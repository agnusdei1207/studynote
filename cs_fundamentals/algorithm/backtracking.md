# 백트래킹 (Backtracking)

## I. 해를 찾는 효율적인 탐색 기법, 백트래킹의 개요

### 가. 백트래킹 (Backtracking)의 정의
- 모든 가능한 경우의 수를 탐색하는 **브루트 포스(Brute Force)** 방식에서, 현재 상태가 해가 될 가능성이 없다고 판단되면 즉시 중단하고 이전 단계로 돌아가 다른 경로를 탐색하는 **제약 조건 만족 문제(Constraint Satisfaction Problem, CSP)** 해결 알고리즘입니다.

### 나. 백트래킹의 등장 배경 및 필요성
1. **탐색 공간의 폭발적 증가**: 단순 전수 조사(Exhaustive Search)는 지수 시간 복잡도($O(2^n)$ 또는 $O(n!)$)를 가져 데이터량이 늘어날 시 수행이 불가능합니다.
2. **효율적 자원 활용**: 무의미한 탐색 비중을 줄여 알고리즘의 평균 수행 시간을 단축할 필요가 있습니다.
3. **최적화 및 결정 문제 해결**: N-Queens, 그래프 색칠하기 등 복잡한 조건이 결합된 문제의 효율적 해결이 요구됩니다.

---

## II. 백트래킹의 핵심 구성요소와 동작 원리

### 가. 핵심 구성요소
| 구성요소 | 설명 | 비고 |
| :--- | :--- | :--- |
| **상태 공간 트리 (State-Space Tree)** | 해를 찾기 위해 탐색해야 할 모든 상태를 트리 형태로 구조화한 모델 | 탐색의 기본 구조 |
| **깊이 우선 탐색 (Depth First Search, DFS)** | 트리의 바닥(Leaf)에 도달할 때까지 깊게 탐색하는 방식 | 백트래킹의 기본 전략 |
| **유망성 판단 (Promising Function)** | 현재 노드가 해의 가능성이 있는지 체크하는 함수 | 조건을 만족하는가? |
| **가지치기 (Pruning)** | 유망하지 않은 노드와 그 하위 트리를 탐색 대상에서 제외하는 기법 | 효율성 향상의 핵심 |

### 나. 백트래킹의 동작 메커니즘
1. **경로 탐색**: DFS를 사용하여 상태 공간 트리를 탐색합니다.
2. **유망성 검사**: 각 노드에서 제약 조건을 만족하는지 확인합니다.
3. **재귀 호출/Backtrack**:
   - **유망함**: 자식 노드로 계속 탐색을 진행합니다.
   - **유망하지 않음**: 해당 노드를 버리고 부모 노드로 돌아가(Backtrack) 다음 자식을 탐색합니다.

---

## III. 백트래킹과 주요 알고리즘 비교

### 가. 백트래킹 vs 브루트 포스 vs 분기한정법 (Branch and Bound)
| 비교 항목 | 브루트 포스 (Brute Force) | 백트래킹 (Backtracking) | 분기한정법 (Branch and Bound) |
| :--- | :--- | :--- | :--- |
| **탐색 방식** | 모든 경로 전수 조사 | DFS 기반 선택적 조사 | BFS 또는 Best-First Search 기반 |
| **최적화 기법** | 없음 | **가지치기 (Pruning)** | **한계치 (Bounding)** 산출 |
| **메모리 사용** | 낮음 | 중간 (재귀 스택) | 높음 (큐/우선순위 큐) |
| **주요 목적** | 단순 해 탐색 | 제약 조건 만족 해 탐색 | 최적해 (Optimization) 탐색 |

### 나. 백트래킹 vs 동적 계획법 (Dynamic Programming, DP)
- **백트래킹**: 주로 해의 형태를 찾는 문제(조합, 순열 등)에 사용하며, 중복 계산이 발생할 수 있으나 가지치기로 극복합니다.
- **DP**: 작은 문제의 해가 큰 문제에 기여하는 '최적 부분 구조'와 '중복 소문제'가 있을 때 사용하며, 메모이제이션(Memoization)으로 중복을 제거합니다.

## IV. 백트래킹의 활용 사례 및 기술사적 판단

### 가. 대표적 활용 사례
1. **N-Queens 문제**: N x N 체스판에 N개의 퀸을 서로 공격할 수 없게 배치하는 문제.
2. **부분 집합의 합 (Subset Sum)**: 원소들의 합이 특정 값이 되는 부분 집합 찾기.
3. **미로 찾기 (Maze)**: 입구에서 출구까지의 유효 경로 탐색.
4. **그래프 색칠 (Graph Coloring)**: 인접한 정점들이 서로 다른 색이 되도록 색칠하는 문제.

### 나. 기술사적 판단 및 향후 전망
- **판단**: 백트래킹은 알고리즘의 시간 복잡도를 혁신적으로 줄이는 기법은 아니나(최악의 경우 동일), **실제 운영 환경에서의 평균 수행 시간을 획기적으로 개선**합니다. 특히 제약 사항이 많은 현실 세계의 스케줄링, 물류 최적화 등에 필수적입니다.
- **전망**: 인공지능(AI)의 탐색(Search) 알고리즘이나 체스/바둑 엔진의 트리 탐색 기법으로 지속 활용되고 있으며, 최근에는 **휴리스틱(Heuristic)** 기법과 결합하여 탐색 효율을 극대화하는 방향으로 발전하고 있습니다.

3. **되돌아가기**: 쥐는 벽 앞에서 가만히 있지 않고, 바로 직전 갈림길로 **휘릭~** 되돌아갑니다. (이게 '백트래킹'이에요)
4. **새로운 길**: 그리고는 안 가본 다른 길로 다시 가기 시작하죠.
