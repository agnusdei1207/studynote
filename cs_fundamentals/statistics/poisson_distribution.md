# 포이송 분포 (Poisson Distribution)

## 1. 개요
### 1.1 정의
**포이송 분포(Poisson Distribution)**는 단위 시간이나 단위 공간 내에서 발생하는 희귀한 사건의 횟수를 확률적으로 모델링하는 이산 확률 분포이다.

### 1.2 핵심 특징
- **희귀성**: 발생 확률이 낮은 사건에 적용된다.
- **단일 매개변수**: 평균($\lambda$)과 분산($\sigma^2$)이 동일하다.
- **이항분포의 극한**: 시행 횟수($n$)가 무한히 크고, 성공 확률($p$)이 0에 가까울 때 유도된다.

## 2. 등장 배경 및 필요성

| 구분 | 내용 |
| :--- | :--- |
| **이항분포의 한계 극복** | $n$이 매우 크고 $p$가 매우 작은 경우, 이항분포 계산($_{n}C_{k}$)의 복잡성 해결 및 근사치 제공 |
| **희귀 사건 모델링** | 시스템 장애, 불량품 발생, 콜센터 전화 수신 등 랜덤하게 발생하는 사건의 빈도 예측 필요 |
| **대기행렬 이론 기초** | 큐잉 이론(Queueing Theory)에서 고객 도착 분포(도착률) 모델링의 기본이 됨 |

## 3. 핵심 원리 및 수식

### 3.1 확률질량함수 (PMF)
단위 구간(시간/공간) 동안 평균 $\lambda$회 발생하는 사건이 $k$회 발생할 확률 $P(X=k)$는 다음과 같다.

$$ P(X=k) = \frac{e^{-\lambda} \lambda^k}{k!} \quad (k = 0, 1, 2, \dots) $$

- **$\lambda$ (Lambda)**: 단위 구간 당 사건의 평균 발생 횟수 (E(X))
- **$e$**: 자연 상수 ($\approx 2.718$)
- **$k$**: 실제 발생한 사건의 횟수

### 3.2 성립 조건 (포이송 과정)
1.  **독립성(Independence)**: 서로 겹치지 않는 구간에서 발생하는 사건의 수는 상호 독립적이다.
2.  **비례성(Proportionality)**: 짧은 구간 동안 사건이 발생할 확률은 그 구간의 길이에 비례한다.
3.  **비집락성(Non-clumping)**: 매우 짧은 구간에서 두 번 이상 사건이 발생할 확률은 무시할 수 있다.

## 4. 이항분포와의 관계 및 비교

이항분포 $B(n, p)$에서 $n \to \infty$, $p \to 0$, $np = \lambda$ (일정)일 때 포이송 분포로 수렴한다.

| 비교 항목 | 이항분포 (Binomial) | 포이송 분포 (Poisson) |
| :--- | :--- | :--- |
| **매개변수** | $n$ (시행 횟수), $p$ (성공 확률) | $\lambda$ (단위 구간 당 평균 발생 수) |
| **적용 대상** | 시행 횟수가 정해져 있고 확률이 명확할 때 | 시행 횟수는 모르지만 평균은 알 때 (희귀 사건) |
| **평균 및 분산** | $E(X)=np$, $V(X)=np(1-p)$ | $E(X)=\lambda$, $V(X)=\lambda$ |

## 5. 컴퓨터 시스템 및 IT 활용 사례

| 활용 분야 | 세부 내용 |
| :--- | :--- |
| **시스템 신뢰성 공학** | 일정 기간 동안 서버 장애, 디스크 오류 발생 횟수 예측을 통한 예비 부품 산정 및 가용성 설계 |
| **네트워크 트래픽 분석** | 단위 시간당 패킷 도착 수, 웹 서버 HTTP 요청 수 모델링 (트래픽 폭주 대비 용량 산정) |
| **보안 (Security)** | 비정상적인 침입 시도 횟수 탐지 (평균 발생 횟수 $\lambda$를 크게 벗어나는 경우 이상 징후로 판단) |
| **고객 서비스 관리** | 콜센터의 시간당 상담 요청 콜 수 예측을 통한 상담원 배치 최적화 (Erlang 모델의 기초) |

## 6. 기술사적 견해 및 고려사항

### 6.1 모델 적용 시 고려사항
- **가정 검증**: 데이터의 평균과 분산이 비슷한지 확인해야 한다. 분산이 평균보다 현저히 큰 경우(과대산포, Overdispersion), **음이항분포(Negative Binomial Distribution)**를 고려해야 한다.
- **지수분포와의 관계**: 포이송 분포가 사건의 '횟수'를 다룬다면, 사건과 사건 사이의 '대기 시간'은 **지수분포(Exponential Distribution)**를 따른다는 점을 이해하여 시스템 성능 분석에 상호 보완적으로 활용해야 한다.

### 6.2 결론
포이송 분포는 IT 인프라의 용량 계획(Capacity Planning)과 성능 분석, 가용성 관리의 수학적 토대를 제공하는 필수적인 도구이다. 현대의 클라우드 컴퓨팅 및 마이크로서비스 아키텍처 환경에서도 오토스케일링(Auto-scaling) 임계치 설정 등에 여전히 유효한 통계적 인사이트를 제공한다.
