# 멀티플렉서와 디멀티플렉서

## 1. 개념

### 1.1 멀티플렉서 (Multiplexer, MUX)

**멀티플렉서(Multiplexer, MUX)**는 다중 입력 신호 중 하나를 선택하여 단일 출력으로 내보내는 **조합 논리 회로**입니다. 일명 **데이터 선택기(Data Selector)**라고도 합니다.

- **N:1 MUX**: N개의 입력 선과 1개의 출력 선, 선택 신호(S)로 구성
- **선택 신호**: $\lceil \log_2 N \rceil$ 개의 선택 선이 필요

### 1.2 디멀티플렉서 (Demultiplexer, DEMUX)

**디멀티플렉서(Demultiplexer, DEMUX)**는 단일 입력 신호를 다중 출력 선 중 하나로 선택하여 전달하는 **조합 논리 회로**입니다. 멀티플렉서와 정반대의 동작을 수행합니다.

- **1:N DEMUX**: 1개의 입력 선과 N개의 출력 선, 선택 신호(S)로 구성
- **선택 신호**: $\lceil \log_2 N \rceil$ 개의 선택 선이 필요

## 2. 등장 배경

### 2.1 역사적 필요성

| 시기 | 배경 |
|-----|------|
| 1950년대 | 전화 교환기에서 다중 통신로 선택 필요 |
| 1960년대 | 초기 컴퓨터에서 버스 시스템의 데이터 전송 효율화 요구 |
| 1970년대 | VLSI 기술 발전으로 집적 회로 내 데이터 경로 최적화 필요 |

### 2.2 기술적 필요성

1. **자원 공유**: 제한된 전송로를 여러 신호가 공유
2. **회로 단순화**: 물리적 배선 감소 및 회로 복잡도 저감
3. **시스템 확장성**: 모듈형 설계를 통한 확장 용이성 확보

## 3. 구성 요소

### 3.1 멀티플렉서 구성

```
        ┌─────────────────────────────────────┐
입력 0 ──┤                                     │
입력 1 ──┤                                     ├─── 출력
...    ──┤          멀티플렉서 (MUX)           │
입력 N-1─┤                                     │
        ┤                                     │
선택 신호─┤             S                     │
        └─────────────────────────────────────┘
```

| 구성 요소 | 설명 |
|----------|------|
| 데이터 입력 (D₀~Dₙ₋₁) | N개의 입력 데이터 선 |
| 선택 신호 (S) | $\lceil \log_2 N \rceil$ 개의 선택 선 |
| 출력 (Y) | 선택된 입력이 나가는 단일 선 |
| 스트로브/인에이블 (E) | 선택적 동작 제어 신호 |

### 3.2 디멀티플렉서 구성

```
        ┌─────────────────────────────────────┐
        │                                     │
입력 ───┤          디멀티플렉서 (DEMUX)        ├──── 출력 0
        │                                     │
선택 신호─┤             S                     ├──── 출력 1
        │                                     │    ...
        └─────────────────────────────────────┘
                                           출력 N-1
```

| 구성 요소 | 설명 |
|----------|------|
| 데이터 입력 (D) | 단일 입력 데이터 선 |
| 선택 신호 (S) | $\lceil \log_2 N \rceil$ 개의 선택 선 |
| 출력 (Y₀~Yₙ₋₁) | N개의 출력 선 중 선택된 하나만 활성화 |

## 4. 핵심 원리

### 4.1 멀티플렉서 동작 원리

**논리 함수 표현**: 4:1 MUX의 경우

$$Y = \overline{S_1}\overline{S_0}D_0 + \overline{S_1}S_0D_1 + S_1\overline{S_0}D_2 + S_1S_0D_3$$

| 선택 신호 S₁S₀ | 선택 입력 | 출력 Y |
|--------------|----------|--------|
| 00 | D₀ | D₀ |
| 01 | D₁ | D₁ |
| 10 | D₂ | D₂ |
| 11 | D₃ | D₃ |

### 4.2 디멀티플렉서 동작 원리

**논리 함수 표현**: 1:4 DEMUX의 경우

$$Y_0 = \overline{S_1}\overline{S_0} \cdot D$$
$$Y_1 = \overline{S_1}S_0 \cdot D$$
$$Y_2 = S_1\overline{S_0} \cdot D$$
$$Y_3 = S_1S_0 \cdot D$$

| 선택 신호 S₁S₀ | 활성화된 출력 |
|--------------|-------------|
| 00 | Y₀ = D, 나머지 = 0 |
| 01 | Y₁ = D, 나머지 = 0 |
| 10 | Y₂ = D, 나머지 = 0 |
| 11 | Y₃ = D, 나머지 = 0 |

### 4.3 회로 구현

**4:1 MUX 구현 (AND-OR 구조)**

```
D0 ───┐
      ├── AND ──┐
S0' ──┘         │
S1' ────────────┤
                ├── OR ─── Y
D1 ───┐         │
      ├── AND ──┘
S0 ───┘
S1' ────────────┘
      ... (D2, D3 유사)
```

## 5. 장단점

### 5.1 멀티플렉서

| 장점 | 단점 |
|-----|------|
| 회로 단순화 및 배선 감소 | 지연 시간 증가 |
| 모든 논리 함수 구현 가능 | 선택 신호 오류 시 데이터 손실 |
| 병렬-직렬 변환 가능 | 복잡도가 선택 선 수에 비례 |
| 시스템 확장 용이 | |

### 5.2 디멀티플렉서

| 장점 | 단점 |
|-----|------|
| 직렬-병렬 변환 가능 | 단일 출력만 활성화로 자원 낭비 |
| 브로드캐스팅 구현 가능 | 선택 신호 오류 시 데이터 오전송 |
| 메모리 디코딩에 활용 | 펄스 왜곡 가능성 |

## 6. 비교

### 6.1 기능적 비교

| 비교 항목 | 멀티플렉서 (MUX) | 디멀티플렉서 (DEMUX) |
|----------|-----------------|-------------------|
| 기본 기능 | 다중 입력 → 단일 출력 | 단일 입력 → 다중 출력 |
| 다른 이름 | 데이터 선택기 | 데이터 분배기 |
| 입력 수 | N개 | 1개 |
| 출력 수 | 1개 | N개 |
| 선택 신호 수 | $\lceil \log_2 N \rceil$ | $\lceil \log_2 N \rceil$ |
| 변환 기능 | 병렬 → 직렬 | 직렬 → 병렬 |

### 6.2 응용 분야 비교

| 응용 분야 | MUX 사용 | DEMUX 사용 |
|----------|----------|-----------|
| 통신 시스템 | 다중 채널 결합 | 채널 분리 |
| 메모리 시스템 | 메모리 선택 | 주소 디코딩 |
| 프로세서 | ALU 입력 선택 | 레지스터 선택 |

### 6.3 디코더와의 관계

```
┌─────────────┐         ┌─────────────┐
│  1:N DEMUX  │  ≡  │   N:1 디코더   │
│  (입력 있음) │         │ (입력을 AND) │
└─────────────┘         └─────────────┘
```

- 디코더: 선택 신호만으로 출력 결정
- DEMUX: 선택 신호 + 입력 데이터로 출력 결정

## 7. 기술사적 판단

### 7.1 현업 활용 분석

**실제 적용 사례:**

| 분야 | 적용 예시 | 기술적 고려사항 |
|-----|----------|---------------|
| CPU 설계 | 레지스터 파일 MUX | 파이프라이닝 지연 최소화 |
| 네트워크 스위치 | 포트 MUX/DEMUX | 대역폭 할당 및 QoS 보장 |
| 메모리 컨트롤러 | 뱅크 선택 DEMUX | 액세스 시간 균일화 |
| 통신 시스템 | TDM MUX/DEMUX | 동기화 및 지터 관리 |

### 7.2 설계 시 고려사항

**1) 성능 vs 면적 트레이드오프**
- 대형 MUX: 단일 단계 구현 → 빠르지만 면적 큼
- 소형 MUX 트리: 면적 절약 but 지연 누적

**2) 전력 소모**
- CMOS 구현 시 출력 변화 없을 때 전력 소모 최소
- 동작 주파수에 따른 동적 전력 고려 필요

**3) 신호 무결성**
- 팬아웃(Fan-out) 고려: 출력이 많은 입력을 드라이브
- 부하 용량(Capacitance)에 따른 드라이버 강도 조정

### 7.3 현대적 구현 동향

**ASIC/FPGA에서의 구현:**

```verilog
// FPGA에서의 4:1 MUX 예시
module mux4_1 (
    input [3:0] data_in,
    input [1:0] sel,
    output reg data_out
);
always @(*) begin
    case(sel)
        2'b00: data_out = data_in[0];
        2'b01: data_out = data_in[1];
        2'b10: data_out = data_in[2];
        2'b11: data_out = data_in[3];
    endcase
end
endmodule
```

**최신 동향:**
- 고속 시리얼 통신에서의 MUX 활용 (PCIe, USB)
- AI 가속기에서의 데이터 경로 MUX
- 저전력 설계를 위한 전력 게이팅 MUX

### 8.1 기술 발전 방향

| 기술 | 발전 내용 |
|-----|----------|
| 광 MUX/DEMUX | 워벌렌스 분할 다중화(WDM) 고도화 |

| 양자 컴퓨팅 | 양자 MUX를 통한 큐비트 제어 |
| 뉴로모픽 | 시냅스 가중치 MUX 구현 |

### 8.2 응용 분야 확대

**1) 차량용 전장 부품**
- ADAS 센서 데이터 융합 MUX
- 차량 통신 게이트웨이 DEMUX

**2) 5G/6G 통신**
- Massive MIMO 안테나 선택 MUX
- 빔포밍을 위한 경로 선택

**3) 엣지 컴퓨팅**
- 센서 데이터 집중 MUX
- 엣지 디바이스 제어 DEMUX

### 8.3 연구 주제

1. **재구성 가능한 MUX**: 런타임 구성 변경 가능한 회로
2. **비휘발성 MUX**: MRAM 기반 상태 유지 MUX
3. **오류 타러런트 MUX**: 방사선 내성을 위한 중복 구조
4. **초저전력 MUX**: 서브스레숄드 동작 회로

## 🧒 어린이를 위한 설명

### 기차역으로 이해하는 MUX와 DEMUX

**멀티플렉서(MUX) = 승객을 한 기차에 태우는 역무원**

🚂 상황: 여러 도시(서울, 부산, 대구, 광주)에서 온 승객들이 각자의 기차에 타 있습니다.

```
서울 기차 ──┐
부산 기차 ──┤            🚃 하나로 합쳐진 기차
대구 기차 ──┼──[MUX]──→ ═════════════════→ 목적지
광주 기차 ──┤
            │
          "어디서 온 사람 탈래?"
```

MUX(역무원)이 **"지금은 서울에서 온 승객 탈 차례!"** 하고 외치면, 서울 기차에 타던 승객들을 하나의 기차에 태워 보냅니다.

**디멀티플렉서(DEMUX) = 도착지에서 승객을 내려주는 역무원**

🏁 상황: 하나의 기차에 다 타고 온 승객들을 각자의 목적지로 안내해야 합니다.

``════════════════ → 하나로 합쳐진 기차
    🚃             │
                    ├──[DEMUX]──→ 서울 가는 승객 🚶
                    ├──[DEMUX]──→ 부산 가는 승객 🚶
                    ├──[DEMUX]──→ 대구 가는 승객 🚶
                    └──[DEMUX]──→ 광주 가는 승객 🚶

                  "서울 가는 사람은 이쪽!"
```

DEMUX(역무원)이 **"서울 가는 사람은 이쪽, 부산 가는 사람은 저쪽!"** 하고 외치면, 승객들을 각자의 목적지로 나누어 보냅니다.

### 더 쉬운 비유: 피자 배달

**MUX = 피자 가게 전화**

- 여러 고객이 동시에 주문을 하지만 📞
- 직원은 하나씩 받아서 주방에 전달합니다 🍕
- **"많은 주문 → 하나의 주방"**

**DEMUX = 피자 배달원**

- 주방에서 나온 피자들은 🛵
- 각 고객의 집으로 배달됩니다 🏠
- **"하나의 주방 → 여러 집으로 배달"**
