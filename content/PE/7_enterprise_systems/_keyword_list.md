# 엔터프라이즈 시스템 및 경영정보/IT 전략 키워드 목록 (심화 확장판)

정보관리기술사, 컴퓨터응용시스템기술사 및 IT 전략/컨설팅 전문가(PE, IT Architect)를 위한 경영정보시스템(MIS), IT 경영전략, 엔터프라이즈 아키텍처, 기업 애플리케이션(ERP, CRM, SCM) 및 통합 시스템 전 영역의 핵심/심화 키워드 800선입니다.

---

## 1. IT 경영 전략 및 거버넌스 (80개)
1. 경영 정보 시스템 (MIS, Management Information System)의 정의
2. IT 거버넌스 (IT Governance) - IT가 기업 전략과 목표를 주도하고 지원하도록 하는 이사회/경영진의 책임/프레임워크
3. IT 거버넌스 5대 도메인 - 전략적 연계, 가치 전달, 위험 관리, 자원 관리, 성과 측정
4. COBIT (Control Objectives for Information and related Technology) - ISACA 주도 IT 거버넌스 통제 프레임워크 최상위 표준
5. IT 투자 성과 평가 지표 - ROI (투자수익률), NPV (순현재가치), IRR (내부수익률), PP (투자회수기간)
6. 총 소유 비용 (TCO, Total Cost of Ownership) - 하드웨어/소프트웨어 구매 비용 외 운영, 유지보수, 교육 등 숨겨진 비용 전체
7. 섀도우 IT (Shadow IT) - IT 부서의 통제를 벗어나 현업 부서가 임의로 도입해 사용하는 SaaS/소프트웨어 (보안 및 거버넌스 리스크)
8. 정보화 전략 계획 (ISP, Information Strategy Planning) - 기업의 중장기 경영 목표 달성을 위한 전사적 IT 마스터플랜 수립
9. ISMP (Information System Master Plan) - 특정 정보시스템 구축 사업을 위한 상세 마스터플랜 (ISP보다 구체적 실행 지침)
10. EA (Enterprise Architecture, 전사적 아키텍처) - 기업의 비즈니스, 데이터, 애플리케이션, 기술 인프라를 체계적으로 구조화한 청사진
11. EA 구성요소 (BA: 비즈니스, DA: 데이터, AA: 애플리케이션, TA: 기술 아키텍처) + SA (보안 아키텍처)
12. 잭맨 프레임워크 (Zachman Framework) - 6x6 매트릭스 (Who, What, Where, When, Why, How vs Planner, Owner, Designer, Builder, Implementer)
13. TOGAF (The Open Group Architecture Framework) - ADM(Architecture Development Method) 기반 국제 표준 EA 프레임워크
14. 범정부 EA 프레임워크 (GEA) - 공공기관 정보화 아키텍처 의무 지침
15. ITA (Information Technology Architecture) - 과거 EA와 동의어로 쓰이던 용어 (법제화 명칭)
16. BPR (Business Process Reengineering) - 마이클 해머, 획기적 성과 향상을 위해 비즈니스 프로세스를 원점에서 근본적으로 재설계
17. AS-IS 분석 (현재 상태) vs TO-BE 모델링 (미래 상태)
18. PI (Process Innovation) - BPR보다 점진적/연속적인 프로세스 혁신
19. 식스 시그마 (Six Sigma) - 통계적 척도를 이용해 불량률을 3.4PPM 이하로 줄이는 품질 혁신 기법 (DMAIC 프로세스)
20. KPI (Key Performance Indicator) - 핵심 성과 지표 (목표 달성 정도 측정)
21. CSF (Critical Success Factor) - 핵심 성공 요인 (목표 달성을 위해 가장 중요한 요소 도출 기법)
22. BSC (Balanced Scorecard, 균형 성과 기록표) - 재무, 고객, 내부 프로세스, 학습과 성장 4가지 관점의 균형 있는 조직 성과 평가 도구
23. 전략 체계도 (Strategy Map) - BSC 관점 간의 인과관계를 시각화한 맵
24. OKR (Objectives and Key Results) - 도전적 목표(O)와 구체적/측정가능한 핵심 결과(KR) 기반의 애자일 성과 관리 기법 (인텔, 구글 방식)
25. MBO (Management by Objectives) - 피터 드러커의 목표 관리 제도의 한계와 OKR의 차이점
26. 가치 사슬 (Value Chain) 분석 - 마이클 포터, 기업의 주활동(본원적 활동)과 지원활동을 분석해 부가가치 창출 원천 파악
27. 주활동 (Inbound Logistics, Operations, Outbound Logistics, Marketing/Sales, Service)
28. 지원활동 (Firm Infrastructure, HR, Tech Development, Procurement)
29. 5 Forces Model (마이클 포터) - 산업 경쟁력 분석 (기존 경쟁자, 신규 진입자, 대체재, 공급자 교섭력, 구매자 교섭력)
30. SWOT 분석 - 강점(Strength), 약점(Weakness), 기회(Opportunity), 위협(Threat) 교차 분석(SO, ST, WO, WT 전략)
31. 3C 분석 - Customer(고객), Company(자사), Competitor(경쟁사)
32. PEST 분석 - 거시 환경 분석 (Political, Economic, Social, Technological) / STEEP 분석
33. 7S 모델 (McKinsey) - Strategy, Structure, Systems, Shared Values, Style, Staff, Skills 조직 진단
34. 포트폴리오 매트릭스 (BCG Matrix) - 시장 점유율과 시장 성장률로 사업부(Cash Cow, Star, Question Mark, Dog) 평가
35. 롱테일 법칙 (Long Tail Theory) - 파레토 법칙(80:20)의 반대, 80%의 비주류 상품이 20%의 주류 상품보다 더 큰 가치를 창출하는 디지털 경제 현상
36. 네트워크 효과 (Network Effect) - 메트칼프의 법칙 (네트워크 가치는 사용자 수의 제곱에 비례)
37. 파괴적 혁신 (Disruptive Innovation) - 크리스텐슨 교수의 하위 시장에서 출발해 주류 시장을 파괴하는 혁신 모델
38. 양손잡이 조직 (Ambidextrous Organization) - 기존 사업의 활용(Exploitation)과 신사업의 탐험(Exploration)을 동시에 수행하는 경영 전략
39. 린 스타트업 (Lean Startup) - 에릭 리스, '만들고-측정하고-학습하는(Build-Measure-Learn)' 피드백 루프 기반의 빠른 검증 전략
40. MVP (Minimum Viable Product, 최소 기능 제품) - 가설 검증을 위한 최소한의 피처만 탑재한 제품 배포
41. 피벗 (Pivot) - 시장 반응 확인 후 사업/제품의 방향성을 전략적으로 전환
42. 그로스 해킹 (Growth Hacking) - 데이터를 기반으로 제품의 사용자 유입, 활성화, 리텐션, 추천, 수익화(AARRR 퍼널)를 최적화하는 마케팅 엔지니어링
43. AARRR 퍼널 (Acquisition, Activation, Retention, Referral, Revenue) 해적 지표
44. 애자일 경영 (Agile Management) / 엔터프라이즈 애자일
45. ITIL (IT Infrastructure Library) - IT 서비스 관리(ITSM)의 글로벌 베스트 프랙티스 프레임워크 (V3 라이프사이클 -> V4 서비스 가치 시스템 SVS)
46. ITSM (IT Service Management) - IT 서비스를 비즈니스 요구에 맞게 제공 및 관리 (SLA 기반 운영)
47. SLA (Service Level Agreement, 서비스 수준 협약서) - 서비스 제공자와 고객 간의 정량화된 서비스 품질 기준 협약
48. SLM (Service Level Management) / OLA (운영 수준 협약) / UC (기초 계약)
49. 서비스 카탈로그 (Service Catalog) - 제공 가능한 IT 서비스 목록 명세서
50. CMDB (Configuration Management Database) - IT 인프라 구성 항목(CI) 자산 및 종속성 관리 데이터베이스
51. 헬프 데스크 (Help Desk) / 서비스 데스크 (Service Desk) - 단일 접점 (SPOC, Single Point of Contact)
52. 인시던트 관리 (Incident Management) - 장애 발생 시 서비스의 신속한 복구 목표 (Workaround 웤어라운드 제공)
53. 문제 관리 (Problem Management) - 인시던트의 근본 원인(Root Cause)을 찾아 영구적 해결책(Known Error) 제시
54. 변경 관리 (Change Management) - 시스템 변경으로 인한 리스크 통제 위원회(CAB)
55. 릴리스 및 배포 관리 (Release and Deployment Management)
56. 비즈니스 연속성 계획 (BCP, Business Continuity Plan) - 재난/재해 시 핵심 업무 기능 유지 지침
57. 재해 복구 시스템 (DR, Disaster Recovery) / BIA (Business Impact Analysis)
58. IT 컴플라이언스 (Compliance) - 사베인스 옥슬리 법(SOX), 바젤 II, ISMS, GDPR 등 규제 준수
59. 정보기술 아웃소싱 (ITO, IT Outsourcing) / BPO (비즈니스 프로세스 아웃소싱)
60. 클라우드 매니지드 서비스 (MSP, Managed Service Provider) - 클라우드 도입/운영 대행 아웃소싱
61. 온프레미스 (On-Premise) - 기업이 자체 전산실에 서버를 직접 구축/운영하는 전통적 방식
62. 코로케이션 (Colocation) - 데이터센터의 공간과 전력만 임대하여 자사 서버 운영
63. 클라우드 락인 (Cloud Vendor Lock-in) 종속 현상과 회피 전략 (멀티 클라우드, 컨테이너 기반 K8s)
64. 클라우드 마이그레이션 전략 (6R: Rehost, Replatform, Refactor, Repurchase, Retire, Retain)
65. 섀도우 데이터 (Shadow Data) - 통제받지 않은 클라우드 내 산재된 기업 민감 데이터 
66. 데이터 거버넌스 (Data Governance) - 데이터 품질, 보안, 프라이버시 전사 관리 체계
67. 데이터 스튜어드 (Data Steward) - 실무 부서 데이터 품질 관리 책임자
68. CDO (Chief Data Officer) / CIO (Chief Information Officer) 역할 분리
69. 메타데이터 (Metadata) 관리 / 데이터 카탈로그 (Data Catalog)
70. 마스터 데이터 관리 (MDM, Master Data Management) - 전사 공통 기준정보 단일화 (Single Source of Truth)
71. 디지털 트랜스포메이션 (DX / DT, Digital Transformation) - AI, 클라우드, 빅데이터로 비즈니스 모델 전면 혁신
72. 플랫폼 비즈니스 (Platform Business) - 양면 시장 (Two-sided Market) 연결 구조
73. 옴니채널 (Omni-Channel) - 오프라인, 온라인, 모바일 등 모든 채널을 통합해 일관된 고객 경험 제공 (O2O의 진화)
74. 데이터 리터러시 (Data Literacy) - 조직원의 데이터 해독 및 활용 능력
75. 애자일 PMO (Project Management Office) - 폭포수 통제를 넘어 애자일 지원 조직으로의 전환
76. 가치 흐름 매핑 (Value Stream Mapping) - Lean 기법, 낭비(Muda) 요소 식별 제거
77. 엔터프라이즈 포털 (EP / EIP) - 기업 내 분산된 정보를 단일 창구로 통합 웹 제공
78. 스마트워크 (Smart Work) / 모바일 오피스 
79. RPA (Robotic Process Automation) - 단순 반복적 사무 프로세스를 소프트웨어 봇이 자동 처리
80. 초자동화 (Hyperautomation) - RPA에 AI, ML, 프로세스 마이닝 등을 결합한 종단간 자동화 트렌드 (가트너)

## 2. 전사적 자원 관리 (ERP) 및 경영 시스템 (60개)
81. ERP (Enterprise Resource Planning) - 기업 내 모든 경영 활동 프로세스(재무, 인사, 생산, 영업 등)를 통합 연동하는 전사적 자원 관리 시스템
82. MRP (Material Requirements Planning) - 자재 소요량 계획 (ERP의 기원 1970년대)
83. MRP II (Manufacturing Resource Planning) - 자재뿐 아니라 설비, 인력 등 생산 자원 전체 포괄 계획 (1980년대)
84. 확장형 ERP (Extended ERP) - 외부 공급망(SCM) 및 고객(CRM)까지 통합 연계 (ERP II)
85. ERP 구축 생명주기 - 패키지 선정 -> 커스터마이징 / CBO (Custom Built Object) -> 데이터 이관 -> 빅뱅/점진적 오픈
86. BPR과 ERP의 관계 - 시스템에 맞춰 업무를 변경할 것인가(BPR 선행), 업무에 맞춰 시스템을 고칠 것인가(커스터마이징)
87. ERP 패키지의 장점 - 베스트 프랙티스(Best Practice) 내재화, 통합 데이터베이스 구축, 유지보수 용이
88. 클라우드 ERP (SaaS ERP) - 2 Tier ERP 구조 (본사는 On-Premise 구축형 무거운 ERP, 지사는 SaaS 형태 가벼운 ERP 혼용)
89. 포스트 모던 ERP (Postmodern ERP) - 거대하고 무거운 단일 모놀리식 벤더 중심에서 벗어나, 코어 ERP와 각 부문별 최적의 SaaS(HR, CRM 등) 솔루션을 느슨하게 결합하는 차세대 전략
90. AI 내재화 ERP (Intelligent ERP) - 머신러닝을 통한 재무 이상 탐지, 수요 예측 자동화 
91. 컴포저블 ERP (Composable ERP) - 비즈니스 블록을 조합하듯 API 중심으로 유연하게 조립/변경 가능한 최신 ERP 아키텍처 
92. SCM (Supply Chain Management, 공급망 관리) - 원자재 조달에서 최종 소비자 전달까지 물류, 정보, 자금의 흐름을 전체 네트워크 관점에서 최적화
93. 불황 효과 / 채찍 효과 (Bullwhip Effect) - 하류(소비자)의 작은 수요 변동이 상류(제조업체)로 갈수록 정보 왜곡으로 증폭되는 현상 (정보 비대칭 원인)
94. 채찍 효과 억제 - SCM 도입을 통한 실시간 정보 공유(POS 데이터 통합) 및 리드 타임 축소
95. SCP (Supply Chain Planning) - 공급망 계획 (수요 예측, 생산 계획)
96. SCE (Supply Chain Execution) - 공급망 실행 (주문 처리, 물류/창고 제어)
97. WMS (Warehouse Management System) - 창고 관리 시스템
98. TMS (Transportation Management System) - 운송 관리 시스템
99. VMI (Vendor Managed Inventory) - 공급자 주도 재고 관리 (월마트 방식, 유통업체 재고를 제조사가 직접 모니터링/보충)
100. 크로스 도킹 (Cross-Docking) - 물류센터 도착 상품을 창고 보관 없이 즉시 배송 차량으로 분류 환적 (창고 비용 제로화)
101. 3PL (Third Party Logistics) / 4PL - 제3자/제4자 물류 아웃소싱 (컨설팅 포함 여부)
102. JIT (Just In Time) - 적기 생산 방식 (도요타 칸반 방식), 재고 제로화 목표
103. S&OP (Sales and Operations Planning) - 영업 및 생산 운영 계획의 주별/월별 전사 통합 조정 회의체
104. APS (Advanced Planning and Scheduling) - 제약 조건 최적화 기반 고급 스케줄링 솔루션 (메모리 상 수학적 시뮬레이션)
105. 디지털 공급망 (Digital Supply Chain) 및 SCM 컨트롤 타워 - 글로벌 물류 가시성 확보 
106. 블록체인 SCM 이력 추적 - 물류 운송 단계별 콜드체인 온도 및 조작 불가 원장 저장 
107. CRM (Customer Relationship Management, 고객 관계 관리) - 신규 고객 획득 및 기존 고객 유지/충성도 극대화를 위한 마케팅, 세일즈, 서비스 통합 전략 
108. LTV (Life Time Value, 고객 생애 가치) - 한 고객이 기업과 거래하는 전 기간 동안 창출하는 총 이익의 현재 가치 (CRM의 핵심 목표)
109. 고객 획득 비용 (CAC, Customer Acquisition Cost) - 신규 고객 1명 유치 비용 (LTV > CAC 필수)
110. 운영 CRM (Operational CRM) - 영업 자동화(SFA), 캠페인 관리, 콜센터 등 프론트 오피스 고객 접점 업무 지원
111. 분석 CRM (Analytical CRM) - 데이터 웨어하우스(DW)/마이닝 기반 고객 세분화(Segmentation), 이탈 예측 분석
112. 협업 CRM (Collaborative CRM) - 다양한 고객 채널(전화, 이메일, 챗봇, 소셜) 간 일관된 커뮤니케이션 지원
113. 소셜 CRM (Social CRM) - 소셜 미디어 상의 고객 반응/여론 청취(Social Listening) 기반 대응
114. AI 기반 CRM - 세일즈포스 아인슈타인 등 (고객 응답 생성, 최적 판매 시점 AI 예측)
115. CDP (Customer Data Platform, 고객 데이터 플랫폼) - 기존 CRM(영업망), DMP(광고 데이터)를 넘어 온라인/오프라인의 자사(1st Party) 고객 행동 로그를 한 곳으로 통합하는 최신 마케팅 플랫폼
116. 서드파티 쿠키리스 (Cookie-less) 시대 대응 퍼스트파티(1st Party) 데이터 확보 전략
117. 마테크 (MarTech / Marketing Technology) - IT와 마케팅의 융합 
118. 프로그래매틱 광고 (Programmatic Advertising) - RTB (실시간 입찰, Real-Time Bidding) 기반 광고 자동 구매 
119. MES (Manufacturing Execution System) - 제조 실행 시스템 (공장 작업 지시, 자재/품질 실시간 추적 제어) (ERP와 공장 설비 사이의 미들웨어 격)
120. POP (Point of Production) - 생산 시점 정보 관리 (현장 단말 바코드 스캔)
121. 스마트 팩토리 (Smart Factory) 4단계 - ICT 미적용 -> 기초 수준(바코드) -> 중간 수준(IoT/실시간) -> 고도화(AI 제어/자율형)
122. PLM (Product Lifecycle Management) - 제품 수명 주기 관리 (기획-설계-생산-폐기) (CAD/CAM/CAE 데이터 연동 BOM 관리)
123. PDM (Product Data Management) - 도면, BOM 등 제품 설계 데이터 관리 (PLM의 전신)
124. BOM (Bill of Materials) - 자재 명세서 (E-BOM 설계 / M-BOM 제조 차이 동기화 이슈)
125. C-Commerce (Collaborative Commerce) - B2B 환경 협업 상거래 설계 도면 공유
126. SCM-ERP-MES 수직적 통합 (Vertical Integration) 아키텍처
127. KMS (Knowledge Management System) - 지식 관리 시스템 (조직의 암묵지를 형식지로 변환 공유)
128. 암묵지 (Tacit Knowledge) - 개인의 경험/노하우 내재된 지식
129. 형식지 (Explicit Knowledge) - 문서/매뉴얼로 겉으로 표출된 지식
130. SECI 모델 (Nonaka 지식 창조 이론) - 이식(Socialization), 표출(Externalization), 연결(Combination), 내면화(Internalization)
131. 그룹웨어 (Groupware) 및 엔터프라이즈 2.0 협업 툴 (Slack, Teams, Confluence)
132. HRIS / e-HR (인적 자원 정보 시스템) 및 인재 관리 (Talent Management)
133. EPM (Enterprise Performance Management) / CPM - 재무 성과, 예산 편성, 예측 통합 관리 모듈
134. SR (Social Responsibility) / ESG 경영 IT 시스템 (환경, 사회, 지배구조 탄소 배출 모니터링)
135. 레그테크 (RegTech, Regulatory Technology) - 금융/보안 컴플라이언스 및 규제 대응을 IT로 자동화 처리 (자금세탁방지 AML 등)
136. 프롭테크 (PropTech) - 부동산 비즈니스와 IT 융합
137. 에듀테크 (EduTech) - 교육 정보 시스템 및 적응형 학습(Adaptive Learning) 통합
138. 디지털 온보딩 (Digital Onboarding) - 신규 고객/직원의 디지털 서비스 초기 적응 자동화 가이드 시스템
139. O2O (Online to Offline) 및 O4O (Online for Offline) 플랫폼 비즈니스
140. 구독 경제 (Subscription Economy) 및 서비스형 플랫폼(XaaS) 비즈니스 모델

## 3. 엔터프라이즈 애플리케이션 아키텍처 (EAI/ESB/MSA) (70개)
141. 애플리케이션 통합 아키텍처 개요
142. 포인트 투 포인트 (Point-to-Point) 통합 - 각 시스템을 1:1로 직접 연결 (N*(N-1)/2 링크 스파게티 구조, 관리 불가)
143. EAI (Enterprise Application Integration) - 전사적 애플리케이션 통합 미들웨어 기반 (Hub & Spoke 모델 중앙 제어)
144. 허브 앤 스포크 (Hub and Spoke) - 중앙 허브를 통해 데이터 포맷 변환 및 라우팅 수행 (허브 장애 시 전면 중단/SPOF 위험)
145. 메시지 브로커 (Message Broker) 기반 동기/비동기 전달
146. 버스 기반 아키텍처 (Bus Architecture) - 공용 메시지 버스를 공유 (EAI의 대안)
147. 데이터 통합 (ETL), 애플리케이션 통합 (API/메시지), 비즈니스 프로세스 통합 (BPM)
148. SOA (Service Oriented Architecture) - 서비스 지향 아키텍처 (2000년대 후반 엔터프라이즈 표준)
149. 서비스 (Service)의 특징 - 비즈니스 의미 단위의 모듈, 독립성, 느슨한 결합(Loosely Coupled), 재사용성, 표준 인터페이스(SOAP/WSDL)
150. SOA 3요소 아키텍처 - 서비스 제공자(Provider), 서비스 요청자(Requester), 서비스 레지스트리(Registry/Broker)
151. UDDI (Universal Description, Discovery, and Integration) - SOA 서비스 레지스트리 표준 규격 디렉터리
152. WSDL (Web Services Description Language) - 서비스의 인터페이스, 위치, 호출 방법(파라미터)을 명세한 XML 문서
153. SOAP (Simple Object Access Protocol) - HTTP 기반 XML 메시지 규약 프로토콜 (무겁고 복잡함의 단점)
154. ESB (Enterprise Service Bus) - SOA 구현을 위한 핵심 백본 미들웨어 (메시지 버스 기반 라우팅, 변환, 보안 추상화 지원)
155. EAI vs ESB - EAI는 벤더 종속적/중앙 집중(Hub), ESB는 개방형 웹 서비스 표준 기반/분산 버스 라우팅 지향
156. REST (Representational State Transfer) - Roy Fielding 제안 아키텍처 스타일, HTTP 메서드(GET, POST, PUT, DELETE)와 URI(자원)를 있는 그대로 활용 (SOAP 대체)
157. RESTful API 성숙도 모델 (Richardson Maturity Model)
158. Level 0 - 단일 URI, 단일 POST 메서드만 사용 (RPC 스타일)
159. Level 1 - 리소스별 고유 URI 할당
160. Level 2 - HTTP 메서드의 적절한 분리 사용 (가장 대중적 단계)
161. Level 3 - HATEOAS (Hypermedia As The Engine Of Application State), 응답에 다음 상태 전이용 하이퍼링크 동적 포함
162. 무상태성 (Statelessness) - REST의 핵심, 서버에 세션 상태를 저장하지 않고 요청 자체만으로 완벽히 문맥 이해
163. 마이크로서비스 아키텍처 (MSA, Microservices Architecture) - 거대한 모놀리식(Monolithic) 앱을 독립적으로 배포 및 확장 가능한 수십~수백 개의 작은 서비스(DB 포함 분리)로 쪼개는 차세대 엔터프라이즈 아키텍처
164. 모놀리식 아키텍처 (Monolithic Architecture) - 프론트, 비즈니스 로직, DB 접근이 한 코드베이스와 단일 DB로 강결합 (빌드 지연, 부분 배포 불가)
165. SOA vs MSA 차이점 - SOA는 전사적 재사용과 ESB 중앙 파이프 집중 (Smart Pipe, Dumb Endpoint), MSA는 독립 배포/자율적 팀 주도, 메시지 큐 비동기 중심 (Dumb Pipe, Smart Endpoint) 결합
166. API 게이트웨이 (API Gateway) - 클라이언트 요청을 단일 진입점으로 받아 적절한 마이크로서비스로 라우팅, 인증/인가, 스로틀링(Throttling), 응답 통합(Aggregation) 담당 모듈
167. BFF (Backend For Frontend) - 모바일, 웹 데스크톱 등 프론트엔드 환경별로 맞춤형 API 게이트웨이를 별도 구축하는 패턴
168. 서비스 디스커버리 (Service Discovery) - 오토스케일링으로 동적 변경되는 컨테이너(Pod)의 IP/Port 위치를 자동 등록하고 조회하는 레지스트리 (Eureka, Consul)
169. 클라이언트 사이드 디스커버리 (Client-side Discovery) vs 서버 사이드 디스커버리 
170. 서킷 브레이커 (Circuit Breaker) 패턴 - 대상 서비스 장애 시 호출을 조기 차단(Open 상태)하여 타임아웃 적체 및 시스템 전체 연쇄 장애(Cascading Failure)를 방지하는 내결함성(Resilience) 패턴
171. 폴백 (Fallback) - 서킷 브레이커 작동 시 에러 대신 기본값(캐시 등)을 반환하는 로직
172. 폴리글랏 퍼시스턴스 (Polyglot Persistence) - 각 마이크로서비스 특성에 맞춰 RDBMS, NoSQL, Graph DB 등을 독립적, 혼합형으로 선택/운영하는 아키텍처
173. 데이터베이스 퍼 서비스 (Database per Service) - 타 마이크로서비스의 DB에 직접 쿼리(조인) 접근 불가, 오직 API로만 통신하도록 강제 캡슐화
174. 분산 트랜잭션 한계 및 2PC (Two-Phase Commit) 배제 이유 - 블로킹 오버헤드
175. 사가 패턴 (Saga Pattern) - MSA 내 긴 트랜잭션 보장, 각 서비스의 로컬 트랜잭션을 연속적으로 실행하고, 중간 실패 시 역순으로 보상 트랜잭션(Compensating Transaction)을 발행해 논리적 롤백 수행 (Eventual Consistency 보장)
176. 코레오그래피 사가 (Choreography Saga) - 중앙 통제 없이 각 서비스가 비동기 이벤트를 발행/구독(Pub/Sub)하며 자율적으로 체인 구성
177. 오케스트레이션 사가 (Orchestration Saga) - 중앙 오케스트레이터(컨트롤러)가 전체 트랜잭션 흐름을 룰 엔진처럼 지휘
178. 트랜잭셔널 아웃박스 (Transactional Outbox) 패턴 - DB 커밋과 메시지 브로커 이벤트 발행의 원자성(Atomicity) 동기화 보장을 위해, 로컬 DB 내 Outbox 테이블에 먼저 기록하고 폴러(Poller)나 CDC 기술로 메시지 전송 보장
179. CQRS (Command Query Responsibility Segregation) 패턴 - MSA의 복잡한 조인 조회 한계 돌파를 위해 상태 변경(Command) 서비스와 조회(Query) 서비스 모델을 물리/논리적으로 분리 분산 운영
180. 이벤트 소싱 (Event Sourcing) - CRUD 상태 최신 덮어쓰기 대신, 상태 변경을 유발한 '이벤트 로그 자체'를 스트림에 불변(Append-Only) 저장하여 완벽한 감사/복구 인프라 구현 (CQRS와 단짝 결합)
181. 서비스 메시 (Service Mesh) - 애플리케이션 비즈니스 코드와 네트워킹/보안 코드를 분리, 인프라 계층(프록시)에서 서비스 간 트래픽 통제 담당 (Istio, Linkerd)
182. 사이드카 패턴 (Sidecar Pattern) - 비즈니스 컨테이너 옆에 프록시 컨테이너(Envoy 등)를 사이드카처럼 부착하여 L7 트래픽 제어 대행
183. 마이크로서비스 샤시 (Microservice Chassis) - 로깅, 메트릭, 분산 추적 등 횡단 관심사 공통 코드 프레임워크 템플릿 기반 부트스트랩 (스프링 부트 스타터)
184. 중앙 집중화된 구성 관리 (Externalized Configuration) - 환경변수 하드코딩 대신 설정 서버(Config Server) 런타임 주입 아키텍처
185. 로그 수집 통합 (Log Aggregation) 아키텍처 - Fluentd -> Elasticsearch 파이프라인
186. 분산 추적 (Distributed Tracing) 인프라 - 다수 서비스를 거치는 API 호출 구간(Span/Trace) 병목 파악 (Zipkin, Jaeger 연동)
187. 스트랭글러 피그 패턴 (Strangler Fig Pattern) - 노후된 모놀리식 엔터프라이즈 시스템을 일시 중단하지 않고, 전면에 API 게이트웨이 라우팅을 조작해 신규 MSA 모듈로 하나씩 점진적 기능 교체(가로채기) 전략
188. 부패 방지 계층 (ACL, Anti-Corruption Layer) 패턴 - 구 레거시 모델과 신규 MSA 도메인 모델 간 개념 오염을 막기 위한 중간 변환 어댑터 계층 배치
189. 멀티 테넌트 (Multi-Tenant) 데이터베이스 아키텍처 - SaaS ERP 구성 시 1개 인스턴스로 논리적 스키마/로우 레벨 식별자를 통한 다수 고객 분리 수용 구조 설계
190. 이벤트 기반 아키텍처 (EDA, Event Driven Architecture) 퍼블리시/서브스크라이브 토폴로지 디커플링망 
191. 컨슈머 그룹 (Consumer Group) 카프카(Kafka) 파티션 분산 부하 분배 아키텍처
192. gRPC (Google Remote Procedure Call) 구조 - HTTP/2 기반 텍스트 JSON 대신 Protocol Buffers 바이너리 직렬화 스트림 통신, 내부 MSA 서비스 간 동기 초고속 통신망 활용
193. 오픈 API 규격 사양 (OpenAPI Specification / Swagger) 명세화 연계망
194. 웹훅 (Webhook) 방식 통합 (역방향 API, 서버가 이벤트 발생 시 클라이언트로 PUSH 호출)
195. B2B 연동망 EAI 프로토콜 변환 - EDI (Electronic Data Interchange), VAN (부가가치통신망) 연동 XML 변환

## 5. 엔터프라이즈 데이터 및 비즈니스 인텔리전스 (BI) (60개)
261. 데이터 웨어하우스 (DW, Data Warehouse) - 전사적 관점의 의사결정을 돕기 위해 여러 시스템의 데이터를 주제별, 통합적, 시계열적, 비휘발성으로 모아둔 데이터베이스 (Bill Inmon 방식)
262. 데이터 마트 (DM, Data Mart) - 특정 부서나 목적(마케팅, 재무)에 특화된 소규모 DW (Ralph Kimball 상향식 방식)
263. ODS (Operational Data Store) - DW로 적재하기 전 운영 시스템의 데이터를 원형 그대로 복사해두는 통합 임시 저장소 (준실시간 리포팅 용도)
264. ETL (Extract, Transform, Load) - 데이터를 소스에서 추출(E)하여 분석 목적에 맞게 변환/정제(T)한 후 DW에 적재(L)하는 배치 프로세스 병목 구조
265. ELT (Extract, Load, Transform) - 데이터를 먼저 DW(클라우드 DW 등)에 적재(L)한 뒤, DW의 강력한 컴퓨팅 파워를 이용해 내부에서 변환(T) 처리하는 현대 클라우드 데이터 파이프라인 표준 
266. 데이터 정제 (Data Cleansing) - 결측치(Missing Value), 이상치(Outlier), 중복 데이터 스크러빙 처리
267. 데이터 프로파일링 (Data Profiling) - 원천 데이터의 통계적 특성, 포맷 오류 등을 사전에 탐색 진단
268. 기준 정보 관리 (MDM, Master Data Management) - 고객, 상품, 조직 등 전사 핵심 식별 데이터의 무결성 동기화 유지 모델 
269. OLTP (On-Line Transaction Processing) - 엔터프라이즈 운영 시스템 실시간 트랜잭션 (빠른 CRUD, 3NF 정규화)
270. OLAP (On-Line Analytical Processing) - 대용량 DW 데이터의 복잡한 다차원 분석 및 리포팅 시스템 (Read 위주, 비정규화)
271. 다차원 모델링 (Dimensional Modeling) 아키텍처 설계
272. 팩트 테이블 (Fact Table) - 분석 대상이 되는 측정 수치 데이터(매출액, 판매량) 및 각 차원 테이블의 외래키(FK) 저장
273. 차원 테이블 (Dimension Table) - 팩트 데이터를 분석하는 관점(시간, 지역, 제품 등 속성) 기준 저장소
274. 스타 스키마 (Star Schema) - 정규화되지 않은 차원 테이블들이 팩트 테이블 하나를 중심으로 방사형 연결 구조 (조인 단순, 조회 속도 극대화, 중복 허용)
275. 스노우플레이크 스키마 (Snowflake Schema) - 차원 테이블들을 제3정규형(3NF)으로 정규화하여 트리/눈송이 구조로 쪼갠 형태 (중복 제거, 조인 복잡도 증가 단점)
276. 서로게이트 키 (Surrogate Key / 인조 키) - 차원 테이블 병합 시 원본 시스템의 비즈니스 PK 충돌 방지를 위해 시스템이 인위적으로 생성하는 정수형 고유 식별자 시퀀스
277. SCD (Slowly Changing Dimension, 서서히 변하는 차원) 모델링 기법 - 고객 주소 등 차원 속성이 변했을 때 이력 관리 방안
278. SCD Type 1 - 덮어쓰기 (과거 이력 삭제)
279. SCD Type 2 - 새로운 로우(Row) 생성 (시작일/종료일/현재여부 플래그 부여, 이력 완벽 보존, 가장 많이 씀)
280. SCD Type 3 - 과거 컬럼을 별도로 추가하여 제한적 이력 보존
281. OLAP 연산 (Drill-down, Roll-up, Slice, Dice, Pivot)
282. 비즈니스 인텔리전스 (BI, Business Intelligence) - 데이터 수집/정제/분석을 통해 기업의 합리적 의사결정을 돕는 기술 체계 (Tableau, Power BI)
283. 데이터 시각화 (Data Visualization) 대시보드 및 리포트 모델링 
284. 데이터 마이닝 (Data Mining) 연관 분석, 분류, 군집 탐색 모형 (CRISP-DM 등 프로세스)
285. 데이터 레이크 (Data Lake) - 스키마를 미리 정의하지 않고 원시(Raw) 상태의 비정형, 반정형 데이터를 무한히 쌓아두는 중앙 저장소 (하둡, 클라우드 스토리지 등)
286. 스키마 온 리드 (Schema on Read) - 데이터 레이크 방식. 데이터를 읽어 들여 분석하는 시점에 스키마(구조)를 동적 적용
287. 스키마 온 라이트 (Schema on Write) - RDB/DW 방식. 데이터를 적재하는 시점에 사전에 정의된 엄격한 테이블 구조에 맞춰 삽입
288. 데이터 레이크의 한계: 데이터 늪 (Data Swamp) 현상 - 메타데이터 카탈로그 관리 부재로 방치된 쓰레기 데이터 적체 현상
289. 데이터 레이크하우스 (Data Lakehouse) - 데이터 레이크(비정형 원시 데이터 무한 저장)와 데이터 웨어하우스(ACID 트랜잭션, 고속 SQL 쿼리)의 장점을 결합한 최신 데이터 아키텍처 (Databricks, Delta Lake 구조)
290. 데이터 카탈로그 (Data Catalog) 시스템 - 데이터 자산 검색성 확보를 위한 통합 메타데이터 저장소 및 태깅 시스템 (AWS Glue 등)
291. 데이터 리니지 (Data Lineage) - 특정 데이터의 기원(Source)부터 변환 과정(ETL), 최종 타겟까지의 흐름 맵핑 가시성 확보 기술 (변경 영향도, 규제 감사 대응 핵심)
292. 클라우드 네이티브 데이터 웨어하우스 (Snowflake, BigQuery, Redshift) 아키텍처 특징 
293. 스토리지와 컴퓨팅의 분리 (Separation of Compute and Storage) 아키텍처 - 데이터 저장소는 무한 확장(S3 등)하고 연산 클러스터는 필요할 때만 기동/확장하여 자원 비용 극대화 절감
294. 제로 카피 클론 (Zero-Copy Cloning) - 디스크 물리적 복제 없이 메타데이터 스냅샷만 분리하여 즉각적인 분석용 독립 DB 클러스터 셋업 기술
295. 데이터 메시 (Data Mesh) - 데이터 관리를 중앙 집중형(사일로) 방식에서 벗어나, 도메인 주도(Domain-Driven) 분산 오너십 체계로 나누고 각 부서가 '데이터 프로덕트'를 생산/제공하는 조직 및 아키텍처 전환 패러다임
296. 데이터 패브릭 (Data Fabric) - 물리적으로 흩어져 있는 데이터 사일로들을 이동(ETL) 없이 논리적으로 가상화 통합 연결하고 AI로 메타데이터를 매핑/관리하는 지능형 통제 아키텍처 레이어 
297. 데이터 가상화 (Data Virtualization) 연방 쿼리 엔진 적용 체제망
298. 빅데이터 분산 처리 프레임워크 - 맵리듀스(MapReduce, 디스크 기반 병목) -> 스파크 (Apache Spark, 인메모리 고속 병렬 처리)
299. 스파크 RDD (Resilient Distributed Dataset) 탄력적 분산 데이터셋 불변성 메모리 구조 기반 지연 평가(Lazy Evaluation) 기술
300. 실시간 데이터 스트리밍(Apache Kafka) 파이프라인 아키텍처 기반 CDC (변경 데이터 캡처) DB 동기망
301. 카프카 토픽(Topic) 파티셔닝(Partition) 기반 컨슈머 그룹 부하 분산 스케일 아웃 매커니즘 
302. 데이터옵스 (DataOps) - 데이터 파이프라인 품질, 테스팅, 배포(버전 관리)를 CI/CD 체계로 자동화 적용(dbt 등 프레임워크 도입)
303. MLOps 피처 스토어(Feature Store) 데이터마트 연동망 분산 오프라인 실시간 피처 캐시 구조 
304. 실시간 커스터머 데이터 플랫폼 (CDP) 아키텍처 - 자사몰 1st Party 실시간 클릭 로그 수집 이벤트 큐 통계 리포트 통합 엔진
