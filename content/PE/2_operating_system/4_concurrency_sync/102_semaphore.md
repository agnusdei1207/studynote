# 세마포어 (Semaphore)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 세마포어는 **"정수 **카운터**와 **Wait**(P), **Signal**(V) **연산**으로 **동기화**하는 **도구"**로, **Dijkstra**가 **제안**했으며 **Counting Semaphore**(0 이상 정수)와 **Binary Semaphore**(0 또는 1)로 **구분**된다.
> 2. **Wait(P) 연산**: **값**을 **감소**시키고 **0보다 **작으면 **Blocking**(대기)하며 **Signal(V) 연산**은 **값**을 **증가**시키고 **대기 중인 **프로세스**를 **깨운다**(Wakeup).
> 3. **융합**: **Producer-Consumer 문제**, **Reader-Writer 문제**, **Dining Philosophers 문제**를 **해결**하며 **POSIX sem_t**, **System V semid**, **Java Semaphore**, **Python threading.Semaphore**가 **구현**을 **제공**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
세마포어는 **"카운터 기반 동기화 도구"**이다.

**세마포어 특징**:
- **정수 카운터**: 리소스 수
- **Wait(P)**: 획득 (감소)
- **Signal(V)**: 해제 (증가)
- **Blocking**: 카운터 0이면 대기

### 💡 비유
세마포어는 **"주차장 표시기****와 같다.
- **주차 공간**: 리소스
- **표시기**: 카운터
- **진입**: Wait
- **나감**: Signal

## Ⅱ. 아키텍처 및 핵심 원리

### Semaphore 연산

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Semaphore Operations                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Wait (P) Operation:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  P(s):                                                                                  │  │
    │      s.value--;                                                                         │  │
    │      if (s.value < 0) {                                                                 │  │
    │          block(process);  // 대기 큐에 추가                                              │  │
    │      }                                                                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Signal (V) Operation:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  V(s):                                                                                  │  │
    │      s.value++;                                                                         │  │
    │      if (s.value <= 0) {                                                                │  │
    │          wakeup(process);  // 대기 큐에서 제거                                             │  │
    │      }                                                                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Producer-Consumer 예시

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Producer-Consumer with Semaphore                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Shared:                                                                                │  │
    │  • Buffer[10]           // 버퍼                                                        │  │
    │  │  • mutex = 1          // 상호 배제                                                   │  │
    │  │  • empty = 10         // 빈 슬롯 수                                                 │  │
    │  │  • full = 0           // 찬 슬롯 수                                                 │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  Producer:                                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  wait(empty);   // 빈 슬롯 확인                                                      │  │  │
    │  │  wait(mutex);   // 버퍼 잠금                                                        │  │  │
    │  │  buffer[in] = item;                                                                  │  │  │
    │  │  in = (in + 1) % 10;                                                                  │  │  │
    │  │  signal(mutex);  // 버퍼 해제                                                        │  │  │
    │  │  signal(full);   // 찬 슬롯 증가                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  Consumer:                                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  wait(full);    // 찬 슬롯 확인                                                      │  │  │
    │  │  wait(mutex);   // 버퍼 잠금                                                        │  │  │
    │  │  item = buffer[out];                                                                 │  │  │
    │  │  out = (out + 1) % 10;                                                                 │  │  │
    │  │  signal(mutex);  // 버퍼 해제                                                        │  │  │
    │  │  signal(empty);  // 빈 슬롯 증가                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Semaphore vs Mutex

| 구분 | Semaphore | Mutex |
|------|-----------|-------|
| **값** | 0 이상 정수 | 0 또는 1 |
| **소유권** | 없음 | 있음 |
| **용도** | 리소스 카운팅 | 상호 배제 |
| **Signal** | 누구나 | 소유자만 |

### Semaphore 종류

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Semaphore Types                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Binary Semaphore (value: 0 or 1):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Mutex와 유사                                                                        │  │
    │  │  • 소유권 없음                                                                       │  │
    │  │  • 신호 동기화에 사용                                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Counting Semaphore (value: 0 to N):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 리소스 수 제한                                                                       │  │
    │  │  • N개 리소스 관리                                                                    │  │
    │  │  • Producer-Consumer에 사용                                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: DB Connection Pool
**상황**: 연결 제한
**판단**: Semaphore로 제어

```c
#include <semaphore.h>

#define MAX_CONN 10

sem_t conn_sem;

void init_pool() {
    sem_init(&conn_sem, 0, MAX_CONN);  // 10개 연결 가능
}

void* query(void* arg) {
    sem_wait(&conn_sem);    // 연결 획득
    // DB 쿼리 실행
    sem_post(&conn_sem);    // 연결 해제
    return NULL;
}
```

## Ⅴ. 기대효과 및 결론

### Semaphore 기대 효과

| 효과 | 없음 | Semaphore |
|------|------|-----------|
| **리소스 관리** | 수동 | 자동 |
| **경쟁 조건** | 발생 | 방지 |
| **복잡도** | 낮음 | 중간 |

### 모범 사례

1. **초기화**: 올바른 값
2. **해제**: 항상 Signal
3. **Deadlock**: 주의 필요
4. **Priority Inversion**: 우선순위

### 미래 전망

1. **Lock-free**: Atomic 연산
2. **Actor Model**: 메시지 전달
3. **Coroutine**: 비동기

### ※ 참고 표준/가이드
- **POSIX**: sem_wait, sem_post
- **System V**: semop, semctl
