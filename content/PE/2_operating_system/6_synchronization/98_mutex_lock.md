# 뮤텍스 (Mutex Lock)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 뮤텍스는 **"상호 배제**(Mutual Exclusion)를 위한 **잠금(Lock) 메커니즘"**으로, **한 번에 하나의 스레드**만 **Critical Section**에 **진입**할 수 있게 하고 **잠금 획득**(acquire/lock)과 **잠금 해제**(release/unlock) 연산으로 **동기화**를 구현한다.
> 2. **가치**: **Race Condition**을 **방지**하고 **공유 자원**을 **보호**하며 **Kernel-level Mutex**는 **Blocking** 방식으로 **CPU 효율**을 높이고 **User-level Mutex**는 **Spinlock**으로 **빠른 잠금**을 제공한다.
> 3. **융합**: **POSIX pthread_mutex**, **Windows CRITICAL_SECTION**, **Java synchronized**, **std::mutex**(C++)로 **프로그래밍** 가능하며 **Deadlock**, **Priority Inversion** 문제를 **우선 순위 상속**(Priority Inheritance)으로 해결한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
뮤텍스는 **"공유 자원에 대한 접근을 제어하는 잠금 도구"**이다.

**뮤텍스의 특징**:
- **Binary**: 0(잠금) 또는 1(해제)
- **Ownership**: 잠금 획득한 스레드만 해제 가능
- **Mutual Exclusion**: 동시 접근 방지

### 💡 비유
뮤텍스는 **"화장실 키"**와 같다.
- **키**: 잠금(1) / 없음(0)
- **사용**: 키 획득 → 사용 → 키 반납
- **대기**: 키 없으면 기다림

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         뮤텍스의 필요성                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Race Condition:
    • 여러 스레드가 공유 변수 접근
    • 결과 예측 불가능
         ↓
Software Solution:
    • Peterson's Algorithm
    • Dekker's Algorithm
    • Busy Waiting (CPU 낭비)
         ↓
Hardware Support:
    • Test-and-Set
    • Compare-and-Swap (CAS)
         ↓
Mutex Lock:
    • Kernel/Atomic Operation
    • Blocking/Spinlock
```

## Ⅱ. 아키텍처 및 핵심 원리

### Mutex 동작 원리

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Mutex Lock 동작                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [잠금 획득 (Lock/Acquire)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Thread 1                              Thread 2                                        │
    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  mutex.lock();                                                                      │  │
    │  │  // [잠금 획득]                                                                      │  │
    │  │  // Critical Section                                                               │  │
    │  │  counter++;                                                                         │  │
    │  │  mutex.unlock();                                                                    │  │
    │  │  // [잠금 해제]                                                                     │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    │                                       ↓                                               │
    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  mutex.lock();              // [대기 시작]                                          │  │
    │  │  │ Thread 1이 해제할 때까지 Block                                                  │  │
    │  │  │ ┌────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │ │ wait_queue.add(Thread 2);                                                   │  │
    │  │  │ │ thread.state = BLOCKED;                                                     │  │
    │  │  │ │ schedule();  // Context Switch                                               │  │
    │  │  │ └────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │  // [Thread 1 해제 후 깨어남]                                                      │  │
    │  │  // Critical Section                                                               │
    │  │  mutex.unlock();                                                                   │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Spinlock vs Mutex

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Spinlock vs Blocking Mutex                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Spinlock]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Busy Waiting (CPU 점유)                                                              │
    │  │  • 짧은 Critical Section에 적합                                                       │
    │  │  • Context Switch 오버헤드 없음                                                        │
    │  │  • 단점: 단일 코어에서는 비효율적                                                       │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  void spinlock_acquire(spinlock_t *lock) {                                         │  │
    │  │      while (test_and_set(&lock->state) == LOCKED) {                                │  │
    │  │          // Spin (CPU 계속 실행)                                                      │  │
    │  │      }                                                                                  │  │
    │  │  }                                                                                      │  │
    │  │                                                                                         │  │
    │  │  void spinlock_release(spinlock_t *lock) {                                          │  │
    │  │      lock->state = UNLOCKED;                                                         │  │
    │  │  }                                                                                      │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  사용처: 인터럽트 핸들러, 커널 내부 짧은 잠금                                              │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Blocking Mutex]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Sleep/Wake (Block)                                                                    │
    │  │  • 긴 Critical Section에 적합                                                          │
    │  │  • Context Switch 발생                                                                │
    │  │  • 장점: CPU 효율 높음                                                                │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  void mutex_acquire(mutex_t *mutex) {                                              │  │
    │  │      if (test_and_set(&mutex->state) == LOCKED) {                                  │  │
    │  │          current->state = BLOCKED;                                                   │  │
    │  │          enqueue(&mutex->wait_queue, current);                                       │  │
    │  │          schedule();  // 다른 스레드로 교체                                           │  │
    │  │      }                                                                                  │  │
    │  │  }                                                                                      │
    │  │                                                                                         │  │
    │  │  void mutex_release(mutex_t *mutex) {                                                │  │
    │  │      if (!queue_empty(&mutex->wait_queue)) {                                          │  │
    │  │          thread = dequeue(&mutex->wait_queue);                                       │  │
    │  │          thread->state = READY;                                                       │  │
    │  │      }                                                                                  │  │
    │  │      mutex->state = UNLOCKED;                                                         │  │
    │  │  }                                                                                      │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  사용처: 일반적인 스레드 동기화                                                           │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Mutex 구현 (Atomic Operation)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Compare-and-Swap (CAS)                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Hardware Atomic Operation                                                            │
    │  │  • 단일 명령어로 읽기-비교-쓰기 수행                                                    │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  bool compare_and_swap(int *ptr, int old, int new) {                                │  │
    │  │      // Atomic: 읽기, 비교, 쓰기가 인터럽트되지 않음                                    │  │
    │  │      if (*ptr == old) {                                                              │  │
    │  │          *ptr = new;                                                                  │  │
    │  │          return true;  // 성공                                                        │  │
    │  │      } else {                                                                         │  │
    │  │          return false; // 실패                                                        │  │
    │  │      }                                                                                  │  │
    │  │  }                                                                                      │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  [CAS 기반 Lock]                                                                         │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  volatile int lock = 0;  // 0: unlocked, 1: locked                                    │  │
    │  │                                                                                       │  │
    │  │  void acquire() {                                                                    │  │
    │  │      while (!compare_and_swap(&lock, 0, 1)) {                                        │  │
    │  │          // Spin (0이면 1로 변경, 1이면 실패 후 재시도)                                │  │
    │  │      }                                                                                  │  │
    │  │  }                                                                                      │  │
    │  │                                                                                       │  │
    │  │  void release() {                                                                    │  │
    │  │      lock = 0;                                                                        │  │
    │  │  }                                                                                      │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Mutex vs Semaphore vs Monitor

| 구분 | Mutex | Semaphore | Monitor |
|------|-------|-----------|---------|
| **값** | 0/1 | 0~N | Condition |
| **소유자** | 있음 | 없음 | Lock |
| **해제** | 소유자만 | 누구나 | Signal |
| **용도** | 상호 배제 | 자원 카운팅 | 복잡한 동기화 |

### Mutex 문제점

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Mutex 문제점과 해결                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Deadlock]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 서로 다른 잠금을 획득하려고 대기                                                         │
    │  │  • 원인: 순환 대기                                                                    │
    │  │  • 해결: 잠금 획득 순서 통일                                                           │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Thread 1                     Thread 2                                             │  │
    │  │  lock(A);                    lock(B);                                              │  │
    │  │  // ...                      // ...                                                 │  │
    │  │  lock(B);  <───────────X    lock(A);                                              │  │
    │  │  (대기)                      (대기)                                                  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Priority Inversion]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 낮은 우선순위 스레드가 잠금 보유 → 높은 우선순위 스레드 대기                                 │
    │  │  • 중간 우선순위 스레드가 실행 → 낮은 우선순위 스레드 선점 불가 → 교착                           │
    │  │  • 해결: Priority Inheritance Protocol                                               │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  High Priority Thread                                                              │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  lock.acquire()  ──X (대기)                                                      │  │  │
    │  │  │                          │                                                      │  │  │
    │  │  │                  Low Priority Thread (잠금 보유)                                 │  │  │
    │  │  │                      │                                                         │  │  │
    │  │  │              Medium Priority Thread (선점)                                      │  │  │
    │  │  │                      → 교착 상태!                                                │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                       │  │
    │  │  [Priority Inheritance 해결]                                                         │  │
    │  │  Low Thread의 우선순위 → High Thread의 우선순위로 일시 상향                              │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: POSIX Mutex 사용
**상황**: 스레드 안전한 카운터
**판단**:

```c
#include <pthread.h>
#include <stdio.h>
#include <stdlib.h>

typedef struct {
    int counter;
    pthread_mutex_t lock;
} safe_counter_t;

void* increment(void *arg) {
    safe_counter_t *ctr = (safe_counter_t*)arg;

    for (int i = 0; i < 100000; i++) {
        pthread_mutex_lock(&ctr->lock);
        ctr->counter++;
        pthread_mutex_unlock(&ctr->lock);
    }
    return NULL;
}

int main() {
    safe_counter_t counter = {0, PTHREAD_MUTEX_INITIALIZER};
    pthread_t t1, t2;

    pthread_create(&t1, NULL, increment, &counter);
    pthread_create(&t2, NULL, increment, &counter);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    printf("Counter: %d\n", counter.counter);  // 200000
    pthread_mutex_destroy(&counter.lock);
    return 0;
}
```

## Ⅴ. 기대효과 및 결론

### Mutex 기대 효과

| 효과 | Mutex 없을 시 | Mutex 있을 시 |
|------|-------------|-------------|
| **Race Condition** | 발생 | 방지 |
| **데이터 무결성** | 깨짐 | 보장 |
| **CPU 효율** | - | Blocking 최적화 |

### 미래 전망

1. **Futex**: Linux Fast Userspace Mutex
2. **RCU**: Read-Copy-Update
3. **Lock-free**: CAS 기반

### ※ 참고 표준/가이드
- **POSIX**: pthread_mutex
- **Linux**: futex(7)
- **C++**: std::mutex
