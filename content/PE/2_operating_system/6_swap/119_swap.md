# 스왑 공간 (Swap Space)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스왑은 **"물리 **메모리**(RAM)**가 **부족**할 때 **디스크**를 **가상 **메모리**로 **사용**하는 **기법\\\"**으로, **Inactive **페이지**를 **디스크**(Swap **Partition, **Swap **File)**로 **이동**시키고 **필요**할 때 **다시 **RAM**으로 **가져온다.
> 2. **교체 정책**: **LRU**(Least **Recently **Used)**, **Clock**, **ARC**(Adaptive **Replacement **Cache)**가 **교체 **대상 **페이지**를 **선택**하고 **Swappiness**값(vm.**swappiness)**으로 **스왑 **사용 **빈도**를 **제어**한다.
> 3. **Thrashing**: **Working **Set**이 **물리 **메모리**보다 **클 **때 **발생**하고 **Disk **I/O**로 **병목**이 **생기**며 **해결**은 **메모리 **증설**, **OOM **Killer**, **프로세스 **종료**로 **한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스왑은 **"디스크를 메모리처럼 사용"**이다.

**스왑 유형**:
| 유형 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **Partition** | 독립 파티션 | 빠름 | 크기 고정 |
| **File** | 일반 파일 | 유연함 | 오버헤드 |
| **zram** | 압축 RAM | 매우 빠름 | CPU 사용 |

### 💡 비유
스왑은 ****책장 **책 ****과 같다.
- **책상**: RAM
- **서재**: Swap
- **이동**: Page In/Out

## Ⅱ. 아키텍처 및 핵심 원리

### 스왑 메커니즘

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Swap Mechanism                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Memory Layout:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Physical Memory (RAM)                         Swap Space (Disk)                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Used Pages (Hot)                   Free/Inactive Pages                             │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  Process A Pages (Active)                                                          │  │  │  │  │
    │  │  │  Process B Pages (Active)                                                          │  │  │  │  │
    │  │  │  Page Cache (File data)      │  Swapped Out Pages                                 │  │  │  │  │
    │  │  │  ─────────────────────────────────────────────────────────────────────────────────  │  │  │  │  │
    │  │  │  Free RAM: 2GB (low!)                                                               │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
    │            │ Low memory condition                                                         │  │
    │            ▼                                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  kswapd (Kernel Swap Daemon)                                                          │  │  │
    │  │  1. Scan for inactive pages (LRU list)                                                 │  │  │
    │  │  2. Select victim: Page Cache > Anonymous Memory                                       │  │  │
    │  │  3. Write selected pages to swap partition                                             │  │  │
    │  │  4. Update page table: Mark as "not present"                                           │  │  │
    │  │  5. Free RAM pages                                                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Page Fault & Swap In

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Page In (Swap In) Process                                           │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Scenario: Process accesses swapped-out page
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU executes: MOV REG, [Virtual Address 0x7fff_0000_1000]                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  MMU translates Virtual → Physical                                                     │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  Page Table Entry:                                                                  │  │  │  │  │
    │  │  │  • Present: 0 (not in RAM!)                                                         │  │  │  │  │
    │  │  │  • Swap Type: 1 (on swap partition)                                                 │  │  │  │  │
    │  │  │  • Swap Offset: 12345                                                              │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  │                                                                                   │  │  │
    │  │  ▼ MMU raises: Page Fault Exception                                                    │  │  │
    │  │  │                                                                                   │  │  │
    │  │  ▼ OS Page Fault Handler                                                               │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  1. Check if page is on swap (yes)                                                 │  │  │  │  │
    │  │  │  2. Allocate free RAM page                                                         │  │  │  │  │
    │  │  │  3. Issue disk I/O: Read from swap offset 12345                                    │  │  │  │  │
    │  │  │  4. Block process until I/O completes (~10ms)                                      │  │  │  │  │
    │  │  │  5. Update page table: Present=1, Physical Address=new page                        │  │  │  │  │
    │  │  │  6. Retry instruction                                                               │  │  │  │  │
    │  │  │  → Penalty: ~10ms (vs ~100ns for RAM access)                                       │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Swappiness

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         vm.swappiness Parameter                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Scale: 0-100
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  │  Value  │  Description                                                                  │  │
    │  │  ─────────────────────────────────────────────────────────────────────────────────────│  │
    │  │  0      │  Disable swap (only swap to avoid OOM)                                       │  │
    │  │  1-10   │  Minimal swapping (server default)                                           │  │
    │  │  60     │  Linux default (balance)                                                     │  │
    │  │  100    │  Aggressive swapping (swap anonymous pages too)                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Calculation (simplified):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  swap_tendency = mapped_file_pages * (swappiness / 100)                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Example: swappiness = 60, mapped_file_pages = 1000                                   │  │  │
    │  │  swap_tendency = 1000 * 0.6 = 600                                                     │  │  │
    │  │                                                                                       │  │  │
    │  │  If anonymous_pages > swap_tendency:                                                  │  │  │
    │  │      → Swap out anonymous pages                                                       │  │  │
    │  │  Else:                                                                                 │  │  │
    │  │      → Reclaim page cache                                                             │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Recommendation:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Database server: swappiness = 1 (keep data in RAM, avoid swap latency)                 │  │
    │  • Desktop: swappiness = 60 (default, reasonable balance)                                │  │
    │  • Embedded: swappiness = 0 (no swap partition)                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 스왑 교체 알고리즘 비교

| 알고리즘 | 대상 선택 | 복잡도 | 오버헤드 | 사용처 |
|----------|-----------|--------|----------|--------|

### Thrashing

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Thrashing (쓰레싱)                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Definition: System spends more time paging than executing
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  CPU Utilization: 100%                                                               │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  • 80% in kernel (paging I/O)                                                    │  │  │  │  │
    │  │  │  • 20% in user space (actual work)                                                │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  → System is "busy" but doing no useful work                                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Working Set Model:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Working Set W(t, Δ): Set of pages accessed in last Δ time units                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  If |W(t, Δ)| > Physical Memory:                                                      │  │  │
    │  │     → Thrashing occurs                                                                │  │  │
    │  │     → Solution: Reduce number of processes or increase RAM                           │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### zram (Compressed RAM Swap)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         zram (Compressed RAM Swap)                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Architecture:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  RAM (8GB)                                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Normal Memory (6GB)                    zram Device (2GB)                             │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  Active pages                       Compressed swap (ratio ~2:1)                     │  │  │  │  │
    │  │  │  • App data                        │  • 2GB zram = ~4GB uncompressed               │  │  │  │  │
    │  │  │  • File cache                      │  • Latency: ~1ms (vs 10ms disk)               │  │  │  │  │
    │  │  │  • Kernel                          │  • CPU overhead: compression                   │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Configuration:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Create 2GB zram device                                                                │  │
    │  modprobe zram                                                                           │  │
    │  echo 2G > /sys/block/zram0/disksize                                                     │  │
    │  mkswap /dev/zram0                                                                       │  │
    │  swapon /dev/zram0                                                                       │  │
    │  → Swap on compressed RAM (much faster than disk)                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 웹 서버 스왑 튜닝
**상황**: 8GB RAM, 4GB Swap, 높은 Swap 사용
**판단**: Swappiness 감소 + zram 추가

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Web Server Swap Tuning                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Problem:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  $ free -h                                                                                │  │
    │  │               total        used        free      shared  buff/cache   available        │  │
    │  │  Mem:           7.6G        6.5G        200M        100M        900M        600M        │  │
    │  │  Swap:          4.0G        2.5G        1.5G                                                     │  │
    │  │                                                                                         │  │
    │  │  → High swap usage (62%)                                                                 │  │
    │  │  → Latency spikes during page faults                                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Solution 1: Reduce Swappiness
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Check current value                                                                    │  │
    │  $ cat /proc/sys/vm/swappiness                                                            │  │
    │  60                                                                                       │  │
    │                                                                                         │  │
    │  # Set to 1 (only swap if critical)                                                       │  │
    │  $ sysctl vm.swappiness=1                                                                 │  │
    │  $ echo "vm.swappiness=1" >> /etc/sysctl.conf                                            │  │
    │  → OS prefers reclaiming page cache over swapping                                         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Solution 2: Add zram for Fast Swap
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Install zram                                                                          │  │
    │  $ apt install zram-config                                                                │  │
    │                                                                                         │  │
    │  # Configure 1GB zram per CPU core                                                       │  │
    │  $ echo 1024 > /sys/block/zram0/disksize                                                 │  │
    │                                                                                         │  │
    │  # After tuning:                                                                         │  │
    │  $ free -h                                                                                │  │
    │  │               total        used        free      shared  buff/cache   available        │  │
    │  │  Mem:           7.6G        6.0G        500M        100M        1.1G        1.2G        │  │
    │  │  Swap:          1.0G        100M        900M (zram)                                         │  │
    │  │  Swap:          4.0G          0B        4.0G (disk)                                          │  │
    │  → Most swap in RAM (zram), disk swap unused                                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Solution 3: Reduce Memory Usage
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Reduce worker processes (reduce memory footprint)                                      │  │
    │  $ vi /etc/nginx/nginx.conf                                                              │  │
    │  worker_processes auto;  # 8 (was 16)                                                    │  │
    │                                                                                         │  │
    │  # Enable response caching (reduce backend queries)                                      │  │
    │  proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=1g;       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 스왑 기대 � 效果

| 구성 | 페이지 폴트 지연시간 | 처리량 |
|------|-------------------|--------|
| **No Swap** | N/A | OOM 위험 |
| **Disk Swap** | ~10ms | 낮음 |
| **SSD Swap** | ~1ms | 중간 |
| **zram** | ~0.5ms | 높음 |

### 모범 사례

1. **서버**: swappiness=1
2. **데스크톱**: swappiness=60
3. **zram**: 임베디드
4. **모니터링**: vmstat, sar

### 미래 전망

1. **Persistent Memory**: Intel Optane
2. **Swap-less**: 충분한 RAM
3. **Compressed**: zRAM
4. **Tiered**: NVM + SSD

- **Linux**: Swap Management
- **Red Hat**: Performance Tuning
