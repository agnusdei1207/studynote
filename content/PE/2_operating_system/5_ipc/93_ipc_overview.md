# IPC (Inter-Process Communication)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPC는 **"프로세스 간 **데이터 교환과 **동기화를 위한 **메커니즘"**으로, **Shared Memory**(데이터 공유), **Message Passing**(메시지 전달), **Synchronization**(동기화) 방식이 있으며 **Kernel-space IPC**(Pipe, Socket)과 **User-space IPC**(Shared Memory)로 **분류**된다.
> 2. **가치**: **독립적인 프로세스** 간 **협력**과 **데이터 교환**을 **가능**하게 하고 **Microkernel Architecture**에서 **서버**와 **클라이언트** 분리를 **지원**하며 **Distributed System**의 **기초**가 된다.
> 3. **융합**: **POSIX IPC**(Message Queue, Semaphore, Shared Memory)가 **표준**이고 **System V IPC**와 **POSIX IPC**가 **공존**하며 **Socket**은 **Network IPC**로 **확장**된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
IPC는 **"프로세스 간 통신 메커니즘"**이다.

**IPC의 필요성****:
- **데이터 전송**: 프로세스 간 정보 공유
- **동기화**: 작업 조정
- **협력**: 공동 작업

### 💡 비유
IPC는 **"사람 간 대화****와 같다.
- **대화**: 정보 교환
- **신호**: 동기화
- **편지**: 비동기 통신

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         IPC 발전                                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Isolated Process:
    • 독립적 주소 공간
    • 통신 불가능
         ↓
Shared Memory:
    • 데이터 공유
    • 빠르지만 복잡
         ↓
Message Passing:
    • 메시지 교환
    • 추상화
```

## Ⅱ. 아키텍처 및 핵심 원리

### IPC 방식 비교

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         IPC 방식                                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Shared Memory]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 빠른 데이터 공유                                                                     │
    │  │  • 동기화 필요                                                                     │
    │  │  • 관계형 복잡                                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Message Passing]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 메시지 큐 통신                                                                       │
    │  │  • Kernel 개입 필요 (pipe, MQ)                                                      │
    │  │  • 느리지만 단순                                                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Socket]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Network IPC 확장                                                                     │
    │  │  • 프로세스 간 + 머신 간                                                           │
    │  │  • 표준화된 인터페이스                                                             │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### IPC 메커니즘 비교

| IPC 방식 | 속도 | 복잡도 | 동기화 | 용도 |
|----------|------|-------|--------|------|
| **Pipe** | 빠름 | 낮음 | X | 부자-자식 |
| **Message Queue** | 중간 | 중간 | O | 비부자-자식 |
| **Shared Memory** | 매우 빠름 | 높음 | 별도 | 대량 데이터 |
| **Socket** | 느림 | 낮음 | X | Network |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 파이프 통신
**상황**: 부모-자식 IPC
**판단**:

```c
#include <unistd.h>
#include <stdio.h>

int main() {
    int pipefd[2];
    char buf[100];

    pipe(pipefd);

    if (fork() == 0) {
        // 자식: 쓰기
        close(pipefd[0]);
        write(pipefd[1], "Hello Parent!", 12);
        close(pipefd[1]);
    } else {
        // 부모: 읽기
        close(pipefd[1]);
        read(pipefd[0], buf, 100);
        printf("Child said: %s\n", buf);
        close(pipefd[0]);
    }

    return 0;
}
```

## Ⅴ. 기대효과 및 결론

### IPC 기대 효과

| 효과 | IPC 없음 | IPC 있음 |
|------|---------|----------|
| **협력** | 불가능 | 가능 |
| **데이터 공유** | 파일 | 빠름 |
| **동기화** | 어려움 | 쉬움 |

### 미래 전망

1. **Zero-Copy**: buf는 복사 없는 IPC
2. **Shared Memory MPI**: 분산 IPC
3. **User-space IPC**: 커널 우회
