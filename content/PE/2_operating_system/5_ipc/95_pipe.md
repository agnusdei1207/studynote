# 파이프 (Pipe)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파이프는 **"프로세스 간 **단방향 통신(Unidirectional Communication)**을 위한 **커널 버퍼"**로, **하나의 프로세스**가 **데이터를 쓰고(Write)** **다른 프로세스**가 **데이터를 읽는(Read)** **생산자-소비자(Producer-Consumer)** 패턴의 **대표적인 IPC(Inter-Process Communication)** 메커니즘이다.
> 2. **가치**: **Anonymous Pipe(익명 파이프)**는 **부모-자식 프로세스** 간 통신에 사용되며 **Named Pipe(FIFO)**는 **관계 없는 프로세스** 간 통신을 가능하게 하고 **Shell 파이프라인(`|`)**은 **명령어 체이닝(Command Chaining)**의 핵심으로 **간결한 스크립트**를 작성하게 한다.
> 3. **융합**: **UNIX Philosophy**(Do one thing and do it well)의 구현으로 **`grep`, `awk`, `sed`** 같은 도구를 **조합**하여 **복잡한 작업**을 수행하며 **Linux Shell**, **PowerShell**, **Node.js Stream**에 응용된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
파이프는 **"한 프로세스의 출력을 다른 프로세스의 입력으로 연결하는 통신 방식"**이다.

**파이프의 특징**:
- **단방향**: 한 방향으로만 데이터 흐름
- **FIFO**: First-In-First-Out
- **바이트 스트림**: 메시지 경계 없음

### 💡 비유
파이프는 **"물관"**과 같다.
- **생산자**: 물을 붓는 곳
- **소비자**: 물을 받는 곳
- **버퍼**: 물통

## Ⅱ. 아키텍처 및 핵심 원리

### Anonymous Pipe (익명 파이프)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Anonymous Pipe 구조                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [pipe() 시스템 콜]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  #include <unistd.h>                                                                    │
    │                                                                                         │
    │  int pipe(int pipefd[2]);  // 성공 시 0, 실패 시 -1 반환                               │
    │                                                                                         │
    │  pipefd[0]: Read End (읽기 전용)                                                        │
    │  pipefd[1]: Write End (쓰기 전용)                                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [부모-자식 파이프]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  [fork() 전]                                                                             │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Parent Process                                                                       │  │
    │  │  pipefd[0] (Read)  ◄────┐                                                          │  │
    │  │  pipefd[1] (Write) ────┼───► Pipe Buffer                                          │  │
    │  └────────────────────────────────┴────────────────────────────────────────────────────┘  │
    │                                     │                                                │
    │                                     ▼ fork()                                        │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  [fork() 후]                                                                          │  │
    │  │                                                                                      │  │
    │  │  ┌─────────────────────┐     ┌─────────────────────┐                               │  │
    │  │  │  Parent Process     │     │  Child Process       │                               │  │
    │  │  │                     │     │                     │                               │  │
    │  │  │  close(pipefd[0])   │     │  close(pipefd[1])   │                               │  │
    │  │  │  pipefd[1] ──Write──►│     │  ◄─Read── pipefd[0] │                               │  │
    │  │  └─────────────────────┘     └─────────────────────┘                               │  │
    │  │         │                           │                                             │  │
    │  │         └─────────── Data Flow ──────────────────────────────────────┘             │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [파이프 버퍼]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Kernel 내부의 원형 버퍼(Circular Buffer)                                              │
    │  │  • 크기: 일반적으로 64KB (Linux 2.6.11~)                                               │
    │  │  • fcntl(F_SETPIPE_SZ)로 조정 가능                                                    │
    │                                                                                         │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐          │  │
    │  │  │ A │ B │ C │ D │ E │ F │   │   │   │   │   │   │   │   │   │   │   │          │  │
    │  │  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘          │  │
    │  │    ▲                       ▼                                                          │  │
    │  │   Read                  Write                                                        │  │
    │  │   Pointer               Pointer                                                      │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 파이프 동작

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         파이프 읽기/쓰기 동작                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Block on Read]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 파이프가 비어 있으면 read()는 Block                                                     │
    │  │  • O_NONBLOCK 플래그 시 즉시 -1(EAGAIN) 반환                                        │
    │                                                                                         │
    │  #include <fcntl.h>                                                                    │
    │  int flags = fcntl(pipefd[0], F_GETFL);                                                 │
    │  fcntl(pipefd[0], F_SETFL, flags | O_NONBLOCK);                                         │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Block on Write]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 파이프가 가득 차면 write()는 Block                                                    │
    │  │  • 버퍼 공간이 생길 때까지 대기                                                         │
    │                                                                                         │
    │  [Write 방식]                                                                           │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  PIPE_BUF (4KB) 보다 작은 write: Atomic (원자적)                                    │  │
    │  │  PIPE_BUF 보다 큰 write: Non-atomic (중간에 다른 데이터 끼어 가능)                     │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [파이프 닫기]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Write End을 닫으면(모든 Writer가 close()):                                         │
    │  │  - read()는 0(EOF) 반환 (파이프 끝)                                                   │
    │  │                                                                                      │
    │  • Read End을 닫으면(모든 Reader가 close()):                                             │
    │  │  - write()는 SIGPIPE Signal 발생, errno = EPIPE                                       │
    │                                                                                         │
    │  #include <signal.h>                                                                   │
    │  signal(SIGPIPE, SIG_IGN);  // SIGPIPE 무시                                              │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Named Pipe (FIFO)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Named Pipe (FIFO)                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [FIFO 생성]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  #include <sys/stat.h>                                                                  │
    │  #include <fcntl.h>                                                                    │
    │                                                                                         │
    │  // mkfifo() 또는 mkfifo 명령어                                                          │
    │  int mkfifo(const char *pathname, mode_t mode);                                          │
    │                                                                                         │
    │  // Shell 명령어                                                                        │
    │  $ mkfifo /tmp/my_pipe                                                                 │
    │  $ mknod /tmp/my_pipe p                                                                │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [FIFO 특징]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 파일 시스템에 경로 이름으로 존재                                                         │
    │  │  • 관계 없는 프로세스 간 통신 가능                                                        │
    │  │  • ls -l로 확인 (prw-rw---- 형태, p는 pipe)                                           │
    │  │  • 사용하지 않으면 unlink로 제거                                                        │
    │                                                                                         │
    │  $ ls -l /tmp/my_pipe                                                                   │
    │  prw-r--r-- 1 user group 0 Jan 1 12:00 /tmp/my_pipe                                     │
    │  │                                                                                      │
    │  # 프로세스 A (Writer)                                                                  │
    │  $ echo "Hello" > /tmp/my_pipe                                                          │
    │                                                                                         │
    │  # 프로세스 B (Reader)                                                                  │
    │  $ cat /tmp/my_pipe                                                                     │
    │  Hello                                                                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Shell Pipeline

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         셸 파이프라인                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [기본 파이프라인]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  $ ps aux | grep nginx | wc -l                                                        │
    │                                                                                         │
    │  ┌────────┐   ┌────────┐   ┌────────┐   ┌────────┐                                   │
    │  │ ps aux │ ─→│ grep   │ ─→│ wc -l  │   │        │                                   │
    │  │        │   │ nginx  │   │        │   │  셸    │                                   │
    │  │stdout  │   │stdin   │   │stdin   │   │        │                                   │
    │  │        │   │stdout  │   │stdout  │   │        │                                   │
    │  └────────┘   └────────┘   └────────┘   │        │                                   │
    │                                    ▼   │                                        │
    │                                 결과 출력                                            │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [내부 동작]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Shell이 pipe() 시스템 콜로 파이프 생성                                                    │
    │  2. fork()로 자식 프로세스 생성                                                              │
    │  3. 각 명령어에 파이프 끝 연결 (dup2)                                                        │
    │  4. exec()로 명령어 실행                                                                   │
    │                                                                                         │
    │  [상세 동작]                                                                              │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  // Pseudocode                                                                          │  │
    │  │  int pipefd[2];                                                                        │  │
    │  │  pipe(pipefd);                                                                         │  │
    │  │                                                                                       │  │
    │  │  if (fork() == 0) {       // Child 1: ps aux                                       │  │
    │  │      dup2(pipefd[1], STDOUT_FILENO);                                                 │  │
    │  │      close(pipefd[0]); close(pipefd[1]);                                            │  │
    │  │      execlp("ps", "ps", "aux", NULL);                                               │  │
    │  │  }                                                                                    │  │
    │  │                                                                                       │  │
    │  │  if (fork() == 0) {       // Child 2: grep nginx                                    │  │
    │  │      dup2(pipefd[0], STDIN_FILENO);                                                  │  │
    │  │      // 다음 파이프 연결...                                                           │  │
    │  │      execlp("grep", "grep", "nginx", NULL);                                          │  │
    │  │  }                                                                                    │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Anonymous Pipe vs Named Pipe

| 구분 | Anonymous Pipe | Named Pipe (FIFO) |
|------|----------------|-------------------|
| **파일 시스템** | X (커널만) | O (경로 있음) |
| **프로세스 관계** | 부모-자식 | 관계 없음 |
| **지속성** | 프로세스 종료 시 소멸 | 파일로 남음 |
| **생성** | pipe() | mkfifo() |

### 과목 융합 관점 분석

#### 1. 네트워크 ↔ 파이프
- **Socket**: 네트워크 파이프
- **SSH Tunnel**: 암호화 파이프

#### 2. 스트림 처리 ↔ 파이프
- **Node.js**: Stream.pipe()
- **Golang**: io.Pipe

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 로그 분석 파이프라인
**상황**: 로그 파일에서 에러 추출
**판단**:

```bash
# 파이프라인 구성
cat /var/log/app.log | \
  grep "ERROR" | \
  awk '{print $1, $6, $7}' | \
  sort | uniq -c | sort -rn | \
  head -10

# 또는 간단히
grep "ERROR" /var/log/app.log | wc -l

# Named Pipe 활용
mkfifo /tmp/log_pipe
tail -f /var/log/app.log > /tmp/log_pipe &
grep "ERROR" < /tmp/log_pipe
```

## Ⅴ. 기대효效果 및 결론

### 파이프 기대 효과

| 효과 | IPC 없을 시 | 파이프 있을 시 |
|------|----------|-------------|
| **통신 복잡도** | 높음 (임시 파일) | 낮음 |
| **동기화** | 어려움 | 자동 |
| **표현력** | 낮음 | 높음 (연산자) |
| **방향성** | 양방향 복잡 | 단방향 간단 |

### 미래 전망

1. **SPSC Queue**: Lock-free
2. **eventfd**: 이벤트 알림
3. **io_uring**: 비동기 I/O

- **POSIX**: pipe, mkfifo
- **Linux**: man 7 pipe
- **Ritchie & Kernighan**: UNIX Philosophy
