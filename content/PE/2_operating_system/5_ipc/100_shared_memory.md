# 공유 메모리 (Shared Memory)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공유 메모리는 **"여러 프로세스가 **동일한 **물리 메모리 영역**을 **접근**하는 **가장 빠른 **IPC 메커니즘"**으로, **Kernel**이 **메모리 영역**을 **할당**하고 **각 프로세스의 **가상 주소 공간**에 **매핑**(Mapping)하며 **System V shmget** 또는 **POSIX shm_open**으로 **생성**한다.
> 2. **동기화**가 **필수**이며 **Race Condition**을 **방지**하기 위해 **Mutex**, **Semaphore**, **Spinlock** 같은 **동기화 도구**를 **사용**하고 **쓰기**는 **상호 배제**되어야 하지만 **읽기**는 **동시에 가능**한 **Reader-Writer Lock**으로 **최적화**할 수 있다.
> 3. **Zero-Copy**로 **데이터 복사**가 **없어** **가장 빠르지만** **Kernel Management**가 **필요**하고 **POSIX mmap**을 **통해** **파일**을 **메모리에 **매핑**하는 **Memory-Mapped File**로도 **사용**된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
공유 메모리는 **"여러 프로세스가 같은 메모리를 공유하는 IPC"**이다.

**공유 메모리 특징**:
- **가장 빠른 IPC**: Kernel 통과 없음
- **Zero-Copy**: 데이터 복사 없음
- **동기화 필수**: Race Condition 위험

### 💡 비유
공유 메모리는 **"공용 화이트보드****와 같다.
- **화이트보드**: 공유 메모리
- **펜**: 쓰기/읽기
- **동시 접근**: 충돌 가능

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         IPC 메커니즘 발전                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Pipe:
    • Kernel 버퍼 복사
    • 단방향
         ↓
Message Queue:
    • Kernel 관리 큐
    • 비동기 통신
         ↓
Shared Memory:
    • 직접 메모리 접근
    • Zero-Copy (가장 빠름)
```

### System V Shared Memory

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         System V Shared Memory API                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. 생성/연결 (shmget, shmat)                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  int shmid = shmget(key, size, IPC_CREAT | 0666);    // 공유 메모리 생성              │  │  │
    │  │  char *ptr = shmat(shmid, NULL, 0);                   // 프로세스에 연결               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  2. 사용 (Mutex로 동기화)                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  mutex_lock();                                                   │  │  │
    │  │  strcpy(ptr, "Hello from Process A");              // 쓰기                              │  │  │
    │  │  mutex_unlock();                                                 │  │  │
    │  │                                                                                       │  │  │
    │  │  mutex_lock();                                                   │  │  │
    │  │  printf("Received: %s\n", ptr);                  // 읽기                                │  │  │
    │  │  mutex_unlock();                                                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  3. 해제 (shmdt, shmctl)                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  shmdt(ptr);                     // 프로세스에서 분리                                   │  │  │
    │  │  shmctl(shmid, IPC_RMID, NULL); // 메모리 삭제                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### POSIX Shared Memory

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         POSIX Shared Memory API                                           │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. 생성/연결 (shm_open, mmap)                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  int fd = shm_open("/my_shm", O_CREAT | O_RDWR, 0666);  // 공유 메모리 열기          │  │  │

    │  │  void *ptr = mmap(NULL, size, PROT_READ | PROT_WRITE,  │  │  │
    │  │                  MAP_SHARED, fd, 0);                  // 메모리 매핑                │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  2. 사용                                                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  sem_wait(sem);                                               │  │  │
    │  │  sprintf(ptr, "Data: %d", value);                 // 쓰기                              │  │  │
    │  │  sem_post(sem);                                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  3. 해제 (munmap, shm_unlink)                                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  munmap(ptr, size);                                        // 매핑 해제               │  │  │
    │  │  close(fd);                                                   // 파일 디스크립터 닫기   │  │  │
    │  │  shm_unlink("/my_shm");                                     // 공유 메모리 삭제       │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### IPC 메커니즘 비교

| IPC | 속도 | 복잡도 | 동기화 | 사용처 |
|-----|------|--------|--------|--------|
| **Pipe** | 느림 | 낮음 | 필요 없음 | 부모-자식 |
| **Message Queue** | 중간 | 중간 | 내장됨 | 비동기 |
| **Shared Memory** | 빠름 | 높음 | 필수 | 대량 데이터 |
| **Socket** | 느림 | 높음 | 필요 없음 | 네트워크 |

### 동기화 기법 비교

| 기법 | 특징 | 성능 | 사용처 |
|------|------|------|--------|
| **Mutex** | 상호 배제 | 중간 | 일반적 |
| **Semaphore** | 카운팅 | 중간 | 리소스 관리 |
| **Spinlock** | 대기 루프 | 빠름(짧게) | 커널 |
| **RW Lock** | 읽기 병렬 | 빠름 | 읽기 많음 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: Producer-Consumer
**상황**: 대량 데이터 공유
**판단**:

```c
// POSIX Shared Memory + Semaphore

#include <fcntl.h>
#include <sys/mman.h>
#include <semaphore.h>

struct shared_data {
    int buffer[100];
    int count;
};

// Producer
void producer() {
    int fd = shm_open("/my_shm", O_CREAT | O_RDWR, 0666);
    ftruncate(fd, sizeof(struct shared_data));
    struct shared_data *data = mmap(NULL, sizeof(struct shared_data),
                                    PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);

    sem_t *empty = sem_open("/empty", O_CREAT, 0666, 100);
    sem_t *full = sem_open("/full", O_CREAT, 0666, 0);
    sem_t *mutex = sem_open("/mutex", O_CREAT, 0666, 1);

    for (int i = 0; i < 1000; i++) {
        sem_wait(empty);
        sem_wait(mutex);
        data->buffer[data->count++] = i;
        sem_post(mutex);
        sem_post(full);
    }
}

// Consumer
void consumer() {
    int fd = shm_open("/my_shm", O_RDWR, 0666);
    struct shared_data *data = mmap(NULL, sizeof(struct shared_data),
                                    PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);

    sem_t *empty = sem_open("/empty", 0);
    sem_t *full = sem_open("/full", 0);
    sem_t *mutex = sem_open("/mutex", 0);

    while (1) {
        sem_wait(full);
        sem_wait(mutex);
        int value = data->buffer[--data->count];
        sem_post(mutex);
        sem_post(empty);
        printf("Consumed: %d\n", value);
    }
}
```

## Ⅴ. 기대효과 및 결론

### 공유 메모리 기대 효과

| 효과 | Message Queue | 공유 메모리 |
|------|--------------|-----------|
| **속도** | 중간 | 빠름 |
| **복사 오버헤드** | 있음 | 없음 |
| **동기화** | 내장됨 | 직접 구현 |
| **데이터 크기** | 작음 | 큼 |

### 모범 사례

1. **데이터 크기**: 큰 데이터에 적합
2. **동기화**: Mutex/Semaphore 필수
3. **해제**: 사용 후 반드시 해제
4. **권한**: 적절한 권한 설정

### 미래 전망

1. **RDMA**: 원격 공유 메모리
2. **Persistent Memory**: 영구 공유 메모리
3. **HSA**: 이기종 공유 메모리
