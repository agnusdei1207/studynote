# IPC (Inter-Process Communication)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPC는 **"프로세스 **간 **데이터 **교환 **메커니즘"**으로, **Pipe**(단방향 **스트림), **Message Queue**(메시지 **지연), **Shared Memory**(가장 **빠름), **Socket**(네트워크 **IPC)가 **대표적**이다.
> 2. **특성**: **Pipe**는 **부자-자식 **관계**에서 **사용**하고 **Named Pipe**(FIFO)는 **관계 **없는 **프로세스**간 **통신**하며 **Message Queue**는 **비동기 **통신**을 **지원**한다.
> 3. **선택**: **데이터 **양**이 **적으면** **Pipe**, **빠른 **통신**이 **필요**하면 **Shared Memory**, **네트워크**는 **Socket**, **비동기**는 **Message Queue**를 **사용**한다.

## Ⅰ. 개요 (Context & Background)

### 개념
IPC는 **"프로세스 간 통신 기법"**이다.

**IPC 종류**:
| IPC | 방향 | 속도 | 사용처 |
|-----|------|------|--------|
| **Pipe** | 단방향 | 중간 | 부자 관계 |
| **FIFO** | 단방향 | 중간 | 관계 없는 프로세스 |
| **Message Queue** | 양방향 | 느림 | 비동기 |
| **Shared Memory** | 양방향 | 빠름 | 대량 데이터 |
| **Socket** | 양방향 | 느림 | 네트워크 |

- **Pipe**: **"전화 **선 ****(일방통행)
- **Message Queue**: **"우편 **함 ****(보관)
- **Shared Memory**: **"공유 **화이트보드 ****(직접 쓰기)

### Pipe (Unnamed Pipe)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Pipe Architecture                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Parent Process                                                                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  pipe(fd) creates file descriptors: fd[0] (read), fd[1] (write)                     │  │  │
    │  │                                                                                       │  │  │
    │  │  fork() creates child process, inherits file descriptors                            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                                 │  │
    │            ▼ Fork                                                                           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Parent                          Child                                              │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  fd[0] (read) ──────────────────→ fd[0] (read, unused)                        │  │  │  │
    │  │  │  fd[1] (write) ←────────────────── fd[1] (write)                               │  │  │  │
    │  │  │                                                                                       │  │  │  │
    │  │  │  Data flow: Parent writes to fd[1] → Child reads from fd[0]                        │  │  │  │
    │  │  │  → Unidirectional (one-way)                                                        │  │  │  │
    │  │  │  → Related processes only (parent-child)                                          │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Message Queue

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Message Queue System                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Kernel Space                                                                           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Message Queue (Linked List in Kernel)                                              │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐                             │  │  │  │
    │  │  │  │ Msg 1   │→ │ Msg 2   │→ │ Msg 3   │→ │ Msg 4   │                             │  │  │  │
    │  │  │  │ Type: 1 │  │ Type: 2 │  │ Type: 1 │  │ Type: 3 │                             │  │  │  │
    │  │  │  │ Prio: 0 │  │ Prio: 1 │  │ Prio: 0 │  │ Prio: 2 │                             │  │  │  │
    │  │  │  └─────────┘  └─────────┘  └─────────┘  └─────────┘                             │  │  │  │
    │  │  │      ▲                                    │                                      │  │  │  │
    │  │  │      │                                    │                                      │  │  │  │
    │  │  │  msgsnd()                        msgrcv()                                         │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │           │                        ▼                                                        │  │
    │  ┌────────────────┐      ┌────────────────┐                                              │  │
    │  │  Process A      │      │  Process B      │                                              │  │
    │  │  msgsnd(qid,    │      │  msgrcv(qid,    │                                              │  │
    │  │         msg,    │      │         &msg,   │                                              │  │
    │  │         type,   │      │         type,   │                                              │  │
    │  │         flags)  │      │         flags)  │                                              │  │
    │  └────────────────┘      └────────────────┘                                              │  │
    │                                                                                         │  │
    │  → Non-blocking: Process continues if queue full/empty                                   │  │
    │  → Priority messages: Higher priority delivered first                                    │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Shared Memory Architecture                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │

    │  │  │                             │      │                                        │  │  │  │
```

## Ⅲ. 융합 비교 및 다각도 분석

### IPC 성능 비교

| IPC | Setup | Data Transfer | 속도 | 동기화 필요 |
|-----|-------|---------------|------|-------------|
| **Pipe** | 낮음 | Context switch | 중간 | X |
| **Message Queue** | 중간 | Context switch + copy | 느림 | X |
| **Shared Memory** | 높음 | Direct access | 빠름 | O |
| **Socket** | 높음 | Protocol overhead | 느림 | X |

### IPC 시스템 호출

```c
// Pipe (Unnamed)
int pipefd[2];
pipe(pipefd);  // pipefd[0]=read, pipefd[1]=write

// Named Pipe (FIFO)
mkfifo("/tmp/myfifo", 0666);
int fd = open("/tmp/myfifo", O_WRONLY);

// Message Queue (System V)
int msgid = msgget(IPC_PRIVATE, 0666 | IPC_CREAT);
msgsnd(msgid, &msg, sizeof(msg.data), 0);
msgrcv(msgid, &msg, sizeof(msg.data), 0, 0);

// Shared Memory (System V)
int shmid = shmget(IPC_PRIVATE, 1024, 0666 | IPC_CREAT);
void *ptr = shmat(shmid, NULL, 0);

// POSIX Shared Memory
int fd = shm_open("/myshm", O_CREAT | O_RDWR, 0666);
ftruncate(fd, 1024);
void *ptr = mmap(NULL, 1024, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
```

## Ⅳ. 실무 적용 및 기술사적 판단

### Python IPC 예시

```python
# Pipe
from multiprocessing import Process, Pipe

def sender(conn):
    conn.send("Hello")
    conn.close()

parent_conn, child_conn = Pipe()
p = Process(target=sender, args=(child_conn,))
p.start()
print(parent_conn.recv())  # "Hello"

# Queue
from multiprocessing import Queue

q = Queue()
q.put("message")
print(q.get())  # "message"

# Shared Memory (Value, Array)
from multiprocessing import Process, Value, Array

def modify(shared_val, shared_arr):
    shared_val.value = 42
    shared_arr[0] = 100

val = Value('i', 0)
arr = Array('i', [0, 0, 0])
p = Process(target=modify, args=(val, arr))
p.start()
p.join()
print(val.value)  # 42
print(arr[:])     # [100, 0, 0]
```

## Ⅴ. 기대효과 및 결론

### IPC 기대 효과

| IPC | 데이터 양 | 속도 | 복잡도 |
|-----|-----------|------|--------|
| **Pipe** | 적음 | 중간 | 낮음 |
| **Message Queue** | 적음 | 느림 | 중간 |
| **Shared Memory** | 대량 | 빠름 | 높음 |

### 모범 사례

1. **단방향 스트림**: Pipe
2. **비동기**: Message Queue
3. **대량 데이터**: Shared Memory
4. **네트워크**: Socket

### 미래 전망

1. **User-Level IPC**: io_uring
2. **Shared Memory**: mmap 기반
3. **Message Passing**: Erlang 스타일
