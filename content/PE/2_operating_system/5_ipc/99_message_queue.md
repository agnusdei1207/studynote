# 메시지 큐 (Message Queue)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메시지 큐는 **"비동기 **프로세스 간 **데이터 전송을 위한 **Kernel Managed **큐 자료구조****로, **Producer**(생산자)가 **데이터를 **넣고** **Consumer**(소비자)가 **데이터를 **꺼내는 **Non-Blocking** 또는 **Blocking** **Queue**이며 **System V IPC** 또는 **POSIX mq_**로 **구현**된다.
> 2. **** **동기화**가 **내장**되어 있어 **Race Condition**을 **방지**하고 **Buffer**가 **꽉 찼**으면 **Producer**는 **Block**되고 **비어 있으면** **Consumer**는 **Block**되며 **Priority Queue**는 **우선순위** **메시지**를 **먼저 **처리**한다.
> 3. **** **Linux Kernel**의 **IPC Queue**(kfifo)**로 **구현**되고 **Tasklet**(Kernel Thread)가 **Async Completion**을 **처리**하며 **Zero-Copy**로 **성능**을 **최적화**할 수 있다.

## Ⅰ. 개요 (Context & Background)

### 개념
메시지 큐는 **"프로세스 간 비동기 통신을 위한 큐 자료구조"**이다.

**메시지 큐의 특징**:
- **비동기**: 메시지 전송 후 즉시 반환
- **Buffer**: 큐 버퍼
- **Blocking**: 꽉 차면 Block

### 💡 비유
메시지 큐는 **"우편함****과 같다.
- **편지**: 메시지
- **함께**: 공유 버퍼
- **우편함**: 큐

### System V Message Queue

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         System V Message Queue                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • msgget/msgrcv: 메시지 수신                                                             │
    │  │  • msgsnd/msgrcv: 메시지 송신                                                             │  │
    │  │  │  • msgrcv: 제어/control                                                                    │  │
    │  │                                                                                      │  │
    │  ┌────────────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  struct msgbuf {                                                                      │  │
    │  │      long mtype;          // 메시지 유형                                                          │  │
    │  │      size_t mszie;          // 메시지 크기                                                          │  │
    │  │      struct msg *next;        // 다음 메시지                                                          │  │
    │  │      void (*msg_copy)(void);   // 데이터 복사                                                           │  │
    │  │      char mtext[1];         │  // 메일 내용 (가변 길이)                                            │  │
    │  │  };                                                                                    │  │
    │  └────────────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: Producer-Consumer
**상황**: 비동기 작업 처리
**판단**:

```c
#include <sys/msg.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

struct msgbuf {
    long mtype;
    char mtext[128];
};

int main() {
    int msqid;
    key_t key;
    struct msgbuf msg;

    // 메시지 큐 생성
    key = 1234;
    msqid = msgget(key, IPC_CREAT | 0666);

    // Consumer
    if (fork() == 0) {
        while (1) {
            msgrcv(msqid, &msg, 0, 0);
            printf("Received: %s\n", msg.mtext);
        }
    }
    // Producer
    else {
        for (int i = 0; i < 5; i++) {
            msg.mtype = 1;
            sprintf(msg.mtext, "Message %d", i);
            msgsnd(msqid, &msg, 0, 0);
            sleep(1);
        }
    }

    return 0;
}
```
