# 메모리 관리 (Memory Management)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 관리는 **"프로세스**에 **메모리**를 **할당**하고 **회수**하는 **기법"**으로, **Paging**(고정 **크기 **블록), **Segmentation**(논리적 **영역), **Virtual Memory**(디스크 **스왑)**으로 **구현**된다.
> 2. **Paging**: **Page**(4KB **고정 **블록)로 **물리 **메모리**를 **관리**하고 **Page Table**로 **가상 → **물리 **주소 **변환**을 **수행**하며 **TLB**(Translation Lookaside Buffer)로 **변환 **속도**를 **향상**시킨다.
> 3. **알고리즘**: **FIFO**(First-In First-Out), **LRU**(Least Recently Used), **Clock**(Circular **List) **교체 **정책**으로 **Page Fault**를 **처리**하고 **Thrashing**을 **방지**하기 **위해 **Working Set Model**을 **사용**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
메모리 관리는 **"RAM 할당 및 회수"**이다.

**관리 기법**:
| 기법 | 단위 | 설명 |
|------|------|------|
| **Paging** | Page (4KB) | 고정 크기 블록 |
| **Segmentation** | Segment (가변) | 논리적 영역 |
| **Combined** | Page + Segment | 둘 다 혼합 |
| **Virtual** | Page | 디스크 스왑 |

### 💡 비유
메모리 관리는 ****책장 **정리 ****와 같다.
- **책장**: 물리 메모리
- **페이지**: 챕터 파일
- **색인**: 페이지 테이블

### Page Fault 처리

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Page Fault Handling                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU tries to access virtual address 0x00400000                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. MMU walks page table                                                             │  │  │
    │  │  2. Page table entry not present (valid=0)                                             │  │  │
    │  │  3. MMU triggers page fault exception                                                │  │  │
    │  │  4. CPU trap to kernel page fault handler                                            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                                 │  │
    │            ▼                                                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Page Fault Handler (Kernel)                                                            │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  1. Check if address is valid (within VMA, permissions OK)                            │  │  │  │
    │  │  │  2. Find free physical frame                                                              │  │  │  │
    │  │  │ 3. If no free frame: Run page replacement algorithm                                      │  │  │  │
    │  │  │ 4. Copy page from disk (or zero) into frame                                          │  │  │  │
    │  │  │ 5. Update page table entry (VPN → PPN, valid=1)                                     │  │  │  │
    │  │  │ 6. Restart instruction                                                             │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → I/O Intensive: Disk access costs ~10ms                                                     │  │
    │  → Optimize with TLB (cache recent translations)                                            │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 핵심

### Linux 메모리 설정

```bash
# 페이지 크기 확인
getconf PAGESIZE  # 4096 bytes (4KB)

# Huge Pages (2MB, 1GB)
cat /proc/meminfo | grep Huge
# HugePages_Total: 0
# HugePages_Free: 0

# Transparent Huge Pages (THP)
cat /sys/kernel/mm/transparent_hugepage/enabled
# [always] [madvise] [never]

# 스왑 영역 확인
free -h
# total        used        free      shared  buff/cache   available
# Mem:           15Gi       4.5Gi       9.2Gi       1.0Gi       5.8Gi       9.2Gi
# Swap:          2.0Gi          0B        2.0Gi

# vm.swappiness (0=swap only if needed, 100=aggressive)
cat /proc/sys/vm/swappiness
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 데이터베이스 서버 메모리
**상황**: 16GB RAM, PostgreSQL
**판단**: Huge Page + swappiness 조정

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Database Memory Tuning                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  PostgreSQL Shared Buffer                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • shared_buffers: 4GB                                                              │  │  │
    │  │  → Use Huge Pages (2MB) to reduce TLB misses                                         │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  vm.swappiness = 10                                                                    │  │
    │  → Swap only when absolutely necessary (protect cache)                                     │  │  │
    │  → Avoid swapping database processes                                                    │  │  │
    │                                                                                         │  │
    │  Result:                                                                                │  │
    │  • TLB misses reduced (more data per TLB entry)                                           │  │  │
    │  • Less page fault overhead                                                             │  │  │
    │  • Stable performance under load                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 메모리 관리 기대 효과

| 정책 | Hit Rate | 복잡도 | 사용처 |
|------|----------|--------|--------|
| **FIFO** | 중간 | 낮음 | 단순 시스템 |
| **LRU** | 높음 | 낮음 | 범용 |
| **Clock** | 높음 | 낮음 | Linux |

### 모범 사례

1. **페이지 크기**: 4KB (일반), 2MB/1GB (DB)
2. **교체**: LU/Clock (범용)
3. **Swappiness**: 낮게 설정 (서버)
4. **모니터링**: page fault rate

### 미래 전망

1. **HugeTLB**: 대규 페이지 지원
2. **NUMA**: 노드 최적화
3. **Non-Uniform**: 특정 워크로드
