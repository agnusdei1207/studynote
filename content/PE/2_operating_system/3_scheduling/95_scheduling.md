# 스케줄링 (Scheduling)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스케줄링은 **"CPU 시간을 **프로세스/스레드에 **배당하는 **OS 기능"**으로, **Ready Queue**에서 **다음 프로세스**를 **선택**하고 **Context Switch**를 **수행**하며 **FCFS**, **SJF**, **RR**(Round Robin), **Priority** 등의 **알고리즘**이 **존재**한다.
> 2. **가치**: **CPU 이용률**을 **최대화**하고 **평균 대기 시간**(Average Waiting Time)을 **최소화**하며 **공정성**(Fairness)과 **효율성**(Efficiency)의 **균형**을 **맞춘**고 **Real-Time System**에서는 **Deadline**을 **준수**한다.
> 3. **융합**: **Batch**(일괄 처리), **Interactive**(대화형), **Real-Time**(실시간) **시스템**에 **맞는 **스케줄링**이 **다르**며 **Linux CFS**(Completely Fair Scheduler), **Windows**(Multilevel Feedback Queue)가 **현대 구현**이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스케줄링은 **"CPU를 누구에게 얼마나 줄 것인가 결정"**이다.

**스케줄링 목표**:
- **CPU Utilization**: 최대화
- **Throughput**: 단위 시간당 작업량
- **Turnaround Time**: 완료 시간
- **Waiting Time**: 대기 시간
- **Response Time**: 응답 시간
- **Fairness**: 공정성

### 💡 비유
스케줄링은 **"은행 창구 직원"**과 같다.
- **고객**: 프로세스
- **창구**: CPU
- **직원**: 스케줄러

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스케줄링 발전                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Batch Systems:
    • FCFS (First Come, First Served)
    • 간단하지만 비효율적
         ↓
Time-Sharing:
    • Round Robin
    • 대화형 시스템
         ↓
Multics:
    • Priority, Multilevel Queue
    • 복잡한 시스템
         ↓
Modern OS:
    • CFS, MLFQ
    • 공정성, 효율성
```

## Ⅱ. 아키텍처 및 핵심 원리

### 스케줄링 알고리즘

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스케줄링 알고리즘                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [FCFS (First-Come, First-Served)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 먼저 온 순서대로 처리                                                                   │
    │  │  • Convoy Effect: 짧은 작업이 긴 작업 뒤에 기다림                                     │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  P1(24) ───────────────────────────────────> P2(3) ──> P3(3)                      │  │
    │  │  │                                                       │                         │  │
    │  │  │  0──────────────────────────────────24 27────30                      │  │
    │  │  │                                                       │                         │  │
    │  │  Average Waiting: (0 + 24 + 27) / 3 = 17                                            │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [SJF (Shortest Job First)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 실행 시간이 짧은 작업 우선                                                           │
    │  │  • 평균 대기 시간 최소화                                                              │
    │  │  • Starvation 문제: 긴 작업 무한 대기                                                │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  P1(24)  P2(3)  P3(3)                                                               │  │
    │  │  │  최단 3 선택                                                      │  │
    │  │  │  P2 → P3 (3+3=6) ──────────────────────────────────>                         │  │
    │  │  │                                                                P1(24)               │  │
    │  │  └───────────────────────────────────────────────────────────────────────────────────┘  │
    │  │                                                                                       │  │
    │  Average Waiting: (0 + 3 + 6) / 3 = 3 (최적!)                                               │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [RR (Round Robin)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Time Quantum(시간 할당량) 만큼 교대 실행                                                  │
    │  │  • 공정성 보장                                                                        │
    │  │  • Context Switch 오버헤드                                                             │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Time Quantum = 4                                                                     │  │
    │  │  P1(10)  P2(5)  P3(2)                                                               │  │
    │  │  │  P1(4) → P2(4) → P3(2) → P1(4) → P2(1)                                            │  │
    │  │  │  ┌───┐  ┌───┐  ┌──┐  ┌───┐  ┌─┐                                             │  │
    │  │  │  │4  │  │4  │  │2 │  │4  │  │1 │                                             │  │
    │  │  │  └───┘  └───┘  └──┘  └───┘  └─┘                                             │  │
    │  │  │ 0  4  8 10 12 14 16 17                                                         │  │
    │  │                                                                                       │  │
    │  Average Waiting: (9 + 7 + 4) / 3 = 6.67                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Multilevel Feedback Queue

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Multilevel Feedback Queue                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 여러 큐에 우선순위                                                                  │
    │  │  • CPU 사용 많으면 하위 큐로 강등 (Aging)                                           │
    │  │  • I/O 많으면 상위 큐로 유지                                                              │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Queue 1 (RR, Q=8)                                                                   │  │
    │  │    ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │    │  • Interactive Process                                                           │  │
    │  │    │  • 짧은 Time Quantum                                                              │  │
    │  │    │  → 빠른 응답 필요                                                                  │  │
    │  │    └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │           ▼ (Time Quantum 초과 시 하위 강등)                                            │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  │  Queue 2 (RR, Q=16)                                                                  │  │
    │  │  │    • Interactive Process (I/O-bound)                                               │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │           ▼                                                                          │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  │  Queue 3 (FCF)                                                                      │  │
    │  │  │    • Batch Process (CPU-bound)                                                      │  │
    │  │  │    • 긴 Time Quantum 또는 FCFS                                                       │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 스케줄링 알고리즘 비교

| 알고리즘 | 평균 대기 시간 | 공정성 | Starvation | Context Switch |
|---------|-------------|--------|-----------|---------------|
| **FCFS** | 높음 | O | X | 낮음 |
| **SJF** | 최소 | X | O | 낮음 |
| **RR** | 중간 | O | X | 높음 |
| **Priority** | 낮음 | △ | O | 중간 |
| **MLFQ** | 중간 | O | X | 높음 |

### Linux CFS (Completely Fair Scheduler)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Linux CFS                                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Red-Black Tree로 Task 관리                                                           │
    │  │  • Virtual Runtime (vruntime) 기반 공정 스케줄링                                      │
    │  │  • O(log N) 스케줄링                                                                   │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  struct task_struct {                                                               │  │
    │  │      u64 vruntime;              // 가상 실행 시간                                    │  │
    │  │      u64 exec_start;            // 실행 시작 시각                                   │  │
    │  │      struct sched_entity se;   // 스케줄링 엔티티                                │  │
    │  │  };                                                                                    │  │
    │  │                                                                                       │  │
    │  │  # vruntime 계산                                                                      │  │
    │  │  vruntime += (delta_exec * NICE_0_TO_LOAD_1) / weight                               │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 우선순위 설정
**상황**: 실시간 프로세스
**판단**:

```bash
# nice 값으로 우선순위 조정 (-19~20)
$ nice -n -10 ./realtime_app   # 높은 우선순위
$ nice -n 10 ./batch_task     # 낮은 우선순위

# renice로 실행 중 프로세스 우선순위 변경
$ renice -n -10 -p 1234

# ionice로 I/O 우선순위 (Real-time, Best-effort, Idle)
$ ionice -c 1 -n 0 ./io_intensive_app
```

## Ⅴ. 기대효과 및 결론

### 스케줄링 기대 효과

| 효과 | 비스케줄링 | 스케줄링 |
|------|----------|--------|
| **CPU 활용** | 유휴 | 최대화 |
| **대기 시간** | 김 | 짧음 |
| **공정성** | 없음 | 있음 |

### 미래 전망

1. **Energy-Aware**: 전력 효율
2. **Real-Time**: Deadline 보장
3. **AI Scheduling**: ML 기반
