# 스케줄링 (Scheduling)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스케줄링은 **"프로세스**가 **CPU**를 **언제**, **어떻게** **사용**할지 **결정**하는 **OS 핵심 기능"**으로, **Preemptive**(선점형)와 **Non-Preemptive**(비선점형)으로 **구분**되며 **Response Time**(응답 시간), **Turnaround Time**(반환 시간), **Throughput**(처리량)을 **최적화**한다.
> 2. **알고리즘**: **FCFS**(선입선출), **SJF**(최단 작업 우선), **SRT**(남은 시간 최소), **Round Robin**(시간 할당), **Multilevel Feedback Queue**(다단계 피드백 큐)로 **구현**되며 **Modern OS**는 **CFS**(Completely Fair Scheduler)를 **사용**한다.
> 3. **융합**: **Linux CFS**(Red-Black Tree), **Windows**(Priority-based, Multilevel Queue), **Real-time OS**(Rate Monotonic, EDF)가 **대표적**이며 **Priority Inversion**을 **방지**하기 위해 **Priority Inheritance**를 **사용**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스케줄링은 **"CPU 할당 순서 결정"**이다.

**스케줄링 목표**:
- **CPU Utilization**: 최대화
- **Throughput**: 단위 시간당 프로세스 수
- **Turnaround Time**: 최소화
- **Response Time**: 최소화
- **Fairness**: 공정성

### 💡 비유
스케줄링은 **"은행 창구 관리****와 같다.
- **고객**: 프로세스
- **창구**: CPU
- **번호표**: Queue

## Ⅱ. 아키텍처 및 핵심 원리

### 스케줄링 알고리즘

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Scheduling Algorithms                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    FCFS (First-Come, First-Served):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  P1 (24) │ P2 (3) │ P3 (3)                                                           │  │
    │  ────────┴────────┴─────────                                                       │  │
    │  Average Turnaround: (24 + 27 + 30) / 3 = 27                                         │  │
    │  → Convoy Effect (긴 프로세스가 뒤 프로세스 지연)                                         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    SJF (Shortest Job First):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  P2 (3) │ P3 (3) │ P1 (24)                                                          │  │
    │  ────────┴────────┴───────────                                                       │  │
    │  Average Turnaround: (3 + 6 + 30) / 3 = 13                                              │  │
    │  → Starvation 문제 (긴 프로세스 기아 현상)                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Round Robin (Quantum = 4):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  P1│ P2│ P3│ P1│ P2│ P3│ P1│ ...                                                 │  │
    │  ───┴───┴───┴───┴───┴───┴───┴───                                                     │  │
    │  → 공정하지만 Context Switch 오버헤드                                                   │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Linux CFS (Completely Fair Scheduler)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Linux CFS                                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Red-Black Tree (vruntime 기반 오름차순)                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  [Task A] │  [Task B]  │  [Task C]   │  [Task D]                              │  │  │
    │  │  vr:10     │  vr:15      │  vr:20      │  vr:25                                │  │  │
    │  │  (min)     │            │            │  (left)                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │         │                                                                                   │  │
    │         ▼ (좌측에서 꺼냄)                                                                  │  │
    │    실행 → vruntime 증가 → 다시 삽입                                                      │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 스케줄링 비교

| 알고리즘 | 평균 반환 시간 | 응답 시간 | 공정성 | 사용처 |
|---------|--------------|----------|--------|--------|
| **FCFS** | 높음 | 중간 | 있음 | 단순 시스템 |
| **SJF** | 낮음 | 높음 | 없음 | 배치 |
| **RR** | 중간 | 낮음 | 있음 | 시분할 |
| **CFS** | 중간 | 낮음 | 있음 | 현대 OS |

| 알고리즘 | 선점 | 주기적 | 사용처 |
|---------|------|--------|--------|
| **RM** | O | O | 주기적 태스크 |
| **EDF** | O | O | 동적 태스크 |

**판단**:

```bash

# 실시간 우선순위
chrt -f 99 ./realtime_app      # SCHED_FIFO
```
