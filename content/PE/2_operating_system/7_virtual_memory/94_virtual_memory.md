# 가상 메모리 (Virtual Memory)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 메모리는 **"물리 **메모리**(RAM)보다 **큰 **논리적 **주소 **공간**을 **프로세스**에 **제공**하는 **기술"**로, **Paging**(페이지 단위 관리)으로 **구현**되며 **Page Table**로 **가상 주소**(Virtual)를 **물리 주소**(Physical)로 **변환**하고 **Swap**(교체)로 **부족한 **메모리**를 **해결**한다.
> 2. **MMU**(Memory Management Unit)가 **주소 변환**을 **수행**하고 **TLB**(Translation Lookaside Buffer)로 **변환 **결과**를 **캐싱**하며 **Page Fault**시 **Demand Paging**(요구 페이징)으로 **페이지**를 **로드**한다.
> 3. **융합**: **Page Replacement**(LRU, Clock)로 **교체 **페이지**를 **선택**하고 **Thrashing**(스래싱)을 **방지**하기 위해 **Working Set Model**을 **사용**하며 **Modern OS**는 **Huge Page**(2MB, 1GB)로 **TLB Miss**를 **줄인다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
가상 메모리는 **"논리적 주소 공간 추상화"**이다.

**가상 메모리 특징**:
- **추상화**: 논리-물리 분리
- **크기**: 물리 메모리보다 큼
- **보호**: 프로세스 격리
- **효율**: 메모리 활용

### 💡 비유
가상 메모리는 ****책**과 ****책장****과 같다.
- **책**: 가상 메모리 (전체)
- **책장**: 물리 메모리 (일부)
- **페이지**: 단위

## Ⅱ. 아키텍처 및 핵심 원리

### 주소 변환

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Address Translation (Paging)                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Virtual Address (32-bit)                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  VPN (20 bits)        offset (12 bits)                                                  │  │  │
    │  │  ┌───────┴─────────┐  ┌──────────┐                                                  │  │  │
    │  │  │       ↓           │  │          │                                                  │  │  │
    │  │  │  Page Table       │  │          │                                                  │  │  │
    │  │  │  VPN ───→ PPN      │  │  offset   ────────→ Physical Address                       │  │  │
    │  │  │                  │  │                                                             │  │  │
    │  │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  │  Page Table Entry (PTE)                                                         │  │  │  │
    │  │  │  │  │  • Present Bit: 0=Not in Memory, 1=In Memory                                │  │  │  │
    │  │  │  │  │  • Dirty Bit: 0=Clean, 1=Modified                                       │  │  │  │
    │  │  │  │  │  • PPN: Physical Page Number                                                 │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  Page Fault 시:                                                                         │  │
    │  1. OS의 Page Fault Handler 호출                                                       │  │
    │  2. Swap 영역에서 페이지 로드                                                           │  │
    │  3. Page Table 업데이트                                                                 │  │
    │  4. 명령어 재실행                                                                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### TLB (Translation Lookaside Buffer)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB (Address Translation Cache)                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  TLB Entry (CAM - Content Addressable Memory)                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  VPN │  │ Valid  PPN       │                                                   │  │  │
    │  │  │  ───┼───┴────────┼───────────────────────────────→                            │  │  │
    │  │  │  0x01 │ 1 │ 0x03     │ Hit! (Fast Path)                                      │  │  │
    │  │  │  0x02 │ 1 │ 0x05     │ Hit!                                                 │  │  │
    │  │  │ 0x03 │ 0 │ (invalid)│ Miss → Page Table Walk (Slow Path)                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  → TLB Hit 시 1 cycle, Miss 시 10~100 cycles                                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 페이지 교체 알고리즘

| 알고리즘 | 페이지 | 구현 | 사용처 |
|---------|------|------|--------|
| **LRU** | 오래된 것 | Reference Bit | 범용 |
| **FIFO** | 첫 번째 | Queue | 단순 |
| **Clock** | 포인터 0 | Clock Bit | Linux |
| **Optimal** | 안 쓸일 | 미래 예측 | 이론 |

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Thrashing & Working Set                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Working Set Model:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 프로세스가 활발히 사용하는 페이지 집합                                                   │  │
    │  │  • Working Set > Physical Memory → Thrashing                                           │  │
    │  │                                                                                       │  │
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 페이지 크기
**상황**: 성능 저하
**판단**: Huge Page 사용

```bash
# Linux Huge Page (2MB)
echo 100 > /proc/sys/vm/nr_hugepages
mkdir /dev/hugepages
mount -t hugetlbfs nodev /dev/hugepages
```
