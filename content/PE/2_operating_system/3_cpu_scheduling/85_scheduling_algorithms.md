# 스케줄링 알고리즘 (Scheduling Algorithms)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스케줄링 알고리즘은 **"준비 상태(Ready Queue)의 프로세스 중 다음에 실행할 프로세스를 선택하는 결정 규칙"**으로, **FCFS(선입선처)**, **SJF(최단 작업 우선)**, **Priority(우선순위)**, **Round Robin(라운드 로빈)**, **MLFQ(다중 레벨 피드백 큐)** 등이 있다.
> 2. **가치**: **평균 반환 시간(Turnaround Time)**, **대기 시간(Waiting Time)**, **응답 시간(Response Time)**, **처리량(Throughput)**, **CPU 이용률(Utilization)**의 **성능 지표**를 최적화하며, **시분할 시스템**의 공정성과 **실시간 시스템**의 **Deadline 보장**을 실현한다.
> 3. **융합**: **Linux CFS(Completely Fair Scheduler)**, **Windows Priority-based Scheduler**, **RTOS**의 **Rate Monotonic**, **EDF(Earliest Deadline First)**로 구현되며, **Kubernetes**, **Docker**의 **CPU Share**, **Quota** 할당의 기반이 된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스케줄링 알고리즘은 **"CPU 시간을 프로세스에 할당하는 순서와 방법을 결정하는 정책"**이다.

**성능 평가 기준**:
- **반환 시간(Turnaround Time)**: 완료 시간 - 도착 시간
- **대기 시간(Waiting Time)**: 반환 시간 - 버스트 시간
- **응답 시간(Response Time)**: 첫 응답까지의 시간
- **처리량(Throughput)**: 단위 시간당 완료된 프로세스 수
- **CPU 이용률(Utilization)**: CPU 사용 시간 / 전체 시간

### 💡 비유
스케줄링은 **"식당 주문 처리"**과 같다.
- **FCFS**: 먼저 온 손님부터
- **SJF**: 빨리 끝나는 음식부터
- **Priority**: VIP 손님부터
- **RR**: 돌아가며 서빙

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스케줄링 알고리즘의 진화                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

일괄처리 (1950년대):
    • FIFO (First In, First Out)
    • 순차 처리
    • 대기 시간 김
         ↓
시분할 시스템 (1960년대):
    • Round Robin 도입
    • Time Slice 개념
    • 응답 시간 개선
         ↓
다중 프로그래밍 (1970년대):
    • SJF, Priority Queue
    • 선점/비선점 구분
    • CPU 이용률 향상
         ↓
멀티레벨 (1980년대):
    • MLFQ (Multilevel Feedback Queue)
    • 다양한 작업 특성 반영
    • 적응형 스케줄링
         ↓
현대 OS (2000년대~):
    • Linux CFS (Red-black tree)
    • Windows Priority-based
    • Real-time 스케줄링
```

### 스케줄링 알고리즘 상세 비교

```
    [예시 프로세스]

    Process   Arrival Time   Burst Time
    ─────────────────────────────────────
       P1         0              8
       P2         1              4
       P3         2              9
       P4         3              5

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. FCFS (First-Come, First-Served) / 선입선처                                         │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  [특징]                                                                                 │
    │  • 도착 순서대로 CPU 할당                                                               │
    │  • 비선점 (Non-preemptive)                                                             │
    │  • 구현 간단                                                                             │
    │                                                                                         │
    │  [간트 차트]                                                                             │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │ P1(0~8) │ P2(8~12) │ P3(12~21) │ P4(21~26)                                       │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  [성능]                                                                                 │
    │  • 평균 반환 시간: (8+11+19+23)/4 = 15.25                                               │
    │  • 평균 대기 시간: (0+7+10+18)/4 = 8.75                                                  │
    │  • Convoy Effect: 긴 프로세스 뒤에 짧은 프로세스가 대기                                 │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  2. SJF (Shortest Job First) / 최단 작업 우선                                           │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  [특징]                                                                                 │
    │  • 버스트 시간이 가장 짧은 프로세스 우선                                                 │
    │  • 비선점형: 한 번 시작하면 완료까지                                                     │
    │  • 최소 평균 반환 시간 보장                                                             │
    │                                                                                         │
    │  [간트 차트]                                                                             │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │ P1(0~8) │ P2(8~12) │ P4(12~17) │ P3(17~26)                                       │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  [성능]                                                                                 │
    │  • 평균 반환 시간: (8+11+15+23)/4 = 14.25                                               │
    │  • 평균 대기 시간: (0+7+7+18)/4 = 8.0                                                    │
    │  • 문제: 기아 현상(Starvation), 버스트 시간 미지                                          │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  3. SRTF (Shortest Remaining Time First) / 최단 잔여 시간 우선 (선점형)                │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  [특징]                                                                                 │
    │  • 남은 버스트 시간이 가장 짧은 프로세스 우선                                           │
    │  • 선점 (Preemptive)                                                                   │
    │  • 새 프로세스 도착 시 잔여 시간 비교                                                     │
    │                                                                                         │
    │  [간트 차트]                                                                             │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │ P1(0~1)│P2(1~5)│P4(5~10)│P1(10~11)│P3(11~20)│P4(20~21)│P3(21~26)                  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │    P1실행  P2도착   P4도착   P1잔여=7  P3도착                                        │
    │    중간    더 짧음   더 짧음   P1재개   잔여=9                                        │
    │                                                                                         │
    │  [성능]                                                                                 │
    │  • 평균 반환 시간: (11+4+10+18)/4 = 10.75                                               │
    │  • 평균 대기 시간: (3+0+2+13)/4 = 4.5                                                    │
    │  • 문제: 기아 현상, 문맥 교환 오버헤드                                                   │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  4. Priority Scheduling / 우선순위 스케줄링                                            │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  [특징]                                                                                 │
    │  • 우선순위가 가장 높은 프로세스 우선                                                   │
    │  • 선점형/비선점형 가능                                                                  │
    │  • 기아 현상 해결: Aging (오래 대기하면 우선순위 증가)                                  │
    │                                                                                         │
    │  [간트 차트: 우선순위 P2>P4>P3>P1]                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │ P1(0~1)│P2(1~5)│P4(5~10)│P3(10~19)│P1(19~27)                                     │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  [성능]                                                                                 │
    │  • 평균 반환 시간: (27+4+17+7)/4 = 13.75                                                │
    │  • 평균 대기 시간: (19+0+8+2)/4 = 7.25                                                   │
    │  • Aging으로 기아 현상 완화                                                             │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  5. Round Robin / 라운드 로빈                                                           │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  [특징]                                                                                 │
    │  • Time Quantum (Time Slice) 만큼만 실행                                                 │
    │  • 선점형: Time Quantum 종료 시 Ready Queue 끝으로                                        │
    │  • 공정한 CPU 할당                                                                       │
    │                                                                                         │
    │  [간트 차트: Time Quantum = 4]                                                          │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │ P1(0~4)│P2(4~8)│P3(8~12)│P4(12~16)│P1(16~20)│P2(20~24)│P3(24~28)│P4(28~32)            │  │
    │  │         │P2완료 │       │P4완료  │P1완료   │         │       │P3완료                  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  [성능]                                                                                 │
    │  • 평균 반환 시간: (20+24+28+32)/4 = 26.0                                               │
    │  • 평균 대기 시간: (12+20+19+27)/4 = 19.5                                                │
    │  • 장점: 응답 시간 짧음, 공정함                                                         │
    │  • 단점: Context Switch 오버헤드                                                       │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  6. MLFQ (Multilevel Feedback Queue) / 다중 레벨 피드백 큐                              │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  [특징]                                                                                 │
    │  • 여러 큐(Queue)에 Time Quantum 다르게 설정                                            │
    │  • 높은 우선순위 큐: 짧은 Time Quantum (대화형 작업)                                     │
    │  • 낮은 우선순위 큐: 긴 Time Quantum (CPU 작업)                                         │
    │  • 피드백: 긴 시간 사용하면 낮은 우선순위로 이동                                         │
    │                                                                                         │
    │  [큐 구조]                                                                              │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Queue 0 (Highest Priority): Time Quantum = 8                                      │  │
    │  │  Queue 1: Time Quantum = 16                                                        │  │
    │  │  Queue 2: Time Quantum = 32                                                        │  │
    │  │  Queue 3 (Lowest Priority): FCFS                                                  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  [이동 규칙]                                                                            │
    │  • Time Quantum 소진: 낮은 우선순위로                                                   │
    │  • CPU 반환(입출력): 동일 우선순위 유지                                                  │
    │  • 새 프로세스: 가장 높은 우선순위                                                       │
    │                                                                                         │
    │  [장점]                                                                                 │
    │  • 대화형 작업에 빠른 응답                                                             │
    │  • 다양한 작업 유형 최적화                                                               │
    │  • 기아 현상 방지 (Aging)                                                               │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
```

### Time Quantum 최적화

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Time Quantum vs 성능                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Time Quantum 너무 작음 (예: 1ms)]

    • Context Switch 빈번
    • 오버헤드 증가
    • 처리량(Throughput) 저하
    • 응답 시간 짧음

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  P1(1)│CS│P2(1)│CS│P3(1)│CS│P4(1)│CS│P1(1)│CS│...                                   │
    │  └── CS: Context Switch (~1-10µs)                                                      │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Time Quantum 너무 큼 (예: 100ms)]

    • Context Switch 감소
    • FCFS와 유사
    • 응답 시간 길음
    • 대화형 작업 부적합

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  P1(8)│P2(4)│P3(9)│P4(5)                                                              │
    │  └── 짧은 프로세스가 긴 프로세스 뒤에 대기                                              │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [최적 Time Quantum]

    • 10 ~ 100ms 일반적
    • 평균 버스트 시간의 80%
    • Context Switch 오버허드 < 10%

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  일반적 가이드라인:                                                                   │
    │  • 대화형 시스템: 10 ~ 50ms                                                           │
    │  • 배치 처리: 100ms 이상                                                              │
    │  • 실시간: 1ms 이하                                                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### OS별 스케줄러 구현

| OS | 스케줄러 | 특징 | 시간 복잡도 |
|----|---------|------|-----------|
| **Linux** | CFS | Red-black Tree, 완전 공정 | O(log N) |
| **Windows** | Priority-based | Multi-level Queue, Dynamic Priority | O(1) |
| **macOS** | MLFQ | Multi-level Feedback Queue | O(1) |
| **Real-time** | EDF/RM | Deadline 기반 | O(N) |

### 과목 융합 관점 분석

#### 1. 실시간 시스템 ↔ 스케줄링
- **Rate Monotonic (RM)**: 주기가 짧을수록 높은 우선순위
- **EDF (Earliest Deadline First)**: 마감 기한이 가장 이른 작업 우선
- **Deadline Monotonic**: 상대 마감 기한 기반

#### 2. 멀티코어 ↔ 스케줄링
- **Load Balancing**: 코어 간 부하 분산
- **CPU Affinity**: 특정 코어에 바인딩
- **NUMA Awareness**: 메모리 로컬리티 고려

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 웹 서버 스케줄링 선택
**상황**: 짧은 요청이 많은 웹 서버
**판단**:
1. **Round Robin**: Time Quantum 50ms
2. **CPU 부하가 높은 요청**: 낮은 우선순위 큐
3. **대화형 요청**: 높은 우선순위 큐

```bash
# Linux에서 CFS 설정
echo 100000 > /proc/sys/kernel/sched_rt_period_us
echo 95000 > /proc/sys/kernel/sched_rt_runtime_us

# 프로세스 우선순위 설정
chrt -f -p 50 <pid>  # FIFO, priority 50
nice -n 10 <command> # 낮은 우선순위
```
