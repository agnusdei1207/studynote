# 스케줄러 (Scheduler)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스케줄러는 **"준비(Ready) 상태의 프로세스 중에서 CPU를 할당할 프로세스를 선택하는 OS 커널 컴포넌트"**로, **장기 스케줄러(Long-term Scheduler)**, **중기 스케줄러(Medium-term Scheduler)**, **단기 스케줄러(Short-term Scheduler)**로 계층화된다.
> 2. **가치**: **CPU 이용률(Utilization)**, **처리량(Throughput)**, **반환 시간(Turnaround Time)**, **대기 시간(Waiting Time)**, **응답 시간(Response Time)**의 **스케줄링 평가 기준**을 최적화하며, **디스패처(Dispatcher)**가 실제 CPU 할당을 수행한다.
> 3. **융합**: **실시간 시스템(Real-time System)**의 **Rate Monotonic**, **EDF(Earliest Deadline First)** 스케줄링으로 확장되며, **멀티코어(Multi-core)**, **클러스터(Cluster)**, **컨테이너(Container)**의 **CPU Share**, **Cgroup**, **Kubernetes** 스케줄링의 기반이 된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스케줄러(Scheduler)는 **"준비 상태(Ready Queue)의 프로세스 중에서 CPU를 할당할 프로세스를 선택하고, 디스패처가 실제 CPU를 할당하는 역할"**을 한다.

**스케줄링의 목표**:
- **시스템 관점**: CPU 이용률 최대화, 처리량最大化
- **사용자 관점**: 반환 시간, 대기 시간, 응답 시간 최소화

### 💡 비유
스케줄러는 **"식당 주인"**과 같다.
- **손님(프로세스)**: 준비 상태에서 대기
- **주인(스케줄러)**: 다음에 누구를 서비스할지 선택
- **종업원(디스패처)**: 실제 주문 받고 서비스 제공

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스케줄러의 진화                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

일괄처리 시스템 (Batch, 1950년대):
    • FIFO (First In, First Out)
    • 작업 순차 처리
    • 대기 시간 긺
         ↓
시분할 시스템 (Time Sharing, 1960년대):
    • Round Robin 도입
    • 응답 시간 개선
    • 대화형 처리 가능
         ↓
다중 프로그래밍 (Multiprogramming, 1970년대):
    • SJF, Priority Queue
    • CPU 이용률 향상
    • 메모리 관리 연계
         ↓
멀티코어 (Multi-core, 2000년대~):
    • SMP(Symmetric Multi-Processing)
    • CPU Affinity, Load Balancing
    • 실시간 스케줄링
```

## Ⅱ. 아키텍처 및 핵심 원리

### 스케줄러 3계층 구조

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스케줄러 3계층 구조                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [장기 스케줄러 (Long-term Scheduler / Job Scheduler)]
    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  역할: 디스크의 작업(Job Pool) → 메모리에 적재할 프로세스 선택                          │
    │  실행 빈도: 초-분 단위 (매우 낮음)                                                      │
    │                                                                                         │
    │  [결정 기준]                                                                             │
    │  • I/O 바운드 vs CPU 바운드 프로세스 비율 조정                                           │
    │  • 시스템의 Degree of Multiprogramming 제어                                             │
    │  • 메모리 여유 공간 확인                                                                │
    │                                                                                         │
    │  Job Pool (디스크)                 Ready Queue (메모리)                                 │
    │  ┌─────────────────────────────┐   ┌─────────────────────────────────────────────────┐  │
    │  │ JOB1, JOB2, JOB3, ...       │──▶│  Selected: P1, P2, P3                         │  │
    │  │  (수백~수천 개)             │   │  (Degree of Multiprogramming = M)              │  │
    │  └─────────────────────────────┘   └─────────────────────────────────────────────────┘  │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
                                │
                                ▼
    [중기 스케줄러 (Medium-term Scheduler / Swapper)]
    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  역할: 메모리 부족 시, 일부 프로세스를 디스크(Swap Area)로 축출(Swap out)                  │
    │  실행 빈도: 수초-수분 단위                                                               │
    │                                                                                         │
    │  [Swapping 결정 기준]                                                                    │
    │  • 메모리 부족 발생                                                                     │
    │  • 우선순위가 낮은 프로세스                                                             │
    │  • 오랫동안 Ready 상태인 프로세스                                                        │
    │                                                                                         │
    │  Memory                 Swap Space (디스크)                                             │
    │  ┌─────────────────────┐     ┌─────────────────────────────────────────────────────────┐  │
    │  │ P1, P2, P3          │────▶│  Swapped Out: P4, P5, P6                              │  │
    │  │ (Active)            │     │  (Suspended state)                                    │  │
    │  └─────────────────────┘     └─────────────────────────────────────────────────────────┘  │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
                                │
                                ▼
    [단기 스케줄러 (Short-term Scheduler / CPU Scheduler)]
    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  역할: Ready Queue → Running 상태로 전환할 프로세스 선택                                │
    │  실행 빈도: 밀리초 단위 (매우 높음)                                                      │
    │                                                                                         │
    │  [스케줄링 알고리즘]                                                                     │
    │  • FCFS (First-Come, First-Served)                                                      │
    │  • SJF (Shortest Job First)                                                             │
    │  • Priority Scheduling                                                                  │
    │  • Round Robin                                                                         │
    │  • Multilevel Queue                                                                     │
    │  • Multilevel Feedback Queue                                                            │
    │                                                                                         │
    │  Ready Queue                  CPU                                                       │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  P1 ▶ P2 ▶ P3 ▶ P4 ▶ P5 ▶ P6 ▶ ...                                                 │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │       │                                                   │                             │
    │       └───────────────────────────────────────────────────┘                             │
    │                         Scheduler 선택 → Dispatcher 할당                                  │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    [디스패처 (Dispatcher)]
    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  역할: 스케줄러가 선택한 프로세스를 실제 CPU에 할당                                       │
    │                                                                                         │
    │  [Dispatcher 동작]                                                                      │
    │  1. 문맥 교환 (Context Switch)                                                          │
    │  2. 사용자 모드 전환 (User Mode Switch)                                                 │
    │  3. 프로그램으로 점프 (Jump to Program)                                                  │
    │                                                                                         │
    │  [Dispatcher Latency]                                                                   │
    │  • 문맥 교환에 소요되는 시간                                                            │
    │  • 짧을수록 좋음 (~수 µs)                                                               │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
```

### 스케줄링 알고리즘 비교

| 알고리즘 | 특징 | 장점 | 단점 | 적용 |
|----------|------|------|------|------|
| **FCFS** | 도착 순서 | 구현 간단 | Convoy Effect | 일괄처리 |
| **SJF** | 실행 시간 최소 | 평균 반환 시간 최소 | 기아 현상 | 배치 |
| **Priority** | 우선순위 | 중요 작업 우선 | 기아 현상 | 실시간 |
| **RR** | Time Slice | 공정한 할당 | Context Switch 오버헤드 | 시분할 |
| **MLFQ** | 다중 큐 + 피드백 | 다양한 작업 최적화 | 복잡함 | 범용 |

### Round Robin 상세 동작

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Round Robin 스케줄링                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Ready Queue (Time Quantum = 4ms)

    초기 상태:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  P1(24) ▶ P2(3) ▶ P3(3)                                                               │
    │  ( ) 안은 남은 CPU 버스트 시간                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Time 0~4ms: P1 실행 (4ms 사용)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU: P1 사용 (0~4ms)                                                                  │
    │  P1 남은 시간: 20ms                                                                    │
    │  Ready Queue: P2(3) ▶ P3(3) ▶ P1(20)                                                  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Time 4~7ms: P2 실행 (3ms 사용 → 완료)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU: P2 사용 (4~7ms)                                                                  │
    │  P2 완료 (Terminated)                                                                  │
    │  Ready Queue: P3(3) ▶ P1(20)                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Time 7~10ms: P3 실행 (3ms 사용 → 완료)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU: P3 사용 (7~10ms)                                                                 │
    │  P3 완료 (Terminated)                                                                  │
    │  Ready Queue: P1(20)                                                                  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Time 10~30ms: P1 계속 실행 (20ms 사용 → 완료)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU: P1 사용 (10~30ms)                                                                │
    │  P1 완료 (Terminated)                                                                  │
    │  Ready Queue: (비어있음)                                                               │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [결과]
    • P1: 30ms 완료 (Turnaround Time = 30ms)
    • P2: 7ms 완료 (Turnaround Time = 7ms)
    • P3: 10ms 완료 (Turnaround Time = 10ms)
    • 평균 Turnaround Time = (30+7+10)/3 = 15.67ms
```

## Ⅲ. 융합 비교 및 다각도 분석

### 스케줄링 알고리즘 성능 비교

**예시 프로세스**:
| 프로세스 | 도착 시간 | 버스트 시간 |
|----------|----------|-----------|
| P1 | 0 | 8 |
| P2 | 1 | 4 |
| P3 | 2 | 9 |
| P4 | 3 | 5 |

| 알고리즘 | 평균 반환 시간 | 평균 대기 시간 |
|----------|---------------|---------------|
| **FCFS** | 13.0 | 7.75 |
| **SJF(비선점)** | 10.25 | 5.0 |
| **SRTF(선점)** | 8.75 | 3.5 |
| **RR(Q=4)** | 12.0 | 6.75 |

### 과목 융합 관점 분석

#### 1. 컴퓨터 구조 ↔ 스케줄러
- **캐시 친화성**: 동일 프로세스 계속 실행
- **파이프라인**: 컨텍스트 스위치 비용
- **멀티코어**: Load Balancing

#### 2. 실시간 시스템 ↔ 스케줄러
- **Rate Monotonic**: 주기가 짧을수록 높은 우선순위
- **EDF**: 마감 기한이 가장 빠른 작업 우선

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 웹 서버 스케줄링 선택
**상황**: 다수의 짧은 요청 처리
**판단**:
1. **Round Robin**: Time Quantum 10-50ms
2. **멀티레벨 큐**:
   - 대화형 작업: 높은 우선순위
   - 배치 작업: 낮은 우선순위
3. **CPU Affinity**: 특정 CPU에 바인딩

```bash
# Linux에서 스케줄러 설정
# chrt: 변경 실시간 우선순위
chrt -f -p 50 <pid>  # FIFO, priority 50

# taskset: CPU affinity 설정
taskset -p 0x3 <pid>  # CPU 0,1에만 할당
```

## Ⅴ. 기대효과 및 결론

### 스케줄러 최적화 기대 효과

| 최적화 방법 | 기대 효과 | 부작용 |
|------------|----------|--------|
| **Time Quantum 증가** | Context Switch 감소 | 응답 시간 증가 |
| **MLFQ 사용** | 다양한 워크로드 최적화 | 복잡도 증가 |
| **CPU Affinity** | 캐시 효율 | 로드 밸런싱 저하 |
| **CPU Share 제한** | 공정성 | 처리량 감소 |

### 미래 전망

1. **CFS**: Completely Fair Scheduler (Linux)
2. **BPF**: 유저레벨 스케줄링 로직
3. **Container Scheduler**: Kubernetes, Docker

### ※ 참고 표준/가이드
- **POSIX**: Scheduling policies
- **Linux Kernel**: kernel/sched/
