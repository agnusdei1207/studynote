# 스케줄링 평가 기준 (Scheduling Criteria)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스케줄링 평가 기준은 **"스케줄링 알고리즘의 성능을 측정하는 지표"**로, **CPU 이용률(Utilization)**, **처리량(Throughput)**, **반환 시간(Turnaround Time)**, **대기 시간(Waiting Time)**, **응답 시간(Response Time)**의 5대 지표로 평가한다.
> 2. **가치**: **시스템 관점**(CPU 이용률, 처리량)과 **사용자 관점**(반환 시간, 대기 시간, 응답 시간)의 **Trade-off(상충 관계)**를 최적화하며, **Time Sharing** 시스템에서는 응답 시간을, **Batch System**에서는 처리량을 최우선한다.
> 3. **융합**: **Linux CFS(Completely Fair Scheduler)**의 **vruntime**(가상 실행 시간), **Windows Scheduler**의 **CPU Cycle**, **Real-time OS**의 **Deadline Miss Rate**로 구현되며, **Kubernetes**, **Docker**의 **CPU Quota**, **Share** 할당의 기준이 된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스케줄링 평가 기준(Scheduling Criteria)은 **"스케줄링 알고리즘의 성능을 정량적으로 평가하는 척도"**이다.

**5대 평가 기준**:
1. **CPU 이용률(Utilization)**: CPU가 사용된 시간의 비율
2. **처리량(Throughput)**: 단위 시간당 완료된 프로세스 수
3. **반환 시간(Turnaround Time)**: 프로세스가 완료될 때까지의 총 시간
4. **대기 시간(Waiting Time)**: Ready Queue에서 대기한 시간
5. **응답 시간(Response Time)**: 첫 응답까지의 시간

### 💡 비유
스케줄링 평가는 **"은행 창구 성과"**와 같다.
- **이용률**: 창구가 열린 시간 비율
- **처리량**: 1시간당 처리한 고객 수
- **대기 시간**: 번호표 받고 기다린 시간
- **응답 시간**: 창구 도착 후 처음 받은 서비스 시간

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스케줄링 평가 기준의 발전                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

일괄처리 (1950년대):
    • 처리량(Throughput) 중심
    • 반환 시간 중요
         ↓
시분할 (1960년대):
    • 응답 시간(Response Time) 중요
    • 대화형 서비스 품질
         ↓
실시간 시스템 (1970년대):
    • Deadline 준수
    • 예측 가능성(Predictability)
         ↓
멀티코어 (2000년대):
    • 로드 밸런싱
    • Energy-aware scheduling
```

## Ⅱ. 아키텍처 및 핵심 원리

### 5대 평가 기준 상세

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스케줄링 평가 기준 5대 지표                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [예시 프로세스]

    Process   Arrival Time   Burst Time   Start Time   Completion Time
    ─────────────────────────────────────────────────────────────────────────
       P1         0              8            0              8
       P2         1              4            8              12
       P3         2              9            12             21
       P4         3              5            21             26

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. CPU 이용률 (CPU Utilization)                                                       │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  정의: CPU가 사용된 시간 / 전체 시간                                                     │
    │                                                                                         │
    │  CPU Utilization = (CPU 사용 시간) / (전체 시간) × 100%                                │
    │                                                                                         │
    │  예시:                                                                                   │
    │  • 전체 시간: 26ms                                                                      │
    │  │  Idle 시간: 0ms                                                                      │
    │  │  CPU Utilization = 26/26 × 100% = 100%                                              │
    │                                                                                         │
    │  목표: 최대화 (100%에 가깝게)                                                            │
    │  상충: 너무 높으면 대기 시간 증가                                                         │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  2. 처리량 (Throughput)                                                                 │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  정의: 단위 시간당 완료된 프로세스 수                                                     │
    │                                                                                         │
    │  Throughput = (완료된 프로세스 수) / (전체 시간)                                        │
    │                                                                                         │
    │  예시:                                                                                   │
    │  • 완료된 프로세스: 4개                                                                 │
    │  │  전체 시간: 26ms                                                                     │
    │  │  Throughput = 4 / 26ms = 0.154 processes/ms                                           │
    │  │                    = 154 processes/sec                                               │
    │                                                                                         │
    │  목표: 최대화                                                                            │
    │  적용: Batch System 중심                                                                │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  3. 반환 시간 (Turnaround Time)                                                         │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  정의: 프로세스가 완료될 때까지의 총 시간                                                 │
    │                                                                                         │
    │  Turnaround Time = Completion Time - Arrival Time                                      │
    │                      = (Waiting Time) + (Burst Time)                                     │
    │                                                                                         │
    │  예시:                                                                                   │
    │  • P1: 8 - 0 = 8ms                                                                      │
    │  │  P2: 12 - 1 = 11ms                                                                   │
    │  │  P3: 21 - 2 = 19ms                                                                   │
    │  │  P4: 26 - 3 = 23ms                                                                   │
    │  │  Average = (8+11+19+23)/4 = 15.25ms                                                   │
    │                                                                                         │
    │  목표: 최소화                                                                            │
    │  적용: Batch System 중심                                                                │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  4. 대기 시간 (Waiting Time)                                                            │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  정의: Ready Queue에서 대기한 시간                                                       │
    │                                                                                         │
    │  Waiting Time = Turnaround Time - Burst Time                                           │
    │               = (CPU 할당 대기 시간의 합)                                                 │
    │                                                                                         │
    │  예시:                                                                                   │
    │  • P1: 8 - 8 = 0ms                                                                      │
    │  │  P2: 11 - 4 = 7ms                                                                    │
    │  │  P3: 19 - 9 = 10ms                                                                   │
    │  │  P4: 23 - 5 = 18ms                                                                   │
    │  │  Average = (0+7+10+18)/4 = 8.75ms                                                     │
    │                                                                                         │
    │  목표: 최소화                                                                            │
    │  적용: 일반적 척도                                                                       │
    └─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  5. 응답 시간 (Response Time)                                                           │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  정의: 프로세스가 첫 응답을 받을 때까지의 시간                                            │
    │                                                                                         │
    │  Response Time = (첫 CPU 할당 시간) - (도착 시간)                                       │
    │                                                                                         │
    │  예시:                                                                                   │
    │  • P1: 0 - 0 = 0ms                                                                      │
    │  │  P2: 8 - 1 = 7ms                                                                     │
    │  │  P3: 12 - 2 = 10ms                                                                   │
    │  │  P4: 21 - 3 = 18ms                                                                   │
    │  │  Average = (0+7+10+18)/4 = 8.75ms                                                     │
    │                                                                                         │
    │  목표: 최소화 (Time Sharing 시스템에서 중요)                                             │
    │  적용: Interactive System                                                              │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
```

### 시스템 유형별 평가 기준 중요도

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         시스템 유형별 평가 기준 중요도                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Batch System]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  중요도 순위:                                                                             │
    │  1. Throughput (처리량): 최대화                                                         │
    │  2. CPU Utilization (이용률): 최대화                                                     │
    │  3. Turnaround Time (반환 시간): 최소화                                                  │
    │  4. Waiting Time (대기 시간): 최소화                                                     │
    │  5. Response Time (응답 시간): 중요하지 않음                                              │
    │                                                                                         │
    │  특징:                                                                                   │
    │  • 대용량 작업 처리                                                                      │
    │  │  사용자 상호작용 적음                                                                  │
    │  │  전체 처리량 중심                                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Interactive System]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  중요도 순위:                                                                             │
    │  1. Response Time (응답 시간): 최소화 ★★★                                              │
    │  2. Waiting Time (대기 시간): 최소화                                                      │
    │  3. CPU Utilization (이용률): 높음                                                       │
    │  4. Throughput (처리량): 적절                                                            │
    │  5. Turnaround Time (반환 시간): 적절                                                     │
    │                                                                                         │
    │  특징:                                                                                   │
    │  • 사용자와 직접 상호작용                                                                │
    │  │  빠른 응답이 사용자 경험에 큰 영향                                                     │
    │  │  Time Sharing 중요                                                                    │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Real-time System]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  중요도 순위:                                                                             │
    │  1. Deadline Miss Rate: 0% ★★★                                                         │
    │  2. Response Time (응답 시간): Deadline 내                                                │
    │  3. Predictability (예측 가능성): 높음                                                   │
    │  4. CPU Utilization (이용률): 적절                                                       │
    │  5. Throughput (처리량): 적절                                                            │
    │                                                                                         │
    │  특징:                                                                                   │
    │  • Deadline 준수가 최우선                                                                │
    │  │  예측 가능성이 중요                                                                   │
    │  │  Hard vs Soft Real-time                                                               │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 평가 기준 간 Trade-off

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         평가 기준 간 상충 관계(Trade-off)                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [CPU Utilization vs Response Time]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU Utilization ↑                                                                      │
    │       │                                                                                 │
    │       │  최대화하면                                                                       │
    │       ▼                                                                                 │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  • 프로세스 수 증가                                                                  │  │
    │  │  • Context Switch 빈번                                                               │  │
    │  │  │  Response Time 증가                                                             │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  → Utilization 100%는 응답 시간 길어짐                                                    │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Throughput vs Turnaround Time]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  SJF(Shortest Job First)                                                                 │
    │  • Throughput: 높음 (짧은 작업 빨리 완료)                                               │
    │  │  Turnaround Time: 평균 낮음                                                           │
    │  │  하지만 긴 작업의 Turnaround Time 매우 김 (기아 현상)                                 │
    │                                                                                         │
    │  Round Robin                                                                             │
    │  • Throughput: 중간                                                                     │
    │  │  Turnaround Time: SJF보다 높음                                                        │
    │  │  하지만 공정하게 모든 프로세스 처리                                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Response Time vs Throughput]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Time Quantum 작게                                                                        │
    │  • Response Time: 짧음                                                                  │
    │  │  Throughput: 낮음 (Context Switch 오버헤드)                                          │
    │                                                                                         │
    │  Time Quantum 크게                                                                       │
    │  • Response Time: 김                                                                    │
    │  │  Throughput: 높음 (오버헤드 감소)                                                      │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 스케줄링 알고리즘별 성능 비교

| 알고리즘 | CPU Util | Throughput | Turnaround | Waiting | Response |
|----------|----------|-----------|------------|---------|----------|
| **FCFS** | 높음 | 중간 | 김 | 김 | 김 |
| **SJF** | 높음 | 높음 | 짧음 | 짧음 | 김 |
| **SRTF** | 높음 | 높음 | 가장 짧음 | 가장 짧음 | 중간 |
| **Priority** | 높음 | 중간 | 중간 | 중간 | 중간 |
| **RR** | 높음 | 중간 | 김 | 김 | 짧음 |

### 과목 융합 관점 분석

#### 1. 성능 공학 ↔ 스케줄링 평가
- **벤치마크**: SPEC CPU, TPC-C
- **프로파일링**: perf, top, htop
- **SLA**: Service Level Agreement

#### 2. 클라우드 ↔ 스케줄링 평가
- **Resource Quota**: CPU Share 제한
- **Auto-scaling**: Load-based
- **Fair Share**: Multi-tenant

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 웹 서버 성능 최적화
**상황**: 높은 응답 시간 문제
**판단**:
1. **측정**:
   - Response Time: p50, p95, p99
   - Throughput: RPS (Request Per Second)
   - CPU Util: vmstat, mpstat
2. **분석**:
   - Context Switch 빈번?
   - Time Quantum 너무 길음?
   - Queue 길이?
3. **최적화**:
   - Nginx Worker Processes = CPU 코어 수
   - Time Quantum = 50ms
   - Connection Pool 최적화

```bash
# Linux 스케줄링 통계 확인
vmstat 1
# procs: r(run), b(block)
# cpu: us(user), sy(system), id(idle)

# Context Switch 확인
pidstat -w 1

# 스케줄러 지연
schedtool -n 10 <pid>  # nice 값 증가 (우선순위 낮춤)
```

## Ⅴ. 기대효과 및 결론

### 스케줄링 평가 기준 활용 효과

| 목적 | 우선 기준 | 최적화 방향 |
|------|----------|-----------|
| **대용량 처리** | Throughput | Context Switch 감소 |
| **빠른 응답** | Response Time | Time Quantum 축소 |
| **실시간 보장** | Deadline | Rate Monotonic |
| **공정한 할당** | Fairness | CFS, MLFQ |

### 미래 전망

1. **AI 기반 스케줄링**: 워크로드 예측 기반
2. **Energy-aware**: 전력 소모 최소화
3. **Container Scheduler**: Kubernetes, Nomad

- **POSIX**: Scheduling metrics
