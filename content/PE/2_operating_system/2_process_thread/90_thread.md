# 스레드 (Thread)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스레드는 **"프로세스 내의 **실행 단위(Execution Unit)**"**로, **독립적인 Stack**과 **Program Counter**를 가지지만 **Code, Data, Heap, File Descriptors**를 **프로세스 내 스레드 간 공유**하는 **경량 프로세스(Lightweight Process)**이다.
> 2. **가치**: **Context Switch** 비용이 프로세스보다 **낮아** **빠른 전환**이 가능하고 **메모리 공유**로 **IPC(Inter-Process Communication)** 오버헤드를 줄이며 **Multiprocessor** 환경에서 **병렬 실행(Parallel Execution)**이 가능하다.
> 3. **융합**: **Pthread(POSIX Thread)**, **Java Thread**, **C++ std::thread**, **Go goroutine**으로 구현되며, **Web Server**(Apache Thread, Nginx Worker), **Database Connection Pool**, **GPU Computing**(CUDA Thread)에서 핵심 역할을 한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스레드는 **"CPU 이용의 기본 단위로, 프로세스 내에서 실행되는 흐름"**이다.

**스레드의 특징**:
- **독립 요소**: PC, Stack, Registers
- **공유 요소**: Code, Data, Heap, Files
- **경량성**: 생성/전환 비용 낮음

### 💡 비유
스레드는 **"회사의 팀원"**과 같다.
- **프로세스**: 회사 (자원 공유)
- **스레드**: 직원 (독립 작업)
- **공유 자원**: 사무실, 장비

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스레드의 발전                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

단일 스레드 프로세스:
    • 프로세스 = 실행 단위
    • 순차 실행만 가능
         ↓
Multithreading 등장 (1960년대):
    • CTSS, Multics
    • 여러 실행 흐름
         ↓
Modern Threads (1990년대):
    • POSIX Threads (pthreads)
    • Win32 Threads
    • Java Threads
         ↓
현대:
    • Native Threads (Kernel-level)
    • Green Threads (User-level)
    • Goroutines, Fibers
```

## Ⅲ. 융합 비교 및 다각도 분석

### Process vs Thread Context Switch

| 구분 | Process | Thread |
|------|---------|--------|
| **Context** | PCB 전체 | 일부 레지스터, Stack |
| **TLB Flush** | 필요 | 불필요 (같은 주소 공간) |
| ** overhead** | 높음 | 낮음 |
| **시간** | 수 마이크로초 | 수백 나노초 |

### 과목 융합 관점 분석

#### 1. 데이터베이스 ↔ 스레드
- **Connection Pool**: 스레드별 연결
- **Query Execution**: 병렬 처리

#### 2. 웹 서버 ↔ 스레드
- **Apache Prefork**: 프로세스 풀
- **Apache Worker**: 스레드 풀
- **Nginx**: Worker 스레드

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 멀티스레드 웹 서버
**상황**: 동시 요청 처리
**판단**:

```c
// Thread Pool 구조
#include <pthread.h>

#define MAX_THREADS 4
#define QUEUE_SIZE 100

typedef struct {
    void (*function)(void*);
    void* arg;
} Task;

Task task_queue[QUEUE_SIZE];
int queue_head = 0, queue_tail = 0;

pthread_mutex_t queue_mutex = PTHREAD_MUTEX_INITIALIZER;
pthread_cond_t queue_cond = PTHREAD_COND_INITIALIZER;

void* worker_thread(void* arg) {
    while (1) {
        pthread_mutex_lock(&queue_mutex);
        while (queue_head == queue_tail) {
            pthread_cond_wait(&queue_cond, &queue_mutex);
        }
        Task task = task_queue[queue_head];
        queue_head = (queue_head + 1) % QUEUE_SIZE;
        pthread_mutex_unlock(&queue_mutex);

        task.function(task.arg);
    }
    return NULL;
}

// Main: 4개 Worker 스레드 생성
// Request: Task 큐에 추가
```

## Ⅴ. 기대효과 및 결론

### 스레드 기대 효과

| 효과 | 단일 스레드 | 멀티스레드 |
|------|----------|----------|
| **CPU 활용** | 단일 코어만 | 다중 코어 병렬 |
| **응답성** | Blocking 시 지연 | 비동기 처리 |
| **오버헤드** | 없음 | Context Switch |
| **동기화** | 불필요 | 필요 (Race Condition) |

### 미래 전망

1. **协程 (Coroutines)**: User-level Light
2. **Actor Model**: 메시지 전달
3. **Fibers**: Cooperative Scheduling

- **POSIX 1003.1c**: pthread
