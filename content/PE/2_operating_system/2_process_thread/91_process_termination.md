# 프로세스 종료 (Process Termination)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 종료는 **"프로세스가 실행을 멈추고 **자원을 반환**하는 과정"**으로, **자발적 종료(Voluntary Exit)**와 **비자발적 종료(Involuntary Termination)**가 있으며, **exit() 시스템 콜**로 **Kernel에게 종료**를 알리고 **부모 프로세스**는 **wait()**로 **자식 상태**를 수집한다.
> 2. **가치**: **자원 회수(Resource Reclamation)**으로 **메모리 누수**, **파일 디스크립터 누수**, **Zombie Process(좀비 프로세스)**를 방지하며, **Graceful Shutdown**으로 **데이터 저장**, **연결 종료**를 안전하게 처리한다.
> 3. **융합**: **Signal**(SIGTERM, SIGKILL)로 **외부 종료**가 가능하며, **Orphan Process(고아 프로세스)**는 **init(PID 1)**이 양육하고 **Zombie Process**는 **부모가 wait() 호출**로 제거한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
프로세스 종료는 **"프로세스의 생명주기를 마무리하고 자원을 반환하는 절차"**이다.

**종료의 유형**:
- **Normal Exit**: return, exit()
- **Error Exit**: abort(), Signal
- **Kill by OS**: OOM Killer
- **Kill by User**: SIGKILL

### 💡 비유
프로세스 종료는 **"퇴사 절차"**와 같다.
- **정상 퇴사**: exit(), 자원 반납
- **해고**: Signal 강제 종료
- **좀비**: 퇴사 처리 미완료

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세스 종료의 필요성                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

초기 시스템:
    • 전원 끄면 모든 종료
    • 자원 정리 없음
         ↓
Graceful Exit:
    • exit() 시스템 콜
    • 자원 반납
         ↓
현대 OS:
    • Signal 기반 종료
    • Zombie, Orphan 처리
    • OOM Killer
```

## Ⅱ. 아키텍처 및 핵심 원리

### 프로세스 종료 절차

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세스 종료 5단계                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [1. 종료 요청]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  자발적 (Voluntary):                                                                   │
    │  • main() return                                                                        │
    │  │  exit(status)                                                                        │
    │  │  _Exit(status)                                                                       │
    │                                                                                         │
    │  비자발적 (Involuntary):                                                                │
    │  • Signal 수신 (SIGTERM, SIGKILL)                                                        │
    │  │  • Kernel 강제 종료 (Segfault)                                                        │
    │  │  • 부모 종료                                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [2. 자원 해제]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 열린 파일 닫기 (fclose, close)                                                       │
    │  │  • 메모리 해제 (malloc free)                                                          │
    │  │  • Semaphore, Mutex 해제                                                             │
    │  │  • POSIX 메시지 큐, 공유 메모리 해제                                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [3. 자식 프로세스 처리]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 자식 프로세스에 SIGHUP 전송                                                          │
    │  │  • 자식이 고아(Orphan) 되면 init(PID 1)에 양입                                        │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [4. PCB 상태 변경]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • State: RUNNING → ZOMBIE                                                              │
    │  │  • Exit Status: 저장 (0~255)                                                         │
    │  │  • PCB 유지 (부모가 수집할 때까지)                                                    │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [5. 부모에게 통지]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • SIGCHLD Signal을 부모에게 전송                                                        │
    │  │  • 부모가 wait() 호출하면 PCB 해제                                                    │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### exit() vs _exit() vs return

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         종료 함수 비교                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [return (main 함수)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  int main() {                                                                           │
    │      // ...                                                                              │
    │      return 0;  // exit(0)와 동일                                                        │
    │  }                                                                                      │
    │                                                                                         │
    │  동작:                                                                                   │
    │  • stdio 버퍼 플러시                                                                    │
    │  │  atexit() 등록 함수 호출                                                             │
    │  │  exit() 호출                                                                        │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [exit()]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  #include <stdlib.h>                                                                    │
    │                                                                                         │
    │  void exit(int status);                                                                 │
    │                                                                                         │
    │  동작:                                                                                   │
    │  • stdio 버퍼 플러시 (fflush)                                                            │
    │  • atexit() 핸들러 호출                                                                 │
    │  • _exit() 호출                                                                        │
    │                                                                                         │
    │  사용: 정상 종료, 데이터 저장 필요                                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [_exit()]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  #include <unistd.h>                                                                    │
    │                                                                                         │
    │  void _exit(int status);                                                                │
    │                                                                                         │
    │  동작:                                                                                   │
    │  • 즉시 커널 모드 전환                                                                   │
    │  │  • 버퍼 플러시 없음                                                                   │
    │  │  • atexit() 호출 없음                                                                 │
    │                                                                                         │
    │  사용: fork() 후 자식 프로세스 종료 (exec 전)                                             │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  비교          │  return   │  exit()   │  _exit()  │                                   │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  버퍼 플러시   │  O        │  O        │  X         │                                   │
    │  atexit 호출   │  O        │  O        │  X         │                                   │
    │  사용 위치     │  main만   │  모든 곳 │  모든 곳  │                                   │
    │  목적          │  정상 종료│  정상 종료│  즉시 종료│                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Zombie Process

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         좀비 프로세스 생성 및 제거                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [좀비 생성 과정]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. 자식 프로세스: exit() 호출                                                          │
    │  2. Kernel: PCB 상태를 ZOMBIE로 변경                                                     │
    │  3. Kernel: 부모에게 SIGCHLD 전송                                                        │
    │  4. 부모: wait() 호출 안 함 (좀비 발생!)                                                  │
    │                                                                                         │
    │  [ZOMBIE 상태]                                                                          │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  PID: 1001                                                                          │  │
    │  │  State: ZOMBIE (메모리 없음, PCB만 존재)                                           │  │
    │  │  Exit Status: 0                                                                     │  │
    │  │  부모가 wait()로 수집 대기                                                           │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [좀비 제거]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  #include <sys/wait.h>                                                                  │
    │                                                                                         │
    │  int main() {                                                                           │
    │      pid_t pid = fork();                                                                │
    │                                                                                         │
    │      if (pid == 0) {  // Child                                                         │
    │          printf("Child running\n");                                                     │
    │          exit(0);  // 좀비 대기                                                         │
    │      } else {  // Parent                                                                │
    │          // wait() 없으면 좀비 발생                                                     │
    │          sleep(10);  // 고의로 좀비 생성                                                │
    │                                                                                         │
    │          int status;                                                                    │
    │          wait(&status);  // 좀비 수집                                                  │
    │          printf("Child reaped\n");                                                      │
    │      }                                                                                  │
    │      return 0;                                                                          │
    │  }                                                                                      │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [좀비 확인]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  $ ps aux | grep Z                                                                      │
    │  user  1001  0.0  0.0   0:00.00 ?        Z    10:20   0:00 [myprog] <defunct>           │
    │                                                                                         │
    │  # 좀비 제거: 부모 프로세스 종료 또는 wait() 호출                                      │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Orphan Process

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         고아 프로세스                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [고아 생성 과정]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. 자식 프로세스: 실행 중                                                               │
    │  2. 부모 프로세스: exit() 또는 강제 종료                                                   │
    │  3. Kernel: 자식의 PPID를 1(init)로 변경                                                   │
    │  4. init: 고아를 양육 (주기적으로 wait())                                                 │
    │                                                                                         │
    │  [Orphan 상태]                                                                           │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  PID: 1002                                                                          │  │
    │  │  PPID: 1 (원래 부모 종료 후 변경)                                                   │  │
    │  │  State: RUNNING (계속 실행)                                                         │  │
    │  │  양부: init (PID 1)                                                                  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  #include <stdio.h>                                                                     │
    │  #include <unistd.h>                                                                    │
    │  #include <stdlib.h>                                                                     │
    │                                                                                         │
    │  int main() {                                                                           │
    │      pid_t pid = fork();                                                                │
    │                                                                                         │
    │      if (pid == 0) {  // Child                                                          │
    │          sleep(10);  // 부모보다 오래 생존                                               │
    │          printf("Child: PPID = %d\n", getppid());  // init(1)이 됨                      │
    │          exit(0);                                                                       │
    │      } else {  // Parent                                                                │
    │          printf("Parent: exiting early\n");                                             │
    │          exit(0);  // 자식보다 먼저 종료 → 고아 생성                                    │
    │      }                                                                                  │
    │  }                                                                                      │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Signal로 종료

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Signal 기반 종료                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Signal           │  동작                            │  기본 핸들러                    │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  SIGTERM (15)     │  Graceful termination 요청        │  종료 (cleanup 가능)         │
    │  SIGKILL (9)      │  강제 종료                        │  즉시 종료 (무시 불가)      │
    │  SIGINT (2)       │  Interrupt (Ctrl+C)               │  종료                         │
    │  SIGHUP (1)       │  Hangup (터미널 종료)              │  종료                         │
    │  SIGSEGV (11)     │  Segmentation Fault               │  Core dump 후 종료            │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  #include <signal.h>                                                                    │
    │  #include <stdlib.h>                                                                    │
    │                                                                                         │
    │  void sigterm_handler(int sig) {                                                         │
    │      printf("Received SIGTERM, cleaning up...\n");                                       │
    │      // 자원 정리                                                                        │
    │      exit(0);                                                                            │
    │  }                                                                                      │
    │                                                                                         │
    │  int main() {                                                                           │
    │      signal(SIGTERM, sigterm_handler);  // SIGTERM 핸들러 등록                          │
    │                                                                                         │
    │      while (1) {  // 무한 루프                                                          │
    │          pause();  // Signal 대기                                                        │
    │      }                                                                                  │
    │      return 0;                                                                          │
    │  }                                                                                      │
    │                                                                                         │
    │  # 사용: kill -SIGTERM <PID> 또는 kill <PID>                                             │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Zombie vs Orphan

| 구분 | Zombie | Orphan |
|------|--------|--------|
| **상태** | 종료됨, PCB만 존재 | 실행 중 |
| **부모** | 살아있음 | 종료됨 |
| **PPID** | 원래 부모 | 1 (init) |
| **해결** | 부모 wait() | init 양육 |

### 과목 융합 관점 분석

#### 1. 데몬 ↔ 종료
- **Double Fork**: Orphan 데몬 생성
- **init 양육**: Session leader

#### 2. Container ↔ 종료
- **PID Namespace**: Container init
- **Signal Propagation**: Container Stop

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 좀비 프로세스 제거
**상황**: 좀비 프로세스 누적
**판단**:

```bash
# 좀비 확인
ps aux | grep Z

# 부모 프로세스 찾기
ps -o ppid= -p <zombie_pid>

# 해결 1: 부모 프로세스에 SIGCHLD 전송
kill -CHLD <parent_pid>

# 해결 2: 부모 프로세스 종료 (좀비 init 양육)
kill <parent_pid>

# 해결 3: 좀비 방지 코드
#include <sys/wait.h>
#include <signal.h>

void sigchld_handler(int sig) {
    while (waitpid(-1, NULL, WNOHANG) > 0);  // 모든 좀비 수집
}

int main() {
    signal(SIGCHLD, sigchld_handler);
    // ...
}
```

## Ⅴ. 기대효과 및 결론

### 프로세스 종료 기대 효과

| 효과 | 정상 종료 | 비정상 종료 |
|------|----------|-----------|
| **자원 해제** | 완전 | 불완전 |
| **데이터 저장** | 가능 | 불가능 |
| **좀비 생성** | 방지 | 가능 |
| **안정성** | 높음 | 낮음 |

### 미래 전망

1. **Graceful Shutdown**: Kubernetes Pod
2. **Cgroup Kill**: OUSD 제어
3. **Userfaultfd**: User-space Page Fault
