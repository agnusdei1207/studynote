# 스레드(Thread)와 병행 처리

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스레드는 **"CPU 스케줄링의 **최소 단위"**로, **프로세스 내에서 **독립적인 **실행 흐름**을 가지며 **Code, Data, Heap**은 **공유**하고 **Stack, PC, Registers**는 **독립적**이다.
> 2. **가치**: **병렬 처리**(Parallelism)로 **멀티코어**를 **활용**하고 **동시성**(Concurrency)으로 **I/O 대기 시간**을 **줄이**며 **응답성**과 **처리량**을 **향상**한다.
> 3. **융합**: **Kernel-Level Thread**(1:1), **User-Level Thread**(N:1), **Hybrid Thread**(M:N) 모델이 **존재**하며 **POSIX pthread**, **Java Thread**, **C++ std::thread**, **Go Goroutine**으로 **구현** 가능하다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스레드는 **"프로세스 내의 실행 단위"**이다.

**스레드 vs 프로세스**:
- **Process**: 독립적 자원
- **Thread**: Process 내 공유

### 💡 비유
스레드는 **"작업장의 작업자"**와 같다.
- **Process**: 공장 (자원)
- **Thread**: 작업자 (실행 단위)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스레드의 발전                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Single-Thread:
    • 한 작업만 실행
    • CPU 유휴
         ↓
Multi-Process:
    • 여러 프로세스
    • Context Switch 비용
         ↓
Multi-Thread (1990년대):
    • 가벼운 Context Switch
    • 자원 공유
```

## Ⅲ. 융합 비교 및 다각도 분석

### 스레드 vs 프로세스

| 구분 | Process | Thread |
|------|---------|--------|
| **자원** | 독립 | 공유 |
| **Context Switch** | 비용 큼 | 비용 작음 |
| **통신** | IPC (복잡) | Memory (간단) |
| **격리** | 강함 | 약함 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 스레드 생성
**상황**: 병렬 처리
**판단**:

```c
// POSIX Thread (pthread)
#include <pthread.h>
#include <stdio.h>

void* thread_function(void* arg) {
    int id = *(int*)arg;
    printf("Thread %d running\n", id);
    return NULL;
}

int main() {
    pthread_t threads[5];
    int ids[5];

    // 스레드 생성
    for (int i = 0; i < 5; i++) {
        ids[i] = i;
        pthread_create(&threads[i], NULL, thread_function, &ids[i]);
    }

    // 스레드 대기
    for (int i = 0; i < 5; i++) {
        pthread_join(threads[i], NULL);
    }

    return 0;
}
```

## Ⅴ. 기대효과 및 결론

### 스레드 기대 효과

| 효과 | Single-Thread | Multi-Thread |
|------|-------------|--------------|
| **CPU 활용** | 하나의 Core | 모든 Core |
| **응답성** | I/O Block 시 느림 | 다른 Thread 계속 |
| **복잡도** | 낮음 | 높음 (동기화) |

### 스레드 모범 사례

1. **작업 분할**: 독립적 단위
2. **동기화**: 최소화
3. **Deadlock 회피**: 순서 통일
4. **Pool 사용**: 생성/소멸 비용 절감
5. **Lock-Free**: Atomic 연산

### 미래 전망

1. **Coroutine**: 가벼운 스레드
2. **Actor Model**: 메시지 전달
3. **Data Parallelism**: SIMD/GPU

### ※ 참고 표준/가이드
- **POSIX**: pthread
- **C++11**: std::thread
- **Java**: Thread, Runnable
