# 프로세스와 스레드 (Process vs Thread)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: **프로세스**는 **"자원 **할당 **단위"**로 **독립적인 **메모리 **공간**, **파일**, **장치**를 **갖고 **스레드**는 **"CPU **스케줄링 **단위"**로 **프로세스 **내에서 **메모리**를 **공유**하며 **Lightweight**(경량) **실행 **단위**이다.
> 2. **차이**: 프로세스는 **PCB**(Process Control Block), **독립적** 주소 공간, **IPC**(Inter-Process Communication) **필요**하지만 **스레드**는 **TCB**(Thread Control Block), **공유** 힙/데이터, **직접 **메모리 **접근**이 **가능**하다.
> 3. **선택**: **독립성**이 **필요**하면 **프로세스**(Chrome **프로세스**당 **탭**), **공유**가 **필요**하면 **스레드**(웹 **서버 **요청 **처리**)를 **사용**하며 **Context Switch** **비용**은 **프로세스**가 **스레드**보다 **크다**.

---

## Ⅰ. 개요 (Context & Background)

### 개념
프로세스는 **"실행 중인 프로그램"**, 스레드는 **"프로세스 내 실행 흐름"**이다.

**비교**:
| 구분 | 프로세스 | 스레드 |
|------|----------|--------|
| **정의** | 자원 할당 단위 | CPU 실행 단위 |
| **메모리** | 독립적 | 공유 (Code, Data, Heap) |
| **문맥 교환** | 큰 비용 | 작은 비용 |
| **통신** | IPC 필요 | 직접 공유 |

### 💡 비유
- **프로세스**: **"집"** (독립된 공간)
- **스레드**: **"가족 구성원"** (집 안에서 함께 생활)

## Ⅲ. 융합 비교 및 다각도 분석

### Context Switch 오버헤드

| 구분 | 스레드 | 프로세스 |
|------|--------|----------|
| **TLB Flush** | 불필요 (같은 주소 공간) | 필수 |
| **Cache Flush** | 일부만 | 전체 |
| **Register Save** | 적음 | 많음 |
| **Page Table** | 유지 | 교체 |

### Context Switch 비용

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Context Switch Cost                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Thread Switch (Same Process):                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Save: SP, PC, General Registers (~10 registers)                                │  │  │
    │  │  2. Update: Scheduler state                                                         │  │  │
    │  │  3. Restore: New thread's registers                                                 │  │  │
    │  │  → Cost: ~1-2 microseconds                                                          │  │  │
    │  │  → No TLB flush (same address space)                                                │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Process Switch:                                                                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Save: All registers (~50 registers)                                             │  │  │
    │  │  2. Save: PC, PSW                                                                    │  │  │
    │  │  3. Update: PCB                                                                     │  │  │
    │  │  4. Switch: Page Table (CR3 register on x86)                                         │  │  │
    │  │  5. Flush: TLB (all entries invalid)                                                 │  │  │
    │  │  6. Restore: New process's context                                                   │  │  │
    │  │  → Cost: ~5-20 microseconds                                                          │  │  │
    │  │  → TLB miss penalty after switch                                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### IPC vs Direct Memory Access

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Communication                                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Process A → Process B (IPC Required):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌─────────────┐                   ┌─────────────┐                                     │  │
    │  │  Process A  │                   │  Process B  │                                     │  │
    │  │  Data       │ ──→ Kernel ───→   │  Copy       │                                     │  │  │
    │  └─────────────┘                   └─────────────┘                                     │  │
    │  → System call required (copy overhead)                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Thread A ↔ Thread B (Direct Access):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Shared Heap                                                                         │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Thread A writes → shared_data.value = 42                                        │  │  │  │
    │  │  │  Thread B reads  → x = shared_data.value                                          │  │  │  │
    │  │  │  → Direct access, no system call                                                 │  │  │  │
    │  │  │  → Synchronization required (mutex)                                               │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 언어별 스레드 생성

```c
// POSIX Thread (pthread)
#include <pthread.h>

void* thread_func(void* arg) {
    int id = *(int*)arg;
    printf("Thread %d\n", id);
    return NULL;
}

int main() {
    pthread_t tid;
    int arg = 1;
    pthread_create(&tid, NULL, thread_func, &arg);
    pthread_join(tid, NULL);
    return 0;
}
```

```java
// Java Thread
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread running");
    }
}

// Or use Runnable
Thread t = new Thread(() -> System.out.println("Lambda thread"));
t.start();
```

```python
# Python threading
import threading

def worker():
    print("Thread working")

t = threading.Thread(target=worker)
t.start()
t.join()
```

## Ⅴ. 기대효과 및 결론

### 프로세스 vs 스레드 기대 효과

| 구분 | 프로세스 | 스레드 |
|------|----------|--------|
| **격리** | 완전 | 없음 |
| **생성 비용** | 큼 | 작음 |
| **통신** | IPC | 직접 |
| **Crash** | 독립 | 전체 영향 |

### 모범 사례

1. **Chrome**: 프로세스당 탭 (격리)
2. **Web Server**: 스레드 풀 (공유)
3. **Database**: 프로세스 + 스레드 혼합

### 미래 전망

1. **Fiber**: 경량 스레드
2. **Actor**: 메시지 전달
3. **Coroutine**: 비동기

### ※ 참고 표준/가이드
- **POSIX**: pthread
- **Windows**: Thread API
- **Linux**: clone()
