# 프로세스 생성 (Process Creation)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 생성은 **"기존 프로세스(Parent)가 새로운 프로세스(Child)를 생성하는 메커니즘"**으로, **UNIX/Linux**에서는 **fork()**로 **자식 프로세스**를 생성하고 **exec()**로 **새로운 프로그램**을 로드하며, **Copy-on-Write(COW)** 기술로 **메모리 복사 비용**을 최소화한다.
> 2. **가치**: **PID(Process ID)** 할당, **PPID(Parent PID)** 설정, **PCB(Process Control Block)** 생성, **자원 상속**(파일 디스크립터, 메모리)을 통해 **프로세스 계층 구조**를 형성하고 **부모-자식 간 통신(IPC)**을 가능하게 한다.
> 3. **융합**: **Linux fork/exec** 모델은 **Shell Command 실행**, **Web Server 요청 처리**(Apache Prefork), **Container**(Docker), **Microservice Architecture**의 기반이 되며, **Windows**는 **CreateProcess()** API를 사용한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
프로세스 생성은 **"운영체제가 새로운 프로세스를 시작하고 실행 환경을 구성하는 절차"**이다.

**프로세스 생성의 필수 요소**:
- **프로세스 식별자(PID)**: 고유 ID 할당
- **PCB 생성**: 프로세스 제어 블록 초기화
- **메모리 할당**: Code, Data, Heap, Stack
- **자원 상속**: 파일, 시그널, 환경 변수

### 💡 비유
프로세스 생성은 **"회사 지사 설립"**과 같다.
- **본사**: Parent Process
- **지사**: Child Process
- **자원 상속**: 정책, 시스템, 데이터 공유

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세스 생성의 발전                                           │
└─────────────────────────────────────────────────────────────────────────────────────────┘

초기 시스템 (1960년대):
    • 단일 프로세스만 실행
    • spawn 방식 (완전히 새로운 프로세스)
         ↓
UNIX fork/exec (1970년대):
    • fork(): 자식 프로세스 생성 (부모 복사)
    • exec(): 새로운 프로그램 로드
    • 유연한 프로세스 생성
         ↓
Copy-on-Write (1980년대):
    • 메모리 복사 지연
    • 페이지 단위 복사
    • 성능 최적화
         ↓
현대 시스템:
    • Linux: clone(), fork(), vfork()
    • Windows: CreateProcess()
    • Container: namespace, cgroup
```

## Ⅱ. 아키텍처 및 핵심 원리

### 프로세스 생성 절차

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세스 생성 6단계                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [1. 새로운 프로세스 식별자 할당]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • PID 할당 (운영체제 커널)                                                            │
    │  • PID 중복 확인                                                                       │
    │  • PPID 설정 (부모 프로세스 ID)                                                        │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [2. PCB(Process Control Block) 생성]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • task_struct (Linux) 생성                                                            │
    │  • 프로세스 상태: NEW → READY                                                          │
    │  • 스케줄링 정보 초기화                                                                │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [3. 주소 공간 생성]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Code Segment: 실행 파일 로드                                                         │
    │  • Data Segment: 전역 변수, 정적 변수                                                   │
    │  • Heap: 동적 할당 영역                                                                │
    │  • Stack: 지역 변수, 함수 호출                                                         │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [4. 자원 상속 및 초기화]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  상속 (Inheritance):                                                                   │
    │  • 열린 파일 디스크립터                                                                │
    │  • 시그널 핸들러                                                                       │
    │  • 파일 시스템 속성                                                                    │
    │  • 환경 변수                                                                           │
    │                                                                                         │
    │  초기화 (Initialization):                                                              │
    │  • CPU 레지스터 (PC=0, SP=Top)                                                         │
    │  • 통계 정보 (CPU 시간=0)                                                              │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [5. Ready Queue에 삽입]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 스케줄러 Ready Queue에 추가                                                         │
    │  • CPU 할당 대기                                                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [6. Context Switch 후 실행]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 스케줄러가 새 프로세스 선택                                                         │
    │  • Context Switch 발생                                                                 │
    │  • 첫 명령어 실행 시작                                                                 │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### UNIX fork() 시스템 콜

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         fork() 실행 과정                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Before fork()]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Parent Process                                                                         │
    │  • PID: 1000                                                                           │
    │  • Code, Data, Heap, Stack                                                             │
    │  • File Descriptors: 0, 1, 2                                                           │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
                              │
                              ▼ fork()
                              │
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  [fork() 실행]                                                                          │
    │  • 커널 모드 전환                                                                       │
    │  • 새 PID 할당 (1001)                                                                  │
    │  • PCB 복사 (task_struct)                                                            │
    │  • 메모리 복사 (Copy-on-Write)                                                         │
    │  • 부모에게는 자식 PID 반환                                                            │
    │  • 자식에게는 0 반환                                                                  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
                              │
                ┌─────────────┴─────────────┐
                ▼                           ▼
    ┌───────────────────────┐   ┌───────────────────────┐
    │  Parent Process       │   │  Child Process        │
    │  • PID: 1000          │   │  • PID: 1001          │
    │  • fork() → 1001      │   │  • fork() → 0         │
    │  (자식 PID 반환)       │   │  (0 반환)             │
    └───────────────────────┘   └───────────────────────┘

    [Copy-on-Write 메모리 구조]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Parent (PID: 1000)                    Child (PID: 1001)                               │
    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Code Page (Read-only, 공유)                                                 │  │
    │  │  ┌────────────────────────────────────────────────────────────────────────────┐ │  │
    │  │  │  printf("Hello\n");                                                     │ │  │
    │  │  │  return 0;                                                               │ │  │
    │  │  └────────────────────────────────────────────────────────────────────────────┘ │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  Data Page (COW)                                                                         │
    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  int x = 10;  →  초기에는 공유                                                │  │
    │  │                                                                                    │  │
    │  │  [Parent 쓰기 시]                                                                 │  │
    │  │  x = 20;  →  페이지 복사 발생                                                   │  │
    │  │  ┌────────────────────────────────────────────────────────────────────────────┐ │  │
    │  │  │  Parent: x = 20 (복사된 페이지)                                           │ │  │
    │  │  │  Child:  x = 10 (원본 페이지)                                             │ │  │
    │  │  └────────────────────────────────────────────────────────────────────────────┘ │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### fork() + exec() 조합

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         fork() + exec() 패턴                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Shell Command 실행 예시]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  $ ls -l                                                                                │
    │                                                                                         │
    │  [Shell Parent Process]                                                                 │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  1. fork() → Child 생성                                                            │  │
    │  │  2. Child에서 exec("/bin/ls", ["ls", "-l"]) 실행                                    │  │
    │  │  3. Child의 메모리 공간을 ls 프로그램으로 교체                                       │  │
    │  │  4. Parent은 wait()로 Child 종료 대기                                               │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [exec() 계열 함수]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • execl("/bin/ls", "ls", "-l", NULL)     →  리스트 인자                               │
    │  • execv("/bin/ls", ["ls", "-l"])        →  벡터 인자                                 │
    │  • execle(...)                           →  환경 변수 지정                             │
    │  • execvp("ls", ["ls", "-l"])            →  PATH 검색                                 │
    │                                                                                         │
    │  exec() 특징:                                                                           │
    │  • PID 유지 (fork()로 생성된 PID)                                                       │
    │  • 메모리 공간 완전 교체                                                                │
    │  • 열린 파일 디스크립터 상속 (close-on-exec 제외)                                        │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### fork() vs vfork() vs clone()

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         생성 방식 비교                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [fork()]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 자식 프로세스 생성                                                                  │
    │  • 메모리 공간 Copy-on-Write                                                           │
    │  • 부모와 자식이 독립적으로 실행                                                        │
    │                                                                                         │
    │  사용: 일반적인 프로세스 생성                                                           │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [vfork()]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 자식 생성 (메모리 복사 없음)                                                          │
    │  • 자식이 부모의 주소 공간 공유                                                         │
    │  • 자식이 exec() 또는 _exit() 호출 시까지 부모 블록                                      │
    │                                                                                         │
    │  사용: exec() 바로 호출 시 (성능 최적화)                                                  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [clone()]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 세밀한 제어 가능                                                                      │
    │  • 공유할 리소스 선택적 지정 (메모리, 파일, 시그널 등)                                   │
    │  • 스레드 생성(pthread_create는 clone 사용)                                          │
    │                                                                                         │
    │  사용: 스레드, Container 생성                                                           │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  특징           │  fork()  │  vfork()  │  clone()                                      │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  메모리 복사     │  COW     │  공유     │  선택적                                        │
    │  부모 실행       │  병행    │  블록     │  병행                                          │
    │  주 용도         │  프로세스│  exec()   │  스레드, 컨테이너                             │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### UNIX vs Windows 프로세스 생성

| 구분 | UNIX/Linux | Windows |
|------|-----------|---------|
| **생성 API** | fork() + exec() | CreateProcess() |
| **단계** | 2단계 (생성+로드) | 1단계 (통합) |
| **메모리** | COW 복사 | 즉시 할당 |
| **PID 재사용** | 지연 재사용 | 즉시 재사용 |

### 과목 융합 관점 분석

#### 1. 스레드 ↔ 프로세스 생성
- **pthread_create**: clone() 기반
- **공유 메모리**: Heap, Data, Code
- **독립 Stack**: 각 스레드별

#### 2. 컨테이너 ↔ 프로세스 생성
- **namespace**: PID, Net, Mnt 격리
- **cgroup**: 자원 제한
- **clone()**: flags로 격리 지정

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: Web Server 프로세스 생성
**상황**: 동시 요청 처리
**판단**:

```bash
# Apache Prefork MPM
# 1. Main 프로세스가 자식 프로세스들 미리 생성 (fork)
# 2. 자식 프로세스들이 요청 대기 (accept)
# 3. 요청 오면 자식이 처리

# Nginx (Master-Worker)
# 1. Master 프로세스가 Worker 프로세스 생성
# 2. Worker들이 event-driven 방식으로 요청 처리

# Systemd 서비스
# 1. Forking 방식: Type=forking
# 2. Simple 방식: Type=simple (fork 없음)
```

## Ⅴ. 기대효과 및 결론

### 프로세스 생성 기대 효과

| 효과 | 단일 프로세스 | 다중 프로세스 |
|------|-------------|-------------|
| **병렬성** | 없음 | 있음 |
| **격리성** | - | 높음 (Crash 격리) |
| **IPC 오버헤드** | - | 있음 |
| **자원 소모** | 낮음 | 높음 |

### 미래 전망

1. **Container**: 경량 프로세스 격리
2. **Unikernel**: 단일 프로세스 OS
3. **Serverless**: 프로세스 생명주기 자동화

### ※ 참고 표준/가이드
- **POSIX**: fork(), exec(), wait()
- **Linux**: man 2 fork, man 2 execve
- **Windows**: CreateProcess API
