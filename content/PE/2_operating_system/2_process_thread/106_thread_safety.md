# 스레드 안전성 (Thread Safety)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스레드 안전성은 **"여러 **스레드가 **동시에 **공유 **데이터**에 **접근**해도 **정확성**을 **보장**하는 **속성"**으로, **Mutex**, **Semaphore**, **Monitor** 같은 **동기화 **기법**과 **Atomic Operation**, **Lock-Free** **자료구**로 **Race Condition**을 **방지**한다.
> 2. **문제**: **Race Condition**(경쟁 조건), **Deadlock**(교착 상태), **Priority Inversion**(우선순위 역전)가 **발생**하며 **volatile** 키워드는 **가시성**과 **가시성**을 **보장**하지 **못하며 **synchronized**(Java), **std::atomic**(C++)로 **해결**한다.
> 3. **융합**: **Java synchronized**, **Lock**, **ConcurrentHashMap**, **C++ std::atomic**, **Python threading.Lock**, **Go sync/atomic**가 **언어별 **안전**한 **동기화 **프라미터웍크**를 **제공**하며 **Thread-Local Storage**(TLS)로 **스레드 **지역 **변수**를 **격리**한다.

## Ⅰ. 개요 (Context & Background)

### 개념
스레드 안전성은 **"멀티스레드 환경에서 정확성 보장"**이다.

**스레드 안전성 위배 요소**:
- **Critical Section**: 공유 데이터 접근
- **Race Condition**: 동시 수정
- **Memory Visibility**: 메모리 가시성
- **Reordering**: 명령어 재배열

### 💡 비유
스레드 안전성은 ******여러 ****사람이 ****하나의 **장소**를 ****동시에 **쓰는 것****과 같다.
- **사람**: 스레드
- **장소**: 공유 데이터
- **충돌**: Race Condition
- **순서**: 동기화

## Ⅱ. 아키텍처 및 핵심 원리

### Race Condition 예시

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Race Condition Example                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Shared Variable: counter = 0                                                               │  │
    │                                                                                         │  │
    │  Thread 1:                    Thread 2:                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  register = counter;    register = counter;                                            │  │  │
    │  │  register = register + 1;  register = register + 1;  │  │  │
    │  │  counter = register;      counter = register;     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │    │                           │                                                        │  │
    │    ▼                           ▼                                                        │  │
    │  counter = 1 (WRONG!)         counter = 1 (WRONG!)                                        │  │
    │  → Should be 2!                                                                         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 해결 방법

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Thread Safety Solutions                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Mutex (상호 배제)                                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  pthread_mutex_lock(&lock);                                                          │  │  │
    │  │  counter++;                                                                            │  │  │
    │  │  pthread_mutex_unlock(&lock);                                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  2. Atomic Operation (Lock-Free)                                                           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  __sync_add(&counter, 1);     // C11 atomic                                   │  │  │
    │  │  counter.fetch_add(1);         // C++ std::atomic<int>                          │  │  │  │
    │  │  incrementAndGet(&counter);    // Java AtomicInteger                        │  │  │
    └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  3. Thread-Local Storage (지역 변수)                                                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  void* thread_func(void* arg) {                                                        │  │  │
    │  │      int local_var = *(int*)arg;  │  │  │
    │  │      // TLS: 각 스레드 독립                                                            │  │  │
    │  │      __thread int tid;           │  │  │
    │  │  }                                                                                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 동기화 기법 비교

| 기법 | 속도 | 복잡도 | Lock-Free |
|------|------|--------|----------|
| **Mutex** | 느림 | 낮음 | X |
| **Semaphore** | 느림 | 중간 | X |
| **Atomic** | 매우 빠름 | 높음 | O |
| **Lock-Free** | 가장 빠름 | 매우 높음 | O |

### volatile 문제점

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         volatile Misconception                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  volatile int counter = 0;                                                                │  │
    │                                                                                         │  │
    │  ❌ NOT thread-safe!                                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  counter++는 NOT atomic!                                                                │  │  │
    │  │   Read-Modify-Write race condition 발생 가능                                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  ✅ Use atomic_int or mutex                                                                │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: Atomic 카운터
**상황**: 고성능 카운터
**판단**: Lock-Free

```c
#include <stdatomic.h>

std::atomic<int> counter(0);

void increment() {
    counter.fetch_add(1, std::memory_order_relaxed);
}
```
