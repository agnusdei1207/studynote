# 프로그램 (Program) vs 프로세스 (Process)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로그램은 하드디스크에 저장된 **수동적(Passive) 코드**이며, 프로세스는 메모리에 적재되어 CPU에 의해 **능동적(Active)으로 실행 중인** 프로그램 인스턴스로, 운영체제는 프로세스 단위로 자원(CPU, 메모리)을 할당하고 스케줄링한다.
> 2. **가치**: 하나의 프로그램이 **여러 프로세스**로 동시 실행 가능(다중 인스턴스)하며, **PCB(Process Control Block)**라는 커널 자료구조로 각 프로세스의 상태(Context: 레지스터, PC, SP)를 저장하여 문맥 교환(Context Switching)을 가능하게 한다.
> 3. **융합**: 멀티태스킹 OS의 기본 단위이자, 가상화(VM), 컨테이너(Docker), 병렬 처리 기술의 근간이며, 프로세스 관리는 시스템 성능(Throughput), 응답 시간(Response Time), 자원 활용도(CPU Utilization)의 핵심 결정 요인이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념

**프로그램(Program)**:
- 디스크에 저장된 실행 파일 (Executable File)
- 수동적 존재 (Passive Entity)
- 명령어 집합 (Instruction Set)

**프로세스(Process)**:
- 메모리에 적재되어 실행 중인 프로그램
- 능동적 존재 (Active Entity)
- **Program + Current State + Allocated Resources**

### 💡 비유
프로그램과 프로세스는 **"요리책과 요리 중인 음식"**과 같다.
- **프로그램**: 책장에 꽂힌 요리책 (수동적)
- **프로세스**: 요리사가 조리 중인 음식 (능동적)
- **PCB**: 요리 중인 레시피, 현재 온도, 조리 시간 등 정보
- **문맥 교환**: 요리사가 다른 음식으로 전환하면 레시피 상태 저장 후 복귀

### 등장 배경 및 발전 과정

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세스 개념의 진화                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

1960년대: 단일 프로그래밍 (Single Programming)
         - 한 번에 하나의 프로그램만 실행
         - CPU 유휴 시간 낮음
         ↓
1970년대: 다중 프로그래밍 (Multiprogramming)
         - 메모리에 여러 프로그램 적재
         - 하나가 I/O 대기 시 다른 프로그램 실행
         - 프로세스 개념 탄생
         ↓
1980년대: 시분할 시스템 (Time-sharing)
         - 여러 사용자가 동시에 시스템 사용
         - 각 사용자에게 전용 프로세스 할당
         - TSS (Time Sharing System)
         ↓
1990년대: 멀티태스킹, 멀티스레딩
         - 하나의 CPU가 여러 프로세스 병렬 실행
         - 프로세스 내 스레드 개념 도입
         ↓
2000년대~현재: 멀티코어, 가상화, 컨테이너
         - 물리적 코어별 프로세스/스레드 할당
         - VM, Container로 프로세스 격리
```

## Ⅱ. 아키텍처 및 핵심 원리

### 프로그램 vs 프로세스 비교

| 비교 항목 | 프로그램 (Program) | 프로세스 (Process) |
|----------|-------------------|-------------------|
| **존재 형태** | 수동적 (Passive) | 능동적 (Active) |
| **저장 위치** | 디스크 (Secondary Storage) | 메모리 (RAM) |
| **수명 주기** | 영구적 | 생성 → 실행 → 종료 |
| **자원 할당** | 없음 | CPU, 메모리, 파일 등 |
| **상태** | 상태 없음 | New, Ready, Running, Waiting, Terminated |
| **목격성** | 공유 가능 (Read-only) | 독립적 (각각의 메모리 공간) |
| **예시** | `/bin/ls` (실행 파일) | `ps aux`로 확인되는 실행 인스턴스 |

### 정교한 구조 다이어그램

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로그램 → 프로세스 변환 과정                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Disk Storage]                  [Memory (RAM)]                 [CPU Execution]
    ┌──────────────────┐            ┌───────────────────────────────┐        ┌─────────────────┐
    │                  │            │                               │        │                 │
    │  Program File    │            │   Process 1   Process 2        │        │  Running        │
    │  (Executable)    │            │   ls -R        grep pattern   │        │  Process        │
    │                  │            │                               │        │                 │
    │  • Code Segment  │            │   ┌─────────┐  ┌─────────┐       │        │  CPU Core 0     │
    │  • Data Segment  │            │   │  Code    │  │  Code    │       │        │  (Current)      │
    │  • BSS Segment   │            │   │  Data    │  │  Data    │       │        │                 │
    │  (Text, .data,   │            │   │  Stack   │  │  Stack   │       │        │                 │
    │   .bss, .rodata) │            │   │  Heap    │  │  Heap    │       │        │                 │
    │                  │            │   └─────────┘  └─────────┘       │        └─────────────────┘
    │                  │            │       │            │             │              │
    │  ls              │────────────┼───────┼────────────┼─────────────┼──────────────┐
    │  grep            │────────────┼───────┼────────────┼─────────────┤              │
    │  firefox         │────────────┼───────┼────────────┼─────────────┤              │
    │                  │            │       │            │             │              │
    └──────────────────┘            └───────────────────────────────┘              │
                                                                              │
    [Process Creation]                                            [Ready Queue]
    ┌──────────────────────────────────────────────────────────────────────────────────┐
    │  1. Loader (OS) reads Program from disk                                      │
    │  2. Allocate Memory (Code, Data, BSS, Stack, Heap)                             │
    │  3. Initialize PCB (Process Control Block)                                     │
    │  4. Create First Thread (Main Thread)                                         │
    │  5. Set Initial State (PC = entry point, SP = stack top)                       │
    │  6. Add to Ready Queue                                                        │
    └──────────────────────────────────────────────────────────────────────────────────┘
```

### 프로세스 제어 블록 (PCB, Process Control Block)

**정의**: 운영체제가 각 프로세스를 관리하기 위한 **커널 내 자료구조**

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         PCB (Process Control Block) 구조                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌───────────────────────────────────────────────────────────────────────────────────┐
    │                              PCB                                                   │
    ├───────────────────────────────────────────────────────────────────────────────────┤
    │  1. 프로세스 식별자 (Process Identifier)                                        │
    │     • PID (Process ID): 고유 번호                                                 │
    │     • PPID (Parent PID): 부모 프로세스 ID                                         │
    │     • UID/GID: 사용자/그룹 ID                                                     │
    │                                                                                  │
    │  2. 프로세스 상태 (Process State)                                               │
    │     • New → Ready → Running → Waiting → Terminated                               │
    │                                                                                  │
    │  3. 프로그램 카운터 (Program Counter)                                            │
    │     • 다음 실행할 명령어 주소                                                     │
    │                                                                                  │
    │  4. CPU 레지스터 (CPU Registers)                                                 │
    │     • 범용 레지스터 (EAX, EBX, ECX, EDX, ESI, EDI)                               │
    │     • 포인터 레지스터 (ESP, EBP, ESI, EDI)                                       │
    │     • 프로그램 상태 레지스터 (EFLAGS)                                              │
    │                                                                                  │
    │  5. CPU 스케줄링 정보 (Scheduling Information)                                   │
    │     • Priority (우선순위)                                                         │
    │     • Time Quantum (할당 시간)                                                    │
    │     • Waiting Time (대기 시간)                                                    │
    │                                                                                  │
    │  6. 메모리 관리 정보 (Memory Management)                                          │
    │     • Program Boundary (코드/데이터 영역 주소)                                   │
    │     • Stack Pointer (스택 포인터)                                                 │
    │     • Heap Pointer (힙 포인터)                                                   │
    │                                                                                  │
    │  7. 입출력 상태 (I/O Status)                                                     │
    │     • Open Files List (열린 파일 목록)                                            │
    │     • I/O Device Status (장치 상태)                                               │
    │                                                                                  │
    │  8. 회계 정보 (Accounting)                                                       │
    │     • CPU Usage Time (CPU 사용 시간)                                              │
    │     • Memory Usage (메모리 사용량)                                                │
    │                                                                                  │
    └───────────────────────────────────────────────────────────────────────────────────┘
```

### 프로세스 상태 전이 (Process State Transition)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세스 상태 전이 다이어그램                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

                        [Admit]
                         │   │
                         ▼   │
    ┌───────────┐    ┌─────────┐
    │   NEW     │───→│  READY  │◄──────────────────────────────────────────────┐
    │ (생성 중) │    │ (준비)  │                                               │
    └───────────┘    └────┬────┘                                               │
                           │                                                     │
                           │ [Scheduler Dispatch]                             │
                           ▼                                                     │
                    ┌───────────┐                                               │
                    │  RUNNING  │───┐ [I/O Request, Event Wait]                │
                    │  (실행 중) │   │                                              │
                    └─────┬─────┘   │                                              │
                          │          │                                              │
         ┌───────────────────┘    │    [Interrupt, Time Quantum]           │
         │                        ▼    ◄────────────────────────────────────────┤
         │              ┌───────────┐                                      │
    [I/O Completion]   │  WAITING  │───────────────────────────────────────┘
    [Event Occurs]     │ (대기 중) │
         │              └─────┬─────┘
         │                    │
         │    [I/O Complete]
         │         [Event Occurs]
         │                    │
         └────────────────────┘
                    │
                    ▼
    ┌───────────┐
    │TERMINATED │
    │  (종료)   │
    └───────────┘

    [상태별 설명]
    • NEW: 프로세스 생성 중, PCB 할당 완료 전
    • READY: 실행 준비 완료, CPU 할당 대기
    • RUNNING: CPU 할당되어 실행 중
    • WAITING: I/O 또는 이벤트 대기 중 (CPU 점유 불가)
    • TERMINATED: 실행 완료, 자원 반납 대기
```

## Ⅲ. 융합 비교 및 다각도 분석

### 과목 융합 관점 분석

#### 1. 컴퓨터 구조 ↔ 프로세스
- **문맥 교환(Context Switching)**: 프로세스 전환 시 레지스터, PC, SP 저장/복원 오버헤드
- **TLB 플러시**: 프로세스 교체 시 페이지 테이블 캐시 무효화
- **캐시 일관성**: 프로세스간 데이터 공유 시 캐시 동기화

#### 2. 프로그래밍 ↔ 프로세스
- **fork() 시스템 콜**: 부모 프로세스를 복제하여 자식 프로세스 생성
- **exec() 시스템 콜**: 프로세스 메모리 공간을 새 프로그램으로 교체
- **wait() 시스템 콜**: 자식 프로세스 종료 대기

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 웹 서버의 동시 접속 처리
**상황**: 동시에 1000명의 사용자가 웹사 접속
**판단**:
1. **프로세스 기반 아파치 (Prefork MPM)**:
   - 각 클라이언트마다 전용 프로세스 할당
   - 메모리 사용량 많음
   - 문맥 교환 오버헤드 큼음
2. **스레드 기반 아파치 (Worker MPM)**:
   - 하나의 프로세스가 여러 스레드 생성
   - 메모리 효율적
   - 공유 자원 잠금 주의
3. **이벤트 기반 (Nginx)**:
   - 단일 프로세스, 비동기 I/O
   - 최소한의 컨텍스트 교환

## Ⅴ. 기대효과 및 결론

### 프로그램 vs 프로세스의 기대 효과

| 측면 | 프로그램만 사용 시 | 프로세스 기반 OS 시 |
|------|------------------|-------------------|
| **병렬 처리** | 불가능 | 가능 (멀티태스킹) |
| **자원 공유** | 불가능 | 가능 (가상 메모리) |
| **격리성** | 없음 (충돌) | 높음 (독립 공간) |
| **신뢰성** | 낮음 (프로그램 오류 → 시스템 다운) | 높음 (프로세스 격리) |
