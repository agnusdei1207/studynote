# 프로세스 상태 (Process State)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 상태는 **생성(New) → 준비(Ready) → 실행(Running) → 대기(Waiting) → 종료(Terminated)**의 5가지 상태를 순환하며, **스케줄러(Scheduler)**가 준비 상태의 프로세스 중 하나를 선택하여 실행 상태로 전이시키고, **I/O 요청이나 이벤트 대기 시 대기 상태로 전이시킨다.
> 2. **가치**: **준비 큐(Ready Queue)**는 CPU를 기다리는 프로세스들의 FIFO/LIFO 대기열이며, **대기 큐(Wait Queue)**는 I/O나 이벤트 완료를 기다리는 프로세스들의 대기열로, 상태 전이를 통해 CPU 효율을 극대화하고 시스템 처리량(Throughput)을 높인다.
> 3. **융합**: **CPU 스케줄링 알고리즘**(FCFS, SJF, RR, Priority)의 핵심 입력이자, 멀티코어 환경에서의 **부하 분산(Load Balancing)**과 **코어 친화성(Core Affinity)** 설정의 기초가 되며, 시스템 성능 병목 진단의 핵심 지표가 된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
프로세스 상태(Process State)는 **"프로세스의 현재 실행 상태를 나타내는 정보"**로, 운영체제가 스케줄링과 자원 할당을 결정하는 근거가 된다.

**5가지 기본 상태**:
1. **New (생성)**: 프로세스가 생성 중인 상태
2. **Ready (준비)**: CPU 할당 대기 상태
3. **Running (실행)**: CPU를 점유하고 명령어 실행 중
4. **Waiting (대기)**: I/O 또는 이벤트 완료 대기
5. **Terminated (종료)**: 실행 완료, 종료 처리 중

### 💡 비유
프로세스 상태는 **"식당 줄 서기기"**와 같다.
- **New**: 손님 입장 (메뉴판 보는 중)
- **Ready**: 줄 서서 기다림 (주문 대기)
- **Running**: 주문하고 식사 중 (음식 받는 중)
- **Waiting**: 주문한 음식 조리 중 (다른 손님에게 자리 비킴)
- **Terminated**: 식사 완료, 매장 leaving

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세스 상태 모델의 진화                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

초기 모델 (2상태):
    Running (실행 중)
    Non-Running (비실행)
    • 간단하지만 I/O 대기 구별 불가

3상태 모델:
    Running (실행)
    Ready (준비)
    Blocked (차단)
    • I/O 대기 구분 가능

5상태 모델 (현대 OS):
    New → Ready → Running → Waiting → Terminated
    • 생성/종료 구체화
    • 대기 원인별 세분화 (이벤트 대기)
```

## Ⅱ. 아키텍처 및 핵심 원리

### 프로세스 상태별 상세

| 상태 | 설명 | 이유 | 결과 |
|------|------|------|------|
| **New** | 프로세스 생성 중 | PCB 할당 전 | 준비(Redy)로 전이 |
| **Ready** | CPU 할당 대기 | 다른 프로세스 실행 중 | 스케줄러 선택 대기 |
| **Running** | 명령어 실행 중 | CPU 점유 | 준비/대기/종료로 전이 |
| **Waiting** | I/O/이벤트 대기 | I/O 요청, 자원 대기 | 완료 시 준비로 전이 |
| **Terminated** | 실행 완료 | exit() 호출, 오류 | PCB 삭제, 자원 반납 |

### 정교한 상태 전이 다이어그램

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세스 상태 전이 (State Transition)                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [스케줄러(Scheduler)]          [I/O 장치]                   [인터럽트]
           │                         │                            │
           │                         │                            │
    ┌───────┴────────┐              │              ┌───────────────┴──────┐
    │                │              │              │                       │
    ▼                ▼              ▼              ▼                       ▼
┌─────────┐      ┌─────────┐      ┌─────────┐      ┌─────────┐        ┌─────────┐
│   NEW   │      │  READY  │      │ RUNNING │      │ WAITING │        │TERMINATED│
└────┬────┘      └────┬────┘      └────┬────┘      └────┬────┘        └────┬────┘
     │                 │                 │                 │                   │
     │  [Admit]        │                 │                 │                   │
     │                 │                 │                 │                   │
     └────────────────►│                 │                 │                   │
                                       │  [Dispatch]      │                   │
                                       │                 │                   │
                                       │                 └───────┬───────────┘                   │
                                       │                         │  [I/O Request]    │
                                       │                         │                   │
                                       │                         ▼                   │
                                       │                    ┌──────────────┐    │
                                       │                    │  WAITING     │    │
                                       │                    │ (I/O Wait)   │    │
                                       │                    └───────┬──────┘    │
                                       │                            │            │
                                       │                [I/O Complete]      │            │
                                       │                            │            │
                                       │                            ▼            │
                                       │                    ┌──────────────┐    │
                                       │                    │  READY       │    │
                                       │                    └───────┬──────┘    │
                                       │                            │            │
                                       │    [Time Quantum/Interrupt]   │            │
                                       │                            │            │
                                       │    ◄───────────────────────────┘            │
                                       │                                         │
                                       │    [exit(), Exception]                   │
                                       │                                         │
                                       │                            ┌───────┴───────┐
                                       │                            │  TERMINATED  │
                                       │                            └──────────────┘

    [상태 전이 트리거]
    • Admit: 생성 완료, PCB 할당
    • Dispatch: 스케줄러가 CPU 할당
    • Time Quantum: 할당 시간 만료
    • I/O Request: 디스크, 네트워크 요청
    • I/O Complete: 입출력 완료 인터럽트
    • exit(): 프로세스 종료 호출
    • Exception: 치명적 오류 발생
```

### 심층 동작 원리

#### 1. 준비 상태 (Ready State)

**정의**: CPU 할당을 기다리는 프로세스들의 상태

**준비 큐(Ready Queue)**:
```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         준비 큐(Ready Queue) 구조                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    PCB 1    PCB 2    PCB 3    PCB 4    PCB 5
      │        │        │        │        │
      ▼        ▼        ▼        ▼        ▼
    ┌───┐    ┌───┐    ┌───┐    ┌───┐    ┌───┐
    │   │    │   │    │   │    │   │    │   │
    │   │    │   │    │   │    │   │    │   │
    │   │    │   │    │   │    │   │    │   │
    └───┘    └───┘    └───┘    └───┘    └───┘
      │        │        │        │        │
      └────────┴────────┴────────┴────────┘
                    │
            [스케줄러 선택 기준]
            • 우선순위 (Priority)
            • RR (Round Robin)
            • SJF (Shortest Job First)
```

**특징**:
- 프로세스는 준비 큐에서 FIFO 순서로 대기
- 선점형 스케줄링: 언제든지 Running 상태로 전이 가능
- 비선점형: 현재 실행 중인 프로세스가 자발적으로 CPU 양보

#### 2. 실행 상태 (Running State)

**정의**: CPU를 점유하고 명령어를 실행 중인 상태

**단일 코어 환경 제약**:
- **오직 하나의 프로세스만 Running 상태**
- 다중 코어: 코어 수만큼 병렬 Running 가능

**Running → Ready 전이 (Preemption)**:
- **Time Quantum(타임 슬라이스) 만료**: RR 스케줄링
- **우선순위 높은 프로세스 도착**: Priority 스케줄링
- **인터럽트 발생**: 하드웨어/소프트웨어 인터럽트 처리

**Running → Waiting 전이**:
- **I/O 요청**: 디스크 읽기/쓰기, 네트워크 송수신
- **이벤트 대기**: 시그널 대기, 자원 대기
- **자원 부족**: 메모리 부족(Swap out)

#### 3. 대기 상태 (Waiting/Blocked State)

**정의**: I/O 완료 또는 이벤트 발생을 대기하는 상태

**대기 큐(Wait Queue)**:
```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         대기 큐(Wait Queue) - I/O 장치별                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [디스크 I/O 대기 큐]           [키보드 대기 큐]           [네트워크 대기 큐]
    ┌─────────────┐              ┌─────────────┐              ┌─────────────┐
    │ PCB A       │              │ PCB B       │              │ PCB C       │
    │ PCB D       │              │             │              │ PCB E       │
    │ PCB F       │              │             │              │             │
    └─────────────┘              └─────────────┘              └─────────────┘
           │                            │                            │
           │                            │                            │
           ▼                            ▼                            ▼
    [Disk I/O Complete]         [Keyboard Input]            [Network RX]
```

**Waiting → Ready 전이**:
- **I/O 완료 인터럽트**: 장치 컨트롤러가 신호
- **시그널 도착**: kill(), signal()
- **자원 할당**: 세마포어 획득

#### 4. 종료 상태 (Terminated State)

**정의**: 프로세스 실행 완료, 자원 반납 대기

**종료 원인**:
- **정상 종료**: exit() 시스템 콜
- **비정상 종료**: SIGKILL, SIGSEGV (Segmentation Fault)
- **기타**: 부모 프로세스 종료에 따른 자식 종료

**자원 반납**:
1. PCB 메모리 해제
2. 열린 파일 닫기
3. 할당된 메모리 반납
4. 자식 프로세스 있으면 init(PID 1)으로 입양

## Ⅲ. 융합 비교 및 다각도 분석

### 스케줄링 알고리즘별 상태 전이 빈도

| 알고리즘 | Ready → Running | Running → Ready | Waiting → Ready | 장점 | 단점 |
|----------|----------------|----------------|-----------------|------|------|
| **FCFS** | FIFO | 프로세스 종료/대기 | I/O 완료 | 공평, 단순 | Convoy 효과 |
| **SJF** | 최단 CPU 버스트 | 프로세스 종료/대기 | I/O 완료 | 평균 대기 시간 최소 | 기아 현상 |
| **RR** | Time Quantum | Time Quantum 만료 | I/O 완료 | 응답 시간 보장 | 문맥 교환 오버헤드 |
| **Priority** | 최고 우선순위 | 높은 우선순위 도착 | I/O 완료 | 중요 작업 우선 | 기아 현상 |

### 과목 융합 관점 분석

#### 1. CPU 스케줄링 ↔ 상태 전이
- **Time Quantum**: RR의 핵심 파라미터 (너무 길면: 응답 저하, 너무 짧으면: 문맥 교환 오버헤드)
- **Context Switch**: 상태 전이 시 발생하는 오버헤드 (수백~수천 사이클)

#### 2. I/O 성능 ↔ 대기 상태
- **I/O Bound 프로세스**: 대기 상태 비중 높음
- **CPU Bound 프로세스**: 실행 상태 비중 높음
- **다중 프로그래밍**: CPU와 I/O 병행 처리

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 웹 서버의 CPU 100% 현상
**상황**: 8코어 서버에서 CPU 사용률 100%, 응답 지연
**진단**:
1. **프로세스 상태 확인**: `top`, `ps aux`
   - Running 상태 프로세스 많음 → CPU 경합
   - Waiting 상태 많음 → I/O 대기, 병목
2. **해결 방안**:
   - **CPU Bound**: 코어 추가, 코드 최적화
   - **I/O Bound**: I/O 병렬화, 비동기 처리
   - **혼합**: 스레드 풀, 비동기 I/O (epoll, io_uring)

### 도입 시 고려사항 (체크리스트)

**모니터링**:
- [ ] 프로세스 상태별 수집: `ps aux`, `top`, `htop`
- [ ] 준비 큐 길이: Load Average (1, 5, 15분)
- [ ] 문맥 교환 빈도: `vmstat`, `sar -w`

## Ⅴ. 기대효과 및 결론

### 프로세스 상태별 CPU 사용 현황

| 상태 | CPU 사용 | 메모리 | I/O | 시스템 영향 |
|------|---------|--------|-----|-----------|
| **Running** | 100% | 사용 중 | 없음 | 처리량 증가 |
| **Ready** | 0% | 할당됨 | 없음 | 대기 시간 증가 |
| **Waiting** | 0% | 할당됨 | 진행 중 | 응답 지연 |

### ※ 참고 표준/가이드
- **Linux procfs**: `/proc/[pid]/status`
- **POSIX**: 프로세스 상태 표준 (S, R, D, Z, T)
