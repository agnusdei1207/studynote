# 교착상태 (Deadlock)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착상태는 **"두 개 이상의 **프로세스가 **서로가 가진 **자원을 기다리며 **무한히 **대기하는 **상태"**로, **Coffman 조건**(Mutual Exclusion, Hold & Wait, No Preemption, Circular Wait)이 **모두 충족**될 때 **발생**한다.
> 2. **가치**: **교착상태 예방**(Prevention), **회피**(Avoidance), **검출**(Detection), **복구**(Recovery) 전략이 **존재**하며 **Banker's Algorithm**, **Resource Allocation Graph**(RAG)로 **안전한 상태**를 **판단**한다.
> 3. **융합**: **운영체제**에서 **프로세스 동기화**의 **중요한 문제**이며 **Database**(Deadlock Detection), **Distributed System**(Distributed Deadlock)에서도 **동일한 문제**가 **발생**하고 **Timeout**, **Wait-Die**, **Wound-Wait** 기법으로 **해결**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
교착상태는 **"프로세스들이 서로의 자원을 기다리며 진행하지 못하는 상태"**이다.

**교착상태의 예시**:
- A가 P1 Lock을 보유 → P2 Lock 대기
- B가 P2 Lock을 보유 → P1 Lock 대기
- → 무한 대기

### 💡 비유
교착상태는 **"교차로에서 서로 양보**하는 상황"과 같다.
- **A 차량**: B에게 양보 대기
- **B 차량**: A에게 양보 대기
- → 교차로 폐색

## Ⅱ. 아키텍처 및 핵심 원리

### Coffman 조건 (필요조건)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Coffman Conditions (교착상태 4가지 조건)                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    1. Mutual Exclusion (상호 배제)

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 자원은 한 번에 하나의 프로세스만 사용 가능                                           │
    │  │  • 예: Lock, Semaphore, Printer                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. Hold & Wait (보유 대기)

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 자원을 보유한 상태에서 다른 자원 대기                                                   │
    │  │  • 예: Lock A를 보유하며 Lock B 대기                                                 │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. No Preemption (비선점)

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 보유한 자원을 강제로 뺏을 수 없음                                                      │
    │  │  • 예: Mutex Lock은 소유자만 해제                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    4. Circular Wait (순환 대기)

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 대기 프로세스들이 원형을 이루어 서로 대기                                               │
    │  │  • P1 → P2 → P3 → P1 (순환)                                                         │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  P1 ──→ Lock A (보유) ──→ Lock B (대기)                                             │  │
    │  │  P2 ──→ Lock B (보유) ──→ Lock C (대기)                                             │  │
    │  │  P3 ──→ Lock C (보유) ──→ Lock A (대기)                                             │  │
    │  │                                                                                       │  │
    │  │  → 순환 대기 발생!                                                                   │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [필요충분조건]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 4가지 조건 모두 충족 시 교착상태 발생 가능                                            │
    │  │  • 하나라도 충족하지 않으면 교착상태 불가능                                            │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Resource Allocation Graph (RAG)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         자원 할당 그래프 (Resource Allocation Graph)                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [그래프 구조]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Process (P): 원형 ○                                                                │
    │  │  • Resource (R): 사각형 □                                                            │
    │  │  • Assignment Edge (R → P): 실선                                                     │
    │  │  • Request Edge (P → R): 점선                                                        │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │    P1         P2         P3         R1         R2         R3                      │  │
    │  │     │           │           │           │           │           │                  │  │
    │  │     └───────────┼───────────┼───────────┼───────────┼           │                  │  │
    │  │                 │           │           │           │           │                  │  │
    │  │                 └───────────┼───────────┼───────────┼───────────┘                  │  │
    │  │                             │           │           │                              │  │
    │  │  [교착상태 없음]                                                     │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  P1 ──(request)──> R1                                                            │  │  │
    │  │  │  P2 ──(request)──> R2                                                            │  │  │
    │  │  │  R1 ──(assign)───> P2                                                            │  │  │
    │  │  │  → 사이클 없음 → 안전                                                              │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                       │  │
    │  │  [교착상태 있음]                                                                      │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  P1 ──(request)──> R1                                                            │  │  │
    │  │  │  P2 ──(request)──> R2                                                            │  │  │
    │  │  │  P3 ──(request)──> R3                                                            │  │  │
    │  │  │  R1 ──(assign)───> P2                                                            │  │  │
    │  │  │  R2 ──(assign)───> P3                                                            │  │  │
    │  │  │  R3 ──(assign)───> P1                                                            │  │  │
    │  │  │  → 사이클 존재 (P1→P2→P3→P1) → 교착상태!                                         │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Banker's Algorithm

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         은행가 알고리즘 (Banker's Algorithm)                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Dijkstra (1965)                                                                      │
    │  │  • 자원 할당 전 안전 상태(Safe State) 확인                                          │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  # 예시: 3개 Process, 12개 Resource                                                │  │
    │  │                                                                                     │  │
    │  │  Process    Max      Alloc    Need     Available                                │  │
    │  │  ──────────────────────────────────────────────────────────────────────────────  │  │
    │  │  P0         10       5        5        3                                         │  │
    │  │  P1         4        2        2                                                  │  │
    │  │  P2         9        2        7                                                  │  │
    │  │                                                                                     │  │
    │  │  # Safe Sequence 찾기                                                               │  │
    │  │  1. P1: Need(2) ≤ Available(3) → 가능                                              │  │
    │  │     Available += Alloc(2) = 5                                                       │  │
    │  │  2. P0: Need(5) ≤ Available(5) → 가능                                              │  │
    │  │     Available += Alloc(5) = 10                                                      │  │
    │  │  3. P2: Need(7) ≤ Available(10) → 가능                                             │  │
    │  │     Safe Sequence: [P1, P0, P2] → 안전 상태!                                         │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Deadlock Prevention (예방)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         교착상태 예방 기법                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    1. Mutual Exclusion 제거

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 자원을 공유 가능하게 만듦                                                             │
    │  │  • 예: Read-Only Lock (공유 가능)                                                    │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. Hold & Wait 제거

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 모든 자원을 한 번에 요청하거나 아무 자원도 보유하지 않은 상태에서만 요청                       │
    │  │  • 자원 낭비 발생                                                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. Preemption 허용

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 보유한 자원을 뺏을 수 있게 함                                                          │
    │  │  • 예: Priority Inheritance                                                         │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    4. Circular Wait 제거

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 자원에 순서 번호 부여                                                                │
    │  │  • 오름차순으로만 자원 요청                                                           │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  자원: R1(1), R2(2), R3(3)                                                          │  │
    │  │  요청 순서: R1 → R2 → R3 (오름차순만 허용)                                             │  │
    │  │  → 순환 대기 불가능!                                                                 │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 교착상태 방지
**상황**: 다중 Lock 획득
**판단**:

```c
// 교착상태 위험 코드
void thread1() {
    lock(A);
    // ...
    lock(B);  // B 대기
    unlock(B);
    unlock(A);
}

void thread2() {
    lock(B);
    // ...
    lock(A);  // A 대기 → 교착상태!
    unlock(A);
    unlock(B);
}

// 해결: 순서 통일
#define LOCK_ORDER_A 1
#define LOCK_ORDER_B 2

void thread1() {
    lock(A, LOCK_ORDER_A);
    lock(B, LOCK_ORDER_B);  // 항상 A → B 순서
    unlock(B);
    unlock(A);
}

void thread2() {
    lock(A, LOCK_ORDER_A);  // 동일 순서
    lock(B, LOCK_ORDER_B);
    unlock(B);
    unlock(A);
}

// 또는 std::lock (C++11)
#include <mutex>
std::mutex mutex1, mutex2;

void safe_lock() {
    // 데드락 방지 알고리즘 사용
    std::lock(mutex1, mutex2);
    std::lock_guard<std::mutex> lock1(mutex1, std::adopt_lock);
    std::lock_guard<std::mutex> lock2(mutex2, std::adopt_lock);
}
```
