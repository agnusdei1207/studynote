# 실시간 시스템 (Real-time System)

## 정의

**실시간 시스템(Real-time System)**은 작업 처리에 **엄격한 시간 제약(Deadline)**이 있는 시스템으로, 정해진 시간 내에 반드시 처리를 완료해야 한다. 경성(Hard)과 연성(Soft) 실시간 시스템으로 구분된다.

```
┌─────────────────────────────────────────────────────────────────┐
│           실시간 시스템 분류                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   경성 실시간 (Hard Real-time)                          │ │
│   │   • 데드라인 위반 = 시스템 실패/재앙                     │ │
│   │   • 예: 항공기 제어, 원자로 제어, ABS 브레이크           │ │
│   │   • 반드시 시간 보장 필요                                │ │
│   │                                                         │ │
│   │   연성 실시간 (Soft Real-time)                          │ │
│   │   • 데드라인 위반 = 품질 저하                           │ │
│   │   • 예: 비디오 스트리밍, 게임, 오디오 재생               │ │
│   │   • 최선 노력(Best-effort)으로 처리                      │ │
│   │                                                         │ │
│   │   확정 실시간 (Firm Real-time)                          │ │
│   │   • 데드라인 위반 시 결과 폐기                          │ │
│   │   • 예: 금융 거래, 날씨 예보                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 실시간 시스템 특성

```
┌─────────────────────────────────────────────────────────────────┐
│           실시간 시스템 핵심 특성                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  특성              │  설명                              │ │
│   ├───────────────────┼──────────────────────────────────┤ │
│   │ 시간 제약성        │ 데드라인(Deadline) 엄수           │ │
│   │ 예측 가능성        │ 최악의 경우 응답시간 보장         │ │
│   │ 신뢰성             │ 시스템 고장 치명적 결과           │ │
│   │ 안전성             │ 인명/환경 피해 방지               │ │
│   │ 결정성             │ 동일 입력 → 동일 시간 내 결과     │ │
│   └───────────────────┴──────────────────────────────────┘ │
│                                                                 │
│   실시간 시스템 요구사항:                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 응답 시간 (Response Time): 이벤트 → 반응까지 시간    │ │
│   │   • 지터 (Jitter): 응답 시간의 편차 최소화               │ │
│   │   • 처리량 (Throughput): 단위 시간당 처리량              │ │
│   │   • 신뢰도 (Reliability): MTBF 최대화                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 주기적/비주기적 태스크

```
┌─────────────────────────────────────────────────────────────────┐
│           실시간 태스크 분류                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   주기적 태스크 (Periodic Task):                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 정해진 주기(T)마다 실행                             │ │
│   │   • 센서 데이터 수집, 제어 신호 생성                    │ │
│   │                                                         │ │
│   │   파라미터:                                              │ │
│   │   • p (Period): 실행 주기                               │ │
│   │   • e (Execution): 실행 시간                            │ │
│   │   • d (Deadline): 마감 시한 (보통 d ≤ p)                │ │
│   │                                                         │ │
│   │   타임라인:                                              │ │
│   │   ├──e──┤──p──├──e──┤──p──├──e──┤                     │ │
│   │   ↑      ↑      ↑      ↑      ↑                        │ │
│   │   t=0   t=p    t=2p   t=3p   t=4p                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   비주기적 태스크 (Aperiodic Task):                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 예측 불가능한 시점에 발생                           │ │
│   │   • 인터럽트, 사용자 입력, 알람                         │ │
│   │   • 최소 응답 시간 보장 필요                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 실시간 스케줄링 알고리즘

```
┌─────────────────────────────────────────────────────────────────┐
│           실시간 스케줄링 알고리즘                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ① RM (Rate-Monotonic) 스케줄링:                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 정적 우선순위 (주기가 짧을수록 높음)                 │ │
│   │   • 주기 p → 우선순위 = 1/p                             │ │
│   │   • 선점형 스케줄링                                     │ │
│   │   • CPU 활용률 ≤ n(2^(1/n) - 1)일 때 스케줄 보장        │ │
│   │     n=1: 100%, n=2: 82.8%, n→∞: 69.3% (ln 2)          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ② EDF (Earliest Deadline First) 스케줄링:                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 동적 우선순위 (데드라인 빠를수록 높음)               │ │
│   │   • 우선순위 = 1/남은데드라인                           │ │
│   │   • 이론적으로 최적 (100% 활용률까지 스케줄 가능)        │ │
│   │   • 오버헤드 큼 (매번 우선순위 재계산)                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ③ DM (Deadline Monotonic) 스케줄링:                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 정적 우선순위 (데드라인이 짧을수록 높음)             │ │
│   │   • d < p인 경우 RM보다 우수                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   비교:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  알고리즘 │ 우선순위  │ 활용률 한계 │ 오버헤드        │ │
│   ├──────────┼───────────┼─────────────┼────────────────┤ │
│   │ RM       │ 정적      │ ~69.3%      │ 낮음            │ │
│   │ EDF      │ 동적      │ 100%        │ 높음            │ │
│   │ DM       │ 정적      │ ~69.3%      │ 낮음            │ │
│   └──────────┴───────────┴─────────────┴────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 우선순위 역전 문제

```
┌─────────────────────────────────────────────────────────────────┐
│           우선순위 역전 (Priority Inversion)                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   문제 상황:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   우선순위: L(낮음) < M(중간) < H(높음)                 │ │
│   │                                                         │ │
│   │   ① L이 자원 S를 획득 (Lock)                            │ │
│   │   ② H가 L을 선점, 자원 S 필요 → 대기                    │ │
│   │   ③ M이 실행 준비, L을 선점 → H가 더 오래 대기          │ │
│   │                                                         │ │
│   │   결과: H(높은 우선순위)가 M(중간)에 의해 간접 차단      │ │
│   │                                                         │ │
│   │   타임라인:                                              │ │
│   │   L: ████░░░░░░░░░░░░████████                          │ │
│   │       ↓ Lock S    ↓ Unlock S                           │ │
│   │   M:      ░░░░░░░░░░░░░░░░░░░░████                    │ │
│   │                        ↓ 선점                           │ │
│   │   H:          ░░░░░░░░░░░░░░░░░░░░░░░░████████        │ │
│   │              ↑ H가 S 대기 시작                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   해결책:                                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 우선순위 상속 (Priority Inheritance):               │ │
│   │      L이 H의 대기 중일 때 L이 H의 우선순위를 임시 상속   │ │
│   │                                                         │ │
│   │   ② 우선순위 상한 (Priority Ceiling):                   │ │
│   │      자원에 최대 우선순위 할당, Lock 시 해당 우선순위로  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **실시간 시스템**: 엄격한 시간 제약(Deadline)이 있는 시스템
> **경성 실시간**: 데드라인 위반 = 치명적 실패
> **연성 실시간**: 데드라인 위반 = 품질 저하

### 2. 빈출 키워드

- **Deadline**: 작업 완료 마감 시한
- **RM (Rate-Monotonic)**: 주기 기반 정적 우선순위
- **EDF (Earliest Deadline First)**: 데드라인 기반 동적 우선순위
- **Priority Inversion**: 우선순위 역전 문제

### 3. 예상 문제

**Q1. 경성 실시간 시스템과 연성 실시간 시스템의 차이점을 설명하시오.**

**Q2. RM 스케줄링과 EDF 스케줄링을 비교하고 각각의 장단점을 기술하시오.**

**Q3. 우선순위 역전 현상을 설명하고 해결 방안을 제시하시오.**
