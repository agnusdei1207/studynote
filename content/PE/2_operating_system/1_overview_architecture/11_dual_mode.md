# 듀얼 모드 (Dual Mode)

## 정의

**듀얼 모드(Dual Mode)**는 CPU가 **사용자 모드(User Mode)**와 **커널 모드(Kernel Mode)** 두 가지 실행 모드를 제공하여, 시스템 자원을 보호하고 운영체제의 안정성을 보장하는 하드웨어 메커니즘이다.

```
┌─────────────────────────────────────────────────────────────────┐
│           듀얼 모드 개념                                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │            ┌─────────────────────────┐                  │ │
│   │            │     커널 모드           │                  │ │
│   │            │   (Kernel Mode)         │                  │ │
│   │            │   • 모든 명령 실행 가능 │                  │ │
│   │            │   • 하드웨어 직접 접근   │                  │ │
│   │            │   • Privilege = 1       │                  │ │
│   │            └───────────┬─────────────┘                  │ │
│   │                        │                                │ │
│   │            시스템 콜   │   트랩/인터럽트 복귀            │ │
│   │            ────────────┼────────────────▶               │ │
│   │                        │                                │ │
│   │            ◀───────────┼────────────────                │ │
│   │                        ▼                                │ │
│   │            ┌─────────────────────────┐                  │ │
│   │            │     사용자 모드         │                  │ │
│   │            │   (User Mode)           │                  │ │
│   │            │   • 제한된 명령만 실행   │                  │ │
│   │            │   • 하드웨어 직접 접근 X │                  │ │
│   │            │   • Privilege = 0       │                  │ │
│   │            └─────────────────────────┘                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 모드별 권한과 기능

```
┌─────────────────────────────────────────────────────────────────┐
│           커널 모드 vs 사용자 모드                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분          │ 커널 모드           │ 사용자 모드       │ │
│   ├────────────────┼─────────────────────┼───────────────────┤ │
│   │ 별칭           │ 관리자 모드,        │ 제한 모드,        │ │
│   │                │ 슈퍼바이저 모드     │ 슬레이브 모드     │ │
│   ├────────────────┼─────────────────────┼───────────────────┤ │
│   │ 모드 비트      │ 0                   │ 1                 │ │
│   ├────────────────┼─────────────────────┼───────────────────┤ │
│   │ 명령어         │ 모든 명령 가능      │ 제한된 명령만     │ │
│   │                │ (특권 명령 포함)    │ (일반 명령만)     │ │
│   ├────────────────┼─────────────────────┼───────────────────┤ │
│   │ 메모리 접근    │ 전체 메모리 접근    │ 할당된 영역만     │ │
│   ├────────────────┼─────────────────────┼───────────────────┤ │
│   │ I/O 접근       │ 직접 가능           │ 시스템 콜 필요    │ │
│   ├────────────────┼─────────────────────┼───────────────────┤ │
│   │ 실행 주체      │ OS 커널             │ 사용자 프로그램   │ │
│   └────────────────┴─────────────────────┴───────────────────┘ │
│                                                                 │
│   특권 명령 (Privileged Instructions):                          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • I/O 명령 (IN, OUT)                                  │ │
│   │   • 메모리 관리 레지스터 변경 (CR3 등)                   │ │
│   │   • 인터럽트 활성/비활성 (CLI, STI)                      │ │
│   │   • 모드 전환 명령                                      │ │
│   │   • 시스템 타이머 설정                                   │ │
│   │   • 캐시 플러시                                         │ │
│   │                                                         │ │
│   │   💡 사용자 모드에서 실행 시 → 예외(Trap) 발생           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 모드 전환 메커니즘

```
┌─────────────────────────────────────────────────────────────────┐
│           모드 전환 과정                                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   사용자 → 커널 모드 전환:                                      │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   사용자 프로그램                                       │ │
│   │        │                                                │ │
│   │        │ 시스템 콜 / 인터럽트 / 예외                    │ │
│   │        ▼                                                │ │
│   │   ┌───────────────────┐                                │ │
│   │   │ ① 트랩 발생        │                                │ │
│   │   └─────────┬─────────┘                                │ │
│   │             ▼                                          │ │
│   │   ┌───────────────────┐                                │ │
│   │   │ ② 모드 비트 1→0   │  ← 하드웨어 자동 설정          │ │
│   │   └─────────┬─────────┘                                │ │
│   │             ▼                                          │ │
│   │   ┌───────────────────┐                                │ │
│   │   │ ③ PC, 레지스터    │  ← 커널 스택에 저장             │ │
│   │   │    저장           │                                │ │
│   │   └─────────┬─────────┘                                │ │
│   │             ▼                                          │ │
│   │   ┌───────────────────┐                                │ │
│   │   │ ④ ISR/시스템 콜   │  ← 커널 코드 실행              │ │
│   │   │    핸들러 실행     │                                │ │
│   │   └───────────────────┘                                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   커널 → 사용자 모드 전환:                                      │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   커널 처리 완료                                        │ │
│   │        │                                                │ │
│   │        ▼                                                │ │
│   │   ┌───────────────────┐                                │ │
│   │   │ ① 레지스터 복원    │  ← 커널 스택에서 로드          │ │
│   │   └─────────┬─────────┘                                │ │
│   │             ▼                                          │ │
│   │   ┌───────────────────┐                                │ │
│   │   │ ② 모드 비트 0→1   │  ← IRET 등 특수 명령           │ │
│   │   └─────────┬─────────┘                                │ │
│   │             ▼                                          │ │
│   │   ┌───────────────────┐                                │ │
│   │   │ ③ 사용자 프로그램 │  ← 원래 위치로 복귀             │ │
│   │   │    재개           │                                │ │
│   │   └───────────────────┘                                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 모드 전환 트리거

```
┌─────────────────────────────────────────────────────────────────┐
│           모드 전환 발생 원인                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 시스템 콜 (System Call)                             │ │
│   │      • 사용자 프로그램의 명시적 요청                     │ │
│   │      • 예: open(), read(), write(), fork()              │ │
│   │      • 소프트웨어 인터럽트 (INT 0x80, SYSCALL)           │ │
│   │                                                         │ │
│   │   ② 인터럽트 (Interrupt)                                │ │
│   │      • 하드웨어 이벤트                                   │ │
│   │      • 예: 타이머 인터럽트, I/O 완료, 키보드 입력        │ │
│   │      • 비동기적 발생                                     │ │
│   │                                                         │ │
│   │   ③ 예외/트랩 (Exception/Trap)                          │ │
│   │      • 프로그램 실행 중 오류                             │ │
│   │      • 예: 0으로 나누기, 페이지 폴트, 보안 위반          │ │
│   │      • 동기적 발생                                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   전환 비용:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 레지스터 저장/복원 오버헤드                          │ │
│   │   • 메모리 접근 (커널 스택)                              │ │
│   │   • 캐시/TLB 영향                                        │ │
│   │   • 파이프라인 플러시                                    │ │
│   │                                                         │ │
│   │   💡 불필요한 시스템 콜은 성능 저하 유발                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 다중 모드 확장

```
┌─────────────────────────────────────────────────────────────────┐
│           x86 Ring Protection (다중 권한 레벨)                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   x86 아키텍처는 4개의 권한 레벨(Ring 0~3) 제공:                │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │      Ring 0 (커널 모드)                                 │ │
│   │      ┌──────────────────────────────────────────────┐  │ │
│   │      │ • OS 커널                                     │  │ │
│   │      │ • 모든 하드웨어 접근                          │  │ │
│   │      │ • 최고 권한                                   │  │ │
│   │      └──────────────────────────────────────────────┘  │ │
│   │      Ring 1, 2 (드라이버/시스템 서비스)                 │ │
│   │      ┌──────────────────────────────────────────────┐  │ │
│   │      │ • 디바이스 드라이버 (과거)                    │  │ │
│   │      │ • 현대 OS에서는 거의 사용 안 함               │  │ │
│   │      └──────────────────────────────────────────────┘  │ │
│   │      Ring 3 (사용자 모드)                               │ │
│   │      ┌──────────────────────────────────────────────┐  │ │
│   │      │ • 사용자 애플리케이션                         │  │ │
│   │      │ • 제한된 권한                                 │  │ │
│   │      └──────────────────────────────────────────────┘  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   현대 OS 활용:                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   • Windows/Linux: 주로 Ring 0과 Ring 3만 사용          │ │
│   │   • 가상화: Ring -1 (VMX root) 추가                     │ │
│   │   • 보안: SGX Enclave (별도 격리 공간)                  │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **듀얼 모드**: 사용자 모드와 커널 모드 두 가지 실행 모드
> **사용자 모드**: 제한된 명령만 실행, 하드웨어 직접 접근 불가
> **커널 모드**: 모든 명령 실행 가능, 하드웨어 직접 제어

### 2. 빈출 키워드

- **모드 비트**: 0=커널, 1=사용자
- **특권 명령**: 사용자 모드에서 실행 불가
- **트랩**: 사용자→커널 모드 전환
- **시스템 콜**: 사용자 프로그램의 커널 서비스 요청

### 3. 예상 문제

**Q1. 듀얼 모드의 필요성과 각 모드의 특징을 설명하시오.**

**Q2. 사용자 모드에서 커널 모드로 전환되는 경우들을 기술하시오.**

**Q3. 특권 명령의 예시를 들고, 사용자 모드에서 실행 시 발생하는 현상을 설명하시오.**
