# 다중 처리 시스템 (Multiprocessing System)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 처리 시스템은 두 개 이상의 CPU(프로세서)가 하나의 메모리와 주변 장치를 공유하며 여러 작업을 동시에 병렬로 처리하는 고성능 컴퓨팅 환경이다.
> 2. **가치**: 단일 프로세서의 물리적 한계를 극복하여 시스템 전체의 처리량(Throughput)을 높이고, 특정 CPU 고장 시에도 업무를 지속할 수 있는 신뢰성(Reliability)을 제공한다.
> 3. **융합**: 대칭 다중 처리(SMP), 비대칭 다중 처리(ASMP), 강결합(Tightly-coupled) 아키텍처 기술이 결합되어 현대의 멀티코어 서버 및 슈퍼컴퓨터의 근간을 이룬다.

---

### Ⅰ. 개요 (Context & Background)

#### 개념
**다중 처리 시스템(Multiprocessing System)**은 한 대의 컴퓨터 시스템 내에 여러 개의 CPU를 장착하여, 운영체제의 통제 하에 복수의 프로세스나 스레드를 실제로 동시에(Simultaneously) 실행하는 방식이다.

#### 💡 비유
다중 처리 시스템은 **"한 주방에 여러 명의 요리사가 있는 식당"**과 같다.
1. **다중 프로그래밍 (요리사 1명)**: 요리사 혼자서 찌개 끓는 동안 양파를 썰며 바쁘게 움직인다 (겉보기엔 병렬이지만 실제론 교대 근무).
2. **다중 처리 (요리사 2명 이상)**: 한 명은 스테이크를 굽고, 다른 명은 샐러드를 만든다. 두 요리가 **실제로 동시에** 만들어진다.
3. **이점**: 요리가 훨씬 빨리 완성되고(처리량), 한 요리사가 손을 다쳐도 다른 요리사가 식당을 계속 운영할 수 있다(신뢰성).

#### 등장 배경 및 발전 과정
1. **단일 프로세서의 한계**: 클럭 속도를 높이는 방식(Scaling-up)이 발열과 전력 소모 문제로 한계에 부딪히자, 프로세서 개수를 늘리는 방식(Scaling-out)으로 패러다임이 전환됨.
2. **경제성 및 효율성**: 별개의 컴퓨터 여러 대를 두는 것보다 메모리와 디스크를 공유하는 다중 처리 시스템이 자원 공유 면에서 경제적임.
3. **발전 과정**:
    - **초기**: 주(Master)-종(Slave) 구조의 비대칭 방식(ASMP).
    - **중기**: 모든 CPU가 동등한 권한을 갖는 대칭 방식(SMP)의 보편화.
    - **현재**: 칩 하나에 여러 코어를 담는 멀티코어(Multi-core) 기술로 보편화.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

#### 구성 요소 (표)

| 요소명 | 상세 역할 | 내부 동작 메커니즘 | 관련 기술 | 비유 |
| :--- | :--- | :--- | :--- | :--- |
| **Processors (CPUs)** | 실제 연산 및 명령어 실행 | 개별 파이프라인 가동 | Multi-core, RISC/CISC | 요리사들 |
| **Shared Memory** | 모든 CPU가 접근 가능한 공용 공간 | 버스(Bus) 또는 스위치 기반 접근 | Tightly-coupled, UMA | 공용 냉장고 |
| **System Bus** | CPU와 메모리, I/O 간 통로 | 데이터 전송 및 중재 | Front-Side Bus, QPI | 주방 복도 |
| **Inter-Processor Comm.** | CPU 간 신호 주고받기 | 인터럽트 발송 및 동기화 | IPI (Inter-Processor Interrupt) | 요리사 간 대화 |
| **Coherence Protocol** | 캐시 데이터 일관성 유지 | 수정된 데이터 알림 및 무효화 | MESI Protocol | 공지사항 공유 |

#### 정교한 구조 다이어그램 (ASCII)

```text
    ┌─────────────┐       ┌─────────────┐
    │    CPU 1    │       │    CPU 2    │
    │ ┌─────────┐ │       │ ┌─────────┐ │
    │ │ Cache   │ │       │ │ Cache   │ │
    └────┬───┬────┘       └────┬───┬────┘
         │   │                 │   │
    ─────┴───┴─────┬───────────┴───┴───── [ System Bus ]
                   │
           ┌───────┴───────┐       ┌─────────────┐
           │ Shared Memory │       │ I/O Devices │
           └───────────────┘       └─────────────┘

    [특징: 강결합(Tightly-coupled) 시스템]
    - 하나의 OS가 모든 CPU 관리
    - 동일한 메모리 주소 공간 공유
    - 고속의 시스템 버스로 연결
```

#### 심층 동작 원리 (Parallelism & Reliability)

1. **병렬 실행(Parallelism)**: 스케줄러가 준비 큐에 있는 여러 프로세스를 가용 CPU들에 각각 배정하여 동시에 실행한다.
2. **부하 균등화(Load Balancing)**: 특정 CPU만 바쁘고 다른 CPU는 노는 일이 없도록 작업을 적절히 분배한다.
3. **결함 허용(Fault Tolerance)**: CPU 하나에 오류가 발생하면 운영체제는 해당 CPU를 시스템에서 제외(Graceful Degradation)시키고 남은 CPU로 작업을 지속한다.
4. **캐시 일관성**: 여러 CPU가 같은 데이터를 각자의 캐시에 가지고 있을 때, 한쪽이 수정하면 다른 쪽도 알 수 있게 하드웨어가 보장한다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

#### 심층 기술 비교

| 항목 | 다중 프로그래밍 | 다중 처리 (Multiprocessing) | 분산 처리 (Distributed) |
| :--- | :--- | :--- | :--- |
| **CPU 개수** | 1개 | **2개 이상** | 여러 대의 독립 컴퓨터 |
| **메모리** | 공유 | **공유 (강결합)** | 개별 (약결합) |
| **실행 방식** | 동시성 (Concurrency) | **병렬성 (Parallelism)** | 병렬성 |
| **OS 개수** | 1개 | **1개** | 여러 개 |
| **신뢰성** | 낮음 | **높음** | 매우 높음 |

#### 과목 융합 관점 분석
- **컴퓨터 구조 연계**: 멀티코어 칩 내부의 L3 캐시 공유 구조 및 코어 간 통신 패브릭(Mesh, Ring) 기술과 직결됩니다.
- **알고리즘 연계**: 여러 CPU를 효율적으로 활용하기 위한 병렬 알고리즘(Divide & Conquer) 설계가 필요합니다.

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 기술사적 판단 (실무 시나리오)
**[시나리오]** 8코어 서버에서 CPU 사용률은 높은데 처리 성능이 기대 이하인 경우
1. **원인 분석**:
    - **동기화 병목**: 여러 CPU가 공유 자원(Lock)을 차지하려고 싸우느라 대기 시간이 길어짐 (Spinlock 경합).
    - **캐시 핑퐁**: 데이터가 여러 CPU 캐시 사이를 왔다 갔다 하느라 버스 트래픽 과부하.
2. **대응 전략**:
    - **Lock-free 알고리즘** 도입으로 동기화 오버헤드 감소.
    - **CPU 친화성(Affinity)** 설정: 특정 프로세스를 특정 CPU에 고정하여 캐시 효율 증대.

#### 도입 시 고려사항 (체크리스트)
- [ ] **병렬화 가능성**: 소프트웨어가 멀티스레드를 지원하도록 설계되었는가? (Amdahl의 법칙 고려)
- [ ] **확장성(Scalability)**: CPU를 2배 늘렸을 때 성능도 그에 근접하게 향상되는가?
- [ ] **전력 및 발열**: 늘어난 CPU의 전력 소모와 열을 충분히 제어할 수 있는가?

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 정량적/정성적 기대효과
- **처리량 향상**: 복잡한 계산이나 대규모 트래픽 처리를 짧은 시간 내 완료.
- **가용성 증대**: 서버 중단 없는 24/7 서비스 운영 가능.

#### 미래 전망 및 진화 방향
- **가속기 통합**: CPU뿐만 아니라 GPU, NPU 등 서로 다른 성격의 프로세서를 혼합하여 사용하는 이기종 다중 처리(Heterogeneous Multiprocessing)가 대세가 되고 있습니다.
- **Many-core 시대**: 수십 개를 넘어 수백, 수천 개의 코어를 효율적으로 관리하는 OS 기술로 진화 중입니다.
