# 가상화 (Virtualization)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상화는 **"물리 **자원**을 **추상화**하여 **여러 **시스템**을 **실행**하는 **기술"**로, **Hypervisor**(VMware ESXi, **KVM**)가 **하드웨어**를 **관리**하고 **Guest OS**를 **격리**한다.
> 2. **유형**: **Type 1**(Bare Metal, **ESXi**, **Hyper-V**)은 **하드웨어**에 **직접 **설치**되고 **Type 2**(Hosted, **VMware Workstation**, **VirtualBox**)는 **Host OS** 위에서 **실행**된다.
> 3. **기술**: **Hardware-Assisted VM**(Intel **VT-x**, AMD-V)로 **성능**을 **향상**하고 **Para-virtualization**(Xen)으로 **트래픽** **오버헤드**를 **줄이며 **SR-IOV**(Single Root I/O Virtualization)로 **I/O**를 **가상화**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
가상화는 **"하드웨어 추상화"**이다.

**가상화 유형**:
| 유형 | 설명 | 예시 |
|------|------|------|
| **서버 가상화** | 전체 시스템 | VMware, KVM |
| **컨테이너** | OS 수준 | Docker |
| **데스크톱** | 사용자 UI | VirtualBox |
| **애플리케이션** | 앱 가상화 | WINE |

### 💡 비유
가상화는 ****건물 **임대 ****와 같다.
- **건물**: Physical Server
- **임대**: Virtual Machines
- **세입자**: Tenants

    │  │           │                                                                             │  │  │

    Type 2 (Hosted):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐

### Hardware-Assisted Virtualization

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Intel VT-x / AMD-V                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Without Hardware Virtualization:                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Guest OS Instruction → Binary Translation → Host Execution → Slow                        │  │  │
    │  │  → Trap every privileged instruction, emulate in software                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  With VT-x (Hardware Virtualization):                                                      │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  VMX Root Mode (Hypervisor)                                                            │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • Full control of hardware                                                          │  │  │  │
    │  │  │  • Manages VMCS (Virtual Machine Control Structure)                                   │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │           │                                                                             │  │  │
    │  │           ▼ VM Entry/Exit                                                              │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  VMX Non-Root Mode (Guest OS)                                                       │  │  │
    │  │  │  • Most instructions run natively                                                  │  │  │
    │  │  │  • Sensitive instructions trap to hypervisor                                        │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  → Near-native performance                                                                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 가상화 기법 비교

| 기법 | 설명 | 성능 | 구현 |
|------|------|------|------|
| **Full Virtualization** | 완전 OS 지원 | 낮음 | HVM |
| **Para-virtualization** | 게스트 OS 수정 | 높음 | Xen PV |
| **Hardware-Assisted** | 하드웨어 지원 | 높음 | VT-x, AMD-V |

### Memory Virtualization (Shadow Page Tables)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Shadow Page Tables                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Guest Physical Memory (GPFN) → Host Physical Memory (HPFN) Mapping                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Shadow Page Table (Hypervisor-managed)                                                │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  GPFN    │  HPFN    │  Dirty  │  Accessed                                    │  │  │  │
    │  │  │  ────────┼─────────┼────────┼───────────                                 │  │  │  │
    │  │  │  0x1000  │  0x5000 │  0      │  1                                          │  │  │  │
    │  │  │  0x2000  │  0x6000 │  1      │  1                                          │  │  │  │
    │  │  │  0x3000  │  0x7000 │  0      │  0 (not yet)                                │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  Guest OS thinks it has physical memory 0x1000-0x3000                                │  │  │
    │  │  Hypervisor maps to scattered host pages                                           │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Guest has contiguous memory view                                                             │  │
    │  → Hypervisor can overcommit (swap if needed)                                                │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### KVM (Kernel-based Virtual Machine)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         KVM Architecture                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Linux Kernel + KVM Module                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  KVM Kernel Module                                                                     │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • /dev/kvm device creates VMs                                                        │  │  │  │
    │  │  │  • Each VM is a regular Linux process                                                │  │  │  │
    │  │  │  │    → Scheduled by Linux scheduler                                                 │  │  │  │
    │  │  │    │    → Managed by CFS (Completely Fair Scheduler)                                  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  QEMU (User Space)                                                                     │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • PC hardware emulation                                                       │  │  │  │
    │  │  │  • Device model (disk, NIC, display)                                              │  │  │  │
    │  │  │  • Provides BIOS, boot sequence                                                   │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                                 │  │
    │            ▼ /dev/km ioctl                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Guest OS (Linux/Windows)                                                               │  │  │
    │  │  → Believes it's running on bare metal                                                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 가상화 기대 효과

| 효과 | 물리 | 가상화 |
|------|------|--------|
| **활용률** | 10-20% | 60-80% |
| **전력** | 높음 | 낮음 |
| **유연성** | 낮음 | 높음 |
| **성능** | 100% | 85-95% |

### 모범 사례

1. **서버**: VMware vSphere, KVM
2. **데스크톱**: VirtualBox, VMware Fusion
3. **네트워크**: VLAN, SR-IOV
4. **스토리**: vSAN, NFS

### 미래 전망

1. **IOMMU**: 강화된 I/O 가상화
2. **GPU Passthrough**: GPGPU 가상화
3. **Nested**: 클라우드상 VM
