# 분기 예측 (Branch Prediction)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분기 예측은 **"파이프라인 **스톨 **방지**를 **위해 **분기 **방향**을 **미리 **예측**하는 **기술\"**으로, **Static**(컴파일 **러 **힌트)**과 **Dynamic**(실행 **시간 **학습)**이 **있고 **BTB**(Branch Target Buffer), **RAS**(Return Address Stack), **PHT**(Pattern History Table)**이 **핵심 **자료구조**이다.
> 2. **알고리즘**: **Bimodal**(1-bit, **2-bit **counter)**, **Two-Level Adaptive**(GAg, **GAp**, **PAp)**, **Hybrid**(Combining **Predictors)**, **Tournament**(Meta **Predictor)**이 **있고 **Global/Local **History**를 **조합**하여 **정확도**를 **향상**시킨다.
> 3. **성능**: **Misprediction Penalty**(파이프라인 **플러시)**은 **10-20 **사이클**이고 **95-98%** **정확도**가 **필요**하며 **Speculative Execution**(추측 **실행)**과 **Recovery**(롤백)**으로 **처리**한다.

## Ⅰ. 개요 (Context & Background)

### 개념
분기 예측은 **"조건문 미리 예측"**이다.

| 유형 | 원인 | 해결 |
|------|------|------|
| **Data** | RAW, WAR | Forwarding |
| **Control** | Branch | Prediction |
| **Structural** | Resource | 복제 |

## Ⅱ. 아키텍처 및 핵심 원리

### 분기 예측 파이프라인

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Branch Prediction Pipeline                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Without Prediction:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  IF: Fetch                                                                                 │  │
    │  ID: Decode ← branch instruction detected                                                │  │
    │  EX: Execute ← condition evaluated                                                        │  │
    │  └─ Stall pipeline until resolved                                                         │  │
    │  → 3+ cycles penalty per branch                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    With Prediction:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  IF: Fetch + Predict (BTB lookup) → Speculative fetch predicted target                    │  │
    │  ID: Decode (speculative)                                                                │  │
    │  EX: Execute + Verify (actual condition)                                                 │  │
    │  └─ If correct: Continue                                                                 │  │
    │     If wrong: Flush pipeline, restart from correct target                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### BTB (Branch Target Buffer)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Branch Target Buffer                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Structure:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  BTB Entry (Branch Instruction Cache)                                                    │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Tag (PC): 0x4000ABC0                      │  Valid: 1                             │  │  │
    │  │  Target: 0x4000AB00                       │  Type: Conditional                      │  │  │
    │  │  History: 11 (strongly taken)            │                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Lookup Process:                                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. IF stage: PC used to index BTB                                                  │  │  │
    │  │  2. If hit: Fetch from predicted target, not PC + 4                                 │  │  │
    │  │  3. Prediction history used to predict direction                                    │  │  │
    │  │  4. EX stage: Verify prediction, update history if wrong                            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 2-Bit Counter (Bimodal Predictor)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         2-Bit Counter State Machine                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    States:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  State    │  Binary  │  Prediction    │  Change on                                  │  │  │
    │  │  ─────────────────────────────────────────────────────────────────────────────────────│  │  │
    │  │  Strongly Taken (ST)   │  11  │  Taken         │  Taken → ST, Not T → WT             │  │  │
    │  │  Weakly Taken (WT)     │  10  │  Taken         │  Taken → ST, Not T → WC             │  │  │
    │  │  Weakly Not Taken (WC) │  01  │  Not Taken     │  Taken → WT, Not T → SC            │  │  │
    │  │  Strongly Not Taken (SC)│  00  │  Not Taken     │  Taken → WC, Not T → SC            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Example: Loop behavior
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Loop: for (i = 0; i < 10; i++) { ... }                                                 │  │
    │                                                                                         │  │
    │  Iteration  Branch Actual  Counter State  Prediction  Result                              │  │
    │  ───────────────────────────────────────────────────────────────────────────────────────│  │
    │  1           Taken       00 → 01        NT          Wrong (flush)                       │  │
    │  2           Taken       01 → 10        NT          Wrong (flush)                       │  │
    │  3-9         Taken       10 → 11        T           Correct                            │  │
    │  10          Not Taken   11 → 10        T           Wrong (flush)                       │  │
    │  11 (exit)   -           -             -           -                                  │  │
    │                                                                                         │  │
    │  → Last iteration always mispredicted (one iteration penalty)                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Two-Level Adaptive Predictor (GAg)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Two-Level Adaptive (GAg)                                           │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    GAg = Global history, Global prediction
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Global History Register (GHR): Tracks last N branch outcomes                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  GHR (8-bit): 10110110                                                               │  │  │
    │  │  └─ 1=Taken, 0=Not Taken (LSB = most recent)                                         │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │           │                                                                              │  │
    │           ▼ Index into PHT                                                               │  │
    │  2. Pattern History Table (PHT): 2-bit counters indexed by GHR                           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Index    │  Counter  │  State                                                          │  │  │
    │  │  ─────────────────────────────────────────────────────────────────────────────────────│  │  │
    │  │  10110110│  11       │  ST (Strongly Taken)                                         │  │  │
    │  │  10110111│  10       │  WT (Weakly Taken)                                           │  │  │
    │  │  ...                                                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Prediction Process:                                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Current GHR = 10110110                                                            │  │  │
    │  │  2. Index PHT[10110110] → state = 11 (ST)                                            │  │  │
    │  │  3. Predict: TAKEN                                                                    │  │  │
    │  │  4. Actual outcome: TAKEN                                                             │  │  │
    │  │  5. Update: Shift GHR ← 1, update counter if needed                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Captures correlation patterns across different branches                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### RAS (Return Address Stack)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Return Address Stack                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  call foo    ← Push return address to RAS                                                │  │
    │      ...                                                                                 │  │
    │  ret         ← Pop return address from RAS                                               │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    RAS Structure:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │

    │  │  │  ... │  ...                                                                      │  │  │  │

    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │

### Hybrid Predictor (Tournament)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Tournament Predictor                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Branch Input                                                                           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │                          │                                                           │  │  │
    │  │            ┌─────────────┴─────────────┐                                             │  │  │
    │  │            ▼                           ▼                                             │  │  │
    │  │  ┌──────────────────┐      ┌──────────────────┐                                    │  │  │
    │  │  │  Local Predictor │      │  Global Predictor│                                    │  │  │
    │  │  │  (Per-branch     │      │  (GAg)          │                                    │  │  │
    │  │  │   history)       │      │                 │                                    │  │  │
    │  │  │  Prediction: T    │      │  Prediction: NT │                                    │  │  │
    │  │  └──────────────────┘      └──────────────────┘                                    │  │  │
    │  │            │                           │                                             │  │  │
    │  │            └─────────────┬─────────────┘                                             │  │  │
    │  │                          ▼                                                           │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │

    │  │  │  ┌──────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  │  History: Which predictor was correct recently                            │  │  │  │
    │  │  │  │  State: 10 (prefer Local)                                                 │  │  │  │
    │  │  │  │  → Final Prediction: TAKED (use Local)                                     │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                          │                                                           │  │  │
    │  │                          ▼                                                           │  │  │
    │  │                   Final Prediction: TAKED                                            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 인텔 분기 예측
**상황**: Skylake 아키텍처
**판단**: Multi-level + RAS + I-TLB

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Intel Skylake Branch Prediction                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Components:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Decode Stream Frontend                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • MITE (Micro-instruction Translation Engine): Legacy decode                         │  │  │
    │  │  • DSB (Decoded Stream Buffer): μop cache                                            │  │  │
    │  │  • LSD (Loop Stream Detector): Lock-free loops                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  2. Branch Prediction Units                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • BTB: 4-way set associative, ~2000 entries                                        │  │  │
    │  │  • RAS: 16-entry return address stack                                                │  │  │
    │  │  • Indirect BTB: 512 entries for indirect jumps                                      │  │  │
    │  │  • TAGE: TAgged GEometric history length predictor                                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  3. Speculative Execution                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • ROB (Reorder Buffer): 224 entries                                                  │  │  │
    │  │  • RS (Reservation Station): 97 entries (integer), 288 (FP)                          │  │  │
    │  │  • Misprediction recovery: Flush ROB, restart fetch                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```
