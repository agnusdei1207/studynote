# IOMMU (Input-Output Memory Management Unit)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IOMMU는 **"I/O 장치의 **DMA 주소를 **변환하고 **메모리 접근을 **제어하는 하드웨어"**로, **CPU의 MMU**와 유사하게 **가상 주소**(Device Virtual Address)를 **물리 주소**(Physical Address)로 **매핑**하고 **격리**를 제공한다.
> 2. **가치**: **DMA 공격**(Malicious Device가 임의 메모리 접근)을 **방지**하고 **가상화 환경**에서 **Device Passthrough**를 지원하며 **32-bit 장치**가 **64-bit 메모리**에 접근할 수 있게 **주소 변환**을 제공한다.
> 3. **융합**: **Intel VT-d**(Virtualization Technology for Directed I/O), **AMD-Vi**(AMD-V I/O Virtualization), **ARM SMMU**로 구현되며 **SR-IOV**(Single Root I/O Virtualization)와 함께 **가상 머신 간 장치 공유**를 가능하게 한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
IOMMU는 **"I/O 장치용 메모리 관리 장치"**이다.

**IOMMU의 기능**:
- **Address Translation**: 가상→물리 주소 변환
- **Memory Protection**: 허용된 영역만 접근
- **Interrupt Remapping**: 인터럽트 가상화

### 💡 비유
IOMMU는 **"입구 보안 검색대"**와 같다.
- **손님**: I/O 장치
- **출입증**: IOMMU 권한
- **구역**: 보호된 메모리 영역

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         IOMMU의 필요성                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

DMA (Direct Memory Access):
    • 장치가 직접 메모리 접근
    • 보안 취약점
         ↓
Malicious Device:
    • 악의적인 DMA 장치
    • 커널 메모리 변조 가능
         ↓
IOMMU (2006년대):
    • Intel VT-d
    • AMD-Vi
    • 장치별 메모리 접근 제어
         ↓
현대:
    • 가상화 필수 구성 요소
    • PCIe 표준
```

└─────────────────────────────────────────────────────────────────────────────────────────┘

    [IOMMU 없는 DMA]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Device                System Bus                    Memory                           │
    │  ┌────────┐            ┌─────────┐                  ┌────────────────────────────────┐  │
    │  │ NIC    │──DMA──────>│         │──────────────────>│ Physical Address 0x10000000    │  │
    │  │ (32bit)│  0x1000    │  Bridge │                  │ (Device가 직접 접근)           │  │
    │  └────────┘            └─────────┘                  └────────────────────────────────┘  │
    │                                                                                         │
    │  문제점:                                                                                 │
    │  • Device가 임의의 물리 주소 접근 가능                                                    │
    │  • 32-bit 장치는 4GB 이상 메모리 접근 불가                                                │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [IOMMU 있는 DMA]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐

    │  │ NIC    │         │ Page    │        │         │           │ Physical Memory       │  │
    │  │        │         │ Table   │        │ Bridge  │           │ ┌───────────────────┐  │  │
    │  │ DMA    │───────>│ Lookup  │───────>│         │───────────>│ │ 0x1000 (IOVA)    │──┼──>│
    │  │ IOVA   │         │ 0x1000  │        │         │           │ │       │          │  │  │
    │  │ 0x1000 │         │    │    │        │         │           │ │       ▼          │  │  │
    │  └────────┘         │    ▼    │        └─────────┘           │ │ 0x80000000 (PA) │  │  │
    │                     │ 0x8000  │                           │ │ (실제 메모리)    │  │  │
    │                     │ 0000    │                           │ └───────────────────┘  │  │
    │                     └─────────┘                           └───────────────────────┘  │
    │                        ▲                                                              │
    │                        │                                                               │
    │                     OS 설정                                                           │
    │                     (Domain Mapping)                                                   │
    │                                                                                         │
    │  장점:                                                                                   │
    │  • IOVA 0x1000 → PA 0x800000000으로 변환                                                │
    │  • Device는 64-bit 메모리 접근 가능                                                      │
    │  • 허용되지 않은 주소 접근 시 Exception 발생                                              │

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         IOMMU Page Table 구조                                           │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • CPU MMU와 유사한 계층 구조                                                            │
    │  │  • Device별로 독립적인 Address Space (Domain)                                        │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Root Entry                                                                          │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  Device ID (BDF) ──> Context Table ──> Page Table                             │  │  │
    │  │  │                                                                              │  │  │
    │  │  │  [Page Table Entry]                                                           │  │  │
    │  │  │  ┌──────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  │  Bit 0: Present (P)                                                       │  │  │  │
    │  │  │  │  Bit 1: Read/Write (R/W)                                                  │  │  │  │
    │  │  │  │  Bit 2-11: Reserved                                                       │  │  │  │
    │  │  │  │  Bit 12-51: Physical Page Number (PPN)                                    │  │  │  │
    │  │  │  │  Bit 52-63: Reserved                                                       │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [주소 변환 예시]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Device DMA Request: IOVA = 0x0000_0000_1000                                           │
    │                                                                                         │
    │  1. IOMMU가 Device ID(BDF) 확인                                                          │
    │  2. Context Table → Page Table 탐색                                                     │
    │  3. Page Table Entry[PTE_INDEX(0x1000)] 조회                                            │
    │     → PTE = 0x80000_0003 (Present + R/W + PPN=0x80000)                                  │
    │  4. Physical Address = 0x80000_0000 + 0x1000 = 0x80000_1000                             │
    │  5. Memory Bus에 PA 0x80000_1000로 전송                                                │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### IOMMU vs MMU

| 구분 | MMU | IOMMU |
|------|-----|-------|
| **대상** | CPU | I/O Device |
| **주소** | VA→PA | IOVA→PA |
| **TLB** | O | O (IOTLB) |
| **목적** | 가상 메모리 | DMA 보호/변환 |

### 과목 융합 관점 분석

#### 1. 운영체제 ↔ IOMMU
- **Device Driver**: DMA Mapping 설정
- **VFIO**: User-space Device 접근

#### 2. 가상화 ↔ IOMMU
- **Passthrough**: 장치 전용
- **SR-IOV**: 장치 공유

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: IOMMU 설정 확인
**상황**: 가상화 환경에서 장치 Passthrough
**판단**:

```bash
# 1. IOMMU 활성화 확인 (Linux)
$ dmesg | grep -e IOMMU -e DMAR
# [    0.000000] DMAR: IOMMU enabled

# 2. IOMMU 그룹 확인
$ ls -l /sys/kernel/iommu_groups/
# total 0
# drwxr-xr-x 2 root root 0 ... 0/
# drwxr-xr-x 2 root root 0 ... 1/
# $ readlink /sys/kernel/iommu_groups/0/devices/0000:00:02.0
# ../../../../devices/pci0000:00/0000:00:02.0

# 3. IOMMU 그룹별 장치 확인
$ find /sys/kernel/iommu_groups/ -type l

# 4. VFIO 장치 바인딩
$ echo "vfio-pci" > /sys/bus/pci/devices/0000:00:02.0/driver_override
$ echo "0000:00:02.0" > /sys/bus/pci/drivers/vfio-pci/bind

# 5. QEMU/KVM Passthrough
$ qemu-system-x86_64 \
    -device vfio-pci,host=00:02.0 \
    ...
```

### IOMMU 기대 효과

| 효과 | IOMMU 없을 시 | IOMMU 있을 시 |
