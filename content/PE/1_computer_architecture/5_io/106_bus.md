# 시스템 버스 (System Bus)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 버스는 **"CPU, Memory, I/O 장치 간 **데이터 전송을 위한 **공유 통신로"**로, **Address Bus**(주소 전달), **Data Bus**(데이터 전송), **Control Bus**(제어 신호)로 구성되며 **Bus Width**에 따라 **전송 가능한 데이터 양**이 결정된다.
> 2. **가치**: **시스템의 모든 구성 요소**를 **연결**하는 **백본**으로 **확장성**(Extension Slot)과 **표준화**(PCI, USB)를 제공하며 **Bus Arbitration**(중재)으로 **충돌을 방지**하고 **전송 효율**을 높인다.
> 3. **융합**: **ISA Bus**(8/16-bit), **PCI Bus**(32/64-bit), **PCIe**(Serial Lane)로 진화하며 **Bus Mastering**, **Plug and Play**, **Hot Swappable** 기능을 지원하고 **SMP**(Symmetric Multiprocessing)에서는 **Front Side Bus(FSB)**가 **CPU 간 통신**을 담당한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
시스템 버스는 **"컴퓨터 내부 구성 요소 간 데이터를 전송하는 공유 통신 경로"**이다.

**버스의 기본 원리**:
- **Shared Medium**: 모든 장치가 공유
- **Addressed Communication**: 주소로 대상 지정
- **Arbitrated**: 동시 접근 중재

### 💡 비유
시스템 버스는 **"고속도로"**와 같다.
- **차선**: Bus Width
- **목적지**: Address
- **신호등**: Control Signal

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         시스템 버스의 발전                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Von Neumann Architecture:
    • 단일 버스
    • 병목 현상
         ↓
ISA Bus (1981):
    • 8/16-bit Data Bus
    • PC/AT 표준
         ↓
PCI Bus (1992):
    • 32/64-bit
    • Plug & Play
         ↓
PCIe (2002):
    • Serial, Lane-based
    • Hot Swappable
```

### Address Bus, Data Bus, Control Bus

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         버스 종류별 특징                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Address Bus]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 용도: 메모리/장치 주소 전송                                                           │
    │  │  • 방향: CPU → Memory/I/O (단방향)                                                    │
    │  │  • 폭: 32-bit = 4GB 주소 공간, 64-bit = 16EB                                         │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  CPU  ──[0x00001000]──>  Memory Controller                                         │  │
    │  │        (Address)                                                                     │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Data Bus]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 용도: 실제 데이터 전송                                                                 │
    │  │  • 방향: 양방향 (Bidirectional)                                                       │
    │  │  • 폭: 64-bit = 8바이트/사이클                                                       │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  CPU  <───[0xFF00AABB]───>  Memory                                                  │  │
    │  │        (Data)                                                                        │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Control Bus]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 용도: 제어 신호 전송                                                                  │
    │  │  • 방향: CPU ↔ 장치 (양방향)                                                          │
    │  │  • 주요 신호:                                                                      │
    │  │  - READ: 메모리 읽기 요청                                                            │
    │  │  - WRITE: 메모리 쓰기 요청                                                           │
    │  │  - IRQ: 인터럽트 요청                                                                │
    │  │  - RESET: 시스템 리셋                                                                │
    │  │  - CLK: 클럭 신호                                                                   │
    │  │  - READY: 데이터 준비 완료                                                           │
    │  └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Bus Arbitration (버스 중재)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Bus Arbitration 방식                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Centralized Arbitration]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 단일 Bus Controller가 중재                                                           │
    │  │  • Daisy Chaining, Polling, Independent Request                                       │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Master 1  ────┐                                                                     │  │
    │  │  Master 2  ────┼──> Bus Controller ──> Bus Grant                                   │  │
    │  │  Master 3  ────┘         (중재)                                                     │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Distributed Arbitration]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 각 Master가 자체 중재                                                                 │
    │  │  • Collision Detection, Priority Resolution                                           │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Master 1 ──┐                                                                       │  │
    │  │  Master 2 ──┼──> Bus (Self-Arbitration)                                             │  │
    │  │  Master 3 ──┘                                                                       │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Bus Timing

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Bus Timing (Synchronous vs Asynchronous)                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Synchronous Bus]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Clock Edge 기반 동기화                                                               │
    │  │  • 고속 전송 가능                                                                    │
    │  │  │  • 모든 장치가 같은 Clock 사용                                                     │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Clock   ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐                                              │  │
    │  │          │  │  │  │  │  │  │  │  │  │                                              │  │
    │  │          └──┘  └──┘  └──┘  └──┘  └──┘                                              │  │
    │  │                ↑      ↑      ↑                                                      │  │
    │  │  Address  ─────┤      │      │                                                      │  │
    │  │  Data     ────────────┤      │                                                      │  │
    │  │  └───────────────────────────────────────────────────────────────────────────────────┘  │
    │  │                                                                                      │
    │  │  예: CPU Front Side Bus, Memory Bus                                                  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Asynchronous Bus]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Handshake 신호 기반                                                                  │
    │  │  • 다양한 속도 장치 지원                                                              │
    │  │  │  • Request/Acknowledge 프로토콜                                                    │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Master  ──[Request]──>  Slave                                                      │  │
    │  │                         ┌─[Acknowledge]─>                                          │  │
    │  │  Master  <───────────────┘                                                          │  │
    │  │  Master  ──[Data]──────>  Slave                                                      │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  예: PCI Bus, ISA Bus                                                                  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 버스 종류 비교

| 구분 | System Bus | I/O Bus | Local Bus |
|------|-----------|--------|-----------|
| **위치** | 메인보드 | 확장 슬롯 | CPU 근접 |
| **속도** | 중간 | 느림 | 빠름 |
| **예시** | ISA, PCI | USB, SATA | AGP, PCIe |

### 과목 융합 관점 분석

#### 1. 운영체제 ↔ Bus
- **DMA**: Bus Mastering
- **Interrupt**: IRQ Line
- **PnP**: Bus Enumeration

#### 2. 가상화 ↔ Bus
- **Passthrough**: 직접 Bus 접근
- **Virtual Device**: Emulated Bus

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: Bus 병목 분석
**상황**: 시스템 성능 저하
**판단**:

```bash
# 버스 정보 확인 (Linux)
$ lspci -vv
$ sudo lshw -businfo

# PCIe Bandwidth 계산
# Bandwidth = Lane Count * Transfer Rate * Encoding
# PCIe 3.0: 8 GT/s, 128b/130b encoding
# x4 = 4 * 8 GT/s * (128/130) ≈ 3.9 GB/s

# Bus utilization 확인
$ pcie-top  # PCIe bandwidth monitor

# DMA Bus Master 확인
$ dmesg | grep -i dma
```

## Ⅴ. 기대효과 및 결론

### 시스템 버스 기대 효과

| 효과 | 버스 없을 시 | 버스 있을 시 |
|------|-----------|-----------|
| **확장성** | 낮음 (납땜) | 높음 (Slot) |

1. **PCIe 6.0**: 64 GT/s
2. **CXL**: Cache Coherent
3. **NVLink**: GPU-GPU Direct
