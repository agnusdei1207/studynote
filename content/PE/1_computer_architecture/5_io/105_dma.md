# DMA (Direct Memory Access)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DMA는 **"CPU 개입 없이 **I/O 장치와 **메모리 간 **데이터 전송을 **직접 수행하는 기술"**로, **DMA Controller(DMAC)**가 **System Bus를 제어**하여 **CPU의 부하**를 줄이고 **데이터 전송 효율**을 높인다.
> 2. **가치**: **Interrupt-Driven I/O**보다 **CPU 효율**이 높으며 **대량 데이터 전송**(디스크 I/O, 네트워크 패킷, 그래픽스)에 **적합**하고 **Bus Mastering**으로 **CPU를 우회**하여 **고속 전송**이 가능하다.
> 3. **융합**: **ISA Bus**, **PCI Bus**의 **Bus Mastering DMA**로 **PnP(Plug and Play)**를 지원하며 **Network Card**, **Sound Card**, **Disk Controller**가 **DMA**를 사용하고 **IOMMU(Input-Output Memory Management Unit)**는 **DMA의 메모리 보호**를 위한 **가상화 지원**을 제공한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
DMA는 **"CPU를 거치지 않고 I/O 장치가 메모리에 직접 접근하는 기술"**이다.

**DMA의 필요성**:
- **CPU 부하 감소**: I/O 전송 중 CPU 해방
- **고속 전송**: 바이트 단위가 아니고 블록 단위
- **병렬 처리**: I/O와 CPU 동작 병행

### 💡 비유
DMA는 **"화물 배달"**와 같다.
- **CPU 없을 시**: 사람이 직접 배달 (PIO)
- **DMA 있을 시**: 택배 회사가 배달

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         DMA의 발전                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Programmed I/O:
    • CPU가 모든 데이터 전송
    • 바이트 단위 느린 전송
         ↓
Interrupt-Driven I/O:
    • 바이트/블록 단위
    • 여전히 CPU 개입
         ↓
DMA (1960년대):
    • IBM System/360
    • DMAC가 Bus 제어
         ↓
현대:
    • Bus Master DMA
    • Scatter-Gather DMA
    • IOMMU (가상화 지원)
```

## Ⅱ. 아키텍처 및 핵심 원리

### DMA 동작 원리

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         DMA 전송 과정                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [1. DMA 설정]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  CPU                                        DMA Controller                           │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  1. I/O 장치가 DMA 요청                                                               │  │  │
    │  │  │  2. CPU가 DMA Controller 설정:                                                        │  │  │
    │  │  │     • Source Address: I/O 장치 버퍼 주소                                           │  │  │
    │  │  │     • Destination Address: 메모리 주소                                              │  │  │
    │  │  │     • Transfer Count: 전송할 바이트 수                                               │  │  │
    │  │  │     • Control: Direction (Read/Write)                                              │  │  │
    │  │  └────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [2. CPU는 다른 작업 수행]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • DMA 전송 동안 CPU는 다른 프로세스 실행                                                     │
    │  │  • CPU는 I/O 전송으로부터 자유로움                                                           │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [3. DMA 전송 (Cycle Stealing)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  ┌─────────┐   ┌──────────────┐   ┌──────────────┐   ┌──────────────┐             │  │
    │  │  │   I/O    │   │ DMA Ctrl     │   │ System Bus   │   │   Memory     │             │  │
    │  │  │ Device  │   │              │   │              │   │              │             │  │
    │  │  └────┬────┘   └──────┬───────┘   └──────┬───────┘   └──────┬───────┘             │  │
    │  │       │              │                  │                  │                     │  │
    │  │       │              ▼                  ▼                  ▼                     │  │
    │  │       │         ┌────────────────────────────────────────────────────────┐        │  │
    │  │       │         │  1. DMA Controller가 Bus Request (BR)                   │        │  │
    │  │       │         │  2. CPU가 Bus Grant (BG) - 현재 사이클 완료 후           │        │  │
    │  │       │         │  3. DMA Controller가 Bus 제어 (Bus Master)             │        │  │
    │  │       │         │  4. 데이터 전송: I/O → Memory (또는 반대)                 │        │  │
    │  │       │         │  5. 전송 완료 후 Bus 반환                                   │        │  │
    │  │       │         └────────────────────────────────────────────────────────┘        │  │
    │  │       │                                                                             │  │
    │  │       └───────────────────── DMA Transfer (CPU는 Bus 사용 못 함) ──────────────┘  │
    │  │                                                                                      │  │
    │  │  [Cycle Stealing]                                                                      │  │
    │  │  • DMA가 Bus를 사용할 때 CPU는 기다려야 함                                               │  │
    │  │  • 전송 단위가 크면 CPU 효율 좋아짐                                                      │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [4. DMA 완료 인터럽트]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  DMA Controller                                         CPU                        │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  1. 전송 완료                                                                        │  │  │
    │  │  │  2. DMA Controller가 CPU에 Interrupt 발생                                              │  │  │
    │  │  │  3. CPU가 Interrupt Service Routine(ISR) 실행                                          │  │  │
    │  │  │     • 전송 완료 처리                                                                  │  │  │
    │  │  │     • 다음 작업 준비                                                                  │  │  │
    │  │  └────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### DMA Mode

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         DMA 전송 모드                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Burst Mode]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 한 번 Bus 획득으로 전체 블록 연속 전송                                                       │
    │  │  • CPU가 오랫동안 Bus 사용 못 함                                                       │
    │  │  • 전송 속도 빠름                                                                      │
    │                                                                                         │
    │  예: 디스크 섹터 읽기 (512Byte x N 연속 전송)                                             │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Cycle Stealing Mode]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 바이트/워드 단위로 전송                                                                 │
    │  │  • CPU와 DMA가 Bus를 번갈아 사용                                                           │
    │  │  • CPU에 방해 적음                                                                      │
    │                                                                                         │
    │  예: 키보드 입력 (1바이트씩 전송)                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Transparent Mode]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • CPU가 메모리를 사용하지 않을 때만 DMA 전송                                               │
    │  │  • CPU와 DMA가 완전히 병행                                                               │
    │  │  • 전송 속도 느림                                                                      │
    │                                                                                         │
    │  예: 배경 데이터 전송                                                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Scatter-Gather DMA

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Scatter-Gather DMA                                               │

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 불연속 메모리 영역 간 전송 자동화                                                         │
    │  │  • CPU 개입 없이 여러 메모리 블록 전송                                                    │
    │                                                                                         │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Example: Network Packet Assembly                                                     │  │
    │  │  ┌──────────┐  ┌──────────┐  ┌──────────┐                                   │  │
    │  │  │ Header   │  │  Body    │  │  Trailer  │                                   │  │

    │  │  └────┬─────┘  └────┬─────┘  └────┬─────┘                                   │  │
    │  │       │              │              │                                          │  │
    │  │       ▼              ▼              ▼                                          │  │

    └──────────────────────────────────────────────────────────────────────────────────────────┘

## Ⅲ. 융합 비교 및 다각도 분석

### PIO vs DMA

| 구분 | PIO (Programmed I/O) | DMA |
|------|---------------------|-----|
| **CPU 개입** | 항상 | 전송 시작/종료만 |
| **전송 단위** | 바이트 | 블록 |
| **속도** | 느림 | 빠름 |
| **복잡도** | 낮음 | 높음 (DMAC) |

### 과목 융합 관점 분석

#### 1. 운영체제 ↔ DMA
- **Buffer Cache**: DMA 전송 최적화
- **Buffer I/O**: 사용공간 DMA

#### 2. 가상화 ↔ DMA
- **IOMMU**: DMA 주소 변환
- **Passthrough**: 장치 전용 DMA

```

## Ⅴ. 기대효과 및 결론

### DMA 기대 효과

| 효과 | PIO | DMA |
|------|-----|-----|
| **CPU 활용** | 낮음 (전송 중) | 높음 |
| **전송 속도** | 느림 | 빠름 |
| **하드웨어** | 간단 | DMAC 필요 |
| **사용** | 소량 데이터 | 대량 데이터 |

### 미래 전망

1. **IOMMU**: 가상화 DMA 격리
2. **NVMe**: Direct Access
3. **RDMA**: Remote DMA

### ※ 참고 표준/가이드
- **Intel**: I/O Architecture
- **PCI**: Bus Mastering
- **ARM**: DMA Controller
