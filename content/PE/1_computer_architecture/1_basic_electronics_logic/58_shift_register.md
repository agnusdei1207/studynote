# 시프트 레지스터 (Shift Register)

## 정의

플립플롭을 직렬로 연결하여 **클럭마다 데이터를 한 비트씩 이동**시키는 순차 논리회로. 직렬-병렬 변환, 지연 회로, 시리얼 통신 등에 활용된다.

```
┌─────────────────────────────────────────────────────────────┐
│          4비트 시프트 레지스터 (Shift Register)              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Serial                                                      │
│  Input ─────┐                                               │
│    (SI)     │                                               │
│             ▼                                               │
│         ┌───────┐     ┌───────┐     ┌───────┐     ┌───────┐│
│         │ D FF  │────▶│ D FF  │────▶│ D FF  │────▶│ D FF  ││
│         │  Q0   │     │  Q1   │     │  Q2   │     │  Q3   ││
│         └───┬───┘     └───┬───┘     └───┬───┘     └───┬───┘│
│             │             │             │             │     │
│             ▼             ▼             ▼             ▼     │
│            Q0            Q1            Q2            Q3    │
│             │             │             │             │     │
│             └─────────────┴─────────────┴─────────────┘     │
│                           │                                 │
│                           ▼                                 │
│                    Parallel Output (PO)                     │
│                                                             │
│              ┌─────────────────┐                           │
│              │       CLK       │  ← 모든 FF에 공통 클럭     │
│              └─────────────────┘                           │
│                                                             │
│                      Serial Output (SO) ────▶ Q3            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4비트 시프트 레지스터 구조 상세 설명

시프트 레지스터는 D 플립플롭을 직렬로 연결하여 데이터가 클럭마다 오른쪽으로 한 비트씩 이동하는 구조이다. 위 다이어그램은 4비트 우측 시프트 레지스터의 기본 구조를 보여준다.

**데이터 흐름**은 왼쪽에서 오른쪽으로 진행된다. Serial Input(SI)으로 들어온 데이터가 첫 번째 D 플립플롭(FF0)의 D 입력단에 인가된다. 클럭 상승 에지가 도달하면, SI 값이 Q0에 저장되고, 동시에 Q0의 이전 값이 Q1으로, Q1의 이전 값이 Q2로, Q2의 이전 값이 Q3으로 이동한다. 이를 **Right Shift**라 한다.

**직렬-병렬 변환(Serial-to-Parallel Conversion)** 기능이 시프트 레지스터의 핵심 응용이다. 4비트 데이터를 SI로 한 비트씩 순차적으로 입력한 후, 4 클럭이 지나면 전체 데이터가 Q0~Q3에 병렬로 저장된다. 이를 통해 직렬 통신으로 수신한 데이터를 CPU가 병렬로 처리할 수 있다.

**병렬 출력(Parallel Output)**은 Q0~Q3에서 동시에 읽을 수 있다. 시리얼 입력이 완료된 후 이 출력들을 한 번에 읽으면 4비트 병렬 데이터가 된다. 반대로 Q3는 Serial Output(SO)로 사용하여, 저장된 데이터를 다시 직렬로 출력할 수도 있다.

```
┌─────────────────────────────────────────────────────────────┐
│         시프트 레지스터 동작 예시 (4 클럭 사이클)             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  입력 데이터: 1 → 0 → 1 → 1 (LSB 우선, SI로 입력)           │
│                                                             │
│  클럭  │  SI  │  Q0  │  Q1  │  Q2  │  Q3  │  병렬 출력      │
│  ──────┼──────┼──────┼──────┼──────┼──────┼──────────────   │
│   초기 │  -   │  0   │  0   │  0   │  0   │  0000          │
│    1   │  1   │  1   │  0   │  0   │  0   │  0001          │
│    2   │  0   │  0   │  1   │  0   │  0   │  0010          │
│    3   │  1   │  1   │  0   │  1   │  0   │  0101          │
│    4   │  1   │  1   │  1   │  0   │  1   │  1011 (완료)   │
│                                                             │
│  ↑ 각 클럭마다 데이터가 오른쪽으로 1비트씩 이동              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 시프트 레지스터 동작 파형 분석

위 타임 테이블은 4비트 데이터 1011이 시프트 레지스터에 로드되는 과정을 보여준다. LSB(최하위 비트)부터 먼저 입력되는 것을 가정하며, 4 클럭 사이클 후에 전체 데이터가 로드된다.

**초기 상태**에서 모든 플립플롭은 0으로 리셋되어 있다. Q0=Q1=Q2=Q3=0이며, 병렬 출력은 0000을 나타낸다.

**클럭 1**에서 SI=1(첫 번째 비트)이 입력된다. 클럭 상승 에지 후 Q0=1, Q1=Q2=Q3=0이 된다. 병렬 출력은 0001이며, 아직 데이터 로드가 완료되지 않았다.

**클럭 2**에서 SI=0(두 번째 비트)이 입력된다. Q0=0, Q1=1(이전 Q0), Q2=Q3=0이 된다. 병렬 출력은 0010이다.

**클럭 3**에서 SI=1(세 번째 비트)이 입력된다. Q0=1, Q1=0, Q2=1, Q3=0이 된다. 병렬 출력은 0101이다.

**클럭 4**에서 SI=1(네 번째 비트)이 입력된다. Q0=1, Q1=1, Q2=0, Q3=1이 되어 병렬 출력은 1011로 **데이터 로드가 완료**된다. 이제 CPU는 Q0~Q3을 병렬로 읽어 4비트 데이터 1011을 획득할 수 있다.

```
┌─────────────────────────────────────────────────────────────┐
```

### 시프트 레지스터의 다양한 유형 분석

시프트 레지스터는 입력/출력 방식에 따라 네 가지 기본 유형과 양방향 유형으로 분류된다. 각 유형은 특정 응용 분야에 최적화되어 있다.

**SISO(Serial In - Serial Out)**는 가장 단순한 형태로, 직렬 입력이 직렬 출력으로 나간다. n비트 레지스터는 n 클럭 사이클의 **지연(Delay)**을 제공하며, 이를 디지털 지연 회로로 활용한다. 예를 들어, 8비트 SISO 레지스터는 입력 신호를 8 클럭 사이클 후에 출력한다.

**SIPO(Serial In - Parallel Out)**는 직렬 데이터를 병렬로 변환한다. 시리얼 통신(UART, SPI, I2C)에서 수신된 데이터를 CPU나 메모리로 전달할 때 필수적이다. n비트 SIPO는 n 클럭 후에 n비트 병렬 데이터를 제공한다.

**PISO(Parallel In - Serial Out)**는 SIPO의 반대로, 병렬 데이터를 직렬로 변환한다. CPU나 메모리의 병렬 데이터를 시리얼 통신으로 전송할 때 사용된다. Load 신호로 병렬 데이터를 한 번에 로드한 후, n 클럭에 걸쳐 직렬로 출력한다.

**PIPO(Parallel In - Parallel Out)**는 일반적인 병렬 레지스터와 유사하지만, 시프트 기능이 추가될 수 있다. 병렬 데이터를 저장하고 필요할 때 시프트 연산을 수행할 수 있다.

**양방향 시프트 레지스터**는 좌우 양방향으로 시프트할 수 있다. ALU의 시프트 연산(<<, >>) 구현에 사용되며, 방향 제어 신호(Left/Right)에 따라 시프트 방향을 선택한다.

```
┌─────────────────────────────────────────────────────────────┐
│         양방향 시프트 레지스터 (Bidirectional Shift)         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Control: S1 S0                                            │
│           │  │                                              │
│           │  └─▶ 0: Hold (유지)                             │
│           └────▶ 0 0: Hold                                  │
│                  0 1: Right Shift (우측 시프트)              │
│                  1 0: Left Shift (좌측 시프트)               │
│                  1 1: Load (병렬 로드)                      │
│                                                             │
│         Left In                            Right In         │
│            (LI)                              (RI)            │
│             │                                 │              │
│             ▼                                 ▼              │
│  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐                │
│  │  MUX  │─▶│  D FF │─▶│  D FF │─▶│  D FF │─▶ ...          │
│  │4-to-1 │  │  Q0   │  │  Q1   │  │  Q2   │                │
│  └───┬───┘  └───┬───┘  └───┬───┘  └───┬───┘                │
│      │          │          │          │                     │
│      └──────────┴──────────┴──────────┘                     │
│                 │                                           │
│   ┌─────────────┼─────────────┐                            │
│   │             │             │                            │
│   ▼             ▼             ▼                            │
│  LI    Q1  Q0  RI                                            │
│  (Hold)(Left)(Right)(Load)                                  │
│                                                             │
│  MUX 입력:                                                   │
│  - 00: Q0 (Hold - 현재 값 유지)                             │
│  - 01: RI (Right Shift - 우측 입력)                         │
│  - 10: Q1 (Left Shift - 다음 비트)                          │
│  - 11: LI (Load - 병렬 로드 입력)                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 양방향 시프트 레지스터 내부 구조 분석

양방향 시프트 레지스터는 각 플립플롭 앞에 4-to-1 MUX를 배치하여 시프트 방향과 동작 모드를 선택한다. 위 다이어그램은 이 구조의 핵심을 보여준다.

**제어 신호 S1, S0**는 4가지 동작 모드를 선택한다. 00은 Hold 모드로, 현재 Q0 값을 그대로 유지한다. 01은 Right Shift 모드로, RI(Right Input)를 Q0에 로드한다. 10은 Left Shift 모드로, Q1을 Q0에 로드하여 데이터가 왼쪽으로 이동한다. 11은 Load 모드로, 병렬 입력 LI를 Q0에 로드한다.

**MUX의 역할**은 각 동작 모드에 맞는 데이터를 선택하여 D 플립플롭의 D 입력에 공급하는 것이다. 예를 들어, Right Shift 시 RI가 선택되어 Q0에 저장되고, Left Shift 시 Q1이 선택되어 Q0에 저장된다. 이를 통해 단일 클럭 사이클에 시프트 방향을 변경할 수 있다.

**Hold 모드의 필요성**은 파이프라인 설계에서 중요하다. 명령어 실행 중 특정 단계에서 레지스터 값을 변경하지 않아야 할 때, 시프트 레지스터를 Hold 모드로 설정하면 값이 유지된다. 이는 스톨(Stall)이나 버블(Bubble) 삽입 시 활용된다.

### 시프트 레지스터 주요 응용 분야 설명

**UART 통신**은 시프트 레지스터의 가장 대표적인 응용 분야이다. UART 송신기는 CPU에서 받은 병렬 데이터(8비트)를 PISO 시프트 레지스터를 통해 직렬 비트 스트림으로 변환하여 전송한다. 수신기는 SIPO 시프트 레지스터를 통해 수신된 직렬 비트 스트림을 병렬 데이터로 변환하여 CPU에 전달한다. Start 비트, Stop 비트, 패리티 비트 처리도 포함된다.

**ALU 시프트 연산**은 프로그래밍에서 자주 사용되는 연산이다. Logical Shift는 빈 자리를 0으로 채우며, 좌측 시프트는 x2, 우측 시프트는 /2 효과를 가진다. Arithmetic Shift는 부호 비트(MSB)를 유지하여 부호 있는 정수의 시프트를 지원한다. Rotate Shift는 빠져나간 비트를 반대쪽에서 다시 입력하여 순환 구조를 만든다.

**디지털 지연 회로**로 SISO 레지스터가 활용된다. n비트 SISO는 n 클럭 사이클의 정확한 지연을 생성하며, 이는 디지털 신호 처리, 파이프라인 타이밍 조정, 동기화 회로 등에 사용된다.

**LFSR(Linear Feedback Shift Register)**은 의사 난수 생성과 CRC(Cyclic Redundancy Check) 계산에 사용된다. 특정 비트들을 XOR하여 입력에 피드백함으로써, 주기적으로 반복되지만 통계적으로는 무작위처럼 보이는 비트열을 생성한다.

## 특징

### 장점
1. **변환 기능**: 직렬↔병렬 데이터 변환
2. **적은 배선**: 직렬 전송 시 적은 와이어 수
3. **유연성**: 다양한 모드 지원 (Hold, Shift, Load)

### 단점
1. **지연**: n비트 변환에 n 클럭 소요
2. **속도**: 병렬 전송보다 느림
3. **복잡도**: 양방향 및 다기능 시프트 레지스터는 설계 복잡

## 기술사 시험 포인트

1. **시프트 레지스터 정의**: 플립플롭 직렬 연결, 클럭마다 1비트 이동
2. **4가지 유형**: SISO, SIPO, PISO, PIPO
3. **제어 모드**: Hold, Right Shift, Left Shift, Load
4. **응용 분야**: UART, ALU 시프트, 직렬-병렬 변환
5. **양방향 구조**: MUX 기반 동작 모드 선택
