# 레지스터 (Register)

## 정의

플립플롭을 병렬로 연결하여 **n비트 데이터를 저장**하는 순차 논리회로. CPU 내부에서 가장 빠른 메모리로, 명령어 실행 중 임시 데이터 저장소 역할을 수행한다.

```
┌─────────────────────────────────────────────────────────────┐
│               4비트 병렬 레지스터 (Parallel Register)         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    D[3]     D[2]     D[1]     D[0]    ← 데이터 입력         │
│      │        │        │        │                          │
│      ▼        ▼        ▼        ▼                          │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐                       │
│  │ D FF │ │ D FF │ │ D FF │ │ D FF │   ← 4개의 D 플립플롭   │
│  │      │ │      │ │      │ │      │                       │
│  │  Q   │ │  Q   │ │  Q   │ │  Q   │                       │
│  └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘                       │
│     │        │        │        │                           │
│     ▼        ▼        ▼        ▼                           │
│    Q[3]    Q[2]    Q[1]    Q[0]     ← 데이터 출력          │
│                                                             │
│     ┌────────────────────────┐                             │
│     │         CLK            │  ← 공통 클럭 (동기식)        │
│     └────────────────────────┘                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4비트 병렬 레지스터 구조 상세 설명

병렬 레지스터는 n개의 D 플립플롭을 병렬로 연결하여 n비트 데이터를 동시에 저장하는 회로이다. 위 다이어그램은 4비트 레지스터의 구조를 보여주며, 각 비트가 독립적인 D 플립플롭에 저장됨을 알 수 있다.

**입력 단계**에서 4비트 데이터 D[3:0]이 각각 대응하는 D 플립플롭의 D 입력단에 인가된다. 이때 모든 플립플롭은 동일한 CLK 신호를 공유하므로, 클럭 에지(주로 상승 에지)가 도달하는 순간 모든 비트가 동시에 저장된다. 이를 **병렬 로드(Parallel Load)**라 한다.

**저장 단계**에서 각 D 플립플롭은 입력된 비트를 Q 출력으로 유지한다. 클럭이 없는 동안에는 출력이 변경되지 않으며, 이는 레지스터가 **메모리 기능**을 수행함을 의미한다. 플립플롭의 정적 특성 덕분에 전원이 공급되는 한 데이터가 무한히 유지된다.

**출력 단계**에서 저장된 4비트 데이터 Q[3:0]이 외부 회로로 출력된다. 이 데이터는 CPU의 산술 논리 연산장치(ALU)로 전달되어 연산에 사용되거나, 다른 레지스터로 전송되거나, 메모리에 쓰기 위해 데이터 버스로 출력될 수 있다.

```
┌─────────────────────────────────────────────────────────────┐
│           레지스터 with 제어 신호 (Control Signals)          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    ┌─────────┐                              │
│     D[3:0] ───────▶│         │                              │
│                    │  4-bit  │                              │
│     CLK ──────────▶│Register │───────▶ Q[3:0]               │
│                    │         │                              │
│     LOAD ─────────▶│ (Load/  │                              │
│           (EN)     │  Hold)  │                              │
│                    │         │                              │
│     CLR ──────────▶│         │                              │
│          (Reset)   └─────────┘                              │
│                                                             │
│  제어 신호 기능:                                            │
│  • LOAD=1: 클럭 에지에서 D 입력을 Q로 로드                   │
│  • LOAD=0: 현재 Q 값 유지 (Hold)                            │
│  • CLR=1: Q를 0000으로 비동기 리셋                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 제어 신호가 포함된 레지스터

실제 CPU 레지스터는 단순한 저장 기능을 넘어 다양한 제어 신호를 지원한다. 위 구조도는 LOAD와 CLR 제어 신호가 추가된 레지스터를 보여준다.

**LOAD 신호(또는 Enable)**는 데이터 로드를 제어한다. LOAD=1일 때만 클럭 에지에서 입력 데이터가 저장되며, LOAD=0이면 현재 저장된 값이 유지된다. 이를 **Hold 모드**라 하며, 레지스터가 불필요한 업데이트를 방지한다. 예를 들어, 명령어 파이프라인에서 이전 단계의 결과를 다음 단계까지 유지해야 할 때 Hold 모드가 사용된다.

**CLR 신호(또는 Reset)**는 레지스터를 0으로 초기화한다. 비동기 리셋인 경우 클럭과 무관하게 즉시 0으로 초기화되며, 동기 리셋인 경우 다음 클럭 에지에서 0으로 초기화된다. CPU 리셋 시 모든 레지스터를 초기화하거나, 프로그램 시작 전 변수를 0으로 설정할 때 사용된다.

### CPU 내 레지스터 종류 상세 분석

CPU는 다양한 목적의 레지스터를 포함하며, 각각 고유한 역할을 수행한다. 위 다이어그램은 크게 세 가지 카테고리로 분류한다.

**범용 레지스터(General Purpose Register, GPR)**는 프로그래머가 자유롭게 사용할 수 있는 레지스터이다. R0, R1, R2, R3...와 같이 번호로 식별되며, 산술 연산의 피연산자, 주소 계산, 임시 변수 저장 등 다양한 용도로 활용된다. x86-64 아키텍처에서는 RAX, RBX, RCX, RDX 등 16개의 64비트 범용 레지스터를 제공하며, ARM64에서는 X0-X30까지 31개의 64비트 레지스터를 제공한다.

**특수 목적 레지스터(Special Purpose Register)**는 CPU 동작 제어를 위한 전용 레지스터이다. **PC(Program Counter)**는 다음에 실행할 명령어의 주소를 저장하며, 분기 명령어 실행 시 자동으로 갱신된다. **SP(Stack Pointer)**는 스택의 최상단 주소를 가리키며, 함수 호출/복귀 및 지역 변수 관리에 사용된다. **FLAGS(또는 Status) 레지스터**는 연산 결과의 상태(Zero, Carry, Overflow, Sign 등)를 저장하여 조건부 분기 명령어의 기준이 된다. **MAR(Memory Address Register)**는 메모리 접근 시 주소를 저장하여 주소 버스로 출력한다.

**부동소수점 레지스터(Floating Point Register)**는 부동소수점 연산을 위한 전용 레지스터이다. 정수 레지스터와 별도로 존재하며, IEEE 754 표준의 단정도(32비트), 배정도(64비트), 확장 정밀도(80비트) 형식을 지원한다. 현대 CPU에서는 SIMD(Single Instruction Multiple Data) 연산을 위해 벡터 레지스터(XMM, YMM, ZMM)와 통합되기도 한다.

### 레지스터 파일 구조 분석

레지스터 파일은 여러 레지스터를 배열 형태로 조직화한 것이다. 위 다이어그램은 32개의 32비트 레지스터를 포함하는 전형적인 레지스터 파일 구조를 보여준다.

**다중 포트 구조**가 레지스터 파일의 핵심 특징이다. 위 예시는 2개의 읽기 포트(Read Addr1, Read Addr2)와 1개의 쓰기 포트(Write Addr)를 가진 **3-포트 레지스터 파일**이다. 이를 통해 한 클럭 사이클에 두 개의 레지스터 값을 읽어 ALU로 전달하고, 연산 결과를 하나의 레지스터에 저장할 수 있다. RISC 프로세서의 3-주소 명령어 형식(예: ADD R1, R2, R3)을 효율적으로 지원한다.

**디코더(Decoder)**는 주소 신호를 받아 해당 레지스터를 선택한다. 5비트 주소로 32개 레지스터 중 하나를 선택할 수 있다(2^5 = 32). 읽기 포트용 디코더와 쓰기 포트용 디코더가 독립적으로 동작하며, 동시에 서로 다른 레지스터에 접근할 수 있다.

**레지스터 배열**은 32개의 32비트 레지스터로 구성된다. 각 레지스터는 독립적인 D 플립플롭 배열이지만, 동일한 클럭과 제어 로직을 공유한다. 쓰기 동작은 클럭 에지에서만 발생하지만, 읽기 동작은 비동기적으로 수행되어 레지스터 값이 즉시 출력된다.

## 특징

### 장점
1. **최고 속도**: CPU 내부에서 가장 빠른 메모리 (1 사이클 접근)
2. **병렬 접근**: 다중 포트로 동시 읽기/쓰기 가능
3. **직접 접근**: 주소 계산 없이 레지스터 번호로 직접 접근

### 한계
1. **용량 제한**: 수십~수백 바이트로 제한 (집적도, 전력 소모)
2. **휘발성**: 전원 차단 시 데이터 손실
3. **비용**: SRAM 기반으로 비트당 비용 높음

### 기억 계층 구조 상세 설명

기억 계층 구조는 속도, 용량, 비용 간의 트레이드오프를 최적화하기 위해 설계되었다. 위 다이어그램은 레지스터에서 SSD까지의 계층을 보여주며, 각 계층의 접근 시간과 용량이 명시되어 있다.

**레지스터**는 계층 최상단에 위치하며 CPU 코어 내부에 직접 통합되어 있다. 접근 시간이 1ns(약 1 클럭 사이클)로 가장 빠르지만, 용량이 1KB 이하로 매우 작다. 모든 연산은 데이터가 레지스터에 로드된 후에만 수행될 수 있다.

**캐시 메모리(L1, L2, L3)**는 레지스터와 메인 메모리 사이의 속도 격차를 완화한다. L1 캐시는 CPU 코어 내부에, L2/L3 캐시는 코어 근처에 위치한다. 캐시는 자주 사용하는 데이터를 저장하여 평균 메모리 접근 시간을 줄인다. 캐시 적중(Cache Hit) 시 레지스터 수준에는 미치지 못하지만 DRAM보다 훨씬 빠른 접근이 가능하다.

**메인 메모리(DRAM)**는 실행 중인 프로그램과 데이터를 저장한다. 캐시보다 훨씬 큰 용량(16-128GB)을 제공하지만, 접근 시간이 100ns 정도로 캐시보다 10-50배 느리다. DRAM의 1트랜지스터-1커패시터 구조는 집적도는 높지만 속도가 제한된다.

**SSD(Solid State Drive)**는 비휘발성 저장 장치로, 전원이 꺼져도 데이터가 유지된다. 접근 시간이 100μs(마이크로초)로 DRAM보다 1000배 느리지만, 용량이 수 TB에 달하며 기계적 움직임이 없어 HDD보다 빠르다.

## 기술사 시험 포인트

1. **레지스터 정의**: 플립플롭 병렬 연결, n비트 데이터 저장
2. **제어 신호**: LOAD(로드/홀드), CLR(리셋)
3. **레지스터 종류**: 범용, 특수 목적(PC, SP, FLAGS), 부동소수점
4. **레지스터 파일**: 다중 포트, 동시 읽기/쓰기
5. **기억 계층**: 레지스터 > L1 > L2 > L3 > DRAM > SSD
