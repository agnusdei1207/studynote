# 병렬 가감산기 (Parallel Adder/Subtractor)

## 정의

**병렬 가감산기(Parallel Adder/Subtractor)**는 덧셈과 뺄셈을 **하나의 회로에서 수행**하는 N비트 연산 장치다. 모드 제어 신호에 따라 가산기 또는 감산기로 동작하며, ALU의 핵심 구성 요소다.

```
┌─────────────────────────────────────────────────────────────────┐
│           병렬 가감산기 개념                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   입력:                                                         │
│   • A[N-1:0]: 피연산자 1                                        │
│   • B[N-1:0]: 피연산자 2                                        │
│   • Mode: 연산 선택 (0=덧셈, 1=뺄셈)                            │
│                                                                 │
│   출력:                                                         │
│   • S[N-1:0]: 연산 결과                                         │
│   • Cout: 자리올림/빌림                                         │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Mode=0 → S = A + B                                    │ │
│   │   Mode=1 → S = A - B = A + B' + 1                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 2의 보수 기반 통합 원리

```
┌─────────────────────────────────────────────────────────────────┐
│           2의 보수 기반 가감산기 원리                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   핵심 아이디어:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   A - B = A + (-B)                                      │ │
│   │         = A + (2의 보수 of B)                           │ │
│   │         = A + (B' + 1)                                  │ │
│   │                                                         │ │
│   │   ∴ 뺄셈 = B 반전 + Cin=1 로 덧셈                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Mode 제어:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Mode = 0 (덧셈):                                      │ │
│   │     • B 그대로 전달 (XOR with 0 = B)                    │ │
│   │     • Cin = 0                                           │ │
│   │     • S = A + B                                         │ │
│   │                                                         │ │
│   │   Mode = 1 (뺄셈):                                      │ │
│   │     • B 반전 (XOR with 1 = B')                          │ │
│   │     • Cin = 1                                           │ │
│   │     • S = A + B' + 1 = A - B                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 상태 플래그 (Status Flags)

```
┌─────────────────────────────────────────────────────────────────┐
│           가감산기 상태 플래그                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ALU 상태 레지스터(Flags) 구성:                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐          │ │
│   │  │  S  │  Z  │  A  │  P  │  C  │  O  │ ... │          │ │
│   │  └─────┴─────┴─────┴─────┴─────┴─────┴─────┘          │ │
│   │                                                         │ │
│   │  S (Sign):     결과의 부호 (MSB)                        │ │
│   │  Z (Zero):     결과가 0이면 1                           │ │
│   │  A (Aux Carry): 하위 4비트 자리올림 (BCD 연산용)        │ │
│   │  P (Parity):   결과의 패리티                            │ │
│   │  C (Carry):    부호 없는 오버플로우                     │ │
│   │  O (Overflow): 부호 있는 오버플로우                     │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   오버플로우 검출 (부호 있는 연산):                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Overflow = Cₙ ⊕ Cₙ₋₁                                 │ │
│   │            = (양수+양수=음수) OR (음수+음수=양수)        │ │
│   │                                                         │ │
│   │   예: 4비트 부호 있는 연산                               │ │
│   │   0111 (7) + 0111 (7) = 1110 (-2) ← 오버플로우!        │ │
│   │   1000 (-8) + 1000 (-8) = 0000 (0) ← 오버플로우!       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **병렬 가감산기**: 덧셈/뺄셈을 하나의 회로로 수행
> **Mode=0**: 덧셈 (S = A + B, Cin = 0)
> **Mode=1**: 뺄셈 (S = A + B' + 1 = A - B)

### 2. 빈출 키워드

- **2의 보수**: 뺄셈을 덧셈으로 변환
- **XOR 게이트**: Mode에 따라 B 반전
- **오버플로우**: 부호 있는 연산의 결과 범위 초과
- **상태 플래그**: Z, S, C, O (Zero, Sign, Carry, Overflow)

### 3. 예상 문제

**Q1. 4비트 병렬 가감산기를 설계하고 동작 원리를 설명하시오.**

**Q2. 2의 보수를 이용한 뺄셈의 원리와 회로 구현 방법을 기술하시오.**

**Q3. 부호 있는 연산에서 오버플로우를 검출하는 방법을 설명하시오.**
