# 동기식 카운터 (Synchronous Counter)

## 정의

모든 플립플롭이 **동일한 클럭**을 공유하여 **동시에 천이**하는 카운터. 리플 지연이 없어 고속 동작이 가능하며, 정밀한 타이밍 제어에 적합하다.

```
┌─────────────────────────────────────────────────────────────┐
│          4비트 동기식 카운터 (Synchronous Counter)           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    ┌───────────────┐                       │
│    CLK ────────────┤   Common      │                       │
│                    │   Clock Line  │                       │
│                    └───────┬───────┘                       │
│            ┌───────────────┼───────────────┐               │
│            │               │               │               │
│            ▼               ▼               ▼               │
│       ┌─────────┐    ┌─────────┐    ┌─────────┐           │
│       │  J K FF │    │  J K FF │    │  J K FF │           │
│       │   Q0    │    │   Q1    │    │   Q2    │           │
│       └────┬────┘    └────┬────┘    └────┬────┘           │
│            │               │               │                │
│            ▼               ▼               ▼                │
│           Q0              Q1              Q2                │
│            │               │               │                │
│            │     ┌─────────┘               │                │
│            │     │     ┌───────────────────┘                │
│            │     │     │                                    │
│            ▼     ▼     ▼                                    │
│        ┌──────────────┐   ┌──────────────┐                 │
│        │    AND       │   │    AND       │                 │
│        │  (2-input)   │   │  (3-input)   │                 │
│        └──────┬───────┘   └──────┬───────┘                 │
│               │                  │                          │
│               ▼                  ▼                          │
│          FF1 토글           FF2 토글                        │
│          조건 (J=K)         조건 (J=K)                      │
│                                                             │
│  토글 조건:                                                  │
│  - FF0: 항상 토글 (J=K=1)                                   │
│  - FF1: Q0=1일 때 토글                                      │
│  - FF2: Q0=Q1=1일 때 토글                                   │
│  - FF3: Q0=Q1=Q2=1일 때 토글                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4비트 동기식 카운터 구조 상세 설명

동기식 카운터는 모든 플립플롭이 공통 클럭 라인에 연결되어 동시에 천이한다. 위 다이어그램은 JK 플립플롭을 사용한 4비트 동기식 업 카운터의 구조를 보여준다.

**공통 클럭(Common Clock)**이 동기식 카운터의 핵심이다. 모든 플립플롭이 동일한 CLK 신호를 받아 정확히 같은 순간에 천이한다. 비동기 카운터에서 발생하던 리플 지연이 완전히 제거되어, 출력 Q0, Q1, Q2, Q3가 동시에 안정화된다. 이는 고속 디지털 시스템에서 필수적이다.

**JK 플립플롭의 토글 모드**를 활용한다. JK 플립플롭은 J=K=1일 때 클럭 에지마다 출력을 토글(Q → Q')시킨다. 각 플립플롭의 J와 K 입력에 적절한 값을 인가하면, 원하는 타이밍에만 토글되도록 제어할 수 있다.

**결합 논리(Combinational Logic)**가 각 단계의 토글 조건을 생성한다. FF0은 항상 토글(J=K=1), FF1은 Q0=1일 때만 토글, FF2는 Q0=Q1=1일 때만 토글, FF3은 Q0=Q1=Q2=1일 때만 토글한다. 이 논리를 수식으로 표현하면:
- J0 = K0 = 1 (항상)
- J1 = K1 = Q0
- J2 = K2 = Q0 · Q1
- J3 = K3 = Q0 · Q1 · Q2

**동작 원리**를 2진수 관점에서 분석하면, 2진수 증가에서 "각 비트는 모든 하위 비트가 1일 때만 토글된다"는 규칙이 있다. 예를 들어, 0111(7)에서 1000(8)으로 증가할 때, Q0, Q1, Q2가 모두 1이므로 Q3가 토글된다. 동기식 카운터의 결합 논리는 이 규칙을 하드웨어로 구현한 것이다.

```
┌─────────────────────────────────────────────────────────────┐
│         동기식 카운터 동작 순서표 (4비트)                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  클럭 │ Q3 Q2 Q1 Q0 │ 10진수 │ 토글하는 FF                  │
│  ────┼─────────────┼────────┼────────────────              │
│   0  │  0  0  0  0 │   0    │ FF0                          │
│   1  │  0  0  0  1 │   1    │ FF0, FF1 (Q0=1)              │
│   2  │  0  0  1  0 │   2    │ FF0                          │
│   3  │  0  0  1  1 │   3    │ FF0, FF1, FF2 (Q0=Q1=1)      │
│   4  │  0  1  0  0 │   4    │ FF0                          │
│   5  │  0  1  0  1 │   5    │ FF0, FF1                     │
│   6  │  0  1  1  0 │   6    │ FF0                          │
│   7  │  0  1  1  1 │   7    │ FF0, FF1, FF2, FF3 (모두)    │
│   8  │  1  0  0  0 │   8    │ FF0                          │
│   9  │  1  0  0  1 │   9    │ FF0, FF1                     │
│  10  │  1  0  1  0 │  10    │ FF0                          │
│  11  │  1  0  1  1 │  11    │ FF0, FF1, FF2                │
│  12  │  1  1  0  0 │  12    │ FF0                          │
│  13  │  1  1  0  1 │  13    │ FF0, FF1                     │
│  14  │  1  1  1  0 │  14    │ FF0                          │
│  15  │  1  1  1  1 │  15    │ FF0, FF1, FF2, FF3 (모두)    │
│  16  │  0  0  0  0 │   0    │ FF0 (리셋/오버플로우)         │
│                                                             │
│  ※ 모든 FF가 동일 클럭에 동시에 천이                        │
│  ※ 리플 지연 없음                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 동기식 카운터 동작 분석

동작 순서표는 4비트 동기식 카운터가 0부터 15까지 세는 과정을 보여준다. 각 클럭에서 어떤 플립플롭이 토글되는지 명시되어 있다.

**패턴 분석**을 하면 다음 규칙을 발견할 수 있다. FF0은 매 클럭마다 토글된다(주파수 = CLK/2). FF1은 Q0=1일 때 토글된다(주파수 = CLK/4). FF2는 Q0=Q1=1일 때 토글된다(주파수 = CLK/8). FF3은 Q0=Q1=Q2=1일 때 토글된다(주파수 = CLK/16).

**토글 조건 생성**은 AND 게이트가 담당한다. FF1의 J=K 입력은 Q0에 직접 연결(1-input AND). FF2의 J=K 입력은 Q0 AND Q1. FF3의 J=K 입력은 Q0 AND Q1 AND Q2. 이렇게 n번째 FF의 토글 조건은 모든 하위 비트가 1인지 확인한다.

**동시 천이의 이점**을 7→8 전이에서 명확히 볼 수 있다. 비동기 카운터에서는 Q0→Q1→Q2→Q3 순서로 천이가 전파되어 일시적으로 0110, 0100, 0000 등의 잘못된 상태가 나타날 수 있다. 동기식 카운터에서는 모든 FF가 동시에 천이하여 0111→1000로 한 번에 전이된다.

```
┌─────────────────────────────────────────────────────────────┐
│         동기식 vs 비동기 타이밍 비교                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │

│                                                             │

│         ─┘ └─┘ └─                                           │
```

### 동기식 vs 비동기 타이밍 상세 비교

타이밍 다이어그램은 두 카운터의 근본적 차이를 명확히 보여준다. 동기식 카운터의 성능 우위를 정량적으로 분석할 수 있다.

**비동기 카운터의 리플 지연**은 누적된다. 3비트 카운터에서 Q2가 안정화되려면 Q0의 전파 지연 + Q1의 전파 지연 + Q2의 전파 지연 = 3 × t_pd가 소요된다. n비트 카운터에서는 최대 n × t_pd의 지연이 발생한다. 따라서 최대 클럭 주파수는 1 / (n × t_pd)로 제한된다.

**동기식 카운터의 동시 천이**는 모든 FF가 같은 시점에 천이한다. 각 FF는 클럭 에지 후 t_pd 시간 내에 출력이 안정화된다. n비트 카운터에서도 지연은 여전히 t_pd이며, 최대 클럭 주파수는 1 / t_pd로 n에 무관하게 일정하다.

**실제 성능 차이**를 예로 들어보자. t_pd = 10ns인 FF로 16비트 카운터를 구현할 때, 비동기 카운터의 최대 클럭 주파수는 1 / (16 × 10ns) = 6.25MHz이다. 반면 동기식 카운터는 1 / 10ns = 100MHz로 동작할 수 있다. 16배의 성능 차이가 발생한다.

```

│         │  Q0     │    │  Q1     │    │  Q2     │          │
│         └────┬────┘    └────┬────┘    └────┬────┘          │
│              │              │              │                │
│              ▼              ▼              ▼                │
```

### 동기식 업/다운 카운터 구조

업/다운 카운터는 제어 신호에 따라 증가 또는 감소 카운팅을 선택할 수 있다. 동기식으로 구현하면 모드 전환 시에도 스파이크 없이 안정적으로 동작한다.

**업/다운 제어 논리**는 각 FF의 J, K 입력에 MUX를 사용하여 업 모드와 다운 모드를 선택한다. 업 모드(Up=1)에서는 Q0, Q1, Q2를 사용하여 토글 조건을 생성한다. 다운 모드(Down=0)에서는 Q0', Q1', Q2'(반전 출력)를 사용한다.

**감소 카운팅 원리**를 분석하면, 2진수 감소에서 "각 비트는 모든 하위 비트가 0일 때만 토글된다"는 규칙이 있다. 예를 들어, 1000(8)에서 0111(7)로 감소할 때, Q0, Q1, Q2가 모두 0이므로 Q3가 토글된다. 반전 출력 Q'을 사용하면 이 논리를 쉽게 구현할 수 있다.

```
┌─────────────────────────────────────────────────────────────┐
│         동기식 BCD 카운터 (Decade Counter)                   │

│                                                             │
│         ┌─────────────────────────────────────┐            │
│         │         4-bit Counter               │            │
│  CLK ──▶│  Q0   Q1   Q2   Q3                  │            │
│         │   │    │    │    │                  │            │
│         └───┼────┼────┼────┼──────────────────┘            │
│             │    │    │    │                                │

│             │    │    └─▶│ Q1·Q3   │───┐                   │
│             │    │      └─────────┘   │                    │
│             │    │                    ▼                    │

```

### 동기식 BCD 카운터(10진 카운터) 구조

BCD 카운터는 0부터 9까지 세고 다시 0으로 돌아가는 모듈로-10 카운터이다. 디지털 시계, 계수기, 7세그먼트 디스플레이 구동에 사용된다.

**리셋 로직**이 핵심이다. 4비트 2진 카운터는 0~15까지 세지만, BCD 카운터는 9(1001) 다음에 0(0000)으로 리셋되어야 한다. 10(1010)이 되는 순간을 감지하려면 Q1=1 AND Q3=1 조건을 사용한다. 이 조건이 참이면 CLR 신호가 활성화되어 카운터가 0으로 리셋된다.

**동기식 리셋 vs 비동기식 리셋**의 선택이 중요하다. 비동기식 리셋은 10이 감지되는 즉시 리셋하지만, 일시적으로 10이 출력될 수 있다. 동기식 리셋은 다음 클럭 에지에서 리셋되며, 더 정확한 타이밍을 제공한다.

**다단계 BCD 카운터**를 구성하려면 각 자리의 Carry 출력이 다음 자리의 Enable 입력으로 연결된다. 1의 자리가 9→0으로 전이할 때 10의 자리가 1 증가한다. 이를 통해 00~99, 000~999 등 확장 가능하다.

```
│  │  │Q0·Q1│          │C0·Q2│          │C1·Q3│      │       │
│  │  └─────┘          └─────┘          └─────┘      │       │
│  │                                                   │       │
│  │  C0 = Q0·Q1                                       │       │
│  │  C1 = C0·Q2 = Q0·Q1·Q2                           │       │
│  │  C2 = C1·Q3 = Q0·Q1·Q2·Q3                        │       │
│  │                                                   │       │
│  │  각 단계에서 2-input AND만 사용                   │       │
│  │  전파 지연 = n × t_AND (여전히 리플 존재)          │       │
│  └─────────────────────────────────────────────────┘       │
│                                                             │
│  대안: 병렬 AND 트리 구조                                    │
│                                                             │
│         Q0  Q1  Q2  Q3                                      │
│          │   │   │   │                                      │
│          ▼   ▼   ▼   ▼                                      │
│         ┌───┐   ┌───┐                                       │
│         │AND│   │AND│  ← Level 1                            │
│         └─┬─┘   └─┬─┘                                       │
│           │       │                                         │
│           ▼       ▼                                         │
│           ┌─────────┐                                       │
│           │   AND   │  ← Level 2                            │
│           └────┬────┘                                       │
│                │                                            │
│                ▼                                            │
│           Q0·Q1·Q2·Q3                                       │
│                                                             │
│  전파 지연 = log₂(n) × t_AND (로그 스케일)                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 결합 논리 최적화 기법

동기식 카운터의 결합 논리는 비트 수가 증가할수록 복잡해진다. 8비트, 16비트 카운터에서는 팬인 문제와 전파 지연을 최적화해야 한다.

**팬인(Fan-In) 문제**는 n번째 FF의 토글 조건이 n-input AND 게이트를 요구한다는 것이다. 실제 게이트는 입력 수에 제한이 있어(보통 4~8입력), 16비트 카운터에서는 16-input AND 게이트를 직접 구현할 수 없다.

**직렬 AND 체인**은 각 단계에서 2-input AND를 사용하여 순차적으로 캐리를 생성한다. C0 = Q0·Q1, C1 = C0·Q2, C2 = C1·Q3... 이 방식은 팬인 문제를 해결하지만, 논리 레벨에서 리플 지연이 발생한다.

**병렬 AND 트리**는 로그 스케일의 전파 지연을 달성한다. n개의 입력을 log₂(n) 레벨의 AND 게이트로 처리한다. 16비트 카운터에서 직렬 체인은 15 레벨, 병렬 트리는 4 레벨의 지연만 발생한다. 고속 카운터 설계의 핵심 기법이다.

## 특징

### 장점
1. **고속 동작**: 리플 지연 없음, 최대 클럭 주파수 = 1/t_pd
2. **스파이크 없음**: 모든 FF 동시 천이로 일시적 오류 방지
3. **정확한 타이밍**: 결합 논리로 정밀한 제어 가능

### 단점
1. **복잡한 회로**: 결합 논리(AND 게이트) 추가 필요
2. **배선 증가**: 공통 클럭 및 다중 AND 연결
3. **팬인 문제**: 비트 수 증가 시 AND 게이트 입력 수 증가

## 기술사 시험 포인트

1. **동기식 카운터 정의**: 공통 클럭, 동시 천이, 리플 지연 없음
2. **토글 조건**: n번째 FF는 모든 하위 비트가 1일 때 토글
3. **결합 논리**: AND 게이트로 토글 조건 생성 (J=K 입력)
4. **장점**: 고속, 스파이크 없음, 정확한 타이밍
5. **최적화**: Look-Ahead Carry, 병렬 AND 트리
6. **응용**: BCD 카운터, 업/다운 카운터, 고속 주파수 분주
