# 카운터 (Counter)

## 정의

클럭 펄스에 따라 **미리 정해진 상태 순서대로 진행**하는 순차 논리회로. 이벤트 발생 횟수 계수, 주파수 분주, 타이밍 생성 등에 활용된다.

### 3비트 비동기 카운터(리플 카운터) 구조 상세 설명

비동기 카운터는 가장 단순한 카운터 구조로, 각 플립플롭의 출력이 다음 단계 플립플롭의 클럭 입력으로 연결된다. 위 다이어그램은 0부터 7까지 세는 3비트 업 카운터를 보여준다.

**T 플립플롭(Toggle Flip-Flop)**이 비동기 카운터의 핵심 요소이다. T 플립플롭은 T=1일 때 모든 클럭 에지에서 출력을 토글(Q → Q')시킨다. 첫 번째 FF(Q0)는 외부 클럭에 의해 구동되고, Q0의 출력이 Q1의 클럭 입력이 되며, Q1의 출력이 Q2의 클럭 입력이 된다. 이를 **Ripple(잔물결) 전파**라 한다.

**동작 원리**를 이진수 관점에서 분석하면, Q0는 매 클럭마다 토글되어 0, 1, 0, 1... 패턴을 반복한다(주파수가 입력 클럭의 1/2). Q1은 Q0가 1→0으로 떨어질 때마다 토글되어 0, 0, 1, 1, 0, 0, 1, 1... 패턴을 반복한다(주파수가 입력 클럭의 1/4). Q2는 Q1이 1→0으로 떨어질 때마다 토글된다(주파수가 입력 클럭의 1/8).

**카운트 순서표**를 보면, Q2Q1Q0는 000→001→010→011→100→101→110→111→000... 순서로 진행한다. 이는 0부터 7까지 세는 3비트 2진 카운터이다. 8번째 클럭에서 111→000으로 리셋되며, 이를 **모듈로-8(Modulo-8)** 카운터라 한다. n비트 카운터는 0부터 2^n - 1까지 세는 모듈로-2^n 카운터가 된다.

### 비동기 카운터 타이밍 파형 분석

타이밍 다이어그램은 비동기 카운터의 동작을 시각적으로 보여준다. 각 신호의 천이 시점에서 발생하는 **리플 지연(Ripple Delay)**이 중요한 특징이다.

**Q0 파형**은 입력 클럭의 매 하강 에지(또는 상승 에지, 설계에 따라 다름)에서 토글된다. Q0의 주파수는 입력 클럭의 정확히 1/2이다. 이를 **주파수 분주(Frequency Division)**라 하며, n번째 단계의 주파수는 입력 클럭 주파수를 2^n으로 나눈 값이다.

**Q1 파형**은 Q0의 하강 에지에서 토글된다. Q0가 1→0으로 떨어질 때만 Q1이 변화한다. Q1의 주파수는 입력 클럭의 1/4이다. Q0의 천이 후 약간의 지연(플립플롭의 전파 지연 시간) 후에 Q1이 변화한다.

**Q2 파형**은 Q1의 하강 에지에서 토글된다. Q1→Q2 전파 과정에서 또 다른 지연이 추가된다. 결과적으로, 입력 클럭의 천이로부터 Q2의 천이까지 누적 지연이 발생한다. 이를 **리플 지연**이라 하며, n비트 비동기 카운터에서 최대 n × t_pd(전파 지연)가 될 수 있다.

**카운트 값**은 Q2Q1Q0를 2진수로 읽은 것이다. 각 클럭마다 1씩 증가하며, 7(111) 다음에는 0(000)으로 돌아간다. 이 카운트 값은 이벤트 발생 횟수를 세거나, 메모리 주소를 순차적으로 생성하는 데 사용될 수 있다.


### 비동기 vs 동기 카운터 구조 비교

비동기와 동기 카운터는 각각 장단점이 있어 용도에 따라 선택된다. 위 다이어그램은 두 구조의 근본적 차이를 보여준다.

**비동기 카운터**는 각 플립플롭이 이전 단계의 출력을 클럭으로 사용한다. 회로가 매우 단순하여 적은 게이트 수와 배선으로 구현할 수 있다. 하지만 리플 지연으로 인해 모든 비트가 안정화될 때까지 시간이 걸린다. 예를 들어, 8비트 비동기 카운터에서 01111111→10000000 전이 시, Q7이 안정화되려면 7 × t_pd의 지연이 발생한다. 또한, 일시적인 잘못된 상태(스파이크)가 발생할 수 있어 디코더와 결합할 때 문제가 될 수 있다.

**동기 카운터**는 모든 플립플롭이 동일한 클럭을 공유한다. 결합 논리(AND 게이트 등)가 각 단계의 토글 조건을 생성하여, 모든 비트가 동시에 천이한다. 리플 지연이 없어 고속 동작이 가능하며, 스파이크도 발생하지 않는다. 하지만 결합 논리가 복잡해지고, 각 단계마다 AND 게이트의 입력 수가 증가(팬인 문제)하여 설계가 어려워진다.

```
┌─────────────────────────────────────────────────────────────┐
│         카운터 종류 (Types of Counters)                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 업 카운터 (Up Counter)                                   │
│     0 → 1 → 2 → 3 → ... → (2^n - 1) → 0 → ...              │
│     가장 일반적인 형태                                       │
│                                                             │
│  2. 다운 카운터 (Down Counter)                               │
│     (2^n - 1) → ... → 3 → 2 → 1 → 0 → (2^n - 1) → ...      │
│     타이머, 카운트다운에 사용                                │
│                                                             │
│  3. 업/다운 카운터 (Up/Down Counter)                         │
│     제어 신호에 따라 업 또는 다운 카운팅                     │
│     ┌──────────┐                                            │
│     │ Up/Down  │                                            │
│     │ Control  │────▶ [Counter]                             │
│     └──────────┘     Up=1: 증가, Down=0: 감소               │
│                                                             │
│  4. BCD 카운터 (Decade Counter)                              │
│     0 → 1 → 2 → ... → 9 → 0 → ...                          │
│     10진수 표시, 디지털 시계                                 │
│                                                             │
│  5. 링 카운터 (Ring Counter)                                 │
│     1000 → 0100 → 0010 → 0001 → 1000                        │
│     하나의 1이 순환                                         │
│                                                             │
│  6. 존슨 카운터 (Johnson Counter)                            │
│     0000 → 1000 → 1100 → 1110 → 1111 → 0111 → ...          │
│     Q를 NOT하여 입력에 연결                                  │
│                                                             │
│  7. 모듈로-N 카운터 (Modulo-N Counter)                       │
│     0 → 1 → ... → (N-1) → 0 → ...                          │
│     N이 2의 거듭제곱이 아닌 경우                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 다양한 카운터 유형 분석

**업 카운터**는 0부터 순차적으로 증가하여 최대값에 도달하면 다시 0으로 돌아간다. n비트 업 카운터는 0부터 2^n - 1까지 세며, 모듈로-2^n 카운터이다. 이벤트 발생 횟수를 세거나 메모리 주소를 순차적으로 생성하는 데 사용된다.

**다운 카운터**는 최대값부터 0까지 감소한 후 다시 최대값으로 돌아간다. 타이머, 카운트다운, 펄스 폭 측정에 사용된다. 다운 카운터의 borrow 출력은 0→최대값 전이 시 생성되어 다음 단계 카운터에 연결할 수 있다.

**업/다운 카운터**는 제어 신호에 따라 증가 또는 감소를 선택할 수 있다. Up/Down = 1이면 증가, Up/Down = 0이면 감소한다. 양방향 위치 제어, 가감산 카운팅에 활용된다.

**BCD 카운터(Decade Counter)**는 0부터 9까지 세고 10진수로 되돌아간다. 4비트 2진 카운터(0~15)에 리셋 로직을 추가하여 10(1010)이 되면 즉시 0으로 리셋한다. 디지털 시계, 계수기, 7세그먼트 디스플레이 구동에 사용된다.

**모듈로-N 카운터**는 N이 2의 거듭제곱이 아닌 경우이다. 예를 들어, 모듈로-6 카운터는 0→1→2→3→4→5→0으로 순환한다. 6(110)이 되면 리셋 로직이 작동한다. 시계의 초/분(모듈로-60), 시(모듈로-24) 구현에 사용된다.

### 카운터 주요 응용 분야 상세 설명

**주파수 분주기**로서 카운터는 높은 주파수의 클럭을 낮은 주파수로 변환한다. 예를 들어, 32.768kHz 크리스탈 오실레이터와 15비트 카운터를 결합하면 정확히 1Hz(1초 간격) 신호를 생성할 수 있다(32768 = 2^15). CPU의 클럭 생성, UART 보레이트 생성, PWM 주파수 설정 등에 활용된다.

**이벤트 카운터**는 외부 펄스의 발생 횟수를 계수한다. 생산 라인에서 제품 수를 세거나, 방사선 계수기에서 방사선 입자 수를 세거나, 디지털 미터기에서 펄스를 세는 데 사용된다. 업 카운터와 다운 카운터를 결합하여 가감산이 가능한 계수기를 만들 수도 있다.

**타이머/타임키퍼**는 클럭 펄스를 시간 단위로 변환한다. 1Hz 클럭을 60으로 나누면 분, 다시 60으로 나누면 시를 얻을 수 있다. 마이크로컨트롤러의 타이머/카운터 모듈은 인터럽트를 생성하여 정기적인 작업을 수행하게 한다.

**메모리 주소 생성**은 메모리를 순차적으로 접근할 때 카운터가 주소를 생성한다. DMA(Direct Memory Access) 컨트롤러는 카운터를 사용하여 소스와 목적지 주소를 자동으로 증가시킨다. 메모리 테스트에서도 모든 주소를 순차적으로 접근하는 데 카운터가 사용된다.

## 특징

### 장점
1. **단순성**: 비동기 카운터는 적은 하드웨어로 구현
2. **주파수 분주**: 쉽게 클럭 주파수를 낮출 수 있음
3. **다양성**: 다양한 모듈로와 모드 지원

### 단점
1. **리플 지연**: 비동기 카운터의 누적 지연
2. **스파이크**: 일시적인 잘못된 상태 가능
3. **속도 제한**: 고속 애플리케이션에서는 동기 카운터 필요

## 기술사 시험 포인트

1. **카운터 정의**: 클럭에 따라 미리 정해진 상태 순서로 진행
2. **비동기 vs 동기**: 리플 지연 vs 동시 천이
3. **카운터 종류**: 업, 다운, 업/다운, BCD, 링, 존슨, 모듈로-N
4. **응용 분야**: 주파수 분주, 이벤트 카운팅, 타이머, 주소 생성
5. **n비트 카운터**: 0 ~ 2^n - 1 범위, 모듈로-2^n
