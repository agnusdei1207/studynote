# 인코더 (Encoder)

## 정의

**인코더(Encoder)**는 **여러 입력선 중 하나**가 활성화되었을 때, 해당 입력의 위치를 나타내는 **이진 코드를 출력**하는 조합 논리회로다. 디코더의 역동작이며, 2ⁿ개 입력 → N개 출력의 구조를 갖는다.

```
┌─────────────────────────────────────────────────────────────────┐
│           인코더 기본 구조                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   2ⁿ입력 → N출력 인코더 (2ⁿ-to-N Encoder)                       │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   입력: 2ⁿ비트 (한 번에 하나만 활성)                     │ │
│   │   출력: N비트 이진 코드                                  │ │
│   │                                                         │ │
│   │   D₃ ───┐                                              │ │
│   │   D₂ ───┤                                              │ │
│   │   D₁ ───┼──▶ [인코더] ──▶ Y₁ Y₀                        │ │
│   │   D₀ ───┘                     │   │                      │ │
│   │                               2비트 코드                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   💡 활성화된 입력선의 번호를 이진수로 출력                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 4×2 인코더

```
┌─────────────────────────────────────────────────────────────────┐
│           4×2 인코더 진리표 및 회로                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   진리표:                                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ D₃ │ D₂ │ D₁ │ D₀ │ Y₁ │ Y₀ │ 설명                   │ │
│   ├───┼───┼───┼───┼───┼───┼────────────────┤ │
│   │ 0 │ 0 │ 0 │ 1 │ 0 │ 0 │ 입력 0 활성 (코드 00)   │ │
│   │ 0 │ 0 │ 1 │ 0 │ 0 │ 1 │ 입력 1 활성 (코드 01)   │ │
│   │ 0 │ 1 │ 0 │ 0 │ 1 │ 0 │ 입력 2 활성 (코드 10)   │ │
│   │ 1 │ 0 │ 0 │ 0 │ 1 │ 1 │ 입력 3 활성 (코드 11)   │ │
│   │ 0 │ 0 │ 0 │ 0 │ X │ X │ 입력 없음 (무정의)      │ │
│   │ 기타 │  │   │   │ X │ X │ 다중 입력 (비정상)    │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   논리식:                                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   Y₁ = D₃ + D₂                                         │ │
│   │   Y₀ = D₃ + D₁                                         │ │
│   │                                                         │ │
│   │   💡 각 출력은 특정 입력선들의 OR                        │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   회로도:                                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   D₃ ─────┬──────────────────────┐                      │ │
│   │           │                      │                      │ │
│   │   D₂ ─────┼──────┤ OR ├── Y₁    │                      │ │
│   │           │                      │                      │ │
│   │   D₁ ─────┼──────────────────────┼──────┤ OR ├── Y₀   │ │
│   │           │                      │      │              │ │
│   │   D₀ ─────┘ (미사용)             │      │              │ │
│   │                                  │      │              │ │
│   │                                  D₃ ───┘              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 우선순위 인코더 (Priority Encoder)

```
┌─────────────────────────────────────────────────────────────────┐
│           우선순위 인코더                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   문제점: 일반 인코더는 여러 입력이 동시 활성 시 오류            │
│   해결: 우선순위를 부여하여 가장 높은 우선순위 입력만 처리       │
│                                                                 │
│   4×2 우선순위 인코더 진리표:                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ D₃ │ D₂ │ D₁ │ D₀ │ Y₁ │ Y₀ │ V (Valid) │           │ │
│   ├───┼───┼───┼───┼───┼───┼────────┤           │
│   │ 0 │ 0 │ 0 │ 0 │ X │ X │   0    │ 입력 없음 │ │
│   │ 0 │ 0 │ 0 │ 1 │ 0 │ 0 │   1    │ D₀ 활성  │ │
│   │ 0 │ 0 │ 1 │ X │ 0 │ 1 │   1    │ D₁ 활성  │ │
│   │ 0 │ 1 │ X │ X │ 1 │ 0 │   1    │ D₂ 활성  │ │
│   │ 1 │ X │ X │ X │ 1 │ 1 │   1    │ D₃ 활성  │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   우선순위: D₃ > D₂ > D₁ > D₀ (D₃이 최고 우선순위)              │
│                                                                 │
│   논리식:                                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   Y₁ = D₃ + D₂                                         │ │
│   │   Y₀ = D₃ + D₂'D₁                                      │ │
│   │   V  = D₃ + D₂ + D₁ + D₀                               │ │
│   │                                                         │ │
│   │   💡 V(Valid): 유효 입력 존재 여부                       │ │
│   │   💡 X(Don't care): 우선순위에 의해 무시                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 8×3 우선순위 인코더

```
┌─────────────────────────────────────────────────────────────────┐
│           8×3 우선순위 인코더 (74LS148)                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   입력: D₇ D₆ D₅ D₄ D₃ D₂ D₁ D₀                       │ │
│   │   출력: Y₂ Y₁ Y₀ (3비트 코드)                           │ │
│   │         V (Valid), GS (Group Signal)                    │ │
│   │         EO (Enable Output, 캐스케이딩용)                │ │
│   │                                                         │ │
│   │   우선순위: D₇ > D₆ > ... > D₀                         │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   진리표 요약:                                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ 입력 활성  │ Y₂ Y₁ Y₀ │ 설명                           │ │
│   ├───────────┼─────────┼────────────────────────┤ │
│   │ D₇        │  0  0  0 │ 입력 7 (코드 000, Active-Low)  │ │
│   │ D₆        │  0  0  1 │ 입력 6                         │ │
│   │ D₅        │  0  1  0 │ 입력 5                         │ │
│   │ D₄        │  0  1  1 │ 입력 4                         │ │
│   │ D₃        │  1  0  0 │ 입력 3                         │ │
│   │ D₂        │  1  0  1 │ 입력 2                         │ │
│   │ D₁        │  1  1  0 │ 입력 1                         │ │
│   │ D₀        │  1  1  1 │ 입력 0                         │ │
│   │ 없음      │  X  X  X │ V=0 (무효)                     │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   논리식:                                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   Y₂ = D₇' + D₆' + D₅' + D₄'                          │ │
│   │   Y₁ = D₇' + D₆' + D₅D₄D₃' + D₅D₄D₂'                │ │
│   │   Y₀ = D₇' + D₆D₅' + D₆D₅D₄D₃' + D₆D₅D₄D₂D₁'      │ │
│   │                                                         │ │
│   │   💡 Active-Low 출력 (74LS148)                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 인코더 응용

```
┌─────────────────────────────────────────────────────────────────┐
│           인코더 응용 분야                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ① 키보드 인코딩:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   키보드 매트릭스 (8×8 = 64키)                          │ │
│   │   행/열 스캔 → 우선순위 인코더 → 스캔 코드               │ │
│   │                                                         │ │
│   │   예: 64키 → 6비트 행 + 6비트 열 = 위치 코드             │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ② 인터럽트 컨트롤러:                                          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   여러 인터럽트 요청                                     │ │
│   │       ↓                                                 │ │
│   │   우선순위 인코더                                        │ │
│   │       ↓                                                 │ │
│   │   인터럽트 벡터 (가장 높은 우선순위)                      │ │
│   │                                                         │ │
│   │   💡 여러 인터럽트 동시 발생 시 우선 처리                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ③ 위치 감지:                                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   센서 배열 (포토다이오드, 홀센서 등)                    │ │
│   │       ↓                                                 │ │
│   │   인코더                                                 │ │
│   │       ↓                                                 │ │
│   │   디지털 위치 코드                                       │ │
│   │                                                         │ │
│   │   예: 로터리 인코더 (각도 → 디지털 코드)                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ④ 멀티플렉서와의 관계:                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   멀티플렉서: 선택선 → 데이터선 선택                     │ │
│   │   인코더:   데이터선 → 선택선 코드 생성                  │ │
│   │                                                         │ │
│   │   💡 인코더는 멀티플렉서의 선택 신호를 생성               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **인코더**: 2ⁿ입력 → N출력, 활성 입력의 위치를 이진 코드로 변환
> **우선순위 인코더**: 여러 입력 동시 활성 시 최우선 입력 처리
> **Valid(V)**: 유효 입력 존재 여부 표시

### 2. 빈출 키워드

- **2ⁿ-to-N**: 입력/출력 비율
- **우선순위**: 동시 입력 시 처리 순서
- **Valid/GS**: 유효 입력 플래그
- **인터럽트 벡터**: 우선순위 인코더 응용

### 3. 예상 문제

**Q1. 4×2 우선순위 인코더의 진리표를 작성하고 논리식을 유도하시오.**

**Q2. 일반 인코더와 우선순위 인코더의 차이점을 설명하시오.**

**Q3. 인코더를 이용한 키보드 스캔 회로를 설계하시오.**
