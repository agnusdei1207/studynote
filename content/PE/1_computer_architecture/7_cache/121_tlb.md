# TLB (Translation Lookaside Buffer)

## 핵싱 인사이트 (3줄 요약)
> 1. **본질**: TLB는 **"가상 주소**를 **물리 주소**로 **변환**하는 **캐시"**로, **Page Table Walk**(페이지 테이블 탐색)의 **오버헤드**를 **줄이고 **CAM**(Content Addressable Memory)로 **Parallel Search**를 **지원**하며 **VPN**(Virtual Page Number)와 **PPN**(Physical Page Number) **쌍**으로 **구성**된다.
>  **2. **ASID**(Address Space ID)로 **프로세스**를 **구분**하고 **TLB Miss** 시 **Page Table Walk**로 **PPN**을 **가져오며 **Page Size**와 **Associativity**(Set-Associative)가 **Hit Rate**에 **영향**을 **미친**다.
> 3. **융합**: **L1 Instruction TLB**(ITLB), **L1 Data TLB**(DTLB), **L2 Unified TLB**로 **계층**되며 **TLB Shootdown**(다중 프로세스가 같은 VPN 경쟁)을 **해결**하기 위해 **ASID**와 **VPN Tagging**을 **사용**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
TLB는 **"주소 변환 캐시"**이다.

**TLB 특징**:
- **캐시**: 주소 변환 결과 저장
- **속도**: CPU에 가까움 (L1)
- **용량**: 작음 (수십~수백 엔트리)
- **Miss Penalty**: 10~100 cycles

### 💡 비유
TLB는 ****주소록****과 같다.
- **전화번호**: VPN
- **실제 번호**: PPN
- **메모**: 캐시
- **찾기**: Parallel Search

## Ⅱ. 아키텍처 및 핵심 원리

### TLB 구조

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB Structure                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  VPN (Virtual Page Number)    │  │  Valid  │  PPN (Physical Page Number)  │  ASID  │                          │  │
    │  ─────────────────────────┴─────────┴─────────────────────────────┴───────┘                          │  │
    │  0x001                        │  1        │  0x03                       │  1      │ Hit!                        │  │
    │  0x002                        │  1        │  0x05                       │  1      │ Hit!                        │  │
    │  0x003                        │  0        │  (invalid)                  │  1      │ Miss → Page Table Walk          │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────┘                          │  │
    │                                                                                         │  │
    │  → 4-way Set-Associative                                                              │  │
    │  → Parallel CAM comparison (all entries simultaneously)                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### TLB Miss 처리

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB Miss Handling                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Virtual Address                                                                        │  │
    │  VPN │  Offset                                                                         │  │
    │  ───┴────┘                                                                               │  │
    │    │                                                                                     │  │
    │    ▼ (TLB Miss)                                                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Hardware Page Table Walk                                                          │  │  │
    │  │     • VPN으로 Page Table Index 접근                                                  │  │  │
    │  │  │  │  → L1 Cache Miss (TLB Miss + Cache Miss)                                   │  │  │
    │  │  │  → L2 Cache Hit (TLB Miss + Cache Hit)                                     │  │  │
    │  │  │  → Memory Access (TLB Miss + Cache Miss + Memory Hit)                           │  │  │
    │  │                                                                                     │  │  │
    │  │  2. PPN 획득                                                                           │  │  │
    │  │  3. TLB 업데이트 (VPN-PPN 쌍 추가)                                                       │  │  │
    │  │  4. Physical Address = PPN + Offset                                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Miss Penalty: 10~100 cycles                                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### TLB 설계 비교

| 구분 | 4-way | 8-way | Full |
|------|-------|-------|------|
| **Hit Rate** | 중간 | 높음 | 가장 높음 |
| **복잡도** | 낮음 | 중간 | 높음 |
| **전력** | 적음 | 중간 | 많음 |

### TLB Shootdown 해결

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB Shootdown Resolution                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Problem: 다중 프로세스가 같은 VPN 경합으로 경쟁                                      │  │
    │                                                                                         │  │
    │  Solution 1: ASID (Address Space ID)                                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  VPN │ ASID  │  Valid  │  PPN    │ Process                              │  │  │
    │  │  ────┼────┼─────┴────────┴────────┼ 1, 2, 3                           │  │  │
    │  │  0x01│  1   │  1      │  0x03   │  │  │  │  │
    │  │  0x01│  2   │  1      │  0x07   │  │  │  │  │
    │  │  │  → 같은 VPN이라도 ASID로 구분                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Solution 2: VPN Tagging                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  VPN │  Process │  Valid  │  PPN                                    │  │  │
    │  │  ────┼─────────┼─────┴────────┴────────┼ 0x03 (Tagged by Process 1)            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

| 효과 | 없음 | TLB |
|------|------|-----|
