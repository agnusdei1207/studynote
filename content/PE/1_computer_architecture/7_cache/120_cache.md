# 캐시 (Cache)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시는 **"속도**가 **빠른 **작은 **메모리**를 **사용**하여 **느린 **메인 **메모리 **접근**을 **줄이는 **기술"**로, **Temporal Locality**(시간 지역성), **Spatial Locality**(공간 지역성)을 **활용**하고 **Cache Line**(블록 단위)으로 **데이터**를 **가져오며 **Hit Ratio**(적중률)로 **성능**을 **결정**한다.
> 2. **교체 정책**: **LRU**(Least Recently Used), **FIFO**, **Random**으로 **캐시 **라인**을 **교체**하며 **Write Policy**(Write-Through, Write-Back, Write-Around)로 **쓰기**를 **최적화**하고 **Multi-Level Cache**(L1, L2, L3)로 **계층**을 **형성**한다.
> 3. **융합**: **Direct-Mapped**(직접 매핑), **Set-Associative**(세트 연관), **Fully-Associative**(완전 연관)로 **구현**되며 **Inclusive**(상위 캐시 포함), **Exclusive**(독립), **Victim Cache**(피해 캐시)로 **계층 간 **데이터**를 **관리**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
캐시는 **"속도 빠른 작은 메모리"**이다.

**캐시 특징**:
- **빠름**: CPU에 가까움
- **작음**: 용량 작음
- **비쌈**: SRAM 사용
- **Locality**: 지역성 활용

### 💡 비유
캐시는 ****책상**과 **책장****과 같다.
- **책상**: 캐시 (빠름)
- **책장**: 메모리 (느림)
- **서재**: 디스크 (더 느림)

### 매핑 방식

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Cache Mapping                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Direct-Mapped:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Block 0 → Cache Set 0                                                               │  │
    │  Block 1 → Cache Set 1                                                               │  │
    │  Block N → Cache Set N (N mod CacheSize)                                            │  │
    │  • Simple                                                                             │  │
    │  │  → Thrashing (Conflict Miss)                                                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    4-Way Set-Associative:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Block 0, 4, 8, 12... → Cache Set 0 [4 lines]                                         │  │
    │  Block 1, 5, 9, 13... → Cache Set 1 [4 lines]                                         │  │
    │  • Flexible                                                                         │  │
    │  → Conflict 감소                                                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Fully-Associative:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Any Block → Any Cache Line                                                          │  │
    │  • Most Flexible                                                                     │  │
    │  → Complex Hardware                                                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 매핑 방식 비교

| 구분 | Direct | Set-Associative | Full |
|------|--------|-----------------|------|
| **하드웨어** | 단순 | 중간 | 복잡 |
| **Hit Rate** | 낮음 | 중간 | 높음 |
| **Conflict Miss** | 많음 | 적음 | 없음 |

### 교체 정책

| 정책 | 설명 | hit rate |
|------|------|----------|
| **LRU** | 오랫간 안 쓴 것 교체 | 높음 |
| **LFU** | 적게 쓰인 것 교체 | 중간 |
| **Random** | 무작위 교체 | 낮음 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 배열 순회
**상황**: 캐시 친화
**판단**: 순차 접근이 아닌 랜델 순회

```c
// Bad (Cache Miss 난무)
int sum = 0;
for (int i = 0; i < N; i++) {
    for (int j = 0; j < M; j++) {
        sum += data[j][i];  // 열 우선 (cache friendly)
    }
}

// Good (Spatial Locality)
int sum = 0;
for (int i = 0; i < N; i++) {
    for (int j = 0; j < M; j++) {
        sum += data[i][j];  // 행 우선
    }
}
```
