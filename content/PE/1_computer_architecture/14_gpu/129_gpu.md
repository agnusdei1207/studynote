# GPU 아키텍처 (GPU Architecture)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: GPU는 **\"대규모 **병렬 **처리**를 **위한 **SIMT**(Single Instruction Multiple Threads)**아키텍처\"**로, **수천 ****코어**(CUDA Cores)**가 **동시 **연산**을 **수행**하며 **그래픽 **렌더링**, **AI**, **과학 **계산**에 **사용**된다.
> 2. **구조**: **SM**(Streaming Multiprocessor)**가 **CUDA Cores**(32/64/128)**를 **관리**하고 **Global Memory**(GDDR, **HBM2), **Shared Memory**(SM 내부), **Registers**로 **구성**되며 **Warp**(32 **threads)**가 **동일 **명령어**를 **실행**한다.
> 3. **프로그래밍**: **CUDA**(NVIDIA), **OpenCL**(범용), **HIP**(AMD)**로 **Kernel **함수**를 **작성**하고 **<<<Grid, Block>>>** **구조**로 **Thread**를 **배열**하며 **Memory Coalescing**, **Warp Divergence**를 **최적화**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
GPU는 **"대규모 병렬 프로세서"**이다.

**CPU vs GPU**:
| 구분 | CPU | GPU |
|------|-----|-----|
| **코어** | 수십 개 | 수천 개 |
| **클럭** | 높음 (3-5GHz) | 낮음 (1-2GHz) |
| **용도** | 복잡 제어 | 단순 반복 |
| **메모리** | 작고 빠름 | 크고 느림 |

### 💡 비유
GPU는 ****공장 **라인 ****과 같다.
- **작업자**: Core
- **부품**: Data
- **동일 작업**: SIMT

### SIMT (Single Instruction Multiple Threads)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         SIMT Execution Model                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Warp (32 threads executing same instruction):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Warp Scheduler dispatches 32 threads to CUDA Cores                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Instruction: FADD                                                                  │  │  │
    │  │  Thread 0: r0 + r1 → r2                                                             │  │  │
    │  │  Thread 1: r0 + r1 → r2                                                             │  │  │
    │  │  ...                                                                                  │  │  │
    │  │  Thread 31: r0 + r1 → r2                                                            │  │  │
    │  │                                                                                       │  │  │
    │  │  → All 32 threads execute same instruction on different data                         │  │  │
    │  │  → SIMD-like execution but with thread-level divergence support                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Warp Divergence (Performance Penalty):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Code: if (threadIdx.x < 16) { A } else { B }                                           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Warp (32 threads):                                                                   │  │  │
    │  │  Threads 0-15: Execute A (threads 16-31 disabled)                                    │  │  │
    │  │  Threads 16-31: Execute B (threads 0-15 disabled)                                    │  │  │
    │  │                                                                                       │  │  │
    │  │  → Serial execution of A then B (2x time)                                             │  │  │
    │  │  → Avoid divergence within warp for best performance                                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Memory Hierarchy

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         GPU Memory Hierarchy                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Memory Types (per SM):                                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Registers (fastest, per-thread)                                                      │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • Size: 255 × 32-bit per thread (max)                                            │  │  │  │
    │  │  │  • Latency: 1 clock                                                                │  │  │  │
    │  │  │  • Scope: Thread-private                                                          │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │           │                                                                             │  │  │
    │  │           ▼                                                                             │  │  │
    │  │  Shared Memory (fast, per-block)                                                       │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • Size: 48KB (configurable with L1)                                               │  │  │  │
    │  │  │  • Latency: ~30 clocks                                                             │  │  │  │
    │  │  │  • Scope: Block-level (all threads in block can access)                            │  │  │  │
    │  │  │  • Use: Data sharing between threads, reduce global memory access                 │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │           │                                                                             │  │  │
    │  │           ▼                                                                             │  │  │
    │  │  L1 Cache (per SM)                                                                     │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • Size: 128KB (configurable with shared memory)                                   │  │  │  │
    │  │  │  • Latency: ~80 clocks                                                             │  │  │  │
    │  │  │  • Scope: SM-private                                                               │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │           │                                                                             │  │  │
    │  │           ▼                                                                             │  │  │
    │  │  L2 Cache (GPU-wide)                                                                   │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • Size: 40960KB                                                                   │  │  │  │
    │  │  │  • Latency: ~200 clocks                                                            │  │  │  │
    │  │  │  • Scope: All SMs share                                                            │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │           │                                                                             │  │  │
    │  │           ▼                                                                             │  │  │
    │  │  Global Memory (DRAM, slowest, largest)                                                 │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • Size: 80GB (HBM2)                                                               │  │  │  │
    │  │  │  • Bandwidth: 1555 GB/s                                                            │  │  │  │
    │  │  │  • Latency: ~500 clocks                                                            │  │  │  │
    │  │  │  • Scope: All threads can access                                                   │  │  │  │
    │  │  │  • Coalescing: Combine 32 thread accesses into 1 transaction                        │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Memory Coalescing

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Memory Coalescing                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Uncoalesced Access (Slow):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Thread 0:访问 global_mem[0]         → 32-byte transaction (128-bit aligned)             │  │
    │  Thread 1:访问 global_mem[100]       → 32-byte transaction (different cache line)        │  │
    │  Thread 2:访问 global_mem[200]       → 32-byte transaction                               │  │
    │  ...                                                                                      │  │
    │  → 32 transactions = high memory latency                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Coalesced Access (Fast):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Thread 0:访问 global_mem[0]                                                             │  │
    │  Thread 1:访问 global_mem[1]                                                             │  │
    │  Thread 2:访问 global_mem[2]                                                             │  │
    │  ...                                                                                      │  │
    │  Thread 31:访问 global_mem[31]                                                           │  │
    │  → 1 transaction (128 bytes) = low latency                                               │  │
    │                                                                                         │  │
    │  Requirement:                                                                            │  │
    │  • Access consecutive 32-bit words                                                       │  │
    │  • Aligned to 128-byte boundary                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### CUDA Programming Model

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         CUDA Grid/Block/Thread Hierarchy                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Kernel Launch:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  // 2D matrix addition: C = A + B                                                        │  │
    │  __global__ void matrixAdd(float *A, float *B, float *C, int N) {                        │  │
    │      int row = blockIdx.y * blockDim.y + threadIdx.y;                                   │  │
    │      int col = blockIdx.x * blockDim.x + threadIdx.x;                                   │  │
    │      if (row < N && col < N) {                                                           │  │
    │          C[row * N + col] = A[row * N + col] + B[row * N + col];                        │  │
    │      }                                                                                    │  │
    │  }                                                                                       │  │
    │                                                                                         │  │
    │  int main() {                                                                            │  │
    │      dim3 blockSize(16, 16);         // 16×16 = 256 threads per block                    │  │
    │      int numBlocks = (N + blockSize.x - 1) / blockSize.x;                                │  │
    │      dim3 gridSize(numBlocks, numBlocks);                                                │  │
    │                                                                                         │  │
    │      matrixAdd<<<gridSize, blockSize>>>(A, B, C, N);                                     │  │
    │      cudaDeviceSynchronize();                                                             │  │
    │  }                                                                                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Grid/Block/Thread Visualization:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Grid (1024×1024 threads total)                                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Block (0,0)      Block (1,0)      Block (2,0)  ...                                   │  │  │
    │  │  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐                                  │  │  │
    │  │  │ (0,0) (1,0)  │ │ (16,0) (17,0)│ │ (32,0) (33,0)│                                  │  │  │
    │  │  │ (0,1) (1,1)  │ │ (16,1) (17,1)│ │ (32,1) (33,1)│                                  │  │  │
    │  │  │  ...         │ │  ...         │ │  ...         │                                  │  │  │
    │  │  └──────────────┘ └──────────────┘ └──────────────┘                                  │  │  │
    │  │  Block (0,1)      Block (1,1)      Block (2,1)  ...                                   │  │  │
    │  │  ...                                                                                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 행렬 곱셈 가속
**상황**: 4096×4096 행렬 곱셈
**판단**: Shared Memory + Tiling

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Matrix Multiplication Optimization                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Naive Approach (Slow):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  __global__ void matMul_naive(float *A, float *B, float *C, int N) {                    │  │
    │      int row = blockIdx.y * blockDim.y + threadIdx.y;                                   │  │
    │      int col = blockIdx.x * blockDim.x + threadIdx.x;                                   │  │
    │      float sum = 0;                                                                     │  │
    │      for (int k = 0; k < N; k++) {                                                      │  │
    │          sum += A[row * N + k] * B[k * N + col];  // Global memory access every iteration│  │
    │      }                                                                                    │  │
    │      C[row * N + col] = sum;                                                             │  │
    │  }                                                                                       │  │
    │  → N global memory accesses per element = high latency                                   │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Optimized (Shared Memory Tiling):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  __global__ void matMul_tiled(float *A, float *B, float *C, int N) {                     │  │
    │      __shared__ float ds_A[TILE_SIZE][TILE_SIZE];                                        │  │
    │      __shared__ float ds_B[TILE_SIZE][TILE_SIZE];                                        │  │
    │                                                                                         │  │
    │      int bx = blockIdx.x, by = blockIdx.y;                                               │  │
    │      int tx = threadIdx.x, ty = threadIdx.y;                                             │  │
    │      int row = by * TILE_SIZE + ty;                                                      │  │
    │      int col = bx * TILE_SIZE + tx;                                                      │  │
    │                                                                                         │  │
    │      float sum = 0;                                                                      │  │
    │                                                                                         │  │
    │      for (int t = 0; t < N / TILE_SIZE; t++) {                                          │  │
    │          // Load tile into shared memory (coalesced access)                              │  │
    │          ds_A[ty][tx] = A[row * N + t * TILE_SIZE + tx];                                 │  │
    │          ds_B[ty][tx] = B[(t * TILE_SIZE + ty) * N + col];                               │  │
    │          __syncthreads();  // Wait for all threads to load                               │  │
    │                                                                                         │  │
    │          // Compute using shared memory (fast)                                           │  │
    │          for (int k = 0; k < TILE_SIZE; k++) {                                          │  │
    │              sum += ds_A[ty][k] * ds_B[k][tx];                                           │  │
    │          }                                                                                │  │
    │          __syncthreads();  // Wait for all threads to compute                            │  │
    │      }                                                                                    │  │
    │                                                                                         │  │
    │      C[row * N + col] = sum;                                                             │  │
    │  }                                                                                       │  │
    │                                                                                         │  │
    │  → Shared memory reduces global memory access by factor of TILE_SIZE                     │  │
    │  → ~10x faster than naive implementation                                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

2. **Ray Tracing**: RT Core
3. **Unified Memory**: CPU/GPU 통합
