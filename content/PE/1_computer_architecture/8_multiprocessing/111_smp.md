# SMP (Symmetric Multi-Processing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SMP는 **"여러 CPU가 **공유 메모리를 **접근하고 **동일한 **권한**으로 **작업을 **분담하는 **아키텍처"**로, **각 CPU가 **독립적으로 **스케줄링**되며 **Shared Memory**를 통해 **데이터 교환**하고 **Interrupt Balancing**으로 **부하**를 **분산**한다.
> 2. **가치**: **병렬 처리**(Parallelism)로 **성능**을 **선형적으로 **향상**하고 **장애 격리**(Fault Isolation)로 **신뢰성**을 **높이며 **Multi-Core** 프로세서의 **표준 아키텍처**로 **채택**되었다.
> 3. **융합**: **UMA**(Uniform Memory Access), **NUMA**(Non-Uniform Memory Access), **COMA**(Cache Only Memory Access)가 **메모리 접근** 방식에 따라 **분류**되며 **x86**(Intel, AMD), **ARM**의 **Multi-Core** 시스템에서 **구현**된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
SMP는 **"여러 CPU가 공유 메모리를 사용하는 시스템"**이다.

**SMP의 특징**:
- **Shared Memory**: 모든 CPU가 접근
- **Symmetric**: 동일한 권한
- **Single OS**: 하나의 운영체제

### 💡 비유
SMP는 **"여러 사람이 **하나의 **공동 작업실**에서 **일하는 것"**과 같다.
- **각자**: 독립 작업
- **공간**: 공유
- **조정**: 필요

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         멀티프로세싱 발전                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Uni-Processor:
    • 단일 CPU
    • 성능 한계
         ↓
ASMP (Asymmetric MP):
    • Master-Slave
    • 비대칭 권한
         ↓
SMP (1980년대):
    • 대칭 권한
    • 공유 메모리
         ↓
Multi-Core (2000년대):
    • 칩 내 다중 코어
    • 표준 아키텍처
```

## Ⅲ. 융합 비교 및 다각도 분석

### SMP vs AMP vs NUMA

| 구분 | SMP | AMP | NUMA |
|------|-----|-----|------|
| **권한** | 대칭 | 비대칭 | 대칭 |
| **메모리** | 공유 | 분리 | 분산 |
| **확장성** | 제한 | 중간 | 높음 |
| **Complexity** | 낮음 | 중간 | 높음 |

## Ⅴ. 기대효과 및 결론

### SMP 기대 효과

| 효과 | Single-Core | SMP |
|------|----------|-----|
| **처리량** | 1x | N x |
| **신뢰성** | 낮음 | 높음 |
| **복잡도** | 낮음 | 높음 |

1. **Heterogeneous MP**: 다양한 Core
2. **Many-Core**: 수백 코어
3. **Accelerator**: GPU/NPU 통합
