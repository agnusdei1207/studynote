# CISC (Complex Instruction Set Computer)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CISC는 **"복잡한 명령어** 세트로 **프로그램 크기**를 **최소화**하고 **메모리 접근**을 **줄이는 **프로세서 아키텍처"**로, **가변 길이 명령어**를 **사용**하고 **Microcode**로 **복잡한 연산**을 **구현**하며 **Memory-to-Memory** 명령어를 **지원**한다.
> 2. **가치**: **코드 크기**가 **작아** **메모리 효율**이 **우수**하고 **복잡한 연산**이 **단일 명령어**로 **가능**하며 **하위 호환성**이 **유지**되어 **레거시 소프트웨어**가 **동작**한다.
> 3. **융합**: **x86**(Intel, AMD), **VAX**(DEC), **IBM System/370**가 **대표적**이며 **RISC**와 **경쟁**하다 **현대에는 x86-64**가 **CISC 외부**에 **RISC 내부**를 **채택**한 **Hybrid 아키텍처**로 **진화**했다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
CISC는 **"복잡한 명령어 세트를 가진 프로세서"**이다.

**CISC 특징**:
- **Complex Instruction**: 다양한 명령어
- **Variable Length**: 가변 길이 명령어
- **Microcode**: 내부 마이크로코드
- **Memory-to-Memory**: 메모리 간 직접 연산

### 💡 비유
CISC는 **"스위스 아미 나이프****와 같다.
- **복잡한 도구**: 다양한 기능
- **한 번에 완료**: 복잡한 작업
- **메모리 절약**: 작은 코드

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세서 아키텍처 발전                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

초기 (1960년대):
    • 메모리 비쌈
    • 코드 크기 최소화 필요
         ↓
CISC (1970년대):
    • IBM System/360
    • x86 (Intel 8086)
         ↓
현대:
    • x86-64 (Hybrid)
    • CISC 외부 + RISC 내부
```

## Ⅱ. 아키텍처 및 핵심 원리

### CISC vs RISC

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         CISC vs RISC                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CISC (Complex Instruction Set Computer)                                                │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  • 복잡한 명령어 (MOV AX, [BX])                                                        │  │
    │  │  • Memory-to-Memory (MOV [SI], [DI])                                                 │  │
    │  │  • 다양한 Addressing Mode                                                            │  │
    │  │  • 1~15 Byte 가변 길이 명령어                                                         │  │
    │  │  │  → Microcode로 복잡한 연산 구현                                                      │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  예: x86 (CISC)                                                                      │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  ADD AX, [BX]         ; 메모리에서 읽어 AX에 더하기                                     │  │  │
    │  │                         ; 3 Byte 명령어 (한 번에 완료)                                    │  │  │
    │  │                                                                                       │  │  │
    │  │  REP MOVSB            ; 문자열 반향 복사 (CX 바이트)                                     │  │  │
    │  │                         ; Hardware Loop 내장                                            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  RISC (Reduced Instruction Set Computer)                                               │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  • 간단한 명령어 (LOAD, STORE, ADD)                                                   │  │
    │  │  • Load/Store Architecture                                                             │  │
    │  │  • 3-address 형식 (ADD R1, R2, R3)                                                    │  │
    │  │  • 고정 길이 명령어                                                                    │  │
    │  │  │  → Pipeline에 유리                                                                  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  예: MIPS (RISC)                                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  LW   R1, 0(R2)        ; R1 ← Mem[0]                                      │  │  │
    │  │  LW   R2, 4(R3)        ; R2 ← Mem[4]                                      │  │  │
    │  │  ADD   R1, R2, R3      ; R3 ← R1 + R2                                    │  │  │
    │  │                         ; 3 명령어 (각 4 Byte)                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

```
    │                                                                                         │  │
    │  Program: ADD [BX], AX      ; 메모리[BX] ← 메모리[BX] + AX                                 │  │

    │    ▼                                                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  1. Instruction Fetch (3 Byte)                                                        │  │
    │  │     OP: ADD   ModR/M: [BX], AX                                                        │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │    │                                                                                     │  │
    │    ▼                                                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  2. Microcode ROM Lookup                                                             │  │
    │  │     ┌────────────────────────────────────────────────────────────────────────────┐    │  │
    │  │     │  uop1: TEMP ← Mem[BX]           ; Load from memory                          │    │  │
    │  │     │  uop2: TEMP ← TEMP + AX          ; ALU Add                                 │    │  │
    │  │     │  uop3: Mem[BX] ← TEMP            ; Store to memory                          │    │  │
    │  │     └────────────────────────────────────────────────────────────────────────────┘    │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │    │                                                                                     │  │
    │    ▼                                                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  3. Micro-operation Execution (RISC 내부)                                             │  │
    │  │     Decoder → uop Queue → Execute Units                                             │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### x86 Addressing Mode

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         x86 Addressing Mode                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Immediate Addressing                                                            │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  MOV AX, 1234h          ; AX ← 0x1234 (상수)                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  2. Register Addressing                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  MOV AX, BX             ; AX ← BX (레지스터)                                         │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  3. Direct Addressing                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  MOV AX, [1000h]        ; AX ← Mem[0x1000]                                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  4. Register Indirect                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  MOV AX, [BX]           ; AX ← Mem[BX]                                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  5. Based Indexed Addressing                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  MOV AX, [BX+SI]        ; AX ← Mem[BX+SI]                                            │  │  │
    │  │  MOV AX, [BX+SI+5]      ; AX ← Mem[BX+SI+5]                                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### CISC vs RISC 비교

| 구분 | CISC | RISC |
|------|------|-----|
| **명령어 수** | 많음 (~500+) | 적음 (~100) |
| **명령어 길이** | 가변 (1-15 Byte) | 고정 (4 Byte) |
| **코드 크기** | 작음 | 큼 |
| **디코딩** | 복잡 (Microcode) | 간단 (Hardwired) |
| **성능 (IPC)** | 낮음~중간 | 높음 |
| **전력 효율** | 낮음 | 높음 |
| **메모리 접근** | Memory-to-Memory | Load/Store만 |

```
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: x86 어셈블리 최적화
**상황**: x86 코드
**판단**:

```assembly
; x86-64 Assembly (CISC)

; 1. 복잡한 명령어 활용 (코드 크기 감소)
movsb                   ; [DS:SI] → [ES:DI], SI++, DI++
rep movsb               ; CX만큼 반복 (Hardware Loop)

; 2. Memory-to-Memory 연산
add [bx], ax            ; Mem[BX] ← Mem[BX] + AX (한 명령어)

; 3. 복잡한 Addressing Mode
lea eax, [ebx + ecx*4]  ; EAX ← EBX + ECX*4 (배열 인덱스 계산)
```

## Ⅴ. 기대효과 및 결론

### CISC 기대 효과

| 효과 | RISC | CISC |
|------|------|------|
| **코드 크기** | 큼 | 작음 |
| **메모리** | 많이 사용 | 적게 사용 |
| **디코딩** | 빠름 | 느림 |
| **전력** | 효율 | 비효율 |

### 미래 전망

1. **Hybrid**: CISC + RISC
2. **x86-64**: 서버/데스크탑
3. **ARM**: 모바일/임베디드
