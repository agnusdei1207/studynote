# VLIW (Very Long Instruction Word)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: VLIW는 **"컴파일러가 **명령어 **병렬성**(ILP)을 **추출**하여 **하나의 **매우 긴 **명령어**(Very Long Instruction Word)로 **묶어** **여러 연산**을 **동시에 **실행**하는 **아키텍처"**로, **Hardware Scheduler**가 **없고 **Compiler가 **모든 **종속성**을 **해결**한다.
> 2. **가치**: **하드웨어 복잡도**가 **낮아** **전력 소모**가 **적고 **Clock Speed**를 **높이기 **쉬우며 **Compiler 최적화**로 **최대의 **명령어 **병렬성**을 **활용**할 수 **있어 **Embedded**, **DSP**(Digital Signal Processor)에서 **주로** 사용된다.
> 3. **융합**: **Intel Itanium**(EPIC), **TI TMS320**(DSP), **HP PA-RISC**, **Transmeta Crusoe**가 **대표적**이며 **Modern GPU**와 **SIMD**(AVX-512, NEON)가 **VLIW 개념**을 **계승**했다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
VLIW는 **"컴파일러가 병렬 명령어를 묶어서 보내는 아키텍처"**이다.

**VLIW 특징**:
- **Long Instruction**: 여러 명령어 포함
- **Compiler Scheduling**: 컴파일러가 병렬화
- **No Hazard Detection**: 하드웨어 단순
- **Fixed Bundle**: 명령어 그룹

### 💡 비유
VLIW는 **"오케스트라 악보****와 같다.
- **악보**: VLIW 명령어
- **악기들**: 실행 유닛
- **지휘자**: 컴파일러

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세서 아키텍처 발전                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Superscalar (1980년대):
    • Hardware가 명령어 스케줄링
    • 복잡한 하드웨어
         ↓
VLIW (1990년대):
    • Compiler가 명령어 스케줄링
    • 단순한 하드웨어
         ↓
EPIC (2000년대):
    • Intel Itanium
    • 예측(Predication)
```

### VLIW vs Superscalar

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         VLIW vs Superscalar                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Superscalar (Hardware Scheduling)                                                      │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Program:                                                                           │  │
    │  │    ADD R1, R2, R3       │  │  │
    │  │    SUB R4, R5, R6       │  │  │
    │  │    MUL R7, R8, R9       │  │  │
    │  │                                                                                       │  │  │
    │  │  Hardware (Runtime):                                                             │  │  │
    │  │  • Hazard Detection                                                               │  │  │
    │  │  │  • Out-of-Order Execution                                                        │  │  │
    │  │  │  • Dynamic Scheduling                                                            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  VLIW (Compiler Scheduling)                                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Program:                                                                           │  │
    │  │    ADD R1, R2, R3       │  │  │
    │  │    SUB R4, R5, R6       │  │  │
    │  │    MUL R7, R8, R9       │  │  │
    │  │                                                                                       │  │  │
    │  │  Compiler (Compile-time):                                                         │  │  │
    │  │  • Dependency Analysis                                                             │  │  │
    │  │  │  • Instruction Bundling                                                           │  │  │
    │  │  │  • NOOP 삽입 (if needed)                                                         │  │  │
    │  │                                                                                       │  │  │
    │  │  VLIW Instruction (128-bit):                                                        │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  [ADD R1,R2,R3] │ [SUB R4,R5,R6] │ [MUL R7,R8,R9] │ [NOOP]                          │  │  │
    │  │  │   32-bit          │   32-bit        │   32-bit        │  32-bit                    │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                       │  │  │
    │  │  Hardware:                                                                           │  │  │
    │  │  • Direct Dispatch to Functional Units                                               │  │  │
    │  │  │  • No Scheduling, No Detection                                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 아키텍처 비교

| 구분 | Superscalar | VLIW | EPIC |
|------|------------|------|------|
| **스케줄링** | Hardware | Compiler | Compiler + Hints |
| **복잡도** | 높음 | 낮음 | 중간 |
| **전력** | 높음 | 낮음 | 중간 |
| **코드 호환** | 좋음 | 나쁨 | 중간 |
