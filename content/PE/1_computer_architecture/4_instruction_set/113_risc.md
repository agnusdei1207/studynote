# RISC (Reduced Instruction Set Computer)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RISC는 **"간단한 명령어 **셋으로 **명령어 **해독과 **실행**을 **고속화**하는 **프로세서 아키텍처"**로, **Load/Store** 구조를 **사용**하고 **Pipeline**으로 **명령어 **병렬** 실행하며 **Compiler**가 **최적화**를 **수행**한다.
> 2. **가치**: **CISC**(Complex Instruction Set Computer)보다 **간단한 명령어**로 **디코딩**이 **간단**하고 **Pipeline**으로 **높은 **IPC**(Instructions Per Cycle)를 **달성**하며 **Power Efficiency**가 **우수**하여 **Mobile**, **Embedded** 시스템에서 **주로** 사용된다.
> 3. **융합**: **ARM**(RISC-V, ARM Cortex), **MIPS**(MIPS I, III), **RISC-V**(오픈 소스), **POWER**(PowerPC, Power ISA)가 **대표적**이며 **x86**(CISC)과 **경쟁**하며 **Apple Silicon**(ARM 기반), **Android**(ARM)가 **RISC**를 **채택**했다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
RISC는 **"간단한 명령어 세트를 가진 프로세서"**이다.

**RISC 특징**:
- **Simple Instruction**: 1 Cycle 실행
- **Load/Store**: 메모리 접근 명령어만
- **Large Register**: 많은 레지스터
- **Pipeline**: 병렬 처리

### 💡 비유
RISC는 **"스위스 군대****와 같다.
- **간단한 명령**: 기동만 수행
- **빠른 실행**: 압속 전술
- **컴파일러 최적화**: 코드 변환

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         프로세서 아키텍처 발전                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

CISC (1970년대):
    • 복잡한 명령어
    • 마이크로코드
         ↓
RISC (1980년대):
    • 간단한 명령어
    • Stanford MIPS, Berkeley RISC
         ↓
 현대:
    • ARM (Mobile)
    • RISC-V (오픈 소스)
    • POWER (Server)
```

## Ⅱ. 아키텍처 및 핵심 원리

### RISC vs CISC

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         RISC vs CISC                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CISC (Complex Instruction Set Computer)                                                │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  • 복잡한 명령어 (MOV AX, [BX])                                                        │  │
    │  │  • Memory-to-Memory (MOV [SI], [DI])                                                 │  │
    │  │  • 다양한 Addressing Mode                                                            │  │
    │  │  • 1~15 Byte 명령어                                                              │  │
    │  │  │  → Microcode로 복잡                                                                   │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  예: x86 (CISC)                                                                      │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  ADD AX, [BX]         ; 메모리에서 읽어 AX에 더하기                                     │  │  │
    │  │                         ; 3 Byte 명령어                                              │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  RISC (Reduced Instruction Set Computer)                                               │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  • 간단한 명령어 (LOAD, STORE, ADD)                                                        │  │
    │  │  • Load/Store Architecture                                                              │  │
    │  │  • 3-address 형식 (Load R1, R2, R3)                                                  │  │
    │  │  • 고정 길이 명령어                                                                 │  │
    │  │  │  → Pipeline에 유리                                                                   │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  예: MIPS (RISC)                                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  LW   R1, 0(R2)        ; R1 ← Mem[0]                                      │  │  │
    │  │  LW   R2, 4(R3)        ; R2 ← Mem[4]                                      │  │  │
    │  │  ADD   R1, R2, R3        ; R3 ← R1 + R2                                   │  │  │
    │  │                         ; 3 명령어 (각 4 Byte)                                  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### RISC Pipeline

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         RISC Pipeline (5-Stage)                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  │  IF     ID     EX     MEM    WB                                                   │
    │  │  ┌───┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐                              │  │
    │  │  │   │  │   │  │   │  │   │  │   │                              │  │
    │  │  └───┘  └───┘  └───┘  └───└───┘  └───┘  (5-stage)              │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  • IF: Instruction Fetch (명령어 인출)                                                     │  │
    │  │  ID: Instruction Decode (해독)                                                        │  │
    │  │  • EX: Execute (연산)                                                                  │  │
    │  │  │  • MEM: Memory Access (메모리 접근)                                                   │  │
    │  │  • WB: Write-Back (결과 저장)                                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### RISC vs CISC 비교

| 구분 | RISC | CISC |
|------|-----|------|
| **명령어 수** | 적음 (~100) | 많음 (~500+) |
| **명령어 길이** | 고정 | 가변 |
| **코드 크기** | 큼음 | 작음 |
| **디코딩** | 간단 | 복잡 |
| **성능 (IPC)** | 높음 | 낮음~중간 |
| **전력 효율** | 높음 | 낮음 |

### RISC 예시 (ARM)

```
┌─────────────────────────────────────────────────────────────────────────────────────────────────┐
│                         ARM Instruction Set (RISC)                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  # Load-Store Architecture                                                           │  │
    │  │                                                                                       │  │
    │  │  # 데이터 이동                                                                         │  │
    │  │  LDR R1, [R2]      ; R1 ← Mem[R2]         # Load Word                           │  │
    │  │  STR R1, [R2]      ; Mem[R2] ← R1         # Store Word                          │  │
    │  │                                                                                       │  │
    │  │  # 연산 명령어                                                                         │  │
    │  │  ADD R1, R1, R2      ; R1 ← R1 + R2       # Add                               │  │
    │  │  SUB R1, R1, R2      ; R1 ← R1 - R2       # Subtract                          │  │
    │  │  MUL R1, R1, R2      ; R1 ← R1 * R2       # Multiply                         │  │  │
    │  │                                                                                       │  │
    │  │  # 분기 명령어                                                                         │  │
    │  │  B   label            ──>  # Branch if condition met                                        │  │
    │  │  BL  target          ──>  # Branch and Link                                                │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: ARM Assembly 최적화
**상황**: ARM 코드
**판단**:

```assembly
# ARM64 (RISC)
# x를 레지스터에 저장
ldr x0, =0x1000     # x0 ← Mem[0x1000]
add x0, x0, #1        # x0 ← x0 + 1
str x0, [x1]          # Mem[x1] ← x0
```

## Ⅴ. 기대효과 및 결론

### RISC 기대 효과

| 효과 | CISC | RISC |
|------|-----|------|
| **디코딩** | 느림 | 빠름 |
| **Pipeline** | 어려움 | 쉬움 |
| **전력** | 낮음 | 높음 |
| **코드 크기** | 작음 | 큼름 |

### 미래 전망

1. **ARM**: Mobile, Embedded
2. **RISC-V**: 오픈 소스
3. **Domain-Specific**: GPU, DSP
