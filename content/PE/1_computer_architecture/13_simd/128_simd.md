# SIMD (Single Instruction Multiple Data)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SIMD는 **\"단일 **명령어**로 **여러 **데이터**를 **병렬 **처리**하는 **기법\"**으로, **벡터 **레지스터**(128-512bit)에 **여러 **값**을 **저장**하고 **동시 **연산**을 **수행**하여 **미디어**, **AI**, **과학 **계산**의 **성능**을 **향상**시킨다.
> 2. **명령어**: **SSE**(Streaming SIMD Extensions, 128bit), **AVX**(Advanced Vector Extensions, 256bit), **AVX-512**(512bit), **ARM NEON**(128bit)가 **대표적**이며 **FMA**(Fused Multiply-Add)로 **곱셈-덧셈 **연산**을 **최적화**한다.
> 3. **작동**: **Load**(벡터 **로드) → **Compute**(동시 **연산) → **Store**(결과 **저장)**으로 **이루어지고 **Loop **Vectorization**(자동 **병환)**과 **Intrinsics**(내장 **함수)**으로 **프로그래밍**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
SIMD는 **\"데이터 병렬 처리\"**이다.

**병렬 처리 유형**:
| 유형 | 설명 | 예시 |
|------|------|------|
| **SISD** | Single Instruction Single Data | 일반 CPU |
| **SIMD** | Single Instruction Multiple Data | 벡터 프로세서 |
| **MIMD** | Multiple Instruction Multiple Data | 멀티코어 |
| **SPMD** | Single Program Multiple Data | GPU |

### 💡 비유
SIMD는 ****요리 **사 ****와 같다.
- **한 손**: Single Instruction
- **여러 재료**: Multiple Data
- **동일 조리**: 동시 연산

## Ⅱ. 아키텍처 및 핵심 원리

### SIMD 레지스터 구조

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         SIMD Register Evolution                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    MMX (64-bit):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  MM0: [63:0] = 8 × 8-bit   or   4 × 16-bit   or   2 × 32-bit   or   1 × 64-bit         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    SSE (128-bit):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  XMM0: [127:0] = 16 × 8-bit   or   8 × 16-bit   or   4 × 32-bit   or   2 × 64-bit       │  │
    │  → 16 registers (XMM0-XMM15)                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    AVX (256-bit):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  YMM0: [255:0] = 32 × 8-bit   or   16 × 16-bit   or   8 × 32-bit   or   4 × 64-bit      │  │
    │  → Lower 128-bit = XMM0, Upper 128-bit = YMM0_hi                                       │  │
    │  → 16 registers (YMM0-YMM15)                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    AVX-512 (512-bit):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ZMM0: [511:0] = 64 × 8-bit   or   32 × 16-bit   or   16 × 32-bit   or   8 × 64-bit      │  │
    │  → Lower 256-bit = YMM0, Upper 256-bit = ZMM0_hi                                       │  │
    │  → 32 registers (ZMM0-ZMM31)                                                          │  │
    │  → Mask registers (K0-K7) for conditional operation                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### SIMD 연산 예시

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Vector Addition (AVX)                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Scalar Code (No SIMD):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  for (int i = 0; i < 8; i++) {                                                          │  │
    │      c[i] = a[i] + b[i];                                                                 │  │
    │  }                                                                                       │  │
    │  → 8 separate add instructions                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    SIMD Code (AVX):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  // Assume aligned arrays                                                                 │  │
    │  __m256i va = _mm256_load_si256((__m256i*)&a[0]);     // Load 8 × 32-bit integers         │  │
    │  __m256i vb = _mm256_load_si256((__m256i*)&b[0]);     // Load 8 × 32-bit integers         │  │
    │  __m256i vc = _mm256_add_epi32(va, vb);               // Add 8 pairs in parallel          │  │
    │  _mm256_store_si256((__m256i*)&c[0], vc);             // Store 8 × 32-bit results         │  │
    │                                                                                         │  │
    │  → 1 add instruction for 8 elements (8x speedup)                                         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Memory Layout:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  a[0]    a[1]    a[2]    a[3]    a[4]    a[5]    a[6]    a[7]                             │  │
    │  ┌───────┐┌───────┐┌───────┐┌───────┐┌───────┐┌───────┐┌───────┐┌───────┐                │  │
    │  │ int32 ││ int32 ││ int32 ││ int32 ││ int32 ││ int32 ││ int32 ││ int32 │                │  │
    │  └───────┘└───────┘└───────┘└───────┘└───────┘└───────┘└───────┘└───────┘                │  │
    │  └─────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                  256-bit YMM Register                                   │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### SIMD 확장 비교

| 확장 | 비트 | 레지스터 | 특징 | 사용처 |
|------|------|----------|------|--------|
| **MMX** | 64 | 8 × MMx | 정수만 | 레거시 |
| **SSE** | 128 | 16 × XMM | 부동소수점 | 미디어 |
| **AVX** | 256 | 16 × YMM | 3-operand | 범용 |
| **AVX-512** | 512 | 32 × ZMM | Masking | AI/HPC |

### FMA (Fused Multiply-Add)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         FMA Operation                                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Without FMA:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  a = b * c + d                                                                          │  │
    │                                                                                         │  │
    │  Step 1: temp = b * c        (MUL instruction)                                           │  │
    │  Step 2: a = temp + d        (ADD instruction)                                           │  │
    │  → 2 instructions, 2 rounding operations                                                │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    With FMA (FMA3: a = b * c + a):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  a = _mm256_fmadd_ps(b, c, d);  // Single instruction                                    │  │
    │  → 1 instruction, 1 rounding operation                                                   │  │
    │  → Better precision (single rounding)                                                    │  │
    │  → 2x throughput for multiply-add operations                                             │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Common pattern: Matrix multiplication
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  for (i = 0; i < N; i++) {                                                               │  │
    │      for (j = 0; j < N; j++) {                                                           │  │
    │          for (k = 0; k < N; k++) {                                                       │  │
    │              C[i][j] += A[i][k] * B[k][j];   // FMA friendly                             │  │
    │          }                                                                                │  │
    │      }                                                                                    │  │
    │  }                                                                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### ARM NEON

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         ARM NEON Architecture                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Register Bank:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  32 × 128-bit registers                                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  View as:   Q0 (128-bit)  = 2 × D (64-bit)  = 4 × S (32-bit)  = 16 × B (8-bit)       │  │  │
    │  │            Q1, Q2, ..., Q31                                                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    NEON Instruction Example:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  // Add 8 × 8-bit integers                                                               │  │
    │  int8x8_t va = vld1_s8(a);           // Load 8 bytes                                     │  │
    │  int8x8_t vb = vld1_s8(b);           // Load 8 bytes                                     │  │
    │  int8x8_t vc = vadd_s8(va, vb);      // Add 8 pairs                                      │  │
    │  vst1_s8(c, vc);                     // Store 8 bytes                                    │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Usage in Mobile:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Image processing: Filter, resize, color conversion                                    │  │
    │  • Video codec: Encode/decode acceleration                                                │  │
    │  • Audio: DSP effects, echo, reverb                                                      │  │
    │  • ML: Inference acceleration (INT8 quantization)                                         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 이미지 처리 최적화
**상황**: RGB → Grayscale 변환
**판단**: AVX2 병렬화

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Image Processing with SIMD                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Problem: Convert RGB image to Grayscale
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Gray = 0.299×R + 0.587×G + 0.114×B                                                     │  │
    │                                                                                         │  │
    │  Scalar Code (8 pixels = 24 values):                                                    │  │
    │  for (int i = 0; i < width * height; i++) {                                             │  │
    │      gray[i] = 0.299f * rgb[3*i] + 0.587f * rgb[3*i+1] + 0.114f * rgb[3*i+2];           │  │
    │  }                                                                                       │  │
    │  → 3 multiply + 2 add per pixel                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    SIMD Optimized (Process 8 RGB pixels = 24 values):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  // Load 8 RGB pixels (24 values) into 3 YMM registers                                   │  │
    │  __m256 r = _mm256_set_ps(r7, r6, r5, r4, r3, r2, r1, r0);                              │  │
    │  __m256 g = _mm256_set_ps(g7, g6, g5, g4, g3, g2, g1, g0);                              │  │
    │  __m256 b = _mm256_set_ps(b7, b6, b5, b4, b3, b2, b1, b0);                              │  │
    │                                                                                         │  │
    │  // Fused multiply-add: gray = 0.299*r + 0.587*g + 0.114*b                               │  │
    │  __m256 gray = _mm256_fmadd_ps(r, _mm256_set1_ps(0.299f),                               │  │
    │                       _mm256_fmadd_ps(g, _mm256_set1_ps(0.587f),                        │  │
    │                       _mm256_mul_ps(b, _mm256_set1_ps(0.114f))));                        │  │
    │                                                                                         │  │
    │  → 8 pixels processed in parallel                                                        │  │
    │  → ~8x speedup theoretical, ~4-6x real-world                                             │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### SIMD 기대 효과

| 워크로드 | 속도향상 | 제약 |
|----------|----------|------|
| **미디어** | 4-8x | 정렬 필요 |
| **과학계산** | 2-4x | 데이터 의존성 |
| **AI 추론** | 4-16x | 양자화 필요 |
| **암호화** | 2-8x | 알고리즘 의존 |

### 모범 사례

1. **정렬**: 16/32-byte boundary
2. **Compiler**: Auto-vectorization (-O3, -mavx2)
3. **Intrinsics**: _mm256_*, _mm512_*
4. **Masking**: 조건부 연산

### 미래 전망

1. **AMX**: Intel Advanced Matrix Extensions
2. **SVE**: ARM Scalable Vector Extension
3. **Matrix Engines**: NPU/GPU fusion
