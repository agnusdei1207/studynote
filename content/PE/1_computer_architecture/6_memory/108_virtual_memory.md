# 가상 메모리 (Virtual Memory)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 메모리는 **"물리 메모리(RAM)보다 **큰 논리적 주소 공간**을 **제공하는 메모리 관리 기법"**으로, **가상 주소**(Virtual Address)를 **물리 주소**(Physical Address)로 **변환**하고 **디스크**(Swap/Page File)를 **확장 메모리**로 사용한다.
> 2. **가치**: **프로세스마다 독립적인 주소 공간**을 제공하여 **메모리 보호**(Memory Protection)를 달성하고 **물리 메모리 한계**를 **초과**하는 **프로그램 실행**을 가능하게 하며 **멀티태스킹**을 **효율적**으로 지원한다.
> 3. **융합**: **Paging**(고정 크기 페이지), **Segmentation**(논리적 단위), **TLB**(Translation Lookaside Buffer)로 **주소 변환 성능**을 높이고 **Demand Paging**(필요 시 로드), **Page Replacement Algorithm**(LRU, LFU)으로 **메모리 관리**를 최적화한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
가상 메모리는 **"논리적 주소 공간을 물리적 메모리에 매핑하는 기술"**이다.

**가상 메모리의 목적**:
- **Abstraction**: 논리 주소 → 물리 주소 분리
- **Protection**: 프로세스 간 메모리 격리
- **Efficiency**: 물리 메모리 효율적 사용

### 💡 비유
가상 메모리는 **"가상 주소"**와 같다.
- **가상 주소**: 1번지, 2번지 (논리적)
- **실제 주소**: 건물 번지, 우편함 번호 (물리적)
- **주소 변환**: 우편배달부 (MMU)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         가상 메모리의 발전                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Base & Limit Register (1950년대):
    • 단일 연속 메모리
    • 메모리 낭비
         ↓
Partitioning (1960년대):
    • Fixed/Dynamic Partition
    • Fragmentation 문제
         ↓
Paging (1961, Atlas):
    • 고정 크기 페이지
    • 외부 단편화 해결
         ↓
Virtual Memory (1970년대):
    • Demand Paging
    • Swap 영역 활용
         ↓
현대:
    • Multi-level Page Table
    • TLB, ASLR
```

    │  │  │  └───────────────────────────────┘  └──────────────────────────────────────┘  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │

### TLB (Translation Lookaside Buffer)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB (주소 변환 캐시)                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Page Table Entry 캐시                                                                │
    │  │  • SRAM (빠름)                                                                       │
    │  │  • Parallel Search (Associative Memory)                                              │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  TLB Entry 구조                                                                     │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  VPN  │  PFN  │  Valid │  Dirty │  Access │  ASID                             │  │  │
    │  │  │  (20) │ (20)  │  (1)   │  (1)   │  (3)    │  (Process ID)                     │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  [TLB Hit/Miss]                                                                         │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  1. CPU가 가상 주소 요청                                                              │  │
    │  │  2. TLB 검색 (Parallel)                                                              │  │
    │  │     • Hit → PFN 즉시 반환 (1 Cycle!)                                                 │  │
    │  │     • Miss → Page Table Walk (10~100 Cycles) → TLB Update                            │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  [TLB Performance]                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  • TLB Hit Rate: 95~99%                                                             │  │
    │  │  • Miss Penalty: 10~100 cycles                                                      │  │
    │  │  • Effective Access Time = (Hit Rate × TLB Time) + (Miss Rate × Page Table Time)   │  │
    │  │    = 0.98 × 1 + 0.02 × 100 = 2.98 cycles                                           │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Paging vs Segmentation

| 구분 | Paging | Segmentation |
|------|--------|-------------|
| **단위** | 고정 크기 (4KB) | 가변 크기 (논리적) |
| **단편화** | 내부 단편화 | 외부 단편화 |
| **관리** | 간단 | 복잡 |
| **보호** | 페이지 단위 | 세그먼트 단위 |

### Page Replacement Algorithm

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         페이지 교체 알고리즘                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [LRU (Least Recently Used)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 가장 오랫동안 사용하지 않은 페이지 교체                                                   │
    │  │  • Time stamp 또는 Reference bit 사용                                                 │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Page │ A │ B │ C │ D │ E │ C │ B │ E │ C │ B │ A  (Reference String)              │  │
    │  │  ──────────────────────────────────────────────────────────────────────────────────  │  │
    │  │  Frames (3):                                                                         │  │
    │  │  │   A    │   A    │   A,B  │   A,B,C│  B,C,D│  C,D,E│  ...                        │  │
    │  │  │ Fault │ Fault │ Fault │ Fault │ Fault│ Fault│                                │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [LFU (Least Frequently Used)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 참조 횟수가 가장 적은 페이지 교체                                                       │
    │  │  • Counter 필요                                                                      │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [FIFO (First In First Out)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 가장 먼저 로드된 페이지 교체                                                            │
    │  │  • Circular Queue                                                                     │
    │  │  • Belady's Anomaly 발생 가능                                                         │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 페이지 폴트 분석
**상황**: 메모리 부족
**판단**:

```bash
# Linux 페이지 폴트 확인
$ ps -o minflt,majflt,cmd -p $$  # PID의 페이지 폴트
# minflt: Minor Fault (메모리에 존재)
# majflt: Major Fault (디스크 로드)

# 전체 시스템 페이지 폴트
$ vmstat -s | grep -i page
# 1234567 page faults
# 56789 major faults

# Page cache 확인
$ free -h
# buff/cache: 캐시된 페이지

# 스왑 사용량
$ swapon -s
$ cat /proc/swaps

# TLB miss 확인 (perf)
$ perf stat -e dTLB-loads,dTLB-load-misses ls
```

## Ⅴ. 기대효과 및 결론

### 가상 메모리 기대 효과

| 효과 | 가상 메모리 없을 시 | 가상 메모리 있을 시 |
|------|-----------------|-----------------|
| **주소 공간** | RAM 한계 | RAM + Disk |
| **멀티태스킹** | 제한적 | 효율적 |
| **메모리 보호** | 어려움 | 격리 |

### 미래 전망

1. **Huge Pages**: 2MB/1GB 페이지
2. **Direct Segment**: Paging 없이 직접 매핑
3. **Non-Volatile Memory**: NVDIMM 활용
