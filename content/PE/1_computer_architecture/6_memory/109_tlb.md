# TLB (Translation Lookaside Buffer)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TLB는 **"가상 주소 → 물리 주소 **변환 정보를 **캐시하는 **하드웨어**로, **Page Table Entry**(PTE)를 **저장**하고 **Parallel Search**(Associative Memory)로 **1 Cycle 내**에 **주소 변환**을 **완료**한다.
> 2. **가치**: **Page Table Memory Access**(10~100 cycles)를 **TLB Hit**(1 cycle)로 **대체**하여 **메모리 접근 성능**을 **획기적으로** 향상하고 **TLB Prefetching**, **Page Coloring**으로 **Hit Rate**를 **높인다**.
> 3. **융합**: **CPU의 MMU**에 **통합**되며 **Split TLB**(I-TLB, D-TLB), **Multi-level TLB**(L1, L2), **ASID**(Address Space ID)로 **Process 격리**를 **지원**하고 **Page Size**(4KB, 2MB, 1GB)에 따라 **다른 TLB Entry**를 **사용**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
TLB는 **"Page Table Entry를 캐시하는 하드웨어"**이다.

**TLB의 특징**:
- **Associative Memory**: 병렬 탐색
- **SRAM**: 빠른 접근
- **Limited Size**: 64~4096 entries

### 💡 비유
TLB는 **"자주 쓰는 번호 단축키"**와 같다.
- **전화번호부**: Page Table (느림)
- **즐겨찾기**: TLB (빠름)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB의 필요성                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Paging (No TLB):
    • 매번 Page Table 접근
    • 최소 2회 Memory Access (Page Table + Data)
         ↓
TLB (1960년대):
    • Page Table Entry 캐시
    • 1 Memory Access로 감소
         ↓
Multi-level TLB:
    • L1 TLB (소용량, 빠름)
    • L2 TLB (대용량, 느림)
```

## Ⅱ. 아키텍처 및 핵심 원리

### TLB 동작 원리

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB Lookup                                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU 가상 주소 요청                                                                     │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Virtual Address: 0x0040_1234                                                      │  │
    │  │  VPN: 0x00401 (Virtual Page Number)                                                │  │
    │  │  Offset: 0x234                                                                      │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                     │                                                   │
    │                                     ▼                                                   │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  TLB (Associative Memory)                                                          │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  [Parallel Search]                                                               │  │  │
    │  │  │  VPN=0x00401 ──→ [Entry 5] Match!                                                │  │  │
    │  │  │                                                                                  │  │  │
    │  │  │  Entry 5:                                                                         │  │  │
    │  │  │  ┌──────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  │  Valid │  Dirty │  Access │  VPN    │  PFN          │  ASID  │  Age    │  │  │  │
    │  │  │  │  (1)   │  (0)   │  (RWX)  │ 0x00401 │  0x0F2A       │  123   │  0     │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                  │  │
    │  │  [TLB Hit]                                                                         │  │
    │  │  → PFN = 0x0F2A                                                                  │  │
    │  │  → Physical Address = 0x0F2A234                                                   │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                     │ (1 Cycle!)                                      │
    │                                     ▼                                                   │
    │  Cache/Memory Access                                                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### TLB Miss 처리

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB Miss Handling                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  [TLB Miss]                                                                            │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  1. Hardware TLB Miss Exception 발생                                               │  │
    │  │  2. OS TLB Miss Handler 실행                                                        │  │
    │  │  3. Page Table Walk (4-level Page Tree)                                           │  │
    │  │     ┌────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │     │  PML4 → PDP → PD → PT                                                       │  │
    │  │     │  각 Level마다 Memory Access                                                    │  │
    │  │     │  총 4회 Memory Access                                                          │  │
    │  │     └────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │  4. PTE 발견 → TLB에 저장 (TLB Fill)                                                │  │
    │  │  5. Fault 발생 시 Page Fault Handler 호출                                           │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  [TLB Miss Penalty]                                                                     │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  • Page Table Walk: ~20-50 cycles                                                   │  │
    │  │  • Memory Access: ~100 cycles each                                                  │  │
    │  │  • Total: ~100-500 cycles                                                           │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Multi-level TLB

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Multi-level TLB                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  L1 TLB (Core-local, Split)                                                       │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  I-TLB: 64-128 entries (Instructions)                                          │  │  │
    │  │  │  D-TLB: 64-128 entries (Data)                                                   │  │  │
    │  │  │  • 1-2 cycles latency                                                            │  │  │
    │  │  │  • Fully Associative                                                            │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                     │ Miss                                            │
    │                                     ▼                                                 │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  L2 TLB (Unified)                                                                   │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  512-4096 entries (Instructions + Data)                                       │  │  │
    │  │  │  • ~10 cycles latency                                                           │  │
    │  │  │  • 4-way or 8-way Set Associative                                              │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                     │ Miss                                            │
    │                                     ▼                                                 │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Page Table Walk (Memory)                                                           │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### TLB vs Cache

| 구분 | TLB | Cache |
|------|-----|-------|
| **저장** | PTE (주소 매핑) | Data/Instructions |
| **용도** | 주소 변환 | 데이터 접근 |
| **크기** | 작음 (KB) | 큼 (MB) |
| **Hit Rate** | 95~99% | 90~95% |

### TLB Flush

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB Flush (Invalidation)                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Flush 필요 상황]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Context Switch (Process Switch)                                                    │
    │     • 각 Process는 독립적인 Address Space                                               │
    │     • Old TLB entries는 무효화                                                           │
    │     │  → ASID(Address Space ID)로 완화 가능                                              │
    │  │                                                                                      │
    │  2. Page Table Update                                                                  │
    │     • Page Protection 변경 (R/W → R/O)                                                  │
    │     │  → 해당 Entry만 Invalidate (Selective Flush)                                      │
    │  │                                                                                      │
    │  3. Page Unmap                                                                          │
    │     • Page 해제, 물리 메모리 반환                                                        │
    │     │  → Global Flush (모든 Entries)                                                    │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [ASID (Address Space ID)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Process별로 고유 ID 부여                                                             │
    │  │  • TLB Entry에 ASID 저장                                                             │
    │  │  • Context Switch 시 Flush 불필요 (ASID로 구분)                                        │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  TLB Entry:                                                                         │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  VPN    │ PFN    │ Valid │ ASID  │                                          │  │  │
    │  │  │ 0x100  │ 0x234  │   1   │   5   │  ← Process 5                          │  │  │
    │  │  │ 0x100  │ 0x789  │   1   │   7   │  ← Process 7 (Same VPN!)                 │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: TLB 성능 분석
**상황**: TLB Miss로 인한 성능 저하
**판단**:

```bash
# Linux perf로 TLB miss 확인
$ perf stat -e dTLB-loads,dTLB-load-misses,iTLB-loads,iTLB-load-misses ls

# TLB miss 비율 계산
# dTLB Load Miss Rate = dTLB-load-misses / dTLB-loads

# /proc/vmstat으로 Page Fault 확인
$ cat /proc/vmstat | grep -i pgfault
# pgfault 1234567

# Huge Page로 TLB miss 감소
# 4KB Page → 512 TLB entries = 2MB
# 2MB Page → 1 TLB entry = 2MB
$ echo 100 > /proc/sys/vm/nr_hugepages
$ hugectl --heap mkdir /mnt/huge

# perf c2c로 Cache/TLB miss 시각화
$ perf record -e dTLB-load-misses ./app
$ perf c2c report
```

## Ⅴ. 기대효과 및 결론

### TLB 기대 효과

| 효과 | TLB 없음 | TLB 있음 |
|------|---------|---------|
| **주소 변환** | 100+ cycles | 1 cycle |
| **Hit Rate** | - | 95~99% |
| **메모리 접근** | 2회+ | 1회 |

### TLB 최적화 기법

1. **Huge Pages**: Entry 수 감소
2. **Page Coloring**: Cache 친화력
3. **TLB Prefetch**: 미리 로드
4. **ASID**: Flush 최소화
5. **Large Working Set**: L2 TLB 활용

### 미래 전망

1. ** larger TLB**: 더 많은 entries
2. **Nested Paging**: 가상화 지원
3. **Shared TLB**: Multi-core
