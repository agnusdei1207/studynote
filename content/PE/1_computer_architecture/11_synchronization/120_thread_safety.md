# 스레드 안전성 (Thread Safety)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스레드 안전성은 **"여러 스레드가 동시에 공유 자원에 접근할 때도 데이터 무결성과 일관성을 보장하는 속성"**으로, **경쟁 상태(Race Condition)**을 방지하고 **상호 배제(Mutual Exclusion)**, **동기화(Synchronization)**, **가시성(Visibility)**, **순서(Ordering)**을 보장한다.
> 2. **가치**: **락(Lock)**, **뮤텍스(Mutex)**, **세마포어(Semaphore)**, **모니터(Monitor)**, **RW Lock** 등의 동기화 프리미티브를 통해 공유 자원을 보호하며, **Lock-Free**, **Wait-Free** 알고리즘으로 **락 경합(Lock Contention)**을 최소화한다.
> 3. **융합**: **Java Memory Model(JMM)**, **C++ Memory Model**, **Python GIL** 등 언어별 **메모리 모델(Memory Model)**로 정의되며, **병렬 프로그래밍(Parallel Programming)**, **동시성 패턴(Concurrency Patterns)**, **Actor Model**의 핵심 개념이 된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스레드 안전성(Thread Safety)은 **"여러 스레드가 동시에 코드를 실행해도 올바른 결과를 보장하는 속성"**이다.

**스레드 안전성 4요소**:
1. **상호 배제(Mutual Exclusion)**: 한 번에 하나의 스레드만 접근
2. **가시성(Visibility)**: 한 스레드의 변경이 다른 스레드에 보임
3. **순서(Ordering)**: 명령어 실행 순서 보장
4. **원자성(Atomicity)**: 연산이 분할되지 않고 실행

### 💡 비유
스레드 안전성은 **"화장실 사용"**과 같다.
- **문 잠금(Mutex)**: 한 사람만 사용
- **대기열(Wait Queue)**: 기다리는 사람들
- **열쇠(Key)**: 잠금 해제 권한

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         스레드 안전성의 필요성                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

단일 스레드 (Single-threaded):
    • 순차적 실행

    • 성능 한계
         ↓
멀티스레드 (Multi-threaded):
    • 병렬 실행
    • 성능 향상
    • 하지만 경쟁 상태 발생
         ↓
동기화 메커니즘 (1970년대):
    • Dijkstra: 세마포어 제안
    • Hoare: 모니터 제안
    • Mutex, Condition Variable
         ↓
Lock-Free (2000년대):
    • CAS, LL/SC 기반
    • 비관성 락 경합 감소
    • 고성능 동시성 자료구조
```

### 경쟁 상태(Race Condition)과 해결

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         경쟁 상태 (Race Condition)                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [문제: 두 스레드가 공유 변수 counter 증가]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  초기: counter = 0                                                                    │
    │                                                                                         │
    │  Thread-1          Thread-2                                                            │
    │     │                  │                                                                │
    │     ▼                  ▼                                                                │
    │  READ counter(0)   READ counter(0)   ← 동시에 읽기                                     │
    │     │                  │                                                                │
    │     ▼                  ▼                                                                │
    │  temp = 0 + 1      temp = 0 + 1     ← 각각 계산                                       │
    │     │                  │                                                                │
    │     ▼                  ▼                                                                │
    │  WRITE counter(1)  WRITE counter(1)  ← 동시에 쓰기                                     │
    │     │                  │                                                                │
    │     ▼                  ▼                                                                │
    │  결과: counter = 1 (예상: 2)                                                          │
    │  → 문제: 한 번의 증가이 손실됨                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [해결: Mutex로 상호 배제]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  mutex.lock()                                                                          │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  counter = counter + 1    → 한 번에 하나의 스레드만 실행                            │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │  mutex.unlock()                                                                        │
    │                                                                                         │
    │  Thread-1          Thread-2                                                            │
    │     │                  │                                                                │
    │     ▼                  │                                                                │
    │  lock() 획득          │                                                                │
    │     │                  │                                                                │
    │     ▼                  ▼                                                                │
    │  counter++            │                                                                │
    │     │                  │                                                                │
    │     ▼                  ▼                                                                │
    │  unlock() 해제        lock() 획득 대기                                                  │
    │                         │                                                              │
    │                         ▼                                                              │
    │                   counter++                                                            │
    │                         │                                                              │
    │                         ▼                                                              │
    │                   unlock() 해제                                                        │
    │                                                                                         │
    │  결과: counter = 2 (예상 대로)                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 동기화 프리미티브 비교

| 프리미티브 | 목적 | 동시 소유자 | 성능 | 사용 사례 |
|-----------|------|-------------|------|----------|
| **Mutex** | 상호 배제 | 1개 | 중간 | 일반적 잠금 |
| **Semaphore** | 카운팅 | N개 | 중간 | 리소스 풀 |
| **RW Lock** | 읽기/쓰기 분리 | 다수 읽기, 1개 쓰기 | 높음 | 읽기 많은 작업 |
| **Monitor** | 고수준 추상화 | 1개 | 중간 | 객체 기반 |
| **Spinlock** | 바쁜 대기 | 1개 | 낮음(짧음) | 커널 |
| **Atomic** | Lock-free | - | 높음 | 단순 연산 |

## Ⅲ. 융합 비교 및 다각도 분석

### 동기화 전략 비교

| 전략 | 장점 | 단점 | 적용 |
|------|------|------|------|
| **Mutex** | 간단, 범용 | 데드락 가능 | 일반적 |
| **Atomic** | 락프리, 빠름 | 복잡함 | 단순 연산 |
| **Lock-Free** | 높은 동시성 | 구현 어려움 | 고성능 |
| **Copy-on-Write** | 읽기 빠름 | 쓰기 느림 | 읽기 많음 |

### 과목 융합 관점 분석

#### 1. 운영체제 ↔ 스레드 안전성
- **커널 동기화**: spinlock, mutex, semaphore
- **유저 동기화**: pthread, std::mutex
- **futex**: userspace + kernel 하이브리드

#### 2. 언어별 메모리 모델
- **Java**: JMM, volatile, synchronized
- **C++**: C++11 Memory Model, atomic
- **Python**: GIL (Global Interpreter Lock)

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 높은 동시성 카운터
**상황**: 1M TPS 카운터
**판단**:
1. **Mutex**: 성능 저하
2. **Atomic**: 카운터 누락 가능
3. **분산 카운터**: Sharded Counter

```python
# Sharded Counter (분산 카운터)
class ShardedCounter:
    def __init__(self, num_shards=16):
        self.shards = [AtomicInt() for _ in range(num_shards)]
        self.num_shards = num_shards

    def increment(self):
        shard_id = thread_id() % self.num_shards
        self.shards[shard_id].increment()

    def get_count(self):
        return sum(shard.get() for shard in self.shards)
```

## Ⅴ. 기대효과 및 결론

### 스레드 안전성 기대 효과

| 기법 | 효과 | 부작용 |
