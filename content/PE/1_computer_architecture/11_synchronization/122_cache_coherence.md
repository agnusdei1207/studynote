# 캐시 일관성 (Cache Coherence)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시 일관성은 **"여러 **코어**가 **같은 **메모리 **블록**을 **캐시**할 때 **데이터 **일관성**을 **보장**하는 **메커니즘"**으로, **Write-Invalidate**(무효화)와 **Write-Update**(갱신) **프로토콜**로 **구현**된다.
> 2. **프로토콜**: **Bus Snooping**(버스 **감시**)과 **Directory-Based**(디렉토리 **관리**)로 **구분**하며 **MESI**(Modified, Exclusive, Shared, Invalid) **프로토콜**이 **가장 **널리 **사용**된다.
> 3. **NUMA**: **Non-Uniform Memory Access**에서는 **Directory-Based** **일관성**이 **필수**이며 **scalability**를 **위해 **분산 **디렉토리**를 **사용**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
캐시 일관성은 **"다중 코어 캐시 동기화"**이다.

**일관성 문제**:
- **Stale Data**: 오래된 데이터 읽기
- **Write Propagation**: 쓰기 전파
- **Transaction Serialization**: 순서 보장

### 💡 비유
캐시 일관성은 ****공유 **문서 ****동기화****와 같다.
- **문서**: 공유 데이터
- **사본**: 각 코어의 캐시
- **동기화**: 일관성 프로토콜

## Ⅱ. 아키텍처 및 핵심 원리

### 캐시 일관성 문제

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Cache Coherence Problem                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Main Memory: X = 10                                                                   │  │
    │                                                                                         │  │
    │  Core 1 (Cache L1)               Core 2 (Cache L1)                                      │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  X = 10                          X = 10                                             │  │  │
    │  │  (Cache Hit)                    (Cache Hit)                                         │  │  │
    │  │                                                                                       │  │  │
    │  │  T1: Core 1 writes X = 20       │                                                   │  │  │
    │  │  → Only Core 1 cache updated    │  Core 2 still has X = 10                          │  │  │
    │  │  → INCOHERENT!                  │  → STALE DATA!                                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Write-Invalidate Protocol로 해결                                                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Bus Snooping Protocol

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Bus Snooping Protocol                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Shared Bus                                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Core 1              Core 2              Core 3              Memory                 │  │  │
    │  │  ┌────────┐        ┌────────┐        ┌────────┐        ┌────────┐                 │  │  │
    │  │  │ Cache  │        │ Cache  │        │ Cache  │        │ Memory │                 │  │  │
    │  │  │ X=10   │        │ X=10   │        │ X=10   │        │  X=10  │                 │  │  │
    │  │  └───┬────┘        └───┬────┘        └───┬────┘        └───┬────┘                 │  │  │
    │  │      │                │                │                │                         │  │  │
    │  │      └────────────────┴────────────────┴────────────────┘                         │  │  │
    │  │                           ▼                                                          │  │  │
    │  │                      ┌─────────┐                                                       │  │  │
    │  │                      │   Bus   │  ← All cores monitor bus transactions               │  │  │
    │  │                      │ Snooping│                                                       │  │  │
    │  │                      └─────────┘                                                       │  │  │
    │  │                                                                                       │  │  │
    │  │  T1: Core 1 writes X = 20                                                             │  │  │
    │  │  → BusRdX (Bus Read Exclusive) on bus                                                 │  │  │
    │  │  → Core 2 snoops: Invalidate X                                                        │  │  │
    │  │  → Core 3 snoops: Invalidate X                                                        │  │  │
    │  │  → Memory: Update X = 20 (Write-Through)                                             │  │  │
    │  │                                                                                       │  │  │
    │  │  Result:                                                                               │  │  │
    │  │  ┌────────┐        ┌────────┐        ┌────────┐                                      │  │  │
    │  │  │ X=20   │        │   I    │        │   I    │                                      │  │  │
    │  │  │ (M)    │        │(Invalid)│        │(Invalid)│                                     │  │  │
    │  │  └────────┘        └────────┘        └────────┘                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### MESI Protocol States

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         MESI State Machine                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  State         │  Other Caches  │  Memory  │  Description                              │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────│  │
    │  Modified (M)  │  No            │  Stale   │  Only this cache has valid copy            │  │
    │  Exclusive (E)│  No            │  Clean   │  Only this cache, memory matches           │  │
    │  Shared (S)   │  Yes           │  Clean   │  Multiple caches may have copy              │  │
    │  Invalid (I)  │  -             │  -       │  Block not in cache                        │  │
    │                                                                                         │  │
    │  State Transitions:                                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  I ──BusRd──→ S (Read from memory, other caches may have copy)                        │  │  │
    │  │  I ──BusRdX──→ M (Read exclusive for write, invalidate others)                        │  │  │
    │  │  S ──BusRdX──→ M (Upgrade to modified, invalidate others)                             │  │  │
    │  │  M ──Flush──→ S (Write back to memory, other caches read)                             │  │  │
    │  │  M/S ──BusRdX──→ I (Other cache writes, invalidate this)                              │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Write-Invalidate vs Write-Update

| 구분 | Write-Invalidate | Write-Update |
|------|------------------|--------------|
| **Write 시** | 무효화 신호 | 데이터 브로드캐스트 |
| **Read 후** | 메모리/캐시 재로드 | 즉시 업데이트 |
| **대역폭** | 적음 | 많음 |
| **구현** | 간단 | 복잡 |
| **대표** | MESI | Firefly |

### Bus Snooping vs Directory-Based

| 구분 | Bus Snooping | Directory-Based |
|------|--------------|-----------------|
| **확장성** | 낮음 (≤16 코어) | 높음 (수백 코어) |
| **지연** | 낮음 | 중간 |
| **복잡도** | 낮음 | 높음 |
| **사용** | SMP | NUMA, CC-NUMA |

### Directory-Based Protocol

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Directory-Based Coherence                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Directory Entry (Memory Block X)                                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  State: Shared                                                                       │  │  │
    │  │  Sharers: {C1, C2, C3}   → Bit vector: 11100000                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Core 1 Write Request:                                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Core 1 → Directory: Write request for X                                          │  │  │
    │  │  2. Directory → Core 2, Core 3: Invalidate X                                        │  │  │
    │  │  3. Core 2, Core 3 → Directory: Ack                                                  │  │  │
    │  │  4. Directory → Core 1: Grant exclusive                                              │  │  │
    │  │  5. Directory: State = Exclusive, Sharers = {C1}                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Point-to-point messages (no broadcast)                                                │  │
    │  → Scalable to hundreds of cores                                                         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### NUMA (Non-Uniform Memory Access)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         NUMA Architecture                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Node 0                           Node 1                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  ┌───────┐  ┌───────┐                   ┌───────┐  ┌───────┐                         │  │  │
    │  │  │ Core 0│  │ Core 1│                   │ Core 2│  │ Core 3│                         │  │  │
    │  │  └───┬───┘  └───┬───┘                   └───┬───┘  └───┬───┘                         │  │  │
    │  │      │          │                            │          │                            │  │  │
    │  │      └────┬─────┘                            └────┬─────┘                            │  │  │
    │  │           │                                       │                                 │  │  │
    │  │  ┌────────┴────────┐                     ┌────────┴────────┐                         │  │  │
    │  │  │   L3 Cache      │                     │   L3 Cache      │                         │  │  │
    │  │  └────────┬────────┘                     └────────┬────────┘                         │  │  │
    │  │           │                                       │                                 │  │  │
    │  │  ┌────────┴────────┐                     ┌────────┴────────┐                         │  │  │
    │  │  │   Memory        │                     │   Memory        │                         │  │  │
    │  │  │   (Local)       │                     │   (Local)       │                         │  │  │
    │  │  │   Access: ~50ns │                     │   Access: ~50ns │                         │  │  │
    │  │  └────────┬────────┘                     └────────┬────────┘                         │  │  │
    │  │           │                                       │                                 │  │  │
    │  │           └────────────────┬──────────────────────┘                                 │  │  │
    │  │                            │                                                        │  │  │
    │  │                    ┌───────┴────────┐                                               │  │  │
    │  │                    │  Interconnect  │  ← Remote access: ~100ns                      │  │  │
    │  │                    │  (QPI, UPI)    │                                               │  │  │
    │  │                    └────────────────┘                                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Directory-based coherence for scalability                                            │  │
    │  → First touch policy for memory allocation                                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: NUMA 최적화
**상황**: 다중 소켓 서버
**판단**: 로컬 메모리 액세스 최적화

```c
// NUMA-aware memory allocation (libnuma)
#include <numa.h>

// Core 0에서 할당 (Node 0)
numa_set_preferred(0);
void *local_mem = numa_alloc_local(1024);

// Thread를 로컬 노드에 바인딩
numa_run_on_node(0);

// → Cross-node 트래픽 최소화
```

## Ⅴ. 기대효과 및 결론

### 캐시 일관성 기대 효과

| 효과 | 없음 | Coherence |
|------|------|-----------|
| **정확성** | 오류 | 보장 |
| **성능** | 빠름(잘못됨) | 오버헤드 |
| **확장성** | 불가 | 가능 |

### 모범 사례

1. **SMP**: Bus Snooping
2. **NUMA**: Directory-Based
3. **최적화**: False Sharing 방지 (Padding)

### 미래 전망

1. **Hybrid**: Snooping + Directory
2. **Hardware Transaction**: TM
3. **Non-Coherent**: GPU-style

### ※ 참고 표준/가이드
- **MESI**: Illinois Protocol
- **MOESI**: AMD Extension
- **IEEE**: Cache Coherency
