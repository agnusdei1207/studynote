# 인터커넥트 (Interconnects)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터커넥트는 **"프로세서 **코어**, **메모리**, **I/O**를 **연결**하는 **데이터 **경로"**로, **Intel QPI**(QuickPath Interconnect), **Intel UPI**(Ultra Path Interconnect), **AMD HyperTransport**, **AMD InfinityFabric**가 **대표적**이다.
> 2. **구조**: **Point-to-Point** **토폴로지**로 **버스 **병목**을 **제거**하고 **Packet-Switching**으로 **데이터**를 **전송**하며 **NUMA** **아키텍처**에서 **노드 **간 **통신**을 **담당**한다.
> 3. **진화**: **FSB**(Front Side Bus) → **QPI** → **UPI**로 **대역폭**이 **증가**하고 **지연**이 **감소**했으며 **CXL**(Compute Express Link)로 **CPU-GPU** **메모리 **공유**가 **가능**해졌다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
인터커넥트는 **"칩 간 데이터 버스"**이다.

**진화 역사**:
| 세대 | 인터커넥트 | 구조 | 대역폭 |
|------|-----------|------|--------|
| **1세대** | FSB | Shared Bus | ~10 GB/s |
| **2세대** | QPI | Point-to-Point | ~25 GB/s |
| **3세대** | UPI | Mesh | ~40 GB/s |
| **4세대** | CXL | Cache Coherent | ~64 GB/s |

### 💡 비유
인터커넥트는 ****고속도로 ****연결 ****과 같다.
- **도시**: 노드 (CPU + Memory)
- **고속도로**: 인터커넥트
- **차량**: 데이터 패킷

## Ⅲ. 융합 비교 및 다각도 분석

### 인터커넥트 비교

| 구분 | QPI | UPI | HyperTransport | InfinityFabric |
|------|-----|-----|----------------|----------------|
| **개발사** | Intel | Intel | AMD | AMD |
| **대역폭** | ~25 GB/s | ~40 GB/s | ~22 GB/s | ~25 GB/s |
| **토폴로지** | Ring | Mesh | HyperTransport | Mesh |
| **지연** | ~40ns | ~30ns | ~50ns | ~35ns |

### CXL (Compute Express Link)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         CXL Architecture                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CXL Protocol (3 protocols over PCIe 5.0 physical layer)                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  CXL.io (I/O):  PCIe-like, non-coherent, configuration                              │  │  │
    │  │  CXL.cache: Device can cache CPU memory (coherent)                                   │  │  │
    │  │  CXL.mem: CPU can access device memory (coherent)                                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Use Cases:                                                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  CPU ──CXL──→ GPU/ accelerator (memory pooling)                                      │  │  │
    │  │  CPU ──CXL──→ Memory expander (capacity scaling)                                      │  │  │
    │  │  CPU ──CXL──→ Smart NIC (coherent access)                                             │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Cache coherency between CPU and device                                                │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 인터커넥트 성능

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Interconnect Bandwidth Calculation                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Intel UPI (2nd Generation Xeon Scalable):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Data rate: 10.4 GT/s                                                                 │  │
    │  • Lanes: 20                                                                            │  │
    │  • Encoding: 8b/10b (80% efficiency)                                                     │  │
    │  • Bandwidth = 10.4 × 20 × 0.8 / 8 = 20.8 GB/s per direction                            │  │
    │  • Full duplex = 20.8 × 2 = 41.6 GB/s                                                  │  │
    │  • 3 UPI links per processor = 124.8 GB/s total                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    AMD InfinityFabric (Zen 3):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Data rate: ~12.5 GT/s                                                                │  │
    │  • Lanes: 32 (2 × 16)                                                                   │  │
    │  • Bandwidth = 12.5 × 32 / 8 = 50 GB/s per link                                        │  │
    │  • 4 InfinityFabric links = 200 GB/s total                                             │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 기술사적 판단 (실무 시나리오)

## Ⅴ. 기대효과 및 결론

### 인터커넥트 기대 효과

| 효과 | FSB | Point-to-Point |
|------|-----|----------------|
| **대역폭** | 공유 | 전용 |
| **확장성** | 8코어 | 28+코어 |

1. **NUMA 인식**: 로컬 메모리 액세스 최적화
2. **코어 바인딩**: 인터커넥트 트래픽 감소
3. **확장성**: Mesh 토폴로지 선택

### 미래 전망

1. **CXL 3.0**: 메모리 풀링
2. **Chiplet**: 모듈러 디자인
3. **Photonic**: 광 인터커넥트
