# 10. 병렬 처리 아키텍처 (Parallel)

## ## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하나의 거대한 문제를 여러 개의 하위 문제로 분할하여 다수의 연산 장치가 동시에 처리하는 성능 극대화 전략.
> 2. **가치**: 명령어 수준(ILP), 데이터 수준(DLP), 스레드 수준(TLP)의 병렬성을 활용하여 무어의 법칙 한계를 아키텍처적으로 극복.
> 3. **융합**: 멀티코어(Multi-core) 및 매니코어(Many-core, GPU) 구조를 통해 대규모 데이터 분석 및 고성능 연산(HPC) 지원.

---

### Ⅰ. 개요 (Context & Background)
병렬 처리는 현대 컴퓨터 성능 향상의 유일한 탈출구다. 단일 코어의 클럭 속도를 높이는 방식이 전력과 발열 문제로 한계에 부딪히면서, "더 빨리" 보다는 "더 많이 동시에" 일하는 방식으로 패러다임이 완전히 전환되었다.

---

### Ⅱ. 아키텍처 및 핵심 원리

#### 1. Flynn의 분류 (Flynn's Taxonomy)
- **SISD**: 단일 명령, 단일 데이터 (고전적 폰 노이만)
- **SIMD**: 단일 명령, 다중 데이터 (벡터 연산, GPU)
- **MISD**: 다중 명령, 단일 데이터 (특수 목적, 오류 검증)
- **MIMD**: 다중 명령, 다중 데이터 (멀티코어 시스템)

#### 2. 병렬 아키텍처 모델 (ASCII)
```text
    [ Parallel Processing Models ]
    
    (SIMD - GPU Style)          (MIMD - Multi-core Style)
    +-------------------+       +---------+   +---------+
    |   Instruction     |       | Core 1  |   | Core 2  |
    +---------+---------+       +----+----+   +----+----+
              |                      |             |
    +----+----+----+----+       +----v-------------v----+
    | D1 | D2 | D3 | D4 |       |    Shared Memory      |
    +----+----+----+----+       +-----------------------+
```

---

### Ⅲ. 융합 비교 및 다각도 분석

#### 공유 메모리 vs 분산 메모리
| 구분 | 공유 메모리 (Shared) | 분산 메모리 (Distributed) |
| :--- | :--- | :--- |
| **통신 방식** | 변수 공유 (Load/Store) | 메시지 패싱 (MPI) |
| **프로그래밍** | 비교적 쉬움 | 어려움 (복잡한 매핑) |
| **확장성** | 낮음 (버스 병목) | 높음 (수천 대 노드 가능) |
| **일관성** | 하드웨어가 관리 (MESI) | 소프트웨어가 관리 |

---

### Ⅳ. 실무 적용 및 기술사적 판단
실무적으로 병렬화의 효율은 **Load Balancing**에 달려 있다. 특정 코어만 과부하가 걸리는 상황을 방지하기 위해 데이터를 균등하게 분할하고, 동기화 오버헤드(Lock Contention)를 최소화하는 소프트웨어-하드웨어 최적화가 필수적이다.

---

### Ⅴ. 기대효과 및 결론
병렬 처리는 인공지능 학습과 대규모 시뮬레이션의 핵심 엔진이다. 향후 수만 개의 코어를 하나로 묶는 가속기 클러스터와 이를 연결하는 고속 패브릭 기술이 국가 경쟁력을 결정할 것이다.
