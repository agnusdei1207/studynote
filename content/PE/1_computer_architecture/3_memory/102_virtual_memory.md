# 가상 메모리 (Virtual Memory)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 메모리는 **"물리 메모리(RAM)보다 큰 **논리적 주소 공간(Virtual Address Space)**을 제공하는 **메모리 관리 기법"**으로, **Demand Paging(요구 페이징)**으로 필요한 페이지만 메모리에 로드하고 **Swap(스왑)** 영역으로 **RAM 부족을 해결**한다.
> 2. **가치**: **Multiprogramming(다중 프로그래밍)**을 지원하여 **메모리 사용량 초과** 프로그램도 실행 가능하게 하며 **Process Isolation(프로세스 격리)**를 통해 **메모리 보호**를 제공하고 **Physical Memory 추상화**로 프로그래머의 부담을 줄인다.
> 3. **융합**: **MMU(Memory Management Unit)**가 **가상 주소 → 물리 주소 변환**을 수행하며, **TLB(Translation Lookaside Buffer)**는 **페이지 테이블 캐시**로 변환 속도를 높이고, **Page Replacement Algorithm**(LRU, LFU)는 **Page Fault**를 최소화한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
가상 메모리는 **"물리 메모리 크기 제약을 없애고, 각 프로세스에 독립적인 주소 공간을 제공하는 기술"**이다.

**가상 메모리의 핵심**:
- **논리 주소(Logical Address)**: 프로세스가 사용하는 주소
- **물리 주소(Physical Address)**: 실제 메모리 주소
- **주소 변환(Address Translation)**: 논리 → 물리 매핑
- **페이지(Page)**: 고정 크기 메모리 블록

### 💡 비유
가상 메모리는 **"가상 서재"**와 같다.
- **책상(RAM)**: 자주 참고하는 책
- **서가(디스크)**: 덜 사용하는 책
- **사서(MMU)**: 책 가져오는 역할

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         가상 메모리의 필요성                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

물리 메모리 한계:
    • RAM 비싸고 용량 제한
    • 큰 프로그램 실행 불가
    • 다중 프로세스 메모리 충돌
         ↓
Overlay (1950년대):
    • 프로그래머가 수동으로 코드 분할
    • 복잡하고 오류 발생
         ↓
가상 메모리 등장 (1961년, Atlas):
    • 자동 메모리 관리
    • 페이징/세그먼테이션
         ↓
현대 OS:
    • Demand Paging
    • Copy-on-Write
    • Memory-mapped File
```

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐

    │  0xC0000000 ┤ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─    │

    │              │   User Space (3GB)                                                        │
    │              │   ┌────────────────────────────────────────────────────────────────────┐   │

    │              │   │  Data Segment (BSS, Data)                                           │   │
    │              │   ├────────────────────────────────────────────────────────────────────┤   │

    │                              │  │  Frame 2  │ Process A, VPN 3            │  │        │
    │  Process B의 가상 페이지 ──→  │  ├──────────────────────────────────────────┤  │        │

    │  VPN 0x00001 ───(not mapped)│  └──────────────────────────────────────────┘  │        │

    │                                                                                         │
    │  Swap Space (디스크):                                                                   │

### 페이징(Paging) 메커니즘

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         주소 변환 과정                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [페이지 테이블 구조]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Virtual Address: 0x00401030                                                            │
    │  ┌─────────────────────────┬─────────────────────────────────────────────────────────┐   │
    │  │  VPN (20비트): 0x00401  │  Offset (12비트): 0x030                            │   │
    │  └─────────────────────────┴─────────────────────────────────────────────────────────┘   │
    │                              │                                                          │
    │                              ▼                                                          │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Page Table (Base Register: 0x1000)                                                │  │
    │  │  ┌────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  Index  │  Valid  │  Protection  │  Frame Number  │  Dirty  │  Accessed     │  │  │
    │  │  │  ─────────────────────────────────────────────────────────────────────────────  │  │  │
    │  │  │  0x400  │  1      │  RW         │  0x3A5F        │  0      │  1            │  │  │
    │  │  │  0x401  │  1      │  RW         │  0x7B2C        │  1      │  1            │  │  │
    │  │  │  ...    │  ...    │  ...        │  ...           │  ...    │  ...          │  │  │
    │  │  │  0x401  │  1      │  R-X        │  0x1C8D        │  0      │  1            │  │  │
    │  │  └────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                              │                                                          │
    │                              ▼ (Page Table[0x401])                                     │
    │                         Frame Number: 0x1C8D                                             │
    │                              │                                                          │
    │                              ▼                                                          │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Physical Address = (Frame Number << 12) | Offset                                   │  │
    │  │                    = (0x1C8D << 12) | 0x030                                         │  │
    │  │                    = 0x1C8D030                                                      │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [TLB (Translation Lookaside Buffer)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  TLB: 페이지 테이블 캐시 (SRAM, 빠름)                                                    │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  VPN      │  Frame    │  Valid  │  ASID  │  Protection                             │  │
    │  │  ─────────────────────────────────────────────────────────────────────────────────  │  │
    │  │  0x00401  │  0x1C8D   │  1      │  100   │  R-X                                   │  │
    │  │  0x003F2  │  0x2A4B   │  1      │  100   │  RW-                                   │  │
    │  │  0x00500  │  0x0F1C   │  1      │  101   │  RW-                                   │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  TLB Hit: 1 클럭 (캐시 적중)                                                            │
    │  TLB Miss: ~10-100 클럭 (메모리 참조)                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Page Fault 처리

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Page Fault 처리 과정                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [1. Page Fault 발생]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. CPU가 가상 주소 접근                                                               │
    │  2. TLB Miss (캐시에 없음)                                                             │
    │  3. Page Table 조회 → Valid Bit = 0 (페이지 없음)                                       │
    │  4. Page Fault Exception 발생                                                           │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [2. OS의 Page Fault Handler]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  [Step 1] 페이지 확인                                                                   │
    │  • 가상 주소가 유효한지 확인                                                            │
    │  • Invalid Access → Segmentation Fault                                                  │
    │                                                                                         │
    │  [Step 2] 빈 프레임 확보                                                                │
    │  • Free Frame이 있으면 사용                                                             │
    │  • 없으면 Page Replacement Algorithm 실행                                               │
    │                                                                                         │
    │  [Step 3] 디스크에서 페이지 로드                                                         │
    │  • Disk I/O: Swap In                                                                    │
    │  • 디스크 주소: Swap Area 또는 Memory-mapped File                                        │
    │                                                                                         │
    │  [Step 4] 페이지 테이블 업데이트                                                        │
    │  • Valid Bit = 1                                                                        │
    │  • Frame Number 업데이트                                                               │
    │                                                                                         │
    │  [Step 5] 명령어 재실행                                                                 │
    │  • Fault 발생 명령어부터 다시 실행                                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Page Replacement Algorithms]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  FIFO (First-In, First-Out):                                                             │
    │  • 가장 오래된 페이지 교체                                                               │
    │  • Belady's Anomaly 발생 가능                                                            │
    │                                                                                         │
    │  LRU (Least Recently Used):                                                             │
    │  • 가장 오랫동안 사용 안 된 페이지 교체                                                   │
    │  • Reference Bit + Aging 또는 Counter 구현                                               │
    │                                                                                         │
    │  LFU (Least Frequently Used):                                                           │
    │  • 사용 횟수가 가장 적은 페이지 교제                                                      │
    │                                                                                         │
    │  Optimal (이론적):                                                                       │
    │  • 앞으로 가장 오랫동안 사용 안 될 페이지 교체                                            │
    │  • 실제 구현 불가 (미래 예지 필요)                                                        │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Paging vs Segmentation

| 구분 | Paging | Segmentation |
|------|--------|--------------|
| **단위** | 고정 크기 Page | 가변 크기 Segment |
| **외부 단편화** | 없음 | 있음 |
| **내부 단편화** | 있음 | 없음 |
| **주소 공간** | 선형 | 논리적 (Code, Data, Stack) |
| **주요 용도** | 현대 OS | historically (x86) |

### 과목 융합 관점 분석

#### 1. 운영체제 ↔ 가상 메모리
- **Process Isolation**: 각 프로세스 독립 주소 공간
- **Copy-on-Write**: fork() 메모리 효율
- **Memory-mapped File**: 파일 I/O 간소화

#### 2. 컴퓨터 구조 ↔ 가상 메모리
- **TLB**: 페이지 테이블 캐시
- **MMU**: 하드웨어 주소 변환
- **Cache Coherence**: 다중 프로세서 메모리

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 메모리 부족 문제
**상황**: Swap으로 인한 성능 저하
**판단**:

```bash
# Swap 사용 모니터링
$ vmstat 1
procs -----------memory---------- ---swap-- -----io---- -system-- ----cpu----
 r  b   swpd   free   buff  cache   si   so    bi    bo   in   cs us sy id wa
 1  0 2097120  16384 262144 5242880  0    0     0     0  100   50 10  5 85  0

# Swapping 발생 시 (si, so > 0)
# → 메모리 추가 또는 프로세스 종료 필요

# Page Fault 확인
$ ps -o min_flt,maj_flt,cmd
  FLTS  FLT CMD
  1234   56 /usr/bin/python
# min_flt: Minor Fault (메모리에서 해결)
# maj_flt: Major Fault (디스크 I/O 필요)

# 해결 방안:
# 1. 물리 메모리 추가
# 2. 스왑 파일/파티션 크기 조정
# 3. 메모리 hungry 프로세스 종료
# 4. OOM Killer 튜닝 (/proc/sys/vm/overcommit_memory)
```

## Ⅴ. 기대효과 및 결론

### 가상 메모리 기대 효과

| 효과 | 가상 메모리 없을 시 | 가상 메모리 있을 시 |
|------|------------------|------------------|
| **주소 공간** | RAM 크기 제한 | RAM + Swap 크기 |
| **프로세스 격리** | 불가능 | 가능 |
| **메모리 관리** | 수동 Overlay | 자동 Paging |
| **프로그래밍** | 복잡 | 단순 (추상화) |

### 미래 전망

1. **Non-Volatile RAM**: Swap 없는 메모리
2. **Huge Pages**: TLB 압력 감소
3. **Sparse Address Space**: 64비트 효율

### ※ 참고 표준/가이드
- **Intel SDM**: Volume 3A (Memory Management)
- **AMD64**: Page Tables
- **POSIX**: mmap, mprotect
