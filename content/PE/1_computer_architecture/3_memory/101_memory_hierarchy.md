# 메모리 계층 (Memory Hierarchy)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 계층은 **"속도, 용량, 가격의 Trade-off를 최적화하기 위해 레지스터, 캐시(Cache), 주기억장치(RAM), 보조기억장치(Storage)을 계층화한 구조"**로, **SRAM(정적 RAM)**은 **CPU 캐시**로, **DRAM(동적 RAM)**은 **주기억장치**로 사용된다.
> 2. **가치**: **Locality(지역성)** 원리(**Temporal Locality**와 **Spatial Locality**)를 통해 **캐시 적중률(Hit Ratio)**를 높이고 **메모리 액세스 시간(Memory Access Time)**을 최소화하며, **가상 메모리(Virtual Memory)**는 **물리 메모리보다 큰 주소 공간**을 제공한다.
> 3. **융합**: **CPU 캐시**(L1/L2/L3)는 **관계형 매핑**, **Set Associative**, **Write-Back/Write-Through** 정책을 사용하며, **TLB(Translation Lookaside Buffer)**는 **페이지 테이블 캐시**로 **Page Fault**를 감소시키고 **MMU**(Memory Management Unit)의 **주소 변환**을 가속화한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
메모리 계층은 **"속도, 용량, 가격의 상충 관계를 해결하기 위해 다양한 저장 장치를 계층화한 구조"**이다.

**계층의 필요성**:
- **CPU 레지스터**: 가장 빠름, 용량 작음
- **캐시(Cache)**: 빠름, 중간 용량
- **주기억장치(RAM)**: 중간 속도, 큰 용량
- **보조기억장치(SSD, HDD)**: 느림, 매우 큰 용량

### 💡 비유
메모리 계층은 **"도서관"**과 같다.
- **책상**: 자주 참고 (CPU Register)
- **옆장**: 1층 (L1 Cache)
- **서가**: 열람실 (L2/L3 Cache)
- **서고**: 본관 (RAM)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         메모리 계층의 필요성                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

단일 메모리 시스템:
    • 속도=용량=가격 (Trade-off 없음)
    • 모든 메모리가 동일 속도
         ↓
메모리 격차 (Memory Gap):
    • CPU 속도: 매년 50~60% 향상
    │  DRAM 속도: 매년 7% 향상
    • 격차 지속 확대
         ↓
계층 메모리 (1970년대~):
    • 캐시(Cache) 도입
    • 다중 계층 구조
    • Locality 활용
```

## Ⅱ. 아키텍처 및 핵심 원리

### 메모리 계층 구조

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         메모리 계층 다이어그램                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [피라미드 구조]

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU Registers                                                                           │
    │  • 용량: 수백 바이트 ~ 수 킬로바이트                                                        │
    │  │  속도: 1 클럭 사이클                                                                   │
    │  │  가격: 매우 비쌈                                                                    │
    │  │  • EAX, EBX, ECX, EDX, ESI, EDI, EBP, ESP                                               │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
                              │  Latency: ~1 cycle
                              ▼
    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  L1 Cache (SRAM)                                                                         │
    │  • 용량: 32KB ~ 64KB                                                                    │
    │  │  속도: 3~4 cycles                                                                     │
    │  │  가격: $/KB 단위                                                                    │
    │  │  • I-Cache (Instruction), D-Cache (Data)                                              │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
                              │  Latency: ~3-4 cycles
                              ▼
    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  L2 Cache (SRAM)                                                                         │
    │  • 용량: 128KB ~ 1MB                                                                    │
    │  │  속도: ~10 cycles                                                                    │
    │  │  가격: 중간                                                                          │
    │  │  • Unified Cache (명령+데이터)                                                      │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
                              │  Latency: ~10 cycles
                              ▼
    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  L3 Cache (SRAM/DRAM)                                                                     │
    │  • 용량: 8MB ~ 64MB (CPU 내장)                                                           │
    │  │  속도: ~40 cycles                                                                    │
    │  │  가격: 저렴                                                                        │
    │  │  • Shared Cache (코어 간 공유)                                                      │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
                              │  Latency: ~40 cycles
                              ▼
    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  Main Memory (DRAM)                                                                     │
    │  • 용량: 8GB ~ 64GB                                                                   │
    │  │  속도: ~100-200 ns (~100-200 cycles)                                                 │
    │  │  가격: $/GB 단위                                                                    │
    │  │  • Volatile (전원 소멸)                                                             │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
                              │  Latency: ~100-200 ns
                              ▼
    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  Secondary Storage (SSD/HDD)                                                            │
    │  • SSD: 100GB ~ 4TB                                                                    │
    │  │    속도: ~100-500 μs                                                                │
    │  │  HDD: 1TB ~ 20TB                                                                    │
    │  │    속도: ~10 ms (10,000,000 cycles)                                                   │
    │  │  가격: 가장 저렴                                                                    │
    │  │  • Non-volatile (전원 보존)                                                          │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
```

### Locality of Reference (참조의 지역성)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Locality 원리                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Temporal Locality (시간적 지역성)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 최근 접근한 데이터를 다시 접근할 가능성 높음                                                 │
    │                                                                                         │
    │  예시:                                                                                   │
    │  for (int i = 0; i < 100; i++) {                                                        │
    │      sum += array[i];  // array[i]를 반복 접근                                            │
    │  }                                                                                        │
    │                                                                                         │
    │  • Loop 내 변수: Temporal Locality 높음                                                 │
    │  │  → 캐시에 유리                                                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Spatial Locality (공간적 지역성)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 인접한 데이터를 함께 접근할 가능성 높음                                                   │
    │                                                                                         │
    │  예시:                                                                                   │
    │  for (int i = 0; i < 100; i++) {                                                        │
    │      sum += array[i];                                                                  │
    │  }                                                                                        │
    │                                                                                         │
    │  • array[i]와 array[i+1]은 인접                                                          │
    │  │  → Sequential access, Prefetching 효과적                                              │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [캐시 적중률(Hit Ratio) 계산]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Hit Ratio = (Cache Hit) / (Cache Hit + Cache Miss)                                       │
    │                                                                                         │
    │  예시:                                                                                   │
    │  • 총 접근: 10,000번                                                                  │
    │  │  Cache Hit: 9,000번                                                                 │
    │  │  Cache Miss: 1,000번                                                                │
    │  │  Hit Ratio = 9000 / 10000 = 90%                                                       │
    │                                                                                         │
    │  평균 액세스 시간 = (Hit Time × Hit) + (Miss Time × Miss)                                 │
    │                   = (3 cycles × 0.9) + (100 cycles × 0.1)                                   │
    │                   = 2.7 + 10 = 12.7 cycles                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 캐시 매핑 (Cache Mapping)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         캐시 매핑 방식                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Direct Mapped Cache (직접 매핑)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 메모리 블록이 캐시의 정해진 라인에만 매핑                                           │
    │                                                                                         │
    │  예: Block Size = 16B, Cache Size = 64B, 4 Lines                                          │
    │  메모리 블록 0 → Cache Line 0 (블록 번호 % 라인 수 = 0)                                     │
    │  메모리 블록 1 → Cache Line 1                                                                │
    │  메모리 블록 2 → Cache Line 2                                                                │
    │  메모리 블록 3 → Cache Line 3                                                                │
    │  메모리 블록 4 → Cache Line 0 (Conflict!)                                                 │
    │                                                                                         │
    │  장점:                                                                                   │
    │  • 구현 간단, Tag 비교 불필요                                                             │
    │  │  • 빠름                                                                             │
    │                                                                                         │
    │  단점:                                                                                   │
    │  • Conflict Miss 빈음                                                                   │
    │  │  • 낮은 유연성                                                                         │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Fully Associative Cache (완전 연관 매핑)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 메모리 블록이 캐시의 모든 라인에 매핑 가능                                             │
    │                                                                                         │
    │  예: Cache Line 4개                                                                     │
    │  메모리 블록 0 → Line 0, 1, 2, 3 (어디든지 가능)                                           │
    │  메모리 블록 1 → Line 0, 1, 2, 3                                                            │
    │  메모리 블록 2 → Line 0, 1, 2, 3                                                            │
    │  메모리 블록 3 → Line 0, 1, 2, 3                                                            │
    │  (가용한 Line 선택: LRU, LFU 등)                                                        │
    │                                                                                         │
    │  장점:                                                                                   │
    │  • Conflict Miss 최소                                                                   │
    │  │  • 유연성 높음                                                                       │
    │                                                                                         │
    │  단점:                                                                                   │
    │  • 구현 복잡                                                                             │
    │  │  • Tag 비교 필요 (모든 Line)                                                            │
    │  │  • 비용 큼음                                                                         │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Set Associative Cache (세트 연관 매핑)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 캐시를 여러 Set으로 분리, 각 Set은 완전 연관                                              │
    │                                                                                         │
    │  예: 4-Way Set Associative, 8 Sets                                                       │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Set 0: 4 Lines (Way 0, 1, 2, 3)                                                    │  │
    │  │  Set 1: 4 Lines (Way 0, 1, 2, 3)                                                    │  │
    │  │  ...                                                                                   │  │
    │  │  Set 7: 4 Lines (Way 0, 1, 2, 3)                                                    │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  메모리 블록 0 → Set(0), Way(0~3) 중 선택                                               │
    │  메모리 블록 8 → Set(0), Way(0~3) 중 선택                                               │
    │  메모리 블록 16 → Set(1), Way(0~3) 중 선택                                              │
    │                                                                                         │
    │  장점:                                                                                   │
    │  • Direct Mapped보다 낮은 Conflict                                                        │
    │  │  완전 연관보다 저렴                                                                   │
    │                                                                                         │
    │  단점:                                                                                   │
    │  • 구현 복잡도 중간                                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### SRAM vs DRAM

| 구분 | SRAM (Static RAM) | DRAM (Dynamic RAM) |
|------|-----------------|-------------------|
| **속도** | 빠름 (~1ns) | 느림 (~50ns) |
| **용량** | 작음 (KB~MB) | 큼음 (GB~TB) |
| **가격** | 비쌈 | 저렴 |
| **유지성** | Flip-flop (6개 트랜지스터/비트) | Capacitor (1개 트랜지스터/비트) |
| **주 사용** | CPU 캐시 | 주기억장치 |

### 과목 융합 관점 분석

#### 1. 운영체제 ↔ 메모리 계층
- **Virtual Memory**: 디스크를 RAM처럼
- **Page Replacement**: LRU, LFU
- **TLB**: 페이지 테이블 캐시

#### 2 컴파일러 ↔ 메모리 계층
- **Register Allocation**: 레지스터 할당
- **Loop Optimization**: Locality 최대화

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 캐시 최적화
**상황**: Cache Miss가 빈번
**판단**:
1. **원인 파악**:
   - Working Set이 캐시보다 큼
   - Sequential access 부족
2. **최적화**:
   - 데이터 구조 변경 (Arrays)
   - Loop Tiling/Blocking
   - Prefetching

```c
// Loop Tiling 예시
for (int i = 0; i < N; i += TILE_SIZE) {
    for (int j = 0; j < N; j += TILE_SIZE) {
        // TILE_SIZE × TILE_SIZE 블록 처리
        // Spatial Locality 증가
    }
}
```
