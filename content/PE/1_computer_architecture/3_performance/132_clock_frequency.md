# 클럭 주파수 (Clock Frequency)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클럭 주파수는 **"CPU의 **동작 **속도**를 **결정**하는 **기준 **신호**"**로, **Hz**(헤르츠) **단위**로 **측정**하며 **1초**당 **발생**하는 **클럭 **사이클**수**를 **의미**한다.
> 2. **성능**: **클럭 **주파수**가 **높을수록** **명령어** **실행 **속도**가 **빨라지지만 **전력 **소모**와 **발열**이 **증가**하며 **DIM**(Dark Silicon) **문제**가 **발생**한다.
> 3. **한계**: **단순 **클럭 **증가**만으로는 **성능 **향상**에 **한계**가 **있어 **파이프라이닝**, **비순차 **실행**, **멀티코어** 등 **아키텍처**적 **최적화**가 **필수**이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
클럭 주파수는 **"CPU 심박수"**이다.

**클럭 신호 특성**:
| 속성 | 설명 | 단위 |
|------|------|------|
| **주파수** | 1초당 클럭 사이클 수 | Hz (MHz, GHz) |
| **주기** | 한 클럭 사이클 시간 | ns (나노초) |
| **듀티 사이클** | ON 시간 비율 | % |

### 💡 비유
클럭 주파수는 **오케스트라 지휘자의 박자**와 같다.
- **클럭**: 지휘봉
- **사이클**: 박자
- **CPU**: 오케스트라 단원

### 성능 방정식

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Performance Equations                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    1. Basic Performance:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Time = Instructions × CPI × Clock Cycle Time                                          │  │
    │  Time = Instructions × CPI / Clock Frequency                                           │  │
    │                                                                                         │  │
    │  Performance = 1 / Time = Clock Frequency / (Instructions × CPI)                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. MIPS (Million Instructions Per Second):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  MIPS = Instruction Count / (Execution Time × 10^6)                                    │  │
    │  MIPS = Clock Frequency (MHz) / CPI                                                    │  │
    │                                                                                         │  │
    │  Example:                                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  CPU: 3 GHz (3000 MHz), CPI: 2.0                                                   │  │  │
    │  │  MIPS = 3000 / 2.0 = 1500 MIPS                                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. Power Consumption:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  P = C × V² × f                                                                          │  │
    │  - C: Capacitance (회로 정전용량)                                                       │  │
    │  - V: Voltage (전압)                                                                     │  │
    │  - f: Clock Frequency (클럭 주파수)                                                      │  │
    │                                                                                         │  │
    │  Key Insight: 전력은 주파수에 비례, 전압 제곱에 비례                                      │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 클럭 스키링 (Clock Skew)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Clock Skew and Jitter                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Clock Skew (클럭 왜곡):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Problem: Same clock edge arrives at different times at different flops                   │  │
    │                                                                                         │  │
    │  Source Flop (Clock @ 0ns)              Destination Flop (Clock @ 0.1ns skew)           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Q changes at t1              │  Data arrives before expected → Setup violation?    │  │  │
    │  │  Clock skew reduces time budget   │                                                │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Solution:                                                                             │  │
    │  • Global Clock Network (H-tree) - minimize skew                                        │  │
    │  • Deskew PLL - dynamically adjust delay                                                 │  │
    │  • Clock Grid - uniform distribution                                                    │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Clock Jitter (클럭 지터):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Problem: Clock edge timing varies cycle-to-cycle (noise, supply voltage)                │  │
    │                                                                                         │  │
    │  Solution:                                                                             │  │
    │  • Low-jitter PLL                                                                       │  │
    │  • Dedicated power supply for clock generator                                           │  │
    │  • Shielded clock lines                                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### DVFS (Dynamic Voltage and Frequency Scaling)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         DVFS Operating Points                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Intel Turbo Boost States (Example):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  P-State │  Frequency  │  Voltage  │  Power    │  Use Case                         │  │
    │  ─────────────────────────────────────────────────────────────────────────────────────│  │
    │  P0      │  5.0 GHz    │  1.3V     │  125W     │  Single-thread burst              │  │  │
    │  P1      │  4.5 GHz    │  1.2V     │  100W     │  Multi-thread boost               │  │  │
    │  P2      │  4.0 GHz    │  1.1V     │  80W      │  Sustained workload               │  │  │
    │  P3      │  3.5 GHz    │  1.0V     │  60W      │  Normal operation                 │  │  │
    │  P4      │  2.0 GHz    │  0.8V     │  30W      │  Power saving                     │  │  │
    │  P5      │  800 MHz    │  0.6V     │  10W      │  Idle                             │  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    DVFS Transition:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  High Load → Increase Frequency/Voltage (Instant)                                       │  │
    │  Low Load → Decrease Frequency/Voltage (Delay for stability)                            │  │
    │  → Trade-off: Responsiveness vs Power Efficiency                                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 고성능 서버 CPU 선택
**상황**: 데이터베이스 서버
**판단**: 단일 코어 성능 vs 코어 수

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         CPU Selection Decision Matrix                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Database Workload Analysis:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • OLTP: Many concurrent transactions → More cores, moderate frequency                  │  │
    │  • OLAP: Complex queries per transaction → Higher frequency, fewer cores                 │  │
    │                                                                                         │  │
    │  Example:                                                                               │  │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  CPU A: 32 cores @ 2.5 GHz = 80 GHz total                                               │  │  │  CPU B: 16 cores @ 4.0 GHz = 64 GHz total                                               │  │  │
    │  → OLTP: Better (handles 32 concurrent queries)                                          │  │  │  → OLAP: Better (faster single query)                                                 │  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

1. **Heterogeneous**: big.LITTLE
2. **Turbo**: 임시 부스트

1. **3D 스태킹**: 단락 경로

3. **근접 연산**: 메모리 내
4. **양자 컴퓨팅**: 새로운 패러다임
