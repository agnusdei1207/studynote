# ILP (Instruction Level Parallelism)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ILP는 **"명령어 **수준의 **병렬성**으로 **동시에 **여러 **명령어**를 **실행**하는 **기술"**으로, **Pipeline**(명령어 파이프라인), **Superscalar**(다중 실행 유닛), **VLIW**(컴파일러 스케줄링), **Out-of-Order**(순서 변경)로 **달성**된다.
> 2. **제약 요소**: **True Dependency**(RAW, 데이터 의존성), **Control Dependency**(분기), **Resource Dependency**(Structural Hazard)가 **ILP**을 **제한**하며 **Dependency Analysis**, **Register Renaming**, **Speculation**으로 **극복**한다.
> 3. **융합**: **Amdahl's Law**로 **병렬화 **효과**를 **계산**하고 **Speedup** = 1 / ((1-P) + P/S) **공식**으로 **이론적 **성능 **향상**을 **예측**하며 **Modern CPU**는 **IPC**(Instructions Per Cycle) > 1을 **달성**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
ILP는 **"명령어 병렬 처리"**이다.

**ILP 기법**:
- **Pipelining**: 오버랩 실행
- **Superscalar**: 다중 명령어
- **VLIW**: 컴파일러 그룹화
- **OoO**: 동적 스케줄링

### 💡 비유
ILP는 **"여러 요리사가 동시 조리****와 같다.
- **단일 요리사**: In-Order
- **팀 요리**: Superscalar
- **레시피**: VLIW

## Ⅱ. 아키텍처 및 핵심 원리

### ILP 기법 비교

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         ILP Techniques                                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Pipelining (5-stage):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Inst1: IF → ID → EX → MEM → WB                                                       │  │
    │  Inst2:    IF → ID → EX → MEM → WB                                                   │  │
    │  Inst3:       IF → ID → EX → MEM → WB                                               │  │
    │  → CPI = 1 (but limited by hazards)                                                    │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Superscalar (Multiple Issue):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Clock 1:  IF    IF    IF    (3 instructions fetched)                                │  │
    │  Clock 2:  ID    ID    ID    ID   (4 decoded)                                       │  │
    │  Clock 3:  EX    EX    EX    (3 executing)                                          │  │
    │  → IPC > 1                                                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    VLIW (Compiler Scheduled):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Long Instruction Word: [ADD] [SUB] [MUL] [LOAD]                                    │  │
    │  → All issue together in single cycle                                                │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Out-of-Order (Dynamic Scheduling):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Instructions executed when ready (independent)                                      │  │
    │  → Maximizes IPC but commits in-order                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Amdahl's Law

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Amdahl's Law                                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Speedup = 1 / ((1 - P) + P / S)                                                        │  │
    │                                                                                         │  │
    │  where:                                                                                  │  │
    │    P = 병렬화 가능 부분의 비율                                                            │  │
    │    S = 병렬화 노드 수 (속도향)                                                           │  │
    │                                                                                         │  │
    │  Example: 50% 병렬화 가능 (P=0.5), 4코어 (S=4)                                             │  │
    │    Speedup = 1 / (0.5 + 0.5/4) = 1 / 0.625 = 1.6x                                    │  │
    │                                                                                         │  │
    │  → 순차 부분(1-P)이 병목!                                                               │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### ILP 한계

| 요소 | 설명 | 해결 |
|------|------|------|
| **Data Dep** | RAW | Forwarding |
| **Control Dep** | Branch | Prediction |
| **Resource Dep** | Structural | Duplication |

### ILP 기법 비교

| 기법 | 하드웨어 | 소프트웨어 | IPC |
|------|----------|----------|-----|
| **Pipeline** | 중간 | 낮음 | ~1 |
| **Superscalar** | 높음 | 낮음 | 2~4 |
| **VLIW** | 낮음 | 높음 | 4~8 |
| **OoO** | 매우 높음 | 낮음 | 3~6 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 코드 최적화
**상황**: ILP 향상
**판단**: 의존성 제거

```c
// 의존성 있는 코드
int a = x + y;
int b = a + z;  // RAW: b depends on a
int c = m * n;  // independent
int d = c + p;  // RAW: d depends on c

// 컴파일러 최적화:
// c, d를 a, b와 병렬 실행 가능
```
