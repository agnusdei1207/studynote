# 분기 예측 (Branch Prediction)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분기 예측은 **"파이프라인 **Control Hazard를 **해결**하기 위해 **분기**(Branch) **방향**을 **미리 **예측**하여 **Pipeline Stall**을 **최소화**하는 **기술"**으로, **Static Prediction**(항상 참/거짓)과 **Dynamic Prediction**(실행 기록)으로 **구분**된다.
> 2. **Dynamic Prediction**: **BHT**(Branch History Table), **BTB**(Branch Target Buffer), **2-Level Adaptive**, **Tournament Predictor**로 **과거 패턴**을 **학습**하며 **Modern CPU**는 **90% 이상의 **정확도**를 **달성**한다.
> 3. **Speculative Execution**: **예측**에 **따라 **미리 **명령어**를 **실행**하고 **예측이 **틀리면 **Pipeline Flush**로 **결과를 **폐기**하며 **Return Address Stack**(RAS)로 **함수 복귀**를 **예측**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
분기 예측은 **"분기 방향을 미리 예측하는 기술"**이다.

**분기 예측 특징**:
- **Target Prediction**: 분기 목적지 예측
- **Direction Prediction**: Taken/Not-That 예측
- **Speculation**: 미리 실행
- **Recovery**: 예측 실패 시 복구

### 💡 비유
분기 예측은 **"나만 아는 지름길****과 같다.
- **분기**: 갈림길
- **예측**: 경험적 판단
- **성공**: 시간 단축
- **실패**: 되돌아감

## Ⅱ. 아키텍처 및 핵심 원리

### Static Prediction

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Static Prediction Strategies                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Always Not Taken                                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • 모든 분기를 Not Taken으로 예측                                                     │  │  │
    │  │  │  • Loop에서는 정확도 낮음                                                           │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  2. Always Taken                                                                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • 모든 분기를 Taken으로 예측                                                          │  │  │
    │  │  │  • Loop에서는 정확도 높음                                                           │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  3. Backward Taken, Forward Not Taken                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • 뒤로 가는 분기(Loop) = Taken                                                        │  │  │
    │  │  │  • 앞으로 가는 분기(if) = Not Taken                                                 │  │  │
    │  │  │  → Loop에서 매우 효과적                                                              │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Dynamic Prediction (BHT)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Branch History Table (1-bit BHT)                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Structure:                                                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  PC (lower bits) ──→ Index ──→ BHT Entry (1 bit: Taken=1, NotTaken=0)                │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  Example Loop:                                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  for (i = 0; i < 100; i++) {                    // Branch at PC=0x1000                │  │  │
    │  │      sum += i;                                                                       │  │  │
    │  │  }                                                                                   │  │  │
    │  │                                                                                       │  │  │
    │  │  Iteration   │  Actual  │  Prediction │  Result  │  BHT Update                       │  │  │
    │  │  ───────────────────────────────────────────────────────────────────────────────────  │  │  │
    │  │  1           │  Taken   │  NT (init)   │  Miss    │  1                               │  │  │
    │  │  2           │  Taken   │  T           │  Hit     │  1                               │  │  │
    │  │  3           │  Taken   │  T           │  Hit     │  1                               │  │  │
    │  │  ...         │  Taken   │  T           │  Hit     │  1                               │  │  │
    │  │  100         │  NT      │  T           │  Miss    │  0                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  → 98% 정확도 (2 miss / 100 predictions)                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 2-Level Adaptive Predictor

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         2-Level Adaptive (GShare)                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Structure:                                                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  BHR (Branch History Register)                                                       │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  Past 4 branch outcomes: 1011 (taken, not-taken, taken, taken)                     │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │         │                                                                              │  │
    │  │         ▼                                                                              │  │
    │  │  PHT (Pattern History Table)                                                          │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  Index = (PC XOR BHR) % TableSize                                                  │  │  │
    │  │  │  Entry: 2-bit saturating counter (00=StrongNT, 01=WeakNT, 10=WeakT, 11=StrongT)     │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  Saturating Counter State Machine:                                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  ┌─────────────────┐     ┌─────────────────┐                                       │  │  │
    │  │  │  11 (Strong T)  │◄───│  10 (Weak T)   │                                       │  │  │
    │  │  │       (T)       │     │      (T)       │                                       │  │  │
    │  │  └──────▲──────────┘     └──────▲──────────┘                                       │  │  │
    │  │         │ Taken                 │ Not Taken                                            │  │  │
    │  │         │                       │                                                   │  │  │
    │  │  ┌──────┴──────────┐     ┌──────┴──────────┘                                       │  │  │
    │  │  │  01 (Weak NT)  │────▶│  00 (Strong NT)│                                       │  │  │
    │  │  │      (NT)       │     │      (NT)       │                                       │  │  │
    │  │  └─────────────────┘     └─────────────────┘                                       │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Prediction 기법 비교

| 기법 | 복잡도 | 정확도 | 사용처 |
|------|--------|--------|--------|
| **Static (Always T)** | 0 | ~80% | Loop |
| **1-bit BHT** | 낮음 | ~85% | 일반적 |
| **2-bit BHT** | 중간 | ~90% | 일반적 |
| **2-Level** | 높음 | ~95% | Modern CPU |
| **Tournament** | 매우 높음 | ~97% | High-end |

### Branch Target Buffer (BTB)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         BTB (Branch Target Buffer)                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Cache-like structure for branch targets                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Tag │  Valid │  Target Address │  Prediction                                     │  │  │
    │  │  ───────────────────────────────────────────────────────────────────────────────────  │  │  │
    │  │  0x1000 │ 1 │ 0x100C │ Taken                                                  │  │  │
    │  │  0x2000 │ 1 │ 0x2050 │ Not Taken                                              │  │  │
    │  │  0x3000 │ 0 │ xxxxx │ -                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  • IF stage에서 Target Address 확인                                                      │  │
    │  │  → Prediction과 함께 fetch 가능                                                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: Loop 최적화
**상황**: 반복문 성능
**판단**: Branch Prediction 활용

```c
// Prediction-friendly code
for (int i = 0; i < n; i++) {  // Backward branch (predict taken)
    if (data[i] > threshold) {    // Inner branch
        count++;
    }
}
```
