# 아웃오브오더 실행 (Out-of-Order Execution)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: OoO는 **"명령어 **순서**를 **바꾸어 **실행**하여 **Pipeline Stall**을 **최소화**하는 **기술"**로, **Data Dependency**(RAW, WAR, WAW)가 **없는 **명령어**를 **먼저 **실행**하고 **Register Renaming**, **Scoreboarding**, **Tomasulo's Algorithm**으로 **의존성**을 **해결**한다.
> 2. **구조**: **Instruction Fetch** → **Decode** → **Rename**(Register Renaming) → **Dispatch**(Reservation Station) → **Issue**(Execute) → **Write-Back** → **Commit**(In-order)으로 **동작**하며 **ROB**(Reorder Buffer)로 **In-order Commit**을 **보장**한다.
> 3. **융합**: **IBM Power**(RS/6000), **Intel P6**(Pentium Pro 이상), **AMD K5**, **ARM Cortex**, **RISC-V**가 **OoO**를 **채택**했으며 **Speculative Execution**과 **결합**하여 **성능**을 **극대화**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
OoO는 **"명령어 순서를 바꿔 실행하는 기술"**이다.

**OoO 특징**:
- **동적 스케줄링**: Runtime
- **명령어 재배치**: 실행 순서 변경
- **In-order Commit**: 결과는 순서대로
- **하드웨어 복잡**: 복잡한 회로

### 💡 비유
OoO는 **"요리사의 주문 최적화****와 같다.
- **주문 순서**: 명령어 순서
- **재료 준비**: 데이터 의존성
- **조리 순서**: 실행 순서 변경

    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │

### Register Renaming

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Register Renaming (WAW/WAR 해결)                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    WAW Dependency Problem:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1: MUL R1, R2, R3                                                                    │  │
    │  2: ADD R1, R4, R5    ← WAW: both write R1                                           │  │
    │                                                                                         │  │
    │  In-order: 2 must wait for 1                                                           │  │
    │  OoO + Renaming:                                                                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Physical Register File (32 architectural, 100 physical)                              │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  RAT (Register Alias Table)                                                      │  │  │
    │  │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  │  Architectural │ Physical                                                    │  │  │  │
    │  │  │  │  ───────────────────────────────────────────────────────────────────────────  │  │  │  │
    │  │  │  │  R1            │ P10 (after inst 1)                                        │  │  │  │
    │  │  │  │  R1            │ P11 (after inst 2) ← different register!                    │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │  → No WAW dependency, both can execute in parallel                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### In-Order vs Out-of-Order

| 구분 | In-Order | Out-of-Order |
|------|----------|--------------|
| **복잡도** | 낮음 | 높음 |
| **IPC** | 낮음 | 높음 |
| **전력** | 적음 | 많음 |
| **설계** | 단순 | 복잡 |

### OoO 요구사항

| 요소 | 설명 |
|------|------|
| **Register Renaming** | WAW/WAR 해결 |
| **Reservation Station** | 명령어 대기 |
| **CDB** | 결과 브로드캐스트 |
| **ROB** | In-order Commit |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 코드 최적화
**상황**: 의존성 제거
**판단**: 컴파일러 최적화로 OoO 돕기

```c
// 의존성 있는 코드
int a = x + y;  // R1 = R2 + R3
int b = a + z;  // R4 = R1 + R5 (RAW)
int c = m * n;  // R6 = R7 * R8 (independent)
int d = c + p;  // R9 = R6 + R10 (independent)

// OoO: c, d를 a, b와 병렬 실행 가능
```
