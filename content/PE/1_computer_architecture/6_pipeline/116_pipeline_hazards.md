# 파이프라인 해저드 (Pipeline Hazards)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파이프라인 해저드는 **"파이프라인 **명령어가 **순차적으로 **실행**되지 **못하게 **하는 **장애"**로, **Data Hazard**(데이터 의존성), **Control Hazard**(분기), **Structural Hazard**(자원 충돌)로 **구분**되며 **Stall**(버블 삽입)이나 **Forwarding**(데이터 전달)으로 **해결**한다.
> 2. **Data Hazard**: **RAW**(Read After Write), **WAR**(Write After Read), **WAW**(Write After Write) **의존성**이 **존재**하며 **Forwarding**(Bypassing)으로 **해결**하고 **Control Hazard**는 **Branch Prediction**으로 **예측**한다.
> 3. **Structural Hazard**: **동일한 **하드웨어 **자원**(예: Memory, Register File)에 **동시 접근**할 때 **발생**하며 **Resource Duplication**(자원 복제) 또는 **Pipeline Stalling**으로 **해결**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
파이프라인 해저드는 **"파이프라인 명령어 실행을 방해하는 장애"**이다.

**해저드 종류**:
- **Data Hazard**: 데이터 의존성
- **Control Hazard**: 분기 명령어
- **Structural Hazard**: 자원 충돌

### 💡 비유
해저드는 **"도로 정체****와 같다.
- **Data Hazard**: 앞차가 가야 뒤차 감
- **Control Hazard**: 진로 불확실
- **Structural Hazard**: 차로 부족

## Ⅱ. 아키텍처 및 핵심 원리

### Data Hazard (RAW)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Data Hazard (RAW - Read After Write)                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Code:
    1: ADD R1, R2, R3    ; R1 ← R2 + R3
    2: SUB R4, R1, R5    ; R4 ← R1 + R5  ← R1을 기다려야 함!

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Pipeline Execution (without Forwarding)                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  ADD   │  IF  │  ID  │  EX  │  MEM │  WB  │                                        │  │  │
    │  │  SUB   │      │      │  IF  │  ID  │  STALL │ STALL │  EX  │  MEM │  WB  │          │  │  │
    │  │              ↑      ↑      2 bubbles inserted (R1을 기다림)                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    With Forwarding (Bypassing):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ADD   │  IF  │  ID  │  EX  │  MEM │  WB  │                                        │  │
    │  SUB   │      │      │  IF  │  ID  │  EX  │  MEM │  WB  │  (no stall!)                │  │
    │              │                          │                                              │  │
    │              └──────────── Forwarded from EX/MEM ─────────→                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Control Hazard (Branch)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Control Hazard (Branch)                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Code:
    1: BEQ R1, R2, Target  ; Branch if R1 == R2
    2: ADD R3, R4, R5      ; Not executed if taken
    Target:
    3: SUB R6, R7, R8

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Pipeline Execution (No Prediction)                                                    │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  BEQ   │  IF  │  ID  │  EX  │  (MEM)│  ──→ Branch calculated here                 │  │  │
    │  │  ADD   │      │  IF  │  ID  │  STALL │ STALL │ STALL │ (flush if taken)           │  │  │
    │  │  SUB   │      │      │  IF  │  ...  │                                        │  │  │
    │  │                  ↑ 3 bubbles inserted                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  With Branch Prediction:                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  BEQ   │  IF  │  ID  │  EX  │  (predict taken, fetch Target)                      │  │  │
    │  │  SUB   │      │      │  IF  │  ID  │  EX  │  (if prediction correct, no penalty)  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Structural Hazard

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Structural Hazard                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Problem: IF and MEM both need Memory access

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Pipeline (with Hazard)                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Inst1 │  IF  │  ID  │  EX  │  MEM │  WB  │                                        │  │  │
    │  │  Inst2 │  STALL│  IF  │  ID  │  EX  │  MEM │  WB  │  (wait for IF to finish)       │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  Solution: Separate I-Cache and D-Cache                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Inst1 │  IF  │  ID  │  EX  │  MEM(D)│ WB  │                                       │  │  │
    │  │  Inst2 │      │  IF  │  ID  │  EX   │ MEM(D)│ WB  │  (no stall!)                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 해저드 해결 기법

| 해저드 | 해결 기법 | 효과 |
|--------|----------|------|
| **Data** | Forwarding | Stall 제거 |
| **Control** | Branch Prediction | Penalty 감소 |
| **Structural** | Resource Duplication | 충돌 방지 |

### Forwarding 경로

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Forwarding Paths                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ALU Output → ALU Input (EX to EX)
    Memory Output → ALU Input (MEM to EX)
```
