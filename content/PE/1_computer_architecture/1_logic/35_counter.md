# 플립플롭 (Flip-Flop)

## 핵심 인사이트 (3줄 요약)
1. 플립플롭은 클럭 에지에서만 상태가 변경되는 1비트 순차 회로로, 카운터, 레지스터, 상태 머신의 기초이다
2. D 플립롭, JK 플립롭, T 플립롭이 있으며, Master-Slave 플립롭은 동기식 설계에 필수적이다
3. 기술사시험에서는 플립플롭의 Setup/Hold 시간, 메타 안정성, 카운터/래치 차이, 동기 설계가 핵심이다

## Ⅰ. 개요 (500자 이상)

플립롭(Flip-Flop)은 **에지 트리거(Edge-Triggered)** 순차 회로로, 클럭 신호의 하강(Rising Edge) 또는 상승(Falling) 에지에서만 출력 상태가 변경된다. 래치(Latch)가 레벨 트리거(Level Triggered)로 동작하는 반면, 플립롭은 에지 트리거(Edge Triggered)로 동작한다. 플립플롭은 **1비트 순차 기억 장비**로, **카운터(Counter)**, **레지스터(Register)**, **상태 머신(State Machine)**의 기본 요소이다.

```
D 플립롭(D Flip-Flop) 회로 기호:
        _______
       |      |
    D  ──┤ D    │
       |      │
    >───┤ CLK  >│
       |  ____ │
    Q  ──┤ Q'   │
       |_______|

입력: D (Data), CLK (Clock)
출력: Q, Q' (Complement)

동작:
CLK 상승 에지(Rising Edge)에서 Q = D로 샘플링
그 외 에지에서는 Q 유지
```

플립플롭의 핵심 특징은 **투명성 부족(No Transparency)**이다. 래치는 Enable=1일 때 입력이 출력으로 바로 통과하지만, 플립플롭은 클럭 에지에서만 상태가 변경되며, 그 외에는 입력이 변해도 출력은 유지된다. 이는 플립플롭이 **동기식(Synchronous)** 디지털 시스템에서 안정적으로 상태를 관리할 수 있게 한다.

**D 플립롭(D Flip-Flop)**은 가장 일반적인 플립플롭으로, D(Data) 입력을 클럭 에지에서 Q로 샘플링한다.

```
D 플립롭 진리표:
| CLK | D | Q(t+1) | 설명 |
|-----|---|---------|------|
| 0→1 | 0 | 0 | 리셋 |
| 0→1 | 1 | 1 | 셋 |
| 1→0 | X | Q(t) | 유지 |
| 1→1 | X | Q(t) | 유지 |
| ↓    ↓ |   | 클럭 상승 에지에서만 동작

X: 0 또는 1 (무관)
```

**JK 플립롭(JK Flip-Flop)**는 J(Set), K(Reset), CLK 입력을 가지며, 다음 진리표를 따른다.

```
JK 플롭 진리표:
| CLK | J | K | Q(t+1) | 설명 |
|-----|---|---|---------|------|
| ↓  | 0 | 0 | Q(t) | 유지 |
| ↓  | 0 | 1 | 0 | 리셋 |
| ↓  | 1 | 0 | 1 | 셋 |
| ↓  | 1 | 1 | Q'(t) | 토글 |

↓는 클럭 상승 에지
```

JK=11일 때 토글(Toggle) 동작을 하며, 이는 T 플립롭(T Flip-Flop)와 동일하다.

**T 플립롭(T Flip-Flop)**는 T 입력이 1일 때마다 클럭마다 상태가 반전(Toggle)한다.

```
T 플롭 진리표:
| CLK | T | Q(t+1) |
|-----|---|---------|
| ↓  | 0 | Q(t) |
| ↓  | 1 | Q'(t) |
```

T 플롭은 주파수 분주기(Frequency Divider)로 사용된다.

## Ⅱ. 아키텍처 및 핵심 원리 (1000자 이상)

### D 플립롭 구조

CMOS D 플롭은 2개 래치(Master-Slave 구조)로 구현된다.

```
Master-Slave D 플롭:
CLK ──┬─────┬───── CLK
     │     │
    ┌┴┐   ┌─┴─┐
    │ │   │ │ │
 D ─┤ ├─┬─┤ ├─── Q
  (Master) (Slave)
    │ │   │ │
  ─┬┘   └─┬─┘
CLK' ────────┘

CLK'는 CLK의 반전(Inverted)
음의 Master-Slave 구조로 0→0과 1→1 전이가 방지
```

### Setup/Hold 시간

플립플립롭이 안정적으로 데이터를 샘플링하기 위해서는 입력이 클럭 에지 전에 일정 시간 동안 안정되어야 한다.

```
Setup 시간(t_su): 입력이 클럭 에지 전에 안정 유지되어야 하는 최소 시간
Hold 시간(t_h): 클럭 에지 후 입력이 일정 시간 동안 유지되어야 하는 최소 시간

제약 조건:
t_su < t_clk - t_hold (t_clk는 클럭 주기)

예) t_clk = 10ns, t_su = 1ns, t_h = 0.5ns:
1ns + 0.5ns < 10ns (제약 충족)
```

### 메타 안정성(Metastability)

플립플롭이 켣속 클럭에 동기하려면 **메타 안정성**이 보장되어야 한다.

```
메타 안정성 조건:
1. Setup 시간(t_su) 충족
2. Hold 시간(t_h) 충족
3. 클럭 주파 범위 내에서 T_j 온도 유지
4. 출력 전압 swing이 V_OL/V_OH 범위 내

불리되지 않으면:
- 클럭 스큼(Clock Skew)로 인한 오류
- 데이터 무효
- 상태 머신 붕괴(State Machine Race Condition)
```

## Ⅲ. 융합 비교 및 다각도 분석

### 래치 vs 플립플롭

| 비교 항목 | 래치 (Latch) | 플립플립롭 (Flip-Flop) |
|----------|-------------|-------------------|
| 트리거 타입 | 레벨 트리거 | 에지 트리거 |
| 클럭 | 필요 (Enable=1) | 불필 (에지 트리거) |
| 투명성 | 있음 (입력→출력) | 없음 (CLK에서만 변경) |
| 사용 | SRAM 셀 | 카운터, 레지스터 |
| 동기 | 비동기 | 동기 |

### 플립플롭 유형 비교

| 플립롭 | 입력 | 출력 | 기능 | 응용 |
|--------|------|------|------|------|
| D | D, CLK | Q | D 저장 | 일반 레지스터 |
| JK | J, K, CLK | Q, Q' | J=1, K=0→Set, J=0, K=1→Reset | 상태 머신 |
| T | T, CLK | Q, Q' | 토글 (Toggle) | 주파수 분주 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 카운터 구현

```
4비트 동기 카운터 (T 플립롭 4개 직렬):
    T0   T1   T2   T3
    │    │    │    │
    ┌────┴────┴────┴────→ 출력 Q[3:0]

CLK 입력 → T0→T1→T2→3

CLK 주파수: f_CLK
출력 주파수: f_OUT = f_CLK / 16 (4비트 카운터)
```

### 레지스트르 구현

```
8비트 레지스터:
D[7:0] ──┬─┬─┬─┬─┬─┬─┬─┬─ 8개 D 플립롭
           │ │ │ │ │ │ │ │
CLK ──────┴─┴─┴─┴─┴─┴─→ 모든 CLK에 병렬
           │
Q[7:0] ──┬─┬─┬─┬─┬─┬─┬─ 8개 Q 출력

Write_Enable → 모든 CLK에 AND 게이트로 쓰기 제어
```

## Ⅴ. 기대효과 및 결론

플립플롭은 동기 디지털 시스템의 핵심이다. D 플립롭은 가장 널리 사용된다.

## 📌 관련 개념 맵

```
플립플롭
├── D 플립롭 (가장 일반)
├── JK 플롭 (토글)
├── T 플롭 (주파수 분주)
└── 파라미터
    ├── Setup 시간
    ├── Hold 시간
    └── 메타 안정성
```

## 👶 어린이를 위한 3줄 비유 설명

1. 플립플립롭은 카메라의 셔터 버튼처럼, 클럭(Shutter)을 누를 순간 입력을 사진하고 유지하는 장치예요
2. 래치는 문이 열려있는 동안에는 입력이 그대로 통과하지만, 플립플립롭은 클럭 틱을 때만 작동해서 상태를 바꿔요
3. CPU의 레지스터는 수천 개의 플립플롭으로 만들어져서, 클럭마다 데이터를 저장하고 계산에 사용해요
