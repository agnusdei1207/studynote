# 이기종 멀티코어 (Heterogeneous Multicore)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이기종 멀티코어는 성능, 전력, 기능이 서로 다른 여러 종류의 프로세서 코어를 단일 칩에 통합하는 아키텍처로, 워크로드 특성에 따라 최적의 코어에 작업을 할당하여 전력 효율과 성능을 동시에 최적화한다
> 2. **가치**: 동일 전력 예산 내에서 최대 40% 이상의 성능 향상 또는 동일 성능에서 50% 이상의 전력 절감을 달성하며, 모바일/엣지 디바이스의 배터리 수명과 데이터센터의 TCO 개선에 핵심적이다
> 3. **융합**: big.LITTLE, DynamIQ, HMP(Heterogeneous Multi-Processing)와 연관되며, OS 스케줄러, 전력 관리, 실시간 시스템 설계와 밀접하게 연동된다

---

## Ⅰ. 개요 (Context & Background)

### 개념
이기종 멀티코어(Heterogeneous Multicore)는 서로 다른 마이크로아키텍처, 성능 특성, 전력 소모를 가진 프로세서 코어들을 단일 칩에 통합하는 설계 방식이다. 대표적으로 ARM의 big.LITTLE 아키텍처가 있으며, 고성능(big) 코어와 저전력(LITTLE) 코어를 조합하여 작업 부하에 따라 최적의 코어로 실행한다. 이는 전력-성능 트레이드오프를 동적으로 관리하여, 모바일 기기에서는 배터리 수명을 연장하고, 데이터센터에서는 전력 비용을 절감한다.

### 💡 비유
이기종 멀티코어는 **"승용차와 스쿠터를 함께 보유한 배달 회사"**에 비유할 수 있다.
- **big 코어 (승용차)**: 빠르고 힘세지만 연료 많이 소모
- **LITTLE 코어 (스쿠터)**: 느리지만 연료 적게 소모
- **스케줄러**: "먼 거리/무거운 짐" → 승용차, "가까운 거리/가벼운 짐" → 스쿠터
- **결과**: 전체 연료 소모 최소화하면서 배달 완료

### 등장 배경 및 발전 과정

#### 1. 기존 기술의 치명적 한계점
- **동종 멀티코어의 전력 문제**: 모든 코어가 동일한 성능/전력 특성 → 경량 작업도 고성능 코어 사용
- **모바일 배터리 한계**: 스마트폰 사용 시간 확보를 위한 전력 최적화 필수
- **Dark Silicon**: 칩 면적 중 활용 가능한 부분이 전력 제약으로 감소

#### 2. 패러다임 변화와 혁신
- **2011년 ARM big.LITTLE**: Cortex-A15(성능) + Cortex-A7(효율)
- **2012년 Samsung Exynos 5 Octa**: 최초 상용 big.LITTLE
- **2015년 ARM DynamIQ**: 클러스터 경계 없는 유연한 구성
- **2017년 Qualcomm Snapdragon 835**: 성능, 효율, 절전 3티어
- **현재**: Apple M시리즈, Intel Hybrid(Alder Lake)

#### 3. 비즈니스적 요구사항
- **모바일 배터리**: 1일 충전 목표 달성
- **데이터센터 TCO**: 전력 비용 30% 절감
- **엣지 AI**: 제한 전력으로 추론 수행

## Ⅱ. 아키텍처 및 핵심 원리

### 구성 요소

| 요소명 | 상세 역할 | 내부 동작 메커니즘 | 관련 기술 | 비유 |
|--------|-----------|-------------------|-----------|------|
| **big 코어** | 고성능 작업 처리 | 높은 IPC, 넓은 파이프라인 | OoO, 수퍼스칼라 | 승용차 |
| **LITTLE 코어** | 저전력 작업 처리 | 낮은 IPC, 좁은 파이프라인 | In-order, 단순 구조 | 스쿠터 |
| **HMP 스케줄러** | 코어 선택 결정 | 부하 추적, 에너지 모델 | EAS (Energy Aware Scheduling) | 배차 담당자 |
| **공유 캐시** | 코어 간 데이터 공유 | DSU (DynamIQ Shared Unit) | L3 Cache Slicing | 창고 |
| **전원 도메인** | 코어별 독립 전원 | Power Gating, DVFS | PMU, PSCI | 시동 키 |

### 심층 동작 원리

#### 1. 작업 할당 전략 (Task Scheduling)

**HMP (Heterogeneous Multi-Processing)**:
```
작업 특성 분석:
┌────────────────────────────────────────────────────────────────┐
│                   │                                            │
│   CPU 사용률      │     높음 ───────► big 코어                  │
│       ▲           │              ↗                              │
│       │           │            ↗                                │
│       │           │          ↗                                  │
│       │           │   중간 ─┘                                   │
│       │           │          ↘                                  │
│       │           │            ↘                                │
│       │           │              ↘                              │
│       │           │     낮음 ───────► LITTLE 코어               │
│       │           │                                            │
│       └───────────┴────────────────────────────────────────────│
│                   짧은 ◄───────────▶ 긴                         │
│                   실행 시간                                     │
└────────────────────────────────────────────────────────────────┘
```

**EAS (Energy Aware Scheduling)**:
- 에너지 모델 기반 코어 선택
- `energy_diff(task, src_cpu, dst_cpu)` 계산
- 가장 낮은 에너지 소모 코어 선택

#### 2. 마이그레이션 (Migration)

**Cluster Migration**:
- big 클러스터 ↔ LITTLE 클러스터 전체 전환
- 구현 단순, 지연 시간 존재

**Task Migration (HMP)**:
- 개별 태스크 단위 마이그레이션
- 유연하지만 컨텍스트 스위칭 오버헤드

**Thread Migration 주요 단계**:
1. 부하 감지 (Load Tracking)
2. 코어 선택 (Target Selection)
3. 컨텍스트 이동 (Context Migration)
4. 캐시 워밍 (Cache Warming)

#### 3. 전력 관리

**코어 레벨 전원 도메인**:
| 상태 | big 코어 | LITTLE 코어 |
|------|----------|-------------|
| Active | 100% | 100% |
| WFI | 20% | 10% |
| Core Off | <1% | <0.1% |
| Cluster Off | 0% | 0% |

**DVFS 전략**:
- big: 높은 클럭, 높은 전압 (2.5-4.0GHz)
- LITTLE: 낮은 클럭, 낮은 전압 (1.0-2.0GHz)

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단

#### 시나리오: 모바일 AP 설계
**상황**: 스마트폰용 AP 코어 구성 결정
**판단**:
- 2개 X4 + 4개 A720 + 4개 A520 = 10코어 구성
- 게임: X4 활성
- 일반 사용: A720
- 대기/음악: A520

### 도입 시 고려사항

- [ ] 워크로드 분석: 성능/전력 요구사항 파악
- [ ] OS 지원: HMP/EAS 스케줄러 지원
- [ ] 마이그레이션 오버헤드: 빈번한 이동 시 성능 저하

### 주의사항

1. **마이그레이션 오버헤드**: 잦은 이동은 성능 저하
2. **캐시 콜드**: 코어 이동 시 캐시 미스
3. **실시간 보장**: 코어 타입에 따른 지연 차이

## 👶 어린이를 위한 3줄 비유 설명

이기종 멀티코어는 **"승용차와 스쿠터를 같이 가진 배달부"**예요.

1. **다른 차들**: 승용차(big)는 빠르지만 기름을 많이 먹어요. 스쿠터(LITTLE)는 느리지만 기름을 적게 먹어요.

2. **똑똑하게 선택**: 먼 곳에 무거운 짐을 배달할 땐 승용차를, 가까운 곳에 가벼운 편지를 배달할 땐 스쿠터를 써요.

3. **기름 아끼기**: 이렇게 하면 기름을 아끼면서도 빨리 배달할 수 있어요! 스마트폰 배터리도 오래가요.
