# 인코더 (Encoder)

## 핵심 인사이트 (3줄 요약)
1. 인코더는 2^n개 입력 중 1개가 활성화된 위치를 n비트 이진 코드로 변환하는 조합 논리 회로로, 10선-4선 인코더가 대표적이다
2. Priority Encoder는 동시 다중 입력 활성화 시 우선순위가 가장 높은 입력을 선택하여 출력하며, 인터럽트 컨트롤러에 필수적이다
3. 기술사 시험에서는 인코더 진리표, Priority Encoder 설계, 10선-4선 인코더 회로 해석이 핵심이다

## Ⅰ. 개요 (500자 이상)

인코더(Encoder)는 2^n개 입력 중 활성화된(1인) 입력의 위치를 n비트 이진 코드로 변환하는 조합 논리 회로이다. 예를 들어 8개 입력(2^3) 중 하나가 1이면, 3비트 이진 코드(000-111)로 위치를 출력한다. 인코더는 키보드(Keyboard) 스캔, 위치 센서(Position Sensor), 인터럽트 컨트롤러(Interrupt Controller) 등에 사용된다.

**10선-4선 인코더(Decimal-to-BCD Encoder)**는 10개 입력(0-9) 중 활성화된 입력을 4비트 BCD 코드(0000-1001)로 변환한다. 예를 들어 입력 5가 활성화되면 출력 0101(5)을 생성한다.

```
10선-4선 인코더:
입력 0-9(10개) → 출력 D C B A (4비트 BCD)

진리표:
| 활성 입력 | 출력 (DCBA) |
|-----------|--------------|
| 0 | 0000 |
| 1 | 0001 |
| 2 | 0010 |
| ... | ... |
| 9 | 1001 |
```

**Priority Encoder(우선순위 인코더)**는 동시에 여러 입력이 활성화되어도 가장 높은 우선순위(Priority) 입력을 선택한다. 예를 들어 입력 7(Priority 7)과 입력 3(Priority 3)이 동시에 활성화되면, 높은 우선순위를 가진 7을 선택하여 출력 111을 생성한다. Priority Encoder는 인터럽트 컨트롤러(PIC, APIC)에서 여러 인터럽트 소스가 동시에 발생했을 때 가장 높은 우선순위 인터럽트를 선택하는 데 사용된다.

인코더의 역회로는 **디코더(Decoder)**로, 디코더는 n비트 입력을 2^n개 출력 중 하나로 변환한다(예: 2-4 디코더는 2비트 입력을 4개 출력 중 하나로 변환).

## Ⅱ. 아키텍처 및 핵심 원리 (1000자 이상)

### 8-3 인코더 (8-to-3 Encoder)

```
8-3 인코더:
입력: D7 D6 D5 D4 D3 D2 D1 D0 (8개, 1-hot)
출력: A2 A1 A0 (3비트)

진리표:
| 활성 입력 | 출력 (A2 A1 A0) |
|-----------|------------------|
| D0 | 0 0 0 |
| D1 | 0 0 1 |
| D2 | 0 1 0 |
| D3 | 0 1 1 |
| D4 | 1 0 0 |
| D5 | 1 0 1 |
| D6 | 1 1 0 |
| D7 | 1 1 1 |

불 대수식:
A2 = D4 + D5 + D6 + D7
A1 = D2 + D3 + D6 + D7
A0 = D1 + D3 + D5 + D7
```

### Priority Encoder (4-2 예시)

```
Priority Encoder (4-2):
입력: D3(최우선) D2 D1 D0(최저우선)
출력: A1 A0 (2비트) + V(Valid)

진리표:
| 입력 (D3 D2 D1 D0) | 출력 (A1 A0 V) |
|--------------------|-------------------|
| 0 0 0 0 | 0 0 0 (모두 비활성) |
| x x x 1 | 0 0 1 (D0 선택) |
| x x 1 x | 0 1 1 (D1 선택) |
| x 1 x x | 1 0 1 (D2 선택) |
| 1 x x x | 1 1 1 (D3 선택) |

V = D0 + D1 + D2 + D3 (적어도 하나 활성화)
```

Priority Encoder는 OR 게이트 트리로 구현된다.

### Priority Encoder 회로 구현

```
4-2 Priority Encoder 회로:
D3 ──┬── OR ────┬── A1
     │          │
D2 ──┴── OR ────┤
     │          │
D1 ──┴── OR ────┤
     │          │
D0 ──┴── OR ────┴── V

OR 게이트 출력:
A1 = D3 OR D2
A0 = D3 OR (D1 AND NOT D2))
V = D0 OR D1 OR D2 OR D3

해석:
- D3=1 → A1=1, A0=1 (최우선)
- D3=0, D2=1 → A1=1, A0=0
- D3=0, D2=0, D1=1 → A1=0, A0=1
- D3=0, D2=0, D1=0, D0=1 → A1=0, A0=0
```

### 8-3 Priority Encoder

```
8-3 Priority Encoder:
입력: I7(최우선) ... I0(최저우선)
출력: A2 A1 A0 + V + GS (Group Select)

진리표 (일부):
| 입력 | 출력 (A2 A1 A0) | V | GS |
|------|------------------|---|-----|
| 0...0 | 0 0 0 | 0 | 0 (무효) |
| x...1 | 0 0 1 | 1 | 1 |
| ... | ... | 1 | 1 |
| 1...x | 1 1 1 | 1 | 1 |

V (Valid) = 적어도 하나 입력 활성화
GS (Group Select) = 우선순위 그룹 활성화 표시
```

## Ⅲ. 융합 비교 및 다각도 분석

### 인코더 vs 디코더

| 비교 항목 | 인코더 (Encoder) | 디코더 (Decoder) |
|----------|-------------------|----------------|
| 입력 수 | 2^n개 | n개 |
| 출력 수 | n개 | 2^n개 |
| 기능 | 1-hot → Binary | Binary → 1-hot |
| 예시 | 8-3 인코더 (8→3) | 2-4 디코더 (2→4) |
| 응용 | 키보드, 인터럽트 | 메모리 어드레스 디코딩 |

### Standard Encoder vs Priority Encoder

| 비교 항목 | Standard Encoder | Priority Encoder |
|----------|-----------------|-------------------|
| 동시 입력 제약 | 오직 1개만 활성화 가능 | 다중 입력 허용 |
| 충돌 처리 | 정의 안됨 (에러) | 우선순위로 자동 해결 |
| 복잡도 | 낮음 (OR 게이트) | 높음 (Priority Logic) |
| 응용 | 단순 위치 인코딩 | 인터럽트 컨트롤러 |

### 10선-4선 vs 8-3 인코더

| 인코더 | 입력 수 | 출력 수 | 입력 타입 | 출력 코드 |
|--------|---------|---------|-----------|-----------|
| 10선-4선 | 10개 | 4개 (BCD) | Decimal digit | BCD (0000-1001) |
| 8-3 | 8개 | 3개 | Binary | 000-111 |
| 16-4 | 16개 | 4개 | Hex | 0000-1111 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 인터럽트 컨트롤러에서의 Priority Encoder

```
인터럽트 요청:
IRQ7(최우선, Non-maskable)
...
IRQ0(최저우선)

Priority Encoder 출력:
- A2 A1 A0: 활성 인터럽트 번호
- V: 적어도 하나 활성화

CPU 동작:
V=1이면 인터럽트 서비스 루틴(ISR) 실행
인터럽트 번호로 ISR 주소 계산 (Interrupt Vector Table)
```

### 키보드 스캔에서의 인코더

```
키보드 매트릭스 (예: 4×4 키):
    C0 C1 C2 C3
R0 1  2  3  A
R1 4  5  6  B
R2 7  8  9  C
R3 *  0 #  D

키 스캔: Row, Column 순차 스캔
키 5 누름 → R1=1, C1=1 → 인코더 입력 0101 → BCD 5
```

### 기술사 시험 대비 문제

**문제**: Priority Encoder(4-2)의 진리표를 작성하고, D3=1, D2=0, D1=0, D0=1일 때 출력을 계산하시오.

**해설**:
```
D3(최우선)=1 → A1=1, A0=1 (무조건 D3 선택)

따라서 출력:
A1 = 1, A0 = 1 (코드 3)

D0=1은 D3=1로 인해 무시 (우선순위 낮음)
```

## Ⅴ. 기대효과 및 결론

인코더는 위치 정보를 디지털 코드로 변환한다. Priority Encoder는 인터럽트 컨트롤러의 핵심이다.

## 📌 관련 개념 맵

```
인코더 (Encoder)
├── 정의: 2^n 입력 → n비트 출력
├── 종류
│   ├── Standard Encoder (1-hot → Binary)
│   └── Priority Encoder (우선순위 선택)
├── 10선-4선 (Decimal → BCD)
├── 8-3 (Binary)
└── 응용
    ├── 키보드

    └── 위치 센서
```

## 👶 어린이를 위한 3줄 비유 설명

1. 인코더는 여러 개의 버튼 중 하나가 눌러지면, 그 위치를 컴퓨터가 이해하는 번호로 바꿔주는 장치예요
2. 엘리베이터에서 10개 버튼(0-9) 중 하나를 누르면, 인코더가 그 번호를 4비트 BCD 코드로 변환해서 CPU에 전달해요
3. Priority Encoder는 여러 사람이 동시에 말할 때, 더 중요한 사람(우선순위 높음)의 말을 먼저 듣는 판사처럼 작동해요
