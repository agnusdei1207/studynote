# 디코더 (Decoder)

## 핵심 인사이트 (3줄 요약)
1. 디코더는 n비트 입력을 2^n개 출력 중 하나로 변환하는 조합 논리 회로로, 2-4 디코더가 가장 일반적이다
2. 메모리 주소 디코딩, 칩 선택(Chip Select), 디스플레이 구동 등에 사용되며, 인코더의 역회로이다
3. 기술사 시험에서는 2-4 디코더 진리표, 활성화 출력(Active-Low/Active-High), 메모리 어드레스 디코딩이 핵심이다

## Ⅰ. 개요 (500자 이상)

디코더(Decoder)는 n비트 입력을 2^n개 출력 중 하나로 변환하는 조합 논리 회로로, **인코더의 역기능**을 수행한다. 예를 들어 2비트 입력(A1, A0)의 4가지 조합(00, 01, 10, 11)에 따라 4개 출력(Y0-Y3) 중 하나를 활성화한다. 2-4 디코더는 2비트 입력에 4개 출력을 가지며, 각 입력 조합에 대해 정확히 하나의 출력만 활성화된다.

```
2-4 디코더:
입력: A1 A0 (2비트)
출력: Y0 Y1 Y2 Y3 (4개, 1-hot)

진리표:
| A1 | A0 | 활성 출력 | 2진수 |
|----|----|----------|--------|
| 0 | 0 | Y0 | 00(0) |
| 0 | 1 | Y1 | 01(1) |
| 1 | 0 | Y2 | 10(2) |
| 1 | 1 | Y3 | 11(3) |
```

디코더의 핵심 응용은 **메모리 주소 디코딩**이다. CPU의 주소 버스(16비트: A15-A0)에서 2^n 바이트 단위로 메모리 칩을 선택할 때, 상위 주소 비트를 디코딩하여 칩 선택(Chip Select, CS) 신호를 생성한다. 또한 디스플레이 구동(LCD, 7-Segment)에서 BCD 코드를 7-segment 출력으로 변환하는 BCD-7세그먼트 디코더에도 사용된다.

## Ⅱ. 아키텍처 및 핵심 원리 (1000자 이상)

### 2-4 디코더 회로 구조

```
2-4 디코더 회로 (Active-Low):
   A1 A0    Y3 Y2 Y1 Y0
    │ │      │  │  │  │
  ──┴─┴───┐  │  │  │
  │      │  └──┴──┴──┴─── NAND 게이트들
  │      │     │  │  │
  INV    INV  │  │  │
  │      │     └──┴──┴──┴─── NOT 게이트들

Y0 = (A1'·A0)' = A1'·A0' (00일 때 활성)
Y1 = (A1'·A0)' = A1'·A0  (01일 때 활성)
Y2 = (A1·A0)' = A1·A0'  (10일 때 활성)
Y3 = (A1·A0)' = A1·A0    (11일 때 활성)
```

### 3-8 디코더

```
3-8 디코더:
입력: A2 A1 A0 (3비트)
출력: Y0-Y7 (8개)

진리표:
| A2 A1 A0 | 활성 출력 | 2진수 |
|----------|----------|--------|
| 0  0  0 | Y0 | 000(0) |
| 0  0  1 | Y1 | 001(1) |
| 0  1  0 | Y2 | 010(2) |
| 0   1  1 | Y3 | 011(3) |
| 1  0  0 | Y4 | 100(4) |
| 1  0  1 | Y5 | 101(5) |
| 1  1  0 | Y6 | 110(6) |
| 1  1  1 | Y7 | 111(7) |
```

### BCD-7세그먼트 디코더

BCD(0000-1001)를 7-segment LED 구동 신호(a-g)로 변환한다.

```
BCD-7세그먼트 디코더:
입력: D3 D2 D1 D0 (4비트 BCD)
출력: a b c d e f g (7개)

예) D3D2D1D0 = 0001 (1)일 때:
a=1, b=1, c=1, d=0, e=1, f=1, g=0 → "1" 표시
```

## Ⅲ. 융합 비교 및 다각도 분석

### 2-4 디코더 vs 인코더

| 비교 항목 | 디코더 | 인코더 |
|----------|--------|--------|
| 입력 | n비트 | 2^n개 |
| 출력 | 2^n개 | n비트 |
| 기능 | Binary → 1-hot | 1-hot → Binary |
| 예시 | 2-4 디코더 (2→4) | 8-3 인코더 (8→3) |
| 역관 | 역관계 | 역관계 |

### Active-Low vs Active-High 디코더

| 타입 | 활성 출력 | 전압 레벨 | 응용 |
|------|-----------|----------|------|
| Active-Low | 활성=0(LOW), 비활성=1(HIGH) | TTL 호환 | TTL 시스템 |
| Active-High | 활성=1(HIGH), 비활성=0(LOW) | CMOS 표준 | CMOS 시스템 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 메모리 어드레스 디코딩

```
16비트 주소 공간, 8KB 메모리 칩:
A15-A14-A13 → 2-4 디코더 → CS (Chip Select)

A15 A14 A13 | CS | 메모리 영역
-------------|----|------------------
0    0   0   | 1  | 0x0000-0x1FFF (ROM)
0    0   1   | 1  | 0x2000-0x3FFF (RAM)
0    1   0   | 1  | 0x4000-0x5FFF (RAM)
... (반복)
```

## Ⅴ. 기대효과 및 결론

디코더는 n비트 입력을 2^n개 출력으로 변환한다. 메모리 어드레스 디코딩의 핵심이다.

## 📌 관련 개념 맵

```
디코더
├── 정의: n비트 → 2^n개 출력
├── 종류
│   ├── 2-4 디코더
│   ├── 3-8 디코더
│   └── BCD-7세그먼트 디코더
└── 응용
    ├── 메모리 주소 디코딩
    ├── 칩 선택 (Chip Select)
    └── 디스플레이 구동
```

## 👶 어린이를 위한 3줄 비유 설명

1. 디코더는 2진수 코드를 해당하는 위치로 변환하는 장치예요
2. "00", "01", "10", "11" 4가지 입력이 있을 때, 각각 0번째, 1번째, 2번째, 3번째 위치에 불을 켜주는 것과 같아요
3. 컴퓨터가 메모리 주소를 읽을 때 디코더가 주소를 칩 위치로 변환해서 해당 칩을 선택해요
