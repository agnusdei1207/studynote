# 3상태 버퍼 (Tri-state Buffer)

## 핵심 인사이트 (3줄 요약)
1. 3상태 버퍼는 출력이 0, 1, Hi-Z(고임피던스) 3가지 상태를 가진 버퍼로, Enable=0일 때 회로를 분리하여 버스 공유(Bus Sharing)를 가능하게 한다
2. 데이터 버스, 주소 버스, 메모리 I/O 인터페이스의 핵심 소자이며, OE(Output Enable) 또는 G로 제어된다
3. 기술사 시험에서는 Hi-Z 상태 조건, 버스 아비트릭, 충돌 방지, 전송 지연 해석이 핵심이다

## Ⅰ. 개요 (500자 이상)

3상태 버퍼(Tri-state Buffer)는 출력이 0, 1, **Hi-Z(High Impedance)** 3가지 상태를 가진 회로이다. Hi-Z 상태에서는 출력이 회로에서 전기적으로 분리되어(Detached), 여러 장치가 단일 버스(Bus)를 공유할 수 있게 한다. 일반 버퍼와 달리 **Enable 핀**을 가지며, Enable=0일 때 Hi-Z, Enable=1일 때 정상 동작(0 또는 1 출력)한다.

```
회로 기호:
    ┌───┐
EN ─┤   │
    │   ├─── Y (Output)
A  ─┤   │
    └───┘

동작:
EN=0: Y = Hi-Z (고임피던스, 회로 분리)
EN=1: Y = A (정상 동작)
```

3상태 버퍼의 핵심 응용은 **버스 아비트릭(Bus Arbitration)**이다. 단일 버스에 여러 장치(CPU, 메모리, I/O 컨트롤러 등)를 연결할 때, 각 장치는 자신의 데이터를 전송할 때만 버스를 구동(Enable=1)하고, 그 외에는 Hi-Z 상태(Enable=0)로 다른 장치가 버스를 사용할 수 있게 한다.

```
데이터 버스 구조:
    CPU       Memory       I/O
     │          │           │
    ┌─┴─┐      ┌─┴─┐       ┌─┴─┐
    │   │      │   │       │   │
    │3상│      │3상│       │3상│ (3상태 버퍼)
    │   │      │   │       │   │
    └─┬─┘      └─┬─┘       └─┬─┘
      │           │           │
      └─────┬─────┴───────┘
            │
         데이터 버스 (공유)

작동:
- CPU Enable=1, 나머지 Enable=0 → CPU 데이터 전송
- Memory Enable=1, 나머지 Enable=0 → Memory 데이터 전송
```

3상태 버퍼는 인버팅 3상태(Inverting)과 논인버팅 3상태(Non-inverting) 두 가지가 있다.
- **논인버팅 3상태**: EN=1, A=0→Y=0, A=1→Y=1
- **인버팅 3상태**: EN=1, A=0→Y=1, A=1→Y=0 (버퍼 + 인버터)

## Ⅱ. 아키텍처 및 핵심 원리 (1000자 이상)

### CMOS 3상태 버퍼 구조

```
CMOS 논인버팅 3상태 버퍼:
        V_DD
         │
    ┌────┴────┐
    │         │
   PMOS      PMOS (Enable 제어)
    │         │
A ──┤         ├─┬─── Y
    │         │ │
EN ─┤         ├─┘
    │         │
   NMOS      NMOS
    │         │
    └────┬────┘
         │
        GND

동작:
EN=1: PMOS/NMOS 활성 → A → Y (정상 동작)
EN=0: 모든 MOSFET OFF → Y = Hi-Z (분리)
```

### Hi-Z 상태 (High Impedance State)

Hi-Z 상태는 출력이 개방 회로(Open Circuit)처럼 동작하여, 부하에서 전기적으로 분리된다.

```
Hi-Z 상태 특성:
- 출력 임피던스: > 1MΩ (실제로는 무한대)
- 출력 전류: < 1μA (누설 전류만)
- 전압: 부하에 의해 결정 (부동 상태)

회로 모델:
    V_DD
     │
   ┌─┴─┐
   │   │ ← OFF (EN=0로 인가)
   └─┬─┘
     │ Y (Hi-Z, 부동)
   ┌─┴─┐
   │   │ ← OFF (EN=0로 인가)
   └─┬─┘
     │
    GND

양쪽 MOSFET가 OFF로 출력이 부동(Floating)
```

### 3상태 버퍼 vs 일반 게이트

| 파라미터 | 일반 게이트 | 3상태 버퍼(EN=1) | 3상태 버퍼(EN=0) |
|----------|-------------|---------------------|-------------------|
| 출력 논리 | Y = f(Inputs) | Y = A (또는 A') | Hi-Z (정의 안됨) |
| 출력 저항 | 수kΩ | 수kΩ | >1MΩ (개방) |
| 전류 구동 | 1-2mA | 1-2mA | <1μA (누설) |
| 버스 공유 | 불가능 | 가능 (EN=1일 때만) | 가능 (Hi-Z로 양보) |

### 버스 충돌 방지(Bus Contention)

```
충돌 상태 (Bus Contention):
CPU Enable=1, Memory Enable=1 → 양쪽이 버스를 구동

CPU Y=0 ──┬─── V_BUS
Memory Y=1 ─┼─── V_BUS
              │
           충돌! (Short circuit)

해결:
- 어느 한 순간에 오직 하나의 Enable만 1이 되도록 설계
- 아비트러(Arbiter)가 Enable 신호 제어
```

## Ⅲ. 융합 비교 및 다각도 분석

### 3상태 버퍼 vs 오픈 컬렉터 vs 일반 버퍼

| 비교 항목 | 3상태 버퍼 | 오픈 컬렉터 | 일반 버퍼 |
|----------|-------------|--------------|----------|
| Hi-Z 상태 | 가능 (EN 제어) | 항상(상단 없음) | 불가능 |
| 버스 공유 | 가능 (활성 제어) | 가능 (Wired-OR) | 불가능 |
| 속도 | 빠름 (능동 Pull-up/Pull-down) | 느림 (수동 Pull-up) | 빠름 |
| 전력 | 낮음 (CMOS) | 낮음 | 낮음 |
| 응용 | 데이터/주소 버스 | Wired-OR 버스 | 신호 완충 |

### 인버팅 vs 논인버팅 3상태 버퍼

| 타입 | EN=1 동작 | EN=0 동작 | 회로 복잡도 | 응용 |
|------|-----------|-----------|-------------|------|
| 논인버팅 | Y=A | Hi-Z | 단순 (버퍼) | 데이터 버스 |
| 인버팅 | Y=A' | Hi-Z | 버퍼+인버터 | 신호 반전+분리 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 마이크로프로세서 버스 인터페이스

```
CPU 버스 인터페이스:
CPU 내부:
    ALU ──┬── 버퍼(Enable) ──┬─ 데이터 버스 (공유)
    레지스터 ──┤                ├─ 메모리 인터페이스
    캐시 ────┘                └─ I/O 인터페이스

Enable 신호 제어:
- ALU→Bus: Enable=1, 나머지=0
- Register→Bus: Enable=1, 나머지=0
- Memory→Bus: Enable=1, 나머지=0

타이밍:
T1: ALU Enable=1 → ALU 데이터 전송
T2: 모든 Enable=0 → 버스 분리(Hi-Z)
T3: Memory Enable=1 → 메모리 데이터 전송
```

### 기술사 시험 대비 문제

**문제**: 3상태 버퍼 2개가 버스에 연결되어 있다. 충돌을 방지하기 위해 Enable 신호를 제어하는 아비트러 설계 원칙을 설명하시오.

**해설**:
```
아비트러 설계 원칙:
1. Mutual Exclusion (상호 배제): 동시에 오직 하나의 Enable=1 허용
2. Priority Encoding (우선순위 부여): CPU > DMA > I/O 순서
3. Round-Robin (라운드 로빈): 순차적으로 Enable 부여

회로 구현:
Request 신호(REQ1, REQ2, ...) → Priority Encoder → Grant(G1, G2, ...)
Grant 신호가 Enable=1로 연결

상태 천이:
S0: IDLE (모든 Enable=0)
S1: G1=1, 나머지=0 (장치1 사용)
S2: G1→0, G2=1 (장치2로 전이)
S3: ...
```

## Ⅴ. 기대효과 및 결론

3상태 버퍼는 버스 공유의 핵심 소자이다.

## 📌 관련 개념 맵

```
3상태 버퍼
├── 3가지 상태
│   ├── 0 (LOW)
│   ├── 1 (HIGH)
│   └── Hi-Z (고임피던스)
├── 제어
│   ├── Enable=1: 정상 동작
│   └── Enable=0: Hi-Z
└── 응용
    ├── 데이터 버스
    ├── 주소 버스
    └── 메모리 인터페이스
```

## 👶 어린이를 위한 3줄 비유 설명

1. 3상태 버퍼는 "켜짐/꺼짐/연결 끊기" 3가지 상태를 가진 특별한 스위치예요
2. 여러 사람이 같은 전화기를 사용할 때, 통화할 때만 연결하고 통화 안 할 때는 전화를 끊는 것처럼 3상태 버퍼도 회로 연결을 끊어요
3. 컴퓨터 내부에서 CPU와 메모리가 데이터를 주고받을 때, 3상태 버퍼가 데이터 버스를 공유해서 충돌 없이 통신하게 돼요
