# 카운터 (Counter)

## 핵심 인사이트 (3줄 요약)
1. 카운터는 클럭 마다 카운트(Count)를 증가 또는 감소시키는 순차 회로로, 주파수 분주기, 타이머, 프로그램 카운터에 사용된다
2. **동기 카운터(Synchronous Counter)**는 모든 플립플롭이 동일 클럭에 의해 동기하며, **비동기 카운터(Asynchronous Counter)**는 각 비트의 플립플롭이 이전 비트의 출력을 클럭으로 사용한다
3. 기술사시험에서는 카운터의 주파수 분주비, 모듈러 카운터 설계, Up/Down 카운터, 리플/시리얼/아일리스 카운터가 핵심이다

## Ⅰ. 개요 (500자 이상)

카운터(Counter)는 클럭(Clock) 사이클마다 상태가 변하는 순차 회로로, **이진 카운터(Binary Counter)**와 **BCD 카운터**가 있다. 이진 카운터는 0 → 1 → 2 → 3 → ... → 최댓값 → 0으로 순환하며, 모듈러 2^n 카운터는 0 → 1 → 2 → ... → (2^n-1) → 0으로 순환한다. **업카운터(Up Counter)**는 카운트를 증가시키고, **다운카운터(Down Counter)**는 감소시키며, **리플 카운터(Reversible Counter)**는 Up/Down 선택이 가능하다.

```
4비트 업카운터 동작:
CLK ──┬───┬───┬───┬─── 4개 T 플립롭 직렬
     │   │   │   │
    Q0  Q1  Q2  Q3

CLK 마다:
0 → 1 → 2 → 3 → ... → 15 → 0

순환: 0 → 1 → 2 → ... → 15 → 0 (16주기)
```

카운터의 핵심 응용은 **주파수 분주기(Frequency Divider)**이다. T 플립롭을 직렬 연결하면 주파수가 2^n로 분주된다.

```
T 플롭을 이용한 주파수 분주기:
CLK 입력 ──┬── T0 ──┬── T1 ──┬── T2 ──→ CLK 출력
             │       │       │
             └───┴───┴─────┘

CLK 출력 = CLK 입력 / 8 (3단계 분주)
```

**모듈러 카운터(Modular Counter)**는 프로그램 카운터(Programmable Counter)의 일종으로, 최종 카운트 값(예: 10, 5, 2, 1)을 미리 설정할 수 있다.

## Ⅱ. 아키텍처 및 핵심 원리 (1000자 이상)

### 동기 카운터 구조

```
4비트 동기 업카운터:
CLK ──┬───┬───┬─── 4개 T 플롭 직렬
     │   │   │   │
    Q0  Q1  Q2  Q3

CLK 마다:
상태 변화: 0000 → 0001 → 0010 → 0011 → ... → 1111 → 0000

타이밍:
Q0: 0→1→0→1→... (CLK/2 주파수)
Q1: 0→0→1→1→... (CLK/4 주파수)
Q2: 0→0→0→0→1111→... (CLK/8 주파수)
Q3: 0→0→0→0→0000→... (CLK/16 주파수)
```

### 리플 카운터(Reversible Counter)

```
Up/Down 리플 카운터:
입력:
- CLK (Clock)
- U/D (Up/Down 제어)

U/D=1: 업 카운터 (증가)
U/D=0: 다운 카운터 (감소)

4비트 리플 카운터 회로:
    U/D
     │
Q3 Q2 Q1 Q0 ──┬─┬─┬─┬─── Adder/Subtractor
     │   │ │ │
   ┌──┴──┴──┴─── 0/1 (Carry/Borrow)
     │
   CLK
```

### 시리얼 카운터(Serial Counter)

```
4비트 시리얼� 카운터:
CLK 입력 → LSB → MSB으로 비트 시프

비트 순차 전이:
CLK 1: Q0 = D_in
CLK 2: Q0 → Q1 (Shift)
CLK 3: Q1 → Q2
CLK 4: Q2 → Q3
```

## Ⅲ. 융합 비교

### 카운터 유형 비교

| 타입 | 클럭 사이클 | 방향 | 리셋 | 응용 |
|------|-----------|------|------|------|
| 업카운터 | 매 | 0→MAX | MAX→0 | 일반 카운터 |
| 다운카운터 | 매 | MAX→0 | 0→MAX | 타이머, 프로그램 |
| 리플 카운터 | 매 | U/D 제어 | 해당 없음 | 양방향 카운터 |
| BCD 카운터 | 매 | 0→9 → 0 | 9→0 | 디스플레이 |
| 모듈러 | 매 | 프로그램 | 프로그램 | PLL |

### 카운터 복잡도

| 비트 수 | 플롭 수 | 주파수 분주비 | 카운터 값 범위 | 전력 |
|--------|---------|----------------|---------------|------|
| 4비트 | 4 | 1:16 | 0-15 | 4T 플롭 |
| 8비트 | 8 | 1:256 | 0-255 | 8T 플롭 |
| 16비트 | 16 | 1:65536 | 0-65535 | 16T 플롭 |
| 32비트 | 32 | 1:4G | 0-4G | 32T 플롭 |

## Ⅳ. 실무 적용

### 32비트 카운터

```
32비트 업카운터 (0 ~ 4,294,967,295):
CLK ──→ 32개 T 플롭

최대 카운트: 2^32 - 1 = 4,294, 967, 295
오버플로(Overflow): 2^32 → 0
```

## Ⅴ. 기대효과 및 결론

카운터는 주파수 분주기, 타이머에 필수적이다.

## 📌 관련 개념 맵

```
카운터
├── 업카운터 (0 → MAX → 0)
├── 다운카운터 (MAX → 0 → MAX)
├── 리플 카운터 (Up/Down 선택)
└── 응용
    ├── 주파수 분주기
    ├── 타이머
    └── RTC
```

## 👶 어린이를 위한 3줄 비유 설명

1. 카운터는 클럭 횟수마다 숫자가 1씩 올라가거나 줄어드는 숫자 세는 장치예요
2. 시계를 생각하면 1초마다 1씩 올라가는 초침기와 비슷해요, 클럭이 1Hz면 카운터가 1Hz로 카운트해요
3. CPU 내부의 프로그램 카운터는 특정 횟수로 클럭을 나누어 각 부품이 작동 속도를 제어할 수 있게 해줘요
