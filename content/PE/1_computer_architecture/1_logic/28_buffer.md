# 버퍼 (Buffer)

## 핵심 인사이트 (3줄 요약)
1. 버퍼는 입력을 그대로 출력하지만 전류 구동 능력을 향상시키는 회로로, Y = A로 표현되며 신호 복원/완충에 사용된다
2. CMOS 버퍼는 2개 인버터를 직렬 연결하여 낮은 출력 저항과 높은 팬아웃을 제공하며, 클럭 분배(Clock Distribution)에 필수적이다
3. 기술사 시험에서는 버퍼의 전류 구동 능력, 전송 지연, 팬아웃 증가, 클럭 트리 설계가 핵심이다

## Ⅰ. 개요 (500자 이상)

버퍼(Buffer)는 입력 신호를 논리적으로 반전 없이 그대로 출력하지만, 전류 구동 능력(Current Drive Capability)을 향상시키는 회로이다. 기능적으로는 Y = A로 동일하나, **신호 복원(Signal Restoration)**, **완충(Buffering)**, **임피던스 매칭(Impedance Matching)** 역할을 수행한다.

```
ANSI 기호          IEC 기호
    ┌─┐              ┌───┐
A ──┤  │          A ─┤   ├─── Y
    │   │─── Y         │   │
    └─┘              └───┘
    (삼각형)           (1을 표시)
```

버퍼의 핵심 기능:
1. **전류 구동**: 낮은 출력 저항으로 높은 부하 용량 구동
2. **신호 완충**: 긴 전송선 또는 다수 부하 구동 시 신호 감쇠 방지
3. **임피던스 매칭**: 50Ω 전송선에 연결 시 반사 방지
4. **클럭 분배**: PLL 또는 클럭 소스에서 다수 플립플롭으로 클럭 전달

CMOS 버퍼는 실제로 **2개 인버터를 직렬 연결**한 구조이다.

```
A ── INV ── INV ── Y
   (제1단)   (제2단)
```

첫 번째 인버터가 신호를 반전하고, 두 번째 인버터가 다시 반전하여 원래 논리 레벨을 복원한다. 그러나 각 인버터는 낮은 출력 저항(수kΩ)을 가지며, 직렬 연결 시 최종 출력 저항이 더 낮아져 높은 전류 구동 능력을 제공한다.

## Ⅱ. 아키텍처 및 핵심 원리 (1000자 이상)

### CMOS 버퍼 구조

```
CMOS 버퍼 (2 인버터 직렬):
    V_DD              V_DD
     │                 │
   ┌─┴─┐             ┌─┴─┐
   │   │             │   │
 PMOS PMOS         PMOS PMOS
   │   │             │   │
A ─┤   ├─┬─── X ────┤   ├─┬─── Y
   │   │ │           │   │ │
   └─┬─┘ │           └─┬─┘ │
     │   │             │   │
   NMOS NMOS         NMOS NMOS
     │   │             │   │
    GND GND           GND

제1단 출력: X = A'
제2단 출력: Y = X' = A'' = A (복원)
```

### 출력 저항 및 전류 구동 능력

```
단일 인버터 출력 저항:
R_out(1) = 1 / g_m1 ≈ 10kΩ

버퍼(2 인버터 직렬) 출력 저항:
R_out(2) ≈ 1 / (2 × g_m) ≈ 5kΩ (이론적, 실제는 더 낮음)

전류 구동 능력:
I_OL = V_DD / R_out

단일 인버터:
I_OL ≈ 5V / 10kΩ = 0.5mA

버퍼:
I_OL ≈ 5V / 5kΩ = 1mA (2배 향상)
```

실제 CMOS 버퍼(74HC07, 74HC125)는 **전류 증폭(Current Amplification)** 트랜지스터를 추가하여 I_OL = 4-8mA를 제공한다.

```
전류 증폭 버퍼 회로:
    V_DD
     │
    ┌─┴─┐
    │   │
   PMOS (Pull-up)
    │   │
A ─┤   ├─┬─── Y
    │   │ │
   └─┬─┘ │
     │   │
   NMOS (Pull-down)
     │   │
    GND

트라이스터 게이트를 증폭하여 V_GS를 높임 → R_DS(on) 감소 → 전류 증가
```

### 전송 지연

```
t_p(buffer) = t_p(inv1) + t_p(inv2)
            ≈ 2 × t_p(single inverter)

예) 74HC07: t_p ≈ 10ns (인버터 2개 직렬)
```

## Ⅲ. 융합 비교 및 다각도 분석

### 버퍼 vs 인버터 vs 일반 게이트

| 비교 항목 | 버퍼 | 인버터 | 일반 게이트(AND/OR) |
|----------|------|--------|---------------------|
| 논리 기능 | Y = A (복원) | Y = A' (반전) | Y = A·B, A+B |
| 출력 저항 | 낮음 (5-10Ω) | 중간 (수kΩ) | 중간 (수kΩ) |
| 전류 구동 | 높음 (4-8mA) | 중간 (1-2mA) | 중간 (1-2mA) |
| 전송 지연 | 2× t_p | 1× t_p | 1× t_p |
| 팬아웃 | 높음 (>50) | 중간 (10-50) | 중간 (10-50) |
| 용도 | 신호 완충, 구동 | 반전 | 논리 연산 |

### 버퍼 유형 비교

| 버퍼 유형 | 구조 | I_OL | 응용 |
|----------|------|------|------|
| 일반 버퍼 | 2 인버터 직렬 | 1-2mA | 신호 복원 |
| 전류 증폭 버퍼 | 트랜지스 증폭 | 4-8mA | 고전류 구동 |
| 3상태 버퍼 | Enable 핀 추가 | 4-8mA (활성), High-Z (비활성) | 버스 아비트릭 |
| 라인 버퍼 | 50Ω 종단 저항 | 높음 | 전송선 구동 |
| 클럭 버퍼 | 저지상(Phase Alignment) | 중간 | 클럭 분배 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 클럭 분배 트리(Clock Distribution Tree)

CPU의 클럭 소스에서 다수 플립플롭(Flip-Flop)으로 클럭을 분배할 때 버퍼를 사용한다.

```
PLL/XTAL ── 클럭 버퍼 ──┬── H-Tree ──┬─ FF
                          │           │
                          └───┬─ FF
                              │
                              └─ FF

버퍼 기능:
1. 클럭 파형 복원 (Edge sharpening)
2. 클럭 스큐(Skew) 최소화
3. 클럭 드라이브 능력 향상
```

### 데이터 버스 구동

```
CPU ──┬── 데이터 버스 (8/16/32/64비트)
     │
     └── 버퍼(8개) ── 메모리/주변 장치

버퍼가 용량성 부하(C_L)를 구동하여 신호 왜곡 방지
```

### 기술사 시험 대비 문제

**문제**: CMOS 인버터 2개 직렬 버퍼의 t_p=5ns, 단일 인버터의 R_out=10kΩ일 때, 버퍼의 전류 구동 능력을 계산하시오.

**해설**:
```
1) 단일 인버터 전류:
I_OL(1) = V_DD / R_out = 5V / 10kΩ = 0.5mA

2) 버퍼(2 인버터) 전류:
R_out(2) ≈ R_out(1) / 2 = 5kΩ
I_OL(2) = V_DD / R_out(2) = 5V / 5kΩ = 1mA

따라서 버퍼는 2배 전류 구동 능력 향상
```

## Ⅴ. 기대효과 및 결론

버퍼는 신호 완충, 전류 구동 향상에 필수적이다.

## 📌 관련 개념 맵

```
버퍼 (Buffer)
├── 정의: Y = A (논리 복원)
├── 구조
│   ├── 2 인버터 직렬
│   └── 전류 증폭 트랜지스터
├── 파라미터
│   ├── 출력 저항: 낮음 (5-10Ω)
│   ├── 전류 구동: 높음 (4-8mA)
│   └── 전송 지연: 2× t_p(inv)
└── 응용
    ├── 클럭 분배
    ├── 데이터 버스 구동
    └── 3상태 버퍼 (Hi-Z)
```

## 👶 어린이를 위한 3줄 비유 설명

1. 버퍼는 전기를 물탱크에 담는 것처럼, 나중에 많은 장치에 공급할 수 있도록 전기 신호를 모아주는 장치예요
2. 입력 신호를 그대로 출력하지만, 더 튼튼하게 해서 여러 곳에 동시에 보낼 수 있어요
3. CPU의 클럭 신호를 여러 칩으로 나누어 보낼 때 버퍼가 사용돼서 모든 칩이 동시에 작동할 수 있어요
