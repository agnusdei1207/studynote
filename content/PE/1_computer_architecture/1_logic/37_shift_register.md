# 레지스터 (Register)

## 핵심 인사이트 (3줄 요약)
1. 레지스터는 여러 플립플롭을 병렬로 배치하여 n비트 데이터를 저장하는 순차 회로로, CLK에서 데이터를 로드(Write)하고, 출력 항상을 통해 데이터를 읽는(Read)다
2. 일반 레스터(General Purpose Register), 특수 레지스터(PC, SP, IR), 시프 레지스터가 있으며, 8/16/32/64비트가 일반적이다
3. 기술사시험에서는 레지스터의 로드/읽기 타이밍, 쓰기/읽기 제어, 클럭 도메인, 파이프라인 레지스터가 핵심이다

## Ⅰ. 개요 (500자 이상)

레지스터(Register)는 **n비트 데이터를 저장하는 순차 회로**로, 일반 레지터(General Purpose Register)와 특수 레지스터(Special Purpose Register)로 분류된다. CPU 내에서 레지스터는 데이터를 임시 저장하여 연산에 사용한다.

```
8비트 레지터:
D[7:0] ──┬─┬─┬─┬─┬─┬─┬─┬─ 8개 D 플립롭
           │ │ │ │ │ │ │ │
CLK ──────┴─┴─┴─┴─┴─┴─→ 모든 CLK에 연결
           │
Q[7:0] ──┬─┬─┬─┬─┬─┬─┬─ 8개 Q 출력
           │ │ │ │ │ │ │ │

Write_Control → 모든 CLK에 AND 게이트로 쓰기 제어
Read_Enable → Q 출력 활성화
```

**일반 레지스트(General Purpose Register)**는 산술/논리 연산에 자유로 사용되는 레지스터이다. ARM, x86 프로세서에는 수십 개의 일반 레지스터가 있다(R0-R15, 또는 X0-X31).

**특수 레지스터**는 고유한 기능을 가진다:
- **PC(Program Counter)**: 다음 명령어 주소를 가리킴
- **SP(Stack Pointer)**: 스택 메모리 주소를 가리킴
- **IR(Instruction Register)**: 현재 실행 중인 명령어를 저장
- **MAR(Memory Address Register)**: 접근할 메모리 주소 저장
- **MDR(Memory Data Register)**: 메모리에서 읽은 데이터 저장
- **AC(Accumulator)**: 연산 결과 누적

레지스터 파일(Register File)은 CPU 내의 가장 빠른 메모리로, 클럭 사이클마다 새로운 데이터를 로드하거나 결과를 저장한다. 레지터의 폭(width)은 데이터 폭을 결정하며, 8비트, 16비트, 32비트, 64비트가 일반적이다.

```
32비트 레지스트(R0):
┌─────────────────────────────┐
│   R31  R30  ...  R1  R0     │
│   (32비트 데이터)              │
└─────────────────────────────┘

RD/WR 제어로 데이터 로드/저장
```

## Ⅱ. 아키텍처 및 핵심 원리 (1000자 이상)

### 레지스터의 읽기/쓰기 타이밍

```
레지스터 타이밍:
설정 시간(Setup time): 데이터가 클럭 에지 전에 안정되어야 함
유지 시간(Hold time): 데이터가 클럭 에지 후에 일정 시간 유지되어야 함

파라미터:
t_setup = D 입력 → Q 출력
t_hold = D → Q 유지
```

### 쓰기/읽기 제어

```
Write_Enable (WE):
WE=1: D → Q (CLK 상승 에지에서)
WE=0: Q 유지

Read_Enable (RE):
RE=1: Q → 출력 버스로 전달
RE=0: 고임피던스(High-Z)
```

### 시프 레지스터(Shift Register)

```
8비트 시프 레지터:
SRL (Shift Right Logical), SLL (Shift Left Logical)

SRL=1: Q[n] = Q[n+1] (우측으로 1비트 시프)
SLL=1: Q[n] = Q[n-1] (좌측으로 1비트 시프)

회로:
Q7 Q6 Q5 Q4 Q3 Q2 Q1 Q0
├──┴──├──├──├──├──┼──→ 시프
SRL=1: Q7=Q6, Q6=Q5, ..., Q0=X

- 회전(Rotate) ROL/ROR
- 산술 논리 시프
- 논리 시프
```

## Ⅲ. 융합 비교

### 레지스터 vs 래치

| 비교 항목 | 레지스터 | 래치 |
|----------|---------|------|
| 크기 | 8/16/32/64비트 | 1비트 |
| 기능 | 데이터 저장, 로드/읽기 | 입력 유지만 |
| 클럭 | 필요 | 필요 (Enable=1) |
| 응용 | 일반 목적 | SRAM 셀 |

### 레지스터 구현 비교

| 구현 | 복잡도 | 속도 | 비용 |
|------|-------|------|------|
| D 플롭 | 낮음 | 빠름 | 일반 레지스터 |
| JK 플롭 | 높음 | 느림 | 특수 응용 |
| T 플롭 | 낮음 | 빠름 | 카운터 |
| SRAM 셀 | 중간 | 빠름 | 고밀도 메모리 |

## Ⅳ. 실무 적용

### 32비트 레지터 파일

```
R0 EAX (Accumulator): 연산 결과 저장
R1 ECX (Count): 카운터/인덱스 레지터
R2 EDX (Data): 데이터 저장
R3 EBX (Base Pointer): 기준 주소
R4 ESP (Stack Pointer): 스택 포인터
R5 EBP (Base Pointer): 스택 프레임 베이스
R6 ESI (Source Index): 소스 인덱스
R7 EDI (Destination Index): 목적지 인덱스
```

## Ⅴ. 기대효과 및 결론

레지스터는 CPU의 워킹 메모리이다.

## 📌 관련 개념 맵

```
레지스터
├── 일반 레지스터 (R0-R15)
├── 특수 레지스터
│   ├── PC (Program Counter)
│   ├── SP (Stack Pointer)
│   └── IR (Instruction Register)
└── 시프 레지스트
    ├── 논리 시프
    └── 회전(Rotate)
```

## 👶 어린이를 위한 3줄 비유 설명

1. 레지스터는 여러 개의 숫자를 한 번에 기억해서 필요할 때 꺼내 쓰는 전자 칠판 같아요
2. CPU 안에 수십 개의 레지스터가 있는데, 각각은 32비트 또는 64비트 데이터를 저장하고 처리에 사용돼요
3. 컴퓨터가 계산한 결과를 레지스터에 저장하고 나중에 다시 사용할 수 있도록 레지스터가 데이터를 잠시 보관해줘요
