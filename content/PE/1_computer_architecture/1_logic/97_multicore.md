# 멀티코어 (Multicore)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멀티코어는 하나의 칩(Die) 내에 여러 개의 독립적인 CPU 코어를 통합하는 프로세서 아키텍처로, SMP 개념을 단일 칩 내부로 축소하여 구현한 것이며 공유 L3 캐시와 온칩 인터커넥트가 핵심 구성요소다
> 2. **가치**: 클럭 속도 증가 한계를 극복하여 성능 향상과 전력 효율을 동시에 달성하며, 코어 간 통신 지연이 매우 낮아 스레드 레벨 병렬성(TLP) 활용에 최적화되어 있다
> 3. **융합**: CMP(Chip Multi-Processor), SMP, SMT와 연관되며, 이기종 멀티코어(big.LITTLE), GPU 통합(APU), AI 가속기 통합(NPU) 등으로 진화하고 있다

---

## Ⅰ. 개요 (Context & Background)

### 개념
멀티코어(Multicore) 프로세서는 하나의 반도체 다이(Die) 또는 패키지 내에 두 개 이상의 독립적인 CPU 코어를 집적한 프로세서를 말한다. 각 코어는 독립적인 L1 캐시와 실행 유닛을 가지며, 일반적으로 L2 또는 L3 캐시와 시스템 인터페이스를 공유한다. 이는 기존의 멀티프로세서 시스템(MP)을 단일 칩 내부에 구현한 것으로, CMP(Chip Multi-Processor)라고도 한다. 멀티코어의 핵심 목표는 클럭 속도 증가 없이 병렬 처리를 통해 성능을 향상시키는 동시에, 전력 소모를 줄이는 것이다.

### 💡 비유
멀티코어는 **"하나의 건물 안에 여러 사무실이 있는 회사"**에 비유할 수 있다.
- **단일 코어 (1인 사무실)**: 한 명이 모든 일을 처리, 일이 많으면 야근
- **멀티코어 (다인 사무실)**:
  - 각 사무실(코어)마다 직원이 따로 있음
  - 복도(인터커넥트)를 통해 빠르게 소통
  - 회의실(L3 캐시)을 공유하여 자원 효율화
- **이기종 멀티코어 (big.LITTLE)**: 부장님 사무실 + 인턴 사무실

### 등장 배경 및 발전 과정

#### 1. 기존 기술의 치명적 한계점
- **전력 벽(Power Wall)**: 클럭 속도 증가 시 전력 소모가 P ∝ f³으로 폭증
- **발열 문제**: 고클럭 CPU의 냉각 비용 급증
- **ILP 한계**: 수퍼스칼라, 비순차 실행으로 추출 가능한 ILP가 2-3에 불과
- **메모리 월**: CPU 속도 대비 메모리 속도 격차 심화

#### 2. 패러다임 변화와 혁신
- **2001년 IBM POWER4**: 최초의 상용 듀얼코어 프로세서
- **2005년 Intel Pentium D / AMD Athlon 64 X2**: x86 듀얼코어 대중화
- **2006년 Intel Core 2 Duo**: 공유 L2 캐시 도입
- **2011년 Intel Sandy Bridge**: 4코어 + 통합 GPU
- **2017년 AMD Ryzen**: 칩렛(Chiplet) 기반 멀티코어
- **현재**: 64코어+ 서버 CPU, 8코어+ 모바일 AP

#### 3. 비즈니스적 요구사항
- **모바일 배터리**: 저전력 코어로 배터리 수명 연장
- **데이터센터 TCO**: 높은 처리량 대비 전력 효율
- **AI/ML 추론**: 다중 코어 + 가속기 통합

### 심층 동작 원리

#### 1. 코어 구조 (Private 자원)

**L1 캐시 (Private)**:
- L1 I-Cache: 32KB, 명령어 전용
- L1 D-Cache: 32-48KB, 데이터 전용
- Latency: 4-5 cycles

**L2 캐시 (Private 또는 Shared)**:
- 크기: 256KB - 2MB per core
- Unified (명령어 + 데이터)
- Latency: 12-15 cycles

**실행 유닛**:
- ALU, FPU, Load/Store Unit
- 수퍼스칼라 발급 (4-6 uop/cycle)

#### 2. 공유 L3 캐시 (LLC)

**동작 원리**:
- Cache Slicing: 각 코어에 가까운 슬라이스 우선 사용
- Inclusive vs Exclusive: L1/L2를 포함하는지 여부
- Directory: 캐시 일관성 추적

**지연 시간**:
- Local Slice: 30-40 cycles
- Remote Slice: 50-70 cycles (Ring hop 추가)

#### 3. 캐시 일관성 (On-Chip)

**MESI 프로토콜 최적화**:
- Directory-based: 스펙트레/멜트다운 이후 스누핑 감소
- Snoop Filter: 불필요한 스누프 차단
- MOESI/O: 공유 수정 상태 추가로 캐시 간 전송 최적화

#### 4. 전력 관리

**코어 레벨 전원 제어**:
| 상태 | 설명 | 전력 | 복귀 시간 |
|------|------|------|----------|
| C0 | 활성 | 100% | - |
| C1 | Halt | ~10% | ~100ns |
| C6 | Deep Power Down | <1% | ~1μs |
| C7/C8/C9 | 더 깊은 슬립 | <0.5% | ~10μs |

**DVFS (Dynamic Voltage Frequency Scaling)**:
- P-State: 주파수/전압 레벨
- Turbo Boost: 짧은 시간 클럭 상향
- Speed Shift: OS 직접 제어

## Ⅲ. 융합 비교 및 다각도 분석

### 심층 기술 비교표

| 특성 | 듀얼코어 | 쿼드코어 | 8코어+ | 이기종 |
|------|----------|----------|--------|--------|
| **일반 워크로드** | 1.8x | 3.2x | 5-6x | 4-5x |
| **병렬 워크로드** | 1.9x | 3.6x | 6-7x | 3-4x |
| **전력 효율** | 좋음 | 중간 | 낮음 | 매우 좋음 |
| **면적 비용** | 낮음 | 중간 | 높음 | 중간 |
| **소프트웨어 요구** | 낮음 | 중간 | 높음 | 매우 높음 |

### 멀티코어 vs 멀티스레딩 (SMT)

| 측면 | 멀티코어 | SMT (Hyper-Threading) |
|------|----------|----------------------|
| **실행 유닛** | 완전히 분리 | 공유 |
| **L1 캐시** | 분리 | 공유 또는 분리 |
| **성능 향상** | 거의 선형 | 20-30% |
| **전력 증가** | 비례 | 5-10% |
| **면적 증가** | 큼 | 매우 작음 |

### 과목 융합 관점 분석

#### 1. OS ↔ 멀티코어: 스케줄링

- **Load Balancing**: 코어 간 부하 분산
- **Affinity**: 캐시 지역성 유지를 위한 코어 고정
- **Energy-aware Scheduling**: 이기종 코어 선택

#### 2. 컴파일러 ↔ 멀티코어: 병렬화

- **Auto-parallelization**: 루프 자동 병렬화
- **OpenMP**: 스레드 병렬 처리 지시어
- **Vectorization**: SIMD 활용

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오 1: 서버 CPU 선택
**상황**: 가상화 호스트용 CPU 선택
**판단**:
- 높은 vCPU 밀도 필요 → 32-64코어
- 메모리 대역폭 중요 → 다채널 메모리
- NUMA 고려 → OS/앱 NUMA 인식

#### 시나리오 2: 모바일 AP 설계
**상황**: 스마트폰용 AP 코어 구성
**판단**:
- 배터리 수명 중요 → big.LITTLE
- 게임 성능 → 1-2개 big 코어 활성
- 대기 시간 → LITTLE 코어만 활성

### 도입 시 고려사항 (체크리스트)

**기술적**:
- [ ] 워크로드 병렬화: 스레드 레벨 병렬성 활용 가능?
- [ ] 메모리 대역폭: 코어 수에 비례하는 대역폭?
- [ ] 열 설계: 코어 밀도에 따른 발열 관리?

**운영적**:
- [ ] 소프트웨어 준비: 멀티스레드 프로그래밍 적용?
- [ ] 라이선스: 코어 기반 라이선스 비용?

### 주의사항 및 안티패턴

1. **Amdahl의 법칙**: 직렬 부분이 성능 향상을 제한
2. **캐시 경합**: 코어 간 캐시 공유로 인한 성능 저하
3. **메모리 병목**: 코어 수 증가 대비 메모리 대역폭 부족

## Ⅴ. 기대효과 및 결론

### 정량적/정성적 기대효과

| 지표 | 듀얼코어 | 쿼드코어 | 8코어 |
|------|----------|----------|-------|
| 처리량 | 1.8-1.9x | 3.2-3.6x | 5.5-6.5x |
| 전력/성능 | 기준 | -10% | -15% |
| 면적 | 1.8x | 3x | 4x |
| 비용/성능 | 기준 | +10% | +20% |

### 미래 전망 및 진화 방향

1. **칩렷(Chiplet)**: 여러 다이를 패키지로 통합 (AMD, Intel)
2. **3D 적층**: 코어와 캐시/메모리 수직 적층
3. **가속기 통합**: CPU + GPU + NPU + FPGA 하이브리드
4. **CXL 메모리**: 확장 가능한 메모리 풀링

### ※ 참고 표준/가이드
- **Intel Architecture Instruction Set Extensions**: 코어 수 쿼리
- **ACPI**: 코어 토폴로지 정보
- **RISC-V multicore**: 오픈 표준 멀티코어

## 👶 어린이를 위한 3줄 비유 설명

멀티코어는 **"하나의 건물에 여러 사무실이 있는 회사"**예요.

1. **각자 다른 일**: 사무실마다 다른 직원이 있어요. 한 직원이 계산할 때, 다른 직원은 파일을 정리할 수 있어요.

2. **빠른 소통**: 복도로 금방 다른 사무실에 갈 수 있어요. "이거 좀 도와줘!" 하고 말 걸면 바로 도와줄 수 있죠.

3. **큰 회의실 공유**: 모든 사무실이 큰 회의실(L3 캐시)을 같이 써요. 자주 쓰는 자료를 회의실에 두면 누구나 빌려볼 수 있어요!
