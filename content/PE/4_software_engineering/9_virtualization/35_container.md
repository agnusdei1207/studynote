# 컨테이너 (Container)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컨테이너는 **"애플리케이션 **실행 **환경**을 **격리**하고 **패키징**하는 **기술"**으로, **VM**(하이퍼바이저)와 **달리 **Host OS **커널**을 **공유**하여 **가볍**고 **빠르다**.
> 2. **기술**: **Linux Namespace**(PID, **NET**, **MNT** 격리)와 **Cgroups**(자원 **제한)로 **구현**되며 **Union Filesystem**(OverlayFS)로 **이미지 **계층**을 **관리**한다.
> 3. **오케스트레이션**: **Kubernetes**가 **Pod**, **Service**, **Deployment**로 **컨테이너 **배포**를 **자동화**하고 **Docker Compose**, **Docker Swarm**이 **소규모 **환경**을 **지원**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
컨테이너는 **"OS 수준 가상화"**이다.

**VM vs Container**:
| 구분 | VM | Container |
|------|----|-----------|
| **격리** | HW level | OS level |
| **커널** | 독립 | 공유 |
| **용량** | GB | MB |
| **시작** | 분 | 초 |

### 💡 비유
- **VM**: **"독립 **집 ****(모든 **것 **포함)
- **Container**: **"아파트 **호수 ****(공유 **시설 + **독립 **공간)

### Namespace & Cgroups

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Container Isolation                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Namespaces (Isolation):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Namespace Type   │  Isolates                      │  Example                          │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────│  │
    │  PID Namespace   │  Process IDs                   │  Container PID 1 = Host PID 1234   │  │
    │  NET Namespace   │  Network stack                 │  Separate network interfaces       │  │
    │  MNT Namespace   │  Mount points                  │  Separate filesystem view          │  │
    │  UTS Namespace   │  Hostname & domain             │  Container sees different hostname │  │
    │  IPC Namespace   │  Inter-Process Communication   │  Isolated message queues          │  │
    │  USER Namespace  │  User & Group IDs              │  Root in container ≠ root on host  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Cgroups (Resource Limits):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  /sys/fs/cgroup/memory/mycontainer/                                                      │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  memory.limit_in_bytes = 536870912  # 512 MB RAM                                   │  │  │
    │  │  memory.memsw.limit_in_bytes = 1073741824  # 1 GB RAM + Swap                       │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  /sys/fs/cgroup/cpu/mycontainer/                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  cpu.cfs_quota_us = 50000        # 50% of 1 CPU core                                │  │  │
    │  │  cpu.cfs_period_us = 100000      # 100ms period                                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  /sys/fs/cgroup/blkio/mycontainer/                                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  blkio.throttle.read_bps_device = "8:0 10485760"  # 10 MB/s disk read                │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

```

```
│                         Dockerfile                                                        │

    │                                                                                         │  │
    │  # Working directory                                                                     │  │
    │  WORKDIR /app                                                                           │  │
    │                                                                                         │  │
    │  # Install dependencies                                                                  │  │
    │  COPY requirements.txt .                                                                 │  │
    │  RUN pip install --no-cache-dir -r requirements.txt                                      │  │
    │                                                                                         │  │
    │  # Copy application code                                                                 │  │
    │  COPY . .                                                                                │  │
    │                                                                                         │  │
    │  # Create non-root user                                                                  │  │
    │  RUN useradd -m -u 1000 appuser                                                          │  │
    │  USER appuser                                                                            │  │
    │                                                                                         │  │
    │  # Expose port                                                                           │  │
    │  EXPOSE 8080                                                                             │  │
    │                                                                                         │  │
    │  # Run command                                                                           │  │
    │  CMD ["python", "app.py"]                                                               │  │
    │                                                                                         │  │
    │  → Layered filesystem: Each RUN/COPY/ADD creates a layer                                 │  │
```

### Union Filesystem (OverlayFS)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         OverlayFS Layers                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Container View (Merged)                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  /bin     /etc     /usr     /app     /data                                           │  │  │
    │  │   │        │        │        │         │                                             │  │
    │  │   ▼        ▼        ▼        ▼         ▼                                             │  │  │
    │  │  All layers merged into single view                                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Actual Layers:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  [Upper Layer - Writable]                                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  /app/main.py (modified)                                                             │  │  │
    │  │  /data/                                                                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │ (Overlay: hides lower layers)                                                │  │
    │            ▼                                                                              │  │
    │  [Layer 3 - Application]                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  /app/main.py                                                                         │  │  │
    │  │  /app/requirements.txt                                                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                              │  │
    │            ▼                                                                              │  │
    │  [Layer 2 - Python Runtime]                                                            │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  /usr/bin/python3                                                                     │  │  │
    │  │  /usr/lib/python3.11/                                                                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                              │  │
    │            ▼                                                                              │  │
    │  [Layer 1 - Base OS]                                                                    │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  /bin, /etc, /lib (Debian slim)                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Layers are cached and reused                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### Docker Compose

```yaml
# docker-compose.yml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "8080:8080"
    environment:
      - DATABASE_URL=postgres://db:5432/app

      - db

    image: postgres:15

      - POSTGRES_DB=app

  redis:
    image: redis:7-alpine
    restart: unless-stopped

volumes:
  db_data:
```

### Kubernetes 예시

```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: web
        image: myapp:1.0
        ports:
        - containerPort: 8080
        resources:
          requests:
            memory: "128Mi"
            cpu: "100m"
          limits:
            memory: "256Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 5

---
# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: web-service
spec:
  selector:
    app: web
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8080
  type: LoadBalancer
```

## Ⅴ. 기대효과 및 결론

### 컨테이너 기대 효과

| 효과 | VM | Container |
|------|----|-----------|
| **시작 시간** | 분 | 초 |
| **이미지 크기** | GB | MB |
| **격리** | 강함 | 중간 |
| **이식성** | 낮음 | 높음 |

### 모범 사례

1. **이미지**: Alpine base (작음)
2. **보안**: Non-root user
3. **Health Check**: Liveness/Readiness
4. **리소스**: Limit 설정

### 미래 전망

1. **WebAssembly**: 더 가벼운 컨테이너
2. **unikernel**: 전용 OS 이미지
3. **Serverless**: FaaS + Containers
