# 포매터 (Formatter)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 포매터는 **"소스 **코드 **스타일**을 **자동으로 **정리**하는 **도구"**로, **린터**(Linter)가 **코드 **품질**을 **검사**하는 **반면** **포매터**는 **코드 **모양**을 **수정**하고 **Consistent**(일관성)을 **보장**하며 **Auto-fix**(자동 수정) 기능을 **제공**한다.
> 2. **가치**: **Code Review**에서 **스타일 **논쟁**을 **줄이고 **프로젝트** **전체**에 **일관된 **코딩 **스타일**을 **적용**하며 **IDE Integration**으로 **저장** 시 **자동 **포맷**이 **가능**하다.
> 3. **융합**: **JavaScript**(Prettier), **Python**(Black, Autopep8), **Go**(gofmt), **C/C++**(clang-format), **Java**(Google Java Format)가 **대표적**이며 **EditorConfig**로 **통합**된 **설정**을 **공유**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
포매터는 **"코드 스타일 정리 도구"**이다.

**포맷 대상**:
- **Indentation**: 들여쓰기
- **Spacing**: 공백
- **Line Break**: 줄바꿈
- **Alignment**: 정렬

### 💡비유
포매터는 **"자동 세탁 기계****와 같다.
- **빨래**: 코드
- **정리**: 포맷
- **기준**: 스타일 가이드

### 포매터 파이프라인

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Formatting Pipeline                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Source Code                                                                           │  │
    │    │                                                                                   │  │
    │    ▼                                                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Parser (AST 생성)                                                                │  │  │
    │  │    Code → AST                                                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │    │                                                                                   │  │
    │    ▼                                                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Formatter (Style 적용)                                                           │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • indent: 2 spaces                                                             │  │  │  │
    │  │  │  │  • quotes: single                                                               │  │  │  │
    │  │  │  │  • semi: always                                                               │  │  │  │
    │  │  │  │  → Styled Code                                                               │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │    │                                                                                   │  │
    │    ▼                                                                                   │  │
    │  Formatted Code                                                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Prettier 설정

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Prettier Configuration (.prettierrc)                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  {                                                                                     │  │
    │    "semi": true,          │  세미콜론 자동 추가                           │  │
    │    "singleQuote": true,    │  작은따옴표                                   │  │
    │    "tabWidth": 2,          │  탭 2칸 = 스페이스 2개                            │  │
    │    "useTabs": false,       │  탭 대신 스페이스 사용                           │  │
    │    "trailingComma": "es5",  │  ES5/Node 스타일에 콤마 추가                   │  │  │
    │    "printWidth": 80        │  줄 길이 80                                   │  │
    │  }                                                                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 저장 시 포맷
**상황**: 자동 포맷
**판단**:

```json
// .vscode/settings.json
{
  "editor.formatOnSave": true,
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[python]": {
    "editor.defaultFormatter": "ms-python.black-formatter"
  }
}
```
