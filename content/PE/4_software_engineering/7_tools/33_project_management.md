# 프로젝트 관리 도구 (Project Management Tools)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로젝트 관리 도구는 **"소프트웨어 **개발 **계획**을 **추적**하고 **관리**하는 **시스템"**으로, **이슈 **추적**(Issue Tracking), **스프린트 **관리**(Sprint Management), **버전 **계획**(Release Planning) **기능**을 **제공**한다.
> 2. **기능**: **Backlog**(미작업), **Sprint**(스프린트), **Kanban Board**(칸반 **보드), **Burndown Chart**(번다운 **차트), **Gantt Chart**(간트 **차트), **Timeline**(타임라인)으로 **진행 **상황**을 **시각화**한다.
> 3. **도구**: **Jira**(Atlassian), **GitHub Projects**, **GitLab Issues**, **Azure DevOps**, **Asana**, **Monday.com**, **Notion**이 **대표적**이며 **Agile/Scrum** **방법론**을 **지원**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
프로젝트 관리 도구는 **"개발 계획 추적 시스템"**이다.

**관리 항목**:
- **Epic**: 대규모 기능
- **Story**: 사용자 스토리
- **Task**: 작업 단위
- **Bug**: 결함
- **Subtask**: 하위 작업

### 💡 비유
프로젝트 관리 도구는 **"작업 **관리 **대장 ****과 같다.
- **작업**: 이슈
- **담당자**: Assignee
- **진행**: Status

## Ⅳ. 실무 적용 및 기술사적 판단

### Jira 워크플로우

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Jira Workflow                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  To Do ──→ In Progress ──→ In Review ──→ Done                                         │  │
    │    │             │                │                                                     │  │
    │    │             │                └─────→ Rejected ──→ To Do                            │  │
    │    │             │                                                                      │  │
    │    │             └─────→ Blocked ─────────────────→ In Progress (unblock)               │  │
    │    │                                                                                    │  │
    │    └─────→ Duplicate ──→ Done                                                           │  │
    │                                                                                         │  │
    │  Transitions:                                                                           │  │
    │  • To Do → In Progress: Assignee starts work                                            │  │
    │  • In Progress → In Review: Pull request created                                        │  │
    │  • In Review → Done: PR approved and merged                                             │  │
    │  • In Review → Rejected: PR needs changes                                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: Jira 대시보드 구성
**상황**: 스프린트 추적
**판단**:

```yaml
# Sprint Dashboard
sprint: "Sprint 24 (2026-03-06 ~ 2026-03-20)"

filters:
  - "project = MYPROJECT"
  - "sprint = Sprint 24"
  - "type in (Story, Task, Bug)"

columns:
  - "Key"
  - "Summary"
  - "Assignee"
  - "Status"
  - "Story Points"
  - "Priority"

quick_filters:
  - name: "Only My Issues"
    query: "assignee = currentUser()"
  - name: "High Priority"
    query: "priority in (High, Critical)"

# Burndown
velocity:
  last_3_sprints: [23, 19, 26]
  average: 22.7
  committed: 50
```

## Ⅴ. 기대효과 및 결론

### 프로젝트 관리 기대 효과

| 효과 | 없음 | 도구 |
|------|------|------|
| **가시성** | 낮음 | 높음 |
| **추적** | 어려움 | 쉬움 |
| **협업** | 산만 | 체계적 |

### 모범 사례

1. **정의**: 명확한 Definition of Done
2. **주기**: 매일 스탠드업
3. **회고**: 스프린트 회고
4. **업데이트**: 실시간 이슈 상태

### 미래 전망

1. **AI 추정**: 자동 Story Point
2. **자동화**: Git ↔ Issue 연동
3. **예측**: 리스크 조기 경보
