# 보안 도구 (Security Tools)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 보안 도구는 **"소프트웨어 개발 **수명 주기**에서 **보안 취약점**(Vulnerability)을 **자동으로 **검사**하고 **식별**하는 **도구 모음"**으로, **SAST**(Static Analysis), **DAST**(Dynamic Analysis), **SCA**(Software Composition Analysis), **Secret Scan**으로 **구분**된다.
> 2. **가치**: **수동 보안 검사**의 **비효율**을 **제거**하고 **CI/CD 파이프라인**에 **통합**(DevSecOps)하여 **코드 커밋 시점**에 **취약점**을 **조기 발견**하며 **OWASP Top 10**, **CVE**(Common Vulnerabilities and Exposures) 데이터베이스를 **활용**한다.
> 3. **융합**: **SAST 도구**(SonarQube, Semgrep, Checkmarx), **DAST 도구**(OWASP ZAP, Burp Suite), **SCA 도구**(Snyk, Dependabot, Trivy), **Secret Scan**(GitGuardian, gitleaks)가 **대표적**이며 **Shift Left** 전략으로 **개발 초기**에 **보안**을 **강화**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
보안 도구는 **"소프트웨어 보안 취약점을 찾는 자동화 도구"**이다.

**보안 도구 분류**:
- **SAST**: 소스 코드 정적 분석
- **DAST**: 실행 중 동적 분석
- **SCA**: 의존성 취약점 분석
- **Secret Scan**: 비밀 정보 노출 검사

### 💡 비유
보안 도구는 **"보안 검색기****와 같다.
- **SAST**: 청소 도구 (코드)
- **DAST**: 침입 테스트 (실행)
- **SCA**: 부품 검사 (라이브러리)
- **Secret Scan**: 열쇠 찾기 (비밀)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         보안 도구 발전                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Manual (1990년대):
    • 코드 리뷰
    • 수동 테스트
         ↓
SAST (2000년대):
    • 정적 분석
    • 자동화
         ↓
DevSecOps (2010년대~):
    • CI/CD 통합
    • Shift Left
```

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  SCA (Software Composition Analysis)                                                    │

    │  │  • CVE 데이터베이스 조회                                                               │  │

## Ⅲ. 융합 비교 및 다각도 분석

### 보안 도구 비교

| 도구 | 검사 대상 | 시점 | 장점 | 단점 |
|------|----------|------|------|------|
| **SAST** | 소스 코드 | 커밋 시 | 조기 발견 | False Positive |
| **DAST** | 실행 앱 | 스테이징 | 실제 취약점 | 늦은 발견 |
| **SCA** | 의존성 | 빌드 시 | CVE 식별 | Transitive 한계 |
| **Secret Scan** | 코드/Git | 푸시 시 | 노출 방지 | False Positive |

### Shift Left 전략

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Shift Left Security                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Traditional:                                   Shift Left:
    개발 ──────> 빌드 ──────> 테스트 ──────> 배포          개발 ──────> 빌드 ──────> 테스트 ──────> 배포
                    │                                            ↑
                    ▼                                            │
                 보안 테스트                                   보안 테스트
                 (늦은 발견)                                   (조기 발견)
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: CI/CD 보안 통합
**상황**: GitHub Actions
**판단**:

```yaml
# .github/workflows/security.yml

name: Security Scan

on:
  push:
    branches: [main]
  pull_request:

jobs:
  sast:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: SonarQube Scan
        uses: sonarsource/sonarqube-scan-action@master
        env:
          SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}

      - name: Semgrep Scan
        uses: returntocorp/semgrep-action@v1
        with:
          config: auto

  sca:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Snyk Test
        uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}

      - name: Trivy Scan
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: 'fs'
          scan-ref: '.'
          format: 'sarif'
          output: 'trivy-results.sarif'

  secret-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0

      - name: Gitleaks Scan
        uses: gitleaks/gitleaks-action@v2
```

## Ⅴ. 기대효과 및 결론

### 보안 도구 기대 효과

| 효과 | 도구 없음 | 도구 있음 |
|------|----------|----------|
| **취약점 발견** | 수동 | 자동 |
| **발견 시점** | 배포 후 | 개발 중 |
| **수정 비용** | 높음 | 낮음 |
| **False Positive** | 없음 | 있음 |

### 모범 사례

1. **Shift Left**: 개발 초기에 보안
2. **여러 도구**: SAST + DAST + SCA
3. **False Positive**: 튜닝 필요
4. **교육**: 개발자 보안 교육

### 미래 전망

1. **AI 보안**: ML 기반 탐지
2. **AST**: Application Security Testing 통합
3. **RASP**: Runtime Protection
