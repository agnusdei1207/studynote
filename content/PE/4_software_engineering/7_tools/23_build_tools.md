# 빌드 도구 (Build Tools)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 빌드 도구는 **"소스 코드를 **실행 가능한 **아티팩트로 **변환하는 **도구"**로, **Compile**(컴파일), **Link**(링크), **Package**(패키징) 과정을 **자동화**하고 **Dependency Management**(의존성 관리)를 제공한다.
> 2. **가치**: **복잡한 빌드 과정**을 **단순화**하고 **재현 가능한 **빌드**를 **보장**하며 **Transitive Dependency**(전이 의존성)를 **관리**하고 **CI/CD Pipeline**에서 **자동화**된다.
> 3. **융합**: **Java**(Maven, Gradle), **JavaScript**(Webpack, Vite, esbuild), **Python**(pip, poetry), **C/C++**(CMake, Meson)가 **언어별 **도구**를 **사용**하며 **Make**, **Ninja**는 **범용** **빌드 시스템**이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
빌드 도구는 **"소스 코드 → 실행 파일 변환 도구"**이다.

**빌드 단계**:
1. **Preprocess**: 전처리
2. **Compile**: 컴파일 (오브젝트 파일)
3. **Link**: 링크 (실행 파일)
4. **Package**: 패키징 (JAR, DLL)

### 💡 비유
빌드 도구는 **"자동화된 요리사"**와 같다.
- **재료**: 소스 코드
- **요리**: 빌드 과정
- **완제품**: 실행 파일

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         빌드 도구 발전                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Manual Build (1970년대):
    • 직접 컴파일 명령어
    • 복잡함, 실수
         ↓
Make (1977):
    • Makefile
    • 의존성 관리
         ↓
Ant (2000년대):
    • XML 기반
    • Java 빌드
         ↓
Modern Build Tools:
    • Maven (POM)
    • Gradle (DSL)
    • Webpack (Module Bundler)
```
