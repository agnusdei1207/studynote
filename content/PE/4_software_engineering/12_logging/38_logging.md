# 로깅 도구 (Logging Tools)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 로깅 도구는 **"애플리케이션 **이벤트**를 **기록**하고 **검색**하는 **시스템"**으로, **Log4j**, **Logback**(Java), **Winston**, **Zap**(Go), **Loguru**(Python)가 **애�리케이션 **내**로깅**을 **작성**하고 **ELK Stack**(Elasticsearch, Logstash, Kibana)가 **로그 **수집**하고 **분석**한다.
> 2. **구조**: **Structured Logging**(JSON **형식), **Log Levels**(DEBUG, **INFO**, **WARN**, **ERROR), **Context**(Trace **ID, **User **ID)로 **추적**하며 **Syslog**(RFC 5424)가 **표준 **프로토콜**이다.
> 3. **파이프라인**: **App** → **Collector**(Fluentd, **Logstash**) → **Storage**(Elasticsearch) → **Visualization**(Kibana/Grafana)로 **구성**되고 **Log Rotation**, **Retention Policy**, **GDPR **준수**가 **필수**다.

## Ⅱ. 아키텍처 및 핵심 원리

### Structured Logging

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Structured Log Format (JSON)                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Unstructured Log:                                                                         │  │
    │  2026-03-06 10:15:30 ERROR Failed to connect to DB                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  → Difficult to parse                                                                  │  │  │
    │  │  → Inconsistent format                                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Structured Log (JSON):                                                                     │  │
    ┌──────────────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  {                                                                                     │  │  │
    │  │    "timestamp": "2026-03-06T10:15:30Z",                                              │  │  │
    │  │    "level": "error",                                                                 │  │  │
    │  │  │  "message": "Failed to connect to database",                                            │  │  │
    │  │  │  "service": "payment",                                                             │  │  │
    │  │  │  "trace_id": "abc123",                                                             │  │  │
    │  │  │  "user_id": 456,                                                                 │  │  │
    │  │  │  "error": {                                                                      │  │  │
    │  │  │    "code": "ECONNREFUSED",                                                         │  │  │
    │  │  │    "host": "db.example.com"                                                      │  │  │
    │  │  │  }                                                                              │  │  │
    │  │  }                                                                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  → Easy to parse and query                                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

    │  ERROR   │  Error        │  Failure but app continues                                     │  │
    │  FATAL   │  Critical     │  App cannot continue                                         │  │

## Ⅳ. 실무 적용 및 기술사적 판단

### Logback 설정 (Java)

```xml
<!-- logback.xml -->
<configuration>
    <appender name="STDOUT" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/app.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
            <fileNamePattern>logs/app-%d{yyyy-MM-dd}.log</fileNamePattern>
            <maxFileSize>100MB</maxFileSize>
            <maxHistory>30</maxHistory>
            <totalSizeCap>10GB</totalSizeCap>
        </rollingPolicy>
        <encoder class="ch.qos.logback.classic.Encoder">
            <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <logger name="com.example" level="DEBUG">
        <appender-ref ref="FILE"/>
    </logger>

    <root level="INFO">
        <appender-ref ref="STDOUT"/>
        <appender-ref ref="FILE"/>
    </root>
</configuration>
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 로그 수집 파이프라인
**상황**: 다중 앱 로그 중앙화
**판단**: Fluentd + S3

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Log Collection Pipeline                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Apps                                                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  App 1 (Go) → structured JSON → stdout → Fluentd                                  │  │  │
    │  │  App 2 (Java) → Log4j → file → Filebeat                                                  │  │  │
    │  │  App 3 (Python) → logging → file → Fluentd                                                │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                                 │  │
    │            ▼                                                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Fluentd (Aggregation)                                                                 │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Filter: Remove sensitive data (password, token, card)                               │  │  │  │
    │  │  │  Enrich: Add hostname, geoip, user info                                             │  │  │  │
    │  │  │  Buffer: Batch 1000 records or 30 seconds                                          │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                                 │  │
    │            ▼                                                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Output: S3 (s3://logs-*/year=2026/month=03/day=06/)                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                                 │  │
    │            ▼                                                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Lambda: Trigger on S3 put → Transform → Elasticsearch                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 모범 사례

1. **구조화**: JSON 형식
2. **문맥**: 항상 동일한 필드
3. **컨텍스트**: Trace ID 추가
4. **보안**: 민감 정보 마스킹

### 미래 전망

1. **Structured Logging**: 표준화
2. **Centralized**: ELK/Loki
3. **Query**: Lucene, KQL
4. **AI**: Log anomaly detection
