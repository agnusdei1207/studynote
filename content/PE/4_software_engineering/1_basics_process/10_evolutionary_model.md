# 진화적 프로세스 모델 (Evolutionary Process Model)

## 정의

**진화적 프로세스 모델(Evolutionary Process Model)**은 소프트웨어를 **점진적으로 발전**시켜 나가는 개발 방법론으로, 초기 버전을 개발한 후 사용자 피드백을 반영하여 반복적으로 확장하고 개선한다. 프로토타이핑과 점진적 개발을 결합한 방식이다.

```
┌─────────────────────────────────────────────────────────────────┐
│           진화적 프로세스 모델 개념                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   초기 요구사항                                         │ │
│   │        │                                                │ │
│   │        ▼                                                │ │
│   │   ┌─────────┐      ┌─────────┐      ┌─────────┐        │ │
│   │   │ 버전 1  │ ───▶ │ 버전 2  │ ───▶ │ 버전 3  │ ───▶...│ │
│   │   │(핵심)   │      │(확장)   │      │(완성)   │        │ │
│   │   └────┬────┘      └────┬────┘      └────┬────┘        │ │
│   │        │                │                │              │ │
│   │        ▼                ▼                ▼              │ │
│   │   ┌─────────┐      ┌─────────┐      ┌─────────┐        │ │
│   │   │사용자   │      │사용자   │      │사용자   │        │ │
│   │   │피드백  │      │피드백  │      │피드백  │        │ │
│   │   └─────────┘      └─────────┘      └─────────┘        │ │
│   │                                                         │ │
│   │   💡 각 버전은 이전 버전의 진화                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 진화적 개발 특성

```
┌─────────────────────────────────────────────────────────────────┐
│           진화적 개발 핵심 특성                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 점진적 확장 (Incremental Growth)                     │ │
│   │      • 작은 버전부터 시작하여 점차 기능 확장              │ │
│   │      • 각 단계별로 명확한 산출물 존재                     │ │
│   │                                                         │ │
│   │   ② 지속적 피드백 (Continuous Feedback)                  │ │
│   │      • 사용자 평가를 통한 요구사항 명확화                 │ │
│   │      • 실제 사용 환경에서 검증                           │ │
│   │                                                         │ │
│   │   ③ 적응적 계획 (Adaptive Planning)                      │ │
│   │      • 고정된 계획보다 상황에 따른 조정                   │ │
│   │      • 요구사항 변화에 유연한 대응                        │ │
│   │                                                         │ │
│   │   ④ 프로토타이핑 통합 (Prototyping Integration)          │ │
│   │      • 실행 가능한 프로토타입을 제품으로 진화            │ │
│   │      • 폐기형이 아닌 진화형 프로토타이핑                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 진화적 개발 사이클

```
┌─────────────────────────────────────────────────────────────────┐
│           진화적 개발 사이클                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │           ┌───────────────┐                             │ │
│   │      ┌───▶│   명세화      │───┐                         │ │
│   │      │    │ (Specification)│    │                       │ │
│   │      │    └───────────────┘    │                       │ │
│   │      │           │              │                       │ │
│   │      │           ▼              │                       │ │
│   │   ┌──┴────┐  ┌───────────┐  ┌──┴────┐                  │ │
│   │   │ 평가  │◀─│   구현    │◀─│ 요구   │                  │ │
│   │   │       │  │ (Implement)│  │ (Reqs) │                  │ │
│   │   └──┬────┘  └───────────┘  └───────┘                  │ │
│   │      │           ▲                                     │ │
│   │      │    ┌──────┴───────┐                             │ │
│   │      └────│  피드백      │                             │ │
│   │           │  (Feedback)  │                             │ │
│   │           └──────────────┘                             │ │
│   │                                                         │ │
│   │   각 사이클:                                             │ │
│   │   1. 요구사항: 다음 버전에 추가할 기능                   │ │
│   │   2. 명세화: 기능의 상세 정의                            │ │
│   │   3. 구현: 코딩 및 테스트                                │ │
│   │   4. 평가: 사용자 피드백 수집                           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 다른 모델과의 비교

```
┌─────────────────────────────────────────────────────────────────┐
│           진화적 모델 vs 다른 모델                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분         │ 폭포수    │ 프로토타입 │ 진화적          │ │
│   ├───────────────┼───────────┼───────────┼─────────────────┤ │
│   │ 요구사항      │ 초기 고정 │ 반복 정제 │ 지속적 진화     │ │
│   │ 산출물        │ 문서 중심 │ 프로토타입 │ 동작하는 버전   │ │
│   │ 사용자 참여   │ 제한적    │ 높음      │ 지속적          │ │
│   │ 변경 용이성   │ 낮음      │ 높음      │ 높음            │ │
│   │ 관리 복잡도   │ 낮음      │ 중간      │ 높음            │ │
│   │ 문서화        │ 완벽      │ 부족      │ 점진적          │ │
│   │ 위험 관리     │ 낮음      │ 중간      │ 높음            │ │
│   └───────────────┴───────────┴───────────┴─────────────────┘ │
│                                                                 │
│   진화적 모델의 장점:                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   • 요구사항 불확실성에 강함                             │ │
│   │   • 조기 사용자 피드백                                   │ │
│   │   • 위험을 초기에 발견                                   │ │
│   │   • 실제 동작하는 소프트웨어 조기 제공                   │ │
│   │   • 비즈니스 가치 조기 실현                              │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   진화적 모델의 단점:                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   • 프로젝트 관리 복잡                                   │ │
│   │   • 아키텍처 부패 가능성                                 │ │
│   │   • 문서화 부족 위험                                     │ │
│   │   • 범위 확장(Scope Creep) 위험                          │ │
│   │   • 일정/비용 예측 어려움                                │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 적용 분야 및 주의사항

```
┌─────────────────────────────────────────────────────────────────┐
│           진화적 모델 적용 분야                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   적합한 경우:                                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │

│   │   ✓ 사용자와 지속적 소통 가능                            │ │
│   │   ✓ 새로운 기술/도메인 영역                               │ │
│   │   ✓ 작은 팀, 단기 프로젝트                                │ │
│   │   ✓ 연구/개발(R&D) 프로젝트                              │ │
│   │   ✓ 사용자 인터페이스 중심 시스템                        │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   부적합한 경우:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ✗ 요구사항이 명확하고 변경 적음                        │ │
│   │   ✗ 대규모, 임베디드, 실시간 시스템                       │ │
│   │   ✗ 안전 중요 시스템 (Safety-critical)                   │ │
│   │   ✗ 계약 기반 고정 가격 프로젝트                         │ │
│   │   ✗ 사용자 참여 어려움                                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   성공 요인:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 명확한 아키텍처 초기 설계                            │ │
│   │   • 각 버전별 명확한 목표 정의                           │ │
│   │   • 지속적인 리팩토링                                    │ │
│   │   • 자동화된 테스트                                      │ │
│   │   • 정기적인 사용자 피드백                               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **진화적 모델**: 초기 버전 개발 후 피드백을 통해 점진적 확장
> **핵심 특성**: 점진적 확장, 지속적 피드백, 적응적 계획
> **프로토타입**: 폐기형이 아닌 제품으로 진화

### 2. 빈출 키워드

- **Evolutionary**: 점진적 발전/진화
- **Incremental**: 점진적 확장
- **Feedback Loop**: 사용자 피드백 순환
- **Adaptive Planning**: 적응적 계획

### 3. 예상 문제

**Q1. 진화적 프로세스 모델의 개념과 특징을 설명하시오.**

**Q2. 진화적 모델과 폭포수 모델의 차이점을 비교하시오.**

**Q3. 진화적 모델의 적용 분야와 주의사항을 기술하시오.**
