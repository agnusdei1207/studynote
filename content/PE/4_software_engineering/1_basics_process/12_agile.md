# 애자일 방법론 (Agile Methodology)

## 정의

**애자일 방법론(Agile Methodology)**은 소프트웨어 개발에서 **변화에 민첩하게 대응**하고, **동작하는 소프트웨어**를 조기에 지속적으로 전달하는 가치 중심의 개발 철학이자 방법론이다. 2001년 애자일 선언문(Agile Manifesto)으로 시작되었다.

```
┌─────────────────────────────────────────────────────────────────┐
│           애자일 선언문 (Agile Manifesto)                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   4가지 핵심 가치:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   공정과 도구보다          개인과 상호작용을              │ │
│   │   (Processes and tools)   (Individuals and interactions) │ │
│   │                                                         │ │
│   │   포괄적인 문서보다        작동하는 소프트웨어를          │ │
│   │   (Comprehensive docs)    (Working software)             │ │
│   │                                                         │ │
│   │   계약 협상보다            고객과의 협력을                │ │
│   │   (Contract negotiation)  (Customer collaboration)       │ │
│   │                                                         │ │
│   │   계획을 따르기보다        변화에 대응하기를               │ │
│   │   (Following a plan)      (Responding to change)         │ │
│   │                                                         │ │
│   │   💡 우측의 가치도 중요하지만, 좌측의 가치를 더 높임       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 애자일 12원칙

```
┌─────────────────────────────────────────────────────────────────┐
│           애자일 12원칙                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 고객 만족: 가치 있는 소프트웨어를 조기/지속적 전달    │ │
│   │                                                         │ │
│   │   ② 변화 수용: 개발 후반도 변경을 환영                    │ │
│   │                                                         │ │
│   │   ③ 작동하는 소프트웨어: 자주 전달 (주/월 단위)           │ │
│   │                                                         │ │
│   │   ④ 협업: 비즈니스 담당자와 개발자의 일일 협력             │ │
│   │                                                         │ │
│   │   ⑤ 동기 부여: 신뢰할 수 있는 환경에서 동기 부여           │ │
│   │                                                         │ │
│   │   ⑥ 대면 소통: 정보 전달의 가장 효율적 방법                │ │
│   │                                                         │ │
│   │   ⑦ 진척도: 작동하는 소프트웨어가 주요 척도                │ │
│   │                                                         │ │
│   │   ⑧ 지속 가능한 속도: 일정한 개발 속도 유지               │ │
│   │                                                         │ │
│   │   ⑨ 기술적 탁월성: 좋은 설계에 지속적 관심                 │ │
│   │                                                         │ │
│   │   ⑩ 단순성: 하지 않아도 되는 일은 최대한 안 함            │ │
│   │                                                         │ │
│   │   ⑪ 자기 조직화 팀: 최고의 아키텍처와 요구사항 도출        │ │
│   │                                                         │ │
│   │   ⑫ 회고: 정기적으로 팀 효율성을 돌아보고 개선             │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 애자일 vs 전통적 방법론

```
┌─────────────────────────────────────────────────────────────────┐
│           애자일 vs 전통적(Waterfall) 방법론 비교                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분         │ 전통적(폭포수)      │ 애자일             │ │
│   ├───────────────┼─────────────────────┼───────────────────┤ │
│   │ 요구사항      │ 초기에 완전 정의    │ 진화적, 지속적    │ │
│   │ 계획          │ 상세, 고정          │ 적응적, 유연      │ │
│   │ 개발 주기     │ 긴 주기 (월/년)     │ 짧은 주기 (주)    │ │
│   │ 문서화        │ 광범위              │ 필요 최소화       │ │
│   │ 고객 참여     │ 시작/끝             │ 지속적            │ │
│   │ 팀 구조       │ 기능별 조직         │ 크로스펑셔널      │ │
│   │ 변화 대응     │ 어려움              │ 환영              │ │
│   │ 성공 척도     │ 계획 준수           │ 고객 가치         │ │
│   │ 통제 방식     │ 프로세스 중심       │ 사람 중심         │ │
│   └───────────────┴─────────────────────┴───────────────────┘ │
│                                                                 │
│   적합한 상황:                                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   전통적:                                                │ │
│   │   • 요구사항 명확하고 안정적                             │ │
│   │   • 안전 중요 시스템                                     │ │
│   │   • 계약 기반 고정 범위                                  │ │
│   │                                                         │ │
│   │   애자일:                                                │ │
│   │   • 요구사항 불확실                                      │ │
│   │   • 빠른 시장 출시 필요                                  │ │
│   │   • 지속적 고객 피드백 가능                              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **애자일**: 변화에 민첩하게 대응하는 가치 중심 개발 방법론
> **선언문 4가치**: 개인/상호작용, 작동 SW, 고객협력, 변화대응
> **핵심 실천**: 반복적 전달, 지속적 피드백, 자기조직화 팀

### 2. 빈출 키워드

- **Agile Manifesto**: 2001년 애자일 선언문
- **Scrum/XP/Kanban**: 주요 애자일 프레임워크
- **Sprint/Iteration**: 짧은 개발 주기
- **Backlog**: 우선순위 있는 요구사항 목록

### 3. 예상 문제

**Q1. 애자일 선언문의 4가지 핵심 가치를 설명하시오.**

**Q2. 애자일 방법론과 전통적 폭포수 모델의 차이점을 비교하시오.**

**Q3. 애자일 도입의 성공 요인과 실패 원인을 기술하시오.**
