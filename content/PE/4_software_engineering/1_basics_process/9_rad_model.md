# RAD (Rapid Application Development) 모델

## 정의

**RAD(Rapid Application Development)** 모델은 단축된 개발 주기(60~90일) 내에 **고품질 시스템을 신속히 개발**하기 위한 점진적 소프트웨어 개발 방법론이다. 사용자 참여, 프로토타이핑, 자동화 도구(CASE)를 핵심으로 한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           RAD 모델 개요                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   핵심 원칙:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 사용자 참여 (User Involvement)                      │ │
│   │      - 전 과정에 사용자 적극 참여                        │ │
│   │      - 의사결정 지연 최소화                              │ │
│   │                                                         │ │
│   │   ② 프로토타이핑 (Prototyping)                          │ │
│   │      - 반복적 개선을 통한 요구사항 명확화                │ │
│   │      - 실행 가능한 결과물 조기 제공                     │ │
│   │                                                         │ │
│   │   ③ CASE 도구 활용 (Computer-Aided Software Eng.)       │ │
│   │      - 코드 자동 생성                                    │ │
│   │      - 개발 시간 단축                                    │ │
│   │                                                         │ │
│   │   ④ 시간 박스 (Time Boxing)                             │ │
│   │      - 60~90일 단축 개발 주기                            │ │
│   │      - 기능 조정보다 일정 준수                           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1. RAD 모델 단계

```

│           RAD 모델 개발 단계                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 요구사항 계획 (Requirements Planning)               │ │
│   │      ┌──────────────────────────────────────────────┐   │ │
│   │      │ • 사용자 참여 워크샵                          │   │ │
│   │      │ • 범위 정의                                    │   │ │
│   │      │ • 비즈니스 요구 분석                           │   │ │
│   │      │ • 기능 우선순위 결정                           │   │ │
│   │      └──────────────────────────────────────────────┘   │ │
│   │                      ↓                                  │ │
│   │   ② 사용자 설계 (User Design)                           │ │
│   │      ┌──────────────────────────────────────────────┐   │ │
│   │      │ • JAD (Joint Application Design) 세션        │   │ │
│   │      │ • 프로토타입 설계 및 검토                     │   │ │
│   │      │ • 사용자 인터페이스 설계                      │   │ │
│   │      │ • 반복적 개선                                 │   │ │
│   │      └──────────────────────────────────────────────┘   │ │
│   │                      ↓                                  │ │
│   │   ③ 구축 (Construction)                                 │ │

│   │      │ • CASE 도구 활용 코딩                         │   │ │
│   │      │ • 컴포넌트 재사용                              │   │ │
│   │      │ • 실시간 사용자 피드백                        │   │ │
│   │      │ • 지속적 통합 및 테스트                       │   │ │
│   │      └──────────────────────────────────────────────┘   │ │
│   │                      ↓                                  │ │
│   │   ④ 전환 (Cutover)                                      │ │
│   │      ┌──────────────────────────────────────────────┐   │ │
│   │      │ • 데이터 변환                                  │   │ │
│   │      │ • 사용자 교육                                  │   │ │
│   │      │ • 시스템 설치                                  │   │ │
│   │      │ • 운영 이관                                    │   │ │
│   │      └──────────────────────────────────────────────┘   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. JAD (Joint Application Design)

```
┌─────────────────────────────────────────────────────────────────┐
│           JAD (공동 응용 설계)                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   정의: 사용자, 관리자, 개발자가 **집중 워크샵**을 통해          │
│         요구사항과 설계를 함께 정의하는 기법                      │
│                                                                 │
│   JAD 세션 구성:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   역할                 │  책임                          │ │
│   │   ─────────────────────┼─────────────────────────────   │ │
│   │   JAD 리더             │  세션 진행, 의사결정 유도       │ │
│   │   경영진 스폰서        │  의사결정 승인, 자원 확보       │ │
│   │   사용자 대표          │  업무 요구사항 제시            │ │
│   │   개발자/분석가        │  기술적 실현가능성 평가        │ │
│   │   서기 (Scribe)        │  회의록 작성                  │ │
│   │   IT 전문가            │  기술 자문                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   JAD 세션 프로세스:                                            │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   1. 준비 단계 (1~2주)                                  │ │
│   │      - 참가자 선정, 목표 정의, 자료 준비                │ │
│   │                                                         │ │
│   │   2. JAD 세션 (3~5일 집중)                              │ │
│   │      - 요구사항 도출                                    │ │
│   │      - 화면/보고서 설계                                 │ │
│   │      - 비즈니스 규칙 정의                                │ │
│   │                                                         │ │
│   │   3. 후속 단계                                          │ │
│   │      - 결과물 정리 및 승인                              │ │
│   │      - 프로토타입 개발                                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. CASE 도구 활용

```
┌─────────────────────────────────────────────────────────────────┐
│           CASE (Computer-Aided Software Engineering) 도구       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   RAD에서 CASE 도구의 역할:                                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌───────────────────────────────────────────────────┐│ │
│   │   │                  Upper CASE                        ││ │
│   │   │   (분석, 설계 자동화)                              ││ │
│   │   │   • 요구사항 관리                                  ││ │
│   │   │   • 모델링 (ERD, DFD, UML)                        ││ │
│   │   │   • 문서 자동 생성                                 ││ │
│   │   └───────────────────────────────────────────────────┘│ │
│   │                         ↓                               │ │
│   │   ┌───────────────────────────────────────────────────┐│ │
│   │   │                  Lower CASE                        ││ │
│   │   │   (구현, 테스트 자동화)                            ││ │
│   │   │   • 코드 자동 생성                                 ││ │
│   │   │   • 테스트 자동화                                  ││ │
│   │   │   • 빌드/배포 자동화                               ││ │
│   │   └───────────────────────────────────────────────────┘│ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   대표 CASE 도구:                                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  • IBM Rational Rose                                    │ │
│   │  • Enterprise Architect                                 │ │
│   │  • Visual Paradigm                                      │ │
│   │  • ER/Studio                                            │ │
│   │  • PowerDesigner                                        │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   RAD 코드 생성 패턴:                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   설계 모델 ──▶ [CASE 도구] ──▶ 소스 코드               │ │
│   │   (UML 등)     (자동 생성)    (70~90%)                  │ │
│   │                                                         │ │
│   │   남은 10~30%: 비즈니스 로직, 특화 기능                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

│   ┌─────────────────────────────────────────────────────────┐ │

│   │   • 초기 품질 문제 조기 발견                            │ │

└─────────────────────────────────────────────────────────────────┘

## 기술사 시험 포인트

### 1. 핵심 정의

> **RAD**: 60~90일 내 고품질 시스템 신속 개발 방법론
> **핵심 요소**: 사용자 참여, 프로토타이핑, CASE 도구, Time Boxing
> **JAD**: 사용자-개발자 집중 워크샵

### 2. 빈출 키워드

- **Time Boxing**: 고정 일정 내 기능 조정
- **JAD (Joint Application Design)**: 공동 설계 워크샵
- **CASE 도구**: 자동화 개발 도구
- **프로토타이핑**: 반복적 개선

### 3. 예상 문제

**Q1. RAD 모델의 4단계와 각 단계의 핵심 활동을 설명하시오.**

**Q2. JAD(Joint Application Design)의 개념과 구성 역할을 기술하시오.**

**Q3. RAD 모델의 적용 조건과 한계점을 설명하시오.**
