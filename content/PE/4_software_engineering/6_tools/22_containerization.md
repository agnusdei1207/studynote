# 컨테이너화 (Containerization)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컨테이너는 **"애플리케이션과 **종속성을 **격리된 **환경**에서 **실행하는 **기술"**로, **Host OS Kernel**를 **공유**하고 **User Space**만 **격리**하며 **Image**(불변 템플릿)를 **Container**(실행 인스턴스)로 **실행**한다.
> 2. **가치**: **"Build once, run anywhere"**로 **환경 일관성**을 **보장**하고 **VM**보다 **가볍고 **빠르**며 **Microservices Architecture**에서 **독립적 **배포**와 **확장**을 **지원**한다.
> 3. **융합**: **Docker**(표준), **Kubernetes**(오케스트레이션), **containerd**(런타임)가 **대표 도구**이며 **OCI**(Open Container Initiative)로 **표준화**되고 **Cloud Native** **기술의 **기반**이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
컨테이너는 **"프로세스와 격리된 실행 환경"**이다.

**컨테이너의 특징**:
- **Isolation**: Namespace, Cgroup
- **Portability**: 이미지 기반
- **Lightweight**: Kernel 공유

### 💡 비유
컨테이너는 **"가상 주방"**과 같다.
- **VM**: 독립 건물
- **Container**: 주방 내 분리

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         컨테이너의 발전                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

VM (2000년대):
    • 완전 격리
    • 무거움 (GB 단위)
         ↓
LXC (2008):
    • Linux Container
    • Kernel 공유
         ↓
Docker (2013):
    • Image 표준
    • 사용성 개선
         ↓
Kubernetes (2014):
    • 오케스트레이션
    • 자동화
```

### Container Isolation (Namespace)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Linux Namespace                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Namespace Type    │  Isolates                      │  Example                       │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  PID               │  Process IDs                   │  Container 내 PID=1          │
    │  NET               │  Network Stack                 │  독립 IP, Port               │
    │  MNT               │  Mount Points                  │  독립 Filesystem             │
    │  IPC               │  Inter-Process Communication   │  독립 Message Queue          │
    │  UTS               │  Hostname & Domain Name        │  독립 hostname              │
    │  USER              │  User & Group IDs               │  Container 내 root          │
    │  CGROUP            │  Resource Limits               │  CPU, Memory 제한           │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### VM vs Container

| 구분 | VM | Container |
|------|-----|-----------|
| **Kernel** | 독립 | 공유 |
| **크기** | GB | MB |
| **시작** | 분~초 | 초 |
| **격리** | 강함 | 중간 |
| **이식성** | 중간 | 높음 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: Dockerfile 작성
**상황**: Python 애플리케이션
**판단**:

```dockerfile
# Multi-stage Build
FROM python:3.11-slim as builder

WORKDIR /app
COPY requirements.txt .
RUN pip install --user --no-cache-dir -r requirements.txt

FROM python:3.11-slim

WORKDIR /app
COPY --from=builder /root/.local /root/.local
COPY . .

# Non-root user
RUN useradd -m appuser
USER appuser

EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```
