# 소프트웨어(Software) 정의

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소프트웨어는 **"하드웨어와 사용자 사이에서 특정 기능을 수행하는 프로그램의 집합"**으로, **저장된 프로그램(Stored Program)**과 **실행 중인 프로그램(Running Process)**의 구분이 명확하며, **시스템 소프트웨어**와 **응용 소프트웨어**로 계층화되어 현대 기술 생태계의 핵심 인프라가 된다.
> 2. **가치**: **시스템 소프트웨어**(OS, 드라이버)는 **하드웨어를 제어하고 자원을 관리**하며, **응용 소프트웨어**(앱플리케이션, 워브 브라우저)는 **최종 사용자에게 서비스를 제공**하는 **2계층 구조(Layered Architecture)**를 통해 **관심사 분리(Separation of Concerns)**를 실현하여 복잡도를 낮추고 유지보수성을 높인다.
> 3. **융합**: **컴퓨터 구조(CS, Client-Server)**의 기반이자 **분산 시스템(Distributed System)**, **마이크로서비스(MSA)**, **클라우드 네이티브(Cloud Native)** 아키텍처로 진화하며, **CI/CD 파이프라인**과 **DevSecOps**로 자동화된 소프트웨어 공학(CI/CD)의 핵심이 되었다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
소프트웨어(Software)는 **"컴퓨터 시스템을 구성하는 프로그램과 데이터의 집합"**으로, 하드웨어를 제어하고 사용자에게 서비스를 제공하는 모든 프로그램을 의미한다.

**핵심 정의 (IEEE Definition)**:
> "소프트웨어는 시스템, 절차 또는 기타 프로세스를 수행하기 위해 설계된 일련된 명령어, 데이터, 프로그램의 집합이다."

**역사적 기원**:
```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         소프트웨어 용어의 역사                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

1935년: Alan Turing "Computing Machinery and Intelligence"
         - 최초의 이론적 제안
         - "Universal Computing Machine"
         ↓
1940~1950년: ENIAC, EDVAC
         • 하드웨어 프로그램 (Switching, Plugboard)
         • 소프트웨어는 존재하지 않음 (하드와이어)
         ↓
1950년대: 어셈블러, 컴파일러 등장
         • 최초의 고급 프로그래밍 언어 (Fortran, COBOL)
         • 소프트웨어 산업 태동
         ↓
1960~1970년대: 유닉(Unix), C 언어
         • OS와 SW를 분리
         • 이식적 공유 코드
         ↓
1980~1990년대: PC 혁명, GUI
         • MS-DOS, Windows, Macintosh
         • 대중화 소프트웨어 시대
         ↓
2000년대~현재: 인터넷, 클라우드, AI
         • SaaS, PaaS, AIaaS
         • DevOps, MSA, Kubernetes
```

### 💡 비유
소프트웨어는 **"스마트폰의 앱�(애플리케이션)"**과 같다.
- **하드웨어**: 스마트폰 본체
- **소프트웨어: 앱리케이션(App)
- **사용자**: 앱을 사용하는 사람

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         소프트웨어 4대 특징                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

1960~70년대: 패키징 및 폭포수(Soft)한 개발
    • 요구사항 → 설계 → 코딩 → 테스트 → 유지보수 순차적 진행
    • 폭포수 모델: 각 단계별 문서화, 관리가 필요
    • Waterfall Model (1970, Winston Royce)

1980~90년대: 구조적 프로그래밍 및 CASE 도구
    • 구조적 설계: SA/SD, DFD(자료 흐름도)
    • CASE 도구: ERWin, Oracle Designer
    • 구조적 분석과 설계 자동화

2000년대~현재: 애자일(Agile) 및 DevOps
    • 애자일 선언: 4가지 가치, 12가지 원칙
    • CI/CD 파이프라인: 자동화된 빌드/배포
    • 클라우드 네이티브, 마이크로서비스
```

## Ⅲ. 융합 비교 및 다각도 분석

### 소프트웨어 개발 모델 비교

| 모델 | 특징 | 장점 | 단점 | 적용 분야 |
|------|------|------|------|----------|
| **폭포수(Waterfall)** | 순차적 개발 | 문서화 철저 | 유연성 낮음 | 안정적 요구 |
| **프로토타입(Prototype)** | 시제품 개발 | 요구사항 명확 | 많은 노력 | 사용자 인터페이스 중심 |
| **나선형(Spiral)** | 위험 분석 반복 | 점진적 개발 | 복잡함 | 대형 프로젝트 |
| **애자일(Agile)** | 반복적 개발, 피드백 | 빠른 피드백 | 문서화 부족 | 변화 많은 프로젝트 |

### 과목 융합 관점 분석

#### 1. 컴퓨터 구조 ↔ 소프트웨어
- **Frontend**: HTML/CSS/JS → React, Vue
- **Back-end**: Java Spring, Python Django
- **Database**: MySQL, PostgreSQL
- **DevOps**: Docker, Kubernetes

#### 2. 운영체제 ↔ 소프트웨어
- **프로세스 관리**: OS의 핵심
- **메모리 관리**: 가상 메모리, 페이지 테이블
- **파일 시스템**: ext4, NTFS

## Ⅴ. 기대효과 및 결론

### 소프트웨어 공학의 3대 축

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         소프트웨어 공학의 3대 축 (The Trilemma)                           │
└─────────────────────────────────────────────────────────────────────────────────────────┘

       [좋음]         [빠름]         [빠름+빠름]
        /                │            │
    (Good)          (Fast)      (Cheap)
     │               │            │
    저렴함          빠름         저렴함+빠름
```

1. **No-Code/Low-Code**: Wix, Bubble (코드 없는 개발)
2. **AI-Assisted Coding**: GitHub Copilot, ChatGPT
3. **Edge Computing**: 엣지에서 연산 수행
