# CI/CD 도구 (CI/CD Tools)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CI/CD 도구는 **"코드 **변경**을 **자동으로 **빌드**, **테스트**, **배포**하는 **시스템"**으로, **CI**(Continuous Integration)는 **자동 **빌드/**테스트**, **CD**(Continuous Delivery/Deployment)는 **자동 **배포**를 **의미**한다.
> 2. **도구**: **Jenkins**(오픈 **소스, **플러그인 **풍부), **GitHub Actions**(Git **통합), **GitLab CI**(DevOps **통합), **CircleCI**(클라우드 **기반), **ArgoCD**(GitOps)가 **대표적**이다.
> 3. **파이프라인**: **Pipeline**으로 **Build** → **Test** → **Deploy** **단계**를 **정의**하고 **Trigger**(Push, **PR**, **Scheduled)로 **자동 **실행**하며 **Notification**(Slack, **Email)으로 **결과**를 **알린다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
CI/CD는 **"지속적 통합 및 배포"**이다.

**용어 정의**:
| 용어 | 설명 |
|------|------|
| **CI** | 코드 병합 후 자동 빌드/테스트 |
| **CD** | 지속적 배포 (Delivery: 수동 승인, Deployment: 자동) |
| **Pipeline** | 자동화 워크플로우 |

### 💡 비유
CI/CD는 ****자동 **생산 **라인 ****과 같다.
- **원료**: 코드
- **조립**: 빌드
- **검사**: 테스트
- **출하**: 배포

## Ⅲ. 융합 비교 및 다각도 분석

### CI/CD 도구 비교

| 도구 | 유형 | 장점 | 단점 |
|------|------|------|------|
| **Jenkins** | Self-hosted | 플러그인, 유연성 | 유지보수 |
| **GitHub Actions** | SaaS | Git 통합 | 비용 |
| **GitLab CI** | SaaS/Self | DevOps 통합 | 복잡함 |
| **CircleCI** | SaaS | 빠름 | Docker 제한 |
| **ArgoCD** | GitOps | K8s 배포 | K8s 종속 |

### Jenkinsfile (Declarative Pipeline)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Jenkinsfile Example                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  pipeline {                                                                              │  │
    │      agent any                                                                            │  │
    │                                                                                         │  │
    │      environment {                                                                        │  │
    │          DOCKER_CREDS = credentials('docker-hub')                                         │  │
    │      }                                                                                    │  │
    │                                                                                         │  │
    │      stages {                                                                             │  │
    │          stage('Build') {                                                                 │  │
    │              steps {                                                                      │  │
    │                  sh 'docker build -t myapp:${BUILD_NUMBER} .'                               │  │
    │              }                                                                            │  │
    │          }                                                                                │  │
    │                                                                                         │  │
    │          stage('Test') {                                                                  │  │
    │              parallel {                                                                  │  │
    │                  stage('Unit Tests') {                                                    │  │
    │                      steps {                                                              │  │
    │                          sh 'docker run myapp:${BUILD_NUMBER} npm test'                       │  │
    │                      }                                                                    │  │
    │                  }                                                                        │  │
    │                  stage('Integration Tests') {                                             │  │
    │                      steps {                                                              │  │
    │                          sh 'docker run myapp:${BUILD_NUMBER} npm run test:int'               │  │
    │                      }                                                                    │  │
    │                  }                                                                        │  │
    │              }                                                                            │  │
    │          }                                                                                │  │
    │                                                                                         │  │
    │          stage('Deploy') {                                                                │  │
    │              when {                                                                       │  │
    │                  branch 'main'                                                             │  │
    │              }                                                                            │  │
    │              steps {                                                                      │  │
    │                  withDockerCredentials([DOCKER_CREDS]) {                                    │  │
    │                      sh 'docker push myapp:${BUILD_NUMBER}'                                │  │
    │                  }                                                                        │  │
    │                  sh 'kubectl set image deployment/myapp myapp=myapp:${BUILD_NUMBER}'         │  │
    │              }                                                                            │  │
    │          }                                                                                │  │
    │      }                                                                                    │  │
    │                                                                                         │  │
    │      post {                                                                               │  │
    │          always {                                                                         │  │
    │              emailext subject: "Pipeline ${env.JOB_NAME} - ${currentBuild.result}",          │  │
    │                       body: "Check console output at ${env.BUILD_URL}",                      │  │
    │                       to: "team@example.com"                                               │  │
    │          }                                                                                │  │
    │      }                                                                                    │  │
    │  }                                                                                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### CI/CD 기대 효과

| 효과 | 수동 | CI/CD |
|------|------|-------|
| **빌드** | 수동 | 자동 |
| **테스트** | 선택 | 필수 |
| **배포** | 느림 | 빠름 |
| **에러** | 늦음 발견 | 조기 발견 |

### 모범 사례

1. **빈번한 커밋**: 작은 단위로
2. **테스트 커버리지**: 80% 이상
3. **문서화**: README, Changelog
4. **모니터링**: 배포 후 지표 확인

### 미래 전망

1. **GitOps**: 모든 것을 코드로
2. **Progressive Delivery**: Feature flags
3. **AI**: 자동 롤백/배포
