# 나선형 모델 (Spiral Model)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 나선형 모델은 **"위험 분석(Risk Analysis)을 중심으로 한 반복적 개발 모델"**으로, 1988년 **보엠(Barry Boehm)**이 제안하였으며, **계획 수립 → 위험 분석 → 엔지니어링 → 고객 평가**의 4단계 사이클을 **나선형(Spiral)**으로 반복하면서 점진적으로 소프트웨어를 개발한다.
> 2. **가치**: **대형 프로젝트의 불확실성(Uncertainty)**을 **위험 분석(Risk Analysis)**으로 체계적으로 관리하며, **프로토타입(Prototype)**을 통해 요구사항을 점진적으로 확정하고 **재작업(Rework)** 비용을 최소화한다.
> 3. **융합**: **애자일(Agile)**의 **반복적 개발(Iterative Development)**과 **지속적 피드백(Continuous Feedback)** 개념의 선구자이자, **DevOps**의 **지속적 통합/배포(CI/CD)**와 **위험 기반 테스팅(Risk-Based Testing)**의 이론적 토대가 된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
나선형 모델(Spiral Model)은 **"위험 분석을 중심으로 한 반복적 소프트웨어 개발 프로세스"**로, 개발 주기마다 위험을 식별하고 해결하면서 점진적으로 시스템을 구축한다.

**4가지 활동(Quadrants)**:
1. **목표 설정(Objective Setting)**: 반복의 목표, 제약조건, 대안 식별
2. **위험 분석 및 감소(Risk Analysis & Reduction)**: 위험 식별, 해결 방안 탐색
3. **개발 및 검증(Engineering & Validation)**: 해당 위계 수준에서의 개발, 테스트
4. **계획(Planning)**: 다음 반복을 위한 계획 수립

### 💡 비유
나선형 모델은 **"계단 오르기"**와 같다.
- **1바퀴(1사이클)**: 계단 한 칸 (프로토타입)
- **반복마다**: 더 높은 곳으로 (기능 확장)
- **위험 분석**: 다칠 위험 확인 후 안전하게 오름

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         나선형 모델의 필요성                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

폭포수 모델의 한계 (1970년대):
    • 요구사항 초기에 완벽히 정의 필요
    • 변경 비용 큼
    • 위험 요소 후반 발견
         ↓
프로토타입 모델 (1980년대 초):
    • 빠른 피드백
    • 하지만 위험 분석 체계 부족
         ↓
나선형 모델 (1988년, Barry Boehm):
    • "A Spiral Model of Software Development and Enhancement"
    • 위험 분석을 핵심에 배치
    • 대형 프로젝트 적합
         ↓
애자일 (2001년):
    • 나선형의 반복성 계승
    • 더 가볍고 유연하게
```

### 위험 분석(Risk Analysis) 상세

| 위험 유형 | 설명 | 분석 방법 | 감소 전략 |
|----------|------|----------|----------|
| **기술적 위험** | 신기술 사용 여부 | PoC, 벤치마킹 | 프로토타입 |
| **프로젝트 위험** | 일정, 예산, 인력 | PERT, CPM | 여유 일정 |
| **비즈니스 위험** | 시장 변화, 경쟁 | 시장 조사 | 애자일 대응 |

### 심층 위험 관리 프로세스

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         위험 분석 4단계                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    1. 위험 식별 (Risk Identification)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 브레인스토밍, 체크리스트, 전문가 의견                                               │
    │  • 기술적, 관리적, 비즈니스 위험 분류                                                   │
    │  • 위험 등록부(Risk Register) 작성                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. 위험 평가 (Risk Assessment)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 발생 가능성(Probability) × 영향도(Impact) = 위험 노출(Risk Exposure)                 │
    │                                                                                         │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  위험 매트릭스 (Risk Matrix)                                                      │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │ Impact       │   Low   │  Medium  │   High   │  Critical  │                 │  │  │
    │  │  ├──────────────────────────────────────────────────────────────────────────────┤  │  │
    │  │  │ High         │ Medium  │   High   │ Critical │  Critical  │                 │  │  │
    │  │  │ Medium       │   Low   │  Medium  │   High   │  Critical  │                 │  │  │
    │  │  │ Low          │   Low   │   Low    │  Medium  │    High    │                 │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. 위험 감소 (Risk Reduction)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 회피(Avoidance): 위험 요소 제거                                                      │
    │  • 전이(Transfer): 보험, 아웃소싱                                                       │
    │  • 완화(Mitigation): 영향 감소                                                          │
    │  • 수용(Acceptance): 위험 인지하고 진행                                                  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    4. 위험 모니터링 (Risk Monitoring)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 정기 위험 검토 회의                                                                  │
    │  │  위험 대시보드 (Risk Dashboard)                                                     │  │
    │  │  • 새로운 위험 식별                                                                  │  │
    │  │  • 기존 위험 상태 업데이트                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### SDLC 모델 비교

| 모델 | 중심 활동 | 장점 | 단점 | 적용 |
|------|----------|------|------|------|
| **폭포수** | 순차적 개발 | 문서화, 관리 용이 | 변경 어려움 | 안정적 요구 |
| **프로토타입** | 빠른 시연 | 요구사항 명확 | 재작업 가능 | UI 중심 |
| **나선형** | 위험 분석 | 위험 관리 체계적 | 복잡함, 비용 | 대형 프로젝트 |
| **애자일** | 반복 + 피드백 | 유연성 | 문서화 부족 | 변화 많음 |

### 과목 융합 관점 분석

#### 1. 프로젝트 관리 ↔ 나선형
- **Risk Management**: PMBOK의 Risk Management와 연결
- **Iterative Planning**: 롤링 웨이브(Rolling Wave) 기법

#### 2. DevOps ↔ 나선형
- **CI/CD**: 각 사이클마다 자동화된 빌드/배포
- **지속적 피드백**: 위험 조기 발견

### 나선형 모델 기대 효과

| 프로젝트 유형 | 적합성 | 이유 |
|-------------|--------|------|

| **소규모 프로젝트** | 낮음 | 오버헤드 큼 |
