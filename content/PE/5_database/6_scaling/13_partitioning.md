# 파티셔닝 (Partitioning)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파티셔닝은 **"대량 테이블을 **여러 파티션(물리적 세그먼트)으로 **분할하여 **관리**하는 **데이터베이스 기법"**으로, **Partition Key**(파티션 키)에 따라 **데이터 분배**되고 **Query Planning**에서 **Partition Pruning**(불필요한 파티션 제외)으로 **성능**을 **향상**한다.
> 2. **가치**: **대량 데이터**에서 **Query 성능**을 **향상**하고 **인덱스 크기**를 **줄이**며 **특정 파티션만 **Backup/Purge** 가능하게 하고 **Partition-wise Join**으로 **조인 성능**을 **개선**한다.
> 3. **융합**: **Range Partition**(날짜/ID 범위), **Hash Partition**(해시 분배), **List Partition**(명시적 값), **Composite Partition**(범위+해시)가 있고 **PostgreSQL**, **MySQL**, **Oracle**, **SQL Server**가 **지원**하며 **Sharding**(수평 분할)과 유사하지만 **단일 DB** 내에서 **실행**된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
파티셔닝은 **"테이블 데이터를 여러 물리적 세그먼트로 분할하는 기법"**이다.

**파티셔닝의 목적**:
- **성능**: 파티션 Pruning으로 스캔 범위 축소
- **관리**: 파티션 단위 Backup/Purge
- **가용성**: 파티션 단위 장애 격리

### 💡 비유
파티셔닝은 **"책을 여러 권으로 나누기"**와 같다.
- **단권**: 10,000페이지 (찾기 어려움)
- **분할**: 권당 500페이지 (빠른 검색)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         파티셔닝의 필요성                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Large Table:
    • 수천만~수억 행
    • Full Scan 느림
    • Index 크기 문제
         ↓
Partitioning:
    • 논리적 1개 테이블
    • 물리적 N개 파티션
    • 필요한 파티션만 접근
         ↓
Sharding:
    • 여러 DB로 분산
    • 수평 확장
```

## Ⅱ. 아키텍처 및 핵심 원리

### 파티셔닝 타입

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         파티셔닝 타입                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Range Partitioning]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 범위 기반 분할 (날짜, ID 등)                                                          │
    │  │  • 예: 연도별, 월별                                                                  │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  -- PostgreSQL 예시                                                                 │  │
    │  │  CREATE TABLE orders (                                                              │  │
    │  │      id BIGINT,                                                                      │  │
    │  │      order_date DATE,                                                                │  │
    │  │      customer_id INT,                                                                │  │
    │  │      amount DECIMAL(10,2)                                                            │  │
    │  │  ) PARTITION BY RANGE (order_date);                                                  │  │
    │  │                                                                                       │  │
    │  │  -- 파티션 생성                                                                       │  │
    │  │  CREATE TABLE orders_2024_q1 PARTITION OF orders                                     │  │
    │  │      FOR VALUES FROM ('2024-01-01') TO ('2024-04-01');                               │  │
    │  │  CREATE TABLE orders_2024_q2 PARTITION OF orders                                     │  │
    │  │      FOR VALUES FROM ('2024-04-01') TO ('2024-07-01');                               │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Hash Partitioning]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 해시 함수 기반 분할                                                                 │
    │  │  • 데이터 균등 분배                                                                  │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  -- MySQL 예시                                                                       │  │
    │  │  CREATE TABLE users (                                                                │  │
    │  │      id INT,                                                                          │  │
    │  │      username VARCHAR(50),                                                           │  │
    │  │      email VARCHAR(100)                                                              │  │
    │  │  )                                                                                    │
    │  │  PARTITION BY HASH(id)                                                                │  │
    │  │  PARTITIONS 4;                                                                       │  │
    │  │                                                                                       │  │
    │  │  -- 4개 파티션으로 균등 분배                                                          │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [List Partitioning]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 명시적 값 목록 기반                                                                  │
    │  │  • 예: 지역별, 카테고리별                                                            │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  -- Oracle 예시                                                                      │  │
    │  │  CREATE TABLE sales (                                                                │  │
    │  │      id NUMBER,                                                                       │  │
    │  │      region VARCHAR2(20),                                                             │  │
    │  │      amount NUMBER                                                                    │  │
    │  │  )                                                                                    │
    │  │  PARTITION BY LIST (region) (                                                         │  │
    │  │      PARTITION north VALUES ('SEOUL', 'INCHEON'),                                     │  │
    │  │      PARTITION south VALUES ('BUSAN', 'DAEGU'),                                       │  │
    │  │      PARTITION etc VALUES (DEFAULT)                                                   │  │
    │  │  );                                                                                  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Partition Pruning

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Partition Pruning (파티션 가지치기)                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Query에서 필요한 파티션만 접근                                                        │
    │  │  • Full Table Scan 회피                                                              │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  -- Query                                                                            │  │
    │  │  SELECT * FROM orders WHERE order_date = '2024-02-15';                              │  │
    │  │                                                                                       │  │
    │  │  [Without Partitioning]                                                              │  │
    │  │  • Full Table Scan (모든 데이터 스캔)                                                 │  │
    │  │                                                                                       │  │
    │  │  [With Range Partitioning (Quarterly)]                                               │  │
    │  │  • orders_2024_q1 파티션만 스캔                                                      │  │
    │  │  • 나머지 파티션 Pruning!                                                            │  │
    │  │                                                                                       │  │
    │  │  EXPLAIN ANALYZE 결과:                                                               │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  Append                                                                           │  │  │
    │  │  │    ->  Seq Scan on orders_2024_q1  (cost=0.00..123.45 rows=1000)               │  │  │
    │  │  │          Filter: (order_date = '2024-02-15'::date)                               │  │
    │  │  │  (다른 파티션은 "Partition Pruned"로 표시)                                        │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Partition-wise Join

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Partition-wise Join                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 동일한 키로 파티션된 테이블 조인                                                      │
    │  │  • 파티션 간 조인 → 병렬 처리 가능                                                    │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  -- 두 테이블 모두 order_date로 Range 파티션                                             │  │
    │  │  SELECT o.*, c.customer_name                                                        │  │
    │  │  FROM orders o                                                                       │  │
    │  │  JOIN customers c ON o.customer_id = c.id                                            │  │
    │  │  WHERE o.order_date BETWEEN '2024-01-01' AND '2024-03-31';                          │  │
    │  │                                                                                       │  │
    │  │  [Execution Plan]                                                                    │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  ->  Hash Join (orders_2024_q1, customers_2024_q1)                               │  │  │
    │  │  │  ->  Hash Join (orders_2024_q2, customers_2024_q2)                               │  │  │
    │  │  │  ->  Hash Join (orders_2024_q3, customers_2024_q3)                               │  │  │
    │  │  │  (각 파티션 쌍별로 병렬 조인!)                                                     │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 파티셔닝 vs 샤딩

| 구분 | 파티셔닝 | 샤딩 |
|------|---------|------|
| **범위** | 단일 DB 내 | 여러 DB |
| **트랜잭션** | 지원 | 제한적 |
| **확장** | 수직 | 수평 |
| **복잡도** | 낮음 | 높음 |

### 파티셔닝 전략 선택

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         파티셔닝 전략 가이드                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  데이터 특성                  │  추천 파티셔닝         │  예시                    │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  시계열 (Log, Trade)         │  Range (날짜)         │  orders_2024_01          │
    │  카디널리티 높음 (ID)        │  Hash                 │  users_p0 ~ p3            │
    │  지역/카테고리               │  List                 │  sales_north             │
    │  복합 조건                   │  Composite            │  Range + Hash             │
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 로그 테이블 파티셔닝
**상황**: 일일 100만 건 로그
**판단**:

```sql
-- PostgreSQL: Range Partitioning (일별)
CREATE TABLE access_log (
    id BIGSERIAL,
    access_time TIMESTAMP NOT NULL,
    user_id INT,
    url TEXT,
    status_code SMALLINT
) PARTITION BY RANGE (access_time);

-- 파티션 자동 생성 (pg_parted extension)
CREATE TABLE access_log_2024_03_06 PARTITION OF access_log
    FOR VALUES FROM ('2024-03-06') TO ('2024-03-07');

-- Query (Partition Pruning)
SELECT * FROM access_log WHERE access_time >= '2024-03-06 00:00:00'
                               AND access_time < '2024-03-07 00:00:00';
-- → access_log_2024_03_06 파티션만 스캔!

-- 오래된 파티션 삭제
DROP TABLE access_log_2023_01_01;  -- 빠른 삭제!

-- MySQL: Range Partitioning (월별)
CREATE TABLE access_log (
    id BIGINT AUTO_INCREMENT,
    access_time DATETIME NOT NULL,
    user_id INT,
    url VARCHAR(500),
    status_code SMALLINT,
    PRIMARY KEY (id, access_time)
) PARTITION BY RANGE (TO_DAYS(access_time)) (
    PARTITION p202401 VALUES LESS THAN (TO_DAYS('2024-02-01')),
    PARTITION p202402 VALUES LESS THAN (TO_DAYS('2024-03-01')),
    PARTITION p202403 VALUES LESS THAN (TO_DAYS('2024-04-01')),
    PARTITION pmax VALUES LESS THAN MAXVALUE
);
```

## Ⅴ. 기대효과 및 결론

### 파티셔닝 기대 효과

| 효과 | 비파티셔닝 | 파티셔닝 |
|------|----------|--------|
| **Query 성능** | 전체 스캔 | 파티션만 |
| **관리** | 전체 Backup | 파티션 단위 |
| **Purge** | DELETE 느림 | DROP 빠름 |
| **인덱스** | 큼 | 작게 분할 |

### 파티셔닝 모범 사례

1. **적절한 키**: 쿼리 패턴 고려
2. **파티션 수**: 너무 많으면 관리 어려움
3. **로컬 인덱스**: 파티션별 인덱스
4. **모니터링**: 파티션 크기 균형
5. **유지보수**: 파티션 추가/삭제 자동화

### 미래 전망

1. **Auto Partitioning**: 자동 생성
2. **Hybrid Sharding**: 파티션 + 샤딩
3. **Cloud Native**: Managed DB 지원
