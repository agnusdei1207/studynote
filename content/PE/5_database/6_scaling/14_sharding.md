# 샤딩 (Sharding)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 샤딩은 **"대량 데이터를 **여러 데이터베이스 **서버(샤드)로 **수평 분할**하는 **확장 기법"**으로, **Shard Key**(분할 키)에 따라 **데이터 분배**되고 **Router**가 **요청**을 **적절한 샤드**로 **전달**한다.
> 2. **가치**: **단일 DB의 **한계**(CPU, Memory, Disk)를 **극복**하고 **읽기/쓰기 **성능을 **선형적으로 **확장**하며 **Geo-Distributed** 구성으로 **전 세계 **지연 시간**을 **단축**한다.
> 3. **융합**: **Hash Sharding**(일관된 해시), **Range Sharding**(범위 기반), **Directory-Based**(중앙 관리)가 있고 **MongoDB**(Sharded Cluster), **Vitess**(MySQL 샤딩), **Citus**(PostgreSQL)가 **구현**을 제공한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
샤딩은 **"데이터를 여러 서버로 분산 저장하는 기법"**이다.

**샤딩의 목적**:
- **확장성**: 수폭 확장
- **성능**: 병렬 처리
- **가용성**: 단일 장애 격리

### 💡 비유
샤딩은 **"여러 창고에 물건 분산"**과 같다.
- **단일 창고**: 용량 한계
- **여러 창고**: 분산 보관

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         샤딩의 필요성                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Single DB:
    • 모든 데이터 1곳
    • 확장 한계
         ↓
Vertical Scaling:
    • 서버 업그레이드
    • 비용 급증
         ↓
Sharding:
    • 수평 분할
    • 무한 확장 가능
```

### 샤딩 전략

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         샤딩 전략                                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Hash Sharding]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Hash(user_id) % N으로 분배                                                           │
    │  │  • 데이터 균등 분배                                                                   │
    │  │  • Rebalancing 어려움                                                               │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  shard_id = hash(user_id) % 3                                                       │  │
    │  │                                                                                       │  │
    │  │  user_id=1  → Shard 1                                                               │  │
    │  │  user_id=2  → Shard 2                                                               │  │
    │  │  user_id=3  → Shard 0                                                               │  │
    │  │  user_id=4  → Shard 1                                                               │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Range Sharding]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 범위 기반 분할                                                                       │
    │  │  • 쿼리 효율 높음 (Range Query)                                                      │
    │  │  • Skew 문제 (불균형)                                                               │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Shard 1: user_id 1 ~ 1000000                                                      │  │
    │  │  Shard 2: user_id 1000001 ~ 2000000                                                 │  │
    │  │  Shard 3: user_id 2000001 ~ 3000000                                                 │  │
    │  │                                                                                       │  │
    │  │  WHERE user_id BETWEEN 1500000 AND 1600000                                         │  │
    │  │  → Shard 2만 스캔!                                                                  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Directory-Based Sharding]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 중앙 Lookup Table                                                                    │
    │  │  • 유연한 Shard 이동                                                                 │
    │  │  │  • Lookup Table이 SPOF                                                           │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  [Directory Table]                                                                   │  │
    │  │  user_id    │  shard_id                                                            │  │
    │  │  ──────────────────────────────────────────────────────────────────────────────  │  │
    │  │  1          │  1                                                                   │  │
    │  │  2          │  2                                                                   │  │
    │  │  3          │  1                                                                   │  │  │
    │  │  4          │  3                                                                   │  │
    │  │  → Shard Key → Shard 매핑 정보                                                       │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Consistent Hashing

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Consistent Hashing                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Hash Ring에 Shard와 Key 배치                                                          │
    │  │  • Shard 추가/제거 시 최소한의 이동                                                     │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │                                                                                       │  │
    │  │          Shard A                                                                   │  │
    │  │        ┌────────────────────────────────────────────────────────────────────────┐    │  │
    │  │       ╱                                                                                    │  │
    │  │      ╱          Shard C                                                                │  │
    │  │     ╱        ┌────────────────────────────────────────────────────────────┐        │  │
    │  │    ╱        ╱                                                                             │  │
    │  │   ╱        ╱          Shard B                                                           │  │
    │  │  ╱        ╱        ┌────────────────────────────────────────────────────────┐        │  │
    │  │ ╱        ╱        ╱                                                                     │  │
    │  └───────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                       │  │
    │  [Shard 추가]                                                                          │  │
    │  • Shard D 추가 → Ring의 한 구간에 배치                                                   │  │
    │  • Shard C의 일부 Key만 Shard D로 이동                                                   │  │
    │  • Shard A, B는 영향 없음                                                               │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 샤딩 vs 파티셔닝 vs 복제

| 구분 | 샤딩 | 파티셔닝 | 복제 |
|------|------|--------|------|
| **범위** | 여러 DB | 단일 DB | 여러 DB |
| **데이터** | 분산 | 분할 | 동일 |
| **확장** | 수평 | 수직 | 읽기 |
| **복잡도** | 높음 | 중간 | 낮음 |

### 샤딩 문제점

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         샤딩 문제점                                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Skew]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Shard 간 데이터 불균형                                                                │
    │  │  • 특정 Shard에 부하 집중                                                             │
    │  │  → Rebalancing 필요                                                                 │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Hotspot]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 특정 Shard Key에 요청 집중                                                            │
    │  │  • 예: 유명인 사용자의 데이터                                                            │
    │  │  → Key 재설계 또은 추가 분할                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Cross-Shard Query]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 여러 Shard에 걸친 Query                                                             │
    │  │  • 성능 저하                                                                          │
    │  │  → Fan-out Query 또는 Broadcast                                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: MongoDB 샤딩
**상황**: 대량 데이터 저장
**판단**:

```javascript
// MongoDB Shard Cluster 구성
sh.enableSharding("mydb")

// Shard Key 선택
// 1. 카디널리티 높음
// 2. 분산 잘 됨
// 3. 자주 조회
sh.shardCollection("mydb.users", { user_id: 1 })

// Range Shard
sh.shardCollection("mydb.logs", { timestamp: 1 })

// Shard 확인
sh.status()
```

## Ⅴ. 기대효과 및 결론

### 샤딩 기대 효과

| 효과 | 비샤딩 | 샤딩 |
|------|--------|------|
| **확장성** | 한계 있음 | 무한 |
| **성능** | 저하 | 선형 확장 |
| **복잡도** | 낮음 | 높음 |
| **운영** | 단순 | 복잡 |

### 샤딩 모범 사례

1. **적절한 Shard Key**: 분산, 조회 패턴
2. **모니터링**: Skew 감지
3. **Rebalancing**: 자동화
4. **다중화**: Shard 복제
5. **Geo-Distribution**: 지리적 분산

### 미래 전망

1. **Auto-Sharding**: 자동 분산
2. **Polyglot Persistence**: 서비스별 DB
3. **Serverless Shard**: 동적 샤드

### ※ 참고 표준/가이드
- **MongoDB**: Sharding
- **Vitess**: MySQL Sharding
- **Citus**: PostgreSQL Extension
