# 복제 (Replication)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 복제는 **"데이터를 **여러 데이터베이스 **서버에 **복제하여 **고가용성**(High Availability)과 **내결합력**(Durability)을 **확보**하는 **기법"**으로, **Primary**(Master)에서 **Secondary**(Slave)로 **데이터를 **복사**하고 **장애 시 **Failover**로 **서비스 중단**을 **방지**한다.
> 2. **가치**: **Read Scalability**을 **향상**하고 **Geographic Distribution**으로 **지연 시간**을 **단축**하며 **Disaster Recovery**를 **위한 **Backup**으로 **활용**되고 **Multi-Leader**(Parallel Writes)로 **쓰기 확장**도 가능하다.
> 3. **융합**: **Synchronous Replication**(동기 복제)은 **Strong Consistency**를 **보장**하고 **Asynchronous Replication**(비동기 복제)는 **성능**을 **향상**하며 **MySQL**, **PostgreSQL**, **MongoDB**, **Redis**가 **각자 다른 방식**으로 **구현**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
복제는 **"데이터를 여러 서버에 복사하는 기술"**이다.

**복제의 목적**:
- **High Availability**: 장애 대비
- **Load Balancing**: 읽기 분산
- **Disaster Recovery**: 재해 복구
- **Backup**: 데이터 보호

### 💡 비유
복제는 **"문서 복사"**와 같다.
- **원본**: Primary DB
- **사본**: Secondary DB
- **목적**: 분실 방지

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         복제의 필요성                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Single DB:
    • SPOF (Single Point of Failure)
    • 장애 시 중단
         ↓
Master-Slave:
    • 1개 Master, 다수 Slave
    • 읽기 분산
    • Master 장애 시 수동 Failover
         ↓
Replica Set:
    • 자동 Failover
    • 자동 재선거
```

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐

    │  │  ┌─────────────────────────────┐                                                          │  │
    │  │  │  (Primary 장애 시)              │                                                          │  │
    │  │  │  → Replica들이 투표                 │                                                          │  │
    │  │  │  → 새 Primary 선출                 │                                                          │  │
    │  │  │  → 나머지 Replica들이 연결        │                                                          │  │
    │  │  └─────────────────────────────┘                                                          │  │

### 동기 vs 비동기 복제

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         동기 vs 비동기 복제                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Synchronous Replication]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Primary가 모든 Replica의 확인을 받을 때까지 대기                                         │
    │  │  • Strong Consistency 보장                                                            │
    │  │  • 쓰기 지연 증가                                                                       │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Client → Primary ─┬─> Replica 1 ──┬─> Replica 2 ──┬─> Replica 3                 │  │
    │  │                 │     ACK          │     ACK          │     ACK                  │  │
    │  │                 └───────┴──────────────┴──────────────┴─────→ ACK 모두 수신      │  │
    │  │                                                                  │                           │  │
    │  │                ────────────────────────> Client (Success)                    │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Asynchronous Replication]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Primary는 Replica 확인을 기다리지 않고 바로 응답                                          │
    │  │  • 빠른 쓰기                                                                          │
    │  │  • Replica Lag 발생 가능                                                              │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Client → Primary ─┬─> Replica 1                                     │  │
    │  │                 │     (비동기 복제 중)                             │  │
    │  │                 └───────> Client (Success)                       │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 복제 전략 비교

| 구분 | 동기 | 비동기 |
|------|------|--------|
| **일관성** | 강함 | 약함 |
| **성능** | 낮음 | 높음 |
|장애 복구| 쉬움 | 복잡 |
| **데이터 손실** | 없음 | 있을 수 있음 |

### 복제 모드

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         복제 모드                                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Statement-Based]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 각 SQL 문장을 복제                                                                    │  │
    │  │  • 유연성                                                                             │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Row-Based]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 변경된 Row만 복제                                                                    │  │
    │  │  • 효율적                                                                             │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Mixed]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 기본: Row-Based                                                                   │  │
    │  │  • 대량 DML/DDL: Statement-Based                                                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: PostgreSQL 복제
**상황**: HA 구성
**판단**:

```sql
-- Primary 설정 (postgresql.conf)
wal_level = replica
max_wal_senders = 3
synchronous_commit = on

-- Replica 설정
standby_mode = 'standby'
primary_conninfo = 'host=primary user=replicator password=xxx'

-- promotion 확인
SELECT pg_is_in_recovery();
```
