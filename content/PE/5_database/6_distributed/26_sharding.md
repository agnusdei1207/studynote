# 데이터베이스 샤딩 (Database Sharding)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 샤딩은 **"대용량 **데이터**를 **여러 **서버**에 **수평 **분할**하는 **확장 **기법"**으로, **Replication**(복제)이 **읽기 **확장**인 **반면** **Sharding**은 **쓰기 **확장**에 **특화**되어 있다.
> 2. **전략**: **Hash-based**(해시 **분할), **Range-based**(범위 **분할), **Geo-based**(지리적 **분할), **Directory-based**(디렉토리 **관리)로 **구분**하며 **Consistent Hashing**으로 **노드 **추가/제거**시 **재분배**를 **최소화**한다.
> 3. **문제**: **Cross-Shard Query**(분산 **조인), **Resharding**(재분할), **Skew**(불균형 **분포) **이슈**가 **발생**하며 **MongoDB**(Sharded Cluster), **Cassandra**(Token Ring), **Vitess**(MySQL)가 **해결책**을 **제공**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
샤딩은 **"데이터 수평 분할"**이다.

**확장 방식**:
| 구분 | Replication | Sharding |
|------|-------------|----------|
| **방향** | 수직 | 수평 |
| **확장** | 읽기 | 읽기 + 쓰기 |
| **용량** | 동일 데이터 증가 | 서버당 데이터 감소 |
| **복잡도** | 낮음 | 높음 |

### 💡 비유
샤딩은 ****여러 **서랍 **나누기 ****와 같다.
- **원본 서랍**: 단일 서버
- **나눈 서랍**: 샤드
- **분류**: Partition Key

### 샤딩 전략

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Sharding Strategies                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    1. Hash-Based Sharding:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  shard_id = hash(key) % num_shards                                                      │  │
    │                                                                                         │  │
    │  Example:                                                                                │  │
    │  user_id = 12345                                                                         │  │
    │  hash(12345) = 987654321                                                                │  │
    │  shard = 987654321 % 4 = 1                                                              │  │
    │  → Store in Shard 1                                                                      │  │
    │                                                                                         │  │
    │  Pros: Even distribution, simple routing                                                 │  │
    │  Cons: Resharding requires moving most data                                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. Range-Based Sharding:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Shard 0: user_id 0-999,999                                                              │  │
    │  Shard 1: user_id 1,000,000-1,999,999                                                    │  │
    │  Shard 2: user_id 2,000,000-2,999,999                                                    │  │
    │                                                                                         │  │
    │  Pros: Range queries efficient                                                           │  │
    │  Cons: Data skew, uneven distribution                                                    │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. Directory-Based Sharding:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Lookup Table:                                                                           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  user_id  │  shard_id                                                                │  │  │
    │  │  ─────────┼───────────                                                               │  │  │
    │  │  12345    │  1                                                                       │  │  │
    │  │  67890    │  3                                                                       │  │  │
    │  │  24680    │  0                                                                       │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Pros: Flexible, easy resharding                                                         │  │
    │  Cons: Extra lookup overhead, directory SPOF                                             │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

    │   │              │   │               │                                                 │  │
    │   │              │   │               │                                                 │  │

    │   │   (to C)     │   │   (to B)       │                                                 │  │
    │   │   key3→90    │   │   key4→180     │                                                 │  │
    │   │   (to A)     │   │   (to B)       │                                                 │  │
    │   │              │   │               │                                                 │  │

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 사용자 데이터 샤딩 설계
**상황**: 1억 사용자, 100GB 데이터
**판단**: Hash-based 샤딩

```python
# 샤드 라우팅
def get_shard(user_id):
    shard_count = 4
    shard_id = hash(user_id) % shard_count
    return f"shard_{shard_id}"

# 샤드 커넥션
shards = {
    "shard_0": "mysql-shard0.example.com",
    "shard_1": "mysql-shard1.example.com",
    "shard_2": "mysql-shard2.example.com",
    "shard_3": "mysql-shard3.example.com",
}

def query_user(user_id):
    shard_name = get_shard(user_id)
    connection = get_connection(shards[shard_name])
    return connection.execute(
        f"SELECT * FROM users WHERE id = {user_id}"
    )
```

## Ⅴ. 기대효과 및 결론

### 샤딩 기대 효과

| 효과 | 단일 서버 | 샤딩 |
|------|-----------|------|
| **저장 용량** | 제한 | 무한 |
| **쓰기 처리량** | 낮음 | 높음 |
| **복잡도** | 낮음 | 높음 |
| **조인** | 빠름 | 제한적 |

### 모범 사례

1. **Key 선택**: 고르게 분포되는 값
2. **금지**: Cross-shard 조인 피하기
3. **확장**: Consistent hashing
4. **모니터링**: Shard skew 확인

### 미래 전망

1. **Auto-sharding**: 자동 분할
2. **Geo-distribution**: 지리적 샤딩
3. **Federation**: 서비스별 샤딩
