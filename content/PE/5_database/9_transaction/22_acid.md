# ACID (트랜잭션)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ACID는 **"트랜잭션**이 **갖춰어야 **할 **4가지 **속성"**으로, **Atomicity**(원자성, All-or-Nothing), **Consistency**(일관성, Valid State), **Isolation**(독립성, Concurrent Execution), **Durability**(내구성, Permanent)으로 **구성**된다.
> 2. **Atomicity**: **COMMIT** 시 **모든 **변경**이 **적용**되거나 **ROLLBACK** 시 **모두 **취소**되고 **Consistency**는 **트랜잭션** **전후**로 **데이터베이스**가 **유효한 **상태**를 **유지**하며 **Referential Integrity**, **Domain Constraint**를 **보장**한다.
> 3. **Isolation**는 **동시 **실행**되는 **트랜잭션**끼리 **서로 **영향**을 **주지 **않게 **하고 **Durability**는 **COMMIT** 후 **영구적으로 **저장**되며 **WAL**(Write-Ahead Log)로 **구현**된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
트랜잭션은 **"논리적 작업 단위"**이다.

**ACID 4가지 속성**:
- **Atomicity**: All or Nothing
- **Consistency**: 유효 상태
- **Isolation**: 독립 실행
- **Durability**: 영구 저장

### 💡 비유
트랜잭션은 **"계좌 이체****와 같다.
- **입출금**: Atomic
- **잔고**: Consistency
- **동시 이체**: Isolation
- **영수증**: Durability

### ACID 상세

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         ACID Properties                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  A - Atomicity (원자성)                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  "모든 연산이 성공하거나 모두 실패"                                                     │  │  │
    │  │                                                                                       │  │  │
    │  │  BEGIN;                                                                                 │  │  │
    │  │  UPDATE accounts SET balance = balance - 100 WHERE id = 1;                                 │  │  │
    │  │  UPDATE accounts SET balance = balance + 100 WHERE id = 2;                                 │  │  │
    │  │  COMMIT;  -- 또는 ROLLBACK;                                                            │  │  │
    │  │                                                                                       │  │  │
    │  │  → 실패 시 아무 일도 일어나지 않음                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  C - Consistency (일관성)                                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  "트랜잭션 전후로 데이터베이스가 유효한 상태"                                               │  │  │
    │  │                                                                                       │  │  │
    │  │  • Primary Key 고유                                                                  │  │  │
    │  │  │  • Foreign Key 제약                                                                │  │  │
    │  │  │  • CHECK 제약                                                                     │  │  │
    │  │  │  → 위반 시 ROLLBACK                                                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  I - Isolation (독립성)                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  "동시 실행되는 트랜잭션끼리 서로 간섭 없음"                                           │  │  │
    │  │                                                                                       │  │  │
    │  │  Isolation Levels:                                                                  │  │  │

    │  │  │  • Read Uncommitted: dirty reads 가능                                             │  │  │
    │  │  │  │  • Read Committed: dirty reads 방지                                                │  │  │
    │  │  │  │  • Repeatable Read: phantom reads 방지                                          │  │  │
    │  │  │  │  • Serializable: 완전 독립                                                         │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  D - Durability (내구성)                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  "COMMIT 후 영구적으로 저장"                                                          │  │  │
    │  │                                                                                       │  │  │
    │  │  WAL (Write-Ahead Logging):                                                           │  │  │
    │  │  1. Log에 먼저 기록                                                                   │  │  │
    │  │  2. 데이터베이스 파일에 반영                                                           │  │  │
    │  │  → 장애 발생해도 Log로 복구                                                             │  │  │

    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Isolation Level

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Isolation Levels & Anomalies                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Isolation Level        │  Dirty Read  │  Non-Repeatable  │  Phantom        │    │  │
    │                         │  (미완료)      │  Read            │  Read          │    │  │
    │  ───────────────────────────────────────────────────────────────────────────────────  │  │  │
    │  Read Uncommitted       │      O        │       O          │      O         │    │  │
    │  Read Committed         │      X        │       O          │      O         │    │  │
    │  Repeatable Read        │      X        │       X          │      O         │    │  │
    │  Serializable           │      X        │       X          │      X         │    │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Dirty Read: 다른 트랜잭션의 미커밋 데이터를 읽음
    Non-Repeatable Read: 같은 데이터를 다시 읽을 때 값이 다름
    Phantom Read: 이전에 없던 행이 새로 나타남
```

## Ⅲ. 융합 비교 및 다각도 분석

### Isolation Level 비교

| Level | 성능 | Anomaly | 사용처 |
|-------|------|---------|--------|
| **Read Uncommitted** | 높음 | O | 거의 없음 |
| **Read Committed** | 중간 | 일부 | 일반적 |
| **Repeatable Read** | 낮음 | Phantom | 분석 |
| **Serializable** | 가장 낮음 | X | 재무 |

### ACID vs BASE

| 구분 | ACID | BASE |
|------|------|------|
| **목적** | 정확성 | 가용성 |
| **일관성** | 강함 | 최종적 |
| **Isolation** | 강함 | 약함 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 계좌 이체
**상황**: ACID 보장
**판단**:

```sql
BEGIN TRANSACTION;

-- Step 1: 출금 (계좌 확인)
UPDATE accounts
SET balance = balance - 100
WHERE id = 1 AND balance >= 100;

-- Step 2: 입금
UPDATE accounts
SET balance = balance + 100
WHERE id = 2;

-- 성공 시에만 커밋
COMMIT;
-- 또는 실패 시 ROLLBACK;
```

## Ⅴ. 기대효과 및 결론

### ACID 기대 효과

| 속성 | 없음 | 있음 |
|------|------|-----|
| **데이터 무결성** | 위험 | 안전 |

| **영구성** | 손실 | 보존 |

### 모범 사례

1. **짧게**: Lock 시간
2. **Isolation**: 적절한 수준
3. **Retry**: 재시 로직

1. **Distributed 트랜잭션**: 2PC, Saga
2. **NewSQL**: ACID + Scale
3. **Cloud**: Managed
