# 클러스터링 (Clustering)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클러스터링은 **"여러 데이터베이스 **서버가 **하나의 **시스템처럼 **동작하는 **구성"**으로, **Load Balancer**가 **읽기**를 **분산**하고 **Data Replication**으로 **데이터 동기화**하며 **Failover**로 **고가용성**(HA)을 **달성**한다.
> 2. **가치**: **단일 장애점**(SPOF)를 **제거**하고 **Read Scaling**(읽기 확장)을 **지원**하며 **Geographic Distribution**으로 **전 세계 **서비스**를 **제공**하고 **Planned Maintenance** 중 **무중단** **업데이트**가 **가능**하다.
> 3. **융합**: **Primary-Replica**(Active-Passive), **Primary-Primary**(Active-Active)가 **아키텍처**이며 **MySQL InnoDB Cluster**, **PostgreSQL Patroni**, **MongoDB Replica Set**가 **구현**을 **제공**하고 **ProxySQL**,HAProxy**, **Pgpool-II**가 **Routing**을 **담당**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
클러스터링은 **"여러 DB가 하나처럼 동작하는 구성"**이다.

**클러스터링 목적**:
- **High Availability**: 장애 시 대체 서버 가용
- **Load Balancing**: 읽기 분산
- **Scalability**: 수평 확장
- **Data Redundancy**: 데이터 중복 저장

### 💡 비유
클러스터링은 **"수영 관제팀"**과 같다.
- **팀**: 여러 DB
- **감독**: Load Balancer
- **협업**: 공동 목표

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         클러스터링 발전                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Single DB:
    • SPOF
    • 다운타임
         ↓
Active-Passive:
    • 1개 Primary, 다수 Replica
    • 자동 Failover
         ↓
Active-Active:
    • 모든 Node가 쓰기 가능
    • 완전 활용
```

## Ⅱ. 아키텍처 및 핵심 원리

### Primary-Replica (Active-Passive)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Primary-Replica Cluster                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Write                 Read                 Read                 Read                    │  │
    │  │    │                     │                     │                            │  │
    │  │    ▼                     ▼                     ▼                            │  │
    │  │  Primary              Replica 1             Replica 2             Replica 3               │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  • Read/Write       │  │  │  │  │  │  │  │  │
    │  │  │  • Data Source       │  │  │  │  │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                              │                         │  │
    │  │                         Replication Stream                                   │  │
    │  │                                                              ▼                         │  │
    │  │                     ┌───┴────┬────────┴────┬───────────────┐                           │  │
    │  │                     │Replica1│Replica2│Replica3│                           │  │
    │  │                     └───────┴────────┴────────┴───────────────────┘                           │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                  │  │
    │  [Failover]                                                                      │  │
    │  • Primary 장애 시                                                                  │  │
    │  → Replica 투표 → 새 Primary 선출                                                     │  │
    │  → 나머지 Replica가 새 Primary가 됨                                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Active-Active Cluster

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Active-Active Cluster                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Write                 Write                 Write                 Write                    │  │
    │  │    │                     │                     │                            │  │  │
    │  │    ▼                     ▼                     ▼                            │  │
    │  │  Node 1               Node 2               Node 3               Node 4                 │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  • 모두 Read/Write 가능                                                              │  │  │
    │  │  │  • Conflict Resolution (해시, 타임스탬프, MVCC)                                  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                  │  │
    │  [Load Balancer]                                                                  │  │
    │  • 모든 Node에 분산                                                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 클러스터링 비교

| 구분 | Active-Passive | Active-Active |
|------|---------------|---------------|
| **쓰기** | Primary만 | 모든 Node |
| **읽기** | Replica만 | 모든 Node |
| **복잡도** | 낮음 | 높음 |
| **성능** | 중간 | 최대 |

### Cluster Nodes 수 결정

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Nodes 수 결정 요인                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    1. Write Load
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 쓰기가 많으면 Node 수 증가 필요                                                            │  │
    │  • Read가 많으면 Replica만 늘려도 됨                                                      │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. Read/Write Ratio
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Read-heavy: 1 Primary + N Replica                                                     │  │
    │  • Write-heavy: Node 수 많아도 병목                                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. Geographic Distribution
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 지리별 배치로 지연 시간 단추                                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: PostgreSQL Cluster
**상황**: HA 구성
**판단**:

```sql
-- Patroni 설정
-- dcs 3 replicas

-- 읽기 라우팅
CREATE USER replica WITH REPLICATION;
CREATE USER load_balancer WITH PASSWORD 'password';

-- HBA 설정
host    replication     replica          10.0.0.0/0      scram-sha-256
host    all             all              10.0.0.0/0      scram-sha-256

-- Replica 구독절 후 Patroni 확인
SELECT * FROM pg_stat_replication;
```
