# 백업 (Backup)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 백업은 **"데이터 **손실**에 **대비**하여 **데이터베이스 **사본**을 **생성**하고 **보관**하는 **기술"**으로, **Full Backup**(전체), **Incremental Backup**(증분), **Differential Backup**(차분)으로 **구분**되며 **RPO**(Recovery Point Objective)와 **RTO**(Recovery Time Objective)를 **만족**해야 한다.
> 2. **Point-in-Time Recovery**(PITR)로 **특정 시점**으로 **복구**가능하고 **Logical Backup**(mysqldump, pg_dump)와 **Physical Backup**(베이스 백업, Snapshot)로 **구현**되며 **Retention Policy**(보관 정책)으로 **저장 공간**을 **관리**한다.
> 3. **융합**: **MySQL**(mysqldump, Percona XtraBackup), **PostgreSQL**(pg_dump, WAL Archiving), **Oracle**(RMAN), **MongoDB**(mongodump)가 **대표적**이며 **Cloud Backup**(AWS RDS Backup, GCP Automate Backup)로 **자동화**된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
백업은 **"데이터 사본을 만드는 기술"**이다.

**백업 목적**:
- **재해 복구**: DR (Disaster Recovery)
- **데이터 보호**: 손실 방지
- **감사**: 데이터 보존
- **테스트**: 개발 환경

### 💡 비유
백업은 **"자동차 보험****과 같다.
- **사고**: 데이터 손실
- **보험**: 백업
- **복구**: 보험금

## Ⅲ. 융합 비교 및 다각도 분석

### 백업 유형 비교

| 구분 | Full | Incremental | Differential |
|------|------|-------------|--------------|
| **속도** | 느림 | 빠름 | 중간 |
| **복구** | 빠름 | 느림 | 중간 |
| **공간** | 큼 | 작음 | 중간 |

### Logical vs Physical

| 구분 | Logical | Physical |
|------|---------|-----------|
| **단위** | SQL | File |
| **속도** | 느림 | 빠름 |
| **크기** | 큼 | 작음 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: PostgreSQL 백업
**상황**: PITR 구성
**판단**:

```bash
# 1. WAL Archiving
postgresql.conf:
wal_level = replica
archive_mode = on
archive_command = 'cp %p /wal_archive/%f'

# 2. Base Backup (Physical)
pg_basebackup -D /backup/base -Ft -z -P

# 3. Logical Backup (SQL)
pg_dump -Fc mydb > /backup/mydb.dump

# 4. Recovery (PITR)
restore_command = 'cp /wal_archive/%f %p'
recovery_target_time = '2026-03-06 10:00:00'
```

## Ⅴ. 기대효과 및 결론

### 백업 기대 효과

| 효과 | 없음 | 백업 |
|------|------|------|
| **데이터 손실** | 치명적 | 복구 가능 |
| **RPO** | 길음 | 짧음 |
| **비용** | 낮음 | 높음 |

### 모범 사례

1. **3-2-1 규칙**: 3 사본, 2 미디어, 1 외부
2. **암호화**: 백업 보안
3. **테스트**: 정기적 복구
4. **모니터링**: 백업 확인

### 미래 전망

1. **Continuous Backup**: 실시간
2. **Cloud**: 자동화
3. **Immutable**: 변경 불가

### ※ 참고 표준/가이드
- **PostgreSQL**: pg_dump, pg_basebackup
- **MySQL**: mysqldump, XtraBackup
- **Oracle**: RMAN
