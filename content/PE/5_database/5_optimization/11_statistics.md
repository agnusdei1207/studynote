# 통계 정보 (Statistics)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 통계 정보는 **"테이블의 **데이터 분포**(Data Distribution)와 **카디널리티**(Cardinality)를 **수집한 정보"**로, **Query Optimizer**가 **실행 계획**을 **수립**할 때 **비용 추정**(Cost Estimation)의 **기초**가 된다.
> 2. **가치**: **정확한 통계**는 **최적의 실행 계획**을 선택하게 하고 **오래된 통계**는 **비효율적인 계획**(Full Table Scan 대신 Index Scan을 선택 못 함)을 **유도**하며 **ANALYZE**, **UPDATE STATISTICS**로 **갱신**한다.
> 3. **융합**: **Histogram**(히스토그램)으로 **데이터 분포**를 **표현**하고 **Most Common Values**(MCV), **Null Fraction**, **Average Width**를 **수집**하며 **Auto-vacuum**(PostgreSQL), **Auto-update statistics**(SQL Server)로 **자동 갱신**된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
통계 정보는 **"테이블과 인덱스에 대한 데이터 분포 통계"**이다.

**수집 항목**:
- **Row Count**: 테이블 전체 행 수
- **Cardinality**: 고유값 개수
- **Histogram**: 값 분포
- **Null Fraction**: NULL 비율

### 💡 비유
통계 정보는 **"도로 교통 상황"**과 같다.
- **통계 없음**: 막무가내 출발
- **오래된 통계**: 알 수 없는 정체
- **정확한 통계**: 최적 경로 선택

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         통계 정보의 필요성                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

System R Optimizer (1970년대):
    • 단순 추정 (균등 분포 가정)
    • 부정확한 계획
         ↓
Histogram (1980년대):
    • 데이터 분포 반영
    • 개선된 비용 추정
         ↓
현대 DBMS:
    • 다중 컬럼 통계 (Multi-column)
    • 자동 갱신 (Auto-vacuum)
```

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Histogram                                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Equi-Height Histogram]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 각 버킷이 비슷한 빈도 (Row Count)                                                     │
    │  │  • 범위가 다름                                                                       │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Bucket  Min      Max      Count                                                 │  │
    │  │  ────────────────────────────────────────────────────────────────────────────────  │  │
    │  │  1        1        1000     1000                                                  │  │
    │  │  2        1001     5000     1000                                                  │  │
    │  │  3        5001     20000    1000                                                  │  │
    │  │  4        20001    100000   1000                                                  │  │
    │  │  5        100001   1000000  1000                                                  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  예: WHERE age = 25 → Bucket 1에 있음 → Selectivity ≈ 1/1000 → 1000 rows 추정              │
    │      WHERE age = 30000 → Bucket 3에 있음 → Selectivity ≈ 1/1000 → 1000 rows 추정           │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Equi-Width Histogram]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 각 버킷이 같은 범위                                                                 │
    │  │  • 빈도가 다름                                                                       │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Bucket  Min      Max      Count                                                 │  │
    │  │  ────────────────────────────────────────────────────────────────────────────────  │  │
    │  │  1        1        200000   950000                                                │  │
    │  │  2        200001   400000   40000                                                 │  │
    │  │  3        400001   600000   8000                                                  │  │
    │  │  4        600001   800000   1500                                                  │  │

    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
```

### Selectivity & Cardinality

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Selectivity & Cardinality                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Selectivity: 조건에 맞는 행의 비율 (0 ~ 1)                                            │
    │  │  • Cardinality: 조건에 맞는 예상 행 수                                                │
    │  │  • Cardinality = Row Count × Selectivity                                            │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  # 예시                                                                               │  │
    │  │  테이블: users (1,000,000 rows)                                                      │  │
    │  │                                                                                       │  │
    │  │  [Equal Condition]                                                                  │  │
    │  │  WHERE status = 'active'                                                            │  │
    │  │  • MCV: 'active' = 60%                                                               │  │
    │  │  • Selectivity = 0.6                                                                │  │
    │  │  • Cardinality = 1,000,000 × 0.6 = 600,000 rows                                      │  │
    │  │                                                                                       │  │
    │  │  [Range Condition]                                                                  │  │
    │  │  WHERE created_at BETWEEN '2026-01-01' AND '2026-03-06'                              │  │
    │  │  • Histogram 기반 추정                                                                │  │
    │  │  • 범위가 전체의 10% 커버                                                             │  │
    │  │  • Selectivity ≈ 0.1                                                                │  │
    │  │  • Cardinality = 100,000 rows                                                        │  │
    │  │                                                                                       │  │
    │  │  [AND Condition]                                                                    │  │
    │  │  WHERE status = 'active' AND created_at > '2026-01-01'                               │  │
    │  │  • Selectivity = 0.6 × 0.1 = 0.06 (독립 가정)                                         │  │
    │  │  • Cardinality = 60,000 rows                                                         │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

| 상황 | 좋은 통계 | 나쁜 통계 |

### 다중 컬럼 통계 (Multi-column Statistics)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         다중 컬럼 통계                                                    │

    │  │  컬럼: country, city                                                                │  │
    │  │                                                                                       │  │
    │  │  단일 컬럼 통계:                                                                      │  │
    │  │  - country = 'KR': 1000 rows (10%)                                                   │  │
    │  │  - city = 'Seoul': 500 rows (5%)                                                     │  │
    │  │                                                                                       │  │
    │  │  WHERE country = 'KR' AND city = 'Seoul'                                            │  │
    │  │  추정: 10% × 5% = 0.5% → 50 rows (독립 가정)                                          │  │

    │  │                                                                                       │  │
    │  │  # 다중 컬럼 통계 (PostgreSQL)                                                       │  │

    │  │  → 상관관계를 반영한 정확한 추정                                                       │  │
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 통계 갱신
**상황**: 대량 데이터 입력 후 느린 쿼리
**판단**:

```sql
-- PostgreSQL
-- 1. 통계 정보 확인
SELECT schemaname, tablename, n_live_tup, last_autovacuum, last_autoanalyze
FROM pg_stat_user_tables
WHERE tablename = 'orders';

-- 2. 통계 수동 갱신
ANALYZE orders;

-- 3. 특정 컬럼만 갱신
ANALYZE orders (status, created_at);

-- 4. 샘플링 비율 조정
ANALYZE orders WITH (sample_percent = 100);

-- 5. 통계 정보 확인
SELECT attname, n_distinct, correlation, null_frac
FROM pg_stats
WHERE tablename = 'orders';


-- MySQL
-- 1. 통계 갱신
ANALYZE TABLE orders;

-- 2. 테이블 정보 확인
SHOW TABLE STATUS LIKE 'orders';

-- 3. 인덱스 통계 갱신
ANALYZE TABLE orders UPDATE HISTOGRAM ON status, created_at WITH 100 BUCKETS;


-- SQL Server
-- 1. 통계 갱신
UPDATE STATISTICS orders WITH FULLSCAN;

-- 2. 전체 데이터베이스 통계 갱신
EXEC sp_updatestats;

-- 3. 통계 정보 확인
DBCC SHOW_STATISTICS ('orders', idx_orders_status);
```

## Ⅴ. 기대효과 및 결론

### 통계 정보 기대 효과

| 효과 | 오래된 통계 | 최신 통계 |
|------|-----------|---------|
| **실행 계획** | 비효율적 | 최적 |
| **Index Scan** | 선택 안 됨 | 적절히 선택 |
| **Join Method** | 잘못된 선택 | 올바른 선택 |

### 통계 정보 관리 모범 사례

1. **정기 갱신**: Auto-vacuum 활용
2. **대량 작업 후**: 즉시 ANALYZE
3. **샘플링 조정**: 큰 테이블은 샘플링
4. **다중 컬럼 통계**: 상관관계 있으면 생성
5. **모니터링**: pg_stats 확인

### 미래 전망

1. **AI Estimation**: 머신러닝 기반
2. **Adaptive Statistics**: 자동 튜닝
3. **Real-time Stats**: 실시간 수집

### ※ 참고 표준/가이드
- **PostgreSQL**: ANALYZE
- **MySQL**: ANALYZE TABLE
- **SQL Server**: UPDATE STATISTICS
