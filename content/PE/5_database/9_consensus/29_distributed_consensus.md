# 분산 합의 (Distributed Consensus)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 합의는 **"분산 **시스템**에서 **노드 **간 **합의**를 **도출**하는 **알고리즘"**으로, **Safety**(모든 **노드가 **같은 **결정), **Liveness**(시스템 **진행), **Fault Tolerance**(f **노드 **고장 **허용)을 **보장**한다.
> 2. **알고리즘**: **Paxos**(Leslie Lamport, **Basic **Paxos), **Raft**(이해 **용이, **Leader **기반), **ZAB**(Zookeeper **Atomic **Broadcast), **PBFT**(Byzantine, **블록체인 **용)가 **대표적**이다.
> 3. **Raft**: **Leader Election**(리더 **선출), **Log Replication**(로그 **복제), **Safety**(Leader **완전성, **Log Matching) **속성**으로 **구현**되며 **etcd**, **Consul**, **TiKV**에 **적용**된다.

## Ⅰ. 개요 (Context & Background)

### 개념
분산 합의는 **"분산 시스템 동기화"**이다.

**FLP Impossibility**:
- **F**ault-tolerant (고장 허용)
- **L**atency (지연 없음)
- **P**artition-tolerant (분할 내성)
→ **동시에 3가지 불가능**

- **안건**: 제안
- **과반수**: Quorum

## Ⅱ. 아키텍처 및 핵심 원리

### Raft Algorithm

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Raft Consensus                                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    States:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌─────────┐  ┌─────────┐  ┌─────────┐                                                     │  │
    │  │ Follower │ → Candidate │ → Leader  │    Follower                                    │  │
    │  └─────────┘  └─────────┘  └─────────┘                                                     │  │
    │       │            ▼            │                                                        │  │
    │       │        Request votes    │                                                        │  │
    │       │                          │                                                        │  │
    │       └──────────────────────────┘                                                        │  │
    │                  Heartbeat timeout → New election                                            │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Leader Election:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Term 1:                                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Node A (Follower)          Node B (Candidate)         Node C (Follower)                 │  │  │
    │  │  ┌───────────────────┐     ┌───────────────────────┐     ┌───────────────────┐         │  │  │
    │  │  │ timeout →          │     │ vote for self        │     │                   │         │  │  │
    │  │  │ Candidate           │     │ term=2, vote=A       │     │                   │         │  │  │
    │  │  └───────────────────┘     └───────────────────────┘     └───────────────────┘         │  │  │
    │  │           │                       │                           │                          │  │  │
    │  │           ▼ Request vote         │                           ▼ vote for B       │  │  │
    │  │  ┌───────────────────────────────────────────────────────────────────────────────┐   │  │  │
    │  │  │ B → A: "Vote for me (term 2)"                                                     │   │  │  │
    │  │  │ B → C: "Vote for me (term 2)"                                                     │   │  │  │
    │  │  │ A → B: "Vote for you (term 2)"                                                    │   │  │  │
    │  │  │ C → B: "Vote for you (term 2)"                                                    │   │  │  │
    │  │  │                                                                                       │   │  │  │
    │  │  │ B receives 2 votes (A + C + B = majority of 3) → Become Leader                │   │  │  │
    │  │  └───────────────────────────────────────────────────────────────────────────────┘   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Log Replication

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Raft Log Replication                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Client Request                                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Client → Leader: SET x = 10                                                        │  │  │  │
    │  │  │  Leader → Log: SET x = 10 (index 1, term 1)                                      │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  Leader → Followers: AppendEntries RPC                                                │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Leader                    Follower 1                Follower 2                       │  │  │  │
    │  │  │  ┌────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  │ Log: [SET x=5]             Log: [SET x=5]             Log: [SET x=5]             │  │  │  │  │
    │  │  │  │ Index: 1                  Index: 1                  Index: 1                  │  │  │  │  │
    │  │  │  │ Term: 1                   Term: 1                   Term: 1                   │  │  │  │  │
    │  │  │  └────────────────────────────────────────────────────────────────────────────┘  │  │  │  │
    │  │  │       │                          │                          │                        │  │  │  │
    │  │  │       └────────────AppendEntries────────┼──────────AppendEntries────────┤  │  │  │
    │  │  │                                  │                          │                        │  │  │  │
    │  │  │                           ▼                          ▼                        │  │  │  │
    │  │  │                     Commit                      Commit                   │  │  │  │
    │  │  │                                  │                          │                        │  │  │  │
    │  │  │                                  └────────────┬───────────────┘                        │  │  │  │
    │  │  │                                               │                                │  │  │  │
    │  │  │                                               ▼                                │  │  │  │
    │  │  │                                    Leader ←───────┴───────AppendEntries OK│  │  │  │
    │  │  │                                                                                       │  │  │  │
    │  │  │  → Once majority committed, Leader responds to client                          │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Paxos Basic Protocol

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Paxos Phases                                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Phase 1: Prepare (Promise)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Proposer → Acceptors: Prepare(n)                                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Acceptor N:                                                                                 │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  if n > promised_n:                                                                │  │  │  │
    │  │  │    promised_n = n                                                                  │  │  │  │
    │  │  │    return Promise(n, accepted_value)                                              │  │  │  │
    │  │  │  else:                                                                               │  │  │  │
    │  │  │    return Reject(n)                                                                 │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Proposer waits for majority promises                                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Phase 2: Accept
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Proposer → Acceptors: Accept(n, value)                                                    │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Acceptor N:                                                                                 │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  if n >= promised_n:                                                               │  │  │  │
    │  │  │    accepted_n = n                                                                  │  │  │  │
    │  │  │    accepted_value = value                                                          │  │  │  │
    │  │  │    return Accept(n)                                                                  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Proposer waits for majority accepts → Learning achieved                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### etcd Raft API

```bash
# etcdctl put key
etcdctl put /config/key "value"

# Background (Raft):
# 1. Leader receives PUT request
# 2. Leader logs entry: {"Op":"PUT","Key":"/config/key","Value":"value"}
# 3. Leader replicates to followers (AppendEntries)
# 4. Followers commit log
# 5. Leader applies to state machine (key-value store)
# 6. Leader responds to client

# Watch for changes
etcdctl watch /config/key --rev

# History
etcdctl get /config/key --rev
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 분산 KV 스토어
**상황**: Kubernetes etcd
**판단**: Raft 기반

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         etcd in Kubernetes                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  etcd Cluster (3 nodes for fault tolerance)                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Node 1 (Leader)          Node 2 (Follower)        Node 3 (Follower)                   │  │  │
    │  │  ┌───────────────────────┐  ┌───────────────────────┐  ┌───────────────────────┐       │  │  │
    │  │  │ Log Index: 10          │  │ Log Index: 10          │  │ Log Index: 10          │       │  │  │
    │  │  │ Term: 1                 │  │ Term: 1                 │  │ Term: 1                 │       │  │  │
    │  │  └───────────────────────┘  └───────────────────────┘  └───────────────────────┘       │  │  │
    │  │           │                       │                       │                          │  │  │
    │  │           └───────────────┬───────┴───────────────────────┘                          │  │  │
    │  │                           │                                                          │  │  │
    │  │                           ▼ Heartbeat (100ms)                                         │  │  │
    │  │                           │                                                          │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Kubernetes API Server writes config: deployment replicas = 3                      │  │  │  │
    │  │  │  → Leader writes to log                                                            │  │  │  │
    │  │  │  → Followers commit log                                                             │  │  │  │
    │  │  │  → All nodes apply to state machine                                               │  │  │  │
    │  │  │  → Consistent cluster state achieved                                             │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Tolerates 1 node failure (3-1=2, majority)                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```
