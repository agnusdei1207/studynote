# 속성 (Attribute)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 속성(Attribute)은 개체(Entity) 또는 관계(Relationship)이 가진 **데이터의 가장 작은 의미 단위**로, **도메인(Domain)**이라는 원자값(Atomic Value)의 집합으로 정의되며, **단순 속성(Simple)**, **복합 속성(Composite)**, **저장/도출 속성(Stored/Derived)**으로 분류된다.
> 2. **가치**: **도메인 무결성(Domain Integrity)**을 보장하는 기본 단위로, 허용되는 값의 **타입(DataType)**과 **크기(Length)**, **NULL 허용 여부**, **기본값(Default)**, **제약조건(Constraint)**을 정의하며, **데이터 사전(Data Dictionary)**에 메타데이터로 등록된다.
> 3. **융합**: **개체-관계 매핑**, **정규화**, **ORM(Object-Relational Mapping)**의 기초가 되며, **스냅샷(Snapshot) 읽기**, **점진적 커밋(Incremental Update)**, **병렬 제어(Concurrency Control)**에 필수적인 데이터 무결성 보장의 최전선 방어선이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
속성(Attribute)은 **"개체나 관계가 가진 데이터의 특성(Properties)으로, 데이터베이스에 저장되는 가장 작은 데이터 단위"**이다.

**속성의 정의**:
- 개체(Entity): EMPLOYEE → 속성: name, salary, hire_date
- 관계(Relationship): enrolls in → 속성: grade

**도메인(Domain)**: 속성이 가질 수 있는 원자값의 집합
- INT 도메인: {1, 2, 3, ...}
- VARCHAR(100) 도메인: 100자 이하의 문자열 집합

### 💡 비유
속성은 **"사람의 이름, 나이, 성별"**과 같다.
- **이름**: 문자열 도메인 (예: "홍길동", "김철수")
- **나이**: 정수 도메인 (예: 30, 25, 40)
- **성별**: 고정 도메인 (예: 'M', 'F')

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         속성 개념의 진화                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

파일 시스템:
    • 속성: 레코드, 구조체, 파일 크기
    • 데이터 타입: 고정 길이, 구조체
    • 무결성: 파일 포맷에 의존

관계형 데이터베이스:
    • 속성: 컬럼, 도메인, 제약조건
    • 데이터 독립성: 스키마로 데이터와 프로그램 분리
    • 무결성: DBMS가 보장 (ACID)
```

## Ⅱ. 아키텍처 및 핵심 원리

### 속성의 분류

| 분류 기준 | 속성 유형 | 설명 | 예시 |
|----------|----------|------|------|
| **단순 vs 복합** | 단순 속성(Simple) | 단일 값 | name, salary |
| | 복합 속성(Composite) | 다른 속성의 조합 | 주소 (도시+시+구+동) |
| **저장 여부** | 저장 속성(Stored) | DB에 물리적으로 저장 | name, birth_date |
| | 도출 속성(Derived) | 다른 속성으로부터 계산 | age (현재년 - 출생년) |
| **필수 여부** | 필수 속성(Required) | NULL 불가 | employee_id, dept_id |
| | 선택 속성(Optional) | NULL 허용 | middle_name, nickname |
| **키 여부** | 키 속성(Key) | 식별자 역할 | employee_id, dept_id |
| | 비키 속성(Non-Key) | 일반 속성 | name, email |

### 심층 동작 원리

#### 1. 단순 속성 vs 복합 속성

**단순 속성(Simple Attribute)**:
- 원자적 indivisible 값
- 예: name, salary, birth_date
- 테이블의 한 컬럼으로 저장

**복합 속성(Composite Attribute)**:
- 여러 속성의 조합
- 예: 주소 (Address)
  - 우편번호(VARCHAR), 도시(VARCHAR), 구(CHAR), 동(CHAR)
- 설계 선택:
  - 단순 속성으로 분리: 우편번호, 도시, 구, 동
  - 복합 속성으로 유지: address 컬럼 (JSON)

#### 2. 저장 속성 vs 도출 속성

**저장 속성(Stored Attribute)**:
- 물리적으로 저장
- 수정 시 UPDATE 필요
- SELECT로 조회

**도출 속성(Derived Attribute)**:
- 다른 속성으로부터 계산
- 변경 시 자동 갱신
- VIEW, VIRTUAL COLUMN로 구현

```sql
-- 저장 속성
CREATE TABLE employees (
    birth_date DATE NOT NULL,
    age INT GENERATED ALWAYS AS (YEAR(CURRENT_DATE) - YEAR(birth_date)) STORED
);

-- 도출 속성 (VIEW 또는 VIRTUAL COLUMN)
CREATE VIEW v_employee_age AS
SELECT
    employee_id,
    name,
    birth_date,
    YEAR(CURRENT_DATE) - YEAR(birth_date) AS age
FROM employees;
```

#### 3. 속성의 제약조건

**NULL 제약조건**:
- 허용 여부 지정
- `NOT NULL`: NULL 비허용
- `NULL`: NULL 허용

**DEFAULT 제약조건**:
- 기본값 지정
- `DEFAULT 'Seoul'`, `DEFAULT NOW()`

**CHECK 제약조건**:
- 조건식 제약
- `CHECK (salary >= 0)`

**UNIQUE 제약조건**:
- 중복 비허
- `UNIQUE (email)`

## Ⅲ. 융합 비교 및 다각도 분석

### 속성별 무결성 제약조건

| 무결성 종류 | 목적 | 제약조건 예시 | 위반 시 |
|----------|------|-----------|--------|
| **개체 무결성** | 기본키 uniqueness | PK 제약 | PK 중복 |
| **참조 무결성** | FK 관계성 | FK 제약 | FK 무효 참조 |
| **도메인 무결성** | 타입 유효성 | CHECK 제약 | 잘못된 데이터 |
| **사용자 정의** | 비즈니스 규칙 | TRIGGER, CHECK | 업무 위배 |

### 과목 융합 관점 분석

#### 1. 컴퓨터 구조 ↔ 속성
- **메모리 할당**: 속성 타입에 따른 바이트 수
- **정렬(Alignment)**: 성능 최적화를 위한 속성 정렬
- **패딩(Padding)**: 구조체 맞춤을 위한 더미 데이터 추가

#### 2. 프로그래밍 언어 ↔ 속성
- **타입 시스템**: 언어별 내장/사용자 정의 타입
- **ORM 매핑**: 객체의 속성 → 테이블의 컬럼

## Ⅴ. 기대효과 및 결론

### 속성 설계 기대 효과

| 설계 측면 | 잘못된 속성 설계 | 올바른 속성 설계 |
|----------|---------------|---------------|
| **타입** | VARCHAR(1) BOOL | CHAR(1), ENUM |
| **길이** | VARCHAR(10) name | VARCHAR(100) name |
| **NULL 허용** | 필수 컬럼에 NULL | 선택 컬럼에 NULL |
| **DEFAULT** | 없음 | DEFAULT NOW() |

### 미래 전망

1. **JSON 속성**: PostgreSQL JSONB, MySQL JSON
2. **ARRAY 속성**: PostgreSQL ARRAY, VARRAY
3. **Computed Columns**: 지속적으로 증가

### ※ 참고 표준/가이드
- **SQL-92**: 속성 정의 표준
- **ANSI/ISO SQL**: 도메인, 무결성 제약
- **GDPR**: 개인정보 보호를 위한 속성 설계
