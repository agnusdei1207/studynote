# 인덱스 (Index)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인덱스는 **"검색 **성능**을 **향상**시키는 **데이터 **구조"**로, **B-Tree**, **Hash Index**, **Bitmap Index**로 **구현**되며 **WHERE**, **JOIN**, **ORDER BY** **절**에 **사용**되는 **열**에 **생성**하여 **Full Table Scan**을 **방지**한다.
> 2. **종류**: **Primary Key**(기본 키, **Unique**), **Secondary Index**(보조 인덱스), **Clustered**(데이터 정렬), **Non-Clustered**(별도 저장), **Composite**(다중 열)로 **구분**되며 **Cardinality**(기수도)가 **높은**열에 **생성**할 **때 **효과**가 **크다.
> 3. **단점**: **INSERT/UPDATE/DELETE** 시 **인덱스 **유지보수**(Overhead)가 **발생**하고 **저장 공간**이 **필요**하며 **불절된 **인덱스**는 **성능**을 **저하**시키므로 **주기적으로 **사용**하는 **쿼리**에 **생성**해야 한다.

## Ⅰ. 개요 (Context & Background)

### 개념
인덱스는 **"데이터 검색을 위한 구조"**이다.

**인덱스 목적**:
- **검색**: 빠른 찾기
- **정렬**: 순서 유지
- **유니크**: 중복 방지

## Ⅱ. 아키텍처 및 핵심 원리

### B-Tree 인덱스

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         B-Tree Index Structure                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  B+ Tree (Database Index)                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Internal Nodes (Internal)  Page                                                      │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  [10, 20, 30, 40, 50]                │  Leaf Node Pointer                            │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │         │  │  │  │  │  │  │  │  │                                              │  │  │
    │  │         ▼  ▼  ▼  ▼  ▼  ▼  ▼  ▼  ▼                                                          │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Leaf Nodes (Data Pages)                      │                                            │  │  │
    │  │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  │  [1, *]    [5, *]    [11, *]   [16, *]   [21, *]                           │  │  │  │  │
    │  │  │  │  Linked List (Sorted)                                                        │  │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Binary Search: O(log n)                                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Clustered vs Non-Clustered

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Index Types                                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Clustered Index (Primary Key):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Table (Sorted by ID)                                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  id │  name   │  email                                                   │  │  │  │
    │  │  ────┼────────┼───────────────────────┐                                           │  │  │  │
    │  │   1  │ Alice  │ alice@ex.com             │                                           │  │  │  │
    │  │  2  │  Bob    │ bob@ex.com               │                                           │  │  │  │
    │  │  │  3  │ Carol  │ carol@ex.com             │                                           │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  → 데이터가 정렬된 상태로 저장 (Leaf Node)                                               │  │
    │  → Range Scan 빠름                                                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Non-Clustered Index (Secondary):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Index (Sorted by name)                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  name │ id         │ pointer                                                   │  │  │  │
    │  │  ─────┼──────────┼───────────────────────┐                                           │  │  │
    │  │  Alice  │ 1         │ → Row 1                                                    │  │  │  │
    │  │  Bob    │ 2         │ → Row 2                                                    │  │  │  │
    │  │  Carol  │ 3         │ → Row 3                                                    │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  → 별도 저장, Random Access                                                           │  │
    │  → Range Scan 느림                                                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 인덱스 타입 비교

| 타입 | 구조 | 검색 | 삽입 | 사용처 |
|------|------|------|------|--------|
| **B-Tree** | 균형 트리 | O(log n) | O(log n) | 범용 |
| **Hash** | 해시 테이블 | O(1) | O(1) | 동등 |
| **Bitmap** | 비트맵 | O(1) | O(1) | 플래그 |

### Cardinality와 인덱스

| Cardinality | 인덱스 효과 |
|-------------|------------|
| **높음** (Unique ID) | 매우 좋음 |
| **중간** (Status) | 좋음 |
| **낮음** (Gender) | 비효율적 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 인덱스 생성
**상황**: 검색 최적화
**판단**:

```sql
-- 단일 인덱스
CREATE INDEX idx_users_email ON users(email);

-- 복합 인덱스
CREATE INDEX idx_orders_user_date ON orders(user_id, created_at);

-- 유니크 인덱스
CREATE UNIQUE INDEX idx_users_id ON users(id);

-- 확인 (EXPLAIN)
EXPLAIN SELECT * FROM users WHERE email = 'user@example.com';
```

## Ⅴ. 기대효과 및 결론

### 인덱스 기대 효과

| 효과 | 없음 | 있음 |
|------|------|------|
| **검색** | 전체 스캔 | 인덱스 스캔 |
| **INSERT** | 빠름 | 느림 |
| **저장 공간** | 작음 | 큼음 |

### 모범 사례

1. **필수**: WHERE, JOIN
2. **주의**: 너무 많은 인덱스
3. **모니터링**: 사용 확인
4. **재구**: 주기적 실행

### 미래 전망

1. **Adaptive Index**: 자동 선택
2. **Full-Text Search: 텍스트
3. **Vector Index**: 임베딩
