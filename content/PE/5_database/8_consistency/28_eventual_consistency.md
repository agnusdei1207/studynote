# 결과적 일관성 (Eventual Consistency)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 결과적 일관성은 **"분산 **시스템**에서 **데이터 **업데이트**가 **시스템 **전체**에 **전파**되면 **최종적으로 **일관성**이 **보장**된다"는 **모델**로, **즉시 **일관성**(Strong)을 **포기**하고 **가용성**을 **우선**한다.
> 2. **특성**: **쓰기 **가능**(항상 **성공), **읽기 **지연**(오래된 **데이터 **가능), **충돌 **해결**(Conflict Resolution), **복구 **가능**(Partition **후 **동기화)이 **특징**이다.
> 3. **구현**: **Quorum**(N/2+1 **노드 **합의), **Version Vector**(버전 **추적), **CRDT**(Conflict-free Replicated Data Types), **Read Repair**(읽기 **시 **복구), **Anti-Entropy**(비동기 **동기화)로 **구현**된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
결과적 일관성은 **"시간 지연 일관성 보장"**이다.

**일관성 수준**:
| 수준 | 일관성 | 지연 | 사용처 |
|------|--------|------|--------|
| **Strong** | 즉시 | 높음 | 금융 거래 |
| **Eventual** | 최종 | 낮음 | SNS, 캐시 |
| **Weak** | 보장 X | 최저 | 로그 |

### 💡 비유
결과적 일관성은 ****뉴스 **전파 ****와 같다.
- **사건**: 데이터 업데이트
- **지연**: 전파 시간
- **최종**: 모두가 알게 됨

### 결과적 일관성 시나리오

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Eventual Consistency Scenario                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  User updates name: "Alice" → "Alice Smith"                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Replicas (N=3)                                                                      │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Node A           Node B           Node C                                      │  │  │  │
    │  │  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐                             │  │  │  │
    │  │  │  │ name:       │  │ name:       │  │ name:       │                             │  │  │  │
    │  │  │  │ Alice Smith │  │ Alice       │  │ Alice       │  → Just updated            │  │  │  │
    │  │  │  │ (v2)        │  │ (v1)        │  │ (v1)        │                             │  │  │  │
    │  │  │  └─────────────┘  └─────────────┘  └─────────────┘                             │  │  │  │
    │  │  │                                                                                       │  │  │  │
    │  │  │  T0: User writes to A (W=1)                                                           │  │  │  │
    │  │  │                                                                                       │  │  │  │
    │  │  │  T1: Read from A → "Alice Smith" (v2) ✓                                            │  │  │  │
    │  │  │                                                                                       │  │  │  │
    │  │  │  T2: Read from B → "Alice" (v1) ✗ (stale)                                         │  │  │  │
    │  │  │                                                                                       │  │  │  │
    │  │  │  T3: Async replication A → B, C                                                       │  │  │  │
    │  │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  │  Node A           Node B           Node C                                      │  │  │  │  │
    │  │  │  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐                             │  │  │  │  │
    │  │  │  │  │ Alice Smith │  │ Alice Smith │  │ Alice Smith │                             │  │  │  │  │
    │  │  │  │  │ (v2)        │  │ (v2)        │  │ (v2)        │                             │  │  │  │  │
    │  │  │  │  └─────────────┘  └─────────────┘  └─────────────┘                             │  │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │  │
    │  │  │                                                                                       │  │  │  │
    │  │  │  T4: Read from B, C → "Alice Smith" (v2) ✓ (consistent)                             │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Eventually consistent (brief period of inconsistency)                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Quorum (합의)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Quorum Reads/Writes                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  N = 3 replicas, W = 2 writes, R = 2 reads                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Write (W=2):                                                                          │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Client → Node A (write v2) ✓                                                      │  │  │  │
    │  │  │  Client → Node B (write v2) ✓                                                      │  │  │  │
    │  │  │  Node C (not contacted, will catch up later)                                      │  │  │  │
    │  │  │  → Write succeeds (2/3 = W)                                                       │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  Read (R=2):                                                                           │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Node A: v2, Node B: v2, Node C: v1                                                  │  │  │  │
    │  │  │  → Return latest (v2, from A or B)                                                  │  │  │  │
    │  │  │  → Read repair: Update C to v2                                                       │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  Condition for strong consistency: R + W > N                                           │  │  │
    │  │  → If R=2, W=2, N=3: 2 + 2 > 3 ✓ Strong read                                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 동기화 기법

| 기법 | 설명 | 오버헤드 |
|------|------|----------|
| **Read Repair** | 읽기 시 오래된 노드 업데이트 | 읽기 시 |

| **Anti-Entropy** | 주기적 전체 동기화 | 백그라운드 |

### Version Vector

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Version Vector                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Data: {"name": "Alice"}                                                                │  │
    │  Version Vector: [{node: A, version: 1}, {node: B, version: 2}]                            │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Node A updates: name = "Alice Smith" (A:1 → A:2)                                     │  │  │
    │  │  Version Vector: [{A:2}, {B:2}]                                                        │  │  │
    │  │                                                                                       │  │  │
    │  │  Node B updates: name = "Alice J" (B:2 → B:3)                                         │  │  │
    │  │  Version Vector: [{A:2}, {B:3}]                                                        │  │  │
    │  │                                                                                       │  │  │
    │  │  Conflict Detection:                                                                   │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  A's version: {A:2, B:2} → "Alice Smith"                                            │  │  │  │
    │  │  │  B's version: {A:1, B:3} → "Alice J"                                                  │  │  │  │
    │  │  │  → Neither dominates: Conflict!                                                     │  │  │  │
    │  │  │  → Resolution: "Alice Smith J" (merge) or timestamp (last write wins)                │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### CRDT (Conflict-free Replicated Data Types)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         CRDT Example (G-Counter)                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  G-Counter (Grow-only Counter)                                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  State: [{A: 5}, {B: 3}]  (Node A counted 5, Node B counted 3)                       │  │  │
    │  │  Value: 5 + 3 = 8                                                                     │  │  │
    │  │                                                                                       │  │  │
    │  │  Node A increment:                                                                     │  │  │
    │  │  • Local: [{A: 6}, {B: 3}] = 9                                                        │  │  │
    │  │  • Send to B: merge                                                                    │  │  │
    │  │  • B receives: max(A:6, A:5) = A:6, max(B:3, B:3) = B:3                                 │  │  │
    │  │  • B state: [{A: 6}, {B: 3}] = 9  (consistent!)                                      │  │  │
    │  │                                                                                       │  │  │
    │  │  → No conflicts, always mergeable                                                     │  │  │
    │  │  → Converges to same value eventually                                                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### Cassandra 일관성 수준

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Cassandra Consistency Levels                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Write Consistency:                                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  ANY: At least 1 node (hinted handoff if replica down)                              │  │  │
    │  │  ONE: At least 1 replica                                                           │  │  │
    │  │  QUORUM: (N/2 + 1) replicas                                                         │  │  │
    │  │  LOCAL_QUORUM: Quorum in local datacenter                                            │  │  │
    │  │  ALL: All replicas                                                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Read Consistency:                                                                      │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  ONE: Fastest, may return stale                                                      │  │  │
    │  │  QUORUM: Returns latest acknowledged write                                           │  │  │
    │  │  ALL: Returns latest write (slowest)                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Example (N=3, RF=3):                                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  CL=ONE, R=ONE: Low latency, eventual consistency                                     │  │  │
    │  │  CL=QUORUM, R=QUORUM: Stronger consistency (R+W>N)                                  │  │  │
    │  │  CL=ALL, R=ALL: Strongest consistency, slowest                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 결과적 일관성 기대 효과

| 효과 | Strong Consistency | Eventual Consistency |
|------|-------------------|----------------------|
| **가용성** | 낮음 | 높음 |
| **지연** | 높음 | 낮음 |
| **복잡도** | 낮음 | 높음 |

### 모범 사례

1. **SNS**: 댓글, 좋아요
2. **캐시**: Redis, Memcached
3. **CDN**: 컨텐츠 배포
4. **DNS**: 레코드 전파

### 미래 전망

1. **CRDT**: 충돌 없는 데이터 타입
2. **CALM**: Consistency as Logical Monotonicity
3. **Hybrid**: Strong + Eventual
