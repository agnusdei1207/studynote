# 시계열 데이터베이스 (Time-Series Database)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시계열 DB는 **"시간 **축**으로 **정렬**된 **데이터**를 **저장**하고 **쿼리**하는 **DB\"**로, **Timestamp**(시간 **정보)**, **Measurement**(측정 **값)**, **Tags**(색인 **가능 **속성)**, **Fields**(측정 **값)**으로 **구성**되고 **모니터링**, **IoT**, **금융**에 **사용**된다.
> 2. **기술**: **LSM Tree**(Log-Structured Merge-Tree)**로 **쓰기 **성능**을 **최적화**하고 **Downsampling**(데이터 **샘플링)**, **Rollup**(시간 **단위 **통합)**, **Retention**(데이터 **보관 **기간)**으로 **저장소**를 **관리**한다.
> 3. **제품**: **InfluxDB**(Open Source, **Flux** 쿼리), **Prometheus**(Pull-based, **PromQL)**, **TimescaleDB**(PostgreSQL **Extension)**가 **대표적**이며 **Grafana**와 **통합**하여 **시각화**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
시계열 DB는 **"시간 순서 데이터 저장소"**이다.

**RDBMS vs TSDB**:
| 구분 | RDBMS | TSDB |
|------|-------|------|
| **주 키** | ID | Timestamp |
| **쓰기** | 무작위 | 순차적 |
| **압축** | 낮음 | 높음 |
| **삭제** | 수동 | 자동 |

### 💡 비유
TSDB는 ****의료 **기록부 ****와 같다.
- **시간**: 진료 시각
- **측정**: 혈압, 체온
- **태그**: 환자 ID

### LSM Tree (Log-Structured Merge Tree)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         LSM Tree Architecture                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Write Path:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Write Request                                                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Append to MemTable (in-memory, sorted by time)                                   │  │  │
    │  │     ┌──────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │     │  MemTable (max size: 256MB)                                                   │  │  │  │
    │  │     │  │  time                 │ value                                             │  │  │  │
    │  │     │  │  ─────────────────────────────────────────────────────────────────────────│  │  │  │
    │  │     │  │  2026-03-06 10:15:30 │ 75.5                                              │  │  │  │
    │  │     │  │  2026-03-06 10:15:31 │ 78.2                                              │  │  │  │
    │  │     │  │  ...                                                                          │  │  │  │
    │  │     │  └──────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │     └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │           │                                                                             │  │  │
    │  │           ▼ MemTable full                                                              │  │  │
    │  │  2. Flush to Sorted String Table (SSTable) on disk                                    │  │  │
    │  │     ┌──────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │     │  SSTable 1 (immutable)                                                         │  │  │  │
    │  │     │  • Sorted by time                                                               │  │  │  │
    │  │     │  • Compressed (Snappy, ZSTD)                                                    │  │  │  │
    │  │     │  • Bloom filter for fast lookup                                                 │  │  │  │
    │  │     └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  3. Compact SSTables (merge sort)                                                      │  │  │
    │  │     ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  SSTable 1 + SSTable 2 → SSTable 3 (merged)                                        │  │  │  │
    │  │  │  → Reduces number of files                                                         │  │  │  │
    │  │  │  → Removes overwritten/deleted data                                                │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → O(1) write performance (append-only, sequential write)                                │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Downsampling & Rollup

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Downsampling Strategy                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Problem: Raw data grows indefinitely
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Raw data: 1 second resolution                                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  2026-03-06 10:15:30: value=75.5                                                    │  │  │
    │  │  2026-03-06 10:15:31: value=78.2                                                    │  │  │
    │  │  ... (86,400 points per day)                                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  → 1 year = 31 million points per metric                                               │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Solution: Tiered downsampling
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Raw Data (1s resolution): Retain 7 days                                                 │  │
    │     │                                                                                    │  │
    │     ▼ Rollup: Average 1 minute                                                          │  │
    │  1m Data (60s resolution): Retain 30 days                                               │  │
    │     │                                                                                    │  │
    │     ▼ Rollup: Average 5 minutes                                                         │  │
    │  5m Data (300s resolution): Retain 365 days                                              │  │
    │     │                                                                                    │  │
    │     ▼ Rollup: Average 1 hour                                                            │  │
    │  1h Data (3600s resolution): Retain 5 years                                              │  │
    │                                                                                         │  │
    │  Storage reduction:                                                                      │  │
    │  • Raw: 31M points/year → 1h: 8,760 points/year (3534x reduction)                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Aggregation functions:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  │  Time Range   │  Mean  │  Min   │  Max   │  Sum   │  Count                         │  │
    │  │  ─────────────────────────────────────────────────────────────────────────────────────│  │
    │  │  10:15:00-    │  76.8  │  70.2  │  82.1  │  768   │  10                            │  │
    │  │  10:15:59     │        │        │        │        │                                │  │
    │  │  (1 minute)  │        │        │        │        │                                │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### PromQL Examples

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         PromQL Query Examples                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  rate(http_requests_total[5m])                                                          │  │
    │  → Calculates requests per second over last 5 minutes                                    │  │
    │                                                                                         │  │

    2. Aggregation:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  sum(rate(http_requests_total[5m])) by (service)                                        │  │
    │  → Total request rate grouped by service                                                 │  │
    │                                                                                         │  │
    │  avg(cpu_usage) by (datacenter)                                                          │  │
    │  → Average CPU usage per datacenter                                                      │  │

    3. Threshold alert:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  avg(cpu_usage) > 80                                                                     │  │
    │  → Alert when average CPU usage exceeds 80%                                             │  │
    │                                                                                         │  │
    │  rate(http_requests_total{status="500"}[5m]) > 0.1                                     │  │
    │  → Alert when 5xx error rate exceeds 0.1 req/s                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 애플리케이션 모니터링
**상황**: 마이크로서비스 지표 수집
**판단**: Prometheus + Grafana

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Application Monitoring Stack                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Components:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Instrumentation (expose metrics)                                                    │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  # Go application (client_golang)                                                    │  │  │
    │  │  import "github.com/prometheus/client_golang/prometheus"                             │  │  │
    │  │                                                                                       │  │  │
    │  │  var (                                                                                  │  │  │
    │  │      httpRequestsTotal = prometheus.NewCounterVec(                                    │  │  │
    │  │          prometheus.CounterOpts{                                                       │  │  │
    │  │              Name: "http_requests_total",                                             │  │  │
    │  │              Help: "Total HTTP requests",                                              │  │  │
    │  │          },                                                                          │  │  │
    │  │          []string{"method", "status"},                                                 │  │  │
    │  │      )                                                                                  │  │  │
    │  │      httpDuration = prometheus.NewHistogramVec(                                        │  │  │
    │  │          prometheus.HistogramOpts{                                                     │  │  │
    │  │              Name: "http_request_duration_seconds",                                     │  │  │
    │  │              Help: "HTTP request latency",                                              │  │  │
    │  │              Buckets: []float64{0.1, 0.5, 1, 2, 5, 10},                                 │  │  │
    │  │          },                                                                          │  │  │
    │  │          []string{"endpoint"},                                                         │  │  │
    │  │      )                                                                                  │  │  │
    │  │  )                                                                                      │  │  │
    │  │                                                                                       │  │  │
    │  │  func handler(w http.ResponseWriter, r *http.Request) {                                │  │  │
    │  │      start := time.Now()                                                               │  │  │
    │  │      // ... handle request ...                                                         │  │  │
    │  │      httpRequestsTotal.WithLabelValues(r.Method, "200").Inc()                           │  │  │
    │  │      httpDuration.WithLabelValues(r.URL).Observe(time.Since(start).Seconds())           │  │  │
    │  │  }                                                                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  2. Prometheus Configuration                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  scrape_configs:                                                                      │  │  │
    │  │    - job_name: 'api-server'                                                           │  │  │
    │  │      scrape_interval: 15s                                                             │  │  │
    │  │      static_configs:                                                                   │  │  │
    │  │        - targets: ['api:8080']                                                         │  │  │
    │  │          labels:                                                                       │  │  │
    │  │            service: 'api'                                                              │  │  │
    │  │            env: 'production'                                                           │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  3. Alerting Rules                                                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  groups:                                                                              │  │  │
    │  │    - name: api_alerts                                                                 │  │  │
    │  │      rules:                                                                           │  │  │
    │  │        - alert: HighErrorRate                                                        │  │  │
    │  │          expr: rate(http_requests_total{status="500"}[5m]) > 0.1                     │  │  │
    │  │          for: 5m                                                                       │  │  │
    │  │          annotations:                                                                  │  │  │
    │  │            summary: "High 5xx error rate"                                              │  │  │
    │  │        - alert: HighLatency                                                           │  │  │
    │  │          expr: histogram_quantile(0.95, http_request_duration_seconds) > 1             │  │  │
    │  │          for: 5m                                                                       │  │  │
    │  │          annotations:                                                                  │  │  │
    │  │            summary: "95th percentile latency > 1s"                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### TSDB 기대 효과

| DB | 쓰기 성능 | 쿼리 성능 | 압축 |
|----|----------|----------|------|
| **InfluxDB** | 높음 | 높음 | 우수 |
| **Prometheus** | 높음 | 중간 | 우수 |
| **TimescaleDB** | 중간 | 우수 | 좋음 |

### 모범 사례

1. **태그 설계**: Low cardinality
2. **압축**: Gorilla encoding
3. **다운샘플링**: 자동화
4. **보관**: 정책 설정

### 미래 전망

1. **Distributed**: 분산 TSDB
2. **Serverless**: Mimir, Cortex
3. **Edge**: Edge TSDB
4. **AI**: 시계열 예측
