# 함수적 종속 (Functional Dependency)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 함수적 종속(Functional Dependency, FD)은 **"속성 X의 값이 결정되면 속성 Y의 값이 유일하게 결정되는 관계"**로, **X → Y**로 표현하며 **정규화(Normalization)**의 **이론적 기초**가 된다.
> 2. **가치**: **암스트롱의 공리(Armstrong's Axioms)**로 함수적 종속의 **함수적 폐포(Functional Closure)**를 구하고 **최소 커버(Minimal Cover)**를 찾아 **중복을 제거**하며, **제1정규형~BCNF**의 **판정 기준**이 된다.
> 3. **융합**: **데이터 마이닝(Data Mining)**의 **연관 규칙(Association Rule)**, **XML Schema**, **종속성 데이터베이스(Deductive Database)**의 **함수적 종속 논리**로 확장되며, **ORM(Object-Relational Mapping)**의 **Lazy Loading**, **Eager Loading** 최적화의 기반이 된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
함수적 종속은 **"한 속성의 값이 다른 속성의 값을 결정하는 제약 조건"**이다.

**함수적 종속의 정의**:
- **X → Y**: X가 Y를 **함수적으로 결정(functionally determine)**
- **결정자(Determinant)**: X (함수적 종속의 왼쪽)
- **종속자(Dependent)**: Y (함수적 종속의 오른쪽)

### 💡 비유
함수적 종속은 **"우편번호 → 주소"**와 같다.
- **우편번호(100-1001)**: **서울 강남구**로 결정됨
- **하지만 주소 → 우편번호**: X (한 동에 여러 우편번호)

### 등장 배경

```
관계형 모델 (1970년, E.F. Codd):

    • 데이터 무결성 보장

    • 1NF, 2NF, 3NF 정의

암스트롱의 공리 (1974년):

    • 최소 커버 찾기
         ↓
현대 데이터베이스:
    • 정규화 자동화 도구
    • 함수적 종속 발견 알고리즘
```

### 암스트롱의 공리(Armstrong's Axioms)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         암스트롱의 공리                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [기본 공리 (Primitive Axioms)]

    1. 반사성 (Reflexivity)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  X ⊇ Y이면 X → Y                                                                        │
    │                                                                                         │
    │  예: {student_id, name} ⊇ {name}                                                         │
    │      → (student_id, name) → name                                                         │
    │                                                                                         │
    │  의미: 속성 집합이 자기 자신을 포함하면 함수적 종속                                        │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. 증가성 (Augmentation)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  X → Y이면 XZ → YZ                                                                      │
    │                                                                                         │
    │  예: student_id → name                                                                   │
    │      → (student_id, email) → (name, email)                                               │
    │                                                                                         │
    │  의미: 함수적 종속의 양쪽에 같은 속성을 추가해도 성립                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. 이행성 (Transitivity)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  X → Y, Y → Z이면 X → Z                                                                  │
    │                                                                                         │
    │  예: student_id → dept_id                                                                │
    │      dept_id → dept_name                                                                 │
    │      → student_id → dept_name                                                           │
    │                                                                                         │
    │  의미: 함수적 종속이 연쇄적으로 전달                                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [추론된 공리 (Derived Axioms)]

    4. 합병성 (Union)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  X → Y, X → Z이면 X → YZ                                                                │
    │                                                                                         │
    │  예: student_id → name                                                                   │
    │      student_id → email                                                                  │
    │      → student_id → (name, email)                                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    5. 분해성 (Decomposition or Pseudo-transitivity)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  X → YZ이면 X → Y, X → Z                                                                │
    │                                                                                         │
    │  예: dept_id → (dept_name, building)                                                     │
    │      → dept_id → dept_name                                                               │
    │         dept_id → building                                                               │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 함수적 폐포(Functional Closure)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         함수적 폐포 계산                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [정의]

    속성 집합 X에 대해 함수적으로 종속된 모든 속성의 집합
    X⁺ = {A | X → A가 성립}

    [예시 계산]

    주어진 함수적 종속:
    • A → B
    • B → C
    • AD → E
    • E → C

    A⁺ (A의 폐포) 계산:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. 초기: A⁺ = {A}                                                                     │
    │  2. A → B 적용: A⁺ = {A, B}                                                            │
    │  3. B → C 적용: A⁺ = {A, B, C}                                                         │
    │  4. 더 이상 적용할 FD 없음                                                               │
    │  결과: A⁺ = {A, B, C}                                                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    (AD)⁺ 계산:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. 초기: (AD)⁺ = {A, D}                                                               │
    │  2. A → B 적용: (AD)⁺ = {A, B, D}                                                      │
    │  3. B → C 적용: (AD)⁺ = {A, B, C, D}                                                    │
    │  4. AD → E 적용: (AD)⁺ = {A, B, C, D, E}                                               │
    │  5. E → C 적용 (이미 C 포함)                                                            │
    │  6. 더 이상 적용할 FD 없음                                                               │
    │  결과: (AD)⁺ = {A, B, C, D, E}                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [활용: 후보키 찾기]

    속성 집합 X가 후보키(Candidate Key)인 조건:
    X⁺ = R (모든 속성 집합)

    예: R = {A, B, C, D, E}
    (AD)⁺ = {A, B, C, D, E} = R → AD는 후보키
```

### 함수적 종속 유형

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         함수적 종속 유형                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [1. 완전 함수적 종속 (Full Functional Dependency)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Y가 X에 완전 함수적으로 종속                                                              │
    │  • X의 진부분집합에 대해 함수적 종속이 성립하지 않음                                         │
    │                                                                                         │
    │  예: (student_id, course_id) → grade                                                     │
    │  • student_id → grade ✗                                                                 │
    │  │  course_id → grade ✗                                                                │
    │  │  (student_id, course_id) → grade ✓                                                    │
    │                                                                                         │
    │  → grade는 (student_id, course_id)에 완전 함수적 종속                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [2. 부분 함수적 종속 (Partial Functional Dependency)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Y가 X의 일부에만 함수적 종속                                                               │
    │  • 진부분집합에 대해 함수적 종속이 성립                                                       │
    │                                                                                         │
    │  예: (student_id, course_id) → name                                                      │
    │  • student_id → name ✓                                                                  │
    │  │  course_id → name ✗                                                                  │
    │                                                                                         │
    │  → name은 (student_id, course_id)에 부분 함수적 종속                                       │
    │  → 제2정규형(2NF) 위배                                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [3. 이행적 함수적 종속 (Transitive Dependency)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  X → Y, Y → Z이며 X → Z                                                                   │
    │                                                                                         │
    │  예: student_id → dept_id → dept_name                                                    │
    │  • student_id → dept_name은 이행적 함수적 종속                                             │
    │                                                                                         │
    │  → 제3정규형(3NF) 위배                                                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 정규화 판정 기준

| 정규형 | 함수적 종속 조건 | 문제 |
|--------|-----------------|------|
| **1NF** | 모든 속성이 원자값 | 반복 그룹 |
| **2NF** | 부분 종속 제거 | 데이터 중복 |
| **3NF** | 이행적 종속 제거 | 갱신 이상 |
| **BCNF** | 모든 결정자가 후보키 | 중복 후보키 |

### 과목 융합 관점 분석

#### 1. 데이터 마이닝 ↔ 함수적 종속
- **연관 규칙(Association Rule)**: X → Y (확률적)
- **함수적 종속**: X → Y (결정론적)

#### 2. 데이터베이스 설계 ↔ 함수적 종속
- **ER 모델**: 개체-관계 → 함수적 종속
- **정규화**: 함수적 종속 분해

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 함수적 종속 발견 및 정규화
**상황**: 대량 데이터 중복 발견
**판단**:
1. **함수적 종속 추출**:
   - 데이터 분석으로 FD 발견
   - `product_id → (category, supplier)`
2. **정규화 수행**:
   - 테이블 분해
   - 중복 제거

```sql
-- 비정규화 (문제)
CREATE TABLE products (
    product_id INT,
    product_name VARCHAR(100),
    category VARCHAR(50),
    category_desc VARCHAR(200),
    supplier VARCHAR(100),
    supplier_addr VARCHAR(200)
);

-- 정규화 (3NF)
CREATE TABLE products (
    product_id INT PRIMARY KEY,
    product_name VARCHAR(100),
    category_id INT,
    supplier_id INT,
    FOREIGN KEY (category_id) REFERENCES categories(id),
    FOREIGN KEY (supplier_id) REFERENCES suppliers(id)
);

CREATE TABLE categories (
    category_id INT PRIMARY KEY,
    category_name VARCHAR(50),
    category_desc VARCHAR(200)
);

CREATE TABLE suppliers (
    supplier_id INT PRIMARY KEY,
    supplier_name VARCHAR(100),
    supplier_addr VARCHAR(200)
);
```
