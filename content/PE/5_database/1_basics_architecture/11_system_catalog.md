# 시스템 카탈로그 (System Catalog)

## 정의

**시스템 카탈로그(System Catalog)**는 데이터베이스의 **메타데이터(데이터에 대한 데이터)**를 저장하는 시스템 테이블 집합이다. 테이블 구조, 인덱스, 사용자 권한, 무결성 제약조건 등 데이터베이스 객체 정보를 관리한다. 데이터 사전(Data Dictionary)이라고도 한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           시스템 카탈로그 개요                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   사용자/응용프로그램                                    │ │
│   │        │                                                │ │
│   │        ▼                                                │ │
│   │   ┌─────────────────────────────────────────────────┐  │ │
│   │   │            DBMS 엔진                             │  │ │
│   │   └────────────────────┬────────────────────────────┘  │ │
│   │                        │                                │ │
│   │         ┌──────────────┴──────────────┐                │ │
│   │         ▼                             ▼                │ │
│   │   ┌───────────────┐           ┌───────────────┐        │ │
│   │   │ 시스템 카탈로그 │           │   사용자      │        │ │
│   │   │ (메타데이터)   │           │   데이터      │        │ │
│   │   │               │           │               │        │ │
│   │   │ • 테이블 정의  │           │ • 테이블      │        │ │
│   │   │ • 인덱스 정보  │           │ • 인덱스      │        │ │
│   │   │ • 사용자 정보  │           │ • 뷰          │        │ │
│   │   │ • 권한 정보    │           │               │        │ │
│   │   └───────────────┘           └───────────────┘        │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. DBMS별 시스템 카탈로그 구현

```
┌─────────────────────────────────────────────────────────────────┐
│           DBMS별 시스템 카탈로그                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Oracle:                                                      │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Data Dictionary Views (DBA_, ALL_, USER_ 접두어)       │ │
│   │   • DBA_TABLES: 관리자 관점 테이블 목록                  │ │
│   │   • ALL_TABLES: 접근 가능한 테이블                       │ │
│   │   • USER_TABLES: 소유한 테이블                          │ │
│   │   • DBA_INDEXES, DBA_USERS, DBA_TAB_COLUMNS 등          │ │
│   │                                                         │ │
│   │   SELECT * FROM USER_TABLES;                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   MySQL/MariaDB:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   INFORMATION_SCHEMA 데이터베이스                        │ │
│   │   • TABLES: 테이블 정보                                 │ │
│   │   • COLUMNS: 컬럼 정보                                  │ │
│   │   • STATISTICS: 인덱스 통계                              │ │
│   │   • USER_PRIVILEGES: 사용자 권한                         │ │
│   │                                                         │ │
│   │   SELECT * FROM INFORMATION_SCHEMA.TABLES               │ │
│   │   WHERE TABLE_SCHEMA = 'mydb';                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   PostgreSQL:                                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   System Catalogs (pg_ 접두사)                           │ │
│   │   • pg_class: 테이블, 인덱스, 시퀀스                     │ │
│   │   • pg_attribute: 컬럼                                  │ │
│   │   • pg_index: 인덱스                                    │ │
│   │   • pg_user: 사용자                                     │ │
│   │                                                         │ │
│   │   SELECT * FROM pg_class WHERE relname = 'students';    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   SQL Server:                                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   System Views                                           │ │
│   │   • sys.tables: 테이블                                  │ │
│   │   • sys.columns: 컬럼                                   │ │
│   │   • sys.indexes: 인덱스                                 │ │
│   │   • sys.database_principals: 사용자                     │ │
│   │                                                         │ │
│   │   SELECT * FROM sys.tables;                             │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 카탈로그 관리 특성

```
┌─────────────────────────────────────────────────────────────────┐
│           시스템 카탈로그 관리 특성                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   자동 관리:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • CREATE TABLE → 카탈로그에 테이블 정보 자동 추가       │ │
│   │   • DROP TABLE → 카탈로그에서 테이블 정보 자동 삭제       │ │
│   │   • ALTER TABLE → 카탈로그 자동 갱신                     │ │
│   │   • CREATE INDEX → 인덱스 정보 자동 등록                 │ │
│   │                                                         │ │
│   │   💡 DDL 실행 시 DBMS가 자동으로 카탈로그 업데이트        │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   읽기 전용 vs 수정 가능:                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   읽기 전용 (대부분):                                    │ │
│   │   • 사용자는 SELECT로만 조회                            │ │
│   │   • 직접 INSERT/UPDATE/DELETE 금지                      │ │
│   │   • 무결성 보호                                         │ │
│   │                                                         │ │
│   │   수정 가능 (일부):                                      │ │
│   │   • 통계 정보 수동 갱신 (ANALYZE, UPDATE STATISTICS)    │ │
│   │   • 힌트 강제 설정                                      │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   캐싱:                                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • Data Dictionary Cache: 자주 접근하는 메타데이터 캐싱│ │
│   │   • 쿼리 최적화 시 캐시된 정보 사용                      │ │
│   │   • DDL 실행 시 캐시 무효화                              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **시스템 카탈로그**: DBMS가 메타데이터를 저장하는 시스템 테이블
> **메타데이터**: 데이터에 대한 데이터 (스키마, 권한, 통계 등)
> **자동 관리**: DDL 실행 시 DBMS가 자동으로 카탈로그 갱신

### 2. 빈출 키워드

- **Data Dictionary**: 시스템 카탈로그의 다른 명칭
- **INFORMATION_SCHEMA**: SQL 표준 카탈로그 뷰
- **통계 정보**: 옵티마이저가 실행 계획 수립에 활용
- **ANALYZE**: 통계 정보 갱신 명령

### 3. 예상 문제

**Q1. 시스템 카탈로그의 저장 정보와 역할을 설명하시오.**

**Q2. Oracle, MySQL의 시스템 카탈로그 구현 방식을 비교하시오.**

**Q3. 시스템 카탈로그의 통계 정보가 쿼리 최적화에 미치는 영향을 기술하시오.**
