# 내부 스키마 (Internal Schema)

## 정의

**내부 스키마(Internal Schema)**는 데이터베이스의 **물리적 저장 구조**를 정의한 것으로, 데이터가 실제로 디스크에 어떻게 저장되는지를 기술한다. 저장 레코드 형식, 인덱스 구조, 데이터 압축 방식 등을 포함한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           3단계 스키마에서의 내부 스키마 위치                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   외부 스키마 (External Schema)                          │ │
│   │   • 사용자 관점                                          │ │
│   │                        │                                │ │
│   │           외부/개념 매핑                                 │ │
│   │                        ▼                                │ │
│   │   개념 스키마 (Conceptual Schema)                        │ │
│   │   • 전체 논리적 구조                                     │ │
│   │                        │                                │ │
│   │           개념/내부 매핑                                 │ │
│   │                        ▼                                │ │
│   │   ┌─────────────────────────────────────────────────┐  │ │
│   │   │      내부 스키마 (Internal Schema)              │  │ │
│   │   │      ★ 물리적 저장 구조                        │  │ │
│   │   │      ★ 저장 레코드, 인덱스, 경로               │  │ │
│   │   │      ★ DBMS마다 다름                          │  │ │
│   │   └─────────────────────────────────────────────────┘  │ │
│   │                        │                                │ │
│   │                        ▼                                │ │
│   │   물리적 저장 장치 (Disk, SSD)                           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 내부 스키마 구성 요소

```
┌─────────────────────────────────────────────────────────────────┐
│           내부 스키마 구성 요소                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 저장 레코드 형식 (Stored Record Format)             │ │
│   │      ┌───────────────────────────────────────────────┐ │ │
│   │      │ • 필드 순서와 길이                             │ │ │
│   │      │ • 고정 길이 vs 가변 길이 레코드                │ │ │
│   │      │ • NULL 비트맵 위치                             │ │ │
│   │      │ • 레코드 헤더/트레일러                         │ │ │
│   │      └───────────────────────────────────────────────┘ │ │
│   │                                                         │ │
│   │   ② 인덱스 구조 (Index Structure)                       │ │
│   │      ┌───────────────────────────────────────────────┐ │ │
│   │      │ • 인덱스 유형 (B-Tree, Hash, Bitmap)          │ │ │
│   │      │ • 인덱스 키 구성                               │ │ │
│   │      │ • 클러스터드 vs 넌클러스터드                   │ │ │
│   │      │ • 인덱스 저장 위치                             │ │ │
│   │      └───────────────────────────────────────────────┘ │ │
│   │                                                         │ │
│   │   ③ 저장 경로 (Access Path)                             │ │
│   │      ┌───────────────────────────────────────────────┐ │ │
│   │      │ • 데이터 접근 방법                             │ │ │
│   │      │ • 조인 전략                                    │ │ │
│   │      │ • 스캔 방식 (Full Scan, Index Scan)           │ │ │
│   │      └───────────────────────────────────────────────┘ │ │
│   │                                                         │ │
│   │   ④ 물리적 배치 (Physical Placement)                    │ │
│   │      ┌───────────────────────────────────────────────┐ │ │
│   │      │ • 파일 구조 (Heap, Sorted, Hash)              │ │ │
│   │      │ • 블록/페이지 크기                             │ │ │
│   │      │ • 익스텐트/세그먼트 할당                       │ │ │

│   │      └───────────────────────────────────────────────┘ │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 저장 레코드 구조

```
┌─────────────────────────────────────────────────────────────────┐
│           저장 레코드 구조 예시                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   고정 길이 레코드:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌────────┬────────┬────────┬────────┬────────┐       │ │
│   │   │ Header │  학번   │  이름  │  학과  │  학년  │       │ │
│   │   │ 4 byte │ 10 byte│ 20 byte│ 10 byte│ 4 byte │       │ │
│   │   └────────┴────────┴────────┴────────┴────────┘       │ │
│   │                                                         │ │
│   │   총 레코드 크기 = 48 byte                              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   가변 길이 레코드 (슬로팅 페이지):                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌─────────────────────────────────────────────────┐  │ │
│   │   │ Header │ Entry │ Entry │ ... │   Free Space   │  │ │
│   │   │ (N,FR) │  1    │  2    │     │                │  │ │
│   │   └────────┴───────┴───────┴─────┴────────────────┘  │ │
│   │                                   ▲                   │ │
│   │                                   │                   │ │
│   │   ┌───────────────────────────────┴─────────────────┐│ │
│   │   │           Data Records (역순 저장)              ││ │
│   │   │   [레코드2] [레코드1]                            ││ │
│   │   └─────────────────────────────────────────────────┘│ │
│   │                                                         │ │
│   │   Header: 레코드 수(N), 여유 공간 시작(FR)              │ │
│   │   Entry: 레코드 위치 오프셋, 길이                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   NULL 비트맵 포함 레코드:                                      │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌────────┬─────────┬──────────────────────────────┐ │ │
│   │   │ Header │ NULL    │      실제 데이터              │ │ │
│   │   │        │ Bitmap  │      (NULL이면 생략)         │ │ │
│   │   └────────┴─────────┴──────────────────────────────┘ │ │
│   │                                                         │ │
│   │   NULL Bitmap: 각 컬럼별 NULL 여부 표시 (1 bit)         │ │
│   │   예: 5개 컬럼 → 1 byte (3 bit unused)                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 인덱스 물리적 구조

```
┌─────────────────────────────────────────────────────────────────┐
│           인덱스 물리적 구조                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   B+Tree 인덱스 구조:                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │                ┌─────────┐                              │ │
│   │                │  Root   │                              │ │
│   │                │  Node   │                              │ │
│   │                └────┬────┘                              │ │
│   │              ┌──────┼──────┐                            │ │
│   │              ▼      ▼      ▼                            │ │
│   │           ┌─────┐┌─────┐┌─────┐                        │ │
│   │           │Non  ││Non  ││Non  │  Branch Nodes          │ │
│   │           │Leaf ││Leaf ││Leaf │  (키+자식 포인터)       │ │
│   │           └──┬──┘└──┬──┘└──┬──┘                        │ │
│   │              ▼      ▼      ▼                            │ │
│   │   ┌─────┬─────┬─────┬─────┬─────┐                      │ │
│   │   │Leaf │◀──▶│Leaf │◀──▶│Leaf │  Leaf Nodes           │ │
│   │   │Node │    │Node │    │Node │  (키+ROWID)            │ │
│   │   └─────┴─────┴─────┴─────┴─────┘                      │ │
│   │                                                         │ │
│   │   리프 노드: 실제 데이터 위치(ROWID) 저장                │ │
│   │   리프 노드 간 링크: 범위 검색 최적화                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ROWID 구조 (Oracle 예시):                                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌──────────┬──────────┬──────────┐                   │ │
│   │   │ 데이터   │ 블록     │ 슬롯     │                   │ │
│   │   │ 파일 번호│ 번호     │ 번호     │                   │ │
│   │   │ 10 bit   │ 22 bit   │ 16 bit   │                   │ │
│   │   └──────────┴──────────┴──────────┘                   │ │
│   │                                                         │ │
│   │   총 48 bit = 데이터의 물리적 주소                      │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 물리적 데이터 독립성

```
┌─────────────────────────────────────────────────────────────────┐
│           물리적 데이터 독립성                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   정의: 내부 스키마가 변경되어도 개념 스키마가 영향을 받지 않음   │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   내부 스키마 변경 예시:                                 │ │
│   │                                                         │ │
│   │   변경 전:                                               │ │
│   │   • B-Tree 인덱스                                       │ │
│   │   • 4KB 페이지 크기                                      │ │
│   │   • Heap 파일 구조                                       │ │
│   │                                                         │ │
│   │   변경 후:                                               │ │
│   │   • B+Tree 인덱스 (인덱스 유형 변경)                    │ │

│   │   • 클러스터드 파일 구조 (파일 구조 변경)                │ │
│   │                                                         │ │
│   │   개념 스키마 영향:                                      │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 개념 스키마: 학생(학번, 이름, 학과)            │    │ │
│   │   │              ↓                                 │    │ │
│   │   │         변경 없음                              │    │ │
│   │   │         (개념/내부 매핑만 변경)                │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   장점:                                                  │ │
│   │   • 성능 튜닝 자유도 확보                                │ │
│   │   • 스토리지 최적화                                      │ │
│   │   • 하드웨어 교체 용이                                   │ │
│   │   • 응용 프로그램 영향 없음                              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **내부 스키마**: 데이터베이스의 물리적 저장 구조 정의
> **구성 요소**: 저장 레코드, 인덱스, 접근 경로, 물리적 배치
> **물리적 독립성**: 내부 스키마 변경 시 개념 스키마 영향 없음

### 2. 빈출 키워드

- **B+Tree**: 가장 널리 사용되는 인덱스 구조
- **ROWID**: 데이터의 물리적 주소
- **Slotted Page**: 가변 길이 레코드 저장 방식
- **물리적 독립성**: 내부 스키마와 개념 스키마 분리

### 3. 예상 문제

**Q1. 내부 스키마의 구성 요소를 설명하시오.**

**Q2. 물리적 데이터 독립성의 개념과 내부 스키마와의 관계를 기술하시오.**

**Q3. 가변 길이 레코드 저장을 위한 Slotted Page 구조를 설명하시오.**
