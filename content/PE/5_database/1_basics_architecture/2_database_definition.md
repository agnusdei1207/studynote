# 데이터베이스 (Database)의 정의

## 정의

**데이터베이스(DataBase)**란 조직의 업무를 수행하는 데 필요한 상호 관련된 데이터들의 **통합된(Integrated)**, **저장된(Stored)**, **운영(Operational)**, **공용(Shared)** 데이터의 집합이다.

```
┌─────────────────────────────────────────────────────────────────┐
│                     데이터베이스의 4대 특성                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────┐   ┌──────────────┐                           │
│  │  Integrated  │   │   Stored     │                           │
│  │   (통합성)    │   │   (저장성)    │                           │
│  │              │   │              │                           │
│  │ 중복 최소화   │   │ 컴퓨터 저장   │                           │
│  │ 일관성 유지   │   │ 직접 접근     │                           │
│  └──────────────┘   └──────────────┘                           │
│                                                                 │
│  ┌──────────────┐   ┌──────────────┐                           │
│  │ Operational  │   │   Shared     │                           │
│  │   (운영성)    │   │   (공용성)    │                           │
│  │              │   │              │                           │
│  │ 실시간 업무   │   │ 다중 사용자   │                           │
│  │ 지속적 갱신   │   │ 동시 접근     │                           │
│  └──────────────┘   └──────────────┘                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 통합된 데이터 (Integrated Data)

```
┌─────────────────────────────────────────────────────────────┐
│                    데이터 통합의 진화                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [Before: 파일 시스템]         [After: 데이터베이스]         │
│                                                             │
│   ┌─────┐ ┌─────┐ ┌─────┐          ┌─────────────┐         │
│   │파일1│ │파일2│ │파일3│    →     │  Database   │         │
│   │학생 │ │강좌 │ │성적 │          │  (통합)      │         │
│   └──┬──┘ └──┬──┘ └──┬──┘          └──────┬──────┘         │
│      │       │       │                    │                │
│   중복데이터  │    중복데이터          ┌───┴───┐            │
│      └───────┴───────┘               │ 통합   │            │
│          데이터 불일치               │ 일관성 │            │
│                                      └───────┘            │
│                                                             │
│  ✓ 데이터 중복 최소화 → 저장 공간 효율화                    │
│  ✓ 데이터 일관성 유지 → 갱신 이상 방지                      │
│  ✓ 데이터 무결성 보장 → 정확성 확보                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**통합의 이점**
- 중복 제어: 동일 데이터의 중복 저장 방지
- 표준화: 데이터 형식, 코딩 체계 통일
- 일관성: 모든 응용에 동일 데이터 제공

### 2. 저장된 데이터 (Stored Data)

```
┌─────────────────────────────────────────────────────────────┐
│                   저장 매체의 계층 구조                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   속도         │  CPU ←── Cache ←── Main Memory            │
│     ↑          │          ↑              ↑                  │
│     │          │       ns 단위        ns~μs                 │
│     │          │                                          │
│     │          │   ┌──────────────────────────────┐        │
│     │          │   │      Secondary Storage       │        │
│     │          │   │    (DB 저장 영역)            │        │
│     │          │   │                              │        │
│     │          │   │  ┌────────┐  ┌────────────┐ │        │
│     │          │   │  │  HDD   │  │    SSD     │ │        │
│     │          │   │  │ms 단위 │  │ μs~ms 단위 │ │        │
│     │          │   │  └────────┘  └────────────┘ │        │
│     │          │   │                              │        │
│     │          │   │  ┌────────────────────────┐ │        │
│     │          │   │  │   NVMe / SAN / NAS     │ │        │
│     │          │   │  │   고속 대용량 스토리지   │ │        │
│     │          │   │  └────────────────────────┘ │        │
│     │          │   └──────────────────────────────┘        │
│     ↓          │                                          │
│   용량         │   GB → TB → PB → EB                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**직접 접근 (Direct Access)**
- 순차 접근 vs 직접 접근
- 인덱스를 통한 빠른 검색
- 블록/페이지 단위 I/O

### 3. 운영 데이터 (Operational Data)

```
┌─────────────────────────────────────────────────────────────┐
│                 운영 데이터 vs 참조 데이터                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  운영 데이터 (Operational/Transaction Data)                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • 실시간 트랜잭션 처리 (OLTP)                        │   │
│  │ • 지속적으로 생성, 수정, 삭제                        │   │
│  │ • 예: 주문, 재고, 계좌, 예약 데이터                  │   │
│  │ • 높은 갱신 빈도                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  참조 데이터 (Reference/Master Data)                        │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • 상대적으로 정적인 데이터                           │   │
│  │ • 마스터 데이터 관리 (MDM)                           │   │
│  │ • 예: 고객 정보, 상품 정보, 코드표                   │   │
│  │ • 낮은 갱신 빈도, 높은 참조 빈도                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  분석 데이터 (Analytical/Data Warehouse)                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • 읽기 전용, 대량 조회 (OLAP)                        │   │
│  │ • 시계열 데이터, 집계 데이터                         │   │
│  │ • 예: 매출 분석, 고객 행동 패턴                      │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```
│                            │                                │
│                    ┌───────┴───────┐                        │

```

## 상세 설명

### 데이터베이스 vs 파일 시스템

```
┌─────────────────────────────────────────────────────────────┐
│                파일 시스템 vs 데이터베이스                   │
├───────────────┬─────────────────┬───────────────────────────┤
│     구분      │   파일 시스템    │      데이터베이스          │
├───────────────┼─────────────────┼───────────────────────────┤
│ 데이터 중복   │     높음        │        최소화              │
│ 데이터 일관성 │    보장 어려움   │        DBMS가 보장         │
│ 데이터 독립성 │       없음      │        3단계 독립성        │
│ 동시 접근     │      제한적      │        완벽 지원           │
│ 보안          │     OS 수준      │     세밀한 권한 제어       │
│ 무결성        │   응용프로그램   │     DBMS가 강제            │
│ 표준화        │       없음      │     데이터 사전 관리        │
│ 백업/복구     │   수동/어려움    │     자동화된 도구          │
│ 확장성        │      제한적      │        뛰어남              │
└───────────────┴─────────────────┴───────────────────────────┘
```

### 데이터베이스의 추가 특성

1. **실시간 접근성 (Real-time Accessibility)**
   - 질의에 대한 실시간 응답
   - 온라인 트랜잭션 처리 (OLTP)

2. **계속적인 변화 (Continuous Evolution)**
   - CRUD 연산으로 지속적 갱신
   - 현재의 정확한 데이터 유지

3. **내용 기반 참조 (Content-based Reference)**
   - 위치가 아닌 내용으로 데이터 식별
   - SQL: WHERE 조건으로 검색

## 기술사 시험 포인트

### 1. 핵심 정의 (반드시 암기)

> "데이터베이스는 조직의 업무를 수행하는 데 필요한 **통합된, 저장된, 운영, 공용** 데이터의 집합이다."

### 2. 빈출 키워드

- **4대 특성**: 통합성, 저장성, 운영성, 공용성
- **파일 시스템과의 차이점**: 중복, 일관성, 독립성, 무결성
- **실시간 접근성**: 질의에 대한 즉각적 응답
- **내용 기반 참조**: 위치가 아닌 내용으로 검색

### 3. 예상 문제

**Q1. 데이터베이스의 4대 특성에 대해 설명하시오.**

**Q2. 파일 시스템 대비 데이터베이스의 장점 5가지를 기술하시오.**

**Q3. 데이터베이스의 "내용 기반 참조"에 대해 설명하고 그 의미를 기술하시오.**

### 4. 실무 연계

- RDBMS vs NoSQL 선택 기준
- 데이터베이스 이중화 및 고가용성 설계
- 데이터 통합 및 마이그레이션 전략
