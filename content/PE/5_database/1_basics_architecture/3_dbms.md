# 데이터베이스 관리 시스템 (DBMS)

## 정의

**DBMS(DataBase Management System)**는 사용자와 데이터베이스 사이에서 **데이터를 정의, 생성, 조작, 제어**하는 기능을 제공하는 소프트웨어 시스템이다. DBMS는 데이터의 **독립성, 보안, 무결성, 동시성**을 보장한다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    DBMS의 위치와 역할                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│    [사용자/응용프로그램]                                        │
│           │                                                     │
│           │ SQL / API                                           │
│           ▼                                                     │
│    ┌─────────────────────────────────────────────┐             │
│    │                   DBMS                       │             │
│    │  ┌─────────────────────────────────────────┐│             │
│    │  │           Query Processor               ││             │
│    │  │  Parser → Optimizer → Executor          ││             │
│    │  └─────────────────────────────────────────┘│             │
│    │  ┌─────────────────────────────────────────┐│             │
│    │  │         Storage Engine                  ││             │
│    │  │  Buffer │ Lock │ Log │ Recovery         ││             │
│    │  └─────────────────────────────────────────┘│             │
│    └─────────────────────────────────────────────┘             │
│                         │                                       │

│    └─────────────────────────────────────────────┘             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 데이터 독립성 제공

```
┌─────────────────────────────────────────────────────────────────┐
│                  3단계 데이터 독립성                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│    [외부 단계 / External Level]                                 │
│    개별 사용자 뷰 (View)                                        │
│    ┌─────────────────────────────────────────────┐             │
│    │  View 1: 고객정보뷰   View 2: 주문뷰        │             │
│    └────────────────────┬────────────────────────┘             │
│                         │                                      │
│    ◀── 외부/개념 매핑 ──┘  (논리적 독립성)                      │
│                         │                                      │
│    [개념 단계 / Conceptual Level]                              │
│    전체 데이터베이스 구조                                       │
│    ┌─────────────────────────────────────────────┐             │
│    │         통합 스키마 (전체 구조)              │             │
│    └────────────────────┬────────────────────────┘             │
│                         │                                      │
│    ◀── 개념/내부 매핑 ──┘  (물리적 독립성)                      │
│                         │                                      │
│    [내부 단계 / Internal Level]                                │
│    물리적 저장 구조                                             │
│    ┌─────────────────────────────────────────────┐             │
│    │  데이터 파일, 인덱스, 저장 레코드            │             │
│    └─────────────────────────────────────────────┘             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**독립성의 이점**
- 물리적 독립성: 저장 구조 변경이 응용프로그램에 영향 없음
- 논리적 독립성: 스키마 변경이 외부 뷰에 영향 없음

### 트랜잭션 관리

```
┌─────────────────────────────────────────────────────────────────┐
│                    트랜잭션 ACID 특성                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ A: Atomicity (원자성)                                     │ │
│  │    All or Nothing - 전부 성공 또는 전부 실패              │ │
│  │    구현: Undo Log, WAL (Write-Ahead Logging)              │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ C: Consistency (일관성)                                   │ │
│  │    트랜잭션 전후 데이터 무결성 보장                        │ │
│  │    구현: 제약조건, 트리거                                 │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ I: Isolation (격리성)                                     │ │
│  │    동시 실행 트랜잭션 간 간섭 방지                         │ │
│  │    구현: Lock, MVCC (Multi-Version Concurrency Control)   │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ D: Durability (지속성)                                    │ │
│  │    완료된 트랜잭션 결과는 영구 보존                        │ │
│  │    구현: Redo Log, Checkpoint                             │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### DBMS 선정 기준

1. **기능적 요구사항**: ACID, 트랜잭션, 복잡한 쿼리
2. **성능 요구사항**: TPS, 응답시간, 동시 사용자
3. **확장성**: 수평/수직 확장, 샤딩
4. **고가용성**: HA 구성, 장애 조치
5. **비용**: 라이선스, 운영 비용

## 기술사 시험 포인트

### 1. 핵심 정의

> "DBMS는 사용자와 데이터베이스 사이에서 데이터를 **정의, 생성, 조작, 제어**하는 기능을 제공하며, **데이터 독립성**을 보장하는 소프트웨어 시스템이다."

### 2. 빈출 키워드

- **4대 기능**: 정의(DDL), 조작(DML), 제어(DCL), 관리
- **데이터 독립성**: 논리적 독립성, 물리적 독립성
- **ACID**: 원자성, 일관성, 격리성, 지속성
- **질의 처리**: 파싱 → 분석 → 최적화 → 실행
