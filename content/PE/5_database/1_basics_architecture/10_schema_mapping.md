# 스키마 매핑 (Schema Mapping)

## 정의

**스키마 매핑(Schema Mapping)**은 ANSI/SPARC 3단계 스키마 아키텍처에서 각 스키마 계층 간의 **대응 관계를 정의**한 것으로, 외부/개념 매핑과 개념/내부 매핑으로 구분된다. 데이터 독립성을 구현하는 핵심 메커니즘이다.

```
┌─────────────────────────────────────────────────────────────────┐
│           스키마 매핑 개요                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   외부 스키마 (External Schema)                          │ │
│   │        │                                                │ │
│   │        │  ┌─────────────────────────┐                   │ │
│   │        │  │  외부/개념 매핑         │                   │ │
│   │        │  │  (External/Conceptual)  │                   │ │
│   │        │  │  • 논리적 독립성 보장    │                   │ │
│   │        │  └─────────────────────────┘                   │ │
│   │        ▼                                                │ │
│   │   개념 스키마 (Conceptual Schema)                        │ │
│   │        │                                                │ │
│   │        │  ┌─────────────────────────┐                   │ │
│   │        │  │  개념/내부 매핑         │                   │ │
│   │        │  │  (Conceptual/Internal)  │                   │ │
│   │        │  │  • 물리적 독립성 보장    │                   │ │
│   │        │  └─────────────────────────┘                   │ │
│   │        ▼                                                │ │
│   │   내부 스키마 (Internal Schema)                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 외부/개념 매핑

```
┌─────────────────────────────────────────────────────────────────┐
│           외부/개념 매핑 (External/Conceptual Mapping)           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   역할: 외부 스키마(사용자 뷰)와 개념 스키마(전체 논리 구조) 간   │
│         대응 관계 정의 → 논리적 데이터 독립성 제공               │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   외부 스키마 (뷰)                                       │ │
│   │   ┌─────────────────────────────────────────┐          │ │
│   │   │ 학생_간단뷰 (학번, 이름)                │          │ │
│   │   └───────────────────┬─────────────────────┘          │ │
│   │                       │                                │ │
│   │         외부/개념 매핑                                  │ │
│   │         (학번→학생.학번, 이름→학생.이름)               │ │
│   │                       ▼                                │ │
│   │   개념 스키마                                           │ │
│   │   ┌─────────────────────────────────────────┐          │ │
│   │   │ 학생 (학번, 이름, 학과, 학년, 주소)     │          │ │
│   │   └─────────────────────────────────────────┘          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   매핑 정보:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   • 뷰와 테이블 간 컬럼 대응                             │ │
│   │   • 선택 조건 (WHERE 절)                                │ │
│   │   • 조인 관계                                           │ │
│   │   • 계산된 컬럼 정의                                     │ │
│   │   • 데이터 타입 변환                                    │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   SQL 뷰 정의 예시:                                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   CREATE VIEW 학생_간단뷰 AS                            │ │
│   │   SELECT 학번, 이름 FROM 학생;                          │ │
│   │                       ↑                                 │ │
│   │                 외부/개념 매핑                           │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 개념/내부 매핑

```
┌─────────────────────────────────────────────────────────────────┐
│           개념/내부 매핑 (Conceptual/Internal Mapping)           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   역할: 개념 스키마(논리 구조)와 내부 스키마(물리 구조) 간        │
│         대응 관계 정의 → 물리적 데이터 독립성 제공               │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   개념 스키마                                           │ │
│   │   ┌─────────────────────────────────────────┐          │ │
│   │   │ 학생 (학번, 이름, 학과, 학년)           │          │ │
│   │   │ PK: 학번                                │          │ │
│   │   └───────────────────┬─────────────────────┘          │ │
│   │                       │                                │ │
│   │         개념/내부 매핑                                  │ │
│   │         • 테이블→파일 매핑                              │ │
│   │         • 컬럼→필드 매핑                                │ │
│   │         • PK→인덱스 매핑                               │ │
│   │                       ▼                                │ │
│   │   내부 스키마                                           │ │
│   │   ┌─────────────────────────────────────────┐          │ │
│   │   │ 파일: /data/student.dat                 │          │ │
│   │   │ 레코드: 48 byte 고정                    │          │ │
│   │   │ 인덱스: B+Tree on 학번                  │          │ │
│   │   └─────────────────────────────────────────┘          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   매핑 정보:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   • 논리 테이블 ↔ 물리 파일                             │ │
│   │   • 컬럼 ↔ 저장 필드                                    │ │
│   │   • 데이터 타입 ↔ 내부 표현                             │ │
│   │   • 제약조건 ↔ 구현 방식                                │ │
│   │   • 인덱스 ↔ 물리적 구조                                │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 매핑을 통한 데이터 독립성

```
┌─────────────────────────────────────────────────────────────────┐
│           매핑과 데이터 독립성                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   논리적 데이터 독립성 시나리오:                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   개념 스키마 변경: 학생 테이블에 '이메일' 컬럼 추가      │ │
│   │                                                         │ │
│   │   변경 전:                                               │ │
│   │   외부: 학생_뷰(학번, 이름)                              │ │
│   │          ↓ 매핑                                          │ │
│   │   개념: 학생(학번, 이름, 학과)                           │ │
│   │                                                         │ │
│   │   변경 후:                                               │ │
│   │   외부: 학생_뷰(학번, 이름) ← 변경 없음                  │ │
│   │          ↓ 매핑 (변경됨: 새 컬럼 무시)                   │ │
│   │   개념: 학생(학번, 이름, 학과, 이메일) ← 컬럼 추가       │ │
│   │                                                         │ │
│   │   💡 외부/개념 매핑만 수정하면 외부 스키마 영향 없음      │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   물리적 데이터 독립성 시나리오:                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   내부 스키마 변경: 인덱스 유형 변경 (B-Tree → B+Tree)   │ │
│   │                                                         │ │
│   │   변경 전:                                               │ │
│   │   개념: 학생(학번, 이름, 학과)                           │ │
│   │          ↓ 매핑                                          │ │
│   │   내부: B-Tree 인덱스                                    │ │
│   │                                                         │ │
│   │   변경 후:                                               │ │
│   │   개념: 학생(학번, 이름, 학과) ← 변경 없음               │ │
│   │          ↓ 매핑 (변경됨: 새 인덱스 구조 반영)            │ │
│   │   내부: B+Tree 인덱스 ← 인덱스 변경                      │ │
│   │                                                         │ │
│   │   💡 개념/내부 매핑만 수정하면 개념 스키마 영향 없음      │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 매핑 구현 방식

```
┌─────────────────────────────────────────────────────────────────┐
│           DBMS에서의 매핑 구현                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   시스템 카탈로그에 저장되는 매핑 정보:                          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   외부/개념 매핑 (뷰 정의):                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ VIEW_DEFINITIONS                               │    │ │
│   │   │ ─────────────────────────────────────────────  │    │ │
│   │   │ view_name    │ 학생_간단뷰                     │    │ │
│   │   │ definition    │ SELECT 학번, 이름 FROM 학생    │    │ │
│   │   │ check_option  │ NONE                           │    │ │
│   │   │ is_updatable  │ YES                            │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   개념/내부 매핑 (테이블 메타데이터):                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ TABLES                                          │    │ │
│   │   │ ─────────────────────────────────────────────  │    │ │
│   │   │ table_name   │ 학생                            │    │ │
│   │   │ tablespace   │ USERS                           │    │ │
│   │   │ num_rows     │ 10000                           │    │ │
│   │   │                                              │    │ │
│   │   │ COLUMNS                                     │    │ │
│   │   │ ─────────────────────────────────────────── │    │ │
│   │   │ column_name  │ data_type  │ position │ ...   │    │ │
│   │   │ 학번         │ CHAR(10)   │ 1        │       │    │ │
│   │   │ 이름         │ VARCHAR(20)│ 2        │       │    │ │
│   │   │ ...          │ ...        │ ...      │       │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   매핑 처리 과정:                                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   SQL: SELECT * FROM 학생_간단뷰                        │ │
│   │                      │                                  │ │
│   │                      ▼                                  │ │
│   │   ① 외부/개념 매핑 조회 (뷰 정의)                        │ │
│   │      → SELECT 학번, 이름 FROM 학생                      │ │
│   │                      │                                  │ │
│   │                      ▼                                  │ │
│   │   ② 개념 스키마 확인                                     │ │
│   │      → 학생 테이블 존재, 컬럼 유효                       │ │
│   │                      │                                  │ │
│   │                      ▼                                  │ │
│   │   ③ 개념/내부 매핑 조회                                  │ │
│   │      → 학생 테이블 위치, 인덱스 정보                     │ │
│   │                      │                                  │ │
│   │                      ▼                                  │ │
│   │   ④ 물리적 데이터 접근                                   │ │
│   │      → 디스크에서 데이터 읽기                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **외부/개념 매핑**: 사용자 뷰와 전체 논리 구조 간 대응 (논리적 독립성)
> **개념/내부 매핑**: 논리 구조와 물리 구조 간 대응 (물리적 독립성)
> **매핑 저장소**: 시스템 카탈로그/데이터 사전

### 2. 빈출 키워드

- **논리적 독립성**: 외부/개념 매핑으로 보장
- **물리적 독립성**: 개념/내부 매핑으로 보장
- **시스템 카탈로그**: 매핑 정보 저장
- **뷰 정의**: 외부/개념 매핑의 SQL 구현

### 3. 예상 문제

**Q1. 외부/개념 매핑과 개념/내부 매핑의 차이점을 설명하시오.**

**Q2. 스키마 매핑이 데이터 독립성에 미치는 영향을 기술하시오.**

**Q3. DBMS에서 스키마 매핑 정보가 저장되는 위치와 관리 방법을 설명하시오.**
