# SNMP (Simple Network Management Protocol)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SNMP는 **"네트워크 **장비 **모니터링 **및 **관리 **프로토콜"**으로, **Manager**(관리 **서버)와 **Agent**(장비 **에이전트)가 **UDP 161**(Query), **UDP 162**(Trap) **포트**로 **통신**한다.
> 2. **구조**: **MIB**(Management Information Base)에 **정의**된 **OID**(Object Identifier)로 **장비 **정보**를 **조회**하며 **GET**(조회), **SET**(설정), **TRAP**(이벤트 **알림) **명령어**를 **사용**한다.
> 3. **버전**: **SNMPv1/v2c**(Community **문자열 **인증), **SNMPv3**(USM **보안**, **VACM **액세스 **제어)로 **구분**되며 **v3**는 **암호화**와 **무결성**을 **보장**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
SNMP는 **"네트워크 관리 표준 프로토콜"**이다.

**구성 요소**:
- **Manager**: NMS (Network Management System)
- **Agent**: 장비 내 에이전트
- **MIB**: 관리 정보 구조
- **OID**: 객체 식별자

### 💡 비유
SNMP는 ****건물 **관리 **시스템 ****과 같다.
- **관리자**: 건물 관리자
- **에이전트**: 각 층 센서
- **MIB**: 센서 매뉴얼
- **Trap**: 화재 경보

## Ⅲ. 융합 비교 및 다각도 분석

### SNMP 명령어

| 명령어 | 설명 | 버전 |
|--------|------|------|
| **GET** | 단일 객체 조회 | v1/v2c/v3 |
| **GETNEXT** | 다음 객체 조회 | v1/v2c/v3 |
| **GETBULK** | 다중 객체 조회 | v2c/v3 |
| **SET** | 객체 값 설정 | v1/v2c/v3 |
| **TRAP** | 이벤트 알림 (비확인) | v1/v2c/v3 |
| **INFORM** | 이벤트 알림 (확인) | v2c/v3 |

### SNMP 버전 비교

| 구분 | v1 | v2c | v3 |
|------|-----|------|-----|
| **인증** | Community | Community | USM (MD5/SHA) |
| **암호화** | X | X | O (DES/AES) |
| **GETBULK** | X | O | O |
| **64bit 카운터** | X | O | O |

### Trap vs Inform

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Trap vs Inform                                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    SNMP Trap (Unconfirmed):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Agent                                       Manager                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  [Link Down Event]                                                                     │  │  │
    │  │         │                                                                              │  │  │
    │  │         ▼                                                                              │  │  │
    │  │  ────── TRAP (UDP 162) ─────────────────→                                            │  │  │
    │  │         │                                                                              │  │  │
    │  │         │ (Fire and forget: No acknowledgment)                                        │  │  │
    │  │                                                                                       │  │  │
    │  │  → Agent doesn't know if Manager received it                                         │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    SNMP Inform (Confirmed):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Agent                                       Manager                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  [Link Down Event]                                                                     │  │  │
    │  │         │                                                                              │  │  │
    │  │         ▼                                                                              │  │  │
    │  │  ────── INFORM (UDP 162) ────────────────→                                            │  │  │
    │  │         │                                 │                                            │  │  │
    │  │         │                                 ▼                                            │  │  │
    │  │         │                          ┌──────────────┐                                  │  │  │
    │  │         │                          │ Process &    │                                  │  │  │
    │  │         │                          │ Acknowledge  │                                  │  │  │
    │  │         │                          └──────────────┘                                  │  │  │
    │  │         │                                 │                                            │  │  │
    │  │         ←───────── RESPONSE ──────────────┘                                            │  │  │
    │  │         │                                                                              │  │  │
    │  │         ▼ (Confirmed received)                                                         │  │  │
    │  │                                                                                       │  │  │
    │  │  → Agent knows Manager received the notification                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### SNMP 도구

```bash
# snmpget (단일 객체 조회)
snmpget -v2c -c public 192.168.1.1 1.3.6.1.2.1.1.1.0
# → SNMPv2-MIB::sysDescr.0 = STRING: Cisco IOS Software...

# snmpwalk (하위 트리 전체 조회)
snmpwalk -v2c -c public 192.168.1.1 1.3.6.1.2.1.2.2.1
# → ifTable 전체

# snmpgetnext (다음 객체)
snmpgetnext -v2c -c public 192.168.1.1 1.3.6.1.2.1.1.1

# snmpset (객체 설정)
snmpset -v2c -c private 192.168.1.1 \
    1.3.6.1.2.1.1.5.0 s "NewSysName"

# snmptrap (Trap 전송)
snmptrap -v2c -c public 192.168.1.100 '' \
    1.3.6.1.4.1.9.9.43.2.0.1 \
    1.3.6.1.2.1.2.2.1.1.3 i 3 \
    1.3.6.1.2.1.2.2.1.7.3 i 2
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 네트워크 모니터링 구성
**상황**: 스위치 트래픽 모니터링
**판단**: SNMP + MRTG/Zabbix

```bash
# 1. 스위치 SNMP 활성화
# Cisco IOS
snmp-server community public RO
snmp-server community private RW
snmp-server host 192.168.1.100 public

# 2. Zabbix 템플릿
# Template SNMP Generic
# - ifInOctets, ifOutOctets → 트래픽 그래프
# - ifOperStatus → 인터페이스 UP/DOWN 알림
# - ifErrors → 에러율 모니터링

# 3. MRTG 설정
Target[switch1]: #Public@192.168.1.1:1.3.6.1.2.1.2.2.1.10.1&1.3.6.1.2.1.2.2.1.16.1:public@192.168.1.1
# → ifInOctets + ifOutOctets = Total Traffic
```

## Ⅴ. 기대효과 및 결론

### SNMP 기대 효과

| 효과 | 없음 | SNMP |
|------|------|------|
| **모니터링** | 수동 | 자동 |
| **알림** | X | Trap/Inform |
| **중앙 관리** | 어려움 | 쉬움 |

### 모범 사례

1. **보안**: v3 사용 (암호화)
2. **Community**: 기본값 변경 (public/private)
3. **필터링**: ACL로 접근 제한
4. **모니터링**: Zabbix, PRTG, SolarWinds

### 미래 전망

1. **SNMP over DTLS**: TLS 암호화
2. **NETCONF/YANG**: 네트워크 자동화
3. **Streaming**: Telemetry 대체

### ※ 참고 표준/가이드
- **RFC 1157**: SNMPv1
- **RFC 1901**: SNMPv2c
- **RFC 3414**: SNMPv3 (USM)
