# VPN (Virtual Private Network)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: VPN은 **"공용 **네트워크(**인터넷)**를 **통해 **안전한 **가상 **전용 **망**을 **구축**하는 **기술"**으로, **Tunneling**(캡슐화)과 **Encryption**(암호화)로 **데이터 **보안**을 **보장**한다.
> 2. **종류**: **Site-to-Site VPN**(사무실 **간 **연결), **Remote Access VPN**(재택 **근무), **SSL VPN**(HTTPS **기반), **IPSec VPN**(IP **계층 **암호화)로 **구분**된다.
> 3. **프로토콜**: **IPSec**(ESP+**AH**), **SSL/TLS**(HTTPS **VPN), **WireGuard**(현대적 **간단 **프로토콜), **L2TP**, **PPTP**(구형, **취약점 **있음)가 **사용**된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
VPN은 **"가상 전용망 서비스"**이다.

**VPN 특징**:
- **Confidentiality**: 암호화
- **Integrity**: 데이터 무결성
- **Authentication**: 사용자 인증
- **Tunneling**: 패킷 캡슐화

### 💡 비유
VPN은 ****자동차 **용 **튜널 ****과 같다.
- **공용 도로**: 인터넷
- **터널**: VPN 터널
- **차량**: 암호화된 데이터

## Ⅱ. 아키텍처 및 핵심 원리

### VPN 터널링

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         VPN Tunneling                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Original Packet:                                                                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  IP Header    │  TCP Header  │  Data                                               │  │  │
    │  │  (Src: A)     │  (HTTP)      │  (Unencrypted)                                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
            │
            ▼ Encrypt & Encapsulate
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  VPN Packet:                                                                            │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Outer IP Header  │  ESP Header  │  Encrypted Original Packet                        │  │  │
    │  │  (Src: Gateway A) │  (SPI, Seq)  │  (IP+TCP+Data encrypted)                         │  │  │
    │  │  (Dst: Gateway B) │              │                                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
            │
            ▼ Through Internet
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  At Gateway B:                                                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Decrypt ESP payload                                                              │  │  │
    │  │  2. Verify authentication                                                             │  │  │
    │  │  3. Decapsulate (remove outer IP/ESP)                                                 │  │  │
    │  │  4. Forward original packet to destination                                           │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐

    │  │                                                                      │              │  │  │
    │  │                                                                      │              │  │  │
    │  │  192.168.1.10 → 1.1.1.1 ────[IPSec Tunnel]───→ 2.2.2.2 → 192.168.2.10            │  │  │

    │  │  → Traffic between sites encrypted                                                      │  │  │
    │  │  → Sites appear as single private network                                              │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

## Ⅲ. 융합 비교 및 다각도 분석

### VPN 프로토콜 비교

| 프로토콜 | 계층 | 암호화 | NAT Traversal | 속도 |
|----------|------|--------|---------------|------|
| **IPSec** | L3 | 강함 | 일부 가능 | 빠름 |
| **SSL/TLS** | L4~L7 | 강함 | 우수 | 중간 |
| **WireGuard** | L3 | 강함 | 우수 | 매우 빠름 |
| **L2TP** | L2 | 없음 | IPsec 필요 | 느림 |
| **PPTP** | L2 | 약함 | 우수 | 빠름 |

## Ⅳ. 실무 적용 및 기술사적 판단

### OpenVPN 설정

```bash
# server.conf
port 1194
proto udp
dev tun
ca ca.crt
cert server.crt
key server.key
dh dh.pem
server 10.8.0.0 255.255.255.0
push "redirect-gateway def1 bypass-dhcp"
push "dhcp-option DNS 192.168.1.1"
keepalive 10 120
cipher AES-256-CBC
auth SHA256
user nobody
group nogroup

# client.ovpn
client
dev tun
proto udp
remote vpn.example.com 1194
resolv-retry infinite
nobind
user nobody
group nogroup
persist-key
persist-tun
ca ca.crt
cert client.crt
key client.key
remote-cert-tls server
cipher AES-256-CBC
auth SHA256
verb 3
```

## Ⅴ. 기대효과 및 결론

### VPN 기대 효과

| 효과 | 없음 | VPN |
|------|------|-----|
| **도청** | 가능 | 방지 |
| **위조** | 가능 | 방지 |
| **원격 접속** | 불안 | 안전 |

### 모범 사례

1. **암호화**: AES-256-GCM
2. **인증**: Certificate + MFA
3. **DNS**: VPN 내부 DNS
4. **Kill Switch**: 연결 끊기면 차단

### 미래 전망

1. **WireGuard**: 기본 VPN
2. **Zero Trust**: 기존 VPN 대체
3. **SASE**: SD-WAN + Security

### ※ 참고 표준/가이드
- **RFC 4301**: IPSec
- **WireGuard**: wireguard.com
- **NIST**: SP 800-77
