# FTP (File Transfer Protocol)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: FTP는 **"파일 **전송 **전용 **프로토콜"**으로, **TCP 20/21포트**를 **사용**하고 **Control Connection**(명령어)과 **Data Connection**(데이터)을 **분리**하며 **ASCII**(Control)와 **Binary**(Data) **모드**를 **지원**한다.
> 2. **모드**: **Active Mode**(서버가 데이터 연결)와 **Passive Mode**(클라이언트가 데이터 연결)로 **작동**하며 **Authentication**(사용자 인증)과 **Anonymous**(익명) 접속을 **지원**한다.
> 3. **보안 문제**: **암호화되지 **않은 **전송**으로 **ID/Password**가 **노출**되며 **FTP over SSL/TLS**(FTPS)나 **SFTP**(SSH File Transfer Protocol)로 **보안**을 **강화**해야 한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
FTP는 **"파일 전송 프로토콜"**이다.

**FTP 특징**:
- **Dual Connection**: 제어 + 데이터
- **TCP**: 신뢰성 보장
- **Authentication**: 사용자 인증
- **Binary/ASCII**: 모드 지원

### 💡 비유
FTP는 ****택배 **배송****와 같다.
- **주문지**: 제어 연결
- **화물**: 데이터 연결
- **운송**: 파일 전송

## Ⅱ. 아키텍처 및 핵심 원리

### FTP 세션

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         FTP Session                                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  FTP Client                              FTP Server (Port 21)                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Control Connection (TCP Port 21)                                                   │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  USER username                                                                        │  │  │  │
    │  │  │  │  → 331 User name okay, need password                                         │  │  │  │
    │  │  │  │  PASS password                                                                        │  │  │  │
    │  │  │  │  → 230 User logged in                                                           │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  Data Connection (TCP Port 20)                                                      │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  PORT 2027  → (Active Mode: Server connects back to client)                        │  │  │  │
    │  │  │  │  PASV  ──────→ (Passive Mode: Client connects to server)                       │  │  │  │
    │  │  │  │  RETR 227 (Entering Passive Mode)                                                │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  File Transfer Commands:                                                            │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  LIST           → 226 Directory listing                                          │  │  │  │
    │  │  │  │  RETR           → 150 Opening Binary mode                                     │  │  │  │
    │  │  │  │  STOR filename  → 150 File status okay                                     │  │  │  │
    │  │  │  │  QUIT           → 221 Goodbye                                             │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Active vs Passive

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Active vs Passive Mode                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Active Mode (Client behind Firewall - Problem):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Client                                                      Server                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Control: Client ──── Port 21 ────→ Server                                        │  │  │
    │  │  Data:    Client ←──── Port 20 ←───── Server (Server initiates)                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Passive Mode (Server behind Firewall - Solution):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Client                                                      Server                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Control: Client ──── Port 21 ────→ Server                                        │  │  │
    │  │  Data:    Client ──── Port 20 ────→ Server (Client initiates)                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### FTP vs SFTP

| 구분 | FTP | SFTP |
|------|-----|-------|
| **포트** | 20/21 | 22 |
| **암호화** | X | O (SSH) |
| **방화벽** | X | O |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: ftp 클라이언트
**상황**: 파일 전송
**판단**:

```bash
# FTP 접속
ftp ftp.example.com
Name: anonymous
Password: (email)

# Passive mode
passive

# 파일 전송
get filename.txt
put upload.txt

# 종료
bye
```
