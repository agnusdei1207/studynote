# SDN (Software Defined Networking)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SDN은 **\"제어 **평면**(Control Plane)**과 **데이터 **평면**(Data Plane)**을 **분리**하여 **중앙 **컨트롤러**에서 **네트워크**를 **프로그래밍**하는 **기술\"**로, **OpenFlow**가 **표준 **프로토콜**이며 **NFV**(Network Functions Virtualization)**와 **결합**하여 **유연**한 **네트워크**를 **구현**한다.
> 2. **구조**: **Controller**(NOX, **ONOS**, **ODL)**가 **Flow Table**을 **관리**하고 **Switch**(OpenFlow **Switch)**가 **Packet**을 **Forwarding**하며 **Southbound API**(OpenFlow, **Netconf**)와 **Northbound API**(REST, **RESTCONF**)로 **통신**한다.
> 3. **작동**: **Controller** → **Flow Rule **설치** → **Switch** → **Packet **매칭** → **Action**(Forward, **Drop**, **Modify)**으로 **이루어지고 **Network **Virtualization**, **Load **Balancing**, **Security **Policy**를 **동적**으로 **변경**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
SDN은 **"네트워크 프로그래밍 가능성"**이다.

**전통 네트워크 vs SDN**:
| 구분 | 전통 | SDN |
|------|------|-----|
| **제어** | 분산 (각 스위치) | 중앙 (컨트롤러) |
| **프로토콜** | OSPF, BGP | OpenFlow |
| **설정** | CLI per box | Central API |
| **유연성** | 낮음 | 높음 |

### 💡 비유
SDN은 ****교통 **관제 **센터 ****와 같다.
- **제어 센터**: Controller
- **신호등**: Switch
- **차량**: Packet
- **경로**: Flow Rule

    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Packet Processing Pipeline:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Packet Ingress → Table 0 Match → Table 1 Match → ... → Table N Match → Egress           │  │
    │                                                                                         │  │
    │  If Match Found:                                                                        │  │
    │  • Execute actions                                                                      │  │

    │  • Forward to next table or output port                                                 │  │
    │                                                                                         │  │
    │  If No Match:                                                                           │  │
    │  • Send to Controller (Packet-In) for flow rule installation                             │  │

## Ⅴ. 기대효과 및 결론

### SDN 기대 효과

| 효과 | 전통 | SDN |
|------|------|-----|
| **프로비저닝** | 주/월 | 분/초 |
| **유연성** | 낮음 | 높음 |
| **비용** | 높음 | 낮음 |
| **복잡도** | 분산 | 중앙 |

### 모범 사례

1. **Controller**: 고가용성 클러스터
2. **Security**: Role-based access
3. **Monitoring**: Flow telemetry
4. **테스트**: 네트워크 시뮬레이션

### 미래 전망

1. **P4**: Programmable data plane
2. **IBN**: Intent-based networking
3. **AI**: ML for traffic optimization
4. **Edge**: SD-WAN 확장
