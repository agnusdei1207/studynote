# HTTPS (HTTP Secure)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: HTTPS는 **"HTTP에 **TLS/SSL 암호화 계층을 **추가하여 **데이터 기밀성**, **무결성**, **인증**을 제공하는 **보안 프로토콜"**으로, **TCP 443포트**를 사용하고 **Public Key Infrastructure(PKI)** 기반의 **Digital Certificate**로 **서버 인증**을 수행한다.
> 2. **가치**: **중간자 공격**(Man-in-the-Middle), **도청**(Eavesdropping), **데이터 변조**(Tampering)을 **방지**하고 **SEO**(검색 엔진 최적화)에서 **우선 순위**를 받으며 **브라우저**에서 **"Not Secure"** 경고를 **표시하지 않는다**.
> 3. **융합**: **TLS 1.2/1.3**이 **AES-GCM**, **ChaCha20-Poly1305** 암호화를 사용하고 **HSTS**(HTTP Strict Transport Security), **Certificate Pinning**, **Let's Encrypt**(무료 인증서)와 함께 **HTTPS 강제** 정책을 구현한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
HTTPS는 **"TLS(Transport Layer Security)를 통해 암호화된 HTTP 통신"**이다.

**HTTPS의 보장**:
- **기밀성**(Confidentiality): 제3자가 내용 확인 불가
- **무결성**(Integrity): 데이터 변조 감지
- **인증**(Authentication): 서버 신원 확인

### 💡 비유
HTTPS는 **"봉투에 넣은 편지"**와 같다.
- **HTTP**: 엽서 (내용 노출)
- **HTTPS**: 봉투+봉인 (내용 보호)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         HTTPS의 발전                                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

HTTP (1991):
    • 평문 통신
    • 도청 위험
         ↓
SSL 1.0/2.0 (Netscape, 1994):
    • 최초 웹 암호화
    • 보안 취약점
         ↓
SSL 3.0 (1996):
    • RFC 6101 표준화
    • POODLE 공격으로 deprecated
         ↓
TLS 1.0 (1999):
    • SSL 3.0 개선
    • RFC 2246
         ↓
TLS 1.1 (2006) → TLS 1.2 (2008) → TLS 1.3 (2018):
    • 강화된 암호화
    • RFC 8446
```

## Ⅱ. 아키텍처 및 핵심 원리

### HTTPS 통신 과정

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         HTTPS Handshake (TLS 1.2)                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [1. Client Hello]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Client                                                                      Server   │
    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Client Hello ────────────────────────────────────────────────────────────────>  │  │
    │  │  • TLS Version (예: TLS 1.2)                                                      │  │
    │  │  • Random Bytes (Client Random)                                                   │  │
    │  │  • Cipher Suites (암호화 알고리즘 목록)                                              │  │
    │  │    - TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256                                         │  │
    │  │    - TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384                                         │  │
    │  │  • Compression Methods                                                             │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [2. Server Hello + Certificate]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Client                                                                      Server   │
    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  <───────────────────────────────────────────────────────────────── Server Hello  │  │
    │  │  • TLS Version (선택된 버전)                                                        │  │
    │  │  • Random Bytes (Server Random)                                                    │  │
    │  │  • Cipher Suite (선택된 암호화)                                                      │  │
    │  │  • Session ID                                                                      │  │
    │  │                                                                                     │  │
    │  │  <─────────────────────────────────────────────────────────── Certificate          │  │
    │  │  • 서버 인증서 (X.509)                                                               │  │
    │  │  • Public Key                                                                       │  │
    │  │  • CA Signature                                                                     │  │
    │  │                                                                                     │  │
    │  │  <─────────────────────────────────────────────────────────── Server Key Exchange   │  │
    │  │  • Ephemeral Public Key (ECDHE)                                                      │  │
    │  │                                                                                     │  │
    │  │  <─────────────────────────────────────────────────────────── Server Hello Done     │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [3. Client Key Exchange + Change Cipher Spec]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Client                                                                      Server   │
    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  (Certificate 검증)                                                                │  │
    │  │  • CA 서명 확인                                                                     │  │
    │  │  • 유효 기간 확인                                                                    │  │
    │  │  • 도메인 확인                                                                      │  │
    │  │                                                                                     │  │
    │  │  Client Key Exchange ──────────────────────────────────────────────────────────>  │  │
    │  │  • Pre-Master Secret (서버 Public Key로 암호화)                                       │  │
    │  │                                                                                     │  │
    │  │  Change Cipher Spec ─────────────────────────────────────────────────────────────>  │  │
    │  │  • 이제부터 암호화 전송                                                               │  │
    │  │                                                                                     │  │
    │  │  Finished ───────────────────────────────────────────────────────────────────────>  │  │
    │  │  • Handshake 메시지 해시                                                              │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [4. Server Finished]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Client                                                                      Server   │
    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  <────────────────────────────────────────────────────────── Change Cipher Spec  │  │
    │  │                                                                                     │  │
    │  │  <────────────────────────────────────────────────────────────────── Finished      │  │
    │  │                                                                                     │  │
    │  │  [HTTPS 암호화 통신 시작]                                                            │  │
    │  │  • Session Key = PRF(Pre-Master, Client Random, Server Random)                       │  │
    │  │  • 대칭키 암호화 (AES-128/256-GCM)                                                   │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐

    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  ClientHello ─────────────────────────────────────────────────────────────────>   │  │
    │  │  • Key Share (Client Public Key)                                                   │  │

    │  │                                                                                     │  │

    │  │  <───────────────────────────────────────────────────────────────── Finished       │  │
    │  │                                                                                     │  │
    │  │  (Handshake 완료, 데이터 전송 가능)                                                    │  │
    │  │                                                                                     │  │
    │  │  Finished ───────────────────────────────────────────────────────────────────────>  │  │
    │  │  (Application Data) ─────────────────────────────────────────────────────────────>  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  • 1-RTT (Round Trip Time) 완료                                                         │
    │  • 0-RTT Resumption 지원 (이전 세션 재개)                                                │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### HTTP vs HTTPS

| 구분 | HTTP | HTTPS |
|------|------|-------|
| **프로토콜** | TCP 80 | TCP 443 (TLS) |
| **암호화** | X | O |
| **인증** | X | O (Certificate) |
| **SEO** | 불리 | 유리 |
| **비용** | 무료 | 인증서 비용 |

### Cipher Suite 구성

```
TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
  │    │    │    │    │       │
  │    │    │    │    │       └── HMAC (Integrity)
  │    │    │    │    └────────── AES-256 GCM (Encryption)
  │    │    │    └─────────────── Auth (RSA Key Exchange)
  │    │    └──────────────────── Ephemeral (Perfect Forward Secrecy)
  │    └───────────────────────── Elliptic Curve Diffie-Hellman
  └────────────────────────────── Protocol
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: HTTPS 구축
**상황**: 웹사이트 HTTPS 적용
**판단**:

```bash
# 1. Let's Encrypt 인증서 발급 (무료)
$ sudo apt install certbot python3-certbot-nginx
$ sudo certbot --nginx -d example.com -d www.example.com

# 2. 인증서 자동 갱신 (Cron)
$ sudo crontab -e
# 매일 새벽 2시 갱신 시도
0 2 * * * certbot renew --quiet

# 3. Apache 설정
<VirtualHost *:443>
    ServerName example.com
    SSLEngine on
    SSLCertificateFile /etc/letsencrypt/live/example.com/fullchain.pem
    SSLCertificateKeyFile /etc/letsencrypt/live/example.com/privkey.pem
    # 강력한 Cipher만 허용
    SSLCipherSuite HIGH:!aNULL:!MD5
    # HSTS (HTTPS 강제)
    Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"
</VirtualHost>

# 4. HTTP → HTTPS 리디렉션
<VirtualHost *:80>
    ServerName example.com
    Redirect permanent / https://example.com/
</VirtualHost>

# 5. 테스트
$ openssl s_client -connect example.com:443 -servername example.com
$ curl -I https://example.com
```

## Ⅴ. 기대효과 및 결론

### HTTPS 기대 효과

| 효과 | HTTP | HTTPS |
|------|------|-------|
| **도청 방지** | X | O |
| **변조 방지** | X | O |
| **서버 인증** | X | O |
| **SEO** | 불리 | 유리 |
| **성능** | 빠름 | 핸드셰이크 오버헤드 |

### 미래 전망

1. **TLS 1.3**: 1-RTT 핸드셰이크
2. **ECH**: Encrypted Client Hello
3. **Post-Quantum**: 양자 내성 암호

### ※ 참고 표준/가이드
- **RFC 8446**: TLS 1.3
- **RFC 2818**: HTTP over TLS
