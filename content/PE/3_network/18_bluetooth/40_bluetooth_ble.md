# 블루투스 및 BLE (Bluetooth & BLE)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 블루투스는 **"2.4GHz **ISM **대역**(Industrial, **Scientific, **Medical)**에서 **근거리 **통신**(Short-**Range)**을 **제공**하는 **무선 **기술\\\"**으로, **Classic **Bluetooth**(BR/EDR)**가 **높은 **전송 **률**을 **제공**하고 **BLE**(Bluetooth **Low **Energy)**가 **초저전력** **통신**을 **제공**한다.
> 2. **BLE**: **Advertising**(Advertising **Packets)**로 **주변 **기기**를 **발견**하고 **Connection**(Connection **Request)**으로 **연결**하며 **GATT**(Generic **Attribute **Profile)**가 **Service**와 **Characteristic**을 **정의**하여 **데이터**를 **교환**한다.
> 3. **보안**: **Pairing**(쌍 **구성)**으로 **암호화 **키**(LTK, **IRK)**를 **교환**하고 **LE **Secure **Connections**(ECDH)**로 **MITM**(Man-**in-**the-**Middle)** 공격**을 **방지**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
블루투스는 **"근거리 무선 개인 통신망"**이다.

**블루투스 비교**:
| 버전 | 전송률 | 전력 | 범위 | 사용처 |
|------|--------|------|------|--------|
| **Classic (BR/EDR)** | 1-3 Mbps | 높음 | 10-100m | 오디오 |
| **BLE (4.0+)** | 1 Mbps | 낮음 | 10-50m | 센서 |
| **BLE 5.0+** | 2 Mbps | 최소 | 200m | IoT |

블루투스는 ****무선 **이어폰 ****과 같다.
- **페어링**: 연결 설정
- **연결**: 데이터 전송
- **低전력**: 배터리 절약

## Ⅱ. 아키텍처 및 핵심 원리

### BLE Architecture

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         BLE Architecture (GATT)                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    GATT Hierarchy:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Peripheral (e.g., Heart Rate Monitor)                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Service: Heart Rate (UUID: 0x180D)                                                  │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  Characteristic: Heart Rate Measurement (UUID: 0x2A37)                           │  │  │  │  │
    │  │  │  • Properties: Notify                                                            │  │  │  │  │
    │  │  │  • Value: [0x06, 0x4B] (75 BPM)                                                   │  │  │  │  │
    │  │  │  • Descriptor: Client Characteristic Configuration (CCC)                          │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  Characteristic: Body Sensor Location (UUID: 0x2A38)                             │  │  │  │  │
    │  │  │  • Properties: Read                                                               │  │  │  │  │
    │  │  │  • Value: [0x02] (Wrist)                                                          │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                                   │  │
    │            ▼ ATT Protocol (Attribute Protocol)                                              │  │
    │  Central (e.g., Smartphone)                                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Discover All Services: [0x180D, 0x180A]                                             │  │  │
    │  │  Discover Characteristics: [0x2A37, 0x2A38]                                          │  │  │
    │  │  Read 0x2A38 → Response: 0x02 (Wrist)                                                 │  │  │
    │  │  Write 0x2A37 CCC (Enable Notify)                                                      │  │  │
    │  │  → Receive notifications: [0x06, 0x50] (80 BPM)                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### BLE Connection Process

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         BLE Connection Establishment                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Advertising (Peripheral broadcasts presence):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Channel: 37/38/39 (Advertising channels, 2402 MHz / 2426 MHz / 2480 MHz)                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Advertising Packet (31 bytes max)                                                    │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  │  Header  │  AdvA      │  Data                                                   │  │  │  │  │  │
    │  │  │  │  (2B)   │  (6B)      │  (0-23B)                                                │  │  │  │  │  │
    │  │  │  │         │            │                                                         │  │  │  │  │  │
    │  │  │  │  Type=0  │  AA:BB:CC  │  0x01 (Flags)                                         │  │  │  │  │  │
    │  │  │  │         │  DD:EE:FF  │  0x03 (Complete List of 16-bit Service UUIDs)          │  │  │  │  │  │
    │  │  │  │         │            │  0x180D (Heart Rate Service)                            │  │  │  │  │  │
    │  │  │  │         │            │  0x09 (Complete Local Name)                             │  │  │  │  │  │
    │  │  │  │         │            │  "MyHeartRate"                                         │  │  │  │  │  │
    │  │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │  → Broadcast every 100ms (configurable)                                                 │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Scanning & Connection:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Central (Smartphone)                            Peripheral (Heart Rate Monitor)         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Scan Request (Channel 37)                                                                 │  │  │
    │  │  ──────────────────────────────────────────────────────────────────────────────────────>  │  │
    │  │  Scan Response: "MyHeartRate", RSSI=-45 dBm                                              │  │  │
    │  │  <─────────────────────────────────────────────────────────────────────────────────────  │  │
    │  │                                                                                       │  │  │
    │  │  Connection Request (Channel 37)                                                         │  │  │
    │  │  ──────────────────────────────────────────────────────────────────────────────────────>  │  │
    │  │  • Transmit Window: 10ms                                                                │  │  │
    │  │  • Transmit Interval: 30ms                                                              │  │  │
    │  │  • Latency: 0                                                                           │  │  │
    │  │  • Timeout: 500ms                                                                       │  │  │
    │  │                                                                                        │  │  │
    │  │  Connection Complete (Channel 37)                                                        │  │  │
    │  │  <─────────────────────────────────────────────────────────────────────────────────────  │  │
    │  │  • Connection Interval: 30ms                                                            │  │  │
    │  │  • Slave Latency: 0                                                                     │  │  │
    │  │  • Supervision Timeout: 6000ms                                                          │  │  │
    │  │  → Connection established, hop to data channels (0-36)                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### BLE Pairing (LE Secure Connections)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         BLE Pairing (ECDH Key Exchange)                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Phase 1: Pairing Feature Exchange
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Central → Peripheral: Pairing Request                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • IO Capability: DisplayYesNo (has display, yes/no buttons)                         │  │  │
    │  │  • OOB Data: Not present                                                              │  │  │
    │  │  • Auth Req: Bonding, MITM, SC (Secure Connections)                                  │  │  │
    │  │  • Max Key Size: 16 bytes                                                             │  │  │
    │  │  • Initiator Key Distribution: LTK, IRK, CSRK                                         │  │  │
    │  │  • Responder Key Distribution: LTK, IRK, CSRK                                         │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  Peripheral → Central: Pairing Response                                                   │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Phase 2: LE Secure Connections (ECDH)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Public Key Exchange                                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Central → Peripheral: Public Key PkC                                                  │  │  │
    │  │  Peripheral → Central: Public Key PkR                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  2. Diffie-Hellman Shared Secret                                                           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Both calculate: DhKey = ECDH(PkC, PkR)                                                │  │  │
    │  │  → Only Central and Peripheral know DhKey                                             │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  3. Authentication (Numeric Comparison)                                                    │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Both calculate: Confirm = AES-CMAC(DhKey, Random, Address)                            │  │  │
    │  │  Central → Peripheral: Confirm C                                                      │  │  │
    │  │  Peripheral → Central: Confirm R                                                      │  │  │
    │  │  Central → Peripheral: Random C                                                       │  │  │
    │  │  Peripheral → Central: Random R                                                       │  │  │
    │  │  → Verify confirms match                                                                │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  4. User Confirmation (Numeric Comparison)                                                │  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Both display: 6-digit number (e.g., "123456")                                        │  │  │
    │  │  User compares numbers, confirms they match                                           │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Bluetooth Classic vs BLE 비교

| 특성 | Classic (BR/EDR) | BLE |
|------|------------------|-----|
| **전송률** | 1-3 Mbps | 1 Mbps (2 Mbps v5.0) |
| **전력** | 높음 (100mW) | 낮음 (10mW) |
| **연결 시간** | 3-6초 | 6ms |
| **동시 연결** | 7개 | 제한 없음 |
| **프로필** | SPP, A2DP | GATT |
| **사용처** | 오디오, 데이터 | 센서, IoT |

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────────────────────────────────────────┘
```

```
    │  // Custom Notification Service (0xF000)                                                 │  │
```

## Ⅴ. 기대효과 및 결론

### BLE 기대 효과

| 기능 | 전력 소모 | 데이터 전송 | 배터리 수명 |
|------|-----------|-------------|------------|
| **Advertising** | 0.01-10 mW | 31B | 연결 전 |
| **Connection** | 1-20 mW | 1 Mbps | 연결 중 |
| **Sleep** | 0.001 mW | 없음 | 대부분 |

### 모범 사례

1. **Connection Interval**: 최소화
2. **Slave Latency**: 최대화
3. **Data Size**: MTU 최대
4. **Bonding**: 키 저장

### 미래 전망

1. **LE Audio**: LC3 코덱
2. **Auracast**: 브로드캐스트 오디오
3. **Mesh 1.1**: Direction finding
4. **6G**: 향후 통합

### ※ 참고 표준/가이드
- **Bluetooth SIG**: Core Spec 5.4
- **GATT**: Services
- **Mesh**: Mesh Profile 1.1
