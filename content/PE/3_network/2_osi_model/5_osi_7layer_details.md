# OSI 7계층 상세 (OSI 7-Layer Details)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: OSI 7계층은 **"네트워크 통신 기능을 7개 계층으로 계층화하여 표준화한 모델"**로, 각 계층은 **인터페이스(Interface)**, **프로토콜(Protocol)**, **서비스(Service)**를 정의하며 **PDU(Protocol Data Unit)**가 계층을 통과할 때 **헤더(Header)**가 추가되는 **캡슐화(Encapsulation)**가 발생한다.
> 2. **가치**: **L7(응용)~L1(물리)**까지의 **관심사 분리(Separation of Concerns)**를 통해 **복잡성을 감소**시키며, **벤더 무관한 상호 운용성(Interoperability)**을 보장하고 **문제 격리(Troubleshooting)**를 용이하게 한다.
> 3. **융합**: **L3(IP)**는 **라우터(Router)**, **L2(Ethernet)**는 **스위치(Switch)**, **L4(TCP/UDP)**는 **L4 스위치**의 동작 원리를 정의하며, **L7 HTTP**는 **웹**, **L4 TCP**는 **신뢰성**, **L3 IP**는 **라우팅**, **L2 MAC**는 **전송**, **L1 비트**는 **물리적 신호**를 담당한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
OSI 7계층 모델(Open Systems Interconnection 7-Layer Model)은 **"개방형 시스템 간 통신을 위한 국제 표준 아키텍처"**이다.

**7계층의 역할**:
- **L1(물리)**: 비트 전송
- **L2(데이터링크)**: 프레임 전송, 오류 제어
- **L3(네트워크)**: 패킷 라우팅
- **L4(전송)**: 세그먼트 전송, 신뢰성
- **L5(세션)**: 세션 관리
- **L6(표현)**: 데이터 표현, 암호화
- **L7(응용)**: 사용자 애플리케이션 서비스

### 💡 비유
OSI 7계층은 **"우편 배달 시스템"**과 같다.
- **L7**: 편지 내용 작성
- **L4**: 등기/일반 우편 구분
- **L3**: 주소 기반 라우팅
- **L2**: 운송 수단 (비행기, 트럭)
- **L1**: 도로, 항공로

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         OSI 7계층 표준화 역사                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

1970년대: 벤더 특화 네트워크
    • IBM SNA (Systems Network Architecture)
    • DEC DNA (Digital Network Architecture)
    • 상호 운용성 없음
         ↓
1977년: ISO 표준화 시작
    • OSI 참조 모델 개발
    • 개방형 시스템 간 통신 표준
         ↓
1984년: OSI 7계층 표준 발표
    • ISO/IEC 7498
    • 각 계층별 프로토콜 표준화
         ↓
1990년대~: TCP/IP 실용화
    • OSI 복잡하여 실제 구현은 TCP/IP
    • 하지만 OSI는 교육/이론적 표준으로 사용
```

### 계층별 장비와 PDU

| 계층 | 장비 | PDU | 주요 프로토콜 |
|------|------|-----|-------------|
| **L7** | 게이트웨이, 방화벽 | Data | HTTP, FTP, SMTP |
| **L6** | 암호화 장비 | Data | SSL/TLS, SSH |
| **L5** | 프록시 서버 | Data | NetBIOS, RPC |
| **L4** | L4 스위치, 로드밸런서 | Segment/Datagram | TCP, UDP |
| **L3** | 라우터(L3 스위치) | Packet | IP, ICMP, ARP |
| **L2** | 스위치(L2), 브리지 | Frame | Ethernet, PPP |
| **L1** | 허브, 리피터, 케이블 | Bit | RS-232, RJ-45 |

### 과목 융합 관점 분석

#### 1. 보안 ↔ OSI 계층
- **L7 보안**: Application Firewall (WAF)
- **L6 보안**: SSL/TLS 암호화
- **L4 보안**: Packet Filtering Firewall
- **L3 보안**: IPsec, VPN
- **L2 보안**: MACsec, Port Security

#### 2. 클라우드 ↔ OSI 계층
- **L7**: SaaS (Software as a Service)
- **L4~L6**: PaaS (Platform as a Service)
- **L3**: VPC, Subnet
- **L2**: VLAN, VXLAN

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 기업 네트워크 트러블슈팅
**상황**: 특정 서버 접속 불가
**판단**:
1. **L1 확인**: 케이블 연결, LED 점등
2. **L2 확인**: MAC 주소 학습, VLAN 설정
3. **L3 확인**: IP 주소, 라우팅 테이블
4. **L4 확인**: 포트 열림, 방화벽
5. **L7 확인**: 애플리케이션 로그

```bash
# L1: 인터페이스 상태
ethtool eth0

# L2: MAC 테이블
show mac address-table

# L3: 라우팅 테이블
ip route show

# L4: 포트 확인
netstat -tuln | grep 8080

# L7: 애플리케이션 로그
tail -f /var/log/app.log
```

## Ⅴ. 기대효과 및 결론

### OSI 7계층 모델 기대 효과

| 장점 | 설명 |
|------|------|
| **모듈화** | 각 계층 독립적 개발/수정 |
| **상호 운용성** | 표준화된 인터페이스 |
| **문제 격리** | 계층별 트러블슈팅 |
| **학습 용이성** | 복잡한 네트워크를 단순화 |

### 미래 전망

1. **SDN**: 제어평면과 데이터평면 분리
2. **NFV**: 네트워크 기능 가상화
3. **5G**: Network Slicing

### ※ 참고 표준/가이드
- **ISO/IEC 7498**: OSI RM
- **IEEE 802.3**: Ethernet
- **RFC 791, 793**: IP, TCP
