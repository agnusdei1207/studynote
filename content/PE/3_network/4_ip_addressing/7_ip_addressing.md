# IP 주소 체계 (IP Addressing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 주소는 **"인터넷에 연결된 장치를 식별하는 32비트(IPv4) 또는 128비트(IPv6) 고유 주소"**로, **네트워크 ID**(네트워크 식별)와 **Host ID**(호스트 식별)로 구성되며 **서브넷 마스크(Subnet Mask)**로 경계를 나눈다.
> 2. **가치**: **CIDR(Classless Inter-Domain Routing)** 표기법으로 **유연한 네트워크 설계**가 가능하며, **IPv4 주소 고갈** 문제를 **IPv6**로 해결하고 **라우팅 테이블(Routing Table)**을 통해 **패킷 전달(Packet Forwarding)**을 수행한다.
> 3. **융합**: **DHCP**(Dynamic Host Configuration Protocol)로 **자동 IP 할당**을 하고, **NAT**(Network Address Translation)로 **IPv4 주소 절약**, **VPC**(Virtual Private Cloud)의 **서브넷 설계**, **DNS**(Domain Name System)로 **도메인-IP 매핑**의 기반이 된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
IP 주소는 **"인터넷에 연결된 각 장치에게 할당된 고유 번호"**이다.

**IP 주소의 역할**:
- **식별**: 장치 식별
- **위치**: 네트워크 위치
- **라우팅**: 패킷 경로 결정

### 💡 비유
IP 주소는 **"집 주소"**와 같다.
- **도로명(네트워크)**: 강남구
- **번지(호스트)**: 123번지
- **우편 배달**: 패킷 라우팅

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         IP 주소의 역사                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

ARPANET (1969년):
    • NCP 프로토콜 사용
    • 호스트 번호 (8비트)
         ↓
IPv4 개발 (1981년, RFC 791):
    • 32비트 주소 (43억 개)
    • Classful Addressing (A, B, C Class)
    • 당시엔 충분하다고 생각
         ↓
IPv4 고갈 (1990년대):
    • 웹 폭발로 주소 부족
    • CIDR (1993)로 유연성 확보
    • NAT로 임시 해결
         ↓
IPv6 개발 (1998년, RFC 2460):
    • 128비트 주소 (3.4×10^38 개)
    • 현재 천천히 도입 중
```

## Ⅱ. 아키텍처 및 핵심 원리

### IPv4 주소 구조

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         IPv4 주소 32비트 구조                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Dotted Decimal 표기법]

    192 . 168 . 1 . 100
     │    │    │    │
     │    │    │    └─ Host ID (8비트): 호스트 식별 (0~255)
     │    │    └─────────────── Network ID (24비트): 네트워크 식별
     │    └────────────────────── Network ID (16비트)
     └────────────────────────── Network ID (8비트)

    [이진수 표현]

    192      168      1        100
    └────────┴────────┴────────┴────────┘
    11000000 10101000 00000001 01100100

    [클래스풀 주소 체계]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Class │ First Octet │ Network ID    │ Host ID        │ 주소 범위            │ 용도          │
    │  ──────┼─────────────┼────────────────┼────────────────┼──────────────────────┼──────────────  │
    │  A     │ 0xxxxxxx    │ a.b.c.d        │ d              │ 0.0.0.0 ~ 127.255.255.255  │ 대형 네트워크  │
    │  B     │ 10xxxxxx    │ a.b            │ c.d            │ 128.0.0.0 ~ 191.255.255.255│ 중형 네트워크  │
    │  C     │ 110xxxxx    │ a.b.c          │ d              │ 192.0.0.0 ~ 223.255.255.255│ 소형 네트워크  │
    │  D     │ 1110xxxx    │ Multicast       │ -              │ 224.0.0.0 ~ 239.255.255.255│ 멀티캐스트      │
    │  E     │ 1111xxxx    │ Reserved        │ -              │ 240.0.0.0 ~ 255.255.255.255│ 예약            │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 서브넷 마스크(Subnet Mask)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         서브넷 마스크와 CIDR                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [서브넷 마스크 역할]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • IP 주소 중 어디까지가 Network ID인지 표시                                               │
    │  │  1인 비트: Network ID                                                               │
    │  │  0인 비트: Host ID                                                                  │
    │  • AND 연산으로 네트워크 주소 계산                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [서브넷 마스크 예시]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  IP: 192.168.1.100                                                                     │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Subnet Mask: 255.255.255.0 (/24)                                                   │  │
    │  │  ─────────────────────────────────────────────────────────────────────────────────  │  │
    │  │  IP:      11000000 10101000 00000001 01100100                                        │  │
    │  │  Mask:    11111111 11111111 11111111 00000000                                        │  │
    │  │  AND:     11000000 10101000 00000001 00000000 = 192.168.1.0                          │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │
    │  Network ID: 192.168.1.0                                                                 │
    │  Broadcast: 192.168.1.255                                                               │
    │  Usable Hosts: 192.168.1.1 ~ 192.168.1.254 (254개)                                      │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [CIDR 표기법]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CIDR (Classless Inter-Domain Routing)                                                    │
    │  • IP/Prefix Length 형식                                                               │
    │  │  Prefix Length: Network ID 비트 수                                                    │
    │                                                                                         │
    │  예시:                                                                                   │
    │  • 192.168.1.0/24  → Network 24비트, Host 8비트                                     │
    │  │  10.0.0.0/8      → Network 8비트, Host 24비트                                      │
    │  │  172.16.0.0/12   → Network 12비트, Host 20비트                                     │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [서브넷 크기와 호스트 수]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Prefix │  Subnet Mask       │  Network Address   │ Usable Hosts        │  용도              │
    │  ───────┼────────────────────┼───────────────────┼────────────────────┼──────────────────  │
    │  /8     │ 255.0.0.0         │ 10.0.0.0          │ 16,777,214          │  A Class           │
    │  /16    │ 255.255.0.0       │ 172.16.0.0        │ 65,534              │  B Class           │
    │  /24    │ 255.255.255.0     │ 192.168.1.0       │ 254                 │  C Class           │
    │  /25    │ 255.255.255.128    │ 192.168.1.128     │ 126                 │  반분              │
    │  /26    │ 255.255.255.192    │ 192.168.1.192     │ 62                  │ 4분               │
    │  /27    │ 255.255.255.224    │ 192.168.1.224     │ 30                  │ 8분               │
    │  /28    │ 255.255.255.240    │ 192.168.1.240     │ 14                  │ 16분              │
    │  /29    │ 255.255.255.248    │ 192.168.1.248     │ 6                   │ 32분              │
    │  /30    │ 255.255.255.252    │ 192.168.1.252     │ 2                   │ 64분              │
    │  /31    │ 255.255.255.255    │ 192.168.1.254     │ 1                   │ Point-to-Point    │
    │  /32    │ 255.255.255.255    │ 192.168.1.255     │ 0                   │ 단일 호스트        │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 사설 주소(Private IP) vs 공인 주소(Public IP)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         사설 vs 공인 IP 주소                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [사설 IP 주소 (Private IP Address)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 인터넷에서 라우팅되지 않는 주소                                                           │
    │  │  LAN(Local Area Network) 내부에서만 사용                                                 │
    │  │  • NAT(Network Address Translation)로 공인 IP와 변환                                     │
    │                                                                                         │
    │  Class A: 10.0.0.0 ~ 10.255.255.255 (10.0.0.0/8)                                     │
    │  Class B: 172.16.0.0 ~ 172.31.255.255 (172.16.0.0/12)                                   │
    │  Class C: 192.168.0.0 ~ 192.168.255.255 (192.168.0.0/16)                                  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [공인 IP 주소 (Public IP Address)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 인터넷에서 고유하게 식별되는 주소                                                        │
    │  │  • 전 세계적으로 중복 없음                                                             │
    │  │  • ISP(Internet Service Provider)에서 할당받음                                          │
    │  │  • 인터넷에 직접 연결된 장치에 할당                                                     │
    │                                                                                         │
    │  예시: 8.8.8.8 (Google DNS), 1.1.1.1 (Cloudflare DNS)                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [NAT (Network Address Translation)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  [내부 네트워크]                           [인터넷]                                     │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  PC1: 192.168.1.10                         │  │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  PC2: 192.168.1.11                         │  │  │  인터넷                                                                                                           │  │  │
    │  │  PC3: 192.168.1.12                         │  │  │  ┌──────────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  └─────────────────────────────────────────────┘  │  │  │  Server: 203.0.113.10 (Public IP)                                            │  │  │
    │                      │                            │  │  └──────────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                      ▼                            │  └──────────────────────────────────────────────────────────────────────────────────────────┘  │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  NAT Router (공인 IP: 203.0.113.10)                                               │  │
    │  │  ┌────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │ NAT Table:                                                                     │  │  │
    │  │  │  ┌────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  │ Private IP    │ Public IP      │ Port          │ Mapping            │  │  │  │
    │  │  │  │ 192.168.1.10   │ 203.0.113.10   │ 5000          │ 1:1 NAPT           │  │  │  │
    │  │  │  │ 192.168.1.11   │ 203.0.113.10   │ 5001          │ 1:1 NAPT           │  │  │  │
    │  │  │  │ 192.168.1.12   │ 203.0.113.10   │ 5002          │ 1:1 NAPT           │  │  │  │
    │  │  │  └────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  └────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  └──────────────────────────────────────────────────────────────────────────────────────────┘
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### IPv4 vs IPv6

| 항목 | IPv4 | IPv6 |
|------|------|------|
| **주소 길이** | 32비트 | 128비트 |
| **주소 수** | 43억 개 | 3.4×10³⁸ 개 |
| **표기법** | Dotted Decimal | Colon Hexadecimal |
| **헤더** | 20~60바이트 | 40바이트 |
| **주소 자동 할당** | DHCP | SLAAC |
| **보안성** | IPSec (선택) | IPSec (필수) |

### 과목 융합 관점 분석

#### 1. 라우터 ↔ IP 주소
- **Routing Table**: 네트워크 → 다음 홉
- **Longest Prefix Match**: 가장 구체적인 경로 선택

#### 2. DNS ↔ IP 주소
- **A Record**: 도메인 → IPv4
- **AAAA Record**: 도메인 → IPv6

## Ⅴ. 기대효과 및 결론

### IP 주소 설계 기대 효과

| 설계 측면 | 잘못된 설계 | 올바른 설계 |
|----------|-----------|-----------|
| **주소 범위** | 너무 크거나 작음 | 적절한 크기 |
| **서브넷 분리** | 혼재 | 티어별 분리 |
| **예비** | 미래 고려 안 함 | 여유분 확보 |

### 미래 전망

1. **IPv6**: 전면 도입
2. **SDN**: 중앙 집중 IP 관리
3. **IoT**: 수십억 기기 연결

### ※ 참고 표준/가이드
- **RFC 791**: IPv4
- **RFC 2460**: IPv6
- **RFC 4632**: IPv6 SLAAC
