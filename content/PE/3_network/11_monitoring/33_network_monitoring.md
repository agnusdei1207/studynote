# 네트워크 모니터링 (Network Monitoring)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 모니터링은 **"네트워크 **상태**를 **실시간**으로 **관찰**하고 **이상**을 **탐지**하는 **기술"**로, **SNMP**(Polling), **NetFlow/sFlow**(Export), **Streaming Telemetry**(Push)로 **데이터**를 **수집**한다.
> 2. **메트릭**: **Traffic**(트래픽 **양), **Error**(패킷 **손실), **Availability**(연결 **가용성), **Latency**(지연 **시간), **Jitter**(지연 **변동)를 **측정**한다.
> 3. **도구**: **SNMP**(장비 **조회), **Wireshark**(패킷 **캡처), **PRTG**, **SolarWinds**, **Zabbix**, **Grafana**(시각화), **Telegraf**(수집)가 **대표적**이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
네트워크 모니터링은 **"네트워크 가시성 확보"**이다.

**모니터링 레벨**:
| 레벨 | 범위 | 도구 |
|------|------|------|
| **Link** | 단일 포트 | ifconfig, ethtool |
| **Device** | 전체 장비 | SNMP, CLI |
| **Network** | 전체 망 | NetFlow, sFlow |
| **Application** | 애플리케이션 | APM |

### 💡 비유
네트워크 모니터링은 ****교통 **CCTV ****와 같다.
- **카메라**: NetFlow/sFlow
- **감시자**: NMS
- **알림**: SNMP Trap

## Ⅲ. 융합 비교 및 다각도 분석

### NetFlow vs sFlow

| 구분 | NetFlow | sFlow |
|------|---------|-------|
| **벤더** | Cisco | HP (표준) |
| **방식** | Sampled (1:N) | Random N |
| **오버헤드** | 중간 | 낮음 |
| **계층기** | Router-only | Router/Switch |

### TCP Ping Check

```bash
# Measure TCP connection latency
hping2 -i eth0 -p 80 -c 10 server.example.com

# Output:
# HPING server.example.com (eth0 192.168.1.1): 80 data bytes
# seq 0: ttl=64 id=1 time=10.2 ms
# seq 1: ttl=64 id=1 time=10.1 ms
# seq 2: ttl=64 id=1 time=10.3 ms
# ...
#
# --- server.example.com hping statistic ---
# 10 packets transmitted, 10 packets received, 0% packet loss
# round-trip min/avg/max = 10.1/10.2/10.3 ms
```

## Ⅴ. 기대효과 및 결론

### 모니터링 기대 효과

| 효과 | 없음 | 모니터링 |
|------|------|---------|
| **장애 감지** | 늦음 | 빠름 |
| **원인 파악** | 어려움 | 쉬움 |
| **용량 파악** | 불가 | 가능 |

### 모범 사례

1. **Ping**: 기본 연결 확인
2. **SNMP**: 장비 상태
3. **NetFlow**: 트래픽 분석
4. **Syslog**: 로그 수집

### 미래 전망

1. **Telemetry**: Streaming metrics
2. **AI**: Anomaly detection
3. **Auto-Remediation**: 자진 복구

### ※ 참고 표준/가이드
- **SNMP**: RFC 1157
- **NetFlow**: v9 RFC
- **sFlow**: RFC 3176
