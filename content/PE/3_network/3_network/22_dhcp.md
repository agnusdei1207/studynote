# DHCP (Dynamic Host Configuration Protocol)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DHCP는 **"호스트**가 **IP 주소**, **Subnet Mask**, **Gateway**, **DNS**를 **자동으로 **할당**받는 **프로토콜"**으로, **UDP 67/68포트**를 **사용**하고 **DHCP Server**가 **주소 풀**(Address Pool)을 **관리**하며 **Lease**(임대) **기간**으로 **IP 재사용**을 **지원**한다.
> 2. **DORA 프로세스**: **Discover**(브로드캐스트), **Offer**(서버 제안), **Request**(요청), **ACK**(확인)로 **IP 할당**을 **협상**하며 **DHCP Relay**(Agent)가 **다른 네트워크**의 **DHCP 요청**을 **전달**한다.
> 3. **융합**: **RFC 2131**이 **표준**이며 **ISC dhcpd**, **Windows Server DHCP**, **Cisco DHCP**가 **구현**을 **제공**하고 **Static IP**(고정)와 **Dynamic IP**(동적)를 **혼용**하며 **Reservation**으로 **특정 MAC**에 **고정 IP**를 **할당**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
DHCP는 **"IP 주소를 자동으로 할당하는 프로토콜"**이다.

**DHCP 할당 정보**:
- **IP Address**: 호스트 주소
- **Subnet Mask**: 네트워크 마스크
- **Default Gateway**: 라우터 주소
- **DNS Server**: DNS 서버
- **Lease Time**: 대여 기간

### 💡 비유
DHCP는 **"호텔 체크인****와 같다.
- **객실**: IP 주소
- **체크인**: 할당
- **체크아웃**: 반납
- **예약**: Reservation

    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  SRC: 0.0.0.0:68    →    DST: 255.255.255.255:67                                  │  │  │

    │  │  │  CHADDR: Client MAC                                                               │  │  │

    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  4. DHCP Request (Broadcast)                                                         │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  SRC: 0.0.0.0:68    →    DST: 255.255.255.255:67                                  │  │  │
    │  │  │  XID: 0x12345678                                                                 │  │  │
    │  │  │  Options: 53 (Message Type = Request)                                              │  │  │
    │  │  │           54 (Server ID = DHCP Server IP)                                          │  │  │
    │  │  │           50 (Requested IP = 192.168.1.100)                                        │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │

    │
    │  DHCP Server
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐
    │  │  2. DHCP Offer (Unicast/Broadcast)                                                   │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  │  SRC: 192.168.1.1:67 → DST: 255.255.255.255:68 (or Client MAC)                      │  │  │
    │  │  │  XID: 0x12345678                                                                 │  │  │
    │  │  │  YIADDR: 192.168.1.100 (Offered IP)                                               │  │  │

    │  │  │           51 (Lease Time = 86400)                                                   │  │  │
    │  │  │           1 (Subnet Mask = 255.255.255.0)                                           │  │  │
    │  │  │           3 (Router = 192.168.1.1)                                                  │  │  │
    │  │  │           6 (DNS = 8.8.8.8, 8.8.4.4)                                                │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                       │  │
    │  │  3. DHCP ACK (Unicast/Broadcast)                                                      │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  SRC: 192.168.1.1:67 → DST: 255.255.255.255:68                                      │  │  │
    │  │  │  XID: 0x12345678                                                                 │  │  │
    │  │  │  YIADDR: 192.168.1.100 (Confirmed IP)                                              │  │  │
    │  │  │  Options: 53 (Message Type = ACK)                                                   │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │

    │  │     │  ▼                                                                     ▼            │  │  │

    │  │     │  ────────────────────────────────────────>                                    │  │  │

## Ⅲ. 융합 비교 및 다각도 분석

### IP 할당 방식 비교

| 방식 | 장점 | 단점 | 사용처 |
|------|------|------|--------|
| **Static** | 고정 | 수동 관리 | 서버, 프린터 |
| **Dynamic** | 자동 | 변동 | PC, 모바일 |
| **Reservation** | 자동+고정 | 설정 필요 | 특정 장비 |

### DHCP vs BOOTP

| 구분 | BOOTP | DHCP |
|------|-------|------|
| **IP 할당** | 수동 | 자동 |
| **Lease** | 영구 | 임대 |
| **임시 IP** | 불가 | 가능 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: ISC dhcpd 설정
**상황**: DHCP 서버 구축
**판단**:

```conf
# /etc/dhcp/dhcpd.conf

# 서브넷 정의
subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.100 192.168.1.200;    # 할당 범위
    option routers 192.168.1.1;           # 게이트웨이
    option subnet-mask 255.255.255.0;
    option domain-name-servers 8.8.8.8, 8.8.4.4;
    option broadcast-address 192.168.1.255;
    default-lease-time 86400;              # 24시간
    max-lease-time 172800;                 # 48시간
}

# 예약 (고정 IP)
host printer {
    hardware ethernet 00:11:22:33:44:55;
    fixed-address 192.168.1.10;
}
```

## Ⅴ. 기대효과 및 결론

### DHCP 기대 효과

| 효과 | Static IP | DHCP |
|------|----------|------|
| **관리** | 수동 | 자동 |
| **충돌** | 가능 | 방지 |
| **이동성** | 불편 | 편리 |

### 모범 사례

1. **예약**: 서버 장비
2. **Lease**: 적절한 시간
3. **Relay**: 다중 서브넷
4. **Failover**: 이중화

### 미래 전망

1. **IPv6**: SLAAC
2. **Stateless**: RA 기반
3. **Zero Touch**: 자동 구성

### ※ 참고 표준/가이드
- **RFC 2131**: DHCP
- **RFC 3315**: DHCPv6
