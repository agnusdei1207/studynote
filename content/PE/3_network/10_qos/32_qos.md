# QoS (Quality of Service)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: QoS는 **"네트워크 **트래픽 **품질**을 **보장**하는 **기술"**으로, **Prioritization**(우선순위 **부여), **Traffic Shaping**(트래픽 **제어), **Congestion Management**(혼잡 **관리)로 **중요 **애플리케이션**의 **성능**을 **보장**한다.
> 2. **모델**: **IntServ**(Integrated Services)는 **RSVP**로 **경로 **예약**하고 **DiffServ**(Differentiated Services)는 **DSCP**(Differentiated Services Code Point)로 **패킷 **분류**한다.
> 3. **기법**: **Policing**(속도 **제한), **Shaping**(평활화), **Queueing**(WFQ, **RED), **Admission Control**(허용 **제어)로 **구현**되며 **VoIP**, **스트리밍**, **게임**에 **적용**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
QoS는 **"네트워크 서비스 품질 보장"**이다.

**QoS 파라미터**:
| 파라미터 | 설명 | 단위 |
|----------|------|------|
| **Bandwidth** | 전송 속도 | Mbps |
| **Latency** | 지연 시간 | ms |
| **Jitter** | 지연 변동 | ms |
| **Loss** | 패킷 손실률 | % |

### 💡 비유
QoS는 ****고속도로 **차로 ****와 같다.
- **일반 차**: 일반 트래픽
- **차선**: 우선순위 트래픽
- **통행료**: QoS 정책

## Ⅲ. 융합 비교 및 다각도 분석

### IntServ vs DiffServ

| 구분 | IntServ | DiffServ |
|------|---------|----------|
| **예약** | Per-flow | Per-class |
| **확장성** | 낮음 | 높음 |
| **신호** | RSVP | 없음 |
| **복잡도** | 높음 | 낮음 |

### 큐잉 알고리즘

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Queueing Disciplines                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    FIFO (First-In, First-Out):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Packet1 → Packet2 → Packet3 → Packet4 → ... → Out                             │  │  │
    │  │  → No prioritization                                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Priority Queuing:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Queue 1 (High)  → → → → → Out                                                       │  │  │
    │  │  Queue 2 (Med)   → → → → → (only if Queue 1 empty)                                 │  │  │
    │  │  Queue 3 (Low)   → → → → → (only if Queue 1, 2 empty)                                │  │  │
    │  │  → High priority always served first                                                   │  │  │
    │  │  → Low priority may starve                                                             │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    WFQ (Weighted Fair Queuing):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Class 1 (Weight 3)  → → → → (30% bandwidth)                                      │  │  │
    │  │  Class 2 (Weight 2)  → → → → (20% bandwidth)                                      │  │  │
    │  │  Class 3 (Weight 1)  → → → → (10% bandwidth)                                      │  │  │
    │  │  → Fair allocation based on weights                                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### Cisco QoS 설정

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Cisco QoS Configuration                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ! Class maps
    class-map match-any VOICE
     match dscp ef
    class-map match-any VIDEO
     match dscp af41

    ! Policy map
    policy-map WAN-EDGE
     class VOICE
      priority percent 30        ! 30% bandwidth, strict priority
     class VIDEO
      bandwidth percent 20        ! 20% bandwidth (WFQ)
     class class-default
      fair-queue                 ! Remaining bandwidth

    ! Interface
    interface GigabitEthernet0/0
     service-policy output WAN-EDGE
```

### Traffic Shaping vs Policing

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Shaping vs Policing                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Traffic Shaping (Leaky Bucket):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Incoming Burst → Buffer → Token Bucket → Out (smoothed)                               │  │  │
    │  │                                                                                       │  │  │
    │  │  • Excess traffic queued (buffered)                                                   │  │  │
    │  │  • Delayed, not dropped                                                                │  │  │
    │  │  → Good for egress (outgoing) traffic                                                │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Traffic Policing:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Incoming → Meter → Within limit? Pass / Exceed → Drop                                 │  │  │
    │  │                                                                                       │  │  │
    │  │  • Excess traffic dropped immediately                                                   │  │  │
    │  │  • No buffering                                                                       │  │  │
    │  │  → Good for ingress (incoming) traffic                                                │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

**상황**: VoIP + 데이터 혼합
**판단**: DiffServ + Priority Queuing

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Enterprise WAN QoS Design                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Traffic Classes:                                                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Voice: DSCP EF (46), Priority Queue, 20% bandwidth                                 │  │  │
    │  │  • Video: DSCP AF41 (34), 30% bandwidth                                               │  │  │
    │  │  • Critical Apps: DSCP AF31 (26), 20% bandwidth                                       │  │  │

    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  → Voice gets lowest latency and jitter                                                    │  │
    │  → Data gets guaranteed minimum bandwidth                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### QoS 기대 효과

| 효과 | 없음 | QoS |
|------|------|-----|
| **지연** | 불규형 | 보장 |
| **대역폭** | 경쟁 | 할당 |
| **손실** | 높음 | 낮음 |

### 모범 사례

1. **VoIP**: EF + Priority Queue
2. **스트리밍**: AF41 + WFQ
3. **데이터**: AF11 + Best Effort
4. **모니터링**: DSCP 카운터 확인

### 미래 전망

1. **SDN**: 중앙 집중 QoS
2. **AI**: 자동 트래픽 분류
3. **Segment Routing**: 경로 기반 QoS

### ※ 참고 표준/가이드
- **RFC 2474**: DSCP
- **RFC 2475**: DiffServ
- **RFC 2205**: IntServ
