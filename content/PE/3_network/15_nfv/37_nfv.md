# NFV (Network Functions Virtualization)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: NFV는 **"네트워크 **기능**(Firewall, **Load Balancer, **DPI)**을 **전용 **하드웨어**에서 **가상 **머신**(VM)/**컨테이너**로 **이전**하는 **기술\"**로, **VNF**(Virtual Network Function)**가 **범용 **서버**에서 **실행**되고 **Service Chaining**(서비스 **연결)**로 **트래픽**을 **처리**한다.
> 2. **구조**: **NFVI**(NF **Infrastructure)**가 **컴퓨팅**, **스토리지**, **네트워크**를 **제공**하고 **VNF Manager**(Lifecycle **관리)**와 **Orchestrator**(서비스 **배치)**가 **자동화**하며 **MANO**(Management **and **Orchestration)** **프레임워크**를 **따른다.
> 3. **이점**: **CapEx**(장비 **투자) **감소**, **OpEx**(운영 **비용) **절감**, **Time-to-Market**(서비스 **출시 **기간)** **단축**, **유연성**(Elastic **Scaling)**이 **있고 **SDN**과 **결합**하여 **완전 **가상화 **네트워크**를 **구현**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
NFV는 **"네트워크 장비 가상화"**이다.

**전통 장비 vs NFV**:
| 구분 | 전통 장비 | NFV |
|------|-----------|-----|
| **형태** | 전용 하드웨어 | 소프트웨어 |
| **확장** | 장비 추가 | VM/Container 추가 |
| **위치** | 물리 위치 | 어디든 |
| **수명** | 5-10년 | 몇 달 |

### 💡 비유
NFV는 ****앱 **스토어 ****와 같다.
- **장비**: 앱
- **서버**: 스마트폰
- **설치**: 다운로드

### Service Chaining

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Service Chaining (SFC)                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Traditional (Physical):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Internet ──[Cable]──> [FW] ──[Cable]──> [IPS] ──[Cable]──> [LB] ──[Cable]──> Servers       │  │
    │  → Fixed order, physical cabling, expensive                                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    NFV Service Chaining:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Forwarding Graph (flexible):                                                            │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Traffic Classifier (Source: subscriber_type)                                        │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  if (subscriber == "gold"):                                                     │  │  │  │
    │  │  │      chain = FW → IPS → LB → WAN Optimizer                                     │  │  │  │
    │  │  │  elif (subscriber == "silver"):                                                 │  │  │  │
    │  │  │      chain = FW → LB                                                           │  │  │  │
    │  │  │  else:                                                                          │  │  │  │
    │  │  │      chain = FW                                                                │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  SFC Encapsulation (NSH - Network Service Header):                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Original Packet                                                                     │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Eth │ IP │ TCP │ Payload                                                       │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │           │ Add NSH                                                                   │  │  │
    │  │           ▼                                                                            │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Eth │ NSH (SPI=100, SI=1) │ IP │ TCP │ Payload                                  │  │  │  │
    │  │  │  └─────────────────────────────────────────────────────────────────────────────┘  │  │  │  │
    │  │  │      ↑ NSH tells VNFs: "You're service #1, forward to #2"                             │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### NFV 기대 효과

| 효과 | 전통 | NFV |
|------|------|-----|
| **CapEx** | 높음 | 40-60% 절감 |
| **OpEx** | 높음 | 30-50% 절감 |
| **배포** | 주/월 | 일/시 |
| **유연성** | 낮음 | 높음 |

### 모범 사례

1. **트래픽**: DPDK, SR-IOV
2. **배치**: Automation
3. **모니터링**: Real-time
4. **HA**: Active-Active

### 미래 전망

1. **CNF**: Cloud-native VNF
2. **5G**: Core virtualization
3. **Edge**: MEC + NFV
4. **AI**: VNF orchestration
