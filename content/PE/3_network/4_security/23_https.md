# HTTPS/SSL (HTTP Secure)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: HTTPS는 **"HTTP를 **SSL/TLS로 **암호화**하여 **전송 계층 **보안**을 **제공**하는 **프로토콜"**으로, **TCP 443포트**를 **사용**하고 **X.509 Certificate**(디지털 인증서)로 **서버 **신원을 **확인**하며 **PKI**(Public Key Infrastructure)로 **인증서**를 **관리**한다.
> 2. **TLS Handshake**: **Client Hello**, **Server Hello**, **Certificate Exchange**, **Key Exchange**(Diffie-Hellman/RSA), **Finished**로 **세션 키**를 **협상**하고 **Symmetric Encryption**(AES)으로 **데이터**를 **암호화**한다.
> 3. **융합**: **SSL 3.0**(deprecated), **TLS 1.2**, **TLS 1.3**(1-RTT)가 **버전**이며 **Certificate Authority**(CA, Let's Encrypt)가 **인증서**를 **발급**하고 **HSTS**, **Certificate Pinning**으로 **보안**을 **강화**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
HTTPS는 **"암호화된 HTTP"**이다.

**HTTPS 특징**:
- **기밀성**: 암호화
- **무결성**: 위변조 방지
- **인증**: 서버 확인

### 💡 비유
HTTPS는 **"봉투에 넣은 편지****와 같다.
- **편지**: HTTP
- **봉투**: TLS
- **우표**: 인증서

## Ⅱ. 아키텍처 및 핵심 원리

### TLS 1.2 Handshake

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLS 1.2 Handshake                                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Client                                                Server
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Client Hello                                                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • TLS Version (1.2)                                                                 │  │  │
    │  │  │  • Random (32 bytes)                                                              │  │  │
    │  │  │  • Cipher Suites (TLS_AES_256_GCM_SHA384, ...)                                    │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │  ──────────────────────────────────────────────────────────────────────────────────────────▶│
    │                                                                                         │  │
    │  2. Server Hello                                                                       │  │
    │  ◀──────────────────────────────────────────────────────────────────────────────────────────│
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  • TLS Version Selected                                                               │  │  │
    │  │  │  • Random (32 bytes)                                                              │  │  │
    │  │  │  • Cipher Suite Selected                                                          │  │  │
    │  │  │  • Session ID                                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  3. Certificate + Server Key Exchange                                                  │  │
    │  ◀──────────────────────────────────────────────────────────────────────────────────────────│
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • X.509 Certificate                                                                 │  │  │
    │  │  │  • Public Key (RSA/ECDSA)                                                         │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  4. Client Key Exchange + Change Cipher Spec                                            │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────────▶│
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Pre-Master Secret (Encrypted with Server Public Key)                              │  │  │
    │  │  │  • Change Cipher Spec                                                             │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  5. Server Change Cipher Spec + Finished                                               │  │
    │  ◀──────────────────────────────────────────────────────────────────────────────────────────│
    │                                                                                         │  │
    │  6. Client Finished                                                                     │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────────▶│
    │                                                                                         │  │
    │  → Secure Channel Established (Symmetric Encryption)                                    │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### TLS 1.3 Handshake (1-RTT)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLS 1.3 Handshake (Improved)                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Client                                                Server
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ClientHello                                                                          │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────────▶│
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Key Share (DH Public Key)                                                         │  │  │
    │  │  │  → Server가 즉시 Key 계산 가능                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  ServerHello + Encrypted Extensions + Finished                                         │  │
    │  ◀──────────────────────────────────────────────────────────────────────────────────────────│
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Key Share (DH Public Key)                                                         │  │  │
    │  │  │  • Certificate (if needed)                                                         │  │  │
    │  │  │  • Finished                                                                       │  │  │
    │  │  │  → 1-RTT 완료 (TLS 1.2는 2-RTT)                                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │
    │                                                                                         │  │
    │  Client Finished                                                                        │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────────▶│
    │                                                                                         │  │
    │  → Application Data Begins                                                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

| 버전 | RTT | 보안 | 속도 |
|------|-----|------|------|
| **SSL 3.0** | 2 | 취약 | 느림 |
| **TLS 1.0** | 2 | 약함 | 중간 |
| **TLS 1.2** | 2 | 강함 | 중간 |
| **TLS 1.3** | 1 | 매우 강함 | 빠름 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: Nginx HTTPS 설정
**상황**: HTTPS 배포
**판단**:

```conf
server {
    listen 443 ssl http2;
    server_name example.com;

    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers on;

    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    add_header Strict-Transport-Security "max-age=31536000" always;
}
```

### HTTPS 기대 효과

| 효과 | HTTP | HTTPS |
|------|------|-------|
| **도청** | 가능 | 방지 |

2. **Certificate Transparency**: 공개 기록
