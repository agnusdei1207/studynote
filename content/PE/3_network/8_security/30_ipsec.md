# IPSec (IP Security)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPSec는 **"IP **계층 **보안 **프로토콜"**로, **AH**(Authentication Header)와 **ESP**(Encapsulating Security Payload)로 **데이터 **인증**과 **암호화**를 **제공**하고 **IKE**(Internet Key Exchange)로 **키 **교환**을 **관리**한다.
> 2. **모드**: **Transport Mode**(데이터 **부분만 **보호)와 **Tunnel Mode**(전체 **패킷 **캡슐화)로 **작동**하며 **Tunnel Mode**는 **VPN**(Virtual Private Network) **구성**에 **사용**된다.
> 3. **프로토콜**: **IKEv1/IKEv2**로 **SA**(Security Association)를 **협상**하고 **IPcomp**로 **압축**하며 **Replay Attack** 방지를 **위해 **Sequence Number**와 **Anti-Replay Window**를 **사용**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
IPSec는 **"IP 계층 보안 표준"**이다.

**보안 기능**:
- **기밀성**: ESP 암호화
- **무결성**: AH/ESP 인증
- **인증**: SA 인증
- **반재전송**: Sequence Number

### 💡 비유
IPSec는 ****보안 **등나무 ****우편 ****서비스 ****와 같다.
- **봉투**: ESP 캡슐화
- **서명**: AH 인증
- **키**: IKE 교환

### IPSec 아키텍처

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         IPSec Architecture                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Original IP Packet                                                                      │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  IP Header    │  TCP Header  │  Data                                               │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
            │
            ▼
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Transport Mode (End-to-End)                                                            │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  IP Header    │  ESP Header  │  TCP Header  │  Encrypted Data  │  ESP Trailer    │  │  │
    │  │  (Original)  │  (Added)    │  (Original) │  (Encrypted)      │  (Added)        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  → Payload (TCP + Data) encrypted, IP header unchanged                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
            │
            ▼
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Tunnel Mode (Gateway-to-Gateway, VPN)                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  New IP Header  │  ESP Header  │  Original IP Header  │  Encrypted Packet  │  ESP   │  │
    │  │  (Added)       │  (Added)    │  (Original)          │  (Entire packet)    │ Trailer│  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  → Entire original packet encrypted, new IP header added (tunnel)                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### SA (Security Association)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Security Association                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  SA Triple (Unique Identifier)                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  <SPI, IP Destination, Protocol>                                                     │  │  │
    │  │                                                                                       │  │  │
    │  │  • SPI (Security Parameter Index): 32-bit identifier in packet header                │  │  │
    │  │  • IP Destination: Destination IP address                                            │  │  │
    │  │  • Protocol: AH (51) or ESP (50)                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  SA Parameters:                                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Encryption Algorithm: AES-CBC, AES-GCM                                           │  │  │
    │  │  • Encryption Key: 128/192/256-bit                                                   │  │  │
    │  │  • Authentication Algorithm: HMAC-SHA256                                            │  │  │
    │  │  • Authentication Key: 256-bit                                                       │  │  │
    │  │  • Mode: Transport or Tunnel                                                         │  │  │
    │  │  • Lifetime: Time (seconds) or Data (bytes)                                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  • Two SAs per direction: One for inbound, one for outbound                              │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### AH vs ESP

| 구분 | AH | ESP |
|------|----|-----|
| **Protocol ID** | 51 | 50 |
| **인증** | O | O |
| **암호화** | X | O |
| **NAT Traversal** | 어려움 | 가능 |
| **사용** | 거의 안함 | 일반적 |

## Ⅳ. 실무 적용 및 기술사적 판단

### IPSec 설정 (Linux strongSwan)

```bash
# /etc/ipsec.conf
config setup
    charondebug="ike 2, knl 2, cfg 2"

conn site-to-site
    left=1.1.1.1              # Gateway A
    leftsubnet=192.168.1.0/24  # Site A network
    leftid=@gatewayA.example.com
    leftauth=pubkey
    leftcert=gatewayA.cert.pem

    right=2.2.2.2             # Gateway B
    rightsubnet=192.168.2.0/24 # Site B network
    rightid=@gatewayB.example.com
    rightauth=pubkey
    rightcert=gatewayB.cert.pem

    keyexchange=ikev2
    ike=aes256gcm16-prfsha256-modp3072!
    esp=aes256gcm16-prfsha256!
    dpdaction=restart
    auto=start

# /etc/ipsec.secrets
gatewayA.example.com : RSA gatewayA.key.pem
```

## Ⅴ. 기대효과 및 결론

### IPSec 기대 효과

| 효과 | 없음 | IPSec |
|------|------|-------|
| **도청** | 가능 | 방지 |
| **변조** | 가능 | 방지 |
| **VPN** | 불가 | 가능 |

### 모범 사례

1. **알고리즘**: AES-GCM (암호화 + 인증)
2. **키 교환**: IKEv2 (간단, 빠름)
3. **모드**: Tunnel (VPN), Transport (End-to-End)
4. **모니터링**: SA 상태, 암호화 알고리즘

### 미래 전망

1. **IPsec/IKEv2**: 계속 사용
2. **WireGuard**: 간단한 VPN 대안
3. **TLS 1.3**: 애플리케이션 계층

### ※ 참고 표준/가이드
- **RFC 4301**: IPSec Architecture
- **RFC 7296**: IKEv2
- **NIST**: SP 800-77 (Guide to IPSec)
