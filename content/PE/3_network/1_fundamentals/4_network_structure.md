# 정보통신망 구조 (Network Structure)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 정보통신망 구조는 **"데이터 통신을 위한 계층적(Layered) 아키텍처"**로, **OSI 7계층 모델**과 **TCP/IP 4계층 모델**로 표준화되며, 각 계층은 **인터페이스(Interface)**, **프로토콜(Protocol)**, **서비스(Service)**를 제공한다.
> 2. **가치**: **계층화(Layering)**를 통해 **관심사 분리(Separation of Concerns)**를 실현하며, **캡슐화(Encapsulation)**와 **역캡슐화(Decapsulation)**를 통해 데이터가 계층을 통과할 때 **헤더(Header)**와 **트레일러(Trailer)**가 추가/제거된다.
> 3. **융합**: **OSI 7계층**은 네트워크 장비(라우터, 스위치, 허브)의 동작 원리를 정의하며, **TCP/IP**는 **인터넷(Internet)**의 실질적인 표준으로 **클라우드**, **IoT**, **5G**, **SDN(Software-Defined Networking)**의 기반이 된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
정보통신망 구조는 **"컴퓨터 간 데이터 통신을 위해 계층별로 기능을 분담하는 아키텍처"**이다.

**계층화의 목적**:
- **복잡성 감소**: 복잡한 문제를 작은 단위로 분해
- **표준화**: 각 계층의 인터페이스 표준화
- **상호 운용성**: 다른 벤더 장비 간 통신 보장

### 💡 비유
계층 구조는 **"우편 배달 시스템"**과 같다.
- **응용 계층**: 편지 내용 작성
- **전송 계층**: 등기/우편물 구분
- **네트워크 계층**: 주소 기반 라우팅
- **데이터링크 계층**: 운송 수단 (비행기, 트럭)
- **물리 계층**: 도로, 항공로

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         네트워크 아키텍처 진화                                           │
└─────────────────────────────────────────────────────────────────────────────────────────┘

 초기: 벤더 특화 (1960년대)
    • IBM SNA, DEC DNA
    • 상호 운용성 없음
    • 폐쇄형 네트워크
         ↓
 OSI 7계층 (ISO, 1984)
    • 개방형 시스템 간 통신 표준
    • 이론적 모델
    • 너무 복잡하여 실제 구현 어려움
         ↓
 TCP/IP (ARPANET, 1970년대~)
    • 실용적 인터넷 프로토콜
    • 4계층 간소 모델
    • 현재 인터넷 표준
         ↓
 현대 네트워킹 (2000년대~)
    • SDN (Software-Defined Networking)
    • NFV (Network Function Virtualization)
    • 5G, IoT
```

## Ⅱ. 아키텍처 및 핵심 원리

### OSI 7계층 vs TCP/IP 4계층

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         OSI 7계층 vs TCP/IP 4계층                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    OSI 7계층 모델                     TCP/IP 4계층 모델
    (이론적 표준)                      (실용적 표준)

    ┌─────────────────────────────────────────────────────────────────────────────────────────┐
    │  7. 응용 계층 (Application Layer)                                                      │
    │     • 사용자 애플리케이션에 네트워크 서비스 제공                                        │
    │     • HTTP, FTP, SMTP, DNS, Telnet                                                    │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  6. 표현 계층 (Presentation Layer)                                                     │
    │     • 데이터 표현 방식, 암호화, 압축                                                    │
    │     • JPEG, MPEG, ASCII, SSL/TLS                                                      │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  5. 세션 계층 (Session Layer)                                                          │
    │     • 세션 설정, 유지, 종료                                                             │
    │     • NetBIOS, RPC, SOCKS                                                             │
    │  └─────────────────────────────────────────────────────────────────────────────────────  │
    │                                                                                         │
    │                                  ┌───────────────────────────────────────────────────┐ │
    │                                  │  응용 계층 (Application Layer)                   │ │
    │                                  │  • OSI 7, 6, 5계층 기능 통합                      │ │
    │                                  │  • HTTP, FTP, SMTP, DNS, ...                     │ │
    │                                  └───────────────────────────────────────────────────┘ │
    │                                                                                         │
    │  4. 전송 계층 (Transport Layer)                                                         │
    │     • 종단 간(End-to-End) 신뢰성 있는 데이터 전송                                      │
    │     • TCP, UDP, SCTP                                                                   │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │                                                                                         │
    │                                  ┌───────────────────────────────────────────────────┐ │
    │                                  │  전송 계층 (Transport Layer)                     │ │
    │                                  │  • TCP, UDP                                      │ │
    │                                  └───────────────────────────────────────────────────┘ │
    │                                                                                         │
    │  3. 네트워크 계층 (Network Layer)                                                       │
    │     • 라우팅(Routing), 패킷 포워딩                                                      │
    │     • IP, ICMP, IGMP, ARP                                                             │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │                                                                                         │
    │                                  ┌───────────────────────────────────────────────────┐ │
    │                                  │  인터넷 계층 (Internet Layer)                     │ │
    │                                  │  • IP, ICMP, ARP                                 │ │
    │                                  └───────────────────────────────────────────────────┘ │
    │                                                                                         │
    │  2. 데이터링크 계층 (Data Link Layer)                                                   │
    │     • 프레임 전송, 오류 제어, 흐름 제어                                                  │
    │     • Ethernet, PPP, HDLC, Frame Relay                                                │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │                                                                                         │
    │  1. 물리 계층 (Physical Layer)                                                          │
    │     • 비트 전송, 전기적, 기계적, 기능적 특성                                              │
    │     • Ethernet Cable, Fiber, Radio                                                     │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │                                                                                         │
    │                                  ┌───────────────────────────────────────────────────┐ │
    │                                  │  네트워크 접근 계층 (Network Access Layer)        │ │
    │                                  │  • OSI 2, 1계층 기능 통합                         │ │
    │                                  │  • Ethernet, Wi-Fi, ...                          │ │
    │                                  └───────────────────────────────────────────────────┘ │
    └─────────────────────────────────────────────────────────────────────────────────────────┘
```

### 캡슐화(Encapsulation) 과정

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         데이터 캡슐화 (Encapsulation)                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    송신 측 (Sender)

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  [응용 계층] 데이터 생성                                                                │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  Data (애플리케이션 데이터: HTTP 요청, 이메일 내용 등)                              │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │                              │                                                          │
    │                              ▼                                                          │
    │  [전송 계층] TCP 헤더 추가                                                              │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │ TCP Header │ Data                                                                 │  │
    │  │ • Src Port, Dst Port                                                               │  │
    │  │ • Sequence Number                                                                  │  │
    │  │ • Acknowledgment Number                                                            │  │
    │  │ • Checksum                                                                        │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │    = 세그먼트 (Segment)                                                                │
    │                              │                                                          │
    │                              ▼                                                          │
    │  [네트워크 계층] IP 헤더 추가                                                           │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │ IP Header │ TCP Header │ Data                                                     │  │
    │  │ • Src IP, Dst IP                                                                  │  │
    │  │ • TTL, Protocol                                                                    │  │
    │  │ • Checksum                                                                        │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │    = 패킷 (Packet)                                                                     │
    │                              │                                                          │
    │                              ▼                                                          │
    │  [데이터링크 계층] Ethernet 프레임 헤더/트레일러 추가                                   │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │ Ethernet │ IP Header │ TCP │ Data │ FCS                                         │  │
    │  │ Header   │            │ Header│       │ (Frame Check Sequence)                   │  │
    │  │ • Src MAC, Dst MAC                                                               │  │
    │  │ • Type                                                                             │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │    = 프레임 (Frame)                                                                    │
    │                              │                                                          │
    │                              ▼                                                          │
    │  [물리 계층] 비트 스트림으로 변환                                                        │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │
    │  │ 0101010101010101010101010101010101010101010101010101010101010101...                   │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    │    = 비트 (Bits)                                                                       │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    수신 측 (Receiver)

    역캡슐화(Decapsulation): 위 과정의 역순으로 헤더 제거
```

### 계층별 장비와 프로토콜

| 계층 | 장비 | 프로토콜 | 데이터 단위 |
|------|------|----------|-----------|
| **7(응용)** | 게이트웨이, PC | HTTP, FTP, SMTP, DNS | Data |
| **6(표현)** | 게이트웨이 | SSL/TLS, JPEG | Data |
| **5(세션)** | 게이트웨이 | NetBIOS, RPC | Data |
| **4(전송)** | L4 스위치 | TCP, UDP | Segment |
| **3(네트워크)** | 라우터(L3) | IP, ICMP | Packet |
| **2(데이터링크)** | 스위치(L2), 브리지 | Ethernet, PPP | Frame |
| **1(물리)** | 허브, 리피터, 케이블 | RS-232, V.35 | Bit |

## Ⅲ. 융합 비교 및 다각도 분석

### OSI vs TCP/IP 비교

| 구분 | OSI 7계층 | TCP/IP 4계층 |
|------|----------|--------------|
| **표준화** | ISO 국제 표준 | IETF 실용 표준 |
| **계층 수** | 7개 (명확한 분리) | 4개 (통합) |
| **구현** | 복잡하여 드놈 | 간단하여 보편 |
| **속도** | 계층 통과 오버헤드 | 상대적으로 빠름 |
| **적용** | 교육용, 이론적 | 인터넷 실제 표준 |

### 과목 융합 관점 분석

#### 1. 보안 ↔ 네트워크 구조
- **SSL/TLS**: 전송 계층 보안 (5-6계층)
- **IPsec**: 네트워크 계층 보안 (3계층)
- **MACsec**: 데이터링크 계층 보안 (2계층)

#### 2. 클라우드 ↔ 네트워크 구조
- **VPC**: 가상 네트워크 2-3계층
- **Load Balancer**: L4/L7 스위치
- **CDN**: 응용 계층 캐싱

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 기업 네트워크 설계
**상황**: 본사-지사 간 안전한 통신 필요
**판단**:
1. **L3계층**: 라우터로 지사 간 연결
   - OSPF 라우팅 프로토콜
   - VPN (IPsec) 보안
2. **L2계층**: 스위치로 사내 LAN
   - VLAN으로 부서 분리
   - Spanning Tree로 루프 방지
3. **L4계층**: L7 스위치로 서버 로드 밸런싱

## Ⅴ. 기대효과 및 결론

### 계층형 아키텍처 기대 효과

| 장점 | 설명 |
|------|------|
| **모듈화** | 각 계층 독립적 개발/수정 가능 |
| **상호 운용성** | 표준화된 인터페이스로 장비 간 통신 |
| **복잡성 감소** | 계층별 관심사 분리 |
| **재사용성** | 상위 계층은 하위 계층 변경 무관 |

### 미래 전망

1. **SDN**: 제어평면과 데이터평면 분리
2. **NFV**: 네트워크 기능 가상화
3. **6G**: 계층 간 통합 네트워크

### ※ 참고 표준/가이드
- **ISO/IEC 7498**: OSI 7계층 모델
- **RFC 791, 793**: IP, TCP
- **IEEE 802.3**: Ethernet
