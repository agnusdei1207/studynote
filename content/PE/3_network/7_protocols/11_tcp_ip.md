# TCP/IP 프로토콜 (TCP/IP Protocol Suite)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TCP/IP는 **"인터넷의 **통신 프로토콜 슈트**"로, **TCP(Transmission Control Protocol)**가 **전송 계층(L4)**에서 **신뢰성 있는 데이터 전송**을 담당하고 **IP(Internet Protocol)**가 **네트워크 계층(L3)**에서 **패킷 라우팅**을 담당하며 **4계층 모델**로 구성된다.
> 2. **가치**: **신뢰성(Reliability)**, **상호 연결성(Interoperability)**, **확장성(Scalability)**로 **서로 다른 하드웨어와 소프트웨어** 간 **통신**을 가능하게 하며 **End-to-End 원칙**으로 **네트워크 중계 장비**의 복잡도를 **단순화**한다.
> 3. **융합**: **OSI 7계층**을 간소화한 **4계층 모델**로 **Application → Transport → Internet → Link** 계층이 **FTP, HTTP, SMTP** 등 **상위 프로토콜**의 기반이 되며 **IPv6**로 확장되고 **QUIC**, **HTTP/3**로 진화하고 있다.

## Ⅰ. 개요 (Context & Background)

### 개념
TCP/IP는 **"인터넷 표준 프로토콜"**으로, **TCP**와 **IP**를 핵심으로 하는 **프로토콜 모음**이다.

**TCP/IP의 구성**:
- **TCP**: 신뢰성 있는 전송
- **IP**: 패킷 전달
- **UDP**: 비연결형 전송
- **ICMP**: 제어 메시지
- **ARP**: 주소 변환

### 💡 비유
TCP/IP는 **"우편 시스템"**과 같다.
- **TCP**: 등기/내용 확인
- **IP**: 주소/배송

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TCP/IP의 발전                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

ARPA (1969년):
    • ARPANET 개발
    • NCP 프로토콜 사용
         ↓
TCP/IP 개발 (1970년대):
    • Vint Cerf, Bob Kahn
    • TCP(1974), IP(1981) 분리
    • RFC 791, 793
         ↓
인터넷 확산 (1980~90년대):
    • UNIX 4.2BSD 탑재
    • NFS, DNS, HTTP 추가
         ↓
현대:
    • IPv6, QUIC
    • HTTP/3
```

### 과목 융합 관점 분석

#### 1. 애플리케이션 ↔ TCP/IP
- **HTTP**: Web (TCP 80)
- **HTTPS**: Secure Web (TCP 443)
- **FTP**: File Transfer (TCP 20/21)

#### 2. 소켓 프로그래밍 ↔ TCP/IP
- **Socket API**: Application과 TCP/IP 간 인터페이스
- **bind(), connect(), listen(), accept()**

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: TCP 성능 최적화
**상황**: 대용량 파일 전송
**판단**:

```bash
# TCP Tuning Parameters

# 1. Window Size 증가 (Linux)
sysctl -w net.core.rmem_max=134217728
sysctl -w net.core.wmem_max=134217728
sysctl -w net.ipv4.tcp_rmem="4096 87380 67108864"
sysctl -w net.ipv4.tcp_wmem="4096 65536 67108864"

# 2. Congestion Control
sysctl -w net.ipv4.tcp_congestion_control=bbr  # Google BBR

# 3. Keepalive
sysctl -w net.ipv4.tcp_keepalive_time=600
sysctl -w net.ipv4.tcp_keepalive_intvl=30
sysctl -w net.ipv4.tcp_keepalive_probes=3

# 4. Fast Open
sysctl -w net.ipv4.tcp_fastopen=3

# nginx.conf 예시
http {
    sendfile on;              # Zero-copy
    tcp_nopush on;            # Packet aggregation
    tcp_nodelay on;           # Nagle algorithm disable
    keepalive_timeout 65;      # HTTP keepalive
}
```

## Ⅴ. 기대효과 및 결론

### TCP/IP 기대 효과

| 효과 | TCP/IP 없을 시 | TCP/IP 있을 시 |
|------|--------------|---------------|
| **상호 연결** | 불가능 | 범용 |
| **신뢰성** | 낮음 | 높음 |
| **확장성** | 제한적 | 무한 |
| **표준화** | 없음 | RFC |

### 미래 전망

1. **QUIC**: UDP 기반 신뢰성
2. **HTTP/3**: QUIC 위 HTTP
3. **IPv6**: IPv4 고갈 해결

### ※ 참고 표준/가이드
- **RFC 791**: IP
- **RFC 793**: TCP
- **RFC 2616**: HTTP/1.1
