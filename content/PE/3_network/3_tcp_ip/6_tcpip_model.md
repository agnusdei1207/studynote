# TCP/IP 모델 (TCP/IP Model)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TCP/IP 모델은 **"인터넷(Internet)의 실제 표준 프로토콜 스위트"**로, **OSI 7계층의 단순화된 4계층(애플리케이션, 전송, 인터넷, 네트워크 접근)**으로 구성되며, **TCP(신뢰성)**와 **IP(라우팅)**가 핵심 프로토콜이다.
> 2. **가치**: **DoD(미 국방부)**의 후원으로 **인터넷의 기반이** 되었으며, **이기종 시스템 간 상호 운용성(Interoperability)**을 **실용적으로** 보장하고 **OSI 모델**보다 **간단하고 빠르게** 구현되었다.
> 3. **융합**: **World Wide Web(HTTP)**, **이메일(SMTP, POP3, IMAP)**, **파일 전송(FTP)**의 응용 계층을 지원하며, **IP 주소**, **서브넷 마스크**, **라우팅 테이블**의 **라우터(Router)** 동작, **TCP 3-way Handshake**의 **신뢰성 있는 연결**을 실현한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
TCP/IP(Transmission Control Protocol/Internet Protocol)는 **"인터넷에서 컴퓨터들이 서로 통신하기 위한 프로토콜 모음"**이다.

**TCP/IP의 핵심 구성**:
- **TCP**: 신뢰성 있는 바이트 스트림 전송
- **IP**: 패킷 라우팅과 전달
- **UDP**: 비신뢰성 있는 데이터그램 전송
- **ICMP**: 진단 및 오류 보고

### 💡 비유
TCP/IP는 **"우편 시스템"**과 같다.
- **IP**: 주소 기반 배송 (라우팅)
- **TCP**: 등기 우편 (신뢰성)
- **UDP**: 일반 우편 (빠르지만 분실 가능)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TCP/IP의 역사                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

ARPANET (1969년):
    • DoD(미 국방부) 후원
    • 패킷 교환 네트워크
    • NCP(Network Control Protocol) 사용
         ↓
TCP/IP 개발 (1973~1974년):
    • Vinton Cerf, Robert Kahn
    • "A Protocol for Packet Network Interconnection"
    • TCP와 IP 분리
         ↓
BSD Unix 채택 (1983년):
    • BSD 4.2에 TCP/IP 내장
    • 대학, 연구소 보급
    • 인터넷 확산 시작
         ↓
WWW 발명 (1989년):
    • HTTP(Hypertext Transfer Protocol)
    • TCP/IP 위에서 동작
    • 인터넷 대중화
         ↓
현재 인터넷 표준:
    • OSI 7계층: 이론적 모델
    • TCP/IP: 실제 표준
```

### TCP vs UDP

| 특징 | TCP | UDP |
|------|-----|-----|
| **연결** | Connection-oriented | Connectionless |
| **신뢰성** | 높음 (ACK, 재전송) | 낮음 |
| **순서** | 보장 | 불보장 |
| **속도** | 느림 (오버헤드) | 빠름 |
| **사용** | 웹, 이메일 | 스트리밍, 게임 |

## Ⅲ. 융합 비교 및 다각도 분석

### TCP/IP vs OSI 모델

| 구분 | OSI 7계층 | TCP/IP 4계층 |
|------|----------|--------------|
| **표준화** | ISO 국제 표준 | IETF 실무 표준 |
| **계층 수** | 7개 (명확) | 4개 (통합) |
| **구현** | 복잡 | 간단 |
| **채택** | 이론적 | 실제 인터넷 |

### 과목 융합 관점 분석

#### 1. 보안 ↔ TCP/IP
- **TLS/SSL**: 전송 계층 보안
- **IPsec**: 네트워크 계층 보안
- **HTTPS**: 응용 계층 보안

#### 2. 클라우드 ↔ TCP/IP
- **VPC**: 가상 네트워크 (L3)
- **Subnet**: IP 주소 대역 (L3)
- **Security Group**: 포트 기반 방화벽 (L4)

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 웹 서버 네트워크 설계
**상황**: 공개 웹 서비스, 방화벽 설정
**판단**:
1. **IP 주소**: 공인 IP (예: 203.0.113.10)
2. **서브넷**: /24 (254개 호스트)
3. **방화벽**:
   - 허용: TCP 80 (HTTP), 443 (HTTPS)
   - 차단: TCP 22 (SSH), 3306 (MySQL)
4. **TCP Keepalive**: 연결 유지

```bash
# iptables 규칙
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
iptables -A INPUT -p tcp --dport 22 -s 203.0.113.0/24 -j ACCEPT  # 사무실 IP만
iptables -A INPUT -j DROP
```

## Ⅴ. 기대효과 및 결론

### TCP/IP 모델 기대 효과

| 효과 | 기대 결과 |
|------|----------|
| **상호 운용성** | 다양한 기기 통신 |
| **확장성** | 새로운 프로토콜 추가 용이 |
| **신뢰성** | TCP로 데이터 보장 |
| **단순성** | 4계층으로 구현 용이 |

### 미래 전망

1. **IPv6**: 128비트 주소, NAT 불필요
2. **QUIC**: UDP 기반 HTTP/3
3. **Segment Routing**: SDN 기반 라우팅

### ※ 참고 표준/가이드
- **RFC 791**: Internet Protocol (IP)
- **RFC 793**: Transmission Control Protocol (TCP)
- **RFC 791**: IP Datagrams
