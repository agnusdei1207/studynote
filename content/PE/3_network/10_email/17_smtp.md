# SMTP (Simple Mail Transfer Protocol)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SMTP는 **"이메일 **전송을 위한 **텍스트 기반 **프로토콜"**으로, **TCP 25포트**를 사용하고 **Sender**(보내는 사람)가 **SMTP Server**에 **연결**하여 **Mail Transaction**(HELO → MAIL FROM → RCPT TO → DATA)을 **수행**한다.
> 2. **가치**: **전 세계 **이메일 **전송의 **표준**이며 **Store-and-Forward** 방식으로 **신뢰성**을 높이고 **ESMTP**(Extension SMTP)로 **인증**(AUTH), **암호화**(STARTTLS), **메일링**(MAIL FROM=NULL)을 **지원**한다.
> 3. **융합**: **DNS MX**(Mail Exchange) 레코드로 **수신 서버**를 **찾고 **MIME**(Multipurpose Internet Mail Extensions)로 **첨부 파일**을 **전송**하며 **IMAP**(143/993), **POP3**(110/995)와 함께 **이메일 시스템**을 **구성**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
SMTP는 **"이메일 전송 프로토콜"**이다.

**SMTP 특징**:
- **Push**: 보내는 사람이 시작
- **Text-based**: 사람이 읽기 쉬움
- **Store-and-Forward**: 중계 서버 저장

### 💡 비유
SMTP는 **"우편 집배원"**과 같다.
- **편지**: 이메일
- **집배원**: SMTP
- **우편함**: POP3/IMAP

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         이메일 프로토콜 발전                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

RFC 821 (1982):
    • SMTP 정의
    • 단순 텍스트 전송
         ↓
MIME (1992):
    • 첨부 파일
    • 멀티미디어
         ↓
ESMTP (1995):
    • 인증
    • STARTTLS
         ↓
현대:
    • SPF, DKIM, DMARC
    • 보안 강화
```

## Ⅱ. 아키텍처 및 핵심 원리

### SMTP 전송 과정

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         SMTP Mail Transaction                                           │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Client (MUA)                SMTP Server                  Mail Server                │
    │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │
    │  │  1. Connection                                                                      │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  220 mail.example.com ESMTP Postfix Ready                                      │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                  │  │
    │  │  2. HELO/EHLO (Handshake)                                                           │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  EHLO client.example.com                                                          │  │  │
    │  │  │  250-mail.example.com Hello client.example.com                                  │  │  │
    │  │  │  250-SIZE 10485760                                                               │  │  │
    │  │  │  250-8BITMIME                                                                    │  │  │
    │  │  │  250-STARTTLS                                                                   │  │  │
    │  │  │  250 HELP                                                                        │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                  │  │
    │  │  3. MAIL FROM (보내는 사람)                                                         │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  MAIL FROM:<sender@example.com>                                                  │  │  │
    │  │  │  250 2.1.0 Ok                                                                    │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                  │  │
    │  │  4. RCPT TO (받는 사람)                                                             │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  RCPT TO:<recipient@example.com>                                                │  │  │
    │  │  │  250 2.1.5 Ok                                                                    │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                  │  │
    │  │  5. DATA (본문 전송)                                                                 │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  DATA                                                                              │  │  │
    │  │  │  354 End data with <CRLF>.<CRLF>                                                   │  │  │
    │  │  │  Subject: Test Email                                                              │  │  │
    │  │  │  From: sender@example.com                                                         │  │  │
    │  │  │  To: recipient@example.com                                                       │  │  │
    │  │  │                                                                                  │  │  │
    │  │  │  This is a test email.                                                            │  │  │
    │  │  │  .                                                                                │  │  │
    │  │  │  250 2.0.0 Ok: queued as ABC123                                                   │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  │                                                                                  │  │
    │  │  6. QUIT                                                                             │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  QUIT                                                                              │  │  │
    │  │  │  221 2.0.0 Bye                                                                    │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────┘  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### SMTP vs POP3 vs IMAP

| 구분 | SMTP | POP3 | IMAP |
|------|------|------|------|
| **용도** | 전송 | 수신 | 수신/동기화 |
| **Port** | 25, 587 | 110, 995 | 143, 993 |
| **저장** | Server | Client | Server |
| **폴더** | - | Inbox | 다중 |

### 이메일 보안

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         이메일 보안                                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [SPF (Sender Policy Framework)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 발신 서버 인증                                                                      │
    │  │  • DNS TXT 레코드에 허용 IP 목록                                                    │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  example.com.  IN  TXT  "v=spf1 ip4:192.0.2.0/24 include:_spf.google.com ~all"    │  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [DKIM (DomainKeys Identified Mail)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 디지털 서명                                                                          │
    │  │  • 위조 방지                                                                          │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [DMARC (Domain-based Message Authentication)]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • SPF + DKIM + 정책                                                                     │
    │  │  • 실패 시 처리 방법 지정                                                             │
    │  │                                                                                      │
    │  ┌────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  example.com.  IN  TXT  "v=DMARC1; p=reject; rua=mailto:dmarc@example.com"       │  │  │
    │  └────────────────────────────────────────────────────────────────────────────────────┘  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 메일 서버 구축
**상황**: SMTP 서버 설정
**판단**:

```bash
# Postfix 설정 (Ubuntu)
$ sudo apt install postfix

# /etc/postfix/main.cf
myhostname = mail.example.com
mydomain = example.com
myorigin = $mydomain
inet_interfaces = all
inet_protocols = ipv4
mynetworks = 127.0.0.0/8 [::ffff:127.0.0.0]/104 [::1]/128
home_mailbox = Maildir/

# SMTP 인증 (SASL)
smtpd_sasl_auth_enable = yes
smtpd_tls_auth_only = yes

# TLS 암호화
smtpd_tls_cert_file = /etc/ssl/certs/smtpd.crt
smtpd_tls_key_file = /etc/ssl/private/smtpd.key
smtpd_use_tls = yes

# 릴레이 제어
smtpd_relay_restrictions =
    permit_mynetworks
    permit_sasl_authenticated
    defer_unauth_destination

# 서비스 재시작
$ sudo systemctl restart postfix

# 테스트
$ telnet localhost 25
$ echo "TEST" | mail -s "Test Email" user@example.com
```
