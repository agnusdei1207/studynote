# POP3 (Post Office Protocol v3)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: POP3는 **"이메일 **수신을 위한 **프로토콜"**로, **TCP 110포트**(일반) 또는 **995포트**(SSL/TLS)를 사용하고 **Client**가 **Server**에 **연결**하여 **인증** 후 **메일을 **다운로드**하고 **서버에서 **삭제**하는 **Store-and-Forward** 방식이다.
> 2. **가치**: **오프라인 메일 확인**이 **가능**하고 **서버 저장 공간**을 **절약**하지만 **IMAP**와 달리 **폴더 동기화**가 **불가능**하고 **여러 장치**에서 **메일 관리**가 **어렵다**.
> 3. **융합**: **POP3 Commands**(USER, PASS, LIST, RETR, DELE)로 **동작**하며 **APOP**(MD5 인증), **TOP**(메일 헤더만), **UIDL**(Unique ID)로 **확장**되고 **IMAP**과 **경쟁**하며 **현대**는 **IMAP**이 **주류**이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
POP3는 **"이메일을 클라이언트로 가져오는 프로토콜"**이다.

**POP3 특징**:
- **Download**: 서버 → 클라이언트
- **Delete**: 가져온 후 서버에서 삭제
- **Offline**: 오프라인 확인 가능

### 💡 비유
POP3는 **"우편물 수령****와 같다.
- **POP3**: 집으로 가져옴 (원본 없어짐)
- **IMAP**: 우편함에서 읽기 (원본 유지)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         이메일 수신 프로토콜 발전                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

POP1 (1984):
    • 기본 기능
    • 간단하지만 불완전
         ↓
POP2 (1985):
    • 인증 추가
    └─> 표준화 실패
         ↓
POP3 (1988, RFC 1939):
    • 표준화
    • 널리 보급
         ↓
IMAP (1994):
    • 서버 관리
    • 다중 장치
```

    │  │  │  [Email Content]                                                                   │  │  │

### POP3 명령어

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         POP3 Commands                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  명령어    │  인증 전 │  인증 후 │  설명                                           │
    │  ─────────────────────────────────────────────────────────────────────────────────────  │
    │  USER      │    O    │    X     │  사용자 이름                                     │
    │  PASS      │    O    │    X     │  비밀번호                                        │
    │  STAT      │    O    │    O     │  메일 개수, 총 크기                               │
    │  LIST      │    X    │    O     │  메일 목록                                       │
    │  RETR      │    X    │    O     │  메일 가져오기                                    │
    │  DELE      │    X    │    O     │  메일 삭제 표시                                   │
    │  RSET      │    X    │    O     │  삭제 표시 해제                                   │
    │  QUIT      │    O    │    O     │  세션 종료                                       │
    │  TOP       │    X    │    O     │  메일 헤더 n줄                                    │
    │  UIDL      │    X    │    O     │  Unique ID 목록                                   │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### POP3 vs IMAP

| 구분 | POP3 | IMAP |
|------|------|------|
| **저장** | Client | Server |
| **동기화** | 불가 | 가능 |
| **폴더** | Inbox만 | 다중 |
| **속도** | 빠름 | 느림 |

### POP3 모드

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         POP3 Modes                                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Delete Mode]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • RETR 후 서버에서 삭제                                                                │
    │  │  • 한 장치에서만 확인 가능                                                           │
    │  │  • 서버 저장 공간 절약                                                               │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Keep Mode]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • RETR 후 서버 유지                                                                  │
    │  │  • DELE 명령으로 명시적 삭제                                                           │
    │  │  • 여러 장치에서 확인                                                               │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: POP3 서버 설정
**상황**: Dovecot POP3
**판단**:

```conf
# /etc/dovecot/conf.d/10-pop3.conf
service pop3 {
  inet_listener pop3 {
    port = 110
  }
  inet_listener pop3s {
    port = 995
    ssl = yes
  }
}

# POP3만 허용
protocols = pop3

# 인증
auth_mechanisms = plain login

# SSL/TLS
ssl = required
ssl_cert = </etc/ssl/certs/dovecot.pem
ssl_key = </etc/ssl/private/dovecot.key
```
