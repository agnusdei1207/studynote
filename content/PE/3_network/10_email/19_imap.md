# IMAP (Internet Message Access Protocol)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IMAP는 **"이메일을 **서버에서 **관리하고 **클라이언트가 **접근하는 **프로토콜"**로, **TCP 143포트** 또는 **993포트**(SSL/TLS)를 사용하고 **Server-side**에 **메일을 **저장**하며 **Folder**(Inbox, Sent, Trash) **동기화**를 **지원**하고 **Multiple Client** 접속이 **가능**하다.
> 2. **가치**: **여러 장치**에서 **메일 확인**이 **가능**하고 **Server-side** **검색**/**필터링**을 **지원**하며 **Partial Download**로 **대역폭 데이터**를 **효율적**으로 **처리**할 수 있다.
> 3. **융합**: **IMAP4**(RFC 3501)가 **현재 표준**이며 **IDLE**(Push) **Extension**으로 **실시간 **알림**을 **지원**하고 **Microsoft Exchange**, **Google Gmail**, **Dovecot**, **Courier**가 **구현**을 **제공**한다.

## Ⅰ. 개요 (Context & Background)

### 개념
IMAP는 **"이메일을 서버에서 관리하는 프로토콜"**이다.

**IMAP 특징**:
- **Server-Side**: 서버 저장
- **Folder Sync**: 폴더 동기화
- **Multiple Client**: 다중 접속
- **Stateful**: 상태 유지

### 💡 비유
IMAP는 **"우편함****과 같다.
- **POP3**: 집으로 가져옴 (원본 없어짐)
- **IMAP**: 우편함에서 읽기 (원본 유지)

### 등장 배경

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         이메일 수신 프로토콜 발전                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

POP3 (1988):
    • 다운로드 후 삭제
    • 단일 장치
         ↓
IMAP2 (1994):
    • 폴더 지원
    • 다중 장치
         ↓
IMAP4 (2003, RFC 3501):
    • 현대 표준
    • IDLE Push
```

## Ⅲ. 융합 비교 및 다각도 분석

### IMAP vs POP3

| 구분 | IMAP | POP3 |
|------|------|------|
| **저장** | Server | Client |
| **폴더** | 다중 | Inbox만 |
| **동기화** | O | X |
| **공간** | 서버 필요 | 클라이언트 |

### IDLE (Push)

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: IMAP 서버 설정
**상황**: Dovecot IMAP
**판단**:

```conf
# /etc/dovecot/conf.d/20-imap.conf
service imap {
  inet_listener imap {
    port = 143
  }
  inet_listener imaps {
    port = 993
    ssl = yes
  }
}

protocol imap {
  # Maildir format
  mail_location = maildir:~/Maildir

  # IMAP capabilities
  mail_plugins = $mail_plugins autocreate
}

# SSL/TLS
ssl = required
ssl_cert = </etc/ssl/certs/dovecot.pem>
ssl_key = </etc/ssl/private/dovecot.key>
```

## Ⅴ. 기대효과 및 결론

### IMAP 기대 효과

| 효과 | POP3 | IMAP |
|------|------|------|
| **다중 장치** | 어려움 | 쉬움 |
| **폴더 관리** | 불가 | 가능 |
| **검색** | 클라이언트 | 서버 |
| **공간** | 절약 | 소모 |
