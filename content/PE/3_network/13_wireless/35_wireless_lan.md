# 무선 LAN (Wireless LAN)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 무선 LAN은 **"무선 **주파수**를 **사용**하여 **근거리 **네트워크**를 **구성**하는 **기술\"**로, **IEEE 802.11** **표준**에 **기반**하고 **WiFi**(Wireless Fidelity)**가 **대표적**이다.
> 2. **기술**: **CSMA/CA**(Carrier Sense Multiple Access/Collision Avoidance)**로 **충돌**을 **방지**하고 **OFDM**(Orthogonal Frequency Division Multiplexing), **MIMO**(Multiple Input Multiple Output)**로 **전송 **속도**를 **향상**시킨다.
> 3. **보안**: **WEP**(약함, **WPA**(TKIP), **WPA2**(AES-CCMP), **WPA3**(SAE, **Opportunistic Wireless Encryption)**로 **보안**이 **강화**되고 **Enterprise **모드**(802.1X, **RADIUS)**로 **인증**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
무선 LAN은 **"무선 근거리 통신망"**이다.

**유선 vs 무선**:
| 구분 | 유선 (Ethernet) | 무선 (WiFi) |
|------|-----------------|-------------|
| **매체** | 케이블 | 전파 |
| **CSMA** | CD (Collision Detection) | CA (Collision Avoidance) |
| **속도** | 1-400Gbps | 0.1-10Gbps |
| **이동성** | X | O |

### 💡 비유
무선 LAN은 ****무전 **통신 ****과 같다.
- **채널**: 주파수
- **충돌**: 동시 전송
- **암호**: 보안

### CSMA/CA (Collision Avoidance)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         CSMA/CA with RTS/CTS                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Problem: Cannot detect collisions in wireless (hidden terminal problem)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Station A              Station B              Station C                                 │  │
    │  ┌──────────┐           ┌──────────┐           ┌──────────┐                            │  │
    │  │          │◄──────────│          │──────────►│          │                            │  │
    │  │  (Hidden)│           │  (AP)    │           │  (Hidden)│                            │  │
    │  └──────────┘           └──────────┘           └──────────┘                            │  │
    │                            ▲                                                             │  │
    │                            │                                                             │  │
    │                    A and C can't hear each other                                        │  │
    │                    → Both transmit to B → Collision at B                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Solution: RTS/CTS (Request to Send/Clear to Send)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Station A                              AP (Access Point)                                │  │
    │  ┌───────────────────────────────────┐ ┌──────────────────────────────────────────────┐  │  │
    │  │  1. RTS (Request to Send)         │                                                 │  │  │
    │  │     Duration: 100ms                │                                                 │  │  │
    │  │  ──────────────────────────────────────────────────────────────────────────────────────>  │  │
    │  │                                   │  2. CTS (Clear to Send)                        │  │  │
    │  │  <──────────────────────────────────────────────────────────────────────────────────────  │  │
    │  │                                   │     Duration: 100ms (all hear this)               │  │  │
    │  │  3. Data transmission (100ms)     │                                                 │  │  │
    │  │  ──────────────────────────────────────────────────────────────────────────────────────>  │  │
    │  │                                   │  4. ACK                                       │  │  │
    │  │  <──────────────────────────────────────────────────────────────────────────────────────  │  │
    │  └───────────────────────────────────┘                                                 │  │  │
    │                                                                                            │  │
    │  → All stations hear CTS and defer transmission for 100ms                                 │  │
    │  → Hidden terminal problem solved                                                         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### MIMO (Multiple Input Multiple Output)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         MIMO Technology                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    SISO (Single Input Single Output) - Traditional:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Transmitter                            Receiver                                       │  │
    │  ┌──────────┐                        ┌──────────┐                                    │  │

    │  └──────────┘                        └──────────┘                                    │  │
    │  → 1 spatial stream                                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    MIMO 2×2:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Transmitter                            Receiver                                       │  │
    │  ┌──────────────────────────────────┐ ┌──────────────────────────────────────────────┐  │  │
    │  │ Ant1 ────────┐                   │                                                 │  │  │
    │  │              ├──────┐             │  Ant1 ────────┐                                │  │  │
    │  │ Ant2 ────────┤      │             │              │                                │  │  │
    │  │              │      │             │  Ant2 ────────┤                                │  │  │
    │  └──────────────┴──────┴─────────────┘              │                                │  │  │
    │                                                 └────────────────────────────────────┘  │  │
    │  → 2 spatial streams (2× capacity)                                                          │  │
    │  → Spatial multiplexing: Send different data on each antenna                             │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    MU-MIMO (Multi-User MIMO):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  AP (4×4)                               Stations                                        │  │
    │  ┌───────────────────────────────────┐                                                 │  │
    │  │ Ant1 ────────────────────────┐   ┌──────────┐  ┌──────────┐  ┌──────────┐           │  │
    │  │ Ant2 ──────────────────┐   │   │ Station1 │  │ Station2 │  │ Station3 │           │  │
    │  │ Ant3 ────────────┐   │   │   │ (2×2)    │  │ (2×2)    │  │ (2×2)    │           │  │
    │  │ Ant4 ────────┐   │   │   │   └──────────┘  └──────────┘  └──────────┘           │  │
    │  └──────────────┴───┴───┴───┴───┘                                                 │  │
    │                                                                                         │  │
    │  → AP serves multiple stations simultaneously                                          │  │
    │  → Increases total system capacity                                                      │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### WiFi 보안

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         WiFi Security Evolution                                            │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    WEP (Wired Equivalent Privacy) - Broken:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  RC4 stream cipher + 24-bit IV (Initialization Vector)                               │  │  │
    │  │  Key: 40-bit or 104-bit (shared key)                                                 │  │  │
    │  │                                                                                       │  │  │
    │  │  Vulnerabilities:                                                                     │  │  │
    │  │  • 24-bit IV: Only 16 million possibilities → reuse inevitable                        │  │  │
    │  │  • RC4 key recovery: Statistical attacks recover key in minutes                       │  │  │
    │  │  • No replay protection: Captured packets can be replayed                             │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  → DO NOT USE                                                                            │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    WPA2 (WiFi Protected Access 2):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Authentication: 4-Way Handshake                                                       │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  1. AP → STA: ANonce (random number)                                              │  │  │  │
    │  │  │  2. STA → AP: SNonce + MIC (Message Integrity Check)                              │  │  │  │
    │  │  │  3. AP → STA: GTK + MIC                                                           │  │  │  │
    │  │  │  4. STA → AP: ACK + MIC                                                            │  │  │  │
    │  │  │                                                                                     │  │  │  │
    │  │  │  PTK = PBKDF2(Passphrase, SSID, 4096 iterations)                                  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  Encryption: AES-CCMP (Counter Mode CBC-MAC Protocol)                                 │  │  │
    │  │  • 128-bit AES in CTR mode for encryption                                            │  │  │
    │  │  • CBC-MAC for integrity                                                              │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    WPA3 (Current Standard):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Authentication: SAE (Simultaneous Authentication of Equals)                          │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • Dragonfly Key Exchange (RFC 7664)                                             │  │  │  │
    │  │  │  • Resistant to offline dictionary attacks                                        │  │  │  │
    │  │  │  • Forward secrecy: Compromised password doesn't expose past sessions             │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  Encryption: GCMP-256 (Galois/Counter Mode Protocol)                                  │  │  │
    │  │  • 256-bit AES                                                                        │  │  │
    │  │  • 256-bit Galois field for MAC                                                       │  │  │
    │  │                                                                                       │  │  │
    │  │  OWE (Opportunistic Wireless Encryption):                                             │  │  │
    │  │  • Unencrypted AP → Encrypted connection (no password needed)                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 무선 LAN 기대 효과

| 대역 | 속도 | 거리 | 벽통과 |
|------|------|------|--------|
| **2.4GHz** | 최대 300Mbps | 길음 | 좋음 |
| **5GHz** | 최대 8.6Gbps | 중간 | 중간 |
| **6GHz** | 최대 11Gbps | 짧음 | 나쁨 |

### 모범 사례

1. **설치**: 천장 높이
2. **채널**: 중복 최소화
3. **보안**: WPA3-Enterprise
4. **모니터링**: Heat map

### 미래 전망

1. **WiFi 7**: 320MHz, 40Gbps
2. **6GHz**: 새로운 대역
3. **WiFi 6E**: 6GHz 지원
4. **Li-Fi**: Visible light
