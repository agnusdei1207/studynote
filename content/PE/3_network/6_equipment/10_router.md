# 라우터 (Router)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 라우터는 **"네트워크 계층(Layer 3)**에서 **패킷의 경로를 결정**하고 **전달(Forwarding)**하는 **네트워크 장비"**로, **Routing Table**을 참조하여 **Next Hop**을 결정하고 **서로 다른 네트워크** 간 **통신**을 중계한다.
> 2. **가치**: **Broadcast Domain 분리**, **Collision Domain 분리**, **NAT(Network Address Translation)**으로 **IP 주소 절약**, **ACL(Access Control List)**로 **보안 정책** 적용, **QoS(Quality of Service)**로 **트래픽 우선순위** 제어가 가능하다.
> 3. **융합**: **Home Router**(공유기)는 **Router + Switch + Wireless AP**가 통합된 형태이며, **Core Router**(백본), **Edge Router**(접속), **CSR(Cisco Virtual Router)**, **AWS Transit Gateway** 등 다양한 형태로 존재한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
라우터는 **"서로 다른 네트워크 간 패킷을 전달하는 L3 장비"**이다.

**라우터의 핵심 기능**:
- **Routing**: 최적 경로 결정
- **Forwarding**: 패킷 전송
- **Filtering**: 보안 정책 적용

### 💡 비유
라우터는 **"우편물 분류 센터"**와 같다.
- **패킷**: 우편물
- **IP 주소**: 주소
- **Routing Table**: 배송 경로

### 라우터 동작 과정

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         패킷 포워딩 과정                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    [Step 1: L2 수신]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Ethernet Frame 수신                                                                   │
    │  • Destination MAC: Router Interface MAC                                                │
    │  • L2 헤더 제거                                                                         │
    │  • L3 패킷 추출                                                                         │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Step 2: L3 Lookup]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Destination IP 확인                                                                     │
    │  • FIB(Routing Table) 조회                                                              │
    │  • Longest Prefix Match                                                                │
    │  • Next Hop 결정                                                                         │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Step 3: TTL 감소]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  TTL(Time To Live) 감소                                                                 │
    │  • TTL = 0이면 패킷 폐기                                                                │
    │  • ICMP Time Exceeded 전송                                                              │
    │  • Loop 방지                                                                             │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Step 4: L2 재작성]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  새로운 L2 헤더 생성                                                                     │
    │  • Destination MAC: Next Hop MAC                                                        │
    │  • Source MAC: Outgoing Interface MAC                                                    │
    │  • CRC 재계산                                                                            │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    [Step 5: 전송]

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Outgoing Interface로 전송                                                               │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Router vs Switch

| 구분 | Router (L3) | Switch (L2) |
|------|------------|------------|
| **계층** | L3 (Network) | L2 (Data Link) |
| **주소** | IP Address | MAC Address |
| **Forwarding** | Routing Table | MAC Table |
| **Broadcast** | 차단 | 전송 |
| **Collision Domain** | 분리 | 분리 |
| **Broadcast Domain** | 분리 | 전체 |

### 과목 융합 관점 분석

#### 1. 클라우드 ↔ 라우터
- **AWS Transit Gateway**: VPC 간 라우팅
- **Azure Route Server**: 경로 제어
- **Cloud Router**: VPN, Interconnect

#### 2. SDN ↔ 라우터
- **OpenFlow**: Centralized Routing
- **Segment Routing**: Source-based
- **NFV**: Virtual Router

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 기업 네트워크 라우터 구성
**상황**: 본사-지사 연결
**판단**:

```bash
# Cisco IOS Configuration

# Router 1 (본사)
interface GigabitEthernet0/0
 ip address 203.0.113.1 255.255.255.252
 no shutdown

interface GigabitEthernet0/1
 ip address 192.168.1.1 255.255.255.0
 no shutdown

# 라우팅
router ospf 1
 network 192.168.1.0 0.0.0.255 area 0
 network 203.0.113.0 0.0.0.3 area 0
 default-information originate

# NAT
ip nat inside source list 1 interface GigabitEthernet0/0 overload
access-list 1 permit 192.168.1.0 0.0.0.255

# Router 2 (지사)
# 유사하게 구성
```

## Ⅴ. 기대효과 및 결론

### 라우터 기대 효과

| 효과 | Router 없을 시 | Router 있을 시 |
|------|-------------|-------------|
| **네트워크 분리** | Flat | 계층 구조 |
| **Broadcast** | 전체 전파 | 차단 |
| **라우팅** | 불가능 | 가능 |
| **보안** | 낮음 | ACL, NAT |

### 미래 전망

1. **Programmable Router**: NETCONF/YANG
2. **Segment Routing**: SRv6
3. **AI-based Routing**: 자동 최적화
