# 동기식 전송 vs 비동기식 전송

## 정의

**동기식 전송(Synchronous)**은 송수신측이 **공통의 클럭**으로 동기화하여 연속적인 데이터를 전송하고, **비동기식 전송(Asynchronous)**은 **시작/정지 비트**로 각 문자 단위의 동기화를 수행한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           동기식 vs 비동기식 전송                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   비동기식 전송 (Asynchronous):                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   [Start][D0][D1][...][D7][Parity][Stop]  유휴          │ │
│   │    1bit   8bits 데이터         1bit  1~2bits           │ │
│   │                                                         │ │
│   │   • 문자 단위 전송                                      │ │
│   │   • 시작/정지 비트로 프레이밍                           │ │
│   │   • 문자 간 간격 허용                                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   동기식 전송 (Synchronous):                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   [SYN][SYN][데이터][데이터]...[데이터][CRC][ETX]       │ │
│   │   동기문자     연속 데이터 스트림      오류검출 종료    │ │
│   │                                                         │ │
│   │   • 블록/프레임 단위 전송                               │ │
│   │   • 공통 클럭 또는 동기 문자 사용                       │ │
│   │   • 고속 대량 전송에 유리                               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 비동기식 전송 상세

```
┌─────────────────────────────────────────────────────────────────┐
│           비동기식 전송 프레임 구조                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   UART 프레임 구조:                                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌─────┬───┬───┬───┬───┬───┬───┬───┬───┬───┬─────┐  │ │
│   │   │Start│D₀ │D₁ │D₂ │D₃ │D₄ │D₅ │D₆ │D₇ │ P │Stop │  │ │
│   │   │  1  │   │   │   │   │   │   │   │   │   │1~2  │  │ │
│   │   └─────┴───┴───┴───┴───┴───┴───┴───┴───┴───┴─────┘  │ │
│   │     ↑                                             ↑   │ │
│   │     │                                             │   │ │
│   │   Low → High                                    High │ │
│   │   (1→0 전이로 시작 감지)                (High 유지) │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   각 비트 구성:                                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   Start Bit: 1비트 (0), 데이터 시작 신호                 │ │
│   │   Data Bits: 5~8비트 (보통 8비트)                        │ │
│   │   Parity:    0~1비트 (짝수/홀수/없음)                    │ │
│   │   Stop Bits: 1~2비트 (1), 데이터 종료 신호               │ │
│   │                                                         │ │
│   │   총 프레임 길이: 10~12비트 (8N1 = 10비트)              │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   전송 효율:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   8N1 설정 (8데이터, 패리티 없음, 1 정지비트):          │ │
│   │                                                         │ │
│   │   전송 효율 = 8 / 10 = 80%                              │ │
│   │                                                         │ │
│   │   8E2 설정 (8데이터, 짝수 패리티, 2 정지비트):          │ │
│   │                                                         │ │
│   │   전송 효율 = 8 / 12 = 66.7%                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 비동기식 전송 특성

```
┌─────────────────────────────────────────────────────────────────┐
│           비동기식 전송 장단점                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   장점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  • 구현 단순 (저렴한 하드웨어)                           │ │
│   │  • 클럭 동기화 불필요                                    │ │
│   │  • 문자 간 임의 간격 허용                                │ │
│   │  • 간헐적 전송에 적합                                    │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  • 오버헤드 큼 (Start/Stop 비트)                         │ │
│   │  • 전송 효율 낮음 (~80%)                                 │ │
│   │  • 고속 전송 부적합                                      │ │
│   │  • 클럭 드리프트에 취약                                  │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   클럭 드리프트 문제:                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   송신측과 수신측 클럭이 약간 다르면:                     │ │
│   │                                                         │ │
│   │   송신: │▔▔│__│▔▔│__│▔▔│__│▔▔│__│                     │ │
│   │          ↑                                               │ │
│   │   수신:  │▔▔│__│▔▔│__│▔▔│__│▔▔│__│  (샘플링 위치 어긋남)│ │
│   │           ↑                                              │ │
│   │                                                         │ │
│   │   해결: 수신측은 Start 비트 후 중앙에서 샘플링           │ │
│   │   (각 비트의 중간 지점에서 읽기)                        │ │
│   │                                                         │ │
│   │   허용 클럭 오차: 약 ±5%                                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   응용: RS-232, 키보드, 마우스, UART                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 동기식 전송 상세

```
┌─────────────────────────────────────────────────────────────────┐
│           동기식 전송 구조                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   문자 동기 방식 (Character-oriented, BSC):                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌──────┬──────┬──────┬────────┬──────┬──────┐        │ │
│   │   │ SYN  │ SYN  │ STX  │ 데이터 │ ETX  │ BCC  │        │ │
│   │   │ 0x16 │ 0x16 │ 0x02 │        │ 0x03 │      │        │ │
│   │   └──────┴──────┴──────┴────────┴──────┴──────┘        │ │
│   │    동기문자(2개)  시작  본문     종료  체크섬            │ │
│   │                                                         │ │
│   │   SYN: 동기 문자 (수신측 동기화)                         │ │
│   │   STX: 본문 시작 (Start of Text)                        │ │
│   │   ETX: 본문 종료 (End of Text)                          │ │
│   │   BCC: Block Check Character (오류 검출)                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   비트 동기 방식 (Bit-oriented, HDLC):                          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌──────┬───────┬────────┬───────┬──────┐             │ │
│   │   │Flag  │Address│Control │ Data  │ FCS  │ Flag        │ │
│   │   │011111│ 8bit  │ 8/16bit│ 가변  │16/32 │ 01111110    │ │
│   │   │  10  │       │        │       │ bit  │             │ │
│   │   └──────┴───────┴────────┴───────┴──────┘             │ │
│   │                                                         │ │
│   │   Flag: 프레임 구분자 (01111110)                        │ │
│   │   Address: 수신지 주소                                  │ │
│   │   Control: 제어 정보 (프레임 유형, 시퀀스 번호)         │ │
│   │   Data: 상위 계층 데이터                                │ │
│   │   FCS: Frame Check Sequence (CRC)                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   비트 스터핑 (Bit Stuffing):                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   데이터에 연속된 1이 5개 나오면 0을 삽입                │ │
│   │                                                         │ │
│   │   원본: 01111110  → Flag과 혼동                        │ │
│   │   변환: 011111010 → 0 삽입                             │ │
│   │                                                         │ │
│   │   수신측: 0111110 다음의 0을 제거                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분         │ 비동기식        │ 동기식               │ │
│   ├───────────────┼─────────────────┼───────────────────────┤ │
│   │ 동기화 단위   │ 문자 (1바이트)  │ 블록/프레임          │ │
│   │ 클럭 공유     │ 불필요          │ 필요                  │ │
│   │ 오버헤드      │ 높음 (~20%)     │ 낮음 (~5%)            │ │
│   │ 전송 효율     │ 낮음 (~80%)     │ 높음 (~95%)           │ │

│   │ 구현 난이도   │ 쉬움            │ 어려움                │ │
│   │ 오류 회복     │ 문자 재전송     │ 블록 재전송           │ │
│   │ 응용          │ 키보드, RS-232  │ HDLC, 이더넷, USB     │ │
│   └───────────────┴─────────────────┴───────────────────────┘ │
│                                                                 │
│   사용 기준:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   비동기식 선택:                                         │ │
│   │   • 저속, 간헐적 데이터                                  │ │
│   │   • 단거리 통신                                          │ │
│   │   • 비용 민감                                            │ │
│   │   • 인터랙티브 터미널                                    │ │
│   │                                                         │ │
│   │   동기식 선택:                                           │ │
│   │   • 고속, 대량 데이터                                    │ │
│   │   • 장거리 통신                                          │ │
│   │   • 높은 전송 효율 필요                                  │ │
│   │   • 네트워크, 백본 통신                                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **비동기식**: Start/Stop 비트로 문자 단위 동기화, 오버헤드 높음
> **동기식**: 공통 클럭/동기 문자로 블록 단위 동기화, 효율 높음
> **Bit Stuffing**: HDLC에서 Flag과 데이터 구분을 위한 0 삽입

### 2. 빈출 키워드

- **Start/Stop Bit**: 비동기식 프레이밍
- **SYN 문자**: 동기식 전송의 동기 신호
- **8N1**: 8데이터, 패리티 없음, 1 정지비트
- **Bit Stuffing**: 연속 1 다섯 개 후 0 삽입

### 3. 예상 문제

**Q1. 비동기식 전송과 동기식 전송의 차이점을 비교 설명하시오.**

**Q2. HDLC 프레임 구조와 비트 스터핑의 필요성을 기술하시오.**

**Q3. 비동기식 전송에서 8N1 설정의 전송 효율을 계산하시오.**
