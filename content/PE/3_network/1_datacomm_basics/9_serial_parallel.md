# 직렬 전송 vs 병렬 전송

## 정의

**직렬 전송(Serial Transmission)**은 데이터를 **1비트씩 순차적**으로 전송하고, **병렬 전송(Parallel Transmission)**은 **여러 비트를 동시에** 전송한다. 전송 거리, 속도, 비용에 따라 적절한 방식을 선택한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           직렬 전송 vs 병렬 전송                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   직렬 전송 (Serial):                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   송신측:  [D7][D6][D5][D4][D3][D2][D1][D0]              │ │
│   │                    ↓                                    │ │
│   │   전송선:  ═════════════════════════════════▶           │ │
│   │                    ↓ (1비트씩 순차 전송)                 │ │
│   │   수신측:  [D7][D6][D5][D4][D3][D2][D1][D0]              │ │
│   │                                                         │ │
│   │   회선 수: 1개 (단방향) 또는 2개 (전이중)                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   병렬 전송 (Parallel):                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   송신측:  [D7][D6][D5][D4][D3][D2][D1][D0]              │ │
│   │              ↓   ↓   ↓   ↓   ↓   ↓   ↓   ↓              │ │
│   │   전송선: ═════════════════════════════════▶           │ │
│   │            ═════════════════════════════════▶           │ │
│   │            ═════════════════════════════════▶           │ │
│   │            ... (8개 회선)                               │ │
│   │            ═════════════════════════════════▶           │ │
│   │              ↓   ↓   ↓   ↓   ↓   ↓   ↓   ↓              │ │
│   │   수신측:  [D7][D6][D5][D4][D3][D2][D1][D0]              │ │
│   │                                                         │ │
│   │   회선 수: N개 (데이터 비트 수) + 제어선                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 직렬 전송 특성

```
┌─────────────────────────────────────────────────────────────────┐
│           직렬 전송 (Serial Transmission)                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   장점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  • 전송선 수 적음 (1~2개)                                │ │
│   │  • 장거리 전송에 유리                                    │ │
│   │  • 케이블 비용 저렴                                      │ │
│   │  • 크로스토크(Crosstalk) 문제 적음                       │ │
│   │  • 스큐(Skew) 문제 없음                                  │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  • 전송 속도 상대적으로 느림                             │ │
│   │  • 직렬-병렬 변환 필요                                   │ │
│   │  • 클럭 동기화 중요                                      │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   전송 속도:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   N비트 데이터 전송 시간 = N × (1비트 전송 시간)          │ │
│   │                                                         │ │
│   │   예: 8비트 @ 1Gbps = 8ns                               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   응용 분야:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  • USB (Universal Serial Bus)                            │ │
│   │  • SATA (Serial ATA)                                     │ │
│   │  • PCIe (PCI Express)                                    │ │
│   │  • 이더넷 (Ethernet)                                     │ │
│   │  • RS-232, RS-485                                        │ │
│   │  • 광섬유 통신                                           │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 병렬 전송 특성

```
┌─────────────────────────────────────────────────────────────────┐
│           병렬 전송 (Parallel Transmission)                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   장점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  • 높은 전송 속도 (N비트 동시 전송)                       │ │
│   │  • 인터페이스 구조 단순                                  │ │
│   │  • 단거리 고속 전송에 유리                               │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  • 전송선 수 많음 (N+α개)                                │ │
│   │  • 케이블 비용高, 굵음                                   │ │
│   │  • 스큐(Skew) 문제 발생                                  │ │
│   │  • 크로스토크(Crosstalk) 간섭                            │ │
│   │  • 장거리 전송 부적합                                    │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   스큐(Skew) 문제:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   각 전송선의 지연 시간 차이로 비트 도착 시간 불일치      │ │
│   │                                                         │ │
│   │   비트 7: ══════════════════════▶│ (늦음)              │ │
│   │   비트 6: ══════════════▶│        (빠름)              │ │
│   │   비트 5: ═════════════════▶│      (중간)              │ │
│   │   ...                                                  │ │
│   │         ↑ 스큐로 인한 도착 시간 차이                    │ │
│   │                                                         │ │
│   │   주파수 높아질수록 스큐 영향 증가 → 속도 제한           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   응용 분야:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  • 내부 버스 (CPU-Memory)                                │ │
│   │  • 병렬 ATA (PATA/IDE)                                   │ │
│   │  • 프린터 포트 (Centronics)                              │ │
│   │  • PCI (Parallel)                                        │ │
│   │  • SCSI (Wide)                                           │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 직렬-병렬 변환

```
┌─────────────────────────────────────────────────────────────────┐
│           직렬-병렬 변환 회로                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   송신측 (병렬→직렬):                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   병렬 데이터   ┌──────────┐    직렬 데이터              │ │
│   │   [D7..D0] ───▶│ PISO     │───▶ (1비트씩)               │ │
│   │                │ Shift    │                             │ │
│   │   Load ───────▶│ Register │                             │ │
│   │   Clock ──────▶│          │                             │ │
│   │                └──────────┘                             │ │
│   │                                                         │ │
│   │   PISO: Parallel-In Serial-Out                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   수신측 (직렬→병렬):                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   직렬 데이터   ┌──────────┐    병렬 데이터              │ │
│   │   (1비트씩) ───▶│ SIPO     │───▶ [D7..D0]               │ │
│   │                │ Shift    │                             │ │
│   │   Clock ──────▶│ Register │                             │ │
│   │   Load ───────▶│          │                             │ │
│   │                └──────────┘                             │ │
│   │                                                         │ │
│   │   SIPO: Serial-In Parallel-Out                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 전송 방식 선택 기준

```
┌─────────────────────────────────────────────────────────────────┐
│           전송 방식 선택 기준                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  조건           │ 추천 방식    │ 이유                  │ │
│   ├─────────────────┼─────────────┼───────────────────────┤ │
│   │ 단거리 (<1m)    │ 병렬        │ 스큐 영향 적음        │ │
│   │ 장거리 (>10m)   │ 직렬        │ 케이블 비용, 스큐    │ │
│   │ 초고속 (>1Gbps) │ 직렬        │ 스큐 한계 극복       │ │
│   │ 저속 (<100Mbps) │ 병렬/직렬   │ 상황에 따라          │ │
│   │ 케이블 간소화   │ 직렬        │ 회선 수 적음         │ │
│   └─────────────────┴─────────────┴───────────────────────┘ │
│                                                                 │
│   현대적 추세:                                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   병렬 → 직렬 전환 이유:                                 │ │
│   │                                                         │ │
│   │   ① 고속화 한계:                                        │ │
│   │      병렬은 스큐로 인해 수GHz 이상 어려움                │ │
│   │      직렬은 차동 신호로 수십 GHz 가능                   │ │
│   │                                                         │ │
│   │   ② 비용 절감:                                          │ │
│   │      회선 수 감소 → 커넥터/케이블 작아짐                │ │
│   │      예: PATA(40선) → SATA(7선)                         │ │
│   │            PCI(32/64선) → PCIe(1~16레인)                │ │
│   │                                                         │ │
│   │   ③ 차동 신호 품질:                                     │ │
│   │      직렬은 LVDS/PCML 등 고품질 신호 사용               │ │
│   │      잡음 내성, EMI 특성 우수                           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **직렬 전송**: 1비트씩 순차 전송, 장거리 유리
> **병렬 전송**: N비트 동시 전송, 단거리 고속 유리
> **Skew**: 병렬 회선 간 지연 차이

### 2. 빈출 키워드

- **PISO/SIPO**: 병렬입력직렬출력/직렬입력병렬출력
- **Skew (스큐)**: 병렬 전송의 지연 차이
- **Crosstalk**: 인접 회선 간 간섭
- **LVDS**: 저전압 차동 신호 (직렬 고속 전송)

### 3. 예상 문제

**Q1. 직렬 전송과 병렬 전송의 장단점을 비교하시오.**

**Q2. 병렬 전송의 속도 한계 원인과 직렬 전송으로의 전환 이유를 설명하시오.**

**Q3. SATA가 PATA를 대체하게 된 기술적 배경을 설명하시오.**
