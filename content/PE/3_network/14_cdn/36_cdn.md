# CDN (Content Delivery Network)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CDN은 **"콘텐츠**를 **사용자**와 **가까운 **엣지 **서버**(Edge Server)**에 **캐싱**하여 **전송 **지연**(Latency)**을 **줄이고 **전송 **속도**(Throughput)**을 **향상**시키는 **분산 **서비스 **네트워크\"**이다.
> 2. **기술**: **DNS CNAME**(도메인 **위임)**, **Anycast**(동일 **IP, **최근 **라우팅)**, **GeoIP**(지리적 **위치 **기반 **라우팅)**, **Cache-Control**(HTTP **캐싱 **헤더)**로 **요청**을 **최근 **엣지**로 **라우팅**한다.
> 3. **제품**: **Cloudflare**(전 세계 **엣지), **AWS CloudFront**(S3 **통합), **Fastly**(VCL **캐시), **Akamai**(엔터프라이즈)**가 **대표적**이며 **Dynamic Content**, **Video Streaming**, **API Acceleration**을 **지원**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
CDN은 **"콘텐츠 전송 최적화 네트워크"**이다.

**원본 서버 vs CDN**:
| 구분 | 원본 서버 | CDN |
|------|----------|-----|
| **위치** | 단일 | 전 세계 |
| **지연** | 높음 | 낮음 |
| **대역폭** | 병목 | 분산 |
| **가용성** | 단일 장애 | 다중 |

### 💡 비유
CDN은 ****편의점 ****체인 ****과 같다.
- **본사**: Origin Server
- **지점**: Edge Node
- **고객**: User

## Ⅱ. 아키텍처 및 핵심 원리

### CDN 아키텍처

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         CDN Architecture                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  User                                                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  URL: https://cdn.example.com/images/logo.png                                       │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                             │  │
    │            ▼ DNS Resolution                                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Query: cdn.example.com                                                          │  │  │
    │  │  2. Local DNS → CDN authoritative DNS                                                │  │  │
    │  │  3. Return: CNAME to cdn.provider.net                                                │  │  │
    │  │  4. DNS resolves to nearest edge IP (Anycast)                                        │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                             │  │
    │            ▼ HTTP Request to Edge Node                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Edge Server (Seoul)                                                                  │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Cache MISS → Fetch from Origin → Cache → Return to User                        │  │  │  │
    │  │  │  Cache HIT → Return immediately (low latency)                                    │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │ (if cache miss)                                                               │  │
    │            ▼ Origin Fetch                                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Origin Server (your datacenter)                                                      │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  • S3 bucket, custom server, or another CDN as origin                            │  │  │  │
    │  │  │  • Edge fetches and caches content                                               │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Caching Strategies

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         HTTP Caching Headers                                                │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Cache-Control:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  │  Header                            │  Meaning                                       │  │
    │  │  ─────────────────────────────────────────────────────────────────────────────────────│  │
    │  │  Cache-Control: public             │  CDN and browser can cache                    │  │
    │  │  Cache-Control: private            │  Only browser can cache                       │  │
    │  │  Cache-Control: max-age=3600        │  Cache for 1 hour                            │  │
    │  │  Cache-Control: no-cache            │  Always validate (ETag)                      │  │
    │  │  Cache-Control: no-store            │  Don't cache at all                          │  │
    │  │  Cache-Control: s-maxage=300        │  CDN cache for 5 min (browser: no)           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Cache Validation:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  // First request                                                                         │  │
    │  GET /image.jpg                                                                           │  │
    │  Response:                                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Cache-Control: max-age=3600                                                          │  │  │
    │  │  ETag: "abc123"                                                                       │  │  │
    │  │  Last-Modified: Wed, 06 Mar 2026 10:00:00 GMT                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  // Request after max-age expires                                                        │  │
    │  GET /image.jpg                                                                           │  │
    │  If-None-Match: "abc123"                                                                  │  │
    │  Response:                                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  304 Not Modified (if unchanged) → Use cached copy                                   │  │  │
    │  │  200 OK + new content (if changed)                                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Anycast Routing

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Anycast vs Unicast                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Unicast (Traditional):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Server A: 1.2.3.4 (New York)                                                           │  │
    │  Server B: 5.6.7.8 (London)                                                              │  │
    │  Server C: 9.10.11.12 (Tokyo)                                                            │  │
    │                                                                                         │  │
    │  DNS resolves to specific IP → User connects to that specific server                      │  │
    │  → If server A fails, DNS must update to server B                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Anycast (CDN):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  All edge servers: 1.2.3.4 (same IP address!)                                           │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  BGP announces 1.2.3.4 from multiple locations                                      │  │  │
    │  │  • Router in Seoul → Routes to closest edge (Tokyo)                                  │  │  │
    │  │  • Router in NY → Routes to closest edge (New York)                                  │  │  │
    │  │  • If Tokyo edge fails, BGP automatically reroutes to next closest                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  User in Seoul connects to 1.2.3.4 → Reaches Tokyo edge                                 │  │
    │  User in NY connects to 1.2.3.4 → Reaches NY edge                                       │  │
    │                                                                                         │  │
    │  → Automatic failover, load distribution                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         CDN Edge Capabilities                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
```

### Cache Purge Strategies

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Cache Invalidation Methods                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    1. Time-Based (Passive):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • max-age expires → Next request fetches fresh                                        │  │
    │  • Simple but stale until expiry                                                       │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. Purge API (Active):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Purge specific URL                                                                    │  │
    │  POST /api/purge/url                                                                     │  │
    │  {"url": "https://cdn.example.com/images/logo.png"}                                     │  │
    │                                                                                         │  │
    │  # Purge by tag                                                                          │  │
    │  POST /api/purge/tag                                                                     │  │
    │  {"tag": "v2.0"}  # Purge all content with cache-tag: v2.0                              │  │
    │                                                                                         │  │
    │  # Purge all (emergency)                                                                 │  │
    │  POST /api/purge/all                                                                     │  │
    │  → Immediate invalidation but increases origin load                                      │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. Cache Tags (Smart):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  // Response headers                                                                      │  │
    │  Cache-Tag: product-123, category-shoes, v2.0                                            │  │
    │                                                                                         │  │
    │  // Purge all shoes                                                                      │  │
    │  POST /api/purge/tag {"tag": "category-shoes"}                                          │  │
    │                                                                                         │  │
    │  // Deploy new version                                                                   │  │
    │  POST /api/purge/tag {"tag": "v2.0"}                                                    │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Global CDN Setup                                                    │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Architecture:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Static Assets                                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Domain: static.example.com → CNAME: example.cloudflare.com                          │  │  │
    │  │  Origin: S3 bucket (us-east-1)                                                        │  │  │
    │  │  Cache-Control: max-age=31536000 (1 year)                                           │  │  │
    │  │  → Images, CSS, JS cached for 1 year                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  2. Dynamic Content                                                                       │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Domain: api.example.com → CNAME: api.cloudflare.com                                │  │  │
    │  │  Origin: ALB (us-east-1)                                                              │  │  │
    │  │  Cache-Control: s-maxage=60, max-age=0                                               │  │  │
    │  │  → CDN caches for 60s, browser doesn't cache                                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  3. Page Rules                                                                            │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  URL Pattern: *.jpg, *.png, *.css, *.js                                              │  │  │
    │  │  Settings: Cache Level: Cache Everything, Edge Cache TTL: 1 month                    │  │  │
    │  │                                                                                       │  │  │
    │  │  URL Pattern: /api/*                                                                  │  │  │
    │  │  Settings: Bypass Cache, Disable Performance                                          │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```
