# Web 3.0 (Web3)

## 정의

**Web 3.0** = **Read-Write-Own** 웹으로, 탈중앙화(Decentralization)와 블록체인 기술을 기반으로 사용자가 데이터와 디지털 자산의 **소유권**을 가지는 차세대 인터넷 환경입니다.

## 웹의 진화

```
┌─────────────────────────────────────────────────────────┐
│                   웹의 진화 단계                         │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Web 1.0 (1990~2003)                                    │
│  └─ Read-Only: "읽기 전용"                               │
│  └─ 사용자: 콘텐츠 소비자                                │
│  └─ 예: 야후, 초기 포털                                  │
│                                                          │
│  Web 2.0 (2004~2020)                                    │
│  └─ Read-Write: "읽기+쓰기"                              │
│  └─ 사용자: 콘텐츠 생산자+소비자                         │
│  └─ 예: 페이스북, 유튜브, 인스타그램                     │
│  └─ 문제: 플랫폼 독점, 데이터 주권 상실                  │
│                                                          │
│  Web 3.0 (2021~)                                        │
│  └─ Read-Write-Own: "읽기+쓰기+소유"                     │
│  └─ 사용자: 데이터와 자산의 진정한 소유자                │
│  └─ 예: 이더리움, NFT, DeFi, DAO                        │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 탈중앙화 (Decentralization)
```
┌─────────────────────────────────────────────────────────┐
│              중앙화 vs 탈중앙화                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  [중앙화 Web 2.0]              [탈중앙화 Web 3.0]        │
│                                                          │
│      ┌───┐                         ┌───┐               │
│      │서버│                        │노드│               │
│      └─┬─┘                         └─┬─┘               │
│        │                 ┌───┐  ┌───┐  ┌───┐           │
│   ┌────┼────┐           │노드│──│노드│──│노드│         │
│   ↓    ↓    ↓            └─┬─┘  └─┬─┘  └─┬─┘         │
│ ┌───┐┌───┐┌───┐           │  ┌───┐  │                 │
│ │클1││클2││클3│           └──│노드│──┘                 │

```

### 2. Web 3.0 핵심 기술

| 기술 | 설명 | 예시 |
|------|------|------|
| **블록체인** | 분산 원장 기술 (DLT) | 이더리움, 솔라나 |
| **스마트 컨트랙트** | 자동 실행 계약 | Solidity, Vyper |
| **토크노믹스** | 토큰 경제 설계 | ERC-20, ERC-721 |
| **IPFS** | 분산 파일 시스템 | 영구 저장, 검열 저항 |
| **DID** | 탈중앙화 신원 | Self-sovereign Identity |

## Web 2.0 vs Web 3.0 비교

| 구분 | Web 2.0 | Web 3.0 |
|------|---------|---------|
| **데이터 소유권** | 플랫폼 기업 | 사용자 |
| **신원 증명** | 중앙 인증 (OAuth) | DID, 지갑 |
| **결제** | 신용카드, PG사 | 암호화폐, 스마트컨트랙트 |
| **신뢰 모델** | 신뢰 기반 (Trust) | 검증 기반 (Trustless) |
| **프라이버시** | 기업 통제 | 사용자 통제 |
| **검열 저항성** | 낮음 | 높음 |
| **인프라** | 중앙 서버 | 분산 노드 |

```
│                  Web 3.0 스택                            │
├─────────────────────────────────────────────────────────┤
│                                                          │

│  └─ DApps, NFT 마켓플레이스, DeFi 프론트엔드            │

│  [Layer 3: Protocol]                                    │

│                                                          │

│  └─ 이더리움, 솔라나, 폴리곤, 아발란체                   │
│                                                          │

│  └─ IPFS, Filecoin, Arweave, Chainlink                 │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 실무 적용

### 기업 Web 3.0 도입 전략

1. **파일럿 프로젝트**
   - NFT 멤버십 프로그램
   - 브랜드 커뮤니티 DAO
   - 디지털 자산 발행

2. **기술 스택 선정**
   - L1: 이더리움 (보안) / 폴리곤 (비용)
   - L2: Arbitrum, Optimism
   - 스토리지: IPFS + Filecoin

3. **규제 대응**
   - VASP 등록 (국가별)
   - AML/KYC 준수
   - 토큰 증권 여부 검토

## 기술사 시험 포인트

### ⭐ 핵심 정의
> **Web 3.0** = 블록체인 기반 탈중앙화 웹으로, 사용자가 데이터와 디지털 자산의 소유권을 가지며 신뢰 없이도 거래 가능한 환경

### ⭐ 빈출 키워드
1. **Decentralization**: 탈중앙화, 단일 실패점 제거
2. **Trustless**: 제3자 신뢰 없는 검증
3. **Self-sovereign**: 자기 주권 데이터
4. **Smart Contract**: 자동 실행 계약
5. **Token Economy**: 인센티브 설계

### ⭐ 예상 문제
**Q. Web 2.0의 한계와 Web 3.0의 해결 방안을 기술하시오.**

**A.**

1. **Web 2.0의 한계**
   - 데이터 주권 상실: 플랫폼 기업이 사용자 데이터 독점
   - 검열 위험: 중앙 기관의 임의 차단 가능
   - 플랫폼 리스크: 서비스 종료 시 데이터 소멸
   - 프라이버시 침해: 행동 타겟팅 광고

2. **Web 3.0의 해결 방안**
   - 데이터 주권: 사용자가 프라이빗 키로 소유권 증명
   - 검열 저항: 분산 네트워크로 단일 통제 불가
   - 영속성: 블록체인/분산 스토리지로 데이터 보존
   - 프라이버시: DID로 최소 정보만 제공
