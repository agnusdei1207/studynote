# Web 1.0 / Web 2.0

## 정의

### Web 1.0 (Read-Only Web)
1990년대 초~중반, 인터넷 초창기의 정적 웹 환경으로, 사용자는 콘텐츠를 **수동적으로 소비**하는 것만 가능했습니다.

### Web 2.0 (Read-Write Web)
2000년대 중반부터, 사용자가 **콘텐츠를 생성·공유·수정**할 수 있는 플랫폼 중심의 참여형 웹입니다.

## 핵심 개념

### Web 1.0 특징
```
┌─────────────────────────────────────────────────────┐
│                    Web 1.0 구조                      │
├─────────────────────────────────────────────────────┤
│  ┌─────────┐    정적 HTML     ┌─────────┐          │
│  │  서버   │ ───────────────→ │  사용자  │          │
│  │ (Content│                   │ (Consumer│          │
│  │ Provider│                   │  Only)  │          │
│  └─────────┘                   └─────────┘          │
│                                                      │
│  특징: 단방향, 정적, 읽기 전용, 포털 중심            │
└─────────────────────────────────────────────────────┘
```

| 구분 | Web 1.0 |
|------|---------|
| **상호작용** | 단방향 (서버 → 사용자) |
| **콘텐츠** | 정적 HTML, 텍스트 위주 |
| **사용자 역할** | 소비자 (Consumer) |
| **대표 서비스** | 야후, 다음, 네이버 초기 포털 |
| **기술** | HTML, CGI, Perl |

### Web 2.0 특징
```
┌─────────────────────────────────────────────────────┐
│                    Web 2.0 구조                      │
├─────────────────────────────────────────────────────┤
│  ┌─────────┐    API/JSON     ┌─────────┐          │
│  │  플랫폼  │ ←─────────────→ │  사용자  │          │
│  │ (Platform│    양방향       │ (Prosumer│          │
│  │  Provider│                 │ Creator)│          │
│  └─────────┘                   └─────────┘          │
│       ↑                              │              │
│       └──────── UGC ─────────────────┘              │
│                                                      │
│  특징: 양방향, 동적, 참여형, 플랫폼 중심, UGC        │
└─────────────────────────────────────────────────────┘
```

| 구분 | Web 2.0 |
|------|---------|
| **상호작용** | 양방향 (플랫폼 ↔ 사용자) |
| **콘텐츠** | 동적, UGC (User Generated Content) |
| **사용자 역할** | 생산자+소비자 (Prosumer) |
| **대표 서비스** | 유튜브, 페이스북, 위키피디아, 블로그 |
| **기술** | AJAX, REST API, JavaScript, CSS3 |

## Web 2.0 핵심 기술

### AJAX (Asynchronous JavaScript and XML)
```
┌────────────────────────────────────────────────────┐
│                AJAX 동작 방식                       │
├────────────────────────────────────────────────────┤
│                                                     │
│  1. 사용자 이벤트 발생                              │
│         ↓                                           │
│  2. JavaScript → XMLHttpRequest 객체 생성          │
│         ↓                                           │
│  3. 비동기 서버 요청 (페이지 새로고침 없음)         │
│         ↓                                           │
│  4. 서버 응답 (XML/JSON)                           │
│         ↓                                           │
│  5. DOM 조작으로 화면 일부 갱신                     │
│                                                     │
│  장점: 빠른 응답, 서버 부하 감소, UX 향상           │
└────────────────────────────────────────────────────┘
```

### RIA (Rich Internet Application)
- 플래시, 실버라이트, 자바FX 등
- 데스크톱 앱 수준의 인터랙션 제공
- 현재는 HTML5, WebAssembly로 대체

### Mashup
- 여러 API를 조합하여 새로운 서비스 생성
- 예: 구글맵 + 부동산 데이터 = 부동산 지도 서비스

## Web 1.0 vs Web 2.0 비교

| 비교 항목 | Web 1.0 | Web 2.0 |
|----------|---------|---------|
| **출현 시기** | 1990년대 | 2004년~ |
| **패러다임** | 읽기 (Read) | 읽기+쓰기 (Read-Write) |
| **주체** | 콘텐츠 제공자 | 사용자 (참여형) |
| **데이터 흐름** | 단방향 | 양방향 |
| **페이지 성격** | 정적 (Static) | 동적 (Dynamic) |
| **링크 구조** | 하이퍼링크 | 태그, RSS, 트랙백 |
| **검색** | 디렉토리 기반 | 알고리즘 기반 (PageRank) |
| **수익 모델** | 배너 광고 | 타겟팅 광고, 플랫폼 비즈니스 |

## 실무 적용

### 기업 관점에서의 Web 2.0 전략
1. **플랫폼 비즈니스**: 사용자 간 연결 및 중개
2. **데이터 수집**: UGC를 통한 빅데이터 확보
3. **소셜 마케팅**: 바이럴, 입소문 효과 활용
4. **오픈 API**: 외부 개발자 생태계 구축

### 보안 고려사항
- XSS (Cross-Site Scripting)
- CSRF (Cross-Site Request Forgery)
- 사용자 프라이버시 보호
- UGC 검증 및 필터링

## 기술사 시험 포인트

### ⭐ 핵심 정의
> **Web 2.0** = 사용자 참여형 플랫폼 중심 웹으로, 읽기와 쓰기가 모두 가능한 양방향 인터랙션 환경

### ⭐ 빈출 키워드
1. **UGC (User Generated Content)**: 사용자 생성 콘텐츠
2. **Prosumer**: Producer + Consumer
3. **AJAX**: 비동기 통신 기술
4. **Mashup**: API 조합 서비스
5. **Long Tail**: 니치 시장의 집합적 가치

### ⭐ 예상 문제
**Q. Web 1.0과 Web 2.0의 차이를 기술적, 비즈니스적 관점에서 설명하시오.**

**A.**
1. **기술적 관점**
   - Web 1.0: 정적 HTML, 단방향 HTTP 요청, 페이지 단위 새로고침
   - Web 2.0: AJAX 비동기 통신, REST API, DOM 조작, 실시간 갱신

2. **비즈니스적 관점**
   - Web 1.0: 콘텐츠 제공자 중심, 포털, 배너 광고
   - Web 2.0: 사용자 참여형 플랫폼, 소셜 네트워크, 타겟팅 광고
