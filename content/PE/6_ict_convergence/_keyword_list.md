# ICT 융합 및 신기술 키워드 목록 (심화 확장판)

정보관리기술사, 컴퓨터응용시스템기술사 합격을 위한 최신 ICT 융합, 4차 산업혁명 핵심 기술, Web 3.0, 자율주행, 퀀텀 컴퓨팅, 공간 컴퓨팅 및 스마트 시티 등 전 영역을 망라한 800대 핵심 키워드입니다.

---

## 1. Web 3.0 및 블록체인 (Blockchain) (100개)
1. Web 1.0 (Read-only) / Web 2.0 (Read-Write, 플랫폼 중심)
2. Web 3.0 (Read-Write-Own) - 탈중앙화(Decentralization)와 개인 소유권 보장 웹
3. 시맨틱 웹 (Semantic Web) - 온톨로지(Ontology) 기반 의미 연결망
4. 블록체인 (Blockchain) - 분산 원장 기술(DLT, Distributed Ledger Technology)
5. 제네시스 블록 (Genesis Block) - 블록체인의 첫 번째 블록
6. 블록의 구조 - 블록 헤더 (버전, 이전 블록 해시, 머클 루트, 타임스탬프, 난이도, 논스) + 바디
7. 머클 트리 (Merkle Tree / Hash Tree) - 트랜잭션 무결성 검증을 위한 해시 트리
8. 머클 루트 (Merkle Root) - 모든 트랜잭션 해시를 묶은 최종 해시값
9. 해시 포인터 (Hash Pointer) - 데이터의 위치와 무결성 정보를 동시에 지님
10. 탈중앙화 (Decentralization) - 단일 장애점(SPOF) 제거 및 투명성 확보
11. 합의 알고리즘 (Consensus Algorithm) - 분산 노드 간 상태 일치 달성 매커니즘
12. 비잔틴 장애 허용 (BFT, Byzantine Fault Tolerance) - 1/3 미만의 악의적 노드가 있어도 정상 합의 보장
13. PBFT (Practical BFT) - 다수결 기반 상태 기계 복제 합의 (텐더민트, 하이퍼레저)
14. 작업 증명 (PoW, Proof of Work) - 해시 퍼즐 연산 경쟁 (비트코인), 막대한 전력 소모
15. 지분 증명 (PoS, Proof of Stake) - 보유 지분(Coin)에 비례해 블록 생성 권한 부여 (이더리움 2.0)
16. 위임 지분 증명 (DPoS, Delegated PoS) - 대표자(BP)를 투표로 선출해 합의 위임 (빠른 속도, EOS)
17. 권위 증명 (PoA, Proof of Authority) - 신원 인증된 노드만 합의 참여 (프라이빗 블록체인)
18. 공간/시간 증명 (PoST, Proof of Space and Time) - 스토리지 자원 증명 (Chia Network)
19. 퍼블릭 블록체인 (Public Blockchain) - 누구나 참여 가능 (비트코인, 이더리움)
20. 프라이빗 블록체인 (Private Blockchain) - 허가된 노드만 참여 (하이퍼레저 패브릭)
21. 컨소시엄 블록체인 (Consortium Blockchain) - 여러 기업이 연합하여 노드 운영
22. 스마트 컨트랙트 (Smart Contract) - 조건이 충족되면 블록체인 상에서 자동 실행되는 프로그램 코드 (닉 자보 제안)
23. EVM (Ethereum Virtual Machine) - 이더리움 스마트 컨트랙트 런타임 환경
24. 가스 (Gas) - 스마트 컨트랙트 실행 및 트랜잭션 처리에 지불하는 네트워크 수수료
25. 튜링 완전성 (Turing Completeness) - 이더리움 스마트 컨트랙트 연산의 무한루프 방지를 위해 Gas 도입
26. 토큰 이코노미 (Token Economy) - 블록체인 생태계 내의 인센티브 보상 구조
27. ICO (Initial Coin Offering) / STO (Security Token Offering, 증권형 토큰 발행) / IEO
28. RWA (Real World Asset) 토큰화 - 실물 자산(부동산, 미술품 등)의 블록체인 토큰 변환
29. 대체 불가능 토큰 (NFT, Non-Fungible Token) - 고유한 식별자를 가진 디지털 자산 (ERC-721 규격)
30. 암호화폐 지갑 (Cryptocurrency Wallet) - 핫 월렛 (온라인) vs 콜드 월렛 (오프라인/하드웨어)
31. 니모닉 (Mnemonic) - 지갑 복구를 위한 12~24개의 무작위 영단어 조합 (시드 구문)
32. 디앱 (DApp, Decentralized Application) - 백엔드가 블록체인(스마트 컨트랙트)인 애플리케이션
33. 디파이 (DeFi, Decentralized Finance) - 탈중앙화 금융 (중개자 없는 대출, 예치, 스왑)
34. 유니스왑 (Uniswap) / AMM (Automated Market Maker) - 자동화된 시장 조성자 알고리즘 (유동성 풀)
35. 플래시 론 (Flash Loan) - 트랜잭션 블록 1개 내에서 무담보 대출 및 상환을 동시에 진행하는 구조
36. 블록체인 오라클 (Oracle) 문제 - 블록체인 외부(Off-chain)의 현실 데이터를 블록체인 내부(On-chain)로 가져올 때 발생하는 신뢰성 문제
37. 영지식 증명 (ZKP, Zero-Knowledge Proof) - 비밀을 공개하지 않고도 그 비밀을 안다는 것을 증명 (Zcash 등 프라이버시 보호)
38. zk-SNARKs (Zero-Knowledge Succinct Non-Interactive Argument of Knowledge) - 비대화형 영지식 증명
39. 레이어 1 (Layer 1) - 비트코인, 이더리움 등 메인 블록체인 네트워크 (트릴레마 딜레마 직면)
40. 블록체인 트릴레마 (Blockchain Trilemma) - 확장성(Scalability), 탈중앙화(Decentralization), 보안성(Security) 세 가지를 동시에 완벽히 만족할 수 없는 문제
41. 레이어 2 (Layer 2) 솔루션 - L1의 확장성 문제를 해결하기 위해 메인넷 밖에서 연산 수행 후 결과만 L1에 기록
42. 롤업 (Rollup) - 수천 개의 오프체인 트랜잭션을 하나로 묶어(Rollup) L1에 검증
43. 옵티미스틱 롤업 (Optimistic Rollup) - 기본적으로 트랜잭션이 유효하다고 낙관하고, 사기 증명(Fraud Proof) 기간을 두는 방식 (Arbitrum, Optimism)
44. ZK 롤업 (Zero-Knowledge Rollup) - 트랜잭션 배치마다 영지식 증명(Validity Proof)을 생성해 L1에 즉시 확정 (속도 빠름, 연산 복잡)
45. 사이드체인 (Sidechain) - 메인체인과 브리지로 연결된 별도의 독립된 블록체인 (Polygon 등)
46. 샤딩 (Sharding) - 메인 체인을 여러 개의 조각(Shard)으로 분할하여 트랜잭션을 병렬 처리하는 L1 확장성 기술
47. 하드 포크 (Hard Fork) - 기존 블록체인과 호환되지 않는 중대한 프로토콜 업그레이드 (체인 분리)
48. 소프트 포크 (Soft Fork) - 이전 버전과 호환성을 유지하는 프로토콜 업그레이드
49. 블록체인 브리지 (Cross-chain Bridge) - 서로 다른 이기종 블록체인 간 자산 및 데이터 이동 통로
50. 소울바운드 토큰 (SBT, Soulbound Token) - 양도/전송이 불가능한 NFT (학위, 자격증, 신원 증명용)
51. 탈중앙화 신원증명 (DID, Decentralized Identity) - W3C 표준, 사용자가 자신의 신원 정보를 직접 관리하고 통제 (SSI, Self-Sovereign Identity)
52. DID 아키텍처 - Issuer(발행자), Holder(소유자/지갑), Verifier(검증자), Verifiable Credential(VC), Verifiable Presentation(VP)
53. 분산 식별자 (DID Document) - 퍼블릭 키, 인증 정보, 서비스 엔드포인트 포함 
54. 다오 (DAO, Decentralized Autonomous Organization) - 중앙 관리자 없이 스마트 컨트랙트와 거버넌스 토큰 투표로 운영되는 자율 조직
55. IPFS (InterPlanetary File System) - P2P 분산 파일 시스템, 위치 기반(HTTP URL)이 아닌 콘텐츠 해시(CID) 기반 검색
56. 스마트 컨트랙트 보안 취약점 - 재진입 (Re-entrancy, DAO 해킹 사태 원인), 오버플로우/언더플로우, 권한 탈취
57. 솔리디티 (Solidity) - 이더리움 스마트 컨트랙트 개발 언어
58. 하이퍼레저 패브릭 (Hyperledger Fabric) - IBM 주도 기업용 프라이빗 블록체인 프레임워크 
59. 체인코드 (Chaincode) - 하이퍼레저의 스마트 컨트랙트
60. 하이퍼레저 아키텍처 - 피어(Peer), 오더러(Orderer, 합의 노드), MSP(Membership Service Provider, 인증)
61. CBDC (Central Bank Digital Currency) - 중앙은행이 직접 발행하는 디지털 화폐 (소매형 vs 도매형)
62. 비트코인 반감기 (Halving) - 약 4년마다 채굴 보상이 절반으로 줄어드는 메커니즘
63. 트랜잭션 풀 (Mempool / Memory Pool) - 블록에 포함되지 않은 대기 중인 트랜잭션 저장소
64. BFT 합의의 3단계 - Pre-prepare, Prepare, Commit
65. 합의 완결성 (Finality) - 블록이 체인에 기록되어 뒤집히지 않음이 보장되는 상태 (PoW는 확률적 완결성, BFT는 즉각적 완결성)
66. 지향성 비순환 그래프 (DAG, Directed Acyclic Graph) - 블록체인 대신 트랜잭션들이 거미줄처럼 서로를 증명하는 분산 원장 구조 (IOTA의 Tangle)
67. 51% 공격 (51% Attack) - 악의적 노드가 전체 해시 파워의 51% 이상을 장악해 장부를 조작하는 공격
68. 이클립스 공격 (Eclipse Attack) - 특정 노드의 주변 P2P 연결을 악성 노드가 장악하여 네트워크를 고립시키고 허위 정보를 주입하는 공격
69. 크립토재킹 (Cryptojacking) - 타인의 PC나 서버 리소스를 해킹하여 몰래 암호화폐를 채굴하는 공격
70. 시빌 공격 (Sybil Attack) - 한 명이 여러 개의 가짜 노드(신분)를 생성하여 투표율/합의를 조작하는 공격
71. 블록체인 서비스형 (BaaS, Blockchain as a Service) - 클라우드 기반 블록체인 인프라 제공 서비스
72. ERC-20 (이더리움 대체 가능 토큰 표준)
73. ERC-1155 (다중 토큰 표준 - NFT와 FT 동시 발행 가능, 게임 아이템에 유리)
74. 영지식 이더리움 가상머신 (zkEVM) - ZK 롤업 내에서 이더리움 스마트 컨트랙트를 그대로 실행할 수 있게 호환성 제공
75. 분산 스토리지 보상 시스템 (Filecoin, Arweave)
76. 무허가형 (Permissionless) vs 허가형 (Permissioned) 블록체인
77. 암호화폐 믹싱 (Coin Mixing / Tumbler) - 거래 자금 출처 추적을 어렵게 하는 트랜잭션 섞기 (Tornado Cash 등, 자금 세탁 악용)
78. 웹3.0 소셜 네트워크 (Mastodon, Bluesky, Nostr 기반 프로토콜)
79. 마이데이터 (MyData) 시스템의 블록체인 영수증 및 인증 활용
80. CBDC 오프라인 결제 프로토콜
81. 블록체인 확장성 지표 - TPS (Transactions Per Second), 레이턴시 (Latency)
82. 라이트 노드 (Light Node / SPV, Simplified Payment Verification) - 블록 헤더만 다운로드하여 트랜잭션을 검증하는 모바일/경량 환경용 노드
83. 풀 노드 (Full Node) - 제네시스 블록부터 모든 거래 내역을 보관하고 검증하는 노드
84. 블록체인 상호운용성 (Interoperability) 폴카닷(Polkadot), 코스모스(Cosmos) 네트워크
85. 가상자산 사업자 (VASP) 트래블 룰 (Travel Rule) - 자금 세탁 방지를 위해 가상자산 송/수신자 정보를 확인하는 규제
86. 지갑 멀티시그 (Multi-Sig, Multi-Signature) - 출금을 위해 N명 중 M명 이상의 서명이 필요한 보안 지갑 구조
87. 계정 추상화 (Account Abstraction, ERC-4337) - 이더리움 지갑(EOA)을 스마트 컨트랙트(CA)처럼 프로그래밍 가능하게 만들어 소셜 복구, 가스비 대납 등을 지원
88. EOA (Externally Owned Account) - 개인키로 통제되는 일반 사용자 계정
89. CA (Contract Account) - 코드에 의해 통제되는 컨트랙트 계정
90. 거버넌스 51% 방어 체계 (분산 슬래싱 Slashing) - 악의적 행동 적발 시 스테이킹한 지분을 몰수하는 PoS 방어 기법
91. 합성 자산 (Synthetic Assets) 토큰 구조
92. 탈중앙화 신탁 관리 (Decentralized Escrow)
93. 스마트 컨트랙트 정형 검증 (Formal Verification) - 수학적 모델링을 통해 컨트랙트 코드 무결성 증명
94. 데이터 가용성 (Data Availability, DA) 계층 - 롤업 트랜잭션 데이터가 L1에 안전하게 게시되었는지 보장하는 블록체인 모듈러 분리 계층 (Celestia, EigenLayer)
95. 모듈러 블록체인 (Modular Blockchain) - 실행(Execution), 합의(Consensus), 정산(Settlement), 데이터 가용성(DA) 계층을 분리하여 확장성 극대화 
96. 모놀리식 블록체인 (Monolithic Blockchain) - 모든 작업을 단일 체인(솔라나, 앱토스 등)에서 처리
97. 타원곡선 디지털 서명 알고리즘 (ECDSA) 및 슈노르 서명 (Schnorr Signature - 다중 서명 병합 축소)
98. 블록체인 데이터 인덱싱 프로토콜 (The Graph)
99. BRC-20 표준 - 비트코인 네트워크 상에서의 토큰 발행 프로토콜 (오디널스 Ordinals)
100. 양자 내성 블록체인 서명 체계 전환 연구

## 2. 사물인터넷(IoT), 모빌리티 및 무선 신기술 (80개)
101. 사물인터넷 (IoT, Internet of Things) 개념 - 모든 사물이 센서와 통신 기능을 내장하여 인터넷에 연결되는 기술
102. IoT 3대 구성 요소 - 디바이스(센서/액추에이터), 네트워크, 플랫폼/클라우드
103. 센서 네트워크 (WSN, Wireless Sensor Network) / 싱크 노드(Sink Node)
104. 초소형 운영체제 (TinyOS, RIOT, FreeRTOS)
105. 엣지 컴퓨팅 (Edge Computing) - 클라우드로 모든 데이터를 보내지 않고 디바이스 주변(엣지)에서 데이터를 실시간 처리하여 지연 단축 및 대역폭 절감
106. 포그 컴퓨팅 (Fog Computing) - 시스코 제안, 클라우드와 엣지 사이의 지역 노드(게이트웨이) 단에서 분산 처리
107. 초연결 사회 (Hyper-connected Society)
108. IoT 무선 통신 기술 분류 - WPAN, WLAN, LPWAN
109. 저전력 광역 통신망 (LPWAN, Low-Power Wide-Area Network) - 수십 km 커버리지, 수년 배터리 수명, 저용량 데이터
110. 비면허 대역 LPWAN - LoRaWAN (Chirp Spread Spectrum 방식), Sigfox (Ultra Narrow Band)
111. 면허 대역 (이동통신 기반) LPWAN - NB-IoT (LTE 보호대역 200kHz 사용), LTE-M (eMTC, 음성 및 이동성 지원)
112. 지그비 (ZigBee) - IEEE 802.15.4 기반 저전력 메시(Mesh) 네트워크 WPAN 기술
113. Z-Wave - 홈 네트워크 특화 저주파(900MHz) 무선 통신 
114. 블루투스 저전력 (BLE, Bluetooth Low Energy) - 비컨(Beacon) 활용 위치 기반 서비스
115. 스레드 (Thread) 프로토콜 - IPv6 기반 스마트홈 IoT 메시 네트워크 (Google 주도)
116. 매터 (Matter) 표준 - CSA 주도, 애플/구글/아마존 스마트홈 기기 간 상호 연동성(Interoperability)을 보장하는 단일 앱/프로토콜 규격
117. 6LoWPAN - 저전력/소용량 네트워크(802.15.4)에서 IPv6 패킷을 전송하기 위해 헤더 압축 및 단편화 수행
118. MQTT (Message Queuing Telemetry Transport) - TCP 기반, 퍼블리시/서브스크라이브 (Pub/Sub) 모델의 초경량 IoT 메시지 프로토콜, 브로커(Broker) 필수
119. MQTT QoS (Quality of Service) - Level 0 (최대 1회, 유실 가능), Level 1 (최소 1회, 중복 가능), Level 2 (정확히 1회, 오버헤드 큼)
120. CoAP (Constrained Application Protocol) - UDP 기반, RESTful(GET/POST/PUT/DELETE) 구조를 제공하는 경량 IoT 프로토콜, 브로커리스 P2P 지원
121. LwM2M (Lightweight M2M) - CoAP 기반의 IoT 디바이스 원격 관리 프로토콜
122. oneM2M - 이기종 사물인터넷 플랫폼 간 상호 연동을 위한 글로벌 공통 아키텍처 표준
123. OCF (Open Connectivity Foundation) 표준
124. IoT 봇넷 (Botnet) 위협 - 미라이(Mirai) 악성코드, 디폴트 패스워드 악용 DDoS 전진기지화
125. 무선 스니핑 및 리플레이 공격 방어
126. 디지털 트윈 (Digital Twin) - 물리적 세계(객체, 시스템, 도시)를 가상 공간에 1:1로 동일하게 구현하고 실시간 동기화하여 시뮬레이션 및 예측하는 기술
127. 디지털 트윈 3요소 - 물리 공간 (Physical), 가상 공간 (Virtual), 실시간 데이터 연결 (Data/Connection)
128. 가상/증강/혼합 현실 (VR / AR / MR / XR)
129. 공간 컴퓨팅 (Spatial Computing) - 현실 공간을 3D로 스캔하여 디지털 정보와 상호작용하는 기술 (Apple Vision Pro 등)
130. 6자유도 (6DoF) 트래킹 - X, Y, Z 이동 및 Pitch, Yaw, Roll 회전 추적
131. SLAM (Simultaneous Localization and Mapping) - 로봇/AR 기기가 미지의 환경을 돌아다니며 지도를 작성함과 동시에 자신의 위치를 추정하는 자율주행/AR 핵심 알고리즘
132. V-SLAM (Vision SLAM) - 카메라 시각 정보 기반 SLAM
133. 볼류메트릭 비디오 (Volumetric Video) - 다수의 카메라로 인물/사물을 360도 3D로 캡처하여 홀로그램처럼 재생하는 기술
134. 메타버스 (Metaverse) - 가상 현실 기술을 기반으로 경제/사회/문화 활동이 이루어지는 3D 가상 세계
135. 자율주행 자동차 (Autonomous Vehicle) 기술 단계 (SAE J3016 기준 0~5단계)
136. 레벨 2 (부분 자동화) - ADAS (차선 유지, 크루즈 컨트롤) 결합
137. 레벨 3 (조건부 자율주행) - 시스템 제어, 비상 시 운전자 개입 (시선 이탈 금지)
138. 레벨 4 (고도 자율주행) - 특정 구역 내에서 시스템이 완전 주행 책임 (운전자 개입 불필요)
139. 센서 퓨전 (Sensor Fusion) - 카메라, 라이다(LiDAR), 레이더(Radar), 초음파 센서의 데이터를 결합하여 인식 정확도 극대화
140. 라이다 (LiDAR, Light Detection And Ranging) - 레이저를 쏴서 반사되는 시간(ToF)을 측정해 주변을 고정밀 3D 포인트 클라우드(Point Cloud)로 맵핑
141. V2X (Vehicle to Everything) 통신 - 차량과 차량(V2V), 인프라(V2I), 보행자(V2P), 네트워크(V2N) 간 무선 통신
142. WAVE (Wireless Access in Vehicular Environments) - IEEE 802.11p 기반 근거리 전용 DSRC 자율주행 통신 (과거 표준)
143. C-V2X (Cellular V2X) - 3GPP 표준, LTE/5G 이동통신망 기반 차량 통신 (현대 대세 기술)
144. 플래투닝 (Platooning / 군집 주행) - 여러 대의 트럭이 통신으로 연결되어 좁은 간격으로 줄지어 동시 자율 주행 (연비 및 효율 향상)
145. UAM (Urban Air Mobility / 도심 항공 교통) - 에어 택시, 수직이착륙기(eVTOL)를 이용한 도심 3차원 교통 체계
146. K-UAM (한국형 도심항공교통) 그랜드 챌린지 및 회랑(Corridor) 설계
147. UTM (Unmanned Aircraft System Traffic Management) - 무인 비행체 교통 관제 시스템
148. 5G 통신망의 3대 초격차 특성 - eMBB (초고속), uRLLC (초저지연/고신뢰 1ms), mMTC (초연결 IoT)
149. 네트워크 슬라이싱 (Network Slicing) - 5G 핵심, 물리적 통신망 하나를 논리적으로 분할하여 자율주행(uRLLC 슬라이스), IoT(mMTC 슬라이스) 등에 맞춤형 보장 제공
150. 5G SA (Standalone) 아키텍처 - LTE 코어망(EPC)을 버리고 순수 5G 코어(5GC)로만 구성하여 진정한 5G 초저지연 성능 발휘
151. SBA (Service Based Architecture) - 5G 코어망 네트워크 기능(NF)들을 마이크로서비스 및 REST API 기반으로 클라우드 네이티브하게 모듈화
152. 6G 통신망 비전 - 테라헤르츠(THz) 대역 사용, Tbps 속도, 수중/우주 통신 커버리지 확장, AI 내재화 (AI-Native)
153. 지능형 반사 표면 (RIS, Reconfigurable Intelligent Surface) - 6G 메타물질, 전파의 반사/투과 방향을 인위적으로 조절하여 고주파(음영지역)의 커버리지 극복
154. 비지상 네트워크 (NTN, Non-Terrestrial Network) - 저궤도 위성(LEO), 성층권 드론(HAPS)을 기지국으로 활용하여 전 세계(바다, 사막)를 커버하는 6G 위성 통신 
155. 오픈 랜 (O-RAN, Open Radio Access Network) - 기지국 장비의 하드웨어와 소프트웨어를 분리하고 인터페이스를 개방형(오픈 API) 표준으로 전환하여 벤더 종속성(Lock-in) 탈피 (화이트박스 스위치 활용)
156. C-RAN (Cloud RAN) - 기지국의 데이터 처리부(BBU/DU)를 한 곳의 클라우드 센터에 모아 풀링(Pooling) 관리하고 안테나(RU)만 엣지에 분산 배치하는 구조 
157. 와이파이 6 (Wi-Fi 6 / IEEE 802.11ax) - OFDMA 및 양방향 MU-MIMO 도입, 밀집 지역 통신 지연 최소화
158. 와이파이 6E - 기존 2.4G/5G 외에 깨끗한 6GHz 대역(1.2GHz 폭)을 추가 사용
159. 와이파이 7 (IEEE 802.11be) - 320MHz 초광대역폭, 4K-QAM, MLO(Multi-Link Operation: 2.4/5/6GHz 동시 송수신) 도입 극초고속 무선랜
160. UWB (Ultra-Wideband / 초광대역) 통신 - 매우 짧은 임펄스(Impulse) 신호를 넒은 주파수에 걸쳐 송신, 수 센티미터 단위 정밀 위치 측정(스마트 태그, 디지털 키)
161. 스마트 그리드 (Smart Grid) - 전력망에 ICT를 접목하여 양방향으로 전력/정보를 교환, 에너지 효율 극대화
162. AMI (Advanced Metering Infrastructure) - 지능형 원격 검침 인프라
163. 마이크로그리드 (Microgrid) - 기존 광역 전력망과 독립적으로 분산 전원(태양광 등)과 ESS를 갖춘 소규모 지역 자급자족 전력망
164. ESS (Energy Storage System) - 남는 전력을 배터리에 저장했다가 피크 타임에 방전하는 에너지 저장 장치
165. V2G (Vehicle to Grid) - 전기차 배터리의 남는 전력을 전력망으로 역송전하여 전력 피크 부하를 줄이는 기술
166. 스마트 팩토리 (Smart Factory) 
167. CPS (Cyber-Physical System / 가상물리시스템) - 컴퓨팅 연산 체계(Cyber)가 물리(Physical) 공정 프로세스를 실시간 제어하고 피드백하는 시스템 (스마트팩토리 두뇌)
168. 산업용 이더넷 (Industrial Ethernet) 및 TSN (Time-Sensitive Networking) - 마이크로초 단위 데이터 전달 시간을 절대적으로 보장(결정론적)하는 산업 공장 통신망
169. OPC UA - 스마트 팩토리 기기 간 상호 운용성을 보장하는 산업 자동화 표준 프로토콜
170. 프라이빗 5G (특화망 / 이음5G) - 통신사가 아닌 일반 기업/공장이 자신의 건물 내에 직접 구축하고 특정 목적으로만 운영하는 사설 5G 네트워크
171. 스마트 시티 (Smart City) 플랫폼 아키텍처
172. 마스 (MaaS, Mobility as a Service) - 대중교통, 공유차량, 킥보드 등 모든 이동 수단을 하나의 플랫폼에서 통합 결제 및 최적 경로로 제공하는 서비스
173. C-ITS (협력형 지능형 교통 체계)
174. 엣지 AI (Edge AI) 기술 적용 아키텍처
175. 백스캐터 통신 (Ambient Backscatter Communication) - 배터리 없이 공기 중의 TV, Wi-Fi 전파 에너지를 흡수/반사하여 무전원으로 통신하는 초소형 IoT 기술
176. 웨어러블 디바이스 (Wearable Device) - 신체 부착형 기기 통신 체계 (WBAN, Body Area Network)
177. HMD (Head Mounted Display) 시선 추적(Eye Tracking), 포비티드 렌더링 (Foveated Rendering - 시선이 머무는 중심부만 초고화질 렌더링하여 GPU 부담 감소 기술)
178. 디지털 후각 / 촉각 텔레햅틱 (Tele-haptics) 통신 지연 제어
179. BCI (Brain-Computer Interface) - 뇌파를 직접 인식하여 기계를 제어하는 신경망 인터페이스 기술 (뉴럴링크)
180. 드론 스웜 (Drone Swarm) - 군집 비행 제어 및 충돌 회피 알고리즘

## 3. 클라우드 컴퓨팅 및 컴퓨팅 인프라 신기술 (80개)
181. 클라우드 컴퓨팅 (Cloud Computing) 5대 특징 (NIST) - 주문형 셀프 서비스, 광범위한 네트워크 접근, 자원 풀링, 신속한 탄력성, 측정 가능한 서비스
182. 클라우드 서비스 모델
183. IaaS (Infrastructure as a Service) - 서버, 스토리지, 네트워크 가상화 제공 (AWS EC2)
184. PaaS (Platform as a Service) - 개발 런타임, 데이터베이스, 배포 환경 제공 (AWS Elastic Beanstalk)
185. SaaS (Software as a Service) - 완제품 소프트웨어 제공 (Office 365, Salesforce)
186. BaaS (Backend as a Service) - 모바일/웹 앱용 백엔드(DB, 인증, 푸시) 제공 (Firebase)
187. FaaS (Function as a Service / Serverless) - 인프라 관리 없이 함수 단위로 코드만 배포 및 호출 시 과금 (AWS Lambda)
188. 클라우드 배포 모델 - 퍼블릭(Public), 프라이빗(Private), 하이브리드(Hybrid), 멀티 클라우드(Multi-Cloud)
189. 멀티 클라우드 (Multi-Cloud) 전략 - 특정 클라우드 종속성(Lock-in) 방지 및 최적화 조합 활용 (쿠버네티스 필수)
190. 가상화 (Virtualization) 컴퓨팅 아키텍처
191. 하이퍼바이저 (Hypervisor) Type 1 (베어메탈 - ESXi, Xen) vs Type 2 (호스트형 - VMware Workstation)
192. 전가상화 (Full Virtualization) vs 반가상화 (Para-virtualization)
193. 하드웨어 보조 가상화 (Intel VT-x)
194. 컨테이너 (Container) 가상화 - Guest OS 없이 호스트 OS 커널을 공유하며 프로세스 격리 (초경량, 초고속 기동)
195. 도커 (Docker) 아키텍처 - 도커 데몬, 도커 이미지(레이어 구조), 컨테이너, 레지스트리
196. 쿠버네티스 (Kubernetes, K8s) - 수백/수천 개의 컨테이너를 스케줄링, 배포, 로드밸런싱, 스케일링하는 오케스트레이션 도구
197. K8s 컴포넌트 - 마스터 노드 (API Server, etcd, Scheduler, Controller Manager), 워커 노드 (Kubelet, Kube-proxy, 컨테이너 런타임)
198. Pod (포드) - 쿠버네티스의 최소 배포 단위 (1개 이상의 컨테이너 그룹)
199. 클라우드 네이티브 (Cloud Native) 아키텍처 - 클라우드 환경의 이점을 극대화하기 위한 앱 구축/운영 방식 (마이크로서비스, 컨테이너, CI/CD, DevOps 결합)
200. 12-Factor App - SaaS 애플리케이션 개발을 위한 12가지 베스트 프랙티스 (코드베이스, 종속성 격리, 설정 외부화, 무상태 프로세스 등)
201. 서버리스 (Serverless) 아키텍처 한계점 분석 - 콜드 스타트(Cold Start: 함수 최초 호출 시 컨테이너 로딩 지연), 장기 실행 제약, 상태 비저장(Stateless)
202. 프로비저닝된 동시성 (Provisioned Concurrency) - 콜드 스타트 해결을 위해 유휴 컨테이너를 미리 예열해두는 서버리스 옵션
203. 인프라스트럭처 애즈 코드 (IaC, Infrastructure as Code) - 수동 UI 클릭 대신 코드(JSON, YAML, HCL)로 인프라를 정의/프로비저닝 (Terraform, Ansible)
204. 불변 인프라 (Immutable Infrastructure) - 서버 구성을 배포 후 수정하지 않고, 변경이 필요하면 새 이미지로 컨테이너/VM을 갈아치우는 패러다임 (구성 편류 Configuration Drift 방지)
205. 컨테이너 이미지 레이어 (Image Layer) 및 OCI (Open Container Initiative) 표준 규격
206. 쿠버네티스 오토스케일링 - HPA (Horizontal Pod Autoscaler - Pod 개수 증가), VPA (수직 스케일링), Cluster Autoscaler (워커 노드 자체 추가)
207. 헬름 (Helm) - 쿠버네티스 패키지 매니저
208. 서비스 디스커버리 (Service Discovery) 및 분산 클라우드 로드밸런싱
209. 스팟 인스턴스 (Spot Instance) - 클라우드 사업자의 남는 자원을 저렴하게 임대(갑자기 회수될 수 있음), 컨테이너/배치 처리와 결합하여 비용 최적화
210. FinOps (Cloud Financial Operations) - 기술, 비즈니스, 재무 팀이 협력하여 클라우드 비용을 투명하게 관리/최적화하는 클라우드 재무 관리 문화
211. 클라우드 마이그레이션 (Cloud Migration) 전략 - 6R (Rehost/Lift & Shift, Replatform, Repurchase, Refactor, Retire, Retain)
212. 리호스트 (Rehost / Lift and Shift) - 온프레미스 앱을 변경 없이 그대로 VM으로 마이그레이션
213. 리팩토링 (Refactor) - 클라우드 네이티브 성능(PaaS/Serverless)을 끌어내기 위해 애플리케이션 아키텍처(MSA 등)를 전면 수정
214. SDDC (Software Defined Data Center) - 컴퓨트, 스토리지, 네트워크 등 데이터센터의 모든 인프라를 가상화하여 소프트웨어로 통제
215. SDN (Software Defined Networking) - 제어 평면(Control Plane)과 데이터 평면(Data Plane)을 분리, 중앙 집중식 네트워크 컨트롤러 배포
216. SDS (Software Defined Storage) - 범용 x86 서버에 소프트웨어를 탑재하여 스토리지 기능(프로비저닝, 이중화) 구현 (Ceph 등)
217. 하이퍼컨버지드 인프라 (HCI, Hyper-Converged Infrastructure) - 서버, 스토리지, 네트워킹 가상화 및 관리 도구를 단일 x86 어플라이언스 박스로 통합 패키징하여 제공 (IT 인프라 단순화)
218. 양자 컴퓨팅 (Quantum Computing) 핵심 원리
219. 양자 중첩 (Superposition) - 0과 1의 상태가 동시에 존재하는 양자적 특성 (연산 속도 지수적 증가)
220. 양자 얽힘 (Entanglement) - 두 큐비트가 서로 멀리 떨어져 있어도 하나의 상태가 변하면 다른 하나도 즉시 결정되는 성질
221. 큐비트 (Qubit) - 양자 컴퓨터의 기본 정보 단위
222. 노이즈 있는 중간 규모 양자 기술 (NISQ, Noisy Intermediate-Scale Quantum) - 현재의 양자 컴퓨터 발전 단계 (오류 정정이 완벽하지 않음)
223. 양자 우위 (Quantum Supremacy) - 양자 컴퓨터가 슈퍼컴퓨터를 능가하여 특정 연산을 압도적 속도로 풀어낸 기점
224. 쇼어 알고리즘 (Shor's Algorithm) - 양자 푸리에 변환 적용 시 대용량 소인수분해를 다항 시간에 처리 (RSA 공개키 암호 붕괴 위협)
225. 그로버 알고리즘 (Grover's Algorithm) - 비정렬 데이터 검색 복잡도를 O(N)에서 O(√N)으로 단축 (AES 대칭키 탐색 위협)
226. 슈퍼컴퓨팅 / HPC (High Performance Computing) 인프라 아키텍처
227. MPI (Message Passing Interface) 기반 분산 클러스터 병렬 컴퓨팅
228. 엣지 컴퓨팅 하드웨어 오프로딩 (SmartNIC / DPU - Data Processing Unit)
229. DPU (Data Processing Unit) / IPU - 서버 CPU가 처리하던 네트워킹, 보안, 스토리지 패킷 처리 기능을 전용 반도체 카드로 오프로딩하여 CPU는 애플리케이션 연산에만 집중하게 하는 칩 (클라우드 인프라 가속기)
230. CXL (Compute Express Link) - PCIe 버스 기반 차세대 인터커넥트 기술, CPU-메모리-가속기 간 메모리 공유(풀링)와 캐시 일관성 보장
231. 메모리 풀링 (Memory Pooling) - CXL을 이용해 물리적으로 떨어진 서버 메모리를 거대한 단일 논리 메모리로 묶어 가상머신에 동적 할당
232. UCIe (Universal Chiplet Interconnect Express) - 칩렛(Chiplet) 간 상호 연결 통신 고속화 범용 표준
233. 칩렛 (Chiplet) 아키텍처 - 거대한 단일 칩(Monolithic) 대신 여러 개의 작은 기능별 칩 조각(Chiplet)을 따로 제조하여 하나의 패키지판(2.5D/3D 패키징)에 이어 붙여 수율 및 비용을 개선하는 최신 반도체 제조 공법
234. 뉴로모픽 반도체 (Neuromorphic Chip) - 폰 노이만 아키텍처 한계 극복, 인체 두뇌 신경망 스파이크 시그널을 하드웨어 회로로 모방 (초저전력 AI 연산)
235. 액침 냉각 (Immersion Cooling) - 데이터센터 발열 해소를 위해 서버 전체를 비전도성 특수 액체 냉매에 담가 식히는 고효율 친환경 냉각 기술
236. 콜드 아일 (Cold Aisle) / 핫 아일 (Hot Aisle) 차폐 데이터센터 공조 설계
237. PUE (Power Usage Effectiveness) - 데이터센터 전력 효율 지표 (총 소비 전력 / IT 장비 전력), 1에 가까울수록 고효율
238. 그린 IT / 탄소 인지 컴퓨팅 (Carbon-Aware Computing) 클라우드 전력 리전 스위칭
239. 베어메탈 클라우드 (Bare Metal Cloud) - 가상화 오버헤드 없이 물리 서버 자체를 클라우드처럼 임대 (DB, 고성능 연산용)
240. 클라우드 재해 복구 아키텍처 (파일럿 라이트 Pilot Light, 웜 스탠바이 Warm Standby 비교)
241. BDI (Bridge Domain Interface) VXLAN 오버레이 네트워크 인프라 연동
242. 분산 클라우드 (Distributed Cloud) - 퍼블릭 클라우드 서비스를 다양한 물리적 위치(고객사, 통신사 엣지)에 배포하면서 중앙 집중형 통제권은 CSP가 유지하는 모델
243. 마이크로VM (MicroVM, Firecracker) - 서버리스(FaaS) 환경에서 극강의 부팅 속도(수 밀리초)와 가상머신급 보안 격리를 동시 제공하는 초경량 하이퍼바이저 기술
244. gRPC 기반 서비스 메시 동기 통신망
245. RESTful API 성숙도 모델 (Richardson Maturity Model - Level 3 HATEOAS)
246. GraphQL - 클라이언트가 필요한 데이터 필드만 명시하여 한 번의 API 쿼리로 복합 데이터 수신 (오버패칭 Overfetching 문제 해결)
247. 오픈 API (Open API) 및 API 게이트웨이 보안 스로틀링 
248. 마이크로 프론트엔드 (Micro Frontends) 아키텍처 - UI 프론트엔드 파트도 독립적 배포 가능한 마이크로서비스 뷰로 분할 개발
249. 이벤트 소싱 (Event Sourcing) - 상태(State)의 최종 결과만 DB에 저장하는 게 아니라, 상태를 변화시킨 모든 '이벤트 이력'을 스트림으로 저장하여 불일치 방지 및 재생(Replay) 허용
250. CQRS (Command Query Responsibility Segregation) - 명령(Insert/Update) 모델과 조회(Select) 모델을 물리적/논리적으로 분리 분산 인프라 설계
251. 엣지 네이티브 (Edge Native) 설계 패턴
252. AIOps (Artificial Intelligence for IT Operations) - 클라우드 인프라/네트워크의 방대한 로그, 텔레메트리 데이터를 머신러닝으로 분석해 장애 전조를 예측하고 자동 치유(Auto-healing)하는 운영 자동화 
253. 디지털 전환 (DT/DX, Digital Transformation) - 기업 생태계 전반을 디지털 인프라 기반으로 혁신
254. 클라우드 락인 (Vendor Lock-in) 회피 기술 아키텍처
255. 데이터 파이프라인 (Apache Airflow) DAG (Directed Acyclic Graph) 배치 플로우 관리
256. 카프카 (Apache Kafka) - 대규모 실시간 로그 및 이벤트 스트리밍 처리를 위한 분산형 메시지 큐 (Pub/Sub 모델, 파티션 분산 저장)
257. 옵저버빌리티 (Observability) 도구 연동 (OpenTelemetry 오픈 표준)
258. 로우코드 / 노코드 (Low-Code / No-Code) 플랫폼 개발론 
259. 시민 개발자 (Citizen Developer) - IT 비전문가 현업 부서원이 로우코드 툴을 이용해 스스로 앱을 개발
260. 소프트웨어 정의 차량 (SDV, Software Defined Vehicle) - 테슬라 등 하드웨어 부품(ECU) 중심 통제에서 벗어나 스마트폰처럼 통합 OS 소프트웨어와 OTA 업데이트를 통해 차량 성능이 진화하는 구조

## 4. 인공지능 (AI) 심화 및 초거대 언어 모델 (LLM) 트렌드 (100개)
261. 파운데이션 모델 (Foundation Model) - 스탠퍼드 HAI 정의, 방대한 무라벨 데이터를 자기지도 학습하여 여러 다운스트림 태스크에 전이 가능한 거대 범용 모델 
262. 트랜스포머 (Transformer) 어텐션 기반 병렬 연산 구조
263. LLM (Large Language Model) - 수백억 이상의 파라미터를 가진 거대 언어 모델 (GPT-4, Gemini, Claude, Llama 3)
264. sLLM / SLM (Small Large Language Model) - 7B 이하의 경량화 파운데이션 모델, 엣지 기기 오프라인 구동 및 특정 도메인 파인튜닝용 (Llama-3-8B, Phi-3, Gemma 등)
265. 창발성 (Emergent Abilities) - 파라미터와 훈련 데이터 규모가 특정 임계치를 넘겼을 때, 사전에 프로그래밍/학습하지 않은 복잡한 추론 논리 능력이 스스로 발현되는 현상
266. 자기 지도 학습 (Self-Supervised Learning) - 다음 단어 예측(Next Token Prediction), 빈칸 채우기(MLM) 방식으로 사람이 라벨링하지 않아도 텍스트 구조 자체를 정답 삼아 자가 학습
267. 인스트럭션 튜닝 (Instruction Tuning) - 범용 LLM을 "인간의 명령(지시문)"과 "그에 따른 모범 답변" 데이터셋(Prompt-Response)으로 추가 지도 미세조정(SFT)하여 대화형 챗봇 형태로 가다듬는 과정
268. 인간 피드백 기반 강화학습 (RLHF, Reinforcement Learning from Human Feedback) - LLM이 내뱉은 여러 답변을 인간이 랭킹 매겨 보상 모델(Reward Model)을 훈련시키고, 이를 PPO 강화학습 알고리즘으로 본 모델에 적용하여 유해성/편향성을 통제 (정렬, Alignment 기법)
269. RLAIF (AI 피드백 기반 강화학습) - 인간 라벨러 대신 GPT-4 등 고성능 AI가 답변 품질 랭킹 채점을 대행 (헌법적 AI / Constitutional AI - Anthropic 제안)
270. DPO (Direct Preference Optimization) - 강화학습(PPO)과 보상 모델 구축의 복잡한 2단계를 생략하고, 선호/비선호 데이터셋으로 LLM을 직접 미세 조정하는 최신 최적화 기법
271. 프롬프트 엔지니어링 (Prompt Engineering) - LLM의 숨겨진 능력을 이끌어내기 위해 질문 컨텍스트를 구조화 설계
272. 제로샷 (Zero-shot) / 원샷 (One-shot) / 퓨샷 (Few-shot) 프롬프팅
273. 생각의 사슬 (CoT, Chain-of-Thought) - "단계별로 차근차근 생각해 보자"는 명령을 통해 중간 추론 과정(Reasoning Path)을 생성하게 유도하여 수학/논리 문제 정답률 극대화
274. ToT (Tree of Thoughts) / GoT (Graph of Thoughts) - 단일 사슬을 넘어 여러 추론 경로를 탐색/평가하며 분기하는 고도화 프롬프팅
275. 환각 (Hallucination / 할루시네이션) - LLM이 학습 데이터의 공백이나 패턴 오류로 인해 그럴싸한 허위 사실(거짓말)을 진실처럼 생성하는 한계점
276. RAG (Retrieval-Augmented Generation / 검색 증강 생성) - 할루시네이션 극복 아키텍처. 사용자 질문 수신 시 ①외부 사내 DB/문서에서 관련 문단 검색(Retrieve) -> ②검색된 문단을 프롬프트에 주입(Augment) -> ③LLM이 참조하여 답변 생성(Generate)
277. 벡터 데이터베이스 (Vector Database) - RAG 검색의 핵심, 텍스트/이미지를 다차원 임베딩 텐서로 변환 저장하고 코사인 유사도 연산으로 문맥적(Semantic) 의미가 가까운 문서를 빛의 속도로 추출 (Milvus, Pinecone 등)
278. 임베딩 (Embedding) - 비정형 데이터의 의미적 관계를 다차원 실수 배열(벡터) 공간에 좌표로 투영하는 변환 과정
279. 하이브리드 검색 (Hybrid Search) - 전통적인 키워드 정확 일치 검색(BM25)과 벡터 의미 유사도 검색(Dense)을 결합하여 검색 정확도 상호 보완
280. 시맨틱 캐시 (Semantic Cache) - 이전 질문의 의미 벡터값을 저장해두고, 유사한(정확히 일치하지 않아도) 질문이 오면 LLM API를 호출하지 않고 캐시에서 답변 즉시 반환 (지연시간 단축, 비용 절감)
281. 파인튜닝 (Fine-Tuning / 미세 조정) 전이 학습 방법론
282. 파라미터 효율적 미세 조정 (PEFT, Parameter-Efficient Fine-Tuning) - 수천억 파라미터 거대 모델 전체를 업데이트하기엔 GPU 자원이 부족하므로, 원래 가중치는 동결(Freeze)하고 극히 일부의 추가 파라미터 모듈만 훈련
283. 로라 (LoRA, Low-Rank Adaptation) - PEFT의 대표 기법. 거대 가중치 행렬 사이에 낮은 차원(Rank)의 랭크 분해 행렬 2개를 삽입하여 그것만 훈련시킨 후 나중에 원본 행렬에 더함 (VRAM 사용량 획기적 절약)
284. 양자화 (Quantization / QLoRA) - 모델 가중치의 정밀도를 FP32(32비트)에서 INT8, INT4 등 정수형으로 깎아내어 모델 용량과 추론 메모리를 대폭 압축 (모바일 온디바이스 탑재 핵심)
285. 지식 증류 (Knowledge Distillation) - 성능이 좋은 무거운 교사(Teacher) 모델의 산출물 확률 분포를 가벼운 학생(Student) 모델이 모방 학습하도록 하여 성능은 유지하되 모델 사이즈를 경량화
286. 멀티모달 (Multimodal) AI - 텍스트뿐만 아니라 이미지, 영상, 오디오 등 이기종 데이터를 동시에 입력받고 상호 교차 이해/생성하는 파운데이션 모델 (GPT-4o, Gemini)
287. 디퓨전 모델 (Diffusion Model) - 원본 이미지에 노이즈를 미세하게 반복 주입해 파괴(Forward)한 뒤, 그 노이즈를 단계적으로 제거(Denoising)하는 역과정을 신경망이 학습하여 텍스트로부터 고해상도 이미지를 생성 (Stable Diffusion, Midjourney)
288. LDM (Latent Diffusion Model) - 픽셀 공간이 아닌 압축된 잠재(Latent) 공간에서 디퓨전 연산을 수행해 이미지 생성 속도와 자원 효율 극대화
289. 생성적 적대 신경망 (GAN)과의 차이 (디퓨전은 생성 품질이 압도적, 속도는 GAN 우위)
290. 오토레그레시브 (Autoregressive) 생성 (트랜스포머 디코더 토큰 생성 메커니즘)
291. KV 캐시 (Key-Value Cache) - LLM 텍스트 추론 단계에서 매 토큰 생성 시 과거 텍스트의 Key, Value 어텐션 행렬 연산을 다시 하지 않고 메모리에 캐싱해두어 텍스트 생성 속도 가속화
292. 페이즈드 어텐션 (PagedAttention / vLLM) - OS의 가상 메모리 페이징 기법을 차용하여 VRAM 내 KV 캐시 파편화를 방지하고 연속 할당하여 GPU 메모리 활용률을 극대화하는 서빙 아키텍처 최적화 기법
293. AI 에이전트 (AI Agents) / 에이전틱 AI - 챗봇 수준을 넘어, 목표가 주어지면 LLM이 스스로 필요한 도구(웹 브라우저, 파이썬, API)를 선택해 실행하고 중간 결과를 바탕으로 다음 행동을 자율 계획(Planning)하여 임무를 완수하는 독립적 AI
294. 함수 호출 (Function Calling / Tool Use) - LLM이 JSON 포맷으로 외부 API 매개변수를 출력하면, 서버가 해당 API를 실행하고 결과를 다시 LLM에 주입하는 에이전트 핵심 기술
295. MoE (Mixture of Experts / 전문가 혼합 모델) 아키텍처 - 거대 신경망 내에 특정 분야별 소규모 전문가(Expert) 네트워크 여러 개를 병렬 배치. 라우팅 게이트 네트워크가 입력 질문에 따라 가장 적합한 전문가 1~2개만 활성화하여 연산 (파라미터는 거대하지만 실제 추론 연산량은 획기적으로 줄어드는 GPT-4 채용 추정 구조)
296. MLOps (Machine Learning Operations) - AI 모델 데이터 수집, 학습, 평가, 배포, 모니터링을 지속적 배포 파이프라인(CI/CD/CT)으로 묶는 운영 체계
297. LLMOps - 기존 MLOps에 더해 대규모 모델 튜닝(PEFT), 프롬프트 관리, RAG 파이프라인 지속 개선, 환각 모니터링 기능이 추가된 LLM 특화 운영 파이프라인
298. 모델 드리프트 (Model Drift / Data Drift) - 배포 후 입력 데이터의 통계적 분포나 환경이 변하여 AI 모델의 정확도가 시간이 지남에 따라 하락하는 현상 (재학습 트리거 원인)
299. 피처 스토어 (Feature Store) - 전처리된 머신러닝 피처 셋을 팀 간 공유하고 서빙 레이어에 고속 공급하기 위한 중앙 저장소
300. 벡터 인덱싱 (ANN: Approximate Nearest Neighbor) - 정확성 대신 탐색 속도를 취한 벡터 DB 인덱스 (HNSW, IVFFlat, PQ 등)
301. AI 안전 (AI Safety) 및 레드티밍 (Red Teaming) - AI 모델 배포 전 보안/윤리 결함을 의도적으로 찾아내기 위해 공격자 관점에서 취약점(탈옥 등)을 테스트하는 기법
302. 프롬프트 인젝션 (Prompt Injection / Jailbreak) 공격 - LLM의 행동 제어 프롬프트를 악의적 명령으로 무시하게 만들어 차별, 혐오, 기밀 코드를 내뱉게 하는 공격
303. 설명 가능한 AI (XAI, eXplainable AI) - 딥러닝 블랙박스 도출 결과의 논리적 근거(피처 기여도, 활성화 히트맵)를 인간이 이해할 수 있도록 시각화 및 수치화
304. LIME / SHAP 지표 (부분적 / 게임이론 전역적 변수 기여 해석)
305. 연합 학습 (Federated Learning) - 엣지 디바이스(스마트폰)에 원본 데이터를 남겨두고 로컬에서 학습한 가중치(Gradient) 업데이트 값만 클라우드 중앙 서버로 모아 병합하는 프라이버시 보존형 머신러닝 (데이터 유출 원천 차단)
306. 그래프 신경망 (GNN, Graph Neural Network) - 노드(Node)와 간선(Edge)의 네트워크 관계를 학습하는 딥러닝 구조 (분자 구조 분석, 소셜 네트워크 추천)
307. 트랜스포머 윈도우 한계 크기 (Context Window) 확장 기술 (RoPE, ALiBi 등 포지셔널 인코딩 개선)
308. 장기 문맥 (Long-context) 처리 - 1M 토큰(책 수권 분량) 동시 입력 처리 모델 아키텍처 구조 
309. 코사인 어닐링 (Cosine Annealing) 및 학습률 스케줄링 튜닝
310. AI 윤리, 편향성 (Bias), 저작권 (Copyright) 이슈 - 크롤링 데이터 공정 이용(Fair Use) 논란, 생성물 지식재산권 인정 불가 등 규제 거버넌스 동향 (EU AI Act 등)
311. 오디오/비디오 생성 AI - Sora (OpenAI 비디오 모델), 텍스트 투 스피치(TTS, 딥페이크 오디오 보이스 클로닝)
312. 클립 (CLIP, Contrastive Language-Image Pre-training) - 이미지와 그에 해당하는 텍스트 설명 쌍(Pair)을 같은 잠재 공간에 가깝게 임베딩하는 대조 학습(Contrastive Learning) 기법 (제로샷 이미지 분류 혁신)
313. 자율주행 강화학습 모방 학습 (Imitation Learning / Behavior Cloning)
314. 강화학습 벨만 방정식 (Bellman Equation) 및 상태-행동 가치함수(Q-Value)
315. 오토인코더 (Autoencoder) 구조 및 변이형 오토인코더 (VAE) 잠재 벡터 생성망
316. AI 반도체 생태계 - 텐서 코어(Tensor Core) GPU 가속, HBM (High Bandwidth Memory) 대역폭 병목 파훼
317. LPU (Language Processing Unit) - LLM 디코더의 순차적 토큰 생성 지연시간(Latency) 단축에 특화된 추론 전용 초고속 반도체 (Groq 등)
318. AI 컴파일러 (TensorRT, ONNX) 모델 최적화 및 타겟 하드웨어 런타임 변환 엔진
319. 데이터 파이프라인 (ETL) 내 비정형 데이터 OCR 및 LLM 문서 파싱 파이프라인 
320. 엣지 컴퓨팅 기반 온디바이스 SLM 구동 경량 아키텍처

## 5. 데이터 과학 (Data Science) 및 통계/최적화 (100개)
321. 데이터 마이닝 (Data Mining) 프레임워크 (CRISP-DM, KDD)
322. 교차 분석 (Cross Tabulation) / 카이제곱 검정 (Chi-Square Test) - 범주형 데이터 독립성 검정
323. T-검정 (t-Test) - 두 집단 간 평균 차이 검정 (단일 표본, 독립 표본, 대응 표본)
324. 분산 분석 (ANOVA) - 3개 이상 집단 평균 차이 비교 (F-분포)
325. 상관 분석 (Correlation Analysis) - 피어슨 상관계수(선형, 연속형), 스피어만 서열 상관계수(비모수)
326. 회귀 분석 (Regression Analysis) - 단순/다중 선형 회귀 모형
327. 최소 제곱법 (OLS, Ordinary Least Squares) - 잔차(오차) 제곱합 최소화 선형 방정식 도출
328. 결정 계수 (R-Squared, R^2) - 0~1 사이, 회귀 모형의 데이터 설명력 크기
329. 다중 공선성 (Multicollinearity) - 독립변수들 간 높은 상관관계로 회귀 계수 왜곡 (VIF 지수 10 이상 시 변수 제거)
330. 더미 변수 (Dummy Variable) - 범주형 -> 이진(0, 1) 변환
331. 로지스틱 회귀 (Logistic Regression) - 종속 변수가 이진(범주형)일 때, 오즈비(Odds Ratio)에 로그를 취해 선형 결합으로 매핑, 시그모이드(S자) 확률 반환
332. 최대 우도 추정법 (MLE, Maximum Likelihood Estimation)
333. 베이즈 정리 (Bayes' Theorem) - 사전 확률과 우도를 통해 사후 확률 계산
334. 마르코프 체인 (Markov Chain) - 미래 상태 확률이 오직 현재 상태에만 의존하는 확률적 상태 전이 과정
335. 중심 극한 정리 (CLT, Central Limit Theorem) - 표본의 크기(n)가 커지면 모집단 분포 무관하게 표본 평균의 분포가 정규 분포에 수렴
336. 1종 오류 (Type I Error, 알파) / 2종 오류 (Type II Error, 베타)
337. 유의 확률 (p-value) - 귀무가설이 맞다고 가정할 때 관측치 이상 극단값이 나올 확률 (유의수준 0.05보다 작으면 귀무가설 기각)
338. 차원 축소법 PCA (주성분 분석) - 데이터 분산을 최대로 보존하는 직교 축 도출
339. 차원 축소법 LDA (선형 판별 분석) - 클래스 간 분산을 최대화하고 클래스 내 분산을 최소화하는 지도 학습
340. t-SNE / UMAP - 고차원 데이터의 비선형적 이웃 관계를 저차원에 보존하는 시각화 특화 차원 축소 기법
341. 시계열 분석 - AR (자기회귀), MA (이동평균), ARMA 모델
342. ARIMA (Auto-Regressive Integrated Moving Average) - 차분(Integrated)을 통해 비정상 시계열을 정상성 시계열로 변환하여 예측
343. 정상성 (Stationarity) 검정 (ADF Test) - 평균과 분산이 시간에 따라 일정한 시계열 속성
344. 지수 평활법 (Exponential Smoothing) / 이동 평균
345. 협업 필터링 (Collaborative Filtering) - 사용자 기반 / 아이템 기반 추천 시스템 핵심 
346. 콘텐츠 기반 필터링 (Content-based Filtering) - 아이템의 메타 속성 분석 추천
347. 콜드 스타트 (Cold Start) 문제 - 추천 시스템에서 신규 유저/아이템에 대한 정보 부족으로 추천 불가 현상
348. 행렬 분해 (Matrix Factorization) - 거대 사용자-아이템 평점 행렬을 잠재 요인(Latent Factor) 행렬 두 개로 쪼개어 빈 공간(예측 평점) 추론
349. SVD (특이값 분해) 및 ALS (Alternating Least Squares) 추천 알고리즘 연산
350. K-Means 클러스터링의 엘보우 기법 (Elbow Method) / 실루엣 계수 (Silhouette Score) 최적 K값 탐색
351. DBSCAN - 밀도 기반 클러스터링 (원 모양이 아닌 불규칙한 모양 군집, 노이즈/이상치 판별 가능)
352. K-최근접 이웃 (K-NN) / 거리 척도 - 유클리디안 (L2), 맨해튼 (L1), 민코프스키, 마할라노비스 거리
353. 랜덤 포레스트 (Random Forest) - 다수의 결정 트리를 배깅(Bagging)과 피처 무작위 선택으로 결합하여 과적합 억제
354. 부스팅 기법 (AdaBoost, GBM) - 트리 오차 수정 반복 연산
355. XGBoost / LightGBM - 분류/회귀 대회의 우승 단골 부스팅 모델, 정규화(L1/L2) 도입 및 병렬 트리 생성으로 GBM 속도/과적합 개선
356. 불균형 데이터 처리 (Imbalanced Data) - 언더샘플링 vs 오버샘플링
357. SMOTE (Synthetic Minority Over-sampling Technique) - 소수 클래스의 데이터 포인트를 K-NN 기반으로 선형 보간하여 합성 생성 데이터 증강
358. TF-IDF (Term Frequency - Inverse Document Frequency) - 텍스트 단어 중요도 가중치. 특정 문서에 많이 나올수록(TF 증가), 전체 문서군에 드물게 나올수록(IDF 증가) 가중치 높음
359. 코사인 유사도 (Cosine Similarity) - 텍스트 벡터 간 각도 측정 (-1 ~ 1)
360. 앙상블 편향-분산 수식 증명 (배깅은 분산 감소, 부스팅은 편향 감소)
361. 그래디언트 소실 (Vanishing Gradient) 대비 가중치 초기화 (Xavier, He)
362. 서포트 벡터 머신 (SVM)의 하드 마진 (Hard Margin) vs 소프트 마진 (Soft Margin / 슬랙 변수 허용)
363. 엔트로피 (Entropy) - 불확실성 정보량 척도 (결정 트리 정보 획득량 계산)
364. 베이즈 정리 사후 확률 갱신 구조망 파악
365. 결정 계수(R^2)와 조정된 결정 계수(Adjusted R^2) 비교
