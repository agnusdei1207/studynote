# 빅데이터 5V 모델 (3V + Veracity + Value)

## 정의

**빅데이터 5V 모델**은 Doug Laney의 3V(Volume, Velocity, Variety) 모델을 확장하여 **Veracity(정확성)**와 **Value(가치)**를 추가한 빅데이터 정의 프레임워크다. 3V가 데이터의 기술적 특성을 기술하는 데 반면, 5V 모델은 데이터의 **품질(Veracity)**과 **비즈니스 임팩트(Value)**를 명시적으로 포함하여 빅데이터의 **실용적 가치**를 강조한다. IBM과 다른 주요 기업들이 5V 모델을 적극 수용하면서 빅데이터 도입의 비즈니스 정당성을 확보하는 데 활용하고 있다.

```
+------------------------------------------------------------------+
|                    빅데이터 5V 모델 확장                           |
+------------------------------------------------------------------+
|                                                                   |
|                         +-----------+                             |
|                        /    Value    \     <- 비즈니스 가치       |
|                       /     (가치)     \                          |
|                      +-------------------+                        |
|                     /                     \                       |
|                    /    Veracity           \   <- 데이터 품질     |
|                   /      (정확성)           \                     |
|                  +---------------------------+                    |
|                 /                             \                   |
|                /            5V                \                  |
|               +-------------------------------+                  |
|              /                                 \                 |
|             /         +-----------+             \                |
|            +----------+-----------+----------+                   |
|            |  Volume  |  Velocity |  Variety |                   |
|            |  (규모)  |  (속도)   | (다양성) |                   |
|            +----------+-----------+----------+                   |
|                                                                   |
|  "3V: 기술적 특성 -> 5V: 기술 + 품질 + 가치"                     |
+------------------------------------------------------------------+
```

### 5V vs 3V: 확장의 필요성

```
+------------------------------------------------------------------+
|                    WHY 5V OVER 3V?                                 |
+------------------------------------------------------------------+
|                                                                   |
|  [3V 모델의 한계]                                                 |
|  +-------------------------------------------------------------+  |
|  |                                                             |  |
|  |  1. 품질 간과                                               |  |
|  |     - 3V는 데이터 양/속도/형태만 다룸                       |  |
|  |     - "Garbage In, Garbage Out" 문제 미해결                 |  |
|  |                                                             |  |
|  |  2. 비즈니스 연결 부재                                      |  |
```

## 관련 키워드

- **3V 모델**: Volume, Velocity, Variety (Doug Laney, 2001)
- **7V 모델**: 5V + Visualization + Variability
- **데이터 품질 관리 (DQM)**: 정확성, 완전성, 일관성, 적시성, 유일성, 유효성
- **DAMA-DMBOK**: 데이터 관리 지식 체계
- **Gartner Analytics Maturity Model**: 4단계 분석 성숙도 모델
- **Descriptive Analytics**: 기술적 분석 (BI, Reporting)
- **Diagnostic Analytics**: 진단적 분석 (Root Cause Analysis)
- **Predictive Analytics**: 예측적 분석 (ML, Statistical Modeling)
- **Prescriptive Analytics**: 처방적 분석 (Optimization, Decision Science)
- **Great Expectations**: 오픈소스 데이터 품질 검증 도구
- **Deequ**: AWS 기반 데이터 품질 라이브러리
- **데이터 옵저버빌리티 (Data Observability)**: 데이터 품질 실시간 모니터링
- **데이터 계보 (Data Lineage)**: 데이터 이력 및 영향 분석
- **ROI (Return on Investment)**: 투자 대비 수익률
- **TCO (Total Cost of Ownership)**: 총 소유 비용
