### Ⅰ. 개요

#### 1. 개념
메모리 관리는 **운영체제가 한정된 물리 메모리를 프로세스에 효율적으로 할당하고 회수하는 기법**이다. 가상 메모리를 통해 물리 메모리보다 큰 주소 공간을 제공한다.

> 비유: "도서관 자리 배정" - 한정된 자리를 효율적으로 써요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 메모리 관리 기법
```
┌────────────────────────────────────────────────────────┐
│           메모리 관리 기법                             │
├────────────────────────────────────────────────────────┤
│                                                        │
│  📦 연속 할당:                                         │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 고정 분할: 크기 고정, 내부 단편화           │   │
│  │  • 동적 분할: 가변 크기, 외부 단편화           │   │
│  │  • 버디 시스템: 2의 거듭제곱 단위              │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  📄 불연속 할당:                                       │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 페이징: 고정 크기 페이지                    │   │
│  │  • 세그멘테이션: 논리적 단위                   │   │
│  │  • 페이지드 세그멘테이션: 혼합                 │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  💾 가상 메모리:                                       │
│  ┌────────────────────────────────────────────────┐   │
│  │  • Demand Paging: 필요시 로드                  │   │
│  │  • 페이지 교체: FIFO, LRU, OPT                 │   │
│  │  • 스와핑: 디스크와 메모리 교환                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 페이지 교체 알고리즘
| 알고리즘 | 설명 | 성능 |
|---------|------|------|
| FIFO | 먼저 들어온 것 교체 | 단순, Belady |
| OPT | 미래에 안 쓸 것 | 최적, 구현 불가 |
| LRU | 최근에 안 쓴 것 | 좋음, 오버헤드 |
| Clock | 참조 비트 활용 | LRU 근사 |
| LFU | 사용 빈도 낮은 것 | 지역성 고려 |

#### 4. 단편화
| 유형 | 설명 | 해결 |
|-----|------|------|
| 내부 단편화 | 할당 > 사용 | 슬랙 감소 |
| 외부 단편화 | 분산된 빈 공간 | 압축, 페이징 |

#### 5. 메모리 계층
| 계층 | 속도 | 용량 | 비용 |
|-----|------|------|------|
| 레지스터 | 가장 빠름 | KB | 가장 높음 |
| L1 캐시 | 매우 빠름 | KB | 높음 |
| L2/L3 캐시 | 빠름 | MB | 중간 |
| RAM | 보통 | GB | 낮음 |
| 디스크 | 느림 | TB | 가장 낮음 |
