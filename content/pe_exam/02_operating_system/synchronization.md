### Ⅰ. 개요

#### 1. 개념
프로세스 동기화는 **여러 프로세스나 스레드가 공유 자원에 접근할 때 데이터 일관성을 보장하기 위한 기법**이다. 경쟁 상태(Race Condition)를 방지하고 임계 영역을 보호한다.

> 비유: "화장실 열쇠" - 한 번에 한 사람만 사용할 수 있어요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 동기화 문제
```
┌────────────────────────────────────────────────────────┐
│           동기화 문제 상황                             │
├────────────────────────────────────────────────────────┤
│                                                        │
│  ⚠️ 경쟁 상태 (Race Condition):                       │
│  ┌────────────────────────────────────────────────┐   │
│  │  Thread A: read count (0)                      │   │
│  │  Thread B: read count (0)  ← 동시 읽기!        │   │
│  │  Thread A: count = 0 + 1 = 1                   │   │
│  │  Thread B: count = 0 + 1 = 1  ← 덮어쓰기!      │   │
│  │  결과: count = 1 (기대: 2)                     │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🔒 임계 영역 (Critical Section):                     │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 공유 자원에 접근하는 코드 영역               │   │
│  │  • 한 번에 하나의 프로세스만 진입 가능          │   │
│  │  • 상호 배제(Mutual Exclusion) 필요             │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 4. 세마포어 연산
| 연산 | 설명 |
|-----|------|
| P(wait) | S--; if S<0 then block |
| V(signal) | S++; if S<=0 then wakeup |

#### 5. 동기화 메커니즘
| 메커니즘 | 장점 | 단점 |
|---------|------|------|
| Mutex | 단순, 효율 | 소유권 필요 |
| Semaphore | 유연, 자원 관리 | 데드락 위험 |
| Monitor | 안전, 구조화 | 언어 지원 필요 |
| Condition Variable | 유연한 대기 | 복잡한 사용 |

#### 6. 고전적 동기화 문제
| 문제 | 설명 |
|-----|------|
| 생산자-소비자 | 버퍼 공유 |
| 독자-저자 | 읽기/쓰기 권한 |
| 식사하는 철학자 | 원형 대기 |
| 흡연자 문제 | 자원 조합 |

### Ⅲ. 기술 비교 분석

#### 3. 동기화 도구 비교
| 도구 | 타입 | 값 범위 | 용도 | 복잡도 |
|-----|------|--------|------|--------|
| 뮤텍스 | 이진 | 0, 1 | 상호 배제 | 낮음 |
| 세마포어 | 카운팅 | 0~N | 자원 관리 | 중간 |
| 모니터 | 고수준 | - | 캡슐화 | 높음 |
| 스핀락 | 바쁜 대기 | - | 짧은 대기 | 낮음 |

#### 7. 장단점
| 장점 | 단점 |
|-----|------|
| 데이터 일관성 | 성능 저하 |
| 예측 가능한 실행 | 데드락 위험 |
| 버그 예방 | 구현 복잡성 |

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|

#### 결론
> **프로세스 동기화 (Process Synchronization)**은(는) 운영체제의 핵심 메커니즘들은 클라우드·컨테이너 시대에도 동일한 원리로 작동하며, eBPF 등 새로운 커널 확장 기술과 결합하여 더 유연하고 관측 가능한 시스템으로 진화하고 있다.
