### Ⅰ. 개요

#### 1. 개념
파일 시스템은 **운영체제가 저장장치의 데이터를 파일 단위로 체계적으로 관리하는 방법**이다. 파일 생성, 삭제, 읽기, 쓰기 등의 기능을 제공하며 디스크 공간을 효율적으로 관리한다.

> 비유: "도서관 분류 시스템" - 책들을 체계적으로 정리해요

#### 3. 디스크 할당 방식
| 방식 | 설명 | 장점 | 단점 |
|-----|------|------|------|
| 연속 할당 | 연속된 블록 | 빠른 접근 | 외부 단편화 |
| 연결 할당 | 링크드 리스트 | 외부 단편화 없음 | 순차 접근만 |
| 색인 할당 | inode 사용 | 직접 접근 | 오버헤드 |
| FAT | 파일 할당 테이블 | 단순 | 테이블 크기 |

#### 4. Inode 구조
| 구성 요소 | 설명 |
|----------|------|
| 메타데이터 | 권한, 소유자, 크기 |
| 직접 블록 | 12개 직접 참조 |
| 간접 블록 | 1단계 간접 |
| 이중 간접 | 2단계 간접 |
| 삼중 간접 | 3단계 간접 |

#### 5. 디렉터리 구조
| 구조 | 설명 |
|-----|------|
| 단일 레벨 | 하나의 디렉터리 |
| 2단계 | 사용자별 디렉터리 |
| 트리 구조 | 계층적 디렉터리 |
| 비순환 그래프 | 심볼릭 링크 |

#### 6. 파일 시스템 종류
| 파일 시스템 | OS | 특징 |
|------------|-----|------|
| ext4 | Linux | 저널링, 대용량 |
| NTFS | Windows | ACL, 압축 |
| APFS | macOS | 스냅샷, 암호화 |
| FAT32 | 범용 | 호환성, 4GB 제한 |
| ZFS | Unix | 풀, 중복 제거 |
