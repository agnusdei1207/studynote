### Ⅰ. 개요

#### 1. 개념
데이터베이스 트랜잭션은 **하나의 논리적 작업 단위를 구성하는 연산들의 집합**으로, 모두 성공하거나 모두 실패해야 하는 원자성을 갖는다. 데이터베이스의 일관성을 보장하는 핵심 메커니즘이다.

> 비유: "은행 이체" - 출금과 입금이 하나의 단위로 처리돼요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. ACID 속성
```
┌────────────────────────────────────────────────────────┐
│           ACID 속성                                    │
├────────────────────────────────────────────────────────┤
│                                                        │
│  🔴 Atomicity (원자성):                                │
│  ┌────────────────────────────────────────────────┐   │
│  │  • All or Nothing                              │   │
│  │  • 전부 성공 또는 전부 실패                      │   │
│  │  • 실패 시 롤백                                 │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🟠 Consistency (일관성):                              │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 트랜잭션 전후 상태 일관성 유지                │   │
│  │  • 무결성 제약 조건 준수                        │   │
│  │  • 데이터 유효성 보장                           │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🟡 Isolation (격리성):                                │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 동시 실행 트랜잭션 간 간섭 없음               │   │
│  │  • 격리 수준에 따른 가시성 제어                  │   │
│  │  • 동시성 문제 방지                             │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🟢 Durability (지속성):                               │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 커밋된 트랜잭션은 영구적                      │   │
│  │  • 시스템 장애 후에도 복구 가능                  │   │
│  │  • 로그를 통한 복구                             │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 트랜잭션 상태
| 상태 | 설명 |
|-----|------|
| Active | 실행 중 |
| Partially Committed | 마지막 연산 완료 |
| Committed | 완료됨 |
| Failed | 실패 |
| Aborted | 철회됨 |

#### 4. 격리 수준
| 수준 | Dirty Read | Non-Repeatable | Phantom |
|-----|-----------|----------------|---------|
| Read Uncommitted | O | O | O |
| Read Committed | X | O | O |
| Repeatable Read | X | X | O |
| Serializable | X | X | X |

#### 5. 동시성 문제
| 문제 | 설명 |
|-----|------|
| Dirty Read | 미커밋 데이터 읽기 |
| Non-Repeatable Read | 같은 쿼리 다른 결과 |
| Phantom Read | 새로운 행 출현 |
| Lost Update | 갱신 손실 |

#### 6. Lock 종류
| Lock | 설명 |
|------|------|
| Shared Lock (S) | 읽기 잠금 |
| Exclusive Lock (X) | 쓰기 잠금 |
| Update Lock (U) | 갱신 잠금 |
| Intent Lock | 의도 잠금 |
