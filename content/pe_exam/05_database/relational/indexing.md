# 데이터베이스 인덱싱 (Database Indexing)

## 핵심 인사이트 (3줄 요약)

---

### Ⅰ. 개요

#### 1. 개념
데이터베이스 인덱스는 **테이블의 데이터를 빠르게 검색하기 위해 생성하는 별도의 자료구조**다. 책의 색인처럼 원하는 데이터의 위치를 빠르게 찾을 수 있게 한다.

> 비유: "책의 색인" - 페이지를 찾기 쉬워요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 인덱스 구조
```
┌────────────────────────────────────────────────────────┐
│           인덱스 동작 원리                             │
├────────────────────────────────────────────────────────┤
│                                                        │
│  📚 테이블 스캔 (인덱스 없음):                         │
│  ┌────────────────────────────────────────────────┐   │
│  │  Row 1 → Row 2 → Row 3 → ... → Row 1000000    │   │
│  │  시간 복잡도: O(n)                             │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🔍 인덱스 사용:                                       │
│  ┌────────────────────────────────────────────────┐   │
│  │  B-Tree 검색: 루트 → 중간 → 리프               │   │
│  │  시간 복잡도: O(log n)                         │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  📊 B-Tree 구조:                                       │
│  ┌────────────────────────────────────────────────┐   │
│  │              [50]                              │   │
│  │             /     \                            │   │
│  │        [25]       [75]                         │   │
│  │       /    \     /    \                        │   │
│  │   [10,20] [30,40] [60,70] [80,90]              │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 인덱스 종류
| 종류 | 구조 | 용도 | 특징 |
|-----|------|------|------|
| B-Tree | 트리 | 범위 검색 | 범용 |
| Hash | 해시 | 등가 검색 | 빠름 |
| Bitmap | 비트맵 | 저카디널리티 | OLAP |
| Clustered | 정렬 | PK | 물리 순서 |
| Covering | 복합 | 포함 | 빠름 |
| Full-text | 역색인 | 텍스트 검색 | 검색 엔진 |

#### 4. 인덱스 속성
| 속성 | 설명 |
|-----|------|
| Unique | 중복 불가 |
| Primary | 기본 키 |
| Composite | 다중 컬럼 |
| Partial | 조건부 |
| Expression | 함수 기반 |

#### 5. 인덱스 선정 기준
| 고려사항 | 권장 |
|---------|------|
| 카디널리티 | 높을수록 좋음 |
| 선택도 | 낮을수록 좋음 |
| WHERE 절 | 자주 사용 |
| JOIN | 외래 키 |
| ORDER BY | 정렬 컬럼 |

#### 6. 인덱스 비용
| 장점 | 단점 |
|-----|------|
| 검색 빠름 | 저장 공간 |
| 정렬 빠름 | 삽입 느림 |
| JOIN 빠름 | 갱신 느림 |
| | 삭제 느림 |

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **데이터 무결성** | ACID 트랜잭션·정규화로 데이터 정합성 보장 | 데이터 이상 현상(Anomaly) 100% 방지 |

#### 결론
> **데이터베이스 인덱싱 (Database Indexing)**은(는) 데이터베이스는 HTAP(하이브리드 거래·분석 처리)와 AI 통합(벡터 DB, RAG 파이프라인)으로 진화하며, 단순 저장소를 넘어 비즈니스 인텔리전스의 핵심 엔진이 될 것이다.
