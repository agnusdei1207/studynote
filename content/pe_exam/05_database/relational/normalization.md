### Ⅰ. 개요

#### 1. 개념
데이터베이스 정규화는 **데이터의 중복을 최소화하고 무결성을 유지하기 위해 테이블을 분해하는 과정**이다. 이상 현상(Anomaly)을 방지하고 효율적인 저장 구조를 만든다.

> 비유: "옷장 정리" - 같은 옷은 한 곳에 모아요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 이상 현상 (Anomaly)
```
┌────────────────────────────────────────────────────────┐
│           이상 현상                                    │
├────────────────────────────────────────────────────────┤
│                                                        │
│  ⚠️ 삽입 이상 (Insertion Anomaly):                    │
│  ┌────────────────────────────────────────────────┐   │
│  │  문제: 학생 정보 없이 과목만 등록 불가          │   │
│  │  원인: 불필요한 속성이 NOT NULL                 │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  ⚠️ 삭제 이상 (Deletion Anomaly):                     │
│  ┌────────────────────────────────────────────────┐   │
│  │  문제: 마지막 수강 정보 삭제 시 학생 정보도 삭제 │   │
│  │  원인: 한 튜플에 여러 정보 혼재                  │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  ⚠️ 갱신 이상 (Update Anomaly):                       │
│  ┌────────────────────────────────────────────────┐   │
│  │  문제: 학생 주소 변경 시 여러 행을 모두 수정     │   │
│  │  원인: 데이터 중복                              │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 정규형 단계
| 정규형 | 조건 | 해결 |
|-------|------|------|
| 1NF | 원자성 | 복합 속성 분해 |
| 2NF | 부분 함수 종속 제거 | 완전 함수 종속 |
| 3NF | 이행 함수 종속 제거 | 직접 종속 |
| BCNF | 결정자 함수 종속 | 모든 결정자 후보키 |
| 4NF | 다치 종속 제거 | 독립 다치 종속 |
| 5NF | 조인 종속 제거 | 무손실 분해 |

#### 4. 함수 종속
| 종류 | 표기 | 설명 |
|-----|------|------|
| 완전 함수 종속 | A → B | A 전체가 B 결정 |
| 부분 함수 종속 | A의 부분집합 → B | 일부만으로 결정 |
| 이행 함수 종속 | A → B → C | 간접 결정 |

#### 5. 정규화 예시
| 단계 | Before | After |
|-----|--------|-------|
| 1NF | 주소(시, 구) | 시, 구 분리 |
| 2NF | (학번, 과목) → 학과 | 학번 → 학과 분리 |
| 3NF | 학번 → 학과 → 학과장 | 학과 테이블 분리 |

#### 6. 역정규화
| 이유 | 방법 |
|-----|------|
| 성능 향상 | 테이블 병합 |
| 조인 감소 | 중복 허용 |
| 읽기 최적화 | 파생 컬럼 |
