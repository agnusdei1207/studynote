### Ⅰ. 개요

#### 1. 개념
쿼리 최적화는 **SQL 문을 가장 효율적으로 실행하기 위한 최적의 실행 계획을 수립하는 과정**이다. 옵티마이저가 다양한 실행 방법 중 비용이 가장 낮은 방법을 선택한다.

> 비유: "네비게이션 경로 선택" - 가장 빠른 길을 찾아요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 옵티마이저 종류
```
┌────────────────────────────────────────────────────────┐
│           쿼리 옵티마이저                              │
├────────────────────────────────────────────────────────┤
│                                                        │
│  📏 규칙 기반 (Rule-Based Optimizer, RBO):            │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 미리 정의된 규칙 사용                        │   │
│  │  • 인덱스 존재 시 인덱스 사용                   │   │
│  │  • 통계 정보 미사용                             │   │
│  │  • 단순하지만 비효율적 가능                     │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  💰 비용 기반 (Cost-Based Optimizer, CBO):            │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 통계 정보 기반 비용 계산                     │   │
│  │  • I/O, CPU, 메모리 고려                       │   │
│  │  • 동적 결정                                    │   │
│  │  • 현대 DBMS 표준                               │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  📊 비용 계산 요소:                                    │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 디스크 I/O 비용                              │   │
│  │  • CPU 처리 비용                                │   │
│  │  • 네트워크 비용 (분산)                         │   │
│  │  • 메모리 사용량                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 실행 계획 연산
| 연산 | 설명 | 비용 |
|-----|------|------|
| Table Scan | 전체 스캔 | 높음 |
| Index Scan | 인덱스 사용 | 낮음 |
| Index Seek | 인덱스 직접 | 매우 낮음 |
| Nested Loop | 중첩 반복 | 높음 |
| Hash Join | 해시 조인 | 중간 |
| Merge Join | 병합 조인 | 낮음 |

#### 4. 조인 알고리즘
| 방식 | 설명 | 적합한 경우 |
|-----|------|------------|
| Nested Loop | 행별 반복 | 소규모 |
| Hash Join | 해시 테이블 | 대규모 |
| Sort Merge | 정렬 후 병합 | 정렬됨 |

#### 5. 통계 정보
| 정보 | 설명 |
|-----|------|
| 행 수 | 테이블 크기 |
| 카디널리티 | 고유 값 수 |
| 분포도 | 값 분포 |
| 히스토그램 | 값 빈도 |

#### 6. 최적화 기법
| 기법 | 설명 |
|-----|------|
| 조건 순서 변경 | 선택적 조건 우선 |
| 인덱스 활용 | 적절한 인덱스 |
| 조인 순서 | 작은 테이블 우선 |
| 서브쿼리 변환 | JOIN으로 변경 |
