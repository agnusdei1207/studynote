# 인터럽트 (Interrupt)

## 핵심 인사이트 (3줄 요약)
> CPU가 **외부/내부 이벤트에 비동기적으로 응답**하는 메커니즘. 폴링(Polling) 대비 효율적이며, 입출력, 예외, 타이머 등을 처리한다. 인터럽트 벡터와 서비스 루틴(ISR)으로 구성된다.

### Ⅰ. 개요

#### 1. 개념
인터럽트(Interrupt)는 **CPU에게 급한 일이 생겼음을 알리고 처리를 요청**하는 신호다. CPU는 하던 일을 멈추고 인터럽트 서비스 루틴(ISR)을 실행한다.

> 비유: "수업 중 교실 비상벨" - 하던 일을 멈추고 대피

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 인터럽트 종류
| 종류 | 원인 | 예시 |
|------|------|------|
| 하드웨어 | 외부 장치 | 키보드, 타이머 |
| 소프트웨어 | 프로그램 | 시스템 콜 |
| 예외 | 실행 오류 | 0으로 나누기 |
| 트랩 | 의도적 | 디버깅 |

#### 3. 인터럽트 처리 과정
```
1. 인터럽트 요청 (IRQ)
   ┌─────────┐
   │  키보드  │ ──── 인터럽트 신호 ────→
   └─────────┘

2. CPU 현재 상태 저장
   PC, 레지스터 → 스택

3. 인터럽트 벡터 확인
   벡터 테이블에서 ISR 주소 찾기

4. ISR 실행
   인터럽트 서비스 루틴 수행

5. 상태 복원
   스택 → PC, 레지스터

6. 원래 작업 재개
```

#### 5. 인터럽트 벡터 테이블
```
┌────────┬─────────────────┐
│ 번호   │ ISR 주소         │
├────────┼─────────────────┤
│   0    │ 0x0000 (리셋)   │
│   1    │ 0x0004 (NMI)    │
│   2    │ 0x0008 (HardFault)│
│  ...   │ ...             │
│  100   │ 0x0190 (UART)   │
└────────┴─────────────────┘

CPU는 인터럽트 번호로 ISR 주소를 찾음
```

#### 6. 인터럽트 우선순위
```
우선순위 예시:
1. 리셋 (최고)
2. NMI (Non-Maskable Interrupt)
3. 하드 폴트
4. 타이머
5. UART
6. GPIO (최저)

높은 우선순위 인터럽트가 낮은 것을 선점 가능
→ 중첩 인터럽트 (Nested Interrupt)
```

#### 7. ISR 작성 예시
```c
// ARM Cortex-M 인터럽트
void USART1_IRQHandler(void) {
    if (USART1->SR & USART_SR_RXNE) {
        // 수신 데이터 있음
        char data = USART1->DR;
        process_data(data);
    }
}

// 인터럽트 활성화
void enable_uart_interrupt(void) {
    USART1->CR1 |= USART_CR1_RXNEIE;  // 수신 인터럽트 활성화
    NVIC_EnableIRQ(USART1_IRQn);      // NVIC에서 활성화
}
```

#### 8. 인터럽트 지연 (Latency)
```
인터럽트 지연 시간:
- 하드웨어 지연: 신호 전달 시간
- 소프트웨어 지연: 현재 명령어 완료 대기
- ISR 진입 시간: 컨텍스트 저장

실시간 시스템에서 중요한 지표
```

### Ⅲ. 기술 비교 분석

#### 4. 인터럽트 vs 폴링
```
폴링 (Polling):
while(1) {
    if (키보드_입력_있음()) {
        처리();
    }
    if (마우스_입력_있음()) {
        처리();
    }
    // 계속 확인...
}

→ CPU가 계속 확인해야 함 (비효율)

인터럽트:
// 메인 프로그램
while(1) {
    // 다른 일 함
}

// 인터럽트 발생 시 자동 호출
ISR_키보드() {
    키보드_처리();
}

→ 이벤트 발생 시에만 CPU 사용 (효율)
```

#### 9. 장단점
| 장점 | 단점 |
|-----|------|
| CPU 효율성 | 오버헤드 |
| 실시간 응답 | 복잡한 설계 |
| 폴링 대비 저전력 | 우선순위 관리 |

### Ⅳ. 실무 적용 방안

#### 10. 실무에선? (기술사적 판단)
- **RTOS**: 인터럽트 지연이 실시간성 결정
- **임베디드**: 모든 I/O를 인터럽트로 처리
- **OS**: 시스템 콜, 스케줄링에 인터럽트 활용
- **성능**: 인터럽트 빈도가 성능 영향
