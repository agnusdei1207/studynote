### Ⅰ. 개요

#### 1. 개념
포그 컴퓨팅(Fog Computing)은 **클라우드와 엣지 디바이스 사이에 위치하여 지역적으로 분산된 컴퓨팅 리소스를 제공하고, 데이터를 실시간으로 처리하며 클라우드와의 효율적인 데이터 동기화를 가능하게 하는 분산 컴퓨팅 아키텍처**이다.

> 비유: "동네 도서관" - 본점(클라우드)과 집(엣지) 사이

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 컴퓨팅 계층
```
┌────────────────────────────────────────────────────────┐
│           컴퓨팅 계층 구조                            │
├────────────────────────────────────────────────────────┤
│                                                        │
│  ☁️ 클라우드 (Cloud)                                 │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 전역 처리                                    │   │
│  │  • 장기 저장                                    │   │
│  │  • 빅데이터 분석                                │   │
│  │  • 지연: 100ms+                                 │   │
│  └────────────────────────────────────────────────┘   │
│                        ↕                              │
│  🌫️ 포그 (Fog) - 중간 계층                          │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 지역 처리                                    │   │
│  │  • 중기 저장                                    │   │
│  │  • 데이터 집계                                  │   │
│  │  • 지연: 10-100ms                               │   │
│  │  • 위치: 지사, 게이트웨이                       │   │
│  └────────────────────────────────────────────────┘   │
│                        ↕                              │
│  📦 엣지 (Edge)                                       │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 현장 처리                                    │   │
│  │  • 단기 저장                                    │   │
│  │  • 실시간 응답                                  │   │
│  │  • 지연: 1-10ms                                 │   │
│  └────────────────────────────────────────────────┘   │
│                        ↕                              │
│  📱 디바이스 (IoT Device)                            │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 센싱                                         │   │
│  │  • 액추에이션                                   │   │
│  │  • 지연: <1ms                                   │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 포그 컴퓨팅 특징
| 특징 | 설명 |
|-----|------|
| 지역성 | 지리적으로 분산된 노드 |
| 계층성 | 다중 계층 구조 |
| 이기종 | 다양한 장치 지원 |
| 실시간 | 낮은 지연 보장 |
| 상호연결 | 노드 간 통신 |

---

### Ⅲ. 기술 비교 분석

#### 4. 포그 vs 엣지 vs 클라우드
| 특징 | 클라우드 | 포그 | 엣지 |
|-----|--------|------|------|
| 위치 | 데이터센터 | 지역 노드 | 현장 |
| 지연 | 높음 | 중간 | 낮음 |
| 처리량 | 대규모 | 중간 | 소규모 |
| 저장 | 장기 | 중기 | 단기 |
| 관리 | 중앙 | 분산 | 로컬 |

#### 6. 장단점
| 장점 | 단점 |
|-----|------|
| 지연 감소 | 관리 복잡성 |
| 대역폭 절약 | 보안 분산 |
| 가용성 향상 | 표준 부재 |
| 지역 처리 | 리소스 제한 |
