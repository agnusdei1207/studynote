# 락인 (Lock-in)

## 핵심 인사이트 (3줄 요약)
> **고객이 특정 제품/서비스에서 벗어나기 어렵게 만드는 전략**. 전환 비용을 높여 이탈 방지. 플랫폼 비즈니스의 핵심 경쟁력.

### Ⅰ. 개요

#### 1. 개념
락인(Lock-in)은 **고객이 특정 기업의 제품이나 서비스를 사용하다가 다른 경쟁사로 이동하기 어렵게 만드는 현상이나 전략**이다. 벤더 락인(Vendor Lock-in)이라고도 한다.

> 비유: "호텔 방에서 나갈 수 없게 만들기" - 편의시설, 포인트, 데이터가 묶여있음

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 락인의 종류
```
┌────────────────────────────────────────────────────────┐
│           락인의 종류                                 │
├────────────────────────────────────────────────────────┤
│                                                        │
│  1. 기술적 락인 (Technical Lock-in):                 │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  • 독점 포맷 (예: .docx, .psd)                │   │
│  │  • 전용 API / SDK                              │   │
│  │  • 호환되지 않는 프로토콜                     │   │
│  │                                                │   │
│  │  예: 아이폰 ↔ 안드로이드 앱 호환 안 됨       │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  2. 데이터 락인 (Data Lock-in):                      │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  • 데이터 이관 어려움                         │   │
│  │  • 클라우드 데이터 묶임                       │   │
│  │  • 플랫폼 간 데이터 비호환                    │   │
│  │                                                │   │
│  │  예: 인스타그램 사진을 다운로드하기 어려움    │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  3. 계약적 락인 (Contractual Lock-in):               │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  • 장기 계약                                   │   │
│  │  • 위약금                                      │   │
│  │  • 약정 기간                                   │   │
│  │                                                │   │
│  │  예: 통신사 2년 약정                          │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  4. 경제적 락인 (Economic Lock-in):                  │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  • 높은 전환 비용                              │   │
│  │  • 포인트 / 마일리지                           │   │
│  │  • 구독 할인                                   │   │
│  │                                                │   │
│  │  예: 아마존 프라임 멤버십                     │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  5. 심리적 락인 (Psychological Lock-in):             │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  • 학습 곡선 (익숙함)                         │   │
│  │  • 브랜드 로열티                               │   │
│  │  • 네트워크 효과                               │   │
│  │                                                │   │
│  │  예: 엑셀 단축키를 다 배워서 바꾸기 싫음     │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 전환 비용 (Switching Cost)
```
┌────────────────────────────────────────────────────────┐
│           전환 비용 구성                              │
├────────────────────────────────────────────────────────┤
│                                                        │
│  전환 비용 =                                          │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  금전적 비용                                   │   │
│  │  + 시간 비용 (학습, 이관)                     │   │
│  │  + 심리적 비용 (불안, 스트레스)               │   │
│  │  + 기회 비용 (이관 중 업무 공백)              │   │
│  │  + 데이터 손실 위험                           │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  전환 비용이 혜택보다 크면 → 락인 효과 발생          │
│                                                        │
│  예시:                                                 │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  은행 계좌 이체:                               │   │
│  │  • 자동이체 변경: 10개 기관                   │   │
│  │  • 거래 내역: 과거 데이터 없음                │   │
│  │  • 급여 계좌: 회사에 요청 필요                │   │
│  │  → 그냥 안 바꾸는 게 편함                     │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### Ⅲ. 기술 비교 분석

#### 4. 락인 vs 개방성
| 구분 | 락인 전략 | 개방성 전략 |
|-----|----------|------------|
| 목표 | 고객 유지 | 시장 확대 |
| 수단 | 독점 생태계 | 표준/호환성 |
| 장점 | 높은 수익성 | 빠른 성장 |
| 단점 | 규제 위험 | 낮은 충성도 |
| 예시 | 애플 생태계 | 안드로이드 |

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **비즈니스 혁신** | 디지털 전환 가속화 및 신규 비즈니스 모델 창출 | 시장 출시 시간(TTM) 50% 단축 |
| **운영 효율** | AI·자동화로 수작업 제거 및 의사결정 지원 강화 | 운영 비용 30~40% 절감 |
| **경쟁력 강화** | 최신 기술 도입으로 시장 경쟁 우위 확보 | 고객 만족도(CSAT) 20점 향상 |

#### 결론
> **락인 (Lock-in)**은(는) ICT 융합 기술은 AI-First 전략, 탄소 중립(Net Zero) 목표, EU AI Act 등 글로벌 규제 환경에 대응하면서 기술적 혁신과 사회적 책임을 동시에 실현하는 방향으로 발전하고 있다.

> **※ 참고 표준**: NIST AI RMF 1.0, EU AI Act(2024), ISO/IEC 42001(AI 관리 시스템), 과기정통부 AI 기본법
