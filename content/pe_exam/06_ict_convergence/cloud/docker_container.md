### Ⅰ. 개요

#### 1. 개념
컨테이너는 **호스트 OS의 커널을 공유하면서 애플리케이션을 격리된 환경에서 실행**하는 가상화 기술이다. 도커(Docker)는 컨테이너를 쉽게 생성, 배포, 실행할 수 있게 해주는 플랫폼이다.

> 비유: "컨테이너는 표준화된 화물 상자" - 어떤 배에 실어도 똑같이 운반 가능

#### 3. 도커 핵심 개념
```
┌────────────────────────────────────────────────────────┐
│                  도커 핵심 개념                         │
├────────────────────────────────────────────────────────┤
│                                                        │
│  1. 이미지 (Image)                                     │
│     ┌────────────────────────────────────────────┐    │
│     │ • 컨테이너 실행을 위한 읽기 전용 템플릿    │    │
│     │ • 레이어 구조 (Union Filesystem)           │    │
│     │ • Dockerfile로 생성                        │    │
│     │ • Docker Hub에서 공유                      │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  2. 컨테이너 (Container)                               │
│     ┌────────────────────────────────────────────┐    │
│     │ • 이미지의 실행 인스턴스                    │    │
│     │ • 격리된 프로세스                           │    │
│     │ • 읽기/쓰기 레이어 추가                     │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  3. Dockerfile                                         │
│     ┌────────────────────────────────────────────┐    │
│     │ FROM ubuntu:20.04                          │    │
│     │ RUN apt-get update && apt-get install -y python3 │
│     │ COPY . /app                                │    │
│     │ WORKDIR /app                               │    │
│     │ CMD ["python3", "app.py"]                  │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  4. 레지스트리 (Registry)                              │
│     - Docker Hub (공개)                               │
│     - Private Registry (사설)                         │
│     - 이미지 저장 및 배포                             │
│                                                        │
│  5. 레이어 구조                                        │
│     ┌────────────────────────────────────────────┐    │
│     │  App Layer (R/W)  ← 컨테이너 전용          │    │
│     │  ─────────────────────────                 │    │
│     │  Config Layer                              │    │
│     │  Dependencies Layer                        │    │
│     │  Base OS Layer                             │    │
│     │  (모두 읽기 전용, 공유 가능)               │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### Ⅱ. 구성 요소 및 핵심 원리

#### 4. 컨테이너 격리 기술
```
┌────────────────────────────────────────────────────────┐
│              리눅스 컨테이너 격리 기술                   │
├────────────────────────────────────────────────────────┤
│                                                        │
│  1. Namespace (이름 공간)                              │
│     ┌────────────────────────────────────────────┐    │
│     │ • PID: 프로세스 ID 격리                     │    │
│     │ • NET: 네트워크 격리                        │    │
│     │ • MNT: 마운트 포인트 격리                   │    │
│     │ • UTS: 호스트명 격리                        │    │
│     │ • IPC: 프로세스 간 통신 격리                │    │
│     │ • USER: 사용자 격리                        │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  2. Cgroups (Control Groups)                          │
│     ┌────────────────────────────────────────────┐    │
│     │ • CPU 사용량 제한                           │    │
│     │ • 메모리 사용량 제한                        │    │
│     │ • I/O 대역폭 제한                           │    │
│     │ • 리소스 모니터링                           │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  3. Union Filesystem                                  │
│     - 여러 파일시스템을 하나로 병합                    │
│     - Copy-on-Write (CoW) 전략                        │
│     - AUFS, OverlayFS, Btrfs                          │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### Ⅲ. 기술 비교 분석

#### 2. 컨테이너 vs 가상머신
```
┌────────────────────────────────────────────────────────┐
│              가상머신 vs 컨테이너                        │
├────────────────────────────────────────────────────────┤
│                                                        │
│  가상머신 (VM):                                        │
│  ┌────────────────────────────────────────────────┐   │
│  │  App A  │  App B  │  App C  │  App D          │   │
│  │ Bins/Lib│ Bins/Lib│ Bins/Lib│ Bins/Lib        │   │
│  │ Guest OS│ Guest OS│ Guest OS│ Guest OS        │   │
│  ├─────────┴─────────┴─────────┴────────────────┤   │
│  │              Hypervisor                        │   │
│  ├───────────────────────────────────────────────┤   │
│  │              Host OS                           │   │
│  ├───────────────────────────────────────────────┤   │
│  │              Physical Server                   │   │
│  └───────────────────────────────────────────────┘   │
│  • 각 VM마다 완전한 OS 필요 (GB 단위)                 │
│  • 시작 시간: 분 단위                                 │
│  • 리소스 오버헤드 큼                                 │
│                                                        │
│  컨테이너 (Container):                                │
│  ┌────────────────────────────────────────────────┐   │
│  │  App A  │  App B  │  App C  │  App D          │   │
│  │ Bins/Lib│ Bins/Lib│ Bins/Lib│ Bins/Lib        │   │
│  ├─────────┴─────────┴─────────┴────────────────┤   │
│  │           Container Runtime (Docker)           │   │
│  ├───────────────────────────────────────────────┤   │
│  │              Host OS                           │   │
│  ├───────────────────────────────────────────────┤   │
│  │              Physical Server                   │   │
│  └───────────────────────────────────────────────┘   │
│  • 호스트 OS 커널 공유 (MB 단위)                      │
│  • 시작 시간: 초 단위                                 │
│  • 리소스 효율적                                      │
│                                                        │
└────────────────────────────────────────────────────────┘
```
