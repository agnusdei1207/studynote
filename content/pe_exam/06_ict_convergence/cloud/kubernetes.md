### Ⅰ. 개요

#### 1. 개념
쿠버네티스(Kubernetes, K8s)는 **컨테이너화된 애플리케이션의 배포, 확장, 관리를 자동화하는 오픈소스 플랫폼**이다.

> 비유: "컨테이너의 오케스트라 지휘자" - 수백 개의 컨테이너를 조율하여 하나의 서비스처럼 동작

#### 3. 핵심 개념
```
┌────────────────────────────────────────────────────────┐
│                  쿠버네티스 핵심 개념                    │
├────────────────────────────────────────────────────────┤
│                                                        │
│  1. Pod (파드)                                         │
│     ┌────────────────────────────────────────────┐    │
│     │ • 가장 작은 배포 단위                        │    │
│     │ • 하나 이상의 컨테이너 그룹                  │    │
│     │ • 공유 IP, 스토리지                         │    │
│     │ • 함께 생성되고 삭제됨                      │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  2. ReplicaSet & Deployment                           │
│     ┌────────────────────────────────────────────┐    │
│     │ ReplicaSet: 지정된 수의 파드 복제본 유지    │    │
│     │ Deployment: 롤링 업데이트, 롤백 관리        │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  3. Service                                            │
│     ┌────────────────────────────────────────────┐    │
│     │ • 파드에 접근하기 위한 안정적인 엔드포인트  │    │
│     │ • ClusterIP: 내부 통신                      │    │
│     │ • NodePort: 외부 노출                       │    │
│     │ • LoadBalancer: 클라우드 로드밸런서         │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  4. ConfigMap & Secret                                │
│     - ConfigMap: 설정 데이터                          │
│     - Secret: 민감 정보 (암호화 권장)                 │
│                                                        │
│  5. Ingress                                            │
│     - HTTP/HTTPS 라우팅                               │
│     - 도메인 기반 라우팅                              │
│     - SSL 종료                                        │
│                                                        │
│  6. Persistent Volume (PV) & Claim (PVC)              │
│     - 스토리지 추상화                                  │
│     - 파드 재시작해도 데이터 유지                     │
│                                                        │
│  7. Namespace                                          │
│     - 리소스 논리적 분리                               │
│     - 권한 격리                                        │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 4. 워크로드 리소스

│  Pending → Running → Succeeded/Failed → Unknown       │
