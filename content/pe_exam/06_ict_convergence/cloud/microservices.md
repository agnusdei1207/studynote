# 마이크로서비스 (Microservices)

## 핵심 인사이트 (3줄 요약)
> **단일 애플리케이션을 작은 독립 서비스들로 분해**하는 아키텍처 패턴. 각 서비스는 독립적으로 배포, 확장 가능. Netflix, Amazon 등 대규모 시스템에서 검증.

### Ⅰ. 개요

#### 1. 개념
마이크로서비스 아키텍처는 **애플리케이션을 작고 독립적으로 배포 가능한 서비스들의 집합으로 구성**하는 아키텍처 스타일이다. 각 서비스는 특정 비즈니스 기능을 담당한다.

> 비유: "특수부대 팀" - 각 팀이 독립적으로 임무 수행하지만 전체 목표를 위해 협력

### Ⅱ. 구성 요소 및 핵심 원리

#### 3. 마이크로서비스 패턴
```
┌────────────────────────────────────────────────────────┐
│              마이크로서비스 디자인 패턴                  │
├────────────────────────────────────────────────────────┤
│                                                        │
│  1. API 게이트웨이                                     │
│     ┌────────────────────────────────────────────┐    │
│     │ 클라이언트 → [API Gateway] → 서비스들       │    │
│     │                                            │    │
│     │ • 라우팅                                   │    │
│     │ • 인증/인가                                │    │
│     │ • 로드 밸런싱                              │    │
│     │ • 속도 제한                                │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  2. 서비스 디스커버리                                  │
│     ┌────────────────────────────────────────────┐    │
│     │ • 서비스 위치 등록 및 조회                  │    │
│     │ • Eureka, Consul, etcd                     │    │
│     │ • 클라이언트 사이드/서버 사이드             │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  3. 서비스 간 통신                                     │
│     ┌────────────────────────────────────────────┐    │
│     │ 동기: REST, gRPC, GraphQL                  │    │
│     │ 비동기: 메시지 큐 (Kafka, RabbitMQ)        │    │
│     │                                            │    │
│     │ 이벤트 기반 통신:                          │    │
│     │ 주문서비스 ──(주문생성이벤트)──→ 메시지큐  │    │
│     │                                  ↓         │    │
│     │    재고서비스 ←──┘  알림서비스 ←──┘       │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  4. 서킷 브레이커                                      │
│     ┌────────────────────────────────────────────┐    │
│     │ Closed ──(실패 임계치)──→ Open             │    │
│     │   ↑                          │              │    │
│     │   └── Half-Open ←───────────┘              │    │
│     │                                            │    │
│     │ • 연쇄 장애 방지                           │    │
│     │ • 빠른 실패 (Fail Fast)                    │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  5. CQRS (Command Query Responsibility Segregation)   │
│     ┌────────────────────────────────────────────┐    │
│     │ 쓰기 모델 (Command) │ 읽기 모델 (Query)    │    │
│     │     정규화된 DB     │  비정규화된 Read DB  │    │
│     │                     │                      │    │
│     │ 성능 최적화, 복잡도 증가                    │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
│  6. Saga 패턴                                         │
│     ┌────────────────────────────────────────────┐    │
│     │ 분산 트랜잭션 관리                          │    │
│     │                                            │    │
│     │ 주문 → 재고차감 → 결제 → 배송              │    │
│     │   ↑        ↓                               │    │
│     │   └── 보상 트랜잭션 (실패 시)              │    │
│     └────────────────────────────────────────────┘    │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### Ⅲ. 기술 비교 분석

#### 2. 모놀리식 vs 마이크로서비스
```
┌────────────────────────────────────────────────────────┐
│         모놀리식 vs 마이크로서비스                       │
├────────────────────────────────────────────────────────┤
│                                                        │
│  모놀리식 (Monolithic):                               │
│  ┌────────────────────────────────────────────────┐   │
│  │              단일 애플리케이션                   │   │
│  │  ┌──────────────────────────────────────────┐ │   │
│  │  │ UI │ 비즈니스 로직 │ DB │ 인증 │ 결제    │ │   │
│  │  └──────────────────────────────────────────┘ │   │
│  │                                                │   │
│  │  • 단일 코드베이스                             │   │
│  │  • 전체가 함께 배포                            │   │
│  │  • 확장: 전체 복제                             │   │
│  │  • 기술 스택: 통일                             │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  마이크로서비스 (Microservices):                       │
│  ┌────────────────────────────────────────────────┐   │
│  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐     │   │
│  │  │사용자│ │상품 │ │주문 │ │결제 │ │알림 │     │   │
│  │  │서비스│ │서비스│ │서비스│ │서비스│ │서비스│     │   │
│  │  └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘     │   │
│  │     │       │       │       │       │         │   │
│  │  ┌──┴──┐ ┌──┴──┐ ┌──┴──┐ ┌──┴──┐ ┌──┴──┐     │   │
│  │  │ DB  │ │ DB  │ │ DB  │ │ DB  │ │ DB  │     │   │
│  │  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘     │   │
│  │                                                │   │
│  │  • 독립 배포                                   │   │
│  │  • 서비스별 확장                               │   │
│  │  • 기술 스택: 자유                             │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 4. 장단점
```
┌────────────────────────────────────────────────────────┐
│                    장단점 비교                          │
├────────────────────────────────────────────────────────┤
│                                                        │
│  장점:                                                 │
│  ┌────────────────────────────────────────────────┐   │
│  │ ✓ 독립 배포: 서비스별 개별 배포 가능           │   │
│  │ ✓ 기술 다양성: 각 서비스에 최적 기술 선택      │   │
│  │ ✓ 탄력적 확장: 필요한 서비스만 확장            │   │
│  │ ✓ 장애 격리: 한 서비스 장애가 전파 안됨        │   │
│  │ ✓ 팀 자율성: 작은 팀이 독립 개발               │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  단점:                                                 │
│  ┌────────────────────────────────────────────────┐   │
│  │ ✗ 복잡성: 분산 시스템 관리 어려움              │   │
│  │ ✗ 네트워크: 지연, 장애 처리                   │   │
│  │ ✗ 데이터 일관성: 분산 트랜잭션 복잡           │   │
│  │ ✗ 운영 오버헤드: 모니터링, 로깅 분산          │   │
│  │ ✗ 테스트: 통합 테스트 어려움                  │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```
