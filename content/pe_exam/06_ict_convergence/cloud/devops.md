### Ⅰ. 개요

#### 1. 개념
DevOps(Development + Operations)는 **소프트웨어 개발과 IT 운영을 통합하여 조직이 제품을 더 빠르게 출시하고 지속적으로 개선할 수 있게 하는 문화, 철학, 실천 방법의 집합**이다.

> 비유: "요리사와 서빙팀의 원팀" - 주방과 홀의 경계를 허물어 빠른 서비스

### Ⅱ. 구성 요소 및 핵심 원리

#### 3. DevOps 핵심 요소 (CALMS)
| 요소 | 설명 |
|-----|------|
| **C**ulture | 협업 문화, 공동 책임 |
| **A**utomation | 자동화 (CI/CD, IaC) |
| **L**ean | 린 방식, 낭비 제거 |
| **M**easurement | 측정, 모니터링 |
| **S**haring | 지식 공유, 투명성 |

#### 6. 핵심 실천 방법
| 실천 방법 | 설명 |
|----------|------|
| CI (지속적 통합) | 코드를 자주 병합하고 자동 테스트 |
| CD (지속적 배포) | 자동으로 프로덕션에 배포 |
| IaC | 인프라를 코드로 관리 |
| 마이크로서비스 | 작은 독립적 서비스로 분해 |
| 모니터링 | 실시간 시스템 상태 추적 |
| 피드백 루프 | 빠른 학습과 개선 |
