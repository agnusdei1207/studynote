### Ⅰ. 개요

#### 1. 개념
방화벽(Firewall)은 **내부 네트워크와 외부 네트워크 사이에서 트래픽을 검사하고, 보안 정책에 따라 허용/차단하는 시스템**이다. 외부 공격으로부터 내부망을 보호하는 1차 방어선이다.

> 비유: "건물 출입구 경비원" - 초대장 없으면 못 들어가요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 방화벽 유형
```
┌────────────────────────────────────────────────────────┐
│           방화벽 진화 과정                             │
├────────────────────────────────────────────────────────┤
│                                                        │
│  1세대: 패킷 필터링 방화벽                             │
│  ┌────────────────────────────────────────────────┐   │
│  │  • IP, Port 기준 필터링                        │   │
│  │  • 상태 비저장 (Stateless)                     │   │
│  │  • 빠르지만 취약점 존재                         │   │
│  └────────────────────────────────────────────────┘   │
│                      ↓                                 │
│  2세대: 상태 검사 방화벽 (Stateful Inspection)        │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 연결 상태 추적                               │   │
│  │  • 세션 테이블 유지                             │   │
│  │  • 보안성 향상                                  │   │
│  └────────────────────────────────────────────────┘   │
│                      ↓                                 │
│  3세대: 프록시 방화벽 (Application Layer)             │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 애플리케이션 계층 검사                       │   │
│  │  • 프록시 서버 역할                             │   │
│  │  • 콘텐츠 필터링 가능                           │   │
│  └────────────────────────────────────────────────┘   │
│                      ↓                                 │
│  4세대: 차세대 방화벽 (NGFW)                          │
│  ┌────────────────────────────────────────────────┐   │
│  │  • DPI (Deep Packet Inspection)                │   │
│  │  • IPS/IDS 통합                                │   │
│  │  • 앱 식별, 사용자 식별                         │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 방화벽 유형별 비교
| 유형 | 계층 | 장점 | 단점 |
|-----|------|------|------|
| 패킷 필터 | 3/4 | 빠름, 저렴 | 상태 미인식 |
| 상태 검사 | 3/4 | 연결 추적 | 메모리 사용 |
| 프록시 | 7 | 상세 검사 | 느림 |
| NGFW | 3-7 | 통합 보안 | 비용 |

#### 4. 방화벽 규칙
```
┌────────────────────────────────────────────────────────┐
│           방화벽 규칙 예시                             │
├────────────────────────────────────────────────────────┤
│                                                        │
│  규칙  출발지      목적지       포트    동작           │
│  ────  ──────     ──────       ────    ────           │
│  1     Any        Web_Server   80      허용           │
│  2     Any        Web_Server   443     허용           │
│  3     Internal   Any          53      허용 (DNS)     │
│  4     Internal   Any          25      허용 (SMTP)    │
│  5     Any        Any          Any     차단 (기본)    │
│                                                        │
│  • First Match: 위에서부터 순차 적용                  │
│  • Default Deny: 기본 정책은 차단                     │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 5. 배치 구조
| 구역 | 설명 |
|-----|------|
| 외부망 (Untrusted) | 인터넷 |
| DMZ (Demilitarized Zone) | 공개 서버 |
| 내부망 (Trusted) | 사내망 |

```
인터넷 ──[FW1]── DMZ (웹서버) ──[FW2]── 내부망
                  │
                  └── 이중화 구성
```

#### 6. NAT (Network Address Translation)
| 유형 | 설명 |
|-----|------|
| SNAT | 출발지 IP 변환 (내부→외부) |
| DNAT | 목적지 IP 변환 (외부→내부) |
| PAT | 포트 기반 변환 (1개 IP 공유) |

#### 7. 주요 공격 방어
| 공격 | 방어 방법 |
|-----|----------|
| DDoS | 속도 제한, 차단 |
| 포트 스캔 | 탐지 및 차단 |
| IP 스푸핑 | ingress 필터링 |
| 웹 공격 | WAF 규칙 |

### Ⅳ. 실무 적용 방안

#### 10. 실무에선? (기술사적 판단)
**선정 기준:**
- 처리량 (Throughput)
- 동시 연결 수
- 보안 기능 (IPS, AV 등)
- 관리 편의성

**구성 원칙:**
- Defense in Depth (다층 방어)
- Least Privilege (최소 권한)
- Default Deny (기본 차단)

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **비즈니스 혁신** | 디지털 전환 가속화 및 신규 비즈니스 모델 창출 | 시장 출시 시간(TTM) 50% 단축 |
| **운영 효율** | AI·자동화로 수작업 제거 및 의사결정 지원 강화 | 운영 비용 30~40% 절감 |
| **경쟁력 강화** | 최신 기술 도입으로 시장 경쟁 우위 확보 | 고객 만족도(CSAT) 20점 향상 |
