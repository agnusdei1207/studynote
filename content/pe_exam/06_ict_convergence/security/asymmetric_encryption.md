### Ⅰ. 개요

#### 1. 개념
비대칭키 암호는 **공개키와 개인키라는 두 개의 서로 다른 키를 사용하는 암호 방식**이다. 공개키로 암호화하고 개인키로 복호화하거나, 개인키로 서명하고 공개키로 검증한다.

> 비유: "우편함" - 누구나 넣을 수 있지만 주인만 열 수 있어요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 공개키 암호 구조
```
┌────────────────────────────────────────────────────────┐
│           공개키 암호 활용                             │
├────────────────────────────────────────────────────────┤
│                                                        │
│  🔐 암호화/복호화 (기밀성):                            │
│  ┌────────────────────────────────────────────────┐   │
│  │  송신자                    수신자              │   │
│  │     │                         │                  │   │
│  │  평문 ──[공개키로 암호화]──→ 암호문             │   │
│  │                              ↓                   │   │
│  │                         [개인키로 복호화]        │   │
│  │                              ↓                   │   │
│  │                             평문                 │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  ✍️ 디지털 서명 (인증):                                │
│  ┌────────────────────────────────────────────────┐   │
│  │  서명자                    검증자              │   │
│  │     │                         │                  │   │
│  │  문서 ──[개인키로 서명]──→ 서명+문서            │   │
│  │                              ↓                   │   │
│  │                         [공개키로 검증]          │   │
│  │                              ↓                   │   │
│  │                            진위 확인             │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 주요 알고리즘
| 알고리즘 | 기반 | 키 길이 | 용도 |
|---------|------|--------|------|
| RSA | 소인수분해 | 2048~4096 | 암호화, 서명 |
| DSA | 이산대수 | 2048 | 서명 |
| ECDSA | 타원곡선 | 256~521 | 서명 |
| ECDH | 타원곡선 | 256~521 | 키 교환 |
| Ed25519 | 타원곡선 | 256 | 서명 |

#### 5. 디지털 서명 과정
| 단계 | 서명 | 검증 |
|-----|------|------|
| 1 | 해시 계산 | 해시 계산 |
| 2 | 개인키로 암호화 | 공개키로 복호화 |
| 3 | 서명 생성 | 비교 검증 |

#### 6. 키 교환 프로토콜
| 프로토콜 | 설명 |
|---------|------|
| Diffie-Hellman | 키 교환 |
| ECDH | 타원곡선 키 교환 |
| RSA 키 교환 | 공개키 암호화 |

---

### Ⅲ. 기술 비교 분석

#### 4. RSA vs ECC
| 항목 | RSA | ECC |
|-----|-----|-----|
| 기반 | 소인수분해 | 타원곡선 |
| 키 길이 | 긴 (2048비트+) | 짧은 (256비트) |
| 속도 | 느림 | 빠름 |
| 용도 | 범용 | 모바일, IoT |

#### 7. 장단점
| 장점 | 단점 |
|-----|------|
| 키 분배 용이 | 속도 느림 |
| 디지털 서명 | 키 길이 김 |
| 부인 방지 | 계산 복잡 |
