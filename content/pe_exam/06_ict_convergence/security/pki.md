### Ⅰ. 개요

#### 1. 개념
PKI(Public Key Infrastructure)는 **공개키 암호방식을 기반으로 사용자의 신원을 확인하고, 공개키의 진위를 보증하는 하드웨어, 소프트웨어, 정책의 통합 시스템**이다.

> 비유: "여권 발급 시스템" - 정부가 본인임을 보증해요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. PKI 구성요소
```
┌────────────────────────────────────────────────────────┐
│           PKI 구성요소                                 │
├────────────────────────────────────────────────────────┤
│                                                        │
│  🏛️ 인증 기관 (CA - Certificate Authority):            │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 인증서 발급 및 관리                          │   │
│  │  • 신원 검증                                    │   │
│  │  • 디지털 서명                                  │   │
│  │  • 예: Verisign, DigiCert, 국가정보원          │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  📋 등록 기관 (RA - Registration Authority):           │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 사용자 신원 확인                             │   │
│  │  • 인증서 요청 접수                             │   │
│  │  • CA에 발급 요청                               │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  📦 인증서 저장소 (Repository):                        │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 인증서 저장                                  │   │
│  │  • CRL (폐기 목록) 관리                         │   │
│  │  • LDAP, HTTP 등으로 접근                       │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  👤 사용자 (End Entity):                               │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 인증서 신청자                                │   │
│  │  • 인증서 사용자                                │   │
│  │  • 개인키 보유                                  │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. X.509 인증서 구조
| 필드 | 설명 |
|-----|------|
| Version | 인증서 버전 (v3) |
| Serial Number | 일련번호 |
| Signature Algorithm | 서명 알고리즘 |
| Issuer | 발급자 (CA) |
| Validity Period | 유효기간 |
| Subject | 소유자 |
| Subject Public Key Info | 공개키 정보 |
| Extensions | 확장 필드 |
| Signature | CA 서명 |

#### 4. PKI 프로세스
| 단계 | 설명 |
|-----|------|
| 1. 키 쌍 생성 | 사용자가 공개키/개인키 생성 |
| 2. 인증서 요청 | CSR(Certificate Signing Request) 생성 |
| 3. 신원 검증 | RA가 사용자 신원 확인 |
| 4. 인증서 발급 | CA가 인증서 서명 후 발급 |
| 5. 인증서 사용 | SSL/TLS, 전자서명 등 |
| 6. 폐기/갱신 | 만료 또는 폐기 시 CRL 등록 |

#### 5. 인증서 유형
| 유형 | 용도 | 검증 수준 |
|-----|------|----------|
| DV (Domain) | 도메인 소유 | 낮음 |
| OV (Organization) | 조직 확인 | 중간 |
| EV (Extended) | 엄격한 검증 | 높음 |
| Code Signing | 코드 서명 | 중간 |
| Client | 클라이언트 인증 | 중간 |
