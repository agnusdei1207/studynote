### Ⅰ. 개요

#### 1. 개념
블록체인(Blockchain)은 **데이터를 블록 단위로 저장하고 체인 형태로 연결하여, 분산 네트워크에서 위변조가 불가능한 원장을 유지하는 기술**이다. 중앙 관리자 없이 참여자 간 신뢰를 보장한다.

> 비유: "공개 장부" - 모두가 볼 수 있고, 한 번 쓰면 지울 수 없어요

### Ⅱ. 구성 요소 및 핵심 원리

#### 3. 블록체인 유형
| 유형 | 설명 | 예시 |
|-----|------|------|
| 퍼블릭 | 누구나 참여 | 비트코인, 이더리움 |
| 프라이빗 | 허가된 참여자 | 하이퍼레저, R3 Corda |
| 컨소시엄 | 그룹 간 공동 운영 | 금융 컨소시엄 |
| 하이브리드 | 혼합형 | 일부 공개, 일부 비공개 |

#### 4. 합의 알고리즘
| 알고리즘 | 설명 | 장단점 |
|---------|------|--------|
| PoW (작업 증명) | 연산으로 경쟁 | 보안 강/에너지 낭비 |
| PoS (지분 증명) | 지분 비례 선정 | 에너지 절약/부자 우대 |
| DPoS (위임 지분) | 대표 선출 | 빠름/중앙화 우려 |
| PBFT (비잔틴 장애) | 투표 기반 | 빠름/참여자 제한 |

#### 5. 핵심 기술
| 기술 | 설명 |
|-----|------|
| 해시 함수 | 데이터 무결성 (SHA-256) |
| 공개키 암호 | 디지털 서명 |
| 머클 트리 | 효율적 거래 검증 |
| P2P 네트워크 | 분산 노드 통신 |
| 스마트 컨트랙트 | 자동 실행 계약 |

#### 6. 스마트 컨트랙트
```
┌────────────────────────────────────────────────────────┐
│           스마트 컨트랙트 동작                         │
├────────────────────────────────────────────────────────┤
│                                                        │
│  if (조건 충족) {                                     │
│      자동 실행                                        │
│  }                                                    │
│                                                        │
│  예: 보험금 지급                                      │
│  ┌────────────────────────────────────────────────┐   │
│  │  조건: 비행기 지연 > 3시간                      │   │
│  │  결과: 보험금 자동 지급                         │   │
│  │  중재자: 없음 (코드가 판단)                     │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

- 다자간 거래 (중개 없이)
- 이력 추적 (공급망)
