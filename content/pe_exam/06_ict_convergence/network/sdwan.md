### Ⅰ. 개요

#### 1. 개념
SD-WAN(Software-Defined Wide Area Network)은 **기존 하드웨어 기반 WAN을 소프트웨어로 가상화하여, 여러 네트워크 회선(MPLS, 인터넷, LTE 등)을 중앙에서 지능적으로 관리하고 트래픽을 최적 경로로 라우팅하는 기술**이다.

> 비유: "스마트 내비게이션" - 상황에 따라 최적 경로 선택

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 3. SD-WAN 핵심 기능
| 기능 | 설명 |
|-----|------|
| 동적 경로 선택 | 트래픽 유형별 최적 경로 |
| 로드 밸런싱 | 회선 간 부하 분산 |
| 애플리케이션 인식 | 앱별 우선순위 |
| 보안 | 암호화, 방화벽 |
| 중앙 관리 | 클라우드 컨트롤러 |
| 자동 장애 조치 | 회선 장애 시 자동 전환 |

### Ⅲ. 기술 비교 분석

#### 2. 기존 WAN vs SD-WAN
```
┌────────────────────────────────────────────────────────┐
│           기존 WAN vs SD-WAN                          │
├────────────────────────────────────────────────────────┤
│                                                        │
│  기존 WAN (MPLS 중심):                                │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  본사 ──── MPLS (비싸고 안정) ──── 지사       │   │
│  │                                                │   │
│  │  문제:                                         │   │
│  │  • 높은 비용                                   │   │
│  │  • 단일 경로                                   │   │
│  │  • 느린 배포                                   │   │
│  │  • 수동 설정                                   │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  SD-WAN (멀티 패스):                                  │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  본사 ─┬─ MPLS (중요 트래픽)                  │   │
│  │        ├─ 인터넷 (일반 트래픽)                 │   │
│  │        └─ LTE (백업)                          │   │
│  │        ↓                                       │   │
│  │  🧠 SD-WAN 컨트롤러 (지능형 라우팅)            │   │
│  │        ↓                                       │   │
│  │       지사                                     │   │
│  │                                                │   │
│  │  장점:                                         │   │
│  │  • 비용 절감                                   │   │
│  │  • 경로 다양화                                 │   │
│  │  • 빠른 배포                                   │   │
│  │  • 중앙 관리                                   │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **통신 성능** | 최적화된 프로토콜·라우팅으로 지연 및 패킷 손실 감소 | 네트워크 지연 50% 단축 |
| **확장성** | 소프트웨어 정의 방식으로 트래픽 급증에도 유연 대응 | 대역폭 활용률 80% 이상 |
| **보안·안정성** | 계층적 보안 아키텍처로 가용성 및 무결성 보장 | SLA 99.99% (4-nine) 달성 |

#### 결론
> **SD-WAN (Software Defined WAN)**은(는) 네트워크 기술은 5G·SDN·NFV를 통해 소프트웨어 중심으로 진화하고 있으며, AI 기반 자율 네트워크(Autonomous Network)가 차세대 통신 인프라의 핵심이 될 것이다.

> **※ 참고 표준**: RFC 표준 시리즈, ETSI NFV ISG, 3GPP TS 23.501, ITU-T 권고안
