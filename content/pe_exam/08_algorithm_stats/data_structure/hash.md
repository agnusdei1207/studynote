# 해시 테이블 (Hash Table)

## 핵심 인사이트 (3줄 요약)
> **키를 해시 함수로 인덱스 변환하여 빠른 접근**. O(1) 평균 검색. 충돌 해결 핵심.

### 📌 예상 문제

---

### Ⅰ. 개요

#### 1. 개념
해시 테이블(Hash Table)은 **키(Key)를 해시 함수(Hash Function)를 통해 배열 인덱스로 변환하여, 값을 저장하고 검색하는 자료구조**이다. 평균 O(1)의 빠른 검색, 삽입, 삭제가 가능하다.

> 비유: "사물함 번호표" - 이름으로 번호를 찾아요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 해시 테이블 구조
```
┌────────────────────────────────────────────────────────┐
│           해시 테이블 구조                             │
├────────────────────────────────────────────────────────┤
│                                                        │
│  🔑 키 → 해시 함수 → 인덱스 → 버킷                     │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                 │   │
│  │  Key: "apple"                                  │   │
│  │    ↓                                           │   │
│  │  hash("apple") = 5                             │   │
│  │    ↓                                           │   │
│  │  Bucket[5] → "사과"                            │   │
│  │                                                 │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  📊 버킷 배열:                                         │
│  ┌────┬────┬────┬────┬────┬────┬────┬────┐          │
│  │ 0  │ 1  │ 2  │ 3  │ 4  │ 5  │ 6  │ 7  │          │
│  ├────┼────┼────┼────┼────┼────┼────┼────┤          │
│  │    │바나나│   │체리 │   │사과 │   │포도 │          │
│  │    │    │   │    │   │    │   │    │          │
│  └────┴────┴────┴────┴────┴────┴────┴────┘          │
│                                                        │
│  ⚠️ 충돌 (Collision):                                  │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 서로 다른 키가 같은 인덱스                  │   │
│  │  • 해결: 체이닝, 오픈 어드레싱                 │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 충돌 해결 방법
| 방법 | 설명 | 장단점 |
|-----|------|--------|
| 체이닝 | 연결리스트로 연결 | 간단, 메모리 |
| 선형 탐사 | 다음 빈 슬롯 | 캐시, 군집 |
| 이차 탐사 | 제곱 간격 이동 | 군집 완화 |
| 이중 해시 | 두 번째 해시 사용 | 분산 좋음 |

#### 4. 해시 함수 특성
| 특성 | 설명 |
|-----|------|
| 결정성 | 같은 키 → 같은 해시 |
| 균등성 | 고르게 분산 |
| 효율성 | 빠른 계산 |
| 최소 충돌 | 다른 키 → 다른 해시 |

#### 5. 적재율과 성능
| 적재율 | 체이닝 | 오픈 어드레싱 |
|-------|--------|---------------|
| 0.5 | O(1) | O(1) |
| 0.75 | O(1) | O(1~2) |
| 0.9 | O(1~2) | O(2~3) |
| 1.0 | O(1~3) | O(n) |
