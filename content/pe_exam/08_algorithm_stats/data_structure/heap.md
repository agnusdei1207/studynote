# 힙 자료구조 (Heap Data Structure)

## 핵심 인사이트 (3줄 요약)

### Ⅰ. 개요

#### 1. 개념
힙(Heap)은 **완전 이진 트리 구조에서 부모 노드가 자식 노드보다 항상 크거나(최대 힙) 작은(최소 힙) 특성을 유지**하는 자료구조다.

> 비유: "응급실 대기 순위" - 항상 가장 급한 환자(최대 우선순위)가 맨 위에

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 3. 힙 연산 (삽입·삭제)
```
최대 힙 삽입 (Heapify-Up):
1. 완전 이진 트리 마지막 위치에 추가
2. 부모와 비교하여 더 크면 교환 (Sift-Up)
3. 제자리 찾을 때까지 반복

예: 힙에 90 삽입
        100                      100
       /    \        →           /    \
     40       80               90       80
    /  \    /  \              /  \    /   \
   20  30  50  [90]         20  30  50   40
   
[90] → 부모 40과 비교 → 교환 → 부모 100과 비교 → 유지

시간복잡도: O(log n)

최대 힙 삭제 (루트 삭제):
1. 루트 원소 반환
2. 마지막 원소를 루트로 이동
3. 자식과 비교하여 더 작으면 교환 (Sift-Down)
4. 제자리 찾을 때까지 반복

       100                      80
      /    \       →           /    \
    40       80               40       70
   /  \    /  \              /  \    /
  20  30  50  70            20  30  50
  
70→루트 → 80과 비교 → sink → O(log n)
```

#### 4. 시간·공간 복잡도
| 연산 | 시간복잡도 | 설명 |
|------|-----------|------|
| 삽입 (insert) | O(log n) | Heapify-Up |
| 삭제 (delete/pop) | O(log n) | Heapify-Down |
| 최대/최소 조회 | O(1) | 루트 참조 |
| 힙 생성 (Heapify) | O(n) | 배열→힙 변환 |
| 힙 정렬 | O(n log n) | 힙 생성 + 반복 삭제 |
| 공간 복잡도 | O(n) | 배열 저장 |

#### 5. 힙 정렬 (Heap Sort)
```
과정:
1단계: 배열을 최대 힙으로 변환 (Heapify) - O(n)
2단계: 루트(최댓값)를 배열 마지막과 교환 후 힙 크기 감소
3단계: 힙 성질 복원 (Heapify-Down) - O(log n)
4단계: 2~3 반복 - O(n log n)

결과: 오름차순 정렬

예시: [4, 10, 3, 5, 1] 정렬
1. 힙 생성: [10, 5, 3, 4, 1]
2. 교환: [1, 5, 3, 4] + [10] → Heapify → [5, 4, 3, 1]
3. 교환: [1, 4, 3] + [5, 10] → Heapify → [4, 1, 3]
4. 교환: [1, 3] + [4, 5, 10] → ...
결과: [1, 3, 4, 5, 10] ✓

특징: In-place 정렬, O(n log n) 보장
      But 캐시 효율 낮아 QuickSort보다 느림
```

#### 6. 정렬 알고리즘 최종 비교
| 알고리즘 | 평균 | 최악 | 공간 | 안정성 | 비고 |
|---------|------|------|------|--------|------|
| 버블 | O(n²) | O(n²) | O(1) | ✓ | 가장 비효율 |
| 선택 | O(n²) | O(n²) | O(1) | X | 최소 교환 |
| 삽입 | O(n²) | O(n²) | O(1) | ✓ | 거의 정렬 데이터 유리 |
| **힙** | **O(n log n)** | **O(n log n)** | **O(1)** | **X** | **최악 보장** |
| **합병** | **O(n log n)** | **O(n log n)** | **O(n)** | **✓** | **안정, 외부정렬** |
| **퀵** | **O(n log n)** | **O(n²)** | **O(log n)** | **X** | **실무 최속** |
| 기수 | O(kn) | O(kn) | O(n+k) | ✓ | 비교 안 함 |

### Ⅲ. 기술 비교 분석

#### 2. 최대 힙 vs 최소 힙
```
최대 힙 (Max Heap):                최소 힙 (Min Heap):
         100                              1
        /    \                         /     \
      40       80                    10       5
     /  \    /   \                  /  \    /   \
   20   30  50   70                15  20  8   40

특성: 부모 ≥ 자식                  특성: 부모 ≤ 자식
top() = 최댓값 (100)               top() = 최솟값 (1)

배열 표현 (인덱스 1부터):
최대 힙: [-, 100, 40, 80, 20, 30, 50, 70]
인덱스:     0   1   2   3   4   5   6   7

부모 인덱스 = i // 2
왼쪽 자식  = 2 * i
오른쪽 자식= 2 * i + 1
```

#### 7. 우선순위 큐 vs 일반 큐 비교
| 항목 | 일반 큐(FIFO) | 우선순위 큐 (힙) |
|------|-------------|----------------|
| 순서 | 도착 순서 | 우선순위 기준 |
| 삽입 | O(1) | O(log n) |
| 삭제 | O(1) | O(log n) |
| 활용 | BFS | 다익스트라, 허프만 |

### Ⅳ. 실무 적용 방안

#### 9. 활용 사례
- **다익스트라 최단경로**: 거리가 최소인 노드를 빠르게 추출
- **허프만 코딩**: 빈도 낮은 문자부터 처리
- **OS 프로세스 스케줄링**: 우선순위 기반 스케줄러
- **이벤트 시뮬레이션**: 이벤트 발생 시간 순 처리
- **K번째 큰/작은 원소**: 크기 K 힙 유지

#### 10. 실무에선? (기술사적 판단)
- **Python**: `heapq` 모듈 (최소 힙)
- **Java**: `PriorityQueue` (최소 힙)
- **힙 생성 O(n)**: 단순 삽입 n번 O(n log n)보다 효율적
- **기술사 포인트**: 힙 성질, 삽입/삭제 과정, 힙 정렬 복잡도
