# 정렬 알고리즘 (Sorting Algorithms)

## 핵심 인사이트 (3줄 요약)

### Ⅰ. 개요

#### 1. 개념
정렬 알고리즘(Sorting Algorithm)은 **데이터 집합을 특정 기준(오름차순, 내림차순 등)에 따라 순서대로 재배열하는 알고리즘**이다. 효율적인 검색과 데이터 처리의 기반이 된다.

> 비유: "도서관 책 정리" - 순서대로 꽂아요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 정렬 알고리즘 분류
```
┌────────────────────────────────────────────────────────┐
│           정렬 알고리즘 분류                          │
├────────────────────────────────────────────────────────┤
│                                                        │
│  🔄 비교 기반 정렬 (Comparison-based):                 │
│  ┌────────────────────────────────────────────────┐   │
│  │  O(n²):                                        │   │
│  │  • 버블 정렬 (Bubble Sort)                      │   │
│  │  • 선택 정렬 (Selection Sort)                   │   │
│  │  • 삽입 정렬 (Insertion Sort)                   │   │
│  │                                                 │   │
│  │  O(n log n):                                   │   │
│  │  • 병합 정렬 (Merge Sort)                       │   │
│  │  • 퀵 정렬 (Quick Sort)                         │   │
│  │  • 힙 정렬 (Heap Sort)                          │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  ⚡ 비비교 기반 정렬 (Non-comparison):                 │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 계수 정렬 (Counting Sort) - O(n+k)          │   │
│  │  • 기수 정렬 (Radix Sort) - O(d(n+k))          │   │
│  │  • 버킷 정렬 (Bucket Sort) - O(n+k)            │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🔀 하이브리드 정렬:                                   │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 팀소트 (Timsort) - Python, Java            │   │
│  │  • 인트로소트 (Introsort) - C++ STL            │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 4. 알고리즘 선택 기준
| 상황 | 추천 알고리즘 |
|-----|-------------|
| 데이터 작음 | 삽입 정렬 |
| 거의 정렬됨 | 삽입 정렬 |
| 안정성 필요 | 병합 정렬 |
| 평균 빠름 | 퀵 정렬 |
| 최악 보장 | 힙, 병합 정렬 |
| 메모리 제약 | 힙 정렬 |
| 범위 제한 정수 | 계수/기수 정렬 |

### Ⅲ. 기술 비교 분석

#### 3. 시간복잡도 비교
| 알고리즘 | 최선 | 평균 | 최악 | 공간 | 안정성 |
|---------|------|------|------|------|--------|
| 버블 | O(n) | O(n²) | O(n²) | O(1) | O |
| 선택 | O(n²) | O(n²) | O(n²) | O(1) | X |
| 삽입 | O(n) | O(n²) | O(n²) | O(1) | O |
| 병합 | O(n log n) | O(n log n) | O(n log n) | O(n) | O |
| 퀵 | O(n log n) | O(n log n) | O(n²) | O(log n) | X |
| 힙 | O(n log n) | O(n log n) | O(n log n) | O(1) | X |

#### 5. 장단점
| 알고리즘 | 장점 | 단점 |
|---------|------|------|
| 퀵 | 평균 빠름 | 최악 O(n²) |
| 병합 | 안정, 보장 | 공간 O(n) |
| 힙 | 공간 효율 | 불안정 |
| 삽입 | 간단, 거의 정렬시 빠름 | 느림 |
