# 백트래킹 (Backtracking)

## 핵심 인사이트 (3줄 요약)
> **가능한 모든 경우를 탐색하며 실패 시 되돌아감**. DFS 기반 가지치기. 제약 충족 문제에 적합.

### Ⅰ. 개요

#### 1. 개념
백트래킹(Backtracking)은 **해를 찾는 과정에서 막다른 길에 도달하면 이전 상태로 되돌아가 다른 경로를 탐색하는 알고리즘 기법**이다. 모든 가능한 경우를 체계적으로 탐색하며, 불가능한 경로는 조기에 포기한다.

> 비유: "미로 찾기" - 막다른 길이면 돌아와요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 백트래킹 구조
```
┌────────────────────────────────────────────────────────┐
│           백트래킹 동작 원리                           │
├────────────────────────────────────────────────────────┤
│                                                        │
│  🔄 핵심 과정:                                         │
│  ┌────────────────────────────────────────────────┐   │
│  │  1. 상태 선택 (Choose)                         │   │
│  │     - 가능한 선택지 중 하나를 선택              │   │
│  │                                                 │   │
│  │  2. 제약 확인 (Constraint Check)               │   │
│  │     - 선택이 유효한지 검증                      │   │
│  │                                                 │   │
│  │  3. 목표 확인 (Goal Check)                     │   │
│  │     - 해를 찾았는지 확인                        │   │
│  │                                                 │   │
│  │  4. 재귀 탐색 (Explore)                        │   │
│  │     - 다음 단계로 진행                          │   │
│  │                                                 │   │
│  │  5. 백트랙 (Backtrack)                         │   │
│  │     - 실패 시 이전 상태로 복구                  │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  ✂️ 가지치기 (Pruning):                               │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 유망하지 않은 경로 조기 제거                 │   │
│  │  • 탐색 공간 대폭 감소                          │   │
│  │  • 최적화를 통해 성능 향상                      │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 대표 문제
| 문제 | 설명 | 시간복잡도 |
|-----|------|-----------|
| N-Queen | N×N 체스판에 N개 퀸 배치 | O(N!) |
| 부분 집합 | 모든 부분 집합 생성 | O(2^n) |
| 순열 | 모든 순열 생성 | O(n!) |
| 조합 | n개 중 r개 선택 | O(C(n,r)) |
| 미로 찾기 | 출발점에서 도착점 경로 | O(4^(m×n)) |
| 스도쿠 | 9×9 퍼즐 해결 | 지수 |

### Ⅲ. 기술 비교 분석

#### 4. 백트래킹 vs 브루트포스
| 구분 | 브루트포스 | 백트래킹 |
|-----|-----------|----------|
| 탐색 | 모든 경우 | 유망한 경우만 |
| 가지치기 | 없음 | 있음 |
| 효율성 | 낮음 | 상대적 높음 |
| 구현 | 단순 | 복잡 |

#### 5. 장단점
| 장점 | 단점 |
|-----|------|
| 모든 해 탐색 가능 | 지수 시간 복잡도 |
| 최적해 보장 | 문제 크기에 민감 |
| 구조적 탐색 | 메모리 사용 (재귀) |

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **알고리즘 효율** | 최적 알고리즘 적용으로 복잡도 대폭 감소 | O(n²) → O(n log n) 수준 개선 |
| **시스템 성능** | 빠른 자료구조·탐색 알고리즘으로 응답 시간 단축 | 대용량 데이터 처리 10배 향상 |
| **의사결정 품질** | 통계적 검증으로 신뢰 있는 데이터 기반 판단 제공 | 의사결정 오류율 50% 감소 |
