# 탐욕 알고리즘 (Greedy Algorithm)

## 핵심 인사이트 (3줄 요약)
> **매 순간 최적 선택이 전체 최적으로 이어짐**. 지역 최적 → 전역 최적. 조건 확인 필수.

### Ⅰ. 개요

#### 1. 개념
탐욕 알고리즘(Greedy Algorithm)은 **각 단계에서 현재 상황에서 가장 최적인 선택을 하여 전체 문제의 최적해를 구하는 알고리즘**이다. 매 순간의 선택이 이후 선택에 영향을 주지 않는다는 가정하에 동작한다.

> 비유: "매장에서 가장 싼 것만 사기" - 그 순간 최선이에요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 탐욕 알고리즘 조건
```
┌────────────────────────────────────────────────────────┐
│           탐욕 알고리즘 적용 조건                      │
├────────────────────────────────────────────────────────┤
│                                                        │
│  ✅ 필수 조건:                                         │
│  ┌────────────────────────────────────────────────┐   │
│  │  1. 탐욕적 선택 속성 (Greedy Choice Property)   │   │
│  │     - 지역 최적 선택 = 전역 최적해의 일부        │   │
│  │     - 이전 선택이 이후 선택에 영향 없음          │   │
│  │                                                 │   │
│  │  2. 최적 부분 구조 (Optimal Substructure)       │   │
│  │     - 최적해의 부분도 최적해                    │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🔄 알고리즘 흐름:                                     │
│  ┌────────────────────────────────────────────────┐   │
│  │  1. Selection: 현재 최적 선택                   │   │
│  │  2. Feasibility: 선택이 유효한지 확인           │   │
│  │  3. Solution: 완료되었는지 확인                 │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  ⚠️ 주의: 모든 문제에 적용 가능하지 않음              │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 배낭 문제 (분할 가능 → O, 0-1 → X)          │   │
│  │  • 최단 경로 (음수 간선 → X)                   │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 대표 문제
| 문제 | 전략 | 시간복잡도 |
|-----|------|-----------|
| 동전 거스름 | 큰 단위부터 | O(n) |
| 활동 선택 | 종료 시간 기준 정렬 | O(n log n) |
| 최소 신장 트리 | Kruskal, Prim | O(E log V) |
| 최단 경로 | 다익스트라 | O(E log V) |
| 허프만 코딩 | 빈도 기반 | O(n log n) |
| 분할 가능 배낭 | 가치/무게 비율 | O(n log n) |

### Ⅲ. 기술 비교 분석

#### 4. Greedy vs DP
| 구분 | Greedy | DP |
|-----|--------|-----|
| 선택 | 매 순간 최적 | 모든 경우 고려 |
| 복잡도 | 낮음 | 높음 |
| 최적해 | 조건부 보장 | 항상 보장 |
| 메모리 | 적음 | 많음 |
| 적용 | 제한적 | 범용적 |

#### 5. 장단점
| 장점 | 단점 |
|-----|------|
| 구현 간단 | 최적해 보장 안 됨 |
| 빠른 수행 | 조건 확인 필요 |
| 메모리 효율 | 근사해일 수 있음 |
