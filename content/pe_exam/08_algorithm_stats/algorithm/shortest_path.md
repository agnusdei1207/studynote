# 최단 경로 알고리즘 (Shortest Path)

## 핵심 인사이트 (3줄 요약)

### Ⅰ. 개요

#### 1. 개념
최단 경로 알고리즘은 **가중치 그래프에서 두 정점 사이의 최단 거리 또는 경로를 찾는 알고리즘**이다. 네비게이션, 네트워크 라우팅 등에 활용된다.

> 비유: "내비게이션" - 가장 빠른 길 찾아줘요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 주요 알고리즘 비교
```
┌────────────────────────────────────────────────────────┐
│           최단 경로 알고리즘 비교                      │
├────────────────────────────────────────────────────────┤
│                                                        │
│  📍 다익스트라 (Dijkstra)                             │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 양의 가중치만 허용                           │   │
│  │  • 단일 출발점 → 모든 정점                      │   │
│  │  • 시간복잡도: O((V+E)logV)                    │   │
│  │  • 자료구조: 우선순위 큐                        │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🔄 벨만-포드 (Bellman-Ford)                          │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 음의 가중치 허용                             │   │
│  │  • 음수 사이클 탐지 가능                        │   │
│  │  • 시간복잡도: O(VE)                           │   │
│  │  • 느리지만 범용적                              │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🔗 플로이드-워셜 (Floyd-Warshall)                    │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 모든 쌍 최단 경로                            │   │
│  │  • 음의 가중치 허용                             │   │
│  │  • 시간복잡도: O(V³)                           │   │
│  │  • 동적 프로그래밍 기반                         │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 3. 알고리즘 선택 기준
| 상황 | 추천 알고리즘 |
|-----|-------------|
| 양수 가중치, 단일 출발점 | 다익스트라 |
| 음수 가중치 가능 | 벨만-포드 |
| 모든 쌍 최단 경로 | 플로이드-워셜 |
| 가중치 없음 | BFS |

#### 4. 다익스트라 알고리즘
```
동작 과정:
1. 출발점 거리 = 0, 나머지 = ∞
2. 방문하지 않은 정점 중 최소 거리 선택
3. 인접 정점들의 거리 갱신 (완화)
4. 모든 정점 방문까지 반복

예시:
    A --2--> B --3--> D
    |        |
    5        1
    |        |
    v        v
    C --2--> E

A→B: 2, A→C: 5, A→D: 5, A→E: 3
```

#### 5. 벨만-포드 알고리즘
```
동작 과정:
1. 출발점 거리 = 0, 나머지 = ∞
2. 모든 간선에 대해 완화 (V-1회 반복)
3. 음수 사이클 확인

특징:
- 음수 간선 처리 가능
- 음수 사이클 탐지
- 다익스트라보다 느림
```

#### 6. 플로이드-워셜 알고리즘
```
동작 과정:
D[i][j] = min(D[i][j], D[i][k] + D[k][j])

모든 정점 쌍에 대해 중간 정점 k를 거쳐가는 경로 고려

특징:
- O(V³) 시간복잡도
- 밀집 그래프에 적합
- 동적 프로그래밍
```

### Ⅲ. 기술 비교 분석

#### 7. 장단점
| 알고리즘 | 장점 | 단점 |
|---------|------|------|
| 다익스트라 | 빠름 | 음수 가중치 불가 |
| 벨만-포드 | 음수 처리 | 느림 |
| 플로이드-워셜 | 모든 쌍 | 매우 느림 |

---

### Ⅳ. 실무 적용 방안

#### 8. 활용 분야
| 분야 | 적용 |
|-----|------|
| 네비게이션 | 경로 안내 |
| 네트워크 | 라우팅 |
| 게임 | 길찾기 |
| 물류 | 배송 경로 |

#### 10. 실무에선? (기술사적 판단)
**알고리즘 선택:**
- 대부분: 다익스트라 (빠름)
- 음수 가중치: 벨만-포드
- 모든 쌍: 플로이드-워셜

**최적화:**
- 우선순위 큐 사용
- 인접 리스트 vs 인접 행렬

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **알고리즘 효율** | 최적 알고리즘 적용으로 복잡도 대폭 감소 | O(n²) → O(n log n) 수준 개선 |
| **시스템 성능** | 빠른 자료구조·탐색 알고리즘으로 응답 시간 단축 | 대용량 데이터 처리 10배 향상 |
| **의사결정 품질** | 통계적 검증으로 신뢰 있는 데이터 기반 판단 제공 | 의사결정 오류율 50% 감소 |
