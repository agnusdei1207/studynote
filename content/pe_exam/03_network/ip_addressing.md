### Ⅰ. 개요

#### 1. 개념
IP(Internet Protocol) 주소는 **네트워크 상에서 각 장치를 고유하게 식별하는 논리 주소**다.

> 비유: "집 주소" - 우편번호(네트워크 주소)와 상세주소(호스트 주소)로 구성

### Ⅱ. 구성 요소 및 핵심 원리

#### 3. IPv4 클래스 체계
```
┌──────────────────────────────────────────────────────────┐
│                IPv4 클래스 분류                           │
├────────┬───────────────────┬──────────┬──────────────────┤
│ 클래스 │ 범위              │ 네트워크 │ 호스트 수       │
├────────┼───────────────────┼──────────┼──────────────────┤
│   A    │ 1.0.0.0 ~         │ /8       │ 16,777,214       │
│        │ 126.255.255.255   │ (8비트)  │ (대형 ISP)      │
├────────┼───────────────────┼──────────┼──────────────────┤
│   B    │ 128.0.0.0 ~       │ /16      │ 65,534           │
│        │ 191.255.255.255   │ (16비트) │ (중형 기관)     │
├────────┼───────────────────┼──────────┼──────────────────┤
│   C    │ 192.0.0.0 ~       │ /24      │ 254              │
│        │ 223.255.255.255   │ (24비트) │ (소형 네트워크) │
├────────┼───────────────────┼──────────┼──────────────────┤
│   D    │ 224.0.0.0 ~       │ 멀티캐스트용              │
│        │ 239.255.255.255   │                              │
├────────┼───────────────────┼──────────┼──────────────────┤
│   E    │ 240.0.0.0 ~       │ 연구/예약용               │
└────────┴───────────────────┴──────────┴──────────────────┘

사설 주소 (RFC 1918):
- 10.0.0.0/8       (A클래스)
- 172.16.0.0/12    (B클래스 일부)
- 192.168.0.0/16   (C클래스)
```

#### 4. 서브네팅 (Subnetting)
```
서브네팅: 하나의 네트워크를 더 작은 서브넷으로 분할

서브넷 마스크: 네트워크 부분과 호스트 부분을 구분
예) 255.255.255.0 = /24 = 11111111.11111111.11111111.00000000

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
예제: 192.168.10.0/24를 4개 서브넷으로 분할

필요 서브넷: 4개 → 2² = 4 → 2비트 차용
새 마스크: /26 = 255.255.255.192

서브넷 1: 192.168.10.0/26   (0~63)   호스트: .1~.62
서브넷 2: 192.168.10.64/26  (64~127) 호스트: .65~.126
서브넷 3: 192.168.10.128/26 (128~191) 호스트: .129~.190
서브넷 4: 192.168.10.192/26 (192~255) 호스트: .193~.254

각 서브넷 사용 가능 호스트: 2⁶ - 2 = 62개
(첫 번째: 네트워크 주소, 마지막: 브로드캐스트 제외)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

VLSM (Variable Length Subnet Masking):
서로 다른 크기의 서브넷을 혼합해 IP 주소 효율화
```

#### 5. CIDR (Classless Inter-Domain Routing)
```
클래스 구분 없이 프리픽스 길이로 표기:
192.168.1.0/24 → 네트워크: 24비트, 호스트: 8비트

슈퍼네팅(Supernetting): 작은 네트워크를 합쳐 큰 블록
192.168.0.0/24 + 192.168.1.0/24 → 192.168.0.0/23

라우팅 테이블 축소 효과 (경로 집약, Route Aggregation)
```

#### 6. NAT (Network Address Translation)
```
사설 IP ←→ 공인 IP 변환

                ┌──────────────┐
사설망          │   NAT 라우터  │          인터넷
192.168.1.10 → │ 사설IP:포트  → 공인IP:포트 → 목적지
192.168.1.11 → │ 매핑 테이블  │
                └──────────────┘

NAT 유형:
- Static NAT: 1:1 고정 매핑
- Dynamic NAT: 풀에서 자동 할당
- PAT (Port Address Translation, Masquerading):
  여러 사설IP → 단일 공인IP, 포트로 구분 (가장 일반적)

장점: IPv4 주소 절약, 내부 구조 숨김
단점: 종단간 연결 어려움, P2P/VoIP 문제
```

### Ⅲ. 기술 비교 분석

#### 2. IPv4 vs IPv6 비교
| 항목 | IPv4 | IPv6 |
|------|------|------|
| 비트 수 | 32비트 | 128비트 |
| 표기 | 점으로 구분 (192.168.1.1) | 콜론 구분 (2001:db8::1) |
| 주소 수 | 약 43억 (2³²) | 3.4×10³⁸ (2¹²⁸) |
| 헤더 크기 | 20~60 바이트 | 40 바이트 (고정) |
| NAT 필요 | 필요 (주소 부족) | 불필요 |
| 보안 | IPSec 선택적 | IPSec 내장 |
| QoS | 제한적 | Flow Label로 향상 |
| 체크섬 | 있음 | 없음 (상위 계층 처리) |
| 브로드캐스트 | 있음 | 없음 (멀티캐스트 사용) |

#### 7. 다른 것과 비교
| 항목 | IPv4 | IPv6 | NAT |
|------|------|------|-----|
| 주소 고갈 | 문제 | 해결 | 임시 해결 |
| 헤더 복잡성 | 복잡 | 단순 | - |
| 보안 | 별도 설정 | 내장 | 내부 숨김 |
| 전환 비용 | - | 높음 | 낮음 |
| 현재 상용 | 주류 | 증가 중 | 광범위 |

### Ⅳ. 실무 적용 방안

#### 8. 실무에선? (기술사적 판단)
- **이중 스택**: IPv4/IPv6 동시 운영 전환 방식
- **터널링**: IPv6 패킷을 IPv4로 캡슐화 (6to4, Teredo)
- **클라우드**: VPC 설계에서 서브네팅 직접 사용
- **기술사 포인트**: 서브네팅 계산, NAT 동작, IPv4→IPv6 전환 전략

> **IP 주소와 서브네팅 (IP Addressing & Subnetting)**은(는) 네트워크 기술은 5G·SDN·NFV를 통해 소프트웨어 중심으로 진화하고 있으며, AI 기반 자율 네트워크(Autonomous Network)가 차세대 통신 인프라의 핵심이 될 것이다.
