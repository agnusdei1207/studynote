# TCP와 UDP (Transmission Control Protocol / User Datagram Protocol)

## 핵심 인사이트 (3줄 요약)
> **TCP**: 연결형, 신뢰성 보장, 순서 보장. **UDP**: 비연결형, 빠름, 신뢰성 없음. 용도에 따라 선택: 파일 전송은 TCP, 실시간 스트리밍은 UDP.

### Ⅰ. 개요

#### 1. 개념
### TCP (Transmission Control Protocol)
**연결 지향형 프로토콜**로, 신뢰성 있는 데이터 전송을 보장한다.

> 비유: "등기 우편" - 수신 확인, 순서 보장

### UDP (User Datagram Protocol)
**비연결형 프로토콜**로, 빠른 전송에 중점을 둔다.

> 비유: "엽서" - 보내고 끝, 확인 없음

### Ⅱ. 구성 요소 및 핵심 원리

#### 3. TCP 상세
### 3.1 TCP 헤더
```
┌────────────────────────────────────────────────────┐
│     Source Port (16)    │  Destination Port (16) │
├────────────────────────────────────────────────────┤
│                 Sequence Number (32)               │
├────────────────────────────────────────────────────┤
│               Acknowledgment Number (32)           │
├───────┬───────┬───────────┬───────────────────────┤
│ HLEN  │  Flag │  Window   │     Checksum (16)     │
│ (4)   │ (6)   │  (16)     │                       │
├───────┴───────┴───────────┴───────────────────────┤
│               Urgent Pointer (16)                  │
├────────────────────────────────────────────────────┤
│                   Options                           │
└────────────────────────────────────────────────────┘

주요 플래그:
- SYN: 연결 설정
- ACK: 확인 응답
- FIN: 연결 종료
- RST: 연결 재설정
- PSH: 데이터 밀어넣기
- URG: 긴급 데이터
```

### 3.2 3-Way Handshake
```
연결 설정 과정:

Client                    Server
   │                         │
   │ ──── SYN ────────────→ │ (연결 요청)
   │      (SEQ=100)         │
   │                         │
   │ ←── SYN+ACK ─────────── │ (요청 수락 + 확인)
   │    (SEQ=300, ACK=101)  │
   │                         │
   │ ──── ACK ────────────→ │ (확인)
   │      (ACK=301)         │
   │                         │
   │      [연결 완료]         │

단계:
1. SYN: 클라이언트가 연결 요청
2. SYN+ACK: 서버가 요청 수락
3. ACK: 클라이언트가 확인
```

### 3.3 4-Way Handshake
```
연결 종료 과정:

Client                    Server
   │                         │
   │ ──── FIN ────────────→ │ (종료 요청)
   │                         │
   │ ←──── ACK ──────────── │ (확인)
   │                         │
   │ ←──── FIN ──────────── │ (종료 요청)
   │                         │
   │ ──── ACK ────────────→ │ (확인)
   │                         │
   │      [연결 종료]         │

TIME_WAIT:
- 마지막 ACK 유실 대비
- 2MSL(Max Segment Lifetime) 대기
```

### 3.4 슬라이딩 윈도우
```
흐름 제어 기법:

송신 윈도우:
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │
└───┴───┴───┴───┴───┴───┴───┴───┘
  ↑           ↑
  전송됨      전송 가능

수신 ACK에 따라 윈도우 이동:
- ACK 1 → 윈도우 오른쪽으로 이동
- 연속 전송 가능

장점:
- 효율적 전송
- 대기 시간 최소화
```

### 3.5 혼잡 제어
```
네트워크 혼잡 방지 알고리즘:

1. Slow Start
   - 초기 cwnd = 1 MSS
   - ACK마다 cwnd 2배
   - ssthresh까지 지수 증가

2. Congestion Avoidance
   - ssthresh 이후 선형 증가
   - RTT마다 cwnd + 1

3. Fast Retransmit
   - 3번의 중복 ACK → 즉시 재전송

4. Fast Recovery
   - ssthresh = cwnd/2
   - cwnd = ssthresh + 3
```

#### 4. UDP 상세
### 4.1 UDP 헤더
```
┌─────────────────────────────────────┐
│  Source Port (16) │ Dest Port (16) │
├───────────────────┼─────────────────┤
│   Length (16)     │  Checksum (16)  │
└───────────────────┴─────────────────┘

특징:
- 8바이트로 매우 간단
- 최소한의 기능만
```

### 4.2 UDP 특성
```
장점:
- 빠른 전송
- 오버헤드 적음
- 멀티캐스트/브로드캐스트 가능

단점:
- 신뢰성 없음
- 순서 보장 없음
- 흐름 제어 없음

적용:
- 실시간 스트리밍
- 온라인 게임
- DNS
- VoIP
```

#### 5. 포트 번호
```
잘 알려진 포트 (0~1023):
- 20/21: FTP
- 22: SSH
- 23: Telnet
- 25: SMTP
- 53: DNS
- 80: HTTP
- 110: POP3
- 143: IMAP
- 443: HTTPS
- 3306: MySQL
- 3389: RDP

등록된 포트 (1024~49151):
- 3000: 개발 서버
- 8080: 대체 HTTP

동적 포트 (49152~65535):
- 클라이언트 임시 포트
```

### Ⅲ. 기술 비교 분석

#### 2. TCP vs UDP 비교
| 항목 | TCP | UDP |
|------|-----|-----|
| 연결 | 연결형 | 비연결형 |
| 신뢰성 | 보장 | 미보장 |
| 순서 | 보장 | 미보장 |
| 흐름제어 | 있음 | 없음 |
| 혼잡제어 | 있음 | 없음 |
| 속도 | 느림 | 빠름 |
| 헤더 | 20바이트 | 8바이트 |
| 용도 | 파일, 웹 | 스트리밍, 게임 |

#### 7. 장단점
### TCP
| 장점 | 단점 |
|-----|------|
| 신뢰성 보장 | 오버헤드 큼 |
| 순서 보장 | 느린 속도 |
| 흐름 제어 | 연결 관리 복잡 |
| 혼잡 제어 | 멀티캐스트 불가 |

### UDP
| 장점 | 단점 |
|-----|------|
| 빠른 속도 | 신뢰성 없음 |
| 오버헤드 적음 | 순서 보장 없음 |
| 멀티캐스트 가능 | 흐름 제어 없음 |
| 단순함 | 혼잡 제어 없음 |

### Ⅳ. 실무 적용 방안

#### 8. 실무에선? (기술사적 판단)
- **웹 서비스**: TCP (HTTP/3는 QUIC 사용)
- **스트리밍**: UDP + 애플리케이션 계층 신뢰성
- **게임**: UDP (지연 민감)
- **파일 전송**: TCP (신뢰성 중요)
