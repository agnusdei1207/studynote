# 라우팅 (Routing)

## 핵심 인사이트 (3줄 요약)
> **출발지에서 목적지까지 최적 경로를 결정**하는 과정. 라우팅 알고리즘/프로토콜이 경로를 계산하고 라우터가 패킷을 전달한다. 정적 라우팅과 동적 라우팅으로 구분된다.

### Ⅰ. 개요

#### 1. 개념
라우팅은 **네트워크에서 패킷이 목적지까지 도달하기 위한 최적 경로를 결정**하고, 그 경로로 패킷을 전달하는 과정이다.

> 비유: "네비게이션" - 목적지까지 가장 빠른 길을 찾아 안내

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 라우팅 기본 요소
```
┌─────────────────────────────────────────┐
│              라우팅 테이블              │
├──────────┬──────────┬──────────┬───────┤
│ 목적지    │  다음 홉 │  인터페이스│ 메트릭 │
├──────────┼──────────┼──────────┼───────┤
│ 10.0.1.0 │ 10.0.0.1 │   eth0   │   1   │
│ 10.0.2.0 │ 10.0.0.2 │   eth1   │   2   │
│ 0.0.0.0  │ 10.0.0.1 │   eth0   │   -   │
└──────────┴──────────┴──────────┴───────┘

용어:
- 홉(Hop): 패킷이 거치는 라우터
- 메트릭(Metric): 경로의 비용
- 다음 홉(Next Hop): 다음 목적지
```

#### 3. 라우팅 분류
### 3.1 정적 라우팅 (Static Routing)
```
관리자가 수동으로 경로 설정

특징:
- 경로 고정
- 설정 단순
- 변경 시 관리자 개입 필요
- 소규모 네트워크에 적합

예시:
ip route 192.168.2.0 255.255.255.0 192.168.1.1
```

### 3.2 동적 라우팅 (Dynamic Routing)
```
라우팅 프로토콜이 자동으로 경로 계산

특징:
- 경로 자동 갱신
- 장애 시 우회 가능
- 프로토콜 오버헤드
- 대규모 네트워크에 적합
```

#### 4. 라우팅 알고리즘
### 4.1 거리 벡터 (Distance Vector)
```
개념: 인접 라우터와 거리 정보 교환

알고리즘: 벨만-포드 (Bellman-Ford)
프로토콜: RIP, IGRP

동작:
1. 인접 라우터에게 자신의 거리 벡터 전송
2. 수신한 정보로 자신의 테이블 갱신
3. 변화가 있으면 다시 전파

특징:
- 구현 단순
- 느린 수렴
- 루프 가능성 (Count-to-Infinity)
```

### 4.2 링크 상태 (Link State)
```
개념: 전체 네트워크 토폴로지 파악 후 최단 경로 계산

알고리즘: 다익스트라 (Dijkstra's SPF)
프로토콜: OSPF, IS-IS

동작:
1. 인접 링크 상태 정보 수집
2. 전체 네트워크 맵 구성
3. SPF로 최단 경로 계산

특징:
- 빠른 수렴
- 루프 방지
- 메모리/CPU 사용 많음
```

### 4.3 경로 벡터 (Path Vector)
```
개념: 경로 정보를 포함하여 루프 방지

프로토콜: BGP

동작:
- AS(자율시스템) 경로 정보 포함
- 경로상의 AS 목록 확인
- 루프 감지 및 방지

특징:
- 대규모 네트워크용
- 정책 기반 라우팅 가능
```

#### 5. 주요 라우팅 프로토콜
### 5.1 RIP (Routing Information Protocol)
```
타입: 거리 벡터
메트릭: 홉 수 (최대 15)
갱신: 30초마다 브로드캐스트

특징:
- 구현 간단
- 소규모용
- 느린 수렴
- 홉 수 제한

RIP v2:
- 서브넷 마스크 지원
- 멀티캐스트 사용
- 인증 지원
```

### 5.2 OSPF (Open Shortest Path First)
```
타입: 링크 상태
메트릭: 비용 (대역폭 기반)
갱신: 변경 시 즉시

특징:
- 빠른 수렴
- 계층적 구조 (Area)
- VLSM 지원
- 인증 지원

구조:
┌─────────────────────────────┐
│       Area 0 (Backbone)      │
│    ┌─────┐     ┌─────┐      │
│    │     │─────│     │      │
│    └─────┘     └─────┘      │
├──────────────┬──────────────┤
│   Area 1     │    Area 2    │
│   ┌─────┐    │    ┌─────┐   │
│   │     │    │    │     │   │
│   └─────┘    │    └─────┘   │
└──────────────┴──────────────┘
```

### 5.3 BGP (Border Gateway Protocol)
```
타입: 경로 벡터
용도: AS 간 라우팅 (인터넷)
메트릭: AS-PATH, 정책

특징:
- 인터넷의 핵심 프로토콜
- 정책 기반 경로 선택
- 신뢰성 중시
- 느린 수렴

동작:
eBGP: 다른 AS와 통신
iBGP: 같은 AS 내부 통신
```

#### 7. 라우팅 결정 과정
```
패킷 수신
    ↓
┌─────────────────┐
│ 목적지 IP 확인   │
└────────┬────────┘
         ↓
┌─────────────────┐
│ 라우팅 테이블 조회│
│ 최장 프리픽스 매칭│
└────────┬────────┘
         ↓
    ┌────┴────┐
    │ 매칭 있음? │
    └────┬────┘
    ┌────┴────┐
   Yes       No
    │         │
    ↓         ↓
┌───────┐ ┌────────┐
│전달   │ │기본 경로│
│       │ │또는 폐기│
└───────┘ └────────┘
```

### Ⅲ. 기술 비교 분석

#### 6. 라우팅 프로토콜 비교
| 항목 | RIP | OSPF | EIGRP | BGP |
|------|-----|------|-------|-----|
| 타입 | 거리벡터 | 링크상태 | 하이브리드 | 경로벡터 |
| 메트릭 | 홉 | 비용 | 복합 | AS경로 |
| 수렴 | 느림 | 빠름 | 매우빠름 | 느림 |
| 규모 | 소규모 | 중~대 | 중~대 | 초대형 |
| 계층 | 없음 | Area | AS | AS |

#### 9. 장단점
### 정적 라우팅
| 장점 | 단점 |
|-----|------|
| 단순 | 수동 관리 |
| 예측 가능 | 확장성 낮음 |
| 리소스 적음 | 장애 대응 못함 |

### 동적 라우팅
| 장점 | 단점 |
|-----|------|
| 자동 장애 복구 | 복잡도 증가 |
| 확장성 | 오버헤드 |
| 유연성 | 수렴 시간 |

### Ⅳ. 실무 적용 방안

#### 10. 실무에선? (기술사적 판단)
- **기업망**: OSPF가 주류
- **인터넷**: BGP가 필수
- **소규모**: 정적 라우팅 충분
- **SDN**: 중앙 집중형 라우팅으로 변화

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **통신 성능** | 최적화된 프로토콜·라우팅으로 지연 및 패킷 손실 감소 | 네트워크 지연 50% 단축 |
| **확장성** | 소프트웨어 정의 방식으로 트래픽 급증에도 유연 대응 | 대역폭 활용률 80% 이상 |
| **보안·안정성** | 계층적 보안 아키텍처로 가용성 및 무결성 보장 | SLA 99.99% (4-nine) 달성 |

#### 결론
> **라우팅 (Routing)**은(는) 네트워크 기술은 5G·SDN·NFV를 통해 소프트웨어 중심으로 진화하고 있으며, AI 기반 자율 네트워크(Autonomous Network)가 차세대 통신 인프라의 핵심이 될 것이다.

> **※ 참고 표준**: RFC 표준 시리즈, ETSI NFV ISG, 3GPP TS 23.501, ITU-T 권고안
