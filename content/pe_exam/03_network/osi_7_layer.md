# OSI 7계층 (OSI 7 Layer Model)

## 핵심 인사이트 (3줄 요약)
> 네트워크 통신을 **7개 계층으로 표준화**한 국제 참조 모델. 각 계층은 독립적인 기능을 수행하며 상하위 계층과 인터페이스한다. 실제 프로토콜은 TCP/IP가 주류이지만, OSI 모델은 학습과 문제 해결의 기준이 된다.

### Ⅰ. 개요

#### 1. 개념
OSI(Open Systems Interconnection) 7계층은 **ISO(국제표준화기구)가 제정한 네트워크 통신 표준 모델**로, 통신 과정을 7단계로 나누어 체계적으로 설명한다.

> 비유: "택배 배송 과정" - 포장 → 하차 → 운송 → 허브 → 지역 배송 → 수령

#### 2. 등장 배경
- **1970년대**: 다양한 통신 시스템 간 호환성 문제
- **해결**: 표준화된 참조 모델 필요
- **1984년**: ISO에서 OSI 참조 모델 표준화

### Ⅱ. 구성 요소 및 핵심 원리

#### 3. 7계층 구조 (상위→하위)
| 계층 | 이름 | PDU | 주요 기능 | 장비/프로토콜 |
|------|------|-----|----------|--------------|
| 7 | 응용 (Application) | Data | 사용자 인터페이스 | HTTP, FTP, SMTP |
| 6 | 표현 (Presentation) | Data | 데이터 변환, 암호화 | SSL/TLS, JPEG |
| 5 | 세션 (Session) | Data | 세션 관리, 동기화 | RPC, NetBIOS |
| 4 | 전송 (Transport) | Segment | 종단간 신뢰성 | TCP, UDP |
| 3 | 네트워크 (Network) | Packet | 라우팅, 논리 주소 | IP, 라우터 |
| 2 | 데이터링크 (Data Link) | Frame | 프레이밍, MAC 주소 | 이더넷, 스위치 |
| 1 | 물리 (Physical) | Bit | 비트 전송 | 케이블, 허브 |

#### 4. 계층별 상세 설명
### 4.1 물리 계층 (Physical Layer)
```
기능: 비트를 전기/광 신호로 변환하여 전송

전송 매체:
- 유선: UTP, 광케이블, 동축케이블
- 무선: 전파, 적외선

장비: 허브, 리피터, 케이블
```

### 4.2 데이터링크 계층 (Data Link Layer)
```
기능:
- 프레이밍: 비트를 프레임으로 구성
- 주소 지정: MAC 주소 사용
- 오류 제어: CRC 검사
- 흐름 제어: 수신 버퍼 관리

하위 계층:
- LLC (Logical Link Control)
- MAC (Media Access Control)

장비: 스위치, 브리지
```

### 4.3 네트워크 계층 (Network Layer)
```
기능:
- 라우팅: 최적 경로 결정
- 논리 주소: IP 주소 할당
- 패킷 전달: 종단 간 전송

프로토콜: IP, ICMP, ARP
장비: 라우터, L3 스위치
```

### 4.4 전송 계층 (Transport Layer)
```
기능:
- 종단 간 신뢰성 보장
- 세그먼트 분할/재조립
- 포트 주소 사용
- 흐름 제어, 혼잡 제어

프로토콜:
- TCP: 연결형, 신뢰성
- UDP: 비연결형, 빠름
```

### 4.5 세션 계층 (Session Layer)
```
기능:
- 세션 생성, 관리, 종료
- 동기화 포인트 설정
- 전이중/반이중 제어

프로토콜: RPC, NetBIOS, SIP
```

### 4.6 표현 계층 (Presentation Layer)
```
기능:
- 데이터 형식 변환
- 암호화/복호화
- 압축/해제

표준: ASN.1, X.509
```

### 4.7 응용 계층 (Application Layer)
```
기능:
- 사용자 인터페이스 제공
- 네트워크 서비스 접근

프로토콜: HTTP, FTP, SMTP, DNS, SSH
```

#### 5. 데이터 캡슐화/역캡슐화
```
[송신 측 - 캡슐화]
응용 계층: Data
    ↓ + 헤더
표현 계층: Data
    ↓ + 헤더
세션 계층: Data
    ↓ + 헤더
전송 계층: Segment (TCP 헤더)
    ↓ + 헤더
네트워크 계층: Packet (IP 헤더)
    ↓ + 헤더
데이터링크 계층: Frame (이더넷 헤더)
    ↓
물리 계층: Bits

[수신 측 - 역캡슐화]
반대 과정
```

### Ⅲ. 기술 비교 분석

#### 6. OSI vs TCP/IP 비교
| OSI 7계층 | TCP/IP 4계층 |
|----------|-------------|
| 응용 | 응용 |
| 표현 | 응용 |
| 세션 | 응용 |
| 전송 | 전송 |
| 네트워크 | 인터넷 |
| 데이터링크 | 네트워크 접근 |
| 물리 | 네트워크 접근 |

#### 7. 장단점
| 장점 | 단점 |
|-----|------|
| 표준화된 모델 | 실제 구현 복잡 |
| 계층별 독립성 | TCP/IP가 실질적 표준 |
| 문제 해결 용이 | 일부 계층 모호 |
| 학습 도구로 최적 | 구현 예시 부족 |

### Ⅳ. 실무 적용 방안

#### 8. 실무 활용
### 문제 해결
```
1계층 문제: 케이블 단선, 포트 불량
2계층 문제: MAC 충돌, 스위치 설정
3계층 문제: 라우팅 오류, IP 설정
4계층 문제: 포트 차단, 방화벽
7계층 문제: 애플리케이션 설정
```

### 네트워크 장비 선택
```
L2 스위치: 2계층 처리 (MAC 기반)
L3 스위치: 3계층 처리 (IP 라우팅)
L4~L7 스위치: 상위 계층 처리 (로드밸런싱)
```

#### 10. 실무에선? (기술사적 판단)
- **트러블슈팅**: 계층별로 문제 격리하여 해결
- **보안**: 각 계층별 보안 적용 (물리→암호화)
- **성능 최적화**: 병목 계층 식별
- **장비 선정**: 필요한 계층 처리 능력 고려

| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **통신 성능** | 최적화된 프로토콜·라우팅으로 지연 및 패킷 손실 감소 | 네트워크 지연 50% 단축 |

| **보안·안정성** | 계층적 보안 아키텍처로 가용성 및 무결성 보장 | SLA 99.99% (4-nine) 달성 |

#### 결론
> **OSI 7계층 (OSI 7 Layer Model)**은(는) 네트워크 기술은 5G·SDN·NFV를 통해 소프트웨어 중심으로 진화하고 있으며, AI 기반 자율 네트워크(Autonomous Network)가 차세대 통신 인프라의 핵심이 될 것이다.
