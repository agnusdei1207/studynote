# 순환 신경망 (RNN / LSTM / GRU)

## 핵심 인사이트 (3줄 요약)
> **RNN(Recurrent Neural Network)**은 순차 데이터(시계열, 자연어)를 처리하기 위해 내부에 순환 구조(Hidden State)를 가진 신경망이다. RNN의 고질적 문제인 장기 의존성 소실을 해결하기 위해 게이트 매커니즘을 도입한 **LSTM과 GRU**가 순차 데이터의 표준으로 사용되었다. 2017년 이후 Transformer에 많은 자리를 내주었으나, 시계열 예측·경량 환경에서 여전히 활발히 사용된다.

---

### Ⅰ. 개요  ↔  개념 + 등장 배경

**개념**: 은닉층의 결과가 다시 동일한 은닉층의 입력으로 들어가는 순환 구조를 통해 과거의 정보를 기억하고, 이를 바탕으로 현재의 출력을 결정하는 인공신경망 아키텍처이다.

> 비유: "책을 읽을 때 앞 문장의 맥락을 기억하면서 현재 문장을 이해하는 과정"

**등장 배경**:
- 기존 FNN/CNN 한계: 입력/출력 길이가 고정되어 있고, 연속성(순서)을 고려하지 못함
- 자연어, 주가 데이터, 음성 신호 등 길이를 알 수 없는 순차 데이터 처리 필요
- **RNN 구조 제안**: 과거 상태를 기억하는 은닉 상태(Hidden State) 도입
- **RNN의 한계**: 문장이 길어지면 역전파 시 기울기 소실(Vanishing Gradient) 발생 → 장기 기억 불가
- **LSTM(1997, Hochreiter) 등장**: 게이트(Gate)와 셀 상태(Cell State) 도입으로 장기 의존성 문제 해결

---

### Ⅱ. 구성 요소 및 핵심 원리

**기본 RNN 구조**:
```
h_t = tanh(W_hh * h_{t-1} + W_xh * x_t + b_h)
y_t = W_hy * h_t + b_y

- 현재 상태(h_t)는 이전 상태(h_{t-1})와 현재 입력(x_t)의 결합
- 문제점: 거리가 먼 입력의 기울기가 W의 곱셈 반복으로 0 수렴
```

**LSTM (Long Short-Term Memory) 핵심 원리**:
내부에 3개의 게이트(Gate)를 통해 '어떤 정보를 버리고, 어떤 정보를 기억할지' 제어.
| 구성 요소 | 역할 | 비유 |
|----------|------|------|
| 망각 게이트 (Forget) | 과거 정보를 얼마나 버릴지 결정 (0~1) | 지우개 |
| 입력 게이트 (Input) | 현재 정보를 셀 상태에 얼마나 추가할지 | 새 메모 쓰기 |
| 셀 상태 (Cell State) | 핵심 기억을 시퀀스 끝까지 전달하는 컨베이어 벨트 | 장기 기억 병렬 회선 |
| 출력 게이트 (Output)| 셀 상태와 현재 입력을 바탕으로 최종 출력 | 요약본 제출 |

**GRU (Gated Recurrent Unit)**:
LSTM의 복잡한 구조를 간소화하여 속도를 높인 모델.
- 셀 상태와 은닉 상태 통합
- 망각/입력 게이트를 업데이트 게이트(Update Gate) 하나로 통합
- 연산량 감소, 소량 데이터에서 LSTM과 유사하거나 나은 성능

**코드 예시** (PyTorch 기반 LSTM 시계열 예측):
```python
import torch
import torch.nn as nn

class LSTMModel(nn.Module):
    def __init__(self, input_size=1, hidden_size=64, num_layers=2, output_size=1):
        super(LSTMModel, self).__init__()
        self.hidden_size = hidden_size
        self.num_layers = num_layers
        
        # batch_first=True → (Batch, Sequence Length, Features)
        self.lstm = nn.LSTM(input_size, hidden_size, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_size, output_size)
        
    def forward(self, x):
        # h0, c0 초기화
        h0 = torch.zeros(self.num_layers, x.size(0), self.hidden_size).requires_grad_()
        c0 = torch.zeros(self.num_layers, x.size(0), self.hidden_size).requires_grad_()
        
        # 은닉 상태, 셀 상태 업데이트하며 순전파
        out, (hn, cn) = self.lstm(x, (h0.detach(), c0.detach()))
        
        # 시퀀스의 마지막 출력값만 사용하여 예측
        out = self.fc(out[:, -1, :]) 
        return out

# 주가/온도 예측 등의 [Batch, 10일연속데이터(Seq), 1차원] 형태의 입력
# model = LSTMModel()
# pred = model(x)
```

### Ⅲ. 기술 비교 분석  ↔  장단점 + 시퀀스 모델 비교

**장단점**:
| 장점 | 단점 |
|-----|------|
| 순차적 패턴(가변 길이) 처리 탁월 | 순차적 연산으로 병렬 처리 불가 (Transformer 대비 매우 느림) |
| 시계열 예측, 음성 인식 등 전통적 강자 | 매우 긴 시퀀스에선 여전히 정보 손실 우려 |
| 파라미터 수 상대적으로 적음(가중치 공유) | 학습 속도 느림 |

**RNN vs LSTM vs Transformer 비교**:
| 기준 | RNN | LSTM/GRU | Transformer (LLM) |
|------|-----|----------|-------------------|
| 병렬 처리 | 불가 (순차적) | 불가 (순차적) | 가능 (전체 시퀀스 동시 입력) |
| 장기 기억력| 매우 나쁨 | 좋음 (`Cell State`) | 완벽 (Attention 메커니즘 1-Hop) |
| 활용처 | 간단 시계열 | 금융 시계열, 신호 처리 | 현대 대규모 NLP, 범용 시퀀스 |

> **선택 기준**: 초거대 언어 모델, 긴 문장 번역 → Transformer; 리소스가 제한된 센서 시계열 예측, 금융 파생상품 시계열 모델링 → LSTM/GRU

### Ⅳ. 실무 적용 방안

**기술사적 판단**:
| 적용 분야 | 활용 방법 | 기대 효과 |
|---------|--------|--------|
| 금융/주요 수요 예측 | 주식 가격, 전력 수요 예측 (LSTM) | 과거 30일 시퀀스 기반 오차 축소 |
| 엣지 디바이스 음성인식 | Wake-word 인식 ("시리야", GRU) | 모델 경량화 + 실시간 반응 구현 |
| 센서 이상치 탐지 | 제조업 설비 진동 데이터 (LSTM-AutoEncoder) | 비정상 패턴 조기 감지 |
| 감성 분석 / 챗봇 | 문장 토큰 기반 감성 분류 (Bi-LSTM) | 문맥의 양방향 이해 (과거/미래) | 

**핵심 아키텍처 패턴**:
1. **One-to-Many**: 이미지 캡셔닝 (사진 1개 → 문자열 설명 출력)
2. **Many-to-One**: 감성 분류 (문장 1개 → 긍정/부정 스칼라 출력)
3. **Many-to-Many**: 기계 번역 (Seq2Seq 구조), 비디오 프레임 분석

**주의사항 / 흔한 실수**:
- LSTM/GRU라고 기울기 소실 문제가 100% 해결된 것은 아님 (수백 Step 초과 시 성능 저하).
- 시계열 예측 시 룩어헤드 편향(Look-ahead Bias): 학습 데이터에 미래 데이터를 포함해버리는 실수 유의.

### Ⅴ. 기대 효과 및 결론

| 효과 영역 | 내용 | 정량적 효과 |
|---------|-----|----------|
| 시계열 성능 | 시간 종속적 피처 추출 | 선형 회귀 대비 오차 30% 이상 개선 |
| 구조적 유연성| 가변 길이 음성/텍스트 입출력 | 패딩/잘라내기를 통한 범용 처리 |
| 활용 다각화 | AutoEncoder와 결합된 탐지 모델 | 제조 불량 탐지율 고도화 |

> **결론**: RNN과 LSTM은 순차 데이터 처리의 표준을 정립한 고전 명작 기법이다. NPL 영역에서는 Transformer(LLM)에 의해 주도권을 넘겨주었으나, 1차원 센서 데이터 기반의 이상 탐지, 금융 시계열, 경량 기기에서의 음성 인식 등 산업계 실무에서는 여전히 비용 효율성 면에서 강력한 선택지다.

## 어린이를 위한 종합 설명

**RNN은 "기억력을 가진 똑똑한 책 읽기 로봇"이야!**

```
보통 로봇 (다 까먹음):
문장: "고양이가 쥐를 쫓았다."
1. "고양이" 읽음
2. "쥐" 읽음 (고양이 잊어버림)
3. "쫓았다" 읽음 (누가 쫓았지? 모름!)

RNN 로봇 (기억함!):
1. "고양이" 읽고 메모함.
2. "쥐" 읽을 때 메모(고양이)를 함께 봄.
3. "쫓았다" 읽을 때 메모(고양이 쥐)를 봄. 
→ "아! 고양이가 쥐를 쫓은 거구나!" 이해!
```

**LSTM의 등장**:
그런데 너무 긴 문장은 앞부분을 까먹어버렸어. 그래서 중요한 단어(주인공, 장소 등)는 **형광펜으로 칠하고 절대 까먹지 않게 보관하는 특수 메모장(Cell State)**을 추가한 똑똑한 로봇이 LSTM이야! 🤖📝
