# 기계학습 기초 (Machine Learning Fundamentals)

## 핵심 인사이트 (3줄 요약)
> **기계학습(ML)**은 데이터로부터 패턴을 학습하여 규칙을 자동으로 추론하는 AI 하위 분야다. **지도·비지도·강화학습**의 3대 패러다임을 기반으로 하며, 모델 선택→데이터 전처리→학습→평가→배포의 ML 워크플로우가 핵심이다. 과적합 방지와 편향-분산 트레이드오프 이해가 기술사급 ML 설계의 핵심 역량이다.

---

### Ⅰ. 개요  ↔  개념 + 등장 배경

**개념**: 기계학습은 명시적 프로그래밍 없이 데이터로부터 경험적으로 성능이 향상되는 알고리즘과 시스템을 연구하는 분야다 (Mitchell 1997: "A program learns from experience E with respect to task T and performance measure P").

> 비유: "물고기 잡는 법을 가르치는 대신 낚시 방법을 스스로 터득하게 하는 것"

**등장 배경**:
- 규칙 기반 AI 한계: 전문가 지식 인코딩의 비용·불완전성
- 데이터 폭발: 인터넷·IoT로 대규모 데이터 가용
- 컴퓨팅 향상: GPU 병렬 처리로 대규모 학습 가능
- 통계학+컴퓨터과학 융합: Vapnik(SVM, 1995), Breiman(RF, 2001)

---

### Ⅱ. 구성 요소 및 핵심 원리

**ML 학습 패러다임**:
| 패러다임 | 입력 데이터 | 학습 방식 | 대표 알고리즘 |
|---------|---------|--------|------------|
| 지도학습 (Supervised) | 레이블 있음 | 오차 최소화 | 선형회귀, SVM, 결정트리, 딥러닝 |
| 비지도학습 (Unsupervised) | 레이블 없음 | 패턴 발견 | K-means, PCA, Autoencoder, GMM |
| 강화학습 (RL) | 보상 신호 | 보상 최대화 | Q-Learning, PPO, A3C |
| 준지도학습 (Semi-supervised) | 일부 레이블 | 혼합 | Self-training, Label Propagation |
| 자기지도학습 (Self-supervised) | 레이블 자체 생성 | 사전학습 | GPT, BERT, SimCLR |

**핵심 개념 - 편향-분산 트레이드오프**:
```
Expected Error = Bias² + Variance + Noise

Bias (편향): 모델의 가정이 실제 데이터와 얼마나 다른가
  → 높은 편향 = 과소적합 (Underfitting)
  → 단순한 모델 (선형 회귀로 비선형 데이터 학습)

Variance (분산): 다른 학습 데이터에서 예측이 얼마나 달라지는가
  → 높은 분산 = 과적합 (Overfitting)
  → 복잡한 모델 (훈련 데이터 외우기)

최적 모델:
      ↑ 에러
      |  `-.
   총  |    `-._____
  에러  |     Bias  `---___
      |              Variance
      +---------------→ 모델 복잡도
      
과소적합 ←────────→ 과적합
```

**주요 알고리즘 한눈에 보기**:
| 알고리즘 | 유형 | 특징 | 적합 상황 |
|---------|------|------|--------|
| 선형 회귀 | 지도(회귀) | 해석 간단 | 선형 관계 데이터 |
| 로지스틱 회귀 | 지도(분류) | 확률 출력 | 이진 분류 기준선 |
| SVM | 지도(분류/회귀) | 최대 마진 | 고차원, 소량 데이터 |
| 결정 트리 | 지도 | 해석 직관적 | 규칙 기반 분류 |
| 랜덤 포레스트 | 지도(앙상블) | 안정적, 강건 | 범용 |
| XGBoost/LightGBM | 지도(부스팅) | 경쟁 1위 | 정형 데이터 대회 |
| K-means | 비지도(클러스터링) | 단순·빠름 | 그룹 발견 |
| DBSCAN | 비지도(클러스터링) | 이상값 제거 | 불규칙 패턴 |
| PCA | 비지도(차원축소) | 분산 최대화 | 시각화, 전처리 |

**ML 워크플로우**:
```
1. 문제 정의 → 지도/비지도/강화? 분류/회귀/생성?
2. 데이터 수집 → Web Scraping, API, DB
3. EDA (탐색적 분석) → 분포, 상관관계, 이상값
4. 전처리 → 결측값, 스케일링, 인코딩, 증강
5. 특성 공학 → 새 특성 생성, 선택
6. 모델 선택 → 알고리즘 실험
7. 학습 & HPO → 교차검증, 하이퍼파라미터 튜닝
8. 평가 → Test Set 최종 성능
9. 배포 → API/엣지/클라우드
10. 모니터링 → 드리프트, 성능 유지
```

### Ⅲ. 기술 비교 분석  ↔  장단점 + 알고리즘 선택 전략

**정형 데이터 알고리즘 선택 가이드**:
```
데이터가 작고 (< 10K)?
  → SVM / 로지스틱 회귀 / 결정트리

데이터가 크고 (> 100K)?
  → XGBoost / LightGBM / 딥러닝

해석 가능성 중요?
  → 선형 모델 / 결정트리 / SHAP 적용

빠른 학습이 중요?
  → LightGBM

최고 성능이 중요?
  → XGBoost / 딥러닝 + 앙상블
```

**과적합 방지 기법**:
| 기법 | 원리 | 효과 |
|------|------|------|
| 데이터 증강 | 학습 데이터 다양화 | Variance↓ |
| 조기 중단 (Early Stopping) | 검증 성능 저하 시 중단 | Variance↓ |
| 정규화 (L1/L2) | 가중치 크기 제한 | Variance↓ |
| 드롭아웃 | 학습 중 무작위 노드 비활성화 | Variance↓ |
| 교차 검증 | K-fold로 견고한 평가 | 신뢰성↑ |
| 앙상블 | 다수 모델 결합 | Bias+Variance 동시↓ |

### Ⅳ. 실무 적용 방안

**기술사적 판단**:
| 비즈니스 문제 | ML 방식 | 추천 알고리즘 |
|------------|--------|------------|
| 고객 이탈 예측 | 지도(분류) | XGBoost + SHAP |
| 수요 예측 | 지도(회귀) | LightGBM / LSTM |
| 고객 세분화 | 비지도(클러스터링) | K-means / DBSCAN |
| 이상치 탐지 | 비지도 | Isolation Forest / Autoencoder |
| 추천 시스템 | 협력 필터링+딥러닝 | DNN + Matrix Factorization |

**ML 평가 지표**:
```
분류:
  Accuracy = (TP+TN) / Total     → 클래스 불균형 시 부적절
  Precision = TP / (TP+FP)       → 양성 예측의 정확도
  Recall = TP / (TP+FN)          → 실제 양성 중 탐지율
  F1 = 2·(P·R)/(P+R)             → P와 R의 조화평균
  AUC-ROC                        → 임계값 무관 종합 성능

회귀:
  MAE = mean(|y - ŷ|)
  RMSE = sqrt(mean((y - ŷ)²))
  R² = 1 - SSR/SST               → 설명력 (1이 완벽)
```

**관련 개념**: 지도학습, 비지도학습, 강화학습, XGBoost, 과적합, 교차검증, 앙상블, 특성 공학

### Ⅴ. 기대 효과 및 결론

| 효과 영역 | 내용 | 정량적 효과 |
|---------|-----|----------|
| 예측 정확도 | 규칙 기반 → ML | 오류율 20~50% 감소 |
| 자동화 | 수동 결정 → ML 자동 | 의사결정 속도 10~100배 |
| 패턴 발견 | 데이터 내 숨은 패턴 | 인간이 찾기 어려운 인사이트 발견 |

> **결론**: ML은 데이터 기반 의사결정의 핵심 엔진. 편향-분산 트레이드오프를 이해하고 문제-데이터-알고리즘의 올바른 매핑이 기술사급 ML 설계 역량이다. 정형 데이터는 XGBoost, 비정형/대규모는 딥러닝이 2024~2025 표준이다.

## 어린이를 위한 종합 설명

**기계학습은 "경험으로 배우는 AI"야!**

```
규칙 기반 AI (옛날 방식):
프로그래머가 모든 규칙 작성
"고양이 = 귀가 뾰족하고, 수염이 있고, 야옹 소리를 내고..."
→ 엄청 많은 규칙 → 새로운 동물 나오면 규칙 또 추가!

기계학습:
고양이 사진 100만 장 보여주기 → "이게 고양이야!"
→ AI가 스스로 특징 학습!
→ 새 고양이 사진도 인식!
```

세 가지 학습 방법:
```
지도학습: 선생님이 정답 알려줌 → "이건 고양이, 이건 강아지"
비지도학습: 혼자 그룹 짓기 → "비슷한 것끼리 묶어봐"
강화학습: 게임으로 배우기 → "잘하면 점수, 못하면 감소"
```

> 기계학습 = AI가 경험을 통해 스스로 뇌를 키우는 방법! 📊🧠💡
