### Ⅰ. 개요

#### 1. 개념
MDM(Master Data Management)은 **기업의 핵심 비즈니스 데이터(고객, 제품, 공급자, 직원 등)를 중앙에서 통합 관리하여, 모든 시스템에서 일관되고 정확한 데이터를 사용할 수 있게 하는 관리 체계**이다.

> 비유: "등기소" - 집 주인이 누구인지 한 곳에서 확인해요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 마스터 데이터란?
```
┌────────────────────────────────────────────────────────┐
│           마스터 데이터 vs 트랜잭션 데이터            │
├────────────────────────────────────────────────────────┤
│                                                        │
│  📊 마스터 데이터 (Master Data):                      │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 변화가 적은 핵심 데이터                      │   │
│  │  • 비즈니스 엔티티                              │   │
│  │  • 여러 시스템에서 공유                         │   │
│  │                                                │   │
│  │  예:                                           │   │
│  │  👤 고객 (Customer)                            │   │
│  │  📦 제품 (Product)                             │   │
│  │  🏭 공급자 (Supplier)                          │   │
│  │  📍 위치 (Location)                            │   │
│  │  👨‍💼 직원 (Employee)                           │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  📝 트랜잭션 데이터 (Transaction Data):               │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 시간에 따라 계속 생성                        │   │
│  │  • 이벤트 기록                                  │   │
│  │                                                │   │
│  │  예:                                           │   │
│  │  🛒 주문 (Order)                               │   │
│  │  💳 결제 (Payment)                             │   │
│  │  📋 송장 (Invoice)                             │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 4. MDM 데이터 도메인
| 도메인 | 설명 | 예시 속성 |
|--------|------|----------|
| 고객 (Customer) | 고객 정보 | 이름, 연락처, 주소 |
| 제품 (Product) | 상품 정보 | SKU, 이름, 가격 |
| 공급자 (Supplier) | 공급업체 | 공급자명, 계약정보 |
| 직원 (Employee) | 인력 정보 | 사번, 부서, 직급 |
| 위치 (Location) | 장소 정보 | 주소, 지역 |

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **경영 효율** | 프로세스 자동화·통합으로 업무 생산성 향상 | 행정 업무 처리 시간 40% 단축 |
| **의사결정** | 실시간 BI·분석으로 데이터 기반 의사결정 지원 | 의사결정 속도 50% 향상 |
| **IT 거버넌스** | 표준화된 거버넌스 체계로 IT 리스크 관리 강화 | IT 감사 지적 사항 60% 감소 |

#### 결론
> **MDM (Master Data Management)**은(는) 기업 정보 시스템은 ERP·CRM에서 시작하여 DX(디지털 전환)·초자동화(Hyper-automation)·AI 통합으로 진화하며, 기업의 모든 운영 영역을 데이터로 연결하는 디지털 기업(Digital Enterprise)의 근간이 될 것이다.

> **※ 참고 표준**: ITIL v4(AXELOS), COBIT 2019(ISACA), ISO/IEC 20000-1:2018, ISO 9001
