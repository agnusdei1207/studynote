# 데이터 거버넌스 (Data Governance)

## 핵심 인사이트 (3줄 요약)
> **데이터 자산의 체계적 관리 체계**. 품질, 보안, 준법성 확보. 전사 데이터 관리 표준화.

---

## 📝 기술사 모의답안 (2.5페이지 분량)

### 📌 예상 문제
> **"데이터 거버넌스 (Data Governance)의 개념과 주요 기능을 설명하고, 기업 정보화 전략 관점에서의 도입 방안과 성공 요인을 논하시오."**

---

### Ⅰ. 개요

#### 1. 개념
데이터 거버넌스(Data Governance)는 **기업의 데이터 자산에 대한 책임, 정책, 프로세스, 표준을 정의하고, 데이터 품질, 보안, 프라이버시, 규정 준수를 체계적으로 관리하는 조직 차원의 관리 체계**이다.

> 비유: "데이터 헌법" - 데이터를 어떻게 다룰지 규칙을 정해요

---

#### 3. 데이터 거버넌스 조직
| 역할 | 책임 |
|-----|------|
| 데이터 오너 | 데이터 자산에 대한 최종 책임 |
| 데이터 스튜어드 | 일상적 데이터 관리 |
| 데이터 커스토디언 | 기술적 데이터 관리 |
| 거버넌스 위원회 | 정책 승인 및 의사결정 |

#### 4. 데이터 품질 차원
| 차원 | 설명 | 측정 방법 |
|-----|------|----------|
| 정확성 | 실제 값과 일치 | 오류율 |
| 완전성 | 필수 데이터 존재 | 결측치율 |
| 일관성 | 데이터 간 모순 없음 | 불일치율 |
| 적시성 | 최신 데이터 | 갱신 주기 |
| 유일성 | 중복 없음 | 중복률 |

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **경영 효율** | 프로세스 자동화·통합으로 업무 생산성 향상 | 행정 업무 처리 시간 40% 단축 |
| **의사결정** | 실시간 BI·분석으로 데이터 기반 의사결정 지원 | 의사결정 속도 50% 향상 |
| **IT 거버넌스** | 표준화된 거버넌스 체계로 IT 리스크 관리 강화 | IT 감사 지적 사항 60% 감소 |

#### 결론
> **데이터 거버넌스 (Data Governance)**은(는) 기업 정보 시스템은 ERP·CRM에서 시작하여 DX(디지털 전환)·초자동화(Hyper-automation)·AI 통합으로 진화하며, 기업의 모든 운영 영역을 데이터로 연결하는 디지털 기업(Digital Enterprise)의 근간이 될 것이다.

> **※ 참고 표준**: ITIL v4(AXELOS), COBIT 2019(ISACA), ISO/IEC 20000-1:2018, ISO 9001
