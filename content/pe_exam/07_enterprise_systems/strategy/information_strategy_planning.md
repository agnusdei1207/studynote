# 정보전략계획 (ISP)

## 핵심 인사이트 (3줄 요약)
> **기업 목표 달성을 위한 IT 전략 수립**. 현황 분석, 목표 설정, 실행 계획. 경영 전략과 IT 정렬.

---

## 📝 기술사 모의답안 (2.5페이지 분량)

### 📌 예상 문제
> **"정보전략계획 (ISP)의 개념과 주요 기능을 설명하고, 기업 정보화 전략 관점에서의 도입 방안과 성공 요인을 논하시오."**

---

### Ⅰ. 개요

#### 1. 개념
ISP(Information Strategy Planning)는 **기업의 경영 목표를 달성하기 위해 정보시스템을 어떻게 구축하고 활용할 것인가를 체계적으로 계획하는 활동**이다. 경영 전략과 IT 전략의 정렬(Alignment)을 핵심으로 한다.

> 비유: "건축 설계도" - 집을 지으려면 먼저 설계도가 필요해요

---

#### 3. ISP 분석 기법
| 기법 | 설명 | 용도 |
|-----|------|------|
| BPR | 업무 프로세스 재설계 | 프로세스 혁신 |
| CSF | 핵심 성공 요인 | 우선순위 결정 |
| Value Chain | 가치 사슬 분석 | 업무 분석 |
| SWOT | 강점/약점/기회/위협 | 전략 수립 |
| BSC | 균형 성과표 | 성과 관리 |

#### 4. 정보 아키텍처 구성
| 아키텍처 | 설명 |
|---------|------|
| 업무 아키텍처 | 업무 프로세스 구조 |
| 데이터 아키텍처 | 데이터 구조 및 흐름 |
| 애플리케이션 아키텍처 | 시스템 구조 |
| 기술 아키텍처 | 하드웨어/네트워크 |

#### 6. 우선순위 결정 기준
| 기준 | 설명 |
|-----|------|
| 전략적 기여도 | 경영 목표 달성 기여 |
| 투자 효과성 | 비용 대비 효과 |
| 기술적 용이성 | 구현 난이도 |
| 긴급성 | 즉시 필요성 |

### Ⅲ. 기술 비교 분석

#### 5. AS-IS vs TO-BE
| 구분 | AS-IS | TO-BE |
|-----|-------|-------|
| 의미 | 현재 상태 | 목표 상태 |
| 분석 | 현황 파악 | 개선 방안 |
| 산출물 | 현황 문서 | 목표 모델 |

#### 7. 장단점
| 장점 | 단점 |
|-----|------|
| 체계적 IT 투자 | 시간/비용 소요 |
| 경영-IT 정렬 | 환경 변화 대응 어려움 |
| 중복 투자 방지 | 실행력 부족 가능 |

---

### Ⅳ. 실무 적용 방안

#### 9. 실무에선? (기술사적 판단)
**수립 시기:**
- 경영 환경 변화 시
- 신규 시스템 도입 전
- 정기적(3~5년 주기)

**성공 요인:**
- 최고경영층 지원
- 명확한 비전 설정
- 실행 가능한 계획

**주요 산출물:**
- 현황 분석서
- 목표 아키텍처
- 시스템 구축 계획서
- 투자 타당성 분석서

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **경영 효율** | 프로세스 자동화·통합으로 업무 생산성 향상 | 행정 업무 처리 시간 40% 단축 |
| **의사결정** | 실시간 BI·분석으로 데이터 기반 의사결정 지원 | 의사결정 속도 50% 향상 |
| **IT 거버넌스** | 표준화된 거버넌스 체계로 IT 리스크 관리 강화 | IT 감사 지적 사항 60% 감소 |

#### 결론
> **정보전략계획 (ISP)**은(는) 기업 정보 시스템은 ERP·CRM에서 시작하여 DX(디지털 전환)·초자동화(Hyper-automation)·AI 통합으로 진화하며, 기업의 모든 운영 영역을 데이터로 연결하는 디지털 기업(Digital Enterprise)의 근간이 될 것이다.

> **※ 참고 표준**: ITIL v4(AXELOS), COBIT 2019(ISACA), ISO/IEC 20000-1:2018, ISO 9001
