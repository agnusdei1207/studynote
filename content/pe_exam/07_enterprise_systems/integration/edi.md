# EDI (Electronic Data Interchange)

## 핵심 인사이트 (3줄 요약)
> **기업 간 전자 문서 교환 표준**. 종이 없는 무역 구현. 자동화된 B2B 거래.

---

## 📝 기술사 모의답안 (2.5페이지 분량)

### 📌 예상 문제
> **"EDI (Electronic Data Interchange)의 개념과 주요 기능을 설명하고, 기업 정보화 전략 관점에서의 도입 방안과 성공 요인을 논하시오."**

---

### Ⅰ. 개요

#### 1. 개념
EDI(Electronic Data Interchange)는 **기업 간 비즈니스 거래에 필요한 주문서, 송장, 납품서 등의 문서를 정해진 표준 형식에 따라 전자적으로 교환하는 시스템**이다.

> 비유: "디지털 우편" - 종이 없이 컴퓨터끼리 편지를 주고받아요

---

│                         │                             │

│                         │                             │

│  │                                                │   │

#### 3. EDI 표준
```
┌────────────────────────────────────────────────────────┐
│           EDI 표준 종류                               │
├────────────────────────────────────────────────────────┤
│                                                        │
│  🌍 UN/EDIFACT (국제 표준):                           │
│  ┌────────────────────────────────────────────────┐   │
│  │  • UN 산하 표준                                │   │
│  │  • 전 세계적으로 가장 널리 사용                 │   │
│  │  • 한국: KEDIFACT (EDIFACT 기반)               │   │
│  │                                                │   │
│  │  예: ORDERS (주문서), INVOIC (송장)            │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🇺🇸 ANSI X12 (미국 표준):                            │
│  ┌────────────────────────────────────────────────┐   │
│  │  • 미국 ANSI 표준                              │   │
│  │  • 북미 지역 주로 사용                         │   │
│  │                                                │   │
│  │  예: 850 (주문서), 810 (송장)                  │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  📄 XML/EDI (차세대):                                 │
│  ┌────────────────────────────────────────────────┐   │
│  │  • XML 기반                                    │   │
│  │  • 웹 친화적                                   │   │
│  │  • 읽기 쉬운 형식                              │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  🇰🇷 KEDIFACT (한국 표준):                            │
│  ┌────────────────────────────────────────────────┐   │
│  │  • EDIFACT 기반 한국 표준                       │   │
│  │  • 관세청, 무역협회 등 사용                     │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 4. 주요 EDI 문서 유형
| 문서 유형 | EDIFACT | ANSI X12 | 용도 |
|----------|---------|----------|------|
| 주문서 | ORDERS | 850 | 구매 주문 |
| 송장 | INVOIC | 810 | 청구서 |
| 납품통지 | DESADV | 856 | 출하 통지 |
| 수취확인 | RECADV | 861 | 수령 확인 |
| 재고보고 | INVRPT | 846 | 재고 현황 |
| 견적서 | QUOTES | 843 | 가격 제안 |

#### 결론
> **EDI (Electronic Data Interchange)**은(는) 기업 정보 시스템은 ERP·CRM에서 시작하여 DX(디지털 전환)·초자동화(Hyper-automation)·AI 통합으로 진화하며, 기업의 모든 운영 영역을 데이터로 연결하는 디지털 기업(Digital Enterprise)의 근간이 될 것이다.

> **※ 참고 표준**: ITIL v4(AXELOS), COBIT 2019(ISACA), ISO/IEC 20000-1:2018, ISO 9001
