# CMMI (능력 성숙도 모델 통합)

## 핵심 인사이트 (3줄 요약)
> **조직의 프로세스 성숙도를 평가하고 개선**하는 모델. 1~5단계로 성숙도 측정. 소프트웨어 개발, 서비스, 조달 분야의 품질 향상 도구.

### 📌 예상 문제

---

### Ⅰ. 개요

#### 1. 개념
CMMI(Capability Maturity Model Integration, 능력 성숙도 모델 통합)는 **조직의 프로세스 개발과 관리 능력을 평가하고 개선**하기 위한 프레임워크다. 카네기멜런 대학교 SEI가 개발했다.

> 비유: "조직의 성장 단계" - 아이에서 어른으로 성장하듯 프로세스가 성숙해간다

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 2. 성숙도 단계
```
┌────────────────────────────────────────────────────────┐
│               CMMI 성숙도 5단계                         │
├────────────────────────────────────────────────────────┤
│                                                        │
│  Level 5: 최적화 (Optimizing)                         │
│  ┌────────────────────────────────────────────────┐   │
│  │ • 지속적 프로세스 개선                          │   │
│  │ • 혁신과 기술 도입                              │   │
│  │ • 예측적 문제 해결                              │   │
│  │ • 조직 전체 학습                                │   │
│  └────────────────────────────────────────────────┘   │
│                      ↑                                │
│  Level 4: 정량적 관리 (Quantitatively Managed)        │
│  ┌────────────────────────────────────────────────┐   │
│  │ • 통계적 프로세스 관리                          │   │
│  │ • 정량적 품질 목표                              │   │
│  │ • 성능 예측                                     │   │
│  └────────────────────────────────────────────────┘   │
│                      ↑                                │
│  Level 3: 정의됨 (Defined)                            │
│  ┌────────────────────────────────────────────────┐   │
│  │ • 표준화된 프로세스                             │   │
│  │ • 조직 차원의 프로세스 자산                     │   │
│  │ • 엔지니어링 활동 통합                          │   │
│  └────────────────────────────────────────────────┘   │
│                      ↑                                │
│  Level 2: 관리됨 (Managed)                            │
│  ┌────────────────────────────────────────────────┐   │
│  │ • 프로젝트 계획 및 추적                         │   │
│  │ • 요구사항 관리                                 │   │
│  │ • 형상 관리                                     │   │
│  │ • 측정 및 분석                                  │   │
│  └────────────────────────────────────────────────┘   │
│                      ↑                                │
│  Level 1: 초기 (Initial)                              │
│  ┌────────────────────────────────────────────────┐   │
│  │ • 애드혹(Ad-hoc) 프로세스                       │   │
│  │ • 개인 의존적                                   │   │
│  │ • 예측 불가능                                   │   │
│  │ • 성공은 개인 능력에 달림                       │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

```
│                                                        │
│  CMMI for Services (CMMI-SVC):                        │
│  ┌────────────────────────────────────────────────┐   │
│  │ • 서비스 제공                                   │   │
│  │ • 서비스 관리                                   │   │
│  │ • 고객 지원                                     │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  CMMI for Acquisition (CMMI-ACQ):                     │
│  ┌────────────────────────────────────────────────┐   │
│  │ • 조달 관리                                     │   │

│  └────────────────────────────────────────────────┘   │
│                                                        │
│  프로세스 영역 (Process Areas):                       │
│  ┌────────────────────────────────────────────────┐   │
│  │ 요구사항 관리 │ 프로젝트 계획 │ 기술 솔루션    │   │

│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **경영 효율** | 프로세스 자동화·통합으로 업무 생산성 향상 | 행정 업무 처리 시간 40% 단축 |
| **의사결정** | 실시간 BI·분석으로 데이터 기반 의사결정 지원 | 의사결정 속도 50% 향상 |
| **IT 거버넌스** | 표준화된 거버넌스 체계로 IT 리스크 관리 강화 | IT 감사 지적 사항 60% 감소 |

#### 결론
> **CMMI (능력 성숙도 모델 통합)**은(는) 기업 정보 시스템은 ERP·CRM에서 시작하여 DX(디지털 전환)·초자동화(Hyper-automation)·AI 통합으로 진화하며, 기업의 모든 운영 영역을 데이터로 연결하는 디지털 기업(Digital Enterprise)의 근간이 될 것이다.

> **※ 참고 표준**: ITIL v4(AXELOS), COBIT 2019(ISACA), ISO/IEC 20000-1:2018, ISO 9001
