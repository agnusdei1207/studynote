# RPA (로봇 프로세스 자동화)

## 핵심 인사이트 (3줄 요약)
> **소프트웨어 로봇으로 업무 자동화**. 반복 작업 무인 처리. 생산성 및 정확성 향상.

---

## 📝 기술사 모의답안 (2.5페이지 분량)

### 📌 예상 문제
> **"RPA (로봇 프로세스 자동화)의 개념과 주요 기능을 설명하고, 기업 정보화 전략 관점에서의 도입 방안과 성공 요인을 논하시오."**

---

### Ⅰ. 개요

#### 1. 개념
RPA(Robotic Process Automation)는 **사람이 컴퓨터로 수행하던 규칙적이고 반복적인 업무를 소프트웨어 로봇이 자동으로 수행하도록 하는 기술**이다. 기존 시스템 변경 없이 UI 레벨에서 자동화한다.

> 비유: "디지털 직원" - 반복 일을 대신해줘요

### Ⅲ. 기술 비교 분석

#### 3. RPA vs 전통적 자동화
| 구분 | RPA | 전통적 자동화 |
|-----|-----|-------------|
| 접근 방식 | UI 레벨 | API/DB 직접 |
| 시스템 변경 | 불필요 | 필요 |
| 구현 속도 | 빠름 | 느림 |
| 비용 | 낮음 | 높음 |
| 유연성 | 높음 | 낮음 |
| 적용 범위 | 규칙 기반 | 전체 |

#### 5. 장단점
| 장점 | 단점 |
|-----|------|
| 구현 용이 | UI 변경에 취약 |
| 비용 절감 | 예외 처리 한계 |
| 정확성 | 관리 부담 |
| 24/7 가동 | 보안 이슈 |

---

### Ⅳ. 실무 적용 방안

#### 4. RPA 적용 영역
| 영역 | 예시 | 효과 |
|-----|------|------|
| 재무/회계 | 결제 처리, 세금 계산 | 정확성 향상 |
| 인사 | 급여 계산, 입사 처리 | 시간 단축 |
| 고객서비스 | 문의 응대, 데이터 입력 | 응답 속도 |
| IT | 계정 관리, 백업 | 운영 효율 |

---

### Ⅴ. 기대 효과 및 결론


| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **경영 효율** | 프로세스 자동화·통합으로 업무 생산성 향상 | 행정 업무 처리 시간 40% 단축 |
| **의사결정** | 실시간 BI·분석으로 데이터 기반 의사결정 지원 | 의사결정 속도 50% 향상 |
| **IT 거버넌스** | 표준화된 거버넌스 체계로 IT 리스크 관리 강화 | IT 감사 지적 사항 60% 감소 |

#### 결론
> **RPA (로봇 프로세스 자동화)**은(는) 기업 정보 시스템은 ERP·CRM에서 시작하여 DX(디지털 전환)·초자동화(Hyper-automation)·AI 통합으로 진화하며, 기업의 모든 운영 영역을 데이터로 연결하는 디지털 기업(Digital Enterprise)의 근간이 될 것이다.

> **※ 참고 표준**: ITIL v4(AXELOS), COBIT 2019(ISACA), ISO/IEC 20000-1:2018, ISO 9001
