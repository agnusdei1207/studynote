# 부동 소수점 수 (Floating Point Number)

## 핵심 인사이트 (3줄 요약)
> **실수를 근사적으로 표현**하는 방식. 가수부와 지수부로 나누어 매우 크거나 작은 수를 표현한다. IEEE 754 표준이 있으며, 정밀도 문제가 존재한다.

## 1. 개념
부동 소수점 수(Floating Point Number)는 **소수점이 움직일 수 있는(浮動)** 형태로 실수를 표현한다. 과학적 표기법을 2진수로 구현한 것이다.

> 비유: "과학적 표기법" - 1230000 = 1.23 × 10^6

## 2. 표현 방식

```
부동 소수점 표현:

값 = (-1)^부호 × 가수 × 밑수^지수

IEEE 754 단정도 (32-bit):
┌────┬─────────┬───────────────────┐
│부호│  지수   │       가수        │
│1bit│ 8 bits  │     23 bits       │
└────┴─────────┴───────────────────┘

IEEE 754 배정도 (64-bit):
┌────┬──────────┬───────────────────────┐
│부호│   지수   │        가수           │
│1bit│ 11 bits  │      52 bits          │
└────┴──────────┴───────────────────────┘
```

## 3. IEEE 754 형식

```
단정도 (32-bit):
- 부호: 1 bit (0=양수, 1=음수)
- 지수: 8 bits (bias = 127)
- 가수: 23 bits (1.xxxxx의 xxxxx)

예: -6.625 표현
1. 2진수 변환: -110.101
2. 정규화: -1.10101 × 2^2
3. 부호: 1
4. 지수: 2 + 127 = 129 = 10000001
5. 가수: 10101000...

결과: 1 10000001 10101000000000000000000
```

## 4. 특수 값

| 값 | 지수 | 가수 | 의미 |
|----|------|------|------|
| 0 | 0 | 0 | 0 |
| 비정규수 | 0 | ≠0 | 아주 작은 수 |
| 정규수 | 1~254 | 임의 | 일반 수 |
| 무한대 | 255 | 0 | ±∞ |
| NaN | 255 | ≠0 | Not a Number |

## 5. 정밀도

```
단정도 (float):
- 유효 자릿수: 약 7자리
- 범위: ±3.4 × 10^38

배정도 (double):
- 유효 자릿수: 약 15자리
- 범위: ±1.8 × 10^308
```

## 6. 정밀도 문제

```python
# 부동 소수점 오차 예시
>>> 0.1 + 0.2
0.30000000000000004  # 정확히 0.3이 아님!

>>> 0.1 + 0.2 == 0.3
False

이유: 0.1, 0.2를 2진수로 정확히 표현 불가
0.1(10진) = 0.0001100110011...(2진, 무한소수)
```

## 7. 반올림 모드

| 모드 | 설명 |
|------|------|
| Round to Nearest | 가장 가까운 값 (기본) |
| Round toward 0 | 0 쪽으로 |
| Round toward +∞ | 양의 무한대 쪽으로 |
| Round toward -∞ | 음의 무한대 쪽으로 |

## 8. 장단점

| 장점 | 단점 |
|-----|------|
| 넓은 범위 표현 | 정밀도 손실 |
| 과학 계산에 적합 | 근사값 |
| 하드웨어 지원 | 비교 어려움 |

## 9. 코드 예시

```c
#include <stdio.h>
#include <float.h>

int main() {
    float f = 0.1f;
    double d = 0.1;

    printf("float: %.20f\n", f);
    printf("double: %.20lf\n", d);

    printf("FLT_DIG: %d\n", FLT_DIG);   // 6
    printf("DBL_DIG: %d\n", DBL_DIG);   // 15

    return 0;
}
```

## 10. 실무에선? (기술사적 판단)
- **금융**: 부동소수점 사용 금지 (BigDecimal)
- **게임**: float 사용 (속도)
- **과학**: double 또는更高 정밀도
- **AI**: FP16, BF16 (반정밀도)

## 어린이를 위한 종합 설명

**부동 소수점은 "움직이는 소수점"이야!**

### 큰 수 표현하기
```
123000000000 = 1.23 × 10^11
              ↑ 소수점이 움직였어요!
```

### 컴퓨터에서는
```
32비트로 저장:

[부호][지수][가수]
  ↓     ↓     ↓
 +/−  10^11  1.23

지수로 소수점 위치를 정해요!
```

### 문제점
```
0.1을 정확히 저장할 수 없어요! 😱

0.1 = 0.0001100110011... (무한 반복)

→ 컴퓨터는 근사값만 저장
0.1 + 0.2 ≠ 0.3 (아주 조금 다름)
```

### 해결책
```
돈 계산할 때: 정수로 계산 (12345원)
과학 계산: 오차 허용
비교: 아주 작은 차이는 무시
```

**비밀**: 컴퓨터는 완벽하지 않아요! 아주 큰 수와 아주 작은 수를 다 표현하려다 보니 조금 부정확해질 수 있어요! 🔢
