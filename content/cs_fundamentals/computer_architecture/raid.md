# RAID (Redundant Array of Independent Disks)

## 핵심 인사이트 (3줄 요약)
> 여러 디스크를 **하나의 논리적 디스크처럼** 구성하여 성능과 신뢰성을 향상시키는 기술. RAID 0(성능), RAID 1(미러링), RAID 5(패리티)가 대표적이다. 기업용 스토리지의 필수 기술이다.

## 1. 개념
RAID(Redundant Array of Independent Disks)는 **여러 개의 물리적 디스크를 하나의 논리적 디스크로 묶어** 성능 향상 및 데이터 보호를 제공하는 기술이다.

> 비유: "여러 명이 짐을 나눠 듦" - 혼자보다 빠르고, 한 명이 아파도 짐을 잃지 않음

## 2. RAID 레벨 비교

| 레벨 | 최소 디스크 | 특징 | 용도 |
|------|------------|------|------|
| RAID 0 | 2 | 스트라이핑, 빠름 | 임시 데이터 |
| RAID 1 | 2 | 미러링, 안전 | OS, 중요 데이터 |
| RAID 5 | 3 | 패리티, 균형 | 일반 서버 |
| RAID 6 | 4 | 이중 패리티 | 중요 서버 |
| RAID 10 | 4 | 0+1 결합 | 데이터베이스 |

## 3. RAID 0 (Striping)

```
데이터: [A][B][C][D][E][F]

Disk 1: [A][C][E]
Disk 2: [B][D][F]

장점: 읽기/쓰기 속도 2배
단점: 디스크 1개 고장 = 모든 데이터 손실
```

## 4. RAID 1 (Mirroring)

```
데이터: [A][B][C][D]

Disk 1: [A][B][C][D]
Disk 2: [A][B][C][D]  ← 복사본

장점: 1개 디스크 고장해도 안전
단점: 디스크 효율 50% (비용 2배)
```

## 5. RAID 5 (Distributed Parity)

```
데이터: [A][B][C][D]

Disk 1: [A][B][P1]
Disk 2: [C][P2][D]
Disk 3: [P3][E][F]

P = 패리티 (A⊕B⊕C = P)

장점: 1개 디스크 고장 복구 가능, 공간 효율 좋음
단점: 쓰기 시 패리티 계산 오버헤드
```

## 6. RAID 6 (Double Parity)

```
Disk 1: [A][B][P1][Q1]
Disk 2: [C][P2][D][Q2]
Disk 3: [P3][E][F][Q3]
Disk 4: [Q4][G][H][P4]

장점: 2개 디스크 동시 고장도 복구
단점: 쓰기 성능 저하, 최소 4개 필요
```

## 7. RAID 10 (1+0)

```
RAID 1 미러 2개를 RAID 0으로 스트라이핑

미러1: [A][B]    미러2: [C][D]
  ↓       ↓        ↓       ↓
[D1]    [D2]     [D3]    [D4]

장점: 읽기 빠름 + 1개 디스크 고장 안전
단점: 공간 효율 50%
```

## 8. 효율성 비교

| RAID | 총 용량 | 가용 용량 | 효율 |
|------|---------|----------|------|
| 0 | 4TB (2×2TB) | 4TB | 100% |
| 1 | 4TB (2×2TB) | 2TB | 50% |
| 5 | 6TB (3×2TB) | 4TB | 67% |
| 6 | 8TB (4×2TB) | 4TB | 50% |
| 10 | 8TB (4×2TB) | 4TB | 50% |

## 9. 장단점

| 장점 | 단점 |
|-----|------|
| 성능 향상 | 복잡성 증가 |
| 데이터 보호 | 비용 증가 |
| 용량 확장 | 복구 시간 |

## 10. 실무에선? (기술사적 판단)
- **OS/부팅**: RAID 1
- **데이터베이스**: RAID 10 (성능+안정성)
- **파일 서버**: RAID 5/6
- **백업**: RAID 6 또는 RAID 10
- **SSD RAID**: 성능보다 내구성 고려

## 11. 관련 개념
- 스트라이핑 (Striping)
- 미러링 (Mirroring)
- 패리티 (Parity)

## 어린이를 위한 종합 설명

**RAID는 "여러 친구가 숙제를 나눠서 해요!"**

### RAID 0 (분담하기)
```
숙제: 1~10페이지

친구1: 1,3,5,7,9 페이지 📄
친구2: 2,4,6,8,10 페이지 📄

→ 2배 빨라요! 하지만 한 명이 아프면... 😱
```

### RAID 1 (복사하기)
```
친구1: 1~10 페이지 📄
친구2: 1~10 페이지 📄 (복사본)

→ 한 명이 아파도 괜찮아요! 하지만 비효율적...
```

### RAID 5 (나누기 + 확인)
```
친구1: 1, 4, 7 페이지 + 확인✓
친구2: 2, 5, 8 페이지 + 확인✓
친구3: 3, 6, 9 페이지 + 확인✓

→ 한 명이 아파도 확인✓으로 복구해요! ✨
```

**기업에서는**: 중요한 자료를 잃으면 안 되니까 RAID를 꼭 써요! 💼
