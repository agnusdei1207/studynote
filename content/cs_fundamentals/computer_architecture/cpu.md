# 중앙처리장치 (CPU, Central Processing Unit)

## 핵심 인사이트 (3줄 요약)
> 컴퓨터의 **두뇌** 역할을 하는 핵심 프로세서. 명령어를 해석(Instruction Decode)하고 실행(Execute)하여 프로그램을 수행한다. ALU, 제어장치, 레지스터로 구성되며, 클럭에 동기화되어 동작한다.

## 1. 개념
CPU(Central Processing Unit)는 **컴퓨터 시스템의 중앙 처리 장치**로, 프로그램의 명령어를 해석하고 실행하는 역할을 한다. 모든 연산과 제어를 담당한다.

> 비유: "요리사" - 레시피(프로그램)를 보고 요리(연산)를 수행

## 2. CPU 내부 구조

```
┌─────────────────────────────────────┐
│                CPU                  │
│  ┌─────────┐  ┌─────────┐          │
│  │   ALU   │  │  제어   │          │
│  │(산술논리)│  │  장치   │          │
│  └────┬────┘  └────┬────┘          │
│       │            │                │
│  ┌────┴────────────┴────┐          │
│  │      레지스터        │          │
│  └──────────────────────┘          │
└─────────────────────────────────────┘
```

## 3. 구성 요소

| 구성 요소 | 역할 | 설명 |
|----------|------|------|
| ALU | 연산 | 산술(+,-,*,/) 및 논리(AND,OR) 연산 |
| 제어 장치 | 제어 | 명령어 해석, 신호 생성 |
| 레지스터 | 저장 | 빠른 임시 저장 공간 |
| 버스 인터페이스 | 통신 | 메모리/I/O와 데이터 전송 |

## 4. 명령어 사이클

```
명령어 실행 과정:

1. Fetch (인출): 메모리에서 명령어 가져오기
2. Decode (해석): 명령어가 무엇인지 분석
3. Execute (실행): 명령어 수행
4. Memory (접근): 필요시 메모리 R/W
5. Writeback (쓰기): 결과를 레지스터에 저장

   ┌─────────────────────────────────┐
   │  Fetch → Decode → Execute → ... │
   └─────────────────────────────────┘
              ↑ 반복
```

## 5. 주요 레지스터

| 레지스터 | 역할 |
|----------|------|
| PC (Program Counter) | 다음 명령어 주소 |
| IR (Instruction Register) | 현재 명령어 |
| MAR (Memory Address Register) | 메모리 주소 |
| MDR (Memory Data Register) | 메모리 데이터 |
| ACC (Accumulator) | 연산 결과 |
| SP (Stack Pointer) | 스택 최상단 주소 |

## 6. 성능 지표

```
성능 = 클럭 속도 × IPC × 코어 수

- 클럭 속도: GHz (초당 사이클)
- IPC: Instructions Per Cycle (사이클당 명령어)
- 코어 수: 병렬 처리 능력

예: 3GHz, IPC=2, 8코어
  = 3×10^9 × 2 × 8 = 48 GFLOPS
```

## 7. 파이프라이닝

```
5단계 파이프라인:

시간 →
IF:  [1] [2] [3] [4] [5]
ID:      [1] [2] [3] [4] [5]
EX:          [1] [2] [3] [4]
MEM:             [1] [2] [3]
WB:                  [1] [2]

→ 한 클럭에 한 명령어 완료 (이상적)
```

## 8. 캐시 계층

```
CPU ─ L1 (32KB) ─ L2 (256KB) ─ L3 (8MB) ─ RAM
     1 cycle      4 cycles      12 cycles    100+ cycles
```

## 9. 장단점

| 장점 | 단점 |
|-----|------|
| 범용성 높음 | 전력 소모 큼 |
| 높은 성능 | 발열 문제 |
| 유연한 프로그래밍 | GPU보다 병렬 처리 느림 |

## 10. 실무에선? (기술사적 판단)
- **데스크톱**: 4~8코어, 높은 클럭
- **서버**: 16~64코어, 높은 처리량
- **모바일**: BIG.little, 저전력
- **AI**: CPU보다 GPU/NPU 선호

## 11. 관련 개념
- GPU (Graphics Processing Unit)
- 마이크로프로세서
- 명령어 집합 (ISA)

## 어린이를 위한 종합 설명

**CPU는 "컴퓨터의 두뇌"야!**

요리사가 레시피를 보고 요리하듯, CPU는 프로그램을 보고 일을 해요!

```
CPU의 하루:

1. 📖 레시피 읽기 (Fetch)
2. 🤔 이게 뭐지? (Decode)
3. 🔥 요리하기 (Execute)
4. 🍽️ 접시에 담기 (Writeback)
5. → 다음 요리로!
```

**CPU의 주머니들 (레지스터)**:
- PC: "다음엔 5페이지 봐야지!" 📑
- IR: "지금 4페이지 보는 중" 📄
- ACC: "요리 결과는 여기에!" 🍳

**빠른 비밀**: 캐시라는 "친구 책상"이 옆에 있어서, 도서관(RAM)까지 안 가도 돼요! 📚

**요즘 CPU**: 한 두뇌가 아니라 4개, 8개, 16개... 여러 두뇌가 같이 일해요! 🧠🧠🧠
