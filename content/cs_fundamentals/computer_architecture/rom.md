# ROM (Read Only Memory)

## 핵심 인사이트 (3줄 요약)
> **읽기 전용 비휘발성 메모리**. 전원이 꺼져도 데이터가 보존된다. BIOS/펌웨어 저장에 사용되며, PROM, EPROM, EEPROM 등으로 발전했다.

## 1. 개념
ROM(Read Only Memory)은 **데이터를 읽기만 할 수 있는 비휘발성 메모리**다. 제조 시 데이터가 기록되거나, 특수한 방법으로 수정 가능하다.

> 비유: "독서용 책" - 읽기만 하고 쓰기는 안 됨 (특별한 경우 제외)

## 2. ROM 종류

| 종류 | 특징 | 수정 |
|------|------|------|
| Mask ROM | 제조 시 기록 | 불가능 |
| PROM | 한 번 기록 | 1회만 |
| EPROM | UV로 삭제 | 가능 |
| EEPROM | 전기로 삭제 | 가능 |
| Flash | 블록 단위 | 가능 |

## 3. EEPROM vs Flash

```
EEPROM:
- 바이트 단위 수정
- 느림, 비쌈
- 설정 저장용

Flash Memory:
- 블록 단위 수정
- 빠름, 저렴
- SSD, USB용
```

## 4. 구조 (NOR Flash)

```
NOR Flash 셀:

   Bit Line
       │
    ┌──┴──┐
    │ FG  │ ← 플로팅 게이트 (전하 저장)
    └──┬──┘
       │
    Control Gate
       │
    Source ─ Drain

읽기: 게이트 전압 → 전류 흐름 감지
쓰기: 높은 전압 → 전하 주입
삭제: 높은 전압 → 전하 제거
```

## 5. NOR vs NAND Flash

| 항목 | NOR | NAND |
|------|-----|------|
| 구조 | 병렬 | 직렬 |
| 읽기 | 빠름 (XIP) | 느림 |
| 쓰기 | 느림 | 빠름 |
| 밀도 | 낮음 | 높음 |
| 용도 | 펌웨어 | SSD |

## 6. 장단점

| 장점 | 단점 |
|-----|------|
| 비휘발성 | 쓰기 속도 느림 |
| 내구성 좋음 | 쓰기 횟수 제한 |
| 전력 소모 적음 | RAM보다 느림 |

## 7. 활용 분야

1. **BIOS/UEFI**: 부팅 펌웨어
2. **임베디드**: 펌웨어 저장
3. **SSD**: 대용량 저장
4. **마이크로컨트롤러**: 프로그램 메모리

## 8. Flash 내구성

```
Flash 수명 (P/E Cycle):
- SLC: 100,000 회
- MLC: 3,000~10,000 회
- TLC: 1,000~3,000 회
- QLC: 100~1,000 회

→ Wear Leveling으로 수명 연장
```

## 9. 실무에선? (기술사적 판단)
- **SSD**: NAND Flash가 대세
- **임베디드**: Flash 기반 MCU
- **보안**: OTP(One-Time Programmable) ROM
- **BIOS**: SPI Flash 사용

## 10. 관련 개념
- RAM (Random Access Memory)
- 플래시 메모리
- BIOS/UEFI

## 어린이를 위한 종합 설명

**ROM은 "새겨진 돌비석"이야!**

책상(RAM) 위의 글은 지워지지만, 돌비석(ROM)에 새긴 글은 영원히 남아요!

```
전기가 켜져도: ✅ 내용 유지
전기가 꺼져도: ✅ 내용 유지
```

**ROM의 진화**:
1. **Mask ROM**: 공장에서 새겨서 못 바꿔요 🪨
2. **PROM**: 한 번만 새길 수 있어요
3. **EPROM**: 햇빛에 지워요 🌞
4. **EEPROM/Flash**: 전기로 지워요 ⚡

**어디에 쓸까요?**
- 컴퓨터가 처음 켜질 때 읽는 "시동 안내서"
- 전원 꺼져도 있어야 하는 "기본 규칙"

ROM은 컴퓨터의 **기억해야 할 가장 중요한 것**들을 저장해요! 📜
