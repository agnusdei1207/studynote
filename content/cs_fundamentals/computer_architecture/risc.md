# RISC (Reduced Instruction Set Computer)

## 핵심 인사이트 (3줄 요약)
> **단순하고 적은 수의 명령어**를 가진 프로세서 아키텍처. ARM, RISC-V가 대표적이며, 명령어가 단순해 **파이프라이닝과 슈퍼스칼라** 구현이 쉽다. 모바일·임베디드 시장을 지배한다.

## 1. 개념
RISC(Reduced Instruction Set Computer)는 **적은 수의 단순한 명령어**로 구성된 아키텍처다. 각 명령어가 한 클럭에 실행되도록 설계되어 파이프라이닝에 최적화되어 있다.

> 비유: "전문 요리팀" - 각자 단순한 일만 빠르게 수행

## 2. 등장 배경
- 1980년대 연구: 복잡한 명령어 사용 빈도가 낮음 발견
- 파이프라이닝: 단순 명령어가 파이프라인에 유리
- 컴파일러 최적화: 컴파일러가 단순 명령어 조합으로 최적화 가능

## 3. RISC 설계 원칙

| 원칙 | 설명 |
|------|------|
| 단순 명령어 | 1클럭에 1명령어 실행 |
| 고정 길이 | 명령어 길이 일정 (보통 32비트) |
| Load/Store | 메모리 접근은 Load/Store만 |
| 많은 레지스터 | 메모리 접근 최소화 |
| 하드웨어 단순 | 컴파일러가 복잡도 담당 |

## 4. 대표 예시: ARM, RISC-V

```
RISC 명령어 예시 (RISC-V):

ADD  x1, x2, x3    ; x1 = x2 + x3
SUB  x4, x5, x6    ; x4 = x5 - x6
LW   x7, 0(x8)     ; x7 = Memory[x8 + 0]
SW   x9, 4(x10)    ; Memory[x10 + 4] = x9
BEQ  x1, x2, label ; if x1 == x2, jump
```

## 5. 장단점

| 장점 | 단점 |
|-----|------|
| 파이프라이닝 쉬움 | 코드 크기가 큼 |
| 전력 소모 적음 | 컴파일러 복잡 |
| 하드웨어 단순 | 명령어 수 많음 |
| 슈퍼스칼라 유리 | - |

## 6. RISC vs CISC 비교

| 항목 | RISC | CISC |
|------|------|------|
| 명령어 수 | ~100개 | ~1000개 |
| 명령어 길이 | 고정 (32bit) | 가변 |
| 실행 시간 | 1 클럭 | 여러 클럭 |
| 레지스터 | 많음 (32+) | 적음 (16) |
| 메모리 접근 | Load/Store | 모든 명령어 |
| 대표 | ARM, RISC-V | x86 |

## 7. RISC-V 아키텍처

```
RISC-V 명령어 포맷 (32비트):

R-type: [funct7][rs2][rs1][funct3][rd][opcode]
I-type: [imm12    ][rs1][funct3][rd][opcode]
S-type: [imm7 ][rs2][rs1][funct3][imm5][opcode]
B-type: [imm7 ][rs2][rs1][funct3][imm5][opcode]

특징: 오픈 소스, 모듈화 (RV32I + 확장)
```

## 8. 활용 분야

1. **모바일**: ARM (iPhone, Android)
2. **IoT/임베디드**: ARM Cortex-M, RISC-V
3. **서버**: AWS Graviton (ARM), RISC-V
4. **AI 가속기**: RISC-V 기반

## 9. 파이프라이닝 효과

```
5단계 파이프라인 (RISC):

IF | ID | EX | MEM | WB |
   IF | ID | EX  | MEM | WB |
      IF | ID  | EX  | MEM | WB |

CPI 이상적 = 1 (한 클럭에 한 명령어 완료)
```

## 10. 실무에선? (기술사적 판단)
- **모바일 시장**: ARM이 95% 이상 점유
- **오픈 소스**: RISC-V가 새로운 표준으로 부상
- **AI/ML**: RISC-V 커스터마이징 용이
- **서버**: ARM 서버가 점유율 확대 중

## 11. 관련 개념

## 어린이를 위한 종합 설명

**RISC는 "분업하는 요리팀"이야!**

RISC 주방에서는 각 요리사가 딱 하나씩만 해요:

```
요리사 A: 채소 썰기만 해요 🥕
요리사 B: 불에 볶기만 해요 🔥
요리사 C: 접시에 담기만 해요 🍽️
```

각자 단순한 일만 해서 **엄청 빨라요!**

**비교**:
- CISC: "파스타 만들어!" → 한 요리사가 다 함 (복잡)
- RISC: "물 끓여! 면 넣어! 저어!" → 분업 (빠름)

**전기도 적게 써요!** 그래서 핸드폰 배터리가 오래가요! 🔋

**RISC-V**: 누구나 무료로 쓸 수 있는 RISC 설계도예요! 레고처럼 원하는 대로 만들 수 있어요!
