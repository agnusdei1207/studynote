# DSP (Digital Signal Processor)

## 핵심 인사이트 (3줄 요약)
> **디지털 신호 처리**에 특화된 프로세서. 오디오, 비디오, 통신 신호 처리에 최적화된 하드웨어 구조를 가진다. 실시간 처리를 위한 고정 소수점 연산과 MAC 연산에 강하다.

## 1. 개념
DSP(Digital Signal Processor)는 **디지털 신호를 실시간으로 처리**하기 위해 설계된 특수 목적 프로세서다. 오디오, 비디오, 통신 등에서 사용된다.

> 비유: "음악 전문가" - 일반 요리사(CPU)보다 음악(신호)을 더 잘 다룸

## 2. DSP 특징

| 특징 | 설명 |
|------|------|
| MAC 연산 | Multiply-Accumulate (곱셈+누적) 하드웨어 |
| 하버드 구조 | 명령어/데이터 메모리 분리 |
| 고정 소수점 | 빠른 실수 연산 |
| 제로 오버헤드 루프 | 루프 오버헤드 없음 |
| SIMD | 병렬 처리 |

## 3. MAC 연산

```
MAC (Multiply-Accumulate):

Y = Σ(a[i] × x[i])  // 필터, FFT의 핵심

일반 CPU:
1. a[i] 로드
2. x[i] 로드
3. 곱셈
4. 누적
→ 여러 클럭 소요

DSP:
1. a[i] × x[i] + Y → Y (1 클럭!)
→ 전용 하드웨어로 한 클럭에 수행
```

## 6. DSP vs 일반 CPU

| 항목 | DSP | CPU |
|------|-----|-----|
| MAC 연산 | 1 클럭 | 4+ 클럭 |
| 실시간성 | 보장 | 불확실 |
| 전력 | 낮음 | 높음 |
| 유연성 | 낮음 | 높음 |
| 가격 | 저렴 | 비쌈 |

## 8. DSP 프로그래밍 예시

```c
// FIR 필터 (DSP 최적화)
void fir_filter(int *input, int *output, int *coeff, int n) {
    int i, j;
    int acc;

    for (i = 0; i < n; i++) {
        acc = 0;
        for (j = 0; j < TAPS; j++) {
            // MAC 연산 (DSP에서 1클럭)
            acc += coeff[j] * input[i - j];
        }
        output[i] = acc;
    }
}
```

## 어린이를 위한 종합 설명

**DSP는 "소리 전문가"야!**

일반 CPU는 만능 요리사지만, DSP는 **소리만 전문으로 다루는 요리사**예요!

```
일반 CPU: 곱하기 → 더하기 → 곱하기 → 더하기...
          (여러 단계)

DSP: 곱하기+더하기를 한 번에! 🎵
     (MAC 연산)
```

**어디에 쓸까요?**
- 🎧 노이즈 캔슬링 이어폰
- 📱 휴대폰 통화 품질
- 🚗 자동차 레이더
- 🎤 음성 비서

**DSP의 비밀**: 소리는 "곱하고 더하고"를 엄청 많이 해야 하는데, DSP는 이걸 한 번에 할 수 있어요! 그래서 실시간으로 소리를 깨끗하게 만들 수 있어요! 🎶
