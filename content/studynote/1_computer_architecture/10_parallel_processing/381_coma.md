# 381. COMA (Cache-Only Memory Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 메모리를 캐시로 구성하여 데이터를 동적으로 이동시키는 구조
> 2. **가치**: 데이터 지역성 자동 최적화, 높은 유연성
> 3. **융합**: 분산 캐시, 마이그레이션, NUMA 변형

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

COMA(Cache-Only Memory Access)는 **모든 메인 메모리를 캐시처럼 취급하여, 데이터가 필요한 노드로 자동으로 이동하는 공유 메모리 아키텍처**다. NUMA의 변형으로, 하드웨어가 데이터 배치를 자동으로 최적화한다.

### 💡 비유: 이동식 물류

COMA는 **이동식 물류**와 같다. 창고가 있다. 하지만 물건이 고정되지 않는다. 서울에서 많이 쓰면 서울로 이동한다. 부산에서 많이 쓰면 부산으로 이동한다. 물건이 스스로 이동한다. COMA가 똑같다. 데이터가 필요한 곳으로 이동한다.

│  │  • Attraction Memory: 각 노드의 로컬 메모리가 캐시처럼 동작       │ │
│  │  • 동적 마이그레이션: 데이터가 필요한 노드로 자동 이동             │ │

## Ⅱ. COMA 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                COMA 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【COMA 동작 원리】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 초기 상태:                                                          │ │
│  │     데이터 A가 노드 0에 있음                                       │ │
│  │                                                               │ │
│  │  2. 노드 1이 A에 접근:                                                   │ │
│  │     • 노드 0에서 A를 복사                                          │ │
│  │     • 노드 1의 attraction memory에 저장                           │ │
│  │     • 이후 노드 1에서 로컬 접근                                     │ │
│  │                                                               │ │
│  │  3. 데이터 이동 (Migration):                                             │ │
│  │     • 노드 1만 A를 사용하면 A가 노드 1으로 이동                     │ │
│  │     • 노드 0의 복사본은 제거                                        │ │
│  │                                                               │ │
│  │  4. 복제 (Replication):                                                  │ │
│  │     • 여러 노드가 읽기만 하면 복제본 유지                           │ │
│  │     • 쓰기 시 복제본 무효화 또는 갱신                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【COMA 장단점】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 자동 지역성 최적화                                             │ │
│  │  • 프로그래머 부담 감소                                           │ │
│  │  • 동적 워크로드 대응                                             │ │
│  │  • 로컬 접근률 향상                                               │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 하드웨어 복잡성                                                │ │
│  │  • 디렉터리 오버헤드                                              │ │
│  │  • 메모리 중복 (복제)                                             │ │
│  │  • 구현 비용                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【COMA 구현 예시】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • KSR-1 (Kendall Square Research): 최초 상용 COMA              │ │
│  │  • S-COMA (Simple COMA): 단순화된 구현                           │ │
│  │  • CC-NUMA with page migration: 소프트웨어적 COMA 유사            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• COMA: 캐시 전용 메모리 접근
• 특징: 모든 메모리가 캐시처럼 동작
• Attraction Memory: 노드별 대용량 캐시
• 마이그레이션: 데이터 자동 이동
• 복제: 읽기 전용 데이터 복사
• 장점: 자동 지역성, 프로그래머 부담 감소
• 단점: 하드웨어 복잡, 메모리 중복
• 구현: KSR-1, S-COMA
• 현대: AutoNUMA로 소프트웨어적 구현
• 비교: NUMA vs COMA vs CC-NUMA
```
