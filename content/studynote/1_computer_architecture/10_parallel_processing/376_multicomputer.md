# 376. 다중 컴퓨터 (Multicomputer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 메모리를 가진 여러 독립 컴퓨터가 네트워크로 연결된 시스템
> 2. **가치**: 높은 확장성, 비용 효율, 유연성
> 3. **융합**: 클러스터, MPP, 메시지 패싱, MPI

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

다중 컴퓨터(Multicomputer)는 **각각 독립적인 메모리를 가진 여러 대의 컴퓨터가 네트워크로 연결되어 메시지 패싱을 통해 통신하며 협력하는 분산 시스템**이다. 공유 메모리가 없는 MIMD 시스템이다.

### 💡 비유: 분점 체인

다중 컴퓨터는 **분점 체인**과 같다. 각 매장이 독립적이다. 자기 창고가 있다. 자기 재고가 있다. 전화로 연락한다. "재고 있어?" "있어, 보낼게" 배달으로 물건 이동. 다중 컴퓨터가 똑같다. 각 노드가 독립, 메시지로 통신, 데이터 전송.

## Ⅱ. 다중 컴퓨터 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                다중 컴퓨터 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【다중 컴퓨터 유형】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 클러스터 (Cluster):                                                 │ │
│  │     • 상용 하드웨어 연결                                         │ │
│  │     • 이더넷/InfiniBand                                         │ │
│  │     • 비용 효율                                                  │ │
│  │     • 범용 컴퓨터 조합                                           │ │
│  │                                                               │ │
│  │  2. MPP (Massively Parallel Processor):                                 │ │
│  │     • 전용 하드웨어                                              │ │
│  │     • 고속 인터커넥트                                            │ │
│  │     • 슈퍼컴퓨터                                                  │ │
│  │     • 수천~수만 노드                                             │ │
│  │                                                               │ │
│  │  3. COW (Cluster of Workstations):                                      │ │
│  │     • 워크스테이션 모음                                          │ │
│  │     • 일반 네트워크                                              │ │
│  │     • 유휴 자원 활용                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메시지 패싱】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  MPI (Message Passing Interface):                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │  MPI_Send(buffer, count, type, dest, tag, comm)         │  │ │
│  │  │  MPI_Recv(buffer, count, type, source, tag, comm, stat) │  │ │
│  │  │                                                         │  │ │
│  │  │  노드 0                         노드 1                   │  │ │
│  │  │  ┌─────────┐                   ┌─────────┐              │  │ │
│  │  │  │ 데이터   │ ──Send()──────→  │ Recv()  │              │  │ │
│  │  │  │ 준비    │                   │ 수신    │              │  │ │
│  │  │  └─────────┘                   └─────────┘              │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  │  통신 패턴:                                                              │ │
│  │  • Point-to-Point: 1:1 통신                                      │ │
│  │  • Collective: 1:N, N:1, N:N (Broadcast, Scatter, Gather)        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 높은 확장성 (수천 노드)                                        │ │
│  │  • 비용 효율 (상용 HW)                                            │ │
│  │  • 내결함성 (노드 장애 격리)                                      │ │
│  │  • 유연성 (다양한 구성)                                           │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 프로그래밍 복잡 (메시지 패싱)                                  │ │
│  │  • 통신 오버헤드                                                  │ │
│  │  • 지연 시간                                                      │ │
│  │  • 데이터 분배 어려움                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 다중 컴퓨터 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                다중 컴퓨터 × 타 과목 융합                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • MIMD: 다중 컴퓨터는 MIMD 구현                                    │
│  • 분산 메모리: 각 노드 독립 메모리                                 │
│  • 인터커넥트: 네트워크 패브릭                                      │
│  • 노드: CPU + 메모리 + 네트워크                                   │
│                                                                     │
│  【운영체제】                                                          │
│  • 분산 OS: 여러 OS 협력                                            │
│  • 통신: 소켓, RPC                                                  │
│  • 파일 시스템: 분산 파일 시스템                                    │
│  • 프로세스: 분산 프로세스 관리                                     │
│                                                                     │
│  【네트워크】                                                          │
│  • 프로토콜: TCP/IP, InfiniBand                                     │
│  • 토폴로지: Fat-tree, Torus                                        │
│  • 대역폭: 고속 인터커넥트                                          │
│  • 지연: RDMA, RoCE                                                 │
│                                                                     │
│  【프로그래밍】                                                        │
│  • MPI: 메시지 패싱 인터페이스                                      │
│  • PVM: 병렬 가상 머신                                              │
│  • 분산: MapReduce, Spark                                           │
│  • 통신: gRPC, Thrift                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  다중 컴퓨터 예시:                                                               │ │
│  │  ──────────────────────                                              │ │
│  │  시스템        노드 수      용도             인터커넥트              │ │
│  │  ──────        ──────      ────             ──────────              │ │
│  │  Beowulf       수십~수백    연구 클러스터     이더넷                  │ │
│  │  HPC 클러스터  수천        과학 연산         InfiniBand              │ │
│  │  구글 데이터   수만        웹 서비스         커스텀                  │ │
│  │  슈퍼컴퓨터     수십만      시뮬레이션       전용 패브릭              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 다중 컴퓨터 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                다중 컴퓨터 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【클러스터 정보 확인】                                                            │)
│  ──────────────────                                                  │
│  $ pdsh -w node[0-15] hostname                                     │
│  # 병렬로 여러 노드 명령 실행                                        │
│                                                                     │
│  【MPI 프로그램 실행】                                                              │)
│  ──────────────────                                                  │
│  $ mpirun -np 16 -hostfile hosts ./mpi_program                     │
│  # 16개 프로세스로 MPI 실행                                         │
│                                                                     │
│  【네트워크 확인】                                                                │)
│  ──────────────────                                                  │
│  $ ibv_devinfo                                                      │
│  # InfiniBand 장치 정보                                             │
│                                                                     │
│  【노드 간 통신 지연】                                                              │)
│  ──────────────────                                                  │
│  $ ping node1                                                       │
│  $ ibping node1                                                     │
│  # 지연 시간 확인                                                   │
│                                                                     │
│  【다중 컴퓨터 최적화 팁】                                                            │)
│  ──────────────────                                                  │
│  1. 통신: 메시지 크기 최적화                                        │
│  2. 계산/통신: 비율 균형                                            │
│  3. 데이터: 지역성 활용                                             │
│  4. 부하: 균등 분배                                                 │
│  5. 네트워크: 고속 인터커넥트 사용                                  │
│  6. 측정: 통신 패턴 분석                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 다중 컴퓨터: 분산 메모리 + 네트워크
• MIMD: 다중 명령 다중 데이터
• 통신: 메시지 패싱 (MPI)
• 유형: 클러스터, MPP, COW
• 장점: 확장성, 비용 효율, 내결함
• 단점: 프로그래밍 복잡, 통신 오버헤드
• 인터커넥트: 이더넷, InfiniBand
• 프로그래밍: MPI, PVM, MapReduce
• 활용: HPC, 웹 서비스, 빅데이터
• 비교: 다중 프로세서 vs 다중 컴퓨터
```
