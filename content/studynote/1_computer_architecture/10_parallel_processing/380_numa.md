# 380. NUMA (Non-Uniform Memory Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 접근 시간이 위치에 따라 다른 공유 메모리 구조
> 2. **가치**: 높은 확장성, 대규모 시스템 지원
> 3. **융합**: 멀티소켓, 로컬/원격 메모리, OS 최적화

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

NUMA(Non-Uniform Memory Access)는 **다중 프로세서 시스템에서 각 CPU가 로컬 메모리와 원격 메모리를 가지며, 로컬 메모리 접근이 원격보다 빠른 공유 메모리 아키텍처**다. 멀티소켓 서버의 표준이다.

### 💡 비유: 지역 창고 체인

NUMA는 **지역 창고 체인**과 같다. 서울 창고가 있다. 부산 창고가 있다. 서울 직원은 서울 창고에서 바로 꺼낸다. 빠르다. 부산 창고는 택배로 받는다. 느리다. 같은 회사지만 거리가 다르다. NUMA가 똑같다. 로컬 메모리는 빠르고, 원격 메모리는 느리다.

## Ⅱ. NUMA 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                NUMA 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NUMA 구성 요소】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  NUMA 노드 (Node):                                                      │ │
│  │  • CPU 코어 그룹 + 로컬 메모리                                    │ │
│  │  • 하나의 소켓 또는 칩렛                                          │ │
│  │                                                               │ │
│  │  로컬 메모리 (Local Memory):                                            │ │
│  │  • 해당 노드에 직접 연결된 메모리                                 │ │
│  │  • 가장 빠른 접근                                                │ │
│  │                                                               │ │
│  │  원격 메모리 (Remote Memory):                                           │ │
│  │  • 다른 노드의 메모리                                             │ │
│  │  • 인터커넥트 경유                                                │ │
│  │  • 로컬보다 느림                                                 │ │
│  │                                                               │ │
│  │  인터커넥트 (Interconnect):                                             │ │
│  │  • QPI/UPI (Intel)                                               │ │
│  │  • HyperTransport/Infinity Fabric (AMD)                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NUMA 장단점】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 높은 확장성 (수십~수백 코어)                                   │ │
│  │  • 로컬 메모리 높은 대역폭                                        │ │
│  │  • 멀티소켓 지원                                                  │ │
│  │  • 비용 효율                                                      │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 프로그래밍 복잡 (지역성 고려)                                  │ │
│  │  • 원격 접근 지연                                                 │ │
│  │  • OS/앱 최적화 필요                                              │ │
│  │  • 불균형 성능                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NUMA 최적화 전략】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 메모리 할당: 로컬 노드 우선                                  │ │
│  │  2. 스케줄링: 데이터가 있는 노드에서 실행                         │ │
│  │  3. 데이터 배치: 노드별 데이터 분할                               │ │
│  │  4. 마이그레이션: 핫 페이지 이동                                  │ │
│  │  5. 인터리빙: 균등 분산 (일부 경우)                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• NUMA: 비균일 메모리 접근
• 구조: 노드별 로컬 메모리 + 인터커넥트
• 로컬 접근: 빠름 (~100ns)
• 원격 접근: 느림 (~150-200ns)
• 장점: 높은 확장성, 대규모 지원
• 단점: 프로그래밍 복잡, 지역성 고려
• 인터커넥트: QPI/UPI, HT/IF
• OS 최적화: NUMA 스케줄러, AutoNUMA
• 적용: 멀티소켓 서버, 대규모 DB
• 비교: UMA vs NUMA
```
