# 378. 분산 메모리 시스템 (Distributed Memory System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 각 프로세서가 독립적인 로컬 메모리를 가진 병렬 시스템
> 2. **가치**: 높은 확장성, 모듈성, 내결함성
> 3. **융합**: 메시지 패싱, 클러스터, MPP

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

분산 메모리 시스템(Distributed Memory System)은 **각 프로세서가 자신만의 로컬 메모리를 가지며, 다른 프로세서의 메모리에 직접 접근할 수 없고 메시지 패싱을 통해서만 데이터를 교환하는 병렬 컴퓨팅 아키텍처**다.

### 💡 비유: 독립 매장 체인

분산 메모리 시스템은 **독립 매장 체인**과 같다. 각 매장이 자기 창고를 가진다. 다른 매장 창고에 직접 들어갈 수 없다. 전화로 요청한다. "물건 좀 보내줘" 배달으로 받는다. 분산 메모리가 똑같다. 각 CPU, 자기 메모리, 메시지로 통신.

## Ⅱ. 분산 메모리 시스템 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분산 메모리 시스템 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 특징】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 독립 메모리:                                                       │ │
│  │     • 각 노드가 전용 로컬 메모리 보유                               │ │
│  │     • 다른 노드 메모리 직접 접근 불가                               │ │
│  │                                                               │ │
│  │  2. 메시지 패싱 통신:                                                   │ │
│  │     • Send/Recv로 데이터 교환                                      │ │
│  │     • 명시적 통신                                                  │ │
│  │                                                               │ │
│  │  3. 캐시 일관성 불필요:                                                 │ │
│  │     • 각 메모리가 단일 소유자                                        │ │
│  │     • 코히런시 오버헤드 없음                                        │ │
│  │                                                               │ │
│  │  4. 높은 확장성:                                                       │ │
│  │     • 노드 추가로 성능 선형 확장 가능                               │ │
│  │     • 수천~수만 노드까지                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메시지 패싱 방식】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  동기 메시지 패싱 (Synchronous):                                        │ │
│  │  • Send가 Recv 완료까지 대기                                       │ │
│  │  • rendezvous 방식                                                │ │
│  │  • 프로그래밍 쉬움, 지연 있음                                        │ │
│  │                                                               │ │
│  │  비동기 메시지 패싱 (Asynchronous):                                     │ │
│  │  • Send가 즉시 반환                                                │ │
│  │  • 버퍼 사용                                                       │ │
│  │  • 높은 성능, 프로그래밍 복잡                                        │ │
│  │                                                               │ │
│  │  MPI 예:                                                                │ │
│  │  MPI_Send()    - 동기/비동기 모드                                   │ │
│  │  MPI_Isend()   - 비동기                                            │ │
│  │  MPI_Recv()    - 수신                                               │ │
│  │  MPI_Wait()    - 완료 대기                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 높은 확장성 (수만 노드)                                         │ │
│  │  • 캐시 일관성 오버헤드 없음                                        │ │
│  │  • 내결함성 (노드 격리)                                            │ │
│  │  • 비용 효율 (상용 HW)                                             │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 프로그래밍 복잡 (명시적 통신)                                   │ │
│  │  • 통신 오버헤드                                                  │ │
│  │  • 데이터 분배 어려움                                             │ │
│  │  • 지연 시간                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 분산 메모리 시스템 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분산 메모리 시스템 분석                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MPI 프로그래밍 예시】                                                           │)
│  ──────────────────                                                  │
│  #include <mpi.h>                                                   │
│  int main(int argc, char** argv) {                                  │
│      MPI_Init(&argc, &argv);                                        │
│      int rank, size;                                                │
│      MPI_Comm_rank(MPI_COMM_WORLD, &rank);                          │
│      MPI_Comm_size(MPI_COMM_WORLD, &size);                          │
│                                                                     │
│      if (rank == 0) {                                               │
│          int data = 42;                                             │
│          MPI_Send(&data, 1, MPI_INT, 1, 0, MPI_COMM_WORLD);         │
│      } else {                                                       │
│          int data;                                                  │
│          MPI_Recv(&data, 1, MPI_INT, 0, 0, MPI_COMM_WORLD, ...);    │
│      }                                                              │
│      MPI_Finalize();                                                │
│  }                                                                  │
│                                                                     │
│  【네트워크 성능 확인】                                                            │)
│  ──────────────────                                                  │
│  $ ibstat                         # InfiniBand 상태                  │
│  $ ibping <remote>                # 지연 시간 확인                    │
│  $ qperf <remote> tcp_bw          # 대역폭 측정                      │
│                                                                     │
│  【분산 메모리 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 통신 최소화: 메시지 수 줄이기                                   │
│  2. 비동기: 비블로킹 통신 활용                                      │
│  3. 중첩: 계산과 통신 병렬                                         │
│  4. 데이터: 지역성 활용                                            │
│  5. 부하: 균등 분배                                                │
│  6. 측정: 통신 패턴 분석                                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 분산 메모리: 각 노드 독립 로컬 메모리
• 통신: 메시지 패싱만 가능
• 장점: 확장성, 내결함, 비용 효율
• 단점: 프로그래밍 복잡, 통신 오버헤드
• MPI: 메시지 패싱 표준 인터페이스
• 인터커넥트: 이더넷, InfiniBand
• 구현: 클러스터, MPP, 그리드
• 스케일아웃: 노드 추가로 확장
• RDMA: 원격 직접 메모리 접근
• 비교: 공유 메모리 vs 분산 메모리
```
