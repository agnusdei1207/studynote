# 373. 벡터 프로세서 (Vector Processor)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 벡터(배열) 데이터를 효율적으로 처리하는 SIMD 기반 프로세서
> 2. **가치**: 과학 연산, 행렬 연산, 고성능 컴퓨팅
> 3. **융합**: 파이프라인, 체인닝, 마스킹

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

벡터 프로세서(Vector Processor)는 **벡터(1차원 배열) 데이터에 대해 단일 명령어로 일련의 연산을 수행하는 특수 목적 프로세서**다. SIMD 방식의 대표적 구현으로, 과학 계산, 시뮬레이션, 그래픽스 등에서 고성능을 발휘한다.

### 💡 비유: 연속 공격

벡터 프로세서는 **연속 공격**과 같다. 검객이 있다. 한 번 휘두르면 여러 적을 벤다. 같은 동작으로 여러 대상. 순서대로 빠르게. 검 1번에 적 10명. 벡터 프로세서가 똑같다. 명령 1개로 데이터 여러 개를 연속 처리한다.

## Ⅱ. 벡터 프로세서 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                벡터 프로세서 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 특징】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 벡터 레지스터 (Vector Registers):                                  │ │
│  │     • 여러 요소를 저장하는 전용 레지스터                           │ │
│  │     • 예: 64 x 64비트 = 512바이트                                │ │
│  │                                                               │ │
│  │  2. 벡터 파이프라인 (Vector Pipeline):                                 │ │
│  │     • 각 요소가 파이프라인 단계를 통과                              │ │
│  │     • 매 클럭마다 결과 생성                                      │ │
│  │                                                               │ │
│  │  3. 체인닝 (Chaining):                                                 │ │
│  │     • 벡터 연산 결과를 즉시 다음 연산에 사용                        │ │
│  │     • VADD → VMUL 연속 실행                                      │ │
│  │                                                               │ │
│  │  4. 마스킹 (Masking):                                                  │ │
│  │     • 조건부 연산 지원                                           │ │
│  │     • 마스크 비트으로 요소별 선택                                 │ │
│  │                                                               │ │
│  │  5. 스트라이드 (Stride):                                               │ │
│  │     • 비연속 메모리 접근 지원                                    │ │
│  │     • 행렬의 행/열 접근                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【체인닝 예시】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  A = B + C  (VADD)                                                │ │
│  │  D = A * E  (VMUL)  ← A가 VADD 결과                              │ │
│  │                                                               │ │
│  │  체인닝 없이:                                                        │ │
│  │  VADD 완료 → VMUL 시작 = 직렬                                    │ │
│  │                                                               │ │
│  │  체인닝:                                                             │ │
│  │  VADD 첫 결과 → 바로 VMUL 입력                                   │ │
│  │  → 병렬로 진행, 성능 2배                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대표 벡터 프로세서】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • Cray-1 (1976): 최초 상용 벡터 슈퍼컴퓨터                       │ │
│  │  • NEC SX 시리즈: 일본 벡터 슈퍼컴퓨터                            │ │
│  │  • Fujitsu A64FX: ARM 기반 벡터 (富岳)                           │ │
│  │  • RISC-V V 확장: 오픈 벡터 확장                                  │ │
│  │  • ARM SVE: 확장 가능 벡터 확장                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 벡터 프로세서 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                벡터 프로세서 분석                                           │

│  【벡터 확장 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/cpuinfo | grep -i vector                              │
│  # RISC-V 벡터 확장                                                 │
│                                                                     │

│  # ARM SVE 지원 확인                                                │
│                                                                     │
│  【ARM SVE 벡터 길이】                                                            │)
│  ──────────────────                                                  │
│  $ aarch64-linux-gnu-gcc -dM -E - < /dev/null | grep SVE           │
│  # SVE 벡터 길이 (가변: 128-2048비트)                               │
│                                                                     │
│  【벡터화 성능 측정】                                                            │)
│  ──────────────────                                                  │
│  $ perf stat -e simd_fp_256 ./vector_program                       │
│  # SIMD 연산 카운트                                                 │
│                                                                     │
│  【벡터 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 데이터 배치: 연속 메모리                                        │
│  2. 정렬: 캐시 라인 정렬                                            │
│  3. 컴파일러: -O3 -march=native                                    │
│  4. 라이브러리: 최적화 BLAS 활용                                    │
│  5. 측정: 벡터화 효율 확인                                          │
│  6. 스트라이드: 접근 패턴 최적화                                    │

```
