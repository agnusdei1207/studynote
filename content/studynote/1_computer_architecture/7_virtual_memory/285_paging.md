# 285. 페이징 (Paging)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고정 크기 블록(페이지) 단위 메모리 관리
> 2. **가치**: 외부 단편화 해결, 효율적 메모리 활용
> 3. **융합**: 페이지, 프레임, 페이지 테이블, TLB

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

페이징(Paging)은 **메모리를 고정 크기 블록(페이지/프레임)으로 나누어 관리하는 기법**이다. 외부 단편화를 해결하고, 물리 메모리를 불연속적으로 할당할 수 있다.

### 💡 비유: 책 페이지와 책장

페이징은 **책 페이지와 책장**과 같다. 책은 페이지 단위로 나뉘고, 책장에는 페이지 단위로 보관한다. 100페이지 책이 있어도, 책장에 순서대로 있을 필요 없다. 1~10페이지는 1층, 11~20페이지는 3층에 둬도 된다. 필요할 때 모으면 된다.

### 페이징 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이징 개요                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이징 기본 개념】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  페이지 (Page):                                                       │ │
│  │  • 논리 메모리(가상 메모리)의 고정 크기 블록                             │ │
│  │  • 프로세스가 보는 메모리 단위                                          │ │
│  │  • 크기: 4KB가 표준 (2MB, 1GB도 가능)                                  │ │
│  │                                                               │ │
│  │  프레임 (Frame):                                                      │ │
│  │  • 물리 메모리(RAM)의 고정 크기 블록                                     │ │
│  │  • 페이지와 동일한 크기                                                │ │
│  │  • 실제 RAM의 슬롯                                                   │ │
│  │                                                               │ │
│  │  페이지 크기 = 프레임 크기 (필수!)                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이징 매핑】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  논리 메모리 (프로세스 A)            물리 메모리 (RAM)              │ │
│  │  ┌─────────────┐                 ┌─────────────┐               │ │
│  │  │  Page 0     │ ──────────────→ │  Frame 3    │               │ │
│  │  ├─────────────┤                 ├─────────────┤               │ │
│  │  │  Page 1     │ ──────────────→ │  Frame 7    │               │ │
│  │  ├─────────────┤                 ├─────────────┤               │ │
│  │  │  Page 2     │ ──────────────→ │  Frame 1    │               │ │
│  │  ├─────────────┤                 ├─────────────┤               │ │
│  │  │  Page 3     │ ──────────────→ │  Frame 9    │               │ │
│  │  └─────────────┘                 └─────────────┘               │ │
│  │                                                               │ │
│  │  → 페이지는 연속적이지만, 프레임은 불연속적                               │ │
│  │  → 페이지 테이블이 매핑 정보 저장                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이징 장단점】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 외부 단편화 없음 (고정 크기)                                        │ │
│  │  • 불연속 할당 가능                                                   │ │
│  │  • 메모리 효율적 활용                                                  │ │
│  │  • 공유 메모리 구현 쉬움                                               │ │
│  │  • 간단한 메모리 할당                                                  │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 내부 단편화 발생 가능                                               │ │
│  │  • 페이지 테이블 오버헤드                                              │ │
│  │  • 주소 변환 오버헤드                                                  │ │
│  │  • 메모리 접근 시간 증가                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 페이징 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이징 상세                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 구조】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  32비트 주소, 4KB 페이지:                                              │ │
│  │  ┌────────────────────────────────┬────────────────────────┐    │ │
│  │  │        페이지 번호 (20비트)       │     오프셋 (12비트)       │    │ │
│  │  │           VPN                   │       Offset           │    │ │
│  │  └────────────────────────────────┴────────────────────────┘    │ │
│  │  │←───────── 2^20 = 1M 페이지 ─────────→│←─ 2^12 = 4KB ─→│     │ │
│  │                                                               │ │
│  │  64비트 주소, 4KB 페이지:                                              │ │
│  │  ┌────────────────────────────────┬────────────────────────┐    │ │
│  │  │     페이지 번호 (52비트)          │     오프셋 (12비트)       │    │ │
│  │  │            VPN                  │       Offset           │    │ │
│  │  └────────────────────────────────┴────────────────────────┘    │ │
│  │  │←───────── 2^52 페이지 ────────────→│←─ 2^12 = 4KB ─→│     │ │
│  │                                                               │ │
│  │  → 오프셋은 변환 없이 그대로 사용                                        │ │
│  │  → VPN만 PFN으로 변환                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 크기 선택】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  페이지 크기   오프셋 비트   페이지 수      내부 단편화    TLB 커버리지      │ │
│  │  ──────        ─────────     ──────        ──────────     ──────────        │ │
│  │  4KB          12비트         많음          적음           작음              │ │
│  │  8KB          13비트         중간          중간           중간              │ │
│  │  16KB         14비트         적음          많음           큼                │ │
│  │  2MB          21비트         매우 적음      많음           매우 큼            │ │
│  │  1GB          30비트         극히 적음      매우 많음       극히 큼            │ │
│  │                                                               │ │
│  │  작은 페이지: 내부 단편화 적음, 페이지 테이블 큼, TLB 압박                   │ │
│  │  큰 페이지: 내부 단편화 많음, 페이지 테이블 작음, TLB 효율                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 테이블 엔트리 (PTE)】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌────────────────────────────────────────────────────────┐    │ │
│  │  │ PFN (물리 프레임 번호)    │ 권한 비트 │ 상태 비트 │ ...    │    │ │
│  │  │     (20~52비트)           │  R W X   │  P D A   │        │    │ │
│  │  └────────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  권한 비트:                                                            │ │
│  │  • R (Read): 읽기 가능                                                │ │
│  │  • W (Write): 쓰기 가능                                                │ │
│  │  • X (Execute): 실행 가능                                             │ │
│  │                                                               │ │
│  │  상태 비트:                                                             │ │
│  │  • P (Present): 메모리에 존재                                          │ │
│  │  • D (Dirty): 수정됨                                                   │ │
│  │  • A (Accessed): 접근됨                                                │ │
│  │  • U/S (User/Supervisor): 사용자/커널 모드                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이징 vs 세그멘테이션】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성            페이징              세그멘테이션                  │ │
│  │  ──────           ──────              ────────────                  │ │
│  │  블록 크기         고정                가변                          │ │
│  │  외부 단편화       없음                있음                          │ │
│  │  내부 단편화       있음                없음                          │ │
│  │  메모리 효율       높음                중간                          │ │
│  │  구현 복잡도       낮음                높음                          │ │
│  │  주소 변환         단순                복잡                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 페이징: 고정 크기 블록(페이지/프레임) 단위 메모리 관리
• 페이지: 논리 메모리의 고정 크기 블록 (보통 4KB)
• 프레임: 물리 메모리의 고정 크기 블록 (페이지와 동일 크기)
• 장점: 외부 단편화 없음, 불연속 할당, 메모리 효율
• 단점: 내부 단편화, 페이지 테이블 오버헤드
• 주소: [페이지 번호(VPN) | 오프셋]
• VPN: 페이지 번호, 변환 대상
• Offset: 페이지 내 위치, 변환 없이 그대로
• PTE: 페이지 테이블 엔트리 (PFN + 권한 + 상태)
• 페이지 크기: 4KB(표준), 2MB, 1GB(Huge Page)
```
