# 282. 가상 메모리 (Virtual Memory)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 메모리보다 큰 주소 공간을 소프트웨어적으로 제공
> 2. **가치**: 메모리 확장, 프로세스 격리, 보안
> 3. **융합**: MMU, 페이지, 스와핑, TLB

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

가상 메모리(Virtual Memory)는 **물리 메모리보다 큰 주소 공간을 프로세스에 제공하는 메모리 관리 기술**이다. 디스크를 메모리처럼 사용하여, 실제 RAM 크기에 관계없이 큰 프로그램을 실행할 수 있다.

### 💡 비유: 가상 도서관

가상 메모리는 **가상 도서관**과 같다. 실제 책장(물리 메모리)은 100권만 담을 수 있다. 하지만 전산 시스템으로 1000권의 책을 관리한다. 필요한 책만 책장에 두고, 나머지는 창고(디스크)에 둔다. 필요하면 창고에서 가져온다.

### 가상 메모리 개요

│                가상 메모리 개요                                             │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  핵심 아이디어:                                                       │ │
│  │  • 각 프로세스에 연속적인 주소 공간 제공                                 │ │
│  │  • 실제 물리 메모리보다 큰 공간 제공 가능                                │ │
│  │  • 필요한 부분만 물리 메모리에 로드                                     │ │
│  │  • 나머지는 디스크에 저장                                              │ │
│  │                                                               │ │
│  │  목적:                                                               │ │
│  │  1. 메모리 확장: RAM보다 큰 프로그램 실행                                │ │
│  │  2. 프로세스 격리: 각 프로세스 독립적 주소 공간                           │ │
│  │  3. 메모리 보호: 프로세스 간 메모리 침벌 방지                             │ │
│  │  4. 효율적 사용: 필요한 부분만 메모리에                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【가상 vs 물리 주소 공간】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  가상 주소 공간 (Virtual Address Space):                               │ │
│  │  ┌────────────────────────────────────────────────────────┐    │ │
│  │  │  프로세스 A (4GB)                                        │    │ │
│  │  │  0x0000_0000 ────────────────────── 0xFFFF_FFFF          │    │ │
│  │  └────────────────────────────────────────────────────────┘    │ │
│  │  ┌────────────────────────────────────────────────────────┐    │ │
│  │  │  프로세스 B (4GB)                                        │    │ │
│  │  │  0x0000_0000 ────────────────────── 0xFFFF_FFFF          │    │ │
│  │  └────────────────────────────────────────────────────────┘    │ │
│  │  → 각 프로세스는 자신만의 4GB 공간 가짐                                  │ │

│  │  ┌────────────────────────────────────────────────────────┐    │ │

│  │  │  0x0000_0000 ────────────────────── 0x3FFF_FFFF          │    │ │
│  │  └────────────────────────────────────────────────────────┘    │ │

│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │

│  │   │  (캐시)      │     │  (변환표)    │     │  (물리)      │        │ │

## Ⅱ. 가상 메모리 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                가상 메모리 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 변환 과정】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. CPU가 가상 주소 요청: VA = [VPN | Offset]                        │ │
│  │     • VPN: Virtual Page Number (페이지 번호)                         │ │
│  │     • Offset: 페이지 내 위치                                          │ │
│  │                                                               │ │
│  │  2. TLB 조회:                                                       │ │
│  │     • Hit: PFN(Physical Frame Number) 즉시 획득                       │ │
│  │     • Miss: 페이지 테이블 조회                                         │ │
│  │                                                               │ │
│  │  3. 페이지 테이블 조회:                                                │ │
│  │     • VPN으로 PFN 찾기                                               │ │
│  │     • Present bit 확인: 메모리에 있는지                                │ │
│  │                                                               │ │
│  │  4. 물리 주소 생성: PA = [PFN | Offset]                              │ │
│  │                                                               │ │
│  │  5. 메모리 접근 또는 Page Fault:                                      │ │
│  │     • Present=1: 메모리 접근                                         │ │
│  │     • Present=0: Page Fault → 디스크에서 로드                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 크기와 주소 구조】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  페이지 크기   Offset 비트   32비트 VA   64비트 VA                     │ │
│  │  ──────        ─────────     ─────────   ─────────                   │ │
│  │  4KB          12비트         VPN=20비트   VPN=52비트                   │ │
│  │  8KB          13비트         VPN=19비트   VPN=51비트                   │ │
│  │  16KB         14비트         VPN=18비트   VPN=50비트                   │ │
│  │  4MB          22비트         VPN=10비트   VPN=42비트                   │ │
│  │                                                               │ │
│  │  32비트, 4KB 페이지:                                                 │ │
│  │  ┌────────────────────────┬────────────────┐                       │ │
│  │  │    VPN (20비트)          │  Offset (12비트) │                       │ │
│  │  └────────────────────────┴────────────────┘                       │ │
│  │                                                               │ │
│  │  → 2^20 = 1M 페이지, 2^12 = 4KB 페이지 크기                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【가상 메모리 장단점】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 큰 주소 공간: 물리 메모리보다 큰 프로그램 실행 가능                      │ │
│  │  • 프로세스 격리: 각 프로세스 독립적 주소 공간                            │ │
│  │  • 메모리 보호: 접근 권한 관리 (R/W/X)                                  │ │
│  │  • 메모리 효율: 필요한 부분만 로드                                      │ │
│  │  • 공유 메모리: 프로세스 간 메모리 공유 가능                              │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 오버헤드: 주소 변환 비용                                            │ │
│  │  • Page Fault: 디스크 접근으로 느려짐                                  │ │
│  │  • 복잡성: MMU, 페이지 테이블 관리                                      │ │
│  │  • 스래싱: 과도한 페이지 교체                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 가상 메모리 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                가상 메모리 × 타 과목 융합                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • MMU: 주소 변환 하드웨어                                            │
│  • TLB: 페이지 테이블 캐시                                             │
│  • 페이지 테이블: 메모리에 저장                                         │
│  • Cache: 가상 vs 물리 주소 캐시                                       │
│                                                                     │
│  【운영체제】                                                          │
│  • 메모리 관리: 페이지 할당, 해제                                        │
│  • 스케줄링: 페이지 교체 알고리즘                                        │
│  • 스와핑: 디스크와 메모리 간 이동                                        │
│  • 페이지 폴트: OS가 처리                                              │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 버퍼 풀: 페이지 단위 관리                                            │
│  • 메모리 맵 파일: 가상 메모리 활용                                      │
│  • 대용량 데이터: 가상 메모리로 처리                                      │
│  • 공유 메모리: 프로세스 간 통신                                         │
│                                                                     │
│  【보안】                                                              │
│  • ASLR: 주소 공간 배치 난수화                                          │
│  • DEP/NX: 실행 방지                                                  │
│  • 메모리 격리: 프로세스 간 보호                                         │
│  • 권한 비트: R/W/X 제어                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  가상 메모리 없이 vs 있이:                                                        │ │
│  │  ──────────────────────                                              │ │
│  │  특성              가상 메모리 없이       가상 메모리 있이                │ │
│  │  ──────             ────────────         ────────────                    │ │
│  │  주소 공간          물리 메모리 크기       거의 무제한 (TB)                 │ │
│  │  프로세스 격리      어려움                 자동                           │ │
│  │  메모리 보호        어려움                 자동                           │ │
│  │  큰 프로그램        실행 불가              실행 가능                        │ │
│  │  공유 메모리        복잡                   쉬움                           │ │
│  │  오버헤드           없음                   있음                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 가상 메모리 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                가상 메모리 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【가상 메모리 정보 확인】                                                        │
│  ──────────────────                                                  │
│  $ cat /proc/meminfo                                               │
│  MemTotal:        16384000 kB    # 전체 물리 메모리                    │
│  MemFree:          8000000 kB    # 사용 가능 메모리                     │
│  SwapTotal:       16777216 kB    # 스왑 공간                          │
│  Virtual:        33554432 kB    # 가상 메모리 총계                     │
│                                                                     │
│  # 프로세스별 가상 메모리                                              │
│  $ cat /proc/[pid]/maps                                            │
│  $ pmap -x [pid]                                                   │
│                                                                     │
│  【페이지 크기 확인】                                                              │)
│  ──────────────────                                                  │
│  $ getconf PAGESIZE                                                │
│  4096    # 4KB                                                     │
│                                                                     │
│  # Huge Page 사용                                                   │
│  $ cat /proc/meminfo | grep Huge                                   │
│  HugePages_Total:    1024                                          │
│  HugePages_Free:      512                                          │
│  Hugepagesize:     2048 kB                                         │
│                                                                     │
│  【가상 메모리 통계】                                                              │)
│  ──────────────────                                                  │
│  $ vmstat 1                                                        │
│  procs -----------memory---------- ---swap-- -----io----            │
│   r  b   swpd   free   buff  cache   si   so    bi    bo             │
│   1  0  1024   8000    200   2000     0    0     5    10             │
│                                                                     │
│  # si: swap in (디스크→메모리)                                        │
│  # so: swap out (메모리→디스크)                                        │
│                                                                     │
│  【가상 메모리 튜닝】                                                              │)
│  ──────────────────                                                  │
│  # 스와핑 설정                                                        │
│  $ sysctl vm.swappiness                                            │
│  vm.swappiness = 60                                                │
│                                                                     │
│  # 페이지 캐시 설정                                                   │
│  $ sysctl vm.dirty_ratio                                           │
│  vm.dirty_ratio = 20                                               │
│                                                                     │
│  【가상 메모리 최적화 팁】                                                         │
│  ──────────────────                                                  │
│  1. 워킹셋 관리: 자주 쓰는 데이터는 메모리에 유지                            │
│  2. Huge Page: 대용량 데이터베이스                                     │
│  3. 스와핑 최소화: 충분한 RAM                                          │
│  4. TLB 미스 감소: 큰 페이지, 지역성                                     │
│  5. NUMA 고려: 멀티소켓 시스템                                         │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 가상 메모리: 물리 메모리보다 큰 주소 공간 제공
• 핵심 기술: MMU, 페이지, TLB, 스와핑
• 목적: 메모리 확장, 격리, 보호
• 주소 변환: 가상 주소 → 물리 주소 (MMU)
• 페이지: 고정 크기 블록 (보통 4KB)
• 스와핑: 디스크를 메모리처럼 사용
• 장점: 큰 주소 공간, 격리, 보호, 효율
• 단점: 오버헤드, Page Fault, 복잡성
• 현대: 모든 OS에서 기본 사용
• 튜닝: swappiness, Huge Page, TLB
```
