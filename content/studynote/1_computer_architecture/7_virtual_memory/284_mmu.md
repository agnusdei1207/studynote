# 284. MMU (Memory Management Unit)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 논리 주소를 물리 주소로 변환하는 하드웨어
> 2. **가치**: 가상 메모리 구현의 핵심
> 3. **융합**: TLB, 페이지 테이블, 보안, 캐시

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

MMU(Memory Management Unit)는 **논리 주소(가상 주소)를 물리 주소로 변환하는 전용 하드웨어**다. CPU와 메모리 사이에 위치하며, 가상 메모리 시스템의 핵심 구성 요소다.

### 💡 비유: 도서관 사서

MMU는 **도서관 사서**와 같다. 손님이 분류번호(논리 주소)를 말하면, 사서가 실제 책장 위치(물리 주소)로 변환한다. 사서는 위치록(페이지 테이블)을 참조하고, 자주 찾는 책은 기억해둔다(TLB).

## Ⅱ. MMU 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                MMU 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MMU 주소 변환 과정】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. CPU가 논리 주소 생성                                             │ │
│  │     예: 0x7FFF_1234                                                │ │
│  │                                                               │ │
│  │  2. MMU가 TLB 조회                                                   │ │
│  │     ┌─────────────────────────────────────────────────┐          │ │
│  │     │  VPN     │  PFN    │  권한  │  Valid  │          │          │ │
│  │     │  0x7FFF  │  0x1234 │  RW-   │    1    │ ← Hit!   │          │ │
│  │     └─────────────────────────────────────────────────┘          │ │
│  │                                                               │ │
│  │  3a. TLB Hit:                                                        │ │
│  │      • PFN 즉시 획득: 0x1234                                         │ │
│  │      • 물리 주소 생성: 0x1234_1234 (PFN + Offset)                     │ │
│  │      • 1~2 클럭 소요                                                 │ │
│  │                                                               │ │
│  │  3b. TLB Miss:                                                        │ │
│  │      • Page Table Walk 시작                                          │ │
│  │      • 메모리에서 페이지 테이블 조회                                   │ │
│  │      • PFN 획득 후 TLB 갱신                                           │ │
│  │      • 10~100 클럭 소요                                              │ │
│  │                                                               │ │
│  │  4. 권한 확인                                                         │ │
│  │     • 쓰기 요청인데 R-only? → Page Fault                              │ │
│  │     • 사용자 모드인데 커널 페이지? → Page Fault                        │ │
│  │                                                               │ │
│  │  5. 물리 주소로 메모리 접근                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MMU 예외 처리】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예외 종류            원인                    처리                    │ │
│  │  ──────────            ────                    ────                    │ │
│  │  Page Fault           페이지가 메모리에 없음    OS가 디스크에서 로드     │ │
│  │  Protection Fault     권한 위반                OS가 프로세스 종료       │ │
│  │  TLB Miss             TLB에 변환 정보 없음     Page Table Walk        │ │
│  │  Access Fault         잘못된 주소              OS가 시그널 전송         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MMU 레지스터】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  레지스터             설명                    설정자                   │ │
│  │  ──────────            ────                    ────                     │ │
│  │  CR3 (x86)            페이지 테이블 베이스        OS                      │ │
│  │  TTBR0/TTBR1 (ARM)    페이지 테이블 베이스        OS                      │ │
│  │  SATP (RISC-V)        페이지 테이블 베이스        OS                      │ │
│  │  Page Table Root      현재 페이지 테이블         OS                      │ │
│  │                                                               │ │
│  │  → 컨텍스트 스위치 시 OS가 이 레지스터 변경                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. MMU × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                MMU × 타 과목 융합                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • CPU-Memory 인터페이스: MMU가 중간 위치                              │
│  • 파이프라인: MMU 지연이 CPI 영향                                     │
│  • 캐시: Virtually Indexed vs Physically Indexed                      │
│  • TLB: MMU 내부의 작은 캐시                                           │
│                                                                     │
│  【운영체제】                                                          │
│  • 페이지 테이블 관리: MMU가 참조할 테이블 유지                           │
│  • 컨텍스트 스위치: CR3/TTBR 레지스터 변경                               │
│  • 페이지 폴트 처리: MMU가 감지하면 OS가 처리                             │
│  • 메모리 보호: MMU의 권한 비트 활용                                     │
│                                                                     │
│  【보안】                                                              │
│  • ASLR: MMU로 주소 무작위화 구현                                       │
│  • NX/DEP: 실행 권한 비트로 방어                                        │
│  • SMEP/SMAP: 커널/사용자 공간 격리                                     │
│  • 메모리 격리: MMU가 각 프로세스 보호                                   │
│                                                                     │
│  【성능】                                                              │
│  • TLB 미스: 성능 큰 저하                                              │
│  • Huge Page: TLB 커버리지 증가                                        │
│  • Page Walk: 다단계 테이블 비용                                        │
│  • 캐시 일관성: VIPT vs PIPT                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  MMU 없이 vs 있이:                                                               │ │
│  │  ──────────────────────                                              │ │
│  │  특성               MMU 없이               MMU 있이                   │ │
│  │  ──────              ────────               ────────                    │ │
│  │  가상 메모리         불가능                  가능                        │ │
│  │  메모리 보호         소프트웨어만             하드웨어 지원                │ │
│  │  프로세스 격리       어려움                  자동                        │ │
│  │  주소 변환           소프트웨어               하드웨어                    │ │
│  │  오버헤드            변환 없음                있음 (TLB로 최소화)          │ │
│  │  복잡도              단순                    복잡                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### MMU 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                MMU 분석                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLB 정보 확인】                                                                │)
│  ──────────────────                                                  │
│  $ cat /proc/cpuinfo | grep -i tlb                                  │
│  TLB size: 64 4KB pages                                             │
│                                                                     │
│  # perf로 TLB 미스 측정                                               │
│  $ perf stat -e dTLB-load-misses,dTLB-loads ./prog                  │
│  dTLB-loads: 100,000,000                                            │
│  dTLB-load-misses: 500,000                                          │
│  → TLB 미스율: 0.5%                                                   │
│                                                                     │
│  【페이지 테이블 설정 확인】                                                        │)
│  ──────────────────                                                  │
│  $ cat /proc/cpuinfo | grep -i "address sizes"                      │
│  address sizes: 39 bits physical, 48 bits virtual                   │
│  → 48비트 가상 주소, 39비트 물리 주소                                   │
│                                                                     │
│  【Huge Page 사용】                                                                 │)
│  ──────────────────                                                  │
│  # TLB 커버리지 증가로 성능 향상                                        │
│  $ cat /proc/meminfo | grep Huge                                    │
│  HugePages_Total:    1024                                           │
│  Hugepagesize:     2048 kB                                          │
│                                                                     │
│  【MMU 성능 분석】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시나리오             TLB Hit    TLB Miss    비율                    │ │
│  │  ──────────            ──────     ──────     ────                     │ │
│  │  작은 워킹셋           99%        1%         TLB가 커버                  │ │
│  │  큰 워킹셋             80%        20%        TLB 부족                   │ │
│  │  랜덤 접근             50%        50%        TLB 효과 낮음               │ │
│  │  Huge Page            99%        1%         TLB 효율 높음                │ │
│  │                                                               │ │
│  │  → TLB 미스가 성능에 큰 영향                                           │ │
│  │  → Huge Page로 TLB 효율 향상 가능                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MMU 최적화 팁】                                                                 │)
│  ──────────────────                                                  │
│  1. Huge Page: TLB 커버리지 증가                                        │
│  2. 워킹셋 관리: TLB 크기에 맞게                                         │
│  3. 지역성 향상: TLB 히트률 증가                                         │
│  4. 페이지 크기: 워크로드에 맞게 선택                                     │
│  5. TLB 플러시 최소화: 컨텍스트 스위치 시                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• MMU: 논리 주소를 물리 주소로 변환하는 하드웨어
• 위치: CPU와 메모리 사이
• 기능: 주소 변환, 메모리 보호, 캐시 제어
• TLB: MMU 내부 변환 캐시
• Page Table Walk: TLB 미스 시 메모리 조회
• 레지스터: CR3(x86), TTBR(ARM), SATP(RISC-V)
• 예외: Page Fault, Protection Fault
• 지연: TLB Hit 1~2클럭, TLB Miss 10~100클럭
• 최적화: Huge Page, 지역성, 워킹셋 관리
• 보안: NX, SMEP, SMAP 구현에 필수
```
