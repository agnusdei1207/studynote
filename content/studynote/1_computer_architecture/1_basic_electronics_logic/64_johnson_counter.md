# 존슨 카운터 (Johnson Counter)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 존슨 카운터(Johnson Counter)는 마지막 단 플립플롭의 **반전 출력($Q'$)**을 첫 번째 단의 입력($D$)으로 피드백하여, $n$개의 플립플롭으로 $2n$개의 상태를 생성하는 변형 링 카운터다.
> 2. **가치**: 일반 링 카운터보다 2배 많은 상태를 표현할 수 있어 회로 면적 효율성이 높으며, 인접한 상태 간에 오직 한 비트만 변하는 특성으로 인해 동작 중 발생하는 **글리치(Glitch) 현상을 억제**한다.
> 3. **융합**: '트위스티드 링 카운터(Twisted Ring Counter)'라고도 불리며, 위상 천이(Phase Shifting)가 필요한 통신 모듈이나 정교한 디지털 파형 생성기의 핵심 로직으로 활용된다.

---

## Ⅰ. 존슨 카운터 (Johnson Counter)의 개요

### 1. 정의 및 명칭
- **정의**: 마지막 플립플롭의 상보 출력(Complementary Output)을 첫 단으로 되돌려, '1'의 꼬리가 길게 늘어났다가 다시 '0'의 꼬리가 길게 늘어나는 방식으로 순환하는 카운터다.
- **별칭**: Twisted Ring Counter, Moebius Counter (뫼비우스의 띠처럼 뒤집혀서 연결됨).

### 2. 💡 비유: 뫼비우스의 기차
- 존슨 카운터는 **"끝이 뒤집혀 연결된 기차 레일"**과 같다.
- 기차가 한 바퀴 돌고 오면 기차의 색깔이 검은색에서 흰색으로 반전되어 돌아온다. 다시 한 바퀴를 더 돌아야 원래의 검은색 기차로 돌아오는 것과 같다.

---

## Ⅱ. 아키텍처 및 동작 원리 (Deep Dive)

### 1. 4비트 존슨 카운터 회로 구조 (ASCII)
D 플립플롭을 사용한 구조 예시다.

```text
       CLK ────────────────┬───────────────┬───────────────┬───────────────┐
                           │               │               │               │
                    ┌──────▼──────┐ ┌──────▼──────┐ ┌──────▼──────┐ ┌──────▼──────┐
             ┌─────▶│      D      │ │      D      │ │      D      │ │      D      │
             │      │    FF0      │ │    FF1      │ │    FF2      │ │    FF3      │
             │      │  (Q0)(Q0')  │ │  (Q1)(Q1')  │ │  (Q2)(Q2')  │ │  (Q3)(Q3')  │
             │      └──────┬──────┘ └──────┬──────┘ └──────┬──────┘ └──────┬──────┘
             │             └───────────────┤               │               │
             │                             └───────────────┤               │
             │                                             └───────────────┤
             └─────────────────────────────────────────────────────────────┘
               (FF3의 Q3' 출력을 FF0의 D입력으로 Twisted Feedback)
```

**[다이어그램 해설]**
- **피드백 루프**: 링 카운터는 $Q_3 \to D_0$로 연결되지만, 존슨 카운터는 **$Q_3' \to D_0$**로 뒤집어서 연결한다.
- **상태 전이**: 초기값이 $0000$일 때, 다음 클럭에서 $Q_3'$인 $1$이 $D_0$로 들어가 $1000$이 된다. 이 과정이 반복되며 $1100 \to 1110 \to 1111 \to 0111 \dots$ 순으로 비트가 '채워지고 비워지는' 형태를 띤다.

### 2. 상태표 (4-bit Johnson Counter)
| 클럭 (CLK) | Q0 | Q1 | Q2 | Q3 | 설명 |
|:---:|:---:|:---:|:---:|:---:|:---|
| 0 | 0 | 0 | 0 | 0 | 초기 상태 |
| 1 | **1** | 0 | 0 | 0 | '1'이 채워지기 시작 |
| 2 | 1 | **1** | 0 | 0 | |
| 3 | 1 | 1 | **1** | 0 | |
| 4 | 1 | 1 | 1 | **1** | '1'로 가득 참 |
| 5 | **0** | 1 | 1 | 1 | '0'이 채워지기 시작 ($Q_3'$가 0이므로) |
| 6 | 0 | **0** | 1 | 1 | |
| 7 | 0 | 0 | **0** | 1 | |
| 8 | 0 | 0 | 0 | **0** | 다시 초기 상태로 복귀 |

## Ⅲ. 링 카운터 vs 존슨 카운터 비교

| 비교 항목 | 링 카운터 (Ring) | 존슨 카운터 (Johnson) |
|:---|:---|:---|
| **피드백 경로** | $Q_n \to D_0$ (Direct) | **$Q_n' \to D_0$ (Twisted)** |
| **상태 수** | $n$개 | **$2n$개** |
| **디코딩** | 불필요 (One-hot) | 2입력 AND 게이트 필요 |
| **장점** | 구조적 직관성 | 자원 효율성, 저노이즈 |
| **단점** | 하드웨어 낭비 ($2^n$ 대비) | 비정상 상태 진입 시 복구 회로 필요 |

- **📢 섹션 요약 비유**: 똑같은 개수의 의자(플립플롭)를 가지고, 한 명씩만 앉는 것이 링 카운터라면, 앞사람부터 차례로 앉아 가득 채웠다가 다시 차례로 일어나는 것이 존슨 카운터입니다.

---

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy)

### 1. 고속 디지털 위상 제어
- **기술사적 인사이트**: 존슨 카운터의 각 출력 신호($Q_0, Q_1 \dots$)는 일정한 시간 간격으로 위상이 밀려있는 구형파를 생성한다. 
- **실무 활용**: 클럭의 위상을 $45^\circ, 90^\circ$ 단위로 쪼개어 사용하는 통신용 주파수 합성기(Frequency Synthesizer) 설계 시 존슨 카운터는 가장 저전력으로 이를 구현하는 표준적인 방법이다.

### 2. 비정상 상태(Illegal State) 대응
- **판단**: 존슨 카운터는 $2^n$개의 가능한 상태 중 $2n$개만 사용하므로, 전원 투입 시나 노이즈로 인해 나머지 $2^n - 2n$개의 상태로 빠질 위험이 있다.
- **결단**: 실무 설계 시에는 비정상 상태 감지 시 자동으로 초기 상태($00\dots0$)로 리셋하는 'Self-Starting' 로직을 반드시 포함해야 한다.

---

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **면적 최적화**: 링 카운터 대비 절반의 플립플롭만으로 동일한 상태 수 구현 가능.
- **동적 전력 절감**: 매 클럭마다 오직 하나의 플립플롭만 상태가 변하므로 스위칭 노이즈와 전력 소모가 적음.

### 2. 미래 전망
나노 공정으로 갈수록 타이밍 스큐와 전력 소모가 칩 설계의 성패를 좌우한다. 존슨 카운터는 그 구조적 안정성 덕분에 최신 저전력 모바일 AP나 고속 인터페이스 회로 내부에서 여전히 '작지만 강력한' 역할을 수행하고 있다. 기본 원리에 충실하면서도 효율을 극대화한 아키텍처의 전형이라 할 수 있다.

- **📢 섹션 요약 비유**: 불필요한 장식을 버리고 실용성을 챙긴 미니멀리즘 인테리어와 같은 영리한 회로 설계입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. 존슨 카운터는 친구들이 손을 잡고 서서 **'흰 장갑'을 한 명씩 차례로 나눠 끼는 게임**과 같아요.
2. 모두가 장갑을 다 끼면, 이번에는 맨 앞 친구부터 **한 명씩 장갑을 벗어서 옆 사람에게** 주죠.
3. 링 카운터보다 훨씬 많은 순서를 셀 수 있어서, 컴퓨터가 복잡한 일을 할 때 아주 효율적인 비서가 되어준답니다!
