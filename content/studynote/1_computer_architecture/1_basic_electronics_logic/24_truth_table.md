> **핵심 인사이트**
> 1. 진리표(Truth Table)는 디지털 논리 회로에서 가질 수 있는 **모든 입력 조건의 조합과 그에 따른 출력 결과를 0과 1로 나열한 표**이다.
> 2. 컴퓨터 아키텍처 설계의 첫 단추로, 모호한 인간의 요구사항(기능 명세)을 하드웨어가 이해할 수 있는 **명확한 수학적 모델(부울 함수)로 매핑(Mapping)**하는 역할을 한다.
> 3. 진리표를 작성한 후, 최소항(Minterm)들을 뽑아내어 카르노 맵(Karnaugh Map)으로 넘겨 최적화하는 것이 반도체 설계의 기본 워크플로우이다.

---

## Ⅰ. 진리표의 구조와 작성법
진리표는 크게 입력을 적는 좌측과 출력을 적는 우측으로 나뉩니다. 입력 변수의 개수가 $N$개라면, 발생할 수 있는 모든 경우의 수는 $2^N$개가 됩니다. 입력값은 보통 2진수 카운트 순서(000, 001, 010, ... )대로 빠짐없이 기입하여 누락을 방지합니다.

예를 들어, "두 개의 스위치(A, B)가 다를 때만 전구(F)가 켜진다"는 요구사항(XOR 게이트)을 진리표로 만들면 다음과 같습니다.

```ascii
[XOR (배타적 논리합) 진리표 예시]

 입력 변수 (N=2, 경우의 수 4개) | 출력 (F)
 -----------------------------+---------
   A (스위치1)   B (스위치2)    |  F (전구)
      0             0         |    0   (둘 다 같음 -> 꺼짐)
      0             1         |    1   (다름 -> 켜짐)
      1             0         |    1   (다름 -> 켜짐)
      1             1         |    0   (둘 다 같음 -> 꺼짐)
```

📢 **섹션 요약 비유**: 진리표는 식당의 '경우의 수 매뉴얼'입니다. 손님이 '짜장면(0)시킬 때, 짬뽕(1)시킬 때', '현금(0)낼 때, 카드(1)낼 때' 발생할 수 있는 모든 4가지 상황을 미리 표로 적어두고 직원이 절대 당황하지 않게(오류 없이 작동하게) 만드는 것입니다.

## Ⅱ. 진리표에서 논리식 도출하기 (SOP, Sum of Products)
진리표를 그리는 궁극적인 목적은 이 표를 보고 **부울 대수식(Boolean Expression)**을 뽑아내기 위함입니다. 가장 대표적인 방법은 출력이 '1(True)'인 줄(Row)만 골라내어 식을 만드는 **곱의 합(SOP, Sum of Products)** 방식입니다.

위의 XOR 진리표에서 출력이 1인 경우는 두 가지입니다.
1. A=0, B=1 일 때 -> 논리곱(AND)으로 묶으면: $A' \cdot B$
2. A=1, B=0 일 때 -> 논리곱(AND)으로 묶으면: $A \cdot B'$

이 두 경우를 논리합(OR)으로 더해주면 완벽한 논리식이 탄생합니다.
**최종 논리식:** $F = (A' \cdot B) + (A \cdot B')$

```ascii
[진리표 -> 논리식 (SOP) 도출 프로세스]

 A  B | F
 -----+---
 0  0 | 0
 0  1 | 1   ----->  A'B   (최소항 1)
 1  0 | 1   ----->  AB'   (최소항 2)
 1  1 | 0
        |
        +-- (OR 결합) -->  F = A'B + AB'
```

📢 **섹션 요약 비유**: 진리표에서 식을 뽑아내는 SOP 방식은 '합격자 명단만 부르기'입니다. 시험에서 불합격한(0인) 사람들은 무시하고, 합격한(1인) 사람들의 이름(A'B, AB')만 쏙쏙 뽑아내어 "합격자는 얘 **또는(+)** 얘야"라고 발표하는 것과 같습니다.

## Ⅲ. 아키텍처 관점: 진리표의 폭발과 설계의 한계
설계해야 할 시스템이 덧셈기(Adder)처럼 간단하면 입력 변수가 2~3개라 진리표가 4~8줄로 끝납니다. 하지만 최신 CPU의 명령어 해독기(Instruction Decoder)나 ALU는 입력 핀(변수)이 32개, 64개에 달합니다. 

입력 변수가 32개($N=32$)라면 진리표의 줄 수는 $2^{32}$개, 즉 **약 43억 줄**이 됩니다. 인간이 손으로 진리표를 그리는 것은 불가능해집니다. 따라서 현대 반도체 아키텍처 설계에서는 진리표를 직접 그리지 않고, **하드웨어 기술 언어(HDL: Verilog, VHDL)**를 사용하여 "A가 B보다 크면 F는 1이다"라고 코드를 짜면, EDA(전자 설계 자동화) 소프트웨어가 내부적으로 수십억 줄의 진리표를 계산하고 최적화하여 칩 회로도를 자동으로 생성(Synthesis)해 냅니다.

```ascii
[진리표 규모의 지수적 폭발]

 입력(N) | 진리표 행(Row) 개수 (2^N)
   2     | 4
   4     | 16  (손으로 계산 가능: 카르노 맵 활용)
   8     | 256 (손 계산 한계)
  32     | 약 4,294,967,296 줄 (HDL 프로그래밍 및 컴퓨터 EDA 툴 필수!)
```

📢 **섹션 요약 비유**: 진리표 작성은 '구구단 외우기'입니다. 1단부터 9단(입력 3~4개)까지는 표를 그려서 직접 머릿속에 넣을 수 있지만, 40억 단(32비트 입력)을 외우는 건 불가능하므로 반드시 슈퍼컴퓨터(EDA 도구)라는 계산기의 도움을 받아야만 합니다.

## 개념 맵 (Concept Map)
```ascii
[진리표 (Truth Table)]
   │
   ├── 기본 정의: N개 입력에 대한 2^N개 조합과 출력(0/1)을 명시한 표
   │
   ├── 논리 설계 워크플로우
   │     └── 요구사항 -> 진리표 작성 -> 출력 1(SOP) 추출 -> 논리식(부울식) 도출
   │
   └── 현대 아키텍처 설계의 한계
         └── 입력 변수 증가에 따른 조합의 지수적 폭발 (State Explosion)
               └── 해결: HDL(Verilog) 코딩과 EDA 툴을 통한 자동 합성(Synthesis)
```

## 어린이 비유
진리표는 피자 가게의 **절대 틀리지 않는 마법의 주문 자판기**예요! 손님이 낼 수 있는 모든 주문 버튼 조합(버튼 A, 버튼 B)을 표에 다 적어 놓고, 각각의 경우에 "콜라를 준다(1)" 아니면 "안 준다(0)"를 미리 다 정해놓은 거예요. 컴퓨터는 생각할 필요 없이 이 표만 쓱 보고 정해진 답을 0.001초 만에 딱 내어놓는 거랍니다!
