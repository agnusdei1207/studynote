> **핵심 인사이트**
> 1. JK 플립플롭은 SR 플립플롭의 치명적 약점인 S=1, R=1 동시 입력 시의 붕괴 현상을 극복하기 위해, **토글(Toggle, 상태 뒤집기)** 기능을 추가한 완전체 플립플롭이다.
> 2. J=1, K=1이 들어오면 에러가 나는 대신, 내부 피드백 루프를 통해 **현재 자신의 출력 상태(Q)를 정반대($Q'$)로 휙 뒤집어버린다.**
> 3. 이 '토글' 기능 덕분에 클럭이 칠 때마다 0, 1, 0, 1로 변하는 특성을 이용해, 시스템의 숫자를 세는 **카운터(Counter)** 회로나 클럭 주파수를 절반으로 쪼개는 **분주기(Divider)** 회로의 핵심 부품으로 사용된다.

---

## Ⅰ. SR의 한계 극복과 완벽한 진리표
JK 플립플롭(발명자 Jack Kilby의 이니셜이라는 썰도 있음)은 겉보기엔 SR 플립플롭과 똑같습니다. J 핀은 Set(1) 역할을 하고, K 핀은 Reset(0) 역할을 합니다.
진짜 차이는 J=1, K=1이 들어왔을 때 일어납니다. SR 플립플롭은 모순이 생겨서 죽어버렸지만, JK 플립플롭은 출력을 입력으로 다시 꽂아버리는 아주 기발한 추가 피드백 전선을 달아서 이 상태를 **'토글(Toggle, 뒤집기)'** 모드로 창조해 냈습니다.

* **J=1, K=0**: Set (무조건 1)
* **J=0, K=1**: Reset (무조건 0)
* **J=0, K=0**: Hold (현재 값 기억)
* **J=1, K=1**: Toggle (현재가 1이면 0으로, 0이면 1로 뒤집힘!)

```ascii
[JK 플립플롭의 진리표 (상승 에지 기준)]

 CLK | J | K || Q(다음 상태) | 설명
 ----+---+---++--------------+-------------------------
  ↑  | 0 | 0 || 이전 값 (Q)  | Hold (기억 유지)
  ↑  | 0 | 1 ||      0       | Reset (0으로 초기화)
  ↑  | 1 | 0 ||      1       | Set (1로 설정)
  ↑  | 1 | 1 || 이전의 반대(Q')| Toggle (상태 뒤집기! JK의 핵심)
```

📢 **섹션 요약 비유**: JK 플립플롭은 '고장 나지 않는 양방향 스위치'입니다. 위로 누르면(J) 불이 켜지고, 아래로 누르면(K) 꺼집니다. 실수로 양쪽을 동시에 팍! 누르면 고장 나는 대신, 찰칵! 하고 지금 불 켜진 상태를 완전히 반대(켜졌으면 끄고, 꺼졌으면 켬)로 바꿔버리는 스마트한 스위치입니다.

## Ⅱ. 토글(Toggle)의 마법과 타이밍 다이어그램
J와 K에 항상 1을 고정해 두고(계속 Toggle 모드) 클럭(CLK)을 줘봅시다.
클럭이 상승 에지($\uparrow$)를 칠 때마다 Q는 0에서 1로, 다음 클럭에서는 1에서 0으로 계속해서 춤을 춥니다. 

재미있는 현상이 발생합니다. 입력되는 클럭 파형의 주파수에 비해, **출력 Q의 파형은 주파수가 정확히 절반(1/2)으로 줄어들고 주기가 2배로 길어집니다.** 클럭이 두 번 쳐야만 Q가 한 바퀴(0->1->0)를 완성하기 때문입니다. 이 성질이 시스템 타이밍 설계에서 엄청난 역할을 합니다.

```ascii
[Toggle 모드 (J=1, K=1)에서의 주파수 분주 현상]

 클럭(CLK): ___|~|___|~|___|~|___|~|___|~|___ (빠름: 예 100MHz)
              (1)   (2)   (3)   (4)   (5)
 ---------------------------------------------
 출력(Q)  : ___|~~~~~~~|___|~~~~~~~|___|~~~~~ (느림: 예 50MHz, 2분주)
              (1에 1) (2에 0) (3에 1) (4에 0)
```

📢 **섹션 요약 비유**: 토글 분주는 '행군할 때 발맞추기'입니다. 조교가 박수를 "짝(왼), 짝(오), 짝(왼), 짝(오)" 하고 네 번 치면(클럭), 병사는 "왼발~, 오른발~" 하고 크게 두 걸음(출력 Q)만 걷게 됩니다. 즉, 조교의 박수 속도보다 내 발걸음 템포가 정확히 절반으로 느려지는 원리입니다.

## Ⅲ. 아키텍처 관점: 카운터(Counter) 회로의 심장
CPU 내부의 프로그램 카운터(PC)나 디지털시계, 타이머 등은 모두 0, 1, 2, 3... 숫자를 세는 **카운터(Counter)** 회로로 만들어집니다.
아키텍처 설계자들은 JK 플립플롭(특히 토글 모드)을 여러 개 사슬처럼 엮어서 이 카운터를 만듭니다.

첫 번째 JK의 출력을 두 번째 JK의 클럭(CLK) 핀에 꽂고, 두 번째의 출력을 세 번째의 클럭에 꽂으면 (비동기식 리플 카운터),
* 1번 플립플롭은 주파수가 $1/2$이 되고 (일의 자리: 0, 1, 0, 1)
* 2번 플립플롭은 주파수가 $1/4$이 되고 (이의 자리: 0, 0, 1, 1)
* 3번 플립플롭은 주파수가 $1/8$이 됩니다 (사의 자리: 0, 0, 0, 0, 1, 1, 1, 1)
이 출력들을 나란히 읽으면 이진수로 `000, 001, 010, 011...` 즉 $0$부터 $7$까지 완벽하게 숫자를 세는 놀라운 회로가 완성됩니다!

```ascii
[3비트 리플 카운터(Ripple Counter) 구조]

 항상 1(Toggle) ---> J0          J1          J2
                     |   (Q0를)  |   (Q1을)  |
 클럭(빠름) -------> CLK [JK 0] CLK [JK 1] CLK [JK 2]
                     |           |           |
                     Q0 (비트0)   Q1 (비트1)   Q2 (비트2)  => [Q2 Q1 Q0] = 이진수 카운트!
```

📢 **섹션 요약 비유**: 카운터 설계는 '도미노 혹은 물레방아 연속 돌리기'입니다. 첫 번째 물레방아가 두 바퀴를 돌아야 두 번째 톱니를 한 칸 밀고, 두 번째가 두 바퀴 돌아야 세 번째 톱니를 미는 식으로 연결해 놓으면, 복잡한 계산기 없이도 톱니바퀴의 위치만 보고 몇 바퀴 돌았는지(몇 분 지났는지) 정확히 숫자를 셀 수 있습니다.

## 개념 맵 (Concept Map)
```ascii
[JK 플립플롭 (JK Flip-Flop)]
   │
   ├── 존재 의의: SR 플립플롭의 S=1,R=1 금지 상태 한계 극복
   │
   ├── 특수 기능: 토글 (Toggle)
   │     └── J=1, K=1 일 때, 매 클럭 에지마다 현재 상태(Q)를 반전시킴(Q')
   │
   └── 아키텍처 응용
         ├── 주파수 분주기(Frequency Divider): 클럭 주파수를 1/2, 1/4 로 쪼갬
         └── 카운터(Counter) 회로 구축: 프로그램 카운터(PC), 타이머, 시계 회로의 기본 단위
```

## 어린이 비유
JK 플립플롭은 **절대 헷갈리지 않는 동전 뒤집기 기계**예요! 위로 올리라는 버튼(J)과 아래로 내리라는 버튼(K)이 있는데, 장난꾸러기가 두 개를 동시에 팍 누르면 이 기계는 고장 나는 대신 "어? 둘 다 눌렀어? 그럼 내가 지금 앞면(1)이니까 뒷면(0)으로 휙 뒤집을게!" 하고 멋지게 토글(뒤집기)을 한답니다. 이 똑똑한 기계들을 줄줄이 세워놓으면 시계처럼 1, 2, 3, 4 숫자를 척척 셀 수 있어요!
