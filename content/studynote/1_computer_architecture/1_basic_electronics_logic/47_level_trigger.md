> **핵심 인사이트**
> 1. 레벨 트리거(Level Trigger)는 클럭 신호가 특정 전압 수준(High 또는 Low)을 **유지하는 전체 시간 구간(Duration)** 동안 입력의 변화를 계속해서 출력으로 내보내는 타이밍 제어 방식이다.
> 2. 상태가 변하는 찰나(Edge)에만 반응하는 플립플롭과 달리, 래치(Latch)라는 메모리 소자를 구동하는 기본 방식이다.
> 3. 신호가 열려 있는 동안 데이터가 무한 반복되어 시스템이 폭주하는 '레이스 컨디션(Race Condition)'의 위험 때문에 현대 CPU 파이프라인의 메인 레지스터로는 쓰이지 않으나, 아주 좁고 특정된 타이밍 제어에는 여전히 쓰인다.

---

## Ⅰ. 레벨 트리거의 넓은 '개방 시간'
레벨 트리거는 클럭이나 제어 신호(Enable)가 1(High) 상태를 유지하는 **넓은 시간 구간 전체**에 걸쳐서 반응합니다. (이를 Positive Level Trigger라 합니다. 반대는 Negative).
이 말은 문(Gate)이 0.001초 만에 닫히는 게 아니라, 클럭이 1인 2나노초 동안 문이 계속 활짝 열려 있다는 뜻입니다. 

문이 열려 있는 시간 동안 만약 외부에서 입력 데이터가 0에서 1로, 다시 1에서 0으로 막 바뀌면, 그 바뀌는 족족 실시간으로 출력도 똑같이 춤을 추며 따라 변하게 됩니다. 즉, 열려 있는 동안은 조합 논리회로처럼 투명하게 통과시키는 성질을 가집니다.

```ascii
[클럭 파형에서의 레벨(Level) 구간]

      (Positive Level 구간: 이 시간 내내 문이 열려 있음!)
    +==================+
    |                  |
    |                  |
 ---+                  +------- (Negative Level: 문이 닫히고 현재 값 저장/유지)
  에지(찰나)
```

📢 **섹션 요약 비유**: 레벨 트리거는 '수동식 카메라의 장노출(벌브 모드)'입니다. 셔터를 누르고 있는 10초 내내 렌즈가 열려 있어서, 그사이에 별이 움직이면 움직인 궤적이 사진에 선으로 쭈욱 다 찍혀버립니다. 짧은 순간을 포착하는 에지 트리거(순간 촬영)와 완전히 다릅니다.

---

## Ⅱ. 치명적 버그: 레이스 컨디션 (Race Condition)
순차 회로(끝말잇기)에서는 출력이 다시 입력으로 되돌아오는 피드백 루프가 있습니다. 
만약 레벨 트리거로 된 래치(Latch)를 메인 레지스터로 쓴다고 상상해 봅시다.
클럭이 1(High)이 되어 문이 열립니다. 입력 A가 들어와서 덧셈기를 거쳐 출력 C가 됩니다. 그런데 이 출력 C가 전선을 타고 0.1나노초 만에 다시 입력으로 빙 돌아옵니다.
아뿔싸, 아직 클럭이 1인 시간(예: 1나노초 유지)이 끝나지 않아서 문이 닫히지 않았습니다! 
돌아온 C는 다시 회로로 들어가서 D를 만들고, 또 돌아와서 E를 만들며, 클럭 문이 닫힐 때까지 이 미친 폭주를 반복합니다. 결국 클럭 문이 닫혔을 때 레지스터에 저장된 값은 원래 원했던 C가 아니라 엉뚱한 쓰레기 값이 됩니다.

이것을 **레이스 컨디션(경쟁 상태)**이라고 하며, 초기 컴퓨터 설계자들의 가장 큰 골칫거리였습니다. 이 문제를 완벽히 해결하기 위해 등장한 구세주가 바로 '에지 트리거(플립플롭)'인 것입니다.

```ascii
[레벨 트리거 회로의 폭주 과정]

 클럭(High) 유지 시간: 10초.  전기가 루프 한 바퀴 도는 시간: 1초.
 [ 문 열림! ]
 1초: 입력 A -> 연산 -> 결과 C (정상) -> C가 피드백으로 다시 돌아옴!
 2초: 결과 C -> 연산 -> 결과 D (원치않는 쓰레기값 발생)
 ...
 10초: 결과 엉망진창
 [ 문 닫힘. 엉망진창 값 저장됨 ㅠ ]
```

📢 **섹션 요약 비유**: 레벨 트리거 피드백은 '마이크와 스피커 삐익 소리(하울링)'와 같습니다. 마이크(입력)가 열려있는 동안, 스피커(출력)로 나간 소리가 다시 마이크로 들어가고, 그게 증폭되어 또 스피커로 나오는 폭주가 순식간에 일어나 귀를 찢는 굉음(에러)을 만들어냅니다.

## Ⅲ. 아키텍처 관점: 그런데도 레벨 트리거(래치)를 쓰는 이유
치명적 단점에도 불구하고 레벨 트리거 방식의 래치(Latch)는 플립플롭보다 **크기가 절반 수준으로 작고 게이트 수가 적어 칩 면적을 크게 아낄 수 있다**는 장점이 있습니다.

현대의 초고성능 CPU에서는 래치를 플립플롭처럼 안전하게 쓰기 위해, 서로 다른 위상을 가진 두 개의 클럭 신호를 번갈아 가며 쏘아주는 **2상 클럭(Two-phase Clocking)** 등의 고도의 설계 기법을 동원하여, 피드백 폭주를 막으면서도 면적은 아끼는 래치 기반의 파이프라인(Latch-based Design)을 채택하기도 합니다.

```ascii
[2상 클럭킹(Two-phase Clocking)을 통한 래치 제어]

 클럭 1 (Phase 1) : 래치 A만 문을 엶 (래치 B는 꽉 닫아둠. 폭주 차단!)
    +---+         +---+
    |   |         |   |
 ---+   +---------+   +---

 클럭 2 (Phase 2) : 래치 B만 문을 엶 (래치 A는 닫음. 안전한 전달!)
          +---+         +---+
          |   |         |   |
 ---------+   +---------+   +---
```

📢 **섹션 요약 비유**: 2상 클럭 래치는 우주선의 '에어록(Air-lock)' 시스템입니다. 바깥 문(래치 A)과 안쪽 문(래치 B)을 절대 동시에 열지(레벨 중첩) 않게 통제합니다. 바깥 문이 열릴 때 우주인이 중간방에 들어오고 문이 꽉 닫힌 뒤에야, 안쪽 문을 열어 우주선 안으로 들여보내 공기(데이터)가 우주로 폭주해 빠져나가는 것을 막습니다.

```ascii

   │
   ├── 정의: 클럭이 1(High) 또는 0(Low)을 유지하는 전체 구간 동안 입력을 출력으로 전송
   │
   ├── 치명적 문제: 레이스 컨디션 (Race Condition)

   │     └── (현대 CPU 메인 레지스터는 이를 피하기 위해 에지 트리거 플립플롭 사용)
   │
   └── 아키텍처 적용
         └── 장점: 회로가 가장 단순하고 면적이 작음 (래치 구현)
         └── 솔루션: 2상 클럭(Two-phase Clocking)과 결합하여 면적 최적화 설계(Latch-based Design)에 사용
```

## 어린이 비유
레벨 트리거는 문이 활짝 열려있는 **동네 슈퍼마켓 문**이에요! 슈퍼 아저씨가 "문 엽니다!(High)" 하고 10분 동안 문을 열어놓으면, 그 10분 동안 강아지도 들어왔다 나가고, 고양이도 들어왔다 나가며 슈퍼 안이 정신없어질 수 있어요(레이스 컨디션). 하지만 문지기(플립플롭)를 따로 두는 것보다 그냥 문만 덜렁 열어두는 게 돈(칩 크기)이 제일 적게 들기 때문에, 똑똑한 방법으로 통제해서 가끔씩 쓰기도 한답니다.
