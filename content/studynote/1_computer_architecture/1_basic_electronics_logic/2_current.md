# 전류 (Current)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전류 (Current)는 단위 시간당 도체의 단면을 통과하는 전하의 흐름 ($I = dQ/dt$)으로, 디지털 회로에서 커패시터를 충·방전시켜 상태 전이 (State Transition)를 물리적으로 완성하는 에너지 운반체다.
> 2. **가치**: 현대 초미세 공정 반도체에서 전류 관리는 스위칭 시 소모되는 동적 전류 (Dynamic Current)의 최적화를 넘어, 수 나노미터(nm)의 얇은 산화막을 뚫고 새어나가는 정적 누설 전류 (Static Leakage Current)를 차단하는 칩 면적과 배터리 수명의 핵심 결정 요인이다.
> 3. **융합**: 집적도 향상으로 인한 국부적 고전류 밀도는 전자가 금속 원자를 타격해 배선을 파괴하는 일렉트로마이그레이션 (Electromigration)을 유발하며, 이는 성능 튜닝 (오버클럭), 쿨링 솔루션, 하드웨어 신뢰성 (MTBF) 전체를 관통하는 시스템 엔지니어링의 궁극적 제약 조건이다.

## Ⅰ. 개요 (Context & Background)

- **개념**: 전류 (Current)는 전위차 (전압)가 존재하는 두 지점 사이를 이동하는 전하(주로 전자)의 연속적인 흐름을 뜻한다. 기호는 $I$를 사용하며 단위는 암페어 (A, Ampere)다. 컴퓨터 아키텍처 관점에서 전류는 논리 회로의 게이트를 구동하여 이진 신호(`0`과 `1`)를 생성하고 전파하는 실질적인 물리적 액션 (Action)이다.

- **💡 비유**: 전류는 전압이라는 펌프가 밀어내는 "배관 속의 물줄기"다. 수압 (전압)이 아무리 높아도 밸브 (트랜지스터)가 닫혀 있으면 물 (전류)은 흐르지 않으며, 밸브를 열었을 때 쏟아지는 물의 양이 연산의 속도를 결정한다. 하지만 파이프가 감당할 수 없을 만큼 너무 많은 물이 흐르면 파이프가 터져버린다 (발열 및 칩 파괴).

- **등장 배경 및 기술적 도약**:
  1. **BJT (Bipolar Junction Transistor)의 한계**: 초창기 트랜지스터인 BJT는 동작을 유지하기 위해 기저 전류 (Base Current)가 지속적으로 흘러야 했기 때문에 대규모 집적 회로(VLSI)로 구성할 경우 발열을 감당할 수 없었다.
  2. **MOSFET (Metal-Oxide-Semiconductor Field-Effect Transistor) 패러다임**: 전압만으로 채널을 제어하고 스위칭 순간에만 찰나의 동적 전류가 흐르는 CMOS (Complementary MOS) 구조가 등장하면서 마이크로프로세서의 전력 소모가 혁신적으로 감소했다.
  3. **누설 전류의 역습과 3D 트랜지스터**: 트랜지스터의 게이트 선폭이 20nm 이하로 줄어들면서, 채널 제어력이 약화되어 스위치를 꺼도 전자가 새어 나가는 터널링 (Tunneling) 기반의 누설 전류가 급증했다. 이를 막기 위해 핀펫 (FinFET)과 GAA (Gate-All-Around) 같은 입체 구조가 필수적으로 도입되었다.

공정 미세화에 따른 동적 전류와 누설 전류의 전력 소모 비중 역전 현상을 시각화하면, 현재 반도체 설계가 성능 향상보다 '새는 물 막기'에 집중하고 있음을 알 수 있다.

```text
  ┌───────────────────────────────────────────────────────────────┐
  │             공정 미세화에 따른 전류 소모 비중의 패러다임 전환            │
  ├───────────────────────────────────────────────────────────────┤
  │                                                               │
  │   100% ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐  │
  │        │          │ │          │ │          │ │ 누설 전류│  │
  │        │ 동적 전류│ │ 동적 전류│ │ 동적 전류│ │(Leakage) │  │
  │        │ (Active) │ │ (Active) │ │ (Active) │ │   > 50%  │  │
  │        ├──────────┤ ├──────────┤ ├──────────┤ ├──────────┤  │
  │        │ 누설 전류│ │ 누설 전류│ │ 누설 전류│ │ 동적 전류│  │
  │     0% └──────────┘ └──────────┘ └──────────┘ └──────────┘  │
  │           130nm         65nm         28nm       10nm 이하     │
  │                                                               │
  │  * 동적 전류 (Dynamic Current): 스위칭(연산)할 때만 흐르는 유효 전류 │
  │  * 누설 전류 (Leakage Current): 스위치를 꺼도 미세하게 새는 낭비 전류 │
  └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 이 그래프는 무어의 법칙 (Moore's Law) 이면에 숨겨진 전류 관리의 한계를 극명하게 보여준다. 130nm 공정 시절에는 칩 전체 전력의 대부분이 실제 연산을 수행하는 동적 전류에 의해 발생했다. 그러나 게이트 선폭이 원자 단위로 얇아지는 10nm 이하 초미세 공정에서는 스위치를 완전히 차단해도 양자 터널링 효과로 인해 미세한 전자가 절연막을 뚫고 지나간다. 수십억 개의 트랜지스터에서 발생하는 이 누설 전류를 합치면 전체 소모 전력의 절반을 넘어서게 되며, 이는 스마트폰이 주머니 속에서 아무 작업을 하지 않아도 배터리가 줄어들고 발열이 생기는 근본 원인이다.

- **📢 섹션 요약 비유**: 과거의 컴퓨터 설계가 "얼마나 빨리 달릴 것인가(동적 전류 집중)"의 문제였다면, 현대의 나노 공정 설계는 "쉬는 동안 어떻게 하면 숨소리조차 내지 않게 할 것인가(누설 전류 억제)"의 치열한 사투와 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 전류의 물리적 구성 요소 및 주요 파라미터

| 요소명 | 역할 | 물리적 의미 | 관련 파라미터 및 단위 | 비유 |
|:---|:---|:---|:---|:---|
| **Ion (On-Current)** | 트랜지스터 턴온 시 채널을 통해 흐르는 구동 전류 | 논리 상태 전이를 위한 커패시터 충방전 능력 | $mA/\mu m$ (밀리암페어 당 마이크로미터) | 완전히 열린 수문에서 쏟아지는 폭포수 |
| **Ioff (Off-Current)** | 트랜지스터 턴오프 시 새어 나가는 누설 전류 | 게이트 누설 (Gate Leakage) 및 문턱 이하 누설 (Sub-threshold) | $nA$ 또는 $pA$ (나노/피코 암페어) | 꽉 잠근 수도꼭지 틈새로 떨어지는 물방울 |
| **Current Density (J)** | 금속 배선의 단위 단면적당 흐르는 전류의 밀도 | 일렉트로마이그레이션 (EM) 등 배선 파괴 임계값 | $A/cm^2$ (제곱센티미터당 암페어) | 좁은 골목길에 한꺼번에 몰린 군중의 밀집도 |
| **Inrush Current** | 칩에 전원이 처음 인가될 때 발생하는 돌입 전류 | 커패시터 뱅크가 순간적으로 충전되며 발생하는 스파이크 현상 | $I_{peak}$ (최대 피크 전류) | 빈 물탱크에 밸브를 열었을 때 치솟는 초기 수압 |

### 디지털 회로 (CMOS)의 전류 흐름 메커니즘

모든 프로세서의 논리 연산은 결국 CMOS (Complementary Metal-Oxide-Semiconductor) 인버터 회로 내부의 커패시터를 채우고 (충전) 비우는 (방전) 전류의 흐름으로 환원된다. 

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               CMOS 인버터의 동적 전류 흐름 (Switching)            │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │     [출력을 '1'로 (Pull-up)]         [출력을 '0'으로 (Pull-down)] │
 │                                                               │
 │          Vdd (전원 공급)                   Vdd (차단됨)          │
 │           │                                │                  │
 │     ┌─────┴─────┐                    ┌─────┴─────┐            │
 │     │ PMOS (ON) │ ◀── 전류 유입      │ PMOS (OFF)│            │
 │     └─────┬─────┘    (Charging)      └─────┬─────┘            │
 │           ├───────▶ Out (Vdd, 1)          ├───────▶ Out (0V, 0)│
 │     ┌─────┴─────┐                    ┌─────┴─────┐            │
 │     │ NMOS (OFF)│                    │ NMOS (ON) │ ◀── 전류 유출│
 │     └─────┬─────┘                    └─────┬─────┘    (Discharge) │
 │           │                                │                  │
 │          Vss (접지)                       Vss (접지)            │
 │                                                               │
 │   충전 (Charge): Vdd에서 부하 커패시터 방향으로 흐르는 동적 전류     │
 │   방전 (Discharge): 커패시터에 저장된 전하가 Vss로 빠져나가는 전류    │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 상단의 PMOS와 하단의 NMOS는 결코 동시에 켜지지 않는 상보적 (Complementary) 특성을 가진다. 출력을 HIGH ('1')로 만들 때, PMOS가 켜지면서 외부 전원 ($V_{dd}$)으로부터 전류가 흘러들어와 부하 커패시터를 전자로 가득 채운다 (Pull-up). 반대로 출력을 LOW ('0')로 만들 때는 하단의 NMOS가 켜지며 커패시터에 모인 전하를 접지 ($V_{ss}$)로 순식간에 흘려버린다 (Pull-down). 즉, 컴퓨터가 클럭에 맞춰 계산을 한다는 것은 초당 수십억 번씩 ($GHz$) 이 미세한 전류의 폭포 (충전과 방전)를 일으키는 과정이며, 이 과정에서 발생하는 마찰열 ($I^2 R$)이 CPU의 동적 전력 소모의 실체다.

### 하드웨어 신뢰성의 킬러: 일렉트로마이그레이션 (Electromigration)

초미세 공정의 가장 큰 물리적 난제는 배선의 두께가 좁아지면서 발생하는 전류 밀도 (Current Density)의 상승이다. 전류 밀도가 한계치를 넘으면 전자가 금속 원자를 물리적으로 밀어내는 치명적인 파괴 현상이 발생한다.

```text
  ┌───────────────────────────────────────────────────────────────┐
  │        일렉트로마이그레이션 (EM) 현상에 의한 금속 배선의 파괴 과정       │
  ├───────────────────────────────────────────────────────────────┤
  │                                                               │
  │  [정상 상태의 배선]                                              │
  │  Metal Ions (구리 원자):  ○   ○   ○   ○   ○   ○   ○            │
  │  Electrons (전자 흐름):   →  →  →  →  →  →  →               │
  │                                                               │
  │  [EM 발생: 과도한 전류 밀도에 의한 전자풍(Electron Wind) 충돌]       │
  │                           강력한 충돌로 원자가 이탈                 │
  │                           ↓                                  │
  │  Metal Ions (구리 원자):  ○       ○   ○   ○           ○       │
  │  Electrons (전자 흐름):   →  →  →  →  →  →  →               │
  │                          (Void 형성)         (Hillock 형성)    │
  │                                                               │
  │  - Void (공극): 원자가 빠져나가 배선이 가늘어짐 → 결국 단선 (Open)  │
  │  - Hillock (돌출): 밀려난 원자가 쌓여 혹이 생김 → 이웃 배선과 합선   │
  └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 거시 세계에서는 전기가 전선을 통과할 때 형태가 변하지 않지만, 나노 미터 크기의 좁은 배선에 강한 전류가 흐르면 이동하는 전자들이 가지는 운동량이 '전자풍 (Electron Wind)'이라는 강력한 물리적 타격력을 발휘한다. 전자들이 배선을 구성하는 구리 (Cu) 원자들을 계속해서 때려 원래 자리에서 밀어내게 되는데, 원자가 빠져나간 자리는 구멍 (Void)이 되어 저항이 급증하다가 결국 선이 끊어진다 (단선, Open). 반대로 밀려난 원자들이 한곳에 쌓인 혹 (Hillock)은 피복을 뚫고 인접한 선과 닿아버린다 (단락, Short). 오버클럭을 무리하게 하면 CPU가 1~2년 만에 돌연사하는 가장 큰 원인이 바로 이 EM 현상 때문이다.

- **📢 섹션 요약 비유**: 수만 명의 인파 (전류)가 아주 좁은 지하철 통로 (미세 배선)를 질주할 때, 통로의 기둥 (금속 원자)에 계속 부딪혀 결국 기둥이 뽑히고 터널이 무너지는 대참사 (EM)와 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 1. 동적 전류 (Dynamic I) vs 정적 누설 전류 (Static Leakage I)

현대 칩 설계 아키텍트는 두 가지 완전히 다른 성질의 전류와 동시에 싸워야 한다.

| 비교 항목 | 동적 전류 (Dynamic Current, $I_{dyn}$) | 정적 누설 전류 (Static Leakage Current, $I_{leak}$) | 트레이드오프 및 설계 전략 |
|:---|:---|:---|:---|
| **발생 원인** | 회로가 '0' $\leftrightarrow$ '1'로 스위칭할 때 부하 정전 용량을 충/방전하기 위함 | 공정 미세화로 절연막 두께가 얇아져 스위치가 OFF된 상태에서도 전자가 양자 터널링함 | $I_{dyn}$은 주파수와 전압에 비례하므로 성능 향상 시 필연적으로 증가하나, $I_{leak}$는 공정의 물리적 한계로 발현됨 |
| **의존성 인자** | 클럭 주파수 ($f$), 인가 전압의 제곱 ($V^2$), 활성도 ($\alpha$) | 임계 전압 ($V_{th}$), 온도 ($T$, 온도가 오를수록 기하급수적 증가) | 온도가 높아지면 $I_{leak}$가 폭증하고 이로 인해 발열이 가속되는 열 폭주 (Thermal Runaway) 위험 발생 |
| **억제 기법** | **클럭 게이팅 (Clock Gating)**: 미사용 블록에 공급되는 클럭 신호 자체를 차단 | **파워 게이팅 (Power Gating)**: 미사용 블록으로 향하는 전원 (Vdd) 자체를 하드웨어 스위치로 완전 단절 | 동적 억제는 반응속도가 빠르지만 누설 전류를 막지 못함. 파워 게이팅은 누설을 막지만 켤 때 복구(Wake-up) 지연이 큼. |

### 2. 과목 융합 관점: 전류가 하드웨어 보안(Security)에 미치는 영향

- **부채널 공격 (Side-Channel Attack)**: 암호화 모듈(예: AES 가속기)이 특정 연산을 수행할 때마다 소모되는 전류의 미세한 파형 변화 (전력 소모 패턴)를 오실로스코프로 측정하여 DPA (Differential Power Analysis) 분석 기법을 적용하면, 내부의 비밀 키(Secret Key)를 역산할 수 있다. 즉, 칩이 소비하는 전류의 출렁임 그 자체가 데이터 유출의 통로가 된다.

- **📢 섹션 요약 비유**: 동적 전류는 자동차가 달릴 때 소모하는 '주행 연비'라면, 정적 누설 전류는 시동을 끄고 주차해 두었는데도 연료 탱크 미세한 틈으로 새어나가는 '기름 누수'입니다. 최신 차일수록 주행 연비는 좋지만, 엔진이 너무 정밀해 누수를 막는 기술이 더 까다롭습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 모바일 AP의 열 폭주 (Thermal Runaway) 통제

1. **상황**: 스마트폰에서 4K 해상도의 고사양 3D 게임을 20분 이상 실행 시, 애플리케이션 프로세서 (AP) 패키지의 표면 온도가 85도를 돌파하며 프레임이 60FPS에서 15FPS로 급락하는 쓰멀 스로틀링 (Thermal Throttling) 현상 발생.
2. **원인 분석**:
   - 고부하 연산으로 GPU의 **동적 전류** 소모가 극대화되며 1차 발열이 생성됨.
   - 칩의 온도가 70도를 넘어서자, 반도체 물성상 **정적 누설 전류 ($I_{leak}$)**가 지수함수적으로 폭증하기 시작함.
   - 새어 나가는 누설 전류가 다시 열을 만들고, 그 열이 다시 누설 전류를 부추기는 최악의 악순환인 **열 폭주 (Thermal Runaway)** 고리에 진입함.
3. **아키텍처적 대응 (DVFS 및 Power Gating 복합 제어)**:
   - AP 내부의 전력 관리 유닛 (PMU)이 온도를 감지하여, 즉시 GPU의 클럭 주파수를 강제로 절반으로 깎아 동적 전류의 유입을 물리적으로 제한함.
   - 동시에, 현재 사용하지 않는 NPU (신경망 처리 장치)나 백그라운드 코어의 전원 블록을 파워 게이팅 (Power Gating) 기술로 완전히 잘라내어 누설 전류가 빠져나가는 파이프 자체를 봉쇄함.
4. **결과**: 발열량이 패키지의 열 설계 전력 (TDP) 방열 한계선 아래로 수렴하며, 칩의 소손(파괴)을 막고 시스템 다운 없이 서비스를 지속함.

### 전력 무결성 및 발열 제어 운영 의사결정 플로우

운영체제와 펌웨어가 온도를 감지했을 때, 단순하게 클럭만 낮추는 것이 아니라 상황의 원인 (동적 vs 정적)에 따라 타격 포인트를 다르게 설정해야 한다.

```text
  ┌───────────────────────────────────────────────────────────────┐
  │         시스템 온도 상승 시 전류 제어(전력 관리) 의사결정 트리         │
  ├───────────────────────────────────────────────────────────────┤
  │                                                               │
  │   [칩 온도가 Thermal Limit (TjMax) 임계치에 근접함 감지]           │
  │                │                                              │
  │                ▼                                              │
  │      활성화된 코어의 연산 부하율(Utilization)이 높은가?              │
  │          ├─ 예 ─────▶ [동적 전류 과다에 의한 발열 상태]             │
  │          │                     │                              │
  │          │                     └─▶ [DVFS를 통해 클럭과 전압 하향] │
  │          │                          (성능 저하 감수)             │
  │          │                                                    │
  │          └─ 아니오                                            │
  │                │                                              │
  │                ▼                                              │
  │      유휴 상태(Idle)임에도 온도가 떨어지지 않는가?                    │
  │          ├─ 예 ─────▶ [누설 전류(Leakage) 폭증 및 열 폭주 상태]    │
  │          │                     │                              │
  │          │                     └─▶ [미사용 블록 파워 게이팅 실행]   │
  │          │                          (물리적 전원 완전 차단)       │
  │          │                                                    │
  │          └─ 아니오 ──▶ [시스템 냉각부(팬, 방열판) 물리적 결함 점검]  │
  └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 발열이 감지되었을 때의 대응 로직이다. 연산량이 폭주하여 생기는 발열은 클럭 주파수와 전압을 낮추는 DVFS로 해결할 수 있다. 하지만 칩이 별다른 일을 하지 않는데도 뜨겁다면, 이는 칩 내부의 트랜지스터들이 누설 전류에 의해 통제력을 잃고 불필요한 열을 뿜어내는 '다크 실리콘' 현상의 일환이다. 이때는 해당 구역의 전원 자체를 물리적으로 끊어버리는 파워 게이팅이 유일한 해법이다. 이러한 펌웨어 수준의 미세한 전류 제어가 최신 노트북과 스마트폰 배터리 수명의 핵심 경쟁력이다.

### 안티패턴 및 실무 체크리스트
- **안티패턴 (돌입 전류 방치)**: 파워 게이팅으로 꺼두었던 코어를 다시 켤 (Wake-up) 때, 전원부에서 막대한 충전 전류 (Inrush Current)가 한꺼번에 몰려들어 전압 스파이크 (Voltage Droop)를 유발, 인접 코어까지 시스템 프리징을 발생시키는 설계 결함. 반드시 '소프트 스타트 (Soft-start)' 회로를 넣어 전류가 서서히 유입되도록 해야 한다.
- **도입 체크리스트**:
  - 고부하 연산 유닛 (FPU, SIMD 등)을 바닥 평면에 한곳에 집중적으로 배치하여 전류 밀도를 높이는 핫스팟 (Hot-spot) 설계가 아닌, 열원 분산 레이아웃이 적용되었는가?
  - Black's Equation (블랙의 방정식)을 기반으로, 설계된 배선 폭과 예상 전류 밀도가 시스템 목표 수명 (예: 서버용 5년, 자동차용 10년)을 만족하는지 EM (Electromigration) 마진을 검증했는가?

- **📢 섹션 요약 비유**: 댐이 무너지기 전에 물의 양을 줄이는 것(DVFS)도 중요하지만, 댐에 미세한 실금이 가서 뿜어져 나오는 물(누설 전류)을 막기 위해 아예 수문을 이중으로 닫아버리는 것(파워 게이팅)이 현대 제어 기술의 핵심입니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 최적화 전 (단순 전압/클럭 제어) | 최적화 후 (파워 게이팅 및 EM 회피 레이아웃) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (전력)** | 유휴 상태에서도 누설 전류로 전력 낭비 | 미사용 블록의 서브 시스템 전원 완벽 차단 | 유휴 전력 소모 **50~70% 감축**, 배터리 수명 극대화 |
| **정량 (수명)** | 좁은 배선에 높은 전류 밀도 집중 방치 | 전류 분산 배선(Mesh) 및 블랙의 방정식 검증 | EM으로 인한 단선 지연, MTBF **2배 이상 증가** |
| **정성 (보안)** | 일관된 전류 소모 패턴 노출 | 전류 파형 마스킹 (Randomization) 및 평탄화 도입 | 부채널 공격 (DPA) 방어로 하드웨어 암호 키 유출 원천 차단 |

### 미래 전망
- **BSPDN (Backside Power Delivery Network)**: 2nm 이하 나노 공정에서는 신호선과 얽혀 좁아진 전원 배선으로 인해 발생하는 저항 증가와 전류 밀도 한계를 극복하기 위해, 칩의 뒷면 (후면)을 통째로 전력 공급망으로 사용하는 BSPDN 아키텍처가 전면 도입된다. 이는 전류 병목을 해결하는 궁극의 설계 혁신이다.
- **실리콘 포토닉스 (Silicon Photonics)**: 구리 배선을 통과하는 '전하(전자)'의 물리적 마찰과 발열 한계를 근본적으로 넘어서기 위해, 전자를 빛 (광자, Photon)으로 변환하여 데이터를 전송함으로써 열 발생 없이 테라바이트급 초거대 대역폭을 달성하려는 연구가 본격적으로 상용화 궤도에 오르고 있다.

### 참고 표준
- **JEDEC JESD** 및 **MIL-HDBK-217**: 반도체 디바이스의 전류 스트레스, 가속 수명 시험 (ALT, Accelerated Life Testing), 그리고 일렉트로마이그레이션(EM) 신뢰성 평가에 대한 국제 산업 표준 규격.

- **📢 섹션 요약 비유**: 자동차가 더 이상 도로의 정체를 뚫고 매연을 뿜으며 달리는 대신, 하늘을 나는 비행기 (빛, 광자)로 진화하거나 도로의 지하 층 (뒷면 전원망)을 통째로 뚫어버리는 혁명적인 방식으로 전류의 한계를 넘어서고 있습니다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **전압 (Voltage)** | 전류가 흐르도록 밀어내는 전기적 압력으로, 두 물리량의 곱 ($P = V \times I$)이 프로세서의 실질적인 전력 소모량이 된다. |
| **저항 (Resistance)** | 전류의 흐름을 물리적으로 방해하는 요소로, 배선에서 발생하는 $I^2 R$ 손실은 칩 전체의 발열 (Thermal)을 주도한다. |
| **일렉트로마이그레이션 (EM)** | 초미세 배선에서 과도한 전류 밀도의 전자풍이 금속 원자를 밀어내어 단선이나 합선을 유발하는 하드웨어 조기 사망의 주범이다. |
| **파워 게이팅 (Power Gating)** | 미사용 블록의 $V_{dd}$와 $V_{ss}$ 연결을 하드웨어적으로 완전히 끊어내어, 누설 전류 (Leakage Current)를 원천 차단하는 다크 실리콘 대응 기술이다. |
| **부채널 공격 (Side-Channel Attack)** | 암호화 연산 시 발생하는 미세한 전류 소모량의 변화 패턴을 분석하여 내부의 비밀 키를 역으로 빼내는 강력한 하드웨어 해킹 기법이다. |

2. 그런데 길이 좁은데 너무 많은 요정들이 한꺼번에 질주하면 서로 부딪혀서 땀을 뻘뻘 흘리고(컴퓨터가 뜨거워짐), 심지어 길이 부서져버릴 수도 있어요(고장).
3. 그래서 똑똑한 기술자 어른들은 요정들이 필요할 때만 달리고, 쉬는 시간에는 문을 꽉 닫아주어서 요정들이 힘을 빼앗기지 않고 배터리도 오래갈 수 있게 마법의 집(반도체 설계)을 만들어준답니다!
