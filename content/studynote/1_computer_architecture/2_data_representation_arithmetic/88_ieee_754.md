# IEEE 754 표준 (IEEE 754 Standard)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IEEE 754는 1985년 IEEE (Institute of Electrical and Electronics Engineers)가 제정한 부동소수점 연산의 세계 표준으로, 실수의 저장 포맷(단정밀도/배정밀도), 특수값(무한대, NaN), 반올림 모드, 예외 처리 방식을 엄격히 규정하여 CPU (Central Processing Unit)·GPU (Graphics Processing Unit)·AI 가속기 간의 수치 연산 결과를 호환 가능하게 만든다.
> 2. **가치**: 이 표준 이전에는 CPU 벤더마다 부동소수점 연산 결과가 달라 동일 알고리즘이 시스템마다 다른 결과를 내는 이식성 문제가 심각했다. IEEE 754는 오늘날 x86, ARM, RISC-V, CUDA 등 모든 현대 아키텍처의 공통 기반이 되어 수십억 디바이스의 수학적 일관성을 보장한다.
> 3. **융합**: 딥러닝의 폭발적 성장과 함께 IEEE 754의 변형(bfloat16, FP8)이 등장했지만, 근본적인 NaN (Not a Number)·무한대·서브노멀 처리 방식은 IEEE 754를 그대로 계승한다. AI 추론·과학 계산·금융 시뮬레이션 모두 이 표준 위에서 동작한다.

### Ⅰ. 개요 (Context & Background) - [500자+]

- **개념**: IEEE 754 (IEEE Standard for Floating-Point Arithmetic)는 컴퓨터 시스템에서 부동소수점(Floating-Point) 수를 표현하고 연산하는 방법에 대한 국제 표준이다. 이 표준은 데이터의 표현 방식(비트 레이아웃), 반올림(Rounding) 알고리즘, 연산 결과 예외(Exception) 처리 등을 포함한 포괄적인 아키텍처를 정의한다. 1985년 초판이 공표된 이후, 2008년(IEEE 754-2008)과 2019년(IEEE 754-2019)을 거쳐 개정되었으며, 현재 DEC (Decimal Floating-Point) 포맷과 같은 확장 기능을 포함하여 산술 연산의 기준이 되고 있다. 이 표준의 핵심 철학은 "수치 계산의 이식성(Portability)과 재현성(Reproducibility)"을 보장하는 것이다.

- **💡 비유**: IEEE 754는 **'전 세계 모든 계산기 제조사가 지켜야 할 사용자 매뉴얼'**과 같다. 버튼을 눌렀을 때 어떤 숫자가 나와야 하고, 0으로 나누면 어떤 에러를 표시해야 하며, 반올림은 어떻게 할지를 모든 제조사가 똑같이 따르도록 강제한다. 덕분에 미국산 앱과 일본산 서버가 통신해도 서로 같은 수학적 결과를 보장받을 수 있다.

- **등장 배경 및 발전 과정**:
  1. **표준 이전의 혼란 (Pre-1985)**: 1970년대까지만 해도 각 하드웨어 벤더(Intel, DEC, IBM, Cray 등)는 고유의 부동소수점 포맷을 사용했다. 이로 인해 동일한 소프트웨어 코드라도 실행되는 하드웨어에 따라 결과값이 달라지거나, 한 기계에서는 작동하던 코드가 다른 기계에서는 오류를 발생시키는 심각한 호환성 문제가 존재했다.
  2. **윌리엄 카한(William Kahan)과 8087 FPU**: 인텔의 8087 수치 코프로세서(FPU, Floating-Point Unit) 설계를 주도하던 윌리엄 카한 교수는 이러한 혼란을 해결하기 위해 수학적 엄밀성과 하드웨어 구현의 효율성을 절충하는 표준안을 제안했다. 이것이 오늘날의 IEEE 754의 기원이 된다.
  3. **현대적인 확장**: 최근에는 AI (Artificial Intelligence) 연산의 효율을 위해 FP8 (8-bit Floating Point), FP16 (16-bit Floating Point), BF16 (Brain Floating Point)과 같은 낮은 비트 수의 포맷이 표준화되거나 산업 표준(de facto standard)으로 자리 잡고 있으나, 그 근간에는 여전히 IEEE 754의 기본 정의(NaN 처리, 무한대 등)가 유지되고 있다.

- **📢 섹션 요약 비유**: IEEE 754 표준의 등장은 마치 **'제각각인 철도궤도의 폭을 전 세계가 표준 궤도(Standard Gauge)로 통일한 것'**과 같아서, 이제 데이터라는 기차가 국가와 하드웨어를 넘나들며 탈 없이 운행될 수 있게 되었다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive) - [1,000자+]

IEEE 754의 핵심은 크게 **데이터 저장 형식(Format)**, **특수 값(Special Values)**, **반올림 방식(Rounding Modes)**, 그리고 **예외 처리(Exceptions)**로 구성된다. 각 구성 요소의 내부 동작 메커니즘을 심층 분석한다.

#### 1. 구성 요소 및 비트 레이아웃 (Bit Layout)

IEEE 754는 수치를 부호(S), 지수(E), 가수(M)의 세 부분으로 나누어 저장한다. 가수부의 경우, 정규화된 수(Normalized Number)에서는 암시적 선행 비트(Implicit Leading Bit, 정수부 1)를 가정하여 유효 자릿수(Precision)를 1비트 추가로 확보하는 독특한 구조를 사용한다.

| 포맷 | 총 비트 | 부호 (S) | 지수 (E) | 가수 (M) | 편향 값 (Bias) | 유효 자릿수 (Precision) | 특징 |
|:---|:---:|:---:|:---:|:---:|:---:|:---:|:---|
| **FP16 (Half)** | 16 | 1 | 5 | 10 | 15 | 11비트 | 딥러닝, 그래픽스 |
| **FP32 (Single)** | 32 | 1 | 8 | 23 | 127 | 24비트 | 일반 연산, 표준 |
| **FP64 (Double)** | 64 | 1 | 11 | 52 | 1023 | 53비트 | 과학 계산, 고정밀도 |
| **FP128 (Quadruple)** | 128 | 1 | 15 | 112 | 16383 | 113비트 | 초고정밀도 금융/과학 |

#### 2. 값의 해석 및 상태 전이 (Value Interpretation)

IEEE 754의 가장 강력한 기능 중 하나는 지수 필드의 값에 따라 비트 패턴을 **0, 정규화 수(Normalized), 비정규화 수(Denormalized/Subnormal), 무한대(Infinity), NaN** 등 5가지 상태로 해석하는 유연한 메커니즘이다.

```text
  ┌────────────────────────────────────────────────────────────────────────────┐
  │           IEEE 754 상태 머신 (32bit Single Precision 기준)                  │
  │                                                                            │
  │  ┌────────────────┬─────────────────────────────────────────────────┐      │
  │  │   E (지수)     │                    해석 규칙                     │      │
  │  ├────────────────┼─────────────────────────────────────────────────┤      │
  │  │ E = 0          │ [비정규화 수 (Subnormal)]                        │      │
  │  │ 1 ~ 254        │ [정규화 수 (Normalized)]                         │      │
  │  │ E = 255        │ [특수값 (Special: ∞ or NaN)]                     │      │
  │  └────────────────┴─────────────────────────────────────────────────┘      │
  │                                                                            │
  │  ┌──────────────────────────────────────────────────────────────────────┐ │
  │  │  1. 비정규화 수 (Denormalized / Subnormal Number)                    │ │
  │  │     조건: E=0, M≠0                                                    │ │
  │  │     값: (-1)^S × (0.M) × 2^(-126)       <-- 암시적 1 아님, 지수는 -126 │ │
  │  │     목적: 0에 근접한 매우 작은 값 표현 (언더플로우 방지)              │ │
  │  └──────────────────────────────────────────────────────────────────────┘ │
  │                                                                            │
  │  ┌──────────────────────────────────────────────────────────────────────┐ │
  │  │  2. 정규화 수 (Normalized Number)                                   │ │
  │  │     조건: 1 ≤ E ≤ 254                                                │ │
  │  │     값: (-1)^S × (1.M) × 2^(E-127)      <-- 암시적 1 포함            │ │
  │  │     예: S=0, E=128(10000000), M=0.5 → 1.5 × 2^1 = 3.0                │ │
  │  └──────────────────────────────────────────────────────────────────────┘ │
  │                                                                            │
  │  ┌──────────────────────────────────────────────────────────────────────┐ │
  │  │  3. 특수값 (Special Values)                                         │ │
  │  │     - Zero:      E=0, M=0 (±0 존재)                                  │ │
  │  │     - Infinity:  E=255, M=0 (±∞ 존재)                                │ │
  │  │     - NaN:       E=255, M≠0 (Not a Number)                           │ │
  │  │       └─ SNaN (Signaling):  M의 최상위 비트 0 → 예외 트랩 유발        │ │
  │  │       └─ QNaN (Quiet):     M의 최상위 비트 1 → 조용히 전파           │ │
  │  └──────────────────────────────────────────────────────────────────────┘ │
  └────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 위 다이어그램은 IEEE 754의 32비트 포맷이 지수 필드 $E$의 값에 따라 어떻게 다르게 해석되는지를 보여주는 유한 상태 머신(Finite State Machine)의 일종이다. 가장 중요한 점은 **'서브노멀(Subnormal)'**의 존재다. $E=0$일 때, 하드웨어는 암시적 선행 1을 사용하지 않고(0을 사용), 지수를 $1-Bias$가 아닌 $-Bias$로 고정한다. 이는 표현 가능한 가장 작은 정규화 수($2^{-126}$)와 0 사이의 간극을 매워주어 **'점진적 언더플로우(Gradual Underflow)'**를 구현하는 핵심 메커니즘이다. 이를 통해 0.000001과 0.000002를 더할 때 갑자기 0이 되어버리는 '소실(Sudden Underflow)' 현상을 방지한다. 반면 $E=255$일 때는 수치적 한계를 초과했거나 정의되지 않은 연산을 처리하기 위한 특수값(Infinity, NaN)으로 할당된다.

#### 3. 반올림 모드 (Rounding Modes)

정확한 연산 결과가 주어진 비트 수(예: 23비트)로 표현 불가능할 때, IEEE 754는 다음과 같은 5가지 반올림 방식을 제공한다. 기본값은 RNE(Round to Nearest, ties to Even)이다.

```text
  ┌───────────────────────────────────────────────────────────────────────────┐
  │               반올림 모드별 동작 원리 (Rounding Decision)                  │
  ├───────────────────────────────────────────────────────────────────────────┤
  │                                                                           │
  │  원래 값 v가 정확히 두 표현 가능한 값 a와 b 사이에 있다고 가정 (a < v < b)  │
  │                                                                           │
  │  1. RNE (Round to Nearest, ties to Even) [기본값]                         │
  │     - 가장 가까운 값 선택                                                 │
  │     - v가 a와 b의 정확히 중간(Tie)일 경우, 짝수(Even)인 값 선택           │
  │     - ex) 2.5 → 2 (짝수), 3.5 → 4 (짝수)                                 │
  │     - 편향(Bias)를 0으로 만들어 통계적 오차 최소화                         │
  │                                                                           │
  │  2. RNA (Round to Nearest, ties to Max Magnitude)                         │
  │     - 가장 가까운 값 선택                                                 │
  │     - 중간일 경우 절�댓값이 더 큰 값 선택                                 │
  │     - ex) 2.5 → 3, -2.5 → -3                                             │
  │                                                                           │
  │  3. RU (Round toward +∞) / Ceiling                                       │
  │     - 값보다 크거나 같은 가장 가까운 값                                   │
  │     - ex) 2.1 → 3, -2.1 → -2                                             │
  │                                                                           │
  │  4. RD (Round toward -∞) / Floor                                         │
  │     - 값보다 작거나 같은 가장 가까운 값                                   │
  │     - ex) 2.9 → 2, -2.1 → -3                                             │
  │                                                                           │
  │  5. RZ (Round toward 0) / Truncate                                       │
  │     - 0 방향으로 자름                                                    │
  │     - ex) 2.9 → 2, -2.9 → -2                                             │
  │                                                                           │
  └───────────────────────────────────────────────────────────────────────────┘
```

#### 4. 핵심 코드 및 수식 (C/C++ 구현)

IEEE 754의 정확한 이해를 위해서는 비트 연산을 통한 값의 재구성이 필수적이다. 아래는 `union`을 활용하여 `float` (FP32)의 비트 패턴을 분해하고 재조립하는 코드이다.

```c
#include <stdio.h>
#include <stdint.h>

typedef union {
    float f;
    uint32_t u;
} float_bits;

// IEEE 754 부동소수점 값 분해
void inspect_float(float f) {
    float_bits fb;
    fb.f = f;
    uint32_t bits = fb.u;

    uint32_t sign = (bits >> 31) & 0x1;
    uint32_t exp  = (bits >> 23) & 0xFF;
    uint32_t mant = bits & 0x7FFFFF; // 23 bits

    printf("Value: %f\n", f);
    printf("Sign: %d, Exp: 0x%02X, Mant: 0x%06X\n", sign, exp, mant);

    // 역추적 공식 (정규화 수의 경우)
    if (exp == 0 && mant != 0) {
        printf("Type: Subnormal (Denormalized)\n");
        // value = (-1)^S * 0.M * 2^(-126)
    } else if (exp == 255) {
        if (mant == 0) printf("Type: Infinity\n");
        else printf("Type: NaN (Quiet: %d)\n", (mant >> 22) & 0x1);
    } else {
        printf("Type: Normalized\n
