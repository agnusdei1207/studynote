# 117. ALU (산술 논리 연산 장치)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU의 핵심 연산 유닛으로, 산술 연산(加减乘除)과 논리 연산(AND, OR, XOR, NOT)을 수행하는 디지털 회로
> 2. **가치**: 모든 컴퓨터 계산의 물리적 기반으로, 가산기·시프터·비교기 등으로 구성되어 1~수 클럭 내에 결과 생성
> 3. **융합**: 제어 유닛(CU), 레지스터 파일, 캐시와 협력하여 명령어 사이클(fetch-decode-execute)의 execute 단계 담당

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

ALU(Arithmetic Logic Unit, 산술 논리 연산 장치)는 CPU 내에서 실제 연산을 수행하는 하드웨어 모듈이다. 두 개의 입력 피연산자와 제어 신호를 받아 산술 연산(ADD, SUB, MUL, DIV)과 논리 연산(AND, OR, XOR, NOT), 시프트 연산을 수행하고 결과와 상태 플래그(Carry, Zero, Sign, Overflow)를 출력한다.

### 💡 비유: 만능 계산기

ALU는 **스마트폰의 계산기 앱 + 로직 게임**이다. 숫자 두 개를 입력하면 더하기, 빼기뿐 아니라 AND/OR 같은 논리 연산도 한다. 결과뿐 아니라 "0인가?", "음수인가?", "넘쳤는가?" 같은 추가 정보도 알려준다.

### 등장 배경

| 시기 | 배경 | 발전 |
|:---:|:---|:---|
| 1945 | 폰 노이만 구조 제안 | 연산 장치 개념 정립 |
| 1970 | Intel 74181 | 최초의 단일 칩 ALU (4비트) |
| 1978 | Intel 8086 | 16비트 ALU 내장 |
| 현재 | 현대 CPU | 슈퍼스칼라, SIMD ALU |

### 전가산기 기반 ALU 구현

```verilog
// 1비트 ALU 모듈 (Verilog)
module alu_1bit (
    input  a, b,         // 피연산자
    input  cin,          // 입력 캐리
    input  [2:0] op,     // 연산 선택
    output result,       // 결과
    output cout,         // 출력 캐리
    output zero          // 0 플래그
);
    wire sum, and_out, or_out, xor_out;

    // 각 연산 수행
    assign sum     = a ^ b ^ cin;
    assign and_out = a & b;
    assign or_out  = a | b;
    assign xor_out = a ^ b;
    assign cout    = (a & b) | (b & cin) | (a & cin);

    // 연산 선택 (멀티플렉서)
    assign result = (op == 3'b000) ? sum :      // ADD
                    (op == 3'b001) ? sum :      // SUB (b 반전)
                    (op == 3'b010) ? and_out :  // AND
                    (op == 3'b011) ? or_out :   // OR
                    (op == 3'b100) ? xor_out :  // XOR
                    1'b0;

    assign zero = ~result;
endmodule

// 32비트 ALU (Ripple Carry)
module alu_32bit (
    input  [31:0] a, b,
    input  [3:0]  op,
    output [31:0] result,
    output        zero, negative, carry, overflow
);
    wire [31:0] cout_chain;
    wire [31:0] alu_result;

    // 32개의 1비트 ALU 연결
    genvar i;
    generate
        for (i = 0; i < 32; i = i + 1) begin : alu_slice
            alu_1bit slice (
                .a(a[i]),
                .b(b[i] ^ op[0]),  // SUB 시 b 반전
                .cin(i == 0 ? op[0] : cout_chain[i-1]),
                .op(op[2:0]),
                .result(alu_result[i]),
                .cout(cout_chain[i])
            );
        end
    endgenerate

    assign result    = alu_result;
    assign zero      = (alu_result == 32'b0);
    assign negative  = alu_result[31];
    assign carry     = cout_chain[31];
    assign overflow  = (a[31] == b[31]) && (alu_result[31] != a[31]);
endmodule
```

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: 고성능 CPU 설계**
- 요구: 4GHz 동작, 4 ALU 병렬
- 설계: 캐리 예측 가산기(CLA), 바렐 시프터
- 결과: CPI 0.25 달성

**시나리오 2: 저전력 임베디드**
- 요구: 10μW 미만, 단순 연산
- 설계: Ripple Carry Adder, 최소 기능
- 결과: 에너지 효율 극대화

**시나리오 3: AI 가속기**
- 요구: INT8 행렬 연산
- 설계: 텐서 코어 (다중 MAC ALU)
- 결과: 100 TOPS

### ALU 설계 트레이드오프

| 설계 선택 | 장점 | 단점 |
|:---|:---|:---|
| Ripple Carry | 단순, 저면적 | 느림 (O(n)) |
| Carry Look-ahead | 빠름 (O(log n)) | 복잡, 면적 큼 |
| Carry Select | 중간 | 면적 증가 |
| Carry Skip | 중간 | 불균형 지연 |

### 👶 어린이를 위한 3줄 비유 설명

**개념**: ALU는 컴퓨터의 '계산 담당 선생님'이에요! 덧셈, 뺄셈뿐 아니라 AND, OR 같은 논리 퍼즐도 풀 수 있어요.

**원리**: 두 숫자를 받아서 무슨 계산을 할지 알려주면(더하기? 빼기?), 결과와 함께 "0이야?", "음수야?" 같은 힌트도 같이 알려줘요.

**효과**: 컴퓨터가 게임을 하든, 영상을 편집하든, 모든 계산이 이 작은 회로 하나에서 일어나요. CPU의 진짜 일꾼이에요!
