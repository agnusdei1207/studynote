# 109. 홀수 패리티 (Odd Parity)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 비트와 패리티 비트를 합쳤을 때 1의 총 개수가 홀수가 되도록 패리티 비트를 설정하는 방식
> 2. **가치**: 0x00(모두 0) 상태를 비정상으로 간주하여 "무음" 상태와 "실제 0 데이터"를 구별 가능
> 3. **융합**: 일부 통신 시스템에서 송수신기 활성 상태 확인용으로 활용, 짝수 패리티와 상호 보완

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

홀수 패리티(Odd Parity)는 데이터 비트열에 1개의 패리티 비트를 추가하여, 전체 비트(데이터 + 패리티)에서 1의 개수가 항상 홀수가 되도록 하는 오류 검출 방식이다. 데이터 비트의 1의 개수가 짝수면 패리티 비트를 1로, 홀수면 0으로 설정한다.

### 💡 비유: 홀수 인원만 출발하는 버스

홀수 패리티는 **버스가 출발하려면 승객이 반드시 홀수 명이어야 하는 규칙**과 같다. 정류장에 아무도 없으면(0명=짝수) 버스는 출발하지 않는다. 즉, "아무것도 없는 상태"와 "데이터가 있는 상태"를 확실히 구별할 수 있다.

### 등장 배경

| 구분 | 내용 |
|:---|:---|
| **개발 목적** | 짝수 패리티와 상호 보완적 역할 |
| **특별 용도** | 0x00 상태 검증, 활성 상태 확인 |
| **사용처** | 일부 산업용 통신, 마크/스페이스 패리티 변형 |

```
┌─────────────────────────────────────────────────────────────────────┐
│                    홀수 패리티 동작 원리                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  규칙: 전체 비트에서 1의 개수 = 홀수 (1, 3, 5, 7...)               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터         1의 개수   패리티 비트   전체       1의 개수  │ │
│  ├───────────────────────────────────────────────────────────────┤ │
│  │  0000000        0 (짝수)      1        10000000     1 (홀수)  │ │
│  │  0000001        1 (홀수)      0        00000001     1 (홀수)  │ │
│  │  0100001        2 (짝수)      1        10100001     3 (홀수)  │ │
│  │  0111111        6 (짝수)      1        10111111     7 (홀수)  │ │
│  │  1111111        7 (홀수)      0        01111111     7 (홀수)  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  패리티 비트 결정 공식:                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  P = ¬(D0 ⊕ D1 ⊕ D2 ⊕ D3 ⊕ D4 ⊕ D5 ⊕ D6 ⊕ D7)               │ │
│  │      └────────────────────────────────────────────┘           │ │
│  │                    데이터 XOR의 NOT                           │ │
│  │                                                               │ │
│  │  • 데이터 1의 개수가 홀수면 P = 0 (총 홀수)                  │ │
│  │  • 데이터 1의 개수가 짝수면 P = 1 (총 홀수)                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소 | 설명 |
|:---|:---|
| 데이터 비트 | 원본 데이터 (예: 7비트) |
| 패리티 비트 | P = NOT XOR(모든 데이터 비트) |
| 검사 결과 | 전체 XOR = 1이면 정상, 0이면 오류 |

### 홀수 패리티 생성/검사 회로

```
┌─────────────────────────────────────────────────────────────────────┐
│                    홀수 패리티 생성 회로                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│       D6 D5 D4 D3 D2 D1 D0                                         │
│        │  │  │  │  │  │  │                                         │
│        └──┴──┴──┴──┴──┴──┴──┐                                      │
│                               ↓                                    │
│                        ┌──────────┐                                │
│                        │   XOR    │                               │
│                        │  TREE    │                               │
│                        └────┬─────┘                               │
│                             │                                      │
│                        ┌────┴────┐                                │
│                        │   NOT   │──→ P (패리티 비트)             │
│                        └─────────┘                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  짝수 vs 홀수 패리티 생성 비교                                │ │
│  │                                                               │ │
│  │  짝수:  XOR ─────────────────────────────→ P                 │ │
│  │  홀수:  XOR ────→ NOT ────→ P                                │ │
│  │                          ↑                                    │ │
│  │                    추가 게이트 필요                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【수신 측 검사】                                                  │
│                                                                     │
│  전체 XOR = 1 → 정상 (1의 개수가 홀수)                            │
│  전체 XOR = 0 → 오류 (1의 개수가 짝수)                            │
└─────────────────────────────────────────────────────────────────────┘
```

### 소프트웨어 구현

```c
#include <stdint.h>

// 홀수 패리티 비트 생성
uint8_t odd_parity_generate(uint8_t data) {
    uint8_t count = 0;
    uint8_t temp = data;
    while (temp) {
        count++;
        temp &= (temp - 1);
    }
    return !(count & 1);  // 홀수면 0, 짝수면 1 (짝수 패리티의 반대)
}

// 홀수 패리티 검사
int odd_parity_check(uint8_t data_with_parity) {
    uint8_t count = 0;
    uint8_t temp = data_with_parity;
    while (temp) {
        count++;
        temp &= (temp - 1);
    }
    // 홀수면 정상(0), 짝수면 오류(1)
    return !(count & 1);
}

// XOR를 이용한 효율적 구현
uint8_t odd_parity_generate_fast(uint8_t data) {
    data ^= data >> 4;
    data ^= data >> 2;
    data ^= data >> 1;
    return !(data & 1);  // NOT 추가
}
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 짝수 vs 홀수 패리티 상세 비교

| 항목 | 짝수 패리티 | 홀수 패리티 |
|:---|:---|:---|
| **1의 총개수** | 짝수 (0,2,4,6,8) | 홀수 (1,3,5,7) |
| **P = XOR(데이터)** | 그대로 | NOT |
| **0x00 전송** | 00000000 | 10000000 |
| **수신 0x00** | 정상 | 오류 (선 단선 의심) |
| **하드웨어** | XOR만 | XOR + NOT |
| **주요 용도** | 일반 데이터 | 활성 상태 확인 |

### 마크/스페이스 패리티 (Mark/Space Parity)

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: 산업용 통신선 감시**
- 상황: 긴 통신 케이블, 노이즈 환경
- 선택: 홀수 패리티
- 이유: 선 단선 시 모든 0 → 홀수 패리티로 즉시 감지

**시나리오 2: 데이터와 주소 구분**
- 상황: 멀티드롭 RS-485 통신
- 선택: Mark/Space 패리티 활용
- 이유: 주소 바이트는 패리티=1, 데이터 바이트는 패리티=0

### 도입 체크리스트

| 체크 항목 | 홀수 패리티 권장 상황 |
|:---|:---|
| ☐ 통신선 상태 감지 | 선 단선/단락 감지 필요 시 |
| ☐ 활성 상태 확인 | 무음=오류로 처리해야 할 때 |
| ☐ 시스템 호환성 | 기존 시스템이 홀수 패리티 사용 |
| ☐ 하드웨어 여유 | NOT 게이트 추가 가능 |

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 홀수 패리티는 항상 1, 3, 5... 같은 홀수만 좋아하는 규칙이에요. 전체에서 1의 개수가 홀수여야만 "정상!"이라고 해요.

**원리**: 1의 개수를 세어서 짝수면 1을 더하고, 홀수면 0을 더해서 무조건 홀수로 만들어요.

**효과**: 아무것도 안 보내면(모두 0) 1의 개수가 0개(짝수)라서 "뭔가 이상해!" 하고 바로 알 수 있어요. 컴퓨터가 잠들었는지 확인할 수 있답니다!
