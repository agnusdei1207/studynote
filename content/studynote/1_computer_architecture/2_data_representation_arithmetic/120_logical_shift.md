# 120. 논리 시프트 (Logical Shift)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비트열을 이동시키며 빈 자리를 항상 0으로 채우는 시프트 연산으로, 부호 없는 정수의 2^n 곱셈/나눗셈에 대응
> 2. **가치**: 1클럭에 수행되는 고속 연산으로, 비트 마스킹, 플래그 추출, 부호 없는 산술의 기본 연산
> 3. **융합**: 비트 필드 조작, 색상 채널 추출, 네트워크 프로토콜 파싱의 핵심 기법

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

논리 시프트(Logical Shift)는 비트열을 왼쪽이나 오른쪽으로 이동시키며, 어느 방향이든 빈 자리는 항상 0으로 채우는 연산이다. 왼쪽 시프트(SHL/LSL)와 오른쪽 시프트(SHR/LSR)가 있으며, 부호 없는 정수(unsigned) 연산에 주로 사용된다.

### 💡 비유: 틈새로 밀어내기

논리 시프트는 **서랍의 물건을 한쪽으로 밀어내는 것**과 같다. 왼쪽으로 밀면 왼쪽 끝의 물건은 떨어지고, 오른쪽에 빈 공간이 생긴다. 그 빈 공간을 새 물건(0)으로 채우는 것이다.

### 등장 배경

| 구분 | 내용 |
|:---|:---|
| **기원** | 초기 컴퓨터의 효율적 곱셈/나눗셈 |
| **발전** | 모든 CPU의 기본 명령어 |
| **용도** | 부호 없는 산술, 비트 조작 |

```
┌─────────────────────────────────────────────────────────────────────┐
│                    논리 시프트 상세                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【논리 왼쪽 시프트 (SHL / LSL / <<)】                             │
│  ───────────────────────────────────                               │
│                                                                     │
│  원본 8비트: 0 0 1 1 0 1 0 1 (53)                                   │
│                                                                     │
│  << 1:   0│1 1 0 1 0 1 0│0   → 01101010 (106) = 53×2              │
│          └─버림─┘       └─0추가─┘                                  │
│                                                                     │
│  << 2:   1│1 0 1 0 1 0 0│0   → 11010100 (212) = 53×4              │
│          └─버림─┘     └─00추가─┘                                   │
│                                                                     │
│  << 3: 1│1 0 1 0 1 0 0 0│  → 오버플로우 (원래 53×8=424 > 255)    │
│         └버림┘   └─000추가─┘                                       │
│                                                                     │
│  【논리 오른쪽 시프트 (SHR / LSR / >>)】                           │
│  ────────────────────────────────────                              │
│                                                                     │
│  원본 8비트: 0 0 1 1 0 1 0 1 (53)                                   │
│                                                                     │
│  >> 1:   0│0 0 1 1 0 1 0│1   → 00011010 (26) = 53÷2               │
│          └─0추가─┘       └─버림─┘                                  │
│                                                                     │
│  >> 2: 0 0│0 0 0 1 1 0 1│1   → 00001101 (13) = 53÷4               │
│        └─00추가─┘     └─버림─┘                                     │
│                                                                     │
│  >> 4: 0 0 0 0│0 0 1 1│0 1 0 1 → 00000011 (3) = 53÷16              │
│        └─0000추가─┘ └─버림─┘                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  공식                                                          │ │
│  │  ────                                                          │ │
│  │  A << n  =  A × 2^n   (오버플로우 가능)                       │ │
│  │  A >> n  =  A ÷ 2^n   (소수점 버림, floor)                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 논리 시프트 vs 산술 시프트

```
┌─────────────────────────────────────────────────────────────────────┐
│                논리 vs 산술 시프트 비교                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  8비트 값: 1 1 1 0 0 1 0 0 (-28 in 2's complement)                 │
│                                                                     │
│  오른쪽으로 2비트 시프트:                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  논리 시프트 (SHR):                                           │ │
│  │  0 0│1 1 1 0 0 1│0 0                                          │ │
│  │  ↑↑└──────────┘└──┘                                          │ │
│  │  00추가      원본   버림                                      │ │
│  │  결과: 00111001 = 57 (잘못된 결과!)                          │ │
│  │                                                               │ │
│  │  산술 시프트 (SAR):                                           │ │
│  │  1 1│1 1 1 0 0 1│0 0                                          │ │
│  │  ↑↑└──────────┘└──┘                                          │ │
│  │  부호비트    원본   버림                                      │ │
│  │  결과: 11111001 = -7 (-28÷4 = -7) ✓                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ⚠️ 부호 있는 수에 논리 시프트 사용 시 주의!                       │
└─────────────────────────────────────────────────────────────────────┘
```

### 논리 시프트 응용

```c
#include <stdint.h>

// 비트 마스크 생성
uint32_t mask_n_bits(int n) {
    return (1 << n) - 1;  // n=4 → 0b1111
}

// 상위 n비트 추출
uint32_t extract_high_bits(uint32_t x, int n) {
    return x >> (32 - n);
}

// 하위 n비트 추출
uint32_t extract_low_bits(uint32_t x, int n) {
    return x & ((1 << n) - 1);
}

// 중간 비트 추출 (from ~ to)
uint32_t extract_bits(uint32_t x, int from, int to) {
    int width = to - from + 1;
    return (x >> from) & ((1 << width) - 1);
}

// RGB 색상 분리
void split_rgb(uint32_t rgb, uint8_t *r, uint8_t *g, uint8_t *b) {
    *r = (rgb >> 16) & 0xFF;
    *g = (rgb >> 8) & 0xFF;
    *b = rgb & 0xFF;
}

// RGB 색상 결합
uint32_t combine_rgb(uint8_t r, uint8_t g, uint8_t b) {
    return ((uint32_t)r << 16) | ((uint32_t)g << 8) | b;
}

// IP 주소 파싱
void parse_ip(uint32_t ip, uint8_t *a, uint8_t *b, uint8_t *c, uint8_t *d) {
    *a = (ip >> 24) & 0xFF;
    *b = (ip >> 16) & 0xFF;
    *c = (ip >> 8) & 0xFF;
    *d = ip & 0xFF;
}
```

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: 비트맵 인덱스**
- 상황: 32개 항목의 존재 여부를 1비트로 표현
- 연산: `(bitmap >> index) & 1`로 특정 비트 확인
- 장점: O(1) 시간, 메모리 효율

**시나리오 2: 고정소수점 연산**
- 상황: Q16.16 형식 곱셈
- 연산: `(a * b) >> 16`으로 소수점 조정
- 용도: 임베디드 DSP

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 논리 시프트는 모든 빈 자리를 0으로 채우는 밀기 놀이예요! 왼쪽으로 밀면 ×2, 오른쪽으로 밀면 ÷2!

**원리**: 숫자 0 1 0 1이 있으면, 왼쪽으로 한 칸 가면 1 0 1 0이 되고(×2), 오른쪽으로 한 칸 가면 0 0 1 0이 돼요(÷2).

**효과**: 컴퓨터가 곱하기와 나누기를 엄청 빠르게 할 수 있어요. 단, 음수에는 쓰면 안 돼요!
