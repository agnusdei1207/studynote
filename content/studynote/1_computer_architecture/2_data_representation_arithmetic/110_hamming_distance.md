# 110. 해밍 거리 (Hamming Distance)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 두 같은 길이의 비트열에서 서로 다른 비트의 개수를 의미하며, 오류 검출 및 수정 능력을 정량화하는 핵심 지표
> 2. **가치**: 최소 해밍 거리 d로 d-1비트 오류 검출, ⌊(d-1)/2⌋비트 오류 수정 가능을 수학적으로 보장
> 3. **융합**: 해밍 코드 설계, 통신 프로토콜 신뢰성 분석, 생체 인식 시스템의 기초 이론

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

해밍 거리(Hamming Distance)는 같은 길이의 두 문자열(또는 비트열)에서 대응하는 위치의 문자가 다른 개수를 의미한다. Richard Hamming이 1950년에 정의했으며, 오류 검출 및 수정 코드의 설계에 필수적인 개념이다. 두 비트열의 XOR 연산 결과에서 1의 개수와 같다.

### 💡 비유: 단어 철자 비교

"KOREA"와 "KAREA"를 비교하면 2번째 글자(O→A)와 3번째 글자(R→R, 같음)를 비교한다. 2글자가 다르므로 해밍 거리는 2다. 마찬가지로 비트열 10110과 11101에서는 2, 4, 5번째 비트가 달라 해밍 거리는 3이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소 | 설명 |
|:---|:---|
| 비트열 A, B | 비교 대상 문자열 |
| XOR 연산 | 비트별 차이 계산 |
| 1의 개수 | XOR 결과에서 1 카운트 = 해밍 거리 |
| 최소 해밍 거리 | 코드북 내 모든 쌍 중 최솟값 |

### 해밍 거리 계산

```c
#include <stdint.h>

// 해밍 거리 계산
int hamming_distance(uint8_t a, uint8_t b) {
    uint8_t xor_result = a ^ b;
    int distance = 0;

    // 1의 개수 세기 (Brian Kernighan 알고리즘)
    while (xor_result) {
        distance++;
        xor_result &= (xor_result - 1);
    }
    return distance;
}

// 확장: 임의 길이 비트열
int hamming_distance_bytes(const uint8_t* a, const uint8_t* b, int len) {
    int distance = 0;
    for (int i = 0; i < len; i++) {
        uint8_t xor_result = a[i] ^ b[i];
        while (xor_result) {
            distance++;
            xor_result &= (xor_result - 1);
        }
    }
    return distance;
}
```

### 공식 유도

```
┌─────────────────────────────────────────────────────────────────────┐
│                오류 검출/수정 공식 유도                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【오류 검출】                                                      │
│  • 두 유효 코드 간 최소 거리가 d                                   │
│  • 오류 e비트 발생 시: d(code, error) = e                          │
│  • 오류를 검출하려면: error가 다른 유효 코드가 되면 안 됨          │
│  • 즉, e < d일 때만 오류임을 알 수 있음                           │
│  • 따라서 최대 d-1비트 오류까지 검출 가능                          │
│                                                                     │
│  【오류 수정】                                                      │
│  • 오류 코드를 가장 가까운 유효 코드로 수정                        │
│  • 수정이 유일하려면: 오류 코드가 한 유효 코드에만 가장 가까워야 함│
│  • 즉, 두 유효 코드의 구체(sphere)가 겹치지 않아야 함              │
│  • 구체 반경 r = ⌊(d-1)/2⌋ 이하이면 보장                          │
│  • 따라서 ⌊(d-1)/2⌋비트 오류까지 수정 가능                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  d_min = 5인 예시                                             │ │
│  │                                                               │ │
│  │  코드 A ────┬──── 2 ────┬──── 오류코드1                      │ │
│  │            d=5          │                                     │ │
│  │  코드 B ────┴──── 3 ────┴──── 오류코드2                      │ │
│  │                                                               │ │
│  │  오류코드1은 A와 거리 2 → A로 수정 (가장 가까움)             │ │
│  │  오류코드2는 A와 거리 3, B와 거리 2 → B로 수정               │ │
│  │  ⇒ 최대 2비트 오류 수정 가능: ⌊(5-1)/2⌋ = 2                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: 통신 프로토콜 설계**
- 요구: 10⁻⁶ BER 환경에서 99.99% 신뢰성
- 분석: 평균 오류 비트 수 기반 d_min 결정
- 선택: d_min = 5 이상의 코딩 방식

**시나리오 2: 홍채 인식 시스템**
- 요구: FAR < 0.001%, FRR < 0.1%
- 설계: 2048비트 코드, 임계값 d=30
- 근거: 정상인 평균 d=10, 타인 평균 d=500

### 설계 체크리스트

| 항목 | 고려사항 |
|:---|:---|
| ☐ 채널 오류율 | BER 측정 또는 예측 |
| ☐ 요구 신뢰성 | 검출/수정 필요 수준 |
| ☐ 대역폭 제약 | 오버헤드 허용 범위 |
| ☐ 지연 요구 | 복잡도 vs 속도 트레이드오프 |

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 해밍 거리는 두 단어에서 다른 글자가 몇 개인지 세는 것이에요. "강아지"와 "강아짜"는 1개가 다르니까 거리가 1이에요!

**원리**: 두 비트열에서 서로 다른 자리의 개수를 세는데, XOR로 계산하면 1이 나온 자리가 다른 거예요. 거리가 멀수록 완전히 다른 코드라는 뜻이에요.

**효과**: 거리가 3 이상이면 1개가 틀려도 원래 코드를 찾을 수 있어요. 마치 "가나다"와 "가나타"가 다르지만, "가나다"가 원래 단어라는 걸 알 수 있는 것처럼요!
