# NaN (Not a Number)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: NaN(Not a Number)은 IEEE 754 부동소수점 표준에서 정의된 특수값으로, 수학적으로 정의되지 않는 연산(0÷0, ∞−∞, √(-1) 등)의 결과를 나타내며, "숫자가 아님"을 의미한다.
> 2. **가치**: NaN은 프로그램을 중단하지 않고 오류 상태를 전파(Propagate)할 수 있게 하여, 수치 연산 파이프라인의 연속성을 보장한다. NaN이 연산에 참여하면 결과도 NaN이 되어, 오류가 "오염"되어 전파된다.
> 3. **융합**: NaN은 SNaN(Signaling NaN, 예외 발생)과 QNaN(Quiet NaN, 조용 전파)로 구분된다. 딥러닝에서 NaN은 그래디언트 폭주(Exploding Gradient)나 연산 오류의 지표가 되며, 데이터 분석에서는 결측치(Missing Value) 표현에 활용된다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: NaN(Not a Number)은 IEEE 754 부동소수점 표준에서 수학적으로 정의할 수 없는 결과를 표현하기 위해 예약된 특수값이다. IEEE 754 포맷에서 지수 필드가 모두 1이고(E = Max), 가수 필드가 0이 아닌(M ≠ 0) 경우 NaN으로 해석된다. NaN의 핵심 속성은 `NaN == NaN`이 `false`라는 점이다(NaN은 자기 자신과도 같지 않다).

- **💡 비유**: NaN은 **'계산기의 "Error" 표시'**와 같다. 0으로 나누거나 음수의 제곱근을 계산하려고 하면, 계산기는 "Error"라고 표시한다. 컴퓨터의 NaN도 비슷하지만, 한 가지 다른 점이 있다: NaN이 포함된 추가 연산을 계속할 수 있다는 것이다. "Error" 값을 가지고 계속 계산하면 결과도 "Error"가 된다.

- **등장 배경 및 발전 과정**:
  1. **초기 컴퓨터의 오류 처리**: IEEE 754 이전에는 정의되지 않은 연산이 발생하면 프로그램이 중단되거나, 예측 불가능한 값이 출력되었다. 이는 과학 계산에서 치명적이었다.
  2. **IEEE 754-1985의 NaN 도입**: 윌리엄 카한(William Kahan)은 "결과가 정의되지 않음"을 표현하는 특수값으로 NaN을 제안했다. NaN은 프로그램을 중단하지 않고, 오류 상태를 전파한다.
  3. **SNaN vs QNaN 구분**: 일부 NaN은 연산 시 즉시 예외(Signal/Exception)를 발생시키고(SNaN), 일부는 조용히 전파만 된다(QNaN). 이 구분은 디버깅과 런타임 오류 감지에 활용된다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### NaN의 비트 표현

NaN은 지수 필드가 모두 1이고, 가수 필드가 0이 아닌 값이다. 가수 필드의 최상위 비트가 SNaN/QNaN을 구분한다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         NaN의 비트 표현 (FP32 기준)                                           │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   FP32 NaN 조건: E = 255 (11111111), M ≠ 0                                  │
  │                                                                              │
  │   ┌────┬──────────────┬───────────────────────────────────────────────────┐ │
  │   │ S  │  11111111    │              M (가수, 23비트)                       │ │
  │   └────┴──────────────┴───────────────────────────────────────────────────┘ │
  │    1비트     8비트                         23비트                           │
  │                                                                              │
  │   NaN 종류 구분 (가수 최상위 비트 M[22]):                                     │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  M[22] = 1 → QNaN (Quiet NaN)                                          ││
  │   │            연산 시 예외를 발생시키지 않고, 조용히 NaN으로 전파             ││
  │   │                                                                         ││
  │   │  M[22] = 0 → SNaN (Signaling NaN)                                      ││
  │   │            연산 시 Invalid Exception을 발생시킴                          ││
  │   │            (나머지 가수 비트가 0이 아니어야 함)                           ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   예시 (FP32):                                                                │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  QNaN: 0 11111111 1xxxxxxxxxxxxxxxxxxxxxxxxxxx                        ││
  │   │               ↑ M[22] = 1                                              ││
  │   │                                                                         ││
  │   │  SNaN: 0 11111111 0xxxxxxxxxxxxxxxxxxxxxxxxxxx (단, x 중 하나는 1)     ││
  │   │               ↑ M[22] = 0                                              ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ⚠ NaN의 나머지 비트(페이로드, Payload)는 사용자 정의 정보를 저장할 수 있음    │
  │      예: JavaScript 엔진은 NaN의 페이로드에 추가 정보를 저장하기도 함         │
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** NaN은 지수가 최댓값(255 for FP32)이고 가수가 0이 아닌 값이다. 가수의 최상위 비트(M[22])가 1이면 QNaN(Quiet NaN), 0이면 SNaN(Signaling NaN)이다. QNaN은 연산 시 조용히 NaN으로 전파되고, SNaN은 연산 시 Invalid Exception을 발생시킨다. 가수의 나머지 비트(페이로드)는 사용자 정의 정보를 저장할 수 있다. 예를 들어 JavaScript 엔진은 NaN에 추가 정보를 저장하기도 한다.

### NaN 발생 조건과 전파 규칙
NaN은 다양한 수학적으로 정의되지 않는 연산에서 발생하며, 연산에 참여하면 결과도 NaN이 된다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         NaN 발생 조건과 전파 규칙                                              │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   NaN 발생 조건:                                                              │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  연산                  │  결과   │  설명                                ││
  │   │  ─────────────────────────────────────────────────────────────────      ││
  │   │  0 / 0                │  NaN   │  0을 0으로 나눔 (불확정)              ││
  │   │  ∞ / ∞               │  NaN   │  무한대를 무한대로 나눔               ││
  │   │  ∞ - ∞               │  NaN   │  무한대에서 무한대를 뺌               ││
  │   │  0 × ∞               │  NaN   │  0과 무한대의 곱                       ││
  │   │  √(-1)              │  NaN   │  음수의 제곱근 (실수 범위)            ││
  │   │  log(-1)            │  NaN   │  음수의 로그                          ││
  │   │  asin(2)            │  NaN   │  정의역 밖의 역삼각함수               ││
  │   │  NaN ○ any          │  NaN   │  NaN이 포함된 모든 연산               ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   NaN 전파 규칙:                                                             │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  NaN + 5 = NaN       (덧셈)                                            ││
  │   │  NaN × 2 = NaN       (곱셈)                                            ││
  │   │  sqrt(NaN) = NaN     (함수)                                            ││
  │   │  NaN < 5 → false     (비교, 항상 false)                                ││
  │   │  NaN == NaN → false  (동등 비교, 핵심 속성!)                           ││
  │   │  NaN != NaN → true   (부등 비교)                                       ││
  │   │                                                                         ││
  │   │  ⚠ NaN은 모든 비교에서 false (자기 자신과의 == 제외하고는)            ││
  │   │     단, NaN != NaN은 true (NaN은 자기 자신과도 다름)                    ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   NaN 검출 방법:                                                             │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  C/C++:  isnan(x)     또는  x != x                                    ││
  │   │  Python: math.isnan(x)                                               ││
  │   │  JavaScript: Number.isNaN(x) (isNaN(x)은 형변환 문제 있음)            ││
  │   │  Java:   Double.isNaN(x)                                             ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** NaN은 0÷0, ∞−∞, 음수의 제곱근 등 수학적으로 정의되지 않는 연산에서 발생한다. NaN의 핵심 속성은 연산에 참여하면 결과도 NaN이 된다는 것이다(전파). 그리고 `NaN == NaN`이 `false`라는 점이다. 이 때문에 `if (x != x)` 또는 `isnan(x)`로 NaN을 검출할 수 있다. JavaScript의 `isNaN(x)`은 문자열 "hello"도 NaN으로 판단하는 문제가 있어, `Number.isNaN(x)` 사용이 권장된다.

### SNaN vs QNaN: Signaling vs Quiet
SNaN(Signaling NaN)과 QNaN(Quiet NaN)은 오류 처리 방식에서 차이가 있다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         SNaN vs QNaN 동작 비교                                                │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   QNaN (Quiet NaN):                                                         │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  • 연산 시 예외 발생 안 함                                              ││
  │   │  • 결과는 QNaN으로 전파                                                ││
  │   │  • 일반적인 연산에서 발생 (0/0, √-1 등)                                ││
  │   │  • 프로그램 흐름 유지                                                   ││
  │   │                                                                         ││
  │   │  예: QNaN + 5 → QNaN (예외 없이 조용히 NaN 반환)                       ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   SNaN (Signaling NaN):                                                     │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  • 연산 시 Invalid Exception 발생 (기본적으로는)                        ││
  │   │  • 예외 핸들러가 호출되거나 플래그 설정                                  ││
  │   │  • 사용자가 직접 생성 (표준 연산에서는 생성 안 됨)                      ││
  │   │  • 디버깅, 오류 추적, 초기화되지 않은 변수 표시에 사용                 ││
  │   │                                                                         ││
  │   │  예: SNaN + 5 → Invalid Exception 발생 → 핸들러 호출 또는 QNaN 반환     ││
  │   │      (대부분의 시스템에서는 기본적으로 SNaN을 QNaN으로 변환)              ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   실제 동작 (x86 기본 설정):                                              │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  1. SNaN을 연산에 사용                                                ││
  │   │  2. Invalid Exception 플래그 설정                                         ││
  │   │  3. 결과는 QNaN으로 반환 (SNaN → QNaN 변환)                              ││
  │   │  4. 예외 마스크가 켜져 있지 않으면 프로그램 계속 실행                              ││
  │   │  5. 예외 마스크가 켜져 있으면 SIGFPE 시그널 발생                              ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ⚠ 대부분의 시스템에서 SNaN은 QNaN으로 변환되어 반환됨                             │
  │      예외 마스크 설정에 따라 동작이 달라짐                                           │
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** SNaN(Signaling NaN)은 연산 시 Invalid Exception을 발생시켜, 프로그래머가 오류를 조기에 감지할 수 있게 한다. 반면 QNaN(Quiet NaN)은 조용히 전파만 된다. 일반적인 연산(0÷0, √-1 등)에서 발생하는 NaN은 QNaN이다. SNaN은 사용자가 직접 생성하여, 초기화되지 않은 변수나 디버깅 목적으로 사용한다. x86에서는 SNaN을 연산에 사용하면 Invalid Exception 플래그가 설정되고, 결과는 QNaN으로 변환된다. 예외 마스크가 켜져 있지 않으면 프로그램이 계속 실행되고, 켜져 있으면 SIGFPE 시그널이 발생한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교: NaN vs ∞ vs 0

| 비교 항목 | NaN | ∞ (Infinity) | 0 |
|:---|:---|:---|:---|
| **의미** | 정의되지 않은 결과 | 오버플로우 결과 | 언더플로우 또는 실제 0 |
| **비교 연산** | 항상 false (자기 자신과도) | 정상 동작 | 정상 동작 |
| **전파** | NaN으로 전파 | ∞로 전파 (규칙에 따라) | 0으로 전파 |
| **연산 참여** | 결과 NaN | 결과 ∞ 또는 NaN | 결과 0 또는 NaN |

### 과목 융합 관점
- **딥러닝(ML)**: NaN은 그래디언트 폭주(Exploding Gradient)나 연산 오류의 지표다. PyTorch/TensorFlow는 `torch.isnan()`, `tf.math.is_nan()`으로 NaN을 감지하고, `torch.nan_to_num()`으로 대체할 수 있다
- **데이터 과학(Data Science)**: Pandas와 NumPy는 NaN을 결측치(Missing Value) 표현에 사용한다. `np.nan`, `pd.NA`가 이에 해당하며, `fillna()`로 대체한다
- **웹 개발(Web)**: JavaScript의 `NaN`은 형변환 오류의 결과로 자주 등장한다. `parseInt("hello")`는 NaN을 반환한다. `Number.isNaN()`으로 검출해야 한다

## Ⅳ. 실무 적용 및 기술사적 판단
### 실무 시나리오
1. **시나리오 — 딥러닝 훈련 중 NaN 발생**: GPT-3 훈련 중 Loss가 갑자기 NaN이 되었다. 로그를 확인하니 그래디언드가 폭주하여 NaN이 발생했다. 기술사는 ① Learning Rate를 1/10로 낮추고, ② Gradient Clipping을 적용하고, ③ NaN 발생 시 Early Stopping하도록 코드를 수정한다
2. **시나리오 — 데이터 분석에서 NaN 처리**: 고객 데이터에 NaN(결측치)이 포함되어 있어, 평균 계산이 NaN으로 나온다. 기술사는 `df.dropna()`로 결측치를 제거하거나, `df.fillna(df.mean())`으로 평균값으로 대체한다
3. **시나리오 — JavaScript 형변환 오류**: `parseInt(userInput)`이 NaN을 반환하여, 후속 연산이 모두 NaN이 되었다. 기술사는 `if (Number.isNaN(value))`로 검출하고, 기본값을 할당하거나 에러를 반환한다
### 도입 체크리스트
- **기술적**: 모든 부동소수점 연산 후 NaN 검출 코드가 있는가? `isnan()` 매크로/함수 사용
- **운영·보안적**: NaN이 외부 API 응답에 포함될 수 있는가? NaN 감지 및 대체 로직이 있는가?
### 안티패턴
- **NaN 검출 없이 연속 연산**: NaN이 발생해도 감지하지 않고 계속 연산하면, 최종 결과가 NaN이 되어 디버깅이 어렵다
- **`isNaN(x)` 대신 `Number.isNaN(x)` 미사용 (JavaScript)**: `isNaN("hello")`는 true이지만, "hello"는 NaN이 아니다

### 미래 전망
- **AI 시스템의 NaN**: 대규모 언어 모델(LLM) 추론 시 NaN 발생은 치명적이다. NaN 방지를 위한 Stable Training, Quantization Aware Training이 연구되고 있다
- **Rust/Go의 NaN 처리**: 안전한 언어에서도 부동소수점 연산은 IEEE 754를 따르므로 NaN 발생 가능. Result 타입으로 래핑하여 명시적 처리가 권장된다

1. NaN은 **'계산기의 "Error" 표시'** 같아요. 0으로 나누거나, 음수의 제곱근을 계산하려고 하면 "Error"라고 떠요
2. 재미있는 건, "Error"끼리 더해도 "Error"이고, "Error"에 5를 더해도 "Error"라는 거예요! 오류가 전염되는 것처럼 퍼져요
3. 그리고 "Error" == "Error"는 거짓이에요! 똑같아 보여도, 컴퓨터는 "Error는 Error와 같지 않다"고 생각한답니다. 이걸 이용해서 오류를 찾아요
