# 107. 패리티 비트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 전송 중 1비트 오류를 검출하기 위해 데이터 비트에 추가하는 1개의 중복 비트로, 1의 개수를 짝수 또는 홀수로 맞추는 방식
> 2. **가치**: 하드웨어 구현이 극도로 단순하며(1개 XOR 게이트), 50%의 오류 검출 확률로 메모리·통신 시스템의 기본 안전망 제공
> 3. **융합**: RAM 패리티, 직렬 통신(UART), PCI 버스 등 하드웨어 계층의 오류 검출 기반 기술로, 해밍 코드·CRC의 선행 개념

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

패리티 비트(Parity Bit)는 데이터 비트열에 추가하여 1의 개수를 짝수(Even) 또는 홀수(Odd)로 만드는 1비트의 중복 정보다. 수신 측에서 1의 개수를 다시 계산해 패리티 규칙을 위반하면 오류가 발생했음을 알 수 있다. 1비트 오류만 검출 가능하며, 2비트 이상 동시 오류는 검출하지 못한다.

### 💡 비유: 교실 출석부

패리티 비트는 **교실 출석부의 총원 확인**과 같다. 선생님이 "반 전체가 30명이면 짝수야"라고 정해두면, 아침마다 학생 수를 세어서 31명이면(홀수) 누가 들어왔거나 나갔다는 것을 바로 알 수 있다. 단, 2명이 동시에 들어와도 32명이 되어 짝수라서 변화를 모르는 것이 한계다.

### 등장 배경

| 시기 | 배경 | 해결책 |
|:---:|:---|:---|
| 1940s | 초기 컴퓨터 메모리 신뢰성 낮음 | 오류 검출 메커니즘 필요 |
| 1950s | 벨 연구소 통신 품질 향상 연구 | 패리티 체크 개발 |

### 패리티 체크 회로

```c
// 짝수 패리티 생성 및 검사 (소프트웨어 구현)
#include <stdint.h>

// 패리티 비트 생성 (짝수 패리티)
uint8_t generate_even_parity(uint8_t data) {
    uint8_t parity = 0;
    uint8_t temp = data;

    while (temp) {
        parity ^= (temp & 1);  // XOR 누적
        temp >>= 1;
    }
    return parity;  // 1의 개수가 홀수면 1, 짝수면 0
}

// 패리티 검사 (데이터 + 패리티 비트)
int check_even_parity(uint8_t data_with_parity) {
    uint8_t parity = 0;
    uint8_t temp = data_with_parity;

    while (temp) {
        parity ^= (temp & 1);
        temp >>= 1;
    }
    return parity;  // 0이면 정상, 1이면 오류
}

// 사용 예시
void example() {
    uint8_t data = 0b01000001;  // 'A' = 65

    // 송신: 패리티 생성
    uint8_t parity = generate_even_parity(data);
    uint8_t transmitted = (parity << 7) | data;  // 8비트

    // 수신: 패리티 검사
    if (check_even_parity(transmitted) == 0) {
        // 정상: 오류 없음
    } else {
        // 오류: 1비트 오류 발생
    }
}
```

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: DRAM ECC 설계**
- 요구: 서버급 메모리 1비트 오류 자동 수정
- 선택: SEC-DED (Single Error Correction, Double Error Detection)
- 구현: 8비트 데이터 + 5비트 해밍 코드 (패리티 확장)

**시나리오 2: 임베디드 UART 통신**
- 요구: 노이즈 많은 환경에서 신뢰성 확보
- 선택: 짝수 패리티 + 재전송 프로토콜
- 비용: 하드웨어 XOR 게이트 1개만 추가

**시나리오 3: RAID 5 스토리지**
- 요구: 디스크 1개 손실 시 데이터 복구
- 선택: XOR 패리티 분산 저장
- 복구: 나머지 디스크 XOR로 손실 데이터 복원

### 도입 체크리스트

| 기술적 체크 | 운영 체크 |
|:---|:---|
| ☐ 짝수/홀수 패리티 결정 | ☐ 오류 발생 시 로그 기록 |
| ☐ 하드웨어/소프트웨어 구현 선택 | ☐ 주기적 패리티 검증 스케줄 |
| ☐ 2비트 오류 미검출 한계 인지 | ☐ 패리티 오류율 모니터링 |
| ☐ 오류 발생 시 복구 전략 수립 | ☐ 대체 오류 검출 방식 고려 |

| 항목 | 패리티 없음 | 패리티 사용 | 효과 |

| **하드웨어 비용** | 최소 | +1 XOR 게이트 | 무시할 수준 |
| **오류 검출률** | 0% | ~50% | 1비트 오류 100% 검출 |
| **성능 영향** | 없음 | 1 클럭 지연 | 무시할 수준 |
| **복구 가능** | - | ❌ | 검출만 가능 |

### 미래 전망

1. **ECC 메모리 보급**: 패리티 → SEC-DED로 확장
2. **NVMe/PCIe**: 패리티에서 CRC로 강화
3. **AI 가속기**: 높은 오류율로 ECC 필수

**개념**: 패리티 비트는 퍼즐 조각이 몇 개인지 세어보는 것과 같아요. 조각이 항상 짝수 개여야 한다고 정해두면, 하나가 없어지거나 생기면 바로 알 수 있어요.
