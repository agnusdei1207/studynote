# 111. 해밍 코드 (Hamming Code)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 최소 해밍 거리 3을 갖는 선형 블록 코드로, 1비트 오류를 검출하고 수정(SEC)하며 2비트 오류를 검출(DED)할 수 있는 오류 정정 코드
> 2. **가치**: RAM ECC, 위성 통신, 저장 장치의 신뢰성 보장에 필수적이며, 패리티 비트 수는 ⌈log₂(n+1)⌉로 최적화됨
> 3. **융합**: 정보 이론의 핵심 응용으로, 확장 해밍 코드(SEC-DED), Reed-Solomon 등 고급 코딩 이론의 기반

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

해밍 코드(Hamming Code)는 1950년 Richard Hamming이 개발한 선형 오류 수정 코드로, 1비트 오류를 자동으로 수정(Single Error Correction, SEC)할 수 있다. (7,4) 해밍 코드는 4비트 데이터에 3비트 패리티를 추가해 7비트 코드워드를 만들며, 최소 해밍 거리가 3이다.

### 💡 비유: 겹치는 영역의 투표

해밍 코드는 **3명의 감시자가 서로 다른 조합으로 감시**하는 것과 같다. 각 감시자가 담당하는 비트가 겹치게 설정되어, 어느 한 비트가 틀리면 세 감시자의 보고서를 교차 검증하여 정확히 어느 비트가 틀렸는지 찾아낼 수 있다.

### 등장 배경

| 시기 | 배경 | 해결책 |
|:---:|:---|:---|
| 1947 | 벨 연구소 펀치카드 판독 오류 | Hamming의 주말 연구 |
| 1950 | 논문 발표 | (7,4) 해밍 코드 정립 |
| 1970s | IBM 메인프레임 | 메모리 ECC 적용 |
| 현재 | 서버급 RAM, SSD | SEC-DED 표준 |

### 소프트웨어 구현

```c
#include <stdint.h>

// (7,4) 해밍 코드 인코딩
uint8_t hamming74_encode(uint8_t data) {
    // data: 하위 4비트만 사용 (D1,D2,D3,D4)
    uint8_t d1 = (data >> 0) & 1;
    uint8_t d2 = (data >> 1) & 1;
    uint8_t d3 = (data >> 2) & 1;
    uint8_t d4 = (data >> 3) & 1;

    uint8_t p1 = d1 ^ d2 ^ d4;
    uint8_t p2 = d1 ^ d3 ^ d4;
    uint8_t p4 = d2 ^ d3 ^ d4;

    // 코드워드: [p1, p2, d1, p4, d2, d3, d4]
    return (p1 << 0) | (p2 << 1) | (d1 << 2) | (p4 << 3) |
           (d2 << 4) | (d3 << 5) | (d4 << 6);
}

// (7,4) 해밍 코드 디코딩 (오류 수정 포함)
uint8_t hamming74_decode(uint8_t codeword) {
    uint8_t p1 = (codeword >> 0) & 1;
    uint8_t p2 = (codeword >> 1) & 1;
    uint8_t d1 = (codeword >> 2) & 1;
    uint8_t p4 = (codeword >> 3) & 1;
    uint8_t d2 = (codeword >> 4) & 1;
    uint8_t d3 = (codeword >> 5) & 1;
    uint8_t d4 = (codeword >> 6) & 1;

    // 신드롬 계산
    uint8_t s1 = p1 ^ d1 ^ d2 ^ d4;
    uint8_t s2 = p2 ^ d1 ^ d3 ^ d4;
    uint8_t s4 = p4 ^ d2 ^ d3 ^ d4;

    uint8_t syndrome = (s4 << 2) | (s2 << 1) | s1;

    // 오류 수정
    if (syndrome != 0) {
        codeword ^= (1 << (syndrome - 1));  // 해당 위치 비트 반전
        // 수정된 값 다시 읽기
        d1 = (codeword >> 2) & 1;
        d2 = (codeword >> 4) & 1;
        d3 = (codeword >> 5) & 1;
        d4 = (codeword >> 6) & 1;
    }

    return (d4 << 3) | (d3 << 2) | (d2 << 1) | d1;
}
```

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 해밍 코드는 선생님이 숙제를 3번 다르게 검사하는 것과 같아요. 하나만 틀려도 3개의 검사를 조합하면 정확히 어디가 틀렸는지 찾을 수 있어요!

**원리**: 4개의 데이터 비트에 3개의 패리티 비트를 추가하는데, 각 패리티가 서로 다른 조합의 비트를 감시해요. 오류가 나면 3개의 패리티 결과로 정확한 위치를 알아내요.

**효과**: 컴퓨터 메모리에서 1비트가 깨져도 스스로 고칠 수 있어요. 서버 컴퓨터가 며칠 동안 켜 있어도 안전한 비밀이에요!
