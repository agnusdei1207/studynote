# 122. 순환 시프트 (Rotate / Circular Shift)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비트열의 양끝이 연결된 고리처럼 비트가 순환하는 시프트 연산으로, 어떤 비트도 손실되지 않음
> 2. **가치**: 암호화 알고리즘(AES, DES), 해시 함수(SHA), CRC 계산의 핵심 연산으로, 1클럭에 수행 가능
> 3. **융합**: 보안(Cipher), 통신(CRC), 그래픽스(비트 조작)의 기본 연산으로 하드웨어 가속 대상

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

순환 시프트(Rotate, Circular Shift)는 비트열을 왼쪽이나 오른쪽으로 이동시키되, 한쪽 끝에서 밀려난 비트가 반대쪽 끝으로 들어오는 시프트 연산이다. 어떤 비트도 손실되지 않으며, n비트 레지스터를 n번 순환하면 원래 상태로 돌아온다.

### 💡 비유: 원형 컨베이어 벨트

순환 시프트는 **원형 컨베이어 벨트**와 같다. 벨트의 물건이 한쪽으로 이동하면 반대쪽에서 다시 나타난다. 어떤 물건도 떨어지지 않고 계속 순환한다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 순환 시프트 수식

```
┌─────────────────────────────────────────────────────────────────────┐
│                순환 시프트 수학적 표현                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  n비트 값 x를 k비트만큼 순환                                       │
│  ───────────────────────────                                       │
│                                                                     │
│  ROL(x, k) = ((x << k) | (x >> (n - k))) & ((1 << n) - 1)         │
│  ROR(x, k) = ((x >> k) | (x << (n - k))) & ((1 << n) - 1)         │
│                                                                     │
│  예시 (8비트):                                                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x = 10110011 (179)                                          │ │
│  │  k = 3                                                        │ │
│  │                                                               │ │
│  │  ROL(x, 3):                                                   │ │
│  │  x << 3 = 10011000 (152)   ← 상위 3비트(101) 손실            │ │
│  │  x >> 5 = 00000101 (5)     ← 하위 3비트(011) 보존            │ │
│  │  OR    = 10011101 (157)                                      │ │
│  │                                                               │ │
│  │  검증: 10110011 → 10011101                                   │ │
│  │        101 | 10011 → 10011101 ✓                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### C 언어 구현

```c
#include <stdint.h>

// 8비트 왼쪽 순환
uint8_t rol8(uint8_t x, int k) {
    k &= 7;  // k를 0~7 범위로
    return (x << k) | (x >> (8 - k));
}

// 8비트 오른쪽 순환
uint8_t ror8(uint8_t x, int k) {
    k &= 7;
    return (x >> k) | (x << (8 - k));
}

// 32비트 왼쪽 순환
uint32_t rol32(uint32_t x, int k) {
    k &= 31;
    return (x << k) | (x >> (32 - k));
}

// 32비트 오른쪽 순환
uint32_t ror32(uint32_t x, int k) {
    k &= 31;
    return (x >> k) | (x << (32 - k));
}

// GCC/Clang 내장 함수
// uint32_t __builtin_rol(uint32_t x, int k);
// uint32_t __builtin_ror(uint32_t x, int k);

// MSVC 내장 함수
// unsigned __int64 __rotl64(unsigned __int64 value, unsigned char shift);
// unsigned __int64 __rotr64(unsigned __int64 value, unsigned char shift);
```

```
│  │                                                               │ │
│  │  S=0 (0회전): [D7 D6 D5 D4 D3 D2 D1 D0]                       │ │
│  │                                                               │ │
│  │  S=1 (1회전): [D6 D5 D4 D3 D2 D1 D0 D7]  ← D7이 끝으로       │ │
│  │                                                               │ │
│  │  S=2 (2회전): [D5 D4 D3 D2 D1 D0 D7 D6]                       │ │
│  │                                                               │ │
│  │  S=7 (7회전): [D0 D7 D6 D5 D4 D3 D2 D1]                       │ │
│  │                                                               │ │
│  │  다단계 멀티플렉서 구조 (log₂(n) 단계):                       │ │
│  │                                                               │ │
│  │  Stage 1 (S[0]): 0 또는 1비트 회전                           │ │
│  │  Stage 2 (S[1]): 0 또는 2비트 회전                           │ │
│  │  Stage 3 (S[2]): 0 또는 4비트 회전                           │ │
│  │                                                               │ │
│  │  예: S=5 (101) → Stage 1(1) + Stage 2(0) + Stage 3(4) = 5   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: SHA-256 구현**
- 요구: 해시 계산 고속화
- 구현: ROR 연산을 하드웨어 명령어 사용
- 효과: 20% 성능 향상

**시나리오 2: 난수 생성 (LFSR)**
- 요구: 의사 난수 생성
- 구현: 순환 레지스터 + XOR
- 용도: CRC, 스크램블링

**시나리오 3: 비트마스크 생성**
- 요구: n비트 마스크 순환
- 구현: `mask = rol(mask, 1)`
- 용도: 비트 순회

### 안티패턴: 일반 시프트로 순환 구현

```c
// ❌ 비효율적: 여러 연산 필요
uint32_t rotate_wrong(uint32_t x, int k) {
    uint32_t saved = (x >> k) & ((1 << k) - 1);  // 하위 k비트 저장
    x >>= k;
    x |= (saved << (32 - k));
    return x;
}

// ✅ 효율적: 단일 식
uint32_t rotate_right(uint32_t x, int k) {
    return (x >> k) | (x << (32 - k));
}

// ✅ 가장 효율적: 컴파일러 내장 함수
uint32_t rotate_best(uint32_t x, int k) {
    #if defined(__GNUC__)
    return __builtin_rotateright32(x, k);
    #elif defined(_MSC_VER)
    return _rotr(x, k);
    #else
    return (x >> k) | (x << (32 - k));
    #endif
}
```

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 순환 시프트는 원형으로 연결된 기차 같아요! 맨 앞 칸이 이동하면 맨 뒤로 가고, 어떤 칸도 사라지지 않아요!

**효과**: 암호를 만들 때 아주 중요해요! 비트를 섞어도 어떤 것도 잃어버리지 않아서, 나중에 다시 풀 수 있어요!
