# 119. 시프트 연산 (Shift Operation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비트열을 좌우로 이동시키는 연산으로, 2의 거듭제곱 곱셈/나눗셈, 비트 필드 조작, 곱셈기 구현의 기본 연산
> 2. **가치**: 단일 클럭에 수행되는 고속 연산으로, 하드웨어 곱셈기/제수기, 암호화 알고리즘, 비트 마스킹의 핵심
> 3. **융합**: 컴파일러 최적화(곱셈→시프트), 그래픽스(색상 채널), 네트워크(비트 필드 파싱)에 광범위 활용

## Ⅰ. 개요 (Context & Background)

### 개념 정의

시프트 연산(Shift Operation)은 비트열의 모든 비트를 왼쪽 또는 오른쪽으로 지정된 횟수만큼 이동시키는 연산이다. 빈 자리는 0으로 채우거나(논리 시프트), 부호 비트로 채우거나(산술 시프트), 순환시킨다(회전).

### 💡 비유: 엘리베이터 승객 이동

시프트 연산은 **엘리베이터에서 승객이 이동하는 것**과 같다. 왼쪽 시프트는 모두 왼쪽으로 한 칸씩 이동하고, 가장 왼쪽 사람은 내리고 오른쪽에는 새 사람(0)이 들어온다. 오른쪽 시프트는 반대다.

### 등장 배경

| 구분 | 내용 |
|:---|:---|
| **기원** | 초기 컴퓨터의 효율적 곱셈/나눗셈 |
| **발전** | 바렐 시프터(Barrel Shifter)로 상수 클럭 구현 |
| **현재** | 모든 CPU의 기본 명령어 |

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 시프트 연산 비교

| 연산 | 기호 | 빈 자리 채움 | 용도 |
|:---|:---:|:---|:---|
| **SHL** | `<<` | 0 | ×2^n, 비트 조작 |
| **SHR** | `>>` | 0 | ÷2^n (부호 없음) |
| **SAR** | `>>>` | 부호 비트 | ÷2^n (부호 있음) |
| **ROL** | `<<<` | 순환 | 암호화, CRC |
| **ROR** | `>>>` | 순환 | 암호화, CRC |

### 시프트를 이용한 산술

```c
#include <stdint.h>

// 시프트를 이용한 곱셈
uint32_t multiply_by_shift(uint32_t a, uint32_t b) {
    uint32_t result = 0;
    int shift = 0;

    while (b) {
        if (b & 1) {
            result += (a << shift);
        }
        b >>= 1;
        shift++;
    }
    return result;
}

// 시프트를 이용한 나눗셈 (몫)
uint32_t divide_by_shift(uint32_t dividend, uint32_t divisor) {
    if (divisor == 0) return 0;  // 에러

    uint32_t quotient = 0;
    int shift = 31;

    // 가장 큰 시프트부터 시도
    while (shift >= 0) {
        if ((divisor << shift) <= dividend) {
            dividend -= (divisor << shift);
            quotient |= (1 << shift);
        }
        shift--;
    }
    return quotient;
}

// 2의 거듭제곱 곱셈/나눗셈
uint32_t mul_pow2(uint32_t x, int n) { return x << n; }
uint32_t div_pow2(uint32_t x, int n) { return x >> n; }
int32_t  div_pow2_signed(int32_t x, int n) {
    // 산술 시프트 (부호 유지)
    return x >> n;  // C 표준: 구현 정의, 보통 산술 시프트
}
```

│  │  └───────────────────────────┘                                │   │
│  │  │   │   │   │   │   │   │   │                               │   │
│  │  ▼   ▼   ▼   ▼   ▼   ▼   ▼   ▼  ← Stage 1 결과              │   │
│  │                                                               │   │
│  │  단계 2: S[1]=1이면 2비트 시프트, 아니면 그대로              │   │
│  │  ┌───────────────────────────┐                                │   │
│  │  │     MUX (S[1])            │                                │   │

│  │  ┌───────────────────────────┐                                │   │
│  │  │     MUX (S[2])            │                                │   │
│  │  │  0: 그대로 / 1: 4비트     │                                │   │
│  │  └───────────────────────────┘                                │   │
│  │  ▼   ▼   ▼   ▼   ▼   ▼   ▼   ▼                               │   │
│  │  Y7  Y6  Y5  Y4  Y3  Y2  Y1  Y0  ← 최종 출력                │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  예: S=5 (101₂) = 1 + 4 = 5비트 시프트                            │
│  • Stage 1: S[0]=1 → 1비트 시프트                                  │
│  • Stage 2: S[1]=0 → 그대로                                        │
│  • Stage 3: S[2]=1 → 4비트 시프트                                  │
│  • 총 시프트: 1 + 4 = 5비트                                        │
└─────────────────────────────────────────────────────────────────────┘

### 시프트 응용: 비트 조작

```c
// 비트 마스킹
#define BIT_SET(x, n)    ((x) | (1 << (n)))
#define BIT_CLEAR(x, n)  ((x) & ~(1 << (n)))
#define BIT_TOGGLE(x, n) ((x) ^ (1 << (n)))
#define BIT_CHECK(x, n)  (((x) >> (n)) & 1)

// RGB 색상 추출
uint8_t get_red(uint32_t rgb)   { return (rgb >> 16) & 0xFF; }
uint8_t get_green(uint32_t rgb) { return (rgb >> 8) & 0xFF; }
uint8_t get_blue(uint32_t rgb)  { return rgb & 0xFF; }

// 바이트 추출 (엔디안 독립적)
uint8_t get_byte(uint32_t x, int n) {
    return (x >> (n * 8)) & 0xFF;
}
```

### 시프트 vs 곱셈/나눗셈

| 연산 | 시프트 | 곱셈/나눗셈 |
|:---|:---:|:---:|
| **지연** | 1클럭 | 3~32클럭 |

| **2의 거듭제곱** | ✅ 동등 | ✅ 동등 |
| **일반 수** | ❌ 불가 | ✅ 가능 |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: 컴파일러 최적화**
- 상황: `x * 1024`를 효율적으로 컴파일
- 변환: `x << 10`
- 효과: 1클럭 vs 수십 클럭

**시나리오 2: 고정소수점 연산**
- 상황: Q15.16 형식 곱셈
- 구현: `(a * b) >> 16`
- 용도: 임베디드 오디오/비디오

**시나리오 3: 해시 함수 구현**
- 상황: SHA-256 σ 함수
- 구현: ROTR(x, n) = (x >> n) | (x << (32-n))
- 용도: 보안, 무결성

### 안티패턴: 부호 있는 나눗셈에 논리 시프트

```c
// ❌ 잘못된 코드
int divide_by_4(int x) {
    return x >> 2;  // 부호 있는 수에서 문제
}
// -7 >> 2 = -2 (구현 의존적, 원하는 -1이 아닐 수 있음)

// ✅ 올바른 코드
int divide_by_4(int x) {
    return x / 4;  // 컴파일러가 최적화
}
// 또는
int divide_by_4_arith(int x) {
    // 산술 시프트 (C 표준은 구현 정의)
    #if defined(__GNUC__) || defined(__clang__)
    return x >> 2;  // GCC/Clang은 산술 시프트 보장
    #else
    return x / 4;
    #endif
}
```

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 시프트는 숫자의 자릿수를 옮기는 것이에요! 123을 왼쪽으로 한 칸 옮기면 1230이 되고(×10), 오른쪽으로 한 칸 옮기면 12가 돼요(÷10).

**원리**: 이진수에서 왼쪽으로 한 칸 가면 2배가 되고, 오른쪽으로 한 칸 가면 반반이 돼요. 빈 자리는 0으로 채워요!

**효과**: 컴퓨터가 ×2, ÷2 같은 계산을 아주 빠르게 할 수 있어요. 곱하기보다 시프트가 훨씬 쉬워요!
