> **핵심 인사이트**
> 1. IEEE 754는 과거 제조사마다 제멋대로였던 부동소수점 처리 방식을 통일하기 위해 만든 **전 세계 컴퓨터 시스템의 '실수(Real Number) 표현 절대 표준 규격'**이다.
> 2. 정규화(Normalization), 바이어스(Bias), 숨겨진 비트(Hidden Bit)라는 세 가지 기발한 압축 마법을 사용하여 **저장 공간의 효율성을 극한으로 끌어올렸다.**
> 3. 단순한 숫자뿐만 아니라, 0으로 나누기 에러를 처리하기 위한 무한대($\pm \infty$)와 숫자가 아님을 뜻하는 $NaN$(Not a Number) 같은 **특수 상태 코드까지 완벽하게 정의**하고 있다.

---

## Ⅰ. 저장 공간을 아끼는 마법: 정규화와 Hidden Bit
이진수 실수를 메모리에 넣기 전, 무조건 **정규화(Normalization)**를 거칩니다. 소수점을 이동시켜서 맨 앞에 딱 '1' 하나만 남기는 것입니다.
* 예: `1011.01` -> `1.01101` $\times 2^3$
* 예: `0.0011` -> `1.1` $\times 2^{-3}$

이렇게 하면 이진수 과학적 표기법은 **항상 "1. xxxx" 모양**으로 시작하게 됩니다. 십진수는 맨 앞이 1~9로 다양하지만, 2진수는 무조건 1밖에 올 수 없습니다.
IEEE 754 위원회는 여기서 천재적인 결정을 합니다. **"어차피 무조건 '1.'으로 시작할 거면, 메모리에 1을 저장하지 말고 빼버리자! 그리고 남은 1비트 공간에 알맹이(가수)를 하나 더 넣어서 정밀도를 높이자!"** 이것이 바로 저장 효율을 극대화한 **Hidden Bit(숨겨진 비트)** 원리입니다.

```ascii
[Hidden Bit 압축 원리]

 숫자: 1.01101 x 2^3

 IEEE 754 가수부 저장: [ 01101000...000 ] (맨 앞의 1. 은 버리고 저장함)
 나중에 CPU가 읽을 때: "아 맞다, 맨 앞에 1. 이 생략되어 있지!" 하고 알아서 붙여서 계산함.
```

📢 **섹션 요약 비유**: Hidden Bit는 '주민등록번호의 1900년대 생략'과 같습니다. 예전에 주민번호 앞자리를 적을 때 어차피 다 1900년대생이니까 '1985'라고 다 적지 않고 '85'라고만 적어서 종이 칸(비트)을 아꼈던 것과 완벽히 동일한 발상입니다.

## Ⅱ. 음수 지수 처리를 위한 바이어스(Bias)
소수점을 뒤로 밀면 지수가 음수가 됩니다. ($2^{-3}$)
지수를 저장하는 칸(8비트)에 이 음수를 어떻게 표현할까요? 2의 보수를 쓸 수도 있지만, IEEE 754는 크기 비교기(Comparator) 하드웨어를 단순하게 만들기 위해 **바이어스(Bias)**라는 편법을 씁니다.

지수부 8비트가 가질 수 있는 값은 0~255입니다. 무조건 이 범위 안에 들어가도록, **진짜 지수 값에 '+127(바이어스)'을 무조건 더해서 메모리에 저장**합니다. (배정밀도 64비트의 바이어스는 +1023)
* 실제 지수가 3이면? -> $3 + 127 = 130$ 을 이진수(`10000010`)로 저장.
* 실제 지수가 -3이면? -> $-3 + 127 = 124$ 를 이진수(`01111100`)로 저장.

이렇게 하면 지수부에 음수가 아예 사라지고 오직 양수만 남기 때문에, 두 실수의 크기를 비교할 때 그냥 이진수 자체가 큰 놈이 더 큰 수라고 직관적으로 계산(하드웨어 단순화)할 수 있습니다.

```ascii
[지수 바이어스(Bias)의 크기 비교 마법]

 -3 (124 저장) < 3 (130 저장)
 -> 컴퓨터는 복잡한 2의 보수 계산 없이, 그냥 저장된 숫자(124 vs 130) 크기만
    비교해도 지수의 대소를 0.001초만에 판별할 수 있음!
```

📢 **섹션 요약 비유**: 바이어스는 '골프장 핸디캡'입니다. 마이너스 점수(음수 지수)를 가진 초보자와 플러스 점수(양수 지수)를 가진 고수가 대결할 때 헷갈리니까, 모두에게 기본 점수 127점(Bias)을 얹어줘서 모든 점수를 플러스 양수로 만들어버린 뒤 아주 편하게 순위를 매기는 것입니다.

## Ⅲ. 아키텍처 관점: 무한대와 NaN (예외 처리)
수학에서 0으로 나누기($1 \div 0$)를 하거나, 음수의 루트를 구하면($\sqrt{-1}$) 계산이 불가능하여 과거의 컴퓨터는 에러를 뿜으며 뻗어버렸습니다.
IEEE 754는 칩이 죽는 것을 막기 위해, 지수부의 특정 숫자 조합을 아예 **'오류 메시지 전용 코드'**로 빼두었습니다.

* **무한대 (Infinity)**: 지수부가 모두 1(`11111111`)이고 가수부가 모두 0일 때. ($1 \div 0$ 같은 연산의 결과)
* **NaN (Not a Number)**: 지수부가 모두 1(`11111111`)이고 가수부가 0이 아닐 때. ($\sqrt{-1}$ 이나 $\infty - \infty$ 같은 불가능한 연산의 결과)

CPU의 FPU(부동소수점 연산장치)는 계산 결과가 무한대나 NaN이 되면 멈추지 않고 이 특별한 코드를 다음 연산으로 넘깁니다. 그러면 소프트웨어(운영체제)가 이 코드를 보고 우아하게 예외 처리(Exception)를 할 수 있습니다.

```ascii
[IEEE 754의 예외 상태 표현법]

 [ 부호 ] [ 지수부 (8비트) ] [ 가수부 (23비트) ]
    0       1111 1111        0000...000      -> +무한대 (+Infinity)
    1       1111 1111        0000...000      -> -무한대 (-Infinity)
    X       1111 1111        (0이 아닌 값)    -> NaN (숫자 아님! 에러 발생!)
```

📢 **섹션 요약 비유**: 무한대와 NaN 코드는 택배 기사님의 '반송 스티커'입니다. 배달을 갔는데 집이 없거나 주소가 엉망이라 배달(계산)을 못 하게 생겼을 때, 트럭을 부수고 우는 대신 상자에 노란색 반송 스티커(NaN)를 딱 붙여서 본사(운영체제)로 깔끔하게 돌려보내는 위기 대처 매뉴얼입니다.

## 개념 맵 (Concept Map)
```ascii
[IEEE 754 부동소수점 표준]
   │
   ├── 메모리 최적화: Hidden Bit
   │     └── 정규화(1.xx)된 숫자의 MSB '1'을 생략하여 가수부 1비트의 정밀도 추가 확보
   │
   ├── 하드웨어 최적화: Bias (바이어스) 처리
   │     └── 실제 지수에 +127(단정밀도)을 더해 저장 -> 음수 지수를 없애 하드웨어 대소 비교 로직 단순화
   │
   └── 시스템 안정성: 예외(Exception) 값 정의
         ├── Infinity(무한대): 지수 All 1, 가수 All 0 (Division by Zero 방어)
         └── NaN(Not a Number): 지수 All 1, 가수 Not 0 (연산 불능 방어)
```

## 어린이 비유
IEEE 754는 전 세계 컴퓨터들이 소수를 다룰 때 싸우지 않게 만든 **우주 대통합 법전**이에요! 예전에는 미국 컴퓨터랑 한국 컴퓨터가 소수를 적는 법이 달라서 서로 편지를 주면 에러가 났어요. 그런데 이 법전에서 "맨 앞의 1은 무조건 빼고 쓰자(Hidden bit)!", "음수에는 127을 무조건 더하자(Bias)!"라고 규칙을 딱 정해주니까, 이제는 전 세계 어떤 로봇이든 완벽하게 똑같이 소수를 읽고 계산할 수 있게 되었답니다!
