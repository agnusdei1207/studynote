> **핵심 인사이트**
> 1. 진법(Numeral System)은 사용할 수 있는 **숫자의 개수와 자리올림의 기준(기수, Radix/Base)**을 정의하는 수학적 체계이다.
> 2. 컴퓨터는 스위치(트랜지스터)의 On/Off라는 물리적 특성 때문에 **오직 2진법(0, 1)**만을 근본적인 언어로 사용한다.
> 3. 8진법과 16진법은 인간이 끝없이 길어지는 2진수를 읽고 쓰기 쉽게 **3비트(8진수)나 4비트(16진수) 단위로 압축해서 묶어보는 프로그래머용 편의 도구**이다.

---

## Ⅰ. 컴퓨터의 언어: 2진법 (Binary)
인간은 손가락이 10개라서 0부터 9까지 10개의 기호를 쓰는 10진법(Decimal)에 익숙합니다. 반면, 컴퓨터를 구성하는 트랜지스터는 오직 '전기가 통한다(1)'와 '안 통한다(0)' 두 가지 상태만 존재합니다. 따라서 컴퓨터는 밑(Base)이 2인 **2진법(Binary)**을 쓸 수밖에 없습니다.

2진법에서는 숫자가 2가 되는 순간 해당 자리의 용량이 꽉 차서 윗자리로 올림(Carry)이 발생합니다.
* 10진수: $0 \rightarrow 1 \rightarrow 2 \rightarrow 3 \rightarrow 4$
* 2진수: $0 \rightarrow 1 \rightarrow 10 \rightarrow 11 \rightarrow 100$

```ascii
[진법별 자리의 가치 (가중치)]

 십진수 (Base 10) : 345 = (3 * 10^2) + (4 * 10^1) + (5 * 10^0)
 이진수 (Base 2)  : 101 = (1 * 2^2)  + (0 * 2^1)  + (1 * 2^0) = 십진수 5
```

📢 **섹션 요약 비유**: 10진법은 '10개들이 계란판'입니다. 계란이 10개 차면 새 판을 꺼냅니다. 2진법은 '2개들이 작은 초콜릿 상자'입니다. 초콜릿이 딱 2개만 들어가면 꽉 차버려서 계속해서 새 상자를 꺼내야 하므로 상자(자릿수)가 엄청나게 길어집니다.

## Ⅱ. 압축의 마법: 8진법(Octal)과 16진법(Hexadecimal)
2진법은 컴퓨터에겐 좋지만 인간에게는 끔찍합니다. 십진수 1,000을 2진수로 쓰면 `1111101000`입니다. 너무 길고 읽다 보면 눈이 핑핑 돕니다.
이때 $8 = 2^3$, $16 = 2^4$ 이라는 수학적 사실이 구세주가 됩니다. **2진수를 3개씩 끊어 읽으면 완벽한 8진수가 되고, 4개씩 끊어 읽으면 완벽한 16진수가 됩니다.**

* **8진법 (0~7 사용)**: 유닉스/리눅스의 파일 권한(예: `chmod 755`) 설정 등 3비트 묶음이 편한 곳에 주로 쓰입니다.
* **16진법 (0~9, A, B, C, D, E, F 사용)**: 10부터 15까지는 알파벳 A~F로 대체합니다. 4비트는 반 바이트(Nibble)이므로, **1바이트(8비트)를 정확히 16진수 두 글자로 압축**할 수 있습니다. (예: `1111 1111` -> `FF`). 이 완벽한 압축비율 때문에 현대 컴퓨터 아키텍처(메모리 주소, 색상 코드, MAC 주소)는 모두 16진수를 표준으로 씁니다.

```ascii
[이진수를 16진수로 압축하는 방법]

 이진수 1바이트: 1 0 1 1  0 1 1 0
                 ---+---  ---+---
                    |        |
 4자리씩 끊기:   (11)     (6)    -> 10진수 환산
 16진수 기호:      B        6

 압축 결과: 0xB6 (앞에 0x를 붙여 16진수임을 표시)
```

📢 **섹션 요약 비유**: 이진수는 '100원짜리 동전 수천 개'입니다. 너무 무겁고 세기 힘듭니다. 16진수는 이걸 은행에 가서 '만 원짜리 지폐'로 교환한 것입니다. 가치(데이터)는 완전히 똑같지만, 인간이 지갑에 넣고 다니거나 숫자를 세기가 훨씬 편해진 것입니다.

## Ⅲ. 아키텍처 관점: 진법 변환과 하드웨어의 무관함
소프트웨어 초보자들이 흔히 하는 착각이 "컴퓨터 안에 16진수 회로나 10진수 회로가 따로 있나?" 하는 것입니다.
**절대 아닙니다.** CPU 연산장치(ALU), 메모리, 하드디스크 내부에는 오직 전압의 높고 낮음(2진법)만이 존재합니다.

우리가 코딩할 때 `int a = 0xFF;` (16진수)라고 치든 `int a = 255;` (10진수)라고 치든, 컴파일러는 이를 무조건 `11111111`(2진수)로 싹 다 번역해서 CPU에 밀어 넣습니다. 그리고 계산이 끝나고 모니터에 결과를 띄워줄 때(I/O 장치), 출력 프로그램이 다시 2진수를 인간이 보기 편한 10진수나 16진수 글자 폰트로 '포장'해서 보여줄 뿐입니다. 진법은 철저히 인간과 기계의 인터페이스(UI) 문제일 뿐, 아키텍처 내부의 물리적 본질은 영원히 2진법입니다.

```ascii
[시스템 내 진법의 흐름]

 인간(키보드)    -> [ 컴파일러 번역 ] -> 하드웨어(CPU/RAM) -> [ I/O 번역기 ] -> 인간(모니터)
 입력: 255 (10진)                      처리: 11111111 (2진)                    출력: FF (16진)
 "십진수로 넣을게!"                    "난 이진수로 더할게"                  "니들 편하게 16진수로 봐라"
```

📢 **섹션 요약 비유**: 진법은 '언어 번역기'입니다. 컴퓨터라는 외국인은 오직 '이진어(0과 1)'밖에 할 줄 모릅니다. 우리가 어떤 화려한 언어(16진법, 10진법)로 말을 걸어도 통역사(소프트웨어)가 이진어로 바꿔서 전달하고, 컴퓨터가 이진어로 대답한 것을 다시 우리 언어로 예쁘게 번역해서 스피커로 들려주는 것뿐입니다.

## 개념 맵 (Concept Map)
```ascii
[진법 체계 (Numeral System)]
   │
   ├── 2진법 (Binary): 밑(Base)=2, 숫자 0~1
   │     └── 컴퓨터 하드웨어(트랜지스터 On/Off)의 절대적인 물리적 언어
   │
   ├── 8/16진법 (Octal / Hexadecimal): 인간을 위한 이진수 압축 표기법
   │     ├── 8진법: 2진수 3비트를 1자리로 압축 (0~7)
   │     └── 16진법: 2진수 4비트(Nibble)를 1자리로 압축 (0~F) -> 1바이트를 2글자로 표현 (표준)
   │
   └── 아키텍처적 시사점
         └── H/W 내부는 100% 2진 처리, 8/10/16진법 변환은 S/W(컴파일러, I/O) 단계의 추상화 계층
```

## 어린이 비유
컴퓨터는 '네(1)'와 '아니오(0)'밖에 말하지 못하는 **이진법 외계인**이에요. 우리가 숫자 '15'를 알려주면 외계인은 "네-네-네-네(1111)"라고 엄청 길게 중얼거려요. 이 소리가 너무 길고 헷갈려서, 과학자 삼촌들이 "외계인이 네-네-네-네 라고 네 번 말하면, 우리는 그냥 알파벳 'F'(16진법)라고 짧게 적자!"라고 우리끼리 약속을 만든 거랍니다. 외계인 머릿속은 여전히 0과 1뿐이에요!
