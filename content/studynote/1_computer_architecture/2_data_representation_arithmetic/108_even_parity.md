# 108. 짝수 패리티 (Even Parity)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 비트와 패리티 비트를 합쳤을 때 1의 총 개수가 짝수가 되도록 패리티 비트를 설정하는 방식
> 2. **가치**: 1의 개수가 홀수가 되면 즉시 오류 감지, 하드웨어 구현이 XOR 게이트 하나로 가능한 가장 단순한 오류 검출
> 3. **융합**: 대부분의 통신 프로토콜·메모리 시스템의 기본 설정으로, 0x00(모두 0)의 올바른 전송 보장

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

짝수 패리티(Even Parity)는 데이터 비트열에 1개의 패리티 비트를 추가하여, 전체 비트(데이터 + 패리티)에서 1의 개수가 항상 짝수가 되도록 하는 오류 검출 방식이다. 데이터 비트의 1의 개수가 홀수면 패리티 비트를 1로, 짝수면 0으로 설정한다.

### 💡 비유: 짝수만 받는 엘리베이터

짝수 패리티는 **짝수 층만 멈추는 엘리베이터**와 같다. 건물에 들어갈 때 정확히 짝수 명이 타야 한다고 규칙을 정하면, 내릴 때 인원이 홀수면 누가 빠졌거나 새로 탄 것이다. 단, 2명이 동시에 들어오거나 나가면 여전히 짝수라서 변화를 모른다.

### 등장 배경

| 구분 | 내용 |
|:---|:---|
| **개발 목적** | 단순하고 빠른 1비트 오류 검출 |
| **주요 사용처** | 메모리, 직렬 통신, 저장 장치 |
| **홀수 패리티와 차이** | 0x00 전송 시 올바름 검증 가능 |

```
┌─────────────────────────────────────────────────────────────────────┐
│                    짝수 패리티 동작 원리                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  규칙: 전체 비트에서 1의 개수 = 짝수 (0, 2, 4, 6, 8...)            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터         1의 개수   패리티 비트   전체       1의 개수  │ │

│  │  0000000        0 (짝수)      0        00000000     0 (짝수)  │ │
│  │  0000001        1 (홀수)      1        10000001     2 (짝수)  │ │
│  │  0100001        2 (짝수)      0        00100001     2 (짝수)  │ │
│  │  0111111        6 (짝수)      0        00111111     6 (짝수)  │ │
│  │  1111111        7 (홀수)      1        11111111     8 (짝수)  │ │
│  └───────────────────────────────────────────────────────────────┘ │

│  패리티 비트 결정 공식:                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  P = D0 ⊕ D1 ⊕ D2 ⊕ D3 ⊕ D4 ⊕ D5 ⊕ D6 ⊕ D7                 │ │

│  │                    데이터 비트들의 XOR                        │ │
│  │                                                               │ │
│  │  • 데이터 1의 개수가 홀수면 P = 1 (총 짝수)                  │ │
│  │  • 데이터 1의 개수가 짝수면 P = 0 (총 짝수)                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 소프트웨어 구현

```c
#include <stdint.h>

// 짝수 패리티 비트 생성
uint8_t even_parity_generate(uint8_t data) {
    // 1의 개수 세기 (Brian Kernighan 알고리즘)
    uint8_t count = 0;
    uint8_t temp = data;
    while (temp) {
        count++;
        temp &= (temp - 1);  // 최하위 1 비트 제거
    }
    return count & 1;  // 홀수면 1, 짝수면 0
}

// 짝수 패리티 검사 (데이터 + 패리티)
int even_parity_check(uint8_t data_with_parity) {
    uint8_t count = 0;
    uint8_t temp = data_with_parity;
    while (temp) {
        count++;
        temp &= (temp - 1);
    }
    // 짝수면 정상(0), 홀수면 오류(1)
    return count & 1;
}

// XOR를 이용한 더 효율적인 구현
uint8_t even_parity_generate_fast(uint8_t data) {
    data ^= data >> 4;
    data ^= data >> 2;
    data ^= data >> 1;
    return data & 1;
}
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 짝수 패리티 vs 홀수 패리티

| 항목 | 짝수 패리티 | 홀수 패리티 |
|:---|:---|:---|
| **1의 개수** | 짝수 | 홀수 |
| **P = XOR(데이터)** | 그대로 사용 | NOT 후 사용 |
| **0x00 전송** | 정상 (1이 0개=짝수) | 오류 (1이 0개≠홀수) |
| **0xFF 전송** | 정상 (1이 8개=짝수) | 오류 (1이 8개≠홀수) |
| **주요 사용처** | 대부분의 시스템 | 일부 통신 시스템 |

### 짝수 패리티가 선호되는 이유

```
┌─────────────────────────────────────────────────────────────────────┐
│                짝수 패리티 선호 이유                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ① 0x00 (널 바이트) 올바른 전송 검증                               │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  짝수 패리티: 00000000 → 1의 개수 0 (짝수) → 정상 ✓          │ │
│  │  홀수 패리티: 00000000 → 1의 개수 0 (짝수) → 오류 ✗          │ │
│  │                                                               │ │
│  │  → 문자열 종료(null), 패딩 등 0x00 전송이 빈번               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ② 하드웨어 구현 단순                                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  짝수: P = D0 ⊕ D1 ⊕ ... ⊕ Dn                                │ │
│  │  홀수: P = ¬(D0 ⊕ D1 ⊕ ... ⊕ Dn)  ← NOT 게이트 추가 필요    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ③ 표준 프로토콜 준수                                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • PCI 버스: 짝수 패리티                                      │ │
│  │  • 대부분의 UART: 짝수 패리티 기본값                          │ │
│  │  • ISA 버스: 짝수 패리티                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: UART 통신 설정**
- 상황: 마이크로컨트롤러 UART 통신
- 설정: 8E1 (8데이터, Even 패리티, 1 스탑 비트)
- 효과: 노이즈 환경에서 1비트 오류 즉시 검출

**시나리오 2: 메모리 패리티 체크**
- 상황: 구형 SIMM 메모리 (패리티 있는 9비트)
- 동작: 8비트 데이터 + 1비트 짝수 패리티
- NMI: 패리티 오류 시 Non-Maskable Interrupt 발생

### 도입 체크리스트

| 체크 항목 | 확인 사항 |
|:---|:---|
| ☐ 데이터 폭 | 7비트, 8비트 등 데이터 비트 수 확인 |
| ☐ 전체 비트 수 | 데이터 + 패리티 = 8, 9, 16, 17 등 |
| ☐ 0x00 처리 | 널 바이트 전송 필요 시 짝수 패리티 필수 |
| ☐ 오류 처리 | 오류 검출 시 재전송/중단 정책 |

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 짝수 패리티는 발에 신발이 항상 짝수 개여야 한다는 규칙과 같아요. 신발이 한 짝이면(홀수) 뭔가 잘못된 거예요!

**원리**: 1이라는 숫자가 몇 개 있는지 세어서, 홀수 개면 0을, 짝수 개면 1을 더해서 무조건 짝수로 만들어요.

**효과**: 컴퓨터가 데이터를 받을 때 1의 개수가 홀수면 "어? 뭔가 잘못됐네!" 하고 바로 알 수 있어요.
