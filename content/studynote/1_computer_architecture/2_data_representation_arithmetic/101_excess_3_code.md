# 101. 3초과 코드 (Excess-3 Code)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 3초과 코드(Excess-3 Code, XS-3)는 8421 BCD(Binary Coded Decimal) 코드에 상수 3을 더하여 생성하는 **자기 보수 코드(Self-Complementing Code)**입니다. 0~9의 십진수를 4비트의 3~12 범위에 매핑하여, 비트 반전만으로 9의 보수를 얻는 수학적 구조를 제공합니다.
> 2. **가치**: 초기 컴퓨터(UNIVAC I 등)의 ALU(Arithmetic Logic Unit) 설계에서 뺄셈 회로의 복잡도를 획기적으로 낮춘 핵심 기술입니다. NOT 게이트만으로 뺄셈 논리를 구현하여 하드웨어 비용을 절감하고, 사용하지 않는 6개의 비트 조합(Invalid States)을 통해 오류를 감지할 수 있는 내장형 안전 장치를 제공합니다.
> 3. **융합**: 현대의 2의 보수(Two's Complement) 기반 아키텍처에서는 산술 연산 용도로 쓰이지 않으나, 디지털 논리 회로 설계(Digital Logic Design) 교과서적으로 **Error Detection** 메커니즘과 유한 상태 머신(FSM, Finite State Machine)의 상태 천이 이해를 돕는 중요한 표준 모델로 활용됩니다.

---

### Ⅰ. 개요 (Context & Background)

**개념 및 정의**
3초과 코드(Excess-3 Code)는 10진수(Decimal Number)를 2진수(Binary Number)로 변환하는 코딩 체계 중 하나로, 속성상 **자기 보수 코드(Self-Complementing Code)**에 속합니다. 가장 대표적인 BCD 코드인 8421 코드에 상수 3(이진수 `0011`)을 더하여 표현하는 것이 특징입니다.
일반적인 8421 BCD가 `0000`(0)부터 `1001`(9)까지의 값을 갖는 것과 달리, 3초과 코드는 `0011`(0)부터 `1100`(9)까지의 값을 가집니다. 이러한 **편향(Bias)**을 가짐으로써, 모든 비트를 논리 반전(Logical Inversion)시켰을 때 원래 수에 대한 9의 보수(Nine's Complement)가 되는 독특한 수학적 성질을 가집니다.

**💡 비유**
일반적인 건물 엘리베이터가 1층부터 9층까지 직선으로 연결되는 것과 달리, **3초과 코드는 지하 3층(Bias -3)에 출발지를 둔 엘리베이터**와 같습니다. 0층(1층)에 도달하려면 3칸을 올라가야 하고(0→0011), 9층에 도달하려면 총 12칸을 올라가야 합니다(9→1100). 이 구조에서 "전체를 뒤집는 것(Inversion)"은 "지하 3층 기준에서 반대편으로 이동하는 것"과 수학적으로 동일하여, 복잡한 계산 없이도 뺄셈 연산을 단순한 반전 작업으로 처리할 수 있습니다.

**등장 배경 및 기술적 진화**
1.  **진공관 시대의 자원 제약 (1950년대 초)**: 초기 컴퓨터인 UNIVAC I(UNIVersal Automatic Computer I) 설계 당시, 하드웨어 자원(진공관, 릴레이)은 고가였고 발열이 심했습니다. BCD 방식으로 10진수 뺄셈을 수행하려면 9의 보수를 구하는 복잡한 가산기 회로가 필요했으나, 3초과 코드는 단순히 비트를 반전시키는 **Inverter(NOT Gate)** 만으로 이를 대체할 수 있어 회로 면적과 소비 전력을 획기적으로 줄였습니다.
2.  **오류 검출의 필요성**: 4비트는 16가지 상태를 표현할 수 있지만, 10진수는 10가지 상태만 필요합니다. 8421 BCD는 나머지 6가지 상태를 사용하지 않아 회로 오류 시 이를 구별하기 어려운 경우가 있었으나, 3초과 코드는 `0000`~`0010`, `1101`~`1111` 등 양 끝단의 6가지 코드를 **Invalid State(비유효 상태)**로 명확히 지정하여, 잘못된 연산 결과를 즉시 감지할 수 있는 메커니즘을 제공했습니다.
3.  **CMOS와 2의 보수의 등장**: 트랜지스터 기술이 발전하고 정수 연산의 표준이 2의 보수(Two's Complement)로 자리 잡으면서, 3초과 코드는 산술 연산용으로는 퇴조하였으나, 현재는 디지털 시스템의 **Error Detection** 및 교육용 코딩 이론(Coding Theory) 모델로 그 가치를 유지하고 있습니다.

> **📢 섹션 요약 비유**: 고가의 부품을 줄이기 위해, 모든 층의 높이를 3씩 들어 올린 후(Biasing), 내려갈 땐 그냥 바닥을 뚫어버리는 방식(Inversion)으로 뺄셈을 해결한 초기 컴퓨터 공학자들의 기발한 궁여지책입니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 및 매핑 테이블**
3초과 코드 시스템은 4비트 시스템(4-bit System) 내에서 16가지(`0000`~`1111`) 가능한 조합 중 6가지(`0000`~`0010`, `1101`~`1111`)를 **오류 코드(Invalid Code)**로 예약하고, 나머지 10가지를 유효 코드(Valid Code)로 할당합니다.

| 요소명 | 십진수 (Decimal) | 8421 BCD | 편향 연산 (Bias +3) | 3초과 코드 (XS-3) | 내부 동작 및 수학적 성질 | 비유 |
|:---:|:---:|:---:|:---:|:---:|:---|:---|
| **Zero** | 0 | 0000 | 0 + 3 | **0011** | 가중치 8421에 3을 가산. 반전 시 `1100`(9) | 지하 3층에서 0층 도달 |
| **Five** | 5 | 0101 | 5 + 3 | **1000** | 8(1000)로 표현. 반전 시 `0111`(4) | 중앙 값 (대칭점) |
| **Nine** | 9 | 1001 | 9 + 3 | **1100** | 최상위 비트 2개가 1. 반전 시 `0011`(0) | 최상층 (정상) |
| **Invalid** | - | - | - | **0000 ~ 0010** | **Underflow Zone**: 시스템 오류 플래그 | 지하 3층보다 낮음 (고장) |
| **Invalid** | - | - | - | **1101 ~ 1111** | **Overflow Zone**: 시스템 오류 플래그 | 12층보다 높음 (고장) |

**ASCII 구조 다이어그램: 비트 레이아웃 및 상태 공간 (Bit Layout & State Space)**
아래 다이어그램은 4비트 16진 상태 공간(Hexadeci-mal State Space)에서 3초과 코드가 차지하는 위치와, **Self-Complementing** 속성이 어떻게 중심축(Center Axis `15/2`)을 기준으로 대칭을 이루는지를 시각화한 것입니다.

**[다이어그램 해설]**
이 다이어그램은 3초과 코드의 핵심 메커니즘인 **비대칭적 편향(Asymmetric Bias)**과 **대칭적 반전(Symmetric Inversion)**을 보여줍니다.
1.  **편향(Bias)**: 8421 BCD 값에 `0011`(3)을 더함으로써, 2진수 연산에서 발생하기 쉬운 **Carry Lookahead**나 **Borrow** 로직의 간섭을 최소화하는 공간을 확보합니다. 특히 `0000`에서 `0001`로 넘어갈 때 모든 비트가 변하는 문제(Hazards)를 회피합니다.
2.  **자기 보수(Self-Complementing)**: 4비트 시스템에서 모든 비트를 반전($\sim x$)하면 $15-x$가 됩니다. 3초과 코드는 수식 $Code(x) = x + 3$을 가지므로, $\sim Code(x) = 15 - (x+3) = 12 - x = (9-x) + 3 = Code(9-x)$가 됩니다. 즉, 코드 자체를 반전하면 9의 보수에 대한 코드가 나옵니다. 이는 하드웨어적으로 뺄셈기를 NOT 게이트 하나로 구현 가능하게 하는 결정적인 이유입니다.

**심층 동작 원리: 3초과 코드 가산기 (Excess-3 Adder Logic)**
3초과 코드로 덧셈을 수행할 때는 두 피연산자에 이미 3이 더해져 있으므로, 결과적으로 $3+3=6$ 만큼의 초과분(Bias)이 발생합니다. 이를 보정(Correction)하기 위해 다음의 3단계 알고리즘이 적용됩니다.

1.  **Raw Addition**: 두 3초과 코드를 단순 2진수로 더한다.
2.  **Carry Check**:
    *   **Case A (Carry-out 발생 = 16 이상)**: 결과값에서 16을 빼고(Carry 처리), 모자른 만큼 보정하기 위해 **3을 더(Add 3)**한다. (총 보정: -16 + 3 = -13)
    *   **Case B (No Carry = 15 이하)**: 결과값이 6만큼 초과된 상태이므로 **3을 뺄(Sub 3)**(혹은 2진수 1101(13)을 더하여 언더플로우 유도)하여 정상화한다.
3.  **Output**: 보정된 결과는 다시 3초과 코드 형태가 된다.

```text
   [실무 레벨 회로 의사 코드 (HDL Style)]

   module excess3_adder(input [3:0] a, input [3:0] b, 
                        output [3:0] sum, output cout);
       
       wire [4:0] raw_sum = a + b;  // 5-bit for overflow detection
       
       // Correction Logic based on Carry Out
       assign cout = raw_sum[4];    // Carry Flag
       
       // If Carry exists (Result >= 16): We subtracted 16 but need +3 bias
       // If No Carry (Result < 16): We have excess 6, so we must subtract 3
       
       always @(*) begin
           if (raw_sum[4]) 
              sum = raw_sum[4:1] + 4'b0011; // Add 3 correction
           else 
              sum = raw_sum[3:0] - 4'b0011; // Sub 3 correction
       end
   endmodule
```

> **📢 섹션 요약 비유**: 모든 사원이 3cm 높이의 구두를 신고 키를 재는 회사와 같습니다. 두 사람의 키 합을 구할 땐, 6cm의 구두 굽 높이만큼 초과된 키를 다시 빼주는 복잡한 보정(Correction) 과정이 필요하지만, 대신 누군가의 키를 180도 뒤집으면(Looking in a mirror) 자연스럽게 9의 보수 사람이 되는 마법이 생깁니다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교표: BCD vs 3초과 코드 vs 그레이 코드**
디지털 시스템 설계에서 3초과 코드의 위치를 명확히 하기 위해 인접 기술들과의 구조적, 기능적 차이를 분석합니다.

| 분석 지표 (Metric) | 8421 BCD (Natural BCD) | 3초과 코드 (Excess-3 XS-3) | 그레이 코드 (Gray Code) |
|:---|:---|:---|:---|
| **정의 (Definition)** | 십진수 0~9를 0~9 2진수에 1:1 매핑 | 십진수 0~9에 3을 더하여 3~12로 매핑 | 인접한 수가 1비트만 차이나도록 매핑 |
| **가중치 (Weight)** | 가중치 있음 (8, 4, 2, 1) | **가중치 없음 (Unweighted)** | 가중치 없음 (Cyclic) |
| **자기 보수 (Self-Complement)** | **아니오 (
