# 310. 메모리 맵 I/O (Memory-Mapped I/O)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 장치를 메모리 주소 공간에 매핑하여 접근
> 2. **가치**: 단일 주소 공간, 풍부한 명령어, 단순한 하드웨어
> 3. **융합**: 주소 공간, 캐시, 페이지 테이블

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

메모리 맵 I/O(Memory-Mapped I/O)는 **I/O 장치의 레지스터를 메모리 주소 공간의 일부에 매핑하여, 일반 메모리 접근 명령어로 I/O 장치를 제어하는 방식**이다. 별도의 I/O 명령어 없이 load/store 명령어로 장치에 접근한다.

### 💡 비유: 같은 책장에 꽂기

메모리 맵 I/O는 **같은 책장에 꽂기**와 같다. 일반 책과 특별한 책(장치)을 같은 책장에 꽂는다. 읽을 때 같은 방법으로 읽는다. 다른 책장에 따로 꽂지 않는다. 위치만 다를 뿐 읽는 법은 같다.

### 메모리 맵 I/O 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 맵 I/O 개요                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 공간 구조】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  메모리 맵 I/O 주소 공간:                                            │ │
│  │                                                               │ │
│  │  ┌────────────────────────────────────────────────────┐      │ │
│  │  │ 0xFFFFFFFF  ┌─────────────────────────┐            │      │ │
│  │  │             │  I/O 장치 영역            │            │      │ │
│  │  │             │  (Memory-Mapped I/O)     │            │      │ │
│  │  │             ├─────────────────────────┤            │      │ │
│  │  │             │  ROM / Flash             │            │      │ │
│  │  │             ├─────────────────────────┤            │      │ │
│  │  │             │  RAM (실제 메모리)        │            │      │ │
│  │  │             ├─────────────────────────┤            │      │ │
│  │  │ 0x00000000  └─────────────────────────┘            │      │ │
│  │  └────────────────────────────────────────────────────┘      │ │
│  │                                                               │ │
│  │  → I/O 장치가 메모리처럼 보임                                      │ │
│  │  → 일반 메모리 명령어로 접근                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【접근 방식】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  C 코드 예시:                                                       │ │
│  │                                                               │ │
│  │  // I/O 레지스터 주소 (메모리 맵)                                    │ │
│  │  #define GPIO_BASE  0x40020000                                    │ │
│  │  #define GPIO_DATA  (*(volatile uint32_t *)(GPIO_BASE + 0x00))   │ │
│  │  #define GPIO_DIR   (*(volatile uint32_t *)(GPIO_BASE + 0x04))   │ │
│  │                                                               │ │
│  │  // 일반 변수처럼 접근                                              │ │
│  │  GPIO_DIR = 0xFF;           // 출력 방향 설정                      │ │
│  │  GPIO_DATA = 0x55;          // 데이터 출력                         │ │
│  │  value = GPIO_DATA;         // 데이터 읽기                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 맵 I/O 장단점】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 단일 주소 공간 (통합)                                             │ │
│  │  • 모든 메모리 명령어 사용 가능                                       │ │
│  │  • 하드웨어 단순 (별도 I/O 버스 불필요)                               │ │
│  │  • C 언어로 직접 접근 가능                                           │ │
│  │  • 풍부한 주소 지정 방식                                              │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 메모리 주소 공간 일부 사용                                         │ │
│  │  • 캐시 문제 (volatile 필요)                                         │ │
│  │  • 메모리 보호 복잡                                                  │ │
│  │  • 주소 디코딩 복잡                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 메모리 맵 I/O 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 맵 I/O 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 맵 I/O vs 분리형 I/O】                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  메모리 맵 I/O:                                                      │ │
│  │  ┌────────────────────────────────────────┐                    │ │
│  │  │         단일 주소 공간                    │                    │ │
│  │  │  ┌──────────┬──────────┬──────────┐   │                    │ │
│  │  │  │   RAM    │   ROM    │  I/O     │   │                    │ │
│  │  │  └──────────┴──────────┴──────────┘   │                    │ │
│  │  │                                        │                    │ │
│  │  │  접근: LOAD/STORE (일반 명령어)          │                    │ │
│  │  └────────────────────────────────────────┘                    │ │
│  │                                                               │ │
│  │  분리형 I/O (Isolated I/O):                                          │ │
│  │  ┌────────────────────────────────────────┐                    │ │
│  │  │  메모리 공간       │      I/O 공간       │                    │ │
│  │  │  ┌──────────┬─────┐  ┌──────────┐     │                    │ │
│  │  │  │   RAM    │ ROM │  │  I/O     │     │                    │ │
│  │  │  └──────────┴─────┘  └──────────┘     │                    │ │
│  │  │                                        │                    │ │
│  │  │  접근: LOAD/STORE   │  IN/OUT          │                    │ │
│  │  └────────────────────────────────────────┘                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 문제와 volatile】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  문제:                                                               │ │
│  │  • CPU가 I/O 레지스터를 캐시에 저장                                   │ │
│  │  • 쓰기가 지연되거나 순서가 바뀜                                       │ │
│  │  • 읽기가 실제 장치가 아닌 캐시에서 발생                               │ │
│  │                                                               │ │
│  │  해결:                                                               │
│  │  • volatile 키워드: 컴파일러 최적화 방지                              │ │
│  │  • Uncacheable 영역: 페이지 테이블에서 캐시 비활성화                   │ │
│  │  • 메모리 배리어: 순서 보장                                           │ │
│  │                                                               │ │
│  │  예:                                                                 │ │
│  │  volatile uint32_t *reg = (volatile uint32_t *)0x40020000;        │ │
│  │  *reg = 0x01;    // 즉시 메모리에 쓰기 (캐시 안 함)                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ARM vs x86 접근 방식】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ARM:                                                                │ │
│  │  • 메모리 맵 I/O만 사용                                               │ │
│  │  • 모든 장치를 메모리 주소로 접근                                      │ │
│  │  • 단순한 명령어 집합                                                 │ │
│  │                                                               │ │
│  │  x86:                                                                │ │
│  │  • 메모리 맵 I/O + 분리형 I/O 모두 지원                                │ │
│  │  • IN/OUT 명령어로 분리형 I/O                                         │ │
│  │  • 레거시 장치 지원                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 메모리 맵 I/O 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 맵 I/O 분석                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 맵 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/iomem                                                  │
│  00000000-00000fff : Reserved                                       │
│  00001000-0009efff : System RAM                                     │
│  000a0000-000bffff : PCI Bus 0000:00                                │
│  fe000000-feffffff : PCI MMCONFIG                                   │
│  fffc0000-ffffffff : Reserved                                       │
│                                                                     │
│  【커널에서 MMIO 매핑】                                                          │)
│  ──────────────────                                                  │
│  // Linux 커널 ioremap 예시                                          │
│  void __iomem *base;                                                │
│                                                                     │
│  // 물리 주소를 가상 주소로 매핑                                       │
│  base = ioremap(0x40020000, 0x1000);                                │
│                                                                     │
│  // MMIO 읽기/쓰기                                                   │
│  value = readl(base + REG_DATA);                                    │
│  writel(value, base + REG_DATA);                                    │
│                                                                     │
│  // 매핑 해제                                                        │
│  iounmap(base);                                                     │
│                                                                     │
│  【ARM 디바이스 트리】                                                            │)
│  ──────────────────                                                  │
│  gpio@40020000 {                                                    │
│      compatible = "vendor,gpio";                                    │
│      reg = <0x40020000 0x1000>;  // MMIO 주소 범위                    │
│      interrupts = <0 10 4>;                                         │
│  };                                                                 │
│                                                                     │
│  【MMIO 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. volatile: 컴파일러 최적화 방지                                     │
│  2. ioremap: 커널에서 MMIO 매핑                                       │
│  3. 캐시 비활성화: 페이지 테이블 설정                                   │
│  4. 메모리 배리어: 순서 보장                                           │
│  5. 접근 폭: 32비트/64비트 맞춰 접근                                    │
│  6. 레지스터 정렬: 정렬된 접근 필수                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 메모리 맵 I/O: I/O를 메모리 주소에 매핑
• 접근: 일반 load/store 명령어
• 장점: 단일 주소 공간, 풍부한 명령어, 단순 하드웨어
• 단점: 주소 공간 사용, 캐시 문제
• volatile: 컴파일러 최적화 방지
• ARM: MMIO만 사용
• x86: MMIO + 분리형 I/O 모두 지원
• ioremap: 커널 MMIO 매핑 함수
• 캐시: MMIO 페이지 캐시 비활성화
• 관련: 분리형 I/O, 주소 공간, 캐시
```
