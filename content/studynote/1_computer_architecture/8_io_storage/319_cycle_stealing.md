# 319. 사이클 스틸링 (Cycle Stealing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 버스를 안 쓸 때 DMA가 버스를 훔쳐서 사용
> 2. **가치**: CPU 방해 최소화, 효율적 버스 공유
> 3. **융합**: DMA, 버스, 메모리 사이클

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

사이클 스틸링(Cycle Stealing)은 **CPU가 버스를 사용하지 않는 순간을 이용해 DMA 컨트롤러가 버스를 빌려서(훔쳐서) 데이터를 전송하는 기법**이다. CPU의 메모리 사이클을 한 번씩 "훔쳐서" DMA 전송에 사용한다.

### 💡 비유: 틈타서 사용하기

사이클 스틸링은 **틈타서 사용하기**와 같다. 가족이 하나의 화장실을 쓴다. 아빠가 쓰다가 잠깐 나갔다. 엄마가 그 틈에 양치를 한다. 아빠가 다시 들어와서 쓴다. 화장실을 빌려서 쓴다. 서로 기다리지 않고 틈틈이 사용한다.

## Ⅱ. 사이클 스틸링 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                사이클 스틸링 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 과정】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. DMA가 버스 요청 (BR):                                            │ │
│  │     • DMA가 데이터 전송 필요                                         │ │
│  │     • BR 신호 활성화                                                │ │
│  │                                                               │ │
│  │  2. CPU가 버스 사용 중인지 확인:                                      │ │
│  │     • CPU가 메모리 접근 중이면 대기                                   │ │
│  │     • CPU가 버스 안 쓰면 승인                                        │ │
│  │                                                               │ │
│  │  3. CPU가 버스 승인 (BG):                                            │ │
│  │     • BG 신호 활성화                                                │ │
│  │     • CPU는 버스 제어 해제                                          │ │
│  │                                                               │ │
│  │  4. DMA가 한 워드 전송:                                               │ │
│  │     • 메모리 읽기/쓰기                                               │ │
│  │     • 카운트 감소                                                   │ │
│  │                                                               │ │
│  │  5. DMA가 버스 해제:                                                 │ │
│  │     • BR 신호 비활성화                                               │ │
│  │     • CPU가 버스 다시 사용                                           │ │
│  │                                                               │ │
│  │  6. 반복:                                                            │ │
│  │     • 전송 완료까지 1~5 반복                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CPU 효율 분석】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  예: 메모리 사이클 = 100ns, CPU 사용 = 90%, DMA 사용 = 10%           │ │
│  │                                                               │ │
│  │  CPU 단독: 100ns마다 메모리 접근                                    │ │
│  │  사이클 스틸링: 90ns CPU, 10ns DMA                                  │ │
│  │                                                               │ │
│  │  CPU 효율: 90% (10% 감소)                                          │ │
│  │  DMA 전송: 동시에 진행                                              │ │
│  │                                                               │ │
│  │  vs 버스트 모드:                                                     │ │
│  │  • DMA 전송 중 CPU 0%                                              │ │
│  │  • 전송 완료 후 CPU 100%                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적합한 사용 사례】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 실시간 시스템: CPU 지연 최소화                                    │ │
│  │  • 낮은 대역폭 I/O: 천천히 전송해도 되는 경우                          │ │
│  │  • CPU 바운드 작업: CPU가 계속 실행해야 하는 경우                      │ │
│  │  • 멀티태스킹: 다른 프로세스가 실행 가능                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 사이클 스틸링: CPU가 버스 안 쓸 때 DMA 사용
• 방식: 한 워드씩 번갈아 전송
• 장점: CPU 방해 최소화, 응답성 유지
• 단점: 전송 완료까지 시간 걸림
• vs 버스트: 느리지만 CPU 영향 적음
• 적합: 실시간, CPU 바운드 작업
• BR/BG: 버스 요청/승인 신호
• 효율: CPU 90%+ 유지 가능
• 관련: DMA, 버스트 모드, 버스
```
