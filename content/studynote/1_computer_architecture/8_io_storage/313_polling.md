# 313. 폴링 (Polling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 주기적으로 장치 상태를 확인하는 방식
> 2. **가치**: 단순한 구현, 예측 가능한 타이밍
> 3. **융합**: 프로그램 제어 I/O, 스핀락, 실시간 시스템

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

폴링(Polling)은 **CPU가 주기적으로 I/O 장치의 상태를 확인하여 준비 여부를 검사하는 기법**이다. 장치가 인터럽트를 발생시키지 않고, CPU가 능동적으로 상태를 확인한다.

### 💡 비유: 주기적으로 확인하기

폴링은 **주기적으로 확인하기**와 같다. 택배가 오기로 했다. 알림이 안 온다. 5분마다 문 앞을 확인한다. "왔나?" "아니요." 5분 뒤 "왔나?" "아니요." 반복한다. 언젠가 택배가 온다. 이렇게 주기적으로 확인하는 게 폴링이다.

### 폴링 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                폴링 개요                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【폴링 동작】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  타임라인:                                                           │ │
│  │                                                               │ │
│  │  CPU:  ┌─────┬─────┬─────┬─────┬─────┬─────┐                  │ │
│  │        │작업 │폴링 │작업 │폴링 │작업 │폴링 │ ...               │ │
│  │        └──┬──┴──┬──┴──┬──┴──┬──┴──┬──┴──┬──┘                  │ │
│  │           │     │     │     │     │     │                     │ │
│  │  장치:    │     │     │     │     │     │                     │ │
│  │        ───┴─────┴─────┴─────┴─────┴─────┴─── Busy ─────→       │ │
│  │                                              │                  │ │
│  │                                             Ready               │ │
│  │                                               │                  │ │
│  │  폴링:    No     No     No     No     No    Yes!                │ │
│  │                                                               │ │
│  │  → 주기적으로 장치 상태 확인                                         │ │
│  │  → 준비되면 데이터 처리                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【폴링 코드 구조】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  // 기본 폴링 루프                                                   │ │
│  │  while (true) {                                                    │ │
│  │      if (device_is_ready()) {                                      │ │
│  │          process_data();                                           │ │
│  │          break;                                                    │ │
│  │      }                                                             │ │
│  │      // 다른 작업 가능 (선택적)                                      │ │
│  │  }                                                                 │ │
│  │                                                               │ │
│  │  // 타임아웃 있는 폴링                                               │ │
│  │  int timeout = MAX_TIMEOUT;                                        │ │
│  │  while (!device_is_ready() && timeout > 0) {                       │ │
│  │      timeout--;                                                    │ │
│  │      delay(POLL_INTERVAL);                                         │ │
│  │  }                                                                 │ │
│  │  if (timeout == 0) return -ETIMEDOUT;                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【폴링 종류】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Busy-wait (스핀 폴링):                                          │ │
│  │     • 지연 없이 계속 확인                                            │ │
│  │     • CPU 100% 사용                                                 │ │
│  │     • 매우 빠른 응답                                                 │ │
│  │                                                               │ │
│  │  2. 주기적 폴링 (Periodic Polling):                                  │ │
│  │     • 일정 간격으로 확인                                              │ │
│  │     • 간격 동안 다른 작업 가능                                        │ │
│  │     • 타이머 사용                                                    │ │
│  │                                                               │ │
│  │  3. 적응적 폴링 (Adaptive Polling):                                 │ │
│  │     • 상황에 따라 간격 조절                                           │ │
│  │     • 장치 활동에 따라 빈도 변경                                      │ │
│  │     • 전력/성능 균형                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 폴링 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                폴링 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【폴링 vs 인터럽트】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  폴링:                                                                │ │
│  │  ┌─────────────────────────────────────────┐                   │ │
│  │  │ CPU               장치                    │                   │ │
│  │  │  │                 │                      │                   │ │
│  │  │  ├── 확인 ───────→│                      │                   │ │
│  │  │  │←─ No ───────────┤                      │                   │ │
│  │  │  ├── 확인 ───────→│                      │                   │ │
│  │  │  │←─ No ───────────┤                      │                   │ │
│  │  │  ├── 확인 ───────→│                      │                   │ │
│  │  │  │←─ Yes ──────────┤                      │                   │ │
│  │  │  ├── 처리 ───────→│                      │                   │ │
│  │  └─────────────────────────────────────────┘                   │ │
│  │  → CPU가 능동적으로 확인                                            │ │
│  │                                                               │ │
│  │  인터럽트:                                                            │ │
│  │  ┌─────────────────────────────────────────┐                   │ │
│  │  │ CPU               장치                    │                   │ │
│  │  │  │                 │                      │                   │ │
│  │  │  ├── 다른 작업    │                      │                   │ │
│  │  │  ├── 다른 작업    │                      │                   │ │
│  │  │  ├── 다른 작업    │                      │                   │ │
│  │  │  │←─ 인터럽트 ────┤ Ready!               │                   │ │
│  │  │  ├── ISR 실행 ───→│                      │                   │ │
│  │  └─────────────────────────────────────────┘                   │ │
│  │  → 장치가 준비되면 CPU에 알림                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【폴링이 유리한 경우】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 매우 빠른 장치:                                                  │ │
│  │     • 인터럽트 오버헤드 > 폴링 오버헤드                               │ │
│  │     • 데이터가 즉시 도착                                             │ │
│  │                                                               │ │
│  │  2. 높은 데이터율:                                                   │ │
│  │     • 데이터가 연속적으로 도착                                       │ │
│  │     • 인터럽트 폭주 방지                                             │ │
│  │                                                               │ │
│  │  3. 실시간 시스템:                                                   │ │
│  │     • 결정적 타이밍 필요                                             │ │
│  │     • 인터럽트 지터 없음                                             │ │
│  │                                                               │ │
│  │  4. 단순 시스템:                                                     │ │
│  │     • 인터럽트 컨트롤러 없음                                         │ │
│  │     • 매우 제한된 자원                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【폴링 오버헤드 분석】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  폴링 오버헤드 = (폴링 횟수 × 폴링 비용) / 총 시간                     │ │
│  │                                                               │ │
│  │  예: 장치 준비 시간 10ms, 폴링 간격 1us                               │ │
│  │  폴링 횟수 = 10ms / 1us = 10,000회                                  │ │
│  │  폴링 비용 = 10 사이클 = 5ns (2GHz)                                  │ │
│  │  총 폴링 시간 = 10,000 × 5ns = 50us                                 │ │
│  │                                                               │ │
│  │  → 상대적으로 작지만, CPU는 10ms 동안 다른 작업 불가                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  • 결정적: 폴링은 예측 가능                                            │
│  • 지터: 인터럽트 지터 없음                                            │
│  • 우선순위: 폴링 스레드 우선순위                                       │

│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  폴링 vs 인터럽트 선택 기준:                                                              │ │
│  │  ──────────────────────                                              │ │

│  │  ────              ────────    ────                                    │ │

│  │  높은 데이터율      폴링         인터럽트 폭주 방지                       │ │
│  │  낮은 데이터율      인터럽트      효율적                                 │ │
│  │  실시간 시스템      폴링         결정적 타이밍                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅳ. 실무 적용

### 폴링 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                폴링 분석                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 폴링 API】                                                         │)
│  ──────────────────                                                  │
│  // Busy-wait 폴링                                                   │
│  #define wait_event_cmd(wq, condition, cmd)                         │
│                                                                     │
│  // 타임아웃 있는 폴링                                                │
│  long wait_event_interruptible_timeout(wq, condition, timeout)       │
│                                                                     │
│  // 폴링 지연                                                        │
│  udelay(us);    // 마이크로초                                        │
│  mdelay(ms);    // 밀리초                                            │
│  ndelay(ns);    // 나노초                                            │
│                                                                     │
│  【NAPI (New API) 폴링】                                                         │)
│  ──────────────────                                                  │
│  // 네트워크 드라이버에서 폴링 + 인터럽트 혼합                            │
│  // 높은 부하: 폴링 모드                                              │
│  // 낮은 부하: 인터럽트 모드                                           │
│                                                                     │
│  int my_poll(struct napi_struct *napi, int budget) {                │
│      int work_done = 0;                                             │
│      while (work_done < budget && packet_available()) {             │
│          process_packet();                                          │
│          work_done++;                                               │
│      }                                                              │
│      if (work_done < budget)                                        │
│          napi_complete_done(napi, work_done);                       │
│      return work_done;                                              │
│  }                                                                  │
│                                                                     │
│  【DPDK 폴링 예시】                                                              │)
│  ──────────────────                                                  │
│  // 전적으로 폴링 기반 패킷 처리                                       │
│  while (1) {                                                        │
│      nb_rx = rte_eth_rx_burst(port, queue, mbufs, MAX_PKT_BURST);   │
│      for (i = 0; i < nb_rx; i++) {                                  │
│          process_packet(mbufs[i]);                                  │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【폴링 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. 타임아웃: 무한 루프 방지                                          │
│  2. 지연: 적절한 폴링 간격                                            │
│  3. cpu_relax: 전력 절감 힌트                                         │
│  4. 혼합: NAPI처럼 폴링 + 인터럽트 혼합                                │
│  5. 예산: 폴링 처리량 제한                                            │
│  6. 우선순위: 폴링 스레드 우선순위 설정                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 폴링: CPU가 주기적으로 장치 상태 확인
• 종류: Busy-wait, 주기적, 적응적
• 장점: 단순, 예측 가능, 결정적
• 단점: CPU 낭비, 전력 소비
• 유리한 경우: 빠른 장치, 높은 데이터율, 실시간
• 불리한 경우: 느린 장치, 낮은 데이터율
• 스핀락: 폴링 기반 동기화
• NAPI: 폴링 + 인터럽트 혼합
• DPDK: 전적 폴링 방식
• 관련: 인터럽트, 프로그램 제어 I/O
```
