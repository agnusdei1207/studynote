# 316. 인터럽트 벡터 (Interrupt Vector)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 번호와 ISR 주소를 매핑하는 테이블
> 2. **가치**: 빠른 인터럽트 처리, 표준화된 메커니즘
> 3. **융합**: IDT, IVT, 인터럽트 핸들러

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

인터럽트 벡터(Interrupt Vector)는 **인터럽트 번호(벡터 번호)와 해당 인터럽트를 처리할 ISR(Interrupt Service Routine)의 주소를 매핑하는 테이블**이다. CPU가 인터럽트를 받으면 벡터 번호로 ISR을 찾아 실행한다.

### 💡 비유: 전화번호부

인터럽트 벡터는 **전화번호부**와 같다. 초인종이 울렸다. 누군지 모른다. 전화번호부를 찾는다. "101번 = 배달원", "102번 = 친구". 101번이면 배달원 처리 루틴으로 간다. 102번이면 친구 맞이 루틴으로 간다. 전화번호부가 있어야 누구인지 알고 어떻게 처리할지 안다.

### 인터럽트 벡터 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 벡터 개요                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 벡터 테이블 구조】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  인터럽트 벡터 테이블 (IVT/IDT):                                       │ │
│  │                                                               │ │
│  │  ┌──────────┬────────────────────────┐                         │ │
│  │  │ 벡터 번호 │     ISR 주소            │                         │ │
│  │  ├──────────┼────────────────────────┤                         │ │
│  │  │    0     │  divide_error_addr     │  # 0으로 나누기          │ │
│  │  │    1     │  debug_addr            │  # 디버깅               │ │
│  │  │    2     │  nmi_addr              │  # NMI                  │ │
│  │  │    3     │  breakpoint_addr       │  # 브레이크포인트         │ │
│  │  │   ...    │  ...                   │                         │ │
│  │  │   14     │  page_fault_addr       │  # 페이지 폴트           │ │
│  │  │   ...    │  ...                   │                         │ │
│  │  │   32     │  timer_irq_addr        │  # 타이머               │ │
│  │  │   33     │  keyboard_irq_addr     │  # 키보드               │ │
│  │  │   ...    │  ...                   │                         │ │
│  │  └──────────┴────────────────────────┘                         │ │
│  │                                                               │ │
│  │  → 인터럽트 발생 시 벡터 번호로 ISR 주소 조회                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 처리 과정】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 인터럽트 발생:                                                   │ │
│  │     • 장치 또는 소프트웨어가 인터럽트 신호                               │ │
│  │     • 벡터 번호 함께 전달                                            │ │
│  │                                                               │ │
│  │  2. 벡터 번호 확인:                                                   │ │
│  │     • CPU가 IDTR 레지스터에서 테이블 위치 확인                         │ │
│  │     • 벡터 번호 × 엔트리 크기 = 테이블 오프셋                           │ │
│  │                                                               │ │
│  │  3. ISR 주소 로드:                                                   │ │
│  │     • 테이블에서 ISR 주소 읽기                                        │ │
│  │     • 필요한 경우 세그먼트 셀렉터도 로드                               │ │
│  │                                                               │ │
│  │  4. ISR 실행:                                                        │ │
│  │     • PC를 ISR 주소로 변경                                           │ │
│  │     • 인터럽트 처리                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【x86 IDT vs ARM 벡터 테이블】                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  x86 IDT (Interrupt Descriptor Table):                               │ │
│  │  • 256개 엔트리                                                     │ │
│  │  • 각 엔트리 16바이트 (게이트 디스크립터)                               │ │
│  │  • IDTR 레지스터로 위치 지정                                         │ │
│  │  • LIDT 명령어로 로드                                                │ │
│  │                                                               │ │
│  │  ARM 벡터 테이블:                                                      │ │
│  │  • 고정 위치 (0x00000000 또는 0xFFFF0000)                            │ │
│  │  • 각 엔트리 4바이트 (분기 명령어)                                     │ │
│  │  • 8개 기본 엔트리 + 확장 가능                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 인터럽트 벡터 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 벡터 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86 IDT 엔트리 구조】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  16바이트 게이트 디스크립터:                                            │ │
│  │                                                               │ │
│  │  ┌────────────────────────────────┬────────────────────────┐  │ │
│  │  │ Offset [15:0]                  │ Segment Selector       │  │ │
│  │  │ (ISR 주소 하위 16비트)            │ (코드 세그먼트)          │  │ │
│  │  ├────────────────────────────────┴────────────────────────┤  │ │
│  │  │ Reserved        │ P │ DPL │ 0  │ Type                   │  │ │
│  │  │                 │   │     │    │ (인터럽트/트랩 게이트)    │  │ │
│  │  ├────────────────────────────────┬────────────────────────┤  │ │
│  │  │ Offset [63:32] (reserved)      │ Offset [31:16]         │  │ │
│  │  │                                │ (ISR 주소 상위 16비트)   │  │ │
│  │  └────────────────────────────────┴────────────────────────┘  │ │
│  │                                                               │ │
│  │  P (Present): 엔트리 유효 여부                                        │ │
│  │  DPL: Descriptor Privilege Level (0~3)                              │ │
│  │  Type: 게이트 타입 (인터럽트, 트랩, 태스크)                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【x86 예약된 벡터 번호】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  0-19: CPU 예외 (Exception)                                          │ │
│  │  ┌────┬────────────────────┐                                  │ │
│  │  │ 0  │ #DE Divide Error   │                                  │ │
│  │  │ 1  │ #DB Debug          │                                  │ │
│  │  │ 2  │ NMI                │                                  │ │
│  │  │ 3  │ #BP Breakpoint     │                                  │ │
│  │  │ 6  │ #UD Invalid Opcode │                                  │ │
│  │  │ 8  │ #DF Double Fault   │                                  │ │
│  │  │ 13 │ #GP General Prot.  │                                  │ │
│  │  │ 14 │ #PF Page Fault     │                                  │ │
│  │  └────┴────────────────────┘                                  │ │
│  │                                                               │ │
│  │  20-31: Intel 예약                                                   │ │
│  │  32-255: 사용자 정의 인터럽트                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ARM 벡터 테이블】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  오프셋      예외 타입             모드                         │ │
│  │  ──────      ──────────           ────                         │ │
│  │  0x00        Reset                Supervisor                   │ │
│  │  0x04        Undefined Instr      Undefined                    │ │
│  │  0x08        SVC (SWI)            Supervisor                   │ │
│  │  0x0C        Prefetch Abort       Abort                        │ │
│  │  0x10        Data Abort           Abort                        │ │
│  │  0x14        Reserved             -                            │ │
│  │  0x18        IRQ                  IRQ                          │ │
│  │  0x1C        FIQ                  FIQ                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 인터럽트 벡터 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 벡터 분석                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux IDT 초기화】                                                            │)
│  ──────────────────                                                  │
│  // arch/x86/kernel/idt.c                                           │
│  static const struct idt_data def_idts[] = {                        │
│      INTG(X86_TRAP_DE,      asm_exc_divide_error),                  │
│      INTG(X86_TRAP_NMI,     asm_exc_nmi),                           │
│      INTG(X86_TRAP_BR,      asm_exc_int3),                          │
│      INTG(X86_TRAP_OF,      asm_exc_overflow),                      │
│      INTG(X86_TRAP_PF,      asm_exc_page_fault),                    │
│      // ...                                                         │
│  };                                                                 │
│                                                                     │
│  【ISR 등록 예시】                                                              │)
│  ──────────────────                                                  │
│  // 인터럽트 게이트 설정                                               │
│  void set_intr_gate(unsigned int n, void *addr)                     │
│  {                                                                  │
│      struct idt_data data;                                          │
│      init_idt_data(&data, n, addr);                                 │
│      idt_setup_from_table(idt_table, &data, 1, false);              │
│  }                                                                  │
│                                                                     │
│  【인터럽트 벡터 확인】                                                            │)
│  ──────────────────                                                  │
│  $ cat /proc/interrupts                                             │
│  # IRQ 번호는 IDT 벡터 번호와 매핑됨                                   │

│  $ dmesg | grep -i idt                                              │
│  [    0.000000] IDT: 0xffffffff8190b000                             │
│                                                                     │
│  【인터럽트 벡터 최적화 팁】                                                         │)
│  ──────────────────                                                  │
│  1. 캐시 정렬: IDT 캐시 효율                                          │
│  2. 최소화: 필요한 벡터만 등록                                         │
│  3. DPL 설정: 권한 최소화                                             │
│  4. Per-CPU: CPU별 IDT (필요시)                                      │
│  5. 보안: IDT 무결성 검사                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 인터럽트 벡터: 벡터 번호 → ISR 주소 매핑
• IDT: x86 인터럽트 디스크립터 테이블
• IVT: 인터럽트 벡터 테이블 (일반적)
• 엔트리: 16바이트 (x86), 4바이트 (ARM)
• 벡터 번호: 0-255 (x86)
• 예약: 0-19 CPU 예외, 20-31 Intel 예약
• 사용자: 32-255 IRQ/소프트웨어
• DPL: 디스크립터 권한 레벨
• IDTR: IDT 베이스 주소 레지스터
• 관련: ISR, 인터럽트, 게이트 디스크립터
```
