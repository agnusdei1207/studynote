# 315. 인터럽트 (Interrupt)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU에 비동기적 이벤트를 알리는 하드웨어 신호
> 2. **가치**: 빠른 응답, 효율적 자원 사용, 이벤트 기반 처리
> 3. **융합**: ISR, 인터럽트 컨트롤러, 우선순위

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

인터럽트(Interrupt)는 **CPU에게 현재 실행 중인 작업을 중단하고 긴급한 이벤트를 처리하도록 알리는 하드웨어 또는 소프트웨어 신호**이다. 외부 장치, 예외, 소프트웨어 요청 등이 인터럽트를 발생시킬 수 있다.

### 💡 비유: 초인종

인터럽트는 **초인종**과 같다. 집에서 TV를 보고 있다. "딩동!" 초인종이 울린다. TV를 멈추고 문으로 간다. 누군지 확인한다. 처리하고 돌아온다. TV를 다시 본다. 초인종이 언제 울릴지 모른다. 울리면 즉시 반응한다.

## Ⅱ. 인터럽트 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 컨트롤러】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  역할:                                                               │ │
│  │  • 여러 장치의 인터럽트 관리                                          │ │
│  │  • 우선순위 판별                                                    │ │
│  │  • CPU에 하나씩 전달                                                │ │
│  │  • 인터럽트 마스킹                                                  │ │
│  │                                                               │ │
│  │  종류:                                                               │ │
│  │  • PIC (Programmable Interrupt Controller): 8259A                 │ │
│  │  • APIC (Advanced PIC): 현대 x86                                   │ │
│  │  • GIC (Generic Interrupt Controller): ARM                        │ │
│  │  • NVIC (Nested Vectored Interrupt Controller): ARM Cortex-M      │ │
│  │                                                               │ │
│  │         장치들                                                     │ │
│  │     ┌──┬──┬──┬──┬──┐                                           │ │
│  │     │A │B │C │D │E │                                           │ │
│  │     └┬─┴┬─┴┬─┴┬─┴┬─┘                                           │ │
│  │      ↓  ↓  ↓  ↓  ↓                                            │ │
│  │    ┌─────────────┐                                            │ │
│  │    │인터럽트 컨트롤러│                                           │ │
│  │    └──────┬──────┘                                            │ │
│  │           ↓                                                    │ │
│  │       ┌───┴───┐                                               │ │
│  │       │  CPU  │                                               │ │
│  │       └───────┘                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 vs 폴링】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  인터럽트:                                                            │ │
│  │  • 이벤트 발생 시 즉시 알림                                          │ │
│  │  • CPU가 다른 작업 가능                                             │ │
│  │  • 오버헤드 있음 (컨텍스트 저장/복원)                                │ │
│  │                                                               │ │
│  │  폴링:                                                               │ │
│  │  • CPU가 주기적으로 확인                                            │ │
│  │  • CPU가 확인에 시간 소비                                            │ │
│  │  • 오버헤드 없음                                                    │ │
│  │                                                               │ │
│  │  선택 기준:                                                           │ │
│  │  • 자주 발생: 폴링 유리                                              │ │
│  │  • 드물게 발생: 인터럽트 유리                                        │ │
│  │  • 실시간: 폴링 (결정적)                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 금지/허용】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  CLI (Clear Interrupt Flag):                                        │ │
│  │  • 인터럽트 비활성화                                                 │ │
│  │  • IF 플래그 = 0                                                   │ │
│  │  • 크리티컬 섹션 보호                                                │ │
│  │                                                               │ │
│  │  STI (Set Interrupt Flag):                                          │ │
│  │  • 인터럽트 활성화                                                  │ │
│  │  • IF 플래그 = 1                                                   │ │
│  │  • 크리티컬 섹션 종료                                               │ │
│  │                                                               │ │
│  │  주의:                                                               │ │
│  │  • 인터럽트 금지 시간 최소화                                          │ │
│  │  • 너무 길면 시스템 응답 불가                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 인터럽트: CPU에 비동기 이벤트 알림
• 종류: 하드웨어, 소프트웨어, 예외
• 처리: 저장 → ISR → 복원 → 재개
• 컨트롤러: PIC, APIC, GIC
• 우선순위: 중요도에 따른 처리 순서
• NMI: 마스크 불가능 인터럽트
• ISR: 인터럽트 서비스 루틴
• 인터럽트 금지: CLI/STI
• 폴링 vs 인터럽트: 상황에 따라 선택
• 관련: ISR, 인터럽트 컨트롤러, 예외
```
