# 314. 인터럽트 구동 I/O (Interrupt-driven I/O)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장치가 준비되면 인터럽트로 CPU에 알리는 방식
> 2. **가치**: CPU 대기 시간 활용, 효율적 자원 사용
> 3. **융합**: 인터럽트, ISR, 컨텍스트 스위칭

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

인터럽트 구동 I/O(Interrupt-driven I/O)는 **I/O 장치가 준비되면 인터럽트를 발생시켜 CPU에 알리고, CPU가 다른 작업을 하다가 인터럽트를 처리하는 방식**이다. CPU가 장치를 기다리지 않고 다른 작업을 수행할 수 있다.

### 💡 비유: 알림 기다리기

인터럽트 구동 I/O는 **알림 기다리기**와 같다. 친구에게 문자를 보냈다. 폰을 계속 보지 않는다. 다른 일을 한다. 숙제도 하고 TV도 본다. 폰이 "딩동" 하고 알림이 온다. 그때 답장을 읽는다. 기다리는 동안 다른 일을 할 수 있다.

```

│                인터럽트 구동 I/O 상세                                        │
├─────────────────────────────────────────────────────────────────────┤

│  【인터럽트 처리 과정】                                                           │)

│  │     • 인터럽트 컨트롤러가 CPU에 전달                                  │ │
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 인터럽트 구동 I/O: 장치가 준비되면 인터럽트 발생
• 동작: 요청 → 다른 작업 → 인터럽트 → ISR → 재개
• 장점: CPU 대기 시간 활용, 높은 이용률
• 단점: 인터럽트 오버헤드, 컨텍스트 스위칭
• ISR: 인터럽트 서비스 루틴
• 인터럽트 폭주: 너무 잦은 인터럽트
• 코알레싱: 여러 이벤트 묶어서 인터럽트
• NAPI: 인터럽트 + 폴링 혼합
• Bottom Half: ISR 지연 처리
• 관련: 인터럽트, ISR, 폴링, DMA
```
