# 312. 프로그램 제어 I/O (Programmed I/O)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 직접 I/O 장치를 제어하고 대기하는 방식
> 2. **가치**: 단순한 구현, 저렴한 하드웨어
> 3. **융합**: 폴링, CPU 대기, 단순 장치

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

프로그램 제어 I/O(Programmed I/O, PIO)는 **CPU가 직접 I/O 장치를 제어하고, 데이터 전송이 완료될 때까지 기다리는 방식**이다. CPU가 장치 상태를 계속 확인(폴링)하며 준비될 때까지 대기한다.

### 💡 비유: 직접 기다리기

프로그램 제어 I/O는 **직접 기다리기**와 같다. 친구에게 문자를 보냈다. 답장이 올 때까지 계속 폰을 확인한다. 다른 일을 못 한다. "왔나?" "아직?" "왔나?" 계속 확인한다. 답장이 오면 읽는다. 기다리는 동안 아무것도 못 한다.

### 프로그램 제어 I/O 개요

```
│  【동작 과정】                                                                  │)

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  CPU                      I/O 장치                               │ │
│  │  ┌───┐                    ┌─────┐                              │ │
│  │  │   │  1. 명령어 전송 ───→│     │                              │ │
│  │  │   │                    │     │                              │ │

│  │  │   │       ↓            │     │                              │ │
│  │  │   │  "아직 준비 안 됨"   │     │                              │ │
│  │  │   │       ↓            │     │                              │ │
│  │  │   │  3. 상태 확인 ←────│     │                              │ │
│  │  │   │       ↓            │     │                              │ │

│  │  │   │       ↓            │Ready│                              │ │

│  │  │   │       ↓            │     │                              │ │

│  │  └───┘                    └─────┘                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │

│  │                                                               │ │
│  │  // 상태 레지스터 확인 (폴링)                                        │ │

│  │      // 장치가 준비될 때까지 대기                                     │ │

│  │                                                               │ │
│  │  // 준비되면 데이터 전송                                             │ │

│  │  // 또는                                                            │ │
│  │  data = DATA_REG; // 읽기                                          │ │
│  │                                                               │ │

│                                                                     │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │

│  │  • 작은 데이터에 적합                                                │ │
│  │  • 단순 장치에 적합                                                  │ │

│  │  단점:                                                               │ │
│  │  • CPU 낭비 (대기 시간)                                             │ │
│  │  • 폴링 오버헤드                                                    │ │
│  │  • 느린 장치에 부적합                                                 │ │

│  │                                                               │ │
```

## Ⅱ. 프로그램 제어 I/O 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                프로그램 제어 I/O 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【폴링 vs 인터럽트】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  폴링 (Polling):                                                     │ │
│  │  • CPU가 주기적으로 장치 상태 확인                                    │ │
│  │  • CPU가 능동적으로 확인                                             │ │
│  │  • 준비 안 되어도 확인 오버헤드                                       │ │
│  │                                                               │ │
│  │  인터럽트 (Interrupt):                                                │ │
│  │  • 장치가 준비되면 CPU에 알림                                         │ │
│  │  • CPU가 수동적으로 대기                                             │ │
│  │  • 준비될 때만 CPU 개입                                              │ │
│  │                                                               │ │
│  │  폴링이 유리한 경우:                                                   │ │
│  │  • 매우 빠른 장치                                                    │ │
│  │  • 데이터가 자주 도착                                                 │ │
│  │  • 인터럽트 오버헤드 > 폴링 오버헤드                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프로그램 제어 I/O 흐름도】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │        ┌─────────┐                                           │ │
│  │        │  시작   │                                           │ │
│  │        └────┬────┘                                           │ │
│  │             ↓                                                 │ │
│  │        ┌────┴────┐                                           │ │
│  │        │ 명령어  │                                           │ │
│  │        │ 전송   │                                           │ │
│  │        └────┬────┘                                           │ │
│  │             ↓                                                 │ │
│  │   ┌─────────┴─────────┐                                      │ │
│  │   │   상태 확인        │                                      │ │
│  │   │   (Ready?)        │                                      │ │
│  │   └─────────┬─────────┘                                      │ │
│  │        No   │   Yes                                          │ │
│  │      ┌──────┴──────┐                                         │ │
│  │      ↓             ↓                                         │ │
│  │  ┌───┴───┐    ┌────┴────┐                                    │ │
│  │  │ 대기  │    │ 데이터  │                                    │ │
│  │  │(폴링) │    │  전송   │                                    │ │
│  │  └───┬───┘    └────┬────┘                                    │ │
│  │      │             ↓                                         │ │
│  │      │        ┌────┴────┐                                    │ │
│  │      │        │  완료   │                                    │ │
│  │      │        └─────────┘                                    │ │
│  │      │                                                       │ │
│  │      └───────────────────┐                                   │ │
│  │                          ↓                                   │ │
│  │                   (상태 확인으로)                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CPU 효율 분석】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  I/O 장치 준비 시간: 1ms                                        │ │
│  │  CPU 폴링 주기: 1us                                            │ │
│  │  폴링 횟수: 1000회                                               │ │
│  │                                                               │ │
│  │  CPU 활용:                                                      │ │
│  │  • 폴링 시간: 1000 × 1us = 1ms (전부 낭비)                        │ │
│  │  • 실제 전송: 수 us                                              │ │
│  │  • 효율: 거의 0%                                                 │ │
│  │                                                               │ │
│  │  → 느린 장치에 프로그램 제어 I/O 사용 시 CPU 낭비 심각               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 프로그램 제어 I/O 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                프로그램 제어 I/O 분석                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【폴링 예시 (Linux 커널)】                                                       │)
│  ──────────────────                                                  │
│  // 장치 준비 대기 (폴링)                                             │
│  int wait_for_ready(struct device *dev)                             │
│  {                                                                  │
│      int timeout = 1000;                                            │
│      while (!(readl(dev->status) & READY) && timeout > 0) {         │
│          timeout--;                                                 │
│          udelay(1);  // 1us 대기                                    │
│      }                                                              │
│      return timeout > 0 ? 0 : -ETIMEDOUT;                           │
│  }                                                                  │
│                                                                     │
│  【스핀락 (폴링 기반 동기화)】                                                      │)
│  ──────────────────                                                  │
│  // SMP 시스템에서 스핀락                                             │
│  void spin_lock(spinlock_t *lock)                                   │
│  {                                                                  │
│      while (test_and_set(lock) == 1) {                              │
│          // 락이 풀릴 때까지 폴링                                      │
│          cpu_relax();  // CPU 힌트                                   │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【적합한 사용 사례】                                                            │)
│  ──────────────────                                                  │
│  • GPIO: 버튼 상태 확인                                              │
│  • LED: 상태 토글                                                    │
│  • 센서: 빠른 응답 센서                                               │
│  • 초기화: 부팅 시 장치 초기화                                        │
│  • 폴링 드라이버: 인터럽트 없는 장치                                   │
│                                                                     │
│  【프로그램 제어 I/O 최적화 팁】                                                     │)
│  ──────────────────                                                  │
│  1. 타임아웃: 무한 루프 방지                                          │
│  2. 지연: 적절한 폴링 간격                                            │
│  3. 조건: 복잡한 조건은 인터럽트                                       │
│  4. 데이터: 큰 데이터는 DMA                                           │
│  5. cpu_relax: 전력 절감 힌트                                         │
│  6. 캐시: 상태 변수 캐시 고려                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 프로그램 제어 I/O: CPU가 직접 I/O 제어 및 대기
• 방식: 폴링으로 상태 확인
• 장점: 구현 단순, 하드웨어 저렴
• 단점: CPU 낭비, 느린 장치에 부적합
• 폴링: 상태 레지스터 반복 확인
• 적합: 단순, 빠른 장치 (LED, 버튼)
• 부적합: 느린 장치, 대량 데이터
• 대안: 인터럽트, DMA
• 실제: 스핀락, 초기화에 사용
• 관련: 인터럽트 구동 I/O, DMA
```
