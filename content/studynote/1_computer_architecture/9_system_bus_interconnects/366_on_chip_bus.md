# 온칩 버스 (AMBA, AXI, AHB, APB)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 온칩 버스(On-Chip Bus)는 거대한 단일 칩(SoC) 내부에 집적된 수많은 IP(Intellectual Property) 블록들(CPU, GPU, 메모리 컨트롤러, I/O 등)을 서로 연결하는 통신 프로토콜 및 하드웨어 배선 규격이다.
> 2. **가치**: ARM사가 제정한 AMBA(Advanced Microcontroller Bus Architecture)가 모바일 및 임베디드 시장의 사실상 표준(De facto standard)이며, IP 코어의 재사용성을 극대화하여 칩 설계 기간(TTM)을 획기적으로 단축시켰다.
> 3. **융합**: 요구 대역폭과 전력 소모에 따라 AXI(초고속), AHB(고속 시스템), APB(저속/저전력 주변기기)로 계층을 나누어 칩 전체의 통신 효율을 최적화하는 버스 브리지(Bridge) 아키텍처의 정수를 보여준다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 과거에는 칩 외부(메인보드)의 컴포넌트들을 연결하기 위해 시스템 버스(FSB, PCI 등)를 사용했다면, 온칩 버스는 칩(Chip) 내부의 마이크로미터 단위 세계에서 모듈 간 데이터를 주고받기 위한 통신 고속도로 규격이다.
- **💡 비유**: SoC가 거대한 '복합 쇼핑몰'이라면, 온칩 버스는 쇼핑몰 내부의 **'에스컬레이터와 화물 엘리베이터 시스템'**이다. 고성능이 필요한 영화관(GPU)이나 마트(메모리)에는 초고속 대형 엘리베이터(AXI)를 놓고, 작은 상점(키보드/타이머 IP)에는 느린 에스컬레이터(APB)를 설치하여 효율을 높인다.
- **등장 배경**: 반도체 미세 공정의 발전으로 하나의 실리콘 다이에 수십 개의 기능 블록(IP)을 집어넣는 **SoC(System on Chip)** 시대가 도래했다. 다양한 벤더(ARM, Imagination, Synopsys 등)에서 만든 IP들을 칩 하나로 결합하려면, 이들이 공통의 언어와 물리적 규격으로 통신할 수 있는 '표준 인터페이스'가 필수적이었다. 이에 따라 1996년 ARM이 AMBA 규격을 공개하며 생태계를 통일했다.

---

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### AMBA 프로토콜 계층 및 구성 요소

AMBA 아키텍처는 성능과 전력 효율의 트레이드오프를 고려하여 버스를 계층적으로 분리 설계했다.

| 요소명 | 약자 / 의미 | 주요 특징 | 주 연결 장치 | 비유 |
|:---|:---|:---|:---|:---|
| **AXI** | Advanced eXtensible Interface | 최고 대역폭, 다중 마스터, 독립된 읽기/쓰기 채널 (버스트 모드) | 멀티 코어 CPU, 고성능 GPU, 메모리 컨트롤러 | 초고속 KTX (상하행 분리) |
| **AHB** | Advanced High-performance Bus | 단일 클럭 에지 파이프라인 버스, AXI 도입 전 메인 버스 역할 | 구형 CPU 코어, DMA 컨트롤러, 빠른 외부 메모리 | 고속 국도 |
| **APB** | Advanced Peripheral Bus | 최저 전력 소모, 파이프라이닝 없음, 단순 제어 레지스터 접근 | 타이머, UART, I2C, 키보드 컨트롤러 등 | 동네 이면도로 |
| **Bridge** | 버스 브리지 (Bus Bridge) | 고속 버스(AXI/AHB)와 저속 버스(APB) 간의 속도/프로토콜 변환 | 프로토콜 간 변환 로직 | 고속도로 톨게이트 |

### AMBA 아키텍처 토폴로지 다이어그램

모바일 AP(Application Processor) 내부에서 AMBA 프로토콜이 어떻게 계층적으로 배치되어 있는지 보여주는 다이어그램이다.

```text
 ┌────────────────────────────────────────────────────────────────────────┐
 │                   [ 일반적인 SoC 내부의 AMBA 계층 구조 ]                 │
 ├────────────────────────────────────────────────────────────────────────┤
 │                                                                        │
 │   ┌──────────┐      ┌──────────┐                                       │
 │   │ CPU Core │      │   GPU    │                                       │
 │   └────┬─────┘      └────┬─────┘        High-Performance / High Freq   │
 │        │(AXI Master)     │(AXI Master)                                 │
 │ ═══════╧═════════════════╧════════════════════════════ AXI Interconnect│
 │        │                 │                 │                           │
 │   ┌────┴─────┐      ┌────┴─────┐      ┌────┴─────┐                     │
 │   │ Memory   │      │ DMA Ctrl │      │ AXI to   │                     │
 │   │ Ctrl     │      │          │      │ AHB / APB│ ◀─ Bridge           │
 │   └──────────┘      └──────────┘      │ Bridge   │                     │
 │     (Slave)           (Master)        └────┬─────┘                     │
 │                                            │                           │
 │                                            │  Low-Power / Low Freq     │
 │ ───────────────────────────────────────────┴────────── APB (Peripheral)│
 │            │                  │                  │                     │
 │      ┌─────┴────┐       ┌─────┴────┐       ┌─────┴────┐                │
 │      │   UART   │       │  Timer   │       │   I2C    │                │
 │      └──────────┘       └──────────┘       └──────────┘                │
 │        (Slave)            (Slave)            (Slave)                   │
 │                                                                        │
 │  * AXI: 독립적인 5개의 채널(읽기 주소, 읽기 데이터, 쓰기 주소, 쓰기    │
 │         데이터, 쓰기 응답)로 구성되어 양방향 동시 전송 극대화.         │
 │  * APB: 전력 소모를 최소화하기 위해 클럭 주파수를 낮추고 단순화함.     │
 └────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** AMBA 기반 SoC 구조는 트래픽의 중요도에 따라 네트워크망을 분리한다. CPU와 GPU, 메인 메모리 컨트롤러처럼 수십 기가바이트(GB/s)의 대역폭이 필요한 핵심 컴포넌트들은 **AXI 인터커넥트(Interconnect)**로 묶인다. AXI는 읽기와 쓰기 채널이 물리적으로 완전히 분리되어 있어 전이중(Full-duplex) 통신이 가능하며, 주소만 한 번 보내고 데이터를 연속으로 받는 버스트 전송(Burst Transfer)을 지원한다. 반면, UART나 타이머 같은 단순 센서/I/O 장치들은 고속 버스에 직접 연결할 경우 클럭 동기화로 인한 전력 낭비가 심하므로, **브리지(Bridge)**를 거쳐 느린 속도로 동작하는 **APB**에 연결된다. 브리지는 AXI의 고속 신호를 APB의 느린 클럭에 맞게 버퍼링하고 변환해주는 댐퍼 역할을 한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 심층 기술 비교: AXI vs AHB vs APB

| 비교 항목 | AXI (Advanced eXtensible Interface) | AHB (High-performance Bus) | APB (Peripheral Bus) |
|:---|:---|:---|:---|
| **채널 구조** | 5개의 독립 채널 (주소/데이터 분리) | 공유된 단일 버스 (파이프라인) | 공유된 단일 버스 (단순) |
| **전송 방식** | 버스트(Burst) 전송 특화, Out-of-Order | 파이프라인(Pipelined) 전송 | 비파이프라인, 레지스터 접근 |
| **클럭 주파수** | 매우 높음 (수 GHz) | 높음 (수백 MHz) | 낮음 (수십 MHz 이하) |
| **전력 소모** | 높음 (게이트 복잡도가 큼) | 중간 | 매우 낮음 |
| **주 사용처** | DDR 컨트롤러, 멀티코어 캐시망, GPU | DMA, 구형 SoC 메인 버스 | 저속 센서, 직렬 통신 인터페이스 |

### 과목 융합 관점
- **소프트웨어 공학 및 운영체제**: SoC 드라이버 개발자는 메모리 맵(Memory Map) I/O 방식을 통해 APB에 연결된 UART의 레지스터 주소에 직접 데이터를 쓴다. 이때 버스 아키텍처(Bridge)가 주소 변환과 클럭 동기화를 하드웨어적으로 투명하게 처리하므로, 소프트웨어 개발자는 버스의 물리적 속도 차이를 의식하지 않고 코딩할 수 있다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **커스텀 하드웨어 가속기(NPU) 설계**: 스타트업에서 새로운 AI 가속기 IP를 설계할 때, 이를 시장에 팔거나 기존 모바일 AP에 통합시키기 위해서는 반드시 데이터 입력 및 출력 포트를 **AXI4 표준**으로 래핑(Wrapping)해야 한다. 독자적인 버스를 고집하면 시스템 통합(Integration)이 불가능하여 제품 가치가 0이 된다.
2. **저전력 IoT 칩 설계**: 배터리로 몇 년간 동작해야 하는 스마트 워치 칩셋을 설계할 때, 시스템 대기(Idle) 상태에서는 AXI 버스 클럭을 완전히 차단(Clock Gating)하고, 최소한의 웨이크업(Wake-up) 타이머만 돌아가는 APB 버스 영역에만 전력을 공급하도록 전력 도메인을 분리(Power Gating)해야 한다.

### 도입/평가 체크리스트
- **기술적**: IP 블록을 결합할 때, 각 마스터(Master)의 피크(Peak) 대역폭 합이 버스 매트릭스(Interconnect)의 총 허용 대역폭을 초과하지 않도록 버스 중재기(Arbiter)의 QoS(Quality of Service) 우선순위를 적절히 설정했는가?
- **안티패턴**: 초고속 DMA 컨트롤러를 느린 APB 버스 쪽에 배치하는 설계 오류. 이 경우 DMA가 메모리에 접근할 때마다 브리지에 심각한 트래픽 병목이 발생하여 시스템 전체가 일시 정지(Stall)되는 현상이 발생한다.

## Ⅴ. 기대효과 및 결론

### 기대효과
- **Plug-and-Play 생태계**: AMBA라는 공통 언어 덕분에 애플, 퀄컴, 삼성 같은 팹리스(Fabless) 기업들은 서로 다른 벤더의 CPU 코어, 그래픽 코어, 통신 모뎀 IP를 블록 조립하듯 쉽게 결합하여 강력한 모바일 AP를 짧은 주기로 출시할 수 있게 되었다.
- 하드웨어 IP 설계의 파편화가 방지되고 글로벌 검증 툴(EDA) 생태계가 통합되었다.

### 미래 전망
- 단일 칩에 들어가는 코어 수가 수십 개를 넘어 수백 개(Many-core)에 달하면서, 선을 그물망처럼 연결하는 AXI 기반의 크로스바 스위치(Crossbar Switch) 토폴로지조차 배선 복잡도와 지연 시간 한계에 직면했다.
- 이를 해결하기 위해 차세대 SoC와 칩렛(Chiplet) 구조에서는 인터넷 라우터의 원리를 칩 내부에 적용한 **NoC(Network on Chip)** 아키텍처 (AMBA CHI 규격 등) 로 패러다임이 빠르게 전환되고 있다.

1. 스마트폰 안의 칩셋은 마치 여러 상점이 모여 있는 아주 거대한 **초대형 복합 쇼핑몰**과 같아요.
2. 이 쇼핑몰 안에서 택배 기사(데이터)들이 물건을 나르려면 길이 필요한데, 가장 무거운 물건을 나르는 넓고 빠른 고속 엘리베이터를 **AXI**라고 불러요.
3. 반대로 가벼운 영수증이나 스위치 같은 걸 누르러 가는 좁고 느린 에스컬레이터는 **APB**라고 한답니다. 이 길들을 모두 합쳐서 '온칩 버스'라고 해요!
