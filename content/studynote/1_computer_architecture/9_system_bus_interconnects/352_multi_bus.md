# 352. 멀티 버스 구조 (Multi-Bus Architecture)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 버스로 시스템 성능과 유연성 향상
> 2. **가치**: 병렬성, 격리, 확장성
> 3. **융합**: 계층적 버스, 브리지, 대역폭

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

멀티 버스 구조(Multi-Bus Architecture)는 **단일 버스의 한계를 극복하기 위해 성능과 용도에 따라 여러 버스를 계층적으로 구성하는 시스템 아키텍처**다.

### 💡 비유: 도로 체계

멀티 버스는 **도로 체계**와 같다. 고속도로가 있다. 국도가 있다. 시내 도로가 있다. 각각 목적이 다르다. 고속도로는 멀리 빨리. 시내 도로는 가까이 천천히. 연결된다. 교차로가 있다. 버스도 똑같다. 고속 버스, 저속 버스가 따로 있다.

### 멀티 버스 구조 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                멀티 버스 구조 개요                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【단일 버스 vs 멀티 버스】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  단일 버스 문제:                                                        │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │                     단일 버스                          │    │ │
│  │  │  CPU ────┬──── Memory ────┬──── I/O ────┬──── ...     │    │ │
│  │  │          │                │             │               │    │
│  │  │  → 모든 장치가 같은 버스 공유                              │    │
│  │  │  → 대역폭 경합                                         │    │
│  │  │  → 느린 장치가 빠른 장치 지연                              │    │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  멀티 버스 해결:                                                        │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │  CPU ──── 고속 버스 ──── Memory                        │    │ │
│  │  │              │                                         │    │ │
│  │  │           브리지                                        │    │
│  │  │              │                                         │    │ │
│  │  │  I/O ──── 저속 버스 ──── 주변장치                        │    │ │
│  │  │                                                       │    │ │
│  │  │  → 용도별 버스 분리                                     │    │
│  │  │  → 대역폭 독립                                         │    │
│  │  │  → 성능 향상                                           │    │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스 계층 구조】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Level 1: 시스템 버스 (System Bus)                                    │ │
│  │  • CPU와 메모리 간 고속 연결                                          │ │
│  │  • 최고 성능 요구                                                     │ │
│  │                                                               │ │
│  │  Level 2: 고속 I/O 버스 (High-Speed I/O Bus)                          │ │
│  │  • PCIe, 그래픽 등 고속 장치                                          │ │
│  │  • 높은 대역폭                                                        │ │
│  │                                                               │ │
│  │  Level 3: 표준 I/O 버스 (Standard I/O Bus)                            │ │
│  │  • SATA, USB 등 일반 I/O                                             │ │
│  │  • 중간 성능                                                         │ │
│  │                                                               │ │
│  │  Level 4: 저속 버스 (Low-Speed Bus)                                   │ │
│  │  • I2C, SPI, UART 등 센서/제어                                        │ │
│  │  • 저속, 저전력                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 멀티 버스 구조 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                멀티 버스 구조 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【현대 PC 아키텍처】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                    ┌─────────────┐                            │ │
│  │                    │     CPU     │                            │ │
│  │                    └──────┬──────┘                            │ │
│  │                           │                                     │ │
│  │                    ┌──────▼──────┐                            │ │
│  │                    │   QPI/UPI   │ ← CPU 간 인터커넥트          │ │
│  │                    └──────┬──────┘                            │ │
│  │                           │                                     │ │
│  │                    ┌──────▼──────┐                            │ │
│  │                    │   메모리 버스 │ ← DDR                      │ │
│  │                    └──────┬──────┘                            │ │
│  │                           │                                     │ │
│  │                    ┌──────▼──────┐                            │ │
│  │                    │   PCIe 버스  │ ← GPU, SSD                 │ │
│  │                    └──────┬──────┘                            │ │
│  │                           │                                     │ │
│  │                    ┌──────▼──────┐                            │ │
│  │                    │   PCH       │ ← 플랫폼 컨트롤러            │ │
│  │                    └──────┬──────┘                            │ │
│  │           ┌───────────────┼───────────────┐                   │ │
│  │           │               │               │                    │ │
│  │     ┌─────▼─────┐   ┌─────▼─────┐   ┌─────▼─────┐            │ │
│  │     │   SATA    │   │   USB     │   │   LPC     │            │ │
│  │     │  (디스크)  │   │  (주변)   │   │  (BIOS)   │            │ │
│  │     └───────────┘   └───────────┘   └───────────┘            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스 브리지】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  역할:                                                                 │ │
│  │  • 서로 다른 버스 간 연결                                               │ │
│  │  • 속도 변환                                                          │ │
│  │  • 프로토콜 변환                                                       │ │
│  │  • 버퍼링                                                             │ │
│  │                                                               │ │
│  │  예:                                                                   │ │
│  │  • Host Bridge: CPU 버스 ↔ PCIe                                       │ │
│  │  • PCI-to-PCI Bridge: PCIe 간 연결                                     │ │
│  │  • USB Controller: PCIe ↔ USB                                         │ │
│  │  • SATA Controller: PCIe ↔ SATA                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │
│  【운영체제】                                                          │
│  • DMA: 버스 간 전송                                                │

│  • 인터럽트: 버스별 처리                                              │

│  │  버스 계층별 특성:                                                              │ │

│  │  ────      ────        ──────        ────                          │ │
│  │  1        DDR5        50+ GB/s      메모리                         │ │
│  │  2        PCIe Gen5   64 GB/s       GPU/SSD                       │ │
│  │  3        USB 4       40 Gbps       주변장치                       │ │
│  │  4        I2C         3.4 Mbps      센서                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
