# NoC (Network on Chip)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: NoC(Network on Chip)는 기존 공유 버스(Shared Bus) 방식의 한계를 극복하기 위해, 수십~수백 개의 IP 코어를 인터넷과 같은 **패킷 스위칭(Packet Switching) 통신망** 형태로 단일 칩 내부에 구현한 차세대 인터커넥트 아키텍처다.
> 2. **가치**: 코어 수가 늘어나도 통신 대역폭이 선형적으로 확장(Scalability)되며, 동시 다발적인 데이터 전송이 가능하여 병목을 없애고 매니코어(Many-core) 프로세서와 대규모 AI 가속기의 성능을 극대화한다.
> 3. **융합**: AMBA CHI, UCIe 같은 프로토콜과 결합되어 단일 다이(Die)를 넘어 여러 개의 칩을 하나로 묶는 **칩렛(Chiplet)** 기반 패키징 아키텍처의 핵심 통신망으로 발전하고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: SoC 내부의 통신 구조를 전통적인 '전선(Bus) 연결' 방식에서 벗어나, 라우터(Router)와 스위치, 링크(Link)로 구성된 미니 컴퓨터 네트워크로 대체하는 설계 방법론이다. 데이터는 전기적 신호의 연속이 아닌, 헤더와 페이로드를 갖춘 '패킷(Packet) 또는 플릿(Flit)' 형태로 분할되어 목적지로 라우팅된다.
- **💡 비유**: 기존 버스(Bus)가 모든 집이 하나의 공용 도로를 공유해서 누가 길을 막으면 아무도 못 지나는 **'동네 1차선 골목길'**이라면, NoC는 교차로마다 신호등(라우터)이 있고 여러 경로로 우회할 수 있는 **'바둑판 모양의 대도시 도로망(그리드)'**이다.
- **등장 배경**: 
  1. **버스(Bus) 구조의 확장성 한계**: 코어가 8개, 16개를 넘어 64개 이상 집적되면서, 단일 버스나 크로스바 스위치(Crossbar Switch) 구조는 극심한 병목 현상과 막대한 전력 소모, 복잡한 배선 문제(Wire Routing)를 일으켰다.
  2. **매니코어와 AI의 부상**: 멀티코어를 넘어선 매니코어 시대와 수백 개의 연산 유닛이 동시다발적으로 데이터를 주고받는 NPU/TPU 아키텍처의 등장으로, 새로운 패러다임의 온칩 통신망이 절실히 요구되었다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 프로토콜 계층 비유 | 비유 |
|:---|:---|:---|:---|:---|
| **PE (Processing Element)** | 데이터 생산 및 소비 주체 | CPU 코어, DSP, 메모리 컨트롤러 등의 IP 블록 | 애플리케이션 계층 | 아파트 단지 주민 |
| **NI (Network Interface)** | PE와 NoC 라우터 간의 번역기 | AMBA(AXI) 트랜잭션을 패킷/플릿(Flit)으로 캡슐화 | 트랜스포트 계층 | 택배 포장/해체 센터 |
| **Router / Switch** | 패킷의 경로 설정 및 전달 | 버퍼링, 중재(Arbitration), 라우팅 알고리즘 수행 | 네트워크 계층 | 우체국 라우팅 기계 |
| **Link (링크)** | 라우터 간 물리적 연결 | 구리 배선, 2D/3D 채널을 통한 물리적 신호 전송 | 물리 계층 | 실제 도로 / 케이블 |
| **Flit (플릿)** | 흐름 제어 단위 | 패킷을 잘게 쪼갠 Flow control Unit | 데이터 링크 계층 | 분할된 작은 택배 상자 |

### NoC 토폴로지와 라우팅 아키텍처

NoC의 가장 보편적인 토폴로지인 **2D Mesh (메시) 구조**의 동작 원리를 다이어그램으로 나타낸다.

```text
 ┌────────────────────────────────────────────────────────────────────────┐
 │                    [ NoC 2D Mesh 토폴로지 아키텍처 ]                   │
 ├────────────────────────────────────────────────────────────────────────┤
 │                                                                        │
 │         ┌─────┐             ┌─────┐             ┌─────┐                │
 │         │ PE 0│             │ PE 1│             │ PE 2│                │
 │         └──┬──┘             └──┬──┘             └──┬──┘                │
 │            │(NI)               │(NI)               │(NI)               │
 │     ... ──(R0)──────Link──────(R1)──────Link──────(R2)── ...           │
 │            │                   │                   │                   │
 │           Link                Link                Link                 │
 │            │                   │                   │                   │
 │         ┌──┴──┐             ┌──┴──┐             ┌──┴──┐                │
 │         │ PE 3│             │ PE 4│             │ PE 5│                │
 │         └──┬──┘             └──┬──┘             └──┬──┘                │
 │            │                   │                   │                   │
 │     ... ──(R3)────────────────(R4)────────────────(R5)── ...           │
 │            │                   │                   │                   │
 │                                                                        │
 │  * R (Router): 5개의 포트(동,서,남,북,로컬 PE)를 가진 스위치.              │
 │  * NI (Network Interface): PE의 AXI 신호를 패킷(Flit)으로 쪼개어 주입.   │
 │                                                                        │
 │  [ 라우팅 동작 원리 (XY Routing) ]                                     │
 │   - PE 0에서 PE 5로 데이터 전송 시:                                      │
 │     1. (R0)에서 X축(가로)으로 이동: R0 → R1 → R2                       │
 │     2. (R2)에서 목표 X좌표에 도달하면 Y축(세로)으로 이동: R2 → R5        │
 │     3. (R5)에서 로컬 NI를 통해 PE 5로 데이터(Flit) 전달 완료               │
 └────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** NoC의 핵심은 칩 내부에 거대한 바둑판(Mesh) 모양의 네트워크망을 구축하는 것이다. 각 노드에는 연산 유닛(PE)과 이를 망에 연결해 주는 라우터(R)가 한 쌍을 이룬다. PE가 데이터를 보낼 때 버스를 통째로 점유하는 것이 아니라, 데이터를 작은 단위인 플릿(Flit)으로 쪼개어 라우터로 던진다. 라우터는 칩 내부의 좌표계(예: XY 라우팅)를 기반으로 막히지 않는 길을 찾아 플릿을 이웃 라우터로 전달(Hop)한다. 이 구조에서는 (PE 0 → PE 5) 통신과 (PE 3 → PE 1) 통신이 서로 다른 도로를 이용하므로 **동시에 충돌 없이(Concurrency)** 데이터를 전송할 수 있다. 코어 수가 100개, 1000개로 늘어나도 바둑판을 계속 넓히기만 하면 되므로 확장성(Scalability)이 무한에 가깝다.

## Ⅲ. 융합 비교 및 다각도 분석

### 통신 아키텍처 비교: 버스(Bus) vs 크로스바 스위치 vs NoC

| 비교 항목 | 공유 버스 (Shared Bus, AHB 등) | 크로스바 스위치 (AXI Crossbar) | NoC (Network on Chip) |
|:---|:---|:---|:---|
| **연결 방식** | 단일 공용 통로 | 모든 마스터-슬레이브 1:1 연결 매트릭스 | 분산 라우터 기반 네트워크 |
| **동시 전송** | 불가능 (1회 1마스터 점유) | 제한적 다중 전송 가능 | **다중 경로 동시 전송 극대화** |
| **확장성 (Scalability)**| 매우 나쁨 (코어 4~8개 한계) | 나쁨 (코어 수 제곱비례로 배선 증가) | **매우 우수 (선형적 확장)** |
| **지연 시간 (Latency)** | 트래픽 적을 때 짧음, 혼잡 시 최악 | 항상 일정하게 짧음 | 홉(Hop) 거리에 따라 변동적 지연 |
| **면적 / 전력** | 적게 차지함 | 극도로 큼 (라우팅 라인 낭비) | 적정 수준 (스위치 분산 배치) |

### 과목 융합 관점
- **컴퓨터 네트워크**: NoC 설계는 인터넷 라우팅의 축소판이다. 웜홀 라우팅(Wormhole Routing), 가상 채널(Virtual Channel), 흐름 제어(Flow Control), 데드락 방지(Deadlock avoidance) 알고리즘 등 L3/L4 네트워크망의 핵심 이론이 실리콘 반도체 내부의 마이크로 아키텍처에 그대로 융합되어 사용된다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **AI 가속기(NPU) 칩 아키텍처 설계**: 거대한 행렬 곱셈을 수행하기 위해 256개의 텐서 코어(Tensor Core)를 칩 하나에 집적해야 한다. 이때 코어 간 가중치(Weight)와 활성화(Activation) 데이터를 교환해야 하는데, 버스 방식으로는 대역폭을 감당할 수 없다. 아키텍트는 Ring 또는 2D Torus 형태의 NoC를 도입하고 캐시 코히어런시(Coherency) 트래픽을 처리할 디렉터리 기반 프로토콜을 네트워크 위에 올려 시스템 병목을 해소한다.
2. **다크 실리콘(Dark Silicon) 대비 전력 관리**: NoC 라우터 자체가 소모하는 전력(스태틱 파워)도 상당하다. 전체 칩이 유휴 상태일 때 통신이 없는 지역의 라우터 전원을 동적으로 차단(Power Gating)하는 우회 라우팅(Bypass Routing) 기법을 칩 설계에 반영하여 발열을 잡아야 한다.

### 도입 체크리스트
- **기술적**: 코어 수가 늘어나더라도 최장 거리 노드 간의 통신 홉(Hop) 수 지연시간이 시스템 요구사항(예: 실시간 처리) 내에 들어오는지 시뮬레이터(gem5, BookSim 등)를 통해 철저히 분석했는가?
- **안티패턴 (Deadlock)**: 패킷이 서로의 자원을 요구하며 무한 대기에 빠지는 데드락(Deadlock) 상태. NoC 설계 시 턴(Turn) 제한 모델이나 가상 채널(VC) 분리 설계를 누락하면 칩 전체가 먹통이 되는 치명적인 결함이 실리콘 생산 후에 발견될 수 있다.

## Ⅴ. 기대효과 및 결론

### 기대효과
- 버스 병목으로 인한 코어들의 대기 시간(Stall)이 사라져 시스템 전체의 연산 처리량(Throughput)이 극대화된다.
- 새로운 IP 모듈을 추가할 때, 시스템 전체 배선을 다시 짤 필요 없이 NoC 라우터 포트에 연결(플러그인)만 하면 되므로 설계 유연성이 비약적으로 향상된다.

### 미래 전망
- **3D NoC**: 칩을 2차원 평면으로 나열하는 것을 넘어, TSV(Through Silicon Via) 기술을 통해 메모리 다이와 로직 다이를 샌드위치처럼 수직으로 쌓고, 이를 상하로 관통하는 3D NoC 통신 기술이 도입되고 있다.
- **포토닉 NoC (Optical NoC)**: 전자 신호의 발열과 저항 한계를 극복하기 위해, 칩 내부의 라우터 간 통신을 미세한 레이저 빛(Silicon Photonics)으로 전송하는 광학 NoC가 차세대 엑사스케일 컴퓨팅의 핵심으로 연구되고 있다.

### 참고 표준
- **AMBA CHI (Coherent Hub Interface)**: ARM에서 다중 코어 캐시 일관성 유지를 위해 설계한 NoC 기반의 차세대 인터커넥트 표준 프로토콜.

## 👶 어린이를 위한 3줄 비유 설명
1. 옛날 컴퓨터 안에는 동네 골목길(버스) 하나만 있어서, 데이터를 실은 차들이 항상 빵빵거리며 막혔어요.
2. 그래서 엔지니어들은 컴퓨터 칩 안에 **뉴욕이나 맨해튼처럼 거대한 바둑판 모양의 고속도로망(NoC)**을 만들고 신호등(라우터)을 세웠어요.
3. 이제 차들이 여러 갈래 길로 동시에 달릴 수 있어서, 코어가 아무리 많아져도 데이터가 막히지 않고 순식간에 배달된답니다!
