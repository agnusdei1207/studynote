# 348. 버스 타이밍 (Bus Timing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 버스 신호의 시간적 순서와 제약
> 2. **가치**: 신뢰성, 동기화, 성능 보장
> 3. **융합**: 클럭, 전파지연, 셋업/홀드

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

버스 타이밍(Bus Timing)은 **버스 신호가 유효하게 되는 시점과 지속 시간을 정의하는 것으로, 셋업 시간, 홀드 시간, 전파 지연 등을 포함**한다.

### 💡 비유: 기차 시간표

버스 타이밍은 **기차 시간표**와 같다. 기차가 정해진 시간에 도착한다. 정해진 시간에 출발한다. 늦으면 탈 수 없다. 일찍 와서 기다려야 한다. 기차 시간표를 지켜야 한다. 버스 타이밍도 똑같다. 신호가 정해진 시간에 유효하다.

### 버스 타이밍 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스 타이밍 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【타이밍 파라미터】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │        ←────── 셋업 시간 ──────→←── 홀드 시간 ──→                  │ │
│  │                          │               │                      │ │
│  │  ────────────────┐       ┌───────────────────────┐             │ │
│  │  클럭            │       │        데이터          │             │ │
│  │  ────────────────┘       └───────────────────────┘             │ │
│  │                          ▲               ▲                      │ │
│  │                          │               │                      │ │
│  │                    샘플링 시점          샘플링 종료                 │ │
│  │                                                               │ │
│  │  셋업 시간 (Setup Time):                                               │ │
│  │  • 클럭 에지 전에 데이터가 안정되어야 하는 시간                           │ │
│  │  • 위반 시: 메타스테이블 발생                                          │ │
│  │                                                               │ │
│  │  홀드 시간 (Hold Time):                                                │ │
│  │  • 클럭 에지 후에 데이터가 유지되어야 하는 시간                           │ │
│  │  • 위반 시: 데이터 손실                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【타이밍 다이어그램】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │         ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐                 │ │
│  │  CLK ───┘   └───┘   └───┘   └───┘   └───┘   └───              │ │
│  │                                                               │ │
│  │         ┌───────────────┐           ┌───────────────┐         │ │
│  │  ADDR ──┘               └───────────┘               └──       │ │
│  │          ←─ Tco ─→      ←─ Tvalid ─→                          │ │
│  │                                                               │ │
│  │                     ┌───────────────┐                         │ │
│  │  DATA ──────────────┘               └───────────────────      │ │
│  │                     ←─ Tsetup ─────→                          │ │
│  │                                                               │ │
│  │  Tco (Clock to Output): 클럭 에지부터 출력까지                       │ │
│  │  Tvalid: 데이터 유효 시간                                              │ │
│  │  Tsetup: 다음 클럭까지 셋업 여유                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 버스 타이밍 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스 타이밍 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 타이밍 파라미터】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 전파 지연 (Propagation Delay):                                     │ │
│  │     • 신호가 소스에서 목적지까지 이동하는 시간                             │ │
│  │     • 전선 길이, 부하에 비례                                            │ │
│  │     • 일반적으로 ns/10cm                                               │ │
│  │                                                               │ │
│  │  2. 클럭 주기 (Clock Period):                                          │ │
│  │     • 한 클럭 사이클의 시간                                             │ │
│  │     • 100MHz = 10ns, 1GHz = 1ns                                       │ │
│  │                                                               │ │
│  │  3. 버스 사이클 (Bus Cycle):                                           │ │
│  │     • 한 트랜잭션에 필요한 클럭 수                                       │ │
│  │     • 읽기: 2-4 사이클, 쓰기: 1-2 사이클                                │ │
│  │                                                               │ │
│  │  4. 대기 상태 (Wait States):                                           │ │
│  │     • 슬레이브가 준비될 때까지 추가 클럭                                  │ │
│  │     • 느린 장치에서 발생                                                │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │  파라미터    기호      전형값        영향               │        │ │
│  │  │  ────────   ────      ──────        ────               │        │ │
│  │  │  셋업 시간   Tsu       1-5 ns        타이밍 마진         │        │ │
│  │  │  홀드 시간   Th        0-3 ns        데이터 유지         │        │ │
│  │  │  전파 지연   Tpd       1-10 ns       최대 거리          │        │ │
│  │  │  클럭-출력   Tco       2-10 ns       성능               │        │ │
│  │  │                                                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【타이밍 마진】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  타이밍 마진 = 클럭 주기 - (Tco + Tpd + Tsetup)                        │ │
│  │                                                               │ │
│  │  예:                                                                 │ │
│  │  • 클럭 주기: 10ns (100MHz)                                            │ │
│  │  • Tco: 3ns                                                          │ │
│  │  • Tpd: 4ns                                                          │ │
│  │  • Tsetup: 2ns                                                       │ │
│  │  • 마진 = 10 - (3 + 4 + 2) = 1ns                                      │ │
│  │                                                               │ │
│  │  → 마진이 양수여야 타이밍 만족                                          │ │
│  │  → 마진이 음수면 타이밍 위반                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 버스 타이밍 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스 타이밍 × 타 과목 융합                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 클럭: 시스템 타이밍 기준                                            │
│  • 파이프라인: 스테이지 타이밍                                          │
│  • 캐시: 버스 타이밍                                                  │
│  • 메모리: DRAM 타이밍                                                │
│                                                                     │
│  【운영체제】                                                          │
│  • 스케줄링: 타이밍 고려                                              │
│  • I/O: 장치 타이밍                                                  │
│  • 실시간: 엄격한 타이밍                                               │
│  • 타이머: 시스템 타이밍                                               │
│                                                                     │
│  【성능】                                                              │
│  • 클럭 주파수: 타이밍 제약                                            │
│  • 대기 상태: 성능 저하                                               │
│  • 버스트: 타이밍 효율                                                │
│  • 지연: 전파 지연                                                   │
│                                                                     │
│  【설계】                                                              │
│  • 셋업/홀드: 정적 타이밍 분석                                          │
│  • 전파 지연: 배선 길이                                               │
│  • 클럭 분배: 스큐 최소화                                              │
│  • 마진: 신뢰성 확보                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  버스 타이밍 문제 해결:                                                            │ │
│  │  ──────────────────────                                              │ │
│  │  문제           원인              해결                               │ │
│  │  ────           ────              ────                               │ │
│  │  셋업 위반      클럭 너무 빠름      클럭 낮춤, 파이프라인               │ │
│  │  홀드 위반      신호 너무 빠름      지연 삽입                          │ │
│  │  전파 지연      배선 길이          버퍼, 리피터                        │ │
│  │  클럭 스큐      분배 불균등        클럭 트리                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 버스 타이밍: 신호 시간적 순서
• 셋업 시간: 클럭 전 안정 시간
• 홀드 시간: 클럭 후 유지 시간
• 전파 지연: 신호 이동 시간
• 클럭 주기: 한 사이클 시간
• 대기 상태: 추가 대기 클럭
• 타이밍 마진: 여유 시간
• Tco: 클럭-출력 지연
• 위반: 메타스테이블, 데이터 손실
• 설계: 마진 확보 중요
```
