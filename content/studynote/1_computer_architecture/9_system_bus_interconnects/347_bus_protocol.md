# 347. 버스 프로토콜 (Bus Protocol)

#### 핵심 인사이트 (3줄 요약)

> 3. **융합**: 타이밍, 핸드셰이크, 트랜잭션

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

버스 프로토콜(Bus Protocol)은 **버스에서 데이터를 전송하기 위한 신호 순서, 타이밍, 전기적 특성 등의 규칙 집합**이다. 마스터와 슬레이브 간의 통신 방식을 정의한다.

### 💡 비유: 전화 통화

버스 프로토콜은 **전화 통화**와 같다. 전화를 건다. 상대방이 받는다. "여보세요?" 인사한다. 대화한다. "끊을게요" 작별한다. 끊는다. 순서가 있다. 규칙이 있다. 프로토콜이 없으면 엉망이 된다. 버스도 똑같다.

## Ⅱ. 버스 프로토콜 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스 프로토콜 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동기식 vs 비동기식】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  동기식 (Synchronous):                                                 │ │
│  │  • 클럭 신호에 맞춰 전송                                                │ │
│  │  • 모든 동작이 클럭 에지에 발생                                         │ │
│  │  • 단순, 빠름                                                         │ │
│  │  • 클럭 주파수에 의해 속도 제한                                         │ │
│  │                                                               │ │
│  │  비동기식 (Asynchronous):                                               │ │
│  │  • 핸드셰이크 신호로 전송                                               │ │
│  │  • READY/ACK 신호 사용                                                │ │
│  │  • 다양한 속도 장치 연결 가능                                           │ │
│  │  • 오버헤드 있음                                                       │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │  특성        동기식           비동기식                 │        │ │
│  │  │  ────        ────             ────                    │        │ │
│  │  │  클럭        필요             불필요                   │        │ │
│  │  │  핸드셰이크   없음             필요                    │        │ │
│  │  │  속도        고정             가변                     │        │ │
│  │  │  복잡도      낮음             높음                     │        │ │
│  │  │                                                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【핸드셰이크 프로토콜】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  전체 핸드셰이크 (Full Handshake):                                      │ │
│  │                                                               │ │
│  │  마스터                          슬레이브                          │ │
│  │     │                               │                              │ │
│  │     │──── REQ (요청) ──────────────→│                              │ │
│  │     │                               │                              │ │
│  │     │←─── ACK (응답) ──────────────│                              │ │
│  │     │                               │                              │ │
│  │     │──── DATA (데이터) ───────────→│                              │ │
│  │     │                               │                              │ │
│  │     │──── REQ (해제) ──────────────→│                              │ │
│  │     │                               │                              │ │
│  │     │←─── ACK (해제) ──────────────│                              │ │
│  │     ▼                               ▼                              │ │
│  │                                                               │ │
│  │  → 양쪽 모두 준비 확인                                               │ │
│  │  → 신뢰성 높음                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 버스 프로토콜: 통신 규칙 집합
• 읽기: 주소→READ→데이터→READY
• 쓰기: 주소→데이터→WRITE→ACK
• 동기식: 클럭 기반, 빠름
• 비동기식: 핸드셰이크, 유연
• 핸드셰이크: REQ/ACK 교환
• 타이밍: 신호 순서 중요
• ACK: 전송 확인
• 예: I2C, SPI, PCI, PCIe
• 설계: 대역폭, 지연, 신뢰성
```
