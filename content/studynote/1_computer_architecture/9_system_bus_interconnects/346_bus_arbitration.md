# 346. 버스 중재 (Bus Arbitration)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 마스터 간 버스 사용권 할당
> 2. **가치**: 공정성, 우선순위, 성능 최적화
> 3. **융합**: 중재 알고리즘, 대기 시간, 공정성

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

버스 중재(Bus Arbitration)는 **여러 마스터 장치(CPU, DMA, I/O 프로세서 등)가 동시에 버스 사용을 요청할 때, 어떤 장치가 버스를 사용할지 결정하는 메커니즘**이다.

### 💡 비유: 회의실 예약

버스 중재는 **회의실 예약**과 같다. 회의실이 하나다. 여러 팀이 쓰고 싶다. 동시에 요청한다. 누가 먼저 쓸까? 규칙이 있다. 빈 회의실이면 쓴다. 두 팀이면 우선순위다. 아니면 선착순이다. 버스도 똑같다. 여러 장치가 경쟁한다.

### 버스 중재 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스 중재 개요                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【다중 마스터 환경】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐         │ │
│  │  │  CPU 1  │  │  CPU 2  │  │   DMA   │  │   GPU   │         │ │
│  │  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘         │ │
│  │       │            │            │            │                │ │
│  │       │    버스 요청 (BR)       │            │                │ │
│  │       │    ─────────────────→  │            │                │ │
│  │       │                        │            │                │ │
│  │       └────────────┬───────────┴────────────┘                │ │
│  │                    │                                          │ │
│  │                    ▼                                          │ │
│  │       ┌──────────────────────────────┐                       │ │
│  │       │       버스 중재기             │                       │ │
│  │       │    (Bus Arbiter)             │                       │ │
│  │       │                              │                       │ │
│  │       │  1. 요청 수집                 │                       │ │
│  │       │  2. 우선순위 판단             │                       │ │
│  │       │  3. 버스 승인 (BG)            │                       │ │
│  │       └──────────────────────────────┘                       │ │
│  │                    │                                          │ │
│  │                    ▼                                          │ │
│  │       ┌──────────────────────────────┐                       │ │
│  │       │       공유 버스               │                       │ │
│  │       └──────────────────────────────┘                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【중재 신호】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  BR (Bus Request): 버스 사용 요청                                     │ │
│  │  BG (Bus Grant): 버스 사용 승인                                        │ │
│  │  BB (Bus Busy): 버스 사용 중                                          │ │
│  │                                                               │ │
│  │  순서:                                                                 │ │
│  │  1. 마스터가 BR 신호 활성화                                            │ │
│  │  2. 중재기가 우선순위 판단                                             │ │
│  │  3. 승자에게 BG 신호 활성화                                            │ │
│  │  4. 승자가 BB 신호 활성화 후 버스 사용                                  │ │
│  │  5. 사용 완료 후 BB 비활성화                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 버스 중재 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스 중재 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【중재 알고리즘】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 고정 우선순위 (Fixed Priority):                                    │ │
│  │     • 각 마스터에 고정 우선순위 할당                                    │ │
│  │     • 높은 우선순위 마스터가 항상 우선                                  │ │
│  │     • 문제: 낮은 우선순위 기아                                         │ │
│  │                                                               │ │
│  │  2. 라운드 로빈 (Round Robin):                                         │ │
│  │     • 순차적으로 기회 부여                                             │ │
│  │     • 공정성 보장                                                    │ │
│  │     • 문제: 높은 우선순위 요구 불만족                                   │ │
│  │                                                               │ │
│  │  3. 데이지 체인 (Daisy Chain):                                         │ │
│  │     • 물리적 순서대로 우선순위                                         │ │
│  │     • 구현 간단                                                      │ │
│  │     • 문제: 전파 지연, 멀리 있는 장치 기아                              │ │
│  │                                                               │ │
│  │  4. 폴링 (Polling):                                                   │ │
│  │     • 중재기가 순차적으로 마스터 조회                                   │ │
│  │     • 유연한 우선순위                                                 │ │
│  │     • 문제: 조회 오버헤드                                              │ │
│  │                                                               │ │
│  │  5. 독립 요청 (Independent Request):                                   │ │
│  │     • 각 마스터가 개별 요청/승인 라인                                   │ │
│  │     • 빠른 응답                                                      │ │
│  │     • 문제: 라인 수 많음                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【데이지 체인 구조】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐        │ │
│  │  │ Arbiter│────│Device 1│────│Device 2│────│Device 3│        │ │
│  │  └───┬────┘    └───┬────┘    └───┬────┘    └───┬────┘        │ │
│  │      │             │             │             │               │ │
│  │      BG ──────────→│    BG ────→│    BG ────→│               │ │
│  │                    │             │             │               │ │
│  │      ←─────────────┴─────────────┴───────────── BR            │ │
│  │                                                               │ │
│  │  우선순위: Device 1 > Device 2 > Device 3                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  │  고정우선순위   낮음     빠름      낮음        가능                   │ │
│  │  라운드로빈    높음     보통      낮음        없음                   │ │
│  │  데이지체인    낮음     느림      낮음        가능                   │ │
│  │  독립요청      보통     빠름      높음        설정가능               │ │
│  │                                                               │ │

## Ⅳ. 실무 적용

### 버스 중재 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스 중재 분석                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【버스 사용 현황】                                                              │)
│  ──────────────────                                                  │
│  $ perf stat -e bus-cycles ./program                                │
│  # 버스 사이클 측정                                                   │
│                                                                     │
│  【멀티코어 버스 경합】                                                            │)
│  ──────────────────                                                  │
│  $ perf stat -e cycles,bus-cycles ./program                         │
│  # 코어 사이클 vs 버스 사이클                                          │
│                                                                     │
│  【NUMA 환경 버스】                                                              │)
│  ──────────────────                                                  │
│  $ numactl --hardware                                               │
│  # NUMA 노드별 메모리/버스 구성                                        │
│                                                                     │
│  【버스 중재 최적화 팁】                                                            │)
│  ──────────────────                                                  │
│  1. 우선순위: 중요 작업에 높은 우선순위                                  │
│  2. 버스트: 연속 전송으로 중재 횟수 감소                                 │
│  3. 캐시: 버스 사용 감소                                              │
│  4. NUMA: 로컬 메모리 우선                                             │
│  5. Lock: 버스 잠금 최소화                                             │
│  6. 백오프: 충돌 시 대기 시간 조절                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 버스 중재: 다중 마스터 버스 할당
• 신호: BR (요청), BG (승인), BB (사용중)
• 알고리즘: 고정, 라운드로빈, 데이지체인
• 공정성: 기아 방지
• 우선순위: 중요 장치 우선
• 데이지체인: 물리적 순서 우선순위
• 라운드로빈: 공정한 기회 부여
• 대기시간: 중재 지연 고려
• 오버헤드: 중재 시간 최소화
• 관련: DMA, 멀티프로세서
```
