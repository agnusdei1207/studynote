# 361. 인피니밴드 (InfiniBand)

> 1. **본질**: 인피니밴드 (InfiniBand, IB)는 고성능 컴퓨팅 (HPC) 및 데이터센터 내부의 장치 간 통신을 위해 설계된 **스위치 패브릭 (Switched Fabric)** 기반의 점대점 (Point-to-Point) 상호 연결 아키텍처다.
> 2. **가치**: RDMA (Remote Direct Memory Access)와 커널 바이패스 (Kernel Bypass) 기술을 통해 마이크로초 (µs) 단위의 극도로 낮은 레이턴시와 NDR (Next Data Rate) 기준 최대 400Gbps 이상의 초고대역폭을 제공한다.
> 3. **융합**: 거대 언어 모델 (LLM) 학습을 위한 대규모 GPU 클러스터링의 핵심 인프라로 자리 잡았으며, 이더넷과의 수렴 기술인 RoCE (RDMA over Converged Ethernet)의 모태가 되었다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 인피니밴드 (InfiniBand)는 서버와 서버, 서버와 스토리지 간의 I/O 병목 현상을 해결하기 위해 탄생한 통신 규격이다. 하드웨어 기반의 전송 계층을 사용하여 CPU 부하를 최소화하며, 모든 장치가 고유의 로컬 ID (LID)를 가진 스위치 네트워크를 통해 격리된 채널로 직접 소통한다.

- **💡 비유**: 인피니밴드는 **"데이터센터 전용 초고속 진공 튜브 열차"**와 같다. 일반 도로(이더넷)처럼 신호등(소프트웨어 스택)에 멈춰 설 필요 없이, 출발지에서 목적지까지 정해진 궤도를 따라 마찰(CPU 개입) 없이 순식간에 화물을 전달한다.

- **등장 배경**:
  1. **TCP/IP 소프트웨어 스택의 오버헤드**: 데이터 전송 시 CPU가 패킷 복사, 체크섬 계산 등에 너무 많은 자원을 소모하여 실제 연산 성능이 저하되는 문제가 발생했다.
  2. **I/O 대역폭의 한계**: 기존 버스 방식 아키텍처로는 멀티코어 프로세서와 고성능 스토리지의 속도를 감당할 수 없었다.
  3. **클러스터 확장성 요구**: 수천 대의 노드가 하나의 시스템처럼 동작해야 하는 슈퍼컴퓨팅 환경에서 일관된 저지연 통신망이 절실해졌다.

- **📢 섹션 요약 비유**: 소프트웨어의 간섭을 최소화하고 하드웨어 통로를 직접 개방하여, 데이터 이동의 자유도를 극대화한 하이엔드 상호 연결 솔루션입니다.

### 인피니밴드 레이어 구조 및 RDMA 동작

인피니밴드는 OSI 7계층과 유사하지만, 전송 계층 (Transport Layer)까지 하드웨어로 구현되어 있다. 이는 CPU가 커널 스택을 거치지 않고 직접 HCA에 명령을 내리는 '커널 바이패스'를 가능케 한다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               인피니밴드 아키텍처 및 데이터 전송 흐름             │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   ┌──────────────────┐               ┌──────────────────┐     │
 │   │  Application (A) │               │  Application (B) │     │
 │   └────────┬─────────┘               └────────┬─────────┘     │
 │            │ (Kernel Bypass / RDMA)           │               │
 │   ┌────────▼─────────┐               ┌────────▼─────────┐     │
 │   │     HCA (A)      │ <───────────> │     HCA (B)      │     │
 │   │  [QP] [Memory]   │   IB Fabric   │  [QP] [Memory]   │     │
 │   └──────────────────┘               └──────────────────┘     │
 │                                                               │
 │   IB 계층 구조:                                                │
 │   1. 상위 계층 (Upper Layer): MPI, Sockets Direct, IPoIB     │
 │   2. 전송 계층 (Transport): 하드웨어 기반 신뢰성 보장         │
 │   3. 네트워크 (Network): 서브넷 간 라우팅 (GRH)               │
 │   4. 링크 (Link): 패킷 포워딩 및 흐름 제어 (LID)              │
 │   5. 물리 (Physical): 고속 차동 신호 (SDR, DDR, QDR... NDR)   │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 인피니밴드 통신의 핵심은 애플리케이션이 OS 커널을 거치지 않고 네트워크 카드(HCA)와 직접 대화한다는 점이다. 애플리케이션은 사용자 공간(User Space)에서 HCA의 큐 페어(QP)에 작업을 요청한다. HCA는 상대방 노드의 메모리 주소에 직접 데이터를 쓰거나 읽는 RDMA 작업을 수행하며, 이 과정에서 CPU는 데이터 복사 작업을 전혀 하지 않는다. 전송 계층이 하드웨어에 내장되어 있어 패킷 유실 시 재전송이나 흐름 제어도 하드웨어가 알아서 처리하므로 지연 시간이 극단적으로 짧아진다.

---

### 심층 동작 원리: 대역폭 진화 (Data Rates)

인피니밴드는 '레인(Lane)' 시스템을 사용하며, 보통 4개의 레인을 묶어 사용한다 (4x 구성).
1. **SDR/DDR/QDR/FDR**: 초기 규격으로 10Gbps~56Gbps 대역폭 제공.
2. **EDR (Enhanced Data Rate)**: 레인당 25Gbps, 4x 구성 시 100Gbps 달성.
3. **HDR (High Data Rate)**: 레인당 50Gbps, 4x 구성 시 200Gbps 달성.
4. **NDR (Next Data Rate)**: 레인당 100Gbps, 4x 구성 시 400Gbps 달성. (현대 AI 클러스터의 주력)

- **📢 섹션 요약 비유**: 물량이 늘어날 때마다 선로 수를 늘리거나(멀티 레인), 열차의 속도를 높여(데이터 레이트) 정체 없는 물류 시스템을 계속해서 확장하는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: 인피니밴드 (IB) vs 이더넷 (Ethernet)

| 비교 항목 | 인피니밴드 (InfiniBand) | 이더넷 (Ethernet) |
|:---|:---|:---|
| **주요 철학** | 저지연, 신뢰성, 고성능 (HPC용) | 보편성, 호환성, 확장성 (범용) |
| **전송 계층** | 하드웨어 기반 (Offload) | 주로 소프트웨어 기반 (TCP/IP) |
| **지연 시간** | 1µs 이하 (극도로 낮음) | 10µs ~ 수백µs (상대적 높음) |
| **흐름 제어** | 크레딧 기반 (Credit-based, 무손실) | 창 기반 (Window-based, 손실 가능) |
| **CPU 점유율** | RDMA 활용으로 매우 낮음 | 패킷 처리에 따른 부하 발생 |

인피니밴드는 처음부터 '무손실(Lossless)' 네트워크를 지향한다. 수신 측에 데이터를 담을 공간(Credit)이 있을 때만 데이터를 보내므로 패킷 드롭이 발생하지 않으며, 이는 HPC 연산 중 데이터 재전송으로 인한 성능 급락을 원천 차단한다.

### 비교 2: 인피니밴드 vs RoCE (RDMA over Converged Ethernet)

| 항목 | InfiniBand | RoCE v2 |
|:---|:---|:---|
| **물리 계층** | 전용 IB 케이블/커넥터 | 표준 이더넷 인프라 |
| **프로토콜** | IB Native Stack | UDP/IP 캡슐화 |
| **관리 복잡도** | 서브넷 매니저 필요 (독립망) | 기존 L3 라우팅 활용 가능 |
| **성능** | 최고 수준의 일관성 보장 | 이더넷 설정(PFC 등)에 따라 가변적 |

### 과목 융합 관점

- **데이터베이스 (DB)**: Oracle RAC나 SAP HANA와 같은 인메모리 DB 클러스터는 노드 간 데이터 일관성 유지를 위해 인피니밴드를 사용한다. Cache Fusion 성능을 극대화하여 분산 DB의 확장성을 보장한다.
- **인공지능 (AI)**: NVIDIA의 DGX 시스템은 다수의 GPU를 인피니밴드(Quantum 스위치)로 묶어 하나의 거대한 가속기처럼 동작하게 한다. NCCL (NVIDIA Collective Communications Library)은 인피니밴드의 멀티캐스트 성능을 활용해 가중치 동기화 속도를 높인다.

- **📢 섹션 요약 비유**: 일반적인 도심 도로(이더넷)를 개조하여 고속도로처럼 쓰는 기술(RoCE)도 나왔지만, 태생부터 전용 궤도를 달리는 고속열차(인피니밴드)의 효율을 넘어서기는 어렵습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **AI 대규모 학습 클러스터 병목 해결**: 수천 개의 GPU가 동원되는 LLM 학습 시, 체크포인트 저장과 가중치 전파 과정에서 네트워크 병목이 발생한다. 인피니밴드 NDR 400Gbps를 도입하고, 'Sharp (Scalable Hierarchical Aggregation and Reduction Protocol)' 기술을 사용하여 스위치 레벨에서 데이터 집계 연산을 수행함으로써 통신량을 절반 이하로 줄이는 아키텍처를 적용한다.
2. **HPC 환경에서의 토폴로지 최적화**: 노드 수가 늘어남에 따라 팻 트리 (Fat Tree), 드래곤플라이 (Dragonfly), 토러스 (Torus) 중 최적의 IB 토폴로지를 선택해야 한다. 실무에서는 케이블 비용과 레이턴시 간의 트레이드오프를 고려하여, 3계층 Fat Tree 구조를 통해 모든 노드 간 논-블로킹 (Non-blocking) 통신을 보장하는 설계를 주로 채택한다.
3. **VMware 가상화 환경의 고성능 스토리지 연결**: vMotion이나 vSAN 트래픽이 폭증할 때 이더넷으로는 한계가 있다. iSER (iSCSI Extensions for RDMA) 프로토콜을 인피니밴드 위에서 구동하여 스토리지 I/O 레이턴시를 획기적으로 낮추고 CPU 자원을 비즈니스 로직에 더 많이 할당한다.

### 도입 체크리스트
- **서브넷 매니저 (SM) 가용성**: 이중화된 SM이 구성되어 있는가? (SM 장애 시 전체 패브릭 마비)
- **케이블 거리 제약**: 구리 케이블(DAC)은 3~5m, 그 이상은 광케이블(AOC)이 필요한데 환경에 맞는 선정이 되었는가?
- **파티셔닝 (Partitioning)**: 보안 및 성능 격리를 위해 P_Key 기반의 테넌트 분리가 적용되었는가?

### 안티패턴
- **이종 속도 혼용에 따른 성능 하락**: FDR 장비와 EDR 장비를 혼용할 경우, 전체 패브릭의 속도가 하위 규격으로 고정되거나 특정 경로에서 극심한 병목이 발생할 수 있다. 동일한 성능 대역으로의 일관성 있는 구축이 중요하다.

- **📢 섹션 요약 비유**: 아무리 빠른 슈퍼카라도 한 대의 구식 트럭이 길을 막으면 정체가 생기듯, 초고속 네트워크일수록 모든 구성 요소의 사양을 상향 평준화하는 것이 필수적입니다.

### 미래 전망
인피니밴드는 인공지능 시대를 맞아 제2의 전성기를 맞이하고 있다. 향후 **XDR (1.6Tbps 이상)**급의 속도로 진화할 것이며, 스위치 내부에서 연산을 처리하는 'In-Network Computing' 기능이 더욱 강화되어 단순한 전송망을 넘어 '분산 컴퓨터의 백본' 역할을 수행할 것이다.

### 참고 표준
- **InfiniBand Trade Association (IBTA)**: 인피니밴드 규격 제정 및 인증 주관
- **RDMA Verbs API**: 소프트웨어가 인피니밴드 하드웨어를 제어하는 표준 API
- **MPI (Message Passing Interface)**: IB를 활용하는 HPC 소프트웨어 통신 표준

- **📢 섹션 요약 비유**: 정보를 전달만 하던 단순한 길에서, 정보를 스스로 처리하고 흐름을 조절하는 지능형 신경망으로의 진화는 계속될 것입니다.
