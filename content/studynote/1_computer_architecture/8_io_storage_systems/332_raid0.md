# 332. RAID 0 (Striping)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 여러 디스크에 분산 저장
> 2. **가치**: 최고 성능, 100% 용량 효율
> 3. **융합**: 스트라이핑, 병렬 I/O, 무결함

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

RAID 0(Striping)은 **데이터를 일정 크기(스트라이프)로 나누어 여러 디스크에 분산 저장함으로써 병렬 처리로 최고 성능을 달성하는 RAID 레벨**이다. 중복이 없어 내결함성은 없다.

### 💡 비유: 책 여러 권 동시에 읽기

RAID 0은 **책 여러 권 동시에 읽기**와 같다. 책 한 권을 읽는다. 처음부터 끝까지 읽는다. 오래 걸린다. 4권으로 나눠서 4명이 읽는다. 각자 1/4씩 읽는다. 4배 빠르다. 하지만 한 사람이 읽다가 멈추면 전체를 못 읽는다. RAID 0도 똑같다. 빠르지만 한 디스크 고장 시 전체 손실이다.

### RAID 0 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 0 개요                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스트라이핑 원리】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  데이터: [A1][A2][A3][A4][A5][A6][A7][A8]                            │ │
│  │                                                               │ │
│  │  스트라이프 크기: 64KB (예)                                          │ │
│  │                                                               │ │
│  │         Disk 0      Disk 1      Disk 2      Disk 3             │ │
│  │         ┌────┐      ┌────┐      ┌────┐      ┌────┐             │ │
│  │         │ A1 │      │ A2 │      │ A3 │      │ A4 │             │ │
│  │         │ A5 │      │ A6 │      │ A7 │      │ A8 │             │ │
│  │         │ .. │      │ .. │      │ .. │      │ .. │             │ │
│  │         └────┘      └────┘      └────┘      └────┘             │ │
│  │           ↑           ↑           ↑           ↑                 │ │
│  │           └───────────┴───────────┴───────────┘                 │ │
│  │                        병렬 읽기/쓰기                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 특성】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  읽기 성능: n배 향상 (n = 디스크 수)                                   │ │
│  │  쓰기 성능: n배 향상                                                  │ │
│  │  용량 효율: 100% (모든 디스크 용량 합)                                  │ │
│  │  내결함성: 없음                                                       │ │
│  │                                                               │ │
│  │  디스크 수에 따른 성능:                                                │ │
│  │  • 2 disks: 2배 성능                                                │ │
│  │  • 4 disks: 4배 성능                                                │ │
│  │  • 8 disks: 8배 성능                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【위험성】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  고장 확률: n배 증가 (디스크 수에 비례)                                 │ │
│  │                                                               │ │
│  │  예: 디스크 1개 고장률 5%/년                                          │ │
│  │      4디스크 RAID 0: 약 19%/년 고장 확률                               │ │
│  │                                                               │ │
│  │  → 하나라도 고장 나면 모든 데이터 손실                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. RAID 0 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 0 상세                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스트라이프 크기 영향】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  작은 스트라이프 (예: 16KB):                                          │ │
│  │  • 장점: 작은 파일도 병렬 처리                                         │ │
│  │  • 단점: 랜덤 I/O 시 많은 디스크 참여                                   │ │
│  │  • 적합: 작은 파일 위주                                                │ │
│  │                                                               │ │
│  │  큰 스트라이프 (예: 256KB):                                           │ │
│  │  • 장점: 순차 I/O 효율적                                              │ │
│  │  • 단점: 작은 파일은 단일 디스크만 사용                                  │ │
│  │  • 적합: 대용량 파일 위주                                              │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │  스트라이프   순차 성능   랜덤 성능   권장 용도         │        │ │
│  │  │  ────────   ────────   ────────   ────────           │        │ │
│  │  │  16KB       보통        좋음       데이터베이스        │        │ │
│  │  │  64KB       좋음        보통       일반 용도          │        │ │
│  │  │  256KB      최고        낮음       비디오/이미지      │        │ │
│  │  │                                                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최소/최대 디스크】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  최소: 2개                                                           │ │
│  │  최대: 제한 없음 (컨트롤러 제한까지)                                    │ │
│  │  권장: 2-8개 (관리 용이성)                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【용량 계산】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  총 용량 = 디스크 용량 × 디스크 수                                      │ │
│  │                                                               │ │
│  │  예: 1TB 디스크 4개                                                 │ │
│  │  총 용량 = 1TB × 4 = 4TB                                            │ │
│  │                                                               │ │
│  │  → 100% 용량 활용                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. RAID 0 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 0 × 타 과목 융합                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 병렬 처리: 동시 디스크 액세스                                        │
│  • 대역폭: 디스크 수에 비례                                           │
│  • 컨트롤러: 스트라이핑 관리                                           │
│  • 버스: 다중 경로 필요                                               │
│                                                                     │
│  【운영체제】                                                          │
│  • mdadm: RAID 0 생성 관리                                           │
│  • I/O 스케줄링: 병렬 요청                                            │
│  • 파일 시스템: 스트라이프 인식                                         │
│  • 블록 장치: 논리 볼륨 관리                                           │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 성능: 높은 처리량                                                  │
│  • 위험: 데이터 손실 가능성                                            │
│  • 백업: 필수                                                        │
│  • 임시 테이블: RAID 0 적합                                           │
│                                                                     │
│  【성능】                                                              │
│  • 순차 읽기: n배 향상                                                │
│  • 순차 쓰기: n배 향상                                                │
│  • 랜덤 I/O: 스트라이프 크기 영향                                       │
│  • IOPS: 디스크 수에 비례                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RAID 0 성능 예시:                                                             │ │
│  │  ──────────────────────                                              │ │
│  │  구성           순차 읽기    순차 쓰기    IOPS                      │ │
│  │  ────           ────────    ────────    ────                      │ │
│  │  단일 HDD       150 MB/s    150 MB/s    100                       │ │
│  │  2x HDD RAID 0  300 MB/s    300 MB/s    200                       │ │
│  │  4x HDD RAID 0  600 MB/s    600 MB/s    400                       │ │
│  │  4x SSD RAID 0  14 GB/s     10 GB/s     2,000,000                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### RAID 0 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 0 분석                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RAID 0 생성】                                                                │)
│  ──────────────────                                                  │
│  # 4디스크 RAID 0 생성                                               │
│  $ mdadm --create /dev/md0 --level=0 --raid-devices=4 \             │
│        /dev/sda1 /dev/sdb1 /dev/sdc1 /dev/sdd1                      │
│                                                                     │
│  # 스트라이프 크기 지정 (64KB)                                        │
│  $ mdadm --create /dev/md0 --level=0 --raid-devices=2 \             │
│        --chunk=64 /dev/sda1 /dev/sdb1                               │
│                                                                     │
│  【RAID 0 상태 확인】                                                            │)
│  ──────────────────                                                  │
│  $ cat /proc/mdstat                                                 │
│  md0 : active raid0 sda1[0] sdb1[1] sdc1[2] sdd1[3]                 │
│        4190208 blocks super 1.2 64k chunks                          │
│                                                                     │
│  $ mdadm --detail /dev/md0                                          │
│  Raid Level : raid0                                                 │
│  Array Size : 4190208 (4.00 TiB)                                    │
│                                                                     │
│  【RAID 0 적합 용도】                                                            │)
│  ──────────────────                                                  │
│  1. 임시 데이터: 캐시, 스왑, /tmp                                      │
│  2. 비디오 편집: 대용량 순차 처리                                       │
│  3. 게임: 빠른 로딩                                                  │
│  4. scratch 공간: 중간 결과물                                         │
│  5. 백업된 데이터: 복원 속도 중요                                       │
│  6. 테스트 환경: 성능 테스트                                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• RAID 0: 스트라이핑만 사용
• 성능: n배 향상 (디스크 수)
• 용량: 100% 활용
• 내결함성: 없음
• 위험: 1개 디스크 고장 = 전체 손실
• 스트라이프: 16KB ~ 256KB
• 적합: 임시, 캐시, 백업된 데이터
• 부적합: 중요 데이터, 프로덕션
• 최소 디스크: 2개
• 백업: 필수
```
