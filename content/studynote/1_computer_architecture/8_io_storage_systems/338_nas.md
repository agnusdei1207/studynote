# 338. NAS (Network Attached Storage)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크로 연결된 파일 레벨 스토리지
> 2. **가치**: 파일 공유, 간편한 관리, 비용 효율
> 3. **융합**: NFS, SMB/CIFS, 파일 시스템

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

NAS(Network Attached Storage)는 **TCP/IP 네트워크에 연결되어 파일 레벨로 데이터에 접근할 수 있는 스토리지 장치**다. 서버의 파일 시스템처럼 동작하며 여러 클라이언트가 네트워크를 통해 파일을 공유할 수 있다.

### 💡 비유: 공용 사물함

NAS는 **공용 사물함**과 같다. 학교에 큰 사물함이 있다. 누구나 열 수 있다. 파일을 넣고 뺀다. A학생이 파일을 넣는다. B학생이 그 파일을 꺼낸다. 공유된다. 네트워크로 연결된다. 어디서든 접근할 수 있다. NAS도 똑같다. 네트워크로 파일을 공유한다.

## Ⅱ. NAS 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                NAS 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파일 공유 프로토콜】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. NFS (Network File System):                                       │ │
│  │     • Unix/Linux 중심                                                │ │
│  │     • RFC 표준                                                       │ │
│  │     • 버전: NFSv3, NFSv4, NFSv4.1                                    │ │
│  │     • 상태 비저장 (v3) / 상태 유지 (v4)                                │ │
│  │                                                               │ │
│  │  2. SMB/CIFS (Server Message Block):                                 │ │
│  │     • Windows 중심                                                   │ │
│  │     • Active Directory 통합                                          │ │
│  │     • 파일 잠금, 권한                                                 │ │
│  │     • SMB 3.0: 고성능, 암호화                                         │ │
│  │                                                               │ │
│  │  3. AFP (Apple Filing Protocol):                                     │ │
│  │     • macOS 중심                                                     │ │
│  │     • Time Machine 지원                                              │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │  프로토콜    OS         특징                          │        │ │
│  │  │  ────────   ────        ────                          │        │ │
│  │  │  NFS        Linux/Unix  상태 비저장, 간단              │        │ │
│  │  │  SMB/CIFS   Windows     AD 통합, 권한                 │        │ │
│  │  │  AFP        macOS       Time Machine                  │        │ │
│  │  │                                                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NAS vs SAN】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  NAS:                                                                 │ │
│  │  • 파일 레벨 접근                                                    │ │
│  │  • TCP/IP 네트워크                                                   │ │
│  │  • 파일 시스템이 NAS에 있음                                           │ │
│  │  • \\nas\share 또는 /mnt/nas                                        │ │
│  │                                                               │ │
│  │  SAN:                                                                 │ │
│  │  • 블록 레벨 접근                                                    │ │
│  │  • 전용 네트워크 (FC/iSCSI)                                           │ │
│  │  • 파일 시스템이 서버에 있음                                           │ │
│  │  • /dev/sdb 처럼 로컬 디스크                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### NAS 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                NAS 분석                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NFS 마운트】                                                                 │)
│  ──────────────────                                                  │
│  # NFS 공유 마운트                                                   │
│  $ mount -t nfs 192.168.1.100:/share /mnt/nas                       │
│                                                                     │
│  # /etc/fstab 영구 마운트                                            │
│  192.168.1.100:/share  /mnt/nas  nfs  defaults  0  0               │
│                                                                     │
│  【SMB/CIFS 마운트】                                                            │)
│  ──────────────────                                                  │
│  # SMB 공유 마운트                                                   │
│  $ mount -t cifs //192.168.1.100/share /mnt/nas \                   │
│           -o username=user,password=pass                            │
│                                                                     │
│  # Windows에서 접근                                                  │
│  \\192.168.1.100\share                                              │
│                                                                     │
│  【NFS 서버 설정】                                                              │)
│  ──────────────────                                                  │
│  # /etc/exports                                                     │
│  /share  192.168.1.0/24(rw,sync,no_subtree_check)                   │
│  /backup *(ro,sync)                                                 │
│                                                                     │
│  # 내보내기 적용                                                     │
│  $ exportfs -a                                                      │
│                                                                     │
│  【NAS 적합 용도】                                                              │)
│  ──────────────────                                                  │
│  1. 파일 서버: 홈 디렉토리, 문서                                      │
│  2. 백업: 데이터 백업                                                │
│  3. 미디어: 동영상, 음악 공유                                         │
│  4. 협업: 팀 파일 공유                                               │
│  5. 가상화: NFS 데이터스토어                                          │
│  6. 로그: 중앙 로그 저장                                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• NAS: 네트워크 연결 파일 스토리지
• 접근: 파일 레벨
• 프로토콜: NFS, SMB/CIFS, AFP
• 장점: 공유 용이, 설정 간편
• 단점: 블록 I/O보다 느림
• NFS: Unix/Linux 중심
• SMB: Windows 중심
• RAID: 데이터 보호
• 적합: 파일 공유, 백업, 미디어
• 비용: SAN보다 저렴
```
