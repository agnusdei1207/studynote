# 327. SSD (Solid State Drive)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 플래시 메모리 기반 비휘발성 저장장치
> 2. **가치**: 빠른 속도, 무소음, 저전력, 내충격성
> 3. **융합**: 플래시, FTL, 마모 평준화, GC

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SSD(Solid State Drive)는 **NAND 플래시 메모리를 사용하여 데이터를 저장하는 비휘발성 반도체 저장장치**다. 회전하는 디스크가 없고 전자적으로 데이터에 접근하므로 HDD보다 훨씬 빠르다.

### 💡 비유: USB 메모리

SSD는 **대용량 USB 메모리**와 같다. USB 메모리는 디스크가 돌지 않는다. 반도체 칩에 데이터를 저장한다. SSD는 USB 메모리 기술로 만든 대용량 저장장치다. 더 빠르고 더 크다. 하지만 원리는 같다. 전기로 데이터를 읽고 쓴다. 움직이는 부품이 없다.

## Ⅱ. SSD 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SSD 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FTL (Flash Translation Layer)】                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  역할:                                                                │ │
│  │  • 논리 주소(LBA) → 물리 주소(플래시) 변환                             │ │
│  │  • 파일 시스템과 플래시 간 인터페이스                                  │ │
│  │                                                               │ │
│  │  필요 이유:                                                           │ │
│  │  • 플래시는 덮어쓰기 불가 (지운 후 쓰기)                               │ │
│  │  • 블록 단위 삭제 (페이지 단위 쓰기)                                   │ │
│  │                                                               │ │
│  │  논리 주소       FTL          물리 주소                               │ │
│  │  LBA 0    ─────────────→     페이지 10                              │ │
│  │  LBA 1    ─────────────→     페이지 25                              │ │
│  │  LBA 2    ─────────────→     페이지 8                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【마모 평준화 (Wear Leveling)】                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  문제:                                                                │ │
│  │  • 플래시 셀은 유한한 쓰기 수명 (P/E 사이클)                           │ │
│  │  • 특정 블록만 반복 사용 시 조기 고장                                   │ │
│  │                                                               │ │
│  │  해결:                                                                │ │
│  │  • 쓰기를 모든 블록에 골고루 분산                                       │ │
│  │  • 모든 블록이 비슷한 수명까지 사용                                     │ │
│  │                                                               │ │
│  │  종류:                                                                │ │
│  │  • 동적 마모 평준화: 사용 중인 블록만                                   │ │
│  │  • 정적 마모 평준화: 모든 블록 포함                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【가비지 컬렉션 (Garbage Collection)】                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  문제:                                                                │ │
│  │  • 블록 내 일부 페이지만 삭제됨 (무효 페이지)                            │ │
│  │  • 블록 삭제를 위해 유효 페이지 이동 필요                               │ │
│  │                                                               │ │
│  │  과정:                                                                │ │
│  │  1. 무효 페이지가 많은 블록 선택                                        │ │
│  │  2. 유효 페이지를 다른 블록으로 복사                                    │ │
│  │  3. 원래 블록 삭제 (Erase)                                            │ │
│  │  4. 새 블록으로 사용 가능                                              │ │
│  │                                                               │ │
│  │  ┌───────┐     ┌───────┐     ┌───────┐                           │ │
│  │  │A B C D│ →  │A B C E│ →  │       │ ← 삭제                      │ │
│  │  │V I V I│    │V V V V│    │       │                            │ │
│  │  └───────┘     └───────┘     └───────┘                           │ │
│  │  I=무효,V=유효   새 블록으로     빈 블록                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### SSD 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SSD 분석                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SSD 정보 확인】                                                              │)
│  ──────────────────                                                  │
│  $ lsblk -d -o name,rota,size,model                                 │
│  NAME  ROTA  SIZE MODEL                                             │
│  sda      0  500G Samsung SSD 860                                   │

│  # ROTA=0 이면 SSD                                                  │
│                                                                     │
│  【SSD 성능 측정】                                                              │)
│  ──────────────────                                                  │
│  $ hdparm -tT /dev/nvme0n1                                          │
│  Timing cached reads:   25000 MB in  2.00 seconds = 12500 MB/s     │
│  Timing buffered disk reads: 1500 MB in  3.00 seconds = 500 MB/s   │
│                                                                     │
│  【TRIM 설정】                                                                  │)
│  ──────────────────                                                  │
│  # 수동 TRIM                                                         │
│  $ sudo fstrim -v /                                                 │
│  /: 100 GB trimmed                                                  │
│                                                                     │
│  # 자동 TRIM 확인                                                    │
│  $ cat /etc/fstab | grep discard                                    │
│  UUID=xxx  /  ext4  defaults,discard  0  1                          │
│                                                                     │
│  【SSD 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. TRIM: 정기적 실행으로 성능 유지                                   │
│  2. 파티션 정렬: 4K 정렬                                             │
│  │  3. Over-provisioning: 여유 공간 확보                               │ │
│  │  4. 쓰기 최소화: 수명 연장                                           │ │
│  │  5. 스케줄러: none 또는 mq-deadline                                  │ │
│  │  6. SWAP: SSD에서 OK                                                │ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SSD: NAND 플래시 기반 비휘발성 저장장치
• 구조: 컨트롤러 + NAND 플래시 칩
• 장점: 빠름, 무소음, 저전력, 내충격
• 단점: 제한 수명, 높은 가격
• FTL: 논리→물리 주소 변환
• 마모 평준화: 쓰기 분산
• 가비지 컬렉션: 무효 페이지 정리
• NAND: SLC, MLC, TLC, QLC
• 인터페이스: SATA, NVMe (PCIe)
• 관련: HDD, 플래시, NVMe
```
