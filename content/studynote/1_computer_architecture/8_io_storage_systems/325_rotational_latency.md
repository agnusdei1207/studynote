# 325. 회전 지연 (Rotational Latency)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 목표 섹터가 헤드 아래 올 때까지 대기하는 시간
> 2. **가치**: RPM에 반비례, 디스크 성능 영향
> 3. **융합**: 탐색 시간, RPM, 액세스 시간

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

회전 지연(Rotational Latency)은 **헤드가 목표 트랙에 도착한 후, 목표 섹터가 헤드 아래로 회전해 올 때까지 대기하는 시간**이다. 디스크가 회전해야 하므로 물리적으로 불가피한 대기 시간이다.

### 💡 비유: 룰렛

회전 지연은 **룰렛**과 같다. 룰렛이 돈다. 원하는 숫자가 보인다. 하지만 바로 멈출 수 없다. 룰렛이 돌아서 그 숫자가 다시 올 때까지 기다린다. 빨리 오면 0.1초, 늦으면 한 바퀴 돌아야 한다. 평균은 반 바퀴다. 디스크도 똑같다. 섹터가 올 때까지 기다린다.

### 회전 지연 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                회전 지연 개요                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【회전 지연 원리】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 헤드가 목표 트랙 도착 (탐색 완료)                                  │ │
│  │                                                               │ │
│  │  2. 목표 섹터 확인                                                    │ │
│  │     • 섹터가 헤드 바로 아래? → 즉시 읽기                               │ │
│  │     • 섹터가 반대편? → 반 바퀴 대기                                    │ │
│  │                                                               │ │
│  │  3. 회전 대기                                                        │ │
│  │     • 섹터가 헤드 아래 올 때까지 기다림                                │ │
│  │                                                               │ │
│  │              ↓ 헤드                                                 │ │
│  │         ┌─────────────────┐                                        │ │
│  │         │  섹터           │ ← 지금 여기                              │ │
│  │         │    ↻            │                                        │ │
│  │         │      목표 섹터  │ ← 여기 올 때까지 대기                    │ │
│  │         └─────────────────┘                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【평균 회전 지연 계산】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  평균 회전 지연 = 회전 주기 / 2                                         │ │
│  │                                                               │ │
│  │  RPM별 계산:                                                          │ │
│  │                                                               │ │
│  │  5400 RPM:                                                            │ │
│  │  • 회전/초 = 5400 / 60 = 90회                                        │ │
│  │  • 회전 주기 = 1 / 90 = 11.11ms                                      │ │
│  │  • 평균 지연 = 11.11 / 2 = 5.56ms                                    │ │
│  │                                                               │ │
│  │  7200 RPM:                                                            │ │
│  │  • 회전/초 = 7200 / 60 = 120회                                       │ │
│  │  • 회전 주기 = 1 / 120 = 8.33ms                                      │ │
│  │  • 평균 지연 = 8.33 / 2 = 4.17ms                                     │ │
│  │                                                               │ │
│  │  10000 RPM:                                                           │ │
│  │  • 회전/초 = 10000 / 60 = 166.67회                                   │ │
│  │  • 회전 주기 = 1 / 166.67 = 6.00ms                                   │ │
│  │  • 평균 지연 = 6.00 / 2 = 3.00ms                                     │ │
│  │                                                               │ │
│  │  15000 RPM:                                                           │ │
│  │  • 회전/초 = 15000 / 60 = 250회                                      │ │
│  │  • 회전 주기 = 1 / 250 = 4.00ms                                      │ │
│  │  • 평균 지연 = 4.00 / 2 = 2.00ms                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최소/최대 회전 지연】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  최소 회전 지연: 0ms (섹터가 바로 아래)                                │ │
│  │  최대 회전 지연: 회전 주기 (섹터가 막 지나감)                           │ │
│  │  평균 회전 지연: 회전 주기 / 2                                         │ │
│  │                                                               │ │
│  │  7200 RPM 예:                                                         │ │
│  │  • 최소: 0ms                                                          │ │
│  │  • 최대: 8.33ms                                                       │ │
│  │  • 평균: 4.17ms                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 회전 지연 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                회전 지연 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【액세스 시간에서 회전 지연 비중】                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  총 액세스 시간 = 탐색 시간 + 회전 지연 + 전송 시간                      │ │
│  │                                                               │ │
│  │  7200 RPM, 평균 탐색 8ms 예시:                                        │ │
│  │                                                               │ │
│  │  ┌───────────────────────────────────────────────────┐        │ │
│  │  │                총 액세스 시간 (~12ms)               │        │ │
│  │  │  ┌──────────────┬──────────┬──────────┐           │        │ │
│  │  │  │   탐색 8ms   │회전지연 4ms│ 전송 <1ms │           │        │ │
│  │  │  │    (67%)    │  (33%)   │  (~0%)   │           │        │ │
│  │  │  └──────────────┴──────────┴──────────┘           │        │ │
│  │  └───────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  │  → 회전 지연은 액세스 시간의 약 30%                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【회전 지연 감소 기법】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 높은 RPM:                                                         │ │
│  │     • 회전 속도 높이기                                                │ │
│  │     • 5400 → 7200 → 10000 → 15000 RPM                               │ │
│  │                                                               │ │
│  │  2. 섹터 인터리빙 (Sector Interleaving):                               │ │
│  │     • 섹터를 건너뛰어 배치                                             │ │
│  │     • 연속 읽기 시 다음 섹터 준비                                     │ │
│  │                                                               │ │
│  │  3. 미리 읽기 (Read Ahead):                                           │ │
│  │     • 한 번 읽을 때 여러 섹터 읽기                                    │ │
│  │     • 회전 지연 한 번으로 여러 데이터                                  │ │
│  │                                                               │ │
│  │  4. 캐싱:                                                             │ │
│  │     • 자주 접근 데이터 캐시                                           │ │
│  │     • 회전 지연 회피                                                  │ │
│  │                                                               │ │
│  │  5. 회전 최적화 스케줄링:                                              │ │
│  │     • 같은 트랙 내 섹터 순서 고려                                     │ │
│  │     • 회전 위치 기반 스케줄링                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【섹터 인터리빙】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  문제:                                                                │ │
│  │  섹터 1 읽음 → 처리 → 섹터 2 읽으려는데 이미 지나감 → 한 바퀴 대기      │ │
│  │                                                               │ │
│  │  해결: 인터리빙 (2:1)                                                 │ │
│  │  ┌───┬───┬───┬───┬───┬───┬───┬───┐                               │ │
│  │  │ 1 │   │ 2 │   │ 3 │   │ 4 │   │                               │ │
│  │  └───┴───┴───┴───┴───┴───┴───┴───┘                               │ │
│  │     ↑       ↑       ↑       ↑                                     │ │
│  │   읽기    처리시간   다음 섹터 도착                                  │ │
│  │                                                               │ │
│  │  → 섹터 사이에 빈 공간으로 처리 시간 확보                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 회전 지연 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                회전 지연 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【디스크 RPM 확인】                                                              │)
│  ──────────────────                                                  │
│  $ hdparm -I /dev/sda | grep Rotation                               │
│  Nominal Media Rotation Rate: 7200                                  │
│                                                                     │
│  【회전 지연 포함 성능 측정】                                                      │)
│  ──────────────────                                                  │
│  $ iostat -x 1                                                      │
│  Device   await  svctm                                              │
│  sda      5.00   2.00                                               │
│  # await: 탐색 + 회전지연 + 전송                                      │
│                                                                     │
│  【순차 vs 랜덤 I/O】                                                              │)
│  ──────────────────                                                  │
│  # 순차: 회전 지연 최소                                               │
│  # 랜덤: 평균 회전 지연 발생                                          │
│                                                                     │
│  【회전 지연 최적화 팁】                                                           │)
│  ──────────────────                                                  │
│  1. 높은 RPM: 빠른 디스크 선택                                        │
│  2. 순차 I/O: 연속 접근                                              │
│  3. 미리 읽기: readahead 증가                                        │
│  4. 캐시: hot 데이터 캐싱                                            │
│  5. 인터리빙: 최신 디스크는 자동                                      │
│  6. SSD: 회전 지연 없음                                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 회전 지연: 목표 섹터가 헤드 아래 올 때까지 대기
• 평균: 회전 주기 / 2
• RPM: 높을수록 회전 지연 감소
• 5400 RPM: 5.56ms 평균
• 7200 RPM: 4.17ms 평균
• 10000 RPM: 3.00ms 평균
• 15000 RPM: 2.00ms 평균
• 비중: 액세스 시간의 약 30%
• 감소: 높은 RPM, 인터리빙, 캐싱
• SSD: 회전 지연 없음
• 관련: 탐색 시간, RPM, 액세스 시간
```
