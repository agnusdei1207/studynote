# 324. 탐색 시간 (Seek Time)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 헤드가 목표 트랙으로 이동하는 시간
> 2. **가치**: 디스크 성능의 주요 병목
> 3. **융합**: HDD, 액세스 시간, 스케줄링

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

탐색 시간(Seek Time)은 **디스크 헤드가 현재 위치에서 목표 트랙으로 이동하는 데 걸리는 시간**이다. 기계적 이동이 필요하므로 상대적으로 느리며, 전체 디스크 액세스 시간의 큰 부분을 차지한다.

### 💡 비유: 엘리베이터

탐색 시간은 **엘리베이터**와 같다. 1층에 있다. 10층으로 가야 한다. 엘리베이터가 이동하는 시간이 걸린다. 1층에서 2층은 빠르다. 1층에서 10층은 오래 걸린다. 이동 거리에 따라 시간이 다르다. 엘리베이터가 이동하는 동안 기다려야 한다.

│  │  │  │  (Accelerate)│  (Coast)    │  (Decelerate)│          │   │ │

│  │                                                               │ │

│  │     • 무작위 트랙 간 이동 시간의 평균                                   │ │
│  │     • 3 ~ 15 ms (디스크 성능 지표)                                     │ │

│  │     • 0.5 ~ 2 ms                                                     │ │
│  │                                                               │ │

│  │   5ms├────────●●●●●●●●●●●●●●●●●●●●●●                          │ │

## Ⅱ. 탐색 시간 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                탐색 시간 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【액세스 시간에서 탐색 시간 비중】                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  총 액세스 시간 = 탐색 시간 + 회전 지연 + 전송 시간                      │ │
│  │                                                               │ │
│  │  예: 7200 RPM, 평균 탐색 8ms                                          │ │
│  │                                                               │ │
│  │  ┌───────────────────────────────────────────────────┐        │ │
│  │  │                총 액세스 시간 (~12ms)               │        │ │
│  │  │  ┌──────────┬──────────┬──────────┐              │        │ │
│  │  │  │ 탐색 8ms  │회전지연 4ms│ 전송 <1ms │              │        │ │
│  │  │  │  (67%)   │  (33%)   │  (~0%)   │              │        │ │
│  │  │  └──────────┴──────────┴──────────┘              │        │ │
│  │  └───────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  │  → 탐색 시간이 가장 큰 비중 차지                                       │ │
│  │  → 탐색 최소화가 성능 핵심                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【탐색 최소화 기법】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 디스크 스케줄링:                                                   │ │
│  │     • SCAN: 한 방향으로 이동하며 처리                                  │ │
│  │     • C-SCAN: 순환 방향 처리                                          │ │
│  │     • SSTF: 가장 가까운 요청 먼저                                     │ │
│  │                                                               │ │
│  │  2. 데이터 배치:                                                      │ │
│  │     • 연속 할당: 관련 데이터 인접 배치                                  │ │
│  │     • 조각 모음: 단편화 해결                                          │ │
│  │     • 실린더 기반 배치                                                │ │
│  │                                                               │ │
│  │  3. 캐싱:                                                             │ │
│  │     • 디스크 캐시: 자주 접근 데이터 메모리                              │ │
│  │     • 버퍼 캐시: 읽기/쓰기 버퍼링                                     │ │
│  │     • 미리 읽기: 순차 접근 예측                                       │ │
│  │                                                               │ │
│  │  4. I/O 병합:                                                         │ │
│  │     • 인접 요청 병합                                                  │ │
│  │     • 배치 I/O                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SSD와 비교】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  HDD: 탐색 시간 존재 (기계적 이동)                                     │ │
│  │  SSD: 탐색 시간 없음 (전자적 접근)                                     │ │
│  │                                                               │ │
│  │  HDD 랜덤 액세스: 10~15ms                                             │ │
│  │  SSD 랜덤 액세스: 0.1ms                                               │ │
│  │                                                               │ │
│  │  → SSD가 100배 빠른 이유: 탐색 시간 없음                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 탐색 시간 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                탐색 시간 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【디스크 성능 확인】                                                            │)
│  ──────────────────                                                  │
│  $ hdparm -I /dev/sda | grep -i seek                                │
│  Buffer size: 8192kB                                                │

│  ──────────────────                                                  │
│  $ iostat -x 1                                                      │
│  Device   await  svctm  %util                                       │
│  sda      5.00   2.00   80.0                                        │
│  # await: 평균 대기 시간 (탐색+회전지연+전송)                           │
│                                                                     │
│  【디스크 스케줄러 확인】                                                        │)
│  ──────────────────                                                  │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  [mq-deadline] none bfq                                             │
│                                                                     │
│  【탐색 최소화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 연속 할당: 관련 파일을 인접하게                                   │
│  2. 조각 모음: 정기적 실행                                            │
│  3. 캐시 활용: 자주 쓰는 데이터                                       │
│  4. 스케줄러: 적절한 알고리즘 선택                                    │
│  5. RAID 0: 스트라이핑으로 분산                                       │
│  6. SSD: 탐색 없음, 랜덤 I/O 빠름                                    │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 탐색 시간: 헤드가 목표 트랙으로 이동하는 시간
• 구성: 가속 + 등속 + 감속
• 종류: 평균(3~15ms), 트랙간(0.5~2ms), 전체(15~25ms)
• 비중: 전체 액세스 시간의 60~70%
• 영향: 이동 거리에 비선형 비례
• 최소화: 스케줄링, 캐싱, 데이터 배치
• SSD: 탐색 시간 없음
• 병목: HDD 성능의 주요 원인
• 관련: 회전 지연, 액세스 시간, 스케줄링
```
