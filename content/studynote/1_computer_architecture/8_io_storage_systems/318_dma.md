# 318. DMA (Direct Memory Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 개입 없이 장치가 직접 메모리에 접근
> 2. **가치**: CPU 부하 감소, 대량 데이터 전송 효율
> 3. **융합**: 버스, 인터럽트, 사이클 스틸링

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

DMA(Direct Memory Access)는 **CPU의 개입 없이 I/O 장치가 메모리와 직접 데이터를 전송하는 기술**이다. DMA 컨트롤러가 버스를 제어하여 장치와 메모리 간 데이터를 전송한다.

### 💡 비유: 비서

DMA는 **비서**와 같다. 사장님(CPU)이 파일을 복사해야 한다. 직접 하면 1시간이 걸린다. 비서(DMA)에게 시킨다. "이 파일 1000장 복사해." 사장님은 다른 일을 한다. 비서가 복사한다. 다 하면 "완료했습니다"라고 보고한다. 사장님은 복사하는 동안 중요한 일을 했다.

### DMA 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                DMA 개요                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【DMA 없는 데이터 전송】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장치 ←─(1)─→ CPU ←─(2)─→ 메모리                                │ │
│  │                                                               │ │
│  │  1. 장치 → CPU: CPU가 장치에서 데이터 읽기                          │ │
│  │  2. CPU → 메모리: CPU가 메모리에 데이터 쓰기                        │ │
│  │                                                               │ │
│  │  문제:                                                               │ │
│  │  • 모든 데이터가 CPU를 거침                                          │ │
│  │  • CPU가 데이터 이동에 바쁨                                          │ │
│  │  • 대량 데이터에 비효율적                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DMA 있는 데이터 전송】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장치 ←───────→ DMA 컨트롤러 ←───────→ 메모리                     │ │
│  │                      ↑                                          │ │
│  │                      │ 설정                                     │ │
│  │                    CPU                                          │ │
│  │                                                               │ │
│  │  1. CPU: DMA 컨트롤러에 전송 설정 (소스, 목적지, 크기)               │ │
│  │  2. CPU: 다른 작업 수행                                            │ │
│  │  3. DMA: 장치 ↔ 메모리 직접 전송                                   │ │
│  │  4. DMA: 완료 시 CPU에 인터럽트                                    │ │
│  │                                                               │ │
│  │  장점:                                                               │
│  │  • CPU가 다른 작업 가능                                             │ │
│  │  • 대량 데이터 전송 효율적                                           │ │
│  │  • CPU 부하 감소                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DMA 전송 과정】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 초기화 (CPU):                                                    │ │
│  │     • 소스 주소, 목적지 주소 설정                                    │ │
│  │     • 전송 크기 설정                                                 │ │
│  │     • 전송 방향 설정 (장치→메모리, 메모리→장치)                       │ │
│  │     • DMA 시작                                                     │ │
│  │                                                               │ │
│  │  2. 전송 (DMA 컨트롤러):                                              │ │
│  │     • 버스 요청 (BR) → 버스 승인 (BG)                               │ │
│  │     • 버스 마스터가 되어 전송                                        │ │
│  │     • 사이클 스틸링 또는 버스트 모드                                  │ │
│  │                                                               │ │
│  │  3. 완료 (DMA 컨트롤러):                                              │ │
│  │     • 전송 완료 시 인터럽트 발생                                      │ │
│  │     • CPU가 후처리                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. DMA 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                DMA 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【DMA 컨트롤러 구조】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │                    DMA 컨트롤러                          │  │ │
│  │  │  ┌──────────┐  ┌──────────┐  ┌──────────┐              │  │ │
│  │  │  │ 소스     │  │ 목적지   │  │ 카운트   │              │  │ │
│  │  │  │ 주소     │  │ 주소     │  │ 레지스터 │              │  │ │
│  │  │  │ 레지스터 │  │ 레지스터 │  │          │              │  │ │
│  │  │  └──────────┘  └──────────┘  └──────────┘              │  │ │
│  │  │  ┌──────────┐  ┌──────────┐  ┌──────────┐              │  │ │
│  │  │  │ 제어     │  │ 상태     │  │ 버스     │              │  │ │
│  │  │  │ 레지스터 │  │ 레지스터 │  │ 인터페이스│              │  │ │
│  │  │  └──────────┘  └──────────┘  └──────────┘              │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  │  레지스터:                                                           │ │
│  │  • 소스 주소: 데이터 읽을 위치                                        │ │
│  │  • 목적지 주소: 데이터 쓸 위치                                        │ │
│  │  • 카운트: 전송할 바이트 수                                          │ │
│  │  • 제어: 전송 방향, 모드, 인터럽트 설정                               │ │
│  │  • 상태: 전송 진행 상태                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【전송 모드】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 사이클 스틸링 (Cycle Stealing):                                  │ │
│  │     • CPU가 버스 안 쓸 때마다 DMA가 버스 사용                        │ │
│  │     • 한 번에 한 워드씩 전송                                        │ │
│  │     • CPU와 DMA가 번갈아 버스 사용                                  │ │
│  │     • CPU 속도 저하 적음                                           │ │
│  │                                                               │ │
│  │  2. 버스트 모드 (Burst Mode):                                        │ │
│  │     • DMA가 버스를 독점하여 전송                                     │ │
│  │     • 전체 데이터를 한 번에 전송                                     │ │
│  │     • CPU는 전송 완료까지 대기                                       │ │
│  │     • 전송 속도 빠름                                                │ │
│  │                                                               │ │
│  │  3. 투명 모드 (Transparent Mode):                                   │ │
│  │     • CPU가 버스를 안 쓸 때만 DMA 사용                               │ │
│  │     • CPU에 영향 없음                                               │ │
│  │     • 복잡한 타이밍                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DMA 버스 신호】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  BR (Bus Request): DMA가 버스 사용 요청                              │ │
│  │  BG (Bus Grant): CPU가 버스 사용 승인                                │ │
│  │                                                               │ │
│  │  CPU          버스          DMA                                    │ │
│  │  ┌───┐        ┌───┐        ┌───┐                                 │ │
│  │  │   │←───────│   │←─ BR ──│   │                                 │ │
│  │  │   │── BG ─→│   │───────→│   │                                 │ │
│  │  │   │        │   │←─ 데이터 ─│                                  │ │
│  │  └───┘        └───┘        └───┘                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                DMA × 타 과목 융합                                           │
├─────────────────────────────────────────────────────────────────────┤

│  【컴퓨터 구조】                                                      │
│  • 버스: DMA가 버스 마스터가 됨                                        │
│  • 버스 중재: CPU와 DMA 간 버스 할당                                   │
│  • 메모리: DMA가 메모리 직접 접근                                      │
│  • 캐시: 캐시 일관성 문제                                              │
│                                                                     │
│  【운영체제】                                                          │
│  • 버퍼: DMA 전용 버퍼 관리                                            │
│  • I/O 스케줄링: DMA 요청 큐                                          │
│  • 메모리: DMA 영역 할당                                              │

│  • 대량 로드: DMA로 효율적 로딩                                        │

│                                                                     │
│  【네트워크】                                                          │

│  • Zero-copy: CPU 복사 없음                                          │
│  • 고속 네트워크: DMA 필수                                             │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  I/O 제어 방식 비교:                                                              │ │
│  │  ──────────────────────                                              │ │
│  │  방식              CPU 관여    전송 단위    적합 장치                  │ │
│  │  ────              ────────    ────────    ────────                  │ │
│  │  프로그램 제어      100%       바이트      단순 장치                  │ │
│  │  인터럽트 구동      설정+완료   바이트/블록 일반 장치                  │ │
│  │  DMA              설정+완료   블록        디스크, NIC                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• DMA: 장치가 직접 메모리 접근
• 목적: CPU 부하 감소, 대량 데이터 전송
• 구성: 소스/목적지/카운트 레지스터
• 모드: 사이클 스틸링, 버스트, 투명
• 신호: BR (요청), BG (승인)
• 과정: 설정 → 전송 → 인터럽트
• 장점: CPU 자유, 고속 전송
• 단점: 하드웨어 복잡, 버스 경합
• 사용처: 디스크, NIC, 사운드
• 관련: 버스, 인터럽트, IOMMU
```
