# 311. 분리형 I/O (Isolated I/O)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 장치를 위한 별도의 주소 공간과 명령어
> 2. **가치**: 메모리 공간 보존, 명확한 I/O 구분
> 3. **융합**: IN/OUT 명령어, I/O 버스, x86

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

분리형 I/O(Isolated I/O, Port-Mapped I/O)는 **I/O 장치를 위한 별도의 주소 공간을 두고, 전용 명령어(IN/OUT)로만 접근하는 방식**이다. 메모리 주소 공간과 I/O 주소 공간이 분리되어 있다.

### 💡 비유: 다른 책장에 꽂기

분리형 I/O는 **다른 책장에 꽂기**와 같다. 일반 책은 A책장에, 특별한 책(장치)은 B책장에 꽂는다. B책장은 다른 방에 있다. A책장에서 읽는 방법과 B책장에서 읽는 방법이 다르다. 따로 관리하니 헷갈릴 일이 없다.

### 분리형 I/O 개요

```
│                분리형 I/O 개요                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 공간 구조】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  분리형 I/O 주소 공간:                                               │ │
│  │                                                               │ │
│  │  ┌──────────────────┐       ┌──────────────────┐            │ │
│  │  │  메모리 공간       │       │  I/O 공간         │            │ │
│  │  │  ┌──────────┐    │       │  ┌──────────┐    │            │ │
│  │  │  │   RAM    │    │       │  │  Port    │    │            │ │
│  │  │  ├──────────┤    │       │  │  0x60    │    │            │ │
│  │  │  │   ROM    │    │       │  │  키보드   │    │            │ │
│  │  │  └──────────┘    │       │  ├──────────┤    │            │ │
│  │  │                  │       │  │  Port    │    │            │ │
│  │  │  접근:           │       │  │  0x3F8   │    │            │ │
│  │  │  MOV, LOAD, STORE│       │  │  시리얼   │    │            │ │
│  │  │                  │       │  └──────────┘    │            │ │
│  │  └──────────────────┘       │                  │            │ │
│  │                             │  접근: IN, OUT   │            │ │
│  │                             └──────────────────┘            │ │
│  │                                                               │ │
│  │  → 메모리와 I/O가 완전히 분리                                         │ │
│  │  → 각각 전용 명령어로 접근                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【x86 IN/OUT 명령어】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  IN 명령어 (I/O → 레지스터):                                          │ │
│  │  IN AL, 60h       ; Port 60h → AL (1바이트)                        │ │
│  │  IN AX, 60h       ; Port 60h → AX (2바이트)                        │ │
│  │  IN EAX, 60h      ; Port 60h → EAX (4바이트)                       │ │
│  │                                                               │ │
│  │  OUT 명령어 (레지스터 → I/O):                                         │ │
│  │  OUT 60h, AL      ; AL → Port 60h (1바이트)                        │ │
│  │  OUT 60h, AX      ; AX → Port 60h (2바이트)                        │ │
│  │  OUT 60h, EAX     ; EAX → Port 60h (4바이트)                       │ │
│  │                                                               │ │
│  │  가변 포트:                                                            │ │
│  │  IN AL, DX        ; DX가 가리키는 Port → AL                         │ │
│  │  OUT DX, AL       ; AL → DX가 가리키는 Port                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【분리형 I/O 장단점】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 메모리 공간 보존                                                   │ │
│  │  • I/O와 메모리 명확히 구분                                           │ │
│  │  • 캐시 문제 없음 (별도 공간)                                         │ │
│  │  • 보호 쉬움 (I/O 권한 비트)                                          │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 전용 명령어 필요 (IN/OUT)                                          │ │
│  │  • 하드웨어 복잡 (별도 I/O 버스)                                       │ │
│  │  • 제한된 주소 지정 방식                                               │ │
│  │  • 이식성 낮음 (x86 특수)                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 분리형 I/O 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분리형 I/O 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 I/O 포트 (x86)】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  포트 번호      장치                   용도                       │ │
│  │  ────────      ────                   ────                       │ │
│  │  0x00-0x0F     DMA 컨트롤러            DMA 제어                    │ │
│  │  0x20-0x21     PIC (마스터)            인터럽트 컨트롤러            │ │
│  │  0x40-0x43     PIT                    타이머                      │ │
│  │  0x60-0x64     키보드 컨트롤러          키보드 입력                  │ │
│  │  0x70-0x71     RTC                    실시간 시계                  │ │
│  │  0x3F8-0x3FF   COM1                   시리얼 포트 1                │ │
│  │  0x2F8-0x2FF   COM2                   시리얼 포트 2                │ │
│  │  0x378-0x37F   LPT1                   병렬 포트 1                  │ │
│  │  0x1F0-0x1F7   IDE Primary            하드 디스크                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/O 권한 (x86)】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  IOPL (I/O Privilege Level):                                         │ │
│  │  • FLAGS 레지스터의 2비트                                             │ │
│  │  • 0~3 값, 0이 가장 높은 권한                                          │ │
│  │                                                               │ │
│  │  I/O 허용 비트맵 (IOPM):                                              │ │
│  │  • TSS에 존재                                                        │ │
│  │  • 각 포트별 접근 권한 표시                                            │ │
│  │  • 0 = 허용, 1 = 금지                                                 │ │
│  │                                                               │ │
│  │  접근 규칙:                                                            │ │
│  │  • CPL ≤ IOPL: 모든 I/O 허용                                         │ │
│  │  • CPL > IOPL: IOPM 확인                                             │ │
│  │  • 사용자 모드: 일반적으로 I/O 금지                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux I/O 포트 접근】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  #include <asm/io.h>                                                 │ │
│  │                                                               │ │
│  │  // 포트 요청                                                        │ │
│  │  request_region(0x3F8, 8, "my_serial");                             │ │
│  │                                                               │ │
│  │  // I/O 읽기                                                         │ │
│  │  value = inb(0x3F8);         // 1바이트                              │ │
│  │  value = inw(0x3F8);         // 2바이트                              │ │
│  │  value = inl(0x3F8);         // 4바이트                              │ │
│  │                                                               │ │
│  │  // I/O 쓰기                                                         │ │
│  │  outb(value, 0x3F8);         // 1바이트                              │ │
│  │  outw(value, 0x3F8);         // 2바이트                              │ │
│  │  outl(value, 0x3F8);         // 4바이트                              │ │
│  │                                                               │ │
│  │  // 포트 해제                                                        │ │
│  │  release_region(0x3F8, 8);                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 분리형 I/O × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분리형 I/O × 타 과목 융합                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 버스: 별도 I/O 버스 또는 공유                                       │
│  • 명령어: IN, OUT 전용 명령어                                         │
│  • 권한: IOPL, IOPM                                                  │
│  • x86: 분리형 I/O 지원 아키텍처                                        │
│                                                                     │
│  【운영체제】                                                          │
│  • 권한: 커널만 I/O 접근                                               │
│  • 드라이버: I/O 포트 요청/해제                                         │
│  • 시스템 콜: iopl(), ioperm()                                        │
│  • 추상화: 장치 파일로 감싸기                                           │
│                                                                     │
│  【임베디드 시스템】                                                      │
│  • x86 기반: 분리형 I/O 사용 가능                                       │
│  • ARM 기반: 분리형 I/O 없음                                           │
│  • 이식성: 아키텍처 의존적                                               │
│  • 레거시: 구형 PC 하드웨어                                             │
│                                                                     │
│  【보안】                                                              │
│  • I/O 보호: 커널만 직접 접근                                           │
│  • IOPL: 권한 레벨 제어                                                │
│  • IOPM: 포트별 접근 제어                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분리형 I/O vs 메모리 맵 I/O:                                                              │ │
│  │  ──────────────────────                                              │ │
│  │  항목              분리형 I/O      MMIO                              │ │
│  │  ────              ────────        ────                              │ │
│  │  주소 공간          별도             통합                               │ │
│  │  명령어            IN/OUT          LOAD/STORE                        │ │
│  │  메모리 공간        보존             사용                               │ │
│  │  캐시 문제          없음             있음                               │ │
│  │  하드웨어          복잡             단순                               │ │
│  │  사용 아키텍처      x86             ARM, RISC-V                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 분리형 I/O 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분리형 I/O 분석                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【I/O 포트 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/ioports                                                │
│  0000-001f : dma1                                                   │
│  0020-0021 : pic1                                                   │
│  0040-0043 : timer0                                                 │
│  0060-0060 : keyboard                                               │
│  0070-0071 : rtc0                                                   │
│  0080-008f : dma page reg                                           │
│  00a0-00a1 : pic2                                                   │
│  00c0-00df : dma2                                                   │
│  00f0-00ff : fpu                                                    │
│  03f8-03ff : serial                                                 │
│                                                                     │
│  【I/O 권한 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/self/status | grep -i iopl                             │
│  Cpu_allowed:   f                                                   │
│                                                                     │
│  【시리얼 포트 직접 접근 예시】                                                    │)
│  ──────────────────                                                  │
│  // COM1 (0x3F8) 초기화                                              │
│  outb(0x00, 0x3F8 + 1);    // 인터럽트 비활성화                        │
│  outb(0x80, 0x3F8 + 3);    // DLAB 활성화                            │
│  outb(0x03, 0x3F8 + 0);    // 보드레이트 하위 (38400)                  │
│  outb(0x00, 0x3F8 + 1);    // 보드레이트 상위                          │
│  outb(0x03, 0x3F8 + 3);    // 8비트, 패리티 없음, 1 스톱비              │
│  outb(0xC7, 0x3F8 + 2);    // FIFO 활성화                            │
│  outb(0x0B, 0x3F8 + 4);    // IRQ 활성화                             │
│                                                                     │
│  【분리형 I/O 최적화 팁】                                                         │)
│  ──────────────────                                                  │
│  1. 권한: ioperm()로 필요한 포트만 요청                                │
│  2. 요청: request_region()으로 중복 방지                              │
│  3. 해제: 사용 후 release_region()                                    │
│  4. 폭: inb/inw/inl 맞춰 사용                                         │
│  5. 이식성: 아키텍처 독립 코드는 MMIO 선호                              │
│  6. 대안: 현대 장치는 주로 MMIO 사용                                   │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 분리형 I/O: 별도 I/O 주소 공간
• 명령어: IN, OUT (x86 전용)
• 장점: 메모리 공간 보존, 캐시 문제 없음
• 단점: 전용 명령어, 하드웨어 복잡
• 주요 포트: 키보드(0x60), 시리얼(0x3F8)
• 권한: IOPL, IOPM으로 제어
• Linux: inb/outb 함수
• 아키텍처: x86만 지원
• 비교: MMIO는 ARM에서 사용
• 추세: 현대는 MMIO가 주류
```
