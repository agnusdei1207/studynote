# 331. RAID (Redundant Array of Independent Disks)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 디스크를 묶어 성능/신뢰성 향상
> 2. **가치**: 데이터 보호, 성능 향상, 용량 확장
> 3. **융합**: 스트라이핑, 미러링, 패리티

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

RAID(Redundant Array of Independent Disks)는 **여러 개의 물리적 디스크를 논리적으로 하나의 디스크처럼 관리하여 성능 향상, 데이터 신뢰성 보장, 또는 용량 확장을 달성하는 기술**이다.

### 💡 비유: 여러 명이 짐 나르기

RAID는 **여러 명이 짐 나르기**와 같다. 혼자서 무거운 짐을 나른다. 힘들고 느리다. 4명이서 나른다. 각자 1/4씩 나른다. 4배 빠르다. 한 명이 아파도 3명이 계속 나른다. 백업도 된다. RAID도 똑같다. 여러 디스크가 함께 일한다.

### RAID 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 개요                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【필요 이유】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  단일 디스크 문제:                                                     │ │
│  │  • 성능 제한: 물리적 한계                                             │ │
│  │  • 신뢰성: 고장 시 데이터 손실                                         │ │
│  │  • 용량: 단일 디스크 용량 한계                                         │ │
│  │                                                               │ │
│  │  RAID 해결:                                                            │ │
│  │  • 성능: 병렬 처리로 향상                                             │ │
│  │  • 신뢰성: 중복으로 데이터 보호                                        │ │
│  │  • 용량: 여러 디스크 용량 결합                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAID 핵심 기술】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 스트라이핑 (Striping):                                             │ │
│  │     • 데이터를 여러 디스크에 분산                                       │ │
│  │     • 병렬 읽기/쓰기로 성능 향상                                        │ │
│  │                                                               │ │
│  │  2. 미러링 (Mirroring):                                               │ │
│  │     • 동일 데이터를 여러 디스크에 복제                                   │ │
│  │     • 디스크 고장 시 데이터 보호                                        │ │
│  │                                                               │ │
│  │  3. 패리티 (Parity):                                                  │ │
│  │     • 오류 검출/복구 정보 저장                                         │ │
│  │     • 디스크 고장 시 데이터 복구                                        │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │  기술        목적           장점           단점        │        │ │
│  │  │  ────        ────           ────           ────        │        │ │
│  │  │  스트라이핑   성능           빠름           중복 없음    │        │ │
│  │  │  미러링      신뢰성         보호           용량 손실    │        │ │
│  │  │  패리티      신뢰성         복구           계산 오버헤드│        │ │
│  │  │                                                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAID 레벨 개요】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  RAID 0: 스트라이핑만 (성능 최우선)                                    │ │
│  │  RAID 1: 미러링만 (신뢰성 최우선)                                    │ │
│  │  RAID 5: 스트라이핑 + 분산 패리티 (균형)                               │ │
│  │  RAID 6: RAID 5 + 이중 패리티 (높은 신뢰성)                           │ │
│  │  RAID 10: RAID 1 + RAID 0 조합 (성능 + 신뢰성)                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. RAID 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스트라이핑 상세】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  데이터: [A1][A2][A3][A4][A5][A6][A7][A8]                            │ │
│  │                                                               │ │
│  │  스트라이프 크기: 2 블록                                             │ │
│  │                                                               │ │
│  │  Disk 0    Disk 1    Disk 2    Disk 3                         │ │
│  │  ┌────┐    ┌────┐    ┌────┐    ┌────┐                         │ │
│  │  │ A1 │    │ A2 │    │ A3 │    │ A4 │                         │ │
│  │  │ A5 │    │ A6 │    │ A7 │    │ A8 │                         │ │
│  │  └────┘    └────┘    └────┘    └────┘                         │ │
│  │                                                               │ │
│  │  → 4개 디스크 병렬 읽기/쓰기 = 4배 성능                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【미러링 상세】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  데이터: [A1][A2][A3][A4]                                            │ │
│  │                                                               │ │
│  │  Disk 0    Disk 1                                               │ │
│  │  ┌────┐    ┌────┐                                               │ │
│  │  │ A1 │    │ A1 │  ← 복제                                       │ │
│  │  │ A2 │    │ A2 │                                               │ │
│  │  │ A3 │    │ A3 │                                               │ │
│  │  │ A4 │    │ A4 │                                               │ │
│  │  └────┘    └────┘                                               │ │
│  │                                                               │ │
│  │  → Disk 0 고장 → Disk 1에서 복구                                    │ │
│  │  → 용량 효율: 50%                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【패리티 상세】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  패리티 계산: XOR 연산                                                │ │
│  │  P = D1 ⊕ D2 ⊕ D3                                                  │ │
│  │                                                               │ │
│  │  예:                                                                 │ │
│  │  D1 = 1010, D2 = 1100, D3 = 0110                                   │ │
│  │  P = 1010 ⊕ 1100 ⊕ 0110 = 0000                                     │ │
│  │                                                               │ │
│  │  D2 손실 시 복구:                                                     │ │
│  │  D2 = P ⊕ D1 ⊕ D3 = 0000 ⊕ 1010 ⊕ 0110 = 1100                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │
│  【운영체제】                                                          │
│  • 소프트웨어 RAID: OS 레벨 구현                                      │
│  • mdadm: Linux RAID 관리                                           │
│  • 파일 시스템: RAID 위에 구축                                        │
│  • I/O 스케줄링: RAID 인식                                           │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 성능: RAID 0, 10                                                 │

│                                                                     │
│  【네트워크】                                                          │
│  • SAN: RAID 기반 스토리지                                           │
│  • NAS: RAID 볼륨 공유                                               │

│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RAID 레벨 비교:                                                              │ │
│  │  ──────────────────────                                              │ │
│  │  레벨  최소 디스크  용량 효율   읽기    쓰기   내구성                   │ │
│  │  ────  ─────────  ────────   ────    ────   ────                    │ │
│  │  0     2          100%      최고    최고   없음                      │ │

│  │  5     3          (n-1)/n   좋음    보통   1디스크                   │ │
│  │  6     4          (n-2)/n   좋음    낮음   2디스크                   │ │
│  │  10    4          50%       최고    좋음   1디스크/미러              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅳ. 실무 적용

### RAID 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RAID 상태 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/mdstat                                                 │
│  Personalities : [raid1] [raid6] [raid5] [raid4]                    │
│  md0 : active raid5 sda1[0] sdb1[1] sdc1[2]                         │
│        2095104 blocks super 1.2 level 5, 512k chunk                 │
│                                                                     │
│  【RAID 생성】                                                                   │)
│  ──────────────────                                                  │
│  # RAID 1 생성                                                       │
│  $ mdadm --create /dev/md0 --level=1 --raid-devices=2 /dev/sda1 /dev/sdb1│
│                                                                     │
│  # RAID 5 생성                                                       │
│  $ mdadm --create /dev/md0 --level=5 --raid-devices=3 /dev/sd{a,b,c}1│
│                                                                     │
│  【RAID 성능 측정】                                                              │)
│  ──────────────────                                                  │
│  $ fio --name=test --ioengine=libaio --rw=randread --bs=4k \        │
│        --direct=1 --size=1G --numjobs=4 /dev/md0                    │
│                                                                     │
│  【RAID 선택 가이드】                                                            │)
│  ──────────────────                                                  │
│  1. 성능 최우선: RAID 0 (백업 필수)                                   │
│  2. 신뢰성 최우선: RAID 1                                            │
│  3. 균형: RAID 5 (3디스크 이상)                                       │
│  4. 높은 신뢰성: RAID 6 (4디스크 이상)                                 │
│  5. 성능 + 신뢰성: RAID 10 (4디스크 이상)                              │
│  6. 비용 고려: RAID 5가 일반적                                        │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• RAID: 여러 디스크를 논리적으로 결합
• 스트라이핑: 데이터 분산, 성능 향상
• 미러링: 데이터 복제, 신뢰성 향상
• 패리티: 오류 복구 정보
• RAID 0: 스트라이핑만, 성능 최고
• RAID 1: 미러링만, 신뢰성 최고
• RAID 5: 스트라이핑 + 패리티, 균형
• RAID 6: 이중 패리티, 높은 신뢰성
• RAID 10: 미러링 + 스트라이핑
• 선택: 용도에 따라 레벨 선택
```
