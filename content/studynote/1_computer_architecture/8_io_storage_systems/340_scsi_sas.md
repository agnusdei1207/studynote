# 340. SCSI & SAS (Small Computer System Interface / Serial Attached SCSI)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 엔터프라이즈급 스토리지 인터페이스 표준
> 2. **가치**: 높은 신뢰성, 다중 장치, 핫스왑
> 3. **융합**: 명령어 세트, 직렬 전송, SAN

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SCSI(Small Computer System Interface)는 **컴퓨터와 주변장치 간 데이터 전송을 위한 표준 인터페이스**이며, SAS(Serial Attached SCSI)는 **SCSI 명령어 세트를 사용하는 직렬 전송 방식의 엔터프라이즈 스토리지 인터페이스**다.

### 💡 비유: 회사 전화망

SCSI는 **회사 전화망**과 같다. 회사에 전화망이 있다. 내선 번호가 있다. 101, 102, 103... 각 부서가 번호를 갖는다. 동시에 여러 통화가 가능하다. 전화기를 바꿔도 번호는 그대로다. SAS는 디지털 전화망이다. 더 빠르고 더 많이 연결된다.

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SCSI: 스토리지 인터페이스 표준
• SAS: 직렬 SCSI, 엔터프라이즈
• 속도: 3→6→12→24 Gbps
• 명령어: SCSI 명령어 세트
• 특징: 핫스왑, 멀티패스, 높은 신뢰성
• SATA 호환: SAS에 SATA 연결 가능
• Expander: 다중 장치 연결
• WWN: 세계 고유 식별자
• MTBF: 200만 시간
• 용도: 엔터프라이즈 스토리지
```
