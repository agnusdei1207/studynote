# 343. NVMe-oF (NVMe over Fabrics)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: NVMe를 네트워크로 확장한 프로토콜
> 2. **가치**: 원격 NVMe 성능, 낮은 지연
> 3. **융합**: RDMA, TCP, Fibre Channel

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

NVMe-oF(NVMe over Fabrics)는 **NVMe 프로토콜을 네트워크 패브릭을 통해 확장하여 원격 스토리지에 로컬 NVMe 수준의 성능으로 접근할 수 있게 하는 기술**이다.

### 💡 비유: 원격 고속도로

NVMe-oF는 **원격 고속도로**와 같다. 서울에 고속도로가 있다. 부산에서도 탈 수 있다. 네트워크로 연결되어 있다. 부산에서 서울 고속도로를 쓴다. 로컬처럼 빠르다. 거리가 멀어도 성능은 비슷하다. NVMe-oF도 똑같다. 네트워크로 원격 NVMe를 쓴다.

## Ⅳ. 실무 적용

### NVMe-oF 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                NVMe-oF 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NVMe-oF 타겟 설정】                                                          │)
│  ──────────────────                                                  │
│  # NVMe 타겟 모듈 로드                                                │
│  $ modprobe nvmet                                                   │
│  $ modprobe nvmet-rdma                                              │
│                                                                     │
│  # 서브시스템 생성                                                    │
│  $ cd /sys/kernel/config/nvmet/subsystems                           │
│  $ mkdir subsys1                                                    │
│                                                                     │
│  # 네임스페이스 추가                                                  │
│  $ cd subsys1/namespaces                                            │
│  $ mkdir 1                                                          │
│  $ echo -n /dev/nvme0n1 > 1/device_path                            │
│  $ echo 1 > 1/enable                                                │
│                                                                     │
│  【NVMe-oF 이니시에이터 설정】                                                    │)
│  ──────────────────                                                  │
│  # RDMA 모듈 로드                                                    │
│  $ modprobe rdma_cm                                                 │
│                                                                     │
│  # 타겟 검색                                                         │
│  $ nvme discover -t rdma -a 192.168.1.100                           │
│                                                                     │
│  # 타겟 연결                                                         │
│  $ nvme connect -t rdma -a 192.168.1.100 -n subsys1                 │
│                                                                     │
│  # 연결 확인                                                         │
│  $ nvme list                                                        │
│  Node        SN                   Model              Namespace       │
│  /dev/nvme1n1 S5H2NS0N300001X    NVMe-oF Target     1               │
│                                                                     │
│  【NVMe-oF 성능 측정】                                                            │)
│  ──────────────────                                                  │
│  $ fio --name=test --ioengine=io_uring --rw=randread --bs=4k \      │
│        --direct=1 --size=1G --numjobs=32 /dev/nvme1n1               │
│  read: IOPS=800K, BW=3.1GB/s                                        │
│                                                                     │
│  【NVMe-oF 적합 용도】                                                            │)
│  ──────────────────                                                  │
│  1. 분산 스토리지: Ceph, SPDK                                        │
│  2. 데이터베이스: 분산 DB, 공유 스토리지                               │
│  3. 가상화: VM 공유 스토리지                                          │
│  4. 컨테이너: 영구 볼륨                                               │
│  5. AI/ML: 고속 데이터 파이프라인                                     │
│  6. 하이퍼스케일: 대규모 데이터센터                                   │
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• NVMe-oF: 네트워크 NVMe 프로토콜
• 패브릭: RDMA, TCP, FC
• RDMA: 초저지연, CPU 오프로드
• RoCE: 이더넷 기반 RDMA
• 지연: <10us
• IOPS: 100만+ 원격
• CPU 부하: 매우 낮음
• 확장: 스토리지 분리
• 용도: 분산 DB, 가상화, AI
• 미래: 데이터센터 표준
```
