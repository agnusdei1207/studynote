# 309. 입출력 모듈 (I/O Module)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU와 외부 장치 간 데이터 전송을 관리하는 하드웨어
> 2. **가치**: CPU 부하 감소, 장치 추상화, 데이터 버퍼링
> 3. **융합**: 버스, 인터럽트, DMA, 제어기

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

입출력 모듈(I/O Module)은 **CPU와 외부 장치(주변장치) 사이에서 데이터 전송을 제어하고 관리하는 하드웨어 구성요소**이다. CPU 속도와 주변장치 속도 차이를 조정하고, 데이터 형식 변환, 오류 검출 등을 수행한다.

### 💡 비유: 통역사

I/O 모듈은 **통역사**와 같다. 한국어(CPU)와 영어(주변장치)는 속도와 방식이 다르다. 통역사가 중간에서 조정한다. 말의 속도를 맞추고, 의미를 변환하고, 양쪽이 이해하게 한다. 통역사가 없으면 직접 소통해야 한다.

## Ⅱ. 입출력 모듈 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                입출력 모듈 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【I/O 제어 방식 비교】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 프로그램 제어 I/O (Programmed I/O):                               │ │
│  │     • CPU가 직접 모든 I/O 제어                                       │ │
│  │     • 폴링으로 상태 확인                                             │ │
│  │     • CPU 부하 높음                                                 │ │
│  │     • 간단한 장치에 적합                                              │ │
│  │                                                               │ │
│  │  2. 인터럽트 구동 I/O (Interrupt-driven I/O):                          │ │
│  │     • 장치 준비 시 인터럽트 발생                                       │ │
│  │     • CPU 대기 시간 감소                                             │ │
│  │     • 인터럽트 오버헤드                                               │ │
│  │     • 일반적 방식                                                   │ │
│  │                                                               │ │
│  │  3. DMA (Direct Memory Access):                                      │ │
│  │     • DMA 컨트롤러가 직접 메모리 접근                                  │ │
│  │     • CPU 개입 최소화                                                │ │
│  │     • 대량 데이터 전송에 적합                                         │ │
│  │     • 고속 장치에 사용                                                │ │
│  │                                                               │ │
│  │  4. 채널 I/O (Channel I/O):                                           │ │
│  │     • 독립 I/O 프로세서                                               │ │
│  │     • I/O 프로그램 실행                                               │ │
│  │     • 대형 시스템에서 사용                                             │ │
│  │     • 최고 효율                                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/O 모듈 vs I/O 컨트롤러】                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  I/O 모듈:                                                            │ │
│  │  • 범용적 개념                                                        │ │
│  │  • 시스템 버스와 장치 간 인터페이스                                     │ │
│  │  • 기본 제어 기능                                                    │ │
│  │                                                               │ │
│  │  I/O 컨트롤러 (Device Controller):                                     │ │
│  │  • 특정 장치 전용                                                     │ │
│  │  • 장치별 특수 기능                                                   │ │
│  │  • 예: 디스크 컨트롤러, USB 컨트롤러                                   │ │
│  │                                                               │ │
│  │  관계: I/O 모듈의 구체적 구현 = I/O 컨트롤러                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/O 주소 지정】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  메모리 맵 I/O (Memory-Mapped I/O):                                    │ │
│  │  • I/O 레지스터를 메모리 주소 공간에 매핑                               │ │
│  │  • 일반 메모리 명령어로 접근                                           │ │
│  │  • 주소 공간 일부 사용                                                │ │
│  │                                                               │ │
│  │  분리형 I/O (Isolated I/O):                                            │ │
│  │  • 별도 I/O 주소 공간                                                  │ │
│  │  • IN/OUT 전용 명령어 사용                                             │ │
│  │  • 메모리 주소 공간과 분리                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• I/O 모듈: CPU와 주변장치 간 인터페이스

```
