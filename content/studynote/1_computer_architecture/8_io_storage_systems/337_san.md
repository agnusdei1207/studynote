# 337. SAN (Storage Area Network)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스토리지를 위한 전용 고속 네트워크
> 2. **가치**: 블록 레벨 접근, 중앙 관리, 유연성
> 3. **융합**: Fibre Channel, iSCSI, LUN

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SAN(Storage Area Network)은 **서버와 스토리지 장치를 연결하는 전용 고속 네트워크로, 서버가 스토리지를 로컬 디스크처럼 블록 레벨로 접근할 수 있게 하는 스토리지 아키텍처**다.

### 💡 비유: 회사 전용 도로

SAN은 **회사 전용 도로**와 같다. 일반 도로는 붐빈다. 많은 차가 다닌다. 느리다. 그래서 회사 전용 도로를 만든다. 지게차와 화물차만 다닌다. 빠르다. 창고까지 바로 간다. 공용 도로와 분리되어 있다. SAN도 똑같다. 스토리지 전용 네트워크다.

## Ⅱ. SAN 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SAN 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SAN 프로토콜】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Fibre Channel (FC):                                              │ │
│  │     • 광섬유 기반                                                    │ │
│  │     • 8/16/32 Gbps                                                 │ │
│  │     • 높은 성능, 낮은 지연                                            │ │
│  │     • 비용 높음                                                      │ │
│  │                                                               │ │
│  │  2. iSCSI (Internet SCSI):                                           │ │
│  │     • IP 네트워크 기반                                                │ │
│  │     • 1/10/25/40 Gbps (이더넷)                                       │ │
│  │     • 비용 효율적                                                    │ │
│  │     • 기존 네트워크 활용                                              │ │
│  │                                                               │ │
│  │  3. FCoE (Fibre Channel over Ethernet):                              │ │
│  │     • 이더넷 위에 FC                                                  │ │
│  │     • 10/25/40/100 Gbps                                             │ │
│  │     • FC와 이더넷 융합                                                │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │  프로토콜    속도        비용    복잡도    용도        │        │ │
│  │  │  ────────   ────        ────    ──────    ────        │        │ │
│  │  │  FC         32 Gbps     높음    높음      엔터프라이즈 │        │ │
│  │  │  iSCSI      40 Gbps     낮음    낮음      일반 기업   │        │ │
│  │  │  FCoE       100 Gbps    중간    중간      데이터센터   │        │ │
│  │  │                                                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LUN (Logical Unit Number)】                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정의: 스토리지에서 서버에 할당하는 논리적 단위                          │ │
│  │                                                               │ │
│  │  예:                                                                 │ │
│  │  Storage Array                                                    │ │
│  │  ┌─────────────────────────────────────┐                         │ │
│  │  │  LUN 0: Server A (100GB)            │                         │ │
│  │  │  LUN 1: Server B (200GB)            │                         │ │
│  │  │  LUN 2: Server C (500GB)            │                         │ │
│  │  │  LUN 3: 공유 (1TB)                   │                         │ │
│  │  └─────────────────────────────────────┘                         │ │
│  │                                                               │ │
│  │  → 서버는 LUN을 로컬 디스크로 인식                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. SAN × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SAN × 타 과목 융합                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 블록 I/O: 로컬 디스크처럼                                          │
│  • 버스: FC, 이더넷                                                  │
│  • HBA: 호스트 버스 어댑터                                            │
│  • 대역폭: Gbps 단위                                                 │
│                                                                     │
│  【운영체제】                                                          │
│  • 장치 인식: /dev/sdX                                              │
│  • 파일 시스템: ext4, xfs 등                                         │
│  • 멀티패싱: 경로 이중화                                              │
│  • LUN 마스킹: 접근 제어                                              │
│                                                                     │
│  【네트워크】                                                          │
│  • FC Fabric: 전용 네트워크                                          │
│  • IP 네트워크: iSCSI                                                │
│  • 스위치: SAN 패브릭                                                │
│  • WWN: 식별자                                                      │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 성능: 높은 IOPS                                                   │
│  • 가용성: 스토리지 이중화                                            │
│  • 백업: 스냅샷 활용                                                  │
│  • 확장: 유연한 용량                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SAN vs NAS vs DAS:                                                              │ │
│  │  ──────────────────────                                              │ │
│  │  항목      SAN           NAS           DAS                          │ │
│  │  ────      ────           ────          ────                         │ │
│  │  접근      블록          파일          블록                          │ │
│  │  네트워크  전용          공용          직접 연결                     │ │
│  │  성능      최고          보통          높음                          │ │
│  │  비용      높음          중간          낮음                          │ │
│  │  공유      가능          가능          불가                          │ │
│  │  확장      쉬움          쉬움          어려움                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### SAN 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SAN 분석                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【iSCSI 설정】                                                                 │)
│  ──────────────────                                                  │
│  # iSCSI 이니시에이터 설치                                            │
│  $ yum install iscsi-initiator-utils                                │
│                                                                     │
│  # 타겟 검색                                                          │
│  $ iscsiadm -m discovery -t st -p 192.168.1.100                     │
│  192.168.1.100:3260,1 iqn.2001-04.com.example:storage               │
│                                                                     │
│  # 로그인                                                            │
│  $ iscsiadm -m node -T iqn.2001-04.com.example:storage \            │
│              -p 192.168.1.100 --login                               │
│                                                                     │
│  # LUN 확인                                                          │
│  $ lsblk                                                            │
│  sdb    8:16   0  100G  0 disk   ← 새 LUN                            │
│                                                                     │
│  【LUN 마스킹】                                                                 │)
│  ──────────────────                                                  │
│  # 특정 서버만 LUN 접근 허용                                          │
│  # 스토리지 컨트롤러에서 설정                                          │
│  WWN: 50:01:43:80:00:00:00:00 → LUN 0, 1 접근 허용                   │
│                                                                     │
│  【멀티패싱】                                                                   │)
│  ──────────────────                                                  │
│  # multipath 설정                                                    │
│  $ multipath -ll                                                    │
│  mpatha (3600144f00000000) dm-0                                     │
│  size=100G features='0' hwhandler='0' wp=0                          │
│  `-+- policy='round-robin 0' prio=1 status=active                   │
│    |- 0:0:1:0 sdb 8:16  active ready  ← 경로 1                       │
│    `- 1:0:1:0 sdc 8:32  active ready  ← 경로 2                       │
│                                                                     │
│  【SAN 적합 용도】                                                              │)
│  ──────────────────                                                  │
│  1. 데이터베이스: 높은 IOPS, 블록 접근                                 │
│  2. 가상화: VM 스토리지                                               │
│  3. 엔터프라이즈: 미션 크리티컬                                        │
│  4. 대용량: 중앙 집중 관리                                             │
│  5. 백업: 스토리지 통합                                               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SAN: 스토리지 전용 네트워크
• 접근: 블록 레벨
• 프로토콜: FC, iSCSI, FCoE
• LUN: 논리적 단위 할당
• 장점: 성능, 유연성, 중앙 관리
• 단점: 비용, 복잡성
• HBA: 호스트 버스 어댑터
• WWN: 식별자
• 멀티패싱: 경로 이중화
• 적합: DB, 가상화, 엔터프라이즈
```
