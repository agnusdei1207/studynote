# 336. RAID 10 (Mirrored Stripes)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: RAID 1 미러 + RAID 0 스트라이프 조합
> 2. **가치**: 최고 성능 + 높은 신뢰성
> 3. **융합**: 미러 그룹, 스트라이프, 50% 효율

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

RAID 10(RAID 1+0)은 **RAID 1의 미러링과 RAID 0의 스트라이핑을 결합하여 높은 성능과 신뢰성을 동시에 제공하는 RAID 레벨**이다. 미러링된 디스크 쌍을 스트라이프로 구성한다.

### 💡 비유: 팀별 복사본

RAID 10은 **팀별 복사본**과 같다. 8명이 2명씩 4팀이다. 각 팀은 같은 일을 한다. 팀1: A와 A'. 팀2: B와 B'. 팀3: C와 C'. 팀4: D와 D'. A가 아파도 A'가 한다. 팀1 전체가 아파도 팀2, 3, 4는 일한다. RAID 10도 똑같다. 미러 쌍 + 스트라이프.

### RAID 10 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 10 개요                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RAID 10 구조】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  RAID 10 = RAID 0 (스트라이핑) + RAID 1 (미러링)                       │ │
│  │                                                               │ │
│  │  데이터: [A1][A2][A3][A4][A5][A6][A7][A8]                            │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │                    RAID 0 스트라이프                      │  │ │
│  │  │  ┌───────────┬───────────┬───────────┬───────────┐     │  │ │
│  │  │  │  미러 1   │  미러 2   │  미러 3   │  미러 4   │     │  │ │
│  │  │  │ ┌───┬───┐│ ┌───┬───┐│ ┌───┬───┐│ ┌───┬───┐│     │  │ │
│  │  │  │ │A1 │A1'││ │A2 │A2'││ │A3 │A3'││ │A4 │A4'││     │  │ │
│  │  │  │ │A5 │A5'││ │A6 │A6'││ │A7 │A7'││ │A8 │A8'││     │  │ │
│  │  │  │ └───┴───┘│ └───┴───┘│ └───┴───┘│ └───┴───┘│     │  │ │
│  │  │  │  Disk0,1 │  Disk2,3 │  Disk4,5 │  Disk6,7 │     │  │ │
│  │  │  └───────────┴───────────┴───────────┴───────────┘     │  │ │
│  │  │         ↑ RAID 1 미러링 (각 쌍)                          │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 특성】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  읽기 성능: 최고 (n/2 디스크 병렬 + 미러 분산)                          │ │
│  │  쓰기 성능: 좋음 (n/2 디스크 병렬, 미러 쓰기)                            │ │
│  │  용량 효율: 50%                                                      │ │
│  │  내결함성: 각 미러당 1디스크씩 고장 허용                                  │ │
│  │                                                               │ │
│  │  고장 허용:                                                           │ │
│  │  • 최선: 각 미러에서 1개씩 = n/2디스크                                  │ │
│  │  • 최악: 같은 미러에서 2개 = 전체 손실                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAID 10 vs RAID 01】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  RAID 10: 미러 후 스트라이프                                            │ │
│  │  • (Disk0, Disk1) 미러 → (Disk2, Disk3) 미러 → 스트라이프             │ │
│  │  • 더 안전함                                                          │ │
│  │                                                               │ │
│  │  RAID 01: 스트라이프 후 미러                                            │ │
│  │  • (Disk0, Disk2) 스트라이프 → (Disk1, Disk3) 미러                     │ │
│  │  • 덜 안전함                                                          │ │
│  │                                                               │ │
│  │  → RAID 10이 더 일반적                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. RAID 10 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 10 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【고장 시나리오】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시나리오 1: Disk 0 고장                                              │ │
│  │  → Disk 1에서 데이터 읽기 → 정상 동작                                  │ │
│  │                                                               │ │
│  │  시나리오 2: Disk 0, Disk 1 고장 (같은 미러)                            │ │
│  │  → 미러 1 전체 손실 → 전체 데이터 손실                                  │ │
│  │                                                               │ │
│  │  시나리오 3: Disk 0, Disk 2 고장 (다른 미러)                            │ │
│  │  → 미러 1: Disk 1 사용, 미러 2: Disk 3 사용 → 정상 동작                 │ │
│  │                                                               │ │
│  │  시나리오 4: Disk 0, Disk 2, Disk 4 고장                               │ │
│  │  → 각 미러에서 1개씩 → 정상 동작                                       │ │
│  │                                                               │ │
│  │  고장 확률 비교 (4디스크, 각 5%/년 고장률):                               │ │
│  │  • RAID 10: 0.25% (같은 미러 2개 고장)                                │ │
│  │  • RAID 5: 0.5% (리빌드 중 1개 더 고장)                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【복구 과정】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Disk 0 고장 감지                                                  │ │
│  │  2. Disk 1에서 계속 서비스                                             │ │
│  │  3. 새 디스크 교체                                                     │ │
│  │  4. Disk 1 → 새 디스크로 복사 (전체 볼륨 아님!)                          │ │
│  │  5. 미러링 복구 완료                                                   │ │
│  │                                                               │ │
│  │  리빌드 속도:                                                           │ │
│  │  • RAID 5: 전체 볼륨 스캔                                              │ │
│  │  • RAID 10: 미러 디스크만 복사 → 훨씬 빠름                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【용량 계산】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  총 용량 = (n/2) × 디스크 용량                                         │ │
│  │                                                               │ │
│  │  예: 1TB 디스크 8개                                                  │ │
│  │  총 용량 = 4 × 1TB = 4TB                                            │ │
│  │  용량 효율 = 50%                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### RAID 10 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 10 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RAID 10 생성】                                                               │)
│  ──────────────────                                                  │
│  # 4디스크 RAID 10 생성                                              │
│  $ mdadm --create /dev/md0 --level=10 --raid-devices=4 \            │
│        /dev/sd{a,b,c,d}1                                            │
│                                                                     │
│  # 레이아웃 지정                                                      │
│  $ mdadm --create /dev/md0 --level=10 --layout=n2 --raid-devices=4 \│
│        /dev/sd{a,b,c,d}1                                            │
│  # n2 = near, 2복사본                                                │
│                                                                     │
│  【RAID 10 상태 확인】                                                            │)
│  ──────────────────                                                  │
│  $ cat /proc/mdstat                                                 │
│  md0 : active raid10 sdd1[3] sdc1[2] sdb1[1] sda1[0]                │
│        2095104 blocks super 1.2 512K chunks 2 near-copies           │
│                                                                     │
│  $ mdadm --detail /dev/md0                                          │
│  Raid Level : raid10                                                │
│  Array Size : 2095104 (2.00 TiB)                                    │
│  Layout : near=2                                                    │
│                                                                     │
│  【복구】                                                                       │)
│  ──────────────────                                                  │
│  # 디스크 고장 표시 및 제거                                            │
│  $ mdadm --manage /dev/md0 --fail /dev/sda1 --remove /dev/sda1      │
│                                                                     │
│  # 새 디스크 추가 (자동 리빌드)                                        │
│  $ mdadm --manage /dev/md0 --add /dev/sde1                          │
│                                                                     │
│  【RAID 10 적합 용도】                                                            │)
│  ──────────────────                                                  │
│  1. 데이터베이스 OLTP: 높은 랜덤 I/O                                  │
│  2. 이메일 서버: 빠른 읽기/쓰기                                       │
│  3. 가상화: VM 스토리지                                               │
│  4. ERP/CRM: 트랜잭션 처리                                           │
│  5. 고성능 웹 서버: 빠른 응답                                         │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• RAID 10: RAID 1 + RAID 0 조합
• 구조: 미러 쌍을 스트라이프
• 최소: 4개 디스크
• 용량: 50% 효율
• 읽기: 최고 성능
• 쓰기: 높은 성능
• 신뢰성: 각 미러당 1디스크
• 복구: 빠름 (미러만 복사)
• 레이아웃: near, far, offset
• 적합: DB OLTP, 가상화, 이메일
```
