# 317. ISR (Interrupt Service Routine)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 발생 시 실행되는 처리 함수
> 2. **가치**: 빠른 응답, 하드웨어 이벤트 처리
> 3. **융합**: 인터럽트, 컨텍스트, Top/Bottom Half

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

ISR(Interrupt Service Routine)은 **인터럽트가 발생했을 때 CPU가 실행하는 전용 처리 함수**이다. 하드웨어 인터럽트, 예외, 소프트웨어 인터럽트 등 각 인터럽트마다 해당하는 ISR이 존재한다.

### 💡 비유: 비상 대응 팀

ISR은 **비상 대응 팀**과 같다. 화재 경보가 울렸다. 소방팀(ISR)이 출동한다. 소방팀은 화재만 진압한다. 화재 후 복구는 다른 팀이 한다. 소방팀은 빨리 끝내야 한다. 오래 걸리면 다른 화재를 못 간다. ISR도 빨리 끝내야 한다.

### ISR 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                ISR 개요                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ISR 실행 흐름】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 인터럽트 발생:                                                   │ │
│  │     • 하드웨어/소프트웨어가 인터럽트 신호                               │ │
│  │     • CPU가 현재 명령어 완료                                         │ │
│  │                                                               │ │
│  │  2. 하드웨어 자동 처리:                                               │ │
│  │     • 플래그 레지스터 저장                                            │ │
│  │     • 인터럽트 금지 (IF=0)                                           │ │
│  │     • CS:IP 저장                                                   │ │
│  │                                                               │ │
│  │  3. ISR 진입:                                                        │ │
│  │     • 벡터 테이블에서 ISR 주소 로드                                    │ │
│  │     • ISR 시작                                                     │ │
│  │                                                               │ │
│  │  4. ISR 본체:                                                        │ │
│  │     • 레지스터 저장 (Push)                                           │ │
│  │     • 인터럽트 처리                                                   │ │
│  │     • 레지스터 복원 (Pop)                                            │ │
│  │                                                               │ │
│  │  5. ISR 종료 (IRET):                                                 │ │
│  │     • 저장된 상태 복원                                                │ │
│  │     • 인터럽트 복귀                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ISR 구조 예시】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  my_isr:                                                            │ │
│  │      push    eax            ; 레지스터 저장                        │ │
│  │      push    ebx                                                 │ │
│  │      push    ecx                                                 │ │
│  │      ;                                                            │ │
│  │      ;  인터럽트 처리 코드                                          │ │
│  │      ;  (빠르게 수행해야 함)                                         │ │
│  │      ;                                                            │ │
│  │      pop     ecx            ; 레지스터 복원                        │ │
│  │      pop     ebx                                                 │ │
│  │      pop     eax                                                 │ │
│  │      iret                     ; 인터럽트 복귀                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ISR 작성 규칙】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 빠른 실행: 다른 인터럽트 지연 방지                                 │ │
│  │  2. 레지스터 저장/복원: 컨텍스트 보호                                  │ │
│  │  3. 재진입 불가: ISR 중 같은 ISR 실행 안 됨                           │ │
│  │  4. 블로킹 금지: sleep, I/O 대기 불가                                 │ │
│  │  5. 인터럽트 금지 최소화                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. ISR 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ISR 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Top Half vs Bottom Half】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Top Half (ISR 본체):                                                │ │
│  │  • 인터럽트 금지 상태로 실행                                          │ │
│  │  • 최소한의 작업만 수행                                              │ │
│  │  • 빠른 실행 필수                                                   │ │
│  │  • 하드웨어 처리, 데이터 복사                                        │ │
│  │                                                               │ │
│  │  Bottom Half (지연 처리):                                            │ │
│  │  • 인터럽트 허용 상태로 실행                                          │ │
│  │  • 시간이 걸리는 작업                                                │ │
│  │  • 나중에 스케줄됨                                                   │ │
│  │  • Softirq, Tasklet, Workqueue                                     │ │
│  │                                                               │ │
│  │  ┌─────────────┐                    ┌─────────────┐             │ │
│  │  │   Top Half  │                    │ Bottom Half │             │ │
│  │  │    (ISR)    │ ── 스케줄 ──→       │  (지연처리)  │             │ │
│  │  │             │                    │             │             │ │
│  │  │ • 하드웨어  │                    │ • 데이터    │             │ │
│  │  │   처리      │                    │   처리      │             │ │
│  │  │ • 빠른      │                    │ • 느린      │             │ │
│  │  │ • 인터럽트  │                    │ • 인터럽트  │             │ │
│  │  │   금지      │                    │   허용      │             │ │
│  │  └─────────────┘                    └─────────────┘             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux Bottom Half 종류】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Softirq:                                                            │ │
│  │  • 가장 빠른 Bottom Half                                            │ │
│  │  • 인터럽트 컨텍스트에서 실행                                         │ │
│  │  • 네트워크, 블록 I/O                                               │ │
│  │                                                               │ │
│  │  Tasklet:                                                            │ │
│  │  • Softirq 기반                                                     │ │
│  │  • 동시 실행 방지                                                   │ │
│  │  • 일반 드라이버                                                    │ │
│  │                                                               │ │
│  │  Workqueue:                                                          │ │
│  │  • 프로세스 컨텍스트에서 실행                                         │ │
│  │  • sleep 가능                                                      │ │
│  │  • 복잡한 작업                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ISR 제약사항】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ISR에서 하지 말아야 할 것:                                           │ │
│  │  • sleep / schedule()                                               │ │
│  │  • 사용자 메모리 접근 (copy_to_user 등)                               │ │
│  │  • 긴 루프                                                          │ │
│  │  • 다른 ISR 대기                                                    │ │
│  │  • 페이지 폴트 유발                                                  │ │
│  │  • 뮤텍스 획득 (세마포는 가능하나 권장 안 함)                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. ISR × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ISR × 타 과목 융합                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 컨텍스트: 레지스터 저장/복원                                        │
│  • 인터럽트 금지: IF 플래그                                           │
│  • IRET: 인터럽트 복귀 명령어                                         │
│  • 우선순위: ISR 중 중첩 인터럽트                                      │
│                                                                     │
│  【운영체제】                                                          │
│  • Softirq: 커널 Bottom Half                                         │
│  • Tasklet: 동적 Bottom Half                                         │
│  • Workqueue: 프로세스 컨텍스트                                       │
│  • 스케줄링: ISR 완료 후 스케줄                                        │
│                                                                     │
│  【디바이스 드라이버】                                                    │
│  • IRQ 핸들러: request_irq() 등록                                     │
│  • 데이터 전송: 하드웨어 → 버퍼                                        │
│  • 이벤트 알림: wait queue 깨움                                       │
│  • DMA 완료: DMA 인터럽트 처리                                        │
│                                                                     │
│  【네트워크】                                                          │
│  • NAPI: ISR → 폴링 전환                                             │
│  • 패킷 수신: NIC 인터럽트                                            │
│  • Softirq: NET_RX_SOFTIRQ                                          │
│  • 부하 분산: RPS/RFS                                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Bottom Half 비교:                                                              │ │
│  │  ──────────────────────                                              │ │
│  │  종류        컨텍스트    Sleep   실행 시점    용도                     │ │
│  │  ────        ────────    ─────   ────────    ────                     │ │
│  │  Softirq     인터럽트    X       즉시        네트워크, 블록            │ │
│  │  Tasklet     인터럽트    X       Softirq    일반 드라이버              │ │
│  │  Workqueue   프로세스    O       나중        복잡한 작업               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### ISR 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ISR 분석                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux ISR 등록】                                                              │)
│  ──────────────────                                                  │
│  // 인터럽트 핸들러                                                   │
│  irqreturn_t my_interrupt(int irq, void *dev_id)                    │
│  {                                                                  │
│      // Top Half: 최소 작업                                          │
│      data = read_hardware();                                        │
│                                                                     │
│      // Bottom Half 스케줄                                           │
│      tasklet_schedule(&my_tasklet);                                 │
│                                                                     │
│      return IRQ_HANDLED;                                            │
│  }                                                                  │
│                                                                     │
│  // 등록                                                             │
│  request_irq(IRQ_NUM, my_interrupt, IRQF_SHARED, "mydev", &dev);    │
│                                                                     │
│  【Tasklet 예시】                                                              │)
│  ──────────────────                                                  │
│  void my_tasklet_func(unsigned long data)                           │
│  {                                                                  │
│      // Bottom Half: 시간 걸리는 작업                                 │
│      process_data(data);                                            │
│      wake_up(&wait_queue);                                          │
│  }                                                                  │
│                                                                     │
│  DECLARE_TASKLET(my_tasklet, my_tasklet_func, 0);                   │
│                                                                     │
│  【ISR 성능 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/softirqs                                               │
│                    CPU0       CPU1                                   │
│  HI:              123        456       # High priority tasklets    │
│  TIMER:        123456     123456       # Timer softirqs            │
│  NET_TX:          123        456       # Network TX                 │
│  NET_RX:        12345      12345       # Network RX                 │
│  BLOCK:          1234       2345       # Block device               │
│                                                                     │
│  【ISR 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. Top Half 최소화: 하드웨어 처리만                                  │
│  2. Bottom Half 사용: 긴 작업은 위임                                  │
│  3. Tasklet vs Workqueue: 상황에 맞게 선택                            │
│  4. 인터럽트 코알레싱: 인터럽트 횟수 감소                               │
│  5. NAPI: 네트워크 고부하 시                                         │
│  6. IRQ 부하 분산: SMP 시스템에서                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• ISR: 인터럽트 서비스 루틴
• 실행: 인터럽트 발생 시 CPU가 호출
• 구조: 저장 → 처리 → 복원 → IRET
• 규칙: 빠른 실행, 레지스터 보존, 재진입 불가
• Top Half: ISR 본체, 빠른 처리
• Bottom Half: 지연 처리, 시간 걸리는 작업
• Softirq: 가장 빠른 Bottom Half
• Tasklet: 동적, 동시 실행 방지
• Workqueue: 프로세스 컨텍스트, sleep 가능
• 관련: 인터럽트, 인터럽트 벡터
```
