# 335. RAID 6 (Dual Parity)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이중 패리티로 2디스크 동시 고장 대응
> 2. **가치**: 높은 신뢰성, 대용량에 적합
> 3. **융합**: P+Q 패리티, 리드-솔로몬, 최소 4디스크

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

RAID 6(Dual Parity)은 **두 개의 독립적인 패리티(P와 Q)를 사용하여 최대 2개 디스크의 동시 고장까지 데이터를 복구할 수 있는 RAID 레벨**이다. RAID 5보다 높은 신뢰성을 제공하지만 쓰기 오버헤드가 더 크다.

### 💡 비유: 두 개의 암호

RAID 6은 **두 개의 암호**와 같다. 비밀 금고가 있다. 비밀번호가 필요하다. 하지만 잊어버릴 수 있다. 그래서 두 가지 방법을 만든다. 첫 번째 비밀번호와 두 번째 지문. 둘 중 하나만 있어도 열린다. 첫 번째를 잊어도 지문으로 열린다. RAID 6도 똑같다. 패리티 P와 Q 두 개로 보호한다.

### RAID 6 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 6 개요                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【이중 패리티 원리】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  패리티 P: XOR 기반 (RAID 5와 동일)                                   │ │
│  │  패리티 Q: 갈루아 필드 연산 (P와 독립적)                                │ │
│  │                                                               │ │
│  │         Disk 0   Disk 1   Disk 2   Disk 3   Disk 4             │ │
│  │         ┌────┐   ┌────┐   ┌────┐   ┌────┐   ┌────┐             │ │
│  │  스트라이프0│ A1 │   │ A2 │   │ A3 │   │ P  │   │ Q  │             │ │
│  │  스트라이프1│ A4 │   │ A5 │   │ P  │   │ Q  │   │ A6 │             │ │
│  │  스트라이프2│ A7 │   │ P  │   │ Q  │   │ A8 │   │ A9 │             │ │
│  │         └────┘   └────┘   └────┘   └────┘   └────┘             │ │
│  │                                                               │ │
│  │  → P와 Q가 서로 다른 디스크에 분산                                      │ │
│  │  → 2디스크 동시 고장까지 복구 가능                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 특성】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  읽기 성능: 좋음 (n-2 디스크 병렬)                                     │ │
│  │  쓰기 성능: 낮음 (이중 패리티 갱신)                                     │ │
│  │  용량 효율: (n-2)/n                                                 │ │
│  │  내결함성: 2디스크 고장 허용                                            │ │
│  │                                                               │ │
│  │  디스크 수에 따른 용량 효율:                                            │ │
│  │  • 4 disks: 50% (2/4)                                               │ │
│  │  • 5 disks: 60% (3/5)                                               │ │
│  │  • 6 disks: 67% (4/6)                                               │ │
│  │  • 8 disks: 75% (6/8)                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAID 5 vs RAID 6】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  항목          RAID 5           RAID 6                         │ │
│  │  ────          ──────           ──────                         │ │
│  │  최소 디스크    3               4                              │ │
│  │  고장 허용     1               2                              │ │
│  │  용량 효율     (n-1)/n         (n-2)/n                         │ │
│  │  쓰기 성능     보통             낮음                            │ │
│  │  신뢰성        보통             높음                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. RAID 6 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 6 상세                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【패리티 P와 Q】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  패리티 P (XOR):                                                      │ │
│  │  P = D1 ⊕ D2 ⊕ D3                                                   │ │
│  │                                                               │ │
│  │  패리티 Q (갈루아 필드):                                               │ │
│  │  Q = g^0·D1 ⊕ g^1·D2 ⊕ g^2·D3 ⊕ ...                                 │ │
│  │                                                               │ │
│  │  1디스크 복구:                                                         │ │
│  │  P만 사용 (RAID 5와 동일)                                             │ │
│  │                                                               │ │
│  │  2디스크 복구:                                                         │ │
│  │  P와 Q 연립 방정식으로 계산                                            │ │
│  │  D1, D2 손실 시:                                                      │ │
│  │  P = D1 ⊕ D2 ⊕ D3                                                   │ │
│  │  Q = g^0·D1 ⊕ g^1·D2 ⊕ g^2·D3                                       │ │
│  │  → 2개 방정식으로 D1, D2 복구                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【쓰기 페널티】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  작은 쓰기 (Small Write):                                            │ │
│  │  1. 기존 데이터 읽기                                                  │ │
│  │  2. 기존 패리티 P 읽기                                                │ │
│  │  3. 기존 패리티 Q 읽기                                                │ │
│  │  4. 새 패리티 P 계산                                                  │ │
│  │  5. 새 패리티 Q 계산                                                  │ │
│  │  6. 새 데이터 쓰기                                                   │ │
│  │  7. 새 패리티 P 쓰기                                                 │ │
│  │  8. 새 패리티 Q 쓰기                                                 │ │
│  │                                                               │ │
│  │  → 6 I/O 작업 필요 (읽기 3 + 쓰기 3)                                  │ │
│  │  → 쓰기 페널티: 6                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【용량 계산】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  총 용량 = (n-2) × 디스크 용량                                         │ │
│  │                                                               │ │
│  │  예: 2TB 디스크 6개                                                  │ │
│  │  총 용량 = 4 × 2TB = 8TB                                            │ │
│  │  용량 효율 = 67%                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. RAID 6 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 6 × 타 과목 융합                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 갈루아 필드: Q 패리티 계산                                         │
│  • XOR: P 패리티 계산                                               │
│  • 컨트롤러: 복잡한 연산                                              │
│  • 캐시: 쓰기 성능 향상                                               │
│                                                                     │
│  【운영체제】                                                          │
│  • mdadm: RAID 6 관리                                                │
│  • ZFS: RAID-Z2 (RAID 6 유사)                                        │
│  • 리빌드: 오래 걸림                                                  │
│  • 스크럽: 정기적 검증                                                │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 아카이브: 높은 신뢰성 필요                                          │
│  • 백업: 데이터 보호                                                  │
│  • 읽기: 좋은 성능                                                    │
│  • 쓰기: 높은 오버헤드                                                │
│                                                                     │
│  【신뢰성】                                                            │
│  • MTTDL: 매우 높음                                                  │
│  • 대용량: URE 위험 감소                                              │
│  • 리빌드 중: 추가 고장 허용                                           │
│  • 엔터프라이즈: 권장                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  대용량 디스크에서 RAID 5 vs RAID 6:                                               │ │
│  │  ──────────────────────                                              │ │
│  │  구성            리빌드 중 URE 확률   데이터 손실 위험                  │ │
│  │  ────            ────────────────   ────────────                     │ │
│  │  RAID 5 8TB     약 50%             높음                             │ │
│  │  RAID 6 8TB     약 0%              매우 낮음                          │ │
│  │                                                               │ │
│  │  → 대용량 디스크에서는 RAID 6 권장                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### RAID 6 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 6 분석                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RAID 6 생성】                                                                │)
│  ──────────────────                                                  │
│  # 6디스크 RAID 6 생성                                               │
│  $ mdadm --create /dev/md0 --level=6 --raid-devices=6 \             │
│        /dev/sd{a,b,c,d,e,f}1                                        │
│                                                                     │
│  【RAID 6 상태 확인】                                                            │)
│  ──────────────────                                                  │
│  $ cat /proc/mdstat                                                 │
│  md0 : active raid6 sdf1[5] sde1[4] sdd1[3] sdc1[2] sdb1[1] sda1[0] │
│        8386560 blocks super 1.2 level 6, 512k chunk                 │
│                                                                     │
│  $ mdadm --detail /dev/md0                                          │
│  Raid Level : raid6                                                 │
│  Array Size : 8386560 (8.00 TiB)                                    │
│  Failed Disks: 0                                                    │
│                                                                     │
│  【ZFS RAID-Z2】                                                                │)
│  ──────────────────                                                  │
│  # RAID-Z2 풀 생성 (RAID 6 유사)                                     │
│  $ zpool create tank raidz2 /dev/sd{a,b,c,d,e,f}                    │
│                                                                     │
│  【RAID 6 적합 용도】                                                            │)
│  ──────────────────                                                  │
│  1. 대용량 스토리지: 4TB+ 디스크                                      │
│  2. 아카이브: 장기 보관                                              │
│  3. 백업: 높은 신뢰성                                                │
│  │  4. 엔터프라이즈: 미션 크리티컬                                      │ │
│  │  5. NAS: 가정/소호                                                │ │
│  │  6. 오브젝트 스토리지: 높은 내구성                                    │ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• RAID 6: 이중 패리티 (P + Q)
• 최소: 4개 디스크
• 용량: (n-2)/n 효율
• 신뢰성: 2디스크 고장 허용
• 읽기: 좋음 (n-2 디스크)
• 쓰기: 낮음 (6 I/O 페널티)
• P: XOR 기반
• Q: 갈루아 필드 기반
• 대용량: RAID 5보다 안전
• 적합: 아카이브, 백업, 엔터프라이즈
```
