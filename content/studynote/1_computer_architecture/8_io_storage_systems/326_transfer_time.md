# 326. 전송 시간 (Transfer Time)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 디스크에서 읽거나 쓰는 실제 시간
> 2. **가치**: 대역폭 활용, 전송 속도 결정
> 3. **융합**: 액세스 시간, 전송률, 버스트

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

전송 시간(Transfer Time)은 **헤드가 목표 섹터에 도달한 후, 실제 데이터를 읽거나 쓰는 데 걸리는 시간**이다. 데이터 양과 전송 속도에 따라 결정되며, 탐색 시간과 회전 지연에 비해 매우 짧다.

### 💡 비유: 복사기 출력

전송 시간은 **복사기 출력**과 같다. 복사기에 갔다. 복사할 차례다. 버튼을 눌렀다. 종이가 나온다. 1장은 1초다. 10장은 10초다. 종이 수에 비례한다. 복사기 속도에 따라 다르다. 빠른 복사기는 1초에 10장 나온다.

### 전송 시간 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                전송 시간 개요                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전송 시간 계산】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  전송 시간 = 데이터 양 / 전송 속도                                       │ │
│  │                                                               │ │
│  │  예:                                                                 │ │
│  │  • 데이터: 4KB (4096 bytes)                                          │ │
│  │  • 전송 속도: 100 MB/s                                                │ │
│  │  • 전송 시간 = 4096 / (100 × 10^6) = 40.96us                         │ │
│  │                                                               │ │
│  │  → 마이크로초 단위로 매우 빠름                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【액세스 시간에서 전송 시간 위치】                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  총 액세스 시간: 12ms (예)                                             │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │                                                       │  │ │
│  │  │  ┌──────────┬──────────┬──────────┐                   │  │ │
│  │  │  │ 탐색 8ms  │회전지연4ms│ 전송 <1ms │                   │  │ │
│  │  │  │  (67%)   │  (33%)   │  (~0%)   │                   │  │ │
│  │  │  └──────────┴──────────┴──────────┘                   │  │ │
│  │  │                                                       │  │ │
│  │  │  → 전송 시간은 전체 액세스 시간의 아주 작은 부분             │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【전송 속도 요인】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 회전 속도 (RPM):                                                  │ │
│  │     • 높은 RPM = 높은 전송 속도                                        │ │
│  │                                                               │ │
│  │  2. 트랙 밀도:                                                        │ │
│  │     • 높은 밀도 = 한 회전당 더 많은 데이터                              │ │
│  │                                                               │ │
│  │  3. 인터페이스:                                                       │ │
│  │     • SATA, SAS, NVMe                                                │ │
│  │                                                               │ │
│  │  4. 버스트 전송:                                                      │ │
│  │     • 캐시를 통한 버스트                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 전송 시간 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                전송 시간 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전송 속도 계산】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  이론적 전송 속도:                                                     │ │
│  │                                                               │ │
│  │  전송 속도 = RPM/60 × 섹터/트랙 × 바이트/섹터                           │ │
│  │                                                               │ │
│  │  예: 7200 RPM, 1000 섹터/트랙, 512B/섹터                              │ │
│  │                                                               │ │
│  │  120회전/초 × 1000섹터 × 512B = 61,440,000 B/s                      │ │
│  │                                  ≈ 61 MB/s (이론적)                  │ │
│  │                                                               │ │
│  │  실제 전송 속도:                                                       │ │
│  │  • 오버헤드, ECC, 포맷팅 등으로 인해 낮아짐                             │ │
│  │  • 일반적으로 80 ~ 200 MB/s                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대용량 vs 소용량 전송】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  소용량 (4KB):                                                        │ │
│  │  • 전송 시간: ~40us                                                   │ │
│  │  • 액세스 시간 대비 무시 가능                                          │ │
│  │  • 전체 시간의 <1%                                                    │ │
│  │                                                               │ │
│  │  대용량 (1MB):                                                        │ │
│  │  • 전송 시간: ~10ms                                                   │ │
│  │  • 액세스 시간과 비슷                                                   │ │
│  │  • 전체 시간의 ~50%                                                   │ │
│  │                                                               │ │
│  │  대용량 (1GB):                                                        │ │
│  │  • 전송 시간: ~10s                                                    │ │
│  │  • 액세스 시간 대비 지배적                                              │ │
│  │  • 전체 시간의 ~99%                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순차 vs 랜덤 전송】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  순차 전송:                                                            │ │
│  │  • 연속 섹터 읽기/쓰기                                                 │ │
│  │  • 탐색, 회전 지연 최소                                                │ │
│  │  • 최대 전송 속도 달성                                                 │ │
│  │                                                               │ │
│  │  랜덤 전송:                                                            │ │
│  │  • 불연속 섹터 접근                                                   │ │
│  │  • 매번 탐색, 회전 지연 발생                                           │ │
│  │  • 전송 속도 급감                                                     │ │
│  │                                                               │ │
│  │  예:                                                                  │ │
│  │  순차: 150 MB/s                                                       │ │
│  │  랜덤 4KB: ~0.5 MB/s (300배 느림)                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 전송 시간: 실제 데이터 읽기/쓰기 시간
• 계산: 데이터 양 / 전송 속도
• 특징: 마이크로초~초 단위, 데이터 양 비례
• 비중: 소량 <1%, 대량 >50%
• 순차: 전송 시간 활용 최적화
• 랜덤: 탐색/회전 지연이 지배적
• HDD: 100-200 MB/s
• SSD: 500-7000 MB/s
• 인터페이스: SATA, SAS, NVMe
• 관련: 탐색, 회전 지연, 액세스 시간
```
