# 329. 마모 평준화 (Wear Leveling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 플래시 블록에 쓰기를 고르게 분산
> 2. **가치**: SSD 수명 연장, 조기 고장 방지
> 3. **융합**: 플래시, P/E 사이클, GC

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

마모 평준화(Wear Leveling)는 **플래시 메모리의 모든 블록에 쓰기 및 삭제 사이클을 균등하게 분산시켜 특정 블록의 조기 마모를 방지하고 전체 SSD 수명을 연장하는 기술**이다.

### 💡 비유: 타이어 교체

마모 평준화는 **타이어 위치 교체**와 같다. 차의 앞바퀴가 뒷바퀴보다 마모가 심하다. 그대로 두면 앞바퀴만 먼저 닳는다. 그래서 정기적으로 타이어 위치를 바꾼다. 앞→뒤, 뒤→앞. 모든 타이어가 고르게 닳게 한다. 그러면 모든 타이어를 오래 쓸 수 있다. 플래시도 똑같다. 모든 블록이 고르게 닳게 한다.

### 마모 평준화 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                마모 평준화 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【필요 이유】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  플래시 수명:                                                          │ │
│  │  • NAND 플래시는 유한한 쓰기/삭제 사이클 (P/E Cycle)                    │ │
│  │  • SLC: ~100,000 사이클                                              │ │
│  │  • MLC: ~3,000 사이클                                                │ │
│  │  • TLC: ~1,000 사이클                                                │ │
│  │  • QLC: ~100-300 사이클                                              │ │
│  │                                                               │ │
│  │  문제:                                                                │ │
│  │  • 특정 파일 자주 수정 → 특정 블록만 집중 사용                          │ │
│  │  • 해당 블록만 빨리 수명 종료 → 전체 SSD 고장                          │ │
│  │                                                               │ │
│  │  해결:                                                                │ │
│  │  • 모든 블록에 쓰기를 균등 분산                                        │ │
│  │  • 모든 블록이 비슷한 시점에 수명 종료                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【마모 평준화 없이】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  블록별 쓰기 횟수:                                                      │ │
│  │                                                               │ │
│  │  블록 0: ████████████████████ 100,000회 (수명 종료!)                  │ │
│  │  블록 1: ███ 3,000회                                               │ │
│  │  블록 2: ██ 2,000회                                                │ │
│  │  블록 3: █ 1,000회                                                 │ │
│  │  ...                                                              │ │
│  │  블록 N:   100회                                                   │ │
│  │                                                               │ │
│  │  → 블록 0 고장 → 전체 SSD 사용 불가                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【마모 평준화 적용】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  블록별 쓰기 횟수:                                                      │ │
│  │                                                               │ │
│  │  블록 0: ████████ 10,000회                                         │ │
│  │  블록 1: ████████ 10,000회                                         │ │
│  │  블록 2: ████████ 10,000회                                         │ │
│  │  블록 3: ████████ 10,000회                                         │ │
│  │  ...                                                              │ │
│  │  블록 N: ████████ 10,000회                                         │ │
│  │                                                               │ │
│  │  → 모든 블록 고르게 사용 → 수명 최대화                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 마모 평준화 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                마모 평준화 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【마모 평준화 종류】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 동적 마모 평준화 (Dynamic Wear Leveling):                          │ │
│  │     • 활성(사용 중) 블록만 대상                                        │ │
│  │     • 쓰기가 많은 블록을 덜 쓴 블록으로 이동                             │ │
│  │     • 구현 간단                                                       │ │
│  │     • 정적 데이터는 이동 안 함                                          │ │
│  │                                                               │ │
│  │  2. 정적 마모 평준화 (Static Wear Leveling):                            │ │
│  │     • 모든 블록(사용 중 + 비어있음) 대상                                │ │
│  │     • 자주 안 쓰는 블록의 데이터도 이동                                 │ │
│  │     • 더 완벽한 평준화                                                 │ │
│  │     • 오버헤드 큼                                                      │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │               동적 vs 정적 비교                      │        │ │
│  │  │  ──────────────────────                              │        │ │
│  │  │  항목          동적           정적                    │        │ │
│  │  │  ────          ────           ────                    │        │ │
│  │  │  대상          사용 블록만    모든 블록               │        │ │
│  │  │  평준화 품질    보통           높음                   │        │ │
│  │  │  오버헤드       낮음           높음                   │        │ │
│  │  │  수명 연장      보통           높음                   │        │ │
│  │  │                                                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【마모 평준화 동작 과정】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 쓰기 요청 도착                                                     │ │
│  │                                                               │ │
│  │  2. 가장 적게 사용된 블록 찾기                                          │ │
│  │     • P/E 카운트 확인                                                  │ │
│  │     • 최소 카운트 블록 선택                                             │ │
│  │                                                               │ │
│  │  3. 데이터 쓰기                                                        │ │
│  │     • 선택된 블록에 쓰기                                               │ │
│  │     • P/E 카운트 증가                                                  │ │
│  │                                                               │ │
│  │  4. (정적) 정적 데이터 이동                                             │ │
│  │     • 자주 안 쓰는 블록의 데이터 이동                                   │ │
│  │     • 해당 블록 사용 가능하게                                          │ │
│  │                                                               │ │
│  │  5. FTL 매핑 테이블 갱신                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【P/E 사이클과 수명】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  TBW (Terabytes Written) 계산:                                         │ │
│  │  TBW = 용량 × P/E 사이클 / 1000                                        │ │
│  │                                                               │ │
│  │  예: 1TB TLC SSD (1000 P/E)                                           │ │
│  │  TBW = 1TB × 1000 / 1000 = 1000 TB                                    │ │
│  │  = 1 PB (페타바이트)                                                   │ │
│  │                                                               │ │
│  │  일반 사용: 하루 100GB 쓰기                                             │ │
│  │  수명 = 1000TB / (100GB × 365일) = 27년                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 마모 평준화 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                마모 평준화 × 타 과목 융합                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • NAND 플래시: 물리적 수명                                          │
│  • P/E 사이클: 쓰기/삭제 횟수                                        │
│  • 컨트롤러: 마모 평준화 알고리즘                                     │
│  • FTL: 주소 매핑으로 평준화 구현                                     │
│                                                                     │
│  【운영체제】                                                          │
│  • TRIM: 삭제 통지로 평준화 지원                                      │
│  • 파일 시스템: 쓰기 패턴 영향                                        │
│  • I/O 스케줄링: 쓰기 패턴 조정                                       │
│  • 캐시: 쓰기 감소로 수명 연장                                        │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 쓰기 패턴: 로그, 체크포인트                                        │
│  • WAL: 순차 쓰기로 평준화                                            │
│  • 버퍼 풀: 쓰기 모아서 처리                                          │
│  • SSD 최적화: 쓰기 증폭 고려                                         │
│                                                                     │
│  【성능】                                                              │
│  • 오버헤드: 마모 평준화로 인한 추가 I/O                               │
│  • 수명: 수명 ↔ 성능 트레이드오프                                     │
│  • TBW: 총 쓰기 보장량                                               │
│  • DWPD: 일일 쓰기량                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  NAND 타입별 수명:                                                              │ │
│  │  ──────────────────────                                              │ │
│  │  타입    셀당 비트  P/E 사이클    용도                               │ │
│  │  ────    ────────  ──────────    ────                               │ │
│  │  SLC     1         100,000      엔터프라이즈                          │ │
│  │  MLC     2         3,000        프로/엔터프라이즈                      │ │
│  │  TLC     3         1,000        일반 소비자                          │ │
│  │  QLC     4         100-300      보관용/저렴                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 마모 평준화 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                마모 평준화 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SSD 수명 확인】                                                              │)
│  ──────────────────                                                  │
│  $ smartctl -a /dev/nvme0n1                                         │
│  Data Units Written:                 10,000,000 [5.12 TB]           │
│  Percentage Used:                    5%                             │
│  # 5.12TB 쓰기, 수명 5% 사용                                          │
│                                                                     │
│  【TBW 확인】                                                                   │)
│  ──────────────────                                                  │
│  # SSD 제조사 사양서 확인                                             │
│  # 예: Samsung 970 EVO 1TB                                           │
│  # TBW: 600 TB                                                       │
│  # 보증: 5년                                                          │
│                                                                     │
│  【쓰기 증폭 확인】                                                              │)
│  ──────────────────                                                  │
│  $ nvme get-feature -f 0x0a -H /dev/nvme0n1                          │
│  get-feature:0xa (LBA Range Type)                                   │
│                                                                     │
│  【마모 평준화 최적화 팁】                                                         │)
│  ──────────────────                                                  │
│  1. TRIM 활성화: 마모 평준화 지원                                    │
│  2. 여유 공간: 10-20% 확보                                           │
│  3. 쓰기 최소화: 불필요한 쓰기 회피                                   │
│  4. 로그 파일: RAM 디스크 고려                                       │
│  5. SWAP: 충분한 RAM으로 최소화                                      │
│  6. 정기 모니터링: SMART 확인                                        │
│  7. 백업: 수명 종료 전 교체                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 마모 평준화: 모든 블록에 쓰기 균등 분산
• 목적: SSD 수명 연장
• 동적: 사용 블록만 평준화
• 정적: 모든 블록 평준화
• P/E 사이클: 쓰기/삭제 횟수
• SLC: ~100,000, MLC: ~3,000
• TLC: ~1,000, QLC: ~100-300
• TBW: 총 쓰기 보장량
• DWPD: 일일 쓰기량
• 관련: SSD, GC, FTL, TRIM
```
