# 330. FTL (Flash Translation Layer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 논리 주소를 물리 주소로 변환하는 SSD의 핵심 계층
> 2. **가치**: 플래시 특성 숨기고 일반 블록 장치처럼 동작
> 3. **융합**: 주소 매핑, GC, 마모 평준화, 캐시

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

FTL(Flash Translation Layer)은 **호스트의 논리 블록 주소(LBA)를 플래시 메모리의 물리 주소(PPN)로 변환하고, 플래시의 특수한 특성(덮어쓰기 불가, 블록 단위 삭제)을 숨겨 일반 저장장치처럼 보이게 하는 소프트웨어 계층**이다.

### 💡 비유: 호텔 리셉션

FTL은 **호텔 리셉션**과 같다. 손님이 "101호 예약했어요"라고 한다. 리셉션은 예약 시스템을 본다. 101호가 아니라 305호가 배정되어 있다. "305호로 가세요"라고 안내한다. 손님은 101호라고 생각하지만 실제는 305호다. 다음에 또 오면 201호일 수도 있다. 리셉션이 방 배정을 관리한다. FTL도 주소 변환을 관리한다.

## Ⅱ. FTL 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                FTL 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 매핑 방식】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 페이지 매핑 (Page Mapping):                                       │ │
│  │     • LBA → 페이지 단위 매핑                                          │ │
│  │     • 유연성 높음                                                     │ │
│  │     • 매핑 테이블 큼                                                  │ │
│  │                                                               │ │
│  │  2. 블록 매핑 (Block Mapping):                                        │ │
│  │     • LBA → 블록 단위 매핑                                            │ │
│  │     • 매핑 테이블 작음                                                │ │
│  │     • 유연성 낮음                                                     │ │
│  │                                                               │ │
│  │  3. 하이브리드 매핑 (Hybrid Mapping):                                  │ │
│  │     • 블록 + 페이지 혼합                                               │ │
│  │     • 로그 블록 사용                                                  │ │
│  │     • 균형 잡힌 방식                                                  │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │  매핑 방식    테이블 크기   성능    복잡도              │        │ │
│  │  │  ────────    ─────────   ────    ──────              │        │ │
│  │  │  페이지        큼        높음     높음                 │        │ │
│  │  │  블록          작음      낮음     낮음                 │        │ │
│  │  │  하이브리드    중간       중간     중간                 │        │ │
│  │  │                                                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【매핑 테이블】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  논리 주소 (LBA)        물리 주소 (PPN)                              │ │
│  │  ─────────────        ─────────────                              │ │
│  │     LBA 0         →      Page 100                               │ │
│  │     LBA 1         →      Page 250                               │ │
│  │     LBA 2         →      Page 88                                │ │
│  │     LBA 3         →      Page 301                               │ │
│  │     ...                   ...                                    │ │
│  │                                                               │ │
│  │  → 쓰기마다 새 페이지 할당, 매핑 테이블 갱신                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FTL 핵심 기능】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 주소 변환: LBA → PPN                                              │ │
│  │  2. 가비지 컬렉션: 무효 페이지 정리                                     │ │
│  │  3. 마모 평준화: 쓰기 분산                                             │ │
│  │  4. 캐시: 쓰기 버퍼, 읽기 캐시                                         │ │
│  │  5. 에러 정정: ECC 관리                                               │ │
│  │  6. 배드 블록 관리: 불량 블록 처리                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  │  방식        장점           단점            사용처                   │ │

│  │  페이지      유연, 빠름      RAM 많이 필요   고성능 SSD               │ │
│  │  블록        RAM 적음       수정 시 복사    저가 SSD                 │ │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• FTL: 논리 주소 → 물리 주소 변환
• 위치: SSD 컨트롤러 내부
• 기능: 매핑, GC, 마모 평준화, 캐시
• 매핑: 페이지, 블록, 하이브리드
• 페이지 매핑: 유연하지만 RAM 많이 필요
• 블록 매핑: RAM 적지만 유연성 낮음
• 하이브리드: 균형 잡힌 방식
• 쓰기 증폭: FTL로 인한 추가 쓰기
• TRIM: FTL 효율화 핵심
• 관련: SSD, GC, 마모 평준화
```
