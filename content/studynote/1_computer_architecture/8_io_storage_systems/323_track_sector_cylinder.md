# 323. 트랙, 섹터, 실린더 (Track, Sector, Cylinder)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크 데이터 조직화의 기본 단위
> 2. **가치**: 데이터 주소 지정, 효율적 접근
> 3. **융합**: HDD 구조, CHS 주소, LBA

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

- **트랙(Track)**: 플래터 위의 동심원 경로, 데이터가 기록되는 원형 라인
- **섹터(Sector)**: 트랙을 나눈 최소 단위, 일반적으로 512바이트 또는 4KB
- **실린더(Cylinder)**: 여러 플래터에서 같은 반지름 트랙의 집합

### 💡 비유: 도서관 구조

디스크 구조는 **도서관**과 같다. 책장이 여러 층 있다(플래터). 각 층의 같은 줄이 실린더다. 한 줄의 선반이 트랙이다. 선반의 칸이 섹터다. "3층 5번 줄 2번 칸"처럼 주소를 지정한다. 도서관 사서는 층→줄→칸 순서로 책을 찾는다.

## Ⅱ. 상세 분석

### CHS 주소와 LBA

```
┌─────────────────────────────────────────────────────────────────────┐
│                CHS 주소와 LBA                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CHS 주소 지정】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  CHS (Cylinder-Head-Sector):                                         │ │
│  │                                                               │ │
│  │  • Cylinder: 실린더 번호 (0 ~ N-1)                                   │ │
│  │  • Head: 헤드 번호 = 표면 번호 (0 ~ H-1)                              │ │
│  │  • Sector: 섹터 번호 (1 ~ S)                                         │ │
│  │                                                               │ │
│  │  예: CHS = (100, 2, 5)                                               │ │
│  │  • 100번 실린더                                                      │ │
│  │  • 2번 헤드 (표면)                                                   │ │
│  │  • 5번 섹터                                                         │ │
│  │                                                               │ │
│  │  최대 주소 (IDE 제한):                                                │ │
│  │  • Cylinder: 1024                                                   │ │
│  │  • Head: 16                                                         │ │
│  │  • Sector: 63                                                       │ │
│  │  • 최대 = 1024 × 16 × 63 × 512B = 504MB (구형)                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LBA 주소 지정】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  LBA (Logical Block Addressing):                                     │ │
│  │                                                               │ │
│  │  • 섹터를 선형 번호로 지정                                             │ │
│  │  • 0부터 시작, 순차적 증가                                            │ │
│  │  • CHS 번역 불필요                                                   │ │
│  │                                                               │ │
│  │  LBA → CHS 변환:                                                     │ │
│  │  Cylinder = LBA / (Heads × Sectors)                                  │ │
│  │  Head = (LBA % (Heads × Sectors)) / Sectors                          │ │
│  │  Sector = (LBA % Sectors) + 1                                        │ │
│  │                                                               │ │
│  │  예: LBA = 10000, Heads = 16, Sectors = 63                           │ │
│  │  Cylinder = 10000 / (16 × 63) = 9                                    │ │
│  │  Head = (10000 % 1008) / 63 = 9                                      │ │
│  │  Sector = (10000 % 63) + 1 = 55                                      │ │
│  │                                                               │ │
│  │  CHS = (9, 9, 55)                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【디스크 용량 계산】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  총 용량 = Cylinders × Heads × Sectors × Sector_Size                  │ │
│  │                                                               │ │
│  │  예:                                                                 │ │
│  │  • Cylinders: 16383                                                  │ │
│  │  • Heads: 16                                                         │ │
│  │  • Sectors: 63                                                       │ │
│  │  • Sector Size: 512 bytes                                            │ │
│  │                                                               │ │
│  │  총 용량 = 16383 × 16 × 63 × 512                                     │ │
│  │         = 8,455,200,768 bytes                                        │ │
│  │         ≈ 8.4 GB (CHS 제한)                                          │ │
│  │                                                               │ │
│  │  현대 디스크: LBA만 사용, 48-bit LBA로 수 TB 지원                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 트랙: 플래터 위 동심원 데이터 경로
• 섹터: 최소 주소 지정 단위 (512B/4KB)
• 실린더: 같은 반지름 트랙 집합
• CHS: 물리적 주소 (Cylinder-Head-Sector)
• LBA: 논리적 주소 (선형 번호)
• 용량: C × H × S × 섹터크기
• 장점: 구조적 데이터 조직화
• 현대: LBA 주로 사용
• 관련: HDD, 파일 시스템, 파티션
```
