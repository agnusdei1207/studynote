# 320. 버스트 모드 (Burst Mode)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: DMA가 버스를 독점하여 연속 데이터 전송
> 2. **가치**: 최고 전송 속도, 대량 데이터에 적합
> 3. **융합**: DMA, 버스, 사이클 스틸링

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

버스트 모드(Burst Mode)는 **DMA 컨트롤러가 버스를 독점하여 여러 데이터를 연속적으로 전송하는 방식**이다. CPU는 전송이 완료될 때까지 대기하며, DMA가 전체 데이터를 한 번에 전송한다.

### 💡 비유: 전세 계약

버스트 모드는 **전세 계약**과 같다. 집을 1년 동안 빌린다. 집주인(CPU)은 1년간 집에 못 들어온다. 세입자(DMA)가 1년간 독점 사용한다. 월세처럼 매달매달 들락날락하는 게 아니다. 한 번에 오래 빌린다. 전세금은 비싸지만 확실하게 쓴다.

## Ⅱ. 버스트 모드 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스트 모드 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 과정】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. DMA 요청 (CPU):                                                  │ │
│  │     • 소스, 목적지, 크기 설정                                        │ │
│  │     • DMA 시작 명령                                                 │ │
│  │                                                               │ │
│  │  2. 버스 요청 (BR):                                                  │ │
│  │     • DMA가 버스 요청                                               │ │
│  │     • CPU가 버스 해제                                               │ │
│  │                                                               │ │
│  │  3. 버스 승인 (BG):                                                  │ │
│  │     • DMA가 버스 마스터 됨                                           │ │
│  │     • CPU는 대기 상태                                               │ │
│  │                                                               │ │
│  │  4. 연속 전송:                                                        │ │
│  │     • 모든 데이터를 연속으로 전송                                     │ │
│  │     • 버스 요청/해제 없음                                            │ │
│  │     • 최대 전송 속도                                                 │ │
│  │                                                               │ │
│  │  5. 버스 해제:                                                        │ │
│  │     • 전송 완료 후 버스 반환                                          │ │
│  │     • CPU 실행 재개                                                 │ │
│  │                                                               │ │
│  │  6. 인터럽트:                                                         │ │
│  │     • 완료 인터럽트 발생                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스트 전송 예시】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1KB 전송:                                                            │ │
│  │  • 데이터 크기: 1024바이트                                            │ │
│  │  • 버스 폭: 64비트 (8바이트)                                          │ │
│  │  • 전송 횟수: 128회                                                   │ │
│  │  • 버스 클록: 100MHz                                                 │ │
│  │  • 전송 시간: 128 × 10ns = 1.28us                                    │ │
│  │                                                               │ │
│  │  vs 사이클 스틸링:                                                    │ │
│  │  • 버스 요청/해제: 128회 × 오버헤드                                   │ │
│  │  • 전송 시간: 1.28us + 128 × 오버헤드                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적합한 사용 사례】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 대량 데이터 전송: 디스크 블록, 네트워크 패킷                         │ │
│  │  • 높은 대역폭 필요: 그래픽, 비디오                                    │ │
│  │  • 배치 처리: 실시간 응답 불필요                                      │ │
│  │  • 블록 전송: 섹터 단위 디스크 I/O                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 버스트 모드: DMA가 버스 독점하여 연속 전송
• 특징: 전송 속도 최고, CPU 대기
• 장점: 빠른 전송, 오버헤드 없음
• 단점: CPU 대기, 응답성 저하
• vs 사이클 스틸링: 빠르지만 CPU 영향 큼
• 적합: 대량 데이터, 높은 대역폭
• 사용처: 디스크, 네트워크, 그래픽
• 버스: DMA 독점 사용
• 관련: DMA, 사이클 스틸링, 버스
```
