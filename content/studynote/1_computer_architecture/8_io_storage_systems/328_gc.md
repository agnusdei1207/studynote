# 328. 가비지 컬렉션 (Garbage Collection in SSD)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 무효 페이지가 있는 블록을 정리하여 공간 확보
> 2. **가치**: 쓰기 공간 확보, 성능 유지, 마모 관리
> 3. **융합**: FTL, 마모 평준화, TRIM

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SSD의 가비지 컬렉션(Garbage Collection, GC)은 **무효 페이지가 많은 블록을 찾아 유효 페이지를 다른 곳으로 이동시키고, 원래 블록을 삭제(Erase)하여 재사용 가능하게 만드는 과정**이다.

### 💡 비유: 도서관 정리

가비지 컬렉션은 **도서관 정리**와 같다. 책장이 꽉 찼다. 하지만 많은 책이 오래된 책이다. 오래된 책을 없애고 싶다. 한 칸에 오래된 책과 새 책이 섞여 있다. 오래된 책을 버리고, 새 책만 다른 칸으로 옮긴다. 그러면 한 칸이 비워진다. 새 책을 꽂을 수 있다.

```
│  │  • 페이지 단위 쓰기 (4KB ~ 16KB)                                       │ │

│  │  • 덮어쓰기 불가 (지운 후 쓰기만 가능)                                   │ │
│  │                                                               │ │
│  │  문제 발생:                                                            │ │
│  │  • 파일 삭제 → 논리적으로만 삭제                                        │ │
│  │  • 플래시에는 데이터 남음 (무효 페이지)                                  │ │
│  │  • 쓰기 공간 부족                                                      │ │

│  │                                                               │ │

│  │                                                               │ │
│  │  단계 1: 정리 대상 블록 선택                                            │ │
│  │  ┌─────────────────────────────────────┐                       │ │
│  │  │ 블록 A                               │                       │ │
│  │  │ ┌───┬───┬───┬───┬───┬───┬───┬───┐ │                       │ │
│  │  │ │ V │ I │ V │ I │ I │ V │ I │ V │ │                       │ │
│  │  │ │ 1 │ - │ 2 │ - │ - │ 3 │ - │ 4 │ │                       │ │
│  │  │ └───┴───┴───┴───┴───┴───┴───┴───┘ │                       │ │
│  │  │ V=유효, I=무효                       │                       │ │
│  │  └─────────────────────────────────────┘                       │ │
│  │                                                               │ │
│  │  단계 2: 유효 페이지 복사                                               │ │
│  │  ┌──────────────────────┐   ┌──────────────────────┐           │ │
│  │  │ 블록 A               │   │ 블록 B (새 블록)       │           │ │
```

## Ⅱ. 가비지 컬렉션 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                가비지 컬렉션 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【GC 종류】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 배경 GC (Background GC):                                          │ │
│  │     • 유휴 시간에 실행                                                 │ │
│  │     • 사용자 I/O에 영향 최소화                                         │ │
│  │     • 미리 여유 공간 확보                                              │ │
│  │                                                               │ │
│  │  2. 강제 GC (Forced GC):                                              │ │
│  │     • 여유 공간 부족 시 실행                                           │ │
│  │     • 사용자 I/O 지연 발생                                             │ │
│  │     • 성능 저하 가능                                                   │ │
│  │                                                               │ │
│  │  3. 아이들 GC (Idle GC):                                              │ │
│  │     • 시스템 유휴 시 자동 실행                                          │ │
│  │     • 미리 공간 확보                                                   │ │
│  │     • 성능 향상                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【GC와 TRIM】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  TRIM:                                                                │ │
│  │  • OS가 SSD에 삭제 정보 알림                                           │ │
│  │  • 어떤 LBA가 더 이상 필요 없는지 알려줌                                │ │
│  │  • GC가 더 효율적으로 작동                                             │ │
│  │                                                               │ │
│  │  TRIM 없이:                                                           │ │
│  │  • GC가 유효한지 무효한지 모름                                          │ │
│  │  • 모든 데이터를 유효로 간주                                            │ │
│  │  • GC 비효율                                                          │ │
│  │                                                               │ │
│  │  TRIM 있이:                                                           │ │
│  │  • 무효 페이지 정확히 식별                                              │ │
│  │  • GC 효율 향상                                                       │ │
│  │  • 쓰기 증폭 감소                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【쓰기 증폭 (Write Amplification)】                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정의:                                                                │ │
│  │  • 실제 쓰기량 > 사용자 요청 쓰기량                                     │ │
│  │  • WA = 실제 플래시 쓰기량 / 사용자 쓰기량                              │ │
│  │                                                               │ │
│  │  원인:                                                                │ │
│  │  • GC: 유효 페이지 복사                                                │ │
│  │  • FTL: 주소 매핑                                                     │ │
│  │  • 마모 평준화: 블록 이동                                              │ │
│  │                                                               │ │
│  │  예:                                                                  │ │
│  │  사용자 4KB 쓰기 → GC로 인해 실제 20KB 쓰기                            │ │
│  │  WA = 20KB / 4KB = 5                                                 │ │
│  │                                                               │ │
│  │  영향:                                                                │ │
│  │  • 높은 WA = 수명 단축                                                │ │
│  │  • 높은 WA = 성능 저하                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  $ cat /sys/block/sda/queue/discard_granularity                     │

│                                                                     │
│  【TRIM 실행】                                                                  │)
│  ──────────────────                                                  │
│  # 수동 TRIM                                                         │
│  $ sudo fstrim -v /                                                 │
│  /: 100 GB trimmed                                                  │
│                                                                     │
│  # 모든 파일시스템 TRIM                                               │
│  $ sudo fstrim -av                                                  │
│                                                                     │
│  【GC 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. TRIM 활성화: GC 효율 향상                                        │
│  2. 여유 공간: 10-20% 비워두기                                        │
│  3. 대용량 SSD: GC 부하 분산                                         │
│  4. 연속 쓰기: 랜덤 쓰기보다 GC 영향 적음                              │
│  5. 정기적 TRIM: GC 효율 유지                                        │
│  6. Over-provisioning: 메이커 도구로 설정                             │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 가비지 컬렉션: 무효 페이지 정리
• 필요: 플래시는 덮어쓰기 불가
• 과정: 블록 선택 → 복사 → 삭제
• 종류: 배경, 강제, 아이들 GC
• TRIM: GC 효율화 핵심
• 쓰기 증폭: WA = 실제쓰기/요청쓰기
• WA: 낮을수록 좋음 (이상적 1.0)
• 영향: 성능, 수명에 직접 영향
• 최적화: TRIM, 여유 공간
• 관련: FTL, 마모 평준화, SSD
```
