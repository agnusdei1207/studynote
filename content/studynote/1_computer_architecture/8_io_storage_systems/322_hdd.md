# 322. 하드 디스크 드라이브 (HDD)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 회전하는 자기 디스크에 데이터를 저장하는 장치
> 2. **가치**: 대용량, 저렴한 비용, 비휘발성
> 3. **융합**: 트랙, 섹터, 실린더, 탐색 시간

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

하드 디스크 드라이브(HDD, Hard Disk Drive)는 **회전하는 자기 디스크(플래터)에 데이터를 자기 방식으로 기록하고 읽는 비휘발성 저장장치**이다. 헤드가 회전하는 플래터 위를 이동하며 데이터를 읽고 쓴다.

### 💡 비유: 오래된 LP 레코드

HDD는 **오래된 LP 레코드**와 같다. 검은 원반(플래터)이 돈다. 바늘(헤드)이 원반 위를 움직인다. 바늘이 홈을 따라가면 소리(데이터)가 나온다. 원반이 돌아야 소리가 나온다. 바늘이 이동하는 데 시간이 걸린다. LP처럼 회전하는 원반에 데이터가 있다.

### HDD 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                HDD 개요                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【HDD 구조】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                     ┌─────────────────┐                       │ │
│  │                     │   액추에이터     │                       │ │
│  │                     │    암(ARM)      │                       │ │
│  │                     └────────┬────────┘                       │ │
│  │                              │                                 │ │
│  │                     ┌───────┴───────┐                         │ │
│  │                     │   헤드(HEAD)   │ ← 읽기/쓰기               │ │
│  │                     └───────┬───────┘                         │ │
│  │                             ↓                                  │ │
│  │              ┌────────────────────────────┐                   │ │
│  │              │    플래터(PLATTER)          │ ← 회전 디스크      │ │
│  │              │  ┌──────────────────┐     │                   │ │
│  │              │  │    트랙(Track)    │     │                   │ │
│  │              │  │  ┌────────────┐  │     │                   │ │
│  │              │  │  │  섹터      │  │     │                   │ │
│  │              │  │  │ (Sector)  │  │     │                   │ │
│  │              │  │  └────────────┘  │     │                   │ │
│  │              │  └──────────────────┘     │                   │ │
│  │              └────────────┬───────────────┘                   │ │
│  │                           │                                    │ │
│  │                    ┌──────┴──────┐                             │ │
│  │                    │   스핀들     │ ← 회전 축                   │ │
│  │                    │  (SPINDLE)  │                             │ │
│  │                    └─────────────┘                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【HDD 용어】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  플래터(Platter):                                                     │ │
│  │  • 데이터가 기록되는 원반                                             │ │
│  │  • 자성 물질로 코팅                                                  │ │
│  │  • 양면 사용 가능                                                   │ │
│  │                                                               │ │
│  │  트랙(Track):                                                          │ │
│  │  • 플래터上的 동심원                                                  │ │
│  │  • 데이터가 기록되는 원형 경로                                        │ │
│  │                                                               │ │
│  │  섹터(Sector):                                                         │ │
│  │  • 트랙의 최소 단위                                                   │ │
│  │  • 일반적으로 512바이트 또는 4KB                                      │ │
│  │                                                               │ │
│  │  실린더(Cylinder):                                                     │ │
│  │  • 여러 플래터에서 같은 반지름 트랙의 집합                              │ │
│  │  • 헤드 이동 없이 접근 가능                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【HDD 성능 지표】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  회전 속도(RPM):   5400, 7200, 10000, 15000 RPM                      │ │
│  │  용량:             500GB ~ 20TB                                       │ │
│  │  전송 속도:        100 ~ 300 MB/s                                     │ │
│  │  평균 탐색 시간:   3 ~ 15 ms                                          │ │
│  │  평균 대기 시간:   회전속도에 따라 다름 (7200 RPM = 4.17ms)            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. HDD 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                HDD 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【액세스 시간】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  총 액세스 시간 = 탐색 시간 + 회전 지연 + 전송 시간                      │ │
│  │                                                               │ │
│  │  1. 탐색 시간(Seek Time):                                              │ │
│  │     • 헤드가 목표 트랙으로 이동하는 시간                                │ │
│  │     • 평균 탐색 시간: 3~15 ms                                         │ │
│  │     • 최대 탐색 시간: 전체 트랙 이동                                    │ │
│  │                                                               │ │
│  │  2. 회전 지연(Rotational Latency):                                     │ │
│  │     • 목표 섹터가 헤드 아래 올 때까지 대기                              │ │
│  │     • 평균 회전 지연 = 0.5 × 회전 주기                                  │ │
│  │     • 7200 RPM: 4.17ms 평균                                           │ │
│  │                                                               │ │
│  │  3. 전송 시간(Transfer Time):                                          │ │
│  │     • 데이터를 읽거나 쓰는 시간                                        │ │
│  │     • 매우 빠름 (마이크로초 단위)                                      │ │
│  │                                                               │ │
│  │  예: 7200 RPM, 평균 탐색 8ms                                          │ │
│  │  총 액세스 시간 = 8ms + 4.17ms + 전송 ≈ 12~15ms                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【디스크 스케줄링 알고리즘】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. FCFS (First Come First Served):                                   │ │
│  │     • 요청 순서대로 처리                                              │ │
│  │     • 단순하지만 비효율적                                             │ │
│  │                                                               │ │
│  │  2. SSTF (Shortest Seek Time First):                                 │ │
│  │     • 현재 위치에서 가장 가까운 요청 처리                               │ │
│  │     • 기아 현상 가능                                                 │ │
│  │                                                               │ │
│  │  3. SCAN (엘리베이터):                                                 │ │
│  │     • 한 방향으로 끝까지 이동, 반대 방향으로 처리                       │ │
│  │     • 균등한 대기 시간                                                │ │
│  │                                                               │ │
│  │  4. C-SCAN (Circular SCAN):                                          │ │
│  │     • 한 방향으로만 처리, 끝에서 처음으로 점프                          │ │
│  │     • 더 균등한 대기 시간                                             │ │
│  │                                                               │ │
│  │  5. LOOK / C-LOOK:                                                    │ │
│  │     • SCAN/C-SCAN 변형, 끝까지 안 가고 마지막 요청에서 방향 전환         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【HDD vs SSD】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  항목              HDD              SSD                        │ │
│  │  ────              ────              ────                        │ │
│  │  속도              느림 (100MB/s)     빠름 (500MB/s+)            │ │
│  │  액세스 시간        느림 (10ms)        빠름 (0.1ms)               │ │
│  │  용량              큼 (20TB+)         작음 (4TB)                 │ │
│  │  가격              저렴              비쌈                        │ │
│  │  내구성            기계적 고장         제한된 쓰기 수명            │ │
│  │  소음              있음              없음                        │ │
│  │  전력              높음              낮음                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. HDD × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                HDD × 타 과목 융합                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • I/O: 디스크 컨트롤러 인터페이스                                     │
│  • DMA: 디스크 데이터 전송                                            │
│  • 인터럽트: I/O 완료 알림                                            │
│  • 버스: SATA, SAS 인터페이스                                         │
│                                                                     │
│  【운영체제】                                                          │
│  • 파일 시스템: ext4, NTFS, FAT32                                    │
│  • 스케줄링: 디스크 스케줄링 알고리즘                                   │
│  • 캐시: 페이지 캐시, 버퍼 캐시                                        │
│  • I/O: 비동기 I/O, 직접 I/O                                         │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 스토리지: 데이터 파일, 인덱스, 로그                                  │
│  • 버퍼 풀: 디스크 I/O 최소화                                          │
│  • RAID: 디스크 어레이                                                │
│  • 성능: IOPS, 처리량                                                 │
│                                                                     │
│  【성능 튜닝】                                                          │
│  • 파티션: 정렬, 최적화                                               │
│  • 조각 모음: 단편화 해결                                             │
│  • RAID: 성능, 신뢰성 향상                                            │
│  • 캐시: 디스크 캐시 활용                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  HDD RPM별 성능:                                                              │ │
│  │  ──────────────────────                                              │ │
│  │  RPM       회전 주기    평균 지연    용도                            │ │
│  │  ────      ────────    ────────    ────                            │ │
│  │  5400      11.1ms      5.56ms      저전력, 대용량                   │ │
│  │  7200      8.33ms      4.17ms      일반 용도                       │ │
│  │  10000     6.0ms       3.0ms       고성능                          │ │
│  │  15000     4.0ms       2.0ms       엔터프라이즈                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• HDD: 회전하는 자기 디스크에 데이터 저장
• 구조: 플래터, 트랙, 섹터, 실린더, 헤드
• 액세스 시간: 탐색 + 회전 지연 + 전송
• RPM: 5400, 7200, 10000, 15000
• 스케줄링: FCFS, SSTF, SCAN, C-SCAN
• 인터페이스: SATA, SAS
• 장점: 대용량, 저렴
• 단점: 느림, 기계적 고장
• vs SSD: 느리지만 저렴하고 대용량
• 관련: SSD, RAID, 파일 시스템
```
