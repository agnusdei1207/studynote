# 321. IOP (I/O Processor)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 전담 전용 프로세서
> 2. **가치**: CPU 부하 완전 분리, 최고 효율
> 3. **융합**: 채널, 스마트 NIC, 오프로딩

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

IOP(I/O Processor)는 **I/O 작업을 전담하는 전용 프로세서**이다. CPU의 I/O 부하를 완전히 분리하여 처리하며, IBM 대형 시스템에서는 채널(Channel)이라고도 부른다.

### 💡 비유: 비서실

IOP는 **비서실**과 같다. 사장님(CPU)은 중요한 결정만 한다. 모든 서류 작성, 전화, 일정 관리는 비서실(IOP)이 한다. 비서실은 독립적으로 일한다. 사장님은 "이 일 해줘"라고만 하면 된다. 비서실이 알아서 처리하고 결과만 보고한다. 사장님은 비서실 일에 신경 쓰지 않는다.

│  ──────────────────                                                  │

│  │  │  └─────┘    └──┬──┘    └──┬──┘    └──┬──┘              │  │ │

│  │                                                               │ │
│  │  채널 종류:                                                           │ │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• IOP: I/O 전담 전용 프로세서
• 기능: I/O 프로그램 실행, 장치 제어, 데이터 변환
• IBM: 채널(Channel)이라 부름
• 채널 프로그램: CCW, CAW, CSW
• vs DMA: IOP는 프로그래밍 가능
• CPU: I/O에서 완전히 분리
• 현대: SmartNIC, SSD 컨트롤러, RAID
• 장점: 최고 효율, CPU 부하 완전 분리
• 단점: 하드웨어 복잡, 비용
• 관련: DMA, 채널, 오프로딩
```
