# 333. RAID 1 (Mirroring)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동일 데이터를 여러 디스크에 복제
> 2. **가치**: 최고 신뢰성, 빠른 읽기
> 3. **융합**: 미러링, 읽기 분산, 50% 효율

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

RAID 1(Mirroring)은 **동일한 데이터를 두 개 이상의 디스크에 복제하여 저장함으로써 디스크 고장 시 데이터 보호를 보장하는 RAID 레벨**이다. 읽기는 병렬로 수행할 수 있지만 쓰기는 모든 디스크에 해야 한다.

### 💡 비유: 중요 서류 복사본

RAID 1은 **중요 서류 복사본**과 같다. 중요한 계약서다. 잃어버리면 큰일이다. 그래서 복사해서 두 군데 보관한다. 하나 잃어버려도 다른 하나가 있다. 읽을 때는 가까운 것을 읽는다. 빠르다. 수정할 때는 둘 다 수정한다. 느리다. 하지만 안전하다.

### RAID 1 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 1 개요                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【미러링 원리】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  데이터: [A1][A2][A3][A4]                                            │ │
│  │                                                               │ │
│  │         Disk 0                Disk 1                             │ │
│  │         ┌────┐                ┌────┐                             │ │
│  │         │ A1 │ ←─── 복제 ───→ │ A1 │                             │ │
│  │         │ A2 │                │ A2 │                             │ │
│  │         │ A3 │                │ A3 │                             │ │
│  │         │ A4 │                │ A4 │                             │ │
│  │         └────┘                └────┘                             │ │
│  │                                                               │ │
│  │  → Disk 0 고장 → Disk 1에서 계속 서비스                                │ │
│  │  → 데이터 손실 없음                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 특성】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  읽기 성능: 향상 (분산 가능)                                           │ │
│  │  쓰기 성능: 단일 디스크와 비슷 (모든 디스크에 쓰기)                         │ │
│  │  용량 효율: 50% (2디스크 기준)                                         │ │
│  │  내결함성: 1디스크 고장 허용                                            │ │
│  │                                                               │ │
│  │  읽기 분산 예:                                                          │ │
│  │  • 요청 1: Disk 0에서 읽기                                            │ │
│  │  • 요청 2: Disk 1에서 읽기 (병렬)                                      │ │
│  │  → 읽기 성능 2배 향상 가능                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【복구 과정】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Disk 0 고장 감지                                                  │ │
│  │  2. Disk 1에서 계속 서비스                                             │ │
│  │  3. 새 디스크 교체                                                     │ │
│  │  4. Disk 1 → 새 디스크로 데이터 복사                                   │ │
│  │  5. 미러링 복구 완료                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. RAID 1 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 1 상세                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【읽기 분산 전략】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  라운드 로빈:                                                          │ │
│  │  • 요청을 번갈아가며 분산                                               │ │
│  │  • 균등한 부하 분산                                                   │ │
│  │                                                               │ │
│  │  최소 탐색:                                                            │ │
│  │  • 헤드 위치가 가까운 디스크 선택                                       │ │
│  │  • 탐색 시간 최소화                                                   │ │
│  │                                                               │ │
│  │  부하 기반:                                                            │ │
│  │  • 현재 부하가 적은 디스크 선택                                         │ │
│  │  • 동적 부하 분산                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【쓰기 처리】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  동기 쓰기:                                                            │ │
│  │  • 모든 디스크에 쓰기 완료 후 반환                                       │ │
│  │  • 데이터 일관성 보장                                                  │ │
│  │  • 느린 디스크가 병목                                                  │ │
│  │                                                               │ │
│  │  비동기 쓰기:                                                          │ │
│  │  • 하나의 디스크에 쓰기 후 반환                                          │ │
│  │  • 빠른 응답                                                         │ │
│  │  • 정전 시 불일치 가능                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【용량 계산】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  총 용량 = 단일 디스크 용량 (2디스크 기준)                               │ │
│  │                                                               │ │
│  │  예: 1TB 디스크 2개                                                  │ │
│  │  총 용량 = 1TB (50% 손실)                                            │ │
│  │                                                               │ │
│  │  3디스크 미러링:                                                       │ │
│  │  총 용량 = 1TB (33% 활용)                                            │ │
│  │  → 2디스크 고장 허용                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. RAID 1 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 1 × 타 과목 융합                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 미러링: 하드웨어 레벨 복제                                          │
│  • 컨트롤러: 쓰기 분산 관리                                            │
│  • 캐시: 쓰기 버퍼링                                                  │
│  • 버스: 동시 쓰기                                                   │
│                                                                     │
│  【운영체제】                                                          │
│  • mdadm: RAID 1 관리                                                │
│  • 복구: 자동 리빌드                                                  │
│  • 읽기 분산: OS 레벨 지원                                             │
│  • 부팅: 미러링된 부팅 파티션                                          │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 로그 파일: RAID 1 권장                                             │
│  • 트랜잭션: 안전한 쓰기                                               │
│  • 읽기 성능: 분산으로 향상                                             │
│  • 쓰기 성능: 단일 디스크 수준                                          │
│                                                                     │
│  【신뢰성】                                                            │
│  • MTBF: 크게 향상                                                   │
│  • 복구: 디스크 교체로 즉시                                             │
│  • 데이터 손실: 거의 없음                                              │
│  • 가용성: 99.99% 이상                                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RAID 1 vs 단일 디스크 신뢰성:                                                   │ │
│  │  ──────────────────────                                              │ │
│  │  구성           MTBF           데이터 손실 확률                        │ │
│  │  ────           ────           ────────────                        │ │
│  │  단일 디스크     100,000시간    5%/년                               │ │
│  │  RAID 1 (2디스크) 200,000시간   0.025%/년                           │ │
│  │  RAID 1 (3디스크) 300,000시간   0.000125%/년                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### RAID 1 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 1 분석                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RAID 1 생성】                                                                │)
│  ──────────────────                                                  │
│  # 2디스크 RAID 1 생성                                               │
│  $ mdadm --create /dev/md0 --level=1 --raid-devices=2 \             │
│        /dev/sda1 /dev/sdb1                                          │
│                                                                     │
│  【RAID 1 상태 확인】                                                            │)
│  ──────────────────                                                  │
│  $ cat /proc/mdstat                                                 │
│  md0 : active raid1 sdb1[1] sda1[0]                                 │
│        1047552 blocks super 1.2 [2/2] [UU]                          │
│                                                                     │
│  $ mdadm --detail /dev/md0                                          │
│  Raid Level : raid1                                                 │
│  Array Size : 1047552 (1023.00 MiB)                                 │
│  Failed Disks: 0                                                    │
│                                                                     │
│  【디스크 고장 및 복구】                                                          │)
│  ──────────────────                                                  │
│  # 고장난 디스크 제거                                                 │
│  $ mdadm --manage /dev/md0 --fail /dev/sda1                         │
│  $ mdadm --manage /dev/md0 --remove /dev/sda1                       │
│                                                                     │
│  # 새 디스크 추가                                                    │
│  $ mdadm --manage /dev/md0 --add /dev/sdc1                          │
│  # 자동 리빌드 시작                                                   │
│                                                                     │
│  【RAID 1 적합 용도】                                                            │)
│  ──────────────────                                                  │
│  1. 운영체제: 부팅 드라이브                                            │
│  2. 데이터베이스 로그: 트랜잭션 안전                                    │
│  3. 중요 문서: 데이터 보호                                             │
│  4. 부팅 파티션: 시스템 가용성                                          │
│  5. 작은 용량: 비용 효율적                                             │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• RAID 1: 미러링으로 데이터 복제
• 읽기: 병렬로 분산 가능, 성능 향상
• 쓰기: 모든 디스크에 쓰기, 단일 수준
• 용량: 50% 활용 (2디스크)
• 신뢰성: 1디스크 고장 허용
• 복구: 디스크 교체로 즉시
• 적합: OS, 로그, 중요 데이터
• 비용: 높음 (용량 손실)
• 최소 디스크: 2개
• MTBF: 단일 대비 2배
```
