# 345. 주소/데이터/제어 버스 (Address/Data/Control Bus)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 버스의 세 가지 구성 요소
> 2. **가치**: 주소 지정, 데이터 전송, 동기화
> 3. **융합**: 메모리 맵, 프로토콜, 타이밍

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

주소 버스, 데이터 버스, 제어 버스는 **시스템 버스를 구성하는 세 가지 신호 그룹으로, 각각 메모리 주소 지정, 데이터 전송, 동작 제어를 담당**한다.

### 💡 비유: 우편 배달

세 버스는 **우편 배달**과 같다. 편지를 보낸다. 주소를 쓴다 (주소 버스). 편지 내용이 있다 (데이터 버스). "보내기", "받기" 표시가 있다 (제어 버스). 세 가지가 있어야 편지가 배달된다.

## Ⅱ. 버스 타입 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스 타입 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 버스 상세】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  주소 공간 계산:                                                        │ │
│  │  • n비트 주소 버스 → 2^n 개 주소                                        │ │
│  │                                                               │ │
│  │  16비트: 2^16 = 64K 주소                                               │ │
│  │  20비트: 2^20 = 1M 주소 (8086)                                         │ │
│  │  32비트: 2^32 = 4G 주소                                                │ │
│  │  64비트: 2^64 = 16E 주소                                               │ │
│  │                                                               │ │
│  │  메모리 맵:                                                              │ │
│  │  ┌────────────────────────────────┐                                 │ │
│  │  │ 0xFFFFFFFF ← 64비트 주소 공간     │                                 │ │
│  │  │ ─────────────────────────────   │                                 │ │
│  │  │         (예약됨)                 │                                 │ │
│  │  │ ─────────────────────────────   │                                 │ │
│  │  │         I/O 공간                 │                                 │ │
│  │  │ ─────────────────────────────   │                                 │ │
│  │  │         메모리                   │                                 │ │
│  │  │ ─────────────────────────────   │                                 │ │
│  │  │ 0x00000000                      │                                 │ │
│  │  └────────────────────────────────┘                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【제어 버스 신호】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  신호         방향       기능                                        │ │
│  │  ────         ────       ────                                        │ │
│  │  READ (RD)    CPU→MEM    읽기 요청                                    │ │
│  │  WRITE (WR)   CPU→MEM    쓰기 요청                                    │ │
│  │  CLK          CPU→ALL    클럭 신호                                    │ │
│  │  RESET        CPU→ALL    초기화                                       │ │
│  │  INT          DEV→CPU    인터럽트 요청                                 │ │
│  │  INTA         CPU→DEV    인터럽트 확인                                 │ │
│  │  BUSRQ        DEV→CPU    버스 요청                                    │ │
│  │  BUSAK        CPU→DEV    버스 승인                                    │ │
│  │  READY        MEM→CPU    데이터 준비됨                                 │ │
│  │  WAIT         MEM→CPU    대기 요청                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【데이터 버스 특성】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  폭에 따른 전송량:                                                       │ │
│  │  • 8비트: 1바이트/전송                                                  │ │
│  │  • 16비트: 2바이트/전송                                                 │ │
│  │  • 32비트: 4바이트/전송                                                 │ │
│  │  • 64비트: 8바이트/전송                                                 │ │
│  │  • 128비트: 16바이트/전송                                               │ │
│  │                                                               │ │
│  │  양방향 전송:                                                            │ │
│  │  • 읽기: 메모리 → CPU                                                  │ │
│  │  • 쓰기: CPU → 메모리                                                  │ │
│  │  • 트라이스테이트 버퍼로 방향 제어                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 주소 버스: 단방향, 주소 전송
• 데이터 버스: 양방향, 데이터 전송
• 제어 버스: 양방향, 제어 신호
• 주소 공간: 2^n (n=주소 버스 폭)
• 대역폭: 데이터 버스 폭 × 클럭
• 신호: RD, WR, CLK, INT 등
• 트라이스테이트: 양방향 구현
• 메모리 맵: 주소 공간 할당
• I/O 맵: 메모리 맵/포트 맵
• 설계: 폭, 속도, 전력 트레이드오프
```
