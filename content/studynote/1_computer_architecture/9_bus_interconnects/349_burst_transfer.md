# 349. 버스트 전송 (Burst Transfer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연속 데이터를 주소 한 번으로 고속 전송
> 2. **가치**: 오버헤드 감소, 대역폭 활용 극대화
> 3. **융합**: 캐시 라인, 메모리, DMA

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

버스트 전송(Burst Transfer)은 **시작 주소를 한 번만 보내고 연속된 여러 데이터를 연속적으로 전송하는 방식**으로, 주소 전송 오버헤드를 줄여 대역폭 효율을 높인다.

### 💡 비유: 박스 배달

버스트 전송은 **박스 배달**과 같다. 주소를 말한다. "101동". 그러면 1호, 2호, 3호, 4호에 연속으로 배달한다. 주소를 매번 안 말해도 된다. 한 번만 말하면 된다. 빠르다. 버스트도 똑같다. 주소 한 번, 데이터 여러 개.

### 버스트 전송 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스트 전송 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【단일 전송 vs 버스트 전송】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  단일 전송 (Single Transfer): 4개 데이터 전송                          │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │ 사이클 1: 주소=100 → 데이터=A                          │    │ │
│  │  │ 사이클 2: 주소=104 → 데이터=B                          │    │ │
│  │  │ 사이클 3: 주소=108 → 데이터=C                          │    │ │
│  │  │ 사이클 4: 주소=112 → 데이터=D                          │    │ │
│  │  │                                                       │    │ │
│  │  │ → 4회 주소 전송 + 4회 데이터 전송 = 8회 전송             │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  버스트 전송 (Burst Transfer): 4개 데이터 전송                         │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │ 사이클 1: 주소=100 → 데이터=A                          │    │ │
│  │  │ 사이클 2:            → 데이터=B                        │    │ │
│  │  │ 사이클 3:            → 데이터=C                        │    │ │
│  │  │ 사이클 4:            → 데이터=D                        │    │ │
│  │  │                                                       │    │ │
│  │  │ → 1회 주소 전송 + 4회 데이터 전송 = 5회 전송             │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  효율 개선: (8 - 5) / 8 = 37.5% 감소                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스트 전송 특징】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                │ │
│  │  • 주소 전송 오버헤드 감소                                             │ │
│  │  • 버스 대역폭 효율 증가                                               │ │
│  │  • 연속 데이터에 적합                                                  │ │
│  │  • 캐시 라인 전송에 최적                                               │ │
│  │                                                               │ │
│  │  단점:                                                                │ │
│  │  • 버스를 오래 점유                                                   │ │
│  │  • 다른 마스터 대기                                                   │ │
│  │  • 비연속 데이터에 비효율                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 버스트 전송 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                버스트 전송 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【버스트 길이】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  버스트 길이 = 한 번에 전송하는 데이터 수                               │ │
│  │                                                               │ │
│  │  일반적인 길이:                                                         │ │
│  │  • 캐시 라인: 64바이트 (4 × 16바이트 또는 8 × 8바이트)                   │ │
│  │  • DDR: 8-버스트 (8 × 데이터 폭)                                       │ │
│  │  • PCIe: 최대 1024 더블워드                                           │ │
│  │                                                               │ │
│  │  예: 64비트 버스, 4-버스트                                             │ │
│  │  → 8바이트 × 4 = 32바이트 전송                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스트 모드】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 고정 버스트 (Fixed Burst):                                         │ │
│  │     • 모든 전송이 같은 주소로                                           │ │
│  │     • FIFO, 레지스터 쓰기에 사용                                        │ │
│  │                                                               │ │
│  │  2. 증가 버스트 (Incrementing Burst):                                   │ │
│  │     • 주소가 매 전송마다 증가                                           │ │
│  │     • 메모리 읽기/쓰기에 사용                                           │ │
│  │                                                               │ │
│  │  3. 래핑 버스트 (Wrapping Burst):                                       │ │
│  │     • 캐시 라인 경계에서 래핑                                            │ │
│  │     • 캐시에 최적화                                                    │ │
│  │                                                               │ │
│  │  예: 4-버스트, 시작주소 0x04, 래핑 (라인 크기 16바이트)                    │ │
│  │  → 0x04, 0x08, 0x0C, 0x00 (래핑)                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스트 vs 파이프라인】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  버스트:                                                                │ │
│  │  • 한 마스터가 연속 전송                                               │ │
│  │  • 주소 오버헤드 감소                                                  │ │
│  │                                                               │ │
│  │  파이프라인:                                                             │ │
│  │  • 여러 트랜잭션이 겹침                                                │ │
│  │  • 버스 유휴 시간 감소                                                 │ │
│  │                                                               │ │
│  │  조합: 버스트 + 파이프라인 = 최고 성능                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
