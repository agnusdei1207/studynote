# 344. 시스템 버스 (System Bus)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컴퓨터 구성요소 간 데이터 전송 통로
> 2. **가치**: 모듈화, 표준화, 확장성
> 3. **융합**: 주소, 데이터, 제어 신호

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

시스템 버스(System Bus)는 **CPU, 메모리, I/O 장치 등 컴퓨터의 주요 구성요소를 연결하고 데이터, 주소, 제어 신호를 전송하는 공유 통로**다.

### 💡 비유: 도로망

시스템 버스는 **도로망**과 같다. 집에서 회사로 간다. 도로를 탄다. 많은 사람이 같은 도로를 쓴다. 신호등이 있다. 교통정리를 한다. 도로가 넓으면 많은 차가 지나간다. 도로가 좁으면 막힌다. 버스도 똑같다. 신호들이 지나간다.

## Ⅱ. 시스템 버스 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 버스 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【버스 대역폭】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  대역폭 = 버스 폭 × 버스 클럭                                           │ │
│  │                                                               │ │
│  │  예:                                                                 │ │
│  │  • 64비트 버스, 100MHz 클럭                                           │ │
│  │  • 대역폭 = 8바이트 × 100MHz = 800 MB/s                               │ │
│  │                                                               │ │
│  │  • 128비트 버스, 200MHz 클럭                                          │ │
│  │  • 대역폭 = 16바이트 × 200MHz = 3.2 GB/s                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스 특성】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 폭 (Width):                                                        │ │
│  │     • 한 번에 전송할 수 있는 비트 수                                    │ │
│  │     • 8, 16, 32, 64, 128비트                                          │ │
│  │                                                               │ │
│  │  2. 클럭 (Clock):                                                       │ │
│  │     • 동기식: 클럭 신호에 맞춰 전송                                      │ │
│  │     • 비동기식: 핸드셰이크로 전송                                        │ │
│  │                                                               │ │
│  │  3. 전송 방식:                                                           │ │
│  │     • 직렬: 1비트씩 순차 전송                                          │ │
│  │     • 병렬: 여러 비트 동시 전송                                         │ │
│  │                                                               │ │
│  │  4. 공유 방식:                                                           │ │
│  │     • 단일 마스터: CPU만 버스 제어                                      │ │
│  │     • 다중 마스터: 여러 장치가 버스 요청                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스 사이클】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  읽기 사이클:                                                            │ │
│  │  1. CPU가 주소를 주소 버스에 출력                                        │ │
│  │  2. CPU가 READ 신호 활성화                                             │ │
│  │  3. 메모리가 데이터를 데이터 버스에 출력                                 │ │
│  │  4. CPU가 데이터를 읽음                                                │ │
│  │                                                               │ │
│  │  쓰기 사이클:                                                            │ │
│  │  1. CPU가 주소를 주소 버스에 출력                                        │ │
│  │  2. CPU가 데이터를 데이터 버스에 출력                                    │ │
│  │  3. CPU가 WRITE 신호 활성화                                            │ │
│  │  4. 메모리가 데이터를 저장                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 시스템 버스 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 버스 × 타 과목 융합                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • CPU: 버스 마스터                                                 │
│  • 메모리: 버스 슬레이브                                             │
│  • I/O: 버스 슬레이브                                               │
│  • 대역폭: 성능 결정                                                 │
│                                                                     │
│  【운영체제】                                                          │
│  • DMA: 버스 제어 획득                                              │
│  • I/O: 버스 통해 장치 접근                                          │
│  • 인터럽트: 제어 버스 신호                                           │
│  • 메모리 맵: 주소 버스 활용                                          │
│                                                                     │
│  【성능】                                                              │
│  • 대역폭: MB/s, GB/s                                               │
│  • 지연: 사이클 수                                                   │
│  • 병목: 버스 대역폭 제한                                             │
│  • 효율: 버스 활용률                                                 │
│                                                                     │
│  【설계 트레이드오프】                                                    │
│  • 폭 ↑ → 대역폭 ↑, 비용 ↑                                           │
│  • 클럭 ↑ → 대역폭 ↑, 전력 ↑                                          │
│  • 병렬 → 빠름, 복잡                                                  │
│  • 직렬 → 단순, 고속화 용이                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  버스 종류별 특성:                                                              │ │
│  │  ──────────────────────                                              │ │
│  │  버스        폭      속도       용도                                  │ │
│  │  ────        ────    ────       ────                                  │ │
│  │  ISA         16비트   8 MHz     구형                                   │ │
│  │  PCI         32비트   33 MHz    확장카드                               │ │
│  │  PCIe Gen4   x16     16 GT/s   그래픽                                 │ │
│  │  DDR4        64비트   3.2 GT/s  메모리                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 시스템 버스 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 버스 분석                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【버스 정보 확인】                                                              │)
│  ──────────────────                                                  │
│  $ lspci -vv | grep -A 10 "System bus"                              │
│                                                                     │
│  $ dmidecode -t bus                                                 │
│  # SMBIOS 버스 정보                                                  │
│                                                                     │
│  【메모리 대역폭 측정】                                                            │)
│  ──────────────────                                                  │
│  $ bandwidth -s                                                     │
│  # 메모리 버스 대역폭 측정                                            │
│                                                                     │
│  【PCIe 버스 확인】                                                              │)
│  ──────────────────                                                  │
│  $ lspci -vv                                                        │
│  # PCIe 링크 속도, 폭 확인                                           │
│                                                                     │
│  【버스 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 대역폭: 충분한 버스 폭 확보                                        │
│  2. 클럭: 적절한 클럭 주파수                                          │
│  3. 버스트: 연속 전송 활용                                            │
│  4. 파이프라인: 버스 대기 시간 숨김                                    │
│  5. 멀티 버스: 병렬 버스 구조                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 시스템 버스: 구성요소 연결 통로
• 구성: 주소, 데이터, 제어 버스
• 대역폭: 폭 × 클럭
• 주소 버스: 단방향, 주소 전송
• 데이터 버스: 양방향, 데이터 전송
• 제어 버스: 양방향, 제어 신호
• 사이클: 읽기, 쓰기
• 특성: 폭, 클럭, 전송 방식
• 병목: 대역폭 제한
• 진화: 병렬 → 직렬 고속화
```
