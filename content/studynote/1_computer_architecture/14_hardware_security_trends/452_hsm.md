# 452. HSM (Hardware Security Module)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 암호화 키의 안전한 저장과 연산을 위한 전용 하드웨어
> 2. **가치**: 키 보호, 고성능 암호화, 규정 준수
> 3. **융합**: PKI, TLS, 결제, 클라우드

---

## Ⅰ. 개요

### 개념 정의

HSM(Hardware Security Module)은 **암호화 키의 생성, 저장, 관리, 사용을 위한 전용 하드웨어 장치로, 물리적 격리와 높은 보안 수준을 제공**한다.

### 💡 비유: 은행 금고실

HSM은 **은행 금고실**과 같다. 금이 있다. (암호화 키) 금고실에 있다. (HSM) 보안요원. (접근 제어) 감시카메라. (감사) 안전하다. (보호) HSM이 똑같다. 키 보호, 물리적 격리, 접근 제어.

## Ⅳ. 실무 적용

### HSM 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                HSM 분석                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【PKCS#11 사용 예시】                                                            │)
│  ──────────────────                                                  │
│  # 토큰 목록                                                               │
│  $ pkcs11-tool --list-tokens                                            │
│                                                                     │
│  # 키 생성                                                                │
│  $ pkcs11-tool --keygen --key-type RSA:2048                             │
│                                                                     │
│  # 서명                                                                  │
│  $ pkcs11-tool --sign --mechanism SHA256-RSA                            │
│                                                                     │
│  【OpenSSL with HSM】                                                            │)
│  ──────────────────                                                  │
│  # HSM 엔진 로드                                                           │
│  $ openssl engine -t -c pkcs11                                          │
│                                                                     │
│  # HSM 키로 서명                                                           │
│  $ openssl dgst -sha256 -sign "pkcs11:id=%01" file.txt                  │
│                                                                     │
│  【AWS CloudHSM】                                                                │)
│  ──────────────────                                                  │
│  # CloudHSM 클러스터 상태                                                   │
│  $ aws cloudhsmv2 describe-clusters                                     │
│                                                                     │
│  # 키 생성                                                                │
│  $ cloudmgmt key generate --type rsa --size 2048                        │
│                                                                     │
│  【HSM 팁】                                                                      │)
│  ──────────────────                                                  │
│  1. 백업: 키 백업 정기 수행                                                 │
│  2. HA: 고가용성 구성                                                      │
│  3. 감사: 모든 작업 로깅                                                   │
│  4. 접근 제어: RBAC 구성                                                   │
│  5. 모니터링: 성능 및 가용성                                                │
│  6. 재해 복구: DR 계획 수립                                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• HSM: Hardware Security Module
• FIPS 140-2: 보안 인증 표준
• PKCS#11: 암호화 표준 인터페이스
• KMIP: 키 관리 프로토콜
• PKI: 공개키 기반 구조
• TLS: 전송 계층 보안
• 장점: 최고 보안, 규정 준수, 고성능
• 단점: 높은 비용, 복잡성, 벤더 종속
• 활용: PKI, 결제, TLS, 클라우드
• 클라우드 HSM: AWS, Azure, GCP
```
