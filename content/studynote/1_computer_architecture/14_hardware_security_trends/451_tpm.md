# 451. TPM (Trusted Platform Module)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드웨어 기반 보안 암호화 칩
> 2. **가치**: 키 보호, 측정 부팅, 원격 증명
> 3. **융합**: Secure Boot, BitLocker, attestation

---

## Ⅰ. 개요

### 개념 정의

TPM(Trusted Platform Module)은 **암호화 키 생성, 저장, 관리를 위한 하드웨어 보안 칩으로, 시스템 무결성 검증과 데이터 보호를 제공**한다.

### 💡 비유: 금고

TPM은 **금고**와 같다. 중요한 게 있다. (암호화 키) 금고에 넣는다. (TPM) 비밀번호만 안다. (인증) 안전하다. (보호) 꺼낼 수 있다. (사용) TPM이 똑같다. 키 저장, 하드웨어 보호, 안전한 사용.

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• TPM: Trusted Platform Module
• TPM 1.2: SHA-1, RSA 2048
• TPM 2.0: SHA-256, 알고리즘 유연
• PCR: Platform Configuration Register
• 측정 부팅: 부트 과정 기록
• 원격 증명: 시스템 무결성 증명
• Sealing: 조건부 데이터 복호화
• BitLocker: TPM 기반 디스크 암호화
• fTPM: 펌웨어 기반 TPM
• 장점: 하드웨어 보안, 무결성 검증
• 단점: 오버헤드, 복잡성
```
