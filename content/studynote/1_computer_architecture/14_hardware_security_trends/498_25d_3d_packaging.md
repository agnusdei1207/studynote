# 498. 2.5D 및 3D 패키징 기술

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 개의 개별 반도체 다이(Die)를 평면(2D)으로 길게 나열하는 대신, 실리콘 인터포저 위에 나란히 올리거나(2.5D) 다이 자체를 위로 포개어 적층하는(3D) 첨단 어드밴스드 패키징(Advanced Packaging) 기술이다.
> 2. **가치**: 칩 간의 물리적 배선 길이를 극단적으로 단축하여 I/O 대역폭 병목을 해소하고, 신호 전송에 필요한 전력 소모를 줄이면서 전체 칩의 실장 면적(Footprint)을 혁신적으로 축소한다.
> 3. **융합**: TSMC의 CoWoS(2.5D), 인텔의 Foveros(3D) 등 파운드리의 핵심 경쟁력으로 부상했으며, 무어의 법칙의 한계를 돌파하는 '칩렛(Chiplet)'과 'HBM' 생태계를 떠받치는 물리적 토대다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 2.5D 및 3D 패키징은 이기종(Heterogeneous) 또는 동종의 여러 반도체 칩을 단일 패키지 내부에서 고밀도로 연결하는 차세대 후공정 기술이다. 2.5D는 수평 병렬 배치 + 인터포저 연결을, 3D는 수직 적층 + TSV(실리콘 관통 전극) 연결을 의미한다.
- **💡 비유**: 도시 계획에 비유할 수 있다. 2D 패키징이 넓은 땅에 단층 건물을 듬성듬성 짓고 긴 도로(기판 배선)로 연결하는 '교외 신도시'라면, 2.5D는 거대한 백화점(인터포저) 1층 평면에 여러 매장을 오밀조밀 붙여 놓은 것이고, 3D는 매장들을 아예 고층 타워(TSV)로 겹겹이 쌓아 올려 엘리베이터로 이동하게 만든 '초고밀도 주상복합'이다.
- **등장 배경 및 발전 과정**:
  1. **단일 칩(Monolithic) 크기의 한계**: 트랜지스터 미세화가 3nm, 2nm로 접어들면서, 하나의 거대한 칩을 만들 때 발생하는 천문학적인 비용과 곤두박질치는 수율(Yield) 문제가 발생했다. 
  2. **메모리 대역폭의 벽 (Memory Wall)**: 프로세서 연산 속도는 빨라지는데 외부 D램에서 데이터를 가져오는 속도가 느려 병목이 심화되었다. 물리적 거리를 좁혀 대역폭을 넓혀야 했다.
  3. **후공정(Packaging)의 가치 상승**: '칩을 잘 쪼개서 만들고(칩렛), 이를 얼마나 촘촘히 잘 이어 붙이느냐'가 반도체의 성능을 결정하는 패러다임 전환이 일어났다. 파운드리 업체들이 전공정(Fabrication) 못지않게 후공정 기술 개발에 사활을 걸게 된 배경이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 특징 및 적용 기술 | 비유 |
|:---|:---|:---|:---|
| **마이크로 범프 (Micro Bump)** | 다이(Die) 간, 또는 다이와 인터포저를 물리적/전기적으로 접합 | 기존 C4 범프(기판용)보다 훨씬 작음 (피치 30µm 이하) | 층을 붙이는 미세한 레고 요철 |
| **실리콘 인터포저 (Silicon Interposer)** | 2.5D 패키징에서 다이들을 올려놓는 특수 매개 기판 | 일반 PCB와 달리 실리콘 웨이퍼 공정으로 제작되어 배선이 극히 촘촘함 | 다이들을 연결하는 미세 광장 |
| **TSV (Through-Silicon Via)** | 다이 내부, 또는 인터포저를 수직으로 관통하는 미세 구멍/전극 | 상하 층을 최단 거리로 연결 (수직 엘리베이터 역할) | 건물 관통 초고속 엘리베이터 |
| **실리콘 브릿지 (Silicon Bridge)** | 2.5D 패키징에서 큰 인터포저 대신 칩과 칩 사이 연결부위만 이어주는 작은 조각 | 인텔 EMIB 기술 (비용 절감 효과 큼) | 두 건물 사이를 잇는 구름다리 |
| **하이브리드 본딩 (Hybrid Bonding)** | 차세대 3D 패키징 결합 기술 (범프를 아예 없앰) | 구리(Cu) 패드와 절연막을 분자 수준에서 직접 융합 (피치 1µm 급) | 풀 없이 분자 단위로 용접하기 |

### 패키징 차원(Dimension)별 아키텍처 진화

단일 패키지 내부에서 칩들이 어떻게 배치되고 연결되는지에 따라 2D, 2.5D, 3D로 구분된다.

```text
  ┌─────────────────────────────────────────────────────────────────┐
  │                 2D vs 2.5D vs 3D 패키징 단면 비교               │
  ├─────────────────────────────────────────────────────────────────┤
  │                                                                 │
  │ [1] 2D 패키징 (MCM, Multi-Chip Module)                             │
  │     - 칩들을 일반 기판(PCB) 위에 평면으로 나열. 배선이 길고 느림.   │
  │      ┌─────┐        ┌─────┐                                     │
  │      │ Die │        │ Die │                                     │
  │      └─┬─┬─┘        └─┬─┬─┘                                     │
  │   ═════│═│════════════│═│════════ [Organic Substrate (PCB)]   │
  │        └─┴──(긴 배선)──┴─┘                                     │
  │                                                                 │
  │ ─────────────────────────────────────────────────────────────── │
  │                                                                 │
  │ [2] 2.5D 패키징 (예: TSMC CoWoS)                                   │
  │     - 칩들을 실리콘 인터포저 위에 나열. 인터포저 내부에 초정밀 배선 존재.│
  │      ┌──────┐          ┌──────┐                                 │
  │      │ GPU  │          │ HBM  │  ← 코어들이 평면으로 인접 배치됨     │
  │      └─┬──┬─┘          └─┬──┬─┘                                 │
  │   ─────┼──┼──────────────┼──┼───── [Silicon Interposer]       │
  │        │  │ ◀초미세배선▶ │  │      (내부 TSV를 통해 하단으로 연결) │
  │        ▼  ▼              ▼  ▼                                    │
  │   ════════════════════════════════ [Package Substrate]        │
  │                                                                 │
  │ ─────────────────────────────────────────────────────────────── │
  │                                                                 │
  │ [3] 3D 패키징 (예: Intel Foveros, AMD 3D V-Cache)                  │
  │     - 칩 위에 다른 칩을 수직으로 바로 포개어 적층. 최단 거리 연결.    │
  │             ┌─────────┐                                          │
  │             │ SRAM 칩 │  ← (예: L3 Cache)                       │
  │             └─┬───┬───┘                                          │
  │           TSV │   │                                            │
  │             ┌─▼───▼───┐                                          │
  │             │ Logic 칩│  ← (예: CPU 코어)                        │
  │             └─┬───┬───┘                                          │
  │   ═════════════▼═══▼══════════════ [Package Substrate]        │
  └─────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 
* **2D**는 여러 칩을 하나의 초록색 기판(PCB) 위에 얹는 전통적인 방식이다. 기판의 배선은 굵고 간격이 넓어 칩 간 데이터 이동(Bandwidth)에 병목이 생긴다.
* **2.5D**는 이 병목을 해결하기 위해 기판과 칩 사이에 **실리콘 인터포저**라는 특수 층을 삽입한다. 인터포저는 반도체 웨이퍼 공정으로 만들어져 수만 가닥의 초미세 배선을 깔 수 있다. 이 덕분에 GPU 칩과 HBM 칩 사이를 1024비트의 광활한 도로로 연결할 수 있다. (수직 적층된 HBM 자체는 3D지만, GPU와 HBM 간의 배치는 평면이므로 전체 시스템은 2.5D로 부른다.)
* **3D**는 아예 인터포저도 빼고 칩(다이) 바로 위에 다른 칩을 올려버리는 방식이다. AMD가 CPU 코어 칩 위에 캐시 메모리(SRAM) 칩을 수직으로 얹어 용량을 3배로 늘린 '3D V-Cache'가 대표적이다. 상하 층은 TSV(관통 전극)로 직접 연결되어 데이터 지연 시간(Latency)이 거의 0에 가깝다.

### 심층 제조 공정: 하이브리드 본딩 (Hybrid Bonding) 혁신
기존의 2.5D/3D 패키징은 칩과 칩을 '마이크로 범프(미세한 납땜 구슬)'로 이어 붙였다. 하지만 범프 방식은 간격을 30µm 이하로 줄이기 어렵다. 
최신 3D 패키징은 범프를 아예 없앤 **하이브리드 본딩**을 사용한다.
① 두 칩의 표면을 CMP 공정으로 거울처럼 완벽하게 평탄화한다.
② 구리(Cu) 전극 패드와 주변 절연막(SiO2)이 정확히 일치하도록 정렬한다.
③ 상온에서 절연막끼리 먼저 붙이고, 열을 가해 구리가 팽창하면서 구리 패드끼리 원자 단위로 융합되게 만든다.
이 방식을 통해 핀 간격을 1µm 단위로 극단적으로 좁혀, 단위 면적당 I/O 밀도를 1,000배 이상 높일 수 있다.

## Ⅲ. 융합 비교 및 다각도 분석

### 주요 벤더별 Advanced Packaging 기술 분류

| 벤더 | 2.5D 패키징 기술 (인터포저/브릿지) | 3D 패키징 기술 (수직 적층) | 특징 |
|:---|:---|:---|:---|
| **TSMC** | **CoWoS** (Chip-on-Wafer-on-Substrate)<br>INFO (Fan-Out) | **SoIC** (System on Integrated Chips) | 전 세계 AI GPU(NVIDIA, AMD) 패키징 독점. 하이브리드 본딩 주도. |
| **Intel** | **EMIB** (Embedded Multi-die Interconnect Bridge) | **Foveros** (3D Face-to-Face 적층) | 비싼 전체 인터포저 대신 부분 실리콘 브릿지(EMIB)를 사용하여 가성비 확보. |
| **Samsung** | **I-Cube** (Interposer-Cube) | **X-Cube** (eXtended-Cube) | 메모리(HBM)와 파운드리를 모두 갖춘 유일한 턴키(Turn-key) 솔루션 제공. |

### 과목 융합 관점
- **하드웨어 아키텍처 (PPA 한계 돌파)**: 반도체 설계의 3대 요소인 PPA(Power, Performance, Area) 중 전력과 면적의 최적화가 전공정(미세화)에서 한계에 달했다. 2.5D/3D 패키징은 신호 전송 거리를 줄여 전력(Power)을 아끼고, 칩을 위로 쌓아 면적(Area)을 줄이며, HBM을 달아 대역폭(Performance)을 올리는 후공정 발 PPA 혁신이다.
- **방열 시스템 (Thermal Management)**: 3D 패키징의 가장 큰 부작용은 '열섬(Thermal Hotspot)' 현상이다. 로직 칩 위에 메모리 칩을 덮어버리면, 아래쪽 로직 칩에서 발생한 100도 이상의 열이 위로 빠져나가지 못하고 갇히게 된다. 이를 해결하기 위해 다이 사이에 미세 유체 통로(Microfluidic)를 파서 냉각수를 직접 흘려보내는 냉각 아키텍처 융합 연구가 활발히 진행 중이다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 데이터센터 AI 가속기 병목 해소 (NVIDIA H100 설계)**: 
   - **문제**: LLM 모델 파라미터 폭증으로 연산기(GPU)가 쉴 새 없이 메모리 데이터를 요구하지만, GDDR 메모리를 기판에 까는 방식(2D)으로는 대역폭의 한계(수백 GB/s)에 부딪힘.
   - **해결책 (2.5D CoWoS 적용)**: TSMC의 CoWoS 기술을 활용해 거대한 실리콘 인터포저 위에 1개의 괴물 같은 GPU 코어 다이와 6개의 HBM3 스택을 바짝 붙여 패키징했다. 이를 통해 3.3 TB/s라는 경이로운 대역폭을 달성하며 생성형 AI 붐을 견인했다.
2. **시나리오 — PC 게이밍 프로세서의 캐시 용량 한계 (AMD 3D V-Cache)**:
   - **문제**: 게임 성능을 높이려면 L3 캐시 용량이 커야 하지만, 2D 평면에서 캐시 용량을 늘리면 CPU 다이 크기가 너무 커져 수율이 폭락하고 원가가 상승한다.
   - **해결책 (3D Foveros/SoIC 적용)**: 기본 CPU 다이(CCD)를 그대로 양산하되, 캐시가 더 필요한 하이엔드 라인업(X3D 모델)에만 64MB짜리 얇은 SRAM 다이를 수직으로 얹어(3D) 하이브리드 본딩으로 결합시켰다. 다이 면적 증가 없이 캐시 용량을 3배로 늘려 게임 프레임을 비약적으로 상승시켰다.

### 도입 체크리스트
- **기술적**: 시스템 설계 시 2.5D 패키지(예: 대형 인터포저)의 크기가 레티클 한계(Reticle Limit, 노광 장비가 한 번에 찍어낼 수 있는 최대 면적, 약 800mm²)를 초과하지 않는가? 초과한다면 여러 인터포저를 이어 붙이는 CoWoS-L/R 기술 적용을 검토해야 한다.
- **운영·제조 관점**: 패키징 공정의 복잡도가 기하급수적으로 증가하므로, 적층되기 전 칩렛들이 완벽히 정상임을 보증하는 KGD(Known Good Die) 테스트 커버리지가 99.9% 이상 달성되었는가? 불량 칩 하나가 수천만 원짜리 전체 패키지를 불량으로 만든다(Yield Kill).

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 2D 평면 패키징 (기존) | 2.5D / 3D 패키징 (도입 후) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (I/O 대역폭)** | 수십 ~ 수백 GB/s 한계 | 수 TB/s 이상 (HBM 통합) | 데이터 병목 해소, AI 훈련 속도 **10배 향상** |
| **정량 (전력 효율)** | 기판의 긴 배선으로 인한 RC 지연/전력 낭비 | 배선 길이 1/100 이하 축소 (pJ/bit 감소) | 칩 간 데이터 통신 전력 소모 **절반 이하로 급감** |
| **정성 (폼팩터)** | 여러 칩 탑재 시 거대한 메인보드 면적 필요 | 수직 적층으로 손톱만 한 패키지에 수십 억 TR 집적 | 모바일/웨어러블 기기의 고성능화 및 경량화 동시 달성 |

### 미래 전망
- **인터포저의 다양화 (실리콘 → 유기/유리)**: 현재 2.5D 패키징의 주류인 실리콘 인터포저는 비싸고 크게 만들기 어렵다는 단점이 있다. 향후에는 비용이 저렴한 유기(Organic) 소재나, 고주파 신호 손실이 극도로 적고 평탄도가 우수한 **유리(Glass) 기판**이 차세대 패키징의 핵심 소재로 급부상할 것이다.
- **SiP(System in Package)의 궁극적 진화**: CPU, GPU, HBM을 넘어 전력 관리 칩(PMIC), 실리콘 포토닉스(광통신) 칩, 아날로그 센서까지 모든 이기종 반도체가 하나의 패키지 안으로 빨려 들어가는 완전한 SiP 시대가 3D 패키징 기술을 통해 완성될 것이다.

### 참고 표준
- **UCIe (Universal Chiplet Interconnect Express)**: 서로 다른 파운드리에서 2.5D/3D 패키징으로 결합될 칩렛 간의 전기적/논리적 통신을 정의하는 글로벌 오픈 스탠다드.

## 📌 관련 개념 맵 (Knowledge Graph)
- **[칩렛 아키텍처 (Chiplet)](./497_chiplet.md)**: 모놀리식 칩을 여러 개로 쪼갠 설계 사상. 이 쪼개진 칩렛들을 다시 하나로 묶어주는 물리적 그릇이 바로 2.5D/3D 패키징이다.
- **[HBM (High Bandwidth Memory)](./495_hbm.md)**: 3D TSV 적층 기술의 가장 대표적이고 성공적인 상용화 산물. 2.5D 인터포저 위에 GPU와 함께 올라간다.
- **[TSV (Through-Silicon Via)](./496_tsv.md)**: 3D 패키징을 가능하게 하는 가장 핵심적인 미세 구멍 전극 뚫기 공정 기술.
- **[무어의 법칙 (Moore's Law)](../3_architecture_basics_performance/146_moore_law.md)**: 트랜지스터 크기를 줄이는 'More Moore'에서 패키징 기술로 집적도를 높이는 'More than Moore'로 반도체 진화의 축이 넘어갔다.

## 👶 어린이를 위한 3줄 비유 설명
1. 보통 컴퓨터 부품들은 넓은 운동장에 따로따로 건물을 짓고, 그 사이를 긴 도로(2D 패키징)로 연결해서 짐을 배달했어요. 길어서 배달이 참 오래 걸렸죠.
2. **2.5D 패키징**은 부품들을 거대한 '스타필드 백화점(인터포저)' 1층에 옹기종기 모아둔 거예요. 가게들이 바로 옆에 붙어 있어서 물건을 엄청 빨리 주고받을 수 있죠!
3. **3D 패키징**은 여기서 한술 더 떠서, 아예 부품들을 건물 위층, 아래층으로 포개어 올리고(적층) 초고속 엘리베이터로 연결한 최고의 '초고층 아파트'랍니다.
