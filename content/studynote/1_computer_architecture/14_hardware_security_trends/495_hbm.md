# 495. HBM (High Bandwidth Memory)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기존 평면 방식의 램 모듈(DIMM) 대신, 실리콘 관통 전극(TSV)을 이용해 여러 개의 D램 다이를 수직으로 쌓아 올려 패키지(SiP) 내부에 프로세서와 함께 통합한 차세대 3D 메모리다.
> 2. **가치**: 칩과 메모리 간의 물리적 거리를 극단적으로 줄이고 데이터 통로(버스)를 1,024비트 이상으로 넓혀, 병목 현상(메모리 벽)을 해소하고 초당 테라바이트(TB/s) 급의 경이로운 대역폭을 제공한다.
> 3. **융합**: 생성형 AI와 거대 언어 모델(LLM) 훈련/추론을 위한 필수 인프라인 최신 GPU(NVIDIA H100 등)와 AI 가속기의 성능을 결정짓는 핵심 부품으로 자리매김했다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: HBM(High Bandwidth Memory)은 고성능 컴퓨팅 및 그래픽 카드를 위해 JEDEC에서 표준화한 고대역폭 적층형 3D 동기식 동적 랜덤 액세스 메모리(SDRAM) 인터페이스 기술이다.
- **💡 비유**: 일반 메모리(GDDR)가 차선은 32개지만 속도 제한이 없는 고속도로를 통해 물건을 한 줄로 나르는 방식이라면, HBM은 속도는 약간 느려도 차선이 무려 1,024개인 거대한 화물 전용 다리를 놓아 한 번에 엄청난 양의 물건을 쏟아붓는 방식이다. 심지어 이 다리는 프로세서 바로 옆에 붙어 있다.
- **등장 배경 및 발전 과정**:
  1. **GDDR의 물리적 한계**: 그래픽과 AI 연산이 복잡해지면서 더 많은 대역폭이 필요해졌다. 하지만 기존 GDDR 방식은 메모리 칩을 메인보드에 넓게 펼쳐(평면 배열) 핀(Pin)을 연결해야 하므로 공간 차지와 전력 소모(발열)가 너무 컸다.
  2. **수직 적층의 도입 (3D Packaging)**: AMD와 SK하이닉스가 주도하여, 메모리 칩을 아파트처럼 수직으로 쌓고 칩에 미세한 구멍(TSV)을 뚫어 엘리베이터처럼 데이터를 위아래로 이동시키는 혁신적인 패키징 기술을 개발했다 (2013년 1세대 HBM 발표).
  3. **AI 시대의 도래**: 챗GPT 같은 LLM은 파라미터(가중치) 크기가 수백 기가바이트에 달해 연산 속도보다 "데이터를 얼마나 빨리 가져오는가"가 성능을 좌우하게 되었고, HBM(특히 HBM3, HBM3E)이 AI 생태계의 절대적인 병목(Bottleneck) 해결사로 등극했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **코어 다이 (Core Die)** | 실제 데이터가 저장되는 D램 층 | 여러 층(4단, 8단, 12단)으로 수직 적층됨 | 3D Stacking | 아파트의 각 층 (거주 공간) |
| **베이스 다이 (Base Die / Logic Die)** | 코어 다이를 제어하고 외부와 통신하는 칩 | 적층된 맨 아래에 위치하며 테스트/제어 로직 포함 | PHY / 컨트롤러 | 아파트 1층의 관리사무소 |
| **TSV (Through-Silicon Via)** | 적층된 다이들을 수직으로 뚫어 연결하는 미세 전극 | 수천 개의 구멍에 구리(Cu)를 채워 상하 데이터 통로 형성 | 3D 패키징 | 아파트 층간 직통 초고속 엘리베이터 |
| **마이크로 범프 (Microbump)** | 다이와 다이, 또는 다이와 기판을 연결하는 미세한 솔더볼 | 전기적 신호 전달 및 물리적 결합 | 플립칩 결합 (Flip-Chip) | 블록을 연결하는 미세한 접착 핀 |
| **실리콘 인터포저 (Silicon Interposer)** | 프로세서(GPU/CPU)와 HBM을 연결하는 특수 기판 | 미세 공정으로 제작되어 수만 개의 회로 배선을 조밀하게 연결 | 2.5D 패키징 | 프로세서와 HBM을 잇는 거대한 광장 |

### HBM 2.5D 패키징 아키텍처 (TSV 및 Interposer)

HBM은 메모리 칩 자체가 메인보드(PCB)에 꽂히는 것이 아니라, GPU와 함께 2.5D 패키지(SiP) 내부에 통합된다. 

```text
  ┌─────────────────────────────────────────────────────────────┐
  │         HBM과 프로세서의 2.5D 패키징 (SiP) 단면 구조        │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │                  ┌──────────┐ ┌──────────┐  [HBM 칩]        │
  │                  │ Core Die │ │ Core Die │    ▲             │
  │   [Processor]    ├──────────┤ ├──────────┤    │ 4단 적층      │
  │    (GPU/CPU)     │ Core Die │ │ Core Die │    │             │
  │  ┌────────────┐  ├──────────┤ ├──────────┤    ▼             │
  │  │            │  │ Base Die │ │ Base Die │                  │
  │  │   Logic    │  └─┬──┬──┬──┘ └─┬──┬──┬──┘                  │
  │  │            │    │  │  │      │  │  │  ← Microbumps       │
  │  └─┬──┬──┬──┬─┘    ▼  ▼  ▼      ▼  ▼  ▼                     │
  │ ───┼──┼──┼──┼──────┴──┴──┴──────┴──┴──┴──────────────       │
  │    │  │  │  │ ◀── TSV 배선 ──▶ │  │  │    [Silicon          │
  │    ▼  ▼  ▼  ▼                 ▼  ▼  ▼      Interposer]      │
  │ ───────────────────────────────────────────────────         │
  │      │    │    │    │    │    │    │  ← C4 Bumps            │
  │      ▼    ▼    ▼    ▼    ▼    ▼    ▼                        │
  │ ═══════════════════════════════════════════════════       │
  │          [ Package Substrate (일반 칩 기판) ]                 │
  │ ═══════════════════════════════════════════════════       │
  │                   [ 메인보드 (PCB) ]                          │
  │                                                             │
  │ * TSV (Through-Silicon Via): HBM 칩 내부를 수직으로 관통하는 전극.│
  │ * 실리콘 인터포저: 일반 기판보다 배선을 훨씬 조밀하게(µm 단위) 깔 수│
  │                    있어, 1024비트의 초광대역 버스 연결을 가능케 함. │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 그림에서 볼 수 있듯, HBM은 일반 메모리처럼 프로세서와 멀리 떨어져 있지 않다. 실리콘 인터포저(Silicon Interposer)라는 특수한 중간 다리 위에 GPU(또는 AI 가속기)와 HBM 스택이 아주 가까이 나란히 배치된다(이를 2.5D 패키징이라 부름). HBM 내부는 4개~12개의 코어 다이가 수직으로 쌓여 있으며, 이 다이들은 가장자리로 배선을 빼는 것이 아니라 칩 한가운데를 수직으로 뚫은 미세한 구멍(TSV) 수천 개를 통해 데이터를 주고받는다. 이 구조 덕분에 데이터가 이동하는 물리적 거리가 밀리미터(mm) 단위로 짧아져 신호 지연(Latency)이 극도로 낮아지고, 한 번에 데이터를 보낼 수 있는 도로의 폭(Bus Width)을 1,024비트(GDDR은 보통 32비트)로 엄청나게 넓힐 수 있다.

---

### 심층 동작 원리 및 대역폭 비교

* **대역폭(Bandwidth) 공식**: `Bandwidth = Clock Rate × Data Rate(DDR) × Bus Width`
* **GDDR6의 접근**: 클럭 속도를 극단적으로 높이는 방식 (예: 16Gbps × 32-bit = 64 GB/s). 하지만 핀당 속도가 높아질수록 신호 무결성을 잡기 위해 전력 소모가 기하급수적으로 증가한다.
* **HBM의 접근**: 클럭 속도는 낮게 유지하되, 버스 폭(차선 수)을 무식하게 넓히는 방식 (예: 2.4Gbps × 1024-bit = 약 300 GB/s). 클럭이 낮으므로 전압을 낮출 수 있어(1.2V) 와트당 전력 효율(Performance per Watt)이 일반 GDDR 대비 3배 이상 우수하다.

## Ⅲ. 융합 비교 및 다각도 분석

### HBM vs GDDR 심층 비교

| 비교 항목 | GDDR6 (Graphics DDR) | HBM3 (High Bandwidth Memory 3) |
|:---|:---|:---|
| **버스 폭 (Bus Width)** | 칩당 32-bit | **스택당 1,024-bit** (압도적 광대역) |
| **패키징 형태** | PCB 표면 실장 (2D Planar) | **2.5D / 3D 패키징 (SiP, TSV)** |
| **설계 복잡도 / 비용** | 낮음 (대량 양산 용이) | **매우 높음** (수율 확보 어려움, 고가) |
| **전력 소모 (효율)** | 클럭이 높아 와트당 대역폭 낮음 | 클럭이 낮고 거리가 짧아 **와트당 효율 최상** |
| **주요 사용처** | 일반 PC 그래픽카드, 콘솔 게임기 | 하이엔드 AI 가속기, 슈퍼컴퓨터, HPC |

### 과목 융합 관점
- **인공지능 (LLM 추론/학습)**: AI 모델의 파라미터가 수천억 개(GPT-3: 175B)에 달하면서, GPU가 연산(MAC)을 하기 위해 메모리에서 가중치(Weight)를 가져오는 시간이 병목이 되는 "Memory Bound" 워크로드가 주류가 되었다. HBM은 칩당 1TB/s 이상의 대역폭을 제공하여 AI 모델의 훈련 시간과 추론 지연을 획기적으로 줄여주는 필수 기반 기술이다.
- **컴퓨터구조 (PIM, Processing In Memory)**: 최근 HBM 설계의 최전선은 단순히 데이터를 빠르게 나르는 것을 넘어, HBM의 베이스 다이(Base Die)나 메모리 뱅크 내부에 간단한 AI 연산 유닛(ALU)을 박아 넣는 HBM-PIM 아키텍처로 진화하고 있다. 이는 폰 노이만 병목(Von Neumann Bottleneck)을 근본적으로 파괴하려는 시도다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 초대규모 AI 클러스터 구축**: 기업이 온프레미스 LLM 훈련을 위해 서버를 발주할 때, 엔비디아의 L40S(GDDR 탑재)와 H100(HBM3 탑재) 사이에서 고민하는 상황.
   - **의사결정**: 훈련 대상 모델의 파라미터 수가 작고 배치(Batch) 위주의 추론이라면 가성비가 높은 L40S를 선택할 수 있으나, 파라미터 수가 수백억 개 이상이거나 분산 훈련(Distributed Training)을 해야 한다면 HBM의 압도적 대역폭이 필수이므로 수 배 비싸더라도 H100을 선택해야 전체 TCO(학습 시간 단축) 측면에서 유리하다.
2. **시나리오 — HBM 수율 관리와 패키징 불량**: HBM은 칩 제조사(SK하이닉스 등)가 만들지만, 최종적으로 GPU 칩과 하나로 합치는 2.5D 패키징(CoWoS 등)은 파운드리(TSMC)에서 담당한다.
   - **리스크**: 수만 개의 마이크로 범프 중 단 하나만 접착 불량이 나도 수천만 원짜리 전체 패키지를 폐기해야 한다. 따라서 HBM 칩 자체를 완벽하게 검사한 뒤 패키징 업체로 넘기는 **KGD (Known Good Die)** 검증 기술과 결함 허용(Fault Tolerance)을 위한 중복 TSV(Redundant TSV) 설계가 필수적이다.

### 도입 체크리스트
- **기술적**: HBM은 용량 확장이 불가능하다(모듈을 꽂는 방식이 아님). 패키징 될 때 용량(예: 80GB)이 영구적으로 픽스되므로, 도입하려는 서버의 HBM 총 용량이 AI 모델의 파라미터 + KV 캐시 크기를 감당할 수 있는지 사전(Sizing)에 완벽히 검토해야 한다.
- **운영·보안적**: 발열 밀도. HBM이 프로세서 바로 옆에 바짝 붙어 있어 특정 부위에 열이 집중(Hot Spot)된다. 공랭식 냉각이 한계에 달할 수 있으므로, 서버실의 풍량 설계나 다이렉트 수랭(DLC) 도입 여부를 재고해야 한다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 일반 서버 (DDR/GDDR 기반) | HBM 탑재 AI 서버 | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (대역폭)** | 최대 100 ~ 500 GB/s 수준 | 패키지당 3 TB/s ~ 8 TB/s 이상 (HBM3E) | 메모리 대역폭 **10배 이상 향상** |
| **정량 (전력 효율)** | 와트당 대역폭 낮음 | TSV 및 짧은 라우팅으로 전력 손실 최소화 | 전력당 데이터 처리량 3배 이상 개선 |
| **정성 (물리적 공간)** | 칩 주변에 VRAM 칩 10여 개 배치 (면적 차지) | 프로세서 바로 옆(수 mm)에 초고밀도 적층 | 기판 크기 축소 및 라우팅 복잡도 감소 |

### 미래 전망
- **적층 한계 돌파 (하이브리드 본딩)**: 현재의 마이크로 범프(솔더볼) 방식은 간격을 줄이는 데 한계가 있어, 향후 범프 없이 구리와 구리를 직접 맞붙이는 **하이브리드 본딩(Hybrid Bonding)** 기술을 도입하여 16단 이상의 초고층 HBM 스택을 구현할 것이다.
- **Custom HBM 시대**: 표준화된 HBM을 넘어, 특정 AI 칩렛(Chiplet) 아키텍처나 고객사의 요구에 맞춰 베이스 다이의 로직 기능을 커스텀 설계하는 맞춤형 HBM이 대두될 것이다.

### 참고 표준
- **JEDEC JESD238 (HBM3)**: 최신 HBM의 대역폭, 핀 레이아웃, TSV 규격 및 온도에 따른 리프레시 관리 등을 정의한 국제 반도체 표준 규격.
