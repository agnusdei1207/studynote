# 455. 보안 엔클레이브 (Secure Enclave / TEE)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 내 격리된 보안 실행 영역
> 2. **가치**: 기밀 연산, 키 보호, 신뢰 실행
> 3. **융합**: SGX, TrustZone, SEV, confidential

---

## Ⅰ. 개요

### 개념 정의

보안 엔클레이브(Secure Enclave) 또는 TEE(Trusted Execution Environment)는 **CPU 내에 격리된 보안 영역을 생성하여, 민감한 데이터와 코드를 보호하며 실행하는 기술**이다.

### 💡 비유: 은행 금고실 안의 금고

보안 엔클레이브는 **은행 금고실 안의 금고**와 같다. 은행이 있다. (CPU) 금고실이 있다. (TEE) 금고가 있다. (엔클레이브) 이중 보안. 완전 격리. 안전하다. 엔클레이브가 똑같다. CPU 내 격리, 메모리 암호화, 보안 실행.

## Ⅳ. 실무 적용

### 보안 엔클레이브 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                보안 엔클레이브 분석                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SGX 지원 확인】                                                               │)
│  ──────────────────                                                  │
│  # CPU 플래그 확인                                                         │
│  $ cat /proc/cpuinfo | grep -o 'sgx' | head -1                          │
│                                                                     │
│  # SGX 세부 정보                                                           │
│  $ cpuid -l 0x12                                                        │
│                                                                     │
│  【SGX SDK 사용】                                                                │)
│  ──────────────────                                                  │
│  # Enclave 생성                                                            │
│  $ sgx_edger8r enclave.edl                                              │
│                                                                     │
│  # Enclave 빌드                                                            │
│  $ make SGX_MODE=HW                                                     │
│                                                                     │
│  # Enclave 서명                                                           │
│  $ sgx_sign sign -enclave enclave.so -key private.pem                   │
│                                                                     │
│  【TrustZone 확인】                                                              │)
│  ──────────────────                                                  │
│  # Android TEE 확인                                                        │
│  $ adb shell getprop ro.build.tags | grep test                          │
│                                                                     │
│  # Keymaster 상태                                                          │
│  $ adb shell dumpsys android.security.keystore                          │
│                                                                     │
│  【TEE 팁】                                                                      │)
│  ──────────────────                                                  │
│  1. 사이즈: EPC 크기 고려                                                  │
│  2. 호출 최소화: ECALL/OCALL 오버헤드                                      │
│  3. 증명: 원격 증명 구현                                                   │
│  4. 사이드 채널: 방지 코드 작성                                             │
│  5. 키 관리: Sealing 활용                                                  │
│  6. 디버깅: SGX 디버그 모드 활용                                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 보안 엔클레이브: CPU 내 격리된 보안 실행 영역
• TEE: Trusted Execution Environment
• Intel SGX: Software Guard Extensions
• ARM TrustZone: Secure/Normal World
• AMD SEV: Secure Encrypted Virtualization
• EPC: Enclave Page Cache
• MEE: Memory Encryption Engine
• 장점: 강력한 격리, 하드웨어 보안
• 단점: 오버헤드, 메모리 제한
• 활용: 기밀 컴퓨팅, DRM, 결제
```
