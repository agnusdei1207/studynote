# 459. 하드웨어 인증 (Hardware Authentication)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드웨어 기반으로 사용자/장치 신원 확인
> 2. **가치**: 강력한 인증, 피싱 방지, 무결성 보장
> 3. **융합**: FIDO, 생체인식, TPM, YubiKey

---

## Ⅰ. 개요

### 개념 정의

하드웨어 인증(Hardware Authentication)은 **전용 하드웨어 장치를 사용하여 사용자나 시스템의 신원을 확인하는 강력한 인증 방식**이다.

### 💡 비유: 신분증 확인

하드웨어 인증은 **신분증 확인**과 같다. 사람이 있다. (사용자) 신분증이 있다. (하드웨어 토큰) 경찰이 확인. (인증 서버) 진짜다. (검증) 통과. (접근 허용) 안전하다. 하드웨어 인증이 똑같다. 전용 장치, 암호화 검증, 강력한 인증.

│  # 브라우저에서 WebAuthn 테스트                                             │

│  # SSH 접속                                                                │

│  4. 분실: 즉시 계정에서 제거                                                │
│  5. 펌웨어: 최신 펌웨어 유지                                                │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 하드웨어 인증: 전용 장치로 강력한 인증
• FIDO2: 비밀번호 없는 인증 표준
• WebAuthn: 브라우저 FIDO2 API
• U2F: 범용 2단계 인증
• YubiKey: 다기능 하드웨어 토큰
• Windows Hello: 생체 + TPM 인증
• 장점: 피싱 방지, 높은 보안
• 단점: 분실 위험, 비용
• 활용: MFA, SSO, ZTA
```
