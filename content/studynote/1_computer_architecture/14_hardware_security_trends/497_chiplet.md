> **핵심 인사이트**
> 1. **칩렛(Chiplet) 아키텍처**는 CPU, GPU, 캐시 등의 구성 요소를 하나의 거대한 실리콘 다이(Monolithic Die)에 찍어내는 대신, **기능별로 작은 조각(Chiplet)을 따로 만들어 레고 블록처럼 조립**하는 설계 패러다임이다.
> 2. 거대한 칩은 미세한 먼지 하나만 떨어져도 전체를 버려야 해서 수율(Yield)이 낮고 원가가 폭증하지만, 작게 쪼개어 만들면 **수율이 비약적으로 상승하여 제조 비용을 크게 절감**할 수 있다.
> 3. 또한 코어(Core)는 최신 3nm 공정으로, 입출력(I/O) 단자는 저렴한 6nm 공정으로 섞어서 조립하는 **이종 결합(Heterogeneous Integration)**이 가능하여 현대 반도체의 미세 공정 한계를 돌파하는 핵심 기술이 되었다.



## Ⅰ. 단일 칩(Monolithic) 아키텍처의 위기

과거 수십 년 동안 반도체는 모든 코어와 캐시, I/O 컨트롤러를 하나의 커다란 실리콘 도화지(Die) 위에 한 번에 그리는 **단일 칩(Monolithic) 방식**을 사용했습니다.

하지만 공정이 5nm, 3nm로 미세화되면서 치명적인 문제가 발생했습니다.
* 반도체를 만들 때는 실리콘 웨이퍼 위에 미세한 결함(Dust, Defect)이 필연적으로 존재합니다.
* 칩의 크기가 크면, 그 커다란 칩 어딘가에 결함이 포함될 확률이 높아집니다. 결함이 하나라도 포함되면 그 거대한 칩은 불량품으로 버려야 합니다. 
* 칩 크기가 2배 커지면, 불량률은 2배가 아니라 기하급수적으로 폭증하여 칩 가격이 수천만 원으로 치솟게 되었습니다 (수율의 한계).

> 📢 **섹션 요약 비유**: 피자 반죽에 파리가 한 마리 날아와 붙었습니다. 거대한 패밀리 사이즈 피자 1판을 굽고 있었다면 피자 한 판을 통째로 버려야 합니다. 하지만 아주 작은 미니 피자 100조각을 굽고 있었다면 파리가 앉은 딱 1조각만 버리고 99조각은 팔 수 있습니다. 이것이 칩렛의 원리입니다.



## Ⅱ. 칩렛의 구조와 동작 원리

AMD는 라이젠(Ryzen)과 에픽(EPYC) 프로세서에 이 칩렛 구조를 선도적으로 도입하여 인텔을 맹추격하는 데 성공했습니다.

### 구조 (AMD Zen 아키텍처 예시)
* **CCD (Core Complex Die)**: 오직 연산만 담당하는 순수 CPU 코어 덩어리입니다. 아주 작게 쪼개서 최신 비싼 공정(예: 5nm)으로 만듭니다.
* **IOD (I/O Die)**: 메모리 컨트롤러, PCIe 등 외부와 통신하는 덩어리입니다. 성능 향상이 크게 필요 없으므로 구형의 저렴한 공정(예: 12nm)으로 크게 만듭니다.
* **Infinity Fabric (초고속 인터커넥트)**: 이 조각(칩렛)들을 하나의 기판 위에서 마치 하나의 칩처럼 완벽하게 통신하게 묶어주는 초고속 버스 기술입니다.

### ASCII 다이어그램 비교

```text
  [ 기존 Monolithic 칩 ]             [ 칩렛 (Chiplet) 조립 칩 ]
 ┌──────────────────────┐         ┌──────────┐ ┌────────────┐
 │    코어 1   코어 2   │         │ 코어 1,2 │ │ 코어 3,4 │ (비싼 5nm 공정)
 │                      │         └─┬──────┬─┘ └─┬──────┬───┘
 │       L3 캐시        │           │      │     │          │
 │                      │        ┌──▼──────▼─────▼──────▼───┐
 │    코어 3   코어 4   │        │     I/O 컨트롤러 다이    │ (저렴한 12nm 공정)
 │                      │        │     (메모리, PCIe 등)    │
 │    I/O 컨트롤러      │        └──────────────────────────┘
 └──────────────────────┘         (모두 하나의 패키지 안에 조립됨)
```

> 📢 **섹션 요약 비유**: 장난감 로봇을 통짜 플라스틱으로 한 번에 찍어내지 않고(Monolithic), 머리와 팔다리를 따로따로 만들어서 조립(Chiplet)하는 것입니다. 팔 하나가 불량이면 팔만 버리고 새 팔을 끼우면 되며, 머리는 최신형으로, 다리는 구형 부품으로 섞어 쓰는 것도 가능합니다.



## Ⅲ. 칩렛 아키텍처의 장단점

**장점:**
1. **압도적인 수율과 원가 절감**: 작게 찍어내므로 불량률이 극히 낮아집니다.
2. **이종 공정 결합**: 코어는 TSMC 3nm, I/O는 GlobalFoundries 12nm 등 각 부품의 특성에 가장 가성비 좋은 공정을 골라 쓸 수 있습니다.
3. **유연한 확장성**: 서버용 64코어 CPU를 만들려면 8코어짜리 칩렛 블록을 8개 가져다 기판에 붙이기만 하면 됩니다. 설계가 매우 쉽고 빠릅니다.

**단점 (과제):**
1. **통신 지연 (Latency)**: 칩 조각들이 물리적으로 떨어져 있으므로, 칩 안에서 통신할 때보다 칩렛끼리 통신할 때 미세한 지연 시간과 추가 전력이 발생합니다.
2. **첨단 패키징 비용**: 조각들을 아주 촘촘하고 빠르게 연결하기 위해 실리콘 인터포저나 TSV 같은 비싼 후공정(Packaging) 기술이 필요합니다.

> 📢 **섹션 요약 비유**: 레고 블록으로 집을 지으면 조립식이라 싸고 빠르지만, 블록과 블록이 이어지는 이음새(인터커넥트)가 헐거우면 물이 샐 수 있습니다. 이 이음새를 통짜 건물처럼 완벽하게 매끄럽게 붙이는 고급 본드질(첨단 패키징)이 칩렛의 생명입니다.
