# 497. 칩렛 (Chiplet) 아키텍처

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기존에 거대한 하나의 실리콘 조각(Monolithic)으로 만들던 반도체를 코어, I/O, 메모리 컨트롤러 등 기능별로 작은 조각(Chiplet)으로 쪼개어 각각 제조한 뒤 하나의 패키지 안에 레고 블록처럼 이어 붙이는 차세대 설계 기법이다.
> 2. **가치**: 미세 공정(3nm 이하)의 기하급수적인 비용 증가와 수율(Yield) 저하 문제를 해결하는 유일한 경제적 대안이며, 고객이 원하는 코어 수와 기능을 자유롭게 조합할 수 있는 확장성을 제공한다.
> 3. **융합**: AMD의 Zen 아키텍처 성공을 기점으로 표준이 되었으며, 각 칩렛 간 초고속 데이터 통신을 위해 2.5D/3D 고급 패키징 기술(TSV, 인터포저) 및 UCIe 같은 개방형 인터커넥트 표준과 강력히 결합되어 있다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 칩렛(Chiplet) 아키텍처는 집적 회로를 설계할 때 거대한 단일 칩(Monolithic Die)을 만드는 대신, 여러 개의 작은 모듈형 칩(Chiplets)을 개별적으로 생산한 후 고급 패키징 기술을 통해 단일 SoC(System on Chip)처럼 동작하도록 결합하는 방법론이다.
- **💡 비유**: 과거에는 거대한 '일체형 콘크리트 빌딩'을 통째로 찍어내려다 보니 중간에 작은 금이라도 가면 빌딩 전체를 부숴야 했다. 칩렛은 '레고 블록'이다. 거실 블록, 침실 블록, 화장실 블록을 따로따로(심지어 화장실은 싼 재료로) 만든 다음 조립판(인터포저) 위에서 찰칵 붙여 거대한 건물을 완성하는 방식이다.
- **등장 배경 및 발전 과정**:
  1. **무어의 법칙 둔화와 수율(Yield) 절벽**: 칩의 크기(Reticle Limit, 약 800mm²)가 극에 달하면서 칩 하나에 미세한 먼지 하나만 떨어져도 수천 달러짜리 칩 전체를 버려야 했다. 칩 크기가 클수록 수율은 기하급수적으로 떨어진다.
  2. **공정 비용의 비대칭성**: CPU 코어는 3nm 최신 공정이 필요하지만, I/O 컨트롤러나 아날로그 회로는 14nm 레거시 공정으로도 충분하다. 모놀리식 구조에서는 이 둘을 합치기 위해 굳이 비싼 3nm 웨이퍼 면적을 아날로그 회로에 낭비해야 했다.
  3. **AMD의 대반격**: AMD가 Zen 2 (EPYC Rome) 아키텍처부터 코어 블록(CCD)과 I/O 블록(cIOD)을 물리적으로 분리하는 칩렛 방식을 과감히 도입, 인텔의 모놀리식 제국을 원가 경쟁력과 코어 수 확장성으로 무너뜨리며 칩렛 시대의 서막을 열었다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 / 특징 | 비유 |
|:---|:---|:---|:---|
| **컴퓨팅 다이 (Compute Die)** | 실제 핵심 연산(CPU/GPU/AI)을 수행하는 칩렛 | 최선단 미세 공정(예: TSMC 3nm) 적용 | 두뇌가 좋은 비싼 전문가 |
| **I/O 다이 (I/O Die)** | 메모리 컨트롤러, PCIe, USB 등 외부 인터페이스 담당 | 성숙한 레거시 공정(예: TSMC 12nm) 적용 | 수족이 튼튼한 택배 요원 |
| **인터커넥트 (Interconnect)** | 칩렛 간 데이터를 주고받는 초고속 통신 규격 | Infinity Fabric(AMD), EMIB(Intel) 등 자체 규격 사용 | 블록 간의 대화 언어 (통역기) |
| **인터포저 (Interposer)** | 칩렛들을 물리적으로 올려놓고 전기적으로 연결하는 기판 | 실리콘 기반으로 수만 개의 미세 배선(Microbump) 제공 | 레고 블록을 꽂는 넓은 밑판 |
| **어드밴스드 패키징** | 쪼개진 칩렛을 하나의 칩처럼 포장하는 기술 | 2.5D(평면 배치) 또는 3D(수직 적층, Foveros 등) 방식 활용 | 여러 블록을 하나의 상자에 포장 |

### 모놀리식(Monolithic) vs 칩렛(Chiplet) 아키텍처 비교

칩렛의 핵심은 '큰 칩 1개'를 '작은 칩 4개'로 나누었을 때 기하급수적으로 올라가는 수율(경제성)과 이기종 공정 혼용에 있다.

```text
  ┌─────────────────────────────────────────────────────────────┐
  │           모놀리식(Monolithic) vs 칩렛(Chiplet) 설계 비교            │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ [1] 모놀리식 (Monolithic) - "비싸고 위험한 일체형"              │
  │                                                             │
  │      결함(X) 발생 시 칩 전체 폐기!                            │
  │    ┌───────────────────────────────────┐                    │
  │    │  ┌────┐ ┌────┐ ┌────┐ ┌────┐      │                    │
  │    │  │Core│ │Core│ │Core│ │Core│  X   │                    │
  │    │  └────┘ └────┘ └────┘ └────┘      │                    │
  │    │  ===========================      │  전체 3nm 공정       │
  │    │  [    I/O & Memory Ctrl    ]      │  수율 40% (적자)     │
  │    └───────────────────────────────────┘                    │
  │                                                             │
  │ ─────────────────────────────────────────────────────────── │
  │                                                             │
  │ [2] 칩렛 (Chiplet) - "저렴하고 유연한 레고 블록"                │
  │                                                             │
  │     결함(X) 발생 시 작은 코어 다이 1개만 버리면 됨!             │
  │                                                             │
  │      ┌────┐    ┌────┐    ┌────┐   ┌────┐(폐기)            │
  │      │CCD1│    │CCD2│    │CCD3│   │CCD4│ X    ← 3nm (수율 90%)│
  │      └─┬──┘    └─┬──┘    └─┬──┘   └─┬──┘                  │
  │   = = =│= = = = = │ = = = = │ = = = = │ = = = = = = =       │
  │  ║   ┌─▼─────────▼─────────▼─────────▼─┐   ║            │
  │  ║   │   I/O Die (Memory, PCIe 등)   │   ║  ← 12nm (수율 99%)│
  │  ║   └───────────────────────────────┘   ║            │
  │   = = = = = =  Package Substrate  = = = = = =               │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 상단의 모놀리식 칩은 코어 4개와 I/O 컨트롤러가 거대한 하나의 실리콘(예: 3nm)에 그려져 있다. 제조 중 웨이퍼의 먼지 하나(X표)가 코어 4에 떨어지면, 멀쩡한 코어 1~3과 I/O 블록까지 통째로 쓰레기통에 버려야 한다. (거대 칩의 수율 절벽). 하단의 칩렛 아키텍처는 코어(CCD)를 작게 쪼개어 만든다. 칩이 작으므로 웨이퍼 한 장에서 훨씬 많은 칩을 찍어낼 수 있고 수율도 90% 이상으로 극대화된다. 결함(X)이 발생해도 그 작은 칩 하나만 버리면 된다. 더 중요한 것은, I/O 다이 같은 통신/제어 회로는 최신 3nm 공정을 쓰나 구형 12nm 공정을 쓰나 성능 차이가 크지 않다는 점이다. 칩렛 설계는 비싼 3nm는 연산 코어에만 쓰고, I/O 다이는 싸고 튼튼한 12nm 공정으로 만들어 이들을 인터포저(기판) 위에서 연결함으로써 전체 제조 원가를 극적으로 낮춘다.

### 심층 동작 원리 및 통신 오버헤드 (D2D Interconnect)
칩렛 방식의 가장 큰 과제는 쪼개진 칩들이 원래 하나의 칩이었던 것처럼(모놀리식 환상) 빠르게 데이터를 주고받게 만드는 **다이 대 다이(Die-to-Die, D2D) 인터커넥트** 기술이다.
① **신호 직렬화/역직렬화 (SerDes)**: 칩렛 A가 칩렛 B로 데이터를 보낼 때, 내부의 병렬 버스 데이터를 고속 직렬 신호로 변환하여 전송하고 B에서 다시 병렬로 푼다.
② **추가 지연(Latency) 페널티**: 칩 밖으로 데이터가 한 번 나갔다 들어와야 하므로, 모놀리식 대비 약 10~20 나노초(ns)의 추가 지연이 발생한다.
③ **발열 및 신호 무결성 보상**: D2D 통신을 위해 PHY 계층에서 추가 전력이 소모된다. 이를 줄이기 위해 인텔은 EMIB(Embedded Multi-die Interconnect Bridge)라는 작은 실리콘 조각을 기판 내부에 심어 칩렛 간 최단 거리 브릿지를 놓는 방식을 사용한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 인텔 vs AMD 칩렛 패키징 전략 비교

| 비교 항목 | AMD (MCM / Infinity Fabric) | Intel (Tile / EMIB / Foveros) |
|:---|:---|:---|
| **아키텍처 철학** | 경제성 중심의 모듈형 분할 (MCM) | 초고밀도 타일(Tile) 적층 및 브릿지 중심 |
| **인터커넥트 매질** | 유기 기판(Organic Substrate) 위 배선 | 실리콘 브릿지(EMIB) 또는 3D 적층(Foveros) |
| **장점** | 제조 원가가 압도적으로 저렴, 양산성 최고 | 칩 간 거리 최소화로 모놀리식에 가까운 속도 |
| **단점** | 칩 간 물리적 거리가 멀어 레이턴시(지연) 발생 | 패키징 공정 자체가 복잡하고 매우 비쌈 |
| **적용 제품** | Ryzen, EPYC 라인업 | Meteor Lake, Ponte Vecchio GPU |

### 과목 융합 관점
- **네트워크 (개방형 프로토콜 UCIe)**: 과거에는 AMD와 인텔이 각자의 독자적인 언어(프로토콜)로 자사 칩렛끼리만 통신하게 만들었다. 그러나 현재는 PCIe, CXL 표준을 기반으로 한 **UCIe (Universal Chiplet Interconnect Express)**라는 글로벌 표준이 제정되었다. 이로 인해 인텔 공정으로 만든 CPU 칩렛과 TSMC 공정으로 만든 AI 가속기 칩렛을 한 기판 위에서 조립하는 궁극의 '오픈 생태계'가 열리고 있다.
- **하드웨어 보안 (Hardware Trojan)**: 칩렛 시대의 가장 큰 보안 위협은 공급망(Supply Chain) 공격이다. 서드파티(Third-party)에서 사 온 IP 칩렛 내부에 하드웨어 트로이목마(Hardware Trojan)가 심어져 전체 패키지를 장악할 위험이 있으므로, 조립 시 각 칩렛의 서명을 검증하는 RoT(Root of Trust) 기반의 상호 인증 프로토콜이 연구되고 있다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 클라우드 벤더의 커스텀 AI 칩(ASIC) 개발**: AWS가 자사만의 데이터센터 워크로드에 최적화된 새로운 AI 추론 칩을 만들고자 한다.
   - **아키텍처 전략**: 칩 전체를 처음부터 끝까지 다 설계(모놀리식)하려면 수천억 원의 비용과 3년의 시간이 걸린다. 대신, 이미 검증된 ARM의 CPU 칩렛과 TSMC의 범용 I/O 칩렛을 구매하고, AWS가 독자 설계한 AI 가속 연산 칩렛만 별도로 제조한 뒤 이들을 UCIe 표준 기반으로 조합(SiP)한다. 개발 기간을 1년으로 단축하고 투자 위험을 최소화할 수 있다.
2. **시나리오 — 메모리 병목 현상과 3D V-Cache (성능 한계 극복)**: 칩렛 구조상 코어(CCD)에서 I/O 다이를 거쳐 메인 메모리(DDR)까지 가는 지연 시간이 게임 성능을 깎아 먹는 상황(AMD Zen 2의 약점).
   - **해결책**: 코어 칩렛(CCD) 바로 '위'에 64MB의 거대한 SRAM 덩어리(L3 캐시 칩렛)를 수직으로 포개어 올리는 3D 적층(TSV 활용, 3D V-Cache)을 적용한다. 연산 코어가 먼 I/O 다이로 나갈 필요 없이 머리 위에서 즉시 데이터를 꺼내 쓰게 만들어 게임 프레임을 비약적으로 상승시켰다.

### 도입 체크리스트
- **기술적**: 여러 칩렛이 좁은 패키지 내부에 모여 있으므로 특정 칩렛에 발열이 집중되는 **열섬 현상(Thermal Hotspot)**을 해소하기 위한 IHS 밀착 및 패키징 방열 설계가 완벽한가?
- **운영/공급망**: 칩렛 A, B, C가 각기 다른 파운드리(TSMC, Samsung, Intel)에서 생산될 경우, 수율 편차와 납기 불일치로 인해 최종 패키징 공정(OSAT) 라인이 멈추는 병목이 발생하지 않도록 SCM(공급망 관리)이 고도화되어 있는가?

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 모놀리식 설계 (기존) | 칩렛 아키텍처 (현대) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (수율/비용)** | 칩 크기 증가 시 수율 30% 이하 폭락 | 작은 다이 분할로 수율 90% 달성 | 64코어 이상 대형 서버 칩 **제조 원가 절반 이하 절감** |
| **정량 (확장성)** | 코어 8개 추가를 위해 새로운 칩 마스크(Mask) 설계 필수 | 기존 코어 칩렛 1개 더 이어 붙이면 끝 | 신제품 타임투마켓(TTM) 극적 단축 |
| **정성 (설계 유연성)** | 단일 공정의 물리적 한계에 묶임 | CPU, NPU, RAM을 마음대로 섞어 조립 | 고객 맞춤형 도메인 특화 아키텍처(DSA) 대중화 |

### 미래 전망
- **UCIe 표준 기반의 반도체 레고 시대**: 향후 5년 내에 UCIe 규격이 완전히 정착되면, 반도체 회사들은 칩을 파는 것이 아니라 '칩렛 단위의 IP'를 마켓플레이스에서 판매하게 될 것이다. 고객은 웹사이트에서 "TSMC 3nm ARM 코어 4개 + 삼성 4nm GPU 2개 + SK하이닉스 HBM3 2개"를 장바구니에 담아 조립을 주문하는 시대가 열릴 것이다.
- **Glass Substrate (유리 기판)의 부상**: 칩렛 개수가 늘어나고 칩 간 통신 속도가 100Gbps를 넘어가면 기존 플라스틱/실리콘 기판이 휘어지거나 신호가 깨진다. 인텔을 필두로 표면이 평탄하고 전기적 손실이 극도로 적은 '유리 기판(Glass Core Substrate)'이 차세대 칩렛 조립판으로 2020년대 후반 상용화를 앞두고 있다.

### 참고 표준
- **UCIe (Universal Chiplet Interconnect Express)**: 인텔, AMD, ARM, TSMC, 삼성 등 글로벌 반도체 공룡들이 모여 2022년에 출범시킨 오픈 칩렛 상호 연결 표준. 칩렛 계의 PCIe라 불린다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **[무어의 법칙 (Moore's Law)](../3_architecture_basics_performance/146_moore_law.md)**: 칩렛 아키텍처는 트랜지스터를 더 작게 만드는 '무어의 법칙'이 경제성 벽에 부딪혔을 때, 이를 우회하여 트랜지스터 수를 늘리기 위해 등장한 구원투수다.
- **[TSV (Through-Silicon Via)](./496_tsv.md)**: 칩렛들을 수직(3D)으로 쌓아 올릴 때 데이터를 위아래로 전달하는 관통 전극 미세 공정 기술.
- **[HBM (High Bandwidth Memory)](./495_hbm.md)**: 칩렛과 가장 찰떡궁합을 이루는 3D 적층 메모리로, 인터포저 위에서 GPU 칩렛과 짝을 이룬다.
- **[이기종 컴퓨팅 (Heterogeneous Computing)](../12_accelerators_ai_hardware/439_heterogeneous_computing.md)**: CPU, GPU, NPU 등 서로 다른 성격의 코어를 칩렛 형태로 나누어 조립함으로써 이기종 컴퓨팅을 하드웨어 레벨에서 가장 쉽고 강력하게 구현한다.

## 👶 어린이를 위한 3줄 비유 설명
1. 옛날에는 거대한 로봇 장난감을 통째로 플라스틱 틀에 부어서 한 번에 찍어냈어요. 중간에 팔 하나만 불량이 나도 로봇 전체를 버려야 해서 너무 비쌌죠 (모놀리식).
2. 그래서 똑똑한 아저씨들이 머리, 몸통, 팔, 다리를 따로따로 작게 만든 다음 조인트(인터커넥트)로 **레고 블록처럼 찰칵! 하고 조립**하는 방법을 발명했어요. 이게 바로 **칩렛**이에요!
3. 이렇게 하면 불량 난 팔 하나만 버리면 되고, 심지어 친구의 튼튼한 다리 부품을 빌려와서 내 로봇에 끼워 맞출 수도 있어서 엄청 싸고 멋진 로봇을 만들 수 있답니다.
