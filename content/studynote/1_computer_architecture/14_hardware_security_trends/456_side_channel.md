# 456. 사이드 채널 공격 (Side-Channel Attack)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리적 특성을 이용한 간접 정보 추출
> 2. **가치**: 암호화 키 탈취, 보안 우회
> 3. **융합**: 타이밍, 전력, 캐시, 스펙터

---

## Ⅰ. 개요

### 개념 정의

사이드 채널 공격(Side-Channel Attack)은 **시스템의 물리적 특성(시간, 전력, 전자파 등)을 관찰하여 비밀 정보를 추출하는 공격 기법**이다.

### 💡 비유: 금고 따는 소리 듣기

사이드 채널 공격은 **금고 따는 소리 듣기**와 같다. 금고가 있다. (암호화) 비밀번호 모른다. (직접 공격 불가) 소리를 듣는다. (사이드 채널) "틱틱틱" (타이밍) "딸깍" (전력) 조합한다. (분석) 연다. (공격 성공) 사이드 채널이 똑같다. 간접 관찰, 물리적 특성, 정보 추출.

## Ⅱ. 사이드 채널 공격 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                사이드 채널 공격 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Spectre 공격 원리】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 분기 예측 학습:                                                        │ │
│  │     if (x < array_size)  // 보통 참                                         │ │
│  │         y = array2[array1[x] * 256];                                       │ │
│  │                                                               │ │
│  │  2. 악의적 입력:                                                           │ │
│  │     x = 비밀_메모리_주소 - array1_base                                     │ │
│  │                                                               │ │
│  │  3. 추측 실행:                                                             │ │
│  │     CPU가 권한 검사 전에 실행                                               │ │
│  │     array1[x] = 비밀_데이터                                                │ │
│  │     array2[비밀_데이터 * 256] 접근                                         │ │
│  │                                                               │ │
│  │  4. 캐시 분석:                                                             │ │
│  │     Flush+Reload로 array2 캐시 확인                                        │ │
│  │     캐시 히트 = 비밀 데이터 추출                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Flush+Reload 공격】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Flush: clflush로 타겟 캐시 라인 무효화                                 │ │
│  │  2. Victim: 피해자가 타겟 메모리 접근                                       │ │
│  │  3. Reload: time stamp로 접근 시간 측정                                    │ │
│  │     - 빠름 (캐시 히트): 피해자가 접근함                                     │ │
│  │     - 느림 (캐시 미스): 피해자가 접근 안 함                                 │ │
│  │                                                               │ │
│  │  코드 예시:                                                                │ │
│  │  mfence();                                                                │ │
│  │  clflush(&array[i]);                                                       │ │
│  │  mfence();                                                                │ │
│  │  // victim 실행                                                            │ │
│  │  time = rdtsc();                                                          │ │
│  │  access(array[i]);                                                         │ │
│  │  time = rdtsc() - time;  // 캐시 히트/미스 판별                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【방어 기법】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  기법              설명                    적용                   │ │
│  │  ────              ────                    ────                   │ │
│  │  KPTI/KAISER       커널 페이지 격리         Meltdown             │ │
│  │  Retpoline         반환 주소 삽입           Spectre V2           │ │
│  │  IBRS/IBPB         간접 분기 제한           Spectre V2           │ │
│  │  LFENCE            직렬화 펜스              Spectre V1           │ │
│  │  Constant-time     상수 시간 구현           Timing Attack        │ │
│  │  캐시 분할         캐시 색상                Cache Attack         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 사이드 채널 공격: 물리적 특성 이용 정보 유출
• 타이밍 공격: 실행 시간 분석
• 전력 분석: DPA, SPA
• 캐시 공격: Flush+Reload, Prime+Probe
• Spectre: 분기 예측 악용
• Meltdown: 권한 검사 우회
• Rowhammer: DRAM 셀 간섭
• 방어: KPTI, Retpoline, Constant-time
• 영향: 암호화 키 탈취, 메모리 유출
• 대응: 패치, 격리, 하드웨어 가속
```
