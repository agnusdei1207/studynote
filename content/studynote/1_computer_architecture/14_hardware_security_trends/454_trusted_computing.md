# 454. 신뢰 컴퓨팅 (Trusted Computing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드웨어 기반 신뢰 체인으로 시스템 무결성 보장
> 2. **가치**: 측정 부팅, 원격 증명, 데이터 보호
> 3. **융합**: TPM, TCG, attestation, sealing

---

## Ⅰ. 개요

### 개념 정의

신뢰 컴퓨팅(Trusted Computing)은 **하드웨어 기반 신뢰 체인(Chain of Trust)을 통해 부팅부터 애플리케이션까지 시스템 무결성을 검증하고 보장하는 기술**이다.

### 💡 비유: 신분증 확인 체인

신뢰 컴퓨팅은 **신분증 확인 체인**과 같다. 은행이 있다. (시스템) 들어간다. (부팅) 경비원 확인. (BIOS) 창구직원 확인. (부트로더) 매니저 확인. (OS) 모두 믿는다. (신뢰 체인) 안전하다. 신뢰 컴퓨팅이 똑같다. 단계별 검증, 신뢰 전파, 무결성 보장.

### 신뢰 컴퓨팅 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                신뢰 컴퓨팅 개요                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TCG (Trusted Computing Group)】                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  구성 요소           설명                                                   │ │
│  │  ─────────           ────                                                   │ │
│  │  TPM                 Trusted Platform Module                               │ │
│  │  CRTM                Core Root of Trust for Measurement                    │ │
│  │  RTM                 Root of Trust for Measurement                         │ │
│  │  RTR                 Root of Trust for Reporting                           │ │
│  │  RTS                 Root of Trust for Storage                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【신뢰 체인 (Chain of Trust)】                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌───────┐    ┌───────┐    ┌───────┐    ┌───────┐            │ │
│  │  │ CRTM  │ →  │ BIOS  │ →  │Bootldr│ →  │  OS   │            │ │
│  │  │(신뢰  │    │/UEFI │    │       │    │       │            │ │
│  │  │ 루트) │    │       │    │       │    │       │            │ │
│  │  └───────┘    └───────┘    └───────┘    └───────┘            │ │
│  │      ↓            ↓            ↓            ↓                  │ │
│  │  PCR[0]       PCR[0-7]     PCR[4-5]     PCR[8-15]              │ │
│  │                                                               │ │
│  │  • CRTM이 BIOS 측정 → PCR[0]                                            │ │
│  │  • BIOS가 부트로더 측정 → PCR[4-5]                                       │ │
│  │  • 부트로더가 OS 측정 → PCR[8-15]                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 기능】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  기능              설명                    활용                   │ │
│  │  ────              ────                    ────                   │ │
│  │  측정 부팅         부트 과정 기록            무결성 검증           │ │
│  │  원격 증명         시스템 상태 증명          신뢰 검증              │ │
│  │  Sealing           조건부 데이터 복호화      조건부 접근           │ │
│  │  인증된 부트        부팅 무결성 확인          보안 부팅             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 신뢰 컴퓨팅 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                신뢰 컴퓨팅 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【측정 부팅 (Measured Boot)】                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  과정:                                                                    │ │
│  │  1. 전원 ON → CRTM 실행                                                  │ │
│  │  2. CRTM이 자신을 PCR[0]에 측정                                          │ │
│  │  3. CRTM이 BIOS/UEFI 측정 → PCR[0-7]                                     │ │
│  │  4. BIOS가 부트로더 측정 → PCR[4-5]                                       │ │
│  │  5. 부트로더가 커널 측정 → PCR[8-15]                                      │ │
│  │  6. TCG 이벤트 로그 저장                                                 │ │
│  │                                                               │ │
│  │  측정 방식:                                                               │ │
│  │  PCR[new] = SHA256(PCR[old] || measurement)                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【원격 증명 (Remote Attestation)】                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 증명자 (Attester): 대상 시스템                                         │ │
│  │  2. 검증자 (Verifier): 신뢰 서버                                           │ │
│  │                                                               │ │
│  │  프로세스:                                                               │ │
│  │  1. 검증자가 nonce 요청                                                   │ │
│  │  2. 증명자가 TPM으로 PCR 인용 생성                                        │ │
│  │  3. PCR 인용 + TCG 로그 전송                                              │ │
│  │  4. 검증자가 서명 검증 및 PCR 값 비교                                      │ │
│  │  5. 신뢰 여부 결정                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 하드웨어 기반 신뢰                                                    │ │
│  │  • 무결성 검증                                                          │ │
│  │  • 원격 증명 지원                                                        │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 프라이버시 우려                                                       │ │
│  │  • 복잡한 구현                                                          │ │
│  │  • 벤더 종속                                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 신뢰 컴퓨팅 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                신뢰 컴퓨팅 분석                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【PCR 값 확인】                                                                 │)
│  ──────────────────                                                  │
│  # TPM 2.0 PCR 읽기                                                       │
│  $ tpm2_pcrread sha256:0,1,2,3,4,5                                      │
│                                                                     │
│  # TCG 이벤트 로그                                                        │
│  $ tpm2_eventlog /sys/kernel/security/tpm0/binary_bios_measurements     │
│                                                                     │
│  【IMA 상태 확인】                                                               │)
│  ──────────────────                                                  │
│  # IMA 측정 목록                                                           │
│  $ cat /sys/kernel/security/ima/ascii_runtime_measurements              │
│                                                                     │
│  # IMA 정책                                                                │
│  $ cat /sys/kernel/security/ima/policy                                  │
│                                                                     │
│  【원격 증명 예시】                                                              │)
│  ──────────────────                                                  │
│  # 인용 생성                                                               │
│  $ tpm2_quote -c ak -l sha256:0,1,2,3 -q nonce                          │
│                                                                     │
│  # 인용 검증                                                               │
│  $ tpm2_checkquote -u ak.pub -m quote.msg -s quote.sig                  │
│                                                                     │
│  【신뢰 컴퓨팅 팁】                                                              │)
│  ──────────────────                                                  │
│  1. TPM 활성화: BIOS에서 TPM 활성화                                        │
│  2. PCR 모니터링: 정기적 PCR 값 비교                                       │
│  3. 이벤트 로그: 부팅 로그 분석                                             │
│  4. 증명 정책: 조직별 정책 수립                                             │
│  5. 키 관리: AK (Attestation Key) 관리                                     │
│  6. 복구: PCR 초기화 절차 문서화                                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 신뢰 컴퓨팅: 하드웨어 기반 신뢰 체인
• TCG: Trusted Computing Group
• CRTM: Core Root of Trust for Measurement
• 측정 부팅: 부트 과정 기록
• 원격 증명: 시스템 상태 증명
• Sealing: 조건부 데이터 복호화
• IMA: Integrity Measurement Architecture
• PCR: Platform Configuration Register
• 장점: 무결성 검증, 원격 증명
• 단점: 프라이버시, 복잡성
• 활용: 엔터프라이즈, 클라우드, IoT
```
