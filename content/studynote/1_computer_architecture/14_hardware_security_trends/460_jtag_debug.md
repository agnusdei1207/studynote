# 460. JTAG / 디버그 보안 (JTAG / Debug Security)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드웨어 디버그 인터페이스 보안
> 2. **가치**: 펌웨어 보호, IP 보호, 공격 방지
> 3. **융합**: JTAG, SWD, secure debug, fuse

---

## Ⅰ. 개요

### 개념 정의

JTAG/디버그 보안은 **하드웨어 디버그 인터페이스(JTAG, SWD 등)를 통한 무단 접근을 방지하고, 승인된 디버깅만 허용하는 보안 기술**이다.

### 💡 비유: 금고 비밀번호

JTAG 보안은 **금고 비밀번호**와 같다. 금고가 있다. (칩) 비밀번호가 있다. (디버그 보안) 아는 사람만 연다. (인증) 모르면 못 연다. (차단) 안전하다. JTAG 보안이 똑같다. 인증 기반 접근, 무단 디버깅 차단, 펌웨어 보호.

## Ⅱ. JTAG/디버그 보안 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                JTAG/디버그 보안 상세                                      │

│                                                                     │
│  【보안 메커니즘】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 물리적 비활성화:                                                       │
│  │     • 퓨즈(Fuse) 소각: JTAG 영구 비활성화                                  │ │
│  │     • PCB 미장착: 커넥터 제거                                              │ │
│  │     • 물리적 차단: 접점 차폐                                               │ │
│  │                                                               │ │
│  │  2. 인증 기반:                                                             │ │
│  │     • 비밀번호: 128비트 키 인증                                            │ │
│  │     • 챌린지-응답: 암호화 인증                                              │ │
│  │     • 인증서: PKI 기반                                                    │ │
│  │                                                               │ │
│  │  3. 조건부 활성화:                                                         │
│  │     • Secure Debug: 특정 조건에서만 활성화                                  │ │
│  │     • 생산 후 비활성화: 양산 시 퓨즈 소각                                    │ │
│  │     • 개발 모드 구분: DEV/PROD 모드                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【퓨즈(Fuse) 보안】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  퓨즈               설명                    효과                   │ │
│  │  ────               ────                    ────                   │ │
│  │  JTAG_DISABLE      JTAG 비활성화           디버깅 차단             │ │
│  │  SECURE_BOOT       보안 부팅 활성화        부트 보안               │ │
│  │  ENCRYPT           암호화 활성화           펌웨어 보호             │ │
│  │  DEBUG_KEY         디버그 인증 키          Secure Debug           │ │
│  │                                                               │ │
│  │  특징:                                                                  │ │
│  │  • 일방향: 한번 소각하면 복구 불가                                          │ │

│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • IP 보호                                                              │ │
│  │  • 펌웨어 보안                                                           │ │

│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 디버깅 어려움                                                        │ │

│  │  • 개발 복잡성                                                          │ │
```

## Ⅳ. 실무 적용

### JTAG/디버그 보안 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                JTAG/디버그 보안 분석                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【STM32 RDP 설정】                                                              │)
│  ──────────────────                                                  │
│  # STM32CubeProgrammer로 RDP 설정                                          │
│  $ STM32_Programmer_CLI -c port=SWD -ob RDP=0xBB                          │
│                                                                     │
│  # RDP 레벨                                                               │
│  # Level 0 (0xAA): 보호 없음                                               │ │
│  # Level 1 (0xBB): 읽기 보호                                               │ │
│  # Level 2 (0xCC): 영구 보호 (JTAG 차단)                                    │ │
│                                                                     │
│  【ESP32 eFuse】                                                                 │)
│  ──────────────────                                                  │
│  # eFuse 상태 확인                                                          │
│  $ espefuse.py summary                                                    │
│                                                                     │
│  # JTAG 비활성화                                                             │
│  $ espefuse.py burn_efuse JTAG_DISABLE                                     │
│                                                                     │
│  # 디버그 비활성화                                                           │
│  $ espefuse.py burn_efuse DISABLE_DL_ENCRYPT                               │
│                                                                     │
│  【nRF52 APPROTECT】                                                             │)
│  ──────────────────                                                  │
│  # nrfjprog로 설정                                                          │
│  $ nrfjprog --recover                                                     │
│  $ nrfjprog --memwr 0x10001208 --val 0x00000000                            │
│  $ nrfjprog --rbp ALL                                                     │
│                                                                     │
│  【JTAG 보안 팁】                                                                │)
│  ──────────────────                                                  │
│  1. 양산 전: RDP/APPROTECT 설정                                             │
│  2. 키 백업: 디버그 키 안전 저장                                             │
│  3. 물리적 보안: 커넥터 제거/차폐                                            │
│  4. 테스트: 보안 설정 확인                                                   │
│  5. RMA: 복구 절차 문서화                                                   │
│  6. 감사: 보안 설정 로그                                                    │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• JTAG: IEEE 1149.1 하드웨어 디버그 인터페이스
• SWD: Serial Wire Debug (ARM)
• TAP: Test Access Port
• 퓨즈: OTP 보안 설정
• RDP: Read Protection (STM32)
• APPROTECT: Access Port Protection (nRF)
• Secure Debug: 인증 기반 디버깅
• 장점: IP 보호, 펌웨어 보안
• 단점: 디버깅 어려움, RMA 문제
• 공격: 펌웨어 추출, 키 탈취
```
