# 235. 분기 역사 표 (BHT)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분기 결과 이력을 저장하는 테이블
> 2. **가치**: 2비트 예측기 기반, 패턴 학습
> 3. **융합**: 동적 예측, 지역/전역 예측기

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

분기 역사 표(BHT, Branch History Table)는 **각 분기 명령어의 과거 결과(Taken/Not Taken)를 저장하는 테이블**이다. 이 이력을 기반으로 다음 분기 결과를 예측한다.

### 💡 비유: 승패 기록

BHT는 **경기 승패 기록**과 같다. A팀이 최근 3번 연속 이겼다 → 다음에도 이길 것으로 예측. A팀이 1승 1패 반복 → 패턴을 파악해서 예측.

### BHT 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                BHT 구조                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【BHT 기본 구조】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ┌─────────────────────────────────────┐                        │ │
│  │  │  분기 PC (Index)   │  2비트 상태   │                          │ │
│  │  ├─────────────────────────────────────┤                        │ │
│  │  │  0x1000           │  11 (강한 T)  │                          │ │
│  │  │  0x2000           │  01 (약한 NT) │                          │ │
│  │  │  0x3000           │  10 (약한 T)  │                          │ │
│  │  │  ...              │  ...         │                          │ │
│  │  └─────────────────────────────────────┘                        │ │
│  │                    ↑                                            │ │
│  │                    │                                            │ │
│  │            분기 PC로 인덱싱                                        │ │
│  │                                                               │ │
│  │  • 분기 PC: 분기 명령어의 주소 (Index)                                │ │
│  │  • 2비트 상태: 예측기 상태 (00, 01, 10, 11)                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【2비트 예측기 상태 전이도】                                            │
│  ──────────────────────────                                          │
│                                                                     │
│         Taken                Not Taken                              │
│         ────                 ────────                                │
│                                                                     │
│  00 (강한 NT) ────────► 01 (약한 NT) ────────► 00 (강은 NT)          │
│       │                        │                                     │
│       │ Taken               │ Not Taken                            │
│       ▼                        ▼                                     │
│  01 (약한 NT) ────────► 10 (약한 T)  ────────► 00 (강은 NT)          │
│       │                        │                                     │
│       │ Taken               │ Not Taken                            │
│       ▼                        ▼                                     │
│  10 (약한 T)  ────────► 11 (강한 T)  ────────► 01 (약은 NT)          │
│       │                        │                                     │
│       │ Taken               │ Not Taken                            │
│       ▼                        ▼                                     │
│  11 (강한 T)  ────────► 11 (강은 T)  ────────► 10 (약은 T)          │
│       │                        │                                     │
│       │ Not Taken           │ Not Taken                            │
│       ▼                        ▼                                     │
│  10 (약은 T)                                                      │
│                                                                     │
│  【상태별 예측】                                                        │
│  ──────────────────                                                  │
│  상태    예측        설명                                                │
│  ────   ────        ────                                                │
│  00    Not Taken    강한 NT, 2번 연속 NT 필요                           │
│  01    Not Taken    약한 NT, 1번 NT로 전이                              │
│  10    Taken        약한 T, 1번 T로 전이                                 │
│  11    Taken        강한 T, 2번 연속 T 필요                               │
│                                                                     │
│  【루프에서의 동작】                                                      │
│  ──────────────────                                                  │
│  for (i = 0; i < 100; i++) { ... }                                  │
│                                                                     │
│  반복    결과        상태 변화                                          │
│  ────   ────        ─────────                                          │
│  1      T           00→01→10→11 (점점 강해짐)                            │
│  2      T           11→11 (강한 T 유지)                                 │
│  ...    T           11→11 (강한 T 유지)                                 │
│  100    NT          11→10→01→00 (점점 약해짐)                            │
│                                                                     │
│  → 루프 본문에서는 "강한 T" 상태 유지 → 예측 정확                        │
│  → 루프 끝에서만 1번 틀림 (히스테리시스 해결)                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BHT 크기:                                                             │ │
│  │  ──────────────────                                                  │ │
│  │  CPU             BHT 엔트리    예측기 타입                               │ │
│  │  ───────────────  ───────────  ──────────────────                      │ │
│  │  Intel Skylake    4,096        2비트 + 토너먼트                          │ │
│  │  AMD Zen 3        6,144        2비트 + 토너먼트                          │ │
│  │  ARM Cortex-A78   1,024        2비트                                     │ │
│  │  Apple M1         4,096        2비트 + 토너먼트                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. BHT 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                BHT 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1비트 vs 2비트 예측기】                                               │
│  ────────────────────────                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1비트 예측기:                                                          │ │
│  │  • 상태: T 또는 NT (1비트)                                              │ │
│  │  • 규칙: 마지막 결과를 다음 예측으로 사용                                  │ │
│  │  • 문제: 루프 끝에서 2번 틀림                                            │ │
│  │    - 마지막 반복: T → NT (틀림)                                         │ │
│  │    - 첫 반복: NT → T (틀림)                                            │ │
│  │  → 루프당 2번 미스                                                     │ │
│  │                                                               │ │
│  │  2비트 예측기:                                                          │ │
│  │  • 상태: 4개 (00, 01, 10, 11)                                          │ │
│  │  • 규칙: 연속 2번 같은 결과여야 예측 변경                                  │ │
│  │  • 장점: 루프 끝에서 1번만 틀림                                          │ │
│  │    - 마지막 반복: 11→10 (예측 T, 실제 NT → 틀림)                         │ │
│  │    - 첫 반복: 10→11 (예측 T, 실제 T → 맞음)                              │ │
│  │  → 루프당 1번 미스                                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지역 vs 전역 예측기】                                                 │
│  ────────────────────────                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  지역 예측기 (Local Predictor):                                         │ │
│  │  • 각 분기별로 독립적인 2비트 카운터                                       │ │
│  │  • 분기 PC로 직접 인덱싱                                                 │ │
│  │  • 장점: 단순함                                                          │ │
│  │  • 단점: 분기 간 상호작용 무시                                            │ │
│  │                                                               │ │
│  │  전역 예측기 (Global Predictor):                                        │ │
│  │  • 모든 분기가 공유하는 큰 BHT                                           │ │
│  │  • 분기 PC의 일부 비트만 인덱싱 (충돌 가능)                                │ │
│  │  • 장점: 패턴 인식 (상관 분기)                                           │ │
│  │  • 단점: 인덱싱 충돌                                                    │ │
│  │                                                               │ │
│  │  토너먼트 예측기 (Tournament Predictor):                                 │ │
│  │  • 지역 + 전역 예측기 경쟁                                               │ │
│  │  • 더 잘 맞는 예측기 선택                                                │ │
│  │  • 장점: 두 예측기 장점 결합                                             │ │
│  │  • 정확도: 97~99%                                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상관 분기 예측 (Correlating Predictor)】                              │
│  ──────────────────────────────────────                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제: if (a == b) goto L1;                                           │ │
│  │        if (c == d) goto L2;  // 이전 분기 결과가 이 분기에 영향            │ │
│  │                                                               │ │
│  │  (m,n) 예측기:                                                          │ │
│  │  • m: 이전 분기 결과 수 (m개)                                            │ │
│  │  • n: 각 패턴별 예측기 비트 수 (n비트)                                     │ │
│  │  • 예: (2,2) 예측기 = 4개의 2비트 예측기                                   │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────┐                         │ │
│  │  │  이전 결과 (2비트)   │  사용할 예측기      │                         │ │
│  │  ├─────────────────────────────────────────┤                         │ │
│  │  │  TT               │  예측기 0           │                         │ │
│  │  │  TN               │  예측기 1           │                         │ │
│  │  │  NT               │  예측기 2           │                         │ │
│  │  │  NN               │  예측기 3           │                         │ │
│  │  └─────────────────────────────────────────┘                         │ │
│  │  → 이전 분기 결과에 따라 다른 예측기 사용                                  │ │
│  │  → 정확도: 93~97%                                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BHT 정확도 비교:                                                        │ │
│  │  ──────────────────                                                  │ │
│  │  예측기             정확도      하드웨어                                   │ │
│  │  ───────────────  ──────    ────────                                   │ │
│  │  1비트              80~90%    매우 작음                                  │ │
│  │  2비트              90~95%    작음                                        │ │
│  │  (2,2) 상관         93~97%    중간                                        │ │
│  │  지역+전역 토너먼트    95~98%    큼                                        │ │
│  │  TAGE              97~99%    매우 큼                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. BHT × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                BHT × 타 과목 융합                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 파이프라인: BHT로 분기 페널티 감소                                     │
│  • 슈퍼스칼라: 다중 BHT 필요                                             │
│  • 비순차 실행: BHT로 스펙클레이티브 실행                                   │
│  • 전력: BHT는 SRAM, 전력 소모 있음                                       │
│                                                                     │
│  【컴파일러】                                                          │
│  • 분기 패턴 분석                                                       │
│  • 코드 배치 최적화                                                     │
│  • PGO: 프로파일 기반 BHT 초기화                                         │
│  • 루프 최적화                                                          │
│                                                                     │
│  【보안】                                                              │
│  • Spectre: BHT 상태 노출                                              │
│  • 사이드 채널: BHT 접근 시간 측정                                        │
│  • 완화: BHT 플러시                                                     │
│                                                                     │
│  【성능 분석】                                                         │
│  • BHT 적중률 측정                                                      │
│  • 분기 패턴 분석                                                       │
│  • 예측 정확도 분석                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BHT 플러시 시나리오:                                                    │ │
│  │  ──────────────────────                                              │ │
│  │  1. 컨텍스트 스위치: BHT 상태가 새 프로세스에 부적합                       │ │
│  │  2. 시스템 콜: 커널 분기 패턴과 다름                                     │ │
│  │  3. 가상화: VM 입장/출장 시 BHT 부적합                                  │ │
│  │  4. 보안: Spectre 완화를 위해 BHT 플러시                                │ │
│  │                                                               │ │
│  │  해결:                                                                │ │
│  │  • 분기 PC 태그 추가 (충돌 감소)                                        │ │
│  │  • 부분 플러시 (관련 엔트리만)                                           │ │
│  │  • BHT 공유 (하이퍼바이저)                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### BHT 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                BHT 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【BHT 정확도 측정】                                                    │
│  ──────────────────                                                  │
│  $ perf stat -e branches,branch-misses ./prog                       │
│  branches: 1,000,000,000                                            │
│  branch-misses: 20,000,000                                          │
│  → 2% 미스율 (98% 정확도)                                              │
│                                                                     │
│  【BHT 패턴 분석】                                                      │
│  ──────────────────                                                  │
│  $ perf record -e branch-misses ./prog                             │
│  $ perf report                                                      │
│  → 어떤 분기에서 미스 발생하는지 확인                                      │
│                                                                     │
│  【BHT 플러시 효과】                                                      │
│  ──────────────────                                                  │
│  $ perf stat -e context-switches,branch-misses ./prog              │
│  context-switches: 100,000                                          │
│  branch-misses: 5,000,000                                           │
│  → 컨텍스트 스위치 후 분기 미스 증가                                       │
│                                                                     │
│  【BHT 친화적 코드】                                                      │
│  ──────────────────                                                  │
│  # Before (BHT 미스 높음)                                              │
│  if (data[i] % 2 == 0) { ... }  ; 패턴 없음 (랜덤)                      │
│                                                                     │
│  # After (BHT 친화적)                                                  │
│  if (likely(i < limit)) { ... }  ; 95% 확률                          │
│  → BHT가 빠르게 "강한 T" 학습                                            │
│                                                                     │
│  【Spectre 완화】                                                       │
│  ──────────────────                                                  │
│  # retpoline: 분기 예측 비활성화                                          │
│  $ gcc -mretpoline prog.c                                           │
│  → BHT 사용 안 함 (성능 저하, 보안 향상)                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BHT 최적화 팁:                                                         │ │
│  │  ──────────────────                                                  │ │
│  │  1. 분기 패턴 친화적 코드 작성                                            │ │
│  │  2. 루프에서 일관된 분기 방향                                              │ │
│  │  3. 랜덤 분기 피하기                                                    │ │
│  │  4. PGO 활용                                                        │ │
│  │  5. 함수 인라이닝 (분기 감소)                                            │ │
│  │  6. likely/unlikely 매크로                                            │ │
│  │  7. 분기 밀도 낮추기                                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• BHT: 분기 결과 이력을 저장하는 테이블
• 2비트 예측기: 강한/약한 T/NT 상태
• 목적: 분기 패턴 학습으로 예측 정확도 향상
• 히스테리시스: 연속 2번 틀려야 예측 변경
• 루프: 본문에서 "강한 T", 끝에서 1번만 미스
• 상관 예측: 이전 분기 결과 고려
• 토너먼트: 지역 + 전역 예측기 경쟁
• 정확도: 90~99%
```
