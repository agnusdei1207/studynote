# 205. 데이터패스 (Datapath)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 내부 데이터 흐름 경로, 연산 수행
> 2. **가치**: 명령어 실행을 위한 하드웨어 자원 구성
> 3. **융합**: ALU, 레지스터 파일, 버스, 멀티플렉서

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

데이터패스(Datapath)는 **CPU에서 데이터가 흐르고 연산이 수행되는 하드웨어 경로**다. 레지스터 파일, ALU, 멀티플렉서, 버스 등으로 구성되며, 제어 유닛의 신호에 따라 명령어를 실행한다.

### 💡 비유: 공장의 조립 라인

데이터패스는 **공장의 조립 라인**과 같다. 부품(데이터)이 컨베이어 벨트(버스)를 타고 이동한다. 각 공정(레지스터, ALU)에서 가공된다. 작업 지시서(제어 신호)에 따라 다음 공정으로 이동한다.

## Ⅱ. 데이터패스 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터패스 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【R-type 명령어 경로 (add $t0, $t1, $t2)】                          │
│  ────────────────────────────────────────────                       │
│  1. Fetch: IR = Mem[PC]; PC = PC + 4                                │
│  2. Decode: rs=$t1, rt=$t2, rd=$t0                                 │
│  3. Execute: ALUout = $t1 + $t2                                     │
│  4. Write: $t0 = ALUout                                             │
│                                                                     │
│  【Load 명령어 경로 (lw $t0, 100($t1))】                             │
│  ────────────────────────────────────────────                       │
│  1. Fetch: IR = Mem[PC]; PC = PC + 4                                │
│  2. Decode: rs=$t1, imm=100                                         │
│  3. Address: ALUout = $t1 + 100                                     │
│  4. Memory: $t0 = Mem[ALUout]                                       │
│                                                                     │
│  【Store 명령어 경로 (sw $t0, 100($t1))】                            │
│  ────────────────────────────────────────────                       │
│  1. Fetch: IR = Mem[PC]; PC = PC + 4                                │
│  2. Decode: rs=$t1, rt=$t0, imm=100                                │
│  3. Address: ALUout = $t1 + 100                                     │
│  4. Memory: Mem[ALUout] = $t0                                       │
│                                                                     │
│  【Branch 명령어 경로 (beq $t0, $t1, label)】                        │
│  ──────────────────────────────────────────────                     │
│  1. Fetch: IR = Mem[PC]; PC = PC + 4                                │
│  2. Decode: rs=$t0, rt=$t1, offset=label                           │
│  3. Compare: Zero = ($t0 == $t1)                                    │
│  4. Branch: if (Zero) PC = PC + offset*4                           │
│                                                                     │
│  【멀티플렉서(MUX) 역할】                                            │
│  ──────────────────────────                                         │
│  • PC 소스: PC+4 vs Branch 주소                                     │
│  • ALU 입력: 레지스터 vs 즉시값                                      │
│  • ALU 연산: add vs sub vs and vs or                                │
│  • 쓰기 데이터: ALU 결과 vs 메모리 로드                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단일 vs 다중 사이클 데이터패스:                                    │ │
│  │  ────────────────────────────────                                 │ │
│  │  단일 사이클:                                                      │ │
│  │  • 모든 명령어 1 클럭에 완료                                       │ │
│  │  • 클럭 = 가장 느린 명령어 시간                                    │ │
│  │  • 중복 하드웨어 (메모리 분리)                                     │ │
│  │                                                               │ │
│  │  다중 사이클:                                                      │ │
│  │  • 명령어마다 여러 클럭                                            │ │
│  │  • 하드웨어 재사용                                                 │ │
│  │  • 더 효율적인 리소스 활용                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 데이터패스 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터패스 × 타 과목 융합                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【디지털 논리회로】                                                 │
│  • 게이트: AND, OR, NOT, XOR                                        │
│  • MUX: 2:1, 4:1, 8:1 멀티플렉서                                   │
│  • ALU: 가산기, 비교기, 시프터                                      │
│  • 레지스터: D 플립플롭 배열                                         │
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 파이프라인: IF-ID-EX-MEM-WB                                      │
│  • 해저드: 데이터, 제어, 구조적                                      │
│  • 포워딩: ALU 결과 전달                                             │
│  • 스톨: 파이프라인 정지                                              │
│                                                                     │
│  【컴파일러】                                                        │
│  • 명령어 스케줄링: 데이터패스 고려                                  │
│  • 레지스터 할당: 레지스터 파일 활용                                  │
│  • 지연 슬롯: 분기 후 명령어 배치                                    │
│                                                                     │
│  【VLSI 설계】                                                       │
│  • 배치: 데이터패스 레이아웃                                         │
│  • 배선: 버스 설계                                                   │
│  • 타이밍: 크리티컬 패스 분석                                        │
│  • 전력: 데이터패스 전력 최적화                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터패스 설계 트레이드오프:                                      │ │
│  │  ────────────────────────────                                     │ │
│  │  • 성능 vs 비용: 더 많은 포트 = 더 빠름, 더 비쌈                   │ │
│  │  • 단순성 vs 복잡성: 단일 사이클 vs 파이프라인                      │ │
│  │  • 전력 vs 성능: 고성능 = 높은 전력                                │ │
│  │  • 면적 vs 속도: 더 많은 하드웨어 = 더 빠름                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 데이터패스 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터패스 분석                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MIPS 단일 사이클 데이터패스 분석】                                 │
│  ──────────────────────────────────                                 │
│  명령어: add $t0, $t1, $t2                                          │
│                                                                     │
│  1. PC → Instruction Memory                                        │
│     • PC = 0x00400000                                              │
│     • IR = 0x012A4020 (add $t0, $t1, $t2)                          │
│                                                                     │
│  2. Register File Read                                              │
│     • Read Register 1 = $t1 (9)                                    │
│     • Read Register 2 = $t2 (10)                                   │
│     • Read Data 1 = $t1의 값                                        │
│     • Read Data 2 = $t2의 값                                        │
│                                                                     │
│  3. ALU Operation                                                   │
│     • ALU Control = ADD                                            │
│     • ALU Result = $t1 + $t2                                        │
│                                                                     │
│  4. Register File Write                                             │
│     • Write Register = $t0 (8)                                     │
│     • Write Data = ALU Result                                       │
│                                                                     │
│  5. PC Update                                                       │
│     • PC = PC + 4 = 0x00400004                                     │
│                                                                     │
│  【크리티컬 패스 분석】                                               │
│  ──────────────────────────                                         │
│  단일 사이클 데이터패스 지연:                                        │
│  T_clk = T_PC + T_mem + T_RFread + T_ALU + T_RFwrite               │
│        = 30ns + 200ns + 100ns + 200ns + 50ns                       │
│        = 580ns                                                      │
│                                                                     │
│  → 클럭 주파수 = 1/580ns ≈ 1.7 MHz                                  │
│  → 매우 느림! 파이프라인 필요                                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터패스 시뮬레이션 (Verilog):                                   │ │
│  │  ─────────────────────────────                                    │ │
│  │  module datapath(                                               │ │
│  │      input clk, reset,                                          │ │
│  │      input [31:0] instr,                                        │ │
│  │      output [31:0] pc, alu_result                               │ │
│  │  );                                                             │ │
│  │      reg [31:0] pc, rf[0:31];                                   │ │
│  │      wire [4:0] rs, rt, rd;                                     │ │
│  │      wire [31:0] rf_out1, rf_out2;                              │ │
│  │      wire [31:0] alu_out;                                       │ │
│  │      // ...                                                     │ │
│  │  endmodule                                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 데이터패스: CPU 내부 데이터 흐름 경로
• 구성: PC, 메모리, 레지스터 파일, ALU, MUX
• 명령어 실행: Fetch → Decode → Execute → Memory → Write
• 단일 사이클: 모든 명령어 1 클럭, 비효율적
• 다중 사이클: 명령어마다 여러 클럭, 하드웨어 재사용
• 파이프라인: 병렬 처리로 성능 향상
```
