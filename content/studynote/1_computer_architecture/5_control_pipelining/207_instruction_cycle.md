# 207. 명령어 사이클 (Instruction Cycle)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어 실행의 기본 단계 (Fetch-Decode-Execute)
> 2. **가치**: CPU 동작의 기본 단위, 모든 명령어 공통
> 3. **융합**: 인터럽트, 파이프라인, 다중 사이클

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

명령어 사이클(Instruction Cycle)은 **CPU가 한 명령어를 가져와서 실행하는 전체 과정**이다. Fetch(인출) → Decode(해독) → Execute(실행)의 기본 단계로 구성되며, 모든 명령어가 이 기본 사이클을 거친다.

### 💡 비유: 요리 조리 과정

명령어 사이클은 **요리 조리 과정**과 같다. 냉장고에서 재료 꺼내기(Fetch) → 레시피 보기(Decode) → 요리하기(Execute) → 담기(Store). 모든 요리가 이 과정을 거치지만, 요리 종류에 따라 구체 조리법이 다르다.

### 명령어 사이클 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                명령어 사이클 구조                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 사이클】                                                        │
│  ─────────────                                                        │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  1. Fetch Cycle (인출 사이클)                                 │   │
│  │     • PC에서 명령어 주소 가져옴                                 │   │
│  │     • 메모리에서 명령어 읽음                                │   │
│  │     • PC 증가 (PC = PC + 4)                                   │   │
│  │     • 명령어를 IR에 저장                                        │   │
│  │                                                           │   │
│  │  2. Decode Cycle (해독 사이클)                                 │   │
│  │     • IR의 opcode 분석                                         │   │
│  │     • 명령어 형식 해석                                          │   │
│  │     • 필요한 레지스터 읽기                                      │   │
│  │     • 제어 신호 생성                                             │   │
│  │                                                           │   │
│  │  3. Execute Cycle (실행 사이클)                               │   │
│  │     • ALU 연산 수행                                               │   │
│  │     • 메모리 접근 (Load/Store)                                  │   │
│  │     • 결과를 레지스터에 저장                                     │   │
│  │     • 분기 시 PC 갱신                                            │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【확장 사이클】                                                        │
│  ─────────────                                                        │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  4. Indirect Cycle (간접 사이클)                              │   │
│  │     • 간접 주소 지정 시 필요                                    │   │
│  │     • 유효 주소 계산                                              │   │
│  │     • 추가 메모리 접근                                            │   │
│  │                                                           │   │
│  │  5. Interrupt Cycle (인터럽트 사이클)                         │   │
│  │     • 인터럽트 요청 확인                                          │   │
│  │     • PC 저장                                                      │   │
│  │     • ISR로 분기                                                   │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어 사이클 흐름:                                               │ │
│  │  ────────────────────                                              │ │
│  │                                                               │ │
│  │  ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐           │ │
│  │  │ Fetch  │───▶│ Decode │───▶│ Execute│───▶│ 다음   │           │ │
│  │  └────────┘    └────────┘    └────────┘    └────────┘           │ │
│  │       │            │            │            │                  │ │
│  │       ▼            ▼            ▼            ▼                  │ │
│  │  ┌────────────────────────────────────────────────────────────┐│ │
│  │  │ 필요시: Indirect Cycle 또는 Interrupt Cycle 진입           ││ │
│  │  └────────────────────────────────────────────────────────────┘│ │
│  │                                                               │ │
│  │  → 정상 흐름: Fetch → Decode → Execute → Fetch → ...           │ │
│  │  → 간접: Fetch → Decode → Indirect → Execute → Fetch           │ │
│  │  → 인터럽트: Fetch → Decode → Execute → Interrupt → ISR        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 명령어 사이클 상세

### 사이클 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                명령어 사이클 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Fetch Cycle 상세】                                                   │
│  ────────────────────                                                  │
│  마이크로 오퍼레이션:                                                 │
│  MAR ← PC              ; PC를 주소 레지스터로                            │
│  MBR ← M[AR]            ; 메모리에서 명령어 읽기                          │
│  IR ← MBR              ; 명령어 레지스터에 저장                          │
│  PC ← PC + 1            ; PC 증가                                    │
│                                                                     │
│  【Decode Cycle 상세】                                                 │
│  ────────────────────                                                  │
│  마이크로 오퍼레이션:                                                 │
│  decode(IR)             ; opcode 분석                                 │
│  CU ← decode 결과       ; 제어 신호 생성                               │
│  if (opcode == LOAD/STORE)                                            │
│      read registers                                                  │
│  if (opcode == R-type)                                               │
│      read rs, rt                                                       │
│                                                                     │
│  【Execute Cycle 상세】                                                │
│  ────────────────────                                                  │
│  R-type (ADD):                                                        │
│  AC ← AC + DR           ; 누산기 + 데이터 레지스터                    │
│                                                                     │
│  Load:                                                                │
│  AR ← AR + IR(addr)     ; 주소 계산                                   │
│  DR ← M[AR]             ; 메모리에서 데이터 로드                      │
│  AC ← DR                ; 누산기에 저장                                 │
│                                                                     │
│  Store:                                                               │
│  AR ← AR + IR(addr)     ; 주소 계산                                   │
│  M[AR] ← AC             ; 메모리에 저장                                 │
│                                                                     │
│  Branch:                                                              │
│  if (condition)                                                      │
│      PC ← IR(addr)        ; 분기                                        │
│  else                                                                │
│      PC ← PC (유지)                                                   │
│                                                                     │
│  【Indirect Cycle 상세】                                               │
│  ────────────────────────                                              │
│  MAR ← IR(addr)          ; 간접 주소                                   │
│  MBR ← M[MAR]            ; 유효 주소 읽기                              │
│  IR(addr) ← MBR          ; 유효 주소 저장                               │
│                                                                     │
│  【Interrupt Cycle 상세】                                              │
│  ──────────────────────────                                            │
│  SP ← SP - 1             ; 스택 포인터 감소                            │
│  M[SP] ← PC              ; PC를 스택에 저장                             │
│  PC ← ISR_address        ; ISR로 분기                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어별 사이클 수:                                                │ │
│  │  ──────────────────                                                │ │
│  │  명령어        사이클 수   비고                                      │ │
│  │  ───────────  ─────────  ──────────────────                       │ │
│  │  R-type        3         Fetch-Decode-Execute                    │ │
│  │  Load         4         Fetch-Decode-Execute(Mem)              │ │
│  │  Store        4         Fetch-Decode-Execute(Mem)              │ │
│  │  Branch       3         Fetch-Decode-Execute                   │ │
│  │  간접 주소     4~5       +Indirect Cycle                         │ │
│  │  인터럽트     +1         +Interrupt Cycle                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 명령어 사이클 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                명령어 사이클 × 타 과목 융합                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【운영체제】                                                        │
│  • 인터럽트: ISR, 컨텍스트 스위치                                    │
│  • 시스템 콜: Trap 명령어로 전환                                    │
│  • 예외 처리: Page Fault, Divide by Zero                           │
│  • 타이머: 주기적 인터럽트                                           │
│                                                                     │
│  【컴파일러】                                                        │
│  • 코드 생성: 사이클 고려 최적화                                     │
│  • 지연 슬롯: MIPS 분기 후 명령어                                   │
│  • 파이프라인 스케줄링: 해저드 회피                                  │
│                                                                     │
│  【컴퓨터 구조】                                                     │
│  • 파이프라인: 사이클 중첩 실행                                      │
│  • 슈퍼스칼라: 다중 사이클 동시 처리                                  │
│  • CPI: 사이클/명령어 비율                                            │
│  • 성능: CPI × 클럭 주기                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPI (Cycles Per Instruction):                                     │ │
│  │  ────────────────────────────                                      │ │
│  │  • 단일 사이클: CPI = 1 (모든 명령어 1클럭)                       │ │
│  │  • 다중 사이클: CPI = 명령어별 사이클 수                           │ │
│  │  • 파이프라인: CPI ≈ 1 (이상적)                                    │ │
│  │  • 실제 파이프라인: CPI > 1 (해저드로)                            │ │
│  │                                                               │ │
│  │  CPU Time = IC × CPI × CC                                         │ │
│  │  (IC: 명령어 수, CPI: 사이클/명령어, CC: 클럭 주기)                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 명령어 사이클 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                명령어 사이클 분석                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MIPS 다중 사이클 구현】                                            │
│  ──────────────────────                                              │
│  상태 전이:                                                          │
│  FETCH → DECODE → (EXECUTE | MEMORY | WRITEBACK) → FETCH           │
│                                                                     │
│  상태별 마이크로 오퍼레이션:                                          │
│  FETCH:                                                              │
│    IR = Mem[PC]; PC = PC + 4                                        │
│                                                                     │
│  DECODE:                                                             │
│    A = Reg[IR[25:21]]; B = Reg[IR[20:16]]                          │
│    ALUOut = PC + (SignExt(IR[15:0]) << 2)                          │
│                                                                     │
│  EXECUTE (R-type):                                                   │
│    ALUOut = A op B                                                   │
│                                                                     │
│  EXECUTE (Load/Store):                                               │
│    ALUOut = A + SignExt(IR[15:0])                                   │
│                                                                     │
│  MEMORY (Load):                                                      │
│    MDR = Mem[ALUOut]                                                │
│                                                                     │
│  MEMORY (Store):                                                     │
│    Mem[ALUOut] = B                                                   │
│                                                                     │
│  WRITEBACK:                                                          │
│    Reg[IR[15:11]] = ALUOut (또는 MDR)                               │
│                                                                     │
│  【성능 분석】                                                        │
│  ──────────────────                                                  │
│  $ perf stat -e cycles,instructions ./prog                          │
│  cycles: 1,234,567,890                                               │
│  instructions: 987,654,321                                          │
│  CPI: 1.25                                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPI 최적화:                                                        │ │
│  │  ──────────────                                                    │ │
│  │  1. 파이프라인: 사이클 중첩                                        │ │
│  │  2. 캐시: 메모리 지연 감소                                         │ │
│  │  3. 분기 예측: 제어 해저드 감소                                    │ │
│  │  4. 포워딩: 데이터 해저드 감소                                    │ │
│  │  5. 비순차 실행: 스톨 감소                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 명령어 사이클: 명령어 실행의 기본 단위
• 기본: Fetch → Decode → Execute
• 확장: Indirect, Interrupt Cycle
• CPI: 명령어당 사이클 수
• CPU Time = IC × CPI × CC
• 최적화: 파이프라인, 캐시, 분기 예측
```
