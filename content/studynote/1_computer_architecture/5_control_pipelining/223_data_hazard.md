# 223. 데이터 해저드 (Data Hazard)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 의존성으로 인한 파이프라인 스톨
> 2. **가치**: RAW가 가장 흔하며, 포워딩으로 대부분 해결
> 3. **융합**: RAW/WAR/WAW, 포워딩, 스톨, 스케줄링

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

데이터 해저드(Data Hazard)는 **명령어 간의 데이터 의존성으로 인해 파이프라인이 스톨되는 현상**이다. 이전 명령어의 결과가 아직 준비되지 않았는데 다음 명령어가 그 데이터를 필요로 할 때 발생한다.

### 💡 비유: 요리의 재료 대기

데이터 해저드는 **요리의 재료 대기**와 같다. "야채를 썰어서(ADD) → 볶아라(SUB)"라고 하면, 야채가 다 썰릴 때까지 볶을 수 없다. 포워딩은 채반에서 바로 프라이팬으로 옮기는 것과 같다.

### 데이터 해저드 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 해저드 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데이터 해저드 종류】                                                 │
│  ────────────────────────                                            │
│                                                                     │
│  1. RAW (Read After Write) - 가장 흔함                                │
│     ADD R1, R2, R3   ; R1 ← R2 + R3 (Write)                         │
│     SUB R4, R1, R5   ; R4 ← R1 - R5 (Read R1)                       │
│     → SUB가 R1을 읽으려는데 ADD가 아직 R1에 안 씀                       │
│                                                                     │
│  2. WAR (Write After Read)                                           │
│     SUB R4, R1, R5   ; R4 ← R1 - R5 (Read R1)                       │
│     ADD R1, R2, R3   ; R1 ← R2 + R3 (Write R1)                      │
│     → ADD가 R1에 쓰려는데 SUB가 아직 R1을 안 읽음                       │
│     → 파이프라인에서는 거의 발생 안 함                                   │
│                                                                     │
│  3. WAW (Write After Write)                                          │
│     ADD R1, R2, R3   ; R1 ← R2 + R3 (Write R1)                      │
│     SUB R1, R4, R5   ; R1 ← R4 - R5 (Write R1)                      │
│     → SUB가 R1에 쓰려는데 ADD가 아직 R1에 안 씀                         │
│     → 비순차 실행에서 문제                                              │
│                                                                     │
│  【RAW 해저드 타이밍】                                                  │
│  ────────────────────────                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ADD R1, R2, R3                                                  │ │
│  │  SUB R4, R1, R5    ; R1 필요                                      │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    ADD                                                       │ │
│  │   2    SUB   ADD        ← ADD ID: R2, R3 읽기                     │ │
│  │   3          SUB   ADD   ← ADD EX: R2 + R3 (결과 생성)             │ │
│  │   4                SUB   ADD    ← ADD MEM                         │ │
│  │   5    ★     ★     SUB   ADD    ← ADD WB: R1 기록                  │ │
│  │         ↑           ↑                                            │ │
│  │   SUB ID에서 R1 읽기 (아직 R1 안 쓰여짐)                            │ │
│  │                                                               │ │
│  │  ★ = 스톨 필요 또는 포워딩                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결 방안】                                                         │
│  ──────────────────                                                  │
│  1. 스톨 (Stall): 파이프라인 정지, 결과 대기                            │
│  2. 포워딩 (Forwarding): 결과를 바로 전달                               │
│  3. 코드 재배치: 의존성 없는 명령어 배치                                 │
│  4. 레지스터 리네이밍: WAR/WAW 해결                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  해저드별 해결 복잡도:                                                 │ │
│  │  ──────────────────────                                              │ │
│  │  해저드    발생 빈도    해결 난이도    해결 방법                         │ │
│  │  ──────   ─────────   ────────────  ──────────                        │ │
│  │  RAW      매우 높음     쉬움          포워딩                           │ │
│  │  WAR      낮음         중간          리네이밍                          │ │
│  │  WAW      낮음         중간          리네이밍                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 데이터 해저드 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 해저드 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스톨로 해결 (비효율적)】                                             │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ADD R1, R2, R3                                                  │ │
│  │  SUB R4, R1, R5                                                  │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    ADD                                                       │ │
│  │   2    SUB   ADD                                                 │ │
│  │   3    -     -     ADD    ← 스톨                                  │ │
│  │   4    -     -     -     ADD                                     │ │
│  │   5    -     SUB    -     -     ADD                              │ │
│  │   6          -     SUB    -     -                                │ │
│  │                                                               │ │
│  │  → 3클럭 스톨 발생                                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【포워딩으로 해결 (효율적)】                                           │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ADD R1, R2, R3                                                  │ │
│  │  SUB R4, R1, R5                                                  │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    ADD                                                       │ │
│  │   2    SUB   ADD                                                 │ │
│  │   3          SUB   ADD     ← EX→EX 포워딩                         │ │
│  │   4                SUB   ADD                                     │ │
│  │   5                      SUB   ADD                               │ │
│  │                          ↑                                       │ │
│  │                    MEM→EX 포워딩 가능                              │ │
│  │                                                               │ │
│  │  → 스톨 없이 실행                                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Load-Use 해저드 (포워딩으로도 해결 안 됨)】                           │
│  ────────────────────────────────────                                │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  LW  R1, 0(R2)    ; R1 ← Mem[R2]                                 │ │
│  │  ADD R3, R1, R4   ; R3 ← R1 + R4                                 │ │
│  │                                                               │ │
│  │  LW 결과는 MEM 단계 끝나야 사용 가능                                  │ │
│  │  ADD는 EX 단계에서 R1 필요                                         │ │
│  │  → 포워딩해도 1클럭 스톨 필요                                        │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    LW                                                        │ │
│  │   2    ADD   LW                                                  │ │
│  │   3    -     -     LW     ← 1클럭 스톨                              │ │
│  │   4          ADD    -     LW    ← 포워딩                          │ │
│  │   5                      ADD   LW                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  포워딩 경로:                                                         │ │
│  │  ──────────────────                                                  │ │
│  │  1. EX/MEM → EX: ALU 결과 → 다음 명령어 ALU 입력                    │ │
│  │  2. MEM/WB → EX: 메모리 데이터 → 다음 명령어 ALU 입력                 │ │
│  │  3. EX/MEM → MEM: Store 데이터 → 다음 명령어 MEM 입력                │ │
│  │                                                               │ │
│  │  포워딩 조건:                                                        │ │
│  │  • 이전 명령어의 목적지 = 현재 명령어의 소스                           │ │
│  │  • 목적지가 $0 (zero)이 아님                                        │ │
│  │  • WB 아직 안 됨                                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │
│  【알고리즘】                                                          │
│  • 데이터 흐름 분석                                                     │
│  • 의존성 그래프                                                       │
│  • 병렬화 가능성 판단                                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터 해저드 감소 기법:                                              │ │
│  │  ──────────────────────                                              │ │
│  │  기법              설명                    효과                       │ │
│  │  ───────────────  ──────────────────     ───────                    │ │
│  │  포워딩           하드웨어 전달             스톨 50~80% 감소            │ │

│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅳ. 실무 적용

### 데이터 해저드 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 해저드 분석                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【해저드 감지 하드웨어】                                               │
│  ────────────────────────                                            │
│  $ perf stat -e uops_executed.stall_cycles ./prog                   │
│  → 실행 스톨 사이클 측정                                               │
│                                                                     │
│  【포워딩 효과 측정】                                                   │
│  ────────────────────────                                            │
│  # 포워딩 없이 (스톨만 사용)                                           │
│  CPI: 1.5~2.0                                                        │
│                                                                     │
│  # 포워딩 있이                                                         │
│  CPI: 1.0~1.1                                                        │
│  → 30~50% 성능 향상                                                   │
│                                                                     │
│  【코드 최적화 예시】                                                   │
│  ────────────────────────                                            │
│  # Before (해저드 발생)                                               │
│  ADD R1, R2, R3                                                      │
│  SUB R4, R1, R5   ; R1 대기                                          │
│  MUL R6, R7, R8                                                      │
│                                                                     │
│  # After (코드 재배치)                                                 │
│  ADD R1, R2, R3                                                      │
│  MUL R6, R7, R8   ; 의존성 없음                                        │
│  SUB R4, R1, R5   ; R1 준비됨                                         │
│  → 스톨 없이 실행                                                      │
│                                                                     │
│  【Load-Use 해저드 최소화】                                            │
│  ──────────────────────────                                          │
│  # Before                                                            │
│  LW  R1, 0(R2)                                                       │
│  ADD R3, R1, R4   ; 1클럭 스톨                                        │
│  MUL R5, R6, R7                                                      │
│                                                                     │
│  # After                                                             │
│  LW  R1, 0(R2)                                                       │
│  MUL R5, R6, R7   ; 스톨 사이에 다른 명령어                             │
│  ADD R3, R1, R4   ; 스톨 없음                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터 해저드 최적화 팁:                                              │ │
│  │  ──────────────────────                                              │ │
│  │  1. 포워딩 하드웨어 활용 (자동)                                        │ │
│  │  2. 컴파일러 최적화 옵션 사용 (-O2, -O3)                              │ │
│  │  3. Load 후 바로 사용 피하기                                          │ │
│  │  4. 의존성 없는 명령어 배치                                            │ │
│  │  5. 루프 언롤링으로 의존성 분산                                        │ │
│  │  6. SIMD 활용 (벡터화)                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 데이터 해저드: 데이터 의존성으로 스톨 발생
• RAW: Read After Write, 가장 흔함
• WAR: Write After Read, 비순차에서 문제
• WAW: Write After Write, 비순차에서 문제
• 포워딩: EX/MEM → EX로 결과 전달, 스톨 감소
• Load-Use: LW 후 바로 사용, 1클럭 스톨 불가피
• 코드 재배치: 컴파일러로 의존성 없는 명령어 배치
```
