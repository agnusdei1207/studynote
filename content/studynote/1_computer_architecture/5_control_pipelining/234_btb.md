# 234. 분기 목적지 버퍼 (BTB)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분기 타겟 주소를 캐시하는 구조
> 2. **가치**: IF 단계에서 바로 타겟 주소 획득, 페널티 0
> 3. **융합**: BHT와 함께 분기 예측, 캐시

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

분기 목적지 버퍼(BTB, Branch Target Buffer)는 **분기 명령어의 타겟 주소를 캐시하는 구조**이다. 분기 예측 시 BTB에서 타겟 주소를 바로 찾아 IF 단계에서 분기 타겟 명령어를 인출할 수 있다.

### 💡 비유: 즐겨찾기 주소록

BTB는 **전화번호 즐겨찾기**와 같다. 자주 거는 전화번호를 저장해둔다. 필요할 때 바로 찾을 수 있다. 일일이 주소록을 뒤지지 않아도 된다.

### BTB 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                BTB 구조                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【BTB 기본 구조】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │  분기 PC   │  타겟 PC   │  예측 비트  │  태그       │        │ │
│  │  ├─────────────────────────────────────────────────────┤        │ │
│  │  │  0x1000   │  0x2000   │  1 (T)    │  ...       │        │ │
│  │  │  0x3000   │  0x4000   │  0 (NT)   │  ...       │        │ │
│  │  │  0x5000   │  0x6000   │  1 (T)    │  ...       │        │ │
│  │  │  ...      │  ...      │  ...      │  ...       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │                    ↑                                            │ │
│  │                    │                                            │ │
│  │            분기 PC로 인덱싱                                        │ │
│  │                                                               │ │
│  │  • 분기 PC: 분기 명령어의 주소 (Key)                                  │ │
│  │  • 타겟 PC: 분기 시 이동할 주소 (Value)                               │ │
│  │  • 예측 비트: Taken/Not Taken (BHT와 공유 가능)                      │ │
│  │  • 태그: 캐시 일관성 확인용 (선택적)                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【BTB 동작】                                                           │
│  ──────────────────                                                  │
│  1. IF 단계:                                                          │
│     • 현재 PC를 BTB에 조회                                             │
│     • 적중: 타겟 PC 사용, 예측 비트 확인                                  │
│     • 미스: 순차적 인출 (PC + 4)                                        │
│                                                                     │
│  2. EX 단계:                                                          │
│     • 실제 분기 결과 계산                                               │
│     • BTB와 비교                                                       │
│     • 맞으면: 계속 실행                                                  │
│     • 틀리면: BTB 업데이트 + 파이프라인 플러시                              │
│                                                                     │
│  【BTB 크기】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU             BTB 엔트리     연관성                                        │ │
│  │  ───────────────  ───────────  ──────────────────────                   │ │
│  │  Intel Skylake    4,096       I-Cache와 통합                              │ │
│  │  AMD Zen 3        6,144       독립 BTB                                    │ │
│  │  ARM Cortex-A78   512~1024    작은 BTB                                    │ │
│  │  Apple M1         4,096       큰 BTB                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BTB vs I-Cache:                                                      │ │
│  │  ──────────────────                                                  │ │
│  │  특성         BTB             I-Cache                                   │ │
│  │  ──────────  ──────────────  ──────────────────                        │ │
│  │  저장 내용     타겟 주소         명령어                                    │ │
│  │  인덱싱       분기 PC           PC                                       │ │
│  │  크기         작음              큼                                        │ │
│  │  목적         분기 최적화        명령어 인출                                 │ │
│  │  → 현대 CPU: BTB를 I-Cache에 통합하는 추세                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. BTB 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                BTB 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【BTB와 분기 예측】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  IF 단계:                                                         │ │
│  │  ┌──────────────────────────────────────────────────────────┐   │ │
│  │  │  1. PC로 I-Cache 접근                                         │   │ │
│  │  │  2. 분기 명령어인지 확인 (또는 BTB 병렬 조회)                    │   │ │
│  │  │  3. BTB 적중 시:                                                │   │ │
│  │  │     - 예측 비트 확인 (Taken/Not Taken)                          │   │ │
│  │  │     - Taken이면 타겟 PC로 인출                                   │   │ │
│  │  │     - Not Taken이면 PC + 4로 인출                                │   │ │
│  │  │  4. BTB 미스 시:                                                │   │ │
│  │  │     - PC + 4로 인출                                             │   │ │
│  │  │     - 첫 분기 시 BTB에 엔트리 생성                                 │   │ │
│  │  └──────────────────────────────────────────────────────────┘   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【BTB와 파이프라인】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BEQ R1, R2, target   ; 분기 명령어                                  │ │
│  │                                                               │ │
│  │  BTB 적중 (Taken 예측):                                            │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    BEQ                                                       │ │
│  │   2    target  BEQ        ← BTB에서 타겟 PC 바로 인출               │ │
│  │   3          target  BEQ                                          │ │
│  │   4                  target  BEQ                                  │ │
│  │                                                               │ │
│  │  → BTB 적중 시 페널티 0클럭                                          │ │
│  │                                                               │ │
│  │  BTB 미스:                                                         │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    BEQ                                                       │ │
│  │   2    BEQ+4  BEQ       ← BTB 미스 → 순차 인출                      │ │
│  │   3    target  BEQ+4    ← EX에서 분기 결정 → 플러시                   │ │
│  │   4          target    BEQ+4                                      │ │
│  │                                                               │ │
│  │  → BTB 미스 시 페널티 1~2클럭                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【BTB 업데이트】                                                        │
│  ──────────────────                                                  │
│  • 분기 실행 완료 시: BTB 엔트리 업데이트                                  │
│  • 새 분기: BTB에 엔트리 생성                                            │
│  • 교체: LRU 또는 랜덤 교체                                              │
│  • 플러시: 컨텍스트 스위치, I-Cache 플러시                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BTB 성능:                                                            │ │
│  │  ──────────────────                                                  │ │
│  │  측정 항목          값           설명                                   │ │
│  │  ───────────────  ──────────  ──────────────────                  │ │
│  │  적중률            95~99%       대부분의 분기 명령어                   │ │
│  │  평균 페널티        0.1~0.3클럭   BTB 미스 시만 페널티                   │ │
│  │  CPI 기여          +0.01~0.05  분기 예측과 함께                       │ │
│  │  → BTB로 분기 페널티 90% 감소                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. BTB × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                BTB × 타 과목 융합                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 파이프라인: BTB로 IF 단계 분기 처리                                   │
│  • 캐시: BTB는 작은 캐시의 일종                                          │
│  • 성능: BTB 적중률이 CPI에 미치는 영향                                   │
│  • 슈퍼스칼라: 다중 BTB 필요                                             │
│                                                                     │
│  【캐시 메모리】                                                        │
│  • 구조: 직접 매핑, 연관 매핑, 세트 연관                                  │
│  • 교체: LRU, Random, FIFO                                            │
│  • 일관성: 프로세스 전환 시 BTB 플러시                                    │
│                                                                     │
│  【운영체제】                                                          │
│  • 컨텍스트 스위치: BTB 플러시 필요                                       │
│  • 스케줄링: BTB 상태 보존 고려                                          │
│  • 성능: BTB 적중률 모니터링                                             │
│                                                                     │
│  【보안】                                                              │
│  • Spectre: BTB 상태 노출                                              │
│  • 사이드 채널: BTB 타이밍 공격                                          │
│  • 완화: BTB 플러시, 격리                                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BTB 구현 트레이드오프:                                                  │ │
│  │  ──────────────────────                                              │ │
│  │  크기      적중률    전력     면적     적합                                │ │
│  │  ──────   ──────   ──────  ──────  ──────                              │ │
│  │  64       80~85%   매우 낮음  매우 작음  저전력                              │ │
│  │  256      85~90%   낮음     작음      임베디드                           │ │
│  │  1024     90~95%   중간     중간      데스크탑                           │ │
│  │  4096     95~98%   높음     큼        고성능                             │ │
│  │  16384    97~99%   매우 높음  매우 큼   서버/워크스테이션                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### BTB 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                BTB 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【BTB 적중률 측정】                                                     │
│  ──────────────────                                                  │
│  $ perf stat -e branch-instructions,branch-misses ./prog           │
│  branch-instructions: 1,000,000,000                                 │
│  branch-misses: 15,000,000                                          │
│  → 98.5% 적중률 (1.5% 미스율)                                          │
│                                                                     │
│  【BTB 미스 페널티】                                                     │
│  ──────────────────                                                  │
│  $ perf stat -e cycles,branch-misses ./prog                        │
│  cycles: 10,000,000,000                                             │
│  branch-misses: 15,000,000                                          │
│  → 평균 페널티 = 10B / 15M ≈ 667 클럭 (너무 높음, 다른 요인 포함)            │
│  → 실제 BTB 페널티: 10~20클럭                                           │
│                                                                     │
│  【BTB 크기 확인】                                                       │
│  ──────────────────                                                  │
│  $ cat /sys/devices/system/cpu/cpu0/cache/index*                   │
│  → BTB는 일반 캐시와 별도                                               │
│                                                                     │
│  【BTB 친화적 코드】                                                      │
│  ──────────────────                                                  │
│  # Before (BTB 미스 높음)                                              │
│  void process(int x) {                                              │
│      if (x > threshold) { ... }  ; 함수마다 분기                       │
│  }                                                                  │
│                                                                     │
│  for (i = 0; i < n; i++) {                                          │
│      process(data[i]);  ; 매번 다른 분기 주소                            │
│  }                                                                  │
│                                                                     │
│  # After (BTB 친화적)                                                  │
│  // 인라이닝으로 분기 감소                                                │
│  for (i = 0; i < n; i++) {                                          │
│      if (data[i] > threshold) { ... }  ; 같은 분기 주소               │
│  }                                                                  │
│                                                                     │
│  【BTB 플러시】                                                           │
│  ──────────────────                                                  │
│  $ perf stat -e context-switches ./prog                            │
│  context-switches: 100,000                                          │
│  → 컨텍스트 스위치마다 BTB 플러시                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BTB 최적화 팁:                                                         │ │
│  │  ──────────────────                                                  │ │
│  │  1. 인라이닝으로 분기 감소                                             │ │
│  │  2. 루프 언롤링로 분기 패턴 단순화                                      │ │
│  │  3. 컨텍스트 스위치 감소                                               │ │
│  │  4. PGO 활용                                                         │ │
│  │  5. 분기 예측 친화적 코드                                              │ │
│  │  6. 간접 분기 최소화 (가상 함수 등)                                     │ │
│  │  7. 분기 밀도 낮은 코드 작성                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• BTB: 분기 타겟 주소를 캐시
• 구조: 분기 PC → 타겟 PC + 예측 비트
• 목적: IF 단계에서 바로 타겟 주소 획득
• 크기: 256~4096 엔트리
• 적중률: 95~99%
• 페널티: BTB 적중 시 0클럭, 미스 시 1~2클럭
• BHT와 함께: 타겟 + 방향 예측
• 현대 CPU: I-Cache와 통합 추세
```
