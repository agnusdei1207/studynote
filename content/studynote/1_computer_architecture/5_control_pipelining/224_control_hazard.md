# 224. 제어 해저드 (Control Hazard)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분기로 인해 파이프라인에 이미 인출된 명령어가 무효화
> 2. **가치**: CPI 증가의 주요 원인, 분기 예측으로 해결
> 3. **융합**: 분기 예측, 지연 분기, BTB, BHT

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

제어 해저드(Control Hazard)는 **분기 명령어로 인해 파이프라인에 이미 인출된 명령어들이 무효화되는 현상**이다. 분기 방향이 결정되기 전까지 어떤 명령어를 실행할지 알 수 없어 파이프라인이 중단된다.

### 💡 비유: 내비게이션 경로 변경

제어 해저드는 **내비게이션 경로 변경**과 같다. 고속도로에서 분기점(분기)이 나오면 왼쪽/오른쪽 선택지가 있다. 미리 예측해서 갔는데 틀리면 되돌아와야 한다(플러시).

### 제어 해저드 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                제어 해저드 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【발생 상황】                                                         │
│  ──────────────────                                                  │
│  • 조건부 분기 (BEQ, BNE, BLT, ...)                                   │
│  • 무조부 분기 (J, JAL, JR, ...)                                       │
│  • 함수 호출/복귀 (CALL, RET)                                          │
│  • 인터럽트/예외                                                        │
│                                                                     │
│  【분기 문제 타이밍】                                                   │
│  ────────────────────────                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BEQ R1, R2, target   ; if (R1 == R2) goto target               │ │
│  │  ADD R3, R4, R5       ; 분기 안 했으면 실행                        │ │
│  │  SUB R6, R7, R8       ; 분기 안 했으면 실행                        │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    BEQ                                                       │ │
│  │   2    ADD   BEQ        ← BEQ ID: R1, R2 읽기                     │ │
│  │   3    SUB   ADD   BEQ   ← BEQ EX: R1 == R2 비교                  │ │
│  │   4    ???   SUB   ADD   BEQ    ← EX에서 분기 결정!                │ │
│  │         ↑                                                        │ │
│  │   이미 인출된 ADD, SUB 무효 → 파이프라인 플러시                       │ │
│  │   → target에서 다시 인출 필요                                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【분기 페널티】                                                        │
│  ──────────────────                                                  │
│  • 5단계 파이프라인: 분기가 EX에서 결정 → 2클럭 페널티                    │
│  • 분기가 ID에서 결정: 1클럭 페널티                                     │
│  • 분기가 MEM에서 결정: 3클럭 페널티                                    │
│                                                                     │
│  【해결 방안】                                                         │
│  ──────────────────                                                  │
│  1. 스톨 (Stall): 분기 결정까지 대기                                    │
│  2. 분기 예측 (Branch Prediction): 분기 여부 예측                       │
│  3. 지연 분기 (Delayed Branch): 분기 후 명령어 항상 실행                 │
│  4. 분기 타겟 버퍼 (BTB): 분기 타겟 캐시                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  제어 해저드 비용:                                                    │ │
│  │  ──────────────────                                                  │ │
│  │  방식              페널티        CPI 영향                            │ │
│  │  ───────────────  ────────     ──────────                           │ │
│  │  스톨만 사용        2~3클럭      0.2~0.3 (분기 20%)                   │ │
│  │  정적 분기 예측     1~2클럭      0.1~0.2                             │ │
│  │  동적 분기 예측     0.1클럭      0.01~0.05                           │ │
│  │  완벽한 예측        0클럭        0                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 제어 해저드 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                제어 해저드 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스톨로 해결 (비효율적)】                                             │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BEQ R1, R2, target                                              │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    BEQ                                                       │ │
│  │   2    -     BEQ        ← 스톨                                    │ │
│  │   3    -     -     BEQ   ← 스톨 (EX에서 분기 결정)                  │ │
│  │   4    target  -     -     BEQ    ← 분기 타겟 인출                  │ │
│  │                                                               │ │
│  │  → 2클럭 스톨                                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【분기 예측으로 해결】                                                 │
│  ────────────────────────                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예측: 분기 안 함 (Not Taken)                                      │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    BEQ                                                       │ │
│  │   2    ADD   BEQ        ← 예측: 다음 명령어 인출                    │ │
│  │   3    SUB   ADD   BEQ   ← 계속 실행                               │ │
│  │                                                               │ │
│  │  예측 성공: 스톨 없음                                               │ │
│  │  예측 실패: ADD, SUB 플러시 → target 인출                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지연 분기 (Delayed Branch)】                                        │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BEQ R1, R2, target                                              │ │
│  │  NOP              ; 지연 슬롯 (항상 실행)                           │ │
│  │                                                               │ │
│  │  또는 유용한 명령어 배치:                                            │ │
│  │  ADD R3, R4, R5   ; 분기 전에 실행                                 │ │
│  │  BEQ R1, R2, target   ; 지연 슬롯에 ADD                           │ │
│  │  ; 여기서 target으로 분기                                          │ │
│  │                                                               │ │
│  │  지연 슬롯 명령어는 분기 여부와 관계없이 항상 실행                      │ │
│  │  → 파이프라인 플러시 불필요                                          │ │
│  │  → MIPS에서 사용                                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【분기 타겟 버퍼 (BTB)】                                               │
│  ────────────────────────                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ┌─────────────────────────────────────────────────┐             │ │
│  │  │  분기 PC   |   타겟 PC   |   예측 비트            │             │ │
│  │  ├─────────────────────────────────────────────────┤             │ │
│  │  │  0x1000   |   0x2000   |   Taken               │             │ │
│  │  │  0x3000   |   0x4000   |   Not Taken           │             │ │
│  │  │  ...      |   ...      |   ...                  │             │ │
│  │  └─────────────────────────────────────────────────┘             │ │
│  │                                                               │ │
│  │  • 분기 PC로 타겟 PC를 바로 찾음                                    │ │
│  │  • IF 단계에서 분기 예측 가능                                       │ │
│  │  → 페널티 0클럭 (BTB 적중 시)                                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분기 예측 정확도:                                                    │ │
│  │  ──────────────────                                                  │ │
│  │  • 정적 예측 (항상 Not Taken): 50~60%                               │ │
│  │  • 정적 예측 (항상 Taken): 60~70%                                    │ │
│  │  • 동적 예측 (1비트): 80~90%                                         │ │
│  │  • 동적 예측 (2비트): 90~95%                                         │ │
│  │  • 고급 예측 (신경망): 95~99%                                        │ │
│  │  → 현대 CPU: 95%+ 정확도                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 제어 해저드 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                제어 해저드 × 타 과목 융합                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 파이프라인: 제어 해저드가 CPI 증가                                   │
│  • 분기 예측: 하드웨어로 해결                                          │
│  • 슈퍼스칼라: 다중 분기 예측                                           │
│  • 비순차 실행: 분기 후 명령어 추측 실행                                 │
│                                                                     │
│  【컴파일러】                                                          │
│  • 지연 슬롯 채우기: 유용한 명령어 배치                                  │
│  • 분기 정렬: 예측 친화적 코드                                          │
│  • 프로파일 기반 최적화 (PGO): 분기 빈도 활용                            │
│  • 루프 최적화: 분기 감소                                               │
│                                                                     │
│  【운영체제】                                                          │
│  • 컨텍스트 스위치: BTB/BHT 플러시                                      │
│  • 인터럽트: 제어 해저드 유발                                           │
│  • 스케줄링: 분기 예측 상태 고려                                        │
│                                                                     │
│  【보안】                                                              │
│  • Spectre: 분기 예측 악용                                             │
│  • 추측 실행: 사이드 채널 공격                                          │
│  • 완화: 분기 예측 제한                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분기 예측 성능 영향:                                                  │ │
│  │  ──────────────────────                                              │ │
│  │  CPU Time = IC × CPI × CC                                           │ │
│  │  CPI = 1 + (분기율 × 미스율 × 페널티)                                 │ │
│  │                                                               │ │
│  │  예:                                                                │ │
│  │  • 분기율: 20%                                                      │ │
│  │  • 미스율: 5%                                                       │ │
│  │  • 페널티: 15클럭                                                   │ │
│  │  → CPI 증가 = 0.2 × 0.05 × 15 = 0.15                                │ │
│  │  → 15% 성능 저하                                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 제어 해저드 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                제어 해저드 분석                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【분기 예측 분석】                                                    │
│  ──────────────────                                                  │
│  $ perf stat -e branches,branch-misses ./prog                       │
│  branches: 1,000,000,000                                            │
│  branch-misses: 10,000,000                                          │
│  → 1% 분기 미스율 (현대 CPU)                                           │
│                                                                     │
│  【BTB 미스 분석】                                                      │
│  ──────────────────                                                  │
│  $ perf stat -e btb_misses ./prog                                   │
│  → BTB 적중률 확인                                                    │
│                                                                     │
│  【분기 예측 친화적 코드】                                              │
│  ──────────────────────────                                          │
│  # Before (예측 어려움)                                               │
│  if (random() % 2 == 0) {  ; 50% 확률                                │
│      // ...                                                          │
│  }                                                                  │
│                                                                     │
│  # After (예측 친화적)                                                 │
│  if (likely(x > 0)) {  ; 95% 확률                                    │
│      // 자주 실행되는 코드                                             │
│  }                                                                  │
│                                                                     │
│  【함수 인라이닝】                                                      │
│  ──────────────────                                                  │
│  # Before                                                            │
│  for (i = 0; i < n; i++) {                                          │
│      result += foo(i);  ; 함수 호출 = 분기                             │
│  }                                                                  │
│                                                                     │
│  # After (인라이닝)                                                    │
│  for (i = 0; i < n; i++) {                                          │
│      result += i * 2;  ; 분기 제거                                    │
│  }                                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  제어 해저드 최소화 팁:                                               │ │
│  │  ──────────────────────                                              │ │
│  │  1. 분기 예측 친화적 코드 작성                                        │ │
│  │  2. 함수 인라이닝 (분기 감소)                                        │ │
│  │  3. 루프 언롤링 (분기 감소)                                          │ │
│  │  4. PGO (프로파일 기반 최적화)                                       │ │
│  │  5. switch → 테이블 점프 (예측 가능)                                 │ │
│  │  6. likely/unlikely 매크로 활용                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 제어 해저드: 분기로 파이프라인 무효화
• 페널티: 분기 결정까지 대기 클럭 수
• 스톨: 분기까지 대기 (비효율적)
• 분기 예측: 분기 여부 예측 (효율적)
• 지연 분기: 분기 후 명령어 항상 실행
• BTB: 분기 타겟 캐시
• 현대 CPU: 95%+ 분기 예측 정확도
```
