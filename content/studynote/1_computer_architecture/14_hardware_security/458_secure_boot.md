# 458. 보안 부팅 (Secure Boot)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 부팅 과정의 코드 서명 검증으로 악성 코드 차단
> 2. **가치**: 부트킷 방지, 무결성 보장, 신뢰 체인
> 3. **융합**: UEFI, PKI, TPM, 측정 부팅

---

## Ⅰ. 개요

### 개념 정의

보안 부팅(Secure Boot)은 **UEFI 펌웨어가 부팅 시 각 단계의 코드 서명을 검증하여, 서명되지 않거나 변조된 코드의 실행을 차단하는 보안 기능**이다.

### 💡 비유: 공항 보안 검색대

보안 부팅은 **공항 보안 검색대**와 같다. 승객이 있다. (부트 코드) 검색대가 있다. (UEFI) 여권 확인. (서명 검증) 입국 허가. (인증서) 통과시킨다. (부팅) 불법 체류자 차단. (악성 코드) 안전하다. 보안 부팅이 똑같다. 서명 검증, 신뢰 체인, 악성 코드 차단.

│  │  │  펌웨어 │     │ 로더    │     │ 커널    │     │         │ │ │

│                                                                     │

│  │  특성           Secure Boot          Measured Boot            │ │

│  │  결과            부팅 허용/거부         증명 가능                 │ │

## Ⅳ. 실무 적용

### 보안 부팅 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                보안 부팅 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Secure Boot 상태 확인】                                                        │)
│  ──────────────────                                                  │
│  # Linux에서 확인                                                          │
│  $ mokutil --sb-state                                                    │
│                                                                     │
│  # UEFI 변수 확인                                                          │
│  $ efivar -l | grep -i secure                                            │
│                                                                     │
│  # dmesg로 확인                                                            │
│  $ dmesg | grep -i secure                                                │
│                                                                     │
│  【MOK (Machine Owner Key) 관리】                                                 │)
│  ──────────────────                                                  │
│  # MOK 상태                                                                │
│  $ mokutil --list-enrolled                                               │
│                                                                     │
│  # 새 키 등록                                                              │
│  $ mokutil --import public_key.der                                       │
│  # 재부팅 후 MOK 매니저에서 확인                                           │
│                                                                     │
│  【커스텀 커널 서명】                                                            │)
│  ──────────────────                                                  │
│  # 키 쌍 생성                                                               │
│  $ openssl req -new -x509 -newkey rsa:2048 \                            │
│    -keyout MOK.priv -outform DER -out MOK.der                           │
│                                                                     │
│  # 커널 서명                                                                │
│  $ sbsign --key MOK.priv --cert MOK.der \                               │
│    /boot/vmlinuz-linux                                                   │
│                                                                     │
│  【Secure Boot 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 상태 확인: mokutil로 확인                                              │
│  2. 백업: MOK 키 백업                                                     │
│  3. 복구: BIOS에서 비활성화 가능                                            │
│  4. 로그: 부팅 로그 확인                                                   │
│  5. 업데이트: 커널 업데이트 시 재서명                                        │
│  6. 문서화: 커스텀 설정 기록                                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 보안 부팅: UEFI 서명 검증으로 부트킷 방지
• PK: Platform Key
• KEK: Key Exchange Key
• db: Authorized Signature Database
• dbx: Forbidden Signature Database
• shim: Linux Secure Boot 로더
• MOK: Machine Owner Key
• 장점: 부트킷 방지, 신뢰 체인
• 단점: 벤더 종속, 커스텀 제한
• 조합: Measured Boot와 함께 사용
```
