# CMP (Chip Multi-Processor)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CMP(칩 다중 프로세서)는 독립적인 하드웨어 패키지로 존재하던 여러 개의 CPU 칩들을, **단 하나(Single)의 반도체 실리콘 다이(Die) 조각 안에 몽땅 압축해서 박아 넣은 물리적 통합 아키텍처**다. (멀티코어의 학술적/엔지니어링 명칭)
> 2. **가치**: 칩 밖으로 나가서 메인보드 버스를 탈 필요 없이, 수 나노미터 거리에 있는 칩 내부의 고속 버스와 공유 L2/L3 캐시를 통해 코어 간 통신을 수행하므로 **지연 시간(Latency)이 0에 수렴하며 성능이 폭발적으로 향상**된다.
> 3. **융합**: 단일 칩의 남는 잉여 트랜지스터를 가장 효율적으로 소모하기 위한 반도체 공정(무어의 법칙)과 운영체제의 대칭형 다중 처리(SMP) 사상이 칩 한 조각 안에서 완벽히 융합된 형태다.

---

## Ⅰ. CMP의 개요

### 1. 정의
- 하나 이상의 온전한 프로세서 코어(ALU, 파이프라인, 레지스터, L1 캐시 포함)를 단일 반도체 집적회로(IC) 다이(Die) 위에 나란히 복제하여 얹어 놓은 아키텍처 구조다. 대중적으로는 '멀티코어 프로세서'라 불린다.

### 2. 등장 배경: '남아도는 트랜지스터를 어찌할꼬'
- 무어의 법칙에 따라 칩에 쑤셔 넣을 수 있는 트랜지스터 개수가 1억 개를 넘어갔다. 아키텍트들은 고민했다. "이걸로 명령어 수준 병렬성(ILP)을 높이려고 디코더를 엄청 뚱뚱하게 만들자니 전기를 너무 먹는다(효율 박살). 차라리 작고 효율 좋은 CPU 코어를 **복사-붙여넣기(Ctrl+C, Ctrl+V)**해서 똑같은 칩 2개, 4개를 1개 칩 안에 꽉 채워 넣는 게 코딩하기 편하지 않을까?" 이 위대한 '복붙' 철학이 CMP의 탄생이다.

### 3. 💡 비유: '분점 4개 vs 거대 복합 쇼핑몰 1개'
- 과거 다중 프로세서가 강남, 홍대, 신촌에 **'분점 건물을 4개 세운 것(소켓 4개)'**이라면, CMP는 코엑스라는 **'하나의 거대한 복합 쇼핑몰 1층부터 4층에 각 지점을 몽땅 쑤셔 넣은 것(칩 1개)'**과 같다.
- 지점장(코어)들끼리 긴급 회의를 할 때, 옛날엔 차를 타고 강남에서 홍대까지(메인보드 외부 버스) 이동해야 해서 하루가 걸렸다. 하지만 쇼핑몰 안에서는 엘리베이터(내부 링 버스)만 타면 10초 만에 만나서 결재 도장을 찍고 헤어질 수 있다.

- **📢 섹션 요약 비유**: 넓은 대지에 집을 여러 채 지어 살던 가족들이, 통신 효율을 위해 아예 초고층 아파트 한 동 안으로 모두 이사 들어와 버린 완벽한 집적 콤플렉스입니다.

## Ⅱ. 메커니즘 및 칩 내부 결합 구조 (Deep Dive)

CMP의 최대 무기는 '내부에서 모든 걸 끝낸다'는 점이다.

### 1. CMP 내부 통신의 압도적 이점
- 과거 CPU 2개가 통신하려면 칩을 빠져나와 $\to$ 메인보드의 구리선(FSB 등)을 타고 $\to$ 노스브리지 칩셋을 거쳐 $\to$ 다시 2번 CPU 칩으로 들어가야 했다. (속도 수백 나노초, 전기 낭비 극심).
- CMP 구조에서는 코어 1과 코어 2가 칩 내부에 달린 공유 L2 또는 L3 캐시에 데이터를 올려두기만 하면 0.1나노초 만에 즉시 공유가 끝난다. **거리가 수 센티미터에서 수 나노미터로 1천 배 이상 짧아졌기 때문**이다.

### 2. CMP의 전형적인 내부 아키텍처 (ASCII)
```text
    ┌──────────────────────── 단일 실리콘 다이 (CMP 칩) ────────────────────────┐
    │                                                                       │
    │  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐           │
    │  │ Core 0       │     │ Core 1       │     │ Core 2       │           │
    │  │ [ L1 Cache ] │     │ [ L1 Cache ] │     │ [ L1 Cache ] │           │
    │  └──────┬───────┘     └──────┬───────┘     └──────┬───────┘           │
    │         │ (수 밀리미터의 극단적으로 짧은 내부 인터커넥트)                 │
    │  ═══════▼════════════════════▼════════════════════▼════════════       │
    │                                                                       │
    │                  [ 거대한 공유 L2 / L3 Cache ]                        │
    └───────────────────────────────────────────────────────────────────────┘
                                   │ (외부 핀)
                                   ▼
                            [ Main Memory (DRAM) ]
```

### 3. 심층 분석: 코어 간 간섭 (Inter-Core Interference)
- 칩 안에 다 우겨넣어서 좋긴 한데, L3 캐시와 메모리 컨트롤러를 코어 4개가 공유하다 보니 부작용도 생겼다. 코어 0번이 미친 듯이 유튜브 4K 영상을 캐시로 퍼오면, 얌전히 워드를 치던 코어 1번이 쓸 캐시 공간이 모자라 버벅거리는 **캐시 쟁탈전(공유 자원 경합)**이 발생한다. 아키텍트들은 이를 막기 위해 캐시를 코어별로 정밀하게 쪼개주는(Cache Partitioning) 복잡한 QoS(Quality of Service) 스케줄링 로직을 하드웨어에 추가해야만 했다.

- **📢 섹션 요약 비유**: 한 지붕 아래 4형제가 방은 따로 쓰지만 냉장고(L3 캐시)는 1개라서, 큰형이 냉장고에 수박을 다 채워버리면 막내는 우유 하나 넣을 자리가 없어 굶주리는 홈 셰어링의 비극입니다.

## Ⅲ. CMP와 SMT(하이퍼스레딩), SMP의 명확한 차이

비슷해 보이는 세 가지 용어를 완벽하게 분리하자.

| 용어 (약어) | 풀네임 (정의) | 실체 (물리적 vs 논리적) | 비유 (레스토랑) |
|:---:|:---|:---|:---|
| **SMP** | Symmetric Multiprocessing<br>(대칭형 다중 프로세서) | OS가 모든 코어를 공평하게 다루는 **시스템 전체의 논리적 사상(철학)**. | "요리사 4명이 주방을 평등하게 공유하자!"는 식당 운영 철학. |
| **CMP** | Chip Multi-Processor<br>(칩 다중 프로세서, 멀티코어) | 1개 칩 안에 진짜 물리적 코어를 여러 개 박은 **구체적인 하드웨어 설계 형태**. | 1명 서는 조리대에 무리해서 4명 서는 조리대로 **물리적 개조**. |
| **SMT** | Simultaneous Multithreading<br>(동시 멀티스레딩, 하이퍼스레딩) | 코어는 1개인데, 남는 부품을 돌려써서 OS에게 코어가 2개인 척 속이는 **가상 분신술**. | 요리사 1명이 양손을 따로 움직여 2명인 척 **환상**을 보여줌. |

- **📢 핵심 정리**: 현대 우리가 쓰는 i7, 라이젠 CPU는 **CMP(칩 1개에 코어 8개 물리적 탑재)이자, SMT(1코어 2스레드 가상화)를 지원하고, SMP(OS가 공평하게 다룸) 방식으로 굴러가는 기계**다.

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: 암달의 법칙을 깨는 멀티코어의 역설
- **상황**: 2코어(듀얼)에서 8코어(옥타)로 넘어가는 CMP 설계.
- **판단**: 칩 안에 코어만 무식하게 늘려봐야 발열 때문에 기본 클럭 속도(GHz)가 뚝뚝 떨어진다(클럭 다운). 그래서 코딩을 한 줄짜리 순차 실행(Single-thread)으로 짠 게임을 돌리면, 1코어의 속도가 오히려 구형 CPU보다 느려져 게임 프레임이 박살 나는 역설이 터졌다. 기술사는 맹목적인 CMP 코어 수 경쟁(Core Count War)에 속지 않고, 단일 스레드 성능(IPC)과 멀티코어 성능의 최적 타협점을 찾는 6~8코어 설계(스위트 스팟)에 집중해야 함을 깨달았다. 코어 64개가 무조건 8코어보다 좋은 건 슈퍼컴퓨터에서나 통하는 얘기다.

### 2. 최신 실무: 칩렛(Chiplet) - 거대 CMP의 죽음
- **인사이트**: 칩(Die) 하나에 코어 8개를 그리는 건 쉽다. 그런데 64개를 한 칩(Monolithic Die)에 그리려니까 반도체 웨이퍼에 불량이 하나만 나도 64코어 칩 전체를 쓰레기통에 버려야 했다(수율 박살). AMD는 천재적인 발상을 한다. "거대 CMP 하나 굽지 말고, **8코어짜리 싼 CMP를 8개 구워서 레고 블록처럼 기판(Substrate) 위에 본드로 이어 붙이자(Chiplet)!**" 
- 이것이 인텔의 독재를 끝낸 AMD Zen 아키텍처의 핵심이다. 거대한 단일 CMP 시대는 끝나고, 조각난 칩(Chiplet)들을 묶어 가상의 거대 CMP처럼 속이는 21세기 최첨단 패키징 시대가 도래했다.

- **📢 섹션 요약 비유**: 64조각 피자를 한 번에 구우려다 한쪽만 타도 피자 전체를 버리던 바보짓을 멈추고, 8조각짜리 작은 피자 8판을 따로 구워서 상자 하나에 예쁘게 포장해 파는 원가 절감의 기적입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **메인보드 공간 혁명**: 여러 개의 CPU 소켓과 거대한 쿨러가 난립하던 메인보드를 단 하나의 구멍(소켓)으로 정리하여 데스크톱과 스마트폰의 극단적 소형화를 이끌어냄.
- **IPC 장벽 우회**: 명령어 여러 개를 겹치기(수퍼스칼라) 위해 회로를 아무리 꼬아도 오르지 않던 성능을, 그냥 공장 자체를 2개, 4개 복사하는 단순한 아이디어로 수백 배 상승시킴.

### 2. 미래 전망
최초의 IBM POWER4 이후 모든 프로세서는 CMP로 만들어졌다. 미래에는 이 CMP라는 네모난 칩 안에 단순히 CPU 코어만 복사해 넣지 않는다. 한구석엔 GPU 코어, 한구석엔 NPU(인공지능) 코어, 한구석엔 모뎀 칩을 구겨 넣는 **SoC(System on Chip)**이자 **이기종 멀티코어(Heterogeneous CMP)**로 진화하며, 반도체 한 조각이 사실상 지구상에서 가장 완벽하게 돌아가는 초소형 우주로 변모하고 있다. 

- **📢 섹션 요약 비유**: 과거에는 수학 천재 4명을 모아놓은 스터디룸(단순 CMP)이었지만, 이제는 수학 천재, 미술 천재, 체육 천재를 한 방에 몰아넣고 세상의 모든 문제를 해결하는 어벤져스 작전 회의실(이기종 CMP)로 진화했습니다.

1. CMP(칩 다중 프로세서)는 컴퓨터의 뇌(CPU)를 만들 때, **'커다란 피자 한 판(칩) 위에 토핑구역(코어)을 여러 개로 쪼개 놓은 것'**이에요.
2. 옛날엔 1인용 피자를 4판 시켜서 식탁에 놔야 했지만, 이제는 초대형 피자 1판을 4조각으로 나눠서 4가지 맛(코어 4개)을 한 번에 즐길 수 있죠.
3. 식탁(메인보드)도 안 어질러지고, 4가지 맛이 찰싹 붙어있어서 한입에 여러 맛을 빨리빨리 맛볼 수 있는 최고의 발명품이랍니다!
