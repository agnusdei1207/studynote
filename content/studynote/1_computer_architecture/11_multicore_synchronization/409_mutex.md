# 409. 뮤텍스 (Mutex)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 상호 배제를 위한 락킹 메커니즘, 대기 시 스레드 블록
> 2. **가치**: 긴 임계 구역 보호, CPU 효율
> 3. **융합**: 스핀락, 세마포어, 소유권, 우선순위

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

뮤텍스(Mutex, Mutual Exclusion)는 **한 번에 하나의 스레드만 임계 구역에 진입할 수 있도록 보장하는 동기화 객체로, 락 획득 실패 시 스레드를 블록 상태로 만들어 CPU를 양보한다.**

### 💡 비유: 열쇠가 하나뿐인 방

뮤텍스는 **열쇠가 하나뿐인 방**과 같다. 방에 들어가려면 열쇠가 필요하다. 열쇠는 하나. 누가 가지고 있다. 다른 사람은 대기실에서. 열쇠 돌려주면 다음 사람. 대기 중엔 다른 일. 뮤텍스가 똑같다. 열쇠=락, 대기실=블록, 돌려주기=언락.

### 뮤텍스 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                뮤텍스 개요                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【뮤텍스 동작】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────┐   lock()   ┌─────────┐   unlock()  ┌─────────┐  │ │
│  │  │스레드 A │ ─────────► │임계 구역 │ ─────────► │스레드 A │  │ │
│  │  │(실행)   │            │ (락 보유) │            │(락 해제)│  │ │
│  │  └─────────┘            └─────────┘            └─────────┘  │ │
│  │                                                               │ │
│  │  ┌─────────┐   lock()   ┌─────────┐                          │ │
│  │  │스레드 B │ ─────────► │ 대기 큐  │ (블록 상태)               │ │
│  │  │(실행)   │   실패     │         │                          │ │
│  │  └─────────┘            └─────────┘                          │ │
│  │       ▲                      │                               │ │
│  │       │      스레드 A가      │                               │ │
│  │       │      unlock()       ▼                               │ │
│  │       │               ┌─────────┐                            │ │
│  │       └────────────── │스레드 B │ (깨어남)                    │ │
│  │                       │(실행)   │                            │ │
│  │                       └─────────┘                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【뮤텍스 vs 스핀락】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  특성           뮤텍스              스핀락                        │ │
│  │  ────           ─────              ─────                        │ │
│  │  대기 방식      블록 (CPU 양보)      busy-wait                  │ │
│  │  컨텍스트 전환  있음               없음                          │ │
│  │  적합한 구간    긴 임계 구역        짧은 임계 구역                │ │
│  │  오버헤드       높음               낮음                          │ │
│  │  스케줄링       OS 관여            사용자 공간                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 뮤텍스 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                뮤텍스 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【뮤텍스 특성】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 소유권 (Ownership):                                                   │ │
│  │     • 락을 획득한 스레드만 해제 가능                                    │ │
│  │     • 다른 스레드가 unlock 시 에러                                      │ │
│  │                                                               │ │
│  │  2. 재귀적 뮤텍스 (Recursive Mutex):                                      │ │
│  │     • 같은 스레드가 여러 번 lock 가능                                   │ │
│  │     • lock 횟수만큼 unlock 필요                                         │ │
│  │                                                               │ │
│  │  3. 우선순위 상속 (Priority Inheritance):                                  │ │
│  │     • 우선순위 역전 방지                                                │ │
│  │     • 낮은 우선순위 스레드가 높은 우선순위 상속                          │ │
│  │                                                               │ │
│  │  4. 적응형 뮤텍스 (Adaptive Mutex):                                       │ │
│  │     • 짧은 대기는 스핀, 긴 대기는 블록                                   │ │
│  │     • 하이브리드 방식                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상태 전이】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │           lock() 성공                                         │ │
│  │  ┌───────────┐ ──────────► ┌───────────┐                      │ │
│  │  │  해제     │             │  잠김     │                      │ │
│  │  │ (Unlocked)│             │ (Locked)  │                      │ │
│  │  └───────────┘ ◄────────── └───────────┘                      │ │
│  │           unlock()          │                                │ │
│  │                             │                                │ │
│  │                             ▼                                │ │
│  │                       ┌───────────┐                          │ │
│  │                       │  대기 큐  │                          │ │
│  │                       │ (Waiters) │                          │ │
│  │                       └───────────┘                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • CPU 효율: 대기 중 CPU 양보                                         │ │
│  │  • 긴 구역: 오래 걸리는 작업에 적합                                    │ │
│  │  • 스케줄링: OS가 공정하게 관리                                        │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 오버헤드: 컨텍스트 전환 비용                                        │ │
│  │  • 교착상태: 잘못된 사용 시                                            │ │
│  │  • 우선순위 역전: 가능                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 뮤텍스 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                뮤텍스 분석                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ mutex 사용】                                                              │)
│  ──────────────────                                                  │
│  #include <mutex>                                                   │
│                                                                     │
│  std::mutex mtx;                                                    │
│                                                                     │
│  // 방법 1: 직접 lock/unlock                                         │
│  mtx.lock();                                                        │
│  // 임계 구역                                                         │
│  mtx.unlock();                                                      │
│                                                                     │
│  // 방법 2: RAII (권장)                                               │
│  {                                                                  │
│      std::lock_guard<std::mutex> lock(mtx);                         │
│      // 임계 구역                                                     │
│  }  // 자동 unlock                                                   │
│                                                                     │
│  // 방법 3: unique_lock (더 유연)                                     │
│  std::unique_lock<std::mutex> lock(mtx);                            │
│  // lock.unlock() 가능                                               │
│                                                                     │
│  【pthread mutex】                                                               │)
│  ──────────────────                                                  │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│                                                                     │
│  pthread_mutex_lock(&mutex);                                        │
│  // 임계 구역                                                         │
│  pthread_mutex_unlock(&mutex);                                      │
│                                                                     │
│  【성능 분석】                                                                   │)
│  ──────────────────                                                  │
│  $ perf lock record ./program                                      │
│  $ perf lock report                                                 │
│                                                                     │
│  【뮤텍스 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. RAII: lock_guard 사용                                            │
│  2. 임계 구역: 최소화                                                 │
│  3. 경합: 분할, 세분화                                                │
│  4. 교착상태: lock 순서, try_lock                                    │
│  5. 측정: 대기 시간                                                   │
│  6. 대안: RCU, 읽기-쓰기 락                                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 뮤텍스: 상호 배제 동기화 객체
• 특성: 소유권, 블록 대기
• 장점: CPU 효율, 긴 구역
• 단점: 컨텍스트 전환 오버헤드
• vs 스핀락: 김=뮤텍스, 짧음=스핀
• 구현: futex, adaptive
• C++: std::mutex, lock_guard
• 교착상태: 주의 필요
• RAII: 자동 unlock
• 최적화: 임계 구역 최소화
```
