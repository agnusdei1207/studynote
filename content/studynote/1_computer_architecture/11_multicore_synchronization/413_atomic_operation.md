# 413. 원자적 연산 (Atomic Operation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 중단 없이 한 번에 완료되는 연산
> 2. **가치**: 경쟁 조건 없는 안전한 동시성
> 3. **융합**: CAS, 원자적 증가, 메모리 순서

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

원자적 연산(Atomic Operation)은 **실행 도중 인터럽트되거나 다른 연산과 섞이지 않고, 완전히 실행되거나 전혀 실행되지 않는 것처럼 보이는 연산**이다. 멀티스레드 환경에서 안전한 데이터 접근을 보장한다.

### 💡 비유: 자판기 버튼

원자적 연산은 **자판기 버튼**과 같다. 버튼을 누른다. 누르는 도중에 멈출 수 없다. 다 누르거나 안 누르거나. 중간 상태가 없다. 다른 사람이 끼어들 수 없다. 원자적 연산이 똑같다. 시작하면 끝까지, 중단 없음, 원자성.

### 원자적 연산 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                원자적 연산 개요                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【원자성 보장】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  비원자적:                                                                  │ │
│  │  ┌──────────────────────────────────────────────┐              │ │
│  │  │  count++                                     │              │ │
│  │  │  = read count → increment → write count      │              │ │
│  │  │        ↑              ↑              ↑        │              │ │
│  │  │      중단        중단           중단          │              │ │
│  │  │    가능        가능           가능            │              │ │
│  │  └──────────────────────────────────────────────┘              │ │
│  │                                                               │ │
│  │  원자적:                                                                    │ │
│  │  ┌──────────────────────────────────────────────┐              │ │
│  │  │  atomic_increment(&count)                    │              │ │
│  │  │  = 한 번에 증가 (중단 불가)                   │              │ │
│  │  └──────────────────────────────────────────────┘              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 원자적 연산】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Load: 원자적 읽기                                                    │ │
│  │  2. Store: 원자적 쓰기                                                   │ │
│  │  3. Exchange: 원자적 교환                                                │ │
│  │  4. CAS: 비교 후 교환                                                    │ │
│  │  5. Fetch-and-Add: 읽고 더하기                                            │ │
│  │  6. Test-and-Set: 테스트 후 설정                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 순서】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  relaxed: 순서 보장 없음                                                 │ │
│  │  acquire: 이후 연산이 위로 올라가지 않음                                  │ │
│  │  release: 이전 연산이 아래로 내려가지 않음                                │ │
│  │  acq_rel: acquire + release                                             │ │
│  │  seq_cst: 순차 일관성 (기본값)                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 원자적 연산 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                원자적 연산 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ atomic 연산】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  #include <atomic>                                                        │ │
│  │                                                               │ │
│  │  std::atomic<int> counter{0};                                             │ │
│  │                                                               │ │
│  │  // 원자적 읽기                                                            │ │
│  │  int val = counter.load(std::memory_order_relaxed);                       │ │
│  │                                                               │ │
│  │  // 원자적 쓰기                                                            │ │
│  │  counter.store(10, std::memory_order_release);                            │ │
│  │                                                               │ │
│  │  // 원자적 증가                                                            │ │
│  │  counter.fetch_add(1);  // 이전 값 반환                                    │ │
│  │  counter++;  // 연산자 오버로딩                                            │ │
│  │                                                               │ │
│  │  // 원자적 교환                                                            │ │
│  │  int old = counter.exchange(5);                                           │ │
│  │                                                               │ │
│  │  // CAS                                                                   │ │
│  │  int expected = 5;                                                        │ │
│  │  bool success = counter.compare_exchange_strong(expected, 10);            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【원자적 포인터】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  std::atomic<Node*> head;                                                 │ │
│  │                                                               │ │
│  │  Node* old_head = head.load();                                            │ │
│  │  new_node->next = old_head;                                               │ │
│  │  while (!head.compare_exchange_weak(old_head, new_node));                 │ │
│  │  // lock-free 스택 push                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 락 없음: 교착상태 없음                                               │ │
│  │  • 효율: 낮은 오버헤드                                                 │ │
│  │  • 확장성: 경합 시에도                                                 │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 단일 변수만: 여러 변수은 불가                                       │ │
│  │  • 복잡성: 메모리 순서 이해 필요                                        │ │
│  │  • ABA: CAS의 문제                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 원자적 연산 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                원자적 연산 분석                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【참조 카운팅】                                                                │)
│  ──────────────────                                                  │
│  class SharedPtr {                                                  │
│      std::atomic<int>* ref_count;                                   │
│  public:                                                            │
│      void add_ref() { ref_count->fetch_add(1, relaxed); }           │
│      void release() {                                               │
│          if (ref_count->fetch_sub(1, acq_rel) == 1) {               │
│              delete ref_count;                                      │
│          }                                                          │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【성능 측정】                                                                   │)
│  ──────────────────                                                  │
│  $ perf stat -e instructions,cycles ./atomic_test                   │
│                                                                     │
│  【어셈블리 확인】                                                              │)
│  ──────────────────                                                  │
│  $ objdump -d program | grep lock                                   │
│  lock xadd %rax,(%rdi)    # fetch_add                               │
│                                                                     │
│  【원자적 연산 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 순서 선택: 필요한만큼만 (relaxed 우선)                             │
│  2. 경합 감소: 로컬 복사 후 원자적                                      │
│  3. 백오프: CAS 실패 시                                               │
│  4. 타입: atomic_flag 가장 가벼움                                     │
│  │  5. 측정: 경합률 모니터링                                            │
│  6. 대안: 높은 경합은 락                                               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 원자적 연산: 중단 없는 연산
• 종류: load, store, CAS, fetch-add
• 메모리 순서: relaxed ~ seq_cst
• 장점: 락 없음, 교착상태 없음
• 단점: 단일 변수만
• C++: std::atomic<T>
• 하드웨어: LOCK, CAS
• 활용: 카운터, 플래그, 포인터
• 최적화: 메모리 순서 선택
• 성능: 낮은 경합에 유리
```
