# 416. 메모리 모델 (Memory Model)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로그램의 메모리 연산 순서와 가시성을 정의하는 규칙
> 2. **가치**: 하드웨어 독립적 동시성, 이식성
> 3. **융합**: SC, TSO, ARM, C++ 메모리 모델

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

메모리 모델(Memory Model)은 **프로그램의 메모리 읽기/쓰기 연산이 어떻게 하드웨어에 의해 실행되고 다른 스레드에 어떻게 관찰되는지 정의하는 규칙 집합**이다. 언어 메모리 모델과 하드웨어 메모리 모델로 구분된다.

### 💡 비유: 교통 규칙

메모리 모델은 **교통 규칙**과 같다. 도로가 있다. 차들이 다닌다. 규칙이 있다. "빨간 불에 멈춤" "오른쪽 우선" 규칙을 알면 안전. 모두가 같은 규칙. 메모리 모델이 똑같다. 연산 순서, 가시성 규칙, 모두가 합의.

│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────┐          │ │
│  │  │          프로그램 (고수준 언어)                     │          │ │

## Ⅱ. 메모리 모델 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 모델 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ 메모리 모델】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  memory_order 옵션:                                                        │ │
│  │                                                               │ │
│  │  1. relaxed:                                                                   │ │
│  │     • 순서 보장 없음                                                 │ │
│  │     • 원자성만 보장                                                   │ │
│  │     • 카운터 등                                                       │ │
│  │                                                               │ │
│  │  2. acquire:                                                                   │ │
│  │     • 이후 읽기/쓰기가 위로 올라가지 않음                                │ │
│  │     • 락 획득                                                        │ │
│  │                                                               │ │
│  │  3. release:                                                                   │ │
│  │     • 이전 읽기/쓰기가 아래로 내려가지 않음                              │ │
│  │     • 락 해제                                                        │ │
│  │                                                               │ │
│  │  4. acq_rel:                                                                   │ │
│  │     • acquire + release                                               │ │
│  │     • RMW 연산                                                       │ │
│  │                                                               │ │
│  │  5. seq_cst (기본값):                                                          │ │
│  │     • 순차 일관성                                                      │ │
│  │     • 전역 순서                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【하드웨어 메모리 모델】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  x86 (TSO):                                                                │ │
│  │  • Store-Load 재배치만 허용                                             │ │
│  │  • 스토어 버퍼 존재                                                     │ │
│  │  • 상대적으로 강함                                                      │ │
│  │                                                               │ │
│  │  ARM (Weak):                                                               │ │
│  │  • 모든 재배치 허용                                                     │ │
│  │  • 명시적 배리어 필요                                                   │ │
│  │  • 약함                                                                 │ │
│  │                                                               │ │
│  │  → C++ 메모리 모델은 모든 하드웨어에서 동일 의미 보장                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  강한 모델 (SC, TSO):                                                      │ │
│  │  • 장점: 이해 쉬움, 디버깅 쉬움                                         │ │
│  │  • 단점: 성능 제한                                                      │ │
│  │                                                               │ │
│  │  약한 모델 (ARM, release/acquire):                                          │ │
│  │  • 장점: 성능 최적화                                                    │ │
│  │  • 단점: 이해 어려움, 버그 위험                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 메모리 모델 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 모델 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ 메모리 순서 사용】                                                          │)
│  ──────────────────                                                  │
│  // 단순 카운터 (relaxed)                                             │
│  counter.fetch_add(1, std::memory_order_relaxed);                    │
│                                                                     │
│  // 플래그 전달 (release/acquire)                                      │
│  // 생산자                                                            │
│  data = 42;                                                          │
│  ready.store(true, std::memory_order_release);                       │
│                                                                     │
│  // 소비자                                                            │
│  while (!ready.load(std::memory_order_acquire));                     │
│  use(data);  // 42 보장                                               │
│                                                                     │
│  // 복잡한 동기화 (seq_cst)                                            │
│  std::atomic<bool> flag(false);                                      │
│  flag.store(true);  // 기본 seq_cst                                   │
│                                                                     │
│  【TSan 사용】                                                                   │)
│  ──────────────────                                                  │
│  $ clang++ -fsanitize=thread program.cpp                             │
│  $ ./a.out                                                           │
│  # 데이터 경쟁 탐지                                                    │
│                                                                     │
│  【아키텍처 확인】                                                              │)
│  ──────────────────                                                  │
│  $ uname -m                                                          │
│  x86_64  # TSO                                                       │
│  aarch64 # Weak                                                      │
│                                                                     │
│  【메모리 모델 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 기본: seq_cst로 시작                                              │
│  2. 최적화: 필요시 relaxed/acquire/release                            │
│  3. 측정: 성능 이득 확인                                               │
│  4. 검증: TSan으로 테스트                                              │
│  5. 문서화: 순서 의도 명시                                             │
│  6. 리뷰: 동시성 코드 집중                                             │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 메모리 모델: 연산 순서와 가시성 규칙
• 언어 모델: C++, Java, Rust 등
• 하드웨어 모델: x86 TSO, ARM Weak
• C++: relaxed ~ seq_cst
• 강한 모델: 이해 쉬움, 성능 제한
• 약한 모델: 성능 좋음, 복잡
• 이식성: 언어 모델로 하드웨어 독립
• 검증: TSan, 모델 체킹
• 최적화: 필요한만큼만 강한 순서
• 트렌드: ARM 증가로 약한 모델 중요
```
