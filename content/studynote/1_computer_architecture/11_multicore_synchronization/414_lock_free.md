# 414. 락 프리 자료구조 (Lock-Free Data Structure)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 없이 CAS 등으로 구현한 동시성 자료구조
> 2. **가치**: 교착상태 없음, 진행 보장, 확장성
> 3. **융합**: CAS, 원자적 포인터, ABA 문제, 메모리 재사용

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

락 프리 자료구조(Lock-Free Data Structure)는 **뮤텍스 같은 락을 사용하지 않고 CAS 등의 원자적 연산만으로 구현된 동시성 자료구조**다. 시스템 전체가 항상 진행됨을 보장한다.

### 💡 비유: 번갈아 쓰는 칠판

락 프리 자료구조는 **번갈아 쓰는 칠판**과 같다. 칠판이 있다. 여러 사람이 쓴다. 지우개는 없다. 그 대신 새 칠판. 내용을 쓴다. 다 쓰면 교체. 실패? 다시. 누가 막지 않는다. 계속 시도. 락 프리가 똑같다. 락 없음, CAS로 교체, 재시도.

### 락 프리 자료구조 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 프리 자료구조 개요                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락 기반 vs 락 프리】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  락 기반:                                                                   │ │
│  │  • 한 스레드가 락 보유 시 다른 스레드 대기                              │ │
│  │  • 락 보유 스레드가 멈추면 전체 멈춤                                   │ │
│  │  • 교착상태 가능                                                      │ │
│  │                                                               │ │
│  │  락 프리:                                                                   │ │
│  │  • 항상 어떤 스레드는 진행                                            │ │
│  │  • 한 스레드가 멈춰도 다른 스레드 진행                                 │ │
│  │  • 교착상태 없음                                                      │ │
│  │                                                               │ │
│  │  진행 보장 수준:                                                                  │ │
│  │  • Obstruction-free: 단독 실행 시 완료 보장                            │ │
│  │  • Lock-free: 항상 어떤 스레드는 진행                                  │ │
│  │  • Wait-free: 모든 스레드가 유한 시간 내 완료                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락 프리 스택 예시】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  push(new_node):                                                          │ │
│  │  ┌────────────────────────────────────────┐                    │ │
│  │  │  do {                                  │                    │ │
│  │  │      old_head = head.load()            │                    │ │
│  │  │      new_node->next = old_head         │                    │ │
│  │  │  } while (!head.compare_exchange_weak( │                    │ │
│  │  │               old_head, new_node))     │                    │ │
│  │  └────────────────────────────────────────┘                    │ │
│  │                                                               │ │
│  │  pop():                                                                    │ │
│  │  ┌────────────────────────────────────────┐                    │ │
│  │  │  do {                                  │                    │ │
│  │  │      old_head = head.load()            │                    │ │
│  │  │      if (!old_head) return null        │                    │ │
│  │  │  } while (!head.compare_exchange_weak( │                    │ │
│  │  │               old_head, old_head->next))                    │    │ │
│  │  │  return old_head                       │                    │ │
│  │  └────────────────────────────────────────┘                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 락 프리 자료구조 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 프리 자료구조 상세                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ABA 문제】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시나리오:                                                                 │ │
│  │  1. 스레드 1: old = head (A)                                              │ │
│  │  2. 스레드 2: pop A, pop B, push A (head = A again)                        │ │
│  │  3. 스레드 1: CAS(head, A, C) → 성공!                                      │ │
│  │     → A의 next는 여전히 B를 가리킴 (B는 이미 해제됨)                        │ │
│  │                                                               │ │
│  │  해결:                                                                    │ │
│  │  • 더블 워드 CAS (포인터 + 버전)                                         │ │
│  │  • Hazard Pointer                                                       │ │
│  │  • RCU (Read-Copy-Update)                                                │ │
│  │  • Epoch-based 재사용                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락 프리 큐】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Michael-Scott 큐:                                                         │ │
│  │  • dummy 노드로 시작                                                     │ │
│  │  • tail과 head 분리                                                     │ │
│  │  • enqueue: tail CAS                                                    │ │
│  │  • dequeue: head CAS                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 교착상태 없음                                                        │ │
│  │  • 진행 보장                                                            │ │
│  │  • 스케일링: 높은 경합에서도                                             │ │
│  │  • 우선순위 역전 없음                                                   │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 구현 복잡성                                                          │ │
│  │  • ABA 문제                                                             │ │
│  │  • 메모리 재사용 어려움                                                 │ │
│  │  • 높은 경합 시 재시도 비용                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 락 프리 자료구조 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 프리 자료구조 분석                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ lock-free 스택】                                                          │)
│  ──────────────────                                                  │
│  #include <atomic>                                                  │
│                                                                     │
│  template<typename T>                                               │
│  class LockFreeStack {                                              │
│      struct Node { T data; Node* next; };                           │
│      std::atomic<Node*> head;                                       │
│  public:                                                            │
│      void push(T value) {                                           │
│          Node* new_node = new Node{value, nullptr};                 │
│          new_node->next = head.load();                              │
│          while (!head.compare_exchange_weak(new_node->next, new_node)); │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【boost::lockfree 사용】                                                        │)
│  ──────────────────                                                  │
│  #include <boost/lockfree/queue.hpp>                                │
│                                                                     │
│  boost::lockfree::queue<int> q(128);                                │
│  q.push(1);                                                         │
│  int value;                                                         │
│  q.pop(value);                                                      │
│                                                                     │
│  【성능 측정】                                                                   │)
│  ──────────────────                                                  │
│  $ perf stat -e instructions,cycles ./lockfree_test                 │
│                                                                     │
│  【락 프리 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 백오프: 지수 백오프로 경합 감소                                    │
│  2. 메모리: Hazard Pointer로 안전한 재사용                            │
│  │  3. ABA: 더블 워드 CAS                                             │
│  │  4. 측정: CAS 성공률                                                │
│  │  5. 대안: 높은 경합은 파티셔닝                                       │
│  6. 검증: 모델 체킹, TSan                                             │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 락 프리: 락 없이 원자적 연산만 사용
• 진행 보장: 항상 어떤 스레드는 진행
• 장점: 교착상태 없음, 스케일링
• 단점: 구현 복잡, ABA 문제
• CAS: 비교 후 교환 기반
• ABA 해결: 더블 워드, Hazard Pointer
• 종류: 스택, 큐, 리스트, 해시
• 메모리: 안전한 재사용 필요
• 성능: 높은 경합에 유리
• 검증: TSan, 모델 체킹
```
