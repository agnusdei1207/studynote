# 411. 모니터 (Monitor)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 뮤텍스와 조건 변수를 캡슐화한 고수준 동기화 추상화
> 2. **가치**: 동기화 로직 단순화, 데이터 캡슐화
> 3. **융합**: 조건 변수, wait/signal, 상호 배제

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

모니터(Monitor)는 **공유 데이터와 이를 조작하는 메서드를 하나로 캡슐화하고, 상호 배제와 조건 동기화를 자동으로 제공하는 고수준 동기화 추상화**다. 내부적으로 뮤텍스와 조건 변수를 사용한다.

### 💡 비유: 은행 창구

모니터는 **은행 창구**와 같다. 창구가 하나다. 한 명만 들어간다. 줄을 선다. 번호표. 내 차례 기다림. 창구원이 부르면 감. 일이 끝나면 나옴. 모니터가 똑같다. 상호 배제 자동, 조건 대기, signal로 깨움.

## Ⅱ. 모니터 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                모니터 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【생산자-소비자 모니터】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  class BoundedBuffer {                                                    │ │
│  │      int buffer[N];                                                       │ │
│  │      int count = 0;                                                       │ │
│  │      condition not_full, not_empty;                                       │ │
│  │                                                               │ │
│  │      void put(int item) {                                                 │ │
│  │          while (count == N) {     // 버퍼 가득 참                         │ │
│  │              wait(not_full);      // 대기                                 │ │
│  │          }                                                                │ │
│  │          buffer[count++] = item;                                          │ │
│  │          signal(not_empty);       // 소비자 깨움                          │ │
│  │      }                                                                    │ │
│  │                                                               │ │
│  │      int get() {                                                          │ │
│  │          while (count == 0) {     // 버퍼 비어있음                         │ │
│  │              wait(not_empty);     // 대기                                 │ │
│  │          }                                                                │ │
│  │          int item = buffer[--count];                                      │ │
│  │          signal(not_full);        // 생산자 깨움                          │ │
│  │          return item;                                                     │ │
│  │      }                                                                    │ │
│  │  }                                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【while 루프 중요성】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  if 대신 while을 쓰는 이유:                                                 │ │
│  │  • 허위 깨어남 (Spurious wakeup): 조건 없이 깨어날 수 있음                │ │
│  │  • 여러 스레드 깨우기: 조건이 변했을 수 있음                               │ │
│  │  • 안전한 패턴: 항상 조건 재확인                                          │ │
│  │                                                               │ │
│  │  올바른 패턴:                                                              │ │
│  │  while (!condition) {                                                     │ │
│  │      wait(cond_var);                                                      │ │
│  │  }                                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 단순화: 동기화 로직 캡슐화                                          │ │
│  │  • 안전성: 자동 상호 배제                                               │ │
│  │  • 추상화: 고수준 인터페이스                                            │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 오버헤드: 뮤텍스+조건변수                                            │ │
│  │  • 유연성: 세밀한 제어 어려움                                           │ │
│  │  • 언어: 일부 언어만 직접 지원                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 모니터 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                모니터 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ 조건 변수】                                                              │)
│  ──────────────────                                                  │
│  #include <mutex>                                                   │
│  #include <condition_variable>                                      │
│                                                                     │
│  std::mutex mtx;                                                    │
│  std::condition_variable cv;                                        │
│  bool ready = false;                                                │
│                                                                     │
│  // 대기 스레드                                                       │
│  std::unique_lock<std::mutex> lock(mtx);                            │
│  cv.wait(lock, []{ return ready; });  // 조건 확인                   │
│  // 작업 수행                                                         │
│                                                                     │
│  // 신호 스레드                                                       │
│  {                                                                  │
│      std::lock_guard<std::mutex> lock(mtx);                         │
│      ready = true;                                                  │
│  }                                                                  │
│  cv.notify_one();  // 또는 notify_all()                              │
│                                                                     │
│  【Java synchronized】                                                           │)
│  ──────────────────                                                  │
│  class BoundedBuffer {                                              │
│      private int[] buffer = new int[N];                             │
│      private int count = 0;                                         │
│                                                                     │
│      public synchronized void put(int item) throws InterruptedException { │
│          while (count == N) wait();                                 │
│          buffer[count++] = item;                                    │
│          notifyAll();                                               │
│      }                                                              │
│                                                                     │
│      public synchronized int get() throws InterruptedException {     │
│          while (count == 0) wait();                                 │
│          int item = buffer[--count];                                │
│          notifyAll();                                               │
│          return item;                                               │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【모니터 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. while 루프: 조건 재확인                                           │
│  2. notify_all: 안전, notify_one: 효율                               │
│  3. 임계 구역: 최소화                                                 │
│  4. 타임아웃: wait_for, wait_until                                   │
│  5. 측정: 대기 시간                                                   │
│  6. 설계: 모니터 클래스로 캡슐화                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 모니터: 뮤텍스+조건 변수 캡슐화
• 상호 배제: 자동 보장
• 조건 변수: wait/signal
• while 루프: 허위 깨어남 방지
• 장점: 단순화, 안전성
• 단점: 오버헤드, 유연성 제한
• Java: synchronized
• C++: mutex + condition_variable
• 패턴: 생산자-소비자
• 설계: 스레드 안전 클래스
```
