# 410. 세마포어 (Semaphore)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 정수 카운터와 대기 큐를 가진 동기화 객체
> 2. **가치**: 자원 관리, 생산자-소비자, 상호 배제
> 3. **융합**: P/V 연산, 이진 세마포어, 카운팅 세마포어

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

세마포어(Semaphore)는 **정수 카운터와 대기 큐를 가진 동기화 객체로, P(wait) 연산으로 카운터를 감소시키고 V(signal) 연산으로 증가시키며, 카운터가 0 이하면 대기한다.**

### 💡 비유: 주차장 빈 자리

세마포어는 **주차장 빈 자리**와 같다. 주차장에 10자리 있다. 입구에 표시판. "빈 자리: 10". 차가 들어오면 9. 또 들어오면 8. 0이 되면 "만차". 기다려야. 나가면 1 증가. 대기하던 차 진입. 세마포어가 똑같다. 카운터, wait, signal.

## Ⅱ. 세마포어 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                세마포어 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【세마포어 유형】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 이진 세마포어 (Binary Semaphore):                                     │ │
│  │     • 카운터: 0 또는 1                                                │ │
│  │     • 뮤텍스와 유사 (소유권 없음)                                       │ │
│  │     • 상호 배제용                                                    │ │
│  │                                                               │ │
│  │  2. 카운팅 세마포어 (Counting Semaphore):                                  │ │
│  │     • 카운터: 0 이상 정수                                              │ │
│  │     • 제한된 자원 관리                                                │ │
│  │     • 예: 연결 풀, 스레드 풀                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【생산자-소비자 패턴】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  세마포어 2개 사용:                                                        │ │
│  │  • empty: 빈 슬롯 수 (초기값: N)                                       │ │
│  │  • full: 찬 슬롯 수 (초기값: 0)                                        │ │
│  │                                                               │ │
│  │  생산자:                                                                   │ │
│  │  wait(empty);  // 빈 슬롯 대기                                         │ │
│  │  // 데이터 생성                                                          │ │
│  │  signal(full);  // 찬 슬롯 증가                                        │ │
│  │                                                               │ │
│  │  소비자:                                                                   │ │
│  │  wait(full);   // 찬 슬롯 대기                                         │ │
│  │  // 데이터 소비                                                          │ │
│  │  signal(empty);  // 빈 슬롯 증가                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 유연성: 카운팅 가능                                                │ │
│  │  • 자원 관리: 제한된 자원                                              │ │
│  │  • 동기화: 다양한 패턴                                                │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 복잡성: P/V 쌍 맞추기                                               │ │
│  │  • 교착상태: 잘못된 순서                                               │ │
│  │  • 소유권 없음: 누구나 signal 가능                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 세마포어 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                세마포어 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【pthread 세마포어】                                                            │)
│  ──────────────────                                                  │
│  #include <semaphore.h>                                             │
│                                                                     │
│  sem_t sem;                                                         │
│  sem_init(&sem, 0, 3);  // 초기값 3                                  │
│                                                                     │
│  // P 연산 (wait)                                                    │
│  sem_wait(&sem);                                                    │
│  // 자원 사용                                                         │
│                                                                     │
│  // V 연산 (signal)                                                  │
│  sem_post(&sem);                                                    │
│                                                                     │
│  sem_destroy(&sem);                                                 │
│                                                                     │
│  【C++20 세마포어】                                                              │)
│  ──────────────────                                                  │
│  #include <semaphore>                                               │
│                                                                     │
│  std::counting_semaphore<10> sem(3);  // 최대 10, 초기 3             │
│                                                                     │
│  sem.acquire();  // wait                                            │
│  // 자원 사용                                                         │
│  sem.release();  // signal                                          │
│                                                                     │
│  【System V 세마포어】                                                            │)
│  ──────────────────                                                  │
│  #include <sys/sem.h>                                               │
│  int semid = semget(key, 1, IPC_CREAT | 0666);                      │
│  struct sembuf op = {0, -1, 0};  // P 연산                           │
│  semop(semid, &op, 1);                                              │
│                                                                     │
│  【세마포어 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 카운터: 적절한 초기값                                              │
│  2. 쌍 맞춤: wait/signal 쌍                                           │
│  3. 교착상태: 획득 순서                                                │
│  4. 타임아웃: sem_timedwait                                           │
│  5. 측정: 대기 시간, 카운터                                            │
│  6. 대안: 조건 변수, 이벤트                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 세마포어: 정수 카운터 동기화 객체
• P(wait): 카운터--, 음수면 대기
• V(signal): 카운터++, 대기 중 깨움
• 이진: 0 또는 1 (뮤텍스 유사)
• 카운팅: 0 이상 정수 (자원 관리)
• 장점: 유연성, 자원 관리
• 단점: 복잡성, 교착상태
• 패턴: 생산자-소비자, Reader-Writer
• vs 뮤텍스: 소유권 없음
• 활용: 풀, 버퍼, IPC
```
