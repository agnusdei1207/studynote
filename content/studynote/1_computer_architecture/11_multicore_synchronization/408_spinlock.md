# 408. 스핀락 (Spinlock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 획득까지 루프를 돌며 대기하는 동기화 기법
> 2. **가치**: 짧은 대기에 효율, 컨텍스트 전환 없음
> 3. **융합**: CAS, 백오프, 스핀 카운트, 선점

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

스핀락(Spinlock)은 **락을 획득할 때까지 루프(busy-wait)를 돌며 계속 시도하는 동기화 기법으로, 컨텍스트 전환 없이 대기하여 짧은 임계 구역에 효율적이다.**

### 💡 비유: 화장실 문 앞에서 기다리기

스핀락은 **화장실 문 앞에서 기다리기**와 같다. 화장실이 사용 중이다. 문 앞에 선다. 계속 확인한다. "나왔나?" "나왔나?" 문을 계속 두드린다. 나오면 바로 들어간다. 자리를 비우지 않는다. 스핀락이 똑같다. 락 획득까지 루프, CPU 사용, 바로 진입.

### 스핀락 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                스핀락 개요                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스핀락 동작】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  락 획득 시도:                                                            │ │
│  │  ┌──────────────────────────────────────────────────────┐      │ │
│  │  │  while (lock == 1) {  // 락이 잠겨있으면              │      │ │
│  │  │      // 계속 확인 (스핀)                               │      │ │
│  │  │  }                                                    │      │ │
│  │  │  lock = 1;  // 락 획득                                 │      │ │
│  │  └──────────────────────────────────────────────────────┘      │ │
│  │                                                               │ │
│  │  문제: 위 코드는 원자적이지 않음 → CAS 사용 필요                         │ │
│  │                                                               │ │
│  │  올바른 구현 (CAS):                                                       │ │
│  │  ┌──────────────────────────────────────────────────────┐      │ │
│  │  │  while (CAS(&lock, 0, 1) == 실패) {                   │      │ │
│  │  │      // 스핀 (빈 루프 또는 백오프)                      │      │ │
│  │  │  }                                                    │      │ │
│  │  │  // 임계 구역                                          │      │ │
│  │  │  lock = 0;  // 락 해제                                 │      │ │
│  │  └──────────────────────────────────────────────────────┘      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스핀락 vs 뮤텍스】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  스핀락:                                                                  │ │
│  │  • 대기: busy-wait (CPU 사용)                                         │ │
│  │  • 컨텍스트 전환: 없음                                                 │ │
│  │  • 적합: 짧은 임계 구역                                                │ │
│  │                                                               │ │
│  │  뮤텍스:                                                                  │ │
│  │  • 대기: 블록 (CPU 양보)                                               │ │
│  │  • 컨텍스트 전환: 있음                                                 │ │
│  │  • 적합: 긴 임계 구역                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 스핀락 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                스핀락 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스핀락 변형】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 단순 스핀락:                                                          │ │
│  │     while (CAS(&lock, 0, 1) == 실패);                                  │ │
│  │     → 높은 경합 시 캐시 트래픽                                          │ │
│  │                                                               │ │
│  │  2. 백오프 스핀락 (Exponential Backoff):                                  │ │
│  │     실패 시 대기 시간 증가                                               │ │
│  │     → 경합 감소                                                        │ │
│  │                                                               │ │
│  │  3. 큐 기반 스핀락 (MCS, CLH):                                            │ │
│  │     대기 스레드를 큐로 관리                                               │ │
│  │     → 공정성, 캐시 트래픽 감소                                           │ │
│  │                                                               │
│  │  4. 티켓 락 (Ticket Lock):                                               │ │
│  │     번호표 방식                                                        │ │
│  │     → 공정성 보장                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【선점형 스핀락】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  문제: 스핀 중 선점되면 락 보유 시간 길어짐                               │ │
│  │                                                               │ │
│  │  해결:                                                                    │ │
│  │  • preempt_disable(): 선점 비활성화                                     │ │
│  │  • preempt_enable(): 선점 활성화                                        │ │
│  │                                                               │ │
│  │  Linux: spin_lock() / spin_unlock()                                     │ │
│  │         (선점 비활성화 포함)                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 낮은 오버헤드: 컨텍스트 전환 없음                                    │ │
│  │  • 빠른 응답: 락 해제 즉시 획득                                         │ │
│  │  • 단순: 구현 간단                                                     │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • CPU 낭비: 대기 중 CPU 사용                                          │ │
│  │  • 경합: 높으면 성능 급감                                               │ │
│  │  • 우선순위 역전 가능                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 스핀락 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                스핀락 분석                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 스핀락】                                                               │)
│  ──────────────────                                                  │
│  #include <linux/spinlock.h>                                        │
│                                                                     │
│  spinlock_t my_lock;                                                │
│  spin_lock_init(&my_lock);                                          │
│                                                                     │
│  spin_lock(&my_lock);                                               │
│  // 임계 구역                                                         │
│  spin_unlock(&my_lock);                                             │
│                                                                     │
│  【pthread 스핀락】                                                              │)
│  ──────────────────                                                  │
│  pthread_spinlock_t spin_lock;                                      │
│  pthread_spin_init(&spin_lock, PTHREAD_PROCESS_PRIVATE);            │
│                                                                     │
│  pthread_spin_lock(&spin_lock);                                     │
│  // 임계 구역                                                         │
│  pthread_spin_unlock(&spin_lock);                                   │
│                                                                     │
│  【C++ atomic_flag】                                                            │)
│  ──────────────────                                                  │
│  std::atomic_flag lock = ATOMIC_FLAG_INIT;                          │
│  while (lock.test_and_set(std::memory_order_acquire)) {             │
│      // 스핀                                                          │
│  }                                                                  │
│  // 임계 구역                                                         │
│  lock.clear(std::memory_order_release);                             │
│                                                                     │
│  【스핀락 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 임계 구역: 최소화                                                 │
│  2. 경합: 낮은 경우만                                                 │
│  3. 백오프: 높은 경합 시                                               │
│  4. 대안: 오래 걸리면 뮤텍스                                           │
│  5. 인터럽트: irqsave/irqrestore                                     │
│  6. 측정: 스핀 횟수, 대기 시간                                         │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 스핀락: 락 획득까지 busy-wait
• 구현: CAS 기반
• 장점: 컨텍스트 전환 없음
• 단점: CPU 사용, 경합 시 성능 저하
• 적합: 짧은 임계 구역
• 변형: 백오프, MCS, CLH, 티켓
• 선점: preempt_disable 필요
• vs 뮤텍스: 짧음=스핀, 김=뮤텍스
• 활용: 커널, 인터럽트, SMP
• 최적화: 임계 구역 최소화
```
