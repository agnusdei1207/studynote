# 415. RCU (Read-Copy-Update)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 읽기는 락 없이, 쓰기는 복사 후 교체로 동기화
> 2. **가치**: 읽기 많은 워크로드에서 높은 성능
> 3. **융합**: Grace Period, 콜백, 커널 동기화

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

RCU(Read-Copy-Update)는 **읽기 연산은 동기화 없이 수행하고, 쓰기 연산은 데이터를 복사하여 수정한 후 원자적으로 포인터를 교체하는 동기화 메커니즘**이다. Linux 커널에서 널리 사용된다.

### 💡 비유: 도서관 책 업데이트

RCU는 **도서관 책 업데이트**와 같다. 책이 있다. 많은 사람이 읽는다. 락 없이 읽는다. 새 버전이 나왔다. 사서가 복사본을 만든다. 복사본을 수정한다. 다 하면 교체한다. 예전 책은? 아직 읽는 사람이 있다. 다 읽으면 폐기. RCU가 똑같다. 읽기 자유, 쓰기는 복사, Grace Period 후 해제.

### RCU 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                RCU 개요                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RCU 동작 원리】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 읽기 (락 없음):                                                       │ │
│  │     rcu_read_lock();                                                     │ │
│  │     ptr = rcu_dereference(data);                                         │ │
│  │     // ptr 사용                                                           │ │
│  │     rcu_read_unlock();                                                   │ │
│  │                                                               │ │
│  │  2. 쓰기 (복사 후 교체):                                                  │ │
│  │     new_data = copy(old_data);                                           │ │
│  │     modify(new_data);                                                    │ │
│  │     rcu_assign_pointer(data, new_data);                                  │ │
│  │     synchronize_rcu();  // Grace Period 대기                             │ │
│  │     free(old_data);                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Grace Period】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Grace Period: 모든 기존 리더가 읽기를 완료하는 기간                      │ │
│  │                                                               │ │
│  │  시간 →                                                                    │ │
│  │  ──────────────────────────────────────────────────────→        │ │
│  │  리더 1: [======rcu_read_lock/_unlock======]                    │ │
│  │  리더 2:    [===rcu_read===]                                    │ │
│  │  리더 3:              [=======rcu_read=======]                   │ │
│  │                         ↑                                       │ │
│  │                    업데이트 발생                                  │ │
│  │                         │                                       │ │
│  │                    Grace Period 시작                            │ │
│  │                                              │                  │ │
│  │                                    Grace Period 끝              │ │
│  │                                        (이전 데이터 해제 가능)     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. RCU 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RCU 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 RCU API】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  읽기 측:                                                                  │ │
│  │  • rcu_read_lock(): 읽기 섹션 시작 (preempt_disable)                      │ │
│  │  • rcu_read_unlock(): 읽기 섹션 끝                                        │ │
│  │  • rcu_dereference(p): RCU 보호 포인터 읽기                               │ │
│  │                                                               │ │
│  │  쓰기 측:                                                                  │ │
│  │  • rcu_assign_pointer(p, v): 새 포인터 할당                               │ │
│  │  • synchronize_rcu(): Grace Period 대기                                  │ │
│  │  • call_rcu(&head, callback): Grace Period 후 콜백                       │ │
│  │  • kfree_rcu(ptr, rcu_head): Grace Period 후 메모리 해제                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RCU 활용 예시】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  // 연결 리스트 노드 교체                                                  │ │
│  │  void update_node(struct list *head, int old_val, int new_val) {          │ │
│  │      struct node *old, *new;                                              │ │
│  │      rcu_read_lock();                                                     │ │
│  │      old = rcu_dereference(head->next);                                   │ │
│  │      rcu_read_unlock();                                                   │ │
│  │                                                               │ │
│  │      new = kmalloc(sizeof(*new), GFP_KERNEL);                             │ │
│  │      *new = *old;  // 복사                                                │ │
│  │      new->value = new_val;  // 수정                                       │ │
│  │                                                               │ │
│  │      rcu_assign_pointer(head->next, new);  // 교체                         │ │
│  │      kfree_rcu(old, rcu);  // Grace Period 후 해제                        │ │
│  │  }                                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 읽기 락 없음: 최고 성능                                             │ │
│  │  • 읽기-많은 워크로드에 최적                                            │ │
│  │  • 교착상태 없음                                                        │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 쓰기: 메모리 복사 비용                                               │ │
│  │  • Grace Period: 지연                                                   │ │
│  │  • 복잡성: 올바른 사용 어려움                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. RCU × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RCU × 타 과목 융합                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 캐시: 읽기는 캐시 친화적                                           │
│  • 원자적: 포인터 교체만 원자적                                        │
│  • 메모리: 메모리 배리어                                              │
│  • 성능: 읽기 O(1), 쓰기 O(1) + 복사                                  │
│                                                                     │
│  【운영체제】                                                          │
│  • Linux: 커널 전반 사용                                              │
│  • 선점: preempt_disable                                             │
│  • 스케줄링: context switch = quiescent state                        │
│  • 타이머: Grace Period 감지                                          │
│                                                                     │
│  【프로그래밍】                                                        │
│  • Userspace RCU (liburcu)                                           │
│  • 패턴: Copy-on-Write                                               │
│  • 동시성: 읽기-많은-쓰기-적게                                         │
│  • 구조: 링크드 리스트, 해시 테이블                                    │
│                                                                     │
│  【성능】                                                              │
│  • 읽기: 락 없음, 최고 성능                                            │
│  • 쓰기: 복사 + 교체                                                  │
│  • Grace Period: 수~수백 ms                                          │
│  • 워크로드: 읽기 90%+에 적합                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RCU vs 다른 동기화:                                                                  │ │
│  │  ──────────────────────                                              │ │
│  │  메커니즘    읽기        쓰기           적합                          │ │
│  │  ────────    ────        ────           ────                          │ │
│  │  뮤텍스      락          락             일반적                        │ │
│  │  RW락       읽기락      쓰기락          읽기 많음                      │ │
│  │  RCU        락 없음     복사+교체       읽기 매우 많음                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### RCU 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RCU 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 RCU 사용】                                                          │)
│  ──────────────────                                                  │
│  // 읽기 측                                                           │
│  rcu_read_lock();                                                    │
│  struct task_struct *task = rcu_dereference(current->task);          │
│  // task 사용                                                         │
│  rcu_read_unlock();                                                  │
│                                                                     │
│  // 쓰기 측                                                           │
│  struct task_struct *new_task = alloc_task();                        │
│  rcu_assign_pointer(current->task, new_task);                        │
│  call_rcu(&old_task->rcu, free_task_callback);                       │
│                                                                     │
│  【Userspace RCU】                                                               │)
│  ──────────────────                                                  │
│  #include <urcu.h>                                                   │
│                                                                     │
│  // 읽기                                                              │
│  rcu_read_lock();                                                    │
│  // 데이터 읽기                                                       │
│  rcu_read_unlock();                                                  │
│                                                                     │
│  // 쓰기                                                              │
│  synchronize_rcu();                                                  │
│                                                                     │
│  【성능 모니터링】                                                              │)
│  ──────────────────                                                  │
│  # RCU 통계                                                           │
│  $ cat /proc/rcu/*/rcugp                                             │
│  # Grace Period 횟수                                                  │
│                                                                     │
│  【RCU 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 읽기 섹션: 짧게                                                   │
│  2. 쓰기: call_rcu 사용 (비동기)                                       │
│  3. 메모리: kfree_rcu                                                 │
│  4. 워크로드: 읽기 90%+ 확인                                           │
│  5. Grace Period: 긴 섹션 피하기                                       │
│  6. 디버깅: RCU_LOCKDEP                                               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• RCU: Read-Copy-Update 동기화
• 읽기: 락 없음, 최고 성능
• 쓰기: 복사 후 교체
• Grace Period: 이전 리더 완료 대기
• 장점: 읽기 락 없음, 교착상태 없음
• 단점: 쓰기 복사 비용, 지연
• Linux: 커널 핵심 동기화
• API: rcu_read_lock/unlock, synchronize_rcu
• Userspace: liburcu
• 적합: 읽기-많은 워크로드
```
