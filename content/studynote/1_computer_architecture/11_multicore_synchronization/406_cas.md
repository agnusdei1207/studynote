# 406. CAS (Compare-And-Swap)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 조건부 원자적 교환 명령어
> 2. **가치**: Lock-free 자료구조, 락 구현, 동시성
> 3. **융합**: ABA 문제, LL/SC, 원자적 연산

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

CAS(Compare-And-Swap)는 **메모리 위치의 현재 값을 예상값과 비교하여, 같으면 새 값으로 원자적으로 교체하고, 다르면 아무것도 하지 않는 하드웨어 원자적 명령어**다. 동기화와 lock-free 알고리즘의 기반이다.

### 💡 비유: 좌석 예약

CAS는 **좌석 예약**과 같다. 비행기 좌석이 있다. 빈 좌석을 확인한다. 예약하려 한다. 하지만 다른 사람이 먼저? 충돌. CAS는 확인 후 예약을 한 번에. "빈 좌석이면 내 것" 원자적. 다른 사람이 먼저면 실패. 재시도.

### CAS 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                CAS 개요                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CAS 동작 원리】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  CAS(addr, expected, new):                                               │ │
│  │                                                               │ │
│  │  1. addr 위치의 현재 값을 읽음                                          │ │
│  │  2. expected와 비교                                                    │ │
│  │  3. 같으면: new 값을 addr에 쓰고, true 반환                             │ │
│  │  4. 다르면: 아무것도 안 하고, false 반환                                 │ │
│  │                                                               │ │
│  │  모든 과정이 원자적 (중단 불가)                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CAS vs 일반 읽기-쓰기】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  일반 연산 (비원자적):                                                    │ │
│  │  ┌────────────────────────────────────────┐                    │ │
│  │  │  temp = *addr;      // 읽기            │                    │ │
│  │  │  if (temp == expected) {               │                    │ │
│  │  │      *addr = new;   // 쓰기            │                    │ │
│  │  │  }                                     │                    │ │
│  │  │  → 읽기와 쓰기 사이에 다른 스레드 개입 가능 │                    │ │
│  │  └────────────────────────────────────────┘                    │ │
│  │                                                               │ │
│  │  CAS (원자적):                                                            │ │
│  │  ┌────────────────────────────────────────┐                    │ │
│  │  │  CAS(addr, expected, new)              │                    │ │
│  │  │  → 읽기-비교-쓰기가 한 번에 실행         │                    │ │
│  │  │  → 중간에 다른 스레드 개입 불가          │                    │ │
│  │  └────────────────────────────────────────┘                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. CAS 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CAS 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CAS 기반 카운터 구현】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  void atomic_increment(int* counter) {                                   │ │
│  │      int old_val;                                                        │ │
│  │      do {                                                                │ │
│  │          old_val = *counter;                                             │ │
│  │      } while (!CAS(counter, old_val, old_val + 1));                      │ │
│  │  }                                                                       │ │
│  │                                                               │ │
│  │  // CAS가 실패하면 재시도 (다른 스레드가 먼저 변경)                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ABA 문제】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시나리오:                                                                 │ │
│  │  1. 스레드 1: addr 읽음 → A                                                │ │
│  │  2. 스레드 2: addr를 A → B → A로 변경                                      │ │
│  │  3. 스레드 1: CAS(addr, A, C) → 성공!                                      │ │
│  │                                                               │ │
│  │  문제: 스레드 1은 변화를 감지 못함                                        │ │
│  │        중간에 B가 있었지만 A로 돌아옴                                     │ │
│  │                                                               │ │
│  │  해결:                                                                    │ │
│  │  • 더블 워드 CAS (값 + 버전)                                             │ │
│  │  • LL/SC 사용                                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • Lock-free: 락 없이 동기화                                          │ │
│  │  • 진행 보장: 항상 어떤 스레드는 진행                                  │ │
│  │  • 효율: 낮은 경합 시 빠름                                             │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • ABA 문제                                                            │ │
│  │  • 높은 경합: 많은 재시도                                               │ │
│  │  • 단일 변수만 지원                                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### CAS 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CAS 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ CAS 사용】                                                               │)
│  ──────────────────                                                  │
│  #include <atomic>                                                  │
│                                                                     │
│  std::atomic<int> value{0};                                         │
│                                                                     │
│  // CAS 루프                                                         │
│  int expected = value.load();                                       │
│  while (!value.compare_exchange_weak(expected, expected + 1)) {     │
│      // expected가 자동으로 갱신됨                                    │
│  }                                                                  │
│                                                                     │
│  // compare_exchange_strong vs weak                                 │
│  // weak: 거짓 실패 가능 (SPARC 등에서 유리)                           │
│  // strong: 항상 정확                                                │
│                                                                     │
│  【어셈블리 확인】                                                              │)
│  ──────────────────                                                  │
│  $ objdump -d program | grep cmpxchg                                │
│  lock cmpxchg %edx,(%rax)    # CAS 명령어                            │
│                                                                     │
│  【성능 측정】                                                                   │)
│  ──────────────────                                                  │
│  $ perf stat -e instructions,cycles ./cas_test                      │
│  # 높은 경합 시 CPI 증가                                              │
│                                                                     │
│  【CAS 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 백오프: 지수 백오프로 경합 감소                                    │
│  2. 로컬 복사: 읽기 후 로컬에서 연산                                   │
│  3. 타임아웃: 무한 루프 방지                                          │
│  4. ABA 방지: 버전 번호 추가                                         │
│  5. 측정: 실패율 모니터링                                            │
│  6. 대안: 높은 경합은 뮤텍스                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• CAS: 비교 후 원자적 교환
• 동작: 읽기-비교-쓰기 원자적
• 반환: 성공/실패
• 장점: Lock-free, 진행 보장
• 단점: ABA 문제, 높은 경합
• ABA 해결: 더블 워드, LL/SC
• x86: CMPXCHG + LOCK
• C++: compare_exchange
• 활용: 스핀락, 원자적 카운터
• 최적화: 백오프, 버전 번호
```
