> **핵심 인사이트**
> 1. 명령어 세트(Instruction Set)는 CPU가 알아듣고 실행할 수 있는 **모든 기계어 명령어들의 총집합이자 사전(Dictionary)**이다.
> 2. 프로그램(소프트웨어)이 하드웨어에게 요구할 수 있는 모든 물리적 동작(데이터 이동, 산술 연산, 논리 연산, 제어 흐름 변경 등)의 한계를 정의한다.
> 3. ISA(명령어 집합 구조)의 핵심 구성 요소이며, 이 사전의 두께(명령어의 개수와 복잡도)에 따라 **CISC와 RISC라는 양대 아키텍처 철학**으로 나뉜다.

---

## Ⅰ. 컴퓨터의 어휘력 (Vocabulary)
인간의 언어에 단어장이 있듯, 컴퓨터의 뇌(CPU)에도 단어장이 있습니다. 이 단어장에 없는 말(명령어)을 하면 CPU는 "이해할 수 없는 명령입니다"라며 에러(Illegal Instruction)를 내뿜습니다. 이 단어장 전체를 '명령어 세트'라고 부릅니다.

일반적인 명령어 세트는 크게 네 가지 카테고리로 묶입니다.
1. **데이터 이동 (Data Transfer)**: 메모리와 레지스터 사이, 혹은 레지스터끼리 데이터를 복사하고 옮기는 가장 흔한 명령입니다. (예: `LOAD`, `STORE`, `MOV`)
2. **산술/논리 연산 (Arithmetic/Logic)**: 실제 계산을 하는 일꾼 명령어입니다. (예: `ADD`, `SUB`, `AND`, `OR`, `SHIFT`)
3. **제어 흐름 (Control Flow)**: 프로그램 카운터(PC)의 값을 강제로 바꿔서 실행 순서를 비틀어버리는 점프 명령입니다. (예: `JUMP`, `BRANCH`, `CALL`, `RETURN`)
4. **특수 제어 (System Control)**: 운영체제(OS)만 쓸 수 있는 특권 명령어로, 컴퓨터 전원을 끄거나 인터럽트를 제어합니다. (예: `HALT`, `SYSCALL`)

```ascii
[명령어 세트의 카테고리별 비중 예시]

 [ 명령어 세트 (Instruction Set) 사전 ]
 ├── 데이터 이동 (40%) : MOV A, B (가장 많이 쓰임)
 ├── 산술/논리 (30%) : ADD R1, R2
 ├── 제어 흐름 (20%) : BEQ R1, 0, Target (조건문/반복문의 핵심)
 └── 시스템 제어 (10%) : HALT (프로그램 종료)
```

📢 **섹션 요약 비유**: 명령어 세트는 강아지 훈련 매뉴얼입니다. "앉아(데이터 이동)", "기다려(제어 흐름)", "먹어(산술 연산)"라는 단어(명령어)들을 미리 훈련해 놓았습니다. 주인이 매뉴얼에 없는 "춤춰!"라고 명령하면 강아지는 알아듣지 못하고 멍하니 서 있을 수밖에 없습니다.

## Ⅱ. 직교성(Orthogonality): 좋은 명령어 세트의 조건
아키텍처 설계자들은 명령어 세트를 만들 때 **직교성(Orthogonality)**이라는 철학을 가장 중요하게 생각합니다.
직교성이란 "명령어(Opcode)와 재료(Operand)가 서로 완벽하게 독립적이어서, 프로그래머가 어떤 조합을 짜든 제약 없이 100% 돌아가야 한다"는 뜻입니다.

* **나쁜 예 (직교성 파괴)**: `ADD` 명령어는 레지스터 1번부터 32번까지 다 쓸 수 있는데, 유독 `SUB` 명령어는 레지스터 1번에서 5번까지만 쓸 수 있게 하드웨어를 제한해 둔 경우입니다. 프로그래머(컴파일러)는 코딩할 때마다 예외 규칙을 외워야 해서 미쳐버립니다.
* **좋은 예 (완벽한 직교성)**: 덧셈이든 뺄셈이든 곱셈이든, 모든 연산 명령어는 32개의 레지스터를 아무런 차별 없이 자유롭게 쓸 수 있습니다. 

명령어 세트가 이 직교성을 잘 지킬수록(RISC 방식), 컴파일러를 만들기가 엄청나게 쉬워지고 프로그램 최적화가 완벽하게 이루어집니다.

```ascii
[직교성(Orthogonality)의 유무 비교]

 [직교성이 낮은 아키텍처 (CISC 구형)]
 ADD R1, R2 (성공)
 SUB R1, R2 (에러! SUB는 R1, R2 지원 안 함!) -> 컴파일러 멘붕!

 [직교성이 완벽한 아키텍처 (현대 RISC)]
 ADD R1, R2 (성공)
 SUB R1, R2 (성공)
 MUL R1, R2 (성공) -> 모든 명령어가 모든 레지스터를 공평하게 공유함!
```

📢 **섹션 요약 비유**: 직교성은 '블록 장난감의 완벽한 호환성'입니다. 빨간색 4칸 블록이든 파란색 2칸 블록이든 아무 제약 없이 찰칵찰칵 끼워져야 훌륭한 장난감입니다. 만약 "빨간색 블록 위에는 절대 노란색 블록을 끼울 수 없다"는 이상한 제약(직교성 파괴)이 있다면 아이들이 성(프로그램)을 만들 때 짜증을 낼 것입니다.

## Ⅲ. 아키텍처 관점: 사전의 두께 (CISC vs RISC)
컴퓨터 역사상 이 '사전(명령어 세트)'의 두께를 두고 거대한 철학적 전쟁이 벌어졌습니다.

1. **CISC (두꺼운 사전)**: 
   * 인텔 x86은 명령어 세트에 수백, 수천 개의 단어를 다 욱여넣었습니다. 심지어 "메모리에 있는 글자열을 통째로 복사해서 저쪽 메모리로 옮겨라(`MOVS`)" 같은 거대한 단어도 있습니다. 
   * 단어가 워낙 풍부해서 프로그래머가 글을 짧게 쓸 수 있지만, CPU 내부의 해독기(디코더)가 이 두꺼운 사전을 뒤지느라 전기를 너무 많이 먹습니다.
2. **RISC (얇은 사전)**: 
   * ARM 아키텍처는 "어차피 쓰는 단어만 쓰잖아? 어려운 단어 싹 다 버려!"라며 명령어 세트를 50개 수준으로 확 줄여버렸습니다(Reduced). 
   * 대신 "A를 10번 더해"라는 식으로 쉬운 단어들을 아주 빠르게 여러 번 말해서 어려운 단어의 뜻을 만들어냅니다. 사전이 얇으니 CPU가 즉각즉각 반응(1클럭)하여 극강의 속도와 전성비(배터리 효율)를 달성했습니다.

```ascii
[사전의 두께에 따른 아키텍처 전쟁]

 [ CISC 명령어 세트 (백과사전) ]
 - 단어 수: 1000개 이상
 - 특수 단어: "배열_A의_모든원소에_5를더하라" (명령어 1줄)
 - 특징: 무겁고 전력 소모가 큼 (PC/서버)

 [ RISC 명령어 세트 (포켓 사전) ]
 - 단어 수: 50개
 - 특수 단어: 없음. (LOAD, ADD, STORE의 조합으로 만듦)
 - 특징: 극도로 가볍고 빠름 (스마트폰/태블릿)
```

📢 **섹션 요약 비유**: CISC는 '법률 전문 용어'입니다. 단어 하나에 엄청난 뜻이 함축되어 있어 전문가는 짧게 말할 수 있지만, 초보자(CPU)는 사전 펴고 해석하느라 진땀을 뺍니다. RISC는 '쉬운 일상 언어'입니다. "그거 저기다 갖다 놔"처럼 단어가 아주 쉽고 직관적이어서 누구든 0.1초 만에 듣고 즉시 행동할 수 있는 빠르고 효율적인 언어입니다.

## 개념 맵 (Concept Map)
```ascii
[명령어 세트 (Instruction Set)]
   │
   ├── 정의: CPU가 하드웨어적으로 이해하고 실행할 수 있는 머신 인스트럭션(Machine Instruction)의 총집합
   │
   ├── 4대 기능 분류
   │     ├── 데이터 전송 (Load, Store, Move)
   │     ├── 산술/논리 연산 (Add, Sub, And, Or)
   │     ├── 제어 흐름 제어 (Jump, Branch, Call)
   │     └── 시스템/특권 제어 (Halt, Interrupt)
   │
   └── 아키텍처 설계 철학
         ├── 직교성(Orthogonality): 명령어와 주소 지정 방식의 완벽한 상호 독립성 (컴파일러 최적화 용이)
         └── 세트의 규모: 복잡/방대함(CISC) vs 단순/최소화(RISC)
```
