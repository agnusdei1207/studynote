# 178. 레지스터 간접 주소 지정 (Register Indirect Addressing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 레지스터 값이 메모리 주소인 주소 지정 방식
> 2. **가치**: 포인터 연산, 배열 순회, 동적 메모리 접근
> 3. **융합**: 힙 할당, 스택 프레임, 문자열 처리

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

레지스터 간접 주소 지정(Register Indirect Addressing)은 **레지스터에 저장된 값을 메모리 주소로 사용**하여 데이터에 접근하는 방식이다. 레지스터가 포인터 역할을 하며, 메모리 간접보다 빠르다.

### 💡 비유: 사서함 번호가 적힌 메모지

레지스터 간접 주소 지정은 **메모지에 적힌 사서함 번호로 찾아가는 것**과 같다. 메모지(레지스터)에 "101번 사서함"이라고 적혀있으면, 그 번호로 가서 편지를 꺼낸다. 메모지만 바꾸면 다른 사서함도 볼 수 있다.

### 레지스터 간접 주소 지정 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                레지스터 간접 주소 지정 구조                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│                                                                     │
│  명령어:  ┌─────────────┬─────────────┐                             │
│          │   Opcode    │    (Rn)     │  (레지스터 번호)             │
│          └─────────────┴─────────────┘                             │
│                    │                                                │
│                    ▼ Rn의 값 = 메모리 주소                          │
│  CPU 레지스터:  ┌─────────────────┐                                 │
│                │  Rn = 0x1000    │  ← 주소 값                       │
│                └────────┬────────┘                                 │
│                         │                                          │
│                         ▼                                          │
│  메모리:       ┌─────────────────┐                                 │
│     0x1000 →  │    데이터        │  ← 실제 데이터                   │
│                └─────────────────┘                                 │
│                                                                     │
│  EA = [Rn]  (유효 주소 = 레지스터 값)                               │
│                                                                     │
│  【ISA별 문법】                                                     │
│  ──────────────                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      문법               예시                               │ │
│  │  ───────  ─────────────────  ──────────────────                │ │
│  │  x86      [reg]              MOV EAX, [EBX]                    │ │
│  │  ARM      [Rn]               LDR R0, [R1]                      │ │
│  │  MIPS     (rs)               lw $t0, 0($t1)   # 변위 포함      │ │
│  │  RISC-V   (rs1)              lw x5, 0(x6)     # 변위 포함      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【특징】                                                           │
│  ─────────                                                          │
│  • 1회 메모리 접근 (메모리 간접보다 빠름)                           │
│  • 실행 중 주소 변경 가능                                           │
│  • 배열, 포인터 연산에 적합                                         │
│  • 짧은 명령어 (레지스터 번호만 필요)                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  성능 비교:                                                       │ │
│  │  ──────────                                                       │ │
│  │  직접 주소:     MOV EAX, [0x1000]  ; 1회 메모리                  │ │
│  │  레지스터 간접: MOV EAX, [EBX]      ; 1회 메모리                 │ │
│  │  메모리 간접:   MOV EAX, [[0x1000]] ; 2회 메모리                 │ │
│  │                                                               │ │
│  │  → 레지스터 간접이 메모리 간접보다 빠름                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 레지스터 간접 주소 지정 확장

### 확장 형태

```
┌─────────────────────────────────────────────────────────────────────┐
│                레지스터 간접 주소 지정 확장                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 기본 레지스터 간접】                                           │
│  ──────────────────────────                                         │
│  MOV EAX, [EBX]         ; EAX = Mem[EBX]                            │
│  LDR R0, [R1]           ; R0 = Mem[R1]                              │
│                                                                     │
│  【2. 자동 증가/감소 (ARM)】                                         │
│  ──────────────────────────                                         │
│  LDR R0, [R1], #4       ; Post-index: R0=Mem[R1], R1+=4            │
│  LDR R0, [R1, #4]!      ; Pre-index: R1+=4, R0=Mem[R1]             │
│                                                                     │
│  ; 배열 순회에 유용                                                  │
│  loop:                                                              │
│      LDR R0, [R1], #4   ; arr[i++]                                  │
│      SUBS R2, R2, #1                                               │
│      BNE loop                                                       │
│                                                                     │
│  【3. 변위 포함 (MIPS/RISC-V)】                                      │
│  ─────────────────────────────                                      │
│  lw $t0, 0($t1)         ; Mem[$t1+0]                                │
│  lw $t0, 4($t1)         ; Mem[$t1+4]                                │
│  lw $t0, -4($t1)        ; Mem[$t1-4]                                │
│                                                                     │
│  【4. 인덱스 레지스터 (x86)】                                        │
│  ──────────────────────────                                         │
│  MOV EAX, [EBX+ECX*4]   ; Mem[EBX + ECX*4]                          │
│  MOV EAX, [EBX+ECX*4+8] ; Mem[EBX + ECX*4 + 8]                      │
│                                                                     │
│  【5. 다중 레지스터 전송 (ARM LDM/STM)】                             │
│  ─────────────────────────────────────                              │
│  LDMIA R0!, {R1-R4}     ; R0에서 R1~R4까지 로드, R0 증가            │
│  STMDB R0!, {R1-R4}     ; R1~R4를 R0에 저장, R0 감소                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  활용 예시:                                                       │ │
│  │  ──────────                                                       │ │
│  │  // C 코드                                                        │ │
│  │  int *ptr = malloc(100 * sizeof(int));                           │ │
│  │  for (int i = 0; i < 100; i++)                                   │ │
│  │      ptr[i] = i;                                                  │ │
│  │                                                               │ │
│  │  // 어셈블리 (레지스터 간접)                                       │ │
│  │  MOV R0, #0             ; i = 0                                  │ │
│  │  MOV R1, ptr            ; 포인터                                  │ │
│  │  loop:                                                            │ │
│  │  STR R0, [R1], #4       ; ptr[i] = i, ptr++                      │ │
│  │  ADD R0, R0, #1         ; i++                                    │ │
│  │  CMP R0, #100                                                    │ │
│  │  BLT loop                                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 레지스터 간접 주소 지정 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                레지스터 간접 주소 지정 분석                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【배열 합계 계산】                                                  │
│  ──────────────────────                                             │
│  // C 코드                                                           │
│  int sum_array(int *arr, int n) {                                   │
│      int sum = 0;                                                   │
│      for (int i = 0; i < n; i++)                                    │
│          sum += arr[i];                                             │
│      return sum;                                                    │
│  }                                                                  │
│                                                                     │
│  // ARM 어셈블리                                                     │
│  sum_array:                                                         │
│      MOV R2, #0           ; sum = 0                                 │
│  loop:                                                              │
│      CMP R1, #0           ; n == 0?                                 │
│      BEQ done                                                       │
│      LDR R3, [R0], #4     ; sum += *arr++ (레지스터 간접 + 증가)    │
│      ADD R2, R2, R3                                                 │
│      SUB R1, R1, #1       ; n--                                     │
│      B loop                                                         │
│  done:                                                              │
│      MOV R0, R2           ; return sum                              │
│      BX LR                                                          │
│                                                                     │
│  【문자열 길이 계산】                                                │
│  ──────────────────────                                             │
│  // C: int strlen(char *s)                                          │
│  strlen:                                                            │
│      XOR EAX, EAX         ; len = 0                                 │
│  loop:                                                              │
│      CMP BYTE PTR [RDI], 0  ; *s == '\0'?                           │
│      JE done                                                        │
│      INC EAX              ; len++                                   │
│      INC RDI              ; s++                                     │
│      JMP loop                                                       │
│  done:                                                              │
│      RET                                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  GDB 디버깅:                                                      │ │
│  │  ──────────                                                       │ │
│  │  (gdb) x/10x $rdi         ; RDI가 가리키는 메모리 10개           │ │
│  │  (gdb) p/x $rdi           ; 레지스터 값 확인                      │ │
│  │  (gdb) watch *(int*)$rdi  ; RDI가 가리키는 값 감시               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 레지스터 간접: 레지스터 값 = 메모리 주소
• EA = [Rn]
• 장점: 동적 주소, 포인터 연산, 1회 메모리 접근
• 확장: 자동 증가/감소, 변위, 인덱스
• 활용: 배열, 포인터, 문자열, 동적 메모리
```
