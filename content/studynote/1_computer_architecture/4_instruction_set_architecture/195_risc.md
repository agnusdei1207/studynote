# RISC (Reduced Instruction Set Computer)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RISC (Reduced Instruction Set Computer)는 복잡한 기능의 명령어를 제거하고 **단순하고 표준화된 명령어들만으로 구성**하여, 하드웨어의 실행 속도를 극대화하고 파이프라이닝 효율을 높인 명령어 집합 구조다.
> 2. **가치**: 명령어의 길이를 고정하고 메모리 접근을 로드/스토어(Load/Store) 명령으로 제한함으로써, 대부분의 명령어를 **단일 사이클 내에 완료**할 수 있게 하여 현대의 고성능·저전력 프로세서(ARM, Apple Silicon 등)의 표준이 되었다.
> 3. **융합**: 하드웨어의 복잡도를 소프트웨어(컴파일러)로 전가하는 설계를 취하며, 최근에는 오픈 소스 기반의 **RISC-V** 아키텍처를 통해 누구나 자유롭게 맞춤형 프로세서를 설계할 수 있는 생태계 혁신을 주도하고 있다.

+++

## Ⅰ. RISC의 개요

### 1. 정의
- 빈도가 높은 단순한 명령어들로 구성된 집합을 사용하며, 복잡한 명령어는 단순한 명령어들의 조합으로 처리하는 아키텍처다.

### 2. 등장 배경: '80/20 법칙의 발견'
- $1970$년대 중반, IBM의 존 콕(John Cocke) 등은 실제 프로그램에서 사용되는 명령어의 $80\%$가 전체 명령어 집합 중 단 $20\%$에 불과하다는 사실을 발견했다. 나머지 복잡한 명령어들은 칩 면적만 차지하고 실행 속도를 늦추고 있었다. 이에 "안 쓰는 복잡한 기능을 빼고, 자주 쓰는 단순한 기능을 빛의 속도로 만들자"는 철학으로 탄생한 것이 RISC다.

### 3. 💡 비유: '레고 블록과 조립식 가구'
- RISC는 **'기본적인 모양의 레고 블록'**과 같다.
- 블록 하나는 단순하지만(단순 명령어), 이들을 조합하면 성도 만들고 비행기도 만들 수 있다. 블록 모양이 일정해서 끼우기(파이프라이닝)가 아주 쉽고 빠르다. 반면 CISC가 완성된 장난감 세트라면, RISC는 조립하는 사람(컴파일러)의 실력이 중요해지는 방식이다.

## Ⅱ. RISC의 핵심 특징 및 메커니즘 (Deep Dive)

### 1. 기술적 5대 특징
| 특징 | 내용 | 효과 |
|:---|:---|:---|
| **고정 길이 명령어** | 모든 명령어가 32비트 또는 64비트로 동일 | 명령어 인출 및 해독 속도 최적화 |
| **로드/스토어 구조** | 오직 Load/Store만 메모리에 접근, 연산은 레지스터에서만 수행 | 메모리 병목 최소화, 연산 예측 가능성 향상 |
| **많은 레지스터** | 32개 이상의 풍부한 범용 레지스터 제공 | 느린 메모리 대신 빠른 레지스터 활용 극대화 |
| **단순한 주소 지정** | 복잡한 모드를 제거하고 직접/상대 주소 위주 사용 | 하드웨어 제어 로직 간소화 |
| **단일 사이클 실행** | 거의 모든 명령어가 1클럭 내에 완료 | **CPI 1.0** 달성 및 높은 클럭 달성 용이 |

### 2. RISC 연산 프로세스 (ASCII)
- CISC가 한 줄로 하던 일을 RISC는 3단계로 명확히 분리하여 수행한다.

```text
    [ RISC 연산 과정: C = A + B ]
    
    Instruction 1: LOAD  R1, Addr_A  (메모리 A를 레지스터 1로)
    Instruction 2: LOAD  R2, Addr_B  (메모리 B를 레지스터 2로)
    Instruction 3: ADD   R3, R1, R2  (레지스터 간 덧셈)
    Instruction 4: STORE Addr_C, R3  (결과를 메모리 C에 저장)
    
    * 각 단계가 균일하게 쪼개져 파이프라인에 빈틈없이 들어감.
```

### 3. 파이프라인 최적화 (Efficiency)
- 명령어의 형식이 일정하므로, CPU는 다음에 올 명령어가 무엇인지 알기도 전에 미리 가져오고(Fetch) 겹쳐서 실행하기가 매우 수월하다. 이는 처리량(Throughput) 향상의 핵심이다.

- **📢 섹션 요약 비유**: 모든 부품의 규격이 똑같은 공장 라인과 같아서, 막힘없이 쉼 없이 제품(결과)을 찍어낼 수 있는 초고속 자동화 시스템입니다.

## Ⅲ. RISC vs CISC 성능 방정식 관점 분석

| 성능 변수 | CISC 영향 | RISC 영향 | 비고 |
|:---|:---:|:---:|:---|
| **명령어 수 ($N$)** | **적음** | 많음 | RISC는 여러 줄로 나눠 써야 함 |
| **평균 사이클 ($CPI$)** | 높음 | **매우 낮음 ($\approx 1$)** | RISC의 최대 강점 |
| **클럭 주기 ($T$)** | 긺 (복잡 로직) | **짧음 (단순 로직)** | RISC는 클럭을 높이기 쉬움 |

- **📢 섹션 요약 비유**: CISC가 '무거운 짐을 지고 천천히 걷는 거인'이라면, RISC는 '가벼운 짐을 들고 엄청나게 빨리 뛰는 수많은 난쟁이'와 같습니다.

## Ⅳ. 실무 적용 및 아키텍처 트렌드 (Strategy)

### 1. 실무 시나리오: 모바일 및 IoT 기기 설계
- **상황**: 배터리 한 번 충전으로 이틀 이상 버텨야 하는 최신 스마트폰 개발.
- **판단**: 주저 없이 **RISC 기반의 ARM** 아키텍처를 선택해야 한다. 복잡한 마이크로코드 제어 장치가 필요 없는 RISC는 칩 면적을 적게 차지하고 전력 소모가 매우 낮기 때문이다. 또한, 수십 년간 최적화된 ARM 컴파일러 생태계를 통해 RISC의 단점인 '코드 양 증가' 문제도 상당 부분 해결할 수 있다.

### 2. 기술사적 인사이트: RISC-V의 파괴적 혁신
- **인사이트**: 특정 기업(ARM, Intel)에 종속되지 않는 **오픈 소스 RISC-V**의 부상은 하드웨어 설계의 민주화를 가져왔다. 기술사는 라이선스 비용 절감 차원을 넘어, 우리 서비스만의 특화 명령어를 직접 추가할 수 있는 '확장성' 관점에서 RISC-V 도입을 전략적으로 검토해야 한다. AI 전용 확장 명령어를 넣은 RISC-V 코어는 범용 칩보다 전성비 면에서 압도적 우위를 점할 수 있다.

- **📢 섹션 요약 비유**: 이미 만들어진 비싼 명품 정장(ARM)을 살 수도 있지만, 누구나 쓸 수 있는 도면(RISC-V)으로 내 몸에 딱 맞는 옷을 직접 지어 입는 시대가 왔습니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **연산 처리량 극대화**: 파이프라인 효율 극대화를 통한 단위 시간당 작업 완료 수 향상.
- **에너지 효율성**: 단순한 로직 설계를 통한 저전력 컴퓨팅 실현.

### 2. 미래 전망
과거 RISC와 CISC는 치열하게 싸웠지만, 현대의 고성능 프로세서들은 서로의 장점을 흡수하며 닮아가고 있다. 하지만 클라우드 데이터 센터와 모바일, 그리고 AI 가속기 시장이 커질수록 **'단순함이 곧 성능이자 효율'**이라는 RISC의 기본 원칙은 더욱 강력한 영향력을 발휘할 것이다. 특히 소프트웨어가 하드웨어 형상을 결정하는 차세대 컴퓨팅 환경에서 RISC는 가장 유연하고 강력한 기초 골조가 될 것이다.

- **📢 섹션 요약 비유**: 복잡한 장식보다는 튼튼한 기초가 건물을 더 높이 세울 수 있게 하듯, 단순한 명령어가 더 거대한 지능(AI)을 지탱하게 될 것입니다.
