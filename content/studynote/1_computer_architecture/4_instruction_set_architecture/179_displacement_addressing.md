# 179. 변위 주소 지정 (Displacement Addressing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 베이스 주소 + 고정 오프셋으로 유효 주소 계산
> 2. **가치**: 구조체 멤버 접근, 지역 변수, 배열 요소
> 3. **융합**: 컴파일러 최적화, 스택 프레임, 객체 지향

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

변위 주소 지정(Displacement Addressing)은 **레지스터(베이스) 값에 고정된 변위(오프셋)를 더해 유효 주소를 계산**하는 방식이다. 구조체 멤버, 지역 변수, 배열 요소 접근에 널리 사용된다.

### 💡 비유: 아파트 동+호수

변위 주소 지정은 **"101동 502호"**와 같다. 동(베이스)은 건물 위치이고, 호수(변위)는 그 건물 내에서의 위치다. "101동에서 5층 올라가서 2호"라고 생각할 수 있다.

### 변위 주소 지정 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                변위 주소 지정 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│                                                                     │
│  명령어:  ┌─────────────┬─────────────┬─────────────┐               │
│          │   Opcode    │   Base Reg  │ Displacement │               │
│          │   연산코드   │   베이스     │    변위      │               │
│          └─────────────┴─────────────┴─────────────┘               │
│                                                                     │
│  EA = [Base Register] + Displacement                                │
│  유효 주소 = 베이스 레지스터 값 + 변위                               │
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  Base Reg (R1):  0x1000  ─────┐                                     │
│                                │                                    │
│  Displacement:   +8      ─────┼──── EA = 0x1008                    │
│                                │                                    │
│                                ▼                                    │
│  메모리:   ┌─────────────────────────────────┐                      │
│    0x1000 │  [0]                             │                      │
│    0x1004 │  [4]                             │                      │
│    0x1008 │  [8] ← EA (여기서 데이터 읽기)   │                      │
│    0x100C │  [12]                            │                      │
│           └─────────────────────────────────┘                      │
│                                                                     │
│  【ISA별 문법】                                                     │
│  ──────────────                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      문법                 예시                             │ │
│  │  ───────  ───────────────────  ───────────────────             │ │
│  │  x86      [base+disp]          MOV EAX, [EBP+8]                │ │
│  │  ARM      [Rn, #disp]          LDR R0, [R1, #8]                │ │
│  │  MIPS     disp(rs)             lw $t0, 8($t1)                  │ │
│  │  RISC-V   disp(rs1)            lw x5, 8(x6)                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  변위 크기:                                                       │ │
│  │  ──────────                                                       │ │
│  │  • x86: 8/16/32비트                                               │ │
│  │  • ARM: 12비트 (±4095)                                           │ │
│  │  • MIPS: 16비트 부호있음 (±32767)                                 │ │
│  │  • RISC-V: 12비트 부호있음 (±2047)                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 변위 주소 지정 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                변위 주소 지정 활용                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 구조체 멤버 접근】                                             │
│  ──────────────────────────                                         │
│  struct Point {                                                     │
│      int x;     // offset 0                                         │
│      int y;     // offset 4                                         │
│      int z;     // offset 8                                         │
│  };                                                                 │
│                                                                     │
│  // C: p->y                                                         │
│  LDR R0, [R1, #4]       ; R1 = p, R0 = p->y                         │
│                                                                     │
│  【2. 지역 변수 접근】                                               │
│  ──────────────────────                                             │
│  void func() {                                                      │
│      int a;  // [RBP-4]                                             │
│      int b;  // [RBP-8]                                             │
│      int c;  // [RBP-12]                                            │
│  }                                                                  │
│                                                                     │
│  MOV EAX, [RBP-4]        ; a 로드                                   │
│  MOV EBX, [RBP-8]        ; b 로드                                   │
│  MOV [RBP-12], EAX       ; c 저장                                   │
│                                                                     │
│  【3. 함수 인자 접근】                                               │
│  ──────────────────────                                             │
│  // x86-64 (스택 전달)                                               │
│  MOV EAX, [RBP+16]       ; 첫 번째 인자                              │
│  MOV EBX, [RBP+24]       ; 두 번째 인자                              │
│                                                                     │
│  【4. 배열 요소 접근 (고정 인덱스)】                                 │
│  ────────────────────────────────────                               │
│  int arr[10];                                                       │
│  MOV EAX, [R1+0]         ; arr[0]                                   │
│  MOV EBX, [R1+4]         ; arr[1]                                   │
│  MOV ECX, [R1+8]         ; arr[2]                                   │
│                                                                     │
│  【5. 문자열 접근】                                                  │
│  ──────────────────                                                  │
│  char str[] = "Hello";                                              │
│  LDRB R0, [R1, #0]       ; str[0] = 'H'                             │
│  LDRB R0, [R1, #1]       ; str[1] = 'e'                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  변위 주소 지정 최적화:                                           │ │
│  │  ────────────────────────────                                     │ │
│  │  1. 작은 변위 사용 (명령어 길이 짧아짐)                           │ │
│  │  2. 자주 쓰는 멤버를 작은 오프셋에                                 │ │
│  │  3. 구조체 패딩 최소화                                             │ │
│  │  4. 캐시 라인 고려한 배치                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 변위 주소 지정 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                변위 주소 지정 × 타 과목 융합                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • 구조체 멤버: p->field → [base + offset]                         │
│  • 지역 변수: [RBP - offset]                                        │
│  • 인자 접근: [RBP + offset]                                        │
│  • 배열 상수 인덱스: [base + index*sizeof]                          │
│                                                                     │
│  【자료구조】                                                        │
│  • 구조체: 멤버 접근                                                 │
│  • 클래스: 멤버 변수                                                 │
│  • 배열: 고정 인덱스                                                 │
│  • 튜플: 원소 접근                                                   │
│                                                                     │
│  【운영체제】                                                        │
│  • 스택 프레임: 지역 변수, 인자                                      │
│  • TCB/PCB: 필드 접근                                               │
│  • 파일 구조체: 멤버 접근                                           │
│  • 버퍼 헤더: 필드 접근                                             │
│                                                                     │
│  【프로그래밍 언어】                                                 │
│  • C/C++: struct/class 멤버                                         │
│  • Java: 객체 필드                                                   │
│  • Python: 속성 접근                                                 │
│  • 레코드: 필드 접근                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스택 프레임 구조 (x86-64):                                       │ │
│  │  ──────────────────────────                                      │ │
│  │  RBP+16  │  arg1         │                                       │ │
│  │  RBP+8   │  return addr  │                                       │ │
│  │  RBP+0   │  saved RBP    │  ← RBP                                │ │
│  │  RBP-4   │  local1       │                                       │ │
│  │  RBP-8   │  local2       │                                       │ │
│  │  RBP-12  │  local3       │                                       │ │
│  │                                                               │ │
│  │  → 모든 변위 주소 지정으로 접근                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 변위 주소 지정 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                변위 주소 지정 분석                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조체 접근 예시】                                                │
│  ──────────────────────                                             │
│  struct Student {                                                   │
│      int id;           // offset 0                                  │
│      char name[32];    // offset 4                                  │
│      float gpa;        // offset 36                                 │
│  };                                                                 │
│                                                                     │
│  // C: student->gpa                                                 │
│  // ARM:                                                            │
│  LDR R0, [R1, #36]      ; R1 = student, R0 = gpa                    │
│                                                                     │
│  【objdump 분석】                                                    │
│  ──────────────────                                                  │
│  $ objdump -d program                                               │
│  mov eax, DWORD PTR [rbp-0x4]   ; 지역 변수                         │
│  mov eax, DWORD PTR [rbp+0x10]  ; 함수 인자                         │
│  mov eax, DWORD PTR [rdi+0x8]   ; 구조체 멤버                       │
│                                                                     │
│  【GCC 구조체 오프셋 확인】                                          │
│  ──────────────────────────                                         │
│  $ pahole struct_name                                               │
│  struct Student {                                                   │
│      int    id;                  /* 0  4 */                         │
│      char   name[32];            /* 4 32 */                         │
│      float  gpa;                 /* 36 4 */                         │
│  }                                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  디버깅 팁:                                                        │ │
│  │  ──────────                                                       │ │
│  │  (gdb) p &((struct Student*)0)->gpa  ; 오프셋 확인              │ │
│  │  (gdb) p/x $rbp-4                   ; 변위 계산된 주소          │ │
│  │  (gdb) x/x $rbp-4                   ; 해당 주소의 값            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 변위 주소 지정: EA = Base + Displacement
• 구조: [베이스 레지스터 + 오프셋]
• 활용: 구조체 멤버, 지역 변수, 함수 인자
• 장점: 고정 오프셋으로 간단한 계산
• 변위 크기: ISA마다 제한 (±2047 ~ ±32767)
```
