# 197. 로드/스토어 아키텍처 (Load/Store Architecture)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 접근은 오직 LOAD/STORE로만 수행
> 2. **가치**: 메모리 지연 분리, 캐시 최적화, RISC 핵심
> 3. **융합**: ARM, MIPS, RISC-V, 모든 RISC ISA

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

로드/스토어 아키텍처(Load/Store Architecture)는 **메모리 접근을 전용 LOAD/STORE 명령어로만 제한**하고, 모든 연산은 레지스터 간에만 수행하는 설계 방식이다. RISC 아키텍처의 핵심 원칙이다.

### 💡 비유: 창고와 작업대

로드/스토어 아키텍처는 **창고(메모리)와 작업대(레지스터)**와 같다. 물건(데이터)을 가지고 작업(연산)하려면 먼저 창고에서 작업대로 가져와야 한다(LOAD). 작업이 끝나면 다시 창고로 옮긴다(STORE). 작업대에서는 작업만 한다.

### 로드/스토어 아키텍처 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                로드/스토어 아키텍처 구조                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 원칙】                                                       │
│  ─────────────                                                      │
│  1. 메모리 접근: LOAD/STORE 명령어만                                 │
│  2. 연산: 레지스터 간만                                               │
│  3. 주소 계산: 별도 명령어                                           │
│  4. 장점: 파이프라인 단순화                                          │
│                                                                     │
│  【vs 메모리-레지스터 아키텍처】                                       │
│  ──────────────────────────────                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  아키텍처      메모리 연산       코드 예시                       │ │
│  │  ──────────  ─────────────────  ────────────────────────       │ │
│  │  Load/Store   레지스터 간만       LOAD R0, [a]                  │ │
│  │  (RISC)                         LOAD R1, [b]                  │ │
│  │                                ADD R2, R0, R1                 │ │
│  │                                STORE R2, [c]                  │ │
│  │                                                               │ │
│  │  Memory-Reg   메모리 직접 가능    MOV EAX, [a]                  │ │
│  │  (CISC)                         ADD EAX, [b]                  │ │
│  │                                MOV [c], EAX                   │ │
│  │                                                               │ │
│  │  → RISC: 메모리 접근 명시적, 파이프라인 단순                      │ │
│  │  → CISC: 메모리 접근 암시적, 파이프라인 복잡                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【명령어 분류】                                                     │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  유형          명령어           동작                              │ │
│  │  ──────────  ───────────────  ────────────────────────         │ │
│  │  Load         LDR/LW/LD        Mem[addr] → Reg                 │ │
│  │  Store        STR/SW/ST        Reg → Mem[addr]                 │ │
│  │  연산         ADD, SUB, ...    Reg ← Reg op Reg                 │ │
│  │  분기         B, BL, ...       PC 변경                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ISA별 로드/스토어 명령어】                                         │
│  ─────────────────────────────                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      로드              스토어            크기              │ │
│  │  ───────  ────────────────  ────────────────  ────────────      │ │
│  │  ARM      LDR Rd, [Rn]      STR Rd, [Rn]      B/H/W/D/Q        │ │
│  │  MIPS     lw rt, offset(rs) sw rt, offset(rs) B/H/W/D          │ │
│  │  RISC-V   ld rd, offset(rs1) sd rs2, offset(rs1) B/H/W/D       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  로드/스토어 장점:                                                  │ │
│  │  ──────────────────────                                             │ │
│  │  1. 파이프라인 단순화: 모든 명령어 비슷한 단계                      │ │
│  │  2. 메모리 지연 숨김: 연산 중 메모리 접근 없음                      │ │
│  │  3. 캐시 최적화: 명시적 접근 패턴                                   │ │
│  │  4. 레지스터 최적화: 컴파일러가 레지스터 할당                        │ │
│  │  5. 정렬 강제: 자연스러운 정렬                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 로드/스토어 명령어 상세

### 명령어 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                로드/스토어 명령어 상세                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ARM 로드/스토어】                                                 │
│  ──────────────────────                                             │
│  ; 기본 형식                                                         │
│  LDR R0, [R1]            ; R0 = Mem[R1]                             │
│  STR R0, [R1]            ; Mem[R1] = R0                             │
│                                                                     │
│  ; 변위 포함                                                         │
│  LDR R0, [R1, #4]        ; R0 = Mem[R1+4]                           │
│  STR R0, [R1, #8]        ; Mem[R1+8] = R0                           │
│                                                                     │
│  ; 인덱스 레지스터                                                   │
│  LDR R0, [R1, R2]        ; R0 = Mem[R1+R2]                          │
│  LDR R0, [R1, R2, LSL#2] ; R0 = Mem[R1+R2*4]                        │
│                                                                     │
│  ; 자동 증가/감소                                                    │
│  LDR R0, [R1], #4        ; R0 = Mem[R1], R1 += 4                    │
│  LDR R0, [R1, #4]!       ; R1 += 4, R0 = Mem[R1]                    │
│                                                                     │
│  ; 크기 변형                                                         │
│  LDRB R0, [R1]           ; byte (8비트)                             │
│  LDRH R0, [R1]           ; halfword (16비트)                        │
│  LDR R0, [R1]            ; word (32비트)                            │
│  LDRD R0, R1, [R2]       ; double (64비트)                          │
│                                                                     │
│  【MIPS 로드/스토어】                                                │
│  ──────────────────────                                             │
│  ; 기본 형식                                                         │
│  lw $t0, 0($t1)          ; $t0 = Mem[$t1+0]                         │
│  sw $t0, 4($t1)          ; Mem[$t1+4] = $t0                         │
│                                                                     │
│  ; 크기 변형                                                         │
│  lb $t0, 0($t1)          ; byte (부호 확장)                         │
│  lbu $t0, 0($t1)         ; byte (0 확장)                            │
│  lh $t0, 0($t1)          ; halfword (부호 확장)                      │
│  lw $t0, 0($t1)          ; word                                     │
│                                                                     │
│  【RISC-V 로드/스토어】                                              │
│  ──────────────────────────                                         │
│  ; 기본 형식                                                         │
│  lw x5, 0(x6)            ; x5 = Mem[x6+0]                           │
│  sw x5, 4(x6)            ; Mem[x6+4] = x5                           │
│                                                                     │
│  ; 크기 변형                                                         │
│  lb x5, 0(x6)            ; byte (부호 확장)                         │
│  lbu x5, 0(x6)           ; byte (0 확장)                            │
│  lh x5, 0(x6)            ; halfword (부호 확장)                      │
│  lw x5, 0(x6)            ; word                                     │
│  ld x5, 0(x6)            ; double (64비트)                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  로드/스토어 패턴:                                                  │ │
│  │  ──────────────────────                                            │ │
│  │  // a = b + c (모두 메모리에 있음)                                  │ │
│  │  LDR R0, [b]           ; b를 레지스터로                           │ │
│  │  LDR R1, [c]           ; c를 레지스터로                           │ │
│  │  ADD R2, R0, R1        ; 레지스터에서 연산                         │ │
│  │  STR R2, [a]           ; 결과를 메모리로                          │ │
│  │                                                               │ │
│  │  // a++ (메모리 직접 증가 안 됨)                                    │ │
│  │  LDR R0, [a]           ; 로드                                      │ │
│  │  ADD R0, R0, #1        ; 증가                                      │ │
│  │  STR R0, [a]           ; 스토어                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 로드/스토어 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                로드/스토어 분석                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【배열 합계 함수 (ARM)】                                            │
│  ──────────────────────────                                         │
│  ; int sum_array(int *arr, int n)                                   │
│  sum_array:                                                         │
│      MOV R2, #0            ; sum = 0                                │
│  loop:                                                              │
│      CMP R1, #0            ; n == 0?                                │
│      BEQ done                                                       │
│      LDR R3, [R0], #4      ; 로드 + 포인터 증가                      │
│      ADD R2, R2, R3        ; sum += *arr                            │
│      SUB R1, R1, #1        ; n--                                    │
│      B loop                                                         │
│  done:                                                              │
│      MOV R0, R2            ; return sum                             │
│      BX LR                                                          │
│                                                                     │
│  【MIPS 버전】                                                       │
│  ──────────────────                                                  │
│  sum_array:                                                         │
│      li $t0, 0              # sum = 0                               │
│  loop:                                                              │
│      beq $a1, $zero, done   # if (n == 0) done                      │
│      lw $t1, 0($a0)         # load *arr                             │
│      add $t0, $t0, $t1      # sum += *arr                           │
│      addi $a0, $a0, 4       # arr++                                 │
│      addi $a1, $a1, -1      # n--                                   │
│      j loop                                                          │
│  done:                                                              │
│      move $v0, $t0          # return sum                            │
│      jr $ra                                                          │
│                                                                     │
│  【RISC-V 버전】                                                     │
│  ──────────────────────                                             │
│  sum_array:                                                         │
│      li t0, 0               # sum = 0                               │
│  loop:                                                              │
│      beq a1, zero, done     # if (n == 0) done                      │
│      lw t1, 0(a0)           # load *arr                             │
│      add t0, t0, t1         # sum += *arr                           │
│      addi a0, a0, 4         # arr++                                 │
│      addi a1, a1, -1        # n--                                   │
│      j loop                                                          │
│  done:                                                              │
│      mv a0, t0              # return sum                            │
│      ret                                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  성능 분석:                                                          │ │
│  │  ──────────                                                         │ │
│  │  $ perf stat -e cycles,instructions,cache-misses ./program         │ │
│  │  cycles: 1,234,567,890                                             │ │
│  │  instructions: 987,654,321                                         │ │
│  │  cache-misses: 12,345                                               │ │
│  │                                                               │ │
│  │  로드/스토어 비율 확인                                               │ │
│  │  $ perf stat -e ld_retired, st_retired ./program                  │ │
│  │  ld_retired: 40%                                                   │ │
│  │  st_retired: 15%                                                   │ │
│  │  → 로드가 스토어보다 많음                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
