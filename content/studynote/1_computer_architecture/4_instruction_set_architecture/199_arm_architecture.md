# 199. ARM 아키텍처 (ARM Architecture)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 1985년 Acorn부터 현재까지 이어진 RISC ISA
> 2. **가치**: 모바일/임베디드 지배, 에너지 효율, Apple Silicon
> 3. **융합**: 스마트폰, IoT, 클라우드 서버, big.LITTLE

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

ARM(Advanced RISC Machine) 아키텍처는 **1985년 Acorn Computers에서 개발된 RISC ISA**로, 에너지 효율성으로 모바일/임베디드 시장을 지배하며 최근 데스크톱/서버로 확장하고 있다.

### 💡 비유: 경량 자전거 vs 대형 트럭

ARM은 **경량 자전거**와 같다. 가볍고(단순 명령어), 연비가 좋고(에너지 효율), 좁은 곳도 다닌다(임베디드). x86은 대형 트럭이다. 무겁고 복잡하지만 많은 짐을 한 번에 나른다. 최근 ARM도 전기 트럭(Apple M1)으로 진화했다.

## Ⅱ. ARM 명령어 집합

### 명령어 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                ARM 명령어 집합                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데이터 이동】                                                     │
│  ──────────────────                                                  │
│  MOV R0, R1          ; R0 = R1                                      │
│  MOV R0, #10         ; R0 = 10                                      │
│  LDR R0, [R1]        ; R0 = Mem[R1]                                 │
│  STR R0, [R1]        ; Mem[R1] = R0                                 │
│  LDR R0, [R1, #4]    ; R0 = Mem[R1+4]                               │
│  LDR R0, [R1], #4    ; R0 = Mem[R1], R1 += 4                        │
│  LDR R0, [R1, R2]    ; R0 = Mem[R1+R2]                              │
│                                                                     │
│  【산술】                                                            │
│  ──────────────────                                                  │
│  ADD R0, R1, R2      ; R0 = R1 + R2                                 │
│  ADD R0, R1, #10     ; R0 = R1 + 10                                 │
│  SUB R0, R1, R2      ; R0 = R1 - R2                                 │
│  MUL R0, R1, R2      ; R0 = R1 * R2 (하위 32비트)                   │
│  UMULL R0, R1, R2, R3 ; R1:R0 = R2 * R3 (64비트)                    │
│  SDIV R0, R1, R2     ; R0 = R1 / R2 (부호 있음)                     │
│                                                                     │
│  【논리】                                                            │
│  ──────────────────                                                  │
│  AND R0, R1, R2      ; R0 = R1 & R2                                 │
│  ORR R0, R1, R2      ; R0 = R1 | R2                                 │
│  EOR R0, R1, R2      ; R0 = R1 ^ R2                                 │
│  BIC R0, R1, R2      ; R0 = R1 & ~R2 (비트 클리어)                  │
│  MVN R0, R1          ; R0 = ~R1                                     │
│                                                                     │
│  【비교】                                                            │
│  ──────────────────                                                  │
│  CMP R0, R1          ; R0 - R1 (플래그만)                           │
│  CMN R0, R1          ; R0 + R1 (플래그만)                           │
│  TST R0, R1          ; R0 & R1 (플래그만)                           │
│  TEQ R0, R1          ; R0 ^ R1 (플래그만)                           │
│                                                                     │
│  【분기】                                                            │
│  ──────────────────                                                  │
│  B label             ; 무조건 분기                                   │
│  BEQ label           ; Z=1이면 분기                                  │
│  BNE label           ; Z=0이면 분기                                  │
│  BLT label           ; N≠V면 분기 (부호 있음 <)                      │
│  BLE label           ; Z=1 또는 N≠V                                 │
│  BL label            ; LR = PC+4, 분기                              │
│  BX LR               ; 복귀                                          │
│                                                                     │
│  【조건부 실행 (ARM 모드)】                                          │
│  ────────────────────────────────                                   │
│  CMP R0, #0                                                         │
│  MOVEQ R0, #1        ; R0==0이면 R0=1                               │
│  MOVNE R0, #2        ; R0!=0이면 R0=2                               │
│  ADDGT R0, R0, #1    ; R0>0이면 R0++                                │
│  ; 분기 없이 조건부 실행!                                            │
│                                                                     │
│  【Thumb 명령어 (16비트)】                                           │
│  ──────────────────────────────                                     │
│  MOV R0, R1          ; 16비트 버전                                   │
│  ADD R0, #10         ; 16비트 버전                                   │
│  ; 코드 크기 30% 감소                                                │
│  ; ARM/Thumb 혼합 가능                                               │
│                                                                     │
│  【NEON SIMD】                                                       │
│  ──────────────────                                                  │
│  VADD.F32 Q0, Q1, Q2  ; 4개 float 병렬 덧셈                         │
│  VMLA.F32 Q0, Q1, Q2  ; Q0 += Q1 * Q2                               │
│  VLD1.32 {Q0}, [R0]   ; 128비트 로드                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ARM 명령어 인코딩 (32비트):                                        │ │
│  │  ──────────────────────────                                        │ │
│  │  ┌────┬────┬────┬────┬────────────────────────┐                │ │
│  │  │cond│ 00 │ I  │opcode│ S │ Rn │ Rd │operand2│                │ │
│  │  └────┴────┴────┴────┴───┴────┴────┴─────────┘                │ │
│  │  31   28   25   24  21  20  19  16  15  12  11   0             │ │
│  │                                                               │ │
│  │  cond: 조건부 실행 (EQ, NE, GT, LE, ...)                       │ │
│  │  → 모든 명령어가 조건부 실행 가능!                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### ARM 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ARM 분석                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ARM 어셈블리 분석】                                               │
│  ──────────────────────────                                         │
│  // int factorial(int n)                                            │
│  factorial:                                                         │
│      CMP R0, #1           ; n <= 1?                                 │
│      MOVLE R0, #1         ; return 1                                │
│      BXLE LR              ; 조건부 복귀                              │
│      PUSH {R4, LR}        ; 레지스터 저장                            │
│      MOV R4, R0           ; n 저장                                   │
│      SUB R0, R0, #1       ; n - 1                                    │
│      BL factorial         ; 재귀 호출                                │
│      MUL R0, R4, R0       ; n * factorial(n-1)                       │
│      POP {R4, PC}         ; 복귀                                     │
│                                                                     │
│  【AArch64 버전】                                                    │
│  ──────────────────                                                  │
│  factorial:                                                         │
│      CMP W0, #1                                                      │
│      B.LE .Lreturn1                                                  │
│      STP X29, X30, [SP, #-16]!                                      │
│      MOV X29, SP                                                    │
│      SUB W0, W0, #1                                                 │
│      BL factorial                                                   │
│      ADD SP, SP, #16                                                │
│      ; MUL 결과는 W0에                                              │
│      RET                                                            │
│  .Lreturn1:                                                         │
│      MOV W0, #1                                                     │
│      RET                                                            │
│                                                                     │
│  【GDB 디버깅 (ARM)】                                                │
│  ──────────────────────                                              │
│  $ gdb ./program                                                    │
│  (gdb) set architecture arm                                         │
│  (gdb) info registers                                               │
│  (gdb) p/x $r0                                                      │
│  (gdb) p/x $cpsr          ; 상태 레지스터                            │
│  (gdb) x/10i $pc          ; 명령어 덤프                              │
│                                                                     │
│  【QEMU ARM 에뮬레이션】                                             │
│  ──────────────────────────                                         │
│  $ qemu-arm -g 1234 ./program                                      │
│  $ qemu-system-aarch64 -M virt -cpu cortex-a57                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Apple Silicon 성능:                                               │ │
│  │  ──────────────────                                                │ │
│  │  M3 Max (2023):                                                   │ │

│  │                                                               │ │
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• ARM: 1985년부터 이어진 RISC ISA

• 시장: 모바일 99%, 서버 증가 중
• 미래: ARMv9, Apple Silicon, 클라우드
```
