## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RISC-V(리스크 파이브)는 UC 버클리에서 개발을 시작한, 특정 기업의 소유가 아닌 완전한 **무료 개방형 명령어 집합 구조(Open Source ISA)**다.
> 2. **가치**: ARM이나 x86처럼 막대한 라이선스 비용과 설계 제약 없이, 누구나 마음대로 칩을 설계하고 수정할 수 있는 자유를 제공하여 **반도체 하드웨어의 '리눅스(Linux)' 혁명**을 이끌고 있다.
> 3. **융합**: 고정된 40개의 기본 정수 명령어(RV32I) 모듈에 곱셈, 부동소수점, 벡터 등의 확장 모듈을 레고 블록처럼 선택적으로 조립(Modular Design)할 수 있어, **초소형 IoT부터 AI 가속기, 슈퍼컴퓨터까지 완벽한 융합 설계**가 가능하다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: RISC-V(리스크 파이브)는 특정 기업의 소유가 아닌, 비영리 기구인 RISC-V 인터내셔널에서 관리하는 완전한 **무료 개방형 명령어 집합 구조(Open Source ISA)**다. 누구나 로열티 지불 없이 이 규격에 기반한 프로세서를 설계하고 제조할 수 있는 하드웨어의 자유로운 도화지다.

- **필요성**: RISC-V는 반도체 시장의 **'특정 기업 독점(ARM, x86) 탈피'와 '하드웨어 설계의 민주화'**를 위해 반드시 필요하다. 기존 상용 아키텍처는 막대한 라이선스 비용과 더불어 사용자가 함부로 명령어를 추가하거나 수정할 수 없는 폐쇄적인 구조였으나, RISC-V는 40여 개의 기본 명령어(RV32I) 모듈에 필요한 기능만을 선택적으로 조립(Modular)할 수 있는 유연성을 제공한다. 이는 중소 스타트업도 적은 비용으로 AI, 보안, IoT 등 특정 도메인에 최적화된 **커스텀 가속기(Custom Extensions)**를 융합한 독자 칩을 개발할 수 있게 하며, 하드웨어 소스코드가 투명하게 공개됨으로써 국가 안보 및 보안 무결성을 확보할 수 있는 핵심적인 기술 자립의 근간이 된다.
- **💡 비유**: RISC-V는 컴퓨터 아키텍처의 '오픈소스 레시피 북'이다. "세상에서 가장 빠르고 깔끔한 햄버거 레시피를 무료로 풉니다. 이대로 만들어 팔든, 패티를 10장 넣어서 팔든 아무도 돈 내라고 안 하고 터치 안 합니다"라는 선언이다. 레시피를 마음대로 변주하여 나만의 시그니처 요리(전용 칩)를 만들 수 있는 무한한 자유를 준다.
- **등장 배경**: 
  2010년대, 인텔(x86)과 ARM이 반도체 ISA 시장을 독점하며 수백억 원의 라이선스 비용을 요구하자, 학계와 스타트업들은 칩 하나 만들기 너무 힘들다고 절망했다. 이때 RISC 아키텍처의 창시자인 UC 버클리의 데이비드 패터슨(David Patterson) 교수를 주축으로 한 연구진이 빡쳐서 선언했다. **"인류의 발전을 위해 아무도 소유할 수 없는 5번째 RISC 설계도를 만들자!"** 그렇게 2010년 RISC-V가 탄생했고, 이는 곧이어 구글, 삼성, 엔비디아 같은 거인들마저 자사 칩에 도입하게 만드는 거대한 눈덩이가 되었다.

레거시를 다 쳐내고 가장 본질적인 40개의 명령어만 남긴 RISC-V의 베이스 모듈을 시각화하면 다음과 같다.

```text
  ┌────────────────────────────────────────────────────────────────────────┐
  │         RISC-V의 극단적 미니멀리즘: RV32I 베이스 명령어 셋             │
  ├────────────────────────────────────────────────────────────────────────┤
  │                                                                        │
  │   [ 인텔 x86 / ARM의 설계 ]                                            │
  │     - 1,000개가 넘는 명령어들. 쓰든 안 쓰든 칩 안에 다 구워 넣어야 함. │
  │     - 디코더가 너무 뚱뚱하고 무거움. 라이선스 비용도 엄청남.           │
  │                                                                        │
  │   [ RISC-V의 모듈형 설계 (RV32I Base) ]                                │
  │     - 컴퓨터가 튜링 완전성을 갖기 위한 **최소한의 명령어 딱 40개!**    │
  │     - "이것만 넣으면 무조건 컴퓨터로 작동한다. 나머진 네 맘대로 해라!" │
  │     ──▶ 칩 면적이 바늘구멍만큼 작아짐. (IoT 센서의 구세주)             │
  └────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 기존 아키텍처는 'All-or-Nothing'이었다. ARM 코어를 쓰려면 내가 안 쓰는 기능까지 몽땅 칩에 넣고 라이선스비를 내야 했다. 하지만 RISC-V는 **모듈화(Modularity)**의 정수다. 가장 기초적인 정수 연산 40개(RV32I)만 기본으로 강제하고, 곱셈/나눗셈(M), 단정밀도 부동소수점(F), 압축 명령어(C) 등은 설계자가 레고 블록처럼 '끼워도 그만, 빼도 그만'이다. 이 극강의 유연성 덕분에 100원짜리 시계 칩부터 수천만 원짜리 AI 칩까지 똑같은 언어로 말할 수 있게 되었다.

- **📢 섹션 요약 비유**: RISC-V의 모듈 설계는 '피자 도우만 팔기'와 같습니다. 기본적으로 빵(RV32I)은 무료로 줍니다. 페퍼로니를 올리든(M확장), 파인애플을 올리든(F확장), 아니면 직접 잡은 생선을 올리든(사용자 커스텀 확장) 요리사 마음입니다. 기존 프랜차이즈가 무조건 토핑이 다 올라간 비싼 피자(ARM, x86)만 강매하던 것과는 완전히 다른 철학입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 (오픈소스 실리콘을 지탱하는 문법)

RISC-V는 기존 RISC들의 실수(예: 분기 지연 슬롯, 상태 레지스터)를 철저히 연구하고 반면교사 삼아 만들어진 가장 깨끗한 도화지다.

| 구성 요소 | 기술적 특징 | 아키텍처적 가치 | 비유 |
|:---|:---|:---|:---|
| **Base Integer ISA (I)**| 40개의 기초 명령어 (RV32I / RV64I) | 튜링 완전성을 달성하는 **초소형 면적 코어 설계** | 레고의 초록색 기본 판 |
| **Standard Extensions** | M(곱셈), A(원자적 연산), F/D(소수점) | 필요에 따라 칩의 **기능을 선택적으로 확장** | 레고 부품 세트 |
| **Custom Extensions** | 사용자 정의 명령어(Opcode) 공간 개방 | AI 가속, 암호화 등 **기업별 비밀 무기 융합 공간**| 3D 프린터로 뽑은 커스텀 조각 |
| **No Condition Codes** | 상태 레지스터(Flags)를 과감히 삭제 | 플래그 병목 해소로 **슈퍼스칼라 병렬성 극대화** | 일기장(Flags) 찢어버리기 |

---

### 심층 동작 원리: "상태 레지스터(Flags)의 폐지와 통합 분기"

x86이나 ARM은 연산을 한 뒤 상태 레지스터에 기록(Zero, Sign 등)하고, 그걸 보고 점프(JUMP)했다. RISC-V는 이 거추장스러운 과정을 하나로 융합했다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │         RISC-V의 우아함: 비교와 분기의 일체화 (Compare & Branch)        │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │   [ ARM / x86의 분기 방식 (2단계) ]                                     │
  │     1. CMP R1, R2  ──▶ ALU가 뺄셈 후 Flags 레지스터 세팅 (병목!)        │
  │     2. BEQ TARGET  ──▶ Flags 레지스터를 읽어서 같으면 점프              │
  │                                                                         │
  │   [ RISC-V의 분기 방식 (1단계 일체화) ]                                 │
  │     1. BEQ R1, R2, TARGET  ──▶ "R1과 R2가 같으면 TARGET으로 뛰어라!"    │
  │                                                                         │
  │ * 혁명: 모든 명령어가 참조하던 중앙 집중식 'Flags' 장부를 태워버렸다!   │
  │   ──▶ 파이프라인의 데이터 의존성(Data Hazard)이 마법처럼 사라진다.      │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** CPU 설계자들이 골머리를 앓던 병목이 바로 'Flags(상태 레지스터)'였다. A연산도 Flags를 고치고 B연산도 Flags를 고치려고 하니 파이프라인에서 교통사고가 났다. RISC-V 설계자들은 "그럼 Flags를 없애면 되잖아!"라는 미친 발상을 실현했다. `BEQ`(Branch if Equal), `BLT`(Branch if Less Than)처럼 레지스터 두 개를 주고 바로 비교해서 뛰게 만들었다. 덧셈기(ALU) 대신 아주 단순한 비교기(Comparator)를 분기 회로에 융합함으로써, 하드웨어가 꼬일 여지를 원천 차단한 천재적인 설계다.

- **📢 섹션 요약 비유**: 상태 레지스터 폐지는 '중앙 게시판 없애기'와 같습니다. 예전엔 직원들이 일할 때마다 중앙 게시판(Flags)에 결과를 적고 다른 직원이 그걸 읽어야 해서 줄이 엄청 길었습니다. 지금은 게시판을 떼어버리고 두 직원이 그냥 현장에서 눈빛 교환(레지스터 간 비교) 한 번으로 다음 할 일(점프)을 결정하니 속도가 훨씬 빨라집니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 심층 기술 비교: RISC-V vs ARM

모바일의 현재 지배자(ARM)와 차기 패권을 노리는 도전자(RISC-V)의 대결이다.

| 비교 항목 | ARM (Advanced RISC Machine) | RISC-V | 아키텍처 판단 포인트 |
|:---|:---|:---|:---|
| **소유권 및 비용** | ARM 홀딩스 소유 (수십억 라이선스 비용)| **오픈소스 (무료, 누구나 사용 가능)**| 비즈니스 모델의 개방성 |
| **명령어 셋 크기** | 매우 방대함 (수십 년 치 누적) | **극도로 단순함 (베이스 40개)** | 디코더 복잡도 및 칩 면적 |
| **커스텀 명령어** | **절대 불가 (ARM이 허락 안 함)** | **완전 자유 (나만의 명령어 융합 가능)**| 도메인 특화(DSA) 칩 설계 난이도 |
| **생태계 (Ecosystem)**| 완성형 (모바일, OS, 툴체인 완벽) | 성장 중 (Linux 지원은 끝남, 앱 부족)| 소프트웨어 파워 (컴파일러, OS) |
| **비유** | 애플 iOS (폐쇄적이지만 완벽함) | **구글 안드로이드 (개방적, 무한 변형)**| 통제냐 자유냐의 철학적 선택 |

### 과목 융합 관점

- **시스템 반도체 (Domain Specific Architecture, DSA)**: 무어의 법칙이 끝나면서, 범용 CPU의 클럭을 올리는 것은 한계에 달했다. 이제는 AI, 암호화, 자율주행 등 특정 작업만 기가 막히게 잘하는 전용 칩(DSA)을 만드는 시대다. ARM은 내 맘대로 명령어를 추가할 수 없지만, RISC-V는 **사용자 정의 명령어 공간(Custom Opcode Space)**이 표준으로 비워져 있다. 엔비디아나 웨스턴디지털이 자사 컨트롤러의 뼈대를 전부 RISC-V로 갈아엎은 이유가 바로 이 '마음대로 개조할 수 있는 자유' 때문이다.
- **오픈소스 생태계 (Linux & GNU Toolchain)**: 하드웨어가 오픈소스라 한들 소프트웨어가 없으면 돌덩이다. RISC-V 커뮤니티는 칩을 만들기도 전인 2010년대 중반부터 GCC(컴파일러), Glibc, Linux Kernel에 RISC-V 패치를 융합하기 시작했다. 하드웨어와 소프트웨어 커뮤니티의 완벽한 앙상블 덕분에, 지금은 내가 설계한 RISC-V 코어를 FPGA에 굽기만 하면 1분 만에 리눅스 OS가 부팅되는 기적을 맛볼 수 있다.

```text
  ┌──────────────────────────────────────────────────────────────────────────┐
  │         아키텍처의 기적: 16비트 압축 명령어(C-Extension)의 융합          │
  ├──────────────────────────────────────────────────────────────────────────┤
  │                                                                          │
  │   [ 문제의식 ] : 32비트 고정 길이는 코드가 뚱뚱해서 L1 캐시가 터진다!    │
  │                                                                          │
  │   [ RISC-V의 천재적 해결책: 'RVC' 모듈 융합 ]                            │
  │     1. 가장 자주 쓰이는 명령어(예: 0을 더하거나, 레지스터 1개만 쓸 때)를 │
  │        '16비트'로 압축하는 규칙을 표준안에 넣었다.                       │
  │     2. 디코더가 16비트를 읽으면 0.1ns 만에 32비트 원본으로 팽창시킨다.   │
  │                                                                          │
  │ * 위력: ARM Thumb보다 더 우아하게, x86 수준의 코드 압축률을 달성했다!    │
  └──────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** RISC의 영원한 숙제인 '코드 비대화(Code Bloat)'를 해결한 방식이다. ARM은 16비트 모드(Thumb)와 32비트 모드(ARM)를 스위칭하는 상태 레지스터를 두고 두 개의 세계를 오갔다(매우 복잡함). 하지만 RISC-V는 모드 스위칭 따위는 없다. 그냥 16비트짜리 짧은 명령어를 32비트 사이에 자연스럽게 섞어(Mix) 쓴다. CPU 앞단의 프리디코더(Pre-decoder)가 16비트 명령어를 마시는 즉시 32비트 표준 포맷으로 뻥튀기(Expand)해 파이프라인에 던져주기 때문에 코어 내부는 100% 32비트로 깔끔하게 돌아간다.

- **📢 섹션 요약 비유**: 압축 명령어 확장은 '동결건조 블록 국'과 같습니다. 가방(캐시 메모리)에 넣고 다닐 때는 부피가 아주 작은 16비트 동결건조 블록이지만, 뜨거운 물(디코더)에 넣는 순간 원래의 32비트짜리 완벽한 국으로 팽창해서 요리사(파이프라인)가 맛있게 먹을 수 있게 됩니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 스타트업의 초저전력 IoT 칩 설계**: 
  **상황**: 배터리 하나로 10년을 버텨야 하는 스마트 농업용 토양 센서 칩을 10만 원 미만의 예산으로 자체 설계해야 함.
  **판단**: "RV32IC 베이스의 극단적 다이어트 설계"다. 엔지니어는 오픈소스 저장소에서 32비트 정수 연산(I)과 압축 명령어(C)만 딱 두 개 융합된 RISC-V 코어(예: PicoRV32) 도면을 다운받는다. 곱셈기나 부동소수점, 운영체제(MMU) 관련 회로를 모조리 들어내 칩 면적을 먼지 크기로 줄인다. 라이선스 비용 0원으로 설계된 이 칩은 타사 ARM Cortex-M 칩보다 전기를 1/3만 먹으며 성공적으로 양산된다.

2. **시나리오 — 구글 TPU(Tensor Processing Unit) 내부 컨트롤러**:
  **상황**: 구글이 AI 행렬 연산을 가속하는 거대한 칩(TPU)을 만들었는데, 이 TPU 내부에 수많은 연산 유닛들을 통제할 작은 지휘관(Microcontroller) 코어가 필요함.
  **판단**: "ISA 커스텀 융합의 자유도 활용"이다. ARM 코어를 쓰면 라이선스 문제로 구글 마음대로 명령어를 뜯어고칠 수 없다. 구글 하드웨어 팀은 RISC-V를 채택하여, **사용자 정의 명령어(Custom Extension)** 공간에 행렬 데이터를 직접 쏴주는 특수 DMA(직접 메모리 접근) 명령어를 박아 넣는다. 완벽하게 구글 AI 생태계만을 위한 커스텀 코어가 탄생한 순간이다.

```text
  ┌────────────────────────────────────────────────────────────────────────┐
  │         마이크로아키텍처 합성: RISC-V 칩셋 설계 라이프사이클           │
  ├────────────────────────────────────────────────────────────────────────┤
  │                                                                        │
  │   [ 타겟: 인공지능 가속용 칩을 설계하자! ]                             │
  │                                                                        │
  │   1단계: RV64I (64비트 기본 정수명령어) 블록을 가져온다.               │
  │   2단계: F, D 모듈을 붙인다 (실수 연산 필요).                          │
  │   3단계: V (Vector) 확장 모듈을 붙인다 (SIMD 행렬 연산 가속).          │
  │   4단계: X (Custom) 확장을 직접 설계해 나만의 텐서 명령어를 융합한다.  │
  │                                                                        │
  │  결과물: [ RV64IMAFDVX ] 라는 문자열 자체가 이 칩의 유전자(DNA)다!     │
  └────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** RISC-V 생태계에서 칩을 설계하는 것은 '레고 조립'과 같다. 칩의 능력을 나타내는 알파벳 문자열(ISA String)이 존재한다. I(정수), M(곱셈), A(원자성), F/D(부동소수점), C(압축), V(벡터) 등이다. 컴파일러(GCC)에 저 문자열만 `-march=rv64imac`처럼 넘겨주면, 컴파일러가 알아서 "아, 이 칩은 곱셈기는 있는데 벡터는 없구나"라고 판단하여 완벽하게 최적화된 기계어를 뱉어낸다. 하드웨어의 모듈화가 소프트웨어 컴파일러와 1:1로 융합된 궁극의 우아함이다.

### 도입 체크리스트
- **Privilege Levels (M, S, U)**: 칩이 RTOS(실시간 OS)나 리눅스를 돌려야 한다면, 단순한 머신 모드(Machine)를 넘어 슈퍼바이저(Supervisor)와 유저(User) 권한 모드를 하드웨어적으로 완벽히 융합했는가?
- **Ecosystem Fragmentation (파편화 방어)**: 커스텀 확장을 너무 많이 넣어서, 표준 리눅스나 컴파일러가 이 칩을 이해하지 못해 나 혼자만 써야 하는 '나 홀로 칩(Fragmentation)'이 되어버리지는 않았는가? (안드로이드 파편화와 같은 문제)

### 안티패턴
- **비표준 확장에 메인 로직 묶어두기**: 내가 만든 RISC-V 칩 전용으로 아주 기가 막힌 커스텀 확장 명령어를 만들었다 치자. 그런데 소프트웨어 커널이나 주요 라이브러리의 뼈대를 모조리 그 비표준 명령어로 도배해 버리는 행위. 나중에 칩을 업그레이드하거나 범용 RISC-V 코드로 이식하려 할 때, 코드가 전혀 호환되지 않아 소프트웨어 자산을 통째로 쓰레기통에 버려야 하는 끔찍한 벤더 락인(Vendor Lock-in)의 함정이다.

- **📢 섹션 요약 비유**: 비표준 확장 남용은 '나만의 비밀어 쓰기'와 같습니다. 나랑 내 친구(우리 칩) 둘이서만 아는 비밀어(커스텀 명령어)로 일기를 잔뜩 썼는데, 나중에 다른 사람(범용 생태계)에게 그 일기를 보여주거나 출판하려고 하니 아무도 읽을 수가 없어 결국 전부 처음부터 다시 써야 하는 바보 같은 짓입니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 레거시 상용 아키텍처 (ARM/x86) | RISC-V 오픈 아키텍처 융합 | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 수백억 원의 초기 ISA 라이선스 비용 | **라이선스 비용 0원 (무료)** | 스타트업의 칩 개발 진입장벽 **완전 파괴** |
| **정량** | 안 쓰는 기능까지 포함된 무거운 코어 | **필요한 모듈(RV32I)만 조립한 코어** | 칩 다이(Die) 면적 및 전력 **수십 % 절감** |
| **정성** | 블랙박스로 감춰진 하드웨어 백도어 위험 | 투명하게 공개된 RTL (Chisel/Verilog) 소스 | 국방/보안 분야의 **하드웨어 무결성** 확보 |

### 미래 전망
- **오토모티브(Automotive) 차량용 반도체의 지배**: 전기차와 자율주행차에는 수백 개의 마이크로컨트롤러(MCU)가 들어간다. 테슬라 등 빅테크들은 벤더 종속성을 끊고 원가를 낮추기 위해, 차세대 차량용 제어 칩의 뼈대로 일제히 RISC-V를 융합 도입하고 있다.
- **국가 안보와 반도체 자립**: 미-중 패권 경쟁 속에서, 특정 국가(미국, 영국)의 소유인 x86이나 ARM 라이선스가 차단될 것을 두려워한 중국, 유럽, 인도 등은 국가적 차원에서 아예 주인이 없는 스위스 기반의 RISC-V를 국가 표준 아키텍처로 육성하고 있다.

### 참고 표준
- **RISC-V ISA Specification**: 더할 나위 없이 깔끔하게 쓰인 명령어 집합 명세서. 볼륨 I(비특권 User-Level)과 볼륨 II(특권 Privileged)로 완벽히 분리되어 하드웨어 엔지니어들의 성서로 불린다.
- **RoCC (Rocket Custom Coprocessor)**: UC 버클리에서 만든 칩 설계 언어인 Chisel로 RISC-V 코어 옆에 커스텀 가속기(AI 연산기 등)를 아주 쉽게 융합해 붙일 수 있는 표준 인터페이스.

"독점과 폐쇄"의 실리콘 제국에 "공유와 자유"의 깃발을 꽂으며 반도체 민주화를 이룩한, 하드웨어의 리눅스 'RISC-V'의 진화 로드맵은 다음과 같다.

```text
  ┌─────────────────────────────────────────────────────────────────────────────────────────────┐
  │         자유와 공유: RISC-V 아키텍처 진화 로드맵                                            │
  ├─────────────────────────────────────────────────────────────────────────────────────────────┤
  │                                                                                             │
  │   [1단계: 대학 연구실의 반란]  [2단계: IoT와 임베디드 정복]  [3단계: 서버와 클라우드 진격]  │
  │                                                                                             │
  │   ARM 비싸서 우리가 만든다 ──▶ 웨스턴디지털, 엔비디아 탑재 ──▶ 고성능 멀티코어 칩셋 등장    │
  │  (가장 깨끗한 RISC의 탄생)   (무료인데 가볍고 커스텀 되네?)  (ARM의 왕좌를 직접 겨냥하다)   │
  │ "누구나 칩을 만들게 하라"    "우리 SSD 컨트롤러 다 이걸로 해" "스마트폰 다음은 데이터센터다"│
  └─────────────────────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** RISC-V는 기술의 우월함을 넘어선 '철학의 승리'다. 1단계: 버클리 연구실에서 교육용으로 시작된 이 작은 불씨는, 가장 더러운 레거시가 1도 없는 '순백의 아키텍처'라는 찬사를 받았다. 2단계: 라이선스비 0원과 내 맘대로 개조할 수 있다는 점에 홀린 전 세계 기업들이 IoT, SSD 컨트롤러, AI 가속기 구석구석에 ARM을 뽑아내고 RISC-V를 심었다. 3단계: 이제 RISC-V는 변방을 넘어 리눅스를 쾌적하게 돌리고, 안드로이드를 올리며, 스마트폰 AP와 클라우드 데이터센터 서버의 거대한 심장으로 커나가고 있다. 반도체 생태계의 판이 뒤집히는 거대한 지각 변동의 한가운데 서 있다.

- **📢 섹션 요약 비유**: RISC-V의 등장은 '위키백과(Wikipedia)의 탄생'과 같습니다. 예전엔 똑똑한 학자들이 모인 브리태니커 백과사전(ARM, x86)을 비싼 돈 주고 사야 했지만, 이제는 전 세계 누구나 무료로 지식을 추가하고 고칠 수 있는 위키백과(RISC-V)가 생겼습니다. 처음엔 엉성해 보였지만 전 세계인의 지성이 모이자 결국 가장 거대하고 정확한 인류의 표준 지식창고가 된 것과 완벽히 같은 이치입니다.

|:---|:---|
| **Open Source Hardware**| RISC-V의 영혼. 설계도가 인터넷에 다 공개되어 있어서, 돈 한 푼 안 내고도 나만의 CPU를 붕어빵처럼 찍어낼 수 있다. |

| **Custom Extension (확장)** | 기업들이 RISC-V를 사랑하는 이유. 기본 명령어 외에 나만의 비밀 무기(AI, 암호화)를 하드웨어 회로로 직결할 공간이 열려 있다. |
| **ARM Architecture** | RISC-V가 타도해야 할 1순위 독점 제국. 명령어 철학은 둘 다 순수 RISC로 거의 똑같지만, 폐쇄적이냐 개방적이냐의 차이만 있다. |
| **Fragmentation (파편화)** | RISC-V 생태계의 유일한 적. 누구나 자기 맘대로 커스텀을 만들다 보니, 공통 소프트웨어(OS)가 안 돌아갈까 봐 떨고 있다. |
