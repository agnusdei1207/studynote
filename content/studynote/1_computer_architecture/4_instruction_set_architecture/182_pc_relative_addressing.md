# PC 상대 주소 지정 (PC-Relative Addressing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: PC 상대 주소 지정(PC-Relative Addressing)은 **프로그램 카운터(PC)의 현재 값에 명령어의 변위(Offset)를 더하여** 이동할 목표 주소를 계산하는 방식이다.
> 2. **가치**: 루프(Loop)나 조건문(If-else) 등 프로그램 내부의 짧은 점프(Branch)를 처리할 때, 절대 주소 대신 상대적인 이동 거리만 기록함으로써 명령어 길이를 단축하고 메모리를 절약한다.
> 3. **융합**: 운영체제의 동적 링킹 및 공유 라이브러리 환경에서 필수적인 **위치 독립 코드 (PIC: Position Independent Code)**를 생성하는 핵심 하드웨어 메커니즘이다.

+++

## Ⅰ. PC 상대 주소 지정의 개요

### 1. 정의
- 다음에 실행할 명령어의 주소를 계산할 때, 현재 실행 중인 명령어의 위치(PC)를 기준으로 앞으로 또는 뒤로 몇 칸 이동할지(변위)를 지정하는 방식이다: $EA = (PC) + A$.

### 2. 등장 배경: '코드의 덩어리 이동'
- 프로그램 안에서 `if`문을 만나 코드를 건너뛰거나 루프를 돌기 위해 돌아갈 때, 그 거리는 보통 몇 줄 차이 나지 않는다. 굳이 32비트 전체 주소를 명령어에 다 쓸 필요 없이, "여기서부터 10칸 앞으로 가라"고만 쓰면 명령어 길이를 크게 줄일 수 있다. 또한 이 방식을 쓰면 코드를 메모리의 다른 곳으로 통째로 복사해도 "10칸 앞"이라는 사실은 변하지 않으므로 코드가 정상 작동한다.

### 3. 💡 비유: '보드게임의 주사위 이동'
- PC 상대 주소 지정은 **'부루마불 게임에서 현재 내 말(PC)의 위치를 기준으로 주사위 숫자(변위)만큼 앞으로 이동하는 것'**과 같다.
- "무조건 서울(절대 주소)로 가라"가 아니라, "지금 있는 곳에서 3칸 뒤로(상대 주소) 가라"는 식의 규칙이다.

- **📢 섹션 요약 비유**: 내 현재 위치를 세상의 중심으로 삼고, 앞뒤로 움직일 거리만 재는 자기중심적 내비게이션입니다.

+++

## Ⅱ. 메커니즘 및 동작 원리 (Deep Dive)

명령어가 인출(Fetch)되는 순간, PC는 이미 다음 명령어 주소를 가리키고 있음을 명심해야 한다.

### 1. 명령어 구조 및 유효 주소 (EA)
- **명령어 형식**: `[ Opcode (Branch) | Offset A ]`
- **유효 주소 (EA)**: $EA = PC_{next} + A$ (A는 보통 부호 있는 정수)
- **특징**: $A$가 양수면 순방향(Forward) 점프, 음수면 역방향(Backward, 루프 등) 점프를 수행한다.

### 2. 동작 프로세스 시각화 (ASCII)
```text
    [ Program Memory (Code Segment) ]
    
    Addr 100: [ ADD R1, R2 ]
    Addr 104: [ JUMP +16   ] ──┐ (현재 PC는 108을 가리킴)
    Addr 108: [ SUB R3, R1 ]   │
    ...                        │
    Addr 120: [ Target Cmd ] ◀─┘ (EA = 108 + 16 = 124)
    
    * 주의: 파이프라인 구조상 JUMP 명령어를 해독할 때 PC는 이미 그 다음 명령어 주소(108)로 증가해 있다. 따라서 실제 타겟은 124번지가 된다.
```

### 3. 심층 동작 원리: 부호 있는 변위 (Signed Offset)
- PC 상대 주소의 변위값 $A$는 2의 보수(2's Complement) 형태를 취한다. 예를 들어 8비트 오프셋이라면 $+127$부터 $-128$까지 점프할 수 있다. 분기 거리가 이 범위를 넘어서면 컴파일러는 PC 상대 지정 대신 긴 절대 주소 점프(Long Jump) 명령어를 사용한다.

- **📢 섹션 요약 비유**: "앞으로 3보 가시오", "뒤로 2보 가시오" 처럼 내 발밑을 0점(Zero point)으로 삼는 상대 좌표계입니다.

## Ⅲ. PC 상대 지정 vs 베이스 레지스터 지정

| 비교 항목 | PC 상대 지정 (PC-Relative) | 베이스 지정 (Base Register) |
|:---|:---|:---|
| **기준 레지스터** | **PC (프로그램 카운터)** | 범용 레지스터 또는 베이스 레지스터 |
| **주된 목적** | 프로그램 내부의 **제어 흐름 분기(Branch)** | 데이터 구조(배열/구조체) 접근, 전체 재배치 |
| **변위값의 성격** | **부호 있는 수 (앞뒤 양방향)** | 보통 양수 (시작점부터의 거리) |
| **동적 변경** | 매 명령어마다 PC가 변하므로 자동 갱신됨 | OS나 프로그램이 명시적으로 값을 변경할 때만 |

- **📢 섹션 요약 비유**: 걷는 걸음마다 기준이 바뀌는 만보계(PC 상대)와, 시작할 때 한 번 말뚝을 박아두는 줄자(베이스)의 차이입니다.

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 실무 시나리오: 위치 독립 코드 (Position Independent Code) 컴파일
- **상황**: 리눅스 환경에서 `.so` (Shared Object) 공유 라이브러리를 빌드함 (`gcc -fPIC`).
- **판단**: 공유 라이브러리는 여러 프로그램이 동시에 로드할 수 있어, 메모리상에서 고정된 주소를 가질 수 없다. 컴파일러는 함수 내부의 분기문과 데이터 참조를 전부 **PC 상대 주소 지정** 방식으로 컴파일한다. 이렇게 만들어진 바이너리는 메모리의 `0x1000`에 올라가든 `0x8000`에 올라가든 명령어 내부의 상대적 거리가 변하지 않으므로 문제없이 실행된다.

### 2. 기술사적 인사이트: 분기 예측(Branch Prediction)과의 충돌 방지
- **인사이트**: PC 상대 주소 지정은 명령어를 읽어야만 목적지를 알 수 있다. 깊은 파이프라인을 가진 현대 CPU에서는 목적지를 계산하는 단 1~2사이클의 지연조차 페널티가 된다. 기술사는 하드웨어 설계 시 분기 목적지 버퍼(BTB: Branch Target Buffer)를 두어, PC 상대 주소 연산이 끝나기 전에 이전 실행 기록을 바탕으로 다음 타겟 주소를 미리 파이프라인에 밀어 넣는 정교한 예측 메커니즘을 필수적으로 고려해야 한다.

- **📢 섹션 요약 비유**: 다음 코너에서 꺾을지 말지 표지판(PC 연산)을 보고 결정하면 늦으니, 예전 기억(BTB)을 떠올려 미리 핸들을 돌려두는 레이싱 기술과 같습니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **분기 명령어 최적화**: JUMP 명령어의 크기를 극적으로 줄여 Instruction Cache 공간을 절약.
- **시스템 보안 강화**: ASLR(Address Space Layout Randomization) 적용 시 코드가 임의의 주소에 배치되어도 완벽한 실행 보장.

### 2. 미래 전망
모든 최신 아키텍처(ARM64, RISC-V 등)는 코드의 유연성을 위해 절대 주소 점프를 최대한 지양하고 PC 상대 점프를 기본으로 채택하고 있다. 특히 RISC-V는 명령어 집합 자체가 위치 독립 코드를 매우 자연스럽게 생성하도록 설계되었다. PC 상대 주소 지정은 소프트웨어가 물리적 하드웨어 주소라는 족쇄에서 벗어나 자유롭게 이동할 수 있게 해주는 가장 우아한 수학적 트릭이다.

- **📢 섹션 요약 비유**: 땅바닥에 묶이지 않고 구름 위를 떠다니며 어디서든 자신의 모습을 유지할 수 있는 자유로운 소프트웨어의 날개입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **프로그램 카운터 (PC)**: 상대 지정의 기준점이 되는 핵심 레지스터.
- **분기 명령어 (Branch Instruction)**: PC 상대 지정이 주로 쓰이는 명령어.
- **위치 독립 코드 (PIC)**: 이 방식을 통해 구현되는 소프트웨어 아키텍처.
- **ASLR**: PIC를 기반으로 성립하는 운영체제 메모리 보안 기법.
