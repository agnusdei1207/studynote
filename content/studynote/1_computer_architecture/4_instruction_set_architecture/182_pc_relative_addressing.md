# PC 상대 주소 지정 (PC-Relative Addressing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: PC 상대 주소 지정 (PC-Relative Addressing)은 현재 프로그램 카운터(PC)의 값에 명령어에 포함된 변위(Displacement)를 더하여 실제 메모리 주소인 **유효 주소 (Effective Address)**를 결정하는 방식이다.
> 2. **가치**: 프로그램이 메모리의 어느 위치에 로드되더라도 코드 간의 상대적 거리만 알면 정상 동작하는 **위치 독립 코드 (PIC, Position Independent Code)**를 가능케 하며, 짧은 변수 비트만으로 근처의 코드로 빠르게 분기(Jump)할 수 있게 한다.
> 3. **융합**: 주로 조건부/무조건 분기 명령어(Branch)에서 실행 흐름을 제어하기 위해 사용되며, 현대의 공유 라이브러리(DLL, SO) 시스템을 지탱하는 핵심 기술이다.

---

## Ⅰ. PC 상대 주소 지정의 정의 및 메커니즘

### 1. 정의
- 유효 주소 $EA$를 구하기 위해 현재 명령어의 다음 주소를 가리키는 $PC$ 값과 명령어 내의 주소 필드 $A$(변위)를 합산하는 방식이다. ($EA = PC + A$)

### 2. 주요 특징: '현재 위치 중심의 길 찾기'
- **코드 재배치성**: 절대 주소를 쓰지 않으므로, 프로그램 전체 덩어리가 메모리 내에서 이동해도 명령어 내부의 변수($A$)를 고칠 필요가 없다.
- **짧은 주소 필드**: 대부분의 분기는 현재 실행 지점 근처(Loop, If문 등)에서 일어나므로, 8~16비트 정도의 짧은 변위만으로도 충분히 목적지를 가리킬 수 있다.
- **부호 있는 정수 사용**: 변위 $A$는 양수(앞으로 점프)와 음수(뒤로 점프)를 모두 표현할 수 있는 2의 보수 형식을 취한다.

### 3. 💡 비유: '현재 위치 기반 내비게이션'
- PC 상대 주소 지정은 **'지도에 "현재 있는 곳에서 300m 앞의 편의점으로 가시오"라고 적어둔 것'**과 같다.
- 내가 서울에 있든 부산에 있든(메모리 로드 위치), 현재 내 위치만 알면 추가 지시 사항(변위)을 따라 목적지에 정확히 도착할 수 있다.

---

## Ⅱ. PC 상대 주소 지정의 동작 구조 (Deep Dive)

### 1. 연산 과정
1. **Fetch**: 명령어를 인출한다. 이때 PC는 이미 다음 명령어 주소로 증가(PC = PC + 4)한 상태이다.
2. **Decode**: 명령어 내의 변위 $A$를 추출한다.
3. **Add**: 하드웨어 가산기가 현재의 **갱신된 $PC$** 값과 변위 $A$를 더한다.
4. **Update PC**: 계산된 유효 주소 $EA$를 다시 PC에 로드하여 다음 실행 지점을 바꾼다. (분기 발생)

### 2. ASCII 구조 다이어그램
```text
    [ CPU ]                             [ Main Memory ]
    ┌──────────────────────────┐        ┌──────────────────┐
    │  Program Counter (PC)    │ ──┐     │                  │
    └──────────────────────────┘   │     │  ...             │
                                   │     ├──────────────────┤
    [ Instruction ]                ├────▶│  Target Address  │ ◀── (Branch Target)
    ┌────────┬────────┬────────┐   │     │  (Jump Point)    │
    │ Opcode │  ...   │ A: +12 │ ──┘     ├──────────────────┤
    └────────┴────────┴────────┘         │  ...             │
                                         └──────────────────┘
    (EA = 갱신된 PC + 12)
```

- **📢 섹션 요약 비유**: "지금 읽고 있는 페이지에서 5페이지 뒤로 넘겨!"라고 하는 것과 같습니다. 책이 책꽂이 어디에 꽂혀 있든 상관없이 항상 정확한 다음 내용을 찾을 수 있습니다.

## Ⅲ. PC 상대 주소 지정의 장점과 한계

| 구분 | 장점 (Pros) | 한계점 (Cons) |
|:---|:---|:---|
| **유연성** | **위치 독립 코드 (PIC)** 생성 가능 | 변위 비트 수에 따라 점프 가능한 **거리 제한** 존재 |
| **효율성** | 명령어 비트 수 절약 (짧은 변수 사용) | 주소 계산을 위한 가산기 오버헤드 발생 |
| **안정성** | 공유 라이브러리의 동적 로딩 최적화 | PC 갱신 타이밍에 따른 정밀한 하드웨어 설계 필요 |

- **📢 섹션 요약 비유**: 근거리 이동(루프, 조건문)에는 최고의 기동성을 발휘하지만, 아주 먼 곳(다른 메모리 영역)으로 가려면 비행기(절대 주소)를 타야 하는 것과 같습니다.

---

## Ⅳ. 실무 적용 및 아키텍처 인사이트 (Strategy)

### 1. 실무 시나리오: 동적 링크 라이브러리 (DLL) 최적화
- **상황**: 여러 실행 파일이 하나의 `libc.so`를 공유 메모리에 올려서 사용함.
- **판단**: 각 프로그램마다 라이브러리가 로드되는 가상 주소가 다를 수 있다. 이때 라이브러리 내부의 모든 분기 명령어는 **PC 상대 주소 지정**을 사용해야 한다. 그래야만 어떤 주소 공간에서도 코드의 논리적 연결이 깨지지 않고 '공유'의 이점을 누릴 수 있다.

### 2. 기술사적 인사이트: 분기 예측과 PC의 관계
- 기술사라면 PC 상대 주소가 **분기 예측 (Branch Prediction)**에 미치는 영향을 논해야 한다. PC 값이 결정되어야 유효 주소가 나오기 때문에, 파이프라인 구조에서는 실제 계산 전까지 다음 명령어를 인출하기 어렵다. 이를 해결하기 위해 **BTB (Branch Target Buffer)**에 'PC + 변위'의 결과를 미리 캐싱하여 주소 계산 단계를 뛰어넘는 기법이 현대 CPU 설계의 핵심임을 강조해야 한다.

- **📢 섹션 요약 비조**: 정답(주소)을 미리 계산해두는 똑똑한 비서(분기 예측)가 있어야만, 상대 경로를 찾는 시간차를 극복하고 고성능을 유지할 수 있습니다.

---

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **메모리 활용도 극대화**: 위치 독립 코드를 통한 대규모 라이브러리 공유 실현.
- **코드 크기 슬림화**: 32비트 절대 주소 대신 8/16비트 변위 사용으로 메모리 점유 감소.

### 2. 미래 전망
앞으로는 보안을 위해 메모리 구조를 실시간으로 섞는 **ASLR** 기술이 더욱 고도화되면서, 이에 대응하기 위한 하드웨어 수준의 PC 상대 주소 지정 엔진의 정밀도가 더욱 중요해질 것이다. 또한, 가상화 환경에서의 주소 오버헤드를 줄이기 위한 다단계 상대 주소 지정 기술이 발전할 것이다. PC 상대 주소 지정은 소프트웨어에 '자유'를, 하드웨어에 '효율'을 선사한 아키텍처의 혁명이다.

- **📢 섹션 요약 비유**: 내 발자취(PC)를 기준으로 세상을 바라보는 지혜가, 가장 복잡한 컴퓨터 세상을 가장 단순하게 제어하는 힘이 됩니다.
