# 180. 베이스 레지스터 주소 지정 (Base Register Addressing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기준 주소(베이스)를 레지스터에 저장하고 변위를 더하는 방식
> 2. **가치**: 세그먼트, 스택 프레임, 객체 데이터 접근
> 3. **융합**: 메모리 관리, 프로세스 격리, 재배치

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

베이스 레지스터 주소 지정(Base Register Addressing)은 **베이스 레지스터가 기준 주소를 저장하고, 여기에 변위를 더해 유효 주소를 계산**하는 방식이다. 변위 주소 지정의 한 형태로, 베이스 레지스터의 역할을 강조한 명칭이다.

### 💡 비유: 건물 출발점

베이스 레지스터 주소 지정은 **건물 입구에서 시작**하는 것과 같다. "입구에서 5m 오른쪽"처럼 기준점(베이스)에서 상대적 위치(변위)로 찾아간다. 건물이 이사해도 내부 위치는 같다.

### 베이스 레지스터 주소 지정 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                베이스 레지스터 주소 지정 구조                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│                                                                     │
│  EA = [Base Register] + Displacement                                │
│  유효 주소 = 베이스 레지스터 값 + 변위                               │
│                                                                     │
│  【베이스 레지스터 역할】                                            │
│  ──────────────────────────                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  용도              베이스 레지스터    변위                       │ │
│  │  ────────────────  ──────────────────  ──────────────          │ │
│  │  스택 프레임        RBP/FP            지역 변수 오프셋          │ │
│  │  데이터 세그먼트    DS/데이터 베이스   변수 오프셋              │ │
│  │  구조체/객체        객체 포인터        멤버 오프셋              │ │
│  │  배열              배열 시작 주소     요소 오프셋               │ │
│  │  코드              CS/코드 베이스     명령어 오프셋             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【x86 세그먼트 + 베이스】                                           │
│  ──────────────────────────                                         │
│                                                                     │
│  논리 주소 = 세그먼트:오프셋                                         │
│  물리 주소 = 세그먼트 베이스 + 오프셋                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  세그먼트      베이스         용도                               │ │
│  │  ──────────  ─────────────  ────────────                       │ │
│  │  CS (Code)   코드 세그먼트    명령어                            │ │
│  │  DS (Data)   데이터 세그먼트   전역 변수                        │ │
│  │  SS (Stack)  스택 세그먼트     스택                             │ │
│  │  ES/FS/GS    추가 세그먼트     특수 용도                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ARM 베이스 레지스터】                                             │
│  ──────────────────────────                                         │
│  LDR R0, [R1, #offset]   ; R1 = 베이스, offset = 변위               │
│  LDR R0, [FP, #-4]       ; FP = 프레임 포인터 (베이스)              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  베이스 레지스터 장점:                                            │ │
│  │  ──────────────────────────                                      │ │
│  │  • 위치 독립 코드: 베이스만 바꾸면 전체 이동                     │ │
│  │  • 재배치 용이: 로더가 베이스만 수정                              │ │
│  │  • 세그먼트 관리: 격리된 메모리 영역                              │ │
│  │  • 모듈식 코드: 라이브러리 로드                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 베이스 레지스터 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                베이스 레지스터 활용                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 스택 프레임 (Frame Pointer)】                                  │
│  ────────────────────────────────────                               │
│  push rbp                                                          │
│  mov rbp, rsp           ; RBP = 스택 프레임 베이스                  │
│  sub rsp, 16            ; 지역 변수 공간                            │
│                                                                     │
│  mov [rbp-4], eax       ; local1 저장                               │
│  mov eax, [rbp-8]       ; local2 로드                               │
│  mov eax, [rbp+16]      ; arg1 로드                                 │
│                                                                     │
│  【2. 구조체/객체 접근】                                             │
│  ──────────────────────                                             │
│  class Object {                                                     │
│      int field1;     // offset 0                                    │
│      int field2;     // offset 4                                    │
│  };                                                                 │
│                                                                     │
│  mov rax, [rdi]         ; this->field1                              │
│  mov rbx, [rdi+4]       ; this->field2                              │
│                                                                     │
│  【3. 세그먼트 기반 주소 지정 (x86 16비트)】                          │
│  ───────────────────────────────────────                            │
│  MOV AX, [DS:0x1000]    ; DS 베이스 + 0x1000                        │
│  MOV AX, [SS:0xFFFC]    ; SS 베이스 + 0xFFFC                        │
│                                                                     │
│  【4. GOT(Global Offset Table) 접근】                               │
│  ───────────────────────────────────                                │
│  ; x86-64 RIP-relative (GOT 베이스)                                 │
│  mov rax, [rip + offset@GOT]  ; 전역 변수                           │
│  call [rip + func@GOT]        ; 함수 포인터                         │
│                                                                     │
│  【5. 라이브러리 로딩 (베이스 재배치)】                               │
│  ────────────────────────────────────                               │
│  ; 공유 라이브러리가 임의 주소에 로드                                │
│  ; 베이스 = 로드 주소                                                │
│  ; 모든 심볼 = 베이스 + 오프셋                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  베이스 레지스터 vs 인덱스 레지스터:                               │ │
│  │  ──────────────────────────────────                              │ │
│  │  베이스: 고정 기준점, 구조체/프레임 시작                          │ │
│  │  인덱스: 변동 인덱스, 배열 요소 접근                              │ │
│  │                                                               │ │
│  │  혼합: [base + index*scale + disp]                              │ │
│  │  예: MOV EAX, [RBP + RCX*4 + 8]                                 │ │
│  │      RBP=베이스, RCX=인덱스, 8=변위                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 베이스 레지스터 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                베이스 레지스터 × 타 과목 융합                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【운영체제】                                                        │
│  • 세그멘테이션: 세그먼트 베이스 레지스터                            │
│  • 페이징: 페이지 테이블 베이스 (CR3)                                │
│  • 프로세스: 코드/데이터/스택 베이스                                 │
│  • 공유 메모리: 매핑 베이스                                          │
│                                                                     │
│  【컴파일러】                                                        │
│  • 스택 프레임: FP/RBP 베이스                                        │
│  • 객체 접근: this 포인터 베이스                                     │
│  • 전역 변수: GOT 베이스                                             │
│  • 문자열 리터럴: 문자열 풀 베이스                                   │
│                                                                     │
│  【링커/로더】                                                       │
│  • 재배치: 베이스 주소 수정                                          │
│  • 공유 라이브러리: 로드 베이스                                      │
│  • PIC: 위치 독립 코드                                              │
│  • ASLR: 무작위 베이스                                               │
│                                                                     │
│  【컴퓨터 구조】                                                     │
│  • x86 세그먼트: CS, DS, SS, ES                                     │
│  • ARM: 워드 정렬 베이스                                             │
│  • 메모리 보호: 베이스 + 한계                                        │
│  • TLB: 페이지 테이블 베이스 캐시                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86-64 주소 지정 형식:                                           │ │
│  │  ──────────────────────────                                      │ │
│  │  EA = Base + (Index * Scale) + Displacement                      │ │
│  │                                                               │ │
│  │  구성요소:                                                        │ │
│  │  • Base: 베이스 레지스터 (RAX~R15)                               │ │
│  │  • Index: 인덱스 레지스터 (RAX~R15, SP 제외)                     │ │
│  │  • Scale: 1, 2, 4, 8                                            │ │
│  │  • Displacement: 0, 8, 32비트 부호있는 정수                       │ │
│  │                                                               │ │
│  │  예: MOV EAX, [RBX + RCX*4 + 0x10]                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 베이스 레지스터 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                베이스 레지스터 분석                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【함수 프롤로그/에필로그】                                          │
│  ──────────────────────────                                         │
│  func:                                                              │
│      push rbp             ; 이전 프레임 저장                        │
│      mov rbp, rsp         ; 새 베이스 설정                          │
│      sub rsp, 32          ; 지역 변수 공간                          │
│      ; ... 함수 본문 ...                                            │
│      mov rsp, rbp         ; 스택 복원                               │
│      pop rbp              ; 이전 베이스 복원                        │
│      ret                                                           │
│                                                                     │
│  【GDB로 베이스 확인】                                               │
│  ──────────────────────────                                         │
│  (gdb) p/x $rbp           ; 현재 프레임 베이스                      │
│  (gdb) x/10x $rbp-32     ; 지역 변수                                │
│  (gdb) info frame         ; 프레임 정보                             │
│                                                                     │
│  【객체 멤버 접근】                                                  │
│  ──────────────────                                                  │
│  // C++ 메서드                                                       │
│  void Object::method() {                                            │
│      this->field = 10;    // this = 베이스                          │
│  }                                                                  │
│                                                                     │
│  // x86-64 (this = RDI)                                             │
│  mov DWORD PTR [rdi], 10  ; this->field = 10                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  베이스 레지스터 선택 가이드:                                      │ │
│  │  ────────────────────────────                                     │ │
│  │  용도              권장 레지스터                                  │ │
│  │  ────────────────  ────────────────                              │ │
│  │  스택 프레임        RBP (x86-64), FP (ARM)                        │ │
│  │  객체 포인터        RDI (첫 인자), this                          │ │
│  │  배열 베이스        RAX~R15                                      │ │
│  │  버퍼              RSI/RDI (문자열 함수)                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 베이스 레지스터: 기준 주소를 저장하는 레지스터
• EA = Base + Displacement
• 용도: 스택 프레임, 세그먼트, 객체 접근
• 장점: 위치 독립, 재배치 용이, 격리
• 예시: RBP(스택), RDI(객체), CS/DS/SS(세그먼트)
```
