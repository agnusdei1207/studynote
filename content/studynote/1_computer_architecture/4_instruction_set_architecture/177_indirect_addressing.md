# 177. 간접 주소 지정 (Indirect Addressing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어의 주소 필드가 가리키는 위치에서 실제 주소를 찾는 방식
> 2. **가치**: 동적 주소 지정, 포인터 구현, 다단계 참조
> 3. **융합**: 연결 리스트, 다형성, 동적 메모리 할당

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

간접 주소 지정(Indirect Addressing)은 **명령어에 있는 주소가 실제 데이터의 주소가 아니라, 데이터 주소가 저장된 위치**를 가리키는 방식이다. 한 번의 간접 참조를 통해 실제 유효 주소를 얻는다.

### 💡 비유: 이리저리 물어보기

간접 주소 지정은 **"A씨한테 물어보세요"**와 같다. 주소를 직접 모르고, 아는 사람(A씨)에게 물어봐서 실제 주소를 얻는다. 한 단계를 더 거치지만, 유연하게 주소를 바꿀 수 있다.

### 간접 주소 지정 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                간접 주소 지정 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│                                                                     │
│  명령어:  ┌─────────────┬─────────────┐                             │
│          │   Opcode    │   @Addr     │  (@ = 간접 표시)            │
│          └─────────────┴─────────────┘                             │
│                    │                                                │
│                    ▼                                                │
│          ┌─────────────────────┐                                    │
│          │  메모리[Addr]       │  ← 포인터 (실제 주소 저장)         │
│          │  = 실제 데이터 주소 │                                    │
│          └─────────┬───────────┘                                    │
│                    │                                                │
│                    ▼                                                │
│          ┌─────────────────────┐                                    │
│          │  실제 데이터        │  ← 실제 데이터                     │
│          └─────────────────────┘                                    │
│                                                                     │
│  EA = Memory[Address Field]  (유효 주소 = 메모리[주소 필드])        │
│                                                                     │
│  【특징】                                                           │
│  ─────────                                                          │
│  • 2회 메모리 접근 필요                                              │
│  • 실행 속도 느림                                                    │
│  • 동적 주소 지정 가능                                               │
│  • 포인터 구현에 적합                                                │
│                                                                     │
│  【직접 vs 간접 비교】                                               │
│  ────────────────────────                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  방식     접근 횟수   유연성   용도                             │ │
│  │  ───────  ─────────  ───────  ────────────                    │ │
│  │  직접     1회        낮음    전역, 정적 변수                   │ │
│  │  간접     2회+       높음    포인터, 동적 데이터               │ │
│  │                                                               │ │
│  │  직접: MOV EAX, [0x1000]    ; 0x1000의 값                     │ │
│  │  간접: MOV EAX, [[0x1000]]  ; 0x1000이 가리키는 주소의 값      │ │
│  │        (x86은 직접 지원 안함, 레지스터 간접 사용)               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  메모리 간접 주소 지정 (Memory Indirect):                       │ │
│  │  ──────────────────────────────────                             │ │
│  │  • x86: 제한적 지원 (JMP [addr])                                │ │
│  │  • VAX: 완전 지원                                                │ │
│  │  • 대부분: 레지스터 간접으로 대체                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 간접 주소 지정 유형

### 유형 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                간접 주소 지정 유형                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 메모리 간접 (Memory Indirect)】                                │
│  ───────────────────────────────────                                │
│  EA = Mem[Address]                                                  │
│                                                                     │
│  예 (VAX):  MOV @1000, R0   ; Mem[Mem[1000]] → R0                   │
│  예 (x86):  JMP [0x1000]    ; Mem[0x1000]으로 점프                  │
│                                                                     │
│  【2. 레지스터 간접 (Register Indirect)】                            │
│  ─────────────────────────────────────                              │
│  EA = Reg                                                           │
│                                                                     │
│  예:  MOV EAX, [EBX]    ; Mem[EBX] → EAX                            │
│       LDR R0, [R1]      ; Mem[R1] → R0                              │
│                                                                     │
│  【3. 다중 간접 (Multilevel Indirect)】                              │
│  ───────────────────────────────────                                │
│  EA = Mem[Mem[Mem[...]]]                                            │
│                                                                     │
│  C 코드 예시:                                                        │
│  int a = 10;                                                        │
│  int *p = &a;                                                       │
│  int **pp = &p;                                                     │
│  int value = **pp;    // 2단계 간접                                  │
│                                                                     │
│  【4. 자동 증가/감소 (Auto-increment/decrement)】                    │
│  ────────────────────────────────────────                           │
│  LDR R0, [R1], #4     ; R0 = Mem[R1], R1 += 4                       │
│  LDR R0, [R1, #4]!    ; R1 += 4, R0 = Mem[R1]                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  간접 주소 지정 활용:                                             │ │
│  │  ────────────────────────────                                     │ │
│  │  • 포인터 변수                                                    │ │
│  │  • 연결 리스트 노드                                               │ │
│  │  • 함수 포인터                                                    │ │
│  │  • 가상 함수 테이블 (vtable)                                      │ │
│  │  • 동적 라이브러리 (GOT)                                          │ │
│  │  • 콜백 함수                                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 간접 주소 지정 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                간접 주소 지정 분석                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【연결 리스트 순회】                                                │
│  ──────────────────────                                             │
│  // C 코드                                                           │
│  struct Node { int data; struct Node *next; };                      │
│  void traverse(struct Node *head) {                                 │
│      while (head != NULL) {                                         │
│          printf("%d", head->data);                                  │
│          head = head->next;  // 간접 참조                            │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  // x86-64 어셈블리                                                 │
│  traverse:                                                          │
│      test rdi, rdi         ; head == NULL?                          │
│      je .done                                                      │
│  .loop:                                                             │
│      mov eax, [rdi]        ; head->data (간접)                      │
│      call print_int                                               │
│      mov rdi, [rdi+8]      ; head = head->next (간접)               │
│      test rdi, rdi                                                │
│      jne .loop                                                     │
│  .done:                                                            │
│      ret                                                           │
│                                                                     │
│  【함수 포인터 호출】                                                │
│  ──────────────────────                                             │
│  void (*callback)(int) = handler;                                   │
│  callback(42);             ; 간접 호출                               │
│                                                                     │
│  ; x86-64                                                           │
│  call [callback]           ; 메모리 간접 호출                        │
│  ; 또는                                                             │
│  mov rax, [callback]                                                │
│  call rax                  ; 레지스터 간접 호출                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  간접 주소 지정 디버깅:                                           │ │
│  │  ────────────────────────────                                     │ │
│  │  (gdb) p *ptr             ; ptr이 가리키는 값                    │ │
│  │  (gdb) p **pptr           ; 이중 포인터 역참조                   │ │
│  │  (gdb) x/x $rdi           ; 레지스터가 가리키는 메모리           │ │
│  │  (gdb) watch *ptr         ; ptr이 가리키는 값 변경 감시          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 간접 주소 지정: 주소 필드가 실제 주소를 가리킴
• EA = Memory[Address Field]
• 장점: 동적 주소, 포인터 구현, 유연성
• 단점: 2회 메모리 접근, 느린 실행
• 활용: 포인터, 연결 리스트, 함수 포인터, vtable
```
