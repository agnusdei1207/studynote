# 289. 다단계 페이지 테이블 (Multi-Level Page Table)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이지 테이블을 계층적으로 나눈 구조
> 2. **가치**: 메모리 절약, 희소 주소 공간 지원
> 3. **융합**: 2단계, 4단계, 5단계, TLB, Page Walk

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

다단계 페이지 테이블(Multi-Level Page Table)은 **페이지 테이블을 여러 단계로 나누어 메모리를 절약하는 구조**다. 사용되지 않는 주소 공간에 대한 페이지 테이블을 생성하지 않아 메모리를 절약한다.

### 💡 비유: 다단계 색인

다단계 페이지 테이블은 **다단계 색인**과 같다. 도서관에서 "과학-물리-양자역학-001"을 찾을 때, 1단계: 과학 섹션, 2단계: 물리 책장, 3단계: 양자역학 칸, 4단계: 001번 책. 각 단계마다 디렉토리가 있다. 과학 섹션이 없으면 그 아래는 색인도 없다.

### 다단계 페이지 테이블 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                다단계 페이지 테이블 개요                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【선형 vs 다단계】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  선형 페이지 테이블 (Linear):                                         │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │ PTE[0] │ PTE[1] │ PTE[2] │ ... │ PTE[N-2] │ PTE[N-1] │   │ │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │  → 모든 VPN에 대한 엔트리 필요                                        │ │
│  │  → 64비트에서 수백 GB 필요                                           │ │
│  │  → 사용 안 하는 공간도 할당                                           │ │
│  │                                                               │ │
│  │  다단계 페이지 테이블 (Multi-Level):                                   │ │
│  │                         ┌───────────┐                            │ │
│  │                         │ L1 Table  │ (Page Directory)           │ │
│  │                         └─────┬─────┘                            │ │
│  │                    ┌──────────┼──────────┐                       │ │
│  │                    ▼          ▼          ▼                       │ │
│  │               ┌────────┐ ┌────────┐ ┌────────┐                  │ │
│  │               │L2 Table│ │L2 Table│ │  NULL  │ (할당 안 됨)        │ │
│  │               └────┬───┘ └────┬───┘ └────────┘                  │ │
│  │                    │          │                                 │ │
│  │               ┌────┴───┐ ┌────┴───┐                            │ │
│  │               │ L3     │ │ L3     │                            │ │
│  │               └────────┘ └────────┘                            │ │
│  │  → 사용하는 주소만 테이블 할당                                         │ │
│  │  → 메모리 크게 절약                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【다단계의 필요성】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  64비트 가상 주소 (48비트 사용):                                       │ │
│  │  • 가상 주소 공간: 256TB                                             │ │
│  │  • 페이지 수: 64G                                                   │ │
│  │  • 선형 페이지 테이블: 64G × 8B = 512GB!                              │ │
│  │  → 선형으로는 불가능                                                  │ │
│  │  → 다단계로 메모리 절약 필수                                           │ │
│  │                                                               │ │
│  │  희소 주소 공간 (Sparse Address Space):                               │ │
│  │  • 실제 사용: 전체의 1% 미만                                          │ │
│  │  • 힙, 스택 사이 큰 빈 공간                                            │ │
│  │  • 다단계: 빈 공간에 대한 테이블 불필요                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 다단계 페이지 테이블 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                다단계 페이지 테이블 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86-64 4단계 페이지 테이블 (4-Level Paging)】                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  48비트 가상 주소 (4KB 페이지):                                        │ │
│  │  ┌─────────┬─────────┬─────────┬─────────┬────────────────┐     │ │
│  │  │ PML4    │ PDPT    │ PD      │ PT      │ Offset         │     │ │
│  │  │ (9비트)  │ (9비트)  │ (9비트)  │ (9비트)  │ (12비트)        │     │ │
│  │  └─────────┴─────────┴─────────┴─────────┴────────────────┘     │ │
│  │                                                               │ │
│  │  변환 과정:                                                          │ │
│  │                                                               │ │
│  │  CR3 ──→ PML4[PFN] ──→ PDPT[PFN] ──→ PD[PFN] ──→ PT[PFN]        │ │
│  │           (L1)          (L2)         (L3)        (L4)            │ │
│  │                                                        │         │ │
│  │                                                        ▼         │ │
│  │                                                    물리 페이지      │ │
│  │                                                               │ │
│  │  각 단계별:                                                          │ │
│  │  • PML4 (Page Map Level 4): 최상위, 512 엔트리                      │ │
│  │  • PDPT (Page Directory Pointer Table): 512 엔트리                 │ │
│  │  • PD (Page Directory): 512 엔트리                                  │ │
│  │  • PT (Page Table): 512 엔트리, 실제 PFN 저장                        │ │
│  │                                                               │ │
│  │  각 엔트리: 8바이트, 512 엔트리 = 4KB (페이지 크기)                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 절약 효과】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  예: 1GB 힙 사용 (가상 주소 공간 256TB 중)                              │ │
│  │                                                               │ │
│  │  선형 페이지 테이블:                                                   │ │
│  │  64G × 8B = 512GB (불가능!)                                         │ │
│  │                                                               │ │
│  │  4단계 페이지 테이블:                                                   │ │
│  │  • PML4: 1개 × 4KB = 4KB                                           │ │
│  │  • PDPT: 1개 × 4KB = 4KB                                           │ │
│  │  • PD: 1개 × 4KB = 4KB                                             │ │
│  │  • PT: 256K/512 ≈ 512개 × 4KB = 2MB                                │ │
│  │  → 총 약 2MB (선형 대비 25만 분의 1!)                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단계별 특징】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단계      이름                  크기        커버리지               │ │
│  │  ────      ────                  ────        ───────               │ │
│  │  1단계    PML4                  4KB         512GB                  │ │
│  │  2단계    PDPT                  4KB         1GB                    │ │
│  │  3단계    PD                    4KB         2MB                    │ │
│  │  4단계    PT                    4KB         4KB                    │ │
│  │                                                               │ │
│  │  5단계 (LA57): PML5 추가로 57비트 주소 지원                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 다단계 페이지 테이블 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                다단계 페이지 테이블 × 타 과목 융합                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • MMU: 다단계 Page Walk 수행                                        │
│  • TLB: Page Walk 결과 캐시                                          │
│  • Page Walker: 하드웨어 순회                                         │
│  • 지연: 단계별 메모리 접근                                             │
│                                                                     │
│  【운영체제】                                                          │
│  • 페이지 테이블 관리: 계층별 생성/삭제                                   │
│  • 희소 할당: 필요한 부분만 생성                                         │
│  • Copy-on-Write: 페이지 테이블 복사 최적화                              │
│  • Huge Page: 여러 단계 스킵                                          │
│                                                                     │
│  【성능】                                                              │
│  • TLB 미스 비용: 4번의 메모리 접근                                      │
│  • Page Walk Cache: 최근 Page Walk 캐시                               │
│  • Huge Page: 단계 축소 (4KB → 2MB → 1GB)                             │
│  • 메모리 절약 vs 접근 비용 트레이드오프                                    │
│                                                                     │
│  【아키텍처별 차이】                                                       │
│  • x86-64: 4단계 (PML4, PDPT, PD, PT)                                │
│  • ARM64: 3~4단계 (PGD, PUD, PMD, PTE)                               │
│  • RISC-V: 3단계 (Sv39) 또는 4단계 (Sv48)                              │
│  • LA57: 5단계 (57비트 가상 주소)                                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단계 수에 따른 트레이드오프:                                                          │ │
│  │  ──────────────────────                                              │ │
│  │  단계 수      메모리 절약     Page Walk 비용    지원 주소 공간           │ │
│  │  ──────       ──────────     ─────────────     ────────────            │ │
│  │  2단계        적음            2번 접근           32비트                  │ │
│  │  3단계        중간            3번 접근           39비트                  │ │
│  │  4단계        큼              4번 접근           48비트                  │ │
│  │  5단계        매우 큼          5번 접근           57비트                  │ │
│  │                                                               │ │
│  │  → 단계가 많을수록 메모리 절약 but Page Walk 비용 증가                    │ │
│  │  → TLB가 Page Walk 비용 상쇄                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  dTLB-load-misses: 500,000                                         │

│  → TLB 미스 = Page Walk 발생                                         │
│                                                                     │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 다단계 페이지 테이블: 페이지 테이블을 계층적으로 나눈 구조
• 목적: 메모리 절약, 희소 주소 공간 지원
• x86-64: 4단계 (PML4 → PDPT → PD → PT)
• ARM64: 3~4단계 (PGD → PUD → PMD → PTE)
• 메모리 절약: 사용하는 주소만 테이블 할당
• Page Walk: 다단계 순회, 단계별 메모리 접근
• 비용: TLB 미스 시 N번의 메모리 접근
• 해결: TLB, Huge Page, Page Walk Cache
• Huge Page: 2MB(3단계), 1GB(2단계)
• 5단계: LA57, 57비트 가상 주소 지원
```
