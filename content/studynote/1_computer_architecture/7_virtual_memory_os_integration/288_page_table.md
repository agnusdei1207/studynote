# 288. 페이지 테이블 (Page Table)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: VPN과 PFN 매핑 정보를 저장하는 테이블
> 2. **가치**: 가상-물리 주소 변환의 핵심 자료구조
> 3. **융합**: MMU, TLB, 다단계, 역페이지 테이블

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

페이지 테이블(Page Table)은 **가상 페이지 번호(VPN)와 물리 프레임 번호(PFN)의 매핑 정보를 저장하는 테이블**이다. 각 프로세스마다 하나씩 존재하며, MMU가 주소 변환 시 참조한다.

### 💡 비유: 도서관 위치록

페이지 테이블은 **도서관 위치록**과 같다. 분류번호(가상 주소)를 책장 위치(물리 주소)로 변환한다. "과학-물리-001" → "3층 B구역 5번 책장". 위치록이 있어야 분류번호로 책을 찾을 수 있다. 각 도서관(프로세스)마다 위치록이 다르다.

### 페이지 테이블 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 테이블 개요                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 테이블 구조】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  VPN  │    PTE (Page Table Entry)                              │ │
│  │  ──── │ ─────────────────────────────────────────────────────  │ │
│  │   0   │ PFN=5   │ P │ R │ W │ X │ D │ A │ U │ ...            │ │
│  │   1   │ PFN=2   │ P │ R │ W │ - │ D │ A │ U │ ...            │ │
│  │   2   │ PFN=9   │ P │ R │ - │ - │ D │ A │ S │ ...            │ │
│  │   3   │ PFN=-   │ 0 │ - │ - │ - │ - │ - │ - │ ... (디스크)    │ │
│  │  ...  │  ...    │...│...│...│...│...│...│...│                │ │
│  │  N-1  │ PFN=7   │ P │ R │ W │ X │ D │ A │ U │ ...            │ │
│  │                                                               │ │
│  │  PTE 구성:                                                          │ │
│  │  • PFN: Physical Frame Number (물리 프레임 번호)                    │ │
│  │  • P: Present bit (메모리에 존재 여부)                               │ │
│  │  • R/W/X: 읽기/쓰기/실행 권한                                        │ │
│  │  • D: Dirty bit (수정 여부)                                         │ │
│  │  • A: Accessed bit (접근 여부)                                      │ │
│  │  • U/S: User/Supervisor 모드                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 테이블 위치】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │   CPU                    MMU                   Memory           │ │
│  │    │                      │                       │              │ │
│  │    │  가상 주소           │                       │              │ │
│  │    │  VPN = 2            │                       │              │ │
│  │    │ ─────────────────→  │                       │              │ │
│  │    │                      │                       │              │ │
│  │    │                      │  Page Table Walk      │              │ │
│  │    │                      │ ─────────────────→    │              │ │
│  │    │                      │                       │              │ │
│  │    │                      │     ┌──────────────┐  │              │ │
│  │    │                      │     │ Page Table   │  │              │ │
│  │    │                      │     │ VPN 2 → PFN 9│  │              │ │
│  │    │                      │     └──────────────┘  │              │ │
│  │    │                      │                       │              │ │
│  │    │                      │  PFN = 9              │              │ │
│  │    │                      │ ←─────────────────    │              │ │
│  │    │                      │                       │              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 테이블 특징】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 프로세스별 존재: 각 프로세스마다 독립적                              │ │
│  │  • 메모리에 저장: RAM에 상주                                         │ │
│  │  • 크기: 가상 주소 공간 크기에 비례                                    │ │
│  │  • 참조: MMU가 TLB 미스 시 접근                                       │ │
│  │  • 관리: OS가 생성, 수정, 삭제                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 페이지 테이블 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 테이블 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 테이블 크기 계산】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  32비트 주소, 4KB 페이지:                                            │ │
│  │  • 가상 주소 공간: 4GB                                              │ │
│  │  • 페이지 수: 4GB / 4KB = 1M                                       │ │
│  │  • PTE 크기: 4바이트                                                │ │
│  │  • 페이지 테이블 크기: 1M × 4B = 4MB                                 │ │
│  │  → 프로세스마다 4MB 페이지 테이블 필요!                                 │ │
│  │                                                               │ │
│  │  64비트 주소, 4KB 페이지 (48비트 가상 주소 사용):                        │ │
│  │  • 가상 주소 공간: 256TB                                            │ │
│  │  • 페이지 수: 256TB / 4KB = 64G                                     │ │
│  │  • PTE 크기: 8바이트                                                │ │
│  │  • 페이지 테이블 크기: 64G × 8B = 512GB!                              │ │
│  │  → 선형 페이지 테이블 불가능 → 다단계 필요                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【PTE (Page Table Entry) 상세】                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  x86-64 PTE 구조 (64비트):                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │   비트   │  이름   │  설명                                   │   │ │
│  │  │   ────   │  ────   │  ────                                   │   │ │
│  │  │   0      │  P      │  Present (존재 여부)                     │   │ │
│  │  │   1      │  R/W    │  Read/Write (0=R, 1=W)                  │   │ │
│  │  │   2      │  U/S    │  User/Supervisor (0=S, 1=U)              │   │ │
│  │  │   3      │  PWT    │  Write-Through                          │   │ │
│  │  │   4      │  PCD    │  Cache Disable                          │   │ │
│  │  │   5      │  A      │  Accessed (읽힘)                         │   │ │
│  │  │   6      │  D      │  Dirty (수정됨)                          │   │ │
│  │  │   7      │  PS     │  Page Size (큰 페이지)                    │   │ │
│  │  │   8      │  G      │  Global (TLB 유지)                       │   │ │
│  │  │   9-11   │  AVL    │  Available (OS 사용)                     │   │ │
│  │  │   12-51  │  PFN    │  Physical Frame Number                  │   │ │
│  │  │   52-62  │  예약    │  Reserved                               │   │ │
│  │  │   63     │  NX     │  No Execute (실행 방지)                   │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 테이블 접근 과정】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. CPU가 가상 주소 생성: VA = [VPN | Offset]                        │ │
│  │  2. MMU가 TLB 조회:                                                 │ │
│  │     - Hit: PFN 즉시 반환 (1~2 클럭)                                  │ │
│  │     - Miss: 페이지 테이블 조회 (Page Table Walk)                      │ │
│  │  3. Page Table Walk:                                                │ │
│  │     - CR3(또는 TTBR)에서 페이지 테이블 베이스 읽기                      │ │
│  │     - VPN으로 PTE 찾기                                               │ │
│  │     - PTE에서 PFN 읽기                                               │ │
│  │     - (다단계면 여러 번 반복)                                          │ │
│  │  4. TLB 갱신: VPN→PFN 저장                                          │ │
│  │  5. 물리 주소 생성: PA = [PFN | Offset]                              │ │
│  │  6. 메모리 접근                                                      │ │
│  │                                                               │ │
│  │  → TLB Hit: 1~2 클럭                                               │ │
│  │  → TLB Miss + Page Table Walk: 10~100 클럭                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 페이지 테이블: VPN과 PFN 매핑 정보 저장
• PTE: 페이지 테이블 엔트리 (PFN + 권한 + 상태)
• 위치: 메모리에 저장, MMU가 참조
• 크기: 가상 주소 공간에 비례 (32비트: 4MB, 64비트: 수십 GB)
• 프로세스별: 각 프로세스마다 독립적 페이지 테이블
• 레지스터: CR3(x86), TTBR(ARM), SATP(RISC-V)
• 권한 비트: R/W/X, Present, Dirty, Accessed
• 보안: NX, U/S 비트로 메모리 보호
• 최적화: TLB, 다단계, Huge Page
• 현대 OS: 다단계 페이지 테이블 사용
```
