# 283. 물리 주소와 논리 주소 (Physical & Logical Address)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 보는 주소(논리) vs 실제 메모리 주소(물리)
> 2. **가치**: 메모리 추상화, 프로세스 격리
> 3. **융합**: MMU 변환, 페이지, 세그먼트

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

논리 주소(Logical Address, 가상 주소)는 **CPU가 생성하는 주소**이고, 물리 주소(Physical Address)는 **실제 메모리 하드웨어의 주소**다. MMU가 논리 주소를 물리 주소로 변환한다.

### 💡 비유: 도서관 분류번호 vs 책장 위치

논리 주소는 **도서관 분류번호**다. "과학-물리-양자역학-001" (논리적 위치). 물리 주소는 **실제 책장 위치**다. "3층 B구역 5번째 책장 2단" (물리적 위치). 사서(MMU)가 분류번호를 책장 위치로 변환한다.

### 주소 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                물리/논리 주소 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【논리 주소 (Logical/Virtual Address)】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정의: CPU가 프로그램 실행 중 생성하는 주소                              │ │
│  │                                                               │ │
│  │  특징:                                                              │ │
│  │  • 0부터 시작하는 연속적인 주소 공간                                    │ │
│  │  • 프로세스마다 독립적                                               │ │
│  │  • 프로그램이 보는 주소                                               │ │
│  │  • 컴파일 시 결정                                                   │ │
│  │  • 최대 크기: CPU 아키텍처에 따라 다름                                   │ │
│  │    - 32비트: 4GB (0 ~ 2^32-1)                                       │ │
│  │    - 64비트: 16EB (0 ~ 2^64-1)                                      │ │
│  │                                                               │ │
│  │  예: 프로그램에서 변수 x의 주소 &x = 0x7fff5fbff8ac                   │ │
│  │      → 이것은 논리 주소 (가상 주소)                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【물리 주소 (Physical Address)】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정의: 실제 메모리(RAM) 하드웨어의 주소                                 │ │
│  │                                                               │ │
│  │  특징:                                                              │ │
│  │  • 메모리 컨트롤러가 접근하는 실제 주소                                 │ │
│  │  • 모든 프로세스가 공유                                               │ │
│  │  • 하드웨어가 보는 주소                                               │ │
│  │  • 실행 시 결정 (MMU 변환)                                           │ │
│  │  • 최대 크기: 설치된 RAM에 따라 다름                                   │ │
│  │    - 16GB RAM: 0 ~ 0x3FFFFFFFF                                     │ │
│  │    - 64GB RAM: 0 ~ 0xFFFFFFFFF                                     │ │
│  │                                                               │ │
│  │  예: 실제 RAM 칩의 0x12345678 번지                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 변환】                                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │   CPU                    MMU                   Memory           │ │
│  │    │                      │                       │              │ │
│  │    │  논리 주소           │                       │              │ │
│  │    │  0x7FFF_1234        │                       │              │ │
│  │    │ ─────────────────→  │                       │              │ │
│  │    │                      │  주소 변환             │              │ │
│  │    │                      │  (Page Table)         │              │ │
│  │    │                      │ ─────────────────→    │              │ │
│  │    │                      │                       │              │ │
│  │    │                      │  물리 주소             │              │ │
│  │    │                      │  0x1234_5678          │              │ │
│  │    │                      │                       │              │ │
│  │    │                      │                       │  데이터       │ │
│  │    │  ←─────────────────────────────────────────  │              │ │
│  │    │                      │                       │              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 물리/논리 주소 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                물리/논리 주소 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 공간 비교】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  논리 주소 공간 (각 프로세스마다):                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │  프로세스 A (4GB)                                         │   │ │
│  │  │  ┌──────────────┐                                        │   │ │
│  │  │  │   Stack      │ ← 높은 주소                              │   │ │
│  │  │  ├──────────────┤                                        │   │ │
│  │  │  │     ↓        │                                        │   │ │
│  │  │  │   (free)     │                                        │   │ │
│  │  │  │     ↑        │                                        │   │ │
│  │  │  ├──────────────┤                                        │   │ │
│  │  │  │   Heap       │                                        │   │ │
│  │  │  ├──────────────┤                                        │   │ │
│  │  │  │   BSS        │                                        │   │ │
│  │  │  ├──────────────┤                                        │   │ │
│  │  │  │   Data       │                                        │   │ │
│  │  │  ├──────────────┤                                        │   │ │
│  │  │  │   Text       │ ← 낮은 주소                              │   │ │
│  │  │  └──────────────┘                                        │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  │  물리 주소 공간 (전체 시스템):                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │  실제 RAM (16GB)                                         │   │ │
│  │  │  ┌────────────────────────────────────────────────────┐ │   │ │
│  │  │  │ 프로세스 A 일부 │ 프로세스 B │ 커널 │ 캐시 │ ...  │ │   │ │
│  │  │  └────────────────────────────────────────────────────┘ │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 변환 방식】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 페이징 (Paging):                                                 │ │
│  │     • 고정 크기 블록 (페이지/프레임) 단위 변환                            │ │
│  │     • 논리 주소: [페이지 번호 | 오프셋]                                 │ │
│  │     • 물리 주소: [프레임 번호 | 오프셋]                                 │ │
│  │     • 오프셋은 변환 없이 그대로                                         │ │
│  │                                                               │ │
│  │  2. 세그멘테이션 (Segmentation):                                      │ │
│  │     • 가변 크기 블록 (세그먼트) 단위 변환                                │ │
│  │     • 논리 주소: [세그먼트 번호 | 오프셋]                                │ │
│  │     • 물리 주소: 세그먼트 베이스 + 오프셋                                │ │
│  │                                                               │ │
│  │  3. 혼합 (Paged Segmentation):                                       │ │
│  │     • 세그먼트 내에서 페이징                                            │ │
│  │     • 두 방식의 장점 결합                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MMU 변환 예시】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  논리 주소: 0x0000_1234 (32비트, 4KB 페이지)                           │ │
│  │                                                               │ │
│  │  ┌────────────────────────┬────────────────┐                       │ │
│  │  │  VPN (20비트)           │ Offset (12비트) │                       │ │
│  │  │    0x00001              │    0x234        │                       │ │
│  │  └────────────────────────┴────────────────┘                       │ │
│  │                          │                                        │ │
│  │                          ▼                                        │ │
│  │  Page Table: VPN 0x00001 → PFN 0x12345                            │ │
│  │                          │                                        │ │
│  │                          ▼                                        │ │
│  │  물리 주소: 0x12345_234                                            │ │
│  │  ┌────────────────────────┬────────────────┐                       │ │
│  │  │  PFN (20비트)           │ Offset (12비트) │                       │ │
│  │  │    0x12345              │    0x234        │                       │ │
│  │  └────────────────────────┴────────────────┘                       │ │
│  │                                                               │ │
│  │  → 오프셋(0x234)은 그대로, VPN만 PFN으로 변환                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 물리/논리 주소 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                물리/논리 주소 × 타 과목 융합                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • MMU: 주소 변환 하드웨어                                            │
│  • TLB: 변환 결과 캐시                                                │
│  • 페이지 테이블: 변환 테이블                                          │
│  • 주소 버스: 물리 주소 전송                                           │
│                                                                     │
│  【운영체제】                                                          │
│  • 메모리 관리: 페이지 할당                                            │
│  • 프로세스 격리: 논리 주소 공간 분리                                    │
│  • 페이지 테이블 관리: 생성, 수정, 삭제                                  │
│  • 컨텍스트 스위치: 페이지 테이블 교체                                   │
│                                                                     │
│  【컴파일러】                                                          │
│  • 주소 할당: 논리 주소 생성                                            │
│  • 링킹: 논리 주소 연결                                                │
│  • 로딩: 논리 → 물리 매핑                                              │
│  • PIE: 위치 독립 코드                                                │
│                                                                     │
│  【보안】                                                              │
│  • ASLR: 논리 주소 무작위화                                            │
│  • DEP: 실행 권한 제어                                                │
│  • 메모리 격리: 주소 공간 분리                                          │
│  • 커널 보호: 커널 공간 vs 사용자 공간                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  주소 변환 없이 vs 있이:                                                        │ │
│  │  ──────────────────────                                              │ │
│  │  특성              변환 없이             변환 있이 (가상 메모리)         │ │
│  │  ──────             ────────             ────────────────              │ │
│  │  주소              직접 물리 주소          논리 → 물리 변환               │ │
│  │  격리              없음                   있음                         │ │
│  │  보호              어려움                  쉬움                         │ │
│  │  재배치            어려움                  불필요                        │ │
│  │  메모리 활용       비효율                  효율                          │ │
│  │  오버헤드          없음                    있음                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  【프로세스 메모리 맵 확인】                                                        │)
│  ──────────────────                                                  │
│  $ cat /proc/self/maps                                             │
│  address           perms offset  dev   inode   pathname            │
│  55a5c2a000-55a5c2d000 r--p 000000 08:01 12345  /bin/ls            │
│  55a5c2d000-55a5c3e000 r-xp 000030 08:01 12345  /bin/ls            │
│  55a5c3e000-55a5c41000 r--p 001040 08:01 12345  /bin/ls            │
│  55a5c42000-55a5c43000 r--p 001060 08:01 12345  /bin/ls            │
│  55a5c43000-55a5c44000 rw-p 001070 08:01 12345  /bin/ls            │
│  7fff5fbff000-7fff5fc00000 rw-p ...           [stack]              │
│                                                                     │
│  # address: 논리 주소 범위                                            │

│                                                                     │

│  ──────────────────                                                  │
│  # root 권한 필요                                                    │
│  $ cat /proc/[pid]/pagemap                                          │
│                                                                     │
│  【주소 변환 시뮬레이션 (C)】                                                        │)
│  ──────────────────                                                  │
│  #include <stdio.h>                                                 │
│  int x = 42;  // 전역 변수                                           │
│  int main() {                                                        │
│      int y = 10;  // 지역 변수                                        │
│      printf("x 논리 주소: %p\n", &x);  // 예: 0x55a5c43010          │
│      printf("y 논리 주소: %p\n", &y);  // 예: 0x7fff5fbfe4          │
│      return 0;                                                       │
│  }                                                                  │
│                                                                     │
│  【물리/논리 주소 요약】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           논리 주소              물리 주소                      │ │
│  │  ──────          ──────                ──────                        │ │
│  │  생성자          CPU/프로그램           메모리 하드웨어                  │ │
│  │  크기            아키텍처 따라           RAM 크기 따라                   │ │
│  │  공유            프로세스별 독립          전체 시스템 공유                │ │
│  │  변환            MMU 필요               변환 불필요                    │ │
│  │  시작            0x0                    하드웨어 결정                   │ │
│  │  연속성          연속적일 수 있음         불연속적일 수 있음              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 논리 주소: CPU가 생성하는 주소, 프로세스별 독립
• 물리 주소: 실제 RAM의 주소, 전체 시스템 공유
• MMU: 논리 주소를 물리 주소로 변환
• 페이징: 페이지 단위 변환, 오프셋 유지
• 세그멘테이션: 세그먼트 단위 변환
• 32비트: 4GB 논리 주소 공간
• 64비트: 16EB 논리 주소 공간
• 페이지 테이블: 변환 정보 저장
• TLB: 변환 결과 캐시
• 목적: 격리, 보호, 메모리 효율
```

**효과**: 분류번호는 쉽고, 책장 위치는 정확해요!
