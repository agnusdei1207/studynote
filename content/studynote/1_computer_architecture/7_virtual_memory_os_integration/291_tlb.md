# 291. TLB (Translation Lookaside Buffer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상-물리 주소 변환 결과를 캐시하는 하드웨어
> 2. **가치**: Page Walk 생략, 빠른 주소 변환
> 3. **융합**: MMU, 페이지 테이블, 캐시 일관성, ASID

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

TLB(Translation Lookaside Buffer)는 **가상 페이지 번호(VPN)와 물리 프레임 번호(PFN) 매핑을 캐시하는 하드웨어**다. MMU 내부에 위치하며, TLB 히트 시 페이지 테이블 접근 없이 즉시 변환한다.

### 💡 비유: 사서의 기억력

TLB는 **사서의 기억력**과 같다. 자주 찾는 책은 위치를 기억해둔다. "과학-물리-001"을 달라고 하면 바로 "3층 B구역 5번 책장"을 알려준다. 위치록(페이지 테이블)을 안 봐도 된다. 기억력(TLB)에 없으면 위치록을 찾아본다.

### TLB 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                TLB 개요                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLB 위치와 역할】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │   CPU                    MMU                   Memory           │ │
│  │    │                      │                       │              │ │
│  │    │  가상 주소           │                       │              │ │
│  │    │  VPN = 2            │                       │              │ │
│  │    │ ─────────────────→  │                       │              │ │
│  │    │                      │                       │              │ │
│  │    │                      │  TLB 조회              │              │ │
│  │    │                      │  ┌─────────────┐      │              │ │
│  │    │                      │  │ VPN │ PFN   │      │              │ │
│  │    │                      │  │  2  │  9    │ ←Hit!│              │ │
│  │    │                      │  └─────────────┘      │              │ │
│  │    │                      │       │               │              │ │
│  │    │                      │       │ Hit          │              │ │
│  │    │                      │       ▼               │              │ │
│  │    │                      │  PFN = 9 즉시 반환     │              │ │
│  │    │                      │  (1~2 클럭)           │              │ │
│  │    │                      │                       │              │ │
│  │    │                      │  ←──────────────────  │              │ │
│  │    │                      │                       │              │ │
│  │    │                      │  Miss 시 Page Table Walk              │
│  │    │                      │ ─────────────────→    │              │ │
│  │    │                      │                       │ Page Table   │
│  │    │                      │ ←─────────────────    │ (10~100클럭) │
│  │    │                      │                       │              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 구조】                                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │ VPN   │ PFN   │ 권한 │ ASID │ Valid │ 기타              │   │ │
│  │  │ ────  │ ────  │ ──── │ ──── │ ────  │ ────              │   │ │
│  │  │ 0x001 │ 0x123 │ RW   │  5   │  1    │ ...               │   │ │
│  │  │ 0x7FF │ 0x456 │ R    │  5   │  1    │ ...               │   │ │
│  │  │ 0x200 │ 0x789 │ RWX  │  3   │  1    │ ...               │   │ │
│  │  │ ...   │ ...   │ ...  │ ...  │ ...   │ ...               │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  │  • VPN: Virtual Page Number (태그)                                  │ │
│  │  • PFN: Physical Frame Number (변환 결과)                           │ │
│  │  • 권한: R/W/X                                                       │ │
│  │  • ASID: Address Space ID (프로세스 구분)                             │ │
│  │  • Valid: 유효 비트                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 특징】                                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 크기: 작음 (64~128 엔트리 일반적)                                   │ │
│  │  • 구조: 완전 연관 (Fully Associative)                               │ │
│  │  • 위치: MMU 내부                                                    │ │
│  │  • 속도: 1~2 클럭 (매우 빠름)                                          │ │
│  │  • 관리: 하드웨어 자동 (일부 OS 관여)                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. TLB 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                TLB 상세                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLB 종류】                                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 데이터 TLB (dTLB): 데이터 접근용                                  │ │
│  │  2. 명령어 TLB (iTLB): 명령어 fetch용                                │ │
│  │  3. 통합 TLB (Unified TLB): 데이터 + 명령어 통합                      │ │
│  │  4. L2 TLB: 큰 크기의 2단계 TLB                                      │ │
│  │                                                               │ │
│  │  계층 구조:                                                          │ │
│  │  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐          │ │
│  │  │ L1 TLB  │ → │ L2 TLB  │ → │ Page    │ → │ Memory  │          │ │
│  │  │ 64엔트리│   │512엔트리│   │ Table   │   │         │          │ │
│  │  └─────────┘   └─────────┘   └─────────┘   └─────────┘          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 미시간 (Miss Penalty)】                                                     │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  TLB Hit: 1~2 클럭 (매우 빠름)                                        │ │
│  │  TLB Miss + Page Table Walk:                                        │ │
│  │  • 2단계 PT: 2번 메모리 접근 ≈ 20~40 클럭                             │ │
│  │  • 4단계 PT: 4번 메모리 접근 ≈ 40~100 클럭                            │ │
│  │                                                               │ │
│  │  AMAT (Average Memory Access Time) with TLB:                        │ │
│  │  AMAT = TLB_시간 + TLB_미스율 × Page_Walk_시간 +                      │ │
│  │         메모리_접근_시간 + 캐시_미스율 × 캐시_미스_페널티                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 관리】                                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. TLB 플러시 (Flush):                                              │ │
│  │     • 전체 플러시: 모든 엔트리 무효화                                   │ │
│  │     • 부분 플러시: 특정 ASID만 무효화                                  │ │
│  │     • 시점: 컨텍스트 스위치, 페이지 테이블 변경                            │ │
│  │                                                               │ │
│  │  2. ASID (Address Space ID):                                        │ │
│  │     • 각 프로세스에 고유 ID 할당                                        │ │
│  │     • TLB 엔트리에 ASID 저장                                          │ │
│  │     • 컨텍스트 스위치 시 플러시 불필요                                    │ │
│  │                                                               │ │
│  │  3. PCID (Process-Context Identifier):                              │ │
│  │     • x86-64의 ASID 구현                                            │ │
│  │     • 12비트 (4096개 프로세스 구분)                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 교체 정책】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • LRU: Least Recently Used (일반적)                                │ │
│  │  • Random: 하드웨어 단순화                                            │ │
│  │  • Round-Robin: 순차 교체                                            │ │
│  │  • NRU: Not Recently Used                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. TLB × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                TLB × 타 과목 융합                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • MMU: TLB가 MMU 내부에 위치                                         │
│  • 캐시: VIPT vs PIPT 문제                                            │
│  • 파이프라인: TLB 미스 시 스톨                                          │
│  • 성능: TLB 히트율이 전체 성능 영향                                      │
│                                                                     │
│  【운영체제】                                                          │
│  • 컨텍스트 스위치: TLB 플러시/ASID                                      │
│  • 페이지 테이블 변경: TLB 무효화                                        │
│  • Huge Page: TLB 커버리지 증가                                        │
│  • 메모리 관리: TLB와 페이지 테이블 동기화                                 │
│                                                                     │
│  【보안】                                                              │
│  • ASID: 프로세스 격리                                                │
│  • TLB 플러시: 민감 정보 보호                                           │
│  • Meltdown/Spectre: TLB 기반 공격                                    │
│  • KPTI: 커널 페이지 격리                                              │
│                                                                     │
│  【성능】                                                              │
│  • TLB 미스: 큰 성능 저하                                              │
│  • Huge Page: TLB 미스 감소                                           │
│  • 지역성: 높을수록 TLB 히트율 높음                                       │
│  • Working Set: TLB 크기보다 작아야 함                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  TLB 크기 vs 커버리지:                                                               │ │
│  │  ──────────────────────                                              │ │
│  │  TLB 엔트리   페이지 크기   커버리지         사용처                       │ │
│  │  ──────────   ──────       ────────         ────                         │ │
│  │  64          4KB          256KB           일반적                       │ │
│  │  64          2MB          128MB           DB, 대용량                   │ │
│  │  64          1GB          64GB            HPC                          │ │
│  │  128         4KB          512KB           서버                         │ │
│  │  512         4KB          2MB             L2 TLB                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### TLB 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                TLB 분석                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLB 정보 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/cpuinfo | grep -i tlb                                  │
│  TLB size: 64 4KB pages                                             │
│  TLB entries: 64                                                    │
│                                                                     │
│  # 또는 lscpu 사용                                                   │
│  $ lscpu | grep -i tlb                                              │
│                                                                     │
│  【TLB 미스 측정】                                                                │)
│  ──────────────────                                                  │
│  $ perf stat -e dTLB-loads,dTLB-load-misses,itlb-loads,itlb-misses ./prog │
│  dTLB-loads: 100,000,000                                            │
│  dTLB-load-misses: 500,000                                          │
│  dTLB 미스율: 0.5%                                                    │
│                                                                     │
│  itlb-loads: 50,000,000                                             │
│  itlb-misses: 100,000                                               │
│  iTLB 미스율: 0.2%                                                    │
│                                                                     │
│  【Huge Page로 TLB 효율 향상】                                                      │)
│  ──────────────────                                                  │
│  # 4KB 페이지: 64 엔트리 × 4KB = 256KB 커버리지                          │
│  # 2MB 페이지: 64 엔트리 × 2MB = 128MB 커버리지 (512배 증가)               │
│                                                                     │
│  $ cat /proc/meminfo | grep Huge                                    │
│  Hugepagesize:     2048 kB                                          │
│                                                                     │
│  【TLB 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. Huge Page: TLB 커버리지 증가                                        │
│  2. 지역성 향상: 같은 페이지 집중 접근                                     │
│  3. Working Set 관리: TLB 크기에 맞게                                   │
│  4. ASID 활성화: 컨텍스트 스위치 TLB 플러시 방지                             │
│  5. 페이지 정렬: TLB 효율 향상                                           │
│  6. TLB 미스 프로파일링: 핫스팟 찾기                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• TLB: VPN→PFN 변환 결과를 캐시하는 하드웨어
• 위치: MMU 내부
• 크기: 작음 (64~128 엔트리 일반적)
• 속도: 1~2 클럭 (TLB Hit 시)
• 미스 페널티: 10~100 클럭 (Page Walk)
• 종류: dTLB, iTLB, Unified TLB, L2 TLB
• ASID: 프로세스 구분, TLB 플러시 방지
• 교체 정책: LRU, Random
• 최적화: Huge Page, 지역성, Working Set
• 보안: KPTI, TLB 플러시
```
