# 287. 내부 단편화 (Internal Fragmentation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 할당된 블록 내부의 사용되지 않는 공간
> 2. **가치**: 페이징의 대가, 공간 낭비 이해
> 3. **융합**: 페이지, 세그먼트, 슬랩 할당

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

내부 단편화(Internal Fragmentation)는 **할당된 메모리 블록 내부에서 사용되지 않는 공간**이다. 요청 크기가 할당 단위보다 작으면 발생한다. 페이징의 주요 단점이다.

### 💡 비유: 버스 좌석

내부 단편화는 **버스 좌석**과 같다. 50인승 버스에 45명이 탄다. 5좌석이 비어 있다. 이 5좌석은 다른 버스에 할당할 수 없다. 버스(페이지) 내부에서 낭비된다. 좌석 단위로만 할당하기 때문이다.

### 내부 단편화 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                내부 단편화 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【내부 단편화 정의】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  할당된 블록:  ┌─────────────────────────────────────┐         │ │
│  │               │ 사용 중 │    사용 안 함 (낭비)         │         │ │
│  │               │ (요청)  │    (내부 단편화)              │         │ │
│  │               └─────────────────────────────────────┘         │ │
│  │               │←── 할당 단위 (페이지) ──→│                  │ │
│  │                                                               │ │
│  │  예: 4KB 페이지에 3KB 요청                                         │ │
│  │  ┌─────────────────────────────────────────────────────┐       │ │
│  │  │              3KB 사용             │    1KB 낭비       │       │ │
│  │  └─────────────────────────────────────────────────────┘       │ │
│  │  │←────────────── 4KB 페이지 ──────────────────→│            │ │
│  │                                                               │ │
│  │  → 1KB가 페이지 내부에서 낭비됨                                      │ │
│  │  → 다른 페이지에서 사용할 수 없음                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【외부 단편화 vs 내부 단편화】                                                     │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  외부 단편화 (External Fragmentation):                               │ │
│  │  ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐                    │ │
│  │  │ 사용 │   │ 빈공간│   │ 사용 │   │ 빈공간│                    │ │
│  │  └──────┘   └──────┘   └──────┘   └──────┘                    │ │
│  │            ↑                ↑                                  │ │
│  │            └────── 사용 불가 ──────┘ (분산됨)                    │ │
│  │  → 빈 공간이 분산되어 큰 할당 불가                                   │ │
│  │  → 세그멘테이션, First Fit 등에서 발생                               │ │
│  │                                                               │ │
│  │  내부 단편화 (Internal Fragmentation):                               │ │
│  │  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │ │
│  │  │ 사용 │ 낭비  │  │ 사용 │ 낭비  │  │ 사용 │ 낭비  │            │ │
│  │  └──────────────┘  └──────────────┘  └──────────────┘          │ │
│  │        ↑                ↑                ↑                      │ │
│  │        └────────── 각 블록 내부 낭비 ──────────┘                  │ │
│  │  → 할당 블록 내부에서 낭비                                           │ │
│  │  → 페이징, Slab 할당에서 발생                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【내부 단편화 특징】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 원인: 고정 크기 할당 단위                                          │ │
│  │  • 위치: 할당된 블록 내부                                             │ │
│  │  • 해결: 작은 할당 단위, 가변 크기 할당                                 │ │
│  │  • 대가: 외부 단편화 제거를 위한 트레이드오프                             │ │
│  │  • 발생: 페이징, Slab, Buddy 할당                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 내부 단편화 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                내부 단편화 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【내부 단편화 계산】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  단편화율 = (할당 크기 - 요청 크기) / 할당 크기 × 100%                  │ │
│  │                                                               │ │
│  │  예: 4KB 페이지에 3KB 요청                                          │ │
│  │  단편화율 = (4KB - 3KB) / 4KB = 25%                                │ │
│  │                                                               │ │
│  │  평균 내부 단편화 (균등 분포 가정):                                    │ │
│  │  평균 단편화 = 페이지 크기 / 2                                        │ │
│  │  평균 단편화율 = 50% (최악)                                          │ │
│  │  실제 단편화율 ≈ 25~30% (요청 분포 고려)                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 크기별 내부 단편화】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  페이지 크기    평균 낭비    최대 낭비    단편화율      사용처           │ │
│  │  ──────         ──────       ──────       ──────       ────             │ │
│  │  4KB           2KB          4KB          ~50%         일반적            │ │
│  │  8KB           4KB          8KB          ~50%         특수             │ │
│  │  16KB          8KB          16KB         ~50%         일부 ARM          │ │
│  │  2MB           1MB          2MB          ~50%         Huge Page        │ │
│  │  1GB           512MB        1GB          ~50%         Huge Page        │ │
│  │                                                               │ │
│  │  → 페이지가 클수록 내부 단편화 심함                                     │ │
│  │  → 하지만 페이지 테이블은 작아짐                                        │ │
│  │  → 트레이드오프 고려 필요                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【내부 단편화 발생 사례】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 페이징 (Paging):                                                │ │
│  │     • 마지막 페이지에서 발생                                          │ │
│  │     • 4KB 페이지, 10KB 요청 → 2KB 낭비                               │ │
│  │                                                               │ │
│  │  2. Slab 할당:                                                      │ │
│  │     • 객체 크기가 슬랩 크기보다 작으면 발생                              │ │
│  │     • 64바이트 슬랩, 50바이트 객체 → 14바이트 낭비                       │ │
│  │                                                               │ │
│  │  3. Buddy 할당:                                                     │ │
│  │     • 2의 거듭제곱 크기로 할당                                         │ │
│  │     • 5KB 요청 → 8KB 할당 → 3KB 낭비                                 │ │
│  │                                                               │ │
│  │  4. 파일 시스템 블록:                                                  │ │
│  │     • 4KB 블록, 1KB 파일 → 3KB 낭비                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 내부 단편화: 할당된 블록 내부의 낭비 공간
• 원인: 고정 크기 할당 단위
• 발생: 페이징, Slab, Buddy 할당
• 평균 낭비: 페이지 크기의 약 50% (최악)
• 외부 단편화와 반대: 페이징은 내부, 세그멘테이션은 외부
• 트레이드오프: 작은 페이지 = 적은 단편화, 큰 페이지 테이블
• 해결: 작은 할당 단위, 가변 크기, 정렬
• Huge Page: 내부 단편화 증가 vs TLB 효율
• 실무: VSZ vs RSS 차이로 확인
• 현대 OS: 4KB 기본, Huge Page 선택적 사용
```
