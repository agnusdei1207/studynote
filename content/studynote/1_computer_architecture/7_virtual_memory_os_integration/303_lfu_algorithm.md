# 303. LFU 알고리즘 (Least Frequently Used)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용 빈도가 가장 낮은 페이지를 교체
> 2. **가치**: 인기도 기반, 장기적 패턴 반영
> 3. **융합**: 카운터, LRU-K, LFU-Aging

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

LFU(Least Frequently Used) 알고리즘은 **가장 적게 사용된(참조 횟수가 가장 적은) 페이지를 교체하는 페이지 교체 알고리즘**이다. 페이지마다 참조 카운터를 유지하며, 교체 시 카운터가 가장 작은 페이지를 선택한다.

### 💡 비유: 인기 없는 책

LFU는 **인기 없는 책 정리**와 같다. 책장이 꽉 찼다. 어떤 책을 뺄까? 1년 동안 1번만 읽은 책을 뺀다. 100번 읽은 책은 둔다. 인기 있는 책은 자주 읽으니까. 인기 없는 책은 앞으로도 안 읽을 확률이 높다.

### LFU 알고리즘 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                LFU 알고리즘 개요                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LFU 기본 원리】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  교체 대상:                                                           │ │
│  │  • 참조 횟수(사용 빈도)가 가장 적은 페이지                                │ │
│  │  • 인기도가 가장 낮은 페이지                                            │ │
│  │                                                               │ │
│  │  구현:                                                               │
│  │  • 각 페이지에 카운터 유지                                              │ │
│  │  • 페이지 접근 시 카운터++                                              │ │
│  │  • 교체 시 카운터가 가장 작은 페이지 선택                                 │ │
│  │                                                               │ │
│  │  페이지 카운터 예시:                                                    │ │
│  │  ┌──────────────────────────────────────────┐                      │ │
│  │  │ 페이지  │ 카운터  │ 상태                  │                      │ │
│  │  │ ────    │ ────    │ ────                  │                      │ │
│  │  │   A     │   100   │ 인기 많음              │                      │ │
│  │  │   B     │   50    │ 보통                  │                      │ │
│  │  │   C     │    5    │ 인기 적음 (교체 후보)   │                      │ │
│  │  │   D     │   75    │ 많음                  │                      │ │
│  │  └──────────────────────────────────────────┘                      │ │
│  │                                                               │ │
│  │  → C가 카운터가 가장 적으므로 교체 대상                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LFU 동작 예시】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  프레임: 3개, 참조 스트링: A, B, C, A, B, D, A, B, C                 │ │
│  │                                                               │ │
│  │  참조  메모리      카운터        폴트?    교체                       │ │
│  │  ────  ──────      ──────       ────     ────                        │ │
│  │   A    [A,-,-]     A:1          Yes                                 │ │
│  │   B    [A,B,-]     A:1,B:1      Yes                                 │ │
│  │   C    [A,B,C]     A:1,B:1,C:1  Yes                                 │ │
│  │   A    [A,B,C]     A:2,B:1,C:1  No                                  │ │
│  │   B    [A,B,C]     A:2,B:2,C:1  No                                  │ │
│  │   D    [D,B,C]     A:2,B:2,C:1  Yes     C 교체 (카운터 1로 가장 적음)   │ │
│  │                  D:1,B:2,C:1                                      │ │
│  │   A    [D,B,C]     D:1,B:2,C:1  No                                  │ │
│  │      [D,A,C]     D:1,A:3,B:2,C:1 No                                 │ │
│  │                                                               │ │
│  │  총 4번 폴트                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LFU 장단점】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 장기적 인기도 반영                                                 │ │
│  │  • 반복적 패턴에 강함                                                 │ │
│  │  • Belady's Anomaly 없음                                             │ │
│  │  • 지역성과 인기도 모두 고려                                           │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 구현 복잡 (카운터 관리)                                            │ │
│  │  • 오래된 페이지가 계속 남을 수 있음                                    │ │
│  │  • 최근 패턴 반영 약함                                                 │ │
│  │  • 카운터 오버플로우 문제                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. LFU 알고리즘 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                LFU 알고리즘 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LFU 문제점】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 과거 인기도 고정 문제:                                            │ │
│  │     • 처음에 많이 쓴 페이지가 계속 높은 카운터 유지                        │ │
│  │     • 이제 안 써도 교체 안 됨                                          │ │
│  │     • 최근 패턴 반영 안 됨                                             │ │
│  │                                                               │ │
│  │  2. 새 페이지 문제:                                                   │ │
│  │     • 새로 로드된 페이지은 카운터 0                                     │ │
│  │     → 바로 교체 대상                                                  │ │
│  │     • 곧 많이 쓸 페이지도 바로 교체될 수 있음                             │ │
│  │                                                               │ │
│  │  3. 카운터 오버플로우:                                                │ │
│  │     • 카운터가 계속 증가하면 오버플로우 가능                              │ │
│  │     • 해결: 주기적 리셋 또는 Aging 사용                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LFU 개선: LFU with Aging】                                                     │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  카운터 Aging (시간 경과에 따른 감소):                                  │ │
│  │                                                               │ │
│  │  1. 주기적으로 모든 카운터 감소 (예: /2)                                 │ │
│  │  2. 오래된 사용은 영향 감소                                             │ │
│  │  3. 최근 사용이 더 중요해짐                                             │ │
│  │                                                               │ │
│  │  예:                                                                 │ │
│  │  초기: A:100, B:50, C:10                                            │ │
│  │  Aging 후: A:50, B:25, C:5                                          │ │
│  │  C 접근: A:50, B:25, C:6                                            │ │
│  │                                                               │ │
│  │  → 최근 접근이 상대적 중요도 증가                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LRU-K (LFU 변형)】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  마지막 K번 참조 시간 고려:                                            │ │
│  │  • LRU-2: 마지막 2번 참조 시간                                         │ │
│  │  • LRU-3: 마지막 3번 참조 시간                                         │ │
│  │                                                               │ │
│  │  교체 기준:                                                           │ │
│  │  1. 참조 횟수 < K: 가장 오래된 참조 시간 기준                            │ │
│  │  2. 참조 횟수 ≥ K: K번째 참조 시간 기준                                 │ │
│  │                                                               │ │
│  │  장점:                                                               │ │
│  │  • 일회성 접근과 반복적 접근 구분                                        │ │
│  │  • DB 버퍼 풀에서 많이 사용                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │
│  【컴퓨터 구조】                                                      │

│  • 이미지 캐시: 자주 요청 이미지 유지                                    │

│  │  ──────────────────────                                              │ │

## Ⅳ. 실무 적용

### LFU 알고리즘 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                LFU 알고리즘 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LFU 구현 의사코드】                                                            │)
│  ──────────────────                                                  │
│  struct Page {                                                       │
│      int frame;                                                      │
│      int counter;    // 참조 카운터                                   │
│  } pages[N];                                                         │
│                                                                     │
│  on_page_access(page):                                               │
│      page.counter++                                                  │
│                                                                     │
│  page_replace():                                                     │
│      min_counter = infinity                                          │
│      victim = -1                                                     │
│      for i in range(N):                                              │
│          if pages[i].counter < min_counter:                          │
│              min_counter = pages[i].counter                          │
│              victim = i                                              │
│      return victim                                                   │
│                                                                     │
│  【LFU with Aging】                                                              │)
│  ──────────────────                                                  │
│  aging():                                                            │
│      for i in range(N):                                              │
│          pages[i].counter = pages[i].counter / 2                     │
│                                                                     │
│  // 주기적으로 호출 (예: 1초마다)                                       │
│                                                                     │
│  【LRU-K (DB에서 사용)】                                                          │)
│  ──────────────────                                                  │
│  // PostgreSQL, MySQL InnoDB 등에서 변형 사용                          │
│  // 최근 K번 참조 시간 추적                                             │
│  // 참조 횟수 + 시간 기준 교체                                          │
│                                                                     │
│  【LFU 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. Aging: 주기적 카운터 감소로 최근성 반영                              │
│  2. 최소 힙: O(log N)으로 최소 카운터 찾기                               │
│  3. 카운터 크기: 오버플로우 방지                                         │
│  4. 새 페이지 보호: 초기 카운터 부여                                     │
│  5. LRU-K: DB에서 LFU + LRU 결합                                      │
│  6. 하이브리드: LFU + LRU (2Q, ARC)                                    │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• LFU: 참조 횟수가 가장 적은 페이지 교체
• 기준: 인기도 (참조 카운터)
• 장점: 장기적 패턴 반영, 반복적 패턴에 강함
• 단점: 과거 인기 고정, 최근성 반영 약함, 구현 복잡
• 문제: 새 페이지 카운터 0, 카운터 오버플로우
• 개선: LFU-Aging (주기적 감소)
• 변형: LRU-K (마지막 K번 참조 시간)
• 사용처: DB 버퍼 풀 (LRU-K, 2Q)
• 비교: LFU = 장기, LRU = 단기
• 실제 OS: 잘 안 씀 (LRU/Clock 선호)
```
