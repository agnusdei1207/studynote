# 307. 워킹셋 (Working Set)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스가 활발히 사용하는 페이지 집합
> 2. **가치**: 메모리 할당 기준, 스래싱 방지
> 3. **융합**: 지역성, 시간 윈도우, PFF

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

워킹셋(Working Set)은 **프로세스가 일정 시간(Δ) 동안 활발히 참조하는 페이지들의 집합**이다. Peter Denning이 1968년 제안했다. 워킹셋만큼 메모리를 보장하면 스래싱을 방지할 수 있다.

### 💡 비유: 지금 읽는 책들

워킹셋은 **지금 읽는 책들**과 같다. 도서관에서 100권을 빌렸다. 하지만 지금 책상에 펼쳐둔 건 3권이다. 이 3권이 워킹셋이다. 나머지 97권은 창고에 있어도 된다. 책상(메모리)에 워킹셋만 있으면 돼!

### 워킹셋 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                워킹셋 개요                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【워킹셋 정의】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  W(t, Δ) = 시간 t에서 Δ 시간 동안 참조된 페이지 집합                       │ │
│  │                                                               │ │
│  │  t: 현재 시간                                                         │ │
│  │  Δ: 워킹셋 윈도우 (Working Set Window)                                 │ │
│  │                                                               │ │
│  │  예: Δ = 10,000 참조                                                 │ │
│  │  최근 10,000번의 메모리 참조에서 사용된 페이지들                            │ │
│  │                                                               │
│  │  시간축:                                                               │ │
│  │  ─────────────────────────────────────────────────────→            │ │
│  │         │←─────── Δ (워킹셋 윈도우) ──────→│                       │ │
│  │         │                                 │                        │ │
│  │  참조: A B C A D A B C E A B C A D A B C A ...                     │ │
│  │         ↑                                                            │ │
│  │        현재 시간 t                                                    │ │
│  │                                                               │ │
│  │  워킹셋: {A, B, C, D, E}  (Δ 내 참조된 모든 페이지)                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【워킹셋 특성】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 지역성 반영:                                                       │ │
│  │     • 시간 지역성: 최근 쓴 페이지는 다시 씀                                │ │
│  │     • 워킹셋 = 현재 지역성 영역                                         │ │
│  │                                                               │ │
│  │  2. 시간에 따라 변화:                                                   │
│  │     • 워킹셋 크기가 변함 (phase 변화)                                   │ │
│  │     • 작을 때: 수 KB, 클 때: 수 MB                                      │ │
│  │                                                               │ │
│  │  3. Δ 선택이 중요:                                                     │
│  │     • Δ 너무 작음: 워킹셋 과소 추정 → 페이지 폴트 증가                     │ │
│  │     • Δ 너무 큼: 워킹셋 과대 추정 → 메모리 낭비                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【워킹셋 모델과 스래싱 방지】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  원리:                                                               │
│  │  • 각 프로세스에 워킹셋만큼 프레임 보장                                  │ │
│  │  • 워킹셋 > 사용 가능 프레임 → 프로세스 일시 중단                         │ │
│  │  • ∑(모든 워킹셋) ≤ 물리 메모리 유지                                    │ │
│  │                                                               │ │
│  │  구현:                                                               │
│  │  • OS가 각 프로세스 워킹셋 추정                                        │ │
│  │  • 워킹셋 기반 프레임 할당                                              │ │
│  │  • 메모리 부족 시 일부 프로세스 스왑 아웃                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 워킹셋 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                워킹셋 상세                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【워킹셋 크기 변화】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  프로그램 실행 단계별 워킹셋:                                            │ │
│  │                                                               │ │
│  │  워킹셋 │        ┌───┐                       ┌───┐               │ │
│  │    크기 │       /│   │\                     /│   │\              │ │
│  │         │      / │   │ \        ┌─────────/ │   │ \              │ │
│  │         │     /  │   │  \      /         \  │   │  \             │ │
│  │         │    /   │   │   \    /           \ │   │   \            │ │
│  │         │───/────│───│----\──/-------------\│───│----\────       │ │
│  │         │ 초기   컴파일   링킹    실행     루프   함수   종료         │ │
│  │         │ 로드   (큼)    (중간)  (작음)   (중간) (큼)  (작음)        │ │
│  │         │                                                               │ │
│  │  → 프로그램 단계(phase)에 따라 워킹셋 크기 변화                           │ │
│  │  → OS는 동적으로 할당 프레임 조절                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Δ 선택】                                                                     │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Δ (워킹셋 윈도우) 선택:                                                │ │
│  │                                                               │ │
│  │  너무 작은 Δ:                                                          │
│  │  • 워킹셋 과소 추정                                                    │ │
│  │  • 필요한 페이지도 제외                                                │ │
│  │  • 페이지 폴트 증가                                                    │ │
│  │                                                               │ │
│  │  너무 큰 Δ:                                                             │ │
│  │  • 워킹셋 과대 추정                                                    │ │
│  │  • 불필요한 페이지도 포함                                              │ │
│  │  • 메모리 낭비                                                        │ │
│  │                                                               │ │
│  │  적절한 Δ:                                                              │
│  │  • 지역성 윈도우와 일치                                                │ │
│  │  • 일반적으로 수천~수만 참조                                            │ │
│  │  • 실험적으로 결정                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Page Fault Frequency (PFF)】                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  워킹셋의 실용적 대안:                                                   │ │
│  │                                                               │ │
│  │  원리:                                                               │ │
│  │  • 페이지 폴트율 기반 프레임 할당                                        │ │
│  │  • 폴트율 높으면 프레임 증가                                            │ │
│  │  • 폴트율 낮으면 프레임 감소                                            │ │
│  │                                                               │ │
│  │  구현:                                                               │ │
│  │  • 상한(upper bound): 폴트율 > f_upper → 프레임++                       │ │
│  │  • 하한(lower bound): 폴트율 < f_lower → 프레임--                       │ │
│  │  • 구간 내: 프레임 유지                                                │ │
│  │                                                               │ │
│  │  장점:                                                               │
│  │  • 구현 간단                                                         │ │
│  │  • 워킹셋 추정 오버헤드 없음                                            │ │
│  │  • 실제 OS에서 많이 사용                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 워킹셋 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                워킹셋 × 타 과목 융합                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 지역성: 워킹셋은 지역성 영역                                        │
│  • 캐시: 워킹셋이 캐시에 있으면 성능 좋음                               │
│  • TLB: 워킹셋 페이지 TLB에 있으면 빠름                                 │
│  • 메모리: 워킹셋이 메모리에 있어야 함                                   │
│                                                                     │
│  【운영체제】                                                          │
│  • 프레임 할당: 워킹셋 기반                                            │
│  • 스래싱 방지: 워킹셋 보장                                            │
│  • 스왑: 워킹셋 외 페이지 스왑 아웃                                     │
│  • VAD (Virtual Address Descriptors): 워킹셋 추적                      │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 버퍼 풀: 핫 페이지 = 워킹셋                                         │
│  • 워킹셋: 자주 접근하는 데이터 페이지                                   │
│  • 쿼리: 쿼리마다 워킹셋 다름                                          │
│  • 메모리: 워킹셋이 버퍼 풀에 있어야 함                                  │
│                                                                     │
│  【성능】                                                              │
│  • 워킹셋 < 메모리: 성능 좋음                                          │
│  • 워킹셋 > 메모리: 스래싱                                             │
│  • 워킹셋 < 캐시: 캐시 히트率高                                        │
│  • 워킹셋 < TLB: TLB 히트率高                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  워킹셋 vs 전체 메모리:                                                                │ │
│  │  ──────────────────────                                              │ │
│  │  상황              워킹셋    전체 메모리    성능                        │ │
│  │  ────              ──────    ──────────    ────                        │ │
│  │  작은 프로그램      1MB       10MB         좋음                        │ │
│  │  큰 프로그램        10MB      1GB          워킹셋만 로드하면 좋음         │ │
│  │  멀티태스킹        50MB      4GB          워킹셋 합 < 메모리면 좋음       │ │
│  │  스래싱            2GB       1GB          매우 나쁨                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 워킹셋 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                워킹셋 분석                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【워킹셋 추정】                                                                │)
│  ──────────────────                                                  │
│  # 프로세스별 활성 페이지 확인                                         │
│  $ cat /proc/[pid]/stat | awk '{print $24}'                         │
│  12345  # RSS (Resident Set Size) in pages                          │
│                                                                     │
│  # 실제 메모리 사용                                                   │
│  $ ps -o pid,vsz,rss,comm -p [pid]                                  │
│  PID   VSZ    RSS   COMMAND                                        │
│  1234  100000 50000  myapp  # RSS가 워킹셋 근사치                       │
│                                                                     │
│  【워킹셋 모니터링】                                                              │)
│  ──────────────────                                                  │
│  $ vmstat -s | grep -i "active"                                     │
│  active anon pages: 345678                                          │
│  active file pages: 456789                                          │
│                                                                     │
│  # Active list가 워킹셋 근사                                          │
│                                                                     │
│  【워킹셋 기반 튜닝】                                                            │)
│  ──────────────────                                                  │
│  # 워킹셋이 메모리보다 크면 메모리 증설 필요                              │
│  # RSS 합이 물리 메모리에 근접하면 주의                                 │
│                                                                     │
│  $ free -h                                                          │
│                total    used    free                                │
│  Mem:           15Gi    14Gi    1Gi   # 거의 꽉 참 → 워킹셋 의심        │
│                                                                     │
│  【워킹셋 최적화 팁】                                                             │)
│  ──────────────────                                                  │
│  1. 워킹셋 측정: RSS, Active pages 확인                               │
│  2. 메모리 확보: 워킹셋 > 메모리면 증설                                 │
│  3. 데이터 구조: 워킹셋 감소를 위한 구조 최적화                           │
│  4. 알고리즘: 지역성 높은 알고리즘 사용                                  │
│  5. 프로세스 수: 워킹셋 합 < 메모리 유지                                 │
│  6. DB 버퍼 풀: 워킹셋에 맞게 크기 설정                                  │
│  7. Huge Page: 워킹셋 페이지 수 감소                                   │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 워킹셋: 일정 시간(Δ) 동안 참조된 페이지 집합
• 제안: Peter Denning (1968)
• 목적: 메모리 할당 기준, 스래싱 방지
• W(t, Δ): 시간 t에서 Δ 윈도우 내 페이지 집합
• Δ 선택: 너무 작음→과소, 너무 큼→과대
• 스래싱 방지: ∑(워킹셋) ≤ 물리 메모리
• PFF: 폴트율 기반 프레임 할당 (실용적 대안)
• 특성: 프로그램 단계(phase)에 따라 크기 변화
• 측정: RSS, Active pages로 근사
• 관련: 지역성, 캐시, TLB
```
