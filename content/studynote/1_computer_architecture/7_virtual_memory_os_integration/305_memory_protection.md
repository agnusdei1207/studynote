# 305. 메모리 보호 (Memory Protection)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 간 메모리 격리 및 접근 제어
> 2. **가치**: 보안, 안정성, 멀티태스킹 필수
> 3. **융합**: MMU, 권한 비트, 커널/사용자 모드

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

메모리 보호(Memory Protection)는 **각 프로세스가 다른 프로세스의 메모리에 접근하지 못하도록 하고, 권한 없는 메모리 접근을 방지하는 메커니즘**이다. MMU와 페이지 테이블의 권한 비트로 구현된다.

### 💡 비유: 방마다 잠금

메모리 보호는 **방마다 잠금**과 같다. 집에 방이 많다. 각 방에 잠금이 있다. 내 방 열쇠로만 내 방에 들어갈 수 있다. 다른 사람 방에 못 들어간다. 부모님 방(커널)은 더 강한 잠금이다. 잠금 없이 들어가면 경비(MMU)가 막는다.

### 메모리 보호 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 보호 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 보호 목적】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 프로세스 격리 (Process Isolation):                                │ │
│  │     • 한 프로세스가 다른 프로세스 메모리 접근 방지                          │ │
│  │     • 버그, 악의적 접근으로부터 보호                                    │ │
│  │                                                               │ │
│  │  2. 커널 보호 (Kernel Protection):                                    │ │
│  │     • 사용자 프로그램이 커널 메모리 접근 방지                              │ │
│  │     • 시스템 안정성 유지                                              │ │
│  │                                                               │ │
│  │  3. 권한 제어 (Access Control):                                       │ │
│  │     • 읽기, 쓰기, 실행 권한 구분                                        │ │
│  │     • 코드는 실행만, 데이터는 읽기/쓰기                                 │ │
│  │                                                               │ │
│  │  4. 보안 (Security):                                                  │ │
│  │     • 버퍼 오버플로우 방지                                              │ │
│  │     • 코드 인젝션 방지                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 보호 구현】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. PTE 권한 비트:                                                     │ │
│  │     ┌────────────────────────────────────────────────────────┐  │ │
│  │     │ PFN     │ R │ W │ X │ U/S │ ...                         │  │ │
│  │     │         │ 1 │ 0 │ 0 │  1  │  (읽기 전용, 사용자)          │  │ │
│  │     └────────────────────────────────────────────────────────┘  │ │
│  │     • R (Read): 읽기 가능                                            │ │
│  │     • W (Write): 쓰기 가능                                           │ │
│  │     • X (Execute): 실행 가능                                         │ │
│  │     • U/S (User/Supervisor): 사용자/커널 모드                         │ │
│  │                                                               │ │
│  │  2. MMU 검사:                                                          │ │
│  │     • 모든 메모리 접근 시 MMU가 권한 검사                                │ │
│  │     • 위반 시 Protection Fault → OS가 처리                             │ │
│  │                                                               │ │
│  │  3. 커널/사용자 모드:                                                   │ │
│  │     • 사용자 모드: 제한적 권한                                          │ │
│  │     • 커널 모드: 모든 메모리 접근 가능                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 보호 위반 시】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Protection Fault 발생:                                               │ │
│  │  • 다른 프로세스 메모리 접근 → Segmentation Fault                      │ │
│  │  • 커널 메모리 접근 (사용자 모드) → General Protection Fault            │ │
│  │  • 읽기 전용 메모리에 쓰기 → Protection Fault                           │ │
│  │  • 실행 권한 없는 메모리 실행 → Execution Fault                         │ │
│  │                                                               │ │
│  │  처리:                                                               │ │
│  │  • OS가 시그널 전송 (SIGSEGV, SIGBUS)                                 │ │
│  │  • 프로세스 종료 또는 예외 처리                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 메모리 보호 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 보호 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【권한 비트 상세】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  R/W/X 조합:                                                           │
│  │  ┌────────────────────────────────────────────────────────────┐  │ │
│  │  │ 조합      의미              사용처                          │  │ │
│  │  │ ────      ────              ────                            │  │ │
│  │  │ R-X       읽기/실행         코드 세그먼트                      │  │ │
│  │  │ RW-       읽기/쓰기         데이터, 힙, 스택                    │  │ │
│  │  │ R--       읽기 전용         상수, rodata                      │  │ │
│  │  │ --X       실행 전용         일부 ROM                          │  │ │
│  │  │ RWX       모든 권한         JIT 코드, 디버깅                   │  │ │
│  │  │ ---       접근 금지         가드 페이지                        │  │ │
│  │  └────────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  │  U/S 비트:                                                              │ │
│  │  • U=1, S=0: 사용자 모드 접근 가능                                    │ │
│  │  • U=0, S=1: 커널 모드만 접근 가능                                    │ │
│  │  • 사용자 모드에서 커널 페이지 접근 → Fault                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NX/DEP 비트 (실행 방지)】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  No-eXecute (NX) / Data Execution Prevention (DEP):                   │ │
│  │  • 데이터 페이지(힙, 스택)에서 코드 실행 방지                              │ │
│  │  • 버퍼 오버플로우 공격 방어                                           │ │
│  │  • 코드 인젝션 방지                                                  │ │
│  │                                                               │ │
│  │  구현:                                                               │ │
│  │  • x86: XD (eXecute Disable) 비트                                    │ │
│  │  • ARM: XN (eXecute Never) 비트                                      │ │
│  │  • x86-64: NX 비트 (63번째 비트)                                       │ │
│  │                                                               │ │
│  │  예: 스택은 RW- (실행 불가) → 스택 버퍼 오버플로우 공격 방지              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【가드 페이지 (Guard Page)】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  접근 금지 페이지:                                                      │
│  │  • 스택 오버플로우 감지                                                │ │
│  │  • 힙 오버플로우 감지                                                  │ │
│  │  • 버퍼 오버플로우 감지                                                │ │
│  │                                                               │ │
│  │  스택 예시:                                                            │ │
│  │  ┌─────────────┐                                                    │ │
│  │  │   Stack     │ ← 스택 영역                                          │ │
│  │  ├─────────────┤                                                    │ │
│  │  │ Guard Page  │ ← 접근 시 즉시 폴트 (오버플로우 감지)                  │ │
│  │  ├─────────────┤                                                    │ │
│  │  │   Heap      │ ← 힙 영역                                            │ │
│  │  └─────────────┘                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 메모리 보호 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 보호 × 타 과목 융합                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • MMU: 권한 검사 하드웨어                                            │
│  • TLB: 권한 비트도 TLB에 저장                                         │
│  • Ring 레벨: x86 Ring 0~3                                           │
│  • CPU 모드: User Mode, Kernel Mode                                  │
│                                                                     │
│  【운영체제】                                                          │
│  • 페이지 테이블: 권한 비트 설정                                        │
│  • 시그널: SIGSEGV, SIGBUS                                           │
│  • 컨텍스트 스위치: 페이지 테이블 교체                                   │
│  • mprotect: 권한 변경 시스템 콜                                       │
│                                                                     │
│  【보안】                                                              │
│  • NX/DEP: 실행 방지                                                 │
│  • ASLR: 주소 무작위화                                                │
│  • Stack Canary: 스택 보호                                            │
│  • SMEP/SMAP: 커널 보호                                               │
│  • CFI: 제어 흐름 보호                                                │
│                                                                     │
│  【프로그래밍】                                                        │
│  • NULL 포인터: 페이지 0 보호                                          │
│  • 버퍼 오버플로우: 가드 페이지로 감지                                   │
│  • Use-After-Free: 메모리 보호와 무관                                  │
│  • Double Free: 힙 보호                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  메모리 보호 기법 비교:                                                                │ │
│  │  ──────────────────────                                              │ │
│  │  기법              구현 위치    보호 대상           효과                  │ │
│  │  ────              ────────     ────────           ────                  │ │
│  │  PTE 권한 비트      MMU/OS      페이지              접근 제어              │ │
│  │  NX/DEP            MMU/OS      코드/데이터         코드 인젝션 방지        │ │
│  │  가드 페이지        OS          스택/힙            오버플로우 감지         │ │
│  │  ASLR              OS          전체 주소          주소 예측 방지          │ │
│  │  Stack Canary      컴파일러     스택               오버플로우 감지         │ │
│  │  SMEP/SMAP         CPU/OS      커널               커널 보호              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 메모리 보호 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 보호 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 권한 확인】                                                            │)
│  ──────────────────                                                  │
│  $ cat /proc/self/maps                                              │
│  address           perms offset  dev   inode   pathname             │
│  55a5c2a000-55a5c2d000 r--p 000000 08:01 12345  /bin/ls   # 읽기     │
│  55a5c2d000-55a5c3e000 r-xp 000030 08:01 12345  /bin/ls   # 실행     │
│  55a5c42000-55a5c43000 r--p 001060 08:01 12345  /bin/ls   # 읽기     │
│  55a5c43000-55a5c44000 rw-p 001070 08:01 12345  /bin/ls   # 읽기/쓰기 │
│  7fff5fbff000-7fff5fc00000 rw-p ...           [stack]    # 스택      │
│                                                                     │
│  # perms: r=읽기, w=쓰기, x=실행, p=private, s=shared                │
│                                                                     │
│  【mprotect로 권한 변경】                                                         │)
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                              │
│                                                                     │
│  // 메모리 권한 변경                                                  │
│  mprotect(addr, size, PROT_READ | PROT_WRITE);                      │
│  mprotect(addr, size, PROT_READ | PROT_EXEC);                       │
│  mprotect(addr, size, PROT_NONE);  // 접근 금지                       │
│                                                                     │
│  【NX/DEP 확인】                                                                │)
│  ──────────────────                                                  │
│  $ cat /proc/cpuinfo | grep -i nx                                    │
│  flags: ... nx ...                                                   │
│                                                                     │
│  # 실행 불가 확인                                                     │
│  $ readelf -l /bin/ls | grep GNU_STACK                              │
│  GNU_STACK      0x0000000000000000 0x0000000000000000 0x0000000000000000│
│                0x0000000000000000 0x0000000000000000  RW    0x8        │
│  # RW = 읽기/쓰기, 실행(X) 없음                                        │
│                                                                     │
│  【메모리 보호 최적화 팁】                                                         │)
│  ──────────────────                                                  │
│  1. 최소 권한: 필요한 권한만 부여                                       │
│  2. NX 활성화: 데이터 페이지 실행 방지                                  │
│  3. 가드 페이지: 스택/힙 보호                                          │
│  4. ASLR: 주소 무작위화                                               │
│  5. Stack Canary: 컴파일러 옵션 (-fstack-protector)                   │
│  6. mprotect: 동적 권한 관리                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 메모리 보호: 프로세스 격리 및 접근 제어
• 구현: MMU + PTE 권한 비트
• 권한 비트: R (읽기), W (쓰기), X (실행)
• U/S 비트: 사용자/커널 모드 구분
• NX/DEP: 데이터 페이지 실행 방지
• 가드 페이지: 오버플로우 감지
• 위반 시: Protection Fault → SIGSEGV
• MMU: 모든 접근 시 권한 검사
• 목적: 보안, 안정성, 멀티태스킹
• 관련: ASLR, Stack Canary, SMEP/SMAP
```
