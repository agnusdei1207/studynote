# 295. Copy-on-Write (COW)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 쓰기 시점에 복사하는 최적화 기법
> 2. **가치**: fork 성능 향상, 메모리 절약
> 3. **융합**: 페이지 공유, 페이지 폴트, 스냅샷

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

Copy-on-Write(COW)는 **페이지를 공유하다가 쓰기가 발생할 때만 복사하는 기법**이다. 읽기는 공유 페이지로 처리하고, 쓰기 시에만 복사본을 만든다. fork() 시스템 콜에서 가장 많이 사용된다.

### 💡 비유: 시험지 배포

COW는 **시험지 배포**와 같다. 선생님이 시험지 한 장만 준다. 학생들이 돌려가며 본다(읽기). 누군가 답을 적어야 하면(쓰기) 그때 복사해서 준다. 아무도 안 쓰면 한 장으로 충분하다.

### Copy-on-Write 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                Copy-on-Write 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【COW 동작 원리】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  초기 상태 (fork 직후):                                              │ │
│  │                                                               │ │
│  │  부모 프로세스            자식 프로세스                               │ │
│  │  ┌──────────┐            ┌──────────┐                          │ │
│  │  │ Page 0   │            │ Page 0   │                          │ │
│  │  │ → PFN 5  │            │ → PFN 5  │ ← 같은 프레임 공유!         │ │
│  │  │ (R/O)    │            │ (R/O)    │                          │ │
│  │  └──────────┘            └──────────┘                          │ │
│  │        │                       │                                │ │
│  │        └───────────┬───────────┘                                │ │
│  │                    ▼                                            │ │
│  │               물리 프레임                                         │ │
│  │            ┌──────────────┐                                     │ │
│  │            │   Frame 5    │                                     │ │
│  │            │  (공유 중)    │                                     │ │
│  │            └──────────────┘                                     │ │
│  │                                                               │ │
│  │  모든 페이지를 읽기 전용(R/O)으로 공유                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【쓰기 발생 시】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  자식이 Page 0에 쓰기 시도:                                          │ │
│  │                                                               │ │
│  │  1. 쓰기 시도 → Page Fault 발생 (R/O 위반)                           │ │
│  │                                                               │ │
│  │  2. OS가 COW 처리:                                                  │ │
│  │     a) 새 프레임 할당 (Frame 10)                                      │ │
│  │     b) Frame 5 → Frame 10 복사                                      │ │
│  │     c) 자식 PTE: PFN 5 → PFN 10, R/W                                │ │
│  │                                                               │ │
│  │  부모 프로세스            자식 프로세스                               │ │
│  │  ┌──────────┐            ┌──────────┐                          │ │
│  │  │ Page 0   │            │ Page 0   │                          │ │
│  │  │ → PFN 5  │            │ → PFN 10 │ ← 이제 다름!               │ │
│  │  │ (R/O)    │            │ (R/W)    │                          │ │
│  │  └──────────┘            └──────────┘                          │ │
│  │        │                       │                                │ │
│  │        ▼                       ▼                                │ │
│  │  ┌──────────┐            ┌──────────┐                          │ │
│  │  │ Frame 5  │            │ Frame 10 │                          │ │
│  │  │ (부모)    │            │ (자식)    │                          │ │
│  │  └──────────┘            └──────────┘                          │ │
│  │                                                               │ │
│  │  → 쓴 페이지만 복사됨                                                 │ │
│  │  → 안 쓴 페이지는 계속 공유                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. Copy-on-Write 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                Copy-on-Write 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【fork()와 COW】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  fork() 없이 COW:                                                   │ │
│  │  부모 메모리 100MB → 자식에 100MB 복사 → 100MB 메모리 사용            │ │
│  │  시간: 수백 ms                                                      │ │
│  │                                                               │ │
│  │  fork() + COW:                                                      │ │
│  │  부모 메모리 100MB → 페이지 테이블만 복사 → 수 KB 메모리 사용           │ │
│  │  시간: 수 ms                                                        │ │
│  │                                                               │ │
│  │  → COW로 fork()가 매우 빨라짐                                        │ │
│  │  → 실제 복사는 쓰기 시점까지 지연                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【COW 처리 과정】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 쓰기 시도:                                                       │ │
│  │     CPU가 R/O 페이지에 쓰기 → Page Fault                               │ │
│  │                                                               │ │
│  │  2. Page Fault 핸들러:                                               │ │
│  │     if (PTE가 COW 마크됨) {                                          │ │
│  │         if (참조 카운트 == 1) {                                       │ │
│  │             // 마지막 참조, 복사 불필요                                  │ │
│  │             PTE를 R/W로 변경                                          │ │
│  │         } else {                                                    │ │
│  │             // 다른 프로세스와 공유 중                                   │ │
│  │             새 프레임 할당                                              │ │
│  │             데이터 복사                                                │ │
│  │             PTE 업데이트 (새 PFN, R/W)                                 │ │
│  │             참조 카운트--                                              │ │
│  │         }                                                           │ │
│  │         재실행                                                        │ │
│  │     }                                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【COW 장단점】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • fork() 매우 빠름                                                  │ │
│  │  • 메모리 절약 (안 쓰는 페이지 복사 안 함)                                │ │
│  │  • 읽기 많은 워크로드에 효과적                                          │ │
│  │  • 스냅샷, 백업에 활용                                                │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 쓰기 시 Page Fault 오버헤드                                        │ │
│  │  • 쓰기 많은 워크로드에 비효율                                          │ │
│  │  • 구현 복잡도                                                       │ │
│  │  • 메모리 부족 시 문제 (실제 복사 필요)                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                Copy-on-Write × 타 과목 융합                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │

│  • PTE: COW 비트, R/W 비트                                            │
│  • TLB: COW 처리 후 TLB 갱신                                          │
│  • 캐시: 데이터 복사 시 캐시 활용                                        │
│                                                                     │
│  【운영체제】                                                          │
│  • fork(): COW로 빠른 프로세스 생성                                    │
│  • 프로그램 로드: fork 후 로드 패턴                                     │
│  • 메모리 관리: 참조 카운팅                                             │
│  • 페이지 폴트: COW 핸들러                                              │

│  • MVCC: 다중 버전 동시성 제어                                          │
│  • 백업: COW로 온라인 백업                                              │
│                                                                     │
│  【파일 시스템】                                                        │
│  • ZFS: COW 파일 시스템                                               │
│  • Btrfs: COW 스냅샷                                                 │
│  • APFS: Apple COW 파일 시스템                                        │
│  • LVM: 스냅샷 생성                                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  COW 사용 사례:                                                                    │ │
│  │  ──────────────────────                                              │ │
│  │  분야              사용처                    효과                      │ │
│  │  ────              ────                      ────                      │ │
│  │  OS               fork()                    빠른 프로세스 생성           │ │
│  │  DB               MVCC, 스냅샷              동시성, 백업                │ │
│  │  파일시스템         ZFS, Btrfs, APFS         스냅샷, 일관성               │ │
│  │  가상화            VM 스냅샷                 빠른 복구                   │ │
│  │  컨테이너          Docker 레이어             저장소 절약                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### Copy-on-Write 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                Copy-on-Write 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【fork 시간 비교】                                                              │)
│  ──────────────────                                                  │
│  # COW 덕분에 fork는 매우 빠름                                         │
│  # 메모리 크기와 관계없이 수 ms 내 완료                                   │
│                                                                     │
│  【COW Page Fault 측정】                                                         │)
│  ──────────────────                                                  │
│  $ perf stat -e page-faults ./cow_test                              │
│  page-faults: 12345                                                 │
│                                                                     │
│  【메모리 절약 확인】                                                              │)
│  ──────────────────                                                  │
│  # 같은 프로그램 여러 인스턴스 실행                                      │
│  $ ps -o pid,vsz,rss,comm -C myapp                                  │
│  PID   VSZ    RSS   COMMAND                                        │
│  1234  100000 10000  myapp   # 인스턴스 1                              │
│  1235  100000 2000   myapp   # 인스턴스 2 (대부분 공유)                   │
│  1236  100000 2000   myapp   # 인스턴스 3 (대부분 공유)                   │
│                                                                     │
│  【COW 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. fork 후 바로 로드: 쓰기 전에 새 프로그램 로드하면 COW 효과 최대         │
│  2. 읽기 많은 데이터: COW에 적합                                        │
│  3. 쓰기 많은 데이터: COW 비효율, 처음부터 복사 고려                       │
│  4. 스냅샷: COW 활용                                                 │
│  5. 메모리 여유: 실제 복사 시 메모리 필요                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• Copy-on-Write: 쓰기 시점에만 페이지 복사
• 목적: fork() 성능 향상, 메모리 절약
• 동작: 읽기는 공유, 쓰기 시 복사
• Page Fault: 쓰기 시도 시 발생, OS가 처리
• 참조 카운트: 공유 페이지 관리
• 장점: 빠른 fork, 메모리 절약, 스냅샷
• 단점: 쓰기 오버헤드, 쓰기 많으면 비효율
• 사용처: fork, DB 스냅샷, ZFS, Docker
• PTE: COW 마크 + R/O
• 현대 OS: fork()에 필수
```
