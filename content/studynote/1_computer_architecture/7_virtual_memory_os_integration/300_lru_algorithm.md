# 300. LRU 알고리즘 (Least Recently Used)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가장 오래 전에 사용된 페이지를 교체
> 2. **가치**: 지역성 활용, 우수한 성능
> 3. **융합**: 시간 국소성, 스택 알고리즘, Clock

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

LRU(Least Recently Used) 알고리즘은 **가장 오랫동안 사용되지 않은 페이지를 교체하는 페이지 교체 알고리즘**이다. 시간 지역성(Temporal Locality)을 활용하여 최근에 사용된 페이지는 곧 다시 사용될 가능성이 높다는 가정에 기반한다.

### 💡 비유: 오래 안 읽은 책

LRU는 **오래 안 읽은 책 정리**와 같다. 책장이 꽉 찼다. 어떤 책을 뺄까? 1년 전에 읽고 안 본 책을 뺀다. 어제 읽은 책은 둔다. 곧 다시 읽을 수 있으니까. 최근에 안 읽은 책이 앞으로도 안 읽을 확률이 높다.

### LRU 알고리즘 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                LRU 알고리즘 개요                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LRU 기본 원리】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  핵심 가정:                                                           │ │
│  │  • 최근에 사용된 페이지 → 곧 다시 사용될 가능성 높음                        │ │
│  │  • 오래 안 쓴 페이지 → 앞으로도 안 쓸 가능성 높음                          │ │
│  │  → 시간 지역성(Temporal Locality) 활용                               │ │
│  │                                                               │ │
│  │  교체 대상:                                                           │ │
│  │  • 가장 오래 전에 사용된 페이지                                          │ │
│  │  • LRU 리스트의 끝에 있는 페이지                                         │ │
│  │                                                               │ │
│  │  LRU 리스트 예시 (왼쪽이 최근, 오른쪽이 오래됨):                             │ │
│  │  ┌───┬───┬───┬───┬───┐                                            │ │
│  │  │ C │ A │ D │ B │ E │  ← E가 가장 오래됨 (교체 대상)                   │ │
│  │  └───┴───┴───┴───┴───┘                                            │ │
│  │   ↑                                                             │ │
│  │  최근 사용                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LRU 동작 예시】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  프레임: 3개, 참조 스트링: A, B, C, D, A, B, C, D                   │ │
│  │                                                               │ │
│  │  참조  메모리 상태      LRU 순서       폴트?                        │ │
│  │  ────  ──────────      ────────       ────                         │ │
│  │   A    [A, -, -]       [A]            Yes                         │ │
│  │   B    [A, B, -]       [B, A]         Yes                         │ │
│  │   C    [A, B, C]       [C, B, A]      Yes                         │ │
│  │   D    [D, B, C]       [D, C, B]      Yes (A 교체, 가장 오래됨)       │ │
│  │   A    [D, A, C]       [A, D, C]      Yes (B 교체)                 │ │
│  │   B    [D, A, B]       [B, A, D]      Yes (C 교체)                 │ │
│  │   C    [C, A, B]       [C, B, A]      Yes (D 교체)                 │ │
│  │   D    [C, D, B]       [D, C, B]      Yes (A 교체)                 │ │
│  │                                                               │ │
│  │  총 8번 폴트                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LRU 장단점】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • OPT에 근접한 성능                                                 │ │
│  │  • 지역성 활용                                                       │ │
│  │  • Belady's Anomaly 없음 (Stack Algorithm)                          │ │
│  │  • 직관적 이해                                                       │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 구현 복잡 (정확한 LRU는 비용 높음)                                  │ │
│  │  • 매 접근마다 리스트 갱신 필요                                        │ │
│  │  • 하드웨어 지원 필요할 수 있음                                        │ │
│  │  • 오버헤드                                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. LRU 알고리즘 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                LRU 알고리즘 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LRU 구현 방법】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 카운터 기반 (Counter-based):                                      │ │
│  │     • 각 페이지에 타임스탬프 저장                                       │ │
│  │     • 접근 시 현재 시간 기록                                           │ │
│  │     • 교체 시 가장 작은 타임스탬프 선택                                  │ │
│  │     • 단점: 타임스탬프 비교 오버헤드                                    │ │
│  │                                                               │ │
│  │  2. 스택 기반 (Stack-based):                                           │ │
│  │     • 접근한 페이지를 스택 top으로 이동                                  │ │
│  │     • 교체 시 스택 bottom 페이지 선택                                   │ │
│  │     • 단점: 리스트 조작 오버헤드                                        │ │
│  │                                                               │ │
│  │  3. 행렬 기반 (Matrix-based):                                         │ │
│  │     • N×N 행렬로 접근 순서 관리                                         │ │
│  │     • 접근 시 해당 행은 1, 해당 열은 0으로                                │ │
│  │     • 교체 시 행이 모두 0인(또는 1인) 페이지 선택                          │ │
│  │     • 단점: 공간 복잡도 O(N²)                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LRU 근사 (Approximate LRU)】                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정확한 LRU 구현 비용이 높아 근사 방법 사용:                              │ │
│  │                                                               │ │
│  │  1. 참조 비트 (Reference Bit):                                        │ │
│  │     • 페이지 접근 시 참조 비트 = 1                                       │ │
│  │     • 주기적으로 참조 비트 = 0으로 리셋                                  │ │
│  │     • 교체 시 참조 비트 = 0인 페이지 선택                                 │ │
│  │                                                               │ │
│  │  2. Aging 레지스터:                                                    │ │
│  │     • 각 페이지에 N비트 레지스터                                         │ │
│  │     • 주기적으로 우측 시프트                                            │ │
│  │     • 접근 시 최상위 비트 = 1                                           │ │
│  │     • 교체 시 가장 작은 값 선택                                          │ │
│  │     ┌─────────┐                                                    │ │
│  │     │ 10000000 │ ← 최근 접근                                          │ │
│  │     │ 01000000 │                                                     │ │
│  │     │ 00000001 │ ← 오래됨 (교체 대상)                                   │ │
│  │     └─────────┘                                                    │ │
│  │                                                               │ │
│  │  3. Clock 알고리즘 (Second Chance):                                   │ │
│  │     • 참조 비트 + 원형 리스트                                           │ │
│  │     • 포인터가 돌며 참조 비트 확인                                       │ │
│  │     • 참조 비트 1 → 0으로 변경, 다음으로                                  │ │
│  │     • 참조 비트 0 → 교체                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Stack Algorithm】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  LRU는 Stack Algorithm:                                               │ │
│  │  • n 프레임의 페이지 집합 ⊂ n+1 프레임의 페이지 집합                      │ │
│  │  → 프레임이 늘어나면 폴트율이 절대 증가하지 않음                           │ │
│  │  → Belady's Anomaly 없음                                             │ │
│  │                                                               │ │
│  │  FIFO는 Stack Algorithm 아님:                                         │ │
│  │  → 프레임 늘어도 폴트율 증가 가능 (Belady's Anomaly)                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │

│  • 성능: OPT에 가까운 성능                                              │
│  • 구현: 대부분 Clock으로 근사                                          │

│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  LRU vs 다른 알고리즘 비교:                                                            │ │
│  │  ──────────────────────                                              │ │
│  │  알고리즘         폴트율      Belady   구현     사용처                  │ │

│  │  OPT              최적        없음     불가     기준                    │ │

│  │  Clock            좋음        없음     쉬움     대부분 OS                │ │
│  │  FIFO             나쁨        있음     쉬움     거의 안 씀               │ │
│  │  Random           중간        없음     쉬움     일부 시스템              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅳ. 실무 적용

### LRU 알고리즘 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                LRU 알고리즘 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux LRU 구현】                                                             │)
│  ──────────────────                                                  │
│  # Linux는 Active/Inactive List 사용                                 │
│  $ cat /proc/zoneinfo | grep -A10 "pages"                           │
│  pages free     1234567                                             │
│  pages inactive 234567    # 비활성 리스트 (LRU 후보)                    │
│  pages active   345678    # 활성 리스트                                │
│                                                                     │
│  【LRU 동작 확인】                                                              │)
│  ──────────────────                                                  │
│  # 페이지 교체 통계                                                   │
│  $ vmstat -s | grep -i "inactive"                                   │
│  inactive anon pages: 123456                                        │
│  inactive file pages: 234567                                        │
│                                                                     │
│  【LRU 캐시 라이브러리】                                                          │)
│  ──────────────────                                                  │
│  # Python LRU 캐시                                                   │
│  from functools import lru_cache                                    │
│                                                                     │
│  @lru_cache(maxsize=128)                                            │
│  def fib(n):                                                        │
│      if n < 2: return n                                             │
│      return fib(n-1) + fib(n-2)                                     │
│                                                                     │
│  # LinkedHashMap (Java)                                              │
│  Map<K,V> cache = new LinkedHashMap<>(16, 0.75f, true);             │
│                                                                     │
│  【LRU 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. 워킹셋 관리: 활성 페이지가 캐시에 유지되도록                            │
│  2. 지역성 향상: LRU가 잘 작동하도록 데이터 접근 패턴 개선                   │
│  3. 프리페칭: LRU와 함께 사용                                          │
│  4. Active/Inactive 비율: 튜닝                                        │
│  5. 스캔 저항: 큰 파일 순차 읽기 시 LRU 오염 방지                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• LRU: 가장 오래 안 쓴 페이지 교체
• 기반: 시간 지역성 (Temporal Locality)
• 성능: OPT에 근접, 매우 좋음
• 장점: 지역성 활용, Belady's Anomaly 없음
• 단점: 정확한 구현 비용 높음
• 구현: 카운터, 스택, 행렬
• 근사: 참조 비트, Aging, Clock
• Stack Algorithm: 프레임 ↑ → 폴트율 ↓ 보장
• 사용처: 캐시, OS, DB 버퍼 풀, 웹 캐시
• Linux: Active/Inactive List로 LRU 구현
```
