# 286. 페이지와 프레임 (Page & Frame)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상/물리 메모리의 고정 크기 관리 단위
> 2. **가치**: 메모리 할당 단순화, 외부 단편화 해결
> 3. **융합**: 페이징, 페이지 테이블, MMU, TLB

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

**페이지(Page)**는 가상 메모리(논리 메모리)의 고정 크기 블록이고, **프레임(Frame)**은 물리 메모리(RAM)의 고정 크기 블록이다. 둘은 크기가 동일하며, 페이지 테이블이 매핑한다.

### 💡 비유: 책 페이지와 책장 칸

페이지는 **책의 페이지**다. 프레임은 **책장의 칸**이다. 책은 100페이지씩 묶여 있고, 책장 칸도 100페이지 분량이다. 어떤 페이지가 어느 칸에 들어가든 상관없다. 1~100페이지는 3층 칸에, 101~200페이지는 1층 칸에 있어도 된다.

### 페이지/프레임 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지와 프레임 개요                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 (Page)】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정의: 가상 메모리(논리 메모리)의 고정 크기 블록                           │ │
│  │                                                               │ │
│  │  특징:                                                              │ │
│  │  • 프로세스가 보는 메모리 단위                                          │ │
│  │  • 0부터 시작하는 연속적인 번호 (Page 0, 1, 2, ...)                        │ │
│  │  • 크기: 보통 4KB (4096 bytes)                                       │ │
│  │  • 가상 주소 공간을 페이지로 나눔                                          │ │
│  │                                                               │ │
│  │  가상 주소 구조:                                                        │ │
│  │  ┌────────────────────────────┬──────────────────────┐          │ │
│  │  │     페이지 번호 (VPN)        │    오프셋 (Offset)      │          │ │
│  │  │       20비트 (4KB)          │      12비트             │          │ │
│  │  └────────────────────────────┴──────────────────────┘          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프레임 (Frame)】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정의: 물리 메모리(RAM)의 고정 크기 블록                                   │ │
│  │                                                               │ │
│  │  특징:                                                              │ │
│  │  • 실제 RAM의 슬롯                                                │ │
│  │  • 0부터 시작하는 연속적인 번호 (Frame 0, 1, 2, ...)                        │ │
│  │  • 크기: 페이지와 동일 (보통 4KB)                                        │ │
│  │  • 물리 메모리를 프레임으로 나눔                                          │ │
│  │                                                               │ │
│  │  물리 주소 구조:                                                        │ │
│  │  ┌────────────────────────────┬──────────────────────┐          │ │
│  │  │     프레임 번호 (PFN)        │    오프셋 (Offset)      │          │ │
│  │  │       N비트                  │      12비트             │          │ │
│  │  └────────────────────────────┴──────────────────────┘          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 = 프레임 크기 (필수!)】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 페이지와 프레임은 항상 같은 크기                                       │ │
│  │  • 이유: 1:1 매핑을 위해                                               │ │
│  │  • 오프셋 비트가 동일하게 유지됨                                          │ │
│  │  • 변환 시 VPN만 PFN으로 변경                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 페이지/프레임 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지/프레임 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지-프레임 매핑】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  가상 메모리 (프로세스 A)            물리 메모리 (RAM)              │ │
│  │  ┌──────────────┐                 ┌──────────────┐             │ │
│  │  │  Page 0      │ ─────────────→ │  Frame 5     │             │ │
│  │  │  (4KB)       │                 │  (4KB)       │             │ │
│  │  ├──────────────┤                 ├──────────────┤             │ │
│  │  │  Page 1      │ ─────────────→ │  Frame 2     │             │ │
│  │  │  (4KB)       │                 │  (4KB)       │             │ │
│  │  ├──────────────┤                 ├──────────────┤             │ │
│  │  │  Page 2      │ ─────────────→ │  Frame 9     │             │ │
│  │  │  (4KB)       │                 │  (4KB)       │             │ │
│  │  ├──────────────┤                 ├──────────────┤             │ │
│  │  │  Page 3      │ ─── (디스크) ──→│  (없음)       │             │ │
│  │  │  (4KB)       │                 │              │             │ │
│  │  └──────────────┘                 └──────────────┘             │ │
│  │                                                               │ │
│  │  → 페이지는 연속적이지만, 프레임은 불연속적                               │ │
│  │  → 일부 페이지는 디스크에 있을 수 있음                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지/프레임 크기 선택】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  크기      오프셋 비트   페이지/프레임 수    내부 단편화    사용처          │ │
│  │  ────      ─────────     ────────────     ──────────     ────            │ │
│  │  4KB      12비트         많음             적음           일반적           │ │
│  │  8KB      13비트         중간             중간           특수             │ │
│  │  16KB     14비트         적음             많음           일부 ARM         │ │
│  │  64KB     16비트         매우 적음         많음           임베디드         │ │
│  │  2MB      21비트         매우 적음         매우 많음       Huge Page      │ │
│  │  1GB      30비트         극히 적음         극히 많음       Huge Page      │ │
│  │                                                               │ │
│  │  트레이드오프:                                                        │ │
│  │  • 작은 페이지: 내부 단편화 적음, 페이지 테이블 큼                           │ │
│  │  • 큰 페이지: 내부 단편화 많음, 페이지 테이블 작음, TLB 효율 높음             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지/프레임 수 계산】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예: 32비트 주소 공간, 4KB 페이지                                     │ │
│  │                                                               │ │
│  │  가상 주소 공간: 2^32 = 4GB                                         │ │
│  │  페이지 크기: 4KB = 2^12                                            │ │
│  │  페이지 수: 2^32 / 2^12 = 2^20 = 1M 페이지                           │ │
│  │                                                               │ │
│  │  예: 물리 메모리 16GB, 4KB 프레임                                    │ │
│  │                                                               │ │
│  │  물리 메모리: 16GB = 2^34                                           │ │
│  │  프레임 크기: 4KB = 2^12                                            │ │
│  │  프레임 수: 2^34 / 2^12 = 2^22 = 4M 프레임                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 페이지/프레임 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지/프레임 × 타 과목 융합                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │

│  • 캐시 라인: 페이지 크기와 별개                                         │
│  • 페이지 테이블: 매핑 정보 저장                                        │
│                                                                     │

│  • 페이지 교체: 프레임 확보                                             │
│  • 프레임 관리: free frame list                                       │
│  • 페이지 폴트: 프레임 로딩                                             │
│                                                                     │
│  【데이터베이스】                                                        │

│  • NX 비트: 실행 불가 페이지                                            │
│  • ASLR: 페이지 배치 무작위화                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  페이지 vs 세그먼트:                                                              │ │
│  │  ──────────────────────                                              │ │
│  │  특성              페이지                    세그먼트                    │ │

│  │  메모리 단위       프레임                      세그먼트                    │ │
│  │  외부 단편화       없음                       있음                        │ │
│  │  내부 단편화       있음                       없음                        │ │
│  │  구현              간단                       복잡                        │ │
│  │  현대 OS           대부분 사용                 거의 안 함                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 페이지/프레임 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지/프레임 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 크기 확인】                                                              │)
│  ──────────────────                                                  │
│  $ getconf PAGESIZE                                                │
│  4096    # 4KB                                                     │
│                                                                     │
│  # 또는                                                             │
│  $ pagesize                                                        │
│  4096                                                              │
│                                                                     │
│  【프레임 정보 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/meminfo                                               │
│  MemTotal:        16384000 kB    # 전체 프레임 수 = 16384000/4 = 약 4M │
│  MemFree:          8000000 kB    # 사용 가능 프레임                   │
│                                                                     │
│  【Huge Page (대형 페이지)】                                                       │)
│  ──────────────────                                                  │
│  # Huge Page 확인                                                   │
│  $ cat /proc/meminfo | grep Huge                                   │
│  HugePages_Total:    1024        # 2MB Huge Page                   │
│  HugePages_Free:      512                                          │
│  Hugepagesize:     2048 kB       # 2MB 페이지                       │
│                                                                     │
│  # Huge Page 할당                                                   │
│  $ echo 2048 > /proc/sys/vm/nr_hugepages                           │
│                                                                     │
│  【페이지/프레임 통계】                                                            │)
│  ──────────────────                                                  │
│  $ vmstat -s | head -20                                            │
│  16384000 K total memory                                           │
│  8000000 K used memory                                             │
│  4000000 K active memory                                           │
│  2000000 K inactive memory                                         │
│  8000000 K free memory                                             │
│  ...                                                               │
│                                                                     │
│  【페이지/프레임 최적화 팁】                                                        │)
│  ──────────────────                                                  │
│  1. 페이지 크기: 워크로드에 맞게 선택                                    │
│  2. Huge Page: DB, 대용량 처리용                                     │
│  3. 페이지 정렬: 캐시 성능 향상                                        │
│  4. 프레임 관리: free list 튜닝                                      │
│  5. NUMA: 멀티소켓에서 프레임 배치 고려                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 페이지: 가상 메모리의 고정 크기 블록 (보통 4KB)
• 프레임: 물리 메모리의 고정 크기 블록 (페이지와 동일 크기)
• VPN: Virtual Page Number, 페이지 번호
• PFN: Physical Frame Number, 프레임 번호
• 오프셋: 페이지/프레임 내 위치 (변환 없이 그대로)
• 매핑: 페이지 테이블이 VPN→PFN 저장
• 외부 단편화: 없음 (고정 크기)
• 내부 단편화: 있음 (마지막 페이지)
• Huge Page: 2MB, 1GB 대형 페이지
• 현대 OS: 페이징 기반 메모리 관리
```
