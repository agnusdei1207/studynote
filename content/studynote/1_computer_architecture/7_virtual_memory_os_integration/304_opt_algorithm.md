# 304. OPT 알고리즘 (Optimal Page Replacement)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가장 오랫동안 사용되지 않을 페이지를 교체
> 2. **가치**: 이론적 최적, 다른 알고리즘 평가 기준
> 3. **융합**: 미래 예측, MIN, Belady 알고리즘

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

OPT(Optimal) 알고리즘은 **앞으로 가장 오랫동안 사용되지 않을 페이지를 교체하는 알고리즘**이다. 미래의 페이지 참조 패턴을 알면 폴트율을 최소화할 수 있다. Belady가 제안하여 Belady's Algorithm, MIN Algorithm이라고도 부른다.

### 💡 비유: 미래를 아는 사서

OPT는 **미래를 아는 사서**와 같다. 사서가 미래를 안다. "이 책은 1년 뒤에야 다시 나와요. 이건 내일 나와요." 그러면 1년 뒤 책을 창고로 보낸다. 미래를 알면 완벽하게 정리할 수 있다. 하지만 현실에선 미래를 모른다.

│  │   B    [A,B,C]     A:∞,C:7,B:∞   No                                  │ │

## Ⅱ. OPT 알고리즘 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                OPT 알고리즘 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【OPT의 용도】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 알고리즘 평가:                                                    │
│  │     • 새 알고리즘을 OPT와 비교                                        │ │
│  │     • 폴트율이 OPT에 얼마나 가까운지                                   │ │
│  │     • 효율성 척도                                                    │ │
│  │                                                               │ │
│  │  2. 성능 한계:                                                        │
│  │     • 어떤 알고리즘도 OPT보다 좋을 수 없음                               │ │
│  │     • Upper Bound                                                   │ │
│  │                                                               │ │
│  │  3. 시뮬레이션:                                                        │
│  │     • 알려진 참조 스트링으로 시뮬레이션                                 │ │
│  │     • 실제 워크로드로 오프라인 분석                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【알고리즘 성능 비교】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  참조 스트링: 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5 (프레임 3개)         │ │
│  │                                                               │ │
│  │  알고리즘         폴트 수     OPT 대비                               │ │
│  │  ──────           ──────      ────────                               │ │
│  │  OPT              6           100% (기준)                             │ │
│  │  LRU              8           133%                                    │ │
│  │  Clock            8           133%                                    │ │
│  │  FIFO             9           150%                                    │ │
│  │                                                               │ │
│  │  → LRU가 OPT에 가장 근접                                             │ │
│  │  → FIFO가 가장 나쁨                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【OPT 구현 (시뮬레이션용)】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  오프라인 알고리즘:                                                    │
│  │  1. 참조 스트링 미리 알 때                                             │ │
│  │  2. 각 페이지의 다음 참조 시간 계산                                     │ │
│  │  3. 가장 늦은 페이지 선택                                              │ │
│  │                                                               │ │
│  │  find_victim(frames, future_refs):                                   │ │
│  │      max_distance = -1                                               │ │
│  │      victim = -1                                                     │ │
│  │      for page in frames:                                             │ │
│  │          next_use = find_next_use(page, future_refs)                 │ │
│  │          if next_use > max_distance:                                 │ │
│  │              max_distance = next_use                                 │ │
│  │              victim = page                                           │ │
│  │      return victim                                                   │ │
│  │                                                               │ │
│  │  // 다시 안 쓸 페이지면 distance = ∞                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### OPT 알고리즘 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                OPT 알고리즘 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【OPT 시뮬레이션】                                                              │)
│  ──────────────────                                                  │
│  # 알려진 참조 스트링으로 OPT 성능 측정                                 │
│                                                                     │
│  def opt_simulate(frames, ref_string):                               │
│      page_faults = 0                                                 │
│      memory = []                                                     │
│      for i, page in enumerate(ref_string):                           │
│          if page not in memory:                                      │
│              page_faults += 1                                        │
│              if len(memory) < frames:                                │
│                  memory.append(page)                                 │
│              else:                                                   │
│                  # OPT: 가장 늦게 쓰일 페이지 찾기                       │
│                  future = ref_string[i+1:]                           │
│                  victim = find_farthest(memory, future)              │
│                  memory.remove(victim)                               │
│                  memory.append(page)                                 │
│      return page_faults                                              │
│                                                                     │
│  【실제 알고리즘과 OPT 비교】                                                      │)
│  ──────────────────                                                  │
│  # 워크로드 트레이스로 LRU, Clock, FIFO vs OPT 비교                     │
│  $ ./page_sim -f trace.txt -a OPT,LRU,Clock,FIFO                    │
│                                                                     │
│  【OPT 활용 예시】                                                              │)
│  ──────────────────                                                  │
│  1. 새 알고리즘 개발 시: OPT 대비 성능 측정                              │
│  2. 메모리 크기 결정: OPT 기준으로 충분한 프레임 수 결정                   │
│  3. 워크로드 분석: OPT 폴트율로 최적화 가능 여부 판단                      │
│  4. 교육: 이론적 최적 이해                                              │
│                                                                     │
│  【OPT 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. OPT 불가능: 실제로는 LRU/Clock 사용                                 │
│  2. 프리페칭: 미래 예측으로 OPT 근사                                    │
│  3. 워킹셋: 실제 사용 패턴 분석                                         │
│  4. 평가: OPT를 기준으로 알고리즘 성능 측정                              │
│  5. 메모리 크기: OPT 폴트율이 낮아질 때까지 프레임 증가                    │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• OPT: 가장 오랫동안 안 쓸 페이지 교체
• 조건: 미래 참조 패턴을 알아야 함
• 특징: 이론적 최적, 구현 불가능
• 장점: 폴트율 최소화, Belady's Anomaly 없음
• 단점: 미래를 모르므로 실제 구현 불가
• 용도: 다른 알고리즘 평가 기준 (Upper Bound)
• 별명: Belady's Algorithm, MIN Algorithm
• 성능: 모든 알고리즘 중 최소 폴트
• 비교: LRU가 OPT에 가장 근접
• 실제: LRU, Clock으로 근사
```
