# 297. 스와핑 (Swapping)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리와 디스크 간 페이지 이동
> 2. **가치**: 물리 메모리 확장, 큰 프로그램 실행
> 3. **융합**: 페이지 교체, 스왑 공간, 성능 저하

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

스와핑(Swapping)은 **물리 메모리가 부족할 때 페이지를 디스크(스왑 공간)로 이동시키는 기법**이다. Swap Out은 메모리→디스크, Swap In은 디스크→메모리다. 가상 메모리의 핵심 메커니즘이다.

### 💡 비유: 창고와 책장

스와핑은 **창고와 책장**과 같다. 책장(메모리)이 꽉 찼다. 새 책을 두려면 기존 책을 창고(디스크)로 옮겨야 한다. 자주 안 보는 책을 창고로 보낸다(Swap Out). 나중에 필요하면 다시 꺼낸다(Swap In). 창고가 있으면 책장보다 많은 책을 관리할 수 있다.

### 스와핑 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                스와핑 개요                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스와핑 기본 개념】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  물리 메모리 (RAM)              디스크 (Swap Space)              │ │
│  │  ┌─────────────────┐          ┌─────────────────┐             │ │
│  │  │ Frame 0: Page A │          │ Swap Slot 0     │             │ │
│  │  │ Frame 1: Page B │  Swap Out│ Swap Slot 1: C  │             │ │
│  │  │ Frame 2: (빔)   │ ←─────── │ Swap Slot 2     │             │ │
│  │  │ Frame 3: Page D │  Swap In │ Swap Slot 3     │             │ │
│  │  └─────────────────┘ ───────→ └─────────────────┘             │ │
│  │                                                               │ │
│  │  Swap Out: 메모리 → 디스크 (페이지 내보내기)                           │ │
│  │  Swap In: 디스크 → 메모리 (페이지 가져오기)                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스와핑 발생 시나리오】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 페이지 폴트 발생                                                  │
│  │     • 프로세스가 메모리에 없는 페이지 접근                                 │
│  │                                                               │ │
│  │  2. 빈 프레임 확인                                                    │
│  │     • 빈 프레임 있음 → Swap In                                        │ │
│  │     • 빈 프레임 없음 → 페이지 교체 (Swap Out + Swap In)                  │ │
│  │                                                               │ │
│  │  3. Swap Out (필요 시)                                               │
│  │     • 희생자(Victim) 페이지 선택                                       │ │
│  │     • 수정된(Dirty) 페이지만 디스크에 저장                               │ │
│  │     • PTE: Present = 0, Swap 위치 저장                               │ │
│  │                                                               │ │
│  │  4. Swap In                                                         │
│  │     • 디스크에서 페이지 읽기                                            │ │
│  │     • 프레임에 로드                                                   │ │
│  │     • PTE 갱신: PFN, Present = 1                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스와핑 장단점】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 물리 메모리보다 큰 프로그램 실행 가능                                  │ │
│  │  • 더 많은 프로세스 동시 실행                                          │ │
│  │  • 메모리 부족 시 시스템 유지                                           │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 디스크 I/O로 성능 저하 (100배~1000배 느림)                           │ │
│  │  • 스래싱 위험                                                       │ │
│  │  • 지연 시간 증가                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 스와핑 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                스와핑 상세                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Swap Space (스왑 공간)】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  종류:                                                               │ │
│  │  • 스왑 파티션: 전용 파티션 (빠름)                                      │ │
│  │  • 스왑 파일: 파일 시스템 내 파일 (유연함)                               │ │
│  │                                                               │ │
│  │  크기 권장:                                                            │
│  │  • RAM ≤ 2GB: RAM × 2                                              │ │
│  │  • RAM 2~8GB: RAM × 1                                              │ │
│  │  • RAM > 8GB: RAM × 0.5 (최소 8GB)                                  │ │
│  │                                                               │ │
│  │  Linux 예:                                                           │
│  │  $ swapon --show                                                   │
│  │  NAME      TYPE SIZE USED PRIO                                     │
│  │  /dev/sda2 partition  8G  1G   -2                                  │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Dirty Page와 스와핑】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Clean Page (수정 안 됨):                                            │
│  │  • 원본이 디스크(실행 파일, 파일 시스템)에 있음                            │ │
│  │  • Swap Out 불필요 (버리면 됨)                                         │ │
│  │  • 필요 시 원본에서 다시 로드                                           │ │
│  │                                                               │ │
│  │  Dirty Page (수정됨):                                                │ │
│  │  • 메모리에서 수정됨                                                   │ │
│  │  • Swap Out 필요 (디스크에 저장)                                       │ │
│  │  • Dirty bit = 1로 표시                                              │ │
│  │                                                               │ │
│  │  예:                                                                 │
│  │  • 코드 페이지: Clean (수정 안 됨)                                      │ │
│  │  • 힙, 스택 페이지: Dirty (수정됨)                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 영향】                                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  접근 시간 비교:                                                        │
│  │  • RAM: 100 ns (100억분의 1초)                                        │ │
│  │  • SSD: 100,000 ns (100마이크로초) = RAM의 1,000배                     │ │
│  │  • HDD: 10,000,000 ns (10밀리초) = RAM의 100,000배                    │ │
│  │                                                               │ │
│  │  스와핑 발생 시:                                                        │
│  │  • 페이지 폴트 → 디스크 읽기 → 메모리 로드                                │ │
│  │  • 1회 Swap In: 1~10 ms                                             │ │
│  │  • 빈번한 스와핑 = 심각한 성능 저하                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  • mlock(): 스왑 방지 메모리 잠금                                       │
│  • OOM Killer: 메모리 부족 시 프로세스 종료                               │

## Ⅳ. 실무 적용

### 스와핑 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                스와핑 분석                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스왑 상태 확인】                                                              │)
│  ──────────────────                                                  │
│  $ swapon --show                                                    │
│  $ cat /proc/swaps                                                  │
│  Filename    Type      Size    Used   Priority                      │
│  /dev/sda2   partition 8388604 1048576  -2                          │
│                                                                     │
│  $ free -h                                                          │
│                total    used    free    shared  buff/cache   available│
│  Mem:           15Gi    8Gi    2Gi    1Gi       5Gi         6Gi       │
│  Swap:         8.0Gi    1.0Gi   7.0Gi                                  │
│                                                                     │
│  【스와핑 통계】                                                                │)
│  ──────────────────                                                  │
│  $ vmstat 1                                                         │
│  procs -----------memory---------- ---swap-- -----io----            │
│   r  b   swpd   free   buff  cache   si   so    bi    bo             │
│   1  0  1024   8000    200   2000     0    0     5    10             │
│                                                                     │
│  # si: swap in (페이지/초) - 디스크에서 메모리로                          │
│  # so: swap out (페이지/초) - 메모리에서 디스크로                         │
│                                                                     │
│  【스와핑 튜닝】                                                                │)
│  ──────────────────                                                  │
│  # swappiness 확인 및 설정                                           │
│  $ cat /proc/sys/vm/swappiness                                      │
│  60                                                                  │
│  $ sysctl vm.swappiness=10    # 스왑 적게 사용                          │
│                                                                     │
│  # DB 서버는 보통 swappiness=0 또는 1                                  │
│  $ sysctl vm.swappiness=1                                           │
│                                                                     │
│  【스왑 방지】                                                                    │)
│  ──────────────────                                                  │
│  # mlock으로 메모리 잠금                                              │
│  #include <sys/mman.h>                                              │
│  mlock(ptr, size);  // 스왑 안 됨                                      │
│                                                                     │
│  # 전체 프로세스 메모리 잠금                                            │
│  mlockall(MCL_CURRENT | MCL_FUTURE);                                │
│                                                                     │
│  【스와핑 최적화 팁】                                                             │)
│  ──────────────────                                                  │
│  1. 충분한 RAM: 스와핑 최소화                                           │
│  2. SSD 스왑: HDD보다 10~100배 빠름                                    │
│  3. swappiness: 워크로드에 맞게 조정                                    │
│  4. mlock: 중요 데이터 스왑 방지                                        │
│  5. 모니터링: vmstat, sar로 si/so 확인                                 │
│  6. 스래싱 감지: si/so 높으면 조치                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 스와핑: 메모리와 디스크 간 페이지 이동
• Swap Out: 메모리 → 디스크
• Swap In: 디스크 → 메모리
• 목적: 물리 메모리 확장, 더 많은 프로세스
• Dirty Page: 수정된 페이지, Swap Out 필요
• Clean Page: 수정 안 됨, 버리면 됨
• 성능: 디스크 I/O로 100~1000배 느려짐
• 스왑 공간: 파티션 또는 파일
• swappiness: 스왑 적극성 (0~100)
• 스래싱: 과도한 스와핑, 시스템 마비
```
