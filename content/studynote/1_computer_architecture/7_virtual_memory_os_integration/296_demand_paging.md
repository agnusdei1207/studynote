# 296. 요구 페이징 (Demand Paging)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 필요할 때만 페이지를 메모리에 로드
> 2. **가치**: 빠른 시작, 메모리 효율, I/O 절약
> 3. **융합**: 페이지 폴트, Lazy Loading, 스와핑

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

요구 페이징(Demand Paging)은 **프로세스가 페이지에 접근할 때만 해당 페이지를 메모리에 로드하는 기법**이다. 프로그램 시작 시 모든 페이지를 로드하지 않고, 필요한 페이지만 로드한다.

### 💡 비유: 필요할 때만 책 꺼내기

요구 페이징은 **필요할 때만 책 꺼내기**와 같다. 도서관에서 책 100권을 다 빌릴 필요 없다. 지금 필요한 1권만 빌린다. 나중에 다른 책이 필요하면 그때 가서 빌린다. 책장(메모리)도 절약하고 시간도 절약한다.

### 요구 페이징 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                요구 페이징 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【요구 페이징 vs 전체 로드】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  전체 로드 (All at Once):                                            │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │ 프로그램 시작 → 모든 페이지 로드 → 실행                    │        │ │
│  │  │ 100MB      →    100MB 로드    → 실행                  │        │ │
│  │  │            →    10초          →                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │  → 시작 느림, 메모리 많이 사용                                        │ │
│  │                                                               │ │
│  │  요구 페이징 (Demand Paging):                                         │ │
│  │  ┌─────────────────────────────────────────────────────┐        │ │
│  │  │ 프로그램 시작 → 필요한 페이지만 → 실행                     │        │ │
│  │  │ 100MB      →    5MB 로드     → 실행                   │        │ │
│  │  │            →    0.5초         →                       │        │ │
│  │  └─────────────────────────────────────────────────────┘        │ │
│  │  → 시작 빠름, 메모리 적게 사용                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【요구 페이징 동작】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 프로그램 시작:                                                    │ │
│  │     • 페이지 테이블 생성                                               │ │
│  │     • 모든 PTE의 Present bit = 0                                     │ │
│  │     • 어떤 페이지도 메모리에 없음                                       │ │
│  │                                                               │ │
│  │  2. 첫 접근:                                                         │ │
│  │     CPU가 Page 0 접근 → PTE.P = 0 → Page Fault!                      │ │
│  │                                                               │ │
│  │  3. Page Fault 처리:                                                │ │
│  │     OS가 Page 0를 디스크에서 메모리로 로드                               │ │
│  │     PTE 갱신: PFN = 5, P = 1                                        │ │
│  │                                                               │ │
│  │  4. 이후 접근:                                                        │ │
│  │     Page 0: PTE.P = 1 → 정상 접근                                     │ │
│  │     Page 1: PTE.P = 0 → 또 Page Fault → 로드                         │ │
│  │                                                               │ │
│  │  → 접근한 페이지만 메모리에 존재                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【요구 페이징 장단점】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 빠른 프로그램 시작                                                  │ │
│  │  • 메모리 효율 (필요한 것만)                                           │ │
│  │  • I/O 절약 (안 쓰는 페이지 로드 안 함)                                  │ │
│  │  • 큰 프로그램 실행 가능                                               │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • Page Fault 오버헤드                                              │ │
│  │  • 초기 실행 느림 (워밍업 필요)                                        │
│  │  • 지연 시간 예측 어려움                                              │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 요구 페이징 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                요구 페이징 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Pure Demand Paging】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  극단적 요구 페이징:                                                  │
│  │  • 시작 시 메모리에 단 1개 페이지도 없음                                 │ │
│  │  • 첫 명령어 fetch부터 Page Fault                                    │ │
│  │  • 필요할 때마다 페이지 로드                                           │ │
│  │                                                               │ │
│  │  실제 구현:                                                           │
│  │  • 일부 페이지 미리 로드 (코드 시작 부분)                                │ │
│  │  • 스택 최소 1페이지 할당                                              │ │
│  │  • 나머지는 요구 페이징                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지역성과 요구 페이징】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  지역성이 좋을 때:                                                     │
│  │  • 한 번 로드한 페이지 계속 사용                                        │ │
│  │  • Page Fault 빈도 감소                                              │ │
│  │  • 성능 향상                                                         │ │
│  │                                                               │ │
│  │  지역성이 나쁠 때:                                                     │
│  │  • 랜덤 접근 시 매번 Page Fault                                        │ │
│  │  • 스래싱 발생 가능                                                   │ │
│  │  • 성능 급격히 저하                                                   │ │
│  │                                                               │ │
│  │  시간별 Page Fault:                                                   │
│  │  ┌─────────────────────────────────────────────┐                  │ │
│  │  │ 실행 시간                                     │                  │ │
│  │  │  ↑                                           │                  │ │
│  │  │  │    ****                                   │                  │ │
│  │  │  │   *    *     ----                         │                  │ │
│  │  │  │  *      *                                │                  │ │
│  │  │  │ *        *                               │                  │ │
│  │  │  └────────────────────────────────────→     │                  │ │
│  │  │  시작    워밍업   안정화                      │                  │ │
│  │  │  (많음)  (감소)   (적음)                      │                  │ │
│  │  └─────────────────────────────────────────────┘                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Pre-paging (미리 로드)】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  요구 페이징 단점 보완:                                                │
│  │  • Page Fault 시 주변 페이지도 같이 로드                               │
│  │  • 공간 지역성 활용                                                  │ │
│  │  • Page Fault 횟수 감소                                             │
│  │                                                               │ │
│  │  단점:                                                               │
│  │  • 불필요한 페이지 로드 가능                                          │ │
│  │  • 메모리 낭비                                                       │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 요구 페이징 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                요구 페이징 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Page Fault 통계】                                                            │)
│  ──────────────────                                                  │
│  $ ps -o pid,maj_flt,min_flt,comm -p [pid]                          │
│  PID   MAJFLT  MINFLT  COMMAND                                     │
│  1234  10      5000    myapp                                        │
│                                                                     │
│  # MAJFLT: 메이저 폴트 (디스크에서 로드)                                 │
│  # MINFLT: 마이너 폴트 (메모리에서 해결)                                 │
│                                                                     │
│  【페이지 폴트 측정】                                                              │)
│  ──────────────────                                                  │
│  $ perf stat -e page-faults,major-faults ./prog                     │
│  page-faults: 12345                                                 │
│  major-faults: 100                                                  │
│                                                                     │
│  【요구 페이징 동작 확인】                                                         │)
│  ──────────────────                                                  │
│  $ /usr/bin/time -v ./prog                                          │
│  Page size: 4096 bytes                                              │
│  Major (requiring I/O) page faults: 100                             │
│  Minor (reclaiming a frame) page faults: 5000                       │
│                                                                     │
│  【Pre-paging 효과】                                                              │)
│  ──────────────────                                                  │
│  # Linux readahead 확인                                             │
│  $ cat /sys/block/sda/queue/read_ahead_kb                           │
│  128                                                                 │
│                                                                     │
│  # readahead 튜닝                                                   │
│  $ echo 256 > /sys/block/sda/queue/read_ahead_kb                    │
│                                                                     │
│  【요구 페이징 최적화 팁】                                                         │)
│  ──────────────────                                                  │
│  1. 지역성 향상: 연속된 데이터 접근                                       │
│  2. Pre-paging: readahead 활용                                       │
│  3. Huge Page: Page Fault 감소                                       │
│  4. 메모리 충분: 스와핑 방지                                             │
│  5. 워밍업: 시작 시 필요한 데이터 로드                                     │
│  6. mmap: 파일을 요구 페이징으로 매핑                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 요구 페이징: 필요할 때만 페이지 로드
• 동작: Page Fault 시 디스크에서 로드
• 장점: 빠른 시작, 메모리 절약, I/O 절약
• 단점: Page Fault 오버헤드, 초기 지연
• Pure Demand Paging: 시작 시 0개 페이지
• Pre-paging: 주변 페이지 미리 로드
• 지역성: 좋을수록 효율적
• 측정: MAJFLT (디스크), MINFLT (메모리)
• 현대 OS: 모두 요구 페이징 사용
• 관련: Lazy Loading, 코드 스플리팅
```
