# 290. 역페이지 테이블 (Inverted Page Table)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 프레임 기준의 페이지 테이블
> 2. **가치**: 시스템 전체 하나만 존재, 메모리 크기에 비례
> 3. **융합**: 해시, PowerPC, IA-64, 탐색 오버헤드

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

역페이지 테이블(Inverted Page Table)은 **물리 프레임 번호(PFN)를 인덱스로 사용하는 페이지 테이블**이다. 각 엔트리가 어떤 프로세스의 어떤 가상 페이지인지 저장한다. 시스템 전체에 하나만 존재하여 메모리를 절약한다.

### 💡 비유: 호텔 투숙부

역페이지 테이블은 **호텔 투숙부**와 같다. 일반 페이지 테이블은 각 손님별로 "어느 방에 있나"를 기록한다. 역페이지 테이블은 객실 기준으로 "이 방에 누가 있나"를 기록한다. 객실이 100개면 100줄만 있으면 된다. 손님이 1000명이어도 100줄이다.

### 역페이지 테이블 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                역페이지 테이블 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【일반 vs 역페이지 테이블】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  일반 페이지 테이블 (Forward):                                        │ │
│  │  ┌────────────────────────────────────────┐                     │ │
│  │  │ VPN │ PFN │ 권한 │ ...                  │  프로세스 A           │ │
│  │  │  0  │  5  │ RW  │ ...                  │                     │ │
│  │  │  1  │  2  │ R   │ ...                  │                     │ │
│  │  │ ... │ ... │ ... │ ...                  │                     │ │
│  │  └────────────────────────────────────────┘                     │ │
│  │  ┌────────────────────────────────────────┐                     │ │
│  │  │ VPN │ PFN │ 권한 │ ...                  │  프로세스 B           │ │
│  │  │  0  │  9  │ RWX │ ...                  │                     │ │
│  │  │  1  │  3  │ R   │ ...                  │                     │ │
│  │  │ ... │ ... │ ... │ ...                  │                     │ │
│  │  └────────────────────────────────────────┘                     │ │
│  │  → 프로세스마다 페이지 테이블 존재                                        │ │
│  │  → 가상 주소 공간 크기에 비례                                          │ │
│  │                                                               │ │
│  │  역페이지 테이블 (Inverted):                                          │ │
│  │  ┌────────────────────────────────────────────────────────┐      │ │
│  │  │ PFN │ PID │ VPN │ 권한 │ ...                             │      │ │
│  │  │  0  │  1  │  5  │ RW  │ ...                             │      │ │
│  │  │  1  │  2  │  3  │ R   │ ...                             │      │ │
│  │  │  2  │  1  │  1  │ R   │ ...                             │      │ │
│  │  │  3  │  2  │  1  │ R   │ ...                             │      │ │
│  │  │  4  │  -  │  -  │ -   │ ...  (빈 프레임)                  │      │ │
│  │  │  5  │  1  │  0  │ RW  │ ...                             │      │ │
│  │  │ ... │ ... │ ... │ ... │ ...                             │      │ │
│  │  └────────────────────────────────────────────────────────┘      │ │
│  │  → 시스템 전체 하나만 존재                                            │ │
│  │  → 물리 메모리 크기에 비례                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【역페이지 테이블 장단점】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 시스템 전체 하나만 존재                                            │ │
│  │  • 크기가 물리 메모리에 비례 (가상 메모리 아님)                            │ │
│  │  • 많은 프로세스에서 메모리 절약                                        │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • VPN으로 PFN 찾기 어려움 (해시/탐색 필요)                              │ │
│  │  • 주소 변환 느림                                                    │ │
│  │  • 공유 메모리 구현 복잡                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 역페이지 테이블 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                역페이지 테이블 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【역페이지 테이블 구조】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  엔트리 구조:                                                          │ │
│  │  ┌────────────────────────────────────────────────────────┐      │ │
│  │  │ PID │ VPN │ 체인 │ 권한 │ 기타                             │      │ │
│  │  │ 16비트│ 48비트│ 32비트│ 8비트 │ ...                             │      │
│  │  └────────────────────────────────────────────────────────┘      │ │
│  │                                                               │ │
│  │  • PID: 프로세스 ID (어느 프로세스의 페이지인가)                           │ │
│  │  • VPN: Virtual Page Number (가상 페이지 번호)                         │ │
│  │  • 체인: 충돌 시 연결 리스트                                           │ │
│  │  • 권한: R/W/X 등                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해시 기반 탐색】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  역페이지 테이블은 VPN으로 직접 인덱싱 불가                               │ │
│  │  → 해시 함수 사용: Hash(PID, VPN) → 인덱스                             │ │
│  │                                                               │ │
│  │  탐색 과정:                                                          │ │
│  │  1. 해시 계산: idx = Hash(PID, VPN)                                  │ │
│  │  2. 역페이지 테이블[idx] 접근                                          │ │
│  │  3. (PID, VPN) 일치 확인                                             │ │
│  │     - 일치: PFN = idx (인덱스가 곧 프레임 번호)                          │ │
│  │     - 불일치: 체인 따라 다음 엔트리 확인                                  │ │
│  │  4. 찾으면 PFN 반환, 없으면 Page Fault                                 │ │
│  │                                                               │ │
│  │  체이닝 (Chaining):                                                   │ │
│  │  ┌─────┐    ┌─────┐    ┌─────┐                                   │ │
│  │  │Entry│ → │Entry│ → │Entry│ → NULL                              │ │
│  │  └─────┘    └─────┘    └─────┘                                   │ │
│  │  → 해시 충돌 시 연결 리스트로 연결                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【크기 비교】                                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시스템: 100개 프로세스, 각 4GB 가상 주소 공간, 16GB 물리 메모리           │ │
│  │                                                               │ │
│  │  일반 페이지 테이블:                                                   │ │
│  │  100 × 4GB / 4KB × 4B = 100 × 4MB = 400MB                        │ │
│  │                                                               │ │
│  │  역페이지 테이블:                                                       │ │
│  │  16GB / 4KB × 12B = 4M × 12B ≈ 48MB                               │ │
│  │                                                               │ │
│  │  → 역페이지 테이블이 약 8배 작음                                         │ │
│  │  → 프로세스가 많을수록 절약 효과 큼                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 역페이지 테이블 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                역페이지 테이블 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【역페이지 테이블 사용 시스템 확인】                                                 │)
│  ──────────────────                                                  │
│  # PowerPC 시스템에서                                               │
│  $ cat /proc/cpuinfo | grep -i "page table"                        │
│                                                                     │
│  # x86/ARM은 일반 페이지 테이블 사용                                    │
│  $ cat /proc/cpuinfo | grep -i "address sizes"                     │
│                                                                     │
│  【메모리 절약 효과 계산】                                                           │)
│  ──────────────────                                                  │
│  # 물리 메모리 확인                                                   │
│  $ cat /proc/meminfo | grep MemTotal                               │
│  MemTotal:        16384000 kB    # 16GB                            │
│                                                                     │
│  # 역페이지 테이블 크기 추정                                           │
│  # 프레임 수 = 16GB / 4KB = 4M                                       │
│  # 엔트리 크기 = 약 16바이트 (PID + VPN + 체인 + 기타)                   │
│  # 역페이지 테이블 = 4M × 16B = 64MB                                  │
│                                                                     │
│  # 일반 페이지 테이블 (100개 프로세스, 4GB 각각)                           │
│  # 100 × 1M × 4B = 400MB                                            │
│  # → 역페이지 테이블로 336MB 절약 (84%)                                  │
│                                                                     │
│  【해시 함수 선택】                                                                │)
│  ──────────────────                                                  │
│  • 간단한 해시: (PID XOR VPN) mod 테이블_크기                            │
│  • 좋은 해시: CRC32, MurmurHash                                      │
│  • 하드웨어 지원: PowerPC의 해시 페이지 테이블                              │
│                                                                     │
│  【최적화 팁】                                                                    │)
│  ──────────────────                                                  │
│  1. TLB 적극 활용: 탐색 오버헤드 감소                                   │
│  2. 좋은 해시 함수: 충돌 최소화                                         │
│  3. 체인 길이 제한: 탐색 시간 보장                                       │
│  4. 병렬 탐색: 멀티코어 활용                                            │
│  5. 캐시: 자주 접근하는 엔트리                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 역페이지 테이블: PFN을 인덱스로 사용하는 페이지 테이블
• 크기: 물리 메모리에 비례 (가상 메모리 아님)
• 특징: 시스템 전체 하나만 존재
• 장점: 메모리 절약, 특히 많은 프로세스
• 단점: VPN으로 PFN 찾기 어려움 (해시 필요)
• 해시: Hash(PID, VPN)로 인덱스 계산
• 체이닝: 해시 충돌 시 연결 리스트
• 엔트리: PID + VPN + 권한 + 체인
• 사용처: PowerPC, IA-64, IBM mainframe
• 미사용: x86, ARM (일반 페이지 테이블 사용)
```
