# 308. 메모리 맵 파일 (Memory-Mapped File)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일을 메모리처럼 직접 접근하는 기법
> 2. **가치**: 빠른 I/O, 간단한 코드, 공유 메모리
> 3. **융합**: 가상 메모리, 페이지 폴트, mmap, 공유

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

메모리 맵 파일(Memory-Mapped File)은 **파일을 가상 메모리 주소 공간에 매핑하여, 파일을 메모리처럼 직접 읽고 쓸 수 있게 하는 기법**이다. read/write 시스템 콜 없이 포인터로 파일에 접근한다.

### 💡 비유: 책을 책장에 바로 꽂기

메모리 맵 파일은 **책을 책장에 바로 꽂기**와 같다. 일반 I/O는 사서에게 달라고 부탁하는 거다. "123페이지 주세요." 메모리 맵은 책을 책장에 바로 꽂아두는 거다. 필요하면 바로 읽는다. 사서(read/write) 안 거쳐도 된다.

### 메모리 맵 파일 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 맵 파일 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【일반 I/O vs 메모리 맵】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  일반 I/O (read/write):                                               │
│  │                                                               │ │
│  │  프로세스         커널 버퍼        디스크                            │ │
│  │  ┌────────┐      ┌────────┐      ┌────────┐                       │ │
│  │  │ 버퍼   │ ←─── │ 버퍼   │ ←─── │ 파일   │                       │ │
│  │  │(사용자) │ read │(커널)  │ I/O  │        │                       │ │
│  │  └────────┘      └────────┘      └────────┘                       │ │
│  │                                                               │ │
│  │  → 데이터 복사 2번 (커널 버퍼, 사용자 버퍼)                              │ │
│  │  → 시스템 콜 오버헤드                                                │ │
│  │                                                               │ │
│  │  메모리 맵 (mmap):                                                      │
│  │                                                               │ │
│  │  프로세스                파일                                         │ │
│  │  ┌────────────────┐     ┌────────┐                                  │ │
│  │  │ 가상 주소 공간  │ ←── │ 파일   │                                  │ │
│  │  │    (매핑)      │     │        │                                  │ │
│  │  └────────────────┘     └────────┘                                  │ │
│  │                                                               │ │
│  │  → 파일이 가상 메모리에 직접 매핑                                        │ │
│  │  → 데이터 복사 없음 (페이지 폴트 시만 로드)                               │ │
│  │  → 포인터로 직접 접근                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【mmap 동작 과정】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. mmap() 시스템 콜 호출:                                             │ │
│  │     void *addr = mmap(NULL, length, PROT_READ, MAP_SHARED, fd, 0);│ │
│  │                                                               │ │
│  │  2. 가상 주소 공간에 매핑 생성:                                          │ │
│  │     • 페이지 테이블 엔트리 생성                                         │ │
│  │     • 파일 오프셋과 가상 주소 매핑                                       │ │
│  │     • 아직 데이터는 메모리에 없음                                        │ │
│  │                                                               │ │
│  │  3. 접근 시 Page Fault 발생:                                           │ │
│  │     • 처음 접근 → Page Fault                                          │ │
│  │     • OS가 파일에서 페이지 로드                                         │ │
│  │     • 요구 페이징 (Demand Paging)                                      │ │
│  │                                                               │ │
│  │  4. 이후 접근:                                                          │ │
│  │     • 메모리에 있으면 바로 접근                                         │ │
│  │     • 쓰기 시 Dirty bit 설정 → 나중에 파일에 반영                         │ │
│  │                                                               │ │
│  │  5. munmap()으로 매핑 해제                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 맵 장단점】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 빠른 I/O (복사 없음, 시스템 콜 없음)                                 │ │
│  │  • 간단한 코드 (포인터로 접근)                                          │ │
│  │  • 자동 캐싱 (OS가 페이지 캐시 관리)                                    │ │
│  │  • 공유 가능 (여러 프로세스가 같은 파일 매핑)                              │ │
│  │  • 큰 파일 처리 용이                                                   │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 메모리 주소 공간 사용                                                │ │
│  │  • 포인터 위험 (버그 시 파일 손상)                                       │ │
│  │  • 동기화 필요 (공유 시)                                               │ │
│  │  • 파일 크기 변경 어려움                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 메모리 맵 파일 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 맵 파일 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【mmap 파라미터】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  void *mmap(void *addr, size_t length, int prot,                      │ │
│  │              int flags, int fd, off_t offset);                         │ │
│  │                                                               │ │
│  │  addr:    매핑 시작 주소 (NULL이면 OS가 선택)                           │ │
│  │  length:  매핑 길이 (바이트)                                           │ │
│  │  prot:    보호 권한                                                     │ │
│  │           PROT_READ  - 읽기 가능                                        │ │
│  │           PROT_WRITE - 쓰기 가능                                        │ │
│  │           PROT_EXEC  - 실행 가능                                        │ │
│  │           PROT_NONE  - 접근 불가                                        │ │
│  │  flags:   매핑 타입                                                     │ │
│  │           MAP_SHARED  - 변경 사항 파일에 반영                            │ │
│  │           MAP_PRIVATE - Copy-on-Write, 파일 변경 안 함                  │ │
│  │           MAP_ANONYMOUS - 파일 없이 익명 매핑                           │ │
│  │  fd:      파일 디스크립터                                               │ │
│  │  offset:  파일 시작 오프셋 (페이지 크기 정렬)                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MAP_SHARED vs MAP_PRIVATE】                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  MAP_SHARED:                                                            │ │
│  │  • 쓰기 시 파일에 반영                                                  │ │
│  │  • 여러 프로세스가 공유                                                 │ │
│  │  • IPC에 활용 가능                                                     │ │
│  │  • 쓰기 시 모든 매핑에 보임                                              │ │
│  │                                                               │ │
│  │  MAP_PRIVATE:                                                           │ │
│  │  • 쓰기 시 COW (Copy-on-Write)                                         │ │
│  │  • 원본 파일 변경 안 됨                                                │ │
│  │  • 각 프로세스 독립적 복사본                                            │ │
│  │  • 파일 읽기 전용으로 수정은 로컬                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【공유 메모리 (Shared Memory)】                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  익명 매핑 + MAP_SHARED:                                                │ │
│  │  mmap(NULL, size, PROT_READ|PROT_WRITE,                              │ │
│  │       MAP_SHARED|MAP_ANONYMOUS, -1, 0);                               │ │
│  │                                                               │ │
│  │  • 파일 없이 메모리 공간 생성                                           │ │
│  │  • fork() 시 자식과 공유                                                │ │
│  │  • IPC로 사용 가능                                                     │ │
│  │  • shm_open + mmap과 유사                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 메모리 맵 파일 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 맵 파일 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【mmap 사용 예시 (C)】                                                           │)
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                              │
│  #include <fcntl.h>                                                 │
│  #include <unistd.h>                                                │
│                                                                     │
│  int fd = open("file.txt", O_RDONLY);                               │
│  size_t size = lseek(fd, 0, SEEK_END);                              │
│  char *data = mmap(NULL, size, PROT_READ, MAP_PRIVATE, fd, 0);      │
│                                                                     │
│  // 파일 데이터를 메모리처럼 직접 접근                                  │
│  printf("%c\n", data[0]);                                           │
│                                                                     │
│  munmap(data, size);                                                │
│  close(fd);                                                         │
│                                                                     │
│  【mmap 사용 예시 (Python)】                                                       │)
│  ──────────────────                                                  │
│  import mmap                                                        │
│                                                                     │
│  with open("file.txt", "r+b") as f:                                 │
│      mm = mmap.mmap(f.fileno(), 0)                                  │
│      print(mm[0:10])  # 첫 10바이트 읽기                               │
│      mm.close()                                                     │
│                                                                     │
│  【메모리 맵 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/[pid]/maps | grep -i mmap                              │
│  7f... 7f... rw-s 00000000 08:01 12345 /path/to/file                │
│  # 's' = shared mapping                                             │
│                                                                     │
│  【메모리 맵 통계】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/meminfo | grep -i mapped                                │
│  Mapped:            123456 kB    # mmap된 페이지                      │
│                                                                     │
│  【메모리 맵 최적화 팁】                                                         │)
│  ──────────────────                                                  │
│  1. 큰 파일: mmap이 효율적                                            │
│  2. 랜덤 접근: mmap이 빠름                                             │
│  3. 순차 접근: read/write도 좋음                                      │
│  4. 공유: MAP_SHARED로 IPC                                           │
│  5. 읽기 전용: MAP_PRIVATE로 안전                                      │
│  6. 페이지 정렬: offset을 페이지 크기로                                 │
│  7. msync: 변경 사항 동기화                                            │
│  8. munmap: 사용 후 해제                                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 메모리 맵 파일: 파일을 가상 메모리에 매핑
• 시스템 콜: mmap(), munmap(), msync()
• 동작: 요구 페이징으로 페이지 로드
• 장점: 빠른 I/O, 복사 없음, 간단한 코드, 공유 가능
• 단점: 주소 공간 사용, 포인터 위험, 동기화 필요
• MAP_SHARED: 변경 사항 파일에 반영, 공유
• MAP_PRIVATE: COW, 원본 파일 변경 안 됨
• MAP_ANONYMOUS: 파일 없이 메모리 공간 생성
• 사용처: 큰 파일, 랜덤 접근, IPC, DB
• 관련: 가상 메모리, 페이지 폴트, 캐시
```
