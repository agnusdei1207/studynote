# 299. 페이지 교체 (Page Replacement)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리가 꽉 찼을 때 희생자 페이지를 선택해 교체
> 2. **가치**: 제한된 메모리로 많은 페이지 관리
> 3. **융합**: 알고리즘, 스와핑, 스래싱, 워킹셋

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

페이지 교체(Page Replacement)는 **물리 메모리가 꽉 찼을 때 희생자(Victim) 페이지를 선택하여 디스크로 내보내고, 새 페이지를 로드하는 과정**이다. 페이지 교체 알고리즘이 희생자를 선택한다.

### 💡 비유: 책장 정리

페이지 교체는 **책장 정리**와 같다. 책장이 꽉 찼다. 새 책을 두려면 기존 책을 빼야 한다. 어떤 책을 뺄까? 안 읽는 책? 오래된 책? 자주 읽는 책은 두고, 덜 읽는 책을 창고로 보낸다.

### 페이지 교체 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 교체 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 교체 발생 상황】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Page Fault 발생                                                  │ │
│  │     → 새 페이지 필요                                                 │ │
│  │                                                               │ │
│  │  2. 빈 프레임 확인                                                     │ │
│  │     → 빈 프레임 있음: 바로 로드                                         │ │
│  │     → 빈 프레임 없음: 페이지 교체 필요                                   │ │
│  │                                                               │ │
│  │  3. 희생자 선택 (Victim Selection)                                    │ │
│  │     → 페이지 교체 알고리즘으로 선택                                      │ │
│  │                                                               │ │
│  │  4. 희생자 페이지 처리                                                  │ │
│  │     → Clean: 바로 덮어쓰기                                             │ │
│  │     → Dirty: 디스크에 저장 후 덮어쓰기                                  │ │
│  │                                                               │ │
│  │  5. 새 페이지 로드                                                      │ │
│  │     → PTE 갱신                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【기본 알고리즘 종류】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. OPT (Optimal):                                                   │ │
│  │     • 가장 오랫동안 안 쓸 페이지 교체                                     │ │
│  │     • 이론적 최적 (실현 불가)                                           │ │
│  │     • 다른 알고리즘 평가 기준                                           │ │
│  │                                                               │ │
│  │  2. FIFO (First-In First-Out):                                       │ │
│  │     • 가장 먼저 들어온 페이지 교체                                        │ │
│  │     • 구현 간단                                                       │ │
│  │     • Belady's Anomaly 문제                                          │ │
│  │                                                               │ │
│  │  3. LRU (Least Recently Used):                                       │ │
│  │     • 가장 오래 안 쓴 페이지 교체                                        │ │
│  │     • 지역성 활용                                                      │ │
│  │     • 성능 좋음, 구현 복잡                                              │ │
│  │                                                               │ │
│  │  4. Clock (Second Chance):                                           │ │
│  │     • FIFO + 참조 비트                                                 │ │
│  │     • LRU 근사                                                       │ │
│  │     • 구현 쉬움, 성능 좋음                                              │ │
│  │                                                               │ │
│  │  5. LFU (Least Frequently Used):                                     │ │
│  │     • 사용 빈도가 가장 낮은 페이지 교체                                    │ │
│  │     • 인기도 기반                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Dirty Page vs Clean Page】                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Clean Page:                                                          │ │
│  │  • 디스크 원본과 동일 (수정 안 됨)                                        │ │
│  │  • 교체 시 디스크 쓰기 불필요                                            │ │
│  │  • 교체 빠름 (버리면 됨)                                               │ │
│  │                                                               │ │
│  │  Dirty Page:                                                          │ │
│  │  • 메모리에서 수정됨 (Dirty bit = 1)                                    │ │
│  │  • 교체 시 디스크에 저장 필요                                            │ │
│  │  • 교체 느림 (I/O 필요)                                               │ │
│  │                                                               │ │
│  │  → 교체 알고리즘은 Dirty 페이지를 피하는 게 좋음                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 페이지 교체 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 교체 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 교체 비용】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Clean Page 교체:                                                     │ │
│  │  • 메모리 덮어쓰기만                                                   │ │
│  │  • 비용: 수 μs                                                       │ │
│  │                                                               │ │
│  │  Dirty Page 교체:                                                     │ │
│  │  • 디스크 쓰기 + 메모리 덮어쓰기                                         │ │
│  │  • 비용: 1~10 ms (1000배 느림)                                         │ │
│  │                                                               │ │
│  │  교체 피하기 (Avoidance):                                              │ │
│  │  • Dirty 비율 낮추기                                                   │ │
│  │  • 자주 쓰는 페이지 유지                                                │ │
│  │  • 워킹셋을 메모리에 유지                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Belady's Anomaly (벨레이디의 이상 현상)】                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  FIFO에서 프레임 수를 늘려도 페이지 폴트가 증가하는 현상                    │ │
│  │                                                               │ │
│  │  예: 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5                             │ │
│  │                                                               │ │
│  │  3 프레임: 9 폴트                                                    │ │
│  │  4 프레임: 10 폴트 (더 많음!)                                          │ │
│  │                                                               │ │
│  │  → FIFO의 문제점                                                    │ │
│  │  → LRU, Clock는 이상 현상 없음 (Stack Algorithm)                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지역 교체 vs 전역 교체】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  지역 교체 (Local Replacement):                                        │ │
│  │  • 각 프로세스 자신의 프레임에서만 교체                                     │ │
│  │  • 프로세스 격리                                                      │ │
│  │  • 메모리 활용 비효율                                                  │ │
│  │                                                               │ │
│  │  전역 교체 (Global Replacement):                                       │ │
│  │  • 전체 시스템 프레임에서 교체                                            │ │
│  │  • 메모리 활용 효율                                                   │ │
│  │  • 프로세스 간 영향                                                   │ │
│  │                                                               │ │
│  │  → 현대 OS는 대부분 전역 교체 사용                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 페이지 교체: 메모리 꽉 찼을 때 희생자 선택 후 교체
• 목적: 제한된 메모리로 더 많은 페이지 관리
• 알고리즘: OPT, FIFO, LRU, Clock, LFU
• OPT: 이론적 최적, 실현 불가
• FIFO: 간단하지만 Belady's Anomaly
• LRU: 성능 좋음, 구현 복잡
• Clock: LRU 근사, 구현 쉬움, 많이 사용
• Dirty Page: 교체 시 디스크 쓰기 필요
• Clean Page: 바로 덮어쓰기 가능
• 전역 vs 지역: 현대 OS는 전역 교체
• 성능: 알고리즘 선택이 폴트율에 큰 영향
```
