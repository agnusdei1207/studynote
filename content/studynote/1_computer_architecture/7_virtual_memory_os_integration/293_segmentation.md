# 293. 세그멘테이션 (Segmentation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가변 크기 블록(세그먼트) 단위 메모리 관리
> 2. **가치**: 논리적 단위 관리, 내부 단편화 없음
> 3. **융합**: 페이징과 결합, x86 세그먼트, 보호

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

세그멘테이션(Segmentation)은 **가변 크기 블록(세그먼트) 단위로 메모리를 관리하는 기법**이다. 코드, 데이터, 스택 등 논리적 단위로 메모리를 나눈다. 내부 단편화가 없지만 외부 단편화가 발생한다.

### 💡 비유: 챕터 단위 책

세그멘테이션은 **챕터 단위 책**과 같다. 책이 챕터별로 나뉜다. 1장은 50페이지, 2장은 30페이지, 3장은 80페이지. 각 챕터(세그먼트) 크기가 다르다. 챕터는 논리적 단위다. 페이지(고정 크기)와 달리 챕터는 가변 크기다.

### 세그멘테이션 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                세그멘테이션 개요                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【세그먼트 정의】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  세그먼트 (Segment):                                                  │ │
│  │  • 논리적 단위로 구성된 가변 크기 메모리 블록                              │ │
│  │  • 크기: 가변 (코드, 데이터, 스택 등 크기 다름)                           │ │
│  │  • 주소: [세그먼트 번호 | 오프셋]                                       │ │
│  │                                                               │ │
│  │  주소 구조:                                                            │ │
│  │  ┌────────────────────────┬────────────────────────┐            │ │
│  │  │    세그먼트 번호          │       오프셋             │            │ │
│  │  │    (Segment #)          │     (Offset)           │            │ │
│  │  └────────────────────────┴────────────────────────┘            │ │
│  │                                                               │ │
│  │  대표 세그먼트:                                                        │ │
│  │  • Code Segment: 실행 코드                                           │ │
│  │  • Data Segment: 전역 변수, 정적 변수                                  │ │
│  │  • Stack Segment: 지역 변수, 호출 스택                                 │ │
│  │  • Heap Segment: 동적 할당 메모리                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【세그먼트 테이블】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  세그먼트 번호  │  Base    │  Limit   │  권한  │ 기타            │ │
│  │  ────────       │  ────    │  ─────   │  ────  │ ────            │ │
│  │  0 (Code)       │  0x1000  │  4KB     │  R-X   │ ...             │ │
│  │  1 (Data)       │  0x5000  │  8KB     │  RW-   │ ...             │ │
│  │  2 (Stack)      │  0xF000  │  2KB     │  RW-   │ ...             │ │
│  │  3 (Heap)       │  0x8000  │  16KB    │  RW-   │ ...             │ │
│  │                                                               │ │
│  │  • Base: 세그먼트 시작 주소 (물리 메모리)                                │ │
│  │  • Limit: 세그먼트 크기                                               │ │
│  │  • 권한: R/W/X                                                       │ │
│  │                                                               │ │
│  │  주소 변환:                                                            │ │
│  │  물리 주소 = Base[세그먼트 번호] + 오프셋                               │ │
│  │  단, 오프셋 < Limit이어야 함                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【세그멘테이션 장단점】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 논리적 단위 관리 (코드, 데이터, 스택)                                  │ │
│  │  • 내부 단편화 없음                                                    │ │
│  │  • 보호 단위가 논리적                                                  │ │
│  │  • 공유 단위가 논리적                                                  │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 외부 단편화 발생                                                    │ │
│  │  • 메모리 할당 복잡                                                   │ │
│  │  • 메모리 압축(Compaction) 필요                                        │ │
│  │  • 구현 복잡도 높음                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 세그멘테이션 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                세그멘테이션 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 변환 과정】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  논리 주소: [세그먼트 1 | 오프셋 0x500]                                 │ │
│  │                                                               │ │
│  │  1. 세그먼트 테이블 조회:                                               │ │
│  │     Base[1] = 0x5000, Limit[1] = 8KB (0x2000)                      │ │
│  │                                                               │ │
│  │  2. 경계 확인:                                                        │ │
│  │     오프셋 0x500 < Limit 0x2000? → Yes (정상)                         │ │
│  │     (No면 Segment Fault)                                           │ │
│  │                                                               │ │
│  │  3. 물리 주소 계산:                                                   │ │
│  │     물리 주소 = Base + 오프셋 = 0x5000 + 0x500 = 0x5500              │ │
│  │                                                               │ │
│  │  4. 권한 확인:                                                        │ │
│  │     쓰기 요청인데 Data 세그먼트? → OK                                   │ │
│  │     쓰기 요청인데 Code 세그먼트? → Fault                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【외부 단편화 문제】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  물리 메모리 상태:                                                      │ │
│  │  ┌────────┐ ┌───┐ ┌──────────┐ ┌───┐ ┌──────┐ ┌─┐ ┌───────┐    │ │
│  │  │ Code   │ │   │ │ Data     │ │   │ │Stack │ │ │ │ Heap  │    │ │
│  │  │ 4KB    │ │2KB│ │ 8KB      │ │1KB│ │2KB   │ │1│ │ 16KB  │    │ │
│  │  └────────┘ └───┘ └──────────┘ └───┘ └──────┘ └─┘ └───────┘    │ │
│  │             ↑                 ↑           ↑                     │ │
│  │             └────── 빈 공간 (외부 단편화) ──────┘                  │ │
│  │                                                               │ │
│  │  총 빈 공간: 2KB + 1KB + 1KB = 4KB                                  │ │
│  │  하지만 4KB 세그먼트 할당 불가 (연속되지 않음)                           │ │
│  │                                                               │ │
│  │  → 외부 단편화: 총 빈 공간은 충분하지만 연속되지 않아 할당 불가             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【세그멘테이션 vs 페이징】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성              세그멘테이션         페이징                      │ │
│  │  ──────            ────────────         ──────                    │ │
│  │  블록 크기          가변                  고정                      │ │
│  │  블록 단위          논리적 (코드, 데이터)   물리적 (페이지)             │ │
│  │  내부 단편화        없음                  있음                      │ │
│  │  외부 단편화        있음                  없음                      │ │
│  │  주소 변환          Base + Offset         VPN → PFN                │ │
│  │  보호 단위          논리적                물리적                     │ │
│  │  공유              논리적 단위             페이지 단위                 │ │
│  │  구현 복잡도        높음                  낮음                      │ │
│  │  현대 OS            거의 안 씀             대부분 사용                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 세그멘테이션 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                세그멘테이션 × 타 과목 융합                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 세그먼트 레지스터: CS, DS, SS, ES (x86)                             │
│  • GDT/LDT: Global/Local Descriptor Table                            │
│  • 세그먼트 보호: 권한 비트 (R/W/X)                                      │
│  • x86-64: 세그멘테이션 거의 사용 안 함                                   │
│                                                                     │
│  【운영체제】                                                          │
│  • 메모리 할당: First Fit, Best Fit, Worst Fit                        │
│  • 외부 단편화 해결: Compaction                                        │
│  • 현대 OS: 페이징 사용, 세그멘테이션 안 씀                                │
│  • 결합: Paged Segmentation (세그먼트 내 페이징)                         │
│                                                                     │
│  【보안】                                                              │
│  • 세그먼트 보호: 논리적 단위별 권한                                       │
│  • Code 세그먼트: 읽기/실행만                                           │
│  • Data 세그먼트: 읽기/쓰기                                             │
│  • 스택 보호: Stack Canary와 연계                                       │
│                                                                     │
│  【현대 시스템】                                                        │
│  • x86-64: Flat Model (세그먼트 0~전체)                                │
│  • ARM: 세그멘테이션 없음                                               │
│  • RISC-V: 세그멘테이션 없음                                            │
│  • 역사적: 16비트 x86에서 중요                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  세그멘테이션과 페이징 결합:                                                            │ │
│  │  ──────────────────────                                              │ │
│  │  Paged Segmentation:                                                             │ │
│  │  1. 논리 주소 → 세그먼트 테이블 → 세그먼트 내 주소                         │ │
│  │  2. 세그먼트 내 주소 → 페이지 테이블 → 물리 주소                           │ │
│  │                                                               │ │
│  │  장점:                                                               │ │
│  │  • 논리적 단위 보호 (세그멘테이션)                                      │ │
│  │  • 외부 단편화 없음 (페이징)                                          │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 2번의 변환 필요                                                   │ │
│  │  • 복잡도 증가                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 세그멘테이션 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                세그멘테이션 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86 세그먼트 확인】                                                           │)
│  ──────────────────                                                  │
│  # 세그먼트 레지스터                                                   │
│  $ gdb                                                              │
│  (gdb) info registers                                               │
│  cs    0x33    51    # Code Segment                                 │
│  ds    0x0     0     # Data Segment (64비트에서 대부분 0)               │
│  ss    0x2b    43    # Stack Segment                                │
│                                                                     │
│  # 64비트에서는 Flat Model 사용                                        │
│  # 모든 세그먼트가 전체 주소 공간 커버                                     │
│                                                                     │
│  【메모리 맵 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/self/maps                                              │
│  address           perms offset  dev   inode   pathname             │
│  55a5c2a000-55a5c2d000 r--p ...  /bin/ls    # Code (Read-only)      │
│  55a5c2d000-55a5c3e000 r-xp ...  /bin/ls    # Code (Executable)     │
│  55a5c42000-55a5c43000 r--p ...  /bin/ls    # Data (Read-only)      │
│  55a5c43000-55a5c44000 rw-p ...  /bin/ls    # Data (Read-Write)     │
│  7fff5fbff000-7fff5fc00000 rw-p ... [stack] # Stack                 │
│                                                                     │
│  → 현대 OS는 페이징으로 구현하지만 논리적 세그먼트 개념 유지                  │
│                                                                     │
│  【외부 단편화 시뮬레이션】                                                         │)
│  ──────────────────                                                  │
│  # 메모리 할당/해제 패턴                                               │
│  malloc(1000) → 1KB 세그먼트 할당                                      │
│  malloc(5000) → 5KB 세그먼트 할당                                      │
│  free(1KB)    → 1KB 빈 공간 발생                                       │
│  malloc(2000) → 2KB 요청, 1KB 빈 공간 사용 불가 → 외부 단편화!           │
│                                                                     │
│  【세그멘테이션 최적화 팁】                                                         │)
│  ──────────────────                                                  │
│  1. 현대 OS: 페이징 사용 권장                                           │
│  2. 외부 단편화: Compaction 또는 페이징으로 해결                           │
│  3. 논리적 구분: 페이지 권한 비트로 대체                                   │
│  4. 보호: NX 비트, R/W 권한으로 세그먼트 보호 대체                         │
│  5. 공유: 페이지 공유로 대체                                             │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 세그멘테이션: 가변 크기 블록(세그먼트) 단위 메모리 관리
• 세그먼트: 논리적 단위 (코드, 데이터, 스택, 힙)
• 주소: [세그먼트 번호 | 오프셋]
• 변환: 물리 주소 = Base + 오프셋
• 장점: 내부 단편화 없음, 논리적 보호/공유
• 단점: 외부 단편화, 할당 복잡
• x86: CS, DS, SS, ES 세그먼트 레지스터
• 현대 OS: 페이징 사용, 세그멘테이션 거의 안 씀
• 결합: Paged Segmentation (장점 결합)
• 대체: 페이지 권한 비트로 세그먼트 기능 대체
```
