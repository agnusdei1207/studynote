# 294. 페이지 공유 (Page Sharing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 프로세스가 같은 물리 페이지를 공유
> 2. **가치**: 메모리 절약, IPC, 코드 공유
> 3. **융합**: COW, shared memory, mmap, 라이브러리

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

페이지 공유(Page Sharing)는 **여러 프로세스가 동일한 물리 프레임을 가리키는 기법**이다. 각 프로세스의 페이지 테이블 엔트리가 같은 PFN을 가리킨다. 메모리 절약, IPC, 코드 공유에 사용된다.

### 💡 비유: 도서관 참고서

페이지 공유는 **도서관 참고서**와 같다. 여러 학생이 같은 참고서를 본다. 각자 사본을 가질 필요 없다. 한 권만 있으면 된다. 읽기만 하면 문제없다. 쓰려면 그때 사본을 만든다.

### 페이지 공유 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 공유 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 공유 구조】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  프로세스 A                프로세스 B                            │ │
│  │  ┌──────────┐            ┌──────────┐                          │ │
│  │  │ Page 0   │            │ Page 0   │                          │ │
│  │  │ → PFN 5  │            │ → PFN 5  │                          │ │
│  │  ├──────────┤            ├──────────┤                          │ │
│  │  │ Page 1   │            │ Page 1   │                          │ │
│  │  │ → PFN 2  │            │ → PFN 3  │                          │ │
│  │  ├──────────┤            ├──────────┤                          │ │
│  │  │ Page 2   │            │ Page 2   │                          │ │
│  │  │ → PFN 9  │            │ → PFN 5  │                          │ │
│  │  └──────────┘            └──────────┘                          │ │
│  │        │                       │                                │ │
│  │        │     ┌─────────────────┘                                │ │
│  │        │     │                                                  │ │
│  │        ▼     ▼                                                  │ │
│  │  물리 메모리 (RAM):                                               │ │
│  │  ┌──────────┐                                                    │ │
│  │  │ Frame 5  │ ← A의 Page 0, B의 Page 0, B의 Page 2 공유           │ │
│  │  │ (공유)    │                                                    │ │
│  │  └──────────┘                                                    │ │
│  │                                                               │ │
│  │  → 같은 PFN을 가리키면 페이지 공유                                    │ │
│  │  → 읽기만 하면 안전                                                 │ │
│  │  → 쓰기는 Copy-on-Write 필요                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 공유 사용 사례】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 코드 공유 (Code Sharing):                                        │ │
│  │     • 공유 라이브러리 (libc.so 등)                                     │ │
│  │     • 실행 파일 코드 세그먼트                                          │ │
│  │     • 읽기 전용 → 쓰기 없음 → 안전                                     │ │
│  │                                                               │ │
│  │  2. 프로세스 생성 (fork):                                             │ │
│  │     • 부모-자식 간 페이지 공유                                         │ │
│  │     • 쓰기 시 Copy-on-Write                                          │ │
│  │                                                               │ │
│  │  3. 공유 메모리 IPC:                                                 │ │
│  │     • shmget, shmat                                                │ │
│  │     • 프로세스 간 통신                                                 │ │
│  │                                                               │ │
│  │  4. 메모리 맵 파일:                                                   │ │
│  │     • mmap으로 파일을 메모리에 매핑                                     │ │
│  │     • 여러 프로세스가 같은 파일 매핑                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 공유 주의점】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 읽기 전용만 안전                                                   │ │
│  │  • 쓰기 필요 시 Copy-on-Write 사용                                    │ │
│  │  • 페이지 교체 시 참조 카운트 관리                                       │ │
│  │  • 일관성 문제 고려                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 페이지 공유 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 공유 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【공유 라이브러리 예시】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  100개 프로세스가 libc.so 사용:                                       │ │
│  │                                                               │ │
│  │  공유 없이:                                                           │ │
│  │  100 × 2MB (libc) = 200MB                                         │ │
│  │                                                               │ │
│  │  공유 사용:                                                           │ │
│  │  1 × 2MB (libc) = 2MB                                             │ │
│  │                                                               │ │
│  │  절약: 198MB (99% 절약!)                                            │ │
│  │                                                               │ │
│  │  libc.so 공유 구조:                                                   │ │
│  │  ┌────────────────────────────────────────────────────┐          │ │
│  │  │ 물리 메모리 (libc 코드)                              │          │ │
│  │  │ Frame 1000~1500 (2MB)                              │          │ │
│  │  └────────────────────────────────────────────────────┘          │ │
│  │        ↑         ↑         ↑              ↑                       │ │
│  │     프로세스1   프로세스2   프로세스3  ...  프로세스100              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Copy-on-Write (COW)】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  쓰기 공유 페이지 처리:                                                │ │
│  │                                                               │ │
│  │  1. 초기: 페이지를 읽기 전용으로 공유                                    │ │
│  │     PTE: Read-only, Shared                                       │ │
│  │                                                               │ │
│  │  2. 쓰기 시도: Page Fault 발생                                       │ │
│  │     → 하드웨어가 쓰기 시도 감지                                         │ │
│  │     → OS가 Page Fault 핸들러 호출                                    │ │
│  │                                                               │ │
│  │  3. COW 처리:                                                        │ │
│  │     a) 새 물리 프레임 할당                                             │ │
│  │     b) 원본 페이지 내용 복사                                           │ │
│  │     c) 페이지 테이블 업데이트                                          │ │
│  │     d) 쓰기 권한 부여                                                 │ │
│  │     e) 재실행                                                        │ │
│  │                                                               │ │
│  │  4. 이후: 각자 독립적인 페이지                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【참조 카운팅】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  각 물리 프레임마다 참조 카운트 관리:                                      │ │
│  │                                                               │ │
│  │  PFN    참조 카운트    상태                                         │ │
│  │  ───    ──────────    ────                                         │ │
│  │  100    3           3개 프로세스가 공유                                 │ │
│  │  101    1           1개 프로세스만 사용                                 │ │
│  │  102    0           Free (해제 가능)                                   │ │
│  │                                                               │ │
│  │  페이지 해제 시:                                                        │ │
│  │  참조 카운트-- → 0이면 실제 프레임 해제                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 페이지 공유 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 공유 × 타 과목 융합                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • MMU: 공유 페이지 권한 관리                                          │
│  • TLB: 공유 페이지도 TLB 캐시                                         │
│  • 캐시 일관성: 공유 메모리 캐시 동기화                                   │
│  • 다중 코어: 공유 메모리 접근 동기화                                     │
│                                                                     │
│  【운영체제】                                                          │
│  • fork(): COW로 페이지 공유                                          │
│  • 프로그램 로드: 코드 세그먼트 공유                                      │
│  • 공유 메모리: shmget, shmat, shmdt                                  │
│  • mmap(): 파일 매핑 공유                                              │
│  • 페이지 교체: 참조 카운트 고려                                          │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 버퍼 풀 공유: 여러 쿼리가 같은 페이지                                   │
│  • 공유 메모리: DB 버퍼 캐시                                            │
│  • mmap: 데이터 파일 매핑                                              │
│  • 동시성: 락, 래치                                                    │
│                                                                     │
│  【보안】                                                              │
│  • 권한 관리: 공유 페이지 접근 제어                                       │
│  • 격리: 프로세스 간 보호                                               │
│  • Side-channel: 공유 메모리 공격                                      │
│  • 권한 상승: 공유 라이브러리 취약점                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  공유 메모리 IPC vs 파이프:                                                          │ │
│  │  ──────────────────────                                              │ │
│  │  특성              공유 메모리         파이프                          │ │
│  │  ──────            ──────              ────                            │ │
│  │  속도              매우 빠름            느림 (복사)                     │ │
│  │  동기화            직접 구현            자동                           │ │
│  │  복잡도            높음                 낮음                           │ │
│  │  용도              대용량 데이터        작은 데이터                      │ │
│  │  방향              양방향               단방향                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 페이지 공유: 여러 프로세스가 같은 물리 페이지 공유
• 메모리 절약: 코드, 라이브러리 공유
• 읽기 전용: 안전, 쓰기 없으면 문제없음
• 쓰기: Copy-on-Write로 처리
• 사용처: 공유 라이브러리, fork, IPC, mmap
• 참조 카운트: 공유 페이지 관리
• 장점: 메모리 절약, 빠른 IPC
• 단점: 동기화 필요, 복잡도 증가
• COW: 쓰기 시 복사, 공유 유지
• 실무: ldd, ipcs, pmap으로 확인
```
