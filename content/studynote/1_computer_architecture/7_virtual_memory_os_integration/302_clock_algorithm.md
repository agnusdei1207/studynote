# 302. Clock 알고리즘 (Second Chance)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: FIFO + 참조 비트로 LRU 근사
> 2. **가치**: 구현 쉬움, 성능 좋음, 실용적
> 3. **융합**: 원형 리스트, 참조 비트, 핸드 포인터

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

Clock 알고리즘은 **FIFO에 참조 비트를 추가하여 LRU를 근사하는 페이지 교체 알고리즘**이다. 원형 리스트와 포인터(Hand)를 사용하여 마치 시계처럼 페이지를 순회하며 교체 대상을 찾는다. Second Chance라고도 부른다.

### 💡 비유: 회전식 차례

Clock은 **회전식 차례**와 같다. 놀이기구 앞에 줄이 아니라 원형으로 서 있다. 차례가 돌아오면 한 명씩 확인한다. "아직 놀고 싶어?" (참조 비트). "네!" 하면 다음으로 넘어간다. "아니요" 하면 나간다. 기회를 두 번 주니까 Second Chance다.

### Clock 알고리즘 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                Clock 알고리즘 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Clock 구조】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │           ┌─────────┐                                            │ │
│  │           │ Page A  │ R=1                                        │ │
│  │           │         │                                            │ │
│  │     ┌─────┴─────────┴─────┐                                      │ │
│  │     │                     │                                      │ │
│  │ ┌───┴───┐           ┌─────┴───┐                                  │ │
│  │ │Page E │           │ Page B  │                                  │ │
│  │ │ R=0   │           │ R=1     │                                  │ │
│  │ └───┬───┘           └─────┬───┘                                  │ │
│  │     │                     │                                      │ │
│  │     │     ┌─────────┐     │                                      │ │
│  │     └────►│ Page D  │◄────┘                                      │ │
│  │           │ R=0     │                                            │ │
│  │           │         │◄── Hand (포인터)                            │ │
│  │           └─────────┘                                            │ │
│  │                                                               │ │
│  │  R = 참조 비트 (Reference Bit)                                       │ │
│  │  Hand = 현재 가리키는 위치                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Clock 동작 과정】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  교체 필요 시:                                                        │
│  │                                                               │ │
│  │  1. Hand가 가리키는 페이지 확인                                        │ │
│  │                                                               │ │
│  │  2. 참조 비트 확인:                                                    │ │
│  │     if (R == 1):                                                    │ │
│  │         R = 0        // Second Chance 끝                             │ │
│  │         Hand++       // 다음으로 이동                                  │ │
│  │         goto 1        // 다시 확인                                     │ │
│  │     else (R == 0):                                                  │ │
│  │         이 페이지 교체                                                  │ │
│  │         새 페이지 로드, R = 1                                          │ │
│  │         Hand++       // 다음으로 이동                                  │ │
│  │                                                               │ │
│  │  → 참조된 페이지(R=1)는 한 번 더 기회를 받음                              │ │
│  │  → 참조 안 된 페이지(R=0)는 교체됨                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Clock 장단점】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 구현 간단 (FIFO + 참조 비트)                                        │ │
│  │  • LRU 근사 성능                                                     │ │
│  │  • Belady's Anomaly 없음                                             │ │
│  │  • 하드웨어 지원 (참조 비트)                                           │ │
│  │  • 실제 OS에서 많이 사용                                               │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 정확한 LRU보다는 성능 떨어짐                                        │ │
│  │  • Hand 이동 오버헤드                                                 │ │
│  │  • 참조 비트 리셋 타이밍 중요                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. Clock 알고리즘 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                Clock 알고리즘 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Clock 동작 예시】                                                             │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  프레임: 3개, 참조 스트링: A, B, C, D, A, B, C, D                   │ │
│  │  초기: Hand → A, R = [0, 0, 0]                                      │ │
│  │                                                               │ │
│  │  참조  메모리      R 비트      Hand    동작                          │ │
│  │  ────  ──────      ──────      ────    ────                          │ │
│  │   A    [A,-,-]     [1,0,0]     A      폴트, A 로드                   │ │
│  │   B    [A,B,-]     [1,1,0]     B      폴트, B 로드                   │ │
│  │   C    [A,B,C]     [1,1,1]     C      폴트, C 로드                   │ │
│  │   D    [D,B,C]     [1,1,0]     B      폴트, A 교체 (R=1→0→다음)        │ │
│  │        [D,B,C]     [0,1,0]     C      B(R=1→0)                      │ │
│  │        [D,B,C]     [0,0,0]     D      C(R=1→0)                      │ │
│  │        [D,B,C]     [1,0,0]     B      D 로드 (A 자리)                │ │
│  │  ...                                                             │ │
│  │                                                               │ │
│  │  → R=1인 페이지는 한 번 R=0으로 바뀌고 다음으로                         │ │
│  │  → R=0인 페이지가 교체됨                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Enhanced Clock (개선된 Clock)】                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  참조 비트 + 수정 비트 사용:                                            │ │
│  │                                                               │ │
│  │  클래스:                                                              │ │
│  │  0: R=0, D=0  (최우선 교체 - Clean, 안 쓴 페이지)                      │ │
│  │  1: R=0, D=1  (차선 - Dirty지만 안 쓴 페이지)                          │ │
│  │  2: R=1, D=0  (쓴 Clean 페이지)                                       │ │
│  │  3: R=1, D=1  (쓴 Dirty 페이지 - 최후)                                 │ │
│  │                                                               │ │
│  │  교체 순서: 0 → 1 → 2 → 3                                             │ │
│  │  → Clean 페이지를 Dirty보다 먼저 교체                                   │ │
│  │  → 디스크 쓰기 최소화                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【GClock, WSClock 등 변형】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  GClock: 카운터 사용, 더 정교한 LRU 근사                               │ │
│  │  WSClock: 워킹셋 + Clock 결합                                         │ │
│  │  Aging Clock: Aging 레지스터 + Clock                                 │ │
│  │                                                               │ │
│  │  → 실제 OS는 복잡한 변형 사용                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### Clock 알고리즘 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                Clock 알고리즘 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Clock 구현 의사코드】                                                          │)
│  ──────────────────                                                  │
│  struct Page {                                                       │
│      int frame;                                                      │
│      int ref_bit;    // 참조 비트                                      │
│  } pages[N];                                                         │
│  int hand = 0;       // 핸드 포인터                                    │
│                                                                     │
│  page_replace():                                                     │
│      while True:                                                     │
│          if pages[hand].ref_bit == 0:                                │
│              victim = pages[hand].frame                              │
│              pages[hand].frame = new_page                            │
│              pages[hand].ref_bit = 1                                 │
│              hand = (hand + 1) % N                                   │
│              return victim                                           │
│          else:                                                       │
│              pages[hand].ref_bit = 0  // Second Chance               │
│              hand = (hand + 1) % N                                   │
│                                                                     │
│  on_page_access(page):                                               │
│      page.ref_bit = 1  // 참조 비트 설정                              │
│                                                                     │
│  【Linux Clock 구현】                                                            │)
│  ──────────────────                                                  │
│  # Linux는 Active/Inactive List 사용                                 │
│  $ cat /proc/zoneinfo | grep -A5 "pages"                             │
│  pages free     1234567                                              │
│  pages inactive 234567                                               │
│  pages active   345678                                               │
│                                                                     │
│  # pgscan: Clock 스캔 횟수                                            │
│  $ cat /proc/vmstat | grep pgscan                                    │
│  pgscan_kswapd_pages 123456                                          │
│                                                                     │
│  【Clock 최적화 팁】                                                               │)
│  ──────────────────                                                  │
│  1. Enhanced Clock: R+D 비트로 Dirty 회피                             │
│  2. 핸드 속도: 너무 느리면 교체 느림, 너무 빠르면 R 비트 무효화             │
│  3. Active/Inactive: Linux 방식으로 정교화                            │
│  4. 스캔 저항: 큰 파일 순차 읽기 시 페이지 회피                          │
│  5. 워킹셋 고려: 활성 페이지 유지                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• Clock: FIFO + 참조 비트로 LRU 근사
• 구조: 원형 리스트 + Hand 포인터
• 동작: R=1 → R=0, 다음 / R=0 → 교체
• Second Chance: R=1인 페이지에 한 번 더 기회
• 장점: 구현 쉬움, 성능 좋음, Belady's Anomaly 없음
• 단점: 정확한 LRU보다는 성능 떨어짐
• Enhanced Clock: R + D 비트 사용
• 교체 우선순위: (R=0,D=0) > (R=0,D=1) > (R=1,D=0) > (R=1,D=1)
• 실제 OS: Linux, Windows 등 많이 사용
• 성능: OPT > LRU ≥ Clock > FIFO
```
