# 298. 페이지 폴트 (Page Fault)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 접근한 페이지가 메모리에 없을 때 발생하는 예외
> 2. **가치**: 요구 페이징, COW, 보호 구현의 핵심
> 3. **융합**: MMU, 스와핑, 페이지 교체, OS 핸들러

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

페이지 폴트(Page Fault)는 **CPU가 접근하려는 페이지가 물리 메모리에 없을 때 발생하는 예외(Exception)**다. MMU가 감지하고 OS가 처리한다. 요구 페이징, COW, 메모리 보호의 핵심 메커니즘이다.

### 💡 비유: 도서관에서 없는 책

페이지 폴트는 **도서관에서 없는 책**과 같다. 손님이 "과학-물리-001"을 달라고 했다. 사서가 책장을 봤는데 없다! 창고(디스크)에 있거나 아예 없다. 손님에게 "잠시만요" 하고 창고에서 가져온다.

### 페이지 폴트 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 폴트 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 폴트 발생 상황】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 유효하지 않은 페이지 접근 (Valid bit = 0):                         │ │
│  │     • 페이지가 메모리에 없음 (디스크에 있음)                             │ │
│  │     • 요구 페이징: 아직 로드 안 됨                                      │ │
│  │                                                               │ │
│  │  2. 권한 위반:                                                │ │
│  │     • R/O 페이지에 쓰기 시도                                           │ │
│  │     • 실행 권한 없는 페이지 실행                                        │ │
│  │     • COW 트리거                                                    │ │
│  │                                                               │ │
│  │  3. 존재하지 않는 주소:                                                │ │
│  │     • 잘못된 주소 접근                                                │ │
│  │     • NULL 포인터 역참조                                              │ │
│  │     • Segmentation Fault                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 폴트 처리 과정】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. CPU가 가상 주소 접근                                              │ │
│  │     MMU가 페이지 테이블 조회                                           │ │
│  │                                                               │ │
│  │  2. Page Fault 발생                                                  │ │
│  │     PTE의 Valid/Present bit = 0 또는 권한 위반                        │ │
│  │                                                               │ │
│  │  3. CPU 예외 발생                                                     │ │
│  │     현재 상태 저장, 커널 모드 전환                                       │ │
│  │                                                               │ │
│  │  4. OS Page Fault 핸들러 실행                                         │ │
│  │     a) 폴트 원인 분석                                                  │ │
│  │     b) 잘못된 주소? → SIGSEGV (프로세스 종료)                           │ │
│  │     c) 디스크에 있음? → 프레임 할당, 디스크에서 로드                        │ │
│  │     d) COW? → 페이지 복사                                             │ │
│  │     e) PTE 갱신                                                      │ │
│  │                                                               │ │
│  │  5. 재실행                                                           │ │
│  │     폴트 발생한 명령어부터 다시 실행                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 폴트 종류】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  종류              원인                    처리                     │ │
│  │  ────              ────                    ────                     │ │
│  │  Hard Fault        디스크에서 로드 필요        디스크 I/O (느림)         │ │
│  │  Soft Fault        메모리에서 해결 가능        페이지 교체 (빠름)         │ │
│  │  Invalid Fault     잘못된 주소              프로세스 종료              │ │
│  │  Protection Fault  권한 위반                COW 또는 종료             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 페이지 폴트 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 폴트 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Hard Fault vs Soft Fault】                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Hard Fault (Major Page Fault):                                     │ │
│  │  • 페이지가 디스크(스왑, 파일)에 있음                                   │ │
│  │  • 디스크 I/O 필요                                                   │ │
│  │  • 처리 시간: 1~10 ms (느림)                                          │ │
│  │  • 성능에 큰 영향                                                    │ │
│  │                                                               │ │
│  │  Soft Fault (Minor Page Fault):                                     │ │
│  │  • 페이지가 메모리에 있지만 매핑 안 됨                                   │ │
│  │  • 디스크 I/O 불필요                                                  │ │
│  │  • 처리 시간: 수 μs (빠름)                                            │ │
│  │  • 페이지 교체만 필요                                                 │ │
│  │                                                               │ │
│  │  예시:                                                               │ │
│  │  • Hard: 스와핑, 파일 mmap 첫 접근                                     │ │
│  │  • Soft: Copy-on-Write, 페이지 재사용                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 폴트 비용】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  작업                    시간                                       │ │
│  │  ────                    ────                                       │ │
│  │  TLB Hit               1~2 ns                                      │ │
│  │  TLB Miss + Page Table   10~100 ns                                 │ │
│  │  Soft Fault             1~10 μs                                    │ │
│  │  Hard Fault (SSD)       100 μs ~ 1 ms                              │ │
│  │  Hard Fault (HDD)       1~10 ms                                    │ │
│  │                                                               │ │
│  │  → Hard Fault는 10,000~1,000,000배 느림                              │ │
│  │  → Hard Fault 최소화가 성능 핵심                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 폴트와 성능】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  폴트율        성능 영향                                               │ │
│  │  ────          ────                                                  │ │
│  │  0.01%         무시 가능                                               │ │
│  │  0.1%          약간 느려짐                                              │ │
│  │  1%            눈에 띄게 느려짐                                          │ │
│  │  10%           매우 느려짐                                              │ │
│  │  50%+          스래싱 (시스템 마비)                                      │ │
│  │                                                               │ │
│  │  → Hard Fault 1% = 성능 10% 이상 저하                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 페이지 폴트: 접근한 페이지가 메모리에 없음
• 발생: Valid=0, 권한 위반, 잘못된 주소
• 처리: OS 핸들러가 디스크 로드, COW, 또는 종료
• Hard Fault: 디스크 I/O 필요 (느림, 1~10ms)
• Soft Fault: 메모리에서만 처리 (빠름, 수μs)
• 성능: Hard Fault 1% = 성능 10%+ 저하
• 요구 페이징: 폴트 시 페이지 로드
• COW: 쓰기 폴트 시 복사
• 측정: maj_flt, min_flt, perf, vmstat
• 최적화: 메모리 충분, 지역성, Huge Page
```
