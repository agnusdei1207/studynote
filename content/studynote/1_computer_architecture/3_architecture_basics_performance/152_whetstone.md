# Whetstone

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Whetstone은 $1972$년 해롤드 커노(Harold Curnow)가 개발한 **부동소수점 연산 중심의 합성 벤치마크 (Synthetic Benchmark)**로, 주로 과학 기술 계산에서의 수치 처리 성능을 측정하기 위해 설계되었다.
> 2. **가치**: 연산 능력을 **MWIPS (Millions of Whetstone Instructions Per Second)** 단위로 산출하며, 단순 사칙연산을 넘어 초월함수(Trigonometric), 배열 접근, 조건부 분기 등 복잡한 실수 연산 패턴을 종합적으로 평가한다.
> 3. **융합**: 정수 연산 중심의 Dhrystone과 상보적 관계를 형성하며, 현대의 FLOPS 지표가 보편화되기 전까지 워크스테이션과 미니컴퓨터의 수치 해석 능력을 검증하는 전 세계적 표준으로 군림했다.

+++

## Ⅰ. Whetstone의 개요

### 1. 정의
- 과학적 수치 계산 알고리즘에서 흔히 발생하는 명령어들의 혼합 비율을 통계적으로 구현하여, 시스템의 부동소수점 유닛(FPU) 성능을 측정하는 표준 프로그램이다.

### 2. 등장 배경: '계산 성능의 객관적 증명'
- 1970년대 초, 영국 웻스톤(Whetstone) 지역의 알골(ALGOL) 컴파일러 성능을 개선하기 위해 시작되었다. 당시 컴퓨터들은 기종마다 실수 처리 방식이 달랐기에, "어떤 컴퓨터가 기상 예측이나 물리 시뮬레이션에 더 적합한가?"를 판단할 공통된 시험 문제가 필요했고, 그 결과물이 바로 Whetstone이다.

### 3. 💡 비유: '공학용 계산기의 성능 테스트'
- Whetstone은 **'단순한 덧셈 뺄셈이 아니라, 사인(sin), 코사인(cos), 로그(log) 같은 복잡한 공식이 가득한 수학 시험지'**와 같다.
- 일반인이 아닌 '수학자(과학자)'가 쓰는 전용 도구의 성능을 재는 것이기에, 문제 하나하나가 매우 까다롭고 정밀한 계산력을 요구한다.

## Ⅱ. Whetstone의 메커니즘 및 지표 (Deep Dive)

### 1. MWIPS (Millions of Whetstone Instructions Per Second)
- Whetstone 루프를 1초 동안 얼마나 많이 실행했는지를 백만 단위로 나타낸다.
- **인사이트**: 주파수가 높더라도 부동소수점 가속기가 부실하거나 함수 라이브러리가 최적화되지 않으면 MWIPS 수치는 낮게 나온다.

### 2. 테스트 구성 요소 (Mix of Instructions)
Whetstone은 실제 과학 프로그램의 통계 데이터를 기반으로 다음 8가지 모듈을 가중치 있게 실행한다.
| 모듈 종류 | 측정 내용 | 비중 및 특징 |
|:---|:---|:---|
| **Simple Identifiers** | 단순 변수 참조 및 할당 | 기본적인 데이터 흐름 체크 |
| **Array Elements** | 다차원 배열 접근 연산 | 메모리 주소 계산 능력 테스트 |
| **Array as Parameter** | 함수 인자로 배열 전달 | 스택 및 프로시저 오버헤드 측정 |
| **Conditional Jumps** | 분기문 처리 | 파이프라인 효율 확인 |
| **Integer Arithmetic** | 정수 연산 보조 | 부동소수점과의 혼합 성능 |
| **Floating Point** | **핵심 실수 연산 (+, -, *, /)** | FPU의 핵심 처리 능력 |
| **Trig Functions** | **초월함수 (sin, cos, exp, log)** | 복잡한 수치 해석 라이브러리 효율 |

### 3. 연산 흐름 시각화 (ASCII)
```text
    [ Whetstone Benchmark Flow ]
    
    1. Initialize Large Arrays
    2. Loop Start (N iterations)
       ┌───────────────────────────────┐
       │ - Real Arithmetic Operations  │ (FPU Active)
       │ - Logical & Branch Tests      │
       │ - Math Library Calls (sin/cos)│ (Complex Ops)
       │ - Array Indexing              │ (Mem Access)
       └───────────────────────────────┘
    3. Loop End
    4. Calculate Time & MWIPS Score
```

- **📢 섹션 요약 비유**: 수천 번의 정교한 수학 올림피아드 문제를 시간 제한 안에 풀게 하여, 수학적 근력(FPU)과 문제 풀이 요령(컴파일러)을 동시에 재는 방식입니다.

## Ⅲ. Whetstone vs Dhrystone 기술 비교

| 비교 항목 | Whetstone | Dhrystone |
|:---|:---|:---|
| **주요 대상** | **부동소수점 (Real Number)** | 정수 (Integer) |
| **핵심 유닛** | **FPU (Floating Point Unit)** | ALU, Control Unit |
| **단위** | **MWIPS** | DMIPS |
| **주요 용도** | 과학, 공학, 그래픽 연산 | 시스템 SW, 일반 사무, 제어 |
| **특징** | 라이브러리 함수(sin 등) 의존도 높음 | 문자열 및 포인터 연산 위주 |

- **📢 섹션 요약 비유**: Whetstone이 '설계 사무소'용 성능 테스트라면, Dhrystone은 '행정 사무소'용 성능 테스트입니다.

## Ⅳ. 실무 적용 및 아키텍처 판단 (Strategy)

### 1. 실무 시나리오: 수치 제어 보드(CNC)의 MCU 선정
- **상황**: 정밀한 곡선 가공을 위해 삼각함수 연산을 실시간으로 수행해야 하는 제어 보드 설계.
- **판단**: 단순히 클럭 주파수나 Dhrystone 점수만 보고 코어를 선택하면 실시간 응답에 실패할 수 있다. 기술사는 반드시 **Whetstone(또는 MWIPS/MHz)** 수치를 확인하여, 하드웨어 FPU가 초월함수 연산을 얼마나 효율적으로 가속하는지 검증해야 한다. FPU가 없거나 약한 코어는 소프트웨어 에뮬레이션으로 인해 수천 배 더 느려질 수 있기 때문이다.

### 2. 기술사적 인사이트: 현대적 대안과 한계
- **인사이트**: Whetstone은 현대의 대규모 캐시와 비순차 실행(OoO) 엔진을 충분히 괴롭히지 못한다. 코드 크기가 너무 작아 캐시에 상주하기 때문이다. 기술사는 수십 년 된 Whetstone 수치에만 의존하지 말고, 병렬 연산 성능을 재는 **Linpack**이나 실제 앱 기반의 **SPECfp** 결과를 교차 확인하여 고성능 연산 시스템의 진정한 한계를 가늠해야 한다.

- **📢 섹션 요약 비유**: 기초 체력장 점수(Whetstone)도 중요하지만, 올림픽 본선(SPECfp)에서 어떤 성적을 내는지가 진정한 프로의 기준입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **과학적 근거 확보**: 수치 해석 시뮬레이션의 예상 완료 시간을 정량적으로 예측.
- **아키텍처 검증**: 다양한 하드웨어 아키텍처의 실수 처리 효율성을 동일 선상에서 비교.

### 2. 미래 전망
비록 수십 년 된 벤치마크이지만, Whetstone의 연산 믹스 철학은 현대의 **AI 벤치마크**에서도 계승되고 있다. 딥러닝에서 쓰이는 활성화 함수(Sigmoid, Tanh) 역시 복잡한 지수 연산을 포함하며, 이를 하드웨어가 얼마나 빨리 처리하느냐가 성능의 핵심이기 때문이다. Whetstone은 디지털 데이터가 '실제 세계의 연속성'을 계산하는 한, 그 상징적인 가치를 유지할 것이다.

- **📢 섹션 요약 비유**: 세월이 흘러도 변하지 않는 '정석 수학 문제집'처럼, Whetstone은 모든 계산 성능의 영원한 기초가 될 것입니다.

### 📌 관련 개념 맵 (Knowledge Graph)
- **Dhrystone**: Whetstone과 짝을 이루는 정수 연산 벤치마크.
- **Linpack**: Whetstone을 계승한 대규모 부동소수점 병렬 연산 표준.
- **FPU (Floating Point Unit)**: Whetstone 성능을 결정짓는 하드웨어 연산 장치.
- **FLOPS**: Whetstone의 MWIPS를 대체한 현대적 성능 단위.
