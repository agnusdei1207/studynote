# 152. Whetstone

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 부동소수점 연산 성능을 측정하는 고전 벤치마크 (1976)
> 2. **가치**: FP 성능 비교, 수치 연산 중심 워크로드 성능 평가
> 3. **융합**: SPEC FP 벤치마크의 기반, 레거시지만 교육적 가치 유지

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

Whetstone은 1976년 Harold Curnow가 개발한 벤치마크로, **부동소수점(Floating-Point) 연산 성능**을 측정하기 위해 설계되었다. Algol 60으로 작성되었으며, 나중에 Fortran으로 재작성되었다.

### 💡 비유: 부동소수점 시험

Whetstone은 **수학 시험(부동소수점)**과 같다. 소수점 계산(삼각함수, 로그 등)을 얼마나 빨리 하는지 측정한다. 정수 계산(덧셈, 뺄셈)이 중심인 Dhrystone과 대비된다.

```
│  • 삼각함수 (SIN, COS, ATAN, EXP)                                   │
│  • 배열 연산                                                        │
│  • 조건문, IF 문                                                    │
│  • 서브루틴 호출                                                     │
│                                                                     │
│  【연산 비율 예시 (원본 기준)】                                      │
│  ─────────────────────────────                                      │
│  연산 유형       비율     설명                                      │
│  ─────────────  ─────  ─────────────────────────                  │
│  ADD, SUB       31%    기본 산술/논리                              │
│  MUL, DIV       35%    곱셈, 나눗셈                                  │
│  SIN, COS, ATAN 25%    삼각함수 (수학/그래픽스)                    │
│  SQRT            5%    제곱근 (비용 높음)                          │
│  IF, GOTO        4%     제어 흐름 (파이프라인 효과)                 │
│                                                                     │
│  【장점】                                                           │
│  ────────                                                           │
│  • 부동소수점 연산에 특화                                           │
│  • FP 유닛 성능 측정에 적합                                         │
│  • 과학/공학 계산 워크로드 반영                                     │
│  • 컴파일러 최적화 효과 검증                                         │
│                                                                     │
│  【단점】                                                           │
│  ────────                                                           │
│  • 정수 연산 성능 미반영                                            │
│  • 메모리 계층 고려 없음                                            │
│  • 현대 SIMD 미반영                                                 │
│  • 작은 코드로 단순                                                  │
└─────────────────────────────────────────────────────────────────────┘
```

### MWIPS (Million Whetstones Per Second)

```
┌─────────────────────────────────────────────────────────────────────┐
│                MWIPS 계산                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【공식】                                                            │
│  ────────                                                            │
│              Whetstone 연산 횟수                                     │
│  MWIPS = ─────────────────────────────────                           │
│         실행 시간(초) × 10⁶                                        │
│                                                                     │
│  【예시 계산】                                                       │
│  ──────────────                                                     │
│  • 100,000,000회 Whetstone 연산                                     │
│  • 실행 시간: 0.1초                                                 │
│  • MWIPS = 100,000,000 / (0.1 × 10⁶) = 1,000 MWIPS                 │
│                                                                     │
│  【다른 척도】                                                       │
│  ──────────────                                                     │
│  MFLOPS (Million FLOPS): 부동소수점 연산                            │
│  MIPS (Million IPS): 정수 연산                                      │
│                                                                     │
│  【MWIPS 의미】                                                      │
│  ──────────────                                                     │
│  • FPU 성능 간접 지표                                               │
│  • 벡엔더/그래픽스/스테이션 시스템 성능 지표                           │
│  • 과학 계산 중심 워크로드 평가                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 연산 비율 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                Whetstone 연산 비율 상세                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【연산 유형별 분석】                                               │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  1. 기본 산술 연산 (31%)                                            │
│     • ADD, SUB: 정수/부동소수점                                       │
│     • 정수 연산은 주로 레지스터 기반                               │
│     • 파이프라인 효과 높음                                            │
│                                                                     │
│  2. 복잡한 산술 연산 (6%)                                            │
│     • MUL, DIV, MOD: 곱셈, 나눗셈, 나머지                               │
│     • 나눗셈은 가장 비용이 높은 연산                                │
│     • 일반적으로 파이프라인 또는 별도 유닛 사용                           │
│                                                                     │
│  3. 삼각함수 (10%)                                                   │
│     • SIN, COS, ATAN, EXP                                            │
│     • CORDIC 알고리즘 또는 테이블 룩업 사용                             │
│     • 근사화 정확도에 따라 연산량 차이                                │
│                                                                     │
│  4. 기타 연산 (53%)                                                │
│     • 배열 접근, 조건문, 서브루틴 등                                 │
│     • 간접적으로 성능에 영향                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  성능 영향:                                                    │  │
│  │  ──────────────                                                 │  │
│  │  • FPU 유무: 큰 차이                                             │  │
│  │    - FPU: 초당 수십만 연산 가능                                │  │
│  │    - No FPU: 소프트웨어 에뮬레이션 (매우 느림)                             │  │
│  │  • SIMD: 벡터화로 성능 향상 가능                                │  │
│  │  • 파이프라인 깊이: 현대 CPU는 깊은 파이프라인                              │  │
│  │    → FP 연산 효율 향상                                            │  │
│  └───────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Dhrystone vs Whetstone

| 구분 | Dhrystone | Whetstone |
|:---|:---|:---:|
| **대상** | 정수 연산 | 부동소수점 연산 |
| **출처** | Reinhold Weiler (1984) | Harold Curnow (1972) |
| **년도** | 1984 | 1972 |
| **연산** | 논리, 정수, 부동소수점 | 부동소수점 중심 |
| **SIMD** | 제한적 | 제한적 |
| **메모리** | 일부 | 없음 |
| **현황** | 레거시 | 레거시 |
| **용도** | 정수 성능 | FP 성능 |

### 과목 융합

```
┌─────────────────────────────────────────────────────────────────────┐
│                Dhrystone/Whetstone × 타 과목 융합                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터구조】                                                         │

│  • Whetstone: 부동소수점 처리 능력 평가                                │
│  • 두 벤치마크 결과 비교해 성능 최적화 검증                              │

│                                                                     │
│  【네트워크】                                                         │
│  • 패킷 처리: FP 연산(체크섬, CRC) 필요                               │
│  • 라우팅: 삼각함수 기반 경로 계산                                │

│  【데이터베이스】                                                      │

│  • 인덱싱: 해시 함수(FP) vs 비교(정수)                               │
│                                                                     │
│  【임베디드 시스템】                                                   │
│  • DSP: 디지털 신호 처리 (FP 연산 중심)                              │
│  • 그래픽: 3D 렌더링 (삼각함수)                                        │
│  • 제어: 센서 데이터 처리 (정수 연산 중심)                               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 임베디드 DSP 선택**
- 요구: 실시간 오디오 처리 (48kHz 샘플링)
- 평가: Whetstone으로 FP 성능 확인
- 선택: 높은 MWIPS DSP

**시나리오 2: 컴파일러 최적화 검증**
- 전: 최적화 전/ 후 성능 비교
- 지표: MWIPS
- 목표: 10% 이상 향상

**시나리오 3: 과학 계산 시스템**
- 요구: 높은 정확도 FP 연산
- 평가: Whetstone + 정밀도 분석
- 선택: FPU + 높은 정밀도 시스템

**시나리오 4: 마이크로컨트롤러 선정**
- 요구: 저전력, 적당한 FP 성능
- 평가: MWIPS/Watt
- 선택: 높은 효율 MCU (ARM Cortex-M4F)
