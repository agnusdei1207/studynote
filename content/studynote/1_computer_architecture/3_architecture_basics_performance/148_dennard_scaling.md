# 148. 데나드 스케일링 (Dennard Scaling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트랜지스터 축소 시 전력 밀도가 일정하게 유지된다는 법칙 (1974)
> 2. **가치**: 30년간 CPU 클럭 증가와 전력 증가 분리 가능했던 원리
> 3. **융합**: 2006년 붕괴, Dark Silicon 문제 대두, 멀티코어 시대 개막

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

데나드 스케일링(Dennard Scaling)은 1974년 IBM의 로버트 데나드(Robert Dennard)가 발표한 법칙으로, **트랜지스터의 크기를 줄이면 전압과 전류도 비례하여 감소하여 전력 밀도가 일정하게 유지된다**는 원리다.

### 💡 비유: 작아질수록 효율적인 전구

전구 크기를 반으로 줄이면, 전력도 반으로 줄어 같은 밝기(단위 면적당)를 유지한다. 그래서 같은 면적에 전구를 4개 넣어도 전력은 2배만 증가한다.

### 기본 공식

```
┌─────────────────────────────────────────────────────────────────────┐
│                데나드 스케일링                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 원리】                                                      │
│  ──────────────                                                     │
│                                                                     │
│  트랜지스터 크기: K로 축소                                          │
│  ────────────────────────                                           │
│  • 전압(V): K배 감소                                                │
│  • 전류(I): K배 감소                                                │
│  • 면적(A): K²배 감소                                               │
│  • 전력(P=V×I): K²배 감소                                           │
│  • 전력 밀도(P/A): 일정 유지!                                       │
│  • 지연(T): K배 감소 → 클럭 1/K배 증가                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시: 트랜지스터 크기를 1/2로 축소                            │ │
│  │  ────────────────────────────────────                          │ │
│  │  • 전압: 1/2                                                    │ │
│  │  • 전류: 1/2                                                    │ │
│  │  • 면적: 1/4                                                    │ │
│  │  • 전력: 1/4                                                    │ │
│  │  • 전력 밀도: (1/4)/(1/4) = 1 (일정)                           │ │
│  │  • 클럭: 2배 증가                                               │ │
│  │                                                               │ │
│  │  → 같은 칩 면적에 4× 트랜지스터,                               │ │
│  │    전력 밀도는 동일, 클럭은 2×                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【결과】                                                           │
│  ────────                                                           │
│  • 더 많은 트랜지스터 (무어의 법칙)                                 │
│  • 더 높은 클럭                                                     │
│  • 동일한 전력 밀도                                                 │
│  → "공짜 점심" (Free Lunch)                                        │
└─────────────────────────────────────────────────────────────────────┘
```

│  │  • "클럭만 높이면 된다"는 믿음                                  │ │

### 데나드 스케일링 붕괴 (2006~)

```
┌─────────────────────────────────────────────────────────────────────┐
│                데나드 스케일링 붕괴                                 │
├─────────────────────────────────────────────────────────────────────┤

│  【붕괴 원인】                                                      │
│  ──────────────                                                     │
│  1. 전압 하한                                                       │

│  3. 전력 밀도                                                       │
│     • 클럭↑ → 전력↑ (비례)                                          │
│     • 전력 밀도 급증                                                 │
│     • 냉각 한계 도달                                                 │
│                                                                     │
│  【붕괴 후 CPU 발전】                                               │
│  ──────────────────────                                             │

│  ─────   ───────────────  ─────────  ─────   ─────                 │
│  2006    Core 2 Duo       2.4 GHz      2      65W                   │
│  2008    Core i7          3.2 GHz      4      95W                   │
│  2012    Core i7          3.5 GHz      4      77W                   │
│  2017    Ryzen 7          3.6 GHz      8      95W                   │
│  2023    Core i9          6.0 GHz     24     253W                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  관찰:                                                         │ │
│  │  ────────                                                      │ │
│  │  • 클럭: 2004(3.8GHz) → 2023(6GHz) = 1.6×만 증가               │ │
│  │  • 코어: 1 → 24 = 24× 증가                                     │ │
│  │  • 전력: 115W → 253W = 2.2× 증가                               │ │
│  │                                                               │ │
│  │  → 클럭 증가 둔화, 코어 증가로 전환                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### Dark Silicon (어두운 실리콘)

```
┌─────────────────────────────────────────────────────────────────────┐
│                Dark Silicon 문제                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정의】                                                           │
│  ────────                                                           │
│  전력/열 예산 때문에 동시에 켤 수 없는 칩 영역                      │
│                                                                     │
│  【원인】                                                           │
│  ────────                                                           │
│  • 무어의 법칙: 트랜지스터 수 증가 지속                             │
│  • 데나드 붕괴: 전력 밀도 증가                                      │
│  • 냉각 한계: TDP 제한                                              │
│  → 모든 트랜지스터 동시 활용 불가                                   │
│                                                                     │
│  【예시】                                                           │
│  ──────────────                                                     │
│  • 칩: 100W 냉각 가능                                               │
│  • 총 코어: 16개                                                    │
│  • 코어당 최대: 15W                                                 │
│  • 동시 활용: 100/15 = 6.7개                                        │
│  • Dark Silicon: 16 - 6 = 10개 (62%)                               │
│                                                                     │
│  【해결 전략】                                                      │
│  ──────────────                                                     │
│  1. 이종 코어 (Heterogeneous)                                       │
│     • Big + Little 코어                                             │
│     • 특화 가속기                                                   │
│                                                                     │
│  2. 전력 관리                                                       │
│     • DVFS (동적 전압/주파수)                                       │
│     • 파도메인                                                      │
│                                                                     │
│  3. 특수 목적 가속기                                                │
│     • NPU, GPU, VPU                                                 │
│     • 필요시만 활성화                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Apple M 시리즈 예시:                                          │ │
│  │  ────────────────────────                                      │ │
│  │  • 성능 코어 (P): 고성능, 고효율                               │ │
│  │  • 효율 코어 (E): 저성능, 초저전력                             │ │
│  │  • GPU, NPU: 필요시만 활성                                     │ │
│  │  → Dark Silicon 활용 극대화                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 서버 CPU 선택**
- 2004식: 높은 클럭 단일 코어
- 2024식: 낮은 클럭 다중 코어
- 이유: 데나드 붕괴로 클럭 한계

**시나리오 2: 모바일 AP 설계**
- 요구: 높은 성능, 낮은 전력
- 해결: big.LITTLE 구조
- Dark Silicon 활용

**시나리오 3: 데이터센터 전력**
- 문제: CPU당 전력 증가
- 해결: 저전력 ARM 서버
- 효율: FLOPS/W 최적화

### 전력 공식

```
┌─────────────────────────────────────────────────────────────────────┐
│                전력 공식                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동적 전력】                                                      │
│  ──────────────                                                     │
│  P_dynamic = α × C × V² × f                                         │
│                                                                     │
│  • α: 활동률                                                        │
│  • C: 정전 용량                                                     │
│  • V: 전압                                                          │
│  • f: 주파수                                                        │
│                                                                     │
│  【정적 전력】                                                      │
│  ──────────────                                                     │
│  P_static = I_leak × V                                              │
│                                                                     │
│  【데나드 스케일링 시절】                                           │
│  ────────────────────────                                           │
│  f↑ (K배) → V↓ (K배) → P_dynamic 일정                              │
│                                                                     │
│  【데나드 붕괴 후】                                                 │
│  ──────────────────                                                 │
│  f↑ → V 일정 → P_dynamic↑ (f² 비례)                                │
│  V 일정 → P_static 증가                                             │
│  → 전력 밀도 급증                                                   │
└─────────────────────────────────────────────────────────────────────┘
```
