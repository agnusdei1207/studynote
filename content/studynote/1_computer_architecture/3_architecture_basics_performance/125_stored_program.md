# 125. 프로그램 내장 방식 (Stored Program Concept)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로그램(명령어)을 하드와이어가 아닌 메모리에 저장하여, 메모리 내용만 바꾸면 다른 작업을 수행할 수 있는 혁신적 개념
> 2. **가치**: 소프트웨어 산업의 탄생을 가능케 한 근본적 패러다임으로, 하드웨어 변경 없이 무한한 기능 확장
> 3. **융합**: 운영체제, 컴파일러, 가상화, 클라우드 컴퓨팅의 이론적 기반

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

프로그램 내장 방식(Stored Program Concept)은 컴퓨터 프로그램을 메모리에 저장해두고, CPU가 메모리에서 명령어를 읽어 실행하는 방식이다. 이전의 하드와이어(hardwired) 방식과 달리, 프로그램을 교체하면 동일한 하드웨어로 완전히 다른 작업을 수행할 수 있다.

### 💡 비유: 교체 가능한 게임 카트리지

프로그램 내장 방식은 **게임기에 카트리지를 꽂는 것**과 같다. 게임기(하드웨어)는 그대로인데, 카트리지(프로그램)만 바꾸면 슈팅 게임, 퍼즐 게임, RPG 등 완전히 다른 게임을 할 수 있다. 프로그램이 '저장'되어 있어서 언제든 바꿀 수 있다.

### 등장 배경

| 시기 | 방식 | 한계 |
|:---:|:---|:---|
| 1930s | 하드와이어 | 회로를 다시 연결해야 함 |
| 1940s | 플러그보드 | 교체에 시간 소요 |
| 1945 | 프로그램 내장 | 메모리만 교체 → 혁신! |
| 현재 | 클라우드 | 네트워크로 프로그램 다운로드 |

### 자가 수정 코드 (Self-Modifying Code)

```c
// 프로그램 내장 방식의 특수 활용: 자가 수정 코드
// (현대 시스템에서는 보안상 DEP로 인해 잘 사용 안 함)

#include <stdio.h>
#include <string.h>
#include <sys/mman.h>

// 예시: JIT 컴파일러의 원리
void jit_example() {
    // 메모리에 실행 코드를 동적으로 생성
    unsigned char code[] = {
        0xB8, 0x2A, 0x00, 0x00, 0x00,  // mov eax, 42
        0xC3                            // ret
    };

    // 실행 가능한 메모리 할당
    void *mem = mmap(NULL, 4096, PROT_READ | PROT_WRITE | PROT_EXEC,
                     MAP_PRIVATE | MAP_ANONYMOUS, -1, 0);

    // 코드 복사
    memcpy(mem, code, sizeof(code));

    // 함수 포인터로 실행
    int (*func)() = mem;
    int result = func();

    printf("Result: %d\n", result);  // 42
    munmap(mem, 4096);
}

// 실제 활용:
// • JIT 컴파일러 (V8, JVM, .NET)
// • 동적 코드 생성
// • 바이러니/악성코드 (악용 사례)
```

### 프로그램 내장 방식의 파급 효과

| 영역 | 파급 효과 |
|:---|:---|
| **소프트웨어 산업** | 하드웨어와 분리된 산업 탄생 |
| **운영체제** | 다중 프로그래밍, 멀티태스킹 |

| **가상화** | VM, 컨테이너 기술 |
| **클라우드** | SaaS, FaaS |

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 프로그램 내장 방식은 프로그램을 '저장'해두고 필요할 때 실행하는 거예요!

**원리**: 마치 스마트폰에 게임을 '저장'해두면 언제든 실행할 수 있는 것처럼, 컴퓨터도 프로그램을 메모리에 저장해두고 필요할 때 꺼내 써요.

**효과**: 컴퓨터를 다시 만들지 않아도, 새로운 프로그램만 저장하면 완전히 다른 일을 할 수 있어요! 앱 스토어가 가능한 이유예요!
