# 155. 다크 실리콘 (Dark Silicon)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 칩 내에서 전력 제한으로 사용하지 못하는 트랜지스터 영역, Dennard 스케일링 붕괴로 발생
> 2. **가치**: 전력 밀도 한계 극복, 이기종 멀티코어 설계의 핵심 동인
> 3. **융합**: big.LITTLE, 가속기, 전력 게이팅 기술로 활용 전략 수립

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

다크 실리콘(Dark Silicon)은 집적된 트랜지스터 중 **전력·발열 제한으로 동시에 켤 수 없는 영역**을 의미한다. Dennard 스케일링이 붕괴된 2005년 이후, 트랜지스터 수는 계속 증가하지만 전력 밀도 한계로 모든 코어를 동시에 활성화할 수 없는 현상이다.

### 💡 비유: 만원 버스의 빈 좌석

다크 실리콘은 **만원 버스의 빈 좌석**과 같다. 50명이 탈 수 있는 버스가 있지만, 냉방이 약해서 30명 이상 타면 너무 더워진다. 결국 20석은 비워둬야 한다. 칩도 마찬가지로 모든 트랜지스터를 켜면 과열되어 일부는 꺼둬야 한다.

│  → 칩의 일부분만 활성화 가능 = 다크 실리콘                          │

│  32nm     ~30%                 70%                                  │

│  │  ────────────                                                   │ │

│  │   하지만 Dennard 스케일링 붕괴로                               │ │

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 다크 실리콘 시각화

```
┌─────────────────────────────────────────────────────────────────────┐
│                다크 실리콘 시각화                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전통적 멀티코어 (Dennard 시대)】                                  │
│  ────────────────────────────────────                               │
│  ┌────┬────┬────┬────┬────┬────┬────┬────┐                        │
│  │ C0 │ C1 │ C2 │ C3 │ C4 │ C5 │ C6 │ C7 │  8코어 모두 활성       │
│  ├────┼────┼────┼────┼────┼────┼────┼────┤                        │
│  │ C8 │ C9 │C10 │C11 │C12 │C13 │C14 │C15 │  16코어 모두 활성      │
│  └────┴────┴────┴────┴────┴────┴────┴────┘                        │
│  모든 코어를 최대 클럭으로 동시 실행 가능                           │
│                                                                     │
│  【다크 실리콘 시대】                                                │
│  ────────────────────────                                           │
│  ┌────┬────┬────┬────┬────┬────┬────┬────┐                        │
│  │ C0 │ C1 │░░░░│░░░░│ C4 │ C5 │░░░░│░░░░│                        │
│  ├────┼────┼────┼────┼────┼────┼────┼────┤                        │
│  │░░░░│ C9 │C10 │░░░░│░░░░│C13 │░░░░│C15 │                        │
│  └────┴────┴────┴────┴────┴────┴────┴────┘                        │
│  ░░ = Dark Silicon (비활성)                                        │
│  일부 코어만 활성, 나머지는 전력 게이팅                            │
│                                                                     │
│  【전력 밀도 제약】                                                  │
│  ──────────────────────                                             │
│                                                                     │
│  TDP (열설계전력) = 125W                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  코어당 전력: 15W × 활성 코어 수 ≤ TDP                        │ │
│  │  15W × 8코어 = 120W ✓ (가능)                                   │ │
│  │  15W × 16코어 = 240W ✗ (불가능)                                │ │
│  │                                                               │ │
│  │  → 16코어 중 8코어만 동시 활성 가능                            │ │
│  │  → 나머지 8코어는 다크 실리콘                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결책: 이기종 멀티코어】                                         │
│  ────────────────────────────                                       │
│  ┌──────────┬──────────┬──────────┬──────────┐                     │
│  │   BIG    │   BIG    │  little  │  little  │                     │
│  │  (고성능)│  (고성능)│  (저전력)│  (저전력)│                     │
│  ├──────────┼──────────┼──────────┼──────────┤                     │
│  │  little  │  little  │  little  │  little  │                     │
│  └──────────┴──────────┴──────────┴──────────┘                     │
│                                                                     │
│  • BIG 코어 2개 + little 코어 6개                                   │
│  • 같은 전력으로 더 많은 코어 활성                                  │
│  • 워크로드별 적절한 코어 사용                                       │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  가속기 배치 예시:                                             │ │
│  │  ────────────────                                               │ │

## Ⅲ. 융합 비교 및 다각도 분석

### 다른 전력 관리 기법과 비교

| 구분 | DVFS | Clock Gating | Power Gating | Dark Silicon |
|:---|:---|:---|:---|:---|
| **방식** | 전압/주파수 조절 | 클럭 차단 | 전원 차단 | 공간 비활성 |
| **복구 시간** | μs~ms | ns | μs~ms | N/A |
| **전력 절감** | 10~50% | 20~40% | 90%+ | 100% |
| **면적 오버헤드** | 없음 | 작음 | 중간 | 없음 |
| **적용 단위** | 도메인 | 블록 | 블록 | 코어/IP |

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 모바일 SoC 설계**
- 요구: 고성능 + 배터리 수명
- 접근: 4× big + 4× LITTLE + NPU + GPU
- 결과: 다크 실리콘을 가속기로 활용

**시나리오 2: 서버 칩 설계**
- 요구: 최대 처리량 + 전력 효율
- 접근: 다수의 저전력 코어 + 가속기
- 결과: 전력당 성능 최적화

**시나리오 3: 클라우드 인스턴스 타입 선택**
- 요구: 워크로드별 최적 인스턴스
- 평가: vCPU 타입, 가속기 유무
- 선택: 워크로드 특성에 맞는 인스턴스

**시나리오 4: 애플리케이션 최적화**
- 요구: 전력 효율적 실행
- 접근: E-core 선호 스레드, 가속기 활용
- 결과: 배터리 수명 연장
