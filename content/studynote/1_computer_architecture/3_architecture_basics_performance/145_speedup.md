# 145. 속도 향상도 (Speedup)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 개선 후 성능 / 개선 전 성능, 최적화 효과를 정량화하는 지표
> 2. **가치**: 투자 대비 효과, 의사결정 근거, 성능 개선 평가 기준
> 3. **융합**: 암달의 법칙으로 최대 향상도 예측, 초선형 향상도 가능

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

속도 향상도(Speedup)는 시스템 개선 전후의 성능 비율을 나타내는 무차원 지표다. 개선 전 실행 시간 대비 개선 후 실행 시간이 얼마나 단축되었는지를 보여준다.

### 💡 비유: 주행 시간 단축

서울에서 부산까지 4시간 걸리던 여정이 고속도로 확장 후 2시간으로 단축되었다면, **속도 향상도는 2×**다. "2배 빨라졌다"는 의미다.

### 기본 공식

```
┌─────────────────────────────────────────────────────────────────────┐
│                속도 향상도 공식                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 정의】                                                      │
│  ──────────────                                                     │
│                                                                     │
│              이전 성능 (Old Performance)                            │
│  Speedup = ──────────────────────────────                           │
│             이후 성능 (New Performance)                             │
│                                                                     │
│  【실행 시간 기준】                                                 │
│  ──────────────────                                                 │
│                                                                     │
│              이전 실행 시간 (T_old)                                 │
│  Speedup = ───────────────────────────────                          │
│             이후 실행 시간 (T_new)                                  │
│                                                                     │
│  【처리량 기준】                                                    │
│  ──────────────                                                     │
│                                                                     │
│              이후 처리량 (TP_new)                                   │
│  Speedup = ───────────────────────────────                          │
│             이전 처리량 (TP_old)                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시: 프로그램 실행 시간                                       │ │
│  │  ────────────────────────                                      │ │
│  │  이전: 100초                                                    │ │
│  │  이후: 40초                                                     │ │
│  │  Speedup = 100 / 40 = 2.5×                                     │ │
│  │                                                               │ │
│  │  → 2.5배 빨라짐 (60% 시간 단축)                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해석】                                                           │
│  ────────                                                           │
│  • Speedup > 1: 개선됨                                              │
│  • Speedup = 1: 변화 없음                                           │
│  • Speedup < 1: 악화됨                                              │
│  • Speedup = n: n배 향상                                            │
└─────────────────────────────────────────────────────────────────────┘
```

### 향상도 vs 개선율

```
┌─────────────────────────────────────────────────────────────────────┐
│                향상도 vs 개선율                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【속도 향상도 (Speedup)】                                          │
│  ──────────────────────────                                         │
│  Speedup = T_old / T_new                                            │
│  • 단위: 없음 (배수)                                                │
│  • 예: 2×, 2.5×, 10×                                               │
│                                                                     │
│  【개선율 (Improvement Rate)】                                      │
│  ──────────────────────────                                         │
│  Improvement = (T_old - T_new) / T_old × 100%                       │
│  • 단위: %                                                          │
│  • 예: 50%, 60%, 90%                                               │
│                                                                     │
│  【관계】                                                           │
│  ────────                                                           │
│  Improvement = (1 - 1/Speedup) × 100%                               │
│  Speedup = 1 / (1 - Improvement/100)                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  변환표:                                                       │ │
│  │  ────────                                                      │ │
│  │  Speedup     개선율     시간 단축                              │ │
│  │  ────────    ──────     ──────────                             │ │
│  │  1.25×       20%        20% 감소                               │ │
│  │  1.5×        33%        33% 감소                               │ │
│  │  2×          50%        50% 감소                               │ │
│  │  5×          80%        80% 감소                               │ │
│  │  10×         90%        90% 감소                               │ │
│  │  100×        99%        99% 감소                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 향상도와 암달의 법칙

```
┌─────────────────────────────────────────────────────────────────────┐
│                암달의 법칙과 최대 향상도                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【공식】                                                           │
│  ────────                                                           │
│                                                                     │
│              1                                                      │
│  Speedup = ─────────────────────────                                │
│            (1-f) + f/s                                              │
│                                                                     │
│  • f: 개선되는 부분의 비율                                          │
│  • s: 개선되는 부분의 향상 배수                                     │
│                                                                     │
│  【최대 향상도 (s→∞)】                                             │
│  ────────────────────                                               │
│  Max Speedup = 1 / (1-f)                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시: 프로그램의 80%를 10배 최적화                            │ │
│  │  ───────────────────────────────────                           │ │
│  │  f = 0.8, s = 10                                               │ │
│  │                                                                │ │
│  │  Speedup = 1 / (0.2 + 0.8/10)                                 │ │
│  │          = 1 / (0.2 + 0.08)                                    │ │
│  │          = 1 / 0.28                                            │ │
│  │          = 3.57×                                               │ │
│  │                                                                │ │
│  │  최대 향상도 (s→∞) = 1/0.2 = 5×                               │ │
│  │  → 무한히 최적화해도 5배 이상 불가                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【핵심 통찰】                                                      │
│  ──────────────                                                     │
│  • 나머지 20%가 병목이 됨                                           │
│  • 전체 10배 빠르게 하려면 100% 최적화 필요                         │
│  • 직렬 구간이 최대 향상도 결정                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 향상도의 종류

```
┌─────────────────────────────────────────────────────────────────────┐
│                향상도 종류                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【절대 향상도 (Absolute Speedup)】                                 │
│  ──────────────────────────────────                                 │
│  • 기준: 단일 프로세서                                              │
│  • 공식: T_1 / T_n                                                  │
│  • 의미: n개 프로세서 사용 시 이론적 향상                           │
│                                                                     │
│  【상대 향상도 (Relative Speedup)】                                 │
│  ──────────────────────────────────                                 │
│  • 기준: 이전 버전/시스템                                           │
│  • 공식: T_old / T_new                                              │
│  • 의미: 개선 효과                                                  │
│                                                                     │
│  【선형 향상도 (Linear Speedup)】                                   │
│  ────────────────────────────────                                   │
│  • 공식: Speedup = n (n = 프로세서 수)                              │
│  • 조건: 완벽한 병렬화                                              │
│  • 현실: 달성 어려움                                                │
│                                                                     │
│  【초선형 향상도 (Superlinear Speedup)】                            │
│  ────────────────────────────────────                               │
│  • 공식: Speedup > n                                                │
│  • 원인: 캐시 효과, 메모리 계층                                     │
│  • 예: 4코어에서 5× 향상                                           │
│                                                                     │
│  【효율 (Efficiency)】                                              │
│  ────────────────────                                               │
│  Efficiency = Speedup / n                                           │
│  • 이상적: 1.0 (100%)                                               │
│  • 현실: 0.7~0.9                                                    │
└─────────────────────────────────────────────────────────────────────┘
```

### 초선형 향상도 현상

```
┌─────────────────────────────────────────────────────────────────────┐
│                초선형 향상도 (Superlinear Speedup)                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【현상】                                                           │
│  ────────                                                           │
│  프로세서 수보다 더 큰 향상도 달성                                  │
│  Speedup > n                                                        │
│                                                                     │
│  【원인】                                                           │
│  ────────                                                           │
│  1. 캐시 효과 (Cache Effect)                                        │
│     • 각 코어의 캐시 합 > 단일 코어 캐시                            │
│     • 더 많은 데이터가 캐시에 상주                                  │
│                                                                     │
│  2. 메모리 계층 (Memory Hierarchy)                                  │
│     • 병렬로 더 많은 메모리 대역폭 활용                             │
│     • 각 코어별 독립 메모리 채널                                    │
│                                                                     │
│  3. 작업 분할 (Work Distribution)                                   │
│     • 데이터 분할로 locality 향상                                   │
│     • TLB 효율 증가                                                 │
│                                                                     │
│  【예시】                                                           │
│  ────────                                                           │
│  단일 코어: 4GB 캐시, 100초                                         │
│  4코어: 4×4GB=16GB 캐시                                             │
│  → 캐시 미스 대폭 감소                                              │
│  → 4코어에서 6× 향상 (초선형!)                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  주의:                                                         │ │
│  │  ────────                                                      │ │
│  │  초선형 향상도는 특수한 경우에만 발생                          │ │
│  │  일반적으로는 이상적 선형보다 낮음                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: CPU 업그레이드**
- 이전: 4코어, 100초
- 이후: 8코어, 55초
- 향상도 = 100/55 = 1.82×
- 효율 = 1.82/8 = 22.8% (낮음)

**시나리오 2: 알고리즘 변경**
- 이전: O(n²), n=10000, 10초
- 이후: O(n log n), 0.1초
- 향상도 = 100× (대폭 향상)

**시나리오 3: 캐시 추가**
- 이전: 캐시 없이, 50초
- 이후: L1+L2 캐시, 5초
- 향상도 = 10×

**시나리오 4: SSD 교체**
- 이전: HDD, 30초
- 이후: NVMe SSD, 0.5초
- 향상도 = 60×

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 개선 | 전제 | 향상도 |
|:---|:---|:---:|
| **캐시** | 메모리 bound | 2~10× |
| **SSD** | I/O bound | 10~100× |
| **알고리즘** | 비효율 | 10~1000× |
| **병렬화** | 90% 병렬 | 최대 10× |

### 핵심 공식

```
Speedup = T_old / T_new
Max Speedup (Amdahl) = 1 / (1-f)
Efficiency = Speedup / n
```

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 속도 향상도는 예전보다 얼마나 빨라졌는지 숫자로 나타낸 거예요!

**원리**: 걸어서 1시간 걸리던 길을 자전거로 20분에 갔다면, 3배 빨라진 거예요 (향상도 3×)!

**효과**: 향상도가 높을수록 좋아요. 하지만 아무리 노력해도 한계가 있어요!
