# 139. 암달의 법칙 (Amdahl's Law)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 일부만 개선할 때 전체 성능 향상의 한계를 예측하는 법칙
> 2. **가치**: 병렬화·최적화의 실질적 효과 계산, 투자 대비 효과 분석 도구
> 3. **융합**: 멀티코어·GPU 활용 전략, 직렬 구간이 병목임을 증명

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

암달의 법칙(Amdahl's Law)은 1967년 진 암달(Gene Amdahl)이 발표한 법칙으로, **시스템의 일부분만 개선했을 때 전체 성능 향상의 한계**를 수학적으로 예측한다. 병렬 처리에서 직렬 구간이 전체 성능을 제한함을 보여준다.

### 💡 비유: 고속도로 확장

100km 도로 중 90km만 2차선으로 확장해도, 나머지 10km가 1차선이면 **전체 통행 시간은 10배 빨라질 수 없다**. 병목 구간(1차선)이 전체를 지배한다.

### 기본 공식

```
┌─────────────────────────────────────────────────────────────────────┐
│                암달의 법칙                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【성능 향상 공식】                                                  │
│  ──────────────────                                                 │
│                                                                     │
│              1                                                      │
│  Speedup = ─────────────────────────                                │
│            (1 - f) + f/s                                            │
│                                                                     │
│  • f: 개선되는 부분의 비율 (0 ~ 1)                                  │
│  • s: 개선되는 부분의 향상 배수                                     │
│  • Speedup: 전체 성능 향상 배수                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시: 프로그램의 80%를 10배 빠르게 최적화                     │ │
│  │  ────────────────────────────────────────                      │ │
│  │  f = 0.8, s = 10                                               │ │
│  │                                                                │ │
│  │  Speedup = 1 / ((1-0.8) + 0.8/10)                             │ │
│  │          = 1 / (0.2 + 0.08)                                    │ │
│  │          = 1 / 0.28                                            │ │
│  │          = 3.57×                                               │ │
│  │                                                                │ │
│  │  → 80%를 10배 빠르게 해도 전체는 3.57배만 빨라짐               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 병렬 처리 버전

```
┌─────────────────────────────────────────────────────────────────────┐
│                암달의 법칙 - 병렬 처리                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【병렬화 공식】                                                    │
│  ───────────────                                                    │
│                                                                     │
│              1                                                      │
│  Speedup = ─────────────────────                                    │
│            (1 - p) + p/n                                            │
│                                                                     │
│  • p: 병렬화 가능한 비율 (0 ~ 1)                                    │
│  • n: 프로세서(코어) 수                                             │
│  • (1-p): 직렬 실행 비율                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시: 프로그램의 90%가 병렬화 가능, 코어 100개 사용           │ │
│  │  ────────────────────────────────────────────────              │ │
│  │  p = 0.9, n = 100                                              │ │
│  │                                                                │ │
│  │  Speedup = 1 / ((1-0.9) + 0.9/100)                            │ │
│  │          = 1 / (0.1 + 0.009)                                   │ │
│  │          = 1 / 0.109                                           │ │
│  │          = 9.17×                                               │ │
│  │                                                                │ │
│  │  → 코어를 100개 써도 10배도 못 빨라짐!                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【극한: n → ∞】                                                    │
│  ──────────────                                                     │
│                                                                     │
│  Speedup_max = 1 / (1 - p)                                         │
│                                                                     │
│  → 무한히 많은 코어를 써도 직렬 비율만큼만 향상 가능               │
│  p=0.9이면 최대 10×, p=0.99이면 최대 100×                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 그래프로 보는 암달의 법칙

```
┌─────────────────────────────────────────────────────────────────────┐
│                병렬화 비율별 성능 향상                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Speedup                                                            │
│     ▲                                                               │
│  20 │                              ────── p=95%                     │
│     │                        ──────                                  │
│  15 │                  ────── p=90%                                  │
│     │            ──────                                              │
│  10 │      ────── p=80%                                              │
│     │  ──────                                                        │
│   5 │────── p=50%                                                    │
│     │                                                                │
│   1 ├────────────────────────────────────────────► 코어 수          │
│     0    4    8    16   32   64   128   256   512                  │
│                                                                     │
│  【관찰】                                                           │
│  ────────                                                           │
│  • 코어 수 증가 → 향상율 감소 (수렴)                                │
│  • 직렬 비율(1-p)이 높을수록 빠르게 포화                            │
│  • p=80%면 100코어 써도 5× 향상이 한계                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  최대 향상 (n→∞):                                              │ │
│  │  ──────────────────                                            │ │
│  │  p=50% → 2×, p=90% → 10×, p=95% → 20×, p=99% → 100×           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 암달의 법칙 vs 구스타프손의 법칙

```
┌─────────────────────────────────────────────────────────────────────┐
│                암달 vs 구스타프손                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【암달의 법칙】                                                    │
│  ───────────────                                                    │
│  • 고정된 문제 크기                                                 │
│  • "코어를 늘리면 얼마나 빨라지나?"                                 │
│  • 직렬 비율이 한계                                                 │
│  • 비관적 관점                                                      │
│                                                                     │
│  【구스타프손의 법칙】                                              │
│  ────────────────────────                                           │
│  • 고정된 실행 시간                                                 │
│  • "코어가 늘어나면 더 큰 문제를 풀 수 있나?"                       │
│  • 확장 가능한 문제에서는 선형 스케일링 가능                        │
│  • 낙관적 관점                                                      │
│                                                                     │
│  Speedup_Gustafson = (1 - p) + n × p                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시: p=90%, n=100                                            │ │
│  │  ─────────────────                                             │ │
│  │  암달: 9.17× (같은 문제)                                       │ │
│  │  구스타프손: 90.1× (10배 큰 문제, 같은 시간)                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 실제 사례

```
┌─────────────────────────────────────────────────────────────────────┐
│                암달의 법칙 실제 사례                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【사례 1: 데이터베이스 쿼리 최적화】                               │
│  ────────────────────────────────────                               │
│  • 쿼리 실행 시간: 100초                                            │
│  • 80%가 DB 접근, 20%가 네트워크                                   │
│  • DB를 10배 빠르게 최적화                                          │
│                                                                     │
│  Speedup = 1 / (0.2 + 0.8/10) = 1/0.28 = 3.57×                    │
│  새 시간 = 100/3.57 = 28초                                         │
│                                                                     │
│  → DB을 10배 빠르게 해도 3.57배만 빨라짐                           │
│  → 네트워크(20%)가 새로운 병목                                     │
│                                                                     │
│  【사례 2: 멀티코어 렌더링】                                        │
│  ────────────────────────                                           │
│  • 렌더링 시간: 1000초                                              │
│  • 95% 병렬화 가능, 5% 직렬 (I/O, 메모리)                          │
│  • 32코어 사용                                                      │
│                                                                     │
│  Speedup = 1 / (0.05 + 0.95/32) = 1/0.08 = 12.5×                   │
│  새 시간 = 1000/12.5 = 80초                                        │
│                                                                     │
│  → 32코어로 12.5배 향상 (이상적 32×가 아님)                        │
│  → 최대 향상 = 1/0.05 = 20× (무한 코어도 20×가 한계)               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: CPU 업그레이드 결정**
- 현재: 4코어
- 계획: 16코어
- 분석: 프로그램 85% 병렬화
- 향상: 1/(0.15+0.85/16) = 4.9×
- 결정: 4.9배 향상 vs 4배 비용 → 투자

**시나리오 2: GPU 도입 평가**
- 현재: CPU 16코어
- 계획: GPU 4000코어
- 분석: 프로그램 99% 병렬화
- 향상: 1/(0.01+0.99/4000) = 80× (이론)
- 실제: 메모리 전송 등으로 30~50×

**시나리오 3: 최적화 우선순위**
- A: 80%를 2배 빠르게 → 1.67×
- B: 20%를 10배 빠르게 → 1.22×
- 결정: A 우선 (더 큰 향상)

### 핵심 교훈

1. **직렬 구간이 지배한다**: 10% 직렬 = 최대 10×
2. **코어만 늘린다고 해결 안 됨**: p가 낮으면 한계
3. **알고리즘이 중요하다**: 병렬화 가능하게 설계
4. **현실적 목표 설정**: 이론적 한계 vs 실제
