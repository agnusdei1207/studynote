# 149. 벤치마크 프로그램 (Benchmark)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 성능을 객관적으로 측정·비교하기 위한 표준화된 테스트 프로그램
> 2. **가치**: 하드웨어/소프트웨어 성능 비교, 구매 의사결정, 최적화 검증
> 3. **융합**: CPU·GPU·메모리·스토리지·네트워크 계층별 전용 벤치마크 존재

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

벤치마크(Benchmark)는 컴퓨터 시스템의 성능을 측정하기 위해 설계된 표준화된 테스트 프로그램이다. CPU, 메모리, 스토리지, 네트워크 등 각 계층의 성능을 객관적으로 평가하고 비교하는 데 사용된다.

### 💡 비유: 자동차 연비 테스트

벤치마크는 **자동차 연비 측정**과 같다. 도심, 고속도로, 복합 주행 등 표준화된 코스에서 연비를 측정하여 객관적 비교가 가능하다. "제조사 말"만 믿지 않고 표준 테스트로 확인한다.

### 벤치마크 종류

```
┌─────────────────────────────────────────────────────────────────────┐
│                벤치마크 종류                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【계층별 벤치마크】                                                │
│  ──────────────────                                                 │
│                                                                     │
│  계층        대표 벤치마크          측정 항목                      │
│  ────────   ─────────────────────   ────────────────────            │
│  CPU        SPEC CPU, Geekbench     정수/부동소수점 연산            │
│  GPU        3DMark, GFXBench        그래픽 렌더링, AI              │
│  메모리     STREAM, Memtest         대역폭, 지연                    │
│  스토리지   CrystalDiskMark, AS SSD  읽기/쓰기 속도, IOPS          │
│  네트워크   iperf, netperf          대역폭, 지연                    │
│  종합       PCMark, SYSmark         실제 사용 환경                 │
│                                                                     │
│  【용도별 벤치마크】                                                │
│  ──────────────────                                                 │
│  • 마이크로 벤치마크: 단일 기능 측정 (예: memcpy 속도)              │
│  • 애플리케이션 벤치마크: 실제 프로그램 (예: MySQL 성능)            │
│  • 커널 벤치마크: OS 성능 (예: 파일 시스템)                         │
│  • 종합 벤치마크: 전체 시스템 (예: PCMark)                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### SPEC CPU 벤치마크

```
┌─────────────────────────────────────────────────────────────────────┐
│                SPEC CPU 벤치마크                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【개요】                                                           │
│  ────────                                                           │
│  • SPEC (Standard Performance Evaluation Corporation)               │
│  • CPU 성능 측정의 사실상 표준                                      │
│  • 정수(INTEGER) + 부동소수점(FP)                                   │
│                                                                     │
│  【SPEC CPU 2017 구성】                                             │
│  ────────────────────────                                           │
│                                                                     │
│  구분           벤치마크        워크로드                            │
│  ─────────────  ─────────────   ──────────────────                  │
│  SPECspeed      정수 (INT)      600.perlbench,                      │
│  (단일 스레드)                  603.bwaves,                          │
│                                 607.cactuBSSN...                    │
│                                                                     │
│  SPECrate       정수 (INT)      10개 워크로드                       │
│  (멀티 스레드)                                                      │
│                                                                     │
│  SPECfpspeed    부동소수점     603.bwaves,                          │
│  (단일 스레드)                  607.cactuBSSN...                    │
│                                                                     │
│  SPECfprate    부동소수점      10개 워크로드                        │
│  (멀티 스레드)                                                      │
│                                                                     │
│  【점수 예시 (2024)】                                               │
│  ────────────────────                                               │
│  CPU                     SPECint2017    SPECfp2017                  │
│  ─────────────────────   ───────────    ──────────                  │
│  Intel i9-13900K         84             134                         │
│  AMD Ryzen 9 7950X       77             119                         │
│  Apple M3 Max            ~70            ~90                         │
│  Ampere Altra Max        60             80                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SPEC 점수 해석:                                               │ │
│  │  ──────────────────                                            │ │
│  │  • 기준: SPECref (참조 시스템) = 1.0                           │ │
│  │  • 84 = 기준 시스템 대비 84× 성능                              │ │
│  │  • 높을수록 좋음                                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

| **TPC-C** | DB | 트랜잭션 | DBA |

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 서버 구매 결정**
- 비교: Intel Xeon vs AMD EPYC
- 벤치마크: SPEC CPU 2017
- 결과: AMD가 20% 높음
- 의사결정: AMD 선택

**시나리오 2: GPU 선택 (AI 학습)**
- 비교: NVIDIA A100 vs H100
- 벤치마크: MLPerf
- 결과: H100이 2.5× 빠름
- 비용 분석: H100 선택

**시나리오 3: 스토리지 성능 확인**
- 요구: 100K IOPS 이상
- 벤치마크: fio
- 측정: 4K 랜덤 읽기
- 결과: NVMe SSD 필요

**시나리오 4: 최적화 효과 검증**
- 이전: SPECint = 70
- 최적화 후: SPECint = 84
- 향상: 20%
