# 143. 응답 시간 (Response Time)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 요청부터 시스템 응답 완료까지의 총 시간, 사용자 체감 성능
> 2. **가치**: UX·고객 만족도에 직결, 웹·앱·게임에서 핵심 지표
> 3. **융합**: 처리량·대역폭과 트레이드오프, 지연 시간 + 처리 시간으로 구성

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

응답 시간(Response Time)은 사용자가 요청을 시작한 시점부터 시스템이 완전한 응답을 완료하는 시점까지의 총 경과 시간이다. 지연 시간(Latency) + 처리 시간(Processing Time) + 전송 시간(Transmission Time)의 합이다.

### 💡 비유: 식당 주문부터 식사 완료까지

응답 시간은 **손님이 주문하고 나서 식사를 다 마칠 때까지**의 총 시간이다. 주문 전달(지연) + 요리(처리) + 식사(전송) 모두 포함한다.

### 응답 시간 구성

```
┌─────────────────────────────────────────────────────────────────────┐
│                응답 시간 구성                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Response Time = Latency + Processing Time + Transmission Time      │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  사용자                                                       │   │
│  │    │                                                          │   │
│  │    │ ① 요청 시작                                              │   │
│  │    ▼                                                          │   │
│  │  ┌─────────────────────────────────────────────────────────┐ │   │
│  │  │                 네트워크/시스템                          │ │   │
│  │  │                                                          │ │   │
│  │  │  ←── Latency ──→←── Processing ──→←── Transmission ──→  │ │   │
│  │  │  (요청 전달)      (서버 처리)      (응답 전송)           │ │   │
│  │  │                                                          │ │   │
│  │  └─────────────────────────────────────────────────────────┘ │   │
│  │    │                                                          │   │
│  │    │ ② 응답 완료                                              │   │
│  │    ▼                                                          │   │
│  │  사용자                                                       │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【구성요소 상세】                                                  │
│  ──────────────────                                                 │
│  • Latency: 요청이 서버에 도달하는 시간 (RTT/2)                     │
│  • Processing Time: 서버가 요청을 처리하는 시간                     │
│  • Transmission Time: 응답 데이터를 전송하는 시간                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 응답 시간 기준 (웹 서비스)

```
┌─────────────────────────────────────────────────────────────────────┐
│                응답 시간 기준                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Doherty Threshold】                                              │
│  ──────────────────────                                             │
│  0.1초 (100ms) 이하: 즉각적 반응, 사용자 개입 없음                  │
│  1초 이하: 자연스러운 대화, 주의 유지                               │
│ 10초 이하: 사용자 주의 유지 가능                                    │
│ 10초 이상: 사용자 이탈 시작                                         │
│                                                                     │
│  【Google 기준】                                                    │
│  ──────────────                                                     │
│  • 100ms: 이상적                                                    │
│  • 300ms: 양호                                                      │
│  • 500ms: 주의 필요                                                 │
│  • 1초+: 개선 필수                                                  │
│  • 3초+: 사용자 50% 이탈                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  사용자 체감:                                                  │ │
│  │  ────────────                                                  │ │
│  │  0~100ms: "즉시 응답"                                          │ │
│  │  100~300ms: "약간의 지연"                                      │ │
│  │  300~1000ms: "느리지만 견딜 만함"                              │ │
│  │  1~3초: "답답함"                                               │ │
│  │  3초+: "포기"                                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 응답 시간 vs 처리량

```
┌─────────────────────────────────────────────────────────────────────┐
│                응답 시간 vs 처리량 트레이드오프                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Little's Law】                                                   │
│  ────────────────                                                   │
│  L = λ × W                                                          │
│                                                                     │
│  L: 시스템 내 요청 수                                               │
│  λ: 도착률 (요청/초)                                                │
│  W: 평균 응답 시간                                                  │
│                                                                     │
│  【트레이드오프】                                                   │
│  ──────────────                                                     │
│  • 처리량↑ → 대기 시간↑ → 응답 시간↑                               │
│  • 동시 사용자↑ → 큐 대기↑ → 응답 시간↑                            │
│                                                                     │
│            응답 시간                                                │
│               ▲                                                     │
│               │                  ╱                                  │
│               │               ╱                                     │
│               │            ╱                                        │
│               │         ╱                                           │
│               │      ╱                                              │
│               │   ╱                                                 │
│               │╱                                                    │
│               └────────────────────────► 처리량                     │
│                                                                     │
│  【운영 지표】                                                      │
│  ──────────────                                                     │
│  • Saturation Point: 처리량 증가 불가 지점                          │
│  • Response Time Degradation: 급격히 증가하는 지점                  │
│  • Optimal Operating Point: 70~80% 활용                             │
└─────────────────────────────────────────────────────────────────────┘
```

### 백분위수 응답 시간

```
┌─────────────────────────────────────────────────────────────────────┐
│                백분위수 (Percentile)                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정의】                                                           │
│  ────────                                                           │
│  P50: 50% 요청이 이 시간 내 완료                                    │
│  P95: 95% 요청이 이 시간 내 완료                                    │
│  P99: 99% 요청이 이 시간 내 완료                                    │
│  P99.9: 99.9% 요청이 이 시간 내 완료                                │
│                                                                     │
│  【예시】                                                           │
│  ────────                                                           │
│  P50 = 50ms: 절반은 50ms 내 응답                                    │
│  P95 = 200ms: 95%는 200ms 내 응답                                   │
│  P99 = 500ms: 99%는 500ms 내 응답                                   │
│  P99.9 = 2s: 0.1%는 2초까지 걸림                                    │
│                                                                     │
│  【중요성】                                                         │
│  ──────────────                                                     │
│  • 평균만 보면 오해 가능                                            │
│  • P99/P99.9가 사용자 경험 결정 (Tail Latency)                      │
│  • SLA는 보통 P99 기준                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SLO/SLA 예시:                                                 │ │
│  │  ─────────────                                                 │ │
│  │  • P50 < 100ms                                                 │ │
│  │  • P95 < 500ms                                                 │ │
│  │  • P99 < 1s                                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 웹 서비스 성능 목표 설정**
- 측정: P50=150ms, P95=800ms, P99=3s
- 목표: P95 < 500ms
- 개선: 캐싱, 쿼리 최적화, CDN

**시나리오 2: 게임 서버 응답 시간**
- 요구: 50ms 이하
- 현재: 평균 30ms, P99 100ms
- 문제: 1% 사용자가 100ms 경험
- 해결: 지역 서버, 예측

**시나리오 3: API 타임아웃 설정**
- 외부 API 응답: P95=2s, P99=10s
- 타임아웃 설정: 5s (P99.5 수준)
- 폴백: 캐시된 데이터
