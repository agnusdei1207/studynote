# 153. Linpack

#### 핵심 인사이트 (3줄 요약)

## Ⅰ. 개요 (Context & Background)

### 개념 정의

Linpack은 1970년대 Jack Dongarra가 개발한 벤치마크로, **선형대수 연산(특히 행렬 곱셈과 LU 분해)** 성능을 측정한다. 슈퍼컴퓨터 성능 순위인 **TOP500**의 공식 벤치마크로 사용된다.

### 💡 비유: 스포츠 경기 공식 기록

Linpack은 **100m 달리기 세계 기록**과 같다. 누가 세상에서 가장 빠른 슈퍼컴퓨터인지 공식적으로 결정한다. TOP500 순위는 이 점수로 매겨진다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### HPL (High Performance Linpack)

```
┌─────────────────────────────────────────────────────────────────────┐
│                HPL 벤치마크                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【테스트 문제】                                                     │
│  ──────────────                                                     │
│                                                                     │
│  1. dense 선형 방정식 시스템: Ax = b                               │
│  2. A: n×n 밀집 행렬 (랜덤 생성)                                    │
│  3. LU 분해 (가우시안 소거법)                                       │
│  4. 전진/후진 대입                                                   │
│                                                                     │
│  【성능 측정】                                                      │
│  ──────────────                                                     │
│                                                                     │
│  FLOPS = (2/3)n³ + (1/2)n²  /  실행 시간                           │
│                                                                     │
│  • n=100,000: 약 6.67×10¹⁴ FLOP                                    │
│  • 1초 실행: 66.7 TFLOPS                                           │
│                                                                     │
│  【알고리즘】                                                        │
│  ──────────────                                                     │
│  1. 행렬 분할 (2D 블록 순환)                                        │
│  2. LU 분해 (부분 피봇팅)                                           │
│  3. 병렬 처리 (MPI)                                                  │
│  4. BLAS 호출 (Level 3)                                             │
│                                                                     │
│  【최적화 요소】                                                     │
│  ──────────────                                                     │
│  • BLAS (Basic Linear Algebra Subprograms)                          │
│  • 행렬 곱셈 최적화 (DGEMM)                                         │
│  • 캐시 친화적 블록 크기                                             │
│  • MPI 통신 최소화                                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BLAS 레벨:                                                    │ │
│  │  ────────────                                                   │ │
│  │  Level 1: 벡터-벡터 (y = αx + y)                              │ │
│  │  Level 2: 행렬-벡터 (y = αAx + βy)                            │ │
│  │  Level 3: 행렬-행렬 (C = αAB + βC) ← HPL에서 주로 사용         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### HPL-AI (HPL with AI Accelerators)

```
┌─────────────────────────────────────────────────────────────────────┐
│                HPL-AI (AI 가속기 포함)                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【배경】                                                           │
│  ────────                                                           │
│  • AI 가속기(GPU, TPU)은 FP64가 아닌 FP16/BF16에 최적화              │
│  • 기존 HPL: FP64 필수                                              │
│  • AI 시대: 혼합 정밀도 벤치마크 필요                                │
│                                                                     │
│  【HPL-AI 특징】                                                     │
│  ────────────────                                                   │
│  • FP64 문제를 FP16/FP32로 변환                                     │
│  • 반복적 정밀도 향상 (Iterative Refinement)                         │
│  • AI 가속기 활용                                                    │
│                                                                     │
│  【결과 예시 (2024)】                                                │
│  ──────────────────                                                 │
│  시스템                HPL (FP64)      HPL-AI (혼합)                 │
│  ───────────────────  ───────────────  ──────────────               │
│  Frontier             1.7 EFLOPS      7.1 EFLOPS                     │
│  Aurora              1.0 EFLOPS      4.2 EFLOPS                      │
│  Eagle               0.56 EFLOPS     2.8 EFLOPS                      │
│                                                                     │
│  → AI 가속기로 4~5× 높은 성능                                        │
└─────────────────────────────────────────────────────────────────────┘
```

### HPCG (High Performance Conjugate Gradients)

```
┌─────────────────────────────────────────────────────────────────────┐
│                HPCG 벤치마크                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【등장 배경】                                                       │
│  ──────────────                                                     │
│  • HPL: dense 행렬에만 특화                                          │
│  • 실제 과학 계산: sparse(희소) 행렬이 많음                            │
│  • 메모리/통신 패턴이 다름                                           │
│                                                                     │
│  【HPCG 특징】                                                       │
│  ──────────────                                                     │
│  • Sparse 행렬 연산                                                  │
│  • Conjugate Gradient(CG) 알고리즘                                   │
│  • 메모리 대역폭·지연 반영                                            │
│  • 현대 애플리케이션에 더 가까운 성능                                   │
│                                                                     │
│  【TOP500 순위】                                                     │
│  ──────────────                                                     │
│  • HPL과 HPCG 동시 발표                                              │
│  • HPCG는 실제 응용 성능에 가까움                                      │
│  • HPL은 최대 이론 성능에 가까움                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  비교 (Frontier, 2024):                                        │ │
│  │  ──────────────────────                                         │ │
│  │  HPL:   1.7 EFLOPS                                              │ │
│  │  HPCG:  0.055 EFLOPS (HPL의 3.2%)                                │ │
│  │                                                               │ │
│  │  → 실제 응용에서는 이론적 최대보다 훨씬 낮음                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### Linpack vs SPEC CPU

| 구분 | Linpack | SPEC CPU |
|:---|:---|:---|
| **대상** | FP 연산 | 정수 + FP |
| **연산** | 행렬 | 다양한 워크로드 |
| **메모리** | 중요 | 중요 |
| **병렬성** | 매우 높음 | 중간 |
| **용도** | 슈퍼컴퓨터 | 일반 CPU |
| **단위** | FLOPS | SPEC 점수 |

### 타 과목 융합

```
┌─────────────────────────────────────────────────────────────────────┐
│                Linpack × 타 과목 융합                                 │
├─────────────────────────────────────────────────────────────────────┤

│  【선형대수】                                                          │
│  • LU 분해: 가우시안 소거법                                            │
│  • 행렬 곱셈: DGEMM (BLAS Level 3)                                      │

│  • 혼합 정밀도: FP16/FP32/FP64                                       │
│  • HPL-AI: AI 가속기 성능 평가                                          │
└─────────────────────────────────────────────────────────────────────┘
```

- 요구: 높은 행렬 연산 성능

### Linpack 결과 해석

```
┌─────────────────────────────────────────────────────────────────────┐
│                Linpack 결과 해석                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FLOPS 수준】                                                       │
│  ─────────────────────────────────────────────────────────────────    │
│  • GFLOPS (10⁹): 고성능 워크스테이션, 개인용 GPU                           │
│  • TFLOPS (10¹²): 소규모 클러스터, 데이터센터 GPU                           │
│  • PFLOPS (10¹⁵): 슈퍼컴퓨터                                             │
│  • EFLOPS (10¹⁸): 엑사스케일 슈퍼컴퓨터                                   │
│                                                                     │
│  【효율 분석】                                                         │
│  ─────────────────────────────────────────────────────────────────    │
│  Efficiency = 실제 FLOPS / 이론적 피크 FLOPS                             │
│                                                                     │
│  • 90%+: 매우 효율적 (최적화 잘됨)                                        │
│  • 70~90%: 양호                                                         │
│  • 50~70%: 개선 여지 있음                                               │
│  • <50%: 병목 존재 (통신, 메모리)                                         │
│                                                                     │
│  【확장성 분석】                                                       │
│  ─────────────────────────────────────────────────────────────────    │
│  • Strong Scaling: 문제 크기 고정, 노드 증가                          │
│  • Weak Scaling: 노드당 문제 크기 증가                                │
│  • 이상적: 선형 확장 (n 노드 = n× 성능)                                   │
│  • 현실: 통신 오버헤드로 인한 이탈                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  TOP500 분석 예시:                                               │  │
│  │  ─────────────────────────────────────────────────────────────  │  │
│  │  Frontier (2024):                                                │  │
│  │  • 피크: 1.7 EFLOPS                                              │  │
│  │  • 노드: 9,408개                                                 │  │
│  │  • 노드당: 181 TFLOPS                                              │  │
│  │  • 효율: ~70%                                                     │  │
│  │  → 통신 오버헤드로 30% 손실                                         │  │
```
