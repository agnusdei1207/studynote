# 146. 무어의 법칙 (Moore's Law)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 반도체 집적회로의 트랜지스터 수가 약 2년마다 2배로 증가한다는 경험적 법칙
> 2. **가치**: 50년간 반도체 산업의 발전 가이드라인, 비용 절감·성능 향상 동시 달성
> 3. **융합**: 물리적 한계 도달, 칩렛·3D 적층으로 새로운 방향 전환

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

무어의 법칙(Moore's Law)은 1965년 인텔 공동 창업자 고든 무어(Gordon Moore)가 발표한 법칙으로, **반도체 칩의 트랜지스터 밀도가 약 18~24개월마다 2배로 증가**한다는 관측이다. 엄밀한 물리 법칙이 아닌, 산업의 발전 속도를 예측한 경험적 관찰이다.

### 💡 비유: 도시 인구 성장

도시가 매년 인구가 2배로 늘어난다면, 10년 후에는 1000배, 20년 후에는 100만 배가 된다. 무어의 법칙은 이런 기하급수적 성장을 반도체에 적용한 것이다.

### 역사와 전개

```
┌─────────────────────────────────────────────────────────────────────┐
│                무어의 법칙 역사                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  연도     사건                                                     │
│  ─────   ─────────────────────────────────────────────────────     │
│  1965    무어 논문 발표: "1년마다 2배"                              │
│  1975    수정: "2년마다 2배"                                        │
│  1971    Intel 4004: 2,300 트랜지스터                               │
│  1993    Pentium: 310만 트랜지스터                                  │
│  2000    Pentium 4: 5,500만 트랜지스터                              │
│  2010    10억 트랜지스터 돌파                                        │
│  2020    Apple M1: 160억 트랜지스터                                 │
│  2024    NVIDIA B200: 1,000억 트랜지스터                            │
│                                                                     │
│  【50년간 성장】                                                    │
│  ──────────────                                                     │
│  1971 → 2024: 약 430만 배 증가                                      │
│  2,300 → 100,000,000,000                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  무어의 법칙 수학적 표현:                                      │ │
│  │  ──────────────────────────                                    │ │
│  │  T(n) = T₀ × 2^(n/2)                                           │ │
│  │                                                               │ │
│  │  T₀: 초기 트랜지스터 수                                         │ │
│  │  n: 경과 연도                                                   │ │
│  │  2: 2년마다 2배                                                 │ │
│  │                                                               │ │
│  │  50년 후: T(50) = T₀ × 2^25 ≈ 33,000,000×                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 무어의 법칙의 의미

```
┌─────────────────────────────────────────────────────────────────────┐
│                무어의 법칙의 3가지 의미                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 성능 향상】                                                   │
│  ──────────────                                                     │
│  • 더 많은 트랜지스터 = 더 많은 연산 유닛                           │
│  • 더 많은 캐시                                                     │
│  • 더 복잡한 아키텍처 가능                                          │
│                                                                     │
│  【2. 비용 절감】                                                   │
│  ──────────────                                                     │
│  • 칩 크기 감소 → 단위당 비용 감소                                  │
│  • 동일 가격에 더 높은 성능                                         │
│  • 반대: 동일 성능에 더 낮은 가격                                   │
│                                                                     │
│  【3. 전력 효율】                                                   │
│  ──────────────                                                     │
│  • 트랜지스터 축소 → 동작 전압 감소                                 │
│  • 더 작은 전력으로 동일 성능                                       │
│  • (단, 데나드 스케일링 붕괴 후 제한적)                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  무어의 법칙 = "더 많이, 더 싸게, 더 효율적으로"              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 무어의 법칙 vs 데나드 스케일링

| 구분 | 무어의 법칙 | 데나드 스케일링 |
|:---|:---|:---|
| **정의** | 트랜지스터 수 2배/2년 | 전력 밀도 일정 |
| **기간** | 1965~현재 | 1974~2006 |
| **상태** | 둔화 중 | 붕괴 |
| **결과** | 성능↑ | 전력↑ |


## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 5년 후 시스템 설계**
- 현재: 64코어, 256GB 메모리
- 5년 후 예상: 256코어 (무어 기준)
- 설계: 확장 가능한 아키텍처

**시나리오 2: 비용 대비 성능 분석**
- 2년 전: $1000 CPU
- 현재: 동일 가격 2× 성능
- 2년 후: 동일 가격 4× 성능
- 의사결정: 업그레이드 시기

**시나리오 3: 전력 예산 계획**
- 데나드 붕괴로 전력 밀도 증가
- 새 서버: 더 높은 냉각 비용
- 해결: ARM 서버, 효율형 코어
