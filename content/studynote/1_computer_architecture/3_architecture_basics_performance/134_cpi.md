# 134. CPI (Cycles Per Instruction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하나의 명령어를 실행하는 데 필요한 평균 클럭 사이클 수
> 2. **가치**: CPU 아키텍처 효율성의 핵심 지표, 낮을수록 효율적
> 3. **융합**: IPC의 역수, 파이프라인·캐시·분기예측이 CPI 결정

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

CPI(Cycles Per Instruction)는 프로그램 실행에 필요한 총 클럭 사이클 수를 총 명령어 수로 나눈 값이다. 즉, **명령어 하나를 실행하는 데 평균 몇 개의 클럭 사이클이 필요한지**를 나타낸다.

### 💡 비유: 식당에서 음식 하나 만드는 시간

CPI는 **식당에서 메뉴 하나를 만드는 평균 시간**과 같다. 단순한 샐러드는 1분(CPI=1), 복잡한 스테이크는 5분(CPI=5). 평균 CPI가 낮을수록 주방이 효율적이다.

### 기본 공식

```
CPI = 총 클럭 사이클 수 / 총 명령어 수

┌────────────────────────────────────────────────────┐
│  예시:                                                            │
│  • 10억 개 명령어 실행에 25억 사이클 소요                        │
│  • CPI = 25억 / 10억 = 2.5                                        │
│  • 평균적으로 명령어 1개당 2.5 사이클 필요                        │
└────────────────────────────────────────────────────┘
```

### CPI와 성능의 관계

```
┌─────────────────────────────────────────────────────────────────────┐
│                성능 공식                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  실행 시간 = 명령어 수 × CPI × 클럭 주기                           │
│            = 명령어 수 × CPI / 클럭 주파수                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  성능 향상 전략:                                               │ │
│  │  ────────────────                                             │ │
│  │  1. 명령어 수 감소: 컴파일러 최적화, 효율적 ISA                │ │
│  │  2. CPI 감소: 파이프라인, 캐시, 분기 예측                     │ │
│  │  3. 클럭 증가: 빠른 공정, 파이프라인 깊이 증가                │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CPU Time 공식】                                                  │
│  ────────────────                                                   │
│                                                                     │
│       Instructions     CPI                                          │
│  CPU Time = ─────────── × ─────                                     │
│                 Program    Clock Cycle Time                         │
│                                                                     │
│       IC × CPI                                                        │
│  = ────────────────                                                  │
│         f                                                              │
│                                                                     │
│  (IC = Instruction Count, f = Clock Frequency)                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 명령어 유형별 CPI

```
┌─────────────────────────────────────────────────────────────────────┐
│                명령어 유형별 CPI 예시                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  명령어 유형       CPI     이유                                    │
│  ───────────────  ────   ─────────────────────────────             │
│  정수 ALU (ADD)    1      파이프라인, 캐시 히트                    │
│  정수 ALU (MUL)    3~4    복잡한 연산                               │
│  정수 ALU (DIV)    10~30  매우 복잡한 연산                          │
│  부동소수 (ADD)    3~5    FPU 파이프라인                            │
│  부동소수 (MUL)    4~7    FPU 파이프라인                            │
│  부동소수 (DIV)    10~30  FPU 비파이프라인                          │
│  메모리 로드       1~100+ 캐시 히트/미스에 따라                     │
│  메모리 스토어     1~50   캐시 히트/미스에 따라                     │
│  분기 (예측 히트)  1      분기 예측 성공                            │
│  분기 (예측 미스)  10~20  파이프라인 플러시                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  평균 CPI 계산 예시:                                           │ │
│  │  ──────────────────                                            │ │
│  │  명령어 분포: ALU 60%, Load 20%, Store 10%, Branch 10%         │ │
│  │                                                               │ │
│  │  CPI_ALU = 1, CPI_Load = 4, CPI_Store = 2, CPI_Branch = 3    │ │
│  │                                                               │ │
│  │  평균 CPI = 0.6×1 + 0.2×4 + 0.1×2 + 0.1×3                    │ │
│  │           = 0.6 + 0.8 + 0.2 + 0.3                            │ │
│  │           = 1.9                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### CPI에 영향을 미치는 요인

```
┌─────────────────────────────────────────────────────────────────────┐
│                CPI 영향 요인                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CPI 감소 요인 (좋음)】                                           │
│  ──────────────────────                                             │
│                                                                     │
│  1. 파이프라인                                                       │
│     • 명령어 중첩 실행                                               │
│     • 이상적 CPI → 1 (또는 그 이하, 슈퍼스칼라)                     │
│                                                                     │
│  2. 캐시 메모리                                                      │
│     • 메모리 접근 시간 단축                                          │
│     • L1 히트: 1~4 사이클                                           │
│                                                                     │
│  3. 분기 예측                                                        │
│     • 분기 미스 감소                                                 │
│     • 정확도 95%+ 달성                                               │
│                                                                     │
│  4. 슈퍼스칼라                                                        │
│     • 사이클당 여러 명령어 실행                                     │
│     • CPI < 1 가능 (IPC > 1)                                        │
│                                                                     │
│  【CPI 증가 요인 (나쁨)】                                           │
│  ──────────────────────                                             │
│                                                                     │
│  1. 캐시 미스                                                        │
│     • L1 미스: 10~20 사이클                                         │
│     • L2 미스: 30~50 사이클                                         │
│     • L3 미스: 100+ 사이클                                          │
│     • 메인 메모리: 200~300 사이클                                   │
│                                                                     │
│  2. 분기 예측 미스                                                   │
│     • 파이프라인 플러시                                              │
│     • 10~20 사이클 손실                                              │
│                                                                     │
│  3. 데이터 의존성                                                     │
│     • 스톨(Stall) 발생                                               │
│     • RAW, WAW, WAR 해저드                                          │
│                                                                     │
│  4. TLB 미스                                                         │
│     • 페이지 테이블 접근 필요                                        │
│     • 100+ 사이클                                                    │
└─────────────────────────────────────────────────────────────────────┘
```

### 파이프라인과 CPI

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 CPI                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【이상적 파이프라인】                                               │
│  ────────────────────                                              │
│  CPI_ideal = 1 (사이클당 1 명령어 완료)                             │
│                                                                     │
│  【실제 파이프라인 CPI】                                            │
│  ──────────────────────                                             │
│  CPI_actual = CPI_ideal + CPI_stall                                 │
│                                                                     │
│  CPI_stall 요인:                                                    │
│  • 구조적 해저드: 리소스 충돌                                       │
│  • 데이터 해저드: 의존성 대기                                       │
│  • 제어 해저드: 분기 미스                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPI 계산 예시:                                                │ │
│  │  ─────────────                                                │ │
│  │  • 기본 CPI = 1                                                │ │
│  │  • 캐시 미스율: 5%, 미스 페널티: 20 사이클                     │ │
│  │  • 분기 미스율: 2%, 미스 페널티: 15 사이클                     │ │
│  │  • 분기 비율: 20%                                              │ │
│  │                                                               │ │
│  │  CPI = 1 + (0.05 × 20) + (0.20 × 0.02 × 15)                  │ │
│  │      = 1 + 1.0 + 0.06                                         │ │
│  │      = 2.06                                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 아키텍처별 CPI 비교

| 아키텍처 | 특징 | 평균 CPI |
|:---|:---|:---:|
| **단일 사이클** | 간단, 느린 클럭 | 1 |
| **다중 사이클** | 빠른 클럭, 복잡 | 2~5 |
| **파이프라인** | 병렬 처리 | 1~1.5 |
| **슈퍼스칼라** | 다중 실행 유닛 | <1 (IPC>1) |
| **VLIW** | 컴파일러 스케줄링 | <1 |

### CPI vs IPC

```
┌─────────────────────────────────────────────────────────────────────┐
│                CPI vs IPC 관계                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  IPC (Instructions Per Cycle) = 1 / CPI                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPI       IPC       설명                                     │ │
│  │  ────     ────     ─────────────────────────                 │ │
│  │  4.0      0.25     비효율적 (과거 CPU)                        │ │
│  │  2.0      0.50     보통                                       │ │
│  │  1.5      0.67     양호                                       │ │
│  │  1.0      1.00     파이프라인 이상적                           │ │
│  │  0.5      2.00     슈퍼스칼라 (사이클당 2명령어)               │ │
│  │  0.33     3.00     고성능 슈퍼스칼라                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ※ 현대 CPU는 IPC로 표현하는 것이 더 직관적                        │
│     (IPC > 1인 경우 CPI < 1, 분수로 표현하기 어려움)                │
└─────────────────────────────────────────────────────────────────────┘
```

### 과목 융합

```
┌─────────────────────────────────────────────────────────────────────┐
│                CPI × 타 과목 융합                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                       │
│  • 명령어 스케줄링: CPI 감소                                        │
│  • 레지스터 할당: 메모리 접근 감소                                  │
│  • 루프 최적화: 분기 감소                                           │
│                                                                     │
│  【운영체제】                                                       │
│  • 페이지 폴트: TLB 미스 → CPI 증가                                 │
│  • 컨텍스트 스위치: 캐시 오염 → CPI 증가                            │

│                                                                     │
│  【메모리 계층】                                                    │
│  • 캐시 크기: 미스율 → CPI 영향                                     │
│  • 캐시 연관도: 충돌 미스 감소                                      │
│  • 프리페칭: 미스 감소                                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: CPU 성능 분석**
- 문제: 프로그램 실행 시간 측정
- 방법: CPI × 명령어 수 / 클럭
- 결과: 병목 구간 식별

**시나리오 2: 컴파일러 최적화**
- 목표: CPI 감소
- 방법: 명령어 스케줄링, 레지스터 할당
- 결과: 성능 10~30% 향상

**시나리오 3: 캐시 설계**
- 목표: 캐시 미스율 감소 → CPI 감소
- 방법: 캐시 크기, 연관도 조정
- 결과: 메모리 bound 프로그램 가속

### CPI 측정 방법

```
┌─────────────────────────────────────────────────────────────────────┐
│                CPI 측정 방법                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드웨어 카운터】                                                │
│  ──────────────────                                                │
│  • CPU_CYCLES: 총 사이클 수                                        │
```

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 최적화 | CPI 변화 | 성능 변화 |
|:---|:---:|:---:|
| **파이프라인** | 4.0 → 1.5 | +166% |
| **캐시 추가** | 2.0 → 1.3 | +54% |
| **분기 예측** | 1.5 → 1.2 | +25% |
| **슈퍼스칼라** | 1.0 → 0.5 | +100% |

### 핵심 공식 정리

```
CPI = 총 사이클 / 총 명령어
IPC = 1 / CPI
실행 시간 = 명령어 × CPI / 클럭주파수
```

### 👶 어린이를 위한 3줄 비유 설명

**개념**: CPI는 일 하나를 끝내는 데 몇 번의 "째깍"이 필요한지 평균이에요!

**원리**: 쉬운 일은 1번째깍, 어려운 일은 5번째깍. 평균 CPI가 낮을수록 빠른 컴퓨터예요!

**효과**: CPI를 낮추면 같은 일을 더 빨리 끝낼 수 있어요!
