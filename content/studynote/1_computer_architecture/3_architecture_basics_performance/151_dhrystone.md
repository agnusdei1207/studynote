# 151. Dhrystone (Dhrystone)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 정수 연산 성능 측정하는 고전 합성 벤치마크, 1976년 John Henness가 만든
> 2. **가치**: 역사적 중요,3. CISC 아키텍처 성능 비교에 적합
> 3. **융합**: FP 성능 벤치마크 유사, 부동소수점 수 벤치

### 💡 비유: 사관관 시험 vs 부동소수점의 계산 시험

**Dhrystone**: 시계 30ms. Whetstone 부동소수., 104년 670ms 처리 속도을 비교.

            **단점: 전력 소비 높음,            **결론: RISC가 더 효율적이다

            - 단일 명령어 길이 증가 지연
            - 더 많은 레지스터, 더 유연한 코드 구조

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 서버 구매**
- Intel Xeon vs AMD EPYC
- 비교: SPECrate 2017_int
- 분석: 단일 스레드는 멀티 코어 병렬 처리 능력
- 결정: SPECint로 SPECrate 비교
- 예산, 발열, 전력 비용 고려

**시나리오 2: 임베디드 시스템 비교**
- Intel Core i9-13900K: 84 SPECint,2017
- AMD Ryzen 9 7950X: 77 SPECint 2017
- Apple M3 Max: ~70 SPECint 2017

→ M3 Max이 전력 효율 뛠 좋음

**시나리오 3: 클라우드 GPU 선택 (Deep Learning)**
- 지표: SPECspeed (단일), SPECrate (멀티)
- 예산: AI 학습 시간, 처리량 중요
- **MLPerf**로 비교 (NVIDIA H100 기준)
- 결정: MLPerf Inference - NVIDIA가 더 빠름
- 비용은 NVIDIA H100가 더 비싸

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 항목 | 이점 |
|:---|:---:|
| **코드 밀도** | **실행 성능 비** |
| **컴파일러 최적화 효과** | **실행 연산 성능 비교 욨}{ 목적별 |
| **실제 워크로드 반영**** | Dhrystone은 훨씬 빠르고 부동소수점 수치 연산이 복잡한 구조를 가지 잘 수 있다.

### Dhrystone의 장단점

1. **단일 코어 테스트 부적절**: 2코어 Dhrystone 성능 = 4코어 Dhrystone 성능 × 2
   - 코어 수 증가 → CPI 증가하지 않
   - 4코어와 2 코어 Dhrystone: CPI ≈ 0.4 (더 나쁨)
   - 64코어: CPI ≈ 0.25 (16×)
   - 128코어: CPI ≈ 0.125 (8×)
   ```
   → **코어 수 증가할수록 CPI 감소하는 효과 감소**
   ```

### Whetstone의 장점

1. **부동소수점 전문**: FP 연산에 최적화
   - Whetstone은 805년에 작성, 이후 Fortran으로 대체
   - **SIMD 벡터화**: 현대 CPU는 SIMD로 8~16개의 32비트 부동소수를 한 번에 4개의 FP 연산 수행
   - 웨플러 수치 개선 (PDE 등 웨플러 수치이 웨플러 수치 수정)
   - Whetstone은 FP 연산에 비교적 최습니다 (예: FP 곱셈 vs FP 나눗)
