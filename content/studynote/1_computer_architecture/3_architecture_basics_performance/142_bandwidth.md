# 142. 대역폭 (Bandwidth)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단위 시간당 전송 가능한 최대 데이터 양, bps/Gbps/GB/s 단위
> 2. **가치**: 대용량 전송·스트리밍·데이터센터의 핵심 지표, 높을수록 좋음
> 3. **융합**: 지연 시간과 독립적, 버스·메모리·네트워크 계층 모두 적용

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

대역폭(Bandwidth)은 주어진 시간 동안 전송될 수 있는 최대 데이터 양을 의미한다. 네트워크, 메모리, 버스 등 다양한 계층에서 사용되며, 단위는 bps(초당 비트), GB/s(초당 기가바이트) 등을 사용한다.

### 💡 비유: 도로 차선 수

대역폭은 **도로의 차선 수**와 같다. 1차선 도로는 1차량/초, 8차선 도로는 8차량/초 통과 가능하다. 차선이 많을수록 동시에 많은 차가 지나갈 수 있다. 하지만 도로 길이(지연)와는 별개다.

### 대역폭 단위

```
┌─────────────────────────────────────────────────────────────────────┐
│                대역폭 단위                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【네트워크 (비트 기준)】                                           │
│  ──────────────────────                                             │
│  단위         값              예시                                  │
│  ──────────  ─────────────  ───────────────────────                │
│  bps         1 bit/s        구형 모뎀                               │
│  Kbps        10³ bit/s      전화선                                  │
│  Mbps        10⁶ bit/s      일반 인터넷                             │
│  Gbps        10⁹ bit/s      기가 인터넷, DC 내부                    │
│  Tbps        10¹² bit/s     백본 네트워크                           │
│                                                                     │
│  【메모리/버스 (바이트 기준)】                                      │
│  ────────────────────────────                                       │
│  단위         값              예시                                  │
│  ──────────  ─────────────  ───────────────────────                │
│  MB/s        10⁶ B/s        SATA SSD                                │
│  GB/s        10⁹ B/s        DDR4, PCIe 4.0                          │
│  TB/s        10¹² B/s       HBM3, NVLink                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ※ 1 Byte = 8 bits                                            │ │
│  │  100 Mbps = 12.5 MB/s                                         │ │
│  │  1 Gbps = 125 MB/s                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 대역폭 계산

```
┌─────────────────────────────────────────────────────────────────────┐
│                대역폭 계산                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 공식】                                                      │
│  ──────────────                                                     │
│  Bandwidth = 데이터 전송량 / 시간                                   │
│            = 데이터 크기 / 전송 시간                                │
│                                                                     │
│  【메모리 대역폭】                                                  │
│  ──────────────────                                                 │
│  Memory BW = 데이터 폭 × 전송 속도 × 채널 수                        │
│                                                                     │
│  예: DDR5-5600 듀얼 채널                                            │
│  • 데이터 폭: 8 Bytes (64-bit)                                      │
│  • 전송 속도: 5600 MT/s                                             │
│  • 채널 수: 2                                                       │
│  • 대역폭 = 8 × 5600 × 2 = 89,600 MB/s = 89.6 GB/s                 │
│                                                                     │
│  【네트워크 대역폭】                                                │
│  ──────────────────                                                 │
│  Effective BW = (패킷 크기 - 오버헤드) / 전송 시간                  │
│                                                                     │
│  예: 1Gbps 이더넷, 1500바이트 패킷                                  │
│  • 오버헤드: 38바이트 (이더넷 헤더 + IP + TCP + 프리앰블)          │
│  • 효율 = 1462 / 1500 = 97.5%                                       │
│  • 유효 대역폭 ≈ 975 Mbps                                           │
│                                                                     │
│  【버스 대역폭】                                                    │
│  ──────────────                                                     │
│  Bus BW = 폭 × 클럭 × 인코딩 효율                                   │
│                                                                     │
│  예: PCIe 4.0 x16                                                   │
│  • 폭: 16 lanes                                                     │
│  • 클럭: 16 GT/s (per lane)                                         │
│  • 인코딩: 128b/130b (98.5% 효율)                                   │
│  • 대역폭 = 16 × 16 × 2 × 0.985 = 64 GB/s (양방향 합)              │
└─────────────────────────────────────────────────────────────────────┘
```

### 대역폭 vs 처리량

```
┌─────────────────────────────────────────────────────────────────────┐
│                대역폭 vs 처리량                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정의 차이】                                                      │
│  ──────────────                                                     │
│  • 대역폭 (Bandwidth): 최대 전송 용량 (이론적 상한)                 │
│  • 처리량 (Throughput): 실제 전송된 데이터 양                       │
│                                                                     │
│  【관계식】                                                         │
│  ──────────────                                                     │
│  Throughput ≤ Bandwidth                                             │
│  Utilization = Throughput / Bandwidth                               │
│                                                                     │
│  【처리량 감소 요인】                                               │
│  ──────────────────────                                             │
│  • 프로토콜 오버헤드 (헤더, 체크섬)                                 │
│  • 재전송 (패킷 손실)                                               │
│  • 혼잡 제어                                                        │
│  • 오버헤드 (인터럽트, 컨텍스트 스위치)                             │
│  • 병목 구간                                                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시: 1 Gbps 링크                                             │ │
│  │  ─────────────────                                             │ │
│  │  대역폭: 1 Gbps (100%)                                         │ │
│  │  프로토콜 오버헤드: -5%                                        │ │
│  │  혼잡/재전송: -10%                                             │ │
│  │  처리량: 850 Mbps (85%)                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 데이터센터 네트워크 설계**
- 요구: AI 클러스터 간 통신
- 분석: H100 GPU × 8 = 7.2 TB/s 필요
- 선택: NVLink + InfiniBand 400G

**시나리오 2: 스토리지 시스템**
- 요구: 100TB 백업 1시간 내 완료
- 계산: 100TB / 3600s = 28 GB/s
- 선택: NVMe SSD 4장 RAID 0

**시나리오 3: 스트리밍 서비스**
- 요구: 동시 10만 시청자, 4K
- 계산: 10만 × 25 Mbps = 2.5 Tbps
- 선택: CDN + 100G 백본

**시나리오 4: 메모리 대역폭 병목**
- 문제: HPC 애플리케이션 성능 저하
- 분석: 메모리 대역폭 80% 활용
- 해결: DDR5, 쿼드 채널, NUMA 최적화

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 증설 | 대역폭 | 효과 |
|:---|:---:|:---|
| **1G→10G** | 10× | 대용량 전송 10× |
| **DDR4→DDR5** | 2× | 메모리 bound 앱 가속 |
| **PCIe 4→5** | 2× | GPU/SSD 병목 해소 |

### 대역폭 vs 지연 시간 정리

```
• 대역폭↑ = 더 많은 데이터 동시 전송
• 지연↓ = 더 빠른 응답
• 대용량 전송 → 대역폭 중요
• 실시간 응답 → 지연 중요
```

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 대역폭은 한 번에 얼마나 많은 데이터를 보낼 수 있는지예요!

**원리**: 넓은 도로(높은 대역폭)는 많은 차가 동시에 지나가요. 좁은 도로(낮은 대역폭)는 한 번에 조금밖에 못 지나가요!

**효과**: 동영상이나 큰 파일을 받을 때 대역폭이 높으면 빨리 받아요!
