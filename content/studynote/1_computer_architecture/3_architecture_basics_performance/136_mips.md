# 136. MIPS (Million Instructions Per Second)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 1초에 실행하는 명령어 수를 백만 단위로 표현한 성능 지표
> 2. **가치**: 직관적인 성능 비교 가능, IPC × 클럭주파수 / 10⁶ 으로 계산
> 3. **융합**: 단순하지만 워크로드 의존적, FLOPS·SPEC 점수가 더 정확한 비교

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

MIPS(Million Instructions Per Second)는 CPU가 1초 동안 실행하는 명령어 수를 백만 단위로 나타낸 성능 측정 지표다. 예를 들어 1000 MIPS = 1초에 10억 개의 명령어 실행을 의미한다.

### 💡 비유: 공장의 생산량

MIPS는 **공장이 1초에 완성하는 제품 수(백만 개 단위)**와 같다. 1000 MIPS 공장은 1초에 10억 개 부품을 조립한다. 하지만 단순한 부품(단순 명령어)과 복잡한 부품(복잡 명령어)은 다르다.

### 기본 공식

```
MIPS = 명령어 수 / (실행 시간 × 10⁶)
MIPS = IPC × 클럭 주파수 (MHz)
MIPS = 클럭 주파수 (MHz) / CPI

┌────────────────────────────────────────────────────┐
│  예시:                                                            │
│  • 클럭: 3 GHz = 3000 MHz                                         │
│  • IPC: 2.0                                                       │
│  • MIPS = 2.0 × 3000 = 6000 MIPS                                  │
│  → 1초에 60억 개 명령어 실행                                      │
└────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### MIPS 계산 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                MIPS 계산                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【공식 1: 기본】                                                   │
│  ────────────────                                                  │
│                                                                     │
│       명령어 수 (Instructions)                                      │
│  MIPS = ──────────────────────────                                  │
│         실행 시간 (초) × 10⁶                                        │
│                                                                     │
│  【공식 2: IPC 활용】                                               │
│  ────────────────────                                               │
│                                                                     │
│  MIPS = IPC × 클럭 주파수 (MHz)                                     │
│                                                                     │
│  【공식 3: CPI 활용】                                               │
│  ────────────────────                                               │
│                                                                     │
│       클럭 주파수 (MHz)                                             │
│  MIPS = ─────────────────                                           │
│              CPI                                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  계산 예시:                                                    │ │
│  │  ─────────────                                                │ │
│  │  CPU: 4 GHz, IPC 2.5                                          │ │
│  │                                                               │ │
│  │  MIPS = 2.5 × 4000 = 10,000 MIPS                              │ │
│  │                                                               │ │
│  │  → 1초에 100억 개 명령어 실행                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실행 시간 역산】                                                 │
│  ──────────────────                                                 │
│                                                                     │
│  실행 시간 = 명령어 수 / (MIPS × 10⁶)                              │
│                                                                     │
│  예: 100억 명령어, 5000 MIPS                                       │
│  실행 시간 = 10¹⁰ / (5000 × 10⁶) = 2초                              │
└─────────────────────────────────────────────────────────────────────┘
```

### MIPS의 한계

```
┌─────────────────────────────────────────────────────────────────────┐
│                MIPS의 한계                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【한계 1: 명령어 복잡도 차이】                                     │
│  ──────────────────────────                                         │
│  • CISC (x86): 복잡한 명령어 1개 = 여러 연산                        │
│  • RISC (ARM): 단순한 명령어 여러 개 = 같은 연산                    │
│  → MIPS만으로는 공정 비교 불가                                      │
│                                                                     │
│  【한계 2: 워크로드 의존성】                                        │
│  ────────────────────────                                           │
│  • 정수 연산: 높은 MIPS                                             │
│  • 메모리 접근: 낮은 MIPS                                           │
│  • 부동소수점: 낮은 MIPS                                            │
│  → 같은 CPU도 워크로드에 따라 MIPS 다름                             │
│                                                                     │
│  【한계 3: 멀티코어**                                               │
│  ──────────────────                                                 │
│  • 단순 합산이 의미 없을 수 있음                                    │
│  • 병렬화 효율에 따라 실제 성능 다름                                │
│                                                                     │
│  【한계 4: 캐시/메모리 영향】                                       │
│  ──────────────────────                                             │
│  • 캐시 미스 시 MIPS 급감                                           │
│  • 메모리 bound 프로그램은 MIPS 낮음                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  "MIPS는 속도계가 아니라 엔진 RPM과 같다"                      │ │
│  │  ──────────────────────────────────────────                    │ │
│  │  RPM이 높아도 기어비(아키텍처)에 따라 속도(성능)가 다름        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### Peak MIPS vs Native MIPS

```
┌─────────────────────────────────────────────────────────────────────┐
│                MIPS 분류                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Peak MIPS】                                                      │
│  ─────────────                                                      │
│  이론적 최대 성능                                                   │
│  Peak MIPS = 클럭(MHz) × 실행 유닛 수                               │
│                                                                     │
│  예: 4 GHz, 4-wide 실행                                            │
│  Peak MIPS = 4000 × 4 = 16000 MIPS                                 │
│                                                                     │
│  【Native MIPS】                                                    │
│  ──────────────                                                     │
│  실제 워크로드에서 측정된 MIPS                                      │
│  일반적으로 Peak의 30~60% 수준                                      │
│                                                                     │
│  【Relative MIPS】                                                  │
│  ──────────────────                                                 │
│  기준 머신 대비 상대적 MIPS                                         │
│  VAX 11/780 = 1 MIPS (역사적 기준)                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  MIPS/MHz (IPC와 동일):                                        │ │
│  │  ─────────────────────                                         │ │
│  │  • Intel Core: 1.8~2.2 MIPS/MHz                               │ │
│  │  • AMD Zen: 1.6~2.0 MIPS/MHz                                  │ │
│  │  • Apple M: 2.5~3.0 MIPS/MHz                                  │ │
│  │  • ARM Cortex-A: 1.5~2.0 MIPS/MHz                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### MIPS vs FLOPS

| 지표 | 의미 | 용도 | 단위 |
|:---|:---|:---|:---|
| **MIPS** | 초당 백만 명령어 | 범용 연산 | MIPS |
| **FLOPS** | 초당 부동소수점 연산 | 과학 연산 | GFLOPS, TFLOPS |
| **IPS** | 초당 명령어 | 절대적 수치 | GIPS |
| **OPS** | 초당 연산 | AI, 정수 | TOPS |

```
┌─────────────────────────────────────────────────────────────────────┐
│                MIPS vs FLOPS 비교                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MIPS】                                                           │
│  ────────                                                           │
│  • 모든 명령어 포함 (정수, 분기, 메모리)                            │
│  • 범용 CPU 성능 지표                                               │
│  • 워크로드에 따라 편차 큼                                          │
│                                                                     │
│  【FLOPS】                                                          │
│  ──────────                                                          │
│  • 부동소수점 연산만 측정                                           │
│  • 슈퍼컴퓨터, GPU 성능 지표                                        │
│  • 과학/그래픽 연산에 적합                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시: NVIDIA RTX 4090                                        │ │
│  │  ────────────────────────                                     │ │
│  │  • FP32: 82.6 TFLOPS                                          │ │
│  │  • INT8 (AI): 660 TOPS                                        │ │
│  │  • MIPS: 측정 어려움 (GPU는 명령어 기반이 아님)               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 서버 용량 계획**
- 요구: 1000명 동시 사용자
- 분석: 사용자당 10 MIPS 필요
- 선택: 10000 MIPS 이상 서버

**시나리오 2: MCU 선택**
- 요구: 실시간 제어
- 분석: 제어 루프 100μs, 5000명령어
- 필요: 50 MIPS 이상 MCU

**시나리오 3: 알고리즘 비교**
- 알고리즘 A: 10억 명령어
- 알고리즘 B: 5억 명령어
- 5000 MIPS CPU: A=0.2초, B=0.1초

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 활용 | 장점 | 단점 |
|:---|:---|:---|
| **빠른 비교** | 직관적 수치 | 아키텍처 차이 무시 |
| **용량 계획** | 정량적 기준 | 워크로드 의존 |
| **트렌드 파악** | 역사적 비교 | 현대 CPU는 더 복잡 |

### 대안 지표

- **SPEC 점수**: 표준 벤치마크
- **FLOPS**: 부동소수점 성능
- **Geekbench**: 종합 점수
- **TPS**: 초당 트랜잭션
