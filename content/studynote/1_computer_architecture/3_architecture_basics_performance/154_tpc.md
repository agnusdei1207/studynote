# TPC 벤치마크 (TPC Benchmark)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TPC (Transaction Processing Performance Council) 벤치마크는 비영리 단체인 TPC에서 정의한 **데이터베이스 및 트랜잭션 처리 성능 측정 표준**으로, 실제 비즈니스 환경의 작업 부하를 모방하여 시스템의 종합적 처리 능력을 평가한다.
> 2. **가치**: 단순히 연산 속도만 재는 것이 아니라, 디스크 I/O, 메모리 관리, 네트워크 통신 등 시스템 전반의 상호작용을 측정하며, **가성비($/tpmC$)** 지표를 통해 하드웨어 도입의 경제적 타당성을 제시한다.
> 3. **융합**: 온라인 트랜잭션 중심의 **TPC-C**, 의사결정 지원(빅데이터 분석) 중심의 **TPC-H**, 클라우드 및 가상화 환경을 위한 **TPC-V** 등으로 전문화되어 엔터프라이즈 아키텍처 설계의 표준 가이드라인으로 활용된다.

+++

## Ⅰ. TPC 벤치마크의 개요

### 1. 정의
- 데이터베이스 관리 시스템(DBMS)과 이를 구동하는 하드웨어 서버의 통합 성능을 측정하기 위해 고안된 표준 시나리오 집합이다. 트랜잭션의 성공적 완료율과 응답 시간을 기준으로 성적을 매긴다.

### 2. 등장 배경: '상업용 성능의 실질적 잣대'
- CPU의 연산력(MIPS, FLOPS)이 아무리 좋아도, 정작 은행이나 쇼핑몰 시스템에서 고객의 주문을 처리할 때 디스크가 느리면 시스템은 버벅거린다. 하드웨어 제조사들의 과장 광고를 막고, 기업 고객들이 "실제 내 비즈니스 업무에서 얼마나 빠를 것인가"를 객관적으로 비교할 수 있도록 상업용 표준인 TPC가 탄생했다.

### 3. 💡 비유: '대형 마트의 계산대 처리 능력'
- TPC 벤치마크는 **'주말 대형 마트에서 계산대 직원들이 1시간 동안 몇 명의 손님을 결제 완료시킬 수 있는지 재는 것'**과 같다.
- 직원의 손놀림(CPU)도 중요하지만, 바코드 스캐너의 속도(I/O), 영수증 종이의 공급(메모리), 손님들이 줄 서는 공간의 배치(네트워크)가 모두 조화를 이루어야 '높은 매출(성능)'을 낼 수 있는 원리를 측정한다.

+++

## Ⅱ. 주요 TPC 스위트 및 측정 지표 (Deep Dive)

비즈니스 성격에 따라 서로 다른 테스트 시나리오를 적용한다.

### 1. 대표적인 TPC 벤치마크 유형
| 유형 | 명칭 | 주 측정 대상 | 상세 시나리오 |
|:---|:---|:---|:---|
| **TPC-C** | **OLTP** | 실시간 거래 처리 | 주문 입력, 결제, 배송 현황 조회 등 반복적 작업 |
| **TPC-H** | **OLAP** | 의사결정 지원 | 대규모 데이터에 대한 복잡한 통계 쿼리 분석 |
| **TPC-DS** | **Big Data** | 데이터 웨어하우스 | 정형/비정형 데이터가 혼재된 현대적 분석 환경 |
| **TPC-V** | **Virtualization** | 가상화 효율 | 가상 머신(VM) 간의 자원 경합 및 격리 성능 |

### 2. 핵심 성능 지표 (Metric)
- **tpmC (Transactions Per Minute C)**: TPC-C 시나리오에서 분당 처리 가능한 최대 트랜잭션 건수.
- **$/tpmC (Price/Performance)**: 성능 1단위를 얻기 위해 투입되는 비용. 시스템의 경제성을 판단하는 결정적 지표다.

### 3. TPC 테스트 환경 구성 (ASCII)
```text
    [ Remote Terminal Emulators ] ──▶ [ Network ] ──▶ [ Database Server ]
           (사용자 요청 모방)                                │
                                                     ┌──────┴──────┐
                                                     │ Disk Storage │
                                                     └──────────────┘
    * 요청 수신 -> 데이터 조회/갱신 -> 로그 기록 -> 응답 전송
    * 이 모든 과정이 ACID 특성을 만족하며 완료되어야 성능으로 인정됨.
```

- **📢 섹션 요약 비유**: 단순히 '계산'만 잘하는 것이 아니라, 창고에서 물건을 가져오고 장부에 기록하는 전 과정을 얼마나 안정적으로 수행하느냐를 봅니다.

## Ⅲ. TPC 벤치마크의 특징 및 차별점

| 특징 | 상세 내용 | 기대 효과 |
|:---|:---|:---|
| **Full System Test** | CPU뿐만 아니라 OS, DBMS, Storage 전체를 측정 | 실제 운영 성능과 높은 상관관계 |
| **ACID 준수** | 모든 트랜잭션의 원자성, 일관성, 고립성, 지속성 검증 | 신뢰할 수 없는 데이터 처리 배제 |
| **엄격한 감사** | TPC 공식 감사관의 승인을 거친 결과만 공표 가능 | 결과 조작 및 치팅(Cheating) 원천 차단 |

- **📢 섹션 요약 비유**: 정해진 룰 없이 자기들끼리 잘났다고 싸우던 벤더들에게 '공인 심판'이 등장하여 공정하게 등급을 매기는 것과 같습니다.

+++

## Ⅳ. 실무 적용 및 서버 도입 가이드 (Strategy)

### 1. 실무 시나리오: 뱅킹 시스템 주전산기 교체
- **상황**: 초당 수만 건의 계좌 이체 처리가 필요한 은행 코어 뱅킹 서버 도입.
- **판단**: 연산 위주의 SPEC 점수보다는 **TPC-C** 결과를 최우선으로 검토해야 한다. 특히 단순히 높은 tpmC 수치만 볼 것이 아니라, 우리가 예산 내에서 달성 가능한 **$/tpmC**가 가장 우수한 벤더를 선정하여 비용 대비 효율성을 극대화해야 한다.

### 2. 기술사적 인사이트: 데이터 일관성(Consistency)의 무게
- **인사이트**: TPC 벤치마크가 가혹한 이유는 고성능을 내면서도 **ACID**를 단 하나도 어겨서는 안 되기 때문이다. 로그 기록을 생략하거나 캐시를 부적절하게 쓰면 점수는 높아지겠지만, TPC 인증을 통과할 수 없다. 기술사는 성능 수치 이면에 숨겨진 '데이터 무결성 보장 메커니즘'의 견고함을 함께 분석하여, 장애 시 복구 능력까지 고려한 시스템 설계를 지향해야 한다.

- **📢 섹션 요약 비유**: 아무리 결제가 빠른 무인 계산기라도, 가끔 돈을 잘못 거슬러 준다면(데이터 오류) 절대 사용할 수 없는 것과 같습니다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **정확한 용량 산정(Capacity Planning)**: tpmC 수치를 기반으로 예상 동시 접속자 수에 필요한 서버 대수 산출.
- **시장 투명성 제고**: 공인된 지표를 통한 벤더 간의 건전한 기술 경쟁 유도.

### 2. 미래 전망
최근에는 머신러닝 연산을 포함한 **TPCx-AI**, IoT 데이터 처리를 재는 **TPCx-IoT** 등 4차 산업혁명의 흐름을 반영한 새로운 표준이 속속 등장하고 있다. 데이터가 자산이 되는 시대에, 그 자산을 얼마나 빠르고 정확하게 가공하느냐를 재는 TPC의 권위는 엔터프라이즈 컴퓨팅의 영원한 '품질 보증 수표'로 남을 것이다.

- **📢 섹션 요약 비유**: 건물을 짓기 전 설계 도면의 안전성을 꼼꼼히 따지듯, 시스템을 구축하기 전 TPC 점수를 따지는 것은 상업용 컴퓨팅의 기본 상식입니다.

### 📌 관련 개념 맵 (Knowledge Graph)
- **벤치마크 (Benchmark)**: TPC가 속한 성능 측정 도구의 상위 개념.
- **ACID 특성**: TPC 트랜잭션이 반드시 준수해야 하는 무결성 원칙.
- **OLTP / OLAP**: TPC-C와 TPC-H가 각각 측정하려는 작업 부하의 종류.
- **SPEC 벤치마크**: 연산 중심의 라이벌 성능 표준.

1. TPC 벤치마크는 컴퓨터들이 **'마트 계산대 직원처럼 물건 계산을 얼마나 꼼꼼하고 빠르게 하는지'** 재보는 시험이에요.
2. 그냥 계산만 하는 게 아니라 돈을 정확히 거슬러 주는지, 영수증은 잘 뽑는지 전 과정을 다 지켜보고 점수를 매기죠.
3. 이 점수가 높은 컴퓨터일수록 우리가 쓰는 은행 앱이나 쇼핑몰을 고장 없이 아주 빠르게 움직여줄 수 있답니다!
