# 135. IPC (Instructions Per Cycle)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 클럭 사이클 동안 CPU가 완료하는 평균 명령어 수, CPI의 역수
> 2. **가치**: 현대 CPU 아키텍처 효율성의 핵심 지표, IPC > 1이면 슈퍼스칼라
> 3. **융합**: 높은 IPC = 아키텍처 효율성, Apple/ARM은 낮은 클럭·높은 IPC 전략

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

IPC(Instructions Per Cycle)는 한 개의 클럭 사이클 동안 CPU가 실행 완료(Commit/Retire)하는 평균 명령어 수를 나타낸다. CPI(Cycles Per Instruction)의 역수로, **IPC = 1 / CPI** 관계가 성립한다.

### 💡 비유: 요리사가 1분에 몇 개 요리를 완성하나

IPC는 **요리사가 1분에 완성하는 요리 수**와 같다. 숙련된 요리사(IPC=2)는 1분에 2개를 완성하고, 초보(IPC=0.5)는 2분에 1개를 겨우 완성한다.

### 기본 공식

```
IPC = 총 완료된 명령어 수 / 총 클럭 사이클 수

IPC = 1 / CPI

┌────────────────────────────────────────────────────┐
│  IPC 범위 해석:                                                    │
│  ──────────────                                                    │
│  • IPC < 1: 파이프라인 비효율, 스톨 많음                          │
│  • IPC = 1: 이상적 파이프라인                                      │
│  • IPC > 1: 슈퍼스칼라, 다중 실행 유닛                             │
│  • IPC > 2: 고성능 슈퍼스칼라 (Apple, Intel 최신)                 │
└────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: CPU 선택 - 모바일**
- 요구: 배터리 효율
- 선택: 높은 IPC, 낮은 클럭 (ARM/Apple)
- 이유: 성능/전력 비율 최적

**시나리오 2: CPU 선택 - 게이밍**
- 요구: 싱글 스레드 성능
- 선택: 높은 클럭 + 높은 IPC
- 이유: 게임은 클럭과 IPC 모두 중요

**시나리오 3: 성능 최적화**
- 분석: IPC 프로파일링
- 병목: 캐시 미스, 분기 실패
- 해결: 캐시 최적화, 분기 감소

### 👶 어린이를 위한 3줄 비유 설명

**개념**: IPC는 "째깍" 한 번에 몇 가지 일을 끝내는지 숫자예요!

**원리**: 보통은 1개지만, 똑똑한 CPU는 "째깍" 한 번에 2~3개를 끝내요. 여러 손이 있어서 동시에 일하거든요!

**효과**: IPC가 높으면 전기를 덜 쓰고도 같은 일을 빨리 끝낼 수 있어요!
