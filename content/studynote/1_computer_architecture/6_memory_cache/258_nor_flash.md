# 258. NOR 플래시 (NOR Flash)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 셀이 병렬 연결된 고신뢰성 비휘발성 메모리
> 2. **가치**: 펌웨어, BIOS, XIP 실행에 최적
> 3. **융합**: 임베디드, IoT, 자동차 시스템

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

NOR 플래시는 **메모리 셀이 NOR 게이트처럼 병렬로 연결된 플래시 메모리**이다. 낮은 밀도와 높은 비용이지만, 빠른 랜덤 읽기와 XIP(eXecute In Place)가 가능하여 펌웨어 저장에 최적화되어 있다.

### 💡 비유: 개별 사물함

NOR 플래시는 **개별 사물함**과 같다. 각 사물함에 따로 접근할 수 있다. 100번 사물함을 열고 싶으면 바로 100번을 열면 된다. 다른 사물함을 거칠 필요가 없다. 하지만 공간을 많이 차지한다.

## Ⅱ. NOR 플래시 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                NOR 플래시 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NOR 동작 특성】                                                             │|
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 읽기: 바이트/워드 단위                                           │ │
│  │     • 랜덤 읽기: 50~100ns (매우 빠름)                                 │ │
│  │     • 순차 읽기: 50~100MB/s                                        │ │
│  │     • XIP: CPU가 직접 실행 가능                                     │ │
│  │                                                               │ │
│  │  2. 쓰기: 바이트/워드 단위 (느림)                                     │ │
│  │     • 프로그램 시간: 10~50μs                                         │ │
│  │     • 순차 쓰기: 1~10MB/s                                          │ │
│  │                                                               │ │
│  │  3. 소거: 블록/섹터 단위 (매우 느림)                                  │ │
│  │     • 소거 시간: 100ms~1s                                          │ │
│  │     • NAND 대비 10~100배 느림                                       │ │
│  │                                                               │ │
│  │  4. 수명:                                                          │ │
│  │     • P/E 사이클: 10,000~100,000회                                  │ │
│  │     • NAND와 비슷하거나 더 높음                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【XIP (eXecute In Place)】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  메모리에서 코드를 직접 실행 (복사 불필요)                             │ │
│  │                                                               │ │
│  │  일반적인 경우 (NAND):                                               │ │
│  │  NAND → DRAM → CPU 실행                                            │ │
│  │  (복사 단계 필요)                                                   │ │
│  │                                                               │ │
│  │  XIP (NOR):                                                          │ │
│  │  NOR → CPU 실행                                                     │ │
│  │  (복사 없이 직접 실행)                                               │ │
│  │                                                               │ │
│  │  장점:                                                            │ │
│  │  • 빠른 부팅 (복사 없음)                                             │ │
│  │  • RAM 절약                                                        │ │
│  │  • 실시간 응답                                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NOR vs NAND 비교】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           NOR                NAND                           │ │
│  │  ──────────     ───────────────     ────────────────                │ │
│  │  셀 연결        병렬                직렬                            │ │
│  │  밀도           낮음 (1/2~1/4)       높음                            │ │
│  │  비용/비트       높음                낮음                            │ │
│  │  랜덤 읽기       50~100ns            25~100μs (1000배 느림)          │ │
│  │  순차 읽기       50~100MB/s          50~100MB/s                     │ │
│  │  쓰기 속도       느림                빠름                            │ │
│  │  소거 속도       매우 느림            느림                            │ │
│  │  XIP           가능                불가능                          │ │
│  │  용도           펌웨어, BIOS          대용량 저장                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### NOR 플래시 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                NOR 플래시 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NOR 플래시 인터페이스】                                                      │
│  ──────────────────                                                  │
│  • Parallel NOR: 병렬 버스, 빠름, 많은 핀                               │ │
│  • SPI NOR: 직렬 버스, 느림, 적은 핀                                    │ │
│  • QSPI: Quad SPI, 4배 빠름                                           │ │
│  • OSPI: Octal SPI, 8배 빠름                                          │ │
│                                                                     │
│  【SPI NOR 플래시 확인】                                                        │
│  ──────────────────                                                  │
│  $ cat /proc/mtd                                                    │
│  dev:    size   erasesize  name                                     │
│  mtd0: 01000000 00010000 "BIOS"                                      │
│  → 16MB SPI NOR 플래시                                                │
│                                                                     │
│  【NOR 플래시 읽기/쓰기】                                                         │
│  ──────────────────                                                  │
│  # Linux MTD 도구                                                    │
│  $ flashcp firmware.bin /dev/mtd0                                   │
│  → NOR 플래시 펌웨어 쓰기                                              │
│                                                                     │
│  $ dd if=/dev/mtd0 of=backup.bin                                    │
│  → NOR 플래시 백업                                                    │
│                                                                     │
│  【NOR 플래시 최적화】                                                         │
│  ──────────────────                                                  │
│  1. XIP 활용: RAM 절약, 빠른 부팅                                     │
│  2. 압축: 펌웨어 압축으로 용량 절약                                     │
│  3. A/B 파티션: 무중단 업데이트                                        │
│  4. ECC: 오류 정정으로 신뢰성 향상                                      │
│  5. 쓰기 보호: 중요 영역 보호                                          │
│                                                                     │
│  【NOR vs NAND 선택 기준】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  용도                    추천             이유                        │ │
│  │  ──────────               ──────────         ──────────                  │ │
│  │  BIOS/UEFI               NOR             XIP, 빠른 부팅                 │ │
│  │  부트로더                 NOR             XIP, 신뢰성                   │ │
│  │  대용량 파일 저장           NAND            높은 밀도                   │ │
│  │  임베디드 펌웨어            NOR             XIP, 실시간                  │ │
│  │  스마트폰 저장             NAND            용량                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• NOR 플래시: 셀이 병렬 연결된 비휘발성 메모리
• 특징: 빠른 랜덤 읽기, XIP 가능, 높은 신뢰성
• 장점: 바이트 단위 읽기, 코드 직접 실행, 빠른 부팅
• 단점: 낮은 밀도, 높은 비용, 느린 쓰기/소거
• vs NAND: 10배 비쌈, 1000배 빠른 랜덤 읽기
• XIP: 메모리에서 직접 코드 실행 (복사 불필요)
• 인터페이스: Parallel, SPI, QSPI, OSPI
• 용도: BIOS/UEFI, 부트로더, 임베디드 펌웨어
```
