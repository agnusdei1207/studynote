# 252. SDRAM (Synchronous DRAM)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 클럭에 동기화된 DRAM
> 2. **가치**: 버스트 모드, 파이프라이닝으로 고속화
> 3. **융합**: DDR, 클럭 주파수, 메모리 컨트롤러

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SDRAM(Synchronous DRAM)은 **시스템 클럭에 동기화되어 동작하는 DRAM**이다. 비동기 DRAM과 달리 클럭 신호에 맞춰 명령을 처리하여 더 빠르고 예측 가능한 성능을 제공한다.

### 💡 비유: 오케스트라 연주

SDRAM은 **오케스트라 연주**와 같다. 비동기 DRAM은 각자 마음대로 연주한다. SDRAM은 지휘자(클럭)에 맞춰 모두 같이 연주한다. 클럭에 맞추면 예측 가능하고 파이프라이닝이 가능하다.

## Ⅱ. SDRAM 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SDRAM 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SDRAM 명령어】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어         설명              용도                            │ │
│  │  ──────────     ───────────────   ────────────────                │ │
│  │  ACTIVATE      행 활성화           특정 행 접근 준비                  │ │
│  │  READ          데이터 읽기          버스트 읽기                      │ │
│  │  WRITE         데이터 쓰기           버스트 쓰기                      │ │
│  │  PRECHARGE     행 비활성화           다른 행 접근 준비                  │ │
│  │  REFRESH       리프레시            전하 보충                        │ │
│  │  MODE REG SET  모드 설정            타이밍, 버스트 길이 설정             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스트 모드】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  연속된 주소를 한 번의 명령으로 읽기/쓰기                              │ │
│  │                                                               │ │
│  │  클럭  명령      데이터                                              │ │
│  │  ────  ────      ────                                               │ │
│  │   1    READ      -                                                  │ │
│  │   2    -         -      (CAS 지연)                                  │ │
│  │   3    -         -                                                  │ │
│  │   4    -         D0     ← 첫 데이터                                  │ │
│  │   5    -         D1     ← 버스트                                    │ │
│  │   6    -         D2     ← 버스트                                    │ │
│  │   7    -         D3     ← 버스트 (버스트 길이 = 4)                      │ │
│  │  → 한 번의 READ로 4개 데이터 인출                                     │ │
│  │                                                               │ │
│  │  버스트 길이: 1, 2, 4, 8, Full Page                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【뱅크 인터리빙】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  여러 뱅크를 병렬로 사용하여 대기 시간 숨김                              │ │
│  │                                                               │ │
│  │  클럭  뱅크0      뱅크1      뱅크2      뱅크3                          │ │
│  │  ────  ──────     ──────     ──────     ──────                      │ │
│  │   1    ACT        -          -          -                          │ │
│  │   2    -          ACT        -          -                          │ │
│  │   3    -          -          ACT        -                          │ │
│  │   4    -          -          -          ACT                        │ │
│  │   5    READ       -          -          -                          │ │
│  │   6    D0         READ       -          -                          │ │
│  │   7    D1         D0         READ       -                          │ │
│  │   8    D2         D1         D0         READ                       │ │
│  │  → 뱅크 4개로 파이프라이닝, 대역폭 4배                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  • 효율: 버스트 활용률                                                  │
```

## Ⅳ. 실무 적용

### SDRAM 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SDRAM 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SDRAM 타이밍 확인】                                                      │

│  $ sudo dmidecode --type memory | grep -E "Speed|Configured"        │
│  Speed: 3200 MT/s                                                    │
│  Configured Memory Speed: 3200 MT/s                                  │
│                                                                     │
│  【버스트 모드 활용】                                                        │
│  ──────────────────                                                  │
│  // 순차 접근: 버스트 모드 활용                                         │
│  for (i = 0; i < n; i++) {                                           │
│      sum += a[i];    // 버스트로 연속 인출                              │
│  }                                                                  │
│                                                                     │
│  // 랜덤 접근: 버스트 모드 비효율                                        │
│  for (i = 0; i < n; i++) {                                           │
│      sum += a[index[i]];  // 버스트 활용 어려움                         │
│  }                                                                  │
│                                                                     │
│  【SDRAM 성능 측정】                                                         │
│  ──────────────────                                                  │
│  $ bandwidth-latency -a 1GB -t 10                                   │
│  → 메모리 대역폭 및 지연 측정                                            │
│                                                                     │
│  【SDRAM 최적화 팁】                                                         │
│  ──────────────────                                                  │
│  1. 순차 접근: 버스트 모드 활용                                          │

│  3. 멀티 채널: 듀얼/쿼드 채널 활용                                         │
│  4. XMP/EXPO: 고성능 타이밍 적용                                         │
│  5. NUMA 인식: 로컬 메모리 우선                                          │
│                                                                     │
│  【SDRAM 대역폭 계산】                                                        │
│  ──────────────────                                                  │
│  DDR4-3200 듀얼 채널:                                                 │
│  대역폭 = 3200 MT/s × 8 bytes × 2 채널 = 51.2 GB/s                    │
│  실제 효율 ~70%: ~35 GB/s                                             │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SDRAM: 시스템 클럭에 동기화된 DRAM
• 특징: 예측 가능한 타이밍, 파이프라이닝 가능
• 명령어: ACTIVATE, READ, WRITE, PRECHARGE, REFRESH
• 버스트 모드: 연속 주소 한 번에 인출
• 뱅크 인터리빙: 여러 뱅크 병렬 사용
• 장점: 높은 대역폭, 효율적 파이프라이닝
• 기반: DDR, DDR2, DDR3, DDR4, DDR5의 기초
• 발전: SDR → DDR → DDR2 → DDR3 → DDR4 → DDR5
```
