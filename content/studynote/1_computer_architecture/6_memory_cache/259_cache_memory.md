# 259. 캐시 메모리 (Cache Memory)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU와 주 메모리 사이의 고속 버퍼 메모리
> 2. **가치**: 메모리 접근 지연을 숨기고 평균 접근 시간 단축
> 3. **융합**: L1/L2/L3 계층, 적중률, 교체 정책

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

캐시 메모리(Cache Memory)는 **CPU와 주 메모리(DRAM) 사이에 위치한 고속 메모리**로, 자주 사용하는 데이터를 저장하여 평균 메모리 접근 시간을 단축한다. SRAM 기반으로 빠르지만 비싸고 용량이 작다.

### 💡 비유: 책상 위

캐시 메모리는 **책상 위**와 같다. 자주 보는 책은 책상 위에 둔다. 필요할 때 바로 꺼내 볼 수 있다. 책상이 가득 차면 덜 쓰는 책을 책장(주 메모리)으로 보낸다. 책상이 클수록 많은 책을 가까이 둘 수 있다.

## Ⅱ. 캐시 메모리 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                캐시 메모리 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 성능 지표】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 적중률 (Hit Rate):                                            │ │
│  │     Hit Rate = Hit / (Hit + Miss)                                │ │
│  │     일반적: 90~99%                                                │ │
│  │                                                               │ │
│  │  2. 미스율 (Miss Rate):                                           │ │
│  │     Miss Rate = 1 - Hit Rate                                     │ │
│  │     일반적: 1~10%                                                 │ │
│  │                                                               │ │
│  │  3. 평균 메모리 접근 시간 (AMAT):                                    │ │
│  │     AMAT = Hit Time + Miss Rate × Miss Penalty                   │ │
│  │     예: 1ns + 0.05 × 100ns = 6ns                                 │ │
│  │                                                               │ │
│  │  4. 캐시 효율:                                                      │ │
│  │     효율 = Hit Rate × 100%                                        │ │
│  │     95% 적중률 → 95% 효율                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 구조】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  캐시 라인 (Cache Line/Block):                                      │ │
│  │  ┌───────────────────────────────────────────────────────────┐ │ │
│  │  │  태그 (Tag)  │  인덱스 (Index)  │  오프셋 (Offset)  │ 데이터    │ │ │
│  │  │   주소 상위    │   캐시 인덱스     │  라인 내 위치    │ 64B     │ │ │
│  │  └───────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │  캐시 구조:                                                          │ │
│  │  ┌───────────────────────────────────────────────────────────┐ │ │
│  │  │  인덱스 │  태그  │  V  │  D  │  데이터 (64B)               │ │ │
│  │  │  0     │  ...   │  1  │  0  │  ...                        │ │ │
│  │  │  1     │  ...   │  1  │  1  │  ...                        │ │ │
│  │  │  ...   │  ...   │  0  │  -  │  -                          │ │ │
│  │  │  N-1   │  ...   │  1  │  0  │  ...                        │ │ │
│  │  └───────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │  • V (Valid): 유효 비트 - 데이터 유효 여부                           │ │
│  │  • D (Dirty): 더티 비트 - 수정 여부                                  │ │
│  │  • Tag: 주소 상위 비트 - 캐시 라인 식별                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【현대 CPU 캐시 구성】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU             L1           L2           L3                    │ │
│  │  ─────────────   ──────────   ──────────   ──────────             │ │
│  │  Intel i9        64KB/코어   512KB/코어   32MB (공유)             │ │
│  │  AMD Ryzen 9     64KB/코어   512KB/코어   64MB (공유)             │ │
│  │  Apple M2        192KB/코어  16MB (공유)   -                      │ │
│  │  ARM Cortex-A78  64KB/코어   256KB/코어   4MB (공유)              │ │
│  │  → 코어당 L1/L2, 전체 공유 L3                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  cache-references: 1,000,000,000                                    │

│  【캐시 최적화 팁】                                                             │|
│  ──────────────────                                                  │

│  3. 순차 접근: 배열 순차 순회                                           │

│  5. 루프 최적화: 타일링, 언롤링                                          │
│  6. 분기 최소화: 예측 가능한 제어 흐름                                    │
│  7. 간접 접근 회피: 포인터 체이싱 감소                                    │

│  │  최적화 전            최적화 후            향상율                    │ │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 캐시 메모리: CPU와 주 메모리 사이의 고속 버퍼
• 기술: SRAM 기반, 빠름, 비쌈, 작음
• 계층: L1 (32~64KB) → L2 (256KB~2MB) → L3 (8~64MB)
• 적중률: 90~99% (높을수록 좋음)
• AMAT: Hit Time + Miss Rate × Miss Penalty
• 구조: 태그 + 인덱스 + 오프셋 + 데이터
• 메타데이터: Valid, Dirty 비트
• 성능: 캐시 적중률이 시스템 성능 결정
• 최적화: 지역성, 정렬, 타일링, 프리패칭
```
