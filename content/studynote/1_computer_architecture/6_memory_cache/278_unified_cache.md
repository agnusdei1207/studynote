# 278. 통합 캐시 (Unified Cache)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어와 데이터를 하나의 캐시에 저장
> 2. **가치**: 공간 효율, 유연한 할당
> 3. **융합**: Von Neumann, L2/L3, 트레이드오프

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

통합 캐시(Unified Cache)는 **명령어와 데이터를 하나의 캐시에 함께 저장하는 구조**다. Von Neumann 아키텍처 기반이며, 공간 효율이 좋고 I/D 비율이 유연하다.

### 💡 비유: 통합 책장

통합 캐시는 **통합 책장**과 같다. 교과서와 참고서를 같은 책장에 둔다. 교과서가 많이 필요하면 더 많은 칸을 쓰고, 참고서가 많이 필요하면 참고서를 더 둔다. 유연하게 공간 활용.

## Ⅱ. 통합 캐시 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                통합 캐시 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Split vs Unified 비교】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           Split Cache          Unified Cache              │ │
│  │  ──────          ──────────            ──────────                  │ │
│  │  구조            I-Cache + D-Cache      단일 캐시                   │ │
│  │  병렬 접근        가능                   제한                        │ │
│  │  공간 효율        낮음                   높음                        │ │
│  │  I/D 비율        고정                   유연                        │ │
│  │  복잡도           높음                   낮음                        │ │
│  │  구조적 위험      없음                   가능                        │ │
│  │  사용 계층       L1                    L2/L3                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【통합 캐시 장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 공간 효율: 전체 공간 활용                                          │ │
│  │  • 유연한 할당: I/D 비율 자동 조절                                     │ │
│  │  • 단순한 구조: 하나의 캐시만 관리                                      │ │
│  │  • 균형: 워크로드에 맞게 자동 조절                                      │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 구조적 위험: 동시 I/D 접근 불가                                      │ │
│  │  • 충돌: I/D 간 캐시 경쟁                                             │ │
│  │  • 대역폭: 단일 포트로 제한                                            │ │
│  │  • 파이프라인: 스톨 가능                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/D 비율 유연성】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  워크로드          I 사용률    D 사용률    Unified 장점               │ │
│  │  ──────────          ──────      ──────      ──────────               │ │
│  │  코드 중심 (컴파일)    80%         20%         I-Cache 공간 더 사용     │ │
│  │  데이터 중심 (DB)      20%         80%         D-Cache 공간 더 사용     │ │
│  │  균형 (일반)           50%         50%         공평하게 사용            │ │
│  │                                                               │ │
│  │  Split Cache: I=50%, D=50% 고정                                     │ │
│  │  Unified: 워크로드에 따라 자동 조절                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【포트 수와 대역폭】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단일 포트 Unified:                                                   │ │
│  │  • 동시에 하나만 접근 가능                                            │ │
│  │  • I/D 충돌 시 대기                                                  │ │
│  │  • 구현 단순                                                        │ │
│  │                                                               │ │
│  │  듀얼 포트 Unified:                                                    │ │
│  │  • 동시 2개 접근 가능                                                │ │
│  │  • I/D 동시 접근 가능                                               │ │
│  │  • 비용 증가                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 통합 캐시 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                통합 캐시 × 타 과목 융합                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • L2/L3: Unified Cache 표준                                        │
│  • Von Neumann: 통합 메모리 모델                                       │
│  • 멀티코어: L3 Unified로 코어 간 공유                                   │
│  • 면적: Split보다 작음                                                │
│                                                                     │
│  【운영체제】                                                          │
│  • 페이지 캐시: Unified (파일 + 코드)                                   │
│  • 버퍼 캐시: Unified                                                │
│  • 메모리: Unified 메모리 공간                                         │
│  • 공유 메모리: Unified 개념                                           │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 버퍼 풀: Unified (인덱스 + 데이터)                                    │
│  • 캐시: Unified                                                    │
│  • 공간 효율: 유연한 할당                                               │
│  • 워크로드: 다양한 I/D 비율                                            │
│                                                                     │
│  【임베디드】                                                          │
│  • MCU: Unified Cache 많음                                          │
│  • 저비용: 단일 캐시                                                  │
│  • 단순함: 설계 복잡도 낮음                                             │
│  • 절충: 성능 vs 비용                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  현대 CPU 계층별 캐시 타입:                                                      │ │
│  │  ──────────────────────                                              │ │
│  │  CPU             L1            L2            L3                      │ │
│  │  ─────────────   ──────────    ──────────    ──────────                │ │
│  │  Intel i9        Split         Unified       Unified                  │ │
│  │  AMD Zen 3       Split         Unified       Unified                  │ │
│  │  Apple M2        Split         Unified       Unified                  │ │
│  │  ARM A78         Split         Unified       Unified                  │ │
│  │  → 모두 L1=Split, L2/L3=Unified                                     │ │
│  │  → 계층마다 적합한 타입 사용                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  │  → 워크로드에 따라 다름                                               │ │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 통합 캐시: 명령어와 데이터를 하나의 캐시에 저장
• 구조: Von Neumann 아키텍처 기반
• 장점: 공간 효율, 유연한 I/D 비율, 단순한 구조
• 단점: 구조적 위험, I/D 충돌, 대역폭 제한
• 사용처: L2/L3 캐시 표준
• Split vs Unified: 병렬성 vs 공간 효율
• 현대 CPU: L1 Split + L2/L3 Unified
• I/D 비율: 워크로드에 따라 자동 조절
• 포트: 단일/듀얼 포트 선택 가능
• 트레이드오프: 성능 vs 비용 vs 면적
```
