# 247. 시간적 지역성 (Temporal Locality)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 최근 접근한 데이터를 가까운 시간에 다시 접근
> 2. **가치**: 캐시, 레지스터 할당의 핵심 원리
> 3. **융합**: 루프 최적화, 핫스팟 분석

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

시간적 지역성(Temporal Locality)은 **최근에 접근한 메모리 위치가 가까운 시간 내에 다시 접근될 확률이 높은 현상**을 말한다. 이를 활용하면 자주 사용하는 데이터를 빠른 메모리에 유지하여 성능을 향상시킬 수 있다.

### 💡 비유: 즐겨찾기

시간적 지역성은 **즐겨찾기**와 같다. 최근에 방문한 웹사이트는 다시 방문할 확률이 높다. 그래서 브라우저는 최근 방문한 사이트를 빠르게 접근할 수 있게 해둔다.

### 시간적 지역성 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                시간적 지역성 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시간적 지역성 개념】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시간 t:   t0    t1    t2    t3    t4    t5    t6    t7        │ │
│  │          ────  ────  ────  ────  ────  ────  ────  ────        │ │
│  │  접근:     A     B     A     A     C     B     A     B          │ │
│  │            ↑           ↑     ↑                 ↑         ↑      │ │
│  │            └───────────┴─────┴─────────────────┴─────────┘      │ │
│  │                      A의 재접근 (시간적 지역성)                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시간적 지역성이 높은 패턴】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 루프 변수                                                     │ │
│  │     for (i = 0; i < n; i++) {   // i가 매 턴마다 사용              │ │
│  │         sum += a[i];                                             │ │
│  │     }                                                           │ │
│  │                                                               │ │
│  │  2. 누적 변수                                                     │ │
│  │     sum = 0;                    // sum이 매 턴마다 갱신/사용        │ │
│  │     for (...) sum += ...;                                        │ │
│  │                                                               │ │
│  │  3. 스택 변수                                                     │ │
│  │     지역 변수가 함수 실행 중 반복 사용                               │ │
│  │                                                               │ │
│  │  4. 카운터                                                       │ │
│  │     count++;                    // 카운터가 빈번하게 갱신            │ │
│  │                                                               │ │
│  │  5. 포인터 역참조                                                  │ │
│  │     p->field = ...;             // 같은 포인터 반복 사용            │ │
│  │     p->other = ...;                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시간적 지역성 활용】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  레지스터 할당:                                                    │ │
│  │  • 자주 사용하는 변수를 레지스터에 유지                                │ │
│  │  • 메모리 접근 회수 → 성능 향상                                      │ │
│  │                                                               │ │
│  │  캐시 유지:                                                        │ │
│  │  • 자주 사용하는 데이터를 캐시에 유지                                 │ │
│  │  • LRU 교체 정책이 시간적 지역성 활용                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 시간적 지역성 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                시간적 지역성 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시간적 지역성 척도】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  재사용 거리 (Reuse Distance):                                     │ │
│  │  • 같은 데이터에 대한 연속 접근 사이의 고유 접근 수                     │ │
│  │  • 재사용 거리가 작을수록 시간적 지역성 높음                           │ │
│  │                                                               │ │
│  │  예시:                                                            │ │
│  │  접근: A B C A D A B A                                            │ │
│  │        └─1─┘ └3─┘ └─2─┘                                          │ │
│  │        거리=2 거리=3 거리=1                                         │ │
│  │  → 평균 재사용 거리 = (2+3+1)/3 ≈ 2                                │ │
│  │                                                               │ │
│  │  캐시 크기 > 재사용 거리 → 적중                                      │ │
│  │  캐시 크기 < 재사용 거리 → 미스                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시간적 지역성 vs 공간적 지역성】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           시간적 지역성        공간적 지역성                   │ │
│  │  ──────────     ────────────────    ────────────────            │ │
│  │  대상           같은 주소            인접 주소                      │ │
│  │  패턴           반복 접근            순차 접근                      │ │
│  │  예시           루프 변수            배열 순회                      │ │
│  │  활용           레지스터, 캐시        캐시 라인, 프리패칭             │ │
│  │  → 두 지역성은 독립적이나 보완적                                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시간적 지역성 최적화】                                                    │
│  ──────────────────                                                  │
│  1. 레지스터 할당: 컴파일러가 자주 사용 변수를 레지스터에                     │
│  2. 루프 불변 코드 이동: 루프 밖으로 이동                                   │
│  3. 스칼라 교체: 배열 요소를 스칼라 변수로                                   │
│  4. 핫스팟 분석: 자주 실행 코드 식별                                        │
│  5. 프로파일 가이드 최적화: 실행 빈도 기반 최적화                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 시간적 지역성 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                시간적 지역성 분석                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핫스팟 분석】                                                            │
│  ──────────────────                                                  │
│  $ perf record -e cycles ./prog                                    │
│  $ perf report                                                      │
│  → 자주 실행되는 코드(핫스팟) 식별                                        │
│                                                                     │
│  【캐시 재사율 분석】                                                        │
│  ──────────────────                                                  │
│  $ perf stat -e cache-references,cache-misses ./prog               │
│  → 캐시 적중률 = 시간적 지역성 활용도                                      │
│                                                                     │
│  【시간적 지역성 높이는 코드】                                                │
│  ──────────────────                                                  │
│  // Before: 낮은 시간적 지역성                                          │
│  for (i = 0; i < n; i++) {                                          │
│      sum += arr[i];        // sum은 시간적 지역성 높음                  │
│      temp = arr[i] * 2;    // temp도 높음                           │
│      process(&temp);        // 하지만 temp를 매번 재계산                  │
│  }                                                                  │
│                                                                     │
│  // After: 높은 시간적 지역성 (레지스터 할당)                               │
│  register int sum_reg = sum;                                        │
│  for (i = 0; i < n; i++) {                                          │
│      sum_reg += arr[i];                                             │
│  }                                                                  │
│  sum = sum_reg;                                                      │
│                                                                     │
│  【시간적 지역성 저해 요인】                                                  │
│  ──────────────────                                                  │
│  1. 큰 워킹셋: 캐시 크기 초과                                            │
│  2. 스트라이드 접근: 큰 간격의 접근                                        │
│  3. 랜덤 접근: 패턴 없는 접근                                             │
│  4. 포인터 체이싱: 연결 리스트 순회                                        │
│  5. 캐시 스래싱: 과도한 캐시 교체                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 시간적 지역성: 최근 접근 데이터의 재접근 경향
• 척도: 재사용 거리 (작을수록 높은 지역성)
• 높은 패턴: 루프 변수, 누적 변수, 스택 변수
• 활용: 레지스터 할당, 캐시 유지, LRU 교체
• 최적화: 핫스팟 분석, 레지스터 할당, 불변 코드 이동
• 저해 요인: 큰 워킹셋, 랜덤 접근, 캐시 스래싱
• 측정: perf, cachegrind
```
