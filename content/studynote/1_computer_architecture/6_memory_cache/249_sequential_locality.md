# 249. 순차적 지역성 (Sequential Locality)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터가 순차적으로 접근되는 패턴
> 2. **가치**: 명령어 인출, 스트림 처리 최적화 기반
> 3. **융합**: 프리패칭, 파이프라이닝과 연동

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

순차적 지역성(Sequential Locality)은 **메모리가 주소 순서대로 연속적으로 접근되는 패턴**을 말한다. 특히 명령어 인출과 스트림 데이터 처리에서 두드러지며, 프리패칭의 기반이 된다.

### 💡 비유: 노래 재생

순차적 지역성은 **노래 재생**과 같다. 노래는 처음부터 끝까지 순서대로 재생된다. 다음 부분을 미리 버퍼에 로드하면 끊김 없이 재생할 수 있다.

### 순차적 지역성 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                순차적 지역성 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【순차적 지역성 개념】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  주소:     100   104   108   10C   110   114   118   11C        │ │
│  │           ────  ────  ────  ────  ────  ────  ────  ────        │ │
│  │  접근:      ▼     ▼     ▼     ▼     ▼     ▼     ▼     ▼          │ │
│  │           t0    t1    t2    t3    t4    t5    t6    t7           │ │
│  │           └─────────────────────────────────────────┘            │ │
│  │                     순차적 접근 패턴                               │ │
│  │                                                               │ │
│  │  예측: t0에 접근 → t1~t7도 접근할 것으로 예측                       │ │
│  │  프리패칭: 미리 108~11C를 캐시에 로드                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순차적 지역성이 높은 패턴】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 명령어 인 fetch                                                │ │
│  │     PC → PC+4 → PC+8 → PC+12 → ...                               │ │
│  │     → 분기 없이 순차적 실행                                         │ │
│  │     → I-Cache 적중률 높음                                          │ │
│  │                                                               │ │
│  │  2. 배열 순회                                                      │ │
│  │     for (i = 0; i < n; i++) sum += a[i];                          │ │
│  │     → a[0], a[1], a[2], ... 순차 접근                              │ │
│  │                                                               │ │
│  │  3. 스트림 데이터                                                    │ │
│  │     오디오/비디오 스트림, 네트워크 패킷                               │ │
│  │     → 순차적 데이터 처리                                            │ │
│  │                                                               │ │
│  │  4. 파일 순차 읽기                                                   │ │
│  │     read(fd, buffer, size);                                       │ │
│  │     → 파일 오프셋이 순차적 증가                                       │ │
│  │                                                               │ │
│  │  5. 로그 파일 쓰기                                                   │ │
│  │     append-only 로그                                               │ │
│  │     → 항상 파일 끝에 추가                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순차적 지역성 vs 공간적 지역성】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           공간적 지역성         순차적 지역성                   │ │
│  │  ──────────     ────────────────     ────────────────            │ │
│  │  방향성          무방향                단방향                        │ │
│  │  예측성          낮음                  높음                         │ │
│  │  프리패칭        제한적                매우 효과적                    │ │
│  │  예시           구조체 필드             명령어, 스트림                 │ │
│  │  → 순차적 지역성은 프리패칭에 더 유리                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 순차적 지역성 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                순차적 지역성 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프리패칭과 순차적 지역성】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  하드웨어 프리패칭:                                                │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │  캐시                                                         │ │
│  │  │  ┌────┬────┬────┬────┬────┬────┬────┬────┐                │   │ │
│  │  │  │ L0 │ L1 │ L2 │ L3 │ L4 │ L5 │ L6 │ L7 │ ← 미리 로드      │   │ │
│  │  │  └────┴────┴────┴────┴────┴────┴────┴────┘                │   │ │
│  │  │                          ↑                                 │   │ │
│  │  │                       프리패처                              │   │ │
│  │  │                          ↑                                 │   │ │
│  │  │  접근: L0 → L1 → L2 → L3 → ...                             │   │ │
│  │  │  → 접근 패턴 감지 → 다음 라인들 미리 인출                       │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  │  스트림 프리패처:                                                   │ │
│  │  • 연속된 캐시 미스 감지                                            │ │
│  │  • 스트림 버퍼에 다음 N개 라인 로드                                   │ │
│  │  • 순차적 지역성 활용                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순차적 지역성 최적화】                                                    │
│  ──────────────────                                                  │
│  1. 명령어 캐시: 분기 예측 + 프리패칭                                       │
│  2. 데이터 캐시: 스트림 프리패칭                                           │
│  3. 메모리 컨트롤러: 버스트 모드                                          │
│  4. SSD/HDD: 순차 읽기 최적화                                           │
│  5. 네트워크: TCP 윈도우, 버퍼링                                          │
│                                                                     │
│  【순차적 지역성 저해 요인】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  요인                영향                  예시                   │ │
│  │  ────────────────    ────────────────      ──────────            │ │
│  │  분기 명령어           명령어 순차성 파괴        if, for           │ │
│  │  랜덤 접근             순차성 없음            포인터 체이싱          │ │
│  │  역순 순회             역방향 순차성           for (i=n; i>0; i--) │ │
│  │  인터리브 접근          여러 스트림 혼재        a[i] + b[i]         │ │
│  │  → 순차적 지역성이 높으면 프리패칭 효과적                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 순차적 지역성 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                순차적 지역성 분석                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프리패칭 효과 측정】                                                      │
│  ──────────────────                                                  │
│  $ perf stat -e L1-dcache-prefetches,L1-dcache-loads ./prog        │
│  → 프리패칭 횟수 vs 실제 로드 횟수                                       │
│                                                                     │
│  【순차적 지역성 활용 코드】                                                  │
│  ──────────────────                                                  │
│  // 명시적 프리패칭                                                    │
│  for (i = 0; i < n; i++) {                                          │
│      __builtin_prefetch(&a[i+8]);  // 8개 앞서 프리패칭                  │
│      sum += a[i];                                                    │
│  }                                                                  │
│                                                                     │
│  // 파일 순차 읽기                                                      │
│  // 커널이 read-ahead 자동 수행                                        │
│  while ((n = read(fd, buf, BUFSIZE)) > 0) {                         │
│      process(buf, n);                                                │
│  }                                                                  │
│                                                                     │
│  【순차적 지역성 높이는 팁】                                                  │
│  ──────────────────                                                  │
│  1. 배열 순차 접근: 인덱스 순서 유지                                       │
│  2. 분기 최소화: 순차적 제어 흐름                                          │
│  3. 큰 블록 I/O: 작은 I/O 대신 큰 블록                                    │
│  4. 정렬된 데이터: 정렬된 상태 유지                                         │
│  5. 버퍼링: 스트림 데이터 버퍼링                                           │
│                                                                     │
│  【SSD 순차 vs 랜덤 성능】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SSD 타입     순차 읽기      랜덤 읽기      차이                      │ │
│  │  ──────────    ──────────     ──────────     ─────                  │ │
│  │  SATA SSD     550 MB/s       40 MB/s       14배                     │ │
│  │  NVMe SSD     3500 MB/s      60 MB/s       58배                     │ │
│  │  → 순차적 지역성 활용이 중요                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 순차적 지역성: 메모리가 주소 순서대로 접근되는 패턴
• 높은 패턴: 명령어 인출, 배열 순회, 스트림 데이터
• 활용: 프리패칭, 버스트 모드, read-ahead
• 장점: 높은 예측성, 효과적인 프리패칭
• 저해 요인: 분기, 랜덤 접근, 역순 순회
• 최적화: 순차 접근, 큰 블록 I/O, 버퍼링
• 성능 차이: 순차 vs 랜덤 = 최대 50배
```
