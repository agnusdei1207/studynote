# 141. 지연 시간 (Latency)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 요청이 발생하고 응답이 완료될 때까지의 경과 시간, 밀리초(ms)~마이크로초(μs) 단위
> 2. **가치**: 실시간 시스템·게임·HFT에서 핵심 지표, 낮을수록 좋음
> 3. **융합**: 대역폭과 트레이드오프, 캐시·CDN·엣지 컴퓨팅으로 최소화

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

지연 시간(Latency)은 데이터가 소스에서 목적지까지 이동하는 데 걸리는 시간, 또는 요청이 시작되어 응답이 완료될 때까지의 총 경과 시간을 의미한다. 네트워크, 스토리지, CPU, 메모리 등 모든 컴퓨팅 계층에서 발생한다.

### 💡 비유: 식당 주문부터 서빙까지

지연 시간은 **주문서를 내고 음식이 나올 때까지 기다리는 시간**이다. 주방(CPU)이 빨라도 서빝(네트워크)이 느리면 지연이 발생한다. 고객은 이 총 시간을 체감한다.

### 지연 시간 구성요소

```
┌─────────────────────────────────────────────────────────────────────┐
│                지연 시간 구성                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Total Latency = Processing + Transmission + Propagation + Queuing  │
│                                                                     │
│  【구성요소】                                                       │
│  ──────────────                                                     │
│  • Processing Latency: 처리 시간 (CPU, 애플리케이션)               │
│  • Transmission Latency: 전송 시간 (패킷 크기 / 대역폭)            │
│  • Propagation Latency: 전파 시간 (거리 / 빛의 속도)               │
│  • Queuing Latency: 대기 시간 (버퍼, 라우터 큐)                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시: 웹 요청 지연 시간                                       │ │
│  │  ────────────────────────                                      │ │
│  │  DNS 조회: 20ms                                                │ │
│  │  TCP 연결: 30ms (3-way handshake)                              │ │
│  │  TLS 핸드셰이크: 50ms                                          │ │
│  │  HTTP 요청 전송: 5ms                                           │ │
│  │  서버 처리: 100ms                                              │ │
│  │  응답 전송: 5ms                                                │ │
│  │  ─────────────────                                             │ │
│  │  총 지연: 210ms                                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 계층별 지연 시간

```
┌─────────────────────────────────────────────────────────────────────┐
│                계층별 지연 시간 비교                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  계층                   지연 시간         비고                     │
│  ─────────────────────  ─────────────    ───────────────           │
│  CPU 레지스터           0.3 ns          1 클럭 (3GHz)              │
│  L1 캐시                1 ns            3~4 클럭                   │
│  L2 캐시                4 ns            10~15 클럭                 │
│  L3 캐시                12 ns           30~40 클럭                 │
│  메인 메모리 (DRAM)     100 ns          300 클럭                   │
│  SSD (NVMe)             25 μs           75,000 클럭                │
│  HDD                    10 ms           30,000,000 클럭            │
│  네트워크 (같은 DC)     0.5 ms          데이터센터 내              │
│  네트워크 (인터넷)      50~200 ms       지역/거리                  │
│  위성 통신              600 ms          지구 궤도 왕복             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  지연 시간 스케일:                                             │ │
│  │  ──────────────────                                            │ │
│  │  ns (10⁻⁹) → μs (10⁻⁶) → ms (10⁻³) → s                       │ │
│  │                                                               │ │
│  │  CPU vs 메모리: 100× 차이                                      │ │
│  │  메모리 vs SSD: 250× 차이                                      │ │
│  │  SSD vs HDD: 400× 차이                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 메모리 계층 지연

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 계층 지연 시간                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 미스 지연】                                                 │
│  ──────────────────                                                 │
│                                                                     │
│        ┌────────────┐                                              │
│        │    CPU     │                                              │
│        └─────┬──────┘                                              │
│              │ 1 ns                                                │
│        ┌─────┴──────┐                                              │
│        │  L1 Cache  │ 32KB                                         │
│        └─────┬──────┘                                              │
│              │ 4 ns (L1 miss)                                       │
│        ┌─────┴──────┐                                              │
│        │  L2 Cache  │ 256KB~1MB                                    │
│        └─────┬──────┘                                              │
│              │ 12 ns (L2 miss)                                      │
│        ┌─────┴──────┐                                              │
│        │  L3 Cache  │ 8~64MB                                       │
│        └─────┬──────┘                                              │
│              │ 40 ns (L3 miss)                                      │
│        ┌─────┴──────┐                                              │
│        │   DRAM     │ 16~128GB                                     │
│        └────────────┘                                              │
│              │ 100+ ns                                              │
│                                                                     │
│  【메모리 대역폭 vs 지연】                                          │
│  ──────────────────────────                                         │
│  • DDR4-3200: 25.6 GB/s, 70ns 지연                                 │
│  • DDR5-5600: 44.8 GB/s, 60ns 지연                                 │
│  • HBM3: 819 GB/s, 30ns 지연                                       │
│                                                                     │
│  → 대역폭↑ ≠ 지연↓ (독립적 특성)                                   │
```

### 네트워크 지연

```
┌─────────────────────────────────────────────────────────────────────┐
│                네트워크 지연 시간                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【지연 구성】                                                      │
│  ──────────────                                                     │
│                                                                     │
│  Network Latency = Propagation + Transmission + Processing         │
│                                                                     │
│  • Propagation: 거리 / 빛의 속도                                    │
│    예: 서울~뉴욕 11,000km → 37ms (광속)                            │
│                                                                     │
│  • Transmission: 패킷 크기 / 대역폭                                 │
│    예: 1500B / 1Gbps = 12μs                                        │
│                                                                     │
│  • Processing: 라우터, 스위치 처리                                  │
│    예: 홉당 0.5~2ms                                                 │
│                                                                     │
│  【실제 지연 예시】                                                 │
│  ──────────────────                                                 │
│                                                                     │
│  구간                    지연                                       │
│  ────────────────────  ─────────                                   │
│  같은 랙 (Same Rack)    0.1~0.5 ms                                 │
│  같은 DC (Same DC)      0.5~2 ms                                   │
│  같은 리전              1~10 ms                                    │
│  다른 리전 (국가 간)    50~200 ms                                  │
│  위성 인터넷            500~700 ms                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CDN/엣지 활용:                                                │ │
│  │  ──────────────                                                │ │
│  │  • 오리진 서버 (미국): 150ms                                   │ │
│  │  • CDN 엣지 (한국): 10ms                                       │ │
│  │  → 15배 지연 감소                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 지연 시간 vs 대역폭

| 특성 | 지연 시간 | 대역폭 |
|:---|:---|:---|
| **정의** | 요청~응답 시간 | 단위 시간당 데이터량 |
| **단위** | ms, μs, ns | Gbps, GB/s |
| **영향** | 실시간성, 반응성 | 처리량, 대용량 전송 |
| **개선** | 거리 단축, 캐시 | 채널 증설, 압축 |
| **비유** | 파이프 길이 | 파이프 굵기 |

```
┌─────────────────────────────────────────────────────────────────────┐
│                지연 시간 vs 대역폭 트레이드오프                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파이프라인 비유】                                                │
│  ──────────────────                                                 │
│                                                                     │
│  지연: ─────────────────────────────────────► (파이프 길이)         │
│        시작                                    끝                   │
│                                                                     │
│  대역폭: ═══════════════════════════════════► (파이프 굵기)         │
│          물의 양/시간                                               │
│                                                                     │
│  【시나리오】                                                       │
│  ──────────────                                                     │
│  • 게임: 낮은 지연 중요 (클릭 → 반응)                               │
│  • 동영상 스트리밍: 높은 대역폭 중요 (버퍼링)                       │
│  • HFT (초단타매매): 극저지연 필수 (μs 단위)                       │
│  • 백업: 높은 대역폭 중요 (대용량 전송)                             │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  • 지터 (Jitter): 지연 변동                                         │
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 게임 서버**
- 요구: 30ms 이하 지연
- 해결: 리전 서버, UDP, 예측
- 결과: 플레이어 만족도 향상

**시나리오 2: HFT (고빈도매매)**
- 요구: 1ms 이하 지연
- 해결: 코로케이션, FPGA, 직접 시장 접근
- 결과: 수익성 직결

**시나리오 3: CDN 선택**
- 비교: A사 50ms vs B사 10ms
- 의사결정: B사 선택 (20% 비용↑)
- 결과: 사용자 체험 개선

**시나리오 4: 데이터베이스 쿼리**
- 문제: 평균 500ms 지연
- 분석: 인덱스 미사용, 풀 스캔
- 해결: 인덱스 추가, 쿼리 최적화
- 결과: 5ms 지연 달성

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 최적화 | 지연 감소 | 효과 |
|:---|:---:|:---|
| **CDN** | 100ms→10ms | 전환율 7%↑ |
| **SSD** | 10ms→0.1ms | IOPS 100× |
| **캐시** | 50ms→1ms | 응답 50× |
| **엣지** | 50ms→5ms | UX 개선 |

### 핵심 지표

```
• P50 Latency: 50% 요청의 지연
• P95 Latency: 95% 요청의 지연
• P99 Latency: 99% 요청의 지연
• Tail Latency: 상위 1%의 지연 (중요!)
```

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 지연 시간은 요청하고 나서 대답 받을 때까지 기다리는 시간이에요!
