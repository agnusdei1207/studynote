# 154. TPC 벤치마크 (TPC Benchmark)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트랜잭션 처리 및 데이터베이스 성능 측정의 업계 표준, TPC-C/TPC-H/TPC-DS 제공
> 2. **가치**: 실제 OLTP/OLAP 워크로드 기반 성능 비교, tpmC/QphH@Size 단위
> 3. **융합**: 스토리지·DBMS·서버 종합 성능 평가, 가격 대비 성능($/tpmC) 지표

## Ⅰ. 개요 (Context & Background)

### 개념 정의

TPC(Transaction Processing Performance Council)는 1988년 설립된 비영리 단체로, **트랜잭션 처리 시스템과 데이터베이스 성능**을 측정하는 표준 벤치마크를 개발·관리한다. OLTP(온라인 트랜잭션 처리)와 OLAP(온라인 분석 처리) 성능을 객관적으로 평가한다.

### 💡 비유: 은행 창구 성능 측정

TPC 벤치마크는 **은행 창구 업무 능력 평가**와 같다. "1시간에 몇 건의 계좌 이체를 처리할 수 있는가?", "복잡한 통계 조회를 얼마나 빨리 하는가?"를 표준화된 방식으로 측정한다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### TPC-C (OLTP 벤치마크)

TPC-C는 **대표적인 OLTP 벤치마크**로, 도매상 주문 처리 시스템을 모델링한다. 5가지 트랜잭션 유형으로 구성되며, 복잡한 트랜잭션 혼합을 처리하는 능력을 측정한다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                TPC-C 워크로드 구성                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데이터베이스 스키마】                                             │
│  ────────────────────────                                           │
│                                                                     │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐          │
│  │  WAREHOUSE  │────<│   DISTRICT  │────<│   CUSTOMER  │          │
│  │  (창고)     │     │   (지점)    │     │   (고객)    │          │
│  └─────────────┘     └─────────────┘     └─────────────┘          │
│        │                   │                   │                   │
│        │                   │                   │                   │
│        ▼                   ▼                   ▼                   │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐          │
│  │    STOCK    │     │    ORDER    │────<│ ORDER-LINE  │          │
│  │   (재고)    │     │   (주문)    │     │  (주문항목) │          │
│  └─────────────┘     └─────────────┘     └─────────────┘          │
│                            │                                        │
│                            ▼                                        │
│                      ┌─────────────┐                               │
│                      │   NEW-ORDER │                               │
│                      │  (신규주문) │                               │
│                      └─────────────┘                               │
│                                                                     │
│  【트랜잭션 유형 (5종)】                                             │
│  ────────────────────────                                           │
│  트랜잭션         비율     설명               ACID 특성              │
│  ─────────────  ──────  ────────────────   ─────────────            │
│  New-Order       45%    신규 주문 생성      Update, Insert          │
│  Payment         43%    결제 처리           Update                  │
│  Order-Status     4%    주문 상태 조회      Read-only               │
│  Delivery         4%    배송 처리           Update                  │
│  Stock-Level      4%    재고 수준 조회      Read-only               │
│                                                                     │
│  【성능 지표】                                                       │
│  ──────────────                                                     │
│  • tpmC: 분당 처리된 New-Order 트랜잭션 수                           │
│  • $/tpmC: 성능당 가격 (낮을수록 좋음)                               │
│  • Availability Date: 시스템 가용 일자                               │
└─────────────────────────────────────────────────────────────────────┘
```

### TPC-C 트랜잭션 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                TPC-C 트랜잭션 상세                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【New-Order (신규 주문) - 45%】                                     │
│  ────────────────────────────                                       │
│  동작:                                                              │
│  1. 고객 정보 조회                                                  │
│  2. 새 주문 레코드 생성                                              │
│  3. 5~15개 주문 항목 추가                                           │
│  4. 재고 업데이트                                                    │
│  5. 트랜잭션 커밋                                                    │
│                                                                     │
│  특성:                                                              │
│  • 가장 빈번한 트랜잭션                                              │
│  • 쓰기 작업 집중                                                    │
│  • ACID 보장 필수                                                    │
│  • 응답 시간: 5초 이내 (90%)                                         │
│                                                                     │
│  【Payment (결제) - 43%】                                            │
│  ────────────────────────                                           │
│  동작:                                                              │
│  1. 고객 잔액 업데이트                                               │
│  2. 지점 수입 업데이트                                               │
│  3. 결제 이력 기록                                                   │
│                                                                     │
│  특성:                                                              │
│  • 간단하지만 빈도 높음                                              │
│  • 핫스팟(잔액) 경합 발생                                             │
│  • 응답 시간: 5초 이내 (90%)                                         │
│                                                                     │
│  【Order-Status (주문 상태) - 4%】                                   │
│  ────────────────────────────────                                   │
│  • 읽기 전용 트랜잭션                                                │
│  • 고객의 최근 주문 상태 조회                                         │
│  • 응답 시간: 5초 이내 (90%)                                         │
│                                                                     │
│  【Delivery (배송) - 4%】                                            │
│  ────────────────────────                                           │
│  • 배치 처리 가능                                                    │
│  • 10개 지역 주문 배송 처리                                          │
│  • 응답 시간: 5초 이내 (90%)                                         │
│                                                                     │
│  【Stock-Level (재고 수준) - 4%】                                    │
│  ────────────────────────────────                                   │
│  • 읽기 전용                                                         │
│  • 저재고 항목 카운트                                                │
│  • 응답 시간: 5초 이내 (90%)                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ACID 요구사항:                                                 │ │
│  │  ────────────────                                               │ │
│  │  • Atomicity: 전체 성공 또는 전체 실패                           │ │
│  │  • Consistency: 무결성 제약 준수                                 │ │
│  │  • Isolation: 트랜잭션 간 간섭 없음                              │ │
│  │  • Durability: 장애 후에도 결과 보존                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │

## Ⅲ. 융합 비교 및 다각도 분석

### TPC 벤치마크 비교

| 구분 | TPC-C | TPC-H | TPC-DS |
|:---|:---|:---|:---|
| **유형** | OLTP | OLAP | Data Warehouse |
| **트랜잭션/쿼리** | 5종 | 22개 | 99개 |
| **특성** | 쓰기 중심 | 읽기 중심 | 읽기 중심 |
| **ACID** | 필수 | 선택적 | 선택적 |
| **단위** | tpmC | QphH@Size | QphDS@Size |
| **스케일** | Warehouse 수 | SF (GB) | SF (GB) |
