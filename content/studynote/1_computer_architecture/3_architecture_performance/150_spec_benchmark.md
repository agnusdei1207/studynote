# 150. SPEC 벤치마크 (SPEC Benchmark)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 성능 측정의 업계 표준, 정수·부동소수점·그래픽 벤치마크 제공
> 2. **가치**: 공정한 성능 비교, 구매 의사결정, 최적화 검증의 객관적 근거
> 3. **융합**: 실제 워크로드 기반, 컴파일러·OS·하드웨어 종합 성능 반영

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SPEC(Standard Performance Evaluation Corporation)는 1988년 설립된 비영리 단체로, 컴퓨터 시스템의 성능을 측정하는 **표준 벤치마크**를 개발·관리한다. SPEC CPU는 CPU 성능 측정의 가장 신뢰받는 벤치마크다.

### 💡 비유: 자동차 공인 연비

SPEC 점수는 **정부 공인 연비**와 같다. 제조사가 임의로 측정한 것이 아니라, 표준화된 방법으로 제3자가 측정하여 신뢰할 수 있다.

### SPEC 점수 계산

```
┌─────────────────────────────────────────────────────────────────────┐
│                SPEC 점수 계산                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 원리】                                                      │
│  ──────────────                                                     │
│  • 참조 시스템(Reference Machine) 대비 비율                         │
│  • 각 테스트별 점수 → 기하 평균                                     │
│  • 높을수록 좋음                                                    │
│                                                                     │
│  【SPECspeed 계산】                                                 │
│  ──────────────────                                                 │
│  Speed_i = 참조 시간 / 측정 시간                                   │
│  SPECspeed = (Π Speed_i)^(1/n)  (기하 평균)                        │
│                                                                     │
│  【SPECrate 계산】                                                  │
│  ──────────────────                                                 │
│  Rate_i = (초당 작업 수) × (작업 크기)                              │
│  SPECrate = (Π Rate_i)^(1/n)  (기하 평균)                          │
│                                                                     │
│  【기하 평균 사용 이유】                                            │
│  ────────────────────────                                           │
│  • 비율(배수)의 평균에 적합                                         │
│  • 이상치 영향 적음                                                 │
│  • 산술 평균보다 대표성                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시:                                                         │ │
│  │  ────────                                                      │ │
│  │  테스트        참조 시간   측정 시간   Speed                   │ │
│  │  ──────────    ─────────   ─────────   ──────                  │ │
│  │  perlbench     1000s       50s         20×                     │ │
│  │  gcc           2000s       100s        20×                     │ │
│  │  mcf           3000s       300s        10×                     │ │
│  │                                                               │ │
│  │  기하평균 = (20 × 20 × 10)^(1/3) = 15.87                       │ │
│  │  산술평균 = (20 + 20 + 10) / 3 = 16.67                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### SPEC vs 타 벤치마크

| 벤치마크 | 특징 | 장점 | 단점 |
|:---|:---|:---|:---|
| **SPEC CPU** | 실제 워크로드 | 공정, 신뢰 | 비용, 복잡 |
| **Geekbench** | 간단, 빠름 | 무료, 접근성 | 대표성 논란 |
| **PassMark** | 종합 | 다양한 지표 | 표준 아님 |
| **Cinebench** | 3D 렌더링 | 시각적 | 단일 워크로드 |

```
┌─────────────────────────────────────────────────────────────────────┐
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 서버 구매 비교**
- Intel Xeon: SPECint 70, $3,000
- AMD EPYC: SPECint 77, $2,500
- 분석: AMD 10% 성능↑, 17% 비용↓
- 선택: AMD EPYC

**시나리오 2: 컴파일러 최적화 검증**
- 이전: SPECint 60
- 컴파일러 업그레이드 후: SPECint 66
- 향상: 10%

**시나리오 3: 클럭 vs 코어 트레이드오프**
- 16코어 3GHz: SPECrate 800
- 32코어 2GHz: SPECrate 1200
- 결정: 32코어 (50% 향상)
