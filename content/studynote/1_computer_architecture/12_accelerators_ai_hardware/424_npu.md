# NPU (Neural Processing Unit)

> 1. **본질**: NPU (Neural Processing Unit)는 딥러닝 인공 신경망 (Artificial Neural Network)의 핵심 연산인 **대규모 행렬 곱셈 (Matrix Multiplication)과 활성화 함수 (Activation Function)**를 하드웨어적으로 가속하기 위해 설계된 도메인 특화 프로세서 (DSA)다.
> 2. **가치**: GPU 대비 불필요한 그래픽 연산 로직을 제거하고 신경망 구조에 최적화된 데이터 흐름 (Dataflow)을 구축함으로써, 압도적인 에너지 효율 (TOPS/W)과 낮은 지연 시간을 제공하여 모바일 및 에지 (Edge) 장치의 온디바이스 AI를 가능케 한다.
> 3. **융합**: 자율주행차의 실시간 객체 인식, 스마트폰의 사진 보정 및 음성 비서, 데이터센터의 대규모 추론 가속기 등 현대 AI 인프라의 물리적 중추 역할을 수행하고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **概念**: NPU는 뇌의 신경망 구조를 모방한 하드웨어다. 수만 개의 데이터를 한꺼번에 연산하는 시스톨릭 어레이 (Systolic Array)나 벡터 엔진을 탑재하여, 딥러닝 모델의 수많은 파라미터를 최소한의 전력으로 빠르게 처리한다. 범용성보다는 AI 연산의 효율에 모든 것을 건 아키텍처다.

- **💡 비유**: NPU는 **"인공지능 연산만 전문으로 하는 전용 계산기"**와 같다. 다용도로 쓰는 컴퓨터(CPU)나 그림도 잘 그리는 도구(GPU)보다, 오직 딥러닝이라는 수학 문제만 풀도록 설계된 특수 기계라서 그 분야만큼은 누구보다 빠르고 전기도 적게 쓰는 것과 같다.

- **등장 배경**:
  1. **AI 연산 부하의 폭증**: 딥러닝 모델이 거대해지면서 범용 CPU나 GPU만으로는 전력 소모와 발열을 감당하기 어려워졌다.
  2. **온디바이스 AI (On-device AI) 요구**: 클라우드를 거치지 않고 스마트폰이나 IoT 기기 자체에서 개인 정보를 안전하고 빠르게 처리해야 할 필요성이 커졌다.
  3. **데이터 중심 아키텍처로의 전환**: 폰 노이만 구조의 고질적 문제인 메모리 병목을 해결하기 위해, 연산 유닛 옆에 데이터를 밀착시키는 새로운 구조가 필요했다.

- **📢 섹션 요약 비유**: 복잡한 제어 로직을 과감히 덜어내고 딥러닝의 핵심인 '행렬 연산'이라는 단일 목적에 모든 자원을 집중한 고효율 연산 엔진입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **MAC 어레이 (MAC Array)** | 행렬 곱셈 및 누산 수행 | 수천 개의 곱셈-더하기 유닛이 병렬 동작 | Systolic Array | 수천 개의 컨베이어 벨트가 맞물린 자동화 공정 |
| **로컬 버퍼 (On-chip Buffer)** | 데이터 재사용성 극대화 | 외부 메모리 접근 없이 가중치와 특징 맵 저장 | SRAM (Static RAM) | 요리사 바로 옆에 준비된 신선한 재료 바구니 |
| **활성화 유닛 (Activation Unit)** | 비선형 함수 적용 | ReLU, Sigmoid 등을 하드웨어로 구현 | Look-up Table (LUT) | 특정 조건에 따라 합격/불합격을 가리는 판독기 |
| **데이터 압축 엔진 (Compression)** | 메모리 대역폭 절약 | 0이 많은 텐서(Sparse Tensor)를 압축 처리 | Sparsity Acceleration | 부피를 줄여서 한꺼번에 많이 옮기는 압축 팩 |
| **DMA 컨트롤러 (DMA Controller)** | 호스트와 NPU 간 데이터 고속 이동 | CPU 개입 없이 대량의 텐서 데이터 전송 | PCIe / AXI Bus | 원자재를 창고에서 공장으로 바로 쏘아주는 전용 관로 |

### NPU의 핵심: 데이터 흐름 중심 아키텍처 (Dataflow Architecture)

NPU는 명령어 하나하나를 가져오는 방식이 아니라, 데이터가 하드웨어 연산 유닛 사이를 물 흐르듯 지나가며 결과가 나오는 방식을 사용한다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               NPU 내부의 시스톨릭 어레이 연산 구조                │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   입력 데이터 (In) ──▶ [MAC] ──▶ [MAC] ──▶ [MAC] ──▶ 결과 (Out) │
 │                         ▲         ▲         ▲                 │
 │   가중치 (Weight) ─────┼─────────┼─────────┼─────             │
 │                         │         │         │                 │
 │   [특징]                                                       │
 │   1. Data Reuse: 한 번 읽은 가중치를 여러 연산에 반복 사용      │
 │   2. Local Interconnect: 옆 칸의 연산기에 결과를 직접 전달     │
 │   3. Zero Idle: 데이터가 계속 흐르며 연산 유닛 가동률 100% 유지 │
 │                                                               │
 │   구조: [Input Buffer] ─▶ [MAC Array] ─▶ [Activation] ─▶ [Out] │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 상기 구조는 NPU의 대표적 방식인 시스톨릭 어레이 (Systolic Array)를 간략화한 것이다. 데이터가 마치 심장 박동(Systole)처럼 일정한 박자에 맞춰 연산기 사이를 흐른다. 한 번 메모리에서 읽어온 '가중치'나 '입력 데이터'는 버려지지 않고 옆이나 아래에 있는 연산기로 전달되어 재사용된다. 이는 폰 노이만 구조의 고질적 병목인 '메모리 읽기/쓰기' 횟수를 획기적으로 줄여준다. 연산이 끝난 값은 즉시 하드웨어 활성화 유닛을 거쳐 비선형 변환된 후 다음 레이어로 전달되거나 메모리에 저장된다. 이 모든 과정이 하드웨어 파이프라인으로 굳어져 있어 전력 소모가 극도로 낮다.

### 심층 동작 원리: 양자화 (Quantization) 가속

NPU는 높은 정밀도의 부동소수점 (FP32) 대신 낮은 정밀도의 정수 (INT8, INT4) 연산에 특화되어 있다.
1. **가중치 압축**: 32비트 데이터를 8비트로 줄여 메모리 대역폭과 저장 공간을 4배 절약한다.
2. **저정밀도 MAC**: 정수 연산기는 부동소수점 연산기보다 회로가 훨씬 작고 단순하여, 같은 면적에 더 많은 연산기를 집적할 수 있다.
3. **정확도 유지**: 양자화 기법을 통해 정확도 손실을 최소화하면서 성능은 5~10배 향상시킨다.

- **📢 섹션 요약 비유**: 복잡한 소수점 계산 대신 대략적인 정수 계산으로 속도를 높이되, 결과에는 지장이 없도록 영리하게 타협한 실용주의적 설계입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: CPU vs GPU vs NPU (AI 연산 관점)

| 비교 항목 | CPU (Central Processing) | GPU (Graphics Processing) | NPU (Neural Processing) |
|:---|:---|:---|:---|
| **설계 철학** | 제어 중심 (Universal) | 병렬 처리 중심 (Throughput) | 데이터 흐름 중심 (Efficiency) |
| **연산 효율** | 낮음 | 보통 | 매우 높음 |
| **유연성** | 매우 높음 (모든 소프트웨어) | 높음 (다양한 병렬 작업) | 낮음 (AI 모델 한정) |
| **전력 소모** | 높음 | 매우 높음 | 매우 낮음 |
| **주 사용처** | 일반 프로그램 실행 | AI 학습, 렌더링 | AI 추론, 모바일 가속 |

### 비교 2: NPU의 유형별 분류

| 구분 | 모바일 NPU (Apple ANE) | 서버용 NPU (Google TPU) | 에지 NPU (Hailo) |
|:---|:---|:---|:---|
| **목표 지표** | 저전력, 실시간성 | 초고성능, 대규모 확장성 | 초소형, 저비용 |
| **연산 단위** | 수 TOPS (Tera Ops) | 수백~수천 TFLOPS | 수십 TOPS |
| **특징** | 배터리 수명 중시 | 고대역폭 메모리 (HBM) 탑재 | 팬리스 (Fanless) 구동 |

### 과목 융합 관점

- **컴퓨터 구조 (CA)**: NPU는 메모리 계층 구조를 파괴하고 연산기 내부에 데이터를 가두는 'Data-centric' 설계의 정수다. 이는 향후 PIM (Processing-In-Memory) 기술로 이어지는 가교 역할을 한다.
- **인공지능 (AI)**: NPU의 성능은 하드웨어 자체보다 '컴파일러' 성능에 좌우된다. TensorFlow나 PyTorch 모델을 NPU 최적화 명령어로 변환하는 TVM, ONNX와 같은 소프트웨어 스택과의 융합이 필수적이다.

- **📢 섹션 요약 비유**: 만능 요리사(CPU)와 대형 식당(GPU)을 지나, 오직 특정 메뉴만 초고속으로 찍어내는 자동화 전용 공장(NPU)으로 진화한 모습입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **스마트폰 카메라의 야간 모드 촬영**: 어두운 곳에서 여러 장의 사진을 찍어 합성하는 복잡한 AI 연산을 수행할 때, GPU를 쓰면 배터리가 광속으로 소모되고 발열이 생긴다. NPU를 사용하여 0.1초 만에 저전력으로 노이즈를 제거함으로써, 사용자에게 쾌적한 촬영 경험을 제공한다.
2. **자율주행 ADAS (Advanced Driver Assistance Systems)**: 전방 카메라의 영상을 분석하여 사람과 장애물을 0.01초 내에 판단해야 한다. 클라우드 지연 시간을 견딜 수 없으므로, 차량 내 탑재된 고성능 NPU를 통해 초저지연 온디바이스 추론을 수행한다.
3. **스마트 팩토리의 비전 검사**: 불량품을 판별하기 위해 24시간 내내 가동되는 공정 라인에서, 저전력 에지 NPU를 도입하여 서버 운영 비용과 대역폭 비용을 획기적으로 절감한다.

### 도입 체크리스트
- **모델 지원 범위**: 해당 NPU가 CNN, RNN뿐만 아니라 최신 Transformer (ViT) 모델을 하드웨어적으로 지원하는가?
- **양자화 툴 체인**: 기존 FP32 모델을 정확도 저하 없이 INT8로 변환해주는 성숙한 SDK가 제공되는가?
- **전력 효율 (TOPS/W)**: 한정된 전력 예산 내에서 목표하는 처리 성능을 달성할 수 있는가?

### 안티패턴
- **오버핏 하드웨어 (Over-fitted Hardware)**: 특정 AI 모델(예: 3년 전 CNN)에 너무 최적화되어, 최신 모델(예: 최신 LLM)이 나왔을 때 아예 구동조차 못 하거나 오히려 느려지는 설계는 피해야 한다. 하드웨어 레벨의 최소한의 유연성(Programmability) 확보가 필수적이다.

- **📢 섹션 요약 비유**: 아무리 빠른 기계라도 특정 부품만 만들 수 있다면 금방 쓸모가 없어지듯, 변화하는 AI 트렌드를 수용할 수 있는 영리한 전문성이 필요합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과 (표)

| 구분 | 도입 효과 | 비즈니스 가치 |
|:---|:---|:---|
| **전력 효율** | GPU 대비 동일 연산당 전력 10배 이상 절감 | 배터리 수명 연장 및 친환경 인프라 구축 |
| **개인정보 보호** | 데이터를 서버로 보내지 않고 로컬에서 처리 | 프라이버시 강화 및 보안 사고 예방 |
| **비용 절감** | 고가의 서버 인프라 없이 독립적 구동 가능 | 소형 가전 및 저가형 기기로 AI 대중화 |

### 미래 전망
NPU는 장기적으로 **'뉴로모픽 (Neuromorphic) 컴퓨팅'**으로 나아갈 것이다. 현재의 행렬 연산을 넘어, 실제 인간의 뇌처럼 스파이크 신호를 주고받는 SNN (Spiking Neural Network) 방식이 도입되어 전력 소모를 한 차원 더 낮출 것이다. 또한 모든 반도체 (AP, SSD, RAM) 내부에 아주 작은 NPU가 내장되어, 장치 스스로 성능을 최적화하는 지능형 하드웨어 시대가 올 것으로 예측된다.

### 참고 표준
- **ONNX (Open Neural Network Exchange)**: 다양한 AI 프레임워크와 NPU 간의 호환 표준
- **MLPerf Tiny**: 에지/모바일 NPU의 성능 측정 표준
- **IEEE P2844**: 인공지능 프로세서의 성능 및 전력 효율 평가 표준

- **📢 섹션 요약 비유**: 인간의 지능을 칩 속에 그대로 옮겨 담으려는 시도는, 결국 모든 사물이 스스로 생각하고 반응하는 진정한 '지능의 대중화'를 이끌 것입니다.
