# IPU (Intelligence Processing Unit)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPU(지능형 처리 장치)는 영국 스타트업 그래프코어(Graphcore)가 개발한 AI 가속기로, GPU의 단순한 물량 공세(SIMD)를 거부하고 뇌신경 세포의 얽힘과 같은 **거대 그래프(Graph) 구조의 병렬 연산을 처리하기 위해 태어난 극단적인 MIMD(다중 명령어 다중 데이터) 아키텍처**다.
> 2. **가치**: 값비싸고 느린 외부 HBM 메모리를 완전히 없애버리고, 대신 칩 내부에 수백 메가바이트의 **거대한 초고속 SRAM(In-Processor Memory)을 꽉 채워 넣어 메모리 병목(Memory Wall)을 물리적으로 파괴**했다.
> 3. **융합**: 고정된 행렬 곱셈만 잘하는 GPU/TPU와 달리, 희소성(Sparsity)이 높고 예측이 불가능한 복잡한 미래형 AI 알고리즘과 **BSP(Bulk Synchronous Parallel) 동기화 모델을 융합**하여 독보적인 추론 속도를 뽐낸다.

---

## Ⅰ. IPU의 개요

### 1. 정의
- 기계 학습(Machine Learning) 모델을 거대한 노드와 간선의 '그래프(Graph)'로 정의하고, 수천 개의 독립적인 프로세서 코어(IPU Core)가 외부 DRAM 개입 없이 칩 내부의 거대 SRAM에서 데이터를 공유하며 비동기적(MIMD)으로 연산하는 인공지능 전용 프로세서다.

### 2. 등장 배경: 'GPU는 바보 같고 TPU는 융통성이 없다'
- 딥러닝 초창기에는 단순한 행렬 곱셈(CNN)이 대세여서 GPU(SIMD)와 TPU(시스톨릭 어레이)로 충분했다. 하지만 AI가 발전하며 그래프 신경망(GNN)이나 비정형 데이터(희소 행렬)가 쏟아졌다. GPU는 1만 개의 코어가 똑같은 행동만 해야 해서 이런 복잡한 뇌신경망 구조 앞에서는 버벅거렸고 파이프라인이 깨졌다. 그래프코어는 "진짜 뇌처럼 각 코어가 독립적으로 생각(MIMD)하고, 메모리는 칩 밖으로 나가지 말고 칩 안에 다 품자!"라며 IPU라는 돌연변이를 내놓았다.

### 3. 💡 비유: '1만 명의 독립된 천재 작가들'
- GPU가 **'1명의 감독 통제하에 똑같은 춤을 추는 1만 명의 아이돌(SIMD)'**이라면, IPU는 **'각자 자기 방에 거대한 백과사전(SRAM)을 꽂아놓고 각기 다른 소설을 쓰는 1만 명의 천재 작가들(MIMD)'**이다.
- 아이돌(GPU)은 대열(행렬)이 무너지면 아무것도 못 하지만, 작가들(IPU)은 대열 따위 없이 서로 복잡하게 메신저(그래프 통신)를 주고받으며 가장 기괴하고 창의적인 문제(차세대 AI)를 풀어낸다.

- **📢 섹션 요약 비유**: 군대식 제식 훈련(GPU)의 한계를 깨닫고, 각자의 개성과 창의력을 100% 발휘하면서도 유기적으로 소통하는 실리콘 밸리의 거대한 애자일(Agile) 스타트업 조직도와 같습니다.

## Ⅱ. 메커니즘 및 하드웨어 아키텍처 (Deep Dive)

IPU의 심장은 외부 메모리의 완전한 배제와 BSP 동기화 모델이다.

### 1. In-Processor Memory (인 프로세서 메모리)
- 일반적인 AI 칩은 연산기 옆에 작은 캐시를 두고 외부 HBM(램)에서 데이터를 미친 듯이 퍼온다(Memory Wall 발생).
- **IPU의 결단**: 외부 RAM 슬롯을 아예 없애버렸다! 대신 칩 면적의 절반을 희생해 **무려 900MB에 달하는 어마어마한 용량의 SRAM**을 칩 내부에 직접 박았다. 칩 내부에서 초당 65TB/s라는 상상 초월의 속도로 데이터가 움직이기 때문에 메모리 병목 자체가 존재할 수 없다.

### 2. BSP (Bulk Synchronous Parallel) 병렬 통신 모델 (ASCII)
MIMD 구조에서 수천 개의 뇌가 지 멋대로 움직이면 꼬이기 때문에, IPU는 '신호등' 체계를 도입했다.
```text
   [ Compute Phase (계산) ]      [ Sync (대기) ]     [ Exchange Phase (통신) ]
  
  코어 1 ─(자기 일 미친 듯이 함)───▶ | 멈춤(Sync) | ◀── (서로 결과물 교환) ──▶ (다음 계산)
  코어 2 ──(자기 일 함)────────────▶ | 멈춤(Sync) | ◀── (서로 결과물 교환) ──▶ (다음 계산)
  코어 3 ─(제일 빨리 끝남)─(쉼)────▶ | 멈춤(Sync) | ◀── (서로 결과물 교환) ──▶ (다음 계산)
  
  * 핵심: 각자 일하다가 일이 다 끝나면(Sync), 그때 비로소 서로 칩 내부 네트워크를 통해 데이터를 교환한다. 레이스 컨디션(충돌) 버그가 수학적으로 원천 차단된다.
```

### 3. 심층 분석: 희소성(Sparsity) 처리의 절대 강자
- GPU는 행렬에 '0'이 많아도 0을 곱하며 낭비한다. 구조적 가지치기(2:4)를 해야만 겨우 빨라진다.
- IPU는 애초에 각 코어가 독립적(MIMD)이므로, 데이터에 0이 있으면 그냥 그 연산을 건너뛰고 바로 다음 데이터를 처리해 버린다. 즉, **비정형 희소성(Unstructured Sparsity)** 환경에서 GPU를 10배 이상 압살하는 극강의 효율을 뿜어낸다. 

- **📢 섹션 요약 비유**: 0이 섞인 서류를 받았을 때, "규정대로 0도 다 읽어라"라고 강요하는 공무원(GPU)과, 0은 1초 만에 휴지통에 던져버리고 진짜 알맹이 서류만 처리하는 센스 있는 전문가(IPU)의 일머리 차이입니다.

## Ⅲ. GPU vs IPU의 철학적 대결

| 비교 항목 | GPU (NVIDIA A100 등) | IPU (Graphcore Bow 등) |
|:---|:---:|:---:|
| **병렬 처리 방식** | **SIMD / SIMT (군무)** | **MIMD (각자도생)** |
| **메인 메모리** | 외부 HBM (수십 GB, 지연 있음) | **내부 SRAM (수백 MB, 지연 0)** |
| **강점 영역** | 조밀한 행렬 곱(Dense Matrix), LLM | 희소 데이터(Sparse), GNN, MCMC |
| **스레드 동기화** | 프로그래머가 락(Lock) 걸다 피 토함 | **하드웨어가 BSP 모델로 깔끔히 맞춤** |
| **시장 상황** | **전 세계 AI 시장 독점 1위** | 혁신적이나 범용성 한계로 고전 중 |

- **📢 섹션 요약 비유**: 무조건 힘센 망치로 모든 못을 때려 박는 범용 공구(GPU)의 독주 앞에서, 특정 꼬인 나사(그래프 연산)를 풀 때는 세상에서 제일 완벽한 맞춤형 드라이버(IPU)의 눈물겨운 생존 경쟁입니다.

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 실무 시나리오: 자율주행 및 금융권 리스크 모델링
- **상황**: 금융권에서 몬테카를로 마르코프 체인(MCMC) 알고리즘을 돌려 주식 시장 리스크를 시뮬레이션해야 한다. 이 알고리즘은 행렬 곱셈이 아니라, 상태가 꼬리에 꼬리를 무는 '확률적 그래프 모델'이다.
- **판단**: GPU에 이 코드를 넣으면 성능이 10%도 안 나온다. GPU의 장기인 SIMD 연산 구조와 완전히 엇나가기 때문이다. 퀀트(Quant) 개발자는 팝아웃(Pop-out) 툴킷(Poplar)을 이용해 코드를 IPU 전용 그래프 C++로 컴파일한다. IPU의 수천 개 MIMD 코어들이 이 확률 그래프를 칩 내부의 거대한 SRAM 안에서 지연 없이 파고들면서, GPU 대비 10~20배 이상의 압도적인 추론(Inference) 속도업을 이뤄낸다. 도메인 맞춤형 하드웨어(DSA) 도입의 가장 완벽한 성공 사례다.

### 2. 기술사적 인사이트: 메모리 용량 부족의 치명적 한계
- **인사이트**: IPU의 "램(HBM)을 없애고 SRAM만 쓴다"는 철학은 양날의 검이었다. ChatGPT와 같은 LLM(거대 언어 모델)이 세상을 지배하자 문제가 터졌다. LLM 모델 사이즈가 수백 기가바이트(GB)에 달하는데, IPU의 칩 내부 SRAM은 고작 900MB에 불과했다. 
- GPU는 느려도 HBM 80GB가 있어서 모델을 담을 수 있었지만, IPU는 모델을 찢어서 수백 대의 칩에 나눠 담아야 했고 통신 오버헤드 때문에 장점이 다 죽어버렸다. 결국 "아키텍처가 아무리 혁신적이어도, 킬러 워크로드(LLM)의 메모리 용량 트렌드를 맞추지 못하면 도태된다"는 냉혹한 반도체 시장의 진리를 보여주었다.

- **📢 섹션 요약 비유**: 수영을 세상에서 제일 빨리하는 천재(IPU)를 키워놨더니, 갑자기 올림픽 종목이 100km 자전거 타기(초거대 LLM)로 바뀌어버려 체력(메모리 용량) 부족으로 예선 탈락해 버린 슬픈 타이밍의 비극입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **메모리 월(Memory Wall) 돌파의 극단적 해답**: 칩 바깥으로 나가는 물리적 선을 잘라버리고 모든 걸 칩 내부에서 끝낸다는 발상으로 전력 소모와 지연 시간을 이론적 최소치로 달성.
- **AI 연산 패러다임의 확장**: 딥러닝=행렬곱이라는 고정관념을 깨고, 차세대 AI인 뇌 모방 신경망이나 그래프 데이터 구조에 가장 최적화된 마이크로아키텍처의 청사진 제시.

### 2. 미래 전망
거대 자본의 논리(LLM의 거대화)에 밀려 그래프코어는 위기를 맞았고, 최근 일본 소프트뱅크(ARM 소유사)에 인수되었다. 그러나 IPU가 남긴 "거대 SRAM 탑재"와 "MIMD 기반의 희소성 가속" 철학은 결코 죽지 않았다. 현재 AI 씬에서 트랜스포머(Transformer) 모델을 경량화하려는 시도(희소 주의력, Sparse Attention 등)가 대세가 되면서, 연산의 효율성을 극단으로 쥐어짜는 IPU의 설계 사상은 미래의 뉴로모픽(Neuromorphic) 칩과 엣지(Edge) AI 생태계의 부활을 알리는 가장 강력한 기술적 떡잎으로 남을 것이다.

- **📢 섹션 요약 비유**: 비록 거인(GPU)과의 힘싸움(용량)에서는 졌을지언정, 그들이 남긴 세련된 검술(MIMD, 거대 SRAM)은 후대 AI 무술가들에게 전설적인 비급으로 전수되어 새로운 전장을 지배할 것입니다.

## 👶 어린이를 위한 3줄 비유 설명
1. IPU는 공장에서 똑같은 장난감만 수만 개 찍어내는 로봇(GPU)과 달리, **'각자 방에 앉아 자기만의 예술 작품을 만드는 1만 명의 장인들'**이에요.
2. 로봇은 레고 설명서가 조금만 꼬여도 고장 나지만, 장인들은 복잡한 퍼즐이나 꼬인 실타래(그래프)를 각자 머리를 굴려 엄청나게 빨리 풀어내죠.
3. 게다가 각자의 방에 모든 재료(거대 캐시)가 다 들어있어서 재료를 가지러 밖에 나갈 필요가 없으니 밥도 안 먹고 빛의 속도로 일한답니다!
