# MAC 연산 (Multiply-Accumulate)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: MAC (Multiply-Accumulate) 연산은 두 숫자를 곱한 뒤 그 결과를 누산기 (Accumulator)에 더하는 과정을 단일 하드웨어 명령어 또는 사이클로 처리하는 핵심 디지털 신호 처리 및 AI 연산 단위다.
> 2. **가치**: 딥러닝의 핵심인 행렬 곱셈 (Matrix Multiplication) 및 합성곱 (Convolution) 연산의 90% 이상을 차지하며, MAC 유닛의 개수와 처리 속도가 NPU (Neural Processing Unit)의 초당 연산 횟수 (TOPS, Tera Operations Per Second)를 결정한다.
> 3. **융합**: 고성능 MAC 연산을 위해 INT8, FP16, BF16과 같은 저정밀도 데이터 포맷 (Data Format)이 도입되었으며, 이는 메모리 대역폭 (Memory Bandwidth)과 직결되어 PIM (Processing-In-Memory) 아키텍처로 진화하는 촉매가 되었다.

## Ⅰ. 개요 (Context & Background)

- **개념**: MAC (Multiply-Accumulate) 연산은 수학적으로 $a \leftarrow a + (b \times c)$ 형태의 수식을 단일 하드웨어 사이클 내에 수행하는 하드웨어 유닛 및 연산 방식을 의미한다. 곱셈기 (Multiplier)와 덧셈기 (Adder), 그리고 결과를 저장하는 누산 레지스터 (Accumulator Register)가 긴밀하게 결합된 구조다.
- **💡 비유**: 마트의 계산대 점원과 같습니다. 물건의 개수(b)와 단가(c)를 곱하여 해당 품목의 가격을 계산(Multiply)하고, 이를 전체 장바구니 총액(a)에 계속해서 더해나가는(Accumulate) 과정을 한 번의 숨결에 처리하는 것과 같습니다.
- **등장 배경**:
  1. **폰 노이만 병목과 분리된 명령어의 한계**: 기존 범용 CPU (Central Processing Unit) 아키텍처에서는 $a = a + b \times c$를 수행하기 위해 곱셈 명령어를 실행하여 임시 레지스터에 저장하고, 다시 덧셈 명령어를 실행해야 했다. 이는 여러 사이클을 소모하고 레지스터 접근 빈도를 높여 성능 저하와 전력 소모를 유발했다.
  2. **DSP (Digital Signal Processor)의 요구**: FIR/IIR 필터, 퓨리에 변환 (FFT) 등 신호 처리 알고리즘은 본질적으로 다량의 내적 (Dot Product) 연산을 필요로 했으며, 이를 고속화하기 위해 하드웨어적으로 통합된 MAC 유닛이 DSP에 최초로 도입되었다.
  3. **AI 시대로의 패러다임 전환**: 인공신경망 (ANN, Artificial Neural Network)의 추론 및 학습 과정은 거대한 가중치 (Weight) 행렬과 입력 (Activation) 행렬의 곱셈으로 귀결된다. 이를 가속하기 위해 GPU 및 NPU 내부에 수만 개의 MAC 유닛을 집적하는 방향으로 발전했다.
- **📢 섹션 요약 비유**: 마치 컨베이어 벨트에서 물건을 조립하고 (곱셈) 바로 상자에 담아 (덧셈) 한 번에 포장하는 일체형 공정 라인과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 프로토콜/구조 | 비유 |
|:---|:---|:---|:---|:---|
| **곱셈기 (Multiplier)** | 두 입력 피연산자 (Operand) 곱셈 | 부분 곱 (Partial Product) 생성 및 덧셈 트리 (Wallace Tree) 압축 | 부스 알고리즘 (Booth's Algorithm) | 재료 가공 |
| **덧셈기 (Adder)** | 곱셈 결과와 누산기 값을 덧셈 | 캐리 전파를 최소화하는 고속 덧셈 | CLA (Carry-Lookahead Adder) | 재료 결합 |
| **누산기 (Accumulator)** | 이전 연산 결과를 저장 | 클럭 에지에 맞춰 결과를 래치 (Latch)하여 다음 사이클의 입력으로 피드백 | 폭이 넓은 레지스터 (예: 32-bit) | 완성품 창고 |
| **파이프라인 레지스터** | 동작 주파수 향상을 위한 단간 분리 | 곱셈과 덧셈 사이에 위치하여 크리티컬 패스 (Critical Path) 분할 | D 플립플롭 (Data Flip-Flop) | 공정 간 대기 구역 |
| **오버플로우 제어 로직** | 누산 시 발생할 수 있는 데이터 넘침 방지 | 최상위 비트 캐리 발생 시 클리핑 (Clipping) 또는 포화 (Saturation) 연산 수행 | Saturation Arithmetic | 창고 넘침 방지 센서 |

### Fused Multiply-Add (FMA) / MAC 유닛 아키텍처

현대의 MAC 유닛은 단순한 곱셈 후 덧셈을 넘어, 정밀도 손실을 최소화하는 FMA (Fused Multiply-Add) 구조로 설계된다. FMA는 곱셈 후 반올림을 수행하지 않고 전체 정밀도를 유지한 채 덧셈을 수행한 후 마지막에 한 번만 반올림을 수행한다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               MAC (FMA) 하드웨어 아키텍처 다이어그램                 │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   입력 b (N-bit)       입력 c (N-bit)                           │
 │        │                    │                                 │
 │        ▼                    ▼                                 │
 │     ┌──────────────────────────┐                              │
 │     │    Multiplier (곱셈기)     │ ── 부스 인코딩 & Wallace Tree │
 │     └──────────────────────────┘                              │
 │                  │  (2N-bit 곱셈 결과, 반올림 없음)                │
 │                  ▼                                            │
 │     ┌──────────────────────────┐                              │
 │     │       Adder (덧셈기)       │ ◀──────┐                     │
 │     └──────────────────────────┘        │                     │
 │                  │  (2N+1-bit 덧셈)       │                     │
 │                  ▼                      │                     │
 │     ┌──────────────────────────┐        │                     │
 │     │   Rounding & Saturation  │        │                     │
 │     └──────────────────────────┘        │ 피드백               │
 │                  │                      │                     │
 │                  ▼                      │                     │
 │     ┌──────────────────────────┐        │                     │
 │     │ Accumulator Register (a) │ ────────┘                     │
 │     └──────────────────────────┘                              │
 │                  │                                            │
 │                  ▼  최종 결과 a (N-bit 또는 2N-bit)                │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 입력 b와 c가 곱셈기에 인가되면, 하드웨어는 부분 곱을 생성하고 이를 압축하여 2N-bit의 결과를 도출한다. 전통적인 설계(단순 연쇄)에서는 이 시점에서 반올림이 발생하여 오차가 누적될 수 있지만, FMA 구조에서는 정밀도를 유지한 채 바로 덧셈기로 전달된다. 덧셈기는 누산기(Accumulator) 레지스터에 저장되어 있던 이전 값 'a'와 새로운 곱셈 결과를 더한다. 덧셈 후 포화 연산(Saturation)을 거쳐 최종적으로 누산기에 업데이트된다. 이 전체 과정이 강력한 파이프라이닝을 통해 매 클럭 사이클마다 1개의 MAC 결과를 쏟아내도록(Throughput=1) 설계된다.

### 심층 동작 원리 (행렬 곱셈 매핑)

1. **데이터 인출**: 피연산자 메모리 (예: SRAM)에서 가중치(Weight)와 입력 특징 맵(Input Feature Map) 데이터를 인출한다.
2. **배치/분배**: 데이터를 다수의 MAC 유닛에 분배한다 (예: 시스톨릭 어레이 구조).
3. **병렬 곱셈**: `w_i * x_i` 연산이 다수의 Multiplier에서 동시에 수행된다.
4. **누산/축적**: 생성된 부분 합들이 Adder Tree 또는 로컬 Accumulator를 통해 더해진다 (`sum += w_i * x_i`).
5. **활성화 및 출력**: 누산된 결과는 활성화 함수 (ReLU 등)를 거쳐 다음 레이어의 입력 또는 메모리로 반환된다.

```c
// MAC 연산을 설명하는 실무 수준의 C 코드 스니펫
void mac_operation(float *a, const float *b, const float *c, int n) {
    // 폰 노이만 구조에서의 스칼라 연산 (현대 프로세서는 SIMD/MAC 명령어 사용)
    float acc = *a; 
    for (int i = 0; i < n; i++) {
        // vfmadd231ps (AVX) 또는 fmla (ARM NEON) 명령어 1개로 컴파일 됨
        acc = acc + (b[i] * c[i]); // FMA (Fused Multiply-Add)
    }
    *a = acc;
}
```

- **📢 섹션 요약 비유**: 곱셈기(망치)와 덧셈기(못)가 일체형으로 결합된 타카(Tacker) 도구처럼, 조준과 격발을 동시에 수행하여 작업 속도를 극한으로 끌어올린 기계장치입니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 비교 1: 분리된 MUL/ADD vs 단일 FMA (MAC)

| 비교 항목 | 분리된 명령어 (MUL -> ADD) | 단일 FMA (Fused Multiply-Add) |
|:---|:---|:---|
| **명령어 수** | 2개 (MUL, ADD 별도 인출/해독) | 1개 (단일 명령어 인출) |
| **레지스터 접근** | 중간 결과 저장을 위해 레지스터 R/W 발생 | 하드웨어 내부 패스로 전달 (레지스터 미접근) |
| **정밀도 (Rounding)** | 곱셈 후 1번, 덧셈 후 1번 (총 2번 오차 발생) | 덧셈 후 1번만 반올림 (높은 정확도) |
| **레이턴시 (단일)** | $T_{mul} + T_{add}$ | $T_{mul} + T_{add}$ (유사하나 패스 최적화됨) |
| **처리량 (Throughput)**| 2 사이클당 1 MAC | 1 사이클당 1 MAC |

### 비교 2: 프로세서별 MAC 유닛 집적도 및 특성

| 프로세서 유형 | MAC 유닛 집적 방향 | 데이터 포맷 | 주 사용처 |
|:---|:---|:---|:---|
| **CPU (Central Processing Unit)** | SIMD (AVX-512 등) 확장 명령어를 통한 벡터 MAC | FP64, FP32, INT8 | 범용 컴퓨팅, 소규모 추론 |
| **GPU (Graphics Processing Unit)** | 텐서 코어 (Tensor Core) 기반 거대 행렬 MAC | FP16, TF32, FP8 | 딥러닝 대규모 학습/추론 |
| **NPU (Neural Processing Unit)** | 시스톨릭 어레이 (Systolic Array) 기반 수만 개 집적 | INT8, INT4 | 엣지 디바이스/모바일 저전력 추론 |

### 과목 융합 관점
- **컴퓨터 구조 (CA)**: MAC의 파이프라인 깊이는 프로세서의 IPC (Instructions Per Cycle) 및 브랜치 페널티에 영향을 주며, 데이터 해저드 (Data Hazard) 해결을 위해 데이터 포워딩 (Data Forwarding) 로직이 필수적이다.
- **알고리즘 (Algorithm)**: 딥러닝의 컨볼루션 연산은 Im2Col (Image to Column) 알고리즘을 통해 거대한 행렬 곱셈(GEMM, General Matrix Multiply)으로 변환되어 MAC 유닛 배열의 병렬성을 극대화한다.

- **📢 섹션 요약 비유**: 과거에는 두 명의 직원이 각각 자르고 붙이는 작업을 했다면, FMA/MAC은 한 명의 장인이 한 번의 칼질과 접착을 동시에 수행하여 품질(정밀도)과 속도를 모두 잡은 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오
1. **시나리오 — NPU 설계 시 누산기 오버플로우 문제**: 8-bit 정수(INT8) 데이터로 수천 번의 MAC 연산을 수행할 때, 누산기(Accumulator)를 8-bit로 유지하면 수십 번의 덧셈만으로 오버플로우가 발생하여 데이터가 파괴된다.
   - **의사결정**: 곱셈 결과는 16-bit로 커지고, 이를 수천 번 더하기 위해 누산기는 최소 32-bit (INT32) 폭을 갖도록 설계해야 한다. 이후 다음 레이어로 넘길 때 양자화 파라미터를 적용하여 다시 INT8로 스케일링(Re-quantization)하는 로직을 추가해야 한다.
2. **시나리오 — 부동소수점 비정규 수 (Denormalized Number) 처리 지연**: FP32 MAC 연산 시 0에 매우 가까운 비정규 수가 입력되면, 하드웨어가 이를 처리하기 위해 수십 사이클의 페널티를 발생시켜 실시간 처리에 병목이 생겼다.
   - **의사결정**: AI 추론과 같은 도메인에서는 극소값의 정밀도가 결과에 큰 영향을 미치지 않으므로, 레지스터 설정(예: FTZ - Flush To Zero, DAZ - Denormals Are Zero)을 활성화하여 비정규 수를 즉시 0으로 처리, 일관된 파이프라인 성능을 보장한다.

### 도입 체크리스트
- **기술적**: 타겟 모델(ResNet, Transformer 등)의 연산량(FLOPs) 대비 MAC 유닛의 수가 충분한가? 연산기(MAC)에 데이터를 공급할 메모리 대역폭(SRAM/DRAM)이 충분하여 연산기가 노는(Starvation) 현상이 없는가?
- **운영/보안**: 저정밀도(INT8) 양자화 시 모델의 정확도(Accuracy) 저하 폭이 비즈니스 요구사항(예: < 1% drop)을 만족하는가?

### 안티패턴
- **단순 루프 기반 연산**: 최적화되지 않은 컴파일러를 사용하여 스칼라 단위로 MAC을 수행하면 레지스터 병목으로 인해 캐시 및 SIMD 유닛의 장점을 전혀 살리지 못한다. 반드시 하드웨어에 최적화된 BLAS (Basic Linear Algebra Subprograms) 라이브러리를 활용해야 한다.

- **📢 섹션 요약 비유**: 물통(누산기)이 너무 작으면 물(데이터)이 넘쳐버리듯, 정밀도와 데이터 폭을 설계 단계에서 치밀하게 계산하지 않으면 연산 결과 전체가 무용지물이 됩니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량/정성 기대효과

| 구분 | 최적화 전 (스칼라 연산) | 최적화 후 (FMA/NPU MAC 어레이) | 기대 효과 |
|:---|:---|:---|:---|
| **정량 (속도)** | 1 MAC / 2 Cycle | 수천 MAC / 1 Cycle (병렬화) | 연산 처리량 **수천 배 향상** (TOPS 확보) |
| **정량 (전력)** | 명령어 인출 비용 과다 (pJ/op 큼) | 데이터 재사용성 극대화 (시스톨릭) | 에너지 효율 **10배 이상 개선** (mW/TOPS) |
| **정성 (구조)** | 복잡한 제어 로직 | 데이터 흐름 중심의 단순 반복 구조 | 실리콘 면적 대비 연산기 밀도 극대화 |

### 미래 전망
- **아날로그 MAC (Analog MAC)**: 디지털 로직 기반의 한계를 넘어, 멤리스터(Memristor)나 플래시 메모리 셀 자체의 물리적 특성(옴의 법칙, 키르히호프의 법칙)을 활용하여 전력 소모를 1/1000 수준으로 줄이는 아날로그 PIM 연구가 활발히 진행 중이다.
- **가변 정밀도 MAC (Dynamic Precision MAC)**: 레이어나 토큰의 중요도에 따라 4-bit, 8-bit, 16-bit 연산을 동적으로 선택하여 하드웨어를 재구성하는 혼합 정밀도(Mixed-Precision) 기술이 표준화될 것이다.

### 참고 표준
- **IEEE 754-2008**: 부동소수점 표준 규격으로 FMA (Fused Multiply-Add) 연산의 반올림(Rounding) 방식을 공식 정의.

- **📢 섹션 요약 비유**: MAC 유닛은 AI 반도체라는 거대한 피라미드를 쌓아 올리는 '가장 완벽한 형태의 벽돌'로, 향후 스스로 생각하고 변형되는 스마트 벽돌(가변 정밀도)로 진화할 것입니다.

## 👶 어린이를 위한 3줄 비유 설명
1. MAC (맥) 연산은 숫자 두 개를 '곱하고' 그 결과를 창고에 '더하는' 작업을 한 번에 쓱싹 해치우는 마법의 계산기예요.
2. 인공지능이 똑똑해지려면 이 곱하고 더하는 계산을 수억 번 해야 하는데, 옛날 컴퓨터는 곱하기와 더하기를 따로 해서 시간이 오래 걸렸어요.
3. 그래서 요즘 AI 반도체 안에는 이 마법 계산기(MAC) 수만 개를 꽉꽉 채워 넣어서, 순식간에 복잡한 문제를 풀어낸답니다!
