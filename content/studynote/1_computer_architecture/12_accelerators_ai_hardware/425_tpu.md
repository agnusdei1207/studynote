# TPU (Tensor Processing Unit)

> 1. **본질**: TPU (Tensor Processing Unit)는 구글 (Google)이 자사의 머신러닝 프레임워크인 텐서플로우 (TensorFlow)에 최적화하여 설계한 **맞춤형 주문형 반도체 (ASIC)** 기반 AI 가속기다.
> 2. **가치**: 시스톨릭 어레이 (Systolic Array) 아키텍처를 통해 대규모 행렬 곱셈 연산을 극도로 낮은 지연 시간과 높은 에너지 효율로 처리하며, TPU Pod 구성을 통해 수천 개의 칩을 하나의 거대한 슈퍼컴퓨터처럼 확장할 수 있다.
> 3. **융합**: 구글 검색, 번역, 포토, 유튜브 추천 알고리즘의 핵심 엔진이며, 구글 클라우드 플랫폼 (GCP)을 통해 전 세계 개발자들에게 강력한 딥러닝 학습 및 추론 인프라를 제공한다.

---

## Ⅰ. 개요 (Context & Background)

- **概念**: TPU는 딥러닝의 기본 단위인 '텐서 (Tensor)' 연산을 전문으로 처리하는 프로세서다. 일반적인 CPU나 GPU가 명령어 인출(Fetch)과 해독(Decode)에 많은 전력을 쓰는 것과 달리, TPU는 데이터가 연산기 사이를 직접 흐르게 설계하여 전력당 연산 성능을 극대화했다.

- **💡 비유**: TPU는 **"행렬 곱셈이라는 한 종류의 제품만 찍어내는 대규모 자동화 스마트 팩토리"**와 같다. 다품종 소량 생산을 하는 수공업자(CPU)나 대형 마트(GPU)보다, 특정 부품 하나를 전 세계에서 가장 빠르고 싸게 대량 생산하는 데 특화된 거대 공장 시스템인 셈이다.

- **등장 배경**:
  1. **데이터센터의 전력 위기**: 구글 사용자들이 하루 3분씩만 음성 인식을 써도 구글 데이터센터를 두 배로 늘려야 한다는 계산이 나오자, 극강의 효율을 가진 전용 칩이 필요해졌다.
  2. **딥러닝 연산의 정형화**: 신경망 연산의 90% 이상이 행렬 곱셈이라는 사실에 착안하여, 다른 기능은 과감히 버리고 행렬 연산에만 몰두한 하드웨어를 기획했다.
  3. **수직 계층 최적화**: 소프트웨어(TensorFlow)와 하드웨어(TPU)를 직접 설계함으로써, 범용 하드웨어가 가질 수 없는 최적화 이점을 누리고자 했다.

- **📢 섹션 요약 비유**: 특정 용도의 도구가 범용 도구보다 뛰어날 수 있다는 '도메인 특화'의 가치를 증명하며, 인공지능 시대의 컴퓨팅 문법을 새롭게 쓴 하드웨어입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **MXU (Matrix Multiply Unit)** | TPU의 핵심 심장부 | 64K개 이상의 MAC 유닛이 행렬 연산 수행 | Systolic Array | 수만 대의 로봇 팔이 동시에 조립하는 라인 |
| **Unified Buffer (HBM)** | 대용량 데이터 캐시 및 저장 | 연산기에 데이터를 초고속으로 공급 | High Bandwidth Memory | 공장 바로 옆에 붙어있는 거대한 자재 창고 |
| **HBM (High Bandwidth Memory)** | 모델 파라미터(가중치) 저장 | 고대역폭 메모리 인터페이스 제공 | TSV (Through Silicon Via) | 창고와 라인을 잇는 초광폭 컨베이어 벨트 |
| **벡터 처리기 (Vector Unit)** | 활성화 함수 및 정규화 처리 | ReLU, Softmax 등 요소별 연산 수행 | Activation Engine | 완성된 제품을 검수하고 포장하는 단계 |
| **ICI (Inter-Connect Interconnect)** | TPU 칩 간 고속 통신 | 호스트 CPU 없이 TPU끼리 직접 데이터 교환 | Torus Topology | 공장 단지 내의 전용 지하 물류 터널 |

### TPU의 핵심: 시스톨릭 어레이 (Systolic Array)

TPU 성능의 90%는 MXU의 시스톨릭 어레이 구조에서 나온다. 이는 데이터를 레지스터에 썼다 지웠다 하는 과정 없이, 연산기들 사이로 직접 밀어 넣는 방식이다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               TPU MXU (Matrix Multiply Unit) 동작 원리           │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   Data In (Act) ──▶ [PE] ──▶ [PE] ──▶ [PE] ──▶ [PE]           │
 │                      ▲        ▲        ▲        ▲             │
 │   Weight In (W) ─────┼────────┼────────┼────────┼───          │
 │                      │        │        │        │             │
 │   [동작 과정]                                                  │
 │   1. 가중치(W)가 각 PE(Processing Element)에 미리 배치됨        │
 │   2. 입력 데이터가 심장 박동처럼 한 칸씩 오른쪽으로 이동         │
 │   3. 각 PE는 (데이터 * 가중치)를 계산하고 아래로 누적값을 전달    │
 │   4. 마지막 줄에서 최종 행렬 곱셈 결과가 산출됨                 │
 │                                                               │
 │   장점: 메모리 접근 최소화, 전력 효율 극대화, 100% 연산기 가동률  │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 위 그림은 TPU 내부의 시스톨릭 어레이를 보여준다. 기존 CPU는 연산 한 번마다 메모리에서 데이터를 가져오고 결과를 저장하는 과정을 반복하지만, TPU는 데이터를 한 번만 밀어 넣으면 수백 개의 연산기를 지나가며 결과가 완성될 때까지 멈추지 않는다. 가중치를 PE 내부에 고정(Stationary)해 두고 입력 데이터만 흐르게 함으로써, 하드웨어의 병목 현상을 원천적으로 차단한다. 이를 통해 TPU v4 기준 칩 하나가 초당 수백 조 번의 연산을 수행할 수 있게 된다.

### 심층 동작 원리: 세대별 TPU 진화

1. **TPU v1 (추론 전용)**: 정수(INT8) 연산에만 집중하여 구글 검색 엔진 등에 적용.
2. **TPU v2/v3 (학습 및 추론)**: 부동소수점(bfloat16) 지원. 수천 개 칩을 묶은 TPU Pod를 통해 학습 시간을 수주에서 수시간으로 단축.
3. **TPU v4/v5 (생성형 AI 최적화)**: 광학 회로 스위치 (OCS)를 도입하여 칩 간 네트워크를 실시간으로 재구성. 거대 언어 모델 (LLM) 학습에 최적화된 아키텍처.

- **📢 섹션 요약 비유**: 작은 구멍 가게용 계산기(v1)에서 시작하여, 전 세계의 지식을 한꺼번에 학습하는 거대한 지능형 도시(v4/v5)로 진화한 과정입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: 구글 TPU vs NVIDIA GPU (H100)

| 비교 항목 | Google TPU v5p | NVIDIA H100 (GPU) |
|:---|:---|:---|
| **설계 목표** | 텐서플로우/JAX 최적화 (ASIC) | 범용 병렬 연산 (GPGPU) |
| **아키텍처** | 시스톨릭 어레이 기반 데이터 흐름 | SIMT 기반 멀티프로세서 (SM) |
| **소프트웨어** | 구글 클라우드 전용 (폐쇄형) | CUDA 생태계 (범용적 활용 가능) |
| **강점** | 대규모 포드(Pod) 확장성, 가성비 | 프레임워크 호환성, 개발 편의성 |
| **주요 모델** | Gemini, PaLM 2 학습 | GPT-4, Llama 3 학습 |

### 비교 2: TPU의 정밀도 형식 (bfloat16)

| 형식 | 부호(Sign) | 지수(Exponent) | 가수(Fraction) | 특징 |
|:---|:---:|:---:|:---:|:---|
| **FP32** | 1 | 8 | 23 | 표준 정밀도, 메모리 많이 차지 |
| **FP16** | 1 | 5 | 10 | 메모리 적으나 숫자의 표현 범위 좁음 |
| **bfloat16** | 1 | 8 | 7 | **FP32와 동일한 표현 범위**, 하드웨어 친화적 |

구글이 고안한 **bfloat16**은 정밀도는 낮추되 표현할 수 있는 숫자의 크기 범위는 FP32와 똑같이 유지하여, 딥러닝 학습 시 값이 폭주하거나 사라지는 문제를 해결하면서도 속도는 두 배 높였다.

### 과목 융합 관점

- **컴퓨터 네트워크 (Network)**: TPU Pod는 독자적인 ICI 네트워크와 OCS (Optical Circuit Switch)를 사용한다. 이는 전통적인 이더넷 망을 쓰지 않고 직접 광통신으로 칩들을 묶어, 수만 대의 TPU가 마치 하나의 거대한 GPU처럼 동작하게 한다.
- **운영체제 (OS)**: TPU는 호스트 CPU로부터 대량의 명령어를 일괄 수신하는 'Push' 방식을 사용한다. OS 커널의 개입을 최소화하기 위해 전용 유저 공간 드라이버를 통해 하드웨어에 직접 접근한다.

- **📢 섹션 요약 비유**: 모든 장비와 길, 그리고 언어까지 하나로 통일하여 오직 한 가지 목표(AI)만을 위해 움직이는 고도로 중앙 집권화된 제국과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **대규모 언어 모델 (LLM) 프리트레이닝**: 수조 개의 토큰을 학습해야 하는 경우, GPU 클러스터보다 TPU Pod가 비용 효율적일 수 있다. 구글 클라우드에서 JAX나 XLA 컴파일러를 사용하여 모델을 최적화하면, 수천 개의 TPU 노드에서 선형적인 성능 향상을 얻을 수 있다.
2. **실시간 번역 및 음성 합성 서비스**: 수억 명의 동시 요청을 처리해야 하는 서비스 백엔드에 TPU 추론 인스턴스를 배치한다. 시스톨릭 어레이 특유의 정형화된 처리 시간 덕분에, 응답 속도 (P99 Latency)를 매우 안정적으로 관리할 수 있다.
3. **알파고 (AlphaGo)와 같은 강화학습**: 수많은 시뮬레이션을 동시에 돌리고 승률을 계산해야 하는 강화학습에서 TPU의 높은 처리량은 필수적이다. 분산 강화학습 알고리즘을 TPU 아키텍처에 매핑하여 학습 효율을 극대화한다.

### 도입 체크리스트
- **프레임워크 종속성**: 프로젝트가 PyTorch나 JAX, TensorFlow를 사용하는가? (TPU는 이 프레임워크들에 최적화되어 있음)
- **데이터 로딩 병목**: TPU의 연산 속도가 너무 빨라 구글 클라우드 스토리지 (GCS)에서 데이터를 가져오는 속도가 못 따라오지는 않는가? (tf.data 파이프라인 최적화 필수)
- **비용 시뮬레이션**: 온프레미스 GPU 서버 구축 비용과 클라우드 TPU 대여 비용 중 어느 쪽이 장기적으로 유리한가?

### 안티패턴
- **소규모 배치 사이즈 (Small Batch Size)**: TPU의 MXU는 거대한 행렬을 한꺼번에 처리할 때만 효율이 나온다. 배치 사이즈를 너무 작게 잡으면 연산기의 대부분이 놀게 되어 성능이 CPU보다 안 나올 수도 있다.

- **📢 섹션 요약 비유**: 거대한 덤프트럭(TPU)에 모래 한 주머니(소규모 데이터)만 싣고 운행하는 낭비를 피하고, 트럭의 적재함을 꽉 채울 수 있는 규모의 경제를 실현해야 합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과 (표)

| 구분 | 도입 효과 | 비즈니스 가치 |
|:---|:---|:---|
| **학습 속도** | GPU 대비 동일 비용 당 2~3배 빠름 | 신규 AI 서비스 출시 기간 (TTM) 획기적 단축 |
| **운영 효율** | 구글의 풀 매니지드 서비스 활용 | 인프라 관리 인력 및 유지보수 비용 절감 |
| **에너지 가치** | 저전력 ASIC 설계로 탄소 배출 감소 | ESG 경영 실천 및 친환경 데이터센터 구현 |

### 미래 전망
TPU는 이제 칩을 넘어 **'데이터센터가 곧 컴퓨터'**가 되는 시대를 열고 있다. 구글은 TPU v6와 그 이후를 통해 연산과 네트워크, 스토리지가 완전히 하나로 녹아든 AI 전용 행성을 건설하려 할 것이다. 또한 머지않아 일반 기업의 온프레미스 환경에서도 구글의 기술이 녹아든 AI 칩렛이 사용될 가능성이 높다.

### 참고 표준
- **XLA (Accelerated Linear Algebra)**: TPU를 위한 구글의 머신러닝 최적화 컴파일러 표준
- **bfloat16**: 구글이 제안하고 현재는 IEEE 표준 및 타사 칩에서도 널리 쓰이는 수치 형식
- **TPU v4 OCS Spec**: 광 회로 스위칭 네트워크 표준 규격

- **📢 섹션 요약 비유**: 구글이라는 거인의 어깨 위에 올라타서, 세상의 모든 지능을 계산해내려는 하드웨어의 정점이 바로 TPU입니다.

### 📌 관련 개념 맵 (Knowledge Graph)
- **NPU (Neural Processing Unit)**: TPU가 속해 있는 범용 AI 가속기 범주
- **시스톨릭 어레이 (Systolic Array)**: TPU 연산의 근간이 되는 하드웨어 구조
- **MAC 연산 (Multiply-Accumulate)**: MXU에서 수행되는 최소 단위 연산
- **텐서플로우 (TensorFlow)**: TPU의 성능을 100% 이끌어내는 소프트웨어 프레임워크
- **HBM (High Bandwidth Memory)**: TPU에 데이터를 쏟아붓는 초고속 적층 메모리
