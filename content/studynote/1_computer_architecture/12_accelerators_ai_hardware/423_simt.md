# SIMT (Single Instruction Multiple Threads)

> 1. **본질**: SIMT (Single Instruction Multiple Threads)는 하나의 명령어를 수많은 독립적인 스레드가 각자의 데이터를 가지고 동시에 실행하는 **GPU 특유의 병렬 실행 모델**이다.
> 2. **가치**: SIMD (Single Instruction Multiple Data)의 하드웨어 효율성과 멀티스레딩의 소프트웨어 유연성을 결합하여, 개발자가 복잡한 벡터 연산 없이도 대규모 병렬 프로그램을 손쉽게 작성할 수 있게 해준다.
> 3. **융합**: CUDA 아키텍처의 근간을 이루며, 워프 (Warp) 단위의 실행 및 하드웨어 기반의 컨텍스트 스위칭과 결합하여 현대 GPGPU 연산 성능을 결정짓는 핵심 패러다임이다.

---

## Ⅰ. 개요 (Context & Background)

- **概念**: SIMT는 엔비디아(NVIDIA)가 제안한 실행 모델로, CPU의 SIMD와 유사하지만 근본적인 차이가 있다. SIMD는 하나의 명령어가 '벡터 레지스터' 내의 여러 데이터를 처리하는 방식인 반면, SIMT는 하나의 명령어가 '여러 개의 독립적인 스레드'를 동시에 구동하는 방식이다.

- **💡 비유**: SIMT는 **"에어로빅 강사와 수강생들"**과 같다. 강사(Instruction Unit)가 "팔을 올리세요!"라고 한 번 외치면(Single Instruction), 수만 명의 수강생(Multiple Threads)이 각자의 자리(Data)에서 동시에 팔을 올리는 것과 같다. 수강생들은 각자 독립된 사람이지만, 강사의 구령에 맞춰 일사불란하게 움직인다.

- **등장 배경**:
  1. **SIMD 프로그래밍의 난해함**: 기존 벡터 프로세서(SIMD)는 개발자가 데이터를 벡터 규격에 맞춰 직접 정렬해야 하는 번거로움이 컸다.
  2. **병렬성 극대화 요구**: 픽셀 연산처럼 독립적인 작업이 수백만 개 존재하는 그래픽 환경에서는, 각 작업을 '스레드'라는 익숙한 개념으로 다루는 것이 생산성 면에서 유리했다.
  3. **하드웨어 제어 오버헤드 감소**: 수천 개의 코어에 각기 다른 명령어를 주면 제어 회로가 너무 커진다. 명령 유닛을 공유하되 데이터만 따로 쓰는 SIMT는 칩 면적 효율성을 극대화한다.

- **📢 섹션 요약 비유**: 소프트웨어적으로는 자유로운 스레드들처럼 보이지만, 하드웨어적으로는 하나의 명령에 묶여 움직이는 '집단 지성' 형태의 연산 모델입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **명령어 유닛 (Instruction Unit)** | 공통 명령어 인출 및 해독 | 워프 내 모든 스레드에 동일 명령어 하달 | Fetch / Decode Stage | 에어로빅 강사의 마이크 |
| **스레드 전용 레지스터 (Register)** | 스레드별 개별 데이터 보관 | 각 코어에 할당된 독립적인 물리 저장소 | Scalar Registers | 수강생 각자의 운동 공간 |
| **워프 (Warp)** | SIMT 실행의 물리적 묶음 | 32개 스레드가 하나의 그룹으로 동작 | Hardware Scheduling Unit | 32명씩 묶인 체조 대형 |
| **PC (Program Counter)** | 스레드의 현재 실행 위치 추적 | 워프 내 공통 PC를 사용하되 발산 시 관리 | Active Mask | 강사가 부르는 진도 번호 |
| **액티브 마스크 (Active Mask)** | 조건문에 따른 스레드 활성/비활성 제어 | 조건에 맞지 않는 스레드의 연산 결과 차단 | Predicated Execution | "안경 쓴 사람만 손 들어!" 할 때의 필터링 |

### SIMD vs SIMT 아키텍처 비교

SIMT는 SIMD의 효율성을 유지하면서도, 각 스레드가 자신만의 실행 경로(Program Counter)를 논리적으로 가질 수 있게 설계되었다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               SIMD vs SIMT 실행 모델 비교                       │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   [ SIMD (Vectorized) ]           [ SIMT (Threaded) ]         │
 │                                                               │
 │   Instruction: ADD V1, V2         Instruction: ADD R1, R2     │
 │                                                               │
 │   ┌───────┐                       ┌───────┐                   │
 │   │  Inst │                       │  Inst │ (Shared)          │
 │   └───┬───┘                       └───┬───┘                   │
 │       ▼                               ▼                       │
 │   ┌───┴───┐               ┌───┬───┬───┬───┬───┐               │
 │   │  ALU  │ (Vector)      │ T0│ T1│ T2│ T3│ Tn│ (Threads)     │
 │   └───────┘               └───┴───┴───┴───┴───┘               │
 │   [D1,D2,D3,D4]           [D0] [D1] [D2] [D3] [Dn]            │
 │                                                               │
 │   SIMD: 프로그래머가 데이터를     SIMT: 프로그래머는 단일 스레드 │
 │         벡터로 직접 묶어야 함     코드만 짜면 하드웨어가 복제함 │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** SIMD (Single Instruction Multiple Data)는 개발자가 `VADD (Vector Add)`와 같은 전용 명령어를 사용하여 4개 혹은 8개의 데이터를 직접 묶어 처리해야 한다. 반면 SIMT는 개발자가 "A와 B를 더해라"라는 단순한 스칼라 코드를 작성하면, 하드웨어가 이를 수천 번 복제하여 각 스레드(T0, T1...)에 배분한다. 이때 각 스레드는 고유의 ID (`threadIdx`)를 기반으로 서로 다른 메모리 주소(D0, D1...)를 계산하여 접근한다. 이 '추상화' 덕분에 개발자는 벡터 규격에 얽매이지 않고 자유롭게 병렬 로직을 설계할 수 있으며, 실제 실행은 하드웨어가 워프 단위로 묶어 SIMD처럼 효율적으로 처리한다.

---

### 심층 동작 원리: 조건 분기와 스레드 발산 (Divergence)

SIMT의 가장 큰 특징이자 약점은 조건문 처리 방식이다.
1. **Branch Discovery**: 워프 내 스레드들이 `if-else`문에서 서로 다른 길로 가야 함을 감지한다.
2. **Serializing**: 하드웨어는 `if`절을 만족하는 스레드들만 먼저 실행하고, `else`절에 속한 스레드들은 잠시 멈춘다(Masking). 이후 반대로 실행한다.
3. **Convergence**: 두 경로가 모두 끝나면 다시 모든 스레드가 합쳐져 동일한 명령어를 실행한다.

- **📢 섹션 요약 비유**: "청군 이겨라!"라고 할 때 청군 팀만 응원하고 백군 팀은 쉬었다가, 나중에 백군 팀만 따로 응원하는 식으로 진행하여 결국 모두가 한 번씩은 움직이게 하는 방식입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: SIMD vs SIMT vs SMT (CPU Hyper-Threading)

| 비교 항목 | SIMD (Vector) | SIMT (GPU) | SMT (Hyper-Threading) |
|:---|:---|:---|:---|
| **기본 단위** | 데이터 요소 (Lane) | 독립 스레드 (Thread) | 하드웨어 컨텍스트 (Core) |
| **명령어 제어** | 명시적 벡터 명령어 | 단일 스레드 코드 복제 | 여러 명령어 스트림 혼합 |
| **유연성** | 낮음 (정렬 필수) | 높음 (조건 분기 가능) | 매우 높음 (완전 독립 작업) |
| **처리량** | 높음 | 매우 높음 | 보통 |
| **주체** | 컴파일러/개발자 | 하드웨어 (Warp Scheduler) | 하드웨어 (Core Resource) |

### 비교 2: SIMT 모델의 성능 결정 요인

| 요인 | 내용 | 최적화 방향 |
|:---|:---|:---|
| **워프 발산 (Divergence)** | 워프 내 스레드의 경로 불일치 | 조건문을 최소화하거나 32단위로 정렬 |
| **메모리 연합 (Coalescing)** | 워프 내 스레드의 메모리 접근 주소 | 인접한 주소를 동시에 읽도록 데이터 배치 |
| **점유율 (Occupancy)** | SM 내 활성 워프의 밀도 | 레지스터 사용량을 줄여 동시 실행 워프 확보 |

### 과목 융합 관점

- **컴퓨터 구조 (CA)**: SIMT는 '폰 노이만 구조'의 직렬 실행 모델을 병렬화하기 위한 변형이다. PC (Program Counter)를 공유하되 스택과 레지스터를 분리하여 하드웨어 효율과 소프트웨어 유연성을 동시에 잡았다.
- **프로그래밍 언어 (PL)**: CUDA C/C++는 SIMT를 지원하기 위해 `<<<grid, block>>>` 이라는 특수 문법을 도입했다. 이는 'SPMD (Single Program Multiple Data)' 프로그래밍 모델의 구체적 구현체이기도 하다.

- **📢 섹션 요약 비유**: 군대식 제식 훈련(SIMD)에 개인별 자율 행동(Thread)을 허용한 하이브리드 부대 운영 방식과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **이미지 처리 알고리즘의 SIMT 최적화**: 픽셀 단위 필터링 작업을 짤 때, 경계선 부분에서만 특수 처리를 하는 `if`문을 넣으면 워프 발산이 발생한다. 실무에서는 경계선 픽셀들만 모아서 별도의 커널로 돌리거나, 분기 없이 연산할 수 있는 수학적 트릭(Predication)을 사용하여 발산을 제거한다.
2. **행렬 곱셈 (GEMM)의 데이터 정렬**: SIMT 환경에서 각 스레드가 행렬의 열(Column)을 무작위로 읽으면 메모리 성능이 급락한다. 데이터를 32개 스레드가 순차적으로 접근하는 행(Row) 방향으로 배치하거나, 공유 메모리로 먼저 옮겨 담는 'Tiling' 기법을 사용하여 SIMT의 메모리 대역폭을 극대화한다.
3. **가변 정밀도 연산 (Mixed Precision)**: 최신 SIMT 아키텍처는 하나의 명령어(`WMMA`)로 16x16 행렬 곱셈을 한꺼번에 수행하는 텐서 코어를 지원한다. 이는 전통적인 SIMT를 넘어선 '매트릭스 연산 가속' 단계로의 진화를 의미한다.

### 도입 체크리스트
- **워프 발산률 확인**: 프로파일링 도구(Nsight 등)를 통해 워프 발산으로 인한 유효 지시어 실행률(IPC)이 급락하지 않았는지 확인했는가?
- **스레드 ID 활용**: `threadIdx`, `blockIdx`를 사용하여 데이터 인덱스를 계산할 때, SIMT의 정렬 규칙에 맞게 설계했는가?
- **하드웨어 제약 준수**: 한 워프가 사용하는 레지스터 수가 너무 많아 동시 실행 가능한 워프 수(Occupancy)가 줄어들지는 않았는가?

### 안티패턴
- **스칼라 연산의 GPU 강제 이행**: 병렬성이 전혀 없는 단순 직렬 코드를 SIMT 모델로 돌리는 경우. 수천 개의 코어 중 1개만 일하고 나머지는 놀게 되어, CPU보다 수백 배 느려지는 결과가 초래된다.

- **📢 섹션 요약 비유**: 수만 명의 병사들에게 각자 다른 음식을 요리하게 시키면 주방(ALU)이 하나뿐이라 줄을 서야 하는 것과 같으므로, 최대한 똑같은 일을 시키는 것이 관건입니다.

### 미래 전망
SIMT는 이제 고정된 32스레드 묶음을 넘어, 각 스레드가 자신만의 독립적인 스케줄링 가용성을 갖는 **'독립적 스레드 스케줄링 (Independent Thread Scheduling)'** 아키텍처(Volta 이후)로 진화하고 있다. 이는 워프 발산 시에도 스레드 간의 데이터 교환(Warp Shuffle)이나 동기화를 더 안전하고 빠르게 만들어, 더 복잡한 알고리즘을 GPU로 옮길 수 있게 할 것이다.
