# 시스톨릭 어레이 (Systolic Array)

> 1. **본질**: 시스톨릭 어레이 (Systolic Array)는 단순한 연산 소자 (PE: Processing Element)들이 격자 형태로 연결되어, 데이터가 일정한 박자에 맞춰 이웃 소자로 전달되며 연산이 이루어지는 **데이터 흐름 중심의 병렬 처리 아키텍처**다.
> 2. **가치**: 메모리에서 데이터를 한 번 읽어오면 수많은 연산 소자에서 재사용 (Data Reuse)함으로써, 폰 노이만 병목 현상을 해결하고 전력 소모를 획기적으로 줄여 대규모 행렬 곱셈 연산의 효율을 극대화한다.
> 3. **융합**: 구글의 TPU (Tensor Processing Unit) MXU의 핵심 엔진으로 채택되며 화려하게 부활했으며, 현대 인공지능 가속기 (NPU) 아키텍처 설계의 가장 중요한 기술적 근간이 되고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **概念**: 시스톨릭 (Systolic)은 생물학적 용어인 '심장 수축'에서 유래했다. 심장이 혈액을 전신으로 뿜어내듯, 중앙 제어 장치가 데이터를 연산기 네트워크로 쏘아주면 데이터가 소자들 사이를 규칙적으로 흐르며 결과가 만들어지는 방식이다. 명령어 중심의 전통적 구조와 달리 데이터의 '흐름' 그 자체가 연산 과정이 된다.

- **💡 비유**: 시스톨릭 어레이는 **"거대한 자동화 김밥 조립 라인"**과 같다. 김(입력 데이터)이 컨베이어 벨트를 따라 이동하면, 첫 번째 로봇(PE)이 밥을 얹고, 두 번째 로봇이 단무지를 얹는 식이다. 재료(가중치)는 각 로봇이 쥐고 있고, 김(데이터)만 옆으로 옮겨가며 최종적으로 김밥(결과값)이 완성되어 나오는 효율적인 공정이다.

- **등장 배경**:
  1. **폰 노이만 병목 (Von Neumann Bottleneck)**: 연산 속도보다 메모리 접근 속도가 느려 데이터 전송에 에너지를 다 쓰는 문제를 해결해야 했다.
  2. **반복적 연산의 지배**: 딥러닝이나 이미지 처리처럼 동일한 데이터를 가지고 수많은 곱셈과 덧셈을 반복해야 하는 작업이 늘어났다.
  3. **VLSI 설계의 용이성**: 동일한 작은 소자(PE)를 복사하여 이어 붙이기만 하면 되므로, 거대한 칩을 설계하고 검증하기에 매우 유리했다.

- **📢 섹션 요약 비유**: 데이터를 창고(메모리)에서 꺼내오는 횟수를 최소화하기 위해, 공장 내부에서 물건을 돌려 막으며 조립을 끝내는 지능형 생산 체계입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **PE (Processing Element)** | 최소 단위 연산 소자 | 곱셈-누산 (MAC) 연산 수행 | 8-bit / 16-bit MAC | 라인의 개별 로봇 팔 |
| **로컬 레지스터 (Local Reg)** | 임시 데이터 및 가중치 보관 | 입력값과 부분합(Partial Sum)을 1박자 유지 | Flip-Flop | 로봇 팔의 손아귀 |
| **데이터 웨이브프런트 (Wavefront)** | 데이터가 흐르는 물결 모양 | 타이밍에 맞춰 데이터가 대각선 방향으로 진행 | Synchronous Clock | 컨베이어 벨트의 일정한 속도 |
| **가중치 고정 (Weight Stationary)** | 메모리 접근 최소화 전략 | 가중치를 PE 내부에 고정해두고 데이터만 이동 | Architecture Strategy | 요리사가 한 자리에 고정된 상태 |
| **출력 버퍼 (Output Buffer)** | 최종 결과값 수집 | 마지막 PE 열에서 나온 데이터를 취합 | High Bandwidth SRAM | 완성품을 담는 최종 박스 |

### 시스톨릭 어레이의 행렬 곱셈 메커니즘

시스톨릭 어레이는 행렬 A와 B를 곱할 때, 각 데이터를 박자에 맞춰 밀어 넣음으로써 모든 PE가 동시에 일하게 만든다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               2x2 시스톨릭 어레이 데이터 흐름 (예시)               │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │      [B 데이터 유입 ↓]                                         │
 │         b11   b12                                             │
 │          │     │                                              │
 │   [A] ──▶[PE11]──▶[PE12]──▶ [결과]                             │
 │   a11    │     │                                              │
 │   a12 ──▶[PE21]──▶[PE22]──▶ [결과]                             │
 │          │     │                                              │
 │                                                               │
 │   1박자: a11과 b11이 PE11에서 만남 (곱셈 시작)                  │
 │   2박자: a11은 PE12로, b11은 PE21로 이동하여 다음 데이터와 만남 │
 │   3박자: 모든 소자가 연산 중 (최대 처리량 달성)                 │
 │                                                               │
 │   특징: 데이터 재사용률 극대화, 글로벌 버스 점유 최소화           │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 위 도식은 시스톨릭 어레이가 어떻게 행렬 데이터를 처리하는지 보여준다. 입력 행렬 A는 왼쪽에서 오른쪽으로, 행렬 B는 위에서 아래로 흐른다. 핵심은 데이터가 이웃 PE로 전달될 때마다 메모리 접근 없이 바로 다음 연산에 투입된다는 것이다. PE11에서 사용된 b11 데이터는 버려지지 않고 다음 박자에 아래쪽 PE21로 전달되어 a12와 만난다. 마찬가지로 a11은 오른쪽 PE12로 전달되어 b12와 만난다. 이 '이웃 간 직접 전송' 덕분에 메모리 읽기 횟수가 N배에서 1배 수준으로 급감하며, 이것이 TPU가 GPU보다 수십 배 높은 전성비(Watt당 성능)를 내는 물리적 이유다.

---

### 심층 동작 원리: 데이터 고정 (Stationary) 전략

1. **Weight Stationary (가중치 고정)**: 가중치를 PE 내부에 미리 저장해 두고, 입력 데이터와 부분합만 흐르게 한다. 가중치가 변하지 않는 인공지능 추론 (Inference)에 최적이다. (Google TPU 방식)
2. **Output Stationary (결과 고정)**: 부분합을 PE 내부에 누적하고, 입력 데이터와 가중치가 흐르게 한다. 최종 결과가 완성될 때까지 PE 밖으로 데이터를 내보내지 않는다.
3. **Input Stationary (입력 고정)**: 입력 데이터를 고정하고 가중치와 부분합을 흐르게 한다. 특정 입력에 대해 여러 모델을 적용할 때 유리하다.

- **📢 섹션 요약 비유**: 어떤 물건을 고정하고 무엇을 움직일지 결정하여, 불필요한 이동 동선을 최소화한 최첨단 공정 설계 전략입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: 폰 노이만 CPU vs 시스톨릭 가속기

| 비교 항목 | 폰 노이만 아키텍처 (CPU) | 시스톨릭 어레이 (NPU/TPU) |
|:---|:---|:---|
| **제어 방식** | 명령어 중심 (Instruction-driven) | 데이터 흐름 중심 (Data-driven) |
| **메모리 접근** | 연산마다 읽기/쓰기 발생 (고비용) | 한 번 읽기로 수천 번 재사용 (저비용) |
| **병렬성** | 수십 개 코어 (MIMD) | 수만 개 PE (Systolic) |
| **유연성** | 매우 높음 (General Purpose) | 낮음 (Matrix Ops 특화) |
| **전력 효율** | 낮음 | 매우 높음 |

### 비교 2: GPU (SIMT) vs 시스톨릭 (Dataflow)

| 항목 | GPU (NVIDIA 방식) | 시스톨릭 (Google 방식) |
|:---|:---|:---|
| **데이터 공유** | 레지스터/공유 메모리 경유 | 이웃 PE 간 직접 배선 연결 |
| **스케줄링** | 복잡한 하드웨어 스케줄러 필요 | 고정된 하드웨어 파이프라인 흐름 |
| **확장성** | SM 단위 확장 | PE 어레이 크기 확장 |
| **성능 일관성** | 워프 발산 등에 의해 가변적 | 결정론적 (Deterministic) 성능 |

### 과목 융합 관점

- **VLSI 설계 (Hardware)**: 시스톨릭 어레이는 구조가 규칙적이고 반복적이어서 레이아웃 설계(Place & Route)가 매우 쉽다. 이는 칩의 수율을 높이고 설계 비용을 낮추는 핵심 요인이다.
- **인공지능 알고리즘 (AI)**: 시스톨릭 어레이의 크기가 256x256이라면, 행렬 연산 시 이 크기에 맞춰 데이터를 쪼개는 '타일링 (Tiling)' 알고리즘 최적화가 필수적이다.

- **📢 섹션 요약 비유**: 유연한 프리랜서 팀(CPU/GPU)보다, 한 가지 공정만 완벽하게 수행하는 거대한 컨베이어 벨트 군단이 대량 생산에는 압도적으로 유리하다는 교훈입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **대규모 언어 모델 (LLM) 추론 가속**: Transformer의 핵심인 Attention 연산은 거대한 행렬 곱셈의 연속이다. 시스톨릭 어레이를 탑재한 가속기를 사용하면, 수천억 개의 파라미터를 메모리에서 읽어오는 시간을 가중치 고정 (Weight Stationary) 기법으로 상쇄하여 실시간 챗봇 서비스를 가능케 한다.
2. **에지 AI 영상 분석**: 전력 공급이 제한된 CCTV 카메라 내부에서 사람을 감지해야 할 때, 시스톨릭 기반의 소형 NPU를 사용한다. 글로벌 버스 트래픽이 거의 발생하지 않아 발열을 최소화하면서 30FPS 이상의 실시간 분석을 수행한다.
3. **컴파일러 최적화 (Mapping)**: 개발자가 짠 모델을 시스톨릭 어레이에 매핑할 때, 어레이의 빈 공간(Bubble)이 생기지 않도록 데이터를 정렬하는 것이 중요하다. 실무에서는 XLA와 같은 컴파일러가 데이터 흐름을 분석하여 어레이 가동률을 90% 이상으로 유지하도록 튜닝한다.

### 도입 체크리스트
- **데이터 재사용성**: 연산하려는 알고리즘이 시스톨릭 어레이의 장점인 데이터 재사용 (Reuse)을 충분히 이끌어낼 수 있는 구조인가?
- **어레이 크기 (Array Size)**: 대상 모델의 행렬 크기가 하드웨어 어레이 크기와 잘 맞는가? (너무 작으면 낭비, 너무 크면 조각화 발생)
- **메모리 대역폭**: 어레이 내부 속도는 빠르지만, 입구와 출구(HBM)에서 데이터를 공급하는 속도가 병목이 되지는 않는가?

### 안티패턴
- **희소 행렬 (Sparse Matrix) 처리**: 데이터에 0이 너무 많을 경우, 시스톨릭 어레이는 0을 곱하는 무의미한 연산에도 박자를 소모한다. 희소성 (Sparsity)이 높은 모델은 시스톨릭보다는 전용 희소 가속기를 쓰는 것이 옳다.

- **📢 섹션 요약 비유**: 물이 계속 흐르지 않으면 물레방아(시스톨릭)가 돌지 않듯, 끊임없는 데이터 공급과 규칙적인 데이터 구조가 아키텍처 성공의 열쇠입니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과 (표)

| 구분 | 도입 효과 | 비즈니스 가치 |
|:---|:---|:---|
| **에너지 가치** | 동일 연산량 당 전력 소모 90% 절감 | 데이터센터 전기료 절감 및 탄소 중립 기여 |
| **지연 시간** | 메모리 대기 시간 제거로 레이턴시 최소화 | 실시간 자율주행 및 금융 거래 지원 |
| **비용 효율** | 단순한 구조로 칩 집적도 향상 | 동일 면적 대비 압도적 연산력 (TFLOPS/mm²) |

### 미래 전망
시스톨릭 어레이는 이제 2D 격자를 넘어 3D로 적층되는 **'3D 시스톨릭 어레이'**나, 메모리 내부의 뱅크마다 시스톨릭 유닛을 배치하는 **'PIM (Processing-In-Memory)'** 기술로 확장되고 있다. 또한 정적인 박자를 넘어 데이터가 준비될 때만 움직이는 비동기식 (Asynchronous) 시스톨릭 구조도 연구되고 있어, 전력 효율의 한계에 계속해서 도전할 전망이다.

### 참고 표준
- ** systolic array 아키텍처**는 특정 표준 기구보다는 학계(H.T. Kung)의 논문과 구글 TPU의 구현 사례가 사실상의 기술 표준 역할을 수행한다.
- **IEEE P2844**: 인공지능 하드웨어의 성능 평가 표준 (시스톨릭 가속기 포함)

- **📢 섹션 요약 비유**: 1970년대에 설계된 고전적 아이디어가 40년 뒤 인공지능이라는 완벽한 짝을 만나 세상을 바꾸는 기술의 기적을 보여준 사례입니다.
