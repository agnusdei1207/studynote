# 가지치기 (Pruning) 지원 하드웨어

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가지치기 (Pruning) 지원 하드웨어는 인공신경망의 가중치 중 연산에 기여도가 낮은 0에 가까운 값들을 제거(Pruning)하여 발생한 희소성(Sparsity)을 물리적 연산 스킵(Skip) 및 데이터 압축으로 연결하여 가속하는 전용 아키텍처다.
> 2. **가치**: 불필요한 0과의 곱셈을 생략하여 연산 처리량을 이론적으로 수 배 향상시키고, 0인 데이터를 메모리에 저장하지 않음으로써 대역폭 효율을 극대화하여 메모리 월(Memory Wall) 문제를 해결한다.
> 3. **융합**: 비정형 희소성(Unstructured)의 유연성과 정형 희소성(Structured)의 하드웨어 친화적 특성을 절충한 2:4 Structured Sparsity (NVIDIA Ampere 아키텍처) 등 하드웨어-소프트웨어 코디자인 기술로 진화하고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 가지치기 (Pruning)는 딥러닝 모델의 성능에 영향이 적은 가중치(Weight)를 0으로 만들어 모델을 가볍게 만드는 기술이다. 하드웨어 관점에서의 가지치기 지원은 이렇게 생성된 '0' 값들을 단순히 저장하는 데 그치지 않고, 연산 시 하드웨어가 이를 감지하여 **실제 연산을 수행하지 않고 넘어가거나(Computation Skip)**, **압축된 형태로 데이터를 전송(Data Compression)**하는 기능을 의미한다.
- **💡 비유**: 가지치기는 과수원에서 실하지 않은 가지를 쳐내어 영양분을 열매에 집중시키는 것과 같습니다. 가지치기 지원 하드웨어는 이렇게 쳐낸 빈 공간(0)을 굳이 비싼 트럭에 실어 나르지 않고, 빈 가지에는 물을 주지 않는(연산 생략) 지능형 농장 자동화 시스템과 같습니다.
- **등장 배경**:
  1. **딥러닝 모델의 거대화와 중복성(Redundancy)**: 최신 AI 모델은 수천억 개의 파라미터를 갖지만, 실제로는 많은 가중치가 0에 가까워 연산 결과에 큰 영향을 미치지 않는다. 이를 모두 계산하는 것은 엄청난 자원 낭비다.
  2. **에너지 및 대역폭 병목**: 0을 곱하는 연산도 전력을 소모하고, 0 데이터를 메모리에서 읽어오는 것도 대역폭을 점유한다. 희소성(Sparsity)을 활용하면 동일한 자원으로 훨씬 큰 모델을 돌릴 수 있다.
  3. **소프트웨어와 하드웨어의 미스매치**: 소프트웨어적으로 가지치기를 해도 일반적인 CPU/GPU는 0인 값도 똑같이 처리(Dense 연산)하므로 속도가 빨라지지 않는다. 이를 해결하기 위해 0을 건너뛰는 전용 하드웨어 로직이 필요해졌다.
- **📢 섹션 요약 비유**: 빈 상자(0)가 섞인 컨베이어 벨트에서 빈 상자만 골라내어 버리고 알맹이가 든 상자만 다닥다닥 붙여서(압축) 배송하는 스마트 물류 센터와 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 및 메커니즘

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **Sparse Indexing Unit** | 0이 아닌 값의 위치를 추적 | 비트맵(Bitmap)이나 CSR(Compressed Sparse Row) 인덱스 관리 | 인덱스 디코딩 로직 | 보물지도 (유효 데이터 위치) |
| **Zero-Skip Logic** | 가중치가 0인 경우 MAC 연산 생략 | 입력 데이터와 가중치를 비교하여 0 감지 시 클럭 게이팅(Clock Gating) | Conditional Execution | 빈 접시 건너뛰기 |
| **Compression Engine** | 0을 제외한 데이터만 압축 저장 | 유효 데이터 + 메타데이터(위치 정보) 구조로 변환 | Run-Length Encoding | 압축 포장기 |
| **Load Balancer** | PE 간 작업 불균형 해소 | 희소 데이터 분포가 불균형할 때 연산기(PE)에 작업 재배분 | Work-stealing | 바쁜 계산대에 손님 배정 |
| **Fine-grained Sparsity Unit** | 미세 단위 가지치기 가속 | NVIDIA의 2:4 스파시티 등 특정 패턴 가속 | Structured Sparsity 지원 | 규격화된 포장 용기 |

### NVIDIA 2:4 Structured Sparsity 아키텍처

현대 GPU(Ampere 이후)에서 채택한 가장 실용적인 방식은 4개의 연속된 가중치 중 2개는 반드시 0이어야 한다는 '2:4 구조적 희소성'이다. 이는 비정형 희소성의 유연성과 정형 희소성의 연산 효율성을 결합한 형태다.

```text
 ┌──────────────────────────────────────────────────────────────────┐
 │            2:4 Structured Sparsity 하드웨어 가속 원리                │
 ├───────────────────────────────────────────────────────────────────┤
 │                                                                   │
 │   [원본 Dense 가중치 (1x4)]         [가지치기 및 압축 가중치]           │
 │   ┌───┬───┬───┬───┐               ┌───┬───┐   ┌───────┐         │
 │   │W0 │W1 │W2 │W3 │  ── Pruning ─▶ │W0 │W2 │ + │Index  │         │
 │   └───┴───┴───┴───┘   (2:4 패턴)   └───┴───┘   │(00,10)│         │
 │                                                └───────┘         │
 │                                                    │             │
 │   [Sparse Tensor Core 연산 과정]                   │             │
 │                                                    ▼             │
 │   입력 데이터 X: [X0, X1, X2, X3] ────┐      ┌─────────────┐     │
 │                                       ├─────▶│ MUX / Select │     │
 │   압축 가중치 W: [W0, W2]         ────┘      └─────────────┘     │
 │                                                    │             │
 │   ※ 하드웨어가 Index 정보를 보고                   │(X0, X2 선택)  │
 │      X1, X3는 버리고 X0, X2만 추출함.               ▼             │
 │                                            ┌───────────────┐     │
 │   결과 = (W0 * X0) + (W2 * X2)   ◀──────── │ 2-wide MAC    │     │
 │                                            └───────────────┘     │
 │                                                                   │
 │   ▶ 효과: 데이터량 1/2 감소, 연산 처리량(Throughput) 2배 향상       │
 └──────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 일반적인 연산기라면 4개의 숫자를 곱하고 더하기 위해 4-wide MAC 유닛이 필요하다. 하지만 2:4 구조적 희소성이 적용된 하드웨어에서는 4개 중 2개가 0인 것이 보장되므로, 실제로는 2-wide MAC 유닛만 있으면 된다. 하드웨어 내부의 MUX(Multiplexer)가 메타데이터인 Index 정보를 읽어, 입력 데이터 $X$ 중에서 0이 아닌 가중치와 짝을 이루는 $X_0, X_2$만 골라낸다. 결과적으로 메모리에서 읽어오는 가중치 양은 절반으로 줄어들고, 텐서 코어(Tensor Core)의 연산 처리량은 동일 시간 대비 2배로 늘어난다. 이는 소프트웨어의 가지치기 기법을 하드웨어의 병렬 연산 구조에 완벽하게 일치시킨 혁신적인 설계다.

### 심층 동작 원리 (Sparse-Dense 곱셈 가속)

1. **Pruning (SW)**: 학습 단계에서 가중치 절대값이 작은 하위 50%를 제거하되, 4개 블록 당 2개가 남도록 강제하여 2:4 패턴을 만든다.
2. **Encoding (HW/SW)**: 0이 아닌 가중치 값들과 그들의 원래 위치 정보를 담은 인덱스(2-bit per non-zero value)를 생성하여 압축 저장한다.
3. **Fetching (HW)**: 연산 시 압축된 가중치와 인덱스를 동시에 읽어 들인다. 이때 메모리 대역폭 이득이 발생한다.
4. **Decoding & Execution (HW)**: 인덱스 유닛이 입력 액티베이션 벡터에서 유효한 값만 추출(Selection)하고, MAC 어레이가 곱셈-누산을 수행한다.

- **📢 섹션 요약 비유**: 시험 문제 4개 중 2개는 항상 '정답 없음'이라고 알려주고, 학생(하드웨어)이 나머지 2개 문제만 집중해서 풀게 하여 시험 시간(연산 시간)을 절반으로 단축하는 기법과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 비교 1: Unstructured vs Structured Sparsity 지원 하드웨어

| 비교 항목 | 비정형 희소성 (Unstructured) | 정형 희소성 (Structured / Block) |
|:---|:---|:---|
| **가지치기 자유도** | 매우 높음 (어느 위치든 제거 가능) | 낮음 (정해진 블록 내에서만 제거) |
| **모델 정확도** | 높음 (중요한 값 보존 용이) | 상대적으로 낮음 (제약 조건 존재) |
| **하드웨어 가속** | **매우 어려움 (불규칙한 접근)** | **매우 쉬움 (규격화된 병렬 연산)** |
| **인덱스 오버헤드** | 큼 (모든 0이 아닌 값의 위치 저장) | 작음 (블록 단위 정보만 저장) |
| **대표 아키텍처** | EIE (Stanford), Cambricon-X | **NVIDIA Ampere/Hopper Tensor Core** |

### 비교 2: 가지치기 vs 양자화 (경량화 하이브리드)

| 가속 기법 | 핵심 원리 | 하드웨어 변화 | 시너지 효과 |
|:---|:---|:---|:---|
| **양자화 (Quantization)**| 데이터의 **비트 수(폭)**를 축소 | ALU/MAC 면적 축소, 정수 연산 | 연산 유닛 밀도 극대화 |
| **가지치기 (Pruning)** | 데이터의 **개수**를 축소 | 연산 스킵 로직, 인덱싱 유닛 | 연산 횟수 자체를 감소 |
| **하이브리드 적용** | **폭과 개수**를 동시에 줄임 | Sparse-Quantized MAC 유닛 | **10~50배 이상의 비약적 가속** |

### 과목 융합 관점
- **컴퓨터 구조 (CA)**: 가지치기 가속은 본질적으로 데이터 의존적(Data-dependent) 연산이다. 이는 정적인 파이프라인 설계를 어렵게 만들며, 하드웨어 수준에서 동적 스케줄링(Dynamic Scheduling)이나 로드 밸런싱(Load Balancing) 기술이 요구된다.
- **알고리즘 (Optimization)**: 하드웨어의 제약 조건을 반영하여 가지치기를 수행하는 하드웨어 친화적 가지치기(Hardware-friendly Pruning) 알고리즘이 연구의 핵심이며, 이는 신경망 구조 탐색(NAS)과 융합되고 있다.

- **📢 섹션 요약 비유**: 비정형 희소성이 길바닥에 흩어진 동전을 줍는 것이라면, 정형 희소성은 자판기 동전 통(블록)만 비우는 것과 같습니다. 전자는 꼼꼼하지만(정확도) 시간이 걸리고, 후자는 빠르고 효율적(하드웨어 가속)입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오
1. **시나리오 — 데이터센터 추론 처리량(Throughput) 극대화**: 수만 개의 검색 쿼리를 실시간으로 처리해야 하는 상황에서, GPU 자원은 한정되어 있고 전력 소모는 한계치에 도달함.
   - **의사결정**: 기존 FP16 모델에 2:4 Structured Sparsity를 적용하여 하드웨어 가속 모드 활성화. 정확도 손실 1% 이내에서 연산 처리량을 2배로 높여 추가 서버 증설 없이 트래픽 수용.
2. **시나리오 — 자율주행차의 객체 인식 모델 지연 시간(Latency) 단축**: 카메라 프레임이 들어온 후 브레이크를 밟기까지의 지연 시간이 너무 길어 사고 위험이 있음.
   - **의사결정**: 비정형 가지치기를 지원하는 전용 NPU(예: 인덱스 압축 지원 가속기)를 도입. 모델의 90%를 제거(Sparsity 90%)하여 연산 횟수를 1/10로 줄임으로써, 추론 지연 시간을 50ms에서 5ms로 단축하여 안전성 확보.

### 도입 체크리스트
- **기술적**: 보유한 하드웨어가 특정 희소성 패턴(예: 2:4)을 지원하는 전용 유닛을 내장하고 있는가? 압축된 가중치를 저장하고 해제하는 과정에서의 오버헤드가 연산 이득보다 크지 않은가?
- **운영/보안**: 가지치기 적용을 위한 리트레이닝(Retraining) 비용과 시간이 전체 프로젝트 일정 내에 소화 가능한가?

### 안티패턴
- **희소성이 낮은 모델에서의 적용**: 모델의 Sparsity가 20~30% 정도로 낮을 때 가지치기 가속 하드웨어를 사용하면, 인덱스를 관리하고 데이터를 압축/해제하는 제어 오버헤드가 실제 연산 절감 효과보다 커져서 오히려 성능이 저하될 수 있다. 통상 Sparsity가 50% 이상일 때 가속 효과가 유의미하다.

- **📢 섹션 요약 비유**: 가지치기가 안 된 울창한 숲(Dense Model)에 가지치기용 톱(Sparse 가속기)을 들고 들어가면, 거추장스러운 장비 무게 때문에 맨손으로 걷는 것보다 더 느려지는 것과 같습니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량/정성 기대효과

| 구분 | 도입 전 (Dense 연산) | 도입 후 (Sparse 가속 하드웨어) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (연산 처리량)** | 1x (기준) | 2x ~ 10x (Sparsity에 비례) | **동일 시간 내 더 많은 처리** |
| **정량 (메모리 대역폭)** | 100% 사용 | 10% ~ 50% 절감 (압축 효과) | **메모리 병목 해소** |
| **정성 (모델 복잡도)** | 중간 규모 모델만 가능 | 초거대 모델(LLM) 엣지 구동 가능 | 인공지능의 범용적 확산 |

### 미래 전망
- **Dynamic Pruning HW**: 고정된 가지치기 패턴이 아니라, 입력 이미지의 복잡도에 따라 실시간으로 레이어의 일부를 생략(Skip)하는 동적 가지치기 하드웨어가 주류가 될 것이다 (예: 하늘 이미지를 볼 때는 복잡한 특징 추출기 생략).
- **Sparsity-native Algorithm**: 처음부터 0이 많이 생기도록 유도하는 학습 기법과 이를 100% 활용하는 아키텍처가 결합되어, 인간의 뇌(극도로 희소한 연산)와 닮은 초효율 컴퓨팅이 실현될 것이다.

### 참고 표준
- **NVIDIA Ampere/Hopper Specification**: 2:4 Structured Sparsity의 사실상 산업 표준(De facto Standard) 역할.
- **MLPerf**: 희소성이 적용된 모델의 추론 성능을 공정하게 측정하기 위한 벤치마크 표준 정립 중.

- **📢 섹션 요약 비유**: 가지치기 지원 하드웨어는 데이터라는 원석에서 쓸모없는 흙(0)을 골라내고 순수한 보석(유효 데이터)만 가공하는 '지능형 제련소'이며, 이는 AI 반도체가 양적 성장을 넘어 질적 진화로 나아가는 핵심 지표가 될 것입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **양자화 (Quantization)**: 데이터의 정밀도를 줄여 가속하는 기술. 가지치기와 병행 시 시너지 극대화.
- **텐서 코어 (Tensor Core)**: 2:4 구조적 희소성 가속 기능을 내장한 NVIDIA의 핵심 연산 유닛.
- **시스톨릭 어레이 (Systolic Array)**: 정형화된 데이터 흐름을 중시하는 구조로, 가지치기 적용 시 로드 밸런싱이 매우 중요한 과제가 됨.
- **NPU (Neural Processing Unit)**: 가지치기 및 압축 기능을 하드웨어적으로 내장하여 전력 효율을 극대화한 가속기.

## 👶 어린이를 위한 3줄 비유 설명
1. 가지치기 하드웨어는 인공지능 공부 내용 중에서 **"몰라도 되는 0점짜리 문제"**를 아주 잘 골라내는 친구예요.
2. 0점이 섞여 있으면 계산할 필요가 없으니까, 0점을 쓱 건너뛰고 **진짜 중요한 문제만 쏙쏙 골라서** 풀기 때문에 엄청나게 빨라요.
3. 빈 상자(0)는 빼고 알맹이(중요한 값)만 꽉꽉 채워서 가방(메모리)에 넣으니까, 작은 가방에도 아주 많은 내용을 담을 수 있답니다!
