# 양자화 (Quantization, INT8, INT4)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 양자화 (Quantization)는 딥러닝 모델의 가중치(Weight)와 활성화 함수 결과(Activation)를 고정밀 부동소수점(FP32)에서 저정밀 고정소수점(INT8, INT4)으로 변환하여, 모델 크기를 줄이고 연산 속도를 가속하는 핵심 경량화 기술이다.
> 2. **가치**: 메모리 대역폭 요구사항을 1/4~1/8로 절감하고 NPU 내 MAC 연산기 면적을 기하급수적으로 줄여, 엣지(Edge) 기기에서 실시간 초거대 모델 추론을 가능하게 하는 경제적/기술적 필수 관문이다.
> 3. **융합**: 하드웨어 가속기(Tensor Core, DSP)의 전용 정수 연산 유닛과 긴밀히 결합하며, 정확도 손실을 최소화하기 위한 QAT(Quantization-Aware Training)와 PTQ(Post-Training Quantization) 기법이 소프트웨어 알고리즘과 융합되어 발전하고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 양자화 (Quantization)는 연속적인 값을 갖는 부동소수점(Floating Point) 데이터를 이산적인 값을 갖는 정수(Integer) 데이터로 매핑하는 과정이다. 예를 들어, -1.0 ~ 1.0 사이의 실수를 0 ~ 255 사이의 8비트 정수로 변환하여 저장하고 연산한다.
- **💡 비유**: 고해상도 4K 영화(FP32)를 스마트폰 화면에 맞게 적당한 화질의 HD 영화(INT8)로 인코딩하는 것과 같습니다. 약간의 화질 손상(정확도 하락)은 있지만, 파일 용량(메모리)이 획기적으로 줄어들고 끊김 없이(고속 연산) 감상할 수 있게 됩니다.
- **등장 배경**:
  1. **온디바이스 AI의 메모리 제약**: 모바일 SoC나 임베디드 기기의 SRAM/DRAM 용량은 제한적이다. 수십억 개의 파라미터를 가진 LLM(FP32 기준 수십 GB)을 탑재하기 위해서는 데이터 다이어트가 필수적이다.
  2. **에너지 효율 및 실리콘 면적**: 디지털 회로에서 32비트 부동소수점 곱셈기는 8비트 정수 곱셈기보다 면적은 약 10배, 에너지 소모는 약 20배 더 많이 필요하다. 정수 연산 기반의 하드웨어 설계가 저전력의 핵심이다.
  3. **추론 전용 가속기(NPU)의 부상**: 학습은 정밀도가 중요하지만, 추론(Inference) 단계에서는 약간의 오차가 결과에 큰 영향을 주지 않는 딥러닝의 견고성(Robustness) 특성을 활용하여 저정밀도 연산기가 주류로 자리 잡았다.
- **📢 섹션 요약 비유**: 복잡한 소수점 계산 대신 대략적인 반올림 숫자로 암산하여, 계산 속도를 높이고 머리(메모리)의 부담을 줄이는 지혜로운 계산법과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 및 파라미터

| 요소명 | 역할 | 내부 동작 | 수식/관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **Scale Factor (S)** | 실수 범위를 정수 범위로 축소/확대 | $S = (R_{max} - R_{min}) / (Q_{max} - Q_{min})$ | 아핀 변환 (Affine Mapping) | 지도상의 축척 |
| **Zero Point (Z)** | 실수의 0점이 정수 어디에 위치하는지 정의 | $Z = Q_{min} - R_{min} / S$ | 비대칭 양자화 (Asymmetric) | 온도계의 0도 보정 |
| **Clipping Range** | 양자화할 실수의 유효 범위 설정 | 특이값(Outlier)을 제거하여 양자화 정밀도 향상 | KL Divergence, MSE | 사진의 크롭(Crop) |
| **Quantizer** | 실수를 정수로 변환하는 연산 엔진 | $Q = clamp(round(R/S + Z))$ | Linear / Log Quantization | 반올림 처리기 |
| **De-quantizer** | 연산 후 결과를 다시 실수로 복원 | $R = S \times (Q - Z)$ | 추론 최종 단계에서 수행 | 암호 해독기 |

### 양자화 매핑 아키텍처 (Linear Quantization)

가장 널리 사용되는 선형 양자화는 부동소수점 영역의 값들을 균일한 간격의 정수 버킷(Bucket)으로 이동시킨다.

```text
 ┌──────────────────────────────────────────────────────────────────┐
 │               부동소수점 (FP32) ──▶ 정수 (INT8) 양자화 매핑               │
 ├──────────────────────────────────────────────────────────────────┤
 │                                                                  │
 │   [실수 영역 Real Domain (FP32)]                                   │
 │   R_min (-1.2)        0.0 (Zero)        R_max (1.5)              │
 │   ───┴─────────────────┼──────────────────┴───                    │
 │      │                 │                  │                      │
 │      ▼ (S, Z 파라미터 적용) ▼                  ▼                      │
 │      │                 │                  │                      │
 │   ───┴─────────────────┼──────────────────┴───                    │
 │   Q_min (0)           Z (110)            Q_max (255)             │
 │   [정수 영역 Quantized Domain (INT8)]                             │
 │                                                                  │
 │   ※ 수식: Q = Round(R/S + Z)                                     │
 │   ※ S (Scale): 한 버킷이 차지하는 실제 값의 크기                    │
 │   ※ Z (Zero Point): 실제 0.0이 정수 110으로 매핑됨                 │
 │                                                                  │
 │   [장점]                                                          │
 │   1. 데이터 크기 4배 감소 (32-bit -> 8-bit)                       │
 │   2. 하드웨어에서 정수 ALU/MAC 사용으로 고속화/저전력 구현           │
 └──────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** FP32로 표현된 연속적인 데이터 분포에서 최소값($R_{min}$)과 최대값($R_{max}$)을 확정한 후, 이를 8비트 정수가 가질 수 있는 0~255 범위로 선형적으로 투영한다. 이때 실제 0.0이 정수 0으로 매핑되지 않는 경우가 많은데, 이를 보정해주는 값이 제로 포인트($Z$)다. 이 $Z$값이 있으면 비대칭(Asymmetric) 양자화, $Z$가 0이면 대칭(Symmetric) 양자화라고 한다. 대칭 양자화는 계산이 단순하여 하드웨어 구현이 쉽지만, ReLU처럼 한쪽으로 치우친 데이터에서는 비대칭 양자화가 더 높은 정밀도를 유지한다.

### 심층 동작 원리 (추론 가속 프로세스)

1. **Calibration (PTQ 기준)**: 대표 데이터를 모델에 흘려보내 각 레이어의 활성화 값(Activation) 분포를 파악하고 최적의 Scale과 Zero-point를 정적(Static)으로 산출한다.
2. **Weight Quantization**: 학습된 FP32 가중치를 미리 정수(INT8/INT4)로 변환하여 저장한다.
3. **Quantized Inference**: 추론 시 입력 데이터도 양자화하여, 하드웨어 내부의 정수 MAC 유닛 어레이에서 $Q_{out} = \sum(Q_w \times Q_{in})$ 연산을 수행한다.
4. **Re-quantization**: 한 레이어의 결과인 INT32 누산값을 다음 레이어의 입력 형식인 INT8로 다시 스케일링한다.

- **📢 섹션 요약 비유**: 돋보기(Scale)로 범위를 좁게 보고 눈금(Zero Point)을 맞춰서 복잡한 치수를 정수로 딱딱 끊어 읽음으로써, 정밀 측정이 아닌 신속한 가공에 집중하는 공정과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 비교 1: PTQ vs QAT 기법 비교

| 비교 항목 | PTQ (Post-Training Quantization) | QAT (Quantization-Aware Training) |
|:---|:---|:---|
| **수행 시점** | 학습 완료 후 (Inference 전) | 학습 단계 (Training 중) |
| **추가 학습** | 필요 없음 (수 분 내 완료) | 필요함 (Fine-tuning 수행) |
| **정확도** | 정밀도 하락 위험 존재 (특히 4-bit) | **높은 정확도 유지 (하드웨어 특성 학습)** |
| **난이도** | 매우 낮음 (자동화 도구 풍부) | 높음 (학습 파이프라인 수정 필요) |
| **주 사용처** | 범용 모델, 빠른 배포 요구 시 | **최첨단 엣지 AI, 초저비트(INT4) 적용 시** |

### 비교 2: 정밀도별 하드웨어 특성 (FP32 vs INT8 vs INT4)

| 데이터 타입 | 비트 수 | 모델 크기 (7B 기준) | 에너지 효율 (pJ/op) | NPU 면적 비율 |
|:---|:---|:---|:---|:---|
| **FP32** | 32-bit | 약 28 GB | 100% (기준) | 100% (기준) |
| **INT8** | 8-bit | 약 7 GB | **~5% (20배 효율)** | **~10% (1/10)** |
| **INT4** | 4-bit | 약 3.5 GB | **~2% (50배 효율)** | **~3% (극소화)** |

### 과목 융합 관점
- **알고리즘 (ML/DL)**: 양자화는 손실 압축의 일종이다. 정보 이론 관점에서 볼 때, 가중치의 분포(Entropy)가 특정 영역에 집중되어 있을수록 양자화 오류(Quantization Error)가 줄어든다.
- **컴퓨터 구조 (CA)**: 하드웨어 설계자는 SIMD(Single Instruction Multiple Data) 유닛을 설계할 때, 32비트 레지스터 하나를 8비트 4개로 쪼개어 연산하는 'Sub-word Parallelism'을 통해 양자화된 데이터의 병렬 처리 성능을 극대화한다.

- **📢 섹션 요약 비유**: PTQ가 기성복을 사서 대충 몸에 맞게 수선해 입는 것이라면, QAT는 옷을 만들 때부터 내 몸의 치수(하드웨어 제약)를 재서 맞춤복을 짓는 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오
1. **시나리오 — 모바일 뱅킹 앱의 얼굴 인식 모델 탑재**: 서버 지연을 줄이기 위해 얼굴 인식 모델을 스마트폰 내부에서 돌려야 하는데, 모델 파일 크기가 500MB로 너무 커서 앱 용량 제한에 걸리고 실행 시 램 부족으로 앱이 강제 종료됨.
   - **의사결정**: PTQ 기반 INT8 양자화 적용. 모델 크기를 125MB로 줄이고 연산 속도를 3배 개선. 만약 정확도가 임계치 이하로 떨어진다면, 중요한 레이어만 FP16으로 남기는 혼합 정밀도(Mixed-Precision) 양자화를 적용한다.
2. **시나리오 — LLM (Llama-3) 온디바이스 서비스**: 최신 거대 언어 모델을 8GB RAM을 가진 PC에서 구동해야 함.
   - **의사결정**: 4-bit 양자화(예: AWQ, GPTQ 기법) 적용. 7B 모델 기준 가중치 용량을 3.5GB 수준으로 압축하여 VRAM 내에 상주시킴으로써 실시간 토큰 생성이 가능하도록 아키텍처를 최적화한다.

### 도입 체크리스트
- **기술적**: 하드웨어 가속기(NPU)가 INT8/INT4 MAC 연산을 지원하는가? (지원이 안 되면 소프트웨어 에뮬레이션으로 인해 오히려 느려짐)
- **운영/보안**: 양자화 후 Top-1 Accuracy 하락 폭이 비즈니스 요구사항(예: 0.5% 이내)을 만족하는가? 아웃라이어(Outlier) 값이 많아 양자화 효율이 떨어지지 않는가?

### 안티패턴
- **활성화 함수 고려 없는 양자화**: 가중치만 양자화하고 활성화 값(Activation)을 FP32로 남겨두면, 하드웨어 연산 시마다 정수 <-> 실수 변환 오버헤드가 발생하여 성능이 향상되지 않는다. 전체 데이터 흐름(Data Path)이 정수로 유지되도록 설계해야 한다.

- **📢 섹션 요약 비유**: 다이어트를 한다고 몸무게(가중치)만 줄이고 먹는 양(입력 데이터)을 줄이지 않으면, 결국 몸의 균형(시스템 성능)이 깨지고 건강(정확도)을 잃게 되는 것과 같습니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량/정성 기대효과

| 구분 | 도입 전 (FP32 시스템) | 도입 후 (INT8 Quantization) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (연산 속도)** | 10 FPS | 40 ~ 50 FPS | **4배 이상 빠른 실시간성 확보** |
| **정량 (메모리 절감)**| 1.0 GB | 250 MB | **메모리 점유율 75% 감소** |
| **정량 (전력 효율)** | 고발열, 배터리 급방전 | 저발열, 긴 배터리 수명 | 엣지 디바이스 사용성 극대화 |
| **정성 (보안/개인정보)**| 서버로 데이터 전송 필요 | 로컬 기기 내 보안 처리 가능 | Privacy-preserving AI 실현 |

### 미래 전망
- **Sub-1-bit Quantization (Binary/Ternary)**: 데이터를 오직 -1, 0, 1 세 가지 값으로만 표현하는 극단적인 양자화 기술이 연구 중이며, 이는 곱셈기 없이 덧셈만으로 AI를 구현하게 할 것이다.
- **Dynamic Quantization**: 추론 중 입력 데이터의 특성에 따라 비트 수를 실시간으로 조절(예: 중요 토큰은 8-bit, 일반 토큰은 2-bit)하는 가변 정밀도 기술이 표준이 될 것이다.

### 참고 표준
- **ONNX (Open Neural Network Exchange)**: 양자화된 모델 정보를 하드웨어 독립적으로 전달하기 위한 표준 포맷 제정.
- **IEEE 754 (Extended)**: 정수 기반의 AI 연산 정밀도 및 반올림 표준에 대한 논의 지속.

- **📢 섹션 요약 비유**: 양자화는 인공지능이라는 거대한 거인을 작고 가벼운 요정으로 변신시켜, 우리의 주머니 속(스마트폰)과 모든 사물(IoT) 속으로 들어가게 하는 마법의 축소 광선과 같습니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **MAC 연산 (Multiply-Accumulate)**: 양자화가 가장 직접적으로 가속하고자 하는 핵심 하드웨어 연산 단위.
- **NPU (Neural Processing Unit)**: 저정밀도(INT8) 연산에 최적화된 하드웨어 아키텍처.
- **가지치기 (Pruning)**: 양자화와 함께 모델 경량화의 양대 산맥을 이루는 기술. (데이터 크기 vs 데이터 개수)
- **지식 증류 (Knowledge Distillation)**: 양자화로 인한 정확도 손실을 보전하기 위해 큰 모델의 지식을 작은 모델에 전수하는 기술.

## 👶 어린이를 위한 3줄 비유 설명
