# GPU (Graphics Processing Unit)

> 1. **본질**: GPU (Graphics Processing Unit)는 수천 개의 단순하고 효율적인 코어를 병렬로 배치하여, 대규모 데이터 세트를 동시에 처리하도록 설계된 **처리량 중심 (Throughput-oriented) 프로세서**다.
> 2. **가치**: 초기 3D 그래픽 렌더링 가속에서 시작하여, 현재는 딥러닝 학습, 과학적 시뮬레이션, 암호화폐 채굴 등 병렬성이 높은 연산 작업에서 CPU 대비 수십 배 이상의 성능을 제공하는 컴퓨팅의 핵심 동력이다.
> 3. **융합**: 고대역폭 메모리 (HBM) 및 텐서 코어 (Tensor Core)와 결합하여 AI 가속기의 표준으로 진화했으며, 멀티 GPU 클러스터링을 통해 슈퍼컴퓨팅의 성능 한계를 돌파하고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **概念**: GPU는 수백 개의 복잡한 명령어를 빠르게 처리하는 CPU와 달리, 수백만 개의 단순한 픽셀 연산이나 수치 연산을 동시에 처리하는 데 최적화되어 있다. '지연 시간 (Latency)'을 줄이는 것보다 단위 시간당 '처리량 (Throughput)'을 극대화하는 아키텍처를 가진다.

- **💡 비유**: CPU가 **"한 번에 한 명의 승객을 아주 빠르게 목적지까지 실어 나르는 스포츠카"**라면, GPU는 **"수천 명의 승객을 한꺼번에 천천히 실어 나르는 대형 여객기"**와 같다. 개별 승객의 도착 시간은 스포츠카가 빠를 수 있지만, 만 명을 옮겨야 한다면 여객기가 압도적으로 효율적인 것과 같은 이치다.

- **등장 배경**:
  1. **고정 기능 파이프라인의 한계**: 초기 그래픽 카드는 정해진 공식으로만 그림을 그렸으나, 더 사실적인 그래픽을 위해 개발자가 직접 연산 로직을 짤 수 있는 '프로그래머블 쉐이더 (Shader)'가 요구되었다.
  2. **대규모 병렬 데이터의 증가**: 3D 그래픽뿐 아니라 시뮬레이션, AI 등 수천 개의 데이터에 동일한 연산을 적용해야 하는 작업이 늘어났다.
  3. **CPU의 병렬성 한계**: CPU 코어 수를 늘리는 것만으로는 행렬 연산과 같은 단순 반복 작업을 효율적으로 처리하기 어려웠다.

- **📢 섹션 요약 비유**: 복잡한 문제 해결사 한 명보다, 단순한 반복 업무를 한꺼번에 처리하는 수천 명의 직원을 고용하여 전체 작업 속도를 혁신적으로 높인 조직 시스템과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **SM (Streaming Multiproc.)** | GPU의 핵심 연산 단위 | 수십 개의 코어와 스케줄러를 포함 | NVIDIA Hopper/Ada Lovelace | 공장의 개별 생산 라인 |
| **CUDA 코어 / ALU** | 실제 수치 연산 (FP32 등) 수행 | 단순 산술 및 논리 연산 유닛 | Floating Point Operations | 생산 라인에 배치된 개별 작업자 |
| **HBM / VRAM** | 대용량 고속 데이터 저장 공간 | 넓은 버스 폭을 통한 초고속 데이터 전송 | HBM3e, GDDR6X | 공장 바로 옆에 있는 거대한 원자재 창고 |
| **L2 캐시 (L2 Cache)** | 코어 간 데이터 공유 및 메모리 접근 완화 | 모든 SM이 공유하는 중앙 집중식 캐시 | Shared Memory Hierarchy | 모든 부서가 공용으로 쓰는 중간 부품 보관소 |
| **GigaThread 엔진** | 수만 개의 스레드를 코어에 할당 | 하드웨어 스케줄러가 작업 분산 | Thread Scheduling | 전체 공장 가동을 총괄하는 스마트 관제 시스템 |

### GPU 아키텍처의 핵심: SIMT (Single Instruction, Multiple Threads)

GPU는 하나의 명령어를 수많은 스레드가 서로 다른 데이터를 가지고 동시에 실행하는 SIMT 방식을 사용한다. 이는 CPU의 SIMD보다 유연하며 대규모 병렬 처리에 최적화되어 있다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               CPU vs GPU 아키텍처 비교 (병렬성 관점)             │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   [ CPU: Latency Oriented ]       [ GPU: Throughput Oriented ]│
 │   ┌───────────────────────┐       ┌───────────────────────┐   │
 │   │  ┌───────┐ ┌───────┐  │       │ ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐  │   │
 │   │  │ Control│ │ Cache  │  │       │ └──┘└──┘└──┘└──┘└──┘└──┘  │   │
 │   │  └───────┘ └───────┘  │       │ ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐  │   │
 │   │  ┌─────────────────┐  │       │ └──┘└──┘└──┘└──┘└──┘└──┘  │   │
 │   │  │       ALU       │  │       │ ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐  │   │
 │   │  └─────────────────┘  │       │ └──┘└──┘└──┘└──┘└──┘└──┘  │   │
 │   │ (Few Powerful Cores)  │       │ (Thousands of Simple) │   │
 │   └───────────────────────┘       └───────────────────────┘   │
 │                                                               │
 │   CPU: 거대한 캐시와 제어 로직이   GPU: 제어 로직을 최소화하고  │
 │        칩 면적의 대부분을 차지     칩 대부분을 연산기(ALU)로   │
 │                                   채워 물량 공세 수행         │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** CPU 아키텍처는 분기 예측 (Branch Prediction)이나 복잡한 데이터 의존성을 해결하기 위해 거대한 제어 유닛 (Control)과 대용량 캐시를 배치한다. 반면 GPU는 이러한 제어 로직을 수십 개의 코어가 공유하게 만들어 다이 (Die) 면적의 대부분을 실제 연산기 (ALU)로 채운다. 데이터가 준비되지 않아 코어가 쉴 때는 다른 스레드로 즉시 전환 (Zero-overhead Context Switch)하여 하드웨어가 노는 시간 없이 계속 연산하게 만든다. 이것이 수 테라플롭스 (TFLOPS)의 압도적인 연산 능력을 내는 물리적 근거다.

---

### 심층 동작 원리: 렌더링 파이프라인에서 범용 연산으로

1. **초기 (Fixed Function)**: 정해진 기하학적 연산과 색상 입히기만 가능했다.
2. **쉐이더 모델 도입 (Unified Shader)**: 정점 (Vertex) 처리와 픽셀 (Pixel) 처리를 동일한 연산 유닛에서 수행하게 되면서 프로그래밍 가능성이 열렸다.
3. **GPGPU 시대 (CUDA)**: 그래픽과 상관없는 행렬 연산 등을 수행할 수 있게 되면서, GPU는 '그래픽 카드'에서 '병렬 가속기'로 진화했다.

- **📢 섹션 요약 비유**: 단순히 도장만 찍던 기계가, 프로그램만 바꾸면 그림도 그리고 수학 문제도 풀 수 있는 다목적 로봇 군단으로 진화한 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: CPU vs GPU 기술적 제원 비교

| 비교 항목 | CPU (Central Processing Unit) | GPU (Graphics Processing Unit) |
|:---|:---|:---|
| **설계 목표** | Low Latency (지연 시간 최소화) | High Throughput (처리량 최대화) |
| **코어 수** | 수십 개 (8~64개) | 수천 개 (2,000~16,000개+) |
| **캐시 메모리** | 대용량 (수십 MB), 복잡한 계층 | 소용량 (SM당 수십 KB), 단순 구조 |
| **제어 로직** | 매우 복잡 (Out-of-Order 실행) | 단순 (In-Order, 코어 간 공유) |
| **강점 작업** | 직렬 로직, OS 커널, 복잡한 조건문 | 행렬 연산, 렌더링, 신호 처리 |

### 비교 2: GPU 종류별 시장 세그먼트

| 구분 | 워크스테이션/서버용 (A100/H100) | 소비자용/게이밍용 (RTX 4090) |
|:---|:---|:---|
| **주요 기능** | FP64/FP16 연산, ECC 메모리 | 레이 트레이싱, DLSS, 높은 클럭 |
| **인터커넥트** | NVLink (노드 간 초고속 통신) | PCIe (표준 인터페이스) |
| **메모리** | HBM (최대 대역폭) | GDDR (가성비 고속 메모리) |
| **가격대** | 수천만 원 ~ 억 단위 | 수백만 원대 |

### 과목 융합 관점

- **컴퓨터 구조 (CA)**: GPU의 메모리 월 (Memory Wall) 문제를 해결하기 위해 도입된 HBM (High Bandwidth Memory)은 TSV (Through-Silicon Via) 공정을 활용한 3D 적층 기술의 정수다.
- **인공지능 (AI)**: 트랜스포머 (Transformer) 아키텍처의 부상은 GPU의 병렬 연산 능력과 찰떡궁합을 이뤘으며, 이는 곧 생성형 AI의 폭발적 성장을 이끌었다.

- **📢 섹션 요약 비유**: 수뇌부(CPU)가 전략을 짜면, 수많은 병사(GPU)가 일사불란하게 전장을 휩쓰는 현대전의 연합 작전과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **딥러닝 모델 학습 시간 단축**: ResNet-50과 같은 모델을 CPU로 학습하면 수주가 걸리지만, 고사양 GPU를 사용하면 몇 시간 내에 완료된다. 실무에서는 다중 GPU (Multi-GPU) 환경을 구축하고 Data Parallelism 기법을 적용하여 학습 속도를 선형적으로 확장한다.
2. **실시간 기상 예측 및 유체 시뮬레이션**: 수조 개의 입자 움직임을 계산해야 하는 시뮬레이션에서 GPU의 병렬성은 필수다. CUDA나 OpenCL을 사용하여 계산 물리 로직을 GPU로 옮김으로써, 과거 슈퍼컴퓨터로만 가능했던 정밀 예측을 워크스테이션 급에서 수행한다.
3. **클라우드 게이밍 및 VDI (Virtual Desktop)**: 서버 한 대의 GPU 자원을 여러 사용자가 나눠 쓰는 'GPU 가상화 (vGPU)' 기술을 적용한다. 이를 통해 사용자 기기 사양에 상관없이 고사양 게임이나 3D 설계를 스트리밍으로 제공한다.

### 도입 체크리스트
- **전력 공급 및 쿨링**: 최신 GPU(TDP 450W+)는 엄청난 전력을 소모하고 열을 내뿜는다. 서버실의 PDU (Power Distribution Unit) 용량과 냉각 인프라가 충분한가?
- **메모리 용량 (VRAM)**: 처리하려는 모델이나 데이터가 GPU 메모리에 다 올라가는가? 부족할 경우 스왑 (Swap)이 발생하여 성능이 90% 이상 하락한다.
- **CPU 병목 (Bottleneck)**: GPU 성능에 비해 CPU가 너무 낮으면, GPU에 데이터를 공급하는 속도가 느려져 GPU가 노는 현상이 발생한다.

### 안티패턴
- **오버헤드 간과**: 아주 작은 연산을 GPU로 보내는 경우, PCIe 버스를 통한 데이터 복사 시간이 실제 연산 시간보다 길어져 오히려 전체 속도가 느려진다.

- **📢 섹션 요약 비유**: 거대한 댐(GPU)을 지어도 물길(PCIe/메모리)이 좁으면 발전기(코어)를 제대로 돌릴 수 없는 것과 같으므로, 전체 밸런스 고려가 필수적입니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과 (표)

| 구분 | 도입 효과 | 비즈니스 가치 |
|:---|:---|:---|
| **연산 속도** | 행렬 연산에서 CPU 대비 20~100배 향상 | R&D 사이클 단축 및 시장 선점 |
| **가성비** | 동일 연산 능력당 도입 비용 1/10 수준 | 저비용 고효율 인프라 구축 |
| **범용성** | 표준 라이브러리(PyTorch, TensorFlow) 지원 | 개발 생산성 향상 및 인력 확보 용이 |

### 미래 전망
GPU는 단순히 그래픽 코어를 늘리는 단계를 넘어, 특정 연산 전용 유닛 (Tensor Core, Ray Tracing Core)을 섞는 **'헤테로지니어스 (Heterogeneous)'** 구조로 진화하고 있다. 또한 칩 간 통신 기술인 NVLink의 발전에 따라, 수백 대의 GPU가 마치 하나의 거대한 GPU처럼 동작하는 '데이터센터 규모의 컴퓨팅'이 가속화될 것이다.

### 참고 표준
- **IEEE 754**: GPU 부동소수점 연산의 표준 규격
- **Khronos Group (Vulkan/OpenGL)**: 개방형 그래픽 API 표준
- **NVIDIA CUDA**: 사실상의 업계 표준 병렬 컴퓨팅 플랫폼

- **📢 섹션 요약 비유**: 화면을 그리던 예쁜 도구에서, 인류의 지능을 확장하는 가장 강력한 엔진으로의 변신은 인류 기술사에서 가장 극적인 전환 중 하나입니다.

### 📌 관련 개념 맵 (Knowledge Graph)
- **GPGPU (General-Purpose GPU)**: GPU를 범용 연산에 활용하는 기술적 흐름
- **CUDA (Compute Unified Device Architecture)**: NVIDIA의 독보적인 GPU 프로그래밍 플랫폼
- **스트리밍 멀티프로세서 (SM)**: GPU의 물리적 연산 유닛 구조
- **텐서 코어 (Tensor Core)**: AI 연산 속도를 수십 배 높이는 전용 로직
- **HBM (High Bandwidth Memory)**: GPU의 폭발적 데이터 갈증을 해소하는 적층형 메모리
