# IPU (Intelligence Processing Unit)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPU (Intelligence Processing Unit)는 그래프 컴퓨팅 (Graph Computing)에 최적화된 MIMD (Multiple Instruction, Multiple Data) 아키텍처 기반의 AI 프로세서로, 대규모 병렬 연산과 초고속 온칩 메모리(SRAM)를 결합하여 딥러닝 모델의 희소성(Sparsity)을 극대화한다.
> 2. **가치**: 기존 GPU의 SIMD (Single Instruction, Multiple Data) 구조가 가진 정형 연산의 한계를 넘어, 비정형 데이터와 복잡한 조건부 연산이 많은 차세대 AI 모델(GNN, Probabilistic Model 등)에서 압도적인 속도와 전력 효율을 제공한다.
> 3. **융합**: BSP (Bulk Synchronous Parallel) 통신 모델을 하드웨어 레벨에서 구현하여 수천 개의 독립 코어 간 동기화 오버헤드를 최소화하며, 소프트웨어 스택인 Poplar와 결합하여 고도의 병렬 프로그래밍 환경을 구축한다.

## Ⅰ. 개요 (Context & Background)

- **개념**: IPU (Intelligence Processing Unit)는 영국 Graphcore 사에서 개발한 인공지능 전용 프로세서다. 수천 개의 독립적인 IPU-Core가 각자의 로컬 메모리(SRAM)를 가지고 독자적인 명령어를 수행하는 **MIMD 아키텍처**를 채택하여, 고도로 복잡하고 비정형적인 AI 모델 연산에 특화되어 있다.
- **💡 비유**: GPU가 수천 명의 군인이 지휘관의 구령 한 번에 똑같은 동작을 수행하는 '제식 훈련'이라면, IPU는 수천 명의 특공대원이 각자의 임무를 띠고 흩어져서 교신하며 작전을 수행하는 '게릴라전'과 같습니다.
- **등장 배경**:
  1. **SIMD 아키텍처의 한계**: GPU는 동일한 연산을 대량의 데이터에 적용할 때 유리하지만, 데이터가 듬성듬성하거나(Sparsity) 연산 흐름이 복잡하게 갈리는(Conditionals) 최신 AI 알고리즘에서는 유닛 활용도가 급격히 떨어진다.
  2. **메모리 지연 및 대역폭 병목**: 외부 DRAM에서 데이터를 계속 가져와야 하는 GPU와 달리, 모델 전체 혹은 주요 부분을 칩 내부의 초고속 SRAM에 상주시킴으로써 메모리 월(Memory Wall) 문제를 근본적으로 해결하려는 시도가 필요했다.
  3. **그래프 기반 AI 모델의 부상**: 소셜 네트워크 분석, 분자 구조 예측 등에 쓰이는 GNN (Graph Neural Network)은 데이터 간 연결 관계가 비정형적이어서 기존 행렬 곱셈 중심의 가속기로는 처리가 어렵다.
- **📢 섹션 요약 비유**: 중앙 통제실의 명령만 기다리는 로봇 부대 대신, 현장에서 스스로 판단하고 협력하는 지능형 드론 군단으로 시스템 전체의 유연성과 반응 속도를 높인 것과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **IPU-Core** | 독립적인 연산 수행 단위 | 6개의 하드웨어 스레드를 가진 MIMD 기반 프로세서 | Multi-threading | 개별 작전 요원 |
| **In-Processor Memory** | 데이터를 연산기 바로 옆에 상주 | 코어마다 할당된 대용량 SRAM (총 900MB+) | Scratchpad Memory | 요원의 개인 배낭 (보급품) |
| **Exchange Fabric** | 코어 간 초고속 데이터 교환 | 모든 코어를 올-투-올(All-to-all)로 연결하는 비차단 스위치 | Non-blocking Crossbar | 무전 통신망 |
| **IPU-Exchange** | 칩 간/보드 간 확장 인터커넥트 | 다수의 IPU를 엮어 거대 클러스터 구축 | IPU-Link | 부대 간 무선 링크 |
| **BSP Controller** | 전역 동기화 및 통신 제어 | 연산(Compute) - 통신(Exchange) - 동기화(Sync) 단계 관리 | Bulk Synchronous Parallel | 작전 개시 신호기 |

### IPU의 MIMD 및 BSP 아키텍처 다이어그램

IPU는 수천 개의 코어가 각자의 메모리에서 데이터를 처리하고, BSP (Bulk Synchronous Parallel) 모델에 따라 일제히 데이터를 교환한다.

```text
 ┌──────────────────────────────────────────────────────────────────┐
 │               IPU (Intelligence Processing Unit) 내부 구조           │
 ├───────────────────────────────────────────────────────────────────┤
 │                                                                   │
 │   [ IPU Chip ]                                                    │
 │   ┌────────────────────────────────────────────────────────────┐  │
 │   │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐        │  │
 │   │  │ Core 0  │  │ Core 1  │  │ Core 2  │  │ Core N  │        │  │
 │   │  │ [SRAM]  │  │ [SRAM]  │  │ [SRAM]  │  │ [SRAM]  │        │  │
 │   │  └─────────┘  └─────────┘  └─────────┘  └─────────┘        │  │
 │   │       │            │            │            │             │  │
 │   │  ─────┴────────────┴─────┬──────┴────────────┴─────        │  │
 │   │             Exchange Fabric (Non-blocking)                 │  │
 │   │  ─────┬────────────┬─────┴──────┬────────────┬─────        │  │
 │   │       │            │            │            │             │  │
 │   │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐        │  │
 │   │  │ Core... │  │ Core... │  │ Core... │  │ Core... │        │  │
 │   │  └─────────┘  └─────────┘  └─────────┘  └─────────┘        │  │
 │   └────────────────────────────────────────────────────────────┘  │
 │                                                                   │
 │   [ BSP (Bulk Synchronous Parallel) 동작 사이클 ]                 │
 │                                                                   │
 │   ──▶ [ 1. Compute ] ──▶ [ 2. Exchange ] ──▶ [ 3. Sync ] ───┐     │
 │   │    (독립 연산)         (코어 간 통신)       (전역 동기화)   │     │
 │   └─────────────────────────────────────────────────────────┘     │
 └──────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** IPU 내부에는 약 1,472개 이상의 독립적인 IPU-Core가 격자 형태로 배치되어 있다. 각 코어는 약 600KB 이상의 전용 SRAM을 가지고 있어, 전체 칩은 약 900MB에 달하는 거대한 **In-Processor Memory**를 형성한다. 이는 외부 DRAM 접근 없이 대규모 모델의 가중치를 칩 내부에 통째로 올릴 수 있게 한다. IPU의 가장 큰 특징은 **BSP (Bulk Synchronous Parallel)** 모델이다. ① 모든 코어가 자신의 SRAM 데이터로 독립적인 연산을 수행(Compute)하고, ② 연산이 끝나면 Exchange Fabric을 통해 서로 데이터를 주고받으며(Exchange), ③ 모든 코어의 교환이 완료될 때까지 기다린 후(Sync) 다음 단계로 넘어간다. 이 구조는 데이터 레이스 컨디션을 방지하고 병렬 프로그래밍의 복잡도를 획기적으로 낮춘다.

### 심층 동작 원리 (Graph Computing 가속)

1. **Graph Mapping**: 모델을 작은 연산 단위(Vertex)와 데이터 의존성(Edge)을 가진 그래프로 분해하여 각 IPU-Core에 분산 배치한다.
2. **Fine-grained Parallelism**: 각 코어는 MIMD 방식이므로 서로 다른 연산(예: 어떤 코어는 Conv, 어떤 코어는 ReLU)을 동시에 수행할 수 있어, 자원 활용도를 극대화한다.
3. **No-Stall Exchange**: 연산과 통신이 하드웨어적으로 철저히 분리되어 있어, 통신 단계에서 충돌 없이 모든 코어 간 올-투-올 데이터를 전송한다.
4. **Sparsity Harvesting**: 가중치가 0인 부분을 하드웨어가 감지하여 연산과 메모리 전송을 아예 생략하는 '희소성 수확' 기능을 통해 실질 성능을 수 배 높인다.

- **📢 섹션 요약 비유**: 모든 마을(코어)이 각자 농사를 짓고(Compute), 수확이 끝나면 장터(Fabric)에 모여 물건을 바꾼 뒤(Exchange), 마을 이장님의 종소리(Sync)에 맞춰 다시 농사지으러 돌아가는 평화롭고 질서 정연한 공동체와 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 비교 1: GPU vs IPU 아키텍처 비교

| 비교 항목 | GPU (NVIDIA H100 등) | IPU (Graphcore Bow 등) |
|:---|:---|:---|
| **연산 모델** | SIMD/SIMT (동일 명령 대량 처리) | **MIMD (독립 명령 병렬 처리)** |
| **메모리 구조** | 외부 DRAM (HBM) 중심 | **내부 SRAM (In-Processor) 중심** |
| **동기화 방식** | 소프트웨어/배리어 동기화 | **하드웨어 BSP 동기화** |
| **특화 워크로드**| Dense GEMM (행렬 곱셈), CNN | **Sparse Data, GNN, Probabilistic AI** |
| **유연성** | 정형 데이터에 최적 | 비정형/복잡한 로직에 최적 |

### 비교 2: AI 가속기 세대별 진화

| 세대 | 대표 장치 | 특징 | 한계점 |
|:---|:---|:---|:---|
| **1세대** | CPU/GPU | 범용성, SIMD 기반 가속 | 전력 효율 및 메모리 월 병목 |
| **2세대** | TPU / NPU | 시스톨릭 어레이, 정형화된 연산 | 비정형 연산 및 희소성 처리 미흡 |
| **3세대** | **IPU** | **MIMD, BSP, 거대 온칩 메모리** | 대규모 모델 수용 시 칩 간 확장성 과제 |

### 과목 융합 관점
- **알고리즘**: IPU는 확률적 그래픽 모델 (Probabilistic Graphical Models)이나 강화학습(RL)처럼 에이전트마다 서로 다른 환경을 탐색해야 하는 워크로드에서 GPU보다 수십 배 빠른 성능을 보여준다.
- **컴파일러 (Poplar)**: IPU의 성능은 소프트웨어 스택인 Poplar에 전적으로 의존한다. Poplar는 수만 개의 연산 그래프를 분석하여 물리적인 IPU-Core에 정적으로 배치(Static Allocation)하고 통신 스케줄링을 미리 확정함으로써 런타임 오버헤드를 제거한다.

- **📢 섹션 요약 비유**: GPU가 거대한 캔버스에 한 가지 색을 동시에 칠하는 거대한 붓이라면, IPU는 수천 명의 세밀화가들이 각자 맡은 부분을 그리면서 조화를 맞추는 정교한 예술 팀과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오
1. **시나리오 — 제약 회사의 신약 후보 물질 탐색**: 수조 개의 분자 구조 간의 상호작용을 예측해야 하는데, 데이터가 극도로 희소(Sparse)하고 그래프 구조가 매번 바뀌어 기존 GPU 클러스터로는 연산 효율이 5% 미만으로 떨어지는 상황.
   - **의사결정**: IPU 클러스터 도입. IPU의 MIMD 특성을 이용해 비정형 분자 그래프를 코어별로 최적 맵핑하고, BSP 모델로 통신 오버헤드를 제거하여 연구 기간을 1년에서 1개월로 단축.
2. **시나리오 — 금융권의 초고속 이상 거래 탐지 (Fraud Detection)**: 수만 건의 거래 데이터 중 복잡한 조건문(If-else)이 얽힌 신경망을 통과시켜 1ms 이내에 승인 여부를 결정해야 함.
   - **의사결정**: IPU 기반 추론 서버 구축. 조건부 연산 시 발생하는 분기 예측 실패 페널티가 적은 IPU의 강점을 활용하여, 복잡한 앙상블 모델을 실시간성(Low Latency) 있게 구동.

### 도입 체크리스트
- **기술적**: 우리가 사용하는 AI 모델이 행렬 곱셈 중심(Dense)인가, 아니면 비정형 데이터/그래프 구조(Sparse)인가? Poplar SDK가 현재 사용 중인 PyTorch/TensorFlow 버전을 완벽히 지원하는가?
- **운영/확장성**: 단일 칩 메모리(900MB)보다 큰 거대 모델을 돌리기 위해 다수의 IPU를 연결할 때 발생하는 IPU-Link 대역폭 병목을 계산했는가?

### 안티패턴
- **단순 대규모 행렬 곱셈 (Dense GEMM)**: 단순한 CNN이나 BERT 같은 정형화된 모델을 돌릴 때는 이미 최적화가 끝판왕 수준인 GPU(H100 등)가 가성비 면에서 압도적이다. IPU의 강점인 MIMD와 온칩 메모리 특성을 살릴 수 없는 워크로드에 IPU를 도입하는 것은 낭비다.

- **📢 섹션 요약 비유**: 고속도로에서 직진만 하면 되는 덤프트럭(GPU)이 필요한 곳에, 좁고 복잡한 골목길을 잘 달리는 스포츠카(IPU)를 배치하는 것은 자원의 미스매치입니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량/정성 기대효과

| 구분 | GPU 기반 시스템 | IPU 기반 시스템 | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (GNN 연산 속도)** | 1x (기준) | 10x ~ 50x | **비정형 연산 비약적 가속** |
| **정량 (전력 효율)** | 대규모 냉각 장치 필요 | 온칩 메모리 활용으로 저전력 | **TCO (총소유비용) 절감** |
| **정성 (모델 혁신)** | 정형 모델에 안주 | 차세대 비정형/확률적 모델 시도 가능 | AI 연구의 지평 확대 |

### 미래 전망
- **Bow IPU와 Wafer-level 통합**: 전력 공급 효율을 높인 Bow 아키텍처를 넘어, 웨이퍼 전체를 하나의 칩으로 만드는 WSE (Wafer Scale Engine) 급의 거대 IPU가 등장하여 초거대 모델을 100% 온칩 메모리에서 구동할 가능성이 크다.
- **AI-Scientific Convergence**: 기상 예측, 양자 역학 시뮬레이션 등 전통적인 HPC 영역과 AI가 결합되는 지점에서 IPU의 그래프 처리 능력이 핵심 인프라로 자리 잡을 것이다.

### 참고 표준
- **Open Graph 라이브러리**: IPU의 그래프 연산을 표준화하기 위한 오픈소스 프레임워크와의 연동 표준화 진행.
- **BSP (Bulk Synchronous Parallel)**: 병렬 컴퓨팅의 이론적 기반이자 IPU 하드웨어 통신 표준 모델.

- **📢 섹션 요약 비유**: IPU는 인공지능이 인간의 뇌처럼 '유연하고 복잡하게' 생각할 수 있도록 뒷받침하는 물리적 토대이며, 정형화된 계산의 시대에서 지능형 사고의 시대로 넘어가는 가교가 될 것입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **가지치기 (Pruning) 지원 하드웨어**: IPU의 핵심 경쟁력 중 하나인 희소성(Sparsity) 가속과 직결된 기술.
- **시스톨릭 어레이 (Systolic Array)**: IPU의 MIMD 방식과 대조되는 2세대 AI 가속기(TPU 등)의 핵심 구조.
- **뉴로모픽 컴퓨팅 (Neuromorphic Computing)**: IPU가 지향하는 궁극적인 비정형/병렬 지능형 컴퓨팅의 지향점.
- **메모리 월 (Memory Wall)**: IPU가 거대한 온칩 메모리로 돌파하고자 하는 현대 컴퓨팅의 최대 난제.

## 👶 어린이를 위한 3줄 비유 설명
1. 보통 컴퓨터(GPU)는 전교생이 운동장에 모여서 똑같은 체조를 하는 것처럼 공부하지만, IPU는 학생들이 각자 자기 방에서 자기가 맡은 과목을 공부하는 것과 같아요.
2. 공부하다가 모르는 게 생기면 중간에 한 번씩 모여서 친구들과 정보를 교환하고(Exchange), 다시 자기 방으로 돌아가서 공부해요.
3. 아주 복잡하고 어려운 퀴즈(그래프 데이터)를 풀 때, 다 같이 똑같은 생각만 하는 것보다 각자 다르게 생각하는 IPU 친구들이 훨씬 더 빨리 정답을 맞힌답니다!
