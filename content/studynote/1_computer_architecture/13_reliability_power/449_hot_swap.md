# 449. 핫 스왑 (Hot Swap/Hot Plug)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 전원을 켠 상태에서 하드웨어 교체
> 2. **가치**: 무중단 유지보수, 가용성 극대화, 비용 절감
> 3. **융합**: SATA, PCIe, USB, 서버, 스토리지

---

## Ⅰ. 개요

### 개념 정의

핫 스왑(Hot Swap)은 **시스템 전원을 켜고 운영 중인 상태에서 하드웨어 장치를 안전하게 제거하거나 추가하는 기술**이다.

### 💡 비유: 주유 중인 자동차

핫 스왑은 **주유 중인 자동차**와 같다. 자동차가 달린다. (시스템 운영) 기름이 떨어진다. (장치 교체 필요) 멈추지 않는다. (무중단) 주유한다. (핫 스왑) 계속 달린다. (서비스 지속) 핫 스왑이 똑같다. 무중단 교체, 서비스 지속.

## Ⅱ. 핫 스왑 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                핫 스왑 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SATA 핫 스왑】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  AHCI 모드 필요:                                                          │ │
│  │  • Native Command Queuing (NCQ)                                          │ │
│  │  • Hot Plug 지원                                                         │ │
│  │  • Port Multiplier                                                       │ │
│  │                                                               │ │
│  │  핫 스왑 절차:                                                            │ │
│  │  1. echo 1 > /sys/block/sdX/device/delete                                │ │
│  │  2. 물리적 디스크 제거                                                     │ │
│  │  3. 새 디스크 삽입                                                        │ │
│  │  4. echo "- - -" > /sys/class/scsi_host/hostX/scan                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【PCIe 핫 플러그】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  핫 플러그 지원:                                                            │ │
│  │  • PCIe 슬롯: Hot Plug Capable                                            │ │
│  │  • 전원 관리: Perst# 신호                                                  │ │
│  │  • 서비스: PCIe Hot-Plug Service                                          │ │
│  │                                                               │ │
│  │  주의사항:                                                                │ │
│  │  • 슬롯 전원 차단 후 삽입/제거                                              │ │
│  │  • 드라이버 언로드 선행                                                    │ │
│  │  • BIOS/UEFI 지원 확인                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 무중단 유지보수                                                        │ │
│  │  • 가용성 극대화                                                          │ │
│  │  • 유지보수 비용 절감                                                      │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 하드웨어 비용 증가                                                      │ │
│  │  • 소프트웨어 복잡성                                                       │ │
│  │  • 데이터 손실 위험 (오사용 시)                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 핫 스왑 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                핫 스왑 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【디스크 핫 스왑 (Linux)】                                                       │)
│  ──────────────────                                                  │
│  # 디스크 제거 전 준비                                                       │
│  $ echo 1 > /sys/block/sdb/device/delete                                │
│                                                                     │
│  # 새 디스크 스캔                                                          │
│  $ echo "- - -" > /sys/class/scsi_host/host0/scan                      │
│                                                                     │
│  # 디스크 확인                                                            │
│  $ lsblk                                                                │
│  $ fdisk -l                                                             │
│                                                                     │
│  【PCIe 장치 핫 플러그】                                                           │)
│  ──────────────────                                                  │
│  # PCIe 장치 제거                                                         │
│  $ echo 1 > /sys/bus/pci/devices/0000:01:00.0/remove                    │
│                                                                     │
│  # PCIe 장치 스캔                                                         │
│  $ echo 1 > /sys/bus/pci/rescan                                         │
│                                                                     │
│  【udev 모니터링】                                                               │)
│  ──────────────────                                                  │
│  $ udevadm monitor                                                      │
│  # 장치 이벤트 실시간 모니터링                                               │
│                                                                     │
│  【핫 스왑 팁】                                                                    │)
│  ──────────────────                                                  │
│  1. 백업: 교체 전 데이터 백업                                               │
│  2. 마운트 해제: 파일시스템 언마운트                                          │
│  3. RAID: 리빌드 상태 확인                                                 │
│  4. LED: 디스크 식별 LED 확인                                              │
│  5. 로그: 교체 이력 기록                                                   │
│  6. 테스트: 새 장치 정상 동작 확인                                           │
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 핫 스왑: 전원 켜진 상태에서 장치 교체
• 핫 플러그: PCIe 장치 실시간 추가
• 지원: USB, SATA, SAS, PCIe, NVMe
• 절차: 오프라인 → 제거 → 삽입 → 인식 → 온라인
• AHCI: SATA 핫 스왑 지원 모드
• RAID: 핫 스페어, 자동 리빌드
• udev: 장치 자동 인식
• 장점: 무중단 유지보수, 가용성
• 단점: 비용, 복잡성
• 활용: 서버, 스토리지, 데이터센터
```
