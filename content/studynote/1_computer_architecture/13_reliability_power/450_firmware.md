# 450. 펌웨어 / BIOS / UEFI

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드웨어와 OS 사이의 초기화 및 부트 소프트웨어
> 2. **가치**: 하드웨어 추상화, 부팅, 설정, 보안
> 3. **융합**: POST, 드라이버, Secure Boot, BIOS

---

## Ⅰ. 개요

### 개념 정의

펌웨어(Firmware)는 **하드웨어에 내장된 소프트웨어로, BIOS/UEFI는 시스템 부팅 시 하드웨어를 초기화하고 OS를 로드하는 펌웨어**다.

### 💡 비유: 건물 입주 전 점검

BIOS/UEFI는 **건물 입주 전 점검**과 같다. 건물이 있다. (하드웨어) 입주 전이다. (부팅) 점검한다. (POST) 전기 확인. 수도 확인. (하드웨어 초기화) 열쇠 받는다. (OS 로드) 입주한다. (부팅 완료) BIOS/UEFI가 똑같다. 하드웨어 점검, 초기화, OS 시작.

### 펌웨어 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                펌웨어 / BIOS / UEFI 개요                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【BIOS vs UEFI】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  특성           BIOS               UEFI                       │ │
│  │  ────           ────               ────                       │ │
│  │  전체 이름       Basic I/O System    Unified Extensible        │ │
│  │                                   Firmware Interface           │ │
│  │  저장 위치       ROM/Flash           Flash (SPI)               │ │
│  │  부팅 모드       Legacy (MBR)        UEFI (GPT)                │ │
│  │  최대 디스크      2TB (MBR)           9.4 ZB (GPT)             │ │
│  │  파티션 수        4개 (주)            128개+                    │ │
│  │  드라이버        16비트              32/64비트                  │ │
│  │  UI             텍스트              GUI 지원                    │ │
│  │  네트워크 부팅    PXE                PXE + HTTP                 │ │
│  │  보안            없음                Secure Boot               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【부팅 과정】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  전원 ON → POST → 하드웨어 초기화 → 부트로더 → OS                │ │
│  │                                                               │ │
│  │  1. POST (Power-On Self-Test):                                            │ │
│  │     • CPU, 메모리, 비디오 테스트                                           │ │
│  │     • 키보드, 저장장치 확인                                                │ │
│  │     • 비프음으로 에러 코드                                                 │ │
│  │                                                               │ │
│  │  2. 하드웨어 초기화:                                                       │ │
│  │     • 메모리 매핑                                                          │ │
│  │     • 장치 열거                                                            │ │
│  │     • ACPI 테이블 생성                                                     │ │
│  │                                                               │ │
│  │  3. 부트로더 로드:                                                         │ │
│  │     • BIOS: MBR → 부트로더                                                │ │
│  │     • UEFI: EFI 파티션 → 부트로더                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 펌웨어 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                펌웨어 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │

│  ──────────────────                                                  │
│  $ dmidecode -t bios                                                    │
│  # BIOS 정보                                                            │

│  # UEFI 변수 목록                                                        │

│  ──────────────────                                                  │
│  $ mokutil --sb-state                                                   │
│  # Secure Boot 상태                                                      │
│                                                                     │
│  $ cat /sys/firmware/efi/efivars/SecureBoot-*                          │
│  # Secure Boot 설정                                                      │
│                                                                     │
│  【부팅 모드 확인】                                                              │)

│                                                                     │
│  【BIOS/UEFI 팁】                                                                │)

│  3. 보안: BIOS 비밀번호 설정                                                │
│  4. Secure Boot: 필요시 활성화                                             │
│  5. 부팅 순서: 보안 고려 설정                                               │
│  6. 가상화: VT-x/AMD-V 활성화                                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 펌웨어: 하드웨어 내장 소프트웨어
• BIOS: Basic Input/Output System
• UEFI: Unified Extensible Firmware Interface
• POST: Power-On Self-Test
• Secure Boot: 부트 보안
• MBR: Master Boot Record (BIOS)
• GPT: GUID Partition Table (UEFI)
• ACPI: Advanced Configuration & Power Interface
• 장점: 하드웨어 추상화, 표준화
• 단점: 취약점, 호환성 문제
• 활용: 모든 컴퓨터 시스템
```
