# 435. 재해 복구 (Disaster Recovery)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장애 발생 시 시스템 복구를 위한 전략과 기술
> 2. **가치**: 비즈니스 연속성, 데이터 보호, RTO 최소화
> 3. **융합**: 백업, HA, DR 사이트, RPO, MTTR

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

재해 복구(Disaster Recovery, DR)는 **시스템 장애나 재해 발생 시 서비스를 복구하여 비즈니스 연속성을 보장하는 전략, 기술, 절차**다.

### 💡 비유: 소방 훈련

재해 복구는 **소방 훈련**과 같다. 화재가 난다. (장애) 대피한다. 복구한다. 미리 계획이 필요하다. (DR 계획) 정기 훈련한다. (DR 테스트) 실전처럼. 재해 복구가 똑같다. 계획, 훈련, 실행.

### 재해 복구 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                재해 복구 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【DR 지표】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. RTO (Recovery Time Objective):                                           │ │
│  │     • 복구 목표 시간                                                        │ │
│  │     • 장애 발생부터 서비스 복구까지                                          │ │
│  │     • 시간 단위 (분, 시간)                                                 │ │
│  │                                                               │ │
│  │  2. RPO (Recovery Point Objective):                                          │ │
│  │     • 복구 목표 지점                                                        │ │
│  │     • 허용 가능한 데이터 손실                                                │ │
│  │     • 시간 단위 (마지막 백업 시점)                                           │ │
│  │                                                               │ │
│  │  3. MTTR (Mean Time To Recovery):                                            │ │
│  │     • 평균 복구 시간                                                        │ │
│  │     • 장애 감지 + 진단 + 복구                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DR 전략】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  전략              RTO           비용         복잡성             │ │
│  │  ────              ────         ────         ────               │ │
│  │  Active-Active    즉시          높음          높음              │ │
│  │  Warm Standby     분~시간       중간          중간              │ │
│  │  Cold Standby     시간~일        낮음          낮음              │ │
│  │  Backup Only      일~주          최저          최저              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 재해 복구 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                재해 복구 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【DR 아키텍처】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────────┐                ┌─────────────────┐         │ │
│  │  │   Primary Site   │                │  Secondary Site  │         │ │
│  │  │                 │                │                 │         │ │
│  │  │  ┌───────────┐  │   Replication   │  ┌───────────┐  │         │ │
│  │  │  │   App     │  │ ────────────→  │  │   App     │  │         │ │
│  │  │  └───────────┘  │                │  └───────────┘  │         │ │
│  │  │  ┌───────────┐  │                │  ┌───────────┐  │         │ │
│  │  │  │   DB      │  │                │  │   DB      │  │         │ │
│  │  │  └───────────┘  │                │  └───────────┘  │         │ │
│  │  │  ┌───────────┐  │                │  ┌───────────┐  │         │ │
│  │  │  │  Storage  │  │                │  │  Storage  │  │         │ │
│  │  │  └───────────┘  │                │  └───────────┘  │         │ │
│  │  │                 │                │                 │         │ │
│  │  └─────────────────┘                └─────────────────┘         │ │
│  │                                                               │ │
│  │  Failover: Primary → Secondary                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【백업 유형】                                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 전체 백업 (Full Backup):                                                 │ │
│  │     • 모든 데이터 복사                                                    │ │
│  │     • 시간 소요, 공간 많이                                                 │ │
│  │                                                               │ │
│  │  2. 증분 백업 (Incremental Backup):                                          │ │
│  │     • 마지막 백업 이후 변경분만                                             │ │
│  │     • 빠름, 공간 절약                                                      │ │
│  │                                                               │ │
│  │  3. 차등 백업 (Differential Backup):                                         │ │
│  │     • 마지막 전체 백업 이후 변경분                                           │ │
│  │     • 증분보다 크지만 복구 쉬움                                            │ │
│  │                                                               │ │
│  │  4. 스냅샷 (Snapshot):                                                       │ │
│  │     • 특정 시점 상태 저장                                                   │ │
│  │     • 빠른 복구                                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 비즈니스 연속성                                                        │ │
│  │  • 데이터 보호                                                            │ │
│  │  • 규정 준수                                                              │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 높은 비용                                                              │ │
│  │  • 복잡성                                                                 │ │
│  │  • 정기 테스트 필요                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 재해 복구 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                재해 복구 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【HA 클러스터 구성 (Pacemaker)】                                                  │)
│  ──────────────────                                                  │
│  # 클러스터 상태 확인                                                   │
│  $ pcs status                                                        │
│                                                                     │
│  # 리소스 Failover                                                     │
│  $ pcs resource move <resource> <node>                               │
│                                                                     │
│  【데이터베이스 복제】                                                              │)
│  ──────────────────                                                  │
│  # MySQL 복제 상태                                                     │
│  SHOW SLAVE STATUS\G                                                 │
│                                                                     │
│  # PostgreSQL 복제 상태                                                │
│  SELECT * FROM pg_stat_replication;                                  │
│                                                                     │
│  【스냅샷 백업】                                                                  │)
│  ──────────────────                                                  │
│  # LVM 스냅샷                                                          │
│  $ lvcreate -L 10G -s -n snap01 /dev/vg01/lv01                       │
│                                                                     │
│  # ZFS 스냅샷                                                          │
│  $ zfs snapshot pool/dataset@snap01                                  │
│                                                                     │
│  【DR 테스트】                                                                    │)
│  ──────────────────                                                  │
│  1. 계획: 테스트 시나리오 작성                                          │
│  2. 통지: 관련 부서에 알림                                              │
│  3. 실행: Failover 수행                                                │
│  4. 검증: 서비스 정상 확인                                              │
│  5. 복구: 원복 수행                                                    │
│  6. 문서화: 결과 기록                                                  │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 재해 복구: 장애 시 서비스 복구
• RTO: 복구 목표 시간
• RPO: 복구 목표 지점 (데이터 손실)
• MTTR: 평균 복구 시간
• 전략: Active-Active, Warm, Cold, Backup
• 백업: 전체, 증분, 차등, 스냅샷
• HA: 고가용성 클러스터
• Failover: 장애 조치
• DR 사이트: 보조 사이트
• 장점: 연속성, 데이터 보호
• 단점: 비용, 복잡성
```
