# 230. 분기 지연 (Delayed Branch)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분기 후 명령어가 항상 실행되는 슬롯
> 2. **가치**: 제어 해저드 스톨 활용, MIPS 특징
> 3. **융합**: 컴파일러 스케줄링, 지연 슬롯 채우기

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

분기 지연(Delayed Branch)은 **분기 명령어 다음의 명령어(지연 슬롯)가 분기 여부와 관계없이 항상 실행되는 기법**이다. 파이프라인에서 분기로 인한 스톨을 유용한 작업으로 채우는 방식이다.

### 💡 비유: 엘리베이터 문 닫힘

분기 지연은 **엘리베이터 문이 닫히는 시간**과 같다. 문이 닫히는 동안(지연) 아무것도 안 하는 게 아니라, 버튼을 누르거나 짐을 정리한다(지연 슬롯 활용).

### 분기 지연 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                분기 지연 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                         │
│  ──────────────────                                                  │
│  • 분기 명령어: BEQ, BNE, J, JAL, ...                                 │
│  • 지연 슬롯: 분기 직후의 한 명령어                                      │
│  • 실행: 분기 여부와 관계없이 지연 슬롯 항상 실행                           │
│  • 분기: 지연 슬롯 실행 후 분기                                          │
│                                                                     │
│  【지연 슬롯 없이 (스톨 발생)】                                           │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BEQ R1, R2, target                                             │ │
│  │  ADD R3, R4, R5   ; 분기 시 실행 안 됨                              │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    BEQ                                                       │ │
│  │   2    ADD   BEQ        ← 분기 결정 전                              │ │
│  │   3    -     -     BEQ   ← 스톨 또는 플러시                          │ │
│  │   4    target ...                                                │ │
│  │                                                               │ │
│  │  → 스톨 또는 플러시 발생                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지연 슬롯 있이 (스톨 없음)】                                           │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BEQ R1, R2, target                                             │ │
│  │  ADD R3, R4, R5   ; 지연 슬롯 (항상 실행)                            │ │
│  │  target:                                                         │ │
│  │    SUB R6, R7, R8                                                │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    BEQ                                                       │ │
│  │   2    ADD   BEQ        ← 지연 슬롯                                │ │
│  │   3    SUB   ADD   BEQ   ← 지연 슬롯 실행, 분기 결정                 │ │
│  │   4          SUB   ADD   ...   ← 분기 후 실행                       │ │
│  │                                                               │ │
│  │  → 스톨 없이 분기 처리                                               │ │
│  │  → ADD는 분기 여부와 관계없이 실행                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지연 슬롯 활용도】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  활용도      설명                    빈도                         │ │
│  │  ────────  ──────────────────────  ──────                       │ │
│  │  매우 좋음   유용한 명령어 배치          60~70%                    │ │
│  │  좋음       분기 전 명령어 이동          15~20%                    │ │
│  │  보통       타겟에서 명령어 이동          10~15%                    │ │
│  │  안 좋음    NOP (채울 명령어 없음)       5~15%                     │ │
│  │  → 평균 85~95% 활용도                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 분기 지연 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분기 지연 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【지연 슬롯 채우기 전략】                                               │
│  ────────────────────────                                            │
│                                                                     │
│  1. From Before (분기 전에서 이동)                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  # Before                                                        │ │
│  │  ADD R1, R2, R3                                                  │ │
│  │  BEQ R4, R5, target                                              │ │
│  │  NOP                                                             │ │
│  │                                                               │ │
│  │  # After (ADD를 지연 슬롯으로)                                       │ │
│  │  BEQ R4, R5, target                                              │ │
│  │  ADD R1, R2, R3   ; 지연 슬롯                                     │ │
│  │  target:                                                         │ │
│  │    SUB R6, R7, R8                                                │ │
│  │                                                               │ │
│  │  조건: ADD가 분기에 영향 없음 (R4, R5 안 씀)                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  2. From Target (타겟에서 이동)                                        │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  # Before                                                        │ │
│  │  BEQ R1, R2, target                                              │ │
│  │  NOP                                                             │ │
│  │  target:                                                         │ │
│  │    ADD R3, R4, R5                                                │ │
│  │    SUB R6, R7, R8                                                │ │
│  │                                                               │ │
│  │  # After (ADD를 지연 슬롯으로)                                       │ │
│  │  BEQ R1, R2, target                                              │ │
│  │  ADD R3, R4, R5   ; 지연 슬롯                                     │ │
│  │  target:                                                         │ │
│  │    ADD R3, R4, R5   ; 여기도 있어야 함 (분기 안 했을 때)              │ │
│  │    SUB R6, R7, R8                                                │ │
│  │                                                               │ │
│  │  조건: 분기가 항상 일어날 때 유효                                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  3. From Fall-Through (분기 후에서 이동)                                │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  # Before                                                        │ │
│  │  BEQ R1, R2, target                                              │ │
│  │  NOP                                                             │ │
│  │  ADD R3, R4, R5   ; 분기 안 했을 때 실행                             │ │
│  │                                                               │ │
│  │  # After (ADD를 지연 슬롯으로)                                       │ │
│  │  BEQ R1, R2, target                                              │ │
│  │  ADD R3, R4, R5   ; 지연 슬롯                                     │ │
│  │  ; 분기 안 했을 때 ADD 실행 후 계속                                   │ │
│  │  ; 분기 했을 때도 ADD 실행 후 target으로                              │ │
│  │                                                               │ │
│  │  조건: 분기가 거의 안 일어날 때 유효                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지연 슬롯 제약】                                                      │
│  ──────────────────                                                  │
│  • 지연 슬롯에 분기 명령어 불가 (일부 아키텍처)                              │
│  • 지연 슬롯의 명령어가 분기 조건에 영향 주면 안 됨                          │
│  • 지연 슬롯의 명령어가 예외를 일으키면 안 됨                                │
│  • 디버깅 복잡도 증가                                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  지연 분기 장단점:                                                     │ │
│  │  ──────────────────                                                  │ │
│  │  장점:                                                               │ │
│  │  • 하드웨어 단순 (분기 예측 불필요)                                     │ │
│  │  • 스톨 100% 활용                                                    │ │
│  │  • 컴파일러로 최적화 가능                                              │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 새 ISA에 적용 어려움 (호환성)                                       │ │
│  │  • NOP이 많아질 수 있음                                              │ │
│  │  • 디버깅 어려움                                                      │ │
│  │  • 깊은 파이프라인에서 효과 감소                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 분기 지연 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분기 지연 × 타 과목 융합                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • MIPS: 1클럭 지연 슬롯                                               │
│  • SPARC: 1클럭 지연 슬롯                                              │
│  • PA-RISC: 1클럭 지연 슬롯                                            │
│  • x86/ARM: 지연 슬롯 없음 (분기 예측 사용)                              │
│                                                                     │
│  【컴파일러】                                                          │
│  • 지연 슬롯 채우기 최적화                                               │
│  • 의존성 분석: 채울 수 있는 명령어 찾기                                   │
│  • 프로파일 기반 최적화 (PGO)                                           │
│  • 코드 이동: 안전한 명령어 이동                                          │
│                                                                     │
│  【성능 분석】                                                         │
│  • 지연 슬롯 활용도: 85~95%                                            │
│  • 스톨 감소: 분기당 1클럭                                               │
│  • CPI 향상: 0.1~0.2 감소                                              │
│  • 현대 CPU: 분기 예측이 더 효과적                                        │
│                                                                     │
│  【역사적 맥락】                                                        │
│  • 1980년대: MIPS, SPARC에서 도입                                      │
│  • 1990년대: 깊은 파이프라인으로 효과 감소                                │
│  • 2000년대: 대부분 분기 예측으로 전환                                   │
│  • 현재: 일부 임베디드에서만 사용                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  지연 분기 vs 분기 예측:                                               │ │
│  │  ──────────────────────                                              │ │
│  │  특성          지연 분기     분기 예측                                 │ │
│  │  ───────────  ─────────  ──────────                                 │ │
│  │  하드웨어       단순        복잡                                       │ │
│  │  소프트웨어     복잡        단순                                       │ │
│  │  깊은 파이프라인  비효율       효율적                                   │ │
│  │  호환성         낮음        높음                                       │ │
│  │  현대 CPU       드묾        일반적                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 분기 지연 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분기 지연 분석                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MIPS 지연 슬롯 예시】                                                │
│  ────────────────────────                                            │
│  # 전형적인 루프                                                        │
│  loop:                                                               │
│      LW   R1, 0(R2)                                                  │
│      ADD  R3, R1, R4                                                 │
│      SW   R3, 0(R2)                                                  │
│      ADDI R2, R2, 4                                                  │
│      BNE  R2, R5, loop   ; 분기                                       │
│      NOP              ; 지연 슬롯 (개선 필요)                            │
│                                                                     │
│  # 지연 슬롯 최적화                                                     │
│  loop:                                                               │
│      LW   R1, 0(R2)                                                  │
│      ADD  R3, R1, R4                                                 │
│      SW   R3, 0(R2)                                                  │
│      BNE  R2, R5, loop   ; 분기                                       │
│      ADDI R2, R2, 4      ; 지연 슬롯 활용!                              │
│                                                                     │
│  → NOP을 ADDI로 채워 스톨 없이 실행                                      │
│                                                                     │
│  【GCC 지연 슬롯 최적화】                                                │
│  ────────────────────────                                            │
│  $ mips-linux-gcc -O2 -f-delayed-branch prog.c                      │
│  → 지연 슬롯 자동 최적화                                                │
│                                                                     │
│  【현대 CPU에서의 대안】                                                 │
│  ────────────────────────                                            │
│  • 분기 예측: 95%+ 정확도                                              │
│  • 스펙클레이티브 실행: 분기 전에 미리 실행                                │
│  • 비순차 실행: 분기 대기 중 다른 작업                                    │
│                                                                     │
│  【지연 분기가 여전히 쓰이는 곳】                                          │
│  ──────────────────────────                                          │
│  • MIPS 임베디드 시스템                                                 │
│  • SPARC 레거시 시스템                                                  │
│  • 교육용 단순 파이프라인                                                │
│  • 초저전력 마이크로컨트롤러                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  지연 분기 활용 팁:                                                    │ │
│  │  ──────────────────                                                  │ │
│  │  1. 분기 전의 독립적인 명령어를 지연 슬롯으로                             │ │
│  │  2. 루프 카운터 업데이트를 지연 슬롯으로                                 │ │
│  │  3. 타겟의 첫 명령어를 지연 슬롯으로 (분기 빈도 높을 때)                  │ │
│  │  4. 컴파일러 최적화 활용                                               │ │
│  │  5. 현대 CPU에서는 분기 예측 의존                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 분기 지연: 분기 후 명령어 항상 실행
• 지연 슬롯: 분기 직후의 한 명령어 위치
• 목적: 제어 해저드 스톨 활용
• 채우기: From Before, From Target, From Fall-Through
• 활용도: 85~95% (NOP 5~15%)
• MIPS, SPARC: 지연 분기 사용
• x86, ARM: 분기 예측 사용 (지연 분기 없음)
• 현대 CPU: 분기 예측이 더 효과적
```
