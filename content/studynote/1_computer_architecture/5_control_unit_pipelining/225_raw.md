# 225. RAW (Read After Write)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 쓰기 후 읽기, 데이터 의존성
> 2. **가치**: 가장 흔한 데이터 해저드, 포워딩으로 해결
> 3. **융합**: 포워딩, 스톨, Load-Use

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

RAW(Read After Write)는 **한 명령어가 데이터를 쓴 후, 다음 명령어가 그 데이터를 읽으려 할 때 발생하는 데이터 의존성**이다. 가장 흔한 데이터 해저드로, 파이프라인에서 결과가 아직 준비되지 않았을 때 발생한다.

### 💡 비유: 요리의 순서

RAW는 **요리의 순서**와 같다. "야채를 썰어서(Write) → 볶아라(Read)"라면, 야채가 다 썰릴 때까지 볶을 수 없다. 썰기가 완료되어야 볶기가 가능하다.

### RAW 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAW 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RAW 정의】                                                          │
│  ──────────────────                                                  │
│  명령어 i: R1 ← ...        (Write to R1)                             │
│  명령어 j: ... ← R1        (Read from R1)                            │
│  j가 i의 결과를 읽으려는데 i가 아직 R1에 안 씀                            │
│                                                                     │
│  【발생 예시】                                                         │
│  ──────────────────                                                  │
│  ADD R1, R2, R3   ; R1 ← R2 + R3  (Write R1)                        │
│  SUB R4, R1, R5   ; R4 ← R1 - R5  (Read R1)                         │
│  MUL R6, R1, R7   ; R6 ← R1 * R7  (Read R1)                         │
│                                                                     │
│  → SUB과 MUL은 ADD의 결과(R1) 필요                                    │
│  → ADD가 WB 되기 전에는 R1이 준비 안 됨                                 │
│                                                                     │
│  【파이프라인 타이밍】                                                  │
│  ────────────────────────                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ADD R1, R2, R3                                                  │ │
│  │  SUB R4, R1, R5   ; RAW on R1                                    │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    ADD                                                       │ │
│  │   2    SUB   ADD        ← ADD: R2, R3 읽기                        │ │
│  │   3          SUB   ADD   ← ADD: EX (R1 계산)                      │ │
│  │   4                SUB   ADD    ← ADD: MEM                        │ │
│  │   5                      SUB   ADD  ← ADD: WB (R1 기록)            │ │
│  │                          ↑                                       │ │
│  │                    SUB EX에서 R1 필요                              │ │
│  │                    ADD WB 전이므로 R1 없음                          │ │
│  │                                                               │ │
│  │  해결: 포워딩 (EX/MEM → EX)                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAW 발생 빈도】                                                      │
│  ──────────────────                                                  │
│  • 모든 데이터 해저드의 80~90%                                         │
│  • 레지스터 의존성이 가장 흔함                                          │
│  • 메모리 의존성도 존재                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RAW 의존성 거리:                                                     │ │
│  │  ──────────────────                                                  │ │
│  │  거리 1: 바로 다음 명령어 (가장 흔함)                                  │ │
│  │  거리 2: 한 명령어 건너뛰고                                           │ │
│  │  거리 3: 두 명령어 건너뛰고                                           │ │
│  │  → 거리가 멀수록 포워딩으로 해결 가능                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. RAW 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAW 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【해결 방법 1: 스톨】                                                  │
│  ──────────────────────                                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ADD R1, R2, R3                                                  │ │
│  │  SUB R4, R1, R5                                                  │ │
│  │  AND R6, R7, R8                                                  │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    ADD                                                       │ │
│  │   2    SUB   ADD                                                 │ │
│  │   3    -     -     ADD    ← 스톨                                  │ │
│  │   4    -     -     -     ADD                                     │ │
│  │   5    AND   -     -     -     ADD                               │ │
│  │   6          AND    SUB    -     -                               │ │
│  │                                                               │ │
│  │  → 3클럭 스톨 (비효율적)                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결 방법 2: 포워딩】                                                │
│  ──────────────────────                                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ADD R1, R2, R3                                                  │ │
│  │  SUB R4, R1, R5   ← R1을 EX에서 포워딩 받음                        │ │
│  │                                                               │ │
│  │  포워딩 경로:                                                      │ │
│  │  • EX/MEM.ALUResult → ID/EX.Rs 또는 Rt                           │ │
│  │  • MEM/WB.ALUResult 또는 ReadData → ID/EX.Rs 또는 Rt              │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    ADD                                                       │ │
│  │   2    SUB   ADD                                                 │ │
│  │   3          SUB   ADD    ← EX→EX 포워딩                          │ │
│  │   4                      SUB   ADD                               │ │
│  │                                                               │ │
│  │  → 스톨 없음 (효율적)                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Load-Use RAW (포워딩으로도 해결 안 됨)】                              │
│  ─────────────────────────────────────────                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  LW  R1, 0(R2)    ; R1 ← Mem[R2]  (Load)                         │ │
│  │  ADD R3, R1, R4   ; R3 ← R1 + R4  (Use R1)                       │ │
│  │                                                               │ │
│  │  문제: LW 결과는 MEM 단계 끝에 준비됨                                 │ │
│  │       ADD는 EX 단계에서 R1 필요                                     │ │
│  │       → 포워딩해도 1클럭 늦음                                        │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    LW                                                        │ │
│  │   2    ADD   LW                                                  │ │
│  │   3    -     -     LW     ← 1클럭 스톨 (불가피)                      │ │
│  │   4          ADD    -     LW    ← 포워딩                          │ │
│  │   5                      ADD   LW                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  포워딩 조건:                                                          │ │
│  │  ──────────────────                                                  │ │
│  │  1. EX/MEM.Rd == ID/EX.Rs 또는 ID/EX.Rt                            │ │
│  │  2. EX/MEM.RegWrite == 1                                          │ │
│  │  3. EX/MEM.Rd != 0 (zero 레지스터 아님)                             │ │
│  │  4. MEM/WB.Rd == ID/EX.Rs 또는 ID/EX.Rt (우선순위 낮음)              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. RAW × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAW × 타 과목 융합                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 파이프라인: RAW로 CPI 증가                                          │
│  • 포워딩: RAW 해결                                                   │
│  • 해저드 감지: RAW 조건 체크                                          │
│  • MUX: 포워딩 데이터 선택                                             │
│                                                                     │
│  【컴파일러】                                                          │
│  • 명령어 스케줄링: RAW 회피                                           │
│  • 의존성 분석: RAW 파악                                               │
│  • 코드 재배치: 의존성 없는 명령어 배치                                   │
│  • 레지스터 할당: RAW 최소화                                            │
│                                                                     │
│  【프로그래밍】                                                         │
│  • 데이터 의존성: 순차적 실행 보장                                       │
│  • 변수 사용: 순서 중요                                                 │
│  • 최적화: 컴파일러가 자동 처리                                          │
│                                                                     │
│  【병렬 처리】                                                         │
│  • 의존성 그래프: RAW가 병렬화 장애                                      │
│  • 벡터화: RAW 의존성 체크 필요                                         │
│  • 스레드 동기화: RAW 보장                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RAW 해결 효과:                                                       │ │
│  │  ──────────────────                                                  │ │
│  │  방식              스톨/명령어    CPI                                  │ │
│  │  ───────────────  ────────────  ────                                  │ │
│  │  스톨만 사용        2~3클럭       1.5~2.0                              │ │
│  │  포워딩            0~1클럭       1.0~1.1                              │ │
│  │  코드 재배치        0클럭         1.0                                  │ │
│  │  → 포워딩으로 30~50% 성능 향상                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### RAW 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAW 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【포워딩 활성화 확인】                                                 │
│  ────────────────────────                                            │
│  $ cat /proc/cpuinfo | grep flags                                   │
│  → 현대 CPU는 기본적으로 포워딩 활성화                                   │
│                                                                     │
│  【RAW로 인한 스톨 측정】                                               │
│  ──────────────────────────                                          │
│  $ perf stat -e uops_executed.stall_cycles ./prog                   │
│  → RAW 스톨 확인                                                      │
│                                                                     │
│  【코드 최적화 예시】                                                   │
│  ────────────────────────                                            │
│  # Before (RAW 발생)                                                 │
│  ADD R1, R2, R3                                                      │
│  SUB R4, R1, R5   ; R1 대기                                          │
│  MUL R6, R7, R8                                                      │
│  DIV R9, R1, R10  ; R1 대기                                          │
│                                                                     │
│  # After (코드 재배치)                                                 │
│  ADD R1, R2, R3                                                      │
│  MUL R6, R7, R8   ; 의존성 없음                                        │
│  SUB R4, R1, R5   ; R1 준비됨                                         │
│  DIV R9, R1, R10  ; R1 준비됨                                         │
│  → 포워딩 + 코드 재배치로 스톨 없음                                      │
│                                                                     │
│  【Load-Use 최소화】                                                    │
│  ──────────────────                                                  │
│  # Before                                                            │
│  LW  R1, 0(R2)                                                       │
│  ADD R3, R1, R4   ; 1클럭 스톨                                        │
│  MUL R5, R6, R7                                                      │
│                                                                     │
│  # After                                                             │
│  LW  R1, 0(R2)                                                       │
│  MUL R5, R6, R7   ; 스톨 사이에 다른 명령어                             │
│  ADD R3, R1, R4   ; 스톨 없음                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RAW 최소화 팁:                                                       │ │
│  │  ──────────────────                                                  │ │
│  │  1. 포워딩 하드웨어 활용 (자동)                                        │ │
│  │  2. Load 후 바로 사용 피하기 (1클럭 스톨)                              │ │
│  │  3. 의존성 없는 명령어 배치                                            │ │
│  │  4. 루프 언롤링으로 의존성 분산                                        │ │
│  │  5. 컴파일러 최적화 옵션 사용                                          │ │
│  │  6. 레지스터 할당 최적화                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• RAW: Read After Write, 쓰기 후 읽기 의존성
• 발생: 이전 명령어 결과가 다음 명령어 입력
• 빈도: 데이터 해저드의 80~90%
• 해결: 포워딩 (EX/MEM → EX)
• Load-Use: LW 후 바로 사용, 1클럭 스톨 불가피
• 코드 재배치: 컴파일러로 의존성 없는 명령어 배치
• 포워딩 효과: 30~50% 성능 향상
```
