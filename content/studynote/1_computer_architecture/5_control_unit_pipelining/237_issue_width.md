# 237. 명령어 발급 폭 (Issue Width)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 클럭에 발급 가능한 최대 명령어 수
> 2. **가치**: IPC 상한선 결정, 성능과 복잡도의 트레이드오프
> 3. **융합**: 슈퍼스칼라, 파이프라인 깊이와 연동

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

명령어 발급 폭(Issue Width)은 **한 클럭 사이클에 파이프라인으로 발급할 수 있는 최대 명령어 수**이다. 슈퍼스칼라 프로세서의 병렬도를 결정하는 핵심 파라미터로, IPC(Instructions Per Cycle)의 이론적 상한선을 정의한다.

### 💡 비유: 마트 계산대

발급 폭은 **계산대 개수**와 같다. 1개 계산대면 한 번에 1명만 계산할 수 있다. 4개 계산대면 4명이 동시에 계산할 수 있다. 하지만 계산대가 많을수록 관리도 복잡해진다.

### 발급 폭과 프로세서 분류

```
┌─────────────────────────────────────────────────────────────────────┐
│                발급 폭과 프로세서 분류                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【발급 폭 분류】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  발급 폭    분류           IPC 이론값   대표 CPU                  │ │
│  │  ────────  ─────────────  ──────────  ───────────────────       │ │
│  │  1         스칼라          1.0         초기 MIPS, ARM7           │ │
│  │  2~3       좁은 슈퍼스칼라    2.0~3.0     ARM Cortex-A53          │ │
│  │  4~6       중간 슈퍼스칼라    4.0~6.0     Intel Core i7           │ │
│  │  7~8       넓은 슈퍼스칼라    7.0~8.0     Apple M1/M2             │ │
│  │  8+        초넓은 슈퍼스칼라   8.0+        서버급 CPU              │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【발급 폭 vs 실제 IPC】                                                 │
│  ──────────────────                                                  │
│                                                                     │
│       IPC                                                            │
│        ↑                                                            │
│     8  │  ┌─────────────────────── 이론적 상한선 (Issue Width)         │
│        │  │/////////////////////////////////////////////////////    │
│     6  │  │/////////////////////////////////////////////////////    │
│        │  │        //////////////////////////////////////////////////│
│     4  │  │        ////////                                          │
│        │  │        ////////  ← 실제 달성 IPC                         │
│     2  │  │  ////  ////////    (해저드, 의존성으로 인해)               │
│        │  │  ////  ////////                                          │
│     0  │  │  ////  ////////                                          │
│        └──┴──┴──┴──┴──┴──┴──┴──┴──→ 발급 폭                          │
│           1   2   3   4   5   6   7   8                              │
│                                                                     │
│  【발급 폭 증가의 비용】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  발급 폭    면적    전력    설계 복잡도    실제 IPC 증가             │ │
│  │  ────────  ──────  ──────  ────────────  ──────────────        │ │
│  │  1→2       +50%    +40%    +100%         +80%                  │ │
│  │  2→4       +100%   +80%    +200%         +60%                  │
│  │  4→8       +200%   +150%   +400%         +40%                  │
│  │  → 발급 폭 2배 = IPC 2배 아님 (수확 체감 법칙)                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 발급 폭 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                발급 폭 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【발급 폭 결정 요소】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 인출 대역폭 (Fetch Bandwidth)                                │ │
│  │     • I-Cache 포트 수                                            │ │
│  │     • 분기 예측 정확도                                            │ │
│  │     • BTB 크기                                                    │ │
│  │                                                               │ │
│  │  2. 디코드 대역폭 (Decode Bandwidth)                              │ │
│  │     • 디코드 로직 병렬도                                          │ │
│  │     • 명령어 길이 가변성 (x86 vs RISC)                            │ │
│  │     • 마이크로옵스 변환                                           │ │
│  │                                                               │ │
│  │  3. 발급 로직 (Issue Logic)                                       │ │
│  │     • 의존성 검사 복잡도                                          │ │
│  │     • 레지스터 리드 포트                                          │ │
│  │     • 예약 스테이션 크기                                          │ │
│  │                                                               │ │
│  │  4. 실행 유닛 (Execution Units)                                   │ │
│  │     • ALU 개수                                                   │ │
│  │     • FPU 개수                                                   │ │
│  │     • 메모리 유닛 개수                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【발급 폭 vs 파이프라인 단계】                                           │
│  ──────────────────                                                  │
│                                                                     │
│  발급 폭 4의 파이프라인:                                                │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │      IF          ID          EX          MEM         WB       │ │
│  │   ┌───────┐   ┌───────┐   ┌───────┐   ┌───────┐   ┌───────┐   │ │
│  │   │ I1~I4 │ → │ I1~I4 │ → │ I1~I4 │ → │ I1~I4 │ → │ I1~I4 │   │ │
│  │   │ 인출  │   │ 디코드│   │ 실행  │   │ 메모리│   │ 쓰기  │   │ │
│  │   └───────┘   └───────┘   └───────┘   └───────┘   └───────┘   │ │
│  │       ↓           ↓           ↓           ↓           ↓       │ │
│  │   4개씩       4개씩       4개씩       4개씩       4개씩        │ │
│  │   동시        동시        동시        동시        동시         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【현대 CPU 발급 폭 사례】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU              발급 폭   인출 폭   리타이어 폭   실제 IPC       │ │
│  │  ───────────────  ───────  ───────  ──────────  ──────────     │ │
│  │  Intel Skylake    6        6        6           2.5~3.0        │ │
│  │  AMD Zen 3        6        8        6           2.8~3.2        │ │
│  │  Apple M1         8        8        8           3.0~3.5        │ │
│  │  ARM Cortex-A78   4        4        4           1.8~2.2        │ │
│  │  → 실제 IPC < 발급 폭 (항상)                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  • 스케줄링: CPU 발급 폭 고려                                           │

## Ⅳ. 실무 적용

### 발급 폭 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                발급 폭 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【발급 폭 활용률 측정】                                                 │
│  ──────────────────                                                  │
│  $ perf stat -e instructions,cycles ./prog                         │
│  instructions: 10,000,000,000                                       │
│  cycles: 4,000,000,000                                              │
│  → IPC = 10B / 4B = 2.5                                            │
│  → 발급 폭 6인 경우 활용률 = 2.5/6 ≈ 42%                              │
│                                                                     │
│  【발급 슬롯 분석】                                                      │
│  ──────────────────                                                  │
│  $ perf stat -e uops_issued.any,uops_retired.retire_slots ./prog  │
│  uops_issued: 12,000,000,000                                       │
│  uops_retired: 10,000,000,000                                      │
│  → 발급 효율 = 10B / 12B ≈ 83%                                       │
│                                                                     │
│  【발급 폭 최적화 팁】                                                   │
│  ──────────────────                                                  │
│  1. 명령어 스케줄링 최적화                                               │
│  2. 분기 예측 정확도 향상                                               │
│  3. 데이터 의존성 감소                                                  │
│  4. SIMD 명령어 활용                                                   │
│  5. 메모리 대역폭 확보                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  발급 폭별 권장 용도:                                                   │ │
│  │  ──────────────────                                                  │ │
│  │  발급 폭    용도                 대표 예시                            │ │
│  │  ───────  ──────────────────     ────────────────────              │ │
│  │  1~2      임베디드, 저전력         IoT, 웨어러블                     │ │
│  │  3~4      모바일, 일반용           스마트폰, 태블릿                   │ │
│  │  5~6      데스크탑, 노트북          일반 PC, 게임                     │ │
│  │  7~8      고성능 워크스테이션        전문가용, 서버                    │ │
│  │  8+       데이터센터, HPC           서버, 슈퍼컴퓨터                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 발급 폭: 한 클럭에 발급 가능한 최대 명령어 수
• 역할: IPC 이론적 상한선 결정
• 범위: 1 (스칼라) ~ 8+ (넓은 슈퍼스칼라)
• 비용: 발급 폭 증가 = 면적·전력·복잡도 증가
• 한계: 실제 IPC < 발급 폭 (해저드, 의존성)
• 트레이드오프: 성능 vs 효율
• 현대 CPU: 4~8 발급 폭이 일반적
```
