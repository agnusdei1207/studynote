# 211. 간접 사이클 (Indirect Cycle)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 간접 주소 지정 시 유효 주소를 구하는 추가 단계
> 2. **가치**: 메모리 공간 절약, 유연한 주소 지정
> 3. **융합**: 주소 지정 방식, 포인터, 다중 간접

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

간접 사이클(Indirect Cycle)은 **간접 주소 지정 시 유효 주소를 구하기 위한 추가 메모리 접근 단계**다. 명령어에 있는 주소가 실제 데이터의 주소가 아니라, 데이터 주소가 저장된 위치를 가리킬 때 필요하다.

### 💡 비유: 쪽지로 주소 찾기

간접 사이클은 **쪽지로 주소 찾기**와 같다. 배달부에게 "서류함 A에 있는 쪽지를 보고 거기 적힌 주소로 배달해"라고 했다. 먼저 서류함 A(간접 주소)를 열어 쪽지(유효 주소)를 보고, 그다음 실제 목적지로 간다.

### 간접 사이클 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                간접 사이클 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【직접 vs 간접 주소 지정】                                           │
│  ────────────────────────────────                                   │
│                                                                     │
│  직접 주소 지정:                                                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  LOAD 1000        ; Mem[1000] → AC                               │ │
│  │                  ; 1000 = 실제 데이터 주소                        │ │
│  │                  ; 메모리 접근 1회                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  간접 주소 지정:                                                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  LOAD @1000      ; Mem[Mem[1000]] → AC                          │ │
│  │                  ; 1000 = 주소가 저장된 위치                      │ │
│  │                  ; 메모리 접근 2회 (간접 + 실제)                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【간접 사이클 동작】                                                  │
│  ────────────────────────                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. MAR ← IR(addr)           ; 명령어에서 주소 필드              │ │
│  │  2. MBR ← M[MAR]              ; 간접 주소에서 유효 주소 읽기       │ │
│  │  3. IR(addr) ← MBR           ; 유효 주소로 명령어 필드 갱신        │ │
│  │  4. Execute Cycle 진입        ; 유효 주소로 실행                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【다중 간접】                                                        │
│  ──────────────────                                                  │
│  • 단일 간접: 1단계 간접                                              │
│  • 다중 간접: 여러 단계 간접                                          │
│  • 무한 루프 가능성                                                   │
│  • 해결: 간접 비트(indirect bit)로 구분                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  사이클 흐름:                                                       │ │
│  │  ──────────────────                                                │ │
│  │                                                               │ │
│  │  직접:  Fetch → Decode → Execute → Fetch → ...                  │ │
│  │                                                               │ │
│  │  간접:  Fetch → Decode → Indirect → Execute → Fetch → ...       │ │
│  │                                                               │ │
│  │  다중간접: Fetch → Decode → Indirect → ... → Execute            │ │
│  │                                                               │ │
│  │  → 간접 사이클은 추가 메모리 접근 필요                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 간접 사이클 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                간접 사이클 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                            │
│  ──────────────────                                                  │
│  1. 작은 주소 필드: 넓은 주소 공간 접근                               │
│  2. 동적 주소: 실행 시 주소 결정                                      │
│  3. 코드 공유: 포인터 변경으로 다른 데이터                            │
│  4. 간접 참조: 테이블, 배열 구현                                      │
│                                                                     │
│  【단점】                                                            │
│  ──────────────────                                                  │
│  1. 느림: 추가 메모리 접근                                           │
│  2. 복잡: 주소 계산 복잡                                              │
│  3. 해저드: 더 긴 파이프라인                                         │
│  4. 캐시: 간접 참조 시 캐시 미스                                      │
│                                                                     │
│  【현대 아키텍처에서】                                                 │
│  ──────────────────                                                  │
│  • x86: 간접 주소 지정 지원                                          │
│    MOV EAX, [EBX]          ; 레지스터 간접                           │
│    MOV EAX, [EBX+ECX*4]    ; 인덱스 간접                             │
│  • RISC: Load/Store만 간접                                           │
│    LW R0, 0(R1)            ; 베이스+변위                             │
│  • 포인터: 고수준 언어에서 간접 참조                                  │
│    int *p; *p = 10;                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  간접 참조 최적화:                                                  │ │
│  │  ──────────────────                                                │ │
│  │  1. 포인터 캐시: 간접 주소 캐싱                                    │ │
│  │  2. 프리페치: 간접 참조 미리 로드                                   │ │
│  │  3. 인라인: 직접 주소로 변환 (컴파일 시)                           │ │
│  │  4. SSA: 간접 참조 제거                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 간접 사이클 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                간접 사이클 분석                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【간접 참조 어셈블리】                                               │
│  ──────────────────────────                                         │
│  ; int x = *ptr;                                                    │
│  MOV RAX, [ptr]          ; ptr 값 (간접 주소) 로드                   │
│  MOV RBX, [RAX]          ; 실제 데이터 로드 (간접)                   │
│  MOV [x], RBX            ; 결과 저장                                 │
│                                                                     │
│  ; 연결 리스트 순회                                                   │
│  loop:                                                              │
│      MOV RAX, [node]       ; 현재 노드                               │
│      MOV RBX, [RAX+next]   ; 다음 노드 (간접)                        │
│      CMP RBX, 0                                                     │
│      JZ done                                                        │
│      MOV [node], RBX                                                │
│      JMP loop                                                       │
│  done:                                                              │
│                                                                     │
│  【캐시 성능 분석】                                                   │
│  ──────────────────                                                  │
│  $ perf stat -e cache-references,cache-misses ./ptr_test            │
│  → 포인터 추적(간접 참조)은 캐시 미스율 높음                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  간접 참조 최적화:                                                  │ │
│  │  ──────────────────                                                │ │
│  │  1. 데이터 지역성 향상                                              │ │
│  │  2. 포인터 압축 (32비트)                                            │ │
│  │  3. 구조체 배열 (AoS → SoA)                                        │ │
│  │  4. 프리페칭                                                        │ │
│  │  5. 캐시 친화적 자료구조                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

