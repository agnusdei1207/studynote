# 244. EPIC (Explicitly Parallel Instruction Computing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: VLIW의 한계를 극복한 명시적 병렬 명령어 아키텍처
> 2. **가치**: 컴파일러와 하드웨어의 협력, 동적 최적화 지원
> 3. **융합**: Intel Itanium, IA-64 아키텍처

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

EPIC(Explicitly Parallel Instruction Computing)은 **VLIW의 한계를 극복하고 컴파일러가 병렬성을 명시적으로 표현하는 아키텍처**이다. Intel과 HP가 공동 개발하여 Itanium(IA-64) 프로세서에 적용했다.

### 💡 비유: 오케스트라 지휘

EPIC은 **오케스트라 지휘**와 같다. 악보(컴파일러)에 각 악기가 언제 연결할지 명시되어 있다. 하지만 지휘자(하드웨어)가 상황에 따라 조정할 수도 있다. 완전히 고정된 것도 아니고, 완전히 자유로운 것도 아니다.

### EPIC vs VLIW

```
┌─────────────────────────────────────────────────────────────────────┐
│                EPIC vs VLIW                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【VLIW의 한계】                                                        │
│  ──────────────────                                                  │
│  • 고정된 명령어 스케줄: 새 CPU에서 바이너리 호환 안 됨                    │
│  • 캐시 미스 대응 불가: 컴파일 타임에 모든 것 결정                          │
│  • NOP 낭비: 빈 슬롯을 NOP으로 채움                                     │
│  • 런타임 정보 부재: 실행 시점 정보 활용 못 함                            │
│                                                                     │
│  【EPIC의 개선】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           VLIW                EPIC                        │ │
│  │  ──────────     ────────────────    ────────────────            │ │
│  │  스케줄링        완전 고정            컴파일러 + 하드웨어           │ │
│  │  바이너리 호환    X                  부분 지원                    │ │
│  │  캐시 미스        대응 불가           동적 대응                    │ │
│  │  명령어 그룹      고정 슬롯            가변 그룹                    │ │
│  │  프리디케이션     제한적              완전 지원                    │ │
│  │  추측 실행       제한적              완전 지원                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【EPIC 핵심 특징】                                                       │
│  ──────────────────                                                  │
│  1. 명시적 병렬성: 컴파일러가 병렬성을 명시                                 │
│  2. 프리디케이션: 조건부 실행으로 분기 제거                                 │
│  3. 추측 로드: 캐시 미스 숨김                                            │
│  4. 레지스터 회전: 소프트웨어 파이프라이닝 지원                              │
│  5. 번들 템플릿: 명령어 타입 정보 제공                                     │
│                                                                     │
│  【EPIC 명령어 형식 (IA-64)】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │              128비트 번들 (Bundle)                         │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │  슬롯0 (41비트)  │  슬롯1 (41비트)  │  슬롯2 (41비트)  │템플릿│ │
│  │  │                  │                  │                  │(5비트)│ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  │  템플릿:                                                       │ │
│  │  • 실행 유닛 타입 지정 (정수/FP/메모리/분기)                       │ │
│  │  • 스톱(stop) 정보: 명령어 그룹 경계                              │ │
│  │  • 병렬성 정보: 독립 실행 가능한 명령어 그룹                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. EPIC 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                EPIC 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프리디케이션 (Predication)】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분기 제거로 파이프라인 플러시 감소                                  │ │
│  │                                                               │ │
│  │  Before (분기 사용):                                             │ │
│  │    if (x > 0)                                                   │ │
│  │      y = a + b;                                                 │ │
│  │    else                                                         │ │
│  │      y = c - d;                                                 │ │
│  │                                                               │ │
│  │  After (프리디케이션):                                             │ │
│  │    p1 = (x > 0)                                                 │ │
│  │    (p1) y = a + b      ; p1이 참이면 실행                         │ │
│  │    (!p1) y = c - d     ; p1이 거짓이면 실행                        │ │
│  │                                                               │ │
│  │  → 두 경로 모두 실행, 조건에 따라 결과만 반영                          │ │
│  │  → 분기 예측 실패 없음                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【추측 로드 (Speculative Load)】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  캐시 미스를 미리 숨김                                              │ │
│  │                                                               │ │
│  │  일반 코드:                                                      │ │
│  │    if (ptr != NULL) {          ; 분기 예측                       │ │
│  │      x = *ptr;                 ; 캐시 미스 가능                   │ │
│  │    }                                                           │ │
│  │                                                               │ │
│  │  EPIC 추측 로드:                                                  │ │
│  │    ld.s r1 = [ptr]             ; 추측 로드 (미리 인출)             │ │
│  │    chk.s r1, recovery          ; 예외 발생 시 복구                 │ │
│  │    x = r1                                                       │ │
│  │                                                               │ │
│  │  → 로드를 분기 전에 수행                                           │ │
│  │  → 캐시 미스 시간을 다른 작업으로 숨김                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【레지스터 회전 (Register Rotation)】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  소프트웨어 파이프라이닝 지원                                        │ │
│  │                                                               │ │
│  │  루프: for (i=0; i<n; i++) a[i] = b[i] + c[i];                  │ │
│  │                                                               │ │
│  │  일반 파이프라인:                                                  │ │
│  │    LOAD b[i]                                                   │ │
│  │    LOAD c[i]                                                   │ │
│  │    ADD                                                         │ │
│  │    STORE a[i]                                                  │ │
│  │    (매 반복마다 의존성)                                            │ │
│  │                                                               │ │
│  │  회전 레지스터:                                                    │ │
│  │    r32 → r33 → r34 → ... (자동 회전)                             │ │
│  │    각 반복가 다른 레지스터 사용                                       │ │
│  │    → 의존성 없이 병렬 실행                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【EPIC 레지스터 파일】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  타입           개수       특징                                    │ │
│  │  ──────────     ──────     ──────────                            │ │
│  │  정수 레지스터    128        64 정적 + 64 회전                      │ │
│  │  부동소수점      128        64 정적 + 64 회전                       │ │
│  │  프리디케이트     64         1비트 조건 레지스터                      │ │
│  │  분기 레지스터    8          분기 주소 저장                          │ │
│  │  → 매우 큰 레지스터 파일                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. EPIC × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                EPIC × 타 과목 융합                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 파이프라인: 프리디케이션으로 분기 플러시 없음                             │
│  • 캐시: 추측 로드로 캐시 미스 숨김                                       │
│  • 레지스터: 회전 레지스터로 소프트웨어 파이프라이닝                           │
│  • 성능: 높은 IPC 달성 가능                                             │
│                                                                     │
│  【컴파일러】                                                          │
│  • 프리디케이션: if-conversion으로 분기 제거                              │
│  • 추측 실행: 안전한 코드 이동                                           │
│  • 소프트웨어 파이프라이닝: 루프 최적화                                     │
│  • 의존성 분석: 매우 정교한 분석 필요                                      │
│                                                                     │
│  【운영체제】                                                          │
│  • 컨텍스트 스위치: 큰 레지스터 파일 저장                                   │
│  • 예외 처리: 추측 실행 예외 복구                                        │
│  • 성능: EPIC 특성 고려 스케줄링                                         │
│                                                                     │
│  【시장 결과】                                                         │
│  • Intel Itanium: 2001~2021 (상업적 실패)                               │
│  • 원인: x86 호환성, 컴파일러 미성숙, 경쟁심화                               │
│  • 교훈: 컴파일러 기술이 하드웨어보다 느리게 발전                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  EPIC vs 슈퍼스칼라:                                                      │ │
│  │  ──────────────────────                                              │ │
│  │  특성          EPIC              슈퍼스칼라                           │ │
│  │  ──────────    ──────────        ──────────                          │ │
│  │  스케줄링       컴파일러           하드웨어                            │ │
│  │  하드웨어       단순               복잡                               │ │
│  │  전력          낮음               높음                                │ │
│  │  호환성        낮음               높음                                │ │
│  │  시장          실패               성공                               │ │
│  │  → EPIC은 기술적으로 우수하지만 시장에서 실패                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### EPIC 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                EPIC 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Itanium 프로세서 역사】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  연도    모델            특징                                     │ │
│  │  ────    ──────────      ──────────────────                      │ │
│  │  2001   Itanium         최초 IA-64, 733~800MHz                   │ │
│  │  2002   Itanium 2       개선된 캐시, 900MHz~1GHz                  │ │
│  │  2006   Itanium 2 9000  듀얼코어, 하이퍼스레딩                       │ │
│  │  2008   Itanium 9100    쿼드코어                                  │ │
│  │  2010   Itanium 9300    8코어, 24MB 캐시                          │ │
│  │  2017   Itanium 9700    마지막 모델                               │ │
│  │  2021   단종            지원 종료                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【EPIC 기술의 유산】                                                      │
│  ──────────────────                                                  │
│  • 프리디케이션: ARM, RISC-V에서 조건부 실행                               │
│  • 추측 실행: 현대 CPU에서 투기적 실행                                    │
│  • 대형 레지스터 파일: ARM SVE, RISC-V 벡터 확장                          │
│  • 컴파일러 기술: 의존성 분석, 자동 병렬화                                 │
│                                                                     │
│  【EPIC 실패 원인 분석】                                                   │
│  ──────────────────                                                  │
│  1. x86 호환성: x86 에뮬레이션 성능 저하                                  │
│  2. 컴파일러: 초기 컴파일러 품질 낮음                                      │
│  3. 경쟁: x86-64, ARM의 빠른 발전                                       │
│  4. 비용: 높은 개발 비용, 낮은 수익                                       │
│  5. 타이밍: 시장 변화에 늦게 대응                                          │
│                                                                     │
│  【EPIC 교훈】                                                            │
│  ──────────────────                                                  │
│  • 컴파일러 기술 < 하드웨어 발전 속도                                      │
│  • 호환성 > 이론적 성능                                                  │
│  • 시장 적시성 중요                                                     │
│  • 단순한 아키텍처가 복잡한 것을 이길 수 있음                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• EPIC: VLIW를 개선한 명시적 병렬 명령어 아키텍처
• 개발: Intel + HP (IA-64)
• 특징: 프리디케이션, 추측 로드, 레지스터 회전
• 장점: 높은 IPC, 저전력, 분기 없는 실행
• 단점: 컴파일러 복잡, 호환성 문제
• 결과: Itanium (2001~2021) 상업적 실패
• 유산: 프리디케이션, 추측 실행 기술은 현대 CPU에 영향
• 교훈: 이론적 우수성 ≠ 시장 성공
```
