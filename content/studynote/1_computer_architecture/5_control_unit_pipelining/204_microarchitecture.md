# 204. 마이크로아키텍처 (Microarchitecture)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: ISA의 하드웨어 구현 방식, 성능 결정
> 2. **가치**: 동일 ISA라도 마이크로아키텍처에 따라 성능 차이
> 3. **융합**: 파이프라인, 캐시, 분기 예측, Out-of-Order

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

마이크로아키텍처는 **ISA(명령어 집합)를 실제 하드웨어로 구현하는 방식**이다. 동일한 ISA라도 마이크로아키텍처에 따라 성능, 전력, 비용이 크게 달라진다. ISA는 "무엇을 할지", 마이크로아키텍처는 "어떻게 할지"를 정의한다.

### 💡 비유: 요리 레시피 vs 주방 설계

ISA는 **요리 레시피**다. "스테이크를 만들어라"라고 정의한다. 마이크로아키텍처는 **주방 설계**다. 가스레인지를 쓸지, 전자레인지를 쓸지, 요리사가 몇 명일지 결정한다. 같은 레시피도 주방에 따라 속도와 품질이 다르다.

## Ⅱ. 마이크로아키텍처 상세

### 구조 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                마이크로아키텍처 상세                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데이터패스 구조】                                                 │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │    ┌─────────┐    ┌─────────┐    ┌─────────┐                 │ │
│  │    │ 명령어   │───▶│ 레지스터 │───▶│   ALU   │                 │ │
│  │    │ 캐시    │    │  파일   │    │         │                 │ │
│  │    └─────────┘    └─────────┘    └────┬────┘                 │ │
│  │         ▲                              │                       │ │
│  │         │         ┌─────────┐          │                       │ │
│  │         └─────────│  데이터  │◀─────────┘                      │ │
│  │                   │  캐시   │                                   │ │
│  │                   └─────────┘                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파이프라인 단계 (5단계)】                                         │
│  ──────────────────────────                                         │
│  IF  (Instruction Fetch)    : 명령어 인출                           │
│  ID  (Instruction Decode)   : 명령어 해독                           │
│  EX  (Execute)              : 연산 실행                             │
│  MEM (Memory Access)        : 메모리 접근                           │
│  WB  (Write Back)           : 결과 기록                             │
│                                                                     │
│  【비순차 실행 (Out-of-Order)】                                      │
│  ────────────────────────────────                                   │
│  순차 프로그램:  I1 → I2 → I3 → I4                                  │
│  순차 발급:      I1 → I2 → I3 → I4                                  │
│  비순차 실행:    I1 → I3 → I2 → I4  (I2 대기 중 I3 먼저)            │
│  순차 완료:      I1 → I2 → I3 → I4                                  │
│                                                                     │
│  【주요 기술】                                                       │
│  ──────────────────                                                  │
│  • 레지스터 리네이밍: WAW/WAR 해저드 해결                            │
│  • 토마술로 알고리즘: 비순차 실행 스케줄링                           │
│  • 분기 예측: 제어 해저드 완화                                       │
│  • 데이터 포워딩: 데이터 해저드 해결                                 │
│  • 스펙클레이티브 실행: 추측 실행                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Intel Core vs AMD Zen vs Apple M1:                                │ │
│  │  ────────────────────────────────────                              │ │
│  │  특성         Core i9    Ryzen 9    M1 Max                        │ │
│  │  ──────────   ─────────  ─────────   ────────────                 │ │
│  │  코어          24        16          10 (8P+2E)                   │ │
│  │  스레드        32        32          10                           │ │
│  │  클럭          6.0GHz    5.7GHz      3.2GHz                       │ │
│  │  IPC           높음      높음        매우 높음                    │ │
│  │  TDP           125W      170W        30W                          │ │
│  │  공정          Intel 7   TSMC 5nm    TSMC 5nm                     │ │
│  │                                                               │ │
│  │  → M1은 낮은 클럭, 낮은 전력으로 경쟁                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  • Retpoline: 간접 분기 완화                                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  IPC 향상 기술:                                                     │
│  │  ──────────────────                                                │ │
│  │  1. 더 깊은 파이프라인: 더 높은 클럭                              │ │
│  │  2. 더 넓은 발급: 더 많은 병렬성                                  │ │
│  │  3. 더 큰 캐시: 더 낮은 미스율                                    │ │
│  │  4. 더 나은 분기 예측: 더 적은 페널티                             │ │
│  │  5. 비순차 실행: 더 높은 활용률                                   │ │
│  │                                                               │ │
│  │  → 물리적 한계로 IPC 향상 둔화                                     │ │
│  │  → 멀티코어로 전환                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
