# 214. 하드와이어드 제어 (Hardwired Control)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고정된 디지털 회로로 제어 신호 생성
> 2. **가치**: 빠름, 단순, RISC에 적합
> 3. **융합**: 게이트, 디코더, PLA, FSM

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

하드와이어드 제어(Hardwired Control)는 **고정된 디지털 논리 회로(게이트, 디코더, PLA)로 제어 신호를 생성하는 방식**이다. 마이크로프로그래밍보다 빠르지만, 수정이 어렵고 유연성이 낮다.

### 💡 비유: 고정 회로 교통신호

하드와이어드 제어는 **고정 회로 교통신호**와 같다. 신호등이 타이머와 센서에 따라 미리 정해진 대로 작동한다. 시스템을 바꾸려면 회로 자체를 뜯어고쳐야 한다. 빠르지만 유연하지 않다.

### 하드와이어드 제어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                하드와이어드 제어 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구성 요소】                                                        │
│  ──────────────────                                                  │
│  1. 명령어 레지스터 (IR): Opcode 제공                                │
│  2. 디코더: Opcode → 제어 신호 매핑                                   │
│  3. 타이밍 유닛: T1, T2, T3, ... 생성                                 │
│  4. 제어 로직: 게이트, PLA, ROM                                      │
│  5. 상태 플래그: Zero, Carry, ...                                    │
│                                                                     │
│  【기본 구조】                                                        │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                            제어 유닛                             │ │
│  │  ┌─────┐      ┌─────────┐      ┌───────────┐                   │ │
│  │  │ IR  │─────▶│ 디코더   │─────▶│ 제어 로직 │─────▶ 제어 신호  │ │
│  │  └─────┘      │ (Opcode)│      │ (게이트)  │                   │ │
│  │               └─────────┘      └─────┬─────┘                   │ │
│  │                                      ▲                          │ │
│  │               ┌─────────┐           │                          │ │
│  │               │ 타이밍   │───────────┘                          │ │
│  │               │  유닛   │                                      │ │
│  │               └─────────┘                                      │ │
│  │                    ▲                                           │ │
│  │                    │ 클럭                                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【타이밍 신호】                                                      │
│  ──────────────────                                                  │
│  클럭 → 카운터 → T0, T1, T2, T3, T4, T5, T6, T7                    │
│                                                                     │
│  【제어 신호 생성】                                                   │
│  ──────────────────                                                  │
│  제어 신호 = f(Opcode, Timing, Flags)                               │
│                                                                     │
│  예: ADD 명령어의 T4 시                                               │
│  ALUOp = ADD                                                         │
│  RegWrite = 1                                                        │
│  RegDst = 1                                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  하드와이어드 vs 마이크로프로그래밍:                                   │ │
│  │  ───────────────────────────────────                               │ │
│  │  특성        하드와이어드      마이크로프로그래밍                   │ │
│  │  ──────────  ───────────────  ────────────────────                │ │
│  │  속도        빠름             느림                                 │ │
│  │  유연성      낮음             높음                                │ │
│  │  수정        어려움           쉬움                                │ │
│  │  복잡도      단순             복잡                                │ │
│  │  적용        RISC            CISC                                │ │
│  │  예          MIPS, RISC-V     x86                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 하드와이어드 제어 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                하드와이어드 제어 분석                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RISC vs CISC 제어 방식】                                           │
│  ────────────────────────────                                        │
│                                                                     │
│  RISC (MIPS, RISC-V, ARM):                                         │
│  • 하드와이어드 제어                                                  │
│  • 단순한 명령어 (100개 미만)                                         │
│  • 고정 길이 명령어                                                   │
│  • 빠른 디코딩                                                        │
│                                                                     │
│  CISC (x86):                                                         │
│  • 마이크로프로그래밍                                                 │
│  • 복잡한 명령어 (1000개 이상)                                        │
│  • 가변 길이 명령어                                                   │
│  • μOp 변환                                                          │
│                                                                     │
│  【Verilog 구현 예시】                                                │
│  ──────────────────                                                  │
│  module control_unit(                                               │
│      input [5:0] opcode,                                            │
│      input [2:0] timing,                                            │
│      output reg reg_write,                                          │
│      output reg [1:0] alu_op                                        │
│  );                                                                  │
│      always @(*) begin                                              │
│          case({opcode, timing})                                     │
│              {6'b000000, 3'b100}: begin // R-type T4               │
│                  reg_write = 1;                                     │
│                  alu_op = 2'b10;                                    │
│              end                                                     │
│              // ...                                                  │
│          endcase                                                     │
│      end                                                              │
│  endmodule                                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  하드와이어드 최적화:                                                │ │
│  │  ──────────────────                                                │ │
│  │  1. 파이프라인 제어: 각 단계별 제어                                 │ │
│  │  2. 분산 제어: 각 기능별 제어 유닛                                  │ │
│  │  3. ROM 하이브리드: 일부 ROM 활용                                  │ │
│  │  4. 자동 생성: HDL 합성                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
