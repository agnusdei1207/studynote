# 220. 파이프라인 깊이 (Pipeline Depth)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파이프라인 단계의 개수
> 2. **가치**: 깊을수록 높은 클럭, 얕을수록 낮은 지연
> 3. **융합**: 클럭 주파수, 분기 페널티, 전력 소모

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

파이프라인 깊이(Pipeline Depth)는 **파이프라인을 구성하는 단계의 수**를 의미한다. 깊이가 깊을수록 각 단계의 논리가 단순해져 높은 클럭 주파수를 달성할 수 있지만, 분기 페널티와 파이프라인 플러시 비용이 증가한다.

### 💡 비유: 고속도로 톨게이트

파이프라인 깊이는 **고속도로 톨게이트 수**와 같다. 톨게이트가 많으면 각 구간 통과가 빠르다(높은 클럭). 하지만 한 번 잘못 들면 빠져나오는 데 오래 걸린다(분기 페널티).

### 파이프라인 깊이 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 깊이 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【얕은 파이프라인 (Shallow Pipeline)】                                │
│  ────────────────────────────────────                                │
│  ┌────────────────────────────────────────────────────────────────┐│
│  │  5단계: IF → ID → EX → MEM → WB                                 ││
│  │                                                               ││
│  │  장점: 분기 페널티 작음 (최대 4클럭)                              ││
│  │       파이프라인 플러시 비용 낮음                                  ││
│  │       전력 소모 적음                                             ││
│  │                                                               ││
│  │  단점: 낮은 클럭 주파수                                           ││
│  │       각 단계의 논리가 복잡                                        ││
│  │  예: MIPS (5단계), ARM7 (3단계)                                   ││
│  └────────────────────────────────────────────────────────────────┘│
│                                                                     │
│  【깊은 파이프라인 (Deep Pipeline)】                                   │
│  ──────────────────────────────                                      │
│  ┌────────────────────────────────────────────────────────────────┐│
│  │  14단계: IF1→IF2→ID1→ID2→EX1→EX2→EX3→MEM1→MEM2→...             ││
│  │                                                               ││
│  │  장점: 높은 클럭 주파수                                           ││
│  │       각 단계의 논리가 단순                                        ││
│  │       처리량 향상                                                 ││
│  │                                                               ││
│  │  단점: 분기 페널티 큼 (최대 13클럭)                                 ││
│  │       파이프라인 플러시 비용 높음                                   ││
│  │       전력 소모 많음                                              ││
│  │  예: Intel Pentium 4 (31단계), Skylake (14~19단계)               ││
│  └────────────────────────────────────────────────────────────────┘│
│                                                                     │
│  【깊이에 따른 트레이드오프】                                           │
│  ────────────────────────                                            │
│                                                                     │
│  깊이    클럭    분기 페널티    전력      IPC                          │
│  ────   ────   ──────────    ────     ────                           │
│  5      낮음   작음(4)       적음      높음                           │
│  10     중간   중간(9)       중간      중간                           │
│  15     높음   큼(14)        많음      낮음                           │
│  31     매우높 매우큼(30)    매우많음  매우낮음                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  최적 깊이:                                                           │ │
│  │  ──────────────────                                                  │ │
│  │  • 명령어 집합, 분기 빈도, 전력 예산 고려                             │ │
│  │  • 현대 CPU: 10~20단계가 일반적                                       │ │
│  │  • Intel: 14~19단계 (Skylake)                                        │ │
│  │  • AMD: 19단계 (Zen 3)                                               │ │
│  │  • ARM: 11~15단계 (Cortex-A78)                                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 파이프라인 깊이 상세

### 깊이별 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 깊이 상세                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【깊은 파이프라인의 단계 분할】                                        │
│  ──────────────────────────                                          │
│                                                                     │
│  MIPS 5단계:                                                         │
│  IF → ID → EX → MEM → WB                                            │
│                                                                     │
│  Intel NetBurst (Pentium 4) 31단계:                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  IF: Trace Cache Fetch (L1)                                    │ │
│  │  → Instruction Decode (2단계)                                   │ │
│  │  → Register Rename (3단계)                                      │ │
│  │  → uOp Scheduling (5단계)                                       │ │
│  │  → Execution (9단계)                                            │ │
│  │  → Memory (8단계)                                               │ │
│  │  → Writeback (4단계)                                            │ │
│  │  → 총 31단계                                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【클럭 주파수와 깊이】                                                │
│  ────────────────────                                                │
│                                                                     │
│  클럭 주기 = 단계당 지연 (가장 느린 단계)                               │
│  깊이↑ = 단계당 지연↓ = 클럭 주기↓ = 클럭 주파수↑                       │
│                                                                     │
│  예:                                                                 │
│  • 5단계, 단계당 200ps → 클럭 200ps = 5 GHz                          │
│  • 10단계, 단계당 100ps → 클럭 100ps = 10 GHz                        │
│  • 20단계, 단계당 50ps → 클럭 50ps = 20 GHz                          │
│                                                                     │
│  【분기 페널티】                                                       │
│  ──────────────────                                                  │
│                                                                     │
│  분기 페널티 = 깊이 - 1 (최악의 경우)                                  │
│                                                                     │
│  5단계: 페널티 = 4클럭                                                │
│  14단계: 페널티 = 13클럭                                              │
│  31단계: 페널티 = 30클럭                                              │
│                                                                     │
│  → 깊을수록 분기 예측의 중요성 증가                                    │
│                                                                     │
│  【IPC(Instructions Per Cycle)】                                      │
│  ─────────────────────────────                                       │
│                                                                     │
│  이상적 IPC = 1 (파이프라인)                                          │
│  실제 IPC = 1 - (분기 미스율 × 페널티 / 명령어당 분기 수)               │
│                                                                     │
│  예: 20% 분기, 10% 미스, 13클럭 페널티                                 │
│  IPC = 1 - (0.2 × 0.1 × 13) = 1 - 0.26 = 0.74                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Pentium 4 (NetBurst) 실패 원인:                                     │ │
│  │  ────────────────────────────────                                    │ │
│  │  • 31단계 너무 깊음                                                   │ │
│  │  • 분기 미스 시 20~30클럭 손실                                        │ │
│  │  • 높은 전력 소모                                                     │ │
│  │  • 실제 성능는 Core 2(14단계)보다 낮음                                │ │
│  │  → 이후 Intel은 깊이 줄임                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 파이프라인 깊이 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 깊이 × 타 과목 융합                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 클럭 주파수: 깊이↑ → 주파수↑                                       │
│  • CPI: 깊이↑ → 분기 페널티↑ → CPI↑                                   │
│  • IPC: 깊이↑ → 분기 미스 영향↑                                       │
│  • 성능: 클럭 × IPC의 균형 필요                                        │
│                                                                     │
│  【운영체제】                                                          │
│  • 컨텍스트 스위치: 플러시 비용 = 깊이                                  │
│  • 스케줄링: 파이프라인 깊이 고려                                      │
│  • 인터럽트 지연: 깊을수록 지연↑                                       │
│                                                                     │
│  【컴파일러】                                                          │
│  • 분기 예측 최적화: 깊을수록 중요                                      │
│  • 루프 최적화: 분기 감소                                              │
│  • 인라이닝: 함수 호출 오버헤드(파이프라인 플러시) 감소                   │
│                                                                     │
│  【보안】                                                              │
│  • Spectre: 깊은 파이프라인에서 더 위험                                │
│  • 추측 실행: 깊을수록 추측 범위↑                                      │
│  • 사이드 채널: 파이프라인 상태 노출                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  파이프라인 깊이 역사:                                                 │ │
│  │  ──────────────────                                                  │ │
│  │  • 486: 5단계                                                        │ │
│  │  • Pentium: 5단계 (슈퍼스칼라)                                         │ │
│  │  • Pentium Pro: 10~12단계                                            │ │
│  │  • Pentium 4: 31단계 (실패)                                           │ │
│  │  • Core 2: 14단계 (균형)                                              │ │
│  │  • Skylake: 14~19단계                                                 │ │
│  │  • Zen 3: 19단계                                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 파이프라인 깊이 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 깊이 분석                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CPU 아키텍처별 깊이】                                               │
│  ────────────────────────                                            │
│  $ lscpu | grep "Model name"                                        │
│  Model name: Intel(R) Core(TM) i7-10700K                            │
│                                                                     │
│  아키텍처별 파이프라인 깊이:                                           │
│  ────────────────────────                                            │
│  • ARM Cortex-A53: 8단계 (in-order)                                 │
│  • ARM Cortex-A78: 11단계 (out-of-order)                            │
│  • Apple M1: 15단계 (out-of-order)                                  │
│  • Intel Skylake: 14~19단계                                         │
│  • AMD Zen 3: 19단계                                                │
│  • Intel Pentium 4: 31단계 (역사적)                                   │
│                                                                     │
│  【분기 예측 중요성】                                                  │
│  ──────────────────                                                  │
│  $ perf stat -e branches,branch-misses ./prog                       │
│  branches: 1,000,000,000                                            │
│  branch-misses: 5,000,000 (0.5%)                                    │
│                                                                     │
│  깊이 19, 0.5% 미스, 18클럭 페널티:                                    │
│  손실 = 1,000,000,000 × 0.005 × 18 = 90,000,000 클럭                 │
│  → 분기 예측 정확도가 성능 결정                                        │
│                                                                     │
│  【전력 소모】                                                         │
│  ──────────────────                                                  │
│  $ perf stat -e power/energy-cores/ ./prog                          │
│                                                                     │
│  깊은 파이프라인:                                                     │
│  • 파이프라인 레지스터 수↑ → 전력↑                                     │
│  • 클럭 주파수↑ → 전력↑                                               │
│  • 플러시 빈도↑ → 전력↑                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  파이프라인 깊이 최적화:                                              │ │
│  │  ──────────────────────                                              │ │
│  │  1. 분기 예측 정확도 향상                                             │ │
│  │  2. 함수 인라이닝 (분기 감소)                                        │ │
│  │  3. 루프 언롤링 (분기 감소)                                          │ │
│  │  4. 프로파일 기반 최적화 (PGO)                                       │ │
│  │  5. 전력 효율 고려한 깊이 선택                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 파이프라인 깊이: 파이프라인 단계의 수
• 얕은 파이프라인: 낮은 클럭, 작은 분기 페널티
• 깊은 파이프라인: 높은 클럭, 큰 분기 페널티
• 트레이드오프: 클럭 vs IPC vs 전력
• 분기 페널티: 깊이 - 1
• 현대 CPU: 10~20단계가 일반적
• Pentium 4 교훈: 너무 깊은 파이프라인은 비효율
```
