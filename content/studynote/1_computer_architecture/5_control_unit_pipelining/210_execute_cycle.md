# 210. 실행 사이클 (Execute Cycle)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실제 연산 수행, ALU/Memory/PC 처리
> 2. **가치**: 명령어의 핵심 작업 수행, 파이프라인 EX 단계
> 3. **융합**: ALU, FPU, 분기, 메모리 접근

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

실행 사이클(Execute Cycle)은 **해독된 명령어를 실제로 수행하는 단계**다. ALU 연산, 메모리 접근, 분기, 레지스터 쓰기 등 명령어의 핵심 작업이 이 단계에서 수행된다.

### 💡 비유: 요리 실행

실행 사이클은 **실제 요리 과정**과 같다. 레시피를 보고(해독), 이제 실제로 재료를 자르고, 볶고, 끓인다(실행). 모든 준비가 실행을 위한 것이다.

### 실행 사이클 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                실행 사이클 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【명령어 유형별 실행】                                               │
│  ──────────────────────                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  R-type (ADD, SUB, AND, OR, SLT):                                  │ │
│  │  ─────────────────────────────────                                  │ │
│  │  ALU 연산: result = operand1 op operand2                           │ │
│  │  예: $t0 = $t1 + $t2                                              │ │
│  │                                                               │ │
│  │  마이크로 오퍼레이션:                                                 │ │
│  │  1. ALU_A ← Read Data 1                                            │ │
│  │  2. ALU_B ← Read Data 2                                            │ │
│  │  3. ALUOut ← ALU_A op ALU_B                                        │ │
│  │  4. Reg[rd] ← ALUOut                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Load/Store (LW, SW):                                               │ │
│  │  ────────────────────────────                                       │ │
│  │  주소 계산: addr = base + offset                                   │ │
│  │  Load: data ← Mem[addr]                                            │ │
│  │  Store: Mem[addr] ← data                                           │ │
│  │                                                               │ │
│  │  마이크로 오퍼레이션:                                                 │ │
│  │  1. ALU_A ← Read Data 1 (base)                                     │ │
│  │  2. ALU_B ← SignExt(imm)                                           │ │
│  │  3. ALUOut ← ALU_A + ALU_B (주소)                                   │ │
│  │  4. Load: MDR ← Mem[ALUOut]                                        │ │
│  │     Store: Mem[ALUOut] ← Read Data 2                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Branch (BEQ, BNE):                                                 │ │
│  │  ──────────────────────                                             │ │
│  │  비교: Zero = (operand1 == operand2)                               │ │
│  │  분기: if (Zero) PC = PC + offset                                  │ │
│  │                                                               │ │
│  │  마이크로 오퍼레이션:                                                 │ │
│  │  1. ALU_A ← Read Data 1                                            │ │
│  │  2. ALU_B ← Read Data 2                                            │ │
│  │  3. Zero ← (ALU_A == ALU_B)                                        │ │
│  │  4. if (Zero & Branch) PC ← PC + SignExt(offset) << 2             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Jump (J, JAL):                                                     │ │
│  │  ──────────────────                                                 │ │
│  │  PC ← target_address                                               │ │
│  │  JAL: $ra ← PC+4                                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파이프라인 EX 단계】                                               │
│  ──────────────────────                                              │
│  입력: ID/EX 레지스터                                                 │
│  • Read Data 1, Read Data 2                                         │
│  • Sign-extended immediate                                          │
│  • 제어 신호 (ALUOp, ALUSrc, ...)                                    │
│  • PC+4                                                              │
│                                                                     │
│  출력: EX/MEM 레지스터                                                │
│  • ALU 결과                                                          │
│  • Zero 플래그                                                        │
│  • 쓰기 데이터 (Store용)                                             │
│  • 쓰기 레지스터 번호                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  EX 단계 구성:                                                      │ │
│  │  ──────────────                                                     │ │
│  │  • ALU: 산술/논리/비교 연산                                          │ │
│  │  • MUX: ALU 입력 선택 (레지스터 vs 즉시값)                          │ │
│  │  • 분기 로직: Zero & Branch → PCSrc                                │ │
│  │  • 포워딩 멀티플렉서: 데이터 해저드 해결                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 실행 사이클 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                실행 사이클 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ALU 연산 상세】                                                     │
│  ──────────────────                                                  │
│  ALU Control  연산     설명                                           │
│  ───────────  ──────  ──────────────────                              │
│  0000         AND     A & B                                          │
│  0001         OR      A | B                                          │
│  0010         ADD     A + B                                          │
│  0110         SUB     A - B                                          │
│  0111         SLT     (A < B) ? 1 : 0                                │
│  1100         NOR     ~(A | B)                                        │
│                                                                     │
│  【포워딩 (Forwarding)】                                              │
│  ──────────────────────────                                          │
│  데이터 해저드 해결을 위해 EX 단계에서 포워딩:                        │
│                                                                     │
│  Forward A 선택:                                                      │
│  • 00: ID/EX.Read Data 1 (정상)                                      │
│  • 01: EX/MEM.ALU Result (이전 명령어)                                │
│  • 10: MEM/WB.ALU Result (두 번째 이전)                              │
│                                                                     │
│  Forward B 선택:                                                      │
│  • 00: ID/EX.Read Data 2 (정상)                                      │
│  • 01: EX/MEM.ALU Result                                            │
│  • 10: MEM/WB.ALU Result                                            │
│                                                                     │
│  【분기 계산】                                                        │
│  ──────────────────                                                  │
│  Branch Target = PC + (SignExt(offset) << 2)                        │
│  Zero = (Read Data 1 == Read Data 2)                                │
│  PCSrc = Branch & Zero                                              │
│                                                                     │
│  【슈퍼스칼라 EX】                                                     │
│  ──────────────────                                                  │
│  • 다중 ALU: 여러 연산 동시 수행                                      │
│  • 다중 실행 유닛: ALU, FPU, 브랜치                                  │
│  • 발급 로직: 어떤 유닛에 명령어를 보낼지                             │
│  • 예약역: 명령어 대기                                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  EX 단계 지연:                                                      │ │
│  │  ──────────────                                                     │ │
│  │  • ALU: 1 클럭 (정수)                                               │ │
│  │  • MUL/DIV: 3~32 클럭                                               │ │
│  │  • FPU: 4~20 클럭                                                   │ │
│  │  • 분기: 1 클럭 (예측 성공 시)                                       │ │
│  │                                                               │ │
│  │  → 복잡한 연산은 여러 클럭 소요                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실행 사이클 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                실행 사이클 × 타 과목 융합                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 파이프라인: EX 단계                                               │
│  • 포워딩: 데이터 해저드 해결                                         │
│  • 비순차 실행: ROB, Reservation Station                            │
│  • 분기 예측: EX에서 분기 결과 확정                                  │
│                                                                     │
│  【디지털 논리회로】                                                  │
│  • ALU 설계: 가산기, 비교기                                          │
│  • MUX: 입력 선택                                                    │
│  • 제어 로직: 연산 선택                                               │
│  • 타이밍: 크리티컬 패스                                              │
│                                                                     │
│  【알고리즘】                                                         │
│  • 연산 복잡도: EX 단계 비용                                          │
│  • 상수 시간: 단일 ALU 연산                                           │
│  • 최적화: 연산 수 최소화                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  EX 단계 성능 향상:                                                 │ │
│  │  ──────────────────                                                │ │
│  │  1. 빠른 ALU (캐리 예측 가산기)                                     │ │
│  │  2. 다중 실행 유닛                                                  │ │
│  │  3. 효율적 포워딩                                                   │ │
│  │  4. 분기 예측 정확도 향상                                           │ │
│  │  5. 비순차 실행                                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
