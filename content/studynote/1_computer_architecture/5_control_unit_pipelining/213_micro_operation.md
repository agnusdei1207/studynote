# 213. 마이크로 오퍼레이션 (Micro-operation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU의 가장 기본적인 하드웨어 동작 단위
> 2. **가치**: 레지스터 전송, 산술/논리 연산, 메모리 접근
> 3. **융합**: 제어 유닛, 마이크로코드, 파이프라인

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

마이크로 오퍼레이션(Micro-operation, μOp)은 **CPU가 한 클럭 사이클에 수행할 수 있는 가장 기본적인 하드웨어 동작**이다. 레지스터 간 데이터 전송, ALU 연산, 메모리 접근 등이 μOp에 해당한다.

### 💡 비유: 요리의 개별 동작

마이크로 오퍼레이션은 **요리의 개별 동작**과 같다. "야채 썰기", "프라이패", "접시에 담기" 같은 가장 작은 단위의 동작이다. 이 동작들이 모여서 하나의 요리(명령어)가 완성된다.

### μOp 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                μOp 구조                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【μOp 유형】                                                         │
│  ────────────                                                        │
│  1. 전송 (Transfer): R1 ← R2                                        │
│  2. 산술 (Arithmetic): R1 ← R1 + 1                                  │
│  3. 논리 (Logic): R1 ← R1 AND R2                                    │
│  4. 시프트 (Shift): R1 ← R1 << 1                                    │
│  5. 메모리 (Memory): R1 ← M[AR]                                     │
│  6. 제어 (Control): PC ← 주소                                       │
│                                                                     │
│  【μOp 기본 형식】                                                    │
│  ──────────────────                                                  │
│  목적지 ← 출발지        ; 전송                                        │
│  예: MAR ← PC           ; PC 값을 MAR로 전송                         │
│  예: MBR ← M[MAR]       ; 메모리에서 MBR로 전송                      │
│  예: AC ← AC + DR       ; AC와 DR을 더해서 AC에 저장                 │
│                                                                     │
│  【명령어별 μOp 시퀀스】                                              │
│  ──────────────────────                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ADD (누산기 기계):                                               │ │
│  │  ───────────────────                                               │ │
│  │  T1: MAR ← PC                  ; 명령어 주소                       │ │
│  │  T2: MBR ← M[MAR], PC ← PC+1   ; 명령어 읽기                       │ │
│  │  T3: IR ← MBR                  ; 명령어 해독                       │ │
│  │  T4: MAR ← IR(addr)            ; 데이터 주소                       │ │
│  │  T5: MBR ← M[MAR]              ; 데이터 읽기                       │ │
│  │  T6: AC ← AC + MBR             ; 덧셈 수행                         │ │
│  │                                                               │ │
│  │  LOAD: T1-T5 + AC ← MBR                                          │ │
│  │  STORE: T1-T4 + M[MAR] ← AC                                      │ │
│  │  JUMP: T1-T3 + PC ← IR(addr)                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【μOp과 제어 신호】                                                   │
│  ──────────────────                                                  │
│  μOp              제어 신호들                                         │
│  ───────────────  ─────────────────                                  │
│  MAR ← PC         PCout, MARin, Read                                │
│  MBR ← M[MAR]     MEMread, MBRin                                    │
│  IR ← MBR          MBRout, IRin                                     │
│  PC ← PC + 1       PCout, ALUadd, PCin                              │
│  AC ← AC + MBR     ACout, MBRout, ALUadd, ACin                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  μOp vs 명령어:                                                     │ │
│  │  ───────────────────                                                │ │
│  │  • 명령어: 소프트웨어가 보는 단위 (ADD, LOAD, ...)                  │ │
│  │  • μOp: 하드웨어가 수행하는 단위 (전송, 연산)                        │ │
│  │                                                               │ │
│  │  하나의 명령어 = 여러 μOp의 시퀀스                                  │ │
│  │  예: LOAD = 5~6개 μOp                                              │ │
│  │      STORE = 5~6개 μOp                                             │ │
│  │      ADD = 6~7개 μOp                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. μOp × 제어 유닛

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                μOp × 제어 유닛                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【제어 신호 생성】                                                   │
│  ──────────────────                                                  │
│  제어 유닛은 각 μOp에 필요한 제어 신호를 생성:                        │
│                                                                     │
│  μOp: MAR ← PC                                                       │
│  • PCout: PC를 내부 버스에 출력                                      │
│  • MARin: MAR가 버스에서 입력                                        │
│  • Read: 메모리 읽기 시작                                             │
│                                                                     │
│  【내부 버스 구조】                                                   │
│  ──────────────────                                                  │
│                                                                     │
│          ┌─────────────────────────────────────────┐               │
│          │           내부 버스                      │               │
│          └──────────────┬──────────────────────────┘               │
│                         │                                           │
│    ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐                    │
│    │  PC  │ │ MAR  │ │ MBR  │ │  IR  │ │  AC  │                    │
│    └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘                    │
│       │        │        │        │        │                          │
│    Out/In   In      In/Out    In      In/Out                       │
│                                                                     │
│  【타이밍 신호】                                                      │
│  ──────────────────                                                  │
│  T1  T2  T3  T4  T5  T6  ...                                        │
│  ┌───┬───┬───┬───┬───┬───┐                                         │
│  │   │   │   │   │   │   │  각 타임 슬롯에서                         │
│  └───┴───┴───┴───┴───┴───┘  해당 μOp 활성화                          │
│                                                                     │
│  【μOp 스케줄링】                                                     │
│  ──────────────────                                                  │
│  • 직렬: 한 번에 하나씩                                              │
│  • 병렬: 독립적인 것 동시에                                           │
│  • 예: PC+1과 메모리 읽기 동시 가능                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  μOp 최적화:                                                        │ │
│  │  ──────────────                                                    │ │
│  │  1. 병렬 수행: 독립 μOp 동시에                                     │ │
│  │  2. μOp 병합: 여러 μOp를 하나로                                    │ │
│  │  3. 파이프라인: μOp 단계 중첩                                      │ │
│  │  4. 캐시: 메모리 μOp 지연 감소                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

┌─────────────────────────────────────────────────────────────────────┐
│                μOp × 타 과목 융합                                       │

│  • 하드와이어드: μOp 직접 생성                                        │
│  • RISC vs CISC: μOp 복잡도                                          │

│  【컴파일러】                                                          │

│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  μOp과 성능:                                                        │ │
│  │  ──────────────                                                    │ │
│  │  • CPI (Cycles Per Instruction)                                   │ │
│  │  • μOp 수 = CPI                                                    │ │
│  │  • 평균 CPI = Σ(μOp 수 × 빈도)                                     │ │
│  │  • 성능 향상: μOp 수 감소 또는 병렬화                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅳ. 실무 적용

### μOp 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                μOp 분석                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【성능 분석】                                                        │
│  ──────────────────                                                  │
│  $ perf stat -e cycles,instructions ./prog                          │
│  cycles: 5,000,000,000                                              │
│  instructions: 2,000,000,000                                        │
│  CPI: 2.5 → 평균 2.5 μOp/명령어                                      │
│                                                                     │
│  【x86 μOp 분석】                                                     │
│  ──────────────────                                                  │
│  $ perf stat -e uops_issued.any,uops_retired.any ./prog             │
│  uops_issued: 4,000,000,000                                         │
│  uops_retired: 3,800,000,000                                        │
│  → 일부 μOp가 스펙큐레이션으로 취소됨                                 │
│                                                                     │
│  【μOp 캐시】                                                         │
│  ──────────────────                                                  │
│  $ perf stat -e idq.dsb_uops, idq.ms_uops ./prog                    │
│  idq.dsb_uops: 3,500,000,000  ; DSB (μOp 캐시)에서                  │
│  idq.ms_uops: 500,000,000      ; MS (마이크로코드)에서               │
│  → 87% μOp 캐시 적중률                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  μOp 최적화 팁:                                                     │ │
│  │  ──────────────                                                    │ │
│  │  1. 단순 명령어 사용 (적은 μOp)                                    │ │
│  │  2. 메모리 접근 최소화                                              │ │
│  │  3. 레지스터 활용                                                   │ │
│  │  4. 복잡한 명령어 피하기 (문자열 등)                                │ │
│  │  5. SIMD 활용 (μOp 효율 향상)                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• μOp: CPU의 가장 기본적인 하드웨어 동작
• 유형: 전송, 산술, 논리, 시프트, 메모리, 제어
• 형식: 목적지 ← 출발지
• 명령어 = 여러 μOp의 시퀀스
• 제어 신호: 각 μOp에 필요한 신호들
• CPI = μOp 수 / 명령어 수
• 최적화: 병렬 수행, μOp 병합, 캐시
```
