# 정적 분기 예측 (Static Branch Prediction)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 정적 분기 예측(Static Branch Prediction)은 프로그램이 실행되는 도중의 상태(History)를 전혀 보지 않고, 오직 명령어의 종류나 방향이라는 **고정된 하드웨어 규칙 또는 컴파일러의 힌트만으로 분기 여부를 예측**하는 방식이다.
> 2. **가치**: 예측을 위한 거대한 메모리(BHT)나 복잡한 회로가 필요 없으므로 하드웨어 구현이 극도로 단순하며, 저전력 임베디드 프로세서나 초창기 파이프라인에서 제어 해저드를 방어하는 1차원적 가성비 솔루션이다.
> 3. **융합**: 단독으로는 적중률의 한계(70~80%)가 명확하여, 현대 프로세서에서는 복잡한 동적 분기 예측기(Dynamic)가 학습을 완료하기 전 극초반에 사용하는 **백업(Fallback) 메커니즘**으로 융합되어 사용된다.

+++

## Ⅰ. 정적 분기 예측의 개요

### 1. 정의
- CPU 내부의 런타임 과거 기록(History)에 의존하지 않고, 사전에 정의된 휴리스틱(Heuristics) 규칙이나 컴파일러가 심어놓은 특정 비트(Hint bit)를 보고 분기 여부를 결정(예상)하는 기법이다.

### 2. 등장 배경: '복잡한 회로 없이 찍어라'
- 파이프라인 구조가 처음 도입되었을 때 분기 페널티는 치명적이었다. 이를 막으려니 과거를 기억하는 회로(SRAM)를 넣을 칩 면적이 부족했다. 설계자들은 프로그래머들의 코드 작성 패턴을 분석한 결과, "반복문(Loop) 때문에 위(뒤)로 뛰는 점프는 대부분 실제로 뛰고, 에러 처리 때문에 아래(앞)로 뛰는 점프는 거의 안 뛴다"는 절대적인 통계를 발견했다. 이를 하드웨어에 고정 규칙으로 박아버린 것이 정적 예측의 시초다.

### 3. 💡 비유: '우산 챙기기 고정 룰'
- 정적 분기 예측은 **'하늘을 보거나 일기예보를 확인하지 않고, 무조건 "여름 장마철에는 우산을 챙긴다", "겨울에는 안 챙긴다"라고 정해놓고 행동하는 것'**과 같다.
- 매일 날씨를 확인하는 수고(하드웨어 계산)가 전혀 안 들고 꽤 높은 확률로 비를 피할 수 있지만, 갑자기 겨울에 비가 오면(예외 상황) 속수무책으로 비를 맞아야 한다(파이프라인 플러시).

- **📢 섹션 요약 비유**: 상대방의 눈치를 보지 않고, 오직 사회적 통계와 고정관념(규칙)만으로 다음 상황을 단정 짓는 심플한 의사결정입니다.

## Ⅱ. 메커니즘 및 예측 휴리스틱 (Deep Dive)

하드웨어가 런타임에 생각할 필요가 없도록 철저한 룰 기반으로 동작한다.

### 1. 주요 정적 예측 휴리스틱 (Heuristics)
| 예측 룰 | 동작 원리 (의사코드) | 근거 및 특징 |
|:---|:---|:---|
| **항상 안 뜀 (Predict Not Taken)** | 무조건 PC+1 의 순차적 다음 명령어를 Fetch | 하드웨어 구현이 가장 쉬움. if문의 예외 처리 패스에 유리. |
| **항상 뜀 (Predict Taken)** | 무조건 타겟 주소(Target)의 코드를 Fetch | 루프(Loop) 구조에 유리. (단, 타겟 주소 계산이 끝난 후에만 가능) |
| **BTFN (Backward Taken, Forward Not)** | 타겟 주소가 **현재 PC보다 작으면 뛰고(Taken)**, 크면 안 뛴다(Not Taken) | `for`나 `while` 문은 위로 되돌아가므로 뛰고, `if`문 건너뛰기는 아래로 가므로 안 뛴다는 통계적 진리에 기반. 적중률 약 80%. |

### 2. 컴파일러 힌트 (Compiler Hint)
- **개념**: 휴리스틱만으로는 한계가 있으므로, 코드를 가장 잘 아는 컴파일러가 기계어를 만들 때 분기 명령어의 안 쓰는 비트(Branch Hint Bit)를 '1'로 세팅하면 하드웨어가 이를 보고 "아 컴파일러가 뛰라고 힌트를 줬네" 하고 예측하는 방식.
- **예시**: PowerPC나 MIPS 아키텍처에서는 조건부 분기 명령어에 `Likely` / `Unlikely` 비트를 제공하여 정적 예측의 정확도를 컴파일 타임에 끌어올렸다.

- **📢 섹션 요약 비유**: 센서(동적) 없이 그냥 직진하는 무식한 방법에서, 바닥에 "이쪽으로 휠 확률 90%"라는 화살표(컴파일러 힌트)를 그려놓는 똑똑한 이정표 방식으로 발전했습니다.

## Ⅲ. 정적 예측 vs 동적 예측 비교

| 비교 항목 | 정적 분기 예측 (Static) | 동적 분기 예측 (Dynamic) |
|:---|:---:|:---:|
| **예측 근거** | **명령어의 방향(PC 증감) 또는 힌트** | **과거의 실제 실행 결과 기록 (History)** |
| **상태 저장 (Memory)** | **불필요 (0 bit)** | 필요함 (BHT 등 대규모 SRAM 탑재) |
| **적중률 (Accuracy)** | 최대 70 ~ 85% 한계 | **최대 95 ~ 99% 달성 가능** |
| **적용 아키텍처** | 저전력 MCU, 초기 파이프라인 | 현대 고성능 CPU (x86, ARM Cortex-A) |
| **변경 적응성** | 실행 패턴이 중간에 바뀌어도 고정 | 루프가 끝나면 곧바로 패턴 변화 학습 |

- **📢 섹션 요약 비유**: 정적 예측이 돌에 새겨진 고정된 십계명이라면, 동적 예측은 판례에 따라 매번 판결이 유연하게 바뀌는 현대 법원입니다.

+++

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 실무 시나리오: C++20 `[[likely]]` / `[[unlikely]]` 어트리뷰트
- **상황**: 극단적인 지연(Low Latency)을 요구하는 고빈도 거래(HFT) 시스템의 C++ 코드.
- **판단**: C++20부터 표준으로 들어온 `[[likely]]`와 `[[unlikely]]` 속성을 조건문에 명시한다.
  ```cpp
  if (value == 0) [[unlikely]] {
      // 에러 처리 루틴
  }
  ```
  이 힌트를 받은 컴파일러는 기계어 블록을 물리적으로 멀리 떨어진 곳으로 치워버린다. 그 결과, 하드웨어의 **정적 분기 예측기(Forward Not Taken 규칙)**가 자연스럽게 메인 로직을 (점프하지 않고) 순차적으로 쭉 밀고 나가게 되어, 캐시 지역성(Cache Locality)과 파이프라인 예측 적중률이 완벽에 가깝게 튜닝된다. 기술사는 이처럼 하드웨어 아키텍처의 정적 규칙을 소프트웨어로 역이용할 줄 알아야 한다.

### 2. 기술사적 인사이트: 동적 예측의 콜드 스타트(Cold Start) 보완
- **인사이트**: 현대 CPU는 99% 동적 분기 예측기를 쓴다. 그렇다면 정적 예측은 죽은 기술일까? 아니다. 프로그램이 처음 실행되거나 컨텍스트 스위치 직후에는 동적 예측기가 참조할 '과거 기록(BHT)'이 비어있다(Cold Start). 이때 CPU는 과거 기록이 찰 때까지 기본값으로 **BTFN(뒤로 뛰고, 앞으로 안 뜀)** 방식의 정적 예측 로직을 사용해 초반 파이프라인 플러시를 1차 방어한다. 정적 예측은 가장 훌륭한 백업 시스템이다.

- **📢 섹션 요약 비유**: 빅데이터(동적)가 쌓이기 전, 사업 초창기에 직관과 통계(정적)만으로 일단 장사를 시작하는 영리한 린(Lean) 스타트업 전략과 같습니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **칩 면적 및 전력 다이어트**: BHT나 BTB 같은 거대한 트랜지스터 덩어리 없이 논리 게이트 몇 개만으로 80%의 방어율 달성.
- **결정론적 성능**: 실행할 때마다 성능이 들쭉날쭉한 동적 방식과 달리, 코드가 주어지면 언제나 예측 결과가 고정되어 실시간 운영체제(RTOS)의 실행 시간 보장에 유리.

### 2. 미래 전망
거대하고 전기를 많이 먹는 고성능 코어(x86)에서는 메인 예측기의 자리를 내어준 지 오래다. 하지만 수백억 개의 사물인터넷(IoT) 센서에 들어가는 초소형 RISC-V 코어나 ARM Cortex-M 라인업에서는, 단돈 1달러의 칩 단가와 1mW의 전력 소모를 줄이기 위해 여전히 **정적 분기 예측**이 핵심 엔진으로 살아 숨 쉬고 있다. 하드웨어의 극한적 경량화가 요구되는 엣지 컴퓨팅 시대에 정적 예측은 결코 사라지지 않을 미니멀리즘의 정수다.

- **📢 섹션 요약 비유**: 화려한 스마트폰(동적 예측) 시대에도, 배터리 한 알로 10년을 버티는 전자 손목시계(정적 예측)가 여전히 전 세계인의 손목을 지배하고 있는 것과 같습니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **동적 분기 예측 (Dynamic Branch Prediction)**: 정적 예측의 한계를 넘기 위해 진화한 완전체 기술.
- **파이프라인 플러시 (Pipeline Flush)**: 정적 예측의 20% 오답률이 발생시킬 때 겪어야 하는 무서운 벌칙.
- **컴파일러 최적화**: 정적 예측기를 100% 활용하기 위해 코드를 재배치하는 핵심 소프트웨어 기술.

## 👶 어린이를 위한 3줄 비유 설명
1. 정적 분기 예측은 시험 문제를 풀 때 **"모르면 무조건 3번으로 찍어라!"**라는 고정된 규칙을 가지고 있는 거예요.
2. 예전에 답이 뭐였는지 공부(기억)할 필요도 없고, 그냥 규칙대로 팍팍 찍고 넘어가니까 머리가 아플 일도 없고 아주 편하죠.
3. 확률상 "뒤로 돌아가는 길(루프)은 무조건 간다!"라고 찍기만 해도 10문제 중에 8문제는 맞출 수 있는 아주 훌륭한 찍기 비법이랍니다!
