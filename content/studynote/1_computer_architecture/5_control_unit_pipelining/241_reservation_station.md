# 241. 예약 스테이션 (Reservation Station)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실행 대기 명령어를 저장하고 오퍼랜드 대기
> 2. **가치**: 비순차 발급, 데이터 포워딩 지원
> 3. **융합**: 레지스터 리네이밍, ROB와 연동

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

예약 스테이션(Reservation Station)은 **명령어와 오퍼랜드를 대기시키는 버퍼**로, 실행 유닛 앞에 위치한다. 오퍼랜드가 준비되면 해당 명령어를 실행 유닛으로 발급한다. 비순차 실행의 핵심 구조이다.

### 💡 비유: 식당 주방 대기표

예약 스테이션은 **주방의 주문 대기표**와 같다. 웨이터가 주문을 가져오면 요리사가 재료가 있는 것부터 만든다. 재료가 없는 주문은 재료가 도착할 때까지 기다린다. 재료가 도착하면 바로 요리를 시작한다.

### 예약 스테이션 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                예약 스테이션 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【예약 스테이션 위치】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  인출 → 디코드 → 리네이밍 → ┌───────────┐ → 실행 → ROB          │ │
│  │                          │예약 스테이션│                         │ │
│  │                          └───────────┘                          │ │
│  │                              ↑                                 │ │
│  │                         발급 대기                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예약 스테이션 엔트리】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │ 인덱스 │ 연산 │ 목적지 │ 오퍼랜드1 │ 오퍼랜드2 │ 준비 │  │  │ │
│  │  ├───────┼─────┼───────┼─────────┼─────────┼─────┤  │  │ │
│  │  │   0   │ ADD │  P10  │   P5    │   P6    │  O  │  │  │ │
│  │  │   1   │ MUL │  P11  │   P7    │   -     │  X  │  │  │ │
│  │  │   2   │ SUB │  P12  │   P10   │   P8    │  X  │  │  │ │
│  │  │   3   │ LOAD│  P13  │  [addr] │   -     │  O  │  │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  │  • 연산: 수행할 연산 (ADD, MUL, LOAD 등)                         │ │
│  │  • 목적지: 결과물이 저장될 물리 레지스터                            │ │
│  │  • 오퍼랜드: 소스 오퍼랜드 (물리 레지스터 또는 태그)                  │ │
│  │  • 준비: 오퍼랜드 준비 여부 (O: 준비, X: 대기)                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예약 스테이션 동작】                                                   │
│  ──────────────────                                                  │
│  1. 명령어 도착: 리네이밍된 명령어가 예약 스테이션에 도착                    │
│  2. 오퍼랜드 확인: 소스 오퍼랜드가 준비되었는지 확인                         │
│  3. 대기: 오퍼랜드가 없으면 태그로 대기                                    │
│  4. 포워딩: 다른 유닛에서 결과가 나오면 CDB로 전달받음                       │
│  5. 발급: 모든 오퍼랜드 준비되면 실행 유닛으로 발급                         │
│                                                                     │
│  【예약 스테이션 종류】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  유형          담당 연산        크기                             │ │
│  │  ──────────    ──────────      ──────                           │ │
│  │  정수 RS       ADD, SUB, AND   10~20                            │ │
│  │  부동소수점 RS  MUL, DIV, FADD  10~20                            │ │
│  │  메모리 RS     LOAD, STORE     10~20                            │ │
│  │  분기 RS       BRANCH          5~10                             │ │
│  │  → 실행 유닛별로 전용 예약 스테이션                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 예약 스테이션 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                예약 스테이션 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CDB (Common Data Bus)】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                      CDB (공용 데이터 버스)                       │ │
│  │           ┌──────────────────────────────────────┐             │ │
│  │           │  결과: P10 = 100                      │             │ │
│  │           │  태그: P10                            │             │ │
│  │           └──────────────────────────────────────┘             │ │
│  │                    ↑              ↓            ↓               │ │
│  │               ┌────┴────┐    ┌────┴────┐   ┌────┴────┐        │ │
│  │               │  ALU 1  │    │  ALU 2  │   │  FPU    │        │ │
│  │               └─────────┘    └─────────┘   └─────────┘        │ │
│  │                    ↑              ↑            ↑               │ │
│  │               ┌────┴────┐    ┌────┴────┐   ┌────┴────┐        │ │
│  │               │  RS 1   │    │  RS 2   │   │  RS FP  │        │ │
│  │               │ P10 대기 │    │         │   │ P10 대기 │        │ │
│  │               └─────────┘    └─────────┘   └─────────┘        │ │
│  │                                                               │ │
│  │  → ALU 1에서 P10 결과 발생 → CDB로 방송                         │ │
│  │  → RS 2, RS FP에서 P10 대기 중 → 결과 수신                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예약 스테이션 발급 과정】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  클럭  RS 상태                    동작                          │ │
│  │  ────  ────────────────────      ───────────────────            │ │
│  │   1    ADD P10, P5, P6 (준비)    RS에 할당                       │ │
│  │   2    MUL P11, P7, P10 (대기)   P10 대기                        │ │
│  │   3    ADD 실행 → P10=100        ALU로 발급                      │ │
│  │   4    CDB: P10=100 도착         P11 오퍼랜드 갱신                │ │
│  │   5    MUL P11, P7, 100 (준비)   ALU로 발급                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예약 스테이션 크기】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU             총 RS 엔트리   정수   FP    메모리               │ │
│  │  ───────────────  ───────────  ─────  ────  ──────               │ │
│  │  Intel Skylake    97           58     28    11                   │ │
│  │  AMD Zen 3        180          88     60    32                   │ │
│  │  Apple M1         300+         150    100   50+                  │ │
│  │  ARM Cortex-A78   64           32     20    12                   │ │
│  │  → RS 크기가 비순차 능력 결정                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 예약 스테이션: 실행 대기 명령어를 저장하는 버퍼
• 목적: 비순차 발급, 오퍼랜드 대기
• 구조: 연산 + 목적지 + 오퍼랜드 + 준비 상태
• 크기: 32~300+ 엔트리 (유닛별)
• CDB: 결과를 모든 RS에 방송
• 발급 조건: 모든 오퍼랜드 준비
• 비용: 하드웨어 복잡도, 면적 증가
• 보안: Spectre 공격 벡터
```
