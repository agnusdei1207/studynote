# 216. 제어 메모리 (Control Memory)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 마이크로코드를 저장하는 ROM
> 2. **가치**: 마이크로프로그래밍의 핵심, 유연한 제어
> 3. **융합**: ROM, 캐시, x86 μOp 캐시

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

제어 메모리(Control Memory)는 **마이크로프로그램(μOp 시퀀스)을 저장하는 ROM**이다. 마이크로프로그래밍 방식의 제어 유닛에서 핵심 역할을 하며, CPU의 제어 방식을 결정하는 마이크로코드가 저장되어 있다.

### 💡 비유: 레시피 책

제어 메모리는 **레시피 책**과 같다. 각 페이지(주소)에 요리 방법(마이크로코드)이 적혀 있다. 요리사(제어 유닛)는 페이지를 넘기며(CAR 증가) 레시피대로 요리한다.

### 제어 메모리 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                제어 메모리 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                        │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │          제어 메모리 (ROM)                                        │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  주소      마이크로명령어                                    │    │ │
│  │  │  ────────  ─────────────────────────────────────────────    │    │ │
│  │  │  0x000    [Next=1][Branch=0][ALU=ADD][C1=1][C2=0]...       │    │ │
│  │  │  0x001    [Next=1][Branch=0][ALU=PASS][C1=0][C2=0]...       │    │ │
│  │  │  0x002    [Next=1][Branch=0][ALU=SUB][C1=0][C2=1]...       │    │ │
│  │  │  ...                                                          │    │ │
│  │  │  0x100    [Next=0][Branch=1][ALU=...]  ; 분기               │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                  ▲                                               │ │
│  │                  │                                               │ │
│  │            마이크로명령어 레지스터 (μIR)                           │ │
│  │                  │                                               │ │
│  │                  ▼                                               │ │
│  │           ┌────────────────┐                                     │ │
│  │           │    디코더       │ → 제어 신호들                        │ │
│  │           └────────────────┘                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【용량】                                                              │
│  ──────────────────                                                  │
│  • 크기: 수십 ~ 수백 KB                                               │
│  • 워드 수: 수백 ~ 수천                                                │
│  • 워드 길이: 20~100 비트                                              │
│  • 예: 1024 워드 × 32 비트 = 4 KB                                     │
│                                                                     │
│  【주소 지정】                                                        │
│  ──────────────────                                                  │
│  CAR(Control Address Register):                                     │
│  • μAddr 저장                                                        │
│  • 크기: log2(제어 메모리 워드 수) 비트                                │
│  • 증가/분기: μSequencer 제어                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  제어 메모리 조직:                                                  │ │
│  │  ──────────────────                                                │ │
│  │  • 구성: N 워드 × W 비트                                            │ │
│  │  • 접근: CAR → ROM → μIR                                           │ │
│  │  • 속도: ROM 읽기 = 1 클럭                                          │ │
│  │  • 내용: 각 명령어의 μOp 시퀀스                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 제어 메모리 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                제어 메모리 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【마이크로명령어 형식】                                               │
│  ────────────────────                                                │
│  수평형 (Horizontal):                                                 │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  [  제어 필드  ] [   분기 필드   ] [     기타     ]            │    │
│  │  각 비트가 독립적 제어 신호                                         │    │
│  │  • 장점: 유연함, 동시 제어 가능                                      │    │
│  │  • 단점: 긴 워드 길이                                                │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  수직형 (Vertical):                                                   │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  [  Opcode  ] [  분기 조건  ] [  다음 주소  ]                 │    │
│  │  코드화된 제어 신호                                                  │    │
│  │  • 장점: 짧은 워드 길이                                               │    │
│  │  • 단점: 디코딩 필요                                                  │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  【제어 메모리 주소 매핑】                                               │
│  ────────────────────────                                             │
│  방식 1: 직접 매핑                                                    │
│  μAddr = Opcode × n + Offset                                          │
│  • 간단한 구조                                                         │
│  • 낭비 가능                                                           │
│                                                                     │
│  방식 2: 2단계 매핑                                                    │
│  1. Opcode → 시작 주소 (μAddr)                                        │
│  2. 시작 주소 + 순차 → 다음 μAddr                                      │
│  • 효율적                                                               │
│  • 유연함                                                               │
│                                                                     │
│  【x86의 제어 메모리】                                                   │
│  ──────────────────                                                    │
│  • 마이크로코드 ROM: 복잡한 명령어 구현                                 │
│  • μOp 캐시 (DSB): 디코드된 μOp 캐시                                  │
│  • MS ROM: μOp 캐시 미스 시 사용                                       │
│  • 현대 x86: 대부분 μOp 캐시 사용 (빠름)                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  제어 메모리 최적화:                                                 │ │
│  │  ──────────────────                                                │ │
│  │  1. μOp 캐시: ROM → 캐시                                            │ │
│  │  2. 압축: 수직 마이크로명령어                                        │ │
│  │  3. 나노코드: 더 작은 제어 메모리                                    │ │
│  │  4. 하이브리드: 하드와이어드 + 마이크로코드                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                제어 메모리 분석                                        │

│  【x86 마이크로코드 확인】                                              │

│  $ cat /proc/cpuinfo | grep microcode                               │

│                                                                     │

│  $ sudo apt install amd64-microcode                                  │
│                                                                     │
│  # 수동 업데이트                                                        │
│  $ sudo cp microcode.bin /lib/firmware/                             │
│  $ sudo echo 1 > /sys/devices/system/cpu/microcode/reload            │
│                                                                     │
│  【μOp 캐시 분석】                                                     │

│  │  • MDS (RIDL, Fallout) 수정                                        │ │
│  │                                                               │ │

│  │  • Spectre 완화                                                     │ │

│  │                                                               │ │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 제어 메모리: 마이크로코드를 저장하는 ROM
• 구성: N 워드 × W 비트
• 접근: CAR → ROM → μIR → 디코더 → 제어 신호
• 마이크로명령어: 수평형 vs 수직형
• 주소 매핑: 직접 vs 2단계
• 최적화: μOp 캐시, 압축, 나노코드
• 현대: μOp 캐시(DSB)가 ROM 대신 사용
• 업데이트: 보안 패치, 성능 최적화
```
