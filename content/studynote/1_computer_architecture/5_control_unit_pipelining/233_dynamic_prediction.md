# 233. 동적 분기 예측 (Dynamic Prediction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실행 이력 기반으로 분기 결과 예측
> 2. **가치**: 90~99% 정확도, 현대 CPU 필수
> 3. **융합**: 2비트 예측기, 상관 예측, 토너먼트

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

동적 분기 예측(Dynamic Prediction)은 **프로그램 실행 중 수집한 분기 이력을 기반으로 분기 결과를 예측**하는 방식이다. 하드웨어가 분기 패턴을 학습하여 높은 정확도를 달성한다.

### 💡 비유: 교통 상황 학습 내비게이션

동적 분기 예측은 **교통 상황을 학습하는 내비게이션**과 같다. 매일 출근할 때마다 어느 길이 막히는지 학습한다. 월요일은 A 코스, 금요일은 B 코스로 자동 추천한다.

### 동적 분기 예측 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                동적 분기 예측 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                         │
│  ──────────────────                                                  │
│  • 실행 시점에 예측 결정                                                 │
│  • 분기 이력(BHT) 저장                                                  │
│  • 학습으로 정확도 향상                                                  │
│  • 90~99% 정확도 달성                                                  │
│                                                                     │
│  【1비트 예측기 (1-bit Predictor)】                                      │
│  ──────────────────────────────                                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  상태: 1비트 (T 또는 NT)                                            │ │
│  │  규칙: 마지막 결과를 다음 예측으로 사용                                  │ │
│  │                                                               │ │
│  │  예: 루프 100회                                                    │ │
│  │  반복 1~99: Taken → 예측 T (맞음)                                   │ │
│  │  반복 100: Not Taken → 예측 T (틀림)                                │ │
│  │  루프 끝: 예측 NT (틀림) → 다음에 NT로 시작                             │ │
│  │                                                               │ │
│  │  문제: 루프 시작과 끝에서 틀림                                         │ │
│  │  → 중첩 루프에서 2N번 틀림                                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【2비트 예측기 (2-bit Predictor)】                                      │
│  ──────────────────────────────                                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  상태 전이도:                                                        │ │
│  │                                                               │ │
│  │      Not Taken              Taken                               │ │
│  │        ↑                       ↓                                 │ │
│  │    ┌───────┐              ┌───────┐                             │ │
│  │    │ 00    │─────────────▶│ 01    │                             │ │
│  │    │강한 NT│    NT        │약한 NT │                             │ │
│  │    └───────┘◀─────────────└───────┘                             │ │
│  │        │  T                   │  NT                               │ │
│  │        │                      ↓                                   │ │
│  │    ┌───────┐              ┌───────┐                             │ │
│  │    │ 11    │◀─────────────│ 10    │                             │ │
│  │    │강한 T │    T         │약한 T  │                             │ │
│  │    └───────┘─────────────▶└───────┘                             │ │
│  │        ↑ T                    │ T                                 │ │
│  │        └──────────────────────┘                                   │ │
│  │                                                               │ │
│  │  00 (강한 NT): NT 예측, 연속 NT 유지                               │ │
│  │  01 (약한 NT): NT 예측, 한 번 T로 전이                             │ │
│  │  10 (약한 T):  T 예측, 한 번 NT로 전이                              │ │
│  │  11 (강한 T):  T 예측, 연속 T 유지                                 │ │
│  │                                                               │ │
│  │  장점: 연속 2번 틀려야 예측 변경 (히스테리시스)                          │ │
│  │  → 루프 끝에서 1번만 틀림                                            │ │
│  │  → 정확도: 90~95%                                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상관 분기 예측 (Correlating Predictor)】                              │
│  ──────────────────────────────────────                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  (m,n) 예측기: m개의 이전 분기 결과로 n비트 예측기 선택                    │ │
│  │                                                               │ │
│  │  예: (2,2) 예측기                                                 │ │
│  │  • 2비트 전역 이력 (이전 2개 분기 결과)                               │ │
│  │  • 4개의 2비트 예측기 (00, 01, 10, 11)                              │ │
│  │  • 이력으로 적절한 예측기 선택                                        │ │
│  │                                                               │ │
│  │  if (a) ...   ; 분기 1                                          │ │
│  │  if (b) ...   ; 분기 2                                          │ │
│  │  if (c) ...   ; 분기 3 - 분기 1,2 결과에 의존할 수 있음                │ │
│  │                                                               │ │
│  │  → 연관된 분기 간의 패턴 학습                                        │ │
│  │  → 정확도: 93~97%                                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  동적 예측기 정확도 비교:                                                 │ │
│  │  ────────────────────────                                            │ │
│  │  예측기           정확도    하드웨어                                   │ │
│  │  ──────────────  ──────    ───────                                  │ │
│  │  1비트            80~90%    매우 낮음                                  │ │
│  │  2비트            90~95%    낮음                                     │ │
│  │  (1,1) 상관       91~96%    낮음                                     │ │
│  │  (2,2) 상관       93~97%    중간                                     │ │
│  │  토너먼트          95~98%    높음                                     │ │
│  │  TAGE            97~99%    매우 높음                                  │ │
│  │  → 현대 CPU는 토너먼트 또는 TAGE 사용                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 동적 분기 예측 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                동적 분기 예측 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【BHT (Branch History Table)】                                       │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구조:                                                             │ │
│  │  ┌──────────────────────────────────────────────────────────┐   │ │
│  │  │  분기 PC (하위 비트)   │   2비트 예측기   │   타겟 주소        │   │ │
│  │  ├──────────────────────────────────────────────────────────┤   │ │
│  │  │       0x1000         │      11 (강한 T)   │    0x2000       │   │ │
│  │  │       0x2000         │      00 (강한 NT)  │    -            │   │ │
│  │  │       0x3000         │      10 (약한 T)   │    0x4000       │   │ │
│  │  │        ...           │         ...       │     ...         │   │ │
│  │  └──────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  │  크기: 1K~64K 엔트리                                               │ │
│  │  인덱싱: PC 하위 비트                                                │ │
│  │  → 충돌 가능 (다른 분기가 같은 엔트리 사용)                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【토너먼트 예측기 (Tournament Predictor)】                               │
│  ──────────────────────────────────────                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구조: 3개의 예측기 경쟁                                             │ │
│  │                                                               │ │
│  │  ┌────────────────┐   ┌────────────────┐                       │ │
│  │  │ 지역 예측기      │   │ 전역 예측기      │                       │ │
│  │  │ (분기별 2비트)   │   │ (전역 이력)     │                       │ │
│  │  └───────┬────────┘   └───────┬────────┘                       │ │
│  │          │                    │                                 │ │
│  │          └────────┬───────────┘                                 │ │
│  │                   │                                             │ │
│  │            ┌──────▼──────┐                                      │ │
│  │            │ 선택기 (2비트) │                                      │ │
│  │            │ 어떤 예측기   │                                      │ │
│  │            │ 더 잘하는지   │                                      │ │
│  │            └──────┬──────┘                                      │ │
│  │                   │                                             │ │
│  │            ┌──────▼──────┐                                      │ │
│  │            │   최종 예측   │                                      │ │
│  │            └─────────────┘                                      │ │
│  │                                                               │ │
│  │  Intel 구현:                                                     │ │
│  │  • 4K 지역 2비트 예측기                                            │ │
│  │  • 4K 전역 2비트 예측기 (12비트 전역 이력)                             │ │
│  │  • 4K 선택기                                                      │ │
│  │  → 정확도: 95~98%                                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TAGE (TAgged GEometric history length predictor)】                   │
│  ────────────────────────────────────────────                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구조: 여러 BHT를 다른 이력 길이로                                    │ │
│  │                                                               │ │
│  │  T0: 기본 2비트 (이력 없음)                                         │ │
│  │  T1: 2비트 이력                                                   │ │
│  │  T2: 4비트 이력                                                   │ │
│  │  T3: 8비트 이력                                                   │ │
│  │  T4: 16비트 이력                                                  │ │
│  │  ...                                                            │ │
│  │                                                               │ │
│  │  각 테이블에 태그 저장 → 충돌 감지                                     │ │
│  │  가장 긴 이력 일치 테이블 사용                                        │ │
│  │  → 정확도: 97~99%                                                  │ │
│  │  → 현대 고성능 CPU 표준 (Intel, AMD, ARM)                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예측 미시 페널티:                                                      │ │
│  │  ──────────────────                                                  │ │
│  │  • 파이프라인 플러시                                                  │ │
│  │  • 페널티 = 파이프라인 깊이 - 1                                        │ │
│  │  • Intel Skylake: ~15클럭                                           │ │
│  │  • AMD Zen 3: ~12클럭                                               │ │
│  │  • Apple M1: ~10클럭                                                │ │
│  │  → 미스율 1% = 0.15~0.2 CPI 증가                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 동적 분기 예측 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                동적 분기 예측 × 타 과목 융합                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • BHT: 분기 역사 표, 1K~64K 엔트리                                     │
│  • BTB: 분기 타겟 버퍼, 캐시와 유사                                       │
│  • RAS: Return Address Stack, 함수 복귀                               │
│  • 파이프라인: 분기 예측으로 CPI ≈ 1                                     │
│                                                                     │
│  【디지털 논리회로】                                                   │
│  • 2비트 카운터: 상태 머신                                               │
│  • BHT: CAM 또는 캐시 구조                                             │
│  • MUX: 예측기 선택                                                   │
│  • 비교기: 태그 매칭                                                   │
│                                                                     │
│  【운영체제】                                                          │
│  • 컨텍스트 스위치: BHT 상태 유지?                                        │
│  • 스케줄러: BHT 오염 고려                                               │
│  • 멀티스레딩: BHT 공유 문제                                             │
│                                                                     │
│  【보안】                                                              │
│  • Spectre: 분기 예측 악용                                             │
│  • 사이드 채널: BHT 상태 노출                                            │
│  • 완화: BHT 플러시, retpoline                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  현대 CPU 분기 예측기 크기:                                              │ │
│  │  ────────────────────────                                            │ │
│  │  CPU            BHT 크기    BTB 크기    정확도                         │ │
│  │  ────────────  ─────────  ─────────  ──────                          │ │
│  │  Intel Skylake  ~64K      ~4K       > 97%                            │ │
│  │  AMD Zen 3      ~48K      ~6K       > 96%                            │ │
│  │  Apple M1       ~128K     ~8K       > 98%                            │ │
│  │  ARM Cortex-A78 ~32K      ~2K       > 95%                            │ │
│  │  → 하드웨어 비용 vs 성능 트레이드오프                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 동적 분기 예측 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                동적 분기 예측 분석                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【분기 예측 정확도 측정】                                                │
│  ────────────────────────                                            │
│  $ perf stat -e branches,branch-misses ./prog                       │
│  branches: 1,000,000,000                                            │
│  branch-misses: 10,000,000                                          │
│  → 1% 미스율 (99% 정확도)                                              │
│                                                                     │
│  【BHT 미스 분석】                                                       │
│  ──────────────────                                                  │
│  $ perf stat -e btb_misses ./prog                                  │
│  → BTB 적중률 확인                                                    │
│                                                                     │
│  【분기 패턴 분석】                                                       │
│  ──────────────────                                                  │
│  # 패턴 1: 항상 같은 방향 (쉬움)                                         │
│  for (i = 0; i < 1000; i++) { ... }  ; 항상 Taken                    │
│                                                                     │
│  # 패턴 2: 규칙적 패턴 (중간)                                             │
│  for (i = 0; i < 1000; i++) { if (i % 2 == 0) {...} }              │
│  ; T, NT, T, NT, ... 패턴 → 학습 가능                                   │
│                                                                     │
│  # 패턴 3: 불규칙 (어려움)                                               │
│  if (data[random()] > 0) { ... }  ; 랜덤                             │
│  ; 예측 불가 → 50% 미스                                                │
│                                                                     │
│  【Spectre 완화】                                                       │
│  ──────────────────                                                  │
│  $ perf stat -e r20d1 ./prog  ; Intel SDM: BR_MISP_RETIRED.ALL_BRANCHES │
│  → 분기 예측 미스 세부 분석                                               │
│                                                                     │
│  【코드 최적화】                                                          │
│  ──────────────────                                                  │
│  # Before (불규칙 분기)                                                │
│  for (i = 0; i < n; i++) {                                          │
│      if (data[i] > threshold) {  ; 예측 어려움                        │
│          // ...                                                      │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  # After (정렬로 예측 가능하게)                                           │
│  sort(data, n);  ; 정렬                                               │
│  for (i = 0; i < n; i++) {                                          │
│      if (data[i] > threshold) {  ; 예측 가능 (패턴)                    │
│          // ...                                                      │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  동적 예측 최적화 팁:                                                   │ │
│  │  ──────────────────────                                              │ │
│  │  1. 분기 예측 친화적 코드 작성                                        │ │
│  │  2. 랜덤 분기 피하기                                                  │ │
│  │  3. 분기 가능성이 높은 코드 배치                                       │ │
│  │  4. PGO 활용                                                        │ │
│  │  5. 데이터 정렬로 분기 패턴 만들기                                     │ │
│  │  6. 분기 없는 코드 (조건부 이동)                                       │ │
│  │  7. Spectre 완화 필요시 retpoline                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 동적 분기 예측: 실행 이력 기반 예측
• 1비트 예측기: 마지막 결과 사용, 80~90%
• 2비트 예측기: 강한/약한 상태, 90~95%
• 상관 예측: 이전 분기 결과 고려, 93~97%
• 토너먼트: 여러 예측기 경쟁, 95~98%
• TAGE: 태그 + 기하급수 이력, 97~99%
• BHT: 분기 역사 표
• BTB: 분기 타겟 버퍼
• 현대 CPU: 95~99% 정확도
```
