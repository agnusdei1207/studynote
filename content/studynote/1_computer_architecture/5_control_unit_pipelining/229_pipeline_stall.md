# 229. 파이프라인 스톨 (Pipeline Stall)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파이프라인 정지, 해저드 해결의 기본 방법
> 2. **가치**: 구현 간단하지만 성능 저하, 최후 수단
> 3. **융합**: 버블, Load-Use, 분기 페널티

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

파이프라인 스톨(Pipeline Stall)은 **해저드를 해결하기 위해 파이프라인을 일시 정지하는 것**으로, 버블(Bubble) 또는 NOP(No Operation)을 삽입하여 명령어의 진행을 지연시킨다.

### 💡 비유: 공장 라인 멈춤

파이프라인 스톨은 **공장 조립 라인이 잠시 멈추는 것**과 같다. 부품이 늦게 도착하면 라인 전체가 기다린다. 공백(버블)이 생기고 생산량이 줄어든다.

### 파이프라인 스톨 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 스톨 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스톨 발생 원인】                                                     │
│  ──────────────────                                                  │
│  1. 데이터 해저드: RAW (Load-Use)                                      │
│  2. 제어 해저드: 분기 페널티                                            │
│  3. 구조적 해저드: 리소스 경합                                           │
│  4. 캐시 미스: 메모리 접근 대기                                          │
│                                                                     │
│  【버블 삽입 예시】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  LW  R1, 0(R2)    ; Load                                        │ │
│  │  ADD R3, R1, R4   ; R1 필요 → Load-Use 해저드                     │ │
│  │  SUB R5, R6, R7   ; 독립적                                         │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    LW                                                        │ │
│  │   2    ADD   LW                                                  │ │
│  │   3    SUB   ADD   버블   LW     ← 버블 삽입 (스톨)                  │ │
│  │   4          SUB   ADD   버블   LW                                │ │
│  │   5                SUB   ADD   버블   LW                          │ │
│  │   6                      SUB   ADD   버블   LW                    │ │
│  │                                                               │ │
│  │  → 버블 = NOP (아무 작업 안 함)                                     │ │
│  │  → 1클럭 스톨 발생                                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버블 구현】                                                          │
│  ──────────────────                                                  │
│  버블 = 제어 신호를 모두 0으로 설정:                                     │
│  • RegWrite = 0                                                     │
│  • MemRead = 0                                                      │
│  • MemWrite = 0                                                     │
│  • ALUOp = 0                                                        │
│  → 아무런 동작 없이 파이프라인 통과                                       │
│                                                                     │
│  【스톨 vs 포워딩】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  방식          CPI      구현 복잡도    비용                        │ │
│  │  ──────────  ──────    ────────────  ────                        │ │
│  │  스톨만        1.5~2.0   낮음          낮음                        │ │
│  │  포워딩       1.0~1.1   중간          중간                        │ │
│  │  포워딩+스톨   1.1~1.2   중간          중간                        │ │
│  │  → 스톨은 간단하지만 성능 저하                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스톨 발생 통계 (전형적인 프로그램):                                     │ │
│  │  ──────────────────────────                                          │ │
│  │  원인           발생 빈도     스톨 사이클    CPI 기여                  │ │
│  │  ────────────  ──────────  ────────────  ──────                     │ │
│  │  Load-Use      5~10%        1클럭        +0.05~0.1                  │ │
│  │  분기 페널티    10~20%       2~3클럭      +0.2~0.6                   │ │
│  │  캐시 미스      1~5%         10~100클럭   +0.1~0.5                   │ │
│  │  총 스톨                                           +0.3~1.2          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 파이프라인 스톨 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 스톨 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Load-Use 스톨】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  LW  R1, 0(R2)                                                  │ │
│  │  ADD R3, R1, R4   ; R1 필요                                      │ │
│  │                                                               │ │
│  │  감지 조건:                                                        │ │
│  │  • ID/EX.MemRead == 1 (이전이 LW)                                 │ │
│  │  • ID/EX.Rt == IF/ID.Rs 또는 IF/ID.Rt (소스가 목적지와 같음)          │ │
│  │                                                               │ │
│  │  스톨 동작:                                                        │ │
│  │  1. PC 쓰기 방지 (PCWrite = 0)                                    │ │
│  │  2. IF/ID 쓰기 방지 (IF/IDWrite = 0)                              │ │
│  │  3. ID/EX 제어 신호를 0으로 (버블)                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【분기 스톨】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BEQ R1, R2, target                                             │ │
│  │  ADD R3, R4, R5   ; 분기 여부 모름                                 │ │
│  │                                                               │ │
│  │  감지 조건:                                                        │ │
│  │  • IF/ID.Opcode == BEQ                                           │ │
│  │                                                               │ │
│  │  스톨 동작:                                                        │ │
│  │  1. IF/ID에 버블 삽입                                              │ │
│  │  2. 분기 결정까지 대기                                              │ │
│  │  3. 분기 시 이미 인출된 명령어 플러시                                  │ │
│  │                                                               │ │
│  │  페널티: 파이프라인 깊이 - 2 (EX에서 결정 시)                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 미스 스톨】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  I-Cache 미스: IF 단계 스톨                                        │ │
│  │  D-Cache 미스: MEM 단계 스톨                                       │ │
│  │                                                               │ │
│  │  스톨 사이클:                                                       │ │
│  │  • L1 미스: 4~10 사이클                                            │ │
│  │  • L2 미스: 20~40 사이클                                           │ │
│  │  • L3 미스: 50~100 사이클                                          │ │
│  │  • 메모리: 100~300 사이클                                          │ │
│  │                                                               │ │
│  │  → 캐시 미스가 가장 큰 스톨 원인                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스톨 최소화 기법】                                                     │
│  ────────────────────────                                            │
│  1. 포워딩: 데이터 해저드 스톨 감소                                       │
│  2. 분기 예측: 제어 해저드 스톨 감소                                      │
│  3. 비순차 실행: 스톨 동안 다른 명령어 실행                                 │
│  4. 캐시 최적화: 캐시 미스 스톨 감소                                      │
│  5. 코드 스케줄링: 컴파일러로 스톨 회피                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스톨 vs 비순차 실행:                                                   │ │
│  │  ──────────────────────                                              │ │
│  │  • 스톨: 파이프라인 정지, 모든 명령어 대기                               │ │
│  │  • 비순차: 의존성 없는 명령어 계속 실행                                  │ │
│  │  → 현대 CPU는 비순차 실행으로 스톨 효과 최소화                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 파이프라인 스톨 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 스톨 × 타 과목 융합                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • CPI = 1 + (스톨 사이클 / 명령어 수)                                   │
│  • 파이프라인 효율: (이상 CPI) / (실제 CPI)                              │
│  • 처리량: 클럭 / CPI                                                 │
│  • 성능: 1 / (CPI × 클럭 주기)                                         │
│                                                                     │
│  【컴파일러】                                                          │
│  • 명령어 스케줄링: 스톨 회피                                            │
│  • Load 지연 슬롯 채우기                                                │
│  • 분기 지연 슬롯 채우기                                                 │
│  • 의존성 분석                                                          │
│                                                                     │
│  【운영체제】                                                          │
│  • 페이지 폴트: 스톨 + 컨텍스트 스위치                                    │
│  • I/O 대기: 스톨 + 프로세스 대기                                        │
│  • 스케줄링: 스톨 고려                                                   │
│                                                                     │
│  【성능 분석】                                                         │
│  • 스톨 사이클 측정                                                      │
│  • CPI 분석                                                           │
│  • 병목 지점 파악                                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스톨 감소 기법 비교:                                                   │ │
│  │  ──────────────────────                                              │ │
│  │  기법            하드웨어  소프트웨어  스톨 감소율                         │ │
│  │  ─────────────  ───────  ───────  ─────────                         │ │
│  │  포워딩          높음      낮음      70~90%                           │ │
│  │  분기 예측        높음      낮음      80~95%                           │ │
│  │  코드 스케줄링     낮음      높음      20~40%                           │ │
│  │  비순차 실행       매우높음   낮음      90~99%                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 파이프라인 스톨 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 스톨 분석                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스톨 측정】                                                          │
│  ──────────────────                                                  │
│  $ perf stat -e cycles,instructions,stalled-cycles-frontend ./prog │
│  cycles: 10,000,000,000                                             │
│  instructions: 8,000,000,000                                        │
│  stalled-cycles-frontend: 2,000,000,000                            │
│  → CPI = 10/8 = 1.25                                                │
│  → 스톨률 = 20%                                                       │
│                                                                     │
│  【스톨 원인 분석】                                                      │
│  ──────────────────                                                  │
│  $ perf stat -e stalled-cycles-frontend,stalled-cycles-backend ./prog │
│  frontend stalls: I-Cache 미스, 분기                                  │
│  backend stalls: D-Cache 미스, 실행 유닛 대기                          │
│                                                                     │
│  【분기 미스 스톨】                                                      │
│  ──────────────────                                                  │
│  $ perf stat -e branch-misses ./prog                               │
│  branch-misses: 50,000,000                                         │
│  → 분기 미스당 10~20클럭 스톨                                            │
│  → 총 500M~1B 클럭 스톨                                                │
│                                                                     │
│  【캐시 미스 스톨】                                                       │
│  ──────────────────                                                  │
│  $ perf stat -e cache-misses,L1-dcache-load-misses ./prog          │
│  cache-misses: 10,000,000                                          │
│  → 캐시 미스당 50~100클럭 스톨                                           │
│  → 총 500M~1B 클럭 스톨                                                │
│                                                                     │
│  【코드로 스톨 회피】                                                     │
│  ──────────────────                                                  │
│  # Before (Load-Use 스톨)                                            │
│  LW  R1, 0(R2)                                                       │
│  ADD R3, R1, R4   ; 1클럭 스톨                                        │
│  MUL R5, R6, R7                                                      │
│                                                                     │
│  # After (스톨 회피)                                                    │
│  LW  R1, 0(R2)                                                       │
│  MUL R5, R6, R7   ; 다른 명령어로 스톨 활용                               │
│  ADD R3, R1, R4   ; 스톨 없음                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스톨 최소화 팁:                                                       │ │
│  │  ──────────────────                                                  │ │
│  │  1. 포워딩 활용 (RAW 해저드 감소)                                     │ │
│  │  2. 분기 예측 친화적 코드                                              │ │
│  │  3. 캐시 지역성 향상                                                  │ │
│  │  4. Load 후 바로 사용 피하기                                          │ │
│  │  5. 비순차 실행 활용                                                  │ │
│  │  6. 프리페치 활용                                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 파이프라인 스톨: 파이프라인 일시 정지
• 버블: NOP 삽입, 제어 신호 0
• Load-Use: LW 후 바로 사용, 1클럭 스톨
• 분기 페널티: 분기 결정까지 대기
• 캐시 미스: 가장 큰 스톨 원인
• CPI = 1 + (스톨 사이클 / 명령어 수)
• 최소화: 포워딩, 분기 예측, 비순차 실행
```
