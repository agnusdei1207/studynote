# 215. 마이크로프로그래밍 (Microprogrammed Control)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 마이크로코드(ROM)로 제어 신호 생성
> 2. **가치**: 유연함, 수정 용이, CISC에 적합
> 3. **융합**: 제어 메모리, 마이크로명령어, x86

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

마이크로프로그래밍(Microprogrammed Control)은 **제어 메모리(ROM)에 저장된 마이크로코드를 읽어 제어 신호를 생성하는 방식**이다. 하드와이어드보다 느리지만 유연하고, 복잡한 명령어(CISC)에 적합하다.

### 💡 비유: 프로그램 가능한 교통신호

마이크로프로그래밍은 **프로그램 가능한 교통신호**와 같다. 교차로 상황에 맞게 신호 패턴을 바꿀 수 있다. ROM만 교체하면 제어 방식을 바꿀 수 있다. 유연하지만 ROM 읽기 때문에 하드와이어드보다 느리다.

### 마이크로프로그래밍 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                마이크로프로그래밍 구조                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구성 요소】                                                        │
│  ──────────────────                                                  │
│  1. 제어 주소 레지스터 (CAR): μAddr 저장                                │
│  2. 제어 메모리 (CM): 마이크로코드 저장 (ROM)                           │
│  3. 마이크로명령어 레지스터 (μIR): 마이크로명령어 해독                        │
│  4. 마이크로 시퀀서: μAddr 순서 제어                                  │
│  5. 디코더: 제어 신호 생성                                               │
│                                                                     │
│  【구조도】                                                            │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ┌─────────┐    ┌──────────┐    ┌──────────┐                     │ │
│  │  │   CAR   │───▶│    CM    │───▶│   μIR    │                     │ │
│  │  │(μAddr)  │    │  (ROM)   │    │(μCmd)   │                     │ │
│  │  └────┬────┘    └────┬─────┘    └────┬─────┘                     │ │
│  │       │              │               │                           │ │
│  │       │              │               ▼                           │ │
│  │       │              │         ┌──────────────┐                     │ │
│  │       │              │         │   디코더     │                     │ │
│  │       │              │         │  (Control     │                     │ │
│  │       │              │         │   Signals)   │                     │ │
│  │       │              │         └──────────────┘                     │ │
│  │       │              │               │                           │ │
│  │       ▼              ▼               ▼                            │ │
│  │  ┌────────────────────────────────────────────────────────┐       │ │
│  │  │              마이크로 시퀀서                              │       │ │
│  │  │  • μAddr 증가/분기                                            │       │ │
│  │  │  • 조건부 다음 μAddr 선택                                       │       │ │
│  │  └────────────────────────────────────────────────────────┘       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【마이크로명령어 형식】                                               │
│  ────────────────────────                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  필드             내용                                              │ │
│  │  ────────────────  ───────────────────────────────                │ │
│  │  Branch/Next μAddr  다음 마이크로명령어 주소                       │ │
│  │  Condition        조건부 분기 플래그 (Z, C, N, ...)               │ │
│  │  ALU Control      ALU 연산 선택                                   │ │
│  │  Memory Control   메모리 읽기/쓰기                                  │ │
│  │  Register Control 레지스터 전송                                    │ │
│  │  Other Control    기타 제어 신호                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【μAddr 계산】                                                        │
│  ──────────────────                                                  │
│  • 순차: CAR ← CAR + 1                                               │
│  • 분기: CAR ← Branch μAddr                                          │
│  • 조건부: if (Condition) CAR ← Branch μAddr                         │
│  • 매크로: CALL 시 CAR ← 서브루틴 시작                               │
│  • 복귀: RETURN 시 CAR ← 이전 CAR                                    │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 마이크로프로그래밍 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                마이크로프로그래밍 상세                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  1. 유연성: 새로운 명령어 추가 용이                                  │
│  2. 수정 용이: ROM 교체로 수정                                       │
│  3. 복잡한 명령어: CISC에 적합                                        │
│  4. 에뮬레이션: 다른 아키텍처 실행 가능                               │
│  5. 진단: 마이크로코드 디버깅 가능                                    │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  1. 느림: ROM 읽기 지연                                               │
│  2. 복잡함: 추가 하드웨어 필요                                         │
│  3. 전력: ROM 접근 전력 소모                                           │
│  4. 비용: 제어 메모리 비용                                             │
│                                                                     │
│  【수평 vs 수직 마이크로프로그래밍】                                    │
│  ──────────────────────────────                                      │
│  수평 (Horizontal):                                                  │
│  • 긴 마이크로명령어                                                  │
│  • 각 비트가 제어 신호에 직접 매핑                                    │
│  • 빠른 실행                                                          │
│  • 더 큰 ROM                                                          │
│                                                                     │
│  수직 (Vertical):                                                    │
│  • 짧은 마이크로명령어                                                 │
│  • 디코더를 통해 제어 신호 생성                                       │
│  • 더 작은 ROM                                                        │
│  • 느린 실행                                                          │
│                                                                     │
│  【x86 마이크로코드】                                                  │
│  ──────────────────                                                  │
│  • Intel/AMD CPU: CISC → μOp 변환                                    │
│  • μOp 캐시: 변환 결과 캐싱                                           │
│  • 복잡한 명령어: μOp 시퀀스로 분해                                   │
│  • 예: REP MOVSB → 여러 load/store μOp                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  마이크로코드 예시 (LOAD 명령어):                                     │ │
│  │  ───────────────────────────────────                                 │ │
│  │  μAddr  Branch  Cond  ALU   Mem   Reg    설명                      │ │
│  │  ──────  ──────  ────  ────  ────  ────    ────────                  │ │
│  │  000    001    -    -     -     MAR←PC  PC를 MAR로                  │ │
│  │  001    010    -    -     Read  MBR    메모리 읽기                  │ │
│  │  010    011    -    -     -     IR←MBR  MBR를 IR로                  │ │
│  │  011    100    -    -     -     MAR←IR  주소 필드를 MAR로           │ │
│  │  100    101    -    -     Read  MBR    데이터 메모리 읽기           │ │
│  │  101    Exit    -    -     -     AC←MBR  MBR를 AC로                  │ │
│  │  Exit: 다음 명령어로                                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 마이크로프로그래밍 분석

```

│                마이크로프로그래밍 분석                                  │

│                                                                     │
│  【Intel/AMD 마이크로코드 업데이트】                                    │
│  ────────────────────────────                                        │
│  $ cat /proc/cpuinfo | grep microcode                               │
│  microcode       : 0xde                                              │
│                                                                     │
│  # 리눅스에서 마이크로코드 업데이트                                    │
│  $ sudo modprobe microcode                                           │
│  $ cat /sys/devices/system/cpu/cpu0/microcode/reload                │
│                                                                     │
│  【μOp 캐시 분석】                                                    │
│  ──────────────────                                                  │
│  $ perf stat -e idq.all_dsb_cycles_4_uops ./prog                    │
│  # DSB (μOp 캐시) 활용 분석                                          │
│                                                                     │
│  【복잡한 명령어 비교】                                                │
│  ──────────────────                                                  │
│  # REP MOVSB: 문자열 복사                                            │
│  # - CISC: 1 명령어, 내부적으로 여러 μOp                             │
│  # - RISC: 루프 + 여러 LOAD/STORE                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  마이크로코드 팁:                                                    │ │
│  │  ──────────────────                                                │ │
│  │  1. CPU 패치 확인: 마이크로코드 업데이트                            │ │
│  │  2. 복잡한 명령어: 성능 저하 가능 (많은 μOp)                        │ │
│  │  3. 단순한 명령어: μOp 캐시 효율 향상                                │ │
│  │  4. 벤치마크: 마이크로코드 버전별 성능 차이                         │ │
│  │  5. 에뮬레이션: 마이크로코드 기반으로 다른 ISA 실행                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 마이크로프로그래밍: ROM에 저장된 마이크로코드로 제어
• 구성: CAR, CM(ROM), μIR, μSequencer, Decoder
• 마이크로명령어: Branch/Next μAddr, Condition, Control Signals
• 장점: 유연성, 수정 용이, 복잡한 명령어 지원
• 단점: 느림, 복잡함, 추가 비용
• 수평 vs 수직: 긴 μCmd vs 짧은 μCmd+디코더
• 적용: CISC (x86, VAX), 에뮬레이터
```
