# 217. 마이크로명령어 (Microinstruction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제어 메모리의 한 워드, 제어 신호들의 집합
> 2. **가치**: μOp 시퀀스 정의, 유연한 제어
> 3. **융합**: 수평형, 수직형, 조건부 마이크로명령어

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

마이크로명령어(Microinstruction)은 **제어 메모리에 저장된 한 워드로, 하나 이상의 μOp을 수행하기 위한 제어 신호들의 집합**이다. 마이크로코드를 구성하는 기본 단위다.

### 💡 비유: 레시피의 한 줄

마이크로명령어는 **레시피의 한 줄**과 같다. "야채 썰기", "프라이패", "접시에 담기" 같은 한 줄의 지시가 모여서 요리(명령어)가 완성된다.

### 마이크로명령어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                마이크로명령어 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                        │
│  ──────────────────                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐│
│  │    수평형 마이크로명령어                                          │ │
│  │  ┌───────────────────────────────────────────────────────────┐    ││
│  │  │ [C필드1][C필드2]...[C필드n][다음 주소]                      │    ││
│  │  └───────────────────────────────────────────────────────────┘    ││
│  │                                                               │ │
│  │  • 모든 필드가 나란히 배열                                             │ │
│  │  • 디코딩이 간단                                                     │ │
│  │  • 가독성 높음                                                       │ │
│  └─────────────────────────────────────────────────────────────────┘│
│                                                                     │
│  【마이크로명령어 필드】                                                │
│  ────────────────────────                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  필드          내용                    비트 수                             │ │
│  │  ────────────  ────────────────────────  ──────────                 │ │
│  │  ALU 제어       ALU 연산 선택            4비트                             │ │
│  │  시프트 제어    시프트 방향/크기          3비트                             │ │
│  │  레지스터 제어   읽기/쓰기 선택           8비트                             │ │
│  │  메모리 제어     읽기/쓰기/비활성화        3비트                             │ │
│  │  분기 제어      분기 조건/타겟            4비트                             │ │
│  │  다음 주소       다음 μAddr               8비트                             │ │
│  │  조건 플래그    분기 조건 선택           4비트                             │ │
│  │  ──────────────────────────────────────────────────────────────    │ │
│  │  전체 크기: 32~64비트 (시스템에 따라 다름)                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【수평형 vs 수직형】                                                   │
│  ────────────────────────                                            │
│                                                                     │
│  수평형:                                                              │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  [ALU][REG][MEM][BRANCH][NEXT_ADDR]                              │   │
│  │  → 각 기능이 별도 필드로                                           │   │
│  │  → 디코딩 병렬, 빠름                                                  │   │
│  │  → 마이크로명령어 길이 김                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  수직형:                                                              │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  [Opcode][Next]                                                    │   │
│  │  → Opcode로 내부 디코딩                                             │   │
│  │  → 마이크로명령어 짧음                                                  │   │
│  │  → 직렬 실행, 느림                                                      │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【조건부 마이크로명령어】                                               │
│  ──────────────────────────────                                        │
│  if (Zero) Next = 0x10                                               │
│  else     Next = Next + 1                                             │
│  → 조건에 따라 다른 μAddr 선택                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 마이크로명령어 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                마이크로명령어 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【μOp → 마이크로명령어】                                               │
│  ────────────────────────                                            │
│  μOp               마이크로명령어 비트                                   │
│  ─────────────────  ────────────────────                                 │
│  PCout             C1 = 1                                              │
│  MARin             C2 = 1                                              │
│  MEMread           C3 = 1                                              │
│  MBRin             C4 = 1                                              │
│  ALUadd            C5C6 = 00                                          │
│  ALUsub            C5C6 = 01                                          │
│  ACin              C7 = 1                                              │
│                                                                     │
│  【명령어별 마이크로코드 예시】                                           │
│  ──────────────────────────                                            │
│  FETCH:                                                              │
│  Addr  μCmd                                                         │
│  ───── ─────────────────────────────────────────                      │
│  0     PCout, MARin, Read         ; MAR ← PC                          │
│  1     MEMread, MBRin             ; MBR ← M[MAR]                      │
│  2     MBRout, IRin, PCinc        ; IR ← MBR, PC++                      │
│  3     goto DECODE                                                     │
│                                                                     │
│  ADD (R-type):                                                        │
│  Addr  μCmd                                                         │
│  ───── ─────────────────────────────────────────                      │
│  10    R1out, R2out, ALUadd      ; ALU ← R1 + R2                       │
│  11    ALUout, R3in              ; R3 ← ALU                              │
│  12    goto FETCH                                                      │
│                                                                     │
│  【다음 주소 계산】                                                     │
│  ──────────────────                                                  │
│  • 순차: CAR ← CAR + 1                                                 │
│  • 분기: CAR ← 분기 주소                                               │
│  • 조건: if (Condition) CAR ← T1 else CAR ← CAR+1                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  마이크로명령어 최적화:                                              │ │
│  │  ──────────────────────                                            │ │
│  │  1. 나노코드: 자주 쓰는 μOp 조합을 한 μOp로                              │ │
│  │  2. 마이크로명령어 압축: 자주 쓰는 패턴 인코딩                            │ │
│  │  3. 수직형: 코드 길이 감소, 실행은 느림                              │ │
│  │  4. 캐싱: 마이크로명령어 캐시 (μOp 캐시)                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 마이크로명령어: 제어 신호들의 집합 (한 워드)
• 필드: ALU, 레지스터, 메모리, 분기, 다음 주소
• 수평형: 병렬 디코딩, 긴 마이크로명령어
• 수직형: 직렬 디코딩, 짧은 마이크로명령어
• 조건부: 조건에 따라 다른 분기
• 명령어 = 여러 마이크로명령어의 시퀀스
• 최적화: 나노코드, 압축, 캐싱
```
