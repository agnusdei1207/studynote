# 208. 인출 사이클 (Fetch Cycle)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리에서 명령어를 CPU로 가져오는 단계
> 2. **가치**: 모든 명령어의 첫 단계, 파이프라인 IF 단계
> 3. **융합**: 캐시, 분기 예측, 명령어 프리페치

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

인출 사이클(Fetch Cycle)은 **메모리에서 명령어를 읽어 CPU로 가져오는 명령어 사이클의 첫 번째 단계**다. PC(프로그램 카운터)가 가리키는 주소에서 명령어를 읽어 IR(명령어 레지스터)에 저장한다.

### 💡 비유: 도서관에서 책 찾기

인출 사이클은 **도서관에서 책 찾기**와 같다. 도서관 위치(PC)를 확인하고, 서가(메모리)에서 책(명령어)을 꺼내고, 책상(IR)에 놓는다. 그래야 읽을 수 있다.

### 인출 사이클 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                인출 사이클 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 동작】                                                        │
│  ──────────────────                                                  │
│  1. MAR ← PC : PC 값을 주소 레지스터로                               │
│  2. MBR ← M[MAR] : 메모리에서 명령어 읽기                            │
│  3. IR ← MBR : 명령어 레지스터에 저장                                 │
│  4. PC ← PC + 4 : PC 증가 (다음 명령어)                               │
│                                                                     │
│  【데이터패스 경로】                                                  │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │    ┌──────┐      ┌──────┐      ┌──────┐                     │ │
│  │    │  PC  │─────▶│ MAR  │─────▶│ 메모리 │                     │ │
│  │    └──┬───┘      └──────┘      └──┬───┘                     │ │
│  │       │                             │                          │ │
│  │       │ +4                          │ 명령어                   │ │
│  │       │                             ▼                          │ │
│  │       │                       ┌──────────┐                    │ │
│  │       └──────────────────────▶│    MBR   │                    │ │
│  │                               └────┬─────┘                    │ │
│  │                                    │                           │ │
│  │                                    ▼                           │ │
│  │                               ┌──────────┐                    │ │
│  │                               │    IR    │                    │ │
│  │                               └──────────┘                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【마이크로 오퍼레이션】                                               │
│  ────────────────────────                                            │
│  T1: PCout, MARin, Read         ; PC → MAR, 읽기 시작               │
│  T2: MEMread, MBRin             ; 메모리 → MBR                       │
│  T3: MBRout, IRin, PC+4        ; MBR → IR, PC 증가                  │
│                                                                     │
│  【소요 시간】                                                        │
│  ──────────────────                                                  │
│  • 캐시 히트: 1~4 클럭                                              │
│  • 캐시 미스: 100+ 클럭 (메인 메모리)                                 │
│  • 평균: 캐시 적중률에 따라 결정                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  인출 사이클 최적화:                                                │ │
│  │  ──────────────────                                                │ │
│  │  1. 명령어 캐시: 별도 I-Cache                                     │ │
│  │  2. 프리페치: 다음 명령어 미리 읽기                                │ │
│  │  3. 분기 예측: PC 예측                                             │ │
│  │  4. BTB: 분기 타겟 캐시                                            │ │
│  │  5. Trace Cache: 실행 경로 캐시                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 인출 사이클 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                인출 사이클 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파이프라인 IF 단계】                                               │
│  ────────────────────                                                │
│  IF1: PC → I-Cache 인덱스 계산                                       │
│  IF2: I-Cache 접근, 태그 비교                                        │
│  IF3: 명령어 출력, PC 업데이트                                       │
│                                                                     │
│  【분기 예측과 Fetch】                                                │
│  ────────────────────────                                            │
│  • 순차: PC = PC + 4                                                 │
│  • 분기: PC = 예측된 타겟                                            │
│  • BTB 히트: 타겟 바로 사용                                           │
│  • BTB 미스: 순차로 진행, 나중에 수정                                 │
│                                                                     │
│  【I-Cache 미스 처리】                                                │
│  ────────────────────────                                            │
│  1. 파이프라인 스톨                                                   │
│  2. L2 캐시 요청                                                      │
│  3. 메모리 요청                                                       │
│  4. I-Cache 채우기                                                    │
│  5. 파이프라인 재개                                                   │
│                                                                     │
│  【인출 대역폭】                                                       │
│  ──────────────────                                                  │
│  • 단일 인출: 1명령어/클럭                                            │
│  • 듀얼 인출: 2명령어/클럭                                            │
│  • 쿼드 인출: 4명령어/클럭                                            │
│  • 슈퍼스칼라: 발급 폭만큼                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어 인출 병목 현상:                                             │ │
│  │  ──────────────────────                                            │ │
│  │  • Von Neumann 병목: 명령어/데이터 메모리 공유                     │ │
│  │  • 해결: Harvard 아키텍처 (I-Cache, D-Cache 분리)                  │ │
│  │                                                               │ │
│  │  • 캐시 미스 병목: 인출 지연                                        │ │
│  │  • 해결: 프리페치, L2 캐시, 높은 대역폭                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 인출 사이클: 메모리에서 명령어 읽기
• 동작: PC → MAR → 메모리 → MBR → IR, PC+4
• 최적화: I-Cache, 프리페치, 분기 예측
• 성능: 캐시 미스율이 핵심
• 파이프라인: IF 단계
• 병목: Von Neumann 병목, 캐시 미스
```
