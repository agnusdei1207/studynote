# 219. 파이프라인 단계 (Pipeline Stages)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어 처리를 IF-ID-EX-MEM-WB 5단계로 분리
> 2. **가치**: 단계별 중첩 실행으로 처리량 향상
> 3. **융합**: 파이프라인 레지스터, 해저드, 슈퍼스칼라

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

파이프라인 단계(Pipeline Stages)는 **명령어 실행을 여러 독립적인 단계로 나눈 것**으로, MIPS에서는 IF(인출)-ID(해독)-EX(실행)-MEM(메모리)-WB(쓰기)의 5단계가 표준이다.

### 💡 비유: 공장 조립 라인

파이프라인 단계는 **자동차 공장 조립 라인**과 같다. 1단계(차체), 2단계(엔진), 3단계(도장), 4단계(내장), 5단계(검사)로 나뉜다. 각 단계가 동시에 다른 차를 처리하여 완성 속도가 5배 빨라진다.

### 파이프라인 5단계 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 5단계 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【IF: Instruction Fetch - 명령어 인출】                              │
│  ──────────────────────────────────                                  │
│  • 동작: PC → I-Cache → IR                                           │
│  • 리소스: PC, I-Cache, 명령어 메모리                                  │
│  • 출력: 명령어 워드, PC+4                                            │
│  • 시간: 1 클럭                                                       │
│                                                                     │
│  【ID: Instruction Decode - 명령어 해독】                             │
│  ──────────────────────────────────                                  │
│  • 동작: Opcode 해독, 레지스터 파일 읽기                                │
│  • 리소스: 디코더, 레지스터 파일, 제어 유닛                             │
│  • 출력: ReadData1, ReadData2, SignExt(imm), 제어신호                 │
│  • 시간: 1 클럭                                                       │
│                                                                     │
│  【EX: Execute - 실행】                                               │
│  ──────────────────────────────────                                  │
│  • 동작: ALU 연산, 분기 타겟 계산                                      │
│  • 리소스: ALU, MUX, 분기 로직                                         │
│  • 출력: ALU 결과, Zero 플래그                                         │
│  • 시간: 1 클럭                                                       │
│                                                                     │
│  【MEM: Memory Access - 메모리 접근】                                  │
│  ──────────────────────────────────                                  │
│  • 동작: D-Cache 읽기/쓰기                                             │
│  • 리소스: D-Cache, 데이터 메모리                                       │
│  • 출력: 메모리 데이터 (Load 시)                                       │
│  • 시간: 1 클럭                                                       │
│                                                                     │
│  【WB: Write Back - 결과 기록】                                        │
│  ──────────────────────────────────                                  │
│  • 동작: 결과를 레지스터에 저장                                         │
│  • 리소스: 레지스터 파일                                               │
│  • 출력: Reg[rd] ← ALU결과 또는 MemData                               │
│  • 시간: 1 클럭                                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  각 단계별 리소스:                                                    │ │
│  │  ──────────────────                                                  │ │
│  │  IF: PC, I-Cache, Adder (PC+4)                                    │ │
│  │  ID: Register File, Decoder, Sign-Extender                        │ │
│  │  EX: ALU, MUX, Forwarding MUX                                      │ │
│  │  MEM: D-Cache, Data Memory                                         │ │
│  │  WB: MUX (ALU vs Mem), Register File Write Port                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 파이프라인 단계 상세

### 단계별 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 단계 상세                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【IF 단계 상세】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  PC → Adder → I-Cache → IF/ID 레지스터                          │ │
│  │    │                                                            │ │
│  │    └──→ PC+4 (다음 명령어 주소)                                  │ │
│  │                                                               │ │
│  │  입력: PC                                                      │ │
│  │  출력: 명령어(32비트), PC+4                                     │ │
│  │  제어: 없음 (항상 실행)                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ID 단계 상세】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어 → 디코더 → 제어신호                                      │ │
│  │         ↓                                                       │ │
│  │  Reg[rs] → ReadData1 ──┐                                        │ │
│  │  Reg[rt] → ReadData2 ──┼→ ID/EX 레지스터                        │ │
│  │  imm    → SignExt   ──┘                                         │ │
│  │                                                               │ │
│  │  입력: 명령어                                                   │ │
│  │  출력: ReadData1, ReadData2, imm_ext, 제어신호                   │ │
│  │  제어: RegDst, ALUSrc, MemtoReg, RegWrite, ...                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【EX 단계 상세】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ReadData1 ──┐                                                  │ │
│  │              ├→ ALU → ALU결과 ──→ EX/MEM 레지스터               │ │
│  │  ReadData2 ──┤       ↓                                          │ │
│  │        ↑     │    Zero (분기용)                                  │ │
│  │  [포워딩]    │                                                  │ │
│  │              │                                                  │ │
│  │  imm_ext ────┘ (ALUSrc=1 시)                                    │ │
│  │                                                               │ │
│  │  입력: ReadData1, ReadData2/imm, ALU 제어                        │ │
│  │  출력: ALU 결과, Zero 플래그                                      │ │
│  │  제어: ALUOp, ALUSrc                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MEM 단계 상세】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ALU결과 → 주소 → D-Cache → 메모리데이터 → MEM/WB 레지스터       │ │
│  │                    ↓                                            │ │
│  │               Read/Write                                         │ │
│  │                                                               │ │
│  │  입력: ALU 결과 (주소), 쓰기 데이터                                │ │
│  │  출력: 메모리 데이터 (Load 시)                                     │ │
│  │  제어: MemRead, MemWrite                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【WB 단계 상세】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ALU결과 ──┐                                                    │ │
│  │            ├→ MUX → Reg[rd]                                     │ │
│  │  MemData ──┘                                                    │ │
│  │                                                               │ │
│  │  입력: ALU 결과 또는 메모리 데이터                                 │ │
│  │  출력: 레지스터 쓰기                                               │ │
│  │  제어: MemtoReg, RegWrite                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어별 단계 사용:                                                 │ │
│  │  ──────────────────                                                  │ │
│  │  명령어        IF  ID  EX      MEM    WB                           │ │
│  │  ──────────   ──  ──  ───────  ────  ────                           │ │
│  │  ADD rd,rs,rt  O   O   ALU     -     Reg[rd]                       │ │
│  │  LW rt,imm(rs) O   O   주소    Read  Reg[rt]                       │ │
│  │  SW rt,imm(rs) O   O   주소    Write -                             │ │
│  │  BEQ rs,rt     O   O   비교    -     -                             │ │
│  │  J target      O   O   -       -     -                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 파이프라인 단계 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 단계 × 타 과목 융합                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • CPI: 파이프라인 없이 5, 파이프라인 ≈ 1                               │
│  • IPC: 명령어/클럭, 파이프라인에서 ≈ 1                                 │
│  • 클럭 주기: 가장 느린 단계가 결정                                     │
│  • 처리량: 단위 시간당 완료 명령어 수                                    │
│                                                                     │
│  【디지털 논리회로】                                                   │
│  • 파이프라인 레지스터: 각 단계 사이                                     │
│  • MUX: 데이터 선택                                                   │
│  • ALU: EX 단계                                                       │
│  • 디코더: ID 단계                                                     │
│                                                                     │
│  【운영체제】                                                          │
│  • 컨텍스트 스위치: 파이프라인 플러시                                    │
│  • 인터럽트: 파이프라인 드레인                                          │
│  • TLB 미스: MEM 단계에서 처리                                          │
│                                                                     │
│  【컴파일러】                                                          │
│  • 명령어 스케줄링: 해저드 회피                                          │
│  • 지연 슬롯: 분기 후 명령어 채우기                                      │
│  • 루프 언롤링: 파이프라인 활용                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  파이프라인 균형:                                                     │ │
│  │  ──────────────────                                                  │ │
│  │  • 모든 단계가 같은 시간이면 최적                                     │ │
│  │  • 가장 느린 단계가 클럭 주기 결정                                    │ │
│  │  • 불균형 시: 병목 단계 발생                                          │ │
│  │  • 해결: 단계 분할 또는 리소스 추가                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 파이프라인 단계 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 단계 분석                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【클럭 주기 분석】                                                    │
│  ──────────────────                                                  │
│  $ perf stat -e cycles,instructions ./prog                          │
│  cycles: 5,000,000,000                                              │
│  instructions: 5,000,000,000                                        │
│  IPC: 1.00  → CPI ≈ 1                                               │
│                                                                     │
│  【단계별 지연 시간】                                                  │
│  ──────────────────                                                  │
│  IF:  I-Cache 접근 (1~3 클럭)                                         │
│  ID:  레지스터 읽기 + 디코딩 (1 클럭)                                   │
│  EX:  ALU 연산 (1 클럭)                                               │
│  MEM: D-Cache 접근 (1~10+ 클럭)                                       │
│  WB:  레지스터 쓰기 (1 클럭)                                           │
│                                                                     │
│  【현대 CPU 파이프라인 단계】                                          │
│  ────────────────────────────                                       │
│  Intel Skylake: 14~19단계                                            │
│  AMD Zen 3: 19단계                                                   │
│  ARM Cortex-A78: 11단계                                              │
│  → 더 깊은 파이프라인 = 높은 클럭 주파수                                │
│                                                                     │
│  【파이프라인 플러시】                                                  │
│  ──────────────────                                                  │
│  $ perf stat -e branches,branch-misses ./prog                       │
│  branches: 1,000,000,000                                            │
│  branch-misses: 50,000,000                                          │
│  → 5% 분기 미스 = 5% 파이프라인 플러시                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  파이프라인 단계 최적화:                                              │ │
│  │  ──────────────────────                                              │ │
│  │  1. 단계 균형: 모든 단계 비슷한 지연                                  │ │
│  │  2. 캐시 적중률: IF/MEM 단계 지연 감소                                │ │
│  │  3. 분기 예측: 제어 해저드 감소                                       │ │
│  │  4. 포워딩: 데이터 해저드 감소                                        │ │
│  │  5. 단계 분할: 깊은 파이프라인                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• IF: 명령어 인출 (PC → I-Cache → IR)
• ID: 명령어 해독 + 레지스터 읽기
• EX: ALU 연산 + 분기 계산
• MEM: 메모리 읽기/쓰기
• WB: 결과를 레지스터에 저장
• 파이프라인 레지스터: IF/ID, ID/EX, EX/MEM, MEM/WB
• CPI: 파이프라인 없이 5, 있으면 ≈ 1
• 병목: 가장 느린 단계가 클럭 주기 결정
```
