# 242. 토마술로 알고리즘 (Tomasulo's Algorithm)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 예약 스테이션과 레지스터 리네이밍을 결합한 비순차 실행 알고리즘
> 2. **가치**: WAR, WAW 해제, 동적 스케줄링 구현
> 3. **융합**: 현대 CPU 비순차 실행의 기초

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

토마술로 알고리즘(Tomasulo's Algorithm)은 **IBM 360/91에서 처음 구현된 비순차 실행 알고리즘**이다. 예약 스테이션, 레지스터 리네이밍, CDB(Common Data Bus)를 결합하여 WAR, WAW 의존성을 제거하고 동적 스케줄링을 구현한다.

### 💡 비유: 병원 응급실

토마술로 알고리즘은 **병원 응급실 분류**와 같다. 환자가 도착하면 증상을 확인하고 대기실에 배치한다. 의사가 비면 증상이 급한 환자부터 진료한다. 검사 결과가 나오면 모든 대기 환자에게 방송으로 알린다. 자신의 검사 결과를 기다리던 환자는 결과를 듣고 바로 진료를 받을 수 있다.

## Ⅱ. 토마술로 알고리즘 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                토마술로 알고리밍 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【알고리즘 단계】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 발급 (Issue)                                                 │ │
│  │     • 명령어를 예약 스테이션에 할당                                  │ │
│  │     • 소스 오퍼랜드가 준비되면 값 복사                               │ │
│  │     • 소스 오퍼랜드가 준비되지 않으면 태그 저장                       │ │
│  │     • 목적지 레지스터에 태그 설정                                   │ │
│  │                                                               │ │
│  │  2. 실행 (Execute)                                               │ │
│  │     • 모든 오퍼랜드가 준비되면 실행 유닛으로 발급                      │ │
│  │     • 실행 유닛에서 연산 수행                                       │ │
│  │     • 메모리 연산은 주소 계산 후 접근                                │ │
│  │                                                               │ │
│  │  3. 쓰기 (Write Result)                                          │ │
│  │     • CDB가 사용 가능해지면 결과를 CDB에 방송                        │ │
│  │     • 대기 중인 예약 스테이션이 결과를 수신                            │ │
│  │     • 레지스터 파일에 결과 저장                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【토마술로 알고리즘 예시】                                                │
│  ──────────────────                                                  │
│  코드:                                                               │
│    I1: FADD F1, F2, F3    ; F1 = F2 + F3                            │
│    I2: FSUB F4, F1, F5    ; F4 = F1 - F5 (I1 의존)                   │
│    I3: FMUL F6, F4, F7    ; F6 = F4 * F7 (I2 의존)                   │
│    I4: FDIV F8, F9, F10   ; F8 = F9 / F10 (독립)                    │
│                                                                     │
│  RS 상태:                                                             │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RS   연산    목적지  오퍼랜드1  오퍼랜드2  상태                      │ │
│  │  ───  ─────  ──────  ────────  ────────  ─────                   │ │
│  │  1    FADD   F1      F2        F3        실행 중                  │ │
│  │  2    FSUB   F4      RS#1      F5        대기 (RS#1)               │ │
│  │  3    FMUL   F6      RS#2      F7        대기 (RS#2)               │ │
│  │  4    FDIV   F8      F9        F10       실행 가능!                 │ │
│  │  → I4는 I1, I2, I3과 독립 → 먼저 실행 가능                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CDB 방송】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시점    CDB 방송          RS 업데이트                             │ │
│  │  ────    ─────────────     ────────────────                      │ │
│  │  t1     RS#1 = 100         RS#2: F1=100                         │ │
│  │  t2     RS#4 = 2.5         (I4 완료)                             │ │
│  │  t3     RS#2 = 95          RS#3: F4=95                          │ │
│  │  t4     RS#3 = 665         (I3 완료)                             │ │
│  │  → 결과가 나오면 즉시 모든 RS에 방송                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
