# 231. 분기 예측 (Branch Prediction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분기 방향을 미리 예측하여 스톨 감소
> 2. **가치**: 제어 해저드 해결, 20~30% 성능 향상
> 3. **융합**: 정적/동적 예측, BTB, BHT

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

분기 예측(Branch Prediction)은 **분기 명령어의 결과(Taken/Not Taken)를 실행 전에 예측하는 기법**이다. 예측이 맞으면 스톨 없이 실행하고, 틀리면 파이프라인을 플러시하고 올바른 경로로 실행한다.

### 💡 비유: 내비게이션 경로 예측

분기 예측은 **내비게이션이 경로를 미리 예측하는 것**과 같다. 운전자가 집에서 회사로 갈 때, 내비게이션은 평소 가는 길(예측)을 미리 보여준다. 예측이 맞으면 빠르고, 틀리면 다시 계산한다.

### 분기 예측 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                분기 예측 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【분기 예측 필요성】                                                   │
│  ──────────────────                                                  │
│  • 분기 빈도: 프로그램의 15~25%가 분기                                    │
│  • 분기 페널티: 10~20클럭 (깊은 파이프라인)                               │
│  • 스톨 없이 실행하려면 분기 결과 미리 알아야 함                            │
│  • 예측으로 페널티 회피                                                 │
│                                                                     │
│  【예측 종류】                                                          │
│  ──────────────────                                                  │
│  1. 정적 예측 (Static Prediction)                                     │
│     • 컴파일 시 결정                                                    │
│     • 간단한 규칙 기반                                                   │
│     • 정확도: 60~70%                                                  │
│                                                                     │
│  2. 동적 예측 (Dynamic Prediction)                                    │
│     • 실행 시 결정                                                     │
│     • 과거 이력 기반                                                    │
│     • 정확도: 90~99%                                                  │
│                                                                     │
│  【분기 예측 흐름】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  IF 단계:                                                         │ │
│  │  ┌──────────────────────────────────────────────────────────┐   │ │
│  │  │  1. 분기 명령어 감지                                         │   │ │
│  │  │  2. BTB에서 타겟 주소 조회                                   │   │ │
│  │  │  3. BHT에서 예측 결과 조회                                   │   │ │
│  │  │  4. 예측된 주소로 명령어 인출                                 │   │ │
│  │  └──────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  │  EX 단계:                                                         │ │
│  │  ┌──────────────────────────────────────────────────────────┐   │ │
│  │  │  1. 실제 분기 결과 계산                                     │   │ │
│  │  │  2. 예측 vs 실제 비교                                       │   │ │
│  │  │  3. 맞으면: 계속 실행                                        │   │ │
│  │  │  4. 틀리면: 파이프라인 플러시 + 올바른 주소로 인출             │   │ │
│  │  │  5. BHT 업데이트                                            │   │ │
│  │  └──────────────────────────────────────────────────────────┘   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 영향】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  방식              정확도    페널티(틀릴 때)  CPI 기여             │ │
│  │  ───────────────  ──────    ────────────  ──────                  │ │
│  │  스톨만 사용        -        10~20클럭      +0.3~0.5               │ │
│  │  정적 예측         60~70%    10~20클럭     +0.1~0.2                │ │
│  │  동적 예측(1비트)    80~90%   10~20클럭     +0.05~0.1              │ │
│  │  동적 예측(2비트)    90~95%   10~20클럭     +0.02~0.05             │ │
│  │  고급 예측         95~99%    10~20클럭     +0.01~0.02             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예측 정확도와 성능:                                                   │ │
│  │  ──────────────────                                                  │ │
│  │  CPI = 1 + (분기율 × 미스율 × 페널티)                                  │ │
│  │                                                               │ │
│  │  예: 분기율 20%, 페널티 15클럭                                         │ │
│  │  • 미스율 40% (정적): CPI = 1 + 0.2 × 0.4 × 15 = 2.2                │ │
│  │  • 미스율 10% (동적): CPI = 1 + 0.2 × 0.1 × 15 = 1.3                │ │
│  │  • 미스율 5% (고급):  CPI = 1 + 0.2 × 0.05 × 15 = 1.15              │ │
│  │  → 40% → 5%로 개선하면 48% 성능 향상                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 분기 예측 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분기 예측 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정적 예측 방식】                                                     │
│  ──────────────────                                                  │
│  1. 항상 Not Taken: 가장 간단                                         │
│     • 순차적 실행 가정                                                  │
│     • 정확도: 50~60%                                                  │
│                                                                     │
│  2. 항상 Taken: 루프 고려                                               │
│     • 후방 분기는 보통 Taken (루프)                                      │
│     • 정확도: 60~70%                                                  │
│                                                                     │
│  3. BTFN (Backward Taken, Forward Not-taken)                        │
│     • 후방 분기(루프) → Taken                                          │
│     • 전방 분기(if) → Not Taken                                        │
│     • 정확도: 70~80%                                                  │
│                                                                     │
│  4. 프로파일 기반                                                      │
│     • 프로그램 실행 이력 분석                                            │
│     • 컴파일러가 힌트 삽입                                               │
│     • 정확도: 80~90%                                                  │
│                                                                     │
│  【동적 예측 방식】                                                      │
│  ──────────────────                                                  │
│  1. 1비트 예측기                                                       │
│     • 마지막 결과 기억                                                  │
│     • Taken → T, Not Taken → NT                                      │
│     • 문제: 루프 끝에서 2번 틀림                                         │
│                                                                     │
│  2. 2비트 예측기 (강한/약한 상태)                                         │
│     • 00: 강한 NT → NT 예측                                           │
│     • 01: 약한 NT → NT 예측                                           │
│     • 10: 약한 T  → T 예측                                            │
│     • 11: 강한 T  → T 예측                                            │
│     • 연속 2번 틀려야 예측 변경                                          │
│     • 정확도: 90~95%                                                  │
│                                                                     │
│  【고급 예측 기법】                                                       │
│  ──────────────────                                                  │
│  1. 상관 분기 예측 (Correlating Predictor)                            │
│     • 이전 분기 결과 고려                                               │
│     • (m,n) 예측기: m개 이력으로 n비트 선택                               │
│                                                                     │
│  2. 토너먼트 예측기 (Tournament Predictor)                              │
│     • 여러 예측기 경쟁                                                  │
│     • 가장 잘하는 예측기 선택                                            │
│     • Intel: 2개 지역 + 1개 전역 예측기                                  │
│                                                                     │
│  3. 퍼셉트론 예측기 (Perceptron Predictor)                              │
│     • 머신러닝 기반                                                      │
│     • 가중치 학습                                                       │
│     • 정확도: 97~99%                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  현대 CPU 분기 예측:                                                  │ │
│  │  ──────────────────                                                  │ │
│  │  CPU          예측기 종류         정확도                             │ │
│  │  ──────────  ────────────────    ──────                             │ │
│  │  Intel Skylake  토너먼트 + TAGE    > 97%                            │ │
│  │  AMD Zen 3      토너먼트 + TAGE    > 96%                            │ │
│  │  Apple M1       토너먼트           > 98%                            │ │
│  │  ARM Cortex-A78 2비트 + BTB       > 95%                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분기 예측 트레이드오프:                                                 │ │
│  │  ──────────────────────                                              │ │
│  │  예측기          정확도    하드웨어   지연    전력                       │ │
│  │  ─────────────  ──────    ───────  ────   ────                        │ │
│  │  1비트           80~90%    매우 낮음  낮음   낮음                       │ │
│  │  2비트           90~95%    낮음     낮음   낮음                       │ │
│  │  상관 예측       93~97%    중간     중간   중간                       │ │
│  │  토너먼트        95~98%    높음     높음   높음                       │ │

│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅳ. 실무 적용

### 분기 예측 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                분기 예측 분석                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【분기 예측 분석】                                                      │
│  ──────────────────                                                  │
│  $ perf stat -e branches,branch-misses ./prog                       │
│  branches: 1,000,000,000                                            │
│  branch-misses: 15,000,000                                          │
│  → 1.5% 미스율 (97.5% 정확도)                                          │
│                                                                     │
│  【BTB 미스 분석】                                                       │
│  ──────────────────                                                  │
│  $ perf stat -e btb_misses ./prog                                  │
│  → BTB 적중률 확인                                                    │
│                                                                     │
│  【분기 예측 친화적 코드】                                                │
│  ────────────────────────                                            │
│  # Before (예측 어려움)                                               │
│  if (data[random_index] > threshold) {  ; 랜덤 접근                   │
│      // ...                                                          │
│  }                                                                  │
│                                                                     │
│  # After (예측 친화적)                                                 │
│  if (likely(x > 0)) {  ; 95% 확률                                    │
│      // 자주 실행                                                      │
│  } else {                                                           │
│      // 드물게 실행                                                    │
│  }                                                                  │
│                                                                     │
│  【함수 인라이닝】                                                        │
│  ──────────────────                                                  │
│  # Before                                                            │
│  for (i = 0; i < n; i++) {                                          │
│      if (is_valid(data[i])) {  ; 분기                                │
│          // ...                                                      │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  # After (인라이닝 + 분기 제거)                                          │
│  for (i = 0; i < n; i++) {                                          │
│      if ((data[i] & MASK) != 0) {  ; 인라인                           │
│          // ...                                                      │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Spectre 완화】                                                       │
│  ──────────────────                                                  │
│  $ gcc -mretpoline prog.c    ; 리트로폴린 사용                         │
│  → 분기 예측 악용 방지                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분기 예측 최적화 팁:                                                  │ │
│  │  ──────────────────────                                              │ │
│  │  1. 분기 예측 친화적 코드 작성                                        │ │
│  │  2. likely/unlikely 매크로 활용                                      │ │
│  │  3. 함수 인라이닝 (분기 감소)                                        │ │
│  │  4. 루프 언롤링 (분기 감소)                                          │ │
│  │  5. PGO (프로파일 기반 최적화)                                       │ │
│  │  6. 랜덤 분기 피하기                                                 │ │
│  │  7. switch → 테이블 점프                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 분기 예측: 분기 결과를 미리 예측
• 목적: 제어 해저드로 인한 스톨 감소
• 정적 예측: 컴파일 시 결정, 60~70% 정확도
• 동적 예측: 실행 시 결정, 90~99% 정확도
• 2비트 예측기: 강한/약한 상태, 90~95% 정확도
• BTB: 분기 타겟 버퍼
• BHT: 분기 역사 표
• 현대 CPU: 95~99% 정확도
```
