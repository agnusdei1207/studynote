# 228. 데이터 포워딩 (Data Forwarding)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 결과를 WB 전에 바로 전달하여 스톨 감소
> 2. **가치**: RAW 해저드 해결, 30~50% 성능 향상
> 3. **융합**: EX/MEM 포워딩, MEM/WB 포워딩, Load-Use

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

데이터 포워딩(Data Forwarding)은 **이전 명령어의 결과가 레지스터에 기록되기 전에 다음 명령어에게 직접 전달하는 기법**이다. 바이패싱(Bypassing)이라고도 하며, RAW 해저드로 인한 스톨을 줄여 성능을 향상시킨다.

### 💡 비유: 채반에서 프라이팬으로

포워딩은 **요리에서 채반에서 바로 프라이팬으로 옮기는 것**과 같다. 보통은 채반 → 식탁(레지스터) → 프라이팬 순서로 옮기지만, 포워딩은 채반에서 바로 프라이팬으로 옮겨 시간을 절약한다.

### 데이터 포워딩 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 포워딩 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                         │
│  ──────────────────                                                  │
│  • 정상 경로: EX → MEM → WB → 레지스터 → ID → EX                      │
│  • 포워딩: EX → (바로) → 다음 EX                                       │
│  → WB 기다림 없이 결과 사용 가능                                        │
│                                                                     │
│  【포워딩 없이 (스톨 발생)】                                             │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ADD R1, R2, R3                                                  │ │
│  │  SUB R4, R1, R5   ; R1 대기                                      │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    ADD                                                       │ │
│  │   2    SUB   ADD                                                 │ │
│  │   3    -     -     ADD    ← 스톨                                  │ │
│  │   4    -     -     -     ADD                                     │ │
│  │   5    -     SUB    -     -     ADD                              │ │
│  │                                                               │ │
│  │  → 3클럭 스톨                                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【포워딩 있이 (스톨 없음)】                                             │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ADD R1, R2, R3                                                  │ │
│  │  SUB R4, R1, R5   ; R1 포워딩 받음                                 │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    ADD                                                       │ │
│  │   2    SUB   ADD                                                 │ │
│  │   3          SUB   ADD    ← EX→EX 포워딩                          │ │
│  │   4                SUB   ADD                                     │ │
│  │   5                      SUB   ADD                               │ │
│  │                                                               │ │
│  │  → 스톨 없음                                                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【포워딩 경로】                                                        │
│  ──────────────────                                                  │
│  1. EX/MEM → EX: ALU 결과 → 다음 EX                                   │
│  2. MEM/WB → EX: 메모리 또는 ALU 결과 → 다음 EX                          │
│  3. MEM/WB → MEM: Store 데이터 → 다음 MEM                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  포워딩 조건:                                                          │ │
│  │  ──────────────────                                                  │ │
│  │  1. 이전 명령어의 목적지 = 현재 명령어의 소스                            │ │
│  │  2. 이전 명령어가 레지스터에 쓰기 (RegWrite = 1)                        │ │
│  │  3. 목적지가 $0 (zero)이 아님                                         │ │
│  │  4. 더 최근 결과가 우선 (MEM/WB < EX/MEM)                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 데이터 포워딩 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 포워딩 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【포워딩 회로 구조】                                                   │
│  ────────────────────────                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  EX/MEM        ID/EX              MEM/WB                      │ │
│  │  ┌─────┐      ┌─────┐            ┌─────┐                     │ │
│  │  │ALU  │      │ Rs  │            │ALU/ │                     │ │
│  │  │Result    │ Rt  │            │Mem  │                     │ │
│  │  │Rd   │      │     │            │Rd   │                     │ │
│  │  └──┬──┘      └──┬──┘            └──┬──┘                     │ │
│  │     │            │                  │                         │ │
│  │     │    ┌───────┴───────┐          │                         │ │
│  │     │    │               │          │                         │ │
│  │     └───►│   Forwarding  │◄─────────┘                         │ │
│  │          │     MUX       │                                    │ │
│  │          └───────┬───────┘                                    │ │
│  │                  │                                            │ │
│  │                  ▼                                            │ │
│  │             ┌───────┐                                         │ │
│  │             │  ALU  │                                         │ │
│  │             └───────┘                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Forwarding MUX 제어】                                              │
│  ────────────────────────                                            │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ForwardA/B  소스              설명                               │ │
│  │  ──────────  ────────────────  ────────────────────────         │ │
│  │  00          레지스터 파일       기본 (의존성 없음)                  │ │
│  │  01          MEM/WB            이전 명령어 결과 (2사이클 전)        │ │
│  │  10          EX/MEM           바로 전 명령어 결과                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【포워딩 로직】                                                        │
│  ──────────────────                                                  │
│  ForwardA =                                                          │
│    if (EX/MEM.RegWrite && EX/MEM.Rd == ID/EX.Rs && EX/MEM.Rd != 0)  │
│      10  // EX/MEM 포워딩                                            │
│    else if (MEM/WB.RegWrite && MEM/WB.Rd == ID/EX.Rs &&              │
│             MEM/WB.Rd != 0 && !(EX/MEM.RegWrite && EX/MEM.Rd == Rs)) │
│      01  // MEM/WB 포워딩                                            │
│    else                                                              │
│      00  // 레지스터 파일                                              │
│                                                                     │
│  【Load-Use는 포워딩으로도 해결 안 됨】                                  │
│  ────────────────────────────────────                                │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  LW  R1, 0(R2)    ; R1 ← Mem[R2]                                 │ │
│  │  ADD R3, R1, R4   ; R3 ← R1 + R4                                 │ │
│  │                                                               │ │
│  │  문제: LW 결과는 MEM 단계 끝에 준비됨                                 │ │
│  │       ADD는 EX 단계에서 R1 필요                                     │ │
│  │       → 포워딩해도 1사이클 늦음                                      │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────  ────                               │ │
│  │   1    LW                                                        │ │
│  │   2    ADD   LW                                                  │ │
│  │   3    -     -     LW     ← 1클럭 스톨 (불가피)                      │ │
│  │   4          ADD    -     LW    ← 포워딩                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  포워딩 성능 향상:                                                     │ │
│  │  ──────────────────                                                  │ │
│  │  방식              스톨/명령어    CPI    성능                        │ │
│  │  ───────────────  ────────────  ────  ──────                        │ │
│  │  스톨만 사용        2~3클럭       1.5   기준                        │ │
│  │  포워딩            0.2~0.5클럭    1.1   +30~50%                     │ │
│  │  포워딩+스케줄링     0.1클럭      1.05  +40~60%                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 데이터 포워딩 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 포워딩 × 타 과목 융합                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 파이프라인: 포워딩으로 CPI 감소                                      │
│  • 해저드 감지: 포워딩 조건 체크                                        │
│  • MUX: 포워딩 데이터 선택                                             │
│  • 제어 신호: ForwardA, ForwardB                                      │
│                                                                     │
│  【디지털 논리회로】                                                   │
│  • 비교기: Rd == Rs 체크                                              │
│  • MUX: 다중 입력 선택                                                │
│  • 우선순위 인코더: EX/MEM > MEM/WB                                    │
│                                                                     │
│  【컴파일러】                                                          │
│  • 명령어 스케줄링: 포워딩 활용                                          │
│  • Load-Use 회피: LW 후 바로 사용 피함                                  │
│  • 의존성 분석: 포워딩 가능 여부 판단                                    │
│                                                                     │
│  【성능 분석】                                                         │
│  • CPI = 1 + (해저드율 × 스톨 사이클)                                   │
│  • 포워딩 없이: CPI ≈ 1.5                                              │
│  • 포워딩 있이: CPI ≈ 1.1                                              │
│  → 30~40% CPI 감소                                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  포워딩 하드웨어 비용:                                                  │ │
│  │  ──────────────────────                                              │ │
│  │  • 비교기: 2개 (Rs, Rt 각각)                                          │ │
│  │  • MUX: 2개 (ForwardA, ForwardB)                                    │ │
│  │  • 배선: EX/MEM, MEM/WB → EX                                        │ │
│  │  • 비용 증가: 5~10%                                                  │ │
│  │  • 성능 향상: 30~50%                                                 │ │
│  │  → 비용 대비 효과 매우 높음                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 데이터 포워딩 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 포워딩 분석                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【포워딩 효과 측정】                                                   │
│  ────────────────────────                                            │
│  # 포워딩 없이 (시뮬레이션)                                             │
│  CPI: 1.5~2.0                                                        │
│  스톨: 30~40%                                                         │
│                                                                     │
│  # 포워딩 있이 (현대 CPU)                                              │
│  $ perf stat -e cycles,instructions ./prog                          │
│  CPI: 1.0~1.1                                                        │
│  스톨: 5~10%                                                          │
│  → 30~50% 성능 향상                                                   │
│                                                                     │
│  【Load-Use 스톨 분석】                                                 │
│  ────────────────────────                                            │
│  $ perf stat -e ld_blocks.store_forward ./prog                      │
│  → Load-Use 스톨 횟수 확인                                             │
│                                                                     │
│  【컴파일러 최적화】                                                    │
│  ──────────────────                                                  │
│  # Before (Load-Use 발생)                                            │
│  LW  R1, 0(R2)                                                       │
│  ADD R3, R1, R4   ; 1클럭 스톨                                        │
│  MUL R5, R6, R7                                                      │
│                                                                     │
│  # After (컴파일러 스케줄링)                                            │
│  LW  R1, 0(R2)                                                       │
│  MUL R5, R6, R7   ; 스톨 사이에 다른 명령어                             │
│  ADD R3, R1, R4   ; 스톨 없음                                          │
│                                                                     │
│  【포워딩 없는 경우 해결】                                               │
│  ────────────────────────                                            │
│  • MIPS: 지연 슬롯 활용                                                │
│  • VLIW: 컴파일러가 명시적 스케줄링                                       │
│  • 슈퍼스칼라: 비순차 실행로 자동 해결                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  포워딩 활용 팁:                                                       │ │
│  │  ──────────────────                                                  │ │
│  │  1. 현대 CPU는 기본적으로 포워딩 탑재                                   │ │
│  │  2. Load 후 바로 사용 피하기 (1클럭 스톨)                              │ │
│  │  3. 의존성 없는 명령어 배치                                            │ │
│  │  4. 컴파일러 최적화 옵션 사용 (-O2, -O3)                              │ │
│  │  5. 루프 언롤링으로 의존성 분산                                        │ │
│  │  6. SIMD 활용 (벡터화)                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 데이터 포워딩: 결과를 WB 전에 바로 전달
• 목적: RAW 해저드로 인한 스톨 감소
• 경로: EX/MEM → EX, MEM/WB → EX
• ForwardA/B: 00=레지스터, 01=MEM/WB, 10=EX/MEM
• Load-Use: LW 후 바로 사용, 1클럭 스톨 불가피
• 성능 향상: 30~50%
• 비용: 5~10% 하드웨어 증가
• 모든 현대 CPU에 기본 탑재
```
