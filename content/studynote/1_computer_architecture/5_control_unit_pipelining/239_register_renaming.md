# 239. 레지스터 리네이밍 (Register Renaming)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 아키텍처 레지스터를 물리 레지스터로 매핑
> 2. **가치**: WAR, WAW 의존성 제거, 비순차 실행 가능
> 3. **융합**: ROB, 예약 스테이션과 연동

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

레지스터 리네이밍(Register Renaming)은 **아키텍처 레지스터(논리 레지스터)를 물리 레지스터로 동적으로 매핑하는 기술**이다. 이를 통해 가짜 의존성(WAR, WAW)을 제거하고 비순차 실행을 가능하게 한다.

### 💡 비유: 공유 사물함

레지스터 리네이밍은 **공유 사물함 시스템**과 같다. 학생들이 "1번 사물함"을 사용한다고 하자. 매번 같은 사물함을 쓰면 이전 내용물 때문에 기다려야 한다. 하지만 실제로는 빈 사물함을 할당받아 사용한다. "1번"이라는 이름만 같을 뿐, 실제 사물함은 다르다.

### 의존성 문제

```
┌─────────────────────────────────────────────────────────────────────┐
│                레지스터 의존성 문제                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【의존성 종류】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. RAW (Read After Write) - 진짜 의존성                        │ │
│  │     I1: ADD R1, R2, R3   ; R1에 쓰기                              │ │
│  │     I2: SUB R4, R1, R5   ; R1 읽기 (I1 완료 대기)                  │ │
│  │     → 해결 불가: 반드시 기다려야 함                                  │ │
│  │                                                               │ │
│  │  2. WAR (Write After Read) - 가짜 의존성                         │ │
│  │     I1: SUB R4, R1, R5   ; R1 읽기                                │ │
│  │     I2: ADD R1, R2, R3   ; R1에 쓰기 (I1 완료 대기?)               │ │
│  │     → 리네이밍으로 해결 가능                                        │ │
│  │                                                               │ │
│  │  3. WAW (Write After Write) - 가짜 의존성                        │ │
│  │     I1: ADD R1, R2, R3   ; R1에 쓰기                              │ │
│  │     I2: SUB R1, R4, R5   ; R1에 쓰기 (I1 완료 대기?)               │ │
│  │     → 리네이밍으로 해결 가능                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【레지스터 리네이밍 없이】                                               │
│  ──────────────────                                                  │
│  코드:                                                               │
│    I1: LOAD  R1, [addr]    ; 100클럭                                │
│    I2: ADD   R2, R1, R3    ; RAW: I1 대기                           │
│    I3: MUL   R1, R4, R5    ; WAR: I2 대기 (R1 쓰기)                  │
│    I4: SUB   R6, R1, R7    ; RAW: I3 대기                           │
│  → 모든 명령어가 순차 대기                                               │
│                                                                     │
│  【레지스터 리네이밍으로】                                                │
│  ──────────────────                                                  │
│  코드:                                                               │
│    I1: LOAD  R1, [addr]    ; R1 → P1                                │
│    I2: ADD   R2, R1, R3    ; R2 → P2, R1은 P1 사용                   │
│    I3: MUL   R1, R4, R5    ; R1 → P3 (새 물리 레지스터!)              │
│    I4: SUB   R6, R1, R7    ; R6 → P4, R1은 P3 사용                   │
│                                                                     │
│  → I3은 R1에 새로운 P3 할당 → I2와 의존성 없음                           │
│  → I3을 I2보다 먼저 실행 가능!                                          │
│                                                                     │
│  【매핑 테이블 (Register Alias Table)】                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  아키텍처 레지스터    물리 레지스터                                   │ │
│  │  ────────────────  ────────────────                             │ │
│  │  R0                 P10                                          │ │
│  │  R1                 P3      ← 리네이밍됨                          │ │
│  │  R2                 P2                                          │ │
│  │  R3                 P12                                          │ │
│  │  ...               ...                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 레지스터 리네이밍 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                레지스터 리네이밍 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【레지스터 리네이밍 구조】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────────┐                                          │ │
│  │  │  아키텍처 레지스터  │  (프로그래머가 보는 레지스터)                 │ │
│  │  │  R0~R31 (32개)   │                                          │ │
│  │  └────────┬────────┘                                          │ │
│  │           │                                                   │ │
│  │           ↓ 리네이밍                                           │ │
│  │  ┌─────────────────┐      ┌─────────────────┐                 │ │
│  │  │       RAT       │  →   │  물리 레지스터    │                 │ │
│  │  │ (Register Alias │      │  P0~P127 (128개) │                 │ │
│  │  │     Table)      │      │                 │                 │ │
│  │  └─────────────────┘      └─────────────────┘                 │ │
│  │                                                               │ │
│  │  【RAT 구조】                                                       │ │
│  │  ────────────────                                                  │ │
│  │  ┌────────────────────────────────────────────────────────┐     │ │
│  │  │  아키텍처   현재 매핑    유효비트                           │     │ │
│  │  │  ────────  ─────────  ────────                           │     │ │
│  │  │  R0        P10        1                                   │     │ │
│  │  │  R1        P3         1                                   │     │ │
│  │  │  R2        P2         1                                   │     │ │
│  │  │  ...       ...        ...                                 │     │ │
│  │  │  R31       P45        1                                   │     │ │
│  │  └────────────────────────────────────────────────────────┘     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【리네이밍 동작 과정】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 명령어 디코드: ADD R1, R2, R3                                │ │
│  │  2. 소스 레지스터 조회: RAT에서 R2, R3의 물리 레지스터 찾기           │ │
│  │     R2 → P5, R3 → P7                                            │ │
│  │  3. 목적지 레지스터 할당: 프리 리스트에서 새 물리 레지스터 할당         │ │
│  │     R1 → P20 (새로 할당)                                         │ │
│  │  4. RAT 업데이트: R1 → P20으로 매핑 변경                           │ │
│  │  5. 이전 매핑 저장: ROB에 이전 P1 저장 (롤백용)                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【물리 레지스터 풀 크기】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU              아키텍처 레지스터  물리 레지스터                    │ │
│  │  ───────────────  ────────────────  ────────────────            │ │
│  │  Intel Skylake    16 (x86-64)       184                         │ │
│  │  AMD Zen 3        16 (x86-64)       180                         │ │
│  │  Apple M1         31 (ARM64)        380+                        │ │
│  │  ARM Cortex-A78   31 (ARM64)        160                         │ │
│  │  → 물리 레지스터 >> 아키텍처 레지스터                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  $ perf stat -e cycles,retirement.alloc_rate_fpu ./prog           │
│  → FPU 레지스터 할당률 확인                                             │

│  【레지스터 압력 모니터링】                                               │

│  resource_stalls.any: 500,000,000                                  │

│                                                                     │

│  ──────────────────                                                  │
│  1. 레지스터 압력 감소: 변수 수 최소화                                    │
│  2. 루프 언롤링适度: 너무 많으면 레지스터 부족                               │

│  4. SIMD 활용: 벡터 레지스터 사용                                        │
│  5. 스플릿 라이프타임: 변수 사용 범위 최소화                                │
│                                                                     │
│  【아키텍처별 레지스터 수】                                                │
│  ──────────────────                                                  │

│  │  ISA           아키텍처 레지스터  특징                               │ │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 레지스터 리네이밍: 아키텍처 레지스터를 물리 레지스터로 매핑
• 목적: WAR, WAW 의존성 제거
• 구조: RAT (Register Alias Table)
• 물리 레지스터: 아키텍처 레지스터보다 훨씬 많음 (128~380+)
• 이점: 비순차 실행 가능, IPC 향상
• 비용: 하드웨어 복잡도, 면적 증가
• 롤백: 예외/분기 미스 시 이전 매핑 복원
• 보안: Spectre 공격 벡터
```
