# 236. 수퍼스칼라 (Superscalar)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클럭당 여러 명령어를 동시에 인출/실행
> 2. **가치**: IPC > 1, 처리량 2~3배 향상
> 3. **융합**: 다중 파이프라인, 이슈 예측, 비순차 실행

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

슈퍼스칼라(Superscalar)는 **클럭당 여러 명령어를 동시에 인출하여 병렬로 실행하는 CPU 구조**다. 단일 파이프라인은 한 클럭에 하나의 명령어만 완료하지만, 슈퍼스칼라는 여러 명령어를 동시에 처리한다.

### 💡 비유: 다중 계산대 슈퍼스칼라

슈퍼스칼라는 **다중 계산대**와 같다. 일반 계산대: 한 명령어를 5단계로 처리 (5클럭). 슈퍼스칼라: 5명령어를 동시에 처리 (1클럭). 처리량이 5배 향상된다

### 슈퍼스칼라 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                슈퍼스칼라 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 파이프라인 vs 슈퍼스칼라】                                      │
│  ────────────────────────────                                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  기본 파이프라인:                                                │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB                                 │ │
│  │  ────  ────  ────  ────  ────                                  │ │
│  │   1    I1                                                        │ │
│  │   2    I2    I1                                                      │ │
│  │   3    I3    I2    I1                                                │ │
│  │   4    I4    I3    I2    I1                                          │ │
│  │   5    I5    I4    I3    I2    I1                                    │ │
│  │   6    I6    I5    I4    I3    I2    I1                              │ │
│  │                                                               │ │
│  │  → 한 클럭에 하나의 명령어 완료                                     │ │
│  │  → CPI = 1 (이상적)                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  슈퍼스칼라 (2-way):                                               │ │
│  │                                                               │ │
│  │  클럭  IF    ID    EX    MEM   WB   IF    ID    EX    MEM   WB        │ │
│  │  ────  ────  ────  ────  ────  ────  ────  ────  ────  ────                  │ │
│  │   1    I1   I2                                                       │ │
│  │   2    I3   I4    I1   I2                                                 │ │
│  │   3    I5   I6    I3   I4    I1   I2                                          │ │
│  │   4         I7   I8    I5   I6    I3   I4    I1   I2                          │ │
│  │                                                               │ │
│  │  → 한 클럭에 2개의 명령어 완료                                       │ │
│  │  → CPI = 1, IPC = 2 (이상적)                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【슈퍼스칼라 구성 요소】                                                  │
│  ────────────────────────                                            │
│  • 다중 파이프라인: 2개 이상의 독립 파이프라인                            │
│  • 다중 실행 유닛: 2개 이상의 ALU, 메모리 유닛, LOAD/STORE 등                  │
│  • 발급 폭(Issue Width): 클럭당 발급할 수 있는 명령어 수                        │
│  • 병렬 인출: IF 단계에서 여러 명령어 동시 인출                              │
│  • 병렬 실행: EX 단계에서 여러 명령어 동시 실행                              │
│  • 병렬 완료: WB 단계에서 여러 명령어 동시 기록                              │
│                                                                     │
│  【발급 폭 vs 파이프라인 수】                                    │
│  ──────────────────────────                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  발급 폭     파이프라인 수    IPC      예시                          │ │
│  │  ──────────  ────────────  ────     ──────────────────              │ │
│  │  1-way      1           1.0      기본 파이프라인                      │ │
│  │  2-way      2           2.0      2개 명령어 동시                      │ │
│  │  4-way      4           4.0      4개 명령어 동시                      │ │
│  │  6-way      6           6.0      6개 명령어 동시 (고성능)              │ │
│  │  8-way      8           8.0      매우 고성능 (서버)              │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  현대 CPU 발급 폭:                                                     │ │
│  │  ──────────────────                                                  │ │
│  │  CPU             발급 폭    ALU  Load  Store  FP                      │ │
│  │  ───────────────  ──────    ───  ────  ─────  ───                     │ │
│  │  Intel Skylake    6-way     4    2     2     2                       │ │
│  │  AMD Zen 3        6-way     4    2     2     2                       │ │
│  │  Apple M1         8-way     6    3     3     2                       │ │
│  │  ARM Cortex-A78   2-way     2    1     1     2                       │ │
│  │  → 발급 폭이 높을수록 IPC 증가                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 슈퍼스칼라 상세

### 상세 분석

```

│  【파이프라인 구조】                                                  │

│  │                                                               │ │

│  2. 해저드 감지: 복잡한 의존성 검사                                     │
│  3. 발급 로직: 어떤 명령어를 어떤 유닛에 발급?                             │
│  4. 순서 보장: WAW, WAR 해저드 처리                                  │
│  5. 전력 소모: 많은 실행 유닛 활성화                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  슈퍼스칼라 vs VLIW:                                                 │ │
│  │  ─────────────────────                                                │ │
│  │  특성         슈퍼스칼라         VLIW                                  │ │
│  │  ──────────  ──────────────    ────────────────────                  │ │
│  │  하드웨어      복잡           단순                                   │ │
│  │  소프트웨어    단순           복잡 (컴파일러)                            │ │
│  │  발급 폭       동적           정적 (컴파일러 결정)                          │ │
│  │  해저드       하드웨어 처리    컴파일러 회피                              │ │
│  │  유연성        높음           낮음                                  │ │
│  │  → 슈퍼스칼라는 하드웨어로, VLIW는 컴파일러로 복잡도 처리            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

```

│  • 발급 폭 분석: 병렬 처리 능력                                │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 슈퍼스칼라: 클럭당 여러 명령어를 동시에 인출/실행
• 발급 폭(Issue Width): 클럭당 발급할 수 있는 명령어 수
• IPC: 슈퍼스칼라에서 > 1
• CPI: 슈퍼스칼라에서 ≈ 1 / 발급 폭
• 구조: 다중 파이프라인 + 다중 실행 유닛
• 해저드: 포워딩, 분기 예측으로 처리
• 장점: 처리량 향상, 자원 활용
• 복잡도: 하드웨어 복잡, 발급 로직
• 현대 CPU: 4~8-way 슈퍼스칼라
```
