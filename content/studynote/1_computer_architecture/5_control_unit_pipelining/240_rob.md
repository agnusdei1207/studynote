# 240. 재주문 버퍼 (ROB, Reorder Buffer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비순차 실행 명령어를 순차적으로 커밋하는 버퍼
> 2. **가치**: 정확한 예외 처리, 순차 실행 의미 보장
> 3. **융합**: 레지스터 리네이밍, 비순차 실행과 연동

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

재주문 버퍼(ROB, Reorder Buffer)는 **비순차로 실행된 명령어를 프로그램 순서대로 커밋(완료)하는 하드웨어 구조**이다. 비순차 실행의 결과를 순차적으로 아키텍처 상태에 반영하여 정확한 예외 처리와 순차 실행 의미를 보장한다.

### 💡 비유: 번호표 대기줄

ROB는 **번호표 대기줄**과 같다. 은행에서 번호표를 뽑으면 순서가 정해진다. 서비스는 순서와 상관없이 빨리 끝날 수 있다. 하지만 호출은 번호표 순서대로 한다. 10번이 5번보다 먼저 끝나도 5번을 먼저 호출한다.

### 비순차 실행과 ROB

```
┌─────────────────────────────────────────────────────────────────────┐
│                비순차 실행과 ROB                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【비순차 실행 문제】                                                    │
│  ──────────────────                                                  │
│  코드:                                                               │
│    I1: LOAD  R1, [addr]    ; 100클럭                                │
│    I2: ADD   R2, R1, R3    ; I1 의존                                 │
│    I3: MUL   R4, R5, R6    ; 독립, 먼저 실행                           │
│    I4: DIV   R7, R8, R0    ; ← 예외 발생! (0으로 나누기)               │
│                                                                     │
│  실행 순서: I3 → I4(예외) → I1 → I2                                   │
│  → I3이 I4보다 먼저 완료, I3 결과 반영?                                  │
│  → 예외 발생 시점: I1, I2는 아직 실행 중!                                │
│  → 어떤 상태를 보고 예외 처리?                                          │
│                                                                     │
│  【ROB로 해결】                                                         │
│  ──────────────────                                                  │
│                                                                     │
│  ROB (순차적 커밋):                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  HEAD →  I1   I2   I3   I4   ...  ← TAIL                       │ │
│  │          ↓    ↓    ↓    ↓                                      │ │
│  │        대기  대기 완료 예외!                                      │ │
│  │                                                               │ │
│  │  커밋 순서: I1 → I2 → I3 → I4 (예외!)                            │ │
│  │  → I4에서 예외 발견 시 I1, I2, I3은 이미 커밋                     │ │
│  │  → I4 이후 명령어는 플러시                                         │ │
│  │  → 정확한 예외(Precise Exception) 보장                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ROB 구조】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │      ┌─────────────────────────────────────────────────┐      │ │
│  │      │                  ROB 엔트리                       │      │ │
│  │      ├─────────────────────────────────────────────────┤      │ │
│  │      │ 인덱스 │ 명령어 │ 목적지 │ 결과 │ 상태 │ 예외    │      │ │
│  │      ├───────┼───────┼───────┼─────┼─────┼───────┤      │ │
│  │      │   0   │  I1   │  R1   │ 100 │ 완료│  -    │      │ │
│  │      │   1   │  I2   │  R2   │ 50  │ 완료│  -    │      │ │
│  │      │   2   │  I3   │  R4   │ 200 │ 완료│  -    │      │ │
│  │      │   3   │  I4   │  R7   │  -  │ 예외│ DIV-0 │ ← HEAD │ │
│  │      │   4   │  I5   │  ...  │  -  │ 대기│  -    │      │ │
│  │      │  ...  │  ...  │  ...  │ ... │ ... │  ...  │      │ │
│  │      └─────────────────────────────────────────────────┘      │ │
│  │                          ↑                                     │ │
│  │                       TAIL                                     │ │
│  │                                                               │ │
│  │  • HEAD: 다음 커밋할 엔트리                                      │ │
│  │  • TAIL: 다음 입력할 엔트리                                      │ │
│  │  • 순환 버퍼 (Circular Buffer)                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. ROB 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ROB 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ROB 동작 과정】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 할당 (Allocate): 디코드된 명령어를 ROB TAIL에 할당             │ │
│  │  2. 실행 (Execute): 비순차로 실행                                 │ │
│  │  3. 완료 (Complete): 실행 결과를 ROB에 기록                        │ │
│  │  4. 커밋 (Commit): HEAD부터 순차적으로 아키텍처 상태에 반영           │ │
│  │  5. 리타이어 (Retire): ROB에서 제거                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ROB 파이프라인】                                                      │
│  ──────────────────                                                  │
│                                                                     │
│       순차                           비순차              순차          │
│       ────                           ────               ────          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  인출 → 디코드 → 리네이밍 → 발급 → 실행 → 완료 → 커밋              │ │
│  │   ↓      ↓       ↓                ↓      ↓      ↓              │ │
│  │ ROB    ROB     ROB              ROB   ROB    ROB              │ │
│  │ 할당                          결과  대기   리타이어              │ │
│  │                                                               │ │
│  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────────────────────────────┐      │ │
│  │  │I1~I4│→│ROB  │→│발급 │→│ 실행 (비순차)                 │→ 커밋│ │
│  │  │인출 │ │할당 │ │     │ │ I3, I4 먼저 실행 가능         │      │ │
│  │  └─────┘ └─────┘ └─────┘ └─────────────────────────────┘      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ROB 크기】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU             ROB 크기    비순차 윈도우                         │ │
│  │  ───────────────  ────────  ────────────────                     │ │
│  │  Intel Skylake    224        224                                 │ │
│  │  AMD Zen 3        256        256                                 │ │
│  │  Apple M1         630+       630+                                │ │
│  │  ARM Cortex-A78   128        128                                 │ │
│  │  → ROB 크기 = 비순차 윈도우 크기                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ROB의 역할】                                                         │
│  ──────────────────                                                  │
│  1. 순차 커밋: 비순차 실행 결과를 순차적으로 반영                           │
│  2. 정확한 예외: 예외 발생 시 정확한 상태 제공                             │
│  3. 분기 복구: 분기 예측 실패 시 롤백                                    │
│  4. 레지스터 복구: 이전 레지스터 매핑 저장                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. ROB × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ROB × 타 과목 융합                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 비순차 실행: ROB로 순차 커밋 보장                                      │
│  • 예외 처리: 정확한 예외(Precise Exception)                            │
│  • 분기 예측: 예측 실패 시 ROB 롤백                                     │
│  • 인터럽트: 인터럽트 발생 시 정확한 상태                                  │
│                                                                     │
│  【운영체제】                                                          │
│  • 컨텍스트 스위치: ROB 플러시 필요                                       │
│  • 시그널 처리: 정확한 예외로 시그널 전달                                  │
│  • 디버깅: 정확한 명령어 상태                                            │
│                                                                     │
│  【보안】                                                              │
│  • Spectre: ROB 이용한 추측 실행                                       │
│  • 사이드 채널: ROB 타이밍 분석                                          │
│  • 완화: ROB 플러시, 직렬화                                             │
│                                                                     │
│  【신뢰성】                                                            │
│  • ECC: ROB 내용 보호                                                 │
│  • 체크포인트: ROB 상태 저장                                            │
│  • 롤백: 오류 발생 시 ROB 복구                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ROB vs 예약 스테이션:                                                  │ │
│  │  ──────────────────────                                              │ │
│  │  특성          ROB              예약 스테이션                        │ │
│  │  ──────────    ──────────      ──────────                          │ │
│  │  목적          순차 커밋        비순차 발급                          │ │
│  │  순서          프로그램 순서     실행 준비 순서                       │ │
│  │  위치          파이프라인 말단   실행 유닛 앞                         │ │
│  │  데이터        결과 + 상태       오퍼랜드 + 상태                      │ │
│  │  → 두 구조는 상호 보완적                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### ROB 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ROB 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ROB 활용률 측정】                                                    │
│  ──────────────────                                                  │
│  $ perf stat -e cycles,retirement.any ./prog                       │
│  → 리타이어 명령어 수 측정                                              │
│                                                                     │
│  【ROB 스톨 분석】                                                       │
│  ──────────────────                                                  │
│  $ perf stat -e resource_stalls.rob ./prog                         │
│  resource_stalls.rob: 100,000,000                                  │
│  → ROB 가득 참으로 인한 스톨                                             │
│                                                                     │
│  【ROB 최적화 팁】                                                       │
│  ──────────────────                                                  │
│  1. 분기 예측 정확도 향상: ROB 플러시 감소                               │
│  2. 캐시 적중률 향상: ROB 대기 시간 감소                                 │
│  3. 명령어 스케줄링: ROB 활용률 향상                                     │
│  4. ILP 증대: ROB에 더 많은 명령어                                     │
│  5. 메모리 레이턴시 숨김: ROB로 긴 연산 숨김                              │
│                                                                     │
│  【ROB 크기와 성능】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ROB 크기   IPC 증가    적합 용도                                   │ │
│  │  ────────  ────────  ──────────────────                           │ │
│  │  64        기준       저전력, 임베디드                               │ │
│  │  128       +10%      모바일                                        │ │
│  │  224       +20%      데스크탑                                       │ │
│  │  256       +25%      서버                                          │ │
│  │  630+      +40%      고성능 워크스테이션                             │ │
│  │  → ROB 클수록 좋지만, 수확 체감                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• ROB: 비순차 실행 명령어를 순차적으로 커밋
• 목적: 정확한 예외, 순차 실행 의미 보장
• 구조: 순환 버퍼 (HEAD → ... → TAIL)
• 크기: 128~630+ 엔트리
• 동작: 할당 → 실행 → 완료 → 커밋 → 리타이어
• 역할: 순차 커밋, 정확한 예외, 분기 복구
• 비용: 하드웨어 복잡도, 면적 증가
• 보안: Spectre 공격 벡터
```
