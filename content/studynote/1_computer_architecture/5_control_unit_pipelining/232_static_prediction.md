# 232. 정적 분기 예측 (Static Prediction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컴파일 시 결정되는 고정 예측 규칙
> 2. **가치**: 구현 간단, 하드웨어 비용 낮음
> 3. **융합**: BTFN, likely/unlikely, PGO

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

정적 분기 예측(Static Prediction)은 **컴파일 시 또는 아키텍처에 의해 고정된 규칙으로 분기 결과를 예측**하는 방식이다. 실행 이력을 사용하지 않고 단순한 규칙만으로 예측한다.

### 💡 비유: 고정된 운전 경로

정적 분기 예측은 **항상 같은 길로 가는 운전자**와 같다. 교통 상황과 관계없이 항상 A 코스로 간다. 간단하지만, 상황에 따라 비효율적일 수 있다.

### 정적 분기 예측 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                정적 분기 예측 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정적 예측 방식】                                                      │
│  ──────────────────                                                  │
│                                                                     │
│  1. 항상 Not Taken (Always Not Taken)                                │
│  ────────────────────────────────────                                │
│  • 규칙: 모든 분기를 Taken 아님으로 예측                                  │
│  • 장점: 하드웨어 불필요                                                 │
│  • 단점: 루프에서 매번 틀림                                               │
│  • 정확도: 50~60% (분기 특성에 따라 다름)                                  │
│  • 적합: 분기가 드문 코드                                                 │
│                                                                     │
│  2. 항상 Taken (Always Taken)                                        │
│  ─────────────────────────                                          │
│  • 규칙: 모든 분기를 Taken으로 예측                                       │
│  • 장점: 루프에서 유리                                                   │
│  • 단점: if문에서 자주 틀림                                               │
│  • 정확도: 60~70%                                                      │
│  • 적합: 루프 많은 코드                                                   │
│                                                                     │
│  3. BTFN (Backward Taken, Forward Not-taken)                        │
│  ────────────────────────────────────────────                        │
│  • 규칙:                                                              │
│    - 후방 분기 (target < PC) → Taken (루프)                             │
│    - 전방 분기 (target > PC) → Not Taken (if)                         │
│  • 장점: 루프 + if문 조합에 유리                                          │
│  • 단점: 루프 끝에서 틀림                                                 │
│  • 정확도: 70~85%                                                      │
│  • 가장 효과적인 정적 예측                                                │
│                                                                     │
│  4. 프로파일 기반 (Profile-Based)                                      │
│  ────────────────────────────────                                    │
│  • 규칙: 실제 실행 이력 분석 → 힌트 삽입                                   │
│  • 과정:                                                              │
│    1. 프로그램 프로파일 실행                                               │
│    2. 분기 빈도/방향 수집                                                │
│    3. 컴파일러가 힌트 삽입                                                │
│  • 정확도: 85~95%                                                      │
│  • 단점: 대표적 입력 필요                                                 │
│                                                                     │
│  【BTFN 예시】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  # 후방 분기 (루프)                                                 │ │
│  │  loop:                                                           │ │
│  │      ADD R1, R1, 1                                               │ │
│  │      BNE R1, R10, loop  ; target < PC → Taken 예측               │ │
│  │                  ↑                                               │ │
│  │               후방 분기 (루프)                                       │ │
│  │                                                               │ │
│  │  # 전방 분기 (if)                                                   │ │
│  │      CMP R1, 0                                                   │ │
│  │      BEQ R1, skip    ; target > PC → Not Taken 예측              │ │
│  │          ↓                                                       │ │
│  │      ADD R2, R2, 1                                               │ │
│  │  skip:                                                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정적 예측 정확도 비교:                                                 │ │
│  │  ──────────────────────                                              │ │
│  │  방식               정확도    하드웨어   적합 코드                       │ │
│  │  ────────────────  ──────    ───────  ──────────                     │ │
│  │  Always Not Taken   50~60%    없음      분기 드문                     │ │
│  │  Always Taken       60~70%    없음      루프 많은                     │ │
│  │  BTFN               70~85%    매우 낮음   일반적                        │ │
│  │  Profile-Based      85~95%    낮음      최적화된                      │ │
│  │  → BTFN이 가장 실용적                                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 정적 분기 예측 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                정적 분기 예측 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러 힌트】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // GCC/Clang likely/unlikely 매크로                               │ │
│  │  #define likely(x)   __builtin_expect(!!(x), 1)                   │ │
│  │  #define unlikely(x) __builtin_expect(!!(x), 0)                   │ │
│  │                                                               │ │
│  │  // 사용 예                                                       │ │
│  │  if (likely(ptr != NULL)) {    ; 대부분 NULL 아님                   │ │
│  │      // 자주 실행                                                   │ │
│  │  }                                                              │ │
│  │                                                               │ │
│  │  if (unlikely(error)) {        ; 대부분 에러 없음                    │ │
│  │      // 드물게 실행                                                 │ │
│  │  }                                                              │ │
│  │                                                               │ │
│  │  // 컴파일러가 분기 순서 최적화                                        │ │
│  │  if (likely) {                                                  │ │
│  │      // 직접 실행 (점프 없음)                                        │ │
│  │  } else {                                                       │ │
│  │      jmp error_handler  ; 점프                                    │ │
│  │  }                                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【PGO (Profile-Guided Optimization)】                               │
│  ────────────────────────────────────                                │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  # 1단계: 프로파일 생성                                              │ │
│  │  $ gcc -fprofile-generate prog.c -o prog                         │ │
│  │  $ ./prog < input.txt                                            │ │
│  │  → prog.gcda 파일 생성 (분기 이력)                                  │ │
│  │                                                               │ │
│  │  # 2단계: 최적화 컴파일                                               │ │
│  │  $ gcc -fprofile-use prog.c -o prog_opt                         │ │
│  │  → 분기 힌트, 코드 배치 최적화                                        │ │
│  │                                                               │ │
│  │  효과:                                                            │ │
│  │  • 분기 예측 정확도 향상                                              │ │
│  │  • 코드 배치 최적화                                                  │ │
│  │  • 함수 인라이닝 결정 개선                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ARM 분기 힌트 명령어】                                                 │
│  ──────────────────────────                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // ARM 분기 힌트                                                   │ │
│  │  B.EQ  target      ; Equal이면 분기                                │ │
│  │  B.NE  target      ; Not Equal이면 분기                            │ │
│  │                                                               │ │
│  │  // ARM64 분기 힌트                                                 │ │
│  │  B.EQ   target     ; 일반 분기                                       │ │
│  │  B.EQ   target, pt ; 예측: Taken                                   │ │
│  │  B.EQ   target, pn ; 예측: Not Taken                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정적 예측 장단점:                                                      │ │
│  │  ──────────────────                                                  │ │
│  │  장점:                                                               │ │
│  │  • 하드웨어 단순 (BHT 불필요)                                          │ │
│  │  • 전력 소모 낮음                                                     │ │
│  │  • 지연 없음 (BHT 조회 불필요)                                        │ │
│  │  • 컴파일러 제어 가능                                                  │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 정확도 낮음 (60~85%)                                              │ │
│  │  • 동적 행동 대응 불가                                                 │ │
│  │  • 입력 의존적 분기에 약함                                              │ │
│  │  • PGO는 추가 단계 필요                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 정적 분기 예측 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                정적 분기 예측 × 타 과목 융합                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 간단한 파이프라인: 정적 예측만 사용                                    │
│  • 저전력 CPU: BHT 없이 정적 예측                                        │
│  • 임베디드: 비용 절감                                                   │
│  • 초기 RISC: 정적 예측 기반                                            │
│                                                                     │
│  【컴파일러】                                                          │
│  • likely/unlikely 매크로                                            │
│  • 코드 배치 최적화                                                     │
│  • PGO 구현                                                          │
│  • 분기 힌트 생성                                                      │
│                                                                     │
│  【운영체제】                                                          │
│  • 커널 코드: likely/unlikely 많이 사용                                 │
│  • 에러 처리: unlikely로 최적화                                         │
│  • 핫패스: likely로 최적화                                              │
│                                                                     │
│  【프로그래밍】                                                         │
│  • 분기 예측 친화적 코드                                                 │
│  • 자주 발생하는 경우 먼저 처리                                            │
│  • 드문 경우 나중에 처리                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정적 예측 적용 분야:                                                   │ │
│  │  ──────────────────────                                              │ │
│  │  분야           이유                    예측 방식                      │ │
│  │  ───────────  ────────────────────     ───────                      │ │
│  │  임베디드       비용/전력 제약            BTFN                         │ │
│  │  마이크로컨트롤러  단순성                   Not Taken                   │ │
│  │  교육용 CPU     단순성                   BTFN                         │ │
│  │  VLIW          컴파일러 제어             PGO                          │ │
│  │  고성능 CPU     동적 예측과 조합           하이브리드                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 정적 분기 예측 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                정적 분기 예측 분석                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【likely/unlikely 사용】                                              │
│  ────────────────────────                                            │
│  // 커널 코드 예시                                                     │
│  if (likely(cpu_online(cpu))) {                                     │
│      // 대부분 온라인                                                   │
│      schedule_on_cpu(cpu);                                          │
│  }                                                                  │
│                                                                     │
│  if (unlikely(kthread_should_stop())) {                             │
│      // 드물게 발생                                                    │
│      do_exit(0);                                                    │
│  }                                                                  │
│                                                                     │
│  【PGO 적용】                                                           │
│  ──────────────────                                                  │
│  # 1단계: 프로파일 생성                                                 │
│  $ gcc -fprofile-generate -O2 prog.c -o prog_gen                    │
│  $ ./prog_gen < typical_input.txt                                   │
│  → prog.gcda 생성                                                    │
│                                                                     │
│  # 2단계: 최적화 빌드                                                   │
│  $ gcc -fprofile-use -O2 prog.c -o prog_opt                         │
│  → 분기 예측 힌트 자동 삽입                                              │
│                                                                     │
│  # 효과 측정                                                           │
│  $ perf stat -e branch-misses ./prog_opt                            │
│  → 미스율 감소 확인                                                    │
│                                                                     │
│  【분기 예측 친화적 코드】                                                │
│  ────────────────────────                                            │
│  # Before                                                            │
│  if (error_code) {  ; 에러 드묾                                        │
│      handle_error();                                                 │
│  } else {                                                           │
│      normal_path();                                                  │
│  }                                                                  │
│                                                                     │
│  # After                                                             │
│  if (likely(!error_code)) {  ; 정상 흐름 먼저                           │
│      normal_path();                                                  │
│  } else {                                                           │
│      handle_error();                                                 │
│  }                                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정적 예측 최적화 팁:                                                  │ │
│  │  ──────────────────────                                              │ │
│  │  1. likely/unlikely 매크로 활용                                      │ │
│  │  2. 자주 실행되는 코드를 직접 경로로                                   │ │
│  │  3. 드문 에러 처리는 점프 경로로                                       │ │
│  │  4. PGO 활용                                                        │ │
│  │  5. 루프는 후방 분기로 구성                                            │ │
│  │  6. 입력 의존적 분기는 동적 예측 의존                                  │ │
│  │  7. 컴파일러 최적화 옵션 활용                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 정적 분기 예측: 컴파일 시 고정된 예측 규칙
• Always Not Taken: 50~60% 정확도
• Always Taken: 60~70% 정확도
• BTFN: 70~85% 정확도, 가장 실용적
• PGO: 85~95% 정확도
• likely/unlikely: 컴파일러 힌트
• 장점: 하드웨어 단순, 저전력
• 단점: 동적 행동 대응 불가
```
