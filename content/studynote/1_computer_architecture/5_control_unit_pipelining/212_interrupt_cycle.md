# 212. 인터럽트 사이클 (Interrupt Cycle)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 요청 처리, ISR 분기
> 2. **가치**: 예외 상황 대응, 이벤트 기반 처리
> 3. **융합**: OS, 장치 드라이버, 예외 처리

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

인터럽트 사이클(Interrupt Cycle)은 **인터럽트 요청이 있을 때 현재 실행을 중단하고 ISR(인터럽트 서비스 루틴)로 분기하는 단계**다. 하드웨어/소프트웨어 인터럽트, 예외, 시스템 콜 등을 처리한다.

### 💡 비유: 긴급 전화 받기

인터럽트 사이클은 **긴급 전화 받기**와 같다. 하던 일(현재 명령어)을 멈추고, 전화(인터럽트)를 받는다. 중요한 건(컨텍스트)을 적어두고(Save PC), 전화를 받고(ISR), 끝나면 다시 하던 일을 계속한다(Restore).

### 인터럽트 사이클 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 사이클 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 사이클 동작】                                              │
│  ────────────────────────                                            │
│  1. 인터럽트 요청 확인                                                 │
│  2. 현재 PC 저장 (스택 또는 특정 메모리)                              │
│  3. PSW/Flags 저장                                                    │
│  4. ISR 주소로 PC 설정                                                  │
│  5. 인터럽트 비활성화 (선택적)                                          │
│                                                                     │
│  【마이크로 오퍼레이션】                                                │
│  ──────────────────────                                              │
│  T1: SP ← SP - 1               ; 스택 포인터 감소                       │
│  T2: M[SP] ← PC               ; PC를 스택에 저장                        │
│  T3: PC ← ISR_address          ; ISR로 분기                            │
│  T4: (PSW 저장, 인터럽트 비활성화)                                    │
│                                                                     │
│  【인터럽트 처리 흐름】                                                │
│  ──────────────────────                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정상 실행                                                           │ │
│  │  ──────────                                                          │ │
│  │  Fetch → Decode → Execute → ...                                    │ │
│  │       ↓                                                             │ │
│  │  인터럽트 요청                                                       │ │
│  │       ↓                                                             │ │
│  │  인터럽트 사이클                                                      │ │
│  │  ──────────────────                                                  │ │
│  │  1. PC 저장                                                          │ │
│  │  2. PSW 저장                                                          │ │
│  │  3. ISR 분기                                                          │ │
│  │       ↓                                                             │ │
│  │  ISR 실행                                                            │ │
│  │  ────────                                                              │ │
│  │  • 인터럽트 원인 파악                                                  │ │
│  │  • 적절한 처리                                                        │ │
│  │  • IRET (인터럽트 복귀)                                                │ │
│  │       ↓                                                             │ │
│  │  정상 실행 재개                                                       │ │
│  │  ──────────                                                          │ │
│  │  1. PSW 복원                                                          │ │
│  │  2. PC 복원                                                            │ │
│  │  3. 이전 명령어부터 재개                                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 종류】                                                      │
│  ──────────────────                                                  │
│  • 하드웨어 인터럽트: I/O, 타이머, 전원                                 │
│  • 소프트웨어 인터럽트: TRAP, 시스템 콜                               │
│  • 예외(Exception): 0으로 나누기, 오버플로우, 페이지 폴트          │
│  • NMI(Non-Maskable): 마스크 불가능, 치명적                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  인터럽트 vs 폴링:                                                    │ │
│  │  ──────────────────                                                  │ │
│  │  • 인터럽트: 이벤트 발생 시 즉시 반응                                 │ │
│  │  • 폴링: 주기적으로 상태 확인                                         │ │
│  │                                                               │ │
│  │  인터럽트 장점: CPU가 다른 작업 가능, 빠른 반응                      │ │
│  │  폴링 장점: 구현 단순, 예측 가능한 타이밍                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 인터럽트 사이클 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 사이클 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 우선순위】                                                  │
│  ──────────────────────                                              │
│  • 여러 인터럽트 동시 발생 시 우선순위로 처리 순서 결정              │
│  • 고정 우선순위: 0 > 1 > 2 > ...                                     │
│  • 데이지 체인: 하드웨어 연결 순서                                    │
│  • 프로그래밍 가능: 소프트웨어로 조정                                 │
│                                                                     │
│  【인터럽트 벡터】                                                      │
│  ──────────────────                                                  │
│  • 인터럽트 번호 → ISR 주소 매핑                                       │
│  • 인터럽트 벡터 테이블: 메모리의 고정 위치                            │
│  • x86: IDT (Interrupt Descriptor Table)                            │
│  • ARM: Vector Table                                                  │
│                                                                     │
│  【컨텍스트 저장/복원】                                                 │
│  ────────────────────────                                            │
│  저장:                                                                │
│  • PC (복귀 주소)                                                     │
│  • PSW/Flags (상태)                                                   │
│  • 범용 레지스터 (선택적, ISR에서 처리)                                │
│                                                                     │
│  복원:                                                                │
│  • 범용 레지스터                                                       │
│  • PSW/Flags                                                          │
│  • PC                                                                  │
│                                                                     │
│  【인터럽트 지연 (Latency)】                                           │
│  ──────────────────────────                                            │
│  • 현재 명령어 완료까지 대기                                           │
│  • 인터럽트 사이클 수행 시간                                           │
│  • ISR 분기 시간                                                       │
│  • 실시간 시스템에서 중요                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  인터럽트 마스크:                                                    │ │
│  │  ──────────────────                                                  │ │
│  │  • IMR (Interrupt Mask Register): 특정 인터럽트 비활성화            │ │
│  │  • CLI/STI: x86 인터럽트 활성/비활성                                 │ │
│  │  • Priority Level: ARM                                             │ │
│  │                                                               │ │
│  │  → 중요 코드 실행 중 인터럽트 방지                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │
│  【실시간 시스템】                                                      │
│  • 인터럽트 지연 최소화                                                 │
│  • 우선순위 역전 방지                                                   │
│  • 결정론적 타이밍                                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  인터럽트 성능:                                                      │ │
│  │  ──────────────────                                                  │ │
│  │  • 지연: 1~100μs (OS 오버헤드 포함)                                 │ │
│  │  • 처리율: 초당 수십만~수백만                                        │ │
│  │  • 오버헤드: 컨텍스트 저장/복원                                      │ │
