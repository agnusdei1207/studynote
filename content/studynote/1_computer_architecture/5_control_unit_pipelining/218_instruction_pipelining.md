# 218. 명령어 파이프라이닝 (Instruction Pipelining)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어 실행 단계를 중첩시켜 병렬 처리
> 2. **가치**: 처리량(Throughput) 향상, CPI ≈ 1
> 3. **융합**: 해저드, 포워딩, 분기 예측, 슈퍼스칼라

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

명령어 파이프라이닝(Instruction Pipelining)은 **명령어 실행 단계를 여러 단계로 나누고, 각 단계를 중첩시켜 동시에 처리**하는 기술이다. 한 클럭에 여러 명령어가 서로 다른 단계에 있어 처리량이 크게 향상된다.

### 💡 비유: 세탁기 파이프라인

파이프라이닝은 **세탁소 컨베이어 벨트**와 같다. 한 사람은 세탁, 다음은 건조, 다음은 다림질을 한다. 파이프라인에서는 세탁-건조-다림질이 동시에 진행된다. 한 번에 3배 많은 옷을 처리할 수 있다.

### 파이프라인 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                5단계 MIPS 파이프라인                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 5단계】                                                        │
│  ──────────────                                                      │
│  IF (Instruction Fetch): 명령어 인출                                   │
│  ID (Instruction Decode): 명령어 해독, 레지스터 읽기                       │
│  EX (Execute): ALU 연산, 분기 계산                                     │
│  MEM (Memory Access): 메모리 읽기/쓰기                                 │
│  WB (Write Back): 결과를 레지스터에 저장                               │
│                                                                     │
│  【파이프라인 동작】                                                    │
│  ──────────────────                                                  │
│                                                                     │
│  클럭  IF    ID    EX    MEM   WB                                     │
│  ────  ────  ────  ────  ────  ────                                    │
│   1    I1                                                        │
│   2    I2    I1                                                      │
│   3    I3    I2    I1                                                │
│   4    I4    I3    I2    I1                                          │
│   5    I5    I4    I3    I2    I1                                    │
│   6    I6    I5    I4    I3    I2    ← 이상적: 매 클럭 1명령어 완료      │
│                                                                     │
│  【vs 비파이프라인】                                                     │
│  ────────────────────────                                            │
│  비파이프라인:                                                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어 1: [IF][ID][EX][MEM][WB] ──────────────────── 완료       │ │
│  │  명령어 2:                      [IF][ID][EX][MEM][WB] ──── 완료      │ │
│  │  → 5클럭 + 5클럭 = 10클럭 (2명령어)                               │ │
│  │  → CPI = 5                                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  파이프라인:                                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  클럭1: I1[IF]                                                  │ │
│  │  클럭2: I2[IF] I1[ID]                                              │ │
│  │  클럭3: I3[IF] I2[ID] I1[EX]                                        │ │
│  │  클럭4: I4[IF] I3[ID] I2[EX] I1[MEM]                                │ │
│  │  클럭5: I5[IF] I4[ID] I3[EX] I2[MEM] I1[WB]  ← I1 완료           │ │
│  │  클럭6: I6[IF] I5[ID] I4[EX] I3[MEM] I2[WB]  ← I2 완료           │ │
│  │  → 6클럭에 2명령어 완료 = 3클럭/명령어                              │ │
│  │  → CPI ≈ 1 (이상적)                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파이프라인 레지스터】                                                 │
│  ──────────────────────                                              │
│  IF/ID → ID/EX → EX/MEM → MEM/WB                                     │
│  각 단계 사이의 결과를 저장하는 레지스터                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  파이프라인 레지스터 내용:                                           │ │
│  │  ────────────────────────                                          │ │
│  │  IF/ID: PC+4, 명령어                                               │ │
│  │  ID/EX: ReadData1, ReadData2, SignExt(imm), 제어신호, rd           │ │
│  │  EX/MEM: ALU결과, Zero, 쓰기데이터, rd                                │ │
│  │  MEM/WB: 메모리 데이터, ALU결과, rd                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 파이프라인 해저드

### 해저드 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 해저드                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조적 해저드 (Structural Hazard)】                                  │
│  ──────────────────────────────────                                  │
│  • 정의: 하드웨어 리소스 부족으로 발생                                │
│  • 예: 메모리 하나에 IF와 MEM 동시 접근                                │
│  • 해결: 리소스 분리 (Harvard: I-Cache + D-Cache)                      │
│                                                                     │
│  【데이터 해저드 (Data Hazard)】                                        │
│  ────────────────────────────                                        │
│  • 정의: 데이터 의존성으로 발생                                         │
│  • RAW (Read After Write): r1 = r1 + r2; r3 = r1 + r4               │
│    → r1이 아직 WB 전, r3는 ID에서 r1 필요                                │
│  • 해결:                                                             │
│    - 포워딩: EX나 MEM 결과를 바로 전달                                   │
│    - 스톨: 파이프라인 정지                                                   │
│                                                                     │
│  【제어 해저드 (Control Hazard)】                                       │
│  ──────────────────────────────                                      │
│  • 정의: 분기로 파이프라인에 이미 들어간 명령어 무효                    │
│  • 예: BEQ 후 분기 시, 이미 인출된 2개 명령어 버려야                  │
│  • 해결:                                                             │
│    - 분기 예측: 분기 여부 예측                                          │
│    - 지연 분기: 분기 후 명령어 항상 실행 (MIPS)                          │
│    - 스톨: 분기 시 파이프라인 정지                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  해저드 해결 비교:                                                    │ │
│  │  ──────────────────                                                  │ │
│  │  방식         CPI 증가   하드웨어 복잡도                              │ │
│  │  ──────────  ────────  ────────────────                              │ │
│  │  스톨          높음       낮음                                          │ │
│  │  포워딩       낮음       중간                                          │ │
│  │  분기 예측     낮음       높음                                          │ │
│  │  지연 분기     중간       낮음                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 파이프라인 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프라인 × 타 과목 융합                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • CPI: 파이프라인에서 CPI ≈ 1                                         │
│  • IPC: 클럭당 명령어 수, 목표는 IPC > 1                                │
│  • 클럭 주파수: 파이프라인 단계당 1클럭                                 │
│  • 깊이: 단계 수 (5단계, 7단계, 14단계...)                              │
│                                                                     │
│  【컴파일러】                                                          │
│  • 명령어 스케줄링: 해저드 회피                                          │
│  • 지연 슬롯 채우기: MIPS                                               │
│  • 루프 언롤링: 파이프라인 활용                                          │
│  • 소프트웨어 프리페치: 캐시 효율                                          │
│                                                                     │
│  【운영체제】                                                          │
│  • 컨텍스트 스위치: 파이프라인 플러시                                    │
│  • 인터럽트: 파이프라인 드레인                                            │
│  • 시스템 콜: 파이프라인 영향                                            │
│                                                                     │
│  【알고리즘】                                                          │
│  • 분기 예측 친화적 코드                                                 │
│  • 루프 구조와 파이프라인                                               │
│  • 캐시 지역성과 파이프라인                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  파이프라인 성능 방정식:                                              │ │
│  │  ──────────────────────                                            │ │
│  │  CPU Time = IC × CPI × CC                                           │ │
│  │                                                               │ │
│  │  • IC: 명령어 수                                                      │ │
│  │  • CPI: 클럭/명령어 (파이프라인에서 ≈ 1)                              │ │
│  │  • CC: 클럭 주기                                                      │ │
│  │                                                               │ │
│  │  파이프라인 없이: CPI = 5 (각 단계 1클럭)                            │ │
│  │  파이프라인: CPI ≈ 1 + 해저드 오버헤드                                │ │
│  │  → 5배 성능 향상 (이상적)                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 파이프라인: 명령어 실행 단계 중첩 처리
• 5단계: IF → ID → EX → MEM → WB
• 효과: 처리량 향상, CPI ≈ 1
• 해저드: 구조적, 데이터(RAW), 제어(분기)
• 해결: 리소스 분리, 포워딩, 분기 예측
• 성능: CPU Time = IC × CPI × CC
• 최적화: 분기 예측 친화적 코드, 루프 언롤링
```
