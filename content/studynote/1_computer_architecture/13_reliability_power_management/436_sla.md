# 436. SLA (Service Level Agreement)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IT 서비스 품질 수준을 정의한 서비스 제공자와 고객 간 계약
> 2. **가치**: 기대치 관리, 책임 명확화, 보상 체계
> 3. **융합**: SLO, SLI, 가용성, 응답 시간, 처벌

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SLA(Service Level Agreement)는 **IT 서비스 제공자와 고객 간 서비스 품질 수준(가용성, 응답 시간 등)을 정의하고 보장하는 공식 계약**이다.

### 💡 비유: 배달 보증

SLA는 **배달 보증**과 같다. 배달 주문. (서비스 요청) 언제 올까. (불확실) 보장해 줘. (SLA) 30분 내. (SLO) 안 오면 공짜. (처벌) SLA가 똑같다. 서비스 수준, 보장, 보상.

### SLA 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                SLA 개요                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SLA 구성 요소】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. SLO (Service Level Objective):                                         │ │
│  │     • 서비스 수준 목표                                                     │ │
│  │     • 예: 가용성 99.9%                                                    │ │
│  │     • 목표값                                                              │ │
│  │                                                               │ │
│  │  2. SLI (Service Level Indicator):                                         │ │
│  │     • 서비스 수준 지표                                                     │ │
│  │     • 예: 실제 가용성 측정값                                              │ │
│  │     • 측정값                                                              │ │
│  │                                                               │ │
│  │  3. SLA (Service Level Agreement):                                         │ │
│  │     • 서비스 수준 협약                                                     │ │
│  │     • SLO 달성 못 시 보상                                                  │ │
│  │     • 계약서                                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대표적 SLA 지표】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  지표              정의                   목표                    │ │
│  │  ────              ────                   ────                    │ │
│  │  가용성            (총시간-장애시간)/총시간    99.9% 이상           │ │
│  │  응답 시간         요청부터 응답까지          100ms 이하           │ │
│  │  처리량            초당 처리 요청 수          1000 TPS            │ │
│  │  오류율            실패 요청 비율            0.1% 이하            │ │
│  │  RTO               복구 목표 시간            4시간 이내           │ │
│  │  RPO               복구 목표 지점            1시간 이내           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. SLA 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SLA 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【가용성 등급】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  9 수           가용성       연간 장애 시간      용도             │ │
│  │  ────           ────────     ─────────────     ────             │ │
│  │  2개 9 (99%)     99.00%       3.65일           개발/테스트        │ │
│  │  3개 9 (99.9%)   99.90%       8.76시간         일반 서비스        │ │
│  │  4개 9 (99.99%)  99.99%       52.56분          업무용             │ │
│  │  5개 9 (99.999%) 99.999%      5.26분           미션 크리티컬      │ │
│  │  6개 9 (99.9999%)99.9999%     31.54초          초고가용성         │ │
│  │                                                               │ │
│  │  가용성 = (총 시간 - 장애 시간) / 총 시간 × 100                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SLA 보상 체계】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  달성률           보상 (일반적)                                             │ │
│  │  ────────         ──────────────                                            │ │
│  │  99.9% ~ 99.99%   없음                                                      │ │
│  │  99% ~ 99.9%      10% 서비스 크레딧                                         │ │
│  │  95% ~ 99%        25% 서비스 크레딧                                         │ │
│  │  95% 미만         100% 서비스 크레딧                                        │ │
│  │                                                               │ │
│  │  * AWS, Azure, GCP 등 클라우드 제공자 기준                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SLA 계층】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • Bronze: 기본 SLA, 낮은 비용                                            │ │
│  │  • Silver: 표준 SLA, 중간 비용                                            │ │
│  │  • Gold: 프리미엄 SLA, 높은 비용                                           │ │
│  │  • Platinum: 최상위 SLA, 최고 비용                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### SLA 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SLA 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SLO 정의 (SLO.yaml)】                                                          │)
│  ──────────────────                                                  │
│  objectives:                                                        │
│    - name: availability                                             │
│      target: 99.9                                                   │
│      window: 30d                                                    │
│    - name: latency_p99                                              │
│      target: 100ms                                                  │
│      window: 5m                                                     │
│    - name: error_rate                                               │
│      target: 0.1%                                                   │
│      window: 1h                                                     │
│                                                                     │
│  【에러 예산 계산】                                                              │)
│  ──────────────────                                                  │
│  # 30일 기준 99.9% SLO                                              │
│  총 시간 = 30 × 24 × 60 = 43200분                                   │
│  에러 예산 = 43200 × (1 - 0.999) = 43.2분                           │
│  # 즉, 30일 동안 43.2분까지만 장애 허용                                │
│                                                                     │
│  【모니터링 (Prometheus)】                                                        │)
│  ──────────────────                                                  │
│  # 가용성 쿼리                                                         │
│  avg_over_time(uptime)[30d]                                          │
│                                                                     │
│  # 응답 시간 쿼리 (p99)                                               │
│  histogram_quantile(http_request_duration_seconds, 0.99)             │
│                                                                     │
│  【SLA 관리 팁】                                                                  │)
│  ──────────────────                                                  │
│  1. 측정: 모든 지표 수집                                              │
│  2. 목표: 현실적 SLO 설정                                             │
│  3. 모니터링: 실시간 추적                                             │
│  4. 알림: SLO 위반 시 알림                                            │
│  5. 에러 예산: 허용 범위 관리                                         │
│  6. 개선: 지속적 최적화                                               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SLA: 서비스 수준 협약
• SLO: 서비스 수준 목표
• SLI: 서비스 수준 지표
• 가용성: 99.9% (3개 9)
• 응답 시간: p99 < 100ms
• 에러 예산: SLO 위반 허용 범위
• 보상: SLO 미달 시 크레딧
• 계층: Bronze, Silver, Gold, Platinum
• 장점: 기대치 관리, 책임 명확
• 단점: 비용, 복잡성
• 활용: 클라우드, 서비스 관리
```
