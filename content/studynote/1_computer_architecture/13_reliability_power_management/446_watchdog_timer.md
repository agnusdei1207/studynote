# 446. 와치독 타이머 (Watchdog Timer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 정지를 감지하고 자동 복구하는 하드웨어 타이머
> 2. **가치**: 무인 운영, 자동 복구, 고가용성
> 3. **융합**: BMC, NMI, 리셋, 펌웨어

---

## Ⅰ. 개요

### 개념 정의

와치독 타이머(Watchdog Timer, WDT)는 **시스템이 정상적으로 작동하는지 주기적으로 확인하고, 응답이 없으면 자동으로 리셋하는 하드웨어 타이머**다.

### 💡 비유: 심박동 모니터

와치독 타이머는 **심박동 모니터**와 같다. 환자가 있다. (시스템) 심박동을 감지. (킥) 심장이 멈춘다. (타임아웃) 자동으로 충격. (리셋) 살린다. (복구) 와치독이 똑같다. 주기적 확인, 타임아웃 감지, 자동 복구.

│                                                                     │

│  ──────────────────                                                  │

## Ⅳ. 실무 적용

### 와치독 타이머 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                와치독 타이머 분석                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 와치독 설정】                                                          │)
│  ──────────────────                                                  │
│  # 와치독 장치 확인                                                        │
│  $ ls -la /dev/watchdog*                                               │
│                                                                     │
│  # 와치독 모듈 로드                                                        │
│  $ modprobe softdog                                                    │
│                                                                     │
│  # 와치독 상태 확인                                                        │
│  $ cat /proc/sys/watchdog                                              │
│                                                                     │
│  【systemd 와치독】                                                              │)
│  ──────────────────                                                  │
│  # systemd 설정                                                          │
│  # /etc/systemd/system.conf                                            │
│  RuntimeWatchdogSec=60s                                                │
│  RebootWatchdogSec=10min                                               │
│  KExecWatchdogSec=1min                                                 │
│                                                                     │
│  # 서비스에 와치독 추가                                                     │
│  [Service]                                                              │
│  WatchdogSec=30s                                                        │
│                                                                     │
│  【IPMI 와치독】                                                                │)
│  ──────────────────                                                  │
│  $ ipmitool mc watchdog get                                            │
│  # 와치독 상태                                                            │
│                                                                     │
│  $ ipmitool mc watchdog reset                                          │
│  # 와치독 리셋                                                            │
│                                                                     │
│  【와치독 팁】                                                                    │)
│  ──────────────────                                                  │
│  1. 타임아웃: 시스템 부하 고려                                               │
│  2. 킥 주기: 타임아웃의 50% 이내                                             │
│  3. 로깅: 와치독 이벤트 기록                                                 │
│  4. 테스트: 정기적 와치독 테스트                                               │
│  5. NMI: 디버깅용 NMI 액션                                                  │
│  6. 백업: 데이터 보호                                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 와치독 타이머: 시스템 정지 감지 및 자동 복구
• 킥 (Kick): 주기적 타이머 리셋
• 타임아웃: 카운터 만료 시 액션
• 액션: 리셋, NMI, 인터럽트, 로그
• 하드웨어 WDT: 독립 타이머 칩
• 소프트웨어 WDT: 커널 구현
• 장점: 자동 복구, 무인 운영, 신뢰성
• 단점: false positive, 데이터 손실
• 활용: 서버, 임베디드, HA 클러스터
```
