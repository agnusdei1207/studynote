# 447. 메모리 미러링 (Memory Mirroring)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 내용을 실시간 복제하여 오류 발생 시 복구
> 2. **가치**: 데이터 무결성, 무정단 복구, 고가용성
> 3. **융합**: ECC, RAID, DRAM, 서버

---

## Ⅰ. 개요

### 개념 정의

메모리 미러링(Memory Mirroring)은 **메모리의 내용을 두 개의 독립적인 영역에 실시간 복제하여, 하나에 오류가 발생해도 다른 영역에서 데이터를 복구하는 기술**이다.

### 💡 비유: 동전 복사

메모리 미러링은 **동전 복사**와 같다. 문서가 있다. (데이터) 복사해둔다. (미러) 원본이 손상. (오류) 복사본 사용. (복구) 안전하다. 메모리 미러링이 똑같다. 실시간 복제, 오류 감지, 자동 전환.

### 메모리 미러링 개요

│  │  └─────────────┘                 └─────────────┘                │ │
│  │                                                               │ │
│  │  쓰기: 두 영역에 동시 쓰기                                            │ │
│  │  읽기: 한 영역에서 읽기 (오류 시 다른 영역)                              │ │
│  │                                                               │ │

│  │  Adaptive Mirror   적응형 미러링           가변                    │ │
│  │  Sparing           예비 DIMM               75-87.5%               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 보호 기술 비교】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  기술              보호 수준            용량 손실       성능 영향        │ │

│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │

## Ⅱ. 메모리 미러링 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 미러링 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【오류 감지 및 복구】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 정상 동작:                                                            │ │
│  │     • 쓰기: 두 채널에 동시 쓰기                                            │ │
│  │     • 읽기: 기본 채널에서 읽기                                            │ │
│  │                                                               │ │
│  │  2. 오류 감지:                                                            │ │
│  │     • ECC 오류 발생                                                       │ │
│  │     • CRC 불일치                                                         │ │
│  │     • 메모리 컨트롤러 감지                                                 │ │
│  │                                                               │ │
│  │  3. 자동 복구:                                                            │ │
│  │     • 미러 채널에서 데이터 읽기                                           │ │
│  │     • 오류 채널 격리                                                      │ │
│  │     • 무중단 서비스                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【미러링 vs RAID 1】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  특성           메모리 미러링         RAID 1                    │ │
│  │  ────           ────────────         ──────                    │ │
│  │  계층           메모리               스토리지                    │ │
│  │  복제 단위       DIMM/채널            디스크                     │ │
│  │  속도           ns 단위              ms 단위                    │ │
│  │  컨트롤러       메모리 컨트롤러       RAID 컨트롤러              │ │
│  │  오류 감지      ECC                  CRC/패리티                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 무중단 복구                                                          │ │
│  │  • 다중 비트 오류 복구                                                   │ │
│  │  • DIMM 전체 고장 대응                                                   │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 50% 용량 손실                                                        │ │
│  │  • 쓰기 성능 저하                                                        │ │
│  │  • 높은 비용                                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 메모리 미러링: 실시간 메모리 복제
• Full Mirror: 전체 미러링, 50% 용량
• Partial Mirror: 부분 미러링, 75% 용량
• Sparing: 예비 DIMM, 핫 스페어
• 장점: 무중단 복구, 다중 비트 오류 복구
• 단점: 용량 손실, 비용, 쓰기 성능 저하
• ECC: 오류 감지 기반
• 활용: 엔터프라이즈 서버, 미션 크리티컬
• RAID 1: 스토리지 미러링과 유사
```
