# 440. 가상화 (Virtualization)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리적 리소스를 논리적으로 추상화
> 2. **가치**: 리소스 효율, 격리, 유연성
> 3. **융합**: VM, 하이퍼바이저, 컨테이너, 클라우드

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

가상화(Virtualization)은 **물리적 하드웨어 리소스(서버, 저장, 네트워크)를 논리적으로 추상화하여 여러 가상 시스템을 단일 물리 하드웨어에서 실행하는 기술**이다.

### 💡 비유: 아파트 건물

가상화는 **아파트 건물**과 같다. 큰 건물이 있다. (물리 서버) 여러 층이 있다. (가상 머신) 각자 입주. (격리) 독립적 관리. 가상화가 똑같다. 리소스 추상화, 격리, 독립적 관리.

### 가상화 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                가상화 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【가상화 종류】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 전가상화 (Full Virtualization):                                           │ │
│  │     • 하드웨어 완전 추상화                                               │ │
│  │     • 게스트 OS 수정 없음                                                │ │
│  │     • 높은 성능                                                            │ │
│  │     • 예: VMware ESXi, Hyper-V, KVM                                        │ │
│  │                                                               │ │
│  │  2. 반가상화 (Paravirtualization):                                           │ │
│  │     • 하드웨어 일부 노출                                                  │ │
│  │     • 게스트 OS 수정 필요                                                │ │
│  │     • 중간 성능                                                            │ │
│  │     • 예: Xen (일부), 초기 UML                                       │ │
│  │                                                               │ │
│  │  3. 하드웨어 지원 가상화 (Hardware-Assisted):                                 │ │
│  │     • 하드웨어 가속                                                │ │
│  │     • 게스트 OS 수정 필요                                                │ │
│  │     • 높은 성능                                                            │ │
│  │     • 예: KVM (Intel VT-x, AMD-V), Hyper-V                              │ │
│  │                                                               │ │
│  │  4. 컨테이너 (Container):                                              │ │
│  │     • OS 레벨 가상화                                                    │ │
│  │     • 프로세스 격리                                                   │ │
│  │     • 경량 가상화                                                          │ │
│  │     • 예: Docker, containerd, LXC                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 기술】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Intel VT-x:                                                                │ │
│  │     • VMCS (Virtual Machine Control Structure)                              │ │
│  │     • EPT (Extended Page Table)                                             │ │
│  │     • VPID (Virtual Processor ID)                                             │ │
│  │                                                               │ │
│  │  AMD-V:                                                                │ │
│  │     • ASID (Address Space ID)                                               │ │
│  │     • NPT (Nested Page Table)                                             │ │
│  │     • AVIC (Advanced Virtual Interrupt Controller)                           │ │
│  │                                                               │ │
│  │  ARM SVE:                                                                │ │
│  │     • Stage-2 Translation                                               │ │
│  │     • SMM (System Memory Management)                              │ │
│  │     • EL2 (Exception Level 2)                                             │ │
│  │                                                               │ │
│  │  KVM:                                                                 │ │
│  │     • QEMU/KVM (Kernel-based Virtual Machine)                              │ │
│  │     • virtio (I/O 가상화)                                               │ │
│  │     • vhost (네트워크 가상화)                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 가상화 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                가상화 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【VM 관리 (virsh)】                                                              │)
│  ──────────────────                                                  │
│  # VM 목록                                                                │
│  $ virsh list --all                                                    │
│                                                                     │
│  # VM 시작                                                                │
│  $ virsh start <vm_name>                                             │
│                                                                     │
│  # VM 정지                                                                │
│  $ virsh destroy <vm_name>                                            │
│                                                                     │
│  【Docker 컨테이너 관리】                                                    │)
│  ──────────────────                                                  │
│  # 컨테이너 목록                                                              │
│  $ docker ps -a                                                          │
│                                                                     │
│  # 컨테이너 실행                                                              │
│  $ docker run -d <image>                                                │
│                                                                     │
│  # 컨테이너 정지                                                              │
│  $ docker stop <container>                                              │
│                                                                     │
│  【KVM 관리 (libvirt)】                                                     │)
│  ──────────────────                                                  │
│  $ virsh list --all                                                    │
│  # VM 목록                                                                │
│                                                                     │
│  $ virsh dominfo <vm_name>                                             │
│  # VM 상세 정보                                                        │
│                                                                     │
│  $ virsh vcpuinfo <vm_name>                                           │
│  # vCPU 정보                                                             │
│                                                                     │
│  【가상화 팁】                                                                    │)
│  ──────────────────                                                  │
│  1. 리소스: 적절히 할당 (CPU, 메모리)                                          │
│  2. 오버커밋: 피하기 위해 적절한 수                                              │
│  3. 모니터링: 성능 추적                                                    │
│  4. 백업: 정기적 스냅샷                                                    │
│  5. 보안: 네트워크 격리                                                    │
│  6. 마이그레이션: 계획적 수행                                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 가상화: 물리 → 논리 리소스 추상화
• 전가상화: VM, 하이퍼바이저, KVM
• 반가상화: Xen
• 컨테이너: Docker, OS 레벨 가상화
• EPT/NPT/SPT: 페이지 테이블 가상화
• VT-x/AMD-V: 하드웨어 지원
• virtio: I/O 가상화
• 장점: 효율성, 격리, 스냅샷
• 단점: 오버헤드, 성능 손실
• 활용: 클라우드, 서버 통합
```
