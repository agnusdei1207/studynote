# 442. vCPU (Virtual CPU)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 머신에 할당된 논리적 CPU 리소스

> 3. **융합**: 스케줄링, 리소스 풀, 라이센스

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

vCPU(Virtual CPU)는 **가상 머신에 할당된 논리적 CPU 리소스로, 물리적 코어를 시분할하여 여러 VM에 분배할 수 있다.**

### 💡 비유: 컴퓨터 파티션

vCPU는 **컴퓨터 파티션**과 같다. 큰 CPU가 있다. (물리 CPU) 여러 파티션이 있다. (vCPU) 각자 독립. 격리. 운영체제가 관리. vCPU가 똑같다. 코어 분할, 스케줄링, 격리.

## Ⅱ. vCPU 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                vCPU 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【vCPU 스케줄링】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Binpacking: 물리 코어 → vCPU 매핑                                 │ │
│  │     • 다중 vCPU를 하나의 물리 코어에 배치                                   │ │
│  │     • 컨텍스트 스위칭 비용                                              │ │
│  │     • 리소스 경합 향상                                                    │ │
│  │                                                               │ │
│  │  2. Time Slicing: 시간 분할                                                │ │
│  │     • 하나의 vCPU가 시간을 나누어 사용                                        │ │
│  │     • 마이크로커널 아키텍처                                                │ │
│  │     • 저지연                                                              │ │
│  │                                                               │ │
│  │  3. Dedicated: 전용 코어                                                 │ │
│  │     • 특정 vCPU가 전용 코어 사용                                        │ │
│  │     • 높은 성능                                                            │ │
│  │     • 리소스 낭비                                                              │ │
│  │                                                               │ │
│  │  4. Burstable: 버스트 가능                                                 │ │
│  │     • 일시적 고성능 코어 할당                                              │ │
│  │     • 온디맨드 확장                                                            │ │
│  │     • 비용 증가                                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vCPU 라이선스 모델】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  라이선스 모델           설명                    예시                          │ │
│  │  ────────           ────                    ────                          │ │
│  │  기본                vCPU만                Standard_D1, Basic                 │ │
│  │  버스트              버스트 vCPU 포함        Burst                          │ │
│  │  엔터프라이즈          모든 기능 포함           Enterprise                      │ │
│  │  메모리 최적화        메모리 액성화             Memory Optimized              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 유연성: 필요에 따라 리소스 할당                                    │ │
│  │  • 스케일링: 수평/축소 용이                                               │ │
│  │  • 격리: VM 간 독립적 보안                                              │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 오버헤드: 가상화 계층 추가                                             │ │
│  │  • 성능 손실: 물리 하드웨어 대비                                             │ │
│  │  • 복잡성: 리소스 관리 복잡                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### vCPU 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                vCPU 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【vCPU 정보 확인 (Linux)】                                               │)
│  ──────────────────                                                  │
│  $ lscpu                                                             │
│  # CPU 정보 (vCPU 포함)                                               │
│                                                                     │
│  $ cat /proc/cpuinfo                                                  │
│  # CPU 상세 정보                                                        │
│                                                                     │
│  $ nproc                                                             │
│  # CPU 코어 수                                                            │
│                                                                     │
│  【vCPU 상태 (virsh)】                                                        │)
│  ──────────────────                                                  │
│  $ virsh vcpuinfo <vm_name>                                           │
│  # vCPU 정보                                                             │
│                                                                     │
│  $ virsh setvcpus <vm_name> <count>                                    │
│  # vCPU 수 변경                                                         │
│                                                                     │
│  【cgroups vCPU 제한】                                                     │)
│  ──────────────────                                                  │
│  $ cat /sys/fs/cgroup/<path>/cpu.max                                │
│  # 최대 vCPU 수                                                            │
│                                                                     │
│  $ cat /sys/fs/cgroup/<path>/cpu.cfs_quota_usable                   │
│  # 사용 가능 vCPU                                                   │
│                                                                     │
│  【vCPU 팁】                                                                    │)
│  ──────────────────                                                  │
│  1. 모니터링: Steal Time, Ready Time                                             │
│  2. 크기: 워크로드에 맞는 크기                                                │
│  3. 타입: 적절한 vCPU 타입 선택                                              │
│  4. NUMA: NUMA 인식 애플리케이션                                              │
│  5. 오버커밋: 적절한 overcommit                                               │
│  6. cgroups: 리소스 제한 설정                                                │
└─────────────────────────────────────────────────────────────────────┘
```
