# 클럭 게이팅 (Clock Gating)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클럭 게이팅 (Clock Gating)은 현재 사용하지 않는 회로 블록에 클럭 신호의 공급을 차단하여 클럭 트리 (Clock Tree)에서 소비되는 동적 전력을 제거하는 가장 널리 쓰이는 저전력 설계 기법이다.
> 2. **가치**: 현대 SoC에서 클럭 트리가 소비하는 전력은 전체 동적 전력의 30~50%에 달하며, 클럭 게이팅으로 미사용 블록의 클럭을 차단하면 동일 부하에서 전반적인 전력을 20~30% 절감할 수 있다.
> 3. **융합**: 클럭 게이팅은 RTL (Register Transfer Level) 합성 단계에서 자동 삽입 (Automatic Clock Gating)되며, 파워 게이팅(전압 차단)·DVFS(주파수 조정)와 함께 계층화된 저전력 설계의 핵심 3대 기법을 구성한다.

---

## Ⅰ. 개요 및 필요성

디지털 회로에서 플립플롭(FF)은 클럭이 공급되는 한 계속 토글(Toggle) 동작을 수행하며 전력을 소모한다. 코드를 처리하지 않는 유닛(예: FPU가 정수 연산 중, 비디오 코덱이 대기 중)에도 클럭이 공급되면 매 사이클마다 불필요한 전력이 낭비된다.

클럭 게이팅은 이런 미사용 블록의 클럭을 **게이트 (AND/OR 소자)**로 물리적으로 차단하여 그 블록의 플립플롭을 동결시킨다.

**💡 비유**: 사무실에서 사용 안 하는 방의 불을 끄는 것 — 방이 비어있는 동안 조명(클럭)을 차단해 전기(전력)를 아낀다.

```text
┌──────────────────────────────────────────────────────────────┐
│         클럭 게이팅 회로 구조                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [클탁 원본 CLK] ──┬── [게이트 소자] ──▶ [블록 내부 CLK]     │
│                    │       ▲                                 │
│                    │   [Enable 신호]                         │
│                    │   Enable=1: 클럭 통과 (블록 활성)       │
│                    │   Enable=0: 클럭 차단 (블록 동결)       │
│                    │                                         │
│  중요: 게이트 소자는 반드시 래치 기반(ICG) 사용!             │
│  단순 AND 게이트 사용 시 글리치 발생 → 비트 에러 위험        │
│                                                              │
│  ICG (Integrated Clock Gate):                                │
│  래치 + AND 게이트 조합 → Enable 변경을 클럭 하강 에지에만   │
│  캡처하여 글리치 방지 (모든 표준 셀 라이브러리에 포함)       │
└──────────────────────────────────────────────────────────────┘
```

**📢 섹션 요약 비유**: 클럭 게이팅은 TV를 보지 않을 때 플러그를 뽑는 대신 리모컨으로 끄는 것 — 빠르게 껐다 켜지만(클럭 Gates), 대기 전력(누설)은 여전히 존재합니다. (대기 전력까지 0으로 하려면 파워 게이팅이 필요합니다.)

## Ⅱ. 아키텍처 및 핵심 원리

### 자동 클럭 게이팅 (Automatic Clock Gating) — RTL 합성

현대 ASIC·FPGA 설계에서 클럭 게이팅은 수동으로 삽입하지 않는다. EDA(Electronic Design Automation) 도구가 RTL 코드를 분석하여 자동으로 ICG를 삽입한다.

```text
┌──────────────────────────────────────────────────────────────┐
│         RTL 자동 클럭 게이팅 예시 (Verilog)                  │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  // 원래 RTL (게이팅 없음)                                   │
│  always @(posedge clk) begin                                 │
│    if (load_en) reg_data <= data_in;                         │
│  end                                                         │
│                                                              │
│  // EDA 도구가 자동 변환한 게이팅 RTL                        │
│  // ICG 셀 삽입:                                             │
│  // clk_gated = clk AND load_en (글리치 방지 래치 포함)      │
│  always @(posedge clk_gated) begin                           │
│    reg_data <= data_in;                                      │
│  end                                                         │
│                                                              │
│  → load_en=0 시 clk_gated가 차단됨                           │
│  → reg_data FF의 클럭이 멈춰 불필요한 Toggle 전력 = 0        │
└──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 핵심은 `load_en`이 0일 때 레지스터가 어차피 갱신되지 않는다는 것을 합성 도구가 파악하고, 그 조건을 클럭 게이트로 변환한다는 것이다. 데이터가 바뀌지 않는데 클럭이 계속 전파되는 것은 100% 낭비다. 합성 도구는 이런 패턴을 수백만 개의 FF에서 찾아 자동으로 최적화한다.

**📢 섹션 요약 비유**: RTL 자동 게이팅은 스마트 빌딩 센서 — 사람이 없는 방을 자동으로 감지해서 조명(클럭)을 끄는 IoT 시스템과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 3대 저전력 기법 비교

| 기법 | 차단 대상 | 전력 절감 | 웨이크업 속도 | 누설 전력 |
|:---|:---|:---|:---|:---|
| 클럭 게이팅 | 클럭 신호 | 동적 전력 20~30% | 1 클럭 (즉시) | 여전히 존재 |
| DVFS | 전압·주파수 | 동적 전력 50~70% | ~1ms | 감소 |
| 파워 게이팅 | 전원 공급 | 동적 + 누설 전력 ~90% | ~수μs~ms | 0에 가까움 |

클럭 게이팅은 가장 빠른 응답(1 클럭)과 구현 용이성을 갖지만 누설 전력 제거는 불가. 파워 게이팅은 최대 절감이지만 상태 복원 오버헤드가 있어 짧은 유휴 기간에는 부적합.

**📢 섹션 요약 비유**: 클럭 게이팅(조명 끄기), DVFS(전구 밝기 조절), 파워 게이팅(전기 코드 뽑기) — 각자 속도와 절감 효과가 달라 상황에 따라 조합합니다.

## Ⅳ. 실무 적용 및 기술사적 판단

**실무 시나리오**:
1. **스마트폰 SoC**: GPU 블록, DSP, ISP 등 유휴 시 즉시 클럭 게이팅. 화면이 정지 상태에서 대기 전력을 1mW 수준으로 낮추는 핵심.
2. **서버 CPU**: 사용 안 하는 AVX 유닛, 비활성 코어 클럭 게이팅으로 서버 TCO (Total Cost of Ownership) 절감.

**안티패턴**:
- **글리치 발생 게이팅**: 단순 AND 게이트로 클럭 게이팅 → 타이밍 위반·비트 에러 위험. 반드시 ICG (Integrated Clock Gate) 셀 사용.
- **과도한 게이팅** 세분화: ICG 셀 하나당 면적·전력 오버헤드가 있어, 너무 작은 단위 (FF 1~2개)에 삽입하면 오히려 면적 증가.

**📢 섹션 요약 비유**: 방마다 스위치가 있어도, 스위치 자체가 벽 하나를 차지한다면 스위치를 너무 많이 달면 안 되는 것처럼, ICG도 적절한 단위로 묶어야 합니다.

## Ⅴ. 기대효과 및 결론

| 구분 | 클럭 게이팅 없음 | 클럭 게이팅 적용 |
|:---|:---|:---|
| 미사용 블록 전력 | 100% 소모 | 5~10% 수준 (누설만) |
| 전체 칩 전력 절감 | 기준 | 평균 20~30% 절감 |
| 충전 주기 (스마트폰) | 기준 | 수 시간 연장 |

클럭 게이팅은 디지털 IC 설계의 기본 저전력 기법으로, 자동 합성 도구에 의해 현대 ASIC·SoC에 수백만 개가 삽입된다. DVFS·파워 게이팅과 결합하여 최적 전력 관리 계층을 구성한다.

## 👶 어린이를 위한 3줄 비유 설명
1. 클럭 게이팅은 사람이 없는 방 조명 끄기 — 아무도 안 쓰는 회로(방)에 클럭(전기)을 차단해요.
2. 게임 안 할 때 GPU 클럭을 끄고, 사진 안 찍을 때 카메라 칩 클럭을 끄니까 배터리가 훨씬 오래가요.
3. 하지만 전기 코드(전원)는 꽂혀 있어서 아주 조금은 새어나가요 — 완전히 끄려면 파워 게이팅이 필요해요!
