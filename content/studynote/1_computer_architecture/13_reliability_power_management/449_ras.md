# 449. RAS (Reliability, Availability, Serviceability)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템의 신뢰성, 가용성, 서비스 용이성 지표
> 2. **가치**: SLA 달성, 비즈니스 연속성, 고객 신뢰
> 3. **융합**: MTBF, MTTR, 가용성 수식, 수리 보수

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

RAS(Reliability, Availability, Serviceability)는 **시스템의 신뢰성(고장 없이 작동할 확률), 가용성(실제 사용 가능한 비율), 서비스 용이성(수리/유지보수 편의성)을 나타내는 핵심 지표**다.

### 💡 비유: 자동차 품질 지표

RAS는 **자동차 품질 지표**와 같다. 신뢰성은 고장 없이 달리는 것. (Reliability) 가용성은 언제든 탈 수 있는 것. (Availability) 서비스 용이성은 수리하기 쉬운 것. (Serviceability) RAS가 똑같다. 고장 없음, 항상 사용 가능, 수리 편의.

### RAS 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAS 개요                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RAS 구성 요소】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 신뢰성 (Reliability):                                                   │ │
│  │     • 고장 없이 작동할 확률                                                │ │
│  │     • MTBF (Mean Time Between Failures)                                   │ │
│  │     • 목표: 고장 최소화                                                   │ │
│  │                                                               │ │
│  │  2. 가용성 (Availability):                                                 │ │
│  │     • 실제 사용 가능한 비율                                                │ │
│  │     • MTBF / (MTBF + MTTR)                                                │ │
│  │     • 목표: 99.9% 이상                                                    │ │
│  │                                                               │ │
│  │  3. 서비스 용이성 (Serviceability):                                         │ │
│  │     • 수리 및 유지보수 편의성                                              │ │
│  │     • MTTR (Mean Time To Repair)                                          │ │
│  │     • 목표: 신속한 복구                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【핵심 지표】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  지표              정의                   단위                 │ │
│  │  ────              ────                   ────                 │ │
│  │  MTBF              고장 간 평균 시간         시간                 │ │
│  │  MTTR              수리 평균 시간           시간                 │ │
│  │  MTTF              고장까지 평균 시간        시간                 │ │
│  │  가용성            MTBF/(MTBF+MTTR)        %                    │ │
│  │  SLA               서비스 수준 협약          %                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. RAS 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAS 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【가용성 등급】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  등급         가용성       연간 다운타임     용도              │ │
│  │  ────         ───────      ────────────     ────              │ │
│  │  1개 9        90%          36.5일           개발/테스트        │ │
│  │  2개 9        99%          3.65일           일반 서비스        │ │
│  │  3개 9        99.9%        8.76시간         중요 서비스        │ │
│  │  4개 9        99.99%       52.6분           금융/전자상거래    │ │
│  │  5개 9        99.999%      5.26분           통신/결제          │ │
│  │  6개 9        99.9999%     31.5초           초고신뢰 시스템    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MTBF vs MTTF】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  MTBF (Mean Time Between Failures):                                        │ │
│  │  • 고장 간 평균 시간                                                       │ │
│  │  • 수리 가능한 시스템                                                      │ │
│  │  • MTBF = MTTF + MTTR                                                     │ │
│  │                                                               │ │
│  │  MTTF (Mean Time To Failure):                                              │ │
│  │  • 고장까지 평균 시간                                                      │ │
│  │  • 수리 불가능한 시스템                                                    │ │
│  │  • 교체만 가능                                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAS 향상 기법】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  신뢰성 향상:                                                                │ │
│  │  • 이중화 (Redundancy)                                                      │ │
│  │  • ECC 메모리                                                               │ │
│  │  • 결함 허용 (Fault Tolerance)                                             │ │
│  │                                                               │ │
│  │  가용성 향상:                                                                │ │
│  │  • 고가용성 클러스터 (HA Cluster)                                           │ │
│  │  • 로드 밸런싱                                                              │ │
│  │  • 장애 조치 (Failover)                                                     │ │
│  │                                                               │ │
│  │  서비스 용이성 향상:                                                          │)
│  │  • 핫 스왑 가능 부품                                                        │ │
│  │  • 모듈형 설계                                                              │ │
│  │  • 원격 관리                                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### RAS 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAS 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【가용성 계산】                                                                  │)
│  ──────────────────                                                  │
│  # MTBF = 10000시간, MTTR = 1시간                                    │
│  가용성 = MTBF / (MTBF + MTTR)                                       │
│        = 10000 / (10000 + 1)                                        │
│        = 0.9999 = 99.99%                                            │
│                                                                     │
│  【SLA 모니터링】                                                                │)
│  ──────────────────                                                  │
│  $ uptime                                                            │
│  # 시스템 가동 시간                                                    │
│                                                                     │
│  $ cat /proc/uptime                                                  │
│  # 가동 시간 (초)                                                      │
│                                                                     │
│  【HA 클러스터 확인】                                                              │)
│  ──────────────────                                                  │
│  $ pcs status                                                        │
│  # Pacemaker 상태                                                     │
│                                                                     │
│  $ crm_mon -1                                                        │
│  # 클러스터 모니터링                                                   │
│                                                                     │
│  【RAS 향상 팁】                                                                  │)
│  ──────────────────                                                  │
│  1. 이중화: SPOF 제거                                                 │
│  2. 모니터링: 실시간 감시                                              │
│  3. 백업: 정기/자동                                                   │
│  4. 장애 조치: 자동화                                                 │
│  5. DR: 재해 복구 계획                                                │
│  6. 테스트: 정기적 장애 테스트                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• RAS: 신뢰성, 가용성, 서비스 용이성
• MTBF: 고장 간 평균 시간
• MTTR: 수리 평균 시간
• 가용성: MTBF/(MTBF+MTTR)
• 3개 9: 99.9% = 8.76시간/년
• 5개 9: 99.999% = 5.26분/년
• 이중화: SPOF 제거
• HA: 고가용성 클러스터
• Failover: 장애 조치
• SLA: 서비스 수준 협약
• 핫 스왑: 운영 중 교체
```
