# 전력 소모 (Power Consumption)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디지털 회로의 전력 소모는 동적 전력 (Dynamic Power: P = αCV²f)과 정적 전력 (Static Power: 누설 전류)의 합으로 구성되며, 이 두 성분이 현대 나노 공정 CPU 설계의 가장 큰 도전인 **전력 장벽 (Power Wall)**의 근원이다.
> 2. **가치**: 전압(V)이 전력에 제곱으로 영향을 미치므로, 전압을 10% 낮추면 동적 전력은 약 19% 줄어든다. 이 물리 법칙이 DVFS, 저전압 공정, 소전력 아키텍처 설계의 핵심 동기다.
> 3. **융합**: 전력 소모 분석은 회로 설계(EDA), 운영체제(DVFS), 데이터센터 운영(PUE, TCO)에 이르는 모든 계층에 영향을 미치며, AI 가속기·데이터센터의 '에너지 효율(FLOPS/W)'이 21세기 하드웨어 경쟁의 핵심 지표가 되었다.

---

## Ⅰ. 개요 및 필요성

무어의 법칙으로 동일 칩 면적에 들어가는 트랜지스터 수가 2배씩 늘었지만, 전력 소모가 비례해서 늘면 CPU는 녹아버린다. 2004년 무렵 클럭 경쟁이 멈춘 이유도 전력 한계 때문이었다.

**💡 비유**: CPU 성능을 높이는 것은 더 강한 불꽃(클럭·전압)을 쓰는 것 — 결국 화재(과열)가 나지 않는 선에서 설계해야 한다.

```text
┌──────────────────────────────────────────────────────────────┐
│         전력 소모 성분 분석                                 │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  총 전력 = 동적 전력 + 정적 전력                            │
│                                                              │
│  ① 동적 전력 (Dynamic Power):                              │
│     P_dyn = α × C × V² × f                                 │
│     α: 스위칭 활동 계수 (0~1)                               │
│     C: 부하 커패시턴스 (팬아웃·배선 길이에 비례)           │
│     V: 공급 전압                                             │
│     f: 클럭 주파수                                           │
│                                                              │
│     → 전압이 가장 강력한 레버: V² 관계                     │
│        V를 20% 줄이면 전력 36% 감소!                       │
│                                                              │
│  ② 정적 전력 (Static Power / 누설 전류):                   │
│     P_sta = V × I_leak                                      │
│     I_leak: 서브스레숄드 누설 + 게이트 누설                 │
│                                                              │
│     나노 공정에서 트랜지스터 수 증가 → I_leak 급증         │
│     7nm 이하에서 정적 전력이 전체 전력의 30~50%를 차지!   │
└──────────────────────────────────────────────────────────────┘
```

**📢 섹션 요약 비유**: 전력 소모는 두 가지 고지서 — 일하면서 쓰는 전기(동적)와 대기하면서도 새는 전기(정적). 현대 CPU는 이 두 전기세를 모두 잡아야 합니다.

## Ⅱ. 아키텍처 및 핵심 원리

### 전력 벽 (Power Wall)

```text
┌──────────────────────────────────────────────────────────────┐
│         Power Wall — 클럭 상승 한계의 물리적 원인          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [클럭 상승] ──▶ [전압도 높여야 타이밍 마진 확보]          │
│               ──▶ [전력 P = CV²f → 제곱 폭증]              │
│               ──▶ [열 발생 폭증]                            │
│               ──▶ [TjMax 도달 → 스로틀링]                  │
│                                                              │
│  결과:                                                       │
│  2004년 Intel Prescott (Pentium 4): 4GHz, 130W → 실패       │
│  단일 코어 클럭 상승 포기 → 멀티코어 전환의 역사적 배경    │
│                                                              │
│  현대의 해법:                                                │
│  ● 클럭은 유지하되 코어 수 증가 (Multicore)                │
│  ● 전압 낮추기 (DVFS, 저전압 공정)                         │
│  ● 전선 짧게 (칩렛, 3D 패키징)                             │
│  ● 불필요한 회로 끄기 (Power Gating, Clock Gating)         │
└──────────────────────────────────────────────────────────────┘
```

### 에너지 효율 지표

- **FLOPS/W**: AI 가속기 선택의 핵심 지표. NVIDIA H100: ~2,000 TFLOPS / 700W ≈ 2.86 TFLOPS/W
- **pJ/op (pico-Joules per operation)**: 신경망 추론 칩 에너지 효율 지표

**📢 섹션 요약 비유**: 자동차 연비(km/L)처럼 — AI 시대엔 FLOPS/W(연산 당 전력)이 CPU/GPU 구매의 가장 중요한 기준이 되고 있습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 성능 계층별 전력 특성

| 계층 | 대표 소자 | 전력 | 주요 소모 원인 |
|:---|:---|:---|:---|
| 에지 IoT | Arm Cortex-M | 1mW~수십mW | 동적 전력 주도 |
| 스마트폰 SoC | Apple M 시리즈 | 2~10W | 동적+정적 균형 |
| 고성능 CPU | AMD EPYC | 120~400W | 정적 전력 비중 증가 |
| AI GPU | NVIDIA H100 | 700W | 극한 동적 전력 |
| 데이터센터 랙 | 42U 서버 풀 로드 | 10~40kW | 전체 설비 냉각 필수 |

**📢 섹션 요약 비유**: IoT 센서는 건전지 하나로 1년, 스마트폰은 하루 충전, 데이터센터 GPU랙은 아파트 단지 전력 — 계층에 따라 전력 설계 철학이 완전히 달라집니다.

## Ⅳ. 실무 적용 및 기술사적 판단

**실무 시나리오**:
1. **데이터센터 TCO**: 서버 전력 소모 × PUE × 전기요금은 서버 구매 비용의 3~5년 내 추월. FLOPS/W 기준 GPU 선택이 ROI의 핵심.
2. **IoT 배터리 수명**: Cortex-M 기반 센서가 Sleep C-State + 클럭 게이팅으로 평균 소비 10μW → AA 배터리 5년 수명 달성.

**안티패턴**:
- **성능만 보고 전력 무시**: GPU 구매 시 TFLOPS만 비교 → 전력 비용이 3년 후 서버 구매비보다 커지는 사례.
- **타이밍 마진 초과 언더볼팅**: 전압을 너무 낮추면 타이밍 위반으로 시스템 불안정 → 안정성 검증 필수.

**📢 섹션 요약 비유**: 전력 비용 무시는 연비 0km/L 차를 성능만 보고 사는 것 — 주유비(전기세)가 차값(서버)을 금방 넘어섭니다.

## Ⅴ. 기대효과 및 결론

| 구분 | 전압 10% 감소 | 주파수 10% 감소 |
|:---|:---|:---|
| 동적 전력 변화 | -19% (V²효과) | -10% |
| 성능 영향 | 타이밍 마진 감소 | -10% |

전력 소모는 현대 컴퓨팅의 가장 근본적 제약 조건이다. 동적 전력은 DVFS·클럭 게이팅으로, 정적 전력은 파워 게이팅으로 최소화하며, 시스템 수준에서는 에너지 비례 컴퓨팅으로 전체 데이터센터 효율을 극대화한다.

| 개념 | 관계 |

| DVFS | 동적 전력을 V², f 조정으로 줄이는 기법 |

| 전력 벽 (Power Wall) | 동적 전력 증가로 클럭 상승이 한계에 달한 현상 |
| FLOPS/W | AI/HPC 시대 전력 효율의 핵심 지표 |
| PUE | 데이터센터 전체 전력 효율 지표 |

## 👶 어린이를 위한 3줄 비유 설명
1. 전력 소모는 두 가지 — 일할 때 쓰는 전기(동적)와 아무것도 안 해도 새는 전기(정적)가 있어요.
2. 전압을 조금만 낮춰도 쓰는 전기가 많이 줄어요 — 왜냐면 전압은 제곱으로 영향을 주거든요!
3. 현대 컴퓨터는 더 빠른 대신 더 많은 전기를 쓰는 게 아니라, 같은 전기로 더 많은 일을 하는 쪽으로 발전하고 있어요.
