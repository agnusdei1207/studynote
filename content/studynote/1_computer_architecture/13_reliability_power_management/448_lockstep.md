# 448. 락스텝 아키텍처 (Lockstep Architecture)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 CPU가 동일한 명령을 동시에 실행하여 오류 감지
> 2. **가치**: 실시간 오류 탐지, 무정단 복구, 최고 신뢰성
> 3. **융합**: 이중화, 비교기, TMR, 미션 크리티컬

---

## Ⅰ. 개요

### 개념 정의

락스텝 아키텍처(Lockstep Architecture)는 **두 개 이상의 CPU가 완전히 동일한 명령을 동시에 실행하고, 결과를 비교하여 오류를 즉시 감지하는 고신뢰성 아키텍처**다.

### 💡 비유: 듀엣 연주

락스텝 아키텍처는 **듀엣 연주**와 같다. 두 연주자가 있다. (CPU 쌍) 같은 악보. (동일 코드) 동시 연주. (락스텝) 소리가 다르다. (오류 감지) 즉시 멈춤. (복구) 완벽한 연주. 락스텝이 똑같다. 동기 실행, 결과 비교, 즉시 오류 감지.

### 락스텝 아키텍처 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                락스텝 아키텍처 개요                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락스텝 구조】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                      입력                                                     │ │
│  │                        ↓                                                      │ │
│  │              ┌─────────────────────┐                                         │ │
│  │              │    동기화 로직      │                                         │ │
│  │              └─────────────────────┘                                         │ │
│  │                    ↓         ↓                                                │ │
│  │           ┌──────────┐   ┌──────────┐                                        │ │
│  │           │   CPU 0  │   │   CPU 1  │                                        │ │
│  │           │ (Primary)│   │(Secondary)│                                       │ │
│  │           └──────────┘   └──────────┘                                        │ │
│  │                    ↓         ↓                                                │ │
│  │              ┌─────────────────────┐                                         │ │
│  │              │     비교기 (Comparator)   │                                    │ │
│  │              └─────────────────────┘                                         │ │
│  │                    ↓                                                          │ │
│  │              ┌─────────────────────┐                                         │ │
│  │              │   일치: 출력         │                                         │ │
│  │              │   불일치: 오류 처리  │                                         │ │
│  │              └─────────────────────┘                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락스텝 모드】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  모드              설명                    활용                   │ │
│  │  ────              ────                    ────                   │ │
│  │  Duplex            2 CPU 락스텝            일반 고가용성          │ │
│  │  Triplex (TMR)     3 CPU 다수결            최고 신뢰성            │ │
│  │  Quad              4 CPU 락스텝            우주/국방              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락스텝 vs 이중화】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  특성           락스텝                이중화 (Active-Passive)    │ │
│  │  ────           ────────            ────────────────────        │ │
│  │  실행           동시 실행            하나만 실행                  │ │
│  │  오류 감지      즉시 (클럭 단위)      페일오버 시                  │ │
│  │  복구 시간      0 (무중단)           초~분 단위                   │ │
│  │  자원 활용      50%                  50-100%                     │ │
│  │  복잡성         높음                  중간                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 락스텝 아키텍처 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                락스텝 아키텍처 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TMR (Triple Modular Redundancy)】                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                      입력                                                     │ │
│  │                  ┌────┴────┬────────┐                                        │ │
│  │                  ↓         ↓        ↓                                         │ │
│  │            ┌─────────┐ ┌─────────┐ ┌─────────┐                              │ │
│  │            │  CPU 0  │ │  CPU 1  │ │  CPU 2  │                              │ │
│  │            └─────────┘ └─────────┘ └─────────┘                              │ │
│  │                  ↓         ↓        ↓                                         │ │
│  │            ┌───────────────────────────────┐                                 │ │
│  │            │   다수결 투표 (Voter)          │                                 │ │
│  │            │   2/3 이상 일치 → 출력         │                                 │ │
│  │            │   1개 불일치 → 해당 CPU 격리   │                                 │ │
│  │            └───────────────────────────────┘                                 │ │
│  │                          ↓                                                    │ │
│  │                        출력                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【오류 감지 및 처리】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 비교 시점:                                                            │ │
│  │     • 매 클럭 사이클                                                      │ │
│  │     • 메모리 쓰기                                                         │ │
│  │     • I/O 출력                                                           │ │
│  │                                                               │ │
│  │  2. 오류 종류:                                                            │ │
│  │     • 일시적 오류 (Transient): 단일 비트 플립                              │ │
│  │     • 영구적 오류 (Permanent): CPU 고장                                   │ │
│  │                                                               │ │
│  │  3. 복구 액션:                                                            │ │
│  │     • 다수결로 정상 출력                                                  │ │
│  │     • 오류 CPU 격리                                                       │ │
│  │     • 인터럽트 발생                                                       │ │
│  │     • 로그 기록                                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 즉각적 오류 감지                                                      │ │
│  │  • 무중단 복구                                                          │ │
│  │  • 최고 수준 신뢰성                                                      │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 높은 하드웨어 비용                                                    │ │
│  │  • 전력 소모 증가                                                        │ │
│  │  • 설계 복잡성                                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 락스텝: 동일 명령 동시 실행, 결과 비교
• TMR: 3 CPU 다수결 투표
• Duplex: 2 CPU 락스텝
• 비교기: 결과 일치/불일치 판단
• 오류 감지: 클럭 단위 즉시 감지
• 복구 시간: 0 (무중단)
• 장점: 최고 신뢰성, 무중단
• 단점: 높은 비용, 복잡성
• 활용: 우주, 항공, 금융, 국방
• 신뢰성: 99.9999% 이상
```
