# 컴퓨테이셔널 스토리지 (Computational Storage)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컴퓨테이셔널 스토리지 (Computational Storage)는 데이터가 저장된 물리적 위치 근처에서 연산 기능을 직접 수행하여, 호스트 CPU와 스토리지 간의 대규모 데이터 이동을 획기적으로 줄이는 아키텍처적 패러다임이다.
> 2. **가치**: SNIA (Storage Networking Industry Association) 표준을 기반으로 하며, 시스템 전체의 병목 현상 (I/O Bottleneck)을 해결하고 병렬 처리 능력을 확장함으로써 페타바이트 (Petabyte)급 데이터 분석 환경에서 성능과 에너지 효율을 동시 확보한다.
> 3. **융합**: 고성능 컴퓨팅 (HPC), 엣지 컴퓨팅 (Edge Computing), 그리고 클라우드 네이티브 스토리지와 결합하여 데이터 가공, 보안, 검색 작업을 분산 처리하는 '지능형 데이터 인프라'의 근간이 된다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 컴퓨테이셔널 스토리지 (Computational Storage)는 기존의 수동적인 저장 장치에 연산 자원 (FPGA, ASIC, ARM 등)을 통합하여, 데이터를 읽고 쓰는 과정 중에 혹은 저장된 상태에서 직접 가공 (Filtering, Compression, Encryption, Analytics)을 수행하는 차세대 스토리지 기술이다.

- **💡 비유**: 이것은 "원재료를 공장(CPU)으로 가져와서 손질하는 것" 대신, "원재료 산지(스토리지)에 간이 가공 공장을 세워 손질된 완제품만 보내는 것"과 같습니다.

- **등장 배경**:
    1. **데이터 중력 (Data Gravity)**: 데이터의 크기가 너무 커져서 이를 컴퓨팅 자원으로 이동시키는 비용이 연산 비용을 압도하게 됨.
    2. **컴퓨팅-스토리지 성능 불균형**: CPU의 처리 능력 향상 속도에 비해 스토리지 인터페이스 (PCIe/NVMe) 대역폭의 증가 속도가 낮아 병목 현상 심화.
    3. **실시간 분석 요구**: 5G, 자율주행 등에서 발생하는 방대한 데이터를 즉각 처리해야 할 필요성 증대.

- **📢 섹션 요약 비유**: 도서관에서 책을 빌려 집(CPU)에 가서 읽는 대신, 도서관(스토리지) 직원이 필요한 내용만 복사해서 이메일로 보내주는 서비스와 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### SNIA 컴퓨테이셔널 스토리지 모델

SNIA에서는 컴퓨테이셔널 스토리지를 세 가지 형태 (CSD, CSP, CSA)로 정의한다.

| 분류 | 명칭 | 특징 | 내부 구성 | 비유 |
|:---|:---|:---|:---|:---|
| **CSD** | Computational Storage Drive | 드라이브 내부에 연산 장치 통합 | SSD + FPGA/ASIC | 주방이 달린 트럭 |
| **CSP** | Computational Storage Processor | 스토리지와 별개로 연산만 담당하는 가속기 | FPGA/ASIC 전용 카드 | 이동식 조리 가속기 |
| **CSA** | Computational Storage Array | 다수의 CSD와 제어 로직을 갖춘 시스템 | JBOD + CS Controller | 대형 가공 식품 공장 |

---

### 컴퓨테이셔널 스토리지의 계층적 아키텍처

호스트 OS와 스토리지 가속 계층 간의 상호작용을 ASCII 다이어그램으로 도해한다.

```text
 ┌──────────────────────────────────────────────────────────────┐
 │             컴퓨테이셔널 스토리지 시스템 아키텍처              │
 ├──────────────────────────────────────────────────────────────┤
 │                                                              │
 │   [ Host Application Layer ]                                 │
 │   - API calls (e.g., Scan, Compress, Filter)                 │
 │             │                                                │
 │   [ Compute Interface Layer (SNIA Standard) ]                │
 │   - NVMe over Fabrics / PCIe Driver                          │
 │             │                                                │
 │   [ Computational Storage Service ] <────┐ 연산 요청         │
 │   ┌─────────┴──────────┐                 │                   │
 │   │  Processing Unit   │ (FPGA/ASIC)     │                   │
 │   │  - Stream Processor│ ────────────────┘                   │
 │   │  - HW Accelerator  │                                     │
 │   └─────────┬──────────┘                                     │
 │             │ Internal Bus (Low Latency)                     │
 │   ┌─────────▼──────────┐                                     │
 │   │  Storage Media     │ (NAND Flash / SCM)                  │
 │   │  - Flash Interface │                                     │
 │   └────────────────────┘                                     │
 └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 호스트 애플리케이션이 표준 API를 통해 연산 명령을 내리면, 명령은 NVMe 드라이버를 거쳐 스토리지 내부의 Processing Unit으로 전달된다. Processing Unit은 스토리지 미디어 (NAND)에서 데이터를 호스트 버스를 거치지 않고 내부 버스를 통해 즉시 읽어온다. 데이터는 스트림 방식으로 처리되며, 처리된 결과물(추출된 핵심 데이터 등)만 호스트로 전송되므로 외부 버스 트래픽이 최소화된다.

---

### 심층 동작 원리: 스트림 프로세싱 (Stream Processing)

컴퓨테이셔널 스토리지는 주로 **스트림 프로세싱** 방식을 채택한다.
1. **커맨드 큐잉 (Command Queuing)**: 호스트가 스토리지에 '데이터 가공 명령'을 큐에 삽입.
2. **다이렉트 메모리 액세스 (DMA)**: 스토리지 내부 가속기가 NAND 컨트롤러에 직접 접근하여 로컬 버퍼로 데이터 로드.
3. **파이프라인 연산 (Pipelining)**: FPGA 내부에 구현된 하드웨어 로직이 데이터가 흐르는 동안 (In-flight) 실시간으로 연산 수행.
4. **결과 집계 (Aggregation)**: 가공된 데이터를 모아 호스트 응답 메시지 생성.

- **📢 섹션 요약 비유**: 컨베이어 벨트 위를 지나가는 물건들을 사람이 일일이 꺼내지 않고, 벨트 위에 설치된 로봇 팔이 즉석에서 포장하고 분류하는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 기술 비교: CPU 중심 vs 스토리지 중심 컴퓨팅

| 비교 항목 | CPU 중심 (Traditional) | 스토리지 중심 (Computational) |
|:---|:---|:---|
| **데이터 이동 경로** | 스토리지 → RAM → CPU | 스토리지 내 로컬 처리 |
| **주요 병목** | PCIe / Memory Bus | 내부 연산 장치 성능 |
| **확장성** | CPU 코어 수에 제한됨 | 스토리지 용량과 함께 연산력도 선형 확장 |
| **에너지 효율** | 낮음 (데이터 복사 전력 소모) | 높음 (로컬 연산 최소화) |
| **프로그래밍 모델** | 표준 파일 I/O | 오프로딩 API (SNIA) |

### 과목 융합 관점

- **운영체제 (OS)**: 파일 시스템 레이어에서 특정 파일을 '컴퓨테이셔널 스토리지 대상'으로 지정하는 지능형 스케줄링이 필요함.
- **네트워크 (NW)**: NVMe-oF (NVMe over Fabrics)를 통해 원격지에 있는 컴퓨테이셔널 스토리지 자원을 네트워크를 거쳐 가속기로 활용 가능.
- **데이터베이스 (DB)**: BigQuery나 Snowflake와 같은 데이터 웨어하우스에서 'Push-down' 쿼리를 수행할 때 가장 강력한 시너지를 발휘함.

- **📢 섹션 요약 비유**: 모든 업무를 본사(CPU)에서 승인받아야 하는 중앙집권 구조에서, 각 지점(스토리지)에 결정권과 처리 능력을 부여한 지방분권 구조로의 변화와 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
- **로그 분석 가속**: 초당 수백 GB의 서버 로그 중 특정 패턴을 탐색할 때, 컴퓨테이셔널 스토리지는 99%의 불필요한 데이터를 버리고 1%의 유효 로그만 전송하여 분석 시스템의 처리량을 10배 이상 높임.
- **영상 트랜스코딩**: CCTV 저장 장치 내에서 영상을 직접 압축하거나 객체를 인식하여, 네트워크 망으로 전송되는 트래픽을 최소화함.

### 도입 체크리스트
1. **표준 준수 여부**: SNIA Computational Storage Architecture and Programming Model을 따르는가?
2. **가용성 및 신뢰성**: 연산 장치의 발열이 스토리지 미디어의 수명에 영향을 주지 않는가?
3. **보안성**: 스토리지 내에서 연산 중 발생하는 중간 데이터가 안전하게 보호되는가?

- **📢 섹션 요약 비유**: 엔진 성능을 높이는 대신 타이어에 직접 모터를 달아 힘을 분산시키는 4륜 구동 시스템처럼, 시스템 전체의 밸런스를 고려한 설계가 필요합니다.

| **확장성 (Scalability)** | 노드 추가 시 스토리지와 연산력 동시 확장 | 선형적 확장성 확보 |

### 미래 전망
향후 컴퓨테이셔널 스토리지는 **CXL (Compute Express Link)**과 융합하여 호스트 메모리와의 일관성을 유지하면서도 독자적인 연산을 수행하는 형태로 진화할 것이다. 또한, **AI 전용 NPU**가 스토리지에 통합되어 '자가 관리형 지능형 데이터 스토어'로 발전할 것으로 보인다.

- **📢 섹션 요약 비유**: 단순한 수납장(스토리지)이 스스로 물건을 정리하고 관리하는 로봇 옷장으로 진화하여 삶의 편의를 극대화하는 것과 같습니다.

### 📌 관련 개념 맵
- 스마트 SSD (Smart SSD): 컴퓨테이셔널 스토리지의 대표적인 구현체
- NVMe-oF (NVMe over Fabrics): 네트워크를 통한 스토리지 연산 오프로딩 표준
- FPGA (Field Programmable Gate Array): 스토리지 내 연산을 담당하는 핵심 가소성 칩
- 데이터 중심 컴퓨팅 (Data-Centric Computing): 아키텍처적 지향점
