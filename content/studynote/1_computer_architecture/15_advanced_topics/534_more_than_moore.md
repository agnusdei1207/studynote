# 534. 무어의 법칙 이후 (More than Moore)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 반도체 칩의 트랜지스터 집적도가 2년마다 2배씩 늘어난다는 고전적 '무어의 법칙(Moore's Law)'이 물리적 한계(양자 터널링, 발열)로 인해 수명을 다해가자, 이를 극복하기 위해 등장한 새로운 반도체 아키텍처 및 패키징 패러다임이다.
> 2. **가치**: 미세 공정(Scaling)에만 의존하던 성능 향상 방식을 버리고, 서로 다른 기능(아날로그, RF, MEMS, 이기종 코어)을 가진 칩들을 3차원 공간이나 하나의 패키지 안에 집적하여 시스템 전체의 부가가치를 창출한다.
> 3. **융합**: 칩렛(Chiplet), 2.5D/3D 패키징(TSV), 시스템 온 패키지(SiP) 기술과 직접적으로 결합하며, 오늘날 AI 가속기(H100 등)와 고성능 프로세서가 성능을 확장하는 유일한 생존 전략이 되었다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 'More than Moore(무어의 법칙 너머)'는 단순히 트랜지스터 크기를 줄이는 기하학적 미세화(More Moore) 노선을 보완/대체하는 개념이다. 센서, 전력 소자, 통신 칩 등 미세화가 불필요하거나 힘든 이기종 소자들을 하나의 시스템 패키지로 통합하여 성능, 전력, 비용의 효율을 끌어올리는 기술적 방향성을 뜻한다.
- **💡 비유**: 도시가 발전할 때, 예전에는 무조건 좁은 땅에 아파트를 빽빽하게 짓는 것(무어의 법칙)에만 집착했다. 하지만 땅이 좁아 더 이상 건물을 올리기 힘들어지자, 이제는 건물과 건물 사이에 고속철도를 뚫고, 주거 구역 옆에 공장과 쇼핑몰을 유기적으로 배치하여 도시 전체의 가치를 높이는 스마트 시티 모델(More than Moore)로 전략을 바꾼 것이다.
- **등장 배경 및 발전 과정**:
  1. **무어의 법칙 한계 봉착**: 10nm 이하 공정으로 내려가며 트랜지스터 게이트가 원자 몇 개 크기로 얇아졌다. 전자가 벽을 뚫고 지나가는 양자 터널링(누설 전류)과 감당할 수 없는 발열(Power Wall)이 발생했다.
  2. **경제성의 파괴 (More Moore의 위기)**: 미세 공정을 위한 EUV(극자외선) 장비는 수천억 원에 달하고, 칩의 크기(Die Size)가 커지면서 수율(Yield)이 급격히 떨어져 트랜지스터당 생산 단가가 오히려 비싸지는 기현상이 나타났다.
  3. **이기종 통합으로의 선회**: ITRS(국제 반도체 기술 로드맵)는 2010년대 중반부터 미세화에만 집착하는 전략을 버리고, 서로 다른 공정으로 만든 값싼 파편 조각(Chiplet)들을 레고 블록처럼 이어 붙이는 패키징 기술을 미래 전략으로 선포했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 (More than Moore의 핵심 축)

| 요소명 | 역할 | 특징 | 비유 |
|:---|:---|:---|:---|
| **기하학적 미세화 (More Moore)** | 무어의 법칙의 연장선 | FinFET, GAA(Gate-All-Around) 등 소자 구조를 바꿔가며 억지로 미세화를 이어나감 | 좁은 방에 침대 욱여넣기 |
| **이기종 통합 (Heterogeneous Integration)** | 성격이 다른 칩들을 하나로 묶음 | 로직 코어(3nm), 아날로그/RF(14nm), 메모리(HBM)를 가장 효율적인 공정으로 따로 만들어 결합 | 레고 블록 조립 |
| **어드밴스드 패키징 (2.5D/3D)** | 조각난 칩들을 초고속으로 연결 | TSV(실리콘 관통 전극), 실리콘 인터포저(Interposer)를 사용해 칩 간의 대역폭 한계(Memory Wall) 돌파 | 블록 사이를 연결하는 초고속 엘리베이터 |
| **도메인 특화 아키텍처 (DSA)** | 특정 연산에만 극도로 최적화된 칩 | 범용 CPU의 미세화 대신, AI 연산용 NPU나 암호화 전용 칩(ASIC)을 별도로 추가하여 성능 뻥튀기 | 만능 칼 대신 맞춤형 수술칼 도입 |

### 반도체 스케일링 패러다임 전환 (More Moore vs More than Moore)

전통적인 SoC(System on Chip) 방식에서 SiP(System in Package) 방식으로의 이동이 핵심이다.

```text
  ┌─────────────────────────────────────────────────────────────┐
  │         전통적 무어의 법칙(SoC) vs More than Moore (SiP) 비교         │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ [1] More Moore (System on Chip, SoC) - "모든 걸 작게 깎아서 한 판에!"│
  │                                                             │
  │   ┌───────────────────────────────────────────────┐         │
  │   │ [ 3nm 최첨단 단일 다이(Die) - 엄청 비싸고 수율 낮음 ]│         │
  │   │   ┌────────┐   ┌────────┐   ┌─────────┐       │         │
  │   │   │ CPU 코어│   │ GPU 코어│   │ I/O 및 RF│       │         │
  │   │   └────────┘   └────────┘   └─────────┘       │         │
  │   │   * 문제: I/O나 아날로그 회로는 3nm로 줄여봤자 작아       │         │
  │   │           지지도 않는데 굳이 비싼 공정을 써야 함.          │         │
  │   └───────────────────────────────────────────────┘         │
  │                                                             │
  │ ─────────────────────────────────────────────────────────── │
  │                                                             │
  │ [2] More than Moore (System in Package, SiP / Chiplet)      │
  │                                                             │
  │   ┌───────────────────────────────────────────────┐         │
  │   │ [ 패키지 기판 (혹은 실리콘 인터포저) ]                   │         │
  │   │   ┌────────┐   ┌────────┐   ┌─────────┐       │         │
  │   │   │ CPU 코어│   │ HBM(3D)│   │ I/O 및 RF│       │         │
  │   │   │ (3nm)  │   │ (DRAM) │   │ (14nm)  │       │         │
  │   │   └────┬───┘   └───┬────┘   └────┬────┘       │         │
  │   │ ◀──────┴───────────┴──────────────┴───────▶       │
  │   │        초고속 연결 통로 (UCIe, TSV 등)                 │         │
  │   └───────────────────────────────────────────────┘         │
  │     * 장점: 최첨단 3nm는 CPU 코어에만 아껴 쓰고, I/O는 값싼   │
  │             14nm 공정으로 찍어내서 패키지 안에서 이어붙임.      │
  │             경제성과 성능 한계를 동시에 돌파!                   │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 상단의 레거시 **SoC(System on Chip)**는 무어의 법칙을 맹신하던 방식이다. 스마트폰 칩 하나에 CPU, 통신 모뎀, 캐시 메모리를 모조리 집어넣고 통째로 3nm 공정에서 구워낸다. 칩이 너무 커지면 먼지 하나만 떨어져도 전체 칩을 버려야 하므로 생산 단가가 천문학적으로 치솟는다.
하단의 **More than Moore (SiP)** 모델은 이를 잘게 쪼갠다. 성능이 가장 중요한 CPU만 비싼 3nm 공정에서 아주 작게 만들고, 크기를 줄이기 힘든 통신 칩(아날로그)은 저렴한 구형 14nm 공정에서 만든다. 메모리는 아예 수직으로 쌓아 올린 HBM을 쓴다. 이렇게 각자 가장 가성비 좋은 공정에서 만들어진 레고 조각(Chiplet)들을 커다란 기판(Interposer) 위에 올려 초고속 도로(Die-to-Die Interconnect)로 묶어버린다. 제조사는 싼값에 수율을 극대화하면서도, 겉보기에는 마치 거대한 하나의 칩처럼 동작하는 괴물 같은 성능을 얻어낸다.

---

### 심층 극복 원리: TSV(Through Silicon Via)와 열 관리(Thermal)
More than Moore 아키텍처의 물리적 완성은 2.5D/3D 패키징이다. 칩들을 옆으로 나열하는 것을 넘어, 구멍을 뚫고 수직으로 쌓아 올리기 위해 **TSV(실리콘 관통 전극)** 기술이 동원된다. 구리 기둥 수만 개를 칩 사이에 뚫어 데이터를 빛의 속도로 위아래로 나른다.
다만, 이 과정에서 칩들이 이불을 덮은 것처럼 밀집되면서 엄청난 열이 갇히는 **열섬(Thermal Hotspot)** 문제가 폭발한다. 이를 해결하기 위해 방열이 잘 되는 더미 실리콘을 끼워 넣거나, 칩 사이로 냉각수를 흘려보내는 3차원 동적 써멀 관리(3D DTM)가 패키징 기술의 핵심 척도가 되었다.

## Ⅲ. 융합 비교 및 다각도 분석

### 기술 진화 방향 (More Moore vs More than Moore)

| 구분 | More Moore (무어의 법칙) | More than Moore (무어의 법칙 이후) |
|:---|:---|:---|
| **핵심 목표** | 트랜지스터 크기 축소 (Scaling Down) | 기능적 다양화 및 통합 (Functional Diversification) |
| **추구하는 가치** | 단일 코어의 연산 속도 극대화, 원가 절감 | 저전력, 이기종 융합, 폼팩터 소형화 |
| **주요 기술 요소** | EUV 노광 장비, FinFET, GAA 트랜지스터 | **Chiplet, 3D 패키징, TSV, SiP, NPU** |
| **주요 대상 부품** | 디지털 로직 연산기 (CPU, GPU 등) | 아날로그, RF, MEMS, 광반도체(Photonics), 센서 |
| **한계점** | 양자 터널링, 엄청난 누설 전류, 막대한 설비 투자 | 조각난 칩 간의 통신 병목 및 복잡한 열(Thermal) 제어 |

### 과목 융합 관점
- **네트워크 (광 인터커넥트, Silicon Photonics)**: More than Moore의 궁극적인 지향점은 빛이다. 조각난 칩들을 구리선으로 연결하면 저항 때문에 열이 나고 전기를 먹는다. 이를 돌파하기 위해 아예 칩 내부 패키징 안에 레이저를 쏘는 광자 반도체(Silicon Photonics)를 박아 넣어, 빛의 속도로 칩과 칩 사이의 데이터를 전송하는 아키텍처(CPO, Co-Packaged Optics)가 데이터센터 스위치의 표준으로 연구되고 있다.
- **소프트웨어 공학 (소프트웨어 정의 하드웨어)**: 여러 개의 칩렛이 묶인 SiP 환경에서는 소프트웨어가 이 파편화된 하드웨어를 어떻게 인식할지가 중요하다. LLVM 같은 컴파일러는 이제 코드를 짤 때 "이 행렬 곱은 저쪽 3D 패키징 된 텐서 코어로 보내고, 이 제어문은 3nm 메인 코어에서 처리하라"고 이기종 하드웨어를 인지하여 스케줄링(Heterogeneous Compilation)해야 하는 무거운 과제를 떠안았다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — AMD EPYC 서버 프로세서의 칩렛 아키텍처 성공기**: 서버 CPU 시장을 인텔이 독점하던 시절, AMD가 전세를 뒤집은 핵심 무기가 바로 More than Moore 사상의 '칩렛(Chiplet)' 도입이었다.
   - **아키텍처 적용**: 인텔은 28코어짜리 거대한 괴물 칩(Monolithic)을 하나로 통째로 굽다가 수율이 안 나와 단가가 폭등했다. AMD는 8코어짜리 작은 칩렛 여러 개를 7nm로 싸게 찍어내고, 메모리를 컨트롤하는 I/O 칩은 구형 14nm로 찍어낸 뒤, 이를 패키징 위에서 이어 붙여 64코어 CPU를 만들어냈다. 수율은 100%에 육박했고, 인텔 대비 절반의 가격으로 압도적인 코어 수를 제공하며 데이터센터 시장을 장악했다.
2. **시나리오 — 애플 M 시리즈 칩의 통합 메모리 아키텍처 (UMA)**: 노트북 CPU와 RAM을 물리적으로 멀리 띄워놓는 기존 x86의 방식(폰 노이만 병목)을 타파.
   - **설계 구조 (System in Package)**: 애플은 메인보드에 램을 꽂는 슬롯을 없애버렸다. 대신 M1/M2 프로세서 패키지 바로 옆에 LPDDR 메모리 칩을 초근접(SiP)으로 붙여버렸다. 물리적 거리가 몇 밀리미터 단위로 줄어들면서 메모리 대역폭(Bandwidth)이 400GB/s를 돌파했다. 전기를 거의 안 먹으면서도 외장 GPU를 압도하는 비디오 편집 성능을 내는 혁신을 이루어냈다.

### 도입 체크리스트
- **칩렛 간 통신 표준 (UCIe) 확인**: 시스템 아키텍트가 차세대 칩을 설계할 때, "내가 설계한 AI 조각 칩(TSMC 생산)"과 "외부에서 사 온 통신 조각 칩(삼성 생산)"이 패키지 안에서 말이 통하려면 규격이 같아야 한다. 이를 위해 최근 인텔, AMD, ARM 등 전 세계 기업들이 대동단결하여 만든 칩렛 간 연결 표준 규격인 **UCIe (Universal Chiplet Interconnect Express)** 호환성을 반드시 지원하도록 설계해야 벤더 락인을 피할 수 있다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 무어의 법칙 집착 (Monolithic 칩) | More than Moore 도입 (Chiplet / SiP) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (수율 및 생산 원가)**| Die Size가 커질수록 기하급수적 불량률 증가 | 작게 쪼개어 조합하므로 수율 90% 이상 방어 | 고성능 서버 칩 **제조 단가 50% 이상 절감** |
| **정량 (I/O 대역폭)** | PCB 기판을 거치는 병목 현상 발생 (GB/s) | 실리콘 인터포저/TSV를 통한 직결 통신 | 메모리 대역폭 **TB/s 단위의 극한 성능 달성** |
| **정성 (출시 기간, TTM)** | 새로운 공정이 나올 때마다 모든 회로 재설계 | 이미 검증된 구형 I/O 칩렛을 그대로 재활용 | 제품 개발 사이클(Time to Market) 대폭 단축 |

### 미래 전망
- **시스템 온 패키지(SiP)의 궁극적 진화**: 무어의 법칙이 완전히 사망 선고를 받는 순간, 칩 설계의 패권은 '파운드리(공정)'에서 '패키징(조립)'으로 넘어간다. 미래에는 CPU, GPU, 메모리뿐만 아니라 빛으로 통신하는 광 소자(Photonics)와 멤스(MEMS) 센서까지 모두 하나의 3차원 큐브 모양 패키지에 통합되어, 그 패키지 하나가 완벽한 컴퓨터 서버 1대 역할을 하는 세상이 열릴 것이다.
- **웨이퍼 스케일 엔진 (Wafer Scale Engine)**: 칩을 작게 자르지 않고, 아예 30cm짜리 거대한 원형 웨이퍼 하나를 통째로 1개의 초대형 AI 가속기(예: Cerebras WSE)로 사용하는 극단적인 아키텍처도 등장하고 있다. 수백만 개의 코어가 웨이퍼 내부에서 빛의 속도로 데이터를 주고받으며 거대 언어 모델(LLM)을 단숨에 학습시킨다.

### 참고 표준
- **ITRS (International Technology Roadmap for Semiconductors)**: 전 세계 반도체 전문가들이 모여 향후 15년의 기술 방향성을 제시하던 로드맵. 2016년에 "더 이상 크기 축소가 유일한 답이 아니다"라며 해산하고, 'More than Moore'를 정식으로 포함한 **IRDS (다양한 시스템 통합 로드맵)**로 개편되며 기술사의 거대한 변곡점을 찍었다.

## 👶 어린이를 위한 3줄 비유 설명
1. 옛날에는 컴퓨터 성능을 높이려면 장난감 블록을 먼지보다 작게 깎아서(무어의 법칙) 좁은 상자에 억지로 욱여넣는 기술에만 목을 맸어요.
2. 하지만 블록이 너무 작아지니 열도 펄펄 나고 자꾸 부서졌어요. 그래서 이제는 무리하게 작게 만드는 걸 포기했어요. (**More than Moore**)
3. 대신 각자 역할이 다른 여러 모양의 레고 블록들(CPU 블록, 메모리 블록)을 따로따로 만든 다음에, 하나로 멋지게 딱풀로 붙여서(패키징) 훨씬 빠르고 싼 로봇을 만드는 영리한 방법을 쓰기 시작했답니다!
