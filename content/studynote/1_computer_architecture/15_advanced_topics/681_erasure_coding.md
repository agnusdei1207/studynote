# Erasure Coding (삭제 코딩) HW 연산

## 🎯 핵심 인사이트

Erasure Coding(EC)은 **데이터를 여러 조각으로 분할하여 일부가 손실되어도 복구 가능하게 만드는 기술**이다. RAID-6, 분산 스토리지에서 널리 사용되며, XOR/Reed-Solomon 연산의 하드웨어 가속이 성능 핵심이다.

---

## Ⅰ. Erasure Coding 기본 개념

### 1-1. 정의와 원리

```
┌─────────────────────────────────────────────────────────────────────┐
│               Erasure Coding (삭제 코딩)                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  "k개 데이터 조각 + m개 패리티 조각 = n개 중 k개면 복구 가능"       │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                         EC Encoder                          │    │
│  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                           │    │
│  │  │ D1  │ │ D2  │ │ D3  │ │ D4  │   Data (k=4)              │    │
│  │  └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘                           │    │
│  │     │       │       │       │                               │    │
│  │     └───────┴───┬───┴───────┘                               │    │
│  │                 ▼                                           │    │
│  │         ┌───────────────────┐                               │    │
│  │         │ EC Encoding Matrix│                               │    │
│  │         │ (XOR or RS)       │                               │    │
│  │         └─────────┬─────────┘                               │    │
│  │     ┌─────────────┼─────────────┐                           │    │
│  │     ▼             ▼             ▼                           │    │
│  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐          │    │
│  │  │ D1  │ │ D2  │ │ D3  │ │ D4  │ │ P1  │ │ P2  │          │    │
│  │  │     │ │     │ │     │ │     │ │     │ │     │          │    │
│  │  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘ └─────┘          │    │
│  │       Data Chunks        Parity Chunks                      │    │
│  │       (k=4)              (m=2)                             │    │
│  │                                                             │    │
│  │  Total n = k + m = 6 chunks                                │    │
│  │  어느 2개(m)가 손실되어도 4개(k)면 복구 가능!              │    │
│  │                                                             │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  표기법: EC(k, m) 또는 EC(k+m, k)                                  │
│  예: EC(4, 2) = 6개 중 최대 2개 손실 허용                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1-2. RAID vs Erasure Coding

```
┌─────────────────────────────────────────────────────────────────────┐
│                 RAID vs Erasure Coding 비교                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌──────────────┬────────────────────────────────────────────────┐ │
│  │   방식       │                 설명                           │ │
│  ├──────────────┼────────────────────────────────────────────────┤ │
│  │ RAID 1       │ Mirroring - 50% 저장 효율, 1디스크 장애 허용  │ │
│  │ (Mirroring)  │ D ──▶ [D] [D]                                 │ │
│  ├──────────────┼────────────────────────────────────────────────┤ │
│  │ RAID 5       │ 1개 패리티, n-1/n 저장 효율, 1디스크 장애     │ │
│  │ (Parity)     │ D1 D2 D3 ──▶ [D1] [D2] [D3] [P]              │ │
│  ├──────────────┼────────────────────────────────────────────────┤ │
│  │ RAID 6       │ 2개 패리티, n-2/n 저장 효율, 2디스크 장애     │ │
│  │ (DP)         │ D1 D2 D3 ──▶ [D1] [D2] [D3] [P] [Q]          │ │
│  ├──────────────┼────────────────────────────────────────────────┤ │
│  │ EC(k,m)      │ m개 패리티, k/(k+m) 저장 효율, m디스크 장애   │ │
│  │ (General)    │ k 데이터 ──▶ k+m 조각                        │ │
│  └──────────────┴────────────────────────────────────────────────┘ │
│                                                                     │
│  저장 효율 비교 (예: 10개 디스크, 2개 장애 허용):                  │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  RAID 1:  5/10 = 50%                                        │   │
│  │  RAID 6:  8/10 = 80%                                        │   │
│  │  EC(8,2): 8/10 = 80% (RAID 6과 동일)                        │   │
│  │  EC(10,2): 10/12 = 83% (12개 디스크)                        │   │
│  │  EC(12,4): 12/16 = 75% (4개 장애 허용)                      │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **📢 섹션 요약 비유**: Erasure Coding은 퍼즐 조각 나누기와 같다. 원래 그림(D)을 6조각으로 나누는데, 4조각만 있어도 원래 그림을 완성할 수 있게 만든다. 2조각이 없어져도 복구 가능!

## Ⅱ. XOR vs Reed-Solomon Coding

### 2-1. XOR-based Coding (RAID-5, RAID-6)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    XOR-based Erasure Coding                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  XOR 성질: A ⊕ B ⊕ C = P  →  A ⊕ B ⊕ P = C (복구)                 │
│                                                                     │
│  RAID-5 (단일 패리티):                                              │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  D1  │  D2  │  D3  │  P   │                                 │   │
│  │──────│──────│──────│──────│                                 │   │
│  │ 101  │ 110  │ 011  │ 000  │  P = D1 ⊕ D2 ⊕ D3              │   │
│  │      │      │      │      │  (101 ⊕ 110 ⊕ 011 = 000)       │   │
│  │                                                             │   │
│  │  D3 손실 시: D3 = D1 ⊕ D2 ⊕ P = 101 ⊕ 110 ⊕ 000 = 011 ✅  │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  RAID-6 (이중 패리티 - P + Q):                                     │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  D1  │  D2  │  D3  │  P   │  Q   │                          │   │
│  │──────│──────│──────│──────│──────│                          │   │
│  │      │      │      │ XOR  │ Galois│                         │   │
│  │      │      │      │      │ Field │                         │   │
│  │                                                             │   │
│  │  P = D1 ⊕ D2 ⊕ D3         (XOR)                             │   │
│  │  Q = gf(1)·D1 ⊕ gf(2)·D2 ⊕ gf(3)·D3  (Galois Field)         │   │
│  │                                                             │   │
│  │  D2, D3 손실 시: 연립방정식으로 D2, D3 복구                 │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  특징:                                                              │
│  • XOR만으로 m=1 (RAID-5) 가능                                     │
│  • m≥2면 Galois Field 연산 필요                                    │
│  • CPU에서 구현 가능 (SIMD 최적화)                                 │
│  • 일반적으로 m≤2에만 효율적                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2-2. Reed-Solomon Coding

```
┌─────────────────────────────────────────────────────────────────────┐
│                  Reed-Solomon Coding                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  "임의의 m개 장애까지 복구 가능 - Vandermonde/Cauchy 행렬"          │
│                                                                     │
│  수학적 기반:                                                       │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  Encoding: [P1 P2] = [D1 D2 D3] × Encoding Matrix            │   │
│  │                                                             │   │
│  │  ┌────────┐   ┌─────────────────┐   ┌────────┐              │   │
│  │  │ D1 D2  │ × │ 1  1  1  1      │ = │ P1     │              │   │
│  │  │ D3 D4  │   │ 1  2  4  8      │   │ P2     │              │   │
│  │  └────────┘   └─────────────────┘   └────────┘              │   │
│  │               Vandermonde Matrix (k×n)                       │   │
│  │                                                             │   │
│  │  Decoding: 손실된 조각 제외한 submatrix로 역행렬 계산        │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  장점:                                                              │
│  • m값에 제한 없음 (이론상)                                        │
│  • 분산 스토리지에서 m=3,4,6 등 사용                               │
│  • 지리적 분산에 적합                                              │
│                                                                     │
│  단점:                                                              │
│  • 연산 복잡도 높음 (O(k×m×n))                                     │
│  • HW 가속 필수                                                    │
│  • 작은 파일엔 오버헤드 큼                                         │
│                                                                     │
│  사용 사례:                                                         │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │ • Ceph: EC(4,2), EC(8,3), EC(12,4)                          │   │
│  │ • HDFS: EC(6,3), EC(10,4)                                   │   │
│  │ • AWS S3: EC(12,4)                                          │   │
│  │ • Azure: EC(12,2), LRC(12,2,2)                              │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **📢 섹션 요약 비유**: XOR는 간단한 덧셈 뺄셈, Reed-Solomon은 행렬 연산이다. XOR는 쉽지만 2개까지만 복구 가능하고, RS는 어렵지만 몇 개든 복구 가능하다.

### 4-2. 복구 성능 최적화

```
┌─────────────────────────────────────────────────────────────────────┐
│                 EC Recovery Optimization                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  문제: 단일 OSD 장애 시 모든 데이터 복구 필요 → I/O 폭증           │

│  해결책 1: LRC (Local Reconstruction Codes)                        │
│  ┌──────────────────────────────────────────────────────────────┐   │

│  │                                                             │   │
│  │  기존: D1...D12 + P1...P4 (16 chunks)                       │   │

│  │                                                             │   │
│  │  L1 = D1⊕D2⊕D3⊕D4⊕D5⊕D6 (local)                           │   │
│  │  L2 = D7⊕D8⊕D9⊕D10⊕D11⊕D12 (local)                        │   │
│  │  G1, G2 = global parity                                     │   │
│  │                                                             │   │
│  │  단일 장애 시: L1 또는 L2만으로 복구 (7개 읽기 vs 12개)     │   │
│  │  복구 I/O 50% 감소!                                         │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  해결책 2: Partial Read Optimization                               │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  1MB 읽기 요청 시:                                          │   │
│  │  • 전체 복구: 모든 k개 청크 필요                            │   │
│  │  • Partial: 필요한 청크만 읽어서 부분 복구                  │   │
│  │                                                             │   │
│  │  Azure LRC: 90%+ 읽기가 local parity로 처리                 │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **📢 섹션 요약 비유**: LRC는 도시 구조와 같다. 큰 도시(전체 데이터)를 작은 동네(로컬 그룹)로 나누면, 작은 사고는 동네에서 해결하고, 큰 사고는 전체 도시가 협력한다.

## Ⅴ. 시험 핵심 정리

### 5-1. 암기 포인트

```
┌─────────────────────────────────────────────────────────────────────┐
│                     📝 시험 암기 포인트                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. Erasure Coding 정의                                             │
│     • k개 데이터 + m개 패리티 = n개 조각                           │
│     • n개 중 k개면 복구 가능                                       │
│     • 저장 효율: k/(k+m)                                           │
│                                                                     │
│  2. XOR vs Reed-Solomon                                             │
│     • XOR: m=1~2까지만, 단순, 빠름                                 │
│     • Reed-Solomon: m 무제한, 복잡, HW 가속 필요                   │
│                                                                     │
│  3. 하드웨어 가속                                                   │
│     • SIMD (AVX/AVX-512): 벡터 XOR                                 │
│     • GFNI: Galois Field 연산                                      │
│     • VPCLMULQDQ: Carryless 곱셈                                   │
│                                                                     │
│  4. 실제 시스템                                                     │
│     • Ceph: EC(8,3), EC(12,4)                                      │
│     • HDFS: EC(6,3), EC(10,4)                                      │
│     • AWS S3: EC(12,4)                                             │
│                                                                     │
│  5. 최적화 기법                                                     │
│     • LRC: 로컬 패리티로 복구 I/O 감소                             │
│     • ISA-L: Intel Storage Acceleration Library                    │
│                                                                     │
│  6. RAID 비교                                                       │
│     • RAID 5: 1장애, (n-1)/n 효율                                  │
│     • RAID 6: 2장애, (n-2)/n 효율                                  │
│     • EC: m장애, k/(k+m) 효율                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **📢 섹션 요약 비유**: 시험에서 Erasure Coding이 나오면 "퍼즐 조각 나누기"를 떠올려라. k조각만 있어도 완성 가능! XOR은 쉬운 퍼즐, Reed-Solomon은 어려운 퍼즐이다.
