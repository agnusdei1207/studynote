> **핵심 인사이트**
> 1. 앞 장의 플런더버그(Plundervolt)가 OS(소프트웨어) 상에서 전압을 깎아 내리는 공격이었다면, **볼티지 글리칭(Voltage Glitching)**은 아예 **칩의 전원 선(Vcc)에 장비를 물리적으로 꽂아 전기 충격을 주는 순수 하드웨어 해킹**이다.
> 2. CPU가 "비밀번호가 맞는지(If문)" 검사하는 정확히 그 클럭(나노초)에 전압을 0V로 살짝 내렸다(Glitch) 올리면, CPU는 일시적인 기절 상태에서 엉뚱한 값(True)을 뱉어내며 보안 문을 활짝 열어버린다.
> 3. 소니 플레이스테이션, 닌텐도 스위치, 셋톱박스 등을 불법 개조(커펌)할 때 가장 많이 쓰이는 무식하고도 전통적인 하드웨어 파훼법이다.

<br>

## Ⅰ. 전압의 출렁임과 트랜지스터의 기절

디지털 칩은 $1.2V$면 `1`, $0V$면 `0`으로 인식합니다.
명령어를 실행할 때는 트랜지스터가 1과 0을 스위칭하는 물리적인 시간(Setup Time, Hold Time)이 필요합니다.

**볼티지 글리치**의 원리는 간단합니다. 
* 칩이 1.2V로 아주 정상적으로 명령어를 실행하고 있습니다.
* 명령어 실행 중간에 10나노초($10^{-8}$초)라는 찰나의 순간 동안 전압을 0.5V로 팍! 깎았다가(Glitch) 다시 1.2V로 올립니다.
* **칩의 혼란**: CPU는 꺼지지도 않고 켜지지도 않은 몽롱한 상태에 빠집니다. 그 짧은 순간 스위치가 덜 닫히면서 원래는 `False(0)`가 나와야 할 비교 연산이 `True(1)`로 뒤집혀버리는 엄청난 하드웨어 에러가 발생합니다.

> 📢 **섹션 요약 비유**: 경비원(CPU)이 방문증을 검사하는 순간, 밖에서 전기 충격기(글리치)로 경비원의 목을 0.1초 동안 탁 칩니다. 경비원이 눈이 풀려서 멍해진 사이 "저 통과죠?" 하고 스윽 지나가면, 멍해진 경비원이 무의식적으로 "네..." 하고 문을 열어주는 원리입니다.

## Ⅱ. 볼티지 글리칭의 파괴력 (If문 뛰어넘기)

이 기술은 소프트웨어의 가장 중요한 보안 로직을 물리적으로 지워버립니다.

```c
// 칩 내부에 구워진 펌웨어 코드
if (check_signature(firmware) == TRUE) {
    boot_os();   // 서명이 맞으면 부팅
} else {
    halt();      // 틀리면 무한 정지 (보안!)
}
```

해커가 위조된 펌웨어(커스텀 OS)를 넣습니다. 당연히 서명이 틀렸으니 `FALSE`가 반환되어 부팅이 안 되어야 합니다.
하지만 해커는 오실로스코프로 `if` 문이 실행되는 정확한 타이밍(클럭)을 찾아냅니다. 그리고 그 순간 **볼티지 글리치(전압 깎기)**를 쏩니다.
CPU는 멍해져서 `FALSE`를 `TRUE`로 잘못 읽거나, 아예 `if` 명령어 자체를 건너뛰어 버리고(Skip) 곧바로 `boot_os()`를 실행해 버립니다!

### 글리치 파형 (ASCII)

```text
 전압(V)
 1.2V ────┐                    ┌─────── ◀ 정상 전압
          │                    │
          │                    │
 0.5V     └─(10ns 찰나의 기절)─┘        ◀ Glitch! (이때 If문 파괴됨)
```

> 📢 **섹션 요약 비유**: 시험 감독관(If문)이 채점할 때, 뒤통수를 딱 때려서 기절시킵니다. 감독관이 0.1초 뒤 정신을 차려보니 내 시험지는 이미 '합격' 도장이 찍힌 통 안으로 넘어가 있는 마술입니다.

## Ⅲ. 방어 기술 (Glitch Detector)

수천만 원짜리 장비의 락(Lock)이 몇만 원짜리 전압 조작기(Raspberry Pi 피코 등)에 뚫리는 걸 막기 위해, 최신 보안 칩(스마트카드, 금융 IC 칩, 애플 T2 칩 등)은 방어막을 갖춥니다.

* **전압 감지기 (Voltage Detector / Glitch Sensor)**: 칩 내부에 전압이 1나노초라도 비정상적으로 푹 꺼지는 것을 감지하는 독립적인 하드웨어 센서를 둡니다. 센서가 이상을 감지하면 즉시 칩을 리셋(Reset)시켜버리거나 메모리의 암호 키를 날려버립니다.
* **중복 연산 (Redundancy)**: `if`문을 한 번만 쓰지 않고 `if`문을 3번 연속으로 쓰거나, 락스텝(Lockstep) 듀얼 코어를 이용해 동시에 같은 글리치가 맞지 않으면 통과 안 되게끔 소프트웨어/하드웨어를 꼬아버립니다.
