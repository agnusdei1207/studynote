# 메모리 암호화 (Intel MKTME, AMD SME/SEV)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메인 메모리의 데이터를 하드웨어로 자동 암호화하여 물리적 공격과 DMA 공격으로부터 보호
> 2. **가치**: 메모리 덤프 방지, 콜드 부트 공격 차단, VM 격리, 물리적 보안
> 3. **융합**: 기밀 컴퓨팅, TEE, VM 보안, 하드웨어 보안과 통합된 메모리 보호

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

메모리 암호화(Intel MKTME, AMD SME/SEV)는 메인 메모리의 데이터를 하드웨어로 자동 암호화하여 물리적 공격과 DMA 공격으로부터 보호합니다.

**💡 비유**: 메모리 암호화는 자동으로 잠기는 금고와 같습니다. 데이터를 넣으면 자동으로 잠깁니다.

**📢 섹션 요약 비유**: 메모리 암호화는 자동 금고 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **암호화 엔진** | AES 암호화 | 메모리 컨트롤러 내장 | 자물쇠 |
| **키 관리** | 키 저장 | CPU 내부, fuses | 열쇠 |
| **페이지 테이블** | 암호화 플래그 | C-bit, MKTME 비트 | 표시 |
| **코어 간 캐시** | 평문 공유 | 캐시 일관성 | 공유 공간 |

**Intel vs AMD 비교**

| 기능 | Intel TME/MKTME | AMD SME/SEV/SEV-SNP |
|:---|:---|:---|
| **단일 키** | TME | SME |
| **다중 키** | MKTME | SEV (VM별) |
| **무결성** | 없음 | SEV-SNP |
| **태그** | 없음 | ASID |

**핵심 알고리즘**: CPU 액세스 → 키 선택 → 암호화/복호화 → DRAM

**📢 섹션 요약 비유**: CPU에서 메모리 갈 때 자동으로 암호화해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 메모리 암호화 vs 디스크 암호화**

| 비교 항목 | 메모리 암호화 | 디스크 암호화 |
|:---|:---:|:---:|
| **대상** | DRAM | SSD/HDD |
| **시점** | In-Use | At-Rest |
| **성능 영향** | 낮음 | 중간 |
| **공격 방어** | 물리적/DMA | 도난 |

**📢 섹션 요약 비유**: 디스크는 저장, 메모리는 사용 중 보호!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 클라우드 VM** → AMD SEV로 VM 격리
**시나리오 2: 엔터프라이즈** → Intel MKTME로 민감 데이터 보호
**시나리오 3: 보안 서버** → 물리적 공격 방지

**도입 체크리스트**: CPU 지원, BIOS 설정, OS 커널 지원

**📢 섹션 요약 비유**: BIOS에서 켜야 해요!
