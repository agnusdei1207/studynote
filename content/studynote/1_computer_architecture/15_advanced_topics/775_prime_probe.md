# Prime+Probe 기법

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공격자가 캐시를 채우고(Prime) 피해자 실행 후 캐시 상태를 확인(Probe)하여 접근 패턴을 유추하는 사이드 채널 공격
> 2. **가치**: 공유 메모리 불필요, 크로스 VM 공격 가능, 캐시 기반 보안 이해
> 3. **융합**: Flush+Reload, 캐시 타이밍 공격, 스펙터와 통합된 캐시 사이드 채널

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Prime+Probe 기법은 공격자가 캐시를 채우고(Prime) 피해자 실행 후 캐시 상태를 확인(Probe)하여 접근 패턴을 유추하는 사이드 채널 공격입니다. 공유 메모리가 필요 없습니다.

```

├─────────────────────────────────────────────────────────────────────────┤

│   │   │                  L1 Cache (공격자 채움)               │   │ │

│   │   │           교체됨                  교체됨             │   │ │

│   │   - 빠름(히트): 피해자가 안 건드림                            │ │
│   │   - 느림(미스): 피해자가 교체함 → 접근 패턴 유추              │ │
```

> **해설**: Prime+Probe는 캐시를 미리 채우고 교체 여부를 확인합니다.

**💡 비유**: Prime+Probe는 사물함을 확인하는 것과 같습니다. 내 물건이 사라졌으면 누가 썼습니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Prime** | 캐시 채움 | 모든 세트 접근 | 내 물건 채우기 |
| **피해자 실행** | 캐시 교체 | 일부 라인 교체 | 남이 쓰기 |
| **Probe** | 상태 확인 | 시간 측정 | 내 물건 확인 |
| **유추** | 정보 분석 | 패턴 파악 | 누가 썼는지 알기 |

**Prime+Probe 세부 단계**

| 단계 | 동작 | 목적 |
|:---|:---|:---|
| **1. Prime** | 공격자 데이터로 캐시 채움 | 기준 상태 |
| **2. 대기** | 피해자 실행 대기 | 공격 윈도우 |
| **3. Probe** | 각 세트 접근 시간 측정 | 교체 여부 확인 |
| **4. 분석** | 시간 차이로 정보 유추 | 비밀 유출 |

**핵심 알고리즘**: 캐시 채움 → 피해자 실행 → 시간 측정 → 패턴 분석

**📢 섹션 요약 비유**: 채우고 기다리고 확인하고 분석해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: Prime+Probe vs Flush+Reload vs Evict+Time**

| 비교 항목 | Prime+Probe | Flush+Reload | Evict+Time |
|:---|:---:|:---:|:---:|
| **공유 메모리** | 불필요 | 필요 | 불필요 |
| **정밀도** | 중간 | 높음 | 낮음 |
| **크로스 VM** | 가능 | 어려움 | 가능 |
| **복잡도** | 중간 | 낮음 | 낮음 |

**📢 섹션 요약 비유**: 공유 메모리 없이도 공격할 수 있어요!
