# SRAM PUF (Physical Unclonable Function)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SRAM 셀의 전원 인가 시 초기 상태가 무작위로 결정되는 물리적 특성을 이용한 디바이스 고유 식별 기술
> 2. **가치**: 디바이스 인증, 키 저장 없이 키 생성, 복제 불가 식별자, 하드웨어 보안
> 3. **융합**: TRNG, 도전-응답, 안티 탬퍼와 통합된 물리적 식별

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SRAM PUF(Physical Unclonable Function)는 SRAM 셀의 전원 인가 시 초기 상태가 무작위로 결정되는 물리적 특성을 이용한 디바이스 고유 식별 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SRAM PUF (Physical Unclonable Function)            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SRAM 셀 초기화 패턴                               │ │
│   │                                                              │ │
│   │   전원 ON 후 SRAM 초기 상태:                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Address │ Cell 0 │ Cell 1 │ Cell 2 │ Cell 3 │ ... │   │ │
│   │   ├─────────────────────────────────────────────────────┤   │ │
│   │   │  0x0000  │   1    │   0    │   1    │   1    │ ... │   │ │
│   │   │  0x0001  │   0    │   0    │   1    │   0    │ ... │   │ │
│   │   │  0x0002  │   1    │   1    │   0    │   0    │ ... │   │ │
│   │   │  ...     │  ...   │  ...   │  ...   │  ...   │ ... │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   이 패턴은:                                                  │ │
│   │   - 디바이스마다 고유함 (제조 편차)                           │ │
│   │   - 복제 불가능 (물리적 특성)                                 │ │
│   │   - 재현 가능 (같은 디바이스는 같은 패턴)                     │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PUF 동작 원리                                     │ │
│   │                                                              │ │
│   │   ┌──────────┐                    ┌──────────┐              │ │
│   │   │ Challenge│ ───────────────►   │   PUF    │              │ │
│   │   │ (입력)   │                    │  (SRAM)  │              │ │
│   │   └──────────┘                    └────┬─────┘              │ │
│   │                                        │                     │ │
│   │                                        ▼                     │ │
│   │                                   ┌──────────┐              │ │
│   │                                   │ Response │              │ │
│   │                                   │ (고유값)  │              │ │
│   │                                   └──────────┘              │ │
│   │                                                              │ │
│   │   Challenge = 주소, Response = 해당 주소의 초기화 패턴        │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: SRAM PUF는 제조 편차로 인한 고유한 초기화 패턴을 식별자로 사용합니다.

**💡 비유**: SRAM PUF는 지문과 같습니다. 제조 과정의 무작위 편차가 고유한 패턴을 만듭니다.

**📢 섹션 요약 비유**: SRAM PUF는 전자 지문 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **SRAM 어레이** | PUF 소스 | 전원 인가 시 난수 | 지문 |
| **Challenge** | 입력 주소 | 메모리 주소 | 질문 |
| **Response** | 출력 값 | 초기화 패턴 | 답변 |
| **에러 정정** | 잡음 제거 | ECC/BCH | 보정 |

**PUF 유형**

| PUF 유형 | 소스 | 장점 | 단점 |
|:---|:---|:---|:---|
| **SRAM PUF** | SRAM 초기화 | 구현 쉬움 | 에러율 |
| **Arbiter PUF** | 지연 차이 | 고속 | 머신러닝 공격 |
| **Ring Oscillator** | 주파수 차이 | 구현 쉬움 | 면적 |
| **Butterfly PUF** | 래치 | 낮은 전력 | 복잡 |

**핵심 알고리즘**: Challenge → SRAM 패턴 읽기 → ECC → Response

**📢 섹션 요약 비유**: 에러 정정으로 신뢰성을 높여요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: PUF vs 키 저장 vs TRNG**

| 비교 항목 | PUF | 키 저장 | TRNG |
|:---|:---:|:---:|:---:|
| **키 저장** | 없음 | 있음 | 없음 |
| **재현성** | 있음 | 있음 | 없음 |
| **복제성** | 불가능 | 가능 | 해당없음 |
| **용도** | 인증 | 암호화 | 난수 |

**📢 섹션 요약 비유**: PUF는 키 저장 없이 키를 만들어요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: IoT 인증** → SRAM PUF로 디바이스 식별
**시나리오 2: 키 파생** → PUF + KDF로 암호화 키
**시나리오 3: IP 보호** → PUF로 정품 확인

**도입 체크리스트**: 엔트로피, 에러율, ECC, 온도/전압 안정성

**📢 섹션 요약 비유**: 환경 변화에도 안정적이어야 해요!
