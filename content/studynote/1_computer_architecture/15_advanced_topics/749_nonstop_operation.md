# 무정전 운영 아키텍처 (Non-Stop Operation Architecture)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드웨어/소프트웨어 고장, 유지보수, 재해 상황에서도 서비스 중단 없이 연속 운영하는 시스템 설계
> 2. **가치**: 99.999% 가용성, 비즈니스 연속성, 데이터 무결성, 무중단 서비스
> 3. **융합**: 이중화, 장애 조치(Failover), 고가용성 클러스터, 재해 복구(DR)와 통합된 가용성 설계

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

무정전 운영 아키텍처(Non-Stop Operation Architecture)는 하드웨어/소프트웨어 고장, 유지보수, 재해 상황에서도 서비스 중단 없이 연속 운영하는 시스템 설계입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    무정전 운영 아키텍처 구성                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              이중화 (Redundancy)                                │ │
│   │                                                              │ │
│   │   ┌─────────┐     ┌─────────┐                               │ │
│   │   │ Server A │     │ Server B │   Active-Active              │ │
│   │   │ Active   │     │ Active   │   또는 Active-Standby          │ │
│   │   └────┬────┘     └────┬────┘                               │ │
│   │        │                │                                    │ │
│   │        └────────┬───────┘                                    │ │
│   │                 │                                            │ │
│   │                 ▼                                            │ │
│   │          ┌───────────┐                                       │ │
│   │          │ Load      │                                       │ │
│   │          │ Balancer  │                                       │ │
│   │          └───────────┘                                       │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              무정전 구성 요소                                   │ │
│   │                                                              │ │
│   │   1. 전원: 이중화 PSU + UPS + 발전기                          │ │
│   │   2. 네트워크: 이중화 NIC + 다중 경로                          │ │
│   │   3. 스토리지: RAID + 복제 + 백업                              │ │
│   │   4. 서버: 클러스터 + 장애 조치                                │ │
│   │   5. 데이터센터: 다중 사이트 + DR                               │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 무정전 운영은 모든 구성 요소를 이중화하고 자동 장애 조치를 구현합니다.

**💡 비유**: 무정전 운영은 비행기의 다중 엔진과 같습니다. 하나가 고장 나도 나머지로 계속 납니다.

**📢 섹션 요약 비유**: 무정전 운영은 비행기 다중 엔진 같아요. 하나 고장 나도 날아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Deep)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **이중화** | 예비 시스템 | Active-Standby | 예비 타이어 |
| **장애 조치** | 자동 전환 | Failover | 비상 착륙 |
| **하트비트** | 상태 확인 | 주기적 신호 | 맥박 |
| **쿼럼** | 합의 | 다수결 | 투표 |
| **DR** | 재해 복구 | 원격 복제 | 대피소 |

**고가용성 지표**

| 지표 | 공식 | 목표 |
|:---|:---|:---:|
| **가용성** | Uptime / (Uptime + Downtime) | 99.999% |
| **MTBF** | 평균 고장 간격 | 최대화 |
| **MTTR** | 평균 복구 시간 | 최소화 |

**핵심 알고리즘**: 하트비트 → 장애 감지 → Failover → 서비스 복구

**📢 섹션 요약 비유**: 무정전은 5개의 9(99.999%)를 목표로 해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: Active-Active vs Active-Standby**

| 비교 항목 | Active-Active | Active-Standby |
|:---|:---:|:---:|
| **자원 활용** | 100% | 50% |
| **Failover** | 즉시 | 수 초~수 분 |
| **복잡도** | 높음 | 낮음 |

**📢 섹션 요약 비유**: Active-Active는 더 빠르지만 복잡해요!
