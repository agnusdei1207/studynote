# AWS Nitro Enclaves

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: AWS EC2 인스턴스 내에서 격리된 컴퓨팅 환경을 제공하여 민감 데이터를 안전하게 처리
> 2. **가치**: 하드웨어 기반 격리, 암호화 메모리, 증명(Attestation), KMS 연동
> 3. **융합**: Nitro 시스템, KMS, ACM, 기밀 컴퓨팅과 통합된 클라우드 보안

**💡 비유**: Nitro Enclave는 건물 안에 있는 독립형 금고실과 같습니다. 건물의 다른 부분과 완전히 격리되어 있습니다.

**📢 섹션 요약 비유**: Nitro Enclave는 건물 안 금고실 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Enclave vCPU** | 격리 실행 | 전용 vCPU | 금고실 직원 |
| **Enclave 메모리** | 격리 메모리 | 암호화 RAM | 금고 |
| **CID** | 통신 채널 | vsock | 전용 전화 |
| **Attestation** | 증명 | PCR, 인증서 | 신분증 |

**Nitro Enclave 특징**

| 특징 | 설명 |
|:---|:---|
| **네트워크 격리** | 외부 네트워크 접근 불가 |
| **영구 스토리지 없음** | 재부팅 시 데이터 소멸 |
| **암호화 메모리** | 하드웨어 암호화 |
| **KMS 연동** | Attestation 기반 키 복호화 |
| **Certificate** | ACM 연동 |

**핵심 알고리즘**: Enclave 생성 → 격리 실행 → Attestation → KMS 복호화

**📢 섹션 요약 비유**: 네트워크 없고, 저장 없고, 암호화만 있어요!
