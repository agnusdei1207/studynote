> **핵심 인사이트**
> 1. 낸드 플래시는 셀 하나에 데이터를 1비트만 저장하는 SLC(가장 빠름), 3비트를 넣는 TLC, 4비트를 욱여넣는 QLC(가장 느림, 고용량)로 발전해 왔다.
> 2. QLC SSD는 가격이 싸고 용량이 크지만, 데이터를 쓸 때 미세한 전압 조절을 수십 번 해야 해서 속도가 하드디스크(HDD) 수준으로 처참하게 느려지는 단점이 있다.
> 3. 이를 숨기기 위해 SSD 컨트롤러는 QLC 낸드의 일부 빈 공간을 **논리적으로 'SLC처럼' 취급(셀당 1비트만 대충 기록)**하여 초고속으로 데이터를 받아낸 뒤, 사용자가 컴퓨터를 안 쓸 때 몰래 진짜 QLC 영역으로 압축해서 옮겨 적는 **SLC 캐싱(SLC Caching)** 기법을 사용한다.

<br>

## Ⅰ. 셀(Cell) 당 비트 수와 속도의 반비례

낸드 플래시 메모리는 '전자를 가두는 물통'입니다.
* **SLC (Single Level Cell)**: 물통에 물이 '있다/없다' 딱 2가지만 구분합니다 (1비트). 물을 대충 확 들이붓거나 쏟아버리면 되므로 **엄청나게 빠르고 수명이 깁니다(약 10만 회).**
* **TLC (Triple Level Cell)**: 물통의 수위를 8단계로 미세하게 조절해서 3비트를 구별합니다. 눈금을 맞추느라 물을 조금씩 찔끔찔끔 넣어야 해서 **속도가 느리고 수명이 짧습니다(수천 회).**
* **QLC (Quad Level Cell)**: 수위를 16단계로 나눕니다 (4비트). 속도가 절망적으로 느리고 수명도 수백 회에 불과하지만, 1TB, 2TB 대용량을 헐값에 만들 수 있습니다.

> 📢 **섹션 요약 비유**: SLC는 컵에 물이 있는지 없는지만 봅니다(0.1초 소요). QLC는 컵에 그려진 16개의 미세한 눈금 중 13번째 눈금에 정확히 맞춰 물을 따라야 하므로 손을 바들바들 떨며(10초 소요) 채워 넣어야 합니다.

<br>

## Ⅱ. SLC 캐싱의 환상적인 마법

소비자는 1TB의 QLC SSD를 사면서도 초당 5000MB/s의 복사 속도를 기대합니다.
이 불가능한 물리 법칙을 극복하는 것이 바로 컨트롤러 소프트웨어의 **SLC 캐싱**입니다.

1. **가짜 SLC (Pseudo-SLC) 할당**: SSD 컨트롤러는 1TB QLC 공간 중 비어있는 일부(예: 100GB)를 "지금부터 여기는 16개 눈금 보지 말고, 그냥 대충 물이 있다/없다 2단계(SLC)로만 기록해!"라고 선언합니다.
2. **초고속 쓰기 (Cache Hit)**: 사용자가 10GB짜리 영화를 복사하면, 데이터는 이 '가짜 SLC 영역'으로 직행합니다. SLC 모드로 작동하므로 눈금을 잴 필요 없이 빛의 속도로 복사가 완료됩니다. 사용자는 "와 내 QLC SSD 엄청 빠르네!"라고 착각합니다.
3. **은밀한 이사 (Flush/Migration)**: 복사가 끝나고 사용자가 컴퓨터를 그냥 켜두고 쉴 때(Idle time), 컨트롤러가 몰래 깨어납니다. 아까 가짜 SLC 영역에 대충 써둔 10GB의 데이터를 가져와서, 본래의 QLC 영역에 16단계 눈금을 재가며 정성스럽게(느리게) 다시 옮겨 씁니다.

### 데이터 흐름 (ASCII 다이어그램)

```text
 [ 사용자 데이터 입력 (빠름!) ]
           │
           ▼
 ┌─────────────────────────── SSD 전체 영역 ─────────────────────────────┐
 │ ┌─ SLC 캐시 영역 (Pseudo-SLC) ─┐                                      │
 │ │ 1비트씩 대충 기록 (속도 5GB/s)│                                      │
 │ └───────────┬────────────────┘                                      │
 │             │ (사용자가 쉴 때 백그라운드에서 천천히 이사)                   │
 │             ▼                                                       │
 │ ┌─ 진짜 QLC 저장 영역 ───────┐                                      │
 │ │ 4비트씩 정밀하게 꽉꽉 눌러 담음│ (이곳에 직접 쓰면 속도는 100MB/s 수준임)  │
 │ └────────────────────────┘                                      │
 └───────────────────────────────────────────────────────────────────┘
```

> 📢 **섹션 요약 비유**: 서류를 정리할 때, 일단 임시 보관함(SLC 캐시)에 서류를 무작정 확 집어 던집니다(속도 엄청 빠름). 나중에 퇴근 시간이 다 되어 한가해지면, 임시 보관함에 있던 서류를 꺼내 캐비닛(QLC)에 가나다순으로 빽빽하고 정밀하게 다시 꽂아 넣는(느림) 방식입니다.

<br>

## Ⅲ. SLC 캐싱의 민낯 (용량 고갈의 비극)

이 마법은 '캐시 공간'이 넉넉할 때만 작동합니다.

만약 내가 100GB의 SLC 캐시를 가진 SSD에, 한 번에 **200GB짜리 게임을 다운받는다면** 어떻게 될까요?
* 처음 100GB까지는 초당 5000MB/s로 미친 듯이 받아집니다.
* 100GB를 넘기는 순간, SLC 캐시(임시 보관함)가 꽉 찼습니다!
* 이때부터는 꼼수 없이 남은 100GB의 데이터를 **느려터진 QLC 영역에 직접 눈금을 재가며 써야 합니다.** 
* 갑자기 복사 속도가 5000MB/s에서 하드디스크보다 못한 100MB/s 이하로 곤두박질치는 **'속도 저하(Drop)'** 현상이 발생합니다.

따라서 SSD를 살 때는 단순히 '최대 속도'만 볼 것이 아니라, "이 제품의 SLC 캐시 구간이 몇 GB까지 유지되는가(지속 쓰기 속도)?"를 확인하는 것이 컴퓨터 구조 관점에서 가장 합리적인 소비입니다.
