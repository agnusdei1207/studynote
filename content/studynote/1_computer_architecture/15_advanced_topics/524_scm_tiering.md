# 524. 스토리지 클래스 메모리 (SCM) 계층화

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SCM (Storage Class Memory) 계층화는 DRAM (Dynamic Random Access Memory)의 속도와 NAND Flash (낸드 플래시)의 비휘발성을 동시에 갖춘 차세대 메모리를 기존 메모리-스토리지 계층 사이에 삽입하여 성능 단절을 해소하는 아키텍처 기술이다.
> 2. **가치**: 인메모리 데이터베이스 (In-memory Database)의 용량 한계를 극복하고, 시스템 전원 차단 시 데이터 복구 시간 (RTO)을 초 단위 이하로 단축하며, TB급 고속 데이터 접근을 경제적으로 실현한다.
> 3. **융합**: CXL (Compute Express Link) 기반 메모리 풀링, 영구 메모리 (Persistent Memory) 프로그래밍 모델과 결합하여 데이터 중심 컴퓨팅 (Data-Centric Computing)의 핵심 기반이 된다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: SCM (Storage Class Memory)은 바이트 (Byte) 단위 접근이 가능할 만큼 빠르면서도 전원이 꺼져도 데이터를 유지하는 비휘발성 기억장치다. SCM 계층화는 이를 '고속 스토리지' 또는 '저속 메모리' 계층으로 활용하여, DRAM과 SSD (Solid State Drive) 사이의 거대한 성능 격차(약 1,000배)를 메우는 설계 기법이다.

- **💡 비유**: DRAM이 "머릿속의 단기 암기(매우 빠르나 자고 나면 잊음)"이고 SSD가 "서재의 백과사전(영원히 남으나 가지러 가기 느림)"이라면, SCM은 "책상 위에 펼쳐진 다이어리(손만 뻗으면 바로 읽고, 다음 날 와도 그대로 있음)"와 같습니다.

- **등장 배경**:
  1. **성능 단절 (Performance Gap)**: CPU 성능과 메모리 용량은 급증했으나, 보조기억장치인 SSD/HDD의 지연 시간은 여전히 메모리 대비 매우 높아 시스템 병목의 주원인이 되었다.
  2. **빅데이터 및 실시간 분석 요구**: 테라바이트(TB) 단위의 데이터를 메모리에 올려 처리하려는 요구가 커졌으나, DRAM만으로는 비용과 전력 소모를 감당하기 어려워졌다.
  3. **데이터 비휘발성 보장**: 전원 장애 시 메모리 내 데이터 유실을 방지하고 빠른 복구가 필요한 미션 크리티컬 시스템의 요구가 증대되었다.

- **📢 섹션 요약 비유**: 마치 비좁은 국도와 고속도로 사이를 연결하는 '순환 도시고속도로'를 건설하여, 도시 전체의 교통 흐름을 원활하게 만드는 교통 혁신과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **SCM Media** | 비휘발성 데이터 저장 매체 | 상변화, 자기 저항 등을 이용해 데이터 기록 | PRAM, MRAM, ReRAM | 지워지지 않는 특수 잉크 |
| **Memory Controller** | CPU와 SCM 간 통신 제어 | DDR (Double Data Rate) 버스 또는 CXL 프로토콜 처리 | CXL Controller | 다이어리를 읽는 신경계 |
| **NVDIMM (Non-Volatile DIMM)** | SCM 장착 폼팩터 | 메모리 슬롯(DIMM)에 직접 장착 가능케 함 | DDR4/5 Interface | 규격화된 바인더 속지 |
| **DAX (Direct Access)** | OS 커널 우회 접근 기술 | 페이지 캐시 없이 파일 시스템이 SCM에 직접 접근 | ext4-DAX, XFS | 비서 없이 직접 쓰기 |
| **PMDK (Persistent Memory Development Kit)** | 영구 메모리 개발 라이브러리 | 데이터의 원자적 저장 및 트랜잭션 관리 | Libpmem, Libpmemobj | 일기장 쓰기 가이드 |

### SCM 계층화 아키텍처 및 데이터 흐름

SCM은 메모리 버스에 직접 연결되어 CPU가 Load/Store 명령으로 직접 접근(mmap)하는 구조를 가진다.

```text
  ┌─────────────────────────────────────────────────────────────┐
  │              SCM 도입 전후 메모리 계층 구조 비교                │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ [전통적 구조]                         [SCM 계층화 구조]       │
  │                                                             │
  │    ┌──────┐                          ┌──────┐               │
  │    │ CPU  │                          │ CPU  │               │
  │    └──┬───┘                          └──┬───┘               │
  │    ┌──┴───┐ (Hot)                    ┌──┴───┐ (Hot)         │
  │    │ DRAM │                          │ DRAM │               │
  │    └──┬───┘                          └──┬───┘               │
  │       │   ◀── [성능 단층] ──▶        ┌──┴───┐ (Warm)        │
  │    ┌──┴───┐ (Cold)                   │ SCM  │ ◀── [New!]    │
  │    │ SSD  │                          └──┬───┘               │
  │    └──────┘                          ┌──┴───┐ (Cold)        │
  │                                      │ SSD  │               │
  │                                      └──────┘               │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 기존 아키텍처에서 DRAM과 SSD 사이에는 약 1,000배 이상의 성능 차이가 존재하여, 메모리 부족 시 발생하는 스왑(Swap) 현상이 심각한 성능 저하를 유발했다. SCM 계층화는 이 사이에 지연 시간이 DRAM의 수 배 수준인 SCM(예: Optane PMEM)을 삽입한다. 이를 통해 자주 쓰이는 데이터(Hot)는 DRAM에, 빈도가 낮지만 빠른 접근이 필요한 데이터(Warm)는 SCM에, 드물게 쓰이는 데이터(Cold)는 SSD에 배치하는 '이기종 메모리 티어링'이 완성된다. CPU는 SCM을 I/O 장치가 아닌 주소 공간의 일부로 인식하여 시스템 전체의 유효 메모리 용량을 TB급으로 확장할 수 있다.

---

### 심층 동작 원리: Memory Mode vs App Direct Mode
1. **Memory Mode**: SCM을 일반 RAM처럼 사용하는 모드다. DRAM은 SCM의 캐시 역할을 수행하며, OS는 전체 용량을 휘발성 메모리로 인식한다. 별도의 코드 수정 없이 대용량 메모리를 확보할 수 있다.
2. **App Direct Mode**: SCM의 비휘발성 특성을 직접 활용하는 모드다. 애플리케이션은 DAX (Direct Access)를 통해 SCM 영역에 데이터를 직접 쓰고, 전원이 꺼져도 유지되는 영구 메모리(Persistent Memory)로 관리한다.

- **📢 섹션 요약 비유**: 상황에 따라 거실을 확장하여 넓게 쓰거나(Memory Mode), 아예 붙박이장(App Direct Mode)을 설치하여 물건을 영구 보관하는 맞춤형 공간 인테리어와 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### DRAM vs SCM vs NAND Flash

| 비교 항목 | DRAM | SCM (PMEM) | NAND Flash (SSD) |
|:---|:---|:---|:---|
| **접근 단위** | Byte (바이트) | Byte (바이트) | Block/Page (블록) |
| **지연 시간** | < 100 ns | 100 ~ 500 ns | 10 ~ 100 µs |
| **휘발성** | 휘발성 | **비휘발성** | 비휘발성 |
| **내구성** | 거의 무한 | 높음 | 제한적 (수만 회) |
| **가격/밀도** | 비쌈 / 낮음 | 중간 / 높음 | 저렴 / 매우 높음 |

---

### 과목 융합 관점
- **데이터베이스 (DB)**: SCM을 활용하면 WAL (Write Ahead Logging)을 SSD가 아닌 SCM에 기록하여 쓰기 성능을 비약적으로 향상시키고, 재부팅 시 DB 로딩 과정 없이 즉시 서비스 재개가 가능하다.
- **운영체제 (OS)**: 파일 시스템의 페이지 캐시 (Page Cache) 계층을 제거하고 CPU 가상 주소를 SCM 물리 주소에 직접 매핑하여 커널 오버헤드를 제거한다.

- **📢 섹션 요약 비유**: 육상 선수(DRAM)와 마라톤 선수(SSD) 사이의 간극을 메워주는 중거리 선수(SCM)를 영입하여 팀 전체의 레이스 전략을 최적화한 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **인메모리 DB 비용 최적화**: SAP HANA나 Redis 사용 시, 전체 데이터를 비싼 DRAM에 올리는 대신 SCM을 활용하여 메모리 비용을 30~50% 절감하면서 성능을 유지한다.
2. **가상화 서버 밀도 향상**: VDI (Virtual Desktop Infrastructure) 환경에서 호스트 서버의 메모리 용량을 SCM으로 확장하여, 한 대의 서버에 수용 가능한 가상 머신(VM) 수를 2배 이상 늘린다.
3. **체크포인팅 가속화**: 대규모 HPC 연산 중 갑작스러운 장애에 대비해 상태를 저장하는 체크포인팅 작업을 SCM에 수행하여 연산 중단 시간을 최소화한다.

### 도입 체크리스트
- 사용하는 서버 CPU 및 메인보드가 SCM(예: NVDIMM)을 지원하는가?
- 애플리케이션이 DAX 및 PMDK 라이브러리를 활용하도록 수정 가능한가?
- 하드웨어 벤더(Intel 등)의 제품 로드맵 및 단종 여부(Optane 사례)를 고려했는가?

- **📢 섹션 요약 비유**: 값비싼 호텔 펜트하우스(DRAM)만 고집하기보다, 가성비 좋은 프리미엄 비즈니스 호텔(SCM)을 섞어 숙박 예산을 효율적으로 관리하는 것과 같습니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과
- **TCO (Total Cost of Ownership) 절감**: 동일 용량 메모리 구축 시 DRAM 대비 비용 효율성 증대.
- **가용성 극대화**: 시스템 재부팅 후 데이터 리로딩 시간 제거로 서비스 가동률 (SLA) 향상.
- **성능 일관성**: SSD의 GC (Garbage Collection) 페널티가 없는 SCM 접근으로 Tail Latency 억제.

### 미래 전망
- 인텔 옵테인 사업 철수 이후, SCM 매질 자체의 발전보다는 **CXL (Compute Express Link)** 기반의 메모리 티어링 솔루션으로 기술이 통합되고 있다. 미래에는 CXL 인터페이스를 통해 다양한 매질의 SCM이 공유 메모리 풀링 형태로 발전할 전망이다.

- **📢 섹션 요약 비유**: 특정 브랜드의 부품(Optane) 시대는 저물었지만, 모든 부품을 연결하는 새로운 표준(CXL)을 통해 더 거대하고 똑똑한 공유 창고의 시대가 열릴 것입니다.

---

### 📌 관련 개념 맵
- **CXL (Compute Express Link)**: SCM 확장의 새로운 표준
- **NVMe (Non-Volatile Memory Express)**: SCM SSD의 통신 프로토콜
- **가상 메모리**: SCM 매핑의 근간 원리
- **NVDIMM (Non-Volatile DIMM)**: SCM의 물리적 구현 형태
