# Memory Sharing in Heterogeneous Computing

> 1. **본질**: 이종 컴퓨팅 메모리 공유 (Memory Sharing in Heterogeneous Computing)는 CPU, GPU, DSP, NPU 등 서로 다른 아키텍처를 가진 프로세서들이 하나의 가상 주소 공간을 공유하거나, 하드웨어 일관성(Coherency)을 유지하며 데이터를 직접 주고받는 기술이다.
> 2. **가치**: 프로세서 간의 불필요한 데이터 복사(Copy)와 오버헤드를 제거하여 시스템 전체의 성능과 전력 효율을 극대화하며, 프로그래머에게 통합된 메모리 뷰(Unified Memory View)를 제공하여 개발 생산성을 높인다.
> 3. **융합**: CXL (Compute Express Link), SVM (Shared Virtual Memory), HSA (Heterogeneous System Architecture), 그리고 캐시 일관성 인터커넥트 기술과 융합되어 현대 AI 및 고성능 연산 시스템의 핵심 축을 담당한다.

+++

## Ⅰ. 개요 (Context & Background)

- **개념**: 이종 컴퓨팅 메모리 공유는 "서로 다른 언어를 쓰는 사람들이 한 도서관을 공유하는 것"과 같다. CPU와 GPU가 각자 자기만의 책상을 쓰는 게 아니라, 거대한 공용 서재(메모리)에서 같은 책(데이터)을 보고 동시에 작업할 수 있게 해주는 기술이다.

- **💡 비유**: 이종 컴퓨팅 메모리 공유는 "주방과 홀이 공유하는 냉장고"와 같다. 예전에는 주방장(CPU)이 재료를 준비해서 홀 매니저(GPU)에게 일일이 배달해줘야 했지만(Data Copy), 이제는 주방 한가운데에 공용 냉장고(Shared Memory)를 두고 각자 필요할 때 바로 꺼내 쓰는 것과 같다.

- **등장 배경**: 
  1. **데이터 폭증**: AI 연산 등에서 다루는 데이터가 너무 커져서 CPU와 GPU 사이를 복사하며 옮기는 시간(Latency)이 연산 시간보다 길어짐.
  2. **프로그래밍 복잡도**: 개발자가 메모리 할당, 복사, 해제 과정을 수동으로 관리해야 하는 번거로움과 버그 위험 증가.
  3. **실시간 처리 요구**: 자율주행 등에서 센서 데이터를 CPU가 읽고 GPU가 즉시 분석해야 하는 등 초저지연 연동이 필수적이 됨.

- **📢 섹션 요약 비유**: 프로세서 간의 경계를 허물고 데이터를 한곳에서 관리함으로써 '복사의 늪'에서 벗어나는 메모리 혁신 기술입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 및 공유 메커니즘

이종 프로세서 간 메모리 공유를 위해 하드웨어 인터커넥트와 소프트웨어 런타임이 협력한다.

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **IOMMU / SMMU** | 주소 변환 공유 | GPU 등 장치가 CPU와 동일한 가상 주소를 쓰도록 변환 | SVM (Shared Virtual Memory) | 통합 주소록 |
| **Coherent Interconnect** | 하드웨어 일관성 유지 | CPU와 GPU의 캐시 데이터를 실시간 동기화 | CXL.cache / AMBA CHI | 실시간 무전기 |
| **Unified Memory Manager** | 메모리 페이지 관리 | 요청에 따라 데이터를 물리적으로 이동시키거나 매핑 | CUDA Managed Memory | 지능형 사서 |
| **Hardware Page Fault** | 요구불 페이지 로딩 | GPU가 없는 데이터를 요청하면 CPU 메모리에서 즉시 인출 | Demand Paging | 자동 주문 시스템 |
| **Atomic Operations** | 원자적 동기화 | 여러 장치가 동시에 한 데이터를 수정할 때 충돌 방지 | HW Semaphores | 작업 승인 도장 |

### 이종 컴퓨팅 메모리 공유 아키텍처 (HSA/CXL 기반)

CPU와 가속기가 어떻게 하나의 메모리 공간에서 협업하는지 시각화한다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │            이종 컴퓨팅 통합 메모리 구조 (Unified Memory)             │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   [ CPU Core ]      [ GPU / NPU Accelerator ]                 │
 │   ┌──────────┐      ┌───────────────┐                         │
 │   │ L1/L2    │      │ L1/L2 / Local │                         │
 │   └────┬─────┘      └───────┬───────┘                         │
 │        │                    │                                 │
 │   ┌────▼────────────────────▼────┐   [ Coherency Manager ]    │
 │   │    Coherent Interconnect     │ ◀─── (Keep Caches in Sync) │
 │   └────┬────────────────────┬────┘                            │
 │        │                    │                                 │
 │   ┌────▼──────────┐   ┌─────▼──────────┐                      │
 │   │ System Memory │   │ Accelerator Mem│  (Unified View)      │
 │   │ (DDR/HBM)     │   │ (HBM/VRAM)     │  Addr 0x000 ~ 0xFFF  │
 │   └───────────────┘   └────────────────┘                      │
 │                                                               │
 │   ※ CPU와 GPU가 동일한 포인터(0xABCD)로 데이터에 접근 가능        │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 과거에는 CPU 메모리와 GPU 메모리가 물리적으로나 논리적으로 완전히 분리되어 있었다. 데이터를 보내려면 PCIe 버스를 통해 복사해야 했다. 하지만 이종 메모리 공유 기술이 적용되면, CPU와 GPU는 '가상 주소 공간'을 통째로 공유한다. CPU가 `malloc`으로 할당한 메모리 주소를 GPU에게 그냥 던져주면, GPU는 그 주소를 자기 것처럼 바로 읽고 쓸 수 있다. 이때 'Coherent Interconnect'는 CPU 캐시에 있는 최신 데이터가 GPU에게도 보이도록 하드웨어적으로 보장한다. 데이터가 실제로 어디에 있든(DDR에 있든 HBM에 있든) 상관없이 프로세서들은 하나의 거대한 메모리 풀로 인식하며 작업한다.

+++

### 심층 동작 원리: 데이터 이동의 자동화

1. **Explicit Migration**: 프로그래머가 특정 시점에 데이터를 가속기로 옮기라고 명령. (성능 예측 용이)
2. **Implicit Migration**: GPU가 데이터 접근 시 페이지 폴트(Page Fault)를 일으키면, 런타임이 자동으로 데이터를 전송. (개발 편의성 극대화)
3. **Zero-copy**: 데이터 이동 없이 인터커넥트를 통해 상대방 메모리를 직접 읽기(Peer-to-Peer).

- **📢 섹션 요약 비유**: 물건을 직접 가져오지 않아도 필요할 때마다 원격으로 조종하거나(Zero-copy), 비서가 알아서 물건을 갖다 주는(Implicit) 스마트한 물류 시스템과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 전통적인 분리 메모리 vs 통합 공유 메모리

| 비교 항목 | 분리 메모리 (Discrete) | 통합 공유 메모리 (Unified) |
|:---|:---|:---|
| **데이터 전송** | 수동 복사 (memcpy) | 하드웨어 자동 관리 / 공유 |
| **프로그래밍** | 어려움 (포인터 2개 관리) | 쉬움 (단일 포인터) |
| **오버헤드** | 높음 (PCIe 복사 지연) | 낮음 (포인터 전달/일관성 유지) |
| **메모리 활용** | 중복 저장으로 낭비 | 중복 제거로 효율적 |
| **성능 (Latency)** | 데이터 전송 시 병목 발생 | 전송 없이 즉시 접근 가능 |

- **📢 섹션 요약 비유**: 두 사람이 각자 장부를 복사해서 보는 것(분리)과, 하나의 장부를 가운데 두고 같이 쓰는 것(공유)의 효율성 차이입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 대규모 LLM(Large Language Model) 학습**: 수천억 개의 파라미터를 GPU 메모리에 다 담을 수 없는 경우. 기술사는 **CXL 기반의 Memory Pooling**을 적용하여, CPU 메모리를 GPU의 확장 메모리처럼 쓰도록 설계해 Out-of-Memory 오류를 방지한다.
2. **시나리오 — 실시간 영상 처리 가속**: 카메라 센서 데이터를 CPU가 전처리하고 GPU가 필터를 입히는 경우. 기술사는 **Zero-copy SVM**을 활성화하여, 복사 오버헤드 없이 CPU와 GPU가 동일한 프레임 버퍼를 번갈아 가며 작업하게 하여 FPS를 2배 이상 끌어올린다.

### 도입 체크리스트
- **Interconnect Bandwidth**: CPU-GPU 간 공유 트래픽을 감당할 만큼 대역폭(CXL/NVLink)이 충분한가?
- **Coherency Overhead**: 캐시 일관성 유지 신호(Snoop)가 실제 데이터 전송보다 더 많은 대역폭을 뺏고 있지는 않은가?
- **OS/Driver Support**: 운영체제 커널과 장치 드라이버가 이종 메모리 관리 기능을 완벽히 지원하는가?

- **📢 섹션 요약 비유**: 주방과 홀 사이의 복도가 좁지는 않은지(Bandwidth), 그리고 서로 "이거 써도 돼?"라고 묻는 말(Coherency)이 너무 많아 시끄럽지는 않은지 체크해야 합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 전 (Discrete) | 도입 후 (Unified) | 개선 효과 |
|:---|:---|:---|:---|
| **데이터 전송 지연** | 100ms (기준) | 1ms 이하 | **100배 지연 시간 단축** |
| **개발 생산성** | 코드의 30%가 메모리 관리 | 메모리 관리 코드 거의 없음 | **개발 속도 40% 향상** |
| **시스템 전력 효율** | 복사 시 전력 낭비 심함 | 필요한 만큼만 접근 | **와트당 성능 30% 개선** |

### 미래 전망
향후 이종 메모리 공유는 **'Global Memory Fabric'**으로 진화할 것이다. 단순히 칩 내부가 아니라 랙(Rack) 전체, 혹은 데이터센터 전체가 하나의 메모리 공간으로 묶이는 시대로 가고 있다. CXL 3.0 이상이 보급되면 수천 개의 CPU와 GPU가 거대한 메모리 풀에서 자유롭게 협업하게 될 것이며, 이는 '창고형 컴퓨터(Warehouse-scale Computer)'의 실질적인 구현이 될 전망이다.

- **📢 섹션 요약 비유**: 이제는 내 방 냉장고(Local Mem)를 넘어, 아파트 전체가 공유하는 거대한 냉장고(Global Fabric)에서 신선한 데이터를 마음껏 꺼내 쓰는 시대로 가고 있습니다.

## 📌 관련 개념 맵
- **CXL (Compute Express Link)**: 이종 장치 간 메모리 공유를 위한 차세대 표준 인터페이스.
- **IOMMU (Input-Output Memory Management Unit)**: 장치들이 시스템 주소를 공유하게 해주는 하드웨어 유닛.
- **HBM (High Bandwidth Memory)**: 공유 메모리 시스템의 성능을 뒷받침하는 초고속 메모리 기술.
- **가상 메모리 (Virtual Memory)**: 통합 주소 공간을 제공하는 논리적 기반 기술.
