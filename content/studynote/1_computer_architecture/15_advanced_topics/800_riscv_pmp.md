# RISC-V PMP (Physical Memory Protection)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: RISC-V CPU에서 물리적 메모리 영역별 접근 권한을 제어하는 하드웨어 메모리 보호 기능
> 2. **가치**: 권한 분리, 메모리 격리, 보안 모니터, 펌웨어 보호
> 3. **융합**: ePMP, RISC-V 보안, TEE, 신뢰 실행 환경과 통합된 메모리 보호

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RISC-V PMP(Physical Memory Protection)는 RISC-V CPU에서 물리적 메모리 영역별 접근 권한을 제어하는 하드웨어 메모리 보호 기능입니다.

**💡 비유**: PMP는 건물 출입 카드 시스템과 같습니다. 구역별로 다른 권한을 줍니다.

**📢 섹션 요약 비유**: PMP는 출입 카드 시스템 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **pmpaddr** | 주소 경계 | 물리 주소 | 구역 표시 |
| **pmpcfg** | 권한 설정 | R/W/X/L | 출입 권한 |
| **M-mode** | 설정 권한 | 최고 권한 | 관리자 |
| **S/U-mode** | 제어 대상 | 일반 실행 | 사용자 |

**PMP vs MPU 비교**

| 비교 항목 | RISC-V PMP | ARM MPU |
|:---|:---:|:---:|
| **레벨** | 물리적 | 가상/물리 |
| **레지스터** | 16개(기본) | 8-16개 |
| **모드** | 4가지 | 1가지 |
| **잠금** | L 비트 | 없음 |

**핵심 알고리즘**: M-mode 설정 → 메모리 접근 → PMP 체크 → 허용/거부

**📢 섹션 요약 비유**: M-mode만 설정할 수 있어요!
