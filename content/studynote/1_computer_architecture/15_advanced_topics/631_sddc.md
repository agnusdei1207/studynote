> **핵심 인사이트**
> 1. **SDDC (소프트웨어 정의 데이터센터)**는 특정 장비 하나를 가상화하는 수준을 넘어, **데이터센터라는 거대한 건물 전체의 통제권**을 물리적 쇠붙이(하드웨어)에서 소프트웨어 중앙 통제실로 100% 빼앗아 오는 패러다임이다.
> 2. SDDC를 완성하는 3대 기둥은 **SDC (소프트웨어 정의 컴퓨팅/가상머신)**, **SDS (소프트웨어 정의 스토리지)**, **SDN (소프트웨어 정의 네트워크)**이다.
> 3. 이를 통해 관리자는 드라이버를 들고 서버실을 뛰어다닐 필요 없이, 마우스 클릭 몇 번이나 파이썬 스크립트 코드 한 줄(IaC)로 5분 만에 수백 대의 서버와 방화벽, 전용망이 셋팅된 완벽한 인프라를 찍어낼 수 있다.

<br>

## Ⅰ. 하드웨어 종속성의 시대 (과거의 데이터센터)

과거 데이터센터를 구축하려면 수십 명의 하드웨어 장인들이 필요했습니다.
* "A 서비스 오픈해야 하니, HP 서버 10대 가져와서 나사로 랙에 조이세요."
* "Cisco 라우터 콘솔에 콘솔 케이블을 꽂고 CLI 명령어를 쳐서 방화벽 뚫어주세요."
* "NetApp 스토리지에 LUN 볼륨 깎아서 파이버 채널(FC) 선으로 서버랑 연결하세요."

이 과정은 수동적이었고, 장비 제조사(Vendor)가 다르면 호환도 되지 않는 철저한 파편화 구조였습니다. 서비스를 하나 오픈하려면 인프라 셋팅에만 몇 달이 걸리는 꽉 막힌 공장이었습니다.

> 📢 **섹션 요약 비유**: 무대(서비스) 하나를 올릴 때마다, 조명 감독이 사다리 타고 올라가서 조명 달고, 음향 감독이 바닥에 전선 깔고, 무대 감독이 톱질을 해서 세트를 만드는 매우 느리고 노동집약적인 아날로그 극장입니다.

<br>

## Ⅱ. SDDC의 3대 기둥 (모든 것의 가상화)

**SDDC**는 이 노동집약적 인프라를 '추상화(Abstraction)'합니다. 바닥에는 값싸고 똑같이 생긴 범용 x86 깡통 서버만 수만 대 깔아둡니다. 그 위를 덮는 3개의 소프트웨어 마법이 SDDC를 완성합니다.

### 1. SDC (Software Defined Compute)
* **내용**: CPU와 RAM을 가상화합니다. (하이퍼바이저, KVM, VMware vSphere 등)
* **효과**: 물리 서버가 10대든 100대든 상관없이, 소프트웨어가 이를 뭉쳐서 거대한 하나의 CPU 풀(Pool)로 만듭니다. 사용자는 "코어 4개짜리 컴퓨터 1대 만들어 줘"라고 허공(소프트웨어)에 명령하면 즉시 VM이 튀어나옵니다.

### 2. SDS (Software Defined Storage)
* **내용**: 수만 대의 깡통 서버 안에 박혀있는 하드디스크들을 소프트웨어가 하나로 묶어 거대한 1개의 가상 창고로 만듭니다. (Ceph, vSAN 등)
* **효과**: 비싼 전용 스토리지 박스를 살 필요 없이, 노드를 꽂으면 알아서 전체 용량이 수백 테라바이트씩 늘어납니다. 데이터 3중 복제도 소프트웨어가 다 알아서 분산시켜 줍니다.

### 3. SDN (Software Defined Network)
* **내용**: 케이블을 꽂고 뽑는 물리적 행위 없이, 가상 스위치(OVS)를 통해 IP, 대역폭, 라우팅 규칙을 소프트웨어 클릭만으로 세팅합니다. (NSX, OpenFlow 등)
* **효과**: 서버 1번과 10번을 한 묶음(VLAN)으로 만들고 싶을 때, 랜선을 빼서 같은 스위치에 꽂을 필요 없이 마우스 드래그로 선을 이어주면 논리적으로 같은 망이 됩니다.

### SDDC 구조 (ASCII)

```text
 ┌────────────────────────────────────────────────────────┐
 │  SDDC 제어판 (클라우드 오케스트레이터 - OpenStack 등)  │ (중앙 통제실)
 └────┬──────────────────┬──────────────────┬─────────────┘
      ▼ API 명령         ▼                  ▼
 ┌─────────┐        ┌─────────┐        ┌─────────┐
 │   SDC   │        │   SDS   │        │   SDN   │ (순수 소프트웨어 계층)
 │(가상 CPU)│        │(가상 디스크)│       │(가상 네트워크)│
 └────┬────┘        └────┬────┘        └────┬────┘
 ═════▼══════════════════▼══════════════════▼══════
 [ 바보 깡통 서버 A ] [ 바보 깡통 서버 B ] [ 바보 화이트박스 스위치 ] (물리 계층)
```

> 📢 **섹션 요약 비유**: 극장에 **'최첨단 디지털 홀로그램 무대(SDDC)'**를 설치했습니다. 이제 톱질하고 전선 깔 필요 없이, 태블릿 PC에서 "숲속 배경, 빨간 조명, 새소리 BGM"을 클릭만 하면 홀로그램 시스템이 1초 만에 무대를 세팅해 줍니다.

## Ⅲ. IaC (Infrastructure as Code)의 달성

SDDC의 궁극적인 존재 가치는 **코드형 인프라(IaC)**를 가능하게 만들었다는 점입니다.

하드웨어가 소프트웨어 덩어리로 변했기 때문에, 이제 인프라 엔지니어는 서버실에 들어가지 않습니다.
* `create server --cpu 4 --ram 16g --storage 100g --network private`
위와 같은 **문자열 텍스트 파일(코드)**을 작성해서 시스템에 던져 넣으면(Terraform 등 사용), 코드가 실행되면서 SDDC의 모든 자원들이 스스로 배합되어 5분 만에 서비스 환경이 뚝딱 지어집니다. 
데이터센터 전체가 거대한 하나의 '소프트웨어 프로그램'처럼 작동하게 된 것입니다.
