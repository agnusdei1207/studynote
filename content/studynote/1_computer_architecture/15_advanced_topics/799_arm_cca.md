# ARM CCA (Confidential Compute Architecture)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: ARM의 기밀 컴퓨팅 아키텍처로 Realm(격리 실행 환경)을 제공하여 In-Use 데이터 보호
> 2. **가치**: 하드웨어 기반 격리, RME(Realm Management Extension), Granule 보호, 증명
> 3. **융합**: TrustZone, TEE, 기밀 컴퓨팅, 모바일/엣지 보안과 통합된 ARM 보안

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ARM CCA(Confidential Compute Architecture)는 ARM의 기밀 컴퓨팅 아키텍처로 Realm(격리 실행 환경)을 제공하여 In-Use 데이터를 보호합니다.

**💡 비유**: CCA는 건물 내에 별도의 보안 구역을 추가하는 것과 같습니다.

**📢 섹션 요약 비유**: CCA는 건물 안에 보안 구역을 추가해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **RME** | Realm 관리 | ARMv9-A 확장 | 경비 시스템 |
| **Realm** | 격리 환경 | VM/App 실행 | 보안 구역 |
| **GPT** | Granule 보호 | 메모리 격리 | 출입 제어 |
| **RMM** | Realm 관리자 | Firmware | 관리자 |

**CCA 구성 요소**

| 구성 요소 | 설명 |
|:---|:---|
| **RME** | Realm Management Extension |
| **GPT** | Granule Protection Table |
| **RMM** | Realm Management Monitor |
| **GCS** | Guarded Control Stack |

**핵심 알고리즘**: Realm 생성 → GPT 설정 → 격리 실행 → 증명

**📢 섹션 요약 비유**: ARMv9부터 CCA가 있어요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: CCA vs TrustZone vs Intel TDX**

| 비교 항목 | CCA | TrustZone | Intel TDX |
|:---|:---:|:---:|:---:|
| **격리 단위** | Realm | Secure World | TD |
| **다중 격리** | 예 | 아니오 | 예 |
| **플랫폼** | ARM | ARM | x86 |
| **메모리** | Granule | 전체 | EPT |

**📢 섹션 요약 비유**: CCA는 다중 Realm을 만들 수 있어요!
