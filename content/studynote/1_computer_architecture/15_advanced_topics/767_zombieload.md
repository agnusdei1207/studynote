# 좀비로드 (ZombieLoad)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: L1 데이터 캐시의 언도된(uncommitted) 데이터를 유출하는 MDS 계열 사이드 채널 공격
> 2. **가치**: 보안 취약점 이해, L1D 캐시 보안, 완화 조치, CPU 아키텍처 보안
> 3. **융합**: MDS, RIDL, Fallout, 사이드 채널 공격과 통합된 캐시 기반 공격

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

좀비로드(ZombieLoad)는 L1 데이터 캐시의 언도된(uncommitted) 데이터를 유출하는 MDS 계열 사이드 채널 공격입니다. CVE-2019-11091로 등록되었습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    좀비로드 (ZombieLoad) 공격                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ZombieLoad 공격 원리                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │              L1 Data Cache                          │   │ │
│   │   │                                                     │   │ │
│   │   │   ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐         │   │ │
│   │   │   │ Line  │ │ Line  │ │ Line  │ │ Line  │         │   │ │
│   │   │   │   0   │ │   1   │ │   2   │ │   3   │  ...    │   │ │

│   │   │   └───────┘ └───────┘ └───────┘ └───────┘   데이터  │   │ │

│   │                                                              │ │
│   │   공격 단계:                                                  │ │
│   │   1. 트랜잭션이 실패/롤백되어도 L1D에 데이터 잔존            │ │
│   │   2. "좀비" 데이터가 Invalid 상태로 남음                     │ │
│   │   3. 공격자가 좀비 데이터를 로드 시도                        │ │
│   │   4. 타이밍 측정으로 데이터 추론                             │ │
```

> **해설**: ZombieLoad는 트랜잭션 실패 후 L1D에 남은 "좀비" 데이터를 유출합니다.

**💡 비유**: ZombieLoad는 귀신 들린 집과 같습니다. 떠난 사람(실패한 트랜잭션)의 흔적이 남습니다.

**📢 섹션 요약 비유**: ZombieLoad는 귀신 같아요. 죽은 데이터가 살아나요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **L1D Cache** | 1차 데이터 캐시 | 32KB per Core | 보관함 |
| **Uncommitted Data** | 언도된 데이터 | 트랜잭션 실패 후 잔존 | 귀신 |
| **Invalid Line** | 무효 라인 | 데이터 있지만 무효 표시 | 버려진 방 |
| **타이밍 측정** | 채널 감지 | 캐시 히트/미스 | 관찰 |

**ZombieLoad 변종**

| 변종 | 대상 | CVE | 설명 |
|:---|:---|:---|:---|
| **ZombieLoad v1** | L1D | CVE-2019-11091 | 기본 변종 |
| **ZombieLoad v2** | L1D | CVE-2019-11135 | TSX 관련 |

**핵심 알고리즘**: 트랜잭션 실패 → L1D 잔존 → 좀비 데이터 로드 → 유출

**📢 섹션 요약 비유**: 실패한 트랜잭션의 데이터가 "좀비"로 살아나요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: ZombieLoad vs RIDL vs Fallout**

| 비교 항목 | ZombieLoad | RIDL | Fallout |
|:---|:---:|:---:|:---:|
| **대상** | L1D 캐시 | LFB | Store Buffer |
| **데이터** | Uncommitted | In-flight | Committed |
| **트리거** | 트랜잭션 실패 | 캐시 미스 | 커널 전환 |

**📢 섹션 요약 비유**: MDS 계열은 각각 다른 CPU 버퍼를 공격해요!
