> **핵심 인사이트**
> 1. 클라우드에서 AI 모델을 훈련하려면 가상 머신(VM) 안에서 최신 Nvidia GPU를 100% 성능으로 돌려야 한다.
> 2. 이때 하이퍼바이저가 중간에 끼어서 GPU 명령을 통제하면 속도가 급락하므로, 아예 **호스트 서버에 꽂힌 물리 GPU 장치를 통째로 뜯어내어 VM에게 직접 던져주는 'PCIe 패스스루(Passthrough)'**가 필요하다.
> 3. **VFIO (Virtual Function I/O)**는 리눅스 커널에서 이 물리 장치의 제어권을 안전하게 사용자(VM)에게 넘겨주고, IOMMU를 이용해 메모리 침범을 막아주는 가장 핵심적인 하드웨어 직접 접근 프레임워크다.



## Ⅰ. 디바이스 패스스루(Passthrough)의 로망

컴퓨터에 비싼 RTX 4090 그래픽카드를 꽂았습니다.
이 컴퓨터 안에 윈도우 가상 머신(VM)을 띄워서 고사양 3D 게임이나 딥러닝을 하고 싶습니다. 

* **일반 가상화 (에뮬레이션/반가상화)**: 하이퍼바이저가 '가상 그래픽카드'를 만들어줍니다. 아무리 발악해도 RTX 4090의 성능이 10%밖에 안 나오며 3D 게임은 불가능합니다.
* **패스스루 (Passthrough)**: 하이퍼바이저가 호스트 OS(리눅스)에게서 RTX 4090에 대한 모든 통제권을 강제로 빼앗은 뒤, **가상 머신(윈도우)에게 물리적인 선을 직접 연결해 주듯 100% 줘버리는 것**입니다.

이렇게 하면 윈도우 VM 화면에 마치 진짜 PC처럼 'Nvidia RTX 4090'이라는 하드웨어 이름이 정확히 뜨고, 엔비디아 드라이버를 직접 깔아 100%의 물리적 성능을 낼 수 있습니다.

> 📢 **섹션 요약 비유**: 게임기를 거실 TV에 연결해놓고, 아빠(호스트 OS)와 아들(가상 머신)이 화면을 쪼개서 나눠 쓰는 것이 기존 방식입니다. 패스스루는 아예 게임기 선을 뽑아서 아들 방 모니터에 다이렉트로 꽂아줘 버린 것입니다(독점). 아빠는 이제 거실에서 그 게임기를 절대 쓸 수 없습니다.



## Ⅱ. VFIO: 안전한 하드웨어 탈취 프레임워크

이 패스스루의 꿈을 안전하게 실현해 주는 리눅스의 핵심 마법이 **VFIO**입니다.

과거에는 장치를 넘겨주기 위해 커널 코드를 헤집어 놓는 더러운 방식(KVM 장치 할당 등)을 썼지만, VFIO는 이를 아주 깔끔한 표준으로 만들었습니다.

1. **디바이스 격리 (Isolation)**: VFIO 모듈을 켜면, 호스트 리눅스가 잡고 있던 그래픽카드의 기본 드라이버(Nouveau 등)를 강제로 쫓아냅니다.
2. **IOMMU 방어벽 활성화**: 장치를 VM에게 주면 가장 위험한 것은 그 장치(GPU)가 메인 메모리의 엉뚱한 곳을 쑤시고 다니는 것(DMA 공격)입니다. VFIO는 CPU 하드웨어의 **IOMMU(앞 장 참조)**를 철저히 설정하여, 넘겨준 GPU가 딱 자기 가상 머신의 메모리 영역 밖으로는 1바이트도 못 나가게 철창을 칩니다.
3. **유저 공간 노출**: 모든 안전 세팅이 끝나면 이 장치를 `/dev/vfio/`라는 단순한 파일 형태로 던져줍니다. KVM이나 QEMU(유저 프로그램)는 그냥 이 파일을 읽고 쓰기만 하면 물리적 그래픽카드를 직접 조종하게 됩니다.

### VFIO 패스스루 구조 (ASCII)

```text
 ┌─── 물리 서버 (Host OS: 리눅스) ───────────────────────────┐
 │                                                           │
 │ ┌── 가상 머신 (Windows VM) ──┐                            │
 │ │ Nvidia 드라이버 설치 됨    │                            │
 │ │ 100% 네이티브 성능 발휘    │                            │
 │ └────────┬─────────────────┘                              │
 │          │ (VFIO 채널)                                    │
 │ ┌────────▼────────┐                                       │
 │ │      VFIO       │ (IOMMU 하드웨어 철창으로 메모리 보안) │
 │ └────────┬────────┘                                       │
 │          │ (직통 통신)                                    │
 │ ┌────────▼────────┐                                       │
 │ │ RTX 4090 (물리) │ ◀ (호스트 OS는 이 장치에 접근 불가)   │
 │ └─────────────────┘                                       │
 └───────────────────────────────────────────────────────────┘
```

> 📢 **섹션 요약 비유**: 맹수(GPU)의 목줄을 아들(VM)에게 통째로 넘겨주는 작업(VFIO)입니다. 아들이 맹수를 마음껏 부릴 수 있게 하되, 맹수가 아빠의 방(호스트 메모리)으로 넘어오지 못하도록 완벽한 쇠창살(IOMMU)을 미리 쳐주는 안전관리 시스템입니다.



## Ⅲ. VFIO의 확장: DPDK (네트워크 폭주)

VFIO는 가상 머신(VM)에 그래픽카드를 던져주는 것 외에도 **데이터센터의 초고속 네트워크(SDN)**를 구축하는 데 절대적으로 쓰입니다.

100Gbps 랜카드로 쏟아지는 패킷을 OS 커널(TCP/IP)이 처리하면 너무 느립니다. 
그래서 인텔과 리눅스는 아예 랜카드를 VFIO로 떼어내서 유저 모드에 띄워둔 **소프트웨어 라우터(DPDK, 다음 챕터 예정)**에게 직접 던져줍니다. OS가 개입하는 모든 컨텍스트 스위칭 오버헤드를 0으로 만들어, 네트워크 처리 속도를 수십 배 펌핑시키는 근간 기술이 바로 VFIO입니다.
