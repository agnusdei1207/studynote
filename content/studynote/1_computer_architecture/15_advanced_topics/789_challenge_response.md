# 도전-응답 쌍 (Challenge-Response Pair)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: PUF에 특정 입력(Challenge)을 주고 고유한 출력(Response)을 얻는 인증 프로토콜
> 2. **가치**: 디바이스 인증, 복제 방지, 비밀 저장 없는 인증, 하드웨어 보안
> 3. **융합**: SRAM PUF, 인증 프로토콜, 키 교환과 통합된 물리적 인증

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

도전-응답 쌍(Challenge-Response Pair, CRP)은 PUF에 특정 입력(Challenge)을 주고 고유한 출력(Response)을 얻는 인증 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    도전-응답 쌍 (Challenge-Response Pair)             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CRP 인증 프로토콜                                 │ │
│   │                                                              │ │
│   │   ┌─────────────┐                    ┌─────────────┐        │ │
│   │   │   서버      │                    │   디바이스   │        │ │
│   │   │ (Verifier)  │                    │  (Prover)   │        │ │
│   │   └──────┬──────┘                    └──────┬──────┘        │ │
│   │          │                                  │                │ │
│   │          │  1. Challenge (난수)              │                │ │
│   │          │ ────────────────────────────────►│                │ │
│   │          │                                  │                │ │
│   │          │                    2. PUF(Response) 계산          │ │
│   │          │                       Response = PUF(Challenge)   │ │
│   │          │                                  │                │ │
│   │          │  3. Response                      │                │ │
│   │          │ ◄────────────────────────────────│                │ │
│   │          │                                  │                │ │
│   │          │  4. Response 검증                 │                │ │
│   │          │     (저장된 CRP와 비교)            │                │ │
│   │          │                                  │                │ │
│   │          └──────────────────────────────────┘                │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CRP 특성                                          │ │
│   │                                                              │ │
│   │   Challenge (도전):                                           │ │
│   │   - 서버가 생성한 난수 또는 특정 패턴                         │ │
│   │   - PUF의 입력으로 사용                                       │ │
│   │                                                              │ │
│   │   Response (응답):                                            │ │
│   │   - PUF가 Challenge에 대해 생성한 고유 출력                   │ │
│   │   - 디바이스마다 다름                                         │ │
│   │                                                              │ │
│   │   CRP 데이터베이스:                                           │ │
│   │   - 등록 단계에서 다수의 CRP 수집                             │ │
│   │   - 인증 시 저장된 Response와 비교                            │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: CRP는 Challenge를 PUF에 입력하여 Response를 얻고 저장된 값과 비교합니다.

**💡 비유**: CRP는 질문-답변 시험과 같습니다. 미리 정해진 질문에 올바른 답을 해야 합니다.

**📢 섹션 요약 비유**: CRP는 질문-답변 시험 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Challenge** | 입력 값 | 주소/난수 | 질문 |
| **PUF** | 변환 함수 | 물리적 특성 | 채점자 |
| **Response** | 출력 값 | 고유 패턴 | 답변 |
| **CRP DB** | 저장소 | 등록된 쌍 | 정답표 |

**CRP 프로토콜 단계**

| 단계 | 동작 | 목적 |
|:---|:---|:---|
| **1. 등록** | 다수 CRP 수집 | DB 구축 |
| **2. Challenge** | 난수 전송 | 인증 시작 |
| **3. Response** | PUF 계산 | 증명 |
| **4. 검증** | DB 비교 | 인증 완료 |

**핵심 알고리즘**: 등록 → Challenge 전송 → Response 생성 → 검증

**📢 섹션 요약 비유**: 미리 등록하고, 질문하고, 답하고, 확인해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: CRP vs 비밀번호 vs 인증서**

| 비교 항목 | CRP | 비밀번호 | 인증서 |
|:---|:---:|:---:|:---:|
| **저장** | 없음 | 있음 | 있음 |
| **재사용** | 1회용 | 반복 | 반복 |
| **복제** | 불가능 | 가능 | 가능 |
| **물리적** | 예 | 아니오 | 아니오 |

**📢 섹션 요약 비유**: CRP는 1회용이고 복제 불가능해요!
