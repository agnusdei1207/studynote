# 신뢰성 블록 다이어그램 (RBD, Reliability Block Diagram)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 구성요소의 신뢰성 관계를 블록 다이어그램으로 표현하여 전체 신뢰성을 계산하는 기법
> 2. **가치**: 직관적인 신뢰성 모델링, 직렬/병렬 구성 계산, 가용성 정량화
> 3. **융합**: FTA, FMEA, 마르코프 모델, HA 설계와 통합된 신뢰성 공학

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

신뢰성 블록 다이어그램(RBD, Reliability Block Diagram)은 시스템 구성요소의 신뢰성 관계를 블록 다이어그램으로 표현하여 전체 신뢰성을 계산하는 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RBD 기본 구성                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              직렬 구성 (Series)                                │ │
│   │                                                              │ │
│   │   Input ──┬──[ A ]──┬──[ B ]──┬──[ C ]──┬── Output          │ │
│   │           │         │         │         │                    │ │
│   │   Rs = R(A) × R(B) × R(C)                                   │ │
│   │   모든 구성요소가 정상이어야 동작                              │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              병렬 구성 (Parallel)                              │ │
│   │                                                              │ │
│   │               ┌──[ A ]──┐                                    │ │
│   │   Input ──┬──┤         ├──┬── Output                        │ │
│   │           │  ├──[ B ]──┤  │                                 │ │
│   │           │  └──[ C ]──┘  │                                 │ │
│   │                                                              │ │
│   │   Rp = 1 - (1-R(A)) × (1-R(B)) × (1-R(C))                   │ │
│   │   하나라도 정상이면 동작                                      │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              k-out-of-n 구성                                  │ │
│   │                                                              │ │
│   │   n개 중 k개 이상 정상이면 동작                               │ │
│   │   예: 3개 중 2개 정상 (2oo3)                                 │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: RBD는 직렬(AND), 병렬(OR), k-out-of-n 구성으로 신뢰성을 계산합니다.

**💡 비유**: RBD는 파이프라인과 같습니다. 직렬은 모든 파이프가 뚫려야 하고, 병렬은 하나라도 뚫리면 됩니다.

**📢 섹션 요약 비유**: RBD는 파이프 연결 같아요. 직렬은 다 뚫려야 해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **블록** | 구성요소 | 신뢰도 R(t) | 파이프 |
| **직렬** | AND 관계 | R = R1×R2×... | 일렬 연결 |
| **병렬** | OR 관계 | R = 1-(1-R1)(1-R2)... | 병렬 연결 |
| **k-out-of-n** | 다수결 | 조합 계산 | 투표 |

**신뢰도 계산 예시**

| 구성 | R1=R2=0.9 | R1=R2=0.99 | 비고 |
|:---|:---:|:---:|:---|
| **직렬** | 0.81 | 0.9801 | R1×R2 |
| **병렬** | 0.99 | 0.9999 | 1-(1-R1)(1-R2) |
| **2oo3** | 0.972 | 0.999702 | C(3,2)+C(3,3) |

**핵심 알고리즘**: 구성 분석 → 직렬/병렬 분해 → 신뢰도 계산 → 전체 신뢰도

**📢 섹션 요약 비유**: 병렬은 신뢰도가 높아요! 이중화가 중요해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: RBD vs FTA vs FMEA**

| 비교 항목 | RBD | FTA | FMEA |
|:---|:---:|:---:|:---:|
| **표현** | 다이어그램 | 트리 | 표 |
| **방향** | 정상 동작 | 고장 | 고장 |
| **계산** | 신뢰도 | 고장 확률 | RPN |
| **용도** | 가용성 | 사고 분석 | 위험 분석 |

**📢 섹션 요약 비유**: RBD는 정상, FTA/FTA는 고장을 분석해요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: HA 설계** → 병렬 RBD로 가용성 계산
**시나리오 2: SPOF 식별** → 직렬 구성에서 낮은 신뢰도 블록
**시나리오 3: k-out-of-n** → 쿼럼 기반 시스템 (3노드 중 2)

**도입 체크리스트**: 구성요소 신뢰도, 구조 분석, 정량적 목표, 민감도 분석

**📢 섹션 요약 비유**: RBD로 99.999% 가용성을 계산해요!
