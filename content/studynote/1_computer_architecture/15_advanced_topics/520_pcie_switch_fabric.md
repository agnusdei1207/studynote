# PCIe Switch Fabric

> 1. **본질**: PCIe 스위치 패브릭 (PCIe Switch Fabric)은 단일 Root Complex의 제한된 레인(Lane) 수를 확장하고, 다수의 PCIe 장치 간에 점대점(Point-to-Point) 고속 통신 및 팬아웃(Fan-out)을 제공하는 지능형 스위칭 네트워크 아키텍처이다.
> 2. **가치**: 서버 한 대에 수십 개의 GPU나 NVMe 드라이브를 연결할 수 있게 하며, 장치 간 직접 통신(P2P)을 지원하여 CPU의 개입 없이 가속기 간 데이터 전송 효율을 극대화한다.
> 3. **융합**: 고성능 컴퓨팅(HPC), GPU 클러스터, NVMe-oF (NVMe over Fabrics), 그리고 차세대 인터커넥트인 CXL (Compute Express Link) 기술과 융합되어 거대한 데이터 패브릭을 형성한다.

+++

## Ⅰ. 개요 (Context & Background)

- **개념**: PCIe 스위치 패브릭은 "컴퓨터 내부의 고속도로 교차로 및 확장 도로"이다. CPU가 직접 제공하는 통로가 부족할 때, 스위치를 달아 수많은 장치가 서로 엉키지 않고 초고속으로 데이터를 주고받을 수 있게 해주는 그물망 구조의 통신망이다.

- **💡 비유**: PCIe 스위치 패브릭은 "대형 물류 센터의 자동 분류 컨베이어 시스템"과 같다. 트럭(데이터 패킷)들이 들어오는 입구는 몇 개 안 되지만, 내부의 스위치(분류기)가 목적지를 확인하여 수십 개의 하차장(장치)으로 정확하고 빠르게 보내주는 복잡한 네트워크와 같다.

- **등장 배경**: 
  1. **가속기 중심 컴퓨팅**: AI 학습을 위해 한 서버에 8개 이상의 GPU를 꽂아야 하는데, CPU가 제공하는 PCIe 레인 수로는 턱없이 부족해짐.
  2. **장치 간 통신(P2P) 요구**: GPU끼리 데이터를 주고받을 때 굳이 느린 CPU 메모리를 거치지 않고 직접 통신하려는 요구 증가.
  3. **Composable Infrastructure**: 필요에 따라 자원(GPU, SSD)을 유연하게 할당하고 재구성할 수 있는 가변적인 서버 구조의 필요성.

- **📢 섹션 요약 비유**: CPU의 한정된 연결 능력을 무한히 확장하고 장치들끼리 '직거래'를 가능하게 만드는 내부 고속 통신망입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 및 내부 로직

PCIe 스위치는 단순한 허브를 넘어 패킷 라우팅과 중재를 수행하는 지능형 장치이다.

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **Upstream Port** | CPU(Root Complex) 쪽 연결 | 호스트와의 통신 및 제어 신호 수신 | PCIe Link Training | 물류 센터 정문 |
| **Downstream Port** | 주변 장치(GPU, SSD) 연결 | 여러 장치로의 경로 확장(Fan-out) | Hot-plug Support | 개별 하차장 문 |
| **Switch Core (Crossbar)** | 데이터 패킷 고속 교환 | 입력 포트의 패킷을 목적지 출력 포트로 즉시 라우팅 | Non-blocking Fabric | 중앙 자동 분류기 |
| **Packet Buffer / Queue** | 데이터 일시 보관 | 포트 경합 시 패킷을 잠시 저장하여 손실 방지 | Flow Control (Credit) | 임시 하차 대기소 |
| **P2P Bridge** | 장치 간 직접 통신 제어 | CPU를 거치지 않고 Downstream 포트 간 데이터 전송 | Peer-to-Peer DMA | 내부 전용 지름길 |

+++

### PCIe 스위치 패브릭 구조 및 데이터 흐름

CPU와 다수의 GPU/NVMe 장치가 스위치를 통해 어떻게 연결되는지 시각화한다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               PCIe 스위치 패브릭 (Switch Fabric) 아키텍처           │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │       [ CPU / Root Complex ]                                  │
 │               │ (x16 Link)                                    │
 │       ┌───────▼───────┐                                       │
 │       │ Upstream Port │                                       │
 │   ┌───┴───────────────┴───┐                                   │
 │   │    [ Switch Core ]    │◀─── (Packet Routing & Arbiter)    │
 │   └───┬───────┬───────┬───┘                                   │
 │       │       │       │                                       │
 │   ┌───▼───┐┌──▼───┐┌──▼───┐   [ Peer-to-Peer Path ]           │
 │   │ Port  ││ Port  ││ Port  │ ◀─── (GPU 1 to GPU 2 Direct)    │
 │   └─┬───┬─┘└─┬───┬─┘└─┬───┬─┘                                 │
 │     ▼   ▼    ▼   ▼    ▼   ▼                                   │
 │   [GPU 1]  [GPU 2]  [NVMe 1]  [NVMe 2]  ...                     │
 │                                                               │
 │   ※ 스위치를 통해 1개의 x16 슬롯이 수십 개의 장치로 확장됨.           │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** CPU에서 나오는 PCIe 레인은 한정되어 있다(보통 16~48개). 하지만 PCIe 스위치를 사용하면 하나의 x16 링크를 입력받아 이를 여러 개의 x8이나 x4 링크로 쪼개어 수십 개의 장치를 연결할 수 있다(Fan-out). 가장 중요한 특징은 **P2P(Peer-to-Peer)** 통신이다. 예를 들어 GPU 1이 GPU 2에게 데이터를 보낼 때, 기존에는 CPU를 거쳐야 했으나 스위치 패브릭 환경에서는 스위치 내부의 크로스바(Crossbar)를 통해 CPU 쪽으로 올라가지 않고 바로 옆 포트로 데이터를 쏜다. 이는 CPU 부하를 없애고 지연 시간을 획기적으로 줄여, NVIDIA의 DGX 시스템 같은 초고성능 AI 서버의 핵심 기반이 된다.

+++

### 심층 동작 원리: 패킷 라우팅 및 중재

1. **Transaction Layer Packet (TLP)**: PCIe 통신의 기본 단위. 주소 정보와 데이터를 포함함.
2. **ID-based / Address-based Routing**: 패킷의 목적지 ID나 메모리 주소를 보고 어느 포트로 보낼지 0.1마이크로초 이내에 결정.
3. **Credit-based Flow Control**: 수신 측 버퍼에 여유가 있는지 확인하고 패킷을 보내, 데이터 유실이 없는 신뢰성 있는 전송 보장.

- **📢 섹션 요약 비유**: 패킷마다 붙은 주소 라벨을 보고 번개 같은 속도로 길을 찾아주며, 길이 막히면 잠시 세워두는 똑똑한 교통 경찰과 같습니다.

+++

## Ⅲ. 융합 비교 및 다각도 분석

### 직접 연결 (Direct) vs 스위치 패브릭 (Fabric)

| 비교 항목 | 직접 연결 (Direct Connect) | 스위치 패브릭 (Switch Fabric) |
|:---|:---|:---|
| **장치 확장성** | 낮음 (CPU 레인 수에 고정) | 매우 높음 (계층적 확장 가능) |
| **데이터 경로** | 항상 CPU를 거침 | 장치 간 직접 통신(P2P) 가능 |
| **지연 시간** | 가장 낮음 (직통) | 약간의 스위칭 지연 발생 (ns 단위) |
| **시스템 유연성** | 낮음 (물리적 슬롯 고정) | 높음 (가상 할당 및 재구성 가능) |
| **비용/전력** | 낮음 | 높음 (추가 스위치 칩 필요) |

- **📢 섹션 요약 비유**: 집 앞에 바로 차를 세우는 개인 주차장(직접 연결)과, 수천 대의 차가 유기적으로 움직이는 거대 공항 주차 타워(패브릭)의 차이입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 8-GPU AI 학습 서버 구축**: CPU 2개로는 8개의 GPU를 x16 속도로 다 지원할 수 없다. 기술사는 **PLX 스위치(Broadcom 등)**를 활용한 패브릭 보드를 채택하여, GPU 간에 NVLink 급은 아니더라도 고속의 P2P 통신 환경을 구축해 학습 속도를 최적화한다.
2. **시나리오 — NVMe JBOF(Just a Bunch Of Flash) 설계**: 수백 개의 SSD를 외부 인클로저로 뺄 때. 기술사는 **PCIe Switch Fabric**을 사용하여 외부 케이블을 통해 서버와 SSD 뱅크를 연결하고, 필요에 따라 특정 VM에 SSD를 동적으로 할당하는 리소스 풀링 아키텍처를 수립한다.

### 도입 체크리스트
- **Non-blocking Bandwidth**: 모든 장치가 동시에 통신할 때 스위치 내부 대역폭이 포화되지 않는가?
- **P2P Compatibility**: 사용하려는 가속기와 OS 드라이버가 PCIe P2P 기능을 완벽히 지원하는가?
- **Thermal Management**: 고성능 PCIe 스위치 칩 자체의 발열이 시스템 냉각 한계를 넘지 않는가?

- **📢 섹션 요약 비유**: 도로가 넓어도 교차로(Switch Core)가 좁으면 정체가 생기므로, 교차로 처리 용량이 충분한지 체크하는 것이 핵심입니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 전 (Direct) | 도입 후 (Fabric) | 개선 효과 |
|:---|:---|:---|:---|
| **최대 연결 장치 수** | 2~4개 | 32~64개 이상 | **장치 수용량 10배 이상 확장** |
| **장치 간 통신 속도** | CPU 경유로 지연 발생 | P2P로 직접 전송 | **통신 지연 시간 50% 이상 단축** |
| **자원 활용 효율** | 슬롯에 박힌 장치만 사용 | 자유로운 자원 재배치 | **하드웨어 가용성 2배 향상** |

### 미래 전망
앞으로 PCIe 스위치는 **'CXL 스위치'**로 진화할 것이다. 단순히 데이터만 주고받는 게 아니라, 서로의 메모리를 자기 것처럼 공유하는 '캐시 일관성 패브릭'으로 발전하게 된다. CXL 3.0 이상이 보급되면 수천 대의 서버가 거대한 PCIe/CXL 스위치 패브릭으로 묶여 하나의 초거대 컴퓨터처럼 작동하는 **'Fabric-Attached Infrastructure'**가 데이터센터의 표준이 될 전망이다.

- **📢 섹션 요약 비유**: 이제는 건물 안의 복도를 넘어, 도시 전체를 잇는 고속도로 네트워크처럼 컴퓨터 내부의 연결도 거대하고 유연한 패브릭 시대로 진화하고 있습니다.

## 📌 관련 개념 맵
- **PCI Express (PCIe)**: 패브릭을 구성하는 기본 물리 인터페이스 기술.
- **CXL (Compute Express Link)**: PCIe 패브릭 위에서 메모리 공유 기능을 더한 차세대 기술.
- **NVMe-oF (NVMe over Fabrics)**: PCIe 패브릭을 네트워크로 확장하여 저장장치를 공유하는 기술.
- **DMA (Direct Memory Access)**: 패브릭을 통해 장치 간 데이터를 직접 옮기는 핵심 방식.

## 👶 어린이를 위한 3줄 비유 설명
1. PCIe 스위치 패브릭은 **'컴퓨터 안의 아주 큰 기차역'**이에요.
2. 기차(데이터)가 갈 수 있는 길(레인)이 부족할 때, 커다란 역을 만들어서 수많은 기차가 여러 방향으로 빠르게 갈 수 있게 해주죠.
3. 특히 CPU 역장님을 거치지 않고 장치들끼리 바로 옆 기찻길로 물건을 주고받을 수 있어서, 컴퓨터가 아주 복잡한 일도 척척 해낼 수 있게 된답니다!
