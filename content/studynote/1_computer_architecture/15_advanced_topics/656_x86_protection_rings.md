> **핵심 인사이트**
> 1. 과거 MS-DOS 시절에는 워드프로세서가 하드디스크의 0번지 섹터를 마음대로 지우거나 컴퓨터 전원을 끌 수 있었다. (보안 0점)
> 2. 이를 막기 위해 인텔 x86 아키텍처는 CPU의 하드웨어 실행 권한을 **Ring 0 (가장 강력함)부터 Ring 3 (가장 약함)까지 4개의 계급**으로 나누는 **보호 모드(Protected Mode)**를 도입했다.
> 3. 운영체제(커널)는 왕인 Ring 0에서 놀고, 우리가 쓰는 크롬 브라우저나 게임은 천민인 Ring 3에 갇힌다. 천민이 하드웨어를 만지려 하면 CPU가 즉시 "권한 없음(Exception)!"이라며 프로그램을 강제 종료(Crash)시킨다.



## Ⅰ. 권한 분리의 필요성 (커널 패닉 방지)

컴퓨터 안에는 수많은 프로그램이 돕니다. 만약 내가 짠 허접한 테트리스 게임이 메모리 관리를 잘못해서, 운영체제(Windows)가 시스템을 돌리기 위해 쓰는 핵심 메모리 영역을 덮어써버리면 어떻게 될까요?
윈도우가 미쳐버리고 끔찍한 **블루스크린(Kernel Panic)**이 뜨며 컴퓨터가 재부팅됩니다.

이런 짓을 원천 차단하려면 "너희 같은 일반 프로그램은 절대 이 선을 넘지 마라!"라는 물리적인 하드웨어 차단막이 필요했습니다. 그래서 CPU 안에 '계급장'을 박아 넣은 것이 **보호 모드의 링(Ring) 구조**입니다.

> 📢 **섹션 요약 비유**: 회사에서 말단 신입사원(게임 앱)이 사장님 결재 서류(커널 메모리)나 금고 다이얼(하드웨어 제어)을 자기 맘대로 만지면 회사가 망합니다. 그래서 사장님은 신입사원의 사원증 출입 권한(Ring)을 로비(유저 모드)로만 제한해 버렸습니다.



## Ⅱ. 4개의 동심원: Ring 0 ~ 3의 세계

x86 CPU는 구조적으로 4단계의 권한 계층을 지원합니다. 번호가 낮을수록 권한이 막강합니다.

1. **Ring 0 (커널 모드, 최고 존엄)**
   * **주인**: 운영체제의 심장(Windows Kernel, Linux Kernel, 드라이버).
   * **권한**: 100% 무소불위. 하드디스크 포맷, CPU 온도 조절, 모든 메모리 접근 등 '특권 명령어(Privileged Instructions)'를 마음대로 실행할 수 있습니다.
2. **Ring 1, 2 (디바이스 드라이버 영역)**
   * 원래는 그래픽카드나 프린터 드라이버에게 주려고 만든 중간 계급입니다. 하지만 OS 개발자들이 "그냥 복잡하게 나누지 말고 0이랑 3만 쓰자"라고 합의하는 바람에 **현재는 거의 버려진(안 쓰는) 계급**이 되었습니다.
3. **Ring 3 (유저 모드, 천민 구역)**
   * **주인**: 워드, 크롬, 카카오톡, 게임 등 여러분이 까는 99.9%의 모든 프로그램.
   * **권한**: 오직 더하기, 곱하기 같은 바보 같은 산술 연산과 자신에게 허락된 좁은 메모리 공간만 만질 수 있습니다. 하드웨어의 'ㅎ'자도 못 건드립니다.

### 권한 다이어그램 (ASCII)

```text
       (뚫고 들어갈 수 없는 권한의 벽)
 ┌─────────────────────────────────────────────┐
 │ ┌─────────────────────────────────────┐     │
 │ │ ┌─────────────────────────────────┐ │     │
 │ │ │       👑 Ring 0 (OS 커널)       │ │     │
 │ │ │ (모든 하드웨어 및 메모리 제어)      │ │ │
 │ │ └─────────────────────────────────┘ │     │
 │ │             Ring 1 (안 씀)            │   │
 │ └─────────────────────────────────────┘     │
 │                 Ring 2 (안 씀)              │
 └─────────────────────────────────────────────┘
        🙋‍♂️ Ring 3 (일반 사용자 프로그램)
    (이 벽 밖에서 더하기/빼기 연산만 하며 갇혀있음)
```

> 📢 **섹션 요약 비유**: 4겹의 철조망으로 둘러싸인 성입니다. 성의 가장 깊숙한 중심(Ring 0)에는 왕(커널)이 살며 모든 것을 통제합니다. 평민(프로그램)들은 4번째 철조망 밖(Ring 3)에서만 농사를 지을 수 있으며, 함부로 안쪽으로 들어오려 하면 경비병(CPU 하드웨어 예외)에게 즉시 목이 베입니다(프로그램 튕김).



## Ⅲ. Ring 3가 Ring 0의 힘을 빌리는 법 (System Call)

"그럼 게임(Ring 3)이 하드디스크에 세이브 파일을 저장하고 싶을 땐 어떻게 하나요? 디스크 접근은 Ring 0만 할 수 있잖아요!"

이때 사용하는 합법적인 면회 창구가 바로 **시스템 콜(System Call)**입니다.
1. 게임(Ring 3)이 CPU에게 특별한 신호(예: `syscall` 명령어)를 보냅니다. "저기요, 파일 좀 저장하고 싶은데요."
2. 게임은 그 자리에 **정지**하고, CPU는 보안 검색대를 통과시켜 모드를 **Ring 0**로 쑥 올려줍니다 (Context Switch).
3. 진짜 왕인 OS 커널(Ring 0)이 깨어나서, 게임이 요구한 파일 저장 명령이 안전한지 깐깐하게 검사한 뒤, 자기가 직접 하드디스크에 파일을 대신 써줍니다.
4. 다 쓰면 다시 권한을 **Ring 3**로 떨어뜨리고 게임에게 "저장 완료!"라고 알려주며 게임이 재개됩니다.

> 📢 **섹션 요약 비유**: 평민(Ring 3)이 자기 집에 금고(하드디스크)를 달고 싶습니다. 평민이 직접 금고를 달면 불법이므로, 동사무소 민원 창구(시스템 콜)에 가서 정식으로 서류를 냅니다. 그러면 관공서 공무원(Ring 0)이 나와서 안전하게 금고를 대신 설치해주고 돌아가는 완벽한 보안 통제 시스템입니다.
