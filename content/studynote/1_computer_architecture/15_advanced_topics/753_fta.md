# FTA (Fault Tree Analysis)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 최상위 시스템 고장 사건부터 역추적하여 근본 원인을 분석하는 하향식 신뢰성 분석 기법
> 2. **가치**: 사고 원인 체계적 분석, 정량적 고장 확률 계산, 안전 설계 검증
> 3. **융합**: FMEA, RBD, 마르코프 모델, 안전 공학과 통합된 사고 분석

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

FTA(Fault Tree Analysis)는 최상위 시스템 고장 사건부터 역추적하여 근본 원인을 분석하는 하향식 신뢰성 분석 기법입니다. 논리 게이트(AND, OR)로 고장 경로를 모델링합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FTA (Fault Tree) 구조                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                        ┌─────────────┐                              │
│                        │   Top Event │                              │
│                        │ 시스템 고장  │                              │
│                        └──────┬──────┘                              │
│                               │                                     │
│                               ▼                                     │
│                        ┌─────────────┐                              │
│                        │    OR 게이트 │                              │
│                        └──────┬──────┘                              │
│              ┌────────────────┼────────────────┐                    │
│              │                │                │                    │
│              ▼                ▼                ▼                    │
│       ┌───────────┐   ┌───────────┐   ┌───────────┐                │
│       │  PSU 고장  │   │ AND 게이트 │   │  Fan 고장 │                │
│       └───────────┘   └─────┬─────┘   └───────────┘                │
│                             │                                       │
│                    ┌────────┴────────┐                              │
│                    │                 │                              │
│                    ▼                 ▼                              │
│             ┌───────────┐     ┌───────────┐                        │
│             │CPU 과열    │     │Fan 2 고장 │                        │
│             └───────────┘     └───────────┘                        │
│                                                                     │
│   기호: ○ 기본 사건, □ 중간 사건, ◇ 미전개 사건                     │
│        OR: 하나라도 발생, AND: 모두 발생                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: FTA는 Top Event(시스템 고장)에서 시작하여 논리 게이트로 원인을 역추적합니다.

**💡 비유**: FTA는 사고 수사와 같습니다. 결과(사고)에서 원인(범인)을 역추적합니다.

**📢 섹션 요약 비유**: FTA는 사고 수사 같아요. 결과에서 원인을 찾아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Top Event** | 분석 대상 | 시스템 고장 | 사건 현장 |
| **논리 게이트** | 인과 관계 | AND/OR 연산 | 추론 |
| **기본 사건** | 근본 원인 | 더 이상 분해 불가 | 원인 |
| **중간 사건** | 연결 사건 | 게이트 출력 | 중간 단계 |

**FTA 논리 게이트**

| 게이트 | 기호 | 의미 | 확률 계산 |
|:---|:---:|:---|:---|
| **OR** | ∧ 형태 | 하나라도 발생 | P(A)+P(B)-P(A)P(B) |
| **AND** | ⋂ 형태 | 모두 발생 | P(A)×P(B) |
| **XOR** | 배타적 OR | 하나만 발생 | P(A)+P(B)-2P(A)P(B) |

**핵심 알고리즘**: Top Event → 논리 게이트 → 기본 사건 → 확률 계산

**📢 섹션 요약 비유**: FTA는 논리 퍼즐 같아요. 게이트로 연결해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FTA vs FMEA vs RBD**

| 비교 항목 | FTA | FMEA | RBD |
|:---|:---:|:---:|:---:|
| **방향** | 하향식 | 상향식 | 구조적 |
| **시작점** | Top Event | 구성요소 | 시스템 |
| **분석** | 정성+정량 | 정성+정량 | 정량 |
| **용도** | 사고 분석 | 설계 검토 | 가용성 |

**📢 섹션 요약 비유**: FTA는 하향식, FMEA는 상향식, RBD는 구조적이에요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 사고 분석** → FTA로 근본 원인 추적
**시나리오 2: 안전 설계** → FTA로 치명적 경로 식별
**시나리오 3: HA 설계** → 단일 장애점(SPOF) 분석

**도입 체크리스트**: Top Event 정의, 경계 설정, 데이터 수집, 소프트웨어 도구

**📢 섹션 요약 비유**: FTA는 SPOF를 찾는 데 유용해요. 단일 장애점을 제거해요!
