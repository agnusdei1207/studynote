> **핵심 인사이트**
> 1. 과거의 컴퓨터(MS-DOS 시절 등)는 모든 프로그램이 하나의 물리적 메모리(RAM)를 공유하여, A 프로그램의 버그나 악의적인 해킹으로 B 프로그램이나 운영체제(OS)가 죽는 일이 흔했다.
> 2. 이를 해결하기 위해 등장한 **가상 주소 공간 분리(Isolation)**는 각 프로그램(프로세스)에게 "이 64비트(또는 32비트)의 거대한 메모리 공간은 온전히 너 혼자만의 것"이라는 **가짜 환상(가상 메모리)**을 심어주는 기술이다.
> 3. 하드웨어 MMU(메모리 관리 장치)가 이 가짜 주소를 서로 겹치지 않는 흩어진 진짜 물리 주소로 번역해 주므로, 프로그램들은 서로의 존재를 알 수도 없고 남의 메모리에 침범할 수도 없는 완벽한 보안 벽이 형성된다.



## Ⅰ. 물리 메모리 직접 접근의 참사

옛날 컴퓨터에서는 워드프로세서와 게임이 동시에 실행될 때, 진짜 물리 메모리 주소를 그대로 썼습니다.
* 워드는 `0번~1000번` 주소를 씁니다.
* 게임은 `1001번~2000번` 주소를 씁니다.

**문제 발생**: 만약 게임 개발자가 코딩을 실수해서 1000번 주소를 침범하여 값을 써버리면? 애먼 워드프로세서가 에러를 뿜으며 강제 종료됩니다. 
악의적인 해커는 이 점을 이용해 OS 커널이 있는 메모리 주소(예: `0번지`)에 악성코드를 덮어써서 컴퓨터 전체를 장악해버렸습니다. 이것이 물리 메모리 환경의 한계입니다.

> 📢 **섹션 요약 비유**: 강당(물리 메모리)에 100명이 모여 각자 바닥에 분필로 선을 긋고 시험을 칩니다. 누군가 악의를 품고 선을 넘어와 내 시험지를 찢어버리거나 훔쳐봐도, 강당에는 벽이 없기 때문에 막을 방법이 없습니다.



## Ⅱ. 완벽한 독방: 가상 주소 공간의 환상

현대의 운영체제(Windows, Linux, iOS)와 CPU 하드웨어(MMU)는 모든 프로세스를 **완벽한 독방(가상 주소 공간)**에 가둬버립니다.

1. **환상의 제공**: 워드프로세서를 켜면, OS는 워드에게 "자, 여기 `0번부터 4GB`까지 빈 메모리가 있으니 마음껏 써라"고 가짜 방(가상 주소)을 줍니다. 
2. **또 다른 환상**: 1초 뒤에 게임을 켜면, OS는 게임에게도 똑같이 "자, 여기 `0번부터 4GB`까지 빈 메모리야. 너 혼자 쓰는 거다"라고 똑같은 가짜 방을 줍니다.
3. **MMU의 비밀 통역**: 워드가 `10번` 주소에 데이터를 쓰면, 하드웨어 MMU가 몰래 진짜 램의 `8500번` 물리 주소로 번역해서 씁니다. 게임이 똑같이 자기 방의 `10번` 주소에 데이터를 쓰면, MMU는 그걸 진짜 램의 `12000번`으로 번역해서 씁니다.

### 주소 공간 분리 (ASCII 다이어그램)

```text
 [ 프로세스 A (워드) ]               [ 프로세스 B (게임) ]
 가상 주소: 0x0000 ~ 4GB             가상 주소: 0x0000 ~ 4GB
 (자기가 왕인 줄 앎)                  (얘도 자기가 왕인 줄 앎)
          │                                                        │
          ▼                                  ▼
 ┌────── MMU (Page Table A) ─────┐ ┌────── MMU (Page Table B) ─────┐
 │ 가상 10번 ─번역─▶ 물리 8500번 │ │ 가상 10번 ─번역─▶ 물리 12000번│
 └──────────────┬──────────────┘ └──────────────┬──────────────────┘
                │                                                  │
 ┌──────────────▼───────────────────────────────▼──────────────────┐
 │                    [ 물리적 메인 메모리 (RAM) ]                 │
 │   ... [8500번: 워드 데이터] ... [12000번: 게임 데이터] ...      │
 └─────────────────────────────────────────────────────────────────┘
```

> 📢 **섹션 요약 비유**: OS는 방에 들어온 손님마다 완벽한 VR 고글(가상 주소)을 씌워줍니다. 모든 손님은 고글 속에서 "와! 이 넓은 대저택에 나 혼자 있네!"라고 똑같이 착각합니다. 손님들이 물건을 놓으려고 허공에 손을 뻗으면, 직원(MMU)이 몰래 달려와 진짜 창고의 서로 다른 칸에 물건을 차곡차곡 분리해서 넣어줍니다. 손님들은 서로의 존재조차 모릅니다.



## Ⅲ. 분리의 위대함과 예외 (공유 메모리)

이 가상 공간 분리(Isolation) 덕분에, 어떤 프로그램이 미쳐 날뛰며 아무 주소에나 폭탄(오류)을 던져도 자기 혼자 쓰는 가상 방만 터질 뿐(Segmentation Fault), **옆에 있는 다른 프로그램이나 OS 커널은 1%의 타격도 입지 않고 완벽하게 보호**됩니다.

**의도적인 구멍 뚫기 (Shared Memory)**:
만약 워드와 게임이 데이터를 주고받고 싶다면? 완벽히 분리되어 있어 소통이 불가능합니다.
이때 OS에 특별히 요청하면, OS는 두 프로세스의 가상 주소 중 일부(예: 1페이지)를 물리 메모리의 **'똑같은 한 곳'**으로 번역되도록 MMU 장부를 세팅해 줍니다. 이것이 바로 멀티프로세스 통신의 핵심인 **공유 메모리(IPC, Shared Memory)**입니다.
