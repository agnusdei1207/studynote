# 애플 Secure Enclave Processor (SEP)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Apple A 시리즈 칩에 내장된 전용 보안 코프로세서로 Face ID, Touch ID, 키체인을 보호하는 하드웨어
> 2. **가치**: 생체 인식 데이터 보호, 암호화 키 관리, 안전 부팅, 앱 보안
> 3. **융합**: 보안 엔클레이브, TPM, HSM, 생체 인식과 통합된 Apple 보안 아키텍처

### Ⅰ. 개요 (Context & Background)

**개념 정의**

> **해설**: SEP는 A 시리즈 칩 내부에서 완전히 격리된 보안 코프로세서입니다.

**💡 비유**: SEP는 은행 지점 안에 있는 별도 금고실과 같습니다. 건물 내에 있지만 별도 잠금장치가 있습니다.

**📢 섹션 요약 비유**: SEP는 은행 안 금고실 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **SEP 코어** | 보안 실행 | 전용 ARM 코어 | 금고실 직원 |
| **Secure ROM** | 부트 코드 | 불변 펌웨어 | 설립 문서 |
| **Secure SRAM** | 작업 메모리 | 격리된 RAM | 작업 공간 |
| **키 저장소** | 키 보관 | 하드웨어 fuses | 금고 |

**SEP 보호 대상**

| 보호 대상 | 설명 |
|:---|:---|
| **Face ID 데이터** | 얼굴 인식 수학적 표현 |
| **Touch ID 데이터** | 지문 템플릿 |
| **키체인 키** | 암호화 키, 비밀번호 |
| **Apple Pay 토큰** | 결제 토큰 |
| **안전 부팅 키** | 부트 체인 검증 |

**핵심 알고리즘**: 격리 실행 → 키 보호 → 생체 인식 → 인증

**📢 섹션 요약 비유**: 생체 인식 데이터는 SEP에만 있어요!
