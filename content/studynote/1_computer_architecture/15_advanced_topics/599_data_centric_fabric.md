> **핵심 인사이트**
> 1. 전통적인 폰 노이만 아키텍처(CPU-Centric)는 무조건 메인 CPU가 모든 자원(메모리, 디스크, 네트워크)의 중심에 서서 허락하고 통제하는 제왕적 구조였다.
> 2. 빅데이터와 AI 시대에는 처리해야 할 데이터의 양이 너무 방대해져서, 데이터를 중앙의 CPU로 불러오는 것 자체가 막대한 병목(이동 비용)을 유발하게 되었다.
> 3. 이를 뒤집은 **데이터 중심(Data-centric) 패브릭**은 거대한 데이터(메모리 풀)를 중앙에 두고, CPU, GPU, 스마트 SSD 등 다양한 연산 유닛들이 PCIe/CXL 같은 스위치 그물망(Fabric)을 통해 **데이터가 있는 곳으로 직접 다가가서 연산**하는 민주적인 아키텍처다.



## Ⅰ. CPU 중심주의(Compute-centric)의 붕괴

과거 수십 년간 메인보드의 구조는 철저히 **CPU 중심**이었습니다.
* GPU가 하드디스크의 영상을 읽으려면: `디스크 $\rightarrow$ CPU 메모리 $\rightarrow$ GPU 메모리`
* 랜카드에 들어온 데이터를 암호화 칩에 넘기려면: `랜카드 $\rightarrow$ CPU 메모리 $\rightarrow$ 암호화 칩`

모든 길은 로마(CPU)로 통했습니다. 하지만 데이터가 1초에 테라바이트(TB)씩 쏟아지는 환경에서는, 이 'CPU를 거쳐 가는 통행세' 때문에 CPU가 과부하에 걸려 다른 일을 전혀 못 하고 데이터 복사 셔틀로 전락해 버렸습니다.

> 📢 **섹션 요약 비유**: 회사에서 모든 부서의 문서가 반드시 사장님(CPU) 책상을 거쳐서 다른 부서로 전달되어야 하는 꽉 막힌 관료제입니다. 서류가 너무 많아지자 사장님은 하루 종일 서류 결재만 하느라 회사가 망해갑니다.



## Ⅱ. 데이터 중심 패브릭: 데이터가 왕이다

이제 패러다임이 역전되었습니다. **"데이터를 연산기(CPU)로 가져오지 말고, 연산기를 데이터(메모리) 옆으로 보내라!"**

이 거대한 전환을 이루는 물리적 구조가 바로 **데이터 방향성 패브릭(Data-centric Fabric)**입니다.
* 데이터센터의 랙(Rack) 한가운데에 아주 거대한 **공유 메모리(Memory Pool)**와 거대한 스위치 그물망(Fabric)을 놓습니다.
* CPU, GPU, NPU, DPU들은 이 그물망에 동등한 자격으로 연결된 '일개 연산 모듈'로 격하됩니다.
* 만약 거대 메모리에 저장된 이미지를 AI로 돌리고 싶다면, CPU를 부르지 않습니다. GPU가 스위치 그물망을 타고 메모리에 직접 접근하여 자기 할 일만 마치고 결과를 쓱 남겨둡니다.

### 아키텍처 패러다임 변화 (ASCII)

```text
 [ 과거: CPU 중심 (Compute-centric) ]
      GPU      NIC      Storage
       │        │               │
 ┌─────▼────────▼─────────▼─────┐
 │       메인보드의 CPU         │ (모든 데이터 이동의 통제자)
 └──────────────┬───────────────┘
                ▼
            [ 메인 RAM ]

 [ 미래: 데이터 중심 (Data-centric / CXL Fabric) ]
  CPU      GPU      DPU     Smart SSD  (각자 잘하는 연산만 전담)
   │        │        │          │
 ══▼════════▼════════▼═════════▼══ (초고속 CXL 스위치 패브릭)
   │        │        │          │
 [메모리A] [메모리B] [메모리C] [메모리D] (거대한 하나의 메모리 풀로 통합)
```

> 📢 **섹션 요약 비유**: 사장님 중심의 관료제를 폐지하고, 회사 중앙에 거대한 '자료실(Data)'을 만들었습니다. 디자인팀(GPU), 기획팀(CPU), 회계팀(DPU)은 각자 자료실로 바로 가서 자기 분야의 서류에 도장만 찍고 나옵니다. 부서 간의 복잡한 결재 라인이 사라졌습니다.



## Ⅲ. CXL (Compute Express Link): 패브릭의 표준 언어

이 아름다운 민주적 데이터 공유를 실현하기 위해서는 각기 다른 회사에서 만든 칩(인텔 CPU, 엔비디아 GPU, 삼성 메모리)들이 똑같은 언어로 소통해야 합니다.

그 표준어가 바로 **CXL (Compute Express Link)**입니다.
CXL은 기존의 PCIe 버스 물리 규격을 그대로 쓰면서, **"CPU와 장치들이 서로의 메모리를 자신의 램처럼 완벽히 공유(Cache Coherence)"**할 수 있게 해주는 마법의 프로토콜입니다. 

CXL 3.0 스위치 패브릭이 상용화되면, 서버 100대의 메모리를 묶어서 1대의 슈퍼컴퓨터가 1페타바이트(PB)의 RAM을 가진 것처럼 쓸 수 있는 기적(Memory Disaggregation)이 데이터센터의 표준이 될 것입니다.
