# 전력 분석 공격 - DPA (Differential Power Analysis)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 암호화 연산 시 소비되는 전력 패턴을 통계적으로 분석하여 비밀 키를 복구하는 사이드 채널 공격
> 2. **가치**: 스마트카드 보안 평가, 전력 기반 공격 이해, 하드웨어 보안 설계
> 3. **융합**: SPA, EMA, TEMPEST와 통합된 전력 기반 사이드 채널

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DPA(Differential Power Analysis)는 암호화 연산 시 소비되는 전력 패턴을 통계적으로 분석하여 비밀 키를 복구하는 사이드 채널 공격입니다. 1999년 Kocher가 발표했습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DPA (Differential Power Analysis)                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              전력 소비 패턴                                    │ │
│   │                                                              │ │
│   │   전력                                                          │ │
│   │      ▲                                                        │ │
│   │      │   ┌──────────┐                                         │ │
│   │      │   │          │   ┌──┐                                  │ │
│   │      │   │          │   │  │   ┌──────────┐                  │ │
│   │      │   │          │   │  │   │          │                  │ │
│   │      └───┴──────────┴───┴──┴───┴──────────┴────────────→ 시간│ │
│   │          라운드 1      라운드 2      라운드 3                  │ │
│   │                                                              │ │
│   │   전력 변화가 데이터 의존적 (Hamming Weight)                   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DPA 공격 프로세스                                 │ │
│   │                                                              │ │
│   │   1. 전력 트레이스 수집 (Power Trace Collection)              │ │
│   │      - 수천~수만 번 암호화 수행                               │ │
│   │      - 각 실행의 전력 소비 기록                               │ │
│   │                                                              │ │
│   │   2. 가설 함수 계산 (Hypothesis Function)                     │ │
│   │      - 키 후보에 따른 중간값 예측                             │ │
│   │      - Hamming Weight 모델 사용                              │ │
│   │                                                              │ │
│   │   3. 상관 분석 (Correlation Analysis)                         │ │
│   │      - 예측값과 실제 전력의 상관계수 계산                      │ │
│   │      - 상관계수 최대 = 올바른 키                              │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: DPA는 전력 소비와 데이터 간 상관관계를 통계적으로 분석합니다.

**💡 비유**: DPA는 자동차 연료 소비로 속도를 추측하는 것과 같습니다. 가속(연산)할 때 연료(전력)가 많이 듭니다.

**📢 섹션 요약 비유**: DPA는 연료 소비로 속도를 아는 것 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **전력 측정** | 트레이스 수집 | 오실로스코프 | 연료계 |
| **가설 함수** | 예측값 계산 | Hamming Weight | 예측 |
| **상관 분석** | 통계 분석 | Pearson 상관계수 | 비교 |
| **키 후보** | 가능한 키 | 브루트포스 | 후보군 |

**DPA vs SPA 차이**

| 비교 항목 | DPA | SPA |
|:---|:---:|:---:|
| **분석 방식** | 통계적 | 시각적 |
| **필요 샘플** | 수천~수만 | 수십 |
| **노이즈** | 강함 | 약함 |
| **정밀도** | 높음 | 중간 |

**핵심 알고리즘**: 트레이스 수집 → 가설 계산 → 상관 분석 → 키 복구

**📢 섹션 요약 비유**: 많은 데이터를 모아서 통계로 분석해요!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: DPA vs SPA vs EMA**

| 비교 항목 | DPA | SPA | EMA |
|:---|:---:|:---:|:---:|
| **대상** | 전력 | 전력 | 전자기 |
| **방법** | 통계 | 시각 | 통계 |
| **접근** | 접촉 | 접촉 | 비접촉 |
| **정밀도** | 높음 | 중간 | 높음 |

**📢 섹션 요약 비유**: DPA는 접촉, EMA는 비접촉이에요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 스마트카드** → DPA 내성 필수
**시나리오 2: 암호화 칩** → 마스킹, 셔플링 방어
**시나리오 3: 보안 인증** → DPA 평가 필수

**도입 체크리스트**: 전력 노이즈, 마스킹, 연산 셔플링

**📢 섹션 요약 비유**: 마스킹으로 전력 패턴을 숨겨요!
