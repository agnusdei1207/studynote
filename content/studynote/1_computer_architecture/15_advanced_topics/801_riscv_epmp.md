# RISC-V ePMP (Enhanced PMP)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: RISC-V PMP를 확장하여 M-mode 실행도 제어하고 기본 거부 정책을 지원하는 강화된 메모리 보호
> 2. **가치**: M-mode 격리, 실행 권한 제어, 기본 거부, 보안 부팅 지원
> 3. **융합**: PMP, TEE, 보안 부팅, 신뢰 실행 환경과 통합된 강화 메모리 보호

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RISC-V ePMP(Enhanced PMP)는 RISC-V PMP를 확장하여 M-mode 실행도 제어하고 기본 거부 정책을 지원하는 강화된 메모리 보호입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RISC-V ePMP (Enhanced PMP)                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PMP vs ePMP 비교                                  │ │
│   │                                                              │ │
│   │   PMP (기본):                                                  │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │   • M-mode는 PMP 제어 불가                           │   │ │
│   │   │   • M-mode는 모든 메모리 접근 가능                    │   │ │
│   │   │   • 기본 허용 정책                                    │   │ │
│   │   │   • M-mode 실행 제어 불가                             │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   ePMP (확장):                                                 │ │

│   │   │   • M-mode 메모리 접근도 제어                         │   │ │
│   │   │   • 기본 거부(whitelist) 정책                         │   │ │
│   │   │   • M-mode 실행 권한 제어                              │   │ │

│   │   │                                                     │   │ │

│   │                                                              │ │
│   │   After ePMP:                                                 │ │
│   │   M-mode → PMP 룰에 의해 제한됨                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │              메모리                                 │   │ │
```

> **해설**: ePMP는 M-mode도 제어하고 기본 거부 정책으로 보안을 강화합니다.

**💡 비유**: ePMP는 관리자에게도 출입 카드를 검사하는 시스템과 같습니다.

**📢 섹션 요약 비유**: ePMP는 관리자도 검사해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **mseccfg** | 보안 설정 | MMWP, MML, RLB | 정책 설정 |
| **pmpcfg** | 권한 설정 | R/W/X/L + M-mode | 출입증 |
| **기본 거부** | whitelist | 지정만 허용 | 초대장제 |
| **잠금** | 변경 방지 | L 비트, MML | 봉인 |

**ePMP 추가 기능**

| 기능 | 레지스터 | 설명 |
|:---|:---|:---|
| **MMWP** | mseccfg | M-mode 쓰기 보호 |
| **MML** | mseccfg | M-mode 잠금 |
| **RLB** | mseccfg | 룰 잠금 우회 |

**핵심 알고리즘**: mseccfg 설정 → PMP 룰 → M-mode도 체크 → 허용/거부

**📢 섹션 요약 비유**: ePMP는 기본이 거부예요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: ePMP vs PMP vs ARM MPU**

| 비교 항목 | ePMP | PMP | ARM MPU |
|:---|:---:|:---:|:---:|
| **M-mode 제어** | 예 | 아니오 | N/A |
| **기본 정책** | 거부 | 허용 | 설정 |
| **실행 제어** | 예 | 제한 | 예 |
| **복잡도** | 중간 | 낮음 | 높음 |

**📢 섹션 요약 비유**: ePMP가 PMP보다 안전해요!
