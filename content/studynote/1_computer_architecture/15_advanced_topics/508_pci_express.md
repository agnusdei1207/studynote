# 508. PCI Express (PCIe)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컴퓨터 내부에서 CPU, 메모리와 주변기기(GPU, NVMe SSD, NIC 등) 간에 데이터를 병렬이 아닌 **고속 직렬 점대점(Point-to-Point) 방식**으로 주고받는 3세대 범용 입출력(I/O) 버스 표준이다.
> 2. **가치**: 레인(Lane, x1~x16) 개수를 조합하여 대역폭을 자유롭게 조절할 수 있는 확장성을 제공하며, 세대(Gen1~Gen6)를 거듭할 때마다 대역폭이 2배씩 증가하여 현대 컴퓨터의 근간 스켈레톤(Skeleton) 역할을 한다.
> 3. **융합**: 단순히 데이터만 나르는 것을 넘어, SR-IOV를 통한 가상화 가속, CXL(Compute Express Link)의 물리 계층(PHY)으로 차용되는 등 데이터센터 인터커넥트 아키텍처의 절대적인 토대다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: PCI Express(Peripheral Component Interconnect Express)는 2003년 인텔 주도로 발표된 직렬 컴퓨터 확장 버스 표준으로, 기존의 병렬 버스 구조인 PCI와 PCI-X를 대체하기 위해 개발되었다. 스위치(Switch) 기반의 스타 토폴로지(Star Topology)를 사용한다.
- **💡 비유**: 옛날 PCI 버스가 "1차선 왕복 도로에 모든 동네 사람(여러 기기)이 끼어들어 눈치 보며 다니는 동네 길(공유 버스)"이었다면, PCIe는 "집집마다 톨게이트(스위치)까지 직통으로 뚫린 나만의 전용 고속도로(Point-to-Point)"다. 돈이 많으면 차선을 16개(x16) 깔고, 적으면 1개(x1)만 깔 수 있는 매우 합리적이고 유연한 시스템이다.
- **등장 배경 및 발전 과정**:
  1. **병렬 버스(Parallel Bus)의 한계**: 기존 PCI는 32개 또는 64개의 선(비트)을 나란히 배열하여 한 번에 보냈다. 그러나 속도(클럭)가 높아지자, 64개의 선을 달리는 전기 신호들이 목적지에 도착하는 타이밍이 미세하게 어긋나는 **클럭 스큐(Clock Skew)** 현상과 선끼리의 **신호 간섭(Crosstalk)**이 심해져 더 이상 속도를 올릴 수 없었다.
  2. **직렬 버스(Serial Bus)로의 패러다임 전환**: 선 여러 개를 천천히 보내느니, 차라리 "송신 선 1쌍(TX)과 수신 선 1쌍(RX)"만 남기고 여기에 엄청난 고클럭으로 데이터를 직렬로 미친 듯이 쏴버리는(SerDes) 패러다임으로 전환했다. 
  3. **PCIe 세대의 진화**: Gen1(2003, 레인당 250MB/s)에서 시작해 현재 상용화된 Gen5(2019, 레인당 4GB/s)를 넘어, 차세대 Gen6/Gen7 규격이 발표되며 3년마다 대역폭을 2배씩 늘리는 극한의 진화를 거듭하고 있다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 / 특징 | 비유 |
|:---|:---|:---|:---|
| **Root Complex (RC)** | CPU/메모리와 PCIe 트리 구조의 꼭대기를 연결하는 두뇌 | 프로세서의 요청을 PCIe 패킷(TLP)으로 변환, 스위치로 전달 | 고속도로 최상위 IC (인터체인지) |
| **PCIe Switch (스위치)** | 부족한 PCIe 레인을 여러 기기로 분배(Fan-out) | 버스 공유 없이 독립적인 패킷 스위칭(Point-to-Point) 보장 | 고속도로 분기점(JC) 분배기 |
| **Endpoint (EP)** | PCIe 트리의 끝단에 달린 실제 장치 | GPU, NVMe SSD, 네트워크 카드(NIC) 등 | 고속도로 끝의 공장이나 집 |
| **Lane (레인)** | 데이터를 나르는 최소 단위의 물리적 통로 | 1개의 레인은 2쌍의 차동 신호선(TX 2가닥, RX 2가닥)으로 구성 | 고속도로 1차선 (상/하행선) |
| **TLP (Transaction Layer Packet)** | PCIe 계층에서 데이터를 주고받는 실제 우편물 상자 | 헤더(목적지 주소) + 페이로드(데이터) + 꼬리(에러 검증)로 구성 | 컨테이너 박스 |

### 공유 버스(PCI) vs 직렬 스위치(PCIe) 토폴로지 비교

PCIe의 가장 큰 혁신은 '버스(Bus)'라는 이름을 쓰면서도 실제로는 네트워크의 라우터처럼 '스위치' 구조를 가진다는 점이다.

```text
  ┌─────────────────────────────────────────────────────────────┐
  │              구형 PCI 버스 vs 최신 PCIe 스위치 아키텍처             │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ [1] Legacy PCI (공유 버스, Shared Bus)                        │
  │                                                             │
  │      ┌───────┐                                              │
  │      │  CPU  │                                              │
  │      └───┬───┘                                              │
  │   ═══════│══════════════════════════════════ [병렬 버스 32bit] │
  │          │             │              │                     │
  │      ┌───┴───┐     ┌───┴───┐      ┌───┴───┐                 │
  │      │ 사운드 │     │ 랜카드 │      │ 캡처보드 │                 │
  │      └───────┘     └───────┘      └───────┘                 │
  │   * 단점: 대역폭(133MB/s)을 3기기가 쪼개 쓴다. 한 기기가 말할 때        │
  │           다른 기기는 기다려야 한다 (충돌 발생).                   │
  │                                                             │
  │ ─────────────────────────────────────────────────────────── │
  │                                                             │
  │ [2] PCIe (스위치 기반 Point-to-Point)                        │
  │                                                             │
  │                 ┌───────────────┐                             │
  │                 │ CPU (Root Cplx)│                             │
  │                 └──┬───────────┬──┘                             │
  │    전용차선 (x16) │           │ 전용차선 (x4)                    │
  │                 ▼           ▼                             │
  │            ┌───────┐     ┌────────┐                           │
  │            │  GPU  │     │ Switch │                           │
  │            └───────┘     └──┬──┬──┘                           │
  │                   전용 (x4) │  │ 전용 (x4)                     │
  │                             ▼  ▼                             │
  │                       ┌────┐ ┌────┐                         │
  │                       │NVMe│ │ NIC│                         │
  │                       └────┘ └────┘                         │
  │   * 장점: 각 기기는 스위치와 "나만의 전용 차선"으로 직결됨.              │
  │           남들 눈치 보지 않고 100% 대역폭으로 쉼 없이 통신 가능.        │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 상단의 구형 PCI는 하나의 굵은 파이프에 여러 장치가 빨대를 꽂고 있는 형태다. 그래서 랜카드가 대용량 파일을 다운받을 때 사운드 카드의 통신이 밀려 소리가 뚝뚝 끊기는 일이 흔했다. 
하단의 **PCIe**는 완전히 다르다. CPU 내부에 있는 **Root Complex(RC)**에서 출발한 차선(Lane)들은 특정 장치(GPU 등)로 1:1로 직접 꽂히거나, 중간에 분배기인 **Switch**를 거쳐 각 장치(NVMe, NIC)로 배분된다. NVMe SSD가 데이터를 꽉 채워 보내더라도, 그것은 스위치와 NVMe 사이의 '전용 도로'를 쓸 뿐이므로 옆에 꽂힌 NIC의 속도에 전혀 지장을 주지 않는다.

### 심층 동작 원리 (차동 신호 방식, Differential Signaling)
PCIe가 1개의 레인당 수 기가헤르츠(GHz)의 초고속 클럭을 달성할 수 있는 물리적 비결이다.
* **노이즈 상쇄**: 데이터를 보낼 때 전선 1가닥으로 보내면 외부 전자파(노이즈)에 쉽게 오염된다. PCIe는 데이터를 보낼 때 **원래 신호(D+)**와 그 파형을 정확히 **뒤집은 반전 신호(D-)**를 두 가닥의 선으로 동시에 보낸다. 
* 도착지에서 (D+) - (D-)의 차이값만 읽어낸다. 만약 중간에 외부 노이즈가 양쪽 선에 똑같이(+N) 끼어들더라도, 수식으로 `(D+ + N) - (D- + N) = D+ - D-` 가 되어 노이즈(N)가 완벽히 상쇄된다. 이 차동 신호 기술 덕분에 엄청난 고속 직렬 전송이 가능하다.

## Ⅲ. 융합 비교 및 다각도 분석

### PCIe 세대별 대역폭 발전 (레인당 단방향 속도 기준)

| 세대 (버전) | 출시 연도 | 인코딩 방식 | x1 레인 대역폭 (단방향) | x16 레인 대역폭 (GPU 등) | 주요 도입 기기 |
|:---|:---|:---|:---|:---|:---|
| **Gen 3.0** | 2010년 | 128b/130b | 1 GB/s 미만 (985MB/s) | 약 16 GB/s | NVMe SSD의 초기 태동 |
| **Gen 4.0** | 2017년 | 128b/130b | 약 2 GB/s (1.97GB/s) | 약 32 GB/s | PS5, 최신 데스크탑 SSD |
| **Gen 5.0** | 2019년 | 128b/130b | **약 4 GB/s (3.94GB/s)** | **약 64 GB/s** | 데이터센터 AI 가속기 (H100) |
| **Gen 6.0** | 2022년 | **PAM4** / FEC | 약 8 GB/s (7.88GB/s) | 약 128 GB/s | 차세대 CXL 3.0 인프라 기반 |

> *참고: Gen 6.0부터는 0과 1만 구분하던 NRZ 방식을 버리고, 한 번의 신호로 00, 01, 10, 11 네 가지 상태를 표현하는 **PAM4(4진 펄스 진폭 변조)** 기술을 도입하여 클럭을 올리지 않고도 대역폭을 2배로 늘렸다.*

### 과목 융합 관점
- **클라우드/가상화 (SR-IOV)**: 하나의 물리적인 100Gbps PCIe 랜카드(NIC)를 수십 대의 가상머신(VM)에 쪼개어 할당하고 싶을 때, 하이퍼바이저가 소프트웨어로 쪼개면 I/O 세금(Tax)이 막대하다. PCIe 표준인 **SR-IOV (Single Root I/O Virtualization)** 기술을 쓰면, 물리적인 PCIe 카드 1개(PF, Physical Function)가 자신을 가짜 PCIe 기기 수백 개(VF, Virtual Function)인 것처럼 복제하여 VM들에게 직접 꽂아줄 수 있다(Kernel Bypass).
- **스토리지 (NVMe의 탄생)**: 과거 SATA 방식의 SSD는 디스크 컨트롤러(AHCI)를 거쳐야 해서 느렸다. 반면 NVMe(Non-Volatile Memory Express)는 이름 그대로 낸드 플래시 메모리 칩을 **PCIe 버스에 직결(Express)**하여 CPU와 직접 수만 개의 큐(Queue)로 대화하게 만든 아키텍처 혁명이다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 데이터센터 AI 훈련 노드의 PCIe 병목 (GPU Starvation)**: 1대의 서버에 NVIDIA GPU 8장을 꽂았다. CPU가 학습할 이미지 데이터를 GPU로 계속 넘겨줘야 하는데, PCIe Gen4 x16의 대역폭(32GB/s)이 너무 좁아 GPU가 계산을 다 끝내고도 다음 데이터를 받지 못해 놀고 있는 현상 발생.
   - **해결책**: 서버 아키텍처를 교체한다. 1) CPU-GPU 간 통신은 PCIe Gen5(64GB/s)를 지원하는 최신 플랫폼으로 전환하고, 2) GPU 간 통신은 좁고 CPU를 거쳐야 하는 PCIe 버스 대신, GPU끼리 다이렉트로 거미줄처럼 연결된 **NVLink (900GB/s 이상)**를 사용하여 PCIe 병목을 우회하는 위상(Topology)을 구축해야 한다.
2. **시나리오 — M.2 SSD 장착 시 그래픽카드 속도 저하 (Lane 공유 이슈)**: 일반 PC에 고사양 PCIe M.2 SSD 3개를 꽂았더니, 갑자기 그래픽카드(GPU)의 대역폭이 x16에서 x8 모드로 반토막 나는 현상.
   - **원인 분석**: 메인스트림 데스크탑 CPU(예: 일반 Core i7)가 내부적으로 제공하는 PCIe 레인의 총합(보통 20~24개)이 부족하기 때문이다. GPU가 16개를 먹고 나면 남은 레인이 4~8개뿐인데, 여기에 SSD 3개(각 4레인 요구 = 12레인)를 꽂아버리면 칩셋/BIOS가 강제로 GPU의 차선을 빼앗아 SSD에 나눠준다. 이럴 때는 비싸더라도 HEDT(스레드리퍼, 제온-W)처럼 PCIe 레인이 64~128개씩 넉넉한 CPU 폼팩터를 도입해야 한다.

### 도입 체크리스트
- **물리적 호환성 (상하위 호환)**: PCIe는 놀랍도록 완벽한 하위 호환성을 제공한다. Gen5 슬롯에 Gen3 카드를 꽂거나, x16 크기의 슬롯에 짧은 x4 크기의 카드를 꽂아도 핀의 앞부분만 사용하여 정상 작동한다. 단, 통신 속도는 '둘 중 더 낮은 스펙'으로 하향 평준화(Negotiation)됨을 시스템 아키텍트가 숙지해야 한다.
- **신호 무결성 (Signal Integrity)**: Gen5(32GT/s) 이상의 초고주파 시대가 되면서, 마더보드의 구리 배선 길이가 10~15cm만 넘어가도 신호가 깎여나가 인식이 안 되는 문제가 심각하다. 따라서 메인보드 중간에 신호를 증폭시켜 주는 **리타이머(Retimer)**나 리드라이버(Redriver) 칩이 꼼꼼히 설계되어 있는지, 혹은 구리선 대신 Twinax 케이블을 썼는지 확인해야 한다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 레거시 병렬 버스 (PCI) | 최신 직렬 버스 (PCIe Gen5) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (대역폭)** | 133 MB/s (전체 공유) | 64,000 MB/s (x16 1기기 단독) | 기기당 대역폭 **수백 배 향상** |
| **정성 (핀 수/복잡도)** | 32/64핀의 복잡한 라우팅 길이 맞춤 필요 | x1 기준 4핀(TX/RX 쌍)의 단순한 라우팅 | 메인보드 설계 난이도 대폭 하락 및 소형화 가능 |
| **정성 (토폴로지)** | 버스 마스터링 경합(Contention) 발생 | 스위치 기반의 독립된 패킷 라우팅 | 다수 기기 연결 시에도 지연 시간(Jitter) 일정 유지 |

### 미래 전망
- **광학 PCIe (Optical PCIe)로의 전환**: PCIe Gen6/Gen7(최대 256GB/s)으로 속도가 올라가면, 구리선을 통한 신호 감쇠가 물리적 한계에 부딪힌다. 향후에는 PCIe 슬롯 끝에 광 트랜시버(Silicon Photonics)를 달아 구리 대신 '빛'으로 데이터를 전송하는 아키텍처가 데이터센터 스토리지 랙을 중심으로 대중화될 것이다.
- **CXL(Compute Express Link)의 근간**: 현재 IT 업계의 가장 뜨거운 감자인 CXL 메모리 풀링 기술은, 아예 바닥부터 새로 만든 것이 아니라 이 **PCIe Gen5의 물리 계층(PHY) 핀과 스위치**를 그대로 가져다 쓴다. 즉, PCIe 슬롯에 그래픽카드 대신 CXL D램 모듈을 꽂아 메모리를 테라바이트급으로 확장하는 시대의 '물리적 인프라' 역할을 PCIe가 수행하고 있다.

### 참고 표준
- **PCI-SIG (PCI Special Interest Group)**: 인텔, AMD, 엔비디아 등 글로벌 IT 기업들이 모여 PCIe 1.0부터 7.0까지의 모든 전기적, 물리적 규격과 프로토콜을 제정하고 테스트하는 표준화 기구.

## 📌 관련 개념 맵 (Knowledge Graph)
- **[시스템 버스 및 내부 버스](../9_system_bus_interconnects/xxx_system_bus.md)**: CPU 내부나 외곽을 연결하는 고속도로의 총칭. 현대 시스템에서 노스브릿지/사우스브릿지의 모든 기기 연결은 PCIe로 통일되었다.
- **[NVMe (Non-Volatile Memory Express)](../8_io_storage_systems/336_nvme.md)**: 느린 SATA 케이블을 버리고, 초고속 PCIe 차선 위에 SSD를 바로 올려놓아 탄생한 혁명적인 스토리지 프로토콜.
- **[CXL (Compute Express Link)](./470_cxl.md)**: PCIe 슬롯을 활용하여, 메모리나 가속기들이 CPU와 서로의 캐시를 직접(Coherent) 공유할 수 있게 만든 차세대 인터커넥트.
- **[서멀 스로틀링 (Thermal Throttling)](./473_thermal_throttling.md)**: PCIe Gen5 NVMe SSD는 속도가 빠른 만큼 컨트롤러에서 막대한 열이 발생하므로, 두꺼운 방열판이 없으면 10초 만에 스로틀링이 걸려 PCIe Gen3 수준으로 속도가 폭락한다.

## 👶 어린이를 위한 3줄 비유 설명
1. 옛날 컴퓨터 부품들은 1차선 좁은 흙길(PCI) 하나를 다 같이 써서, 누가 짐을 옮기고 있으면 다른 애들은 한참을 기다려야 했어요.
2. 하지만 **PCIe**는 집집마다 톨게이트까지 나만의 '직통 고속도로'를 뻥 뚫어준 거랑 같아요. 중간에 누구랑 부딪힐 일도 없고 눈치 안 보고 전속력으로 달릴 수 있죠!
3. 차(데이터)가 많이 필요한 곳(그래픽카드)에는 16차선을 깔아주고, 조금 필요한 곳(사운드카드)에는 1차선만 깔아주는 아주 합리적이고 똑똑한 도로망이랍니다.
