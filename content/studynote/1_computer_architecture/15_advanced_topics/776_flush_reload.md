# Flush+Reload 기법

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공유 메모리 영역에서 특정 캐시 라인을 플러시하고 피해자 실행 후 리로드 시간을 측정하여 접근 여부를 확인하는 정밀 사이드 채널 공격
> 2. **가치**: 높은 정밀도, 노이즈 저항, 암호화 키 복구, 스펙터/멜트다운 기반 공격
> 3. **융합**: Prime+Probe, 캐시 타이밍 공격, 스펙터와 통합된 고정밀 사이드 채널

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Flush+Reload 기법은 공유 메모리 영역에서 특정 캐시 라인을 플러시하고 피해자 실행 후 리로드 시간을 측정하여 접근 여부를 확인하는 정밀 사이드 채널 공격입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Flush+Reload 기법                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Flush+Reload 공격 단계                           │ │
│   │                                                              │ │
│   │   단계 1: Flush (내보내기)                                     │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                  Shared Memory                       │   │ │
│   │   │                                                     │   │ │
│   │   │   ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐         │   │ │
│   │   │   │ Line│ │ Line│ │ Line│ │ Line│ │ Line│ ...     │   │ │
│   │   │   │  0  │ │  1  │ │  2  │ │  3  │ │  4  │         │   │ │
│   │   │   └──┬──┘ └─────┘ └─────┘ └─────┘ └─────┘         │   │ │
│   │   │      │                                              │   │ │
│   │   │      ▼  CLFLUSH 명령어로 플러시                      │   │ │
│   │   │   ┌─────┐                                           │   │ │
│   │   │   │Empty│ ← 캐시에서 제거됨                          │   │ │
│   │   │   └─────┘                                           │   │ │
│   │   │                                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   단계 2: 피해자 실행                                          │ │
│   │   - 피해자가 공유 메모리 특정 라인 접근                        │ │
│   │   - 접근한 라인은 캐시에 로드됨                                │ │
│   │                                                              │ │
│   │   단계 3: Reload (다시 읽기)                                   │ │
│   │   - 공격자가 같은 라인 접근                                    │ │
│   │   - 빠름(히트): 피해자가 접근함                                │ │
│   │   - 느림(미스): 피해자가 접근 안 함                            │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: Flush+Reload는 CLFLUSH로 캐시를 비우고 리로드 시간으로 접근을 확인합니다.

**💡 비유**: Flush+Reload는 표시된 책을 확인하는 것과 같습니다. 책이 움직였으면 누가 읽었습니다.

**📢 섹션 요약 비유**: Flush+Reload는 표시된 책 확인 같아요. 움직였으면 읽었어요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **CLFLUSH** | 캐시 플러시 | 캐시 라인 무효화 | 책 치우기 |
| **공유 메모리** | 공유 영역 | mmap, shared lib | 공유 책장 |
| **Reload** | 시간 측정 | 메모리 접근 | 책 확인 |
| **타이밍** | 히트/미스 판단 | RDTSC | 시간 측정 |

**Flush+Reload 세부 단계**

| 단계 | 동작 | 명령어 | 목적 |
|:---|:---|:---|:---|
| **1. Flush** | 캐시 라인 제거 | CLFLUSH | 기준 상태 |
| **2. 대기** | 피해자 실행 | - | 공격 윈도우 |
| **3. Reload** | 시간 측정 | MOV + RDTSC | 접근 확인 |
| **4. 분석** | 패턴 유추 | 통계 분석 | 비밀 유출 |

**핵심 알고리즘**: 플러시 → 피해자 실행 → 리로드 시간 측정 → 정보 유추

**📢 섹션 요약 비유**: 비우고 기다리고 읽고 분석해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: Flush+Reload vs Prime+Probe vs Evict+Time**

| 비교 항목 | Flush+Reload | Prime+Probe | Evict+Time |
|:---|:---:|:---:|:---:|
| **공유 메모리** | 필요 | 불필요 | 불필요 |
| **정밀도** | 매우 높음 | 중간 | 낮음 |
| **노이즈** | 낮음 | 높음 | 높음 |
| **스펙터 호환** | 높음 | 중간 | 낮음 |

**📢 섹션 요약 비유**: Flush+Reload가 가장 정밀해요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 스펙터 공격** → Flush+Reload로 유출
**시나리오 2: AES 키 복구** → S-Box 접근 패턴 분석
**시나리오 3: 방어** → CLFLUSH 제한, 공유 메모리 격리

**도입 체크리스트**: 공유 메모리 식별, 타이밍 정밀도, 노이즈 처리

**📢 섹션 요약 비유**: 공유 메모리가 있으면 Flush+Reload를 써요!
