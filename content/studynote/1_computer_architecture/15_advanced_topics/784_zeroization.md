# 제로화 (Zeroization) 회로

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공격 감지 또는 명령 시 보안 데이터(키, 비밀)를 즉시 완전 삭제하는 하드웨어 회로
> 2. **가치**: 키 유출 방지, 인증 요건, 물리적 보안, FIPS 140 준수
> 3. **융합**: 안티 탬퍼, 보안 키 소거, HSM, TPM과 통합된 데이터 파괴

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

제로화(Zeroization) 회로는 공격 감지 또는 명령 시 보안 데이터(키, 비밀)를 즉시 완전 삭제하는 하드웨어 회로입니다.

> **해설**: 제로화는 다양한 트리거로 즉시 키를 0으로 만듭니다.

**💡 비유**: 제로화는 비상 자폭 버튼과 같습니다. 누르면 모든 비밀이 사라집니다.

**📢 섹션 요약 비유**: 제로화는 비상 버튼 같아요. 누르면 다 사라져요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **트리거** | 시작 신호 | 다양한 소스 | 스위치 |
| **제어 회로** | 소거 제어 | FSM | 컨트롤러 |
| **메모리** | 키 저장 | SRAM/Flash | 금고 |
| **소거 로직** | 데이터 삭제 | 0 쓰기 | 소각 |

**제로화 트리거**

| 트리거 | 소스 | 반응 시간 |
|:---|:---|:---:|
| **Tamper** | 안티 탬퍼 | <1μs |
| **Power Loss** | 전원 센서 | <100μs |
| **SW Command** | 소프트웨어 | <1ms |
| **Env** | 온도/전압 | <1ms |

**핵심 알고리즘**: 트리거 감지 → 메모리 소거 → 확인 → 잠금

**📢 섹션 요약 비유**: 트리거가 오면 바로 지워요!
