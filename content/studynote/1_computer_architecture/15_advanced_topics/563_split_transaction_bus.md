> **핵심 인사이트**
> 1. 초기의 단순한 버스 구조는 장치 A가 데이터를 요청하면, 메모리가 데이터를 찾아 응답할 때까지 **버스를 꽉 붙잡고 놓아주지 않아(Bus Lock)** 다른 장치들이 버스를 전혀 쓰지 못하는 심각한 병목 현상이 있었다.
> 2. **분리 트랜잭션 버스 (Split Transaction Bus)**는 하나의 통신 과정을 **'요청(Request)'과 '응답(Reply)'이라는 두 개의 독립된 사건으로 완전히 분리**하는 기술이다.
> 3. CPU가 요청만 쏙 던져놓고 버스를 즉시 놓아주기 때문에, 메모리가 데이터를 찾는 기나긴 지연 시간(Latency) 동안 다른 수많은 기기(GPU, 랜카드 등)가 버스를 사용할 수 있어 시스템 전체의 처리량(Throughput)이 비약적으로 상승한다.

<br>

## Ⅰ. 버스 점유의 비극 (Locked Bus)

CPU는 5GHz로 엄청나게 빠르지만, 메인 메모리(DRAM)는 데이터를 찾아서 꺼내오기까지 무려 100나노초(CPU 기준 500클럭)라는 엄청난 시간이 걸립니다.

만약 버스가 분리형이 아니라 **단일 트랜잭션(원자적 버스)**이라면 대참사가 일어납니다.
1. CPU가 "메모리야, A 데이터 줘"라며 주소 버스를 켭니다.
2. 메모리가 창고(DRAM 셀)로 뛰어가서 데이터를 찾는 **500클럭 동안, CPU는 물론이고 이 버스에 물려있는 다른 모든 장치(그래픽카드, 하드디스크)가 버스를 쓰지 못하고 멈춰 섭니다.**
3. 500클럭 뒤 메모리가 데이터를 건네주면 비로소 버스가 해제됩니다.

버스는 도로인데, 짐 싣는 시간 동안 도로 한가운데 차를 세워두고 모든 차를 500클럭 동안 막아버리는 꼴입니다.

> 📢 **섹션 요약 비유**: 드라이브스루 햄버거집에서, 앞차가 "햄버거 주세요"라고 주문한 뒤, 햄버거가 다 만들어져서 나올 때까지 주문 창구 앞에 차를 그대로 세워두고 버티는 진상 손님(Bus Lock)과 같습니다. 뒷사람은 주문조차 못 합니다.

<br>

## Ⅱ. 분리 트랜잭션 (요청과 응답의 분리)

**분리 트랜잭션 버스(Split Transaction Bus)**는 이 꽉 막힌 도로를 뚫기 위해 도입되었습니다. 한 번의 거래를 '요청(Request)'과 '응답(Reply)'으로 쪼개고, 그 사이의 시간 동안 남들이 도로를 쓰게 해줍니다.

**동작 원리**:
1. **요청 발송**: CPU가 버스에 주소를 띄워 "A 데이터 줘!"라고 요청합니다. (이때 각 요청에 꼬리표(Tag 번호)를 달아 보냅니다.)
2. **버스 해제 (핵심!)**: CPU는 요청을 보낸 즉시 **버스를 놓아줍니다(Release).**
3. **타 장치 버스 사용**: 메모리가 A 데이터를 찾는 수백 클럭 동안, 그래픽카드나 랜카드가 버스를 마음껏 사용합니다.
4. **응답 수신**: 메모리가 드디어 A 데이터를 다 찾았습니다. 그러면 메모리 측에서 역으로 버스 사용 권한을 얻어내어(Bus Master가 됨), 데이터와 함께 "아까 꼬리표 번호에 대한 응답이야!"라고 버스에 쏴줍니다.

### 파이프라인 버스 (ASCII 다이어그램)

```text
 [ 단일 트랜잭션 (비효율) ]
 CPU 요청 ─▶ (.......... 500클럭 도로 통제 ..........) ─▶ 응답 수신
 
 [ 분리 트랜잭션 (효율 극대화) ]
 CPU_1 요청 ─▶ 버스 즉시 해제!
                CPU_2 요청 ─▶ 버스 즉시 해제!
                               GPU 요청 ─▶ 버스 즉시 해제!
                                            (메모리가 데이터 찾음) ─▶ CPU_1에 응답 발송!
```
이렇게 분리 트랜잭션을 쓰면 버스 위에 여러 개의 요청과 응답이 파이프라인처럼 동시에 돌아다닐 수 있습니다(Pipelined Bus).

> 📢 **섹션 요약 비유**: 진동벨(Tag) 시스템입니다. 주문 창구에서 "햄버거 주세요"하고 주문(요청)을 넣으면 즉시 진동벨을 받고 옆으로 빠져줍니다(버스 해제). 햄버거가 만들어지는 동안 다른 수십 명의 손님이 주문을 넣을 수 있고, 햄버거가 다 완성되면 주방이 진동벨을 울려(응답) 손님에게 건네줍니다.

<br>

## Ⅲ. 트레이드오프와 적용

버스를 극도로 효율적으로 쓰게 해 주지만, 공짜는 아닙니다.

* **복잡한 버스 중재기(Arbiter)**: 버스 통제권이 수시로 바뀌어야 하고, 메모리가 응답을 보낼 때 다시 버스를 점유하기 위한 중재 요청 로직이 필요해 컨트롤러 설계가 매우 복잡해집니다.
* **응답 꼬리표 추적 (Tagging)**: 응답이 도착했을 때 이게 아까 CPU 1이 부른 건지 CPU 2가 부른 건지 매칭해 주는 태그 메모리가 필요합니다.

현대의 **PCIe (PCI Express)**, ARM의 **AXI 프로토콜** 등 모든 고성능 시스템/네트워크 버스는 100% 분리 트랜잭션 방식을 사용하여, 수만 개의 패킷이 버스 위를 동시에 날아다니는(In-flight) 마법을 부리고 있습니다.
