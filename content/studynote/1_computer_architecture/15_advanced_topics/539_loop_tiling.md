> **핵심 인사이트**
> 1. AI나 과학 연산에서 사용되는 행렬(Matrix)의 크기는 수 메가바이트(MB)를 넘어가므로, CPU의 작은 **L1 데이터 캐시(수십 KB) 용량을 단번에 초과**해버린다.
> 2. 이렇게 큰 데이터를 단순한 루프로 연산하면, 방금 썼던 데이터가 캐시 밖으로 밀려나고 나중에 다시 메모리에서 가져와야 하는 **캐시 미스(Cache Miss)의 지옥**에 빠지게 된다.
> 3. **루프 타일링(Loop Tiling)**은 전체 행렬을 한 번에 계산하지 않고, 캐시 메모리 크기에 딱 맞는 '작은 사각형(타일, Block)' 단위로 쪼갠 뒤, 타일 단위로 연산을 끝내어 캐시 적중률(Hit rate)을 극한으로 끌어올리는 기법이다.

<br>

## Ⅰ. 행렬 곱셈의 캐시 파괴(Thrashing) 문제

행렬 A와 행렬 B를 곱해서 C를 만드는 코드를 살펴봅시다. (배열 크기 $N = 1000$)
```c
for (int i = 0; i < N; i++) {
    for (int j = 0; j < N; j++) {
        for (int k = 0; k < N; k++) {
            C[i][j] += A[i][k] * B[k][j];
        }
    }
}
```
여기서 `B[k][j]`의 접근 패턴을 보면, C언어는 행 우선(Row-major)으로 메모리에 저장되는데, $k$가 증가할 때마다 B 행렬을 **세로 방향으로 징검다리 뛰듯 접근**합니다. 

메모리 주소가 $1 \rightarrow 1001 \rightarrow 2001$로 엄청나게 널뛰기 때문에 공간적 지역성이 박살 납니다. L1 캐시가 감당하지 못해 이전에 가져온 B의 데이터들을 계속 쫓아내고(Evict), 다시 쓸 때 또 메모리에서 가져오는 최악의 캐시 낭비(Thrashing)가 일어납니다.

> 📢 **섹션 요약 비유**: 도서관에서 책을 보는데, 1권의 1페이지 $\rightarrow$ 2권의 1페이지 $\rightarrow$ 3권의 1페이지 순으로 책을 읽습니다. 내 책상(캐시)은 3권밖에 못 놓는데, 책을 1,000권째 열어봐야 하니 매번 책을 서가(메모리)에 갖다 꽂고 새 책을 꺼내오는 끔찍한 막노동을 하는 셈입니다.

## Ⅱ. 루프 타일링: 캐시 크기에 맞춘 조각 내기

**루프 타일링(또는 블로킹)**은 이 거대한 문제를 작은 조각으로 쪼갭니다.
"1000 x 1000 행렬을 한 번에 곱하지 말고, L1 캐시에 쏙 들어가는 **16 x 16 크기의 작은 타일(Tile) 단위로 쪼개서** 그 조각 안에서 덧셈/곱셈을 전부 끝내버리자!"

### 타일링이 적용된 코드의 개념
```c
int B_SIZE = 16; // 캐시에 딱 맞는 타일(블록) 사이즈
for (int ii = 0; ii < N; ii += B_SIZE) {
    for (int jj = 0; j < N; jj += B_SIZE) {
        for (int kk = 0; k < N; kk += B_SIZE) {
            // 이제 이 안에서는 16x16 작은 타일 안에서만 논다!
            for (int i = ii; i < ii + B_SIZE; i++) {
                for (int j = jj; j < jj + B_SIZE; j++) {
                    for (int k = kk; k < kk + B_SIZE; k++) {
                        C[i][j] += A[i][k] * B[k][j];
                    }
                }
            }
        }
    }
}
```

이렇게 하면 한 번 캐시에 올라온 $16 \times 16$ 크기의 B 행렬 조각은, 연산이 완전히 끝날 때까지 밖으로 쫓겨나지 않고 캐시 안에 머물면서 **100%에 가까운 캐시 히트(Hit)**를 달성합니다.

> 📢 **섹션 요약 비유**: 서가에서 1,000권을 미친 듯이 왔다 갔다 하는 대신, 아예 "1권부터 10권"만 책상(캐시)에 쌓아놓고, 그 10권 안에서 뽑아낼 수 있는 정보는 뽕을 뽑을 때까지 다 읽은 뒤에야 다음 10권을 가져오는 영리한 독서법입니다.

<br>

## Ⅲ. 현대 아키텍처에서의 중요성

루프 타일링은 메모리 병목(Memory Wall)을 해결하는 소프트웨어 차원의 가장 강력한 무기 중 하나입니다.
* 인공지능(AI) 훈련이나 이미지 처리, 기상 예측 등 **텐서(Tensor) 연산**이 들어가는 모든 분야에서 필수적으로 사용됩니다.
* 개발자가 6중 for 문을 직접 짜는 것은 사람이 할 짓이 못 되기 때문에, 현대의 똑똑한 컴파일러나 고성능 수학 라이브러리(BLAS, cuBLAS 등)가 현재 돌아가고 있는 **CPU/GPU의 L1 캐시 용량을 스스로 파악하여 최적의 `B_SIZE`를 찾아내고 코드를 타일링으로 자동 변환**해 줍니다.
