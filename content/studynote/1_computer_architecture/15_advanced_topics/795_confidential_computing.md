# Confidential Computing (기밀 컴퓨팅)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용 중인 데이터(in-use)를 하드웨어 기반 격리 환경에서 보호하여 처리 중인 데이터도 암호화 상태 유지
> 2. **가치**: 전체 데이터 수명주기 보호, 클라우드 신뢰성, 규정 준수, 다중 파티 계산
> 3. **융합**: TEE, SGX, SEV-SNP, Nitro Enclaves와 통합된 데이터 보호 패러다임

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Confidential Computing(기밀 컴퓨팅)은 사용 중인 데이터(in-use)를 하드웨어 기반 격리 환경에서 보호하여 처리 중인 데이터도 암호화 상태를 유지하는 기술입니다.

**💡 비유**: 기밀 컴퓨팅은 안전한 방에서만 문서를 여는 것과 같습니다. 방 밖에서는 문서가 암호화되어 있습니다.

**📢 섹션 요약 비유**: 기밀 컴퓨팅은 안전한 방 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **TEE** | 신뢰 실행 환경 | 하드웨어 격리 | 안전한 방 |
| **Attestation** | 환경 증명 | 증명서, PCR | 신분 확인 |
| **봉인 키** | 키 보호 | TEE 바인딩 | 방 열쇠 |
| **보안 모니터** | 격리 관리 | CPU 모드 | 경비원 |

**기밀 컴퓨팅 기술 비교**

| 기술 | Intel | AMD | ARM | AWS |
|:---|:---:|:---:|:---:|:---:|
| **이름** | SGX/TDX | SEV-SNP | CCA | Nitro |
| **격리 단위** | Enclave | VM | Realm | Enclave |
| **메모리** | EPC | 전체 | Realm | 격리 메모리 |
| **크기** | MB~GB | TB | TB | GB |

**핵심 알고리즘**: TEE 생성 → Attestation → 보안 처리 → 봉인

**📢 섹션 요약 비유**: 각사마다 기술이 달라요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 기밀 컴퓨팅 vs 기존 암호화**

| 비교 항목 | 기밀 컴퓨팅 | 기존 암호화 |
|:---|:---:|:---:|
| **In-Use 보호** | 예 | 아니오 |
| **메모리 공격** | 방지 | 취약 |
| **OS 신뢰** | 불필요 | 필요 |
| **성능 오버헤드** | 있음 | 없음 |

**📢 섹션 요약 비유**: 메모리 공격도 막아요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 클라우드** → 다중 테넌트 환경에서 데이터 격리
**시나리오 2: 금융** → 규정 준수, 민감 데이터 처리
**시나리오 3: 의료** → PHI 처리, HIPAA 준수

**도입 체크리스트**: TEE 지원 하드웨어, Attestation, 키 관리

**📢 섹션 요약 비유**: 클라우드에서도 OS를 안 믿어요!
