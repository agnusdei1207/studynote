# FMEA (Failure Mode and Effects Analysis)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템의 잠재적 고장 모드를 체계적으로 식별하고 영향도를 분석하는 신뢰성 엔지니어링 기법
> 2. **가치**: 사전 위험 식별, RPN 우선순위화, 설계 개선, 예방적 유지보수
> 3. **융합**: FTA, RBD, 신뢰성 공학, 품질 관리와 통합된 체계적 위험 분석

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

FMEA(Failure Mode and Effects Analysis)는 시스템의 잠재적 고장 모드를 체계적으로 식별하고 영향도를 분석하는 신뢰성 엔지니어링 기법입니다. RPN(Risk Priority Number)으로 위험도를 우선순위화합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FMEA 분석 프로세스                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FMEA 워크시트 구조                                 │ │
│   │                                                              │ │
│   │   ┌─────────┬─────────┬─────────┬─────────┬─────────┐       │ │
│   │   │ 구성요소 │ 고장모드 │  영향   │  원인   │  RPN    │       │ │
│   │   ├─────────┼─────────┼─────────┼─────────┼─────────┤       │ │
│   │   │  PSU    │ 과전압  │ 시스템  │레귤레이터│  320    │       │ │
│   │   │         │         │  손상   │  고장   │         │       │ │
│   │   ├─────────┼─────────┼─────────┼─────────┼─────────┤       │ │
│   │   │  Fan    │ 정지    │ 과열    │ 베어링  │  240    │       │ │
│   │   │         │         │ Throttle│  마모   │         │       │ │
│   │   └─────────┴─────────┴─────────┴─────────┴─────────┘       │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RPN 계산                                          │ │
│   │                                                              │ │
│   │   RPN = S (심각도) × O (발생빈도) × D (검출가능성)             │ │
│   │                                                              │ │
│   │   S: 1-10 (경미 ~ 치명적)                                    │ │
│   │   O: 1-10 (희박 ~ 빈번)                                      │ │
│   │   D: 1-10 (확실히 검출 ~ 검출 불가)                           │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: FMEA는 각 고장 모드의 심각도, 발생빈도, 검출가능성을 곱하여 RPN을 산출합니다.

**💡 비유**: FMEA는 건강 검진과 같습니다. 잠재적 질병(고장)을 미리 찾아 예방합니다.

**📢 섹션 요약 비유**: FMEA는 건강 검진 같아요. 미리 위험을 찾아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **고장 모드** | 고장 유형 | 어떻게 고장나는가 | 증상 |
| **영향** | 고장 결과 | 무슨 일이 발생하는가 | 결과 |
| **원인** | 고장 원인 | 왜 고장나는가 | 원인 |
| **RPN** | 위험 점수 | S×O×D | 점수 |

**RPN 등급 분류**

| RPN 범위 | 등급 | 조치 |
|:---|:---:|:---|
| **1-100** | 낮음 | 모니터링 |
| **101-200** | 중간 | 개선 권고 |
| **201-300** | 높음 | 즉시 개선 |
| **301+** | 치명적 | 설계 변경 |

**핵심 알고리즘**: 구성요소 식별 → 고장 모드 분석 → RPN 계산 → 우선순위화

**📢 섹션 요약 비유**: RPN이 높으면 먼저 고쳐요. 우선순위예요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FMEA vs FTA**

| 비교 항목 | FMEA | FTA |
|:---|:---:|:---:|
| **방향** | 상향식 | 하향식 |
| **시작점** | 구성요소 | 시스템 고장 |
| **분석** | 다대일 | 일대다 |
| **용도** | 설계 검토 | 사고 분석 |

**📢 섹션 요약 비유**: FMEA는 부품에서, FTA는 시스템에서 시작해요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 신규 설계** → DFMEA (설계 FMEA)
**시나리오 2: 제조 공정** → PFMEA (공정 FMEA)
**시나리오 3: 운영** → FMEA 기반 예방 유지보수

**도입 체크리스트**: 시스템 분해, 전문가 패널, RPN 기준, 개선 추적

**📢 섹션 요약 비유**: FMEA는 전문가와 함께 해요. 다양한 시각이 필요해요!
