# PACMAN 공격 (ARM PAC 우회)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: ARM PAC(Pointer Authentication Code) 보안 기능을 사이드 채널로 우회하여 포인터 변조 공격을 수행하는 기법
> 2. **가치**: ARM 보안 이해, PAC 한계, 완화 조치, 포인터 인증 보안
> 3. **융합**: 사이드 채널 공격, 스펙터, ARM 아키텍처, 메모리 보안과 통합된 공격

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

PACMAN 공격은 ARM PAC(Pointer Authentication Code) 보안 기능을 사이드 채널로 우회하여 포인터 변조 공격을 수행하는 기법입니다. 2022년 MIT에서 공개했습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PACMAN 공격                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ARM PAC (Pointer Authentication Code)             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │              포인터 구조 (PAC 적용)                   │   │ │
│   │   │                                                     │   │ │
│   │   │   ┌─────────────────────────────────────────────┐   │   │ │
│   │   │   │ 63            48 47                       0 │   │   │ │
│   │   │   │ ┌───────────────┬─────────────────────────┐ │   │   │ │
│   │   │   │ │  PAC (16bit)  │   주소 (48bit)          │ │   │   │ │
│   │   │   │ │  인증 코드     │   실제 메모리 주소       │ │   │   │ │
│   │   │   │ └───────────────┴─────────────────────────┘ │   │   │ │
│   │   │   └─────────────────────────────────────────────┘   │   │ │
│   │   │                                                     │   │ │
│   │   │   PAC 생성: PAC = QARMA(주소, 컨텍스트, 키)          │   │ │
│   │   │   PAC 검증: 인증 실패 시 포인터 무효화               │   │ │
│   │   │                                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PACMAN 공격 원리                                  │ │
│   │                                                              │ │
│   │   1. 공격자가 추측한 PAC 값으로 포인터 변조                   │ │
│   │   2. 스펙터 유사 추론 실행으로 PAC 검증 우회 시도             │ │
│   │   3. 캐시 타이밍 측정으로 검증 성공/실패 유추                 │ │
│   │   4. 올바른 PAC 찾을 때까지 브루트포스                        │ │
│   │                                                              │ │
│   │   PAC 검증 실패 → 예외 (측정 불가)                           │ │
│   │   PAC 검증 성공 → 캐시 상태 변화 (측정 가능)                  │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: PACMAN은 스펙터와 유사한 추론 실행으로 PAC 검증 결과를 유출합니다.

**💡 비유**: PACMAN은 자물쇠 번호를 맞추는 것과 같습니다. 틀리면 경고(예외), 맞으면 열립니다.

**📢 섹션 요약 비유**: PACMAN은 자물쇠 번호 맞추기 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **PAC** | 포인터 인증 | QARMA 암호화 | 자물쇠 |
| **PAC 키** | 비밀 키 | 128-bit | 열쇠 |
| **추론 실행** | 사이드 채널 | Speculative | 시도 |
| **타이밍 측정** | 결과 유추 | 캐시 상태 | 확인 |

**PACMAN 특징**

| 항목 | 내용 |
|:---|:---|
| **대상** | ARMv8.3+ PAC |
| **방법** | 스펙터 + 브루트포스 |
| **필요 시도** | ~5.3만 회 (16-bit PAC) |
| **영향** | PAC 우회, 코드 실행 |

**핵심 알고리즘**: PAC 추측 → 추론 실행 → 타이밍 측정 → 결과 확인 → 반복

**📢 섹션 요약 비유**: PAC는 16비트라서 6만 번 안에 맞춰요!

**기술 비교: PACMAN vs Spectre vs PAC 우회**

| 비교 항목 | PACMAN | Spectre | 일반 PAC 우회 |
