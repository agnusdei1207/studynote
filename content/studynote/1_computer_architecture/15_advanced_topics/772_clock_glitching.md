# 클럭 글리칭 (Clock Glitching)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 클럭 신호에 순간적인 변화(주파수/듀티 사이클)를 가해 명령어 실행 오류를 유발하는 물리적 공격
> 2. **가치**: 하드웨어 보안 이해, 클럭 기반 공격, 방어 설계, 보안 평가
> 3. **융합**: 볼티지 글리칭, EMFI, FA(Fault Attack)와 통합된 물리적 공격

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

클럭 글리칭(Clock Glitching)은 CPU 클럭 신호에 순간적인 변화(주파수/듀티 사이클)를 가해 명령어 실행 오류를 유발하는 물리적 공격입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    클럭 글리칭 (Clock Glitching)                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              정상 클럭 vs 글리치 클럭                          │ │
│   │                                                              │ │
│   │   정상 클럭:                                                   │ │
│   │      ▲   ┌──┐   ┌──┐   ┌──┐   ┌──┐   ┌──┐                 │ │
│   │      │   │  │   │  │   │  │   │  │   │  │                 │ │
│   │      └───┘  └───┘  └───┘  └───┘  └───┘  └───┴────→ 시간     │ │
│   │          T=1    T=2    T=3    T=4    T=5                    │ │
│   │                                                              │ │
│   │   글리치 클럭 (주파수 증가):                                   │ │
│   │      ▲   ┌┐ ┌┐ ┌┐ ┌──┐   ┌──┐   ┌──┐                     │ │
│   │      │   │└─┘└─┘└─┘  │   │  │   │  │                     │ │
│   │      └───┘            └───┘  └───┘  └───┴────→ 시간         │ │
│   │          ↑             T=2    T=3    T=4                    │ │
│   │       글리치 구간                                            │ │
│   │       (주파수 3x 증가)                                       │ │
│   │       → 셋업/홀드 타임 위반                                   │ │
│   │       → 명령어 오류                                          │ │
│   │                                                              │ │
│   │   글리치 클럭 (듀티 사이클 변조):                              │ │
│   │      ▲   ┌──────┐ ┌┐ ┌──┐   ┌──┐                           │ │
│   │      │   │      │ │└─┘  │   │  │                           │ │
│   │      └───┘      └─┘     └───┘  └───┴────→ 시간               │ │
│   │          ↑       ↑                                            │ │
│   │       듀티    글리치                                         │ │
│   │       변조                                                  │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              공격 효과                                         │ │
│   │                                                              │ │
│   │   1. 명령어 스킵 (Instruction Skip)                          │ │
│   │   2. 레지스터 오염 (Register Corruption)                     │ │
│   │   3. 분기 우회 (Branch Bypass)                               │ │
│   │   4. 메모리 읽기/쓰기 오류                                    │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 클럭 글리칭은 클럭 신호 변조로 셋업/홀드 타임을 위반시킵니다.

**💡 비유**: 클럭 글리칭은 시계를 빨리 감는 것과 같습니다. 타이밍이 맞지 않으면 오류가 납니다.

**📢 섹션 요약 비유**: 클럭 글리칭은 시계 조작 같아요. 타이밍을 망쳐요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **클럭 제어** | 주파수 변조 | FLL/PLL | 시계 조절 |
| **타이밍 제어** | 정밀 동기 | ps~ns 단위 | 스톱워치 |
| **셋업/홀드** | 타이밍 제약 | 타이밍 위반 | 오차 |
| **플립플롭** | 상태 저장 | 오류 전파 | 메모리 |

**글리치 파라미터**

| 파라미터 | 범위 | 효과 |
|:---|:---:|:---|
| **주파수 변화** | 1.5x-10x | 타이밍 위반 |
| **듀티 사이클** | 10-90% | 래치 오류 |
| **지속 클럭** | 1-10 클럭 | 오류 범위 |

**핵심 알고리즘**: 명령어 분석 → 타이밍 계산 → 글리치 인가 → 오류 유발

**📢 섹션 요약 비유**: 셋업/홀드 타임만 지키면 안전해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 클럭 글리칭 vs 볼티지 글리칭 vs EMFI**

| 비교 항목 | 클럭 글리칭 | 볼티지 글리칭 | EMFI |
|:---|:---:|:---:|:---:|
| **대상** | 클럭 신호 | 전원 | 회로 전체 |
| **접근** | 클럭선/수정 | VCC/GND | 비접촉 |
| **정밀도** | 매우 높음 | 높음 | 중간 |
| **장비** | 함수 발생기 | FET 스위치 | EM 프로브 |

**📢 섹션 요약 비유**: 클럭 글리칭이 가장 정밀해요!
