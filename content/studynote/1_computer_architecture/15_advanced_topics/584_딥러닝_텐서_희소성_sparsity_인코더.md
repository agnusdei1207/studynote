> **💡 3-line Insight**
> - 딥러닝 모델, 특히 활성화 함수(ReLU 등)를 통과한 텐서(Tensor)에는 값이 '0'인 요소가 절반 이상을 차지하는 희소성(Sparsity)이 존재한다.
> - 희소성 인코더(Sparsity Encoder)는 이러한 0 값들을 제거하고 유의미한 비영(Non-zero) 데이터와 그 위치 정보(Index)만을 압축하여 저장하는 하드웨어 가속기 블록이다.
> - 메모리 대역폭(Bandwidth) 요구량을 극적으로 낮추고, 이어지는 연산 파이프라인에서 불필요한 곱셈-누적(MAC) 연산을 생략할 수 있게 하여 AI 가속기의 전력 효율을 극대화한다.

## Ⅰ. 딥러닝 텐서의 희소성(Sparsity) 특성

딥러닝 네트워크, 특히 합성곱 신경망(CNN)이나 최신 대규모 언어 모델(LLM)의 실행(Inference) 과정에서 발생하는 데이터(텐서)는 고유한 통계적 특성을 가진다. 대표적으로 ReLU (Rectified Linear Unit) 계열의 활성화 함수는 음수 값을 모두 0으로 변환하므로, 출력 텐서의 50~80%가 0으로 채워지는 활성화 희소성(Activation Sparsity)이 발생한다.
또한, 모델 경량화를 위해 진행되는 프루닝(Pruning, 가지치기) 기법은 중요도가 낮은 가중치(Weight)를 0으로 만들어 가중치 희소성(Weight Sparsity)을 생성한다. 이렇게 0이 대다수인 희소 텐서(Sparse Tensor)를 그대로 메모리에 저장하고 연산 유닛으로 전송하는 것은 엄청난 대역폭 낭비와 전력 소모를 초래한다. 이를 해결하기 위해 하드웨어 가속기 내부에 탑재되는 것이 희소성 인코더(Sparsity Encoder)이다.

📢 섹션 요약 비유:
거대한 수박(텐서)을 샀는데 안을 쪼개보니 80%가 껍질(0)이고 20%만 진짜 과육(유효 데이터)인 상황에서, 껍질까지 몽땅 트럭으로 옮기는 대신 현장에서 과육만 발라내어 통에 담아가는 작업입니다.

## Ⅱ. 희소성 인코더의 아키텍처 및 압축 포맷 (CSR/CSC)

희소성 인코더는 메모리(SRAM 또는 DRAM)로 데이터를 보내기 전에 실시간으로 0을 필터링하고 압축 포맷으로 변환하는 하드웨어 논리 회로이다. 주로 CSR (Compressed Sparse Row) 또는 CSC (Compressed Sparse Column) 포맷이 사용된다.

```text
[ 희소성 인코더 동작 로직 (CSR 포맷 예시) ]

원본 텐서 (Dense 4x4 Matrix)
  5  0  0  2
  0  0  3  0
  0  0  0  0
  1  0  0  4
       |
       v
[ Sparsity Encoder Hardware ]
  1. 0 검출 (Zero Detection)
  2. 비영(Non-zero) 값 추출
  3. 위치 인덱스 계산 (Row Pointers, Column Indices)
       |
       v
압축된 데이터 (CSR Format)
  - Values (Data):  [ 5, 2, 3, 1, 4 ]       -> 유효 데이터만 저장 (5개)
  - Col_Index:      [ 0, 3, 2, 0, 3 ]       -> 각 데이터의 열 위치
  - Row_Pointer:    [ 0, 2, 3, 3, 5 ]       -> 각 행이 시작되는 Values 배열의 인덱스
```

인코더는 파이프라인을 통과하는 스트리밍 데이터에서 0을 판별하는 비교기(Comparator)와, 남은 데이터의 원래 위치(Index)를 추적하는 카운터(Counter) 로직으로 구성된다. 압축된 데이터와 메타데이터(Index)를 패킷화하여 버스로 전송한다. 반대로 연산 유닛이 데이터를 요청할 때는 디코더(Decoder)가 이를 원래 모양으로 복원하거나, 압축된 상태 그대로 연산기에 주입한다.

📢 섹션 요약 비유:
출석부(행렬)에 결석한 학생(0)이 너무 많을 때, 전체 명단을 다 부르지 않고 "1번 줄의 1번째와 4번째, 2번 줄의 3번째 자리에 앉은 학생만 출석!(CSR 포맷)"이라고 출석 방식 자체를 압축해버리는 것입니다.

## Ⅲ. 하드웨어 설계의 과제: 불규칙성(Irregularity) 처리

희소성 인코더를 하드웨어로 구현할 때 마주하는 가장 큰 문제는 데이터의 불규칙성(Irregularity)이다. 딥러닝의 희소성은 비정형적(Unstructured)으로 발생하므로, 어떤 캐시 라인은 0이 90%일 수도 있고, 어떤 라인은 0이 전혀 없을 수도 있다.
이로 인해 고정된 대역폭을 가지는 메모리 버스와의 매칭, 가변 길이 데이터에 대한 버퍼링(Buffering), 그리고 여러 연산 유닛 간의 작업 부하 불균형(Load Imbalance)이 발생한다. 이를 완화하기 위해 N:M 희소성(예: NVIDIA Ampere 아키텍처의 2:4 Sparsity)과 같은 정형화된 희소성(Structured Sparsity) 패턴을 강제하거나, FIFO (First-In First-Out) 버퍼를 깊게 설계하여 데이터 흐름의 속도를 조절(Smoothing)하는 메커니즘이 인코더에 함께 통합된다.

📢 섹션 요약 비유:
과육만 발라내서 상자에 담는데, 어떤 수박은 과육이 너무 적고 어떤 수박은 꽉 차 있어서 1분에 채워지는 상자 개수가 들쭉날쭉합니다. 그래서 중간에 임시 보관 창고(FIFO 버퍼)를 두어 컨베이어 벨트가 일정한 속도로 돌아가게 맞추는 것입니다.

## Ⅳ. 희소성 인코더와 AI 가속 성능의 시너지

희소성 인코더는 단순히 데이터를 압축하는 것을 넘어, AI 가속기(NPU, TPU, GPU Tensor Core)의 핵심인 MAC (Multiply-Accumulate) 연산 효율을 극대화하는 촉매제 역할을 한다.
압축된 텐서 포맷(예: Index 정보)을 MAC 유닛이 직접 해석할 수 있게 설계하면, 0을 곱하는 불필요한 연산 사이클(Clock Cycle)을 원천적으로 건너뛸 수 있다 (Zero-skipping). 만약 가중치 행렬과 활성화 행렬 모두에 희소성 인코더가 적용되어 있다면, 연산량은 이론적으로 (가중치 밀도 $\times$ 활성화 밀도) 비율로 급감한다. 이는 대규모 모델 추론 시 지연 시간(Latency)을 획기적으로 줄여준다.

📢 섹션 요약 비유:
빈 의자(0)가 많은 교실에서 선생님이 시험지를 나눠줄 때, 모든 책상에 들러 확인하는 것이 아니라, 앞에서 학생이 앉은 자리 위치만 적힌 지도를 보고 그 자리로만 직진해서 시험지를 나눠주어 시간을 크게 단축하는 원리입니다.

## Ⅴ. 트렌드와 미래 발전 방향

현재 희소성 인코더는 최신 AI 가속기의 필수 구성 요소가 되었다. 특히 LLM (Large Language Model) 시대에 들어서면서, 메모리 대역폭이 모델의 성능을 제한하는 '메모리 바운드(Memory-bound)' 문제가 심화됨에 따라 희소성 압축의 중요성은 더욱 커지고 있다.
향후 발전 방향은 소프트웨어(컴파일러/학습 프레임워크)와 하드웨어의 코디자인(Co-design)에 있다. 모델 학습 단계에서부터 하드웨어 인코더가 가장 효율적으로 압축할 수 있는 형태의 희소성을 가지도록 유도하는 'Sparsity-aware Training' 기술과, 동적으로 변화하는 희소 패턴에 실시간으로 적응하는 가변 정밀도 인코더(Dynamic Precision Encoder)가 차세대 신경망 처리 장치(NPU)의 핵심 아키텍처로 연구되고 있다.

📢 섹션 요약 비유:
예전에는 농부(소프트웨어)가 맘대로 키운 수박을 공장(하드웨어)에서 힘들게 분류했다면, 이제는 아예 농부와 공장이 회의를 해서 씨가 한곳에만 몰려있고 과육은 네모난 모양으로 자라는 '공장 맞춤형 수박(Sparsity-aware Training)'을 재배하는 시대로 진화하고 있습니다.
