# 벡터형 인터럽트 컨트롤러 (VIC, NVIC)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드웨어가 각 인터럽트 소스별로 고유한 서비스 루틴 (ISR)의 시작 주소를 직접 관리하고, CPU에 즉시 제공함으로써 인터럽트 처리 속도를 극대화한 인터럽트 관리 아키텍처다.
> 2. **가치**: 기존의 비벡터형 (Non-vectored) 방식에서 발생하던 '어떤 장치가 불렀는지 일일이 확인하는 소프트웨어 과정 (Polling)'을 제거하여, 마이크로초 단위의 초저지연 (Low-latency) 응답성을 보장한다.
> 3. **융합**: ARM Cortex-M 시리즈의 NVIC (Nested Vectored Interrupt Controller)와 같이 우선순위 기반 중첩 (Nesting) 기능과 결합되어, 현대 실시간 임베디드 시스템 (RTOS)의 핵심 엔진 역할을 수행한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 벡터형 인터럽트 컨트롤러 (VIC, Vectored Interrupt Controller)는 인터럽트 발생 시 해당 장치에 할당된 메모리 주소 (Vector)를 하드웨어가 즉시 참조하여 CPU의 프로그램 카운터 (PC)에 넘겨주는 장치다. NVIC는 여기에 '중첩 (Nested)' 기능을 더해, 실행 중인 ISR보다 높은 우선순위의 인터럽트가 오면 즉시 다시 가로채는 지능형 컨트롤러다.

- **💡 비유**: 벡터형 인터럽트는 "단축 다이얼이 있는 비서"와 같다. 옛날 비서 (Non-vectored)는 전화가 오면 "누구세요?"라고 물어보고 장부를 뒤져서 사장님께 연결했지만, 단축 다이얼 비서 (VIC)는 발신 번호만 보고 즉시 담당 부서 연결 버튼을 눌러 사장님이 바로 통화하게 해주는 것과 같다.

- **등장 배경 및 필요성**:
  1. **응답 속도의 한계**: 비벡터형 방식은 하나의 인터럽트 핀을 여러 장치가 공유하여, 인터럽트가 발생할 때마다 소프트웨어가 모든 장치를 전수 조사 (Polling)해야 하는 오버헤드가 컸다.
  2. **실시간성 강화**: 공장 자동화, 자동차 제어 등에서는 수 마이크로초의 지연도 사고로 이어질 수 있어 하드웨어 차원의 고속 라우팅이 필요했다.
  3. **복잡한 우선순위 관리**: 수십 개의 인터럽트 소스가 동시에 발생할 때, 이를 지능적으로 정렬하고 중요도에 따라 실행 순서를 결정하는 중앙 관리자가 필요해졌다.

- **📢 섹션 요약 비유**: 물어볼 필요 없이 번호표만 보고 담당 창구로 즉시 안내하는 은행의 자동 번호표 시스템과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 비벡터형 vs 벡터형 인터럽트 처리 비교

| 비교 항목 | 비벡터형 (Non-vectored) | 벡터형 (Vectored - VIC) |
|:---|:---|:---|
| **하드웨어 구조** | 공통 인터럽트 라인 1개 사용 | 각 소스별 독립된 벡터 번호 부여 |
| **ISR 찾기 방식** | 소프트웨어 루프 (if-else Polling) | 하드웨어 테이블 참조 (Direct Jump) |
| **처리 지연 (Latency)** | 김 (장치가 많을수록 비례해서 증가) | 매우 짧음 (장치 수와 무관하게 일정) |
| **구현 난이도** | 단순함 | 하드웨어 설계가 복잡함 |
| **주 사용처** | 초기 8비트 마이크로프로세서 | 현대 32/64비트 MCU, 고성능 프로세서 |

---

### NVIC (Nested Vectored Interrupt Controller) 아키텍처

NVIC는 벡터 방식에 '중첩'과 '우선순위 그룹화'를 추가하여 실시간성을 극대화한 구조를 가진다.

```text
 ┌──────────────────────────────────────────────────────────────────┐
 │               NVIC (Nested Vectored Interrupt Controller) 구조      │
 ├──────────────────────────────────────────────────────────────────┤
 │                                                                  │
 │  [인터럽트 소스들]                                                 │
 │  ┌──────┐      ┌──────────────────────────────────────────────┐  │
 │  │Timer │─────▶│ NVIC (Nested Vectored Interrupt Controller) │  │
 │  └──────┘      │                                              │  │
 │  ┌──────┐      │  1. 우선순위 판별 (Priority Resolver)         │  │
 │  │UART  │─────▶│  2. 벡터 테이블 참조 (Vector Table Look-up)   │  │
 │  └──────┘      │  3. 중첩 제어 (Nesting Control)              │  │
 │  ┌──────┐      └──────────────┬───────────────────────────────┘  │
 │  │ADC   │─────▶               │                                  │
 │  └──────┘                     │ (PC: Vector Address 전달)         │
 │                               ▼                                  │
 │  ┌─────────────────────────────────────────────────────────────┐ │
 │  │ CPU Core                                                    │ │
 │  │ ┌──────────────────┐        ┌────────────────────────────┐  │ │
 │  │ │ 현재 실행 중인     │        │ Vector Table (SRAM/Flash)  │  │ │
 │  │ │ 태스크/ISR        │        │ [0] Reset   -> 0x08000000  │  │ │
 │  │ └──────────────────┘        │ [1] NMI     -> 0x08000004  │  │ │
 │  │         │  (Context Switch)  │ [2] Timer   -> 0x08000040  │  │ │
 │  │         ▼                    │ [3] UART    -> 0x08000080  │  │ │
 │  │ [스택에 레지스터 자동 저장]      └────────────────────────────┘  │ │
 │  └─────────────────────────────────────────────────────────────┘ │
 └──────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 여러 인터럽트 소스 (Timer, UART 등)에서 신호가 오면, NVIC는 내부의 **우선순위 판별기 (Priority Resolver)**를 통해 가장 급한 일을 먼저 선별한다. 이때 하드웨어는 즉시 **벡터 테이블 (Vector Table)**을 뒤져 해당 인터럽트에 맞는 ISR의 주소 (예: 0x08000040)를 찾아낸다. CPU Core는 소프트웨어 개입 없이도 NVIC가 알려준 주소로 즉시 점프하며, 동시에 현재 작업의 핵심 상태 (레지스터들)를 스택 (Stack)에 자동으로 저장 (Hardware Stacking)하여 문맥 전환 (Context Switch) 시간을 획기적으로 단축한다.

---

### 심층 동작 원리: 테일 체이닝 (Tail-Chaining)

NVIC의 혁신적인 속도 비결 중 하나는 '테일 체이닝' 기술이다.

1. **상황**: 우선순위가 같은 두 인터럽트 A, B가 연달아 발생함.
2. **일반 처리**: A 실행 → 스택 복구 (Unstacking) → 다시 스택 저장 (Stacking) → B 실행. (낭비 발생)
3. **테일 체이닝**: A 실행 → **Unstacking/Stacking 건너뛰기** → 바로 B 실행.
4. **효과**: 불필요한 메모리 접근을 없애 인터럽트 간 전환 지연시간을 6사이클 이상 단축한다.

- **📢 섹션 요약 비유**: 일을 마치고 집에 갔다가 다시 출근하는 대신, 퇴근하려던 찰나에 바로 다음 업무를 이어서 처리하는 효율적인 야근 시스템과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 주요 인터럽트 컨트롤러 모델 비교

| 모델명 | 아키텍처 | 특징 | 주 사용처 |
|:---|:---|:---|:---|
| **8259A PIC** | Non-vectored (Legacy) | 캐스케이딩(Cascading) 방식, 속도 느림 | 초기 x86 PC |
| **VIC (ARM9)** | Vectored | 하드웨어 주소 제공, 중첩 제한적 | 고전 임베디드 |
| **NVIC (Cortex-M)** | Nested Vectored | 하드웨어 스택 저장, 테일 체이닝, 실시간성 최강 | MCU, IoT, 차량용 |
| **GIC (ARM Cortex-A)** | Multi-core Support | 멀티코어 분산 처리, 가상화 지원 | 스마트폰, 서버 |

---

### 과목 융합 관점

- **운영체제 (OS)**: NVIC의 우선순위 그룹화 (Preemption Priority vs Sub Priority) 기능을 활용하여, RTOS 커널이 선점형 스케줄링 (Preemptive Scheduling)을 하드웨어적으로 구현할 수 있게 한다.
- **전자 회로 (Electronics)**: 인터럽트 핀의 지터 (Jitter)나 노이즈를 방지하기 위해 NVIC 내부에는 디지털 노이즈 필터 (Glitch Filter)가 내장되는 경우가 많다.

- **📢 섹션 요약 비유**: 복잡한 도로 위에서 구급차에게 하이패스 차선을 열어주는 지능형 교통 관제 시스템과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 드론 비행 제어 장치 (FC)의 인터럽트 충돌

드론이 비행 중 자이로 센서 데이터 읽기 (고순위)와 GPS 수신 (저순위)이 동시에 발생했다.
1. **분석**: 만약 단순 인터럽트 컨트롤러라면 GPS 데이터를 처리하느라 자이로 데이터를 놓쳐 드론이 추락할 수 있다.
2. **해결**: NVIC의 **Preemption Priority**를 설정한다. 자이로 센서에 더 높은 우선순위를 부여하면, GPS ISR이 실행 중이더라도 NVIC가 즉시 CPU를 가로채 자이로 ISR로 넘겨준다. GPS는 자이로 처리가 끝난 뒤 **테일 체이닝**으로 나머지 처리를 완료한다. 이를 통해 비행 안정성을 완벽히 확보한다.

### 도입 체크리스트
- **벡터 테이블 배치**: 벡터 테이블을 SRAM에 배치하여 플래시 메모리 접근 지연을 줄였는가? (VTOR 레지스터 설정)
- **우선순위 설계**: 시스템의 모든 인터럽트 소스에 대해 데드라인 (Deadline)을 분석하여 우선순위를 중복 없이 배정했는가?
- **지터 관리**: 인터럽트 응답 시간이 일정 (Deterministic)해야 하는 제어 루프가 있는가? (NVIC의 고정 사이클 응답성 확인)

### 안티패턴
- **ISR 내 긴 대기**: ISR 내부에서 `printf`나 `delay`를 사용하는 것. 이는 아무리 좋은 NVIC라도 시스템 전체를 멈추게 하는 행위다. ISR은 깃발 (Flag)만 세우고 즉시 빠져나와야 한다.

- **📢 섹션 요약 비유**: 번호표를 빨리 뽑아줘도 창구 직원이 한 손님과 1시간 동안 수다를 떨면 (긴 ISR), 전체 은행 업무가 마비되는 것과 같습니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 효과 | 비고 |
|:---|:---|:---|
| **정량 (속도)** | 인터럽트 지연 시간 (Latency) 5~10배 단축 | 수십 사이클 이내 응답 |
| **정량 (효율)** | 소프트웨어 오버헤드 (Polling) 100% 제거 | CPU 자원 절약 |
| **정성 (안정)** | 실시간 결정성 (Determinism) 보장 | 하드웨어 제어 안정성 향상 |

### 미래 전망
- **가상화 지원 VIC**: 하이퍼바이저 환경에서 각 가상 머신 (VM)에 독립적인 가상 인터럽트를 할당하고 라우팅하는 기술이 고도화될 것이다.
- **보안 강화 NVIC**: TrustZone 기술과 결합하여, 보안 영역 (Secure World)의 인터럽트가 비보안 영역에 의해 마스킹되거나 변조되지 않도록 물리적으로 격리하는 기술이 표준이 될 것이다.

### 참고 표준
- **ARM v7-M / v8-M Architecture Reference Manual**: NVIC의 상세 동작 표준.
- **MISRA C**: 고신뢰성 임베디드 소프트웨어에서 인터럽트 핸들러를 작성하는 가이드라인 제공.

- **📢 섹션 요약 비유**: 0.1초의 승부처에서 선수에게 가장 빠른 길을 안내하는 코치처럼, VIC와 NVIC는 디지털 세계의 실시간 응답성을 지키는 수호자입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **NMI (Non-Maskable Interrupt)**: VIC/NVIC조차도 막을 수 없는 최우선 순위 인터럽트.
- **RTOS (Real-Time Operating System)**: NVIC의 하드웨어 스케줄링 기능을 활용하는 소프트웨어 플랫폼.
- **문맥 전환 (Context Switch)**: NVIC가 하드웨어적으로 자동 수행하는 핵심 작업.
- **GIC (Generic Interrupt Controller)**: 멀티코어 환경으로 확장된 인터럽트 컨트롤러.
