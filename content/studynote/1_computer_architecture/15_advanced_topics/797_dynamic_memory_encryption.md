# 동적 메모리 암호화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실행 중인 애플리케이션의 메모리 영역을 실시간으로 암호화하여 런타임 메모리 공격 방지
> 2. **가치**: 런타임 보호, 키 로테이션, 세밀한 제어, 프로세스 격리
> 3. **융합**: MKTME, SEV, TEE, 프로세스 격리와 통합된 동적 보안

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

동적 메모리 암호화는 실행 중인 애플리케이션의 메모리 영역을 실시간으로 암호화하여 런타임 메모리 공격을 방지합니다.

> **해설**: 동적 메모리 암호화는 런타임에 메모리 영역별로 다른 키를 적용합니다.

**💡 비유**: 동적 메모리 암호화는 각 방마다 다른 열쇠를 쓰는 호텔과 같습니다.

**📢 섹션 요약 비유**: 동적 암호화는 각 방마다 다른 열쇠 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **키 관리자** | 키 생성/분배 | 하드웨어/소프트웨어 | 열쇠 관리자 |
| **암호화 엔진** | 실시간 암호화 | MMU/컨트롤러 | 자물쇠 |
| **페이지 테이블** | 영역 매핑 | 암호화 플래그 | 표시판 |
| **로테이션** | 키 교체 | 주기적/이벤트 | 열쇠 교체 |

**동적 암호화 기술**

| 기술 | 구현 | 특징 |
|:---|:---|:---|
| **MKTME** | Intel | 다중 키, 페이지별 |
| **SEV-ES** | AMD | VM + 레지스터 암호화 |
| **PKEY** | Software | Protection Keys |
| **mprotect** | OS | 메모리 보호 플래그 |

**핵심 알고리즘**: 프로세스 생성 → 키 할당 → 페이지 매핑 → 로테이션

**📢 섹션 요약 비유**: 프로세스마다 키가 달라요!
