> **핵심 인사이트**
> 1. 전통적인 가상 머신(VM)은 가짜 메인보드, 가짜 CPU를 하나하나 하드웨어 수준에서 다 모방(Emulation/VT-x)해야 하므로 덩치가 크고 부팅이 느리다.
> 2. 반면 **도커(Docker) 컨테이너**는 하드웨어 가상화를 쓰지 않는다. 대신 리눅스 OS가 제공하는 **네임스페이스(Namespace)**라는 기능으로, **하나의 OS 안에서 '프로세스가 보는 세상'에만 착시 현상을 일으켜 완벽한 가짜 환경(독방)**을 만들어 준다.
> 3. 이 착시 현상을 물리적으로 생성하고 컨테이너를 찍어내는 가장 밑바닥의 표준 핵심 엔진이 바로 **runc (컨테이너 런타임)**이다.



## Ⅰ. 무거운 VM vs 가벼운 컨테이너

서버 1대에 서로 격리된 10개의 독립된 공간을 만들고 싶습니다.
* **VM 방식 (하드웨어 격리)**: 서버에 KVM 하이퍼바이저를 깔고, 10개의 가짜 컴퓨터를 만듭니다. 각각의 가짜 컴퓨터 안에 **리눅스 OS를 10번 새로 부팅시킵니다.** OS가 10개나 켜져 있으니 램(RAM)을 수십 GB 낭비하고 켜는 데 수십 초가 걸립니다.
* **컨테이너 방식 (운영체제 격리)**: 서버에 깔린 1개의 리눅스 커널을 그대로 씁니다. 대신 프로세스(앱) 10개를 실행할 때, 각 앱의 눈에 특수한 '가상현실 고글(네임스페이스)'을 씌웁니다. 고글을 쓴 앱은 "아! 이 컴퓨터에는 나 말고 아무도 없구나!"라고 착각하며 동작합니다. 용량은 MB 수준이고 켜는 데 0.1초면 충분합니다.

> 📢 **섹션 요약 비유**: 10명의 가수를 격리해야 합니다. VM은 진짜 콘크리트 방 10개를 짓고 방마다 전용 에어컨(OS)을 달아주는 방식입니다(안전하지만 비쌈). 컨테이너는 하나의 큰 강당(단일 커널) 안에 10개의 투명한 방음 텐트(네임스페이스)를 치고 중앙 에어컨 바람을 나눠 쐬게 하는 방식입니다(싸고 빠름).



## Ⅱ. 네임스페이스 (Namespace)의 6대 착시 마법

**runc(런타임)**는 리눅스 커널에 지시하여 프로세스에게 6가지의 완벽한 착시(환상)를 걸어줍니다. 이것이 우리가 "컨테이너에 들어왔다"고 느끼는 실체입니다.

1. **PID (프로세스 번호) 네임스페이스**
   실제 서버에는 크롬, 카톡 등 1,000개의 프로세스가 돌고 있고, 내 컨테이너의 진짜 번호는 `4591`번입니다. 하지만 고글을 씌우면 내 눈에는 나 자신만 보이며, 내 번호가 영광의 `PID 1번(시스템 최초 프로세스)`으로 보입니다.
2. **NET (네트워크) 네임스페이스**
   실제 서버의 랜카드가 아니라, 오직 나만을 위한 `eth0`라는 이름의 텅 빈 가짜 랜카드와 가짜 IP 주소가 보입니다.
3. **MNT (마운트/디스크) 네임스페이스**
   실제 서버의 복잡한 하드디스크 폴더가 안 보입니다. 오직 나만의 깨끗한 `C드라이브(Root /)` 폴더 체계만 보입니다.
4. **UTS (호스트명) 네임스페이스**: 서버 컴퓨터 이름(Hostname)을 내 맘대로 바꿀 수 있습니다.
5. **IPC (통신)**: 다른 컨테이너와 메모리를 통한 귓속말 통신을 물리적으로 차단합니다.
6. **User (사용자)**: 실제로는 서버의 천민(일반 유저) 권한으로 실행됐지만, 고글 속 내 세상에서는 내가 이 컴퓨터의 신(Root 관리자)이라고 착각하게 만듭니다.

### 컨테이너 착시 (ASCII)

```text
 ┌─────────── 진짜 물리 서버 (리눅스 커널 1개) ──────────────────┐
 │                                                               │
 │   [ 컨테이너 A (runc가 생성) ]   [ 컨테이너 B (runc가 생성)]
 │   PID=1 로 착각 중           PID=1 로 착각 중                 │
 │   IP: 172.17.0.2             IP: 172.17.0.3                   │
 │   (서로 존재 자체를 인지할 수 없는 완벽한 투명 텐트)          │
 │                                                               │
 └───────────────────────────────────────────────────────────────┘
```

> 📢 **섹션 요약 비유**: 매트릭스 세계입니다. 캡슐에 누워있는 사람(프로세스)에게 뇌파(runc 네임스페이스)를 쏴서, 좁은 캡슐 안이 아니라 혼자 있는 넓은 우주선 안에 있다고 완벽하게 뇌를 속이는 소프트웨어적 감옥입니다.



## Ⅲ. Cgroups (자원 통제)와의 결합

네임스페이스(눈가림)만으로는 부족합니다. 컨테이너 A가 자기가 혼자라고 착각해서 서버의 CPU 100%와 RAM 128GB를 미친 듯이 다 써버리면 옆 텐트의 컨테이너 B가 굶어 죽습니다.

그래서 runc는 눈가림(Namespace)과 더불어, 손발을 묶는 **Cgroups (Control Groups)**라는 리눅스 기능을 함께 사용합니다.
"너는 환상 속에서 네가 왕이라고 생각하겠지만, 물리적으로는 CPU의 10%와 RAM 2GB 이상은 절대 못 쓰게 하드웨어 자원을 끊어버리겠다"고 철저히 통제합니다. 

이 두 가지(Namespace + Cgroups)가 절묘하게 결합한 것이 현대 도커(Docker)와 쿠버네티스(K8s)를 지탱하는 클라우드 혁명의 밑바탕입니다.
