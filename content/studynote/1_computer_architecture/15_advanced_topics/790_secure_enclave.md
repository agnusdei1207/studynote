# 보안 엔클레이브 (Secure Enclave)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메인 프로세서와 격리된 전용 보안 코프로세서로 키 저장, 암호화, 인증을 수행하는 하드웨어
> 2. **가치**: 키 보호, 생체 인식, 안전 부팅, 암호화 가속, 물리적 격리
> 3. **융합**: TPM, HSM, TEE, PUF와 통합된 보안 실행 환경

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

보안 엔클레이브(Secure Enclave)는 메인 프로세서와 격리된 전용 보안 코프로세서로 키 저장, 암호화, 인증을 수행하는 하드웨어입니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **전용 CPU** | 격리 실행 | 보안 코프로세서 | 금고실 직원 |
| **보안 메모리** | 데이터 저장 | 암호화 SRAM | 금고 |
| **키 저장소** | 키 보관 | 하드웨어 fuses | 열쇠함 |
| **암호화 엔진** | 암호화 가속 | AES/RSA/ECC | 암호기 |

**주요 보안 엔클레이브**

| 제품 | 회사 | 용도 |
|:---|:---|:---|
| **Secure Enclave** | Apple | Face ID, 키체인 |
| **Titan** | Google | Android, Cloud |
| **Pluton** | Microsoft | Windows 보안 |
| **TrustZone** | ARM | TEE |

**핵심 알고리즘**: 격리 실행 → 키 보호 → 암호화 → 인증

**📢 섹션 요약 비유**: 각 회사마다 이름이 달라요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: Secure Enclave vs TPM vs TEE**

| 비교 항목 | Secure Enclave | TPM | TEE |
|:---|:---:|:---:|:---:|
| **형태** | 코프로세서 | 별도 칩 | 소프트웨어 격리 |
| **성능** | 높음 | 낮음 | 중간 |
| **비용** | 높음 | 낮음 | 낮음 |
| **보안 수준** | 매우 높음 | 높음 | 중간 |

**📢 섹션 요약 비유**: Secure Enclave가 가장 강력해요!
