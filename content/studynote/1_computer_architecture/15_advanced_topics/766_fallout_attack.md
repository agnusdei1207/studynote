# 폴아웃 (Fallout) 공격

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU Store Buffer를 통해 운영체제 커널 메모리 데이터를 유출하는 MDS 계열 사이드 채널 공격
> 2. **가치**: 보안 취약점 이해, 커널 보안, 완화 조치, CPU 마이크로아키텍처 보안
> 3. **융합**: MDS, RIDL, ZombieLoad, 멜트다운 계열과 통합된 버퍼 기반 공격

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

폴아웃(Fallout) 공격은 CPU Store Buffer를 통해 운영체제 커널 메모리 데이터를 유출하는 MDS 계열 사이드 채널 공격입니다. CVE-2018-12126으로 등록되었습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    폴아웃 (Fallout) 공격                               │

│   │   │                 Store Buffer (SB)                    │   │ │
│   │   │                                                     │   │ │
│   │   │   ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐         │   │ │
│   │   │   │Entry│ │Entry│ │Entry│ │Entry│ │Entry│  ...    │   │ │
│   │   │   │  0  │ │  1  │ │  2  │ │  3  │ │  4  │         │   │ │
```

> **해설**: Fallout은 Store Buffer의 잔류 데이터를 통해 커널 메모리를 유출합니다.

**💡 비유**: Fallout은 공문 훔쳐보기와 같습니다. 관공서(커널)에서 작성 중인 문서를 봅니다.

**📢 섹션 요약 비유**: Fallout은 공문 훔쳐보기 같아요. 커널 데이터를 봐요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Store Buffer** | 쓰기 버퍼링 | 메모리 쓰기 지연 | 발송 전 우편함 |
| **커널-유저 전환** | 권한 변경 | Ring 0 ↔ Ring 3 | 출입문 |
| **잔류 데이터** | 버퍼 잔존 | 이전 쓰기 데이터 | 남은 문서 |
| **타이밍 측정** | 채널 감지 | 캐시 접근 시간 | 관찰 |

**Fallout 특징**

| 항목 | 내용 |
|:---|:---|
| **CVE** | CVE-2018-12126 |
| **대상** | Intel CPU Store Buffer |
| **영향** | 커널 메모리 유출 |
| **완화** | 마이크로코드 + OS 패치 |

**핵심 알고리즘**: 커널 전환 → SB 잔류 → 사용자 모드에서 샘플링 → 유출

**📢 섹션 요약 비유**: 커널이 쓴 흔적을 사용자가 봐요!
