> **핵심 인사이트**
> 1. 윈도우에서 `시스템 정보`를 켜면 메인보드 모델명, RAM 제조사, CPU 시리얼 번호가 뜬다. OS가 컴퓨터 케이스를 뜯어본 것도 아닌데 어떻게 아는 것일까?
> 2. 컴퓨터가 처음 켜질 때, 메인보드 펌웨어(BIOS/UEFI)가 하드웨어 부품들을 쫙 스캔한 뒤, 이 정보들을 정해진 표 형식으로 메모리 구석에 적어놓는데 이를 **SMBIOS (System Management BIOS)**라고 한다.
> 3. 운영체제나 자산 관리 프로그램은 단지 이 SMBIOS 장부를 읽기만 함으로써, 시스템의 하드웨어 형상(Inventory)을 100% 파악할 수 있다.

<br>

## Ⅰ. OS는 하드웨어를 어떻게 아는가?

과거에는 OS가 하드웨어를 파악하려면 각 부품(CPU, 램, 그래픽카드)의 I/O 포트에 일일이 전기 신호를 찔러봐서 반응을 기다려야 했습니다(Probing).
이 방식은 너무 위험했습니다. 잘못 찌르면 기계가 뻗거나 멈춰버렸기 때문입니다.

그래서 델, 인텔 등의 제조사들이 모여서 합의를 합니다.
"앞으로 컴퓨터를 켤 때, 펌웨어가 부품 명세서를 만들어서 메모리의 특정 주소(예: `0xF0000` 근처)에 예쁘게 적어둘게. OS는 위험하게 부품 찌르지 말고 이 명세서만 읽어!"
이 명세서의 표준 포맷이 **SMBIOS**입니다.

> 📢 **섹션 요약 비유**: 이사 온 집에 들어갈 때, 내가 직접 보일러실 문을 따고 들어가서 어느 회사 보일러인지 확인(Probing)하다가 고장 내는 것이 과거 방식입니다. SMBIOS는 집주인이 현관문 앞에 '집 구조와 가전제품 스펙 목록(명세서)'을 붙여놓아, 세입자가 종이만 보고도 집을 다 파악하게 해주는 친절한 시스템입니다.

<br>

## Ⅱ. SMBIOS 테이블의 구조 (Type)

SMBIOS는 정보를 종류(Type)별로 쪼개서 저장합니다. 리눅스에서 `dmidecode` 명령어를 치면 이 장부를 그대로 볼 수 있습니다.

* **Type 0 (BIOS 정보)**: BIOS 제조사, 버전, 출시일
* **Type 1 (시스템 정보)**: HP, Dell 같은 제조사 이름, 제품명, 그리고 가장 중요한 **UUID (기기 고유 식별 번호)**
* **Type 4 (프로세서 정보)**: CPU 제품명(Intel Core i9), 코어 개수, 클럭, L1/L2 캐시 크기
* **Type 17 (메모리 장치)**: 메인보드의 램 슬롯 개수, 각 슬롯에 꽂힌 램의 용량, 속도(3200MHz), 제조사(Samsung)

### SMBIOS 읽기 과정 (ASCII)

```text
 ┌─── 메인보드 펌웨어 (부팅 시) ──────────┐
 │ 1. CPU, RAM, 보드 모델명 스캔        │
 │ 2. RAM 구석에 SMBIOS 형식으로 복사   │
 └─────────────────┬──────────────────┘
                   │
                   ▼
 ╔════════ 물리적 RAM (메모리) ═════════╗
 ║ [ Type 0 ] [ Type 1 ] [ Type 4 ] ║ ◀ (OS가 이 장부의 주소를 찾아 읽음)
 ╚══════════════════════════════════╝
                   │
                   ▼
 [ 윈도우 시스템 정보 / Linux dmidecode ]
```

> 📢 **섹션 요약 비유**: 명세서를 쓸 때 "0번 칸은 집주인 이름, 1번 칸은 집 주소, 4번 칸은 에어컨 스펙"으로 칸(Type)을 깐깐하게 나누어 놓은 국가 표준 서식입니다.

<br>

## Ⅲ. 산업적 활용 (자산 관리와 라이선스)

데이터센터에는 수만 대의 서버가 있습니다. 관리자가 엑셀로 서버 부품을 관리하는 건 불가능합니다.
대신 중앙 서버에서 각 서버에 "네 SMBIOS Type 1과 Type 17을 보내라"고 명령하면, 어떤 서버에 램이 몇 기가인지, 고유 일련번호(UUID)는 무엇인지 1초 만에 전수조사가 끝납니다.

또한 윈도우(Windows)나 오라클(Oracle) 같은 비싼 상용 소프트웨어는 불법 복제를 막기 위해 설치될 때 **SMBIOS의 UUID와 메인보드 일련번호를 긁어가서 암호 키(인증키)로 사용**합니다. 메인보드를 바꾸면 윈도우 정품 인증이 풀리는 이유가 바로 SMBIOS 장부의 내용이 바뀌었기 때문입니다.
