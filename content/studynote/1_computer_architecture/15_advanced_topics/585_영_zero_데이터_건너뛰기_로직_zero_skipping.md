> **💡 3-line Insight**
> - 영 데이터 건너뛰기 로직(Zero-skipping Logic)은 피연산자(Operand) 중 하나라도 0인 경우, 무의미한 곱셈-누적(MAC) 연산을 하드웨어 레벨에서 생략하여 클럭(Clock)과 전력을 아끼는 기술이다.
> - 희소성(Sparsity)이 높은 딥러닝(Deep Learning) 워크로드에서 AI 가속기의 성능을 비약적으로 끌어올리는 핵심 마이크로아키텍처(Microarchitecture) 기법이다.
> - 0을 판별하는 전처리 단계와 연산 스케줄링을 동적으로 재배치하는 로직이 필요하며, 데이터 패스(Datapath)에서 클럭 게이팅(Clock Gating)과 결합하여 극단적인 전력 효율을 달성한다.

## Ⅰ. 무의미한 연산(Zero-MAC)의 문제점과 Zero-skipping의 개념

수학적으로 어떤 숫자에 0을 곱하면 무조건 결과는 0이고, 0을 더하는 것은 기존 값에 아무런 변화를 주지 않는다 ($A \times 0 = 0$, $B + 0 = B$). 그러나 일반적인 CPU나 GPU의 ALUs(Arithmetic Logic Units) 또는 FPU(Floating Point Units)는 명령어가 하달되면 피연산자의 값이 무엇이든 상관없이 정해진 클럭 사이클(Clock Cycle) 동안 전체 논리 회로(Multiplier 등)를 활성화하여 전력을 소모한다.
딥러닝 모델의 활성화 함수(ReLU)나 프루닝(Pruning)된 가중치에는 0이 50% 이상 존재하는 경우가 흔하다. 이러한 환경에서 기존 방식으로 연산하면 전체 에너지의 절반 이상이 아무 의미 없는 '0 곱하기' 연산에 낭비된다. Zero-skipping (0 건너뛰기) 로직은 입력 데이터가 0인지 사전에 검사하여, 0일 경우 연산 파이프라인(Pipeline) 자체를 우회(Bypass)하거나 조기에 종료시켜 불필요한 스위칭 전력 소모와 사이클 낭비를 없애는 하드웨어 최적화 기술이다.

📢 섹션 요약 비유:
세탁기(연산기)에 옷을 넣을 때, 빨래통에 옷이 아예 없는 빈 바구니(0 데이터)인데도 무작정 세탁기 전원을 켜서 한 시간 동안 돌리는 바보 같은 행동을 막기 위해, 바구니가 비었으면 아예 세탁기 전원도 안 켜고 다음 바구니로 넘어가는 똑똑한 센서입니다.

## Ⅱ. Zero-skipping 아키텍처의 동작 메커니즘

Zero-skipping 로직은 주로 MAC (Multiply-Accumulate) 배열 앞단의 데이터 패스(Datapath)에 위치한다.

```text
[ 일반적인 MAC 유닛 ]
 Input A ---> [ Multiplier (A*B) ] ---> [ Adder (+C) ] ---> Output

[ Zero-skipping 로직이 적용된 MAC 유닛 ]
 Input A ---+--> (Zero Detector)
            |      |
            |      +-- (If A==0 or B==0) 
            |      |   1. 클럭 게이팅(Clock Gating) 활성화 -> Multiplier 전원 차단
            |      |   2. 제어 신호 발생 -> Adder 우회, 이전 누적값(C) 그대로 유지
            |      |   3. (선택적) 파이프라인 다음 유효 데이터로 즉시 스킵 (사이클 단축)
 Input B ---+-----> [ Multiplier ] ---------> [ Adder ] ---> Output
```

1. **Zero 검출 (Zero Detection)**: 입력되는 벡터나 텐서 원소(A, B)가 0인지 NOR 게이트 등을 이용해 단일 사이클 내에 판별한다.
2. **클럭 게이팅 (Clock Gating)**: 하나라도 0으로 판별되면, 전력을 가장 많이 먹는 거대한 곱셈기(Multiplier) 트리 회로에 공급되는 클럭 신호를 즉시 차단(Gating)하여 동적 전력 소모를 0으로 만든다.
3. **파이프라인 스킵 (Pipeline Skipping)**: 전력만 아끼는 것이 아니라 성능(Throughput)까지 높이기 위해, 스케줄러(Scheduler)가 0인 데이터를 버리고 FIFO 버퍼에서 다음 비영(Non-zero) 데이터를 즉시 끌어와 빈 클럭 슬롯을 채워버린다.

📢 섹션 요약 비유:
놀이공원(파이프라인)의 롤러코스터(곱셈기) 출발 직전에, 빈 의자(0 데이터)가 감지되면 그 칸의 안전바 잠금 장치 전원을 아예 꺼버려서(클럭 게이팅) 전기를 아끼고, 대기 줄에서 바로 다음 사람(비영 데이터)을 데려와 빈자리를 채우는(파이프라인 스킵) 것입니다.

## Ⅲ. Zero-skipping의 두 가지 방식: Temporal vs Spatial

Zero-skipping의 구현은 아키텍처의 목표에 따라 시간적(Temporal) 스킵과 공간적(Spatial) 스킵으로 나뉜다.
- **시간적 스킵 (Temporal Zero-skipping)**: 단일 연산 유닛에서 0이 들어오면 해당 클럭 사이클을 통째로 생략하고 다음 데이터를 당겨와 처리한다. 처리 시간(Latency)이 단축되어 속도가 빨라지지만, 각 연산 유닛의 완료 시점이 달라져 전체 동기화(Synchronization)가 복잡해지는 단점이 있다.
- **공간적 스킵 (Spatial Zero-skipping)**: 0 데이터를 다른 유휴(Idle) 연산 유닛이나 멀티플렉서(MUX)를 통해 우회시켜, 한 클럭 사이클 내에 더 많은 유효 데이터가 병렬(Parallel)로 연산되도록 공간을 재배치한다. 다수의 MAC이 얽힌 시스톨릭 어레이(Systolic Array) 구조의 NPU(Neural Processing Unit)에서 널리 쓰인다.

📢 섹션 요약 비유:
우체국 창구에서 빈 소포(0)가 올 때, 시간적 스킵은 "빈 거면 1초 만에 치우고 다음 손님 빨리 와!" 하는 방식이고, 공간적 스킵은 빈 소포 자리에 옆 창구 대기자를 끌어와서 창구 전체 공간을 빈틈없이 100% 가동하는 방식입니다.

## Ⅳ. 희소성 인코더(Sparsity Encoder)와의 결합

Zero-skipping 로직 자체는 들어오는 데이터가 0인지 매번 '검사'해야 하는 오버헤드를 가진다. 이를 극복하기 위해 메모리 단계에서부터 아예 0을 제거하고 위치 정보만 남긴 **희소성 인코더(CSR/CSC 포맷 등)**와 결합된다.
압축 포맷(Index 배열)을 지원하는 하드웨어 가속기(예: NVIDIA Tensor Core의 구조적 희소성 지원)는 연산기에 데이터가 도달하기도 전에 스케줄러가 Index 값만 보고 "이번엔 1번과 4번 위치만 곱하면 된다"는 것을 미리 안다. 따라서 하드웨어 차원의 무거운 Zero 검출기를 돌릴 필요 없이, 연산 파이프라인의 MUX(멀티플렉서) 제어 신호만으로 완벽한 Zero-skipping 네트워크를 구성할 수 있다.

📢 섹션 요약 비유:
택배 기사님이 집집마다 벨을 눌러보고 사람이 없으면(Zero 검출) 다음 집으로 가는 게 아니라, 우체국에서 미리 '사람 있는 집 지도(희소성 인코더 인덱스)'를 받아서 사람이 있는 집만 정확히 골라 방문하는 완벽한 배달 루트입니다.

## Ⅴ. 설계의 복잡성과 트레이드오프

Zero-skipping은 훌륭한 최적화 기법이지만 공짜가 아니다. 불규칙한 0 데이터의 분포는 메모리 버퍼 플러시나 파이프라인 스톨(Stall)을 유발할 수 있다. 다음 비영(Non-zero) 데이터를 제때 공급하지 못하면 오히려 연산기가 쉬게 되어 파이프라인 효율이 떨어진다. 
또한, Zero를 판별하고 데이터를 동적으로 스위칭하는 추가적인 제어 회로(Control Logic)와 멀티플렉서 배열 자체가 면적(Area)과 전력(Power)을 소모한다. 따라서 희소성이 10~20% 정도로 낮을 때는 우회 회로를 돌리는 비용이 연산을 그냥 수행하는 비용보다 커질 수 있다. 설계자는 대상 워크로드의 평균 희소율(Sparsity Ratio)을 프로파일링하여 브레이크이븐(Break-even) 포인트를 넘기는 수준에서만 우회 로직이 활성화되도록 정교하게 아키텍처를 튜닝해야 한다.

📢 섹션 요약 비유:
고속도로(파이프라인)에 텅 빈 차(0)가 보일 때마다 차를 강제로 우회로(Zero-skipping 회로)로 빼내는 교통경찰을 고용했는데, 빈 차가 별로 없으면 차라리 그냥 직진하게 놔두는 게 경찰 월급(회로 전력)과 우회로 건설비(면적)를 아끼는 더 현명한 방법일 수 있습니다.
