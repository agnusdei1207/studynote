# 결함 주입 테스트 (Fault Injection Test)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템에 의도적으로 고장(결함)을 발생시켜 복구 능력과 내결함성을 검증하는 테스트 기법
> 2. **가치**: 실제 장애 대응 능력 검증, 복구 절차 확인, HA 구성 검증. 운영 자신감 향상
> 3. **융합**: 카오스 엔지니어링, 무정전 운영. HA 클러스터. 재해 복구 훈련과 통합된 신뢰성 테스트

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

결함 주입 테스트(Fault Injection Test)는 시스템에 의도적으로 고장(결함)을 발생시켜 복구 능력과 내결함성을 검증하는 테스트 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    결함 주입 테스트 유형                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              하드웨어 결함 주입                                 │ │
│   │                                                              │ │
│   │   - PSU 전원 차단                                             │ │
│   │   - NIC 케이블 제거                                           │ │
│   │   - 디스크 제거/고장                                           │ │
│   │   - 메모리 고장 시뮬레이션                                     │ │
│   │   - CPU 코어 비활성화                                          │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              소프트웨어 결함 주입                               │ │
│   │                                                              │ │
│   │   - 프로세스 강제 종료 (kill -9)                               │ │
│   │   - 네트워크 지연/패킷 손실                                    │ │
│   │   - 메모리 누수 유발                                           │ │
│   │   - 디스크 I/O 지연                                            │ │
│   │   - 타임아웃 강제 발생                                         │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              테스트 절차                                       │ │
│   │                                                              │ │
│   │   1. 기준 상태 측정 (Baseline)                                │ │
│   │   2. 결함 주입 (Inject)                                       │ │
│   │   3. 시스템 반응 관찰 (Observe)                                │ │
│   │   4. 복구 확인 (Verify Recovery)                              │ │
│   │   5. 결과 기록 (Document)                                     │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 결함 주입은 하드웨어/소프트웨어 고장을 의도적으로 발생시켜 복구 능력을 검증합니다.

**💡 비유**: 결함 주입 테스트는 소방 훈련과 같습니다. 실제 화재 전에 대피 훈련을 합니다.

**📢 섹션 요약 비유**: 결함 주입은 소방 훈련 같아요. 사고 전에 연습해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **결함 인젝터** | 고장 발생 | HW/SW 제어 | 시뮬레이터 |
| **모니터** | 상태 관찰 | 메트릭 수집 | 관찰자 |
| **오케스트레이터** | 테스트 제어 | 시나리오 실행 | 지휘자 |
| **분석기** | 결과 분석 | 로그/메트릭 | 분석가 |

**주요 결함 주입 도구**

| 도구 | 유형 | 용도 |
|:---|:---|:---|
| **Chaos Monkey** | SW | 인스턴스 종료 |
| **Gremlin** | SW | 다양한 결함 |
| **Litmus** | SW | Kubernetes |
| **IPMI** | HW | 전원 제어 |

**핵심 알고리즘**: 기준 측정 → 결함 주입 → 복구 확인 → 결과 기록

**📢 섹션 요약 비유**: 결함 주입은 "뭐가 문제인지 미리 알아요!" 하는 거예요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

| 비교 항목 | 결함 주입 | 카오스 엔지니어링 |
