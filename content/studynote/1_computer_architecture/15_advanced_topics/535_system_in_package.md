# 535. 시스템 온 패키지 (SiP, System in Package)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서로 다른 공정(Node)으로 제조된 복수의 반도체 다이(Die), 수동 소자(Passive Component), 그리고 시스템 전원 모듈을 하나의 패키지 기판(Substrate) 내에 3차원적으로 집적하여 단일 칩처럼 동작하게 하는 고차원 패키징 아키텍처이다.
> 2. **가치**: **PCB (Printed Circuit Board)** 실장 면적을 최대 90% 이상 축소하고, 칩 간 거리를 단축하여 계산 성능(Performance) 대비 전력 효율(PPW - Performance Per Watt)을 극대화하며, 이기종 공정(Heterogeneous Process)을 통해 원가 경쟁력을 확보한다.
> 3. **융합**: 'More than Moore'를 실현하는 핵심 수단으로, **SoC (System on Chip)**의 한계를 넘어 **TSV (Through-Silicon Via)**, **RDL (Redistribution Layer)**, **Chiplet** 기술과 융합하여 반도체 생태계의 패러다임을 '칩 설계'에서 '시스템 패키징 설계'로 전환시켰다.

---

## Ⅰ. 개요 (Context & Background)

- **개념 정의**: 시스템 온 패키지(SiP, System in Package)는 로직, 메모리, 아날로그, RF(Radio Frequency) 등 서로 다른 기능을 수행하는 복수의 **IC (Integrated Circuit)** 다이를 단 하나의 패키지 캐비티 내에 적층(Stacking)하거나 병렬 배치하여 시스템 레벨의 기능을 구현하는 기술이다. 이는 단일 웨이퍼(Wafer) 공정에서 모든 것을 해결하려는 SoC와 달리, '가장 적합한 공정으로 만든 부품들을 조립'하는 철학을 따른다.
- **💡 비유**: **SoC**는 거대한 주방에서 모든 재료를 한 곳에 넣고 거대한 피자를 통째로 구워내는 것과 같다(모든 재료가 같은 온도/시간 조건을 맞춰야 함). 반면 **SiP**는 샌드위치 가게에서 각기 다른 최적의 재료(빵, 고기, 야채)를 따로 준비해두었다가, 주문이 들어오면 그 순간에 층층이 쌓아올려 포장지에 담는 '세트 메뉴' 조립 과정과 같다.

### 등장 배경 및 기술적 필연성
1.  **미세 공정의 한계 (Physical Limit)**: **Moore's Law(무어의 법칙)**가 다가 3nm, 2nm 극초미세 공정으로 진입하면서, 미세화를 통한 성능 향상과 비용 절감의 효율이 급격히 떨어지는 'Diminishing Return' 현상이 발생했다. 특히 아날로그/RF 회로는 미세 공정을 사용할 경우 노이즈 내성이 악화되어 오히려 성능이 저하되는 문제가 있었다.
2.  **이기종 집적의 요구 (Heterogeneous Integration)**: 스마트폰과 웨어러블 기기는 고성능 디지털 로직(미세 공정 필요), 대용량 메모리(고밀도 공정 필요), 고품질 아날로그/RF(고전압/특수 공정 필요)가 하나의 시스템 안에 공존해야 했다. SoC 방식으로는 이를 한 웨이퍼에 구현할 수 없었다.
3.  **폼팩터(Form Factor)의 혁명**: 스마트워치, TWS(True Wireless Stereo) 이어폰, AR/VR 글래스 등의 등장으로 **PCB (Printed Circuit Board)**의 공간이 물리적으로 사라지게 되었다. 시스템 부피를 줄이기 위해 부품들을 수평으로 배치하는 것이 아니라, 패키지 내부로 수직으로 들어앉히는 SiP가 유일한 대안이 되었다.

```text
  ┌─────────────────────────────────────────────────────────────┐
  │         [시대적 요구에 따른 집적 패러다임의 변화]                  │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ Phase 1: SoC (Monolithic)          Phase 2: SiP (Heterogeneous) │
  │ (모든 것을 하나로 통합)              (최적의 부품을 선택적 조립)        │
  │                                                             │
  │  [┌─────────────────────┐]          [ ┌─────┐ ┌─────┐ ┌─────┐ ]  │
  │  [ │  Logic  │  Mem  │ RF│ ]          [ │ CPU │ │ DRAM│ │ RF  │ ]  │
  │  [ │   (3nm)  │ (3nm) │(3nm)│ ]  vs    [ │(3nm)│ │(14nm)│ │(28nm)│ ]  │
  │  [ └─────────────────────┘]          [ └──┬──┘ └──┬──┘ └──┬──┘ ]  │
  │  [  단일 웨이퍼, 고비용, 수율 리스크 ]       [    공통 기판(Substrate)  ]  │
  │                                     [    각각 최적 공정 사용!   ]  │
  └─────────────────────────────────────────────────────────────┘
```
**(해설)** SoC는 하나의 실리콘 다이 안에 모든 것을 통합하는 방식으로 칩 간 연결 속도는 빠르지만, 하나의 기능이라도 불량이 나면 칩 전체를 폐기해야 하는 '수율(Yield) 악화'와 '비용 상승'의 리스크를 안고 있다. 반면 SiP는 각각의 부품을 독립적으로 생산하고 테스트(KGD)한 후 조립하므로, 전체 시스템의 비용 효율성과 설계 유연성을 동시에 확보할 수 있다.

**📢 섹션 요약 비유**: 마치 연필, 지우개, 다이어리를 따로따로 사서 파우치에 담는 것(SiP)과, 이들을 모두 녹여서 하나의 거대한 플라스틱 덩어리로 만드는 것(SoC)의 차이와 같습니다. 파우치에 담으면 필요할 때마다 최고급 지우개와 싸구려 연필을 자유롭게 섞어 쓸 수 있듯이, SiP는 서로 다른 특성의 칩들을 자유롭게 섞어 쓸 수 있는 기술입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 구성 요소 상세 분석 (Module Analysis)

SiP는 단순한 부품의 집합이 아니라, 그 자체로 하나의 완벽한 **마이크로 시스템**이다. 주요 구성 요소는 다음과 같다.

| 요소명 (Component) | 전체 명칭 (Full Name) | 역할 및 내부 동작 | 핵심 기술/파라미터 | 비유 (Analogy) |
|:---|:---|:---|:---|:---|
| **Bare Die** | 베어 다이 (회로가 노출된 반도체) | 패키징되지 않은 순수 반도체 칩. 각각 독립된 기능을 수행 | KGD (Known Good Die), Wire Bonding / Flip-Chip | 햄버거의 패티, 토마토, 치즈 |
| **Substrate** | 패키지 기판 (회로 기판) | 모든 다이와 소자가 올라가는 베이스. 외부 메인보드와의 **I/O** 인터페이스 역할 | Fine Pitch ABF, Micro-via, Layer Count (4~12L+) | 햄버거의 아래빵(트레이) |
| **Interposer** | 인터포저 (중개판) | 칩과 칩 사이를 초고밀도 배선으로 연결하는 다리. **2.5D SiP**의 핵심 | TSV (Through-Silicon Via), Si/Organic | 고속 데이터 전용 고가 교량 |
| **Passive** | 수동 소자 (R, L, C) | 노이즈 필터링, 전력 안정화. **EMI (Electromagnetic Interference)** 저감 필수 | 0201/01005 Size, Embedding Technology | 샌드위치의 소스와 양념 |
| **Overmold** | 에폭시 몰딩 컴파운드 | 외부 충격, 습기, 먼지로부터 보호 및 기계적 강도 확보 | High Thermal Conductivity EMC, Laser Drill | 햄버거를 감싸는 포장지 |

### 2. 3D 패키징 구조 및 데이터 플로우 (Architecture & Data Flow)

SiP의 핵심은 **어떻게 칩과 칩을 연결하느냐(Connection Method)**이다.

```text
  ┌────────────────────────────────────────────────────────────────┐
  │          [고급 SiP의 단면 구조: 3D 적층 및 TSV 연결]             │
  ├────────────────────────────────────────────────────────────────┤
  │                                                                  │
  │   TOP View (상단)          Side View (측면 - 단면)               │
  │                                                                  │
  │   ┌──────────────────┐         ▲ Mount (Solder Ball)            │
  │   │  Logic Die (CPU) │         │                         ┌────┐│
  │   └───────┬──────────┘         ▼                       ┌────┤PCB │
  │           │ │ (C4 Bump)     ┌──────────────────┐      │    └────┘
  │           │ └───────────────┤    Package       │      │      ▲
  │           ▼                 │    Substrate     │◄─────┘      │
  │   ┌──────────────────┐      └────────┬─────────┘      BGA Solder│
  │   │ Memory Die (DRAM)│               │ (Power/Gnd Signals)       │
  │   │    (Stack)       │               │                          │
  │   └───────┬──────────┘               ▼                          │
  │           │              ┌───────────────────────┐               │
  │           └──────────────┤   Interposer (Silicon)│               │
  │          TSV (Via)       │ (Micro Bumps < 100um) │               │
  │                          └───────────────────────┘               │
  │                                                                  │
  │   [동작 흐름]                                                      │
  │   1. CPU에서 연산 요청                                           │
  │   2. Interposer를 통해 DRAM으로 데이터 요청 (Ultra-Wide Bus)     │
  │   3. DRAM은 TSV를 통해 수직으로 데이터를 전달 (대역폭 최대화)      │
  │   4. Substrate를 통해 외부 전원 공급 및 열 방출                   │
  └────────────────────────────────────────────────────────────────┘
```

**(해설)** 위 다이어그램은 **2.5D/3D SiP** 구조를 보여준다. 가장 큰 변화는 칩 간 연결이 긴 와이어(Wire Bonding)에서 **Micro Bump**와 **TSV**로 바뀌었다는 점이다.
1.  **Wire Bonding**: 칩 가장자리에서 긴 금속 선을 연결하므로 인덕턴스가 커져 고속 동작에 불리.
2.  **Flip-Chip**: 칩 전체 면에 돌기(Bump)를 만들어 뒤집어 붙이는 방식. 저항과 인덕턴스가 획기적으로 줄어든다.
3.  **TSV (Through-Silicon Via)**: 실리콘 관통 전극. 칩 위에 또 다른 칩을 쌓고(Stacking), 수직으로 구멍을 뚫어 구리(Cu)로 채워 연결한다. 이는 고층 빌딩에 특화된 고속 엘리베이터를 설치한 것과 같아서, 데이터가 칩 밖으로 나왔다 다시 들어오는 번거로움 없이 수직으로 이동하여 전력 소모를 줄이고 대역폭(Bandwidth)을 폭발적으로 늘린다.

### 3. 핵심 설계 알고리즘 및 기술 사양 (Design Specs)

SiP 설계의 핵심은 **열(Thermal)과 신호 무결성(Signal Integrity, SI)**의 균형이다.

*   **HBM (High Bandwidth Memory) Integration**:
    ```text
    Total Bandwidth = # of Channels × Channel Speed × # of DRAM Stacks
    예시: AMD MI300 (3D Chiplet)
    - 8개의 HBM 스택
    - 각 스택당 1024-bit I/O
    - 3.2 Gbps 데이터 레이트
    -> 5.3 TB/s (PCB 방식 DDR5의 10배 이상)
    ```
    SiP, 특히 **CoWoS (Chip-on-Wafer-on-Substrate)** 기술을 통해야만 이런 대역폭을 구현할 수 있다.

**📢 섹션 요약 비유**: SoC가 넓은 평원에 단독 주택을 짓는 것이라면, SiP는 좁은 땅에 고층 아파트를 짓고 각 층을 특화된 고속 엘리베이터(TSV)로 연결한 것과 같습니다. 이렇게 하면 땅값(PCB 면적)은 줄이면서도 엘리베이터를 통해 층간 이동(데이터 전송)을 신속하게 할 수 있습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 1. SoC vs SiP vs Multi-Chip Module (MCM) 심층 비교

이 세 가지는 모두 '시스템 집적'을 목표로 하지만, 접근 방식과 성능 특징이 확연히 다르다.

| 비교 항목 | **SoC (System on Chip)** | **SiP (System in Package)** | **PCB (Multi-Chip Module)** |
|:---|:---|:---|:---|
| **집적 위치** | 단일 Silicon Die 내부 | 단일 Package 내부 (Interposer/Substrate) | 메인 보드 상 |
| **제약 조건** | 모든 IP가 동일한 공정 사용 필수 | 각 칩이 상이한 공정 사용 가능 | 각 칩이 독립 패키지 사용 |
| **핀 간 거리** | 나노미터 (nm) 수준 (초근접) | 마이크로미터 (µm) 수준 (근접) | 밀리미터 (mm) 수준 (원거리) |
| **전기적 성능** | ⭐⭐⭐⭐⭐ (최고) | ⭐⭐⭐⭐ (매우 우수) | ⭐⭐ (상대적으로 낮음) |
| **비용/수율** | ⭐ (수율 리스크 큼, 비용 고) | ⭐⭐⭐ (KGD 사용 시 수율 우수) | ⭐⭐⭐⭐ (모듈 교체 용이) |
| **주요 용도** | 스마트폰 AP, MCU | 스마트워치, HBM, 5G 모뎀 | 서버, 데스크탑 GPU |
