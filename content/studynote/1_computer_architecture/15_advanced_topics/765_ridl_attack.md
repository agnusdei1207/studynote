# 리들 (RIDL) 공격

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 내부 버스와 버퍼(Line Fill Buffer, RSB)를 통해 권한 없는 데이터를 유출하는 MDS 계열 공격
> 2. **가치**: 보안 취약점 이해, MDS 계열, 완화 조치, CPU 보안 설계
> 3. **융합**: MDS, Fall Clone, ZombieLoad, 사이드 채널 공격과 통합된 마이크로아키텍처 보안

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

리들(RIDL, Rogue In-Flight Data Load)은 CPU 내부 버스와 버퍼를 통해 권한 없는 데이터를 유출하는 MDS 계열 공격입니다. 2019년 공개되었습니다.

> **해설**: RIDL은 Line Fill Buffer의 인-플라이트 데이터를 샘플링하여 유출합니다.

**💡 비유**: RIDL은 공항 수하물 컨베이어 뒤지기와 같습니다. 남의 짐을 몰래 봅니다.

**📢 섹션 요약 비유**: RIDL은 수하물 컨베이어 뒤지기 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Line Fill Buffer** | L1D 채움 | 캐시 미스 처리 | 컨베이어 |
| **공유 버스** | 데이터 전송 | 코어 간 공유 | 이동 경로 |
| **RSB** | Return Stack | 함수 반환 예측 | 주소록 |
| **타이밍 측정** | 채널 감지 | 캐시 타이밍 | 관찰 |

**RIDL 변종**

| 변종 | 대상 | CVE | 설명 |
|:---|:---|:---|:---|
| **RIDL-LFB** | Line Fill Buffer | CVE-2019-12147 | LFB 데이터 유출 |
| **RIDL-RSB** | Return Stack Buffer | - | RSB 데이터 유출 |
| **RIDL-MDS** | MDS 결합 | - | MDS와 조합 |

**핵심 알고리즘**: 캐시 미스 유발 → LFB 데이터 잔류 → 샘플링 → 유출

**📢 섹션 요약 비유**: LFB에 지나가는 데이터를 몰래 봐요!
