# 보안 키 소거 (Secure Key Erasure)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 암호화 키를 안전하게 삭제하여 복구 불가능하게 만드는 보안 절차 및 하드웨어
> 2. **가치**: 키 유출 방지, 암호화 전방향성, 컴플라이언스, 수명 종료 관리
> 3. **융합**: 제로화, 안티 탬퍼, HSM, 키 관리 시스템과 통합된 키 수명 관리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

보안 키 소거(Secure Key Erasure)는 암호화 키를 안전하게 삭제하여 복구 불가능하게 만드는 보안 절차 및 하드웨어입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    보안 키 소거 (Secure Key Erasure)                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              키 소거 방법                                       │ │
│   │                                                              │ │
│   │   Level 1: 단순 덮어쓰기                                       │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  원본:  A5 B7 C3 D9 E1 F2 A4 B8                       │   │ │
│   │   │  소거:  00 00 00 00 00 00 00 00                       │   │ │
│   │   │                                                     │   │ │
│   │   │  단점: 단일 패스, 잔류 자기 가능성                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   Level 2: 다중 패스 덮어쓰기 (DoD 5220.22-M)                 │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Pass 1: FF FF FF FF FF FF FF FF                      │   │ │
│   │   │  Pass 2: 00 00 00 00 00 00 00 00                      │   │ │
│   │   │  Pass 3: AA 55 AA 55 AA 55 AA 55                      │   │ │
│   │   │  Pass 4: 55 AA 55 AA 55 AA 55 AA                      │   │ │
│   │   │  ...                                                  │   │ │
│   │   │  Pass 7: 랜덤 데이터                                   │   │ │
│   │   │                                                     │   │ │
│   │   │  장점: 잔류 자기 패턴 제거                             │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   Level 3: 암호화 소거 (Cryptographic Erasure)               │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  키를 암호화된 상태로 저장                              │   │ │
│   │   │  KEK(Key Encryption Key)만 소거                       │   │ │
│   │   │  → 모든 데이터 자동 무효화                             │   │ │
│   │   │                                                     │   │ │
│   │   │  장점: 빠름, 확실함                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 보안 키 소거는 단순 삭제부터 암호화 소거까지 다양한 방법이 있습니다.

**💡 비유**: 보안 키 소거는 문서 파쇄와 같습니다. 완전히 파쇄해야 복구 불가합니다.

**📢 섹션 요약 비유**: 보안 키 소거는 파쇄기 같아요. 완전히 없애요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **소거 명령** | 시작 신호 | API/트리거 | 버튼 |
| **소거 로직** | 삭제 수행 | 알고리즘 | 파쇄기 |
| **검증** | 완료 확인 | 체크섬 | 확인 |
| **로그** | 감사 추적 | 이력 기록 | 기록 |

**소거 표준**

| 표준 | 패스 수 | 용도 |
|:---|:---:|:---|
| **DoD 5220.22-M** | 7 | 군사/정부 |
| **NIST SP 800-88** | 1+ | 연방 기관 |
| **HMG IS5** | 3 | 영국 정부 |
| **Cryptographic** | 1 (KEK) | 최신 방식 |

**핵심 알고리즘**: 소거 요청 → 패스 수행 → 검증 → 로그 기록

**📢 섹션 요약 비유**: 암호화 소거가 가장 빨라요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: SSD 폐기** → 암호화 소거 (TRIM + Crypto Erase)
**시나리오 2: HDD 폐기** → DoD 5220.22-M 7패스
**시나리오 3: HSM 키 회전** → 즉시 소거

**도입 체크리스트**: 미디어 타입, 소거 표준, 검증, 로그

**📢 섹션 요약 비유**: 미디어마다 적합한 방법이 달라요!
