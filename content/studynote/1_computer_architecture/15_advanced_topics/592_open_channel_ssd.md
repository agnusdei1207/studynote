> **핵심 인사이트**
> 1. 일반적인 SSD는 내부에 똑똑한 컨트롤러(FTL)가 있어서 마모 평준화나 쓰레기 수집(GC)을 알아서 처리한다(블랙박스). 하지만 이 땜에 클라우드 서버에서는 **알 수 없는 타이밍에 갑자기 속도가 뚝 떨어지는 지연(Latency Spike) 현상**이 발생한다.
> 2. **오픈 채널 SSD**는 이 답답한 블랙박스를 깨부수고, SSD 내부의 물리적인 채널과 플래시 칩 구조를 호스트 컴퓨터(운영체제)에게 100% 투명하게 공개(Open)하는 구조다.
> 3. OS 커널이 FTL 역할을 직접 수행하므로, CPU가 덜 바쁠 때 쓰레기 수집(GC)을 시키는 등 초고도의 최적화가 가능해져 데이터센터의 응답 시간을 완벽하게 통제할 수 있다.

<br>

## Ⅰ. 기존 블랙박스 SSD의 횡포

일반 SSD는 매우 폐쇄적입니다.
서버 OS가 "데이터 읽어줘!"라고 요청했을 때, 하필 SSD 내부 컨트롤러가 "잠깐만! 나 지금 찌꺼기 청소(Garbage Collection) 중이라 바빠!"라고 맘대로 멈춰버리면, OS는 영문도 모른 채 1초 동안 응답을 기다리며 서버 전체가 렉에 걸리게 됩니다. (QoS 하락)

또 OS는 데이터베이스를 저장할 때 나름대로 조각 모음을 해서 보냈는데, SSD 내부 FTL이 마모 평준화를 하겠다고 자기 맘대로 칩 여기저기에 파편화해서 저장해버립니다. (소프트웨어와 하드웨어의 이중 헛수고)

> 📢 **섹션 요약 비유**: 블랙박스 SSD는 독단적인 창고 관리인입니다. 사장님(OS)이 물건을 급하게 빼달라고 해도 "저 지금 바닥 청소 중이니까 10분만 기다리쇼!"라며 배짱을 부리고, 사장님이 정리해준 장부도 자기 맘대로 다시 써버립니다.

<br>

## Ⅱ. 오픈 채널 SSD: 사장님의 직접 통제

클라우드 기업(알리바바, 바이두 등)은 이 꼴을 볼 수 없었습니다. 그래서 **오픈 채널 SSD**를 만들었습니다.

이 SSD 안에는 머리(FTL)가 없습니다. 단순히 낸드 플래시(저장 공간)와 거기로 향하는 수십 개의 물리적인 통로(Channel)만 열려(Open) 있습니다.
* **Host-Based FTL**: SSD가 하던 찌꺼기 청소, 마모 평준화, 주소 변환 등 모든 골치 아픈 일을 **서버의 운영체제(Linux Kernel)가 직접 코드로 처리**합니다.

### 아키텍처 비교 (ASCII)

```text
 [ 전통적인 SSD ]                        [ 오픈 채널 SSD ]
 ┌── Host OS ──┐                       ┌───── Host OS ─────┐
 │ 파일 시스템 │                       │ 파일 시스템       │
 └──────┬──────┘                       │ LightNVM (FTL)  │ ◀ (OS가 청소, 배치 다 함)
        │                              └────────┬──────────┘
 ┌──────▼──────┐                                │ (직통)
 │ FTL (블랙박스)│                                ▼
 │ NAND 플래시 │                       ┌─── Open-Channel ──┐
 └─────────────┘                       │ NAND 플래시       │ (순수 저장 공간)
                                       └───────────────────┘
```

> 📢 **섹션 요약 비유**: 깐깐한 창고 관리인을 해고해버렸습니다. 이제 사장님(OS)이 창고 바닥 구조(채널)를 훤히 들여다보며, "1번 구역은 지금 손님이 많으니 청소하지 말고 놔두고, 손님 없는 4번 구역부터 찌꺼기 치워!"라고 완벽하게 통제합니다. 

<br>

## Ⅲ. Zoned Namespace (ZNS) 로의 진화

오픈 채널 SSD는 성능은 최고였지만 치명적인 단점이 있었습니다. 낸드 플래시의 물리적인 특성(예: 제조사마다 다른 블록 크기와 타이밍)을 OS가 일일이 다 알아야 해서 관리가 너무너무 복잡했다는 점입니다.

그래서 최근에는 오픈 채널 SSD의 철학을 계승하면서도 표준화된 **Zoned Storage (ZNS SSD)**로 발전했습니다. (이는 다음 챕터에서 설명합니다.)
