# TDI (Trust Domain Interconnect)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Intel TDX Trust Domain 간 안전한 통신을 위한 프로토콜 및 인터커넥트 아키텍처
> 2. **가치**: TD 간 격리된 통신, 무결성 보호, 기밀 컴퓨팅 연동
> 3. **융합**: Intel TDX, TEE, VM 격리, 기밀 컴퓨팅과 통합된 보안 통신

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

> **해설**: TDI는 Trust Domain 간 통신을 하이퍼바이저 없이 안전하게 수행합니다.

**💡 비유**: TDI는 건물 내 보안 전화선과 같습니다. 건물 관리자가 엿들을 수 없습니다.

**📢 섹션 요약 비유**: TDI는 보안 전화선 같아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Trust Domain** | 격리 VM | TDX VM | 방 |
| **TDI 프로토콜** | 통신 규약 | 암호화 채널 | 전화 규칙 |
| **SEAM** | 보안 모듈 | CPU 모드 | 경비원 |
| **EPT** | 메모리 격리 | 확장 페이지 테이블 | 출입증 |

**TDX 구성 요소**

| 구성 요소 | 설명 |
|:---|:---|
| **TDX Module** | SEAM 모드에서 실행되는 보안 모듈 |
| **Trust Domain** | 격리된 VM |
| **TD VF** | Virtual Function |
| **TD VMCALL** | TD → TDX Module 호출 |

**핵심 알고리즘**: TD 생성 → 격리 실행 → TDI 통신 → 증명

**📢 섹션 요약 비유**: TDX가 VM을 더 안전하게 만들어요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: TDX/TDI vs SEV-SNP vs SGX**

| 비교 항목 | TDX/TDI | SEV-SNP | SGX |
|:---|:---:|:---:|:---:|
| **격리 단위** | VM | VM | Enclave |
| **크기** | GB~TB | GB~TB | MB~GB |
| **통신** | TDI | SEV-ES | ECall/OCall |
| **무결성** | 예 | 예 | 예 |

**📢 섹션 요약 비유**: TDX는 VM 전체를 보호해요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 클라우드** → TDX로 VM 격리, TDI로 통신
**시나리오 2: 다중 파티** → TD 간 안전한 데이터 교환
**시나리오 3: 규정 준수** → 하드웨어 기반 격리

**도입 체크리스트**: Intel TDX CPU, TDX 모듈, 하이퍼바이저 지원

**📢 섹션 요약 비유**: Intel 클라우드 서버에서 써요!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: VM 간 안전 통신, 하이퍼바이저 차단
