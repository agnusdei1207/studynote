> **핵심 인사이트**
> 1. 우리가 아는 어셈블리어(`ADD`, `MOV` 등)는 CPU가 1차적으로 이해하는 명령어(매크로 인스트럭션)일 뿐, x86 같은 복잡한 프로세서 내부에서는 이를 더 잘게 쪼갠 0과 1의 진짜 기계어인 **마이크로코드(Microcode, $\mu$op)**로 변환하여 실행한다.
> 2. 과거에는 이 변환 사전(ROM)이 하드웨어에 납땜되어 수정이 불가능했으나, 현대 CPU는 부팅 시 운영체제나 BIOS가 임시 메모리(SRAM)에 **새로운 마이크로코드를 덮어쓸 수 있는 기능(Patch)**을 제공한다.
> 3. 멜트다운이나 스펙터 같은 하드웨어 설계 오류가 터졌을 때, 전 세계의 수억 대의 PC를 리콜(환불)하는 대신 인텔과 AMD가 **마이크로코드 업데이트를 배포하여 CPU의 두뇌 회로를 일시적으로 고쳐 쓰는 것**이 이 원리다.



## Ⅰ. 마이크로코드란 무엇인가? (CISc의 비밀)

인텔이나 AMD의 x86 프로세서는 CISC(복잡한 명령어 세트) 구조를 가집니다. 
명령어 하나(`XADD` 등)가 너무 길고 복잡해서 하드웨어 논리 게이트로 한 번에 처리할 수가 없습니다.

그래서 CPU 내부에 작은 **통역사(디코더)**를 둡니다.
디코더는 거대한 명령어 사전을 보고, 복잡한 명령어 1개를 실행하기 쉬운 아주 단순한 마이크로코드 3~4개($\mu$op)로 잘게 쪼갭니다.
* `ADD [메모리], R1` $\rightarrow$ (디코더) $\rightarrow$ `1. Load`, `2. Add`, `3. Store` ($\mu$op로 분해)

이때 사용되는 '거대한 명령어 사전'이 바로 CPU 내부에 내장된 **마이크로코드 ROM**입니다.

> 📢 **섹션 요약 비유**: 레스토랑 주방장(하드웨어)은 멍청해서 "계란 후라이"라는 복잡한 단어를 모릅니다. 그래서 주문을 받으면 매니저(디코더)가 몰래 매뉴얼(마이크로코드)을 펴보고, 주방장에게 "1. 불 켜, 2. 기름 둘러, 3. 계란 깨"라고 아주 단순하게 쪼개서 지시를 내리는 시스템입니다.



## Ⅱ. 마이크로코드 패치의 동작 원리

1994년 인텔의 펜티엄 CPU에서 나눗셈 결과가 틀리게 나오는 치명적 결함(FDIV 버그)이 터져 5천억 원 규모의 리콜 사태가 벌어졌습니다. 이 뼈아픈 교훈 이후, 인텔은 마이크로코드 사전을 **업데이트할 수 있는 우회로**를 만들었습니다.

1. CPU 내부에는 쓰기 불가능한 기본 ROM 사전 외에, 텅 빈 **패치용 SRAM** 공간이 존재합니다.
2. 윈도우(OS)가 부팅될 때나 메인보드 BIOS가 켜질 때, 인텔 서버에서 다운받은 '최신 마이크로코드 패치 파일'을 읽어서 이 CPU의 SRAM에 쏙 밀어 넣습니다.
3. 이제 CPU의 디코더는 특정 명령어가 들어오면 기본 ROM 사전을 보지 않고, 새로 업데이트된 SRAM 사전의 코드를 우선적으로 가로채서(Intercept) 읽어들입니다.

### 동작 다이어그램 (ASCII)

```text
 [ x86 복잡한 명령어 입력 ]
                          │
             ▼
 ┌──────── 디코더 ────────┐ ─▶ [ 패치 매치 레지스터: 이 명령어 패치된 적 있나? ]
 │                        │             │ (Yes)
 │  [ 기본 ROM 사전 ]     │ ◀ (No)      ▼
 │  (공장에서 구워져 나옴) │      [ 임시 SRAM 사전 ] (OS가 부팅 시 주입한 새 마이크로코드)
 └───────────┬────────────┘
             │ (잘게 쪼개진 $\mu$op들)
             ▼
       [ ALU 연산 유닛 ]
```

> 📢 **섹션 요약 비유**: 매니저가 쓰던 구형 매뉴얼(ROM)에 치명적 오타가 발견됐습니다. 책을 새로 찍어주려면 돈이 너무 드니까, 포스트잇(SRAM 패치)에 올바른 레시피를 적어서 구형 매뉴얼 해당 페이지 위에 찰싹 붙여놓는 것입니다.



## Ⅲ. 보안의 구세주, 그러나 영구적이지 않은 마취제

스펙터(Spectre) 사태 때 IBPB 같은 새로운 방어용 명령어가 하늘에서 뚝 떨어진 것이 바로 이 마이크로코드 업데이트 덕분입니다. 기존 CPU 설계도에는 없던 명령어와 논리를 패치를 통해 억지로 쑤셔 넣은 것입니다.

**한계점 (휘발성):**
마이크로코드 패치는 물리적으로 납땜 되는 것이 아니라 램(SRAM)에 임시로 올라가는 것입니다. 
따라서 **컴퓨터 전원을 끄면 패치가 날아갑니다.** 그래서 우리는 윈도우 업데이트를 하거나 메인보드 BIOS를 업데이트하여, **컴퓨터가 부팅될 때마다 매번 0.1초 만에 CPU에 패치를 새로 주사**해 주어야만 합니다. (소프트웨어 의존성)
