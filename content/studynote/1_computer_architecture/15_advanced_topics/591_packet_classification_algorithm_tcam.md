> **Insight**
> - 네트워크 스위치 및 라우터에서 패킷을 수신했을 때, 헤더 정보를 분석하여 어떤 규칙(라우팅, ACL, QoS)을 적용할지 초고속으로 결정하는 기술
> - TCAM(Ternary Content Addressable Memory) 하드웨어를 활용하여 O(1)의 복잡도로 수백만 개의 규칙 중 일치하는 것을 단 1클럭 사이클 만에 검색
> - 고비용/고전력의 한계를 극복하기 위해 소프트웨어 알고리즘(Trie, Hash 등)과 결합한 하이브리드 접근법이 핵심 설계 과제

## Ⅰ. 패킷 분류(Packet Classification) 및 TCAM의 개요

패킷 분류는 네트워크 장비(라우터, 스위치, 방화벽)로 들어오는 데이터 패킷의 헤더(Source IP, Destination IP, Port 번호, Protocol 등)를 검사하여 사전에 정의된 규칙 테이블과 대조한 뒤, 해당 패킷을 포워딩할지, 폐기할지(보안/ACL), 혹은 우선순위를 부여할지(QoS) 결정하는 과정입니다.

네트워크 속도가 수백 Gbps로 빨라짐에 따라 소프트웨어 기반의 순차 탐색 알고리즘으로는 선속도(Line Rate) 처리가 불가능해졌습니다. 이를 해결하기 위해 등장한 하드웨어 솔루션이 **TCAM(Ternary Content Addressable Memory)**입니다. TCAM은 입력된 검색어(패킷 헤더)를 메모리 내의 모든 데이터(규칙)와 병렬로 동시에 비교하여 일치하는 메모리 주소를 즉시 반환하는 특수 목적의 하드웨어 메모리입니다.

📢 **섹션 요약 비유**: 수만 명의 범죄자 몽타주(규칙 테이블)를 경찰 1명(소프트웨어)이 한 장씩 넘기며 대조하면 하루 종일 걸리지만, 수만 명의 경찰(TCAM 메모리 셀)이 각자 몽타주 1장씩을 들고 있다가 용의자 얼굴(패킷)을 보여주자마자 일치하는 경찰이 동시에 손을 번쩍 드는 초고속 수사 기법과 같습니다.

## Ⅱ. TCAM의 동작 원리 및 아키텍처

일반적인 RAM(Random Access Memory)이 '주소'를 입력받아 '데이터'를 출력하는 반면, CAM(Content Addressable Memory)은 '데이터'를 입력받아 그 데이터가 저장된 '주소'를 출력합니다.
특히 TCAM(Ternary CAM)은 데이터를 0, 1, 그리고 **'Don't Care (X)'**의 세 가지 상태로 저장할 수 있어 IP 라우팅의 서브넷 마스크(Subnet Mask)와 같은 와일드카드 매칭에 최적화되어 있습니다.

```text
[TCAM Hardware Architecture]

Search Key (Packet Header: e.g., 192.168.1.5)
      |
      v
+---------------------------------------------------+
|  TCAM Array (Parallel Comparison in 1 Cycle)      |
|  Rule 1: 10.0.0.*        [ Match? No  ]           |
|  Rule 2: 192.168.1.*     [ Match? YES ] --------+ |
|  Rule 3: 192.168.*.*     [ Match? YES ]         | |
+-------------------------------------------------|-+
      | (Priority Encoder selects highest priority) |
      v                                           |
+-------------------------------------------------|-+
|  Priority Encoder (Resolves Multiple Matches)   | | (Rule 2 wins due to longest match)
+-------------------------------------------------|-+
      |
      v (Index Pointer)
+---------------------------------------------------+
|  SRAM (Action Memory)                             |
|  Index 2 Action: Forward to Port 3                |
+---------------------------------------------------+
      |
      v
Result: Forward to Port 3
```

1. **병렬 검색:** 들어온 패킷 헤더 정보가 TCAM 내부의 모든 엔트리에 동시 브로드캐스트되어 1클럭 사이클 내에 비교를 마칩니다.
2. **우선순위 인코더 (Priority Encoder):** 한 패킷이 여러 규칙(예: 서브넷이 다른 규칙들)에 동시에 매칭될 수 있습니다. 이때 하드웨어 로직인 우선순위 인코더가 가장 구체적인 규칙(Longest Prefix Match)의 주소를 선택합니다.
3. **액션 메모리 매핑:** 도출된 TCAM 주소를 바탕으로 일반 SRAM을 참조하여 최종 액션(포워딩 포트, 폐기 등)을 실행합니다.

📢 **섹션 요약 비유**: "서울시 강남구" 주소를 가진 소포가 들어왔을 때, "서울시로 가는 상자"와 "서울시 강남구로 가는 상자"가 동시에 손을 들면 반장(우선순위 인코더)이 더 정확한 "강남구" 상자를 선택해 그 안에 적힌 지시대로 소포를 처리하는 구조입니다.

## Ⅲ. TCAM의 장점과 치명적인 단점

**장점:**
* **O(1) 검색 복잡도:** 규칙의 개수가 10개든 10만 개든 항상 1 사이클(수십 나노초) 만에 검색이 완료되어 확정적인(Deterministic) 성능을 보장합니다.

**단점:**
* **고비용 및 낮은 밀도:** 각 메모리 셀마다 비교 논리 회로(Comparator)가 포함되어 있어 일반 SRAM 대비 칩 면적을 약 6배나 더 차지합니다. 가격이 매우 비쌉니다.
* **막대한 전력 소모 및 발열:** 검색 시마다 모든 회로가 동시에 켜져 전력 소모가 극심합니다. 하이엔드 라우터 전력 소모의 상당 부분을 TCAM이 차지합니다.
* **업데이트의 어려움:** 우선순위에 따라 규칙을 물리적으로 정렬(Sorting)해야 하므로, 새로운 규칙을 중간에 삽입할 때 대규모 데이터 이동이 발생합니다.

📢 **섹션 요약 비유**: 모든 직원이 초능력(동시 병렬 검색)을 가진 엘리트 탐정 사무소(TCAM)라서 사건 해결 속도는 세계 최고지만, 월급(전력)이 너무 비싸고 사무실 공간(칩 면적)도 많이 차지해서 직원을 수백만 명씩 늘리기는 불가능한 상황입니다.

## Ⅳ. 소프트웨어 기반 패킷 분류 알고리즘의 진화

TCAM의 용량 한계와 고비용 문제를 극복하기 위해 소프트웨어 기반의 휴리스틱 알고리즘들이 연구되었으며, 최근에는 하드웨어와 결합하는 형태로 발전하고 있습니다.

1. **Trie (Prefix Tree) 기반:** IP 주소의 비트를 하나씩 따라가며 탐색하는 트리 구조. 검색 속도는 느리지만 메모리 효율이 좋음. (예: Multibit Trie)
2. **Tuple Space Search (TSS):** Open vSwitch(OVS) 등에서 널리 쓰이는 방식으로, 해시(Hash) 테이블을 활용하여 규칙의 길이별로 나누어 검색 속도를 획기적으로 높인 소프트웨어 알고리즘입니다.
3. **Decision Tree (의사결정 트리):** 기하학적 공간 분할 방식(예: HiCuts, HyperCuts)을 사용하여 다차원 규칙을 노드 단위로 분할하여 탐색합니다.

📢 **섹션 요약 비유**: 전화번호부를 맨 앞장부터 찾는 대신, 'ㄱ, ㄴ, ㄷ' 인덱스(Trie)를 만들거나, 가장 자주 연락하는 사람 명단(Hash/TSS)을 따로 빼두어 똑똑하게 찾아가는 방법론들입니다.

## Ⅴ. 하이브리드 아키텍처 및 미래 동향

현대의 고성능 데이터센터 스위치와 DPU(Data Processing Unit)는 TCAM의 하드웨어적 한계를 극복하기 위해 소프트웨어(또는 SRAM 기반 검색 로직)와 TCAM을 결합한 하이브리드 아키텍처를 채택하고 있습니다.

* **Exact Match는 SRAM (Hash), Wildcard는 TCAM:** 정확히 일치해야 하는 수백만 개의 플로우(예: 세션 테이블)는 대용량 SRAM과 해시 알고리즘을 사용하고, IP 서브넷이나 포트 범위 등 복잡한 와일드카드 매칭이 필요한 수만 개의 보안 규칙(ACL)만 TCAM에 할당하는 전략입니다.
* **Algorithmic TCAM:** 물리적인 TCAM 칩 없이, SRAM과 고도로 최적화된 검색 알고리즘 회로를 조합하여 TCAM의 기능을 논리적으로 모사함으로써 전력과 비용을 줄이는 설계가 도입되고 있습니다.

📢 **섹션 요약 비유**: 회사에서 정기적으로 오는 흔한 소포들(Exact Match)은 일반 알바생(SRAM+Hash)들이 빠르게 대조해서 처리하고, 발신자가 모호하거나 복잡한 지시가 적힌 특별한 폭발물 의심 소포(Wildcard 규칙)만 비용이 비싼 베테랑 폭발물 처리반(TCAM)에게 맡겨 효율성을 극대화하는 방식입니다.
