# 392. 다단 연결망 (MIN, Multistage Interconnection Network)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 단계의 스위치로 구성된 확장 가능한 인터커넥트
> 2. **가치**: 비용 효율, 확장성, 비차단/재배치 가능
> 3. **융합**: 배니언, 베네시, 나비, 오메가 네트워크

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

다단 연결망(Multistage Interconnection Network, MIN)은 **여러 단계(stage)의 작은 스위치들을 계단식으로 연결하여, 대규모 입력-출력 연결을 비용 효율적으로 구현하는 상호 연결망 구조**다. 크로스바의 비용 문제와 버스의 성능 문제를 절충한다.

### 💡 비유: 환승 지하철

다단 연결망은 **환승 지하철**과 같다. 1호선이 있다. 2호선이 있다. 환승역에서 갈아탄다. 여러 단계를 거친다. 직접 가지 않아도 된다. 환승으로 목적지까지. 다단 연결망이 똑같다. 여러 스위치 단계, 환승처럼 연결, 목적지 도달.

│  │   └──┘    │    │S0│     │      │S2│     │      │S4│  │    └──┘   │ │

## Ⅱ. 다단 연결망 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                다단 연결망 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MIN 종류】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 배니언 네트워크 (Banyan):                                              │ │
│  │     • 가장 기본적인 MIN                                            │ │
│  │     • 단일 경로                                                    │ │
│  │     • 내부 차단 (blocking)                                         │ │
│  │                                                               │ │
│  │  2. 베네시 네트워크 (Benes):                                                │ │
│  │     • 배니언 + 추가 단계                                           │ │
│  │     • 다중 경로                                                    │ │
│  │     • 재배치 가능 비차단 (rearrangeably non-blocking)              │ │
│  │                                                               │ │
│  │  3. 오메가 네트워크 (Omega):                                                │ │
│  │     • 셔플-익스체인지 패턴                                         │ │
│  │     • 내부 차단                                                   │ │
│  │     • 단순한 하드웨어                                              │ │
│  │                                                               │ │
│  │  4. 나비 네트워크 (Butterfly):                                               │)
│  │     • FFT에 적합                                                   │ │
│  │     • 하이퍼큐브와 유사                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【차단 특성】                                                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 차단 (Blocking): 두 연결이 내부 링크 경합                        │ │
│  │  • 비차단 (Non-blocking): 모든 연결 동시 가능                       │ │
│  │  • 재배치 가능 비차단: 기존 연결 재배치 시 모든 연결 가능             │ │
│  │                                                               │ │
│  │  크로스바: 비차단 (비용 O(N²))                                      │ │
│  │  배니언: 차단 (비용 O(N log N))                                     │ │
│  │  베네시: 재배치 가능 비차단 (비용 O(N log N))                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 비용 효율 (O(N log N))                                           │ │
│  │  • 확장성 좋음                                                      │ │
│  │  • 로그 지연                                                        │ │
│  │  • 모듈화                                                           │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 내부 차단 가능성                                                 │ │
│  │  • 복잡한 제어                                                      │ │
│  │  • 재배치 오버헤드                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• MIN: 다단계 스위치 연결 구조
• 비용: O(N log N) (크로스바보다 효율)
• 지연: log₂N 단계
• 유형: 배니언, 베네시, 오메가, 나비
• 차단: 배니언은 차단, 베네시는 재배치 비차단
• 장점: 비용 효율, 확장성
• 단점: 내부 차단, 복잡한 제어
• 활용: 통신 스위치, 병렬 시스템
• 비교: 버스 < MIN < 크로스바
• 현대: 데이터센터 스위치 패브릭
```
