# 385. 스레드 레벨 병렬성 (TLP, Thread-Level Parallelism)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 스레드가 동시에 실행되는 병렬성
> 2. **가치**: 멀티코어 활용, 처리량 향상, 응답성 개선
> 3. **융합**: 멀티스레딩, 동기화, 락, 경합

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

스레드 레벨 병렬성(Thread-Level Parallelism, TLP)은 **여러 스레드가 동시에 실행되어 전체 시스템 성능을 향상시키는 병렬 처리 방식**이다. 멀티코어 프로세서의 핵심 활용 방법이다.

### 💡 비유: 다수의 직원

TLP는 **다수의 직원**과 같다. 직원이 여러 명이다. 동시에 일한다. 각자 다른 업무를 한다. A는 서류, B는 전화, C는 정리. 동시에 진행. 전체 업무량이 늘어난다. TLP가 똑같다. 여러 스레드, 동시 실행, 처리량 향상.

### TLP 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                TLP 개요                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLP 구조】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  단일 스레드 (Sequential):                                               │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │ 시간 →  T1   T2   T3   T4   T5   T6   T7   T8          │    │ │
│  │  │ CPU    [A ][B ][C ][D ][E ][F ][G ][H ]                │    │ │
│  │  │                                                       │    │ │
│  │  │ 하나씩 순차 실행                                      │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  다중 스레드 (TLP):                                                       │)
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │ 시간 →  T1   T2   T3   T4   T5   T6   T7   T8          │    │ │
│  │  │ Core0  [A1 ][A2 ][A3 ][A4 ]                            │    │ │
│  │  │ Core1  [B1 ][B2 ][B3 ][B4 ]  ← 동시 실행               │    │ │
│  │  │ Core2  [C1 ][C2 ][C3 ][C4 ]                            │    │ │
│  │  │ Core3  [D1 ][D2 ][D3 ][D4 ]                            │    │ │
│  │  │                                                       │    │ │
│  │  │ 4배 처리량 (이상적)                                   │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLP 유형】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 독립 스레드:                                                        │ │
│  │     • 서로 다른 작업                                               │ │
│  │     • 동기화 불필요                                                 │ │
│  │     • 예: 웹 서버 요청 처리                                         │ │
│  │                                                               │ │
│  │  2. 협력 스레드:                                                        │ │
│  │     • 공통 작업 분할                                               │ │
│  │     • 동기화 필요                                                   │ │
│  │     • 예: 병렬 행렬 연산                                            │ │
│  │                                                               │ │
│  │  3. 파이프라인 스레드:                                                   │ │
│  │     • 단계별 처리                                                   │ │
│  │     • 생산자-소비자 패턴                                            │ │
│  │     • 예: 비디오 인코딩 파이프라인                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. TLP 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                TLP 상세                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLP 활용 기법】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 멀티프로세싱 (Multiprocessing):                                     │ │
│  │     • 다중 CPU/코어에서 스레드 실행                                │ │
│  │     • 진정한 병렬 실행                                             │ │
│  │                                                               │ │
│  │  2. 멀티스레딩 (Multithreading):                                        │ │
│  │     • 단일 프로세스 내 다중 스레드                                  │ │
│  │     • 메모리 공유                                                  │ │
│  │                                                               │ │
│  │  3. 동시 멀티스레딩 (SMT/Hyper-Threading):                              │ │
│  │     • 단일 코어에서 다중 스레드                                    │ │
│  │     • 하드웨어 멀티스레딩                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLP 성능 요소】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 스레드 수: 코어 수에 맞춤                                        │ │
│  │  • 동기화: 락, 배리어, 원자 연산                                    │ │
│  │  • 부하 분산: 균등한 작업 분배                                      │ │
│  │  • 오버헤드: 스레드 생성, 동기화 비용                               │ │
│  │  • 확장성: Amdahl의 법칙                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLP 문제점】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 경합 (Contention): 락 대기                                       │ │
│  │  • 데드락 (Deadlock): 순환 대기                                     │ │
│  │  • 레이스 컨디션 (Race Condition): 동시 접근                        │ │
│  │  • 거짓 공유 (False Sharing): 캐시 라인 경합                        │ │
│  │  • 라이브락 (Livelock): 계속 재시도                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### TLP 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                TLP 분석                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스레드 수 확인】                                                              │)
│  ──────────────────                                                  │
│  $ nproc                                                            │
│ 8                                                                   │
│  # 사용 가능한 CPU 수 = 최대 병렬 스레드 수                          │
│                                                                     │
│  【스레드 성능 측정】                                                            │)
│  ──────────────────                                                  │
│  $ sysbench cpu --threads=8 --cpu-max-prime=20000 run              │
│  # 8스레드 CPU 성능                                                 │
│                                                                     │
│  【OpenMP 예시】                                                                 │)
│  ──────────────────                                                  │
│  #pragma omp parallel for                                           │
│  for (int i = 0; i < N; i++) {                                      │
│      a[i] = b[i] + c[i];  // 병렬 실행                               │
│  }                                                                  │
│                                                                     │
│  【TLP 최적화 팁】                                                                │)
│  ──────────────────                                                  │
│  1. 스레드 수: 코어 수에 맞춤                                        │
│  │  2. 동기화: 최소화                                                │ │
│  3. 락: 세분화 (fine-grained)                                       │
│  4. 부하: 균등 분배                                                  │
│  5. 측정: 프로파일링                                                 │
│  6. Affinity: CPU 바인딩 고려                                       │
└─────────────────────────────────────────────────────────────────────┘
```
