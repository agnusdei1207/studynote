# 387. 상호 연결망 (Interconnection Network)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 병렬 시스템의 노드 간 통신을 위한 네트워크 구조
> 2. **가치**: 성능, 확장성, 비용의 트레이드오프
> 3. **융합**: 토폴로지, 라우팅, 교착 방지

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

상호 연결망(Interconnection Network)은 **병렬 컴퓨팅 시스템에서 프로세서, 메모리, I/O 노드 간의 데이터 통신을 가능하게 하는 네트워크 인프라**다. 토폴로지, 라우팅, 교착 방지가 핵심 요소다.

### 💡 비유: 도로망

상호 연결망은 **도로망**과 같다. 건물들이 있다. 도로로 연결된다. 교차로가 있다. 신호등이 있다. 직선 도로, 순환 도로, 복잡한 교차로. 어디서 어디로든 갈 수 있다. 상호 연결망이 똑같다. 노드를 연결, 경로 설정, 데이터 전송.

### 상호 연결망 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                상호 연결망 개요                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【상호 연결망 구성 요소】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 노드 (Node):                                                         │ │
│  │     • 프로세서, 메모리, I/O 장치                                    │ │
│  │     • 네트워크 인터페이스 포함                                      │ │
│  │                                                               │ │
│  │  2. 링크 (Link):                                                         │ │
│  │     • 노드 간 물리적 연결                                          │ │
│  │     • 유선/무선, 직렬/병렬                                         │ │
│  │                                                               │ │
│  │  3. 스위치 (Switch):                                                      │ │
│  │     • 경로 결정                                                   │ │
│  │     • 라우팅, 포워딩                                              │ │
│  │                                                               │ │
│  │  4. 토폴로지 (Topology):                                                  │ │
│  │     • 네트워크 연결 구조                                           │ │
│  │     • 버스, 링, 메시, 하이퍼큐브                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 토폴로지】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  토폴로지        구조              장점          단점             │ │
│  │  ────────        ────              ────          ────             │ │
│  │  버스            공유 선            단순          확장성 낮음       │ │
│  │  링              순환               단순          지연 증가        │ │
│  │  메시            2D 그리드          확장성        거리 증가        │ │
│  │  토러스           링 연결 그리드     대역폭        복잡성          │ │
│  │  하이퍼큐브        n차원 큐브         짧은 거리     복잡성          │ │
│  │  크로스바         전면 연결          최고 성능     비용            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상호 연결망 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                상호 연결망 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【성능 메트릭】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 지연 (Latency):                                                       │ │
│  │     • 소스에서 목적지까지 시간                                      │ │
│  │     • 홉 수 × 홉 지연 + 직렬화 지연                                │ │
│  │                                                               │ │
│  │  2. 대역폭 (Bandwidth):                                                    │ │
│  │     • 단위 시간당 전송량                                            │ │
│  │     • 링크 대역폭 × 링크 수                                        │ │
│  │                                                               │ │
│  │  3. 이분 대역폭 (Bisection Bandwidth):                                     │ │
│  │     • 네트워크를 반으로 나눴을 때 대역폭                             │ │
│  │     • 확장성 척도                                                   │ │
│  │                                                               │ │
│  │  4. 직경 (Diameter):                                                       │ │
│  │     • 최대 홉 수                                                   │ │
│  │     • 최악의 지연                                                   │ │
│  │                                                               │ │
│  │  5. 차수 (Degree):                                                         │ │
│  │     • 노드당 링크 수                                               │ │
│  │     • 비용과 연결성 척도                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【라우팅 알고리즘】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 최단 경로: 최소 홉 수                                            │ │
│  │  • 적응형: 혼잡도 고려                                               │ │
│  │  • 차원 순서: X-Y (메시), 비트 수정 (하이퍼큐브)                     │ │
│  │  • 밸런싱: 부하 분산                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【교착 방지】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 차원 순서 라우팅: 순차적 차원 사용                                │ │
│  │  • 가상 채널: 물리적 링크를 논리적으로 분할                          │ │
│  │  • 턴 모델: 특정 턴 금지                                            │ │
│  │  • 버퍼 관리: 교착 방지 버퍼                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 상호 연결망 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                상호 연결망 × 타 과목 융합                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │

│  • 멀티코어: 코어 간 연결                                            │
│  • SMP: 공유 버스/인터커넥트                                        │
│  • NUMA: QPI/UPI                                                   │
│                                                                     │
│  【네트워크】                                                          │
│  • 토폴로지: 물리/논리 구조                                          │
│  • 라우팅: 경로 결정                                                 │
│  • 스위칭: 패킷 전달                                                 │
│  • QoS: 서비스 품질                                                 │
│                                                                     │
│  【운영체제】                                                          │
│  • 메시지 패싱: 통신 오버헤드                                        │
│  • 동기화: 배리어                                                   │
│  • NUMA: 메모리 접근                                                │
│  • I/O: 주변장치 연결                                               │
│                                                                     │
│  【성능】                                                              │
│  • 지연: 통신 시간                                                  │
│  • 대역폭: 처리량                                                   │
│  • 확장: 노드 추가                                                  │
│  • 병목: 네트워크 한계                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  토폴로지별 특성 비교:                                                            │ │
│  │  ──────────────────────                                              │ │
│  │  토폴로지   노드 수   차수   직경   이분 대역폭                         │ │
│  │  ────────   ──────   ────   ────   ──────────                         │ │
│  │  버스        N        1      1      1                                │ │
│  │  링          N        2      N/2    2                                │ │
│  │  메시 (√N)   N        4      2√N   √N                               │ │
│  │  하이퍼큐브  2^n      n      n      N/2                               │ │
│  │  크로스바    N        N-1    1      N/2                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 상호 연결망: 병렬 시스템 노드 간 통신
• 구성: 노드, 링크, 스위치, 토폴로지
• 토폴로지: 버스, 링, 메시, 하이퍼큐브
• 성능: 지연, 대역폭, 이분 대역폭, 직경
• 라우팅: 최단 경로, 적응형, 차원 순서
• 교착: 가상 채널, 턴 모델, 버퍼 관리
• NoC: 온칩 네트워크
• QPI/UPI: CPU 인터커넥트
• 선택: 워크로드, 비용, 확장성 고려
• 진화: 하이퍼스케일, AI 가속기
```
