# 391. 하이퍼큐브 (Hypercube)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: n차원 큐브 구조의 상호 연결망 토폴로지
> 2. **가치**: 짧은 직경, 높은 연결성, 이진 주소
> 3. **융합**: 로그 복잡도, 비트 라우팅, 병렬 알고리즘

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

하이퍼큐브(Hypercube)는 **n차원 큐브 형태로 노드를 배치하여, 각 노드가 n비트 이진 주소를 갖고 해밍 거리가 1인 노드들과 연결되는 상호 연결망 토폴로지**다. 로그 복잡도의 통신이 특징이다.

### 💡 비유: 다차원 체스판

하이퍼큐브는 **다차원 체스판**과 같다. 1차원은 선이다. 2차원은 정사각형이다. 3차원은 정육면체다. 4차원은? 상상하기 어렵다. 하지만 수학으로는 가능하다. 각 차원에서 점들이 연결된다. 하이퍼큐브가 똑같다. n차원, 이진 주소, 해밍 거리 1 연결.

### 하이퍼큐브 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                하이퍼큐브 개요                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하이퍼큐브 차원】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  0D (점):        •                    1개 노드                 │ │
│  │                                                               │ │
│  │  1D (선):        0 ─────── 1              2개 노드, 1 링크      │ │
│  │                                                               │ │
│  │  2D (정사각형):     00 ──── 01              4개 노드, 4 링크      │ │
│  │                    │        │                                      │ │
│  │                    │        │                                      │ │
│  │                    10 ──── 11                                     │ │
│  │                                                               │ │
│  │  3D (정육면체):        000 ──── 001          8개 노드, 12 링크     │ │
│  │                     /  │    /  │                                  │ │
│  │                   010 ──── 011  │                                  │ │
│  │                    │  100 │  │  101                                │ │
│  │                    │  /    │  /                                    │ │
│  │                   110 ──── 111                                     │ │
│  │                                                               │ │
│  │  4D+: 상상 불가, 수학적 정의                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【하이퍼큐브 특성】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  n차원 하이퍼큐브:                                                          │ │
│  │  • 노드 수: N = 2^n                                                 │ │
│  │  • 차수: n (모든 노드 동일)                                          │ │
│  │  • 직경: n = log₂N (로그 스케일!)                                    │ │
│  │  • 총 링크: n × 2^(n-1)                                             │ │
│  │  • 이분 대역폭: N/2                                                  │ │
│  │                                                               │ │
│  │  연결 규칙: 주소가 한 비트만 다른 노드들끼리 연결                        │ │
│  │  (해밍 거리 = 1)                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 하이퍼큐브 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                하이퍼큐브 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【라우팅 알고리즘】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  비트 수정 라우팅 (Bit-fixing):                                            │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │  function hypercube_route(src, dst):                   │    │ │
│  │  │      current = src                                      │    │ │
│  │  │      for i = 0 to n-1:                                 │    │ │
│  │  │          if current[i] != dst[i]:                       │    │ │
│  │  │              # i번째 비트 수정                           │    │ │
│  │  │              current = current XOR (1 << i)              │    │ │
│  │  │      return path                                        │    │ │
│  │  │                                                       │    │ │
│  │  │  예: 0000 → 1101                                       │    │ │
│  │  │      0000 → 1000 → 1100 → 1101 (3홉)                    │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  특징:                                                                   │ │
│  │  • 최대 n홉 (직경)                                                  │ │
│  │  • 교착 없음 (단조적 라우팅)                                         │ │
│  │  • 단순한 구현                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 짧은 직경 (log₂N)                                                │ │
│  │  • 높은 이분 대역폭 (N/2)                                            │ │
│  │  • 단순한 라우팅                                                    │ │
│  │  • 풍부한 경로 다양성                                                │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 차수 증가 (log₂N)                                                │ │
│  │  • 노드 수가 2의 거듭제곱으로 제한                                    │ │
│  │  • 배선 복잡도                                                       │ │
│  │  • 확장 시 전체 재구성                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【활용 사례】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • Connection Machine (CM-2): 12차원 하이퍼큐브 (4096 노드)         │ │
│  │  • nCUBE: 하이퍼큐브 기반 병렬 컴퓨터                                 │ │
│  │  • Intel iPSC: 하이퍼큐브 인터커넥트                                  │ │
│  │  • 병렬 알고리즘: FFT, 정렬, 행렬 연산                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 하이퍼큐브 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                하이퍼큐브 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하이퍼큐브 차원 계산】                                                            │)
│  ──────────────────                                                  │
│  # 노드 수 → 차원                                                   │
│  n = ceil(log2(N))                                                  │
│  # 1000 노드 → 10차원 (2^10 = 1024)                                 │
│                                                                     │
│  【해밍 거리 계산】                                                                │)
│  ──────────────────                                                  │
│  function hamming_distance(a, b):                                   │
│      return count_ones(a XOR b)                                     │
│  # 0000 XOR 1101 = 1101 → 3                                        │
│                                                                     │
│  【비트 수정 라우팅 예시】                                                            │)
│  ──────────────────                                                  │
│  src = 0000, dst = 1101                                             │
│  비트 3: 0→1 → 1000                                                 │
│  비트 2: 0→1 → 1100                                                 │
│  비트 0: 0→1 → 1101                                                 │
│  경로: 0000 → 1000 → 1100 → 1101 (3홉)                               │
│                                                                     │
│  【하이퍼큐브 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 매핑: 통신 패턴에 맞은 주소 할당                                  │
│  2. 라우팅: 비트 수정 또는 적응형                                    │
│  3. 부하: 균등 분산                                                  │
│  4. 알고리즘: 하이퍼큐브 최적화 버전                                  │
│  5. 측정: 홉 수, 지연                                               │
│  6. 확장: 2^n으로 제한                                               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 하이퍼큐브: n차원 큐브 구조
• 노드 수: 2^n (2의 거듭제곱)
• 차수: n (모든 노드 동일)
• 직경: n = log₂N (로그 스케일)
• 연결: 해밍 거리 1인 노드
• 라우팅: 비트 수정 알고리즘
• 장점: 짧은 직경, 높은 대역폭
• 단점: 차수 증가, 2^n 제한
• 활용: FFT, 정렬, 병렬 알고리즘
• 현대: 메시/토러스로 대체 추세
```
