# 분산 메모리 시스템 (Distributed Memory System)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 메모리 시스템(Distributed Memory System)은 여러 개의 프로세서(노드)가 중앙의 거대한 메모리를 공유하지 않고, **각자 자신만의 100% 독립적인 프라이빗 로컬 메모리(Local Memory)를 꽉 쥐고 있는 병렬 하드웨어 아키텍처**다.
> 2. **가치**: 아무리 많은 컴퓨터(코어)를 이어 붙여도 메모리 병목이나 캐시 일관성 충돌이 발생하지 않으므로, 성능 저하 없이 시스템을 **수만~수십만 노드까지 무한대로 확장(Scale-out)**할 수 있는 유일무이한 인프라다.
> 3. **융합**: 물리적으로 단절된 메모리 간에 데이터를 주고받기 위해, 프로그래머가 직접 통신 코드를 작성하는 **메시지 패싱 인터페이스(MPI, Message Passing Interface)**라는 강력한 소프트웨어 모델과 필연적으로 융합되어 거대 슈퍼컴퓨터 생태계를 창조했다.

+++

## Ⅰ. 분산 메모리 시스템의 개요

### 1. 정의
- 플린의 분류법 중 다중 컴퓨터(Multicomputer) 모델에 해당하며, 각 프로세서가 고유의 주소 공간(Address Space)을 가지는 램(RAM)을 탑재하고 고속 상호 연결망(Interconnection Network)을 통해 명시적으로 패킷을 주고받는 병렬 처리 아키텍처다. (일명 '약결합(Loosely Coupled) 시스템'이라 부른다.)

### 2. 등장 배경: '모여 살다 터진 병목의 대탈출'
- 공유 메모리(SMP)는 코어가 64개만 넘어가도 램(RAM)으로 가는 골목길이 미어터지고 캐시 일관성(Snooping)을 맞추다 칩이 녹아버렸다. 아키텍트들은 깨달았다. "물리법칙상 10만 개의 코어가 한 메모리를 쓸 수는 없다. 그냥 각자 집(로컬 메모리)을 하나씩 주고, 필요한 데이터가 있으면 **전화(네트워크 통신)를 해서 보내달라고 부탁**하게 만들자!" 이 철저한 분리 독립 사상이 세상을 바꿨다.

### 3. 💡 비유: '섬나라들의 무역'
- 분산 메모리 시스템은 **'바다를 사이에 두고 각자 자급자족하며 살아가는 섬나라(노드)들'**과 같다.
- 내 섬(로컬 메모리)에 있는 사과는 내 맘대로 바로 따 먹을 수 있어 엄청 빠르다. 하지만 옆 섬(다른 노드)에 있는 바나나를 먹고 싶다고 내 손을 뻗어 바로 가져올 수는 없다. 반드시 배(네트워크 패킷)를 띄워 "사과 보낼 테니 바나나 보내줘!"라고 명시적으로 무역(Message Passing)을 해야만 데이터를 얻을 수 있다.

- **📢 섹션 요약 비유**: 한 테이블에서 젓가락이 부딪히며 반찬을 뺏어 먹는 대가족 식사(공유 메모리)를 포기하고, 각자 자기 방에서 독상을 받되 필요한 반찬은 카톡으로 요청해 배달받는 철저한 개인주의 시스템입니다.

+++

## Ⅱ. 메커니즘 및 메시지 패싱 (Deep Dive)

이 시스템의 영혼은 '고립(Isolation)'과 '통신(Communication)'에 있다.

### 1. 명시적 통신 (Message Passing)
- 공유 메모리에서는 `A = B`라고 쓰면 끝이지만, 분산 메모리에서는 남의 램 주소를 절대 볼 수 없다.
- **송신자(Node 1)**: `Send(&B, Node_2);` (내 램에 있는 B를 박스에 포장해서 2번 노드로 쏴라!)
- **수신자(Node 2)**: `Receive(&A, Node_1);` (1번 노드에서 오는 박스를 받아서 내 램 A에 풀어라!)
- 이 과정은 운영체제의 네트워크 스택(또는 RDMA)을 거쳐야 하므로 통신 지연(Latency)이 수십 마이크로초에 이른다.

### 2. 분산 메모리 아키텍처 토폴로지 (ASCII)
```text
  [ Node A (192.168.0.1) ]           [ Node B (192.168.0.2) ]
  ┌─────────┬────────────┐           ┌─────────┬────────────┐
  │  CPU A  │ Local RAM A│           │  CPU B  │ Local RAM B│
  │         │ (주소 0번~)│           │         │ (주소 0번~)│
  └────┬────┴────────────┘           └────┬────┴────────────┘
       │                                  │
       │ (데이터 1GB 보냄 = Send)         │ (데이터 받음 = Recv)
       ▼                                  ▼
  ════════════════ 고속 인터커넥트 망 (InfiniBand 등) ══════════════
                            (통신 오버헤드 발생)
```
*주의: Node A의 0번지와 Node B의 0번지는 물리적으로 완전히 다른 공간이다.*

### 3. 심층 분석: 무한 확장성(Scalability)의 축복
- 공유 메모리 시스템의 하드웨어는 노드 수가 늘어날수록 통제 로직 복잡도가 $N^2$으로 커진다.
- 하지만 분산 메모리는 '내 방'만 신경 쓰면 되므로, 노드를 1대 붙이든 10만 대 붙이든 통제 로직이 복잡해지지 않는다. 스위치 포트만 넉넉하다면 레고 블록처럼 영원히 스케일아웃(Scale-out)할 수 있다. 지구상에서 **수퍼컴퓨터 TOP 500에 드는 100%의 컴퓨터가 이 분산 메모리 아키텍처를 채택**하고 있다.

- **📢 섹션 요약 비유**: 100층짜리 거대 건물을 하나 짓는 건 콘크리트 공학상 불가능에 가깝지만(공유 한계), 1층짜리 단독 주택 1만 채를 지어 마을을 만드는 건(분산 확장) 땅만 있으면 무한정 가능한 단순한 원리입니다.

## Ⅲ. 다중 컴퓨터(분산) vs 다중 프로세서(공유) 비교

| 비교 항목 | 다중 프로세서 (공유 메모리) | 다중 컴퓨터 (분산 메모리) |
|:---|:---:|:---:|
| **메모리 아키텍처** | **단일 공유 메모리 (SMP/NUMA)** | **독립된 로컬 메모리 (클러스터)** |
| **운영체제(OS)** | 1개의 OS가 전체 관리 | 노드마다 독립된 수십 개의 OS 동작 |
| **통신 방식** | 변수 읽기/쓰기 (매우 빠름) | **명시적 네트워크 패킷 전송 (상대적 느림)** |
| **확장성 (Scalability)**| 수십~수백 코어에서 한계 | **수만~수백만 노드 무한 확장 가능** |
| **캐시 일관성** | 하드웨어가 목숨 걸고 맞춰야 함 | 필요 없음 (각자 알아서 놂) |

- **📢 섹션 요약 비유**: 한 테이블에서 반찬을 공유하며 싸우는 '대가족(다중 프로세서)'과, 각자 자기 집에서 밥을 먹으며 전화로만 안부를 묻는 '독립한 형제들(다중 컴퓨터)'의 차이입니다.

+++

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: 클러스터(Cluster)와 베오울프(Beowulf)
- **상황**: 1990년대 슈퍼컴퓨터(CRAY 등)는 수백억 원을 호가하는 귀족들의 전유물이었다.
- **판단**: 가난한 대학 연구원들은 "용산에서 파는 100만 원짜리 싸구려 조립 PC(COTS: 상용 기성품) 100대를 사서, 스위치 허브에 랜선으로 엮은 뒤 리눅스와 MPI를 깔자!"라고 반란을 일으켰다. 이것이 전설적인 **베오울프 클러스터(Beowulf Cluster)**의 탄생이다. 1억 원으로 수백억 원짜리 슈퍼컴퓨터의 성능을 발라버린 이 가성비 혁명은, 오늘날 구글과 아마존 데이터센터가 값싼 x86 서버 수만 대를 엮어 클라우드를 짓게 만든 가장 위대한 아키텍처의 승리다.

### 2. 실무 시나리오: 맵리듀스(MapReduce)와 하둡(Hadoop)
- **인사이트**: 1TB짜리 텍스트 파일에서 특정 단어를 찾는다고 치자. 1대의 컴퓨터가 읽으려면 하루가 걸린다. 다중 컴퓨터 사상은 이를 100MB씩 10,000대로 조각내어(Map), 10,000대의 컴퓨터가 각자 자기 로컬 디스크에서 1초 만에 찾아낸 뒤 중앙으로 결과를 던져주면(Reduce) 끝난다. 이 다중 컴퓨터의 완벽한 물리적 격리성을 소프트웨어로 찬란하게 승화시킨 프레임워크가 바로 **하둡(Hadoop)**이며, 이는 빅데이터 시대의 문을 열어젖혔다.

- **📢 섹션 요약 비유**: 천재 교수 1명에게 산더미 같은 서류 정리를 시키는 대신, 시급을 주는 평범한 알바생 1만 명을 고용해 서류를 한 장씩 나눠주고 1분 만에 끝내버리는 현대 자본주의의 물량 공세입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **Fault Tolerance (결함 내성) 극대화**: 노드 100대가 죽어도, 마스터 노드가 그 작업을 다른 살아있는 900대에게 다시 뿌려주면 시스템은 절대 죽지 않는 영생(Immortality)의 아키텍처 달성.
- **가성비 혁명**: 고가의 커스텀 하드웨어(슈퍼컴) 없이 시장에 굴러다니는 범용 부품만으로도 전 세계 최고 성능의 시스템 구축 가능.

### 2. 미래 전망
최근 다중 컴퓨터 아키텍처는 인공지능(AI) 훈련의 유일한 해답이 되었다. H100 GPU 서버 10,000대를 인피니밴드로 엮어 만든 초거대 AI 클러스터(예: 엔비디아 슈퍼팟)가 바로 다중 컴퓨터의 끝판왕이다. 더 나아가 이 노드 간의 느린 통신(네트워크 지연)을 하드웨어 레벨에서 없애버리기 위해 RDMA 기술이 도입되면서, **"물리적으로는 다중 컴퓨터(분리)인데, 동작은 다중 프로세서(공유 메모리)처럼 하는"** 극한의 하이브리드 진화가 벌어지고 있다. 물리적 단절을 소프트웨어와 네트워크로 꿰매는 마법은 무한히 계속될 것이다.

- **📢 섹션 요약 비유**: 뿔뿔이 흩어져 살던 사람들이 통신 기술(인터넷)의 극한 발달로 인해, 마치 한 지붕 아래 사는 것처럼 실시간으로 호흡하는 완벽한 지구촌 공동체가 형성되는 과정입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **MIMD (다중 명령어 다중 데이터)**: 다중 컴퓨터를 감싸 안고 있는 플린의 분류법 상위 카테고리.
- **다중 프로세서 (Multiprocessor)**: 공유 메모리를 놓고 치열하게 다투는 영원한 라이벌이자 형제.
- **인피니밴드 / RDMA**: 분리된 1만 대의 컴퓨터가 메모리를 주고받을 때 생기는 렉을 없애준 은혜로운 통신망.

## 👶 어린이를 위한 3줄 비유 설명
1. 다중 컴퓨터는 전 세계에 흩어진 친구들 1,000명과 **'각자 자기 집 방에 앉아서 스마트폰(네트워크)으로 연락하며 거대한 마인크래프트 성을 짓는 것'**이에요.
2. 한 방에 모여서 지으면 장난감이 모자라고 서로 밟고 넘어져서 싸우지만(다중 프로세서), 자기 방에서 자기가 맡은 블록만 만들어서 보내주면 100만 명이라도 같이 놀 수 있죠.
3. 중간에 한 친구가 낮잠을 자도 나머지 999명이 계속 성을 지을 수 있어서 절대로 게임이 멈추지 않는 가장 크고 멋진 방법이랍니다!
