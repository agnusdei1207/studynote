# 369. SISD (Single Instruction, Single Data)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단일 명령어로 단일 데이터를 처리하는 순차적 아키텍처
> 2. **가치**: 단순성, 예측 가능성, 기본 참조 모델
> 3. **융합**: 폰 노이만 구조, 파이프라이닝, 직렬 처리

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SISD(Single Instruction, Single Data)는 **하나의 제어 유닛이 하나의 명령어를 인출하여, 하나의 처리 유닛이 하나의 데이터에 대해 연산을 수행하는 전통적인 컴퓨터 아키텍처**다. 플린의 분류법에서 가장 기본적인 형태다.

### 💡 비유: 1인 주방

SISD는 **1인 주방**과 같다. 요리사가 1명이다. 레시피를 하나씩 읽는다. 재료 하나를 손질한다. 요리 하나를 만든다. 순서대로 한다. 하나씩 한다. 빠르진 않지만 확실하다. SISD가 똑같다. 명령 하나, 데이터 하나, 결과 하나.

## Ⅱ. SISD 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SISD 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SISD 특징】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 구조 단순, 설계 용이                                           │ │
│  │  • 프로그래밍 모델 단순                                           │ │
│  │  • 디버깅 용이                                                    │ │
│  │  • 예측 가능한 성능                                               │ │
│  │  • 하드웨어 비용 낮음                                             │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 병렬 처리 불가                                                 │ │
│  │  • 성능 한계                                                      │ │
│  │  • 대규모 연산에 부적합                                           │ │
│  │  • 클럭 속도 의존                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SISD 성능 향상 기법】                                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 파이프라이닝 (Pipelining):                                         │ │
│  │     • 명령어 단계 중첩                                             │ │
│  │     • IPC 향상                                                    │ │
│  │                                                               │ │
│  │  2. 캐시 메모리 (Cache Memory):                                       │ │
│  │     • 메모리 지연 감소                                             │ │
│  │     • 대역폭 향상                                                  │ │
│  │                                                               │ │
│  │  3. 클럭 주파수 향상:                                                  │ │
│  │     • 더 빠른 실행                                                 │ │
│  │     • 전력/발열 문제                                               │ │
│  │                                                               │ │
│  │  4. 수퍼스칼라 (Superscalar):                                          │ │
│  │     • SISD 내부에서 다중 실행 유닛                                  │ │
│  │     • 겉보기엔 SISD, 내부는 병렬                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SISD vs 타 분류】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  SISD: 1개 명령 → 1개 데이터 (기준점)                          │ │
│  │  SIMD: 1개 명령 → N개 데이터 (데이터 병렬)                     │ │
│  │  MIMD: N개 명령 → N개 데이터 (태스크 병렬)                     │ │
│  │                                                               │ │
│  │  성능 비교 (이론적):                                                    │ │
│  │  SISD: 1x (기준)                                               │ │
│  │  SIMD: Nx (N = 데이터 수)                                       │ │
│  │  MIMD: Px (P = 프로세서 수)                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. SISD × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SISD × 타 과목 융합                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 폰 노이만: SISD 기반 구조                                         │
│  • 파이프라인: SISD 성능 향상                                        │
│  • 캐시: 메모리 병목 완화                                            │
│  • 수퍼스칼라: 내부 병렬성                                           │
│                                                                     │
│  【운영체제】                                                          │
│  • 단일 태스킹: 초기 OS                                              │
│  • 멀티태스킹: 시분할로 SISD 시뮬레이션                              │
│  • 스케줄링: 단일 CPU 관리                                           │
│  • 인터럽트: 문맥 교환                                               │
│                                                                     │
│  【프로그래밍】                                                        │
│  • 순차 프로그래밍: SISD 모델                                        │
│  • 단일 스레드: SISD 실행                                            │
│  • 디버깅: 단순 실행 흐름                                            │
│  • 알고리즘: 순차 알고리즘                                           │

│  【성능】                                                              │
│  • IPC: 파이프라인 효율                                              │

│  • MIPS: 초당 명령어                                                │

│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SISD 성능 방정식:                                                             │ │
│  │  ──────────────────────                                              │ │
│  │  실행 시간 = 명령어 수 × CPI × 클럭 주기                         │ │
│  │                                                               │ │
│  │  개선 방법:                                                            │ │
│  │  1. 명령어 수 감소 (컴파일러 최적화)                             │ │
│  │  2. CPI 감소 (파이프라인)                                       │ │
│  │  3. 클럭 주기 감소 (고클럭)                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SISD: 단일 명령, 단일 데이터
• 구조: CU + ALU + Memory
• 특징: 순차 처리, 단순 구조
• 장점: 설계 용이, 디버깅 쉬움
• 단점: 병렬 처리 불가, 성능 한계
• 향상: 파이프라인, 캐시, 클럭
• 폰 노이만: SISD 기반
• 수퍼스칼라: 내부 병렬성
• 성능식: 명령어수 × CPI × 클럭
• 기준점: 다른 분류의 비교 기준
```
