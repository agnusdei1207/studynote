# 다중 컴퓨터 (Multicomputer)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 컴퓨터(Multicomputer) 시스템은 메모리를 공유하지 않고, **각각 자신만의 CPU, 로컬 메모리, 운영체제를 가진 수십~수만 대의 독립된 컴퓨터(Node)들이 네트워크로 연결되어 하나의 작업을 처리하는 아키텍처**다.
> 2. **가치**: 메모리를 억지로 공유하려다 발생하는 버스 병목과 캐시 일관성 지옥을 완전히 회피하여, 컴퓨터를 무한대에 가깝게 이어 붙일 수 있는 **궁극의 확장성(Massive Scalability)**을 제공한다.
> 3. **융합**: 노드(Node) 간의 물리적 분리를 극복하기 위해 **메시지 패싱(Message Passing, 예: MPI)**이라는 소프트웨어 통신 기법과 융합되며, 오늘날 전 세계를 덮고 있는 **클러스터(Cluster) 및 클라우드 컴퓨팅의 물리적 실체**다.

+++

## Ⅰ. 다중 컴퓨터의 개요

### 1. 정의
- 플린의 분류법 중 MIMD에 속하며, 프로세서들이 물리적으로 분리된 자신만의 로컬 메모리(Distributed Memory)를 가지고 있어, 데이터 교환을 위해 고속 네트워크(LAN, InfiniBand 등)를 통한 명시적인 메시지 전달이 필수적인 시스템이다. (일명 '약결합(Loosely Coupled) 시스템'이라 부른다.)

### 2. 등장 배경: '한 지붕 아래선 더 이상 살 수 없다'
- 다중 프로세서(SMP) 방식은 메모리 하나를 100명이 나눠 쓰려다 보니 칩이 터질 지경이었다. 아키텍트들은 깨달았다. "물리법칙상 10만 개의 코어가 한 메모리를 쓸 수는 없다. 그냥 각자 집(로컬 메모리)을 하나씩 주고, 필요한 데이터가 있으면 **전화(네트워크 통신)를 해서 보내달라고 부탁**하게 만들자!" 이 철저한 분리 독립 사상이 세상을 바꿨다.

### 3. 💡 비유: '재택근무를 하는 글로벌 팀'
- 다중 컴퓨터는 **'전 세계에 흩어져 각자 자기 집(로컬 메모리)에서 재택근무를 하는 회사'**와 같다.
- 팀원(컴퓨터)들은 옆 사람의 모니터(메모리)를 훔쳐볼 수 없다. 만약 팀장 A가 팀원 B의 자료가 필요하면, 반드시 카카오톡이나 이메일(메시지 패싱)을 보내서 "자료 좀 보내줘"라고 명시적으로 통신해야 한다. 소통 속도는 느리지만, 사무실 크기에 구애받지 않고 전 세계 10만 명의 직원을 고용(확장)할 수 있다.

- **📢 섹션 요약 비유**: 하나의 거대한 괴물(다중 프로세서)을 만드는 것을 포기하고, 평범한 개미 수만 마리를 군단(다중 컴퓨터)으로 엮어 코끼리를 사냥하는 완벽한 인해전술입니다.

+++

## Ⅱ. 메커니즘 및 분산 메모리 구조 (Deep Dive)

이 시스템의 척추는 '약결합(Loosely Coupled)'이라는 물리적 독립성에 있다.

### 1. 약결합 (Loosely Coupled) 시스템
- 컴퓨터(Node)들은 서로의 램 주소에 `LOAD/STORE` 명령어로 직접 접근할 수 없다. `0x1000`번지라고 하면 내 컴퓨터의 램일 뿐, 남의 컴퓨터 램이 아니다.
- 따라서 데이터 일관성(Cache Coherence)을 하드웨어가 맞춰줄 필요가 전혀 없어 하드웨어 제작 비용이 미친 듯이 싸진다.

### 2. 다중 컴퓨터의 토폴로지 시각화 (ASCII)
```text
  [ Node 1 ]                [ Node 2 ]                [ Node 3 ]
  ┌─────────┬────────────┐           ┌─────────┬────────────┐           ┌─────────┬────────────┐
  │  CPU   │ Local RAM  │           │  CPU   │ Local RAM  │           │  CPU   │ Local RAM  │
  │         │ (OS 탑재)  │           │         │ (OS 탑재)  │           │         │ (OS 탑재)  │
  └────┬────┴────────────┘           └────┬────┴────────────┘           └────┬────┴────────────┘
       │                                  │                                  │
       └─────(Message Passing)────────────┼───(Message Passing)──────────────┘
                                          ▼
         ════════ 고속 네트워크 (Ethernet, InfiniBand 등) ════════
```
*핵심: 각 노드는 독립된 OS(리눅스 등)로 돌아가며, 전원도 따로 끌 수 있다. 노드 1개가 죽어도 전체 시스템은 절대 무너지지 않는다.*

### 3. 심층 분석: 메시지 패싱 인터페이스 (MPI)
- 메모리를 못 보니 프로그래머가 고생해야 한다. C언어로 짤 때 `A = B + C` 처럼 짤 수 없다. 노드 1번이 노드 2번에게 `send(data, Node_2)` 함수를 명시적으로 때리고, 노드 2번은 `receive(data, Node_1)` 함수로 받아서 덧셈을 해야 한다. 이 끔찍하고 귀찮은 통신 코딩을 표준화한 라이브러리가 바로 분산 컴퓨팅의 절대 성경인 **MPI(Message Passing Interface)**다.

- **📢 섹션 요약 비유**: 텔레파시(공유 메모리)로 생각을 나누던 초능력을 포기하고, 봉투에 편지를 정성껏 써서 우체통에 넣고 답장을 기다리는(메시지 패싱) 지독한 아날로그 소통법입니다.

+++

## Ⅲ. 다중 컴퓨터(분산) vs 다중 프로세서(공유) 비교

| 비교 항목 | 다중 프로세서 (공유 메모리) | 다중 컴퓨터 (분산 메모리) |
|:---|:---|:---|
| **메모리 아키텍처** | **단일 공유 메모리 (SMP/NUMA)** | **독립된 로컬 메모리 (클러스터)** |
| **운영체제(OS)** | 1개의 OS가 전체 관리 | 노드마다 독립된 수십 개의 OS 동작 |
| **통신 방식** | 변수 읽기/쓰기 (매우 빠름) | **명시적 네트워크 패킷 전송 (상대적 느림)** |
| **확장성 (Scalability)**| 수십~수백 코어에서 한계 | **수만~수백만 노드 무한 확장 가능** |
| **캐시 일관성** | 하드웨어가 목숨 걸고 맞춰야 함 | 필요 없음 (각자 알아서 놂) |

- **📢 섹션 요약 비유**: 한 테이블에서 반찬을 공유하며 싸우는 '대가족(다중 프로세서)'과, 각자 자기 집에서 밥을 먹으며 전화로만 안부를 묻는 '독립한 형제들(다중 컴퓨터)'의 차이입니다.

+++

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: 클러스터(Cluster)와 베오울프(Beowulf)
- **상황**: 1990년대 슈퍼컴퓨터(CRAY 등)는 수백억 원을 호가하는 귀족들의 전유물이었다.
- **판단**: 가난한 대학 연구원들은 "용산에서 파는 100만 원짜리 싸구려 조립 PC(COTS: 상용 기성품) 100대를 사서, 스위치 허브에 랜선으로 엮은 뒤 리눅스와 MPI를 깔자!"라고 반란을 일으켰다. 이것이 전설적인 **베오울프 클러스터(Beowulf Cluster)**의 탄생이다. 1억 원으로 수백억 원짜리 슈퍼컴퓨터의 성능을 발라버린 이 가성비 혁명은, 오늘날 구글과 아마존 데이터센터가 값싼 x86 서버 수만 대를 엮어 클라우드를 짓게 만든 가장 위대한 아키텍처의 승리다.

### 2. 실무 시나리오: 맵리듀스(MapReduce)와 하둡(Hadoop)
- **인사이트**: 1TB짜리 텍스트 파일에서 특정 단어를 찾는다고 치자. 1대의 컴퓨터가 읽으려면 하루가 걸린다. 다중 컴퓨터 사상은 이를 100MB씩 10,000대로 조각내어(Map), 10,000대의 컴퓨터가 각자 자기 로컬 디스크에서 1초 만에 찾아낸 뒤 중앙으로 결과를 던져주면(Reduce) 끝난다. 이 다중 컴퓨터의 완벽한 물리적 격리성을 소프트웨어로 찬란하게 승화시킨 프레임워크가 바로 **하둡(Hadoop)**이며, 이는 빅데이터 시대의 문을 열어젖혔다.

- **📢 섹션 요약 비유**: 천재 교수 1명에게 산더미 같은 서류 정리를 시키는 대신, 시급을 주는 평범한 알바생 1만 명을 고용해 서류를 한 장씩 나눠주고 1분 만에 끝내버리는 현대 자본주의의 물량 공세입니다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **Fault Tolerance (결함 내성) 극대화**: 노드 100대가 죽어도, 마스터 노드가 그 작업을 다른 살아있는 900대에게 다시 뿌려주면 시스템은 절대 죽지 않는 영생(Immortality)의 아키텍처 달성.
- **가성비 혁명**: 고가의 커스텀 하드웨어(슈퍼컴) 없이 시장에 굴러다니는 범용 부품만으로도 전 세계 최고 성능의 시스템 구축 가능.

### 2. 미래 전망
최근 다중 컴퓨터 아키텍처는 인공지능(AI) 훈련의 유일한 해답이 되었다. H100 GPU 서버 10,000대를 인피니밴드로 엮어 만든 초거대 AI 클러스터(예: 엔비디아 슈퍼팟)가 바로 다중 컴퓨터의 끝판왕이다. 더 나아가 이 노드 간의 느린 통신(네트워크 지연)을 하드웨어 레벨에서 없애버리기 위해 RDMA 기술이 도입되면서, **"물리적으로는 다중 컴퓨터(분리)인데, 동작은 다중 프로세서(공유 메모리)처럼 하는"** 극한의 하이브리드 진화가 벌어지고 있다. 물리적 단절을 소프트웨어와 네트워크로 꿰매는 마법은 무한히 계속될 것이다.

- **📢 섹션 요약 비유**: 뿔뿔이 흩어져 살던 사람들이 통신 기술(인터넷)의 극한 발달로 인해, 마치 한 지붕 아래 사는 것처럼 실시간으로 호흡하는 완벽한 지구촌 공동체가 형성되는 과정입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **MIMD (다중 명령어 다중 데이터)**: 다중 컴퓨터를 감싸 안고 있는 플린의 분류법 상위 카테고리.
- **다중 프로세서 (Multiprocessor)**: 공유 메모리를 놓고 치열하게 다투는 영원한 라이벌이자 형제.
- **인피니밴드 / RDMA**: 분리된 1만 대의 컴퓨터가 메모리를 주고받을 때 생기는 렉을 없애준 은혜로운 통신망.

## 👶 어린이를 위한 3줄 비유 설명
1. 다중 컴퓨터는 전 세계에 흩어진 친구들 1,000명과 **'각자 자기 집 방에 앉아서 스마트폰(네트워크)으로 연락하며 거대한 마인크래프트 성을 짓는 것'**이에요.
2. 한 방에 모여서 지으면 장난감이 모자라고 서로 밟고 넘어져서 싸우지만(다중 프로세서), 자기 방에서 자기가 맡은 블록만 만들어서 보내주면 100만 명이라도 같이 놀 수 있죠.
3. 중간에 한 친구가 낮잠을 자도 나머지 999명이 계속 성을 지을 수 있어서 절대로 게임이 멈추지 않는 가장 크고 멋진 방법이랍니다!
