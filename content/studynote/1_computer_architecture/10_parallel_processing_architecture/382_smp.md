# 382. SMP (Symmetric Multiprocessing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 프로세서가 동등하게 메모리와 I/O에 접근하는 대칭형 다중 처리
> 2. **가치**: 단일 OS, 프로그래밍 용이성, 부하 분산
> 3. **융합**: UMA, 캐시 일관성, 스핀락

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SMP(Symmetric Multiprocessing)는 **여러 개의 동등한 프로세서가 단일 공유 메모리와 I/O 시스템에 대칭적으로 접근하며, 하나의 운영체제가 모든 프로세서를 관리하는 병렬 컴퓨팅 아키텍처**다.

### 💡 비유: 동등한 동료

SMP는 **동등한 동료**와 같다. 팀원이 여러 명이다. 모두 같은 권한이다. 모두 같은 자료에 접근한다. 누가 부장이든 상관없다. 동등하다. 한 명이 아파도 다른 사람이 대신한다. SMP가 똑같다. 모든 CPU가 동등, 공유 자원, 단일 OS.

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SMP: 대칭형 다중 처리
• 특징: 모든 CPU 동등, 단일 OS
• 구조: 공유 메모리 + 공유 I/O
• 장점: 프로그래밍 용이, 부하 분산
• 단점: 확장성 제한, 버스 병목
• UMA: SMP는 보통 UMA 기반
• 캐시 일관성: MESI 프로토콜
• 스케줄링: OS가 자동 부하 분산
• 한계: 16~64 CPU가 일반적
• 비교: SMP vs AMP vs NUMA
```
