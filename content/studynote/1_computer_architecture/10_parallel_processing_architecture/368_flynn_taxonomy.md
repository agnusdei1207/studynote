# 368. 플린의 분류법 (Flynn's Taxonomy)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어 스트림과 데이터 스트림에 따른 병렬 처리 분류 체계
> 2. **가치**: 컴퓨터 아키텍처 이해와 분석의 기준 틀
> 3. **융합**: SISD, SIMD, MISD, MIMD 4가지 분류

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

플린의 분류법(Flynn's Taxonomy)은 **1966년 마이클 플린(Michael Flynn)이 제안한 컴퓨터 아키텍처 분류법으로, 명령어 스트림(Instruction Stream)과 데이터 스트림(Data Stream)의 수에 따라 병렬 처리 방식을 4가지로 분류**한다.

### 💡 비유: 식당 주방

플린의 분류법은 **식당 주방**과 같다. 요리사가 있다. 레시피가 있다. 재료가 있다. 요리사 1명이 레시피 1개로 재료 1개를 요리한다. SISD다. 요리사 1명이 레시피 1개로 재료 여러 개를 동시에 요리한다. SIMD다. 요리사 여러 명이 각자 레시피로 각자 재료를 요리한다. MIMD다.

### 플린의 분류법 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                플린의 분류법 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【분류 기준】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  명령어 스트림 (Instruction Stream):                          │ │
│  │  • CPU가 실행하는 명령어의 흐름                                  │ │
│  │  • 단일(Single) vs 다중(Multiple)                              │ │
│  │                                                               │ │
│  │  데이터 스트림 (Data Stream):                                  │ │
│  │  • 명령어가 처리하는 데이터의 흐름                               │ │
│  │  • 단일(Single) vs 다중(Multiple)                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【4가지 분류】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                    데이터 스트림                               │ │
│  │                    Single     Multiple                        │ │
│  │              ┌────────────┬────────────┐                      │ │
│  │   Single    │   SISD     │   SIMD     │                      │ │
│  │ 명령어      ├────────────┼────────────┤                      │ │
│  │  Multiple   │   MISD     │   MIMD     │                      │ │
│  │              └────────────┴────────────┘                      │ │
│  │                                                               │ │
│  │  SISD: Single Instruction, Single Data                        │ │
│  │  SIMD: Single Instruction, Multiple Data                      │ │
│  │  MISD: Multiple Instruction, Single Data                      │ │
│  │  MIMD: Multiple Instruction, Multiple Data                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 플린의 분류법 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                플린의 분류법 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SISD (Single Instruction, Single Data)】                                    │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  특징:                                                                  │ │
│  │  • 전통적인 단일 프로세서                                         │ │
│  │  • 순차적 실행                                                   │ │
│  │  • 폰 노이만 구조                                                │ │
│  │                                                               │ │
│  │  구조:                                                                  │ │
│  │  ┌─────────────────────────────────────────────────┐          │ │
│  │  │     CU      │     ALU     │     Memory        │          │ │
│  │  │  (제어유닛)  │   (연산유닛) │     (메모리)       │          │ │
│  │  │      ↓      │      ↓      │       ↓           │          │ │
│  │  │   1개 명령어  →   1개 데이터  →    1개 결과      │          │ │
│  │  └─────────────────────────────────────────────────┘          │ │
│  │                                                               │ │
│  │  예: 초기 CPU, 단일 코어 프로세서                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SIMD (Single Instruction, Multiple Data)】                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  특징:                                                                  │ │
│  │  • 벡터 프로세서, 배열 프로세서                                  │ │
│  │  • GPU, SIMD 확장 명령어                                        │ │
│  │  • 데이터 병렬성 활용                                           │ │
│  │                                                               │ │
│  │  구조:                                                                  │ │
│  │  ┌─────────────────────────────────────────────────┐          │ │
│  │  │     CU      │    ALU 0  │    ALU 1  │   ALU n  │          │ │
│  │  │  (제어유닛)  │           │           │          │          │ │
│  │  │      ↓      │     ↓     │     ↓     │    ↓     │          │ │
│  │  │   1개 명령어  →  데이터0   →  데이터1   → 데이터n │          │ │
│  │  └─────────────────────────────────────────────────┘          │ │
│  │                                                               │ │
│  │  예: AVX, NEON, GPU, 벡터 프로세서                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MISD (Multiple Instruction, Single Data)】                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  특징:                                                                  │ │
│  │  • 이론적 분류, 실제 거의 없음                                  │ │
│  │  • 내결함성 시스템에서 일부 활용                                 │ │
│  │  • 같은 데이터에 여러 연산 동시 수행                             │ │
│  │                                                               │ │
│  │  구조:                                                                  │ │
│  │  ┌─────────────────────────────────────────────────┐          │ │
│  │  │    CU 0    │    CU 1    │    CU n              │          │ │
│  │  │      ↓     │      ↓     │      ↓               │          │ │
│  │  │    ALU 0   │    ALU 1   │    ALU n             │          │ │
│  │  │      ↓     │      ↓     │      ↓               │          │ │
│  │  │   ──────────→  1개 데이터  ←──────────          │          │ │
│  │  └─────────────────────────────────────────────────┘          │ │
│  │                                                               │ │
│  │  예: 우주선 내결함 시스템 (다중 투표)                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MIMD (Multiple Instruction, Multiple Data)】                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  특징:                                                                  │ │
│  │  • 멀티프로세서, 멀티코어                                       │ │
│  │  • 분산 시스템, 클러스터                                        │ │
│  │  • 가장 일반적인 병렬 처리                                      │ │
│  │                                                               │ │
│  │  구조:                                                                  │ │
│  │  ┌─────────────────────────────────────────────────┐          │ │
│  │  │   CPU 0       │   CPU 1       │   CPU n        │          │ │
│  │  │  CU │ ALU     │  CU │ ALU     │  CU │ ALU      │          │ │
│  │  │   ↓    ↓      │   ↓    ↓      │   ↓    ↓       │          │ │
│  │  │  명령0 데이터0 │ 명령1 데이터1 │ 명령n 데이터n   │          │ │
│  │  └─────────────────────────────────────────────────┘          │ │
│  │                                                               │ │
│  │  예: 멀티코어 CPU, SMP, 클러스터, 분산 시스템                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 플린의 분류법 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                플린의 분류법 × 타 과목 융합                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • SISD: 단일 코어, 폰 노이만                                        │
│  • SIMD: 벡터, GPU, SIMD 확장                                        │
│  • MIMD: 멀티코어, SMP, NUMA                                         │
│  • 파이프라인: SISD에서 성능 향상                                     │
│                                                                     │
│  【운영체제】                                                          │
│  • 스케줄링: MIMD 멀티코어 관리                                       │
│  • 동기화: MIMD에서 필수                                              │
│  • SIMD: GPU 스케줄링                                                │
│  • 병렬: 프로세스/스레드 할당                                         │
│                                                                     │
│  【프로그래밍】                                                        │
│  • SIMD: 벡터화, 자동 벡터라이제이션                                  │
│  • MIMD: 멀티스레딩, MPI                                             │
│  • GPU: CUDA, OpenCL                                                │
│  • 병렬: OpenMP, Pthreads                                            │
│                                                                     │
│  【성능】                                                              │
│  • SISD: 순차 성능                                                   │
│  • SIMD: 데이터 병렬성                                               │
│  • MIMD: 태스크 병렬성                                               │
│  • 확장: 코어/유닛 추가                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  플린 분류별 특성 비교:                                                        │ │
│  │  ──────────────────────                                              │ │
│  │  분류   명령어  데이터   대표 예시       병렬성                   │ │
│  │  ────   ────    ────     ────────       ────                     │ │
│  │  SISD   1       1        단일코어        없음                     │ │
│  │  SIMD   1       N        GPU, AVX       데이터 병렬               │ │
│  │  MISD   N       1        내결함성        거의 사용 안 함           │ │
│  │  MIMD   N       N        멀티코어        태스크 병렬               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│                                                                     │
│  【SIMD 확장 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/cpuinfo | grep flags                                  │
│  flags: ... sse sse2 ssse3 sse4_1 sse4_2 avx avx2 ...              │
│  # SIMD 확장명령어 지원                                              │
│                                                                     │
│  【GPU SIMD 구조】                                                               │)
│  ──────────────────                                                  │
│  $ nvidia-smi -q | grep "Streaming Multiprocessors"                │
│  Streaming Multiprocessors: 28                                      │
│  # GPU는 수천 개의 SIMD 코어                                        │
│                                                                     │
│  【플린 분류 활용 팁】                                                            │)
│  ──────────────────                                                  │
│  1. SISD: 단순 작업, 순차 처리                                       │
│  2. SIMD: 이미지, 행렬, 벡터 연산                                    │
│  3. MIMD: 병렬 태스크, 서버 워크로드                                 │
│  4. 혼합: CPU(MIMD) + GPU(SIMD)                                     │
│  5. 측정: 병렬 성능 확인                                             │
│  6. 최적화: 워크로드에 맞는 분류 선택                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 플린 분류법: 명령어/데이터 스트림 기반 분류
• SISD: 단일 명령, 단일 데이터 (순차)
• SIMD: 단일 명령, 다중 데이터 (벡터)
• MISD: 다중 명령, 단일 데이터 (이론)
• MIMD: 다중 명령, 다중 데이터 (병렬)
• SISD 예: 단일 코어 CPU
• SIMD 예: GPU, AVX, NEON
• MIMD 예: 멀티코어, SMP
• 데이터 병렬성: SIMD 활용
• 태스크 병렬성: MIMD 활용
```
