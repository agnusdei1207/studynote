# 374. 배열 프로세서 (Array Processor)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다수의 처리 요소가 2D 그리드로 구성된 SIMD 병렬 프로세서
> 2. **가치**: 이미지 처리, 행렬 연산, 패턴 인식
> 3. **융합**: 펜티엄, ILLIAC IV, 연관 메모리

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

배열 프로세서(Array Processor)는 **다수의 처리 요소(Processing Element, PE)가 2차원 그리드 형태로 구성되어, 단일 제어 유닛의 명령을 받아 각각의 데이터에 대해 동시에 연산을 수행하는 SIMD 병렬 프로세서**다.

### 💡 비유: 군사 훈련

배열 프로세서는 **군사 훈련**과 같다. 사열장에 병사들이 있다. 2차원으로 서 있다. 교관이 명령 1개를 외친다. "총 들어!" 모든 병사가 동시에 총을 든다. 같은 명령, 각자 자세, 동시 실행. 배열 프로세서가 똑같다. PE들이 그리드로 배치, 같은 명령, 각자 데이터.

│  │                                                               │ │
│  │  1. SIMD 구조:                                                         │ │

│  │     • 각 PE는 자신의 데이터에 대해 독립 연산                       │ │

│  │  2. 데이터 병렬성:                                                      │ │
│  │     • 2D 배열/이미지를 PE에 매핑                                   │ │

│  ──────────────────                                                  │
│  // CUDA 커널 (각 스레드 = PE)                                      │
│  __global__ void imageFilter(uchar* in, uchar* out, int w, int h) {│
│      int x = blockIdx.x * blockDim.x + threadIdx.x;                │
│      int y = blockIdx.y * blockDim.y + threadIdx.y;                │
│      if (x < w && y < h) {                                          │
│          // 각 PE가 1개 픽셀 처리                                   │
│          out[y*w+x] = process(in, x, y);                            │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【배열 프로세서 최적화 팁】                                                        │)
│  ──────────────────                                                  │
│  1. 데이터 매핑: PE에 최적 배치                                     │
│  2. 통신: 인접 PE 간 데이터 공유                                    │
│  3. 마스킹: 불필요한 PE 비활성화                                    │
│  4. 메모리: 로컬 메모리 활용                                        │
│  5. 부하 균형: 균등 분배                                            │
│  6. 측정: 병렬 효율 확인                                            │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 배열 프로세서: 2D PE 그리드 SIMD
• 구조: CU + PE 그리드 + 인터커넥트
• PE: ALU + 레지스터 + 로컬 메모리
• 통신: 근접 PE (상하좌우)
• 마스킹: 조건부 PE 활성화
• 예: ILLIAC IV, CM-2, GPU
• 활용: 이미지, 행렬, 물리 시뮬
• 장점: 높은 데이터 병렬성
• 단점: 유연성 제한, 프로그래밍 복잡
• 현대: GPU가 배열 프로세서 계승
```
