# 389. 메시 토폴로지 (Mesh Topology)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 노드를 2D/3D 그리드로 배치한 인터커넥트 구조
> 2. **가치**: 확장성, 규칙적 구조, 낮은 차수
> 3. **융합**: NoC, 슈퍼컴퓨터, X-Y 라우팅

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

메시 토폴로지(Mesh Topology)는 **노드를 2차원 또는 3차원 격자(grid) 형태로 배치하고, 인접한 노드 간에만 링크로 연결하는 상호 연결망 구조**다. 확장성과 규칙성이 장점이다.

### 💡 비유: 바둑판

메시 토폴로지는 **바둑판**과 같다. 격자가 있다. 각 교차점에 노드가 있다. 상하좌우로 연결된다. 대각선은 안 된다. 바둑돌처럼 규칙적이다. 메시가 똑같다. 2D 그리드, 인접 연결, 규칙적 구조.

## Ⅱ. 메시 토폴로지 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메시 토폴로지 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【라우팅 알고리즘】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  X-Y 라우팅 (차원 순서 라우팅):                                            │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │  1. 먼저 X 방향으로 이동 (목적지 X 좌표까지)            │    │ │
│  │  │  2. 그다음 Y 방향으로 이동 (목적지 Y 좌표까지)          │    │ │
│  │  │                                                       │    │ │
│  │  │  예: N0(0,0) → N11(3,2)                              │    │ │
│  │  │      (0,0) → (1,0) → (2,0) → (3,0)                   │    │ │
│  │  │      → (3,1) → (3,2)                                 │    │ │
│  │  │                                                       │    │ │
│  │  │  장점: 교착(deadlock) 없음                             │    │ │
│  │  │  단점: 항상 최단 경로는 아님                           │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  적응형 라우팅:                                                            │ │
│  │  • 혼잡도 고려                                                   │ │
│  │  • 여러 최단 경로 중 선택                                         │ │
│  │  • 부하 분산                                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【3D 메시】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • X, Y, Z 3차원 격자                                            │ │
│  │  • 차수: 내부 6, 표면 5, 모서리 4, 꼭짓점 3                        │ │
│  │  • 직경: 3(∛N - 1)                                               │ │
│  │  • 활용: 3D 스택 칩, 최신 슈퍼컴퓨터                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 확장성 좋음                                                    │ │
│  │  • 규칙적 구조 (VLSI 용이)                                         │ │
│  │  • 낮은 차수 (적은 링크)                                           │ │
│  │  • 단순한 라우팅                                                  │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 직경 증가 (√N)                                                 │ │
│  │  • 이분 대역폭 낮음                                                │ │
│  │  • 대각선 통신 느림                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 메시 토폴로지 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메시 토폴로지 분석                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NUMA 거리 매트릭스】                                                             │)
│  ──────────────────                                                  │
│  $ numactl --hardware                                               │
│  # 메시 기반 시스템에서 노드 간 거리                                  │
│                                                                     │
│  【NoC 시뮬레이션】                                                               │)
│  ──────────────────                                                  │
│  # Booksim2, Noxim 등 시뮬레이터                                    │
│  $ ./booksim config_mesh                                            │
│                                                                     │
│  【X-Y 라우팅 의사코드】                                                            │)
│  ──────────────────                                                  │
│  function xy_route(src, dst):                                       │
│      path = []                                                       │
│      x, y = src                                                      │
│      while x != dst.x:                                               │
│          x += sign(dst.x - x)                                        │
│          path.append((x, y))                                         │
│      while y != dst.y:                                               │
│          y += sign(dst.y - y)                                        │
│          path.append((x, y))                                         │
│      return path                                                     │
│                                                                     │
│  【메시 최적화 팁】                                                                │)
│  ──────────────────                                                  │
│  1. 매핑: 통신 많은 노드 인접 배치                                   │
│  2. 라우팅: X-Y 또는 적응형                                          │
│  3. 부하: 트래픽 균등 분산                                           │
│  4. 버퍼: 가상 채널로 교착 방지                                      │
│  5. 측정: 홉 수, 지연 분석                                          │
│  6. 3D: 대규모 시 3D 메시 고려                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 메시: 2D/3D 그리드 노드 배치
• 연결: 인접 노드 상하좌우
• 차수: 내부 4, 가장자리 3, 모서리 2
• 직경: 2(√N - 1) (2D 메시)
• 라우팅: X-Y, 차원 순서, 교착 방지
• 장점: 확장성, 규칙성, 낮은 차수
• 단점: 직경 증가, 낮은 이분 대역폭
• 활용: NoC, 멀티코어, 슈퍼컴퓨터
• 3D 메시: 최신 고성능 시스템
• 비교: 메시 vs 토러스 vs 하이퍼큐브
```
