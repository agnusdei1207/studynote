# 377. 공유 메모리 시스템 (Shared Memory System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 프로세서가 단일 주소 공간을 공유하는 병렬 시스템
> 2. **가치**: 프로그래밍 용이성, 저지연 통신, 데이터 공유
> 3. **융합**: UMA, NUMA, 캐시 일관성, 동기화

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

공유 메모리 시스템(Shared Memory System)은 **여러 프로세서가 동일한 물리 메모리 주소 공간을 공유하여, 메모리 읽기/쓰기를 통해 데이터를 교환하는 병렬 컴퓨팅 아키텍처**다.

### 💡 비유: 공동 창고

공유 메모리 시스템은 **공동 창고**와 같다. 여러 직원이 있다. 같은 창고를 쓴다. 누구든 물건을 꺼낼 수 있다. 누구든 물건을 넣을 수 있다. 서로 물건을 볼 수 있다. 공유 메모리가 똑같다. 여러 CPU, 같은 메모리, 데이터 공유.

## Ⅱ. 공유 메모리 시스템 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                공유 메모리 시스템 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【공유 메모리 유형】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  UMA (Uniform Memory Access):                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │  • 모든 메모리 접근 시간 동일                             │  │ │
│  │  │  • SMP (대칭형 다중 처리)                                │  │ │
│  │  │  • 버스 기반 구조                                         │  │ │
│  │  │  • 확장성 제한                                            │  │ │
│  │  │                                                           │  │ │
│  │  │  CPU0   CPU1   CPU2   CPU3                               │  │ │
│  │  │   │      │      │      │                                  │  │ │
│  │  │   └──────┴──────┴──────┘                                  │  │ │
│  │  │         버스                                               │  │ │
│  │  │           │                                                │  │ │
│  │  │       [메모리]                                             │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  │  NUMA (Non-Uniform Memory Access):                                     │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │  • 메모리 접근 시간이 위치에 따라 다름                     │  │ │
│  │  │  • 로컬 메모리: 빠름, 원격 메모리: 느림                    │  │ │
│  │  │  • 멀티소켓 시스템                                        │  │ │
│  │  │  • 확장성 좋음                                            │  │ │
│  │  │                                                           │  │ │
│  │  │  노드0                  노드1                              │  │ │
│  │  │  [CPU0-7]←──인터──→[CPU8-15]                              │  │ │
│  │  │  [로컬mem]   커넥트  [로컬mem]                             │  │ │
│  │  │                                                           │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 일관성】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  문제: CPU0이 A를 캐시, CPU1이 A를 수정 → CPU0의 캐시 무효화      │ │
│  │                                                               │ │
│  │  해결 방법:                                                              │
│  │  • 스누핑 (Snooping): 버스 트래픽 감시                           │ │
│  │  • 디렉터리 (Directory): 중앙 집중 관리                          │ │
│  │  • MESI 프로토콜: Modified, Exclusive, Shared, Invalid           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동기화 메커니즘】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 락 (Lock): 상호 배제                                          │ │
│  │  • 뮤텍스 (Mutex): 이진 세마포어                                  │ │
│  │  • 세마포어 (Semaphore): 카운팅 동기화                           │ │
│  │  • 원자 연산: CAS, Test-and-Set                                  │ │
│  │  • 메모리 배리어: 순서 보장                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                공유 메모리 시스템 분석                                      │
├─────────────────────────────────────────────────────────────────────┤

│  【공유 메모리 IPC】                                                              │)
│  ──────────────────                                                  │

│  # 공유 메모리 세그먼트 확인                                        │

│  ──────────────────                                                  │
│  $ numactl --hardware                                               │
│  available: 2 nodes (0-1)                                           │
│  node 0 size: 65536 MB                                              │
│  node 1 size: 65536 MB                                              │
│                                                                     │
│  【NUMA 메모리 할당】                                                             │)
│  ──────────────────                                                  │
│  $ numactl --cpunodebind=0 --membind=0 ./program                   │
│  # 노드 0에 바인딩                                                  │
│                                                                     │
│  【공유 메모리 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 지역성: NUMA에서 로컬 메모리 우선                               │
│  2. 동기화: 락 경합 최소화                                          │
│  3. 거짓 공유: 캐시 라인 패딩                                       │
│  4. 원자 연산: CAS 활용                                             │
│  5. 메모리 배리어: 필요 시점에만                                    │
│  6. 측정: 프로파일링으로 병목 확인                                  │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 공유 메모리: 단일 주소 공간 공유
• 특징: 직접 통신, 캐시 일관성 필요
• UMA: 균일 메모리 접근 (SMP)
• NUMA: 비균일 메모리 접근
• 캐시 일관성: MESI, 스누핑, 디렉터리
• 동기화: 락, 뮤텍스, 원자 연산
• 장점: 프로그래밍 용이, 저지연
• 단점: 확장성 제한, 캐시 오버헤드
• 프로그래밍: Pthreads, OpenMP
• 활용: 멀티코어, SMP 서버
```
