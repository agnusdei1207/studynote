# 벡터 프로세서 (Vector Processor)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 벡터 프로세서(Vector Processor)는 데이터를 하나씩 찔끔찔끔 가져와서 계산하는 일반 CPU(스칼라)와 달리, 수십~수백 개의 데이터가 담긴 **1차원 배열(Vector)을 통째로 레지스터에 쓸어 담아 단 한 번의 기계어 명령으로 모두 병렬 처리하는 특수 아키텍처**다.
> 2. **가치**: 1970~90년대 일기예보, 유체역학, 우주 궤도 계산 등 방대한 과학 수학 계산을 요구하는 **슈퍼컴퓨터(Supercomputer, 예: CRAY-1)의 절대적인 심장**으로 군림하며 연산 속도의 이정표를 세웠다.
> 3. **융합**: 현대에는 거대한 독립형 벡터 프로세서 칩 자체는 멸종했으나, 그 영혼과 설계 사상은 고스란히 인텔의 AVX 확장 명령어, ARM의 SVE, 그리고 **엔비디아 GPU의 내부 코어 구조(SIMD)로 융합되어 부활**하였다.

---

## Ⅰ. 벡터 프로세서의 개요

### 1. 정의
- 일반적인 스칼라(Scalar, 단일 숫자) 연산이 아닌 벡터(Vector, 1차원 데이터 배열) 단위의 피연산자를 하드웨어적으로 직접 다룰 수 있도록 거대한 벡터 레지스터와 깊게 쪼개진 연산 파이프라인을 탑재한 중앙처리장치다.

### 2. 등장 배경: '루프(Loop) 오버헤드의 저주'
- 과학자들은 배열 A와 배열 B의 원소 1,000개를 더하는 시뮬레이션을 돌려야 했다. 기존 스칼라 CPU는 `1. A[0]가져옴, 2. B[0]가져옴, 3. 더함, 4. 저장, 5. i증가, 6. 조건비교, 7. JUMP`라는 7단계를 1,000번 반복했다. 정작 필요한 '덧셈'은 1번인데, 루프를 돌기 위한 잡다한 명령어(오버헤드)가 6개나 되어 컴퓨터가 너무 느렸다. "아예 하드웨어에 '이 배열 1,000개 몽땅 더해!'라는 명령어 딱 1개만 만들면 되잖아?" 이것이 크레이(Seymour Cray)가 만든 벡터 슈퍼컴퓨터의 철학이다.

### 3. 💡 비유: '바느질 장인과 거대 직조기'
- 스칼라 프로세서는 **'실과 바늘을 들고 한 땀 한 땀 옷을 꿰매는 장인'**이다. 꼼꼼하지만 넓은 천을 짜려면 평생이 걸린다.
- 벡터 프로세서는 **'수백 개의 바늘이 일렬로 꽂혀서 한 번 스위치를 누르면 1미터짜리 천을 한 번에 쾅! 하고 짜내는 거대 공장 직조기(방직기)'**다. 단순한 패턴(배열)의 옷감을 찍어내는 속도는 장인 1,000명이 모여도 이길 수 없다.

- **📢 섹션 요약 비유**: 수백 명의 손님에게 커피를 1잔씩 일일이 서빙하는 알바생을 해고하고, 버튼을 누르면 100잔의 커피가 컨베이어 벨트를 타고 동시에 쏟아지는 자판기를 설치한 셈입니다.

## Ⅱ. 메커니즘 및 2대 벡터 연산 방식 (Deep Dive)

어떻게 한 번에 수많은 데이터를 처리할까? 두 가지 방식이 역사적으로 팽팽히 맞섰다.

### 1. 메모리-메모리 (Memory-to-Memory) 아키텍처
- 데이터를 레지스터로 안 부르고, 메모리에 있는 A배열과 B배열을 쫙 빨아들여 연산한 뒤 다시 메모리 C배열에 직접 쏴버린다.
- **장점**: 레지스터 크기 제한 없이 엄청나게 긴 배열도 무한대로 연산 가능.
- **단점**: 메모리 접근 속도(Latency)가 워낙 느려서, 기껏 연산기가 빨라도 버스 병목 때문에 터져버려 결국 도태되었다. (초기 CDC STAR-100 모델)

### 2. 벡터 레지스터 (Vector-Register) 아키텍처 ★ 현대의 표준
- CPU 내부에 64개, 128개의 원소를 담을 수 있는 엄청나게 길고 거대한 **벡터 레지스터**를 수십 개 박아 넣었다. 일단 메모리에서 블록 단위로 콸콸 퍼와 레지스터를 꽉 채운 뒤, 그 안에서 빛의 속도로 파이프라인 연산을 조진다.
- **장점**: 메모리 병목 회피. CRAY-1 슈퍼컴퓨터가 이 방식을 써서 세상을 제패했고, 현대의 모든 GPU와 AVX 명령어도 100% 이 구조를 따른다.

### 3. 하드웨어 데이터 흐름 시각화 (ASCII)
수식: `V3 = V1 + V2` (V는 64칸짜리 벡터 레지스터)
```text
  [ Vector Register V1 ]          [ Vector Register V2 ]
  [1] [2] [3] ... [64]            [5] [2] [1] ... [8]
   │   │   │        │              │   │   │       │
   └─┬─┴─┬─┴──┬─────┘              └─┬─┴─┬─┴──┬────┘
     │   │    │      (Vector ALU)    │   │    │
     ▼   ▼    ▼                      ▼   ▼    ▼
    [+] [+]  [+] <─ 수십 개의 ALU가 ─> [+] [+] [+]
     │   │    │   동시에 파이프라이닝  │   │    │
     ▼   ▼    ▼                      ▼   ▼    ▼
  [6] [4] [4] ... [72]
  [ Vector Register V3 (결과 완성) ]
```

- **📢 섹션 요약 비유**: 수영장에 물을 채울 때, 수도꼭지(메모리)에서 양동이로 100번 나르는 대신, 양동이보다 수십 배 큰 소방차 물탱크(벡터 레지스터)에 물을 꽉 채워와서 한 방에 부어버리는 구조입니다.

## Ⅲ. 스칼라 vs 벡터 프로세서의 본질적 차이

| 비교 항목 | 스칼라 프로세서 (일반 CPU) | 벡터 프로세서 (슈퍼컴/GPU) |
|:---|:---|:---|
| **처리 데이터 수**| 1클럭당 **1개** (단일 변수) | 1클럭당 **N개** (1차원 배열 전체) |
| **명령어 종류** | `ADD R1, R2` | `VADD V1, V2` (V=벡터 레지스터) |
| **루프(Loop) 의존성**| 배열 덧셈에 `for`문과 분기(Jump) 필수 | **`for`문 자체가 기계어 1줄로 압축됨** |
| **명령어 인출 패널티**| 1만 번 계산하려면 1만 번 Fetch 해야 함 | 1만 번 계산할 때 딱 1번만 Fetch 함 |
| **주 사용처** | OS, 일반 응용 프로그램 (If-Else 많음) | 기상청 예측, 딥러닝 행렬, 3D 렌더링 |

- **📢 섹션 요약 비유**: 스칼라는 복잡한 교차로와 골목길을 자유롭게 빠져나가는 다목적 오토바이고, 벡터 프로세서는 장애물이 없는 사막에서 무식한 속도로 모래를 퍼 나르는 거대 불도저입니다.

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: 스트라이드(Stride)와 스캐터-개더(Scatter-Gather)
- **상황**: 배열이 메모리에 연속으로 예쁘게 모여 있으면 벡터 레지스터로 퍼오기 쉽다. 하지만 2차원 배열에서 '세로' 방향으로만 더하고 싶다면 주소가 100칸, 200칸씩 띄엄띄엄 떨어진다(Memory Stride).
- **판단**: 벡터 프로세서의 진가는 여기서 발휘된다. 일반 CPU는 띄엄띄엄한 주소를 읽을 때마다 캐시 미스가 나서 뻗어버리지만, 벡터 프로세서에는 **스캐터-개더(Scatter-Gather)** 하드웨어 유닛이 있다. 메모리에 지뢰처럼 흩어져 있는 데이터들을 하드웨어가 1나노초 만에 긁어모아(Gather) 벡터 레지스터에 빈틈없이 꽉 채워(Packing) 준다. 계산이 끝나면 다시 흩뿌려(Scatter) 저장한다. 이 기능 덕분에 과학자들은 메모리 구조 신경 안 쓰고 편하게 수학 공식을 코딩할 수 있었다.

### 2. 벡터 마스크 (Vector Mask)의 마법
- **인사이트**: 벡터 프로세서가 제일 무서워하는 건 `if` 조건문이다. 배열 64개를 동시에 더하는데, "A 원소가 0보다 클 때만 더해라"라는 조건이 있으면 병렬 기계가 멈칫한다. 
- 이를 해결하기 위해 아키텍트들은 **벡터 마스크 레지스터(Vector Mask Register)**를 만들었다. 일단 64개를 무식하게 **전부 다 더해버린다**. 그리고 조건에 맞는 놈은 `1`, 틀린 놈은 `0`이 적힌 마스크 판을 결과물 위에 덮어씌워, `0`이 적힌 자리의 계산 결과만 쓰레기통에 쿨하게 버려버린다. 조건문마저도 병렬로 씹어먹어 버리는 극도의 하드웨어 사치다.

- **📢 섹션 요약 비유**: 도장(마스크)을 파서 안 칠할 부분을 미리 가려놓고, 페인트통(벡터 연산)을 벽 전체에 사정없이 들이부어 버리면 1초 만에 원하는 모양만 깔끔하게 남는 페인트칠 기법입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **과학 기술의 스케일 점프**: 일기예보, 우주선 궤도 시뮬레이션, 핵폭발 시뮬레이션 등 몇 달이 걸리던 인류의 난제 계산을 수 시간 단위로 단축시킴.
- **파이프라이닝의 극대화**: 배열 데이터는 서로 의존성이 없는 독립 데이터라 데이터 해저드(Data Hazard)가 발생하지 않아 파이프라인의 스톨(Bubble)을 0%로 만들 수 있음.

### 2. 미래 전망
거대한 냉장고 모양의 CRAY 벡터 슈퍼컴퓨터는 1990년대 이후 가격 경쟁력에 밀려 멸종했다. 대신 값싼 범용 CPU 수만 대를 네트워크로 묶는(클러스터 컴퓨팅) 방식으로 세상이 변했다. "벡터는 죽었다"라고 모두가 생각할 때, 그래픽 처리와 인공지능(AI)이라는 시대의 파도가 덮치면서 상황은 180도 뒤집혔다. 작게 쪼개진 벡터 프로세서 수천 개가 칩 하나에 들어간 **GPGPU (NVIDIA)** 형태로 화려하게 부활했고, 최근 RISC-V는 아예 길이 제한이 없는 **RVV (RISC-V Vector Extension)**를 표준으로 내세우며 제2의 벡터 프로세서 황금기를 열어젖히고 있다.

- **📢 섹션 요약 비유**: 시대를 너무 앞서가 공룡처럼 멸종한 줄 알았던 기술이, 수십 년 뒤 조그마하고 날렵한 랩터(GPU/SIMD)로 진화해 나타나 현대 생태계(인공지능)의 최상위 포식자가 된 이야기입니다.
