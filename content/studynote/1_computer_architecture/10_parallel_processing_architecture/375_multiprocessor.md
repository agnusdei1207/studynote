# 375. 다중 프로세서 (Multiprocessor)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 CPU가 메모리를 공유하는 MIMD 병렬 시스템
> 2. **가치**: 높은 처리량, 확장성, 신뢰성
> 3. **융합**: SMP, NUMA, 캐시 일관성, 동기화

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

다중 프로세서(Multiprocessor)는 **둘 이상의 CPU가 단일 시스템에서 공유 메모리를 통해 통신하며 협력적으로 작업을 처리하는 병렬 컴퓨팅 시스템**이다. MIMD 아키텍처의 대표적 구현이다.

### 💡 비유: 합작 주방

다중 프로세서는 **합작 주방**과 같다. 요리사가 여러 명이다. 같은 주방에서 일한다. 같은 창고를 쓴다. 냉장고를 공유한다. 서로 협력한다. 한 명이 재료 꺼내면 다른 사람도 쓴다. 다중 프로세서가 똑같다. 여러 CPU, 공유 메모리, 협력 처리.

## Ⅱ. 다중 프로세서 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                다중 프로세서 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 기술】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 캐시 일관성 (Cache Coherence):                                     │ │
│  │     • 여러 CPU 캐시 간 데이터 일관성 유지                         │ │
│  │     • 스누핑/디렉터리 프로토콜                                    │ │
│  │     • MESI/MOESI 프로토콜                                        │ │
│  │                                                               │ │
│  │  2. 메모리 일관성 (Memory Consistency):                                │ │
│  │     • 메모리 접근 순서 보장                                       │ │
│  │     • 순차적/완화된 일관성 모델                                   │ │
│  │                                                               │ │
│  │  3. 동기화 (Synchronization):                                          │ │
│  │     • 락, 뮤텍스, 세마포어                                        │ │
│  │     • 원자 연산 (CAS, Test-and-Set)                              │ │
│  │                                                               │ │
│  │  4. 인터커넥트 (Interconnect):                                         │ │
│  │     • 버스, 크로스바, 메시, NoC                                  │ │
│  │     • QPI/UPI, HyperTransport                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 높은 처리량 (Throughput)                                      │ │
│  │  • 프로그래밍 용이 (단일 주소 공간)                               │ │
│  │  • 자원 공유                                                    │ │
│  │  • 신뢰성 (장애 허용)                                            │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 확장성 제한 (버스/메모리 병목)                                 │ │
│  │  • 캐시 일관성 오버헤드                                          │ │
│  │  • 동기화 비용                                                   │ │
│  │  • 복잡성                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【활용 사례】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 서버: 웹, DB, 애플리케이션 서버                               │ │
│  │  • 데이터베이스: Oracle, MySQL, PostgreSQL                      │ │
│  │  • 가상화: VMware, Hyper-V, KVM                                 │ │
│  │  • HPC: 과학 연산, 시뮬레이션                                   │ │
│  │  • 클라우드: AWS, Azure, GCP                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
