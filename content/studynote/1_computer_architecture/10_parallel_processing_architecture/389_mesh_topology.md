# 메시 (Mesh) 토폴로지

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메시(Mesh) 토폴로지는 코어(또는 노드)들을 **2차원 바둑판(Grid) 모양으로 배열하고, 각 노드를 상·하·좌·우의 인접한 4개의 이웃 노드와만 1:1로 직접 연결**하는 정적 상호 연결망 구조다.
> 2. **가치**: 칩 설계(Layout) 시 물리적으로 평면 위에 직각으로 선을 긋기 가장 완벽하고 직관적이어서, **배선 복잡도를 극단적으로 낮추고 코어 확장이 무한히 쉬운 '현대 매니코어(Many-core) 칩셋의 진정한 황금 비율'**이다.
> 3. **융합**: 거리가 멀어지면 스위치를 여러 번 건너야 하는 홉(Hop) 지연의 단점이 있으나, 최적화된 **라우팅 알고리즘(XY Routing 등)**과 결합하여 인텔 제온(Xeon) 등 최상위 서버 CPU의 칩 내부 네트워크(NoC) 표준으로 자리 잡았다.

+++

## Ⅰ. 메시 토폴로지의 개요

### 1. 정의
- $N$개의 노드가 격자무늬로 배열되어, 모서리를 제외한 내부의 모든 노드가 동, 서, 남, 북 4개의 인접 노드와 물리적인 전선(Link)으로 직접 연결되는 2차원(2D Mesh) 네트워크 아키텍처다.

### 2. 등장 배경: '크로스바와 링 버스의 한계 타파'
- 코어가 32개, 64개로 늘어나자 칩 내부 설계자들은 멘붕에 빠졌다. 크로스바 스위치를 쓰자니 선이 수백만 개로 꼬여서 칩 뚜껑이 닫히질 않았고, 링(Ring) 버스를 쓰자니 64코어가 한 줄로 원을 그리며 데이터를 돌리느라 렉(Latency)이 터졌다. 
- "잠깐, 반도체 실리콘 칩은 어차피 네모난 평면(2D)이잖아? 코어도 네모낳고! 그냥 바둑판처럼 예쁘게 깔고 **바로 옆에 있는 애들끼리만 선을 긋자!**" 이 지극히 물리적 공간 친화적인 발상이 메시 아키텍처의 탄생이다.

### 3. 💡 비유: '계획도시의 바둑판 교차로'
- 메시 구조는 맨해튼이나 바르셀로나 같은 **'계획도시의 바둑판식 도로망'**과 완벽히 일치한다.
- 101동(코어1)에서 대각선 끝 909동(코어9)에 가려면 직통 터널(크로스바)은 없다. 무조건 102동, 103동 앞 교차로(라우터)를 신호등 걸리며 거쳐서 가야 한다. 조금 돌아가긴 하지만, 도로를 까는 비용이 가장 싸고 도시를 무한정 크게 짓기도 제일 쉬운 구조다.

- **📢 섹션 요약 비유**: 수백 명의 학생들을 원형으로 빙 둘러앉히는 수건돌리기(링)를 멈추고, 시험 대형처럼 5x5 바둑판으로 쫙 줄을 세운 뒤 옆 짝꿍한테만 쪽지(패킷)를 전달하게 만든 교실입니다.

## Ⅱ. 메커니즘 및 2D 라우팅 (Deep Dive)

메시의 가장 큰 적은 '멀리 떨어진 놈에게 갈 때 거쳐야 하는 교차로(Hop)'의 개수다.

### 1. 2D Mesh 토폴로지 시각화 (ASCII)
```text
  (0,0) ─── (0,1) ─── (0,2) ─── (0,3)
    │         │         │         │
  (1,0) ─── (1,1) ─── (1,2) ─── (1,3)
    │         │         │         │
  (2,0) ─── (2,1) ─── (2,2) ─── (2,3)
  
  * (0,0)이 (2,3)으로 데이터를 보낼 때:
    (0,0) -> (0,1) -> (0,2) -> (0,3) -> (1,3) -> (2,3) 
    총 5번의 다리(Link)를 건너고 4번의 교차로(라우터)를 거쳐야(Hop) 한다.
```

### 2. XY 라우팅 (XY Routing) 알고리즘
- 메시 안에서 길을 찾는 가장 단순무식하고 확실한 방법이다.
- **원리**: 먼저 X축(가로)으로만 쭉 달려서 목적지 열(Column)을 맞춘다. 그다음 Y축(세로)으로 쭉 달려서 목적지 행(Row)을 맞춘다.
- **장점**: 길 찾기가 너무 쉬워 라우터 칩이 단순해진다. 교착 상태(Deadlock)에 절대 빠지지 않는다. 차가 좌회전, 우회전을 빙글빙글 돌다 갇히는 일을 원천 차단하는 알고리즘이다.

### 3. 심층 분석: 이등분 대역폭 (Bisection Bandwidth)
- 메시 네트워크를 가로로 쫙 찢어서 반으로 나눌 때 끊어지는 선의 개수(병목 저항력)는 $\sqrt{N}$ 에 비례한다. (16코어면 4가닥이 끊어짐).
- 크로스바 스위치나 하이퍼큐브보다는 대역폭이 낮지만, 구형 링 버스(무조건 2가닥)보다는 압도적으로 넓은 통신 대역폭을 제공하여 트래픽 정체를 시원하게 해소한다.

- **📢 섹션 요약 비유**: 옆집에 가려면 직진 1번, 뒷동네에 가려면 직진 후 우회전 1번. 길 찾기 룰이 너무나도 직관적이라 택배 기사(라우터)가 길을 잃거나 막혀서 멍때릴(데드락) 일이 없는 완벽한 동네입니다.

## Ⅲ. 토폴로지 전투: Mesh vs Ring vs Crossbar

현대 아키텍처에서 메시는 64코어급 이상의 '헤비급 챔피언'이다.

| 토폴로지 | 장점 (왜 쓰는가?) | 단점 (왜 버렸는가?) | 현대 CPU 적용 사례 |
|:---|:---|:---|:---|
| **Ring (원형)** | 선이 엄청 적고 설계가 극도로 쉽다 | 코어 10개 넘어가면 1바퀴 도는 데 하루 종일 걸림 (지연 최악) | 인텔 코어 i7 (8코어 이하 PC용) |
| **Crossbar (격자)**| 지연 시간 0 (1:1 직결) | 코어 많아지면 스위치가 기하급수적으로 폭발 (설계 불가) | 코어 내부 연산기 간의 초고속 스위칭 |
| **Mesh (바둑판)** | **평면 반도체에 찍어내기 가장 좋고 확장이 쉬움** | 대각선 멀리 갈 때 거쳐야 하는 라우터(Hop) 지연 발생 | **인텔 제온 스케일러블 (대형 서버용)** |

- **📢 섹션 요약 비유**: 작은 소모임은 둥글게 모여 앉고(Ring), 돈 많고 소수면 각자 직통 전화를 뚫고(Crossbar), 1만 명짜리 대기업은 부서를 바둑판 파티션으로 쪼개는(Mesh) 최적화의 결과물입니다.

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: 인텔 제온(Xeon)의 메쉬 아키텍처 도입
- **상황**: 인텔은 서버 CPU(브로드웰) 시절까지 구닥다리 링(Ring) 버스를 2개로 꼬아 만든 듀얼 링 버스를 썼다. 하지만 코어가 22개를 넘어가자 링 2개로도 데이터가 막혀 캐시 동기화가 박살 났다.
- **판단**: 인텔은 스카이레이크-SP (2017) 아키텍처부터 과감하게 링을 부수고 **2D 메쉬(Mesh) 토폴로지**를 서버 칩에 도입했다. 코어, L3 캐시 조각, 메모리 컨트롤러를 바둑판 교차로마다 예쁘게 하나씩 박아 넣었다. 이로 인해 대각선 코어끼리의 통신은 조금 느려졌(Hop 지연)지만, 코어를 64개까지 찍어내도 전체 시스템 대역폭이 끄떡없이 버티는 미친 스케일업(Scale-up) 능력을 손에 쥐게 되었다. 아키텍트의 결단이 서버 시장을 구원한 것이다.

### 2. 실무 시나리오: OS의 스레드 스케줄링 (Affinity) 튜닝
- **인사이트**: 메쉬 구조의 약점은 '물리적 거리가 멀면 패킷이 느리다'는 것이다. (0,0)에 있는 코어와 (3,3)에 있는 코어가 계속 메시지를 주고받는 멀티스레드 코딩을 짜면, 메쉬 도로망 한가운데로 패킷이 미친 듯이 오가며 교통체증을 유발한다(NoC Congestion).
- **해결책**: 백엔드 시스템 엔지니어는 리눅스 `taskset`을 이용해 **서로 엮여서 일하는 스레드들은 반드시 (0,0)과 (0,1)처럼 착 붙어있는 인접 코어에 강제로 묶어버린다(Pinning)**. 이렇게 하면 패킷이 단 1개의 교차로만 건너면 되므로, 네트워크 전체의 트래픽이 텅텅 비고 응답 속도는 극한으로 올라가는 '하드웨어 친화적 튜닝'이 완성된다.

- **📢 섹션 요약 비유**: 회사 사무실이 바둑판 파티션일 때, 영업부와 회계부가 맨날 회의를 하는데 끝에서 끝에 배치하면 서류 들고 복도 횡단하느라 회사가 멈춥니다. 둘을 바로 옆 파티션에 붙여주는 게 경영자(OS 스케줄러)의 센스입니다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **제조 단가의 극단적 억제**: 실리콘 다이(Die)라는 2차원 평면에 레이아웃(Layout)을 그릴 때, 선이 꼬이거나 건너뛰지 않고 최단 거리 직각 배선이 가능하여 칩 원가와 발열을 방어.
- **무한 확장성(Tiling)**: 칩셋 회사가 새로운 32코어 CPU를 만들고 싶으면, 설계도를 처음부터 다시 그릴 필요 없이 기존 16코어 메쉬 타일(Tile) 설계도를 복사-붙여넣기 해서 옆으로 2배 늘리기만 하면 끝나는 마법의 개발 사이클 제공.

### 2. 미래 전망
현재 인공지능(AI) 반도체인 구글 TPU나 신경망 가속기(NPU)의 내부 텐서 코어들은 100% 이 2D 메쉬 구조를 시스톨릭 어레이(Systolic Array) 형태로 활용한다. 데이터를 옆 사람에게 전달(Shift)하며 곱셈을 릴레이로 끝내는 데 메쉬 구조보다 완벽한 형태는 없기 때문이다. 메시는 인간의 뇌신경 구조를 향해 달려가는 매니코어(Many-core) 시대에, 가장 단순하면서도 가장 우아한 반도체 도로망의 클래식으로 영원히 살아남을 것이다.

- **📢 섹션 요약 비유**: 화려하고 복잡한 교차로를 짓다 지친 인류가, "가장 단순하고 규칙적인 바둑판이 결국 가장 완벽한 인프라다"라는 도시 계획의 진리를 반도체 위에서 다시 증명한 위대한 회귀입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **NoC (Network on Chip)**: 메쉬 토폴로지가 물리적 도로망이라면, 그 위에서 패킷을 타고 달리는 통신 사상 그 자체.
- **토러스 (Torus)**: 메쉬의 끝과 끝이 멀어서 발생하는 지연 시간을 없애고자, 바둑판 양끝을 돌돌 말아 도넛으로 만들어버린 개량형 구조.
- **크로스바 스위치**: 메쉬가 극복해야 했던 돈 먹는 하마, 비현실적 무한 직통 도로망.
