# 370. SIMD (Single Instruction, Multiple Data)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하나의 명령어로 여러 데이터를 동시에 처리하는 병렬 아키텍처
> 2. **가치**: 데이터 병렬성 활용, 미디어/과학 연산 가속

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SIMD(Single Instruction, Multiple Data)는 **하나의 명령어가 여러 데이터 요소에 대해 동시에 동일한 연산을 수행하는 병렬 처리 방식**이다. 데이터 레벨 병렬성(DLP)을 활용하여 이미지 처리, 행렬 연산, 과학 계산 등에서 고성능을 제공한다.

### 💡 비유: 벨트 컨베이어 요리

SIMD는 **벨트 컨베이어 요리**와 같다. 요리사 1명이다. 레시피 1개다. 하지만 벨트 위에 재료가 여러 개 있다. 칼질 1번에 재료 여러 개를 썬다. 같은 동작을 여러 재료에 동시에 한다. SIMD가 똑같다. 명령 1개로 데이터 여러 개를 처리한다.

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SIMD: 단일 명령, 다중 데이터
• 구조: CU + 다중 ALU + 벡터 레지스터
• 데이터 병렬성: 동일 연산 다중 데이터
• x86 확장: MMX, SSE, AVX, AVX-512
• ARM 확장: NEON, SVE
• 활용: 이미지, 오디오, 비디오, 과학
• GPU: 대규모 SIMD (SIMT)
• 성능: N배 향상 (N = 벡터 폭)
• 정렬: 데이터 정렬 필수
• 벡터화: 컴파일러 자동/수동
```
