# 386. 데이터 레벨 병렬성 (DLP, Data-Level Parallelism)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동일한 연산을 여러 데이터에 동시에 적용하는 병렬성
> 2. **가치**: 이미지/과학 연산 가속, 높은 처리량
> 3. **융합**: SIMD, 벡터 프로세서, GPU, 자동 벡터화

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

데이터 레벨 병렬성(Data-Level Parallelism, DLP)은 **동일한 연산(명령어)을 서로 다른 여러 데이터 요소에 동시에 적용하는 병렬 처리 방식**이다. SIMD 아키텍처의 기반이다.

### 💡 비유: 다수의 식탁

DLP는 **다수의 식탁**과 같다. 웨이터가 있다. 같은 메뉴를 10테이블에 서빙한다. 각 테이블이 다른 사람이다. 같은 요리를 동시에 배달한다. 한 번에 여러 테이블. DLP가 똑같다. 같은 연산, 여러 데이터, 동시 실행.

### DLP 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                DLP 개요                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【DLP 구조】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  스칼라 처리 (Scalar):                                                   │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │ A[i] = B[i] + C[i]  (i = 0 to 7)                       │    │ │
│  │  │                                                       │    │ │
│  │  │ 시간:  t1   t2   t3   t4   t5   t6   t7   t8           │    │ │
│  │  │       [0 ][1 ][2 ][3 ][4 ][5 ][6 ][7 ]  ← 8번 실행      │    │ │
│  │  │                                                       │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  DLP 처리 (SIMD):                                                        │)
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │ A[0:7] = B[0:7] + C[0:7]  (1번의 벡터 명령어)           │    │ │
│  │  │                                                       │    │ │
│  │  │ 시간:  t1                                              │    │ │
│  │  │       [0  1  2  3  4  5  6  7 ]  ← 1번에 8개 처리       │    │ │
│  │  │                                                       │    │ │
│  │  │ 8배 빠름 (이상적)                                      │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DLP 활용 분야】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 이미지 처리: 픽셀별 필터, 색상 변환                             │ │
│  │  • 오디오/비디오: 샘플 처리, 인코딩                                │ │
│  │  • 과학 연산: 행렬, 벡터 연산                                      │ │
│  │  • 딥러닝: 텐서 연산, 행렬 곱                                      │ │
│  │  • 암호화: 블록 암호, 해시                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. DLP 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                DLP 상세                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【DLP 구현 하드웨어】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. SIMD 확장 (CPU):                                                    │ │
│  │     • MMX, SSE, AVX (Intel/AMD)                                   │ │
│  │     • NEON (ARM)                                                  │ │
│  │     • 128-512비트 벡터 레지스터                                    │ │
│  │                                                               │ │
│  │  2. 벡터 프로세서:                                                        │ │
│  │     • Cray, NEC SX                                               │ │
│  │     • ARM SVE                                                     │ │
│  │     • 매우 긴 벡터 (64+ 요소)                                      │ │
│  │                                                               │ │
│  │  3. GPU:                                                                 │ │
│  │     • 수천 개의 SIMD 코어                                          │ │
│  │     • SIMT 실행 모델                                               │ │
│  │     • 대규모 데이터 병렬                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DLP 프로그래밍】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 자동 벡터화 (Auto-vectorization):                                    │ │
│  │     • 컴파일러가 자동으로 SIMD 코드 생성                            │ │
│  │     • -O3 -march=native                                          │ │
│  │                                                               │ │
│  │  2. 인트린직 (Intrinsics):                                               │ │
│  │     • SIMD 명령어 직접 사용                                        │ │
│  │     • _mm256_add_ps()                                             │ │
│  │                                                               │ │
│  │  3. 컴파일러 프래그마:                                                    │ │
│  │     • #pragma omp simd                                            │ │
│  │     • #pragma vector                                              │ │
│  │                                                               │ │
│  │  4. 라이브러리:                                                           │ │
│  │     • Intel MKL, IPP                                              │ │
│  │     • Apple Accelerate                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DLP 성능 요소】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 벡터 폭: 128/256/512비트                                        │ │
│  │  • 데이터 정렬: 32/64바이트 경계                                    │ │
│  │  • 연속 메모리: 스트라이드, 캐시                                    │ │
│  │  • 데이터 타입: float, double, int                                │ │
│  │  • 분기: 마스킹, 분기 제거                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### DLP 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                DLP 분석                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SIMD 지원 확인】                                                              │)
│  ──────────────────                                                  │
│  $ cat /proc/cpuinfo | grep flags | head -1                        │
│  flags: ... sse sse2 ssse3 sse4_1 sse4_2 avx avx2 avx512f ...       │
│                                                                     │
│  【자동 벡터화 확인】                                                              │)
│  ──────────────────                                                  │
│  $ gcc -O3 -march=native -fopt-info-vec                            │
│  # 벡터화된 루프 정보 출력                                           │
│                                                                     │
│  【AVX2 인트린직 예시】                                                            │)
│  ──────────────────                                                  │
│  #include <immintrin.h>                                             │
│  void add_arrays(float* a, float* b, float* c, int n) {             │
│      for (int i = 0; i < n; i += 8) {                               │
│          __m256 va = _mm256_load_ps(&a[i]);                         │
│          __m256 vb = _mm256_load_ps(&b[i]);                         │
│          __m256 vc = _mm256_add_ps(va, vb);                         │
│          _mm256_store_ps(&c[i], vc);                                │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【DLP 최적화 팁】                                                                │)
│  ──────────────────                                                  │
│  1. 정렬: 32/64바이트 경계                                          │
│  2. 연속: 메모리 연속 배치                                          │
│  3. 컴파일러: -O3 -march=native                                     │
│  4. 루프: 의존성 제거                                               │
│  5. 분기: 마스킹으로 대체                                           │
│  6. 측정: 벡터화 효율 확인                                          │
└─────────────────────────────────────────────────────────────────────┘
```

