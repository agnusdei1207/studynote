# 379. UMA (Uniform Memory Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 프로세서가 메모리에 동일한 접근 시간을 갖는 공유 메모리 구조
> 2. **가치**: 프로그래밍 단순성, 예측 가능한 성능
> 3. **융합**: SMP, 버스 기반, 캐시 일관성

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

UMA(Uniform Memory Access)는 **다중 프로세서 시스템에서 모든 CPU가 메모리의 모든 위치에 동일한 지연 시간과 대역폭으로 접근할 수 있는 공유 메모리 아키텍처**다. SMP(Symmetric Multiprocessing)의 기반이다.

### 💡 비유: 공동 창고

UMA는 **공동 창고**와 같다. 직원들이 여러 명이다. 모두 같은 창고를 쓴다. 어느 선반이든 같은 거리다. 어느 물건이든 같은 시간에 꺼낸다. 공평하다. UMA가 똑같다. 모든 CPU, 모든 메모리, 같은 접근 시간.

## Ⅱ. UMA 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                UMA 상세                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【UMA 구현 방식】                                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 버스 기반 (Bus-based):                                              │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │     CPU0    CPU1    CPU2    CPU3                      │    │ │
│  │  │       │       │       │       │                        │    │ │
│  │  │       └───────┴───────┴───────┘                        │    │ │
│  │  │                   공유 버스                            │    │ │
│  │  │                     │                                  │    │ │
│  │  │                 [메모리]                               │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │  • 구조 단순                                                   │ │
│  │  • 확장성 낮음 (버스 경합)                                      │ │
│  │  • 소규모 시스템                                                │ │
│  │                                                               │ │
│  │  2. 크로스바 (Crossbar):                                                │)
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │     CPU0    CPU1    CPU2    CPU3                      │    │ │
│  │  │       │       │       │       │                        │    │ │
│  │  │       ├───────┼───────┼───────┤                        │    │ │
│  │  │       │ 크로스바 스위치  │     │                        │    │ │
│  │  │       ├───────┼───────┼───────┤                        │    │ │
│  │  │       │       │       │       │                        │    │ │
│  │  │     MEM0   MEM1    MEM2    MEM3                       │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │  • 고성능                                                      │ │
│  │  • 비용 높음                                                   │ │
│  │  • 확장성 제한                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【UMA 장단점】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 프로그래밍 단순 (단일 주소 공간)                                │ │
│  │  • 성능 예측 가능                                               │ │
│  │  • 운영체제 단순                                                │ │
│  │  • 데이터 배치 고려 불필요                                      │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 확장성 제한 (버스 병목)                                        │ │
│  │  • CPU 수 증가 시 대역폭 부족                                    │ │
│  │  • 캐시 일관성 오버헤드                                          │ │
│  │  • 대규모 시스템 부적합                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【UMA 시스템 예시】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 단일 소켓 멀티코어 CPU (Intel i7, AMD Ryzen)                   │ │
│  │  • 초기 SMP 서버 (4-way까지)                                     │ │
│  │  • 임베디드 멀티코어 시스템                                       │ │
│  │  • SoC (System on Chip)                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
```

```
│  │  ────        ────                ────                               │ │

│  │  확장성      낮음                높음                               │ │
│  │  프로그래밍  단순                복잡 (지역성 고려)                  │ │
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• UMA: 균일 메모리 접근
• 특징: 모든 메모리 동일 접근 시간
• 구현: 버스 기반, 크로스바
• SMP: UMA 기반 대칭형 다중 처리
• 장점: 프로그래밍 단순, 예측 가능
• 단점: 확장성 제한, 버스 병목
• 적용: 단일 소켓 멀티코어
• 한계: 4~8코어가 일반적 한계
• 비교: UMA vs NUMA
• 현대: 대규모는 NUMA 사용
```
