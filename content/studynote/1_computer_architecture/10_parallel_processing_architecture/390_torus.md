# 390. 토러스 (Torus)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메시의 가장자리를 링으로 연결한 토폴로지
> 2. **가치**: 높은 이분 대역폭, 균일 지연, 대칭성
> 3. **융합**: 3D 토러스, 슈퍼컴퓨터, 균일 성능

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

토러스(Torus)는 **메시 토폴로지의 가장자리 노드들을 링으로 연결하여, 모든 행과 열이 순환 구조를 갖는 상호 연결망 토폴로지**다. 균일한 노드 특성과 높은 이분 대역폭이 장점이다.

### 💡 비유: 도넛

토러스는 **도넛**과 같다. 표면을 생각해 보자. 어느 방향으로든 돌아가면 제자리로 온다. 끝이 없다. 모든 점이 동등하다. 메시는 직사각형인데, 토러스는 끝을 연결했다. 토러스가 똑같다. 가장자리 연결, 순환, 균일 구조.

## Ⅱ. 토러스 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                토러스 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【토러스 특성】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  2D 토러스 (k×k):                                                        │ │
│  │  • 노드 수: N = k²                                                 │ │
│  │  • 차수: 4 (모든 노드 동일)                                          │ │
│  │  • 직경: k (메시의 절반)                                            │ │
│  │  • 이분 대역폭: 2k (메시의 2배)                                     │ │
│  │                                                               │ │
│  │  3D 토러스 (k×k×k):                                                      │ │
│  │  • 노드 수: N = k³                                                 │ │
│  │  • 차수: 6 (모든 노드 동일)                                          │ │
│  │  • 직경: 3k/2                                                       │ │
│  │  • 이분 대역폭: 2k²                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 균일한 노드 특성 (모든 노드 동등)                                │ │
│  │  • 높은 이분 대역폭 (메시의 2배)                                    │ │
│  │  • 짧은 직경 (메시의 절반)                                          │ │
│  │  • 대칭적 구조                                                      │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 복잡한 배선 (순환 링크)                                          │ │
│  │  • 더 많은 링크 (가장자리 연결)                                     │ │
```

## Ⅳ. 실무 적용

### 토러스 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                토러스 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【토러스 라우팅】                                                                │)
│  ──────────────────                                                  │
│  # 2D 토러스에서 최단 경로 계산                                      │
│  function torus_route(src, dst, k):                                 │
│      dx = min((dst.x - src.x) % k, (src.x - dst.x) % k)             │
│      dy = min((dst.y - src.y) % k, (src.y - dst.y) % k)             │
│      return dx + dy                                                 │
│                                                                     │
│  【토러스 주소 계산】                                                              │)
│  ──────────────────                                                  │
│  # 노드 ID → (x, y) 좌표                                            │
│  x = node_id % k                                                    │
│  y = node_id // k                                                   │
│                                                                     │
│  【토러스 장점 확인】                                                              │)
│  ──────────────────                                                  │
│  # 모든 노드가 동일한 특성                                           │
│  # 가장자리와 내부 노드 차이 없음                                    │
│                                                                     │
│  【토러스 최적화 팁】                                                                │)
│  ──────────────────                                                  │
│  1. 매핑: 통신 패턴에 맞은 배치                                      │
│  2. 라우팅: 순환 고려한 최단 경로                                    │
│  3. 부하: 균등 분산                                                  │
│  4. 3D: 대규모 시 3D 토러스                                         │
│  5. 측정: 이분 대역폭 활용                                           │
│  6. 비용: 메시와 트레이드오프                                        │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 토러스: 가장자리 연결된 메시
• 특징: 순환 구조, 균일 노드
• 차수: 모든 노드 동일 (2D: 4, 3D: 6)
• 직경: √N (메시의 절반)
• 이분 대역폭: 2√N (메시의 2배)
• 장점: 균일성, 높은 대역폭
• 단점: 배선 복잡, 링크 증가
• 활용: 슈퍼컴퓨터, 대규모 시스템
• 3D 토러스: 현대 고성능 시스템
• 비교: 메시 vs 토러스
```
