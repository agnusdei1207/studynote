# 토러스 (Torus)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 토러스(Torus) 토폴로지는 2차원 바둑판 모양의 메시(Mesh) 네트워크에서 **가장자리 끝에 있는 노드들을 서로 빙 돌려(Wrap-around) 묶어버림으로써 '도넛 모양(Torus)'의 끝없는 연결망을 만든 아키텍처**다.
> 2. **가치**: 일반 메시 구조에서 치명적이었던 "끝에서 반대편 끝으로 갈 때 너무 멀다"는 **지름(Diameter) 문제를 절반으로 싹둑 잘라버려**, 극단적인 통신 지연(Latency) 단축을 이뤄냈다.
> 3. **융합**: 모서리나 끝부분이 존재하지 않는 완벽한 대칭성(Symmetry)을 갖추어, 트래픽이 한가운데로 몰리는 병목 현상을 해소하며 IBM BlueGene이나 후가쿠(Fugaku) 같은 **글로벌 최상위 슈퍼컴퓨터의 표준 네트워크로 융합**되었다.

---

## Ⅰ. 토러스 토폴로지의 개요

### 1. 정의
- K-ary N-cube 구조의 일종으로, 메쉬(Mesh) 네트워크의 각 행(Row)과 열(Column)의 끝과 끝 노드를 환형(Ring)으로 연결하여, 네트워크 내에 경계(Edge)가 존재하지 않는 다차원 연결망 구조다. (도넛 형태를 띰)

### 2. 등장 배경: '모서리 주민들의 억울함 해소'
- 평면 바둑판 모양인 메시(Mesh) 망에서 중앙에 있는 코어들은 길이 4방향으로 뻥 뚫려 있어서 통신하기 좋았다. 하지만 구석(모서리)에 처박힌 코어는 길이 2개밖에 없었고, 저 반대쪽 구석으로 데이터를 보내려면 대륙을 횡단하듯 한참을 거쳐 가야 했다(지름이 긺). 아키텍트들은 "지구를 둥글게 말면 동쪽 끝에서 서쪽 끝이 바로 붙잖아? 바둑판을 종이접기하듯 둥글게 말아버리자!"라며 랩어라운드(Wrap-around) 선을 추가해 토러스를 탄생시켰다.

### 3. 💡 비유: '팩맨(Pac-Man) 게임의 화면 밖 이동'
- 토러스는 고전 게임 **'팩맨(Pac-Man)의 둥근 미로 맵'**과 완벽히 똑같다.
- 팩맨이 화면의 오른쪽 끝 구멍으로 쏙 들어가면, 막히는 게 아니라 마법처럼 화면의 왼쪽 끝에서 뿅 하고 튀어나온다. 메시 구조였다면 막힌 벽이라 다시 돌아서 가야 하지만, 토러스는 공간이 둥글게 이어져 있어 끝과 끝이 사실상 '바로 옆집'이 되는 기적의 지름길을 제공한다.

- **📢 섹션 요약 비유**: 평면의 지도를 들고 동쪽 끝에서 서쪽 끝으로 배를 타던 시대(메시)에서, 지구가 둥글다는 것을 깨닫고 서쪽으로 직진해 동쪽으로 1초 만에 도착하는 대항해시대(토러스)의 도래입니다.

## Ⅱ. 메커니즘 및 3D 확장 (Deep Dive)

선 몇 개를 더 이었을 뿐인데 네트워크의 수학적 특성이 완전히 뒤바뀐다.

### 1. 2D Mesh vs 2D Torus 구조 비교 (ASCII)
```text
   [ 2D Mesh (구석이 단절됨) ]           [ 2D Torus (끝이 이어짐) ]
   
   O ── O ── O ── O                 ┌── O ── O ── O ── O ──┐
   │    │    │    │                 │   │    │    │    │   │
   O ── O ── O ── O                 ├── O ── O ── O ── O ──┤
   │    │    │    │                 │   │    │    │    │   │
   O ── O ── O ── O                 └── O ── O ── O ── O ──┘
   
 * 토러스에서는 왼쪽 끝 노드에서 오른쪽 끝 노드로 갈 때 1번만 건너면(Hop=1) 됨!
```

### 2. 토러스의 3대 수학적 장점
| 평가 지표 | 2D Mesh (가로 $N \times$ 세로 $N$) | 2D Torus |
|:---|:---|:---|
| **지름 (Diameter, 최대 거리)**| $2(N - 1)$ | **$N$ (딱 절반으로 뚝 떨어짐!)** |
| **대칭성 (Symmetry)** | 중앙은 좋고 구석은 나쁨 | **모든 노드가 완벽히 평등함 (구석이 없음)** |
| **이등분 대역폭 (반 가르기)**| 약함 ($N$ 가닥 끊어짐) | **2배 강함 ($2N$ 가닥이 끊어져야 반갈죽 됨)** |

### 3. 심층 분석: 라우팅 데드락(Deadlock)의 위험성
- 토러스는 둥글게 원형(Ring)으로 선이 이어져 있다. 이 때문에 4개의 패킷이 원을 그리며 서로 "네가 비키면 내가 갈게"라고 무한 대기하는 교착 상태(Deadlock)에 빠지기 엄청나게 쉽다. 이를 막기 위해 가상 채널(Virtual Channel)이라는 가짜 논리적 도로를 소프트웨어적으로 여러 겹 깔아주어 차들이 비켜 갈 수 있는 입체 교차로를 만들어야 하는 치명적인 하드웨어 설계 오버헤드가 발생한다.

- **📢 섹션 요약 비유**: 순환 고속도로를 뚫어서 뺑 돌아갈 일을 없앴지만, 순환선 특유의 꼬리물기 정체가 터지면 차가 한 대도 못 움직이는 끔찍한 병목이 터지기 때문에 똑똑한 교통경찰(가상 채널)을 많이 세워둬야 합니다.

## Ⅲ. 3D 토러스 (3D Torus)와 슈퍼컴퓨터

토러스는 2차원에 머물지 않고 3차원, 6차원으로 진화한다.

### 1. 3차원(3D) 토러스의 마법
- $X, Y$ 평면으로 도넛을 만든 것에 그치지 않고, 층(Z축)을 쌓아 올려 Z축의 맨 꼭대기 층과 맨 바닥 층마저 선으로 묶어버린다. (마치 꽈배기가 꼬여있는 4차원 입체 도형 형태)
- 이렇게 하면 수만 대의 컴퓨터를 엮어도 어떤 컴퓨터든 대여섯 번의 점프(Hop)만으로 전 세계 모든 노드에 도달할 수 있는 신의 물류망이 탄생한다.

### 2. 슈퍼컴퓨터의 절대 지배자
- 현존하는 전 세계 슈퍼컴퓨터 1, 2위를 다투는 일본의 **'후가쿠(Fugaku)'**는 Tofu(Torus fusion)라는 무려 **6차원(6D) 메쉬/토러스 하이브리드** 네트워크를 쓴다.
- 과거 전설의 슈퍼컴퓨터 IBM **BlueGene/L** 역시 3D 토러스 구조로 수십만 개의 CPU 코어를 랙(Rack)에 욱여넣고도 지연 시간 없이 완벽한 우주 시뮬레이션을 돌려냈다. 극단적인 확장성은 오직 토러스만이 감당할 수 있다.

- **📢 섹션 요약 비유**: 지구본(2D 토러스)을 넘어 우주의 시공간을 휘어버리는 웜홀(3D/6D 토러스)을 수만 개 뚫어, 우주 끝에서 끝으로 1초 만에 워프하는 초공간 이동 기술입니다.

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: Mesh vs Torus, 왜 CPU 안에는 Torus가 없을까?
- **상황**: 토러스가 지름도 짧고 대역폭도 메쉬보다 2배 좋은데, 왜 인텔 제온(Xeon) 코어 내부 칩(NoC)은 토러스를 안 쓰고 메쉬(Mesh)를 쓸까?
- **판단**: **"긴 전선(Long Wire)의 딜레마"** 때문이다. 실리콘 칩 평면에 토러스를 그리려면, 끝에서 반대편 끝으로 연결하는 **랩어라운드(Wrap-around)** 전선을 칩 전체를 가로질러 아주 길게 깔아야 한다. 반도체에서 전선이 길어지면 저항이 커져 신호가 느려지고 전기를 미친 듯이 퍼먹는다. 슈컴처럼 랙(서버 박스) 단위에서 광케이블로 묶는 건 괜찮지만, 나노미터(nm) 단위의 칩 안에서 긴 구리선을 까는 건 물리법칙상 자살 행위다. 아키텍트는 이론상의 우월함(Torus)을 포기하고, 물리적인 제조의 합리성(Mesh)을 선택하는 타협의 미학을 발휘했다.

### 2. 실무 시나리오: 폴디드 토러스 (Folded Torus) 레이아웃
- **인사이트**: 어떻게든 긴 전선을 피하면서 토러스를 만들고 싶은 아키텍트들은 **폴디드 토러스(접힌 토러스)**라는 천재적인 배선 레이아웃을 고안했다.
- 노드 1, 2, 3, 4를 순서대로 `1-2-3-4`로 배치하고 `1`과 `4`를 길게 잇는 대신, 노드 배치를 `1-3-4-2` 순서로 지그재그로 섞어버린다. 이렇게 배치하고 선을 이으면 모든 선의 길이가 동일하게 짧아지면서 논리적으로 완벽한 토러스 링이 칩 안에 구축된다. 하드웨어 엔지니어의 기하학적 퍼즐 풀이 능력이 돋보이는 걸작이다.

- **📢 섹션 요약 비유**: 긴 끈을 끝과 끝으로 이어 묶으려니 끈이 덜렁거려 거추장스럽자, 끈을 아코디언처럼 지그재그로 접어서 묶음으로써 길이를 반으로 쫙 줄여버린 기발한 포장술입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **병목 지점(Hotspot) 0% 달성**: 메시 네트워크 중앙 교차로에 트래픽이 쏠려 터지는 병목을, 트래픽을 외곽으로 둥글게 분산시킴으로써 시스템 전체의 평균 통신 지연(Average Latency)을 극적으로 낮춤.
- **결함 허용(Fault Tolerance)**: 경로가 둥글게 순환하므로, 특정 스위치나 링크가 끊어져도 무조건 반대 방향으로 돌아서 목적지에 도착할 수 있는 100% 생존형 우회로 보장.

### 2. 미래 전망
거대 클러스터 시장에서는 수십만 대의 노드를 효율적으로 엮어야 하므로, 토러스는 팻 트리(Fat Tree)나 드래곤플라이(Dragonfly) 같은 차세대 토폴로지들과 치열하게 패권을 다투고 있다. 토러스는 배선이 규칙적이고 3D 공간(데이터센터 랙 배열)에 딱 들어맞는 기하학적 장점이 압도적이기 때문에, 양자 컴퓨터의 큐비트 배열이나 인간의 뇌신경을 모방하는 뉴로모픽 컴퓨팅의 3차원 입체 연결망 설계에서도 가장 유력한 베이스 뼈대로 부활할 준비를 하고 있다.

- **📢 섹션 요약 비유**: 평평한 땅(Mesh)에 살다 한계를 깨닫고 둥근 행성(Torus)을 발견한 인류가, 이제는 4차원 우주(6D Torus)를 개척하며 데이터 통신의 한계를 끝없이 돌파하고 있는 위대한 여정입니다.
