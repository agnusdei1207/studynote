# NAND 플래시 (NAND Flash)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: NAND 플래시는 NAND 게이트 구조를 기반으로 한 **고밀도 비휘발성 메모리**로, 전원이 꺼져도 데이터를 유지하며 **페이지 단위 쓰기, 블록 단위 삭제** 특성을 가진다.
> 2. **가치**: NOR 플래시 대비 **면적 효율이 높고 비용이 낮아** SSD, USB, eMMC, 스마트폰 저장공간 등 대용량 스토리지의 핵심 소자로 자리 잡았다.
> 3. **융합**: XOR-like 특성으로 인해 **쓰기 전 삭제(Erase Before Write)**가 필요하며, 마모 평준화(Wear Leveling), 가비지 컬렉션(GC), FTL 등의 소프트웨어 기술과 결합하여 신뢰성을 확보한다.

+++

## Ⅰ. NAND 플래시의 개요

### 1. 정의
- NAND 플래시는 플로팅 게이트(Floating Gate) 또는 차지 트랩(Charge Trap) 구조를 사용하여 전자를 가둬두는 방식으로 데이터를 저장하는 비휘발성 메모리다. NAND 게이트처럼 셀이 직렬로 연결된 구조를 가진다.

### 2. 등장 배경: 대용량 스토리지의 필요성
- 하드디스크(HDD)는 기계적 한계로 속도와 내구성에 제약이 있었다.
- NOR 플래시는 빠르지만 밀도가 낮아 대용량에 부적합했다.
- NAND는 **"밀도를 높이고 속도를 조금 포기하자"**는 트레이드오프로 등장하여 SSD 혁명을 주도했다.

### 3. 💡 비유: '주차장의 연결된 주차 공간'
- NAND 플래시은 **'주차 공간들이 일렬로 촘촘하게 연결된 주차장'**과 같다.
- 차(데이터)를 세울 때는 빈 자리에 바로 들어갈 수 있지만, 이미 차가 있는 자리에 다른 차를 세우려면 먼저 기존 차를 빼내야 한다(삭제 후 쓰기). 그리고 한 줄(블록) 단위로만 차를 다 빼낼 수 있다.

- **📢 섹션 요약 비유**: 아파트에 비유하면, 방(페이지)마다 입주(쓰기)는 자유롭지만, 퇴실(삭제)은 한 층(블록) 전체를 비워야 하는 구조입니다.

## Ⅱ. 구조 및 동작 원리 (Deep Dive)

### 1. NAND 셀 구조
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │               NAND 플래시 셀 어레이 구조 (간소화)                 │

    │                                                                 │
    │    Bit Line (BL)                                                │
    │         │                                                       │
    │    ┌────┴────┬────┬────┬────┬────┬────┐                        │
    │    │         │    │    │    │    │    │                        │
    │   (CG)      (CG) (CG) (CG) (CG) (CG)  ← 컨트롤 게이트           │
    │    │         │    │    │    │    │    │                        │
    │   (FG)      (FG) (FG) (FG) (FG) (FG)  ← 플로팅 게이트 (전자 저장)│

    │   │ │       │ │  │ │  │ │  │ │  │ │   ← 채널 (직렬 연결)       │
    │    │         │    │    │    │    │                             │

    │   * 32~64개의 셀이 직렬로 연결됨 (NAND 구조)                    │
    │   * 중간에 있는 셀의 데이터를 읽으려면 나머지를 통과해야 함      │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 읽기/쓰기/삭제 동작
- **읽기(Read)**: 페이지 단위 (4KB~16KB). 전하 상태 감지.
- **쓰기(Programming)**: 페이지 단위. 0은 전자 주입, 1은 유지 (기본이 1).
- **삭제(Erase)**: **블록 단위** (수백 페이지). 전체를 1 상태로 리셋.
- **핵심 제약**: 이미 쓰여진 페이지는 삭제 없이 덮어쓸 수 없음.

### 3. SLC/MLC/TLC/QLC 구분
| 유형 | 비트/셀 | 내구성(PE 사이클) | 밀도 | 용도 |
|:---|:---:|:---:|:---:|:---|
| **SLC** (Single) | 1 | ~100,000 | 낮음 | 산업용, 서버 |
| **MLC** (Multi) | 2 | ~3,000 | 중간 | 일반 SSD |
| **TLC** (Triple) | 3 | ~1,000 | 높음 | 소비자 SSD |
| **QLC** (Quad) | 4 | ~100 | 매우 높음 | 읽기 중심 스토리지 |

- **📢 섹션 요약 비유**: 방 하나에 사람 1명만 들어가면 비싸지만 오래 쓰고(SLC), 4명씩 들어가면 싸지만 금방 닳는다(QLC)입니다.

## Ⅲ. NAND 플래시 vs NOR 플래시 비교

| 비교 항목 | NAND 플래시 | NOR 플래시 |
|:---|:---|:---|
| **구조** | 셀 직렬 연결 | 셀 병렬 연결 |
| **밀도** | 높음 | 낮음 |
| **읽기 속도** | 느림 (순차는 빠름) | 빠름 (랜덤 접근) |
| **쓰기/삭제 속도** | 빠름 | 느림 |
| **XIP (Execute In Place)** | 불가능 | 가능 |
| **주 용도** | SSD, USB, SD 카드 | BIOS, 펌웨어 저장 |

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: P/E 사이클과 수명
- NAND 플래시은 삭제(Erase)와 쓰기(Program)가 반복될수록 산화막이 열화되어 수명이 다했다.
- **Wear Leveling**: 모든 블록이 균등하게 사용되도록 분산.
- **Over-provisioning**: 실제 용량보다 많은 여유 공간 확보하여 수명 연장.

### 2. 안티패턴: 랜덤 쓰기 남용
- NAND는 순차 쓰기에 최적화되어 있다. 랜덤 쓰기는 가비지 컬렉션 부담 증가 → 성능 저하 → 수명 단축.
- 로그 구조 파일 시스템(LFS), FTL의 로그 블록 방식으로 완화.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **HDD 대비 100배 빠른 접근 속도**: 랜덤 IOPS 수만 ~ 수십만.
- **무소음, 저전력, 내충격성**: 모바일 기기와 데이터센터에 필수.

### 2. 미래 전망
- **3D NAND**: 셀을 수직으로 적층(128Layer, 232Layer...)하여 밀도 극대화.
- **PLC (Penta Level Cell)**: 5비트/셀 연구 중 (밀도 ↑, 내구성 ↓).
