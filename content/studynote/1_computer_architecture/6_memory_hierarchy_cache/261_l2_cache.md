# 261. L2 캐시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: L1과 L3 사이의 중간 계층 캐시
> 2. **가치**: L1 미스 처리, 큰 워킹셋 지원
> 3. **융합**: 코어 전용 vs 공유, 유니파이드 캐시

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

L2 캐시(Level 2 Cache)는 **L1 캐시와 L3 캐시(또는 주 메모리) 사이에 위치한 2차 캐시**이다. L1보다 크고 느리지만, 더 많은 데이터를 저장할 수 있어 L1 미스를 효과적으로 처리한다.

### 💡 비유: 책상 옆 책장

L2 캐시는 **책상 옆 책장**과 같다. 책상(L1)에 없으면 책장에서 찾는다. 책장은 책상보다 멀지만 많은 책을 담을 수 있다. 자주 쓰는 책은 책장에도 비치해둔다.

```

│                L2 캐시 개요                                               │
├─────────────────────────────────────────────────────────────────────┤

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │   CPU 코어 0                    CPU 코어 1                      │ │
│  │  ┌───────────────┐            ┌───────────────┐                │ │
│  │  │ L1 I-Cache    │            │ L1 I-Cache    │                │ │
│  │  │ L1 D-Cache    │            │ L1 D-Cache    │                │ │
│  │  │ 64KB (Split)  │            │ 64KB (Split)  │                │ │
│  │  └───────┬───────┘            └───────┬───────┘                │ │
│  │          │                             │                         │ │
│  │    ┌─────┴─────┐                 ┌─────┴─────┐                  │ │
│  │    │  L2 캐시   │                 │  L2 캐시   │                  │ │
│  │    │ 256KB~1MB  │                 │ 256KB~1MB  │                  │ │
│  │    │ (Unified)  │                 │ (Unified)  │                  │ │
│  │    └─────┬─────┘                 └─────┬─────┘                  │ │
│  │          │                             │                         │ │
│  │          └──────────────┬──────────────┘                         │ │
│  │                         │                                        │ │
│  │                 ┌───────┴───────┐                                │ │
│  │                 │    L3 캐시     │                                │ │
│  │                 │   8~64MB      │                                │ │
│  │                 │  (공유)        │                                │ │
│  │                 └───────────────┘                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【L2 캐시 특징】                                                               │
│  ──────────────────                                                  │
│  • 크기: 256KB ~ 2MB (L1의 4~32배)                                    │
│  • 지연: 3~5 클럭 (약 2~5ns)                                          │
│  • 대역폭: 수백 GB/s                                                  │
│  • 연관도: 8~16 way Set Associative                                    │
│  • 라인 크기: 64B                                                     │
│  • 유형: Unified Cache (명령어 + 데이터)                                │
│  • 적중률: 95~98%                                                     │
│                                                                     │
│  【L2 캐시 배치 방식】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Private L2 (코어 전용):                                        │ │
│  │     • 각 코어가 독자적인 L2 보유                                     │ │
│  │     • 빠른 접근, 높은 일관성                                         │ │
│  │     • 예: Intel, AMD                                              │ │
│  │                                                               │ │
│  │  2. Shared L2 (코어 공유):                                          │ │
│  │     • 여러 코어가 하나의 L2 공유                                      │ │
│  │     • 효율적 공간 활용, 통신 가능                                      │ │
│  │     • 예: 멀티코어 ARM                                              │ │
│  │                                                               │ │
│  │  3. Cluster L2 (클러스터 공유):                                     │ │
│  │     • 일부 코어 그룹이 L2 공유                                        │ │
│  │     • big.LITTLE 아키텍처                                           │ │
│  │     • 예: Apple, ARM                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. L2 캐시 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                L2 캐시 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【L2 캐시 동작】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. L1 미스 발생                                                    │ │
│  │  2. L2 캐시 조회                                                    │ │
│  │  3. L2 적중:                                                        │ │
│  │     - 데이터를 L1로 전송                                             │ │
│  │     - L1 캐시 라인 할당                                              │ │
│  │  4. L2 미스:                                                        │ │
│  │     - L3 또는 메모리에서 데이터 가져옴                                 │ │
│  │     - L2, L1에 데이터 저장                                           │ │
│  │                                                               │ │
│  │  Latency:                                                           │ │
│  │  L1 적중: ~1ns                                                       │ │
│  │  L2 적중: ~4ns (L1의 4배)                                            │ │
│  │  L3 적중: ~15ns (L1의 15배)                                          │ │
│  │  DRAM: ~100ns (L1의 100배)                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Inclusive vs Exclusive】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Inclusive (포함):                                                  │ │
│  │  • L2가 L1의 모든 데이터 포함                                         │ │
│  │  • L3가 L2의 모든 데이터 포함                                         │ │
│  │  • 장점: 캐시 일관성 관리 쉬움                                        │ │
│  │  • 단점: 중복 저장으로 공간 낭비                                       │ │
│  │  • 예: Intel                                                        │ │
│  │                                                               │ │
│  │  Exclusive (배타):                                                     │ │
│  │  • 각 계층이 서로 다른 데이터 저장                                      │ │
│  │  • 데이터는 한 계층에만 존재                                           │ │
│  │  • 장점: 공간 효율적                                                  │ │
│  │  • 단점: 캐시 일관성 관리 복잡                                         │ │
│  │  • 예: AMD (일부)                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【현대 CPU L2 캐시 사양】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU             L2 크기        배치 방식     연관도                 │ │
│  │  ─────────────   ──────────     ──────────     ──────                   │ │
│  │  Intel i9        512KB/코어    Private       8-way                  │ │
│  │  AMD Ryzen 9     512KB/코어    Private       8-way                  │ │
│  │  Apple M2        16MB (공유)   Shared        -                      │ │
│  │  ARM Cortex-A78  256KB/코어    Private       8-way                  │ │
│  │  → Apple M2는 큰 공유 L2                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### L2 캐시 분석

```

│                L2 캐시 분석                                                 │

│  ──────────────────                                                  │

│                                                                     │
│  【L2 캐시 최적화 팁】                                                         │
│  ──────────────────                                                  │
│  1. 워킹셋 크기: L2 크기 이내로 유지                                    │
│  2. 데이터 배치: L2 지역성 극대화                                        │
│  3. 루프 타일링: L2 크기에 맞게 타일 크기 설정                             │
│  4. 데이터 구조: L2 친화적 구조                                         │
│  5. 메모리 할당: 연속적 메모리 할당                                       │
│                                                                     │
│  【L2 캐시 문제 해결】                                                         │
│  ──────────────────                                                  │
│  • 높은 미스율: 워킹셋 > L2 → 데이터 구조 변경                            │
│  • 캐시 스래싱: 연관도 부족 → 데이터 재배치                                 │
│  • 캐시 충돌: 같은 셋 → 오프셋 변경                                      │
│  • 메모리 대역폭: L2 미스 다 → 프리패칭                                   │

│  └───────────────────────────────────────────────────────────────┘ │
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• L2 캐시: L1과 L3 사이의 중간 계층 캐시
• 크기: 256KB ~ 2MB (L1의 4~32배)
• 지연: 3~5 클럭 (약 2~5ns)
• 유형: Unified Cache (명령어 + 데이터)
• 배치: Private, Shared, Cluster
• 정책: Inclusive vs Exclusive
• 적중률: 95~98%
• 연관도: 8~16 way Set Associative
• 역할: L1 미스 처리, 큰 워킹셋 지원
• 최적화: 워킹셋 < L2 크기 유지
```
