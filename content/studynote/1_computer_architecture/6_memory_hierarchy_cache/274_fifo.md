# 274. FIFO (First In First Out)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가장 먼저 들어온 항목 교체
> 2. **가치**: 가장 단순한 교체 정책
> 3. **융합**: 큐, Belady's Anomaly, 순환 버퍼

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

FIFO(First In First Out)는 **가장 먼저 들어온 캐시 라인을 교체하는 알고리즘**이다. 큐(Queue) 구조로 구현되며, 가장 단순한 교체 정책이다.

### 💡 비유: 줄 서기

FIFO는 **줄 서기**와 같다. 줄에 가장 먼저 선 사람이 가장 먼저 나간다. 순서대로 들어오고 순서대로 나간다. 가장 오래 기다린 사람이 먼저 나간다.

### FIFO 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                FIFO 개요                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FIFO 원리】                                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  가정: 먼저 들어온 것부터 교체                                        │ │
│  │                                                               │ │
│  │  동작:                                                            │ │
│  │  1. 새 항목은 큐의 끝에 추가                                          │ │
│  │  2. 교체 필요 시 큐의 앞에서 제거                                      │ │
│  │  3. 접근해도 순서 변경 없음 (LRU와 차이)                                 │ │
│  │                                                               │ │
│  │  구조: 선입선출 (Queue)                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FIFO 동작 예시】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  4-way 캐시, 접근 순서: A B C D A B E A B C D                       │ │
│  │                                                               │ │
│  │  접근   큐 상태 [Front→Rear]        Hit/Miss   교체                  │ │
│  │  ────   ──────────────────        ────────    ────                    │ │
│  │  A     [A - - -]                  Miss        -                       │ │
│  │  B     [A B - -]                  Miss        -                       │ │
│  │  C     [A B C -]                  Miss        -                       │ │
│  │  D     [A B C D]                  Miss        -                       │ │
│  │  A     [A B C D]                  Hit         -  (순서 유지)            │ │
│  │  B     [A B C D]                  Hit         -                       │ │
│  │  E     [B C D E]                  Miss        A (가장 오래됨)           │ │
│  │  A     [B C D A]                  Hit         -                       │ │
│  │  B     [B C D A]                  Hit         -                       │ │
│  │  C     [B C D A]                  Hit         -                       │ │
│  │  D     [B C D A]                  Hit         -                       │ │
│  │                                                               │ │
│  │  적중률: 6/11 = 55%                                                 │ │
│  │  → 접근해도 순서 변경 없음 (LRU와 다름)                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FIFO 특징】                                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │
│  │  • 가장 단순한 구현 (큐)                                             │ │
│  │  • 가장 낮은 하드웨어 비용                                            │ │
│  │  • 가장 낮은 전력 소비                                               │ │
│  │  • 예측 가능한 동작                                                  │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 지역성 활용 못함                                                  │ │
│  │  • Belady's Anomaly 발생 가능                                        │ │
│  │  • 낮은 적중률                                                      │ │
│  │  • 자주 사용하는 데이터도 오래되면 교체                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. FIFO 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                FIFO 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Belady's Anomaly】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정의: 캐시 크기를 늘려도 적중률이 감소하는 현상                             │ │
│  │                                                               │ │
│  │  예시: 접근 패턴 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5                 │ │
│  │                                                               │ │
│  │  3-way FIFO:                                                        │ │
│  │  Miss: 1,2,3,4(1교체),1(2교체),2(3교체),5,1,2,3,4,5                  │ │
│  │  총 미스: 9                                                          │ │
│  │                                                               │ │
│  │  4-way FIFO:                                                        │ │
│  │  Miss: 1,2,3,4,1(2교체),2(3교체),5(4교체),1,2,3,4(5교체),5             │ │
│  │  총 미스: 10                                                         │ │
│  │                                                               │ │
│  │  → 캐시가 커졌는데 미스가 늘어남!                                        │ │
│  │  → LRU, OPT는 이런 문제 없음                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FIFO 구현 방법】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 큐 (Queue):                                                       │ │
│  │     • 단순 연결 리스트                                                │ │
│  │     • 삽입: O(1), 삭제: O(1)                                         │ │
│  │     • 포인터 1개로 구현 가능 (순환 버퍼)                                 │ │
│  │                                                               │ │
│  │  2. 순환 버퍼 (Circular Buffer):                                      │ │
│  │     • 고정 크기 배열 + 포인터                                          │ │
│  │     • 포인터가 가리키는 위치 교체                                       │ │
│  │     • 포인터 순환 (next = (ptr + 1) % size)                          │ │
│  │     • 하드웨어 구현 용이                                               │ │
│  │                                                               │ │
│  │  3. 링 버퍼 (Ring Buffer):                                             │ │
│  │     • head, tail 포인터                                             │ │
│  │     • Overwrite 가능                                               │ │
│  │     • Embedded 시스템에서 사용                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FIFO vs LRU vs Random】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           FIFO           LRU           Random              │ │
│  │  ──────          ────           ────           ────                │ │
│  │  구현 복잡도      가장 낮음        중간           가장 낮음            │ │
│  │  하드웨어 비용    가장 낮음        중간           가장 낮음            │ │
│  │  지역성 활용      없음            있음           없음                │ │
│  │  Belady Anomaly  있음            없음           없음                │ │
│  │  적중률          낮음            높음           중간                │ │
│  │  예측성          높음            중간           낮음                │ │
│  │                                                               │ │
│  │  → FIFO는 단순하지만 성능은 낮음                                        │ │
│  │  → Random이 실제로는 FIFO보다 나을 수 있음                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### FIFO 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                FIFO 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FIFO 구현 예시】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // C 순환 버퍼로 FIFO 구현                                          │ │
│  │  int buffer[SIZE];                                                 │ │
│  │  int head = 0;                                                     │ │
│  │                                                               │ │
│  │  void insert(int value) {                                          │ │
│  │      buffer[head] = value;                                         │ │
│  │      head = (head + 1) % SIZE;  // 순환                            │ │
│  │  }                                                                │ │
│  │                                                               │ │
│  │  // Python 큐로 FIFO 구현                                           │ │
│  │  from collections import deque                                    │ │
│  │  cache = deque(maxlen=SIZE)                                       │ │
│  │  cache.append(value)  # 자동으로 오래된 것 제거                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FIFO 사용 사례】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분야            용도               이유                             │ │
│  │  ──────          ────                ────                             │ │
│  │  네트워크         패킷 큐              순서 보장                         │ │
│  │  OS 스케줄링      FCFS                공정성                          │ │
│  │  버퍼            순환 버퍼             단순함                          │ │
│  │  파이프           IPC                 순서 유지                        │ │
│  │  캐시            변형만 사용           성능 문제                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FIFO vs Second Chance】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  FIFO:                                                              │ │
│  │  • 접근해도 순서 유지                                                 │ │
│  │  • Belady Anomaly 발생                                              │ │
│  │  • 낮은 적중률                                                      │ │
│  │                                                               │ │
│  │  Second Chance:                                                     │ │
│  │  • 참조 비트 추가                                                    │ │
│  │  • 참조 비트=1이면 0으로 설정하고 건너뜀                                 │ │
│  │  • 참조 비트=0이면 교체                                               │ │
│  │  • Belady Anomaly 없음                                              │ │
│  │  • 더 높은 적중률                                                    │ │
│  │                                                               │ │
│  │  → Second Chance가 FIFO 개선                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FIFO 최적화 팁】                                                               │
│  ──────────────────                                                  │
│  1. Second Chance 사용: 참조 비트 추가                                   │
│  2. Clock 알고리즘: 순환 포인터                                         │
│  3. 하이브리드: FIFO + LRU 조합                                         │
│  4. 워킹셋: FIFO는 지역성 고려 안 함                                      │
│  5. 캐시 크기: Belady Anomaly 주의                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• FIFO: 가장 먼저 들어온 항목 교체
• 구조: 큐 (Queue), 순환 버퍼
• 구현: O(1) 삽입, O(1) 삭제
• 장점: 가장 단순, 가장 저렴
• 단점: 지역성 무시, Belady Anomaly, 낮은 적중률
• Belady Anomaly: 캐시 커도 적중률 감소 가능
• 접근해도 순서 유지 (LRU와 다름)
• 변형: Second Chance, Clock, Enhanced
• 사용처: 네트워크 큐, 스케줄링, 버퍼
• 캐시에는 변형만 사용 (기본 FIFO 성능 낮음)
```
