# 255. ROM (Read Only Memory)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 읽기 전용 비휘발성 메모리
> 2. **가치**: 펌웨어, 부트로더 저장, 전원 차단 후 데이터 유지
> 3. **융합**: BIOS/UEFI, 임베디드 시스템, IoT

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

ROM(Read Only Memory)은 **데이터를 읽기만 할 수 있는 비휘발성 메모리**이다. 전원이 꺼져도 데이터가 유지되며, 주로 펌웨어, 부트로더, 시스템 설정 등을 저장하는 데 사용된다.

### 💡 비유: 돌에 새긴 글

ROM은 **돌에 새긴 글**과 같다. 한 번 새기면 지울 수 없다. 영구적으로 보존된다. 비가 오고 바람이 불어도(전원 차단) 글은 그대로 남아 있다.

### ROM 종류

```
┌─────────────────────────────────────────────────────────────────────┐
│                ROM 개요                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ROM 발전 과정】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Mask ROM (마스크 ROM)                                            │ │
│  │  • 제조 시 데이터 심음 (변경 불가)                                   │ │
│  │  • 대량 생산용, 저렴                                               │ │
│  │  • 예: 게임 카트리지                                                │ │
│  │                                                               │ │
│  │  PROM (Programmable ROM)                                          │ │
│  │  • 한 번만 프로그래밍 가능                                           │ │
│  │  • 퓨즈 방식, UV 소거                                              │ │
│  │  • 예: 초기 BIOS                                                   │ │
│  │                                                               │ │
│  │  EPROM (Erasable PROM)                                            │ │
│  │  • UV 빛으로 소거 가능                                              │ │
│  │  • 창이 있어 UV 노출                                                │ │
│  │  • 수십 번 재프로그래밍                                              │ │
│  │                                                               │ │
│  │  EEPROM (Electrically EPROM)                                      │ │
│  │  • 전기적으로 소거/재프로그래밍                                        │ │
│  │  • 바이트 단위 수정                                                 │ │
│  │  • 예: BIOS, 설정 저장                                             │ │
│  │                                                               │ │
│  │  Flash Memory (플래시 메모리)                                        │ │
│  │  • 전기적 소거, 블록 단위                                            │ │
│  │  • 대용량, 고속                                                    │ │
│  │  • 예: SSD, USB, 스마트폰                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ROM vs RAM】                                                              │|
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           ROM               RAM                           │ │
│  │  ──────────     ───────────────   ────────────────                │ │
│  │  쓰기           제한적/불가          자유로움                        │ │
│  │  휘발성          비휘발성            휘발성                         │ │
│  │  속도           느림                빠름                           │ │
│  │  용도           펌웨어, 설정         실행 프로그램                   │ │
│  │  예시           BIOS, SSD          DDR, SRAM                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. ROM 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ROM 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ROM 용도】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. BIOS/UEFI: 시스템 부트 펌웨어                                   │ │
│  │  2. 펌웨어: 임베디드 시스템 소프트웨어                                │ │
│  │  3. 마이크로코드: CPU 마이크로프로그램                               │ │
│  │  4. 룩업 테이블: 문자 폰트, 색상표                                   │ │
│  │  5. 게임 데이터: 게임 카트리지                                       │ │
│  │  6. 설정 데이터: 시스템 설정, 캘리브레이션                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【EEPROM 구조】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  셀 구조 (플로팅 게이트):                                           │ │
│  │                                                               │ │
│  │       컨트롤 게이트                                                │ │
│  │           │                                                      │ │
│  │       ┌───┴───┐                                                  │ │
│  │       │ 산화막  │                                                  │ │
│  │       └───┬───┘                                                  │ │
│  │       플로팅 게이트  ← 전하 저장                                     │ │
│  │           │                                                      │ │
│  │       ┌───┴───┐                                                  │ │
│  │       │ 산화막  │                                                  │ │
│  │       └───┬───┘                                                  │ │
│  │       채널 (실리콘)                                                │ │
│  │                                                               │ │
│  │  프로그래밍: 높은 전압으로 전하 주입                                   │ │
│  │  소거: 높은 역전압으로 전하 제거                                      │ │
│  │  → 전하 유무로 0/1 표현                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ROM 성능 특성】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  타입          읽기 속도    쓰기 속도    쓰기 횟수                   │ │
│  │  ──────────     ────────    ────────    ────────                   │ │
│  │  Mask ROM       ns         불가        0                          │ │
│  │  PROM           ns         ms          1                          │ │
│  │  EPROM          ns         min         ~100                       │ │
│  │  EEPROM         ns         ms          ~100K                      │ │
│  │  Flash          μs         ms          ~100K-1M                   │ │
│  │  → Flash가 가장 실용적                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• ROM: 읽기 전용 비휘발성 메모리
• 특징: 전원 차단 후 데이터 유지
• 발전: Mask ROM → PROM → EPROM → EEPROM → Flash
• 용도: BIOS/UEFI, 펌웨어, 설정, 마이크로코드
• 장점: 비휘발성, 안정성, 보안
• 단점: 쓰기 제한, 속도 느림
• 현대: Flash Memory가 ROM 역할
• 응용: BIOS, SSD 펌웨어, IoT, 자동차
```
