# 266. 캐시 맵핑 방식 (Cache Mapping)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 주소를 캐시 위치로 매핑하는 방법
> 2. **가치**: 성능, 비용, 복잡도 간 트레이드오프
> 3. **융합**: 직접, 연관, 집합 연관 사상

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

캐시 맵핑(Cache Mapping)은 **메인 메모리의 블록을 캐시의 어느 위치에 배치할지 결정하는 방식**이다. 직접 사상(Direct), 완전 연관(Fully Associative), 집합 연관(Set Associative) 세 가지 방식이 있다.

### 💡 비유: 주차장 배치

캐시 맵핑은 **주차장 배치 규칙**과 같다. 직접 사상은 번호별 지정 주차(101호는 A1만). 완전 연관은 아무 자리나 가능. 집합 연관은 구역만 정하면 그 안에서 자유. 각각 장단점이 있다.

### 캐시 맵핑 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                캐시 맵핑 방식 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【세 가지 맵핑 방식】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 직접 사상 (Direct Mapped):                                   │ │
│  │     • 각 메모리 블록 → 정확히 1개 캐시 라인                         │ │
│  │     • 가장 단순, 가장 빠른 조회                                    │ │
│  │     • 충돌 빈발                                                  │ │
│  │                                                               │ │
│  │  2. 완전 연관 (Fully Associative):                               │ │
│  │     • 각 메모리 블록 → 모든 캐시 라인                               │ │
│  │     • 충돌 최소, 최고 적중률                                       │ │
│  │     • 복잡한 하드웨어                                              │ │
│  │                                                               │ │
│  │  3. 집합 연관 (Set Associative):                                 │ │
│  │     • 각 메모리 블록 → 특정 셋 내 N개 라인                           │ │
│  │     • 직접 사상과 완전 연관의 절충                                   │ │
│  │     • 현대 캐시의 표준                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【맵핑 방식 비교】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  방식            검색 복잡도    충돌 빈도    하드웨어    적중률         │ │
│  │  ──────────       ──────────     ──────────   ────────    ──────         │ │
│  │  직접 사상         O(1)           높음         단순        낮음          │ │
│  │  완전 연관         O(N)           낮음         복잡        높음          │ │
│  │  집합 연관         O(K)           중간         중간        중간          │ │
│  │    (N-way)                       (K = way 수)                           │ │
│  │                                                               │ │
│  │  → 실제로는 집합 연관이 가장 많이 사용                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 구조】                                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  직접 사상:                                                        │ │
│  │  ┌──────────────┬──────────────┬──────────────┐                 │ │
│  │  │    Tag       │    Index     │   Offset     │                 │ │
│  │  │   (태그)      │   (인덱스)    │   (오프셋)    │                 │ │
│  │  └──────────────┴──────────────┴──────────────┘                 │ │
│  │                                                               │ │
│  │  집합 연관:                                                        │ │
│  │  ┌──────────────┬──────────────┬──────────────┐                 │ │
│  │  │    Tag       │  Set Index   │   Offset     │                 │ │
│  │  │   (태그)      │  (셋 인덱스)  │   (오프셋)    │                 │ │
│  │  └──────────────┴──────────────┴──────────────┘                 │ │
│  │                                                               │ │
│  │  완전 연관:                                                        │ │
│  │  ┌──────────────┬──────────────┐                               │ │
│  │  │    Tag       │   Offset     │                               │ │
│  │  │   (태그)      │   (오프셋)    │                               │ │
│  │  └──────────────┴──────────────┘                               │ │
│  │  → 인덱스 없음, 모든 태그 병렬 검색                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 캐시 맵핑 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                캐시 맵핑 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【직접 사상 (Direct Mapped)】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  메모리 주소: [Tag][Index][Offset]                               │ │
│  │  매핑: Cache Line = Memory Block % 캐시 라인 수                    │ │
│  │                                                               │ │
│  │  장점:                                                            │ │
│  │  • 단순한 하드웨어                                                 │ │
│  │  • 빠른 접근 (1 사이클)                                            │ │
│  │  • 낮은 전력 소비                                                 │ │
│  │                                                               │ │
│  │  단점:                                                            │ │
│  │  • 높은 Conflict Miss                                            │ │
│  │  • 특정 패턴에서 성능 저하                                          │ │
│  │                                                               │ │
│  │  예: 16KB 캐시, 64B 라인 → 256 라인                                 │ │
│  │  메모리 주소 0x000, 0x1000, 0x2000 → 모두 같은 라인                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【완전 연관 (Fully Associative)】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  메모리 주소: [Tag][Offset]                                       │ │
│  │  매핑: 모든 캐시 라인에 배치 가능                                     │ │
│  │                                                               │ │
│  │  장점:                                                            │ │
│  │  • 최저 Conflict Miss                                            │ │
│  │  • 최고 적중률                                                    │ │
│  │  • 유연한 배치                                                    │ │
│  │                                                               │ │
│  │  단점:                                                            │ │
│  │  • 모든 태그 병렬 비교 필요 (CAM)                                    │ │
│  │  • 복잡한 하드웨어                                                 │ │
│  │  • 높은 전력 소비                                                 │ │
│  │                                                               │ │
│  │  사용: TLB, 작은 캐시 (≤64 엔트리)                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【집합 연관 (Set Associative)】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  메모리 주소: [Tag][Set Index][Offset]                            │ │
│  │  매핑: 특정 셋 내 N개 라인 중 하나                                    │ │
│  │                                                               │ │
│  │  N-way Set Associative:                                          │ │
│  │  • 2-way: 셋당 2개 라인                                            │ │
│  │  • 4-way: 셋당 4개 라인                                            │ │
│  │  • 8-way: 셋당 8개 라인                                            │ │
│  │  • 16-way: 셋당 16개 라인                                          │ │
│  │                                                               │ │
│  │  장점:                                                            │ │
│  │  • 적절한 복잡도                                                   │ │
│  │  • 낮은 Conflict Miss                                            │ │
│  │  • 좋은 적중률                                                    │ │
│  │                                                               │ │
│  │  단점:                                                            │ │
│  │  • 직접 사상보다 느림                                               │ │
│  │  • N-way 비교기 필요                                               │ │
│  │                                                               │ │
│  │  현대 캐시 표준: L1 4~8-way, L2 8~16-way, L3 16~32-way              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【연관도 vs 성능】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  연관도      적중률 향상    접근 시간    전력 소비    면적            │ │
│  │  ──────      ──────────     ──────      ──────      ────            │ │
│  │  1-way      기준          기준         기준        기준            │ │
│  │  2-way      +5~10%        +5~10%       +10%        +10%            │ │
│  │  4-way      +3~5%         +10~15%      +20%        +20%            │ │
│  │  8-way      +1~3%         +15~20%      +30%        +30%            │ │
│  │  16-way     +0~1%         +20~25%      +40%        +40%            │ │
│  │  → 8-way 이상은 수확 체감                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 캐시 맵핑 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                캐시 맵핑 × 타 과목 융합                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 캐시 설계: 연관도 선택이 핵심                                        │

│  • 하드웨어: CAM, 비교기 설계                                          │
│  • 전력: 연관도 높을수록 전력 증가                                       │
│                                                                     │

│  • TLB: 완전 연관 또는 집합 연관                                        │

│  • 배열 접근: 스트라이드 패턴 분석                                       │

│                                                                     │
│  【보안】                                                              │
│  • 사이드 채널: 캐시 충돌 이용 공격                                      │
│  • Spectre: 맵핑 특성 이용                                             │
│  • 완화: 캐시 파티셔닝                                                 │
│  • CAT: Cache Allocation Technology                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  현대 CPU 캐시 맵핑 사례:                                                    │ │
│  │  ──────────────────────                                              │ │

│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 캐시 맵핑 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                캐시 맵핑 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 구조 확인】                                                            │
│  ──────────────────                                                  │
│  $ lscpu                                                            │
│  L1d cache: 32 KiB, 8-way, 64 sets                                 │
│  L1i cache: 32 KiB, 8-way, 64 sets                                 │
│  L2 cache: 256 KiB, 8-way, 512 sets                                │
│  L3 cache: 8192 KiB, 16-way, 8192 sets                             │
│                                                                     │
│  【맵핑 방식에 따른 최적화】                                                      │
│  ──────────────────                                                  │
│  1. 직접 사상: 스트라이드 패턴 피하기                                     │
│  2. 집합 연관: 같은 셋에 몰리는 접근 피하기                                 │
│  3. 완전 연관: 데이터 지역성 극대화                                        │
│  4. 공통: 캐시 크기와 연관도 고려                                         │
│                                                                     │
│  【캐시 충돌 분석】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제: 2개 배열 A[1024], B[1024] (int, 4KB씩)                      │ │
│  │  캐시: 32KB, 8-way, 64B 라인 → 64셋                                 │ │
│  │                                                               │ │
│  │  나쁜 패턴 (같은 셋 매핑):                                            │ │
│  │  for (i = 0; i < 1024; i += 64)                                  │ │
│  │    A[i] + B[i];  // A, B 같은 셋 → 충돌                             │ │
│  │                                                               │ │
│  │  좋은 패턴 (다른 셋 매핑):                                            │ │
│  │  for (i = 0; i < 1024; i++)                                      │ │
│  │    A[i] + B[i];  // 순차적 → 다른 셋                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【맵핑 최적화 팁】                                                             │
│  ──────────────────                                                  │
│  1. 배열 패딩: 캐시 라인 크기에 맞춤                                      │
│  2. 데이터 정렬: 캐시 라인 경계                                          │
│  3. 구조체 배치: 핫 데이터 분리                                          │
│  4. 배열 분리: 같은 셋 매핑 회피                                          │
│  5. 타일링: 캐시 크기와 연관도 고려                                        │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 캐시 맵핑: 메모리 블록을 캐시 위치로 매핑
• 직접 사상: 1:1 매핑, 단순, 충돌 많음
• 완전 연관: N:1 매핑, 복잡, 충돌 적음
• 집합 연관: 절충안, 현대 캐시 표준
• 연관도: 2-way, 4-way, 8-way, 16-way
• 트레이드오프: 복잡도 vs 적중률 vs 전력
• 선택 기준: 워크로드, 전력, 면적
• 현대 표준: L1 4~8-way, L2 8~16-way, L3 16~32-way
• 최적화: 캐시 충돌 회피, 데이터 배치
• 측정: lscpu, perf
```
