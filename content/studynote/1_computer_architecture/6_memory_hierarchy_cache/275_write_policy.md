# 275. 캐시 쓰기 정책 (Cache Write Policy)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시 데이터 수정 시 하위 메모리 동기화 방법
> 2. **가치**: 일관성 vs 성능의 트레이드오프
> 3. **융합**: Write-Through, Write-Back, Write-Around

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

캐시 쓰기 정책(Cache Write Policy)은 **캐시의 데이터가 수정될 때 하위 메모리에 언제, 어떻게 반영할지 결정하는 정책**이다. Write-Through, Write-Back, Write-Around 등이 있으며, 일관성과 성능의 트레이드오프다.

### 💡 비유: 메모장 정리

쓰기 정책은 **메모장 정리**와 같다. Write-Through는 쓸 때마다 파일에 저장 (안전하지만 느림). Write-Back은 수정만 해두고 나중에 저장 (빠르지만 위험). Write-Around는 메모장에 안 쓰고 파일에만 저장.

### 쓰기 정책 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                쓰기 정책 개요                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【세 가지 쓰기 정책】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Write-Through (통과 쓰기):                                      │ │
│  │     • 캐시와 메모리에 동시에 쓰기                                       │ │
│  │     • 항상 일관성 유지                                               │ │
│  │     • 쓰기가 느림                                                   │ │
│  │                                                               │ │
│  │  2. Write-Back (후기 쓰기, Copy-Back):                              │ │
│  │     • 캐시에만 쓰고, 교체 시 메모리에 반영                                │ │
│  │     • 쓰기가 빠름                                                   │ │
│  │     • 일관성 관리 복잡                                               │ │
│  │                                                               │ │
│  │  3. Write-Around (회피 쓰기, No-Write-Allocate):                     │ │
│  │     • 메모리에만 쓰고, 캐시에 로드 안 함                                 │ │
│  │     • 쓰기 미스 시 메모리만 갱신                                        │ │
│  │     • 읽기 때만 캐시 로드                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【쓰기 정책 비교】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정책            쓰기 속도    일관성    복잡도    사용처              │ │
│  │  ──────          ──────      ──────    ──────    ──────              │ │
│  │  Write-Through   느림         즉시      단순      중요 데이터          │ │
│  │  Write-Back      빠름         지연      복잡      일반 캐시            │ │
│  │  Write-Around    중간         즉시      중간      쓰기 많은 경우        │ │
│  │                                                               │ │
│  │  → 대부분 Write-Back 사용 (성능)                                     │ │
│  │  → 중요 데이터는 Write-Through                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【쓰기 미스 처리】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Write-Allocate (쓰기 할당):                                     │ │
│  │     • 미스 시 캐시에 블록 로드 후 쓰기                                  │ │
│  │     • Write-Back과 함께 사용                                         │ │
│  │     • 쓰기 지역성 활용                                               │ │
│  │                                                               │ │
│  │  2. No-Write-Allocate (비할당):                                     │ │
│  │     • 미스 시 메모리에만 쓰기                                          │ │
│  │     • Write-Through과 함께 사용                                       │ │
│  │     • 쓰기 후 바로 읽지 않으면 효율적                                   │ │
│  │                                                               │ │
│  │  조합:                                                              │ │
│  │  • Write-Through + No-Write-Allocate                               │ │
│  │  • Write-Back + Write-Allocate                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  │  동작:                                                              │ │

│  │                                                               │ │

│  │                                                               │ │

│  │                                                               │ │
│  │  사용처:                                                              │ │

│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Write-Back 상세】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  동작:                                                              │ │
│  │  1. CPU가 데이터 쓰기 요청                                           │ │
│  │  2. 캐시에만 쓰기, 더티 비트 설정                                      │ │
│  │  3. 교체 시 더티 라인만 메모리에 쓰기                                   │ │
│  │                                                               │ │
│  │  장점:                                                               │ │
│  │  • 쓰기가 빠름 (캐시만)                                              │ │
│  │  • 낮은 메모리 대역폭                                                │ │
│  │  • 여러 쓰기 병합 가능                                               │ │
│  │  • 일반적 사용                                                       │ │
│  │                                                               │ │
│  │  단점:                                                               │ │
│  │  • 일관성 관리 복잡                                                  │ │
│  │  • 더티 비트 필요                                                   │ │
│  │  • 교체 시 쓰기 페널티                                               │ │
│  │  • 장애 시 데이터 손실 가능                                           │ │
│  │                                                               │ │
│  │  사용처:                                                              │ │
│  │  • D-Cache (데이터 캐시)                                            │ │

│                                                                     │
│  【Write Buffer】                                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  목적: 쓰기 속도 향상                                                  │ │
│  │                                                               │ │
│  │  동작:                                                              │ │
│  │  • 캐시에서 쓰기 버퍼로 데이터 전송                                     │ │
│  │  • CPU는 계속 실행                                                  │ │
│  │  • 버퍼가 백그라운드에서 메모리에 쓰기                                   │ │
│  │                                                               │ │
│  │  크기: 4~32 엔트리                                                  │ │
│  │  효과: 쓰기 지연 숨김                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 쓰기 정책: 캐시 수정 시 메모리 동기화 방법
• Write-Through: 캐시+메모리 동시 쓰기, 일관성 최고
• Write-Back: 캐시만 쓰기, 교체 시 반영, 성능 최고
• Write-Around: 메모리만 쓰기, 캐시 로드 안 함
• Write-Allocate: 미스 시 캐시 로드 후 쓰기
• No-Write-Allocate: 미스 시 메모리만 쓰기
• 더티 비트: Write-Back에서 수정 여부 추적
• Write Buffer: 쓰기 지연 숨김
• 트레이드오프: 일관성 vs 성능
• 현대: D-Cache는 Write-Back 표준
```
