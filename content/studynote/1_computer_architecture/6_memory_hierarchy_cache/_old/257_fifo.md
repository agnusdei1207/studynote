# FIFO (First In First Out)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: FIFO(선입선출) 알고리즘은 캐시 메모리가 가득 차서 데이터를 내쫓아야 할 때, **'캐시에 가장 먼저 들어왔던(가장 오래 머물렀던)' 데이터를 무조건 1순위로 방출**하는 가장 단순한 교체 기법이다.
> 2. **가치**: 데이터가 자주 쓰였는지(LFU), 방금 쓰였는지(LRU)를 전혀 따지지 않기 때문에 멍청해 보이지만, **하드웨어 구현이 원형 큐(Circular Queue) 하나로 끝날 만큼 극도로 간단하고 빠르다**는 가성비를 지닌다.
> 3. **융합**: 자주 쓰이는 핵심 데이터마저 들어온 지 오래됐다는 이유로 무참히 죽여버리는 맹점과, 캐시 크기를 늘렸는데 오히려 적중률이 떨어지는 기현상(**Belady's Anomaly**) 때문에 메인 캐시에서는 퇴출되었으나, 분기 예측기나 버퍼 등 단순 큐 구조에서 여전히 융합되어 쓰인다.

---

## Ⅰ. FIFO 알고리즘의 개요

### 1. 정의
- 캐시의 각 세트(Set)에 데이터 블록이 적재된 순서를 큐(Queue)로 기억하고 있다가, 새로운 블록이 들어올 공간이 부족할 때 큐의 맨 앞(가장 먼저 들어온 부분, Front)에 있는 블록을 삭제하고 새로운 블록을 맨 뒤(Rear)에 넣는 교체 방식이다.

### 2. 등장 배경: '가장 싸고 단순하게 막아라'
- 초기 컴퓨터 설계자들은 LRU처럼 매번 '가장 최근에 쓴 놈'을 갱신하는 하드웨어 회로가 칩 면적을 너무 많이 차지한다는 사실에 좌절했다. "그냥 복잡하게 생각하지 말고, 식당 대기줄처럼 먼저 온 놈이 밥 다 먹었을 테니 먼저 쫓아내자!"라는 발상이 도입되었다. 카운터나 정렬 로직 없이 포인터 하나만 빙글빙글 돌리면(Round Robin) 끝나는 예술적인 단순함이 무기였다.

### 3. 💡 비유: '편의점 음료수 진열대'
- FIFO는 **'편의점 냉장고의 뒤에서 채워 넣는 음료수 진열대'**와 완벽히 똑같다.
- 알바생이 새로 들어온 음료수를 뒤쪽(Rear)에서 밀어 넣으면, 가장 오래전에 들어와서 앞에 있던 음료수(Front)가 손님에게 먼저 팔려 나간다. 이 음료수가 제일 인기 있는 맛인지 아닌지는 전혀 중요하지 않고, 오직 '들어온 시간'만이 나갈 시간을 결정한다.

- **📢 섹션 요약 비유**: 성과(자주 쓰였나?)나 능력(방금 쓰였나?)은 깡그리 무시하고, 오직 나이(입사 순서)만 따져서 늙은 순서대로 강제 은퇴시키는 무자비한 철밥통 시스템입니다.

## Ⅱ. 메커니즘 및 치명적 한계 (Deep Dive)

구현의 달콤함 이면에는 수학적인 재앙이 숨어있다.

### 1. FIFO의 큐(Queue) 교체 동작
- 방 4개짜리(4-Way) 캐시에 A, B, C, D가 순서대로 들어왔다. (A가 가장 오래됨)
- E가 들어오려고 한다. CPU는 A가 방금 전까지 미친 듯이 쓰인 핵심 데이터인지 묻지도 따지지도 않고, 큐의 맨 앞에 있다는 이유만으로 **A를 즉시 사살하고** E를 꽂아 넣는다. (결과: E, B, C, D 순서가 됨)

### 2. 치명적 약점: 시간적 지역성(Temporal Locality) 무시
- `sum = sum + i` 라는 루프에서 `sum`은 맨 처음에 메모리에서 캐시로 올라온다. 즉, '가장 먼저 들어온 데이터'다.
- FIFO를 쓰면 캐시가 꽉 찼을 때 이 소중한 `sum`을 가장 먼저 쫓아내 버린다. 1나노초 뒤에 루프가 돌면서 `sum`을 또 찾기 때문에 어마어마한 캐시 미스(Miss)가 발생한다. 지능이 0에 수렴하는 알고리즘의 한계다.

### 3. 심층 분석: Belady의 모순 (Belady's Anomaly)
- 보통 상식적으로 '캐시 방의 개수를 늘리면, 더 많이 저장할 수 있으니 캐시 미스가 줄어든다'고 생각한다.
- 하지만 1969년 Laszlo Belady가 수학적으로 증명했듯, **FIFO 알고리즘에서는 캐시 방의 개수를 3개에서 4개로 늘렸는데 오히려 캐시 미스 횟수가 폭증하는 기현상**이 특정 패턴(1, 2, 3, 4, 1, 2, 5, 1, 2, 3...)에서 발생한다. 멍청한 규칙이 낳은 치명적 논리 오류다.

- **📢 섹션 요약 비유**: 방이 넓어지면 짐 정리가 잘 돼야 정상인데, 오히려 방이 넓어질수록 물건을 어디 뒀는지 더 못 찾아서 집이 난장판이 되는 수학적 저주에 걸린 셈입니다.

## Ⅲ. FIFO vs LRU 비교

현대 시스템에서 FIFO가 버림받고 LRU가 선택받은 이유.

| 비교 항목 | FIFO (First In First Out) | LRU (Least Recently Used) |
|:---|:---|:---|
| **희생양 선택 기준**| **가장 먼저 적재된 놈** (오직 태어난 시간) | **가장 오랫동안 안 쓴 놈** (최근 활동 내역) |
| **적중 시 상태 갱신**| **안 함 (Hit 되어도 큐 위치 불변)** | **함 (Hit 되면 최신 상태로 갱신)** |
| **구현 난이도** | 최하 (포인터만 움직이면 끝) | 높음 (순서 리스트 계속 뒤집어야 함) |
| **지역성 반영** | 0% (바보) | 100% (완벽) |
| **Belady의 모순** | **발생할 수 있음** | 절대로 발생하지 않음 (Stack 알고리즘 특성) |

- **📢 섹션 요약 비유**: 어제 회사에 100억을 벌어준 부장님을 "가장 늙었다"는 이유로 해고하는 바보(FIFO)와, 작년에 입사했어도 일 안 한 신입사원을 먼저 자르는 똑똑한 사장(LRU)의 차이입니다.

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: 캐시에서는 퇴출, 네트워크/버퍼에서는 맹활약
- **상황**: L1, L2 캐시 등 코어의 생살여탈권을 쥐고 있는 메모리 계층에서는 히트율 1%가 아쉬우므로 멍청한 FIFO는 절대 쓰지 않는다.
- **판단**: 하지만 **입출력(I/O) 버퍼, 키보드 입력 버퍼, 파이프라인의 명령어 큐(Fetch Queue)**에서는 이야기가 완전히 다르다. 들어온 순서대로 데이터를 빼주는 것 자체가 '목적'이기 때문에 FIFO는 여기서 절대적인 권력을 쥔다. 라우터가 네트워크 패킷을 처리할 때도 도착한 순서대로 내보내는(FIFO) 것이 공평성과 타이밍을 보장하는 유일한 해답이다. "어디에 쓰느냐"에 따라 알고리즘의 가치가 달라짐을 보여준다.

### 2. 실무 시나리오: Second Chance 알고리즘 (FIFO의 부활)
- **인사이트**: 운영체제의 페이지 교체에서 완벽한 LRU를 돌리자니 OS가 너무 무거워진다. 그래서 멍청한 FIFO 큐에 약간의 지능을 더한 **Second Chance (또는 Clock)** 알고리즘이 현대 OS(리눅스 등)의 표준이 되었다.
- FIFO처럼 포인터가 둥글게 돌면서 가장 오래된 놈을 죽이려 하는데, "잠깐, 얘 최근에 한 번이라도 쓰였니(Reference Bit == 1)?"라고 물어보고, 쓰였다면 비트를 0으로 깎아주면서 **한 번의 기회를 더 주고 살려둔다**. 그리고 다음 타겟으로 넘어간다. 엄청나게 싼 FIFO의 하드웨어 비용으로 비싼 LRU의 적중률을 99% 모방해 낸 전설적인 타협안이다.

- **📢 섹션 요약 비유**: 무조건 늙었다고 죽이는 게 아니라, "최근에 숨 쉬었어? 그럼 한 턴 더 살려줄게"라며 융통성을 발휘하여 가성비와 효율을 싹 다 잡은 명재판관입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **순차적 데이터의 질서 보장**: 선입선출이라는 자연계의 가장 기본적인 순리를 하드웨어에 가장 빠르고 저렴하게 구현.
- **오버헤드 제로(0)**: 데이터가 적중(Hit)할 때마다 큐를 갱신할 필요가 없으므로, 캐시의 읽기 속도가 극단적으로 빨라짐.

### 2. 미래 전망
메인 캐시의 교체 정책으로서 순수한 FIFO는 교과서의 반면교사(Belady's Anomaly)로만 남았다. 하지만 스트리밍 비디오(Netflix)나 대용량 로그 파일 분석처럼 **"한 번 읽고 지나가면 다시는 안 볼 데이터"**를 다루는 최신 워크로드에서는, 캐시에 잔상을 남기지 않고 빨리빨리 밀어내어 다음 데이터에게 자리를 내어주는 FIFO 방식의 변형(예: FIFO 기반의 스트리밍 버퍼)이 오히려 LRU보다 나은 성능을 보여주며 재평가받기도 한다. 

- **📢 섹션 요약 비유**: 복잡한 세상에서 너무 똑똑하게 재는 것보다, 들어온 순서대로 쿨하게 빼주는 우직함이 오히려 가장 빠른 정답이 될 때가 있는 법입니다.
