# 255. ROM (Read Only Memory)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전원이 차단되어도 저장된 데이터가 지워지지 않는 비휘발성(Non-volatile) 메모리 구조로, 시스템의 초기화와 핵심 제어 코드를 영구적으로 보존하는 하드웨어다.
> 2. **가치**: 컴퓨터의 콜드 부팅(Cold Boot) 시 CPU가 가장 먼저 실행할 명령어(BIOS/UEFI)를 제공하며, 임베디드 시스템과 IoT 기기의 펌웨어를 안전하게 구동하는 근간이 된다.
> 3. **융합**: 제조 시 데이터가 고정되는 Mask ROM에서 출발하여, 자외선(EPROM)과 전기적 신호(EEPROM)로 데이터를 수정할 수 있는 형태로 진화하였으며, 이는 현대 플래시(Flash) 메모리 아키텍처의 직접적인 모태가 되었다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: ROM(Read Only Memory)은 읽기 전용으로 설계된 주 기억 장치의 일종이다. 물리적인 회로 결선이나 퓨즈, 고립된 게이트의 전하 상태를 통해 데이터를 표현하므로 전원 공급과 무관하게 데이터가 영구 보존된다.
- **💡 비유**: 일반 RAM이 연필로 썼다 지웠다 할 수 있는 '화이트보드'라면, ROM은 돌에 글자를 새겨 넣은 '비석'이다. 비석은 전기가 끊어지는 것과 상관없이 내용이 영원히 유지되지만, 내용을 바꾸려면 비석을 새로 파거나(PROM) 표면을 깎아내는(EPROM/EEPROM) 아주 특별하고 느린 과정이 필요하다.
- **등장 배경 및 발전 과정**:
  1. **초기 부트스트랩(Bootstrap) 문제**: 컴퓨터 전원을 켰을 때 RAM은 텅 비어 있다. CPU가 OS를 디스크에서 RAM으로 가져오기 위해서는 '가져오는 방법'을 아는 최소한의 코드가 필요한데, 이 코드를 영구적으로 담아둘 장소가 필요했다.
  2. **하드와이어링의 유연성 한계**: 초기의 로직 보드(Hardwired)는 수정이 불가능했다. 반도체 제조 공정 중 마스크(Mask) 패턴을 통해 데이터를 각인하는 Mask ROM이 등장하여 대량 생산이 가능해졌다.
  3. **재프로그래밍의 요구**: 개발 과정에서 펌웨어 버그를 수정하기 위해 칩을 버려야 하는 낭비를 막기 위해, 사용자가 한 번 기록할 수 있는 PROM, 자외선으로 지우는 EPROM, 전기적으로 지우는 EEPROM으로 기술이 발전했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **어드레스 디코더** | 입력된 주소를 해석하여 활성화할 행(Wordline)을 선택 | 논리 AND 게이트 배열로 특정 라인에 High 신호 인가 | Address Decoding | 도서관의 책장 번호판 |
| **ROM 셀 어레이** | 실제 0과 1의 데이터가 하드웨어적으로 각인된 매트릭스 | 트랜지스터의 연결 유무(Mask)나 퓨즈 상태로 데이터 표현 | Crossbar Array | 비석에 새겨진 글자 매트릭스 |
| **비트 라인 (Bitline)** | 선택된 워드 라인의 상태를 감지하여 데이터 출력 | 워드 라인과 교차점의 트랜지스터 유무에 따라 전류 흐름 결정 | Pull-up Resistor | 글자를 읽어내는 스캐너 |
| **센스 앰프 (Sense Amp)** | 비트 라인의 미세한 전류/전압 변화를 증폭하여 디지털 신호로 변환 | 기준 전압과 비교하여 0 또는 1 판별 | 증폭 회로 | 희미한 글씨를 보는 돋보기 |
| **플로팅 게이트 (EPROM/EEPROM)** | 전하를 가두어 트랜지스터의 문턱 전압(Vth)을 영구적으로 변화시킴 | 산화막으로 둘러싸여 전자가 빠져나가지 못함 | Fowler-Nordheim 터널링 | 지워지지 않는 특수 잉크 |

### Mask ROM vs PROM 하드웨어 구조 

전통적인 Mask ROM은 반도체 제조 공정에서 트랜지스터를 연결(1)하거나 연결하지 않음(0)으로써 데이터를 기록한다. 반면 PROM(Programmable ROM)은 모든 교차점에 퓨즈를 연결해두고 강한 전류로 퓨즈를 끊어서 데이터를 기록한다.

```text
  ┌───────────────────────────────────────────────────────────────┐
  │                 Mask ROM vs PROM 회로 매트릭스                  │
  ├───────────────────────────────────────────────────────────────┤
  │                                                               │
  │  [1] Mask ROM (제조 시 고정)                                     │
  │                                                               │
  │  Wordline 0 ─────┬────────────┬───────── (0번지 주소 선택)         │
  │                  │            │                               │
  │              [Tr 접속]    [Tr 미접속]  ← 금속 배선 공정에서 결정        │
  │                  │            │                               │
  │                  ▼            ▼                               │
  │  Bitline      [전류 흐름]   [전류 없음]                          │
  │  출력(Data)       1            0                              │
  │                                                               │
  │ ───────────────────────────────────────────────────────────── │
  │                                                               │
  │  [2] PROM (사용자 1회 기록 - Fusible Link)                        │
  │                                                               │
  │  Wordline 0 ─────┬────────────┬─────────                       │
  │                  │            │                               │
  │              [퓨즈 끊음]    [퓨즈 유지]   ← 고전압 프로그래밍으로 끊음   │
  │                  │            │                               │
  │                  ▼            ▼                               │
  │  Bitline      [전류 없음]   [전류 흐름]                          │
  │  출력(Data)       0            1                              │
  │                                                               │
  │  ※ EPROM/EEPROM은 퓨즈 대신 '플로팅 게이트' 트랜지스터를 사용함.      │
  └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 상단의 Mask ROM 구조에서는 주소 디코더가 Wordline 0에 전압을 가할 때, 제조 공정 시 트랜지스터(Tr)가 연결된 첫 번째 열에서는 Bitline으로 전류가 흘러내려가 '1'로 읽힌다. 두 번째 열은 트랜지스터가 연결되지 않았으므로 전류가 흐르지 않아 '0'으로 읽힌다. 이 구조는 변경이 물리적으로 불가능하다. 하단의 PROM은 초기에 모든 교차점에 퓨즈가 연결되어 있어 모두 '1' 상태다. 롬 라이터(ROM Writer) 장비를 이용해 특정 교차점에 고전압을 가하면 해당 퓨즈가 타서 끊어지고(Blow), 그 자리는 영구적으로 '0'이 된다. 한 번 끊어진 퓨즈는 복구할 수 없으므로 OTP(One-Time Programmable) 메모리라고도 불린다.

### 심층 동작 원리 (EEPROM의 데이터 쓰기/지우기 메커니즘)
EEPROM(Electrically Erasable Programmable ROM)은 플로팅 게이트(Floating Gate) 기술을 사용하여 전기적으로 데이터를 쓰고 지운다.

① **데이터 쓰기 (Programming)**: 컨트롤 게이트(Control Gate)에 고전압(예: +20V)을 가한다. 채널에 있던 전자들이 강한 전기장에 이끌려 절연막을 뚫고(Fowler-Nordheim Tunneling) 플로팅 게이트에 갇힌다. 전자가 갇히면 트랜지스터가 켜지기 위한 문턱 전압(Vth)이 높아져, 일반적인 읽기 전압에서는 전류가 흐르지 않게 된다(Data '0').
② **데이터 보존 (Retention)**: 고전압을 제거해도, 플로팅 게이트는 두꺼운 산화막(Oxide)으로 완전히 고립되어 있어 전자가 수십 년간 빠져나가지 못하고 유지된다.
③ **데이터 지우기 (Erasing)**: 기판(Substrate)에 고전압을 가하고 컨트롤 게이트를 접지(0V)시킨다. 갇혀 있던 전자가 기판 쪽으로 다시 빠져나오며, 트랜지스터의 문턱 전압이 원래대로 낮아진다(Data '1'). EEPROM은 바이트(Byte) 단위로 미세하게 지우고 쓸 수 있는 것이 특징이다.

## Ⅲ. 융합 비교 및 다각도 분석

### ROM 유형별 심층 비교표

| 구분 | 기록 방식 | 소거 방식 | 재기록 가능 횟수 | 단위 지우기 | 주요 사용처 |
|:---|:---|:---|:---|:---|:---|
| **Mask ROM** | 반도체 제조 공정(포토마스크) | 불가능 | 0회 | 불가능 | 게임팩, 옛날 PC 폰트 칩 |
| **PROM (OTP)** | 고전압 퓨즈 용단 | 불가능 | 1회 | 불가능 | RFID 태그, 보안 키 칩 |
| **EPROM** | 고전압 프로그래밍 | 자외선 조사 (칩 창문) | 수백 회 | 전체 칩 동시 | 초기 개발용 펌웨어 테스트 |
| **EEPROM** | 전기적 프로그래밍 | 전기적 소거 | 약 10만~100만 회 | **바이트(Byte) 단위** | 보드 설정값, 소용량 BIOS |
| **Flash Memory** | 전기적 프로그래밍 | 전기적 소거 | 약 1만~10만 회 | **블록/섹터 단위** | SSD, USB, 현대 PC BIOS/UEFI |

### 과목 융합 관점
- **운영체제 (OS 부팅 시퀀스)**: 전원이 인가되면 CPU는 하드웨어적으로 고정된 특정 메모리 주소(Reset Vector, x86의 경우 `0xFFFFFFF0`)를 참조한다. 이 주소는 항상 메인보드의 ROM(또는 Flash) 칩으로 매핑되어 있으며, 여기 위치한 BIOS/UEFI 코드가 하드웨어를 초기화(POST)하고 MBR/GPT를 읽어 OS 커널을 RAM에 적재하는 릴레이를 시작한다.
- **보안 (Hardware Security)**: 하드웨어 보안 모듈(HSM)이나 스마트폰의 AP는 제조 단계에서 칩 내부의 **OTP ROM(One-Time Programmable ROM)**에 고유한 공개키 해시값(RoT, Root of Trust)을 영구 각인한다. 부팅 시 이 ROM의 값을 기준으로 펌웨어의 전자서명을 검증하는 **Secure Boot** 체인이 형성된다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 임베디드 기기의 무선 펌웨어 업데이트 (OTA, Over-The-Air)**: 스마트홈 도어락의 펌웨어 취약점을 패치해야 한다. 과거의 EPROM 구조라면 기기를 분해해 칩에 자외선을 쬐어야 하지만, 현재는 EEPROM 기반의 NOR 플래시가 탑재되어 있다.
   - **아키텍처 설계**: 펌웨어 뱅크를 A/B 듀얼 파티션으로 나눈다. 기기가 A 파티션의 코드로 구동되는 동안 백그라운드에서 B 파티션에 새 펌웨어를 전기적으로 쓰고(EEPROM 원리), 재부팅 시 부트로더가 부팅 포인터를 B로 변경(Safe OTA)하도록 설계한다.
2. **시나리오 — 불법 위변조 방지를 위한 영구적 락(Lock) 적용**: 신용카드 결제 단말기(POS)의 핵심 암호화 로직이 담긴 메모리 영역이 해커에 의해 덮어씌워질 위험이 있다.
   - **해결책**: 제조 공정 마지막 단계에서 해당 EEPROM 영역의 쓰기 보호(Write-Protect) 레지스터 퓨즈를 고전압으로 끊어버린다(OTP 영역화). 이후 어떠한 소프트웨어적 방법으로도 해당 코드를 변경할 수 없는 불변의 상태를 물리적으로 강제한다.
3. **시나리오 — 펌웨어 부팅 속도 병목 (안티패턴)**: ROM이나 저속 SPI 플래시는 RAM에 비해 읽기 속도가 수백 배 느리다. OS 커널이 ROM에 위치한 함수를 직접(XIP, eXecute In Place) 빈번하게 호출하면 시스템 전체 성능이 극심하게 저하된다.
   - **해결책**: 부팅 초기 단계에 ROM의 펌웨어 코드를 초고속 RAM 공간으로 전부 복사(Shadowing)한 뒤, RAM 위에서 코드를 실행하도록 아키텍처를 변경해야 한다.

### 도입 체크리스트
- **기술적**: 디바이스 설정 데이터(네트워크 MAC 주소, 보정 값 등)처럼 바이트 단위의 잦은 갱신이 필요한 데이터는 Flash가 아닌 EEPROM에 저장되도록 메모리 맵이 분리되어 있는가? (Flash는 블록 단위로만 지워지므로 바이트 갱신 시 오버헤드가 큼)
- **운영·보안적**: 펌웨어 저장용 ROM 칩과 CPU 간의 통신 버스(SPI/I2C)를 오실로스코프로 스니핑하여 코드를 탈취할 위험은 없는가? (필요 시 SoC 내부에 내장된 eFlash/eROM 사용 검토)

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 과거 (Mask ROM / EPROM) | 현대 (EEPROM / Flash) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (유지보수 비용)** | 버그 발생 시 물리적 칩 전량 폐기 | 네트워크를 통한 원격 전기적 패치 | 펌웨어 리콜 비용 **99% 절감** |
| **정량 (업데이트 속도)** | 자외선 지우기 약 20분 소요 | 전기적 지우기 수 ms 소요 | 업데이트 시간 수십만 배 단축 |
| **정성 (보안성)** | 칩 추출 후 복제 용이 | 부트로더 단의 암호화 및 OTP 잠금 | 펌웨어 위변조 완벽 차단 |

### 미래 전망
- **플래시 메모리로의 통합**: 순수한 의미의 독립형 ROM 칩은 소형 임베디드 기기를 제외하고는 거의 사라졌으며, 대용량화가 쉽고 블록 단위로 고속 지우기가 가능한 NOR 플래시와 NAND 플래시 메모리가 사실상 ROM의 역할을 완벽히 대체하고 있다.
- **차세대 비휘발성 메모리(SCM)의 대두**: PRAM, MRAM, ReRAM과 같은 차세대 SCM(Storage Class Memory)은 ROM의 비휘발성, RAM의 바이트 단위 접근성, 플래시의 집적도를 모두 갖춘 궁극의 유니버셜 메모리(Universal Memory)로서, 향후 ROM과 RAM의 물리적 경계를 완전히 허물어버릴 것이다.

### 참고 표준
- **SPI (Serial Peripheral Interface)**: 현대 PC 및 임베디드 보드에서 메인 CPU와 BIOS/UEFI ROM(Flash) 간에 코드를 읽어오기 위해 사용하는 사실상의 표준 직렬 버스 인터페이스.

- **[플래시 메모리 (Flash Memory)](./256_flash_memory.md)**: EEPROM 기술을 기반으로 발전하여, 바이트 단위가 아닌 블록 단위로 데이터를 한꺼번에 지우는 방식으로 집적도를 극대화한 현대의 대용량 비휘발성 메모리.
- **[부트스트랩 (Bootstrap / Booting)](../2_operating_system/1_overview_architecture/xx_booting.md)**: 컴퓨터가 스스로를 깨우는 과정. CPU가 전원 인가 직후 최초로 코드를 인출하는 장소가 바로 ROM이다.
- **[하드웨어 루트 오브 트러스트 (Root of Trust)](../14_hardware_security_trends/487_root_of_trust.md)**: 보안 시스템의 닻 역할을 하는 요소로, 절대 변조되지 않는 OTP ROM에 저장된 공개키를 기반으로 구현된다.
- **[SRAM (Static RAM)](./250_sram.md)**: 전원이 꺼지면 데이터가 날아가는 휘발성 메모리. ROM에 저장된 느린 코드를 고속으로 실행하기 위해 부팅 중 SRAM(Shadow RAM)으로 코드를 복사하는 기법이 쓰인다.

## 👶 어린이를 위한 3줄 비유 설명
1. 컴퓨터는 전기를 끄면 머릿속이 새하얗게 지워지는(RAM) 건망증 꾸러기예요.
2. 그래서 깨어났을 때 "숨은 어떻게 쉬지? 눈은 어떻게 뜨지?"를 알려주기 위해, 절대 지워지지 않게 돌에 새겨둔 설명서가 필요한데 이게 바로 **ROM**이랍니다.
3. 옛날에는 한 번 돌에 새기면 고칠 수 없어서 돌을 버려야 했지만, 요즘 나오는 마법의 돌(EEPROM)은 전기를 찌릿! 하고 주면 글씨를 쓱 지우고 다시 쓸 수 있어서 아주 편리해졌어요.
