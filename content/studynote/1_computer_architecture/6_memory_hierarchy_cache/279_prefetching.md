# 279. 프리패칭 (Prefetching)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 필요한 데이터를 미리 캐시에 로드
> 2. **가치**: Compulsory Miss 감소, 메모리 지연 숨김
> 3. **융합**: HW/SW 프리패칭, 스트라이드, 캐시 오염

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

프리패칭(Prefetching)은 **CPU가 실제로 요청하기 전에 필요할 것으로 예측되는 데이터를 미리 캐시에 로드하는 기술**이다. Compulsory Miss를 감소시키고 메모리 지연을 숨긴다.

### 💡 비유: 미리 책 꺼내기

프리패칭은 **미리 책 꺼내기**와 같다. 내일 수업에 필요한 책을 미리 책상에 둔다. 수업 시간에 책장까지 안 가도 된다. 어떤 책이 필요할지 예측하는 게 핵심이다.

### 프리패칭 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                프리패칭 개요                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프리패칭 원리】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  목표: 미스 페널티를 백그라운드 작업으로 숨김                             │ │
│  │                                                               │ │
│  │  동작:                                                              │ │
│  │  1. 접근 패턴 분석                                                   │ │
│  │  2. 다음 접근 예측                                                   │ │
│  │  3. 예측 주소 미리 로드                                               │ │
│  │  4. 실제 요청 시 캐시 히트                                            │ │
│  │                                                               │ │
│  │  효과:                                                              │ │
│  │  • Compulsory Miss 감소                                            │ │
│  │  • 메모리 지연 숨김                                                   │ │
│  │  • AMAT 감소                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프리패칭 종류】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 하드웨어 프리패칭 (Hardware Prefetching):                         │ │
│  │     • CPU가 자동으로 수행                                            │ │
│  │     • 접근 패턴 감지                                                 │ │
│  │     • 투명하게 동작                                                  │ │
│  │     • 예: 순차, 스트라이드                                            │ │
│  │                                                               │ │
│  │  2. 소프트웨어 프리패칭 (Software Prefetching):                         │ │
│  │     • 프로그래머가 명시적 지시                                          │ │
│  │     • prefetch 명령어 사용                                           │ │
│  │     • 워크로드 지식 활용                                              │ │
│  │     • 예: __builtin_prefetch                                        │ │
│  │                                                               │ │
│  │  3. 하이브리드 프리패칭:                                                 │ │
│  │     • HW + SW 조합                                                 │ │
│  │     • 각각 장점 활용                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프리패칭 패턴】                                                              │
│  ──────────────────                                                  │

│  │  패턴           설명                    감지 난이도    효과              │ │
│  │  ──────         ────                    ──────────     ────              │ │
│  │  순차적(Sequential) 연속 주소 접근           쉬움          높음            │ │
│  │  스트라이드(Stride)   일정 간격 접근           중간          높음            │ │
│  │  역순(Reverse)      역방향 순차              쉬움          높음            │ │
│  │  랜덤(Random)       패턴 없음                불가능         없음            │ │
│  │  포인터 체이닝      포인터 따라가기           어려움         중간            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 프리패칭 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                프리패칭 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드웨어 프리패칭】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 순차 프리패칭 (Sequential Prefetching):                           │ │
│  │     • 다음 블록 미리 로드                                            │ │
│  │     • 가장 단순                                                    │ │
│  │     • 높은 적중률                                                  │ │
│  │     • 예: N+1, N+2 블록                                            │ │
│  │                                                               │ │
│  │  2. 스트라이드 프리패칭 (Stride Prefetching):                           │ │
│  │     • 접근 간격 감지                                                │ │
│  │     • 규칙적 패턴 예측                                              │ │
│  │     • 예: A[0], A[16], A[32] → A[48] prefetch                      │ │
│  │                                                               │ │
│  │  3. 스트림 버퍼 (Stream Buffer):                                       │ │
│  │     • 여러 스트림 동시 추적                                           │ │
│  │     • 최근 접근 패턴 기억                                            │ │
│  │     • 하드웨어 테이블 유지                                           │ │
│  │                                                               │ │
│  │  4. Next-Line Prefetching:                                          │ │
│  │     • 현재 라인 접근 시 다음 라인 프리패칭                               │ │
│  │     • 가장 기본적인 방식                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【소프트웨어 프리패칭】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  명령어:                                                             │ │
│  │  • x86: PREFETCHT0, PREFETCHT1, PREFETCHT2, PREFETCHNTA           │ │
│  │  • ARM: PRFM (Prefetch Memory)                                     │ │
│  │  • C/C++: __builtin_prefetch(ptr, rw, locality)                    │ │
│  │                                                               │ │
│  │  사용 예시:                                                          │ │
│  │  for (i = 0; i < N; i++) {                                         │ │
│  │      __builtin_prefetch(&A[i+8]);  // 8개 앞 미리 로드                │ │
│  │      sum += A[i];                                                  │ │
│  │  }                                                                │ │
│  │                                                               │ │
│  │  주의점:                                                             │ │
│  │  • 너무 이르면: 캐시에서 밀려남                                        │ │
│  │  • 너무 늦으면: 효과 없음                                             │ │
│  │  • 너무 많으면: 캐시 오염, 대역폭 낭비                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프리패칭 성공률】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  지표                설명                     목표                    │ │
│  │  ──────              ────                     ────                    │ │
│  │  Prefetch Coverage  실제 미스 중 프리패칭으로   높을수록 좋음           │ │
│  │                     해결된 비율                                       │ │
│  │  Prefetch Accuracy  프리패칭 중 실제 사용된    높을수록 좋음           │ │
│  │                     비율                                             │ │
│  │  Cache Pollution    불필요한 프리패칭으로      낮을수록 좋음           │ │
│  │                     유용 데이터 밀려남                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 프리패칭 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                프리패칭 × 타 과목 융합                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 캐시: L1/L2/L3 모두 프리패칭                                        │
│  • TLB: TLB 프리패칭                                                │
│  • 명령어: I-Cache 프리패칭                                           │
│  • 하드웨어: 스트림 버퍼, 스트라이드 감지기                                 │
│                                                                     │
│  【운영체제】                                                          │
│  • 페이지 프리패칭: 페이지 폴트 시 인접 페이지 로드                          │
│  • 파일 프리패칭: readahead                                          │
│  • Swap: 미리 스왑 인                                                 │
│  • mmap: 페이지 예측 로드                                             │
│                                                                     │
│  【컴파일러】                                                          │
│  • 루프 최적화: 자동 프리패칭 삽입                                        │
│  • 프로파일 기반: 핫스팟에 프리패칭                                        │
│  • 데이터 분석: 접근 패턴 인식                                          │
│  • 프래그마: prefetch 지시어                                          │
│                                                                     │
│  【알고리즘】                                                          │
│  • 행렬 연산: 블록화 + 프리패칭                                         │
│  • 연결 리스트: 포인터 체이싱                                           │
│  • 트리 순회: 자식 노드 프리패칭                                         │
│  • 정렬: 버킷 프리패칭                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  계층별 프리패칭:                                                               │ │
│  │  ──────────────────────                                              │ │
│  │  계층          프리패칭 타입        주요 패턴                           │ │
│  │  ──────         ──────             ──────                             │ │
│  │  L1 I-Cache    Next-Line           순차적 코드                         │ │
│  │  L1 D-Cache    Stride, Stream      배열 접근                          │ │
│  │  L2 Cache      Stride, Next-Line   대용량 데이터                        │ │
│  │  L3 Cache      Limited             공유 데이터                          │ │
│  │  TLB           Page Table Walk     주소 변환                          │ │
│  │  Page Cache    Readahead           파일 접근                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 프리패칭 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                프리패칭 분석                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프리패칭 제어】                                                              │
│  ──────────────────                                                  │
│  # 하드웨어 프리패칭 확인/제어                                           │
│  $ cat /sys/devices/system/cpu/cpu0/cpue/cache/index0/prefetch      │
│                                                                     │
│  # 프리패칭 비활성화 (테스트용)                                           │
│  $ echo 0 > /sys/devices/system/cpu/cpu0/cpue/cache/index0/prefetch │
│                                                                     │
│  【소프트웨어 프리패칭 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 행렬 곱셈 최적화                                                  │ │
│  │  for (i = 0; i < N; i++) {                                         │ │
│  │      for (j = 0; j < N; j++) {                                     │ │
│  │          __builtin_prefetch(&A[i+1][j]);  // 다음 행 미리 로드        │ │
│  │          __builtin_prefetch(&B[i][j+8]);  // 8열 앞 미리 로드         │ │
│  │          C[i][j] = A[i][j] * B[i][j];                              │ │
│  │      }                                                            │ │
│  │  }                                                                │ │
│  │                                                               │ │
│  │  // 연결 리스트 순회                                                  │ │
│  │  while (node) {                                                    │ │
│  │      if (node->next)                                               │ │
│  │          __builtin_prefetch(node->next);  // 다음 노드 미리 로드      │ │
│  │      process(node);                                                │ │
│  │      node = node->next;                                            │ │
│  │  }                                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프리패칭 효과 측정】                                                         │
│  ──────────────────                                                  │
│  $ perf stat -e L1-dcache-prefetches,L1-dcache-prefetch-misses ./prog│
│  L1-dcache-prefetches: 10,000,000                                  │
│  L1-dcache-prefetch-misses: 500,000                                │
│  → 프리패칭 정확도 = 95%                                               │
│                                                                     │
│  【프리패칭 최적화 팁】                                                         │
│  ──────────────────                                                  │
│  1. 거리 조절: 너무 이르지도 늦지도 않게                                   │
│  2. 양 조절: 너무 많으면 캐시 오염                                        │
│  3. 패턴 활용: 순차, 스트라이드 패턴 인식                                   │
│  4. HW vs SW: 기본은 HW, 특수는 SW                                     │
│  5. 측정: 프리패칭 효과 반드시 측정                                        │
│  6. Non-temporal: 한 번만 쓸 데이터는 NT 프리패칭                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 프리패칭: 필요한 데이터를 미리 캐시에 로드
• 목표: Compulsory Miss 감소, 지연 숨김
• 종류: HW 프리패칭, SW 프리패칭, 하이브리드
• 패턴: 순차, 스트라이드, 역순, 포인터 체이닝
• HW: Next-Line, Stride, Stream Buffer
• SW: __builtin_prefetch, PREFETCH 명령어
• 지표: Coverage, Accuracy, Cache Pollution
• 거리: 너무 이르면 밀려남, 너무 늦으면 무효
• 측정: perf로 프리패칭 효과 확인
• 트레이드오프: 정확도 vs 커버리지 vs 오염
```
