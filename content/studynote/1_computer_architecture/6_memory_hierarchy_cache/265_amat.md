# 265. AMAT (Average Memory Access Time)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시 계층의 평균 메모리 접근 시간
> 2. **가치**: 메모리 성능의 핵심 평가 지표
> 3. **융합**: CPI, 파이프라인, 성능 분석과 직결

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

AMAT(Average Memory Access Time)는 **캐시 계층 구조에서 평균 메모리 접근 시간**이다. 히트 시간과 미스 페널티, 적중률을 고려하여 전체 메모리 접근 성능을 나타내는 핵심 지표다.

### 💡 비유: 식당 평균 대기 시간

AMAT는 **식당 평균 대기 시간**과 같다. 바로 자리가 나면 5분(히트), 예약이 필요하면 30분(미스). 전체 손님의 90%가 바로 들어가면 평균 대기는 7분. 이게 AMAT다.

### AMAT 기본 공식

```
┌─────────────────────────────────────────────────────────────────────┐
│                AMAT 개요                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【AMAT 기본 공식】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  AMAT = Hit Time + (Miss Rate × Miss Penalty)                  │ │
│  │                                                               │ │
│  │  또는                                                          │ │
│  │                                                               │ │
│  │  AMAT = (Hit Rate × Hit Time) + (Miss Rate × Miss Time)        │ │
│  │                                                               │ │
│  │  예:                                                           │ │
│  │  Hit Time = 1ns, Miss Rate = 5%, Miss Penalty = 100ns          │ │
│  │  AMAT = 1ns + (0.05 × 100ns) = 1 + 5 = 6ns                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【계층별 AMAT 확장】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  2계층 (L1 + L2):                                               │ │
│  │  AMAT = L1 Hit Time + L1 Miss Rate × L2 Hit Time               │ │
│  │                    + L1 Miss Rate × L2 Miss Rate × L2 Penalty   │ │
│  │                                                               │ │
│  │  3계층 (L1 + L2 + L3):                                          │ │
│  │  AMAT = L1 HT + L1 MR × L2 HT                                  │ │
│  │       + L1 MR × L2 MR × L3 HT                                  │ │
│  │       + L1 MR × L2 MR × L3 MR × DRAM Penalty                    │ │
│  │                                                               │ │
│  │  ┌────────────────────────────────────────────────────────┐    │ │
│  │  │  계층   Hit Time   Miss Rate   누적 AMAT                   │    │ │
│  │  │  ────   ─────────   ─────────   ──────────                   │    │ │
│  │  │  L1     1ns        5%          1 + 0.05 × ?                  │    │ │
│  │  │  L2     4ns        20%         1 + 0.05 × (4 + 0.2 × ?)     │    │ │
│  │  │  L3     15ns       20%         1 + 0.05 × (4 + 0.2 ×        │    │ │
│  │  │  DRAM   100ns      -                    (15 + 0.2 × 100))    │    │ │
│  │  │  ────────────────────────────────────────────────────────    │    │ │
│  │  │  AMAT = 1 + 0.05 × (4 + 0.2 × (15 + 0.2 × 100))              │    │ │
│  │  │       = 1 + 0.05 × (4 + 0.2 × 35)                            │    │ │
│  │  │       = 1 + 0.05 × 11                                        │    │ │
│  │  │       = 1 + 0.55 = 1.55ns                                    │    │ │
│  │  └────────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【AMAT vs 적중률】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  적중률    미스율    AMAT          성능 (vs L1만)                   │ │
│  │  ──────    ──────    ──────────     ──────────────                   │ │
│  │  100%      0%       1ns           100%                            │ │
│  │  99%       1%       2ns           50%                             │ │
│  │  98%       2%       3ns           33%                            │ │
│  │  95%       5%       6ns           17%                            │ │
│  │  90%       10%      11ns          9%                             │ │
│  │  80%       20%      21ns          5%                             │ │
│  │  → 미스율 1% 증가 = AMAT 크게 증가                                   │ │
│  │                                                               │ │
│  │  계산: AMAT = 1ns + MissRate × 100ns                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. AMAT 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                AMAT 상세                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【AMAT 구성 요소】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Hit Time (히트 시간):                                         │ │
│  │     • 캐시에서 데이터를 찾는 시간                                   │ │
│  │     • L1: 1~2ns (1~2 클럭)                                        │ │
│  │     • L2: 3~5ns (3~5 클럭)                                        │ │
│  │     • L3: 10~20ns (10~20 클럭)                                    │ │
│  │                                                               │ │
│  │  2. Miss Rate (미스율):                                           │ │
│  │     • 캐시 미스 비율                                               │ │
│  │     • L1: 5~10%                                                  │ │
│  │     • L2: 10~20% (L1 미스 중)                                      │ │
│  │     • L3: 10~20% (L2 미스 중)                                      │ │
│  │                                                               │ │
│  │  3. Miss Penalty (미스 페널티):                                    │ │
│  │     • 미스 시 하위 계층 접근 시간                                    │ │
│  │     • L1 미스 → L2: ~3ns                                          │ │
│  │     • L2 미스 → L3: ~10ns                                         │ │
│  │     • L3 미스 → DRAM: ~100ns                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【AMAT 최적화 전략】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전략              효과                 적용 방법                     │ │
│  │  ──────────         ──────────            ──────────                  │ │
│  │  Hit Time 감소      직접 AMAT 감소         작은 캐시, 단순 구조          │ │
│  │  Miss Rate 감소     AMAT 크게 감소         큰 캐시, 높은 연관도          │ │
│  │  Miss Penalty 감소  AMAT 감소              프리패칭, 인터리빙            │ │
│  │  계층 추가          전체 AMAT 감소         L4 캐시, HBM                 │ │
│  │                                                               │ │
│  │  트레이드오프:                                                      │ │
│  │  • Hit Time ↓ = Miss Rate ↑ (작은 캐시)                            │ │
│  │  • Miss Rate ↓ = Hit Time ↑ (큰 캐시, 높은 연관도)                   │ │
│  │  • 최적 균형점 찾기가 핵심                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【현대 시스템 AMAT 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템             L1 적중률   L2 적중률   L3 적중률   AMAT            │ │
│  │  ─────────────      ──────      ──────      ──────      ──────            │ │
│  │  Desktop PC         95%         80%         80%         1.5ns           │ │
│  │  Server             97%         85%         90%         1.2ns           │ │
│  │  HPC                98%         90%         95%         1.1ns           │ │
│  │  Mobile             92%         75%         70%         2.0ns           │ │
│  │                                                               │ │
│  │  Desktop AMAT 계산:                                                │ │
│  │  AMAT = 1 + 0.05 × (4 + 0.2 × (15 + 0.2 × 100))                  │ │
│  │       = 1 + 0.05 × 11 = 1.55ns                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. AMAT × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                AMAT × 타 과목 융합                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • CPI: AMAT가 메모리 CPI에 직접 영향                                  │
│  • 파이프라인: AMAT > 1 클럭이면 스톨                                   │
│  • 성능: AMAT가 전체 성능 결정                                         │
│  • 설계: AMAT 목표로 캐시 설계                                          │
│                                                                     │
│  【운영체제】                                                          │
│  • 페이지 폴트: 페이지 폴트 시 AMAT 급증                                 │
│  • TLB: TLB 미스가 AMAT에 영향                                         │
│  • 스케줄링: 캐시 웜업이 AMAT 영향                                       │
│  • 메모리 관리: 페이지 배치가 AMAT 영향                                  │
│                                                                     │
│  【컴파일러】                                                          │
│  • 최적화: AMAT 감소 목표                                              │
│  • 데이터 배치: 지역성 향상으로 AMAT 감소                                 │
│  • 루프 최적화: 캐시 효율로 AMAT 감소                                    │
│  • 프로파일링: AMAT 핫스팟 식별                                         │
│                                                                     │
│  【성능 분석】                                                         │
│  • 벤치마크: AMAT 측정                                                 │
│  • 프로파일링: 메모리 병목 식별                                          │
│  • 시뮬레이션: AMAT 예측                                               │
│  • 튜닝: AMAT 최적화                                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  AMAT와 CPI 관계:                                                          │ │
│  │  ──────────────────────                                              │ │
│  │  CPI = Base CPI + Memory CPI                                    │ │
│  │  Memory CPI = Memory Accesses/Inst × AMAT / Cycle Time           │ │
│  │                                                               │ │
│  │  예:                                                            │ │
│  │  Base CPI = 1, Memory Accesses/Inst = 1.3                        │ │
│  │  AMAT = 6ns, Cycle Time = 0.3ns (3.3GHz)                         │ │
│  │  Memory CPI = 1.3 × 6 / 0.3 = 26                                │ │
│  │  Total CPI = 1 + 26 = 27                                        │ │
│  │  → 메모리가 CPI 크게 저하                                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### AMAT 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                AMAT 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【AMAT 측정】                                                                │
│  ──────────────────                                                  │
│  $ perf stat -e cache-references,cache-misses,cycles ./prog        │
│  cache-references: 1,000,000,000                                    │
│  cache-misses: 50,000,000                                           │
│  cycles: 10,000,000,000                                             │
│  → 미스율 = 5%                                                       │
│  → 캐시 사이클/cycle = 캐시 접근당 사이클 계산                            │
│                                                                     │
│  【계층별 AMAT 분석】                                                         │
│  ──────────────────                                                  │
│  $ perf stat -e L1-dcache-loads,L1-dcache-load-misses \             │
│               -e LLC-loads,LLC-load-misses ./prog                   │
│  → L1, L3 미스율로 AMAT 계산                                          │
│                                                                     │
│  【AMAT 최적화 사례】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  최적화 전:                                                         │ │
│  │  L1 적중률: 90%, L2 적중률: 80%                                     │ │
│  │  AMAT = 1 + 0.1 × (4 + 0.2 × 100) = 3.4ns                         │ │
│  │                                                               │ │
│  │  최적화 후 (타일링, 데이터 재구성):                                     │ │
│  │  L1 적중률: 95%, L2 적중률: 90%                                     │ │
│  │  AMAT = 1 + 0.05 × (4 + 0.1 × 100) = 1.7ns                        │ │
│  │  → AMAT 50% 감소                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【AMAT 목표 설정】                                                           │
│  ──────────────────                                                  │
│  • L1 중심: AMAT < 2ns                                               │
│  • L2 중심: AMAT < 3ns                                               │
│  • L3 중심: AMAT < 5ns                                               │
│  • 메모리 중심: AMAT < 10ns                                           │
│                                                                     │
│  【AMAT 개선 팁】                                                             │
│  ──────────────────                                                  │
│  1. 적중률 향상: 캐시 크기, 연관도                                       │
│  2. 미스 페널티 감소: 프리패칭                                          │
│  3. 히트 시간 감소: 작은 캐시                                           │
│  4. 계층 최적화: 계층 간 비율                                           │
│  5. 데이터 지역성: 알고리즘 변경                                        │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• AMAT: 평균 메모리 접근 시간
• 공식: Hit Time + Miss Rate × Miss Penalty
• 계층별: L1 + L1 MR × (L2 + L2 MR × (L3 + L3 MR × DRAM))
• 구성: Hit Time, Miss Rate, Miss Penalty
• 적중률 영향: 1% 미스율 = AMAT 크게 증가
• CPI 연관: Memory CPI = Memory Accesses × AMAT / Cycle
• 최적화: 적중률 향상, 미스 페널티 감소
• 목표: L1 중심 < 2ns
• 측정: perf, cachegrind
• 트레이드오프: Hit Time vs Miss Rate
```
