# SDRAM (Synchronous DRAM)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SDRAM(Synchronous DRAM)은 시스템 클럭(Clock)에 동기화되어 동작하는 DRAM으로, 비동기식 DRAM 대비 **예측 가능한 타이밍과 버스트 전송**을 통해 대역폭을 획기적으로 향상시켰다.
> 2. **가치**: 클럭 동기화로 파이프라인드 동작이 가능해져, 연속된 데이터 접근 시 **평균 대역폭이 2~4배** 향상되며, 현대 DDR 메모리의 직접적 조상이다.
> 3. **융합**: 메모리 컨트롤러와의 정밀한 타이밍 협력, 다중 뱅크(Bank) 구조, CAS/RAS 신호의 클럭 동기화 등 현대 메모리 아키텍처의 기반을 확립했다.

+++

## Ⅰ. SDRAM의 개요

### 1. 정의
- SDRAM은 외부 클럭 신호에 맞춰 모든 동작(주소 전송, 데이터 입출력)이 동기화되는 DRAM이다. 1990년대 후반 도입되어 비동기식 DRAM(FPM, EDO)을 대체했다.

### 2. 등장 배경: 비동기 DRAM의 한계
- 기존 비동기 DRAM은 메모리 컨트롤러가 DRAM의 응답을 기다리는 방식이었다.
- CPU 클럭이 빨라지면서 메모리 지연의 불확실성이 병목이 되었다.
- SDRAM은 **"클럭 신호에 맞춰 딱딱 움직이자"**는 접근으로 이 문제를 해결했다.

### 3. 💡 비유: '오케스트라의 지휘봉'
- SDRAM은 **'지휘봉(클럭)에 맞춰 연주하는 오케스트라'**와 같다.
- 비동기 DRAM은 각 악기가 제멋대로 연주하는 반면, SDRAM은 모든 악기가 지휘자의 박자에 정확히 맞춰 연주한다. 협주가 훨씬 매끄러워진다.

- **📢 섹션 요약 비유**: 신호등이 없던 교차로에 신호등(클럭)을 설치해서 차들이 막힘없이 지나가게 만든 도로 시스템입니다.

## Ⅱ. 핵심 기술 및 동작 원리 (Deep Dive)

### 1. 클럭 동기화의 이점
- **예측 가능한 타이밍**: 클럭 사이클 단위로 명령/데이터가 전송되어 지연 시간 계산이 정확.
- **버스트 전송(Burst Transfer)**: 한 번의 명령으로 연속된 여러 워드를 전송.
- **파이프라이닝**: 명령어 디코딩, 행 활성화, 데이터 전송이 겹쳐서 수행 가능.

### 2. SDRAM 내부 구조
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                    SDRAM 내부 구조                               │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  ┌─────────────────────────────────────────────────────────┐   │
    │  │                 메모리 컨트롤러                           │   │
    │  └─────────────────────────────────────────────────────────┘   │
    │                          │                                      │
    │           CLK (클럭) ────┼──────────────────→ 동기화 신호      │
    │           CMD (명령) ────┼────→ READ/WRITE/ACTIVATE           │
    │           ADDR ──────────┼────→ 행/열 주소                     │
    │           DQ ────────────┼←───→ 데이터 입출력                  │
    │                          │                                      │
    │  ┌───────────────────────┴───────────────────────────────┐    │
    │  │                     SDRAM 칩                           │    │
    │  │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  │    │
    │  │  │ Bank 0  │  │ Bank 1  │  │ Bank 2  │  │ Bank 3  │  │    │
    │  │  │ Rows    │  │ Rows    │  │ Rows    │  │ Rows    │  │    │
    │  │  │ Cols    │  │ Cols    │  │ Cols    │  │ Cols    │  │    │
    │  │  └─────────┘  └─────────┘  └─────────┘  └─────────┘  │    │
    │  │                                                        │    │
    │  │  * 다중 뱅크: 한 뱅크가 리프레시 중이어도 다른 뱅크 접근 가능│
    │  └────────────────────────────────────────────────────────┘    │
    └─────────────────────────────────────────────────────────────────┘
```

### 3. 버스트 전송 모드
- SDRAM은 한 번의 READ 명령으로 연속된 주소의 데이터(예: 4, 8 워드)를 순차적으로 전송한다.
- **버스트 길이(Burst Length)**: 1, 2, 4, 8, Full Page 등 설정 가능.
- 캐시 라인 단위 접근(64바이트)에 최적화.

### 4. 다중 뱅크(Multi-Bank) 구조
- SDRAM 칩 내부를 여러 독립적인 뱅크로 분할.
- 한 뱅크에서 행 활성화 중 다른 뱅크 접근 가능 → **인터리빙 효과**.
- 현대 DDR4/DDR5는 8~16개 뱅크 보유.

- **📢 섹션 요약 비유**: 하나의 큰 창고 대신 여러 개의 작은 창고로 나누어, 한 창고에서 물건 찾는 동안 다른 창고에서도 작업할 수 있게 만든 구조입니다.

## Ⅲ. SDRAM vs 비동기 DRAM 비교

| 비교 항목 | 비동기 DRAM (FPM/EDO) | SDRAM |
|:---|:---|:---|
| **동기화** | 없음 (컨트롤러 대기) | 클럭 동기화 |
| **버스트 전송** | 제한적 | 완전 지원 |
| **파이프라이닝** | 불가능 | 가능 |
| **대역폭** | 낮음 (~100 MB/s) | 높음 (SDR: ~1 GB/s) |
| **타이밍 제어** | 불확실 | 정확 (클럭 기반) |

+++

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: SDR → DDR 진화
- **SDR (Single Data Rate)**: 클럭 상승 에지에서만 전송.
- **DDR (Double Data Rate)**: 상승/하강 에지 모두 전송 → 대역폭 2배.
- DDR은 SDRAM의 직접적 후속으로, 기본 원리(클럭 동기화, 버스트, 뱅크)는 동일.

### 2. 타이밍 파라미터 최적화
- SDRAM의 성능은 CL, tRCD, tRP 파라미터 설정에 따라 크게 달라짐.
- BIOS/UEFI에서 메모리 타이밍 조정으로 성능 튜닝 가능.
- 타이트한 타이밍 = 빠르지만 불안정, 느슨한 타이밍 = 안정적.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **대역폭 향상**: 비동기 DRAM 대비 2~4배.
- **시스템 통합 용이**: 클럭 기반 설계로 메모리 컨트롤러 설계 단순화.

### 2. 역사적 의의
- SDRAM은 현대 DDR 메모리의 **원형(Prototype)**이다.
- 클럭 동기화, 버스트 전송, 다중 뱅크 등 핵심 개념이 DDR5까지 이어짐.

## 👶 어린이를 위한 3줄 비유 설명
1. SDRAM은 **'박자에 맞춰 노래하는 합창단'** 같아요.
2. 지휘자(클럭)가 박자를 주면 모두 딱 맞춰서 노래(데이터 전송)를 하니까 소리가 엉키지 않아요.
3. 덕분에 박자 없이 제멋대로 부르던 예전 합창단보다 훨씬 아름다운 노래가 나온답니다!
