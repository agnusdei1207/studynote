# 순차적 지역성 (Sequential Locality)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 순차적 지역성(Sequential Locality)은 프로그램이 메모리의 **연속된 주소 공간을 순서대로 접근**하는 경향성을 의미하며, 공간적 지역성의 특수 형태다.
> 2. **가치**: 배열 순회, 명령어 인출, 스트리밍 데이터 처리 등에서 나타나며, **하드웨어 프리페처(Prefetcher)가 다음 데이터를 예측**하는 핵심 근거가 된다.
> 3. **융합**: 캐시 라인 크기 결정, 메모리 인터리빙, 순차 접근 감지 로직 등 메모리 계층 전반의 설계에 영향을 미친다.

+++

## Ⅰ. 순차적 지역성의 개요

### 1. 정의
- 순차적 지역성은 프로그램이 메모리 주소 A를 접근한 후, A+1, A+2, A+3...과 같이 **연속적이고 순차적인 주소**를 접근할 확률이 높다는 관찰에 기반한다.
- 이는 시간적 지역성(같은 데이터 반복 접근)과 공간적 지역성(인접 데이터 접근)의 하위 개념으로, 특히 **순차적 패턴**에 초점을 맞춘다.

### 2. 등장 배경: 데이터 구조의 본질
- 대부분의 프로그래밍 언어에서 **배열(Array)**은 메모리에 연속적으로 저장된다.
- for 루프로 배열을 순회하는 것은 가장 흔한 프로그래밍 패턴이며, 이는 곧 순차적 메모리 접근을 의미한다.
- 명령어 역시 프로그램 카운터(PC)가 순차적으로 증가하며 인출된다.

### 3. 💡 비유: '책을 처음부터 끝까지 읽기'
- 순차적 지역성은 **'소설책을 1페이지부터 순서대로 읽는 독자'**와 같다.
- 100페이지를 읽은 독자가 다음에 101페이지를 읽을 확률은 거의 100%다. 랜덤하게 페이지를 펼치는 독자보다 예측이 훨씬 쉽다.

- **📢 섹션 요약 비유**: 도서관에서 책을 빌릴 때, 같은 시리즈의 1권을 빌린 사람은 곧 2, 3권을 빌릴 확률이 높다는 원리입니다.

+++

## Ⅱ. 메커니즘 및 하드웨어 활용 (Deep Dive)

### 1. 스트림 프리페처 (Stream Prefetcher)
- 현대 CPU는 순차적 지역성을 활용하여 **다음 캐시 라인을 미리 가져오는(Pre-fetch)** 하드웨어를 내장한다.
- 메모리 접근 패턴이 A → A+64 → A+128...으로 감지되면, 자동으로 A+192, A+256...을 프리페치한다.

```text
    ┌─────────────────────────────────────────────────────────────────┐
    │              순차적 지역성 기반 프리페칭                          │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  CPU 요청:  [Addr 0x1000]                                       │
    │                    ↓                                            │
    │  캐시 미스 → 메모리에서 64바이트 라인 가져옴                       │
    │                    ↓                                            │
    │  하드웨어 감지: "순차적 패턴 시작!"                               │
    │                    ↓                                            │
    │  자동 프리페치: 0x1040, 0x1080, 0x10C0...                       │
    │                                                                 │
    │  결과: CPU가 요청하기 전에 이미 캐시에 데이터 존재!               │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 메모리 인터리빙과의 연계
- 순차적 접근은 여러 메모리 뱅크에 분산되어 병렬 처리될 수 있다.
- **인터리빙**: 주소 A는 뱅크 0, A+64는 뱅크 1, A+128은 뱅크 2...으로 분산.
- 순차적 요청이 여러 뱅크에 분산되어 대역폭 활용이 극대화된다.

### 3. 캐시 라인 크기 결정
- 순차적 지역성이 높을수록 **큰 캐시 라인**이 유리하다. 64바이트 대신 128바이트를 한 번에 가져오면 미스 횟수가 줄어든다.
- 하지만 순차적이지 않은 랜덤 접근(포인터 체이싱)에서는 큰 라인이 낭비가 된다.

- **📢 섹션 요약 비유**: 백화점에서 쇼핑할 때, 한 코너에서 연속으로 물건을 담으면 카트를 크게 준비하는 게 효율적입니다.

+++

## Ⅲ. 순차적 vs 비순차적 접근 비교

| 특성 | 순차적 접근 (Sequential) | 비순차적 접근 (Random/Strided) |
|:---|:---|:---|
| **예시** | 배열 순회, 행렬 연산 | 연결 리스트, 포인터 체이싱 |
| **프리페칭** | 매우 효과적 | 효과 낮음 |
| **캐시 효율** | 높음 (높은 히트율) | 낮음 (불필요한 데이터 로드) |
| **메모리 대역폭** | 효율적 사용 | 낭비 발생 |
| **TLB 효율** | 높음 (페이지 단위 접근) | 낮음 (페이지 폴트 빈번) |

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: 행렬 연산의 캐시 최적화
- C언어에서 2차원 배열 `A[i][j]`는 행 우선(Row-major) 순서로 저장된다.
- `for(i) for(j)` 순서로 접근하면 순차적 지역성 활용 (캐시 히트율 높음).
- `for(j) for(i)` 순서로 접근하면 비순차적 (캐시 스래싱 발생).
- **결론**: 루프 순서 최적화(Loop Interchange)는 성능에 결정적 영향.

### 2. 안티패턴: 포인터 체이싱
- 연결 리스트, 트리 구조는 순차적 지역성이 거의 없다.
- 각 노드가 힙 영역의 임의 위치에 할당되어 캐시 미스가 빈번.
- 해결: 메모리 풀(Memory Pool)로 연속 할당 또는 배열 기반 구조로 재설계.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **프리페치 정확도 향상**: 순차적 패턴 감지 시 90%+ 프리페치 히트율.
- **메모리 대역폭 활용**: 버스트 전송(Burst Transfer)으로 효율 극대화.

### 2. 미래 전망
- AI/ML 워크로드는 주로 행렬 연산(순차적 접근)으로 구성되어 캐시 최적화에 유리.
- NPU 설계에서도 순차적 지역성을 활용한 효율적 메모리 접근이 핵심.

## 👶 어린이를 위한 3줄 비유 설명

2. 1권을 읽고 나면 2권을 읽을 확률이 매우 높으니까, 도서관 사서님(CPU)이 미리 2, 3권을 책상 위에 올려두는 거예요.
3. 덕분에 다음 책을 읽고 싶을 때 바로 읽을 수 있어서 독서 시간이 아껴진답니다!
