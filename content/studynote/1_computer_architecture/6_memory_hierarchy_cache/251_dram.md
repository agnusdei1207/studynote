# DRAM (Dynamic RAM)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DRAM(Dynamic Random Access Memory)은 **1개의 트랜지스터와 1개의 커패시터(1T1C)**로 구성된 셀을 사용하여 전하의 유무로 데이터를 저장하는 휘발성 메모리로, 주기적 리프레시(Refresh)가 필요하다.
> 2. **가치**: SRAM 대비 **면적과 비용이 1/4~1/6** 수준으로, **기가바이트~테라바이트급 대용량 메인 메모리**를 경제적으로 구현할 수 있는 유일한 실용적 기술이다.
> 3. **융합**: DDR(Double Data Rate) 기술로 진화하며, CPU와의 속도 격차를 인터리빙, 버스트 전송, 채널 병렬화로 극복하고 있다.

---

## Ⅰ. DRAM의 개요

### 1. 정의
- DRAM은 커패시터에 저장된 전하(Charge)로 데이터를 표현하는 휘발성 메모리다. '동적(Dynamic)'이라는 이름은 커패시터의 누설 전류로 인해 데이터가 시간이 지나면 사라지기 때문에 **주기적으로 다시 충전(Refresh)**해야 한다는 데서 유래했다.

### 2. 등장 배경: 대용량 메모리의 필요성
- SRAM은 빠르지만 6T 구조로 면적이 커서 대용량 구현이 어렵다.
- 1T1C 구조의 DRAM은 밀도가 높아 **메인 메모리(RAM)**로 전 세계 모든 컴퓨터에 사용된다.

### 3. 💡 비유: '물탱크에 물 채우기'
- DRAM은 **'구멍 난 물탱크에 물을 채워두는 것'**과 같다.
- 물(전하)이 조금씩 새서 시간이 지나면 바닥난다. 그래서 정기적으로 물을 다시 채워줘야(Refresh) 한다. 물이 있으면 1, 없으면 0이다.

- **📢 섹션 요약 비유**: 저렴하지만 관리(리프레시)가 필요한 '대형 창고'입니다.

## Ⅱ. 셀 구조 및 동작 원리 (Deep Dive)

### 1. 1T1C DRAM 셀 구조
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                    1T1C DRAM 셀 구조                             │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │    Bit Line (BL) ─────┬──────────────────────                  │
    │                       │                                       │
    │                    ┌──┴──┐                                     │
    │                    │  T  │ ← 액세스 트랜지스터 (워드라인 제어)   │
    │                    └──┬──┘                                     │
    │                       │                                        │
    │          ┌────────────┴────────────┐                          │
    │          │                         │                          │
    │          │      커패시터 (C)        │ ← 전하 저장 (데이터)      │
    │          │                         │                          │
    │          └────────────┬────────────┘                          │
    │                       │                                        │
    │                    Ground                                       │
    │                                                                 │
    │    Word Line (WL) ────────→ 트랜지스터 게이트 제어              │
    │                                                                 │
    │    데이터 저장: 커패시터 전하 유무                               │
    │    - 전하 있음 = 1                                              │
    │    - 전하 없음 = 0                                              │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 동작 모드
- **읽기(Read)**: 워드라인 활성화 → 커패시터 전하가 비트라인으로 이동 → 센스 앰프로 미세 전압차 증폭 → **파괴적 읽기: 읽으면 전하가 사라짐!**
- **쓰기(Write)**: 워드라인 활성화 → 비트라인에 전압 인가 → 커패시터 충전/방전
- **리프레시(Refresh)**: 주기적으로(일반적으로 64ms 간격) 모든 셀을 읽고 다시 써서 전하 유지

### 3. 리프레시의 부담
- 리프레시 동안에는 일반 읽기/쓰기가 불가능하여 **성능 저하** 발생.
- 고용량 DRAM일수록 리프레시에 걸리는 시간이 증가.
- 최신 기술: **FBGA(Fine Granularity Refresh)**, **SRA(Self Refresh Abort)** 등으로 오버헤드 감소.

- **📢 섹션 요약 비유**: 도서관에서 책을 읽으면 책이 사라지는 구조라, 읽고 나서 바로 복사해서 다시 꽂아야 합니다.

## Ⅲ. DRAM 성능 지표 및 진화 (Evolution)

### 1. 주요 타이밍 파라미터 (CL-tRCD-tRP)
- **CL (CAS Latency)**: 열 주소 전달 후 데이터 출력까지 지연 시간
- **tRCD (RAS to CAS Delay)**: 행 주소 전달 후 열 주소 전달까지 대기 시간
- **tRP (Row Precharge)**: 행 닫고 새 행 열기까지 대기 시간
- 예: DDR4-3200 CL16 → 16 클럭 지연

### 2. DDR 세대별 비교
| 세대 | 전송 속도 | 전압 | 주요 특징 |
|:---|:---|:---|:---|
| DDR3 | 800~2133 MT/s | 1.5V | 대중화 |
| DDR4 | 2133~3200 MT/s | 1.2V | 저전력, 뱅크 그룹 |
| DDR5 | 4800~8400 MT/s | 1.1V | 온디 ECC, 2채널 모듈 |

### 3. 구조적 병목: 행 버퍼(Row Buffer)
- DRAM은 행 단위로 버퍼에 로드 후 열 접근.
- **행 버퍼 히트**: 같은 행 접근 시 빠름.
- **행 버퍼 미스**: 다른 행 접근 시 느림 (Precharge + Activate 필요).
- **행해머(Rowhammer)**: 같은 행 반복 접근 시 인접 행 비트 뒤집힘 (보안 취약점).

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: 메모리 대역폭 계산
```
대역폭 = 데이터 폭 × 클럭 속도 × 전송당 데이터 수
DDR4-3200 듀얼 채널: 64bit × 2 × 1.6GHz × 2 = 51.2 GB/s
```
- 고성능 GPU/서버는 채널 수(8채널, 12채널)를 늘려 대역폭 확보.

### 2. 안티패턴: DRAM 지연을 무시한 설계
- DRAM 랜덤 접근은 수백 클럭 소요.
- 순차적 접근 패턴 설계, 프리페칭, 캐시 활용이 필수.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **경제적 대용량**: $20/GB 수준으로 테라바이트급 메모리 구현 가능.
- **범용성**: 모든 컴퓨팅 시스템의 메인 메모리 표준.

### 2. 미래 전망
- DDR6 개발 진행 중 (2025+)
- 3D DRAM(HBM), CXL 메모리 확장으로 대역폭/용량 계속 확장.
