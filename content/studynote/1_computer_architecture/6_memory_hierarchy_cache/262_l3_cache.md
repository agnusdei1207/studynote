# L3 캐시

## 핵심 인사이트
> 1. **본질**: L3(Level 3) 캐시는 모든 CPU 코어가 데이터를 공유하는 **마지막 보루(LLC, Last Level Cache)**로, 메인 메모리(DRAM)로 나가는 끔찍한 지연을 막는 거대한 수조다.
> 2. **역할**: 코어 간의 데이터 통신(Shared Data)을 가속하며, 멀티코어 간 일관성을 유지하는 핵심 플랫폼 역할을 수행한다.
> 3. **성능**: 최근 AMD의 3D V-Cache처럼 수직 적층 기술을 통해 용량을 극대화하여 게임 및 연산 성능을 비약적으로 끌어올리는 "캐시 용량의 깡패"가 되고 있다.

---

## Ⅰ. L3 캐시의 특성
- **위치**: CPU 다이 내부, 모든 코어의 집합 근처.
- **속도**: 40~80 Cycles (RAM의 300+ Cycles에 비하면 여전히 압도적으로 빠름).
- **용량**: 수십 MB ~ 100MB+ (대용량화 추세).
- **공유 여부**: **모든 코어 공유 (Shared)**. 코어 1이 쓴 데이터를 코어 2가 RAM까지 안 가고 여기서 즉시 읽어올 수 있음.

## Ⅱ. 최후의 캐시 (Last Level Cache)
- CPU 다이 내에서 메모리 컨트롤러를 만나기 전 마지막으로 거치는 캐시 단계다.
- 여기서도 데이터가 없으면(Miss), CPU는 모든 파이프라인 연산을 멈추고(Stall) 수백 배 느린 DRAM이 데이터를 줄 때까지 기다려야 하는 막대한 페널티를 입는다.

## Ⅲ. 혁신 사례: 3D V-Cache
- 기존의 평면적인 다이 설계에서 벗어나, L3 캐시 위에 똑같은 용량의 캐시 다이를 층층이 쌓아 올려 96MB 이상의 거대한 LLC를 구현한 사례로, 데이터 접근을 극대화하여 실제 성능(특히 게임)에서 인텔을 압도하는 결과를 가져왔다.
