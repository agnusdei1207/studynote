# 262. L3 캐시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 코어 간 공유되는 최하위 계층 온칩 캐시
> 2. **가치**: 큰 용량, 코어 간 데이터 공유
> 3. **융합**: LLC, 캐시 일관성, 멀티코어 성능

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

L3 캐시(Level 3 Cache)는 **여러 CPU 코어가 공유하는 최하위 계층의 온칩 캐시**이다. Last Level Cache(LLC)라고도 하며, 큰 용량으로 코어 간 데이터 공유와 L2 미스 처리를 담당한다.

### 💡 비유: 가족 공용 책장

L3 캐시는 **가족 공용 책장**과 같다. 책상(L1)과 옆 책장(L2)에 없으면 거실 책장에서 찾는다. 가족 모두가 공용으로 쓴다. 한 사람이 쓰던 책을 다른 사람도 쓸 수 있다.

## Ⅱ. L3 캐시 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                L3 캐시 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【L3 캐시 분할 방식】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Monolithic L3 (단일):                                          │ │
│  │     • 하나의 큰 L3를 모든 코어가 공유                                 │ │
│  │     • 단순한 설계                                                   │ │
│  │     • 예: Intel 소켓당 하나                                          │ │
│  │                                                               │ │
│  │  2. Distributed L3 (분산):                                          │ │
│  │     • L3를 여러 슬라이스로 분할                                       │ │
│  │     • 각 코어가 일부 슬라이스 접근                                     │ │
│  │     • 해시 함수로 슬라이스 결정                                        │ │
│  │     • 예: Intel Mesh, AMD Chiplet                                   │ │
│  │                                                               │ │
│  │  L3 슬라이스 구조:                                                    │ │
│  │  ┌────────────────────────────────────────────────────────────┐   │ │
│  │  │  코어0     코어1     코어2     코어3     코어4     코어5    │   │ │
│  │  │   │         │         │         │         │         │       │   │ │
│  │  │   ▼         ▼         ▼         ▼         ▼         ▼       │   │ │
│  │  │ ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐            │   │ │
│  │  │ │L3 │   │L3 │   │L3 │   │L3 │   │L3 │   │L3 │  슬라이스    │   │ │
│  │  │ │ 0 │   │ 1 │   │ 2 │   │ 3 │   │ 4 │   │ 5 │            │   │ │
│  │  │ └───┘   └───┘   └───┘   └───┘   └───┘   └───┘            │   │ │
│  │  │                   │                                      │   │ │
│  │  │                   ▼  인터커넥트 (Ring/Mesh)               │   │ │
│  │  └────────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【L3 캐시 일관성】                                                             │|
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  MESI 프로토콜로 L3 일관성 유지:                                     │ │
│  │                                                               │ │
│  │  M (Modified): 수정됨, 이 코어만 보유                                 │ │
│  │  E (Exclusive): 배타적, 이 코어만 보유                                │ │
│  │  S (Shared): 공유, 여러 코어가 보유                                    │ │
│  │  I (Invalid): 무효, 데이터 없음                                        │ │
│  │                                                               │ │
│  │  L3 역할:                                                            │ │
│  │  • 디렉터리: 각 캐시 라인의 상태 추적                                   │ │
│  │  • 중재: 코어 간 캐시 일관성 중재                                       │ │
│  │  • 스누핑: 다른 코어의 접근 감시                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【현대 CPU L3 캐시 사양】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU             L3 크기       구조         코어 수               │ │
│  │  ─────────────   ──────────    ──────────     ──────                   │ │
│  │  Intel i9        32MB         Distributed    24코어                  │ │
│  │  AMD Ryzen 9     64MB         Distributed    16코어                  │ │
│  │  AMD EPYC        256MB        Chiplet       64코어                   │ │
│  │  ARM Neoverse    64MB         Distributed    64코어                  │ │
│  │  → AMD EPYC가 가장 큰 L3                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### L3 캐시 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                L3 캐시 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【L3 캐시 정보 확인】                                                         │
│  ──────────────────                                                  │
│  $ lscpu | grep L3                                                  │
│  L3 cache: 8192 KiB                                                 │
│                                                                     │
│  【L3 캐시 적중률 측정】                                                         │
│  ──────────────────                                                  │
│  $ perf stat -e LLC-loads,LLC-load-misses ./prog                    │
│  LLC-loads: 100,000,000                                             │
│  LLC-load-misses: 2,000,000                                         │
│  → L3 적중률 = 98%                                                    │
│                                                                     │
│  【CAT (Cache Allocation Technology)】                                          │
│  ──────────────────                                                  │
│  # L3 캐시 파티션 확인                                                 │
│  $ cat /sys/fs/resctrl/info/L3/cbm_mask                             │
│  7ff                                                                │
│                                                                     │
│  # 파티션 설정                                                         │
│  $ echo "L3:0=7ff;1=7ff" > /sys/fs/resctrl/schemata                │
│                                                                     │
│  【L3 캐시 최적화 팁】                                                         │
│  ──────────────────                                                  │
│  1. 데이터 배치: L3 크기 고려                                          │
│  2. 코어 간 통신: L3 공유 활용                                          │
│  3. CAT 활용: 워크로드 격리                                             │
│  4. NUMA 인식: 로컬 L3 우선                                            │
│  5. 페이지 컬러링: L3 충돌 감소                                          │
│                                                                     │
│  【L3 vs DRAM 성능 비교】                                                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  계층         지연          대역폭          적중률                   │ │
│  │  ──────────     ──────────     ──────────     ──────                     │ │
│  │  L3            10~20ns        수십 GB/s       98~99%                    │ │
│  │  DRAM          50~100ns       수 GB/s        -                         │ │
│  │  → L3가 5~10배 빠름                                               │ │
│  │  → L3 적중 시 DRAM 접근 불필요                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• L3 캐시: 코어 간 공유되는 최하위 온칩 캐시 (LLC)
• 크기: 8MB ~ 256MB (매우 큼)
• 지연: 10~20 클럭 (약 10~20ns)
• 구조: Monolithic, Distributed, Chiplet
• 공유: 모든 코어가 공유
• 일관성: MESI 프로토콜로 유지
• 적중률: 98~99%
• 연관도: 16~32 way Set Associative
• 인터커넥트: Ring, Mesh, NoC
• 보안: CAT로 파티셔닝, Spectre 완화
```
