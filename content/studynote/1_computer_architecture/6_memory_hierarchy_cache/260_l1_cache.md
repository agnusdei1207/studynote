# L1 캐시

## 핵심 인사이트
> 1. **본질**: L1(Level 1) 캐시는 CPU 코어와 가장 가까이 위치하여 **프로세서 파이프라인과 동일한 속도**로 동작하는 초고속 캐시 메모리다.
> 2. **구조**: 명령어 인출과 데이터 접근의 충돌을 막기 위해 **명령어 캐시(L1i)와 데이터 캐시(L1d)**로 분리된 하버드 아키텍처 구조를 주로 채택한다.
> 3. **특징**: 용량은 수십 KB로 매우 작지만, 1~4 클럭 사이클 내에 응답하여 CPU가 메인 메모리의 느린 속도에 발목 잡히지 않게 하는 최전방 방어선이다.

---

## Ⅰ. L1 캐시의 특성
- **위치**: CPU 코어 내부에 직접 통합되어 있음.
- **속도**: 1~4 Cycles (현존하는 메모리 중 가장 빠름).
- **용량**: 32KB ~ 64KB (보통 코어당 할당).
- **공유 여부**: **코어 독점 (Private)**. 각 코어는 자신만의 L1 캐시를 가짐.

## Ⅱ. 명령어/데이터 분리 (Split Cache)
- L1 캐시는 특이하게 두 개로 쪼개져 있다.
- **L1i (Instruction Cache)**: 실행할 명령어들을 보관.
- **L1d (Data Cache)**: 연산에 필요한 데이터를 보관.
- **이유**: CPU 파이프라인이 명령어를 읽는 동시에 데이터를 읽거나 쓸 수 있도록 하여 구조적 해저드(Structural Hazard)를 방지하기 위함이다.

## Ⅲ. 포함 정책 (Inclusion Policy)
- 주로 L1 캐시는 하위 캐시(L2, L3)의 부분집합으로 존재하거나(Inclusive), 독자적인 데이터를 가짐(Exclusive). 인텔 프로세서의 경우 L1의 내용은 보통 L3에도 존재한다.
