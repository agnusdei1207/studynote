# 260. L1 캐시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 코어 내부의 최고속 1차 캐시
> 2. **가치**: 명령어/데이터 분리, 1~2클럭 지연
> 3. **융합**: 파이프라인과 밀접, 가장 높은 적중률

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

L1 캐시(Level 1 Cache)는 **CPU 코어 내부에 위치한 최고속 1차 캐시**이다. 명령어 캐시(I-Cache)와 데이터 캐시(D-Cache)로 분리되며, 1~2클럭(약 1~2ns)의 접근 시간을 가진다.

### 💡 비유: 책상 서랍

L1 캐시는 **책상 서랍**과 같다. 손을 뻗어 바로 꺼낼 수 있다. 가장 빠르다. 하지만 많이 담을 수 없다. 가장 자주 쓰는 것만 담는다.

### L1 캐시 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                L1 캐시 개요                                               │
├─────────────────────────────────────────────────────────────────────┤

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                      CPU 코어                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │                                                       │   │ │
│  │  │   ┌──────────────┐          ┌──────────────┐            │   │ │
│  │  │   │  L1 I-Cache   │          │  L1 D-Cache   │            │   │ │
│  │  │   │  (명령어)      │          │  (데이터)      │            │   │ │
│  │  │   │  32KB         │          │  32KB         │            │   │ │
│  │  │   │  4-way SA     │          │  8-way SA     │            │   │ │
│  │  │   └───────┬──────┘          └───────┬──────┘            │   │ │
│  │  │           │                         │                    │   │ │
│  │  │           └───────────┬─────────────┘                    │   │ │
│  │  │                       │                                  │   │ │
│  │  │               ┌───────┴───────┐                          │   │ │
│  │  │               │    L2 캐시     │                          │   │ │
│  │  │               │   256KB~1MB    │                          │   │ │
│  │  │               └───────────────┘                          │   │ │
│  │  │                                                       │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Split Cache vs Unified Cache】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Split Cache (L1):                                                │ │
│  │  • 명령어 캐시와 데이터 캐시 분리                                    │ │
│  │  • 동시에 명령어 인출 + 데이터 접근 가능                              │ │
│  │  • Harvard 아키텍처 기반                                            │ │
│  │                                                               │ │
│  │  Unified Cache (L2/L3):                                           │ │
│  │  • 명령어와 데이터를 같은 캐시에 저장                                 │ │
│  │  • 유연한 공간 활용                                                 │ │
│  │  • 폰 노이만 아키텍처 기반                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【L1 캐시 특징】                                                               │
│  ──────────────────                                                  │
│  • 크기: 32~64KB (I-Cache + D-Cache)                                 │
│  • 지연: 1~2 클럭 (약 0.3~1ns)                                        │
│  • 대역폭: 수 TB/s                                                    │
│  • 연관도: 4~8 way Set Associative                                    │
│  • 라인 크기: 64B                                                     │
│  • 적중률: 90~95%                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. L1 캐시 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                L1 캐시 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【L1 I-Cache (명령어 캐시)】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  역할: 명령어 인출 속도 향상                                         │ │
│  │                                                               │ │
│  │  특징:                                                            │ │
│  │  • 읽기 전용 (코드는 수정되지 않음)                                  │ │
│  │  • 더티 비트 불필요                                                 │ │
│  │  • 쓰기 정책 불필요                                                 │ │
│  │  • 높은 공간적 지역성 (순차적 명령어)                                 │ │
│  │                                                               │ │
│  │  구조:                                                            │ │
│  │  • 크기: 32KB (일반적)                                             │ │
│  │  • 연관도: 4-way Set Associative                                   │ │
│  │  • 라인 크기: 64B (16개 명령어 @ 32bit)                              │ │
│  │  • 적중률: 95~98%                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【L1 D-Cache (데이터 캐시)】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  역할: 데이터 접근 속도 향상                                         │ │
│  │                                                               │ │
│  │  특징:                                                            │ │
│  │  • 읽기/쓰기 모두 가능                                              │ │
│  │  • 더티 비트 필요 (수정된 데이터 추적)                                │ │
│  │  • 쓰기 정책 필요 (Write-Through/Write-Back)                        │ │
│  │  • 다양한 접근 패턴                                                 │ │
│  │                                                               │ │
│  │  구조:                                                            │ │
│  │  • 크기: 32KB (일반적)                                             │ │
│  │  • 연관도: 8-way Set Associative                                   │ │
│  │  • 라인 크기: 64B                                                  │ │
│  │  • 적중률: 90~95%                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【L1 캐시 파이프라인】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  I-Cache 파이프라인:                                                │ │
│  │  클럭  IF1    IF2    ID     EX     MEM    WB                       │ │
│  │  ────  ────   ────   ────   ────   ────   ────                    │ │
│  │   1    태그    인덱스                                          │ │
│  │   2    조회    데이터                                          │ │
│  │   3           명령어                                          │ │
│  │                                                               │ │
│  │  → L1 I-Cache 적중: 2 클럭                                         │ │
│  │  → L1 I-Cache 미스: 10~20 클럭 (L2/L3/DRAM)                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                L1 캐시 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │

│  ──────────────────                                                  │
│  $ lscpu | grep L1                                                  │
│  L1d cache: 64 KiB                                                  │

│                                                                     │
│  【L1 캐시 적중률 측정】                                                         │
│  ──────────────────                                                  │
│  $ perf stat -e L1-dcache-loads,L1-dcache-load-misses ./prog       │
│  L1-dcache-loads: 10,000,000,000                                    │
│  L1-dcache-load-misses: 500,000,000                                 │
│  → L1 D-Cache 적중률 = 95%                                           │
│                                                                     │
│  $ perf stat -e L1-icache-load-misses ./prog                        │
│  L1-icache-load-misses: 10,000,000                                  │
│  → I-Cache 미스 확인                                                 │
│                                                                     │
│  【L1 캐시 최적화 팁】                                                         │
│  ──────────────────                                                  │
│  1. 코드 크기: L1 I-Cache 크기에 맞게                                  │
│  2. 핫 코드: 자주 실행 코드를 작게 유지                                  │
│  3. 데이터 지역성: L1 크기에 맞는 워킹셋                                 │
│  4. 루프 최적화: L1 크기 고려 타일링                                     │
│  5. 함수 인라이닝: 적절한 크기 유지                                       │
│                                                                     │
│  【L1 캐시 문제 해결】                                                         │
│  ──────────────────                                                  │
│  • 높은 미스율: 워킹셋이 L1보다 큼 → L2/L3 활용                          │
│  • 코드 팽창: 인라이닝 과다 → 크기 조정                                   │
│  • 데이터 스필: 지역 변수 과다 → 구조체 활용                               │
│  • 캐시 충돌: 같은 인덱스 접근 → 데이터 재배치                              │
│                                                                     │
│  【L1 vs L2 vs L3 성능 비교】                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  계층    크기         지연       대역폭       적중률                 │ │
│  │  ────    ──────        ────       ──────       ──────                 │ │
│  │  L1      32~64KB      1~2ns      수 TB/s      90~95%                 │ │
│  │  L2      256KB~1MB    3~5ns      수백 GB/s    95~98%                 │ │
│  │  L3      8~64MB       10~20ns    수십 GB/s    98~99%                 │ │
│  │  → L1이 가장 빠르지만 가장 작음                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• L1 캐시: CPU 코어 내부의 최고속 1차 캐시
• 구조: Split Cache (I-Cache + D-Cache)
• I-Cache: 명령어 전용, 읽기 전용, 32KB
• D-Cache: 데이터 전용, 읽기/쓰기, 32KB
• 지연: 1~2 클럭 (약 0.3~1ns)
• 대역폭: 수 TB/s (최고속)
• 적중률: 90~95%
• 연관도: 4~8 way Set Associative
• 파이프라인: L1 미스 시 스톨 발생
• 최적화: 코드/데이터 크기를 L1에 맞게
```
