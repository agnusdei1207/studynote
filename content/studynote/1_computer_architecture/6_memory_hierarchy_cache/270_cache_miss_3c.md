# 270. 3C 모델 (3C Model)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시 미스의 세 가지 원인 분류
> 2. **가치**: 미스 원인별 최적화 전략 수립
> 3. **융합**: Compulsory, Capacity, Conflict

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

3C 모델(3C Model)은 **캐시 미스를 세 가지 원인으로 분류하는 모델**이다. Compulsory(필수), Capacity(용량), Conflict(충돌) 미스로 나누며, 각 원인에 따른 최적화 전략을 제시한다.

### 💡 비유: 도서관에서 책 못 찾는 이유

3C 모델은 **도서관에서 책 못 찾는 이유**와 같다. ①처음 오는 책(Compulsory), ②책장이 너무 작아서(Capacity), ③같은 코너에 몰려서(Conflict). 각각 해결책이 다르다.

### 3C 모델 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                3C 모델 개요                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【세 가지 미스 원인】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Compulsory Miss (Cold Start Miss, 필수 미스):                   │ │
│  │     • 처음 접근하는 데이터                                            │ │
│  │     • 캐시에 한 번도 없었던 블록                                       │ │
│  │     • 해결: 프리패칭                                                 │ │
│  │     • 특징: 캐시 크기와 무관                                           │ │
│  │                                                               │ │
│  │  2. Capacity Miss (용량 미스):                                       │ │
│  │     • 캐시 용량 부족으로 데이터가 밀려남                                  │ │
│  │     • 워킹셋 > 캐시 크기                                              │ │
│  │     • 해결: 캐시 크기 증가                                            │ │
│  │     • 특징: 완전 연관에서도 발생                                        │ │
│  │                                                               │ │
│  │  3. Conflict Miss (Collision Miss, 충돌 미스):                       │ │
│  │     • 같은 셋에 매핑되는 여러 블록 경쟁                                   │ │
│  │     • 연관도 부족                                                   │ │
│  │     • 해결: 연관도 증가, 데이터 재배치                                    │ │
│  │     • 특징: 직접 사상에서 가장 심각                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【3C 모델 시각화】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  전체 미스                                                        │ │
│  │  ───────────────────────────────────────────────────            │ │
│  │  │████████████│████████████████│████████████████████│           │ │
│  │  │ Compulsory │   Capacity     │      Conflict        │           │ │
│  │  │   5~10%    │    30~50%      │       40~60%         │           │ │
│  │  ───────────────────────────────────────────────────            │ │
│  │                                                               │ │
│  │  전형적 분포:                                                        │ │
│  │  • Compulsory: 5~10% (최초 접근)                                   │ │
│  │  • Capacity: 30~50% (용량 부족)                                    │ │
│  │  • Conflict: 40~60% (매핑 충돌)                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【미스 비율 vs 캐시 구조】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  캐시 구조          Compulsory   Capacity   Conflict   총 미스       │ │
│  │  ──────────          ──────────   ────────   ────────   ──────         │ │
│  │  작은 직접 사상        5%           30%        55%        90%          │ │
│  │  큰 직접 사상          5%           15%        30%        50%          │ │
│  │  4-way 집합 연관       5%           25%        20%        50%          │ │
│  │  8-way 집합 연관       5%           25%        10%        40%          │ │
│  │  완전 연관             5%           30%         0%        35%          │ │
│  │  → Conflict는 연관도로, Capacity는 크기로 감소                         │ │
│  │  → Compulsory는 구조와 무관                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 3C 모델 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                3C 모델 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Compulsory Miss 상세】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정의: 데이터를 처음 접근할 때 발생                                       │ │
│  │                                                               │ │
│  │  발생 조건:                                                          │ │
│  │  • 캐시에 한 번도 로드되지 않은 블록                                      │ │
│  │  • 프로그램 시작 시 모든 접근                                           │ │
│  │  • 새로운 데이터 영역 접근                                             │ │
│  │                                                               │ │
│  │  특징:                                                              │ │
│  │  • 캐시 크기와 무관                                                   │ │
│  │  • 연관도와 무관                                                      │ │
│  │  • 전체 미스의 5~10%                                                  │ │
│  │                                                               │ │
│  │  해결책:                                                             │ │
│  │  • 프리패칭 (Prefetching)                                           │ │
│  │  • 순차적 접근 유도                                                    │ │
│  │  • 데이터 재구성                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Capacity Miss 상세】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정의: 캐시 용량이 부족하여 발생                                         │ │
│  │                                                               │ │
│  │  발생 조건:                                                          │ │
│  │  • 워킹셋 > 캐시 크기                                                 │ │
│  │  • 자주 사용하는 데이터가 캐시에 다 들어가지 않음                             │ │
│  │  • 완전 연관 캐시에서도 발생                                             │ │
│  │                                                               │ │
│  │  특징:                                                              │ │
│  │  • 캐시 크기에 직접 비례                                               │ │
│  │  • 연관도와 무관 (완전 연관에서도 발생)                                    │ │
│  │  • 전체 미스의 30~50%                                                  │ │
│  │                                                               │ │
│  │  해결책:                                                             │ │
│  │  • 캐시 크기 증가                                                    │ │
│  │  • 워킹셋 감소 (알고리즘 변경)                                           │ │
│  │  • 데이터 압축                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Conflict Miss 상세】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정의: 같은 셋에 매핑되는 블록 간 충돌                                     │ │
│  │                                                               │ │
│  │  발생 조건:                                                          │ │
│  │  • 여러 블록이 같은 셋에 매핑                                           │ │
│  │  • 셋 내 way 수 초과                                                 │ │
│  │  • 직접 사상에서 가장 심각                                               │ │
│  │                                                               │ │
│  │  특징:                                                              │ │
│  │  • 연관도에 반비례                                                     │ │
│  │  • 완전 연관에서는 발생하지 않음                                           │ │
│  │  • 전체 미스의 40~60%                                                  │ │
│  │                                                               │ │
│  │  해결책:                                                             │ │
│  │  • 연관도 증가 (8-way, 16-way)                                        │ │
│  │  • 데이터 재배치 (패딩, 정렬)                                            │ │
│  │  • 페이지 컬러링                                                      │ │
│  │  • Victim Cache                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 3C 모델: 캐시 미스의 세 가지 원인
• Compulsory: 첫 접근, 5~10%, 프리패칭으로 감소
• Capacity: 용량 부족, 30~50%, 크기 증가로 감소
• Conflict: 매핑 충돌, 40~60%, 연관도 증가로 감소
• 구조 무관: Compulsory는 캐시 구조와 무관
• 연관도 효과: Conflict만 감소
• 크기 효과: Capacity만 감소
• 최적화: 미스 원인별 맞춤 전략
• 분석: cachegrind, 시뮬레이터
• 트레이드오프: 비용 vs 효과
```
