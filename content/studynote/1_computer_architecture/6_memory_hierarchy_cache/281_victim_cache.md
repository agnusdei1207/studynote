# 281. 희생 캐시 (Victim Cache)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시에서 교체된 블록을 임시 저장하는 작은 캐시
> 2. **가치**: Conflict Miss 감소, 교체된 데이터 2차 기회
> 3. **융합**: Direct Mapped 보완, Fully Associative, 작은 크기

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

희생 캐시(Victim Cache)는 **주 캐시에서 교체된(victim) 블록을 저장하는 작은 완전 연관 캐시**다. 1990년 Norman Jouppi가 제안했으며, 직접 사상 캐시의 Conflict Miss를 효과적으로 줄인다.

### 💡 비유: 임시 보관함

희생 캐시는 **임시 보관함**과 같다. 책상(주 캐시)이 꽉 차서 책을 치워야 할 때, 바로 버리지 않고 임시 보관함에 둔다. 나중에 다시 필요하면 책상에 다시 올린다. 보관함이 꽉 차면 그때 오래된 것부터 버린다.

## Ⅱ. 희생 캐시 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                희생 캐시 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【희생 캐시 동작 과정】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. CPU가 주소 A 요청                                               │ │
│  │  2. L1 캐시 조회:                                                   │ │
│  │     - Hit: 데이터 반환, 종료                                         │ │
│  │     - Miss: 3번으로                                                 │ │
│  │  3. Victim Cache 조회:                                              │ │
│  │     - Hit:                                                         │ │
│  │       a) Victim에서 A 가져옴                                        │ │
│  │       b) L1 교체 대상(V)을 Victim으로                                │ │
│  │       c) A를 L1에 로드                                              │ │
│  │       d) 데이터 반환 (빠름!)                                          │ │
│  │     - Miss:                                                        │ │
│  │       a) L1 교체 대상(V)을 Victim으로 (Victim 꽉 차면 오래된 것 교체)    │ │
│  │       b) L2/DRAM에서 A 가져옴                                        │ │
│  │       c) A를 L1에 로드                                              │ │
│  │       d) 데이터 반환 (느림)                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【희생 캐시 동작 예시】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  L1: 4-way, 인덱스 0~3  |  Victim: 4 엔트리                         │ │
│  │  접근: A, B, C, D, E, A, B, C, D  (모두 같은 인덱스 충돌)             │ │
│  │                                                               │ │
│  │  시간  L1[idx]     Victim       결과                              │ │
│  │  ────  ─────────   ──────────    ────                              │ │
│  │   A    [A,-,-,-]   []           L1 Miss, L2→L1                    │ │
│  │   B    [B,A,-,-]   []           L1 Miss, L2→L1                    │ │
│  │   C    [C,B,A,-]   []           L1 Miss, L2→L1                    │ │
│  │   D    [D,C,B,A]   []           L1 Miss, L2→L1                    │ │
│  │   E    [E,D,C,B]   [A]          L1 Miss, A→Victim                  │ │
│  │   A    [A,E,D,C]   [B]          L1 Miss, Victim Hit! (A 복구)       │ │
│  │   B    [B,A,E,D]   [C]          L1 Miss, Victim Hit! (B 복구)       │ │
│  │   C    [C,B,A,E]   [D]          L1 Miss, Victim Hit! (C 복구)       │ │
│  │   D    [D,C,B,A]   [E]          L1 Miss, Victim Hit! (D 복구)       │ │
│  │                                                               │ │
│  │  → Victim 없으면 모두 L2 미스                                        │ │
│  │  → Victim로 L2 접근 4회 절약 (80% 감소)                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【희생 캐시 효과】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  워크로드: 충돌이 많은 패턴 (루프, 스트라이드)                             │ │
│  │  L1: Direct Mapped, 32KB                                           │ │
│  │                                                               │ │
│  │  Victim 크기    L1 적중률    Victim 적중률    전체 적중률    개선        │ │
│  │  ──────────     ──────        ──────────      ──────        ────        │ │
│  │  없음           80%           -              80%           기준          │ │
│  │  4 엔트리       80%           12%            92%           +15%         │ │
│  │  8 엔트리       80%           15%            95%           +19%         │ │
│  │  16 엔트리      80%           17%            97%           +21%         │ │
│  │                                                               │ │
│  │  → 작은 Victim Cache로 큰 적중률 향상                                 │ │
│  │  → Conflict Miss가 많을수록 효과 큼                                   │ │
│  │  → 비용 효율적 (면적 대비 효과)                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 희생 캐시 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                희생 캐시 × 타 과목 융합                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • L1 보조: Direct Mapped + Victim으로 Conflict 감소                  │
│  • 비용 효율: 작은 면적으로 큰 효과                                      │
│  • 파이프라인: Victim Hit으로 스톨 감소                                  │
│  • 전력: 작은 크기로 오버헤드 적음                                        │
│                                                                     │
│  【캐시 계층】                                                        │
│  • L1 → Victim → L2 → L3 → DRAM                                     │
│  • 계층 간 미스 체인 완화                                               │
│  • AMAT 감소                                                         │
│  • L2 접근 감소                                                       │
│                                                                     │
│  【워크로드 특성】                                                       │
│  • 충돌 많음: Victim 효과 매우 큼 (루프, 스트라이드)                        │ │
│  • 순차적: 효과 적음                                                   │ │
│  • 랜덤: 효과 중간                                                    │ │
│  • Working Set 작음: 효과 큼                                          │ │
│                                                                     │
│  【현대 CPU 동향】                                                       │
│  • 집합 연관 캐시 발전으로 Victim 사용 감소                               │ │
│  • 대신 8-way, 16-way 집합 연관 사용                                     │ │
│  • 임베디드/저전력 CPU에서는 여전히 유용                                   │ │
│  • ARM Cortex-A53 등에서 사용                                          │ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Victim Cache vs 집합 연관:                                                    │ │
│  │  ──────────────────────                                              │ │
│  │  특성              Direct + Victim    8-way Set Associative          │ │
│  │  ──────             ────────────       ────────────                    │ │
│  │  L1 면적            작음                큼                              │ │
│  │  추가 면적          Victim (작음)        없음                            │ │
│  │  전체 면적          비슷                 비슷                            │ │
│  │  복잡도             중간                 낮음                            │ │
│  │  적중률             비슷                 비슷                            │ │
│  │  접근 지연          1~2클럭              1클럭                           │ │
│  │                                                               │ │
│  │  → 현대는 집합 연관이 더 선호됨                                        │ │
│  │  → 하지만 특수 상황에서 Victim 유용                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  │                                                               │ │
│  │  4. 데이터 교환:                                                      │ │
│  │     • L1-Victim 간 Swap 동작                                         │ │
│  │     • Victim Hit 시 데이터 교환                                       │ │
│  │     • 1~2 클럭 내 완료                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【희생 캐시 사용 CPU 예】                                                        │
│  ──────────────────                                                  │

│  │  CPU                  Victim Cache    특징                           │ │
│  │  ─────────────        ──────────      ────                             │ │
│  │  ARM Cortex-A53       있음            저전력, 임베디드                  │ │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 희생 캐시: L1에서 교체된 블록을 저장하는 작은 캐시
• 크기: 4~16 엔트리 (매우 작음, 비용 효율적)
• 구조: 완전 연관 (Fully Associative)
• 목적: Conflict Miss 감소, 교체 블록 2차 기회
• 동작: L1 Miss → Victim Hit → Swap (빠름)
• 효과: 작은 면적으로 큰 적중률 향상 (15~20%)
• 발명: 1990년 Norman Jouppi
• 사용처: ARM Cortex-A53, MIPS R10000, 임베디드
• 현대: 집합 연관 발전으로 사용 감소
• 대안: 8-way/16-way 집합 연관, 큰 L1
```
