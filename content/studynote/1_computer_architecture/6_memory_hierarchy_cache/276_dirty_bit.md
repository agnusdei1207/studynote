# 276. 더티 비트 (Dirty Bit)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시 라인 수정 여부를 나타내는 플래그
> 2. **가치**: Write-Back 정책의 핵심, 불필요한 쓰기 방지
> 3. **융합**: 캐시, 페이지, 버퍼 풀에서 사용

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

더티 비트(Dirty Bit)는 **캐시 라인이나 페이지가 수정되었는지 나타내는 1비트 플래그**이다. Write-Back 정책에서 필수적이며, 수정된 데이터만 메모리에 다시 쓰도록 한다.

### 💡 비유: 수정 표시

더티 비트는 **문서 수정 표시**와 같다. 문서를 수정하면 표시를 해둔다. 저장할 때 표시된 문서만 저장한다. 수정 안 한 문서는 다시 저장할 필요 없다.

### 더티 비트 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                더티 비트 개요                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【더티 비트 역할】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정의: 캐시 라인이 메모리와 다른지 (수정되었는지) 나타내는 비트           │ │
│  │                                                               │ │
│  │  위치: 캐시 라인 메타데이터                                           │ │
│  │                                                               │ │
│  │  상태:                                                              │ │
│  │  • 0 (Clean): 메모리와 동일, 수정 없음                                │ │
│  │  • 1 (Dirty): 메모리와 다름, 수정 있음                                │ │
│  │                                                               │ │
│  │  동작:                                                              │ │
│  │  • 읽기: 더티 비트 변경 없음                                           │ │
│  │  • 쓰기: 더티 비트 = 1 설정                                           │ │
│  │  • 교체: 더티=1이면 메모리에 쓰기 후 교체                                │ │
│  │  • 교체: 더티=0이면 바로 교체 (쓰기 불필요)                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 라인 구조】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌────────┬────────┬────────┬──────────────────────┐           │ │
│  │  │ Valid  │ Dirty  │  Tag   │        Data          │           │ │
│  │  │ 1비트   │ 1비트   │ t비트  │      B바이트          │           │ │
│  │  └────────┴────────┴────────┴──────────────────────┘           │ │
│  │                                                               │ │
│  │  Valid: 데이터 유효 여부                                             │ │
│  │  Dirty: 데이터 수정 여부                                             │ │
│  │  Tag: 메모리 블록 식별자                                              │ │
│  │  Data: 실제 캐시 데이터                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【더티 비트 동작 예시】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  초기: 캐시 비어 있음, 모든 Dirty = 0                                  │ │
│  │                                                               │ │
│  │  1. A 읽기: A 로드, Dirty = 0                                       │ │
│  │  2. A 쓰기: A 수정, Dirty = 1                                       │ │
│  │  3. A 읽기: 캐시에서 읽기, Dirty = 1 유지                              │ │
│  │  4. A 교체: Dirty=1 → 메모리에 A 쓰기 후 B 로드                        │ │
│  │  5. B 교체: Dirty=0 → 바로 C 로드 (메모리 쓰기 불필요)                   │ │
│  │                                                               │ │
│  │  효과: 수정된 데이터만 메모리에 쓰기                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 더티 비트 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                더티 비트 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【더티 비트 vs Write-Through】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Write-Through:                                                    │ │
│  │  • 캐시와 메모리 동시 쓰기                                            │ │
│  │  • 더티 비트 불필요 (항상 동일)                                        │ │
│  │  • 단순하지만 느림                                                  │ │
│  │                                                               │ │
│  │  Write-Back:                                                        │ │
│  │  • 캐시에만 쓰기                                                    │ │
│  │  • 더티 비트로 수정 추적                                              │ │
│  │  • 교체 시에만 메모리 쓰기                                             │ │
│  │  • 빠르지만 복잡                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【더티 비트 상태 전이】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │      ┌─────────┐                                               │ │
│  │      │ Invalid │ (Valid = 0)                                   │ │
│  │      └────┬────┘                                               │ │
│  │           │ 로드                                                │ │
│  │           ▼                                                    │ │
│  │      ┌─────────┐                                               │ │
│  │      │  Clean  │ (Valid = 1, Dirty = 0)                        │ │
│  │      │ (수정X) │                                               │ │
│  │      └────┬────┘                                               │ │
│  │           │ 쓰기                                                │ │
│  │           ▼                                                    │ │
│  │      ┌─────────┐                                               │ │
│  │      │  Dirty  │ (Valid = 1, Dirty = 1)                        │ │
│  │      │ (수정O) │                                               │ │
│  │      └────┬────┘                                               │ │
│  │           │ 플러시/교체                                          │ │
│  │           ▼                                                    │ │
│  │      ┌─────────┐                                               │ │
│  │      │  Clean  │ (메모리와 동기화됨)                               │ │
│  │      └─────────┘                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【더티 비트 이점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 쓰기 감소:                                                       │ │
│  │     • 수정된 블록만 메모리에 쓰기                                        │ │
│  │     • 읽기 전용 블록은 쓰기 불필요                                       │ │
│  │     • 메모리 대역폭 절약                                              │ │
│  │                                                               │ │
│  │  2. 쓰기 병합:                                                        │ │
│  │     • 같은 블록에 여러 쓰기 → 1회 메모리 쓰기                              │ │
│  │     • 쓰기 버스트 처리                                                │ │
│  │                                                               │ │
│  │  3. 성능 향상:                                                        │ │
│  │     • 교체 시 메모리 쓰기 선택적                                        │ │
│  │     • 전체 성능 향상                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【더티 비트 관리 비용】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 저장: 캐시 라인당 1비트 추가                                         │ │
│  │  • 예: 64KB 캐시, 64B 라인 → 1024라인 → 1024비트 = 128B               │ │
│  │  • 오버헤드: 무시할 수준 (캐시의 0.2%)                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │
│  【컴퓨터 구조】                                                      │

│  • 스와핑: Dirty Page만 스왑 아웃                                       │

│                                                                     │
│  【파일 시스템】                                                        │
│  • 페이지 캐시: Dirty Page                                            │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  계층별 더티 비트 사용:                                                         │ │
│  │  ──────────────────────                                              │ │

│  │  ──────          ──────       ────                  ────                │ │
│  │  L1 D-Cache     있음         캐시 교체              즉시               │ │

### 더티 비트 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                더티 비트 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【더티 비트 확인】                                                              │
│  ──────────────────                                                  │
│  # 페이지 캐시 더티 페이지 확인                                          │
│  $ cat /proc/meminfo | grep Dirty                                   │
│  Dirty:             1024 kB                                         │
│                                                                     │
│  # 더티 페이지 플러시                                                  │
│  $ sync                                                             │
│                                                                     │
│  【더티 페이지 관리】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Linux 커널 파라미터:                                                 │ │
│  │                                                               │ │
│  │  vm.dirty_ratio = 20                                              │ │
│  │  → 메모리 20%까지 Dirty Page 허용                                    │ │
│  │                                                               │ │
│  │  vm.dirty_background_ratio = 10                                   │ │
│  │  → 메모리 10%에서 백그라운드 플러시 시작                                 │ │
│  │                                                               │ │
│  │  vm.dirty_writeback_centisecs = 500                               │ │
│  │  → 5초마다 백그라운드 플러시                                           │ │
│  │                                                               │ │
│  │  # 설정 확인                                                       │ │
│  │  $ sysctl vm.dirty_ratio                                          │ │
│  │  $ sysctl vm.dirty_background_ratio                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DB 버퍼 풀 더티 페이지】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  MySQL InnoDB:                                                       │ │
│  │  • innodb_max_dirty_pages_pct = 90                                │ │
│  │  → 90%까지 Dirty Page 허용                                          │ │
│  │                                                               │ │
│  │  # 상태 확인                                                       │ │
│  │  SHOW ENGINE INNODB STATUS\G                                     │ │
│  │  SHOW STATUS LIKE 'Innodb_buffer_pool_pages_dirty';              │ │
│  │                                                               │ │
│  │  # 강제 플러시                                                       │ │
│  │  SET GLOBAL innodb_max_dirty_pages_pct = 0;                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【더티 비트 최적화 팁】                                                         │
│  ──────────────────                                                  │
│  1. 주기적 플러시: 데이터 손실 방지                                        │
│  2. fsync: 중요 데이터 즉시 동기화                                        │
│  3. 임계값 설정: 메모리 부족 방지                                          │
│  4. 백그라운드 플러시: 성능 영향 최소화                                      │
│  5. 체크포인트: DB 복구 시간 단축                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 더티 비트: 캐시 라인/페이지 수정 여부 플래그
• 위치: 캐시 라인 메타데이터 (1비트)
• 상태: Clean (0), Dirty (1)
• 용도: Write-Back 정책에서 필수
• 이점: 불필요한 쓰기 방지, 대역폭 절약
• 동작: 쓰기 시 1 설정, 교체 시 1이면 쓰기
• I-Cache: Dirty Bit 불필요 (읽기 전용)
• OS: 페이지 테이블 Dirty Bit
• DB: 버퍼 풀 Dirty Page
• 관리: 주기적 플러시, fsync, 체크포인트
```
