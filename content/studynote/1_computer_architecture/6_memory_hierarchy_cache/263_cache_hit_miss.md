# 263. 캐시 히트 및 미스 (Cache Hit & Miss)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 요청 데이터가 캐시에 있으면 히트, 없으면 미스
> 2. **가치**: 적중률이 캐시 성능의 핵심 지표
> 3. **융합**: 3C 모델, 미스 페널티, 성능 최적화

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

캐시 히트(Cache Hit)는 **CPU가 요청한 데이터가 캐시에 있어서 빠르게 접근하는 경우**이고, 캐시 미스(Cache Miss)는 **데이터가 캐시에 없어서 하위 메모리에서 가져와야 하는 경우**이다. 적중률(Hit Rate)은 캐시 성능의 핵심 지표이다.

### 💡 비유: 도서관 이용

캐시 히트/미스는 **도서관 이용**과 같다. 내 책상(캐시)에 책이 있으면 바로 읽을 수 있다(히트). 없으면 책장(메모리)까지 가서 찾아야 한다(미스). 책상에 많은 책을 둘수록 히트율이 높아진다.

### 히트와 미스

```
┌─────────────────────────────────────────────────────────────────────┐
│                캐시 히트 및 미스 개요                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【히트(Hit) vs 미스(Miss)】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │   CPU 요청: 주소 A의 데이터                                       │ │
│  │          │                                                      │ │
│  │          ▼                                                      │ │
│  │   ┌──────────────────┐                                          │ │
│  │   │   캐시 조회        │                                          │ │
│  │   └────────┬─────────┘                                          │ │
│  │            │                                                      │ │
│  │     ┌──────┴──────┐                                              │ │
│  │     │              │                                              │ │
│  │  ┌──┴──┐        ┌──┴──┐                                          │ │
│  │  │ 히트  │        │ 미스  │                                          │ │
│  │  └──┬──┘        └──┬──┘                                          │ │
│  │     │              │                                              │ │
│  │     ▼              ▼                                              │ │
│  │  캐시에서      하위 메모리에서                                       │ │
│  │  데이터 반환    데이터 인출                                          │ │
│  │  (~1ns)        (~100ns)                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 지표】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  적중률 (Hit Rate):                                                │ │
│  │  Hit Rate = Hit / (Hit + Miss) × 100%                             │ │
│  │  예: 950만 번 히트 / 1000만 번 접근 = 95%                            │ │
│  │                                                               │ │
│  │  미스율 (Miss Rate):                                               │ │
│  │  Miss Rate = 1 - Hit Rate                                         │ │
│  │  예: 100% - 95% = 5%                                              │ │
│  │                                                               │ │
│  │  히트 시간 (Hit Time):                                              │ │
│  │  • L1: ~1ns                                                        │ │
│  │  • L2: ~4ns                                                        │ │
│  │  • L3: ~15ns                                                       │ │
│  │                                                               │ │
│  │  미스 페널티 (Miss Penalty):                                         │ │
│  │  • L1 미스 → L2: ~3ns                                              │ │
│  │  • L2 미스 → L3: ~10ns                                             │ │
│  │  • L3 미스 → DRAM: ~100ns                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【AMAT (평균 메모리 접근 시간)】                                                   │
│  ──────────────────                                                  │
│  AMAT = Hit Time + Miss Rate × Miss Penalty                        │
│  예: 1ns + 0.05 × 100ns = 6ns                                       │
│  → 95% 적중률로 평균 접근 시간 6ns 달성                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 히트/미스 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                히트/미스 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【3C 모델: 캐시 미스의 원인】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Compulsory Miss (Cold Start Miss, 필수 미스):                   │ │
│  │     • 처음 접근하는 데이터                                            │ │
│  │     • 캐시에 한 번도 없었던 데이터                                     │ │
│  │     • 해결: 프리패칭                                               │ │
│  │     • 비율: 전체 미스의 5~10%                                         │ │
│  │                                                               │ │
│  │  2. Capacity Miss (용량 미스):                                       │ │
│  │     • 캐시 용량 부족으로 데이터가 밀려남                                  │ │
│  │     • 워킹셋 > 캐시 크기                                              │ │
│  │     • 해결: 캐시 크기 증가                                            │ │
│  │     • 비율: 전체 미스의 30~50%                                        │ │
│  │                                                               │ │
│  │  3. Conflict Miss (Collision Miss, 충돌 미스):                       │ │
│  │     • 같은 셋에 매핑되는 여러 주소                                       │ │
│  │     • 연관도 부족                                                   │ │
│  │     • 해결: 연관도 증가, 웨이 증가                                      │ │
│  │     • 비율: 전체 미스의 40~60%                                        │ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐     │ │
│  │  │  미스 원인      비율        해결책                          │     │ │
│  │  │  ──────────     ──────       ──────────                      │     │ │
│  │  │  Compulsory     5~10%       프리패칭                        │     │ │
│  │  │  Capacity       30~50%      캐시 크기 증가                    │     │ │
│  │  │  Conflict       40~60%      연관도 증가                       │     │ │
│  │  └─────────────────────────────────────────────────────────┘     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【계층별 적중률】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  계층    전체 접근 중    조건부 적중률    지연                       │ │
│  │  ────    ──────────      ────────────     ────                      │ │
│  │  L1      95%             95%              1ns                      │ │
│  │  L2      4%              80% (L1 미스)     4ns                      │ │
│  │  L3      0.8%            80% (L2 미스)     15ns                     │ │
│  │  DRAM    0.2%            -                100ns                     │ │
│  │  → 대부분 L1에서 처리                                               │ │
│  │  → L1 미스의 80%가 L2에서 적중                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  4. 프리패칭: Compulsory 미스 감소                                        │
│  5. 연관도: Conflict 미스 감소                                           │
│  6. 캐시 크기: Capacity 미스 감소                                          │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 캐시 히트: 요청 데이터가 캐시에 있음
• 캐시 미스: 요청 데이터가 캐시에 없음
• 적중률: Hit / (Hit + Miss) × 100%
• 미스율: 1 - Hit Rate
• AMAT: Hit Time + Miss Rate × Miss Penalty
• 3C 모델: Compulsory, Capacity, Conflict
• Compulsory: 첫 접근, 프리패칭으로 감소
• Capacity: 용량 부족, 캐시 크기 증가로 감소
• Conflict: 매핑 충돌, 연관도 증가로 감소
• 성능: 적중률 5% 감소 = 성능 6배 저하
```
