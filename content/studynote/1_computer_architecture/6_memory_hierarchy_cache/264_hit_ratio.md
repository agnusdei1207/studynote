# 264. 적중률 (Hit Ratio)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시 접근 중 히트 비율
> 2. **가치**: 캐시 효율성의 핵심 지표
> 3. **융합**: 성능, CPI, AMAT과 직접 연관

## Ⅰ. 개요 (Context & Background)

### 개념 정의

적중률(Hit Ratio)은 **전체 메모리 접근 중 캐시에서 히트한 비율**이다. 캐시 성능을 나타내는 가장 중요한 지표로, 높을수록 캐시가 효과적으로 동작함을 의미한다.

### 💡 비유: 야구 타율

적중률은 **야구 타율**과 같다. 타자가 공을 칠 때마다 안타를 치는 비율이다. 타율이 높을수록 좋은 타자다. 마찬가지로 적중률이 높을수록 좋은 캐시다.

### 적중률 계산

```
┌─────────────────────────────────────────────────────────────────────┐
│                적중률 개요                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【적중률 공식】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                  Hit                                           │ │
│  │  Hit Ratio = ───────────────── × 100%                          │ │
│  │              Hit + Miss                                        │ │
│  │                                                               │ │
│  │  예: 950만 번 히트, 50만 번 미스                                    │ │
│  │      Hit Ratio = 950 / 1000 × 100% = 95%                        │ │
│  │                                                               │ │
│  │  Miss Ratio = 1 - Hit Ratio                                    │ │
│  │             = 1 - 0.95 = 5% (또는 0.05)                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【계층별 적중률】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌──────────────────────────────────────────────────────┐      │ │
│  │  │     전체 메모리 접근                                    │      │ │
│  │  │                    100%                               │      │ │
│  │  └───────────────────────┬───────────────────────────────┘      │ │
│  │                          │                                       │ │
│  │           ┌──────────────┴──────────────┐                       │ │
│  │           │                              │                       │ │
│  │        L1 히트                       L1 미스                      │ │
│  │         95%                           5%                         │ │
│  │           │                              │                       │ │
│  │           │               ┌──────────────┴──────────────┐        │ │
│  │           │               │                              │        │ │
│  │           │            L2 히트                       L2 미스     │ │
│  │           │             80%                           20%        │ │
│  │           │               │                              │        │ │
│  │           │               │               ┌──────────────┴───┐    │ │
│  │           │               │               │                  │    │ │
│  │           │               │            L3 히트             L3 미스│ │
│  │           │               │             80%                 20%  │ │
│  │           │               │               │                  │    │ │
│  │           │               │               │              DRAM    │ │
│  │           │               │               │               100%  │ │
│  │           │               │               │                  │    │ │
│  │  전체 L1 적중률:  95%                                            │ │
│  │  전체 L2 적중률:  5% × 80% = 4%                                   │ │
│  │  전체 L3 적중률:  1% × 80% = 0.8%                                 │ │
│  │  DRAM 접근:      0.2% × 100% = 0.2%                               │ │
│  │  ─────────────────────────────────────────────────────────────   │ │
│  │  합계              100%                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【계층별 일반적 적중률】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  계층    적중률 (전체)    조건부 적중률       지연                    │ │
│  │  ────    ──────────        ────────────        ────                     │ │
│  │  L1      90~95%            90~95%             1ns                      │ │
│  │  L2      4~8%              80~90%             4ns                      │ │
│  │  L3      0.8~1.6%          80~90%             15ns                     │ │
│  │  DRAM    0.2~0.4%          -                  100ns                     │ │
│  │  → L1에서 대부분 처리                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 적중률 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                적중률 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【적중률에 영향을 미치는 요소】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 캐시 크기 (Cache Size):                                         │ │
│  │     • 클수록 높은 적중률                                             │ │
│  │     • 2배 크기 → ~10% 적중률 향상                                    │ │
│  │     • 한계: 수확 체감 법칙                                            │ │
│  │                                                               │ │
│  │  2. 연관도 (Associativity):                                          │ │
│  │     • 높을수록 Conflict Miss 감소                                    │ │
│  │     • 2-way → 4-way → 8-way → 16-way                               │ │
│  │     • 한계: 16-way 이상은 효과 미미                                   │ │
│  │                                                               │ │
│  │  3. 블록 크기 (Block Size):                                          │ │
│  │     • 클수록 공간적 지역성 향상                                       │ │
│  │     • 너무 크면 미스 페널티 증가                                       │ │
│  │     • 최적: 32~128B                                                 │ │
│  │                                                               │ │
│  │  4. 워킹셋 (Working Set):                                            │ │
│  │     • 작을수록 높은 적중률                                             │ │
│  │     • 워킹셋 < 캐시 크기가 이상                                        │ │
│  │                                                               │ │
│  │  5. 접근 패턴 (Access Pattern):                                      │ │
│  │     • 순차적: 높은 적중률                                             │ │
│  │     • 랜덤: 낮은 적중률                                               │ │
│  │     • 스트라이드: 패턴에 따라 다름                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적중률 vs 성능】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  적중률    미스율    AMAT         성능 (vs 100%)                    │ │
│  │  ──────    ──────    ──────────     ──────────────                   │ │
│  │  99%       1%       2ns          50%                               │ │
│  │  98%       2%       3ns          33%                               │ │
│  │  95%       5%       6ns          17%                               │ │
│  │  90%       10%      11ns         9%                                │ │
│  │  80%       20%      21ns         5%                                │ │
│  │  → 1% 미스율 증가 = 성능 큰 저하                                      │ │
│  │  → 95% 이상 목표                                                    │ │
│  │                                                               │ │
│  │  AMAT 계산:                                                        │ │
│  │  AMAT = 1ns + MissRate × 100ns                                    │ │
│  │  95% 적중률: 1 + 0.05 × 100 = 6ns                                   │ │
│  │  90% 적중률: 1 + 0.10 × 100 = 11ns                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적중률 벤치마크】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  워크로드           L1 적중률    L2 적중률    L3 적중률               │ │
│  │  ──────────           ──────        ──────        ──────               │ │
│  │  SPECint 2017        95~98%        80~90%        80~90%               │ │
│  │  SPECfp 2017         92~96%        75~85%        75~85%               │ │
│  │  Database OLTP       90~95%        70~80%        70~80%               │ │
│  │  HPC                 85~92%        60~75%        60~75%               │ │
│  │  Gaming              95~98%        85~90%        85~90%               │ │
│  │  → 워크로드에 따라 적중률 다름                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 적중률 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                적중률 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 적중률 측정】                                                         │
│  ──────────────────                                                  │
│  $ perf stat -e cache-references,cache-misses ./prog               │
│  cache-references: 1,000,000,000                                    │
│  cache-misses: 50,000,000                                           │
│  → 적중률 = (1,000M - 50M) / 1,000M = 95%                            │
│                                                                     │
│  【계층별 적중률 측정】                                                         │
│  ──────────────────                                                  │
│  $ perf stat -e L1-dcache-loads,L1-dcache-load-misses \             │
│               -e LLC-loads,LLC-load-misses ./prog                   │
│  → L1, L3 적중률 확인                                                 │
│                                                                     │
│  【적중률 vs 캐시 크기】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  캐시 크기    적중률    AMAT                                       │ │
│  │  ──────────     ──────    ──────                                    │ │
│  │  16KB          85%        16ns                                    │ │
│  │  32KB          90%        11ns                                    │ │
│  │  64KB          93%        8ns                                     │ │
│  │  128KB         95%        6ns                                     │ │
│  │  256KB         96%        5ns                                     │ │
│  │  512KB         97%        4ns                                     │ │
│  │  → 2배 크기 → ~1% 적중률 향상                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적중률 목표 설정】                                                         │
│  ──────────────────                                                  │
│  • L1: 95%+ (목표)                                                   │
│  • L2: 80%+ (L1 미스 중)                                              │
│  • L3: 80%+ (L2 미스 중)                                              │
│  • 전체: 99%+ (DRAM 미스 1% 미만)                                       │
│                                                                     │
│  【적중률 향상 팁】                                                             │
│  ──────────────────                                                  │
│  1. 워킹셋 최적화: 캐시 크기에 맞게                                       │
│  2. 데이터 배치: 지역성 극대화                                             │
│  3. 알고리즘 변경: 캐시 친화적                                            │
│  4. 프리패칭: 미리 로드                                                  │
│  5. 데이터 구조: SoA, 타일링                                             │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 적중률: 캐시 히트 비율 = Hit / (Hit + Miss) × 100%
• 미스율: 1 - Hit Ratio
• 계층별: L1(95%) → L2(80% of miss) → L3(80% of miss)
• 영향 요소: 캐시 크기, 연관도, 블록 크기, 워킹셋, 접근 패턴
• 성능 관계: 1% 미스율 = 큰 성능 저하
• AMAT: Hit Time + Miss Rate × Miss Penalty
• 목표: L1 95%+, 전체 99%+
• 측정: perf, cachegrind
• 향상: 지역성, 프리패칭, 데이터 재구성
```
