# 245. 메모리 계층 구조 (Memory Hierarchy)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 속도와 용량의 트레이드오프를 계층으로 해결
> 2. **가치**: 빠른 메모리의 속도 + 느린 메모리의 용량
> 3. **융합**: 캐시, 가상 메모리, 스토리지의 기초

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

메모리 계층 구조(Memory Hierarchy)는 **속도, 용량, 비용이 다른 메모리들을 계층적으로 배치하여 전체 시스템 성능을 최적화하는 구조**이다. 빠르고 비싼 메모리는 소량, 느리고 저렴한 메모리는 대량으로 배치한다.

### 💡 비유: 책장과 도서관

메모리 계층은 **개인 책상 vs 도서관**과 같다. 자주 보는 책은 책상 위에(빠르고 비싼 공간), 가끔 보는 책은 집 책장에, 거의 안 보는 책은 도서관에(느리고 저렴한 공간) 둔다. 책상은 작지만 접근이 빠르고, 도서관은 크지만 가기 멀다.

### 메모리 계층 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 계층 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【피라미드 구조】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                          ↑ 속도                               │ │
│  │                          │ ↓ 용량                              │ │
│  │                          │ ↓ 비용/비트                          │ │
│  │                          │                                    │ │
│  │                     ┌────┴────┐                               │ │
│  │                     │ 레지스터 │  ~1ns    <1KB    $$$$         │ │
│  │                     └────┬────┘                               │ │
│  │                    ┌─────┴─────┐                              │ │
│  │                    │  L1 캐시   │  ~1ns    ~64KB   $$$         │ │
│  │                    └─────┬─────┘                              │ │
│  │                 ┌────────┴────────┐                           │ │
│  │                 │    L2 캐시       │  ~4ns    ~256KB  $$        │ │
│  │                 └────────┬────────┘                           │ │
│  │            ┌─────────────┴─────────────┐                      │ │
│  │            │        L3 캐시             │  ~12ns   ~32MB  $      │ │
│  │            └─────────────┬─────────────┘                      │ │
│  │       ┌──────────────────┴──────────────────┐                 │ │
│  │       │            DRAM (주 메모리)           │  ~100ns  ~128GB      │ │
│  │       └──────────────────┬──────────────────┘                 │ │
│  │  ┌───────────────────────┴───────────────────────┐            │ │
│  │  │              SSD (플래시 메모리)                 │  ~10μs  ~4TB        │ │
│  │  └───────────────────────┬───────────────────────┘            │ │
│  │  ┌───────────────────────┴───────────────────────┐            │ │
│  │  │              HDD (하드 디스크)                  │  ~10ms  ~20TB        │ │
│  │  └───────────────────────────────────────────────┘            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【계층 간 관계】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  계층    접근 시간      용량        비용/비트   적중률           │ │
│  │  ────    ──────────     ──────       ────────   ──────          │ │
│  │  레지스터  0.3~1ns       <1KB        최고       100%           │ │
│  │  L1       1~2ns         32~64KB      높음       90~95%         │ │
│  │  L2       3~5ns         256KB~1MB    중간       95~98%         │ │
│  │  L3       10~20ns       8~64MB       낮음       98~99%         │ │
│  │  DRAM     50~100ns      8~256GB      더 낮음    99%+           │ │
│  │  SSD      10~100μs      256GB~8TB    낮음       -              │ │
│  │  HDD      5~15ms        1~20TB       최저       -              │ │
│  │  → 상위 계층: 빠름, 작음, 비쌈                                        │ │
│  │  → 하위 계층: 느림, 큼, 쌈                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 메모리 계층 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 계층 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【계층 간 데이터 이동】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │   CPU ←→ 레지스터 ←→ L1 ←→ L2 ←→ L3 ←→ DRAM ←→ SSD ←→ HDD    │ │
│  │           워드     라인    라인    라인    페이지   블록    블록    │ │
│  │                                                               │ │
│  │   전송 단위:                                                     │ │
│  │   • 레지스터 ↔ 캐시: 캐시 라인 (64B)                               │ │
│  │   • 캐시 ↔ DRAM: 캐시 라인 (64B)                                  │ │
│  │   • DRAM ↔ SSD: 페이지 (4KB)                                     │ │
│  │   • SSD ↔ HDD: 블록 (4KB~64KB)                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【평균 접근 시간 (AMAT)】                                                 │
│  ──────────────────                                                  │
│  AMAT = Hit Time + Miss Rate × Miss Penalty                        │
│                                                                     │
│  예시:                                                               │
│  • L1 Hit Time: 1ns, Miss Rate: 5%                                 │
│  • L2 Hit Time: 5ns, Miss Rate: 10%                                │
│  • L3 Hit Time: 15ns, Miss Rate: 5%                                │
│  • DRAM Access: 100ns                                               │
│                                                                     │
│  AMAT = 1 + 0.05 × (5 + 0.10 × (15 + 0.05 × 100))                   │
│  AMAT = 1 + 0.05 × (5 + 0.10 × 20)                                  │
│  AMAT = 1 + 0.05 × 7 = 1.35ns                                       │
│  → DRAM만 쓰면 100ns, 계층 구조로 1.35ns                               │
│                                                                     │
│  【지역성 (Locality)】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 시간적 지역성: 최근 접근한 데이터 재접근 확률 높음                  │ │
│  │     예: 루프 변수, 카운터                                          │ │
│  │                                                               │ │
│  │  2. 공간적 지역성: 접근한 데이터 근처 재접근 확률 높음                   │ │
│  │     예: 배열 순차 접근                                            │ │
│  │                                                               │ │
│  │  3. 순차적 지역성: 순차적 접근 패턴                                   │ │
│  │     예: 명령어 인출, 스트림 데이터                                   │ │
│  │                                                               │ │
│  │  → 지역성이 계층 구조의 효율성을 보장                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```

│                                                                     │

│  • 가상 메모리: DRAM-디스크 계층                                       │

│  • 전력: 계층별 전력 관리                                              │
│                                                                     │
│  【운영체제】                                                          │
│  • 메모리 관리: 페이지 교체, 스와핑                                      │
│  • 버퍼 캐시: 파일 시스템 캐시                                          │
│  • 스케줄링: 메모리 계층 고려                                           │
│  • 성능: 메모리 대역폭 관리                                             │
│                                                                     │
│  【데이터베이스】                                                        │
│  • 버퍼 풀: DB 페이지 캐시                                             │
│  • 인덱스: 메모리 계층 활용                                             │
│  • 쿼리 최적화: 지역성 고려                                             │
│  • 스토리지: SSD/HDD 계층                                              │
│                                                                     │
│  【네트워크】                                                          │
│  • CDN: 웹 캐시 계층                                                  │
│  • 프록시: 캐시 서버                                                   │
│  • DNS: DNS 캐시                                                     │
│  • 지연: 메모리 vs 네트워크                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  메모리 계층 설계 트레이드오프:                                            │ │
│  │  ──────────────────────                                              │ │
│  │  설계 선택     장점                단점                              │ │
│  │  ──────────    ──────────────      ────────────────                │ │
│  │  더 많은 계층   높은 적중률           복잡도 증가                      │ │
│  │  더 큰 캐시    낮은 미스율           지연 증가, 비용 증가               │ │
│  │  더 빠른 캐시   낮은 히트 타임         비용 증가                        │ │
│  │  → 용도에 맞는 최적 설계 필요                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 메모리 계층: 속도와 용량의 트레이드오프를 계층으로 해결
• 구조: 레지스터 → L1 → L2 → L3 → DRAM → SSD → HDD
• 원리: 상위는 빠르고 작고 비쌈, 하위는 느리고 크고 쌈
• 핵심: 지역성 (시간, 공간, 순차)
• AMAT: 계층 구조로 평균 접근 시간 최소화
• 전송 단위: 라인, 페이지, 블록
• 최적화: 지역성 활용, 캐시 친화적 코드
• 설계: 용도에 맞는 계층 구조 선택
```
