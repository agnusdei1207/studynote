# 254. 메모리 인터리빙 (Memory Interleaving)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 메모리 뱅크/채널에 데이터를 분산 배치
> 2. **가치**: 병렬 접근으로 대역폭 증대, 지연 숨김
> 3. **융합**: 캐시 인터리빙, 디스크 스트라이핑과 유사

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

메모리 인터리빙(Memory Interleaving)은 **메모리를 여러 뱅크나 채널로 나누어 병렬로 접근하는 기술**이다. 연속된 주소가 서로 다른 뱅크에 배치되어 동시에 여러 접근이 가능하다.

### 💡 비유: 계산대 여러 개

메모리 인터리빙은 **마트 계산대 여러 개**와 같다. 계산대가 1개면 한 명씩만 처리할 수 있다. 계산대가 4개면 4명이 동시에 계산할 수 있다. 물건을 4개의 계산대에 나누어 배치하면 대기 시간이 줄어든다.

### 인터리빙 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 인터리빙 개요                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터리빙 없음】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  주소:     0    1    2    3    4    5    6    7                 │ │
│  │           ──── ──── ──── ──── ──── ──── ──── ────                │ │
│  │  뱅크:     B0   B0   B0   B0   B0   B0   B0   B0                 │ │
│  │           └────────────────────────────────────┘                │ │
│  │                          순차 접근                               │ │
│  │                                                               │ │
│  │  접근: 0 → 1 → 2 → 3 → 4 → 5 → 6 → 7                             │ │
│  │  → 한 번에 1개만 접근 가능                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터리빙 (4-way)】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  주소:     0    1    2    3    4    5    6    7                 │ │
│  │           ──── ──── ──── ──── ──── ──── ──── ────                │ │
│  │  뱅크:     B0   B1   B2   B3   B0   B1   B2   B3                 │ │
│  │           └──┬──┴──┬──┴──┬──┴──┬──┘                             │ │
│  │              ↓      ↓      ↓      ↓                             │ │
│  │           병렬 접근 가능                                          │ │
│  │                                                               │ │
│  │  접근: 0,1,2,3 동시 → 4,5,6,7 동시                                 │ │
│  │  → 4개 동시 접근 가능                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터리빙 효과】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시간  인터리빙 없음          4-way 인터리빙                        │ │
│  │  ────  ────────────────      ──────────────────                  │ │
│  │   t0   뱅크0: 접근 0          뱅크0: 접근 0                        │ │
│  │   t1   뱅크0: 접근 1          뱅크1: 접근 1  } 동시                │ │
│  │   t2   뱅크0: 접근 2          뱅크2: 접근 2  } 동시                │ │
│  │   t3   뱅크0: 접근 3          뱅크3: 접근 3  } 동시                │ │
│  │   t4   뱅크0: 접근 4          뱅크0: 접근 4                        │ │
│  │  → 인터리빙 없음: 8 클럭                                         │ │
│  │  → 인터리빙 있음: 2 클럭 (4배 향상)                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 인터리빙 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터리빙 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터리빙 종류】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 뱅크 인터리빙 (Bank Interleaving)                              │ │
│  │     • DRAM 칩 내부의 여러 뱅크 활용                                  │ │
│  │     • 같은 행에 접근하지 않아도 병렬 접근 가능                         │ │
│  │     • 뱅크 수: 4, 8, 16개                                          │ │
│  │                                                               │ │
│  │  2. 채널 인터리빙 (Channel Interleaving)                           │ │
│  │     • 여러 메모리 채널 활용                                          │ │
│  │     • 듀얼/쿼드 채널                                                │ │
│  │     • 채널 수: 2, 4, 8개                                           │ │
│  │                                                               │ │
│  │  3. 랭크 인터리빙 (Rank Interleaving)                              │ │
│  │     • 여러 DRAM 랭크 활용                                           │ │
│  │     • DIMM의 양면 사용                                              │ │
│  │     • 랭크 수: 1, 2, 4개                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터리빙 주소 매핑】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  주소 비트       매핑                                               │ │
│  │  ──────────     ────────────────                                   │ │
│  │  하위 2~3비트    캐시 라인 오프셋 (64B)                               │ │
│  │  다음 2비트      뱅크 선택 (4-way)                                   │ │
│  │  다음 1비트      채널 선택 (듀얼)                                    │ │
│  │  상위 비트       행/열 주소                                          │ │
│  │                                                               │ │
│  │  예: 주소 0x1000                                                   │ │
│  │  0x1000 = 0001 0000 0000 0000                                      │ │
│  │  하위 6비트: 캐시 라인 내 오프셋                                      │ │
│  │  비트 6-7: 뱅크 선택 (00 = 뱅크 0)                                   │ │
│  │  비트 8: 채널 선택 (0 = 채널 0)                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터리빙 효과】                                                            │
│  ──────────────────                                                  │
│  • 대역폭 증대: 병렬 접근으로 대역폭 증가                                  │
│  • 지연 숨김: 한 뱅크의 지연을 다른 뱅크로 숨김                              │
│  • 충돌 감소: 뱅크 충돌 감소                                             │
│  • 효율 향상: 메모리 활용 효율 증가                                        │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 인터리빙 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터리빙 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【듀얼 채널 확인】                                                            │
│  ──────────────────                                                  │
│  $ sudo dmidecode --type memory | grep "Bank"                       │
│  Bank 0: 8GB                                                        │
│  Bank 1: 8GB                                                        │
│  Bank 2: 8GB                                                        │
│  Bank 3: 8GB                                                        │
│  → 4개 뱅크 = 듀얼 채널 × 2 랭크                                        │
│                                                                     │
│  【메모리 대역폭 측정】                                                        │
│  ──────────────────                                                  │
│  $ stream                                                            │
│  Function    Rate (GB/s)                                            │
│  Copy:       35.2                                                   │
│  Scale:      35.0                                                   │
│  Add:        38.5                                                   │
│  Triad:      38.3                                                   │
│  → 듀얼 채널 DDR4-3200 이론값: 51.2 GB/s                               │
│  → 실제 효율 ~70%: 35 GB/s                                            │
│                                                                     │
│  【인터리빙 최적화 팁】                                                        │
│  ──────────────────                                                  │
│  1. 채널 채우기: 모든 채널에 메모리 설치                                   │
│  2. 같은 용량: 각 채널에 같은 용량 모듈                                   │
│  3. 같은 스펙: 같은 속도, 타이밍 모듈 사용                                 │
│  4. 짝수 모듈: 듀얼 채널 활성화                                          │
│  5. 4모듈: 쿼드 채널 또는 듀얼+듀얼                                       │
│                                                                     │
│  【인터리빙 문제 해결】                                                        │
│  ──────────────────                                                  │
│  • 대역폭 저하: 채널 불균형 확인                                         │
│  • 단일 채널: 모듈이 1개만 설치됨                                         │
│  • 비대칭: 다른 용량/속도 모듈 혼용                                        │
│  • BIOS 설정: 채널 인터리빙 활성화 확인                                    │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 메모리 인터리빙: 여러 뱅크/채널에 데이터 분산 배치
• 종류: 뱅크, 채널, 랭크 인터리빙
• 원리: 연속 주소를 다른 뱅크/채널에 배치
• 효과: 대역폭 증대, 지연 숨김, 충돌 감소
• 방식: 주소 하위 비트로 뱅크/채널 선택
• 유사 기술: RAID 스트라이핑, 링크 본딩
• 최적화: 모든 채널 채우기, 같은 스펙 모듈
• 성능: 인터리빙 수준에 따라 2~8배 향상
```
