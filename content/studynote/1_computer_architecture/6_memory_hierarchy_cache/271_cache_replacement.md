# 271. 캐시 교체 정책 (Cache Replacement Policy)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시 미스 시 희생자 라인 선택 방법
> 2. **가치**: 적중률 최적화의 핵심 요소
> 3. **융합**: LRU, LFU, FIFO, Random, OPT

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

캐시 교체 정책(Cache Replacement Policy)은 **캐시 미스가 발생했을 때 어떤 캐시 라인을 교체할지 결정하는 알고리즘**이다. 적중률에 직접적인 영향을 미치며, LRU, LFU, FIFO, Random 등이 있다.

### 💡 비유: 도서관 책장 정리

교체 정책은 **도서관 책장 정리**와 같다. 새 책을 꽂을 자리가 없으면 어떤 책을 뺄까? 가장 안 읽은 책(LRU), 거의 안 읽은 책(LFU), 오래된 책(FIFO), 무작위(Random). 각각 장단점이 있다.

### 교체 정책 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                교체 정책 개요                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 교체 정책】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. LRU (Least Recently Used):                                    │ │
│  │     • 가장 오랫동안 사용하지 않은 라인 교체                              │ │
│  │     • 시간 지역성 기반                                              │ │
│  │     • 가장 널리 사용                                                │ │
│  │                                                               │ │
│  │  2. LFU (Least Frequently Used):                                  │ │
│  │     • 사용 빈도가 가장 낮은 라인 교체                                    │ │
│  │     • 인기도 기반                                                   │ │
│  │     • 장기적 패턴에 유리                                               │ │
│  │                                                               │ │
│  │  3. FIFO (First In First Out):                                    │ │
│  │     • 가장 먼저 들어온 라인 교체                                        │ │
│  │     • 단순한 구현                                                   │ │
│  │     • 순환 버퍼와 유사                                                │ │
│  │                                                               │
│  │  4. Random:                                                        │ │
│  │     • 무작위로 라인 선택                                               │ │
│  │     • 가장 단순                                                    │ │
│  │     • 하드웨어 구현 용이                                               │ │
│  │                                                               │ │
│  │  5. OPT (Optimal, Belady's):                                       │ │
│  │     • 미래에 가장 늦게 사용될 라인 교체                                    │ │
│  │     • 이론적 최적 (실현 불가)                                          │ │
│  │     • 비교 기준으로 사용                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【교체 정책 비교】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정책        적중률     구현 복잡도    하드웨어 비용    사용처           │ │
│  │  ──────      ──────     ──────────     ──────────     ──────           │ │
│  │  OPT         최고       불가능         -            기준              │ │
│  │  LRU         높음       중간           중간          캐시 표준          │ │
│  │  LFU         높음       높음           높음          일부 사용          │ │
│  │  FIFO        중간       낮음           낮음          간단한 캐시        │ │
│  │  Random      중간       가장 낮음       가장 낮음       일부 사용          │ │
│  │                                                               │ │
│  │  → 실제로는 LRU와 변형이 가장 많이 사용                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【교체 정책 선택 기준】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  기준           LRU       LFU       FIFO      Random            │ │
│  │  ──────          ──────    ──────    ──────    ──────            │ │
│  │  시간 지역성       최고       중간      낮음       낮음              │ │
│  │  구현 복잡도       중간       높음      낮음       최저              │ │
│  │  하드웨어 비용     중간       높음      낮음       최저              │ │
│  │  전력 소비        중간       높음      낮음       최저              │ │
│  │  적중률          높음       높음      중간       중간              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 교체 정책 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                교체 정책 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LRU (Least Recently Used)】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  원리: 가장 오랫동안 사용하지 않은 것 교체                                  │ │
│  │                                                               │ │
│  │  예시 (4-way 캐시):                                                 │ │
│  │  접근: A B C D E A B C D                                          │ │
│  │                                                               │ │
│  │  시간  캐시 상태 [MRU→LRU]       교체    설명                           │ │
│  │  ────  ──────────────────       ────    ────                           │ │
│  │  A    [A - - -]                 -       첫 접근                        │ │
│  │  B    [B A - -]                 -                                    │ │
│  │  C    [C B A -]                 -                                    │ │
│  │  D    [D C B A]                 -       캐시 만석                      │ │
│  │  E    [E D C B]                 A       LRU인 A 교체                   │ │
│  │  A    [A E D C]                 B       LRU인 B 교체                   │ │
│  │  B    [B A E D]                 C       LRU인 C 교체                   │ │
│  │  C    [C B A E]                 D       LRU인 D 교체                   │ │
│  │  D    [D C B A]                 E       LRU인 E 교체                   │ │
│  │                                                               │ │
│  │  구현:                                                             │ │
│  │  • 카운터 방식: 접근 시 타임스탬프 저장                                  │ │
│  │  • 스택 방식: MRU→LRU 순서 유지                                        │ │
│  │  • 행렬 방식: N×N 행렬로 순서 추적                                      │ │
│  │  • 근사 LRU: 참조 비트만 사용 (실용적)                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LFU (Least Frequently Used)】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  원리: 사용 빈도가 가장 낮은 것 교체                                      │ │
│  │                                                               │ │
│  │  특징:                                                             │ │
│  │  • 장기적 인기도 반영                                                 │ │
│  │  • 새 데이터에게 불리 (찬발적 캐시 문제)                                 │ │
│  │  • 카운터 오버플로우 문제                                              │ │
│  │                                                               │ │
│  │  구현:                                                             │ │
│  │  • 각 라인별 접근 카운터 유지                                           │ │
│  │  • 카운터가 가장 낮은 라인 교체                                          │ │
│  │  • 주기적 카운터 감소 (Aging)                                         │ │
│  │                                                               │ │
│  │  변형:                                                             │ │
│  │  • LFU with Aging: 오래된 데이터 가중치 감소                            │ │
│  │  • LFU-K: 최근 K번 접근만 고려                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FIFO vs Random】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  FIFO:                                                              │ │
│  │  • 큐 구조로 구현                                                   │ │
│  │  • Belady's Anomaly: 캐시 커도 적중률 감소 가능                          │ │
│  │  • 단순하지만 지역성 활용 못함                                          │ │
│  │                                                               │ │
│  │  Random:                                                             │ │
│  │  • 난수로 교체 대상 선택                                              │ │
│  │  • 구현 가장 단순                                                   │ │
│  │  • 평균적으로 FIFO와 유사                                            │ │
│  │  • 최악의 경우가 없음 (결정적이지 않음)                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 교체 정책 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                교체 정책 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【교체 정책 성능 비교】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  워크로드: 루프 패턴 (A B C D E A B C D E ...)                       │ │
│  │  캐시: 4-way, 4 라인                                               │ │
│  │                                                               │ │
│  │  정책      적중/총 접근    적중률    설명                               │ │
│  │  ──────    ──────────     ──────    ────                               │ │
│  │  OPT       5/10           50%       이론적 최적                         │ │
│  │  LRU       0/10            0%       워킹셋 5 > 캐시 4                    │ │
│  │  FIFO      0/10            0%       LRU와 동일                         │ │
│  │  Random    0~2/10         0~20%     운에 따라 다름                       │ │
│  │                                                               │ │
│  │  → 스래싱 패턴에서는 모든 정책 성능 낮음                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LRU 근사 기법】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. NRU (Not Recently Used):                                       │ │
│  │     • 참조 비트 1비트만 사용                                           │ │
│  │     • 참조 비트 0 중 하나 선택                                         │ │
│  │     • 단순하지만 근사치                                                │ │
│  │                                                               │ │
│  │  2. Clock Algorithm:                                                │ │
│  │     • 순환 리스트 + 참조 비트                                           │ │
│  │     • 포인터가 회전하며 참조 비트 0인 것 선택                             │ │
│  │     • OS 페이지 교체에 사용                                            │ │
│  │                                                               │ │
│  │  3. PLRU (Pseudo-LRU):                                              │ │
│  │     • 트리 구조로 근사                                                │ │
│  │     • log2(N) 비트만 필요                                            │ │
│  │     • CPU 캐시에 많이 사용                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【교체 정책 최적화 팁】                                                         │
│  ──────────────────                                                  │
│  1. 워크로드 분석: 접근 패턴 파악                                          │
│  2. 캐시 크기: 워킹셋 이상 유지                                           │
│  3. 연관도: Conflict Miss 감소                                          │
│  4. 프리패칭: Compulsory Miss 감소                                       │
│  5. 데이터 구조: 지역성 극대화                                             │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 교체 정책: 미스 시 희생자 라인 선택 알고리즘
• LRU: 가장 오래된 것 교체, 시간 지역성 기반, 표준
• LFU: 빈도 낮은 것 교체, 인기도 기반
• FIFO: 먼저 들어온 것 교체, 단순
• Random: 무작위 선택, 가장 단순
• OPT: 미래 기반, 이론적 최적 (불가능)
• 선택 기준: 워크로드, 구현 복잡도, 비용
• 근사 기법: NRU, Clock, PLRU
• 성능: 적중률 직접 영향
• 현대: LRU 근사가 표준
```
