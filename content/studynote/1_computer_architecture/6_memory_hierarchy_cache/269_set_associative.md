# 269. 집합 연관 사상 (Set Associative Mapping)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 직접 사상과 완전 연관의 절충안
> 2. **가치**: 적절한 복잡도로 높은 적중률 달성
> 3. **융합**: N-way, 셋, 웨이, 현대 캐시 표준

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

집합 연관 사상(Set Associative Mapping)은 **캐시를 여러 셋으로 나누고, 각 셋 내에서 N개의 라인에 배치 가능한 방식**이다. 직접 사상의 단순함과 완전 연관의 유연성을 절충한 방식으로, 현대 캐시의 표준이다.

### 💡 비유: 구역 주차장

집합 연관은 **구역 주차장**과 같다. A구역, B구역, C구역이 있다. 번호에 따라 구역은 정해지지만, 그 구역 내에서는 아무 자리나 가능하다. A구역에 4자리가 있으면 4개 중 하나 선택.

### 집합 연관 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                집합 연관 사상 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【집합 연관 원리】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  매핑: 메모리 블록 → 특정 셋 → 셋 내 N개 라인 중 하나                   │ │
│  │                                                               │ │
│  │  메모리 주소 구조:                                                   │ │
│  │  ┌──────────────────┬──────────────────┬──────────────────┐    │ │
│  │  │       Tag        │    Set Index     │      Offset      │    │ │
│  │  │     (t비트)       │     (s비트)       │     (b비트)       │    │ │
│  │  └──────────────────┴──────────────────┴──────────────────┘    │ │
│  │                                                               │ │
│  │  N-way: 각 셋에 N개 라인 (2, 4, 8, 16-way)                         │ │
│  │  셋 수: 2^s                                                       │ │
│  │  총 라인 수: 셋 수 × N-way                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【4-way 집합 연관 구조 예시】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  캐시: 64 라인 = 16 셋 × 4-way                                     │ │
│  │                                                               │ │
│  │         ┌──────────────────────────────────────────────┐       │ │
│  │         │              Set 0 (4-way)                     │       │ │
│  │         │  ┌────┐ ┌────┐ ┌────┐ ┌────┐                  │       │ │
│  │         │  │Way0│ │Way1│ │Way2│ │Way3│                  │       │ │
│  │         │  └────┘ └────┘ └────┘ └────┘                  │       │ │
│  │         └──────────────────────────────────────────────┘       │ │
│  │         ┌──────────────────────────────────────────────┐       │ │
│  │         │              Set 1 (4-way)                     │       │ │
│  │         │  ┌────┐ ┌────┐ ┌────┐ ┌────┐                  │       │ │
│  │         │  │Way0│ │Way1│ │Way2│ │Way3│                  │       │ │
│  │         │  └────┘ └────┘ └────┘ └────┘                  │       │ │
│  │         └──────────────────────────────────────────────┘       │ │
│  │                       ...                                      │ │
│  │         ┌──────────────────────────────────────────────┐       │ │
│  │         │             Set 15 (4-way)                    │       │ │
│  │         │  ┌────┐ ┌────┐ ┌────┐ ┌────┐                  │       │ │
│  │         │  │Way0│ │Way1│ │Way2│ │Way3│                  │       │ │
│  │         │  └────┘ └────┘ └────┘ └────┘                  │       │ │
│  │         └──────────────────────────────────────────────┘       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【N-way 의미】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  N-way    셋 내 라인 수    충돌 저항    비교기 수    사용처          │ │
│  │  ──────    ───────────     ────────     ────────     ──────          │ │
│  │  1-way    1 (직접 사상)     낮음         1           거의 안 씀        │ │
│  │  2-way    2               낮음         2           저전력           │ │
│  │  4-way    4               중간         4           L1 일반적        │ │
│  │  8-way    8               높음         8           L1/L2 표준       │ │
│  │  16-way   16              높음         16          L2/L3 표준       │ │
│  │  32-way   32              매우 높음     32          L3              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 집합 연관 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                집합 연관 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【집합 연관 동작】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 주소 분해:                                                      │ │
│  │     CPU 주소 → [Tag][Set Index][Offset]                           │ │
│  │                                                               │ │
│  │  2. 셋 선택:                                                        │ │
│  │     Set = Index                                                   │ │
│  │                                                               │ │
│  │  3. 셋 내 태그 비교 (N-way 병렬):                                     │ │
│  │     for (i = 0; i < N; i++)                                       │ │
│  │       if (Valid[Set][i] && Tag[Set][i] == Tag)                   │ │
│  │         Hit at Way i                                               │ │
│  │                                                               │ │
│  │  4. Hit:                                                           │ │
│  │     Data = Cache[Set][Way].Data[Offset]                           │ │
│  │     LRU 갱신                                                       │ │
│  │                                                               │ │
│  │  5. Miss:                                                          │ │
│  │     교체 정책으로 희생자 Way 선택                                      │ │
│  │     Victim = Replacement_Policy(Set)                              │ │
│  │     Cache[Set][Victim] = Memory[Address]                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【직접 사상 vs 집합 연관 vs 완전 연관】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성          직접 사상      4-way 집합 연관   완전 연관             │ │
│  │  ──────         ──────        ──────────        ──────             │ │
│  │  인덱스 비트      log(S)        log(S/N)         없음               │ │
│  │  비교기 수       1             4                S                  │ │
│  │  Conflict Miss   높음          중간              없음               │ │
│  │  교체 정책       불필요         필요              필요               │ │
│  │  하드웨어        단순          중간              복잡               │ │
│  │  적중률          낮음          높음              최고               │ │
│  │  전력 소비       낮음          중간              높음               │ │
│  │                                                               │ │
│  │  S = 총 라인 수, N = way 수                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【연관도 vs 성능 트레이드오프】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  연관도      적중률 향상    Hit Time 증가   전력 증가   면적 증가        │ │
│  │  ──────      ──────────     ────────────     ────────   ────────        │ │
│  │  2-way      기준 +3%       +5%             +10%       +10%            │ │
│  │  4-way      기준 +5%       +10%            +20%       +20%            │ │
│  │  8-way      기준 +7%       +15%            +30%       +30%            │ │
│  │  16-way     기준 +8%       +20%            +40%       +40%            │ │
│  │  32-way     기준 +8.5%     +25%            +50%       +50%            │ │
│  │  → 8-way 이상은 수확 체감 (적중률)                                     │ │
│  │  → 하지만 Hit Time, 전력, 면적은 계속 증가                              │ │
│  │  → 최적점: 워크로드에 따라 다름                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【현대 CPU 집합 연관 사양】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU             L1 I-Cache   L1 D-Cache   L2          L3          │ │
│  │  ─────────────   ──────────   ──────────   ──────────   ──────        │ │
│  │  Intel i9        8-way        8-way        4-way        16-way       │ │
│  │  AMD Zen 3       8-way        8-way        8-way        16-way       │ │
│  │  Apple M2        8-way        8-way        8-way        12-way       │ │
│  │  ARM A78         4-way        4-way        8-way        16-way       │ │
│  │  → L1: 4~8-way, L2: 8-way, L3: 12~16-way 일반적                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 집합 연관 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                집합 연관 × 타 과목 융합                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 캐시 설계: 연관도 선택이 핵심                                        │
│  • 성능: 적중률과 Hit Time 균형                                        │
│  • 전력: 연관도 높을수록 전력 증가                                       │
│  • 면적: N-way 비교기 필요                                             │
│                                                                     │
│  【운영체제】                                                          │
│  • 페이지 컬러링: 셋 인덱스와 페이지 할당                                  │
│  • TLB: 작은 집합 연관 또는 완전 연관                                    │
│  • 스케줄링: 캐시 공유 고려                                            │
│  • 메모리: 셋 충돌 최소화                                              │
│                                                                     │
│  【컴파일러】                                                          │
│  • 데이터 배치: 셋 분산 배치                                            │
│  • 타일링: 셋 수 고려                                                  │
│  • 스트라이드: 셋 인덱스 패턴 분석                                       │
│  • 구조체: 캐시 라인 정렬                                              │
│                                                                     │
│  【성능 분석】                                                         │
│  • 벤치마크: 연관도별 성능 측정                                         │
│  • 프로파일링: 셋별 미스 분석                                           │
│  • 시뮬레이션: 최적 연관도 탐색                                          │
│  • 튜닝: 워크로드별 연관도                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  워크로드별 최적 연관도:                                                        │ │
│  │  ──────────────────────                                              │ │
│  │  워크로드         충돌 패턴      추천 연관도    이유                      │ │
│  │  ──────────        ──────        ──────        ────                      │ │
│  │  HPC              높음          16-way       대용량 데이터                │ │
│  │  Database         중간          8-way        다양한 쿼리                  │ │
│  │  Gaming           낮음          4-way        지역성 높음                  │ │
│  │  Web Server       중간          8-way        다양한 요청                  │ │
│  │  Mobile           낮음          4-way        전력 제약                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 집합 연관 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                집합 연관 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 구조 확인】                                                            │
│  ──────────────────                                                  │
│  $ lscpu                                                            │
│  L1d cache: 32 KiB, 8-way, 64 sets                                 │
│  L1i cache: 32 KiB, 8-way, 64 sets                                 │
│  L2 cache: 256 KiB, 8-way, 512 sets                                │
│  L3 cache: 8192 KiB, 16-way, 8192 sets                             │
│                                                                     │
│  【집합 연관 주소 계산】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  L1 D-Cache: 32KB, 8-way, 64B 라인                                │ │
│  │                                                               │ │
│  │  총 라인 수: 32KB / 64B = 512 라인                                  │ │
│  │  셋 수: 512 / 8-way = 64 셋                                        │ │
│  │  Offset 비트: log2(64) = 6비트                                     │ │
│  │  Set Index 비트: log2(64) = 6비트                                  │ │
│  │  Tag 비트: 64 - 6 - 6 = 52비트 (64비트 주소)                          │ │
│  │                                                               │ │
│  │  주소 구조: [Tag:52][Set:6][Offset:6]                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【셋 충돌 분석】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제: 2개 배열 A[64], B[64] (각 256B)                             │ │
│  │  L1: 32KB, 8-way, 64 셋                                           │ │
│  │                                                               │ │
│  │  나쁜 패턴:                                                          │ │
│  │  A[0], A[64], A[128]... 모두 같은 셋 매핑                            │ │
│  │  256B 간격 = 4 라인 간격 = 같은 셋                                   │ │
│  │  → 8개까지만 저장 가능, 그 이후 교체                                   │ │
│  │                                                               │ │
│  │  좋은 패턴:                                                          │ │
│  │  연속 접근: A[0], A[1], A[2]... 다른 셋                              │ │
│  │  → 캐시 효율적 활용                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【집합 연관 최적화 팁】                                                         │
│  ──────────────────                                                  │
│  1. 데이터 정렬: 캐시 라인 경계                                          │
│  2. 스트라이드 패턴: 셋 수 고려                                           │
│  3. 타일링: 셋 내 way 수 이내로                                          │
│  4. 구조체 분리: 핫/콜드 분리                                             │
│  5. 배열 패딩: 셋 분산                                                  │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 집합 연관: 직접 사상과 완전 연관의 절충
• 구조: 셋으로 분리, 셋 내 N-way
• 주소: [Tag][Set Index][Offset]
• N-way: 2, 4, 8, 16, 32-way
• 장점: 적절한 복잡도, 높은 적중률
• 단점: 교체 정책 필요, N개 비교기
• 트레이드오프: 연관도 ↑ = 적중률 ↑, Hit Time ↑
• 현대 표준: L1 4~8-way, L2 8-way, L3 16-way
• 선택: 워크로드, 전력, 면적 고려
• 최적화: 셋 분산, 스트라이드 분석
```
