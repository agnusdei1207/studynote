# 267. 직접 사상 (Direct Mapping)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 각 메모리 블록이 하나의 캐시 라인에만 매핑
> 2. **가치**: 가장 단순하고 빠른 캐시 구조
> 3. **융합**: 충돌 미스, 모듈로 연산, 인덱싱

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

직접 사상(Direct Mapping)은 **각 메모리 블록이 정확히 하나의 캐시 라인에만 매핑되는 방식**이다. 메모리 주소의 인덱스 비트로 캐시 라인을 결정하며, 가장 단순한 캐시 구조다.

### 💡 비유: 번호별 지정 주차

직접 사상은 **번호별 지정 주차**와 같다. 101호는 A1 자리만, 201호도 A1 자리만. 번호 끝자리가 같으면 같은 자리. A1에 101호가 있으면 201호는 못 들어온다.

### 직접 사상 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                직접 사상 개요                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【직접 사상 원리】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  매핑 공식: Cache Line = Memory Block Address % 캐시 라인 수       │ │
│  │                                                               │ │
│  │  메모리 주소 구조:                                                   │ │
│  │  ┌──────────────────┬──────────────────┬──────────────────┐    │ │
│  │  │       Tag        │      Index       │      Offset      │    │ │
│  │  │     (t비트)       │     (s비트)       │     (b비트)       │    │ │
│  │  └──────────────────┴──────────────────┴──────────────────┘    │ │
│  │                                                               │ │
│  │  Tag: 블록 식별 (같은 인덱스 내 구분)                                │ │
│  │  Index: 캐시 라인 선택 (2^s 개)                                    │ │
│  │  Offset: 블록 내 위치 (2^b 바이트)                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【직접 사상 예시】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  캐시: 16 라인 (인덱스 0~15), 블록 크기: 4바이트                       │ │
│  │                                                               │ │
│  │  메모리 주소     인덱스 계산      매핑되는 캐시 라인                     │ │
│  │  ──────────     ──────────      ──────────────                     │ │
│  │  0x0000         0 % 16 = 0      라인 0                              │ │
│  │  0x0004         1 % 16 = 1      라인 1                              │ │
│  │  0x0010         4 % 16 = 4      라인 4                              │ │
│  │  0x0040         16 % 16 = 0     라인 0 (충돌!)                       │ │
│  │  0x0080         32 % 16 = 0     라인 0 (충돌!)                       │ │
│  │  0x0100         64 % 16 = 0     라인 0 (충돌!)                       │ │
│  │                                                               │ │
│  │  → 주소 0, 64, 128, 192, ... 모두 같은 라인                          │ │
│  │  → 16바이트 간격으로 같은 인덱스 반복                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【직접 사상 캐시 구조】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  캐시 라인 구조:                                                     │ │
│  │  ┌───────────┬───────────┬───────────┐                         │ │
│  │  │ Valid (1b)│ Tag (t비트)│ Data (B바이트)│                      │ │
│  │  └───────────┴───────────┴───────────┘                         │ │
│  │                                                               │ │
│  │  캐시 전체 (8 라인 예시):                                            │ │
│  │  ┌───────────────────────────────────────────────────────┐     │ │
│  │  │  라인 0: V │ Tag  │ Data                               │     │ │
│  │  │  라인 1: V │ Tag  │ Data                               │     │ │
│  │  │  라인 2: V │ Tag  │ Data                               │     │ │
│  │  │  라인 3: V │ Tag  │ Data                               │     │ │
│  │  │  ...                                                    │     │ │
│  │  │  라인 7: V │ Tag  │ Data                               │     │ │
│  │  └───────────────────────────────────────────────────────┘     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 직접 사상 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                직접 사상 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【직접 사상 동작】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 주소 분해:                                                      │ │
│  │     CPU 주소 → [Tag][Index][Offset]                               │ │
│  │                                                               │ │
│  │  2. 인덱스로 캐시 라인 선택:                                          │ │
│  │     Cache[Index] 접근                                             │ │
│  │                                                               │ │
│  │  3. 태그 비교:                                                       │ │
│  │     if (Valid && Cache[Index].Tag == Tag) → Hit                  │ │
│  │     else → Miss                                                   │ │
│  │                                                               │ │
│  │  4. Hit: 데이터 반환                                                 │ │
│  │     Data = Cache[Index].Data[Offset]                              │ │
│  │                                                               │ │
│  │  5. Miss: 메모리에서 가져와서 교체                                      │ │
│  │     Cache[Index] = Memory[Address]                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【직접 사상 장단점】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │
│  │  • 가장 단순한 하드웨어 (1개 비교기)                                    │ │
│  │  • 가장 빠른 접근 (1 사이클)                                          │ │
│  │  • 가장 낮은 전력 소비                                                │ │
│  │  • 가장 작은 면적                                                    │ │
│  │  • 교체 정책 불필요 (1개 후보)                                         │ │
│  │                                                               │ │
│  │  단점:                                                               │
│  │  • 높은 Conflict Miss                                               │ │
│  │  • 특정 접근 패턴에서 성능 저하                                         │ │
│  │  • 캐시 공간 비효율                                                   │ │
│  │  • 스래싱 발생 가능                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Conflict Miss 예시】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제: 2개 배열 A[64], B[64]                                       │ │
│  │  캐시: 64바이트 (16 라인 × 4바이트)                                    │ │
│  │                                                               │ │
│  │  for (i = 0; i < 64; i++)                                        │ │
│  │    A[i] = B[i];                                                   │ │
│  │                                                               │ │
│  │  A[0] → 라인 0, B[0] → 라인 0 (충돌!)                                │ │
│  │  A[1] → 라인 1, B[1] → 라인 1 (충돌!)                                │ │
│  │  ...                                                              │ │
│  │  → 모든 접근이 미스! (0% 적중률)                                       │ │
│  │  → 실제로는 캐시 크기보다 작은 데이터도 충돌                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【직접 사상 적중률 분석】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  워크로드          직접 사상    2-way    4-way    8-way              │ │
│  │  ──────────        ──────      ──────    ──────    ──────              │ │
│  │  순차 접근          95%         96%       97%       98%               │ │
│  │  랜덤 접근          70%         75%       80%       85%               │ │
│  │  스트라이드(충돌)    30%         70%       85%       92%               │ │
│  │  혼합              85%         90%       93%       95%               │ │
│  │  → 충돌 패턴에서 큰 차이                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```

│                                                                     │
│  【컴퓨터 구조】                                                      │

│  • 성능: 충돌 패턴에 따라 성능 편차                                       │
│  • 파이프라인: 1 사이클 접근                                            │
│  • 하드웨어: 1개 태그 비교기                                             │
│                                                                     │
│  【운영체제】                                                          │

│  • 버퍼 캐시: 간단한 구조                                               │
│  • 파일 시스템: 블록 캐시                                               │
│                                                                     │
│  【컴파일러】                                                          │
│  • 데이터 배치: 충돌 회피                                               │
│  • 배열 패딩: 같은 인덱스 회피                                           │
│  • 루프 최적화: 스트라이드 변경                                          │
│  • 구조체 배치: 캐시 라인 정렬                                          │
│                                                                     │
│  【알고리즘】                                                          │
│  • 행렬 연산: 블록화로 충돌 회피                                         │
│  • 정렬: 캐시 친화적 알고리즘                                            │
│  • 그래프: 인접 행렬 vs 리스트                                           │
│  • 해시: 캐시 효율                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  직접 사상 최적화 기법:                                                        │ │
│  │  ──────────────────────                                              │ │
│  │  기법                문제              해결                             │ │
│  │  ──────────           ──────            ──────                         │ │
│  │  배열 패딩            충돌 미스          인덱스 분리                       │ │
│  │  데이터 재구성         충돌 미스          접근 패턴 변경                    │ │
│  │  루프 타일링           캐시 스래싱         워킹셋 감소                      │ │
│  │  스트라이드 변경       같은 인덱스          접근 간격 조정                   │ │
│  │  구조체 분리           충돌               핫/콜드 데이터 분리               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 직접 사상 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                직접 사상 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【직접 사상 확인】                                                            │
│  ──────────────────                                                  │
│  $ lscpu | grep -i way                                              │
│  L1d cache: 32 KiB, 8-way  # 8-way = 집합 연관                      │
│                                                                     │
│  # 직접 사상 = 1-way                                                 │
│  # 현대 CPU에서 직접 사상은 거의 사용 안 함                               │
│                                                                     │
│  【직접 사상 충돌 디버깅】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제 코드:                                                          │ │
│  │  int A[1024], B[1024];  // 4KB씩                                   │ │
│  │  for (i = 0; i < 1024; i += 64)  // 256바이트 간격                   │ │
│  │    A[i] = B[i];  // 같은 인덱스 반복                                 │ │
│  │                                                               │ │
│  │  해결 1: 패딩 추가                                                    │ │
│  │  int A[1024], padding[32], B[1024];  // 인덱스 분리                 │ │
│  │                                                               │ │
│  │  해결 2: 순차 접근                                                    │ │
│  │  for (i = 0; i < 1024; i++)  // 연속 접근                           │ │
│  │    A[i] = B[i];                                                    │ │
│  │                                                               │ │
│  │  해결 3: 루프 분리                                                    │ │
│  │  for (i = 0; i < 1024; i++) A[i] = tmp;                            │ │
│  │  for (i = 0; i < 1024; i++) tmp = B[i];                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【직접 사상 사용 사례】                                                         │
│  ──────────────────                                                  │
│  • 작은 캐시 (≤64 엔트리): 단순함 우선                                   │
│  • 임베디드 시스템: 전력, 면적 제약                                        │
│  • 교육용: 캐시 개념 학습                                               │
│  • 특수 목적: 고정 패턴 워크로드                                          │
│                                                                     │
│  【직접 사상 vs 집합 연관 선택】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  기준           직접 사상         집합 연관                          │ │
│  │  ──────          ──────           ──────                           │ │
│  │  하드웨어 복잡도   낮음             높음                              │ │
│  │  접근 속도        빠름             느림                              │ │
│  │  적중률          낮음 (충돌)       높음                              │ │
│  │  전력 소비        낮음             높음                              │ │
│  │  면적            작음             큼                                │ │
│  │  → 성능 중시: 집합 연관, 단순함 중시: 직접 사상                         │ │
│  │  → 현대: 대부분 집합 연관 사용                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 직접 사상: 각 블록 → 정확히 1개 캐시 라인
• 공식: Cache Line = Memory Block % 캐시 라인 수
• 주소 구조: [Tag][Index][Offset]
• 장점: 단순, 빠름, 저전력, 작은 면적
• 단점: 높은 Conflict Miss, 스래싱
• 충돌: 같은 인덱스 블록 간 경쟁
• 적중률: 패턴에 따라 편차 큼
• 사용처: 작은 캐시, 임베디드, 교육용
• 최적화: 패딩, 재구성, 타일링
• 현대: 대부분 집합 연관 사용
```
