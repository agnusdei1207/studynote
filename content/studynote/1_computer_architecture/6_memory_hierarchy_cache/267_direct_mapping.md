> **핵심 인사이트**
> 1. 직접 사상(Direct Mapping)은 메인 메모리의 특정 블록이 **캐시 메모리 안에서 들어갈 수 있는 위치(Index)가 단 1곳으로 고정되어 있는 가장 단순한 캐시 맵핑 방식**이다.
> 2. 데이터를 찾을 때 해당 인덱스 칸 딱 한 곳만 열어보고 Tag(이름표)를 비교하면 되므로, **하드웨어 회로가 극도로 단순하고 검색 속도(Hit Time)가 모든 방식 중 가장 빠르다.**
> 3. 하지만 똑같은 인덱스를 배정받은 두 개의 데이터가 번갈아 가며 캐시에 들어오려고 하면, 캐시 전체 용량이 텅텅 비어있어도 서로를 계속 쫓아내는 **충돌 미스(Conflict Miss, 캐시 스래싱)**가 발생한다는 치명적인 단점이 있다.

---

## Ⅰ. 직접 사상의 주소 배정 원리

메인 메모리의 주소를 캐시의 주소로 맵핑할 때, 직접 사상은 아주 간단한 '나머지 연산(Modulo)'을 사용합니다.

- **수식:** `캐시 인덱스 = (메인 메모리 블록 번호) MOD (캐시의 전체 줄 수)`
- **예시:** 캐시가 총 4칸(0~3)밖에 없다고 가정해 봅시다.
  - 메모리 0번, 4번, 8번 블록은 $\rightarrow$ 무조건 캐시 **0번 칸**으로!
  - 메모리 1번, 5번, 9번 블록은 $\rightarrow$ 무조건 캐시 **1번 칸**으로!

📢 **섹션 요약 비유:**
주차장에 10개의 칸(0~9번)이 있을 때, 자동차 번호판의 제일 끝자리 숫자와 똑같은 번호의 주차 칸에만 무조건 주차해야 하는 아주 엄격하고 단순한 주차장 규칙입니다.

---

## Ⅱ. 주소 분할과 하드웨어 구조

CPU가 32비트 메모리 주소를 요청하면, 하드웨어는 이 주소를 가위로 3등분(`Tag | Index | Offset`)합니다.

- **하드웨어 동작 흐름:**
  1. **Index 추출:** 주소 가운데에 있는 Index 비트를 봅니다. (예: `010`이면 2번 칸으로 직행)
  2. **Tag 비교 (Comparator 1개):** 2번 칸에 이미 누군가 들어와 있다면, 그 데이터의 이마에 붙어있는 Tag와 내가 들고 온 주소의 앞부분 Tag 비트를 비교합니다.
  3. **결과:** 일치하면 Hit! 일치하지 않으면 쓰레기로 간주하고 메모리로 쫓아갑니다(Miss).
- **장점:** 복잡하게 찾을 필요 없이 지정석으로 달려가서 **단 1개의 비교기(Comparator)**만 돌리면 끝나므로 전력 소모가 적고 속도가 광속입니다.

📢 **섹션 요약 비유:**
"2학년 3반(Index) 가서 김철수(Tag) 찾아와!"라고 했을 때, 3반 교실 문 열고 철수 명찰 단 애가 있는지 딱 한 명만 확인하고 1초 만에 나오는 방식입니다.

---

## Ⅲ. 치명적 단점: 충돌 미스 (Conflict Miss)의 공포

직접 사상의 가장 큰 문제는 융통성이 '0'이라는 점입니다.

- **스래싱(Thrashing) 상황:** CPU가 메모리 0번 블록과 4번 블록의 데이터를 번갈아 가며 계속 읽어야 하는 루프(Loop)를 만났습니다.
- **비극의 시작:** 0번과 4번 블록은 둘 다 캐시의 **0번 칸(Index 0)**에만 들어갈 수 있습니다.
  - 0번 블록 로드 $\rightarrow$ (0번 칸에 넣음)
  - 4번 블록 로드 $\rightarrow$ (0번 칸에 있던 0번 블록을 쫓아내고 4번 블록을 넣음)
  - 다시 0번 블록 로드 $\rightarrow$ (방금 넣은 4번 블록을 쫓아내고 다시 0번 블록을 넣음)
- **결과:** 나머지 1번, 2번, 3번 캐시 칸은 텅텅 비어있는데, 오직 0번 칸 하나에서만 치고받고 싸우느라 적중률(Hit Ratio)이 0%로 추락해버립니다.

📢 **섹션 요약 비유:**
번호판 끝자리가 7인 자동차 두 대가 주차장에 들어왔는데, 다른 9개의 주차 칸이 텅텅 비어있음에도 불구하고 굳이 7번 칸 하나를 두고 서로 네가 나가라며 밤새도록 싸우는 멍청한 주차장입니다.

---

## Ⅳ. 현대 아키텍처에서의 활용

충돌 미스의 공포 때문에, 현대의 똑똑한 CPU들은 L2나 L3 캐시에서는 절대 이 방식을 쓰지 않습니다(Set Associative를 씀). 하지만 **L1 캐시**에서는 이야기가 다릅니다.

- **L1 캐시의 부활:** L1 캐시의 최우선 덕목은 적중률(Hit Ratio)이 아니라, "무조건 1클럭 안에 대답해라(Hit Time)"입니다. 
- 직접 사상은 하드웨어가 얇고 비교를 1번만 하므로 속도가 미치도록 빠릅니다. 그래서 일부 고성능 CPU들은 충돌 미스를 감수하고라도 L1 캐시에 한해 (또는 2-way 수준의 아주 얕은 사상으로) 직접 사상의 철학을 도입하여 극한의 속도를 쥐어짜기도 합니다.

📢 **섹션 요약 비유:**
가끔 배달 사고(Conflict Miss)가 나더라도, 일단 주문 들어오면 1초 만에 요리를 빼버리는 극단적인 속도전이 필요한 패스트푸드점(L1 캐시)에서만 쓰는 특별한 시스템입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- `[Cache Mapping]`: 직접 사상이 속해있는 캐시 위치 결정 규칙
- `[Index / Tag]`: 직접 사상이 데이터를 찾기 위해 사용하는 주소 분할의 핵심 필드
- `[Conflict Miss (충돌 미스)]`: 직접 사상의 존재 이유를 위협하는 가장 치명적인 아킬레스건
- `[Set Associative Mapping]`: 직접 사상의 충돌 미스를 해결하기 위해 융통성을 부여한 상위 호환 방식
- `[Hit Time (적중 시간)]`: 직접 사상이 다른 모든 맵핑 방식을 압도하는 단 하나의 장점

## 👶 어린이를 위한 3줄 비유 설명
1. 직접 사상은 학교 신발장에 "학번 끝자리가 5번인 학생은 무조건 5번 신발장만 써!"라고 정해둔 규칙이에요.
2. 찾을 때 5번 칸만 딱 열어보면 되니까, 1초 만에 내 신발을 찾을 수 있어서 엄청 편하고 빨라요.
3. 하지만 5번으로 끝나는 친구가 2명이 되면, 다른 칸이 100개나 비어있어도 둘이서 5번 칸을 차지하려고 매일매일 싸워야 한답니다!
