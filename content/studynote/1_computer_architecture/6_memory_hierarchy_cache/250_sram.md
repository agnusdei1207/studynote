# SRAM (Static RAM)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SRAM(Static Random Access Memory)은 **6개의 트랜지스터로 구성된 플립플롭 구조**를 사용하여 데이터를 저장하는 비휘발성이 아닌 고속 반도체 메모리로, 리프레시 없이 전원이 공급되는 동안 데이터를 유지한다.
> 2. **가치**: DRAM 대비 **10~100배 빠른 접근 속도**(나노초 vs 마이크로초)를 제공하며, CPU 캐시(L1/L2/L3), 레지스터 파일, 고속 버퍼의 핵심 소자다.
> 3. **융합**: 속도는 압도적이지만 셀당 6개의 트랜지스터를 사용하여 **면적과 비용이 DRAM 대비 4~6배** 높아, 용량보다는 속도가 중요한 계층에 제한적으로 사용된다.

+++

## Ⅰ. SRAM의 개요

### 1. 정의
- SRAM은 전원이 공급되는 한 별도의 리프레시(Refresh) 없이 데이터를 유지하는 휘발성 메모리다. '정적(Static)'이라는 이름은 동적(Dynamic)인 리프레시가 필요 없다는 의미에서 유래했다.

### 2. 등장 배경: CPU 속도와 메모리 속도의 격차
- CPU 클럭 속도는 기하급수적으로 증가했으나, 메인 메모리(DRAM) 속도는 상대적으로 정체되었다.
- 이 속도 격차(Memory Wall)를 메우기 위해 CPU에 가까울수록 더 빠른 SRAM을 배치하는 캐시 계층이 도입되었다.

### 3. 💡 비유: '책상 위의 메모지'
- SRAM은 **'언제든 바로 확인할 수 있는 책상 위의 메모지'**와 같다.
- 책장(DRAM)에서 책을 꺼내는 데는 걸어가는 시간이 필요하지만, 책상 위 메모지는 손만 뻗으면 즉시 읽을 수 있다. 단, 책상은 작아서 많은 메모지를 둘 수 없다.

- **📢 섹션 요약 비유**: 용량은 작지만 접근 속도가 빛보다 빠른 '비싼 VIP 전용 창고'입니다.

## Ⅱ. 셀 구조 및 동작 원리 (Deep Dive)

### 1. 6T SRAM 셀 구조
- 표준 SRAM 셀은 6개의 트랜지스터(6T)로 구성된다.

```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                    6T SRAM 셀 회로도                             │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │         Vdd (전원)                                              │
    │           │                                                     │
    │     ┌─────┴─────┐                                               │
    │     │           │                                               │
    │    (P1)       (P2)    ← PMOS 풀업 (Pull-up)                     │
    │     │           │                                               │
    │   ┌─┴─┐       ┌─┴─┐                                             │
    │   │   │       │   │   ← 크로스커플드 인버터 (저장 소자)          │
    │   └─┬─┘       └─┬─┘                                             │
    │     │     ×     │     ← 피드백 루프 (데이터 유지)                │
    │   ┌─┴─┐       ┌─┴─┐                                             │
    │   │   │       │   │   ← NMOS 풀다운 (Pull-down)                  │
    │   └─┬─┘       └─┬─┘                                             │
    │     │           │                                               │
    │    (N1)       (N2)                                              │
    │     │           │                                               │
    │    (N3)       (N4)    ← 액세스 트랜지스터 (WL 제어)              │
    │     │           │                                               │
    │  BL ───┘       └─── BLB   ← 비트라인 (Bit Line / Bit Line Bar)  │
    │           ↑                                                    │
    │          WL (Word Line)                                        │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 동작 모드
- **읽기(Read)**: 워드라인(WL) 활성화 → 비트라인(BL/BLB) 전압 차이 감지 → 센스 앰프로 데이터 판독
- **쓰기(Write)**: 워드라인(WL) 활성화 → 비트라인에 원하는 값 인가 → 플립플롭 상태 강제 전이
- **유지(Hold)**: 워드라인 비활성화 → 크로스커플드 인버터가 상태를 무한히 유지 (전원 공급 시)

### 3. SRAM vs DRAM 셀 비교
| 특성 | SRAM (6T) | DRAM (1T + 1C) |
|:---|:---|:---|
| **트랜지스터 수** | 6개 | 1개 + 1 커패시터 |
| **리프레시** | 불필요 | 필요 (64ms 주기) |
| **접근 속도** | ~1ns | ~50ns |
| **면적** | 큼 | 작음 |
| **소비 전력** | 높음 (누설 전류) | 낮음 (리프레시 제외) |

- **📢 섹션 요약 비유**: 6명의 경비원이 24시간 교대로 지키는 VIP 방(SRAM) vs 1명의 경비원이 가끔씩 순찰하는 일반 창고(DRAM)입니다.

## Ⅲ. SRAM의 성능 특성 (Trade-off)

### 1. 접근 시간 (Access Time)
- L1 캐시: 1~3 클럭 (약 1ns 이내)
- L2 캐시: 5~10 클럭
- L3 캐시: 20~40 클럭

### 2. 전력 소모
- **정적 전력**: 트랜지스터 누설 전류로 인해 대기 전력이 큼.
- **동적 전력**: 읽기/쓰기 시 비트라인 충전/방전.
- 모바일 기기에서는 L3 캐시를 줄이거나 전력 게이팅하는 이유.

### 3. 안정성: SNM (Static Noise Margin)
- SRAM 셀이 노이즈 없이 데이터를 유지할 수 있는 여유도.
- 공정 미세화(5nm 이하)로 SNM이 악화되어 설계 난이도 증가.

+++

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: SRAM이 칩 면적의 절반 이상
- 현대 고성능 CPU(AP)의 다이(die) 면적 중 **50~70%가 SRAM(캐시)**이다.
- Apple M1/M2의 거대한 캐시는 성능의 비결이지만, 동시에 칩 크기와 비용을 증가시키는 요인.

### 2. 안티패턴: SRAM을 메인 메모리로 사용
- SRAM은 너무 비싸고 면적을 많이 차지하여 대용량 메인 메모리로는 경제적이지 않음.
- 특수 상황(우주선, 극한 환경)에서만 예외적으로 사용.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **CPU-메모리 격차 해소**: SRAM 캐시로 메모리 월(Memory Wall) 완화.
- **예측 가능한 지연**: DRAM과 달리 리프레시로 인한 지연이 없음.

### 2. 미래 전망
- 3D 적층 SRAM, eSRAM(embedded SRAM) 기술로 밀도 개선 시도.
- 그러나 물리적 한계(6T 구조)로 인해 DRAM 이상의 용량화는 어려움.
