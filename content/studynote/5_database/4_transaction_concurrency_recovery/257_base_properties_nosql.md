# 257. BASE 속성 - NoSQL의 철학적 뿌리

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: BASE는 **Basically Available(기본적 가용성), Soft-state(유연한 상태), Eventually Consistent(결과적 일관성)**의 약자로, 전통적 RDBMS의 ACID와 정반대로 성능과 가용성을 위해 엄격한 일관성을 양보하는 NoSQL의 핵심 특성이다.
> 2. **가치**: 대규모 분산 환경에서 시스템의 일부가 고장 나더라도 전체 서비스는 중단되지 않게 하며, 기하급수적인 데이터 증가에도 선형적으로 확장(Scale-out)할 수 있는 이론적 토대를 제공한다.
> 3. **융합**: CAP 정리 중 AP(가용성+분단 허용성) 시스템의 철학을 구체화한 것이며, 현대의 웹 스케일(Web-scale) 서비스 아키텍처를 지탱하는 가장 실용적인 데이터 정합성 모델로 자리 잡았다.

+++

### Ⅰ. BASE의 3대 구성 요소

1. **Basically Available (기본적 가용성)**:
    - 어떤 상황에서도(심지어 네트워크 장애 시에도) 시스템은 항상 응답해야 합니다. "정답이 아니더라도, 응답은 준다"는 원칙입니다.
2. **Soft-state (유연한 상태)**:
    - 외부의 입력이 없더라도 데이터의 상태가 시간에 따라 변할 수 있습니다. (노드 간 동기화가 진행 중인 상태)
3. **Eventually Consistent (결과적 일관성)**:
    - 일시적으로는 데이터가 불일치할 수 있으나, 결국에는 모든 복제본이 일치하게 된다는 보장입니다.

```text

      ACID (RDBMS)                     BASE (NoSQL)
  ┌────────────────────┐           ┌────────────────────┐
  │  Strong Consistency│           │ Eventual Consist.  │

  └────────────────────┘           └────────────────────┘
           ▲                                │
           └───────── [ Trade-off ] ────────┘
             (정합성 우선) <-----> (성능/가용성 우선)
```

### Ⅲ. 실무적 활용

- **전자상거래**: "재고가 1개 남았습니다"라고 보여줬는데(Soft-state), 결제 시점에 재고가 없을 수 있음. 하지만 서비스는 멈추지 않고 돌아가야 함.
- **SNS**: 내가 쓴 글이 전 세계 팔로워에게 1초 뒤에 보임(Eventually Consistent). 하지만 서버는 전 세계에서 24시간 가동되어야 함(Basically Available).

- **📢 섹션 요약 비유**: BASE는 **'동네 반상회'**와 같습니다. 모두가 한자리에 모여서 결정하는 깐깐한 회의(ACID) 대신, 일단 결정된 사항을 집집마다 돌아다니며 알려주는 방식입니다. 집집마다 소식을 듣는 시간은 다르겠지만(Soft-state), 결국 밤이 되면 모든 주민이 같은 내용을 알게 되는(Eventually Consistent) 실무적인 소통법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[CAP 정리]**: BASE가 탄생하게 된 이론적 배경.
- **[NoSQL]**: BASE를 실천하는 저장소들의 집합 (Cassandra, MongoDB 등).
- **[확장성]**: BASE를 택함으로써 얻게 되는 최고의 선물.

📢 **마무리 요약**: **BASE**는 완벽함보다는 실용성을 택한 아키텍처입니다. 거대하고 빠른 데이터 세상을 만들기 위해 반드시 이해해야 할 현대의 데이터 철학입니다.
