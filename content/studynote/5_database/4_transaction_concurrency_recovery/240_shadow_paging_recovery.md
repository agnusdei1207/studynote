# 240. 그림자 페이징 (Shadow Paging) 기법 - 로그 없는 복구의 예술

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 그림자 페이징(Shadow Paging)은 트랜잭션 수행 중 원본 페이지 테이블(Shadow)은 보존하고, 복사본인 **현 페이지 테이블(Current)에만 변경 사항을 반영하다가 커밋 시점에 두 테이블을 교체하는 회복 기법**이다.
> 2. **가치**: 로그 파일(Redo/Undo)을 유지하고 관리하는 오버헤드가 없으며, 장애 발생 시 원본 '그림자' 테이블로 즉시 포인터만 돌리면 되므로 복구 속도가 극단적으로 빠르다.
> 3. **융합**: 현대의 **Copy-On-Write (COW)** 파일 시스템이나 스토리지 스냅샷(Snapshot) 기술의 모태가 되는 이론이며, 단순한 구조의 파일 기반 데이터베이스에서 강력한 힘을 발휘한다.

+++

### Ⅰ. 그림자 페이징의 작동 원리

- **정의**: 데이터베이스를 일정한 크기의 페이지 단위로 나누고, 이를 가리키는 두 개의 디렉토리(Current, Shadow)를 운영하는 방식입니다.
- **프로세스**:
    1. 트랜잭션 시작 시, **Shadow 디렉토리**를 복사하여 **Current 디렉토리**를 만듭니다.
    2. 변경이 필요한 페이지는 새로운 빈 블록에 쓰고, **Current 디렉토리**의 포인터만 바꿉니다. (원본 Shadow는 그대로 유지)
    3. **COMMIT**: Current 디렉토리를 디스크에 쓰고, 이를 새로운 Shadow로 승격시킵니다.
    4. **ABORT**: 그냥 Current 디렉토리를 버리고 Shadow 디렉토리를 계속 사용합니다.

+++

### Ⅱ. 그림자 페이징 시각화 (ASCII Structure)

```text
[그림자 페이징의 테이블 교체 메커니즘]

  [ Shadow Directory ] (Stable)      [ Current Directory ] (Working)
  ┌───┬────────────┐                 ┌───┬────────────┐
  │ 1 │  Block 10  │                 │ 1 │  Block 10  │
  │ 2 │  Block 20  │ ──▶ (Modify 2) ──▶ │ 2 │  Block 99* │ ◀── New Block!
  │ 3 │  Block 30  │                 │ 3 │  Block 30  │
  └───┴────────────┘                 └───┴────────────┘
          │                                   │
          └─────▶ [ Disk Blocks ] ◀────────────┘
                  [10][20][30]...[99*]
  
  * COMMIT 시: Current가 Shadow가 됨 (포인터 전환)
  * 장애 시 : Current 버리고 Shadow(10, 20, 30)로 즉시 복구 ✅
```

+++

### Ⅲ. 장단점 분석

- **장점**: 
    - 로그 기록 및 체크포인트 오버헤드가 없습니다.
    - 복구 시 로그를 분석할 필요가 없어 매우 빠릅니다.
- **단점**: 
    - 데이터 파편화(Fragmentation)가 심해집니다.
    - 대규모 트랜잭션 병행 수행 시 디렉토리 관리가 매우 복잡해집니다. (주로 단일 사용자 DB에 적합)

- **📢 섹션 요약 비유**: 그림자 페이징은 **'시험지 두 장 쓰기'**와 같습니다. 한 장은 완벽한 정답지(Shadow)로 보관하고, 다른 한 장(Current)에 낙서를 하며 문제를 풉니다. 다 풀고 확신이 서면 낙서한 종이를 정답지로 바꿔치기하고, 망치면 그냥 낙서장을 찢어버리고 정답지를 그대로 쓰는 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[포인터 전환]**: 커밋의 물리적 실체.
- **[No Log]**: 그림자 페이징의 아이덴티티.
- **[스냅샷]**: 현대 클라우드가 그림자 페이징을 계승한 방식.

📢 **마무리 요약**: **Shadow Paging**은 로그의 복잡함을 포인터의 논리로 치환한 우아한 기법입니다. 데이터의 불변성을 유지하며 원자성을 확보하는 가장 직관적인 모델입니다.
