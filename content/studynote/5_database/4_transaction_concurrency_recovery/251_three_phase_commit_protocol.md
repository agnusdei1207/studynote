# 251. 3단계 커밋 (3PC, Three-Phase Commit) - 블로킹의 해법

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 3단계 커밋(3PC)은 2단계 커밋(2PC)의 고질적 문제인 '코디네이터 장애 시 참여자 무한 대기(Blocking)'를 해결하기 위해, **'예비 커밋(Pre-Commit)' 단계를 중간에 추가한 비차단 분산 합의 프로토콜**이다.
> 2. **가치**: 코디네이터가 준비 완료 후 갑자기 죽더라도, 참여자들이 'Pre-Commit' 상태를 공유하고 있다면 스스로 커밋을 결정할 수 있는 타임아웃 매커니즘을 제공한다.
> 3. **융합**: 고가용성 네트워크 환경에서 데이터 유실 없는 자율 복구 시스템의 기초가 되나, 메시지 전송 횟수가 늘어나 지연 시간(Latency)이 길어지는 단점이 있어 실무에서는 Raft나 Paxos로 대체되기도 한다.

+++

### Ⅰ. 3PC의 3단계 프로세스

2PC의 단일 통제 구조를 보완하기 위해 '확신'의 단계를 하나 더 둡니다.

1. **Can-Commit (질의 단계)**: 코디네이터가 참여자들에게 트랜잭션 수행이 가능한지 묻고 응답을 받습니다. (2PC의 Prepare와 유사)
2. **Pre-Commit (예비 커밋)**: 모든 참여자가 동의하면, "곧 커밋할 테니 준비해!"라고 알립니다. 참여자들은 락을 걸고 연산을 준비합니다.
3. **Do-Commit (최종 커밋)**: 모든 참여자가 Pre-Commit ACK를 보내면 최종적으로 커밋을 지시합니다.

+++

### Ⅱ. 3PC 아키텍처 및 타임아웃 (ASCII Flow)

```text
[3PC의 비차단 복구 매커니즘]

  [ Coordinator ]           [ Node A ]           [ Node B ]
        │                      │                    │
        │ ── 1. CanCommit? ──▶ │                    │
        │ ◀───  YES  ───────── │                    │
        │                      │                    │
        │ ── 2. PreCommit! ──▶ │                    │
        │ ◀───  ACK  ───────── │                    │
        │                      │                    │
        │ ── 3. DoCommit!  ──▶ │                    │
        ▼                      ✅                   ✅
        
  * 핵심: Node A와 B가 'PreCommit'을 받은 상태에서 
    Coordinator가 죽으면, 참여자들끼리 "우리 다 PreCommit 받았지? 
    그럼 그냥 커밋하자!"라고 자율 결정 가능. ──▶ 🔓 UNBLOCK!
```

+++

### Ⅲ. 2PC vs 3PC 비교

| 비교 항목 | 2단계 커밋 (2PC) | 3단계 커밋 (3PC) |
|:---|:---|:---|
| **단계 수** | 2단계 (Prepare, Commit) | 3단계 (Can, Pre, Do) |
| **블로킹 여부** | **Blocking 발생** (코디네이터 의존) | **Non-Blocking** (타임아웃 처리) |
| **성능 (속도)** | 빠름 (메시지 2회) | 느림 (메시지 3회) |
| **구현 복잡도** | 상대적으로 낮음 | 매우 높음 |

- **📢 섹션 요약 비유**: 3PC는 **'결혼식의 예행연습'**과 같습니다. 2PC가 바로 결혼식(Prepare->Commit)이라면, 3PC는 결혼식 전날 리허설(Pre-Commit)을 한 번 더 하는 것입니다. 리허설 때 전원 참석을 확인했으니, 당일 아침에 주례 선생님이 늦으시더라도(장애) 신랑 신부가 "어제 다 약속했으니까 그냥 진행하자!"라고 결정할 수 있는 안전장치입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Pre-Commit]**: 3PC의 존재 의의.
- **[Network Partition]**: 3PC조차 완벽히 해결하지 못하는 분산 시스템의 난제.
- **[Non-blocking]**: 가용성을 높이는 핵심 키워드.

📢 **마무리 요약**: **3PC**는 이론적으로 완벽한 탈출구를 제공합니다. 동기식 조율의 한계를 극복하려는 아키텍트들의 끊임없는 노력이 담긴 산물입니다.
