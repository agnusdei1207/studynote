# 148. Grouping Sets / Rollup / Cube - 다차원 집계 연산

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단일 쿼리 내에서 다양한 차원의 소계(Subtotal)와 총계(Grand Total)를 한 번에 산출하기 위한 확장된 **GROUP BY** 절의 고급 기능이다.
> 2. **계층적 분화**: 계층 구조에 따른 부분합을 구하는 **ROLLUP**, 발생 가능한 모든 차원 조합의 집계를 생성하는 **CUBE**, 그리고 사용자 정의에 따른 특정 그룹 집합만을 선별하는 **GROUPING SETS**로 나뉜다.
> 3. **가치**: 복잡한 UNION ALL 쿼리를 획기적으로 단순화하여 가독성을 높이며, DBMS 내부의 한 번의 스캔만으로 다차원 분석을 완료하여 성능 최적화를 실현한다.

+++

## Ⅰ. 개요 (Context & Background)

### 1. 개념 정의
- **ROLLUP**: 인자의 순서에 따라 계층적인 소계를 생성합니다. (상위 차원부터 단계별 집계)
- **CUBE**: 결합 가능한 모든 차원의 조합에 대해 집계를 수행합니다.
- **GROUPING SETS**: 사용자가 명시한 특정 컬럼 조합들만 골라서 집계를 수행합니다.

### 2. 💡 비유: 마트 영수증의 '항목별 합계'
```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        다차원 집계 연산 비유                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 여러 종류의 과자와 음료수를 샀음.                                        │
│                                                                             │
│  1. ROLLUP (영수증 스타일):                                                   │
│     - [초코파이 합계] -> [과자 전체 합계] -> [영수증 총합계].                     │
│     - 위에서 아래로 순서대로 소계를 착착 쌓아 내려감.                             │
│                                                                             │
│  2. CUBE (통계 전문가 스타일):                                                 │
│     - [과자 합계], [A사 제품 합계], [빨간 포장지 합계]...                         │
│     - 생각할 수 있는 모든 조건으로 다 더해보고 비교함.                            │
│                                                                             │
│  → 복잡한 계산기를 여러 번 두드리지 않고, **'버튼 하나로 소계와 총계를 몽땅'**      │
│     뽑아내는 마법의 정산기입니다!                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 등장 배경
- **의사결정 지원 시스템 (DSS)**: 단순 명세 조회가 아닌, "지역별 소계", "월별 총계" 등 다각도의 요약 정보를 실시간으로 요구함.
- **코드 중복의 한계**: 소계를 위해 `GROUP BY 지역` + `UNION ALL` + `GROUP BY 전체` 처럼 장황하게 쓰던 방식을 개선하여 유지보수성 확보.

- **📢 섹션 요약 비유**: ROLLUP은 '계단을 한 칸씩 올라가며 합산하는 것'이고, CUBE는 '모든 방향에서 사진을 찍어 합산하는 것'입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 3대 다차원 집계 연산자 상세 비교

| 연산자 | 집계 범위 | 결과 행 수 | 특징 |
|:---:|:---|:---|:---|
| **ROLLUP(A, B)** | (A, B), (A), (전체) | $N+1$ 레벨 | **순서가 중요함**. 계층적 소계. |
| **CUBE(A, B)** | (A, B), (A), (B), (전체) | $2^N$ 조합 | 모든 경우의 수. 데이터 폭발 주의. |
| **GROUPING SETS(A, B)** | (A), (B) | 지정된 만큼 | 소계나 총계 없이 딱 정한 것만 조회. |

### 2. ROLLUP vs CUBE 결과 집합 구조 도해 (ASCII)

두 컬럼(지역, 상품)에 대한 집계 결과 차이를 시각화한다.

```text
    [ Raw Data ] : {서울, 폰}, {서울, 폰}, {부산, TV}
    
    (1) GROUP BY ROLLUP(지역, 상품)
    서울 | 폰 | 2
    서울 | ALL| 2  <-- 지역 소계
    부산 | TV | 1
    부산 | ALL| 1  <-- 지역 소계
    ALL  | ALL| 3  <-- 총합계
    
    (2) GROUP BY CUBE(지역, 상품)
    (위의 ROLLUP 결과 포함) +
    ALL  | 폰 | 2  <-- 상품 소계 (지역 무시)
    ALL  | TV | 1  <-- 상품 소계 (지역 무시)
```

### 3. 심층 동작 원리: GROUPING 함수
- **개념**: 해당 행이 소계/총계 행인지(1) 아니면 일반 행인지(0) 판별해주는 함수.
- **용도**: 'ALL'이나 '전체'라는 텍스트를 소계 행에 예쁘게 출력하기 위해 사용.

### 4. 핵심 기술: 데이터 중복 스캔 방지 (Single Scan)
- DBMS 옵티마이저는 이 연산자들을 만나면 내부적으로 데이터를 한 번만 읽고(Full Scan), 메모리 내에서 여러 레벨의 집계를 동시에 수행하여 I/O 효율을 극대화함.

- **📢 섹션 요약 비유**: GROUPING SETS는 '뷔페에서 먹고 싶은 음식(집계 차원)만 골라 담는 것'과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. UNION ALL 방식 vs 다차원 집계 연산자

| 비교 항목 | UNION ALL (전통적) | ROLLUP / CUBE (고급) |
|:---:|:---|:---|
| **가독성** | 매우 낮음 (중복 코드) | **매우 높음 (간결)** |
| **I/O 부하** | **테이블을 N번 읽음** | **단 한 번만 읽음 (Single Pass)** |
| **실수 위험** | 매핑 누락 가능성 큼 | 함수 하나로 완벽 보장 |
| **비유** | 계산기 N번 두드리기 | 자동 정산 프로그램 실행 |

### 2. 과목 융합 관점: 통계 및 데이터 웨어하우스
- **통계학 융합**: 다차원 집계는 데이터의 **교차 분석(Cross Tabulation)**을 수행하는 것이며, 각 차원 간의 상관 관계를 한눈에 파악하는 기술적 토대가 된다.
- **DW 융합**: ROLLUP과 CUBE는 데이터 큐브(Data Cube)의 **슬라이싱(Slicing)과 다이싱(Dicing)**을 SQL 환경에서 가장 가볍게 구현한 형태다.

- **📢 섹션 요약 비유**: UNION ALL 이 '각개전투'라면 다차원 집계 연산자는 '일사불란한 군대'입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 1. 실무 시나리오: 일별/월별/연별 통합 매출 리포트
- **상황**: 하나의 쿼리로 '매일의 합계', '매월의 소계', '올해의 총계'를 다 보여줘야 함.
- **판단**: 클라이언트 앱에서 합산하기엔 데이터 양이 너무 많고 비효율적임.
- **결단**: **ROLLUP(Year, Month, Day) 적용**. 계층 구조에 따른 소계를 자동으로 생성하고, `GROUPING` 함수로 "월계", "연계"라는 라벨을 달아 사용자 친화적인 통합 보고서 완성.

### 2. 안티패턴: 수십 개 컬럼에 대한 CUBE 사용
- **증상**: 분석 차원이 10개인데 `CUBE(A, B, C, D, E, ...)`를 사용.
- **결과**: 결과 행 수가 $2^{10} = 1,024$배로 늘어나며 메모리 부족으로 쿼리 중단.
- **조치**: **반드시 필요한 차원 조합만 GROUPING SETS**로 묶어서 요청하거나, 계층 구조가 뚜렷한 경우 ROLLUP으로 범위를 제한할 것.

### 📢 기술사적 결언
> "다차원 집계 연산자는 데이터베이스를 **'정보의 공장'**으로 만든다. 원재료(Raw Data)를 집어넣으면 완제품(요약 리포트)이 튀어나오기 때문이다. 기술사는 단순한 함수 호출을 넘어, 이 연산이 내부적으로 **정렬(Sort)이나 해시(Hash)** 연산을 얼마나 유발하는지 통찰해야 한다. 특히 대용량 환경에서는 집계 연산의 순서를 조정하여 메모리 내에서 연산을 끝내는 **'인메모리 최적화'** 능력이 전체 분석 시스템의 성패를 가른다."

- **📢 섹션 요약 비유**: CUBE를 잘못 쓰는 것은 '전화번호부의 모든 사람 조합을 출력하라는 것'과 같아, 종이(자원) 낭비만 엄청나게 유발할 수 있습니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 1. 정량/정성 기대효과 (다차원 집계 활용 시)

| 지표 | 도입 전 | 도입 후 | 개선 효과 |
|:---|:---:|:---:|:---:|
| **보고서 생성 속도** | 수 분 (반복 스캔) | **수 초 (단일 스캔)** | 성능 10배 이상 향상 |
| **SQL 코드량** | 200줄 이상 | **20줄 이내** | 유지보수 비용 90% 절감 |
| **데이터 리터러시** | 단편적 조회 | **입체적 분석 가능** | 의사결정 통찰력 확대 |

### 2. 미래 전망
- **AI-Auto Grouping**: 사용자가 어떤 차원의 소계를 주로 보는지 AI가 분석하여, 미리 인메모리에 집계 결과를 캐싱해두는 **지능형 소계 캐시** 기술 고도화.
- **Streaming Aggregation**: 데이터가 들어오는 즉시 윈도우 기반으로 다차원 집계를 수행하여 대시보드에 소계를 실시간으로 갱신하는 기술 보편화.

- **📢 섹션 요약 비유**: 미래의 다차원 집계는 우리가 고민하지 않아도, 데이터들 스스로가 '가장 의미 있는 소계'를 찾아서 우리 앞에 먼저 보여주는 인공지능 정산사가 될 것입니다.
