# 147. PIVOT / UNPIVOT - 행과 열의 회전 및 변환

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터베이스 내 저장된 행(Row) 데이터를 열(Column)로 전환하거나(**PIVOT**), 반대로 열 데이터를 행 데이터로 펼치는(**UNPIVOT**) 데이터 구조 변환 연산이다.
> 2. **변환 로직**: **PIVOT**은 특정 속성값을 새로운 컬럼 헤더로 삼아 집계 연산을 수행하여 교차 표(Crosstab)를 생성하며, **UNPIVOT**은 비정규화된 여러 속성을 하나의 속성으로 통합하여 정형화된 분석용 데이터를 만든다.
> 3. **가치**: 보고서 작성을 위한 '가로형 보기'와 기계 학습/분석을 위한 '세로형 적재' 사이를 자유롭게 넘나들게 하며, 복잡한 CASE 문 없이도 간결하게 데이터 포맷을 최적화할 수 있다.

---

## Ⅰ. 개요 (Context & Background)

### 1. 개념 정의
- **PIVOT (회전)**: 행(Row)으로 나열된 데이터를 열(Column)로 회전시켜 요약 보고서 형태로 만드는 것. (Long $\to$ Wide)
- **UNPIVOT (역회전)**: 여러 열(Column)에 흩어진 데이터를 하나의 열(Row)로 모아서 길게 펼치는 것. (Wide $\to$ Long)

### 2. 💡 비유: 사진첩 레이아웃 변경
```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        PIVOT / UNPIVOT 비유                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. PIVOT (슬라이드 쇼):                                                      │
│     - 한 장씩 넘겨보던 사진(Row)들을 벽에 한꺼번에 쫙 펼쳐놓고 보는 것.           │
│     - "누가, 언제, 어디서"를 표의 가로세로 축으로 만들어 한눈에 파악함.           │
│                                                                             │
│  2. UNPIVOT (앨범 정리):                                                      │
│     - 벽에 붙어있던 여러 사진(Column)을 떼어내어 한 권의 앨범에 순서대로 끼움.      │
│     - 낱개로 분석하거나 보관하기 좋게 한 줄로 줄 세우는 것.                       │
│                                                                             │
│  → 데이터를 **'가로로 넓게 볼지'** 아니면 **'세로로 길게 쌓을지'** 결정하는         │
│     유연한 데이터 변신술입니다!                                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 등장 배경
- **비즈니스 리포팅 요구**: 엑셀의 피벗 테이블처럼 "월별/제품별 매출"을 가로로 길게 보고 싶은 사용자의 요구 급증.
- **분석 전처리**: 비정규화된 테이블(`Sales_Q1`, `Sales_Q2` 등)을 분석 도구에 넣기 위해 하나의 `Sales_Amount` 컬럼으로 통합해야 할 필요성.

- **📢 섹션 요약 비유**: PIVOT은 '전시회용 액자'를 만드는 것이고, UNPIVOT은 '보관용 박스'에 차곡차곡 담는 것입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. PIVOT 작동 원리 및 구문 (Oracle/SQL Server 기준)

```sql
SELECT * FROM (대상_쿼리)
PIVOT (집계함수(컬럼) FOR 회전할_컬럼 IN (값1, 값2, ...));
```

| 구성 요소 | 상세 역할 | 비유 |
|:---:|:---|:---|
| **집계 함수** | 새로운 열에 들어갈 숫자 연산 (SUM, AVG 등). | 바구니에 담긴 물건 세기 |
| **FOR 컬럼** | 어떤 속성값을 새로운 컬럼 제목으로 쓸지 결정. | 새 이름표 달기 |
| **IN 리스트** | 새로운 컬럼으로 만들 실제 값들의 목록. | 칸막이 나누기 |

### 2. PIVOT vs UNPIVOT 데이터 구조 변환 도해 (ASCII)

데이터의 방향성이 어떻게 바뀌는지 시각화한다.

```text
    [ PIVOT: Long to Wide ]
    Name | Year | Sales         Name | 2023 | 2024
    -----|------|------         -----|------|------
    Kim  | 2023 | 100   ──▶     Kim  | 100  | 150
    Kim  | 2024 | 150           Lee  | 200  | 250
    Lee  | 2023 | 200
    
    [ UNPIVOT: Wide to Long ]
    ID | Math | Eng             ID | Subject | Score
    ---|------|-----            ---|---------|------
    01 |  90  |  80     ──▶     01 | Math    |  90
                                01 | Eng     |  80
```

### 3. 심층 동작 원리: CASE 문과의 상관관계
- **전통적 방식**: PIVOT이 지원되지 않는 DB에서는 `SUM(CASE WHEN Year=2023 THEN Sales END)` 처럼 노가다성 쿼리를 짰음.
- **현대적 방식**: PIVOT 연산자는 내부적으로 위와 같은 CASE 문을 최적화된 형태로 자동 생성하여 실행함.

### 4. 핵심 기술: 동적 피벗 (Dynamic PIVOT)
- `IN` 절에 들어갈 값(예: 2023, 2024...)이 계속 늘어날 때, 이를 자동으로 감지하여 컬럼을 생성하는 기술 (주로 프로시저나 클라이언트 언어와 결합).

- **📢 섹션 요약 비유**: UNPIVOT은 '가로로 누워 자는 데이터들을 깨워서 세로로 줄 세우는 기상 나팔'과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. PIVOT vs UNPIVOT 전략적 선택

| 비교 항목 | PIVOT | UNPIVOT |
|:---:|:---|:---|
| **주요 사용자** | **경영진, 기획자 (보고서용)** | **데이터 분석가, 머신러닝 (전처리용)** |
| **행/열 변화** | Row $\to$ Column | **Column $\to$ Row** |
| **데이터 압축** | 수치 집계로 행 수 감소 | 데이터 전개로 행 수 증가 |
| **비유** | **망원경 (멀리서 요약)** | **현미경 (낱낱이 분해)** |

### 2. 과목 융합 관점: 행렬 연산 및 소프트웨어 공학
- **수학(행렬) 융합**: PIVOT은 선형대수의 **전치 행렬(Transpose Matrix)** 연산에 집계(Aggregation)가 추가된 형태의 연산이다.
- **소프트웨어 공학(SE) 융합**: UI 컴포넌트 중 **그리드(Grid) 컨트롤**의 가로/세로 축 데이터를 바인딩할 때, 백엔드에서 PIVOT 연산을 통해 데이터를 전송하면 프론트엔드의 부하를 획기적으로 줄일 수 있다.

- **📢 섹션 요약 비유**: PIVOT이 '결과 보고서'라면, UNPIVOT은 '데이터 원천 회복'입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 1. 실무 시나리오: 월간 프로젝트 투입 인력 현황판
- **상황**: DB에는 [사원, 월, 시간]이 세로로 쌓여 있는데, 사용자는 [사원]을 행으로, [1월, 2월, 3월...]을 열로 보고 싶어 함.
- **판단**: 12번의 조인을 거는 것은 쿼리 작성이 너무 고통스럽고 성능도 안 좋음.
- **결단**: **PIVOT 연산자 사용**. `FOR Month IN (1, 2, 3...)`을 사용하여 단 한 번의 스캔으로 가로형 현황판을 완성. 쿼리 라인 수를 1/5로 줄여 유지보수 효율 증대.

### 2. 안티패턴: UNPIVOT을 통한 억지 데이터 정형화
- **증상**: 이미 잘 설계된 1NF 테이블을 특정 도구의 요구사항에 맞추려고 억지로 UNPIVOT 하여 기하급수적인 행 생성.
- **결과**: 인덱스 효율이 사라지고 조인 성능이 바닥을 치는 '데이터 폭발' 발생.
- **조치**: **필요한 시점에만 뷰(View)를 통해 UNPIVOT** 하고, 원본 저장소의 물리적 구조는 정규화 원칙을 고수할 것.

### 📢 기술사적 결언
> "PIVOT과 UNPIVOT은 데이터의 **'가변적 시각'**을 제공하는 자유로운 기술이다. 데이터는 담는 그릇에 따라 물처럼 변해야 하기 때문이다. 기술사는 단순 문법 사용을 넘어, **데이터의 카디널리티(열로 바뀔 값의 종류)**가 성능에 미치는 파급력을 계산해야 한다. 너무 많은 컬럼 생성(Wide Column)은 메모리 단편화를 유발하므로, **집계의 깊이와 차원의 폭**을 비즈니스 가치에 비추어 정교하게 설계해야 한다."

- **📢 섹션 요약 비유**: 무분별한 PIVOT은 '한 장의 종이에 전 세계 지도를 다 그리려는 것'과 같아, 종이가 찢어지거나(에러) 아무것도 안 보일 수(성능 저하) 있습니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 1. 정량/정성 기대효과 (회전 연산 최적 활용 시)

| 지표 | 도입 전 (CASE 문 방식) | 도입 후 (PIVOT) | 개선 효과 |
|:---|:---:|:---:|:---:|
| **SQL 작성 시간** | 1시간 (오타 위험) | **10분 (정형화)** | 개발 속도 6배 향상 |
| **쿼리 실행 효율** | 반복 연산 발생 | **내부 엔진 최적화** | 응답 속도 20~30% 개선 |
| **비즈니스 가독성** | 난해한 수식 | **직관적 행/열 변환** | 요구사항 반영 정확도 상승 |

### 2. 미래 전망
- **Schema-Agnostic Pivoting**: JSON 등 비정형 데이터의 키(Key)를 분석하여, 사용자가 정의하지 않아도 자동으로 컬럼을 생성해주는 **지능형 자동 피벗** 기술 발전.
- **In-memory Pivot Acceleration**: 대규모 피벗 연산을 CPU가 아닌 **GPU의 병렬 행렬 연산**으로 처리하여 초대용량 데이터를 실시간으로 회전시키는 기술 확산.

- **📢 섹션 요약 비유**: 미래의 데이터 회전은 단순히 가로세로를 바꾸는 것을 넘어, 우리의 질문에 따라 데이터가 스스로 최적의 입체 구조(OLAP Cube)로 재배열되는 지능형 변신을 보여줄 것입니다.
