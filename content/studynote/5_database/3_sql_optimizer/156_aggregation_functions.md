# 156. 집계 함수 (Aggregate Function) - 데이터 요약 및 그룹 연산

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다수의 행(Row) 데이터를 입력받아 하나의 요약된 결과값(Single Value)을 산출하는 함수로, 개별 데이터의 상세함보다는 집합 전체의 통계적 특성을 파악하는 데 목적이 있다.
> 2. **작동 원리**: **`GROUP BY`** 절과 결합하여 특정 그룹 단위로 집계를 수행하며, `NULL` 값을 연산에서 자동 제외하는 특성과 중복을 제거하고 계산하는 `DISTINCT` 옵션 등을 제공한다.
> 3. **가치**: 방대한 로우(Raw) 데이터를 비즈니스 의사결정에 필요한 유의미한 정보(KPI, 통계 리포트)로 변환하며, 인덱스 및 병렬 처리를 통해 대규모 데이터셋에 대한 고속 요약을 실현한다.

+++

## Ⅰ. 개요 (Context & Background)

### 1. 개념 정의
**집계 함수 (Aggregate Function)**는 여러 행의 값을 하나의 값으로 합치는 함수입니다. 대표적으로 합계(`SUM`), 평균(`AVG`), 개수(`COUNT`), 최대값(`MAX`), 최소값(`MIN`) 등이 있으며, 주로 보고서 작성이나 통계 분석에 사용됩니다.

### 2. 💡 비유: 전교생 시험 성적 정산하기
```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        집계 함수 개념 비유                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 시험이 끝나고 선생님이 성적표 뭉치를 들고 계심.                           │
│                                                                             │
│  1. COUNT (인원수): "오늘 시험 본 학생이 총 몇 명이지?" (성적표 장수 세기)         │
│  2. SUM (총점): "전교생 점수를 다 합하면 얼마지?" (모든 점수 더하기)               │
│  3. AVG (평균): "우리 학교 실력이 어느 정도지?" (총점 / 인원수)                   │
│  4. MAX/MIN (최고/최저): "1등 점수랑 꼴등 점수가 몇 점이지?"                     │
│                                                                             │
│  → 수많은 학생의 개별 성적(Data)을 **'학교 전체의 실력(Information)'**으로         │
│     요약해주는 마법의 계산기입니다!                                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 등장 배경
- **데이터 분석의 필요성**: 수만 건의 거래 내역을 하나씩 보는 것은 불가능하므로, "이번 달 총 매출"과 같은 요약 수치가 경영에 필수적임.
- **SQL의 연산 기능 확장**: 단순 조회 기능을 넘어 데이터베이스 엔진 내에서 직접 통계 연산을 수행하여 애플리케이션의 부하를 줄이기 위함.

- **📢 섹션 요약 비유**: 집계 함수는 '흩어진 구슬들을 모아서 하나의 무게를 재는 저울'과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 주요 집계 함수의 특성 및 동작 방식

| 함수 | 대상 데이터 타입 | NULL 처리 방식 | 주요 비유 |
|:---:|:---|:---|:---|
| **COUNT(*)** | 모든 행 | **NULL 포함**하여 행 수 계산. | 출석부 체크 |
| **COUNT(컬럼)** | 특정 컬럼 | **NULL 제외**한 유효 값 계산. | 숙제 제출자 확인 |
| **SUM / AVG** | 숫자형 | NULL 제외하고 연산. | 용돈 합산 |
| **MAX / MIN** | 숫자, 문자, 날짜 | NULL 제외하고 비교. | 키재기 / 가나다순 |

### 2. GROUP BY 와 집계 함수의 협업 도해 (ASCII)

데이터가 그룹으로 묶이고 집계되는 물리적 과정을 시각화한다.

```text
    [ Raw Data ]                [ Grouping : 부서별 ]         [ Aggregation : SUM ]
    ┌────┬──────┬──────┐        ┌─────────────────────┐      ┌────────┬──────────┐
    │이름 │부서  │급여  │        │ Sales 부서 그룹      │      │ 부서   │ 급여합계 │
    ├────┼──────┼──────┤        │ - Kim (500)         │ ──▶  ├────────┼──────────┤
    │Kim │Sales │ 500  │ ──▶    │ - Lee (400)         │      │ Sales  │   900    │
    │Lee │Sales │ 400  │        └─────────────────────┘      ├────────┼──────────┤
    │Park│ IT   │ 700  │ ──▶    ┌─────────────────────┐      │ IT     │   700    │
    └────┴──────┴──────┘        │ IT 부서 그룹         │ ──▶  └────────┴──────────┘
                                │ - Park (700)        │
                                └─────────────────────┘
```

### 3. 심층 동작 원리: 해시 집계 (Hash Aggregation)
- **원리**: `GROUP BY` 절의 컬럼을 해시 키(Hash Key)로 사용하여 메모리에 해시 테이블을 빌드함.
- **작동**: 데이터를 읽으면서 해당 해시 버킷에 집계 값(`SUM` 등)을 실시간으로 누적함.
- **성능**: 정렬(Sort)이 필요 없는 방식이라 대용량 데이터 그룹화에 매우 효율적임.

### 4. 핵심 기술: HAVING 절을 통한 집계 결과 필터링
- `WHERE` 절은 집계 전(개별 행)을 거르고, `HAVING` 절은 집계 후(그룹 결과)를 거르는 선후 관계를 가짐.

- **📢 섹션 요약 비유**: GROUP BY가 '동전 분류기'라면, 집계 함수는 각 통에 담긴 '동전의 총합을 계산하는 장치'입니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 단일 행 함수 vs 집계 함수

| 비교 항목 | 단일 행 함수 (Scalar) | 집계 함수 (Aggregate) |
|:---:|:---|:---|
| **입력 행 수** | 1행 | **N행 (다중 행)** |
| **출력 행 수** | 1행 (1:1 대응) | **1행 (N:1 압축)** |
| **위치** | SELECT, WHERE, ORDER BY | SELECT, HAVING |
| **비유** | 개별 학생의 이름 변경 | **우리 반 전체의 평균** |

### 2. 과목 융합 관점: 통계학 및 분산 처리
- **통계학 융합**: 집계 함수는 기술 통계학(Descriptive Statistics)의 기본 지표인 **중심 경향성(Mean)**과 **변동성**을 측정하는 도구이며, `STDDEV`, `VARIANCE`와 같은 표준 편차 함수로 확장된다.
- **분산 처리 융합**: 대규모 분산 DB(MapReduce 등)에서는 각 노드에서 부분 집계를 먼저 수행(Local Agg)하고 마스터가 이를 다시 합치는 **두 단계 집계(Two-stage Aggregation)** 아키텍처를 통해 네트워크 비용을 최소화한다.

- **📢 섹션 요약 비유**: 단일 행 함수가 '각자의 키를 10cm씩 늘리는 구두'라면, 집계 함수는 '모두의 키를 합쳐서 건물을 세우는 설계'입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 1. 실무 시나리오: 일일 결산 및 마감 프로세스
- **상황**: 매일 밤 12시에 그날 발생한 수백만 건의 결제 내역을 정산해야 함.
- **판단**: 애플리케이션에서 한 건씩 읽어서 합산하는 것은 시간이 너무 오래 걸리고 정합성 위험이 있음.
- **결단**: **DB 엔진 내 집계 함수 활용**. `SUM`과 `COUNT`를 사용하여 엔진 레벨의 병렬 스캔을 유도하고, 결과를 `Materialized View`에 저장하여 아침 보고서 출력 시간을 10분에서 10초로 단축.

### 2. 안티패턴: COUNT(*)와 COUNT(Column)의 혼용 실수
- **증상**: 게시판 댓글 수 표시할 때 작성자가 탈퇴한(NULL) 댓글을 제외하고 세야 하는데 `COUNT(*)`를 사용.
- **결과**: 유령 댓글까지 포함된 엉뚱한 수치가 사용자에게 노출됨.
- **조치**: **비즈니스 목적에 따른 COUNT 방식 결정**. 행 수 자체가 중요하다면 `COUNT(*)`, 유효한 값의 개수가 중요하다면 `COUNT(컬럼명)`를 명확히 구분하여 쓸 것.

### 📢 기술사적 결언
> "집계 함수는 데이터베이스를 **'지능형 분석 엔진'**으로 격상시킨다. 단순 저장소를 넘어 데이터의 의미적 요약을 스스로 수행하기 때문이다. 기술사는 집계 함수의 문법을 넘어, 이것이 유발하는 **메모리(PGA) 부하와 Temp DB 사용량**을 상시 모니터링해야 한다. 특히 **중복 제거(`DISTINCT`)와 집계**를 동시에 수행할 때 발생하는 성능 저하를 인지하고, 이를 적절한 인덱스 전략으로 방어하는 '성능 아키텍트'의 면모를 보여야 한다."

- **📢 섹션 요약 비유**: 잘못된 집계는 '가짜 뉴스를 요약하는 인공지능'과 같아, 결과가 빨라도 내용이 틀리면 아무짝에도 쓸모가 없습니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 1. 정량/정성 기대효과 (집계 함수 최적 활용 시)

| 지표 | 도입 전 (Manual) | 도입 후 (Aggregate) | 개선 효과 |
|:---|:---:|:---:|:---:|
| **보고서 생성 시간** | 수 시간 | **수 초** | 분석 생산성 100배 향상 |
| **데이터 전송량** | 수백 MB (전체 데이터) | **몇 KB (요약값)** | 네트워크 부하 99% 절감 |
| **계산 정확도** | 사람의 실수 상존 | **엔진 레벨의 검증** | 데이터 신뢰도 극대화 |

### 2. 미래 전망
- **Incremental Aggregation**: 전체를 다시 계산하지 않고, 새로 들어온 데이터(Delta)만 기존 집계에 합치는 **증분 집계** 기술 고도화.
- **Approximate Aggregation**: 100% 정확도 대신 99%의 정확도를 보장하며 속도를 수만 배 높이는 **HyperLogLog 기반의 근사 집계** 기술이 빅데이터 표준으로 안착.

- **📢 섹션 요약 비유**: 미래의 집계 함수는 단순히 더하고 나누는 것을 넘어, 데이터의 흐름 속에서 우리가 물어보기도 전에 '오늘의 한 줄 요약'을 내놓는 예언자가 될 것입니다.
