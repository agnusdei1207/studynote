# 162. 정규 표현식 (Regular Expression) - SQL에서의 고급 패턴 매칭

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단순한 와일드카드(`LIKE %`)의 한계를 넘어, 복잡한 문자열 패턴을 메타 문자(Meta-characters) 조합으로 정의하여 검색, 추출, 치환하는 **강력한 데이터 정제 기술**이다.
> 2. **표준 기능**: 이메일 형식 검증, 전화번호 패턴 추출, 불필요한 특수문자 일괄 제거 등 비정형 텍스트 데이터에 대한 **정교한 필터링**을 SQL 문장 내에서 직접 수행할 수 있게 한다.
> 3. **가치**: 애플리케이션의 복잡한 문자열 처리 로직을 데이터베이스 레벨로 이관하여 개발 생산성을 높이고, 데이터 품질 검수(Data Profiling) 단계에서의 정확도를 획기적으로 개선한다.

---

## Ⅰ. 개요 (Context & Background)

### 1. 개념 정의
**정규 표현식 (Regular Expression, Regexp)**은 특정한 규칙을 가진 문자열의 집합을 표현하는 데 사용하는 형식 언어입니다. SQL에서는 `REGEXP_LIKE`, `REGEXP_SUBSTR`, `REGEXP_REPLACE` 등의 함수를 통해 이를 지원합니다.

### 2. 💡 비유: 마법의 '돋보기'와 '핀셋'
```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        정규 표현식 (Regexp) 비유                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. LIKE 연산자 (일반 돋보기):                                                │
│     - "김씨로 시작하는 사람 다 찾아!" (김%)                                     │
│     - 단순한 시작과 끝만 볼 수 있는 일반적인 도구입니다.                          │
│                                                                             │
│  2. 정규 표현식 (마법의 핀셋):                                                │
│     - "이메일 주소 중에서 @ 뒤에 'naver'가 들어있고 끝이 '.com'인 것만 뽑아!"      │
│     - "숫자가 3개-4개-4개로 이어지는 전화번호 형태만 골라내!"                     │
│                                                                             │
│  → 아주 까다롭고 복잡한 규칙도 **'단 한 줄의 기호(패턴)'**로 정확히 찝어내는         │
│     데이터의 정밀 타격 도구입니다!                                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 등장 배경
- **비정형 데이터의 폭증**: 소셜 미디어 게시글, 로그 파일 등 규칙이 일정하지 않은 텍스트 데이터를 DB에서 정제해야 할 필요성 증대.
- **LIKE의 한계**: `LIKE`만으로는 "숫자로만 구성된 값"이나 "특정 문자가 3번 반복되는 값" 등을 찾기가 극도로 어렵고 쿼리가 지저분해짐.

- **📢 섹션 요약 비유**: 정규 표현식은 '데이터라는 모래사장에서 바늘(패턴)을 찾아내는 강력한 자석'과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 정규 표현식의 주요 메타 문자 (Meta-characters)

| 기호 | 의미 | 예시 |
|:---:|:---|:---|
| **.** | 임의의 한 문자. | `a.c` $\to$ abc, a1c, acc |
| **^ / $** | 문자열의 시작 / 끝. | `^abc` $\to$ abc로 시작하는 행 |
| **\d / \D** | 숫자(Digit) / 숫자가 아닌 것. | `\d{3}` $\to$ 숫자 세 자리 |
| **[ ]** | 대괄호 안의 문자 중 하나. | `[abc]` $\to$ a 또는 b 또는 c |
| **{n, m}** | 앞의 패턴이 n번 이상 m번 이하 반복. | `a{2,3}` $\to$ aa 또는 aaa |
| **\|** | 선택(OR) 연산. | `apple\|banana` |

### 2. SQL 정규 표현식 함수 작동 도해 (ASCII)

입력된 문자열이 패턴 엔진을 거쳐 가공되는 과정을 시각화한다.

```text
    [ Raw String ] : "My phone is 010-1234-5678."
           │
    ┌──────▼──────────────────────────────────────────────────────────┐
    │  [ Pattern Engine : '\d{3}-\d{4}-\d{4}' ]                       │
    │  - \d{3}  : 숫자 3개 찾음 (010)                                  │
    │  - '-'    : 하이픈 대조                                          │
    │  - \d{4}  : 숫자 4개 찾음 (1234) ...                             │
    └──────┬──────────────────────────────────────────────────────────┘
           │
    ┌──────▼──────────────────────────────────────────────────────────┐
    │  [ SQL Functions ]                                              │
    │  1. REGEXP_LIKE   ──▶ TRUE (패턴 존재 확인)                      │
    │  2. REGEXP_SUBSTR ──▶ '010-1234-5678' (값 추출)                  │
    │  3. REGEXP_REPLACE──▶ '***-****-****' (보안 마스킹)               │
    └─────────────────────────────────────────────────────────────────┘
```

### 3. 심층 동작 원리: 상태 머신 (FSM) 엔진
- **원리**: DBMS 엔진 내부의 정규식 처리기는 **유한 상태 오토마타(FSM)**를 구축함.
- **작동**: 문자열을 한 글자씩 읽으며 정의된 상태 전이도(State Transition)를 따라가고, 최종 수락 상태에 도달하면 매칭 성공으로 판정.

### 4. 핵심 기술: 후방 참조 (Back-reference)
- 매칭된 괄호 안의 그룹을 `\1`, `\2` 등의 기호로 다시 불러와서 순서를 바꾸거나 재활용하는 기술. (예: 성과 이름의 위치 바꾸기)

- **📢 섹션 요약 비유**: 메타 문자는 '패턴 설계도'이며, 엔진은 이 설계도를 들고 데이터 현장을 누비는 '검사관'입니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. LIKE 연산자 vs 정규 표현식 (REGEXP)

| 비교 항목 | LIKE 연산자 | 정규 표현식 (REGEXP) |
|:---:|:---|:---|
| **표현력** | 낮음 (%, _ 만 가능) | **무한함 (복합 패턴 지원)** |
| **학습 곡선** | 낮음 (매우 쉬움) | **높음 (문법 복잡)** |
| **성능** | 상대적으로 빠름 (인덱스 유리) | **느림 (CPU 연산 집중)** |
| **인덱스 활용** | 전방 일치 시 활용 가능 | **대부분 활용 불가 (Full Scan)** |

### 2. 과목 융합 관점: 컴파일러 및 보안
- **컴파일러 융합**: 정규 표현식은 **어휘 분석(Lexical Analysis)**의 핵심 기술이며, SQL 엔진은 정규식을 실행 가능한 바이트코드로 컴파일하여 성능을 최적화한다.
- **보안(Security) 융합**: 정규 표현식은 웹 방화벽(WAF)이나 DB 보안 도구에서 **SQL Injection** 패턴이나 민감 정보 유출을 탐지하는 패턴 매칭의 심장부 역할을 수행한다.

- **📢 섹션 요약 비유**: LIKE가 '신분증 앞자리만 보는 경비원'이라면, REGEXP는 '지문과 홍채까지 대조하는 정밀 보안 검색대'입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 1. 실무 시나리오: 개인정보 필터링 및 데이터 정제
- **상황**: 고객 상담 게시판에 주민번호나 카드번호를 적은 글을 찾아 숨겨야 함.
- **판단**: "숫자-숫자" 형태가 너무 다양해서 `LIKE`로는 불가능함.
- **결단**: **REGEXP_REPLACE 활용**. `[0-9]{6}-[0-9]{7}` 패턴을 정의하여, 해당 패턴이 발견되는 즉시 `######-#######`로 치환하는 스케줄러 배치 구동. 개인정보 유출 사고를 시스템적으로 예방.

### 2. 안티패턴: 대량 테이블 WHERE 절에서의 정규식 사용
- **증상**: 1억 건 테이블의 검색 조건에 `REGEXP_LIKE`를 사용.
- **결과**: 인덱스를 타지 못하고 1억 번의 복잡한 패턴 연산을 수행하느라 CPU 점유율 폭발 및 타임아웃 발생.
- **조치**: **검색 범위를 먼저 축소**. 날짜나 ID 등의 인덱스 조건으로 1,000건 이하로 줄인 뒤, 마지막 필터 단계에서 정규 표현식을 적용하는 '필터 체이닝' 전략 수립.

### 📢 기술사적 결언
> "정규 표현식은 데이터베이스의 **'언어 지능'**을 상징한다. 텍스트 속에 숨은 의미의 질서를 찾아내기 때문이다. 기술사는 정규 표현식의 강력함에 취해 **'성능의 독'**을 마시는 실수를 범해서는 안 된다. 정규식은 기본적으로 **CPU Bound** 작업임을 인지하고, 가능한 한 **애플리케이션 계층에서의 전처리**와 **DB 계층에서의 인덱스 기반 필터링** 사이에서 최적의 연산 배치 결정을 내려야 한다."

- **📢 섹션 요약 비유**: 정규 표현식을 남발하는 것은 '모든 방 문을 열 때마다 복잡한 퍼즐을 푸는 것'과 같아, 전체적인 이동 속도(시스템 성능)를 크게 떨어뜨립니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 1. 정량/정성 기대효과 (정규 표현식 최적 활용 시)

| 지표 | 도입 전 | 도입 후 | 개선 효과 |
|:---|:---:|:---:|:---:|
| **패턴 매칭 코드량** | 수백 라인 (IF/LIKE 조합) | **SQL 1줄** | 개발 유지보수성 90% 향상 |
| **데이터 정제 정확도** | 낮음 (예외 케이스 발생) | **높음 (엄격한 규칙)** | 데이터 품질 신뢰도 확보 |
| **보안 사고 대응력** | 수동 감시 | **실시간 패턴 탐지** | 리스크 관리 역량 강화 |

### 2. 미래 전망
- **AI-Generated Regex**: 사용자가 원하는 추출 결과 예시만 주면, AI가 가장 최적의(성능과 정확도를 고려한) 정규 표현식을 자동으로 생성해주는 기술 보편화.
- **Vectorized Regex Engine**: 현대의 최신 CPU 명령어(AVX-512 등)를 활용하여 여러 문자열을 동시에 정규식으로 검사하는 **SIMD 기반 정규식 가속** 기술 탑재.

- **📢 섹션 요약 비유**: 미래의 정규 표현식은 우리가 기호를 외우지 않아도, 우리가 '찾고 싶은 느낌'을 말하면 데이터들 사이에서 정답만을 쏙쏙 골라주는 지능형 탐지기가 될 것입니다.
