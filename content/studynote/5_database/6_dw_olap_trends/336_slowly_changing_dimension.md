# 336. 느리게 변하는 차원 (SCD) - 데이터 역사의 기록법

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 느리게 변하는 차원(SCD, Slowly Changing Dimension)은 시간이 지남에 따라 속성 값이 불규칙하고 드물게 변하는 차원 데이터를 데이터 웨어하우스(DW)에서 어떻게 관리할 것인지에 대한 설계 전략이다.
> 2. **가치**: 과거의 분석 결과가 현재의 변화로 인해 왜곡되지 않도록 **역사적 정합성(Historical Integrity)**을 유지하며, 데이터의 변화 궤적을 추적하여 시계열 분석을 가능케 한다.
> 3. **융합**: 대리 키(Surrogate Key) 및 유효 기간(Start/End Date) 속성과 결합되어, 정적인 마스터 데이터를 동적인 이력 데이터로 변환하는 분석 아키텍처의 핵심 공정이다.

+++

### Ⅰ. SCD의 발생 배경 및 필요성

- **상황**: 고객이 이사를 가거나(주소 변경), 상품의 카테고리가 재분류되는 등 현실 세계의 속성은 변합니다.
- **문제**: 단순히 현재 값으로 덮어쓰면, 과거 해당 고객이 옛날 주소지에 살 때 발생했던 매출 통계가 현재 주소지 통계로 바뀌어 '과거의 진실'이 사라집니다.
- **해결**: 변화의 종류와 분석 목적에 따라 유형별(Type 0~4) 대응 방식을 선택합니다.

+++

### Ⅱ. SCD의 핵심 매커니즘 (ASCII Model)

기본적으로 대리 키를 통해 하나의 개체에 대해 여러 개의 '행'을 가질 수 있게 설계합니다.

```text
[ Slowly Changing Dimension Architecture ]

  (Time Axis) ──▶  T1 (Old State)  ──▶  T2 (Update Event) ──▶  T3 (New State)
  
  [ Dimensional Table Structure ]
  ┌────────┬──────────┬──────────┬──────────┬──────────┐
  │  SK    │ Business │ Attribute│ Valid    │ Current  │
  │ (PK)   │ Key      │ (Value)  │ Period   │ Flag     │
  ├────────┼──────────┼──────────┼──────────┼──────────┤
  │ 1001   │ CUST_01  │ Seoul    │ 2025~    │ N        │ ◀── Old Record
  │ 1005   │ CUST_01  │ Pusan    │ 2026~    │ Y        │ ◀── New Record ✅
  └────────┴──────────┴──────────┴──────────┴──────────┘
```

+++

### Ⅲ. 실무적 의의: 분석 정합성

- **Point-in-time Analysis**: "특정 시점에 이 고객은 어느 등급이었는가?"를 정확히 짚어내어 당시의 팩트와 매칭할 수 있습니다.
- **Trend 추적**: 속성이 변해온 과정을 통해 고객의 생애 주기를 분석하거나 상품 인기도의 변화 흐름을 파악할 수 있습니다.

- **📢 섹션 요약 비유**: SCD는 **'졸업 앨범과 현재 사진의 관계'**와 같습니다. 내 얼굴(데이터)은 매일 조금씩 변하지만, 졸업 앨범(과거 기록)은 그 당시의 모습을 그대로 간직하고 있어야 합니다. 현재 모습으로 앨범 사진을 다 바꿔버리면(덮어쓰기) 내 과거를 증명할 길이 사라지듯, 분석 시스템에서도 과거의 흔적을 남기는 규칙이 필요합니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Surrogate Key]**: SCD Type 2 등을 구현하기 위한 필수 전제.
- **[Historical Accuracy]**: SCD가 추구하는 최고의 가치.
- **[ETL Logic]**: 소스 시스템의 변화를 감지하여 SCD 테이블을 갱신하는 복잡한 연산 로직.

📢 **마무리 요약**: **SCD**는 데이터 웨어하우스의 '기억법'입니다. 단순히 데이터를 담는 것을 넘어, 데이터가 살아온 궤적을 온전히 보존하는 것이 분석의 시작입니다.
