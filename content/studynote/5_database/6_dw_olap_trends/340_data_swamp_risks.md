# 340. 데이터 늪 (Data Swamp) - 관리되지 않는 데이터의 역습

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 늪은 데이터 레이크(Data Lake)를 구축한 후, 적절한 **메타데이터 관리와 데이터 거버넌스가 결여되어 무엇이 어디에 있는지 알 수 없게 된 무질서한 상태**를 의미한다.
> 2. **가치**: 데이터의 활용 불능 상태를 경고함으로써, 단순 저장(Storage)과 진정한 분석 환경(Analytics)의 차이를 명확히 하고 데이터 품질 관리의 시급성을 강조한다.
> 3. **융합**: 데이터 카탈로그(Data Catalog), 비즈니스 용어집(Glossary), 데이터 리니지(Lineage) 기술과 융합되어, 늪에 빠진 데이터를 다시 맑은 호수로 정화하는 아키텍처적 장치가 요구된다.

+++

### Ⅰ. 데이터 늪의 발생 원인

- **무분별한 수집**: "나중에 쓸모 있겠지"라며 출처와 용도도 모르는 데이터를 무작정 적재만 하는 행위.
- **메타데이터 부재**: 데이터의 이름, 생성 일시, 소유자, 형식 등에 대한 기록이 없는 상태.
- **품질 관리 포기**: 중복 데이터, 오류 데이터가 필터링 없이 쌓여 분석 결과의 신뢰도를 떨어뜨림.
- **접근 제어 실패**: 보안 정책이 불분명하여 아무도 접근하지 못하거나, 반대로 아무나 수정하여 오염되는 상태.

### Ⅲ. 늪에서 탈출하는 방안 (정화 전략)

1. **데이터 카탈로그 도입**: 모든 데이터 자산에 태그를 달고 검색할 수 있는 '데이터 지도'를 만듭니다.
2. **데이터 라이프사이클 정의**: 유효 기간이 지난 쓸모없는 데이터(Rotten Data)는 주기적으로 삭제하거나 아카이빙합니다.
3. **데이터 소유권(Data Ownership) 명확화**: 각 데이터 파편에 대해 책임지고 관리할 담당 도메인을 지정합니다 (Data Mesh 전략).

- **📢 섹션 요약 비유**: 데이터 늪은 **'정리 안 된 창고'**와 같습니다. 처음에는 넓어서 좋다고 이것저것 다 집어넣었지만(수집), 나중에 물건 하나 찾으려면 온 동네를 뒤져야 하고 결국 못 찾아서 새로 사는 것과 같습니다. 창고가 늪이 되지 않으려면 물건마다 라벨(메타데이터)을 붙이고 선반(거버넌스)을 잘 짜야 합니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Data Governance]**: 늪을 방지하는 가장 강력한 행정적 도구.
- **[Metadata]**: 데이터의 신분증이자 위치 추적기.
- **[Data Lineage]**: 데이터가 어디서 와서 어디로 가는지 추적하는 흐름도.

📢 **마무리 요약**: **Data Swamp**는 데이터 프로젝트의 가장 큰 적입니다. 저장의 양보다 중요한 것은 **'찾아 쓸 수 있는 능력'**임을 잊지 말아야 합니다.
