# 370. 데이터베이스 감사(Audit) 로그 - 사후 추적의 기록

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터베이스 감사(Audit) 로그는 데이터베이스 내에서 발생하는 **모든 활동(접속, 쿼리 실행, 권한 변경 등)을 '누가, 언제, 어디서, 무엇을' 했는지 상세히 기록**하여 보관하는 증적 자료다.
> 2. **가치**: 보안 사고 발생 시 원인을 규명하는 **포렌식(Forensics)** 근거가 되며, 내부자에 의한 데이터 오남용을 심리적으로 억제하고 각종 법적 컴플라이언스(ISMS, GDPR 등)를 준수하게 한다.
> 3. **융합**: 실시간 이상 징후 탐지(SIEM) 및 데이터 거버넌스 기술과 융합되어, 단순한 기록 보관을 넘어 실시간 보안 위협에 대응하는 능동적 방어 체계의 핵심 소스가 된다.

+++

### Ⅰ. 데이터베이스 감사의 주요 대상

- **인증 및 접속**: 로그인 성공/실패 기록, 접속 IP, 터미널 정보.
- **DML (Data Manipulation)**: 민감 테이블에 대한 `SELECT`, `UPDATE`, `DELETE` 실행 내역.
- **DDL (Data Definition)**: 테이블 생성, 삭제, 스키마 변경 이력.
- **DCL (Data Control)**: 권한 부여(`GRANT`) 및 회수(`REVOKE`) 내역.
- **시스템 설정 변경**: 파라미터 수정, 백업 실행 등 관리자 활동.

+++

### Ⅱ. 감사 로그 관리 아키텍처 (ASCII Flow)

감사 로그는 위변조 방지를 위해 DB 외부의 안전한 저장소로 즉시 전송하는 것이 정석입니다.

```text
[ Secure Audit Log Pipeline ]

  [ Database Engine ] ──▶ [ Audit Agent ] ──▶ [ Log Collector ]
          │ (Activity Trace)      │ (Extraction)      │
          ▼                       ▼                   ▼
  ┌───────────────┐       ┌───────────────┐   ┌───────────────┐
  │ Local Log File│       │ Central SIEM  │   │ WORM Storage  │
  └───────────────┘       └───────────────┘   └───────────────┘
   (Temp Storage)          (Real-time Analysis) (Immutable Archive) ✅
```

+++

### Ⅲ. 감사 로그 관리의 3대 원칙

1. **무결성 (Integrity)**: 기록된 로그는 관리자라도 절대 수정하거나 지울 수 없어야 합니다 (WORM 스토리지 활용).
2. **기밀성 (Confidentiality)**: 감사 로그 자체에 민감 정보가 포함될 수 있으므로, 로그에 대한 접근 권한을 엄격히 분리(SoD)해야 합니다.
3. **가용성 (Availability)**: 법적 보존 기간(보통 6개월~5년) 동안 언제든 즉시 조회가 가능하도록 인덱싱 및 백업을 관리해야 합니다.

- **📢 섹션 요약 비유**: 데이터베이스 감사 로그는 **'비행기의 블랙박스'**와 같습니다. 평소에는 운항(서비스)에 아무런 영향을 주지 않고 조용히 모든 비행 기록을 담고 있지만, 사고가 났을 때 비로소 꺼내어 "어느 시점에 어떤 조작이 잘못되었는지"를 밝혀내어 진실을 규명하는 가장 결정적인 증거물입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[SoD (Segregation of Duties)]**: 감사자와 관리자의 역할을 분리하는 원칙.
- **[Fine-grained Auditing]**: 특정 조건(예: 100만 원 이상 인출)에만 로그를 남기는 정밀 감사 기술.
- **[Compliance]**: 감사를 강제하는 법적 규제 (개인정보 보호법 등).

📢 **마무리 요약**: **Database Audit Log**는 시스템의 양심입니다. 투명한 기록 관리를 통해 데이터의 신뢰를 구축하고, 예기치 못한 위기 상황에서 시스템을 지켜내는 최후의 보루입니다.
