# 324. ETL (Extraction, Transformation, Loading) - 데이터 통합의 삼중주

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ETL은 다양한 원천 시스템(Source)에서 데이터를 **추출(Extract)**하여, 분석에 적합한 형태로 **변환(Transform)**한 뒤, 데이터 웨어하우스(DW)와 같은 목적지에 **적재(Load)**하는 일련의 데이터 통합 프로세스다.
> 2. **가치**: 파편화된 원시 데이터를 정제하고 표준화함으로써 데이터 품질을 확보하며, 비즈니스 의사결정에 즉시 활용 가능한 고품질의 정보 자산을 생성한다.
> 3. **융합**: 최근에는 클라우드의 강력한 연산 능력을 활용하여 추출-적재 후 변환하는 **ELT** 방식으로 융합 진화하고 있으며, 데이터 엔지니어링 파이프라인의 핵심 공정으로 작용한다.

+++

### Ⅰ. ETL의 3단계 상세 과정

1. **Extract (추출)**: 
    - ERP, CRM, 로그 파일, 외부 API 등 다양한 소스에서 데이터를 수집합니다. 
    - 전체를 다 가져오는 Full Load와 변경분만 가져오는 Incremental Load 방식이 있습니다.
2. **Transform (변환)**: 
    - **가장 중요한 단계**. 데이터 정제(Cleaning), 중복 제거, 형식 통일, 집계(Aggregation) 등을 수행합니다.
    - 예: 'M/F' 성별 코드를 '남성/여성'으로 통일.
3. **Load (적재)**: 
    - 변환된 데이터를 목적지(DW, Data Mart)에 저장합니다. 
    - 초기 적재 후 주기적인 배치(Batch) 업데이트를 통해 최신성을 유지합니다.

### Ⅱ. ETL 프로세스 시각화 (ASCII Flow)

```text
[ ETL Pipeline Architecture ]

  (Sources)          (Staging Area)          (Target DW)
  ┌────────┐        ┌──────────────┐        ┌────────────┐
  │ RDBMS  │ ──(E)─▶│  Cleaning    │ ──(L)─▶│ Final      │
  ├────────┤        │  Formating   │        │ Fact/Dim   │
  │ Log    │ ──(E)─▶│  (Transform) │ ──(L)─▶│ Tables     │
  └────────┘        └──────────────┘        └────────────┘
```

### Ⅲ. ETL 도입 시 고려사항

- **데이터 품질**: "쓰레기를 넣으면 쓰레기가 나온다(GIGO)"는 원칙에 따라 변환 단계에서의 철저한 검증이 필요합니다.
- **성능 부하**: 소스 시스템에 부하를 주지 않도록 운영 시간이 아닌 심야에 수행하거나 CDC 기술을 활용해야 합니다.
- **메타데이터 관리**: 데이터가 어떻게 변했는지(Lineage) 추적할 수 있어야 합니다.

- **📢 섹션 요약 비유**: ETL은 **'원유 정제 과정'**과 같습니다. 땅속에서 갓 뽑아낸 원유(추출)는 바로 쓸 수 없지만, 정유 공장(변환)에서 불순물을 제거하고 휘발유나 경유로 나누면 비로소 자동차(분석 시스템)를 움직이는 연료(데이터)로 적재하여 사용할 수 있게 되는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Staging Area]**: 변환 작업을 위해 데이터를 임시로 두는 작업장.
- **[Data Cleansing]**: 틀린 데이터를 바로잡는 정제 작업.
- **[ETL Tool]**: Informatica, Talend, DataStage 등 전문 솔루션.

📢 **마무리 요약**: **ETL**은 분석의 원동력입니다. 복잡하고 지저분한 원시 데이터를 가치 있는 비즈니스 자산으로 탈바꿈시키는 가장 정교한 공학적 공정입니다.
