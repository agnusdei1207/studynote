# 330. 다차원 모델링 (Dimensional Modeling) - 분석을 위한 데이터 설계

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다차원 모델링은 데이터 웨어하우스 설계 시 사용자 관점의 분석 편의성을 극대화하기 위해, 데이터를 수치 측정값인 **팩트(Fact)**와 이를 설명하는 속성인 **차원(Dimension)**으로 구분하여 구조화하는 기법이다.
> 2. **가치**: 복잡한 ER 모델을 직관적인 **스타 스키마(Star Schema)** 형태로 단순화하여, 비즈니스 사용자가 쿼리를 이해하기 쉽게 하고 시스템의 대량 조회 성능을 비약적으로 향상시킨다.
> 3. **융합**: 킴벌(Kimball)의 DW 구축 방법론의 핵심이며, 데이터 분석 도구(BI)가 다차원 큐브를 생성하고 드릴다운 연산을 수행하는 물리적 토대가 된다.

+++

### Ⅰ. 다차원 모델의 핵심 구성 요소

1. **팩트 테이블 (Fact Table)**:
    - 비즈니스 프로세스에서 발생하는 수치 데이터(측정값)를 담습니다.
    - 예: 매출액, 판매량, 온도, 클릭 수.
    - 외래키(FK)를 통해 여러 차원 테이블과 연결됩니다.
2. **차원 테이블 (Dimension Table)**:
    - 팩트 데이터에 대한 문맥(Context)을 제공합니다.
    - 예: 시간(언제), 상품(무엇을), 매장(어디서), 고객(누가).
    - 텍스트 중심이며 비정규화된 구조를 가집니다.

+++

### Ⅱ. 스타 스키마 구조 시각화 (ASCII Model)

팩트 테이블을 중심으로 차원 테이블들이 별 모양으로 둘러싼 구조입니다.

```text
[ Star Schema Architecture ]

      [ Dim_Product ]          [ Dim_Time ]
      - ProductID (PK)         - TimeID (PK)
      - Name, Category         - Date, Quarter
             │                        │
             └───────┐        ┌───────┘
                     ▼        ▼
               [ Fact_Sales ] (Central)
               - SalesID (PK)
               - ProductID (FK)
               - TimeID (FK)
               - RegionID (FK)
               - Amount, Quantity (Measures)
                     ▲        ▲
             ┌───────┘        └───────┐
             │                        │
      [ Dim_Region ]           [ Dim_Customer ]
      - RegionID (PK)          - CustomerID (PK)
      - City, State            - Name, Grade
```

+++

### Ⅲ. 다차원 모델링의 설계 단계

1. **비즈니스 프로세스 선택**: 어떤 업무를 분석할 것인가? (예: 일일 판매 현황)
2. **그레인(Grain) 설정**: 데이터의 최소 단위 정의. (예: 영수증 행 단위 vs 일별 요약 단위)
3. **차원 식별**: 분석 관점 결정. (예: 일자별, 상품별, 고객별)
4. **팩트 식별**: 측정할 수치 결정. (예: 판매 수량, 할인 금액)

- **📢 섹션 요약 비유**: 다차원 모델링은 **'가계부 정리'**와 같습니다. 매일 쓴 '돈의 액수(Fact)'를 가운데 적고, 그 옆에 '언제', '어디서', '무엇을 위해' 썼는지(Dimension)를 기록하는 것과 같습니다. 이렇게 정리해 두면 나중에 "지난달에 커피값으로 얼마 썼지?"라는 질문에 바로 대답할 수 있는 것과 같은 원리입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Star Schema]**: 조인을 최소화한 가장 단순하고 빠른 구조.
- **[Snowflake Schema]**: 차원 테이블을 정규화하여 관리 효율을 높인 구조.
- **[Measure]**: 팩트 테이블의 숫자 데이터 (합산 가능한 값).

📢 **마무리 요약**: **Dimensional Modeling**은 데이터 분석의 기초 공사입니다. 데이터를 사용하는 사람의 눈높이에서 가장 편하고 빠르게 답을 얻을 수 있는 구조를 만드는 것이 목표입니다.
