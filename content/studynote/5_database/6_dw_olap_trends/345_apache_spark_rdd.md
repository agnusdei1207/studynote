# 345. Apache Spark & RDD - 인메모리 분산 처리의 혁명

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Apache Spark는 디스크 기반의 맵리듀스 한계를 극복하기 위해 데이터를 메모리 상에 유지하며 고속으로 연산하는 **범용 인메모리 분산 처리 엔진**이다.
> 2. **가치**: 핵심 추상화 모델인 **RDD(Resilient Distributed Dataset)**를 통해 대규모 데이터를 병렬로 처리하면서도, 노드 장애 시 계보(Lineage)를 따라 자동으로 복구되는 강력한 **내결함성**을 제공한다.
> 3. **융합**: 실시간 스트리밍(Spark Streaming), SQL 분석(Spark SQL), 머신러닝(MLlib), 그래프 분석(GraphX) 기능을 하나의 통합된 플랫폼에서 제공하여 현대 데이터 사이언스의 표준 엔진으로 자리 잡았다.

+++

### Ⅰ. RDD (Resilient Distributed Dataset)의 이해

- **Resilient (회복력)**: 메모리 내 데이터가 손실되어도, 데이터 생성 과정을 기록한 **Lineage(계보)**를 통해 언제든 다시 생성할 수 있습니다.
- **Distributed (분산)**: 클러스터의 여러 노드에 나누어 저장되고 병렬로 연산됩니다.
- **Dataset (데이터셋)**: 읽기 전용(Immutable) 객체들의 모음입니다.

+++

### Ⅱ. 스파크 연산 아키텍처 (ASCII Flow)

스파크는 지연 연산(Lazy Evaluation) 방식을 사용하여 실행 계획을 최적화합니다.

```text
[ Spark Execution Model ]

  (Input Data) ──▶ [ Transform 1 ] ──▶ [ Transform 2 ] ──▶ [ ACTION! ]
                         │                    │               │
                         └───────── Lineage ──┴───────────────┤
                                                              ▼
  1. Transformation: 맵, 필터 등 (즉시 실행 안 됨, 계획만 수립)
  2. Action: 카운트, 저장 등 (실제 물리적 연산 트리거)
  3. Result: [ Driver Program ] ◀───▶ [ Executors ]
```

+++

### Ⅲ. 맵리듀스 vs 스파크 비교

| 비교 항목 | MapReduce (Hadoop) | Apache Spark |
|:---|:---|:---|
| **처리 방식** | 디스크 기반 (Step별 I/O) | **인메모리 기반 (RAM 활용)** |
| **속도** | 상대적으로 느림 | **최대 100배 빠름** |
| **복잡도** | 구현이 까다로움 | 풍부한 API (Python, Scala 등) 제공 |
| **주요 용도** | 대량의 배치 작업 | **머신러닝, 실시간 분석, 반복 연산** |

- **📢 섹션 요약 비유**: 스파크와 RDD는 **'고속도로 하이패스'**와 같습니다. 맵리듀스가 톨게이트마다 멈춰 서서 돈을 내고(디스크 쓰기) 다시 출발하는 것이라면, 스파크는 처음부터 끝까지 멈추지 않고 고속으로 달려(인메모리) 목적지까지 도달하는 방식입니다. 중간에 길을 잘못 들어도(장애), 내비게이션의 주행 기록(Lineage)이 있어 바로 원래 길로 돌아올 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[DAG (Directed Acyclic Graph)]**: 스파크가 쿼리를 실행하기 위해 내부적으로 그리는 비순환 작업 그래프.
- **[Lineage]**: RDD 복구의 생명줄.
- **[Lazy Evaluation]**: 마지막 결과가 필요할 때까지 연산을 미뤄두는 최적화 전략.

📢 **마무리 요약**: **Apache Spark**는 데이터 처리의 패러다임을 '속도'와 '편의성'으로 바꾼 주인공입니다. RDD라는 우아한 모델을 통해 빅데이터를 가장 빠르고 안전하게 다루는 법을 제시했습니다.
