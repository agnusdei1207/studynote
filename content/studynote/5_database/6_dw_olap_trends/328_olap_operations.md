# 328. OLAP (Online Analytical Processing) - 다차원 분석의 즐거움

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: OLAP(온라인 분석 처리)은 최종 사용자가 **대규모 데이터를 여러 차원(Dimension)에서 대화식으로 분석**하여 의사결정에 필요한 통찰을 즉시 얻을 수 있게 하는 데이터 처리 기술이다.
> 2. **가치**: 복잡한 집계 연산을 미리 수행하거나 최적화하여, 수백만 건의 데이터를 '지역별', '시간별', '상품별'로 자유롭게 회전(Pivot)시키며 실시간으로 탐색할 수 있게 한다.
> 3. **융합**: 다차원 모델링(스타 스키마) 및 인메모리 큐브(Cube) 기술과 융합되어, 현대 기업의 BI(Business Intelligence) 대시보드와 데이터 시각화 도구의 핵심 엔진으로 작동한다.

+++

### Ⅰ. OLAP의 4대 핵심 연산

사용자가 데이터를 다각도로 살펴보기 위해 사용하는 조작법입니다.

1. **Roll-up (롤업)**: 요약 수준을 높이는 것. (예: 일별 매출 ──▶ 월별 매출)
2. **Drill-down (드릴다운)**: 더 상세한 수준으로 내려가는 것. (예: 연도별 매출 ──▶ 분기별 매출)
3. **Slicing & Dicing (슬라이싱 & 다이싱)**:
    - **Slicing**: 다차원 큐브의 한 단면을 자름. (예: 2026년 데이터만 보기)
    - **Dicing**: 특정 부분 집합을 작은 큐브 형태로 추출. (예: 서울 지역의 1분기 가전제품 매출)
4. **Pivoting (피러팅)**: 보고서의 행과 열을 바꿈으로써 분석 관점을 회전시킴.

+++

### Ⅱ. OLAP 다차원 큐브 시각화 (ASCII Model)

```text
[ OLAP Data Cube ]

          /  Time  /
         /────────/
        /  2026  /
       /────────/  (Dimension 1)
      ┌──────────┐
      │  Product │ (Dimension 2)
      ├──────────┤
      │  Region  │ (Dimension 3)
      └──────────┘
           │
           ▼
  [ Cell Content: Measure ]
  (예: Sales Amount, Quantity)
```

+++

### Ⅲ. OLAP의 구현 유형

- **MOLAP (Multidimensional OLAP)**: 데이터를 다차원 큐브 구조로 물리적으로 저장. 속도는 가장 빠르나 용량 한계가 있음.
- **ROLAP (Relational OLAP)**: 기존 RDBMS 테이블을 사용하여 분석 수행. 대용량 처리에 유리하나 MOLAP보다 상대적으로 느림.
- **HOLAP (Hybrid OLAP)**: 두 방식의 장점을 혼합. 요약 데이터는 MOLAP, 상세 데이터는 ROLAP에 저장.

- **📢 섹션 요약 비유**: OLAP은 **'데이터로 만든 루빅스 큐브'**와 같습니다. 큐브를 이리저리 돌려보고(Pivot), 특정 면을 잘라보며(Slice), 안을 깊이 들여다보는(Drill-down) 과정을 통해 복잡하게 얽힌 숫자들 속에서 비즈니스의 정답을 찾아내는 재미있는 놀이이자 강력한 도구입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[BI (Business Intelligence)]**: OLAP을 통해 달성하고자 하는 기업의 지능화 목표.
- **[Measure]**: 큐브 안에 들어있는 수치 데이터 (합계, 평균 등).
- **[Dimension]**: 데이터를 바라보는 관점 (시간, 장소, 대상).

📢 **마무리 요약**: **OLAP**은 데이터를 정보로, 정보를 지식으로 바꾸는 연금술입니다. 다차원적 사고를 시스템적으로 구현하여 기업의 미래를 예측하는 혜안을 제공합니다.
