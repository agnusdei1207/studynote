# 329. MOLAP, ROLAP, HOLAP - 분석 엔진의 물리적 구현

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: MOLAP은 다차원 배열 구조의 **전용 저장소**를 쓰고, ROLAP은 기존 **관계형 DB**를 활용하며, HOLAP은 이 둘의 장점을 **결합**한 분석 처리 아키텍처다.
> 2. **가치**: 데이터의 규모와 응답 속도 요구사항에 따라 최적의 물리적 저장 방식을 선택함으로써, 대규모 데이터셋에서도 지연 없는 다차원 탐색을 가능케 한다.
> 3. **융합**: 인메모리 기술 및 대규모 병렬 처리(MPP) 기술과 융합되어, 수 테라바이트급 데이터를 1초 내에 집계하는 현대적 BI 시스템의 근간을 형성한다.

+++

### Ⅰ. 유형별 상세 특징 비교

| 비교 항목 | MOLAP (Multidimensional) | ROLAP (Relational) | HOLAP (Hybrid) |
|:---|:---|:---|:---|
| **저장 방식** | **다차원 큐브 (Array)** | **관계형 테이블 (Row/Col)** | 요약은 큐브, 상세는 RDB |
| **성능 (속도)** | **매우 빠름** (미리 집계) | 상대적으로 느림 (동적 집계) | 중간 (효율적 배분) |
| **확장성 (용량)** | 낮음 (데이터 폭증 시 한계) | **매우 높음** (대용량 유리) | 높음 |
| **주요 특징** | 고속 응답, 직관적 구조 | 유연성, 표준 SQL 활용 | 최적의 균형점 지향 |

+++

### Ⅱ. 아키텍처 시각화 (ASCII Model)

```text
[ MOLAP: Pre-computed Cube ]       [ ROLAP: Dynamic Query ]
  
  (Dimension A)                      (SQL Engine)
       │                                  │
  ┌────▼────┐                        ┌────▼────┐
  │  Cube   │ ◀── 미리 집계 완료!      │  RDBMS  │ ◀── 쿼리 시점에 계산
  └─────────┘                        └─────────┘
  
[ HOLAP: The Hybrid ]
  ┌──────────────────┐
  │ Summary (MOLAP)  │ ──▶ 상위 수준 분석 (고속)
  ├──────────────────┤
  │ Detailed (ROLAP) │ ──▶ 하위 수준 상세 분석 (유연)
  └──────────────────┘
```

+++

### Ⅲ. 실무적 선택 가이드

- **MOLAP 선택**: 분석 관점이 명확하고, 데이터량이 아주 방대하지 않으며, '즉각적인' 응답 속도가 생명일 때.
- **ROLAP 선택**: 데이터가 수억 건 이상이며, 수시로 새로운 분석 관점(Dimension)이 추가되어 유연성이 필요할 때.
- **HOLAP 선택**: 전반적인 트렌드는 빨리 보면서도, 필요할 때만 깊게 파고드는(Drill-through) 하이브리드 환경이 필요할 때.

- **📢 섹션 요약 비유**: 이 세 방식은 **'공부 요약본'**과 같습니다. MOLAP은 미리 요약된 '요약 노트'만 들고 있는 것이고(빠르지만 내용 한계), ROLAP은 '전공 서적 전체'를 보며 그때그때 찾는 것이며(느리지만 모든 내용 있음), HOLAP은 '요약 포스트잇'을 책 앞에 붙여두고 필요할 때만 본문을 펼쳐보는 영리한 학생과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Sparsity]**: MOLAP 큐브에서 데이터가 비어있는 현상 (비효율의 원인).
- **[Drill-through]**: HOLAP에서 요약 데이터(MOLAP)를 보다가 상세 내역(ROLAP)으로 바로 이동하는 기능.
- **[MDX]**: 다차원 큐브를 다루기 위한 전용 질의 언어.

📢 **마무리 요약**: **MOLAP, ROLAP, HOLAP**은 성능과 용량 사이의 끝없는 줄다리기입니다. 데이터의 성격과 사용자의 인내심을 고려하여 최적의 엔진을 배치해야 합니다.
