# 355. 데이터 거버넌스 (Data Governance) - 데이터의 법과 질서

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 거버넌스는 전사적 차원에서 데이터의 가용성, 유용성, 무결성, 보안을 보장하기 위해 수립된 **정책, 프로세스, 역할 및 책임(R&R), 표준의 통합적인 관리 체계**다.
> 2. **가치**: "데이터는 누구의 소유이며, 어떤 규칙으로 관리되는가?"에 대한 명확한 답을 제시하여, 데이터 품질 사고를 예방하고 비즈니스 의사결정의 신뢰도를 높인다.
> 3. **융합**: IT 부서의 기술적 통제와 현업 부서의 비즈니스 지식이 융합된 결과물이며, 현대 기업이 인공지능(AI)이나 클라우드로 나아가기 위해 반드시 거쳐야 하는 데이터 민주화의 초석이다.

+++

### Ⅰ. 데이터 거버넌스의 3대 구성 요소

1. **사람 (People)**: 데이터 소유자(Owner), 데이터 관리자(Steward), 거버넌스 위원회 등 조직적 체계 수립.
2. **프로세스 (Process)**: 데이터의 생성부터 폐기까지의 생애주기 관리 및 표준화된 업무 절차 정의.
3. **기술 (Technology)**: 메타데이터 관리 시스템, 데이터 사전, 품질 모니터링 도구 등 거버넌스를 뒷받침하는 IT 인프라.

+++

### Ⅱ. 거버넌스 체계도 시각화 (ASCII Model)

```text
[ Data Governance Framework ]

  ┌───────────────────────────────────────────┐
  │  Data Strategy & Policies (의사결정/표준)   │
  ├───────────────────┬───────────────────────┤
  │ [ Metadata Mgmt ] │ [ Data Quality Mgmt ] │
  │ (데이터 의미/위치) │ (정확성/일관성 검증)  │
  ├───────────────────┼───────────────────────┤
  │ [ Data Security ] │ [ Privacy & Compl. ]  │
  │ (접근제어/암호화)  │ (법적 규제 준수)      │
  └─────────┬─────────┴───────────┬───────────┘
            ▼                     ▼
      [ Single Source of Truth ] (신뢰할 수 있는 단일 진실)
```

+++

### Ⅲ. 데이터 거버넌스의 주요 활동

- **표준화**: 단어의 정의, 도메인, 코드 등을 통일하여 부서 간 소통 오류 제거.
- **품질 관리**: 주기적인 데이터 프로파일링을 통해 오류 데이터를 정화.
- **마스터 데이터 관리 (MDM)**: 고객, 상품 등 핵심 기준 정보를 전사적으로 동기화.
- **컴플라이언스 대응**: GDPR, 개인정보 보호법 등 법적 요구사항을 시스템적으로 이행.

- **📢 섹션 요약 비유**: 데이터 거버넌스는 **'교통 법규와 신호 체계'**와 같습니다. 자동차(데이터)가 아무리 많아도 도로(인프라) 위에서 제멋대로 달리면 대형 사고가 납니다. 신호등(프로세스)을 설치하고 차선(표준)을 그리고 면허증(권한)을 관리해야만 모든 차가 안전하고 빠르게 목적지까지 도착할 수 있는 질서가 생기는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Data Stewardship]**: 현업 접점에서 데이터 품질을 책임지고 관리하는 실무 활동.
- **[Metadata]**: 거버넌스를 구현하기 위한 데이터의 이력 및 명세서.
- **[Data Silo]**: 거버넌스가 부재할 때 발생하는 데이터 고립 및 파편화 현상.

📢 **마무리 요약**: **Data Governance**는 데이터 경제의 주춧돌입니다. 기술보다 어려운 '합의'와 '문화'를 만드는 과정이며, 이를 통해 데이터는 비로소 기업의 진정한 자산으로 거듭납니다.
