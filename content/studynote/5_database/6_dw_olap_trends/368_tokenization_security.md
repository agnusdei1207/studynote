# 368. 토큰화(Tokenization) 기술 - 가치 없는 값으로의 치환

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 토큰화는 민감한 데이터(카드번호 등)를 그 자체로는 아무런 비즈니스 가치가 없는 **임의의 난수 형태인 '토큰(Token)'으로 대체하여 저장하는 보안 기술**이다.
> 2. **가치**: 암호화와 달리 수학적 알고리즘이 아닌 '매핑 테이블' 방식을 사용하여 **복호화 키 탈취를 통한 원본 유출 위험을 원천 차단**하며, PCI-DSS와 같은 규제 준수 범위를 축소시킨다.
> 3. **융합**: 모바일 결제(Samsung Pay, Apple Pay), API 보안, 개인정보 비식별화 시스템과 융합되어 데이터 활용의 편의성과 보안성을 동시에 확보한다.

+++

### Ⅰ. 토큰화의 작동 매커니즘

1. **토큰 생성**: 원본 데이터가 입력되면 토큰 서버가 난수를 생성하여 매핑합니다.
2. **저장 및 유통**: 비즈니스 시스템(DB)에는 오직 '토큰'만 저장됩니다. 해커가 DB를 털어도 쓸모없는 난수만 얻게 됩니다.
3. **토큰 교환 (Detokenization)**: 결제 등 원본 데이터가 필요한 순간에만 권한이 있는 시스템이 토큰 서버에 요청하여 원본을 받아옵니다.

+++

### Ⅱ. 토큰화 아키텍처 시각화 (ASCII Flow)

```text
[ Tokenization Process ]

  (Real Data) : [ Credit Card: 1234-5678-xxxx ]
       │
       ▼ [ Token Vault (Secure Storage) ]
  ┌─────────────────────────────────────────┐
  │ Mapping Table: 1234-5678-xxxx ↔ T-998877 │ ✅
  └─────────────────────────────────────────┘
       │
       ▼ [ Business Database ]
  ┌─────────────────────────────────────────┐
  │ User_ID: 101 | Card_Token: T-998877     │ 🔒 (No value if stolen)
  └─────────────────────────────────────────┘
```

+++

### Ⅲ. 암호화(Encryption) vs 토큰화(Tokenization)

| 비교 항목 | 암호화 (Encryption) | 토큰화 (Tokenization) |
|:---|:---|:---|
| **복원 방식** | 알고리즘 + 키 (Mathematical) | **금고 매핑 (Table-based)** |
| **보안 강도** | 키 유출 시 위험 | **원본이 외부에 없어 더 안전함** |
| **데이터 형태** | 길이/형태가 변할 수 있음 | **원본과 동일한 형태 유지 가능** |
| **규제 영향** | 규제 대상에 포함되는 경우 많음 | **규제 준수 범위(Scope) 축소 가능** |

- **📢 섹션 요약 비유**: 토큰화는 **'카지노의 칩'**과 같습니다. 현금(민감 데이터)은 카지노 금고(Token Vault)에 꽉 잠가두고, 도박장(비즈니스 DB) 안에서는 오직 칩(토큰)만 가지고 놉니다. 밖에서 누군가 칩을 훔쳐 가도 카지노 밖에서는 아무런 가치가 없는 플라스틱 조각일 뿐이므로 돈을 잃어버릴 염려가 없는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Token Vault]**: 토큰과 원본의 매핑 정보를 보관하는 가장 강력한 보안 구역.
- **[Format Preserving]**: 원본의 길이나 형식을 유지하여 기존 시스템 수정을 최소화하는 기술.
- **[PCI-DSS]**: 카드 정보 보호를 위해 토큰화 도입을 강력히 권고하는 표준.

📢 **마무리 요약**: **Tokenization**은 데이터 보안의 패러다임을 '보호'에서 '치환'으로 바꿉니다. 데이터 자체의 가치를 제거함으로써 유출의 동기를 꺾어버리는 가장 스마트한 방어 전략입니다.
