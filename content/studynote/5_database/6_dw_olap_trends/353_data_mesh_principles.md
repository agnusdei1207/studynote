# 353. 데이터 메시(Data Mesh) 4대 원칙 - 분권화의 철학

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 메시(Data Mesh)는 거대한 중앙 집중형 데이터 팀의 병목을 해결하기 위해, **데이터 소유권과 관리 책임을 개별 비즈니스 도메인으로 분산시키는 조직 및 기술적 아키텍처**다.
> 2. **가치**: 데이터를 중앙에서 처리해야 할 '짐'이 아닌, 각 부서가 가치를 창출하여 제공하는 **'제품(Product)'**으로 정의함으로써 데이터 품질과 활용 속도를 극대화한다.
> 3. **융합**: 마이크로서비스(MSA)의 도메인 중심 설계(DDD) 사상을 데이터 세상으로 확장한 것이며, 현대의 복잡한 엔터프라이즈 데이터 거버넌스의 새로운 해법으로 융합된다.

+++

### Ⅰ. 데이터 메시의 4대 핵심 원칙 (Pillars)

1. **도메인 중심의 분권화된 데이터 소유권 (Domain Ownership)**:
    - 데이터를 가장 잘 아는 현업 부서(예: 마케팅팀)가 해당 데이터를 직접 관리하고 책임집니다.
2. **제품으로서의 데이터 (Data as a Product)**:
    - 데이터는 단순한 파일이 아니라, 다른 부서 사용자가 쉽게 발견하고 신뢰하며 사용할 수 있도록 잘 정제된 '제품' 형태(Data Product)로 제공되어야 합니다.
3. **셀프 서비스 데이터 플랫폼 (Self-serve Data Platform)**:
    - 각 부서가 데이터 인프라를 직접 구축할 필요 없이, 중앙 플랫폼 팀이 제공하는 도구들을 통해 스스로 데이터를 제품화하고 분석할 수 있어야 합니다.
4. **연합된 컴퓨팅 거버넌스 (Federated Computational Governance)**:
    - 분산된 도메인들이 서로 소통할 수 있도록 상호운용성, 보안, 품질에 대한 글로벌 표준을 중앙에서 수립하고 자동화된 기술로 강제합니다.

### Ⅲ. 실무적 의의: 왜 메시인가?

- **확장성(Scalability)**: 중앙 데이터 팀의 인원 수에 의존하지 않고, 비즈니스 부서가 늘어나는 만큼 데이터 분석 역량도 비례해서 늘어납니다.
- **데이터 품질(Agility)**: 데이터 발생지와 관리자가 일치하므로 데이터의 의미가 왜곡되지 않고 빠르게 정제됩니다.

- **📢 섹션 요약 비유**: 데이터 메시는 **'전통적인 대형 마트'에서 '재래시장'으로의 변화**와 같습니다. 마트(중앙 DW)는 물건 배치와 관리를 본사가 다 하지만 물건이 들어오는 데 시간이 걸립니다. 재래시장(메시)은 각 상점 주인(도메인)이 자기가 가장 잘 아는 전문 품목을 직접 골라와서 팔기 때문에 훨씬 신선하고 활기찬 것과 같습니다. 시장 전체의 규칙(거버넌스)만 잘 지키면 훨씬 생동감 넘치는 데이터 생태계가 됩니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Data Product]**: 메시 아키텍처에서 유통되는 가치 단위.
- **[DDD (Domain Driven Design)]**: 데이터 메시의 사상적 뿌리.
- **[Interoperability]**: 분산된 데이터 제품들을 서로 엮어 쓰기 위한 상호운용성.

📢 **마무리 요약**: **Data Mesh 4 Principles**는 데이터 관리의 패러다임을 바꿉니다. 통제(Control)에서 위임(Empowerment)으로의 전환을 통해 진정한 데이터 기반 조직을 완성합니다.
