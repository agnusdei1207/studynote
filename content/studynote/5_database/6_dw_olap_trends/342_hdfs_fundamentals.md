# 342. HDFS (Hadoop Distributed File System) - 빅데이터의 대형 창고

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: HDFS는 수십 테라바이트(TB) 이상의 거대 파일을 **블록(Block) 단위로 쪼개어 여러 대의 범용 서버에 분산 저장하고 관리하는 마스터-슬레이브 구조의 분산 파일 시스템**이다.
> 2. **가치**: 동일한 블록을 여러 노드에 복제(Default 3개) 저장하는 **내결함성(Fault Tolerance)** 기능을 통해, 일부 서버가 고장 나도 데이터 유실 없이 서비스를 지속하는 높은 신뢰성을 제공한다.
> 3. **융합**: '한 번 쓰고 여러 번 읽는(Write Once, Read Many)' 대규모 데이터 분석 워크로드에 최적화되어 있으며, 맵리듀스나 스파크와 같은 분산 처리 엔진의 물리적 토대가 된다.

+++

### Ⅰ. HDFS의 아키텍처 구성 요소

1. **NameNode (마스터)**: 
    - 파일 시스템의 **메타데이터**(파일명, 디렉토리 구조, 블록 위치 등)를 관리합니다. 
    - 모든 데이터 노드의 상태를 감시하며 작업 지시를 내립니다.
2. **DataNode (슬레이브)**: 
    - 실제 **데이터 블록**이 저장되는 물리적 노드입니다. 
    - 주기적으로 네임노드에 안부(Heartbeat)를 전하고 데이터를 읽기/쓰기 합니다.
3. **Block (블록)**: 
    - HDFS가 데이터를 나누는 기본 단위 (보통 128MB). 큰 블록 크기는 탐색 비용을 줄여줍니다.

+++

### Ⅱ. HDFS 데이터 분산 및 복제 구조 (ASCII Model)

```text
[ HDFS Replication Architecture ]

      [ NameNode ] (Metadata: "File1 -> B1, B2")
           │
    ┌──────┴────────────────┬────────────────┐
    ▼                       ▼                ▼
 [ DataNode A ]          [ DataNode B ]   [ DataNode C ]
 ┌────────────┐          ┌────────────┐   ┌────────────┐
 │  Block 1   │ (Copy1)  │  Block 1   │   │  Block 2   │
 │  Block 2   │ (Copy1)  │  Block 2   │   │  Block 1   │ (Copy3)
 └────────────┘          └────────────┘   └────────────┘
   (Copy 1)                (Copy 2)
```

+++

### Ⅲ. HDFS의 주요 동작 원리

- **데이터 읽기**: 클라이언트가 네임노드에 위치를 묻고, 가장 가까운(Network Topology) 데이터노드에서 직접 데이터를 가져옵니다.
- **데이터 쓰기**: 클라이언트가 네임노드로부터 저장할 노드 목록을 받고, 파이프라인(Pipeline) 방식으로 노드 간에 데이터를 복제하며 씁니다.
- **자동 복구**: 데이터노드가 죽으면 네임노드가 이를 감지하고, 부족해진 복제본을 다른 살아있는 노드에 새로 생성하라고 지시합니다.

- **📢 섹션 요약 비유**: HDFS는 **'은행의 금고 시스템'**과 같습니다. 네임노드는 '장부와 열쇠를 쥔 지점장'이고, 데이터노드는 '실제 돈이 든 수많은 금고'입니다. 지점장은 돈을 여러 금고에 나누어 보관(복제)하게 함으로써, 금고 하나가 망가져도 손님의 돈을 안전하게 지켜낼 수 있게 관리합니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Secondary NameNode]**: 네임노드의 체크포인트를 관리하는 보조 노드.
- **[Rack Awareness]**: 같은 건물이 아닌 다른 건물(랙)에 복제본을 두어 재해에 대비하는 전략.
- **[High Availability]**: 네임노드 이중화를 통해 마스터 장애를 극복하는 기술.

📢 **마무리 요약**: **HDFS**는 빅데이터의 기초 체력입니다. "하드웨어는 언제든 고장 날 수 있다"는 전제하에 설계된 이 시스템 덕분에 우리는 데이터의 바다를 안전하게 항해할 수 있습니다.
