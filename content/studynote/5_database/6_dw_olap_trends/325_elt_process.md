# 325. ELT (Extraction, Loading, Transformation) - 클라우드 기반의 혁신

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ELT는 데이터를 추출(Extract)한 뒤 변환하지 않고 **목적지인 데이터 레이크나 클라우드 DW에 먼저 적재(Load)하고, 이후 타겟 시스템의 강력한 컴퓨팅 자원을 활용해 변환(Transform)하는 방식**이다.
> 2. **가치**: 중간 변환 서버의 병목을 없애 대량의 비정형 데이터를 초고속으로 수집할 수 있으며, 원천 데이터를 그대로 보존하므로 필요할 때 언제든 다시 다양한 방식으로 변환할 수 있는 유연성을 제공한다.
> 3. **융합**: Snowflake, BigQuery와 같은 클라우드 네이티브 DW와 dbt(Data Build Tool) 같은 최신 변환 프레임워크와 결합되어 현대적 데이터 스택(Modern Data Stack)의 표준으로 자리 잡았다.

+++

### Ⅰ. ETL vs ELT 상세 비교

| 비교 항목 | ETL (전통적) | ELT (현대적) |
|:---|:---|:---|
| **처리 순서** | 추출 -> **변환** -> 적재 | 추출 -> **적재** -> **변환** |
| **변환 주체** | 별도 ETL 서버 | **데이터베이스 본체** (Target) |
| **데이터 유연성** | 정제된 데이터만 보존 | **원천 데이터(Raw) 보존** |
| **주요 환경** | 온프레미스 RDBMS | **클라우드 데이터 레이크/DW** |
| **속도** | 소량 정밀 처리에 유리 | **대량 고속 적재에 유리** |

+++

### Ⅱ. ELT 프로세스 시각화 (ASCII Flow)

```text
[ ELT Pipeline Architecture ]

  (Sources)          (Target System: BigQuery/Snowflake)
  ┌────────┐        ┌────────────────────────────────────┐
  │ RDBMS  │ ──(E)─▶│ [ Raw Data Layer (S3/GCS) ]        │
  ├────────┤        │ (원시 데이터 그대로 저장)            │
  │ Log    │ ──(E)─▶│          │                         │
  └────────┘        │          ▼ (T: In-DB Processing)   │
                    │ [ Refined Data / Analytics View ]  │
                    └────────────────────────────────────┘
```

+++

### Ⅲ. ELT가 각광받는 이유

1. **클라우드 컴퓨팅 파워**: 과거엔 DB에서 변환을 돌리면 업무 성능이 떨어졌지만, 지금은 클라우드의 무한한 자원을 빌려 순식간에 처리가 가능합니다.
2. **비정형 데이터의 폭증**: 일단 다 쌓아두고(Load) 나중에 용도에 맞게 분석(Transform)하는 것이 훨씬 경제적입니다.
3. **Data Lineage**: 원본 데이터가 타겟에 살아있으므로 분석 결과의 근거를 추적하기 매우 쉽습니다.

- **📢 섹션 요약 비유**: ELT는 **'대형 뷔페의 즉석 요리'**와 같습니다. 주방에서 미리 다 만들어 내놓는 것(ETL)이 아니라, 신선한 재료를 손님 앞 테이블에 먼저 가져다 놓고(Load), 손님이 원하는 시점에 원하는 방식대로 즉석에서 조리(Transform)해 먹는 것과 같습니다. 재료가 신선하게 유지(Raw Data)되므로 언제든 다른 요리로 바꿀 수 있는 장점이 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Modern Data Stack]**: ELT를 기반으로 한 현대적 분석 환경.
- **[dbt]**: SQL만으로 ELT의 변환(T) 과정을 관리하는 도구.
- **[Schema-on-Read]**: 적재 시가 아닌 읽는 시점에 스키마를 정의하는 ELT의 철학.

📢 **마무리 요약**: **ELT**는 클라우드 시대의 데이터 처리 문법입니다. 인프라의 한계를 기술로 극복하여, "일단 저장하고 나중에 고민하라"는 데이터 중심의 혁신을 가능케 합니다.
