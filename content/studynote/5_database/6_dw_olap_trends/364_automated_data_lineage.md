# 364. 데이터 리니지 자동화 - 보이지 않는 흐름의 자동 추적

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 리니지 자동화는 SQL 파싱, 로그 분석, API 가로채기 등을 통해 **수작업 없이 데이터의 발생, 변환, 이동 경로를 실시간으로 탐지하고 맵을 그리는 기술**이다.
> 2. **가치**: 수만 개의 테이블과 파이프라인이 얽힌 거대 시스템에서도 **휴먼 에러 없는 정확한 영향도 분석(Impact Analysis)**을 가능케 하며, 규제 준수(Compliance)를 위한 투명성을 즉각 확보한다.
> 3. **융합**: AI 기반의 데이터 카탈로그 기술과 융합되어, 데이터의 흐름뿐만 아니라 '의미적 연관성'까지 자동으로 파악하는 지능형 거버넌스의 핵심으로 진화하고 있다.

+++

### Ⅰ. 리니지 자동화의 핵심 기술 방식

1. **SQL 파싱 (SQL Parsing)**:
    - 데이터베이스의 저장 프로시저, 뷰, 쿼리 로그를 분석하여 `SELECT`, `INSERT INTO` 구문 속의 테이블 간 관계를 자동으로 추출합니다.
2. **메타데이터 하베스팅 (Metadata Harvesting)**:
    - ETL 도구(Informatica, Talend 등)나 데이터 오케스트레이터(Airflow)의 실행 로그를 수집하여 작업 간 선후 관계를 파악합니다.
3. **런타임 관측 (Runtime Observability)**:
    - 데이터가 네트워크를 통해 이동할 때 패킷을 분석하거나 에이전트를 심어 실제 데이터의 흐름을 실시간 캡처합니다.

+++

### Ⅱ. 자동화된 리니지 생성 아키텍처 (ASCII Flow)

```text
[ Automated Lineage Pipeline ]

  (Data Sources) ──▶ [ Activity Logs / Query Logs ]
                            │
                            ▼
  [ Lineage Engine ] ◀── [ SQL Parser / Regex ]
          │                 (추출 및 분석)
          ▼
  [ Graph Metadata ] ──▶ [ Visual Lineage Map ]
  (Nodes: Tables)        ┌───┐    ┌───┐    ┌───┐
  (Edges: Data Flow)     │Src│──▶ │ETL│──▶ │Tgt│ ✅ "Auto-mapped"
                         └───┘    └───┘    └───┘
```

### Ⅲ. 수동 리니지 vs 자동 리니지

| 비교 항목 | 수동 리니지 관리 | 자동 리니지 관리 |
|:---|:---|:---|
| **정확도** | 관리자의 기억에 의존 (부정확) | **시스템 실측 기반 (매우 정확)** |
| **최신성** | 업데이트 누락 빈번 | **실시간 변경 즉시 반영** |
| **확장성** | 소규모 시스템만 가능 | **엔터프라이즈급 대규모 대응 가능** |
| **비용** | 인건비 지속 발생 | 초기 솔루션 구축 비용 발생 |

- **📢 섹션 요약 비유**: 데이터 리니지 자동화는 **'실시간 GPS 경로 기록기'**와 같습니다. 예전에는 여행이 끝나고 기억을 되짚어 지도를 그렸다면(수동), 이제는 차를 운전하는 내내 GPS가 내 위치를 기록하여 내가 어느 길로 와서 어디에 멈췄는지 완벽한 경로도(자동 리니지)를 실시간으로 완성해 주는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Impact Analysis]**: 리니지 자동화로 얻는 가장 큰 실무적 혜택.
- **[Data Catalog]**: 자동화된 리니지가 시각화되어 저장되는 공간.
- **[OpenLineage]**: 리니지 정보 공유를 위한 글로벌 오픈 표준 규격.
