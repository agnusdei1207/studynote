# 335. 대리 키 (Surrogate Key) - 안정적인 데이터의 식별자

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 대리 키(Surrogate Key)는 자연 키(Natural Key)와 달리 비즈니스적 의미가 전혀 없는, **시스템이 자동으로 생성한 단조 증가하는 일련번호 형태의 인공 키**다.
> 2. **가치**: 운영 시스템의 키 값이 바뀌거나 재사용되더라도 데이터 웨어하우스(DW) 내의 이력 데이터를 안전하게 보존하며, 정수형 타입을 사용하여 조인(Join) 성능을 극대화한다.
> 3. **융합**: 느리게 변하는 차원(SCD) 관리 기술과 결합되어 동일한 비즈니스 키를 가진 객체의 시점별 변화를 서로 다른 행으로 분리 식별하는 중추적인 역할을 한다.

+++

### Ⅰ. 자연 키 vs 대리 키 상세 비교

| 비교 항목 | 자연 키 (Natural Key) | 대리 키 (Surrogate Key) |
|:---|:---|:---|
| **의미** | 비즈니스 의미 존재 (주민번호, 상품코드) | **의미 없음 (순번, UUID)** |
| **변경 가능성** | 비즈니스 정책에 따라 변경 가능 | **절대 변경되지 않음** |
| **데이터 타입** | 문자열이 많음 (복합 키 가능) | **단순 정수형 (Integer)** |
| **DW 적합성** | 낮음 (이력 관리 취약) | **매우 높음 (표준 방식)** |

+++

### Ⅱ. 대리 키의 이력 관리 시각화 (ASCII Flow)

운영 DB의 상품코드(Natural Key)는 하나지만, DW에서는 대리 키를 통해 변화 과정을 추적합니다.

```text
[ Slowly Changing Dimension with Surrogate Key ]

  (Source DB) : Prod_CD: "A100" | Price: 5000 ──▶ 6000 (Update)

  (DW Dim_Product Table)
  ┌────────┬─────────┬────────┬──────────┬──────────┐
  │ SK(PK) │ Prod_CD │ Price  │ Start_DT │ End_DT   │
  ├────────┼─────────┼────────┼──────────┼──────────┤
  │   1    │  A100   │ 5000   │ 2025-01  │ 2026-02  │ (Old)
  │   2    │  A100   │ 6000   │ 2026-03  │ 9999-12  │ (Current) ✅
  └────────┴─────────┴────────┴──────────┴──────────┘
  * SK(Surrogate Key) 덕분에 두 버전의 상품 정보를 동시에 식별 가능.
```

+++

### Ⅲ. 대리 키 도입의 장점

1. **데이터 독립성**: 소스 시스템의 데이터 통합이나 분할 시 발생하는 식별자 충돌을 원천 차단합니다.
2. **성능 향상**: 수억 건의 팩트 테이블과 차원 테이블을 조인할 때, 긴 문자열보다는 고정 크기 정수(Integer)를 쓰는 것이 훨씬 빠릅니다.
3. **이력 관리 (Time-Travel)**: 동일한 자연 키에 대해 여러 개의 대리 키를 생성함으로써 시점별 스냅샷 관리가 가능해집니다.

- **📢 섹션 요약 비유**: 대리 키는 **'군대의 군번'**과 같습니다. 사회에서의 이름(자연 키)은 개명할 수도 있고 동명이인이 있을 수도 있지만, 군대 시스템 안에서 부여받은 군번(대리 키)은 평생 변하지 않고 나를 유일하게 증명해 줍니다. 시스템은 내 이름이 무엇으로 바뀌든 군번으로 나를 관리하므로 혼란이 생기지 않습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Identity]**: 대리 키를 생성하는 DB 엔진의 일반적인 기능.
- **[Integration]**: 서로 다른 시스템의 데이터를 합칠 때 대리 키가 빛을 발함.
- **[Natural Key Map]**: 대리 키와 자연 키 사이의 연결 정보를 가진 매핑 테이블.

📢 **마무리 요약**: **Surrogate Key**는 데이터 웨어하우스의 뼈대입니다. 비즈니스의 변화로부터 분석 시스템을 독립시켜, 흔들리지 않는 데이터의 역사(History)를 기록하게 합니다.
