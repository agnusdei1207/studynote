# 343. YARN (Yet Another Resource Negotiator) - 하둡의 운영체제

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: YARN은 하둡 2.0부터 도입된 **분산 자원 관리 및 작업 스케줄링 프레임워크**로, 클러스터 전체의 컴퓨팅 자원(CPU, 메모리)을 통합 관리하고 여러 애플리케이션에 효율적으로 배분하는 역할을 한다.
> 2. **가치**: 맵리듀스(MapReduce)뿐만 아니라 스파크(Spark), 하이브(Hive) 등 다양한 데이터 처리 엔진이 하나의 하둡 클러스터를 공유할 수 있게 하여 **클러스터 활용도와 유연성**을 획기적으로 높였다.
> 3. **융합**: 자원 할당을 담당하는 리소스매니저(RM)와 개별 작업을 관리하는 애플리케이션마스터(AM)를 분리한 아키텍처를 통해 대규모 클러스터의 확장성과 안정성을 동시에 융합해 냈다.

+++

### Ⅰ. YARN의 4대 핵심 구성 요소

1. **ResourceManager (RM, 중앙 마스터)**: 클러스터 전체의 자원을 관리하며 어떤 작업에 자원을 줄지 결정하는 '중앙 정부'입니다.
2. **NodeManager (NM, 각 노드 에이전트)**: 각 서버에서 실행되며, 해당 서버의 자원 사용량을 모니터링하고 컨테이너를 관리하는 '지방 정부'입니다.
3. **ApplicationMaster (AM, 작업별 관리자)**: 각 작업(Job)마다 하나씩 생성되어 자원을 요청하고 실행 과정을 총괄하는 '현장 소장'입니다.
4. **Container (컨테이너)**: CPU, 메모리 등 자원이 할당된 논리적인 작업 단위입니다.

+++

### Ⅱ. YARN 작업 수행 프로세스 (ASCII Flow)

```text
[ YARN Resource Management Flow ]

  (Client) ──▶ [ ResourceManager ] (자원 스케줄링)
                     │
          ┌──────────┴──────────┐
          ▼                     ▼
  [ NodeManager A ]      [ NodeManager B ]
  ┌─────────────┐        ┌─────────────┐
  │ [ AppMaster ]│ ──▶   │ [ Container ]│ (Task 1)
  │ [ Container ]│ (Task)│ [ Container ]│ (Task 2)
  └─────────────┘        └─────────────┘
```

+++

### Ⅲ. YARN 도입의 혁신: 범용 분산 플랫폼

- **멀티 테넌시 (Multi-tenancy)**: 하나의 클러스터에서 배치(Batch) 작업과 실시간(Real-time) 분석 작업을 동시에 수행할 수 있습니다.
- **확장성 개선**: 하둡 1.0의 잡트래커(JobTracker)가 가졌던 병목 현상을 RM과 AM의 역할 분리를 통해 해결했습니다.
- **다양한 엔진 지원**: 맵리듀스 외에도 사용자가 직접 정의한 어떠한 분산 애플리케이션도 YARN 위에서 실행 가능합니다.

- **📢 섹션 요약 비유**: YARN은 **'컴퓨터의 운영체제(OS)'**와 같습니다. 예전 하둡이 엑셀(MapReduce)만 돌아가는 전용기였다면, YARN이 도입된 하둡은 엑셀, 포토샵(Spark), 인터넷(Hive) 등 여러 프로그램을 동시에 띄우고 메모리와 CPU를 골고루 나눠줄 수 있게 된 최신 윈도우나 맥 OS와 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Scheduler]**: RM 내에서 자원 배분 정책(Fair, Capacity 등)을 수행하는 모듈.
- **[Isolation]**: 컨테이너를 통해 서로 다른 작업 간의 자원 간섭을 막는 기술.
- **[Hadoop 2.0]**: YARN이 등장하여 하둡의 르네상스를 이끈 시점.

📢 **마무리 요약**: **YARN**은 분산 컴퓨팅의 효율을 극대화하는 지휘자입니다. 자원의 조율과 통제를 통해 하둡을 단순한 저장소를 넘어 만능 분석 플랫폼으로 격상시켰습니다.
