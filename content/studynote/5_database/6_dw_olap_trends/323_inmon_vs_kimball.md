# 323. Inmon vs Kimball - 데이터 웨어하우스 설계의 양대 철학

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 빌 인먼(Bill Inmon)은 전사적 데이터를 먼저 3정규형(3NF)으로 통합하는 **'Top-down(하향식)'** 접근을 주장하며, 랄프 킴벌(Ralph Kimball)은 부서별 데이터 마트를 먼저 스타 스키마로 구축하는 **'Bottom-up(상향식)'** 접근을 지향한다.
> 2. **가치**: 인먼 방식은 데이터 정합성과 유지보수성이 뛰어나 대기업의 장기적 인프라에 유리하고, 킴벌 방식은 구축이 빠르고 사용자 분석 편의성이 높아 실무 중심의 신속한 성과 도출에 최적화되어 있다.
> 3. **융합**: 현대 기업은 중앙에 인먼식 통합 저장소(EDW)를 두고, 그 위에 킴벌식 차원 모델링(DM)을 얹는 하이브리드 아키텍처로 두 거장의 이론을 융합하여 사용한다.

+++

### Ⅰ. 두 아키텍처의 상세 비교

| 비교 항목 | Inmon (Corporate Information Factory) | Kimball (Dimensional Data Warehouse) |
|:---|:---|:---|
| **설계 방식** | **Top-down (하향식)** | **Bottom-up (상향식)** |
| **데이터 모델** | ER 모델링 (3정규형 중심) | **다차원 모델링 (스타 스키마 중심)** |
| **중심 저장소** | 전사적 데이터 웨어하우스 (EDW) | 데이터 마트의 연합 (Bus Architecture) |
| **데이터 마트** | EDW로부터 생성 (종속적) | 물리적 DW 자체가 마트의 결합 |
| **장점** | 데이터 중복 없음, 정합성 최상 | 구축 속도 빠름, 사용하기 쉬움 |
| **단점** | 초기 비용 높음, 구축 기간 김 | 데이터 중복 발생, 전사 통합 어려움 |

+++

### Ⅱ. 아키텍처 시각화 (ASCII Flow)

```text
[ Inmon (하향식) ]                   [ Kimball (상향식) ]

  (Source)                             (Source)
     │                                    │
     ▼                                    ▼
  [ EDW ] (3NF 통합)                   [ Data Marts ] (Star Schema)
     │                                    │
  ┌──┴──┐ (추출)                       ┌──┴──┐ (공유 차원)
  ▼     ▼                              ▼     ▼
 [DM1] [DM2]                          [  Bus Architecture  ]
```

### Ⅲ. 실무적 선택 가이드

- **Inmon 선택 시**: 데이터의 일관성이 절대적으로 중요하고, 대규모 조직에서 수년 이상의 장기 프로젝트를 추진할 때.
- **Kimball 선택 시**: 특정 부서의 분석 니즈가 시급하고, 변화가 잦은 비즈니스 환경에서 빠른 ROI(투자 대비 효과)가 필요할 때.

- **📢 섹션 요약 비유**: 인먼 방식은 **'도시 전체의 거대한 상수도망'**을 먼저 설계하고 집집마다 물을 대는 꼼꼼한 도시 계획이고, 킴벌 방식은 **'필요한 집 옆에 우물'**을 파서 당장 물을 마시고 나중에 우물들을 수로로 잇는 실용적인 개척 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[EDW]**: 인먼 아키텍처의 핵심 저장소.
- **[Conformed Dimension]**: 킴벌 방식에서 서로 다른 마트를 잇는 고리.
- **[Hybrid Architecture]**: 두 방식의 장점만 섞은 현대의 절충안.

📢 **마무리 요약**: **Inmon과 Kimball**은 데이터의 정석(인먼)과 실전(킴벌)을 대변합니다. 조직의 자금력, 인력, 목표 기간을 고려하여 두 철학의 적절한 균형을 잡는 것이 아키텍트의 예술입니다.
