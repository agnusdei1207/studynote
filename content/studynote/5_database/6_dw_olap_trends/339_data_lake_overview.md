# 339. 데이터 레이크 (Data Lake) - 가공되지 않은 기회의 바다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 레이크는 정형, 반정형, 비정형 데이터를 가리지 않고 **원본 형태 그대로(Raw Format) 대규모로 저장하는 중앙 집중식 저장소**다.
> 2. **가치**: 데이터를 담기 전에 미리 스키마를 정의할 필요가 없는 **'Schema-on-Read'** 방식을 택하여, 데이터 수집의 유연성을 극대화하고 미래의 분석 가능성을 무한히 열어둔다.
> 3. **융합**: Hadoop(HDFS)이나 클라우드 오브젝트 스토리지(S3 등)와 융합되어 저비용 고효율의 빅데이터 기반을 형성하며, 데이터 사이언티스트들의 머신러닝 학습을 위한 원천 데이터 공급원 역할을 한다.

+++

### Ⅰ. 데이터 레이크의 주요 특징

- **저장 형식의 자유**: SQL 테이블뿐만 아니라 JSON, 로그, 이미지, PDF 등 모든 형태를 수용합니다.
- **경제성**: 고가의 고성능 DB 서버 대신, 저렴한 범용 디스크나 오브젝트 스토리지를 사용하여 테라바이트급 이상의 데이터를 장기 보관합니다.
- **접근성**: 데이터 전처리(ETL) 없이 일단 다 쌓아두므로, 데이터 발생과 수집 사이의 지연 시간이 거의 없습니다.

+++

### Ⅱ. DW vs 데이터 레이크 아키텍처 비교 (ASCII)

```text
[ Data Warehouse (정돈된 정원) ]      [ Data Lake (거대한 호수) ]

  ┌───┐ ┌───┐ ┌───┐                  ┌───┐ ┌───┐ ┌───┐
  │SQL│ │CSV│ │Log│ (Sources)        │SQL│ │CSV│ │Log│ (Sources)
  └───┘ └───┘ └───┘                  └───┘ └───┘ └───┘
    │     │     │                      │     │     │
    ▼ (Schema-on-Write)                ▼ (Schema-on-Read)
  ┌─────────────────┐                ┌~~~~~~~~~~~~~~~~~┐
  │  Refined Data   │                │  Raw Data Pool  │
  │  (Warehouse)    │                │  (Lake Storage) │
  └─────────────────┘                └~~~~~~~~~~~~~~~~~┘
```

+++

### Ⅲ. 데이터 레이크 구축 시 주의사항

- **거버넌스 부재의 위험**: 무엇이 들어있는지 기록(메타데이터)하지 않으면 나중에 찾을 수 없는 **데이터 늪 (Data Swamp)**으로 변질됩니다.
- **보안 관리**: 데이터가 원본 그대로 노출되므로, 민감 정보에 대한 접근 제어와 마스킹 처리가 더욱 중요합니다.
- **활용 역량**: 정제되지 않은 데이터를 다룰 수 있는 고도의 데이터 엔지니어링 기술이 요구됩니다.

- **📢 섹션 요약 비유**: 데이터 레이크는 **'편집 전의 영상 원본 데이터'**와 같습니다. 완성된 다큐멘터리(DW)는 보기 편하지만 잘려 나간 부분이 많습니다. 원본 영상(Lake)은 길고 지루할 수 있어도, 나중에 감독이 바뀌었을 때 전혀 다른 영화(새로운 분석 결과)를 만들어낼 수 있는 무궁무진한 재료가 됩니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Schema-on-Read]**: 읽는 시점에 스키마를 입히는 레이크의 철학.
- **[Object Storage]**: 클라우드 시대 데이터 레이크의 물리적 기반.
- **[ELT]**: 데이터 레이크 환경에 가장 적합한 통합 방식.

📢 **마무리 요약**: **Data Lake**는 현대 데이터 아키텍처의 필수 관문입니다. "일단 다 담아라"는 과감한 전략을 통해 데이터로부터 예상치 못한 혁신을 끌어내는 마법의 호수입니다.
