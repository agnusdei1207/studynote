# 344. Apache Hive - 빅데이터 위의 SQL 엔진

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Apache Hive는 페이스북에서 고안한 데이터 웨어하우스 소프트웨어로, **하둡(Hadoop)에 저장된 대규모 데이터를 SQL과 유사한 쿼리 언어인 HiveQL을 사용하여 분석할 수 있게 해주는 도구**다.
> 2. **가치**: 복잡한 자바(Java) 기반의 맵리듀스 코드를 작성할 필요 없이 **친숙한 SQL 문법**만으로 빅데이터 분석이 가능하게 하여, 데이터 분석가들의 진입 장벽을 획기적으로 낮췄다.
> 3. **융합**: HDFS의 분산 저장 능력과 맵리듀스/Tez/Spark 등의 실행 엔진을 융합하여, 수 페타바이트급 데이터에 대한 구조화된 질의와 리포팅을 수행하는 엔터프라이즈 DW의 핵심으로 활용된다.

+++

### Ⅰ. Hive의 주요 구성 요소

1. **HiveQL**: SQL-92 표준을 따르는 하이브 전용 질의 언어입니다.
2. **Driver**: 쿼리를 입력받아 세션을 관리하고 실행 계획을 수립합니다.
3. **Metastore**: 테이블의 구조(스키마), 컬럼명, 데이터 위치 등을 담은 메타데이터 저장소입니다. (보통 RDBMS 사용)
4. **Execution Engine**: HiveQL을 실제 분산 작업(MapReduce 등)으로 변환하여 실행합니다.

+++

### Ⅱ. Hive 쿼리 실행 아키텍처 (ASCII Flow)

```text
[ Hive Query Pipeline ]

  (User) ──▶ [ Hive Interface (CLI/JDBC) ]
                   │ (HiveQL: SELECT * FROM ...)
                   ▼
            [ Hive Driver ] ◀──(Schema Check)── [ Metastore ]
                   │ (Compiler/Optimizer)
                   ▼
            [ Execution Engine ] ──▶ (Submit Job) ──▶ [ YARN / Cluster ]
                   │                                       │
                   └───────────────────────────────────────┘
                                       ▼
                                [ HDFS Data ] ✅
```

+++

### Ⅲ. Hive의 특징과 진화

- **Schema on Read**: 데이터를 저장할 때가 아니라 읽을 때 스키마를 적용하므로 고속 적재가 가능합니다.
- **배치 처리 최적화**: 실시간 쿼리보다는 대규모 데이터를 한꺼번에 처리하는 배치 분석에 특화되어 있습니다.
- **성능 개선**: 초기엔 맵리듀스만 썼으나, 최근엔 Apache Tez나 Spark 엔진을 사용하여 실행 속도를 수십 배 향상시켰습니다.

- **📢 섹션 요약 비유**: 하이브는 **'빅데이터용 번역기'**와 같습니다. 하둡이라는 거대한 창고에는 기계어(MapReduce)만 알아듣는 일꾼들이 사는데, 분석가(사용자)가 익숙한 영어(SQL)로 명령을 내리면 하이브가 이를 기계어로 번역해서 일꾼들에게 시키고 결과만 가져다주는 아주 똑똑한 통역사 역할을 합니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Metastore]**: 하이브의 기억 장치 (스키마 정보).
- **[Partitioning & Bucketing]**: 하이브에서 쿼리 성능을 높이기 위한 데이터 물리적 분할 기법.
- **[LLAP]**: 하이브 2.0부터 도입된 실시간 쿼리 가속 기술.

📢 **마무리 요약**: **Apache Hive**는 SQL의 대중성을 빅데이터의 세계로 이식한 혁명적 도구입니다. 하둡 생태계에서 가장 널리 쓰이는 표준 분석 레이어로 자리 잡고 있습니다.
