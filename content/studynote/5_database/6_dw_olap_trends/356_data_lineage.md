# 356. 데이터 리니지 (Data Lineage) - 데이터의 가계도를 그리다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 리니지는 데이터의 생성 시점부터 가공, 이동, 최종 소비에 이르기까지의 **전체 경로와 변화 과정을 시각화하여 추적하는 기술**이다.
> 2. **가치**: "이 보고서의 숫자는 어디서 왔는가?"라는 질문에 명확한 근거를 제시하여 **데이터 신뢰성**을 확보하고, 소스 변경 시 영향도 분석(Impact Analysis)을 신속히 수행하게 한다.
> 3. **융합**: 메타데이터 관리 시스템 및 ETL 도구와 융합되어, 복잡하게 얽힌 현대 기업의 데이터 파이프라인을 투명하게 관리하는 데이터 거버넌스의 핵심 도구로 작용한다.

+++

### Ⅰ. 데이터 리니지의 주요 관리 대상

- **소스 및 타겟**: 데이터가 어디서 시작되어 어디로 흘러갔는가?
- **변환 로직 (Transformation)**: 중간 단계에서 어떤 함수나 집계 연산이 적용되었는가?
- **의존성 (Dependency)**: 특정 테이블이 변경될 때 영향을 받는 하위 보고서나 API는 무엇인가?
- **사용 주체**: 누가 이 데이터를 최종적으로 조회하고 활용하고 있는가?

+++

### Ⅱ. 데이터 리니지 시각화 (ASCII Flow)

```text
[ Data Journey: Lineage Trace ]

  (Source DB)        (Stage/ETL)         (DW/Mart)         (Consumption)
  ┌──────────┐      ┌───────────┐      ┌───────────┐      ┌─────────────┐
  │ Customer │ ──▶  │  Merge &  │ ──▶  │ Sales     │ ──▶  │ Monthly BI  │
  │ Master   │      │  Cleanse  │      │ Fact Table│      │ Dashboard   │
  └──────────┘      └─────┬─────┘      └───────────┘      └─────────────┘
                          │
  ┌──────────┐            │            [ Analysis ]
  │ Order    │ ───────────┘            "If Order Table changes, 
  │ History  │                          BI Dashboard is IMPACTED!" ✅
  └──────────┘
```

+++

### Ⅲ. 데이터 리니지의 비즈니스 효용

1. **데이터 신뢰성 검증**: 보고서의 수치가 의심될 때, 역방향으로 추적(Back-tracing)하여 원천 데이터의 오류를 찾아낼 수 있습니다.
2. **장애 전파 차단**: 상위 시스템 장애 발생 시 "어느 데이터까지 오염되었는가"를 즉시 파악하여 후속 조치를 취할 수 있습니다.
3. **컴플라이언스 증명**: 감사 기관에 데이터 처리 과정의 투명성을 입증하는 강력한 근거 자료가 됩니다.

- **📢 섹션 요약 비유**: 데이터 리니지는 **'식재료의 이력 추적제'**와 같습니다. 우리가 먹는 스테이크(최종 보고서)가 어느 농장(소스 DB)에서 왔는지, 도축 과정(ETL)에서 어떤 검사를 거쳤는지 한눈에 보여주는 것입니다. 만약 고기가 상했다면(데이터 오류), 리니지를 통해 어느 단계에서 문제가 생겼는지 즉시 찾아낼 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Root Cause Analysis]**: 장애의 근본 원인을 찾는 과정.
- **[Impact Analysis]**: 상류의 변화가 하류에 미치는 영향을 분석하는 것.
- **[Data Catalog]**: 리니지 정보를 포함하여 데이터 자산을 검색하게 돕는 시스템.

📢 **마무리 요약**: **Data Lineage**는 데이터의 족보입니다. 거대해진 데이터 생태계 속에서 길을 잃지 않고 데이터의 진실을 끝까지 추적하게 돕는 네비게이션입니다.
