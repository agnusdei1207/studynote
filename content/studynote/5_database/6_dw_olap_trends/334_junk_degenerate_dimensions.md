# 334. 정크 차원(Junk)과 퇴화 차원(Degenerate) - 모델링의 최적화

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 정크 차원은 여러 개의 잡다한 속성(플래그, 인디케이터)을 하나의 테이블로 묶은 것이며, 퇴화 차원은 별도 테이블 없이 **팩트 테이블 내에 직접 보관되는 식별 속성**이다.
> 2. **가치**: 팩트 테이블의 컬럼 수를 줄여 모델을 단순화(정크 차원)하고, 불필요한 조인을 제거하여 시스템 성능을 향상(퇴화 차원)시킨다.
> 3. **융합**: 데이터 웨어하우스의 복잡성을 관리하는 킴벌(Kimball) 방법론의 정제 기술이며, 설계 효율성과 분석 성능 사이의 최적 균형을 찾기 위해 활용된다.

+++

### Ⅰ. 정크 차원 (Junk Dimension)

- **정의**: '예/아니오', '성공/실패' 등 값의 범위가 작고 서로 독립적인 여러 속성들을 하나의 차원으로 통합한 테이블입니다.
- **필요성**: 이런 속성들을 각각의 차원으로 만들면 팩트 테이블에 외래키(FK)가 너무 많아져 관리가 힘들어집니다 (FK Explosion 방지).
- **구성**: 가능한 모든 조합을 미리 행으로 생성해 둡니다.

+++

### Ⅱ. 퇴화 차원 (Degenerate Dimension)

- **정의**: 주문번호, 영수증번호처럼 그 자체로 고유한 식별자이지만, 연관된 속성이 없어 별도 차원 테이블을 만들 필요가 없는 속성입니다.
- **특징**: 팩트 테이블의 컬럼으로 직접 존재하며, 데이터의 원천(Origin)을 추적하는 용도로 주로 쓰입니다.

+++

### Ⅲ. 차원 모델링 최적화 시각화 (ASCII Model)

```text
[ Fact_Sales Table Before & After ]

  (Before: 복잡함)              (After: 최적화)
  ┌──────────────────┐         ┌──────────────────┐
  │ Order_No         │ ──▶     │ Order_No (퇴화)   │ ✅ "Direct ID"
  │ Payment_Flag_FK  │         ├──────────────────┤
  │ Ship_Status_FK   │ ──▶     │ Junk_ID (FK)     │ ✅ "Combined Flags"
  │ ... 10개 더 있음  │         └──────────────────┘
  └──────────────────┘                  │
                                        ▼
                                 [ Dim_Junk ]
                                 - ID
                                 - Payment_Flag (Y/N)
                                 - Ship_Status (Done/Wait)
```

+++

### Ⅲ. 실무적 의의

- **정크 차원**: 팩트 테이블의 인덱스 개수를 획기적으로 줄여 삽입 및 조회 성능을 보정합니다.
- **퇴화 차원**: 별도의 차원 테이블 조인 없이도 "어느 주문에서 발생한 매출인가?"를 즉시 알 수 있게 합니다.

- **📢 섹션 요약 비유**: 정크 차원은 **'다용도 정리함'**과 같습니다. 집안 곳곳에 굴러다니는 작은 못, 나사, 전구(잡다한 속성)를 각각 전용 상자에 담는 대신 하나의 정리함에 모아두는 것과 같습니다. 퇴화 차원은 **'택배 박스에 인쇄된 송장번호'**와 같습니다. 송장번호를 알기 위해 따로 장부를 뒤질 필요 없이, 물건(팩트) 겉면에 직접 적혀 있어 바로 확인할 수 있는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[FK Explosion]**: 정크 차원이 해결하고자 하는 고질적인 설계 문제.
- **[Audit]**: 퇴화 차원이 주로 활용되는 데이터 감사 및 추적 영역.
- **[Kimball]**: 이러한 기법들을 체계화한 DW의 거장.

📢 **마무리 요약**: **Junk & Degenerate Dimensions**는 다차원 모델링의 조미료와 같습니다. 모델을 가볍고 빠르게 만들어 분석 시스템의 실질적인 기동성을 확보해 줍니다.
