# 367. 데이터 마스킹 (Static vs Dynamic) - 필요한 만큼만 보여주기

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 마스킹은 주민번호, 카드번호 등 민감한 정보의 일부 또는 전부를 **별표(*)나 임의의 값으로 변환하여 노출을 차단하는 보안 기술**이다.
> 2. **가치**: 데이터를 실제로 삭제하지 않고도 형태를 유지함으로써, 보안 사고 위험을 낮추면서 분석이나 개발/테스트 업무에 데이터를 안전하게 활용할 수 있게 한다.
> 3. **융합**: 물리적으로 데이터를 변환하는 정적 마스킹(Static)과 사용자 권한에 따라 실시간으로 가리는 동적 마스킹(Dynamic) 기술이 상황에 맞게 융합되어 사용된다.

+++

### Ⅰ. 정적 마스킹 vs 동적 마스킹 비교

| 비교 항목 | 정적 마스킹 (Static) | 동적 마스킹 (Dynamic) |
|:---|:---|:---|
| **변환 시점** | 데이터 복제/이관 시 (Batch) | **사용자 조회 시 (Runtime)** |
| **원본 유지** | 원본 파괴 (가역 불가능) | **원본 유지** (DB 내부값은 그대로) |
| **주요 용도** | 테스트/개발 데이터 생성 | **운영계 실시간 접근 제어** |
| **성능 영향** | 운영 성능 영향 없음 | 미세한 조회 지연 발생 가능 |

+++

### Ⅱ. 마스킹 아키텍처 시각화 (ASCII Model)

```text
[ Data Masking Scenarios ]

  1. Static Masking (원본 DB ──▶ 테스트 DB)
     [ 900101-1234567 ] ──(Masking Process)──▶ [ 900101-1****** ] 💾 (저장)

  2. Dynamic Masking (사용자별 실시간 필터링)
     [ Database Engine ]
            │ (Original: 900101-1234567)
            ▼
     ┌──────┴──────────────────────┐
     │  Dynamic Masking Layer      │
     └──────┬──────────────┬───────┘
            ▼              ▼
     (Admin: Full View)  (User: Masked View) ✅
     [900101-1234567]    [900101-1******]
```

+++

### Ⅲ. 주요 마스킹 기법

- **치환 (Substitution)**: 다른 임의의 값으로 바꿈. (예: '홍길동' ──▶ '임꺽정')
- **랜덤화 (Shuffling)**: 컬럼 내의 값들을 무작위로 섞음. (총합은 유지되나 개인 특정 불가)
- **부분 가리기 (Partial Masking)**: 특정 위치만 마스킹. (예: 010-1234-****)
- **범주화 (Generalization)**: 구체적 수치를 범위로 바꿈. (예: 28세 ──▶ 20대)

- **📢 섹션 요약 비유**: 데이터 마스킹은 **'방송의 모자이크 처리'**와 같습니다. 정적 마스킹은 아예 원본 필름을 수정해서 배포하는 것이라 나중에 모자이크를 벗길 수 없지만, 동적 마스킹은 원본 필름은 멀쩡한데 시청자(사용자)의 등급에 따라 TV 화면 위에 실시간으로 가림막을 씌워 보여주는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Privacy by Design]**: PbD를 실천하는 가장 대표적인 기술적 수단.
- **[Tokenization]**: 마스킹보다 보안성이 높은 토큰 치환 기술 (상호보완적).
- **[VPD]**: 동적 마스킹을 구현하는 논리적 기반 기술.

📢 **마무리 요약**: **Data Masking**은 보안과 가용성의 타협점입니다. 원본 데이터의 가치를 훼손하지 않으면서도 프라이버시를 완벽히 수호하는 현대 보안의 지혜입니다.
