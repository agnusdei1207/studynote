# 351. RAG (Retrieval Augmented Generation) - LLM의 지식 확장

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RAG(검색 증강 생성)는 거대언어모델(LLM)이 답변을 생성하기 전, **신뢰할 수 있는 외부 지식 베이스(벡터 DB 등)에서 관련 정보를 먼저 검색하여 그 내용을 바탕으로 답변을 보강하는 기술 아키텍처**다.
> 2. **가치**: 모델 재학습(Fine-tuning) 없이도 실시간으로 최신 정보를 반영할 수 있으며, 근거 기반의 답변을 유도하여 AI의 고질적 문제인 **환각(Hallucination) 현상을 획기적으로 줄인다.**
> 3. **융합**: 벡터 데이터베이스(Vector DB), 임베딩 기술, 프롬프트 엔지니어링이 융합된 현대 기업용 AI 시스템의 표준 설계 패턴으로 자리 잡았다.

+++

### Ⅰ. RAG의 작동 프로세스

1. **사용자 질문**: 사용자가 질문을 던집니다. (예: "우리 회사 작년 매출은?")
2. **검색 (Retrieve)**: 질문을 벡터로 변환하여 벡터 DB에서 가장 관련 있는 문서 조각을 찾아냅니다.
3. **증강 (Augment)**: 검색된 문서 조각과 원래 질문을 합쳐 LLM에게 전달할 프롬프트를 만듭니다.
4. **생성 (Generate)**: LLM은 제공된 문서를 참고하여 정확한 답변을 생성합니다.

+++

### Ⅱ. RAG 아키텍처 시각화 (ASCII Flow)

```text
[ RAG Workflow Architecture ]

  (User) ──▶ [ Query ] ──▶ [ Embedding ] ──▶ (Query Vector)
                                               │
          ┌────────────────────────────────────┘
          ▼
  [ Vector Database ] ──▶ [ Top-K Search ] ──▶ [ Relevant Context ]
                                               │
          ┌────────────────────────────────────┘
          ▼
  [ Prompt Builder ] : "제시된 문서를 바탕으로 질문에 답하세요: [Context] + [Query]"
          │
          ▼
  [ LLM (GPT/Claude) ] ──▶ ✅ "정확하고 근거 있는 답변 생성"
```

+++

### Ⅲ. RAG vs Fine-tuning 비교

| 비교 항목 | RAG (검색 증강) | Fine-tuning (미세 조정) |
|:---|:---|:---|
| **지식 최신성** | **실시간 반영 가능** | 재학습 시점까지만 유효 |
| **근거 제시** | **출처 명시 가능 (투명함)** | 근거 제시 불가능 (불투명함) |
| **비용** | 낮음 (DB 업데이트 위주) | 높음 (컴퓨팅 자원 소모) |
| **환각 방지** | 강력함 (제시된 문서 기반) | 취약함 (모델 기억에 의존) |

- **📢 섹션 요약 비유**: RAG는 **'오픈북 테스트'**와 같습니다. 파인튜닝이 시험을 보기 위해 모든 내용을 머릿속에 외우는 것(내부 학습)이라면, RAG는 시험 도중 옆에 있는 백과사전(벡터 DB)을 펼쳐서 관련 내용을 찾아보고(검색) 답을 적는(생성) 것과 같습니다. 훨씬 정확하고 최신 내용을 반영하기 쉽습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Vector DB]**: RAG의 지식 저장고.
- **[Hallucination]**: RAG가 해결하고자 하는 AI의 거짓말 문제.
- **[LangChain / LlamaIndex]**: RAG 아키텍처 구현을 돕는 대표적인 프레임워크.

📢 **마무리 요약**: **RAG**는 생성형 AI를 실무에 적용하기 위한 가장 현실적이고 강력한 방법입니다. 데이터베이스의 '정확함'과 AI의 '창의성'을 연결하는 최고의 가교입니다.
