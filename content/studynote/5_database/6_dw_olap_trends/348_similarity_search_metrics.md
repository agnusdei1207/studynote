# 348. 유사성 검색 (Similarity Search) - 거리로 찾는 정답

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 유사성 검색은 데이터 간의 정확한 일치(Exact Match) 대신, **고차원 벡터 공간에서 두 데이터 사이의 수학적 거리를 계산하여 가장 '가까운' 이웃을 찾아내는 기술**이다.
> 2. **가치**: 코사인 유사도(Cosine)와 유클리드 거리(Euclidean) 등 다양한 척도를 사용하여, 검색어의 의미적 맥락이나 이미지의 시각적 유사도를 기반으로 최적의 결과를 추출한다.
> 3. **융합**: ANN(Approximate Nearest Neighbor) 알고리즘과 결합되어 수조 건의 대규모 데이터셋에서도 실시간에 가까운 검색 속도를 보장하는 현대 AI 검색 엔진의 핵심 로직이다.

+++

### Ⅰ. 주요 유사도 측정 척도 (Metrics)

#### 1. 유클리드 거리 (L2 Distance / Euclidean)
- **정의**: 두 점 사이의 최단 직선거리.
- **특징**: 벡터의 크기(Magnitude)가 중요할 때 사용됩니다. 값이 작을수록 유사합니다.

#### 2. 코사인 유사도 (Cosine Similarity)
- **정의**: 두 벡터 사이의 각도(θ)의 코사인 값을 계산.
- **특징**: 벡터의 길이는 무시하고 **방향성**만 봅니다. 1에 가까울수록(각도가 작을수록) 유사합니다.

#### 3. 내적 (Dot Product)
- **정의**: 두 벡터의 성분별 곱의 합.
- **특징**: 각도와 크기를 모두 고려하며, 신경망 연산에서 가장 흔히 쓰입니다.

+++

### Ⅱ. 유사도 측정 시각화 (ASCII Model)

```text
[ Euclidean vs Cosine Comparison ]

      Y-Axis
        ▲      ● Vector A
        │     /
        │    / θ (Angle for Cosine)
        │   /─────────▶ ● Vector B
        │  / (Distance for Euclidean)
        └──────────────────────────▶ X-Axis

  - Euclidean: 두 점(●) 사이의 직선거리를 직접 측정.
  - Cosine   : 두 선(/, ─)이 벌어진 각도(θ)를 측정.
```

+++

### Ⅲ. 실무적 선택 기준

- **이미지 검색**: 픽셀 값의 차이가 중요한 유클리드 거리를 선호합니다.
- **텍스트 검색**: 문서의 길이에 상관없이 단어 사용 비중(방향)이 중요한 코사인 유사도를 선호합니다.
- **추천 시스템**: 사용자의 선호도 강도와 방향을 모두 고려하기 위해 내적(Dot Product)을 활용합니다.

- **📢 섹션 요약 비유**: 유사성 검색은 **'데이터 세계의 보물찾기'**와 같습니다. 유클리드 거리는 "철수에게서 5m(직선거리) 떨어진 곳"을 찾는 것이고, 코사인 유사도는 "철수가 바라보는 방향과 같은 쪽"을 찾는 것입니다. 무엇을 기준으로 삼느냐에 따라 우리가 발견하는 보물(검색 결과)의 성격이 달라집니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Nearest Neighbor]**: 가장 가까운 이웃을 찾는 검색의 목적.
- **[Curse of Dimensionality]**: 차원이 높아질수록 거리 계산이 무의미해지거나 느려지는 현상.
- **[Normalization]**: 유사도 계산 전 벡터의 크기를 1로 맞추는 전처리 과정.

📢 **마무리 요약**: **Similarity Search**는 데이터의 '관계'를 수치화합니다. 모호한 질문 속에서도 가장 닮은 정답을 찾아내는, 인공지능 시대를 지탱하는 검색의 신기술입니다.
