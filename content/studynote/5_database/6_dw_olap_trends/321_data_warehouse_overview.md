# 321. 데이터 웨어하우스 (Data Warehouse) - 의사결정의 보고

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 웨어하우스(DW)는 기업 내 여러 시스템에 흩어진 데이터를 통합하여, **사용자의 의사결정을 돕기 위해 분석 전용으로 구축한 대규모 데이터 저장소**다.
> 2. **가치**: 운영 DB(OLTP)의 부하를 주지 않으면서 과거부터 현재까지의 시계열 데이터를 보존하고, 다차원 분석을 통해 비즈니스 인사이트를 도출하는 핵심 인프라 역할을 한다.
> 3. **융합**: ETL 프로세스를 통해 정제된 데이터만을 수용하며, 최근에는 클라우드 기반의 대규모 병렬 처리(MPP) 기술과 결합되어 데이터 레이크하우스로 진화하고 있다.

+++

### Ⅰ. 데이터 웨어하우스의 4대 특징 (W.H. Inmon)

1. **주제 지향성 (Subject-Oriented)**: 업무 처리(예: 입금, 출금)가 아닌 주요 주제(예: 고객, 상품, 매출)를 중심으로 데이터를 구성합니다.
2. **통합성 (Integrated)**: 여러 소스 시스템의 서로 다른 명칭과 단위를 일관된 형식으로 통일하여 저장합니다.
3. **시계열성 (Time-Variant)**: 단순히 현재 값이 아닌, 과거 5~10년 동안의 변화 이력을 모두 보존합니다.
4. **비휘발성 (Non-Volatile)**: 한 번 적재된 데이터는 수정되거나 삭제되지 않으며, 오직 읽기 전용으로만 사용됩니다.

+++

### Ⅱ. DW 아키텍처 시각화 (ASCII Flow)

```text
[ Data Warehouse Data Flow ]

  (Source Systems)       (Integration)       (Storage)         (Access)
  ┌────────────┐       ┌───────────┐       ┌───────────┐      ┌────────────┐
  │  ERP / CRM │ ──▶   │    ETL    │ ──▶   │   Data    │ ──▶  │ BI / OLAP  │
  ├────────────┤       │ (Cleanse) │       │ Warehouse │      ├────────────┤
  │  Legacy DB │ ──▶   │ (Transform)│      └─────┬─────┘      │ Data Mining│
  └────────────┘       └───────────┘             │            └────────────┘
                                           ┌─────▼─────┐
                                           │ Data Mart │ (부서별 요약)
                                           └───────────┘
```

### Ⅲ. 운영 DB(OLTP)와의 차이점

| 비교 항목 | 운영 DB (OLTP) | 데이터 웨어하우스 (OLAP) |
|:---|:---|:---|
| **목적** | 일상적 업무 처리 | **전략적 의사결정 분석** |
| **데이터 범위** | 현재 데이터 위주 | **방대한 이력 데이터** |
| **단위 작업** | 짧은 트랜잭션 (Update 위주) | **대량의 데이터 조회 (Read 위주)** |
| **사용자** | 실무 직원 | 관리자, 데이터 분석가 |

- **📢 섹션 요약 비유**: 데이터 웨어하우스는 **'박물관의 수장고'**와 같습니다. 매일 물건을 사고파는 '시장(OLTP)'과 달리, 수장고는 역사적으로 가치 있는 물건들을 시대별로(시계열성), 분류별로(주제 지향성) 잘 정리해서 보관해 두었다가, 연구자(분석가)들이 필요할 때 언제든 꺼내 볼 수 있게 만든 정적인 공간입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Inmon vs Kimball]**: DW를 구축하는 두 가지 철학적 접근법.
- **[Metadata]**: 데이터의 출처와 의미를 설명하는 '데이터의 데이터'.
- **[Data Silo]**: DW가 해결하고자 하는 조직 내 데이터 단절 현상.

📢 **마무리 요약**: **Data Warehouse**는 기업의 기억 장치입니다. 파편화된 정보를 정제된 지식으로 바꾸어 미래의 방향을 제시하는 비즈니스 인텔리전스의 심장입니다.
