# 331. 스타 스키마 (Star Schema) - 분석용 설계의 표준

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스타 스키마는 중앙의 **팩트 테이블(Fact Table)을 중심으로 여러 개의 비정규화된 차원 테이블(Dimension Table)이 직접 연결**된 가장 단순하고 직관적인 다차원 모델링 구조다.
> 2. **가치**: 조인(Join)의 깊이를 1단계로 제한하여 쿼리 처리 속도를 극대화하며, 복잡한 비즈니스 관계를 단순한 별 모양으로 시각화하여 사용자의 분석 편의성을 높인다.
> 3. **융합**: 현대 비즈니스 인텔리전스(BI) 툴과 OLAP 엔진이 다차원 큐브를 생성하는 가장 선호하는 물리적 스키마 구조로 융합되어 사용된다.

+++

### Ⅰ. 스타 스키마의 구조적 특징

- **중앙 집중형**: 대량의 측정값이 담긴 팩트 테이블이 모든 차원의 교차점에 위치합니다.
- **비정규화 (Denormalization)**: 차원 테이블은 중복을 허용하더라도 조인을 줄이기 위해 1단계로 유지됩니다.
- **성능 최적화**: 다차원 검색 시 복잡한 계층 조인 없이 팩트와 각 차원 간의 1:1 조인만 발생하므로 인덱스 스캔 효율이 매우 높습니다.

+++

### Ⅱ. 스타 스키마 시각화 (ASCII Diagram)

```text
[ Star Schema Layout ]

      ┌───────────────┐          ┌───────────────┐
      │  Dim_Product  │          │   Dim_Store   │
      └───────┬───────┘          └───────┬───────┘
              │                          │
              └───────────┐      ┌───────┘
                          ▼      ▼
                    ┌──────────────────┐
                    │    Fact_Sales    │
                    │ (Central Table)  │
                    └──────────────────┘
                          ▲      ▲
              ┌───────────┘      └───────┐
              │                          │
      ┌───────┴───────┐          ┌───────┴───────┐
      │   Dim_Time    │          │  Dim_Region   │
      └───────────────┘          └───────────────┘
```

+++

### Ⅲ. 장점 및 단점

| 장점 | 단점 |
|:---|:---|
| **쿼리 단순성**: SQL 작성이 쉽고 직관적임 | **데이터 중복**: 차원 테이블의 비정규화로 저장 공간 낭비 |
| **고속 조회**: Join 횟수 최소화로 성능 우수 | **데이터 일관성 관리**: 중복 데이터 갱신 시 부하 발생 |
| **BI 툴 최적화**: 대부분의 분석 도구와 찰떡궁합 | **복잡한 계층 표현**: 조직도 등 계층 구조 표현에 한계 |

- **📢 섹션 요약 비유**: 스타 스키마는 **'별모양 사탕'**과 같습니다. 가운데 가장 달콤하고 핵심적인 맛(팩트)이 있고, 그 주변을 여러 장식(차원)이 둘러싸고 있어 한입에 쏙 넣어 맛을 느끼기(분석)에 가장 편하고 빠른 모양입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[De-normalization]**: 스타 스키마 성능의 핵심 비결.
- **[Dimensional Modeling]**: 스타 스키마를 만드는 설계 방법론.
- **[Snowflake Schema]**: 스타 스키마의 중복 문제를 해결하기 위해 정규화한 변형 모델.

📢 **마무리 요약**: **Star Schema**는 데이터 웨어하우스의 영원한 베스트셀러입니다. 단순함 속에 강력한 성능을 감춘, 분석 시스템의 가장 정석적인 형태입니다.
