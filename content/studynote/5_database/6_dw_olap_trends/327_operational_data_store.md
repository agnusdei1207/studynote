# 327. ODS (Operational Data Store) - 운영과 분석의 교차점

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ODS는 여러 운영 시스템(OLTP)으로부터 데이터를 실시간 혹은 근실시간으로 수집하여 통합한, **운영 보고 및 실시간 의사결정을 위한 데이터 저장소**다.
> 2. **가치**: 데이터 웨어하우스(DW)가 과거 이력 분석에 치중한다면, ODS는 **"지금 현재 무슨 일이 일어나고 있는가?"**에 집중하여 운영계의 부하 없이 실시간 통합 조회를 가능케 한다.
> 3. **융합**: 데이터 통합 과정에서 스테이징 영역과 DW 사이의 논리적 계층으로 기능하며, 최근에는 CDC(Change Data Capture) 기술과 융합되어 데이터의 신선도를 극대화한다.

+++

### Ⅰ. ODS의 특징과 역할

- **실시간성**: DW보다 훨씬 자주(수분~수초 단위) 업데이트되어 현재의 업무 상태를 반영합니다.
- **상세 데이터**: 요약된 데이터가 아닌, 운영계와 거의 유사한 수준의 상세 내역(Atomic Data)을 보관합니다.
- **짧은 보존 기간**: 보통 수일에서 수개월 정도의 최신 데이터만 유지하며, 오래된 데이터는 DW로 이관됩니다.
- **통합성**: 서로 다른 여러 운영 DB의 데이터를 하나의 스키마로 통합하여 보여줍니다.

+++

### Ⅱ. 데이터 흐름 내 ODS의 위치 (ASCII Flow)

```text
[ Data Integration Landscape ]

  (Production)         (Integration)         (Analytics)
  ┌──────────┐        ┌─────────────┐       ┌──────────────┐
  │ DB 1     │ ──▶    │             │ ──▶   │      DW      │
  ├──────────┤ (CDC)  │     ODS     │       │ (Historical) │
  │ DB 2     │ ──▶    │  (Current)  │ ──▶   └──────────────┘
  └──────────┘        └──────┬──────┘
                             │
                             ▼
                    (Operational Reports / 
                     Real-time Dashboards)
```

### Ⅲ. ODS vs DW 비교

| 비교 항목 | ODS (운영 데이터 저장소) | DW (데이터 웨어하우스) |
|:---|:---|:---|
| **주요 목적** | 실시간 운영 지원, 현황 파악 | 장기적 추세 분석, 전략 수립 |
| **데이터 시점** | **현재 (Current)** | **과거 ~ 현재 (Historical)** |
| **데이터 수준** | 상세 데이터 (Granular) | 집계/요약 데이터 위주 |
| **갱신 주기** | 실시간 / 근실시간 (자주) | 배치 (주기적) |

- **📢 섹션 요약 비유**: ODS는 **'상황실의 실시간 모니터'**와 같습니다. 1년 전의 통계 기록을 보는 '연감(DW)'과 달리, 지금 현재 도로에 차가 얼마나 막히는지, 사고가 났는지 즉각적으로 확인하고 대처하기 위해 만든 살아있는 데이터 저장소입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[CDC]**: ODS의 실시간성을 보장하는 핵심 기술.
- **[Integration Hub]**: 여러 부서의 현재 데이터를 한눈에 보게 돕는 허브 역할.
- **[Operational Reporting]**: 일일 마감, 재고 현황 등 실무용 보고서 작성.

📢 **마무리 요약**: **ODS**는 데이터의 신선함을 유지하는 저장소입니다. 운영의 속도와 분석의 깊이를 연결하여 비즈니스 현장의 기민한 대응을 지원하는 전략적 아키텍처입니다.
