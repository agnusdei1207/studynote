# 322. 데이터 마트 (Data Mart) - 부서별 맞춤형 데이터

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 마트(Data Mart)는 데이터 웨어하우스(DW) 전체 데이터 중 **특정 부서(마케팅, 재무 등)나 특정 비즈니스 영역에 필요한 데이터만을 추출하여 구축한 소규모 분석 전용 저장소**다.
> 2. **가치**: 불필요한 데이터를 배제하고 부서 맞춤형 데이터 모델을 제공함으로써 **조회 속도를 높이고, 사용자에게 최적화된 분석 환경**을 선사한다.
> 3. **융합**: DW로부터 데이터를 공급받는 종속형(Dependent) 방식과 개별 시스템에서 직접 구축하는 독립형(Independent) 방식으로 나뉘며, 최근에는 가상화 기술을 통한 논리적 데이터 마트로 융합 진화하고 있다.

+++

### Ⅰ. 데이터 마트의 정의 및 목적

- **정의**: DW의 부분 집합으로, 특정 사용자 집단의 요구사항에 맞춰 정제되고 요약된 데이터 저장소입니다.
- **주요 목적**:
    - **빠른 응답성**: 전체 DW를 뒤지는 것보다 훨씬 적은 데이터량으로 신속한 보고서 생성.
    - **편의성**: 현업 사용자가 이해하기 쉬운 비즈니스 용어와 모델로 구성.
    - **보안**: 해당 부서에 허용된 데이터만 격리하여 관리.

+++

### Ⅱ. DW와 데이터 마트의 관계 (ASCII Flow)

```text
[ Data Architecture Hierachy ]

  ┌──────────────┐
  │ Source Data  │ (ERP, CRM, Web Logs...)
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │     DW       │ (전사 통합 저장소, 원본 보존)
  └──────┬───────┘
         │
  ┌──────┴───────────────────────┐
  ▼                             ▼
 [ Marketing DM ]        [ Finance DM ] (특정 목적 데이터)
 - 캠페인 반응률          - 분기별 손익
 - 고객 등급 분석         - 예산 집행 현황
```

+++

### Ⅲ. 구축 방식의 비교

1. **종속형 데이터 마트 (Dependent DM)**:
    - 중앙 DW에서 데이터를 내려받아 생성. 데이터 정합성이 우수하지만 구축 시간이 오래 걸림.
2. **독립형 데이터 마트 (Independent DM)**:
    - DW 없이 운영계에서 바로 구축. 신속하지만 전사적 관점의 데이터 통합이 어려움 (Silo 현상).

- **📢 섹션 요약 비유**: 데이터 마트는 **'편의점'**과 같습니다. 전사 데이터가 다 들어있는 DW가 '대형 물류 창고'라면, 각 부서 근처에 있는 데이터 마트는 자주 찾는 물건들만 골라 놓은 편의점입니다. 창고까지 멀리 갈 필요 없이 집 앞 편의점에서 필요한 것을 바로 살 수 있듯, 마트를 통해 빠르게 업무를 처리할 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Bottom-up Approach]**: 데이터 마트를 먼저 짓고 DW로 합치는 킴벌(Kimball) 방식.
- **[ETL]**: DW에서 마트로 데이터를 정제하여 옮기는 과정.
- **[Silo]**: 독립형 마트들이 서로 소통되지 않아 데이터가 꼬이는 현상.

📢 **마무리 요약**: **Data Mart**는 분석의 효율성을 극대화하는 전술적 도구입니다. 전사적인 정합성을 유지하면서도 부서별 자율성을 보장하는 영리한 아키텍처 배치가 중요합니다.
