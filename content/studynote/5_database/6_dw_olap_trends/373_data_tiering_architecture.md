# 373. 데이터 계층화(Data Tiering) - 가치에 따른 저장의 경제학

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 계층화는 데이터의 접근 빈도와 비즈니스 가치에 따라 **핫(Hot), 웜(Warm), 콜드(Cold) 데이터로 분류**하고, 각 성격에 맞는 최적의 성능과 비용을 가진 스토리지에 분산 저장하는 기술이다.
> 2. **가치**: 자주 쓰는 데이터는 고속의 SSD/RAM에, 거의 안 쓰는 이력 데이터는 저렴한 클라우드 오브젝트 스토리지에 배치하여 **인프라 비용을 획기적으로 절감**하면서도 성능을 유지한다.
> 3. **융합**: 정보 수명 주기 관리(ILM) 정책 및 클라우드 티어링 기술과 결합되어, 대규모 데이터베이스의 지속 가능한 성장을 지원하는 핵심 아키텍처로 기능한다.

+++

### Ⅰ. 데이터 분류 체계 (Hot / Warm / Cold)

| 데이터 유형 | 접근 빈도 | 성능 요건 | 저장 매체 |
|:---|:---|:---|:---|
| **핫 (Hot)** | 매우 빈번 (실시간) | 초고속 (ms 미만) | **RAM, NVMe SSD** |
| **웜 (Warm)** | 가끔 (월/분기 보고서) | 보통 (초 단위) | 저가형 SSD, 고속 HDD |
| **콜드 (Cold)** | 거의 없음 (감사/법적 보존) | 낮음 (분 단위 가능) | **오브젝트 스토리지 (S3), Tape** |

+++

### Ⅱ. 계층화 아키텍처 시각화 (ASCII Model)

데이터가 시간이 지남에 따라 아래 계층으로 자동으로 이동(Aging)합니다.

```text
[ Data Tiering Architecture ]

  (User Requests)
       │
  ┌────▼────┐ [ Hot Layer ]  : Current Month Sales (High Cost, Fast)
  │  NVMe   │
  └────┬────┘
       │ (Auto-migration after 30 days)
  ┌────▼────┐ [ Warm Layer ] : Last 1 Year History (Medium Cost)
  │   SSD   │
  └────┬────┘
       │ (Auto-migration after 1 year)
  ┌────▼────┐ [ Cold Layer ] : Legacy Archive / Audit (Low Cost, Slower)
  │  S3/Gl  │
  └─────────┘
```

+++

### Ⅲ. 데이터 계층화의 주요 이점

1. **비용 효율성**: 모든 데이터를 비싼 고성능 스토리지에 담을 필요가 없어 스토리지 예산을 50% 이상 절감할 수 있습니다.
2. **백업 및 복구 최적화**: 핫 데이터 위주로 백업을 집중 수행하여 복구 시간(RTO)을 단축시킵니다.
3. **시스템 부하 분산**: 최신 트래픽이 집중되는 장치와 과거 분석용 장치를 물리적으로 분리하여 성능 간섭을 제거합니다.

- **📢 섹션 요약 비유**: 데이터 계층화는 **'냉장고와 창고 정리'**와 같습니다. 매일 먹는 반찬(핫 데이터)은 손이 잘 닿는 '냉장고(SSD)'에 두고, 가끔 먹는 통조림(웜 데이터)은 '찬장(HDD)'에, 일 년에 한 번 쓸까 말까 한 제사 도구(콜드 데이터)는 저 멀리 '베란다 창고(S3)'에 보관하는 것과 같습니다. 이렇게 해야 주방 공간을 넓게 쓰면서도 필요한 요리를 빠르게 할 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[ILM (Information Lifecycle Management)]**: 데이터의 탄생부터 폐기까지를 관리하는 거시적 전략.
- **[Atemporal Consistency]**: 계층 이동 중에도 데이터 조회에 문제가 없게 하는 정합성 기술.
- **[Object Storage]**: 콜드 데이터 저장을 위한 현대 클라우드의 표준 저장소.

📢 **마무리 요약**: **Data Tiering**은 지혜로운 자원 배분의 기술입니다. 데이터의 '신선도'를 기준으로 저장소를 정의할 때, 거대해지는 데이터를 가장 경제적이고 효율적으로 통제할 수 있습니다.
