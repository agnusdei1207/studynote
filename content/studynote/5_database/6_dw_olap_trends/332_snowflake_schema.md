# 332. 눈꽃 스키마 (Snowflake Schema) - 정규화의 미학

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 눈꽃 스키마는 스타 스키마의 **차원 테이블들을 정규화(Normalization)하여 다단계 계층 구조로 확장**한 다차원 모델링 형태다.
> 2. **가치**: 차원 데이터의 중복을 제거하여 저장 공간을 절약하고 데이터 무결성을 높이며, 복잡한 분류 체계를 물리적 테이블 구조에 명확히 반영한다.
> 3. **융합**: 고전적인 데이터 정규화 이론과 분석용 다차원 모델링이 결합된 형태이며, 최근에는 클라우드 DW의 유연한 조인 성능과 맞물려 다시 주목받고 있다.

+++

### Ⅰ. 눈꽃 스키마의 구조적 특징

- **정규화된 차원**: 스타 스키마에서 하나의 차원 테이블에 들어있던 정보(예: 상품-카테고리-브랜드)를 각각의 테이블로 분리합니다.
- **복잡한 조인**: 팩트 테이블에서 원하는 속성까지 도달하기 위해 여러 번의 조인(`Fact -> Dim1 -> Dim2`)이 필요합니다.
- **눈꽃 모양**: 중심에서 밖으로 뻗어 나가는 가지들이 정규화로 인해 잘게 쪼개져 눈결정체(Snowflake) 같은 형상을 띱니다.

+++

### Ⅱ. 눈꽃 스키마 시각화 (ASCII Model)

```text
[ Snowflake Schema Architecture ]

      [ Dim_Category ] (Level 2)
             ▲
             │
      [ Dim_Product ]  (Level 1)       [ Dim_Quarter ] (Level 2)
             │                                ▲
             └───────┐        ┌───────────────┘
                     ▼        ▼
               [ Fact_Sales ] (Central)
               - SalesID
               - ProductID (FK)
               - TimeID (FK)
               - Amount
                     ▲        ▲
             ┌───────┘        └───────────────┐
             │                                │
      [ Dim_Time ] (Level 1)           [ Dim_Region ] (Level 1)
                                              │
                                              ▼
                                       [ Dim_Country ] (Level 2)
```

+++

### Ⅲ. 스타 스키마 vs 눈꽃 스키마

| 비교 항목 | 스타 스키마 (Star) | 눈꽃 스키마 (Snowflake) |
|:---|:---|:---|
| **정규화 여부** | 비정규화 (Denormalized) | **정규화 (Normalized)** |
| **데이터 중복** | 많음 | **적음** |
| **쿼리 복잡도** | 단순 (Join 적음) | **복잡 (Join 많음)** |
| **유지보수** | 어려움 (중복 데이터 갱신) | **쉬움 (일관성 유지 유리)** |
| **조회 성능** | **우수** | 상대적으로 낮음 |

- **📢 섹션 요약 비유**: 눈꽃 스키마는 **'잘 정리된 서류 파일링'**과 같습니다. 스타 스키마가 한 장의 큰 종이에 모든 정보를 다 적어놓은 것이라면, 눈꽃 스키마는 제목별로 대분류, 중분류 파일을 따로 만들어 연결해 둔 것입니다. 종이(용량)는 아끼고 정리는 완벽하지만, 내용을 다 보려면 파일을 여러 번 열어봐야(Join) 하는 번거로움이 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Normalization]**: 눈꽃 스키마를 만드는 핵심 동력.
- **[Storage Optimization]**: 대규모 차원 데이터에서 눈꽃 스키마를 쓰는 주된 이유.
- **[Hierarchy]**: 데이터의 계층 관계를 물리적으로 가장 잘 표현하는 구조.

📢 **마무리 요약**: **Snowflake Schema**는 정교한 데이터 설계의 산물입니다. 성능과 용량 사이의 고민 끝에 데이터의 '질서'와 '무결성'을 택한 고차원 모델링 전략입니다.
