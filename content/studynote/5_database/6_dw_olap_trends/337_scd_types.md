# 337. SCD 유형 (Type 0, 1, 2, 3, 4) - 변화에 대응하는 5가지 전략

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SCD 유형은 데이터 변화 시 과거 이력을 보존할 것인지, 아니면 현재 값으로 갱신할 것인지에 대한 **구체적인 데이터 처리 규칙들의 집합**이다.
> 2. **가치**: 분석 목적에 따라 단순 갱신(Type 1)부터 완벽한 이력 보존(Type 2), 현재와 과거 값의 공존(Type 3) 등을 선택하여 **시스템 자원 효율성과 분석 정밀도**를 조율한다.
> 3. **융합**: 기업의 데이터 거버넌스 정책과 융합되어, 고객 등급이나 주소와 같은 민감 정보의 변화를 공학적으로 완벽히 추적 가능하게 한다.

+++

### Ⅰ. SCD 5대 주요 유형 요약

| 유형 | 명칭 | 작동 방식 | 특징 |
|:---:|:---|:---|:---|
| **Type 0** | 고정 (Fixed) | 한 번 적재되면 절대 안 바뀜 | 생년월일 등 불변 데이터 |
| **Type 1** | 덮어쓰기 (Overwrite) | 새로운 값으로 기존 값을 밀어냄 | 이력 보존 안 됨, 가장 단순 |
| **Type 2** | **이력 추가 (New Row)** | **새로운 행을 추가**하고 기간 설정 | **DW의 표준 방식**, 완벽한 이력 |
| **Type 3** | 컬럼 추가 (New Column) | '이전 값' 컬럼을 만들어 보관 | 현재와 바로 직전 값만 비교 가능 |
| **Type 4** | 이력 테이블 분리 | 원본은 갱신하고 이력은 따로 저장 | 성능과 이력을 모두 잡는 하이브리드 |

+++

### Ⅱ. 핵심 유형 시각화 (ASCII Comparison)

고객 'Kim'이 서울에서 부산으로 이사했을 때의 변화입니다.

```text
[ Type 1: Overwrite ]           [ Type 2: Add New Row ]
┌────────┬──────────┐           ┌────────┬──────────┬──────────┐
│ Name   │ City     │           │ SK     │ Name     │ City     │
├────────┼──────────┤           ├────────┼──────────┼──────────┤
│ Kim    │ Pusan    │           │ 1      │ Kim      │ Seoul    │ (Expired)
└────────┴──────────┘           │ 2      │ Kim      │ Pusan    │ (Current)
 (Seoul 정보는 영영 사라짐)        └────────┴──────────┴──────────┘
                                 (역사적 진실이 보존됨 ✅)
```

+++

### Ⅲ. 실무적 선택 기준

- **Type 1 사용**: 데이터 오타 수정이나, 과거 이력이 비즈니스적으로 전혀 가치가 없을 때.
- **Type 2 사용**: "이사 가기 전의 매출"과 "이사 간 후의 매출"을 정확히 분리해서 집계해야 할 때. (가장 권장됨)
- **Type 4 사용**: 운영 성능을 위해 주 테이블은 가볍게 유지하면서도, 감사를 위해 모든 로그를 남겨야 할 때.

- **📢 섹션 요약 비유**: SCD 유형은 **'일기장 수정법'**과 같습니다. Type 1은 틀린 내용을 지우개로 지우고 쓰는 것이고, Type 2는 다음 장에 "오늘부터는 이렇다"라고 새로 적는 것입니다. Type 3은 원래 문장 옆에 괄호 치고 옛날 내용을 적어두는 방식입니다. 나중에 내 인생을 돌아보기(분석)에는 한 장씩 정성껏 새로 쓰는 Type 2가 최고겠죠.

+++

### Ⅳ. 개념 맵 및 요약

- **[Current Flag]**: Type 2에서 현재 유효한 행인지 표시하는 불리언 필드.
- **[Surrogate Key]**: Type 2 구현을 위해 반드시 필요한 인공 키.
- **[Effective Date]**: 데이터의 유효 시작일과 종료일.

📢 **마무리 요약**: **SCD Types**는 데이터 아키텍트의 전략 리스트입니다. 비즈니스의 요구사항과 스토리지 비용을 계산하여 가장 합리적인 이력 관리 모델을 구축해야 합니다.
