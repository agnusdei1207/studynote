# 299. Apache Cassandra의 특징 - 가용성의 제왕

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 카산드라는 페이스북에서 고안하고 아마존 다이나모의 분산 설계와 구글 빅테이블의 데이터 모델을 결합하여 만든 **마스터 없는(Masterless) 피어투피어(P2P) 기반의 고가용성 NoSQL 데이터베이스**다.
> 2. **가치**: 단일 장애 지점(SPOF)이 전혀 없는 링(Ring) 구조 아키텍처를 통해 데이터 센터 전체의 장애 시에도 무중단 서비스를 제공하며, **선형적인 쓰기 확장성**을 보장한다.
> 3. **융합**: 가십 프로토콜(Gossip Protocol)을 통한 노드 상태 전파와 튜너블 컨시스턴시(Tunable Consistency) 기술이 융합되어, 성능과 일관성 사이의 정교한 트레이드오프를 지원한다.

+++

### Ⅰ. 카산드라의 주요 기술적 특징

- **Masterless Architecture**: 모든 노드가 대등한 역할을 수행합니다. 어떤 노드로 요청을 보내도 처리가 가능합니다.
- **Ring-based Topology**: 일관된 해싱(Consistent Hashing)을 사용하여 데이터를 링 구조로 분산 배치합니다.
- **Tunable Consistency**: 읽기/쓰기 시 정족수(Quorum)를 조절하여 강력한 일관성과 결과적 일관성 중 선택할 수 있습니다.
- **LSM-Tree 기반 쓰기**: 디스크 순차 쓰기를 통해 엄청난 양의 쓰기 트래픽을 지연 없이 소화합니다.

+++

### Ⅱ. 카산드라 분산 구조 시각화 (ASCII Ring)

```text
[ Cassandra Masterless Ring ]

          [ Node A ]
        /            \
  [ Node F ]      [ Node B ] ──▶ (Gossip: "I'm alive!")
      |                |
  [ Node E ]      [ Node C ]
        \            /
          [ Node D ]

  * 데이터 복제: Node A에 쓰면 설정(RF=3)에 따라 
    Node B, C에도 복제본이 자동으로 저장됨.
  * 장애 대응: Node A가 죽어도 사용자는 B~F 중 
    아무 곳에나 요청 가능 ✅
```

+++

### Ⅲ. 실무적 강점: 튜너블 컨시스턴시

- **ANY**: 어느 한 노드라도 받으면 성공 (가장 빠름, 유실 위험).
- **ONE**: 데이터의 담당 노드 중 하나만 성공하면 됨.
- **QUORUM**: 과반수 이상의 노드가 성공해야 함 (일관성과 성능의 균형).
- **ALL**: 모든 복제 노드가 성공해야 함 (가장 느림, 강한 일관성).

- **📢 섹션 요약 비유**: 카산드라는 **'리더 없는 자율 주행 함대'**와 같습니다. 대장선(마스터)이 따로 없어서 한두 척이 침몰해도 남은 배들이 즉시 정보를 공유(Gossip)하며 임무를 완수합니다. 명령(데이터)을 내릴 때 "한 명만 알면 돼"라고 할지 "절반은 알아야 해"라고 할지 내 마음대로 정할 수 있는(Tunable) 유연한 함대입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Gossip Protocol]**: 노드 간 안부를 묻는 통신 규약.
- **[SSTable]**: 메모리의 MemTable이 디스크로 플러시된 불변의 데이터 파일.
- **[No SPOF]**: 카산드라가 클라우드 인프라에서 사랑받는 가장 큰 이유.

📢 **마무리 요약**: **Apache Cassandra**는 가용성에 미친 데이터베이스입니다. 멈추지 않는 서비스, 끊임없는 데이터 유입이 필요한 대규모 분산 환경의 절대 강자입니다.
