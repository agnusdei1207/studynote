# 284. 맵리듀스 (MapReduce) - 분산 병렬 처리의 거인

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 맵리듀스(MapReduce)는 대규모 데이터셋을 여러 노드에 나누어 병렬로 처리하기 위해 구글이 고안한 프로그래밍 모델로, **데이터를 필터링/변환하는 맵(Map) 단계와 데이터를 집계하는 리듀스(Reduce) 단계**로 구성된다.
> 2. **가치**: 복잡한 분산 컴퓨팅의 구현(장애 복구, 데이터 배분 등)을 추상화하여, 개발자가 단순한 로직 작성만으로 테라바이트(TB) 급 이상의 빅데이터를 손쉽게 다룰 수 있게 한다.
> 3. **융합**: 하둡(Hadoop) 에코시스템의 핵심 엔진인 HDFS와 결합되어 데이터 중심(Data-centric) 연산의 시대를 열었으며, 현대 스파크(Spark)나 분산 DB 쿼리 엔진의 기초 사상이 되었다.

+++

### Ⅰ. 맵리듀스의 3대 처리 단계

1. **Map (맵)**: 입력 데이터를 (Key, Value) 쌍으로 쪼개고 변환합니다. (예: 텍스트에서 단어 추출)
2. **Shuffle & Sort (셔플 및 정렬)**: 같은 키를 가진 중간 데이터들을 동일한 리듀서 노드로 모으고 정렬합니다. (가장 무거운 네트워크 작업)
3. **Reduce (리듀스)**: 같은 키로 묶인 값들을 합치거나 통계 내어 최종 결과를 도출합니다. (예: 단어 빈도수 합산)

+++

### Ⅱ. 맵리듀스 워크플로우 시각화 (ASCII Flow)

```text
[ MapReduce: Word Count 예시 ]

  Input: "Apple Banana Apple"
    │
  ┌─▼──────────────────┐
  │  MAP Phase         │ ──▶ (Apple, 1), (Banana, 1), (Apple, 1)
  └────────┬───────────┘
           │
  ┌────────▼───────────┐
  │ Shuffle & Sort     │ ──▶ (Apple, [1, 1]), (Banana, [1])
  └────────┬───────────┘
           │
  ┌────────▼───────────┐
  │  REDUCE Phase      │ ──▶ (Apple, 2), (Banana, 1)
  └────────┬───────────┘
           │
  Output: Apple: 2, Banana: 1 ✅
```

+++

### Ⅲ. 맵리듀스의 혁신: 데이터 로컬리티

- **Data Locality**: 데이터를 연산 장치로 옮기는 대신, **연산 로직(코드)을 데이터가 있는 노드로 보내서 실행**합니다. 대규모 데이터 이동에 따른 네트워크 비용을 획기적으로 절감한 맵리듀스의 '신의 한 수'입니다.
- **Fault Tolerance**: 특정 노드가 작업 중 죽으면, 마스터 노드가 다른 노드에 해당 작업(Map/Reduce)을 다시 할당하여 완주를 보장합니다.

- **📢 섹션 요약 비유**: 맵리듀스는 **'대규모 급식 조리'**와 같습니다. 1. 각 조리사들이 재료를 다듬고(Map), 2. 다듬어진 재료 중 같은 종류끼리 냄비에 모아서(Shuffle), 3. 마지막에 푹 끓여서 요리를 완성(Reduce)하는 체계적인 분업 시스템입니다. 주방장(Master)은 누가 쉬고 있는지, 누가 불을 냈는지 감시하며 요리를 지휘합니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Split]**: 데이터를 맵 노드에 배분하는 단위.
- **[Combiner]**: 맵 노드에서 미리 1차 집계를 수행하여 셔플 부하를 줄이는 최적화 도구.
- **[Shared Nothing]**: 노드 간 자원 공유 없이 메시지만으로 통신하는 아키텍처.

📢 **마무리 요약**: **MapReduce**는 빅데이터의 바다를 가르는 거대한 그물입니다. 분산 처리의 복잡함을 'Map'과 'Reduce'라는 단순한 철학으로 녹여낸 데이터 공학의 걸작입니다.
