# 287. 시계열 데이터 관리 - Downsampling & Retention

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다운샘플링(Downsampling)은 데이터의 해상도를 낮추어 저장 공간을 절약하는 기술이며, 보존 정책(Retention)은 일정 기간이 지난 데이터를 자동으로 폐기하여 무한한 데이터 증가를 통제하는 전략이다.
> 2. **가치**: 대량의 초미세 시계열 데이터(High Resolution)를 비즈니스 가치가 높은 통계 데이터로 압축하고 관리함으로써 **스토리지 비용 최적화와 쿼리 성능 향상**을 동시에 달성한다.
> 3. **융합**: 데이터 라이프사이클 관리(DLM) 기술과 결합되어, 최신 데이터는 핫(Hot) 스토리지에, 오래된 집계 데이터는 콜드(Cold) 스토리지에 배치하는 계층화 아키텍처를 완성한다.

+++

### Ⅰ. 다운샘플링 (Downsampling)

- **정의**: 초 단위의 상세 데이터를 분, 시간, 일 단위로 묶어 평균, 최댓값, 최솟값 등의 대표값으로 변환하는 과정입니다.
- **필요성**: 1년 전의 CPU 사용률을 1초 단위로 볼 필요는 거의 없습니다. 이를 1시간 평균값으로 변환하면 데이터 용량을 1/3600로 줄일 수 있습니다.

+++

### Ⅱ. 보존 정책 (Retention Policy)

- **정의**: 특정 시간(예: 30일)이 지난 데이터를 자동으로 삭제하도록 설정하는 규칙입니다.
- **매커니즘**: TSDB는 데이터를 시간 범위별로 파일(Shard)을 나누어 저장하며, 보존 기간이 지난 파일 전체를 삭제(Dropping)하는 방식으로 성능 부하 없이 데이터를 정리합니다.

```text
[시계열 데이터 라이프사이클 시각화]

  (Real-time) 1s Interval  ──▶  (Daily) 1m Interval  ──▶  (Monthly) 1h Interval
  ┌──────────────────────┐      ┌──────────────────┐      ┌────────────────┐
  │  Raw Data (Hot)      │ ──▶  │ Downsampled (Warm)│ ──▶  │ Archive (Cold) │
  │  Retention: 7 days   │      │ Retention: 90 days│      │ Retention: 1 yr│
  └──────────────────────┘      └──────────────────┘      └────────────────┘
```

+++

### Ⅲ. 실무적 이점

1. **지속 가능한 인프라**: 무한정 늘어나는 디스크 비용을 예측 가능한 수준으로 통제합니다.
2. **쿼리 가속**: 수조 건의 원본 데이터를 읽는 대신, 미리 계산된 다운샘플링 데이터를 읽어 차트 로딩 속도를 수백 배 높입니다.
3. **규제 준수**: 개인정보나 로그 보존 법적 기한에 맞춰 데이터를 안전하게 자동 파기합니다.

- **📢 섹션 요약 비유**: 시계열 데이터 관리는 **'CCTV 영상 관리'**와 같습니다. 어제 영상은 고화질(Raw)로 보관하지만, 한 달 전 영상은 중요한 장면만 캡처(Downsampling)해서 용량을 줄이고, 1년이 넘은 영상은 아예 지워버려(Retention) 하드디스크 공간을 확보하는 지혜로운 관리법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Roll-up]**: 다운샘플링의 다른 이름 (데이터를 말아 올림).
- **[Aggregated Data]**: 요약된 통계 데이터.
- **[TTL (Time To Live)]**: 보존 정책을 구현하는 기술적 파라미터.

📢 **마무리 요약**: **Downsampling & Retention**은 시계열 DB의 생존 기술입니다. 데이터의 신선도와 가치를 저울질하여 시스템의 지속 가능성을 확보하는 것이 설계자의 핵심 업무입니다.
