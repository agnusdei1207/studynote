# 261. 분산 데이터베이스(Distributed Database)의 목표

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 데이터베이스의 궁극적 목표는 물리적으로 흩어진 데이터들을 네트워크로 연결하여, 사용자가 **단일 시스템(Single System Image)을 사용하는 것과 동일한 논리적 편의성**을 제공하는 것이다.
> 2. **가치**: 데이터 부하 분산을 통한 **성능 향상**, 특정 노드 장애 시에도 서비스를 유지하는 **고가용성**, 그리고 필요에 따라 노드를 추가하는 **수평적 확장성**을 동시에 달성한다.
> 3. **융합**: '투명성(Transparency)'이라는 6가지 설계 원칙을 통해 복잡한 분산 구조를 사용자로부터 은닉하며, 현대 클라우드 네이티브 인프라의 핵심 저장소 역할을 수행한다.

+++

### Ⅰ. 분산 데이터베이스의 개요

- **정의**: 지리적으로 분산된 여러 컴퓨터 노드에 데이터를 나누어 저장하면서도, 소프트웨어적으로 통합 관리하여 하나의 논리적인 데이터베이스처럼 작동하는 시스템입니다.
- **핵심 키워드**: **Shared Nothing**, **Transparency**, **High Availability**.

+++

### Ⅱ. 분산 데이터베이스의 4대 핵심 목표

1. **위치 투명성 (Location Transparency)**: 데이터가 어디에 저장되어 있는지 몰라도 이름만으로 접근 가능해야 합니다.
2. **분할/복제 투명성**: 데이터가 쪼개져 있거나 여러 곳에 복사되어 있다는 사실을 사용자가 몰라도 됩니다.
3. **지역 자율성 (Local Autonomy)**: 각 노드는 자신의 데이터를 독자적으로 관리할 권한을 가집니다.
4. **효율적 데이터 처리**: 질의 처리 시 네트워크 전송량을 최소화하고 병렬 연산을 극대화합니다.

```text
[분산 데이터베이스의 논리적 뷰]

  (사용자 관점)           (물리적 현실)
  ┌──────────┐         ┌──────────┐   ┌──────────┐
  │ 단일 DB  │ ◀──(통합)── │  Node A  │───│  Node B  │
  │ (Logical)│         └──────────┘   └──────────┘
  └──────────┘               │              │
                             └──────┬───────┘
                                    ▼
                             [ Network Interconnect ]
```

+++

### Ⅲ. 분산 DB 도입의 기대효과

- **성능**: 쿼리를 여러 노드에서 동시 실행(Parallel Processing).
- **신뢰성**: 복제본(Replica)을 통해 데이터 유실 방지.
- **확장**: 서버 한 대의 한계를 넘어 수백 대 규모로 증설 가능.

- **📢 섹션 요약 비유**: 분산 DB의 목표는 **'프랜차이즈 식당의 본사 시스템'**과 같습니다. 고객은 서울이든 부산이든 어느 지점에 가도 똑같은 메뉴를 주문(위치 투명성)할 수 있고, 재료가 어디서 왔는지 신경 쓸 필요 없이 맛있는 음식을 먹을 수 있게 만드는 것이 목표입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[DDBMS]**: 분산 데이터베이스를 관리하는 소프트웨어 계층.
- **[CAP 정리]**: 분산 DB 설계 시 마주치는 물리적 한계.
- **[투명성]**: 분산 DB를 완성하는 설계의 6대 법칙.

📢 **마무리 요약**: **Distributed Database**는 거대한 데이터를 다루는 현대 IT의 심장입니다. 복잡성을 숨기고 사용자에게 단순함을 제공하는 것이 이 아키텍처의 위대한 사명입니다.
