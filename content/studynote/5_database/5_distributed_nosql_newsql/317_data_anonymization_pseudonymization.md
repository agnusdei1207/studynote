# 317. 데이터 익명화 및 가명화 기법 - 개인정보 보호의 기술

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 익명화는 특정 개인을 더 이상 식별할 수 없도록 **데이터를 영구적으로 변형**하는 것이며, 가명화는 추가 정보 없이는 식별할 수 없게 **식별자를 가짜 이름이나 토큰으로 대체**하는 기술이다.
> 2. **가치**: 개인정보 보호법(GDPR, 데이터 3법 등)을 준수하면서도, 원천 데이터의 통계적 특성을 유지하여 **빅데이터 분석 및 AI 학습에 안전하게 활용**할 수 있게 한다.
> 3. **융합**: K-익명성, L-다양성, T-근접성 등의 프라이버시 모델과 차분 프라이버시(Differential Privacy) 기술이 융합되어 데이터의 유용성과 보안 사이의 최적 균형을 찾는다.

+++

### Ⅰ. 익명화와 가명화의 차이

| 비교 항목 | 익명화 (Anonymization) | 가명화 (Pseudonymization) |
|:---|:---|:---|
| **복원 가능성** | **불가능** (비가역적) | 가능 (추가 정보/키 필요) |
| **개인정보 여부** | 개인정보 아님 (활용 자유) | 개인정보로 간주 (보호 조치 필요) |
| **핵심 기법** | 삭제, 총합, 범주화 | 마스킹, 치환, 토큰화 |
| **목적** | 완전한 비식별 데이터 배포 | 데이터 분석의 유연성 확보 |

+++

### Ⅱ. 비식별화 주요 기법 시각화 (ASCII Model)

```text
[ 데이터 변환 예시 ]

  (Original)  ID: 123 | Name: 홍길동 | Age: 28 | Disease: 독감
       │
       ▼ [ 비식별 기법 적용 ]
       
  1. 가명화 (Substitution) : ID: XXX | Name: 임꺽정 | Age: 28 | Disease: 독감
  2. 마스킹 (Masking)      : ID: 1** | Name: 홍*동 | Age: 28 | Disease: 독감
  3. 범주화 (Generalization): ID: --- | Name: --- | Age: 20대 | Disease: 독감
```

+++

### Ⅲ. 프라이버시 보호 모델

1. **K-익명성 (K-Anonymity)**: 동일한 속성 값을 가진 레코드가 최소 K개 이상 존재하게 하여 특정인을 짚어내지 못하게 함.
2. **L-다양성 (L-Diversity)**: K-익명성 내에서도 민감한 정보(질병 등)가 최소 L개 이상의 종류를 갖게 하여 유추 공격 방어.
3. **T-근접성 (T-Closeness)**: 전체 데이터의 민감 정보 분포와 특정 그룹의 분포 차이를 T 이하로 유지하여 정보 쏠림 방지.

- **📢 섹션 요약 비유**: 익명화와 가명화는 **'가면무도회'**와 같습니다. 가명화는 '가면(ID 대체)'을 써서 누구인지 모르지만 주최측은 명부를 보고 나중에 누군지 알 수 있는 상태이고, 익명화는 아예 얼굴 윤곽조차 안 보이게 '안개(범주화)'를 뿌려버려 주최측조차 누가 누구인지 영원히 알 수 없게 만드는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Differential Privacy]**: 데이터에 노이즈를 섞어 통계량은 유지하되 개인 정보는 철저히 숨기는 최신 기술.
- **[Re-identification]**: 비식별 데이터를 다른 정보와 결합하여 다시 개인을 찾아내는 공격 기법.
- **[Data Clean Room]**: 기업 간 익명화된 데이터를 안전하게 공유하고 분석하는 물리적/논리적 공간.

📢 **마무리 요약**: **Anonymization & Pseudonymization**은 데이터 경제의 윤리적 브레이크이자 가속기입니다. 철저한 비식별화 기술만이 데이터의 진정한 가치를 세상 밖으로 끌어낼 수 있습니다.
