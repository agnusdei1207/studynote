# 273. 동종 vs 이종 분산 DB 통합 - 하이브리드 인프라의 조율

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동종(Homogeneous) 분산 DB는 동일한 DBMS 소프트웨어를 사용하는 단순 통합인 반면, **이종(Heterogeneous) 분산 DB는 서로 다른 벤더나 데이터 모델(SQL, NoSQL 등)을 가진 저장소들을 하나의 논리적 시스템으로 묶는 기술**이다.
> 2. **가치**: 이종 통합은 기업 내 파편화된 레거시 시스템을 통합 조회할 수 있게 하며, 각 노드별로 최적화된 DB(예: 오라클+몽고DB)를 선택하는 '폴리글랏 퍼시스턴스' 환경을 지원한다.
> 3. **융합**: 서로 다른 쿼리 언어와 데이터 타입을 통일하기 위해 **게이트웨이(Gateway)** 및 **가상화(Virtualization)** 기술과 융합되어 거대한 가상 데이터망을 형성한다.

+++

### Ⅰ. 동종 분산 DB와 이종 분산 DB의 차이

| 비교 항목 | 동종 (Homogeneous) | 이종 (Heterogeneous) |
|:---|:---|:---|
| **DBMS 벤더** | 동일 (예: All Oracle) | **다양** (예: Oracle + MySQL) |
| **데이터 모델** | 동일 스키마 구조 | **서로 다름** (관계형, 문서형 등) |
| **통신 프로토콜** | 내장된 고유 프로토콜 사용 | 표준 프로토콜 (ODBC, JDBC, API) |
| **통합 난이도** | 낮음 (자동 지원) | **매우 높음** (중계기 필요) |

+++

### Ⅱ. 이종 분산 DB 통합 아키텍처 (ASCII Flow)

서로 다른 뇌를 가진 DB들을 연결하기 위해선 '통역사' 역할을 하는 **Global Schema**가 필요합니다.

```text
[이종 분산 DB 통합: 페더레이티드 모델]

      (User) "SELECT * FROM Global_User_View"
                │
      ┌─────────▼─────────┐
      │  Global Schema    │ (논리적 통합 뷰)
      │  Manager (Query)  │
      └─────────┬─────────┘
                │
      ┌─────────┴─────────┬──────────────┐
      ▼ (Mapping)         ▼ (Mapping)    ▼ (Mapping)
  [ Gateway A ]       [ Gateway B ]  [ Gateway C ]
      │                   │              │
  [ Oracle ]          [ MySQL ]      [ MongoDB ]
  (Relational)        (Relational)   (NoSQL)
```

+++

### Ⅲ. 이종 통합의 핵심 기술: 데이터 가상화

- **정의**: 데이터를 물리적으로 한곳에 모으지 않고, 질의 시점에만 실시간으로 각 DB에서 데이터를 가져와 합쳐주는 기술입니다.
- **도전 과제**:
    - **타입 변환**: 오라클의 `NUMBER`와 몽고DB의 `Double`을 어떻게 맞출 것인가?
    - **성능 저하**: 이종 DB 간의 조인(Join) 연산 시 네트워크 트래픽 제어가 매우 어렵습니다.

- **📢 섹션 요약 비유**: 동종 통합은 **'같은 나라 사람들끼리의 회의'**와 같아서 말이 잘 통하지만, 이종 통합은 **'다국적 정상회담'**과 같습니다. 서로 다른 언어(SQL vs NoSQL)를 쓰는 사람들이 모였으므로, 중간에 동시통역사(게이트웨이)를 세워야만 비로소 대화가 가능해지는 고난도의 작업입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Federated Database]**: 이종 DB 통합의 대표적인 시스템 형태.
- **[Polyglot Persistence]**: 목적에 따라 다른 DB를 쓰는 현대적 트렌드.
- **[Middleware]**: 서로 다른 DB 사이를 연결하는 접착제.

📢 **마무리 요약**: **Heterogeneous Integration**은 현대 엔터프라이즈의 숙명입니다. 파편화된 데이터를 하나의 지식으로 묶어내는 기술이 기업의 진정한 데이터 경쟁력을 결정합니다.
