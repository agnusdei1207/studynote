# 290. NewSQL - RDBMS의 신뢰와 NoSQL의 확장을 융합하다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: NewSQL은 전통적인 관계형 데이터베이스(RDBMS)의 **ACID 트랜잭션 및 SQL 표준을 유지하면서, NoSQL의 수평적 확장성(Scale-out)을 동시에 제공**하는 차세대 데이터베이스 아키텍처다.
> 2. **가치**: "성능을 위해 일관성을 포기(BASE)"해야 했던 과거의 딜레마를 극복하여, 분산 환경에서도 전역적(Global) 데이터 정합성과 고성능 처리를 모두 가능케 한다.
> 3. **융합**: 구글 스패너(Spanner)를 필두로 CockroachDB, TiDB 등이 있으며, 분산 합의 알고리즘(Raft/Paxos)과 클라우드 네이티브 스토리지 기술이 완벽하게 융합된 결과물이다.

+++

### Ⅰ. NewSQL의 등장 배경 및 정의

- **배경**: 
    - RDBMS: 데이터 신뢰성은 높지만 확장성이 한계(Scale-up 위주).
    - NoSQL: 확장은 쉽지만 복잡한 트랜잭션과 일관성 유지에 취약.
- **정의**: 두 패러다임의 장점만 취합하여, SQL을 쓰면서도 서버 수천 대에 데이터를 무한히 늘려 저장할 수 있는 시스템입니다.

+++

### Ⅱ. NewSQL 아키텍처 시각화 (ASCII Flow)

NewSQL은 내부적으로 데이터를 잘게 쪼개어 분산시키면서도, 논리적으로는 하나의 큰 RDBMS처럼 보입니다.

```text
[NewSQL의 계층적 분산 구조]

      (Standard SQL Query) ──▶ [ SQL Layer / Proxy ]
                                     │ (Logical View)
          ┌──────────────────────────┼──────────────────────────┐
          ▼                          ▼                          ▼
    [ Compute Node 1 ]         [ Compute Node 2 ]         [ Compute Node 3 ]
          │                          │                          │
    ┌─────┴─────┐              ┌─────┴─────┐              ┌─────┴─────┐
    │ Raft Group│ ◀──(Consensus)─▶ Raft Group│ ◀──(Consensus)─▶ Raft Group│
    └─────┬─────┘              └─────┬─────┘              └─────┬─────┘
          ▼                          ▼                          ▼
    [ Distributed Storage ]    [ Distributed Storage ]    [ Distributed Storage ]
```

+++

### Ⅲ. NewSQL의 핵심 기술 요소

1. **분산 합의 (Consensus)**: Raft나 Paxos를 사용하여 노드 장애 시에도 데이터 복제본 간의 완벽한 일치 보장.
2. **분산 트랜잭션**: 2단계 커밋(2PC)의 성능 한계를 극복하기 위한 최적화된 잠금 매커니즘.
3. **글로벌 타이머**: 구글 스패너의 TrueTime처럼 원자시계를 사용하여 전 세계 노드 간의 시간 오차를 극복하고 일관성 유지.

- **📢 섹션 요약 비유**: NewSQL은 **'어벤져스 팀'**과 같습니다. 아이언맨(RDBMS)의 정교한 기술과 헐크(NoSQL)의 압도적인 힘을 한 팀으로 묶은 것이죠. 어떤 강력한 적(빅데이터 트래픽)이 나타나도 전술(SQL)을 지키면서 힘(확장성)으로 밀어붙일 수 있는 가장 완벽한 형태의 팀입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Scale-out SQL]**: NewSQL이 지향하는 물리적 지향점.
- **[Strong Consistency]**: 분산 환경에서도 포기하지 않는 RDB의 자존심.
- **[HTAP]**: 트랜잭션과 실시간 분석을 동시에 처리하는 NewSQL의 확장 영역.

📢 **마무리 요약**: **NewSQL**은 데이터베이스 진화의 종착역 중 하나입니다. "타협하지 않는 무결성"을 분산 환경에서 실현함으로써 현대 금융 및 기업용 클라우드 아키텍처의 핵심으로 급부상하고 있습니다.
