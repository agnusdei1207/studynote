# 264. 분할 투명성 (Fragmentation Transparency) - 파편화의 은닉

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분할 투명성은 전역 릴레이션이 물리적으로 여러 개의 단편(Fragment)으로 쪼개져 분산 저장되어 있다는 사실을 사용자로부터 은닉하여, **사용자가 데이터를 단일 논리적 단위로 취급하게 하는 성질**이다.
> 2. **가치**: 쿼리 작성 시 각 파편의 이름을 일일이 명시할 필요가 없으므로 **SQL 작성의 단순성**을 보장하며, 데이터 분할 구조가 변경되어도 응용 프로그램은 영향을 받지 않는다.
> 3. **융합**: 수평 분할(Horizontal)과 수직 분할(Vertical) 기법이 적용된 물리 계층을 논리 계층으로 통합하는 매핑 엔진의 핵심 역량이다.

+++

### Ⅰ. 분할 투명성의 작동 원리

- **정의**: 사용자가 전역 테이블 이름을 사용하면, DDBMS가 내부적으로 해당 테이블을 구성하는 여러 파편(Shards)들을 자동으로 찾아내어 조인(Union 등) 처리하는 능력입니다.
- **구현 방식**:
    - **수평 분할**: 특정 조건(예: 지역='서울')에 따라 행들을 나눔.
    - **수직 분할**: 특정 속성(예: 연봉 정보 분리)에 따라 열들을 나눔.

+++

### Ⅱ. 분할 투명성 시각화 (ASCII Flow)

```text
[분할 투명성을 통한 논리적 통합]

  (User View) : "SELECT * FROM Member;"
       │
  ┌────▼────────────────────────────────┐
  │ DDBMS (Fragmentation Manager)       │
  │ Member = Seoul_Part ∪ Pusan_Part    │
  └────┬────────────────────────────────┘
       │
  ┌────┴─────────┐               ┌─────────┴────────┐
  │  Node 1      │               │  Node 2          │
  │ [Seoul_Part] │               │ [Pusan_Part]     │
  └──────────────┘               └──────────────────┘
```

+++

### Ⅲ. 실무적 의의: 관리적 유연성

- **데이터 재배치**: 성능 향상을 위해 서울 파편을 다른 서버로 옮겨도 사용자는 여전히 `Member` 테이블만 호출하면 됩니다.
- **부하 분산**: 특정 파편에 트래픽이 몰릴 때, 해당 파편만 더 고성능 노드로 이동시키는 최적화가 가능해집니다.

- **📢 섹션 요약 비유**: 분할 투명성은 **'백과사전 세트'**와 같습니다. 전 30권으로 나뉘어(분할) 꽂혀 있지만, 내가 "세종대왕 찾아줘"라고 요청하면 기계가 알아서 몇 권(파편)에 있는지 찾아 내용을 보여주는 것과 같습니다. 사용자는 세종대왕이 몇 권에 있는지 미리 외울 필요가 없습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[단편화 기법]**: 수평, 수직, 혼합 분할의 기초.
- **[전역 스키마]**: 분할된 정보를 하나로 묶어 보여주는 가상의 지도.
- **[샤딩]**: 분할 투명성을 실무적으로 구현한 현대의 기술.

📢 **마무리 요약**: **Fragmentation Transparency**는 데이터의 파편화를 공학적인 아름다움으로 승화시킵니다. 물리적 분할의 복잡성을 제거하여 개발의 편의성을 완성합니다.
