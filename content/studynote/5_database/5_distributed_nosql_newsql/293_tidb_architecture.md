# 293. 티아이디비 (TiDB) - 실시간 HTAP의 선두주자

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TiDB(Titanium DB)는 MySQL과 완벽하게 호환되면서도 **컴퓨팅과 스토리지를 분리한 아키텍처를 통해 수평적 확장성을 실현한 오픈소스 분산 NewSQL 데이터베이스**다.
> 2. **가치**: 행 기반 저장소(TiKV)와 열 기반 저장소(TiFlash)를 동시에 운영하여, **OLTP(트랜잭션)와 OLAP(분석) 워크로드를 하나의 시스템에서 실시간으로 처리(HTAP)**하는 강력한 통합 기능을 제공한다.
> 3. **융합**: Raft 합의 알고리즘 기반의 고가용성과 분산 쿼리 엔진 기술이 융합되어, 복잡한 ETL 과정 없이 실시간 데이터 통찰이 필요한 현대 비즈니스에 최적의 솔루션을 제시한다.

+++

### Ⅰ. TiDB의 계층형 아키텍처

1. **TiDB Server**: 무상태(Stateless) 컴퓨팅 계층. 사용자의 SQL을 받고 최적화하여 실행 계획을 수립합니다.
2. **TiKV**: 행(Row) 기반 분산 키-값 저장소. Raft를 통해 데이터를 복제하며 주로 트랜잭션 업무를 처리합니다.
3. **TiFlash**: 열(Column) 기반 저장소. TiKV의 데이터를 실시간 복제하여 대규모 분석 쿼리를 가속합니다.
4. **PD (Placement Driver)**: 클러스터의 메타데이터 관리 및 데이터 로드 밸런싱을 지휘하는 두뇌 역할을 합니다.

+++

### Ⅱ. HTAP 아키텍처 시각화 (ASCII Flow)

```text
[ TiDB: Real-time HTAP Engine ]

      (User Application)
             │
      ┌──────▼──────┐
      │ TiDB Server │ (MySQL Compatible)
      └──────┬──────┘
             │
      ┌──────┴──────────────────┐
      ▼                         ▼
  [ TiKV Group ]           [ TiFlash Group ]
  (Row-based)              (Column-based)
  - OLTP (Update/Insert)   - OLAP (Aggregations)
  - Strong Consistency     - High Speed Analytics
```

+++

### Ⅲ. 주요 특징 및 이점

- **MySQL 호환성**: 애플리케이션 코드를 거의 수정하지 않고도 단일 MySQL에서 분산 TiDB로 전환이 가능합니다.
- **자동 샤딩 및 리밸런싱**: 데이터가 늘어나면 PD가 알아서 노드 간 균형을 맞추며, 수동 샤딩의 고통을 없애줍니다.
- **실시간 비즈니스 인텔리전스**: 별도의 DW 전송(ETL) 없이 운영 데이터를 즉시 분석하여 최신 트렌드를 파악할 수 있습니다.

- **📢 섹션 요약 비유**: TiDB는 **'만능 주방 도구'**와 같습니다. 재료를 썰고 볶는 일상적인 요리(OLTP)도 잘하면서, 동시에 수만 명의 식사 통계를 내는 복잡한 계산(OLAP)도 한 주방에서 해결하는 것과 같습니다. MySQL이라는 익숙한 손잡이 덕분에 누구나 쉽게 사용할 수 있는 것도 큰 장점입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[HTAP]**: TiDB가 추구하는 궁극의 기술적 지향점.
- **[Placement Driver]**: 클러스터 전체의 오케스트레이터.
- **[Raft]**: 데이터의 생존과 일관성을 책임지는 기저 프로토콜.

📢 **마무리 요약**: **TiDB**는 분산 DB의 복잡함과 분석의 어려움을 동시에 해결한 혁신적인 시스템입니다. 클라우드 네이티브 환경에서 데이터의 활용도를 극대화하는 최고의 선택지입니다.
