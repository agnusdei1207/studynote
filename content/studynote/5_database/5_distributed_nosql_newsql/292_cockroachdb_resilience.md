# 292. 칵로치DB (CockroachDB) - 불사조의 생존력

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 칵로치DB(CockroachDB)는 이름처럼 끈질긴 생존력을 목표로 설계된 **분산형 NewSQL 데이터베이스로, 표준 SQL과 ACID 트랜잭션을 지원하며 데이터 센터 단위의 장애에도 무중단 서비스를 제공**한다.
> 2. **가치**: Raft 합의 알고리즘을 통한 강력한 일관성과 자동 샤딩 기능을 결합하여, 관리자의 수동 개입 없이도 클러스터의 확장과 복구를 스스로 처리한다.
> 3. **융합**: PostgreSQL과의 높은 호환성을 유지하면서도 클라우드 네이티브 환경에 최적화되어 있어, 레거시 시스템의 현대화와 글로벌 확장이 필요한 서비스의 핵심 저장소로 융합된다.

+++

### Ⅰ. 칵로치DB의 핵심 철학 및 기술

- **Design for Failure**: 노드, 랙(Rack), 심지어 리전(Region) 전체가 중단되어도 데이터 유실 없이 서비스를 지속하는 '복원력'이 최우선 순위입니다.
- **Raft Consensus**: 모든 데이터 조각(Range)은 최소 3개 이상의 복제본을 가지며, Raft 알고리즘을 통해 과반수 합의로 데이터를 업데이트합니다.
- **Shared-Nothing**: 각 노드가 대등한 권한을 가지며 데이터를 나누어 가짐으로써 병목 지점을 제거합니다.

+++

### Ⅱ. 데이터 복제 및 복구 구조 (ASCII Diagram)

```text
[CockroachDB Raft Group & Survival]

  [ Range 1: Key A~M ]
  ┌──────────┬──────────┬──────────┐
  │ Node A   │ Node B   │ Node C   │
  │ (Leader) │(Follower)│(Follower)│
  └────┬─────┴────┬─────┴────┬─────┘
       │          │          │
       └──── (Heartbeat) ────┘
       
  (Node A Crash! 💥) ──▶ Node B, C가 감지
  ──▶ Node B가 새로운 Leader로 즉시 선출 👑
  ──▶ 서비스 중단 없이 [ Range 1 ] 접근 가능 ✅
```

+++

### Ⅲ. 주요 특징

1. **PostgreSQL 호환**: 기존 포스트그레스 드라이버와 툴을 그대로 사용할 수 있어 진입 장벽이 낮습니다.
2. **Geo-Partitioning**: 사용자의 물리적 위치와 가장 가까운 리전에 데이터를 자동으로 배치하여 지연 시간을 최소화합니다.
3. **무한 확장**: 서버를 추가하기만 하면 데이터가 자동으로 재분배(Rebalancing)되며 성능이 선형적으로 증가합니다.

- **📢 섹션 요약 비유**: 칵로치DB는 **'군집 생활을 하는 개미들'**과 같습니다. 개미 한두 마리가 없어져도 전체 개미집은 아무 문제 없이 굴러가듯이, 서버 몇 대가 고장 나도 나머지 서버들이 즉시 역할을 대신하여 전체 시스템의 생명을 유지하는 질긴 생명력이 특징입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Raft]**: 칵로치DB의 정합성을 책임지는 합의 엔진.
- **[Range]**: 칵로치DB가 데이터를 쪼개는 기본 단위 (약 64MB).
- **[Distributed SQL]**: 분산 환경에서 돌아가는 표준 SQL 엔진.

📢 **마무리 요약**: **CockroachDB**는 클라우드 시대의 가장 신뢰할 수 있는 저장소 중 하나입니다. "죽지 않는 데이터베이스"라는 가치를 통해 기업의 데이터 주권을 완벽하게 보호합니다.
