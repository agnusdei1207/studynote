# 311. 데이터 메쉬 vs 데이터 패브릭 - 차세대 데이터 거버넌스

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 메쉬(Data Mesh)는 데이터 소유권을 각 비즈니스 도메인에 위임하는 **'분권화된 조직적 접근'**이며, 데이터 패브릭(Data Fabric)은 메타데이터와 AI를 활용해 파편화된 데이터를 가상으로 통합하는 **'기술 중심의 자동화 접근'**이다.
> 2. **가치**: 데이터 메쉬는 중앙 집중형 데이터 팀의 병목을 해소하여 도메인 전문성을 극대화하고, 데이터 패브릭은 복잡한 하이브리드 클라우드 환경에서 실시간 데이터 가시성과 연결성을 제공한다.
> 3. **융합**: 현대 기업은 기술적 통합(Fabric)을 통해 기반을 닦고, 운영 방식(Mesh)을 통해 책임 경영을 실현하는 상호 보완적 전략으로 두 개념을 융합하여 사용한다.

+++

### Ⅰ. 두 패러다임의 상세 비교

| 비교 항목 | 데이터 메쉬 (Data Mesh) | 데이터 패브릭 (Data Fabric) |
|:---|:---|:---|
| **핵심 사상** | **분권화 (Decentralization)** | **통합 (Integration)** |
| **주요 동력** | 사람, 조직, 프로세스 (Domain-driven) | 메타데이터, AI/ML, 자동화 |
| **데이터 형태** | 데이터 제품 (Data as a Product) | 가상 통합 층 (Data Virtualization) |
| **적합 조직** | 도메인별 자율성이 강한 대규모 조직 | 데이터 원천이 극도로 파편화된 복잡한 조직 |

### Ⅲ. 핵심 가치와 선택 기준

- **Data Mesh의 장점**: 도메인 전문가가 데이터를 직접 관리하므로 데이터의 품질과 활용도가 높습니다. "데이터를 가장 잘 아는 사람이 제품화하라"는 철학입니다.
- **Data Fabric의 장점**: 사용자가 데이터의 위치를 몰라도 지능형 엔진이 최적의 경로로 데이터를 연결해 줍니다. "기술이 데이터 사일로를 자동으로 극복하라"는 철학입니다.

- **📢 섹션 요약 비유**: 데이터 메쉬는 **'각국 대사관이 모인 UN'**과 같습니다. 각 나라(도메인)가 자기 정체성을 유지하며 협력하는 구조입니다. 반면 데이터 패브릭은 **'모든 언어를 완벽히 번역하는 지능형 통역기'**와 같습니다. 나라가 수백 개여도 통역기(기술 레이어)만 있으면 누구나 자유롭게 소통할 수 있게 만드는 자동화된 망입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Data Product]**: 데이터 메쉬에서 제공하는 최종 결과물.
- **[Active Metadata]**: 데이터 패브릭이 분석하고 활용하는 살아있는 정보.
- **[Silo]**: 두 기술이 공통적으로 타파하고자 하는 데이터 격립 현상.

📢 **마무리 요약**: **Data Mesh와 Data Fabric**은 데이터 거버넌스의 양대 산맥입니다. 조직의 문화와 기술적 복잡도를 고려하여 분권화와 자동화의 최적점을 찾아야 합니다.
