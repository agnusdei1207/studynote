# 291. 구글 스패너 (Google Cloud Spanner) - 글로벌 일관성의 정점

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 구글 스패너는 전 세계에 흩어진 데이터 센터에 데이터를 분산 저장하면서도, **ACID 트랜잭션과 강력한 외부 일관성(External Consistency)을 보장하는 구글의 완전 관리형 NewSQL 데이터베이스**다.
> 2. **가치**: **TrueTime(원자시계+GPS)** 기술을 사용하여 분산 노드 간의 시간 오차를 극복하고, 글로벌 스케일에서도 정확한 읽기/쓰기 순서를 보장하여 데이터 정합성 문제를 해결한다.
> 3. **융합**: 관계형 데이터 모델의 장점과 NoSQL의 수평적 확장성을 융합하였으며, 현대 글로벌 대규모 금융 및 엔터프라이즈 시스템의 핵심 백엔드로 기능한다.

+++

### Ⅰ. 구글 스패너의 핵심 아키텍처

- **Global Distribution**: 데이터를 대륙별, 지역별로 자동 복제 및 샤딩(Split)합니다.
- **TrueTime API**: 스패너의 가장 독특한 기술로, 각 서버에 원자시계와 GPS를 설치하여 전역적으로 동기화된 타임스탬프 범위를 제공합니다.
- **Paxos 합의**: 복제본 간의 데이터 일치를 위해 팍소스 알고리즘을 변형하여 사용합니다.

+++

### Ⅱ. TrueTime 기반 일관성 메커니즘 (ASCII Diagram)

```text
[ Spanner TrueTime & Commit ]

  1. Request: Write(A) at Node 1
  2. Node 1 asks TrueTime: "What time is it?"
  3. TrueTime: "It's [T_earliest, T_latest]" (오차 범위 포함)
  4. Node 1 sets Commit Time = T_latest
  5. **Commit Wait**: T_latest 시점이 실제로 지나갈 때까지 대기
  6. Finalize: "이제 전 세계 누구도 이 시간보다 과거일 수 없음"
  ──▶ ✅ 글로벌 정합성 확보!
```

+++

### Ⅲ. 주요 특징 및 이점

1. **강력한 일관성 (Strong Consistency)**: 분산 환경임에도 불구하고 단일 DB를 쓰는 것과 같은 데이터 무결성을 제공합니다.
2. **무중단 스키마 변경**: 운영 중인 거대 데이터베이스의 스키마를 다운타임 없이 실시간으로 수정 가능합니다.
3. **고가용성**: 데이터 센터 한두 곳이 통째로 날아가도 99.999%의 가용성을 유지합니다.

- **📢 섹션 요약 비유**: 구글 스패너는 **'전 세계를 하나로 연결하는 거대한 정밀 시계'**와 같습니다. 나라마다 시간이 조금씩 다르듯 서버마다 시계가 다르지만, 스패너는 원자시계라는 절대 기준을 세워 "누가 먼저 글을 썼는지"를 전 지구적으로 정확하게 판별해 내는 놀라운 장치입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[TrueTime]**: 스패너의 심장이자 글로벌 일관성의 근거.
- **[Directory]**: 스패너가 데이터를 샤딩하는 논리적 단위.
- **[External Consistency]**: 트랜잭션이 완료된 순서대로 누구나 데이터를 보게 되는 최상위 일관성.

📢 **마무리 요약**: **Google Cloud Spanner**는 분산 DB 기술의 정점입니다. "확장성 때문에 일관성을 포기하지 않아도 된다"는 것을 증명한 역사적인 시스템입니다.
