# 318. 제로 트러스트(Zero Trust) DB 보안 - 신뢰는 없다, 검증하라

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제로 트러스트 DB 보안은 "내부 네트워크는 안전하다"는 전통적인 경계 보안의 한계를 인정하고, **데이터베이스에 접근하는 모든 주체(사람, 기기, 앱)를 잠재적 위협으로 간주하여 매 순간 철저히 검증하는 보안 모델**이다.
> 2. **가치**: 한 번의 인증으로 모든 권한을 주는 것이 아니라, **최소 권한 원칙(Principle of Least Privilege)**과 컨텍스트 기반의 실시간 인증을 통해 내부자에 의한 데이터 유출 및 지능형 지속 위협(APT)을 원천 차단한다.
> 3. **융합**: 마이크로 세그멘테이션(Micro-segmentation), 다요소 인증(MFA), 상시 모니터링 기술과 결합되어 현대 클라우드 환경의 데이터 보호 아키텍처를 완성한다.

+++

### Ⅰ. 제로 트러스트 DB 보안의 3대 핵심 원칙

1. **명시적 검증 (Verify Explicitly)**: 위치(IP)에 상관없이 사용자 아이디, 기기 상태, 접근 시간, 접속 위치 등 모든 가용 데이터를 활용해 매번 인증합니다.
2. **최소 권한 할당 (Use Least Privileged Access)**: 업무 수행에 꼭 필요한 만큼의 권한만, 필요한 시간 동안만 부여합니다 (Just-in-Time).
3. **침해 가정 (Assume Breach)**: 이미 네트워크가 뚫렸다고 가정하고, 데이터베이스 내부의 움직임을 세밀하게 쪼개어 감시(Micro-segmentation)합니다.

+++

### Ⅱ. 보안 모델의 패러다임 전환 (ASCII Model)

```text
[ 전통적 경계 보안 (Castle & Moat) ]     [ 제로 트러스트 (Verify All) ]

      (외부)  |  (내부: 신뢰)                (어디서든)  |  (모든 요청 검증)
      ────▶  |  ┌────────┐                  ────▶  |  [ PDP/PEP ] ──▶ [ DB ]
      인증 시  |  │ DB 접근  │                  매 요청  |  (Policy Enforcement)
      통과    |  └────────┘                  인증 필수 |
```

+++

### Ⅲ. DB 레벨의 제로 트러스트 구현 기술

- **mTLS (Mutual TLS)**: 앱 서버와 DB 서버가 서로의 인증서를 확인하며 양방향 암호화 통신을 수행합니다.
- **Dynamic Secrets**: 고정된 DB 비밀번호 대신, 일정 시간만 유효한 임시 계정을 발급하여 탈취 위험을 방지합니다 (HashiCorp Vault 등 활용).
- **데이터 마스킹 & 암호화**: 인가된 사용자라도 컨텍스트에 따라 데이터의 일부만 보이게 하거나(Dynamic Data Masking), 저장 데이터(TDE)와 전송 데이터를 모두 암호화합니다.

- **📢 섹션 요약 비유**: 제로 트러스트 보안은 **'고급 아파트의 보안 시스템'**과 같습니다. 예전 방식이 단지 입구(방화벽)만 통과하면 현관문까지 자유롭게 가는 것이었다면, 제로 트러스트는 단지 입구는 물론, 엘리베이터 앞, 그리고 집 현관문 앞에서도 매번 지문 인식(연속 검증)을 요구하며, 내 집 이외의 다른 층 버튼은 아예 누를 수 없게(최소 권한) 막는 철저한 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Identity-Centric]**: IP 주소가 아닌 '신원' 중심의 보안.
- **[Micro-segmentation]**: 네트워크를 잘게 쪼개어 공격자의 횡적 이동(Lateral Movement)을 차단하는 기술.
- **[IAM (Identity & Access Management)]**: 제로 트러스트의 핵심 관리 도구.

📢 **마무리 요약**: **Zero Trust**는 데이터 보안의 새로운 상식입니다. "신뢰하되 검증하라"는 과거의 격언을 버리고, "절대 신뢰하지 말고 항상 검증하라"는 철학으로 소중한 데이터를 지켜야 합니다.
