# 298. MongoDB 아키텍처 - 유연성과 분산의 결착

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: MongoDB는 데이터를 유연한 BSON(Binary JSON) 문서 형태로 저장하며, **레플리카 셋(Replica Set)을 통한 고가용성과 샤딩(Sharding)을 통한 수평적 확장성을 결합한 분산 도큐먼트 데이터베이스 아키텍처**다.
> 2. **가치**: 스키마의 제약 없이 정형/비정형 데이터를 통합 관리하며, 자동 페일오버(Failover)와 데이터 리밸런싱 기능을 통해 대규모 트래픽 환경에서도 안정적인 서비스 운영을 지원한다.
> 3. **융합**: 복잡한 계층 구조를 가진 객체 데이터를 조인 없이 처리하는 임베디드(Embedded) 모델과 클라우드 네이티브 환경의 오케스트레이션 기술이 융합되어 현대 웹 서비스의 표준 저장소로 자리 잡았다.

+++

### Ⅰ. MongoDB의 분산 구성 요소

1. **mongos (Query Router)**: 클라이언트의 요청을 받고 적절한 샤드로 라우팅하는 게이트웨이입니다.
2. **Config Server**: 클러스터의 메타데이터 및 샤딩 정보(어떤 데이터가 어디 있는지)를 저장합니다.
3. **Shard (Data Node)**: 실제 데이터가 분산 저장되는 물리적 노드입니다. 각 샤드는 보통 가용성을 위해 **Replica Set**으로 구성됩니다.

+++

### Ⅱ. MongoDB 클러스터 시각화 (ASCII Flow)

```text
[ MongoDB Sharded Cluster Architecture ]

      (Client App) ──▶ [ mongos ] ◀──(Read Meta)── [ Config Server ]
                          │
          ┌───────────────┼───────────────┐
          ▼               ▼               ▼
    [ Shard A ]     [ Shard B ]     [ Shard C ]
    ┌─────────┐     ┌─────────┐     ┌─────────┐
    │ Primary │     │ Primary │     │ Primary │
    ├─────────┤     ├─────────┤     ├─────────┤
    │ Sec/Sec │     │ Sec/Sec │     │ Sec/Sec │ (Replica Sets)
    └─────────┘     └─────────┘     └─────────┘
```

+++

### Ⅲ. 핵심 운영 매커니즘

- **자동 페일오버**: 레플리카 셋의 Primary 노드가 죽으면 Secondary 노드들이 투표(Election)를 통해 10~30초 내에 새로운 Primary를 선출합니다.
- **Chunk & Balancer**: 데이터를 '청크(Chunk)' 단위로 관리하며, 특정 샤드에 데이터가 쏠리면 백그라운드의 Balancer가 한가한 샤드로 청크를 자동 이동(Migration)시킵니다.
- **WiredTiger 엔진**: 스냅샷 격리 및 문서 레벨의 락킹(Locking)을 지원하는 고성능 스토리지 엔진입니다.

- **📢 섹션 요약 비유**: 몽고DB 아키텍처는 **'거대 물류 센터'**와 같습니다. 고객의 주문(쿼리)을 받는 안내원(mongos)이 있고, 물건이 어디 있는지 기록한 장부(Config)를 확인해 실제 창고(Shard)로 안내합니다. 창고마다 화재에 대비해 똑같은 물건을 둔 보조 창고(Replica Set)가 딸려 있어 사고가 나도 배송에 차질이 없게 설계된 완벽한 시스템입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[BSON]**: JSON의 이진 표현 방식 (공간 효율 및 속도 향상).
- **[Secondary Index]**: 문서 내부 필드에 대한 강력한 검색 지원.
- **[Aggregation Pipeline]**: 복잡한 데이터 분석을 위한 전용 프레임워크.

📢 **마무리 요약**: **MongoDB Architecture**는 확장이 필요한 모든 현대적 서비스의 훌륭한 해답입니다. 유연한 그릇에 거대한 데이터를 담아내면서도 생존과 성장을 동시에 책임지는 아키텍처입니다.
