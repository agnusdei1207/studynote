# 304. 데이터 레이크하우스 (Data Lakehouse) - 통합의 아키텍처

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 레이크하우스는 **데이터 레이크(Data Lake)의 유연하고 저렴한 저장 능력과 데이터 웨어하우스(Data Warehouse)의 강력한 관리 기능(ACID, 스키마 제어)을 단일 시스템으로 결합**한 차세대 데이터 아키텍처다.
> 2. **가치**: 운영 데이터(BI)와 AI/머신러닝(Data Science)을 위해 데이터를 중복으로 옮길 필요 없이 하나의 저장소에서 동시에 처리하여, 데이터 사일로(Silo) 현상을 방지하고 관리 효율을 극대화한다.
> 3. **융합**: Delta Lake, Iceberg, Hudi와 같은 오픈 스토리지 포맷 기술과 결합되어 클라우드 오브젝트 스토리지 위에서도 고성능 쿼리와 트랜잭션을 지원하는 현대적 분석 인프라로 융합된다.

+++

### Ⅰ. 데이터 레이크하우스의 등장 배경

- **DW의 한계**: 정형 데이터 위주이며, 머신러닝 분석에 부적합하고 확장 비용이 비쌉니다.
- **DL의 한계**: 비정형 데이터 저장은 좋으나, 데이터 무결성 보장이 어렵고 정밀한 쿼리 성능이 떨어집니다 (Data Swamp 위험).
- **Lakehouse의 탄생**: "데이터 레이크 위에 웨어하우스의 엔진을 얹자!"는 아이디어로 탄생했습니다.

+++

### Ⅱ. 레이크하우스 아키텍처 시각화 (ASCII Model)

```text
[ Data Lakehouse Layers ]

  ┌─────────────────────────────────────────┐
  │  AI / ML / BI / Real-time Analytics     │ (Unified Query)
  ├─────────────────────────────────────────┤
  │  Metadata / Caching / Indexing / ACID   │ (Warehouse Layer)
  ├─────────────────────────────────────────┤
  │  Open Formats (Parquet, Avro, Delta)    │ (Storage Layer)
  ├─────────────────────────────────────────┤
  │  Object Storage (AWS S3, Azure Blob)    │ (Infrastructure)
  └─────────────────────────────────────────┘
```

+++

### Ⅲ. 레이크하우스의 핵심 기술 요소

1. **ACID 트랜잭션**: 오브젝트 스토리지 위에서도 여러 사용자가 동시에 데이터를 쓰고 읽을 때 무결성을 보장합니다.
2. **스키마 강제 및 진화 (Schema Enforcement)**: 레이크로 들어오는 데이터의 품질을 검증하고, 비즈니스 변화에 맞춰 스키마를 안전하게 변경합니다.
3. **버전 관리 (Time Travel)**: 과거 특정 시점의 데이터로 되돌아가거나 조회할 수 있는 기능을 제공합니다.

- **📢 섹션 요약 비유**: 레이크하우스는 **'똑똑한 거대 창고'**와 같습니다. 예전에는 잡동사니를 다 던져넣는 '마당(레이크)'과 금고에 보물만 넣는 '은행(DW)'이 따로 있었다면, 레이크하우스는 마당처럼 넓고 싸면서도 은행처럼 물건마다 바코드를 붙여(스키마) 도둑이 못 가져가게 감시(ACID)까지 해주는 만능 창고입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Delta Lake]**: 레이크하우스 개념을 처음 대중화시킨 오픈소스 프로젝트.
- **[Medallion Architecture]**: Bronze -> Silver -> Gold로 이어지는 데이터 정제 레이어.
- **[Single Source of Truth]**: 레이크하우스가 지향하는 단일 진실 공급원.

📢 **마무리 요약**: **Data Lakehouse**는 데이터 엔지니어링의 최종 진화형 중 하나입니다. 복잡한 데이터 파이프라인을 단순화하고, 전사적인 데이터 활용 속도를 가속화하는 클라우드 시대의 정답지입니다.
