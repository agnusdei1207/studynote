# 277. 문서 저장소 (Document Store) - 데이터의 자유로운 진화

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 문서 저장소는 데이터를 행과 열의 집합이 아닌, **JSON, BSON, XML 등의 계층적 문서(Document) 형태로 저장하고 관리**하는 비관계형 데이터베이스다.
> 2. **가치**: 스키마리스(Schema-less) 특성을 통해 복잡한 중첩 구조를 하나의 문서에 담을 수 있어, 조인(Join) 없이도 관련 데이터를 한 번에 읽어오는 압도적인 조회 성능을 제공한다.
> 3. **융합**: 객체 지향 프로그래밍 언어의 데이터 구조와 일치하는 특성 덕분에 현대 웹 및 모바일 앱 개발에서 데이터 매핑 오버헤드를 최소화하는 핵심 저장소로 융합된다.

+++

### Ⅰ. 문서 저장소의 핵심 특징

- **자기 기술적 (Self-describing)**: 각 문서 안에 데이터의 키(Key)와 값(Value)이 함께 포함되어 있어, 문서마다 구조가 달라도 상관없습니다.
- **중첩 구조 허용**: 리스트나 하위 객체를 문서 내에 포함(Embedding)할 수 있어 복잡한 엔티티를 하나의 단위로 처리합니다.
- **인덱싱 지원**: 단순 KV와 달리 문서 내부의 특정 필드에 대해 보조 인덱스(Secondary Index)를 생성하여 강력한 검색 기능을 제공합니다.

+++

### Ⅱ. 문서 구조 시각화 (ASCII Model)

```text
[Document 구조: MongoDB 사례]

  {
    "_id": "user777",
    "name": "Gemini",
    "interests": ["AI", "Cloud", "DB"],  <-- Array 지원
    "address": {                         <-- Nested Object
       "city": "Seoul",
       "zip": "12345"
    },
    "version": 1.0
  }
```

+++

### Ⅲ. 대표 제품 및 실무 활용

1. **MongoDB**: 가장 대중적인 문서형 DB. 풍부한 쿼리 언어와 자동 샤딩 지원.
2. **CouchDB**: HTTP/JSON 기반의 복제 및 동기화에 강점을 가진 DB.
3. **Firestore**: 모바일 앱 개발에 최적화된 구글의 클라우드 문서 저장소.

- **📢 섹션 요약 비유**: 문서 저장소는 **'스크랩북'**과 같습니다. 관계형 DB가 칸이 나누어진 '명함첩'이라면, 문서형 DB는 한 페이지에 사진, 일기, 티켓 영수증을 마음대로 붙일 수 있는 스크랩북입니다. 페이지마다 구성이 달라도(스키마리스) 한 권으로 묶을 수 있고, 한 페이지만 펼치면 그날의 모든 기억(연관 데이터)을 한눈에 볼 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Embedding vs Reference]**: 조인을 피하기 위한 문서 설계의 핵심 전략.
- **[BSON]**: 몽고DB에서 사용하는 이진 직렬화 포맷.
- **[Dynamic Schema]**: 비즈니스 변화에 즉각 대응하는 문서 저장소의 자부심.

📢 **마무리 요약**: **Document Store**는 유연함의 대명사입니다. 고정된 틀에 데이터를 맞추는 대신, 데이터의 흐름에 따라 저장소를 진화시키는 현대적 아키텍처의 필수 요소입니다.
