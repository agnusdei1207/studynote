# 286. 시계열 데이터베이스 (TSDB) - 시간의 흐름을 기록하다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시계열 데이터베이스(TSDB)는 시간의 흐름에 따라 발생하는 **이벤트나 수치 데이터를 타임스탬프(Timestamp)와 함께 순차적으로 저장하고 조회하는 데 최적화된 비관계형 저장소**이다.
> 2. **가치**: 초당 수백만 건의 쓰기(Write) 요청을 처리하며, 시간 범위 기반의 집계 연산(Aggregation)과 데이터 압축 기능을 통해 대규모 이력 데이터를 효율적으로 관리한다.
> 3. **융합**: 인프라 모니터링(Prometheus), IoT 센서 데이터 수집, 금융 주식 차트 분석 등 시간축 중심의 의사결정이 필요한 모든 IT 인프라의 핵심 엔진으로 융합된다.

+++

### Ⅰ. TSDB의 핵심 특징

- **Append-only**: 기존 데이터를 수정하기보다 새로운 시간대의 데이터를 계속해서 덧붙여 나갑니다.
- **Timestamp Indexing**: 모든 데이터의 기본 키가 시간(Time)이며, 시간순 정렬이 물리적으로 보장됩니다.
- **높은 처리량**: 대량의 실시간 데이터를 지연 없이 적재하기 위해 LSM-Tree 등의 쓰기 최적화 엔진을 사용합니다.

+++

### Ⅱ. 시계열 데이터 구조 시각화 (ASCII Map)

```text
[ TSDB의 데이터 모델: Metric ]

  (Metric Name) : cpu_usage
  (Tags/Labels) : host=server01, region=seoul
  (Timestamp)   : 2026-03-16 10:00:01
  (Value)       : 45.2%

  [ 물리적 저장 레이아웃 ]
  Time ──▶ [ Value 1 ][ Value 2 ][ Value 3 ] ...
  (10:01)   (45.2)     (46.1)     (44.8)
```

+++

### Ⅲ. 대표 제품 및 활용 분야

1. **InfluxDB**: SQL과 유사한 쿼리를 지원하는 대표적인 오픈소스 TSDB.
2. **Prometheus**: 클라우드 네이티브 환경(Kubernetes) 모니터링의 표준.
3. **TimescaleDB**: PostgreSQL 엔진 위에 시계열 기능을 얹은 하이브리드 모델.

- **📢 섹션 요약 비유**: TSDB는 **'끊임없이 돌아가는 블랙박스'**와 같습니다. 매 순간 일어나는 일을 기록(쓰기)하는 데 아주 특화되어 있고, 사고가 났을 때 "정확히 몇 시 몇 분에 무슨 일이 있었나?"를 찾아보기(시간 기반 조회)에 가장 최적화된 특수 기록 장치입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Downsampling]**: 오래된 데이터의 해상도를 낮추어 용량을 아끼는 기술.
- **[Retention Policy]**: 일정 기간이 지난 데이터를 자동으로 삭제하는 보존 정책.
- **[LSM-Tree]**: TSDB의 폭발적인 쓰기 속도를 지탱하는 물리 구조.

📢 **마무리 요약**: **TSDB**는 과거와 현재를 잇는 데이터의 선입니다. 시간이라는 축 위에서 데이터의 의미를 찾아내는 현대 모니터링 아키텍처의 정수입니다.
