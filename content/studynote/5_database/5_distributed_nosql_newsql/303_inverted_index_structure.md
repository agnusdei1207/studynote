# 303. 역색인 (Inverted Index) - 검색 속도의 비밀

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 역색인(Inverted Index)은 데이터를 문서별로 저장하는 일반적인 방식과 달리, **단어(Term)를 기준으로 해당 단어가 포함된 문서 ID들의 리스트를 매핑**하여 보관하는 검색 전용 자료구조다.
> 2. **가치**: 수억 건의 문서 속에서도 특정 키워드를 포함한 대상을 순식간에 찾아낼 수 있으며, 'LIKE %검색'과 달리 데이터량이 늘어나도 조회 성능 저하가 거의 없다.
> 3. **융합**: Elasticsearch, Lucene 등의 전문 검색 엔진의 핵심 알고리즘이며, 형태소 분석(Tokenizing) 기술과 융합되어 자연어 처리 및 지능형 검색 서비스의 기반이 된다.

+++

### Ⅰ. 정방향 색인 vs 역색인 비교

- **정방향 색인 (Forward Index)**: 문서 ID를 기준으로 단어들을 나열함. (예: 문서1 ──▶ [AI, DB, Cloud])
- **역색인 (Inverted Index)**: 단어를 기준으로 문서 ID를 나열함. (예: AI ──▶ [문서1, 문서3])

+++

### Ⅱ. 역색인 구조 시각화 (ASCII Map)

```text
[ 원본 문서 (Documents) ]
  Doc 1: "Database is good"
  Doc 2: "AI is power"
  Doc 3: "Database and AI"

[ 역색인 테이블 (Inverted Index) ]
  (Term)      (Posting List: Doc IDs)
  ┌──────────┬──────────────┐
  │ AI       │ Doc 2, Doc 3 │
  ├──────────┼──────────────┤
  │ Database │ Doc 1, Doc 3 │
  ├──────────┼──────────────┤
  │ Power    │ Doc 2        │
  └──────────┴──────────────┘
  * "Database" 검색 시 Doc 1, 3을 즉시 반환 ✅
```

+++

### Ⅲ. 역색인의 주요 처리 과정

1. **분석 (Analysis)**: 문장에서 불필요한 조사 등을 제거하고 단어의 원형을 추출합니다.
2. **토큰화 (Tokenizing)**: 문장을 단어(Token) 단위로 쪼갭니다.
3. **색인 생성**: 쪼개진 단어들을 가나다/알파벳 순으로 정렬하고 문서 번호를 매핑하여 저장합니다.

- **📢 섹션 요약 비유**: 역색인은 **'책 뒷면의 인덱스(찾아보기)'**와 같습니다. 책 전체를 처음부터 읽으며(Full Scan) 특정 단어를 찾는 대신, 맨 뒤의 색인 페이지에서 단어만 찾아 "35페이지, 102페이지에 있구나"라고 바로 알아내어 그 페이지만 펼쳐보는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Posting List]**: 특정 단어가 포함된 문서 번호들의 묶음.
- **[Token]**: 역색인에 저장되는 최소 의미 단위.
- **[TF-IDF]**: 검색 결과의 우선순위(연관성)를 계산하는 통계적 수치.

📢 **마무리 요약**: **Inverted Index**는 검색 엔진의 심장입니다. 데이터의 저장을 넘어 '탐색의 효율'을 극대화한 이 구조가 있었기에 오늘날의 구글과 같은 고속 검색 서비스가 가능해졌습니다.
