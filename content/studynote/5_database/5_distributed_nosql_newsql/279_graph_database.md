# 279. 그래프 저장소 (Graph Store) - 관계의 사슬을 잇다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 그래프 저장소는 데이터를 **노드(Node, 객체), 엣지(Edge, 관계), 프로퍼티(Property, 속성)**로 구성된 망(Network) 구조로 저장하여, 데이터 간의 복잡한 연결성을 모델링하는 데이터베이스다.
> 2. **가치**: RDBMS에서 수십 번의 조인(Join)이 필요한 다차원 관계 탐색을 인덱스 없이 '포인터 추적'만으로 순식간에 해결하여 **관계 중심 쿼리 성능을 극대화**한다.
> 3. **융합**: 소셜 네트워크 서비스(SNS), 지능형 추천 엔진, 금융 사기 탐지(FDS), 지식 그래프(Knowledge Graph) 등 연결이 곧 가치인 도메인에서 현대 AI 기술과 융합되어 사용된다.

+++

### Ⅰ. 그래프 저장소의 구성 요소

1. **노드 (Node / Vertex)**: 데이터의 실체. (예: 사람, 상품, 장소)
2. **엣지 (Edge / Relationship)**: 노드 간의 연결. 방향성과 타입(예: '좋아함', '거주함')을 가집니다.
3. **프로퍼티 (Property)**: 노드와 엣지에 붙는 상세 정보. (예: 사람의 나이, 관계가 시작된 날짜)

+++

### Ⅱ. 그래프 모델 시각화 (ASCII Graph)

```text
[그래프 저장소의 논리적 망]

  (Node: User_A) ────[:Follows]───▶ (Node: User_B)
        │                                 │
    [:Bought]                         [:Bought]
        │                                 │
        ▼                                 ▼
  (Node: iPhone 15) ◀───[:Related]─── (Node: MacBook)
```

+++

### Ⅲ. 왜 그래프 저장소인가? (전통 DB와의 차이)

- **인덱스 없는 인접성 (Index-free Adjacency)**: 
    - RDBMS는 조인 시 인덱스를 찾아 헤매야 하지만, 그래프 DB는 노드 자체가 옆 노드의 주소를 알고 있어 물리적 포인터 이동만으로 탐색합니다.
- **유연한 스키마**: 새로운 관계가 생겨도 테이블 구조를 바꿀 필요 없이 엣지 하나만 더 그으면 끝입니다.
- **심층 탐색**: "내 친구의 친구의 친구가 산 물건" 같은 3단계 이상의 깊은 탐색에서 RDBMS 대비 수백 배 이상의 속도를 냅니다.

- **📢 섹션 요약 비유**: 그래프 저장소는 **'인맥 지도'**와 같습니다. 관계형 DB가 '사람들의 이름이 적힌 명부'라면, 그래프 저장소는 그 사람들 사이에 실이 엉켜있는 복잡한 관계망입니다. 명부에서는 두 사람의 관계를 찾으려면 한참을 뒤져야 하지만, 인맥 지도에서는 실(Edge)만 따라가면 누가 누구와 친한지 한눈에 보이는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Cypher / Gremlin]**: 그래프 전용 쿼리 언어.
- **[Property Graph]**: 노드와 관계에 속성을 부여하는 가장 대중적인 모델.
- **[Neo4j]**: 그래프 저장소 업계의 압도적 1위 제품.

📢 **마무리 요약**: **Graph Store**는 연결된 세상의 데이터 언어입니다. 데이터의 '값'만큼이나 '관계'가 중요한 시대에 가장 강력한 통찰을 제공하는 저장소입니다.
