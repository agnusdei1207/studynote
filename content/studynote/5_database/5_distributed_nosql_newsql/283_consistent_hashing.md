# 283. 일관된 해싱 (Consistent Hashing) - 분산 리밸런싱의 최적해

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 일관된 해싱은 해시 테이블의 크기가 변할 때(노드 추가/삭제), 전체 키를 재배치하는 대신 **오직 $K/N$(K:키 개수, N:노드 수)만큼의 데이터만 이동하게 하는 영리한 분산 알고리즘**이다.
> 2. **가치**: 서버 증설 시 시스템 전체의 데이터를 옮겨야 하는 '폭풍 재배치' 문제를 해결하여, **무중단 수평 확장(Scale-out)**을 실무적으로 가능케 한다.
> 3. **융합**: 가상 노드(Virtual Nodes) 개념과 결합되어 노드 간 부하 불균형을 해소하며, Cassandra, DynamoDB, Akamai CDN 등 현대 분산 시스템의 아키텍처적 근간이 된다.

+++

### Ⅰ. 일관된 해싱의 필요성 (Traditional Hashing의 한계)

- **문제**: 전통적인 `Hash(key) % N` 방식은 서버 대수 $N$이 바뀌면 거의 모든 데이터의 저장 위치가 바뀌어 버립니다. 이는 분산 시스템에서 엄청난 네트워크 부하와 캐시 미스(Cache Miss)를 유발합니다.
- **해결**: 서버의 개수가 아닌, 고정된 크기의 **해시 링(Hash Ring)**을 사용하여 매핑 구조를 안정화합니다.

+++

### Ⅱ. 해시 링 구조와 작동 원리 (ASCII Diagram)

```text
[Consistent Hashing Ring]

          (Hash Value: 0)
               ▲
        ┌──────┴──────┐
     [Node 3]      [Node 1]
        │             │
        │      (K1)──▶│ (Key 1은 시계방향으로 
        │             │  가장 가까운 Node 1에 저장)
     [Node 2]      [New Node!] ◀── 신규 서버 추가 시
        └──────┬──────┘           (K1은 여전히 Node 1, 
               ▼                  일부 키만 New Node로 이동)
```

1. **링 형성**: 해시 함수의 출력 범위(예: 0~2^32-1)를 원형 링으로 만듭니다.
2. **배치**: 서버(Node)와 데이터(Key)를 모두 이 링 위에 해시값으로 배치합니다.
3. **매핑**: 특정 키는 링 위에서 **시계 방향**으로 가장 먼저 만나는 서버에 저장됩니다.

+++

### Ⅲ. 핵심 기술: 가상 노드 (Virtual Nodes)

- **문제**: 서버가 링 위에 불균형하게 배치되면 특정 서버만 너무 많은 데이터를 가질 수 있습니다.
- **해결**: 하나의 물리 서버를 링 위의 수백 개 가상 지점(Virtual Nodes)에 분산 배치하여, 데이터가 전체 링에 수학적으로 완벽하게 골고루 퍼지도록 유도합니다.

- **📢 섹션 요약 비유**: 일관된 해싱은 **'의자 뺏기 게임의 역발상'**과 같습니다. 사람이 늘어나거나 줄어들 때 모든 사람이 자리를 옮겨야 하는 게 일반 해싱이라면, 일관된 해싱은 내 옆에 의자가 하나 더 생기면 나만 살짝 옆으로 옮겨 앉고 다른 사람들은 그대로 앉아있게 하는 효율적인 자리 배치법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Rebalancing]**: 서버 추가/삭제 시 데이터를 다시 나누는 과정.
- **[Hash Ring]**: 데이터와 서버가 공존하는 가상의 논리 원.
- **[Scalability]**: 일관된 해싱을 통해 얻게 되는 무한한 확장성.

📢 **마무리 요약**: **Consistent Hashing**은 분산 시스템의 '평화 유지군'입니다. 변화에 민감하게 반응하지 않으면서도 시스템을 안정적으로 키워나갈 수 있게 돕는 핵심 지혜입니다.
