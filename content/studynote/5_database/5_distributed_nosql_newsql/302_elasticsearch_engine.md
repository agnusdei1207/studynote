# 302. Elasticsearch - 검색의 패러다임을 바꾸다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Elasticsearch는 Apache Lucene을 기반으로 구축된 **분산형 RESTful 검색 및 분석 엔진**으로, 정형 및 비정형 데이터를 가리지 않고 실시간에 가까운(Near Real-time) 검색 성능을 제공한다.
> 2. **가치**: **역색인(Inverted Index)** 구조를 사용하여 방대한 양의 텍스트 데이터 속에서 원하는 키워드를 1초 미만의 속도로 찾아내며, 수평적 확장성이 뛰어나 수천 대의 서버 클러스터로 운영 가능하다.
> 3. **융합**: 로그 수집(Logstash), 시각화(Kibana)와 결합된 **ELK 스택**의 핵심으로 활용되며, 현대의 검색 서비스, 애플리케이션 모니터링, 엔터프라이즈 분석의 표준 저장소로 융합된다.

+++

### Ⅰ. Elasticsearch의 주요 특징

- **Schemaless & Document Oriented**: 데이터를 JSON 문서 형태로 저장하며, 스키마를 미리 정의하지 않아도 자동으로 필드 타입을 추론(Dynamic Mapping)합니다.
- **분산 아키텍처**: 데이터를 '샤드(Shard)'로 나누어 여러 노드에 분산하고, '레플리카(Replica)'를 통해 데이터 안전성과 조회 성능을 동시에 확보합니다.
- **RESTful API**: 표준 HTTP 메소드(GET, POST, PUT, DELETE)를 통해 모든 기능을 제어하므로 언어에 구애받지 않는 개발이 가능합니다.

+++

### Ⅱ. Elasticsearch 내부 구조 시각화 (ASCII Model)

```text
[ Elasticsearch Cluster & Sharding ]

  [ Node 1 ]           [ Node 2 ]           [ Node 3 ]
  ┌──────────┐         ┌──────────┐         ┌──────────┐
  │ Shard 0  │ (P)     │ Shard 1  │ (P)     │ Shard 2  │ (P)
  ├──────────┤         ├──────────┤         ├──────────┤
  │ Shard 2  │ (R)     │ Shard 0  │ (R)     │ Shard 1  │ (R)
  └──────────┘         └──────────┘         └──────────┘
  
  * (P): Primary Shard - 쓰기 담당
  * (R): Replica Shard - 읽기 분산 및 백업 담당
```

+++

### Ⅲ. 왜 단순 DB 검색보다 빠른가?

- **RDBMS의 한계**: `LIKE %keyword%` 검색은 모든 행을 다 뒤져야 하는 'Full Table Scan'을 유발합니다.
- **ES의 혁신**: 데이터를 저장할 때 단어별로 어느 문서에 있는지 미리 지도를 그려두는 **역색인** 기술을 사용합니다. 책의 맨 뒤에 있는 '찾아보기' 페이지를 보는 것과 같은 원리입니다.

- **📢 섹션 요약 비유**: Elasticsearch는 **'도서관의 전문 사서'**와 같습니다. 책(데이터)이 수백만 권이어도 사서는 각 단어가 어느 책 몇 페이지에 있는지 완벽한 색인(Index)을 가지고 있습니다. 우리가 질문을 던지면 서가를 헤매지 않고 즉시 해당 페이지를 펼쳐 보여주는 유능한 해결사입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Lucene]**: ES의 심장이 되는 오픈소스 검색 라이브러리.
- **[Inverted Index]**: 검색 속도의 비밀 병기.
- **[Full-text Search]**: ES가 가장 잘하는 전문(全文) 검색 기술.

📢 **마무리 요약**: **Elasticsearch**는 단순한 저장소를 넘어 데이터의 '가치'를 찾아내는 검색 엔진입니다. 복잡한 텍스트 분석이 필요한 현대의 모든 인프라에서 대체 불가능한 존재입니다.
