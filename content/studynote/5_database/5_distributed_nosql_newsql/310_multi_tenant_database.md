# 310. 멀티테넌트(Multi-tenant) DB 구조 - SaaS 인프라의 토대

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멀티테넌시(Multi-tenancy) 데이터베이스는 하나의 소프트웨어 인스턴스에서 **수많은 고객(Tenant)의 데이터를 격리하여 동시에 관리**하는 아키텍처 기술이다.
> 2. **가치**: 인프라 자원을 공유함으로써 운영 비용을 획기적으로 낮추고 통합 관리를 가능케 하며, 고객 간의 데이터 간섭을 방지하는 강력한 격리(Isolation) 매커니즘을 제공한다.
> 3. **융합**: 데이터 격리 수준에 따라 독립 DB, 독립 스키마, 공유 스키마 방식으로 나뉘며, 현대 클라우드 SaaS 아키텍처의 비즈니스 확장성을 결정짓는 핵심 설계 요소다.

+++

### Ⅰ. 멀티테넌트 DB의 3대 격리 모델

1. **독립 DB (Database per Tenant)**:
    - 각 고객마다 물리적으로 다른 DB 서버나 인스턴스를 제공.
    - **장점**: 완벽한 격리, 보안 최상. **단점**: 높은 비용, 관리 복잡.
2. **독립 스키마 (Schema per Tenant)**:
    - 동일 DB 내에서 스키마(또는 Namespace)만 나누어 사용.
    - **장점**: DB 자원 공유로 비용 절감. **단점**: 고객 수 증가 시 스키마 관리 부하.
3. **공유 스키마 (Shared Schema)**:
    - 하나의 테이블에 모든 고객 데이터를 넣고 `tenant_id` 컬럼으로 구분.
    - **장점**: 자원 효율 극대화, 무한 확장 유리. **단점**: 쿼리 실수 시 데이터 노출 위험 (Row-level security 필수).

+++

### Ⅱ. 멀티테넌트 구조 시각화 (ASCII Model)

```text
[ Shared Schema Approach ]

  Table: [ All_Customer_Data ]
  ┌───────────┬───────────┬─────────────────────┐
  │ Tenant_ID │ Record_ID │ Data Content        │
  ├───────────┼───────────┼─────────────────────┤
  │  "CO_A"   │    101    │ "Sales Report..."   │
  │  "CO_B"   │    205    │ "Inventory Log..."  │
  │  "CO_A"   │    102    │ "User List..."      │
  └───────────┴───────────┴─────────────────────┘
       │
       ▼
  (Query Logic): SELECT * FROM Data WHERE Tenant_ID = 'CO_A';
  ✅ "A사 직원은 절대로 B사 데이터를 볼 수 없음"
```

+++

### Ⅲ. 실무적 고려 사항

- **성능 스큐 (Noise Neighbor)**: 특정 대형 고객의 트래픽이 많아지면 같은 서버를 쓰는 다른 소형 고객들의 성능이 느려지는 현상입니다. 이를 위해 자원 할당량(Quota) 제한 기술이 필요합니다.
- **백업 및 복구**: 공유 스키마 모델에서는 특정 고객의 데이터만 1시간 전으로 되돌리는(Point-in-time Recovery) 작업이 매우 어렵습니다.
- **스키마 진화**: 수만 명의 고객이 공유하는 테이블의 컬럼을 추가할 때 발생하는 서비스 중단을 최소화해야 합니다.

- **📢 섹션 요약 비유**: 멀티테넌시 구조는 **'아파트 거주'**와 같습니다. 독립 DB는 '전원주택(비싸지만 자유로움)', 독립 스키마는 '빌라(벽만 공유)', 공유 스키마는 '쉐어하우스(한 거실을 같이 쓰되 자기 방만 점유)'인 셈입니다. 쉐어하우스가 가장 싸고 관리하기 편하지만, 공용 공간(자원)을 매너 있게 나누어 쓰는 규칙이 중요합니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Isolation Level]**: 멀티테넌트 설계의 제1 원칙.
- **[Row-Level Security]**: 공유 스키마의 보안을 책임지는 DB 엔진 기능.
- **[SaaS (Software as a Service)]**: 멀티테넌트 DB가 가장 활발히 쓰이는 산업군.

📢 **마무리 요약**: **Multi-tenant DB**는 클라우드 비즈니스의 수익성을 결정합니다. 서비스의 성장 단계와 보안 요구 수준에 맞는 최적의 격리 모델을 선택하는 것이 성공의 열쇠입니다.
