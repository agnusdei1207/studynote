# 301. 데이터 마이그레이션 도구 및 CDC (Change Data Capture)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CDC(Change Data Capture)는 데이터베이스에서 발생하는 **모든 삽입, 수정, 삭제 등의 변경 사항을 실시간으로 감지하여 타겟 시스템으로 전송하는 기술**이다.
> 2. **가치**: 대량의 데이터를 한꺼번에 옮기는 '벌크 마이그레이션'의 다운타임 문제를 해결하며, 운영 DB에 부하를 거의 주지 않고 실시간 데이터 동기화(Real-time Sync)를 가능케 한다.
> 3. **융합**: 메시지 브로커(Kafka) 및 ETL 도구와 융합되어 데이터 레이크, 검색 엔진, 마이크로서비스 간의 데이터 파이프라인을 구축하는 현대적 데이터 엔지니어링의 핵심이다.

+++

### Ⅰ. CDC의 작동 매커니즘: 로그 기반 방식

- **정의**: DB 엔진의 **트랜잭션 로그(Redo Log, Binary Log)**를 직접 읽어 변경 사항을 추출하는 방식입니다.
- **특징**:
    - **Low Impact**: 실제 테이블을 조회하지 않으므로 운영 DB 성능에 영향이 거의 없습니다.
    - **Real-time**: 데이터가 변경되는 즉시 캡처되어 전송됩니다.
    - **Reliability**: 커밋된 트랜잭션만 추적하므로 일관성이 보장됩니다.

+++

### Ⅱ. CDC 데이터 파이프라인 시각화 (ASCII Flow)

```text
[ CDC기반 실시간 데이터 동기화 ]

  [ Source DB ] ──▶ [ Transaction Log ] ──▶ [ CDC Agent ]
       │                                       │ (Capture)
       │                                       ▼
  (User Update)                       [ Message Bus (Kafka) ]
                                               │ (Produce)
          ┌────────────────────────────────────┼────────────────────────────────┐
          ▼                                    ▼                                ▼
  [ Target DB (Replica) ]              [ Search Engine (ES) ]           [ Data Lake (S3) ]
```

+++

### Ⅲ. 대표적인 도구 및 활용

1. **Debezium**: 오픈소스 CDC의 표준. 카프카 커넥트 기반으로 다양한 DB 지원.
2. **AWS DMS (Database Migration Service)**: 클라우드 이전을 위한 관리형 마이그레이션 도구.
3. **Oracle GoldenGate**: 엔터프라이즈급 고성능 실시간 데이터 통합 솔루션.

- **📢 섹션 요약 비유**: CDC는 **'CCTV 실시간 중계'**와 같습니다. 예전 방식이 하루 일과가 끝나고 장부를 통째로 복사해 가는 것(Batch)이었다면, CDC는 현장에서 일어나는 모든 움직임을 실시간으로 카메라로 찍어(Capture) 전 세계 지점으로 생중계(Sync)하는 고도의 감시 및 배달 시스템입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Zero-Downtime]**: CDC를 통해 달성하고자 하는 마이그레이션 목표.
- **[Schema Evolution]**: 소스 DB의 스키마 변경 시 CDC가 어떻게 대응할 것인가의 문제.
- **[Data Pipeline]**: CDC가 시작점이 되는 거대한 데이터 흐름의 망.

📢 **마무리 요약**: **CDC**는 정적인 데이터를 동적인 흐름으로 바꿉니다. 파편화된 시스템들이 하나의 살아있는 유기체처럼 동일한 데이터를 공유하게 만드는 실시간 혈관과 같습니다.
