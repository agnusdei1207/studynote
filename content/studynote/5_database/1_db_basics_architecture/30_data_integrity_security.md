# 30. 데이터 무결성 (Integrity) 및 보안성 (Security)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 **무결성**은 저장된 데이터의 정확성, 일관성, 유효성을 보장하는 성질이며, **보안성**은 권한이 없는 자로부터 데이터를 보호하고 기밀성을 유지하는 성질이다.
> 2. **상호 보완**: 무결성이 깨지면 데이터는 가치가 없어지며(Garbage in, Garbage out), 보안성이 깨지면 데이터는 기업의 리스크가 된다. 즉, 무결성은 '데이터의 품질'을, 보안성은 '데이터의 안전'을 책임진다.
> 3. **융합**: DBMS는 제약 조건(Constraints)을 통해 무결성을 자동화하고, 접근 제어(DCL) 및 암호화를 통해 보안성을 결착하여 전사적 데이터 거버넌스의 핵심 교두보 역할을 수행한다.

---

## Ⅰ. 개요 (Context & Background)

### 1. 개념 정의
- **데이터 무결성 (Data Integrity)**: 데이터베이스에 저장된 값이 실제 세계의 사실과 일치하며, 논리적으로 오류가 없는 상태를 유지하는 것을 말합니다.
- **데이터 보안성 (Data Security)**: 불법적인 노출, 변조, 파괴로부터 데이터를 보호하기 위한 기술적·관리적 조치를 의미합니다.

### 2. 💡 비유: 은행 금고와 장부
```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        무결성 vs 보안성 비유                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 무결성 (장부의 정확성):                                                   │
│     - 입금액이 100원인데 장부에 1,000원이라 적히면 안 됨. (정확성)               │
│     - 이름 칸에 숫자가 적히면 안 됨. (유효성)                                   │
│                                                                             │
│  2. 보안성 (금고와 자물쇠):                                                   │
│     - 은행 직원만 장부를 볼 수 있어야 함. (기밀성)                               │
│     - 도둑이 장부를 훔쳐가거나 불태우면 안 됨. (가용성/보호)                      │
│                                                                             │
│  → 장부가 정확해도 도둑맞으면 끝이고, 금고가 튼튼해도 장부가 엉터리면 소용없음!      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 등장 배경
- **무결성 필요성**: 파일 시스템에서는 데이터가 여러 곳에 흩어져 있어 값이 서로 달라지는 '불일치'가 빈번하여 데이터 신뢰성 하락.
- **보안성 필요성**: 해킹 기술의 고도화와 개인정보 보호법(GDPR, 데이터 3법 등) 강화로 인해 기업의 법적 책임 강화.

- **📢 섹션 요약 비유**: 무결성은 데이터의 '내실'을 다지는 일이고, 보안성은 데이터의 '외벽'을 쌓는 일입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 데이터 무결성의 종류 및 제약 조건 (Integrity Constraints)

| 종류 | 상세 설명 | DBMS 구현 예시 |
|:---:|:---|:---|
| **개체 무결성 (Entity)** | 기본키(PK)는 NULL일 수 없고 중복될 수 없음. | `PRIMARY KEY` |
| **참조 무결성 (Referential)** | 외래키(FK)는 참조하는 테이블의 PK값이거나 NULL이어야 함. | `FOREIGN KEY (REFERENCES)` |
| **도메인 무결성 (Domain)** | 속성 값은 정의된 데이터 타입과 범위 내에 있어야 함. | `CHECK`, `DEFAULT`, `NOT NULL` |
| **사용자 정의 무결성** | 비즈니스 로직에 따른 특정 규칙 준수. | `TRIGGER`, `ASSERTION` |

### 2. 데이터 보안 아키텍처 (ASCII)

사용자 접근부터 물리적 저장까지 다층 방어(Defense in Depth) 구조를 시각화한다.

```text
    [ 외부 위협 / 사용자 ]
           │
    ┌──────▼──────────────────────────────────────────────────────────┐
    │ 1. [ 식별 및 인증 ] : ID/PW, MFA, 생체 인식 (보안성 시작)          │
    └──────┬──────────────────────────────────────────────────────────┘
           │
    ┌──────▼──────────────────────────────────────────────────────────┐
    │ 2. [ 접근 제어 (DCL) ] : RBAC(역할 기반), ABAC(속성 기반) 권한 관리  │
    │    - GRANT / REVOKE 명령어를 통한 세밀한 통제                     │
    └──────┬──────────────────────────────────────────────────────────┘
           │
    ┌──────▼──────────────────────────────────────────────────────────┐
    │ 3. [ 무결성 엔진 ] : 제약 조건(PK, FK) 실시간 검증 (무결성 보장)     │
    └──────┬──────────────────────────────────────────────────────────┘
           │
    ┌──────▼──────────────────────────────────────────────────────────┐
    │ 4. [ 데이터 암호화 ] : TDE(저장 시), TLS(전송 시) 암호화           │
    └──────┬──────────────────────────────────────────────────────────┘
           │
    [  물리적 디스크 (Encrypted) ]
```

### 3. 심층 동작 원리: 참조 무결성과 Cascade 옵션
데이터 변경 시 무결성을 유지하기 위한 DBMS의 연쇄 동작 방식입니다.
1. **제약 감지**: 부모 테이블의 행을 삭제하려 할 때, 이를 참조하는 자식 행이 있는지 확인.
2. **RESTRICT**: 자식이 있으면 부모 삭제를 거부 (기본 설정).
3. **CASCADE**: 부모를 지우면 자식도 함께 지움 (위험하지만 편리함).
4. **SET NULL**: 부모를 지우면 자식의 FK 값을 NULL로 변경.

### 4. 핵심 보안 기술: 접근 제어 행렬 (Access Control Matrix)
사용자(Subject)와 객체(Object) 간의 권한 관계를 정의하는 핵심 논리입니다.
- **Grantee**: 권한을 받는 자.
- **With Grant Option**: 받은 권한을 남에게 다시 줄 수 있는 위험한 권한.

- **📢 섹션 요약 비유**: 무결성 제약 조건은 '입구에서 신분증과 자격을 검사하는 안내원'과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 무결성(Integrity) vs 보안성(Security) 상세 비교

| 항목 | 데이터 무결성 (Integrity) | 데이터 보안성 (Security) |
|:---:|:---|:---|
| **핵심 목표** | **데이터의 정확성과 품질** | **데이터의 기밀성과 보호** |
| **주요 위협** | 입력 실수, 시스템 오류, 버그 | 해킹, 내부 유출, 악성 코드 |
| **관리 주체** | 설계자, 개발자 | 보안 담당자, DBA |
| **성패 기준** | "데이터가 믿을 수 있는가?" | "데이터가 안전하게 숨겨졌는가?" |

### 2. 과목 융합 관점: 보안 및 네트워크
- **보안(Security) 융합**: CIA 3대 요소(Confidentiality, Integrity, Availability) 중 **Integrity**가 바로 DB 무결성에 해당하며, DB 보안은 이를 실현하는 기술적 수단이다.
- **네트워크(NW) 융합**: 데이터 전송 구간에서 해커가 값을 가로채 조작(Man-in-the-Middle)하는 행위는 무결성과 보안성을 동시에 파괴하는 공격이다.

- **📢 섹션 요약 비유**: 무결성이 '음식이 상하지 않게 관리하는 것'이라면, 보안성은 '음식을 아무나 못 먹게 감시하는 것'입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 1. 실무 시나리오: 애플리케이션 vs DB 제약 조건
- **상황**: 성능 향상을 위해 DB의 Foreign Key를 모두 제거하고 소스 코드에서 체크하자고 함.
- **판단**: FK 제거 시 DB 직접 수정(DML)이나 배치 작업 도중 무결성이 깨질 확률 100%임.
- **결단**: **성능 저하를 감수하더라도 핵심 정합성(PK, FK)은 반드시 DB 레벨에서 강제**해야 함. 인덱스 최적화를 통해 성능 문제를 해결할 것.

### 2. 안티패턴: 무분별한 CASCADE 옵션 사용
- **증상**: 편의성을 위해 모든 FK에 `ON DELETE CASCADE` 설정.
- **결과**: 사원 테이블에서 부서 하나를 지웠는데, 수년간 쌓인 수억 건의 업무 실적 데이터가 순식간에 영구 삭제됨.
- **조치**: 중요한 비즈니스 데이터에는 `RESTRICT`를 사용하여 명시적인 확인 후 삭제하도록 설계.

### 📢 기술사적 결언
> "데이터베이스의 가치는 용량이 아니라 **'신뢰'**에서 나온다. 무결성이 무너진 DB는 단순한 '쓰레기 더미'에 불과하며, 보안이 뚫린 DB는 '시한폭탄'과 같다. 기술사는 무결성 제약 조건을 통해 데이터의 자정 능력을 키우고, 다층 보안 설계를 통해 외부 위협으로부터 방벽을 세우는 '데이터의 수호자'가 되어야 한다."

- **📢 섹션 요약 비유**: FK 없는 DB는 '안전벨트 없는 스포츠카'와 같습니다. 평소엔 빠르지만, 사고가 나면 생명을 지킬 수 없습니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 1. 정량/정성 기대효과 (ROI)

| 지표 | 도입 전 (파일 기반) | 도입 후 (DBMS 무결성/보안) | 개선 효과 |
|:---|:---:|:---:|:---:|
| **데이터 정제 비용** | 매우 높음 | **매우 낮음 (자동 검증)** | 운영 비용 60% 이상 절감 |
| **개인정보 유출 리스크** | 상존 | **통제 가능** | 법적 리스크 및 과징금 방어 |
| **비즈니스 신뢰도** | 낮음 (데이터 오류 빈번) | **높음** | 고객 만족도 및 신뢰 상승 |

### 2. 미래 전망
- **블록체인 기반 무결성**: 위변조가 불가능한 블록체인 기술을 DBMS와 융합하여 '절대적 무결성'을 제공하는 시스템 등장 (예: Amazon QLDB).
- **Zero Trust 보안**: DB 접근 시 "한 번의 인증"이 아닌 "모든 쿼리마다 재인증"하는 제로 트러스트 아키텍처가 DB 보안의 표준이 될 것임.

- **📢 섹션 요약 비유**: 미래의 데이터 관리는 스스로 상처를 치유하는(무결성) 스마트한 요새(보안성)로 진화할 것입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **무결성**은 일기장에 글씨를 **틀리지 않고 똑바르게** 적는 약속이에요.
2. **보안성**은 내 일기장을 **동생이 몰래 보지 못하게** 서랍에 넣고 열쇠로 잠그는 거랍니다.
3. 이 두 가지를 잘 지켜야 내 소중한 기록들이 **망가지지 않고 안전하게** 남을 수 있어요!
