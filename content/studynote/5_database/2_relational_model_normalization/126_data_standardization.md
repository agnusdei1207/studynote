# 126. 데이터 표준화 - 표준 용어, 단어, 도메인 정의

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 간 데이터 교환 및 통합 시 발생하는 혼란을 방지하기 위해, 데이터의 명칭(용어), 구성 요소(단어), 값의 범위(도메인)에 대한 **통일된 기준과 명명 규칙**을 수립하고 준수하는 활동이다.
> 2. **3대 관리 요소**: 최소 의미 단위인 **표준 단어**, 단어의 조합으로 형성된 **표준 용어**, 그리고 속성의 형식을 규정하는 **표준 도메인**을 체계적으로 관리하여 데이터 아키텍처의 일관성을 확보한다.
> 3. **가치**: "이름은 달라도 같은 의미"인 데이터와 "이름은 같아도 다른 의미"인 데이터를 원천 차단하여 **데이터 품질**을 높이고, 개발 생산성과 시스템 유지보수 효율을 획기적으로 향상시킨다.

+++

## Ⅰ. 개요 (Context & Background)

### 1. 개념 정의
**데이터 표준화 (Data Standardization)**는 기업의 모든 정보 시스템에서 공통으로 사용하는 데이터 요소에 대해 동일한 명칭, 동일한 형식, 동일한 의미를 부여하는 과정입니다. 데이터 거버넌스의 가장 구체적인 실행 단계입니다.

### 2. 💡 비유: 전 세계 공통의 '미터법(Metric System)'
```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        데이터 표준화 개념 비유                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 여러 나라(시스템)가 모여서 거대한 다리(통합 시스템)를 지으려 함.          │
│                                                                             │
│  1. 표준화 미흡: 어느 나라는 '인치', 어느 나라는 '척'을 사용함.                 │
│     - 부품을 가져왔는데 크기가 안 맞아 다리가 무너짐. (데이터 통합 실패)          │
│                                                                             │
│  2. 표준화 완료: 모든 나라가 '미터(m)'와 '킬로그램(kg)'이라는 표준을 약속함.      │
│     - 누가 부품을 만들었든 척척 맞물림! (상호 운용성 확보)                       │
│                                                                             │
│  → 데이터 표준화는 데이터들끼리 **'말이 통하게'** 규격을 맞추는 약속입니다!        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 등장 배경
- **의사소통 비용 증가**: '고객명', '성명', '이름' 등 동일 데이터에 대한 명칭 파편화로 인해 현업과 IT 간의 오해 빈번.
- **데이터 통합의 난제**: 시스템 통합(M&A 등) 시 데이터 형식이 달라 매핑 작업에만 막대한 리소스 투입.

- **📢 섹션 요약 비유**: 데이터 표준화는 데이터 세상의 '바른 우리말 대사전'을 편찬하는 일과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 데이터 표준화의 3대 핵심 구성 요소

| 요소 | 상세 설명 | 예시 |
|:---:|:---|:---|
| **표준 단어 (Word)** | 더 이상 분해되지 않는 최소 의미 단위. | 고객, 번호, 일자, 성명, 금액 |
| **표준 용어 (Term)** | 표준 단어를 조합하여 만든 공식 명칭. | 고객번호, 주문일자, 결제금액 |
| **표준 도메인 (Domain)** | 속성이 가질 수 있는 값의 형식과 범위. | 날짜형(YYYYMMDD), 금액형(NUMBER) |

### 2. 표준화 관리 체계 도해 (ASCII)

단어와 도메인이 결합하여 표준 용어와 물리 컬럼이 되는 과정을 시각화한다.

```text
    [ 표준 단어 사전 ]       [ 표준 도메인 사전 ]
    - 고객 (CUST)            - ID형 (VARCHAR 10)
    - 번호 (NO)              - 일자형 (DATE)
           │                        │
           └───────────┬────────────┘
                       ▼
                [ 표준 용어 사전 ]
                - 고객번호 (CUST_NO)
                - 등록일자 (REG_DT)
                       │
           ┌───────────┴───────────┐
           ▼                       ▼
    [ Logical Model ]       [ Physical DB ]
    - 속성명: 고객번호       - 컬럼명: CUST_NO
    - 타입: VARCHAR(10)     - 제약: NOT NULL
```

### 3. 심층 동작 원리: 명명 규칙 (Naming Convention)
- **한글명 규칙**: '명사 + 명사' 형태로 구성하며 조사는 생략. (예: '고객의 번호' $\to$ '고객번호')
- **영문약어 규칙**: 표준 단어마다 대응되는 유일한 영문 약어를 정의. (예: 번호 $\to$ NO, 일자 $\to$ DT)
- **일관성**: 물리 컬럼명은 반드시 영문 약어의 조합으로 생성하여 가독성과 시스템 호환성 확보.

### 4. 핵심 기술: 데이터 표준 준수 검사 (Standard Check)
- 모델링 도구(ERwin, DA# 등)와 연동하여, 표준 사전에 등록되지 않은 단어나 용어를 사용하면 경고를 띄우거나 생성을 막는 자동화 기술.

- **📢 섹션 요약 비유**: 명명 규칙은 '가족 이름을 돌림자로 짓는 것'과 같아, 이름만 봐도 어느 항렬(주제 영역)인지 알 수 있게 합니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 표준 데이터 vs 비표준 데이터의 처리 비용

| 비교 항목 | 표준 데이터 기반 개발 | 비표준 데이터 기반 개발 |
|:---:|:---|:---|
| **설계 단계** | 사전 협의로 시간 소요 | 즉흥적 설계로 빠름 |
| **개발 단계** | **코드 재사용 용이** | 매번 컬럼 의미 파악 필요 |
| **데이터 통합** | **즉시 통합 가능 (Mapping Zero)** | 수작업 매핑 및 변환 로직 필수 |
| **유지보수** | 영향도 분석 명확 | **분석 불가능 (블랙박스화)** |

### 2. 과목 융합 관점: 소프트웨어 공학 및 AI
- **소프트웨어 공학(SE) 융합**: 데이터 표준화는 **유비쿼터스 언어(Ubiquitous Language)**의 실체이며, 도메인 주도 설계(DDD)에서 비즈니스 전문가와 개발자가 공유하는 핵심 개념 모델이 된다.
- **AI 융합**: 데이터 표준화는 **데이터 레이블링(Labeling)**의 품질을 결정하며, 잘 정제된 표준 데이터는 AI 모델 학습 시 노이즈를 줄여 정확도를 획기적으로 높인다.

- **📢 섹션 요약 비유**: 표준화는 '레고 블록의 규격'과 같습니다. 규격이 맞아야 다른 세트의 블록들도 무한히 합칠 수 있습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 1. 실무 시나리오: 차세대 차세대 프로젝트의 표준화 빅뱅
- **상황**: 20년간 제각각 운영된 5개 시스템을 하나로 통합함.
- **판단**: 기존 명칭을 유지하면 통합 뷰 생성이 불가능함.
- **결단**: **전사 데이터 표준 사전 신규 수립**. 5개 시스템의 모든 컬럼을 전수 조사하여 대표 표준명을 정하고, 구 시스템과 신 시스템 간의 '표준 매핑 정의서'를 구축하여 데이터 전환의 무결성 확보.

### 2. 안티패턴: 문서로만 존재하는 죽은 표준
- **증상**: 표준 가이드북은 두껍게 만들었으나, 개발자들은 여전히 `col1`, `temp_table` 등의 임의 명칭 사용.
- **결과**: 표준화 비용만 쓰고 효과는 0이며, 시스템은 더욱 혼탁해짐.
- **조치**: **표준 관리 시스템(MMS) 도입**. DB 생성 권한을 표준 검사 통과 시에만 부여하는 프로세스 강제화(Hard Enforcement)와 자동 명명 도구 제공.

### 📢 기술사적 결언
> "데이터 표준화는 **'데이터 거버넌스의 기초 체력'**이다. 체력이 없는 선수가 기술을 부릴 수 없듯, 표준화 없는 AI나 빅데이터는 허상에 불과하다. 기술사는 단순히 이름을 통일하는 수준을 넘어, **데이터의 비즈니스적 본질**을 규정하는 표준 도메인 체계를 수립해야 한다. 견고한 표준화는 향후 **데이터 메시(Data Mesh)** 아키텍처에서 각 도메인이 자율적으로 성장하면서도 전사적 일관성을 유지하게 하는 유일한 **논리적 구속력**이 된다."

- **📢 섹션 요약 비유**: 표준화를 무시하는 개발은 '각자 다른 주파수로 무전기를 쓰는 군대'와 같아, 전투(운영)가 시작되는 순간 자멸하게 됩니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 1. 정량/정성 기대효과 (표준화 정착 시)

| 지표 | 도입 전 | 도입 후 | 개선 효과 |
|:---|:---:|:---:|:---:|
| **데이터 탐색 및 해석 시간** | 수 시간 | **수 분** | 업무 효율성 80% 향상 |
| **시스템 통합 비용** | 100% | **30% 이하** | IT 프로젝트 ROI 극대화 |
| **데이터 품질 신뢰도** | 낮음 (의구심 상존) | **최상 (표준 준수)** | 데이터 기반 의사결정 문화 정착 |

### 2. 미래 전망
- **AI-Driven Standardization**: AI가 기존의 수만 개 컬럼을 스스로 분석하여 최적의 표준 후보를 제안하고 매핑하는 **지능형 표준화 자동화**.
- **Global Ontology Standards**: 개별 기업을 넘어 산업군 전체가 동일한 표준 언어를 사용하는 **산업 데이터 공유 체계(Data Space)** 확산.

- **📢 섹션 요약 비유**: 미래의 표준화는 보이지 않는 '공기'처럼 존재하여, 우리가 어떤 데이터를 쓰더라도 자동으로 정화되고 일관된 의미를 전달하는 지능형 인프라가 될 것입니다.
