# 429. 인덱스 레인지 스캔(Index Range Scan) - 효율적 탐색의 정석

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인덱스 레인지 스캔은 B+Tree 인덱스의 루트부터 시작하여 특정 시작 지점의 리프 노드를 찾은 뒤, **리프 노드의 연결 리스트를 따라 필요한 범위까지 순차적으로 데이터를 읽어 들이는 방식**이다.
> 2. **가치**: 전체 테이블을 뒤질 필요 없이 필요한 범위의 데이터만 골라 읽으므로 **I/O 비용을 최소화**하며, 데이터가 이미 인덱스 순으로 정렬되어 있어 추가적인 정렬(Sort) 작업 없이도 결과를 반환할 수 있다.
> 3. **융합**: 선택도(Selectivity)가 높은 쿼리에 가장 적합하며, 인덱스에 포함되지 않은 컬럼이 필요한 경우 실제 데이터 페이지로 이동하는 **랜덤 I/O(Table Access by Index ROWID)**와 결합되어 성능이 결정된다.

+++

### Ⅰ. 인덱스 레인지 스캔의 작동 단계

1. **Vertical Scan (수직 탐색)**: 루트 노드에서 브랜치 노드를 거쳐, 검색 조건의 시작 값(예: `ID >= 100`)이 있는 리프 노드까지 내려갑니다.
2. **Horizontal Scan (수평 탐색)**: 찾은 리프 노드에서 형제 노드 포인터를 따라 오른쪽(또는 왼쪽)으로 이동하며 종료 조건(예: `ID <= 200`)까지 데이터를 읽습니다.
3. **Table Access (선택 사항)**: 인덱스에 없는 컬럼을 가져와야 할 경우, 리프 노드에 저장된 ROWID를 이용해 실제 테이블 블록을 한 번 더 읽습니다.

+++

### Ⅱ. 레인지 스캔 프로세스 시각화 (ASCII Model)

```text
[ Index Range Scan: WHERE ID BETWEEN 10 AND 25 ]

          [ Root Node ]
         /      |      \
  [Branch 1] [Branch 2] [Branch 3]
     /   \      /   \      /   \
  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐
  │ 1 │ │ 10│─▶ 15│─▶ 20│─▶ 25│─▶ 30│
  └───┘ └───┘ └───┘ └───┘ └───┘ └───┘
          ▲     │     │     │
          └─────┴───┬─┴─────┘
                    ▼
          [ Read Range Data ] ✅ (10, 15, 20, 25)
```

+++

### Ⅲ. 성능 최적화 포인트

- **인덱스 스캔 효율**: 리프 노드에서 읽어야 할 범위가 너무 넓으면(예: 전체의 20% 이상), 차라리 테이블 풀 스캔(FTS)을 하는 것이 나을 수 있습니다 (Single Block Read의 한계).
- **인덱스 구성 컬럼**: `SELECT` 하는 모든 컬럼이 인덱스에 포함되어 있다면(Covering Index), 테이블 접근 단계를 생략하여 성능이 비약적으로 향상됩니다.
- **클러스터링 팩터**: 인덱스 순서와 실제 테이블 데이터의 물리적 저장 순서가 얼마나 일치하느냐에 따라 테이블 접근 시 발생하는 랜덤 I/O 부하가 결정됩니다.

- **📢 섹션 요약 비유**: 인덱스 레인지 스캔은 **'사전에서 특정 단어 뭉치 찾기'**와 같습니다. "가을"부터 "겨울"까지의 단어를 찾는다면, 먼저 "가을"이 있는 페이지를 한 번에 펼친 뒤(수직 탐색), 거기서부터 한 장씩 넘기며(수평 탐색) "겨울"이 나올 때까지 단어를 읽어 나가는 것과 같습니다. 처음부터 끝까지 다 읽을 필요가 없어 아주 효율적인 독서법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Covering Index]**: 테이블 조회를 생략하게 해주는 최강의 인덱스 기법.
- **[Sequential Read]**: 리프 노드 간 이동 시 발생하는 효율적 읽기 방식.
- **[Filter Predicate]**: 인덱스에서 범위를 결정하지 못하고 체크만 하는 조건.

📢 **마무리 요약**: **Index Range Scan**은 데이터베이스 조회의 표준입니다. 정밀한 조준과 빠른 수집을 통해 대규모 데이터 속에서 필요한 정보만을 낚아채는 가장 세련된 기법입니다.
