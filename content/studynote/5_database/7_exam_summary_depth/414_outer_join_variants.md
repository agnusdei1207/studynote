# 414. 외부 조인(Outer Join) - 소외된 데이터를 보듬다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 외부 조인(Outer Join)은 조인 조건에 만족하지 않는 튜플(행)이라 하더라도 결과에서 삭제하지 않고, **기준이 되는 릴레이션의 모든 데이터를 유지하며 상대측 데이터를 NULL로 채워 보여주는 연산**이다.
> 2. **가치**: 내부 조인(Inner Join) 시 발생하는 '정보의 손실'을 방지하여, "아직 부서가 없는 사원"이나 "주문 내역이 없는 상품" 등 비즈니스적으로 의미 있는 예외 사례를 모두 파악할 수 있게 한다.
> 3. **융합**: 왼쪽(Left), 오른쪽(Right), 전체(Full) 외부 조인으로 구분되며, 데이터 분석 및 보고서 작성 시 데이터의 완전성을 보장하는 필수 기술로 활용된다.

+++

### Ⅰ. 외부 조인의 유형별 작동 원리

1. **Left Outer Join ($R \bowtie^{+} S$)**:
    - 왼쪽 릴레이션 $R$의 모든 데이터를 보존. 짝이 없는 오른쪽 속성은 NULL 처리.
2. **Right Outer Join ($R ^{+}\bowtie S$)**:
    - 오른쪽 릴레이션 $S$의 모든 데이터를 보존. 짝이 없는 왼쪽 속성은 NULL 처리.
3. **Full Outer Join ($R ^{+}\bowtie^{+} S$)**:
    - 양쪽 릴레이션의 모든 데이터를 보존. 매칭되지 않는 모든 부분은 NULL 처리.

+++

### Ⅱ. 외부 조인 프로세스 시각화 (ASCII Model)

```text
[ Relation R: 사원 ]             [ Relation S: 부서 ]
┌────┬──────┬────────┐           ┌────────┬──────────┐
│ID  │이름  │DeptID  │           │DeptID  │부서명    │
├────┼──────┼────────┤           ├────────┼──────────┤
│101 │Kim   │ 10     │           │  10    │ 인사     │
│102 │Lee   │ NULL   │           │  20    │ 개발     │
└────┴──────┴────────┘           └────────┴──────────┘

[ Left Outer Join Result ]
┌────┬──────┬────────┬──────────┐
│ID  │이름  │DeptID  │부서명    │
├────┼──────┼────────┼──────────┤
│101 │Kim   │ 10     │ 인사     │
│102 │Lee   │ NULL   │ NULL     │ ✅ (부서 없는 사원 Lee 보존)
└────┴──────┴────────┴──────────┘
```

+++

### Ⅲ. 실무적 활용 시나리오

- **데이터 전수 조사**: "가입은 했으나 한 번도 구매하지 않은 고객 리스트"를 추출할 때 `Customer LEFT JOIN Order`를 사용하여 구매 정보가 NULL인 대상을 찾습니다.
- **마스터 데이터 검증**: 데이터 이관 후 낙오된 데이터가 없는지 확인할 때 필수적으로 사용됩니다.
- **주의**: 오라클의 고전 문법인 `(+)` 기호는 기준이 아닌 **반대편(NULL이 채워질 쪽)**에 붙입니다. (예: `WHERE R.ID = S.ID(+)` ──▶ R이 기준인 Left Join)

- **📢 섹션 요약 비유**: 외부 조인은 **'모두가 참여하는 소풍'**과 같습니다. 내부 조인이 짝꿍(조인 조건)이 있는 사람만 소풍을 가는 것이라면, 외부 조인은 짝꿍이 없어도 일단 기준이 되는 반 아이들 전원이 소풍을 가는 것입니다. 짝꿍이 없는 아이의 옆자리는 빈자리(NULL)로 남겨두어, 누가 혼자 왔는지를 한눈에 알 수 있게 하는 배려 깊은 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Inner Join]**: 짝이 맞는 데이터만 남기는 비정한 연산.
- **[Null Padding]**: 짝이 없는 자리를 NULL로 채우는 물리적 행위.
- **[Lossless Join]**: 외부 조인을 통해 데이터 손실을 막는 설계 기법.

📢 **마무리 요약**: **Outer Join**은 포용의 연산입니다. 조건에 맞지 않는 데이터 속에 숨겨진 비즈니스의 진실을 찾아내는 가장 강력한 도구입니다.
