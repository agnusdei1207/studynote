# 493. 데이터 거버넌스 3요소 - 사람, 프로세스, 기술의 조화

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 거버넌스는 전사적 차원에서 데이터의 가치를 극대화하기 위해 수립된 **사람(People), 프로세스(Process), 기술(Technology)의 통합적 관리 체계**다.
> 2. **가치**: "데이터는 누가 소유하고, 어떻게 관리하며, 어떤 기술로 보호하는가?"에 대한 표준을 제시하여 데이터 품질 사고를 예방하고 비즈니스 신뢰도를 높인다.
> 3. **융합**: 조직의 문화와 정책(관리적 보안)이 IT 시스템(기술적 보안)과 융합되어, 데이터가 단순한 정보 조각이 아닌 기업의 핵심 자산으로 기능하게 한다.

+++

### Ⅰ. 데이터 거버넌스의 3대 핵심 요소

1. **사람 (People)**:
    - **역할과 책임 (R&R)**: 데이터 소유자(Owner), 데이터 관리자(Steward), 거버넌스 위원회 구성.
    - 데이터 중심 조직 문화를 형성하고 전문 인력을 양성합니다.
2. **프로세스 (Process)**:
    - **절차와 규칙**: 데이터의 생성, 저장, 활용, 폐기에 이르는 생애주기별 표준 업무 절차 정의.
    - 데이터 품질 관리 및 보안 위반 시 대응 시나리오 수립.
3. **기술 (Technology)**:
    - **관리 도구**: 메타데이터 관리 시스템, 데이터 사전, 품질 모니터링 대시보드, 접근 제어 솔루션 등.
    - 거버넌스 정책을 시스템적으로 강제하고 자동화하는 IT 인프라.

+++

### Ⅱ. 거버넌스 삼각 체계도 시각화 (ASCII Model)

세 요소가 균형을 이룰 때 신뢰할 수 있는 데이터가 탄생합니다.

```text
[ Data Governance Triangle ]

           (People)
           조직 & 책임
               ▲
              / \
             /   \
  (Process) /_____\ (Technology)
   절차 & 규칙       도구 & 인프라

  * People + Process = Governance Framework 수립 ✅
  * Process + Tech = Automated Data Pipeline 구현 ✅
  * People + Tech = Data Literacy & Security 확보 ✅
```

### Ⅲ. 실무적 의의: 왜 세 요소가 다 필요한가?

- **기술만 있을 때**: 고가의 솔루션을 도입해도 관리 주체가 불분명하고 프로세스가 없으면 쓰레기 데이터만 쌓입니다. (Data Swamp)
- **사람만 있을 때**: 전문가가 있어도 수동 작업의 한계로 인해 대규모 데이터 품질을 유지할 수 없습니다.
- **프로세스만 있을 때**: 규칙은 거창하지만 이를 실행할 사람과 도구가 없어 문서 속에만 존재하는 거버넌스가 됩니다.

- **📢 섹션 요약 비유**: 데이터 거버넌스는 **'국가의 운영'**과 같습니다. 공무원과 국민(사람)이 있고, 법과 질서(프로세스)가 있으며, 이를 집행할 시설과 전산망(기술)이 있어야 나라가 평화롭고 부강해지는 것과 같습니다. 어느 하나라도 부족하면 무법천지(데이터 오염)가 되거나 성장이 멈추게 됩니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Data Stewardship]**: 현업 부서에서 실질적인 데이터 품질을 챙기는 사람들의 활동.
- **[Metadata Management]**: 기술적 거버넌스를 구현하는 핵심 근거 정보.
- **[Single Source of Truth]**: 거버넌스를 통해 달성하고자 하는 단일 진실 공급원.

📢 **마무리 요약**: **Data Governance**는 기업의 기초 체력입니다. 사람, 프로세스, 기술이라는 세 기둥을 단단히 세울 때, 데이터는 비로소 혁신의 원동력이 됩니다.
