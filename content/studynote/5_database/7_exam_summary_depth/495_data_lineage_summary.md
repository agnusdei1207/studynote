# 495. 데이터 리니지(Data Lineage) - 데이터의 가계도와 영향력

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 리니지는 데이터의 생성부터 가공, 이동, 최종 소비에 이르는 **전체 수명 주기를 시각적으로 추적하고 기록하는 기술**이다.
> 2. **가치**: 보고서 수치의 근거를 명확히 하는 **데이터 신뢰성**을 확보하며, 시스템 변경 시 하류(Downstream)에 미치는 파급 효과를 분석하는 **영향도 분석(Impact Analysis)**의 핵심 도구로 쓰인다.
> 3. **융합**: 메타데이터 관리와 융합되어 복잡한 데이터 파이프라인의 가시성을 제공하며, 규제 준수(Compliance)를 위한 데이터 투명성 입증의 필수 요건이 된다.

+++

### Ⅰ. 데이터 리니지의 주요 추적 항목

- **기원 (Origin)**: 데이터가 어느 소스 시스템의 어느 테이블에서 처음 만들어졌는가?
- **변환 (Transformation)**: 중간 단계에서 어떤 로직(SQL, ETL)에 의해 값이 바뀌었는가?
- **흐름 (Flow)**: 데이터가 어떤 경로(네트워크, API, 파일)를 통해 이동했는가?
- **종착지 (Destination)**: 최종적으로 어떤 보고서나 AI 모델에서 사용되고 있는가?

+++

### Ⅱ. 리니지 및 영향도 분석 시각화 (ASCII Model)

```text
[ Data Lineage & Impact Analysis Flow ]

  (Source DB)        (ETL Process)         (DW/Mart)         (BI Tool)
  ┌──────────┐      ┌─────────────┐      ┌─────────────┐      ┌─────────────┐
  │ Customer │ ──▶  │ Scrubbing   │ ──▶  │ Master Dim  │ ──▶  │ VIP Sales   │
  │ Profile  │      │ & Merging   │      │ Table       │      │ Dashboard   │ 📊
  └────┬─────┘      └─────────────┘      └─────────────┘      └─────────────┘
       │                                        │
       ▼ [ IMPACT ANALYSIS ]                    ▼ [ BACK TRACING ]
  "Source Table 변경 시"                  "지표가 이상할 때"
  "하류 시스템들 장애 위험!" ✅             "어디서 틀렸는지 추적" ✅
```

+++

### Ⅲ. 실무적 의의: 가시성 확보

1. **데이터 신뢰 회복**: 수치가 틀렸을 때 "어디서부터 잘못되었는지"를 역추적(Back-tracing)하여 원인을 빠르게 규명합니다.
2. **효율적 변경 관리**: 테이블 컬럼 하나를 지울 때, 이를 참조하는 수백 개의 프로그램을 일일이 뒤지는 대신 리니지 맵 하나로 모든 위험을 파악합니다.
3. **거버넌스 완성**: 데이터의 생명주기를 한눈에 보여줌으로써 데이터 관리의 투명성을 극대화합니다.

- **📢 섹션 요약 비유**: 데이터 리니지는 **'식재료 이력 추적제'**와 같습니다. 우리가 먹는 스테이크(최종 보고서)가 어느 농장(소스 DB)에서 왔는지, 도축과 가공(ETL)은 어디서 했는지, 배송(흐름)은 누가 했는지를 한눈에 보여주는 것입니다. 만약 고기가 상했다면(데이터 오류), 이 이력 지도를 보고 어느 단계에서 문제가 생겼는지 즉시 찾아낼 수 있는 아주 똑똑한 시스템입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Impact Analysis]**: 상류의 변화가 하류에 미치는 영향을 분석하는 기법.
- **[Data Traceability]**: 데이터의 발자취를 추적할 수 있는 능력.
- **[Metadata]**: 리니지를 그리기 위한 원천 재료 정보.

📢 **마무리 요약**: **Data Lineage**는 데이터의 과거와 현재를 연결하는 선입니다. 복잡하게 얽힌 데이터 파이프라인의 안개를 걷어내고, 진실된 데이터 흐름을 드러내는 가시성의 등불입니다.
