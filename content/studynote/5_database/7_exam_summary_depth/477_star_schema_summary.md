# 477. 스타 스키마(Star Schema) - 분석용 모델링의 표준

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스타 스키마는 대량의 수치 데이터인 **팩트 테이블(Fact Table)을 중심으로, 이를 설명하는 비정규화된 차원 테이블(Dimension Table)들이 별 모양으로 직접 연결**된 다차원 데이터 모델이다.
> 2. **가치**: 복잡한 관계를 단일 계층으로 단순화하여 **조인(Join) 횟수를 최소화**하며, 비즈니스 사용자가 이해하기 쉬운 직관적인 구조를 제공하여 쿼리 성능을 극대화한다.
> 3. **융합**: 킴벌(Kimball)의 DW 상향식 구축 방법론의 핵심이며, 현대의 모든 비즈니스 인텔리전스(BI) 툴과 OLAP 엔진의 물리적 데이터 토대가 된다.

+++

### Ⅰ. 스타 스키마의 구조적 특징

- **팩트 중심 (Fact Centric)**: 모든 차원이 하나의 팩트 테이블을 공통으로 바라봅니다.
- **단일 계층 (Single Layer)**: 차원 테이블은 더 이상 쪼개지지 않는 비정규화된 상태를 유지합니다 (Snowflake와 대조).
- **속도 지향**: 데이터 중복을 허용하더라도 검색 속도를 높이는 데 최적화되어 있습니다.

+++

### Ⅱ. 스타 스키마 구조 시각화 (ASCII Model)

```text
[ Star Schema Architecture ]

      [ Dim_Product ]          [ Dim_Time ]
      - ProductID (PK)         - TimeID (PK)
      - Name, Brand            - Year, Month, Day
             │                        │
             └───────┐        ┌───────┘
                     ▼        ▼
               [ Fact_Sales ] (Central)
               - SalesID (PK)
               - ProductID (FK)
               - TimeID (FK)
               - Amount, Qty (Measures) ✅
                     ▲        ▲
             ┌───────┘        └───────┐
             │                        │
      [ Dim_Region ]           [ Dim_Customer ]
      - RegionID (PK)          - CustomerID (PK)
      - City, State            - Name, Grade
```

+++

### Ⅲ. 실무적 장점

1. **조회 성능**: 팩트 테이블과 차원 테이블 간의 조인이 1단계로 끝나므로 대량의 집계 분석에 매우 유리합니다.
2. **사용자 편의성**: 데이터 구조가 단순하여 현업 사용자가 직접 쿼리를 작성하거나 보고서를 만들기에 적합합니다.
3. **최적화**: 옵티마이저가 스타 조인(Star Transformation) 최적화 기법을 사용하여 처리 속도를 한층 더 높일 수 있습니다.

- **📢 섹션 요약 비유**: 스타 스키마는 **'별 모양 사탕'**과 같습니다. 한가운데 가장 알차고 중요한 맛(팩트)이 있고, 그 주변을 예쁜 장식(차원)들이 직접 감싸고 있어 한입에 쏙 넣어 맛을 느끼기(분석)에 가장 효율적이고 단순한 형태입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Fact Table]**: 비즈니스의 수치적 결과를 담는 중심 테이블.
- **[Dimension Table]**: 분석의 기준(누가, 언제, 어디서)을 제공하는 주변 테이블.
- **[Surrogate Key]**: 차원 테이블 관리를 위해 사용하는 인공 키.

📢 **마무리 요약**: **Star Schema**는 데이터 웨어하우스의 클래식입니다. 단순함이 주는 강력한 성능과 직관성을 통해 복잡한 비즈니스 데이터를 명쾌한 통찰로 바꿔줍니다.
