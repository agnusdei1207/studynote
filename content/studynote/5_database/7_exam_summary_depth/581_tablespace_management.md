# 581. 테이블스페이스(Tablespace) - 물리적 저장의 분산 전략

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 테이블스페이스(Tablespace)는 하나 이상의 데이터 파일(.dbf)로 구성된 **데이터베이스의 가장 큰 논리적 저장 단위**로, 테이블이나 인덱스와 같은 객체들을 물리적 파일 그룹에 할당하는 역할을 한다.
> 2. **가치**: 데이터의 성격에 따라 물리적 저장 위치를 분리하여 **I/O 부하를 분산**시키고, 테이블스페이스 단위의 백업 및 복구를 통해 관리의 유연성을 극대화한다.
> 3. **융합**: 세그먼트(Segment), 익스텐트(Extent), 블록(Block)으로 이어지는 스토리지 계층 구조와 융합되어, 대규모 데이터를 물리적으로 최적 배치하는 인프라 설계의 기초가 된다.

+++

### Ⅰ. 테이블스페이스의 저장 구조 계층

- **Tablespace (논리)**: 여러 개의 데이터 파일 묶음.
- **Data File (물리)**: 실제 디스크에 저장되는 파일.
- **Segment**: 테이블, 인덱스 등 공간을 점유하는 객체 단위.
- **Extent**: 연속된 데이터 블록의 집합 (공간 할당의 단위).
- **Block (Page)**: 데이터를 읽고 쓰는 최소 I/O 단위. ✅

+++

### Ⅱ. 테이블스페이스 물리 배치 시각화 (ASCII Model)

서로 다른 물리 디스크에 테이블스페이스를 분산하여 성능을 높입니다.

```text
[ Database Storage Partitioning ]

  ┌──────────────────────────────────────────────────────────┐
  │  Logical Database                                        │
  └──────┬───────────────────────────┬───────────────────────┘
         ▼                           ▼
  ┌──────────────┐            ┌──────────────┐
  │ TS_DATA (L)  │            │ TS_INDEX (L) │ ✅ "I/O Isolation"
  └──────┬───────┘            └──────┬───────┘
         ▼                           ▼
  ┌──────────────┐            ┌──────────────┐
  │ Disk 1 (P)   │            │ Disk 2 (P)   │ ✅ "Parallel Access"
  │ [DataFile 1] │            │ [DataFile 2] │
  └──────────────┘            └──────────────┘
```

+++

### Ⅲ. 실무적 활용 가이드

1. **I/O 분산**: 테이블 데이터와 인덱스 데이터를 서로 다른 물리 디스크의 테이블스페이스에 저장하여 헤드 경합을 방지합니다.
2. **이력 데이터 관리**: 최신 데이터는 고속 SSD 테이블스페이스에, 과거 이력 데이터는 저가형 HDD 테이블스페이스에 배치합니다 (Tiering).
3. **가용성**: 특정 테이블스페이스에 문제가 생겨도 해당 파일만 오프라인으로 만들고 나머지 서비스는 유지할 수 있습니다.

- **📢 섹션 요약 비유**: 테이블스페이스는 **'대형 도서관의 전문 서가'**와 같습니다. 도서관(DB) 전체를 하나의 큰 방으로 쓰지 않고, '인문학실(TS_DATA)', '과학실(TS_INDEX)', '어린이실(TS_TEMP)' 등으로 구역을 나누어 관리하는 것입니다. 이렇게 나누어 두면 사람들이 한꺼번에 몰려도(I/O 부하) 각자 다른 방으로 흩어져 책을 찾을 수 있고, 한 방에 불이 나도(파일 장애) 다른 방의 책들은 안전하게 지킬 수 있는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[HWM (High Water Mark)]**: 테이블스페이스 내에서 데이터가 채워진 최고 지점.
- **[LMT (Locally Managed Tablespace)]**: 비트맵을 사용하여 익스텐트 할당을 효율적으로 관리하는 방식.
- **[System Tablespace]**: 데이터 사전(Data Dictionary)이 저장되는 가장 중요한 기본 공간.

📢 **마무리 요약**: **Tablespace**는 데이터의 물리적 기초입니다. 논리와 물리를 연결하는 이 계층을 어떻게 설계하느냐에 따라 시스템의 확장성과 안정성이 결정됩니다.
