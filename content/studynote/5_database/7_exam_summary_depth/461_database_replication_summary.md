# 461. 복제(Replication) - 가용성과 성능의 동시 확보

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 복제는 데이터베이스의 가용성과 성능을 높이기 위해, **동일한 데이터를 물리적으로 분리된 여러 서버(Node)에 중복하여 저장하고 실시간으로 동기화하는 기술**이다.
> 2. **가치**: 마스터(Master) 노드 장애 시 슬레이브(Slave)를 승격시켜 서비스 중단을 막는 **고가용성(HA)**과, 다수의 읽기 전용 복제본을 통한 **조회 부하 분산**을 실현한다.
> 3. **융합**: 복제 시점(동기/비동기)과 구조(Master-Slave / Multi-Master)에 따라 다양한 정합성 모델과 융합되며, 현대 글로벌 클라우드 인프라의 핵심 데이터 전략이 된다.

+++

### Ⅰ. 마스터-슬레이브(Master-Slave) 복제 구조

- **마스터 (Primary)**: 모든 쓰기(Write) 연산을 독점 수행하며, 변경 내역을 로그로 기록합니다.
- **슬레이브 (Replica)**: 마스터의 로그를 받아 자신의 데이터를 업데이트하고, 오직 읽기(Read) 요청만 처리합니다.
- **장점**: 쓰기 충돌이 없으며 조회가 많은 서비스에서 슬레이브 증설을 통해 무한 확장이 가능합니다.

+++

### Ⅱ. 복제 아키텍처 및 장애 대응 시각화 (ASCII Flow)

```text
[ Master-Slave Replication & Failover ]

      (Write App)          (Read App)
             │                   │
      ┌──────▼──────┐     ┌──────▼──────┐
      │   Master    │ ──▶ │   Slave 1   │ ✅ (Normal)
      │ (Read/Write)│     └─────────────┘
      └──────┬──────┘     ┌─────────────┐
             └──────────▶ │   Slave 2   │ ✅ (Normal)
             (Sync Log)   └─────────────┘

  (--- 💥 Master Down! ---)

      ┌─────────────┐     ┌─────────────┐
      │ [ Dead ]    │     │ New Master  │ 👑 (Promoted)
      └─────────────┘     └─────────────┘
```

+++

### Ⅲ. 동기 vs 비동기 복제의 트레이드오프

1. **동기식 복제 (Synchronous)**: 모든 복제본에 기록이 완료되어야 성공으로 인정. 데이터 일관성은 완벽하지만 응답 속도가 느리고 노드 하나만 죽어도 시스템 전체가 멈출 수 있습니다.
2. **비동기식 복제 (Asynchronous)**: 마스터에만 쓰고 나중에 복제. 응답은 매우 빠르지만, 찰나의 순간에 데이터가 서로 다를 수 있는 '복제 지연(Replication Lag)'이 발생합니다.

- **📢 섹션 요약 비유**: 복제는 **'유명 작가와 필사생들'**과 같습니다. 작가(마스터)가 원고를 쓰면(쓰기), 여러 명의 필사생(슬레이브)들이 그 원고를 베껴 써서 독자들에게 나누어 줍니다(읽기 분산). 만약 작가가 아프면(장애), 가장 실력이 좋은 필사생이 그 자리를 이어받아 글을 계속 쓰는(Failover) 방식으로 독자들이 끊김 없이 책을 읽을 수 있게 하는 것입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Failover]**: 장애 발생 시 자동으로 주 서버를 교체하는 과정.
- **[Read Replica]**: 읽기 전용으로 운영되는 고성능 복제 노드.
- **[CDC]**: 복제를 구현하기 위한 변경 데이터 캡처 기술.

📢 **마무리 요약**: **Replication**은 현대 DB 운영의 필수 사양입니다. 가용성과 성능이라는 두 마리 토끼를 잡기 위해, 비즈니스의 정합성 허용 수준에 맞는 최적의 복제 전략을 선택해야 합니다.
