# 393. 데이터 사전 (Data Dictionary) - 데이터베이스의 백과사전

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 사전은 데이터베이스에 존재하는 모든 객체(테이블, 인덱스, 뷰 등)의 정의와 속성 정보를 저장하고 있는 **시스템 전용의 읽기 전용 저장소(Meta-Database)**다.
> 2. **가치**: DBMS가 쿼리를 실행할 때 유효성을 검증하고 실행 계획을 세우는 기초 자료로 쓰이며, DBA는 이를 조회하여 시스템의 상태와 구조를 파악한다.
> 3. **융합**: '시스템 카탈로그'라고도 불리며, 데이터 거버넌스의 핵심 도구인 메타데이터 관리 시스템의 물리적 원천이 된다.

+++

### Ⅰ. 데이터 사전의 주요 내용 (Metadata)

데이터 사전에는 실제 데이터가 아닌 '데이터에 대한 정보'가 담깁니다.
- **스키마 정보**: 테이블명, 컬럼명, 데이터 타입, 제약 조건 (PK, FK 등).
- **무결성 정보**: 체크 제약 조건, 트리거 정의.
- **물리적 정보**: 테이블스페이스 위치, 파일 경로, 스토리지 파라미터.
- **통계 정보**: 레코드 수, 블록 사용량, 인덱스 깊이 (Optimizer 활용).
- **권한 정보**: 사용자 계정, 부여된 역할(Role) 및 권한.

+++

### Ⅱ. 데이터 사전의 계층 구조 시각화 (ASCII Model)

오라클의 예시를 통한 데이터 사전의 뷰(View) 계층입니다.

```text
[ Data Dictionary View Hierarchy ]

  ┌────────────────────────────────────────────────────────┐
  │  DBA_ (시스템 전체 정보 - 관리자용)                     │
  ├────────────────────────────────────────────────────────┤
  │  ALL_ (내가 접근 가능한 모든 정보 - 일반 사용자용)      │
  ├────────────────────────────────────────────────────────┤
  │  USER_ (내가 생성한 객체 정보 - 소유자용)               │
  └──────────────────────────┬─────────────────────────────┘
                             ▼
                [ Data Dictionary Base Tables ]
                (DBMS만 직접 수정 가능한 시스템 테이블)
```

+++

### Ⅲ. 데이터 사전의 활용 시나리오

1. **질의 분석**: 사용자가 쿼리를 날리면, DBMS는 데이터 사전을 뒤져 "테이블이 존재하는가?", "컬럼명이 맞는가?", "조회 권한이 있는가?"를 0.1초 만에 확인합니다.
2. **최적화**: 옵티마이저는 데이터 사전의 통계 정보를 보고 가장 빠른 검색 경로를 결정합니다.
3. **스키마 탐색**: 관리자는 `SELECT * FROM USER_TABLES;` 쿼리를 통해 자신이 가진 자원을 확인합니다.

- **📢 섹션 요약 비유**: 데이터 사전은 **'도서관의 통합 도서 목록 장부'**와 같습니다. 실제 책(데이터)이 어디 있는지, 제목(테이블명)은 무엇인지, 대출 가능한지(권한)를 모두 적어둔 두꺼운 장부입니다. 사서(DBMS)는 이 장부를 보고 우리에게 책을 찾아주며, 우리는 이 장부를 보고 도서관에 어떤 책들이 있는지 알 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[System Catalog]**: 데이터 사전의 또 다른 명칭.
- **[Metadata]**: 데이터 사전에 저장되는 '데이터에 관한 데이터'.
- **[Active vs Passive]**: DBMS가 자동으로 갱신하면 Active, 수동으로 기록하면 Passive 사전.

📢 **마무리 요약**: **Data Dictionary**는 데이터베이스의 뇌이자 지도입니다. 모든 운영의 근거가 여기에 담겨 있으므로, 이를 잘 이해하고 조회하는 능력이 전문가의 필수 역량입니다.
