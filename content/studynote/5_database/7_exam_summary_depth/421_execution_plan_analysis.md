# 421. 실행 계획 (Execution Plan) - 쿼리의 내부 지도

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실행 계획은 사용자가 작성한 SQL 쿼리를 실제로 처리하기 위해 **데이터베이스 옵티마이저가 선택한 구체적인 연산 순서와 데이터 접근 경로의 명세서**다.
> 2. **가치**: 쿼리 성능이 느릴 때 어느 단계에서 병목(Full Scan 등)이 발생하는지 시각적으로 보여주며, 인덱스 활용 여부와 조인 방식을 사전에 확인하는 **성능 튜닝의 핵심 도구**다.
> 3. **융합**: 시스템 통계 정보와 옵티마이저의 판단 로직이 융합된 결과물이며, `EXPLAIN` 명령어를 통해 트리 구조의 계층적 연산 과정으로 노출된다.

+++

### Ⅰ. 실행 계획의 주요 분석 항목

- **Operation**: 수행되는 구체적 작업 (TABLE ACCESS FULL, INDEX RANGE SCAN, HASH JOIN 등).
- **Object Name**: 해당 연산이 일어나는 테이블이나 인덱스 명칭.
- **Cost**: 옵티마이저가 예상한 상대적인 자원 소모량.
- **Cardinality (Rows)**: 각 단계에서 반환될 것으로 예상되는 데이터 건수.
- **Access Predicate**: 인덱스를 찾기 위한 검색 조건.
- **Filter Predicate**: 검색 후 데이터를 걸러내기 위한 조건.

+++

### Ⅱ. 실행 계획 시각화 (ASCII Tree)

상위 노드가 하위 노드의 결과를 받아 처리하는 트리 구조입니다.

```text
[ Sample Execution Plan ]

  (Query) SELECT * FROM Emp WHERE Dept = 'IT' AND Salary > 5000;

  Id  | Operation                    | Name       | Rows | Cost |
  ---------------------------------------------------------------
   0  | SELECT STATEMENT             |            |   10 |   50 |
   1  |  FILTER                      |            |   10 |   50 |
   2  |   TABLE ACCESS BY INDEX ROWID| EMP        |   50 |   40 |
   3  |    INDEX RANGE SCAN          | EMP_DEPT_IX|   50 |    2 | ✅ (Start Here)
  ---------------------------------------------------------------

  [ Logic Flow ]
  Step 3 (Index Scan) ──▶ Step 2 (Fetch Data) ──▶ Step 1 (Salary Filter) ──▶ Result ✅
```

+++

### Ⅲ. 실행 계획 분석 시 핵심 포인트

1. **Full Table Scan 여부**: 인덱스를 타야 하는 쿼리가 전체 테이블을 읽고 있지는 않은지 확인합니다.
2. **Join 순서**: 작은 테이블을 먼저 읽어 드라이빙(Driving) 하고 있는지 체크합니다.
3. **인덱스 활용도**: 조건절(WHERE)에 명시한 컬럼이 인덱스를 정상적으로 사용하고 있는지 검증합니다.

- **📢 섹션 요약 비유**: 실행 계획은 **'내비게이션의 상세 경로 안내'**와 같습니다. "강남역까지 가라"는 명령(SQL)을 내렸을 때, 내비게이션(옵티마이저)이 "골목길(인덱스)로 가다가 대로(풀 스캔)로 합류해서 가라"고 알려주는 상세 루트와 같습니다. 길이 막히는지(비용), 예상 도착 시간(응답 시간)은 언제인지를 미리 보고 최적의 길인지 판단하는 도구입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[EXPLAIN PLAN]**: 실행 계획을 확인하기 위한 표준 명령어.
- **[Optimizer]**: 실행 계획을 만드는 주체 (CBO).
- **[Step-by-step]**: 안쪽(가장 깊은 계층)에서 바깥쪽으로 읽는 실행 계획 판독 원칙.

📢 **마무리 요약**: **Execution Plan**은 데이터베이스 튜닝의 시작점입니다. 쿼리의 속살을 들여다보는 지도를 통해, 비효율을 찾아내고 성능의 정점에 도달할 수 있습니다.
