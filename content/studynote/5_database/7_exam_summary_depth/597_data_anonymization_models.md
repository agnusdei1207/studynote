# 597. 개인정보 비식별화 모델 - 프라이버시의 수학적 방벽

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 개인정보 비식별화 모델은 데이터셋에서 특정인을 식별할 수 없도록 **데이터를 변형하고, 그 익명성 수준을 수학적으로 측정하고 보장하는 평가 체계**다.
> 2. **가치**: 연결 공격(Linkage Attack)이나 유추 공격으로부터 정보 주체를 보호하며, **K-익명성, L-다양성, T-근접성** 모델을 통해 데이터 활용의 유용성과 프라이버시 보호 사이의 최적 균형을 찾는다.
> 3. **융합**: 데이터 3법 등 법적 규제 준수 가이드라인과 융합되어, 기업 간 데이터 결합이나 통계 분석을 위한 기술적 안전장치로 작동한다.

+++

### Ⅰ. 3대 비식별화 보호 모델 상세

1. **K-익명성 (K-Anonymity)**:
    - **정의**: 특정인으로 식별될 수 있는 레코드가 최소 K개 이상 존재하게 함.
    - **방어**: 특정인을 1명으로 콕 집어내는 '식별 공격'을 방어.
2. **L-다양성 (L-Diversity)**:
    - **정의**: K-익명성을 만족하는 그룹 내에서 민감한 정보(질병 등)가 최소 L개 이상의 서로 다른 종류를 갖게 함.
    - **방어**: 특정 그룹이 모두 동일한 질병을 가진 경우 발생하는 '동질성 공격' 방어. ✅
3. **T-근접성 (T-Closeness)**:
    - **정의**: 전체 데이터의 민감 정보 분포와 특정 비식별 그룹의 분포 차이를 T 이하로 유지.
    - **방어**: 정보의 쏠림 현상을 이용한 '유추 공격' 방어.

### Ⅲ. 비식별 조치 가이드라인 및 평가 기준

- **적정성 평가**: 비식별 조치가 완료된 후, 외부 전문가가 포함된 '적정성 평가 위원회'를 통해 데이터 배포 여부를 결정합니다.
- **재식별 가능성 모니터링**: 배포된 데이터가 다른 외부 데이터와 결합되어 개인이 특정될 위험이 없는지 지속적으로 감시해야 합니다.
- **기술적 기법**: 가명처리(Pseudonymization), 총계처리(Aggregation), 데이터 마스킹(Masking), 범주화(Generalization) 등을 상황에 맞게 조합합니다.

- **📢 섹션 요약 비유**: 비식별화 모델은 **'가면무도회의 입장 조건'**과 같습니다. K-익명성은 똑같은 가면을 쓴 사람을 최소 5명 이상 있게 하여 얼굴로 누구인지 맞히지 못하게 하는 것이고, L-다양성은 그 가면 쓴 사람들이 들고 있는 가방 색깔을 다 다르게 해서 가방만 보고 주인을 유추하지 못하게 하는 것입니다. T-근접성은 무도회 안의 가방 색깔 비율을 세상 전체의 가방 비율과 비슷하게 맞춰서 "이 동네 사람은 다 빨간 가방만 드네"라는 편견(유추)을 막는 아주 꼼꼼한 규칙입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Quasi-identifier]**: 결합 시 식별 위험이 있는 속성 (우편번호, 나이 등).
- **[Differential Privacy]**: 수학적 노이즈를 섞어 더 강력한 프라이버시를 보장하는 차세대 모델.
- **[DPIA]**: 개인정보 영향 평가 절차.

📢 **마무리 요약**: **Anonymization Models**는 데이터 경제의 안전띠입니다. 수학적으로 증명된 익명성 모델을 통해 프라이버시 침해 걱정 없는 투명한 데이터 세상을 만들어가야 합니다.
