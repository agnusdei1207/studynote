# 503. 데이터 가상화 푸시다운(Push-down) 최적화 - 연산의 현지화

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 푸시다운(Push-down) 최적화는 데이터 가상화 환경에서 전체 데이터를 가상화 서버로 가져오지 않고, **필터링(WHERE), 집계(GROUP BY), 조인 등의 연산을 데이터가 실제 저장된 원천 DB에서 직접 수행하도록 명령을 내려보내는 기술**이다.
> 2. **가치**: 네트워크 전송 데이터량을 획기적으로 줄여 지연 시간(Latency)을 최소화하며, 원천 DB의 인덱스와 최적화된 연산 능력을 100% 활용하여 **분산 질의 성능을 극대화**한다.
> 3. **융합**: 고도의 쿼리 옵티마이저 기술과 소스 시스템별 방언(Dialect) 변환 기술이 융합되어, 마치 하나의 고성능 DB를 쓰는 것과 같은 가상 통합 환경을 실현한다.

+++

### Ⅰ. 푸시다운 최적화의 작동 원리

1. **질의 분석**: 사용자가 가상 뷰를 대상으로 복잡한 SQL을 던집니다.
2. **최적화 판단**: 가상화 엔진은 쿼리의 어느 부분(예: `Age > 30` 조건)을 원천 DB에서 처리할 수 있는지 판단합니다.
3. **명령 전파 (Push-down)**: 해당 조건절을 원천 DB의 문법에 맞게 변환하여 실행하도록 지시합니다.
4. **결과 취합**: 소스 DB가 필터링을 마친 '작은 결과셋'만 네트워크로 보내면, 가상화 엔진이 이를 합쳐 최종 응답합니다.

+++

### Ⅱ. 푸시다운 최적화 전후 비교 (ASCII Model)

```text
[ 1. No Push-down (Bad) ]             [ 2. With Push-down (Good) ]

  (User) "Salary > 5000"                (User) "Salary > 5000"
     │                                     │
  [Virtual Server]                      [Virtual Server]
     ▲                                     ▲
     │ (1,000,000 Rows 전송) 💥             │ (10 Rows 만 전송) ✅
     │                                     │
  [Source DB]                           [Source DB] ◀── "WHERE Salary > 5000"
  (Filter at Virtual Server)            (Filter at SOURCE using Index!)
```

+++

### Ⅲ. 주요 푸시다운 유형

- **Predicate Push-down**: `WHERE` 절의 필터 조건을 소스 DB로 보냄.
- **Projection Push-down**: 필요한 컬럼만 소스에서 읽어오도록 지시.
- **Aggregate Push-down**: `SUM`, `AVG` 등 집계 연산을 소스에서 미리 수행.
- **Join Push-down**: 같은 소스 시스템 내에 있는 테이블들끼리는 소스 내부에서 조인하도록 유도.

- **📢 섹션 요약 비유**: 푸시다운 최적화는 **'현지 식재료 손질'**과 같습니다. 멀리 있는 시장(Source DB)에서 채소 100박스(전체 데이터)를 우리 집(가상화 서버)으로 다 배달시킨 뒤에 상한 것을 골라내는(필터링) 대신, 시장 상인에게 "좋은 것만 1박스로 추려서 보내주세요"라고 미리 부탁하는 것과 같습니다. 배송비(네트워크 비용)가 획기적으로 줄어들고 요리 시간도 훨씬 빨라집니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Predicate]**: 데이터 추출 조건을 나타내는 술어 (WHERE 절).
- **[Network I/O]**: 데이터 가상화의 가장 큰 적이자 푸시다운이 해결해야 할 대상.
- **[Query Rewrite]**: 소스 DB가 이해할 수 있는 언어로 쿼리를 다시 쓰는 과정.

📢 **마무리 요약**: **Push-down Optimization**은 데이터 가상화의 생명선입니다. 데이터를 가져오는 대신 '지능'을 소스로 보낼 때, 비로소 물리적 통합을 넘어서는 진정한 논리적 통합이 완성됩니다.
