# 522. 데이터 거버넌스 3요소 - 원칙, 조직, 프로세스의 균형

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 거버넌스는 전사적 데이터를 자산으로 관리하기 위해 수립된 **원칙(Policies), 조직(Organization), 프로세스(Process)**의 통합적인 체계다.
> 2. **가치**: "데이터는 어떤 규칙으로 관리되고, 누가 책임지며, 어떤 절차를 거치는가?"를 명확히 하여, 데이터 사일로(Silo)를 방지하고 데이터 품질의 상향 평준화를 이룬다.
> 3. **융합**: 비즈니스 전략과 IT 기술이 융합된 형태이며, 최근에는 자동화된 거버넌스 툴을 통해 실시간으로 정책을 집행하는 시스템적 거버넌스로 진화하고 있다.

+++

### Ⅰ. 데이터 거버넌스의 3대 핵심 요소

1. **원칙 (Principle / Policy)**:
    - **데이터 표준**: 단어, 도메인, 코드의 전사적 통일.
    - **관리 정책**: 데이터의 생성부터 파기까지의 생애주기별 보안 및 활용 가이드라인.
2. **조직 (Organization)**:
    - **데이터 소유자(Owner)**: 비즈니스 관점에서 데이터의 가치를 책임지는 임원/팀장.
    - **데이터 관리자(Steward)**: 데이터 품질을 실무적으로 관리하는 담당자.
    - **거버넌스 위원회**: 이해관계 조율 및 최종 의사결정 기구.
3. **프로세스 (Process)**:
    - **업무 절차**: 데이터 변경 요청, 승인, 배포, 품질 점검의 표준화된 워크플로우.
    - **모니터링**: 거버넌스 준수 여부를 상시 감시하고 리포팅하는 활동.

+++

### Ⅱ. 거버넌스 요소 간 상호작용 시각화 (ASCII Model)

세 요소가 맞물려 돌아가야 신뢰할 수 있는 데이터가 생산됩니다.

```text
[ Data Governance Pillar Model ]

           (원칙: Policy) 📜
           "무엇을 지킬 것인가?"
               ▲
              / \
             /   \
  (조직: Org) 👥 ─── (프로세스: Process) ⚙️
 "누가 책임지는가?"   "어떻게 실행하는가?"

  * Result: [ Trusted Data Assets ] ✅
  * Value: Consistency, Security, Quality
```

+++

### Ⅲ. 실무적 의의: 거버넌스 실패의 원인

- **원칙만 있을 때**: 서류상의 규정일 뿐, 현업에서는 아무도 지키지 않는 무거운 규제가 됩니다.
- **조직만 있을 때**: 사람(위원)은 많지만 무엇을 어떻게 해야 할지 몰라 탁상공론만 이어집니다.
- **프로세스만 있을 때**: 기계적인 절차는 있으나 비즈니스 방향성과 맞지 않아 효율성이 떨어집니다.

- **📢 섹션 요약 비유**: 데이터 거버넌스는 **'교통 시스템'**과 같습니다. 교통 법규(원칙)가 있고, 이를 관리하는 경찰과 운전자(조직)가 있으며, 면허 발급과 신호 체계(프로세스)가 조화를 이루어야 합니다. 그래야만 도로 위에서 사고(데이터 유출/오류) 없이 수많은 차(데이터)가 안전하게 달릴 수 있는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Data Stewardship]**: 현업 접점에서 거버넌스를 실천하는 실무 활동.
- **[Metadata]**: 거버넌스를 기술적으로 가능케 하는 핵심 근거 정보.
- **[DMM (Maturity Model)]**: 거버넌스 수준을 평가하는 성숙도 지표.

📢 **마무리 요약**: **Data Governance 3 Elements**는 데이터 자산화의 필수 요건입니다. 어느 하나에 치우치지 않는 균형 잡힌 관리 체계만이 데이터라는 거친 파도를 비즈니스의 순풍으로 바꿀 수 있습니다.
