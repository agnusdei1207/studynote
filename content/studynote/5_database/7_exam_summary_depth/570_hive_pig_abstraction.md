# 570. 하둡 쿼리 엔진(Hive & Pig) - 맵리듀스의 추상화 혁명

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Hive와 Pig는 자바(Java) 기반의 복잡한 **맵리듀스(MapReduce) 코드를 직접 작성하지 않고도**, SQL이나 스크립트 언어를 통해 대규모 분산 데이터를 손쉽게 분석할 수 있게 돕는 상위 레벨의 추상화 도구다.
> 2. **가치**: 데이터 분석가와 개발자들의 진입 장벽을 낮추어 **하둡 대중화**를 이끌었으며, 논리적인 질의를 최적화된 물리적 실행 계획으로 변환하여 분석 생산성을 극대화한다.
> 3. **융합**: HDFS의 분산 저장 능력과 YARN의 자원 관리 기술이 융합되어, 정형(Hive) 및 반정형(Pig) 데이터에 대한 전사적인 배치 분석 인프라를 형성한다.

+++

### Ⅰ. Hive와 Pig의 특징 비교

| 비교 항목 | Apache Hive (하이브) | Apache Pig (피그) |
|:---|:---|:---|
| **언어/문법** | **HiveQL** (SQL과 매우 유사) | **Pig Latin** (데이터 흐름 중심 스크립트) |
| **주요 사용자** | 데이터 분석가, 비즈니스 인텔리전스 | 데이터 엔지니어, 개발자 |
| **데이터 성격** | **정형 데이터** (테이블 구조) | 반정형/비정형 데이터 처리 유리 |
| **추상화 방식** | 선언형 (무엇을 찾을 것인가) | 절차형 (어떻게 흐르게 할 것인가) |

+++

### Ⅱ. 하둡 분석 계층화 시각화 (ASCII Model)

```text
[ Hadoop Analysis Abstraction Layer ]

  (User) ──▶ [ SQL Query ]      (User) ──▶ [ Pig Script ]
                │                          │
  ┌─────────────▼─────────────┐    ┌───────▼─────────────┐
  │  Apache Hive (Logical)    │    │ Apache Pig (Flow)   │ ✅ "Abstraction"
  └─────────────┬─────────────┘    └───────┬─────────────┘
                │                          │
  ┌─────────────▼──────────────────────────▼─────────────┐
  │  Execution Engine: MAPREDUCE / TEZ / SPARK           │ ✅ "Physical"
  └──────────────────────────┬───────────────────────────┘
                             ▼
                    [ HDFS Data Blocks ] 💾 ✅
```

### Ⅲ. 실무적 의의: 왜 추상화가 필요한가?

- **생산성**: 수백 줄의 자바 코드로 작성해야 할 '워드 카운트' 프로그램을 Hive에서는 단 5줄의 SQL로 끝낼 수 있습니다.
- **최적화**: 엔진이 내부적으로 조인 순서를 바꾸거나 필터링 시점을 조정하여, 초보자가 짠 코드보다 더 효율적인 맵리듀스 작업을 생성합니다.
- **유연성 (Schema-on-Read)**: 데이터를 담을 때가 아닌, 읽을 때 스키마를 입히는 방식을 통해 다양한 데이터 소스를 즉시 분석할 수 있습니다.

- **📢 섹션 요약 비유**: 하이브와 피그는 **'빅데이터용 번역기'**와 같습니다. 하둡이라는 거대한 창고에는 기계어(맵리듀스)만 알아듣는 일꾼들이 사는데, 우리가 익숙한 영어(SQL)나 설명서(Pig Latin)를 내밀면 이 번역기들이 일꾼들이 알아들을 수 있는 작업 지시서로 바꾸어 전달하는 것과 같습니다. 덕분에 우리는 기계어를 배우지 않고도 거대한 창고의 물건들을 마음껏 정리하고 조사할 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Metastore]**: 하이브 테이블의 구조 정보가 담긴 저장소.
- **[ETL Pipeline]**: 피그가 주로 활용되는 대규모 데이터 변환 공정.
- **[Tez / Spark Engine]**: 성능 향상을 위해 맵리듀스 대신 사용되는 현대적 실행 엔진.

📢 **마무리 요약**: **Hive & Pig**는 하둡의 뇌를 빌려 쓰는 법을 가르쳐주었습니다. 복잡함을 단순함으로 치환한 이 도구들이 있었기에 빅데이터는 비로소 비즈니스의 언어가 될 수 있었습니다.
