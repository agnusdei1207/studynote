# 574. 공통 차원(Conformed Dimension) - 데이터 마트를 잇는 고리

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공통 차원(Conformed Dimension)은 기업 내 여러 데이터 마트(Data Mart)에서 **동일한 의미와 구조를 가지고 공유되는 차원 테이블(시간, 고객, 상품 등)**을 의미한다.
> 2. **가치**: 부서마다 제각각인 기준 정보로 인해 발생하는 데이터 불일치를 방지하여, 서로 다른 마트의 팩트 데이터를 **하나의 쿼리로 조인(Drill-across)**할 수 있게 하는 전사적 정합성의 근간이 된다.
> 3. **융합**: 킴벌(Kimball)의 **데이터 웨어하우스 버스 아키텍처(Bus Architecture)**의 핵심 요소이며, 분권화된 마트 구축 환경에서도 전사 통합 뷰를 가능케 하는 논리적 접착제 역할을 한다.

+++

### Ⅰ. 공통 차원의 필요성과 조건

- **문제점**: 마케팅팀의 '고객' 테이블과 재무팀의 '고객' 테이블이 다르면, 고객별 매출과 고객별 캠페인 효과를 한눈에 분석할 수 없습니다.
- **성립 조건**:
    - **동일한 기본키**: 모든 마트에서 같은 ID 체계를 사용해야 합니다.
    - **동일한 속성 정의**: 컬럼의 이름과 데이터 타입이 완벽히 일치해야 합니다.
    - **동일한 데이터 값**: 특정 ID에 해당하는 이름이나 주소 값이 모든 곳에서 같아야 합니다.

+++

### Ⅱ. 버스 아키텍처와 공통 차원 시각화 (ASCII Model)

```text
[ Kimball Data Warehouse Bus Matrix ]

               | Time(D) | Product(D)| Customer(D)| Store(D) | ✅ Conformed
  -------------|---------|-----------|------------|----------|
  Sales Mart   |    X    |     X     |      X     |    X     |
  Inventory    |    X    |     X     |            |    X     |
  Marketing    |    X    |           |      X     |          |
  (Fact Tables)

  * X 표시된 차원들은 전사적으로 "동일한 테이블"을 공유함 ✅
  * 효과: Sales와 Inventory를 'Product' 키로 즉시 Join 가능 (Drill-across)
```

+++

### Ⅲ. 실무적 의의: Drill-across 분석

- **정의**: 두 개 이상의 팩트 테이블을 공통 차원을 매개로 결합하여 분석하는 행위.
- **예시**: "상품(공통 차원)"을 기준으로 "매출 팩트"와 "재고 팩트"를 한 줄로 나열하여 '적정 재고 대비 판매율'을 도출합니다.
- **관리 전략**: 중앙의 MDM(Master Data Management) 시스템에서 공통 차원을 먼저 정의하고, 각 마트로 배포하는 '하이브리드' 방식이 권장됩니다.

- **📢 섹션 요약 비유**: 공통 차원은 **'나토(NATO) 표준 규격의 부품'**과 같습니다. 각 나라(부서)가 자기만의 무기(데이터 마트)를 만들더라도, 탄환이나 연료 주입구(차원 데이터)를 표준 규격으로 통일해 두었기 때문에 전쟁 시(전사 분석 시) 서로의 자원을 빌려 쓰거나 연합 작전을 펼칠 수 있는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Bus Architecture]**: 공통 차원을 행렬 형태로 정의한 설계 지도.
- **[Drill-across]**: 공통 차원을 이용해 여러 팩트를 넘나드는 분석 기법.
- **[Surrogate Key]**: 공통 차원의 일관성을 유지하기 위해 사용하는 인공 키.

📢 **마무리 요약**: **Conformed Dimension**은 데이터 분석의 '공용어'입니다. 이 약속이 지켜질 때, 파편화된 데이터 마트들은 비로소 거대한 지식의 연합체로 거듭납니다.
