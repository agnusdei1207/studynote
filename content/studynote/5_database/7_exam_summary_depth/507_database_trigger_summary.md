# 507. 트리거(Trigger) - 데이터의 자동 반응 장치

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트리거(Trigger)는 특정 테이블에 **삽입(INSERT), 수정(UPDATE), 삭제(DELETE)와 같은 DML 이벤트가 발생할 때, 데이터베이스가 자동으로 실행하는 저장 프로시저**의 일종이다.
> 2. **가치**: 복잡한 비즈니스 제약 조건을 DB 레벨에서 강제하고, 데이터 변경 시 연관된 다른 테이블의 데이터를 자동으로 갱신(Audit, Summary 등)하여 **데이터 무결성**을 보장한다.
> 3. **특징**: 명시적인 호출 없이 시스템에 의해 '방격'되는 수동적 자동화 도구이며, 과도한 사용은 성능 저하와 복잡한 연쇄 반응을 일으킬 수 있어 신중한 설계가 요구된다.

+++

### Ⅰ. 트리거의 주요 구성 및 시점

- **이벤트 (Event)**: INSERT, UPDATE, DELETE (어떤 동작이 일어날 때?)
- **시점 (Timing)**:
    - **BEFORE**: 실제 데이터가 바뀌기 전 실행 (값 검증, 보정에 유리).
    - **AFTER**: 데이터가 바뀐 후 실행 (로그 기록, 통계 갱신에 유리).
- **레벨 (Level)**:
    - **Row Level**: 영향받은 각 행마다 실행 (`FOR EACH ROW`).
    - **Statement Level**: SQL 문장당 딱 한 번 실행.

+++

### Ⅱ. 트리거 작동 아키텍처 시각화 (ASCII Model)

```text
[ Database Trigger Mechanism ]

  (User SQL) : UPDATE Stock SET Qty = 0 WHERE ID = 101;
       │
       ▼ [ DB Engine ]
  ┌─────────────────────────────────────────┐
  │ 1. Event Detected (UPDATE Stock)        │
  │ 2. Trigger Fired (🔥 Auto-Execute)      │ ✅
  │    ┌─────────────────────────────────┐  │
  │    │ [ Trigger Logic ]               │  │
  │    │ INSERT INTO Out_Of_Stock_Log... │  │
  │    └─────────────────────────────────┘  │
  │ 3. Final Commit                         │
  └────────────────────┬────────────────────┘
                       ▼
                [ Target Table ] + [ Log Table ] 💾
```

+++

### Ⅲ. 실무적 활용 사례와 주의사항

- **활용**:
    - **Audit (감사)**: 누가 언제 어떤 데이터를 바꿨는지 로그를 자동으로 남김.
    - **통계 자동 갱신**: 주문이 들어오면 실시간 매출 합계 테이블을 업데이트.
    - **제약 조건 확장**: 복잡한 논리적 체크(예: "주말에는 주문 금지")를 수행.
- **주의사항**: 
    - **보이지 않는 로직**: 소스 코드에 드러나지 않아 디버깅이 매우 어렵습니다.
    - **성능 저하**: 대량의 데이터 처리 시 트리거가 반복 실행되어 시스템 전체가 느려질 수 있습니다.
    - **연쇄 트리거 (Mutation)**: 트리거가 다른 트리거를 깨우고, 그것이 다시 나를 깨우는 무한 루프 위험이 있습니다.

- **📢 섹션 요약 비유**: 트리거는 **'집안의 화재경보기'**와 같습니다. 내가 연기(데이터 변경)를 피우면, 누가 시키지 않아도 경보기가 즉시 감지하여 물을 뿌리거나(데이터 보정) 소방서에 신고(로그 기록)하는 자동화 시스템입니다. 편리하지만, 고기를 굽는 연기(정상 작업)에도 사방에 물을 뿌리면 집안 꼴이 엉망(시스템 성능 저하)이 될 수 있으니 꼭 필요한 곳에만 달아야 합니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Procedural SQL]**: 트리거를 작성하는 언어 (PL/SQL, T-SQL 등).
- **[Autonomous Transaction]**: 트리거 내부에서 메인 트랜잭션과 별개로 커밋하는 고도의 기술.
- **[Constraint]**: 트리거가 보조하거나 대체할 수 있는 데이터 제약 조건.

📢 **마무리 요약**: **Trigger**는 데이터베이스의 파수꾼입니다. 보이지 않는 곳에서 데이터의 정합성을 수호하지만, 그 강력한 힘만큼이나 정교하고 신중한 관리가 필요한 양날의 검입니다.
