# 397. 부분 함수 종속과 제2정규형(2NF) - 키의 완전한 지배

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제2정규형(2NF)은 제1정규형을 만족하면서, 기본키가 복합키(Composite Key)일 때 **기본키의 일부분에만 종속되는 '부분 함수 종속'을 제거하여 모든 속성이 기본키 전체에 종속되게 만드는 과정**이다.
> 2. **가치**: 불필요한 데이터 중복을 방지하여 삽입, 삭제, 갱신 시 발생하는 이상 현상을 제거하고 데이터의 논리적 응집도를 높인다.
> 3. **융합**: '완전 함수 종속(Full Functional Dependency)' 상태를 지향하며, 이는 복잡한 대규모 데이터 모델을 관리 가능한 단위로 쪼개는 정규화의 핵심 관문이다.

+++

### Ⅰ. 부분 함수 종속의 정의와 문제점

- **부분 함수 종속 (Partial Functional Dependency)**: 기본키 $\{A, B\}$ 가 있을 때, 일반 속성 $C$ 가 기본키 전체가 아닌 $\{A\}$ 또는 $\{B\}$ 에만 종속되는 경우입니다.
- **발생 사례**: (학번, 과목코드)가 기본키인데, '학생이름' 속성이 포함된 경우. 이름은 '학번'에만 종속되므로 부분 함수 종속이 발생합니다.
- **문제점**: 학생이 수강 신청을 안 하면 이름을 저장할 수 없고(삽입 이상), 과목을 취소하면 이름까지 지워지는(삭제 이상) 문제가 생깁니다.

+++

### Ⅱ. 2정규화 과정 시각화 (ASCII Model)

부분 함수 종속을 끊어내어 테이블을 분리하는 과정입니다.

```text
[ 1NF (Unnormalized) ]
  ┌────────┬──────────┬──────────┬──────────┐
  │ 학번(K)│ 과목코드(K)│ 학생이름  │ 성적     │
  └────────┴──────────┴────┬─────┴──────────┘
             │             └─▶ [ 부분 함수 종속 ] 💥
             └────────────────▶ [ 완전 함수 종속 ] ✅

[ 2NF (Normalized) ]
  ┌────────┬──────────┐      ┌────────┬──────────┐
  │ 학번(K)│ 학생이름  │      │ 학번(K)│ 과목코드(K)│
  └────────┴──────────┘      ├────────┼──────────┤
                             │ 성적   │          │
                             └────────┴──────────┘
```

+++

### Ⅲ. 제2정규형(2NF)의 성립 조건

1. **제1정규형(1NF)**을 만족해야 합니다 (도메인의 원자성).
2. 기본키가 아닌 모든 속성이 기본키에 대하여 **완전 함수 종속**이어야 합니다.
3. **단일 속성 기본키**인 경우: 이미 완전 함수 종속이므로 자동으로 2NF를 만족할 가능성이 매우 높습니다 (복합키일 때 주로 문제 발생).

- **📢 섹션 요약 비유**: 제2정규형은 **'가족사진에서 이웃집 사람 빼기'**와 같습니다. (학번, 과목)이라는 우리 집 대문(복합키)에 걸린 사진인데, 학생이름(일반 속성)은 학번에만 관련 있고 과목과는 상관없는 사람입니다. 이 사람을 자기 집(학번 테이블)으로 보내주어야 우리 집 사진(테이블)이 명확하고 깔끔해지는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Full Functional Dependency]**: 2NF가 달성된 후의 이상적인 상태.
- **[Anomalies]**: 정규화되지 않았을 때 마주치는 3대 이상 현상.
- **[Decomposition]**: 테이블을 쪼개어 정규형을 만드는 무손실 분해 과정.

📢 **마무리 요약**: **2NF**는 키의 권위를 세우는 과정입니다. 속성이 오직 '전체 키'에만 충성하게 함으로써 데이터의 모호함을 없애고 정합성을 확보합니다.
