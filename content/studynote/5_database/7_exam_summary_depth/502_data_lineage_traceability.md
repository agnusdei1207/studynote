# 502. 데이터 리니지(Data Lineage) - 데이터의 가계도와 무결성

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 리니지는 데이터의 발생 원천부터 가공, 이동, 최종 소비 단계에 이르기까지의 **전체 경로와 변화 과정을 시각화하고 기록하는 추적 기술**이다.
> 2. **가치**: 보고서 수치의 근거를 증명하는 **데이터 신뢰성**을 확보하며, 시스템 변경 시 하류(Downstream)에 미치는 파급 효과를 분석하는 영향도 분석의 핵심 도구가 된다.
> 3. **융합**: 데이터 감사(Audit) 및 디지털 포렌식 기술과 융합되어, 데이터 조작 여부를 감시하고 사고 발생 시 원인 지점(Root Cause)을 즉각 찾아내는 무결성 체계를 구축한다.

+++

### Ⅰ. 데이터 리니지의 주요 관리 대상

- **Upstream (소스)**: 데이터가 어느 시스템, 어느 테이블에서 시작되었는가?
- **Transformation (변환)**: 중간에 어떤 SQL 로직이나 함수에 의해 값이 바뀌었는가?
- **Downstream (타겟)**: 최종적으로 어떤 보고서나 AI 모델에서 사용되고 있는가?
- **Ownership**: 각 단계별 데이터의 책임자와 관리 주체는 누구인가?

### Ⅲ. 무결성 감사를 위한 리니지 활용

1. **역추적 (Back-tracing)**: 통계 숫자가 틀렸을 때, 리니지 지도를 따라가며 어느 단계의 로직이 오류를 범했는지 파악합니다.
2. **영향도 분석 (Impact Analysis)**: 특정 컬럼을 삭제하거나 수정할 때, 이와 연결된 수백 개의 하위 시스템 중 무엇이 망가질지 미리 예측합니다.
3. **규제 준수 (Compliance)**: 데이터의 처리 과정이 투명하게 기록되어 있음을 입증하여, 개인정보 보호법이나 금융 규제에 대응합니다.

- **📢 섹션 요약 비유**: 데이터 리니지는 **'식재료 이력 추적제'**와 같습니다. 우리가 먹는 요리(최종 보고서)가 어느 농장(소스 DB)에서 왔는지, 도축과 가공은 어디서 했는지, 배송은 누가 했는지를 한눈에 보여주는 것입니다. 만약 배탈이 났다면(데이터 오류), 이 지도를 보고 어느 단계에서 식재료가 오염되었는지 즉시 찾아낼 수 있는 아주 똑똑한 시스템입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Root Cause Analysis]**: 장애의 근본 원인을 찾는 활동.
- **[Metadata]**: 리니지를 구성하는 원천 데이터 정보.
- **[Data Catalog]**: 리니지 정보를 포함하여 데이터 자산을 검색하게 돕는 플랫폼.

📢 **마무리 요약**: **Data Lineage**는 데이터의 족보입니다. 거대하고 복잡한 데이터 생태계 속에서 길을 잃지 않고 진실을 끝까지 추적하게 돕는 데이터베이스의 나침반입니다.
