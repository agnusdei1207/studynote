# 550. HTAP(Hybrid Transactional/Analytical Processing) - 운영과 분석의 실시간 통합

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: HTAP은 동일한 데이터베이스 엔진 내에서 **온라인 트랜잭션 처리(OLTP)와 온라인 분석 처리(OLAP)를 동시에 수행**하여, 데이터 이동 지연 없이 즉각적인 비즈니스 통찰을 얻는 하이브리드 아키텍처다.
> 2. **가치**: 운영 DB에서 분석용 DW로 데이터를 옮기는 복잡한 **ETL 과정을 제거**하여 인프라 비용을 절감하며, "방금 발생한 거래"를 1초 내에 분석에 반영하는 실시간 의사결정을 지원한다.
> 3. **융합**: 행 기반 저장소(Row-store)와 열 기반 저장소(Column-store)의 병렬 운영, 또는 인메모리 스냅샷 복제 기술과 융합되어 현대 NewSQL의 핵심 성능 지표가 된다.

+++

### Ⅰ. HTAP의 주요 구현 방식

1. **인메모리 컬럼 복제 (In-memory Columnar Replication)**: 
    - 메모리 상에 행 기반 데이터와 동일한 데이터를 열 기반으로 하나 더 유지하여 분석 쿼리 시에만 사용. (예: Oracle In-Memory)
2. **분산 스토리지 분리**: 
    - 하나의 클러스터 내에서 트랜잭션용 노드(Row)와 분석용 노드(Column)를 나누고 Raft 등으로 실시간 동기화. (예: TiDB TiFlash)
3. **단일 포맷 엔진**: 
    - 특수한 압축 및 인덱싱 기술을 통해 하나의 데이터 포맷으로 두 워크로드를 모두 수용.

### Ⅲ. HTAP 도입의 비즈니스 효과

- **실시간 사기 탐지**: 결제가 일어나는 순간(OLTP)의 데이터를 바탕으로 과거 패턴 분석(OLAP)을 즉시 수행하여 이상 거래를 차단합니다.
- **다이내믹 프라이싱**: 실시간 재고와 판매 추이를 즉각 분석하여 상품 가격을 유동적으로 조정합니다.
- **IT 단순화**: DW를 따로 관리할 필요가 없어 시스템 복잡도와 인력 운영 비용이 획기적으로 줄어듭니다.

- **📢 섹션 요약 비유**: HTAP은 **'오픈 키친이 있는 레스토랑'**과 같습니다. 예전에는 주방(OLTP)에서 요리한 음식을 식당(OLAP)으로 옮겨서 손님에게 내놓는 식이었다면, HTAP은 손님이 요리되는 과정을 실시간으로 보면서 즉석에서 간을 맞추고 피드백을 주는 것과 같습니다. 기다림 없는 신선한 데이터 요리가 가능해집니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Freshness]**: 분석 데이터가 운영 데이터와 얼마나 일치하는지를 나타내는 척도.
- **[ETL-less]**: HTAP의 궁극적인 기술적 지향점.
- **[NewSQL]**: HTAP 아키텍처를 가장 적극적으로 도입하고 있는 차세대 DB 진영.

📢 **마무리 요약**: **HTAP**은 데이터 활용의 패러다임을 바꿉니다. "과거를 돌아보는 분석"에서 "현재를 실시간으로 통제하는 분석"으로 나아가는 기술적 도약입니다.
