# 478. 스노우플레이크 스키마 - 정규화된 차원의 눈꽃

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스노우플레이크 스키마(Snowflake Schema)는 스타 스키마의 **차원 테이블들을 정규화(Normalization)하여 다단계 계층 구조로 쪼개어 놓은 다차원 데이터 모델**이다.
> 2. **가치**: 중복된 차원 데이터를 제거하여 저장 공간을 절약하고 데이터의 무결성을 높이며, 복잡한 계층 관계를 물리적 테이블 구조에 명확히 반영한다.
> 3. **융합**: 고전적인 정규화 이론과 분석용 모델링의 절충안이며, 최근에는 클라우드 DW의 강력한 컴퓨팅 자원 덕분에 복잡한 조인 부하를 견디며 다시 활용도가 높아지고 있다.

+++

### Ⅰ. 스노우플레이크 스키마의 특징

- **정규화된 차원**: 예로 '상품' 테이블에서 '카테고리'와 '브랜드' 정보를 별도 테이블로 분리합니다.
- **조인 복잡도**: 팩트 테이블에서 원하는 정보를 얻기 위해 여러 단계의 조인(`Fact -> Dim Level 1 -> Dim Level 2`)을 거쳐야 합니다.
- **저장 효율**: 중복 데이터가 사라지므로 데이터 모델이 날씬해집니다 (Storage efficiency).

+++

### Ⅱ. 스노우플레이크 구조 시각화 (ASCII Model)

팩트에서 뻗어 나가는 가지가 정규화로 인해 계속 갈라지는 눈결정체 모양입니다.

```text
[ Snowflake Schema Architecture ]

      [ Dim_Category ] (L2)
             ▲
             │
      [ Dim_Product ] (L1)             [ Dim_City ] (L2)
             │                                ▲
             └───────┐        ┌───────────────┘
                     ▼        ▼
               [ Fact_Sales ] (Central)
               - SalesID
               - ProductID (FK)
               - StoreID (FK)
               - Amount, Qty
                     ▲        ▲
             ┌───────┘        └───────────────┐
             │                                │
      [ Dim_Time ] (L1)                [ Dim_Store ] (L1)
             │                                │
             ▼                                ▼
      [ Dim_Year ] (L2)                [ Dim_Region ] (L2)
```

+++

### Ⅲ. 스타 vs 스노우플레이크 비교

| 비교 항목 | 스타 스키마 (Star) | 스노우플레이크 (Snowflake) |
|:---|:---|:---|
| **정규화** | 비정규화 (Denormalized) | **정규화 (Normalized)** |
| **중복도** | 높음 | **낮음 (무결성 유리)** |
| **조회 속도** | **빠름 (Join 적음)** | 상대적으로 느림 (Join 많음) |
| **유지보수** | 어려움 (중복 데이터 갱신) | **쉬움 (부모 테이블만 수정)** |

- **📢 섹션 요약 비유**: 스노우플레이크 스키마는 **'잘 분류된 도서관 서가'**와 같습니다. 스타 스키마가 모든 정보를 책 표지에 다 적어둔 무거운 책이라면, 스노우플레이크는 책에는 번호만 있고, 그 번호를 따라가면 장르실이 나오고, 또 거기서 세부 작가실로 연결되는 정교한 분류 체계입니다. 찾기는 조금 더 걸리지만(Join), 정리는 훨씬 완벽하게 되어 있는 구조입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Normalization]**: 스노우플레이크의 정체성을 규정하는 기술적 행위.
- **[Hierarchy]**: 데이터의 상하 관계를 가장 잘 표현하는 구조.
- **[Storage Cost]**: 온프레미스 시대에 스노우플레이크를 선택하게 했던 핵심 요인.

📢 **마무리 요약**: **Snowflake Schema**는 정교함과 무결성을 중시하는 데이터 설계의 산물입니다. 성능과 용량의 트레이드오프를 넘어서, 데이터의 논리적 질서를 완벽하게 세우는 모델입니다.
