# 520. 데이터 가상화 연합 질의(Federated Query) - 파편화된 데이터의 논리적 결합

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연합 질의(Federated Query)는 데이터 가상화 기술의 핵심 엔진으로, **물리적으로 분리된 여러 이기종 데이터 소스(SQL, NoSQL, API 등)에 대해 단일 SQL 질의로 동시에 접근하여 결과를 합쳐주는 기술**이다.
> 2. **가치**: 데이터를 한곳으로 모으는 무거운 물리적 통합 과정 없이, **실시간으로 흩어진 데이터의 '최신 상태'를 조회**할 수 있게 하여 비즈니스 의사결정의 민첩성을 극대화한다.
> 3. **융합**: 소스별 쿼리 최적화와 푸시다운(Push-down) 연산 기술이 융합되어, 네트워크 전송 부하를 최소화하면서도 대규모 분산 조인(Join) 성능을 확보한다.

+++

### Ⅰ. 연합 질의의 작동 아키텍처

1. **사용자 인터페이스**: 사용자는 표준 SQL로 가상 통합 테이블을 조회합니다.
2. **질의 분해 및 변환**: 가상화 엔진이 쿼리를 각 소스 시스템(Oracle, MongoDB, S3 등)이 이해할 수 있는 개별 쿼리로 쪼개고 변환합니다.
3. **분산 실행**: 각 소스 시스템이 자기 몫의 연산을 수행하고 결과만 엔진으로 보냅니다.
4. **글로벌 조인 및 취합**: 엔진이 각지에서 온 데이터를 메모리 상에서 최종적으로 합쳐(Join/Union) 사용자에게 반환합니다.

+++

### Ⅱ. 연합 질의 처리 흐름 시각화 (ASCII Model)

```text
[ Federated Query Processing Pipeline ]

      (User SQL) : "SELECT * FROM Global_Sales_View"
                │
      ┌─────────▼─────────┐
      │ Federation Engine │ (Parser / Optimizer / Executor)
      │ (Virtual Layer)   │ ✅ "Orchestrates Sub-queries"
      └─────────┬─────────┘
                │
      ┌─────────┴─────────┬──────────────┐
      ▼ (Sub-query 1)     ▼ (Sub-query 2) ▼ (Sub-query 3)
  [ Source A: DB ]    [ Source B: API ]  [ Source C: Cloud ]
 (Compute locally)   (Compute locally)  (Compute locally)
      │                   │              │
      └─────────┬─────────┴──────────────┘
                ▼
      [ Global Join in RAM ] ──▶ ✅ Final Results Delivered
```

+++

### Ⅲ. 실무적 도전 과제: 성능 최적화

- **Data Locality의 부재**: 데이터가 네트워크 너머에 있으므로, 대량의 데이터를 가져와 조인하면 네트워크 병목이 발생합니다.
- **최적화 전략 (Push-down)**: 최대한 많은 필터링과 집계를 소스 시스템 내부에서 끝내도록 명령을 내려보내는 것이 연합 질의 성능의 핵심입니다.
- **이기종 호환성**: 각 소스 시스템의 데이터 타입 차이와 SQL 문법의 미묘한 차이를 완벽히 중계(Mapping)해야 합니다.

- **📢 섹션 요약 비유**: 연합 질의는 **'다국적 화상 회의'**와 같습니다. 각 팀(데이터 소스)은 자기 나라에 그대로 머물러 있지만, 의장(연합 엔진)이 공통 언어(SQL)로 질문을 던지면 각 팀이 자기 나라의 정보(로컬 연산)를 정리해서 답변합니다. 의장은 이 답변들을 모아 종합 보고서(최종 결과)를 만드는 것과 같습니다. 팀원들이 직접 비행기를 타고 모일(ETL) 필요가 없어 아주 빠르고 효율적인 회의 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Push-down Optimization]**: 연합 질의의 성능을 책임지는 핵심 기술.
- **[Zero-copy Integration]**: 물리적 복제 없이 논리적으로만 통합하는 철학.
- **[Presto / Trino]**: 대규모 연합 질의를 수행하는 대표적인 오픈소스 엔진.

📢 **마무리 요약**: **Federated Query**는 데이터의 물리적 경계를 허뭅니다. 파편화된 데이터 사일로를 기술적으로 연결함으로써, 전사적 가시성을 확보하는 가장 민첩하고 강력한 방법입니다.
