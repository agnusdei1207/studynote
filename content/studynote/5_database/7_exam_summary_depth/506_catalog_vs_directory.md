# 506. 데이터 디렉터리 vs 시스템 카탈로그 - 메타데이터의 안과 밖

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 디렉터리와 시스템 카탈로그는 모두 데이터베이스의 메타데이터(Metadata)를 저장하는 공간이지만, **카탈로그는 사용자에게 노출되어 조회가 가능한 공간**이고 **디렉터리는 시스템만 접근할 수 있는 내부 공간**이라는 결정적 차이가 있다.
> 2. **가치**: 카탈로그는 사용자가 스키마와 통계 정보를 파악하는 인터페이스가 되며, 디렉터리는 DBMS 내부의 물리적 주소와 인덱스 매핑을 관리하여 시스템 무결성을 지탱한다.
> 3. **융합**: '데이터 사전(Data Dictionary)'이라는 큰 틀 안에서 논리적 접근(카탈로그)과 물리적 접근(디렉터리)의 역할을 분담하여 효율적인 자원 관리를 실현한다.

+++

### Ⅰ. 두 개념의 명확한 정의 비교

| 비교 항목 | 시스템 카탈로그 (System Catalog) | 데이터 디렉터리 (Data Directory) |
|:---|:---|:---|
| **가시성** | **사용자 접근 가능 (Read-only)** | **시스템만 접근 가능 (Hidden)** |
| **조회 방법** | **표준 SQL 사용** (SELECT 등) | 전용 시스템 콜이나 내부 API |
| **저장 내용** | 스키마, 뷰, 권한, 통계 정보 | 물리적 파일 주소, 블록 매핑 정보 |
| **별칭** | 데이터 사전 (Data Dictionary) | 메타-메타데이터, 시스템 내부 장부 |

+++

### Ⅱ. 메타데이터 계층 시각화 (ASCII Model)

```text
[ Database Metadata Hierarchy ]

  (User / DBA)  ──▶ [ System Catalog (Views) ] ✅ "Visible through SQL"
                           │
  -------------------------│------------------------- (System Boundary)
                           ▼
  [ Database Engine ] ──▶ [ Data Directory ] ✅ "Internal Only"
                           │
  -------------------------│------------------------- (Physical Boundary)
                           ▼
                  [ Physical Data Files ] 💾 (.dbf / .ibd)
```

+++

### Ⅲ. 실무적 의의: 왜 분리하는가?

- **보안성**: 실제 데이터가 디스크의 어느 섹터, 어느 블록에 있는지와 같은 극도의 물리적 보안 정보(디렉터리 내용)는 사용자에게 노출하지 않음으로써 해킹이나 임의 수정을 방지합니다.
- **편의성**: 사용자는 `SELECT * FROM USER_TABLES` 처럼 익숙한 SQL로 시스템 정보를 조회하여 데이터 모델을 이해할 수 있습니다.
- **성능**: 시스템 내부의 복잡한 주소 계산은 최적화된 디렉터리 엔진이 담당하고, 통계 분석이나 쿼리 최적화는 카탈로그 정보를 활용하도록 역할을 분리합니다.

- **📢 섹션 요약 비유**: 시스템 카탈로그와 데이터 디렉터리는 **'도서관의 검색 컴퓨터'와 '사서의 비밀 창고 장부'**의 관계와 같습니다. 검색 컴퓨터(카탈로그)는 누구나 쓸 수 있고 책의 제목과 위치(메타데이터)를 알려주지만, 사서만 볼 수 있는 비밀 장부(디렉터리)에는 책장의 정밀한 일련번호와 수리 내역, 열쇠 번호 같은 내부 관리 정보가 적혀 있는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Active Metadata]**: DBMS가 자동으로 갱신하고 활용하는 살아있는 정보.
- **[Information Schema]**: 카탈로그 정보를 표준화하여 보여주는 공통 뷰 규격.
- **[Physical Independence]**: 디렉터리와 카탈로그의 분리를 통해 달성되는 물리적 데이터 독립성.

📢 **마무리 요약**: **Catalog vs Directory**의 차이를 아는 것이 DB 아키텍처의 깊이를 이해하는 척도입니다. 보이는 곳과 보이지 않는 곳의 조화를 통해 데이터베이스는 거대한 정보를 질서 있게 통제합니다.
