# 554. 트리 구조 매핑 - Nested Set vs Nested Path 비교

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트리 구조 매핑은 관계형 데이터베이스(RDBMS)의 평면적인 테이블에서 계층적 데이터를 효율적으로 조회하고 관리하기 위해, **노드 간의 포함 관계(Nested Set)나 경로(Nested Path)를 수치화/문자열화하여 저장하는 기술**이다.
> 2. **가치**: 재귀 쿼리 없이도 한 번의 쿼리로 전체 하위 트리를 즉시 조회할 수 있는 **압도적인 조회 성능**을 제공하며, 폴더 구조나 카테고리 트리 구현의 핵심 설계 패턴이 된다.
> 3. **융합**: 읽기 성능이 극대화된 **Nested Set**과 쓰기/경로 파악이 쉬운 **Nested Path(Materialized Path)** 모델을 비즈니스 요구사항에 맞춰 선택적으로 융합하여 적용한다.

+++

### Ⅰ. 두 매핑 모델의 상세 비교

| 비교 항목 | Nested Set (중첩 집합) | Nested Path (경로 열거) |
|:---|:---|:---|
| **저장 방식** | 좌/우 번호(Left, Right) 할당 | 전체 경로 문자열 저장 (예: 1/2/5) |
| **조회 (Select)** | **매우 빠름** (Between 연산 활용) | 빠름 (Like 연산 활용) |
| **삽입 (Insert)** | **매우 느림** (전체 번호 재할당 필요) | 상대적으로 빠름 (경로만 추가) |
| **경로 파악** | 어려움 | **매우 쉬움** (문자열에 포함됨) |
| **복잡도** | 구현 및 관리 어려움 | 직관적이고 구현 쉬움 |

+++

### Ⅱ. 모델별 데이터 구조 시각화 (ASCII Model)

```text
[ 1. Nested Set Model ]                [ 2. Nested Path Model ]
(모든 하위 노드를 범위로 포함)            (뿌리부터의 경로를 모두 적음)

      [ Root (1, 10) ]                       [ 1: Root ]
       /          \                           /       \
  [A (2, 5)]  [B (6, 9)]                [1/2: A]    [1/3: B] ✅
   /    \      /    \                    /    \      /    \
[C(3,4)][D]  [E]    [F]              [1/2/4:C] ...  [1/3/7:F] ✅

* Query Sub-tree of A:
  - Set : SELECT * WHERE Left > 2 AND Right < 5;
  - Path: SELECT * WHERE Path LIKE '1/2/%';
```

+++

### Ⅲ. 실무적 선택 가이드

- **Nested Set 선택**: 카테고리 정보처럼 데이터 변경은 거의 없으나, 수천 명의 사용자가 동시에 전체 트리를 빈번하게 조회해야 할 때.
- **Nested Path 선택**: 파일 시스템이나 댓글 계층처럼 새로운 노드가 자주 추가되고, 특정 노드의 위치(Depth)와 경로를 직관적으로 알아야 할 때. (가장 대중적)
- **최신 경향**: 최근 RDBMS의 **Recursive CTE(WITH 절)** 성능이 좋아지면서, 복잡한 매핑 모델 대신 단순한 부모 ID(Adjacency List) 방식과 재귀 쿼리를 조합하는 경우도 늘고 있습니다.

- **📢 섹션 요약 비유**: 트리 매핑은 **'가계부 정리법'**과 같습니다. Nested Set은 가족 전체를 '큰 봉투(번호 범위)' 안에 작은 봉투들을 넣어 보관하는 것이라 한 번에 꺼내기(조회)는 편하지만, 새 식구가 들어오면 모든 봉투의 번호를 다시 써야(수정) 합니다. Nested Path는 각 식구의 명찰에 '누구의 자식의 자식'이라고 긴 주소를 적어두는 것과 같아서, 주소만 읽으면 가문을 금방 알 수 있고 명찰 하나 달아주기(삽입)도 편한 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[L/R Values]**: Nested Set에서 노드의 경계를 나타내는 숫자.
- **[Materialized Path]**: Nested Path의 또 다른 학술적 명칭.
- **[Adjacency List]**: 부모 ID만 저장하는 가장 단순한 트리 모델 (매핑 모델의 대척점).

📢 **마무리 요약**: **Tree Mapping**은 차원을 넘는 설계입니다. 평면적인 테이블에 계층의 입체감을 불어넣음으로써, 복잡하게 얽힌 데이터들 사이에서 질서 정연한 통찰을 이끌어냅니다.
