# 509. 클러스터링 팩터(Clustering Factor) - 인덱스 효율의 진실

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클러스터링 팩터(Clustering Factor)는 인덱스의 리프 노드에 정렬된 키 순서와 **실제 테이블 데이터 블록의 물리적 정렬 순서가 얼마나 일치하는지를 나타내는 수치적 지표**다.
> 2. **가치**: 인덱스를 통해 테이블을 읽을 때 발생하는 **랜덤 I/O 부하를 정량적으로 예측**하게 해주며, 옵티마이저가 인덱스 스캔과 테이블 풀 스캔 중 무엇이 더 유리한지 판단하는 결정적 근거가 된다.
> 3. **융합**: 데이터 재정렬(Reorg) 기술과 융합되어, 나빠진 클러스터링 팩터를 개선함으로써 쿼리 성능을 비약적으로 향상시키는 DB 튜닝의 핵심 지표로 활용된다.

+++

### Ⅰ. 클러스터링 팩터의 측정 원리

DBMS는 인덱스를 처음부터 끝까지 스캔하면서 다음과 같이 계산합니다.
- **계산 방식**: 인덱스 리프 노드를 따라가며 가리키는 테이블 블록의 주소(ROWID)가 **직전과 다른 블록으로 바뀔 때마다 카운트를 1씩 증가**시킵니다.
- **결과값의 범위**:
    - **최상 (Good)**: 테이블의 총 블록 수에 가까움. (인덱스 순서와 물리적 순서가 일치)
    - **최악 (Bad)**: 테이블의 총 레코드 수에 가까움. (인덱스 순서와 물리적 순서가 무질서하게 흩어짐)

+++

### Ⅱ. 정렬 상태에 따른 팩터 차이 시각화 (ASCII Model)

```text
[ Case 1: Good Clustering Factor ]     [ Case 2: Bad Clustering Factor ]
  (Index Order: A, B, C, D)              (Index Order: A, B, C, D)

  Index: [A][B] [C][D]                   Index: [A][B] [C][D]
          │  │   │  │                            │  │   │  │
          ▼  ▼   ▼  ▼                            ▼  ▼   ▼  ▼
  Table: [A, B] [C, D] (Block 1, 2)      Table: [A, C] [B, D] (Block 1, 2)
  
  * Read A, B: Block 1 만 읽음 (I/O: 1)   * Read A, B: Block 1 읽고 2로 점프 (I/O: 2) ✅
  * I/O 효율성: 매우 높음 (Sequential)    * I/O 효율성: 매우 낮음 (Random) ✅
```

+++

### Ⅲ. 실무적 튜닝 포인트

- **인덱스 스캔 결정**: 데이터가 아무리 잘 걸러져도(Selectivity가 좋아도), 클러스터링 팩터가 너무 나쁘면 옵티마이저는 인덱스 스캔을 포기하고 풀 스캔(FTS)을 선택합니다.
- **개선 방법**: 테이블을 해당 인덱스 키 순서대로 다시 생성(`CTAS`)하거나, 주기적으로 테이블 재구성(Reorganization) 작업을 수행하여 물리적 정렬도를 높입니다.
- **영향도**: 클러스터링 팩터가 개선되면 동일한 쿼리의 I/O 횟수가 1/10 이하로 줄어들기도 합니다.

- **📢 섹션 요약 비유**: 클러스터링 팩터는 **'마트 장보기 리스트와 진열 순서의 일치도'**와 같습니다. 내 리스트(인덱스)가 우유, 치즈, 버터 순인데 마침 진열대(테이블 블록)에도 유제품들이 모여 있다면 한 곳에서 다 집을 수 있어 편합니다(Good CF). 하지만 우유는 1층, 치즈는 5층, 버터는 지하에 있다면 매번 층을 옮겨 다녀야 하므로(Bad CF) 엄청나게 힘이 드는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[CBO (Cost-Based Optimizer)]**: 클러스터링 팩터 값을 비용 계산의 핵심 변수로 사용.
- **[Random Access]**: 클러스터링 팩터가 나쁠 때 발생하는 성능 저하의 주범.
- **[Clustered Index]**: 클러스터링 팩터를 강제로 최상(테이블 블록 수)으로 고정하는 인덱스.

📢 **마무리 요약**: **Clustering Factor**는 인덱스의 숨겨진 성적표입니다. 단순한 인덱스 생성보다 중요한 것은, 실제 데이터가 그 인덱스를 얼마나 기쁘게 받아들일 수 있는 '정렬된 상태'인가를 관리하는 것입니다.
