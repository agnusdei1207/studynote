# 569. 카산드라 쓰기 경로(Write Path) - 지연 없는 데이터 적재의 기술

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 카산드라의 쓰기 경로는 데이터를 **메모리에 정렬하여 쌓고(Memtable), 동시에 디스크에 순차 로그(Commit Log)를 남긴 뒤 나중에 불변 파일(SSTable)로 내려쓰는** LSM-Tree 기반의 고속 쓰기 프로세스다.
> 2. **가치**: 디스크의 랜덤 I/O를 완전히 제거하고 모든 쓰기를 **순차 I/O(Sequential I/O)**로 처리하여, 데이터 규모가 커져도 쓰기 성능이 저하되지 않는 압도적인 처리량(Throughput)을 제공한다.
> 3. **융합**: 복제 계수(RF)와 정족수(Quorum) 합의 기술이 쓰기 경로와 융합되어, 네트워크 지연 속에서도 정합성과 가용성의 균형을 실시간으로 조율한다.

+++

### Ⅰ. 카산드라 쓰기 3단계 아키텍처

1. **Commit Log (디스크 기록)**: 
    - 장애 복구를 위한 생명줄입니다. 디스크의 특정 위치를 찾지 않고 파일 끝에 무조건 덧붙여(Append-only) 쓰므로 매우 빠릅니다.
2. **Memtable (메모리 기록)**: 
    - 데이터를 메모리에 정렬된 상태로 보관합니다. 사용자는 이 시점에 바로 "쓰기 성공" 응답을 받습니다. ✅
3. **SSTable (최종 저장)**: 
    - 멤테이블이 꽉 차면 디스크에 **불변(Immutable)** 파일 형태로 한꺼번에 내려씁니다. 이후에는 절대 수정되지 않으며 오직 읽기만 가능합니다.

+++

### Ⅱ. 쓰기 경로 시각화 (ASCII Model)

```text
[ Cassandra High-speed Write Path ]

  (Write Request) ──▶ [ Node Coordinator ]
                             │
          ┌──────────────────┴──────────────────┐
          ▼                                     ▼
  [ 1. Commit Log ] 💾 (Disk)           [ 2. Memtable ] 🧠 (RAM) ✅
  (Sequential Append)                   (Sorted Store)
          │                                     │
          │                                     ▼ (Flush when full)
          │                             [ 3. SSTable ] 💾 (Disk) ✅
          └──────────────────┬──────────────────┘
                             ▼
                    "SUCCESS Answer to Client"
```

+++

### Ⅲ. 병목 배제 모델의 핵심 원리

- **No Overwrite**: 기존 데이터를 찾아가서 고치는 작업이 아예 없습니다. 수정이나 삭제조차 새로운 데이터를 덧붙이는 행위(Upsert/Tombstone)로 처리하여 디스크 헤더의 이동을 최소화합니다.
- **Lock-free**: 행 단위 잠금을 사용하지 않고 메모리 버퍼링을 활용하므로 트랜잭션 경합에 따른 병목이 발생하지 않습니다.
- **Compaction**: 나중에 백그라운드에서 여러 SSTable을 합치며 중복과 삭제된 데이터를 정리하여 읽기 성능까지 보완합니다.

- **📢 섹션 요약 비유**: 카산드라의 쓰기 방식은 **'속기사의 메모'**와 같습니다. 취재 현장(데이터 유입)에서 내용을 예쁘게 책(데이터 파일)으로 엮으려다가는 흐름을 놓칩니다. 대신, 들리는 대로 수첩(Commit Log)에 갈겨쓰고 동시에 기억(Memtable)해 둡니다. 상황이 종료되면(Flush) 수첩의 내용을 깔끔하게 정리해 책(SSTable)으로 묶어 내놓는 식입니다. 덕분에 어떤 빠른 대화(대량 트래픽)도 놓치지 않고 다 받아적을 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[LSM-Tree]**: 로그 구조 머지 트리의 실전 구현체.
- **[Flush]**: 메모리(Memtable)를 디스크(SSTable)로 옮기는 결정적 순간.
- **[Tombstone]**: 삭제를 '삭제했다는 기록'으로 처리하는 쓰기 방식.

📢 **마무리 요약**: **Cassandra Write Path**는 속도에 최적화된 공학적 설계의 정수입니다. "쓰기는 저렴하고 읽기는 비싸다"는 철학 아래, 전 세계 수조 건의 데이터를 가장 빠르게 담아내는 그릇이 되었습니다.
