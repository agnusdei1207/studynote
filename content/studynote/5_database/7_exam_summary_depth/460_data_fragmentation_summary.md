# 460. 데이터 단편화(Fragmentation) - 분산 배치의 기본 기법

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 단편화는 전역 데이터베이스의 논리적 릴레이션을 **행(튜플) 단위의 수평 분할 또는 열(속성) 단위의 수직 분할을 통해 여러 개의 작은 물리적 단위(단편)로 나누는 기술**이다.
> 2. **가치**: 데이터 로컬리티(Data Locality)를 향상시켜 네트워크 전송 부하를 줄이며, 자주 쓰이는 속성이나 특정 지역 데이터를 해당 노드에 전진 배치하여 응답 속도를 최적화한다.
> 3. **융합**: 완전성, 재구성, 상호 배제의 3대 설계 원칙 하에 구현되며, 현대 분산 시스템의 샤딩(Sharding) 및 마이크로서비스 데이터 격리의 이론적 근간이 된다.

+++

### Ⅰ. 단편화의 두 가지 핵심 유형

#### 1. 수평 분할 (Horizontal Fragmentation)
- **정의**: 테이블의 행(Row)들을 특정 조건(예: 지역='서울')에 따라 나누는 방식.
- **특징**: 모든 단편은 원본과 동일한 스키마를 가짐. SQL의 **Select($\sigma$)** 연산과 대응.
- **복구**: 단편들을 **Union($\cup$)** 하여 복구.

#### 2. 수직 분할 (Vertical Fragmentation)
- **정의**: 테이블의 열(Column)들을 사용 빈도나 보안 중요도에 따라 나누는 방식.
- **특징**: 각 행을 식별하기 위해 **기본키(PK)를 모든 단편에 중복 포함**해야 함. SQL의 **Project($\pi$)** 연산과 대응.
- **복구**: 단편들을 **Join($\bowtie$)** 하여 복구.

### Ⅱ. 단편화 구조 시각화 (ASCII Model)

```text
[ Original Table: Employee ]
┌────┬──────┬────────┬───────┐
│ ID │ Name │ Dept   │ Salary│
└────┴──────┴────────┴───────┘

  1. 수평 분할 (By Dept)         2. 수직 분할 (By Privacy)
  ┌────┬──────┬────────┐        ┌────┬──────┬────────┐ ┌────┬───────┐
  │ ID │ Name │ Dept   │        │ ID │ Name │ Dept   │ │ ID │ Salary│
  ├────┼──────┼────────┤        └────┴──────┴────────┘ └────┴───────┘
  │ 1  │ Kim  │ Sales  │ (Frag1)   (Public Data)        (Private Data)
  ├────┼──────┼────────┤         * ID(PK)는 양쪽 모두 포함 ✅
  │ 2  │ Lee  │ IT     │ (Frag2)
  └────┴──────┴────────┘
```

+++

### Ⅲ. 단편화 설계의 3대 원칙

1. **완전성 (Completeness)**: 원본의 모든 데이터는 반드시 최소한 하나의 단편에 속해야 합니다. (유실 금지)
2. **재구성 (Reconstructibility)**: 쪼개진 단편들을 합쳤을 때 원래의 모습으로 완벽히 복원 가능해야 합니다.
3. **상호 배제 (Disjointness)**: 하나의 데이터는 원칙적으로 단 하나의 단편에만 존재해야 합니다. (수직 분할의 PK 중복은 예외)

- **📢 섹션 요약 비유**: 데이터 단편화는 **'커다란 피자를 나누는 법'**과 같습니다. 수평 분할은 피자를 조각내어 여러 접시(노드)에 담는 것이고, 수직 분할은 피자의 토핑(속성)만 따로 걷어내어 다른 그릇에 담는 것과 같습니다. 어떻게 나누든 나중에 다시 합치면 원래의 피자 한 판(전역 데이터)이 그대로 복원될 수 있어야 한다는 약속입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Fragmentation Transparency]**: 데이터가 쪼개져 있다는 사실을 사용자가 모르게 하는 것.
- **[Hybrid Fragmentation]**: 수평과 수직을 섞어서 사용하는 방식.
- **[Data Locality]**: 연산이 일어나는 곳 근처에 데이터를 두어 효율을 높이는 가치.

📢 **마무리 요약**: **Fragmentation**은 분산 데이터베이스의 물리적 설계의 정수입니다. 비즈니스 쿼리 패턴에 맞게 데이터를 정교하게 쪼개는 능력이 고성능 분산 시스템의 성패를 가릅니다.
