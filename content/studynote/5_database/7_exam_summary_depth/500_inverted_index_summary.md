# 500. 역색인(Inverted Index) - 검색 속도의 혁명

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 역색인은 데이터를 문서(Document)별로 저장하는 일반적인 방식과 달리, **단어(Term)를 기준으로 해당 단어가 포함된 문서 번호들의 리스트를 매핑**하여 보관하는 검색 전용 자료구조다.
> 2. **가치**: 수억 건의 문서 속에서도 특정 키워드를 포함한 대상을 순식간에 찾아낼 수 있으며, 데이터량이 늘어나도 조회 성능 저하가 거의 없는 **Full-text Search**의 핵심 알고리즘이다.
> 3. **융합**: Elasticsearch, Lucene 등의 전문 검색 엔진의 심장이며, 형태소 분석(Tokenizing) 및 정규화 기술과 융합되어 자연어 처리 기반의 지능형 검색 시스템을 가능케 한다.

+++

### Ⅰ. 정방향 색인 vs 역색인 비교

- **정방향 색인 (Forward Index)**: "문서 1에는 어떤 단어들이 있는가?" (예: 책의 목차)
- **역색인 (Inverted Index)**: "이 단어는 어떤 문서들에 들어있는가?" (예: 책 뒷면의 찾아보기 색인)

+++

### Ⅱ. 역색인 구조 및 검색 원리 시각화 (ASCII Model)

```text
[ Step 1: Raw Documents ]
  Doc 1: "Database is good"
  Doc 2: "AI is power"
  Doc 3: "Database and AI"

[ Step 2: Inverted Index Table ]
  (Term)      (Posting List: Document IDs)
  ┌──────────┬──────────────┐
  │ AI       │ Doc 2, Doc 3 │
  ├──────────┼──────────────┤
  │ Database │ Doc 1, Doc 3 │
  ├──────────┼──────────────┤
  │ Power    │ Doc 2        │
  └──────────┴──────────────┘

  * Query "Database" ──▶ [Doc 1, Doc 3] 즉시 반환! ✅
```

+++

### Ⅲ. 역색인의 주요 장점

1. **초고속 전문 검색**: `LIKE '%keyword%'` 쿼리는 모든 행을 다 읽어야 하지만(Full Scan), 역색인은 단어 사전만 조회하면 결과가 바로 나옵니다.
2. **복합 키워드 연산**: 여러 단어를 조합한 검색(`AI AND Database`) 시, 각 단어의 문서 리스트를 가져와 집합 연산(교집합)만 수행하면 되므로 매우 효율적입니다.
3. **랭킹 및 가중치**: 단어가 문서에서 나타나는 빈도(TF)와 희귀도(IDF)를 함께 저장하여 검색 결과의 정확도 순위를 매길 수 있습니다.

- **📢 섹션 요약 비유**: 역색인은 **'도서관의 인덱스 카드'**와 같습니다. 내가 '세종대왕'에 대한 책을 찾고 싶을 때 수만 권의 책을 처음부터 끝까지 읽어보는 대신(정방향), '세종대왕'이라고 적힌 인덱스 카드를 찾아 거기 적힌 책 번호들만 보고 서가로 달려가는 것과 같습니다. 수고는 1/1000로 줄고 속도는 1000배 빨라지는 마법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Tokenizing]**: 문장을 단어 단위로 쪼개는 역색인의 전처리 과정.
- **[Posting List]**: 특정 단어가 포함된 문서 ID들의 묶음.
- **[Lucene]**: 역색인 구조를 가장 완벽하게 구현한 오픈소스 라이브러리.

📢 **마무리 요약**: **Inverted Index**는 검색 엔진의 존재 이유입니다. 데이터의 저장을 넘어 '탐색의 효율'을 극한으로 끌어올린 이 구조가 있었기에 오늘날의 초고속 정보 검색 시대가 가능해졌습니다.
