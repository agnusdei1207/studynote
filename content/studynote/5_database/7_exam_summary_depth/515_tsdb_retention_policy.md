# 515. 시계열 DB 보존 정책(Retention) - 데이터의 수명 관리

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 보존 정책(Retention Policy)은 시간의 흐름에 따라 기하급수적으로 늘어나는 시계열 데이터를 **특정 기간(예: 30일)이 지나면 자동으로 삭제하거나 압축하여 보관하는 데이터 수명 관리 전략**이다.
> 2. **가치**: 무한한 데이터 유입에 따른 **스토리지 비용 폭증을 방어**하며, 불필요한 과거 데이터를 제거함으로써 최신 데이터에 대한 쿼리 성능을 최상으로 유지한다.
> 3. **융합**: 데이터 계층화(Tiering) 기술과 결합되어, 최신 데이터는 고속 메모리에, 오래된 데이터는 저렴한 콜드 스토리지로 자동 이동시키는 지능형 생명주기를 완성한다.

+++

### Ⅰ. 보존 정책의 작동 매커니즘

- **시간 창 (Time Window)**: 데이터를 유지할 기간을 정의합니다 (예: 1 Week, 1 Year).
- **자동 폐기 (Auto-drop)**: 설정된 기간이 지난 데이터 블록(Shard)을 DB 엔진이 백그라운드에서 물리적으로 삭제합니다.
- **다운샘플링 연계**: 원본 데이터는 짧게 보존(7일)하고, 요약된 통계 데이터는 길게 보존(1년)하는 전략을 주로 사용합니다.

+++

### Ⅱ. 데이터 라이프사이클 시각화 (ASCII Model)

시간이 지날수록 데이터의 정밀도는 낮아지고 저장 비용은 저렴해집니다.

```text
[ Time Series Data Lifecycle ]

  (Ingestion) ──▶ [ Hot Tier ] ──▶ [ Warm Tier ] ──▶ [ Cold Tier / Drop ]
  ┌───────────┐   ┌────────────┐   ┌────────────┐   ┌────────────────┐
  │ Real-time │   │ SSD Store  │   │ Object Str │   │ DELETE / ARCH  │ ✅
  │ 1s Res.   │   │ Ret: 7 Days│   │ Ret: 90 Days│   │ Over 90 Days   │
  └─────┬─────┘   └─────┬──────┘   └─────┬──────┘   └────────────────┘
        │               │                │
        └─────── (Downsampling) ─────────┘ ✅ "Reduce Resolution"
```

+++

### Ⅲ. 실무적 고려사항

1. **법적 규제**: 금융 거래나 로그 기록은 법적으로 최소 보존 기간(보통 6개월~5년)이 정해져 있으므로 이를 준수해야 합니다.
2. **분석 가치**: 데이터가 오래될수록 개별 수치보다는 '추세(Trend)'가 중요해지므로, 효율적인 압축 및 요약 전략이 병행되어야 합니다.
3. **삭제 부하**: 대량의 데이터를 한꺼번에 지울 때 발생하는 I/O 부하를 방지하기 위해 파티션 단위의 삭제(Drop Partition) 기술을 활용합니다.

- **📢 섹션 요약 비유**: 시계열 DB 보존 정책은 **'CCTV 영상 관리'**와 같습니다. 어제 영상은 사고 확인을 위해 고화질로 꼼꼼히 보관하지만(Hot), 한 달 전 영상은 중요한 장면만 캡처해서 남기고(Warm), 일 년이 지난 영상은 용량 확보를 위해 아예 지워버리는(Cold/Drop) 것과 같습니다. 무조건 쌓아두기만 하면 금방 저장 공간이 부족해져서 정작 중요한 최신 영상을 찍지 못하게 되는 법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[TTL (Time To Live)]**: 각 레코드나 블록의 유효 수명.
- **[Downsampling]**: 해상도를 낮추어 용량을 아끼는 기술.
- **[Continuous Query]**: 보존 정책에 따라 데이터를 삭제하기 전 미리 요약본을 만드는 자동화 쿼리.

📢 **마무리 요약**: **Retention Policy**는 데이터의 선순환을 돕는 '비우기의 미학'입니다. 가치 있는 데이터만 남기고 과감히 비울 때, 데이터베이스는 비로소 지치지 않고 달릴 수 있습니다.
