# 501. 스토리지와 컴퓨팅의 분리 - 클라우드 네이티브의 진수

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스토리지와 컴퓨팅의 분리는 데이터가 저장되는 **물리적 디스크 계층(Storage)과 쿼리를 처리하는 연산 계층(Compute)을 독립적인 자원으로 나누어 관리하는 아키텍처**다.
> 2. **가치**: 데이터량과 쿼리 복잡도에 따라 각 자원을 개별적으로 확장(Scale-out)할 수 있어 **비용 효율성과 탄력성**을 극대화하며, 리소스 경합 없는 동시 분석 환경을 제공한다.
> 3. **융합**: 클라우드 기반 DW(Snowflake, BigQuery)의 핵심 설계 원칙이며, '서비스형 데이터베이스(DBaaS)' 가치를 실현하는 현대적 분산 시스템의 표준이다.

+++

### Ⅰ. 스토리지-컴퓨팅 분리 아키텍처의 특징

- **독립적 확장 (Independent Scaling)**: 데이터가 늘어나면 스토리지만 늘리고, 분석 요청이 몰리면 컴퓨팅 노드만 즉시 추가할 수 있습니다.
- **상태 비저장 컴퓨팅 (Stateless Compute)**: 연산 노드는 로컬 데이터를 가지지 않으므로, 장애 발생 시 즉시 새 노드로 교체하여 작업을 재개할 수 있습니다.
- **데이터 공유 용이성**: 모든 컴퓨팅 노드가 하나의 거대한 공유 스토리지 레이어를 바라보므로 데이터 복제 없이 조인이 가능합니다.

+++

### Ⅱ. 아키텍처 비교 시각화 (ASCII Model)

```text
[ Traditional (Coupled) ]             [ Cloud Native (Decoupled) ]

  ┌───────────────┐                    ┌─────────────────────────┐
  │ Compute + Str │ (Fixed)            │ Compute Nodes (Elastic) │ ✅
  └───────────────┘                    └───────────┬─────────────┘
          │                                        │ (Network)
  ┌───────────────┐                    ┌───────────▼─────────────┐
  │ Compute + Str │                    │ Shared Storage Layer    │ ✅
  └───────────────┘                    └─────────────────────────┘
  (확장 시 데이터 이동 필요)               (자원 간 독립적 확장 가능)
```

+++

### Ⅲ. 실무적 장점: 워크로드 격리

- **분석과 운영의 조화**: 동일한 데이터 스토리지에 대해, 마케팅팀(컴퓨팅 A)과 개발팀(컴퓨팅 B)이 각각 독립된 연산 자원을 사용함으로써 서로의 성능에 영향을 주지 않고 분석할 수 있습니다.
- **비용 최적화**: 쿼리를 날리지 않는 시간 동안에는 컴퓨팅 자원을 0으로 줄여 비용을 획기적으로 절감할 수 있습니다 (Serverless 특성).

- **📢 섹션 요약 비유**: 이 분리 기술은 **'식당의 주방과 창고 분리'**와 같습니다. 예전에는 요리사(컴퓨팅)가 자기 방에 재료(스토리)를 다 들고 있어야 해서 요리사를 더 뽑으려면 방(서버)도 더 지어야 했습니다. 하지만 이제는 거대한 중앙 냉장고(공유 스토리지)를 따로 두고, 요리사들은 필요할 때만 와서 요리하는 방식입니다. 손님이 많으면 요리사만 더 부르면 되고, 재료가 많아지면 냉장고만 더 사면 되는 아주 유연한 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Multi-cluster Shared Data]**: 스노우플레이크의 핵심 아키텍처 명칭.
- **[Object Storage]**: 분리된 스토리지 계층으로 주로 쓰이는 클라우드 서비스(S3 등).
- **[Elasticity]**: 분리를 통해 얻게 되는 최고의 가치인 '탄력성'.

📢 **마무리 요약**: **Separation of Storage & Compute**는 클라우드 데이터베이스의 혁명입니다. 물리적 제약을 논리적 유연성으로 극복하여 데이터 활용의 무한한 가능성을 열어줍니다.
