# 400. 다치 종속(MVD)과 제4정규형(4NF) - 관계의 독립성 확보

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제4정규형(4NF)은 BCNF를 만족하면서, 하나의 키에 대해 서로 독립적인 두 개 이상의 속성이 여러 값을 가지는 **'다치 종속(Multi-Valued Dependency)'을 제거하여 데이터 중복을 방지하는 과정**이다.
> 2. **가치**: 관계가 없는 정보들을 억지로 하나의 테이블에 묶음으로써 발생하는 기하급수적인 데이터 증식과 갱신 이상을 해결하여 저장 효율성을 극대화한다.
> 3. **융합**: "서로 관련 없는 다중 값들은 각자의 테이블로 가야 한다"는 원칙을 통해, 논리적으로 분리된 비즈니스 개체들을 물리적으로도 독립시킨다.

+++

### Ⅰ. 다치 종속 (MVD)의 정의와 문제점

- **다치 종속 (Multi-Valued Dependency)**: 속성 $A$ 가 $B$ 의 집합을 결정하고, 동시에 $A$ 가 $C$ 의 집합을 결정하는데, $B$ 와 $C$ 가 서로 아무런 관련이 없는 경우입니다 ($A \twoheadrightarrow B \mid C$).
- **발생 사례**: (교수) ──▶▶ (담당과목), (교수) ──▶▶ (취미). 교수가 가르치는 과목과 교수의 취미는 서로 무관하지만, 하나의 테이블에 두면 모든 조합(Cartesian Product)을 저장해야 합니다.
- **문제점**: 교수가 새 취미를 하나 가지면, 그 교수가 가르치는 모든 과목 수만큼 행을 추가해야 하는 '폭발적 중복'이 발생합니다.

+++

### Ⅱ. 4정규화 과정 시각화 (ASCII Model)

서로 무관한 다중 값들의 결합을 해체합니다.

```text
[ BCNF (with MVD) ]
  ┌────────┬──────────┬──────────┐
  │ 교수(K) │ 과목(K)   │ 취미(K)   │
  ├────────┼──────────┼──────────┤
  │ 김교수  │ DB       │ 골프      │
  │ 김교수  │ AI       │ 골프      │
  │ 김교수  │ DB       │ 등산      │ <-- 중복 발생 시작!
  │ 김교수  │ AI       │ 등산      │
  └────────┴──────────┴──────────┘

[ 4NF (Normalized) ]
  ┌────────┬──────────┐      ┌────────┬──────────┐
  │ 교수(K) │ 과목(K)   │      │ 교수(K) │ 취미(K)   │
  ├────────┼──────────┤      ├────────┼──────────┤
  │ 김교수  │ DB       │      │ 김교수  │ 골프      │
  │ 김교수  │ AI       │      │ 김교수  │ 등산      │
  └────────┴──────────┘      └────────┴──────────┘
```

+++

### Ⅲ. 제4정규형(4NF)의 성립 조건

1. **BCNF**를 만족해야 합니다.
2. 릴레이션 내에 의미 있는 **다치 종속(MVD)이 없어야** 합니다.
3. 즉, 모든 다치 종속 $A \twoheadrightarrow B$ 가 존재할 때, $A$ 가 슈퍼키(Super Key)여야 합니다.

- **📢 섹션 요약 비유**: 제4정규형은 **'취미 동호회와 전공 스터디 분리하기'**와 같습니다. 김교수님이 가르치는 과목 리스트와 좋아하는 취미 리스트는 서로 아무 상관이 없습니다. 이 둘을 한 종이에 적으려다 보니 "등산을 좋아하는 김교수의 DB 수업", "골프를 좋아하는 김교수의 DB 수업" 식으로 중복된 내용이 계속 생기는 것입니다. 두 리스트를 별도의 종이(테이블)로 나누면 중복 없이 깔끔하게 관리할 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Multi-Valued Dependency]**: 4NF가 타파하고자 하는 중복의 근원.
- **[Trivial MVD]**: 전체 속성 집합 내에서 당연히 성립하는 다치 종속 (무시 가능).
- **[Independence]**: 데이터 간의 논리적 무관함을 물리적으로 보장하는 가치.

📢 **마무리 요약**: **4NF**는 복잡하게 얽힌 관계를 풀어헤치는 과정입니다. 서로 상관없는 정보들을 독립시킴으로써 데이터베이스의 유연성과 성능을 완성합니다.
