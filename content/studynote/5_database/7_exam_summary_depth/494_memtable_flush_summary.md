# 494. 멤테이블(MemTable) - 고속 쓰기의 인메모리 관문

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멤테이블(MemTable)은 LSM 트리 구조의 저장 엔진에서 사용자가 입력한 데이터를 디스크에 기록하기 전, **메모리 상에 정렬된 상태로 임시 보관하는 쓰기 버퍼(Buffer)**다.
> 2. **가치**: 디스크 I/O를 기다리지 않고 메모리 속도로 데이터를 즉시 수용하여 **쓰기 응답 시간을 최소화**하며, 일정 크기가 되면 디스크로 일괄 플러시(Flush)하여 효율적인 저장 구조를 만든다.
> 3. **융합**: 장애 복구를 위한 쓰기 로그(WAL) 및 디스크 저장 포맷인 SSTable과 융합되어, 메모리의 휘발성과 디스크의 속도 한계를 동시에 극복하는 분산 저장소의 핵심 계층이 된다.

+++

### Ⅰ. 멤테이블의 작동 원리

- **정렬 유지**: 데이터가 들어올 때마다 메모리 내에서 정렬된 상태(Skip List, Red-Black Tree 등 활용)를 유지합니다. 덕분에 메모리 내 검색과 범위 조회가 매우 빠릅니다.
- **가변성 (Mutable)**: 디스크의 SSTable과 달리 멤테이블은 내용을 수정하거나 추가하는 것이 자유로운 공간입니다.
- **임계치 관리**: 멤테이블의 크기가 미리 정해진 임계치(예: 64MB)에 도달하면, 쓰기를 중단하고 새로운 멤테이블을 생성한 뒤 기존 멤테이블은 **Immutable MemTable**로 전환하여 플러시를 준비합니다.

+++

### Ⅱ. 멤테이블 플러시(Flush) 프로세스 시각화 (ASCII Model)

메모리에서 디스크로 데이터가 이동하며 '불변성'을 얻는 과정입니다.

```text
[ MemTable to SSTable Flush ]

  (Write Traffic) ──▶ [ Active MemTable (Mutable) ] ── (Sort in RAM) ✅
                             │
                             ▼ (When Full)
                      [ Immutable MemTable ] 🔒
                             │
                             ▼ [ Flush Process ] (Sequential Write) 💾
                      ┌──────────────────────┐
                      │   SSTable File       │ ✅ "New Data File on Disk"
                      └──────────────────────┘
```

+++

### Ⅲ. 실무적 의의: 쓰기 성능의 마법

- **Batching I/O**: 수만 개의 작은 쓰기 요청을 모아서 한 번의 큰 연속적인 디스크 쓰기로 바꿈으로써 하드웨어 성능을 극대화합니다.
- **조회 효율**: 최근에 쓴 데이터는 메모리(MemTable)에 있을 확률이 높으므로, 최신 정보를 빠르게 읽어오는 효과(Cache-like)가 있습니다.

- **📢 섹션 요약 비유**: 멤테이블은 **'주방의 임시 조리대'**와 같습니다. 손님이 주문(데이터)을 할 때마다 매번 멀리 있는 대형 냉장고(디스크)를 열어 정리하는 게 아니라, 일단 조리대 위에 재료를 정렬해서 차곡차곡 쌓아둡니다. 조리대가 꽉 차면(임계치), 쌓인 재료들을 가지런히 정리해서 상자(SSTable)에 담아 창고에 한꺼번에 넣는 효율적인 작업 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Skip List]**: 멤테이블에서 주로 사용하는 정렬된 메모리 자료구조.
- **[WAL (Write-Ahead Log)]**: 멤테이블의 휘발성을 보완하기 위해 먼저 남기는 로그.
- **[Immutable MemTable]**: 플러시를 위해 대기 중인 읽기 전용 멤테이블.

📢 **마무리 요약**: **MemTable**은 데이터베이스의 현관입니다. 무질서한 데이터 유입을 정렬된 질서로 바꾸어 고속 성장의 기반을 닦는 인메모리 기술의 정수입니다.
