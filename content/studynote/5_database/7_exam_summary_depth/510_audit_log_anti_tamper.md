# 510. 데이터 감사 로그 위변조 방지 - 증적 무결성의 기술

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 감사 로그 위변조 방지는 데이터베이스 내에서 발생한 행위 기록이 **악의적인 관리자나 해커에 의해 사후에 수정되거나 삭제되지 않도록 보호하는 무결성 강화 기술**이다.
> 2. **가치**: 보안 사고 발생 시 법적 효력을 갖는 **증거 능력(Admissibility)**과 **부인 방지(Non-repudiation)**를 보장하며, 내부 통제의 투명성을 극대화하여 시스템의 신뢰를 유지한다.
> 3. **융합**: WORM(Write Once Read Many) 스토리지, 디지털 서명, 블록체인 해시 체이닝 기술이 융합되어, 로그가 생성되는 즉시 박제되는 '불변의 기록 보관소'를 구현한다.

+++

### Ⅰ. 감사 로그 보호의 핵심 기술

1. **WORM (Write Once Read Many)**: 한 번 기록되면 물리적으로 지우거나 고칠 수 없는 광학 매체나 특수 클라우드 스토리지(Object Lock)를 사용합니다.
2. **디지털 서명 (Digital Signature)**: 각 로그 엔트리에 대해 관리자의 키나 시스템 키로 서명을 수행하여, 1비트만 바뀌어도 즉시 발각되게 합니다.
3. **해시 체이닝 (Hash Chaining)**: 이전 로그의 해시값을 다음 로그가 포함하게 하여, 중간 로그를 삭제하면 전체 체인이 깨지도록 설계합니다 (블록체인 원리).
4. **원격 실시간 전송**: 로그가 생성되자마자 DB 서버 외부의 독립된 보안 서버(SIEM)로 전송하여 본체 탈취 시의 위험을 분산합니다.

+++

### Ⅱ. 위변조 방지 아키텍처 시각화 (ASCII Model)

```text
[ Tamper-proof Audit Logging ]

  [ Database Engine ] ──▶ [ Audit Agent ] ──▶ [ Hashing & Signing ] ✅
          │                                          │
          ▼                                          ▼
  ┌───────────────┐                          ┌──────────────────────┐
  │ Local Temp Log│ (공격 시 유실 가능)        │ Immutable Log Store  │ ✅
  └───────────────┘                          │ (WORM / Object Lock) │ 🔒
                                             └──────────────────────┘
                                             * "Once written, NEVER erased"
```

+++

### Ⅲ. 실무적 의의: 왜 중요한가?

- **내부자 위협 방어**: 시스템 권한을 가진 관리자가 자신의 부정행위 흔적을 지우는 것을 원천적으로 차단합니다.
- **포렌식의 핵심**: 사고 조사 시 "이 로그가 진짜인가?"라는 의심을 기술적으로 잠재워 수사의 속도를 높입니다.
- **컴플라이언스 대응**: ISMS, GDPR 등 정보보호 인증에서 감사 로그의 무결성은 가장 중요한 심사 항목 중 하나입니다.

- **📢 섹션 요약 비유**: 감사 로그 위변조 방지는 **'잉크로 쓴 공증 문서'**와 같습니다. 연필(수정 가능한 일반 파일)로 쓴 기록은 언제든 지우고 다시 쓸 수 있지만, 잉크로 쓰고 그 위에 투명 코팅(WORM)까지 한 문서는 내용을 고치려 하면 종이가 찢어져서 티가 날 수밖에 없습니다. 누구나 믿을 수 있는 역사 기록을 만드는 과정입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Non-repudiation]**: 행위를 한 사실을 나중에 부인할 수 없게 하는 성질.
- **[Object Lock]**: 클라우드 저장소에서 제공하는 데이터 불변성 옵션.
- **[SIEM (Security Information and Event Management)]**: 전사 로그를 수집하고 위변조를 감시하는 통합 시스템.

📢 **마무리 요약**: **Anti-tamper Logging**은 시스템의 마지막 양심입니다. 기술적으로 조작이 불가능한 기록 체계를 갖출 때, 데이터베이스는 비로소 정당한 신뢰를 얻게 됩니다.
