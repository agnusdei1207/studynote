# 521. 데이터 품질 진단(Profiling) - 데이터의 건강검진 기술

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 프로파일링(Profiling)은 데이터베이스 내의 실제 값들을 통계적으로 분석하여, **데이터의 구조, 내용, 관계 및 숨겨진 오류 패턴을 파악하는 품질 진단의 첫 번째 단계**이다.
> 2. **가치**: 주관적인 판단이 아닌 객관적인 지표(NULL 비율, 중복도, 패턴 위반 등)를 통해 **데이터의 오염 정도를 정량화**하며, 정제 작업의 우선순위와 거버넌스 수립의 근거를 제공한다.
> 3. **융합**: 메타데이터 분석 기술과 결합되어 데이터 리니지(Lineage)의 정확성을 검증하고, AI 학습 데이터의 편향성이나 노이즈를 사전에 걸러내는 품질 보증 도구로 융합된다.

+++

### Ⅰ. 데이터 프로파일링의 3대 핵심 기법

1. **컬럼 프로파일링 (Column Profiling)**: 
    - 개별 열의 데이터 타입, 길이, 최소/최대값, NULL 수, 유일 값 수 등을 분석. (기초 체력 검사)
2. **구조 프로파일링 (Structure Profiling)**: 
    - 데이터가 정해진 포맷(예: 이메일 형식, 날짜 형식)을 잘 지키고 있는지 검사. (형식 준수 확인)
3. **관계 프로파일링 (Relationship Profiling)**: 
    - 테이블 간의 참조 무결성이나 중복 관계를 분석. (가족 관계/연결고리 검사)

### Ⅲ. 실무적 활용: GIGO 방지

- **데이터 마이그레이션 전**: 소스 DB의 품질을 미리 진단하여 타겟 DB로 '쓰레기'가 넘어가는 것을 방지합니다.
- **DW/BI 구축**: 분석 결과의 왜곡을 막기 위해 원천 데이터의 통계적 정확성을 검증합니다.
- **거버넌스 기준 수립**: 프로파일링 결과를 바탕으로 "우리 회사 주소 데이터의 정확도는 95% 이상을 유지한다"는 식의 품질 목표(SLA)를 설정합니다.

- **📢 섹션 요약 비유**: 데이터 프로파일링은 **'정수장의 수질 검사'**와 같습니다. 물(데이터)을 가정에 공급하기 전에, 현미경(프로파일링 툴)으로 물속에 세균(오류 데이터)이나 중금속(노이즈)이 얼마나 들어있는지 꼼꼼히 체크하는 것입니다. 검사 결과가 나쁘면 정수 필터(데이터 정제)를 강화하여 깨끗한 물만 흐르게 만드는 기초 작업입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Data Cleansing]**: 프로파일링 결과를 바탕으로 수행되는 데이터 세척 작업.
- **[Anomaly Detection]**: 프로파일링을 통해 평소와 다른 데이터 패턴을 찾는 이상 탐지.
- **[Business Rule]**: 프로파일링 결과를 판정하는 비즈니스적인 품질 기준.

📢 **마무리 요약**: **Data Profiling**은 데이터의 진실을 대면하는 과정입니다. 현상을 정확히 진단할 때, 비로소 데이터를 정화하고 가치 있는 자산으로 바꿀 수 있는 해법이 나옵니다.
