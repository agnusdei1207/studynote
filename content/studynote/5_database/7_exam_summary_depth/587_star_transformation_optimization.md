# 587. 스타 변환(Star Transformation) - DW 쿼리의 고속도로

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스타 변환은 데이터 웨어하우스의 스타 스키마 구조에서 **비트맵 인덱스(Bitmap Index)를 활용하여, 팩트 테이블을 읽기 전 여러 차원의 조건을 먼저 결합하고 필터링하는 옵티마이저 최적화 기법**이다.
> 2. **가치**: 대규모 팩트 테이블과 여러 차원 테이블 간의 복잡한 조인 연산을 **단순한 비트 연산(AND/OR)과 인덱스 스캔**으로 치환하여, 디스크 I/O를 획기적으로 줄이고 조회 속도를 극대화한다.
> 3. **융합**: 비트맵 조인 인덱스 기술과 서브쿼리 최적화 기술이 융합된 형태이며, 주로 오라클 등 엔터프라이즈급 DBMS의 핵심 튜닝 매커니즘으로 작동한다.

+++

### Ⅰ. 스타 변환의 작동 프로세스

1. **서브쿼리 생성**: 메인 쿼리를 분석하여 각 차원 테이블의 조건을 서브쿼리로 분리합니다.
2. **비트맵 결합**: 팩트 테이블의 각 외래키(FK)에 걸린 비트맵 인덱스를 사용하여, 서브쿼리 결과에 해당하는 비트들을 추출합니다.
3. **비트 연산**: 여러 차원에서 온 비트맵들을 **AND 연산**하여, 모든 조건을 만족하는 최종 행의 위치(ROWID)를 도출합니다. ✅
4. **최종 추출**: 계산된 위치의 실제 팩트 데이터만 콕 집어 읽어오고, 나중에 차원 테이블과 조인합니다.

+++

### Ⅱ. 스타 변환 아키텍처 시각화 (ASCII Model)

```text
[ Star Transformation Logic ]

  (Query) "2026년 AND 서울 지역 AND 전자제품" 매출 조회
     │
     ▼ [ Step 1: Multiple Bitmap Scans ]
  ┌──────────────────┬──────────────────┬──────────────────┐
  │ Dim_Time (2026)  │ Dim_Region (SEO) │ Dim_Prod (Elec)  │
  ├──────────────────┼──────────────────┼──────────────────┤
  │ Bitmap: 11001    │ Bitmap: 10101    │ Bitmap: 11100    │ ✅
  └────────┬─────────┴────────┬─────────┴────────┬─────────┘
           │                  │                  │
           ▼ [ Step 2: BITMAP AND Operation ]    ▼
  ( 11001 ) AND ( 10101 ) AND ( 11100 ) = [ 10000 ] ✅ "Only Row 1 matches all"
                                              │
                                              ▼
  [ Step 3: Fetch Data ] ──▶ [ Fact_Sales Row 1 ] ──▶ User ✅
```

+++

### Ⅲ. 실무적 의의: 왜 스타 변환인가?

- **조인 순서의 혁명**: 기존에는 `Fact -> Dim1 -> Dim2` 순으로 하나씩 조인하며 데이터를 불려 나갔지만, 스타 변환은 조인 전에 이미 인덱스만으로 결과 후보를 압축해버립니다.
- **성능 폭발**: 팩트 테이블이 수조 건이더라도, 비트맵 연산을 통해 읽어야 할 블록을 수천 분의 일로 줄일 수 있습니다.
- **필수 조건**: 팩트 테이블의 모든 외래키에 **비트맵 인덱스**가 생성되어 있어야 하며, 옵티마이저 파라미터(`STAR_TRANSFORMATION_ENABLED`)가 활성화되어야 합니다.

- **📢 섹션 요약 비유**: 스타 변환은 **'백화점의 지능형 층별 안내기'**와 같습니다. 예전 방식(일반 조인)이 1층부터 꼭대기 층까지 모든 매장을 돌아다니며 "빨간색(차원1), 95사이즈(차원2), 10만원 이하(차원3) 셔츠"를 찾는 것이라면, 스타 변환은 입구 키오스크에서 세 가지 조건을 동시에 체크하여 "5층 12번 매장에 딱 한 벌 있습니다"라고 정확한 위치를 알려주는 것과 같습니다. 엘리베이터(I/O)를 딱 한 번만 타면 정답을 얻을 수 있는 가장 빠른 방법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Bitmap Index]**: 스타 변환을 가능케 하는 물리적 재료.
- **[Star Join]**: 스타 변환의 또 다른 명칭.
- **[Temporary Table]**: 변환 과정에서 중간 결과를 담기 위해 내부적으로 사용하는 공간.

📢 **마무리 요약**: **Star Transformation**은 데이터 웨어하우스의 치트키입니다. 방대한 팩트의 바다에서 비트 연산이라는 나침반을 통해 최단 경로를 찾아내는 고도의 최적화 기술입니다.
