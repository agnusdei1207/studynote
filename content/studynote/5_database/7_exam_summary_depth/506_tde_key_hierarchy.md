# 506. TDE 암호 키 계층 구조 - 이중 자물쇠의 보안

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TDE(투명한 데이터 암호화) 키 계층 구조는 보안성을 극대화하기 위해, **실제 데이터를 암호화하는 키(DEK)와 그 키를 다시 암호화하는 마스터 키(MK)로 역할을 분리하여 관리하는 이중 보안 체계**다.
> 2. **가치**: 데이터를 암호화한 키가 유출되더라도 상위의 마스터 키가 없으면 무용지물로 만들어, **암호 키 관리의 편의성과 보안성 사이의 난제**를 해결한다.
> 3. **융합**: 외부 키 관리 시스템(KMS)이나 하드웨어 보안 모듈(HSM)과 융합되어, 마스터 키를 데이터베이스 외부의 가장 안전한 장소에 격리 보관하는 엔터프라이즈급 보안 아키텍처를 완성한다.

+++

### Ⅰ. TDE 키 계층의 2단계 구조

1. **DEK (Data Encryption Key)**:
    - **역할**: 실제 테이블스페이스나 데이터 파일을 암호화하는 대칭키입니다.
    - **위치**: 데이터베이스 내부에 저장되지만, **마스터 키에 의해 암호화된 상태**로 존재합니다.
    - **특징**: 성능을 위해 데이터 파일과 가까운 곳에 위치합니다.
2. **MK (Master Key / Master Encryption Key)**:
    - **역할**: DEK를 암호화하여 보호하는 최상위 키입니다.
    - **위치**: 데이터베이스 외부의 **Wallet(지갑), KMS, 또는 HSM**에 별도로 보관됩니다.
    - **특징**: 이 키가 있어야만 DB 부팅 시 DEK를 복호화하여 암호화된 데이터를 읽을 수 있습니다.

+++

### Ⅱ. 키 계층 및 암호화 흐름 시각화 (ASCII Model)

```text
[ TDE Key Management Architecture ]

  ┌───────────────────────────────────────────┐
  │  Layer 1: External Key Store (KMS/HSM)    │
  │  [ Master Key (MK) ] ──▶ 최상위 자물쇠      │ ✅ Secure Isolation
  └─────────────┬─────────────────────────────┘
                │ (Encrypts / Decrypts)
                ▼
  ┌───────────────────────────────────────────┐
  │  Layer 2: Database Metadata / Wallet      │
  │  [ Encrypted DEK ] ──▶ 데이터 보호용 열쇠   │ 🔒 Encrypted at Rest
  └─────────────┬─────────────────────────────┘
                │ (Provides plain DEK to Mem)
                ▼
  ┌───────────────────────────────────────────┐
  │  Layer 3: Physical Storage                │
  │  [ Encrypted Data Blocks (.dbf) ]         │ 💾 Hardened Data
  └───────────────────────────────────────────┘
```

+++

### Ⅲ. 이중 계층 구조의 실무적 이점

- **키 교체(Rotation) 용이성**: 데이터 전체를 다시 암호화할 필요 없이, 상위 마스터 키만 교체하여 하위 DEK를 재암호화하는 것만으로 보안 등급을 갱신할 수 있습니다 (Rekey).
- **물리적 분리 보장**: 데이터 파일이 든 디스크를 통째로 훔쳐가더라도, 외부 장치(HSM)에 들어있는 마스터 키 없이는 절대 데이터를 열어볼 수 없습니다.
- **성능 유지**: 실제 데이터 연산은 메모리에 올라온 대칭키(DEK)로 빠르게 수행되므로 성능 저하를 최소화합니다.

- **📢 섹션 요약 비유**: TDE 키 계층 구조는 **'이중 보안 금고'**와 같습니다. 보석(데이터)을 담은 작은 상자(DEK)는 방(DB) 안에 두지만, 그 상자를 여는 열쇠는 집 밖의 안전한 은행 금고(MK/KMS)에 따로 맡겨두는 것입니다. 도둑이 담을 넘어 방에 들어와서 보석 상자를 훔쳐 가도, 은행 금고에 있는 열쇠가 없으면 상자를 절대 열 수 없는 원리입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Envelope Encryption]**: 봉투 암호화. 데이터 키를 마스터 키로 감싸는 이 방식의 공식 명칭.
- **[HSM (Hardware Security Module)]**: 마스터 키를 물리적으로 보호하는 특수 하드웨어.
- **[Rotation]**: 보안을 위해 주기적으로 마스터 키를 바꾸는 관리 행위.

📢 **마무리 요약**: **TDE Key Hierarchy**는 데이터 보안의 논리적 완결성을 제공합니다. 키와 데이터를 분리하고, 키를 다시 키로 감싸는 이중 보호망이 현대 데이터베이스의 철통 보안을 완성합니다.
