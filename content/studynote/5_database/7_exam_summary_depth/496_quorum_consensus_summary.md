# 496. 쿼럼(Quorum) - 분산 합의의 정족수

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 쿼럼(Quorum)은 분산 데이터베이스에서 데이터의 일관성을 보장하기 위해, **읽기나 쓰기 작업 시 반드시 응답을 얻어야 하는 최소한의 노드 수(정족수)**를 의미한다.
> 2. **가치**: 전체 노드가 아닌 일부 노드의 응답만으로도 '합의'를 도출하게 함으로써, **시스템 가용성과 데이터 정합성 사이의 정교한 균형**을 맞춘다.
> 3. **융합**: 읽기 정족수($R$)와 쓰기 정족수($W$)의 합이 전체 복제본 수($N$)보다 커야 한다는 $R + W > N$ 공식을 통해, 어떤 읽기 요청에서도 반드시 최신 쓰기 결과가 포함되도록 설계된다.

+++

### Ⅰ. 쿼럼의 기본 수식과 조건

- **N (Replication Factor)**: 데이터를 복제하여 저장하는 총 노드 수.
- **W (Write Quorum)**: 쓰기 성공을 위해 응답해야 하는 최소 노드 수.
- **R (Read Quorum)**: 읽기 성공을 위해 확인해야 하는 최소 노드 수.
- **강한 일관성 조건**: **$W + R > N$**
    - 이 조건이 만족되면, 읽기 노드 집합과 쓰기 노드 집합에는 반드시 **최소 하나 이상의 겹치는 노드**가 존재하게 되어 최신 데이터를 읽을 수 있습니다.

+++

### Ⅱ. 쿼럼 작동 매커니즘 시각화 (ASCII Model)

복제본이 3개($N=3$)인 환경에서 $W=2, R=2$ 인 경우입니다.

```text
[ Quorum Overlap Scenario (N=3, W=2, R=2) ]

  Nodes: [ Node A ]  [ Node B ]  [ Node C ]
            │          │          │
  Write:    ●──────────● (W=2 성공) │ ✅ Latest Data in A, B
            │          │          │
  Read:     │          ●──────────● (R=2 성공) ✅
            │          │          │
  * 결과: 읽기 요청 시 Node B가 중복 포함되어 있으므로, 
    Node C는 옛날 값을 주더라도 Node B의 최신 값을 통해 정합성 확보!
```

+++

### Ⅲ. 실무적 튜닝 전략

1. **쓰기 최적화 (W=1, R=N)**: 쓰기는 매우 빠르지만, 읽을 때 모든 노드를 확인해야 하므로 읽기가 느려집니다.
2. **읽기 최적화 (W=N, R=1)**: 읽기는 한 노드만 봐도 되니 빠르지만, 쓰기 시 노드 하나만 장애가 나도 전체 쓰기가 실패합니다.
3. **균형 모델 (W=Q, R=Q)**: 보통 $N/2 + 1$ (과반수) 방식을 사용하여 가용성과 일관성을 모두 적절히 챙깁니다.

- **📢 섹션 요약 비유**: 쿼럼은 **'위원회 의결 정족수'**와 같습니다. 3명의 위원(노드)이 있을 때, 안건을 통과(쓰기)시키려면 최소 2명(W)의 찬성이 필요합니다. 나중에 그 결정을 확인(읽기)할 때도 2명(R)에게 물어본다면, 그중 한 명은 반드시 아까 찬성했던 사람일 수밖에 없으므로 "어떤 결정이 내려졌는지" 정확히 알 수 있는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Sloppy Quorum]**: 장애 시 임시 노드를 포함시키는 느슨한 쿼럼 (가용성 극대화).
- **[Hinted Handoff]**: 쿼럼 부족 시 나중에 데이터를 전달하기 위해 남기는 메모.
- **[Consistency Level]**: 카산드라 등에서 쿼럼 수치를 설정하는 옵션 명칭.

📢 **마무리 요약**: **Quorum**은 분산 시스템의 민주주의이자 물리 법칙입니다. 노드의 개수와 합의의 강도를 조절함으로써, 우리는 비즈니스에 가장 최적화된 데이터 신뢰의 지도를 그릴 수 있습니다.
