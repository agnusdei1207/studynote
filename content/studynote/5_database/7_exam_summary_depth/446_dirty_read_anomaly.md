# 446. 오손 읽기(Dirty Read) - 유령 같은 미확정 데이터

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오손 읽기는 한 트랜잭션이 **다른 트랜잭션이 수정 중이지만 아직 커밋(Commit)하지 않은 중간 상태의 데이터를 읽어 들이는 현상**이다.
> 2. **가치**: 트랜잭션의 원자성(Atomicity) 원칙을 위반할 위험이 크며, 읽어온 데이터가 이후 롤백(Rollback)될 경우 존재하지 않는 가짜 정보를 바탕으로 잘못된 비즈니스 판단을 내리게 된다.
> 3. **융합**: 격리 수준(Isolation Level) 중 'Read Uncommitted'에서 발생하며, 이를 방지하기 위해 현대 대부분의 DB는 'Read Committed'를 기본값으로 융합 설정한다.

+++

### Ⅰ. 오손 읽기의 발생 시나리오

1. **미확정 수정**: 트랜잭션 T1이 계좌 잔액을 100원에서 500원으로 업데이트합니다 (아직 커밋 안 함).
2. **부적절한 읽기**: 트랜잭션 T2가 해당 계좌를 조회하여 '500원'이라는 값을 읽어갑니다.
3. **원인 불명 취소**: T1이 어떤 오류로 인해 `ROLLBACK`을 수행합니다. 계좌 잔액은 다시 100원이 됩니다.
4. **결과**: T2는 세상에 존재하지도 않는 '500원'을 진짜라고 믿고 후속 작업을 진행합니다 (데이터 오염).

+++

### Ⅱ. 오손 읽기 타임라인 시각화 (ASCII Flow)

```text
[ Dirty Read Scenario ]

  Time | Transaction T1 (Writers) | Transaction T2 (Readers) | Actual DB
  -----|--------------------------|--------------------------|----------
   t1  | Update Val = 500         |                          |   100
   t2  |                          | Read Val ──▶ [500] 💥    |   100 (in mem)
   t3  | 💥 ROLLBACK!             |                          |   100
   t4  |                          | Use Val(500) for logic...|   100
  ----------------------------------------------------------------------
  * Result: T2 read a "Dirty" value that never officially existed.
```

+++

### Ⅲ. 대응 방안 및 격리 수준

- **격리 수준 상향**: 'Read Committed' 이상의 격리 수준을 사용하면, 커밋된 데이터만 읽도록 강제하여 오손 읽기를 100% 방지할 수 있습니다.
- **MVCC 활용**: 데이터가 수정 중일 때, 수정 중인 값을 보여주는 대신 Undo 영역에 저장된 '마지막 커밋 버전'을 보여줌으로써 읽기 일관성을 유지합니다.

- **📢 섹션 요약 비유**: 오손 읽기는 **'요리사의 간 보기 단계에서 음식을 훔쳐 먹는 것'**과 같습니다. 요리사(T1)가 찌개에 소금을 왕창 넣었다가(Update) 너무 짜서 다시 물을 붓거나 버리려고 하는데(Rollback), 그 사이를 못 참고 웨이터(T2)가 간만 본 찌개를 손님에게 내가는 상황입니다. 손님은 완성되지 않은, 엉터리 요리(Dirty Data)를 먹게 되는 셈입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Read Uncommitted]**: 오손 읽기가 허용되는 위험한 격리 수준.
- **[Read Committed]**: 오손 읽기를 차단하는 가장 일반적인 격리 수준.
- **[Undo Log]**: 오손 읽기를 막기 위해 과거 데이터를 제공하는 원천.

📢 **마무리 요약**: **Dirty Read**는 데이터베이스의 정직함을 해치는 행위입니다. 미확정된 데이터의 유혹을 뿌리치고, 오직 '확정된 진실(Commit)'만을 읽는 체계를 갖추는 것이 무결성 수호의 기본입니다.
