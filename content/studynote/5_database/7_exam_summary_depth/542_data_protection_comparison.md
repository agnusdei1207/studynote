# 542. 데이터 보호 3대 기술 - 마스킹, 비식별화, 암호화 비교

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 보호 기술은 목적에 따라 **원본을 수학적으로 변환하여 숨기는 '암호화'**, 형태를 유지하며 **일부를 가리는 '마스킹'**, 개인을 특정할 수 없게 **통계적으로 변형하는 '비식별화'**로 구분된다.
> 2. **가치**: 암호화는 기밀성을, 마스킹은 가독성 기반 보안을, 비식별화는 데이터 활용과 프라이버시 보호의 균형을 제공하여 **각기 다른 비즈니스 요구사항을 충족**한다.
> 3. **융합**: 보안 위협 모델에 따라 저장 데이터는 암호화(TDE), 조회 화면은 마스킹(Dynamic Masking), 분석 목적 데이터는 비식별화(K-Anonymity)를 적용하는 다층 방어 체계로 융합된다.

+++

### Ⅰ. 데이터 보호 기술 상세 비교 매트릭스

| 비교 항목 | 암호화 (Encryption) | 데이터 마스킹 (Masking) | 비식별화 (De-identification) |
|:---|:---|:---|:---|
| **복원 가능성** | **가역적** (Key 필요) | 부분 가역적 / 비가역적 | **비가역적 지향** |
| **데이터 형태** | 완전히 변함 (바이너리) | **원본 형태 유지** (* 처리) | **통계적 특성 유지** (범주화) |
| **핵심 목적** | 비인가자 탈취 방지 | **인가된 사용자의 노출 최소화** | **개인 정보 보호 및 분석 활용** |
| **적용 시점** | 저장(At-rest) / 전송 중 | **조회(SELECT) 시점** | **데이터 배포 / 분석 시점** |

+++

### Ⅱ. 기술별 데이터 변환 예시 (ASCII Model)

```text
[ Original Data: 홍길동 | 900101-1234567 | 서울시 강남구 ]

  1. Encryption ──▶ [ X&@#$! (Unreadable Binary) ] ✅ "Full Privacy"
  
  2. Masking    ──▶ [ 홍*동 | 900101-1****** ] ✅ "Readability + Security"
  
  3. Anonymization ──▶ [ 홍씨 | 30대 | 서울특별시 ] ✅ "Utility for Stats"
```

+++

### Ⅲ. 실무적 선택 및 적용 가이드

- **암호화 필수 구역**: DB 데이터 파일, 백업 테이프, 네트워크 통신 구간. (해킹 대비 최후의 보루)
- **마스킹 필수 구역**: 고객 센터 상담 화면, 영수증 출력물, 관리자 조회 대시보드. (내부자 유출 및 실수 방지)
- **비식별화 필수 구역**: 마케팅 통계 분석, 타 기업과의 데이터 결합, AI 학습용 데이터셋. (법적 리스크 없이 데이터 활용)

- **📢 섹션 요약 비유**: 이 세 기술은 **'귀중품을 지키는 세 가지 방법'**과 같습니다. 암호화는 귀중품을 '금고에 넣고 잠그는 것'이고, 마스킹은 '귀중품 위에 검은 보자기(가림막)를 씌워두는 것'이며, 비식별화는 '귀중품의 가치(통계량)는 남기되 원래 주인이 누구인지는 알 수 없게 모양을 뭉개버리는 것'과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[TDE]**: 대표적인 저장 데이터 암호화 기술.
- **[Tokenization]**: 마스킹보다 보안성이 높은 고도의 치환 기술.
- **[K-Anonymity]**: 비식별화의 적정성을 평가하는 대표적 모델.

📢 **마무리 요약**: **Encryption, Masking, De-identification**은 데이터 보안의 삼총사입니다. 데이터의 생애주기와 사용 목적에 맞는 기술을 적재적소에 배치할 때, 데이터는 비로소 '안전한 자산'이 됩니다.
