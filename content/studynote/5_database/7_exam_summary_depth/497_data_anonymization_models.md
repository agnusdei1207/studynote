# 497. 개인정보 비식별화 모델 - 익명성의 수학적 증명

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 개인정보 비식별화는 데이터셋에서 특정인을 식별할 수 있는 정보를 삭제하거나 변형하여, **누구인지 알 수 없게 만들면서도 데이터의 통계적 가치는 보존하는 기술**이다.
> 2. **가치**: 연결 공격(Linkage Attack)이나 유추 공격으로부터 프라이버시를 보호하며, K-익명성, L-다양성, T-근접성 등의 모델을 통해 **익명성의 수준을 수학적으로 측정하고 강제**한다.
> 3. **융합**: 데이터 3법 등 법적 규제 준수와 빅데이터/AI 산업 활성화의 접점에 위치하며, 안전한 데이터 결합과 유통을 가능케 하는 신뢰의 기술이다.

+++

### Ⅰ. 3대 비식별화 보호 모델

1. **K-익명성 (K-Anonymity)**: 
    - 같은 속성 값을 가진 레코드가 최소 K개 이상 존재하게 함. 
    - **목표**: 특정인을 $1/K$ 확률 이하로만 식별 가능하게 함.
2. **L-다양성 (L-Diversity)**: 
    - K-익명성 그룹 내에서 민감한 정보(질병 등)가 최소 L개 이상의 서로 다른 종류를 갖게 함.
    - **목표**: "K-익명성 그룹이 모두 암 환자"인 경우와 같은 동질성 공격 방어.
3. **T-근접성 (T-Closeness)**: 
    - 전체 데이터셋의 민감 정보 분포와 특정 비식별 그룹의 분포 차이를 T 이하로 유지.
    - **목표**: 특정 그룹의 정보 쏠림 현상을 막아 정보 유추 공격 방어.

+++

### Ⅱ. 비식별화 기법 시각화 (ASCII Model)

```text
[ Data Transformation Examples ]

  (Original)  Name: 홍길동 | Age: 28 | Zip: 12345 | Disease: Cancer
       │
       ▼ [ 비식별 기법 적용 ]
       
  1. 삭제 (Suppression)  : Name: ---    | Age: 28 | Zip: 12345 | Disease: Cancer
  2. 범주화 (Generalization): Name: 홍씨   | Age: 20대 | Zip: 123** | Disease: Cancer
  3. 마스킹 (Masking)    : Name: 홍*동   | Age: 28 | Zip: 12345 | Disease: Cancer ✅
```

+++

### Ⅲ. 실무적 의의: 활용과 보안의 균형

- **데이터의 유용성 (Utility)**: 비식별화를 과하게 하면(Age -> '인간') 데이터는 안전해지지만 분석 가치는 사라집니다.
- **재식별 위험 (Re-identification)**: 다른 공공 데이터와 결합하여 비식별 데이터를 다시 실명 데이터로 복원하려는 공격에 상시 대비해야 합니다.
- **적정 수준 평가**: 기업은 '개인정보 비식별 조치 가이드라인'에 따라 적정성 평가 위원회를 거쳐 데이터 외부 배포 여부를 결정합니다.

- **📢 섹션 요약 비유**: 비식별화 모델은 **'가면무도회'**와 같습니다. K-익명성은 같은 가면을 쓴 사람이 최소 K명 이상 있게 하여 누가 누군지 헷갈리게 하는 것이고, L-다양성은 그 가면 쓴 사람들이 손에 든 가방(민감 정보) 색깔을 다 다르게 해서 가방만 보고 주인을 맞히지 못하게 하는 것과 같습니다. 안전하게 파티(데이터 분석)를 즐기기 위한 약속입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Quasi-identifier]**: 결합 시 개인 식별이 가능해지는 속성들 (나이, 성별, 우편번호 등).
- **[Differential Privacy]**: 노이즈를 섞어 더 강력한 프라이버시를 보장하는 차세대 기술.
- **[DPIA]**: 개인정보 영향 평가 절차.

📢 **마무리 요약**: **Anonymization Models**는 데이터 경제의 안전띠입니다. 수학적으로 증명된 익명성 모델을 통해 프라이버시 침해 걱정 없는 투명한 데이터 세상을 만들어가야 합니다.
