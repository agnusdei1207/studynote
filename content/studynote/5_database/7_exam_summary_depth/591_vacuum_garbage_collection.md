# 591. 가비지 컬렉션(MVCC Vacuum) - 보이지 않는 청소의 기술

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가비지 컬렉션(GC), 특히 MVCC 환경의 **진공(Vacuum) 프로세스**는 데이터 업데이트나 삭제 후 더 이상 어떤 트랜잭션에서도 참조하지 않는 **과거 버전의 데이터(Dead Tuples)를 물리적으로 회수하여 공간을 재사용**하는 작업이다.
> 2. **가치**: 불필요한 데이터가 쌓여 성능을 떨어뜨리는 **테이블 팽창(Bloat)** 현상을 방지하며, 트랜잭션 ID 랩어라운드(Wraparound) 장애를 예방하여 시스템의 지속 가능성을 보장한다.
> 3. **융합**: 스토리지 엔진의 블록 관리 기술과 트랜잭션 생명주기 관리 기술이 융합되어, 서비스 성능에 영향을 주지 않는 범위 내에서 자동으로 수행(Auto-vacuum)되는 지능형 청소 체계를 형성한다.

+++

### Ⅰ. MVCC와 가비지(Garbage)의 발생

- **MVCC의 특성**: 데이터를 수정할 때 원본을 고치지 않고 새로운 버전을 생성합니다 (Copy-on-Write 성격).
- **쓰레기 생성**: 트랜잭션이 종료되어 더 이상 예전 데이터를 볼 필요가 없어지면, 디스크에 남은 구버전 데이터는 '가비지'가 됩니다.
- **문제점**: 청소를 안 하면 테이블 크기가 무한히 커지고, 인덱스 검색 효율이 급격히 떨어집니다.

+++

### Ⅱ. Vacuum(진공) 프로세스 시각화 (ASCII Model)

```text
[ Data Block Before & After Vacuum ]

  (Before Vacuum: Bloated)
  ┌─────────────────────────────────────────┐
  │ [Live A][Dead B][Live C][Dead D][Live E]│ 💥 Waste Space
  └─────────────────────────────────────────┘
          │
          ▼ [ Vacuum Execution ]
  ┌─────────────────────────────────────────┐
  │ [Live A][ FREE ][Live C][ FREE ][Live E]│ ✅ Reusable
  └─────────────────────────────────────────┘
          │
          ▼ [ Full Vacuum (Compaction) ]
  ┌─────────────────────────┬───────────────┐
  │ [Live A][Live C][Live E]│ (Disk Return) │ ✅ Shrinked
  └─────────────────────────┴───────────────┘
```

+++

### Ⅲ. 실무적 의의: Vacuum 전략

1. **Auto-vacuum**: 임계치 이상의 변경이 발생하면 시스템이 백그라운드에서 자동으로 수행. (현대 DB의 표준)
2. **Full Vacuum**: 테이블을 통째로 새로 써서 빈 공간을 OS에 반환. (수행 중 배타적 잠금 필요, 서비스 중단 주의)
3. **Transaction ID 관리**: PostgreSQL 등에서는 트랜잭션 번호가 한계에 도달하기 전 Vacuum을 통해 번호를 재사용(Freeze)해야 하는 치명적 관리 포인트가 존재합니다.

- **📢 섹션 요약 비유**: 가비지 컬렉션은 **'호텔의 객실 정비'**와 같습니다. 손님(트랜잭션)이 퇴실(커밋/롤백)하면, 그들이 썼던 침구류와 쓰레기(과거 버전 데이터)를 깨끗이 치워야 다음 손님을 받을 수 있습니다. 만약 청소를 안 하면 호텔 방은 금방 쓰레기장이 되어 아무도 쓸 수 없게 됩니다. 손님이 없는 틈을 타서 방을 정리(Vacuum)하거나, 아예 리모델링(Full Vacuum)을 해서 공간을 효율적으로 쓰는 지혜가 필요합니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Dead Tuple]**: 논리적으로는 삭제되었으나 물리적으로 공간을 점유하고 있는 데이터.
- **[Bloat]**: 가비지가 쌓여 테이블이 비정상적으로 커지는 현상.
- **[Freeze]**: 과거 트랜잭션 ID를 동결하여 ID 고갈을 막는 기술.

📢 **마무리 요약**: **Garbage Collection**은 데이터베이스의 건강을 지키는 비결입니다. 보이지 않는 곳에서 묵묵히 공간을 정화할 때, 데이터베이스는 비로소 지치지 않는 영구적 성능을 발휘합니다.
