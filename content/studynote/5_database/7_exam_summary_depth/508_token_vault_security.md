# 508. 토큰화 Vault(Token Vault) - 암호화의 요새

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 토큰화 Vault(금고)는 민감한 원천 데이터와 이를 대체하는 **무의미한 토큰(Token) 사이의 매핑 정보를 저장하고 관리하는 독립된 고보안 저장 구역**이다.
> 2. **가치**: 실제 데이터는 오직 이 Vault 내에만 존재하게 하여 **공격 표면(Attack Surface)을 최소화**하며, 해커가 비즈니스 DB를 탈취하더라도 아무런 가치 없는 난수(토큰)만 얻게 만든다.
> 3. **융합**: 하드웨어 보안 모듈(HSM) 및 강력한 접근 제어 솔루션과 융합되어, PCI-DSS 등 글로벌 금융 보안 표준을 준수하는 핵심 인프라로 기능한다.

+++

### Ⅰ. 토큰화 Vault의 핵심 기능

- **토큰 생성 및 매핑**: 원본 데이터가 입력되면 난수를 생성하여 1:1 매핑 테이블을 구축합니다.
- **Detokenization 제어**: 권한이 있는 특정 시스템에서만 토큰을 실데이터로 바꿀 수 있도록 엄격히 통제합니다.
- **데이터 격리**: 비즈니스용 DB와 물리적/논리적으로 완전히 분리된 네트워크 망에 위치합니다.

+++

### Ⅱ. Vault 기반 보안 아키텍처 시각화 (ASCII Model)

```text
[ Token Vault Isolation Model ]

  (Untrusted Zone)         (Trusted Zone)           (Secure Vault Zone)
  ┌──────────────┐        ┌──────────────┐        ┌──────────────────┐
  │ Public App   │ ──▶    │ Business DB  │ ──▶    │  Token Vault     │ ✅
  │ (User View)  │        │ (Stored:     │        │  [Mapping Table] │ 🔒
  │ [ Token ]    │        │   T-998877 ) │        │  Real: 1234...   │
  └──────────────┘        └──────────────┘        │  Token: T-998877 │
                                                  └────────┬─────────┘
                                                           │
  (Action: Payment) ◀──────────────────────────────────────┘
  * "결제 시점에만 Vault에 접근하여 실제 카드번호 획득"
```

+++

### Ⅲ. Vault 관리의 보안 원칙

1. **최소 접근 권한**: 오직 토큰화 서버만이 Vault에 접근할 수 있으며, 일반 DBA나 개발자는 접근이 원천 차단됩니다.
2. **강력한 암호화**: Vault 내의 매핑 정보 자체도 최신 암호화 알고리즘으로 이중 보호됩니다.
3. **감사 추적**: 누가 어떤 토큰을 실데이터로 변환(Detokenization)했는지 모든 이력을 실시간으로 기록합니다.

- **📢 섹션 요약 비유**: 토큰화 Vault는 **'은행의 안전 금고실'**과 같습니다. 시중의 상점(비즈니스 DB)에서는 진짜 금괴(민감 데이터) 대신 '금 보관 증서(토큰)'만 주고받습니다. 증서만으로는 아무것도 할 수 없으며, 오직 삼엄한 경비가 쳐진 은행 금고실(Vault)에 가서 정당한 주인임을 확인받아야만 진짜 금괴를 꺼낼 수 있는 것과 같은 원리입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Detokenization]**: 토큰을 다시 원본 데이터로 복원하는 과정 (가장 위험한 단계).
- **[Tokenization as a Service]**: 클라우드 기반으로 Vault 기능을 제공하는 서비스.
- **[PCI-DSS]**: Vault의 물리적/논리적 보안 요건을 규정하는 대표적 표준.

📢 **마무리 요약**: **Token Vault**는 데이터 보안의 최후 보루입니다. 민감한 정보의 서식지를 단 한 곳으로 한정하고 그곳을 요새화함으로써, 전체 시스템의 안전을 보장하는 전략적 핵심 자산입니다.
