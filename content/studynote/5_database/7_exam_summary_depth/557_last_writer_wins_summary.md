# 557. 멀티 마스터 충돌 해결 - Last Writer Wins(LWW)의 명암

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: LWW(Last Writer Wins)는 멀티 마스터 환경에서 동일한 데이터에 대해 동시 수정이 발생했을 때, **각 노드의 타임스탬프를 비교하여 가장 나중에 기록된 데이터를 최종 진실로 인정하는 단순 합의 매커니즘**이다.
> 2. **가치**: 복잡한 인과관계 추적이나 사용자 개입 없이 **충돌을 즉각적이고 자동으로 해결**하여 시스템의 가용성을 극대화하지만, 먼저 기록된 데이터가 소리 없이 사라지는 데이터 유실 리스크를 내포한다.
> 3. **융합**: 카산드라(Cassandra), Riak 등 고가용성을 지향하는 분산 NoSQL 저장소의 기본 충돌 해결 전략으로 융합되어 사용된다.

+++

### Ⅰ. LWW(Last Writer Wins)의 작동 원리

1. **타임스탬프 부여**: 모든 쓰기 요청은 발생 시점의 정밀한 타임스탬프를 메타데이터로 함께 저장합니다.
2. **충돌 감지**: 노드 간 동기화 도중 동일 키에 대해 서로 다른 값이 발견되면 타임스탬프를 비교합니다.
3. **최종 승자 결정**: 타임스탬프 숫자가 더 큰(더 최신인) 데이터를 남기고, 작은 데이터는 폐기합니다.

+++

### Ⅱ. LWW 충돌 해결 시각화 (ASCII Model)

```text
[ Multi-Master Write Collision ]

  Step 1: Concurrent Writes
  (Node A) SET Key='Red'   (TS: 100.001) ──┐
                                           │ (Syncing...)
  (Node B) SET Key='Blue'  (TS: 100.002) ──┼─▶ [ Conflict Resolution ]
                                           │
  Step 2: Compare Timestamps               │
  - Node A TS (100.001) < Node B TS (100.002)
  
  Step 3: Blue Wins! 🏆
  [ Final DB State ] : Key = 'Blue' ✅ (Node A's 'Red' is DELETED) 💥
```

+++

### Ⅲ. LWW의 장단점 및 주의사항

- **장점**: 
    - 성능: 연산이 매우 단순하여 조율 오버헤드가 거의 없습니다.
    - 자율성: 노드들이 서로 대화하지 않고도 독자적으로 결정을 내릴 수 있습니다.
- **단점**: 
    - **데이터 유실**: 동시에 일어난 두 작업 중 하나는 반드시 버려집니다. 
    - **시계 오차(Clock Skew)**: 노드 간 서버 시간이 미세하게 다를 경우, 실제로는 먼저 일어난 사건이 나중 사건으로 오판되어 정합성이 깨질 수 있습니다.
- **보완책**: 인과관계가 중요한 경우 **벡터 시계(Vector Clock)**를 병행 사용하거나, 유실되면 안 되는 데이터(장바구니 등)는 LWW 대신 비즈니스 로직 기반의 병합(Merge)을 수행해야 합니다.

- **📢 섹션 요약 비유**: LWW는 **'초시계로 판정하는 달리기 시합'**과 같습니다. 두 선수가 거의 동시에 결승선에 들어왔을 때, 복잡한 사정은 다 빼고 오직 기록지에 적힌 0.001초 차이만 보고 승자를 정하는 냉정한 룰입니다. 기록지(서버 시계)가 조금이라도 틀리면 엉뚱한 우승자가 나올 수 있고, 2등의 노력(데이터)은 완전히 무시되는 단점이 있지만, 판정 시비 없이 경기를 가장 빨리 끝낼 수 있는 방법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Conflict Resolution]**: 멀티 마스터 분산 시스템의 숙명적 과제.
- **[Clock Skew]**: LWW의 신뢰성을 위협하는 물리적 시간차.
- **[Cassandra]**: LWW를 기본 전략으로 채택한 대표적인 DB.

📢 **마무리 요약**: **Last Writer Wins**는 단순함의 미학이자 위험한 도박입니다. 유실을 감수하더라도 극강의 가용성이 필요한 도메인에서 최고의 위력을 발휘하는 실용적인 합의 기법입니다.
