# 428. 테이블 풀 스캔(Full Table Scan) - 전수 조사의 미학

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 테이블 풀 스캔(FTS)은 쿼리 조건에 부합하는 데이터를 찾기 위해 **데이터베이스 테이블의 첫 번째 블록부터 마지막 블록까지 모든 데이터를 순차적으로 읽어 들이는 접근 방식**이다.
> 2. **가치**: 인덱스가 없거나 데이터의 상당 부분(보통 20% 이상)을 읽어야 할 때 오히려 인덱스 스캔보다 효율적이며, 한 번의 I/O로 여러 블록을 읽는 **Multi-Block Read** 기술을 통해 성능을 보정한다.
> 3. **융합**: 소규모 테이블 조회나 대량의 배치 분석(Batch) 환경에서 옵티마이저의 전략적 선택지가 되며, 랜덤 I/O를 순차 I/O로 전환하는 성능 최적화의 한 축을 담당한다.

+++

### Ⅰ. 테이블 풀 스캔이 발생하는 상황

- **인덱스 부재**: 조회 조건 컬럼에 인덱스가 없는 경우.
- **선택도(Selectivity) 낮음**: 인덱스를 타는 것보다 그냥 다 읽는 게 빠르다고 옵티마이저가 판단한 경우 (대량 데이터 조회).
- **데이터량 적음**: 테이블 자체가 너무 작아 인덱스 구조를 타는 오버헤드가 더 큰 경우.
- **인덱스 무력화**: `LIKE '%word'` 검색이나 컬럼 변형(`WHERE Salary * 12 > 5000`) 시 발생.

+++

### Ⅱ. 풀 스캔 작동 매커니즘 시각화 (ASCII Model)

```text
[ Table Full Scan Process ]

  (Query) SELECT * FROM Orders WHERE Amount > 100;
     │
     ▼ [ Database Engine ]
  ┌───────────────────────────────────────────────────────────┐
  │  Multi-Block Read Start (HWM: High Water Mark 까지)        │
  │                                                           │
  │  [ Block 1 ][ Block 2 ][ Block 3 ][ Block 4 ][ Block 5 ]  │
  │  └────┬────┘└────┬────┘└────┬────┘└────┬────┘└────┬────┘  │
  │       ▼          ▼          ▼          ▼          ▼       │
  │    (Check)    (Check)    (Check)    (Check)    (Check)    │ ✅
  └──────────────────────────────┬────────────────────────────┘
                                 ▼
                        [ Matching Results ] ──▶ User
```

### Ⅲ. 인덱스 스캔 vs 풀 스캔 성능 비교

| 비교 항목 | 인덱스 레인지 스캔 | 테이블 풀 스캔 |
|:---|:---|:---|
| **I/O 방식** | **Single Block Read** (랜덤) | **Multi-Block Read** (순차) |
| **속도** | 소량 데이터 조회 시 매우 빠름 | **대량 데이터 조회 시 효율적** |
| **자원 소모** | CPU 위주 소모 | **Disk I/O 및 대역폭 집중 소모** |
| **적정 지점** | 전체의 10~15% 미만 조회 시 | 전체의 20% 이상 조회 시 |

- **📢 섹션 요약 비유**: 테이블 풀 스캔은 **'시험 범위 전체 정독하기'**와 같습니다. 내가 찾는 내용이 어디 있는지 모를 때(인덱스 없음), 혹은 시험 문제의 80%가 교과서 전체에서 나올 때(낮은 선택도)는 굳이 요약 노트를 뒤지는 것보다 처음부터 끝까지 정독하는 것이 확실하고 빠를 수 있습니다. 특히 한 번에 여러 페이지를 쓱쓱 넘기며(Multi-Block Read) 읽는 숙련된 독서법과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[High Water Mark (HWM)]**: 풀 스캔이 멈춰야 할 테이블의 끝 지점.
- **[Multi-Block Read]**: 한 번의 I/O로 수십 개의 블록을 한꺼번에 읽는 가속 기술.
- **[Full Table Scan Hint]**: `/*+ FULL(table_name) */`

📢 **마무리 요약**: **Full Table Scan**은 무조건 피해야 할 악습이 아닙니다. 데이터의 규모와 성격에 따라 가장 정직하고 효율적인 정공법이 될 수 있는, 옵티마이저의 전략적 카드입니다.
