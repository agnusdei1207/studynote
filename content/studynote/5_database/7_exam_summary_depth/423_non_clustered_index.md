# 423. 넌클러스터드 인덱스(Non-Clustered) - 데이터의 보조 지도

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 넌클러스터드 인덱스는 실제 데이터 파일의 물리적 순서와 상관없이, **인덱스 키 값만 정렬하여 별도의 공간에 저장하고 각 키가 실제 데이터의 물리적 주소(RID/ROWID)를 가리키게 하는 보조 인덱스**다.
> 2. **가치**: 하나의 테이블에 여러 개를 생성할 수 있어 다양한 조회 조건에 대응 가능하며, 테이블 자체를 재정렬하지 않아도 되므로 삽입/수정 성능에 미치는 영향이 클러스터드 인덱스보다 적다.
> 3. **융합**: B+Tree 구조로 관리되며, 리프 노드에서 실제 데이터 페이지로 한 번 더 찾아가는 '이동(Fetch)' 과정이 발생하여 대량 조회 시 랜덤 I/O 부하가 발생할 수 있다.

+++

### Ⅰ. 넌클러스터드 인덱스의 작동 원리

- **인덱스 페이지**: 키 값(예: 이름)과 데이터 주소(ROWID) 쌍으로 구성됩니다. 키 값 순으로 정렬되어 있습니다.
- **데이터 페이지**: 인덱스 순서와 무관하게 데이터가 무질서하게(Heap 구조 등) 들어있을 수 있습니다.
- **조회 과정**: 인덱스 검색 ──▶ ROWID 획득 ──▶ 실제 데이터 블록 접근 (Table Access by Index ROWID).

+++

### Ⅱ. 인덱스와 데이터의 분리 시각화 (ASCII Model)

```text
[ Non-Clustered Index Structure ]

  < Index Pages (Sorted) >           < Data Pages (Unsorted) >
  ┌──────────┬───────────┐           ┌───────────────────────┐
  │  Key     │ Address   │           │ RowID: AAA | Data...  │
  ├──────────┼───────────┤           ├───────────────────────┤
  │  Alice   │ Addr: CCC │──▶───┐    │ RowID: BBB | Data...  │
  │  Bob     │ Addr: AAA │──▶─┐ │    ├───────────────────────┤
  │  Zack    │ Addr: BBB │──▶ │ └──▶ │ RowID: CCC | Data...  │ ✅
  └──────────┴───────────┘    └─▶────┴───────────────────────┘ ✅
```

+++

### Ⅲ. 클러스터드 인덱스와의 주요 차이점

| 비교 항목 | 클러스터드 인덱스 | 넌클러스터드 인덱스 |
|:---|:---|:---|
| **물리적 정렬** | 인덱스 순서 = 실제 데이터 순서 | **인덱스 순서 ≠ 실제 데이터 순서** |
| **개수 제한** | 테이블당 단 1개 | **테이블당 여러 개 생성 가능** |
| **리프 노드 내용** | 실제 데이터 행(Row) 자체 | **데이터를 가리키는 주소(ROWID)** |
| **조회 속도** | 매우 빠름 | 양호 (단, Fetch 시 I/O 발생) |

- **📢 섹션 요약 비유**: 넌클러스터드 인덱스는 **'전공 서적 맨 뒤의 인덱스(찾아보기)'**와 같습니다. 책의 본문(데이터)은 장(Chapter) 순서대로 쓰여 있지만, 인덱스 페이지는 가나다순으로 정렬되어 있어 단어와 페이지 번호만 적혀 있습니다. 우리가 단어를 찾은 뒤(인덱스 검색), 해당 페이지를 직접 넘겨서(ROWID 이동) 내용을 확인해야 하는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[RID (Row Identifier)]**: 행의 물리적 위치 정보.
- **[Secondary Index]**: 넌클러스터드 인덱스의 또 다른 이름.
- **[Random I/O]**: 넌클러스터드 인덱스 사용 시 발생하는 성능상의 주의점.

📢 **마무리 요약**: **Non-Clustered Index**는 유연한 검색의 열쇠입니다. 테이블의 원본을 건드리지 않고도 다양한 각도에서 데이터를 빠르게 찾아낼 수 있게 돕는 데이터베이스의 필수 편의 시설입니다.
