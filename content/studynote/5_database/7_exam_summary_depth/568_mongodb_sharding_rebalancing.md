# 568. MongoDB 샤딩 리밸런싱 - 데이터의 공평한 이사

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: MongoDB 샤딩 리밸런싱은 특정 샤드에 데이터가 쏠리는 불균형을 해결하기 위해, **백그라운드의 밸런서(Balancer)가 데이터 덩어리인 청크(Chunk)를 한가한 샤드로 자동 이동시키는 과정**이다.
> 2. **가치**: 관리자의 수동 개입 없이도 클러스터 전체의 **부하 균형(Load Balancing)**을 유지하며, 서버 추가 시 새로운 자원을 즉시 분석 작업에 투입할 수 있게 한다.
> 3. **융합**: 샤드 키 기반의 범위 판정과 분산 트랜잭션 기술이 융합되어, 데이터가 이동하는 중에도 중단 없는 읽기/쓰기 서비스를 보장한다.

+++

### Ⅰ. 리밸런싱의 핵심 구성 요소

1. **Chunk (청크)**: 샤드 내 데이터의 분할 단위 (기본 64MB). 샤드 키의 특정 범위를 담고 있습니다.
2. **Balancer (밸런서)**: 각 샤드에 담긴 청크 개수를 상시 모니터링하여, 개수 차이가 임계치를 넘으면 이사를 지시하는 감시자입니다.
3. **Migration (마이그레이션)**: 청크를 소스 샤드에서 타겟 샤드로 물리적으로 복사하고, 완료 후 소스 데이터를 삭제하는 실제 이사 작업입니다.

+++

### Ⅱ. 청크 마이그레이션 프로세스 시각화 (ASCII Model)

```text
[ MongoDB Background Balancing ]

  [ Shard A ] (Overloaded)       [ Shard B ] (Idle)
  ┌──────────┐                   ┌──────────┐
  │ Chunk 1  │ ──(1. Copy)──▶    │ [Empty]  │
  │ Chunk 2  │ ──(2. Sync)──▶    │ [Empty]  │
  │ Chunk 3  │                   └──────────┘
  └──────────┘                        │
       │                              │
       └────(3. Drop from Source) ◀───┘ ✅ Migration Complete!

  * 밸런서: "A에 청크가 너무 많네? 1번 청크를 B로 옮겨!"
  * 결과: Shard A(2개), Shard B(1개) ✅ "Balance Restored"
```

+++

### Ⅲ. 실무적 의의 및 주의사항

- **무중단 운영**: 마이그레이션 중에도 `mongos` 라우터가 데이터의 위치를 추적하므로 사용자는 서비스 중단 없이 데이터를 조회할 수 있습니다.
- **성능 저하 주의**: 데이터 이동은 네트워크와 디스크 자원을 소모하므로, 트래픽이 몰리는 피크 타임에는 밸런서 가동 시간을 제한(Window 설정)하는 것이 좋습니다.
- **샤드 키의 중요성**: 샤드 키를 잘못 잡으면(예: 날짜순), 밸런서가 아무리 데이터를 옮겨도 새 데이터가 계속 특정 샤드로만 들어오는 **'무한 이사'** 현상이 발생할 수 있습니다.

- **📢 섹션 요약 비유**: 샤딩 리밸런싱은 **'대형 마트의 계산대 줄 조절'**과 같습니다. 1번 계산대(Shard A)에만 손님(데이터)이 가득 차면, 마트 매니저(Balancer)가 나타나 "손님, 이쪽 2번 계산대(Shard B)가 한가하니 이쪽으로 옮겨서 결제 도와드릴게요"라고 안내하는 것과 같습니다. 손님은 조금 귀찮지만(이동 부하), 전체적으로는 훨씬 빨리 계산을 마칠 수 있게 되는 원리입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[mongos]**: 데이터가 어느 샤드로 이사 갔는지 알고 있는 길잡이.
- **[Config Server]**: 청크의 위치 정보를 최종 관리하는 두뇌.
- **[Shard Key Skew]**: 리밸런싱 부하를 폭증시키는 데이터 불균형 현상.

📢 **마무리 요약**: **MongoDB Balancing**은 분산 시스템의 평화 유지 활동입니다. 보이지 않는 곳에서 끊임없이 데이터를 재배치함으로써, 시스템의 수평적 확장을 완성하는 핵심 매커니즘입니다.
