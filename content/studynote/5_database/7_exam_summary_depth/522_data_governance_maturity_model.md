# 522. 데이터 거버넌스 성숙도 모델(DMM) - 단계별 도약의 지도

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 거버넌스 성숙도 모델(DMM, Data Management Maturity)은 조직의 데이터 관리 능력을 객관적으로 진단하고, **현재의 수준에서 더 높은 단계로 나아가기 위한 로드맵을 제시하는 평가 프레임워크**다.
> 2. **가치**: 데이터 전략, 품질, 데이터 거버넌스, 플랫폼 등 주요 영역별로 **초기(Ad-hoc) 단계부터 최적화(Optimized) 단계까지의 성숙도**를 측정하여 시스템적인 데이터 경영을 가능케 한다.
> 3. **융합**: CMMI 사상과 데이터 공학이 융합된 형태이며, 기업의 디지털 전환(DX) 수준을 가늠하는 핵심적인 경영 지표로 활용된다.

+++

### Ⅰ. DMM의 5단계 성숙도 레벨 (일반적 모델 기준)

1. **1단계: 초기 (Initial / Ad-hoc)**: 프로세스가 없고 개인의 역량에 의존. 데이터 오류가 잦고 관리가 파편화됨.
2. **2단계: 관리 (Managed)**: 부서 단위로 데이터 관리 절차가 생김. 기본적인 품질 체크가 시작됨.
3. **3단계: 정의 (Defined)**: 전사적인 표준과 거버넌스 체계가 수립됨. 공통 언어와 프로세스가 존재함.
4. **4단계: 측정 (Measured)**: 데이터 품질과 프로세스 효율성을 정량적 수치로 모니터링하고 제어함.
5. **5단계: 최적화 (Optimized)**: 지속적인 개선이 자동화됨. 데이터가 비즈니스 혁신의 핵심 동력으로 완벽히 작동함.

### Ⅲ. DMM의 6대 핵심 진단 영역

- **데이터 전략**: 데이터가 비즈니스 목표와 연계되어 있는가?
- **데이터 거버넌스**: 역할과 책임(R&R)이 명확하고 정책이 실행되는가?
- **데이터 품질**: 데이터의 정확성과 신뢰성을 측정하고 관리하는가?
- **데이터 운영**: 데이터 생애주기별 파이프라인이 안정적인가?
- **플랫폼 및 아키텍처**: 데이터를 처리하기 위한 기술적 기반이 견고한가?
- **데이터 지원 프로세스**: 메타데이터 및 용어 표준화가 되어 있는가?

- **📢 섹션 요약 비유**: 데이터 거버넌스 성숙도 모델은 **'요리사의 숙련도 단계'**와 같습니다. 1단계는 레시피도 없이 대충 요리하는 초보(초기)이고, 3단계는 정해진 표준 레시피대로 맛을 내는 요리사(정의)이며, 5단계는 주방 시스템을 자동화하고 손님의 건강 상태(데이터 가치)까지 고려해 최고의 요리를 내놓는 마스터 셰프(최적화)가 되는 과정입니다. 우리 회사가 지금 어떤 수준인지 알아야 다음 비법을 배울 수 있는 것과 같습니다.

### Ⅳ. 개념 맵 및 요약

- **[CMMI]**: DMM 모델의 근간이 된 소프트웨어 프로세스 성숙도 모델.
- **[Gap Analysis]**: 현재 성숙도와 목표 성숙도 사이의 차이를 분석하는 기법.
- **[Data Culture]**: 상위 단계로 가기 위해 기술보다 중요한 조직의 문화적 태도.

📢 **마무리 요약**: **DMM**은 데이터 경영의 성적표이자 안내서입니다. 막연한 목표가 아닌 단계별 실천 과제를 통해, 조직 전체를 데이터 기반의 스마트한 유기체로 변화시키는 나침반 역할을 합니다.
