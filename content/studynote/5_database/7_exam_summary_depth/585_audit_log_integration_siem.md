# 585. 데이터베이스 감사 로그 통합 - SIEM 연계를 통한 입체적 보안

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터베이스 감사 로그 통합은 개별 DB 서버에 흩어진 접속 및 쿼리 실행 이력을 **중앙 집중형 보안 관리 시스템(SIEM)으로 실시간 전송하여 위변조를 방지하고 통합 분석하는 보안 체계**다.
> 2. **가치**: 내부 관리자에 의한 로그 삭제 위험을 원천 차단하며, DB 활동을 웹 로그, 네트워크 로그 등과 결합 분석하여 **지능형 지속 위협(APT)이나 데이터 유출 징후를 조기에 포착**한다.
> 3. **융합**: WORM(불변 저장소) 기술과 상관관계 분석(Correlation) 엔진이 융합되어, 법적 컴플라이언스 준수와 실시간 위협 대응이라는 두 마리 토끼를 잡는다.

+++

### Ⅰ. 감사 로그 통합 관리의 필요성

- **위변조 방지**: 해커나 악의적 내부자가 자신의 흔적을 지우기 위해 로컬 DB 로그를 수정하는 것을 막아야 합니다.
- **가시성 확보**: 수십 대의 DB 서버에서 일어나는 일을 한 화면에서 실시간 감시해야 합니다.
- **복합 위협 탐지**: "웹 서버의 이상 접속"과 "DB의 대량 조회"라는 서로 다른 이벤트를 하나로 엮어 공격 시나리오를 완성해 냅니다.

+++

### Ⅲ. SIEM 연계를 통한 지능형 보안 시나리오

1. **대량 유출 방어**: 특정 사용자가 평소보다 100배 많은 개인정보를 조회하면 즉시 경고를 띄우고 세션을 강제 종료합니다.
2. **우회 접속 탐지**: 정상적인 앱 서버가 아닌, 관리자의 로컬 PC에서 직접 DB로 접근하는 '비정상 경로'를 포착합니다.
3. **권한 오남용 감시**: `GRANT` 명령을 통해 갑자기 높은 권한을 얻은 계정이 평소 접근하지 않던 테이블을 건드리는지 감시합니다.

- **📢 섹션 요약 비유**: 감사 로그 통합은 **'건물 폐쇄회로 TV(CCTV)의 통합 관제 센터'**와 같습니다. 각 방(DB)마다 설치된 카메라는 그 방의 일만 찍지만, 통합 관제 센터(SIEM)에서는 건물 전체의 움직임을 한눈에 봅니다. 범인이 1층에서 창문을 깨고(웹 해킹), 3층 복도를 지나(내부 이동), 5층 금고(DB 조회)를 여는 전체 과정을 연결해서 파악할 수 있으며, 범인이 카메라 녹화본을 지우려 해도 이미 센터로 전송된 영상은 건드릴 수 없는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Correlation]**: 서로 다른 장비의 로그를 연결하여 의미를 찾아내는 상관관계 분석.
- **[Non-repudiation]**: 로그 통합을 통해 달성되는 행위 부인 방지.
- **[Compliance]**: ISMS, GDPR 등에서 요구하는 통합 로그 보존 요건.

📢 **마무리 요약**: **Audit Log Integration**은 보안의 완결성입니다. 흩어진 기록을 하나로 묶어 살아있는 정보로 바꿀 때, 데이터베이스는 비로소 어떤 위협 앞에서도 투명한 진실을 증명할 수 있습니다.
