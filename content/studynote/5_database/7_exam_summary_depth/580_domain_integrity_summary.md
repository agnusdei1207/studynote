# 580. 도메인 무결성과 CHECK 제약 조건 - 값의 정당성 부여

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 도메인 무결성(Domain Integrity)은 속성이 가질 수 있는 **유효한 값의 범위를 제한하는 것**으로, SQL의 **CHECK 제약 조건과 정규 표현식**을 통해 데이터의 물리적 형식을 넘어 논리적 타당성까지 강제하는 기술이다.
> 2. **가치**: "나이는 음수가 될 수 없다"거나 "이메일 형식은 반드시 @를 포함해야 한다"는 등의 비즈니스 규칙을 DB 레벨에서 원천 보호하여, **애플리케이션의 오류로 인한 오염된 데이터 유입을 차단**한다.
> 3. **융합**: 데이터 타입(Domain), 기본값(Default), NULL 허용 여부 기술과 융합되어, 데이터베이스의 가장 기초적인 품질 방어선 역할을 수행한다.

+++

### Ⅰ. 도메인 무결성 실현 수단: CHECK 제약 조건

- **정의**: 특정 컬럼에 데이터가 입력되거나 수정될 때, 설정된 조건식(Boolean)을 만족하는지 검사하는 장치입니다.
- **주요 기능**:
    - **범위 제한**: `Age >= 0 AND Age < 150`
    - **목록 제한**: `Gender IN ('M', 'F')`
    - **정규식 매칭**: `REGEXP_LIKE(Email, '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$')` ✅

+++

### Ⅱ. 도메인 무결성 검증 아키텍처 시각화 (ASCII Model)

데이터가 DB로 들어오는 입구에서 '필터' 역할을 수행합니다.

```text
[ Domain Integrity Filter ]

  (Input Attempt) ──▶ [ CHECK Constraint Engine ]
                             │
          ┌──────────────────┴──────────────────┐
          ▼ (Condition: Salary > 0)             ▼
  [ Value: -100 ]                       [ Value: 5000 ]
          │                                     │
          ▼ [ VIOLATION! ] ❌                   ▼ [ SUCCESS ] ✅
  ┌──────────────────────┐              ┌──────────────────────┐
  │  Transaction Reject  │              │  Stored in Database  │ 💾
  └──────────────────────┘              └──────────────────────┘
```

+++

### Ⅲ. 실무적 의의: 데이터 품질의 파수꾼

- **방어적 설계**: 클라이언트나 서버 앱에서 필터링을 놓치더라도, 최후의 보루인 DB가 "이 값은 논리적으로 틀렸다"라고 거부함으로써 시스템 전체의 정합성을 수호합니다.
- **표준화**: 전사적으로 공통된 데이터 도메인을 정의(예: 전화번호 도메인)하여 모든 테이블에서 동일한 CHECK 규칙을 사용하게 함으로써 관리 효율을 높입니다.
- **성능**: 복잡한 트리거(Trigger)를 쓰는 것보다 CHECK 제약 조건을 쓰는 것이 훨씬 가볍고 옵티마이저가 쿼리를 최적화하는 데에도 도움이 됩니다.

- **📢 섹션 요약 비유**: 도메인 무결성은 **'놀이공원의 키 제한'**과 같습니다. "키가 120cm 이상인 어린이만 탈 수 있다"는 규칙(CHECK)을 입구에서 검사하여, 자격이 안 되는 사람이 무리하게 탑승(잘못된 데이터 입력)하여 사고(시스템 장애)가 나는 것을 원천 차단하는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Attribute]**: 도메인 무결성이 적용되는 최소 단위 (속성).
- **[Regular Expression]**: 문자열의 패턴을 정교하게 제어하는 기술.
- **[User Defined Type (UDT)]**: 사용자가 직접 정의한 복합 도메인.

📢 **마무리 요약**: **Domain Integrity**는 데이터의 '성격'을 정의합니다. 데이터가 가져야 할 본연의 모습을 기술적으로 명시할 때, 데이터베이스는 비로소 지식의 원천으로서 신뢰를 얻습니다.
