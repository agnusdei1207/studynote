# 489. 데이터 메시(Data Mesh) - 데이터 관리의 분권화 선언

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 메시(Data Mesh)는 거대한 중앙 집중식 데이터 팀의 병목 현상을 해결하기 위해, **데이터 소유권과 관리 책임을 개별 비즈니스 도메인 부서로 위임하는 조직 및 아키텍처 패러다임**이다.
> 2. **가치**: 데이터를 중앙으로 수집해야 할 '짐'이 아닌, 각 부서가 가치를 창출하여 배포하는 **'제품(Product)'**으로 정의함으로써, 데이터의 품질과 활용 속도를 획기적으로 높인다.
> 3. **융합**: 마이크로서비스(MSA)의 도메인 주도 설계(DDD) 철학을 데이터 영역으로 확장한 것이며, 현대의 복잡한 멀티 클라우드 데이터 거버넌스의 새로운 해법으로 융합된다.

+++

### Ⅰ. 데이터 메시의 4대 핵심 원칙 (Pillars)

1. **도메인 기반 데이터 소유권 (Domain Ownership)**: 데이터를 생산하는 현업 부서가 해당 데이터의 설계와 품질을 직접 책임집니다.
2. **제품으로서의 데이터 (Data as a Product)**: 다른 부서 사용자가 쉽게 찾고, 읽고, 신뢰할 수 있도록 표준화된 인터페이스로 제공합니다.
3. **셀프 서비스 플랫폼 (Self-serve Platform)**: 각 부서가 인프라 고민 없이 데이터를 제품화할 수 있도록 중앙에서 자동화된 도구를 제공합니다.
4. **연합된 컴퓨팅 거버넌스 (Federated Governance)**: 분산된 데이터 부서들이 지켜야 할 최소한의 전사 표준(보안, 포맷 등)을 자동화된 기술로 강제합니다.

+++

### Ⅱ. 데이터 메시 아키텍처 시각화 (ASCII Model)

```text
[ Data Mesh: Decentralized Flow ]

  ┌──────────────┐      ┌──────────────┐      ┌──────────────┐
  │  마케팅 도메인 │      │  재무 도메인  │      │  물류 도메인  │
  │ (Data Owner) │      │ (Data Owner) │      │ (Data Owner) │
  └──────┬───────┘      └──────┬───────┘      └──────┬───────┘
         │ (Data Product)      │ (Data Product)      │ (Data Product)
         ▼                     ▼                     ▼
  ┌──────────────────────────────────────────────────────────┐
  │      Self-Serve Data Infrastructure (AWS/S3/Spark)       │ ✅
  ├──────────────────────────────────────────────────────────┤
  │      Computational Governance (Global Standards)          │ ✅
  └──────────────────────────────────────────────────────────┘
```

### Ⅲ. 중앙 집중형 DW와의 결정적 차이

| 비교 항목 | 중앙 집중형 (Monolithic DW) | 데이터 메시 (Data Mesh) |
|:---|:---|:---|
| **관리 주체** | 중앙 데이터 전문 팀 | **현업 비즈니스 도메인 팀** |
| **데이터 흐름** | 모든 데이터를 중앙으로 흡수 | **각 도메인에서 제품으로 배포** |
| **확장성** | 중앙 팀의 인력 한계에 봉착 | **도메인 증가에 따라 무한 확장** |
| **품질 책임** | 데이터 팀 (맥락 파악 어려움) | **도메인 팀 (데이터 의미 전문가)** |

- **📢 섹션 요약 비유**: 데이터 메시는 **'대형 할인 마트'에서 '전통 시장'으로의 변화**와 같습니다. 마트(중앙 DW)는 본사가 모든 물건을 관리하지만 구색이 늦게 갖춰질 수 있습니다. 반면 전통 시장(메시)은 각 상점 주인(도메인)이 자기가 가장 잘 아는 전문 품목을 직접 골라와서 팔기 때문에 훨씬 신선하고 활기찬 것과 같습니다. 시장 전체의 규칙(거버넌스)만 잘 지키면 훨씬 생동감 넘치는 데이터 생태계가 됩니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Data Product]**: 메시 아키텍처의 핵심 유통 단위.
- **[Domain Driven Design (DDD)]**: 데이터 메시의 사상적 모태.
- **[Data Democratization]**: 전문가뿐만 아니라 누구나 데이터를 제품화하고 분석하는 지향점.

📢 **마무리 요약**: **Data Mesh**는 데이터 관리의 주권을 사용자에게 돌려주는 혁명입니다. 통제가 아닌 자율을 통해 거대한 데이터 파도를 비즈니스의 성장 동력으로 전환하는 가장 현대적인 전략입니다.
