# 394. 시스템 카탈로그(Catalog) - 메타데이터의 총집합

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 카탈로그는 데이터베이스 관리 시스템(DBMS)이 생성하고 관리하는 **모든 객체의 메타데이터(Metadata)를 체계적으로 모아놓은 전역 저장소**다.
> 2. **가치**: 데이터베이스의 논리적 구조와 물리적 배치를 연결하며, 일반 데이터와 마찬가지로 테이블 형태로 존재하여 표준 SQL로 조회 가능하다는 특징을 가진다.
> 3. **융합**: 데이터 사전(Data Dictionary)의 구현체이며, 데이터베이스 재구성이나 성능 튜닝 시 가장 먼저 참조되는 지식 베이스(Knowledge Base) 역할을 수행한다.

+++

### Ⅰ. 시스템 카탈로그의 주요 저장 정보

- **구조 정보**: 테이블, 뷰, 인덱스의 정의 및 속성.
- **제약 조건**: 기본키, 외래키, 고유값, 체크 제약 조건.
- **통계 정보**: 카디널리티, 인덱스 클러스터링 팩터, 데이터 분포도. (성능 최적화의 핵심)
- **보안 정보**: 사용자 계정, 프로필, 객체별 접근 권한 매트릭스.
- **연결 정보**: 저장 프로시저, 트리거 간의 의존성 관계.

+++

### Ⅱ. 카탈로그 가시성 시각화 (ASCII Model)

카탈로그는 일반 테이블처럼 보이지만, 수정은 오직 DBMS만이 할 수 있습니다.

```text
[ System Catalog Visibility ]

  (User SQL) : "SELECT * FROM PG_TABLES;" (PostgreSQL 예시)
       │
       ▼
  ┌─────────────────────────────────────────┐
  │  System Catalog (Tables / Views)        │
  ├───────────────────┬─────────────────────┤
  │ [ Table Meta ]    │ [ Index Meta ]      │
  │ - name, owner     │ - type, columns     │
  ├───────────────────┼─────────────────────┤
  │ [ Statistic Meta ]│ [ Privilege Meta ]  │
  │ - rows, blocks    │ - grant, revoke     │
  └─────────┬─────────┴───────────┬─────────┘
            │                     │
            ▼                     ▼
      [ Query Optimizer ]   [ Security Manager ]
```

+++

### Ⅲ. 카탈로그와 데이터 사전의 관계

- **이론적 정의**: 데이터 사전 (Data Dictionary)
- **물리적 구현**: 시스템 카탈로그 (System Catalog)
- **차이점**: 학술적으로는 혼용되나, 실무적으로는 DB 엔진이 내부적으로 관리하는 테이블군을 카탈로그라고 부릅니다.

- **📢 섹션 요약 비유**: 시스템 카탈로그는 **'백화점의 안내 데스크 및 관리 장부'**와 같습니다. 고객(사용자)은 안내 데스크(조회용 뷰)를 보고 매장이 어디 있는지(데이터 위치) 알 수 있고, 관리자(DBMS)는 장부(카탈로그 테이블)를 보며 매장의 매출이나 재고 현황(통계 정보)을 관리하여 백화점이 원활하게 돌아가도록 조율하는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Metadata]**: 카탈로그에 담기는 정보의 실체.
- **[Information Schema]**: ANSI 표준에 따라 정의된 공통 카탈로그 뷰 규격.
- **[DDL (Data Definition Language)]**: 카탈로그의 정보를 생성하고 수정하는 언어.

📢 **마무리 요약**: **System Catalog**는 데이터베이스의 지도이자 나침반입니다. 카탈로그를 정교하게 관리하고 이해하는 것이 데이터베이스의 무결성과 성능을 지키는 가장 확실한 방법입니다.
