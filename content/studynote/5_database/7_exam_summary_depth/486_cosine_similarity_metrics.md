# 486. 코사인 유사도(Cosine Similarity) - 각도로 찾는 의미의 거리

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 코사인 유사도는 두 벡터 사이의 **각도($\theta$)의 코사인 값을 이용하여 데이터 간의 유사성을 측정하는 척도**로, 벡터의 크기(Magnitude)보다는 방향성에 집중한다.
> 2. **가치**: 텍스트 분석 시 문서의 길이나 단어 빈도수에 상관없이 **문맥적 의미의 유사함**을 정확히 포착하며, 벡터 데이터베이스의 검색 성능을 결정짓는 핵심 수학적 연산이다.
> 3. **융합**: 임베딩 기술과 융합되어 시맨틱 검색 엔진의 거리 계산 표준으로 쓰이며, 결과값은 -1(반대)에서 1(동일) 사이의 값을 가져 직관적인 해석이 가능하다.

+++

### Ⅰ. 코사인 유사도의 수학적 정의

- **공식**: $Similarity = \cos(\theta) = \frac{A \cdot B}{\|A\| \|B\|}$
- **의미**: 두 벡터의 내적을 각 벡터의 크기의 곱으로 나눈 값입니다.
- **범위**: 
    - **1**: 두 벡터의 방향이 완전히 일치 (매우 유사).
    - **0**: 두 벡터가 수직 (독립적, 관계없음).
    - **-1**: 두 벡터가 정반대 방향.

+++

### Ⅱ. 코사인 유사도 측정 시각화 (ASCII Model)

```text
[ Vector Angle and Similarity ]

      Y-Axis
        ▲      ● Vector A (Document 1)
        │     /
        │    / θ (Small Angle = High Similarity)
        │   /─────────▶ ● Vector B (Document 2)
        │  /
        └──────────────────────────▶ X-Axis

  * 특징: 두 벡터의 길이(값의 크기)가 달라도 
    벌어진 각도(방향)만 같다면 유사도는 1에 수렴함 ✅
```

+++

### Ⅲ. 실무적 활용과 장점

1. **텍스트 검색**: 문서 A가 100단어, 문서 B가 10,000단어라도 핵심 키워드의 분포(방향)가 비슷하면 유사한 문서로 판별합니다 (Length Invariance).
2. **추천 시스템**: 사용자의 과거 평점 패턴과 가장 비슷한 취향을 가진 이웃(Neighbor)을 찾을 때 유용합니다.
3. **이미지 유사도**: 딥러닝으로 추출된 이미지 특징 벡터 간의 유사성을 비교하여 비슷한 사진을 찾아냅니다.

- **📢 섹션 요약 비유**: 코사인 유사도는 **'나침반 바늘의 방향 비교'**와 같습니다. 나침반이 얼마나 크든 작든 상관없이, 두 바늘이 가리키는 방향(각도)이 얼마나 일치하느냐만 보는 것입니다. 북동쪽을 가리키는 작은 바늘과 큰 바늘은 '방향성' 면에서 완벽하게 유사한(Cosine 1) 상태인 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Dot Product]**: 코사인 유사도 계산의 핵심 분자 연산.
- **[Euclidean Distance]**: 각도가 아닌 '직선거리'를 재는 방식 (코사인과 대조).
- **[Normalization]**: 벡터의 크기를 1로 맞추면 코사인 유사도와 내적 연산이 같아지는 성질.

📢 **마무리 요약**: **Cosine Similarity**는 데이터의 '의도'를 파악하는 기술입니다. 수치적인 크기의 환상을 걷어내고, 데이터가 향하는 순수한 방향성을 통해 가장 닮은 정답을 찾아냅니다.
