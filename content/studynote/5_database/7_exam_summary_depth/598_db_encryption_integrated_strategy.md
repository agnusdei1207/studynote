# 598. DB 암호화 통합 체계 - API, Plug-in, TDE의 다층 방어

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DB 암호화 통합 체계는 데이터의 중요도와 시스템 성능 영향도를 고려하여 **애플리케이션(API), DB 커널(Plug-in), 스토리지(TDE) 계층의 암호화 기술을 적재적소에 혼합 구성하는 보안 전략**이다.
> 2. **가치**: 단일 방식의 한계를 극복하여 내부자에 의한 유출(API/Plug-in 방어)과 외부 공격자에 의한 물리적 탈취(TDE 방어)를 동시에 차단하는 **심층 방어(Defense in Depth)**를 실현한다.
> 3. **융합**: 암호화 모듈과 외부 키 관리 시스템(KMS), HSM 기술이 인프라 망 구조와 융합되어, 성능 저하를 최소화하면서도 법적 암호화 요건을 완벽히 충족한다.

+++

### Ⅰ. 3대 암호화 방식의 융합 전략

1. **API 방식 (최상위 보안)**:
    - 앱에서 암호화 후 전송. DB 서버가 뚫려도 데이터는 안전.
    - **적용**: 비밀번호, 바이오 정보 등 극도로 민감한 고유 식별자.
2. **Plug-in 방식 (유연한 통제)**:
    - DB 커널에 설치된 모듈이 암호화. SQL 수정이 적음.
    - **적용**: 일반적인 개인정보(이름, 주소) 및 업무상 중요한 비공개 데이터.
3. **TDE 방식 (투명한 스토리지 보안)**:
    - 엔진이 디스크 저장 시 자동 암호화. 성능 저하 극소.
    - **적용**: 데이터 파일 전체, 백업 파일, 대량의 로그 데이터 보호. ✅

+++

### Ⅱ. 다층 암호화 인프라 시각화 (ASCII Model)

```text
[ Multi-layered DB Encryption Architecture ]

  (Application Server) ───(1) API Encryption (Field Level) ✅
           │
  (Secure Network: TLS)
           │
  (Database Server)    ───(2) Plug-in Encryption (Column Level) ✅
           │
  (Storage Engine)     ───(3) TDE Encryption (File/Block Level) ✅
           │
  (Physical Disk)      ───▶ [ 💾 Encrypted Bits ]
```

+++

### Ⅲ. 체계 검토 시 주요 고려사항

- **키 관리의 통합**: 여러 암호화 방식이 섞여 있어도 암호 키는 중앙 KMS/HSM에서 통합 관리하여 일관성을 유지해야 합니다.
- **성능 오버헤드**: 모든 컬럼을 API 방식으로 암호화하면 인덱스 조회가 불가능해지므로, 검색 빈도가 높은 컬럼은 TDE나 OPE(순서 보존 암호화)를 검토합니다.
- **가용성**: 암호화 모듈 장애 시 서비스가 중단되지 않도록 고가용성(HA) 구성과 적절한 예외 처리 로직이 필요합니다.

- **📢 섹션 요약 비유**: DB 암호화 통합 체계는 **'금고 안의 보석 상자'**와 같습니다. 집 대문(네트워크 보안)을 잠그는 것은 기본이고, 보물이 든 방 전체를 특수 합금(TDE)으로 만들며, 그 안의 금고(Plug-in)를 설치하고, 가장 비싼 다이아몬드는 다시 별도의 작은 자물쇠 상자(API)에 넣어두는 것과 같습니다. 도둑이 대문을 따고 들어와도 보석을 가져가기까지 겹겹이 쌓인 장애물을 만나게 하는 철저한 보안 설계입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Defense in Depth]**: 보안의 기본 원칙인 심층 방어 전략.
- **[KMS (Key Management System)]**: 파편화된 암호화 방식의 열쇠를 통합하는 컨트롤 타워.
- **[Performance Impact]**: 암호화 수준과 비례하는 시스템 자원 소모량.

📢 **마무리 요약**: **Integrated DB Encryption**은 기술의 조합이 아닌 정책의 완성입니다. 데이터의 가치를 분류하고 계층별 보호막을 씌울 때, 비로소 데이터베이스는 난공불락의 요새가 됩니다.
