# 519. Amazon Aurora 스토리지 - 분산 쿼럼과 로그의 혁신

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Amazon Aurora는 **컴퓨팅과 스토리지를 분리**하고, 데이터베이스 페이지가 아닌 **로그 레코드만을 스토리지 계층으로 전송(The Log is the Database)**하여 I/O 효율을 극대화한 클라우드 네이티브 아키텍처다.
> 2. **가치**: 데이터를 3개의 가용 영역(AZ)에 걸쳐 **6개로 복제(6-way Replication)**하며, **4/6 쓰기 쿼럼과 3/6 읽기 쿼럼** 방식을 통해 노드 장애 시에도 데이터 유실 없는 초고가용성을 실현한다.
> 3. **융합**: 서버리스 기술과 결합되어 부하에 따라 리소스를 즉시 확장하고, 스토리지 자체에서 자가 치유(Self-healing) 및 백업 작업을 수행하는 지능형 저장 체계를 완성한다.

+++

### Ⅰ. Aurora 스토리지의 핵심 원리: 쿼럼(Quorum) 모델

- **복제 방식**: 하나의 쓰기 데이터는 3개 AZ에 걸쳐 6개의 스토리지 노드에 복제됩니다.
- **쓰기 쿼럼 (Write Quorum)**: 6개 중 4개의 노드로부터 확인 응답을 받으면 성공으로 간주합니다. (1개 AZ가 통째로 날아가도 쓰기 가능)
- **읽기 쿼럼 (Read Quorum)**: 6개 중 3개의 노드로부터 데이터를 확인하여 최신성을 보장합니다.
- **장점**: 특정 노드가 느려지거나 장애가 나도 전체 성능에 미치는 영향이 매우 적습니다.

+++

### Ⅱ. Aurora 분산 복제 시각화 (ASCII Model)

```text
[ Aurora 6-way Replication & Quorum ]

      (Compute Node) ──▶ [ Log Stream Only ] ──▶ (High Speed Net)
                               │
          ┌────────────────────┼────────────────────┐
          ▼                    ▼                    ▼
      [ AZ 1 ]             [ AZ 2 ]             [ AZ 3 ]
    ┌────┐┌────┐         ┌────┐┌────┐         ┌────┐┌────┐
    │Rep1││Rep2│         │Rep3││Rep4│         │Rep5││Rep6│ ✅
    └────┘└────┘         └────┘└────┘         └────┘└────┘
    
  * Write Success: 4 out of 6 nodes ACK ✅
  * Survivability: 2 nodes fail = No problem, 3 nodes fail = Read-only ✅
```

+++

### Ⅲ. "The Log is the Database"

- **전통적 DB**: 변경된 데이터 페이지 전체를 디스크에 씁니다. (Heavy I/O)
- **Aurora**: 변경 내역인 **Redo 로그**만 스토리지 계층으로 쏩니다. 실제 페이지 조립(Materialization)은 스토리지 노드가 백그라운드에서 스스로 수행합니다.
- **효과**: 네트워크 트래픽을 기존 대비 1/10 수준으로 줄여 응답 속도를 획기적으로 개선합니다.

- **📢 섹션 요약 비유**: 오로라 스토리지는 **'분신술을 쓰는 꼼꼼한 서기'**와 같습니다. 내가 명령(로그) 한 번만 내리면, 6명의 서기(복제본)가 전 세계 3곳에 흩어져서 동시에 기록을 시작합니다. 그중 4명만 기록을 마쳐도 "완료!"라고 답해주니 매우 빠릅니다. 게다가 서기들이 단순히 받아 적는 게 아니라, 나중에 내가 책(데이터 페이지)을 보여달라고 할 때 그동안 적은 메모들을 합쳐서 즉석에서 책을 만들어 보여주는 아주 유능한 시스템입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[6-way Replication]**: 가용성을 위한 오로라의 물리적 전제.
- **[Storage Auto-scaling]**: 데이터가 늘어나는 대로 10GB 단위로 무한 확장되는 스토리지.
- **[Self-healing]**: 고장 난 데이터 블록을 백그라운드에서 자동으로 감지하고 복구하는 기능.

📢 **마무리 요약**: **Amazon Aurora Storage**는 클라우드의 힘을 데이터베이스에 이식한 걸작입니다. 로그 중심의 설계와 쿼럼 기반의 합의를 통해, 성능과 안전이라는 두 마리 토끼를 완벽하게 잡아냈습니다.
