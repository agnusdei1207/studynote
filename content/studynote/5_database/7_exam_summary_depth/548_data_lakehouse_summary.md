# 548. 데이터 레이크하우스(Data Lakehouse) - 분석 인프라의 최종 진화

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 레이크하우스는 **데이터 레이크의 저렴하고 유연한 저장 능력과 데이터 웨어하우스의 강력한 관리 기능(ACID, 스키마 제어)을 단일 시스템으로 결합**한 차세대 데이터 아키텍처다.
> 2. **가치**: 정형/비정형 데이터를 한곳에 모으면서도 트랜잭션 무결성을 보장하여, 별도의 데이터 이동(ETL) 없이 **BI 분석과 머신러닝(ML)을 동시에 수행**하게 한다.
> 3. **융합**: Delta Lake, Apache Iceberg, Hudi와 같은 오픈 테이블 포맷 기술과 클라우드 오브젝트 스토리지가 융합되어, "Schema-on-Read"의 유연성과 "Schema-on-Write"의 견고함을 모두 실현한다.

+++

### Ⅰ. 레이크하우스의 등장 배경: DW와 DL의 한계 극복

- **DW (Data Warehouse)**: 정형 데이터 위주, 고성능이나 비쌈, 비정형 데이터(이미지 등) 분석에 취약.
- **DL (Data Lake)**: 모든 데이터 저장 가능, 저렴하나 관리 부재 시 '데이터 늪(Swamp)'으로 변질, 트랜잭션 지원 안 됨.
- **Lakehouse의 해법**: 데이터 레이크의 물리적 기반 위에 **메타데이터 계층**을 얹어 DW의 관리 기능을 이식.

+++

### Ⅲ. 핵심 기술 요소: 메타데이터 및 테이블 포맷

1. **ACID 트랜잭션**: 오브젝트 스토리지 파일들에 대해 여러 사용자가 동시에 읽고 쓸 때 데이터 일관성을 보장합니다.
2. **스키마 강제 및 진화**: 데이터가 들어올 때 품질을 검증하고, 비즈니스 변화에 따라 스키마를 안전하게 변경합니다.
3. **타임 트래블 (Time Travel)**: 데이터의 변경 이력을 관리하여 과거 특정 시점의 데이터로 되돌아가거나 조회할 수 있습니다.
4. **증분 처리 (Upsert)**: 데이터 레이크에서도 RDB처럼 특정 행만 수정하거나 추가하는 기능을 효율적으로 제공합니다.

- **📢 섹션 요약 비유**: 레이크하우스는 **'바코드가 달린 만능 거대 창고'**와 같습니다. 예전에는 잡동사니를 다 던져넣는 '마당(레이크)'과 금고에 보물만 넣는 '은행(DW)'이 따로 있었습니다. 레이크하우스는 마당처럼 넓고 싸면서도, 모든 물건에 바코드(스키마)를 붙여 목록을 관리하고 도둑(데이터 오염)을 막는 보안 시스템(ACID)까지 갖춘 최첨단 창고입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Delta Lake]**: 레이크하우스 개념을 선도한 대표적 오픈소스 프레임워크.
- **[Medallion Architecture]**: Bronze(Raw) -> Silver(Filtered) -> Gold(Aggregated)로 이어지는 정제 단계.
- **[Decoupled Storage]**: 스토리지와 컴퓨팅이 분리된 클라우드 네이티브의 특징을 계승.

📢 **마무리 요약**: **Data Lakehouse**는 데이터 엔지니어링의 정착지입니다. 파편화된 분석 인프라를 하나로 통합하여, 기업이 데이터의 양과 질 사이에서 더 이상 방황하지 않게 돕는 완벽한 플랫폼입니다.
