# 592. 섀도우 페이징(Shadow Paging) - 로그 없는 원자성의 구현

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 섀도우 페이징은 트랜잭션 수행 시 원본 페이지 테이블(Shadow)은 보존하고, **복사본인 현 페이지 테이블(Current)에만 변경 사항을 반영하다가 커밋 시점에 두 테이블을 교체하는 회복 기법**이다.
> 2. **가치**: 복잡한 로그 기록(Redo/Undo) 오버헤드를 제거하여 **롤백 속도를 극대화**하며, 장애 발생 시 단순히 포인터만 그림자 페이지로 돌리면 되므로 즉각적인 복구가 가능하다.
> 3. **융합**: 현대의 Copy-on-Write(COW) 파일 시스템 사상과 융합되어 있으며, 단일 사용자 환경이나 특정 임베디드 데이터베이스 아키텍처의 핵심 무결성 기술로 사용된다.

+++

### Ⅰ. 섀도우 페이징의 작동 원리

1. **테이블 복제**: 트랜잭션 시작 시, 현재의 페이지 테이블(디렉토리)을 복사하여 **Current Page Table**을 만듭니다.
2. **변경 및 쓰기**: 데이터 변경 시 원본 블록을 덮어쓰지 않고, 새로운 빈 블록에 데이터를 쓴 뒤 **Current Table**의 포인터만 바꿉니다. (Shadow Table은 여전히 원본을 가리킴) ✅
3. **COMMIT**: **Current Table**을 디스크에 영구 저장하고, 이를 새로운 **Shadow Table**로 승격시킵니다.
4. **ROLLBACK/장애**: 그냥 **Current Table**을 버리고 기존 **Shadow Table**을 계속 사용합니다. (복구 시간 0) ✅

+++

### Ⅱ. 섀도우 페이징 구조 시각화 (ASCII Model)

```text
[ Shadow Paging Mechanism ]

  [ Shadow Directory ] (Stable)      [ Current Directory ] (Working)
  ┌───┬────────────┐                 ┌───┬────────────┐
  │ 1 │  Block 10  │                 │ 1 │  Block 10  │
  │ 2 │  Block 20  │ ──▶ (Modify 2) ──▶ │ 2 │  Block 99* │ ◀── New Block!
  │ 3 │  Block 30  │                 │ 3 │  Block 30  │
  └───┴────────────┘                 └───┴────────────┘
          │                                   │
          └─────▶ [ Disk Blocks ] ◀────────────┘
                  [10][20][30]...[99*]
  
  * 장애 발생 시: Current를 버리고 Shadow(10, 20, 30)로 즉시 복구 가능.
```

+++

### Ⅲ. 장점과 기술적 한계

- **장점**: 
    - 로그 파일 관리 및 체크포인트 오버헤드가 없습니다.
    - 롤백과 장애 복구가 포인터 변경만으로 끝나 매우 빠릅니다.
- **한계**: 
    - **데이터 파편화**: 새로운 블록에 계속 쓰기 때문에 데이터가 디스크에 흩어집니다.
    - **동시성 저하**: 여러 트랜잭션이 동시에 테이블을 교체하기가 매우 복잡하여 대규모 다중 사용자 환경에는 부적합합니다. (주로 Single Transaction 위주)

- **📢 섹션 요약 비유**: 섀도우 페이징은 **'시험지 두 장 쓰기'**와 같습니다. 한 장은 완벽하게 쓴 정답지(Shadow)로 보관하고, 다른 한 장(Current)에 연필로 답을 고쳐가며 풉니다. 다 풀고 확신이 서면(Commit) 그 종이를 제출(교체)하고, 틀렸으면(Rollback) 그냥 낙서한 종이를 찢어버리고 원래의 깨끗한 정답지를 그대로 쓰는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Atomicity]**: 섀도우 페이징이 지향하는 가장 완벽한 트랜잭션 속성.
- **[No Log Recovery]**: 별도의 로그 없이 테이블 포인터만으로 복구하는 철학.
- **[Copy-on-Write]**: 섀도우 페이징의 현대적 진화 형태.

📢 **마무리 요약**: **Shadow Paging**은 로그의 복잡함을 포인터의 논리로 풀어낸 우아한 기법입니다. 비록 범용성은 낮으나, 특정 환경에서 절대적인 정합성과 복구 속도를 보장하는 고전적 지혜입니다.
