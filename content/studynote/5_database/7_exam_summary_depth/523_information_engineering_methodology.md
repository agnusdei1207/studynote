# 523. 정보 공학 방법론(IE) - 데이터 주도적 설계의 고전

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 정보 공학 방법론(Information Engineering)은 기업의 정보 시스템을 구축할 때 프로세스보다 **데이터(Information)를 우선시**하며, 상향식(Bottom-up) 데이터 통합과 하향식(Top-down) 업무 분석을 결합한 공학적 접근 방식이다.
> 2. **가치**: "프로세스는 변해도 데이터 구조는 변하지 않는다"는 철학을 바탕으로, 데이터의 독립성과 일관성을 확보하여 장기적으로 유지보수가 용이한 엔터프라이즈 시스템을 구축하게 한다.
> 3. **융합**: ISP(전략 계획) 단계부터 데이터 모델링, 프로세스 모델링, 상관 분석(CRUD Matrix) 기술이 융합되어 시스템 개발의 전 생애주기를 체계적으로 관리한다.

+++

### Ⅰ. 정보 공학 방법론의 4단계 생명주기

1. **정보 전략 계획 (ISP)**: 기업의 비즈니스 목표를 분석하고 전사적인 데이터 모델의 청사진을 수립.
2. **업무 영역 분석 (BA)**: 현업의 구체적인 업무 규칙을 파악하여 데이터 모델(ERD)과 프로세스 모델을 상세화.
3. **업무 시스템 설계 (BSD)**: 분석된 논리 구조를 바탕으로 물리적 DB 설계 및 사용자 인터페이스 설계 수행.
4. **시스템 시스템 구축 (SC)**: 실제 코드를 생성하고 데이터베이스를 생성하여 운영 환경으로 이행.

+++

### Ⅱ. 데이터-프로세스 상관 분석 시각화 (ASCII Model)

정보 공학의 핵심 도구인 **CRUD Matrix**입니다.

```text
[ CRUD Matrix (Data vs Process) ]

               | Customer | Order | Product | Stock | (Entities)
  -------------|----------|-------|---------|-------|
  Create Order |    R     |   C   |    R    |   U   |
  Register Cust|    C     |       |         |       |
  Cancel Order |    R     |   D   |         |   U   |
  Check Stock  |          |       |    R    |   R   |
  (Processes)

  * C: Create, R: Read, U: Update, D: Delete
  * "모든 엔티티는 반드시 최소 한 번의 C가 있어야 함" ✅
```

+++

### Ⅲ. 실무적 의의: 왜 데이터 주도인가?

- **안정성**: 업무 절차(프로세스)는 법규나 정책에 따라 수시로 변하지만, '고객', '상품' 같은 핵심 데이터 구조는 매우 견고합니다. 데이터를 먼저 설계하면 변화에 강한 시스템이 됩니다.
- **통합성**: 부서마다 제각각인 시스템을 하나의 전역 데이터 모델(Global Schema)로 묶어 중복을 방정합니다.
- **자동화**: CASE 도구를 활용하여 모델로부터 코드를 자동 생성하는 생산성 향상을 꾀합니다.

- **📢 섹션 요약 비유**: 정보 공학 방법론은 **'건물의 뼈대(철근)를 먼저 세우는 것'**과 같습니다. 인테리어(프로세스)는 거주자의 취향에 따라 언제든 바뀔 수 있지만, 건물의 뼈대(데이터 구조)가 튼튼하지 않으면 나중에 리모델링조차 불가능해집니다. 처음부터 튼튼한 뼈대를 설계하여 수십 년간 안전하게 건물을 쓸 수 있게 하는 정통 건축 공법과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[ISP (Information Strategy Planning)]**: IE의 출발점인 전략 수립 단계.
- **[ERD (Entity Relationship Diagram)]**: 데이터 주도 설계의 핵심 결과물.
- **[CASE Tool]**: 정보 공학의 복잡한 설계를 자동화해 주는 도구.

📢 **마무리 요약**: **Information Engineering**은 현대 데이터 아키텍처의 근간입니다. "데이터가 먼저다"라는 확고한 원칙을 통해, 파편화된 정보를 기업의 전략적 자산으로 승화시키는 정교한 방법론입니다.
