# 509. ABAC(Attribute-Based Access Control) - 속성 기반의 지능형 통제

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ABAC은 사용자의 직책(Role)뿐만 아니라 사용자의 특성, 데이터의 보안 등급, 접속 시간 및 위치 등 **다양한 속성(Attribute)들의 조합과 논리적 정책을 사용하여 동적으로 접근 권한을 결정하는 보안 모델**이다.
> 2. **가치**: "평일 업무 시간에, 본사 사무실에서만, 1급 기밀 데이터에 접근 가능"과 같은 **정교하고 맥락 중심적인(Context-aware) 보안 정책**을 구현하여 보안 사고의 위험을 극도로 낮춘다.
> 3. **융합**: 제로 트러스트(Zero Trust) 아키텍처의 핵심 엔진으로 융합되며, 클라우드 IAM 및 차세대 DB 보안 게이트웨이의 표준 인증 방식으로 자리 잡았다.

+++

### Ⅰ. ABAC의 4대 주요 속성 (Attributes)

1. **주체 속성 (Subject)**: 사용자의 부서, 직급, 참여 프로젝트, 인증 강도(MFA 여부).
2. **자원 속성 (Resource)**: 데이터의 민감도 등급, 소유자, 데이터 생성일, 파일 형식.
3. **환경 속성 (Environment)**: 현재 시간, 접속 지역(IP), 기기 보안 상태, 네트워크 유형.
4. **행위 속성 (Action)**: 읽기, 쓰기, 삭제, 승인 등 수행하려는 연산의 종류.

+++

### Ⅱ. ABAC 정책 결정 매커니즘 시각화 (ASCII Model)

단순한 역할 체크가 아닌 '조건문 연산'의 결과로 권한이 부여됩니다.

```text
[ ABAC Intelligence Network ]

  (User: Agent_K) ──▶ [ Request: SELECT Secret_Table ]
                             │
          ┌──────────────────▼──────────────────┐
          │  Policy Decision Point (PDP)        │ ✅ Check Attributes
          │ ----------------------------------  │
          │ IF (Subject.Role == 'Manager') AND  │
          │    (Env.Time IN 'Business_Hours')   │
          │    AND (Env.IP IN 'HQ_Range')       │
          │    AND (Resource.Level <= User.Lvl) │
          └──────────────────┬──────────────────┘
                             │
          ┌──────────────────┴──────────────────┐
          ▼ (Decision: Allow)                   ▼ (Decision: Deny)
  [ ACCESS GRANTED! ] ✅                [ ACCESS BLOCKED! ] ❌
```

### Ⅲ. RBAC vs ABAC 상세 비교

| 비교 항목 | RBAC (Role-based) | ABAC (Attribute-based) |
|:---|:---|:---|
| **판단 기준** | 정적인 직책 (역할) | **동적인 속성 조합 (Context)** |
| **정책 유연성** | 낮음 (역할 폭발 문제 발생) | **매우 높음 (세밀한 조건 정의)** |
| **관린 편의성** | 소규모 조직에 유리 | **대규모/복잡한 조직에 필수** |
| **성능** | 매우 빠름 (단순 매칭) | 정책 연산 오버헤드 존재 |

- **📢 섹션 요약 비유**: ABAC은 **'지능형 도어락'**과 같습니다. RBAC이 '열쇠만 있으면 누구든 문을 여는 것'이라면, ABAC은 열쇠가 있어도 "지금 밤 12시가 넘었나?", "함께 있는 사람이 등록된 사람인가?", "집안에 화재가 났나?" 등을 종합적으로 따져서 문을 열어줄지 말지 결정하는 것과 같습니다. 조건이 까다롭지만 그만큼 완벽하게 집(데이터)을 지켜줍니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Policy as Code]**: 보안 정책을 코드로 관리하고 배포하는 방식.
- **[XACML]**: ABAC 정책을 표현하는 표준 XML 언어.
- **[Context-aware]**: 주변 환경을 인지하여 스스로 판단하는 보안의 지능화.

📢 **마무리 요약**: **ABAC**은 보안의 미래입니다. 단순한 '누가'를 넘어 '언제, 어디서, 어떻게'까지 통제하는 정밀한 정책망을 통해, 데이터 중심 시대의 완벽한 거버넌스를 실현합니다.
