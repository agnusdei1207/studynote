# 521. 동적 SQL(Dynamic SQL) - 런타임 질의의 유연성

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동적 SQL은 프로그램 실행 시점에 사용자의 입력이나 비즈니스 로직에 따라 **SQL 문장을 실시간으로 조립하여 실행하는 기술**이다.
> 2. **가치**: 검색 조건이 가변적인 조회 화면 등에서 **유연한 대응**이 가능하게 하며, 컴파일 시점에 쿼리 구조가 고정되는 정적 SQL의 한계를 극복한다.
> 3. **융합**: MyBatis, JPA와 같은 프레임워크의 동적 쿼리 엔진과 융합되어 사용되나, 잘못된 조립은 **SQL 인젝션** 보안 위협과 하드 파싱 부하를 유발하므로 바인드 변수 사용이 필수적이다.

+++

### Ⅰ. 정적 SQL vs 동적 SQL 비교

| 비교 항목 | 정적 SQL (Static) | 동적 SQL (Dynamic) |
|:---|:---|:---|
| **생성 시점** | 컴파일 시점에 확정 | **실행 시점(Runtime)에 확정** |
| **유연성** | 낮음 (구조 고정) | **매우 높음 (필드/조건 가변)** |
| **성능** | 최적화된 상태 유지 | 매번 파싱 부하 발생 가능 (하드 파싱) |
| **보안** | 상대적으로 안전 | **SQL 인젝션 위험 상존** |

+++

### Ⅱ. 동적 SQL 조립 매커니즘 시각화 (ASCII Model)

```text
[ Dynamic SQL Assembly Flow ]

  (Application) : "Search User where City='Seoul' and Age > 20"
       │
       ▼ [ SQL Builder / Parser ]
  ┌──────────────────────────────────────────────────────────┐
  │ 1. Base Query: "SELECT * FROM User WHERE 1=1"            │
  │ 2. Add Condition: " AND City = ?" (Value: 'Seoul')       │ ✅
  │ 3. Add Condition: " AND Age > ?"  (Value: 20)            │
  └──────────────┬───────────────────────────────────────────┘
                 │ (Final SQL: SELECT ... WHERE City=? AND Age>?)
                 ▼
        [ Database Engine ] ──▶ (Binding Values) ──▶ ✅ Execute
```

+++

### Ⅲ. 실무적 의의: 최적화와 보안

- **바인드 변수 필수화**: 동적 SQL 조립 시 문자열을 직접 합치지 말고, 반드시 `?` 나 `:var` 형태의 바인드 변수를 사용하여 **소프트 파싱**을 유도해야 합니다.
- **실행 계획 오염 방지**: 동적 SQL은 조건에 따라 쿼리 구조가 계속 바뀌므로, 비효율적인 실행 계획이 생성되지 않도록 힌트(`HINT`)나 옵티마이저 설정을 정교하게 관리해야 합니다.

- **📢 섹션 요약 비유**: 동적 SQL은 **'즉석 조립 요리'**와 같습니다. 정적 SQL이 미리 만들어진 '완제품 도시락'이라면, 동적 SQL은 손님이 원하는 재료를 담으면 그 자리에서 볶아주는 '마라탕'과 같습니다. 손님의 취향(검색 조건)에 완벽히 맞출 수 있어 인기가 좋지만, 요리사(DBMS) 입장에서는 매번 레시피(실행 계획)를 고민해야 하는 수고가 따릅니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Soft Parsing]**: 바인드 변수를 통해 동적 SQL의 성능을 보장하는 기술.
- **[SQL Injection]**: 동적 SQL 조립 시 가장 주의해야 할 보안 취약점.
- **[Native Dynamic SQL]**: DBMS가 자체적으로 지원하는 동적 쿼리 실행 명령어 (EXECUTE IMMEDIATE 등).

📢 **마무리 요약**: **Dynamic SQL**은 현대 애플리케이션의 필수 역량입니다. 유연함이라는 달콤한 열매를 얻기 위해, 보안과 성능이라는 두 마리 토끼를 놓치지 않는 정교한 조립 기술이 필요합니다.
