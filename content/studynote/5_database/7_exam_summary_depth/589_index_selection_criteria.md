# 589. 인덱스 선정 기준 - 선택도와 카디널리티의 조화

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인덱스 선정은 테이블의 전체 데이터 중 **원하는 행을 얼마나 좁은 범위로 걸러낼 수 있는가(선택도)**와 **데이터 값의 종류가 얼마나 다양한가(카디널리티)**를 분석하여 최적의 검색 경로를 설계하는 작업이다.
> 2. **가치**: 불필요한 전체 테이블 스캔(Full Scan)을 방지하여 디스크 I/O를 획기적으로 줄이며, 동시에 무분별한 인덱스 생성으로 인한 **DML(삽입/수정/삭제) 성능 저하를 예방**한다.
> 3. **융합**: 비즈니스 쿼리의 빈도 분석과 컬럼의 데이터 분포 통계가 융합되어, 최소한의 인덱스로 최대한의 조회 효율을 뽑아내는 전략적 배치를 완성한다.

+++

### Ⅰ. 인덱스 선정의 2대 지표

1. **선택도 (Selectivity)**:
    - **정의**: 특정 조건에 의해 걸러지는 행의 비율. (`선택도 = 1 / Num_Distinct`)
    - **기준**: 보통 선택도가 **5~10% 이내**인 컬럼이 인덱스 후보로 적합합니다. (값이 잘 걸러질수록 좋음)
2. **카디널리티 (Cardinality)**:
    - **정의**: 해당 컬럼이 가진 고유한 값의 개수.
    - **기준**: **카디널리티가 높을수록**(중복이 적을수록) 인덱스 효율이 극대화됩니다. (예: 주민번호 > 성별) ✅

+++

### Ⅱ. 인덱스 효율성 시각화 (ASCII Model)

```text
[ Index Efficiency: Comparison ]

  1. High Cardinality (Good) ✅         2. Low Cardinality (Bad) ❌
     (Column: User_ID)                   (Column: Gender)
     ┌──────┬──────┬──────┐              ┌──────┬──────┬──────┐
     │ 101  │ 102  │ 103  │              │ Male │ Male │ Fem. │
     ├──────┼──────┼──────┤              ├──────┼──────┼──────┤
     │ 104  │ 105  │ 106  │              │ Fem. │ Male │ Fem. │
     └──────┴──────┴──────┘              └──────┴──────┴──────┘
     * 인덱스 타면 딱 1건만 찾음! ✅       * 인덱스 타도 절반을 읽어야 함! 💥
```

+++

### Ⅲ. 최적의 인덱스 설계 원칙

- **Leading Column 선정**: 결합 인덱스 구성 시, 조건절에 가장 자주 쓰이고 선택도가 좋은 컬럼을 **맨 앞(Leading)**에 배치합니다.
- **수정 빈도 고려**: 삽입/삭제가 빈번한 테이블에 인덱스를 너무 많이 만들면 데이터 변경 시마다 인덱스 재정렬 부하가 발생합니다.
- **조인 속성 우선**: `JOIN`의 연결 고리가 되는 컬럼은 인덱스 생성을 최우선으로 검토합니다.

- **📢 섹션 요약 비유**: 인덱스 선정은 **'전화번호부 색인 만들기'**와 같습니다. 이름(고카디널리티)으로 색인을 만들면 사람을 한 번에 찾을 수 있어 아주 유용하지만, '사는 구(저카디널리티)'로 색인을 만들면 한 구에 사는 사람이 너무 많아 결국 수천 명을 다 뒤져야 하므로 색인의 의미가 퇴색되는 것과 같습니다. 가장 확실하게 한 명을 콕 집어낼 수 있는 정보부터 인덱스를 만드는 것이 지혜입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Selectivity]**: 데이터를 얼마나 잘 걸러내는가에 대한 척도.
- **[Covering Index]**: 테이블 접근 없이 인덱스만으로 쿼리를 끝내는 최강의 튜닝 기법.
- **[Clustering Factor]**: 인덱스 순서와 물리 데이터 순서의 일치도 (인덱스 효율의 숨은 변수).

📢 **마무리 요약**: **Index Selection**은 시스템의 기동성을 결정합니다. 데이터의 성질을 꿰뚫어 보고 가장 날카로운 검색 칼날(인덱스)을 갈아두는 것이 데이터베이스 튜닝의 정수입니다.
