# 484. ELT(Extraction, Loading, Transformation) - 클라우드 분석의 표준

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ELT는 데이터를 추출(Extract)한 뒤 **변환 과정 없이 즉시 목적지(Target)에 적재(Load)하고, 이후 타겟 시스템의 강력한 자원을 사용하여 변환(Transform)하는 현대적 데이터 통합 방식**이다.
> 2. **가치**: 중간 변환 단계의 병목을 제거하여 대규모 데이터를 초고속으로 수집하며, 원시 데이터(Raw Data)를 보존함으로써 필요할 때 언제든 다양한 방식으로 재분석할 수 있는 유연성을 제공한다.
> 3. **융합**: Snowflake, BigQuery와 같은 클라우드 네이티브 DW의 대규모 병렬 처리(MPP) 기술과 융합되어, 수 테라바이트의 데이터 변환을 단 몇 초 만에 끝내는 성능 혁신을 실현한다.

+++

### Ⅰ. ELT의 작동 프로세스

1. **Extract (추출)**: 소스 시스템에서 데이터를 가져옵니다.
2. **Load (적재)**: 변환 없이 타겟 시스템(데이터 레이크 또는 클라우드 DW)의 스테이징 영역에 그대로 저장합니다.
3. **Transform (변환)**: 타겟 DB 내부에서 SQL 등을 사용하여 조인, 집계, 정제 작업을 수행합니다. (In-Database Processing)

+++

### Ⅱ. ELT 아키텍처 시각화 (ASCII Model)

```text
[ ELT Pipeline: In-Database Transformation ]

  (Sources)          (Cloud Data Warehouse / Lake)
  ┌────────┐        ┌──────────────────────────────────────┐
  │ RDBMS  │ ──(E)─▶│ 1. Raw Data Layer (S3/GCS)           │
  ├────────┤        │ (원본 데이터 보존 - 유연성 확보)        │
  │ Web Log│ ──(E)─▶│          │                           │
  └────────┘        │          ▼ (2. T: MPP 기반 고속 변환)  │ ✅
                    │ 3. Analytics Layer (Fact/Dim)        │
                    └───────────────────┬──────────────────┘
                                        ▼
                               (BI / Data Science)
```

+++

### Ⅲ. 왜 ELT가 ETL보다 우수한가?

- **확장성**: 별도의 비싼 ETL 서버를 증설할 필요 없이, 클라우드 DB의 컴퓨팅 노드를 늘려 연산 능력을 무한히 확장할 수 있습니다.
- **데이터 신선도**: 적재 단계가 단순하므로 데이터가 소스에서 분석 시스템까지 도달하는 시간이 획기적으로 단축됩니다.
- **스키마 유연성**: 'Schema-on-Read' 철학을 반영하여, 분석 시점에 필요한 스키마를 동적으로 생성하기 유리합니다.

- **📢 섹션 요약 비유**: ELT는 **'대형 뷔페의 즉석 요리 코너'**와 같습니다. 주방에서 미리 다 조리해서 내놓는 것(ETL)이 아니라, 신선한 재료를 손님 앞 테이블(Target)에 먼저 가져다 놓고(Load), 손님이 원하는 시점에 원하는 조리법으로 즉석에서 볶아 먹는(Transform) 것과 같습니다. 재료가 신선하게 살아있으므로 나중에 마음이 바뀌어도 다른 요리로 바꿔 만들 수 있는 장점이 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Modern Data Stack]**: ELT를 기반으로 구축된 최신 데이터 엔지니어링 체계.
- **[In-Database Processing]**: 데이터를 밖으로 빼지 않고 내부 엔진으로 요리하는 기술.
- **[dbt]**: ELT의 변환(T) 과정을 관리하는 현대적 코드 기반 도구.

📢 **마무리 요약**: **ELT**는 클라우드 시대의 데이터 처리 문법입니다. 인프라의 한계를 기술로 정복하여, "일단 저장하고 나중에 고민하라"는 데이터 중심의 혁명을 완성합니다.
