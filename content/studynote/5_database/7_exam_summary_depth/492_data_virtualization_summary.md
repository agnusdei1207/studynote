# 492. 데이터 가상화(Data Virtualization) - 제로 카피의 미학

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 가상화는 데이터를 물리적으로 한곳에 모으지 않고, **논리적인 가상 계층을 통해 여러 원천 시스템의 데이터를 마치 하나의 DB처럼 통합 조회하는 기술**이다.
> 2. **가치**: 데이터를 복제하지 않는 **'제로 카피(Zero Copy)'** 통합을 통해 스토리지 비용을 절감하며, 실시간으로 원천 데이터를 조회하여 최상의 데이터 신선도를 보장한다.
> 3. **융합**: 하이브리드 클라우드 환경에서 흩어진 데이터 사일로(Silo)를 연결하는 추상화 레이어로 융합되며, 복잡한 ETL 과정 없이 빠른 분석 환경을 구축한다.

+++

### Ⅰ. 데이터 가상화의 핵심 프로세스

1. **Connect**: 다양한 소스(RDB, NoSQL, API, CSV)에 어댑터를 통해 접근합니다.
2. **Abstract**: 각 소스의 물리적 형식을 무시하고 공통된 논리적 모델로 변환합니다.
3. **Federate**: 여러 소스의 데이터를 실시간으로 조인하거나 결합하여 가상 뷰를 생성합니다.
4. **Deliver**: 사용자가 쿼리하면, 엔진이 실시간으로 소스에 질의하여 결과를 취합해 반환합니다.

+++

### Ⅱ. 가상화 아키텍처 시각화 (ASCII Model)

```text
[ Data Virtualization: Zero Copy Flow ]

      (User / App) : "SELECT * FROM Integrated_View"
                │
      ┌─────────▼─────────┐
      │ Virtualization    │ (Query Push-down & Optimization)
      │ Layer (Logical)   │ ✅ "No physical storage needed"
      └─────────┬─────────┘
                │ (Request only relevant data)
      ┌─────────┴─────────┬──────────────┐
      ▼                   ▼              ▼
  [ SQL DB ]          [ NoSQL ]      [ Cloud API ]
 (Local Data)        (Raw Data)     (SaaS Data)
```

+++

### Ⅲ. 실무적 의점과 선택의 기로

- **장점**: 인프라 구축 시간이 며칠 내로 짧으며, 데이터 중복 동기화에 따른 정합성 고민이 없습니다.
- **도전 과제**: 실시간으로 원천 DB에 질의를 던지므로 소스 시스템에 부하가 갈 수 있으며, 대량 데이터 조인 시 네트워크 지연(Latency)이 발생할 수 있습니다.
- **해결**: 자주 조회되는 가상 뷰 결과에 대해 **메모리 캐싱** 기술을 병행하여 성능을 보완합니다.

- **📢 섹션 요약 비유**: 데이터 가상화는 **'전 세계 도서관을 연결한 통합 온라인 검색대'**와 같습니다. 책을 우리 집으로 다 사 올(ETL) 필요 없이, 검색창에 입력만 하면 각 도서관에 있는 책 내용을 실시간으로 화면에 띄워주는 것과 같습니다. 책은 각 도서관(Source)에 그대로 있지만, 나는 내 방에서 전 세계의 지식을 한꺼번에 읽는 셈입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Zero Copy]**: 데이터 복제 없이 연결만으로 가치를 만드는 철학.
- **[Logical DW]**: 가상화 기술로 구현된 논리적 데이터 창고.
- **[Push-down Query]**: 연산을 소스 DB 쪽에서 수행하게 하여 전송량을 줄이는 기술.

📢 **마무리 요약**: **Data Virtualization**은 데이터 통합의 지름길입니다. 물리적 통합의 무거움을 벗어던지고, 논리적 연결을 통해 비즈니스의 민첩성을 극대화합니다.
