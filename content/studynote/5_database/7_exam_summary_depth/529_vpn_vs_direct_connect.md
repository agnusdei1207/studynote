# 529. DB 연동 인프라(VPN vs DX) - 안전한 데이터의 통로

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하이브리드 클라우드 환경에서 온프레미스와 클라우드 DB를 연결하기 위한 기술로, **공인망을 암호화하여 터널링하는 VPN 방식**과 **물리적인 전용 회선을 직접 연결하는 전용선(Direct Connect) 방식**이 있다.
> 2. **가치**: 데이터베이스 연동 시 가장 민감한 **보안성과 네트워크 지연 시간(Latency)** 문제를 해결하며, 대규모 데이터 마이그레이션 및 실시간 동기화(CDC)의 안정적인 파이프라인을 제공한다.
> 3. **융합**: 고가용성 설계를 위해 전용선을 주 회선으로, VPN을 백업 회선으로 구성하는 하이브리드 연결 전략으로 융합되어 기업의 비즈니스 연속성을 보장한다.

+++

### Ⅰ. VPN과 전용선(DX)의 상세 비교

| 비교 항목 | IPsec VPN | 전용선 (Direct Connect / FastConnect) |
|:---|:---|:---|
| **물리적 경로** | 공인 인터넷망 공유 | **독립된 전용 물리 회선** |
| **보안 방식** | 데이터 암호화 터널링 | **물리적 격리 (Physical Isolation)** |
| **성능 (지연)** | 가변적 (인터넷 혼잡도 영향) | **고정적 / 매우 낮음 (Low Latency)** |
| **비용** | 저렴함 | 비쌈 (회선료 + 포트 사용료) |
| **구축 기간** | 매우 짧음 (설정 즉시) | 길음 (물리 회선 공사 필요) |


### Ⅲ. 실무적 선택 기준

- **VPN 추천**: 데이터 전송량이 적고 지연 시간에 민감하지 않은 소규모 시스템, 혹은 본선 연결 전 테스트용.
- **전용선 추천**: 대규모 데이터 마이그레이션이 빈번하거나, **금융/공공** 등 극강의 보안과 일관된 응답 성능이 필수인 엔터프라이즈 시스템.
- **최적의 조합**: 전용선을 주 회선으로 쓰되, 통신사 공사나 단선 사고에 대비해 저렴한 VPN을 상시 대기(Standby) 시키는 이중화 구성이 표준입니다.

- **📢 섹션 요약 비유**: DB 연동 인프라는 **'회사의 출퇴근길'**과 같습니다. VPN은 '공용 도로(인터넷)'에 전용 차선을 그려서 암호화된 버스를 타고 가는 것이고, 전용선은 아예 우리 회사만 쓸 수 있는 '지하 비밀 터널'을 뚫는 것과 같습니다. 공용 도로는 차가 막힐 때(트래픽 혼잡) 늦어질 수 있지만, 전용 터널은 언제나 정해진 시간에 가장 빠르게 도착할 수 있는 안전하고 확실한 길입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Latency]**: DB 성능에 직격탄을 주는 네트워크 지연 시간.
- **[BGP (Border Gateway Protocol)]**: 전용선 연결 시 경로 정보를 주고받는 표준 라우팅 프로토콜.
- **[Encryption]**: VPN의 핵심 보안 기술.

📢 **마무리 요약**: **VPN & Direct Connect**는 데이터의 혈관입니다. 데이터의 양과 비즈니스의 중요도를 고려하여, 끊김 없고 안전한 통로를 설계하는 것이 하이브리드 클라우드 성공의 열쇠입니다.
