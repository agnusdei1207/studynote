# 471. 일관된 해싱(Consistent Hashing) - 분산 리밸런싱의 최적해

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 일관된 해싱은 해시 테이블의 크기가 변할 때(노드 추가/삭제), 전체 키를 재배치하는 대신 **오직 $K/N$(K:키 개수, N:노드 수)만큼의 데이터만 이동하게 하는 영리한 분산 알고리즘**이다.
> 2. **가치**: 서버 증설 시 시스템 전체의 데이터를 옮겨야 하는 '폭풍 재배치' 문제를 해결하여, **무중단 수평 확장(Scale-out)**을 실무적으로 가능케 한다.
> 3. **융합**: 가상 노드(Virtual Nodes) 개념과 결합되어 노드 간 부하 불균형을 해소하며, Cassandra, DynamoDB, Akamai CDN 등 현대 분산 시스템의 아키텍처적 근간이 된다.

+++

### Ⅰ. 일관된 해싱의 핵심 매커니즘: 해시 링(Hash Ring)

1. **링 형성**: 해시 함수의 출력 범위(예: 0~2^32-1)를 원형 링으로 구성합니다.
2. **서버 및 데이터 배치**: 서버 노드들과 데이터 키들을 모두 이 링 위의 특정 지점에 해시값으로 배치합니다.
3. **데이터 매핑**: 특정 키는 링 위에서 **시계 방향**으로 가장 먼저 만나는 서버 노드에 저장됩니다.

+++

### Ⅱ. 노드 추가 시 리밸런싱 시각화 (ASCII Model)

```text
[ Consistent Hashing Ring ]

          (Hash Value: 0)
               ▲
        ┌──────┴──────┐
     [Node 3]      [Node 1]
        │             │
        │      (K1)──▶│ (Key 1은 시계방향으로 
        │             │  가장 가까운 Node 1에 저장)
     [Node 2]      [New Node!] ◀── 신규 서버 추가 시
        └──────┬──────┘           (일부 키만 New Node로 이동, 
               ▼                  나머지 노드는 영향 없음!) ✅
```

+++

### Ⅲ. 가상 노드 (Virtual Nodes)의 도입 이유

- **문제점**: 서버가 링 위에 불균형하게 배치되면 특정 서버가 너무 많은 데이터를 담당하는 '쏠림' 현상이 발생합니다.
- **해결책**: 하나의 물리 서버를 링 위의 수백 개 가상 지점(Virtual Nodes)에 분산 배치하여, 데이터가 전체 링에 수학적으로 골고루 퍼지도록 유도합니다.

- **📢 섹션 요약 비유**: 일관된 해싱은 **'의자 뺏기 게임의 역발상'**과 같습니다. 사람이 늘어나거나 줄어들 때 모든 사람이 자리를 옮겨야 하는 게 일반 해싱이라면, 일관된 해싱은 내 옆에 의자가 하나 더 생기면 나만 살짝 옆으로 옮겨 앉고 다른 사람들은 그대로 앉아있게 하는 효율적인 자리 배치법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Rebalancing]**: 노드 변화에 따른 데이터 이동 최소화 과정.
- **[Hash Space]**: 링으로 표현되는 무한한 해시 값의 공간.
- **[Scalability]**: 일관된 해싱을 통해 얻게 되는 무한한 확장성.

📢 **마무리 요약**: **Consistent Hashing**은 분산 시스템의 '평화 유지군'입니다. 노드의 변화라는 거친 풍랑 속에서도 데이터의 위치를 안정적으로 지켜주는 핵심 지혜입니다.
