# 507. 동적 데이터 마스킹(Dynamic Data Masking) - 인가 기반의 가시성 제어

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동적 데이터 마스킹(DDM)은 데이터베이스에 저장된 원본 데이터는 그대로 둔 채, **사용자의 권한이나 컨텍스트에 따라 쿼리 실행 결과에서만 민감 정보를 가리고(Masking) 보여주는 실시간 보안 기술**이다.
> 2. **가치**: 불필요한 데이터 복제 없이 운영계 DB에서 직접 **미세 권한 제어(Fine-grained Access Control)**를 수행하며, 인가되지 않은 내부자의 실수나 악의적인 유출로부터 데이터를 즉각 보호한다.
> 3. **융합**: 가상 프라이빗 데이터베이스(VPD)나 속성 기반 접근 제어(ABAC) 기술과 융합되어, "누가, 언제, 어디서" 조회하느냐에 따라 데이터의 노출 수준을 동적으로 결정하는 지능형 보안 레이어로 작동한다.

+++

### Ⅰ. 동적 마스킹의 작동 매커니즘

1. **사용자 요청**: 사용자가 SQL 조회를 수행합니다.
2. **정책 확인**: DB 엔진은 사용자의 세션 정보(Role, IP, Time 등)를 보안 정책과 대조합니다.
3. **실시간 변환**: 인가되지 않은 정보라면, 데이터를 전송하기 직전에 메모리 상에서만 마스킹 처리합니다.
    - 예: `900101-1234567` ──▶ `900101-1******` ✅
4. **결과 반환**: 마스킹된 가짜 데이터를 최종 응답으로 보냅니다. (DB 원본은 여전히 `1234567` 보존)

+++

### Ⅱ. DDM 아키텍처 및 권한별 뷰 시각화 (ASCII Model)

```text
[ Dynamic Data Masking Workflow ]

  [ Physical Storage ] : Original Data [ 010-1234-5678 ]
           │
           ▼ [ Database Engine / DDM Layer ]
  ┌────────────────────────────────────────────────────────┐
  │  Security Policy: IF Role != 'ADMIN' THEN Mask Phone   │ 🔒
  └──────────────────────────┬─────────────────────────────┘
                             │
          ┌──────────────────┴──────────────────┐
          ▼ (User: Admin)                       ▼ (User: Telemarketer)
  [ SELECT Phone FROM ... ]             [ SELECT Phone FROM ... ]
          │                                     │
          ▼                                     ▼
  ✅ [ 010-1234-5678 ]                  ✅ [ 010-1234-**** ] 🔒
     (Full Visibility)                     (Masked Visibility)
```

+++

### Ⅲ. 정적 마스킹(Static) vs 동적 마스킹(Dynamic)

| 비교 항목 | 정적 마스킹 (Static) | 동적 마스킹 (Dynamic) |
|:---|:---|:---|
| **원본 보존** | **원본 파괴** (별도의 복제 DB 생성) | **원본 유지** (조회 시에만 가림) |
| **변경 편의성** | 마스킹 룰 변경 시 재이관 필요 | **정책 수정만으로 즉시 반영** |
| **주요 용도** | 테스트/개발용 데이터셋 배포 | **운영 시스템 실시간 보안** |
| **리소스** | 스토리지 비용 발생 | CPU 연산 비용 발생 |

- **📢 섹션 요약 비유**: 동적 마스킹은 **'방송국의 실시간 음성 변조'**와 같습니다. 원래 목소리(데이터)는 멀쩡하지만, 시청자(조회자)가 누구냐에 따라 실시간으로 목소리 톤을 바꾸어(마스킹) 내보내는 것과 같습니다. 시청자는 진짜 목소리를 알 수 없지만, 녹음실(DB) 안에서는 여전히 진짜 목소리가 보존되고 있는 것과 같은 원리입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Context-Aware]**: 사용자의 접속 상황을 인지하여 보안을 적용하는 성질.
- **[Anonymization]**: 정적 마스킹이 지향하는 완전한 비식별화.
- **[Row Level Security]**: 동적 마스킹과 함께 쓰여 특정 행까지 가리는 기술.

📢 **마무리 요약**: **Dynamic Data Masking**은 보안과 편의성의 완벽한 타협점입니다. 원본 데이터의 가치를 훼손하지 않으면서도, 인가된 사람에게만 진실을 보여주는 가장 스마트한 데이터 가림막입니다.
