# 517. 다크 데이터 보안 통제 - 잊힌 데이터의 위협 관리

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다크 데이터(Dark Data)는 기업이 수집하고 저장했으나 분석에 활용되지 않고 방치된 방대한 데이터 집합으로, **보안 관리의 사각지대에 놓여 개인정보 유출의 뇌관이 될 수 있는 유휴 자산**이다.
> 2. **가치**: 보이지 않는 데이터를 가시화(Discovery)하고 분류(Classification)함으로써 **공격 표면(Attack Surface)을 획기적으로 줄이며**, 스토리지 비용 절감과 컴플라이언스 준수를 동시에 달성한다.
> 3. **융합**: 데이터 스캔 기술, AI 기반 민감 정보 탐지 기술, 데이터 삭제(Sanitization) 기술이 융합되어 제로 트러스트 보안 체계의 실질적인 데이터 보호 영역을 확장한다.

+++

### Ⅰ. 다크 데이터의 위험 요소

- **보안 취약성**: 관리자가 잊어버린 백업 파일, 테스트용 데이터베이스 등은 최신 보안 패치가 적용되지 않아 해커의 주요 표적이 됩니다.
- **컴플라이언스 위반**: 보유 기간이 지난 개인정보가 다크 데이터 속에 남아있을 경우, GDPR이나 개인정보 보호법 위반으로 막대한 과징금이 부과될 수 있습니다.
- **가용성 저하**: 가치 없는 데이터가 스토리지를 점유하여 검색 속도를 늦추고 클라우드 비용을 낭비하게 만듭니다.

+++

### Ⅱ. 다크 데이터 보안 통제 프로세스 (ASCII Model)

숨겨진 위협을 찾아내어 정화하는 과정입니다.

```text
[ Dark Data Remediation Workflow ]

  1. Discovery ──▶ 2. Classification ──▶ 3. Decision ──▶ 4. Execution
  ┌────────────┐   ┌────────────────┐   ┌────────────┐   ┌──────────────┐
  │ Scan all   │   │  PII Detected? │   │ Keep vs    │   │ Delete /     │
  │ Storages   │ ──▶  Sensitive?    │ ──▶ Purge      │ ──▶ Mask /       │ ✅
  │ (S3, DB)   │   │  (AI Analysis) │   │ (Biz Val)  │   │ Archive      │
  └────────────┘   └────────────────┘   └────────────┘   └──────────────┘
```

+++

### Ⅲ. 핵심 통제 기술

1. **자동화된 데이터 탐색 (Auto-Discovery)**: 클라우드 전역을 훑으며 관리되지 않는 버킷이나 DB 인스턴스를 실시간으로 찾아냅니다.
2. **DLP (Data Loss Prevention)**: 네트워크로 나가는 데이터 중 다크 데이터에서 유출된 패턴이 있는지 감시합니다.
3. **데이터 수명 주기 관리 (DLM)**: 일정 기간 활용되지 않는 데이터는 자동으로 격리하거나 삭제하는 정책을 시스템적으로 강제합니다.

- **📢 섹션 요약 비유**: 다크 데이터 보안 통제는 **'창고 대정리'**와 같습니다. 집안 구석구석(데이터 저장소)에 무엇이 들어있는지도 모른 채 쌓아둔 상자(다크 데이터)들은 먼지가 쌓이고 좀이 슬어(보안 위협) 집 전체를 망칠 수 있습니다. 상자를 다 열어서 귀중품은 금고(보안 DB)에 넣고, 쓰레기는 버려야(삭제) 집도 넓어지고 도둑이 들어와도 훔쳐갈 게 없게 만드는 과정입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Data Minimization]**: 필요한 데이터만 남기는 최소화 원칙.
- **[Metadata Management]**: 다크 데이터를 가시화하기 위한 핵심 재료.
- **[Attack Surface]**: 보안 통제를 통해 줄여야 할 공격 가능한 지점들의 총합.

📢 **마무리 요약**: **Dark Data Control**은 보이지 않는 적과의 싸움입니다. 데이터의 양이 아닌 '가치와 안전'을 중심으로 저장소를 재정의할 때, 기업은 진정한 데이터 주권을 확보할 수 있습니다.
