# 396. Mapping 규칙 (개체->테이블) - 논리에서 물리로의 전환

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Mapping 규칙은 개념적 설계 단계인 **ER 모델(Entity-Relationship)을 논리적 설계 단계인 관계형 데이터베이스의 테이블 구조로 변환**하기 위해 정의된 일련의 변환 절차다.
> 2. **가치**: 현실 세계의 추상적인 객체와 관계를 컴퓨터가 이해할 수 있는 2차원 표(Relation)로 누락 없이 이식하며, 데이터 무결성과 정규화 원칙을 유지하는 가교 역할을 한다.
> 3. **융합**: 개체는 테이블로, 속성은 컬럼으로, 관계는 외래키(FK)나 별도 테이블로 융합되어 최종적인 물리 스키마를 완성한다.

+++

### Ⅰ. 핵심 Mapping 규칙 5단계

1. **개체(Entity)의 변환**: 
    - 강한 개체(Strong Entity)는 독립적인 테이블로 생성.
    - 약한 개체(Weak Entity)는 부모의 기본키를 포함한 테이블로 생성.
2. **속성(Attribute)의 변환**:
    - 일반 속성은 테이블의 컬럼으로 변환.
    - 다중값 속성(Multivalued)은 별도의 테이블로 분리 (제1정규화).
3. **1:1 관계의 변환**:
    - 어느 한쪽 테이블에 상대방의 기본키를 외래키(FK)로 추가하거나, 두 테이블을 합침.
4. **1:N 관계의 변환**:
    - 'N' 쪽의 테이블에 '1' 쪽의 기본키를 외래키(FK)로 추가함.
5. **N:M 관계의 변환**:
    - 두 개체의 기본키를 합친 **교차 테이블(Intersection Table)**을 별도로 생성.

+++

### Ⅱ. Mapping 프로세스 시각화 (ASCII Model)

```text
[ Conceptual (ER Model) ]           [ Logical (Relational Table) ]

  ┌────────┐ (1)      (N) ┌────────┐       Table: [ Department ]
  │ Dept   │──────────────│ Emp    │       - Dept_ID (PK)
  └────────┘              └────────┘       - Dept_Name
       │                      │
       ▼                      ▼            Table: [ Employee ]
  [ Relation Mapping Logic ] ──▶──▶        - Emp_ID (PK)
                                           - Emp_Name
                                           - Dept_ID (FK) ✅ "1:N Mapping"
```

+++

### Ⅲ. 실무적 의의: 관계의 물리적 구현

- **무결성 유지**: 매핑 규칙을 정확히 따르면 개체 무결성(PK)과 참조 무결성(FK)이 자연스럽게 설계에 반영됩니다.
- **성능 최적화**: 매핑 과정에서 빈번한 조인을 방지하기 위해 의도적으로 1:1 관계를 통합하는 등 물리적 보정이 이루어집니다.

- **📢 섹션 요약 비유**: Mapping 규칙은 **'청사진을 보고 레고를 조립하는 설명서'**와 같습니다. 머릿속에 있는 건물의 모습(개체와 관계)을 실제로 하나하나의 블록(테이블)으로 어떻게 끼워 맞춰야 튼튼하고 쓰기 편한 집(데이터베이스)이 되는지 알려주는 친절한 가이드라인입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Intersection Entity]**: N:M 관계를 풀기 위해 탄생한 매핑의 산물.
- **[Foreign Key]**: 관계를 물리적으로 잇는 가장 강력한 도구.
- **[Schema Evolution]**: 매핑 이후 변화하는 스키마를 관리하는 기술.

📢 **마무리 요약**: **Mapping Rules**는 데이터베이스 설계의 연금술입니다. 추상의 세계를 물리적 실체로 바꾸는 이 규칙들을 완벽히 이해해야만 빈틈없는 데이터 세상을 구축할 수 있습니다.
