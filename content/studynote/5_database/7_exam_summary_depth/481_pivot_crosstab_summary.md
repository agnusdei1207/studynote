# 481. 피벗과 크로스탭 보고서 - 관점의 전환

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 피벗(Pivot)은 행(Row)으로 나열된 데이터를 **열(Column)로 회전시켜 2차원 교차 도표(Crosstab) 형태로 재구성**하는 데이터 변환 기술이다.
> 2. **가치**: 나열된 개별 트랜잭션 데이터를 한눈에 파악하기 쉬운 **요약 보고서** 형태로 바꾸어 주며, 서로 다른 두 차원 간의 상관관계를 시각적으로 명확히 드러낸다.
> 3. **융합**: SQL의 `PIVOT` 연산자나 `CASE WHEN` 집계 기술이 사용되며, 엑셀의 피벗 테이블과 같은 데이터 분석 도구의 핵심 UI 로직으로 활용된다.

+++

### Ⅰ. 피벗(Pivot)의 정의 및 과정

- **정의**: 데이터의 특정 속성값을 열 머리글로 올리고, 그에 해당하는 집계값(합계, 평균 등)을 채워 넣는 작업입니다.
- **크로스탭 (Crosstab)**: 피벗의 결과로 만들어진 교차 집계표를 의미합니다.
- **방향**: 세로로 긴(Long) 데이터를 가로로 넓은(Wide) 데이터로 변환합니다.

+++

### Ⅱ. 데이터 변환 시각화 (ASCII Model)

```text
[ Source Table: Sales ]           [ Result: Crosstab Report ]
┌──────┬───────┬───────┐          ┌────────┬───────┬───────┬───────┐
│ Month│ Item  │ Sales │          │ Item   │ Jan   │ Feb   │ Mar   │
├──────┼───────┼───────┤          ├────────┼───────┼───────┼───────┤
│ Jan  │ Phone │ 100   │   ──▶    │ Phone  │ 100   │ 150   │ 120   │ ✅
│ Jan  │ Watch │ 50    │   ──▶    │ Watch  │ 50    │ 60    │ 70    │ ✅
│ Feb  │ Phone │ 150   │          └────────┴───────┴───────┴───────┘
└──────┴───────┴───────┘          (각 월이 컬럼으로 변환됨)
```

+++

### Ⅲ. SQL 구현 방식

1. **표준 PIVOT 연산자**: 일부 DBMS(Oracle, MSSQL 등)에서 지원하는 전용 키워드 사용.
2. **CASE WHEN 방식**: 모든 DBMS에서 통용되는 범용적인 방식.
```sql
SELECT Item,
       SUM(CASE WHEN Month='Jan' THEN Sales ELSE 0 END) AS Jan,
       SUM(CASE WHEN Month='Feb' THEN Sales ELSE 0 END) AS Feb
FROM Sales GROUP BY Item;
```

- **📢 섹션 요약 비유**: 피벗은 **'가로세로 낱말 맞추기'**와 같습니다. 낱말들이 한 줄로 길게 적혀 있을 때는 뜻을 알기 힘들지만, 가로축(날짜)과 세로축(품목)을 정해 표를 그리면 두 정보가 만나는 지점의 의미가 명확해지는 것과 같습니다. 데이터를 회전시켜 새로운 시각을 얻는 마법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Unpivot]**: 열로 펼쳐진 데이터를 다시 행으로 모으는 역과정.
- **[Aggregation]**: 피벗 시 반드시 동반되는 합계/평균 등의 연산.
- **[OLAP Operations]**: 피벗팅은 4대 OLAP 연산 중 하나.

📢 **마무리 요약**: **Pivot & Crosstab**은 데이터 분석의 화룡점정입니다. 복잡한 수치들을 체계적인 표로 정리함으로써, 의사결정권자에게 가장 명쾌한 답을 제시하는 리포팅의 정수입니다.
