# 504. 데이터 메시 셀프 서비스 플랫폼 - 자율 분석의 인프라

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 셀프 서비스 데이터 플랫폼은 각 비즈니스 도메인 팀이 복잡한 인프라 설정 없이 **독립적으로 데이터를 수집, 저장, 가공, 배포할 수 있도록 중앙에서 제공하는 자동화된 도구 및 서비스들의 집합**이다.
> 2. **가치**: 중앙 데이터 팀에 대한 기술적 의존성을 제거하여 **데이터 제품(Data Product)의 생산 속도**를 높이며, 도메인 팀이 인프라 관리가 아닌 '비즈니스 가치 도출'에 집중할 수 있게 한다.
> 3. **융합**: 클라우드 네이티브 기술과 '서비스형 데이터(DaaS)' 사상이 융합되어, 클릭 몇 번으로 표준화된 데이터 파이프라인과 거버넌스 정책이 적용된 분석 환경을 즉시 프로비저닝한다.

+++

### Ⅰ. 셀프 서비스 플랫폼의 주요 기능

- **자동 프로비저닝**: 버튼 하나로 새로운 데이터 레이크 저장소나 SQL 엔진 인스턴스를 생성.
- **데이터 카탈로그 서비스**: 도메인이 만든 데이터 제품을 전사 사용자가 검색하고 권한을 요청할 수 있는 마켓플레이스 제공.
- **표준 파이프라인 템플릿**: 검증된 ETL/ELT 워크플로우를 템플릿화하여 도메인 팀에 배포.
- **통합 보안 및 정책 강제**: 데이터 마스킹, 암호화, 접근 제어 등 거버넌스 룰을 시스템적으로 자동 적용.

### Ⅲ. 플랫폼 팀의 역할 변화

- **과거 (중앙 집중)**: 모든 부서의 쿼리를 대신 짜주고 데이터 품질을 책임짐 (병목의 원인).
- **현재 (데이터 메시)**: 도메인 팀이 쿼리를 직접 짜고 품질을 책임질 수 있도록 **가장 쉽고 강력한 '플랫폼'을 만드는 데** 집중.

- **📢 섹션 요약 비유**: 셀프 서비스 플랫폼은 **'스마트 키친이 갖춰진 공유 주방'**과 같습니다. 각 요리사(도메인 팀)는 자기만의 레시피(비즈니스 로직)와 재료(데이터)만 가져오면 됩니다. 불(컴퓨팅), 냄비(저장소), 설거지 시스템(거버넌스)은 이미 완벽하게 준비되어 있어, 요리사는 오직 맛있는 요리(데이터 제품)를 만드는 데만 온 힘을 쏟을 수 있습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Infrastructure as Code (IaC)]**: 플랫폼 자동화의 핵심 기술.
- **[Data Product]**: 플랫폼을 통해 생산되는 최종 가치 단위.
- **[Data Democratization]**: 전문가가 아니어도 데이터를 다룰 수 있게 하는 데이터 민주화.

📢 **마무리 요약**: **Self-serve Data Platform**은 데이터 메시를 실현하는 엔진입니다. 도구의 단순함이 조직의 자율성을 낳고, 그 자율성이 모여 기업의 거대한 데이터 혁신을 완성합니다.
