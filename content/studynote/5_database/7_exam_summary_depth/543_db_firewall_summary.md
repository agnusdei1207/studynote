# 543. 데이터베이스 방화벽(DB Firewall) - 입구부터 차단하는 보안

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터베이스 방화벽은 DB 서버 앞단에서 유입되는 SQL 패킷을 실시간으로 분석하여, **허용되지 않은 사용자나 비정상적인 쿼리(SQL Injection 등)를 사전에 차단하는 네트워크 기반 보안 솔루션**이다.
> 2. **가치**: DB 자체의 기능을 수정하지 않고도 외부에서 **접근 제어, 마스킹, 실시간 로깅**을 수행하며, 인가된 사용자의 권한 오남용을 감시하는 '외부 감시자' 역할을 한다.
> 3. **융합**: 프록시(Proxy) 방식과 스니핑(Sniffing) 방식이 상황에 맞게 융합되어, 성능 저하 없는 정밀 모니터링과 즉각적인 위협 차단을 동시에 구현한다.

+++

### Ⅰ. DB 방화벽의 2대 주요 구현 방식

1. **Proxy (In-line) 방식**:
    - **원리**: 애플리케이션과 DB 사이의 통로에 직접 위치하여 모든 패킷을 중계함.
    - **장점**: 비정상 쿼리를 물리적으로 **즉시 차단** 가능.
    - **단점**: 솔루션 장애 시 서비스 전체가 중단(SPOF)될 수 있고, 미세한 지연 시간 발생.
2. **Sniffing (Mirroring) 방식**:
    - **원리**: 네트워크 스위치에서 복제된 패킷(Mirror)만 가져와 분석함.
    - **장점**: DB 성능 및 서비스에 영향이 **0%**에 가까움.
    - **단점**: 실시간 차단이 어렵고 사후 탐지 및 경고 위주로 작동.

+++

### Ⅲ. 핵심 보안 통제 기능

- **Access Control**: 허용된 IP, MAC 주소, 시간대, 사용자 계정에 의해서만 접속 허용.
- **Query Filtering**: `DROP TABLE`, `DELETE` 등 위험한 명령어나 `WHERE 1=1` 같은 인젝션 패턴 차단.
- **Data Masking**: 특정 조건의 사용자에게는 쿼리 결과를 실시간 변조하여 마스킹된 데이터 반환.
- **Logging & Auditing**: 누가 어떤 데이터를 조회했는지에 대한 위변조 불가능한 감사 증적 보관.

- **📢 섹션 요약 비유**: DB 방화벽은 **'은행 정문의 보안 검색대'**와 같습니다. 프록시 방식은 모든 고객이 금속 탐지기를 통과해야만 들어갈 수 있는 입구(차단 위주)이고, 스니핑 방식은 입구에 CCTV를 달아두고 수상한 사람을 실시간으로 감시하는 것(모니터링 위주)과 같습니다. 검색대가 튼튼해야 은행원(데이터)이 안심하고 일을 할 수 있는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[SQL Injection]**: DB 방화벽이 막아야 할 가장 대표적인 공격.
- **[Audit Trail]**: 법적 증거 능력을 갖는 정교한 행위 기록.
- **[DAM (Database Activity Monitoring)]**: DB 방화벽 기능을 포함한 포괄적 감시 체계.

📢 **마무리 요약**: **DB Firewall**은 데이터 성채의 성문입니다. 철저한 입구 통제와 기록 관리를 통해, 내부와 외부의 모든 위협으로부터 데이터 자산을 보호하는 필수 방어막입니다.
