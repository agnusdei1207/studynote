# 406. 참조 무결성 (Referential Integrity) - 관계의 끈을 지키다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 참조 무결성은 두 테이블 간의 관계를 맺을 때, **자식 테이블의 외래키(Foreign Key) 값이 반드시 부모 테이블의 기본키(Primary Key)로 존재해야 하거나 NULL이어야 한다**는 제약 조건이다.
> 2. **가치**: 존재하지 않는 부모 데이터를 참조하는 '고아 레코드'의 발생을 막아 데이터 간의 논리적 연결성을 보장하며, 시스템 전체의 일관된 데이터 뷰를 제공한다.
> 3. **융합**: 연쇄 삭제(Cascade), 제한(Restrict), NULL 설정 등 다양한 참조 옵션과 결합되어 비즈니스 규칙에 따른 데이터 생명주기 관리를 자동화한다.

+++

### Ⅰ. 참조 무결성의 핵심 원칙

- **유효한 참조**: 외래키 $FK$ 의 값은 항상 참조되는 테이블의 기본키 $PK$ 값 중 하나와 일치해야 합니다.
- **예외 허용**: 비즈니스 로직상 허용된다면 외래키 값은 NULL이 될 수 있습니다 (아직 부서가 정해지지 않은 신입사원 등).
- **무결성 파괴 금지**: 부모 테이블의 행을 지우거나 수정할 때, 이를 참조하는 자식 데이터가 있다면 함부로 처리하지 못하게 막아야 합니다.

+++

### Ⅱ. 참조 무결성 구조 시각화 (ASCII Model)

```text
[ Parent Table: Dept ]           [ Child Table: Emp ]
┌─────────┬──────────┐           ┌────────┬──────────┬──────────┐
│ DeptID  │ Name     │           │ EmpID  │ Name     │ DeptID   │
├─────────┼──────────┤           ├────────┼──────────┼──────────┤
│   10    │ Sales    │ ◀───Ref───│  501   │ Kim      │   10     │ ✅ OK
│   20    │ IT       │           │  502   │ Lee      │   99     │ ◀── 💥 Error!
└─────────┴──────────┘           └────────┴──────────┴──────────┘
                                 (DeptID 99는 부모에 존재하지 않음)
```

+++

### Ⅲ. 참조 무결성 유지 옵션 (Foreign Key Options)

부모 데이터가 변경/삭제될 때 자식 데이터를 처리하는 방법입니다.
1. **CASCADE (연쇄)**: 부모를 지우면 자식도 함께 지웁니다.
2. **RESTRICT / NO ACTION (제한)**: 자식이 있으면 부모를 지울 수 없게 막습니다.
3. **SET NULL**: 부모를 지우면 자식의 외래키 값을 NULL로 바꿉니다.
4. **SET DEFAULT**: 부모를 지우면 자식의 외래키 값을 미리 정해둔 기본값으로 바꿉니다.

- **📢 섹션 요약 비유**: 참조 무결성은 **'부모와 아이의 외출'**과 같습니다. 아이(자식 테이블)가 "우리 아빠(부모 테이블)는 저 사람이에요"라고 가리켰는데 정작 그 자리에는 아무도 없는(고아 레코드) 상황이 생기면 안 됩니다. 아버지가 자리를 옮기면 아이도 같이 데려가거나(Cascade), 아이가 있으면 아버지가 마음대로 떠나지 못하게(Restrict) 하여 가족의 연결 고리를 유지하는 것입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Foreign Key]**: 참조 무결성을 물리적으로 구현하는 제약 조건.
- **[Orphan Record]**: 참조 무결성이 깨져서 부모를 잃어버린 데이터.
- **[Join Integrity]**: 참조 무결성이 보장되어야만 조인 연산의 결과가 완벽해짐.

📢 **마무리 요약**: **Referential Integrity**는 데이터베이스를 거대한 관계의 망으로 묶어주는 접착제입니다. 이 끈이 단단하게 유지될 때 비로소 파편화된 데이터들은 하나의 의미 있는 정보가 됩니다.
