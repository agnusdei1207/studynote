# 532. 3단계 커밋(3PC) - 블로킹을 넘어서는 비차단 합의

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 3단계 커밋(3PC)은 2PC의 블로킹 문제를 해결하기 위해, **'예비 커밋(Pre-Commit)' 단계를 추가하고 타임아웃 매커니즘을 도입하여 코디네이터 장애 시에도 참여자가 스스로 결정을 내릴 수 있게 한 프로토콜**이다.
> 2. **가치**: 코디네이터가 응답이 없을 때 참여자들이 서로의 상태(Pre-Commit 여부)를 확인하여 **자율적으로 커밋이나 롤백을 수행**함으로써 시스템의 가용성을 유지한다.
> 3. **융합**: 분산 상태 기계(State Machine) 이론과 타임아웃 기반 복구 기술이 융합되어, 네트워크 지연 속에서도 중단 없는 분산 합의를 지향한다.

+++

### Ⅰ. 3PC의 3단계 상세 프로세스

1. **Can-Commit (질의 단계)**: 조정자가 참여자에게 작업 수행 가능 여부를 묻습니다. (2PC의 Prepare와 유사)
2. **Pre-Commit (예비 커밋)**: 모든 참여자가 동의하면, 조정자가 "곧 커밋할 것이니 준비하라"고 지시합니다. 참여자들은 락을 걸고 대기합니다.
3. **Do-Commit (최종 커밋)**: 모든 참여자가 예비 커밋을 확인하면 최종적으로 커밋을 수행합니다.

+++

### Ⅱ. 3PC 비블로킹 구조 시각화 (ASCII Flow)

```text
[ 3PC Non-blocking Workflow ]

  [ Coordinator ]           [ Node A ]           [ Node B ]
        │                      │                    │
        │ ── 1. CanCommit? ──▶ │                    │
        │ ◀───  YES  ───────── │                    │
        │                      │                    │
        │ ── 2. PreCommit! ──▶ │                    │ ✅ "Prepare to commit"
        │ ◀───  ACK  ───────── │                    │
        │                      │                    │
        │ ── 3. DoCommit!  ──▶ │                    │ ✅ "Final Commit"
        ▼                      ✅                   ✅
        
  * 핵심: Node A, B가 'PreCommit'을 이미 받은 상태에서 
    Coordinator가 죽으면, 타임아웃 후 자기들끼리 
    "우리 둘 다 약속 받았으니 그냥 커밋하자!"라고 자율 결정 가능. 🔓 ✅
```

+++

### Ⅲ. 3PC의 성과와 한계

- **성과**: 코디네이터의 단일 장애 지점(SPOF)으로 인한 시스템 전체 마비(Blocking)를 이론적으로 해결했습니다.
- **한계**: 
    - **네트워크 분할(Partition)** 상황에서는 여전히 데이터 불일치가 발생할 수 있습니다 (참여자가 서로 통신을 못 할 경우).
    - 메시지 교환 횟수가 늘어나 지연 시간(Latency)이 2PC보다 길어집니다.
    - 실무에서는 3PC보다 더 정교한 **Raft나 Paxos** 알고리즘이 더 널리 쓰입니다.

- **📢 섹션 요약 비유**: 3PC는 **'결혼식의 리허설'**과 같습니다. 2PC가 바로 결혼식 본식(Prepare->Commit)이라면, 3PC는 결혼식 전날 리허설(Pre-Commit)을 한 번 더 하는 것입니다. 리허설 때 전원 참석과 동의를 확인했으니, 정작 결혼식 당일 주례 선생님이 늦으시더라도(장애), 신랑 신부와 하객들은 "어제 다 약속했으니 그냥 진행하자!"라고 결정할 수 있는 안전장치를 하나 더 둔 셈입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Non-blocking]**: 3PC가 탄생하게 된 가장 큰 목표.
- **[Pre-Commit]**: 자율적 합의를 가능케 하는 '확신'의 상태값.
- **[Timeout Mechanism]**: 기다리다 지치면 스스로 움직이게 하는 트리거.

📢 **마무리 요약**: **3-Phase Commit**은 분산 합의 기술의 진보입니다. 조율자의 부재를 자율적인 규칙으로 보완하여, 멈추지 않는 데이터 시스템을 향한 공학적 의지를 보여줍니다.
