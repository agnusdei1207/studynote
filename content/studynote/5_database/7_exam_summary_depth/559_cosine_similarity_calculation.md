# 559. 코사인 유사도(Cosine Similarity) - 벡터 간 방향의 일치도

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 코사인 유사도는 두 벡터 사이의 **각도($\theta$)의 코사인 값을 이용하여 유사성을 측정하는 방식**으로, 수치의 절대적인 크기(Magnitude)보다 데이터가 향하는 **방향성**에 집중하는 지표다.
> 2. **가치**: 텍스트 분석 시 문서의 길이에 상관없이 **의미적 문맥의 유사함**을 정확히 포착하며, 결과값이 -1에서 1 사이로 정규화되어 있어 직관적인 비교가 가능하다.
> 3. **융합**: 고차원 임베딩(Embedding) 매칭 기술과 융합되어 벡터 데이터베이스의 시맨틱 검색 엔진을 구성하는 가장 대중적인 거리 계측 연산으로 쓰인다.

+++

### Ⅰ. 코사인 유사도의 수학적 원리

- **정의**: 두 벡터의 내적(Dot Product)을 각 벡터의 크기의 곱으로 나눈 값입니다.
- **수식**: $Sim(A, B) = \frac{A \cdot B}{\|A\| \|B\|}$
- **결과 해석**:
    - **1 (0도)**: 두 벡터의 방향이 완전히 일치 (매우 유사).
    - **0 (90도)**: 두 벡터가 직교함 (독립적, 관계없음).
    - **-1 (180도)**: 두 벡터가 정반대 방향.

+++

### Ⅱ. 벡터 각도 기반 유사도 시각화 (ASCII Model)

```text
[ Vector Direction & Cosine Similarity ]

      Y-Axis
        ▲      ● Vector A (Document 1)
        │     /
        │    / θ (Small Angle = High Similarity) ✅
        │   /─────────▶ ● Vector B (Document 2)
        │  /
        └──────────────────────────▶ X-Axis

  * Length Invariance: 벡터의 길이가 길어져도(문서가 길어져도) 
    방향(주제)이 같으면 유사도는 변하지 않음 ✅
```

+++

### Ⅲ. 실무적 의의: 왜 텍스트 임베딩에 쓰는가?

1. **길이 독립성**: "AI"라는 단어가 1번 나온 짧은 글과 100번 나온 긴 글은 유클리드 거리로는 멀게 느껴지지만, 코사인 유사도로는 'AI'라는 동일한 방향성을 가지므로 매우 가깝게 인식됩니다.
2. **시맨틱 매칭**: 단어가 정확히 일치하지 않아도, 임베딩 공간에서 비슷한 맥락을 가진 벡터들은 좁은 각도를 유지하므로 의미론적 검색(Semantic Search)이 가능해집니다.
3. **효율성**: 벡터를 미리 정규화(Unit Vector)해두면, 코사인 유사도는 단순한 **내적(Dot Product)** 연산만으로 계산되어 초고속 조회가 가능합니다.

- **📢 섹션 요약 비유**: 코사인 유사도는 **'나침반 바늘의 방향 비교'**와 같습니다. 나침반이 크든 작든(데이터의 크기) 상관없이, 두 바늘이 북동쪽(특정 주제)을 얼마나 똑같이 가리키느냐만 보는 것입니다. 방향만 정확하다면 크기는 중요하지 않다는 지혜로운 비교법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Dot Product]**: 코사인 유사도 계산의 핵심 분자 연산.
- **[L2 Normalization]**: 벡터의 크기를 1로 맞추어 방향성만 남기는 전처리.
- **[Euclidean Distance]**: 각도가 아닌 '직선거리'를 측정하는 방식 (크기에 민감).

📢 **마무리 요약**: **Cosine Similarity**는 데이터의 '본질적 의도'를 봅니다. 화려한 수치적 외형에 현혹되지 않고 데이터가 지향하는 순수한 방향을 읽어내는 AI 검색의 나침반입니다.
