# 588. 쿼리 옵티마이저 - 최적의 실행 계획을 향한 여정

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 쿼리 옵티마이저는 사용자가 던진 추상적인 SQL 질의를 분석하여, 데이터에 접근하고 결합하는 가장 효율적인 **물리적 실행 계획(Execution Plan)을 생성하는 DBMS의 핵심 지능 엔진**이다.
> 2. **가치**: 시스템 통계 정보(레코드 수, 인덱스 분포 등)를 바탕으로 각 경로의 **비용(Cost)**을 수치화하여 비교하며, 수백만 개의 후보 중 가장 빠르게 답을 낼 지름길을 0.1초 내에 찾아낸다.
> 3. **융합**: 쿼리 변환(Transformation), 비용 모델링, 그리고 동적 실행 기술이 융합되어, 데이터 분포 변화에 실시간으로 대응하는 '자율적 성능 최적화'를 실현한다.

+++

### Ⅰ. 실행 계획 수립의 3대 단계

1. **쿼리 파싱 및 변환 (Query Transformation)**:
    - SQL의 문법을 검사하고, 옵티마이저가 처리하기 쉬운 형태(관계 대수식)로 재작성합니다.
    - 예: 서브쿼리를 조인으로 풀기(Unnesting), 뷰 머징(View Merging).
2. **비용 예측 (Cost Estimation)**:
    - 생성된 수많은 실행 경로 후보들에 대해, **시스템 통계**를 대입하여 I/O 횟수와 CPU 사용량을 계산합니다.
    - **카디널리티(Cardinality)** 추정이 비용 계산의 핵심입니다.
3. **계획 확정 (Plan Selection)**:
    - 계산된 비용이 가장 낮은 최종 계획을 선택하여 실행 엔진(Executor)으로 전달합니다.

+++

### Ⅱ. 옵티마이저 의사결정 시각화 (ASCII Flow)

```text
[ Optimizer Strategy Pipeline ]

```

### Ⅲ. 옵티마이저 성능의 열쇠: 통계 정보

- **정확도**: 통계 정보가 낡으면 옵티마이저는 엉뚱한 길(예: 인덱스 대신 풀 스캔)을 최선이라고 착각합니다.
- **히스토그램**: 컬럼 내 데이터가 특정 값에 쏠려 있는 정도를 파악하여 더욱 정밀한 비용을 산정하게 돕습니다.
- **적응형 최적화**: 런타임에 실제 데이터를 보고 실행 계획을 실시간으로 수정하는 고도의 기능도 탑재되고 있습니다.

- **📢 섹션 요약 비유**: 쿼리 옵티마이저는 **'초행길의 내비게이션'**과 같습니다. 운전자(사용자)가 "목적지(SQL)"만 말해주면, 내비게이션(옵티마이저)은 실시간 교통 정보(시스템 통계)를 분석하여 "고속도로(인덱스)를 탈지 국도(풀 스캔)로 갈지" 결정합니다. 교통 정보가 1년 전 것이라면(통계 미갱신) 내비게이션은 꽉 막힌 길로 우리를 안내하는 실수를 범하게 되는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[CBO (Cost-Based Optimizer)]**: 현대적 성능 최적화의 주류 (비용 중심).
- **[RBO (Rule-Based Optimizer)]**: 정해진 규칙(우선순위)에 따라 길을 찾는 고전 방식.
- **[Cardinality]**: 조건절에 의해 걸러질 것으로 예상되는 행의 수.

📢 **마무리 요약**: **Query Optimizer**는 데이터베이스의 뇌입니다. 보이지 않는 곳에서 수행되는 치열한 수학적 계산이 복잡한 SQL 뒤에 숨겨진 광속의 성능을 만들어냅니다.
