# 573. ODS vs DW - 운영과 분석의 시간차 조율

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ODS(운영 데이터 저장소)는 운영 시스템으로부터 데이터를 **준실시간(Near Real-time)으로 수집하여 통합한 저장소**이며, DW(데이터 웨어하우스)는 이 데이터를 다시 가공하여 **장기적인 추세 분석을 위해 구축한 이력 저장소**다.
> 2. **가치**: ODS는 "지금 당장"의 통합된 업무 현황 파악에 집중하여 운영계 부하를 줄이며, DW는 "과거부터 현재까지"의 다차원 분석을 통해 전략적 의사결정을 지원한다.
> 3. **융합**: CDC(실시간 복제) 기술과 ETL 배치 프로세스가 융합되어, 운영 시스템 ──▶ ODS ──▶ DW로 이어지는 유기적인 데이터 공급망을 형성한다.

+++

### Ⅰ. ODS와 DW의 상세 비교

| 비교 항목 | ODS (Operational Data Store) | DW (Data Warehouse) |
|:---|:---|:---|
| **주요 목적** | 실시간 운영 현황 파악, 운영 리포팅 | **전략적 추세 분석, 경영 의사결정** |
| **데이터 시점** | **현재 (Current)** | **과거 ~ 현재 (Historical)** |
| **데이터 수준** | 상세 내역 (Granular) | **요약/집계 데이터 위주** |
| **갱신 주기** | 수초~수분 단위 (자주) | 일간/주간/월간 단위 (배치) |
| **보존 기간** | 단기 (수일~수개월) | 장기 (수년 이상) |

+++

### Ⅱ. 데이터 흐름 계층 시각화 (ASCII Model)

```text
[ Data Integration Flow ]

  (Source DB) ──▶ [ ODS (Current) ] ──▶ [ DW (Historical) ]
  ┌──────────┐      ┌─────────────┐      ┌──────────────┐
  │ ธุรก 거래 │ ──▶  │ 통합 거래   │ ──▶  │ 연도별 매출  │ ✅
  │ 데이터   │ (CDC) │ 상세 현황   │ (ETL)│ 통계 분석    │ ✅
  └──────────┘      └──────┬──────┘      └──────────────┘
                           │
                           ▼
                  (Operational Reports)
                  "오늘 입금 총액은?"
```

+++

### Ⅲ. 실무적 구축 전략

- **ODS의 가치**: 운영 DB(OLTP)에서 복잡한 통합 조회를 직접 수행하면 서비스 속도가 느려집니다. ODS를 별도로 구축함으로써 운영 서비스에 지장을 주지 않고 '실시간 통합 현황'을 볼 수 있게 됩니다.
- **DW로의 전이**: ODS에 쌓인 상세 데이터 중 분석 가치가 있는 데이터를 추출하여, 차원 모델링(스타 스키마 등)을 거쳐 DW로 적재합니다.
- **하이브리드 추세**: 최근에는 기술의 발전으로 ODS와 DW의 경계가 모호해지는 **HTAP**이나 **실시간 DW** 아키텍처로 진화하고 있습니다.

- **📢 섹션 요약 비유**: ODS는 **'실시간 교통 상황판'**과 같습니다. 지금 어느 도로가 막히는지 즉시 확인하고 대처하기 위해 만든 '현재형' 정보입니다. 반면 DW는 **'연간 교통 통계 연보'**와 같습니다. 작년 한 해 동안 어느 시간대에 사고가 가장 많았는지 분석하여 내년 예산을 짜기 위한 '과거형' 정보입니다. 상황판을 보고 바로 대처(운영)하고, 통계 연보를 보고 전략(기획)을 짜는 차이입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[CDC]**: ODS의 실시간성을 보장하는 핵심 전송 기술.
- **[Staging Area]**: ODS 전단계에서 임시로 데이터를 거치는 작업장.
- **[Batch Layer]**: DW를 담당하는 정적인 연산 레이어.

📢 **마무리 요약**: **ODS & DW**는 데이터의 '나이'와 '용도'를 나누는 기준입니다. 신선한 현재 데이터와 묵직한 과거 데이터가 조화를 이룰 때, 기업은 비로소 완전한 데이터 통찰력을 가질 수 있습니다.
