# 420. 옵티마이저 CBO와 시스템 통계 - 데이터의 경제학

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CBO(Cost-Based Optimizer)는 미리 수집된 **시스템 통계 정보(레코드 수, 데이터 분포 등)를 바탕으로 각 실행 계획의 비용(Cost)을 계산하여 가장 효율적인 경로를 선택하는 지능형 엔진**이다.
> 2. **가치**: 데이터의 실제 부하를 예측하여 실행 계획을 수립하므로, RBO(Rule-based)의 한계를 극복하고 데이터 분포의 변화에 유연하게 대응하는 최적의 성능을 보장한다.
> 3. **융합**: 옵티마이저 통계 수집 기술(Analyze/Stats)과 실행 계획 생성 알고리즘이 융합되어, SQL 성능 고도화 및 데이터베이스 튜닝의 핵심 동력으로 작용한다.

+++

### Ⅰ. CBO (Cost-Based Optimizer)의 작동 원리

1. **후보 생성**: SQL을 실행할 수 있는 다양한 실행 경로(Index Scan, Full Scan, Nested Loop, Hash Join 등)를 생성합니다.
2. **비용 계산**: 각 경로에 대해 시스템 통계 정보를 대입하여 예상 I/O, CPU 사용량, 메모리 소요량을 수치화합니다.
3. **최적 경로 선정**: 계산된 비용(Cost)이 가장 낮은 최적의 실행 계획(Optimal Execution Plan)을 선택합니다.

+++

### Ⅱ. 시스템 통계와 CBO 시각화 (ASCII Flow)

통계 정보가 옵티마이저의 판단 근거가 됩니다.

```text
[ CBO Decision Pipeline ]

  (User SQL) ──▶ [ Optimizer ] ◀──(Reference)── [ System Stats ]
                        │                         - Row Counts
                        │                         - Block Usage
          ┌─────────────┴─────────────┐           - Data Distribution
          ▼                           ▼
  [ Plan A (Full Scan) ]      [ Plan B (Index Scan) ]
  - Cost: 5,000               - Cost: 200 ✅
          │                           │
          └─────────────┬─────────────┘
                        ▼
                [ Chosen Plan: B ] ──▶ [ Execution ]
```

+++

### Ⅲ. 시스템 통계의 주요 항목

- **테이블 통계**: 전체 레코드 수(Num_rows), 사용 중인 데이터 블록 수, 평균 행 길이.
- **컬럼 통계**: 고유 값의 수(Num_distinct), NULL 값의 수, 데이터 분포(Histogram).
- **인덱스 통계**: 리프 블록 수, 인덱스 높이(Level), 클러스터링 팩터(Clustering Factor).

- **📢 섹션 요약 비유**: CBO는 **'스마트폰의 실시간 길 찾기 앱'**과 같습니다. 예전 내비게이션(RBO)이 "무조건 큰길로 가라"는 고정된 규칙만 따랐다면, CBO는 현재의 교통 정보(시스템 통계)를 분석하여 "지금은 큰길이 막히니 좁은 골목길(인덱스)이 더 빠르다"라고 판단하여 최적의 경로를 알려주는 영리한 가이드입니다. 통계 정보가 낡으면(Traffic Data 미갱신) 엉뚱한 길로 안내할 수 있으므로 주기적인 업데이트가 중요합니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Analyze / GATHER_STATS]**: 시스템 통계를 최신으로 갱신하는 명령어.
- **[Cardinality]**: 특정 조건절에 의해 반환될 것으로 예상되는 행의 수 (비용 계산의 핵심).
- **[Histogram]**: 데이터 쏠림 현상을 파악하기 위해 통계 정보를 세분화한 도구.

📢 **마무리 요약**: **CBO와 시스템 통계**는 데이터베이스 성능의 나침반입니다. 정확한 통계를 바탕으로 데이터의 지름길을 찾아내는 능력이 고성능 DB 운영의 성패를 가릅니다.
