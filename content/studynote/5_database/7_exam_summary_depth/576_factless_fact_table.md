# 576. 팩트리스 팩트 테이블 - 숫자가 없는 사건의 기록

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 팩트리스 팩트 테이블(Factless Fact Table)은 금액이나 수량 같은 수치적 측정값(Measure)은 없고, 오직 **여러 차원 테이블의 외래키(FK)들로만 구성되어 '사건의 발생 여부'만을 기록하는 팩트 테이블**이다.
> 2. **가치**: "무엇이 일어났는가" 뿐만 아니라 "무엇이 일어나지 않았는가"를 분석할 수 있게 하며, 출석 체크, 자격 부여, 프로모션 대상 확인 등 **이벤트의 존재(Occurrence)나 범위(Coverage)**를 추적하는 데 사용된다.
> 3. **융합**: 다차원 모델링 기술과 결합되어, 수치가 없는 비즈니스 프로세스를 데이터웨어하우스 내에서 논리적으로 정량화할 수 있게 돕는다.

+++

### Ⅰ. 팩트리스 팩트 테이블의 두 가지 유형

1. **이벤트 기록형 (Event Recording)**: 
    - 특정 사건이 발생했음을 기록. 
    - 예: 학생의 출석부. (수치는 없지만 '출석'이라는 사건이 발생했음을 FK 조합으로 증명)
2. **범위/자격형 (Coverage)**: 
    - 특정 조건에 해당함을 기록. 
    - 예: 특정 기간 동안의 할인 대상 상품 리스트. (매출이 없어도 '대상'이었는지 여부를 분석할 때 사용)

+++

### Ⅱ. 팩트리스 팩트 구조 시각화 (ASCII Model)

수치 컬럼 없이 오직 '관계'로만 이루어진 테이블의 모습입니다.

```text
[ Factless Fact Table: Student Attendance ]

  ┌──────────────┐      ┌──────────────┐      ┌──────────────┐
  │  Dim_Student │      │   Dim_Time   │      │  Dim_Course  │
  └──────┬───────┘      └──────┬───────┘      └──────┬───────┘
         │ (FK)                │ (FK)                │ (FK)
         ▼                     ▼                     ▼
  ┌──────────────────────────────────────────────────────────┐
  │             Fact_Attendance (Factless)                   │ ✅
  │ -------------------------------------------------------- │
  │ [ Student_ID ] [ Time_ID ] [ Course_ID ] [ Attend_Flag ] │ ✅
  └──────────────────────────────────────────────────────────┘
  * "합산할 수치는 없지만, 행의 개수(Count)가 곧 데이터의 가치"
```

+++

### Ⅲ. 실무적 활용: "일어나지 않은 일" 분석

- **문제**: "우리 가게에 왔지만 물건을 사지 않은 사람은 누구인가?"
- **해결**:
    1. '매장 방문' 팩트리스 팩트 테이블 조회.
    2. '실제 구매' 팩트 테이블과 차집합 연산.
- **의의**: 수치 데이터(매출)만 있는 DW는 '구매한 사람'만 알 수 있지만, 팩트리스 팩트가 있으면 '방문만 한 사람'까지 포함한 전체 비즈니스 맥락을 파악할 수 있습니다.

- **📢 섹션 요약 비유**: 팩트리스 팩트 테이블은 **'학교의 출석부'**와 같습니다. 시험 성적표(일반 팩트)에는 '점수'라는 숫자가 중요하지만, 출석부에는 숫자가 없습니다. 오직 "이 학생이, 이 시간에, 이 교실에 있었다"는 사실 자체가 중요합니다. 성적표만 보면 공부 못하는 학생은 가치가 없어 보일지 모르지만, 출석부(팩트리스 팩트)가 있어야 그 학생이 학교에 오긴 왔는지(비즈니스 기회)를 알 수 있는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Occurrence]**: 사건이 발생했다는 사실 그 자체.
- **[Coverage]**: 비즈니스가 커버하는 잠재적 기회의 범위.
- **[Count]**: 팩트리스 팩트 테이블에서 유일하게 의미를 갖는 통계 함수.

📢 **마무리 요약**: **Factless Fact Table**은 분석의 지평을 넓힙니다. 눈에 보이는 '숫자' 너머에 존재하는 '사실'의 궤적을 기록함으로써, 비즈니스의 보이지 않는 빈틈을 찾아내는 정교한 도구입니다.
