# 525. 제로 트러스트 DB 인증 - 신원 중심 보안(IAM)의 실천

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제로 트러스트 DB 인증은 IP 주소나 내부망 접속 여부와 상관없이, **데이터에 접근하는 모든 주체에 대해 강력한 신원 확인(Identity)을 매 요청마다 반복 수행하는 보안 매커니즘**이다.
> 2. **가치**: 한 번의 로그인으로 광범위한 권한을 얻는 고전적 방식의 위험을 제거하며, **다요소 인증(MFA)**과 컨텍스트 분석을 통해 계정 탈취에 의한 데이터 유출을 원천 방어한다.
> 3. **융합**: 중앙 집중식 신원 권한 관리(IAM) 시스템과 DB 접근 제어(DAM) 기술이 융합되어, 클라우드 환경의 파편화된 데이터를 보호하는 현대 보안의 핵심 기둥이 된다.

+++

### Ⅰ. 신원 중심 인증(Identity-Centric)의 3대 원칙

1. **상시 검증 (Never Trust, Always Verify)**: 세션이 유지되는 동안에도 주기적으로 사용자의 상태와 위치를 재인증합니다.
2. **최소 권한 기반의 동적 할당**: 고정된 권한 대신, 업무 목적에 맞는 최소한의 권한을 승인 절차를 통해 일시적으로 부여합니다 (Just-in-Time).
3. **MFA (Multi-Factor Authentication) 필수화**: 비밀번호 외에 생체 인식, 모바일 푸시, 하드웨어 키 등을 결합하여 인증 강도를 높입니다.

+++

### Ⅱ. 제로 트러스트 IAM 인증 아키텍처 (ASCII Flow)

```text
[ Identity-based DB Access Control ]

  (User/App) ──▶ [ IAM / SSO Portal ] ──▶ (Multi-factor Auth)
                        │
                        ▼ [ Policy Decision ]
  (Context) ────▶ [ Trust Engine ] ──▶ (Allow/Deny)
  - Device Health       │
  - Geolocation         ▼ [ Policy Enforcement ]
  - Behavior      [ Database Proxy / Gateway ] ──▶ [ Target DB ] 🔒 ✅
                        │
                        └─────────── (Session Monitoring)
```

+++

### Ⅲ. 실무적 구현 사례: Dynamic Secrets

- **개념**: 데이터베이스에 고정된 계정(예: scott/tiger)을 두지 않고, 사용자가 접근할 때마다 1시간 동안만 유효한 임시 계정과 비밀번호를 생성해 주는 기술입니다 (예: HashiCorp Vault).
- **효과**: 비밀번호가 유출되더라도 잠시 후면 자동으로 만료되므로 대규모 유출 사고를 막을 수 있습니다.
- **감사 연계**: 모든 인증 시도는 중앙 IAM 로그에 기록되어, "누가, 어떤 기기로, 언제 이 권한을 사용했는지" 명확한 증적을 남깁니다.

- **📢 섹션 요약 비유**: 제로 트러스트 DB 인증은 **'얼굴 인식과 지문이 필요한 VIP 금고'**와 같습니다. 건물 정문(방화벽)을 통과했다고 해서 금고를 열 수 있는 게 아닙니다. 금고 앞에 설 때마다 다시 한번 얼굴을 대조하고(MFA), 지금 이 사람이 금고를 열 자격이 있는 시간인지(Context) 확인받아야만 비로소 금고가 열립니다. 열쇠를 한 번 받았다고 해서 영원히 내 것이 아닌, 매번 허락을 받아야 하는 깐깐한 시스템입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[MFA]**: 지식, 소유, 특징 기반의 다중 인증 기술.
- **[Just-in-Time (JIT)]**: 필요할 때만 즉시 권한을 생성하고 회수하는 방식.
- **[Single Sign-On (SSO)]**: 한 번의 강력한 인증으로 여러 DB에 접근하게 돕는 통합 계정 체계.

📢 **마무리 요약**: **Identity-centric Authentication**은 경계가 사라진 클라우드 시대의 새로운 성벽입니다. 신원을 데이터 보안의 최우선 가치로 삼을 때, 비로소 데이터는 어떤 환경에서도 안전하게 보호될 수 있습니다.
