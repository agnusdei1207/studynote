# 528. 오손 읽기(Dirty Read)와 격리 수준 - 미확정 데이터의 차단

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오손 읽기(Dirty Read)는 한 트랜잭션이 **다른 트랜잭션이 수정 중이지만 아직 커밋(Commit)하지 않은 데이터를 읽는 현상**으로, 데이터베이스 격리 수준(Isolation Level) 설정을 통해 제어된다.
> 2. **가치**: 읽어온 데이터가 이후 롤백(Rollback)될 경우 존재하지 않는 가짜 정보를 바탕으로 작업하는 심각한 오류를 방지하며, 시스템의 **원자성(Atomicity)** 보장을 뒷받침한다.
> 3. **융합**: 'Read Committed' 이상의 격리 수준을 설정함으로써 물리적으로는 **잠금(Locking)이나 MVCC 스냅샷 기술**을 활용해 오손 읽기를 융합 회피한다.

+++

### Ⅰ. 오손 읽기의 위험성과 격리 수준별 대응

- **현상**: 트랜잭션 T1이 A를 100에서 200으로 고쳤는데, T2가 200을 읽어감. 이후 T1이 롤백되어 A는 다시 100이 되지만 T2는 여전히 200을 진짜로 알고 있음.
- **격리 수준별 회피 여부**:
    - **Read Uncommitted (Level 0)**: **발생함** (성능은 최상이나 정합성 최악).
    - **Read Committed (Level 1)**: **차단됨** (커밋된 것만 읽음).
    - **Repeatable Read (Level 2)**: **차단됨**.
    - **Serializable (Level 3)**: **차단됨**.

+++

### Ⅱ. 오손 읽기 차단 매커니즘 시각화 (ASCII Model)

'Read Committed' 수준에서의 읽기 차단 방식입니다.

```text
[ Avoiding Dirty Read with Read Committed ]

  T1 (Writer) ────────▶ Update Balance = 500 (NOT COMMITTED)
                          │
  T2 (Reader) ────────▶ Read Balance?
                          │
          ┌───────────────┴───────────────┐
          ▼ [ DBMS Control Layer ]        ▼
    (Read Uncommitted 시)           (Read Committed 시) ✅
    - "500 읽어가세요" 💥            - "기다리거나(Lock)"
                                    - "과거 커밋본(Undo) 읽으세요" ✅
                                    - Result: 100 (Safe)
```

+++

### Ⅲ. 실무적 의의: 왜 Level 1 이상인가?

- **데이터 신뢰도**: 사용자는 당연히 '확정된' 정보만 보기를 원합니다. "방금 입금 확인했는데 왜 잔액이 다시 줄었죠?"라는 민원을 방지하는 최소한의 보루입니다.
- **표준 설정**: Oracle, PostgreSQL, SQL Server 등 대부분의 메이저 DB는 오손 읽기를 허용하지 않는 **Read Committed**를 기본값으로 사용합니다.
- **예외 상황**: 데이터의 정확성보다 '대략적인 실시간 수치'가 훨씬 중요한 특수 통계 시스템에서만 극히 제한적으로 Level 0을 고려합니다.

- **📢 섹션 요약 비유**: 오손 읽기는 **'요리사가 간을 보고 버리려고 따로 빼둔 찌개(롤백 예정 데이터)를, 종업원이 손님상에 내가는 것'**과 같습니다. 손님은 완성되지 않은 잘못된 음식을 먹게 되므로, 반드시 주방장이 "완성!"(Commit)이라고 외칠 때만 음식을 내가야 합니다. 격리 수준은 바로 이 주방과 홀 사이의 '음식 전달 규칙'을 정하는 것입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Isolation Level]**: 오손 읽기 방어의 가장 강력한 설정 도구.
- **[MVCC]**: 읽기 작업을 블로킹하지 않고도 오손 읽기를 막아주는 마법 같은 기술.
- **[Undo Log]**: 오손 읽기를 피하기 위해 과거의 정답(커밋 데이터)을 찾아가는 통로.

📢 **마무리 요약**: **Dirty Read**를 피하는 것은 데이터베이스 매너의 시작입니다. 격리 수준을 적절히 관리하여, 사용자가 오직 '확정된 진실'만을 마주할 수 있도록 보장해야 합니다.
