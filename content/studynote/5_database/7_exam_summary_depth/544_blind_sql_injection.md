# 544. SQL 인젝션 심화 - 에러부터 블라인드 공격까지

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SQL 인젝션은 사용자 입력값에 악의적인 쿼리를 삽입하여 DB를 조작하는 공격으로, 데이터가 직접 유출되지 않아도 참/거짓 응답이나 응답 시간 차이를 이용하는 **블라인드(Blind) 기법**으로 발전하였다.
> 2. **가치**: 눈에 보이지 않는 데이터를 한 글자씩 유추해내는 **추론 공격**의 위험성을 경고하며, 단순한 필터링만으로는 막을 수 없는 지능형 주입 공격의 체계를 분석한다.
> 3. **융합**: 웹 취약점 분석 기술과 데이터베이스 엔진의 연산 로직(Wait, Error msg)이 융합된 공격 형태이며, 이를 방어하기 위해 바인드 변수 사용 및 DB 방화벽의 입체적 대응이 요구된다.

+++

### Ⅰ. SQL 인젝션의 고도화 유형

1. **에러 기반 (Error-based)**:
    - 고의로 구문 오류를 유발하여, DB가 뱉어내는 에러 메시지 안에 포함된 테이블명이나 컬럼 정보를 탈취.
2. **불린 기반 블라인드 (Boolean-based Blind)**:
    - 쿼리 결과가 화면에 나오지 않을 때, `1=1` (참)과 `1=2` (거짓) 조건에 따른 페이지 반응 차이를 보고 데이터를 유추.
3. **시간 기반 블라인드 (Time-based Blind)**:
    - 참일 경우 `SLEEP(5)`를 실행하게 하여, 서버의 응답이 늦게 오면 해당 조건이 참임을 알아내는 지독한 추론 방식.

+++

### Ⅱ. 블라인드 인젝션 추론 매커니즘 시각화 (ASCII Model)

```text
[ Blind SQL Injection Inference ]

  (Attacker) "사용자 비밀번호 첫 글자가 'A'인가?"
       │
       ▼ [ Injected Query ]
  "SELECT * FROM Users WHERE ID='admin' AND SUBSTR(PWD,1,1)='A' AND SLEEP(5);"
       │
       ├─▶ (No Wait) ──▶ "아니군, 첫 글자는 'A'가 아니다."
       └─▶ (5초 대기) ──▶ ✅ "찾았다! 첫 글자는 'A'다."
```

+++

### Ⅲ. 공격 체계망 및 대응 전략

- **공격 단계**: 취약점 탐색 ──▶ DB 엔진 식별 ──▶ 스키마 정보 탈취 ──▶ 데이터 대량 유출.
- **방어 체계**:
    - **입력값 검증**: 특수문자(`'`, `--`, `;`) 필터링.
    - **Prepared Statement**: 쿼리 구조와 데이터를 분리하여 데이터가 명령어로 실행되는 것을 방지. ✅ (최고의 대책)
    - **최소 권한**: DB 계정에 꼭 필요한 권한만 부여하여 뚫려도 피해 최소화.
    - **에러 페이지 은닉**: 상세한 DB 에러 메시지가 사용자에게 보이지 않도록 설정.

- **📢 섹션 요약 비유**: 블라인드 인젝션은 **'스무고개 게임'**과 같습니다. 공격자는 정답(데이터)을 직접 볼 수는 없지만, "비밀번호 첫 글자가 기역인가요?", "니은인가요?"라고 수백 번 질문을 던집니다. 시스템이 대답을 안 해줘도 대답하는 시간(Time-based)이나 미세한 표정 변화(Boolean-based)를 보고 정답을 한 글자씩 알아맞히는 아주 끈질긴 공격입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Union-based]**: 쿼리 결과를 합쳐서 한 번에 보여주는 인젝션 방식.
- **[Out-of-band]**: 이메일이나 DNS 서버를 통해 데이터를 빼돌리는 우회 인젝션.
- **[SQL Map]**: 인젝션 공격을 자동화해 주는 도구 (방어자도 이를 통해 점검 수행).

📢 **마무리 요약**: **SQL Injection**은 고전이지만 가장 파괴적입니다. 보이지 않는 곳에서 데이터를 훔치려는 지능형 공격에 맞서, **바인딩과 검증**이라는 기본 원칙을 철저히 고수해야 합니다.
