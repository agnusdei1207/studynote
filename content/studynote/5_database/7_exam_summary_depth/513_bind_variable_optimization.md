# 513. 라이브러리 캐시와 바인드 변수 - 파싱 부하의 정복

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 바인드 변수(Bind Variable)는 SQL 쿼리 내의 가변적인 조건값을 상수로 직접 기술(Literal)하는 대신 **변수(?, :var)로 처리하여, 동일한 SQL 문장이 라이브러리 캐시(Library Cache)에서 재사용되도록 유도하는 기술**이다.
> 2. **가치**: 매번 새로운 실행 계획을 짜는 '하드 파싱(Hard Parsing)'을 생략하고 이미 검증된 계획을 쓰는 '소프트 파싱'을 극대화하여, **CPU 자원 소모를 줄이고 쿼리 응답 속도를 획기적으로 개선**한다.
> 3. **융합**: SQL 공유(SQL Sharing) 매커니즘의 핵심이며, 대량의 동시 접속자가 발생하는 온라인 트랜잭션(OLTP) 환경에서 시스템 마비를 방지하는 필수 최적화 전략이다.

+++

### Ⅰ. 하드 파싱 vs 소프트 파싱

- **하드 파싱 (Hard Parsing)**: 
    - SQL이 처음 들어왔을 때 문법 체크, 권한 확인, 최적화(계획 수립)를 모두 수행하는 무거운 과정.
- **소프트 파싱 (Soft Parsing)**: 
    - 이미 라이브러리 캐시에 있는 동일한 SQL의 실행 계획을 찾아 즉시 실행하는 가벼운 과정.
- **바인드 변수의 역할**: SQL 문장 자체를 '틀'로 만들어, 값만 바뀌어도 시스템이 '같은 SQL'로 인식하게 함으로써 소프트 파싱을 유도합니다.

+++

### Ⅱ. 바인드 변수 활용 시각화 (ASCII Model)

```text
[ Literal SQL (Bad) ]                 [ Bind Variable SQL (Good) ]

  SQL 1: SELECT * FROM User WHERE ID=1;  SQL 1: SELECT * FROM User WHERE ID=:id;
  SQL 2: SELECT * FROM User WHERE ID=2;  SQL 2: SELECT * FROM User WHERE ID=:id;
  (모두 다른 SQL로 인식 -> Hard Parsing)  (완벽히 같은 SQL로 인식 -> Soft Parsing) ✅

  [ Library Cache (RAM) ]               [ Library Cache (RAM) ]
  ┌───────────────────────────┐         ┌───────────────────────────┐
  │ Plan 1 (for ID=1)         │         │ Shared Plan (for ID=:id)  │ ✅
  │ Plan 2 (for ID=2)         │         └───────────────────────────┘
  └───────────────────────────┘         (메모리 절약 및 CPU 부하 감소)
```

+++

### Ⅲ. 실무적 장점과 주의사항

- **장점**: 
    - 공유 풀(Shared Pool) 메모리 낭비를 막아 'LATCH' 경합(자원 쟁탈전)을 줄입니다.
    - SQL 인젝션 보안 공격을 원천적으로 차단합니다.
- **주의사항 (Bind Peeking)**: 
    - 데이터 분포가 극단적으로 불균형할 때(예: 상태가 '진행중'은 1건, '종료'는 100만 건), 바인드 변수는 하나의 평균적인 실행 계획만 쓰므로 특정 값에서 성능이 떨어질 수 있습니다 (Adaptive Cursor Sharing으로 보완).

- **📢 섹션 요약 비유**: 바인드 변수는 **'식당의 인쇄된 메뉴판'**과 같습니다. 손님이 올 때마다 매번 종이에 메뉴를 새로 적어서(하드 파싱) 주는 대신, "주문하신 메뉴는 [ ]입니다"라고 적힌 인쇄물(바인드 변수)을 준비해두고 빈칸에 이름만 적어 넣는 것과 같습니다. 주문을 받는 속도가 훨씬 빨라지고 종이(메모리)도 아끼는 현명한 업무 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Shared Pool]**: 라이브러리 캐시가 포함된 메모리 영역.
- **[Cursor]**: 파싱된 SQL과 실행 계획이 담긴 메모리 객체.
- **[Latch: shared pool]**: 바인드 변수를 안 써서 파싱이 폭주할 때 발생하는 대표적 대기 이벤트.

📢 **마무리 요약**: **Bind Variable**은 고성능 SQL의 문법입니다. SQL을 '공유 가능한 자산'으로 만듦으로써 데이터베이스의 심장인 CPU를 가장 가치 있게 사용하는 기술입니다.
