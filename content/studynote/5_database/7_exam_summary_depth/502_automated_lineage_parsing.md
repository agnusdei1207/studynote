# 502. 데이터 리니지 자동 파싱 - 숨겨진 데이터 족보의 자동 추출

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 리니지 자동 파싱은 SQL 쿼리, 저장 프로시저, ETL 스크립트 등 **데이터베이스 관련 소스 코드를 구문 분석(Parsing)하여 데이터의 흐름과 의존 관계를 수작업 없이 자동으로 추출하는 기술**이다.
> 2. **가치**: 수만 개의 테이블이 얽힌 복잡한 환경에서 **휴먼 에러를 배제한 정확한 영향도 분석**을 가능케 하며, 데이터의 변경 이력을 실시간으로 반영하여 가시성을 유지한다.
> 3. **융합**: 정적 분석(Static Analysis) 기술과 메타데이터 수집 기술이 융합되어, 데이터의 탄생부터 소멸까지를 한눈에 보여주는 '지능형 데이터 지도'를 완성한다.

+++

### Ⅰ. 리니지 파싱의 주요 기술 단계

1. **소스 수집**: DB의 시스템 카탈로그, 쿼리 로그, 코드 레포지토리에서 분석 대상 소스(SQL 등)를 확보합니다.
2. **어휘 및 구문 분석 (Lexical & Syntactic Analysis)**: SQL 파서를 통해 예약어, 테이블명, 컬럼명, 조인 조건을 분리해 냅니다.
3. **그래프 생성**: 추출된 정보를 바탕으로 소스 테이블(Node)에서 타겟 테이블(Node)로 향하는 연결선(Edge)을 그립니다.
4. **시각화 및 저장**: 완성된 리니지 맵을 사용자 UI로 노출하고 그래프 DB에 저장하여 검색을 지원합니다.

+++

### Ⅱ. SQL 파싱을 통한 리니지 추출 시각화 (ASCII Model)

```text
[ SQL Parsing to Lineage Map ]

  (Input SQL) : INSERT INTO Monthly_Sales 
                SELECT SUM(S.Price), P.Category 
                FROM Sales S JOIN Product P ON S.ID = P.ID;

  ┌───────────────┐       ┌─────────────────┐       ┌────────────────────┐
  │ SQL Parser    │ ──▶   │ Dependency      │ ──▶   │ Lineage Map (Graph)│
  │ (Lex/Yacc)    │       │ Analyzer        │       │                    │
  └───────────────┘       └─────────────────┘       │ [Sales] ──┐        │
                                                    │           ├─▶ [M_S]│ ✅
                                                    │ [Product] ┘        │
                                                    └────────────────────┘
```

+++

### Ⅲ. 자동 파싱 기술의 한계와 도전

- **동적 SQL 처리**: 소스 코드 내에서 런타임에 생성되는 쿼리는 정적 파싱만으로 분석하기 어려워 실행 로그 분석(Dynamic Analysis)이 병행되어야 합니다.
- **다양한 방언(Dialect)**: Oracle, MySQL, BigQuery 등 벤더마다 다른 SQL 문법을 완벽히 해석할 수 있는 멀티 파서 지원이 필수적입니다.
- **성능 오버헤드**: 수십만 줄의 저장 프로시저를 파싱하는 작업은 많은 컴퓨팅 자원을 소모하므로 효율적인 증분 분석 기술이 요구됩니다.

- **📢 섹션 요약 비유**: 데이터 리니지 자동 파싱은 **'CSI의 혈흔 형태 분석'**과 같습니다. 사건 현장(데이터베이스)에 흩어진 흔적(SQL 로그)들을 정밀하게 분석하여, 피(데이터)가 어디서 튀어서 어디에 묻었는지(데이터 흐름)를 과학적으로 재구성하는 것입니다. 일일이 목격자(개발자)를 찾아 물어보지 않아도 객관적인 물증만으로 진실을 밝혀내는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[AST (Abstract Syntax Tree)]**: SQL을 분석하기 위해 내부적으로 만드는 트리 구조.
- **[Impact Analysis]**: 파싱된 리니지를 통해 얻는 가장 큰 업무상 이득.
- **[Metadata Harvesting]**: 여러 시스템에서 리니지 정보를 자동으로 긁어모으는 작업.

📢 **마무리 요약**: **Automated Lineage Parsing**은 데이터 관리의 자동화를 이끄는 핵심 동력입니다. 코드 속에 숨겨진 데이터의 연결 고리를 밝혀냄으로써 데이터 시스템의 투명성을 완성합니다.
