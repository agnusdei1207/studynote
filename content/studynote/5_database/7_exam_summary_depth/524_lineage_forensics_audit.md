# 524. 데이터 리니지 기반 감사 및 포렌식 - 디지털 증거의 추적

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 리니지 기반 감사는 데이터의 흐름 지도를 활용하여 **특정 수치나 결과값이 어느 소스에서, 어떤 과정을 거쳐 도달했는지 역추적(Back-tracing)하여 그 정당성을 입증하는 활동**이다.
> 2. **가치**: 보안 사고나 데이터 변조 의심 시, 리니지를 통해 **사고의 발생 지점(Root Cause)과 오염 범위를 즉각 파악**할 수 있어 디지털 포렌식의 결정적 도구로 쓰인다.
> 3. **융합**: 데이터 리니지 자동화 기술과 감사 로그(Audit Log)가 융합되어, "누가 데이터를 언제, 어디서, 어떻게 바꿨는가"에 대한 완벽한 사후 추적성과 부인 방지 기능을 제공한다.

+++

### Ⅰ. 리니지 기반 포렌식의 주요 시나리오

1. **지표 이상 발견**: 경영 보고서의 매출 수치가 이상할 때, 리니지 맵을 따라가며 중간 단계의 ETL 오류나 원천 데이터의 조작 여부를 확인합니다.
2. **개인정보 유출 조사**: 민감 데이터가 외부로 유출되었을 때, 해당 데이터가 어떤 경로(어느 DB, 어느 API)를 타고 이동했는지 리니지로 추적하여 보안 구멍을 찾아냅니다.
3. **규제 감사 (Audit)**: 금융 당국 등에서 데이터의 투명성을 요구할 때, 리니지를 통해 전 처리 과정이 법적 기준을 준수했음을 입증합니다.

+++

### Ⅱ. 리니지 포렌식 프로세스 시각화 (ASCII Flow)

```text
[ Data Forensic Investigation ]

  (Suspicious Result) ──▶ [ Search Lineage Map ] ──▶ (Trace Backwards)
          │                         │                       │
          ▼                         ▼                       ▼
  [ Report Error ] ◀──(1)── [ Aggregate Logic ] ◀──(2)── [ Raw Source ]
                                    │                       │
                                    ▼                       ▼
                          [ Check Audit Log ]     [ Verify Original ] ✅
                          "Who changed SQL?"      "Was it hacked here?"
```

+++

### Ⅲ. 포렌식 관점의 리니지 핵심 기능

- **시간 여행 (Point-in-time Lineage)**: 현재의 리니지뿐만 아니라, 사고가 발생했던 3일 전의 데이터 흐름이 어떠했는지 재현할 수 있어야 합니다.
- **불변성 보장**: 리니지 정보 자체도 공격자에 의해 조작되지 않도록 별도의 안전한 저장소(WORM 등)에 보관해야 합니다.
- **연관 관계 매핑**: 데이터 테이블뿐만 아니라, 이를 처리한 서버 IP, 실행한 사용자 ID, 사용된 라이브러리 버전까지 리니지에 포함하여 포렌식의 정확도를 높입니다.

- **📢 섹션 요약 비유**: 리니지 기반 감사는 **'CSI의 혈흔 형태 분석'**과 같습니다. 사건 현장(최종 보고서)에 남은 흔적을 보고, 역으로 범행 경로(데이터 흐름)를 추적하여 범인이 어디서 들어오고(소스 DB), 어떤 흉기(SQL)를 썼는지 과학적으로 밝혀내는 과정입니다. 이 지도가 정교할수록 범인을 잡고(원인 규명) 재발을 막는(보안 강화) 것이 쉬워집니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Root Cause Analysis]**: 장애나 오류의 근본 원인을 찾는 활동.
- **[Data Integrity]**: 리니지를 통해 최종적으로 증명하고자 하는 데이터의 무결성.
- **[Non-repudiation]**: 리니지 기록을 통해 "나는 안 했다"고 발뺌하지 못하게 하는 부인 방지.

📢 **마무리 요약**: **Lineage-based Forensics**는 데이터의 결백을 입증하는 과정입니다. 복잡한 시스템일수록 데이터의 족보를 투명하게 관리하는 것이 가장 강력한 보안 전략입니다.
