# 562. B-Tree 디스크 I/O 최적화 - 팬아웃과 블록 매핑의 조화

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: B-Tree 디스크 I/O 최적화는 인덱스 노드의 크기를 운영체제의 **디스크 블록(또는 페이지) 크기와 일치**시키고, 한 노드가 가질 수 있는 자식 노드 수인 **팬아웃(Fan-out)을 극대화**하여 트리의 높이를 낮추는 물리적 설계 기법이다.
> 2. **가치**: 트리의 높이(Level)를 최소화함으로써 단 몇 번의 디스크 읽기만으로 수조 건의 데이터 중 원하는 값을 찾아내는 **로그 시간($O(\log N)$) 검색 성능**을 보장한다.
> 3. **융합**: 하드웨어의 블록 I/O 특성과 B-Tree의 다진 탐색(Multi-way Search) 원리가 융합되어, 현대 데이터베이스 검색 엔진의 성능 표준으로 작용한다.

+++

### Ⅰ. 최적화의 핵심 요소: 팬아웃(Fan-out)

- **정의**: 하나의 노드에서 뻗어 나가는 자식 노드의 개수.
- **차수 (Order, $m$)**: 노드가 가질 수 있는 최대 자식 수.
- **영향**: 팬아웃이 클수록 트리의 높이는 기하급수적으로 낮아집니다.
    - 예: 팬아웃이 100이면, 3층 높이로 100만($100^3$)개의 레코드를 수용 가능. (즉, 단 3번의 디스크 I/O로 검색 완료) ✅

+++

### Ⅱ. 노드 크기와 블록 매핑 시각화 (ASCII Model)

노드 하나가 디스크 블록 하나와 1:1로 매핑되어야 불필요한 I/O 낭비가 없습니다.

```text
[ Optimized B-Tree Node Layout ]

  ┌──────────────────────────────────────────────────────────┐
  │ Disk Block (e.g. 8KB / 16KB)                             │
  │ ┌──────────────────────────────────────────────────────┐ │
  │ │  [ Key 1 ][ Ptr 1 ][ Key 2 ][ Ptr 2 ] ... [ Key M ]  │ │ ✅ High Fan-out
  │ └──────────────────────────────────────────────────────┘ │
  └──────────────────────────────────────────────────────────┘
          │ (1 Block Read)              │
          ▼                             ▼
  ┌───────────────┐             ┌───────────────┐
  │ Child Block 1 │             │ Child Block 2 │  (Lower Level)
  └───────────────┘             └───────────────┘
```

+++

### Ⅲ. 디스크 I/O 절감을 위한 설계 전략

1. **블록 크기 최적화**: 쿼리 패턴에 따라 블록 크기를 8KB, 16KB 등으로 설정하여 한 번의 I/O로 최대한 많은 키 정보를 메모리에 올립니다.
2. **키 압축 (Prefix Compression)**: 인덱스 키의 앞부분 공통 문자열을 압축하여 한 노드에 더 많은 키를 담음으로써 팬아웃을 인위적으로 높입니다.
3. **노드 채우기 비율 (Fill Factor)**: 페이지 스플릿을 방지하기 위해 노드 공간의 일부(보통 10~30%)를 비워두어 쓰기 성능과 읽기 성능 사이의 균형을 맞춥니다.

- **📢 섹션 요약 비유**: B-Tree 최적화는 **'초고층 빌딩의 엘리베이터 설계'**와 같습니다. 팬아웃을 높이는 것은 한 번에 수백 명을 태울 수 있는 '거대 엘리베이터(큰 블록)'를 만드는 것과 같습니다. 엘리베이터가 클수록 몇 번 오가지 않아도 모든 사람(데이터)을 옮길 수 있고, 층수(트리 높이)를 낮게 지을 수 있어 목적지까지 도달하는 시간이 획기적으로 줄어드는 원리입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Block I/O]**: 디스크와 메모리 사이의 최소 전송 단위.
- **[Tree Height]**: 검색 성능에 직접적인 영향을 미치는 계층 수.
- **[Clustering Factor]**: 인덱스와 실제 데이터의 물리적 배치 일치도.

📢 **마무리 요약**: **B-Tree I/O Optimization**은 물리적 한계를 극복하는 수학의 힘입니다. 하드웨어의 블록 단위를 이해하고 데이터의 밀도를 높일 때, 데이터베이스는 비로소 빛의 속도로 응답할 수 있습니다.
