# 582. 동적 성능 뷰(V$, DMV) - 데이터베이스 실시간 검진 창

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동적 성능 뷰는 데이터베이스가 가동되는 동안 **메모리 상에서 실시간으로 변하는 상태 정보(세션, 락, I/O, SQL 통계 등)를 조회할 수 있게 제공하는 가상 테이블**이다.
> 2. **가치**: 장애 발생 시 병목 지점(Bottleneck)을 즉각적으로 파악하고, 현재 실행 중인 쿼리의 성능 지표와 락(Lock) 경합 상태를 트레이싱하여 **신속한 문제 해결**을 돕는다.
> 3. **융합**: 오라클의 **V$ 뷰**와 SQL Server의 **DMV(Dynamic Management Views)** 기술이 대표적이며, 상시 모니터링 시스템과 연계되어 시스템 자율 최적화의 원천 데이터가 된다.

+++

### Ⅰ. 동적 성능 뷰의 특징과 용도

- **가상성**: 디스크에 저장된 정적인 정보가 아니라, 인스턴스 기동 시 메모리에 생성되어 사라지는 휘발성 정보입니다.
- **실시간성**: 현재 접속 중인 세션 수, 방금 실행된 SQL의 CPU 사용량 등을 즉시 확인할 수 있습니다.
- **주요 활용 분야**:
    - **세션 모니터링**: 누가 접속해서 무슨 일을 하고 있는가?
    - **락 트레이싱**: 어떤 쿼리가 다른 쿼리를 블로킹하고 있는가?
    - **성능 튜닝**: 가장 많은 자원을 소모하는 'Top-N SQL'은 무엇인가?

+++

### Ⅱ. 모니터링 아키텍처 시각화 (ASCII Model)

```text

```

+++

### Ⅲ. 실무에서 자주 쓰이는 핵심 뷰 (Oracle 기준)

1. **V$SESSION**: 현재 접속된 모든 세션의 상세 정보 및 상태.
2. **V$SQL / V$SQLAREA**: 실행된 SQL 문장과 통계치 (실행 횟수, 처리 시간).
3. **V$LOCK / V$SESSION_WAIT**: 락 경합과 대기 이벤트 정보. ✅ (장애 분석의 핵심)
4. **V$SYSSTAT**: 시스템 전체의 성능 지표 (버퍼 히트율, 파싱 횟수 등).

- **📢 섹션 요약 비유**: 동적 성능 뷰는 **'중환자실의 생체 신호 모니터'**와 같습니다. 환자(DB)의 심박수(TPS), 혈압(I/O), 호흡(CPU)이 실시간으로 그래프(뷰)에 나타나는 것입니다. 의사(DBA)는 이 모니터를 보고 어디가 아픈지 즉시 진단하여 처방을 내릴 수 있습니다. 환자가 퇴원(DB 종료)하면 모니터 기록도 초기화되는 것처럼 메모리 상에서만 존재하는 살아있는 정보입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Wait Events]**: DB 성능 저하의 구체적인 원인을 나타내는 지표.
- **[AWR (Automatic Workload Repository)]**: 동적 성능 뷰의 스냅샷을 모아 장기 분석하는 저장소.
- **[LATCH]**: 메모리 자원 획득을 위한 내부 경쟁 상태를 보여주는 뷰 정보.

📢 **마무리 요약**: **Dynamic Performance Views**는 데이터베이스의 속살입니다. 숫자가 보내는 실시간 신호를 정확히 해석할 때, 우리는 장애를 예방하고 시스템의 가치를 극대화할 수 있습니다.
