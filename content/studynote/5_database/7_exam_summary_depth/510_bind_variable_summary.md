# 510. 바인드 변수(Bind Variable) - 파싱 부하의 완벽한 통제

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 바인드 변수는 SQL 문장 내의 가변적인 조건값을 상수로 직접 기술(Literal)하는 대신 **변수(?, :var)로 처리하여, 동일한 SQL 문장이 라이브러리 캐시(Library Cache)에서 재사용되도록 유도하는 기술**이다.
> 2. **가치**: 매번 새로운 실행 계획을 수립하는 **하드 파싱(Hard Parsing)을 방지**하여 CPU 자원 소모를 획기적으로 줄이며, 대규모 동시 접속 환경에서 공유 풀(Shared Pool) 경합을 최소화한다.
> 3. **융합**: SQL 공유(SQL Sharing) 매커니즘과 융합되어 시스템의 확장성을 높이며, 보안적으로는 SQL 인젝션 공격을 원천 차단하는 방어막 역할을 한다.

+++

### Ⅰ. 하드 파싱 vs 소프트 파싱의 메커니즘

- **하드 파싱 (Hard Parsing)**: SQL이 라이브러리 캐시에 없어 문법 검사, 권한 확인, 최적화(실행 계획 생성) 과정을 모두 거치는 무거운 작업.
- **소프트 파싱 (Soft Parsing)**: SQL 텍스트가 이미 캐시에 존재하여 수립된 실행 계획을 즉시 재사용하는 가벼운 작업.
- **바인드 변수의 기여**: SQL 문장을 '틀'로 인식하게 하여, 값만 바뀌어도 시스템이 '같은 SQL'로 판별하게 함으로써 소프트 파싱을 극대화합니다.

+++

### Ⅱ. 바인드 변수 활용 전후 비교 (ASCII Model)

```text
[ Literal SQL: Hard Parsing ]         [ Bind Variable: Soft Parsing ]

  SQL 1: SELECT * FROM User WHERE ID=1;  SQL 틀: SELECT * FROM User WHERE ID=:id; ✅
  SQL 2: SELECT * FROM User WHERE ID=2;
  (Text가 달라 매번 새로 최적화)           (변수값만 바꿔서 실행 계획 재사용) ✅

  [ Library Cache (Shared Pool) ]       [ Library Cache (Shared Pool) ]
  ┌───────────────────────────┐         ┌───────────────────────────┐
  │ Plan for ID=1             │         │ Shared Execution Plan     │ ✅
  │ Plan for ID=2             │         └───────────────────────────┘
  └───────────────────────────┘         (메모리 절약 및 CPU 부하 급감)
```

+++

### Ⅲ. 실무적 의의 및 주의사항 (Bind Peeking)

- **성능적 강점**: 수만 명의 사용자가 동시에 접속하는 온라인 트랜잭션(OLTP) 환경에서 바인드 변수를 쓰지 않으면, 시스템은 파싱 부하만으로 마비될 수 있습니다.
- **주의사항 (Bind Peeking)**: 옵티마이저가 첫 번째 입력된 변수값을 기준으로 계획을 세우는 현상입니다. 데이터 분포가 불균형한 경우(예: 특정 값의 데이터가 90%인 경우) 엉뚱한 실행 계획이 고정되어 성능이 나빠질 수 있습니다 (Adaptive Cursor Sharing 등으로 보완).

- **📢 섹션 요약 비유**: 바인드 변수는 **'식당의 인쇄된 주문서'**와 같습니다. 매번 손님이 올 때마다 메뉴와 가격을 종이에 새로 적어서(하드 파싱) 주방에 전달하는 대신, 메뉴 항목은 인쇄되어 있고 빈칸(바인드 변수)에 수량만 적어서 전달하는 것과 같습니다. 주문 처리 속도가 훨씬 빨라지고 종이(메모리)도 아끼는 가장 효율적인 방식입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Shared Pool]**: SQL과 실행 계획이 공유되는 메모리 영역.
- **[Cursor]**: 파싱된 결과가 담긴 메모리 내의 실행 객체.
- **[SQL Injection]**: 바인드 변수를 통해 예방 가능한 대표적 보안 취약점.

📢 **마무리 요약**: **Bind Variable**은 고성능 데이터베이스의 기초 문법입니다. SQL을 개별적인 명령이 아닌 '공유 가능한 지식'으로 전환함으로써, 데이터베이스의 잠재력을 극한으로 끌어올립니다.
