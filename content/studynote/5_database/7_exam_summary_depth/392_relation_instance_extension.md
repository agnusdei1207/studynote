# 392. 릴레이션 인스턴스(Extension) - 데이터의 생동하는 실체

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 릴레이션 인스턴스(Instance)는 특정 시점에 데이터베이스 테이블에 실제로 저장되어 있는 **튜플(행, Row)들의 집합**을 의미한다.
> 2. **가치**: 스키마가 정적인 '틀'이라면, 인스턴스는 삽입, 삭제, 수정에 따라 끊임없이 변하는 **동적인 외연(Extension)**적 속성을 지닌다.
> 3. **융합**: 트랜잭션의 결과물이 인스턴스에 반영되며, 질의(Query)의 최종 응답 결과 또한 스키마 기반의 가공된 인스턴스 형태로 사용자에게 전달된다.

+++

### Ⅰ. 릴레이션 인스턴스의 특징

- **동적 성격**: 데이터의 추가(INSERT)나 삭제(DELETE)가 일어날 때마다 인스턴스의 상태는 변합니다.
- **카디널리티 (Cardinality)**: 인스턴스에 포함된 튜플의 총 개수를 의미합니다.
- **무결성 준수**: 인스턴스 내의 모든 튜플은 해당 릴레이션 스키마가 정의한 제약 조건(도메인, 키 등)을 반드시 만족해야 합니다.

+++

### Ⅱ. 인스턴스 구성 시각화 (ASCII Model)

스키마라는 틀 안에 담긴 실제 데이터들의 모습입니다.

```text
[ Relation: Student ]

  (Schema Area)
  ┌──────────┬──────────┬──────────┬──────────┐
  │ Stud_ID  │ Name     │ Major    │ Grade    │
  ├──────────┼──────────┼──────────┼──────────┤ (Instance Area)
  │ 2026001  │ Kim      │ CS       │ 4        │ <-- Tuple 1
  │ 2026002  │ Lee      │ Math     │ 2        │ <-- Tuple 2
  │ 2026003  │ Park     │ Art      │ 3        │ <-- Tuple 3
  └──────────┴──────────┴──────────┴──────────┘
  (Cardinality: 3)
```

+++

### Ⅲ. 스키마(Intension)와 인스턴스(Extension) 비교

| 비교 항목 | 릴레이션 스키마 (내포) | 릴레이션 인스턴스 (외연) |
|:---|:---|:---|
| **비유** | 아파트 설계도, 붕어빵 틀 | **입주한 주민, 실제 붕어빵** |
| **변화 빈도** | 거의 변하지 않음 (정적) | **매우 빈번하게 변함 (동적)** |
| **정의 위치** | 데이터 사전 (Data Dictionary) | 데이터 파일 (Data Files) |

- **📢 섹션 요약 비유**: 릴레이션 인스턴스는 **'설계도대로 지어진 아파트에 살고 있는 사람들'**과 같습니다. 설계도(스키마)는 그대로지만, 이사 오는 사람(INSERT)과 이사 나가는 사람(DELETE)에 따라 아파트 안의 풍경은 매일 달라집니다. 우리는 인스턴스를 통해 데이터베이스의 현재 모습을 확인하게 됩니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Tuple]**: 인스턴스를 구성하는 개별 행.
- **[State]**: 특정 시점의 인스턴스 전체 모습 (Database State).
- **[Cardinality]**: 인스턴스의 크기를 나타내는 척도.

📢 **마무리 요약**: **Relation Instance**는 데이터베이스의 살아있는 데이터 그 자체입니다. 스키마라는 약속을 실천하는 구체적인 값들의 모임이 비즈니스의 가치를 증명합니다.
