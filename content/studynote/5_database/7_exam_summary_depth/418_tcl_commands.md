# 418. TCL(Transaction Control Language) - 무결성의 마침표

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TCL(트랜잭션 제어 언어)은 DML에 의해 변경된 내용을 **최종적으로 데이터베이스에 확정 짓거나, 취소하여 원래 상태로 되돌리는 제어 명령어**들의 집합이다.
> 2. **가치**: 트랜잭션의 원자성(Atomicity)을 물리적으로 구현하며, 논리적인 작업 단위가 완벽하게 성공했을 때만 영구 저장되도록 보장하여 데이터 정합성을 수호한다.
> 3. **융합**: COMMIT(확정), ROLLBACK(취소), SAVEPOINT(임시 저장)가 유기적으로 융합되어, 복잡한 비즈니스 로직 처리 중 발생할 수 있는 예외 상황에 유연하게 대응한다.

+++

### Ⅰ. TCL의 주요 명령어와 작동 원리

1. **COMMIT (확정)**: 
    - 트랜잭션 내의 모든 변경 사항을 영구적으로 저장합니다. 
    - 실행 후 모든 락(Lock)이 해제되며, 변경된 결과는 다른 모든 사용자가 볼 수 있게 됩니다.
2. **ROLLBACK (취소)**: 
    - 트랜잭션 시작 이후 또는 마지막 커밋 이후의 모든 변경 사항을 취소하고 이전 상태로 원복합니다.
3. **SAVEPOINT (저장점)**: 
    - 트랜잭션 중간에 책갈피를 표시합니다. 전체 취소가 아닌 특정 시점까지만 롤백할 수 있게 해 줍니다.

+++

### Ⅱ. TCL 프로세스 시각화 (ASCII Flow)

```text
[ Transaction Control Workflow ]

  Transaction Start
       │
  1. INSERT (Data A)
  2. UPDATE (Data B)
       │
  3. SAVEPOINT SP1; 🚩 (책갈피)
       │
  4. DELETE (Data C) -- 💥 "실수했다!"
       │
  5. ROLLBACK TO SP1; ──▶ 🚩 지점으로 복구 (1, 2번 작업은 유지)
       │
  6. Final Check...
       │
  7. COMMIT; ✅ ──▶ [ Database Disk ] 💾 (영구 저장 완료)
```

+++

### Ⅲ. TCL 사용 시 주의사항

- **자동 커밋 주의**: 많은 툴들이 기본적으로 'Auto-commit'이 활성화되어 있습니다. 운영 환경에서는 이를 끄고 명시적으로 TCL을 사용하는 것이 안전합니다.
- **락 점유 시간**: `COMMIT`이나 `ROLLBACK`을 날리지 않고 트랜잭션을 오래 끌면, 해당 데이터에 걸린 락이 풀리지 않아 다른 사용자의 업무를 마비시킬 수 있습니다 (Blocking).

- **📢 섹션 요약 비유**: TCL은 **'문서 편집기의 저장과 취소 버튼'**과 같습니다. 글을 열심히 쓰는 과정(DML) 중에 '뒤로 가기(ROLLBACK)'를 할 수 있고, 중요한 대목마다 '임시 저장(SAVEPOINT)'을 해둘 수 있습니다. 최종적으로 '파일 저장(COMMIT)' 버튼을 누르는 순간, 그동안의 모든 노력이 공식 문서로 확정되는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[ACID]**: TCL이 지탱하는 트랜잭션의 핵심 4대 속성.
- **[Redo/Undo]**: TCL 명령 시 내부적으로 돌아가는 회복 엔진 메커니즘.
- **[Atomicity]**: TCL의 가장 큰 존재 가치 (전부 아니면 전무).

📢 **마무리 요약**: **TCL**은 데이터베이스의 안전장치입니다. 신중한 확정과 과감한 취소를 통해 데이터의 진실성을 지켜내는 것이 전문가의 소양입니다.
