# 575. 느리게 변하는 차원(SCD) - 시간의 흐름을 기록하는 기술

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SCD(Slowly Changing Dimension)는 차원 데이터의 속성 값이 시간이 지남에 따라 변할 때, **과거의 이력 정보를 어떻게 보존하거나 갱신할 것인지에 대한 설계 전략**이다.
> 2. **가치**: 과거의 분석 수치가 현재의 변화로 인해 왜곡되는 것을 방지하여 **역사적 정합성(Historical Integrity)**을 유지하며, 데이터의 변화 추적을 통한 시계열 분석을 가능케 한다.
> 3. **융합**: 대리 키(Surrogate Key)와 유효 기간 필드 기술이 융합되어, 정적인 마스터 데이터를 동적인 이력 데이터로 변환하는 DW의 핵심 매커니즘을 형성한다.

+++

### Ⅰ. SCD의 주요 3대 유형 비교

| 유형 | 방식 | 특징 | 비고 |
|:---:|:---|:---|:---|
| **Type 1** | **Overwrite** (덮어쓰기) | 과거 이력을 버리고 최신 값으로 갱신 | 가장 단순, 이력 분석 불가 |
| **Type 2** | **New Row** (행 추가) | 새로운 행을 추가하고 유효 기간 설정 | **DW의 표준 방식**, 완벽한 이력 |
| **Type 3** | **New Column** (열 추가) | '이전 값' 컬럼을 별도로 생성 | 현재와 바로 전 단계만 비교 가능 |

+++

### Ⅱ. SCD Type 2 작동 매커니즘 시각화 (ASCII Model)

고객 'Kim'의 등급이 'Silver'에서 'Gold'로 상향된 경우입니다.

```text
[ SCD Type 2: Historical Versioning ]

  (Step 1: Before Change)
  ┌────────┬──────────┬──────────┬──────────┬──────────┐
  │  SK    │ CustID   │ Grade    │ Start_DT │ End_DT   │
  ├────────┼──────────┼──────────┼──────────┼──────────┤
  │  101   │ C-001    │ Silver   │ 2025-01  │ 9999-12  │ ✅ (Current)
  └────────┴──────────┴──────────┴──────────┴──────────┘

  (Step 2: After Change at 2026-03)
  ┌────────┬──────────┬──────────┬──────────┬──────────┐
  │  SK    │ CustID   │ Grade    │ Start_DT │ End_DT   │
  ├────────┼──────────┼──────────┼──────────┼──────────┤
  │  101   │ C-001    │ Silver   │ 2025-01  │ 2026-02  │ 🔒 (Expired)
  │  205   │ C-001    │ Gold     │ 2026-03  │ 9999-12  │ ✅ (Current)
  └────────┴──────────┴──────────┴──────────┴──────────┘
```

+++

### Ⅲ. 실무적 의의: 분석의 정확도

- **과거 시점 분석**: "2025년에 실적이 좋았던 Silver 등급 고객 리스트"를 뽑을 때, 현재 Gold가 된 Kim을 정확히 Silver 집단에 포함할 수 있습니다.
- **성능 고려**: Type 2는 데이터량이 계속 늘어나므로, 팩트 테이블과 조인 시 인덱스 전략 및 파티셔닝 기술이 함께 뒷받침되어야 합니다.

- **📢 섹션 요약 비유**: SCD 유형은 **'일기장 쓰기'**와 같습니다. Type 1은 틀린 글자를 지우개로 지우고 다시 쓰는 것이고(과거 소멸), Type 2는 다음 장에 새로운 이야기를 계속 써 내려가는 것이며(역사 보존), Type 3은 원래 문장 옆에 작은 글씨로 옛날 내용을 메모해두는 방식입니다. 우리 인생(데이터)의 굴곡을 완벽히 이해하려면 한 장씩 새로 쓰는 Type 2가 가장 정석입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Surrogate Key]**: 동일한 비즈니스 키(CustID)를 가진 여러 버전을 구분하기 위한 필수 도구.
- **[Current Flag]**: 수많은 행 중 어떤 것이 '지금'의 진실인지 알려주는 지표.
- **[Effective Date]**: 데이터의 유효성을 결정짓는 시간의 경계선.

📢 **마무리 요약**: **SCD**는 데이터 웨어하우스의 기억법입니다. 변하지 않는 것은 없다는 진리를 인정하고, 그 변화의 궤적을 정교하게 기록할 때 데이터는 비로소 지혜가 됩니다.
