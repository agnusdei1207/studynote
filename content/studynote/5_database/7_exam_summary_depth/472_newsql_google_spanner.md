# 472. NewSQL과 구글 스패너 - 글로벌 일관성의 정점

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: NewSQL은 전통적인 RDBMS의 **ACID 트랜잭션과 SQL 인터페이스를 유지하면서, NoSQL의 수평적 확장성(Scale-out)을 제공**하는 차세대 데이터베이스 아키텍처다.
> 2. **가치**: 구글 스패너(Spanner)는 **TrueTime(원자시계+GPS)** 기술을 통해 전 세계에 분산된 노드 간의 시간 오차를 극복하고, 글로벌 스케일에서도 강력한 외부 일관성(External Consistency)을 보장한다.
> 3. **융합**: 분산 합의 알고리즘(Paxos)과 클라우드 네이티브 스토리지 기술이 융합되어, "성능을 위해 일관성을 포기해야 한다"는 CAP 정리의 실무적 한계를 기술로 극복했다.

+++

### Ⅰ. 구글 스패너의 핵심 아키텍처

- **Global Scale**: 데이터를 전 지구적 규모로 자동 샤딩 및 복제합니다.
- **TrueTime API**: 전 세계 모든 서버에 설치된 원자시계와 GPS를 이용해, 각 트랜잭션에 전역적으로 정렬 가능한 타임스탬프를 부여합니다.
- **분산 트랜잭션**: 2단계 커밋(2PC)과 팍소스(Paxos)를 결합하여 높은 가용성과 일관성을 동시에 달성합니다.

+++

### Ⅱ. TrueTime 기반 일관성 매커니즘 (ASCII Flow)

```text
[ Spanner TrueTime & Commit Logic ]

  1. Write Request at Node A
  2. Ask TrueTime: "지금 몇 시니?"
  3. TrueTime: "[T_earliest, T_latest]" (오차 범위 제공)
  4. Node A: Commit Time = T_latest 설정
  5. **Commit Wait**: T_latest 시점이 실제로 지나갈 때까지 대기 ⏳
  6. 결과: 전 세계 누구도 이 시간보다 과거의 데이터를 볼 수 없음 ✅
  ──▶ 글로벌 정합성(Strong Consistency) 완성!
```

+++

### Ⅲ. 실무적 의의: 왜 NewSQL인가?

- **RDBMS의 신뢰**: 복잡한 비즈니스 로직과 조인 연산을 그대로 사용 가능합니다.
- **NoSQL의 확장**: 서비스 성장에 맞춰 서버를 추가하기만 하면 무한 확장이 가능합니다.
- **운영 효율**: 수동 샤딩의 고통에서 해방되어 시스템이 스스로 데이터의 균형을 맞춥니다.

- **📢 섹션 요약 비유**: 구글 스패너는 **'전 세계를 하나로 연결하는 거대한 정밀 시계'**와 같습니다. 나라마다 시간이 조금씩 다르듯 서버마다 시계가 다르지만, 스패너는 원자시계라는 절대 기준을 세워 "누가 먼저 글을 썼는지"를 전 지구적으로 정확하게 판별해 내는 놀라운 장치입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[TrueTime]**: 스패너의 심장이자 일관성의 근거.
- **[Paxos Group]**: 데이터 복제와 합의를 담당하는 기본 단위.
- **[External Consistency]**: 트랜잭션이 완료된 순서대로 누구나 데이터를 보게 되는 최상위 일관성.

📢 **마무리 요약**: **NewSQL & Spanner**는 분산 DB 기술의 정점입니다. 정교한 시간 통제와 합의 알고리즘을 통해, 데이터베이스의 '완벽함'과 '거대함'이 공존할 수 있음을 증명했습니다.
