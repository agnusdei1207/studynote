# 552. 일관된 해싱(Consistent Hashing) - 데이터 이동의 최소화

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 일관된 해싱은 클러스터의 크기가 변할 때(노드 추가/삭제), 전체 데이터를 재배치하는 대신 **오직 $K/N$(K:키 개수, N:노드 수) 만큼의 데이터만 이동하게 하는 지능형 분산 알고리즘**이다.
> 2. **가치**: 서버 증설 시 시스템 전체의 데이터를 옮겨야 하는 '리밸런싱 폭풍'을 방지하여, **서비스 무중단 수평 확장(Scale-out)**을 실무적으로 가능케 한다.
> 3. **융합**: 가상 노드(Virtual Nodes) 기술과 융합되어 노드 간 부하 불균형을 해소하며, Cassandra, DynamoDB, CDN 캐시 시스템의 아키텍처적 근간이 된다.

+++

### Ⅰ. 일관된 해싱의 리밸런싱 원리

- **전통적 해싱 (`Hash % N`)**: 서버 대수 $N$이 바뀌면 거의 모든 데이터의 저장 위치가 바뀌어 버립니다 (전체 이동).
- **일관된 해싱 (Hash Ring)**: 
    1. 데이터와 서버를 모두 가상의 원형 링(Ring) 위에 배치합니다.
    2. 데이터는 시계 방향으로 가장 가까운 서버에 저장됩니다.
    3. **노드 추가 시**: 새 노드와 그 인접 노드 사이의 데이터만 이동합니다. 나머지 노드의 데이터는 꿈적도 하지 않습니다. ✅

+++

### Ⅱ. 노드 추가 시 데이터 이동 시각화 (ASCII Model)

```text
[ Consistent Hashing Ring: Node Addition ]

          (Hash Value: 0)
               ▲
        ┌──────┴──────┐
     [Node 3]      [Node 1]
        │             │
        │      (K1)──▶│ (Key 1은 원래 Node 1 소유)
        │             │
     [Node 2] ◀── [New Node!] ✅ (새 노드 투입)
        └──────┬──────┘
               ▼
  * 결과: Node 2가 가졌던 데이터 중 일부만 New Node로 이동.
  * Node 1, 3의 데이터는 이동량 0! ✅ "Minimal Rebalancing"
```

+++

### Ⅲ. 실무적 이점과 가상 노드

- **탄력적 확장**: 트래픽 증가 시 실시간으로 노드를 추가해도 시스템 부하가 매우 적습니다.
- **가상 노드 (Virtual Nodes)**: 하나의 물리 서버를 링 위의 여러 지점에 가상으로 배치함으로써, 특정 노드가 더 많은 데이터를 가지게 되는 불균형(Skew)을 확률적으로 해결합니다.

- **📢 섹션 요약 비유**: 일관된 해싱은 **'의자 뺏기 게임의 역발상'**과 같습니다. 일반 해싱은 사람이 한 명 늘어날 때마다 모든 사람이 자리를 옮겨야 하지만, 일관된 해싱은 내 옆에 의자가 하나 더 생기면 나만 살짝 옆으로 옮겨 앉고 다른 사람들은 원래 자리에 가만히 있게 하는 효율적인 자리 배치법입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Hash Ring]**: 서버와 데이터를 하나로 묶는 논리적 공간.
- **[Rebalancing]**: 클러스터 변화에 대응하는 데이터 재배치 과정.
- **[Virtual Nodes]**: 물리적 하드웨어의 불균형을 해소하는 논리적 분신 기술.

📢 **마무리 요약**: **Consistent Hashing**은 분산 시스템의 평화 유지군입니다. 변화에 민감하게 반응하지 않으면서도 시스템을 안정적으로 키워나갈 수 있게 돕는 핵심 지혜입니다.
