# 505. 마이크로 세그멘테이션 DB 보안 - 격리의 미학

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 마이크로 세그멘테이션(Micro-segmentation)은 네트워크나 데이터베이스 내부를 **매우 작은 단위(테이블, 행, 특정 서비스 등)로 쪼개어 각각 독립적인 보안 정책을 적용하는 미세 격리 기술**이다.
> 2. **가치**: 침해 사고 발생 시 공격자가 옆에 있는 다른 자산으로 이동하는 **횡적 이동(Lateral Movement)**을 원천 차단하여, 피해 범위를 최소화(Blast Radius Reduction)하고 데이터 유출을 방어한다.
> 3. **융합**: 제로 트러스트(Zero Trust) 아키텍처와 융합되어, "한 번 통과하면 다 안전하다"는 경계 보안의 한계를 극복하는 현대 클라우드 보안의 핵심 전략으로 작용한다.

+++

### Ⅰ. DB 보안에서의 마이크로 세그멘테이션

- **전통적 방식**: DB 서버 전체를 하나의 보안 그룹(Security Group)으로 묶어 관리. (한 곳 뚫리면 DB 전체 노출)
- **마이크로 방식**: 
    - **서비스별 격리**: 주문 서비스는 주문 테이블만, 결제 서비스는 결제 테이블만 볼 수 있도록 네트워크 단에서 분리.
    - **행 단위 격리**: 동일 테이블 내에서도 사용자 권한에 따라 특정 행(Row)에만 접근할 수 있도록 동적으로 격리.

+++

### Ⅱ. 세그멘테이션 보안 구조 시각화 (ASCII Model)

```text
[ Micro-segmentation: Stopping Lateral Movement ]

  (External World)
      │
  ┌───▼───┐ [ Traditional Firewall ] (Castle Wall)
  │       │
  └───┬───┘
      │ (If breached here...)
      ▼
  ┌──────────────────────────────────────────────────────────┐
  │  Micro-segments (Individual Vaults)                      │

  │  ┌──────────┐      ┌──────────┐      ┌──────────┐        │
  │  │ Order DB │ ◀──X─▶│ User DB  │ ◀──X─▶│ Payment  │        │ ✅
  │  │ Segment  │      │ Segment  │      │ Segment  │        │
  │  └──────────┘      └──────────┘      └──────────┘        │
  └──────────────────────────────────────────────────────────┘
  * "각 세그먼트 사이에는 개별적인 통제 규칙이 존재함" 🔒
```

+++

### Ⅲ. 마이크로 세그멘테이션의 주요 장점

1. **최소 권한 원칙 실현**: 모든 데이터가 아닌 업무에 '딱 필요한 조각'에만 접근 경로를 열어줍니다.
2. **가시성 확보**: 세그먼트 간의 모든 통신 흐름을 모니터링하여 평소와 다른 이상 징후를 즉시 포착합니다.
3. **규제 준수**: 민감 정보(개인정보, 카드정보)가 포함된 구역만 따로 떼어내어 고도의 보안을 집중 적용함으로써 컴플라이언스 비용을 낮춥니다.

- **📢 섹션 요약 비유**: 마이크로 세그멘테이션은 **'잠수함의 격벽'**과 같습니다. 커다란 배 전체가 하나의 통으로 되어 있으면 구멍 하나만 나도 배 전체가 침몰하지만, 잠수함처럼 내부를 수많은 작은 방(세그먼트)으로 나누어 문(보안 정책)을 잠가두면, 한 방에 물이 차더라도(해킹 발생) 다른 방으로 번지지 않아 배 전체의 안전을 지킬 수 있는 원리입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Zero Trust]**: 마이크로 세그멘테이션을 필요로 하는 상위 보안 철학.
- **[Lateral Movement]**: 공격자가 내부망에서 정보를 찾아 수평으로 이동하는 행위.
- **[Policy Enforcement Point (PEP)]**: 세그먼트 간의 경계에서 정책을 집행하는 지점.

📢 **마무리 요약**: **Micro-segmentation**은 데이터 보안의 정교함을 상징합니다. 거대한 데이터를 잘게 쪼개어 각각 보호할 때, 비로소 빈틈없는 완벽한 데이터 요새가 완성됩니다.
