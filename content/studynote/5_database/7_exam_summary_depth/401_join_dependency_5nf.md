# 401. 조인 종속과 제5정규형(5NF) - 분해의 최종 단계

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제5정규형(5NF) 또는 PJ/NF(Project-Join Normal Form)는 릴레이션이 **더 이상 무손실 분해(Lossless Decomposition)를 할 수 없는 마지막 정규형**으로, 모든 조인 종속(Join Dependency)이 후보키를 통해서만 성립하는 상태를 말한다.
> 2. **가치**: 데이터를 쪼개서 저장했다가 나중에 다시 합쳤을 때(Join), 원래 없던 가짜 데이터(Spurious Tuple)가 생기는 것을 방지하여 완벽한 데이터 재구성을 보장한다.
> 3. **융합**: 고도의 논리적 설계 기술이며, 주로 N항 관계(N-ary Relationship)가 복잡하게 얽힌 특수한 모델링 상황에서 정합성을 수호하기 위해 적용된다.

+++

### Ⅰ. 조인 종속 (Join Dependency)의 정의

- **개념**: 릴레이션 $R$을 $R_1, R_2, \dots, R_n$으로 분해했다가 다시 조인했을 때 원래의 $R$과 똑같아지는 성질을 말합니다.
- **5NF의 규칙**: 릴레이션 $R$에 조인 종속이 존재할 때, 분해된 모든 릴레이션들이 $R$의 후보키를 포함하고 있어야 합니다.
- **특이점**: 4NF가 '두 개의 독립된 다치 종속'을 다룬다면, 5NF는 '세 개 이상의 속성이 순환적으로 얽힌 관계'를 다룹니다.

+++

### Ⅱ. 5정규화(조인 종속 제거) 시각화 (ASCII Model)

세 속성이 서로 얽혀 있어 두 개씩 짝지어 분해하면 정보가 왜곡되는 사례입니다.

```text
[ 4NF (with Cycle Dependency) ]
  ┌────────┬──────────┬──────────┐
  │ 보급업자│ 부품      │ 프로젝트  │
  ├────────┼──────────┼──────────┤
  │ S1     │ P1       │ J1       │
  └────────┴──────────┴──────────┘
  * 만약 (S1, P1), (P1, J1), (J1, S1)으로 쪼갰다가 합쳤을 때 
    원래 없던 (S1, P1, J2) 같은 행이 생긴다면 조인 종속 위반! 💥

[ 5NF (Normalized) ]
  세 개의 테이블로 완벽히 분해하여 순환 관계를 명확히 함 ✅
  1. (보급업자, 부품)
  2. (부품, 프로젝트)
  3. (프로젝트, 보급업자)
```

+++

### Ⅲ. 실무적 의의: 정규화의 끝

- **실무 적용**: 현실에서는 3NF나 BCNF까지만 진행해도 대부분의 이상 현상이 해결되므로, 5NF는 학술적으로 완벽함을 추구하거나 극도로 복잡한 자원 관리 모델에서만 제한적으로 쓰입니다.
- **무손실 결합**: 분해는 쉽지만, 정보의 손실 없이 다시 합칠 수 있는 구조를 만드는 것이 데이터베이스 설계의 정수임을 시사합니다.

- **📢 섹션 요약 비유**: 제5정규형은 **'퍼즐 조각 완벽하게 맞추기'**와 같습니다. 큰 사진을 조각조각 잘랐을 때(분해), 나중에 조각들을 합쳤는데 원래 사진에 없던 이상한 모양(가짜 데이터)이 나타나면 안 됩니다. 각 조각이 고유한 위치 정보(후보키)를 완벽히 가지고 있어, 어떻게 합쳐도 원래 사진으로만 돌아오게 만드는 가장 정교한 설계 단계입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Lossless Join]**: 분해 후 조인 시 데이터 보존 법칙.
- **[PJ/NF]**: Project-Join Normal Form의 약자.
- **[Spurious Tuple]**: 잘못된 분해와 조인으로 발생하는 가짜 데이터.

📢 **마무리 요약**: **5NF**는 정규화 이론의 마침표입니다. 데이터 분해와 합체의 수학적 정당성을 부여하여, 어떤 상황에서도 진실된 데이터만을 보여줄 수 있도록 보장합니다.
