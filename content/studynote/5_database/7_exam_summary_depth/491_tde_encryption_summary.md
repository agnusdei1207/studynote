# 491. 데이터 암호화 TDE - 투명한 저장 데이터 보호

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TDE(Transparent Data Encryption)는 데이터베이스 엔진이 디스크에 저장되는 **데이터 파일(At-rest)을 실시간으로 자동 암호화하고, 읽어올 때 자동으로 복호화**하는 보안 기술이다.
> 2. **가치**: 애플리케이션 수정이 필요 없는 **'투명성'**을 제공하며, 물리적인 디스크 탈취나 백업 파일 유출로부터 핵심 자산을 보호하는 엔터프라이즈 보안의 표준이다.
> 3. **융합**: 마스터 키(Master Key)와 데이터 키(DEK)의 이중 계층 구조를 사용하여 키 관리의 안전성을 확보하며, 하드웨어 보안 모듈(HSM)과 융합되어 보안성을 극대화한다.

+++

### Ⅰ. TDE의 작동 매커니즘

- **암호화 시점**: 메모리 버퍼의 더티 페이지가 디스크로 쓰여질 때(Write) 즉시 암호화.
- **복호화 시점**: 데이터가 디스크에서 메모리로 읽혀질 때(Read) 즉시 복호화.
- **특징**: 인가된 사용자는 평소와 동일한 SQL을 사용하며, 암호화 여부를 인지하지 못함.

+++

### Ⅱ. TDE 키 계층 구조 시각화 (ASCII Model)

```text
[ TDE Key Hierarchy ]

  ┌─────────────────────────────────┐
  │  External Key Store (KMS/HSM)   │ ──▶ [ Master Key (MK) ] 🔒
  └───────────────┬─────────────────┘
                  │ (Encrypts)
                  ▼
  ┌─────────────────────────────────┐
  │  Database Wallet / Metadata     │ ──▶ [ Encrypted DEK ]
  └───────────────┬─────────────────┘
                  │ (Decrypts to Mem)
                  ▼
  ┌─────────────────────────────────┐
  │  Physical Storage (.dbf)        │ ──▶ [ Encrypted Blocks ] 💾
  └─────────────────────────────────┘
```

+++

### Ⅲ. 실무적 의의 및 장점

- **성능 최적화**: 엔진 레벨에서 암호화가 일어나므로 인덱스 검색 성능을 그대로 유지할 수 있습니다.
- **규제 준수**: 개인정보 보호법이나 PCI-DSS 등에서 요구하는 '저장 데이터 암호화' 요건을 가장 효율적으로 충족합니다.
- **관리 편의성**: 테이블 단위, 테이블스페이스 단위로 손쉽게 암호화 정책을 적용할 수 있습니다.

- **📢 섹션 요약 비유**: TDE는 **'특수 잠금장치가 달린 일기장'**과 같습니다. 내가 글을 쓸 때는(메모리) 평소처럼 쓰지만, 일기장을 덮고 금고(디스크)에 넣는 순간 자동으로 암호가 걸린 비밀 문서로 바뀝니다. 누군가 일기장을 통째로 훔쳐가도, 금고 열쇠(Master Key)가 없으면 내용을 절대 볼 수 없게 보호해 주는 장치입니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[DEK (Data Encryption Key)]**: 실제 데이터를 암호화하는 대칭키.
- **[Wallet]**: 마스터 키나 암호화된 DEK를 보관하는 보안 컨테이너.
- **[AES-256]**: TDE에서 주로 사용하는 강력한 표준 암호화 알고리즘.

📢 **마무리 요약**: **TDE**는 저장 데이터 보안의 마침표입니다. 물리적 보안의 빈틈을 메워주는 가장 투명하고 강력한 방패입니다.
