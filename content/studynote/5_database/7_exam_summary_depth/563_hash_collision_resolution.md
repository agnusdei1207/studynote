# 563. 해시 충돌과 해결 기법 - 체이닝과 개방 주소법의 승부

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 해시 충돌(Collision)은 서로 다른 두 개 이상의 키가 해시 함수를 통해 **동일한 버킷(Bucket) 주소로 매핑되는 현상**이며, 이를 해결하기 위해 체이닝(Chaining)이나 개방 주소법(Open Addressing)이 사용된다.
> 2. **가치**: 충돌 발생 시 조회 성능이 $O(1)$에서 $O(N)$으로 급락하는 것을 방지하며, 데이터의 밀도와 메모리 가용성에 따라 최적의 해결 방식을 선택하여 **인덱스 효율**을 유지한다.
> 3. **융합**: DB 인메모리 인덱스 관리와 캐시 시스템의 핵심 알고리즘으로 융합되며, 데이터 분포도에 따라 해시 함수를 재설계하거나 테이블 크기를 늘리는 리해싱(Rehashing) 기술과 결합된다.

+++

### Ⅰ. 해시 충돌 해결의 2대 핵심 기법

1. **체이닝 (Chaining - 폐쇄 주소법)**:
    - **원리**: 각 버킷을 연결 리스트(Linked List)의 헤더로 사용. 충돌 시 리스트 뒤에 데이터를 매담.
    - **장점**: 저장 공간이 유연하며 삭제가 쉬움.
2. **개방 주소법 (Open Addressing)**:
    - **원리**: 충돌 발생 시 다른 빈 버킷을 찾아 데이터를 저장.
    - **세부 기법**: 
        - **선형 탐사 (Linear Probing)**: 바로 다음 빈칸 탐색. (클러스터링 발생 위험)
        - **이차 탐사 (Quadratic Probing)**: 제곱수만큼 떨어진 칸 탐색.
        - **이중 해싱 (Double Hashing)**: 두 번째 해시 함수로 이동 거리 결정.

+++

### Ⅱ. 충돌 해결 방식 시각화 (ASCII Model)

```text
[ 1. Chaining ]                        [ 2. Open Addressing (Linear) ]
(Bucket 에 리스트 연결)                   (빈 자리를 찾아 이동)

  Index | Data                           Index | Data
  ------|----------                      ------|----------
    0   | [Empty]                          0   | [Empty]
    1   | [A] ──▶ [B] ✅ (Collision)       1   | [A] 
    2   | [C]                              2   | [B] ✅ (Moved from 1)
  ------|----------                      ------|----------
```

+++

### Ⅲ. 성능 오버헤드 및 DB 매핑

- **클러스터링 (Clustering)**: 개방 주소법(특히 선형 탐사)에서 특정 구역에 데이터가 뭉치면서 탐사 시간이 길어지는 현상.
- **적재율 (Load Factor)**: 전체 버킷 대비 저장된 데이터의 비율. 보통 0.7(70%)이 넘어가면 충돌이 급증하여 성능이 악화되므로 리해싱을 수행합니다.
- **DB 활용**: PostgreSQL의 Hash Index나 Redis의 내부 저장 구조 등 포인트 쿼리가 중요한 곳에서 각 기법의 특성을 고려하여 튜닝됩니다.

- **📢 섹션 요약 비유**: 해시 충돌은 **'사물함 번호 겹침'**과 같습니다. 체이닝은 번호가 겹치면 그 사물함 안에 주머니(리스트)를 달아 여러 물건을 한꺼번에 넣어두는 것이고, 개방 주소법은 내 번호 주인이 이미 있다면 옆 칸, 그다음 옆 칸(빈자리)을 찾아가서 내 물건을 넣는 것과 같습니다. 칸이 넉넉하면 옆으로 가는 게 빠르지만, 칸이 꽉 차면(높은 적재율) 빈자리를 찾느라 하루 종일 걸리는 것과 같습니다.

+++

### Ⅳ. 개념 맵 및 요약

- **[Hash Function]**: 충돌의 근본 원인을 제공하는 수식.
- **[Rehashing]**: 충돌이 너무 많아질 때 판을 다시 짜는 작업.
- **[Perfect Hashing]**: 충돌이 전혀 발생하지 않도록 설계된 이상적인 해시.

📢 **마무리 요약**: **Collision Resolution**은 해시 인덱스의 완성도를 결정합니다. 데이터의 특성을 고려하여 '깊게 쌓을 것인가(Chaining)' 아니면 '옆으로 넓게 퍼뜨릴 것인가(Open Addressing)'를 결정하는 것이 시스템 성능의 핵심입니다.
