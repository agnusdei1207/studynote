## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 분단(Network Partition)은 분산 시스템에서 네트워크 연결이 두절되어 노드 간 통신이 불가능한 상태를 말하며, 이는 현실에서不可避免하게 발생할 수 있다.
> 2. **가치**: CAP 이론에서 P(Partition Tolerance)는 반드시 선택해야 하며, 시스템 설계자는 C(일관성)와 A(가용성) 사이의 트레이드오프를 선택해야 한다.
> 3. **융합**: 네트워크 분단은 네트워크 이론의 장애 모델(Fault Model)에서 중요한 개념으로, 이중화 和 장애 检测과 관련된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념
네트워크 분단(Network Partition)은 분산 시스템에서 네트워크 장비故障나 네트워크拥塞으로 인해 일부 노드 간 통신이 불가능해진 상태다. 이는 서버故障와 달리, 서버는稼働하지만 네트워크 연결이 두절된 상태를 의미한다.

### 필요성
분산 시스템에서는 네트워크 분단이不可避免하게 발생할 수 있다. 따라서 시스템은 반드시 파티션 내성(Partition Tolerance)을 갖추어야 하며, 네트워크 분단 발생 시 C와 A 중 하나를牺牲해야 한다.

### 섹션 요약 비유
네트워크 분단은 **震焱時の通信障害**と 같くて、电话局（노드）は生きているが电话线（네트워크）が切れて互相に連絡不能になった状態다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 네트워크 분단 시 동작

```text
  ┌───────────────────────────────────────────────────────────────────────┐
  │                    네트워크 분단 시 동작 예시                               │
  ├───────────────────────────────────────────────────────────────────────┤
  │
  │   [정상 상태]                                                        │
  │                                                                       │
  │        Node A ◀──▶ Node B ◀──▶ Node C                             │
  │           │                                                         │
  │           └──▶ (모두 상호 연결)                                       │
  │
  │   [네트워크 분단 발생]                                                │
  │
  │        Node A ◀──▶ Node B    │    │    Node C                    │
  │                               │隔绝│                               │
  │                               └──▶ (Node A,B ↔ Node C 불통)        │
  │
  │   [선택 상황]                                                        │
  │
  │   CP 선택: Node C만 서비스, A와 B 응답 불가                            │
  │   AP 선택: A, B, C 모두 서비스 (단, 데이터 불일치 발생 가능)            │
  │
  └───────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 네트워크 분단이 발생하면, 시스템은 두 가지 파티션으로 나뉘어 개별적으로서비스를 계속한다. CP(Consistency + Partition Tolerance) 시스템에서는 일관성을 위해 한 파티션만 서비스하고, 다른 파티션은 응답하지 않는다. AP(Availability + Partition Tolerance) 시스템에서는 가용성을 위해 모든 파티션이서비스를 계속하지만, 데이터 불일치가 발생할 수 있다. 현실적으로 네트워크 분단은"언제 발생할지 모르는"事件이므로, 시스템 설계 단계에서 반드시 이에 대한 대응을 고려해야 한다.

### 섹션 요약 비유
네트워크 분단 대응은 ** Bizration 대피 계획**と似っていて、communicationが途絶えた状況でもbizが続けれるよう事前に準備しておく必要がある。

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 — 글로벌 서비스의 네트워크 분단**: 글로벌 분산 서비스에서 재해(地震 등)로 특정 지역과 다른 지역 간 네트워크가 두절되면, 해당 지역 사용자에 대한 서비스 계속 여부를 결정해야 한다. 금융 시스템에서는 일관성을優先하여 해당 지역 서비스를 중단하고, 소셜 미디어에서는 가용성을優先하여서비스를 계속한다.

### 섹션 요약 비유
네트워크 분단 대응은 ** biz비상時の远程근무**と似っていて、本社との通信が 不能になっても、現地で業務を続けるかどうかの 判断が 필요하다。

## Ⅴ. 결론

네트워크 분단은 분산 시스템에서不可避免한 현실이다. 따라서 시스템 설계자는 CAP 이론에 따라 시스템이 C와 A 중 어느 것을優先할지를 명확히 설계해야 한다.

### 섹션 요약 비유
네트워크 분단 대응은 **非常時の BCP (Biz継続計画)**と似っていて、日常的なbp와 다른 수준의 대처准备了 필요한다.
