# 범정부 EA 프레임워크 (GEA)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 범정부 EA(Government Enterprise Architecture)는 대한민국 공공기관의 정보화 자원을 효율적으로 관리하기 위해 비즈니스, 데이터, 서비스, 기술, 성과 등 5대 참조 모델(Reference Model)을 기반으로 수립된 국가 차원의 아키텍처 표준이다.
> 2. **가치**: 부처 간 중복 투자를 방지하고 정보 시스템의 상호운용성(Interoperability)을 확보하며, 전자정부 서비스의 통합 및 연계를 가속화하여 행정 효율성을 극대화한다.
> 3. **융합**: 행정안전부 주도로 운영되는 범정부 아키텍처 포털(GEAP)과 연동되며, 공공 정보화 사업의 기획(ISP), 예산 편성, 감리 단계에서 필수적인 통제 기준으로 작용한다.

+++

## Ⅰ. 개요 (Context & Background)

국가 기관들이 각자 시스템을 만들다 보면 "국세청 데이터와 행정안전부 데이터가 서로 맞지 않거나", "비슷한 예산 관리 시스템을 수십 개 부처가 따로 구매하는" 비효율이 발생한다. 이를 해결하기 위해 대한민국 정부는 2000년대 초반부터 **"범정부 EA (GEA)"**라는 거대한 설계도를 구축했다.

GEA는 단순히 문서화된 규정이 아니라, 법률(정보시스템 효율적 도입 및 운영에 관한 법률)에 근거한 **강제력 있는 프레임워크**다. 모든 공공기관은 정보화 사업을 추진할 때 범정부 EA의 표준을 준수해야 하며, 구축된 아키텍처 정보를 국가 중앙 저장소에 등록해야 한다. 이는 전 국가적 IT 자산을 한눈에 파악하고 관리하기 위한 "정부 IT 컨트롤 타워"의 핵심 도구다.

📢 섹션 요약 비유: 범정부 EA는 대한민국이라는 거대한 스마트 시티를 건설하기 위한 총괄 조감도와 같아서, 모든 건물이 같은 규격의 전기 코드(표준)를 쓰고 상하수도(데이터)가 끊김 없이 연결되도록 관리하는 기준입니다.

+++

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

GEA의 핵심은 '어떤 기준으로 분류할 것인가'를 정의한 **5대 참조 모델(Reference Model)**이다.

### 1. 범정부 EA 5대 참조 모델 구조 (ASCII)

```text
    ┌─────────────────────────────────────────────────────────────┐
    │  [PRM] Performance Reference Model (성과 참조 모델)         │
    │  : 정보화 사업이 행정 목표 달성에 얼마나 기여했는가?        │
    └──────────────────────────────┬──────────────────────────────┘
                                   ▼
    ┌─────────────────────────────────────────────────────────────┐
    │  [BRM] Business Reference Model (업무 참조 모델)            │
    │  : 정부가 수행하는 행정 업무의 기능적 분류는 무엇인가?      │
    └───────────────┬─────────────────────────────┬───────────────┘
                    ▼                             ▼
    ┌─────────────────────────────┐ ┌─────────────────────────────┐
    │ [DRM] Data Reference Model  │ │ [SRM] Service Reference Model│
    │ (데이터 참조 모델)          │ │ (서비스 참조 모델)          │
    │ : 교환 데이터의 표준은?     │ │ : 재사용 가능한 컴포넌트는? │
    └───────────────┬─────────────┘ └─────────────┬───────────────┘
                    ▼                             ▼
    ┌─────────────────────────────────────────────────────────────┐
    │  [TRM] Technical Reference Model (기술 참조 모델)           │
    │  : 어떤 표준 기술과 플랫폼을 사용할 것인가?                 │
    └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]**  
GEA는 비즈니스(BRM)를 중심으로 데이터(DRM)와 서비스(SRM)를 정의하고, 이를 뒷받침하는 기술(TRM)을 표준화하며, 최종적으로 이 모든 활동이 어떤 성과(PRM)를 냈는지 측정하는 수직적/수평적 통합 구조를 갖는다.

### 2. 5대 참조 모델 상세 (Table)

| 참조 모델 | 영문 명칭 | 주요 내용 | 핵심 가치 |
|:---|:---|:---|:---|
| **성과 (PRM)** | Performance | 업무/재무/사용자/학습 관점 지표 | 정보화 투자 성과 측정 |
| **업무 (BRM)** | Business | 기능별(대/중/소) 업무 분류 체계 | 업무 중복 식별 및 통합 |
| **서비스 (SRM)** | Service | 공통 서비스 부품(Component) 식별 | 시스템 간 기능 재사용 |
| **데이터 (DRM)** | Data | 데이터 분류, 교환 포맷, 용어 표준 | 기관 간 정보 공유 및 연계 |
| **기술 (TRM)** | Technical | 서비스 접근, 정보 보안, 플랫폼 기술 | 상호운용성 및 기술 표준화 |

### 3. GEA의 3대 구성 요소

1. **아키텍처 정보**: 각 기관이 등록한 도메인별(BA, DA, AA, TA) 산출물.
2. **관리 체계**: EA를 수립하고 현행화하는 조직, 프로세스, 규정.
3. **시스템(GEAP)**: 범정부 아키텍처 포털(Government Enterprise Architecture Portal)을 통한 통합 관리.

📢 섹션 요약 비유: 5대 참조 모델은 음식을 만들 때 사용하는 레시피 표준과 같아서, 어떤 부처(요리사)가 요리를 하더라도 일관된 맛(행정 서비스)과 영양(성과)을 보장하게 합니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

GEA와 민간 EA, 그리고 관련 제도와의 관계다.

### 1. 민간 EA vs 범정부 EA(GEA) 비교

| 구분 | 민간 EA (Enterprise) | 범정부 EA (GEA) |
|:---|:---|:---|
| **주요 목표** | 이윤 극대화, 시장 경쟁력 | 대국민 서비스, 예산 효율성, 상호운용성 |
| **강제성** | 기업 내부 필요에 따른 자율 | 법적 근거에 의한 의무 준수 |
| **참조 모델** | 기업 특성에 따라 자유로운 구성 | 국가 표준 5대 참조 모델 준수 |
| **관리 단위** | 단일 기업 또는 그룹사 | 국가 전체 기관 (행정, 입법, 사법 등) |
| **비유** | 자사 브랜드 매뉴얼 | 국가 건축 법규 및 공공 디자인 가이드 |

### 2. 과목 융합: 전자정부 클라우드 플랫폼과의 시너지

최근 GEA는 **"전자정부 클라우드 플랫폼"**과 융합되고 있다.
- **TRM의 변화**: 개별 기관의 하드웨어 기술 표준에서 '범정부 클라우드(G-Cloud)' 표준 기술로 통합.
- **SRM의 진화**: 기관별 기능 재사용을 넘어, 마이크로서비스(MSA) 기반의 공통 행정 서비스 API 공유 체계로 발전.

📢 섹션 요약 비유: 과거의 GEA가 각 건물의 설계도를 모으는 수준이었다면, 현대의 GEA는 모든 건물이 에너지를 공유하고 함께 관리되는 '스마트 그리드 시티'의 운영 체계로 진화하고 있습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

공공 정보화 사업 기획 및 수행 시 GEA 활용 전략이다.

### 1. 실무 시나리오: "부처 간 데이터 연계 시스템 구축"
- **DRM 적용**: 데이터 참조 모델을 통해 타 부처에서 이미 정의된 데이터 항목이 있는지 확인. 만약 있다면 중복 구축 대신 연계(Interface)를 우선 검토.
- **TRM 확인**: 우리 부처에서 도입하려는 신기술이 범정부 기술 표준(TRM)에 위배되는지 확인하여 향후 유지보수나 보안 심의 문제를 사전에 방지.

### 2. GEA 성숙도 측정 (EA Maturity Model)

범정부 EA는 각 기관의 EA 수준을 매년 측정한다.
- **계획**: EA 수립을 위한 추진력 확보.
- **구축**: 5대 참조 모델 기반의 아키텍처 정보 현행화.
- **활용**: 정보화 사업 기획 및 예산 절감에 EA 정보 활용.
- **관리**: EA 조직 및 프로세스의 지속적 운영.

### 3. 도입 시 고려해야 할 안티패턴
- **"Registry Only"**: 실제 사업 관리에는 쓰지 않고 GEAP 포털에 정보 입력만 하는 경우. -> **해결**: 정보화 예산 신청 시 EA 정보 기반의 타당성 검토 의무화.
- **"Outdated Info"**: 시스템은 바뀌었는데 EA 정보는 수년 전 내용인 경우. -> **해결**: 사업 종료 후 준공 검사 조건으로 EA 현행화(Update) 연동.

📢 섹션 요약 비유: EA 성숙도 측정은 건강검진과 같아서, 단순히 높은 점수를 받는 것이 목적이 아니라 우리 기관의 IT 체질을 개선하고 고장(비효율)을 예방하는 기회로 삼아야 합니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

GEA는 국가 디지털 플랫폼 정부(Digital Platform Government)를 지탱하는 뼈대다.

| 기대효과 | 설명 |
|:---|:---|
| **중복 투자 방지** | 국가 전체 관점의 IT 자산 현황 파악으로 예산 낭비 억제 |
| **상호운용성 강화** | 기관 간 데이터 칸막이를 허물고 원활한 정보 공유 지원 |
| **정책 결정 지원** | 정확한 IT 통계 데이터를 기반으로 한 국가 정보화 정책 수립 |

미래의 GEA는 **"AI 기반 지능형 아키텍처"**로 진화할 것이다. 사람이 일일이 입력하지 않아도 모든 공공 시스템의 데이터 흐름을 실시간으로 시각화하고, 부처 간 유사 사업을 AI가 찾아내어 통합을 제안하는 능동적인 거버넌스 체계가 될 전망이다.

📢 섹션 요약 비유: 범정부 EA는 대한민국 디지털 영토를 지키는 파수꾼이자, 모든 국민이 편리한 행정 서비스를 누릴 수 있게 돕는 보이지 않는 기반 시설(Infrastructure)입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. 우리나라의 모든 관공서(동사무소, 경찰서 등)가 어떤 일을 하고 어떤 컴퓨터를 쓰는지 정리해 놓은 "나라 전체의 설계도"예요.
2. 이 설계도가 있으면 각 기관이 똑같은 물건을 두 번 사서 돈을 낭비하는 것을 막을 수 있고, 서로 정보를 쉽게 주고받을 수 있어요.
3. 우리 나라를 하나의 커다란 스마트 로봇처럼 똑똑하게 움직이게 해주는 두뇌 지도라고 생각하면 돼요.
