# 데스크톱 애널리틱스 (Desktop Analytics) / 작업 마이닝 (Task Mining)

> **약어 (Abbreviation)**: Task Mining (Desktop Analytics)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 작업 마이닝 (Task Mining) 혹은 데스크톱 애널리틱스는 백엔드 서버의 로그를 분석하는 프로세스 마이닝과 달리, 직원의 **개인 PC(프론트엔드)** 백그라운드에 에이전트를 설치하여 **키스트로크(Keystroke), 마우스 클릭, 화면 스크롤, 텍스트 복사** 등의 미시적 사용자 상호작용(User Interaction) 데이터를 수집하고 분석하는 기술이다.
> 2. **가치**: 중앙 서버 로그에 남지 않는 '직원들의 엑셀 작업, 이메일 작성, 메신저 소통' 등 **그림자 노동(Shadow Work)**과 비정형 수작업을 가시화하여, RPA(Robotic Process Automation) 봇을 투입했을 때 가장 효율이 높은 자동화 타겟을 정밀하게 발굴해 낸다.
> 3. **융합**: 거시적인 흐름을 보여주는 **프로세스 마이닝 (Process Mining)**과 결합하여, 거시(Macro)부터 미시(Micro)까지 기업의 모든 움직임을 엑스레이처럼 투시하는 PTM(Process and Task Mining) 하이브리드 아키텍처로 진화하고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 작업 마이닝은 직원들이 화면에서 어떤 순서로 클릭하고 어떤 데이터를 복사해 붙여넣는지(Swivel-chair work)를 초 단위로 기록하여, 특정 업무를 수행하는 데 걸리는 실제 시간과 비효율적인 동선을 분석하는 현미경 같은 기술이다.
- **💡 비유**: 프로세스 마이닝이 헬리콥터를 타고 도시 전체의 자동차 흐름(서버 로그)을 보는 것이라면, 작업 마이닝은 운전석 옆자리에 카메라를 달고 운전자가 내비게이션을 몇 번 터치하고 브레이크를 언제 밟는지(클릭/타이핑)를 기록하는 것과 같습니다.
- **등장 배경**:
  - **기존 한계**: 프로세스 마이닝으로 "발주에서 결재까지 평균 3일이 걸린다"는 서버 병목은 찾아냈다. 하지만 그 3일 동안 담당자가 구체적으로 무슨 일을 하길래 지연되는지(예: 엑셀을 10개 열어서 VLOOKUP을 돌리는지)는 서버 로그(Event Log)에 남지 않아 원인을 규명할 수 없었다.
  - **혁신적 패러다임**: PC 데스크톱에 설치되는 가벼운 에이전트(Agent) 프로그램이 키보드와 마우스 이벤트, 화면의 OCR(문자 인식) 텍스트를 수집하고, AI 클러스터링 알고리즘이 "이 직원은 하루에 30번 똑같은 패턴으로 복사/붙여넣기를 한다"고 자동 군집화해 내는 기술이 개발되었다.
  - **현재의 비즈니스 요구**: 막대한 비용을 들여 RPA를 도입하려는데 "어디에 적용해야 본전을 뽑을지" 모르는 기업들에게, 가장 확실한 RPA 도입 후보지(Automation Pipeline)를 자동으로 추출해 주는 필수 사전 진단 도구로 수요가 폭증했다.

> 다음은 기존 프로세스 마이닝의 사각지대(Blind Spot)를 작업 마이닝이 어떻게 메워주는지 보여주는 도식이다.

```text
┌─────────────────────────────────────────────────────────────┐
│ [Process Mining vs Task Mining (Desktop Analytics)]         │
├──────────────────────────────┬──────────────────────────────┤
│ 1. Process Mining (서버 레벨)│ 2. Task Mining (PC 레벨)     │
│                              │                              │
│ [ ERP System (SAP) ]         │ [ User Desktop (Windows) ]   │
│  - 09:00 : 주문 접수         │  - 09:01: Outlook 이메일 열기│
│  - (이 사이 2시간은 블랙박스)│  - 09:05: 첨부 엑셀 다운로드 │
│  - 11:00 : 결제 완료         │  - 09:10: PDF 보면서 엑셀입력│
│                              │  - 10:50: SAP 화면에 복붙    │
│                              │                              │
│ * 거시적 흐름(Macro) 파악    │ * 미시적 수작업(Micro) 파악  │
└──────────────────────────────┴──────────────────────────────┘
```
- **해설**: 프로세스 마이닝 엔진의 눈에 직원의 PC에서 일어나는 2시간의 엑셀 작업은 보이지 않는 '블랙박스'다. 작업 마이닝은 이 블랙박스를 열어, 직원이 어떤 앱을 오가며(Context Switch) 시간을 낭비하는지 정확한 클릭 로그로 증명해 준다.

> **📢 섹션 요약 비유**: 프로세스 마이닝이 "주방에서 요리가 나오기까지 30분이 걸렸다"는 포스기(POS) 기록이라면, 작업 마이닝은 주방장 어깨에 고프로(카메라)를 달아 "도마와 가스레인지를 20번이나 왔다 갔다 해서 늦어졌다"는 동선을 분석하는 것입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 핵심 구성 요소 (Task Mining Architecture)

작업 마이닝 시스템은 크게 클라이언트의 수집기와 중앙의 분석 엔진으로 나뉜다.

| 요소명 | 역할 | 내부 형태 및 특징 | 비유 |
|:---|:---|:---|:---|
| **Recorder / Agent** | 데이터 수집 | 직원의 PC에 설치되어 키스트로크, 마우스 클릭, 활성화된 앱 이름, 화면 스크린샷 수집 | 블랙박스 카메라 |
| **OCR / Computer Vision**| 화면 문맥 파악 | 캡처된 화면의 픽셀을 분석하여 버튼 이름이나 텍스트를 데이터로 추출 | 영상 판독기 |
| **Clustering Engine** | 패턴 자동 분류 | 무작위로 수집된 수십만 번의 클릭 중, 반복되는 "의미 있는 행동 패턴"을 머신러닝으로 묶음 | 행동 심리 분석가 |
| **Automation Hub** | 대시보드 및 RPA 연계 | 발견된 수작업 패턴 중 "자동화 시 절감 예상 시간(ROI)"을 계산하여 리포팅 | RPA 수주 견적서 |

### 2. 데이터 수집 및 패턴 클러스터링 원리

> 아래 다이어그램은 직원들의 무작위 클릭 데이터가 어떻게 의미 있는 작업 맵(Task Map)으로 변환되는지 보여준다.

```text
[Task Mining Data Pipeline]

1. Data Collection (수집)
  직원 A: [Outlook 클릭] -> [Excel 클릭] -> [Ctrl+C] -> [SAP 클릭] -> [Ctrl+V]
  직원 B: [메신저 클릭] -> [Excel 클릭] -> [Ctrl+C] -> [SAP 클릭] -> [Ctrl+V]

2. Noise Filtering & OCR (노이즈 제거)
  "메신저 클릭(카톡)" 같은 개인적/무관한 행동은 머신러닝이 노이즈로 간주하여 필터링

3. Sequence Clustering (패턴 군집화)
  AI: "A와 B 모두 [Excel 복사 -> SAP 붙여넣기] 패턴을 하루 50번 반복하고 있군."
        │
        v
4. Task Model Generation (작업 모델 생성 및 RPA 추천)
  [ Task: 인보이스 이관 ]
   - 관련 앱: Excel, SAP
   - 1회 소요시간: 45초
   - 연간 절감 예상 비용: $30,000
   - [ 추천: UiPath Bot 생성 ]
```
- **해설**: 작업 마이닝의 핵심은 '노이즈 제거'와 '군집화(Clustering)'다. 사람들은 기계가 아니므로 같은 일을 하더라도 중간에 마우스를 엉뚱한 곳에 클릭하거나 메신저를 본다. 딥러닝 알고리즘(Sequence Mining)은 이런 변칙 속에서 일관된 핵심 줄기(Core Sequence)를 뽑아내어 시각적 도면으로 만들어준다.

### 3. PDD (Process Design Document) 자동 생성
가장 강력한 기능 중 하나는, 군집화된 패턴을 바탕으로 RPA 개발자에게 필요한 **설계 문서(PDD)를 봇이 자동으로 작성해 준다**는 점이다. "1. 엑셀을 연다 -> 2. C열을 복사한다 -> 3. ERP를 켠다"는 식의 스크린샷과 텍스트가 포함된 워드 문서가 클릭 한 번에 튀어나온다.

> **📢 섹션 요약 비유**: 수백 명의 요리사가 제각각 칼질하는 영상을 AI에 넣었더니, AI가 쓸데없는 동작(머리 긁기)은 빼고 가장 효율적인 '황금 레시피와 칼질 동영상 교본(PDD)'을 자동으로 만들어주는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 프로세스 마이닝 vs 작업 마이닝 (Macro vs Micro)

| 비교 항목 | Process Mining (프로세스 마이닝) | Task Mining (작업 마이닝) | 통합(PTM) 시너지 |
|:---|:---|:---|:---|
| **분석 레벨** | **Macro** (부서/시스템 간 흐름) | **Micro** (개인 PC 화면 내 조작) | End-to-End 가시성 확보 |
| **데이터 원천** | 서버 데이터베이스, 이벤트 로그 | 데스크톱 Agent, API 훅(Hook), 화면 | 데이터의 교차 검증 (Cross Check) |
| **주요 목적** | 비즈니스 병목 해결, 컴플라이언스 감사 | **수작업(Manual Work) 발견, RPA 타겟 선정** | 자동화 로드맵의 기획부터 실행까지 연결 |
| **개인정보 침해**| 낮음 (시스템 ID 위주) | **매우 높음** (화면 캡처, 키로깅 발생) | 철저한 화이트리스트 및 마스킹 필요 |

### 2. PTM (Process and Task Mining) 융합 아키텍처
최근 글로벌 선도 기업(Celonis, UiPath 등)은 두 기술을 따로 팔지 않고 **PTM 아키텍처**로 융합하여 제공한다. 서버 로그의 거시적 맥락과 데스크톱 클릭의 미시적 원인을 시간(Timestamp) 기준으로 동기화(Sync)하는 것이다.

> 다음은 PTM 융합 아키텍처를 통해 병목의 진짜 원인을 찾아내는 도식이다.

```text
[PTM (Process + Task Mining) Hybrid Analysis]

[ Macro View (Process Mining) ]
 (Order 접수) ─────(3시간 대기 발생!)─────> (Invoice 발행)
                           │
                           v (Drill-down)
[ Micro View (Task Mining) ]
                           └─> 직원 PC 분석 결과:
                               1. 시스템 오류로 ERP 창이 3번 꺼짐 (Crash 로그 탐지)
                               2. 직원이 엑셀에서 고객 주소를 10번이나 오타 내고 수정함
                               3. 카카오톡으로 주소를 다시 물어봄
```
- **해설**: 경영진이 프로세스 마이닝만 보면 "해당 부서가 게으르다"고 오판할 수 있다. 하지만 작업 마이닝을 겹쳐보면 "ERP 잦은 다운"과 "수작업 주소 입력"이라는 진짜 범인(Root Cause)이 드러난다. 이를 바탕으로 IT 부서는 ERP 메모리를 증설하고, 현업에는 RPA 봇을 투입하는 정확한 처방을 내릴 수 있다.

> **📢 섹션 요약 비유**: 프로세스 마이닝이 인공위성으로 막히는 고속도로(병목)를 찾는 것이라면, 작업 마이닝은 막힌 자동차의 블랙박스를 열어 "앞차가 졸음운전을 하고 있었다"는 진짜 이유를 밝혀내는 완벽한 수사 콤비입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 1. 실무 시나리오와 의사결정
- **상황**: 대기업 재무팀. 내년도 RPA 도입 예산으로 10억 원을 받았으나, 100명의 직원에게 "무엇을 로봇에게 맡기고 싶냐"고 물어보니 "저는 다 중요한 판단 업무라 로봇이 못합니다"라며 핑계를 대거나 제대로 된 아이디어를 내지 않음.
- **판단**: 직원의 인터뷰(주관적 기억과 방어 기제)에 의존하는 RPA 타겟 발굴(Process Discovery)을 즉각 중단한다. 대신 재무팀 100명의 PC에 **작업 마이닝(Task Mining) 에이전트**를 2주간 설치한다. 분석 결과, "직원들이 매일 아침 은행 포털 5곳에 접속해 환율을 복사해서 사내 엑셀에 붙여넣는 작업"에 전체 인력의 15% 시간이 낭비되고 있음을 데이터로 적발했다. 이 구간에 RPA를 즉시 투입하여 연간 1억 원의 비용을 절감한다.

### 2. 도입 시 핵심 체크리스트 (Technical & Operational)
1. **프라이버시 및 노조 협의 (Privacy & Compliance)**: 데스크톱 화면 캡처와 키로깅은 강력한 직원 감시(Big Brother) 논란을 부른다. 도입 전 노조와 합의했는가? 메신저, 개인 이메일, 은행 사이트 등 업무와 무관한 앱은 녹화되지 않도록 철저한 '블랙리스트/화이트리스트' 필터링 룰을 설정했는가?
2. **PC 리소스 오버헤드 (Performance Hit)**: 백그라운드에서 화면을 캡처하고 OCR을 돌리는 에이전트가 CPU와 RAM을 과도하게 점유하여 직원의 엑셀 작업을 느리게 만들면 심각한 클레임이 발생한다. 경량화(Lightweight) 에이전트인지 검증했는가?
3. **분석의 목적성 (Purpose-driven)**: 무작정 데이터를 모으는 것은 쓰레기 산을 만드는 것이다. "SAP와 엑셀을 오가는 데이터 이관 작업 중 자동화할 것을 찾겠다"와 같이 마이닝의 타겟 애플리케이션과 목표를 명확히 정의(Scoping)했는가?

### 3. 치명적 안티패턴 (Anti-Pattern)
- **개인 성과 평가 툴로의 악용 (Weaponization of Data)**: 작업 마이닝으로 수집된 클릭 데이터를 바탕으로 "김대리는 하루에 인터넷 기사를 2시간 본다"며 인사 고과를 깎는 무기로 사용하는 현상.

> 아래는 작업 마이닝의 목적이 왜곡되었을 때 발생하는 안티패턴 도식이다.

```text
[Anti-Pattern: Weaponized Task Mining]

(경영진의 지시) "누가 딴짓하는지 마이닝 데이터로 잡아내!"
       |
       v
(직원들의 반발) "우리를 감시하는 스파이웨어다!"
       |
       v (직원들의 창의적 회피)
 - 에이전트 프로세스 강제 종료
 - 마우스를 1분마다 자동으로 움직이는 매크로(Mouse Jiggler) 설치
       |
       v
(결과) 수집된 데이터는 100% 가짜 데이터가 되어 RPA 발굴 프로젝트 전면 무산.
```
- **해설**: 작업 마이닝의 데이터는 철저히 익명화(Anonymization)되어야 하며, 오직 '반복적인 프로세스를 찾아 기계에 넘기고 사람을 편하게 해주기 위한 목적'으로만 쓰인다는 강력한 리더십의 약속이 선행되어야 한다.

> **📢 섹션 요약 비유**: 수술용 메스(작업 마이닝)는 환자(직원)의 병든 곳(단순 노동)을 도려내어 살리기 위해 써야지, 환자를 위협하는 칼로 쓰면 환자는 병원(회사)을 도망쳐 버립니다.

### 1. 정량/정성적 기대효과

|:---|:---|:---|:---|

### 2. 미래 전망 및 아키텍처 진화
- **생성형 AI (GenAI) 융합 자동화**: 지금은 작업 마이닝이 '문서(PDD)'를 만들어 주면 개발자가 이를 보고 RPA 코드를 짠다. 미래에는 수집된 태스크 로그를 LLM(대규모 언어 모델)이 이해하고, 인간의 개입 없이 즉시 **실행 가능한 RPA 봇 코드를 100% 자동 생성(Auto-generation)**하는 진정한 의미의 'Zero-touch Automation'으로 진화하고 있다.
- **의도 파악 마이닝 (Intent Mining)**: 직원이 단순히 어떤 버튼을 눌렀는지(What)를 넘어, 마우스의 머뭇거림이나 스크롤 속도를 AI가 분석하여 "이 직원은 지금 이 양식을 어떻게 채울지 몰라 헷갈려하고 있다(Intent)"를 파악하고 화면에 팝업 가이드를 띄워주는 UX 최적화 도구로 확장되고 있다.

> **📢 섹션 요약 비유**: 작업 마이닝은 셜록 홈즈의 돋보기입니다. 겉보기에는 조용히 일하는 것 같은 사무실 풍경 속에 숨겨진 '단순 노동이라는 범인'의 지문과 발자국을 초정밀 단위로 찾아내어, 로봇 경찰(RPA)에게 완벽한 체포 영장(PDD)을 쥐여줍니다.

### 📌 관련 개념 맵 (Knowledge Graph)
- 프로세스 마이닝 (Process Mining) | 작업 마이닝과 쌍을 이루는 기술로, 서버의 백엔드 로그를 분석하여 부서 간, 시스템 간의 거시적(Macro) 업무 흐름을 진단하는 기술
- RPA (Robotic Process Automation) | 작업 마이닝이 발굴해 낸 "단순 반복 클릭 업무"를 실제로 대신 수행하기 위해 투입되는 UI 기반의 소프트웨어 로봇
- PDD (Process Design Document) | RPA 봇을 개발하기 전, 현재 업무가 어떤 순서로 화면에서 이루어지는지 화면 캡처와 함께 상세히 기록한 설계 문서 (작업 마이닝이 자동 생성함)
- 스위블 체어 워크 (Swivel-chair Work) | API가 없어 두 개의 시스템 창을 띄워놓고, 사람이 의자를 빙글빙글 돌려가며 수작업으로 데이터를 복사/붙여넣기 하는 비효율적 업무 형태
- OCR (Optical Character Recognition) | 작업 마이닝이 화면 스크린샷을 분석할 때 이미지 속의 글자(예: 영수증 금액, 송장 번호)를 텍스트 데이터로 변환해 주는 핵심 인지 기술
