# 5G SA & SBA: 클라우드 네이티브로 진화한 진정한 5G 코어망 아키텍처

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 5G SA(Standalone)는 기존 LTE 코어망(EPC)을 완전히 배제하고 오직 5G 전용 코어망(5GC)만으로 통신을 수행하는 구조이며, 이 5GC를 구성하는 뼈대가 바로 마이크로서비스 기반의 SBA(Service Based Architecture)이다.
> 2. **가치**: NSA(Non-Standalone) 방식이 갖던 코어망의 지연(Latency) 한계를 돌파하여 1ms 초저지연을 실현하고, 네트워크 슬라이싱(Network Slicing)과 엣지 컴퓨팅(MEC)을 완벽하게 지원하는 '진짜 5G'의 완성이다.
> 3. **융합**: 통신 장비(하드웨어) 중심이었던 과거의 틀을 깨고, IT 업계의 클라우드 네이티브(Cloud-Native), 컨테이너(Container), RESTful API 기술을 통신망에 융합하여 네트워크 기능을 소프트웨어적으로 무한 확장하게 만든 기술 혁신이다.

+++

## Ⅰ. 5G SA 및 SBA 개요

### 1. 정의
- **5G SA (Standalone, 단독 모드)**: 무선 기지국(gNB)부터 코어망(5GC)까지 데이터 전송 및 제어 신호 처리를 모두 5G 표준 규격만으로 처리하는 차세대 이동통신 아키텍처다.
- **SBA (Service Based Architecture)**: 5G 코어망(5GC) 내부의 각 통신 제어 기능(NF, Network Function)들을 독립적인 소프트웨어 서비스로 모듈화하고, 이들이 HTTP/2 기반의 API로 서로 통신하게 만든 구조다.

### 💡 비유: "증축 공사가 끝난 최첨단 스마트 주방"
- **5G NSA**는 **낡은 가스레인지(LTE 코어)를 그대로 두고 그릴(5G 무선망)만 새로 사서 요리하는 것**입니다. 고기는 빨리 굽지만 결국 가스레인지의 불 조절(제어) 한계 때문에 한계가 명확합니다.
- **5G SA와 SBA**는 **낡은 주방을 완전히 허물고 모든 요리사(NF)들이 스마트 패드(API)로 소통하는 최첨단 주방(SBA)을 새로 지은 것**입니다. 요리사들은 각자 재료 다듬기, 굽기 등 맡은 일(마이크로서비스)만 하며, 손님이 몰리면 언제든 요리사를 복제(스케일 아웃)해서 투입할 수 있습니다.

### 3. 등장 배경
- **NSA의 한계 노출**: 초기 5G망은 빠른 보급을 위해 무선망만 5G를 깔고 코어는 LTE를 쓰는 NSA(Option 3) 방식을 썼으나, 이로 인해 자율주행이 요구하는 초저지연(uRLLC) 달성에 실패함.
- **통신망의 IT 클라우드화 요구**: 새로운 서비스를 몇 주 만에 런칭하기 위해서는 크고 무거운 통신 전용 하드웨어(Appliance) 대신, AWS 같은 클라우드 환경에서 도커(Docker)로 통신망을 띄울 수 있는 유연한 구조가 절실해짐.

📢 **섹션 요약 비유**: 껍데기만 5G였던 과거(NSA)를 버리고, 심장(Core)과 혈관(API)까지 모두 5G의 피가 흐르도록 환골탈태하는 완벽한 진화입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 5G 코어망(5GC)의 주요 네트워크 기능 (NF)

| 네트워크 기능 (NF) | 상세 내용 | 역할 (LTE 대비) |
|:---:|:---|:---|
| **AMF** (Access & Mobility) | 단말의 접속 및 이동성(핸드오버) 관리 | LTE의 MME 역할 수행 |
| **SMF** (Session Management)| 데이터 세션(IP 할당 등) 관리 | LTE의 PGW-C/SGW-C 역할 |
| **UPF** (User Plane Function)| 실제 사용자 데이터가 지나가는 통로 (라우팅) | LTE의 PGW-U/SGW-U 역할 |
| **NRF** (NF Repository) | 모든 NF들이 자신을 등록하고 서로를 찾는 전화번호부 | **SBA의 핵심 (신규 도입)** |

### 2. NSA vs SA 구조 및 SBA 연결 아키텍처 (ASCII)

```text
┌─────────────────────────────────────────────────────────────┐
│                 [ NSA vs SA ARCHITECTURE ]                  │
├─────────────────────────────────────────────────────────────┤
│   (5G NSA - Option 3)              (5G SA - Option 2)       │
│                                                             │
│   [ LTE Core (EPC) ]               [ 5G Core (5GC) ]        │
│          ▲   ▲                             ▲                │
│ (Control)│   │ (Data)             (Control & Data)          │
│          │   │                             │                │
│    ┌─────┴───┴─────┐                 ┌─────┴───────┐        │
│    │ LTE Base(eNB) │───(Data)──▶     │ 5G Base(gNB)│        │
│    └──────┬────────┘                 └──────┬──────┘        │
│           │                                 │               │
│        [ 5G Phone ]                    [ 5G Phone ]         │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│         [ 5G SBA (Service Based Architecture) ]             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│         [ AMF ]      [ SMF ]      [ PCF ]      [ UDM ]      │
│            │            │            │            │         │
│  ◀─────────┴────────────┴────────────┴────────────┴────────▶│
│     Service Based Interface (SBI) - HTTP/2 & REST API       │
│                                           │                 │
│                                        [ NRF ] (Directory)  │
└─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 
상단의 **SA 아키텍처**를 보면, NSA에서는 통신 제어(Control)를 구형 LTE 기지국과 코어(EPC)가 전담하지만, SA에서는 오직 5G 기지국(gNB)과 5G 코어(5GC)만이 제어와 데이터를 모두 처리하여 병목이 제거된다.
하단의 **SBA 아키텍처**는 5G 코어망 내부의 모습이다. 과거에는 장비 간 통신 선이 1:1로 얽혀있는 복잡한 스파게티 구조(Point-to-Point)였으나, SBA는 **Service Based Interface (SBI)**라는 하나의 공용 버스(Bus)에 모든 모듈(NF)을 연결한다. 각 모듈은 웹 개발에서 쓰이는 HTTP/2 (REST API)로 대화하며, **NRF(저장소 기능)**에 서로를 등록하고 검색하여 동적으로 통신한다.

### 3. 심층 동작 원리: CUPS (Control and User Plane Separation)
- 5G SA의 핵심 철학은 **제어 평면(Control Plane)과 사용자 데이터 평면(User Plane)의 완벽한 분리(CUPS)**다.
- 길 안내(AMF, SMF)는 중앙 클라우드에서 통합 관리하고, 실제 엄청난 동영상 트래픽이 지나는 길(UPF)은 기지국 바로 옆(Edge)에 마음대로 배치할 수 있게 되어 초저지연을 달성한다.

📢 **섹션 요약 비유**: 사장님(Control Plane)은 서울 본사에 두고, 무거운 짐을 나르는 직원(User Plane)은 부산 현장에 마음대로 쪼개어 배치할 수 있게 된 완벽한 분업 구조입니다.

## Ⅲ. 융합 비교 및 기술 분석

### 1. 5G NSA vs 5G SA의 성능/서비스 비교

| 비교 항목 | 5G NSA (Non-Standalone) | 5G SA (Standalone) |
|:---:|:---|:---|
| **코어망** | LTE 코어망 (EPC) 재사용 | **순수 5G 코어망 (5GC)** |
| **제어 신호 지연** | 느림 (LTE망 의존) | **매우 빠름 (1ms 초저지연 지원)** |
| **배터리 소모** | 많음 (LTE와 5G 두 개 신호 동시 수신) | **적음 (5G 신호만 수신)** |
| **네트워크 슬라이싱**| 불가능 (또는 매우 제한적) | **완벽히 지원 (End-to-End)** |
| **도입 시기** | 5G 초기 상용화 (비용 절감 목적) | **현재 및 미래 지향 (B2B 필수)** |

### 2. 과목 융합 관점 (소프트웨어 공학 & 클라우드 네이티브)
- **MSA (Microservices Architecture)**: SBA는 철저하게 MSA 사상에 입각하여 설계되었다. 과거 거대한 모놀리식(Monolithic) 통신 장비가 수십 개의 작은 NF(컨테이너)로 쪼개져 서로 느슨하게 결합(Loosely Coupled)된다.
- **클라우드 네이티브 인프라**: SBA 구조 덕분에 통신사들은 코어망을 전용 하드웨어가 아닌 K8s(쿠버네티스) 환경에 배포하는 **CNF(Cloud Native Network Function)** 체계로 완벽히 전환하게 되었다.

📢 **섹션 요약 비유**: 무거운 '철제 로봇(LTE 코어)'을 버리고, 레고 블록처럼 언제든 조립하고 교체할 수 있는 '소프트웨어 요정(SBA 기반 5GC)'으로 코어망을 바꾼 것입니다.

+++

## Ⅳ. 실무 적용 및 기술사적 판단

### 1. 실무 시나리오: 자율주행 및 원격 수술을 위한 이음 5G(특화망) 구축
- **상황**: 대형 종합 병원에서 로봇을 이용한 초저지연 원격 수술 환경을 구축하고자 함. 기존 NSA망으로는 지연 시간 튀어 오름(Jitter) 현상 발생.
- **적용**: **5G SA 기반의 이음 5G(Private 5G) 코어망 자체 구축**.
  - 병원 지하 서버실에 소형 서버 1대 분량의 5GC(SBA 기반 컨테이너)를 K8s에 배포.
  - UPF(데이터 전송 처리기)를 수술실 바로 앞의 MEC 장비에 배치하여 트래픽의 외부망 유출 원천 차단.
- **결과**: 수술 로봇의 제어 지연 시간을 1ms 이하로 고정 확보 및 의료 데이터의 완벽한 보안 달성.

### 2. 기술사적 판단 (Insight)
- **마이그레이션 전략 (Migration Strategy)**: NSA에서 SA로 넘어가는 것은 단순히 서버를 교체하는 것이 아니라 통신사의 운영 철학이 IT(클라우드)로 완전히 바뀌는 거대한 도전이다. 기술사는 서비스 중단 없는 무중단 마이그레이션을 위해, 기존 장비와 SBA 장비가 공존하는 **'하이브리드 코어 아키텍처(Option 4/7 등)'** 설계 로드맵을 면밀히 수립해야 한다.
- **API 보안 위협**: 코어망 내부가 HTTP/2 기반의 개방형 API로 통신함에 따라, 과거 폐쇄망에서는 없었던 새로운 웹 기반 해킹(API 인젝션 등)에 노출될 수 있다. 실무에서는 **OAuth 2.0 기반의 인증** 및 **TLS 1.3 암호화**를 코어망 내부에 필수적으로 적용하는 '제로 트러스트' 설계가 동반되어야 한다.

📢 **섹션 요약 비유**: 과거에는 성문을 닫아걸면(폐쇄망) 끝이었지만, 이제는 성 안에 수많은 상점(API)이 열렸으므로 상점마다 철저한 신분증 검사(API 보안)를 해야 합니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성적 기대효과
- **B2B 혁신 가속화**: 진정한 네트워크 슬라이싱이 가능해져 스마트 팩토리, 자율주행 등 기업용(B2B) 5G 시장의 폭발적 성장 견인.
- **OPEX/CAPEX 혁신**: 코어망 구축과 유지보수를 자동화된 클라우드 시스템(CI/CD)으로 처리하여 통신사의 운영 비용 극단적 절감.

### 2. 미래 전망
- **AI-Native Core**: SBA 구조에 의해 생성되는 방대한 API 통신 로그를 AI가 실시간으로 분석하여, 네트워크 장애를 미리 예측하고 스스로 슬라이스 용량을 늘리는 '자율형 코어망'으로 진화할 것이다.
- **6G 아키텍처의 근간**: 5G SBA에서 확립된 '클라우드 네이티브'와 '서비스 기반'이라는 사상은 6G 코어망 아키텍처에서도 변함없는 핵심 철학으로 계승될 것이다.

📢 **섹션 요약 비유**: 5G SA와 SBA는 통신 산업이 무거운 철조망을 벗어 던지고 가벼운 날개(소프트웨어)를 달아, IT 클라우드라는 거대한 하늘로 날아오른 역사적인 도약입니다.

+++

### 📌 관련 개념 맵
- **네트워크 슬라이싱**: 5G SA 환경에서 꽃을 피우는 망 분할 기술.
- **MEC (모바일 엣지 컴퓨팅)**: 5G 코어망의 UPF가 전진 배치되는 물리적 위치.
- **마이크로서비스 아키텍처 (MSA)**: SBA를 탄생시킨 소프트웨어 공학적 설계 사상.
- **이음 5G (Private 5G)**: 5G SA 장비를 기업이 직접 구축하여 쓰는 사설망 기술.

### 👶 어린이를 위한 3줄 비유 설명
1. **가짜 5G vs 진짜 5G**: 예전 5G는 길만 넓고 신호등 아저씨는 옛날 4G 아저씨라서 길이 자주 막혔지만, 진짜 5G(SA)는 신호등 아저씨도 5G 천재로 바뀌어서 절대 막히지 않아요.
2. **똑똑한 요리사들 (SBA)**: 컴퓨터 안에서 일을 처리하는 요리사들이 서로 "이것 좀 해줘!"라고 스마트폰 문자(API)로 빠르고 정확하게 대화하는 멋진 주방이에요.
3. **손님 맞춤 서비스**: 요리사들이 너무 일을 잘해서, 영화를 보는 사람, 자율주행 차를 타는 사람 각각에게 가장 알맞은 서비스를 순식간에 만들어 줄 수 있답니다.
