# DID Document: 신원 증명의 기술적 실체와 검증 규격

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DID Document는 특정 분산 식별자(DID)와 관련된 **공개키 정보, 인증 방법, 서비스 엔드포인트** 등을 담고 있는 JSON-LD 형식의 기술 명세서다.
> 2. **가치**: 블록체인 상에 저장되어 사용자가 자신의 신원을 증명하려 할 때, 검증자가 해당 사용자의 **진짜 공개키**를 찾아낼 수 있게 돕는 '신뢰할 수 있는 데이터 원천' 역할을 한다.
> 3. **융합**: W3C(World Wide Web Consortium) 표준을 따르며, 다양한 블록체인 네트워크(Methods) 간의 상호운용성을 보장하는 탈중앙화 신원 생태계의 공통 언어다.

+++

## Ⅰ. DID Document 개요

### 1. 정의
- 특정 DID 식별자에 맵핑되어 있는 데이터 객체로, 해당 DID의 소유권 확인 및 상호작용에 필요한 모든 메타데이터를 포함한다.
- 블록체인에는 개인정보 원본 대신 이 **기술 명세서**만이 등록되어 보안을 유지한다.

### 2. 💡 비유: "공개 게시판에 붙은 명함 제작 가이드"
- DID Document는 마을 게시판(블록체인)에 붙어있는 **'나와 소통하는 법이 적힌 안내문'**과 같습니다. 안내문에는 내 이름이나 주소는 없지만, "나를 확인하려면 이 도장(공개키)을 쓰세요"라거나 "나에게 연락하려면 이 우체통(서비스 엔드포인트)으로 오세요"라는 정보가 적혀 있습니다. 누구든 게시판의 안내문만 보면 나를 사칭하는 사람을 가려낼 수 있습니다.

### 3. 등장 배경
- **중앙화된 인증서의 한계**: 공인인증기관(CA)이 발행한 인증서는 해당 기관이 마비되면 검증이 불가능함.
- **표준화된 검증 체계 필요**: 서로 다른 블록체인을 쓰더라도 동일한 방식으로 신원을 검증할 수 있는 공통 규격 요구.

📢 **섹션 요약 비유**: 특정 회사의 전용 양식이 아닌, 전 세계 어디서나 통용되는 '표준 규격의 신분 확인 설명서'입니다.

## Ⅱ. 아키텍처 및 핵심 구성 요소 (Deep Dive)

### 1. DID Document의 주요 필드 구조

| 필드명 | 역할 | 설명 | 비유 |
|:---|:---|:---|:---|
| **@context** | 데이터 규격 정의 | JSON-LD 구문 해석을 위한 스키마 정의. | 언어 사전 |
| **id** | DID 식별자 | 해당 문서가 설명하는 고유 DID (`did:method:id`). | 이름표 |
| **verificationMethod** | 공개키 정보 | 서명 검증에 사용할 공개키와 알고리즘 타입. | 진짜 도장 견본 |
| **authentication** | 인증 권한 | 이 DID의 주인임을 증명할 때 사용하는 방법. | 본인 확인 절차 |
| **service** | 상호작용 지점 | 신원 관련 통신을 위한 서버 주소(Endpoint). | 연락처/우체통 |

### 2. DID Document 데이터 구조 다이어그램 (ASCII)

```text
┌─────────────────────────────────────────────────────────────┐
│                   [ DID DOCUMENT (JSON-LD) ]                │
├─────────────────────────────────────────────────────────────┤
│ {                                                           │
│   "@context": "https://www.w3.org/ns/did/v1",               │
│   "id": "did:example:123456789abcdefghi",                   │
│                                                             │
│   "verificationMethod": [{                                  │
│     "id": "did:example:123456789...#key-1",                 │
│     "type": "Ed25519VerificationKey2018",                   │
│     "publicKeyMultibase": "zH3C2AVv... (Public Key)"        │
│   }],                                                       │
│                                                             │
│   "authentication": [                                       │
│     "did:example:123456789...#key-1"                        │
│   ],                                                        │
│                                                             │
│   "service": [{                                             │
│     "id": "did:example:123456789...#vcs",                   │
│     "type": "VerifiableCredentialService",                  │
│     "serviceEndpoint": "https://example.com/vcs"            │
│   }]                                                        │
│ }                                                           │
└───────────────────────────┬─────────────────────────────────┘
                            │
            ┌───────────────┴───────────────┐
    [ Self-Describing ]             [ Cryptographic Proof ]
   (스스로를 설명하는 데이터)      (암호학적 서명의 기초)
```

**[다이어그램 해설]** 
DID Document는 기계가 읽을 수 있는(Machine-readable) 형태다. 검증자는 블록체인에서 이 문서를 가져온 뒤, `verificationMethod`에 적힌 **공개키**를 추출한다. 그리고 사용자가 제출한 VP(Verifiable Presentation)의 서명을 이 공개키로 대조해본다. 만약 일치한다면, 이 신원은 변조되지 않았음을 수학적으로 확신할 수 있다. `service` 항목은 신원 소유자와 실제 데이터를 주고받기 위한 통로(엔드포인트)를 안내한다.

### 3. 심층 동작 원리: DID Resolution (해소)
- 사용자가 "내 DID는 `did:sov:abc`야"라고 말하면, 시스템은 **DID Resolver**를 통해 블록체인 네트워크를 뒤져서 해당 식별자에 연결된 **DID Document**를 찾아낸다. 이 과정을 '해소(Resolution)'라고 하며, 도메인 이름을 IP 주소로 바꾸는 DNS 서비스와 유사한 원리다.

📢 **섹션 요약 비유**: 전화번호(DID)를 보고 전화번호부(블록체인)를 찾아가서, 그 사람의 집 주소와 신분 확인용 지문(Document)을 확인하는 일련의 과정입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 1. 전통적 PKI 인증서 vs DID Document 비교

| 비교 항목 | 전통적 인증서 (X.509 등) | DID Document |
|:---:|:---|:---|
| **발행 주체** | 중앙 집중형 CA (Root CA) | **사용자 본인 (Self-issued)** |
| **저장 위치** | 기업 서버 및 사용자 기기 | **분산 원장 (공개 저장소)** |
| **유효 기간** | 정해진 기간 후 갱신 필수 | **영구적 (필요시 업데이트 가능)** |
| **폐기 확인** | CRL, OCSP 등 별도 조회 필요 | **블록체인 상태값으로 즉시 확인** |
| **신뢰 모델** | 계층적 신뢰 (Chain of Trust) | **수평적 합의 (Consensus)** |

### 2. 과목 융합 관점 (웹 표준 & 데이터베이스)
- **웹 표준 (JSON-LD)**: 단순 JSON이 아닌 의미론적 웹(Semantic Web) 기술인 JSON-LD를 사용하여, 데이터 필드의 의미를 전 세계 소프트웨어가 동일하게 해석할 수 있도록 한다.
- **데이터베이스 (CRUD)**: DID Document는 블록체인 상에서 CRUD 연산 중 **Create(등록), Read(조회), Update(변경)**를 지원한다. 단, 블록체인 특성상 삭제(Delete) 대신 '폐기 상태로의 변경'을 수행한다.

📢 **섹션 요약 비유**: 중앙 관공서에서 도장을 찍어주는 '종이 신분증'과, 내가 직접 관리하지만 누구나 진짜임을 확인할 수 있는 '디지털 신원 설명서'의 차이입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 1. 실무 시나리오: 다중 기기 신원 동기화
- **상황**: 유저가 스마트폰을 잃어버리고 새로 샀을 때, 기존의 DID 신원을 그대로 유지하고 싶다.
- **적용**: **DID Document 업데이트**. 유저는 새로운 기기에서 새 공개키 쌍을 생성한다. 그리고 기존에 설정해둔 '복구 관리자'나 '백업 키'를 이용해 블록체인에 등록된 자신의 **DID Document** 내의 `verificationMethod`를 새 공개키로 업데이트한다. 이제 전 세계 모든 서비스는 유저의 새로운 기기 서명을 정당한 것으로 인식하게 된다.

### 2. 기술사적 판단 (Insight)
- **개인정보 노출 주의**: DID Document는 블록체인에 공개되므로 **PII(개인식별정보)**가 단 1비트라도 포함되어서는 안 된다. 실무에서는 이름, 전화번호 대신 오직 암호학적 파라미터만 넣어야 함을 강조해야 한다.
- **Controller의 권한**: DID Document를 수정할 수 있는 권한인 `controller` 필드를 정밀하게 설계해야 한다. 1인 관리뿐만 아니라 멀티시그(Multi-sig) 형태의 공동 관리를 지원하여 보안성을 높일 수 있다.

📢 **섹션 요약 비유**: 신분증 카드 자체를 바꾸는 게 아니라, 마을 벽보(블록체인)에 적힌 "제 진짜 서명 도장을 이걸로 바꿨습니다"라는 공고문만 수정하여 신뢰를 이어가는 지혜로운 방식입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성적 기대효과
- **글로벌 호환성**: 특정 기업의 솔루션에 갇히지 않고 전 세계 모든 DID 호환 서비스와 연동 가능.
- **신원 관리 효율화**: 인증서 갱신이나 폐기 확인을 위한 시스템 부하를 획기적으로 줄임.

### 2. 미래 전망
- **범용 식별자의 표준**: 사람뿐만 아니라 기기, 기업, 디지털 콘텐츠 등 모든 대상의 'ID 명세서'로서 DID Document가 사용될 것이다.
- **AI-Agent 신원 관리**: 사람이 없는 자율 주행차나 AI 에이전트가 스스로 DID Document를 관리하며 계약을 체결하는 '자율 신원' 시대의 핵심 데이터가 될 전망이다.

📢 **섹션 요약 비유**: DID Document는 디지털 영토를 항해하는 모든 존재들에게 부여된 **'표준 항해 일지'**이자, 서로를 신뢰하게 만드는 **'디지털 약속의 증표'**입니다.

### 📌 관련 개념 맵
- **DID (탈중앙 신원증명)**: DID Document를 사용하는 전체 시스템.
- **W3C DID 스펙**: DID Document의 표준을 정의하는 문서.
- **공개키 기반 구조 (PKI)**: DID Document 보안의 수학적 기초.
- **DID Resolver**: 식별자로 Document를 찾아주는 도구.

### 👶 어린이를 위한 3줄 비유 설명
1. **마법의 자기소개서**: 내 이름은 비밀로 하고, 대신 "나를 믿으려면 이 도장 모양을 확인해!"라고 적어둔 특별한 자기소개서예요.
2. **블록체인 벽보**: 이 자기소개서를 전 세계 친구들이 다 볼 수 있는 튼튼한 벽보(블록체인)에 붙여두는 거예요.
3. **가짜 도장 방지**: 나쁜 친구가 내 행세를 하려고 해도, 벽보에 적힌 도장 모양이랑 다르면 금방 가짜라는 걸 들키게 된답니다.
