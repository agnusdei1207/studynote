# DID 아키텍처 (DID Architecture): 신뢰의 삼각형과 데이터 주권의 기술적 구현

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DID 아키텍처는 신원 정보를 발급하는 **발행자(Issuer)**, 보관하는 **소유자(Holder)**, 확인하는 **검증자(Verifier)**가 블록체인(Registry)을 매개로 형성하는 **'신뢰의 삼각형(Trust Triangle)'** 구조다.
> 2. **가치**: 실제 신원 데이터(VC)는 블록체인이 아닌 사용자의 단말기에 보관하고, 블록체인에는 데이터의 진위와 상태(공개키, 폐기 여부)만 기록하여 **개인정보의 오남용을 원천 차단**한다.
> 3. **융합**: W3C 표준인 DID Document와 암호학적 서명이 결합된 **VC(검증 가능한 자격증명)**, **VP(검증 가능한 프레젠테이션)** 기술을 통해 디지털 환경에서의 '선택적 노출'을 완벽히 구현한다.

+++

## Ⅰ. DID 아키텍처 (DID Architecture) 개요

### 1. 정의
- 탈중앙화 신원증명을 실현하기 위해 필요한 구성 요소(주체, 데이터 표준, 저장소)들과 이들 간의 상호작용 방식을 정의한 시스템 프레임워크다.
- 데이터의 **원본성**과 **무결성**, 그리고 **소유권**을 기술적으로 보장하기 위한 분산 아키텍처다.

### 2. 💡 비유: "스마트폰 속의 암호화된 자격증 꾸러미"
- DID 아키텍처는 **암호화된 주머니(지갑)**와 **마법 도장(블록체인)**이 결합된 시스템과 같습니다. 내가 학교에서 받은 졸업장(VC)을 주머니에 넣고 다니다가, 회사에 취직할 때 졸업장 전체가 아닌 "이 사람은 졸업생이 맞다"는 확인 도장(VP)만 찍어서 보여주는 방식입니다. 회사는 주머니 주인(Holder)이 맞는지, 학교 도장(Issuer 공개키)이 진짜인지만 벽보(블록체인)와 대조해보고 믿어줍니다.

### 3. 주요 설계 목표
- **개인정보 최소화**: 서비스 제공자에게 꼭 필요한 정보만 제공 (Selective Disclosure).
- **데이터 주권**: 사용자가 자신의 데이터 전송 및 폐기 권한을 직접 행사.
- **상호운용성**: 서로 다른 기관에서 발급한 증명서들을 하나의 지갑에서 통합 관리.

📢 **섹션 요약 비유**: 중앙 서버라는 '거대한 창고'에 모든 사람의 정보를 쌓아두는 방식에서, 각자가 자신의 정보를 담은 '보물 상자'를 들고 다니며 필요할 때만 열어 보여주는 분산형 신뢰 체계입니다.

## Ⅱ. 핵심 구성 요소 및 작동 원리 (Deep Dive)

### 1. DID 아키텍처의 4대 핵심 데이터 모델

| 요소명 | 역할 | 주요 내용 | 비유 |
|:---|:---|:---|:---|
| **DID 식별자** | 신원의 고유 주소 | `did:example:1234abcd` 형태의 URI | 디지털 이름표 |
| **DID Document** | 신원 검증 정보 | 공개키(Public Key), 서비스 엔드포인트, 서명 정보 | 신분 확인 설명서 |
| **VC (Credential)** | 신원 증명 원본 | 성명, 학위, 면허 등 발행자가 서명한 데이터 | 주민등록증 카드 |
| **VP (Presentation)** | 제출용 데이터 세트 | 여러 VC 중 일부 정보만 추출하여 사용자가 서명한 데이터 | 제출용 사본 (부분 가림) |

### 2. DID 신뢰의 삼각형(Trust Triangle) 작동 구조 (ASCII)

```text
┌─────────────────────────────────────────────────────────────┐
│                   [ DID ARCHITECTURE FLOW ]                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│      (1) VC 발급 (Issue)       ┌───────────────┐            │
│    ┌──────────────────────────▶│    HOLDER     │            │
│    │                           │ (User Wallet) │            │
│    │                           └───────┬───────┘            │
│    │                                   │ (2) VP 제출        │
│    │                                   │    (Present)       │
│┌───────────────┐                       ▼             ┌───────────────┐
││    ISSUER     │               ┌───────────────┐     │   VERIFIER    │
││(Trust Anchor) │               │    VERIFIER   │     │ (Service Pro) │
│└───────┬───────┘               └───────┬───────┘     └───────┬───────┘
│        │                               │                     │
│        ▼ (3) DID 등록 (Registry)       ▼ (4) 유효성 검증     ▼
│  ┌───────────────────────────────────────────────────────────┐
│  │               [ BLOCKCHAIN / DATA REGISTRY ]              │
│  │  - DID Documents (Public Keys, Auth Info)                 │
│  │  - Revocation Status (폐기 목록)                          │
│  └───────────────────────────────────────────────────────────┘
│                                                             │
└───────────────────────────┬─────────────────────────────────┘
                            │
            ┌───────────────┴───────────────┐
    [ Privacy by Design ]           [ Decentralized Trust ]
   (원본은 지갑에만 존재)          (검증은 블록체인으로)
```

**[다이어그램 해설]** 
1. **발행(Issue)**: 발행자(Issuer)는 사용자의 정보를 확인하고 암호학적으로 서명된 **VC**를 사용자 지갑에 전송한다. 동시에 이 VC를 검증할 수 있는 자신의 공개키 정보를 블록체인(Registry)에 등록한다.
2. **제출(Present)**: 사용자는 Verifier가 요구하는 정보만 VC에서 골라내어 **VP**를 만들고, 자신의 개인키로 서명하여 전달한다.
3. **검증(Verify)**: Verifier는 블록체인에서 Issuer와 Holder의 공개키를 조회하여, 제출된 VP의 서명이 진짜인지, 폐기된 정보는 아닌지 실시간으로 확인한다. 이 모든 과정에서 Issuer와 Verifier는 서로 통신할 필요가 없다.

### 3. 심층 동작 원리: 선택적 노출 (Selective Disclosure)
- 사용자가 "20세 이상임"을 증명해야 할 때, 주민등록번호 전체가 담긴 VC를 다 주는 대신, "성인 여부: 참"이라는 값만 추출하여 VP를 생성한다. 이를 통해 프라이버시를 완벽히 보호하면서도 신뢰할 수 있는 인증을 수행한다.

📢 **섹션 요약 비유**: 면접장에서 이력서(VC)를 통째로 넘기는 게 아니라, 면접관이 질문하는 내용에 대해서만 내 도장이 찍힌 답변지(VP)를 건네주는 '철저한 정보 제어' 방식입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 1. DID vs 기존 ID 관리 시스템 (Silo/Federated)

| 비교 항목 | 기존 시스템 (Siloed/Centralized) | 연합형 ID (Federated - OIDC) | **탈중앙화 ID (DID)** |
|:---:|:---|:---|:---|
| **데이터 주권** | 서비스 업체가 독점 | 플랫폼 업체(구글 등)가 보유 | **사용자 본인이 소유** |
| **로그인 경험** | 사이트마다 ID/PW 생성 | 소셜 로그인 (편리함) | **지갑 하나로 모든 곳 인증** |
| **보안성** | 서버 해킹 시 대량 유출 | 플랫폼 계정 탈취 시 연쇄 피해 | **개인키 보안 (유출 위험 분산)** |
| **프라이버시** | 무분별한 데이터 수집 가능 | 활동 내역 추적 가능 | **필요 정보만 선택적 노출** |
| **가용성** | 서버 다운 시 인증 불가 | 플랫폼 장애 시 인증 불가 | **블록체인 기반 상시 인증** |

### 2. 과목 융합 관점 (정보보호 & 법규)
- **정보보호 (PKI)**: DID는 전통적인 공개키 기반 구조(PKI)를 블록체인으로 확장한 것이다. 인증 기관(CA)이라는 단일 신뢰 지점을 없애고, 분산 원장을 통해 신뢰를 분산(DPKI)한 것이 핵심이다.
- **법규 (GDPR)**: 유럽 개인정보 보호법(GDPR)의 **'잊힐 권리'**를 준수하기 위해, 블록체인에는 개인을 식별할 수 있는 정보를 절대 넣지 않고 오직 암호학적 지문(해시)과 상태값만 기록하는 설계가 실무에서 매우 중요하다.

📢 **섹션 요약 비유**: 모든 열쇠가 호텔 관리실(중앙)에 있는 시스템에서, 손님마다 자기만 아는 '디지털 지문 인식기'를 들고 다니는 시스템으로 보안의 패러다임이 바뀐 것입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 1. 실무 시나리오: 모바일 자격증 및 온/오프라인 통합 출입 시스템
- **상황**: 대기업 사원이 사원증 하나로 회사 전산망 로그인과 오프라인 사무실 출입을 동시에 해결하고 싶다.
- **적용**: **DID 아키텍처** 도입. 회사는 사원증 VC를 발행하고, 사원은 이를 폰에 저장한다. 사무실 입구(Verifier)에서 폰을 태그하면 블록체인 조회를 통해 0.5초 내에 신원이 검증되어 문이 열린다. 퇴사 시 회사가 블록체인 상의 VC 상태를 '폐기'로 바꾸면, 사원의 폰에 데이터가 남아있어도 즉시 권한이 상실된다.

### 2. 기술사적 판단 (Insight)
- **데이터 최소화의 실현**: 실무 설계 시 Verifier가 필요 이상의 데이터를 요구하지 않도록 **'Minimal Data Disclosure'** 정책을 스마트 컨트랙트와 연동하여 자동화해야 한다.
- **Resolver의 보안**: 블록체인에서 DID Document를 조회해오는 **DID Resolver**가 해킹당하면 잘못된 공개키 정보를 가져올 수 있다. Resolver의 무결성을 보장하기 위해 신뢰할 수 있는 노드 운영이 필수적이다.

📢 **섹션 요약 비유**: 출입증 카드를 잃어버렸을 때 카드를 뺏으러 다니는 게 아니라, 건물 입구의 센서(블록체인)에 "저 카드는 이제 가짜다"라고 한 줄만 적어두면 끝나는 '중앙 제어형 분산 시스템'입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성적 기대효과
- **보안 사고 비용 절감**: 개인정보 서버를 운영하지 않아도 되므로 보안 인프라 비용 및 해킹 배상 리스크 90% 이상 절감.
- **사용자 편의성 극대화**: 수많은 아이디와 비밀번호를 외울 필요 없이 지갑 앱 하나로 전 세계 웹 서비스를 안전하게 이용.

### 2. 미래 전망
- **메타버스 신원 표준**: 가상 세계에서 나의 아바타가 나임을 증명하는 유일한 수단으로 DID가 채택될 것이다.
- **범부처/글로벌 상호인정**: 국가 간 DID 표준이 통합되어, 한국에서 발급받은 DID 면허증으로 해외에서 렌터카를 빌리는 '글로벌 신뢰망'이 구축될 것이다.

📢 **섹션 요약 비유**: DID 아키텍처는 디지털 세상의 모든 문을 여는 **'단 하나의 마스터 키'**이며, 그 키의 주인은 오직 나 자신이라는 점이 가장 위대한 혁신입니다.

### 📌 관련 개념 맵
- **DID (탈중앙 신원증명)**: 아키텍처의 상위 개념.
- **VC / VP**: DID 시스템에서 오가는 핵심 데이터 꾸러미.
- **DID Document**: 블록체인에 등록되는 신원 기술 명세서.
- **자기 주권 신원 (SSI)**: DID 아키텍처가 지향하는 최종 목적지.

### 👶 어린이를 위한 3줄 비유 설명
1. **나만의 마법 주머니**: 학교나 시청에서 받은 소중한 카드(증명서)들을 내 스마트폰 마법 주머니에 쏙 넣어두는 거예요.
2. **보여주고 싶은 것만**: "이름이 뭐야?"라고 물어보면 이름만 보여주고, "몇 살이야?"라고 물어보면 나이만 보여줄 수 있는 아주 똑똑한 주머니예요.
3. **가짜는 금방 들통나**: 주머니에서 꺼낸 카드가 진짜인지 가짜인지는 블록체인 요정이 항상 지켜보고 있어서, 나쁜 친구가 속이려 해도 절대로 안 속는답니다.
