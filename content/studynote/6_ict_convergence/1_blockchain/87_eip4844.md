# EIP-4844: 롤업의 가스비를 1/10로 낮추는 블롭(Blob) 데이터 혁명

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: EIP-4844(Proto-Danksharding)는 이더리움 메인넷에 **'블롭(Blob)'**이라는 대용량 데이터 전용 저장 공간을 신설하여, 레이어 2(L2) 롤업들이 훨씬 저렴한 비용으로 데이터를 게시할 수 있게 돕는 업그레이드다.
> 2. **가치**: 기존의 비싼 `Calldata` 대신 일정 기간(약 18일) 후 자동으로 삭제되는 임시 저장소를 활용함으로써, L2 사용자들의 **트랜잭션 수수료를 획기적으로 절감**하고 이더리움의 확장성을 비약적으로 향상시킨다.
> 3. **융합**: 고도의 암호학적 약정 기술인 **KZG 다항식 약정(KZG Commitment)**을 도입하여 데이터 전체를 읽지 않고도 유효성을 검증하며, 장기적으로 '풀 댕크샤딩(Full Danksharding)'으로 나아가는 핵심 징검다리 역할을 한다.

+++

## Ⅰ. EIP-4844 (Proto-Danksharding) 개요

### 1. 정의
- 이더리움의 확장성 로드맵인 'The Surge'의 첫 번째 단계로, 블록체인에 **'Blob-carrying transactions'**라는 새로운 거래 유형을 도입하는 개선 제안이다.
- 연산에 참여하지 않는 '순수 데이터 뭉치'를 위한 저렴한 차선을 도로 옆에 새로 까는 것과 같다.

### 2. 💡 비유: "고속도로 옆의 화물 전용 컨테이너 벨트"
- 기존 방식은 모든 승객(트랜잭션)과 화물(L2 데이터)이 똑같은 비싼 통행료를 내고 1차선 도로(Calldata)를 이용하는 것과 같았습니다.
- **EIP-4844**는 고속도로 옆에 **'임시 화물 전용 벨트(Blob)'**를 설치한 것입니다. 화물들은 이제 비싼 도로를 점유하지 않고 저렴한 전용 벨트를 타고 이동하며, 배달이 완료된 박스(데이터)는 일정 시간이 지나면 자동으로 폐기되어 도로가 막히는 것을 방지합니다.

### 3. 등장 배경
- **L2 수수료의 한계**: 롤업 수수료의 90% 이상이 메인넷에 데이터를 기록하는 비용(Calldata 비용)에서 발생하여, 진정한 대중화에 걸림돌이 됨.
- **데이터 가용성(DA) 문제**: 모든 데이터를 영구 보관할 필요가 없는 롤업 특성을 반영하여, 효율적인 데이터 관리 체계 필요성 대두.

📢 **섹션 요약 비유**: 비싼 양장본 책(메인넷)에 모든 낙서를 다 남기는 대신, 포스트잇(블롭)에 메모를 해서 붙여두었다가 나중에 떼어버리는 식의 '알뜰한 기록법'입니다.

## Ⅱ. 아키텍처 및 핵심 기술 원리 (Deep Dive)

### 1. Calldata vs Blob 비교 분석

| 구분 | Calldata (기존) | Blob (EIP-4844) |
|:---|:---|:---|
| **저장 위치** | 이더리움 블록 내부 (영구) | **비콘 노드 사이드카 (임시, 약 18일)** |
| **EVM 접근** | 실행 가능 (연산 가능) | **접근 불가 (단순 게시/검증용)** |
| **가스 가격** | 일반 가스비와 연동 (비쌈) | **독립된 블록체인 시장 형성 (매우 저렴)** |
| **데이터 크기** | 가변적, 제한적 | **블록당 최대 6개(각 128KB)** |

### 2. EIP-4844 작동 구조 다이어그램 (ASCII)

```text
┌─────────────────────────────────────────────────────────────┐
│                [ EIP-4844 TRANSACTION STRUCTURE ]           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [ L2 Rollup Sequencer ]                                    │
│             │                                               │
│             ▼ (Send Blob Transaction)                       │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ [ Standard TX Data ] | [ KZG Commitment (Pointer) ]   │  │ (Main Block)
│  └───────────────────────────┬───────────────────────────┘  │
│                             │                               │
│                             │      [ BLOB SIDECAR ]         │
│                             └────▶ ┌─────────────────────┐  │ (Separate
│                                    │  Binary Raw Data    │  │  Storage)
│                                    │  (128KB x N)        │  │
│                                    └─────────────────────┘  │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  [ Verification ]                                           │
│  - EVM checks only the 'KZG Commitment'                     │
│  - Proof that Blob exists and matches the Commitment        │
└───────────────────────────┬─────────────────────────────────┘
                            │
            ┌───────────────┴───────────────┐
    [ Data Decoupling ]             [ Auto Pruning ]
   (연산과 데이터의 분리)          (18일 후 자동 삭제)
```

**[다이어그램 해설]** 
EIP-4844의 핵심은 **디커플링(Decoupling)**이다. 롤업의 실제 대용량 데이터는 블록체인 본체에 담기지 않고, 옆에 붙어 다니는 **'사이드카(Sidecar)'** 형태의 **블롭(Blob)**에 저장된다. 메인넷의 가상머신(EVM)은 블롭의 내용 전체를 읽지 않는다. 대신 **KZG Commitment**라는 짧은 증명값만 보고 "이 데이터가 진짜구나"라고 확인만 하고 넘어간다. 이 블롭 데이터는 노드들에게 저장 부담을 주지 않기 위해 18일 정도가 지나면 네트워크에서 자동으로 사라진다.

### 3. 심층 동작 원리: KZG 다항식 약정 (KZG Commitment)
- 데이터를 다항식의 점들로 표현하고, 특정 지점의 값을 증명하는 암호학적 기법이다. 이를 통해 수만 개의 거래 데이터(블롭)를 단 하나의 짧은 해시값으로 요약하면서도, 나중에 데이터의 일부를 검증할 때 매우 빠르게 처리할 수 있다.

📢 **섹션 요약 비유**: 거대한 짐 가방(블롭) 전체를 세관(EVM)에 들고 가는 게 아니라, 가방 안의 물건 목록이 정확하다는 '암호화된 바코드(KZG)'만 보여주고 통과하는 정교한 물류 검문 시스템입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 1. 롤업 경제학의 변화 (Before vs After)

| 비교 항목 | EIP-4844 도입 전 | EIP-4844 도입 후 |
|:---:|:---|:---|
| **L2 거래 수수료** | $0.5 ~ $2.0 | **$0.01 ~ $0.1 (1/10~1/100 절감)** |
| **L1 저장 수단** | Calldata (값비싼 연산 공간) | **Blob (저렴한 전용 데이터 공간)** |
| **네트워크 부하** | 노드의 영구 저장 용량 급증 | **주기적 삭제로 저장 부담 완화** |
| **확장성 성격** | 단일 병목 구조 | **데이터 가용성 위주의 다층 구조** |

### 2. 과목 융합 관점 (운영체제 & 알고리즘)
- **운영체제 (Caching)**: 블롭은 일종의 '네트워크 캐시' 역할을 한다. 영구 저장이 아닌 '단기 고속 버퍼'로서 기능을 수행하며, 운영체제의 스왑(Swap) 메모리 기법을 네트워크 레벨에서 구현한 것과 유사하다.
- **알고리즘 (Polynomials)**: 영지식 증명과 유사한 다항식 연산 알고리즘이 적용되어, 데이터 전체를 공개하지 않고도 유효성을 증명하는 '데이터 최소화 원칙'을 기술적으로 완성한다.

📢 **섹션 요약 비유**: 모든 쓰레기(데이터)를 집 안에 영원히 쌓아두던 방식에서, 정해진 날짜에 내놓으면 수거해가는 '분리수거 시스템'으로 전환하여 집(블록체인)을 쾌적하게 만든 것입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 1. 실무 시나리오: L2 기반 소액 결제 플랫폼 전략
- **상황**: 커피 한 잔을 결제하는데 수수료가 500원이 나와서 사업이 안 된다.
- **적용**: **EIP-4844 활성화 및 블롭 활용**. 
  - ①아비트럼(Arbitrum)이나 옵티미즘(Optimism) 같은 L2가 메인넷 업그레이드에 맞춰 블롭 트랜잭션을 사용하도록 업데이트한다. 
  - ②결과적으로 유저의 수수료는 5원~10원 수준으로 떨어진다. 
  - ③기업은 절감된 수수료 비용을 마케팅 포인트로 활용하거나 가스비를 전액 대납(Paymaster)해주는 '완전 무료 결제' 경험을 제공한다.

### 2. 기술사적 판단 (Insight)
- **데이터 보존의 책임**: 메인넷에서 18일 후 데이터가 지워지므로, 과거 데이터를 영구 보존해야 하는 주체(인덱서, 블록 익스플로러, 개별 앱)는 별도의 저장 대책(예: IPFS, Filecoin 연동)을 수립해야 한다. 기술사는 이를 위해 **'Data Availability Layer'**의 이중화 설계를 제안해야 한다.
- **가스 가격 시장의 분리**: 블롭 가스비는 일반 가스비와 별개로 움직인다. 따라서 메인넷에서 NFT 민팅 전쟁이 일어나 가스비가 폭등해도, 블롭을 쓰는 L2 수수료는 여전히 저렴하게 유지될 수 있는 **'수수료 격리 효과'**를 실무 운영 시 적극 활용해야 한다.

📢 **섹션 요약 비유**: 메인 도로가 명절 정체(가스비 폭등)로 꽉 막혀도, 옆에 뚫린 블롭 전용 차선은 텅텅 비어 있어 화물차(L2)들이 쌩쌩 달릴 수 있는 '차별화된 인프라 운용'이 가능해집니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성적 기대효과
- **Mass Adoption의 기틀**: 누구나 부담 없이 블록체인을 쓸 수 있는 '저비용 고성능' 시대의 개막.
- **롤업 경쟁력 강화**: 사이드체인보다 보안성이 뛰어난 롤업이 비용 경쟁력까지 갖추게 됨.

### 2. 미래 전망
- **Full Danksharding으로의 진화**: 블롭의 개수가 블록당 수백 개로 늘어나며 수십만 TPS를 처리하는 진정한 모듈러 블록체인의 완성.
- **이더리움의 데이터화**: 이더리움은 점차 '연산 플랫폼'에서 전 세계 롤업들을 위한 '가장 신뢰할 수 있는 데이터 게시판'으로 그 정체성이 변화할 것이다.

📢 **섹션 요약 비유**: EIP-4844는 이더리움이라는 거대한 도서관에 '데이터 전용 자동 반납대'를 설치한 것이며, 이를 통해 도서관은 전보다 수만 배 더 많은 지식을 순환시킬 수 있게 되었습니다.

### 📌 관련 개념 맵
- **댕크샤딩 (Danksharding)**: EIP-4844가 최종적으로 지향하는 기술적 목표.
- **레이어 2 (Rollup)**: EIP-4844 업그레이드의 최대 수혜자.
- **데이터 가용성 (DA)**: 블롭이 해결하고자 하는 근본적인 기술적 난제.
- **The Surge**: EIP-4844가 속한 이더리움의 확장성 단계 명칭.

### 👶 어린이를 위한 3줄 비유 설명
1. **임시 포스트잇**: 일기장에 직접 글을 쓰면 종이가 아까우니까, 살짝 붙였다 뗄 수 있는 저렴한 '마법 포스트잇(블롭)'에 메모를 하는 거예요.
2. **저렴한 택배비**: 무거운 짐을 보낼 때 비싼 비행기 대신 아주 싼 화물 기차를 새로 만든 것과 같아서, 이제 코인을 보낼 때 세뱃돈을 아낄 수 있어요.
3. **18일 뒤에 펑!**: 포스트잇은 18일이 지나면 마법처럼 사라져서 일기장이 두꺼워지지 않게 도와준답니다. 중요한 내용은 따로 적어둬야겠죠?
