# 페이즈드 어텐션 (PagedAttention): 운영체제의 지혜로 LLM의 메모리 낭비를 해결한 혁신적 서빙 아키텍처

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이즈드 어텐션(PagedAttention)은 LLM 추론 시 발생하는 KV 캐시를 연속된 메모리 공간이 아닌, 운영체제의 가상 메모리 기법처럼 작은 **'페이지(Page)'** 단위로 쪼개어 비연속적인 물리 메모리에 동적으로 할당하는 알고리즘이다.
> 2. **가치**: 기존 서빙 방식에서 발생하던 메모리 파편화(Fragmentation)와 최대 길이 선점(Over-reservation) 문제를 해결하여, 동일한 GPU 자원에서 추론 처리량(Throughput)을 최대 **2~4배**까지 향상시키는 'AI 인프라의 경제학'을 실현했다.
> 3. **융합**: 컴퓨터 아키텍처의 가상 메모리(Virtual Memory) 및 페이징(Paging) 개념이 생성형 AI의 메모리 병목 문제와 완벽하게 융합된 사례로, 현재 업계 표준 서빙 엔진인 **vLLM**의 핵심 엔진으로 자리 잡았다.

+++

## Ⅰ. 페이즈드 어텐션 (PagedAttention) 개요

### 1. 정의
- UC 버클리 연구진이 개발한 기술로, KV 캐시 데이터를 불연속적인 메모리 블록에 저장하고 이를 **페이지 테이블(Page Table)**을 통해 매핑하여 관리하는 메모리 최적화 기법.

### 💡 비유: "고정된 예약석 vs 자유로운 도서관 좌석"
- **기존 방식 (Static Allocation)**은 식당에 예약할 때, 손님이 1명이 올지 10명이 올지 모르니 일단 **'가장 큰 10인용 테이블(최대 문장 길이)'**을 미리 잡아두는 것과 같습니다. 실제로는 2명만 와도 나머지 8인석은 비어있지만 다른 손님이 앉을 수 없어 엄청난 공간 낭비가 발생합니다.
- **페이즈드 어텐션**은 **'도서관의 칸막이 책상'**과 같습니다. 학생(토큰)이 올 때마다 비어있는 자리(페이지)를 하나씩 찾아 앉힙니다. 자리가 서로 떨어져 있어도 사서(페이지 테이블)가 누가 어디 앉았는지 장부에 다 적어두기 때문에 문제없습니다. 빈자리가 생기면 즉시 다음 학생을 받을 수 있어 공간 효율이 100%에 가깝습니다.

### 3. 등장 배경
- **KV 캐시의 거대함**: LLM은 답변을 생성할수록 메모리 사용량이 늘어나는데, 기존 시스템은 미래에 얼마나 길어질지 몰라 최대 길이를 미리 할당(Pre-allocation)해야 했음.
- **메모리 낭비 (Waste)**: 실제 실험 결과, 기존 방식은 전체 GPU 메모리의 **60~80%**를 실제 데이터 없이 '빈 예약 공간'으로만 낭비하고 있었음.

📢 **섹션 요약 비유**: 덩치 큰 가구(연속 메모리)를 통째로 옮기려다 포기하고, 가구를 조립식(페이지)으로 바꿔 좁은 틈새 공간까지 알뜰하게 활용하는 수납의 마법입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 페이즈드 어텐션의 핵심 구성 요소 (OS 융합)

| 구성 요소 | 영문명 | 역할 (OS 개념 매핑) | 비유 |
|:---:|:---|:---|:---|
| **논리적 블록** | Logical Blocks | 사용자가 보는 연속된 문장의 KV 캐시. | 가상 주소 공간 |
| **물리적 블록** | Physical Blocks | 실제 GPU VRAM 상의 쪼개진 메모리 조각. | 물리 프레임 (Frame) |
| **블록 테이블** | Block Table | 논리 블록과 물리 블록의 매핑 정보를 저장. | **페이지 테이블 (Page Table)** |
| **스케줄러** | Scheduler | 어떤 블록을 비우고 채울지 결정하는 제어부. | 가상 메모리 관리자 |

### 2. vLLM의 메모리 관리 아키텍처 (ASCII)

```text
┌─────────────────────────────────────────────────────────────┐
│                 [ PagedAttention Architecture ]             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [ User Request: "Tell me a story..." ]                    │
│          │                                                  │
│   ┌──────▼───────┐        ┌──────────────────────────────┐  │
│   │ Logical KV   │        │      Block Table (Map)       │  │
│   │   Blocks     │        ├──────────────┬───────────────┤  │
│   ├──────────────┤        │ Logic Block  │ Physical Addr │  │
│   │ Block 0      │ ──────▶│      0       │   0x7F01 (GPU)│  │
│   │ Block 1      │ ──────▶│      1       │   0xA23C (GPU)│  │
│   └──────────────┘        └──────────────┴───────┬───────┘  │
│                                                  │          │
│ ─────────────────────────────────────────────────┼───────── │
│                                                  │          │
│   [ Physical GPU VRAM Memory ]                   ▼          │
│   ┌──────────────┐  ┌──────────────┐      ┌──────────────┐  │
│   │  Used (Blk 0)│  │     Free     │      │ Used (Blk 1) │  │
│   └──────────────┘  └──────────────┘      └──────────────┘  │
│   ┌──────────────┐  ┌──────────────┐      ┌──────────────┐  │
│   │     Free     │  │  Used (New)  │      │     Free     │  │
│   └──────────────┘  └──────────────┘      └──────────────┘  │
│                                                             │
│   * 핵심: 물리 메모리가 파편화되어 있어도 논리적으로는 하나로 연결! │
└─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 
기존에는 KV 캐시를 위해 GPU 메모리를 한 줄로 길게(Contiguous) 빌려야 했다. 하지만 vLLM의 **페이즈드 어텐션**은 메모리를 16개 토큰 단위의 작은 **'블록'**으로 쪼갠다. 
문장이 길어지면 시스템은 GPU의 남는 빈 구석(Physical VRAM) 어디든 블록 하나를 할당해 준다. 이 조각난 주소들을 **Block Table**이 장부에 적어 관리하므로, AI 모델은 마치 메모리가 한 줄로 이어진 것처럼 착각하고 연산을 수행한다. 덕분에 메모리 낭비가 거의 0%에 수렴하며, 남는 공간에 더 많은 사용자를 동시에 태울 수 있게 된다.

### 3. 심층 동작 원리: Copy-on-Write (CoW) 기반 빔 서치(Beam Search)
- 여러 개의 답변 후보를 동시에 만드는 빔 서치(Beam Search) 시, 과거의 공통된 문맥(KV 캐시)은 메모리에 단 하나만 복사해두고 여러 후보군이 공유해서 쓴다. 그러다 내용이 달라지는 시점에만 새로운 메모리 블록을 할당한다(Copy-on-Write). 이는 OS의 프로세스 포크(Fork)와 동일한 원리다.

📢 **섹션 요약 비유**: 한 권의 책을 여러 명이 읽을 때, 각자 책을 한 권씩 사는 게 아니라 한 권의 책을 같이 보다가(메모리 공유), 메모할 때만 자기 포스트잇을 붙이는(CoW) 방식입니다.

## Ⅲ. 융합 비교 및 기술 분석

### 1. 기존 서빙 엔진 vs vLLM (PagedAttention) 비교

| 비교 항목 | 기존 엔진 (HuggingFace 등) | vLLM (페이즈드 어텐션) |
|:---:|:---|:---|
| **메모리 할당** | 정적 / 최대 길이 기반 | **동적 / 페이지 단위 기반** |
| **메모리 낭비** | 60~80% (Over-reservation) | **4% 미만 (최적 활용)** |
| **파편화 문제** | 심각함 (External Frag.) | **거의 없음 (Paging 기술)** |
| **처리량 (Throughput)**| 1.0 (기준) | **2.0 ~ 4.0 (획기적 향상)** |
| **복합 알고리즘 지원**| 빔 서치 시 메모리 폭발 | **공유 블록 기법으로 효율적 지원** |

### 2. 과목 융합 관점 (OS & 인공지능)
- **가상 메모리 시스템**: 1960년대 운영체제에서 고안된 '페이징'과 '페이지 테이블' 개념이 2020년대 인공지능의 'KV 캐시 관리' 문제에 그대로 적용되었다. 이는 컴퓨터 공학의 기본기가 최첨단 기술의 병목을 해결하는 전형적인 사례다.
- **컴퓨터 아키텍처 (NUMA)**: 여러 대의 GPU가 연결된 환경에서 각 GPU의 메모리 페이지를 어떻게 효율적으로 배치하고 통신(RDMA)할 것인지에 대한 하드웨어 융합 설계로 이어진다.

📢 **섹션 요약 비유**: 최신형 우주선(AI)의 연료 누출 문제를 해결하기 위해, 오래된 범선(OS)의 방수 격벽 기술을 가져와 완벽하게 수리한 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 1. 실무 시나리오: 글로벌 SaaS 챗봇의 GPU 원가 절감
- **상황**: 동시 접속자가 늘어날수록 GPU(A100) 대수가 기하급수적으로 늘어나 서버 비용 때문에 적자 발생.
- **적용**: **vLLM 기반 PagedAttention 서빙 전환**.
  - 1단계: 기존의 정적 할당 서빙 모델을 vLLM 엔진으로 교체.
  - 2단계: 블록 사이즈를 16으로 설정하고, 빔 서치를 적용하여 답변 품질 향상.
  - 3단계: 남는 VRAM 공간에 배클(Batch) 크기를 4배로 키움.
- **결과**: 동일 GPU 대수로 3배 더 많은 사용자 수용 성공. 인프라 운영 비용 66% 절감.

### 2. 기술사적 판단 (Insight)
- **최적의 블록 사이즈 결정**: 블록을 너무 작게 나누면 매핑 정보(Page Table)가 너무 많아져 관리 오버헤드가 생기고, 너무 크게 나누면 여전히 내부 파편화가 발생한다. 기술사는 **하드웨어의 캐시 라인 크기**와 **평균 문장 길이**를 고려하여 최적의 블록 사이즈(보통 8 or 16)를 튜닝해야 한다.
- **스왑(Swapping) 아키텍처**: GPU 메모리가 꽉 차면 덜 중요한 페이지를 CPU RAM으로 잠시 밀어냈다가 다시 가져오는 **Swapping** 기법을 적용할 수 있다. 기술사는 이때 발생하는 PCIe 대역폭 지연과 추론 속도 사이의 트레이드오프를 설계에 반영해야 한다.

📢 **섹션 요약 비유**: 창고(GPU VRAM)가 꽉 찼다고 새 창고를 짓는(GPU 증설) 대신, 짐을 박스(페이지)에 잘 담아 천장 끝까지 쌓아 올리고, 자주 안 쓰는 짐은 마당(CPU RAM)으로 잠시 내놓는 영리한 창고지기가 되어야 합니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성적 기대효과
- **AI 서비스 경제성 확보**: 추론 원가를 파격적으로 낮추어 B2C AI 서비스의 유료화 및 수익성 개선에 결정적 기여.
- **대규모 언어 모델 대중화**: 고성능 서빙 기술 덕분에 중소기업도 합리적인 비용으로 거대 모델을 서비스할 수 있는 생태계 조성.

### 2. 미래 전망
- **분산 페이징 (Distributed Paging)**: 수천 대의 서버에 흩어진 GPU 메모리를 하나의 거대한 공유 페이지 풀로 관리하는 클라우드 네이티브 서빙 기술로 진화할 것이다.
- **추론 엔진의 커널화**: 현재는 라이브러리 수준인 PagedAttention이 향후에는 AI 전용 가속기(NPU)의 하드웨어 커널이나 전용 OS의 핵심 기능으로 내재화될 전망이다.

📢 **섹션 요약 비유**: 페이즈드 어텐션은 인공지능이 '어떻게 똑똑해질 것인가'를 고민할 때, '어떻게 효율적으로 기억할 것인가'를 해결하여 비즈니스의 완성도를 높인 실질적인 공학의 승리입니다.
