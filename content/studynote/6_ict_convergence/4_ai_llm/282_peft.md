# PEFT (Parameter-Efficient Fine-Tuning): 거대 모델의 뇌는 동결하고 '보조 뇌'만 가르치는 효율 극대화 기술

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: PEFT는 수천억 개의 파라미터를 가진 거대 모델(LLM) 전체를 재학습시키는 대신, 원본 모델의 가중치는 고정(Freeze)하고 극히 일부의 추가 파라미터(약 1% 미만)만 업데이트하여 특정 업무에 적응시키는 효율적 미세 조정 기법이다.
> 2. **가치**: 모델 전체를 튜닝할 때 드는 막대한 VRAM(비디오 메모리) 사용량과 컴퓨팅 비용을 90% 이상 절감하여, 일반 소비자용 GPU 한 장만으로도 초거대 AI를 우리 회사에 맞게 튜닝할 수 있는 'AI 개발의 민주화'를 실현한다.
> 3. **융합**: 가중치를 분해하는 LoRA, 입력단에 힌트를 넣는 프롬프트 튜닝(Prompt Tuning), 층 사이에 모듈을 끼우는 어댑터(Adapter) 기법으로 나뉘며, 양자화 기술(QLoRA)과 융합되어 모바일 기기 내 학습(On-device Learning)의 기반이 된다.

+++

## Ⅰ. PEFT (Parameter-Efficient Fine-Tuning) 개요

### 1. 정의
- 사전 학습된 거대 언어 모델(PLM)의 대부분의 파라미터를 수정하지 않고 유지하면서, 특정 태스크를 위해 설계된 아주 작은 규모의 파라미터 셋만 훈련시키는 최적화 방법론이다.

### 💡 비유: "천재 피아니스트의 새로운 곡 마스터하기"
- **전체 파인튜닝(Full FT)**은 천재 피아니스트에게 새로운 현대 음악 한 곡을 가르치기 위해, 그가 여태껏 배운 모든 클래식 주법과 손가락 근육 사용법(전체 가중치)을 처음부터 다시 뜯어고쳐 훈련시키는 것과 같습니다. 시간도 오래 걸리고 기존 실력(상식)을 망칠 수도 있습니다.
- **PEFT**는 피아니스트의 천재적인 손가락 실력은 그대로 둔 채, 악보 옆에 붙은 **'작은 주석(보조 파라미터)'** 몇 줄만 외우게 하는 것입니다. 피아니스트는 원래 실력으로 연주하되 주석만 참고하여 완벽하게 새 곡을 소화합니다. 몸(원본 모델)은 힘들지 않고 머리(보조 모듈)만 살짝 써서 효율적입니다.

### 3. 등장 배경
- **Full Fine-Tuning의 물리적 한계**: Llama-3 70B 모델 전체를 튜닝하려면 수 테라바이트의 VRAM이 필요하며, 학습 후 저장되는 모델 용량도 원본만큼 거대하여 배포와 관리가 불가능함.
- **도메인 변화 속도**: 비즈니스 환경은 매일 변하는데, 그때마다 거대 모델을 통째로 재학습시키는 것은 경제적으로 수지타산이 맞지 않음.

📢 **섹션 요약 비유**: 무거운 코끼리(거대 모델)를 직접 움직이는 대신, 코끼리 등에 탄 가벼운 조련사(PEFT 모듈)만 교육시켜 코끼리의 방향을 바꾸는 기술입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. PEFT의 3대 핵심 기술 분류

| 분류 | 영문명 | 작동 원리 | 대표 기법 |
|:---:|:---|:---|:---|
| **가산형** | Additive | 모델의 층 사이나 입력단에 새로운 작은 층을 '추가'하여 그것만 학습. | Adapters, Prompt Tuning |
| **선택형** | Selective | 기존 파라미터 중 중요도가 높은 극일부만 골라내어 '활성화' 학습. | BitFit |
| **재구성형** | Reparameterization | 기존 가중치 행렬을 낮은 차원의 행렬로 '분해'하여 학습 효율화. | **LoRA (가장 대세)** |

### 2. PEFT vs Full Fine-Tuning 비교 아키텍처 (ASCII)

```text
┌─────────────────────────────────────────────────────────────┐
│           [ FULL FINE-TUNING vs PEFT (LoRA) ]               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [ 1. Full Fine-Tuning (과거) ]                            │
│   ┌──────────────────┐                                      │
│   │ [ Large Weights ]│ ◀─── [ Update ALL Gradients ]        │
│   │   (100% Params)  │      (Requires Massive VRAM)         │
│   └──────────────────┘                                      │
│                                                             │
│ ─────────────────────────────────────────────────────────── │
│                                                             │
│   [ 2. PEFT (LoRA - 현재 대세) ]                             │
│                                                             │
│   ┌──────────────────┐      ┌──────────────┐                │
│   │ [ Large Weights ]│      │ [LoRA Module]│                │
│   │    (FROZEN)      │ ───▶ │ (Update 1%)  │                │
│   │ (No Training)    │      └──────────────┘                │
│   └──────────────────┘             │                        │
│            │                       │                        │
│            └───────────┬───────────┘                        │
│                        ▼                                    │
│               [ Final Output Y ]                            │
│                                                             │
│   * 장점: VRAM 사용량 80% 감소, 모델 저장용량 1/10000 감소.      │
└─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 
과거의 방식은 모델의 모든 신경망 연결(Full Weights)을 하나하나 다 계산하고 수정해야 했다. 
하지만 **PEFT의 대표 격인 LoRA**는 원본 뇌(Large Weights)는 **'동결(Frozen)'** 시켜서 건드리지 않는다. 대신 그 옆에 아주 얇고 작은 보조 신경망(LoRA Module)을 붙여서, 들어오는 데이터가 이 보조 신경망만 통과하며 학습되게 한다. 연산량이 극적으로 줄어들고, 학습이 끝나면 원본 모델(70GB)은 그대로 둔 채 10MB짜리 보조 모듈 파일만 저장하면 되므로 관리가 매우 쉽다.

### 3. 심층 동작 원리: 랭크 분해 (Rank Decomposition)
- 행렬 연산 시 $100 \times 100$ 행렬(10,000개 파라미터)을 업데이트하는 대신, 이를 $100 \times 2$와 $2 \times 100$ 행렬(총 400개 파라미터)로 쪼개어 학습하는 방식이다. 수학적으로 모델의 실질적인 정보량(Intrinsic Rank)이 생각보다 낮다는 점을 이용한 고도의 효율화 전략이다.

📢 **섹션 요약 비유**: 두꺼운 백과사전의 모든 내용을 고치는 것이 아니라, 중요한 부분에만 **'포스트잇(PEFT 모듈)'**을 붙여서 내용을 보충하는 효율적인 수정 방식입니다.

## Ⅲ. 융합 비교 및 기술 분석

### 1. PEFT 주요 기법 상세 비교

| 기법 명칭 | 방식 (Method) | 적용 위치 | 추론 속도 저하 | 특징 |
|:---:|:---|:---|:---:|:---|
| **Adapter** | Serial Insert | 트랜스포머 층 사이 | 있음 (연산 추가) | 구현이 직관적이나 추론 지연 발생. |
| **Prefix Tuning** | Virtual Tokens | KV 캐시 앞부분 | 거의 없음 | 입력 텍스트 앞에 가상 토큰을 붙임. |
| **LoRA** | Bypass Matrix | 가중치 행렬 옆 | **없음 (병합 가능)** | 가장 널리 쓰임. 원본과 합칠 수 있음. |
| **Prompt Tuning** | Input Embedding | 입력 임베딩 층 | 없음 | 프롬프트 자체를 벡터로 학습. |

### 2. 과목 융합 관점 (인프라 & 소프트웨어 공학)
- **인프라 비용 최적화 (FinOps)**: PEFT를 도입하면 고가의 A100 GPU 8대를 써야 하던 작업을 일반 RTX 3090 1대로 수행할 수 있다. 이는 클라우드 비용을 1/10 이하로 줄이는 인프라 융합 기술이다.
- **멀티-테넌트(Multi-tenant) 아키텍처**: 하나의 거대 모델을 띄워두고, 사용자 A에게는 'A 업무용 PEFT 모듈'을, 사용자 B에게는 'B 업무용 모듈'을 동적으로 바꿔 끼우며 서빙(Serving)할 수 있다. 소프트웨어 공학의 '플러그인 아키텍처'와 인공지능이 융합된 형태다.

📢 **섹션 요약 비유**: 거대한 엔진(원본 모델) 하나에, 경주용 타이어(모듈 A), 오프로드용 타이어(모듈 B)만 갈아 끼우며 상황에 맞춰 운전하는 레이싱 전략과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 1. 실무 시나리오: 다국어 고객 센터 AI 상담 시스템
- **상황**: 하나의 LLM으로 한국어, 영어, 일본어 상담을 전문적으로 수행해야 함. 각 언어별로 모델을 통째로 만들면 서버비가 감당 안 됨.
- **적용**: **Multi-LoRA 아키텍처**.
  - 베이스 모델로 Llama-3를 선정하여 서버에 1대만 로딩.
  - 한국어 법률 상담용 LoRA 모듈(10MB), 영어 IT 상담용 LoRA 모듈(10MB)을 각각 훈련.
  - 사용자가 접속하면 질문의 언어를 감지하여, 해당 LoRA 모듈만 베이스 모델에 실시간으로 '결합'하여 답변.
- **결과**: 서버 자원은 1대 값만 쓰면서, 전문 상담사 3명을 보유한 효과 달성.

### 2. 기술사적 판단 (Insight)
- **튜닝의 트레이드오프**: PEFT는 빠르고 가볍지만, 학습 데이터가 수백만 건 이상으로 너무 많을 때는 전체 파인튜닝(Full FT)보다 성능이 약간 떨어질 수 있다. 기술사는 데이터 규모가 작을 때는 **PEFT**를, 데이터가 넘쳐나고 모델의 근본적인 지능을 개조해야 할 때는 **Full FT**를 선택하는 기준을 세워야 한다.
- **버전 관리의 혁신**: PEFT는 수십 메가바이트의 파일만 관리하면 되므로, Git과 같은 형상 관리 도구와 융합하여 'AI 모델의 버전 관리'를 소프트웨어 코드처럼 쉽고 빠르게 수행할 수 있게 한다.

📢 **섹션 요약 비유**: 집을 매번 새로 짓는(Full FT) 건축가는 망합니다. 튼튼한 골조(Base Model)는 두고 내부 인테리어(PEFT)만 바꿔서 호텔도 만들고 병원도 만드는 건축가가 진정한 기술사입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성적 기대효과
- **학습 자원 99% 절감**: 파라미터 업데이트 개수를 1/1000로 줄여 GPU 전기세와 임대료 획기적 절감.
- **빠른 실험 사이클 (Time-to-Value)**: 모델을 통째로 굽는 데 걸리던 며칠의 시간을 단 몇 시간으로 단축하여, 비즈니스 아이디어를 즉시 AI에 적용 가능.

### 2. 미래 전망
- **온디바이스 개인화 완성**: 스마트폰 성능이 좋아지면서, 밤사이 사용자의 일기를 PEFT(LoRA)로 학습하여 아침에 완벽한 개인 비서로 변신하는 시나리오가 표준이 될 것이다.
- **Federated PEFT (연합 학습)**: 보안이 중요한 의료/금융 데이터를 서버로 보내지 않고, 로컬에서 PEFT 모듈만 학습시킨 뒤 그 '가중치 증분'만 서버로 보내 합치는 프라이버시 보호형 AI 학습이 대중화될 전망이다.

📢 **섹션 요약 비유**: PEFT는 거대한 인공지능이라는 기차를 멈추지 않고도, 목적지에 따라 객차 한 칸만 갈아 끼우는 '가장 우아한 철도 시스템'입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **커다란 로봇과 작은 조종석**: 엄청나게 큰 로봇(거대 모델)이 있는데, 이 로봇 전체를 고치려면 돈이 너무 많이 들어요.
2. **조종석만 바꾸기**: 그래서 로봇 몸체는 그대로 두고, 조종석에 앉은 조종사(PEFT 모듈)만 새로 교육시키는 거예요.
3. **싸고 똑똑한 로봇**: 그러면 로봇은 힘이 센 그대로이면서, 아주 적은 돈으로 요리도 하고 청소도 하는 새로운 기술을 금방 배울 수 있답니다!
