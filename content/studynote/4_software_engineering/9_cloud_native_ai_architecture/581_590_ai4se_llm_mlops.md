# 581-590. AI4SE, LLM 기반 코드 생성, RAG, MLOps

> **핵심 인사이트**: AI는 소프트웨어 엔지니어링의 **"보조 역(Copilot)"**로 진화하고 있습니다. 개발자를 대체하는 것이 아니라, **생산성을 10배 향상**시키는 도구입니다. 하지만 **할루시네이션(환각)**과 **보안/라이선스 이슈**를 이해하고 적절히 통제해야 합니다.

---

## Ⅰ. AI4SE (AI for Software Engineering)

### 1. 개념
AI를 활용하여 소프트웨어 엔지니어링 생산성을 향상시키는 패러다임입니다.

### 2. 적용 영역
```
┌─────────────────────────────────────────────────────────────────────┐
│                    AI4SE Application Areas                            │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐    │
│  │   Code Generation│  │   Code Review    │  │   Testing        │    │
│  │   (코드 생성)     │  │   (코드 리뷰)     │  │   (테스트)        │    │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘    │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐    │
│  │   Documentation  │  │   Debugging      │  │   Refactoring    │    │
│  │   (문서화)        │  │   (디버깅)        │  │   (리팩토링)     │    │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘    │
└─────────────────────────────────────────────────────────────────────┘
```

### 3. 주요 도구
| 도구 | 개발사 | 특징 |
|-----|-------|------|
| **GitHub Copilot** | GitHub/OpenAI | IDE 통합, 실시간 제안 |
| **Cursor** | Cursor | AI 네이티브 IDE |
| **Claude Code** | Anthropic | CLI 기반 에이전트 |
| **Amazon CodeWhisperer** | AWS | AWS 서비스 특화 |
| **Tabnine** | Tabnine | 프라이빗 배포 지원 |

## Ⅱ. LLM 기반 코드 생성

### 1. 작동 원리
```
[자연어 프롬프트] ──▶ [LLM] ──▶ [코드 생성]
        │                       │
        ▼                       ▼
[컨텍스트 이해]         [패턴 매칭] ──▶ [구문 생성]
```

### 2. 코드 생성 유형
* **완성(Fill-in-the-middle)**: 코드 중간 자동 완성
* **생성(Generation)**: 자연어에서 코드 생성
* **변환(Transformation)**: 코드 변환/리팩토링
* **설명(Explanation)**: 코드 설명 생성

### 3. 할루시네이션(Hallucination) 문제
LLM이 존재하지 않는 API나 잘못된 코드를 생성하는 현상입니다.

| 원인 | 완화 방안 |
|-----|---------|
| **지식 부족** | RAG(Retrieval-Augmented Generation) |
| **잘못된 패턴** | Few-shot 예제 제공 |
| **과신** | 인간 검증 필수 |
| **컨텍스트 부족** | 충분한 컨텍스트 제공 |

### 4. 라이선스/저작권 이슈
* **학습 데이터**: 오픈소스 코드로 학습
* **복제 위험**: 기존 코드와 유사한 생성물
* **라이선스 전염**: GPL 코드 사용 시 전염 가능성
* **대응 전략**: 생성 코드 검토, SBOM 관리

## Ⅲ. RAG (Retrieval-Augmented Generation)

### 1. 개념
외부 지식 베이스에서 관련 정보를 검색하여 LLM 생성 품질을 향상시키는 기법입니다.

### 2. 아키텍처
```
┌─────────────────────────────────────────────────────────────────────┐
│                      RAG Architecture                                 │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐    │
│  │   Query          │  │   Retrieval      │  │   Generation    │    │
│  │   (질문)         │  │   (검색)         │  │   (생성)        │    │
│  │                  │  │                  │  │                 │    │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘    │
│          │                     │                     │             │
│          ▼                     ▼                     ▼             │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐    │
│  │   Embedding      │  │   Vector DB      │  │   LLM           │    │
│  │   (임베딩)        │  │   (벡터 저장소)   │  │   (언어 모델)    │    │
│  │   - OpenAI       │  │   - Pinecone     │  │   - GPT-4       │    │
│  │   - Cohere       │  │   - Weaviate     │  │   - Claude      │    │
│  │   - HuggingFace  │  │   - Milvus       │  │   - Llama       │    │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘    │
└─────────────────────────────────────────────────────────────────────┘
```

### 3. 검색 전략
* **밀도 검색(Dense Retrieval)**: 벡터 유사도 기반
* **희소 검색(Sparse Retrieval)**: 키워드 기반 (BM25)
* **하이브리드**: 밀도 + 희소 결합
* **재순위(Re-ranking)**: 검색 결과 재정렬

### 4. 청킹(Chunking) 전략
| 전략 | 설명 | 적합한 경우 |
|-----|------|-----------|
| **고정 크기** | 일정 토큰 수로 분할 | 일반적인 문서 |
| **의미 단위** | 문단/섹션 단위 | 구조화된 문서 |
| **재귀적** | 계층적 분할 | 복잡한 문서 |
| **슬라이딩 윈도우** | 겹치는 분할 | 문맥 보존 중요 |

## Ⅳ. LangChain 프레임워크

### 1. 개념
LLM 애플리케이션 개발을 위한 오픈소스 프레임워크입니다.

### 2. 핵심 구성 요소
```
┌─────────────────────────────────────────────────────────────────────┐
│                    LangChain Components                               │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐    │
│  │   Models         │  │   Prompts        │  │   Memory         │    │
│  │   - LLM          │  │   - Templates    │  │   - Short-term   │    │
│  │   - Chat Models  │  │   - Few-shot     │  │   - Long-term    │    │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘    │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐    │
│  │   Indexes        │  │   Chains         │  │   Agents        │    │
│  │   - Retrievers   │  │   - Sequential   │  │   - Tools       │    │
│  │   - Vector Store │  │   - Router       │  │   - Executors   │    │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘    │
└─────────────────────────────────────────────────────────────────────┘
```

### 3. 체인(Chain) 패턴
```python
from langchain.chains import LLMChain
from langchain.prompts import PromptTemplate

# 프롬프트 템플릿
template = "Write a {language} function to {task}"
prompt = PromptTemplate(template=template, input_variables=["language", "task"])

# 체인 구성
chain = LLMChain(llm=llm, prompt=prompt)

# 실행
result = chain.run(language="Python", task="sort a list")
```

## Ⅴ. Agentic AI 시스템

### 1. 개념
도구(Tool)를 직접 호출하고 작업을 자율적으로 수행하는 AI 시스템입니다.

### 2. 에이전트 아키텍처
```
┌─────────────────────────────────────────────────────────────────────┐
│                    Agentic AI Architecture                            │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                         Agent                                   │   │
│  │  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐ │   │
│  │  │   Planner       │  │   Executor      │  │   Evaluator     │ │   │
│  │  │   (계획)         │  │   (실행)         │  │   (평가)        │ │   │
│  │  └─────────────────┘  └─────────────────┘  └─────────────────┘ │   │
│  │                          │                     │                │   │
│  │                          ▼                     ▼                │   │
│  │  ┌─────────────────────────────────────────────────────────┐   │   │
│  │  │                    Tools (도구)                            │   │   │
│  │  │  - Code Execution  - Web Search  - Database Query       │   │   │
│  │  │  - File I/O        - API Call     - Calculator          │   │   │
│  │  └─────────────────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
```

### 3. 에이전트 유형
| 유형 | 특징 | 사용 사례 |
|-----|------|----------|
| **ReAct** | 추론 + 행동 | 일반적인 작업 |
| **Plan-and-Execute** | 계획 후 실행 | 복잡한 작업 |
| **Multi-Agent** | 다중 에이전트 협력 | 대규모 시스템 |
| **Hierarchical** | 계층적 구조 | 기업 업무 자동화 |

## Ⅵ. MLOps 파이프라인

### 1. 개념
머신러닝 모델의 전체 생명주기를 자동화하는 DevOps의 확장입니다.

### 2. MLOps 단계
```
┌─────────────────────────────────────────────────────────────────────┐
│                      MLOps Pipeline                                  │
│                                                                     │
│  [데이터 수집] ──▶ [전처리] ──▶ [학습] ──▶ [평가] ──▶ [배포] ──▶ [모니터링]│
│       │              │           │          │          │          │     │
│       ▼              ▼           ▼          ▼          ▼          ▼     │
│   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   │
│   │ Feature │   │ Data    │   │ Model   │   │ Model   │   │ Model   │   │
│   │ Store   │   │ Versioning│   │ Registry│   │ Serving │   │ Monitoring│   │
│   └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘   │
│                                                                     │
│                    [지속적 학습 (Continuous Learning)]                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 3. 핵심 구성 요소
| 구성 요소 | 설명 | 도구 |
|-----------|------|------|
| **Feature Store** | 특징 저장소 | Feast, Tecton |
| **Model Registry** | 모델 버전 관리 | MLflow, Vertex AI |
| **Model Serving** | 모델 서빙 | TensorFlow Serving, TorchServe |
| **Experiment Tracking** | 실험 추적 | MLflow, Weights & Biases |
| **Pipeline Orchestration** | 파이프라인 관리 | Kubeflow, Airflow |

### 4. 모델 드리프트(Model Drift)
시간이 지남에 따라 모델 성능이 저하되는 현상입니다.

| 유형 | 설명 | 탐지 방법 |
|-----|------|----------|
| **Data Drift** | 입력 데이터 분포 변화 | 통계적 검정 |
| **Concept Drift** | 타겟 개념 변화 | 성능 모니터링 |
| **Prediction Drift** | 예측 분포 변화 | 예측 분석 |

## Ⅶ. Edge AI / On-Device AI

### 1. 개념
클라우드가 아닌 엣지 디바이스에서 직접 AI 추론을 수행하는 방식입니다.

### 2. 모델 경량화 기법
```
[원본 모델] ──▶ [경량화] ──▶ [엣지 배포]
                    │
        ┌───────────┼───────────┐
        ▼           ▼           ▼
    [양자화]    [가지치기]    [지식 증류]
    (Quantization) (Pruning) (Distillation)
        │           │           │
        ▼           ▼           ▼
   INT8/FP16   구조 축소    작은 모델로 지식 전이
```

### 3. 경량화 비교
| 기법 | 크기 감소 | 정확도 손실 | 속도 향상 |
|-----|----------|------------|----------|
| **양자화** | 4-8x | 1-3% | 2-4x |
| **가지치기** | 2-10x | 0-5% | 2-5x |
| **지식 증류** | 10-100x | 2-10% | 5-20x |

### 4. 엣지 AI 프레임워크
* **TensorFlow Lite**: 모바일/임베디드
* **ONNX Runtime**: 크로스 플랫폼
* **Core ML**: Apple 디바이스
* **TensorRT**: NVIDIA GPU

📢 **섹션 요약 비유**: AI4SE는 **"비서"**와 같습니다. 문서를 작성하고, 정보를 찾고, 작업을 정리해 줍니다. LLM은 **"천재 작가"**입니다. 엄청난 양의 텍스트를 학습했지만, 가끔 **"할루시네이션"**(존재하지 않는 사실을 지어냄)을 일으킵니다. RAG는 **"도서관 사서"**입니다. 질문에 답하기 위해 관련 책을 찾아줍니다. MLOps는 **"공장 자동화"**입니다. 모델을 학습하고, 배포하고, 모니터링하는 전 과정을 자동화합니다. Edge AI는 **"주머니 과학자"**입니다. 클라우드 없이도 작동하는 작은 AI입니다.
