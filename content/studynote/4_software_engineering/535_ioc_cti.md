# 535. 침해 사고 지표 (IOC) 및 위협 인텔리전스 (CTI)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사이버 위협 인텔리전스 (CTI)는 조직에 위협이 되는 공격자의 동기, 의도, 능력을 분석한 지식이며, 침해 사고 지표 (IOC)는 해커가 시스템에 남긴 악성 파일 해시, IP 주소 등 **구체적인 범행 흔적을 데이터화한 식별자**다.
> 2. **가치**: 파편화된 보안 데이터를 유의미한 위협 정보로 변환하여 선제적 방어(Proactive Defense)를 가능하게 하며, **STIX/TAXII**와 같은 표준 프로토콜을 통해 전 세계 보안 커뮤니티와 실시간으로 위협 정보를 공유함으로써 집단 방어 체계를 구축한다.
> 3. **융합**: SIEM, EDR, SOAR 등 보안 솔루션의 탐지 룰(Rule) 생성의 핵심 소스로 활용되며, 현대 아키텍처에서는 위협 헌팅(Threat Hunting)과 결합되어 보이지 않는 침입자를 찾아내는 '디지털 조기 경보 시스템'의 역할을 수행한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 
  - **IOC (Indicators of Compromise)**: 이미 발생한 침해 사고의 유전 정보(IP, URL, Hash, Registry 등).
  - **CTI (Cyber Threat Intelligence)**: IOC를 포함하여 공격자의 수법(TTP), 타겟 산업군, 캠페인 정보 등을 종합한 고차원적 지식.

- **💡 비유**: IOC는 범인이 현장에 남긴 **"지문이나 신발 자국"**과 같습니다. CTI는 이 지문을 토대로 범인의 **"범행 수법, 선호하는 장소, 다음 타겟을 분석한 프로파일링 보고서"**와 같습니다.

- **등장 배경**:
  1. **고립된 보안의 한계**: 우리 회사만 잘 막는다고 해결되지 않는 글로벌 APT 공격이 증가하면서, 기업 간 위협 정보 공유의 필요성이 커졌다.
  2. **방대한 데이터의 홍수**: 매일 발생하는 수억 건의 로그 중 진짜 위험을 가려내기 위해 신뢰할 수 있는 '블랙리스트(IOC)'와 '맥락(Intelligence)'이 필요해졌다.
  3. **대응 속도의 경쟁**: 해킹 도구의 자동화로 인해 사람이 분석하기 전 이미 공격이 끝나는 상황을 막기 위해 머신러닝 기반의 자동화된 위협 정보 유통이 요구되었다.

- **📢 섹션 요약 비유**: 동네에 도둑이 들었을 때, 도둑의 인상착의(IOC)를 이웃들과 실시간으로 공유하고(STIX/TAXII), 도둑이 주로 밤 12시에 뒷문을 노린다는 사실(CTI)을 미리 알아 대비하는 마을 방범대 시스템과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 위협 정보의 계층 구조: 고통의 피라미드 (Pyramid of Pain)

공격자가 자신의 흔적(IOC)을 변경하기 위해 들여야 하는 고통(노력)의 정도를 나타낸다.

| 계층 | 지표 유형 (IOC) | 공격자의 변경 난이도 | 방어적 가치 |
|:---|:---|:---:|:---|
| **Top** | **TTPs (공격 수법)** | **매우 높음 (습관)** | 가장 강력한 탐지 기준 |
| **High** | **Tools (공격 도구)** | 높음 (새 도구 필요) | 공격 그룹 특정 가능 |
| **Middle** | **Network Artifacts** | 보통 | C2 통신 차단 효율적 |
| **Low** | **IP Addresses** | 낮음 (VPN 등으로 변경) | 단기적 차단 효과 |
| **Bottom** | **Hash Values** | 매우 낮음 (1비트만 바꿔도 변함) | 단순 악성코드 선별 |

---

### 위협 인텔리전스 공유 표준 아키텍처 (STIX & TAXII)

위협 정보를 표현하는 언어와 이를 나르는 트럭의 관계를 시각화한다.

```text
 ┌─────────────────────────────────────────────────────────────────────────────┐
 │                      STIX (표현 표준) 및 TAXII (전송 표준) 연동 구조              │
 ├─────────────────────────────────────────────────────────────────────────────┤
 │                                                                             │
 │ [위협 정보 생성자 (IdP)]           [위협 정보 소비자 (Subscriber)]             │
 │ (보안 연구소, 정부기관)             (기업 보안팀, SOC)                        │
 │        │                                │                                   │
 │  ┌─────┴─────┐                  ┌─────┴─────┐                               │
 │  │ STIX 객체  │                  │ SIEM / EDR│                               │
 │  │ (JSON DB) │                  │ (Rule Map)│                               │
 │  └─────┬─────┘                  └─────┬─────┘                               │
 │        │                              ▲                                     │
 │        ▼      [TAXII Protocol]        │                                     │
 │  ┌────────────┐   (HTTPS/REST)   ┌────────────┐                             │
 │  │ TAXII Server│ ◀──────────────▶ │ TAXII Client│                             │
 │  │ (Hub/Push) │                  │ (Poll/Pull)│                             │
 │  └────────────┘                  └────────────┘                             │
 │                                                                             │
 │  ※ STIX: "무엇이 위험한가?" (위협 객체, 캠페인, 공격기법 등 정의)                 │
 │  ※ TAXII: "어떻게 공유할 것인가?" (정보 교환 프로토콜)                           │
 └─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** CTI 생태계는 두 가지 표준에 의해 돌아간다. **STIX (Structured Threat Information eXpression)**는 위협 정보를 기계가 읽을 수 있도록 JSON 형식으로 구조화한 '언어'다. 여기에는 단순 IP(IOC)뿐만 아니라 공격 그룹, 공격의 목적, 취약점 정보 등이 객체(Object) 간의 관계로 정의된다. **TAXII (Trusted Automated eXchange of Intelligence Information)**는 이 STIX 데이터를 실시간으로 주고받는 '운송 통로'다. 기업은 TAXII 클라이언트를 통해 신뢰할 수 있는 서버에서 최신 위협 정보를 내려받아 자사의 **SIEM(통합 로그 분석)**이나 **방화벽**에 즉시 탐지 룰로 적용한다.

---

### 심층 동작 원리: 위협 헌팅 (Threat Hunting) 프로세스

CTI를 활용하여 시스템 내부에 이미 숨어 있는 적을 찾는 능동적 활동이다.
1. **가설 수립 (Hypothesis)**: CTI 정보를 바탕으로 "해커가 최근 유행하는 기술(TTP)로 우리 DB를 노리고 있을 것이다"라고 가정한다.
2. **데이터 조사 (Investigation)**: EDR이나 트래픽 로그에서 해당 TTP와 일치하는 미세한 징후(IOC)를 검색한다.
3. **탐지 및 대응 (Detection)**: 숨어 있던 백도어나 권한 상승 흔적을 발견하고 즉시 제거한다.
4. **가치**: 수동적인 경보(Alert) 대기 상태에서 벗어나, 공격자가 정보를 빼돌리기 전 먼저 선수를 치는 기술이다.

- **📢 섹션 요약 비유**: CCTV에 도둑이 찍히길 기다리는 게 아니라, 최근 도둑들이 '가스 점검원'으로 위장한다는 정보(CTI)를 입수하고, 단지 내 모든 가스 점검원의 신분을 선제적으로 확인하러 다니는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 전략적(Strategic) vs 전술적(Tactical) vs 기술적(Technical) CTI

| 분류 | 대상 | 내용 | 활용도 |
|:---|:---|:---|:---|
| **전략적 CTI** | 경영진, C-Level | 공격 주체, 동기, 산업 트렌드 | 예산 편성, 보안 전략 수립 |
| **전술적 CTI** | 보안 관리자 | 공격자의 TTP (수법), 도구 | 방어 인프라 아키텍처 개선 |
| **기술적 CTI** | 보안 관제 요원 | **IOC (IP, 해시, URL)** | 실시간 차단 룰, 블랙리스트 |

---

### 기술 융합: SOAR와 CTI의 자동화 결합

위협 정보가 들어오는 즉시 기계가 대응을 마친다.
- **Enrichment**: 새로운 외부 IP가 감지되면, SOAR가 CTI 서버에 자동으로 물어봐서 위험 점수(Risk Score)를 가져온다.
- **Auto-Containment**: 위험 점수가 90점 이상이면 사람이 승인하기 전에도 방화벽에서 해당 IP를 즉시 차단(Playbook 실행)한다.

- **📢 섹션 요약 비유**: 낯선 사람이 나타나면 스마트 도어락이 경찰 데이터베이스(CTI)에 얼굴을 대조해보고, 지명수배자라면 문을 쇠사슬로 꽁꽁 잠가버리는 자동 방범 시스템과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 피싱 메일을 통한 내부망 침투 탐지 및 전파 차단

- **문제 상황**: 일부 직원이 정교한 피싱 메일을 열어본 정황 포착. 메일 내부의 악성 링크는 아직 백신에서 탐지되지 않는 신종임.
- **의사결정 (CTI 기반 대응)**:
  1. **지표 추출**: 해당 메일에 포함된 파일의 해시값과 접속된 URL 정보를 추출하여 내부용 IOC로 등록한다.
  2. **인텔리전스 검색**: 유료/무료 위협 정보 포털(VirusTotal, AlienVault 등)에 해당 IOC를 검색하여, 유사한 공격을 수행한 해커 그룹의 특징을 파악한다.
  3. **STIX 전파**: 파악된 정보를 STIX 포맷으로 변환하여 그룹사 및 파트너사에게 공유한다.
  4. **전사 스캔**: 확보된 IOC를 전사 EDR(엔드포인트 탐지 및 대응) 솔루션에 주입하여, 동일한 파일이나 접속 흔적이 있는 모든 사내 PC를 5분 내로 찾아내어 격리한다.

### 도입 체크리스트
- **Information Freshness**: 위협 정보가 실시간으로 업데이트되는가? (1주일 전 데이터는 무용지물일 수 있음)
- **Standard Support**: 우리 회사의 보안 솔루션이 STIX 2.1 및 TAXII 2.1 표준을 지원하는가?
- **Contextualization**: 외부에서 온 방대한 위협 정보 중 우리 회사의 자산(OS 종류, 서비스 언어 등)과 관련된 것만 필터링하는 기능이 있는가?

### 안티패턴
- **단순 차단 맹신**: IP 기반 IOC 차단에만 몰두하는 행위. 공격자는 IP를 실시간으로 바꿀 수 있으므로 근본 해결책이 되지 않는다. (TTP 분석 병행 필수)
- **정보 공유 기피**: "우리만 알고 있어야지"라는 태도. 사이버 보안은 혼자 지킬 수 없으며, 공유하지 않으면 우리도 다른 회사의 정보를 받을 수 없어 전체적인 생존력이 떨어진다.

- **📢 섹션 요약 비유**: 범인의 인상착의(IOC)만 외우고 있으면 옷을 갈아입었을 때 놓치게 됩니다. 범인의 말투나 걷는 모양새(TTP)를 익혀야 어떤 변장을 해도 잡아낼 수 있다는 점을 명심해야 합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 세부 내용 |
|:---|:---|:---|
| **정량적 (Time)** | 사고 탐지 시간(MTTD) 70% 단축 | 이미 알려진 위협 정보를 통해 분석 단계 생략 및 즉시 차단 |
| **정량적 (Ops)** | 보안 관제 오탐률 감소 | 신뢰도 높은 인텔리전스 데이터를 기반으로 한 정교한 필터링 |
| **정성적 (Resilience)** | 집단 방어 능력 강화 | 표준 프로토콜 기반의 정보 공유로 산업 생태계 전체의 면역력 강화 |
| **정성적 (Insight)** | 선제적 위협 예측 | 공격자의 동향 파악을 통해 미래의 공격 시나리오에 대한 사전 대비 가능 |

### 미래 전망
- **AI-Driven OSINT**: AI가 다크웹, 커뮤니티, SNS 등 방대한 공개 정보(OSINT)를 실시간 크롤링하여 인간 분석가보다 빠르게 새로운 취약점과 공격 징후를 예견할 것이다.
- **Inter-Cloud Threat Sharing**: 클라우드 사업자들(AWS, Azure, GCP) 간의 대규모 위협 정보 공유 망이 구축되어, 플랫폼 수준에서의 글로벌 통합 방어가 실현될 것이다.

### 참고 표준
- **OASIS STIX / TAXII**: 정보 표현 및 전송 국제 표준
- **MITRE ATT&CK**: CTI의 전술/기술 지식 베이스
- **ISO/IEC 27010**: 정보 보안 간 정보 공유 지침

- **📢 섹션 요약 비유**: 예전에는 각자도생하며 각개격파 당하던 시대였다면, 이제는 전 세계 보안 요원들이 실시간 무전(CTI)을 주고받으며 거대한 악의 무리에 맞서 싸우는 '디지털 연합군'의 시대입니다.

## 👶 어린이를 위한 3줄 비유 설명
1. **IOC**는 도둑이 흘리고 간 **"특이한 모양의 단추"**처럼, 나쁜 사람이 누군지 알려주는 작은 증거예요.
2. **CTI**는 이 단추를 보고 **"이 도둑은 밤에만 나타나고, 사탕 가게만 노리는 사람이다"**라고 미리 분석해놓은 똑똑한 정보랍니다.
3. 이 정보들을 전 세계 경찰 아저씨들과 서로 나눠 가지면(STIX/TAXII), 도둑이 우리 동네에 오기도 전에 미리 잡을 수 있어요!
