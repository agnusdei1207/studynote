# 490. CVSS (Common Vulnerability Scoring System)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CVSS (공통 취약점 평가 시스템)는 보안 취약점의 특성과 심각도를 정량화하여 **0.0에서 10.0 사이의 점수로 환산하는 개방형 표준 프레임워크**다.
> 2. **가치**: 취약점의 기본 특성(Base), 시간적 변화(Temporal), 개별 환경 요인(Environmental)을 종합 고려하여, 수많은 취약점 중 어떤 것을 가장 먼저 조치해야 하는지에 대한 객관적인 우선순위 가이드라인을 제공한다.
> 3. **융합**: CVE 식별자와 1:1로 매핑되어 NVD에 공표되며, 기업의 취약점 관리 프로세스(Vulnerability Management)와 정보보호 공시의 핵심 정량 지표로 활용된다. 최신 표준은 v4.0으로 정밀도가 더욱 향상되었다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: CVSS는 보안 사고의 "위험 수준"을 누구나 납득할 수 있는 숫자로 바꾸는 계산기다. 취약점이 얼마나 공격하기 쉬운지(Exploitability)와 성공 시 얼마나 큰 타격을 주는지(Impact)를 수식에 대입하여 최종 점수를 산출한다.

- **💡 비유**: CVSS는 **"태풍의 강도를 나타내는 카테고리(1~5등급)"**와 같습니다. 풍속과 기압을 보고 태풍의 위험도를 미리 알 수 있듯, CVSS 점수를 보고 우리 시스템에 닥칠 '보안 태풍'의 위력을 미리 짐작하는 것입니다.

- **등장 배경**:
  1. **주관적 평가의 한계**: 과거에는 보안 회사마다 같은 취약점을 두고 '상', '중', '하'로 제각각 평가하여 혼란이 컸다.
  2. **우선순위 결정의 난제**: 매년 수만 개의 취약점이 쏟아지는 상황에서, 인력과 예산의 한계로 인해 "가장 위험한 것부터 고치자"는 합리적 근거가 필요했다.
  3. **글로벌 표준의 필요성**: 전 세계 보안 전문가들이 동일한 잣대로 위험을 측정하기 위해 FIRST(Forum of Incident Response and Security Teams) 주도로 개발되었다.

- **📢 섹션 요약 비유**: 복잡한 병 증상을 "통증 점수 1부터 10까지 중 얼마인가요?"라고 물어 수치화함으로써, 의사가 응급 수술 순서를 정할 수 있게 돕는 객관적 진단 체계와 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### CVSS v3.1 메트릭 그룹 구조

CVSS는 세 가지 메트릭 그룹의 조합으로 최종 점수를 산출한다.

| 메트릭 그룹 | 설명 | 주요 항목 | 비유 |
|:---|:---|:---|:---|
| **기본 (Base)** | 취약점 고유의 변하지 않는 속성 | 공격 경로, 복잡도, 권한, CIA 영향도 | 폭탄의 파괴력 |
| **시간 (Temporal)** | 시간에 따라 변하는 속성 | 공격 코드 존재 여부, 패치 가용성 | 도화선의 길이 |
| **환경 (Environmental)** | 실제 시스템 환경에 따른 속성 | 자산의 중요도, 보안 통제 현황 | 주변 건물의 인구수 |

---

### CVSS 점수 산정 아키텍처 및 등급 체계

취약점 특성 값이 수식에 입력되어 최종 등급으로 변환되는 흐름을 시각화한다.

```text
 ┌─────────────────────────────────────────────────────────────────────────────┐
 │                      CVSS v3.1 점수 산출 및 등급 체계 아키텍처                   │
 ├─────────────────────────────────────────────────────────────────────────────┤
 │                                                                             │
 │  [입력 값 (Metrics)] ────▶ [수학적 공식 (Formula)] ────▶ [최종 점수 (Score)]    │
 │                                                                             │
 │  1. 공격 가능성 (Exploitability)                                              │
 │     - Attack Vector (Network, Local 등)                                     │
 │     - Attack Complexity (Low, High)                                         │
 │     - Privileges Required, User Interaction                                 │
 │                                                                             │
 │  2. 영향도 (Impact)                                                          │
 │     - Confidentiality (기밀성)                                               │
 │     - Integrity (무결성)                                                     │
 │     - Availability (가용성)                                                  │
 │                                                                             │
 │  ┌───────────────────────────────────────────────────────────────────────┐  │
 │  │  Score (0.0 ~ 10.0) ──▶ Severity Rating                               │  │
 │  ├───────────────────────────────────────────────────────────────────────┤  │
 │  │  0.0          : None                                                  │  │
 │  │  0.1 ~ 3.9    : Low                                                   │  │
 │  │  4.0 ~ 6.9    : Medium                                                │  │
 │  │  7.0 ~ 8.9    : High                                                  │  │
 │  │  9.0 ~ 10.0   : Critical (⚠ 즉시 패치 필요)                             │  │
 │  └───────────────────────────────────────────────────────────────────────┘  │
 │                                                                             │
 └─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** CVSS 계산의 핵심은 **'기본(Base) 점수'**다. 예를 들어, 인터넷을 통해 원격에서(Network), 특별한 권한 없이(None), 쉽게(Low) 공격할 수 있고, 성공 시 서버의 모든 데이터를 지울 수 있다면(CIA: High) 10.0점 만점에 가까운 **Critical** 등급이 부여된다. 반면, 공격자가 서버 앞에 직접 와야 하고(Physical) 복잡한 조건이 필요하다면 점수는 낮아진다. 이 점수 체계 덕분에 보안 담당자는 수백 개의 경고 중 9.0점 이상의 Critical 항목부터 처리하는 과학적 워크플로우를 가질 수 있다.

---

### 심층 동작 원리: 벡터 문자열 (Vector String)

CVSS는 점수뿐만 아니라, 그 점수가 어떻게 도출되었는지를 압축된 문자열로 표현한다.
- **예시**: `CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H`
- **해석**: 
  - `AV:N`: Attack Vector is Network (원격 공격 가능)
  - `AC:L`: Attack Complexity is Low (공격 난이도 낮음)
  - `C:H/I:H/A:H`: CIA 모두에 심각한 타격(High)을 줌
- **가치**: 보안 장비들은 이 벡터 문자열을 읽고 "네트워크 공격이 가능하므로 방화벽에서 즉시 차단해야지"라고 자동으로 판단할 수 있다.

- **📢 섹션 요약 비유**: 혈액형과 혈압 수치를 `A+/120-80`처럼 기호로 적어두면 전 세계 어떤 의사라도 환자 상태를 즉시 이해하는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### CVSS v3.1 vs v4.0 비교 (진화하는 표준)

최근 발표된 v4.0은 현대 보안 환경의 특성을 더 정밀하게 반영한다.

| 항목 | CVSS v3.1 (기존) | CVSS v4.0 (최신) |
|:---|:---|:---|
| **분류 체계** | Base, Temporal, Environmental | Base, Threat, Environmental, Supplemental |
| **정밀도** | 영향도 평가가 다소 단순함 | **OT/IoT 시스템의 영향력(Safety)** 등 추가 |
| **명칭 변경** | Temporal ──▶ Threat | 실제 위협이 진행 중인지(Threat) 강조 |
| **사용자 체감** | 10.0점이 너무 많다는 불만 | 세분화된 지표로 점수 변별력 강화 |

---

### 융합 시너지: Risk-based Vulnerability Management (RBVM)

CVSS 점수를 자산의 중요도와 결합하여 실제 비즈니스 리스크를 계산한다.
- **공식**: `Business Risk = CVSS Base Score * Asset Criticality`
- **가치**: 똑같은 10.0점 취약점이라도, 홍보용 홈페이지 서버와 핵심 고객 DB 서버 중 어디를 먼저 고칠지 명확한 우선순위를 정해준다.

- **📢 섹션 요약 비유**: 아무리 무서운 맹수(CVSS 10.0)라도 우리 집 마당(중요 자산)에 있는 것과 먼 나라 동물원에 있는 것은 나에게 주는 위협이 완전히 다르다는 원리입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 보안 관제 시스템의 취약점 대응 필터링

- **문제 상황**: 취약점 스캐너에서 하루에 5,000건의 경고가 발생하여 운영팀이 과부하에 걸림.
- **의사결정 (CVSS 기반 필터링)**:
  1. **Tier 1 (Critical)**: CVSS 9.0 이상 항목은 24시간 내 패치를 의무화한다.
  2. **Tier 2 (High)**: 7.0 ~ 8.9 항목은 1주일 내 정기 점검 시간에 패치한다.
  3. **Tier 3 (Others)**: 7.0 미만은 차기 릴리스 일정에 포함하거나 설정 변경으로 대응한다.
  4. **보정 (Threat Metric)**: 만약 5.0점짜리 중간 위험 취약점이라도 실제 해커들이 공격 도구(Exploit)를 유포 중이라는 정보가 있다면 즉시 Critical 등급으로 격상하여 조치한다.

### 도입 체크리스트
- **Calculator**: 조직 내 보안 담당자들이 CVSS 계산기 사용법을 숙지하고 있는가?
- **SLA**: CVSS 점수대별 조치 기한(SLA)이 전사 정책으로 명문화되어 있는가?
- **Context**: NVD에서 제공하는 기본 점수 외에, 우리 회사의 네트워크 환경을 반영한 '환경(Environmental) 점수'를 따로 계산하는가?

### 안티패턴
- **숫자 맹신**: CVSS 점수만 보고 6.9점은 안 고치고 7.0점만 고치는 행위. 점수는 참고치일 뿐, 실제 비즈니스 영향도를 반드시 고려해야 한다.
- **기본 점수만 확인**: 대부분의 관리자가 Base Score만 확인한다. 하지만 패치가 나왔는지(Temporal), 내부망인지(Environmental)에 따라 실제 위험은 10점에서 2점으로 떨어질 수도 있다.

- **📢 섹션 요약 비유**: 수능 성적(CVSS)만 보고 학생을 뽑는 것보다, 학생의 적성과 가정 환경(내부 상황)을 같이 고려해야 가장 훌륭한 인재를 선발할 수 있다는 점과 같습니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 세부 내용 |
|:---|:---|:---|
| **정량적 (Prioritizing)** | 보안 패치 효율성 3배 향상 | 리스크 기반 대응으로 불필요한 단순 업무 감소 및 핵심 위협 집중 |
| **정량적 (Monitoring)** | 전사 보안 점수 지표화 | "우리 회사의 평균 CVSS 노출 지수는 4.5점"과 같은 성과 보고 가능 |
| **정성적 (Communication)** | 부서 간 갈등 해소 | "무서우니까 고쳐라"가 아닌 "표준 점수가 높으니 고쳐라"는 객관적 근거 |
| **정성적 (Compliance)** | 법적 소명 자료 확보 | 사고 발생 시 표준 가이드에 따라 최선을 다해 대응했음을 입증 |

### 미래 전망
- **EPSS (Exploit Prediction Scoring System)**: CVSS가 '심각도'라면, EPSS는 '실제로 공격당할 확률'을 통계적으로 예측한다. 미래에는 CVSS와 EPSS가 결합된 지능형 리스크 모델이 주류가 될 것이다.
- **Real-time Scoring**: 시스템 구성이 변할 때마다(서버 증설, 방화벽 규칙 변경) CVSS 환경 점수가 실시간으로 재계산되는 동적 대시보드가 보편화될 것이다.

### 참고 표준
- **CVSS v3.1 / v4.0 Specification**: 공식 가이드라인
- **FIRST.org**: CVSS 표준 관리 포럼
- **PCI-DSS**: 4.0점 이상 취약점에 대한 패치 의무화 규정 포함

- **📢 섹션 요약 비유**: 예전에는 감으로 날씨를 맞혔지만, 이제는 슈퍼컴퓨터가 계산한 강수 확률(CVSS)을 보고 우산을 챙기는 기상 과학의 시대로 보안도 나아가고 있습니다.

## 👶 어린이를 위한 3줄 비유 설명
1. 학교 가는 길에 있는 **"위험한 곳들"이 얼마나 무서운지 점수를 매겨보는 거예요.**
2. 차가 많이 다니는 큰길은 10점(매우 위험), 친구들이랑 같이 걷는 골목길은 2점(조금 위험)이라고 정하는 규칙과 같아요.
