# 521. 인공지능 모델 공격 방어 (적대적 예제, 데이터 포이즈닝)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인공지능 모델 공격 방어는 머신러닝 모델의 취약점을 악용하여 오판을 유도하는 **적대적 예제(추론 단계)**와 학습 데이터 자체를 오염시키는 **데이터 포이즈닝(학습 단계)** 공격으로부터 AI 시스템의 무결성을 지키는 기술적 보호 체계다.
> 2. **가치**: 자율주행차의 표지판 오인식, 생체 인증 우회 등 AI 기반 서비스의 치명적인 물리적·논리적 사고를 예방하며, 인공지능 판단의 **강건성(Robustness)**과 **신뢰성**을 확보하여 AI의 실전 배치를 가능하게 한다.
> 3. **융합**: 적대적 훈련(Adversarial Training), 입력 정제(Input Sanitization), 차분 프라이버시(Differential Privacy) 기술과 결합되며, 현대 아키텍처에서는 보안 중심의 MLOps(SecMLOps)를 통해 모델의 전 생애 주기를 보호한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 
  - **적대적 예제 (Adversarial Example)**: 사람이 보기엔 정상이지만 AI에게는 오판을 유도하는 미세한 노이즈를 섞은 데이터.
  - **데이터 포이즈닝 (Data Poisoning)**: AI가 특정 상황에서 오답을 내도록 유도하기 위해 나쁜 데이터를 미리 학습 데이터셋에 몰래 섞어넣는 행위.

- **💡 비유**: 
  - **적대적 예제**: 인공지능에게만 보이는 **"디지털 착시 현상"**과 같습니다. 마술사가 눈속임을 하듯, 해커가 AI의 시력을 교묘하게 비틀어 사과를 바나나로 보게 만드는 것입니다.
  - **데이터 포이즈닝**: 요리사(AI)에게 요리를 가르칠 때, 몰래 **"설탕통에 소금을 섞어놓는 것"**과 같습니다. 결국 요리사는 배운 대로 요리를 하지만 맛은 엉망(오판)이 됩니다.

- **등장 배경**:
  1. **딥러닝의 블랙박스 특성**: 모델 내부의 복잡한 의사결정 과정을 완벽히 이해하기 어렵기 때문에, 미세한 입력 변화가 결과를 뒤흔드는 취약 지점이 항상 존재한다.
  2. **AI의 실전 배치**: 국방, 금융, 의료 등 보안이 생명인 분야에 AI가 쓰이면서 공격 성공 시의 파급력이 인명 피해나 금전적 재앙으로 직결되기 시작했다.
  3. **공격의 용이성**: CleverHans, Foolbox 등 적대적 공격을 자동으로 수행해주는 오픈소스 도구가 보급되어 비전문가도 정교한 공격이 가능해졌다.

- **📢 섹션 요약 비유**: 시력이 아주 좋은 로봇(AI)이라도, 렌즈에 특수한 필터(노이즈)를 끼우면 눈앞의 낭떠러지를 꽃밭으로 착각할 수 있다는 위험성을 경고하고 이를 치료하는 기술입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 인공지능 공격 유형 및 단계별 방어 전략

| 공격 유형 | 발생 단계 | 공격 원리 | 주요 방어 기술 |
|:---|:---:|:---|:---|
| **적대적 예제** | **추론 (Inference)** | 데이터에 미세 노이즈 주입 | **적대적 훈련**, 입력 평활화 |
| **데이터 포이즈닝** | **학습 (Training)** | 학습 데이터셋 오염 | **이상치 제거**, 차분 프라이버시 |
| **모델 탈취** | 추론 (Inference) | 쿼리 반복을 통한 지능 복제 | 답변 노이즈 추가, 속도 제한 |
| **백도어 공격** | 학습 (Training) | 특정 트리거 시 오작동 유발 | 신경망 정제, 트리거 탐지 |

---

### 2. 적대적 훈련 (Adversarial Training) 아키텍처

모델의 강건성을 높이는 가장 강력한 방어 기법을 시각화한다.

```text
 ┌─────────────────────────────────────────────────────────────────────────────┐
 │                      적대적 훈련 (Adversarial Training) 모델 강화 흐름             │
 ├─────────────────────────────────────────────────────────────────────────────┤
 │                                                                             │
 │ [1. 공격 생성 단계] ──────▶ [2. 증강 학습 단계] ──────▶ [3. 강건한 모델 완성]      │
 │                                                                             │
 │  정상 데이터 (X)             새로운 학습 데이터셋             보안 강화 AI 모델     │
 │        │                    (X + X')                       (Defended Model)  │
 │        ▼                    ┌──────────────┐               ┌────────────┐    │
 │  적대적 공격 엔진  ────▶    │  정상 샘플    │ ──────▶       │ 노이즈에    │    │
 │  (FGSM, PGD 등)             │  + 공격 샘플  │               │ 면역력 확보 │    │
 │                             └──────────────┘               └────────────┘    │
 │                                                                             │
 │  ┌───────────────────────────────────────────────────────────────────────┐  │
 │  │ ✅ 방어 원리:                                                            │  │
 │  │ "매를 미리 맞아서 매집을 키우는 것" - 공격 데이터를 미리 학습에 포함시켜      │  │
 │  │ 미세한 노이즈가 섞여도 원래의 정답을 유지하도록 결정 경계를 부드럽게 함        │  │
 │  └───────────────────────────────────────────────────────────────────────┘  │
 │                                                                             │
 └─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 아키텍처의 핵심은 **공격 데이터의 내재화**다. 1) **공격 엔진**은 모델의 약점을 찾아내어 가장 치명적인 노이즈가 섞인 가짜 데이터($X'$)를 생성한다. 2) 이 가짜 데이터를 정상 데이터와 섞어서 모델을 **재학습**시킨다. 3) 이 과정을 반복하면 모델은 "이런 식의 노이즈는 무시하고 원래 정체에 집중해"라고 학습하게 된다. 결과적으로 결정 경계(Decision Boundary)가 안정적으로 형성되어, 해커가 웬만큼 노이즈를 섞어도 흔들리지 않는 **강건한 AI**가 탄생한다.

---

### 심층 동작 원리: 차분 프라이버시 (DP)와 포이즈닝 방어

학습 단계에서 한 명의 악성 사용자가 모델을 망치지 못하게 하는 기술이다.
1. **정의**: 특정 데이터가 포함되었을 때와 아닐 때의 결과 차이를 수학적으로 제한하는 기술.
2. **원리**: 학습 과정에서 각 데이터의 가중치를 제한(Clipping)하고 노이즈를 주입한다.
3. **가치**: 공격자가 아주 강력한 오염 데이터를 넣어도, 모델은 그 데이터 하나에 크게 휘둘리지 않고 전체적인 경향성에만 집중하게 된다.

- **📢 섹션 요약 비유**: 한 명의 거짓말쟁이(포이즈닝 데이터)가 큰 소리로 거짓을 말해도, 전교생의 평균 점수(모델)가 크게 변하지 않게 마이크 볼륨을 조절하는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 적대적 예제 생성 알고리즘 비교

| 알고리즘 | 특징 | 장점 | 단점 |
|:---|:---|:---|:---|
| **FGSM** | 단 한 번의 연산으로 노이즈 생성 | 매우 빠름 | 방어(적대적 훈련)에 취약함 |
| **PGD** | 여러 번의 미세 조정을 통해 생성 | **가장 강력한 공격** | 연산 속도가 느림 |
| **C&W Attack** | 최적화 기법을 사용하여 정교하게 생성 | 인간이 구별 불가능 | 수학적 복잡도 매우 높음 |

---

### 기술 융합: 설명 가능한 AI (XAI)와 보안

XAI는 적대적 공격을 탐지하는 훌륭한 도구가 된다.
- **원리**: AI가 왜 이 사진을 '바나나'라고 했는지 시각화했을 때, 물체의 형상이 아닌 구석의 낙서(노이즈)에 기여도가 쏠려 있다면 적대적 공격으로 판단한다.
- **가치**: 블랙박스였던 AI의 오판 이유를 밝혀내어, 공격의 징후를 조기에 포착한다.

- **📢 섹션 요약 비유**: 로봇이 엉뚱한 대답을 할 때 로봇의 머릿속(XAI)을 들여다보고, "아, 안경에 묻은 먼지(노이즈) 때문에 헛것을 보고 있구나"라고 알아차리는 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 자율주행 도로 인프라 보안 시스템 구축

- **문제 상황**: 표지판에 붙은 스티커 하나 때문에 자율주행차가 '정지'를 '직진'으로 오인할 위험 발견.
- **의사결정 (AI 방어 설계 전략)**:
  1. **Input Transformation**: 카메라 영상이 딥러닝 모델로 들어가기 전, 의도적으로 해상도를 낮추거나 노이즈를 살짝 뭉개는(Smoothing) 전처리 단계를 둔다. (미세 노이즈 무력화)
  2. **Multi-modal Defense**: 시각 정보(카메라)뿐만 아니라 물리적 형태(LiDAR) 정보를 결합하여, 시각적 착시가 발생하더라도 실제 장애물의 존재를 교차 검증한다.
  3. **SecMLOps 구축**: 주기적으로 최신 공격 툴로 우리 모델을 '셀프 공격'해보고, 뚫린 지점을 적대적 훈련으로 계속 보강하는 자동화 파이프라인을 운영한다.
  4. **Detection Logic**: 모델의 결과값에 대한 '확신 점수(Confidence Score)'가 평소보다 낮다면 즉시 비상 제동 모드로 전환한다.

### 도입 체크리스트
- **Evaluation**: 모델의 성능 측정 시 단순 정확도(Accuracy)뿐만 아니라 '적대적 정확도(Adversarial Accuracy)'를 함께 측정하는가?
- **Data Provenance**: 학습 데이터의 출처가 명확하며, 오염되지 않았음을 증명할 수 있는 이력 관리가 되는가?
- **Algorithm Freshness**: 이미 방어법이 나온 고전 공격(FGSM 등) 외에 최신 논문에 발표된 공격 기법까지 대응하고 있는가?

### 안티패턴
- **비밀 보장 기반 보안 (Obscurity)**: "우리 모델 구조는 비밀이니까 아무도 공격 못 할 거야"라는 생각. 전이성(Transferability)으로 인해 타 모델의 공격 샘플로도 얼마든지 뚫릴 수 있다.
- **방어 없는 모델 배포**: 일반적인 정확도만 보고 서비스를 오픈하는 행위. 인공지능 시대의 가장 무책임한 배포 방식이다.

- **📢 섹션 요약 비유**: 공부만 잘해서 전교 1등(정확도 우수)을 하더라도, 남의 유혹에 쉽게 넘어가는(방어력 부재) 아이는 험난한 사회(실전 운영 환경)에서 살아남기 힘들다는 점을 명심해야 합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 세부 내용 |
|:---|:---|:---|
| **정량적 (Safety)** | 오인식 사고 발생률 80% 감소 | 적대적 훈련 및 입력 정제를 통한 물리적 사고 리스크 차단 |
| **정량적 (Cost)** | 모델 재학습 비용 절감 | 초기부터 강건하게 설계하여 사고 후 전체 모델 폐기 리스크 방지 |
| **정성적 (Trust)** | 사용자 및 규제 기관 신뢰 확보 | "공격에 강한 안전한 인공지능"이라는 브랜드 이미지 구축 |
| **정성적 (Resilience)** | 시스템 생존력 강화 | 일부 오염된 데이터가 들어와도 전체 판단이 무너지지 않는 강인함 확보 |

### 미래 전망
- **Certifiable Robustness**: 모델이 어떠한 반경 내의 공격에도 100% 안전함을 수학적으로 보증하는 '증명 가능한 강건성' 기술이 상용화될 것이다.
- **Self-Defending AI**: 공격을 받으면 모델 스스로 공격 패턴을 학습하고 즉시 자신의 신경망 구조를 최적화하여 방어하는 '자율 방어 AI'가 등장할 것이다.

### 참고 표준
- **NIST AI Risk Management Framework (AI RMF)**: AI 리스크 관리의 글로벌 표준
- **ISO/IEC 27090**: AI 시스템에 대한 보안 위협 대응 지침
- **MITRE ATLAS**: AI 공격 전술 및 기법 매트릭스

- **📢 섹션 요약 비유**: 이제 인공지능은 단순히 '똑똑함'을 넘어 '성숙함'을 갖춰야 합니다. 공격 방어 기술은 인공지능이 세상의 나쁜 의도로부터 자신과 인간을 지키는 '디지털 인격의 방패'가 될 것입니다.

## 👶 어린이를 위한 3줄 비유 설명
1. 로봇 친구(AI)가 사진을 보고 공부할 때, **나쁜 사람이 사진에 아주 작은 점들을 찍어서 로봇을 헷갈리게 만드는 나쁜 장난**이 적대적 공격이에요.
2. 우리 눈에는 그냥 강아지 사진인데, 로봇은 그 점들 때문에 "이건 사자야!"라고 무서워하며 도망갈 수도 있답니다.
3. 그래서 로봇에게 "나쁜 장난이 섞여 있어도 이건 강아지란다"라고 미리 여러 번 가르쳐주면, 나중에 속지 않고 씩씩하게 우리를 지켜줄 수 있어요!
