# 73-83. XP(eXtreme Programming)와 애자일 실천법

> **핵심 인사이트**: XP는 '코딩의 극한'을 추구한다. 테스트를 먼저 짜고(TDD), 두 명이 함께 코딩하며(Pair Programming), 매 순간 코드를 다듬는(Refactoring) 행위는 단순히 속도를 높이는 게 아니라, 변경에 유연하면서도 결함이 없는 단단한 코드를 만드는 가장 강력한 도구다.

---

## Ⅰ. XP의 5대 가치
1. **의사소통 (Communication)**: 팀원 간의 긴밀한 대화.
2. **단순성 (Simplicity)**: 미래를 걱정해 미리 복잡하게 짜지 말고, 지금 필요한 것만 최소한으로 구현.
3. **피드백 (Feedback)**: 지속적인 테스트와 리뷰로 올바른 방향 확인.
4. **용기 (Courage)**: 코드를 과감히 삭제하거나 리팩토링할 수 있는 자신감.
5. **존중 (Respect)**: 동료의 코드와 인격을 존중.

---

## Ⅱ. 핵심 실천 방법 (Practices)

### 1. 테스트 주도 개발 (TDD, Test Driven Development)
* **절차**: **Red**(실패하는 테스트 작성) $\rightarrow$ **Green**(테스트 통과를 위한 코드 작성) $\rightarrow$ **Refactor**(중복 제거 및 구조 개선).
* **가치**: 설계 결함을 조기에 발견하고, 수정을 두렵지 않게 함.

### 2. 짝 프로그래밍 (Pair Programming)
* **구성**: 드라이버(실제 코딩) + 내비게이터(전략적 검토, 버그 감시).
* **효과**: 지식 공유, 코드 품질 향상, 실시간 코드 리뷰.

### 3. 리팩토링 (Refactoring)
* **정의**: 소프트웨어의 **외부 동작은 유지**하면서, 내부 구조(가독성, 유지보수성)만 개선하는 활동.

---

## Ⅲ. 사용자 스토리 (User Story)
요구사항을 사용자의 관점에서 아주 짧게 기술한 것입니다.
* **형식**: "나는 [누구]로서, [무엇]을 원한다. 왜냐하면 [이유] 때문이다."
* **INVEST 원칙**: Independent, Negotiable, Valuable, Estimable, Small, Testable.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[XP의 실시간 품질 보증 체계]

  [ TDD (테스트) ] <───> [ Pair Programming (함께 짜기) ]
          │                         │
          └──────────┬──────────────┘
                     ▼
              [ Clean Code ] ──> [ Refactoring (다듬기) ]
```

📢 **섹션 요약 비유**: **XP**는 수술실의 의사들처럼 일하는 방식입니다. **짝 프로그래밍**은 집도의(드라이버)와 보조의(내비게이터)가 함께 수술하며 실수를 막는 것이고, **TDD**는 수술 전 미리 성공적인 수술 결과를 시뮬레이션해 보는 체크리스트입니다. **리팩토링**은 수술 후 상처 부위를 흉터 없이 깔끔하게 꿰매는 마무리 작업이며, **사용자 스토리**는 환자가 의사에게 "어디가 어떻게 아픈지" 자신의 말로 설명하는 진술서와 같습니다.
