# 497-506. 입력 데이터 검증 및 암호화 적용

> **핵심 인사이트**: 시큐어 코딩의 핵심은 **"절대 신뢰하지 말기"**입니다. 모든 입력은 악의적일 수 있다고 가정하고, 화이트리스트, 패턴 매칭. 길이 제한. 타입 검증으로 걸러내고. 암호화는 데이터의 **이중 잠금장치**입니다. 전송 중(TLS). 저장 시(AES). 사용 시(메모리 보호).

---

## Ⅰ. 입력 데이터 검증 (Input Validation)

### 1. 검증 원칙
1. **화이트리스트(Whitelist)**: 허용 가능한 값만 통과
2. **블랙리스트(Blacklist)**: 위험한 값만 차단 (보안 권장 X)
3. **길이/형식/범위 제한**: 버퍼 오버플로우 방지

### 2. 검증 위치
* **프론트엔드**: UX 향상 (보안 목적 아님)
* **백엔드**: 보안 필수 (우회 가능하므로)

---

## Ⅱ. 주요 웹 공격 방어

### 1. SQL 인젝션 (SQL Injection) 방어

**공격 원리**: 사용자 입력이 SQL 쿼리의 일부로 실행됨

```
-- 취약한 코드
query = "SELECT * FROM users WHERE id = " + userInput;

-- 공격 입력
userInput = "1 OR 1=1; --"
→ "SELECT * FROM users WHERE id = 1 OR 1=1; --"
```

**방어 기법**:
1. **Prepared Statement (파라미터화된 쿼리)** - 가장 효과적
2. **ORM 사용** (Hibernate, MyBatis 등)
3. **입력값 이스케이프**
4. **최소 권한 DB 계정**

### 2. 크로스 사이트 스크립팅 (XSS) 방어

**공격 원리**: 악성 스크립트가 사용자 브라우저에서 실행됨

| 유형 | 특징 | 예시 |
|------|------|------|
| **Reflected XSS** | URL 파라미터 → 즉시 반영 | 검색 결과 표시 |
| **Stored XSS** | DB 저장 → 지속적 공격 | 게시판 글 |
| **DOM-based XSS** | 클라이언트 JS 조작 | innerHTML 사용 |

**방어 기법**:
1. **출력값 인코딩** (`<` → `&lt;`)
2. **CSP (Content Security Policy)** 헤더
3. **HttpOnly 쿠키** (JavaScript 접근 차단)
4. **입력값 검증**

### 3. 크로스 사이트 요청 위조 (CSRF) 방어

**공격 원리**: 사용자의 인증 상태를 악용한 요청 위조

**방어 기법**:
1. **Anti-CSRF 토큰** (Synchronizer Token Pattern)
2. **SameSite 쿠키 속성** (`SameSite=Strict`)
3. **Referer 헤더 검증**
4. **CAPTCHA** (사용자 확인)

## Ⅳ. 암호화 알고리즘 적용 기준

### 1. 대칭키 암호화 (AES)
* **용도**: 대량 데이터 암호화, 세션 키
* **키 길이**: AES-128 (현재 안전). AES-256 (장기 권장)
* **모드**: GCM (인증+암호화), CBC (레거시)

### 2. 비대칭키 암호화 (RSA, ECC)
* **용도**: 키 교환, 디지털 서명
* **키 길이**: RSA-2048 (현재 안전), RSA-4096 (장기)
* **ECC**: 더 짧은 키로 동등 보안 (P-256, P-384)

### 3. 일방향 해시 함수 (SHA)
* **용도**: 무결성 검증, 비밀번호 저장
* **권장**: SHA-256, SHA-3
* **비권장**: MD5, SHA-1 (충돌 취약점)

### 4. 비밀번호 저장 (KDF)
| 알고리즘 | 특징 | 메모리 하드닝 |
|----------|------|--------------|
| **PBKDF2** | 기본 표준 | X |
| **bcrypt** | 솔트 내장 | X |
| **scrypt** | 메모리 하드닝 | O |
| **Argon2** | 최신 표준 (PHC 우승) | O |

* **필수**: 솔트(Salt) 적용, 충분한 반복 횟수


📢 **섹션 요약 비유**: 입력 검증은 **공항 보안 검색**(모든 짐 검사). Prepared Statement는 **마스크**(입력이 얼굴이 아님). XSS 방어는 **방독면**(해로운 것 가려서 표시). CSRF 토큰은 **신분증**(본인 확인). 암호화는 **금고**(AES는 금고, RSA는 열쇠 보관함). Argon2는 **금고+지문 인식**(최고 수준 비밀번호 보호)입니다.
