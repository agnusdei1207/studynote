# 477-487. OWASP Top 10 (2021) 주요 취약점

> **핵심 인사이트**: OWASP Top 10은 웹 보안의 '10대 건강 검진 항목'이다. 2021년 개정에서는 **아키텍처 수준의 설계 결함**이 중요해졌으며, 특히 Broken Access Control이 1위로 올라섰다. 이는 보안이 코드 수준을 넘어 시스템 설계 전반으로 확장되었음을 시사한다.

---

## Ⅰ. OWASP Top 10 개요
OWASP(Open Web Application Security Project)가 3~4년 주기로 발표하는 웹 애플리케이션 보안 취약점 순위입니다.

| 순위 | 취약점 (2021) | 비고 |
|:---:|:---|:---|
| 1 | **Broken Access Control** | 접근 제어 실패 |
| 2 | **Cryptographic Failures** | 암호화 실패 |
| 3 | **Injection** | 인젝션 공격 |
| 4 | **Insecure Design** | 안전하지 않은 설계 |
| 5 | **Security Misconfiguration** | 보안 설정 오류 |
| 6 | **Vulnerable Components** | 취약한 컴포넌트 |
| 7 | **Auth Failures** | 인증 실패 |
| 8 | **Integrity Failures** | 무결성 실패 |
| 9 | **Logging Failures** | 로깅/모니터링 실패 |
| 10 | **SSRF** | 서버 측 요청 위조 |

---

## Ⅱ. 주요 취약점 상세

### 1. Broken Access Control (취약한 접근 제어) #478
인가되지 않은 사용자가 리소스에 접근하거나 권한을 벗어나는 행위입니다.

* **사례**: URL 조작으로 타인의 정보 조회 (`/user/123` → `/user/124`)
* **방어**: 최소 권한 원칙, 역할 기반 접근 제어(RBAC), 객체 수준 인가 검증

### 2. Cryptographic Failures (암호화 실패) #479
민감 데이터가 암호화 없이 저장/전송되어 유출되는 취약점입니다.

* **사례**: 평문 비밀번호 저장, HTTP로 카드 정보 전송
* **방어**: TLS 전송, 저장 시 암호화, 강력한 암호 알고리즘(AES-256, RSA-2048+)

### 3. Injection (인젝션) #480
신뢰할 수 없는 데이터가 명령어나 쿼리의 일부로 실행되는 공격입니다.

* **유형**: **SQL Injection**, OS Command Injection, NoSQL Injection, LDAP Injection
* **방어**: Prepared Statement, ORM 사용, 입력값 검증, 최소 권한 DB 계정

### 4. Insecure Design (안전하지 않은 설계) #481
**2021 신규**: 아키텍처와 설계 단계의 보안 결함입니다.

* **사례**: 비밀번호 찾기 질문의 정답이 유추 가능, 비즈니스 로직 취약점
* **방어**: 위협 모델링(STRIDE), 보안 사용자 스토리, 안전한 설계 패턴 적용

### 5. Security Misconfiguration (보안 설정 오류) #482
* **사례**: 기본 계정/비밀번호 미변경, 불필요한 포트 오픈, 상세 에러 메시지 노출
* **방어**: 보안 설정 템플릿, 정기 설정 점검, IaC로 설정 표준화

### 6. Vulnerable and Outdated Components (취약한 컴포넌트) #483
* **사례**: 알려진 취약점이 있는 라이브러리 사용, EoS(End of Support) 버전
* **방어**: SCA(Software Composition Analysis), 정기 업데이트, SBOM 관리

### 7. Identification and Authentication Failures (인증 실패) #484
* **사례**: 약한 비밀번호 허용, 세션 고정 공격, Credential Stuffing
* **방어**: MFA, 비밀번호 복잡성 정책, 세션 재생성, 계정 잠금 정책

### 8. Software and Data Integrity Failures (무결성 실패) #485
**2021 신규**: 코드나 데이터의 무결성 검증 없이 신뢰하는 경우입니다.

* **사례**: 서명되지 않은 업데이트, 안전하지 않은 역직렬화(Insecure Deserialization)
* **방어**: 코드 서명, 서명된 커밋, 직렬화 데이터 검증

### 9. Security Logging and Monitoring Failures (로깅 실패) #486
* **사례**: 로그인 실패 미기록, 감사 로그 미보관, 실시간 알림 부재
* **방어**: 보안 이벤트 로깅, 중앙 집중 로그 관리(SIEM), 침입 탐지 시스템(IDS)

### 10. SSRF (Server-Side Request Forgery) #487
서버가 외부 공격자의 요청을 내부 리소스에 전달하는 취약점입니다.

* **사례**: URL 입력값을 통해 내부망 스캔, 클라우드 메타데이터 탈취
* **방어**: URL 화이트리스트, 내부망 접근 차단, 응답 필터링

📢 **섹션 요약 비유**: OWASP Top 10은 **웹 애플리케이션의 10가지 '만성질환'**입니다. Injection은 **세균 감염**(외부 입력이 시스템에 침투), Broken Access Control은 **출입증 분실**(권한 없는 접근), Insecure Design은 **유전적 결함**(태생적 구조 문제)에 비유할 수 있습니다. 정기 검진(보안 스캔)과 건강한 생활 습관(Secure SDLC)이 필수적입니다.
