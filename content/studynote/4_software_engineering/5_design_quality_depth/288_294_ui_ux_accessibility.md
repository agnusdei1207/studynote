# 288-294. UI/UX 설계와 웹 접근성

> **핵심 인사이트**: 기술이 아무리 뛰어나도 사용자가 쓰기 불편하면 실패한 시스템이다. UI/UX는 미적인 화려함보다 사용자의 의도를 정확히 파악하는 '직관성'과 누구도 소외되지 않게 배려하는 '접근성'에 그 본질이 있다. 특히 웹 접근성은 법적 준수를 넘어 디지털 포용성을 실천하는 공학적 의무다.

---

## Ⅰ. UI (User Interface) vs UX (User Experience)
1. **UI**: 사용자가 시스템과 대화하는 물리적/논리적 통로. (화면 레이아웃, 버튼 디자인 등)
2. **UX**: 사용자가 시스템을 이용하며 느끼는 총체적인 경험과 감정.

---

## Ⅱ. UI 설계 원칙 및 휴리스틱
* **4대 원칙**: **직관성**(누구나 알기 쉽게), **유효성**(목표 달성), **학습성**(배우기 쉽게), **유연성**(오류 방지 및 복구).
* **닐슨-노먼 10대 휴리스틱**: 시스템 상태 가시성, 현실 세계와의 일치, 사용자 제어 및 자유, 일관성 및 표준 등.

---

## Ⅲ. 웹 접근성 (Accessibility / WCAG)
장애인이나 고령자 등 모든 사용자가 웹 사이트의 정보를 동등하게 이용할 수 있도록 보장하는 것입니다.

### 웹 콘텐츠 접근성 지침 4대 원칙 (KWCAG)
1. **인식의 용이성 (Perceivable)**: 모든 콘텐츠는 눈으로 보거나 귀로 들을 수 있어야 함. (대체 텍스트 제공)
2. **운용의 용이성 (Operable)**: 마우스 없이 키보드만으로도 모든 기능을 쓸 수 있어야 함. (광점멸 방지)
3. **이해의 용이성 (Understandable)**: 콘텐츠는 이해하기 쉬워야 하며 가독성이 좋아야 함. (기본 언어 표시)
4. **견고성 (Robust)**: 미래의 기술(최신 브라우저, 보조 기기)과도 잘 호환되어야 함. (문법 오류 방지)

---

## Ⅳ. 반응형 웹 디자인 (Responsive Web Design)
* **개념**: 단 하나의 소스로 데스크탑, 태블릿, 스마트폰 등 다양한 기기의 해상도에 맞춰 레이아웃이 유동적으로 변하는 기술. (Media Query 활용)

---

## Ⅴ. 개념 맵 및 요약

```ascii
[사용자 중심 설계의 레이어]

  ( 최외곽 )  [  Web Accessibility  ] : 모두를 위한 배려 (법적/윤리적)
      │
  ( 시각화 )  [  UI Design  ] : 아름답고 편리한 통로 (도구)
      │
  ( 내면화 )  [  UX Design  ] : 만족스러운 총체적 경험 (목표)
```

📢 **섹션 요약 비유**: **UI**는 예쁜 그릇과 식탁이고, **UX**는 그 식당에서 밥을 먹고 난 뒤 느끼는 "참 맛있고 기분 좋다"라는 경험입니다. **웹 접근성**은 휠체어를 탄 사람도 식당에 들어올 수 있게 문턱을 없애는 '경사로'이며, **반응형 웹**은 1인용 작은 상부터 10인용 대형 테이블까지 크기가 자유자재로 변하는 '요술 식탁'과 같습니다.
