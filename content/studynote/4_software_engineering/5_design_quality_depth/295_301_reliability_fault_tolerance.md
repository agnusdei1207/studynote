# 295-301. 시스템 신뢰성과 결함 허용 설계

> **핵심 인사이트**: 완벽한 시스템은 없다. 하지만 '완벽하게 대처하는' 시스템은 가능하다. 결함 허용(Fault Tolerance)은 부품이 고장 나도 전체 기능이 멈추지 않게 중복화(Redundancy)를 꾀하는 것이며, 최악의 경우에도 시스템이 안전한 상태(Fail-Safe)로 멈추게 하는 설계의 미학이다.

---

## Ⅰ. 시스템 신뢰성 모델링
시스템의 구성 요소가 어떻게 연결되어 있느냐에 따라 전체 신뢰도가 결정됩니다.

1. **직렬 모델 (Series)**: 하나의 요소만 고장 나도 전체 시스템이 중단됨. ($R = R_1 \times R_2$)
2. **병렬 모델 (Parallel)**: 모든 요소가 고장 나야만 시스템이 중단됨. ($R = 1 - (1-R_1)(1-R_2)$) ➔ **중복화**를 통해 신뢰도 향상.

---

## Ⅱ. 결함 허용 (Fault Tolerance) 기술
장애가 발생해도 시스템이 정상적인 서비스나 기능을 계속 유지할 수 있게 하는 설계 방식입니다.

* **구현 기법**:
  - **N-Version Programming**: 동일한 요구사항을 서로 다른 팀이 서로 다른 언어로 구현하여 투표(Voting)로 결과 결정.
  - **Recovery Block**: 실행 결과를 검증하여 오류 시 이전 상태로 복구 후 예비 블록 실행.

---

## Ⅲ. 장애 처리 3대 원칙 (Fail 기술)

1. **Fail-Safe (페일 세이프)**: 결함 발생 시 시스템을 무조건 **가장 안전한 상태**로 만드는 것. (예: 철도 건널목 차단기 하락)
2. **Fail-Soft (페일 소프트)**: 결함 발생 시 중요도가 낮은 기능은 포기하되, **핵심 기능은 유지**하는 것. (기능 저하 운전)
3. **Fail-Over (페일 오버)**: 운영 중인 시스템 장애 시, **예비 시스템으로 즉시 자동 전환**하는 것. (이중화 클러스터)

---

## Ⅳ. 결함 회피 (Fault Avoidance)
* 결함이 발생하지 않도록 개발 과정에서 엄격한 관리와 테스트를 수행하는 사전 예방 활동입니다. (클린룸 공학 등과 연계)

---

## Ⅴ. 개념 맵 및 요약

```ascii
[장애 대응 전략의 차이]

  [ 정상 작동 ] ───( 장애 발생 )───┐
                                   │
  ┌────────────────┬───────────────┼───────────────┐
  ▼                ▼               ▼               ▼
[ Fail-Safe ]    [ Fail-Soft ]   [ Fail-Over ]   [ Fail-Passive ]
(안전 중시)       (기능 저하)      (중단 없음)      (동작 멈춤)
```

📢 **섹션 요약 비유**: **직렬 모델**은 한 알만 빠져도 끊어지는 진주 목걸이이고, **병렬 모델**은 전구가 하나 나가도 거실이 어둡지 않은 샹들리에와 같습니다. **페일 세이프**는 엘리베이터 줄이 끊겼을 때 즉시 브레이크가 걸려 멈추는 생명 장치이고, **페일 소프트**는 엔진 하나가 고장 나도 남은 엔진으로 기어가는 비행기이며, **페일 오버**는 주전자가 타버렸을 때 미리 준비해둔 새 주전자로 바로 물을 끓이는 주부의 지혜입니다.
