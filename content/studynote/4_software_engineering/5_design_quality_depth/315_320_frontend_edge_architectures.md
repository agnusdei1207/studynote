# 315-320. 현대 프론트엔드와 엣지 아키텍처

> **핵심 인사이트**: 사용자에게 더 빠른 화면을 보여주기 위한 렌더링 전쟁이 치열하다. 서버에서 다 그려서 보내주거나(SSR), 브라우저가 직접 그리거나(CSR), 혹은 앱처럼 설치해서 오프라인에서도 돌아가게 하는(PWA) 등 목적에 맞는 아키텍처 선택이 사용자 경험(UX)의 질을 결정한다.

---

## Ⅰ. 렌더링 방식의 비교

1. **SSR (Server Side Rendering)**: 
   - 서버에서 HTML을 완성하여 브라우저에 전달.
   - **장점**: 초기 로딩 속도 빠름, 검색 엔진 최적화(SEO) 유리.
2. **CSR (Client Side Rendering)**:
   - 서버는 빈 HTML과 데이터만 주고, 브라우저의 JavaScript가 화면을 그림.
   - **장점**: 페이지 이동 시 부드러운 전환, 서버 부하 감소.

---

## Ⅱ. 현대 웹 애플리케이션 구조

1. **SPA (Single Page Application)**: 페이지를 새로고침 하지 않고 필요한 부분만 동적으로 갈아 끼우는 방식. (React, Vue 등)
2. **PWA (Progressive Web App)**: 웹이지만 앱처럼 푸시 알림, 오프라인 모드, 홈 화면 설치 기능을 제공하는 기술. (Service Worker 활용)
3. **마이크로 프론트엔드**: 거대한 프론트엔드 화면을 여러 팀이 개발한 독립적인 컴포넌트들의 집합으로 구성하는 MSA의 확장판.

---

## Ⅲ. 성능 가속 기술

* **WebAssembly (Wasm)**: 웹 브라우저에서 실행 가능한 저수준 바이너리 포맷. C, C++, Rust 코드를 웹에서 실행하여 JavaScript보다 훨씬 빠른 성능을 냄.
* **엣지 컴퓨팅 아키텍처**: 클라우드가 아닌 사용자 근처(Edge)의 CDN 노드에서 로직을 처리하여 지연 시간을 극도로 줄이는 설계 방식.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[렌더링 위치에 따른 차이]

  ( Server ) ─────( 완성된 HTML )────> ( Browser )  [ SSR ]
  ( Server ) ─────( JS + Data )─────> ( Browser )  [ CSR ]
```

📢 **섹션 요약 비유**: **SSR**은 주방에서 요리를 다 해서 접시에 담아 내어주는 '전통적인 식당'입니다. 손님은 바로 먹기만 하면 되죠. **CSR**은 테이블마다 버너가 있어서 손님이 직접 구워 먹는 '셀프 바비큐'와 같습니다. **PWA**는 식당 음식을 도시락으로 싸서 집(로컬)에서도 먹을 수 있게 한 포장 서비스이며, **WebAssembly**는 주방에 '최신형 고속 믹서기'를 들여놓아 요리 속도를 비약적으로 높인 것과 같습니다.
