# 302-314. 마이크로서비스 설계 패턴 (Circuit Breaker, Saga, BFF)

> **핵심 인사이트**: 마이크로서비스는 '분산의 고통'을 수반한다. 서비스 하나가 죽었을 때 전체가 마비되는 것을 막으려면 전기 차단기(Circuit Breaker)를 달아야 하고, 흩어진 DB 사이의 트랜잭션은 보상 트랜잭션(Saga)으로 해결해야 한다. MSA는 단순히 나누는 것이 아니라, 나뉘어 있어도 하나처럼 동작하게 하는 정교한 패턴들의 집합이다.

---

## Ⅰ. 통신 및 안정성 패턴

1. **API 게이트웨이**: 모든 클라이언트 요청을 한곳에서 받아 인증, 라우팅, 부하 분산을 수행하는 단일 창구.
2. **서킷 브레이커 (Circuit Breaker)**: 특정 서비스의 장애가 감지되면 해당 서비스로의 호출을 즉시 차단(Open)하여 **장애 전파를 방지**함. 정상화되면 다시 연결(Close).
3. **벌크헤드 (Bulkhead)**: 선박의 격벽처럼, 서비스별로 가용 자원(스레드 풀 등)을 격리하여 한쪽의 과부하가 다른 쪽에 영향을 주지 않게 함.

---

## Ⅱ. 데이터 일관성 패턴

1. **Saga 패턴**: 분산 DB 환경에서 각 서비스의 로컬 트랜잭션을 순차적으로 실행. 중간에 실패하면 **보상 트랜잭션**을 실행하여 이전 상태로 원복.
   - **Choreography**: 이벤트 기반으로 자율적으로 연쇄 반응.
   - **Orchestration**: 중앙 조정자가 전체 흐름을 지휘.
2. **트랜잭셔널 아웃박스**: DB 업데이트와 메시지 발행을 하나의 원자적 트랜잭션으로 묶어 데이터 유실 방지.

---

## Ⅲ. 구조 및 인터페이스 패턴

1. **BFF (Backend For Frontend)**: 모바일, 웹 등 각 프론트엔드 기기에 최적화된 전용 백엔드 API를 별도로 두는 패턴.
2. **스트랭글러 피그 (Strangler Fig)**: 거대 레거시 시스템을 한꺼번에 MSA로 바꾸지 않고, 기능별로 조금씩 떼어내어 새 시스템으로 교체해 나가는 점진적 전환 전략.
3. **서비스 디스커버리**: 동적으로 변하는 서비스들의 IP와 포트 위치를 자동으로 찾아주는 기술.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[MSA 장애 격리 메커니즘]

  [ Client ] ──> [ API Gateway ] ──> [ Circuit Breaker ] ──> [ Microservice ]
                                            │ (장애 발생!)
                                            ▼
                                     [ Fallback 응답 ] (차단 및 우회)
```

📢 **섹션 요약 비유**: **API 게이트웨이**는 아파트 단지의 통합 보안실입니다. **서킷 브레이커**는 과부하 시 전기가 차단되는 두꺼비집이고, **벌크헤드**는 배 밑바닥에 물이 샜을 때 배 전체가 침몰하지 않게 막아둔 격벽입니다. **Saga 패턴**은 여행 패키지 예약 중 비행기표는 샀는데 호텔이 꽉 찼을 때, 이미 산 비행기표를 위약금 없이 환불(보상 트랜잭션)해 주는 예약 시스템과 같습니다. **BFF**는 똑같은 영화를 극장판, TV판으로 따로 편집해서 보여주는 맞춤 서비스입니다.
