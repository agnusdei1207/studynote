# 281-287. 아키텍처 품질 전술 (Quality Tactics)

> **핵심 인사이트**: 품질은 우연히 만들어지지 않는다. 가용성을 위해 결함을 감시하고(Heartbeat), 성능을 위해 자원을 분산하며(Load Balancing), 보안을 위해 매 순간 검증(Authentication)하는 구체적인 '설계 기술(Tactic)'들이 아키텍처를 단단하게 만든다.

---

## Ⅰ. 가용성 (Availability) 전술
시스템이 장애 없이 서비스를 지속적으로 제공하게 하는 기술입니다.

* **결함 탐지**: **Heartbeat**(주기적 신호 확인), **Ping/Echo**, 모니터링.
* **결함 복구**: **Redundancy**(중복화), **Check-point**(복구 지점 저장), **Rollback**.
* **결함 예방**: 예방적 유지보수, 트랜잭션 관리.

---

## Ⅱ. 성능 (Performance) 전술
사용자의 요청에 대해 정해진 시간 내에 응답하게 하는 기술입니다.

* **자원 요구 관리**: 요청 빈도 제한, 샘플링, 대기 큐 관리.
* **자원 관리**: **Concurrency**(병행 처리), 복제(Replication), 데이터 압축.
* **스케줄링**: 우선순위 기반 처리, FIFO, Round-Robin.

---

## Ⅲ. 보안성 (Security) 전술
비인가된 접근을 막고 데이터의 무결성을 유지하는 기술입니다.

* **공격 저항**: 인증(Who?), 인가(What?), 데이터 암호화.
* **공격 탐지**: 침입 탐지 시스템(IDS), 로그 감사.
* **공격 복구**: 시스템 복원, 데이터 백업.

---

## Ⅳ. 유지보수성 (Modifiability) 전술
시스템 변경 시 발생하는 비용과 시간을 최소화하는 기술입니다.

* **국소화 (Localize)**: 응집도 높이기, 전역 변수 제한.
* **파급 효과 방지**: **Encapsulation**(캡슐화), 인터페이스 분리.
* **의존성 지연**: 런타임 바인딩, 설정 파일 활용.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[품질별 대표 전술 매칭]

  [ 가용성 ] ──> Heartbeat, Redundancy (장애 극복)
  [ 성  능 ] ──> Caching, Load Balancing (속도 향상)
  [ 보안성 ] ──> Encryption, IDS (위협 방어)
  [ 유지보수] ──> Abstraction, Modularity (변경 용이)
```

📢 **섹션 요약 비유**: **가용성 전술**은 정전에 대비해 '비상 발전기'를 두는 것이고, **성능 전술**은 손님이 몰릴 때 '계산대 직원'을 늘리는 것입니다. **보안성 전술**은 현관문에 '지문 인식기'를 다는 것이며, **유지보수성 전술**은 나중에 수리하기 편하게 전선마다 '라벨'을 붙여두는 지혜와 같습니다.
