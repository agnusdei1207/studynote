# 660. 번다운 차트 작업 진척도

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스프린트 기간 동안 남은 작업량(Remaining Work)과 시간을 시각화하여, 팀이 정해진 기한 내에 목표를 달성할 수 있을지를 예측하는 **애자일 가시화 도구**이다.
> 2. **추세 분석**: 이상적인 진행선과 실제 진행선을 비교함으로써, 팀의 작업 속도가 너무 빠른지(Under-commitment) 혹은 느린지(Over-commitment)를 실시간으로 파악한다.
> 3. **가치**: 장애물(Blockers)이나 범위 변경(Scope Creep)을 조기에 식별하여 데일리 스크럼에서 즉각적인 대응을 할 수 있는 데이터 기반의 근거를 제공한다.

---

## Ⅰ. 개요 (Context & Background)

### 번다운 차트(Burndown Chart)의 목적

전통적인 방식의 '진척률(%)' 보고는 "90% 완료 증후군(나머지 10%를 하는 데 90%의 시간이 걸리는 현상)"에 빠지기 쉽습니다. 번다운 차트는 '한 일'이 아니라 **'남은 일'**에 집중합니다. 이를 통해 팀은 "우리 진짜 다 할 수 있어?"라는 질문에 정직하게 답할 수 있게 됩니다.

### 💡 비유: 마라톤의 남은 거리 표시판

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        번다운 차트 비유                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 42.195km 마라톤(스프린트)을 뛰고 있음.                                  │
│                                                                             │
│  1. 이상적인 선 (가이드):                                                     │
│     일정한 속도로 뛰었을 때 매 km마다 통과해야 하는 시간 지점.                   │
│                                                                             │
│  2. 실제 진행 선:                                                            │
│     내가 지금 실제로 남겨둔 거리.                                              │
│     - 실제 선이 가이드보다 위라면? 페이스가 늦음! (더 빨리 뛰어야 함)             │
│     - 실제 선이 가이드보다 아래라면? 페이스가 좋음! (여유가 있음)                 │
│                                                                             │
│  → 결승선(스프린트 종료)에 도착했을 때 남은 거리가 0이 되는 것이 목표!            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 차트의 구성 요소
- **X축 (Time)**: 스프린트 기간 (일 단위).
- **Y축 (Work)**: 남은 작업량 (스토리 포인트 또는 시간).
- **이상적 진행선 (Ideal Line)**: 첫날의 총 작업량에서 마지막 날 0이 되도록 그은 직선.
- **실제 진행선 (Actual Line)**: 매일 작업 완료 후 업데이트되는 실제 남은 양의 궤적.

### 2. 번다운 차트 해석 패턴 (Pattern Analysis)

| 패턴 모양 | 의미 분석 | 대응 방안 |
|:---:|:---|:---|
| **계단형** | 작업이 한꺼번에 완료됨 (대형 태스크 위주) | 작업을 더 작게 쪼개야 함 (Granularity) |
| **위로 솟구침** | 작업 도중 새로운 요구사항 추가 (Scope Creep) | PO와 상의하여 우선순위 조정 필요 |
| **수평 지속** | 작업이 진행되지 않음 (장애물 발생) | 데일리 스크럼에서 장애물 제거(SM의 역할) |
| **급격한 하강** | 작업량을 너무 적게 잡음 | 다음 스프린트에서 계획 포인트 상향 |

### 3. 차트 시각화 (ASCII)

```text
남은량(Point)
  ^
  │● [시작]
  │ \
  │  \  ● [실제: 약간 지연 중]
  │───\───●────────────────────────── 이상적 진행선 (Ideal)
  │    \ /  \
  │     ●    \  ● [실제: 속도 회복]
  │           \ /
  │            ● ──▶ 0 [목표 달성!]
  └──────────────────────────────────────────> 시간(Days)
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 번다운(Burndown) vs 번업(Burnup) 차트
- **번다운(Burndown)**: 이번 스프린트의 **마감 기한 준수** 여부 확인에 최적화.
- **번업(Burnup)**: 프로젝트 전체 기간 동안 **범위의 변화(Scope)**와 **완료량**을 동시에 확인하는 데 유리.

### 2. 기술적 시너지: 속도 (Velocity)
여러 스프린트의 번다운 차트 데이터를 모으면 팀의 평균 '속도(Velocity)'가 산출됩니다. 이 속도는 다음 스프린트 계획 회의에서 "우리는 이번에 몇 점 어치의 일을 가져올 수 있는가?"를 결정하는 과학적 근거가 됩니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 스프린트 중반 위기 감지
- **상황**: 스프린트 기간의 50%가 지났는데, 번다운 차트의 실제 선이 이상적인 선보다 훨씬 위에 머물러 있음.
- **결단**: **조기 대응**.
    1. 데일리 스크럼에서 모든 팀원이 차트를 확인하며 원인 파악.
    2. 기술적 난관이 발견되면 즉시 페어 프로그래밍으로 전환.
    3. 도저히 불가능하다고 판단되면 PO에게 보고하여 낮은 우선순위의 백로그 아이템을 다음 스프린트로 이월.
- **판단**: 마지막 날에 "못 했습니다"라고 고백하는 대신, 중반에 미리 조정하여 **'가장 중요한 목표'**만큼은 반드시 완수.

### 📢 기술사적 결언
> "번다운 차트는 팀을 감시하는 'CCTV'가 아니라, 팀의 목적지를 알려주는 **'GPS'**다. 차트가 예쁘지 않다고 해서(수평이거나 솟구친다고 해서) 팀원을 탓해서는 안 된다. 오히려 정직한 차트는 팀이 처한 현실을 투명하게 드러내어, 프로세스를 개선할 수 있는 가장 소중한 힌트를 제공한다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **목표 달성률 향상**: 가시적인 압박감과 목표 의식 공유로 인한 몰입도 증대.
- **리스크 조기 발견**: 문제 발생 시점과 해결 시점 사이의 간격 최소화.

### 미래 전망
최근에는 **'예측형 번다운 차트'**가 도입되고 있습니다. 머신러닝이 팀의 과거 수행 이력과 현재 작업의 복잡도를 분석하여, 현재 추세대로라면 며칠에 작업이 끝날지(Probabilistic Forecasting) 여러 개의 시나리오를 시뮬레이션해 주는 지능형 대시보드로 진화하고 있습니다.
