# 633. 페어와이즈 (Pairwise) 직교 배열 (Orthogonal Array)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 대부분의 결함이 2개 요소의 상호작용에서 발생한다는 점에 착안하여, 모든 변수 쌍(Pair)의 조합을 최소 한 번씩 테스트하는 **조합 최적화 테스트 기법**이다.
> 2. **가치**: 가능한 모든 조합(Full Cartesian Product)을 테스트할 때 발생하는 **조합의 폭발(Combinatorial Explosion)**을 막고, 최소한의 테스트 케이스로 높은 결함 발견율을 보장한다.
> 3. **도구**: 수학적 모델인 **직교 배열(Orthogonal Array)**을 활용하거나 PICT(Pairwise Independent Combinatorial Testing)와 같은 자동화 도구를 사용하여 테스트 스위트를 구성한다.

---

## Ⅰ. 개요 (Context & Background)

### 등장 배경: 조합의 폭발 문제

소프트웨어의 입력값이 다양해짐에 따라 모든 경우의 수를 테스트하는 것은 물리적으로 불가능해졌습니다. 예를 들어, 10개의 옵션이 있고 각각 3개의 선택지가 있다면 총 $3^{10} = 59,049$개의 조합이 생깁니다. 하지만 실제 버그의 60~90%는 단일 변수 혹은 2개 변수의 조합에서 발생한다는 연구 결과가 있습니다. **페어와이즈(Pairwise)**는 이 지점을 공략하여 효율성을 극대화합니다.

### 💡 비유: 뷔페 음식 조합 테스트

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                       페어와이즈 뷔페 비유                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 햄버거, 콜라, 감자튀김의 모든 조합이 배탈을 일으키는지 확인해야 함.       │
│                                                                             │
│  1. 모든 조합(Full):                                                        │
│     (치즈/불고기) x (콜라/사이다) x (소금/어니언) = 8가지 조합                 │
│                                                                             │
│  2. 페어와이즈(Pairwise):                                                   │
│     "치즈버거와 콜라", "불고기버거와 사이다", "콜라와 어니언감자" 등            │
│     모든 메뉴 '쌍'이 한 번은 만나도록 조합을 짬.                             │
│     → 4~5가지 조합만으로도 웬만한 배탈 원인(음식 궁합 문제)을 다 찾아냄!         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 직교 배열 (Orthogonal Array, OA)

수학적 통계 원리를 이용하여 행(Row)과 열(Column)을 배치한 표입니다. 임의의 두 열을 선택했을 때 나타날 수 있는 모든 값의 조합이 동일한 횟수로 나타나도록 설계됩니다.

- **표기법**: $L_N(S^K)$ (N: 행의 수, S: 수준/선택지 수, K: 요인/변수 수)
- **예시**: $L_4(2^3)$ → 4개 테스트 케이스로 3개 변수(2개 선택지)의 모든 쌍을 커범함.

### 2. 페어와이즈 알고리즘 작동 방식

변수 A, B, C가 있을 때:
1. (A, B), (B, C), (A, C)의 모든 가능한 쌍의 리스트를 생성합니다.
2. 각 테스트 케이스를 생성할 때, 아직 테스트되지 않은 쌍을 가장 많이 포함하도록 변수 값을 선택합니다.
3. 모든 쌍이 하나 이상의 테스트 케이스에 포함될 때까지 반복합니다.

### 3. 비교 분석: Full vs Pairwise

| 비교 항목 | 모든 조합 (Full Combination) | 페어와이즈 (Pairwise) |
|:---:|:---|:---|
| **테스트 케이스 수** | 지수함수적 증가 ($S^K$) | 로그/선형적 증가 |
| **결함 발견율** | 100% (이론상) | 60~90% (경험적 효율성) |
| **복잡도** | 매우 높음 (수행 불가 수준) | 낮음 (도구 활용 시 매우 빠름) |
| **적합한 상황** | 안전 최우선 시스템 (항공, 의료) | 일반 상용 소프트웨어, 웹/앱 |

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 테스트 기법 간 시너지
- **경계값 분석 + 페어와이즈**: 각 변수의 대푯값으로 경계값(Min, Max, Null 등)을 선정하고, 이 값들을 페어와이즈로 조합하면 결함 발견율이 극대화됩니다.
- **상태 전이 테스트 + 페어와이즈**: 다양한 상태 변화 조건들을 페어와이즈로 조합하여 복잡한 시나리오를 구성합니다.

### 2. 하드웨어/네트워크 융합
- **호환성 테스트**: OS(Windows, Mac, Linux) x 브라우저(Chrome, Safari, Edge) x 해상도 x 네트워크 환경 등의 방대한 조합을 테스트할 때 필수적으로 사용됩니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오: 프린터 드라이버 설정 테스트
- **요인**: 용지 크기(3), 양면 인쇄(2), 컬러 설정(2), 해상도(3)
- **모든 조합**: $3 \times 2 \times 2 \times 3 = 36$회
- **페어와이즈 적용**: 약 **9~12회**의 테스트로 단축 가능.

### 📢 기술사적 결언
> "페어와이즈는 **'완벽함'보다 '경제성'과 '효과성'**을 선택한 지혜로운 기법이다. 자원이 한정된 프로젝트에서 조합의 저주를 푸는 가장 강력한 무기이며, 특히 설정(Configuration)이나 설치(Installation) 환경 테스트에서 그 진가를 발휘한다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **테스트 케이스 감축**: 변수가 많을수록 감축률은 기하급수적으로 커짐 (최대 95% 이상 감축).
- **비용 절감**: 테스트 수행 시간 및 인건비의 획기적 절감.

### 미래 전망
최근에는 **N-way 테스팅**으로 확장되어, 2개가 아닌 3개, 4개 변수 간의 상호작용까지 고려하는 도구들이 발전하고 있습니다. 또한 AI가 결함 이력을 학습하여, 과거에 문제가 자주 발생했던 변수 쌍 위주로 가중치를 두어 조합을 생성하는 '지능형 페어와이즈' 기술이 연구되고 있습니다.
