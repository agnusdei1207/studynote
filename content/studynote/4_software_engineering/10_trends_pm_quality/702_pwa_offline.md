# 702. PWA (Progressive Web App) 오프라인 워커

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 웹 브라우저의 백그라운드 스레드에서 실행되는 **서비스 워커(Service Worker)**를 핵심 엔진으로 삼아, 오프라인 환경에서도 네이티브 앱과 유사한 경험을 제공하는 **차세대 웹 애플리케이션**이다.
> 2. **오프라인 전략**: 네트워크 요청을 가로채어 미리 저장된 캐시(Cache) 응답을 반환하는 **프록시(Proxy) 역할**을 수행하며, 백그라운드 동기화 및 푸시 알림 기능을 지원한다.
> 3. **가치**: 네트워크 연결이 불안정한 환경에서도 서비스 가용성을 보장하고, 별도의 스토어 설치 없이 URL만으로 네이티브 앱의 강력한 기능들을 구현하여 사용자 전환율을 높인다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "웹의 한계를 넘어서는 앱 경험"

전통적인 웹 페이지는 인터넷이 끊기면 "연결되지 않음"이라는 하얀 화면만 보여주었습니다. 또한 주소창이 항상 표시되어 앱다운 몰입감을 주기 어려웠습니다. **PWA (Progressive Web App)**는 이러한 웹의 단점을 보완하기 위해 탄생했습니다. 특히 **서비스 워커**라는 기술은 웹 페이지가 꺼져 있어도 뒤에서 돌아가며 데이터를 미리 챙겨두는 '똑똑한 매니저' 역할을 수행합니다.

### 💡 비유: 단골 가게의 비밀 창고

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        PWA 서비스 워커 비유                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 일반 웹 (일반 식당):                                                     │
│     비가 와서 배달 트럭(인터넷)이 못 오면 요리를 못 함. 손님은 굶어야 함.           │
│                                                                             │
│  2. PWA (단골 가게):                                                        │
│     가게 뒤에 **비밀 창고(Service Worker Cache)**가 있음.                       │
│     평소에 트럭이 올 때마다 재료를 미리 넉넉히 쟁여둠.                            │
│                                                                             │
│  3. 오프라인 상황:                                                           │
│     폭풍우로 트럭이 끊겨도, 점원(서비스 워커)이 창고에서 재료를 꺼내 요리를 함.     │
│     손님은 트럭이 끊긴 줄도 모르고 맛있게 식사를 할 수 있음!                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 서비스 워커(Service Worker)의 3대 역할
- **네트워크 프록시**: 모든 HTTP 요청을 가로채서 캐시에서 줄지 서버에서 가져올지 결정.
- **오프라인 지원**: 로컬에 저장된 `Cache Storage`를 활용하여 네트워크 단절 시에도 리소스 응답.
- **백그라운드 처리**: 웹 페이지가 닫혀 있어도 푸시 알림 수신 및 데이터 동기화 수행.

### 2. 서비스 워커 캐싱 전략 (Caching Strategies)

| 전략 명칭 | 설명 | 적합한 리소스 |
|:---:|:---|:---|
| **Cache First** | 무조건 캐시 먼저 확인, 없으면 네트워크 요청 | 폰트, 로고, 이미지 등 정적 자산 |
| **Network First** | 네트워크 요청 시도 후 실패 시 캐시 반환 | 최신 뉴스, 실시간 주가 등 |
| **Stale-While-Revalidate** | 캐시를 즉시 보여주되, 뒤에서 조용히 업데이트 | 메인 피드, 프로필 정보 |

### 3. PWA 구동 아키텍처 (ASCII)

```text
    [ User Interface ] ──▶ [ Fetch Event ] ──▶ [ Service Worker ]
                                                    │
    ┌───────────────────────────────────────────────┼────────────────────────┐
    │ (Offline Mode)                                │ (Online Mode)          │
    ▼                                               ▼                        │
    [ Cache Storage ] ◀───(Update)─── [ Remote Server (API) ] <───┘
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. PWA vs 네이티브 앱

| 항목 | PWA | 네이티브 앱 (iOS/Android) |
|:---:|:---|:---|
| **설치 방식** | 브라우저 내 '홈 화면 추가' | 앱 스토어 다운로드 |
| **업데이트** | 실시간 (웹 배포 시 즉시) | 스토어 심사 및 재설치 필요 |
| **하드웨어 접근** | 제한적 (점차 확대 중) | **완벽 지원 (GPS, 센서 등)** |
| **개발 비용** | **매우 저렴 (Single Codebase)** | 높음 (OS별 별도 개발) |

### 2. 기술적 시너지: Web App Manifest
`.json` 설정 파일을 통해 앱의 이름, 아이콘, 시작 URL, 배경색 등을 정의합니다. 이를 통해 웹 사이트가 브라우저의 주소창 없이 **'Full Screen'**으로 실행되도록 하여 사용자가 네이티브 앱과 구별할 수 없는 경험을 제공합니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 저사양 기기용 이커머스 앱
- **상황**: 동남아시아 시장 진출을 위해 용량이 작고 네트워크가 느려도 잘 돌아가는 앱이 필요.
- **결단**: **PWA 아키텍처 채택**. 
- **판단**: 서비스 워커를 활용하여 '장바구니'와 '상품 목록'을 선제적 캐싱(Pre-cache). 오프라인에서 주문을 넣으면 서비스 워커가 보관했다가, 인터넷이 연결되는 순간 서버로 전송(Background Sync).
- **효과**: 네이티브 앱 대비 용량 90% 절감, 사용자 이탈률 40% 개선.

### 📢 기술사적 결언
> "PWA는 기술적 진보보다 **'비즈니스 접근성'**에 초점을 맞춘 승부수다. 설치라는 거대한 장벽을 없애고, 서비스 워커라는 강력한 엔진을 통해 웹의 최대 약점인 '네트워크 의존성'을 극복했다. 이제 아키텍트는 하드웨어 성능을 극한으로 쓰는 앱이 아니라면, **'웹의 생산성'**과 **'앱의 지속성'**을 결합한 PWA를 최우선 전략으로 고려해야 한다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **데이터 사용량**: 지능형 캐싱으로 중복 로드 방지, 트래픽 비용 50% 이상 절감.
- **설치율**: 스토어 방문 과정 생략으로 앱 설치 전환율 최대 3배 향상.

### 미래 전망
미래의 PWA는 **'Project Fugu'**를 통해 파일 시스템 접근, 블루투스 제어, 연락처 연동 등 네이티브의 영역을 99% 흡수할 것입니다. 또한 **데스크톱 PWA** 시장이 커지면서 Windows나 macOS 앱조차 PWA로 대체되는 '대통합의 시대'가 올 것입니다. 서비스 워커는 단순 캐싱을 넘어 웹 환경에서의 '마이크로 OS' 역할을 수행하게 될 것입니다.
