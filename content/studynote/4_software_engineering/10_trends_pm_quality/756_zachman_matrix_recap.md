# 756. 잭맨 프레임워크 6x6 매트릭스 (Recap)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기업의 정보 시스템 아키텍처를 정의하기 위해 6가지 관점(Perspective)과 6가지 기술 항목(Abstraction)을 교차시킨 **엔터프라이즈 아키텍처(EA)의 논리적 분류 표준**이다.
> 2. **전체론적 접근**: '데이터, 기능, 네트워크, 사람, 시간, 동기'를 경영자부터 실무자까지 각기 다른 수준에서 조망함으로써 시스템의 누락 없는 식별과 통합을 보장한다.
> 3. **가치**: 특정 기술에 종속되지 않는 고전적이면서도 포괄적인 분류 체계를 제공하여, 복잡한 전사 자산 관리의 기초 데이터를 형성하고 의사소통의 기준점을 제시한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "아키텍처의 주기율표"

존 잭맨이 제안한 이 프레임워크는 단순히 시스템을 나열하는 것이 아니라, 기업이라는 유기체를 구성하는 모든 요소를 36개의 셀(Cell)로 분류합니다. 이는 화학의 주기율표가 원소를 분류하듯, 기업의 무형 자산(비전, 프로세스)과 유형 자산(데이터, 네트워크)을 체계적으로 기술하게 해 줍니다.

### 💡 비유: 거대 도시의 도시 계획 매트릭스

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        잭맨 프레임워크 분류 비유                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [관점: 세로축]                                                               │
│  - 시장(Planner): 도시 전체의 구상.                                           │
│  - 건축주(Owner): 살기 편한 집의 구조.                                         │
│  - 설계사(Designer): 수도와 전기가 흐르는 논리적 도면.                          │
│  - 시공사(Builder): 콘크리트와 철근의 물리적 배치.                             │
│                                                                             │
│  [질문: 가로축]                                                               │
│  - What(자재), How(방법), Where(장소), Who(시민), When(일정), Why(이유).       │
│                                                                             │
│  → 이 모든 것을 한 장의 표로 정리하면, **도시의 과거, 현재, 미래**를 한눈에      │
│     알 수 있는 완벽한 지도가 됨!                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 잭맨 매트릭스의 6대 관점 (Rows)
1. **Planner (경영자)**: 사업의 목적과 범위 정의.
2. **Owner (소유자)**: 비즈니스 개념 모델링.
3. **Designer (설계자)**: 시스템 논리 모델링.
4. **Builder (구축자)**: 기술 물리 모델링.
5. **Sub-contractor (협력업체)**: 구성 요소 상세 명세.
6. **Functional System (운영)**: 실제 가동 중인 시스템.

### 2. 6대 질문 항목 (Columns)
- **Data (What)**: 시스템이 관리하는 정보.
- **Function (How)**: 시스템이 수행하는 작업.
- **Network (Where)**: 하드웨어가 위치한 장소.
- **People (Who)**: 시스템을 사용하는 사람과 조직.
- **Time (When)**: 사건이 발생하는 시점과 순서.
- **Motivation (Why)**: 비즈니스 전략과 규칙.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 잭맨 vs TOGAF vs FEAF
- **Zachman**: 분류 체계 중심 (Classification). "무엇이 있는가?"
- **TOGAF**: 실행 프로세스 중심 (Process). "어떻게 구축할 것인가?"
- **FEAF**: 미 연방정부 표준. 성과 참조 모델(PRM) 등 참조 모델 강조.

### 2. 현대적 시너지: 데이터 거버넌스
잭맨 매트릭스의 'What' 열은 현대 기업의 **데이터 계보(Data Lineage)** 관리의 시초가 되었습니다. 비즈니스 용어(BA)에서 실제 DB 테이블(TA)까지 데이터가 어떻게 흘러가는지 추적하는 기반을 제공합니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 아키텍처 결손 지점 파악
- **상황**: 프로젝트 감리 중, "데이터 보안 체계가 미흡하다"는 지적을 받음.
- **결단**: **잭맨 매트릭스 맵핑 수행**. 
- **판단**: 'Who(사람)'와 'What(데이터)'의 교차점(Row 4, 5)에 해당하는 '데이터 접근 권한 명세'가 비어있음을 식별.
- **효과**: 누락된 설계 영역을 즉시 보강하여 시스템 보안 구멍 차단.

### 📢 기술사적 결언
> "잭맨 프레임워크는 EA의 **'기준 원점'**이다. 너무 오래된 이론이라 치부하기엔 그 논리적 완결성이 매우 높다. 아키텍트는 복잡한 신기술을 도입하기 전에, 잭맨의 36개 셀 중 우리가 놓치고 있는 **'사각지대'**가 어디인지 먼저 점검해야 한다. 채워지지 않은 칸은 곧 미래의 장애 포인트나 보안 사고의 발원지가 될 수 있음을 잊지 말아야 한다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **설계 품질**: 전사 자산 식별의 포괄성 100% 달성.
- **소통 효율**: 계층 간 용어 불일치 및 관점 차이 50% 이상 해소.

### 미래 전망
미래에는 **'AI-Generated Zachman Matrix'**가 등장할 것입니다. 기업의 산출물과 로그를 AI가 상시 분석하여 실시간으로 잭맨 매트릭스를 업데이트하고, 비즈니스 가치(Why)와 기술 구현(How) 사이의 괴리를 자동으로 경고해주는 'Living EA' 체계로 진화할 것입니다.
