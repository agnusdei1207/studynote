# 762. 애자일 에픽, 스토리, 테마, 태스크 계층

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 크고 모호한 비즈니스 요구사항을 실행 가능하고 측정 가능한 작은 단위로 쪼개어 관리하기 위한 **애자일 프로젝트의 작업 분류 체계(Work Hierarchy)**이다.
> 2. **계층 구조**: 공통의 비즈니스 목적을 묶은 **테마(Theme)**, 여러 스프린트에 걸친 거대 기능인 **에픽(Epic)**, 사용자 가치 단위인 **스토리(Story)**, 그리고 개발자의 최소 작업 단위인 **태스크(Task)**로 구성된다.
> 3. **가치**: 비즈니스 가치와 기술적 구현을 정교하게 정렬(Alignment)하고, 가시성 있는 진척 관리를 통해 시장 변화에 기민하게 대응할 수 있는 유연한 백로그 관리 기반을 제공한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "코끼리를 어떻게 먹을 것인가?"

전통적인 방식의 '요구사항 명세서'는 너무 방대하여 당장 무엇을 해야 할지 알기 어렵습니다. 애자일에서는 "코끼리를 먹으려면 한 입 크기로 썰어야 한다"는 원칙에 따라 작업을 분할합니다. 이때 중요한 것은 단순히 크기만 줄이는 것이 아니라, 각 조각이 **'사용자에게 주는 가치'**를 잃지 않도록 계층화하는 것입니다.

### 💡 비유: 여행 계획 수립의 계층

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        애자일 작업 계층 비유                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 테마 (Theme - 여행의 목적): "건강하고 행복한 가족 휴가 보내기".             │
│  2. 에픽 (Epic - 큰 이벤트): "유럽 일주 여행 다녀오기".                        │
│     (한 번에 다 하기엔 너무 큼, 여러 번의 준비 기간 필요)                        │
│                                                                             │
│  3. 스토리 (Story - 실행 단위): "프랑스 파리에서 에펠탑 구경하기".              │
│     (가족이 기뻐하는 구체적인 행동, 한 번의 주말 여행으로 가능)                  │
│                                                                             │
│  4. 태스크 (Task - 할 일): "비행기 티켓 예약", "환전하기", "숙소 짐 풀기".       │
│     (실제로 몸을 움직여서 해야 하는 구체적인 작업들)                             │
│                                                                             │
│  → 높은 곳의 **비전(Theme)**부터 발끝의 **실행(Task)**까지 한 줄로 잇기!         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 애자일 계층별 상세 정의

| 계층 | 영문 명칭 | 정의 및 범위 | 특징 |
|:---:|:---|:---|:---|
| **테마** | **Theme** | 여러 에픽을 관통하는 최상위 비즈니스 목표 | 조직의 전략적 방향성 지시 |
| **에픽** | **Epic** | 하나의 스프린트에 담기 어려운 거대 기능 | 추후 다수의 스토리로 분할 필수 |
| **스토리** | **User Story** | 사용자가 얻는 가치를 기술한 최소 기능 단위 | **INVEST 원칙** 준수, 추정의 단위 |
| **태스크** | **Task** | 스토리를 달성하기 위한 기술적 세부 작업 | 시간(Hours) 단위로 관리, 개발자 할당 |

### 2. 하향식 분할 (Stepwise Refinement) 전략
- **Vertical Slicing**: 기술 계층(UI, DB)으로 쪼개지 않고, 사용자 가치가 관통하도록 세로로 쪼개는 것이 핵심입니다.
- **Just-In-Time (JIT)**: 처음부터 모든 태스크를 쪼개지 않고, 해당 스프린트가 시작될 때 스토리를 태스크로 분해합니다.

### 3. 작업 분류 체계도 (ASCII)

```text
    [ Theme: 매출 증대 및 고객 유입 ]
           │
    ┌──────┴────────────────────────────────┐
    │ [ Epic: 멤버십 혜택 강화 ]             │ [ Epic: 선물하기 기능 도입 ]
    └──────┬────────────────────────────────┘
           │
    ┌──────┴──────────┬───────────────┴──────┐
    │ [ Story: 포인트 ] [ Story: 등급 조회 ]   │ [ Story: 무료 배송 ]
    └──────┬──────────┘
           │
    ┌──────┴──────────┬───────────────┴──────┐
    │ [ Task: API 설계 ] [ Task: DB Table ]   │ [ Task: UI 퍼블리싱 ]
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. WBS(Waterfall) vs Agile Hierarchy
- **WBS**: 산출물과 공정 중심. 모든 하위 작업을 미리 정의. (경직됨)
- **Agile Hierarchy**: 가치와 피드백 중심. 에픽에서 스토리로 유동적으로 진화. (유연함)

### 2. 기술적 시너지: 이슈 트래커 (Jira)
현대의 도구들은 이 계층을 기본 기능으로 지원합니다. **Epic**은 진행 바(Progress Bar)로 가시화되고, **Story**는 칸반 보드의 카드가 되며, **Task**는 체크리스트나 하위 이슈(Sub-task)로 연결되어 프로젝트의 거시적 흐름과 미시적 실행을 동시에 관리하게 해 줍니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: "검색 기능" 백로그 정제
- **에픽**: 통합 검색 시스템 고도화.
- **스토리 1**: 사용자는 상품명으로 검색하여 원하는 물건을 찾을 수 있다.
- **스토리 2**: 사용자는 가격 범위를 지정하여 검색 결과를 필터링할 수 있다.
- **태스크 (스토리 1의 하위)**: ElasticSearch 인덱스 생성, 검색 결과 UI 퍼블리싱, API 연동 테스트.
- **판단**: 에픽이 너무 커서 이번 분기에 못 끝낼 것 같으면, 스토리 1(필수)만 먼저 개발하고 스토리 2(선택)는 다음 에픽으로 미루어 시장 반응 확인.

### 📢 기술사적 결언
> "계층화의 목적은 통제가 아니라 **'이해'**다. 팀원이 지금 하는 사소한 코드 수정(Task)이 회사의 어떤 위대한 비전(Theme)에 기여하고 있는지 보여주는 지도가 되어야 한다. 아키텍트는 스토리가 너무 기술적(예: DB 인덱스 생성)이지 않도록, 항상 **'사용자의 언어'**로 쓰여 있는지 감시해야 한다. 가치가 담기지 않은 조각은 단순한 노가다에 불과하기 때문이다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **가시성 확보**: 전체 프로젝트 대비 완료된 가치(Story)를 실시간 수치화 가능.
- **의사소통 효율**: "이번 주는 어떤 스토리에 집중하나요?" 한마디로 업무 정렬 완료.

### 미래 전망
미래의 작업 계층 관리는 **'AI 기반 자동 분할'**이 주도할 것입니다. 추상적인 비즈니스 요구사항을 AI가 분석하여 관련 에픽과 스토리들을 제안하고, 팀의 Velocity를 고려하여 스프린트에 적절히 배분하는 'Intelligent Planning' 시스템이 구축될 것입니다. 또한 사용자의 실시간 반응에 따라 스토리가 스스로 생성되거나 우선순위가 바뀌는 **'Adaptive Backlog'** 기술이 보편화될 것입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **에픽**은 "거대한 로봇 만들기" 같은 **커다란 꿈**이고, **스토리**는 "로봇의 팔 만들기"처럼 **한 번에 할 수 있는 일**이에요.
2. **태스크**는 팔을 만들기 위해 "나사를 조이고 색칠하기" 같은 **아주 작은 행동**들이죠.
3. 이렇게 큰 꿈을 작은 일들로 나누면, 포기하지 않고 **매일매일 꿈에 가까워지는** 마법이 일어난답니다!
