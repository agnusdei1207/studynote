# 748. 로우코드/노코드 섀도우 IT 거버넌스

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 현업 담당자(**시민 개발자**)가 로우코드/노코드(LCNC) 플랫폼을 사용하여 IT 부서의 통제 밖에서 자체적으로 앱을 개발·사용하는 **섀도우 IT(Shadow IT)** 리스크를 관리하고 제도권 내로 편입시키는 **IT 거버넌스 체계**이다.
> 2. **리스크 요인**: 보안 취약점 방치, 데이터 중복 및 파편화, 라이선스 비용 폭증, 그리고 개발자 퇴사 시 유지보수 불능(Zombie App) 등의 문제를 야기할 수 있다.
> 3. **가치**: LCNC의 민첩성(Agility)은 살리되, 중앙 IT 부서가 가이드라인과 보안 게이트를 제공하는 **'자율과 통제의 조화'**를 통해 기업의 디지털 혁신 안정성을 확보한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "개발의 민주화와 통제의 상실"

비즈니스 현장에서는 앱 하나가 급한데, IT 부서는 개발 인력이 부족해 1년을 기다리라고 합니다. 답답한 현업 부서는 직접 AppSheet나 Power Apps를 써서 앱을 뚝딱 만듭니다. 이것이 섀도우 IT의 시작입니다. 편리하지만, 보안 검수 없이 고객 정보가 담긴 앱이 배포된다면 기업에는 시한폭탄이 됩니다. **LCNC 거버넌스**는 이 시한폭탄을 안전한 혁신의 도구로 바꾸는 작업입니다.

### 💡 비유: 자녀의 요리와 주방 관리

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        LCNC 섀도우 IT 거버넌스 비유                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 배고픈 아이(현업)가 엄마(IT팀)를 기다리다 직접 요리를 시작함.            │
│                                                                             │
│  1. 섀도우 IT (위험한 상황):                                                 │
│     아이가 엄마 몰래 가스레인지를 켜고 칼을 씀. 불이 날 수도 있고 손을 다칠       │
│     수도 있음. 요리법을 몰라 상한 재료를 쓸 수도 있음. (보안/유지보수 리스크)      │
│                                                                             │
│  2. LCNC 거버넌스 (엄마의 교육):                                             │
│     - 안전 가이드: "가스레인지 대신 에어프라이어(LCNC 플랫폼)를 써라."            │
│     - 재료 통제: "냉장고(DB)에서 엄마가 허락한 칸의 재료만 써라."                 │
│     - 확인 절차: "다 만든 요리는 먹기 전에 엄마한테 한 번 보여줘라."              │
│                                                                             │
│  → 아이는 **배고픔을 스스로 해결**하고, 엄마는 **안전을 확인**하는 평화로운 주방!   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. LCNC 섀도우 IT의 주요 리스크

| 분류 | 리스크 내용 | 상세 설명 |
|:---:|:---|:---|
| **보안** | 데이터 유출 | 개인정보나 기밀이 암호화 없이 외부 클라우드에 노출 |
| **데이터** | 데이터 파편화 | 전사 ERP와 연동되지 않는 '그들만의 엑셀/DB' 양산 |
| **유지보수** | 좀비 앱 발생 | 개발한 현업 담당자가 퇴사하면 아무도 코드를 못 고침 |
| **규제** | 컴플라이언스 위반 | 금융/의료법 등 산업별 규제 미준수 앱 배포 |

### 2. LCNC 거버넌스 3대 전략 체계
1. **Platform Center of Excellence (CoE)**: 전사 공통 플랫폼 선정 및 기술 지원 조직 구성.
2. **Access & Security Policy**: 데이터 커넥터 권한 관리 및 SSO(단일 로그인) 연동 강제.
3. **App Lifecycle Management**: 앱 등록제 실시 및 정기적인 미사용 앱 삭제(Purge).

### 3. 거버넌스 아키텍처 (ASCII)

```text
    [ Citizen Developer ] ──▶ [ LCNC Platform (IDP) ]
           │                         │
           │ (1) Request Usage       │ (2) Sandbox Provisioning
           ▼                         ▼
    ┌───────────────────────────────────────────────────────────┐
    │               Central IT / CoE Governance Layer           │
    │  - Connector Guardrails (데이터 접근 제한)                  │
    │  - Automated Security Scan (자동 보안 검사)                 │
    │  - Metadata Catalog (앱 목록 관리)                         │
    └──────────────────────────┬────────────────────────────────┘
                               ▼
    [ Managed Apps ] ──────▶ [ Real Business Value ]
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 전사적 표준화 vs 부서별 자율성
- **표준화**: 관리 효율은 높지만 현업의 속도를 늦춤.
- **자율성**: 속도는 빠르지만 섀도우 IT 사고 유발.
- **해법 (The Middle Way)**: 중앙 IT는 **'플랫폼'**만 관리하고, 그 안에서의 **'창작'**은 현업에 맡기되 로그만 추적합니다.

### 2. 기술적 시너지: 데이터 메시 (Data Mesh)
현업이 직접 데이터를 다루는 LCNC 환경은 데이터 메시의 '도메인 오너십'과 일맥상통합니다. IT 팀이 정제된 데이터 제품(Data Product)을 API로 제공하면, 현업은 LCNC로 이를 안전하게 조리(Application)하는 구조가 가장 이상적인 시너지입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 전사 LCNC 도입 및 거버넌스 수립
- **상황**: 영업팀, 인사팀에서 각자 유료 노코드 툴을 결제하여 사용 중. 보안 사고 위험 및 비용 중복 발생.
- **결단**: **LCNC 통합 거버넌스 수립 및 표준 플랫폼(예: MS Power Platform) 지정**. 
- **판단**: 
    1. 개별 결제 차단 및 전사 라이선스 확보. 
    2. '데이터 손실 방지(DLP)' 정책을 설정하여 사내 데이터를 외부 이메일로 전송하는 기능 차단. 
    3. '앱 제작 커뮤니티'를 만들어 우수 사례 공유 및 보안 교육 실시.
- **효과**: 섀도우 IT를 양지로 끌어올려 보안 리스크를 제거하고, 전사적인 업무 자동화 성공.

### 📢 기술사적 결언
> "섀도우 IT는 시스템의 결함이 아니라 **'프로세스의 병목'**이 만들어낸 현상이다. 막으려 할수록 현업은 더 깊은 곳으로 숨어든다. 아키텍트는 통제자가 아닌 **'플랫폼 제공자(Enabler)'**가 되어야 한다. 로우코드를 공식적인 개발 문화로 인정하되, 보이지 않는 곳에서 자동화된 **'가드레일'**을 설치하는 것이 현대적 IT 거버넌스의 정수다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **IT 부하 감소**: 단순 반복 앱 개발 요청 40% 이상 현업 자체 해결.
- **보안 가시성**: 사내에서 돌아가는 모든 비공식 앱의 100% 모니터링 가능.

### 미래 전망
미래의 LCNC 거버넌스는 **'AI 헌법 (AI Constitution)'**과 결합할 것입니다. 현업이 앱을 만들면 AI가 실시간으로 "이 앱은 개인정보 보호법 제10조를 위반할 소지가 있습니다"라고 코칭해주고, 자동으로 보안 설정을 교정하는 'Self-governing LCNC' 시대가 올 것입니다. 개발은 더 쉬워지되, 그 안전함은 시스템이 보장하는 진정한 '시민 개발자의 시대'가 도래할 것입니다.
