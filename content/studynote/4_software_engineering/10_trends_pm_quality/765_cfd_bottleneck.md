# 765. 누적 흐름도 병목 지점 병목 분석

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시간이 경과함에 따라 각 작업 단계(To-do, In-progress, Testing, Done 등)에 머물러 있는 작업의 수를 누적해서 보여주는 그래프로, 프로세스의 안정성과 **병목(Bottleneck)** 현상을 한눈에 파악하는 칸반의 핵심 도구이다.
> 2. **지표 도출**: 그래프의 수직 거리는 현재 진행 중인 작업량(**WIP**)을, 수평 거리는 작업을 완료하는 데 걸리는 시간(**Cycle Time**)을 나타내어 프로세스의 효율성을 정량적으로 분석한다.
> 3. **가치**: 특정 단계의 띠(Band)가 급격히 넓어지는 현상을 통해 병목 지점을 조기에 발견하고, 리소스 재배치나 WIP 제한 조정을 통해 최적의 흐름(Flow)을 유지하게 한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "어디서 일이 막히고 있는가?"

단순히 "바쁘다"는 말로는 문제를 해결할 수 없습니다. 어떤 팀은 코딩은 빠른데 테스트에서 한 달씩 밀려있고, 어떤 팀은 기획이 안 나와서 개발자가 놀고 있습니다. **누적 흐름도 (CFD, Cumulative Flow Diagram)**는 이러한 작업의 '정체 구간'을 시각적으로 드러내어, 감이 아닌 데이터로 프로세스를 고치게 해 줍니다.

### 💡 비유: 고속도로 구간별 교통량 그래프

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        누적 흐름도(CFD) 비유                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 명절날 고속도로의 소통 원활도를 확인하려 함.                             │
│                                                                             │
│  1. 수직 거리 (차량 대수): 특정 지점에 차가 몇 대나 밀려있는가? (WIP)            │
│  2. 수평 거리 (통과 시간): 톨게이트를 빠져나가는 데 몇 분이나 걸리나? (Cycle Time) │
│                                                                             │
│  3. 병목 분석 (병목 구간):                                                   │
│     지도에서 "수원-천안" 구간의 색깔(띠)이 갑자기 넓게 퍼져 보인다면?            │
│     → 거기가 바로 차가 막히는 **병목 지점**! 사고가 났거나 길이 좁은 것임.        │
│                                                                             │
│  → 그래프의 띠가 일정한 두께로 예쁘게 흘러가야 **길이 안 막히고 잘 빠지는 것**!    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. CFD 그래프의 해석 지표

| 지표 | 그래프 상의 의미 | 비즈니스적 의미 |
|:---:|:---|:---|
| **WIP** | 특정 시점에서의 **띠의 수직 폭** | 현재 시스템 내에 쌓여 있는 재고(일)의 양 |
| **Cycle Time** | 특정 작업의 **수평적 폭** | 한 작업이 시작되어 완료될 때까지 걸린 시간 |
| **Throughput** | 완료(Done) 선의 **기울기** | 단위 시간당 처리할 수 있는 작업의 수 (생산성) |

### 2. 주요 패턴 분석 (Pattern Recognition)
- **띠가 넓어짐**: 작업 유입 속도가 처리 속도보다 빠름 (**병목 발생**).
- **띠가 좁아짐**: 처리 능력이 남음 (리소스 유휴 상태).
- **계단 현상**: 작업이 한꺼번에 쏟아지거나 한꺼번에 완료됨 (배치 작업의 위험).

### 3. CFD 아키텍처 시각화 (ASCII)

```text
    누적 작업 수
      ^
      │                / [ Done ]
      │              / /
      │            / /  <-- 수평 거리: Cycle Time
      │          / / /
      │        / / / / [ Testing ]
      │      / / / / / 
      │    / / / / / / [ In-Progress ] <── 수직 거리: WIP
      │  / / / / / / /
      │/ / / / / / / / [ To-do ]
      └──────────────────────────────────────────> 시간 (Time)
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 번다운 차트 vs 누적 흐름도 (CFD)
- **번다운 차트**: "이번 스프린트를 **제때 끝낼 수 있는가**?"에 집중. (단기 목표 달성용)
- **CFD**: "전체 프로세스에 **문제(병목)는 없는가**?"에 집중. (장기 공정 최적화용)

### 2. 기술적 시너지: 리틀의 법칙 (Little's Law)
CFD는 리틀의 법칙($WIP = \text{Throughput} \times \text{Cycle Time}$)을 시각적으로 증명합니다. 그래프의 수직 폭(WIP)을 줄이면 자연스럽게 수평 폭(Cycle Time)이 줄어든다는 사실을 팀원들에게 설득하는 가장 강력한 증거 자료가 됩니다.

---

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: QA 병목 구간 해결
- **상황**: 개발은 제때 끝나는데 배포가 항상 늦어짐. CFD를 보니 'Testing' 단계의 띠가 점점 벌어지고 있음.
- **결단**: **Swarming (떼 지어 돕기) 전략 및 WIP 제한 강화**. 
- **판단**: 개발팀의 'In-progress' WIP를 낮추고, 남는 인력을 'Testing' 단계로 투입하여 띠의 폭을 강제로 줄임. 
- **효과**: 전체 사이클 타임이 30% 감소하고, 개발-테스트 간의 업무 흐름이 동기화되어 안정적인 배포 체계 확보.

### 📢 기술사적 결언
> "CFD는 조직의 **'혈류 가시화 도구'**다. 혈전(병목)이 생기면 몸 전체가 마비되듯, 특정 공정의 정체는 전체 리드 타임을 망가뜨린다. 아키텍트는 그래프의 '기울기'와 '폭'을 상시 모니터링하여, 개발자가 열심히 일하고 있음에도 결과가 안 나오는 **'가짜 바쁨'**의 함정을 파헤쳐야 한다. 아름다운 CFD는 평행하게 흐르는 여러 색깔의 띠들로 구성된다."

---

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **예측 가능성 향상**: 일정한 처리량(Throughput) 확보를 통해 완료 시점 정밀 예측.
- **WIP 최적화**: 불필요한 동시 작업을 줄여 문맥 전환 비용 20% 이상 절감.

### 미래 전망
미래의 CFD는 **'예측형 흐름 분석 (Predictive Flow)'**으로 진화합니다. 현재의 띠 모양을 보고 2주 뒤에 발생할 병목 지점을 AI가 미리 시뮬레이션하여 경고해주고, 최적의 인력 배정 시나리오를 제안하는 'Self-balancing Workflow' 환경이 구축될 것입니다. 또한 원격 근무 팀의 디지털 도구 데이터를 통합하여 보이지 않는 소통의 병목까지 시각화하는 영역으로 확장될 것입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **누적 흐름도**는 놀이공원 기구마다 **줄이 얼마나 긴지** 보여주는 커다란 전광판과 같아요.
2. 기구 타는 곳(개발)은 줄이 짧은데, 표 검사하는 곳(테스트)에만 사람이 가득 모여 있다면 **"저기가 문제구나!"**라고 금방 알 수 있죠.
3. 전광판을 잘 보고 줄이 긴 곳에 도우미 친구를 더 보내면, 모두가 **기다리지 않고 신나게** 놀 수 있답니다!
