# 689. RASP 런타임 자체 보호

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 애플리케이션의 실행 환경(Runtime)에 내장되어, 실시간으로 발생하는 공격 시도를 탐지하고 차단하는 **애플리케이션 자가 보호 기술**이다.
> 2. **차별점**: 네트워크 외부에서 막는 WAF와 달리, 앱 내부에서 실행 맥락(Context)을 분석하므로 SQL 인젝션이나 XSS 공격이 실제 데이터에 도달하기 직전 가장 정확하게 방어한다.
> 3. **가치**: 미지의 취약점(Zero-day)에 대한 대응력을 높이고, 보안 패치가 즉시 어려운 레거시 시스템에 대해 강력한 가상 패칭(Virtual Patching) 효과를 제공한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "내부로부터의 방어"

전통적인 보안 장비(WAF, IPS)는 트래픽의 '모양'만 보고 판단합니다. 하지만 공격자가 암호화된 채널을 쓰거나 복잡한 로직을 우회하면 무용지물이 됩니다. **RASP (Runtime Application Self-Protection)**는 앱이 구동될 때 보안 엔진이 함께 로드되어, 함수 호출이나 DB 쿼리 실행 직전에 "이 동작이 안전한가?"를 스스로 판단합니다.

### 💡 비유: 몸속의 면역 체계와 백신

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        RASP (런타임 자가 보호) 비유                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. WAF (성문 앞 경비원):                                                    │
│     성 밖에서 들어오는 사람들만 검사함. 변장하거나 몰래 들어오면 놓칠 수 있음.      │
│                                                                             │
│  2. RASP (몸속의 면역 세포):                                                 │
│     음식(데이터)이 이미 입안으로 들어와서 소화되기 직전, 몸속 세포가 직접          │
│     "어? 이건 독극물인데?"라고 알아채고 바로 뱉어내는 것과 같음.                 │
│                                                                             │
│  → 시스템 가장 깊숙한 곳에서 스스로를 지키는 가장 똑똑한 보안 세포!              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. RASP의 작동 메커니즘 (Hooking)
- **에이전트 방식**: JVM이나 .NET 런타임에 에이전트를 삽입하여 핵심 API 호출을 낚아챔(Hooking).
- **라이브러리 방식**: 애플리케이션 프레임워크에 보안 라이브러리를 통합.
- **분석 범위**: 사용자 세션, DB 쿼리 실행, 파일 시스템 접근, 네트워크 호출.

### 2. RASP vs WAF 비교

| 항목 | WAF (Web Application Firewall) | RASP (Runtime Protection) |
|:---:|:---|:---|
| **위치** | 네트워크 경계 (외부) | **애플리케이션 내부 (런타임)** |
| **판단 근거** | 페이로드 시그니처, 패턴 | **코드 실행 맥락(Context)** |
| **정확도** | 중간 (오탐 가능성 높음) | **매우 높음 (실행 흐름 파악)** |
| **암호화 대응** | SSL 복호화 장비 필요 | 복호화된 데이터 직접 분석 가능 |
| **주요 방어** | 알려진 패턴 공격 차단 | 제로데이, 비즈니스 로직 공격 차단 |

### 3. RASP 아키텍처 구조도 (ASCII)

```text
    [ User Request ] ──▶ [ Network / WAF ] ──▶ [ Web Server ]
                                                   │
    ┌──────────────── App Runtime (JVM / CLR) ─────┼─────────────────┐
    │                                              ▼                 │
    │  ┌───────────────┐      ┌─────────────┐      ┌──────────────┐  │
    │  │ Business Code │ ──▶  │ RASP Agent  │ ──▶  │ SQL / IO API │  │
    │  └───────────────┘      └──────┬──────┘      └──────────────┘  │
    │                                │                               │
    │                                ▼ [ Decision ]                  │
    │                         Allow / Block / Alert                  │
    └────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 가상 패칭 (Virtual Patching)
취약점이 발견되었으나 코드를 수정하고 배포할 시간이 없을 때, RASP 설정만으로 해당 공격 패턴을 런타임에 차단할 수 있습니다. 이는 긴급 대응 시 비즈니스 연속성을 지켜주는 핵심 시너지입니다.

### 2. 기술적 시너지: 제로 트러스트 (Zero Trust)
"아무도 믿지 말고 항상 검증하라". RASP는 이미 내부 망으로 들어온 데이터조차 신뢰하지 않고 실행 직전에 최종 검증함으로써, 제로 트러스트 보안 모델의 애플리케이션 계층 구현체가 됩니다.

---

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 패치가 중단된 레거시 시스템 보호
- **상황**: 지원이 끝난 구버전 Struts 프레임워크를 사용 중인 금융 시스템.
- **결단**: **RASP 솔루션 도입**. 
- **판단**: 코드 수정이 불가능하므로, 런타임 에이전트를 심어 취약한 함수 호출을 실시간 감시. 
- **효과**: 신규 취약점(RCE 등)이 발표되어도 코드를 건드리지 않고 RASP 정책 업데이트만으로 실시간 방어망 구축.

### 📢 기술사적 결언
> "RASP는 애플리케이션 보안의 **'최종 수비수'**다. 네트워크 보안 장비가 거를 수 없는 '데이터의 의미'를 이해하는 유일한 기술이기 때문이다. 다만, 실행 환경에 직접 개입하므로 애플리케이션의 성능(Latency)에 미치는 영향도를 반드시 검증해야 한다. 보안과 성능의 정교한 트레이드오프 분석을 통해 **'핵심 트랜잭션'** 위주로 보호 범위를 설정하는 지혜가 필요하다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **오탐율 감소**: WAF 대비 오탐 발생 빈도 70% 이상 감소.
- **운영 가시성**: 공격이 코드의 어느 라인에서 발생했는지 정밀한 스택 트레이스 제공.

### 미래 전망
미래의 RASP는 **'자율 자가 치유(Self-healing)'** 시스템으로 진화합니다. 공격을 차단하는 것을 넘어, 취약한 코드 구간을 AI가 런타임에 안전한 코드로 임시 치환하거나, 공격자의 IP를 실시간으로 판단하여 전체 인프라에 차단 룰을 전파하는 '능동형 보안 유기체'의 중추 역할을 수행하게 될 것입니다.
