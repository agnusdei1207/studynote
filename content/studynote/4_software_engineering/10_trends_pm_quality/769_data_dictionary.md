# 769. 구조적 분석 도구 데이터 사전(DD) 표기법

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자료 흐름도(DFD)에 나타난 모든 자료 요소들의 상세한 정의와 구성, 특성을 표준화된 기호로 명시한 **데이터의 저장소(Metadata Repository)**이다.
> 2. **표기법**: 정의(`=`), 연결(`+`), 반복(`{ }`), 선택(`[ | ]`), 생략(`( )`), 주석(`* *`) 등 6가지 표준 기호를 사용하여 자료의 논리적 구조를 수식처럼 표현한다.
> 3. **가치**: 시스템 내 데이터의 의미를 일관성 있게 관리하여 개발자 간의 오해를 방지하고, 데이터 변경 시 영향도를 즉각 파악할 수 있는 분석 명세서 역할을 한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "그림만으로는 부족한 데이터의 정의"

자료 흐름도(DFD)는 시스템의 기능을 보여주지만, 화살표 위에 적힌 "주문서"라는 데이터가 구체적으로 어떤 항목(이름, 주소, 수량 등)으로 이루어졌는지는 알려주지 않습니다. **데이터 사전 (DD, Data Dictionary)**은 DFD라는 '지도'의 각 지점에 대한 '상세 설명서'와 같습니다.

### 💡 비유: 요리책의 목차와 재료 상세 목록

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        데이터 사전(DD) 표기법 비유                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 맛있는 비빔밥(데이터)을 만들려고 함.                                     │
│                                                                             │
│  1. DFD (요리 순서도): "야채를 썰어서 밥에 넣는다"라고 그림으로 표현함.           │
│                                                                             │
│  2. 데이터 사전 (재료 정의):                                                 │
│     비빔밥 = 밥 + 고추장 + {나물} + [달걀프라이 | 고기] + (참기름)             │
│     - 연결(+): 밥이랑 고추장이 다 있어야 함.                                    │
│     - 반복({ }): 나물은 여러 종류가 들어갈 수 있음.                             │
│     - 선택([ | ]): 달걀이나 고기 중 하나를 고름.                                │
│     - 생략(( )): 참기름은 넣어도 되고 안 넣어도 됨.                             │
│                                                                             │
│  → 데이터가 **무엇으로 이루어졌는지** 수학 공식처럼 딱 정해두는 약속!             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 데이터 사전 표준 표기 기호

| 기호 | 의미 | 상세 설명 | 예시 |
|:---:|:---|:---|:---|
| **=** | **정의 (Definition)** | 자료의 내용을 정의함 (~으로 구성됨) | 주민번호 = 앞자리 + 뒷자리 |
| **+** | **연결 (Sequence)** | 두 개 이상의 자료 항목을 결합 | 성명 = 성 + 이름 |
| **{ }** | **반복 (Iteration)** | 항목이 반복적으로 나타남 | 주문서 = {항목} |
| **[ \| ]** | **선택 (Selection)** | 나열된 항목 중 하나를 선택 | 성별 = [남 \| 여] |
| **( )** | **생략 (Optional)** | 해당 항목이 있을 수도, 없을 수도 있음 | 주소 = 시도 + (구군) + 상세 |
| **\* \*** | **주석 (Comment)** | 부연 설명이나 제약 조건 기술 | * 8자리 숫자 필수 * |

### 2. 자료 사전의 구성 내용
- **자료 원소**: 더 이상 쪼갤 수 없는 최소 단위.
- **자료 구조**: 원소들의 조합.
- **자료 흐름**: DFD의 화살표 명칭.
- **자료 저장소**: DFD의 파일/DB 명칭.

### 3. 분석 모델링 연계 구조 (ASCII)

```text
    [ DFD: Data Flow Diagram ] <──(Mapping)──▶ [ DD: Data Dictionary ]
    
    (Process A) ───▶ [ Order ] ────▶ (Process B)   Order = Cust_Info + {Items}
                       │                           Cust_Info = Name + Phone
                       ▼                           Items = Code + Qty
                 [[ Order_DB ]]
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 데이터 사전(DD) vs 자료 명세(Mini-Spec)
- **데이터 사전**: **데이터(Data)**의 정적인 구조와 의미를 정의. (명사 중심)
- **자료 명세**: **프로세스(Process)**의 동적인 처리 로직을 정의. (동사 중심)
- **시너지**: 이 둘이 합쳐져 시스템의 '기능'과 '데이터'가 완벽히 명세됩니다.

### 2. 기술적 시너지: 데이터베이스 카탈로그
현대의 DBMS(Oracle, MySQL 등) 내부에 있는 시스템 카탈로그(System Catalog)는 데이터 사전의 자동화된 형태입니다. 설계 단계의 DD가 실제 DB의 스키마 정보와 일치하도록 관리하는 것이 **데이터 거버넌스**의 핵심입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 전사 표준 데이터 용어 사전 구축
- **상황**: 부서마다 '고객번호'를 `CUST_NO`, `ID`, `MEMBER_ID` 등으로 제각각 부름.
- **결단**: **표준 DD 수립 및 공표**. 
- **판단**: 모든 프로젝트에서 `Customer_ID = * 10자리 숫자 *` 로 정의를 통일하고, DFD 작성 시 이 용어만 사용하도록 강제.
- **효과**: 시스템 통합 시 데이터 변환 오류 80% 감소 및 유지보수 가독성 향상.

### 📢 기술사적 결언
> "데이터 사전은 시스템의 **'어휘집'**이다. 어휘가 통일되지 않은 조직에서 만들어진 코드는 바벨탑과 같다. 아키텍트는 DFD를 그리는 화려한 솜씨보다, 그 이면의 데이터를 얼마나 **'엄격하고 간결하게'** 정의하느냐에 집중해야 한다. 6가지 기호만으로 복잡한 비즈니스 엔티티를 수식화할 수 있는 능력은, 구조적 분석을 넘어 객체지향 설계의 **'클래스 정의'**로 이어지는 핵심 기본기다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **의사소통 비용**: 데이터 정의 확인을 위한 불필요한 질의응답 50% 이상 제거.
- **데이터 품질**: 표준화된 DD 준수를 통해 데이터 도메인 위배 결함 최소화.

### 미래 전망
미래의 데이터 사전은 **'지능형 메타데이터 관리 시스템'**으로 진화합니다. 사람이 수동으로 기호를 적는 대신, 소스코드를 분석하여 AI가 데이터 구조를 역공학(Reverse Engineering)하고 자동으로 DD를 생성/관리하는 기술이 표준이 될 것입니다. 또한 데이터 메시 아키텍처와 결합하여, 부서별로 분산된 DD들이 실시간으로 동기화되는 '연합 데이터 사전' 체계가 구축될 것입니다.
