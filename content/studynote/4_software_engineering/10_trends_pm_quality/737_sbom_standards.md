# 737. SBOM 규격 SPDX CycloneDX

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소프트웨어를 구성하는 모든 부품(오픈소스 라이브러리, 모듈, 의존성)의 명세서인 **SBOM**을 기계가 읽고 처리할 수 있도록 표준화한 데이터 규격이다.
> 2. **양대 표준**: 라이선스 관리와 법적 컴플라이언스에 강점을 가진 **SPDX**와, 보안 취약점 분석 및 자동화된 빌드 파이프라인(DevSecOps)에 최적화된 **CycloneDX**가 시장을 양분하고 있다.
> 3. **가치**: 공급망 전반의 투명성을 확보하여 취약점 발생 시 영향 범위를 즉각 파악하게 하며, 미 행정명령(EO 14028) 등 글로벌 보안 규제 대응의 필수 요건으로 자리 잡았다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "표준이 없으면 목록은 낙서에 불과하다"

모든 소프트웨어 기업이 제각각의 엑셀 형식으로 부품 목록을 관리한다면, 수만 개의 라이브러리를 사용하는 현대의 복잡한 공급망을 통제할 수 없습니다. 취약점이 터졌을 때 "우리 시스템에 그 부품이 있는가?"라는 질문에 수 초 내로 답하기 위해, 컴퓨터가 서로 데이터를 주고받을 수 있는 공통의 언어(규격)가 필요해졌습니다.

### 💡 비유: 전 세계 공통 바코드와 성분표

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        SBOM 표준 규격 비유                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 전 세계 모든 과자(소프트웨어)의 알레르기 성분(취약점)을 관리해야 함.      │
│                                                                             │
│  1. SPDX (법률 전문가의 성분표):                                              │
│     이 재료를 누가 만들었고, 판권(라이선스)은 누구에게 있는지 아주 꼼꼼하게        │
│     적어둔 두꺼운 서류. (법적 분쟁 방지용)                                       │
│                                                                             │
│  2. CycloneDX (응급실 의사의 성분표):                                         │
│     독성 물질이 들었는지 빨리 확인하기 위해 가볍고 직관적으로 만든 차트.          │
│     - 컴퓨터가 스캔하자마자 "이건 위험해!"라고 바로 알려줌. (보안 자동화용)       │
│                                                                             │
│  → 어떤 양식을 쓰든, **누가 봐도 똑같이 해석**할 수 있게 약속을 정하는 것!        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 양대 SBOM 규격 비교 분석

| 항목 | SPDX (Software Package Data Exchange) | CycloneDX |
|:---:|:---|:---|
| **주관** | Linux Foundation (ISO 표준) | OWASP Foundation |
| **태생적 목적** | **오픈소스 라이선스 및 지식재산권 관리** | **보안 취약점 분석 및 공급망 보호** |
| **데이터 형식** | TagValue, JSON, YAML, RDF | JSON, XML, Protocol Buffers |
| **특징** | 매우 상세하고 포괄적임 (복잡도 높음) | 가볍고 자동화 도구 친화적임 (사용 용이) |
| **핵심 기능** | 라이선스 텍스트 포함, 파일 무결성 검증 | VEX(취약점 상태 공유), 의존성 그래프 |

### 2. SBOM의 핵심 데이터 구조
- **Metadata**: 소프트웨어 이름, 버전, 작성자, 생성 시간.
- **Components**: 라이브러리 목록, 고유 식별자(PURL, CPE), 해시값.
- **Dependencies**: 부품 간의 포함 관계 (그래프 구조).
- **Vulnerabilities (VEX)**: 알려진 취약점 정보 및 조치 상태 (CycloneDX 특화).

### 3. 규격 통합 및 활용 아키텍처 (ASCII)

```text
    [ Build Tool ] ──▶ [ SBOM Tool (Syft/Tern) ]
                             │
          ┌──────────────────┴──────────────────┐
          ▼ (Output Formats)                    ▼
    [ SPDX (ISO/IEC 5962) ]             [ CycloneDX (OWASP) ]

          └───────────▶ [ SBOM Repository ] ◀───┘
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. SBOM vs VEX (Vulnerability Exploitability eXchange)
- **SBOM**: "우리 앱에 A 라이브러리가 **있다**." (정적 목록)
- **VEX**: "A 라이브러리에 취약점이 있지만, 우리 앱에서는 **그 기능을 안 써서 안전하다**." (동적 상태)
- **시너지**: SBOM 규격(특히 CycloneDX) 내에 VEX 정보를 포함하여, 보안 담당자가 무분별한 알람(Alert Fatigue)에 시달리지 않게 돕습니다.

### 2. 기술적 시너지: SLSA (Supply-chain Levels for Software Artifacts)
SBOM은 소프트웨어 인도물의 무결성을 증명하는 SLSA 프레임워크의 핵심 증적 자료입니다. 규격화된 SBOM이 있어야만 빌드 전 과정의 투명성을 입증하고 'L3' 이상의 높은 보안 등급을 획득할 수 있습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 글로벌 시장 진출 가이드라인
- **상황**: 유럽과 미국 수출용 소프트웨어에 대한 보안 증명 요구.
- **결단**: **SPDX와 CycloneDX 병행 생성 전략**. 
- **판단**: 
    1. 법무팀용으로는 ISO 표준인 **SPDX** 문서를 생성하여 라이선스 리스크 관리.
    2. 개발/보안팀용으로는 **CycloneDX**를 생성하여 CI/CD 파이프라인 내 실시간 취약점 스캔 연동.
- **효과**: 글로벌 규제 대응(Compliance)과 실질적인 시스템 방어(Security)라는 두 마리 토끼를 잡음.

### 📢 기술사적 결언
> "규격은 도구일 뿐, 본질은 **'추적 가능성(Traceability)'**이다. SPDX와 CycloneDX 중 무엇을 선택하느냐보다, 우리 조직의 빌드 파이프라인에서 **'누락 없이 실시간으로'** SBOM이 갱신되는 체계를 갖추었는지가 훨씬 중요하다. 아키텍트는 규격의 세부 필드에 매몰되지 말고, 전사적인 **'SBOM 거버넌스'**를 수립하여 공급망 전체의 신뢰 사슬을 설계해야 한다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **취약점 분석 시간**: 수동 조사 대비 90% 이상 단축.
- **라이선스 소송 예방**: 위반 라이브러리 조기 식별로 법적 리스크 제거.

### 미래 전망
미래의 SBOM 규격은 **'Runtime SBOM'**으로 진화할 것입니다. 정적인 빌드 산출물을 넘어, 실제 서버에서 메모리에 로드되어 실행 중인 모듈의 상태를 실시간 규격화하여 보고하는 기술입니다. 또한 **AI 기반 자동 규격 변환**이 보편화되어, 서로 다른 표준을 사용하는 기업 간에도 자유롭게 보안 정보를 교환하는 '글로벌 보안 메쉬'가 형성될 것입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **SBOM 규격**은 전 세계 친구들이 장난감 부품 목록을 적을 때 쓰는 **"공통 약속 양식"**이에요.
2. 어떤 친구는 법률 편지봉투(SPDX)에 담고, 어떤 친구는 번개 탐지기(CycloneDX)에 담지만, **내용물은 똑같이** 적혀 있죠.
3. 이 약속된 양식만 있으면, 멀리 있는 친구라도 내 장난감에 **고장 난 부품이 섞여 있는지** 금방 알려줄 수 있답니다!
