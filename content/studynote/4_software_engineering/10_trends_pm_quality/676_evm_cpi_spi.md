# 676. EVM (Earned Value Management) SPI, CPI 계산

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로젝트의 범위, 일정, 자원을 통합하여 현재의 수행 성과를 측정하고 미래를 예측하는 **성과 측정 및 진척 관리 기법**이다.
> 2. **핵심 지표**: 투입된 비용 대비 성과를 나타내는 **CPI(비용수행지수)**와 계획 대비 성과를 나타내는 **SPI(일정수행지수)**를 통해 프로젝트의 건강 상태를 수치화한다.
> 3. **가치**: "90% 완료 증후군"과 같은 주관적 보고를 배제하고, 객관적인 데이터를 기반으로 프로젝트 완료 시점의 최종 비용(EAC)과 기간을 예측하여 선제적 조치를 가능하게 한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "우리는 정말 잘하고 있는가?"

단순히 "예산 1억 중 5천만 원을 썼다"는 정보만으로는 프로젝트가 잘 돌아가는지 알 수 없습니다. 5천만 원을 써서 계획된 80%의 일을 했는지, 아니면 20%밖에 못 했는지가 중요하기 때문입니다. **EVM (Earned Value Management)**은 '돈'과 '시간', 그리고 '일의 양'을 하나의 저울(Value) 위에 올려놓고 비교하는 기술입니다.

### 💡 비유: 10일간 100km 걷기 여행

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        EVM 성과 지표 비유                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 10일 동안 100km를 걷기로 함 (예산: 10일, 목표: 100km).                  │
│         지금은 **5일째** 되는 날 저녁임.                                        │
│                                                                             │
│  1. PV (계획): 원래 오늘까지 **50km**를 걸었어야 함.                           │
│  2. EV (성과): 실제로는 **40km**밖에 못 걸었음. (지연 발생!)                    │
│  3. AC (실제 비용): 그런데 신발이 닳아서 **6일치** 식량을 이미 다 먹어버림.       │
│                                                                             │
│  - SPI (일정): 40km / 50km = 0.8 (계획보다 20% 느림)                          │
│  - CPI (비용): 40km / 60km(상당액) = 0.67 (예산보다 훨씬 많이 씀)              │
│                                                                             │
│  → "지금처럼 가면 10일이 아니라 15일 걸리고, 밥값도 부족하겠네!"라고 예측!        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. EVM의 3대 기본값 (Standard Terms)

| 약어 | 명칭 | 정의 | 의미 |
|:---:|:---|:---|:---|
| **PV** | **Planned Value** | 특정 시점까지 계획된 작업의 예산 | 무엇을 하기로 했나? |
| **EV** | **Earned Value** | 실제 완료된 작업의 예산 가치 | **진짜 무엇을 했나?** |
| **AC** | **Actual Cost** | 완료된 작업에 투입된 실제 비용 | 돈을 얼마 썼나? |

### 2. 성과 측정 지수 (Performance Indices)

- **SPI (Schedule Performance Index)**: $EV / PV$
    - 1.0 초과: 일정 단축 (Ahead of Schedule)
    - 1.0 미만: **일정 지연 (Behind Schedule)**
- **CPI (Cost Performance Index)**: $EV / AC$
    - 1.0 초과: 예산 절감 (Under Budget)
    - 1.0 미만: **예산 초과 (Over Budget)**

### 3. 미래 예측 공식 (Forecasting)
- **BAC (Budget At Completion)**: 프로젝트 전체 총 예산.
- **EAC (Estimate At Completion)**: 현재 추세 반영 시 최종 예상 비용 ($BAC / CPI$).
- **ETC (Estimate To Complete)**: 남은 작업을 끝내기 위해 필요한 추가 비용 ($EAC - AC$).

### 4. EVM 데이터 흐름도 (ASCII)

```text
    금액($)
      ^
      │                   / [ EAC ] : 최종 예상 비용
      │                /
      │             /------- [ BAC ] : 원래 총 예산
      │          ● [ AC ] : 실제 쓴 돈
      │       /  
      │    / ● [ EV ] : 진짜 한 일의 가치
      │  /  /
      │/--● [ PV ] : 계획했던 양
      └──────────────────────────────────────────> 시간 (Time)
               [ 지금 ]
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. SPI vs 번다운 차트 (Burndown)
- **SPI**: 비용과 결합된 정밀한 성과 지표. 주로 워터폴이나 대규모 공공 프로젝트에서 사용.
- **번다운 차트**: 오직 '남은 일의 양'에만 집중. 애자일 스크럼 팀의 일일 진척 관리에 최적화.

### 2. 기술적 시너지: 프로젝트 관리 도구 연동
현대의 Jira나 MS Project는 타임시트(Time Sheet)와 연동되어 AC를 실시간 수집하고, 티켓 완료 상태를 통해 EV를 자동 계산하여 실시간 CPI/SPI 대시보드를 제공합니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 프로젝트 위기 징후 포착
- **상황**: 겉으로는 모든 팀원이 바쁘게 일하고 있고 보고서상 '진척률 50%'라고 함.
- **결단**: **EVM 분석 실시**. 
- **판단**: 실측 결과 $SPI=0.7$, $CPI=0.8$로 판명. 즉, 일은 계획보다 30% 늦어지고 있으며 돈은 20% 더 쓰고 있음. 
- **조치**: 인력 추가(Crashing)보다는 작업 범위 조정(De-scoping)과 비효율적 프로세스 개선을 통해 CPI를 1.0 근처로 복구하는 전략 수립.

### 📢 기술사적 결언
> "EVM은 프로젝트의 **'거짓말 탐지기'**다. 관리자는 '잘 되고 있다'는 주관적 형용사가 아닌, CPI와 SPI라는 **'객관적 명사'**로 보고해야 한다. 특히 인건비가 대부분인 소프트웨어 프로젝트에서 CPI의 하락은 곧 팀의 생산성 저하를 의미하므로, 지수가 0.9 이하로 떨어지는 순간 즉각적인 근본 원인 분석(RCA)에 착수해야 한다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **예측 정확도**: 프로젝트 종료 20% 시점에서 최종 비용을 $\pm 10\%$ 오차로 예측 가능.
- **투명성 확보**: 발주자와 수주자 간의 객관적인 성과 합의 기준 마련.

### 미래 전망
미래의 EVM은 **'AI기반 예측형 EVM'**으로 진화하고 있습니다. 단순 현재 추세(Linear) 반영을 넘어, AI가 팀의 과거 결함 수정 패턴과 기술적 부채 데이터를 학습하여 비선형적인 EAC를 도출함으로써, 훨씬 더 인간적인(현실적인) 조기 경보 시스템을 구축하게 될 것입니다.
