# 711. KWCAG 웹 접근성 지침

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장애인, 고령자 등 모든 사용자가 신체적·환경적 제약 없이 웹 사이트의 모든 정보를 동등하게 이용할 수 있도록 보장하기 위한 **한국형 웹 콘텐츠 접근성 지침**이다.
> 2. **4대 원칙**: 정보를 명확히 인지할 수 있어야 하는 **인식의 용이성**, 모든 기능을 조작할 수 있어야 하는 **운용의 용이성**, 내용을 이해하기 쉬워야 하는 **이해의 용이성**, 다양한 기술(보조공학 기기 등)과 호환되어야 하는 **견고성**을 기반으로 한다.
> 3. **가치**: 보편적 설계(Universal Design)를 실현하여 정보 격차를 해소하고, 법적 준수 사항(장애인차별금지법 등) 이행과 동시에 검색 엔진 최적화(SEO) 및 전반적인 웹 품질 향상을 견인한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "장벽 없는 디지털 세상"

웹 접근성은 단순한 '배려'가 아닌 '권리'입니다. 시각 장애인이 스크린 리더로 뉴스를 읽고, 손을 쓰지 못하는 사용자가 키보드만으로 쇼핑을 할 수 있어야 합니다. **KWCAG (Korean Web Content Accessibility Guidelines)**는 W3C의 WCAG 표준을 기반으로 국내 실정에 맞게 제정된 국가 표준(KS)입니다.

### 💡 비유: 건물의 경사로와 안내 표지판

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        웹 접근성(KWCAG) 비유                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 도서관(웹사이트)을 짓는데, 누구나 이용할 수 있어야 함.                   │
│                                                                             │
│  1. 인식의 용이성 (음성 안내): 눈이 안 보이는 분을 위해 책 내용을 읽어주는        │
│     시스템(스크린 리더용 대체 텍스트)을 갖춤.                                   │
│                                                                             │
│  2. 운용의 용이성 (경사로): 휠체어를 타거나 계단을 못 오르는 분을 위해           │
│     엘리베이터나 경사로(키보드 전용 조작)를 만듦.                                │
│                                                                             │
│  3. 이해의 용이성 (쉬운 표지판): 길을 잃지 않게 표지판을 크고 쉽게(일관된 메뉴)   │
│     만들어두고, 잘못된 길로 가면 친절히 알려줌(오류 정정).                       │
│                                                                             │
│  → 모든 사람이 **차별 없이** 지식의 창고를 이용할 수 있게 하는 마법의 설계도!      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. KWCAG 2.2 기준 4대 원칙 및 핵심 검사 항목

| 원칙 | 상세 내용 | 주요 검사 항목 (Checkpoint) |
|:---:|:---|:---|
| **인식의 용이성** | 정보를 오감으로 느낄 수 있게 제공 | 대체 텍스트 제공, 자막 제공, 명도 대비(4.5:1 이상) |
| **운용의 용이성** | UI 구성 요소의 조작 가능성 보장 | 키보드 사용 보장, 응답 시간 조절, 광과민성 발작 예방 |
| **이해의 용이성** | 정보와 조작의 이해 가능성 | 언어 명시, 예측 가능성, 콘텐츠의 선형 구조 |
| **견고성** | 현재 및 미래 기술과의 호환성 | 마크업 오류 방지, 보조공학 기기 호환성 (ARIA 활용) |

### 2. 기술적 구현의 핵심: WAI-ARIA
- **ARIA (Accessible Rich Internet Applications)**: HTML의 의미론적(Semantic) 한계를 보완하여, 동적인 웹 컴포넌트(모달, 탭 등)의 역할과 상태를 보조공학 기기에 전달하는 기술 표준입니다.

### 3. 접근성 준수 아키텍처 (ASCII)

```text
    [ Web Content ] ──▶ [ Semantic HTML / ARIA ] ──▶ [ Accessibility Tree ]
    (텍스트, 이미지)      (의미 부여 및 상태 명시)       (브라우저가 생성)
                                                            │
    ┌───────────────────────────────────────────────────────┘
    ▼
    [ Assistive Tech ] ──▶ [ Multi-modal Output ] ──▶ [ User ]
    (Screen Reader)         (Speech, Braille)         (장애인/고령자)
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. UX vs Web Accessibility
- **UX (User Experience)**: 사용자의 '즐거움과 편의성'에 초점.
- **웹 접근성**: 사용자의 '동등한 접근권'에 초점.
- **시너지**: 접근성이 좋아지면 검색 봇의 정보 수집이 쉬워져 **SEO**가 향상되고, 시력이 낮은 비장애인 사용자에게도 더 나은 경험을 제공합니다.

### 2. 관련 법규와 컴플라이언스
- **지능정보화기본법**: 공공기관의 웹 접근성 준수 의무화.
- **장애인차별금지법 (장차법)**: 모든 법인 및 민간 웹사이트의 접근성 준수 강제 (위반 시 과태료 리스크).

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 금융권 뱅킹 앱 접근성 개선
- **상황**: 시각 장애인이 이체 기능을 이용하려 할 때 "버튼 1, 버튼 2"라고만 읽어주는 문제 발생.
- **결단**: **KWCAG 전면 개편 수행**. 
- **판단**: 
    1. 모든 이미지에 구체적인 `alt` 속성 부여.
    2. 복잡한 차트 데이터를 표(Table) 형태의 대체 수단으로 제공.
    3. 키보드 탭 순서(Focus Order)를 논리적인 흐름으로 재배치.
- **효과**: 웹 접근성 품질 인증 마크 획득 및 ESG 경영 지표 달성.

### 📢 기술사적 결언
> "웹 접근성은 **'디지털 민주주의'**의 척도다. 소스코드는 기계가 읽는 것 이전에 인간이 소통하는 수단이다. 시맨틱 마크업을 무시하고 `<div>` 태그로만 도배된 사이트는 보조공학 기기 사용자에게는 거대한 암흑과 같다. 아키텍트는 설계 초기부터 **'접근성 체크리스트'**를 개발 문화에 내재화하여, 기술의 진보가 누군가에게는 소외의 이유가 되지 않도록 방어해야 한다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **잠재 고객 확대**: 고령자 및 장애인 유저 유입으로 시장 점유율 약 10~15% 확대 가능.
- **법적 리스크 제거**: 차별 금지 위반으로 인한 소송 및 시정 권고 예방.

### 미래 전망
미래의 웹 접근성은 **'AI 자율 보정'**으로 진화합니다. 대체 텍스트가 없는 이미지를 AI가 실시간으로 인식하여 음성으로 설명해주거나, 복잡한 레이아웃을 사용자의 장애 유형에 맞게 실시간으로 재배열해주는 지능형 접근성 엔진이 보편화될 것입니다. 또한 가상 현실(VR/AR) 환경에서의 접근성 지침인 **XR Access**로 그 영역이 확장될 것입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **웹 접근성**은 눈이 나쁜 친구나 손을 다친 친구라도 **똑같이 컴퓨터 게임을 할 수 있게** 도와주는 착한 약속이에요.
2. 그림만 보여주는 게 아니라 **글자로 설명(대체 텍스트)**해주고, 마우스가 없어도 **키보드(경사로)**로 다닐 수 있게 길을 터주는 거죠.
3. 이 약속을 지키면, **세상 모든 친구**가 우리 홈페이지에 놀러 와서 즐겁게 놀 수 있답니다!
