# 778. 소프트웨어 테스트 성숙도 모델 (TMMi)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 조직의 소프트웨어 테스트 프로세스를 진단하고 개선하기 위한 **글로벌 테스트 성숙도 평가 모델**로, CMMI의 구조를 벤치마킹하여 5단계 성숙도 레벨을 정의한다.
> 2. **상보성**: CMMI가 개발 프로세스 전반을 다룬다면, **TMMi**는 테스트 전략, 계획, 설계, 실행 및 관리에 특화되어 고도화된 품질 보증(QA) 체계를 구축하게 돕는다.
> 3. **가치**: 주구장창 테스트만 하는 비효율을 제거하고, 리스크 기반 테스팅과 테스트 자동화 자산화를 통해 테스트의 **효과성(Effectiveness)**과 **효율성(Efficiency)**을 극대화한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "개발만 잘한다고 품질이 좋아질까?"

개발 프로세스가 성숙해도(CMMI) 테스트가 주먹구구식이라면 결함은 운영 환경으로 유출됩니다. 테스트를 단순한 '버그 잡기'가 아니라 하나의 '독립된 공학 프로세스'로 격상시켜 관리해야 한다는 필요성에서 **TMMi (Test Maturity Model integration)**가 탄생했습니다.

### 💡 비유: 학생의 시험 공부 전략 진화

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        TMMi 성숙도 5단계 비유                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 레벨 1 (초기): 시험 전날에만 반짝 공부함. 운 좋으면 점수 잘 나옴.            │
│  2. 레벨 2 (관리): 시험 범위를 확인하고 일주일 전부터 계획적으로 공부함.         │
│  3. 레벨 3 (정의): 나만의 오답 노트와 표준 학습법을 만듦. (모든 과목 적용)       │
│  4. 레벨 4 (측정): 문제당 풀이 시간을 재고 정답률을 통계로 관리함.               │
│  5. 레벨 5 (최적화): 틀릴 문제를 미리 예상하고, 가장 효율적인 공부법으로 개선함.   │
│                                                                             │
│  → 단순히 공부(테스트)를 많이 하는 게 아니라, **똑똑하게 하는 방법**을 배우는 것! │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. TMMi 5단계 성숙도 상세 모델

| 레벨 | 명칭 | 핵심 목표 및 특징 | 주요 프로세스 영역 (PA) |
|:---:|:---|:---|:---|
| **1** | **Initial (초기)** | 혼란, 디버깅 수준의 테스트 | (없음) |
| **2** | **Managed (관리)** | 프로젝트별 테스트 계획 및 통제 | 테스트 정책, 계획, 전략, 환경 |
| **3** | **Defined (정의)** | **조직 차원의 표준화**, 비기능 포함 | 테스트 조직, 교육, 설계 및 실행 |
| **4** | **Measured (측정)** | 정량적 측정 및 리스크 기반 통제 | 테스트 정량 관리, 소프트웨어 품질 평가 |
| **5** | **Optimization (최적화)** | 지속적 개선, 결함 예방 | 결함 예방, 테스트 프로세스 최적화 |

### 2. TMMi의 구조적 특징
- **Risk-driven**: 레벨 2부터 리스크를 기반으로 테스트 우선순위를 결정함.
- **Independence**: 테스트 팀의 독립성을 강조하여 객관적 품질 검증 보장.
- **Alignment**: 비즈니스 목표와 테스트 목표의 정렬 중시.

### 3. 성숙도 발전 로드맵 (ASCII)

```text
    [ Level 5: 최적화 ] <─── 통계적 결함 예방 (Prevention)
           ▲
    [ Level 4: 측정 ]   <─── 품질 메트릭 기반 관리 (Prediction)
           ▲
    [ Level 3: 정의 ]   <─── 전사 표준 테스트 프로세스 (Standard)
           ▲
    [ Level 2: 관리 ]   <─── 프로젝트 단위 계획/수행 (Repeatable)
           ▲
    [ Level 1: 초기 ]   <─── 임기응변식 테스트 (Ad-hoc)
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. CMMI vs TMMi

| 항목 | CMMI | TMMi |
|:---:|:---|:---|
| **범위** | 개발 전 공정 (분석~배포) | **테스트 공정 특화** |
| **주체** | 개발 조직 전체 | **테스트/QA 조직 중심** |
| **관계** | 상호 보완적 (CMMI 레벨과 TMMi 레벨은 대략 비례함) | |
| **목표** | 프로젝트 성공 및 성숙도 | **결함 제거 효율 및 테스트 자산화** |

### 2. 기술적 시너지: 테스트 자동화 (Test Automation)
TMMi 레벨 3 이상의 조직은 테스트 자동화를 단순한 '도구 사용'이 아닌 **'재사용 가능한 자산'**으로 관리합니다. 자동화 스크립트의 생명주기를 관리하고, 회귀 테스트의 범위를 최적화하는 전략이 TMMi 체계 안에서 작동합니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 전사 테스트 센터(TCoE) 구축
- **상황**: 프로젝트마다 테스터의 실력이 다르고 테스트 방식이 제각각임.
- **결단**: **TMMi 레벨 3 인증을 목표로 한 표준화**. 
- **판단**: 각 팀의 테스트 계획서 양식을 통일하고, '테스트 설계 기법(경계값 분석 등)' 교육 실시. 
- **효과**: 테스트 전문성 강화로 인해 운영 환경 결함 유출률 40% 감소 및 테스트 케이스 재사용성 2배 향상.

### 📢 기술사적 결언
> "테스트는 개발의 '끝'에 붙는 부속품이 아니라, 개발과 **'병행'**하는 파트너 프로세스다. TMMi는 이 파트너십의 수준을 결정하는 잣대다. 아키텍트는 단순히 버그를 많이 찾는 것에 만족하지 말고, **'어떻게 하면 더 적은 비용으로 치명적 결함을 먼저 찾을 것인가'**를 고민하는 TMMi적 사고를 가져야 한다. 성숙한 테스트 조직이 있는 곳에서만 진정한 '지속적 배포(CD)'가 가능하다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **테스트 효율**: 결함 탐지율(DDP) 향상으로 인한 전체 개발 비용 20% 절감.
- **품질 신뢰**: 정량적 데이터를 통한 시스템 릴리즈 여부(Go/No-Go)의 과학적 판단.

### 미래 전망
미래의 TMMi는 **'AI-Native Testing'**을 수용하는 방향으로 진화할 것입니다. 사람이 테스트 시나리오를 설계하는 단계를 넘어, AI가 비즈니스 변경점을 감지하여 자동으로 성숙도 높은 테스트 스위트를 갱신하는 'Self-maturing Test Process'가 등장할 것입니다. 또한 **Shift-left**와 **Shift-right** 테스팅이 완벽히 통합된 'Full-cycle Quality Governance'가 차세대 TMMi의 핵심 과제가 될 것입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **TMMi**는 우리 학교 축구부가 **"얼마나 연습을 체계적으로 하는지"** 등급을 매기는 거예요.
2. 그냥 공만 차는 게 아니라(레벨 1), 작전을 짜고(레벨 2), 모든 선수가 똑같은 훈련법을 배우고(레벨 3), 체력 수치를 재는(레벨 4) 식이죠.
3. 높은 등급을 받을수록, 실제 경기(서비스 오픈)에서 **실수하지 않고 우승할 확률**이 훨씬 높아진답니다!
