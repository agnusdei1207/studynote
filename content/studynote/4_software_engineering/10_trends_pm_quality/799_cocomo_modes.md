# 799. COCOMO 비용 산정 모드 (Organic, Semi, Embedded)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소프트웨어의 규모(LOC)와 개발 환경의 복잡도에 따라 산정 공식을 차별화하여 적용하는 **경험적 비용 산정 모델**의 핵심 분류 체계이다.
> 2. **3대 모드**: 단순 비즈니스용인 **Organic**, 복잡도가 중간인 **Semi-Detached**, 시스템 제어 등 초고난도인 **Embedded** 모드로 나누어 개발 공수(MM)를 수학적으로 계산한다.
> 3. **가치**: 프로젝트의 성격에 따라 '규모가 커질수록 난이도가 상승하는 비율(지수 b)'을 다르게 설정함으로써, 단순 비례 산정이 가질 수 없는 현실적인 예산과 일정 예측을 가능케 한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "같은 1만 줄이라도 난이도는 다르다"

단순한 가계부 프로그램 1만 줄과, 전투기 제어 프로그램 1만 줄을 만드는 데 드는 비용이 같을까요? 당연히 후자가 훨씬 어렵고 많은 인력이 필요합니다. 배리 보헴은 이를 정량화하기 위해 프로젝트의 **'복잡성 토폴로지'**에 따라 3가지 모드를 정의하고, 각기 다른 상수를 공식에 대입하도록 설계했습니다.

### 💡 비유: 가구 제작의 난이도별 견적

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        COCOMO 3대 모드 비유                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. Organic (유기적 - DIY 선반):                                             │
│     이미 수백 번 만들어본 익숙한 선반. 재료(LOC)가 늘어난 만큼 정직하게 시간도      │
│     늘어남. (쉬운 프로젝트)                                                     │
│                                                                             │
│  2. Semi-Detached (준분리형 - 맞춤형 붙박이장):                               │
│     방 크기도 재야 하고, 가끔 특수 부품도 써야 함. 선반보다는 시간 늘어나는        │
│     속도가 좀 더 가파름. (중간 난이도)                                           │
│                                                                             │
│  3. Embedded (내장형 - 스마트 주방 시스템):                                   │
│     수도, 전기, 가스 배관과 완벽히 연동되어야 하며 한치의 오차도 없어야 함.        │
│     조금만 커져도 난이도가 **폭발적으로(지수적)** 상승함! (초고난도)               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 모드별 상세 비교 및 상수값

기본 공식: $Effort (PM) = a \times (KDSI)^b$

| 모드 명칭 | 시스템 특징 | 규모 (LOC) | 상수 a | **지수 b** |
|:---:|:---|:---:|:---:|:---:|
| **Organic** | 일반 비즈니스 자료 처리, 유틸리티, 익숙한 환경 | 5만 이하 | 2.4 | **1.05** |
| **Semi-Detached** | 트랜잭션 처리, 운영체제 일부, 숙련도 혼재 | 30만 이하 | 3.0 | **1.12** |
| **Embedded** | 실시간 제어, 항공/국방, 엄격한 하드웨어 제약 | 제한 없음 | 3.6 | **1.20** |

- **지수 b의 의미**: 규모가 커질수록 소통 비용과 복잡도가 얼마나 가파르게 상승하는지를 나타냄. (Embedded가 가장 가파름)

### 2. 모드 결정 가이드라인
- **유연성**: 요구사항이 얼마나 자주 바뀌는가?
- **경험**: 팀원이 해당 분야에 얼마나 익숙한가?
- **제약**: 하드웨어나 보안 규제가 얼마나 엄격한가?

### 3. 산정 아키텍처 흐름 (ASCII)

```text
    [ Project Info ] ──▶ [ Select Mode ] ──▶ [ Apply Formula ]
    - Complexity          1. Organic          PM = a * (KDSI)^b
    - Experience          2. Semi-Detached           │
    - Constraints         3. Embedded                ▼
                                              [ Total Man-Month ]
                                                     │
          ┌──────────────────────────────────────────┘
          ▼
    [ Schedule Calc ] ──▶ [ TDEV = c * (PM)^d ] ──▶ [ Project Timeline ]
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 하향식(COCOMO) vs 상향식(FP) 산정
- **COCOMO**: **코드 규모**를 먼저 예측하고 모드를 정해 비용 산출. (시스템 관점)
- **기능점수(FP)**: **사용자 기능**을 먼저 세고 비용 산출. (사용자 가치 관점)
- **시너지**: 초기 기획 단계에선 FP로 가이드라인을 잡고, 상세 설계 후 예상 LOC가 나오면 COCOMO 모드를 적용해 상호 검증(Cross-check)합니다.

### 2. 기술적 시너지: COCOMO II의 진화
799번의 고전 COCOMO는 '폭포수 모델'에 최적화되어 있었습니다. 현대의 **COCOMO II**는 재사용성, 객체지향, 팀 규모 등을 추가 변수로 도입하여 애자일 환경에서도 모드별 특성을 반영할 수 있도록 진화했습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 자율주행 알고리즘 프로젝트 견적
- **상황**: 10만 라인 정도의 자율주행 판단 로직 개발. 
- **결단**: **Embedded 모드 강제 적용**. 
- **판단**: 라인 수는 30만 이하로 Semi-Detached 범위에 해당하지만, 생명과 직결된 실시간성(Real-time)과 하드웨어 제약이 극심하므로 가장 보수적인 **Embedded** 공식 사용.
- **효과**: 초기 예산을 넉넉히 확보하여, 프로젝트 후반부의 복잡성 폭발(Complexity Explosion) 상황에서도 인력 부족 없이 완수.

### 📢 기술사적 결언
> "COCOMO 모드는 프로젝트의 **'유전자'**를 결정하는 작업이다. 단순히 LOC 수치에 매몰되지 말고, 우리 조직의 **'기술적 체력'**과 도메인의 **'치명도'**를 냉정하게 평가하여 모드를 선택해야 한다. 무리하게 Organic으로 낮춰 잡는 '낙관적 편향'은 프로젝트 중반의 일정 파행을 부르는 지름길이다. 아키텍트는 숫자의 뒤에 숨겨진 **'지수(Exponential)의 무서움'**을 경영진에게 경고할 수 있어야 한다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **견적 신뢰도**: 단순 비례 산정 대비 예측 오차 30% 이상 감소.
- **리스크 관리**: 모드별 가중치 적용을 통해 고난도 작업에 대한 자원 예비율 확보.

### 미래 전망
미래의 비용 산정은 **'AI 모드 추천'** 시스템으로 진화할 것입니다. 요구사항 문서를 AI가 읽고 유사 프로젝트의 실패 사례와 성공 사례를 대조하여, 가장 적합한 COCOMO 모드와 보정 계수를 실시간 제안할 것입니다. 또한 **서버리스** 환경처럼 인프라 관리가 사라지는 곳에서는 하드웨어 제약 변수가 줄어들며, 오직 '로직의 순수 복잡도'만을 측정하는 새로운 모드가 등장할 것입니다.
