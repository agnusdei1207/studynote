# 770. 페트리 넷 병행/비동기 시스템 정형 명세

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 칼 아담 페트리(Carl Adam Petri)가 고안한 수학적 모델링 기법으로, 시스템 내에서 발생하는 **병행성(Concurrency), 동기화(Synchronization), 자원 공유** 현상을 시각적이고 정형적으로 표현하는 그래프 도구이다.
> 2. **구성 요소**: 상태를 나타내는 **플레이스(Place)**, 사건을 나타내는 **트랜지션(Transition)**, 그리고 이들을 잇는 아크(Arc)와 상태의 흐름을 나타내는 **토큰(Token)**으로 시스템의 동적 행위를 시뮬레이션한다.
> 3. **가치**: 복잡한 비동기 시스템의 교착 상태(Deadlock)나 기아 상태(Starvation)를 수학적으로 증명할 수 있어, 임베디드, 프로토콜, 유연 생산 시스템(FMS) 등의 고신뢰성 설계에 필수적이다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "동시에 일어나는 일들을 어떻게 묘사할까?"

전통적인 순서도(Flowchart)는 '한 번에 한 일'만 일어나는 시스템을 그리는 데 최적화되어 있습니다. 하지만 현대 시스템은 수많은 프로세스가 동시에 돌아갑니다. "A와 B가 모두 끝나야 C가 시작된다"거나 "자원은 하나인데 둘이 동시에 쓰려 한다"는 복잡한 상황을 완벽하게 묘사하고 수학적으로 검증하기 위해 **페트리 넷 (Petri Net)**이 탄생했습니다.

### 💡 비유: 기차역의 신호기와 선로

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        페트리 넷(Petri Net) 비유                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 기차가 충돌하지 않고 안전하게 역을 통과해야 함.                          │
│                                                                             │
│  1. 플레이스 (Place - 대기 선로): 기차가 머무를 수 있는 장소. (동그라미)          │
│  2. 토큰 (Token - 실제 기차): 지금 어느 선로에 기차가 있는지 나타내는 점.         │
|  3. 트랜지션 (Transition - 신호기): 기차가 다음 선로로 이동하는 사건. (직사각형)   │
│                                                                             │
│  → 규칙: "앞 선로가 비어있고(조건 1), 신호가 바뀌면(조건 2)" 기차가 이동함!        │
│     이 규칙을 따라 점(Token)이 옮겨가는 모습을 보며 **사고가 날지 안 날지**        │
│     미리 계산해보는 마법의 지도!                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 페트리 넷의 4대 구성 요소

| 요소 명칭 | 기호 | 의미 | 상세 설명 |
|:---:|:---:|:---|:---|
| **플레이스** | **○** | **조건 (Condition)**, 상태 | 토큰을 보관하는 장소 (Place) |
| **트랜지션** | **|** | **이벤트 (Event)**, 행위 | 상태를 전이시키는 사건 (Transition) |
| **아크** | **→** | **관계 (Relation)** | 플레이스와 트랜지션을 연결하는 화살표 |
| **토큰** | **●** | **자원 (Resource)** | 플레이스에 놓이며, 시스템의 현재 상태 표시 |

### 2. 실행 규칙: 점화 (Firing)
- **활성화 (Enabled)**: 트랜지션의 모든 입력 플레이스에 토큰이 존재할 때.
- **점화 (Firing)**: 활성화된 트랜지션이 실행되면서 입력 플레이스의 토큰을 제거하고 출력 플레이스에 토큰을 생성함.

### 3. 주요 모델링 패턴 (ASCII)

```text
    [ Parallel (병렬) ]           [ Synchronization (동기) ]
    
       ○ (Place)                     ●      ● (Input Places)
       │                             ↘    ↙
    ───┴─── (Transition)            ───┴─── (Transition)
     ↙   ↘                           │
    ○     ○ (Fork)                   ○ (Join)
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 상태 전이도(STD) vs 페트리 넷(Petri Net)
- **STD**: 한 번에 **하나의 상태**만 가질 수 있음. (순차 시스템)
- **Petri Net**: 토큰이 여러 곳에 있을 수 있어 **동시 다발적인 상태** 표현 가능. (병렬 시스템)

### 2. 기술적 시너지: 정형 검증 (Formal Verification)
페트리 넷으로 모델링된 시스템은 수학적인 행렬 연산을 통해 다음 성질을 검증할 수 있습니다.
- **Liveness**: 시스템이 멈추지 않고 영원히 동작하는가? (Deadlock 방지)
- **Safeness/Boundedness**: 자원(토큰)이 무한정 쌓여 폭발하지 않는가? (Overflow 방지)
- **Reachability**: 우리가 원하는 특정 상태에 도달할 수 있는가?

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 생산 라인 로봇 협업 설계
- **상황**: 로봇 A가 부품을 집어 로봇 B에게 전달해야 함. 서로 엇갈리면 충돌 발생.
- **결단**: **페트리 넷 기반 인터로킹(Interlock) 설계**. 
- **판단**: 로봇 A의 '작업 완료' 플레이스와 로봇 B의 '준비 완료' 플레이스에 모두 토큰이 모여야만 '전달' 트랜지션이 점화되도록 설계.
- **효과**: 시뮬레이션을 통해 어떤 상황에서도 두 로봇이 동시에 자원을 점유하려다 멈추는(Deadlock) 일이 없음을 수학적으로 증명.

### 📢 기술사적 결언
> "페트리 넷은 **'병행성의 언어'**다. 눈에 보이지 않는 비동기 로직의 충돌을 시각화하여 설계자의 실수를 방어한다. 아키텍트는 단순한 기능 구현을 넘어, 시스템이 가진 **'논리적 결함'**을 수학적으로 증명하려는 정형적 태도를 가져야 한다. 특히 자율주행이나 의료 기기처럼 생명이 직결된 분야에서 페트리 넷은 가장 강력한 **'설계 보증 수표'**가 된다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **신뢰성 향상**: 설계 단계에서 논리적 교착 상태 100% 제거 가능.
- **검증 비용 절감**: 구현 후 테스트가 아닌, 모델 단계의 수학적 검증으로 오류 수정 비용 최소화.

### 미래 전망
미래의 페트리 넷은 **'컬러 페트리 넷 (CPN)'**과 **'확률적 페트리 넷'**으로 고도화되어, 데이터의 종류와 시간 흐름까지 정밀하게 묘사할 것입니다. 또한 AI 기반의 **'자동 모델 체크 (Model Checking)'**와 결합하여, 사람이 직접 그래프를 그리지 않아도 소스코드를 읽어 페트리 넷으로 변환하고 취약점을 찾아내는 '자율 검증 시스템'이 구축될 것입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **페트리 넷**은 개미들이 과자를 나르는 **"개미굴 지도"**와 같아요.
2. 과자(토큰)를 나르다가 개미들이 좁은 길에서 마주쳐서 **꼼짝달싹 못 하게(교착)** 되지는 않는지 미리 확인해보는 거죠.
3. 이 지도를 잘 그려두면, 개미들이 부딪히지 않고 **영차영차 끝까지** 과자를 잘 나를 수 있답니다!
