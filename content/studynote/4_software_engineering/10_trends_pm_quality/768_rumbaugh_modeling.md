# 768. 럼바우 객체 모델링 (객체/동적/기능 모델)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제임스 럼바우(James Rumbaugh)가 제안한 **객체 모델링 기술(OMT, Object Modeling Technique)**로, 시스템을 객체, 동적, 기능의 세 가지 관점에서 입체적으로 분석하는 그래픽 모델링 방법론이다.
> 2. **3대 모델**: 정적 구조를 다루는 **객체 모델링**, 상태 변화와 시간 흐름을 다루는 **동적 모델링**, 데이터 변환 과정을 다루는 **기능 모델링**으로 구성된다.
> 3. **가치**: 현대 UML(Unified Modeling Language)의 가장 중요한 학문적 토대가 되었으며, 복잡한 시스템을 '구조-상태-동작'이라는 논리적 체계로 나누어 분석하는 표준을 확립했다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "객체지향 분석의 표준을 세우다"

UML이 등장하기 전, 객체지향 분석 및 설계(OOAD) 방법론은 난립하고 있었습니다. 그중 가장 논리적이고 포괄적이라는 평가를 받은 것이 럼바우의 **OMT**입니다. 럼바우는 시스템을 단순히 객체들의 집합으로 보는 것을 넘어, 그들이 어떻게 변하고(Dynamic) 어떤 계산을 수행하는지(Functional)를 함께 기술해야 한다고 주장했습니다.

### 💡 비유: 영화 제작의 3대 요소

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        럼바우 OMT 3대 모델 비유                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 영화(시스템)를 제작하려 함.                                            │
│                                                                             │
│  1. 객체 모델링 (인물 설정):                                                 │
│     어떤 배우(Object)가 출연하고, 그들의 관계(상속, 연관)는 어떠한가?             │
│     (가장 중요하고 기본이 되는 뼈대!)                                           │
│                                                                             │
│  2. 동적 모델링 (연기 시나리오):                                              │
│     배우가 기쁠 때와 슬플 때 어떻게 행동(State)하는가? 어떤 사건(Event)에         │
│     반응하여 감정이 변하는가?                                                  │
│                                                                             │
│  3. 기능 모델링 (특수 효과 로직):                                             │
│     마법사가 마법을 부릴 때, 지팡이(Input)가 어떻게 불꽃(Output)으로             │
│     변하는가? (데이터의 변환 과정)                                             │
│                                                                             │
│  → 배우를 정하고(객체), 시나리오를 쓰고(동적), 연출(기능)을 하면 영화 완성!        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 럼바우 3대 모델 상세 분석

| 모델 명칭 | 영문 명칭 | 분석 관점 | 주요 도구 (Diagram) | 현대 UML 대응 |
|:---:|:---|:---|:---|:---|
| **객체 모델링** | **Object Modeling** | 시스템의 정적 구조 | **객체 다이어그램** | 클래스 다이어그램 |
| **동적 모델링** | **Dynamic Modeling** | 상태 변화, 시간 흐름 | **상태 다이어그램 (STD)** | 상태 머신 다이어그램 |
| **기능 모델링** | **Functional Modeling** | 데이터 값의 변환 과정 | **자료 흐름도 (DFD)** | 액티비티 다이어그램 |

### 2. 분석 순서 (The Sequence)
럼바우 방법론은 분석의 선후 관계를 명확히 합니다.
1. **객체 모델링 (가장 먼저 수행)**: 시스템의 뼈대 구축.
2. **동적 모델링**: 객체 간의 상호작용 및 생명주기 정의.
3. **기능 모델링**: 상세 알고리즘 및 데이터 가공 로직 정의.

### 3. OMT 모델 간 관계도 (ASCII)

```text
    ┌──────────────────────┐
    │   [ Object Model ]   │ <── 뼈대 (가장 중요)
    └──────────┬───────────┘
               │ (의존)
    ┌──────────▼──────────┐      ┌──────────────────────┐
    │  [ Dynamic Model ]  │ ◀──▶ │  [ Functional Model ]│
    │ (State & Events)    │      │ (Process & Data)     │
    └─────────────────────┘      └──────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 럼바우(OMT) vs 부치(Booch) vs 야콥슨(OOSE)
- **럼바우 (OMT)**: **그래픽 모델링**과 데이터 흐름(DFD) 강조. (UML의 뼈대)
- **부치 (Booch)**: 설계와 구현의 상세화, 마이크로/매크로 프로세스 강조.
- **야콥슨 (OOSE)**: **유스케이스(Use Case)** 중심의 사용자 관점 강조.
- **시너지**: 이 세 거두가 만나 '세 명의 아미고(Three Amigos)'라 불리며 **UML**을 탄생시켰습니다.

### 2. 기술적 시너지: 정형 명세와의 연결
동적 모델링의 '상태 전이' 개념은 이후 **페트리 넷(Petri-Net)**이나 **상태 머신**과 결합하여, 임베디드 시스템의 안전성을 수학적으로 증명하는 정형 기법의 토대가 되었습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 스마트 홈 제어 시스템 분석
- **객체 모델링**: 전등, 온도 조절기, 유저 클래스 도출 및 상속 관계 정의.
- **동적 모델링**: 전등의 'On/Off' 상태와 '타이머 종료' 이벤트에 따른 상태 전이 설계.
- **기능 모델링**: '현재 온도' 데이터가 '희망 온도'와 비교되어 '보일러 가동' 신호로 변하는 로직 기술.
- **효과**: 정적인 구조뿐만 아니라 복잡한 이벤트 처리 로직까지 누락 없이 분석 완료.

### 📢 기술사적 결언
> "럼바우의 OMT는 객체지향을 **'보는 것'에서 '증명하는 것'**으로 승격시켰다. 특히 기능 모델링(DFD)을 객체지향에 포함시킨 것은 데이터의 흐름 또한 시스템의 본질임을 인정한 혜안이다. 현대 아키텍트는 클래스 다이어그램만 그리는 반쪽짜리 설계를 경계해야 한다. 시스템이 '어떤 상태'를 가지며 '데이터를 어떻게 가공'하는지 럼바우의 3대 관점으로 균형 있게 조망할 때, 비로소 완전한 설계가 가능해진다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **분석 완전성**: 3대 모델 교차 검증을 통해 로직 누락률 40% 이상 감소.
- **UML 이해도**: 현대 표준의 뿌리를 이해함으로써 모델링 도구 활용 능력 향상.

### 미래 전망
미래의 모델링은 **'Executable OMT'**로 진화할 것입니다. 럼바우의 3대 모델을 정의하면 코드가 100% 자동 생성되는 **MDA (Model Driven Architecture)** 기술이 AI와 결합하여 고도화될 것입니다. 또한 자율 주행과 같은 복합 시스템 분석을 위해 상태(Dynamic)와 연산(Functional)의 정합성을 AI가 실시간 검증해주는 지능형 모델링 환경이 구축될 것입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **럼바우 모델링**은 복잡한 게임기를 만들 때 **"3가지 방법으로 생각하기"** 약속이에요.
2. 어떤 부품이 있는지(객체), 버튼을 누르면 화면이 어떻게 변하는지(동적), 전기가 들어가서 어떻게 소리가 나오는지(기능)를 **골고루 생각하는 것**이죠.
3. 이 3가지를 다 잘 적어두면, 어떤 어려운 기계라도 **실수 없이 뚝딱** 만들어낼 수 있답니다!
