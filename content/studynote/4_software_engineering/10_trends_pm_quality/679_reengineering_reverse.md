# 679. 소프트웨어 재공학 역공학

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기존의 노후화된 레거시 시스템을 분석하고 구조를 변경하여 새로운 시스템으로 진화시키는 **소프트웨어 유지보수 및 현대화 활동**이다.
> 2. **역공학**: 완성된 제품이나 소스코드로부터 설계 명세서나 모델을 추출해내는 **'거꾸로 분석하는'** 과정으로, 재공학의 가장 중요한 기초 단계가 된다.
> 3. **가치**: 시스템의 유지보수성을 높이고 수명을 연장하며, 문서가 유실된 오래된 시스템의 비즈니스 로직을 복원하여 기술 부채를 해결한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "죽어가는 시스템에 새 생명을"

소프트웨어는 시간이 지남에 따라 '엔트로피'가 증가하여 수정하기 힘든 스파게티 코드가 됩니다. 개발자가 퇴사하고 문서가 사라지면 시스템은 블랙박스가 되어버립니다. 이를 통째로 새로 만드는 것은 비용과 리스크가 너무 큽니다. **재공학(Re-engineering)**은 기존의 가치 있는 로직을 살리면서 낡은 껍데기와 엉킨 구조만 고쳐 쓰는 경제적인 전략입니다.

### 💡 비유: 낡은 한옥 리모델링

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        재공학 및 역공학 비유                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 100년 된 낡은 집을 고쳐서 살고 싶음. (근데 설계도가 없음!)               │
│                                                                             │
│  1. 역공학 (Reverse Engineering):                                            │
│     집의 기둥이 어디 있는지, 벽 속에 전선이 어떻게 지나가는지 벽을 뜯어보며        │
│     그림을 그려냄. (실물 -> 설계도 복원)                                       │
│                                                                             │
│  2. 재공학 (Re-engineering):                                                 │
│     알아낸 설계도를 바탕으로, 썩은 기둥은 바꾸고 구조를 현대식으로 바꿈.           │
│     집의 주소와 주인(핵심 가치)은 그대로지만, 살기 편한 새집이 됨!                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 재공학의 4대 주요 활동

| 단계 | 활동 명칭 | 핵심 내용 |
|:---:|:---|:---|
| **Step 1** | **분석 (Analysis)** | 대상 시스템 식별 및 범위 확정 |
| **Step 2** | **역공학 (Reverse)** | 소스코드로부터 **설계 정보(UML, ERD 등) 추출** |
| **Step 3** | **재구조화 (Restructuring)** | 기능 변경 없이 내부 구조만 개선 (Refactoring 포함) |
| **Step 4** | **순공학 (Forward)** | 개선된 설계를 바탕으로 실제 코드 구현 및 배포 |

### 2. 역공학(Reverse Engineering)의 수준
- **Logic Reverse**: 코드 흐름 파악.
- **Data Reverse**: DB 스키마 및 관계 추출.
- **Design Reverse**: 아키텍처 및 모듈 간 관계 복원.

### 3. 재공학 프로세스 순환 모델 (ASCII)

```text
    [ Legacy Code ] ──────▶ [ Reverse Engineering ] ──────▶ [ Design Models ]
                                (분석 및 추출)                   (복원된 문서)
                                                                    │
          ▲                                                         │
          │                                                         ▼
    [ Modern System ] ◀───── [ Forward Engineering ] ◀───── [ Restructuring ]
                                (신규 기술 적용)                 (구조 개선)
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 재개발 vs 재공학

| 항목 | 전부 재개발 (Replacement) | 재공학 (Re-engineering) |
|:---:|:---|:---|
| **리스크** | 매우 높음 (기존 로직 유실 위험) | 낮음 (검증된 로직 유지) |
| **비용** | 높음 | 상대적 낮음 |
| **기간** | 장기 | 단기/중기 |
| **결과물** | 완전히 새로운 시스템 | 진화된 기존 시스템 |

### 2. 기술적 시너지: 리팩토링(Refactoring)
리팩토링은 재공학의 **'재구조화(Restructuring)'** 단계에서 핵심적인 기법으로 사용됩니다. 작은 단위의 코드 정리는 리팩토링이라 부르고, 시스템 전반의 패러다임(예: 메인프레임 → 클라우드) 전환을 포함하는 큰 범위의 활동을 재공학이라 합니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 금융권 차세대 '시스템 전환'
- **상황**: 20년 된 COBOL 시스템을 Java Spring Boot 기반으로 바꿔야 함. 문서 없음.
- **결단**: **역공학 도구 활용 및 단계적 재공학**. 
- **판단**: 먼저 상용 역공학 도구로 코볼 코드를 읽어 비즈니스 룰을 추출. 이후 **스트랭글러 패턴(Strangler Pattern)**을 적용하여 핵심 기능부터 하나씩 Java 서비스로 재공학하여 교체.
- **효과**: 빅뱅 방식의 전환 실패 리스크를 최소화하고, 구식 언어 전문가 부족 문제 해결.

### 📢 기술사적 결언
> "재공학은 단순한 코드 변환이 아니라 **'지식의 추출과 보존'**이다. 역공학을 통해 우리가 얻어야 하는 것은 소스코드가 아니라, 그 안에 숨겨진 **'비즈니스 인텔리전스'**다. 미래의 소프트웨어 엔지니어링은 '새로 만드는 기술'만큼이나 '오래된 것을 다스리는 기술'이 생존의 열쇠가 될 것이다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **유지보수성 향상**: 복잡도(McCabe) 지수 평균 40% 이상 감소.
- **비용 절감**: 신규 개발 대비 약 30~50%의 비용으로 현대화 달성.

### 미래 전망
미래의 재공학은 **'AI-Native Re-engineering'**으로 진화합니다. 사람이 수동으로 로직을 분석하는 대신, 거대 언어 모델(LLM)이 수백만 줄의 레거시 코드를 읽어 "이 로직의 의도는 이것입니다"라고 설명해주고, 즉시 현대적인 언어(Rust, Kotlin 등)로 변환한 뒤 테스트 코드까지 자동 생성하는 '자율적 현대화' 시대가 열리고 있습니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **역공학**은 완성된 장난감 자동차를 **나사 하나하나 풀어서** 어떻게 만들어졌는지 공부하는 거예요.
2. **재공학**은 그렇게 알아낸 지식으로, 낡은 플라스틱 바퀴를 **튼튼한 고무 바퀴로 바꾸고** 엔진도 더 빠르게 업그레이드하는 거죠.
3. 아예 새 차를 사는 것보다 **내가 아끼던 차를 더 좋게 고쳐서** 계속 타는 아주 멋진 방법이랍니다!
