# 743. 데이터 마스킹 FPE 암호 유지

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 민감한 데이터를 식별할 수 없도록 변환하여 보호하는 기술로, 일부를 가리는 **데이터 마스킹**과 원본의 형식을 그대로 유지하면서 암호화하는 **FPE (Format-Preserving Encryption)**가 핵심이다.
> 2. **사용성 보장**: 데이터의 길이와 형식을 보존하므로, 데이터베이스 스키마나 애플리케이션 로직의 대대적인 수정 없이도 암호화된 데이터를 활용한 검색, 정렬 및 통계 분석이 가능하다.
> 3. **가치**: 운영 데이터의 테스트/분석 환경 이관 시 개인정보 유출 위험을 원천 차단하고, 토큰화(Tokenization) 기술과 결합하여 보안성과 데이터 활용성 사이의 트레이드오프를 해결한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "보호하면서도 써야 하는 딜레마"

전통적인 암호화(AES 등)는 평문을 알아볼 수 없는 바이너리 덩어리로 만듭니다. 이 과정에서 데이터의 길이와 형식이 변하므로, DB 설계나 앱 코드를 고쳐야 하는 막대한 비용이 듭니다. 또한 암호화된 상태로는 '끝자리 검색'이나 '통계'가 불가능합니다. **데이터 마스킹**과 **FPE**는 "데이터의 성질(Format)은 살리고 내용은 죽이는" 고도의 보안 기술입니다.

### 💡 비유: 가짜 돈과 영화 촬영 소품

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                        데이터 마스킹 및 FPE 비유                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [상황] 진짜 보물(민감 데이터)을 안전하게 다뤄야 함.                             │
│                                                                             │
│  1. 데이터 마스킹 (부분 가리기):                                              │
│     보물 지도에서 정확한 위치만 포스트잇으로 가림.                              │
│     - "서울시 강남구 ***" (내용은 알 수 없지만, 서울이라는 건 알 수 있음)         │
│                                                                             │
│  2. FPE (모조품 만들기):                                                     │
│     진짜 다이아몬드 대신, 똑같은 크기와 모양의 '유리 가짜 다이아'를 만듦.          │
│     - 무게와 크기(Format)가 같아서 전시용 상자(DB 스키마)를 바꿀 필요가 없음.      │
│     - 멀리서 보면 진짜 같지만, 훔쳐가도 아무 가치가 없음!                        │
│                                                                             │
│  → 겉모양은 그대로인데 **속내용만 안전하게** 바꾸는 변신술!                     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 데이터 마스킹 (Data Masking) 유형
- **정적 마스킹 (Static)**: 원본 DB를 복제할 때 데이터를 영구적으로 변환. (테스트 환경용)
- **동적 마스킹 (Dynamic)**: 사용자가 조회하는 순간 권한에 따라 실시간 변환. (운영 환경용)

### 2. FPE (Format-Preserving Encryption) 메커니즘
- **정의**: 암호화 전후의 데이터 **도메인(문자 집합, 길이)이 동일**함을 보장하는 암호화 방식. (NIST SP 800-38G 표준)
- **핵심 알고리즘**: FF1, FF3-1 (AES 라운드 기반의 Feistel 네트워크 활용).
- **특징**: "123-456" (카드번호) → "892-104" (FPE 암호문). 여전히 카드번호 형식이므로 DB 제약 조건 위배 안 함.

### 3. 기술 비교 분석 (Comparison)

| 구분 | 일반 암호화 (AES) | 데이터 마스킹 | FPE |
|:---:|:---|:---|:---|
| **복호화** | 가능 | **불가능 (단방향)** | **가능 (키 필요)** |
| **형식 유지** | 불가능 (바이너리) | 가능 | **완벽 유지** |
| **검색 가능성** | 낮음 | 부분적 가능 | **매우 높음 (일치 검색)** |
| **주요 용도** | 데이터 저장/전송 보안 | **통계, 고객 상담 화면** | **레거시 시스템 통합, 분석** |

---

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 토큰화 (Tokenization) vs FPE
- **토큰화**: 데이터를 임의의 난수(Token)로 바꾸고 맵핑 테이블을 별도로 관리. 보안성은 극강이나 관리 부담 큼.
- **FPE**: 수학적 공식을 통해 변환. 별도 테이블 없이 '키'만으로 복호화 가능. 시스템 부하 적음.

### 2. 기술적 시너지: 데이터 합성 (Synthetic Data)
마스킹이 실제 데이터를 변형하는 것이라면, 데이터 합성은 실제 데이터의 통계적 특성만 따서 아예 새로운 가짜 데이터를 만드는 기술입니다. AI 학습 시 마스킹된 데이터의 한계를 극복하기 위해 FPE와 데이터 합성을 병행합니다.

---

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 차세대 시스템 테스트 데이터 구축
- **상황**: 실제 운영 DB(1억 건)를 개발 서버로 옮겨서 성능 테스트를 해야 함.
- **결단**: **정적 마스킹 및 FPE 적용**. 
- **판단**: 
    1. 이름, 주민번호 뒷자리는 '***'로 마스킹. 
    2. 카드번호와 전화번호는 로직 검증을 위해 **FPE** 적용. (형식이 틀리면 프로그램이 에러 나기 때문)
- **효과**: 개발자는 실제와 똑같은 데이터로 완벽하게 테스트하고, 보안 담당자는 데이터 유출 걱정 종료.

### 📢 기술사적 결언
> "데이터 보안의 완성은 **'필요할 때만 보여주는 것'**이다. 마스킹과 FPE는 데이터를 '흉물스러운 암호'가 아닌 '우아한 가짜'로 만드는 기술이다. 아키텍트는 보안 정책 수립 시, **'데이터의 생애주기별 노출 등급'**을 정의하고, 상담원에게는 마스킹을, 분석 시스템에는 FPE를 적용하는 **'맞춤형 보호 전략'**을 구사해야 한다."

---

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **보안 사고 예방**: 테스트 환경을 통한 개인정보 유출 사고 100% 차단.
- **개발 비용 절감**: 암호화 도입에 따른 DB 및 앱 수정 비용 70% 이상 절감.

### 미래 전망
미래의 데이터 보호는 **'자기 방어형 데이터 (Self-defending Data)'**로 진화합니다. 데이터 자체에 접근 정책이 내장되어, 접근하는 주체의 권한에 따라 데이터가 스스로 마스킹되거나 FPE 암호문으로 변하는 지능형 객체 모델이 대세가 될 것입니다. 또한 양자 암호 기술이 FPE에 이식되어, 양자 컴퓨터로도 깰 수 없는 '완벽한 형식 보존 암호' 시대가 열릴 것입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. **데이터 마스킹**은 이름표에서 내 이름만 **검은색 매직으로 슥슥** 지우는 거예요.
2. **FPE**는 내 전화번호를 **모양은 똑같지만 다른 숫자**로 바꾸는 마법이죠. (예: 010-1234 → 010-9999)
3. 이렇게 하면 내 정보를 몰래 훔쳐보려는 나쁜 사람을 **골탕 먹이면서도**, 우리끼리는 재미있게 놀 수 있답니다!
