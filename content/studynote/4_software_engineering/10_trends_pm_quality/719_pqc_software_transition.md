# 719. 양자 컴퓨팅 대비 PQC 소프트웨어 구조 전환

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 양자 컴퓨터의 강력한 연산 능력(Shor 알고리즘 등)으로 무력화될 수 있는 기존 공개키 암호 체계(RSA, ECC)를 대신하여, 양자 공격에도 안전한 **양자 내성 암호(PQC, Post-Quantum Cryptography)**로 전환하는 보안 진화 전략이다.
> 2. **전환 아키텍처**: 암호 알고리즘을 하드코딩하지 않고 필요에 따라 신속하게 교체할 수 있는 **암호 민첩성(Crypto-agility)** 구조를 확보하고, 기존 암호와 PQC를 혼용하는 하이브리드(Hybrid) 방식을 적용한다.
> 3. **가치**: 미래의 'Q-Day(양자 컴퓨터가 현재 암호를 깨는 날)'에 대비하여 국가 및 기업의 기밀 데이터를 선제적으로 보호하고, "지금 수집하고 나중에 해독(Harvest Now, Decrypt Later)"하는 공격으로부터 과거 데이터를 수호한다.

---

## Ⅰ. 개요 (Context & Background)

### 배경: "보안의 시한폭탄, 양자 컴퓨터"

양자 컴퓨터는 기존 슈퍼컴퓨터로 수백 년 걸릴 암호 해독을 단 몇 분 만에 끝낼 수 있습니다. 현재 우리가 쓰는 인터넷 뱅킹, 공인인증서, 메신저 암호화는 모두 양자 컴퓨터 앞에 무방비 상태가 됩니다. **PQC (Post-Quantum Cryptography)**는 양자 컴퓨터가 잘 풀지 못하는 복잡한 수학 문제(격자, 부호, 다변수 등)를 기반으로 하여, 일반 컴퓨터에서도 돌아가면서 양자 공격을 막아내는 차세대 암호입니다.

### 💡 비유: 마법의 열쇠와 초강력 자물쇠

```text

│                        PQC 및 구조 전환 비유                                  │

│                                                                             │
│  [상황] 도둑이 모든 문을 한 번에 여는 "마법의 마스터키(양자 컴퓨터)"를 만듦.      │
│                                                                             │
│  1. 기존 보안 (일반 자물쇠): 마법 키 하나면 전 세계 모든 집이 다 털림.           │
│                                                                             │
│  2. PQC 전환 (신형 디지털 도어락): 마법 키가 통하지 않는 새로운 방식의           │
│     자물쇠로 교체함.                                                           │

│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. PQC의 4대 주요 수학 모델 (NIST 표준 기반)
- **Lattice-based (격자 기반)**: n차원 격자의 최단 벡터 문제 활용 (가장 유망, Kyber/Dilithium).
- **Code-based (부호 기반)**: 오류 교정 부호의 복잡성 활용 (McEliece).
- **Multivariate-based (다변수 기반)**: 다변수 이차 방정식을 푸는 난해함 활용.
- **Hash-based (해시 기반)**: 일방향 해시 함수의 안전성 활용 (서명 전용).

### 2. PQC 소프트웨어 구조 전환 3대 전략 (Crypto-agility)

| 전략 | 설명 | 기대 효과 |
|:---:|:---|:---|
| **알고리즘 추상화** | 암호 로직을 비즈니스 코드와 분리하여 Interface/Provider 화 | 코드 수정 없이 설정만으로 암호 교체 |
| **하이브리드 모드** | 기존 암호(RSA/ECC)와 PQC를 이중으로 중첩 적용 | 한쪽이 뚫려도 최소한의 보안성 유지 |
| **키 길이/성능 최적화** | PQC의 긴 키 사이즈와 연산 부하를 견딜 수 있는 인프라 증설 | 네트워크 지연 및 메모리 고갈 방지 |

### 3. 전환 아키텍처 구조도 (ASCII)

```text
    [ Application Logic ]
           │ (Abstract Call)
    ┌──────▼────────────────────────────────┐
    │      Crypto Agility Layer (Interface) │ <── 정책에 따라 동적 전환
    └──────┬────────────────┬───────────────┘
           │                │
    ┌──────▼──────┐  ┌──────▼──────┐
    │ Classic Provider │  │  PQC Provider  │ <── Crystals-Kyber 등
    │ (RSA, ECDSA)│  │ (Lattice, Code)│
    └─────────────┘  └──────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 양자 암호 통신(QKD) vs 양자 내성 암호(PQC)

| 항목 | 양자 암호 통신 (QKD) | 양자 내성 암호 (PQC) |
|:---:|:---|:---|
| **핵심 기술** | 양자의 물리적 성질 (광자) | **수학적 복잡도 (알고리즘)** |
| **필요 장비** | 전용 하드웨어, 광케이블 필수 | **기존 컴퓨터/네트워크 그대로 사용** |
| **확장성** | 낮음 (물리적 거리 제한) | **매우 높음 (소프트웨어 패치만으로 가능)** |
| **주 용도** | 국가망, 전용선 보안 | **일반 인터넷, 전자서명, 모바일 앱** |

### 2. 기술적 시너지: SBOM (Software Bill of Materials)
어느 소프트웨어 모듈에 취약한 암호가 들어있는지 파악하기 위해 **SBOM**이 필수입니다. SBOM을 통해 "GPL 라이선스 확인"뿐만 아니라 "Quantum-vulnerable 알고리즘 사용 여부"를 실시간으로 탐지하여 PQC 전환 리스트를 작성합니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 적용 시나리오: 국가 행정망 암호 체계 현대화
- **상황**: 10년 이상 보관해야 하는 국가 기밀 데이터가 양자 공격에 노출될 위기.
- **결단**: **하이브리드 PQC 전환 로드맵 수립**. 
- **판단**: 
    1. 현재 암호 체계 영향도 분석 (Inventory).
    2. TLS 통신 구간에 하이브리드 모드(ECDHE + Kyber) 적용. 
    3. 데이터 저장소 암호화 모듈을 인터페이스 기반으로 리팩토링하여 '암호 민첩성' 확보.
- **효과**: 향후 양자 컴퓨터 상용화 시점에 코드 재작성 없이 설정 변경만으로 즉시 대응 체계 완성.

### 📢 기술사적 결언
> "PQC로의 전환은 '선택'이 아닌 **'생존'**의 문제다. 특히 데이터는 '지금' 털려도 '나중'에 해독될 수 있다는 점(Harvest Now, Decrypt Later)이 무서운 지점이다. 아키텍트는 암호 알고리즘을 소스코드에 박아넣는 구시대적 관습을 버리고, 보안 정책에 따라 언제든 알고리즘을 갈아 끼울 수 있는 **'유연한 암호 추상화 계층'**을 설계의 표준으로 삼아야 한다."

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량적 기대효과
- **미래 안전성**: 양자 컴퓨터 시대에도 데이터 기밀성 유지 보장.
- **전환 비용 절감**: 암호 민첩성 확보 시, 알고리즘 교체 비용 80% 이상 절감.

### 미래 전망
미래에는 모든 브라우저와 OS 커널에 PQC가 기본 탑재될 것입니다. 특히 **'양자 내성 칩(PQC Hardware Accelerator)'**이 CPU 내부에 기본 내장되어, 긴 키 길이로 인한 성능 저하 문제도 해결될 것입니다. 소프트웨어 공학의 품질 속성 중 '보안성'의 정의는 이제 '양자 내성'을 포함하는 개념으로 완전히 재편될 것입니다.
