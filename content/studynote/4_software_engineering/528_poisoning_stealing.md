# 528. 데이터 중독 공격 (Poisoning) 및 모델 탈취 공격 (Stealing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 중독 공격 (Data Poisoning)은 학습 단계에서 **악성 데이터를 주입하여 모델의 판단 기준을 타락**시키는 공격이며, 모델 탈취 공격 (Model Stealing)은 추론 API를 반복 호출하여 **원본 모델과 유사하게 작동하는 복제 모델을 생성**하는 지적 재산권 침해 공격이다.
> 2. **가치**: 인공지능의 수명 주기 중 '학습(Training)'과 '자산(Asset)'의 보안을 다루며, 공급망 공격이나 상업적 모델 무단 복제와 같은 고차원적 비즈니스 리스크를 차단하는 데 필수적인 방어 지식이다.
> 3. **융합**: 데이터 무결성 검증, 차분 프라이버시(DP), 모델 워터마킹 기술과 결합되며, 현대 아키텍처에서는 MLOps 전 과정에 보안 필터링을 적용하여 오염된 데이터의 유입을 막고 모델의 고유성을 보호한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 
  - **데이터 중독 (Data Poisoning)**: 공격자가 학습 데이터셋의 일부를 조작하여, 모델이 특정 조건(Backdoor)에서만 오답을 내게 하거나 전체 성능을 떨어뜨리는 기법이다.
  - **모델 탈취 (Model Stealing)**: 타겟 모델에 수많은 질문(Query)을 던지고 그 답변(Label)을 받아 자기 모델의 학습 데이터로 사용하여, 원본의 지능을 그대로 복사하는 기법이다.

- **💡 비유**: 
  - **데이터 중독**: 요리사(AI 모델)에게 요리를 가르칠 때, 몰래 설탕통에 소금을 섞어놓아(악성 데이터) 나중에 요리사가 모든 음식을 짜게 만들게 유도하는 것과 같습니다.
  - **모델 탈취**: 맛집 주방장에게 레시피를 물어보는 대신, 매일 음식을 사 먹어보고(API 호출) 들어간 재료와 조리법을 유추하여 똑같은 맛을 내는 가게를 옆에 차리는 것과 같습니다.

- **등장 배경**:
  1. **사용자 참여형 학습의 증가**: 온라인 피드백이나 크라우드 소싱 데이터를 실시간 학습(Online Learning)에 반영하면서 데이터 오염 위험이 커졌다.
  2. **AI의 비즈니스 자산화**: 기업들이 막대한 비용을 들여 만든 모델이 경쟁사에 의해 단 몇 달러의 API 호출 비용으로 복제되는 리스크가 발생했다.
  3. **공급망 보안의 확장**: 오픈소스 데이터셋이나 사전 학습된(Pre-trained) 모델을 무비판적으로 가져다 쓰면서 '독이 든 성배'를 마시는 사례가 늘어났다.

- **📢 섹션 요약 비유**: 스파이가 학교 교과서를 조작하여 학생들을 세뇌시키거나(중독), 수석 졸업생의 노트를 몰래 베껴서 똑같은 실력을 갖추는(탈취) 지능적 범죄에 대응하는 보안 전략입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 데이터 중독 공격 (Data Poisoning)

| 공격 유형 | 상세 원리 | 공격 목표 | 비유 |
|:---|:---|:---|:---|
| **가용성 공격 (Availability)** | 전체 데이터셋의 품질을 저하시킴 | 모델을 무용지물로 만듦 | 요리 학교 전체를 엉망으로 만들기 |
| **타겟 공격 (Targeted)** | 특정 인물이나 사례에 대해서만 오판 유도 | 스팸 필터 우회, 안면 인식 통과 | 특정 인물만 감옥에서 풀어주기 |
| **백도어 공격 (Backdoor)** | 특정 트리거(점, 무늬) 입력 시에만 오작동 | 평소엔 정상이지만 신호 시 돌변 | 최면 암호를 걸어두기 |

---

### 2. 모델 탈취 공격 (Model Stealing) 및 방어 아키텍처

공격자가 API를 통해 지능을 빨아들이고, 시스템이 이를 방어하는 구조를 시각화한다.

```text
 ┌─────────────────────────────────────────────────────────────────────────────┐
 │                      모델 탈취 공격 시나리오 및 방어 메커니즘                     │
 ├─────────────────────────────────────────────────────────────────────────────┤
 │                                                                             │
 │ [공격자 (Attacker)]          [타겟 모델 API (Victim)]         [복제 모델 생성]    │
 │        │                          │ (Original Intelligence)      │          │
 │  1. 수만 번의 Query ─────────────▶  │                           │          │
 │     (이미지, 텍스트)                 │ 2. 정답(Label) 반환         │          │
 │        │ ◀────────────────────────  │                           │          │
 │        │                            │                           │          │
 │  3. 수집된 [Q, A] 쌍으로 자기 모델 학습 ─────────────────────────▶ │ [Cloned] │  │
 │                                                                  └──────────┘  │
 │  ┌───────────────────────────────────────────────────────────────────────┐  │
 │  │ ✅ 방어 기술:                                                            │  │
 │  │ - 속도 제한 (Rate Limiting): 짧은 시간 대량 호출 차단                      │  │
 │  │ - 답변 노이즈 주입: 정답 점수를 미세하게 변조하여 학습 방해                  │  │
 │  │ - 모델 워터마킹: 특정 입력에 대해 독특한 반응을 하도록 설계하여 복제 증명      │  │
 │  └───────────────────────────────────────────────────────────────────────┘  │
 │                                                                             │
 └─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 모델 탈취의 핵심은 **결과값의 정보량**이다. 공격자가 데이터를 보내면 모델은 "이것은 고양이일 확률 98%, 강아지 2%" 식의 상세한 확률값(Soft Label)을 주는데, 이 숫자들이 모델의 내부 지능을 고스란히 담고 있다. 공격자는 이 데이터를 모아 자신의 모델을 '증류(Distillation)' 학습시킨다. 이를 막기 위해 서버는 1) **속도 제한**을 걸어 대량 수집을 막고, 2) 확률값 대신 최종 카테고리(Hard Label)만 주거나 미세한 오차를 섞어 학습 효율을 떨어뜨린다. 3) 또한 **워터마킹**을 통해, 나중에 복제된 모델이 발견되었을 때 우리 모델의 자식임을 법적으로 증명할 수 있는 장치를 마련한다.

---

### 심층 동작 원리: 중독 데이터 탐지 (Data Sanitization)

학습 전 '독'을 가려내는 기술이다.
1. **Outlier Detection**: 데이터셋의 전체적인 분포에서 너무 멀리 떨어진 데이터(이상치)를 제거한다.
2. **Influence Analysis**: 특정 데이터가 모델 학습에 기여하는 '영향력'을 계산하여, 한 개가 전체를 뒤흔든다면 중독 데이터로 의심한다.
3. **가치**: "나쁜 친구(데이터) 한 명이 반 전체 분위기를 망치지 못하게" 미리 가려내는 선별 작업이다.

- **📢 섹션 요약 비유**: 급식 재료를 검수할 때 색깔이 너무 이상하거나 냄새가 고약한 식재료(이상치)를 미리 버려서 집단 식중독(중독 공격)을 예방하는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 데이터 중독 vs 적대적 공격 (Adversarial Attack)

공격이 발생하는 시점과 방식의 차이를 이해해야 한다.

| 비교 항목 | 데이터 중독 (Poisoning) | 적대적 공격 (Adversarial) |
|:---|:---|:---|
| **발생 단계** | **학습 단계 (Training)** | **추론 단계 (Inference)** |
| **공격 수단** | 학습 데이터셋 오염 | 입력 데이터에 노이즈 추가 |
| **목적** | 모델의 지능 자체를 타락시킴 | 모델을 일시적으로 속임 |
| **지속성** | **영구적** (재학습 전까지 유지) | 일회성 (해당 입력에만 반응) |
| **비유** | 뇌를 세뇌시킴 | 안경에 스티커를 붙임 |

---

### 기술 융합: 차분 프라이버시 (DP)와 모델 보안

차분 프라이버시는 개인정보 보호뿐만 아니라 모델 보안에도 기여한다.
- **Poisoning 방어**: 학습 시 개별 데이터의 영향력을 제한하는 DP 기술을 적용하면, 소수의 중독 데이터가 모델 전체를 오염시키는 것을 수학적으로 방지할 수 있다.
- **Stealing 방어**: 응답 결과에 DP 노이즈를 섞으면 공격자가 정교한 지능을 복제해가는 효율을 극적으로 낮춘다.

- **📢 섹션 요약 비유**: 개별 학생의 목소리가 너무 크지 않게 마이크 볼륨(DP)을 조절하여, 한 명의 거짓말쟁이(중독 데이터)가 전체 합창(모델)을 망치지 못하게 하는 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 자율 학습형 스팸 메일 필터의 보안 설계

- **문제 상황**: 사용자가 "이 메일은 스팸이 아님"이라고 신고하면 모델이 실시간 학습하는데, 해커들이 이를 이용해 악성 메일을 정상 메일로 인식하도록 모델을 오염시키고 있음.
- **의사결정 (종합 방어 전략)**:
  1. **신뢰 등급제 (Reputation)**: 가입 기간이 짧거나 활동이 적은 사용자의 피드백은 학습 가중치를 낮게 설정하여 중독 위험을 줄인다.
  2. **Golden Dataset 대조**: 학습 후 모델의 판단 결과가 사전에 정의된 '절대 변하지 않는 정답지(Golden Set)'와 크게 달라지면 배포를 즉시 중단한다.
  3. **Honey-pot 쿼리 탐지**: 모델 탈취를 방지하기 위해, 짧은 시간 내에 의미 없는 무작위 메일 데이터를 대량으로 넣어보는 계정을 실시간 차단한다.
  4. **Model Watermarking**: 필터 모델 내부에 특정 단어(예: 'Pink Elephant')가 포함된 메일은 반드시 스팸으로 처리하도록 비밀 로직을 심어두어, 경쟁사가 필터를 복제했을 때 증거로 활용한다.

### 도입 체크리스트
- **Lineage**: 학습 데이터의 출처(Origin)와 변경 이력이 완벽하게 추적(Data Lineage)되고 있는가?
- **Robustness**: 중독 데이터가 5~10% 섞여도 모델 성능이 유지되는지 테스트했는가?
- **API Defense**: 외부 노출 API에서 확률값 대신 순위(Rank)나 카테고리만 반환하도록 설정했는가?

### 안티패턴
- **온라인 학습 맹신**: 검증 없이 실시간 데이터를 모델에 즉시 반영하는 행위. 공격자에게 모델 조종권을 넘겨주는 것과 같다.
- **무제한 무료 API**: 호출 횟수 제한 없는 API 제공은 모델 탈취를 위한 고속도로를 깔아주는 격이다.

- **📢 섹션 요약 비유**: "누구나 편집할 수 있는 백과사전"이 편리하긴 하지만, 나쁜 의도를 가진 사람이 거짓 정보를 잔뜩 적어놓을 수 있다는 점(중독)을 항상 경계하고 검수해야 합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 세부 내용 |
|:---|:---|:---|
| **정량적 (Security)** | 모델 오염 사고 예방 | 데이터 정제 및 영향력 분석을 통해 악성 데이터 유입 90% 차단 |
| **정량적 (Legal)** | 지식 재산권 보호 | 워터마킹 및 API 통제를 통해 모델 무단 복제에 따른 매출 손실 방지 |
| **정성적 (Reliability)** | 서비스 신뢰성 유지 | 공격자의 의도와 상관없이 일관된 AI 판단 성능 유지 |
| **정성적 (Standard)** | MLOps 보안 표준 확립 | 데이터 수집부터 모델 배포까지의 안전한 거버넌스 체계 구축 |

### 미래 전망
- **Blockchain for Data Provenance**: 학습 데이터의 모든 이동과 수정을 블록체인에 기록하여 위변조가 절대 불가능한 '청정 데이터 공급망'이 구축될 것이다.
- **Self-Healing Models**: 중독된 부분을 스스로 감지하고 해당 신경망 가중치만 초기화하여 정상 상태로 복구하는 자가 치유 AI 기술이 등장할 것이다.

- **NIST AI RMF (Risk Management Framework)**: 학습 데이터 보호 및 모델 보안 가이드
- **ISO/IEC 27090**: AI 보안 위협 대응 지침

- **📢 섹션 요약 비유**: 과거에는 컴퓨터의 하드웨어와 소프트웨어를 지켰다면, 이제는 컴퓨터가 학습하는 '지식의 순수성'과 '지능의 고유성'을 지키는 고차원 보안의 시대가 되었습니다.
