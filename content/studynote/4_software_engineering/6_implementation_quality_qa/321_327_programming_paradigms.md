# 321-327. 프로그래밍 패러다임 (OOP, FP, Reactive)

> **핵심 인사이트**: 프로그래밍은 단순히 코드를 짜는 것이 아니라 '사고의 방식'을 결정하는 것이다. 사물의 관계를 정의하는 객체지향(OOP)을 넘어, 수학적 순수함을 추구하는 함수형(FP)과 데이터의 변화에 민첩하게 반응하는 반응형(Reactive) 패러다임의 융합이 현대 소프트웨어의 성능과 안정성을 좌우한다.

---

## Ⅰ. 객체지향 프로그래밍 (OOP)
실세계를 객체(Object) 단위로 모델링하여 프로그램을 구성하는 방식입니다.

* **4대 특징**:
  1. **캡슐화 (Encapsulation)**: 데이터와 연산을 하나로 묶고 내부 상세 구현을 은닉.
  2. **상속 (Inheritance)**: 부모의 특성을 자식이 물려받아 재사용.
  3. **다형성 (Polymorphism)**: 동일한 인터페이스로 서로 다른 객체의 동작을 수행. (Overloading, Overriding)
  4. **추상화 (Abstraction)**: 불필요한 부분은 빼고 핵심적인 특징만 추출.

---

## Ⅱ. 함수형 프로그래밍 (FP)
상태 변경과 가변 데이터를 피하고 **순수 함수**를 조합하여 프로그래밍하는 방식입니다.

* **핵심 특징**:
  - **순수 함수 (Pure Function)**: 부작용(Side Effect)이 없고, 같은 입력엔 항상 같은 출력.
  - **불변성 (Immutability)**: 한 번 생성된 데이터는 변하지 않음.
  - **일급 객체 (First-class Object)**: 함수를 변수에 담거나 인자로 전달 가능.
  - **지연 평가 (Lazy Evaluation)**: 결과가 진짜 필요할 때까지 계산을 미룸.

---

## Ⅲ. 반응형 프로그래밍 (Reactive Programming)
데이터 스트림과 변화의 전파에 중점을 둔 프로그래밍 패러다임입니다.

* **원리**: 데이터가 변경될 때마다 이를 관찰하는 구독자들에게 즉시 알림을 보내 처리하는 방식. (Observer 패턴의 확장)
* **사례**: 실시간 채팅, 주식 차트, 복잡한 사용자 인터페이스(UI) 상호작용.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[패러다임별 중심 가치]

  [ OOP ] ──> "관계와 상태" (객체 간의 메시지 교환)
  [  FP ] ──> "값과 계산" (순수 함수의 조합)
  [ Reactive ] ──> "흐름과 반응" (데이터 스트림 처리)
```

📢 **섹션 요약 비유**: **OOP**는 연극 무대의 배우(객체)들이 각자의 대본(메서드)을 가지고 서로 대화하는 연극과 같습니다. **FP**는 수학 공식을 풀듯이 입력값을 넣으면 오직 계산 결과만 나오는 '마법의 계산기'이며, **반응형 프로그래밍**은 도미노처럼 하나가 쓰러지면(데이터 변화) 연결된 모든 것이 순차적으로 촤르르 반응하며 움직이는 역동적인 시스템과 같습니다.
