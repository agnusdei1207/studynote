# 527. 인공지능 보안 (AI Security) 및 Adversarial Attack

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인공지능 보안 (AI Security)은 머신러닝 모델의 수명 주기(데이터 수집, 학습, 추론) 전반에 걸쳐 발생하는 고유한 취약점을 보호하는 체계로, 특히 **적대적 공격 (Adversarial Attack)**은 입력 데이터에 미세한 변조를 가해 AI가 잘못된 판단을 내리게 유도하는 기술이다.
> 2. **가치**: 자율주행차의 표지판 오인식, 생체 인증 우회, 금융 사기 탐지 무력화 등 현실 세계의 치명적 사고를 예방하기 위해 필수적이며, 모델의 강건성(Robustness)과 신뢰성을 정량화하는 핵심 지표로 활용된다.
> 3. **융합**: 적대적 훈련(Adversarial Training), 데이터 증강(Data Augmentation), 모델 워터마킹 기술과 결합되며, 현대 아키텍처에서는 보안 중심의 MLOps (SecMLOps) 파이프라인 구축을 통해 지능형 방어 체계를 완성한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: AI 보안은 전통적인 SW 보안(인젝션, 버그 등)을 넘어, AI 모델의 "논리적 판단 능력" 자체를 공격 대상으로 삼는다. **적대적 공격 (Adversarial Attack)**은 인간의 눈에는 보이지 않는 아주 작은 노이즈(Perturbation)를 이미지나 음성에 섞어, AI가 사과를 바나나로, 혹은 정지 표지판을 속도 제한 표지판으로 잘못 분류하게 만드는 공격이다.

- **💡 비유**: 적대적 공격은 **"인공지능에게만 보이는 특수한 착시 현상"**과 같습니다. 마술사가 관객의 시선을 돌려 속임수를 쓰듯, 공격자는 AI 알고리즘이 중요하게 생각하는 특징점을 교묘하게 비틀어 AI의 뇌(모델)를 혼란에 빠뜨리는 것입니다.

- **등장 배경**:
  1. **딥러닝의 블랙박스 특성**: 모델 내부의 의사결정 과정을 완벽히 이해하기 어렵기 때문에, 아주 작은 입력 변화가 결과값을 크게 바꾸는 취약 지점을 사전에 완벽히 제거하기 어렵다.
  2. **AI의 실전 배치 확산**: 자율주행, 의료 진단, 안면 인식 도어락 등 보안이 중요한 영역에 AI가 도입되면서 공격 성공 시의 파괴력이 수직 상승했다.
  3. **공격 도구의 오픈소스화**: CleverHans, Foolbox 등 적대적 샘플을 생성하는 도구들이 보급되면서 비전문가도 정교한 공격을 수행할 수 있게 되었다.

- **📢 섹션 요약 비유**: 시력이 아주 좋은 로봇(AI)이라도, 렌즈에 특수한 필터(노이즈)를 끼우면 눈앞의 낭떠러지를 꽃밭으로 착각하여 추락할 수 있다는 위험성을 경고하는 보안 영역입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 적대적 공격의 3대 분류 체계

공격자가 모델에 대해 알고 있는 정보의 양에 따라 구분한다.

| 분류 | 설명 | 특징 | 비유 |
|:---|:---|:---|:---|
| **White-box Attack** | 모델의 구조, 가중치(Weights)를 모두 앎 | 가장 강력하고 효율적인 공격 가능 | 내부 설계도를 가진 도둑 |
| **Black-box Attack** | 입력과 출력 결과만 관찰 가능 | 쿼리(Query) 반복을 통해 모델 특성 추론 | 밖에서 소리만 듣는 도둑 |
| **Gray-box Attack** | 모델의 일부 정보나 학습 데이터만 앎 | 특정 계층(Layer) 정보 등을 활용 | 열쇠 구멍으로 엿보는 도둑 |

---

### 적대적 샘플 생성 및 방어 아키텍처

공격자가 노이즈를 생성하고, 시스템이 이를 방어하는 전체 흐름을 시각화한다.

```text
 ┌─────────────────────────────────────────────────────────────────────────────┐
 │                      적대적 공격(Adversarial Attack) 및 방어 프로세스             │
 ├─────────────────────────────────────────────────────────────────────────────┤
 │                                                                             │
 │ [원본 데이터 (X)] ────▶ [공격자 알고리즘 (FGSM/PGD)] ────▶ [적대적 샘플 (X')]   │
 │    (정상 이미지)           (미세 노이즈 δ 추가)             (변조된 이미지)     │
 │                                                                  │          │
 │           ┌──────────────────────────────────────────────────────┘          │
 │           ▼                                                                 │
 │  ┌───────────────────────────────────────────────────────────────────────┐  │
 │  │  방어 적용 AI 모델 (Defended Model)                                     │  │
 │  ├───────────────────────────────────────────────────────────────────────┤  │
 │  │ 1. 입력 정제 (Denoising): 노이즈 필터링 ──▶ [정상 판단]                  │  │
 │  │ 2. 적대적 훈련 (Adv. Training): 공격 데이터를 학습에 포함 ──▶ [내성 확보]   │  │
 │  │ 3. 앙상블 탐지: 여러 모델의 결과 비교 ──▶ [이상 감지]                     │  │
 │  └──────────────────┬────────────────────────────────────────────────────┘  │
 │                    │                                                        │
 │                    ▼ [최종 출력]                                             │
 │           ┌────────────────────────┐                                        │
 │           │  올바른 결과 (Correct)  │ ◀─── 방어 성공 시                       │
 │           └────────────────────────┘                                        │
 │                                                                             │
 └─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 공격의 핵심 알고리즘인 **FGSM (Fast Gradient Sign Method)**은 모델의 오차(Loss)가 가장 커지는 방향으로 원본 데이터($X$)를 살짝 변형($\delta$)시킨다. 결과물인 **적대적 샘플($X'$)**은 인간에게는 원본과 똑같아 보이지만, 모델 내부의 경사도(Gradient)를 뒤흔들어 전혀 엉뚱한 결과를 도출하게 한다. 이에 대응하는 가장 강력한 방어책은 **적대적 훈련 (Adversarial Training)**이다. 모델을 학습시킬 때 미리 생성한 수많은 공격 샘플을 정답과 함께 보여주어, "이런 노이즈가 섞여도 원래 정답은 사과야!"라고 미리 내성을 길러주는 방식이다.

---

### 심층 동작 원리: 전이성 (Transferability)

블랙박스 공격을 가능하게 하는 놀라운 특성이다.
1. **정의**: 특정 모델 A를 공격하기 위해 만든 적대적 샘플이, 구조가 전혀 다른 모델 B에서도 공격에 성공하는 현상.
2. **원리**: 서로 다른 모델이라도 동일한 학습 데이터를 사용했다면, 데이터 내부에 공통적으로 존재하는 '취약한 특징점'을 공격자가 건드리기 때문이다.
3. **가치**: 공격자는 자신이 직접 타겟 모델을 해킹하지 않고도, 유사한 오픈소스 모델을 대상으로 공격 도구를 만들어 배포할 수 있다.

- **📢 섹션 요약 비유**: 수만 개의 자물쇠 회사가 달라도, 공통으로 사용하는 특수 금속(학습 데이터)의 약점을 알아내면 모든 자물쇠를 한 번에 열 수 있는 마스터 키가 된다는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 주요 적대적 공격 알고리즘 비교

| 알고리즘 | 특징 | 장점 | 단점 |
|:---|:---|:---|:---|
| **FGSM** | 단 한 번의 연산으로 노이즈 생성 | 매우 빠름, 실시간 공격 가능 | 방어(적대적 훈련)에 취약함 |
| **PGD** | FGSM을 여러 번 반복하여 정교화 | **현존하는 가장 강력한 공격** | 연산 속도가 상대적으로 느림 |
| **C&W Attack** | 최적화 기법을 사용하여 노이즈 최소화 | 인간이 절대 구별 못 함 | 수학적 복잡도 매우 높음 |
| **JSMA** | 특정 픽셀 몇 개만 골라 변조 | 눈에 덜 띔 (Saliency Map 활용) | 특정 데이터 타입에만 효과적 |

---

### 기술 융합: 적대적 패치 (Adversarial Patch)

디지털 이미지가 아닌 물리적 환경에서의 공격 기술이다.
- **사례**: 옷이나 모자에 특정한 무늬(패치)를 인쇄하여 부착하면, CCTV의 안면 인식 AI가 해당 인물을 '투명 인간'으로 처리하거나 다른 사람으로 오인하게 만든다.
- **가치**: 해커가 시스템에 직접 접속하지 않고도 '현실 세계의 물건'만으로 AI를 무력화할 수 있음을 시사한다.

- **📢 섹션 요약 비유**: 해킹을 하는 대신, AI 보안 요원 앞에서 '나를 나무로 보이게 하는 마법의 망토(패치)'를 두르고 유유히 걸어 들어가는 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 자율주행 도로 인프라의 보안 강화

- **문제 상황**: 도로의 '정지(STOP)' 표지판에 공격자가 검은색 스티커를 교묘하게 붙여, 자율주행차가 이를 '속도 제한'으로 오인해 멈추지 않고 질주할 위험 발견.
- **의사결정 (AI 보안 강화 전략)**:
  1. **Robustness Training**: 다양한 기후 조건(눈, 비) 및 인위적인 오염(스티커, 낙서)이 포함된 적대적 샘플을 대량 생성하여 주행 모델을 재학습시킨다.
  2. **Multi-modal Fusion**: 카메라 영상뿐만 아니라 LiDAR, Radar 정보를 결합하여, 시각적 착시(이미지 노출)가 발생하더라도 물리적 형태(LiDAR)로 물체를 재확인하도록 설계한다.
  3. **Input Sanitization**: 딥러닝 판단 직전, 이미지의 고주파 성분(노이즈)을 강제로 뭉뚱그리는 전처리 필터를 적용하여 미세 변조의 영향을 줄인다.
  4. **Confidence Threshold**: 모델의 판단 결과 점수가 일정 수준(Confidence Score) 이하라면 모호한 상황으로 간주하고 즉시 긴급 제동하거나 운전자에게 제어권을 넘긴다.

### 도입 체크리스트
- **Evaluation**: 우리 모델이 표준 공격 도구(PGD 등)에 대해 얼마나 버티는지 '적대적 강건성 점수'를 측정했는가?
- **Diversity**: 학습 데이터셋에 편향된 정보가 아닌, 극단적인 예외 상황(Edge Cases)이 충분히 포함되었는가?
- **Monitoring**: 운영 환경에서 AI 판단 결과의 분포가 평소와 다르게 급변하는지 실시간 감시하는가?

### 안티패턴
- **단순 정확도 맹신**: 테스트 세트에서 99.9% 정확도가 나왔다고 안심하는 것. 적대적 샘플 1개에 모델이 완전히 무너질 수 있음을 간과하는 행위.
- **비밀 보장 기반 보안 (Obscurity)**: "모델 구조를 안 알려주면 공격 못 하겠지"라는 생각. 전이성(Transferability)으로 인해 블랙박스 공격이 얼마든지 가능하다.

- **📢 섹션 요약 비유**: 공부를 잘해서 전교 1등(정확도 99.9%)을 하더라도, 갑자기 시험지에 잉크를 쏟는 돌발 상황(적대적 공격)에 대처하지 못하면 빵점을 맞을 수 있다는 점을 잊지 말아야 합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 세부 내용 |
|:---|:---|:---|
| **정량적 (Safety)** | 오인식 사고 발생률 80% 감소 | 적대적 훈련 적용 시 외부 노이즈에 대한 모델 회복력 강화 |
| **정량적 (Efficiency)** | 보안 검증 시간 단축 | 자동화된 공격 시뮬레이션 도구를 통해 모델 출시 전 취약점 사전 식별 |
| **정성적 (Trust)** | 사용자 안전 신뢰도 확보 | "공격에 강한 안전한 AI"라는 브랜드 가치 증명 |
| **정성적 (Resilience)** | 시스템 복원력 향상 | 공격이 발생해도 치명적 오답 대신 '판단 유보'를 선택하는 지능적 대처 |

### 미래 전망
- **Certifiable Robustness**: 모델이 어떠한 반경 내의 공격에도 100% 안전함을 수학적으로 보증하는 '증명 가능한 강건성' 기술이 상용화될 것이다.
- **AI-Native SOC**: 보안 관제 요원이 아닌 AI가 다른 AI의 공격을 실시간으로 방어하고 학습하는 'AI vs AI' 자동화 보안 체계가 주류가 될 것이다.

### 참고 표준
- **ISO/IEC 27090**: Guidance on addressing security threats to AI (진행 중)
- **NIST AI Risk Management Framework (AI RMF)**: AI 리스크 관리의 글로벌 표준
- **MITRE ATLAS**: AI 시스템에 대한 공격 전술 및 기법 행렬 (ATT&CK의 AI 버전)

- **📢 섹션 요약 비유**: 이제 인공지능은 단순히 똑똑해지는 단계를 넘어, 세상의 거친 풍파와 나쁜 의도를 견뎌내는 '어른스러운 인공지능'으로 성장해야 하는 숙제를 안고 있습니다.
