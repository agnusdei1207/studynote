# 135-141. 요구사항 도출 기법 (Persona, Journey Map)

> **핵심 인사이트**: 사용자는 자신이 무엇을 원하는지 말로 다 설명하지 못한다. 인터뷰와 설문이라는 전통적 방식을 넘어, 가상의 사용자(Persona)를 세우고 그들의 발자취(Journey Map)를 따라가며 그들이 느끼는 불편함(Pain Point)을 찾아내는 공감 능력이 현대 요구공학의 핵심이다.

---

## Ⅰ. 전통적 도출 기법
1. **인터뷰 (Interview)**: 1:1 대화를 통해 심층적인 정보를 얻음. (비용과 시간 많이 소요)
2. **설문지 (Questionnaire)**: 다수의 사용자에게 빠르게 의견 수집. (답변의 깊이가 낮음)
3. **브레인스토밍**: 자유로운 분위기에서 아이디어를 쏟아냄. (비판 금지 원칙)
4. **워크숍 (JAD)**: 모든 이해관계자가 한자리에 모여 집중적으로 요구사항 결정.

---

## Ⅱ. 사용자 중심 도출 기법

### 1. 페르소나 (Persona)
* **개념**: 실제 사용자를 대표할 수 있는 **가상의 인물**을 구체적으로 설정하는 기법입니다.
* **효과**: "사용자"라는 막연한 대상 대신 "30대 직장인 김철수 씨"라는 구체적 인물을 상정하여 의사결정의 기준을 명확히 합니다.

### 2. 사용자 여정 지도 (User Journey Map)
* **개념**: 사용자가 서비스를 이용하는 전 과정을 시간 순서대로 시각화한 지도입니다.
* **핵심**: 각 단계에서의 행동, 감정, 생각 등을 기록하여 **고충 지점 (Pain Point)**을 발굴합니다.

### 3. 관찰 및 섀도잉 (Shadowing)
* 사용자의 업무 환경에 직접 들어가 그림자처럼 따라다니며, 사용자가 인지하지 못하는 습관적인 요구사항을 포착합니다.

---

## Ⅲ. 프로토타이핑 (Prototyping)
* **Low-fidelity**: 종이나 스케치 도구로 빠르게 핵심만 그려봄.
* **High-fidelity**: 실제 제품과 거의 유사한 디자인과 동작을 구현함.

---

## ④. 개념 맵 및 요약

```ascii
[사용자 경험(UX) 기반 도출]

  (인격 부여) [ Persona ] ──> ( 과정 추적 ) [ User Journey Map ]
                                            │
                                            ▼
  ( 실체 확인 ) [ Prototype ] <── ( 불편함 발견 ) [ Pain Point ]
```

📢 **섹션 요약 비유**: **인터뷰**는 수사관이 용의자를 심문하는 것이고, **페르소나**는 소설가가 주인공의 프로필을 짜는 것과 같습니다. **사용자 여정 지도**는 헨젤과 그레텔이 숲속에 떨어뜨린 빵부스러기를 따라가듯 사용자의 동선을 추적하는 것이며, **섀도잉**은 다큐멘터리 감독이 야생 동물의 생태를 몰래 촬영하며 숨겨진 진실을 찾는 것과 같습니다.
