# 142-147. 요구사항 분석 및 모델링 (Structured, OOA)

> **핵심 인사이트**: 분석은 찰흙 덩어리 같은 요구사항을 깎고 다듬어 '조각상'을 만드는 과정이다. 데이터의 흐름을 쫓는 구조적 방식(DFD)과 실세계의 사물을 추상화하는 객체지향 방식(OOA)을 통해, 모호함을 제거하고 개발자와 설계자가 소통할 수 있는 정밀한 지도를 그려야 한다.

---

## Ⅰ. 요구사항 분석 (Requirements Analysis) 개요
도출된 요구사항들 사이의 충돌, 중복, 누락을 발견하고 수정하여 **범위(Scope)**를 확정하는 단계입니다.

* **목적**: 타당성 검토, 제약 사항 파악, 우선순위 부여.

---

## Ⅱ. 구조적 분석 (Structured Analysis)
프로세스(기능)와 데이터 흐름 중심의 고전적 분석 기법입니다.

1. **DFD (Data Flow Diagram, 자료 흐름도)**: 시스템 내 데이터의 흐름과 가공 과정을 시각화.
   - **구성 요소**: 프로세스(Process), 자료 흐름(Data Flow), 자료 저장소(Data Store), 단말(Terminator).
2. **DD (Data Dictionary, 자료 사전)**: DFD에 나타난 자료들을 정의한 사전.
   - **기호**: `=` (정의), `+` (연결), `{ }` (반복), `[ ]` (선택), `( )` (생략), `* *` (주석).
3. **Mini-Spec (소단위 명세서)**: DFD의 최하위 프로세스가 수행하는 일을 상세히 기술한 문서.

---

## Ⅲ. 객체지향 분석 (OOA, Object-Oriented Analysis)
사용자의 요구사항을 **객체(Object)** 중심으로 분석하여 클래스, 속성, 연산을 정의하는 기법입니다.

1. **객체 모델링 (Information)**: 시스템의 정적 구조 파악 (Class Diagram).
2. **동적 모델링 (Dynamic)**: 상태 변화와 상호작용 파악 (State/Sequence Diagram).
3. **기능 모델링 (Functional)**: 어떤 일을 하는지 파악 (Use Case Diagram).

---

## Ⅳ. 유스케이스 다이어그램 (Use Case Diagram)
사용자의 관점에서 시스템이 제공하는 기능을 모델링하는 UML 도구입니다.

* **구성 요소**:
  - **액터 (Actor)**: 시스템 외부에서 상호작용하는 사람 또는 외부 시스템.
  - **유스케이스 (Use Case)**: 시스템이 수행하는 서비스(기능)의 단위.
* **관계**: 
  - **Include (포함)**: 기능을 수행하기 위해 반드시 필요한 선행 작업.
  - **Extend (확장)**: 특정 조건에서만 수행되는 선택적 작업.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[요구사항 분석의 두 갈래]

  (기능 중심) [ Structured Analysis ] <─── (상호 보완) ───> [ OO Analysis ] (객체 중심)
      │                                                       │
      ├─ DFD (흐름)                                           ├─ Class (구조)
      ├─ DD  (정의)                                           ├─ Sequence (동작)
      └─ Mini-Spec (명세)                                     └─ Use Case (범위)
```

📢 **섹션 요약 비유**: **요구사항 분석**은 엉킨 실타래를 풀어 색깔별로 정리하는 작업입니다. **구조적 분석**은 공장의 컨베이어 벨트를 따라 재료가 어떻게 변하는지 보는 것이고, **자료 사전**은 그 재료들의 성분표를 적어두는 것입니다. **객체지향 분석**은 공장의 로봇(객체) 하나하나가 어떤 능력이 있는지 정의하는 것이며, **유스케이스**는 손님이 키오스크에서 어떤 메뉴를 고를 수 있는지 보여주는 안내 화면과 같습니다.
