# 470. 버퍼 캐시 (Buffer Cache)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크 I/O 성능 향상을 위한 메모리 캐시
> 2. **가치**: 디스크 접근 최소화
> 3. **융합**: 페이지 캐시, 가상 메모리와 연관

---

## Ⅰ. 개요

### 개념 정의

버퍼 캐시(Buffer Cache)는 **디스크 블록 데이터를 메모리에 캐싱하여 디스크 I/O를 줄이는 커널 메커니즘**이다.

### 💡 비유: 책상 위 책
버퍼 캐시는 **책상 위 책**과 같다. 자주 보는 책(디스크 블록)을 책상(메모리)에 두어 서재(디스크)에 가지 않아도 된다.

### 버퍼 캐시 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                버퍼 캐시                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【버퍼 캐시 위치】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────┐                                                 │ │
│  │  │   CPU       │                                                 │ │
│  │  └──────┬──────┘                                                 │ │
│  │         │                                                         │ │
│  │         ▼                                                         │ │
│  │  ┌─────────────────────────────────────────────────────────┐      │ │
│  │  │                    메모리                                 │      │ │
│  │  │  ┌─────────────────────────────────────────────────┐    │      │ │
│  │  │  │            버퍼 캐시                              │    │      │ │
│  │  │  │  • 디스크 블록 캐싱                               │    │      │ │
│  │  │  │  • 읽기/쓰기 버퍼링                               │    │      │ │
│  │  │  │  • 페이지 캐시와 통합 (리눅스)                    │    │      │ │
│  │  │  └─────────────────────────────────────────────────┘    │      │ │
│  │  └─────────────────────────────────────────────────────────┘      │ │
│  │         │                                                         │ │
│  │         ▼                                                         │ │
│  │  ┌─────────────┐                                                 │ │
│  │  │   디스크     │                                                 │ │
│  │  └─────────────┘                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버퍼 캐시 동작】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  읽기 요청:                                                          │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  1. 버퍼 캐시 확인                                         │    │ │
│  │  │  2. 캐시 히트 → 캐시에서 반환 (빠름)                         │    │ │
│  │  │  3. 캐시 미스 → 디스크에서 읽어 캐시에 저장 후 반환           │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  쓰기 요청:                                                          │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  1. 버퍼 캐시에 쓰기                                       │    │ │
│  │  │  2. 더티 페이지로 표시                                     │    │ │
│  │  │  3. 나중에 디스크에 동기화 (지연 쓰기)                       │    │ │
│  │  │  4. 또는 즉시 동기화 (동기 쓰기)                            │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                버퍼 캐시 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【리눅스 통합 캐시】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  리눅스는 버퍼 캐시와 페이지 캐시를 통합:                              │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                    페이지 캐시                            │    │ │
│  │  │  ┌─────────────────────────────────────────────────┐    │    │ │
│  │  │  │  • 파일 데이터 페이지                             │    │    │ │
│  │  │  │  • 메모리 매핑된 파일 (mmap)                     │    │    │ │
│  │  │  │  • 버퍼 캐시 (블록 디바이스)                      │    │    │ │
│  │  │  └─────────────────────────────────────────────────┘    │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  /proc/meminfo에서 확인:                                             │ │
│  │  Buffers: 블록 디바이스 버퍼                                          │ │
│  │  Cached: 페이지 캐시                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 교체 알고리즘】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  LRU (Least Recently Used):                                          │ │
│  │  • 가장 오래 사용하지 않은 것 교체                                     │ │
│  │  • 2개 리스트: 활성(active) + 비활성(inactive)                       │ │
│  │                                                             │ │
│  │  리스트 이동:                                                         │
│  │  새 페이지 → 비활성 → 접근 → 활성 → 오래됨 → 비활성 → 교체            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【더티 페이지 관리】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  더티 페이지: 수정되었지만 디스크에 아직 안 쓰여진 페이지              │ │
│  │                                                             │ │
│  │  동기화 시점:                                                         │ │
│  │  • 백그라운드 쓰기 (pdflush/kworker)                                  │ │
│  │  • sync 명령                                                          │ │
│  │  • 메모리 압박 시                                                      │ │
│  │  • 언마운트 시                                                         │ │
│  │                                                             │ │
│  │  관련 파라미터:                                                       │
│  │  /proc/sys/vm/dirty_ratio: 더티 페이지 최대 비율                      │ │
│  │  /proc/sys/vm/dirty_background_ratio: 백그라운드 쓰기 시작 비율        │ │
│  │  /proc/sys/vm/dirty_writeback_centisecs: 쓰기 주기                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 상태 확인】                                                  │
│  ──────────────────                                                  │
│  // /proc/meminfo                                                    │
│  $ cat /proc/meminfo | grep -E "Buffers|Cached|Dirty"               │
│  Buffers:          102400 kB                                        │
│  Cached:          1024000 kB                                        │
│  Dirty:              512 kB  // 디스크에 안 쓰여진 데이터              │
│                                                                     │
│  // free 명령                                                        │
│  $ free -h                                                           │
│                total    used    free    shared  buff/cache  available│
│  Mem:           8G      2G      1G      0.5G       5G         5G     │
│  // buff/cache = Buffers + Cached                                    │
│                                                                     │
│  // vmstat                                                           │
│  $ vmstat 1                                                          │
│  procs -----------memory---------- ---swap-- -----io----            │
│    r  b   swpd   free   buff  cache   si   so    bi    bo            │
│    1  0      0 102400 10240 1024000    0    0    10    20            │
│  // buff: 버퍼 캐시, cache: 페이지 캐시                                │
│  // bi: 블록 in, bo: 블록 out                                          │
│                                                                     │
│  【캐시 동기화】                                                      │
│  ──────────────────                                                  │
│  // 모든 캐시 동기화                                                   │
│  $ sync                                                              │
│                                                                     │
│  // 파일 시스템 동기화                                                 │
│  $ sync /mnt/data                                                    │
│                                                                     │
│  // 캐시 비우기 (주의!)                                                │
│  $ echo 3 > /proc/sys/vm/drop_caches                                │
│  // 1: 페이지캐시, 2: dentries/inodes, 3: 모두                        │
│                                                                     │
│  【캐시 튜닝】                                                          │
│  ──────────────────                                                  │
│  // 더티 페이지 비율 조정                                              │
│  $ cat /proc/sys/vm/dirty_ratio                                     │
│  20  // 메모리의 20%까지 더티 허용                                      │
│                                                                     │
│  $ cat /proc/sys/vm/dirty_background_ratio                          │
│  10  // 10%에서 백그라운드 쓰기 시작                                    │
│                                                                     │
│  // 쓰기 주기                                                          │
│  $ cat /proc/sys/vm/dirty_writeback_centisecs                       │
│  500  // 5초마다                                                      │
│                                                                     │
│  // vfs_cache_pressure (아이노드/dentry 캐시)                         │
│  $ cat /proc/sys/vm/vfs_cache_pressure                              │
│  100  // 기본값, 높으면 더 적극적으로 회수                               │
│                                                                     │
│  【프로그래밍에서 캐시 제어】                                          │
│  ──────────────────                                                  │
│  #include <unistd.h>                                                 │
│  #include <fcntl.h>                                                  │
│                                                                     │
│  // 캐시 무시하고 직접 I/O                                              │
│  int fd = open("file", O_RDONLY | O_DIRECT);                        │
│                                                                     │
│  // 캐시에 데이터 제거 (POSIX_FADV_DONTNEED)                          │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_DONTNEED);                      │
│                                                                     │
│  // 데이터를 캐시에 미리 로드                                            │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_WILLNEED);                      │
│                                                                     │
│  // 순차 접근 힌트                                                      │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_SEQUENTIAL);                    │
│                                                                     │
│  // 랜덤 접근 힌트                                                      │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_RANDOM);                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 디스크 블록을 메모리에 캐싱
• 리눅스: 페이지 캐시와 통합
• 확인: /proc/meminfo, free, vmstat
• 동작: 읽기 캐싱, 쓰기 버퍼링
• 교체: LRU (활성/비활성 리스트)
• 동기화: sync, 더티 페이지 관리
• 튜닝: dirty_ratio, dirty_background_ratio
• 제어: O_DIRECT, posix_fadvise
```
