# 479. 파일 시스템 조각 모음 (Defragmentation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산된 파일 블록을 연속으로 재배치
> 2. **가치**: 디스크 성능 향상
> 3. **융합**: 파일 시스템, 디스크 성능과 연관

---

## Ⅰ. 개요

### 개념 정의

조각 모음(Defragmentation)은 **파일 시스템에서 분산된 파일 블록들을 연속된 위치로 재배치하여 성능을 향상시키는 작업**이다.

### 💡 비유: 정리된 책장
조각 모음은 **정리된 책장**과 같다. 흩어진 책 페이지들을 한 권으로 모아 찾기 쉽게 한다.

### 조각 모음 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                조각 모음                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【단편화 발생】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  초기:                                                               │ │
│  │  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐                        │ │
│  │  │ A │ A │ A │ B │ B │ B │ C │ C │ C │   │                        │ │
│  │  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘                        │ │
│  │  파일들이 연속 배치                                                   │ │
│  │                                                             │ │
│  │  B 삭제 후:                                                          │ │
│  │  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐                        │ │
│  │  │ A │ A │ A │   │   │   │ C │ C │ C │   │                        │ │
│  │  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘                        │ │
│  │  빈 공간 발생                                                        │ │
│  │                                                             │ │
│  │  D 파일 생성 (빈 공간에 분산):                                         │ │
│  │  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐                        │ │
│  │  │ A │ A │ A │ D │ D │   │ C │ C │ C │ D │                        │ │
│  │  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘                        │ │
│  │  D가 분산됨 (단편화 발생)                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【조각 모음 후】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐                        │ │
│  │  │ A │ A │ A │ C │ C │ C │ D │ D │ D │   │                        │ │
│  │  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘                        │ │
│  │  파일들이 연속 배치, 빈 공간 통합                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                조각 모음 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【단편화 종류】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 내부 단편화 (Internal Fragmentation):                              │ │
│  │     • 블록 내부의 미사용 공간                                           │ │
│  │     • 파일이 블록 크기에 딱 맞지 않음                                    │ │
│  │     • 블록 크기 줄이기로 완화                                            │ │
│  │                                                             │ │
│  │  2. 외부 단편화 (External Fragmentation):                              │ │
│  │     • 파일이 여러 블록에 분산                                           │ │
│  │     • 파일 생성/삭제 반복으로 발생                                       │ │
│  │     • 조각 모음으로 해결                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파일 시스템별 조각 모음 필요성】                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파일 시스템    단편화 발생    조각 모음 필요                            │ │
│  │  ──────────    ──────────    ────────────                              │ │
│  │  FAT32        높음          필요                                        │ │
│  │  NTFS         보통          가끔 필요                                   │ │
│  │  ext4         낮음          거의 불필요                                 │ │
│  │  XFS          낮음          거의 불필요                                 │ │
│  │  Btrfs        매우 낮음      불필요                                     │ │
│  │  SSD          N/A           권장 안 함 (수명 단축)                       │ │
│  │                                                             │ │
│  │  ext4, XFS: 지연 할당, 익스텐트로 단편화 최소화                         │ │
│  │  Btrfs: COW + 단편화 자동 해결                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SSD와 조각 모음】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  SSD는 조각 모음이 필요 없음:                                          │ │
│  │  • 탐색 시간이 0이라 단편화 영향 없음                                    │ │
│  │  • TRIM이 더 중요                                                     │ │
│  │  • 불필요한 쓰기로 수명 단축                                            │ │
│  │  • wear leveling 방해                                                  │ │
│  │                                                             │ │
│  │  대신 TRIM 실행:                                                      │ │
│  │  $ sudo fstrim /                                                      │ │
│  │  $ sudo fstrim -v /  // 상세 출력                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【윈도우 조각 모음】                                                  │
│  ──────────────────                                                  │
│  // GUI 도구                                                          │
│  // 디스크 조각 모음 도구 실행                                          │
│                                                                     │
│  // 명령줄                                                             │
│  > defrag C:                                                         │
│  > defrag C: /O  // 최적화 포함                                        │
│  > defrag C: /A  // 분석만                                             │
│  > defrag C: /V  // 상세 출력                                          │
│                                                                     │
│  // 정기적 예약                                                        │
│  // 기본적으로 자동 실행됨                                              │
│                                                                     │
│  【Linux ext4 조각 모음】                                             │
│  ──────────────────                                                  │
│  // 단편화 확인                                                        │
│  $ sudo e4defrag -c /dev/sda1                                        │
│                                                                     │
│  // 조각 모음 실행                                                     │
│  $ sudo e4defrag /dev/sda1                                           │
│                                                                     │
│  // 특정 디렉토리만                                                     │
│  $ sudo e4defrag /home/user/                                         │
│                                                                     │
│  // 진행률 표시                                                        │
│  $ sudo e4defrag -v /dev/sda1                                        │
│                                                                     │
│  【XFS 조각 모음】                                                      │
│  ──────────────────                                                  │
│  // 단편화 확인                                                        │
│  $ xfs_db -c frag -f /dev/sda1                                       │
│                                                                     │
│  // 조각 모음 실행 (마운트된 상태에서)                                  │
│  $ xfs_fsr /mountpoint                                               │
│                                                                     │
│  // 전체 파일 시스템                                                   │
│  $ xfs_fsr                                                           │
│                                                                     │
│  【Btrfs 조각 모음】                                                   │
│  ──────────────────                                                  │
│  // 단편화 확인                                                        │
│  $ sudo btrfs filesystem df /                                        │
│                                                                     │
│  // 조각 모음 실행                                                     │
│  $ sudo btrfs filesystem defragment /                                │
│                                                                     │
│  // 특정 디렉토리만                                                     │
│  $ sudo btrfs filesystem defragment -r /home                         │
│                                                                     │
│  【TRIM (SSD용)】                                                      │
│  ──────────────────                                                  │
│  // 수동 TRIM                                                         │
│  $ sudo fstrim -v /                                                  │
│  /: 123456789 bytes trimmed                                          │
│                                                                     │
│  // 모든 마운트된 파일 시스템에 TRIM                                     │
│  $ sudo fstrim -a -v                                                 │
│                                                                     │
│  // 정기적 TRIM (systemd timer)                                       │
│  $ sudo systemctl status fstrim.timer                                │
│  // 기본적으로 주 1회 실행                                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 분산된 파일 블록을 연속으로 재배치
• 종류: 내부 단편화, 외부 단편화
• 영향: 순차 읽기 성능 저하
• FAT/NTFS: 조각 모음 필요
• ext4/XFS: 거의 불필요 (지연 할당)
• Btrfs: 자동 해결
• SSD: 조각 모음 금지, TRIM 사용
• 도구: defrag(윈도우), e4defrag(ext4), fstrim(SSD)
```
