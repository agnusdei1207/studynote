# 472. RAID 1 (Mirroring)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 여러 디스크에 동일하게 복제
> 2. **가치**: 높은 데이터 보호와 가용성
> 3. **융합**: RAID 0, 내결함성과 연관

---

## Ⅰ. 개요

### 개념 정의

RAID 1(Mirroring)은 **동일한 데이터를 두 개 이상의 디스크에 복제하여 저장하는 방식**이다.

### 💡 비유: 복사본
RAID 1은 **중요 문서 복사본**과 같다. 원본과 똑같은 복사본을 따로 보관한다.

### RAID 1 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 1 미러링                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  데이터: [A][B][C][D][E][F]                                         │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                                                         │    │ │
│  │  │    Disk 0 (원본)              Disk 1 (미러)               │    │ │
│  │  │   ┌─────────────┐           ┌─────────────┐              │    │ │
│  │  │   │      A      │    ═══    │      A      │              │    │ │
│  │  │   ├─────────────┤           ├─────────────┤              │    │ │
│  │  │   │      B      │    ═══    │      B      │              │    │ │
│  │  │   ├─────────────┤           ├─────────────┤              │    │ │
│  │  │   │      C      │    ═══    │      C      │              │    │ │
│  │  │   ├─────────────┤           ├─────────────┤              │    │ │
│  │  │   │     ...     │    ═══    │     ...     │              │    │ │
│  │  │   └─────────────┘           └─────────────┘              │    │ │
│  │  │                                                         │    │ │
│  │  │         동일한 데이터 복제                                  │    │ │
│  │  │                                                         │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  한 디스크 고장 → 다른 디스크로 서비스 계속                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【읽기 분산】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  읽기 요청: [A] [B] [C] [D]                                         │ │
│  │                                                             │ │
│  │  Disk 0: [A] [C] 읽기  ←─── 분산                                     │ │
│  │  Disk 1: [B] [D] 읽기  ←─── 분산                                     │ │
│  │                                                             │ │
│  │  읽기 성능: 2배 향상 (이론적)                                          │ │
│  │  쓰기 성능: 동일 (양쪽 모두 써야 함)                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 1 상세                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 높은 내결함성:                                                    │ │
│  │     • 디스크 1개 고장 시에도 서비스 지속                                 │ │
│  │     • 데이터 손실 확률 매우 낮음                                        │ │
│  │     • 핫 스왑 지원 시 무중단 교체 가능                                  │ │
│  │                                                             │ │
│  │  2. 빠른 복구:                                                        │ │
│  │     • 디스크 교체 후 미러에서 복사만 하면 됨                              │ │
│  │     • RAID 5/6보다 복구 빠름                                          │ │
│  │     • 복구 중 성능 저하 적음                                           │ │
│  │                                                             │ │
│  │  3. 읽기 성능:                                                        │ │
│  │     • 읽기 요청 분산 가능                                              │ │
│  │     • 2배 읽기 성능 향상 (이론적)                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 용량 비효율:                                                      │ │
│  │     • 50% 용량만 사용 가능                                            │ │
│  │     • 예: 2×1TB = 1TB 사용 가능                                      │ │
│  │     • 비용 대비 효율 낮음                                              │ │
│  │                                                             │ │
│  │  2. 쓰기 성능:                                                        │ │
│  │     • 양쪽 디스크에 모두 써야 함                                        │ │
│  │     • 쓰기 성능 향상 없음                                              │ │
│  │     • 약간의 오버헤드 존재                                             │ │
│  │                                                             │ │
│  │  3. 비용:                                                             │ │
│  │     • 같은 용량을 위해 2배 디스크 필요                                  │ │
│  │     • 스토리지 비용 2배                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【용량 계산】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  총 용량 = min(모든 디스크 크기)                                       │ │
│  │                                                             │ │
│  │  예: 1TB + 1TB = 1TB 사용 가능                                        │ │
│  │      1TB + 2TB = 1TB 사용 가능 (2TB는 1TB만 사용)                      │ │
│  │                                                             │ │
│  │  효율: 50% (2디스크), 33% (3디스크), ...                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux mdadm RAID 1 생성】                                        │
│  ──────────────────                                                  │
│  // RAID 1 어레이 생성                                                │
│  $ sudo mdadm --create /dev/md0 --level=1 --raid-devices=2 \        │
│        /dev/sdb1 /dev/sdc1                                          │
│                                                                     │
│  // 상태 확인                                                        │
│  $ cat /proc/mdstat                                                 │
│  Personalities : [raid1]                                            │
│  md0 : active raid1 sdc1[1] sdb1[0]                                 │
│        1047552 blocks super 1.2 [2/2] [UU]                          │
│  // [UU] = 두 디스크 모두 정상                                          │
│                                                                     │
│  // 상세 정보                                                          │
│  $ sudo mdadm --detail /dev/md0                                     │
│    Number   Major   Minor   RaidDevice State                        │
│       0       8       17        0      active sync   /dev/sdb1       │
│       1       8       33        1      active sync   /dev/sdc1       │
│                                                                     │
│  【디스크 고장 및 교체】                                              │
│  ──────────────────                                                  │
│  // 고장난 디스크 표시                                                  │
│  $ sudo mdadm --manage /dev/md0 --fail /dev/sdb1                    │
│  // [UU] → [_U] 로 변경                                               │
│                                                                     │
│  // 고장난 디스크 제거                                                  │
│  $ sudo mdadm --manage /dev/md0 --remove /dev/sdb1                  │
│                                                                     │
│  // 새 디스크 추가                                                     │
│  $ sudo mdadm --manage /dev/md0 --add /dev/sdd1                     │
│  // 자동 동기화 시작                                                   │
│                                                                     │
│  // 동기화 진행 확인                                                   │
│  $ cat /proc/mdstat                                                 │
│  md0 : active raid1 sdd1[2] sdc1[1]                                 │
│        1047552 blocks super 1.2 [2/1] [_U]                          │
│        [=>...................]  recovery = 5.0%                      │
│                                                                     │
│  【적합한 사용 사례】                                                  │
│  ──────────────────                                                  │
│  • 운영체제 설치 (부팅 디스크)                                          │
│  • 데이터베이스 서버                                                    │
│  • 중요 데이터 저장                                                    │
│  • 이메일 서버                                                        │
│  • 고가용성 필요 시스템                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 데이터를 두 디스크에 동일하게 복제
• 장점: 높은 내결함성, 빠른 복구, 읽기 성능
• 단점: 50% 용량만 사용, 쓰기 성능 향상 없음
• 내결함성: 1디스크 고장 허용
• 용량: N/2 (2디스크), N/3 (3디스크)
• 복구: 미러에서 복사만 하면 됨
• 용도: OS, DB, 중요 데이터
• 비용: 스토리지 비용 2배
```
