## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비순환 그래프 디렉터리 (Acyclic-graph Directory)는 트리 구조를 확장하여, 서로 다른 디렉터리가 동일한 하위 디렉터리나 파일을 공유할 수 있도록 허용한 구조다.
> 2. **가치**: 데이터의 물리적 복사 없이 여러 위치에서 동일한 자원에 접근하는 별칭 (Aliasing) 기능을 제공하여 저장 공간 절약과 협업 효율성을 극대화한다.
> 3. **제약**: 순환 (Cycle)을 허용하지 않는 방향성 비순환 그래프 (DAG, Directed Acyclic Graph) 형태를 유지해야 하며, 파일 삭제 시 참조 횟수 (Reference Count) 관리가 필수적이다.

---

## Ⅰ. 개요 (Context & Background)

- **정의**: 트리 구조의 엄격한 부모-자식 관계를 완화하여, 하나의 노드가 여러 부모를 가질 수 있도록 설계된 디렉터리 구조다.
- **배경**: 여러 사용자가 참여하는 공동 프로젝트에서 동일한 라이브러리나 문서에 각자의 작업 디렉터리에서 접근해야 할 필요성이 커지면서 등장했다.
- **핵심 기술**:
  - **링크 (Link)**: 다른 파일이나 디렉터리를 가리키는 포인터.
  - **별칭 (Aliasing)**: 하나의 물리적 파일이 여러 개의 논리적 이름을 갖는 현상.

📢 **섹션 요약 비유**: 비순환 그래프 구조는 "공유 문서의 바로가기"와 같다. 원본 문서는 하나지만, 여러 사람의 바탕화면에 각각 아이콘을 만들어두고 모두가 실시간으로 내용을 공유하는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 공유 구조 시각화 (ASCII Diagram)

```text
       [ Root ]
       /      \
  [ user1 ]  [ user2 ]
      |          |
 [ projectA ]    |
      |          |
      └────┬─────┘
           ▼
     [ SharedFile ] <─── (Aliasing: 두 경로에서 동일 파일 지시)
```

### 2. 파일 공유의 두 가지 방식
- **하드 링크 (Hard Link)**:
  - 디렉터리 항목이 파일의 실제 물리적 주소(inode 번호)를 직접 가리킨다.
  - 원본과 링크 파일 사이에 구분이 없으며, 모든 링크가 사라져야 파일이 실제 삭제된다.
- **심볼릭 링크 (Symbolic Link)**:
  - 파일의 경로명(Pathname)을 담고 있는 특수한 '포인터 파일'을 만든다.
  - 원본이 삭제되면 심볼릭 링크는 깨진 링크(Dangling Link)가 된다.

📢 **섹션 요약 비유**: 하드 링크는 "한 사람의 이름이 여러 개인 것(예: 본명과 예명)"과 같고, 심볼릭 링크는 "그 사람의 집 주소가 적힌 쪽지"와 같다.

---

## Ⅲ. 삭제 로직과 일관성 문제 (Deletion & Consistency)

- **참조 횟수 (Reference Count)**: 하드 링크 방식에서는 파일 메타데이터에 이 파일을 가리키는 디렉터리 항목의 수를 기록한다. 삭제 시 이 카운트를 감소시키며, **카운트가 0이 될 때만** 실제 데이터 블록을 회수한다.
- **일관성 (Consistency)**: 여러 경로에서 동일 파일을 수정할 때, 모든 사용자가 즉시 변경 사항을 볼 수 있어야 한다.
- **Dangling Link**: 원본 파일은 삭제되었는데, 이를 가리키는 심볼릭 링크가 남아 있어 존재하지 않는 주소를 가리키는 문제다.

📢 **섹션 요약 비유**: 참조 횟수는 "도서관 책의 대출 예약자 수"와 같다. 예약자가 한 명이라도 남아 있다면 책을 폐기하지 않고 보관해야 하는 것과 같은 원리다.

## Ⅳ. 융합 비교 및 분석 (Comparison)

| 비교 항목 | 트리 구조 (Tree) | 비순환 그래프 (DAG) | 일반 그래프 (General Graph) |
|:---|:---|:---|:---|
| **부모 노드 수** | 오직 1개 | 2개 이상 가능 | 무제한 |
| **공유 가능성** | 낮음 (복제 필요) | 높음 (링크 활용) | 매우 높음 |
| **삭제 복잡도** | 단순 | 보통 (Ref Count 필요) | 높음 (Garbage Collection 필요) |
| **순환(Cycle)** | 절대 불가 | 금지됨 (검사 필요) | 허용됨 |

📢 **섹션 요약 비유**: 트리 구조가 "단방향 통행"이라면, 비순환 그래프는 "여러 갈래 길이 한 곳에서 만나는 교차로"와 같지만, 결코 되돌아오는 길은 없는 상태다.

## Ⅴ. 기술적 구현 및 주의사항 (Implementation)

- **순환 방지 (Cycle Prevention)**: 링크를 생성할 때마다 목적지 디렉터리가 현재 디렉터리의 조상인지 확인하는 알고리즘을 수행하여 트리 구조의 근간이 깨지지 않도록 한다.
- **시스템 호출**: 유닉스/리눅스에서는 `link()` 시스템 콜로 하드 링크를, `symlink()`로 심볼릭 링크를 생성한다.
- **사용 사례**: 공용 라이브러리(/lib)를 각 프로그램의 디렉터리에 링크하거나, 최신 버전의 파일을 `current`라는 이름의 심볼릭 링크로 관리하는 방식이 대표적이다.

📢 **섹션 요약 비유**: 유연함을 위해 트리의 규칙을 조금 깨뜨렸지만, "미로에 빠지지 않기 위해(순환 방지)" 최소한의 안전장치를 둔 합리적인 구조다.
