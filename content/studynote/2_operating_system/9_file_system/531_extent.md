# 531. I/O 우선순위 (I/O Priority)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 작업의 중요도를 결정하는 기준
> 2. **가치**: 중요 작업 우선 처리로 시스템 반응성 향상
> 3. **융합**: 스케줄링, 우선순위 큐, QoS와 연관

---

## Ⅰ. 개요

### 개념 정의
**I/O 우선순위(I/O Priority)**는 **I/O 요청의 중요도를 결정하여 처리 순서를 조정하는 메커니즘**입니다.

### 💡 비유: 응급실 환자 분류
I/O 우선순위는 **응급실에서 환자를 응급도에 따라 분류하는 것**과 같습니다. 위급한 환자부터 먼저 치료합니다.

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                I/O 우선순위 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ionice 명령어】                                                  │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  // 현재 I/O 우선순위 확인                                    │ │   │
│  │  $ ionice -p <PID>                                         │ │   │
│  │                                                             │ │   │
│  │  // I/O 우선순위 설정                                        │ │   │
│  │  $ ionice -c <class> -n <level> -p <PID>                      │ │   │
│  │                                                             │ │   │
│  │  // 클래스별 예시                                            │ │   │
│  │  $ ionice -c 1 -n 0 -p 1234    # RT, 최고 우선순위        │ │   │
│  │  $ ionice -c 2 -n 4 -p 1234    # BE, 기본 우선순위           │ │   │
│  │  $ ionice -c 3 -p 1234          # IDLE, 최저 우선순위          │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【CFQ 스케줄러와 I/O 우선순위】                                     │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  // cfq 비율 확인                                              │ │   │
│  │  $ cat /sys/block/sda/queue/iosched/slice_idle              │ │   │
│  │                                                             │ │   │
│  │  // 프로세스 그룹별 우선순위                                │ │   │
│  │  $ cat /proc/<pid>/io_priority                        │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【cgroups I/O 제어】                                               │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  // cgroup 생성                                                │ │   │
│  │  $ mkdir -p /sys/fs/cgroup/io                          │ │   │
│  │                                                             │ │   │
│  │  // I/O 대역폭 제한 (bytes/s)                                │ │   │
│  │  $ echo "8:0 1048576" > /sys/fs/cgroup/io/throttle.read_bps_device  │ │   │
│  │  $ echo "8:0 1048576" > /sys/fs/cgroup/io/throttle.write_bps_device │ │   │
│  │                                                             │ │   │
│  │  // I/O 작업 수 제한 (IOPS)                                  │ │   │
│  │  $ echo "8:0 1000" > /sys/fs/cgroup/io/throttle.read_iops_device    │ │   │
│  │  $ echo "8:0 1000" > /sys/fs/cgroup/io/throttle.write_iops_device   │ │   │
│  │                                                             │ │   │
│  │  // 프로세스 추가                                           │ │   │
│  │  $ echo <pid> > /sys/fs/cgroup/io/cgroup.procs                │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【장단점】                                                 │
│  ──────────────────                                            │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  장점:                                                       │ │   │
│  │  • 중요 작업 우선 처리                                        │ │   │
│  │  • 시스템 반응성 향상                                          │ │   │
│  │  • 백그라운드 작업 격리                                      │ │   │
│  │  • 자원 공정성 확보                                          │ │   │
│  │                                                             │ │   │
│  │  단점:                                                       │ │   │
│  │  • 우선순위 반전 위험                                        │ │   │
│  │  • 구현 복잡성                                              │ │   │
│  │  • 기아 상태 가능성                                              │ │   │
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프로세스 I/O 우선순위 설정】                                  │
│  ──────────────────                                                │
│  #include <sys/syscall.h>                              │
│  #include <unistd.h>                              │
│  #include <sys/resource.h>                            │
│                                                             │
│  int set_io_priority(int class, int level) {                       │
│      struct sched_param param = {0};                              │
│      param.sched_priority = (class << 13) | level;               │
│      return syscall(SYS_sched_setparam, 0, &param, sizeof(param));  │
│  }                                                     │   │   │
│                                                             │
│  // 실시간 I/O                                │
│  set_io_priority(1, 0);  // RT, 최고               │
│                                                             │
│  // 일반 I/O                                │
│  set_io_priority(2, 4);  // BE, 기본               │
│                                                             │
│  // 백그라운드 I/O                            │
│  set_io_priority(3, 7);  // IDLE, 최저             │
│                                                             │
│  【워크로드별 권장 설정】                                            │
│  ──────────────────                                                │
│  용도              클래스    레벨    설명                        │
│  ────            ─────    ────    ────                        │
│  데이터베이스      RT       0-2     높은 우선순위, 최소 지연    │
│  웹서버          BE       4-6     일반적, 균형 작당            │
│  로그 수집        IDLE     7       낮은 우선순위, 백그라운드          │
│  백업            IDLE     7       낮은 우선순위, 사용자 영향 최소화      │
│  파일 인덱싱      BE       4-6     일반적, 균형 작달            │
│  실시간 처리      RT       0       최고 우선순위, 엄격한 지연 요구사항  │
│                                                             │
│  【I/O 스케줄러별 지원】                                            │
│  ──────────────────                                                │
│  스케줄러         I/O 우선순위 지원                           │
│  ────────         ────────────────                           │
│  CFQ             완전 지원                               │
│  Deadline        부분 지원                               │
│  noop            미지원                              │
│  BFQ             완전 지원                               │
│  mq-deadline     부분 지원                               │
│                                                             │
│  【모니터링】                                                 │
│  ──────────────────                                            │
│  // I/O 우선순위 확인                                          │
│  $ ionice -p $$                                              │
│                                                             │
│  // 실시간 모니터링                                              │
│  $ iotat -x 1                                              │
│                                                             │
│  // 프로세스별 I/O 통계                                      │
│  $ pidstat -p $$                                              │
│                                                             │
│  // cgroup I/O 통계                                           │
│  $ cat /sys/fs/cgroup/io/io.stat                        │
│                                                             │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: I/O 요청의 중요도 결정
• 레벨: RT(0-7), BE(0-7), IDLE
• 도구: ionice, cgroups
• CFQ: 완전한 우선순위 지원
• Deadline: 부분 지원
• 설정: syscall(SYS_sched_setparam)
• 모니터링: iotat, pidstat
 cgroup
• 용도: 데이터베이스, 웹서버, 백업
 실시간
```
