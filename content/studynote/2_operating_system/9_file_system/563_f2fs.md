# 563. SATA (Serial ATA)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 직렬 ATA 저장 장치 인터페이스
> 2. **가치**: 고속, 핫 스왑, NCQ 지원
> 3. **융합**: HDD, SSD, AHCI와 연관

---

## Ⅰ. 개요

### 개념 정의
**SATA(Serial ATA)**는 **저장 장치를 위한 직렬 인터페이스 표준**입니다.

### 💡 비유: 고속 데이터 파이프
SATA는 **고속 데이터 파이프**와 같습니다. 저장 장치로 데이터를 빠르게 보냅니다.

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                SATA 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SATA 프로토콜 계층】                                               │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  계층              설명                                         │ │   │
│  │  ────              ────                                         │ │   │
│  │  Application      명령 (READ/WRITE DMA)                        │ │   │
│  │  Command          FIS (Frame Information Structure)            │ │   │
│  │  Transport        CRC, Ack/Nak                                │ │   │
│  │  Link             8b/10b 인코딩                                │ │   │
│  │  Phy              물리 신호 (차동)                              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【NCQ (Native Command Queuing)】                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  • 명령을 재정렬하여 최적화                                      │ │   │
│  │  • 최대 32개 명령 큐잉                                          │ │   │
│  │  • 헤드 이동 최소화 (HDD)                                       │ │   │
│  │                                                             │ │   │
│  │  비 NCQ: [A→B→C] 순차 실행                                      │ │   │
│  │  NCQ: [A→C→B] 최적 경로로 재정렬                                │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【AHCI (Advanced Host Controller Interface)】                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  • SATA 컨트롤러 표준 인터페이스                                 │ │   │
│  │  • NCQ, 핫 플러그 지원                                          │ │   │
│  │  • 메모리 매핑 I/O (MMIO)                                       │ │   │
│  │  • 32개 명령 슬롯                                               │ │   │
│  │                                                             │ │   │
│  │  레거시: IDE 모드 (NCQ 미지원)                                   │ │   │
│  │  최신: AHCI 모드 (NCQ 지원)                                     │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 직렬 ATA 저장 장치 인터페이스
• 버전: 1.5G → 3G → 6G → 16G bps
• 케이블: 7핀 데이터, 15핀 전력
• 프로토콜: Command, Transport, Link, Phy
• NCQ: 최대 32개 명령 큐잉
• AHCI: SATA 컨트롤러 표준
• 핫 스왑: 지원
• 폼팩터: 3.5", 2.5", mSATA, M.2
• Linux: /dev/sdX, hdparm, smartctl
• 비교: PATA보다 빠르고 간편
```
