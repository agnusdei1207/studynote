## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크 캐시(Disk Cache)와 페이지 캐시(Page Cache)는 느린 보조기억장치의 데이터를 빠른 주기억장치(RAM)에 복사해두어 I/O 성능 차이를 극복하는 완충(Buffer) 메커니즘이다.
> 2. **가치**: 반복적인 디스크 접근을 메모리 참조로 대체함으로써 응답 시간을 수천 배 단축하고, 시스템의 전체적인 처리량을 향상시킨다.
> 3. **융합**: 현대 운영체제는 남는 RAM 메모리를 모두 페이지 캐시로 활용하며, 가상 메모리 관리(Virtual Memory Management) 시스템과 긴밀하게 통합되어 작동한다.

---

## Ⅰ. 캐시의 정의와 필요성 (Cache Definition & Necessity)

- **디스크 캐시 (Disk Cache)**: 넓은 의미로는 디스크 성능 향상을 위한 모든 캐시를 뜻하며, 좁게는 디스크 컨트롤러 내부에 내장된 하드웨어 캐시를 의미한다.
- **페이지 캐시 (Page Cache)**: 운영체제 커널이 파일 데이터를 관리하는 단위인 페이지(Page) 단위로 메인 메모리에 저장해두는 소프트웨어 캐시다.
- **지역성 (Locality)**: 한 번 읽은 데이터는 다시 읽을 가능성이 높고(시간 지역성), 근처 데이터도 읽힐 가능성이 높다(공간 지역성)는 원리를 이용한다.

📢 **섹션 요약 비유**: 캐시는 "책상 위에 자주 보는 책을 꺼내두는 것"과 같다. 매번 서재(디스크)에 가기보다 책상(메모리)에서 바로 보는 게 훨씬 빠르다.

---

## Ⅱ. 페이지 캐시의 작동 구조 (Architecture of Page Cache)

커널은 파일 읽기 요청 시 먼저 페이지 캐시를 확인한다.

### 1. 캐시 히트 및 미스 (Hit & Miss)
- **Cache Hit**: 요청한 데이터가 메모리에 있어 즉시 반환 (매우 빠름).
- **Cache Miss**: 데이터가 메모리에 없어 디스크에서 읽어온 후 캐시에 저장 (느림).

### 2. 작동 흐름도 (ASCII Diagram)

```text
 [ Application ]          [ RAM (Page Cache) ]          [ Disk (Storage) ]
        │                        │                             │
        │ 1. Read(File)          │                             │
        ├──────────────────────▶ │ (Check Cache)               │
        │                        │                             │
        │                        │ ── [ Case 1: Hit ] ──┐      │
        │ ◀──────────────────────┤ <── Result Data ─────┘      │
        │                        │                             │
        │                        │ ── [ Case 2: Miss ] ─┐      │
        │                        │                      │      │
        │                        │      2. Fetch Data   ▼      │
        │ ◀──────────────────────┼──────────────────────┤      │
        │        3. Return       │                      │      │
        └────────────────────────┴──────────────────────┴──────┘
```

📢 **섹션 요약 비유**: 캐시 히트는 "냉장고에 우유가 있는 것"이고, 미스는 "우유를 사러 마트(디스크)까지 가야 하는 것"이다.

---

## Ⅲ. 쓰기 전략과 데이터 일관성 (Write Strategies & Consistency)

캐시에 있는 데이터가 수정되었을 때 디스크와 동기화하는 전략이다.

- **Write-Through**: 캐시와 디스크에 동시에 쓴다. 일관성은 완벽하지만 쓰기 성능이 디스크 속도에 제한된다.
- **Write-Back (Delayed Write)**: 캐시에만 먼저 쓰고, 나중에 한꺼번에 디스크에 쓴다. 성능은 뛰어나지만 갑작스러운 전원 차단 시 데이터 손실 위험이 있다.
- **더티 페이지 (Dirty Page)**: 메모리에는 수정되었으나 아직 디스크에는 기록되지 않은 페이지를 의미한다.

📢 **섹션 요약 비유**: Write-Through는 "글자를 쓸 때마다 저장 버튼을 누르는 것"이고, Write-Back은 "글을 다 쓰고 나중에 한꺼번에 저장하는 것"이다.

---

## Ⅳ. 페이지 교체 알고리즘 (Page Replacement Algorithms)

메모리가 가득 찼을 때 어떤 데이터를 내보낼지 결정하는 전략이다.

- **LRU (Least Recently Used)**: 가장 오랫동안 사용되지 않은 데이터를 내보낸다. 가장 일반적이고 효율적이다.
- **LFU (Least Frequently Used)**: 사용 빈도가 가장 낮은 데이터를 내보낸다.
- **Clock Algorithm**: LRU를 근사화하여 적은 비용으로 효율적으로 작동하게 만든 알고리즘이다.

📢 **섹션 요약 비유**: 페이지 교체는 "꽉 찬 옷장에서 안 입는 옷을 골라 버리는 것"과 같다. 유행이 지난(LRU) 옷부터 버리는 게 합리적이다.

---

## Ⅴ. 하드웨어 디스크 캐시와의 차이 (HW vs SW Cache)

- **관리 주체**: 페이지 캐시는 OS 커널이, 디스크 캐시는 디스크 컨트롤러 하드웨어가 관리한다.
- **가시성**: 페이지 캐시는 사용자 프로세스가 시스템 콜을 통해 간접적으로 제어할 수 있지만, 하드웨어 캐시는 투명(Transparent)하게 작동한다.
- **성능 시너지**: 두 캐시가 계층적으로 작동하여 디스크 I/O의 병목을 이중으로 방어한다.

📢 **섹션 요약 비유**: 페이지 캐시는 "내 가방 안의 간식"이고, 디스크 캐시는 "편의점 창고의 재고"와 같다. 둘 다 가까이 있어 내 입(CPU)까지 오는 시간을 줄여준다.
