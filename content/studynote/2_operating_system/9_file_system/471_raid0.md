# 471. RAID 0 (Striping)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 여러 디스크에 분산 저장
> 2. **가치**: 성능 향상 (읽기/쓰기 모두)
> 3. **융합**: RAID, 디스크 성능과 연관

---

## Ⅰ. 개요

### 개념 정의

RAID 0(Striping)은 **데이터를 스트라이프 단위로 나누어 여러 디스크에 분산 저장하는 방식**이다.

### 💡 비유: 고속도로 차선
RAID 0은 **고속도로 차선**과 같다. 1차선보다 4차선이 더 많은 차가 동시에 지나갈 수 있다.

### RAID 0 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 0 스트라이핑                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  데이터: [A][B][C][D][E][F][G][H]                                   │ │
│  │                                                             │ │
│  │  스트라이프 크기: 2 블록                                              │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                                                         │    │ │
│  │  │    Disk 0         Disk 1         Disk 2         Disk 3  │    │ │
│  │  │   ┌─────┐        ┌─────┐        ┌─────┐        ┌─────┐  │    │ │
│  │  │   │  A  │        │  B  │        │  C  │        │  D  │  │    │ │
│  │  │   ├─────┤        ├─────┤        ├─────┤        ├─────┤  │    │ │
│  │  │   │  E  │        │  F  │        │  G  │        │  H  │  │    │ │
│  │  │   ├─────┤        ├─────┤        ├─────┤        ├─────┤  │    │ │
│  │  │   │  I  │        │  J  │        │  K  │        │  L  │  │    │ │
│  │  │   └─────┘        └─────┘        └─────┘        └─────┘  │    │ │
│  │  │                                                         │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  동시에 4개 디스크에 쓰기 → 4배 성능 (이론적)                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스트라이프 크기 영향】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  작은 스트라이프 (16KB-64KB):                                          │ │
│  │  • 많은 작은 I/O에 유리                                                │ │
│  │  • 병렬성 높음                                                         │ │
│  │  • 랜덤 I/O 성능 좋음                                                  │ │
│  │                                                             │ │
│  │  큰 스트라이프 (256KB-1MB):                                            │ │
│  │  • 순차 I/O에 유리                                                     │ │
│  │  • 오버헤드 감소                                                       │ │
│  │  • 대용량 파일 전송 좋음                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 0 상세                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 성능 향상:                                                        │ │
│  │     • 읽기: N개 디스크에서 동시 읽기                                    │ │
│  │     • 쓰기: N개 디스크에 동시 쓰기                                      │ │
│  │     • 이론적 성능: N배 (실제는 약 70-90%)                               │ │
│  │                                                             │ │
│  │  2. 용량 효율:                                                        │ │
│  │     • 모든 디스크 공간 사용 가능                                        │ │
│  │     • 용량 효율: 100%                                                 │ │
│  │     • 예: 4×1TB = 4TB 사용 가능                                       │ │
│  │                                                             │ │
│  │  3. 비용 효율:                                                        │ │
│  │     • 추가 오버헤드 없음                                                │ │
│  │     • 패리티 계산 불필요                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 내결함성 없음:                                                    │ │
│  │     • 디스크 1개 고장 → 모든 데이터 손실                                 │ │
│  │     • 신뢰성: 단일 디스크보다 낮음                                       │ │
│  │     • N개 디스크 → 고장 확률 N배                                        │ │
│  │                                                             │ │
│  │  2. 복구 불가능:                                                      │ │
│  │     • 고장 시 데이터 복구 방법 없음                                      │ │
│  │     • 반드시 백업 필요                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【신뢰성 계산】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  단일 디스크 MTBF: 1,000,000 시간                                     │ │
│  │  4디스크 RAID 0 MTBF: 1,000,000 / 4 = 250,000 시간                   │ │
│  │                                                             │ │
│  │  결론: 디스크 많을수록 고장 확률 높아짐                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux mdadm RAID 0 생성】                                        │
│  ──────────────────                                                  │
│  // RAID 0 어레이 생성                                                │
│  $ sudo mdadm --create /dev/md0 --level=0 --raid-devices=4 \        │
│        /dev/sdb1 /dev/sdc1 /dev/sdd1 /dev/sde1                      │
│                                                                     │
│  // 스트라이프 크기 지정 (기본 512KB)                                  │
│  $ sudo mdadm --create /dev/md0 --level=0 --raid-devices=4 \        │
│        --chunk=64 /dev/sd{b,c,d,e}1                                 │
│                                                                     │
│  // 상태 확인                                                        │
│  $ cat /proc/mdstat                                                 │
│  $ sudo mdadm --detail /dev/md0                                     │
│                                                                     │
│  // 파일 시스템 생성                                                   │
│  $ sudo mkfs.ext4 /dev/md0                                          │
│  $ sudo mount /dev/md0 /mnt/raid0                                   │
│                                                                     │
│  【성능 테스트】                                                      │
│  ──────────────────                                                  │
│  // 순차 읽기/쓰기                                                     │
│  $ sudo hdparm -tT /dev/md0                                         │
│                                                                     │
│  // fio 벤치마크                                                      │
│  $ fio --name=test --ioengine=libaio --rw=randread \                │
│        --bs=4k --numjobs=4 --size=1G --filename=/dev/md0            │
│                                                                     │
│  // dd로 순차 쓰기 테스트                                               │
│  $ dd if=/dev/zero of=/mnt/raid0/testfile bs=1M count=1000         │
│  1048576000 bytes (1.0 GB) copied, 2.5 s, 400 MB/s                  │
│  // 4디스크라 단일 디스크(100MB/s) 대비 4배                              │
│                                                                     │
│  【적합한 사용 사례】                                                  │
│  ──────────────────                                                  │
│  • 임시 데이터 처리                                                    │
│  • 비디오/오디오 편집 (대용량 순차)                                       │
│  • 캐시 저장소                                                        │
│  • 스크래치 디스크                                                      │
│  • 백업된 데이터 (빠른 복원용)                                          │
│                                                                     │
│  【부적합한 사용 사례】                                                │
│  ──────────────────                                                  │
│  • 중요 데이터 저장                                                    │
│  • 데이터베이스 (미러링 필요)                                           │
│  • 운영 환경 서버                                                      │
│  • 장기 보관 데이터                                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 데이터 스트라이핑으로 분산 저장
• 장점: 최고 성능, 100% 용량 활용
• 단점: 내결함성 없음, 디스크 고장 시 전체 손실
• 성능: N배 (이론), 70-90% (실제)
• 스트라이프: 작으면 랜덤, 크면 순차에 유리
• 신뢰성: 디스크 수에 반비례
• 용도: 임시, 캐시, 비디오 편집
• 주의: 반드시 백업 필요
```
