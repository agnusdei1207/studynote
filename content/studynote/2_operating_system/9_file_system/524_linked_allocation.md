# 524. 인터럽트 처리 지연 (Interrupt Latency)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 발생 시 CPU가 직접 처리
> 2. **가치**: 빠른 응답과 하드웨어 돴륑

 3. **융합**: 인터럽트, 예외 처리와 연관

---

## Ⅰ. 개요

### 개념 정의
**인터럽트(Interrupt Latency)**는 인터럽트 발생 시 CPU가 직접 처리하여 ** 빠른 응답을 가능하게 하고, 하드웨어 자원을 보다 효율적으로 만든 기술**입니다.

### 💡 비유: 택배 게임
인터럽트는 **무료 샘플 진행을 수행하는 미니 게임**과 같다. 업데이나 시간을 동적으로 변경하고 복잡한 점수(FPS) 수에 영향

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 인터럽트 → CPU 개입 → 핸들러 → 복귀 실행
• 종류: 외부, 내부, 소프트웨어 인터럽트

• IRQ: ISA 버스, IRQ 할당,• PIC/APIC: 우선순위 관리,• 컨트롤러: 인터럽트 요청 및 응답 관리
• 상/하반응: Top Half(빠름), vs Bottom Half(지연)
 나• 처리 시간 목표: 하드(<100μs), 일반(100μs~1ms), 소프트(수ms~수백ms)
• Latency: 하드웨어 지연 + 소프트웨어 처리
```
