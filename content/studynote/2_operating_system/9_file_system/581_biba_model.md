# 581. 저널링 (Journaling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일시스템 변경 사항을 로그에 기록
> 2. **가치**: 빠른 복구, 데이터 무결성 보장
> 3. **융합**: 트랜잭션, ext4, XFS와 연관

---

## Ⅰ. 개요

### 개념 정의
**저널링(Journaling)**은 **파일시스템 변경 사항을 로그에 먼저 기록하는 기법**입니다.

### 💡 비유: 작업 일지
저널링은 **작업 일지**와 같습니다. 작업 전에 계획을 적어둡니다.

### 저널링 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                저널링 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【저널링 없는 파일시스템】                                            │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  일반 쓰기:                                                    │ │   │
│  │  1. 데이터 쓰기 → 2. 메타데이터 갱신 → 3. 완료                    │ │   │
│  │                                                             │ │   │
│  │  문제점:                                                        │ │   │
│  │  • 1~2 사이 정전: 데이터는 있고 메타데이터 없음                    │ │   │
│  │  • 2~3 사이 정전: 메타데이터 불일치                               │ │   │
│  │  • 복구 위해 전체 파일시스템 스캔 필요 (fsck)                       │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【저널링 파일시스템】                                                 │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  저널링 쓰기:                                                   │ │   │
│  │  1. 저널에 변경 사항 기록 → 2. 실제 데이터 쓰기 → 3. 저널 커밋       │ │   │
│  │                                                             │ │   │
│  │  장점:                                                         │ │   │
│  │  • 빠른 복구 (저널만 재생)                                        │ │   │
│  │  • 데이터 무결성 보장                                             │ │   │
│  │  • fsck 시간 단축                                               │ │   │
│  │                                                             │ │   │
│  │  ┌─────────────────────────────────────────────────────┐    │ │   │
│  │  │                저널 구조                              │    │ │   │
│  │  ├─────────────────────────────────────────────────────┤    │ │   │
│  │  │  [트랜잭션 시작] [데이터] [메타데이터] [커밋]             │    │ │   │
│  │  └─────────────────────────────────────────────────────┘    │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【저널링 모드】                                                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  모드              설명                    성능    안전성        │ │   │
│  │  ────              ────                    ────    ────         │ │   │
│  │  Journal           모든 데이터+메타데이터     낮음    높음          │ │   │
│  │  Ordered           메타데이터만 (기본)       중간    중간          │ │   │
│  │  Writeback         메타데이터만 (빠름)       높음    낮음          │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                저널링 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【저널링 모드 상세】                                                  │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  1. Journal (data=journal):                                   │ │   │
│  │     • 모든 데이터와 메타데이터를 저널에 기록                        │ │   │
│  │     • 가장 안전하지만 느림                                       │ │   │
│  │     • 오버헤드 큼                                               │ │   │
│  │                                                             │ │   │
│  │  2. Ordered (data=ordered) [기본값]:                            │ │   │
│  │     • 메타데이터만 저널에 기록                                    │ │   │
│  │     • 데이터는 메타데이터보다 먼저 디스크에 기록                      │ │   │
│  │     • 균형 잡힌 성능과 안전성                                     │ │   │
│  │                                                             │ │   │
│  │  3. Writeback (data=writeback):                               │ │   │
│  │     • 메타데이터만 저널에 기록                                    │ │   │
│  │     • 데이터 순서 보장 안 함                                      │ │   │
│  │     • 가장 빠르지만 덜 안전                                       │ │   │
│  │     • 정전 시 이전 데이터가 보일 수 있음                             │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【트랜잭션 처리】                                                     │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  트랜잭션 단계:                                                  │ │   │
│  │  ┌─────────────────────────────────────────────────────┐    │ │   │
│  │  │  1. Begin    - 트랜잭션 시작                          │    │ │   │
│  │  │  2. Log      - 저널에 변경 내용 기록                   │    │ │   │
│  │  │   3. Commit   - 저널에 커밋 기록                       │    │ │   │
│  │  │  4. Checkpoint - 실제 디스크에 쓰기                    │    │ │   │
│  │  │  5. Complete  - 저널에서 트랜잭션 제거                  │    │ │   │
│  │  └─────────────────────────────────────────────────────┘    │ │   │
│  │                                                             │ │   │
│  │  원자성 보장:                                                    │ │   │
│  │  • 커밋 전 장애: 트랜잭션 무시                                    │ │   │
│  │  • 커밋 후 장애: 트랜잭션 재생                                     │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【저널링 파일시스템 비교】                                             │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  파일시스템       저널링 방식         특징                       │ │   │
│  │  ────────         ─────────         ────                       │ │   │
│  │  ext3/ext4        메타데이터          journal/ordered/writeback │ │   │
│  │  XFS              메타데이터          고성능                    │ │   │
│  │  NTFS             $LogFile           윈도우                    │ │   │
│  │  JFS              메타데이터          IBM                       │ │   │
│  │  ReiserFS         메타데이터          SUSE                      │ │   │
│  │  Btrfs            COW (Copy-on-Write)  스냅샷                   │ │   │
│  │  ZFS              COW + ZIL          Sun                       │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【저널링 모드 확인/변경】                                             │
│  ──────────────────                                                │
│  // 현재 저널링 모드 확인                                              │
│  $ cat /proc/mounts | grep ext4                                    │
│  $ sudo tune2fs -l /dev/sda1 | grep journal                        │
│  $ sudo dumpe2fs /dev/sda1 | grep "Filesystem features"            │
│                                                                     │
│  // 저널링 모드 변경 (마운트 시)                                        │
│  $ sudo mount -o data=journal /dev/sda1 /mnt                       │
│  $ sudo mount -o data=ordered /dev/sda1 /mnt                       │
│  $ sudo mount -o data=writeback /dev/sda1 /mnt                     │
│                                                                     │
│  // 영구 변경 (tune2fs)                                               │
│  $ sudo tune2fs -o journal_data /dev/sda1                          │
│  $ sudo tune2fs -o journal_data_ordered /dev/sda1                  │
│  $ sudo tune2fs -o journal_data_writeback /dev/sda1                │
│                                                                     │
│  // /etc/fstab                                                       │
│  /dev/sda1  /  ext4  defaults,data=ordered  0  1                   │
│                                                                     │
│  【저널 비활성화/활성화】                                               │
│  ──────────────────                                                │
│  // 저널 비활성화 (성능 향상, 위험)                                      │
│  $ sudo tune2fs -O ^has_journal /dev/sda1                          │
│  $ sudo e2fsck -f /dev/sda1                                        │
│                                                                     │
│  // 저널 활성화 (ext2 → ext3)                                         │
│  $ sudo tune2fs -j /dev/sda1                                       │
│                                                                     │
│  // 저널 크기 확인                                                     │
│  $ sudo dumpe2fs /dev/sda1 | grep -i journal                       │
│                                                                     │
│  【저널 복구】                                                         │
│  ──────────────────                                                │
│  // 저널 재생                                                         │
│  $ sudo e2fsck -p /dev/sda1          // 자동 복구                    │
│  $ sudo e2fsck -f /dev/sda1          // 강제 검사                    │
│                                                                     │
│  // XFS 저널 복구                                                      │
│  $ sudo xfs_repair /dev/sda1                                        │
│  $ sudo xfs_repair -n /dev/sda1      // 읽기 전용 검사               │
│                                                                     │
│  // 저널 초기화 (위험!)                                                 │
│  $ sudo xfs_repair -L /dev/sda1                                    │
│                                                                     │
│  【XFS 저널 설정】                                                      │
│  ──────────────────                                                │
│  // 저널 정보 확인                                                     │
│  $ sudo xfs_info /dev/sda1                                         │
│                                                                     │
│  // 생성 시 저널 설정                                                   │
│  $ sudo mkfs.xfs -l size=256m /dev/sda1     // 저널 크기             │
│  $ sudo mkfs.xfs -l logdev=/dev/sdc1 /dev/sda1  // 외부 저널         │
│                                                                     │
│  // lazy-count 옵션                                                   │
│  $ sudo mkfs.xfs -l lazy-count=1 /dev/sda1                         │
│  // 슈퍼블록 카운터 지연 갱신으로 성능 향상                               │
│                                                                     │
│  【저널링 성능 비교】                                                   │ |
│  ──────────────────                                                │
│  // 벤치마크 (dd)                                                      │
│  $ sudo mount -o data=journal /dev/sda1 /mnt                       │
│  $ dd if=/dev/zero of=/mnt/test bs=1M count=1000                   │
│  $ sudo umount /mnt                                                 │
│                                                                     │
│  $ sudo mount -o data=writeback /dev/sda1 /mnt                     │
│  $ dd if=/dev/zero of=/mnt/test bs=1M count=1000                   │
│  $ sudo umount /mnt                                                 │
│                                                                     │
│  // 일반적인 성능 차이:                                                  │
│  // writeback > ordered > journal                                   │
│                                                                     │
│  【Btrfs/ZFS COW】                                                    │
│  ──────────────────                                                │
│  // Btrfs (Copy-on-Write)                                            │
│  $ sudo mkfs.btrfs /dev/sda1                                       │
│  $ sudo mount /dev/sda1 /mnt                                       │
│  // COW으로 저널링 없이 무결성 보장                                      │
│                                                                     │
│  // ZFS (ZIL - ZFS Intent Log)                                       │
│  $ sudo zpool create mypool /dev/sda1                              │
│  // ZIL로 동기 쓰기 보장                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 파일시스템 변경 사항을 로그에 기록
• 목적: 빠른 복구, 데이터 무결성
• 모드: journal, ordered (기본), writeback
• journal: 데이터+메타데이터, 가장 안전
• ordered: 메타데이터만, 균형
• writeback: 메타데이터만, 가장 빠름
• 트랜잭션: Begin → Log → Commit → Checkpoint
• ext4: data=ordered 기본값
• XFS: 메타데이터 저널링
• Btrfs/ZFS: COW 방식
• 설정: tune2fs -o, mount -o data=
• 복구: e2fsck, xfs_repair
```
