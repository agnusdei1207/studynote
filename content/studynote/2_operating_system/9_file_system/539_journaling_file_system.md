# 539. I/O 가상화 (I/O Virtualization)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 머신에서 I/O 리소스 추상화
> 2. **가치**: 하드웨어 독립성과 리소스 공유
> 3. **융합**: 가상화, 에뮬레이션, 패스스루와 연관

---

## Ⅰ. 개요

### 개념 정의
**I/O 가상화(I/O Virtualization)**는 **가상 머신에서 물리적 I/O 리소스를 추상화하여 공유하는 기술**입니다.

### 💡 비유: 호텔 객실 공유
I/O 가상화는 **호텔에서 객실을 예약하는 것**과 같습니다. 한 건물의 방을 여러 투숙객이 나눠 씁니다.

### I/O 가상화 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                I/O 가상화 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【가상화 방식】                                                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  1. 전 에뮬레이션 (Full Emulation)                              │ │   │
│  │     ────                                                        │ │   │
│  │     가상 머신 ───▶ 하이퍼바이저(에뮬레이션) ───▶ 하드웨어            │ │   │
│  │     장점: 완전한 격리, 이식성                                    │ │   │
│  │     단점: 높은 오버헤드                                         │ │   │
│  │                                                             │ │   │
│  │  2. 패스스루 (Passthrough)                                     │ │   │
│  │     ────                                                        │ │   │
│  │     가상 머신 ───▶ 하이퍼바이저(DMA) ───▶ 하드웨어               │ │   │
│  │     장점: 네이티브 성능                                         │ │   │
│  │     단점: 하드웨어 종속                                         │ │   │
│  │                                                             │ │   │
│  │  3. 반가상화 (Paravirtualization)                              │ │   │
│  │     ────                                                        │ │   │
│  │     가상 머신(프론트엔드) ↔ 하이퍼바이저 ↔ 백엔드 드라이버        │ │   │
│  │     장점: 에뮬레이션 대비 낮은 오버헤드                          │ │   │
│  │     단점: 게스트 OS 수정 필요                                     │ │   │
│  │                                                             │ │   │
│  │  4. SR-IOV (Single Root I/O Virtualization)                    │ │   │
│  │     ────                                                        │ │   │
│  │     PF(물리 기능) ───▶ 여러 VF(가상 기능) ───▶ VM들              │ │   │
│  │     장점: 하드웨어 수준 가상화                                    │ │   │
│  │     단점: 하드웨어 지원 필요                                     │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【IOMMU 역할】                                                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  ┌───────────────────────────────────────────────────────┐   │ │   │
│  │  │                    IOMMU                               │   │ │   │
│  │  │  (Intel VT-d / AMD-Vi)                                 │   │ │   │
│  │  │                                                        │   │ │   │
│  │  │  • DMA 주소 변환 (GPA → HPA)                            │   │ │   │
│  │  │  • 장치 격리 (Device Isolation)                         │   │ │   │
│  │  │  • 인터럽트 리매핑 (Interrupt Remapping)                │   │ │   │
│  │  │  • DMA 보호 (DMA Protection)                            │   │ │   │
│  │  │                                                        │   │ │   │
│  │  └───────────────────────────────────────────────────────┘   │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SR-IOV 설정 예시】                                                │
│  ──────────────────                                                │
│  // VF 수 확인                                                       │
│  $ cat /sys/class/net/eth0/device/sriov_numvfs                      │
│  0                                                                 │
│                                                                     │
│  // VF 생성                                                          │
│  $ echo 4 > /sys/class/net/eth0/device/sriov_numvfs                  │
│  $ cat /sys/class/net/eth0/device/sriov_numvfs                      │
│  4                                                                 │
│                                                                     │
│  // VF 목록 확인                                                      │
│  $ ip link show eth0                                                │
│  2: eth0: <...> mtu 1500 qdisc mq state UP ...                     │
│  3: eth0v0: <...> mtu 1500 qdisc mq state DOWN ...                  │
│  4: eth0v1: <...> mtu 1500 qdisc mq state DOWN ...                  │
│  5: eth0v2: <...> mtu 1500 qdisc mq state DOWN ...                  │
│  6: eth0v3: <...> mtu 1500 qdisc mq state DOWN ...                  │
│                                                                     │
│  // VF를 VM에 할당                                                   │
│  $ virsh nodedev-dumpxml pci_0000_00_19_0                         │
│  $ virsh attach-device vm1 vf0.xml                                 │
│                                                                     │
│  【virtio 설정 (QEMU)】                                               │
│  ──────────────────                                                │
│  // virtio-net 사용                                                  │
│  $ qemu-system-x86_64 -enable-kvm -m 2048 \                         │
│    -drive file=disk.qcow2,if=virtio \                               │
│    -netdev tap,id=net0,ifname=tap0,script=no \                      │
│    -device virtio-net-pci,netdev=net0,mac=52:54:00:12:34:56         │
│                                                                     │
│  // vHost-net 활성화                                                 │
│  $ qemu-system-x86_64 -enable-kvm -m 2048 \                         │
│    -netdev tap,id=net0,ifname=tap0,vhost=on \                        │
│    -device virtio-net-pci,netdev=net0                               │
│                                                                     │
│  【IOMMU 설정 (Intel VT-d)】                                          │
│  ──────────────────                                                │
│  // 커널 파라미터 확인                                                 │
│  $ cat /proc/cmdline | grep intel_iommu                             │
│  intel_iommu=on                                                     │
│                                                                     │
│  // IOMMU 활성화 (GRUB)                                              │
│  GRUB_CMDLINE_LINUX="intel_iommu=on iommu=pt"                       │
│  $ update-grub                                                       │
│                                                                     │
│  // IOMMU 그룹 확인                                                   │
│  $ ls /sys/kernel/iommu_groups/                                      │
│  0  1  2  3  4  5 ...                                               │
│                                                                     │
│  // 장치가 속한 IOMMU 그룹 확인                                        │
│  $ readlink /sys/bus/pci/devices/0000:00:19.0/iommu_group             │
│  ../../../../../kernel/iommu_groups/1                               │
│                                                                     │
│  【PCI 패스스루 설정】                                                 │
│  ──────────────────                                                │
│  // VFIO 드라이버 바인딩                                               │
│  $ echo 0000:00:19.0 > /sys/bus/pci/drivers/vfio-pci/bind            │
│                                                                     │
│  // 또는 새 ID로 바인딩                                                │
│  $ echo 19ee 8086 > /sys/bus/pci/drivers/vfio-pci/new_id            │
│                                                                     │
│  // QEMU에서 패스스루 사용                                             │
│  $ qemu-system-x86_64 -enable-kvm -m 2048 \                         │
│    -device vfio-pci,host=00:19.0                                   │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                │
│  방식              상대 성능    지연 시간     CPU 사용                │
│  ────              ────────    ────────     ────────                │
│  전 에뮬레이션      ~10%        높음         높음                    │
│  Virtio            ~90%        중간         중간                    │
│  vHost             ~95%        낮음         낮음                    │
│  SR-IOV            ~98%        매우 낮음      매우 낮음                 │
│  PCI 패스스루      ~100%       최저         최저                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 가상 머신에서 물리적 I/O 리소스 추상화
• 방식: 전 에뮬레이션, 패스스루, 반가상화, SR-IOV
• IOMMU: DMA 주소 변환, 장치 격리, 인터럽트 리매핑
• Virtio: 프론트엔드/백엔드 아키텍처, 링 버퍼
• SR-IOV: PF(물리 기능), VF(가상 기능)
• vHost: 커널 수준 네트워크 처리
• 성능: 에뮬레이션 < Virtio < vHost < SR-IOV < 패스스루
• 기술: Intel VT-d, AMD-Vi, VFIO
```
