## 핵심 인사이트 (3줄 요약)
> 1. **통합 스토리지**: Ceph는 객체(Object), 블록(Block), 파일(File) 스토리지를 하나의 분산 클러스터에서 모두 제공하는 통합형 오픈소스 저장 플랫폼이다.
> 2. **지능적 분산**: 중앙 집중식 메타데이터 조회 대신 CRUSH 알고리즘을 통한 계산 기반 데이터 배치를 사용하여 병목 현상을 제거하고 무한한 확장성을 확보했다.
> 3. **자가 치유**: 시스템 내부의 OSD(Object Storage Device)들이 서로 상태를 감시하며, 장애 발생 시 관리자의 개입 없이 데이터를 자동으로 재복제하고 복구하는 자가 관리(Self-managing) 기능을 갖추고 있다.

---

## Ⅰ. Ceph의 정의와 통합 스토리지 (Unified Storage)

- **정의**: 하드웨어 독립적인 소프트웨어 정의 스토리지(SDS, Software-Defined Storage)로, 데이터의 고가용성과 확장성을 보장하는 분산 시스템이다.
- **통합 아키텍처 (Three-in-One)**:
    - **Ceph FS**: POSIX 호환 분산 파일 시스템 제공.
    - **RBD (RADOS Block Device)**: 가상 머신(VM)을 위한 블록 스토리지 제공.
    - **RADOSGW (Object Gateway)**: S3/Swift 호환 객체 스토리지 제공.

📢 **섹션 요약 비유**: Ceph는 "만능 수납함"과 같다. 옷(객체), 책(블록), 잡동사니(파일)를 모두 담을 수 있으면서, 수납함 크기를 언제든 무한대로 늘릴 수 있는 똑똑한 시스템이다.

---

## Ⅱ. Ceph 시스템 구조 (RADOS Architecture)

Ceph의 핵심은 '신뢰할 수 있는 분산 객체 저장소'인 RADOS(Reliable Autonomic Distributed Object Store)다.

### 1. 계층적 논리도 (ASCII Diagram)

```text
       [ Clients: CephFS / RBD / RGW ]
    ┌───────────────────────────────────┐
    │          LIBRADOS (API)           │
    └───────────────────────────────────┘
                    ▲
                    │ (CRUSH Algorithm)
                    ▼
 [ Monitors (MON) ]         [ OSD Nodes ]
 ┌────────────────┐   ┌───────────────────────┐
 │ Cluster Map    │   │ [OSD 1] [OSD 2] [OSD 3]│
 │ Consensus (Paxos)  │   │ [OSD 4] [OSD 5] [OSD 6]│
 └────────────────┘   └───────────────────────┘
                    ▲
                    │ (Peering & Replication)
                    ▼
          [ Self-Healing Clusters ]
```

📢 **섹션 요약 비유**: Ceph 아키텍처는 "자율 주행 물류 센터"와 같다. 중앙 통제관(Monitor)은 전체 지도만 관리하고, 실제 물건 이동과 관리는 로봇들(OSD)이 서로 소통하며 스스로 처리한다.

---

## Ⅲ. CRUSH 알고리즘 - 계산 기반 배치

- **개념**: CRUSH (Controlled Replication Under Scalable Hashing) 알고리즘은 데이터를 어디에 저장할지 '찾는' 것이 아니라 '계산'하는 방식이다.
- **특징**:
    - **No Lookup Table**: 중앙의 메타데이터 테이블이 필요 없어 병목 현상이 발생하지 않는다.
    - **확정적 배치**: 동일한 입력값(데이터 ID + 클러스터 맵)에 대해 항상 동일한 위치를 산출한다.
    - **계층 인식**: 서버, 랙, 데이터센터 단위의 물리적 배치를 고려하여 복제본을 분산한다.

📢 **섹션 요약 비유**: CRUSH 알고리즘은 "내비게이션"과 같다. 목적지를 일일이 사람에게 물어보는 대신, 지도 데이터(클러스터 맵)와 알고리즘을 이용해 스스로 길을 찾아가는 것과 같다.

---

## Ⅳ. 주요 구성 요소의 역할

- **OSD (Object Storage Device)**: 실제 데이터를 저장하고 복제, 복구, 리밸런싱을 수행한다. 일반적으로 디스크 하나당 하나의 OSD 프로세스가 동작한다.
- **Monitor (MON)**: 클러스터 맵의 마스터 사본을 유지하며 노드들의 생존 상태를 감시한다. (다수결 합의 Paxos 알고리즘 사용)
- **MDS (Metadata Server)**: Ceph FS(파일 시스템)를 사용할 때만 필요하며, 파일 계층 구조와 메타데이터를 관리한다.
- **Manager (MGR)**: 클러스터의 상태 정보, 성능 지표 등을 수집하고 모니터링 대시보드를 제공한다.

📢 **섹션 요약 비유**: OSD는 "현장 일꾼", Monitor는 "보안 요원", MDS는 "장부 기록원"이다. 일꾼들이 일을 하고 보안 요원이 상태를 체크하며, 기록원이 복잡한 서류를 정리하는 구조다.

---

## Ⅴ. 자가 치유(Self-healing)와 확장성

- **동적 리밸런싱 (Rebalancing)**: 새로운 노드가 추가되거나 기존 노드가 제거되면 CRUSH 알고리즘이 자동으로 데이터 배치를 다시 계산하여 부하를 분산시킨다.
- **피어링 (Peering)**: OSD들끼리 서로의 상태를 체크하며 데이터 일관성을 확인한다.
- **자동 복구 (Recovery)**: OSD 장애 감지 시, 다른 OSD들이 가진 복제본을 이용해 즉시 새로운 복제본을 생성하여 가용성을 복구한다.

📢 **섹션 요약 비유**: 자가 치유는 "도마뱀의 꼬리 재생"과 같다. 일부가 잘려 나가도(노드 장애) 스스로 알아서 원래의 건강한 상태(데이터 가용성)로 되돌리는 능력을 갖추고 있다.
