# 585. fsck (File System Check)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일시스템 무결성 검사 및 복구 도구
> 2. **가치**: 데이터 손상 방지, 복구
> 3. **융합**: 저널링, 슈퍼블록, 파일시스템과 연관

---

## Ⅰ. 개요

### 개념 정의
**fsck(File System Check)**는 **파일시스템의 무결성을 검사하고 오류를 복구하는 도구**입니다.

### 💡 비유: 건물 안전 점검
fsck는 **건물 안전 점검**과 같습니다. 구조적 문제를 찾아 수리합니다.

### fsck 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                fsck 구조                                             │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【fsck 검사 항목】                                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  검사 항목           설명                    오류 유형             │ │   │
│  │  ────────            ────                    ────────             │ │   │
│  │  슈퍼블록            파일시스템 메타데이터        손상, 불일치          │ │   │
│  │  inode              파일 메타데이터           고아 inode            │ │   │
│  │  디렉토리            디렉토리 구조             연결 끊김             │ │   │
│  │  블록               데이터 블록              중복, 누락             │ │   │
│  │  링크               하드/심볼릭 링크           끊어진 링크            │ │   │
│  │  비트맵             할당 비트맵              불일치                │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【파일시스템별 fsck 도구】                                            │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  파일시스템       fsck 도구              설명                    │ │   │
│  │  ────────         ─────────             ────                    │ │   │
│  │  ext2/3/4         e2fsck/fsck.ext4      ext 계열                │ │   │
│  │  XFS              xfs_repair            XFS 전용                │ │   │
│  │  Btrfs            btrfs check           Btrfs 전용              │ │   │
│  │  FAT              fsck.vfat/dosfsck     FAT 계열                │ │   │
│  │  NTFS             ntfsfix               NTFS 기본 복구           │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【fsck 실행 시점】                                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  자동 실행:                                                      │ │   │
│  │  • 부팅 시 /etc/fstab의 pass 필드가 0이 아닌 경우                  │ │   │
│  │  • 마운트 횟수 초과                                               │ │   │
│  │  • 마지막 검사 후 일정 기간 경과                                    │ │   │
│  │  • 비정상 종료 (dirty 플래그)                                      │ │   │
│  │                                                             │ │   │
│  │  수동 실행:                                                      │ │   │
│  │  • 파일시스템 오류 의심 시                                         │ │   │
│  │  • 디스크 문제 발생 시                                             │ │   │
│  │  • 정기 점검                                                      │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                fsck 상세                                             │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【e2fsck 옵션】                                                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  옵션              설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  -p               자동 복구 (safe)                               │ │   │
│  │  -n               읽기 전용 (변경 안 함)                           │ │   │
│  │  -y               모든 질문에 yes                                 │ │   │
│  │  -f               강제 검사                                       │ │   │
│  │  -v               상세 출력                                       │ │   │
│  │  -c               불량 블록 검사                                   │ │   │
│  │  -b superblock    백업 슈퍼블록 사용                               │ │   │
│  │  -B size          블록 크기 지정                                   │ │   │
│  │  -j journal       외부 저널 지정                                   │ │   │
│  │  -D               디렉토리 최적화                                   │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【fsck 실행 단계】                                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  Pass 1: inode 검사                                              │ │   │
│  │  • inode 유형, 모드, 크기 확인                                    │ │   │
│  │  • 고아 inode 탐지                                                │ │   │
│  │                                                             │ │   │
│  │  Pass 2: 디렉토리 구조 검사                                        │ │   │
│  │  • 디렉토리 엔트리 무결성                                          │ │   │
│  │  • 중복 항목 확인                                                  │ │   │
│  │                                                             │ │   │
│  │  Pass 3: 디렉토리 연결 검사                                        │ │   │
│  │  • 부모-자식 관계 확인                                             │ │   │
│  │  • 루트 디렉토리 연결                                               │ │   │
│  │                                                             │ │   │
│  │  Pass 4: 참조 카운트 검사                                          │ │   │
│  │  • 링크 카운트 일치 확인                                           │ │   │
│  │                                                             │ │   │
│  │  Pass 5: 비트맵 검사                                               │ │   │
│  │  • 블록/inode 비트맵 일치                                          │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【자동 fsck 설정】                                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  // 최대 마운트 횟수                                               │ │   │
│  │  $ sudo tune2fs -c 30 /dev/sda1         // 30회마다              │ │   │
│  │  $ sudo tune2fs -c -1 /dev/sda1         // 비활성화              │ │   │
│  │                                                             │ │   │
│  │  // 시간 기반                                                       │ │   │
│  │  $ sudo tune2fs -i 30d /dev/sda1        // 30일마다              │ │   │
│  │  $ sudo tune2fs -i 0 /dev/sda1          // 비활성화              │ │   │
│  │                                                             │ │   │
│  │  // 현재 설정 확인                                                  │ │   │
│  │  $ sudo tune2fs -l /dev/sda1 | grep -i "check"                  │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ext4 fsck】                                                       │
│  ──────────────────                                                │
│  // 마운트 해제 후 실행 (권장)                                         │
│  $ sudo umount /dev/sda1                                           │
│  $ sudo e2fsck -f /dev/sda1                                        │
│                                                                     │
│  // 읽기 전용 검사                                                    │
│  $ sudo e2fsck -n /dev/sda1                                        │
│                                                                     │
│  // 자동 복구                                                        │
│  $ sudo e2fsck -p /dev/sda1                                        │
│                                                                     │
│  // 모든 질문 yes                                                     │
│  $ sudo e2fsck -y /dev/sda1                                        │
│                                                                     │
│  // 불량 블록 검사                                                    │
│  $ sudo e2fsck -c /dev/sda1                                        │
│  $ sudo e2fsck -cc /dev/sda1         // 자세한 불량 블록 검사          │
│                                                                     │
│  // 백업 슈퍼블록 사용                                                 │
│  $ sudo dumpe2fs /dev/sda1 | grep -i superblock                    │
│  $ sudo e2fsck -b 32768 /dev/sda1                                  │
│                                                                     │
│  【루트 파일시스템 검사】                                               │
│  ──────────────────                                                │
│  // 방법 1: 단일 사용자 모드                                           │
│  $ sudo init 1                                                      │
│  $ sudo e2fsck -f /dev/sda1                                        │
│                                                                     │
│  // 방법 2: 재부팅 시 강제 검사                                         │
│  $ sudo touch /forcefsck                                           │
│  $ sudo reboot                                                      │
│                                                                     │
│  // 방법 3: shutdown 시                                              │
│  $ sudo shutdown -F -r now                                         │
│                                                                     │
│  【XFS 복구】                                                         │
│  ──────────────────                                                │
│  // 읽기 전용 검사                                                    │
│  $ sudo xfs_repair -n /dev/sda1                                    │
│                                                                     │
│  // 복구                                                            │
│  $ sudo xfs_repair /dev/sda1                                       │
│                                                                     │
│  // 강제 로그 초기화 (위험!)                                           │
│  $ sudo xfs_repair -L /dev/sda1                                    │
│                                                                     │
│  // 메타데이터 덤프                                                    │
│  $ sudo xfs_metadump /dev/sda1 dump.metadump                       │
│  $ sudo xfs_mdrestore dump.metadump /dev/sda1                      │
│                                                                     │
│  【FAT/NTFS 검사】                                                   │ |
│  ──────────────────                                                │
│  // FAT32                                                           │
│  $ sudo dosfsck -a /dev/sdb1         // 자동 복구                    │
│  $ sudo dosfsck -v /dev/sdb1         // 상세 출력                    │
│  $ sudo fsck.vfat /dev/sdb1                                        │
│                                                                     │
│  // NTFS 기본 복구                                                    │
│  $ sudo ntfsfix /dev/sdb1                                          │
│                                                                     │
│  // Windows에서 chkdsk                                               │
│  > chkdsk E: /f                                                     │
│  > chkdsk E: /f /r                                                  │
│                                                                     │
│  【fsck 결과 해석】                                                   │ |
│  ──────────────────                                                │
│  // 정상                                                             │
│  /dev/sda1: clean, 1234/65536 files, 56789/262144 blocks           │
│                                                                     │
│  // 복구 완료                                                         │
│  /dev/sda1: ***** FILE SYSTEM WAS MODIFIED *****                   │
│  /dev/sda1: 1235/65536 files (0.2% non-contiguous), 56790/262144 blocks│
│                                                                     │
│  // 오류                                                             │
│  /dev/sda1: UNEXPECTED INCONSISTENCY; RUN fsck MANUALLY.           │
│  // -> e2fsck -y /dev/sda1 실행                                     │
│                                                                     │
│  【fstab pass 필드】                                                 │ |
│  ──────────────────                                                │
│  // 0: fsck 안 함                                                    │
│  // 1: 루트 파일시스템 (먼저 검사)                                      │
│  // 2: 기타 파일시스템 (병렬 검사 가능)                                 │
│                                                                     │
│  /dev/sda1  /        ext4  defaults  0  1                          │
│  /dev/sda2  /home    ext4  defaults  0  2                          │
│  /dev/sda3  /data    xfs   defaults  0  0                          │
│                                                                     │
│  【로그 및 진단】                                                      │ |
│  ──────────────────                                                │
│  // fsck 로그                                                        │
│  $ sudo e2fsck -f -v /dev/sda1 2>&1 | tee fsck.log                 │
│                                                                     │
│  // 파일시스템 상태 확인                                                │
│  $ sudo tune2fs -l /dev/sda1 | grep state                          │
│                                                                     │
│  // 마운트 횟수 확인                                                   │
│  $ sudo tune2fs -l /dev/sda1 | grep -i "mount count"               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 파일시스템 무결성 검사 및 복구
• ext4 도구: e2fsck, fsck.ext4
• XFS 도구: xfs_repair
• FAT 도구: dosfsck, fsck.vfat
• NTFS 도구: ntfsfix
• 옵션: -p(자동), -n(읽기전용), -y(yes)
• -f: 강제 검사
• -c: 불량 블록 검사
• -b: 백업 슈퍼블록 사용
• 실행: 마운트 해제 후 실행 권장
• 루트: touch /forcefsck, init 1
• 자동: tune2fs -c, -i 설정
• fstab pass: 0(안함), 1(루트), 2(기타)
```
