# 474. RAID 6 (Block-Level Striping with Double Distributed Parity)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이중 분산 패리티로 높은 내결함성
> 2. **가치**: 2디스크 동시 고장 허용
> 3. **융합**: RAID 5, 내결함성과 연관

---

## Ⅰ. 개요

### 개념 정의

RAID 6은 **두 개의 독립적인 패리티를 사용하여 2개 디스크 동시 고장까지 허용하는 방식**이다.

### 💡 비유: 이중 보험
RAID 6은 **이중 보험**과 같다. 한 보험이 실패해도 다른 보험이 보장한다.

### RAID 6 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 6 이중 패리티                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조 (4디스크)】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  P = XOR 패리티, Q = 리드-솔로몬 패리티                               │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                                                         │    │ │
│  │  │   Disk 0    Disk 1    Disk 2    Disk 3                  │    │ │
│  │  │  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐                 │    │ │
│  │  │  │  A   │  │  B   │  │ P(AB)│  │ Q(AB)│  스트라이프 0     │    │ │
│  │  │  ├──────┤  ├──────┤  ├──────┤  ├──────┤                 │    │ │
│  │  │  │  C   │  │ P(CD)│  │ Q(CD)│  │  D   │  스트라이프 1     │    │ │
│  │  │  ├──────┤  ├──────┤  ├──────┤  ├──────┤                 │    │ │
│  │  │  │ P(EF)│  │ Q(EF)│  │  E   │  │  F   │  스트라이프 2     │    │ │
│  │  │  └──────┘  └──────┘  └──────┘  └──────┘                 │    │ │
│  │  │                                                         │    │ │
│  │  │  2개 패리티(P, Q)가 다른 디스크에 분산                      │    │ │
│  │  │                                                         │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【복구 시나리오】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1디스크 고장: P로 복구 (RAID 5와 동일)                               │ │
│  │                                                             │ │
│  │  2디스크 고장: P와 Q로 복구                                            │ │
│  │     • 연립 방정식 풀이                                                 │ │
│  │     • A + B + P = 0                                                  │ │
│  │     • g*A + h*B + Q = 0  (g, h: 갈루아 필드 계수)                    │ │
│  │     • 두 식으로 A, B 계산                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 6 상세                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 높은 내결함성:                                                    │ │
│  │     • 2디스크 동시 고장 허용                                           │ │
│  │     • 복구 중 1디스크 추가 고장도 안전                                  │ │
│  │     • 대용량 디스크에 적합 (복구 시간 김)                               │ │
│  │                                                             │ │
│  │  2. 용량 효율:                                                        │ │
│  │     • (N-2)/N 용량 사용 가능                                          │ │
│  │     • 4디스크: 50%, 6디스크: 67%, 8디스크: 75%                        │ │
│  │     • RAID 1보다 효율적                                               │ │
│  │                                                             │ │
│  │  3. 읽기 성능:                                                        │ │
│  │     • N-2개 디스크에서 분산 읽기                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 쓰기 페널티 (더 큼):                                              │ │
│  │     • 작은 쓰기: 읽기 2번 + 쓰기 4번 = 6 IOPS                          │ │
│  │     • P와 Q 모두 갱신 필요                                            │ │
│  │     • RAID 5보다 쓰기 느림                                            │ │
│  │                                                             │ │
│  │  2. 복구 시간:                                                        │ │
│  │     • 2디스크 복구는 더 복잡하고 느림                                   │ │
│  │     • 대용량에서 며칠 걸릴 수 있음                                     │ │
│  │                                                             │ │
│  │  3. 최소 디스크:                                                      │ │
│  │     • 최소 4디스크 필요                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAID 5 vs RAID 6 비교】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  특성           RAID 5          RAID 6                             │ │
│  │  ────           ──────          ──────                              │ │
│  │  고장 허용       1디스크          2디스크                              │ │
│  │  용량 효율       (N-1)/N         (N-2)/N                              │ │
│  │  최소 디스크     3               4                                    │ │
│  │  쓰기 페널티     4 IOPS          6 IOPS                               │ │
│  │  복구 속도       빠름            느림                                  │ │
│  │  복구 중 안전성  낮음            높음                                  │ │
│  │                                                             │ │
│  │  대용량 디스크(4TB+)에서는 RAID 6 권장                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux mdadm RAID 6 생성】                                        │
│  ──────────────────                                                  │
│  // 최소 4디스크 필요                                                  │
│  $ sudo mdadm --create /dev/md0 --level=6 --raid-devices=4 \        │
│        /dev/sd{b,c,d,e}1                                            │
│                                                                     │
│  // 상태 확인                                                        │
│  $ cat /proc/mdstat                                                 │
│  md0 : active raid6 sde1[3] sdd1[2] sdc1[1] sdb1[0]                 │
│        2095104 blocks super 1.2 level 6, 64k chunk                  │
│        algorithm 2 [4/4] [UUUU]                                     │
│  // [UUUU] = 4디스크 모두 정상                                         │
│                                                                     │
│  // 상세 정보                                                          │
│  $ sudo mdadm --detail /dev/md0                                     │
│    Raid Level : raid6                                               │
│    Array Size : 2095104 (2046.00 MiB 2145.39 MB)                   │
│    Raid Devices : 4                                                 │
│    Total Devices : 4                                                │
│    Working Devices : 4                                              │
│    Failed Devices : 0                                               │
│                                                                     │
│  【2디스크 고장 시뮬레이션】                                          │
│  ──────────────────                                                  │
│  // 2개 디스크 고장 표시                                               │
│  $ sudo mdadm --manage /dev/md0 --fail /dev/sdb1 /dev/sdc1         │
│  $ cat /proc/mdstat                                                 │
│  [UU__]  // 2디스크 고장이어도 여전히 동작                              │
│                                                                     │
│  // 제거 및 교체                                                       │
│  $ sudo mdadm --manage /dev/md0 --remove /dev/sdb1 /dev/sdc1        │
│  $ sudo mdadm --manage /dev/md0 --add /dev/sdf1 /dev/sdg1           │
│  // 복구 시작                                                         │
│                                                                     │
│  【적합한 사용 사례】                                                  │
│  ──────────────────                                                  │
│  • 대용량 스토리지 (4TB+ 디스크)                                        │
│  • 아카이브 시스템                                                     │
│  • 백업 저장소                                                        │
│  • 높은 가용성 필요 시스템                                              │
│  • 장기 데이터 보관                                                   │
│                                                                     │
│  【성능 최적화】                                                      │
│  ──────────────────                                                  │
│  // 배터리 백업 캐시 사용                                              │
│  // 큰 스트라이프 크기 (256KB+)                                        │
│  // SSD로 캐시 계층 추가                                               │
│  // 쓰기 집약적이면 RAID 10 고려                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 이중 분산 패리티 (P + Q)
• 장점: 2디스크 동시 고장 허용
• 단점: 쓰기 페널티 큼 (6 IOPS)
• 내결함성: 2디스크 고장 허용
• 용량: (N-2)/N, 최소 4디스크
• 패리티: XOR(P) + 리드-솔로몬(Q)
• 복구: 연립 방정식 풀이
• 용도: 대용량, 높은 가용성
```
