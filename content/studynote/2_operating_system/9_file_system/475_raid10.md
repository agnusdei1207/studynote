# 475. RAID 10 (RAID 1+0)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: RAID 1 미러링 + RAID 0 스트라이핑 결합
> 2. **가치**: 최고의 성능과 높은 내결함성
> 3. **융합**: RAID 0, RAID 1과 연관

---

## Ⅰ. 개요

### 개념 정의

RAID 10(RAID 1+0)은 **미러링된 디스크 쌍을 스트라이핑하는 방식으로, RAID 1과 RAID 0의 장점을 결합**한다.

### 💡 비유: 이중 문서와 분할 배송
RAID 10은 **이중 문서를 여러 팀에 분할 배송**하는 것과 같다. 각 부분이 2곳에 복제되어 있고, 여러 부분이 동시에 처리된다.

### RAID 10 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 10 (1+0)                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조 (4디스크)】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  데이터: [A][B][C][D]                                               │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                                                         │    │ │
│  │  │          RAID 0 스트라이핑                                │    │ │
│  │  │  ┌───────────────────┐    ┌───────────────────┐          │    │ │
│  │  │  │    미러 그룹 0     │    │    미러 그룹 1     │          │    │ │
│  │  │  │  ┌─────┐ ┌─────┐  │    │  ┌─────┐ ┌─────┐  │          │    │ │
│  │  │  │  │Disk0│ │Disk1│  │    │  │Disk2│ │Disk3│  │          │    │ │
│  │  │  │  │  A  │ │  A  │  │    │  │  B  │ │  B  │  │          │    │ │
│  │  │  │  │ mir │ │ mir │  │    │  │ mir │ │ mir │  │          │    │ │
│  │  │  │  ├─────┤ ├─────┤  │    │  ├─────┤ ├─────┤  │          │    │ │
│  │  │  │  │  C  │ │  C  │  │    │  │  D  │ │  D  │  │          │    │ │
│  │  │  │  │ mir │ │ mir │  │    │  │ mir │ │ mir │  │          │    │ │
│  │  │  │  └─────┘ └─────┘  │    │  └─────┘ └─────┘  │          │    │ │
│  │  │  │    RAID 1 미러     │    │   RAID 1 미러     │          │    │ │
│  │  │  └───────────────────┘    └───────────────────┘          │    │ │
│  │  │                                                         │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【고장 허용 예시】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  안전한 조합 (동작 유지):                                             │ │
│  │  • Disk 0 + Disk 2 고장 → OK (다른 미러에서 복구 가능)                │ │
│  │  • Disk 0 + Disk 3 고장 → OK                                        │ │
│  │  • Disk 1 + Disk 2 고장 → OK                                        │ │
│  │                                                             │ │
│  │  위험한 조합 (데이터 손실):                                           │ │
│  │  • Disk 0 + Disk 1 고장 → X (같은 미러 그룹)                          │ │
│  │  • Disk 2 + Disk 3 고장 → X (같은 미러 그룹)                          │ │
│  │                                                             │ │
│  │  결론: 각 미러 그룹에서 최대 1개씩 고장 허용                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 10 상세                                           │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 우수한 성능:                                                      │ │
│  │     • 읽기: N개 디스크에서 분산 읽기                                    │ │
│  │     • 쓰기: N/2개 미러 그룹에 분산 (쓰기 페널티 없음)                   │ │
│  │     • RAID 5/6보다 쓰기 성능 좋음                                      │ │
│  │                                                             │ │
│  │  2. 빠른 복구:                                                        │ │
│  │     • 미러에서 직접 복사                                               │ │
│  │     • 패리티 계산 불필요                                               │ │
│  │     • RAID 5/6보다 복구 빠름                                          │ │
│  │                                                             │ │
│  │  3. 높은 가용성:                                                      │ │
│  │     • 각 미러 그룹에서 1개씩 고장 허용                                  │ │
│  │     • 4디스크: 최대 2개 고장 (다른 그룹)                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 용량 비효율:                                                      │ │
│  │     • 50% 용량만 사용 가능                                            │ │
│  │     • 4×1TB = 2TB 사용 가능                                          │ │
│  │     • RAID 5/6보다 비효율적                                           │ │
│  │                                                             │ │
│  │  2. 비용:                                                             │ │
│  │     • 많은 디스크 필요                                                │ │
│  │     • 스토리지 비용 높음                                               │ │
│  │                                                             │ │
│  │  3. 최소 디스크:                                                      │ │
│  │     • 최소 4디스크 필요 (2 미러 × 2)                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAID 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  특성       RAID 0    RAID 1    RAID 5    RAID 6    RAID 10       │ │
│  │  ────       ──────    ──────    ──────    ──────    ───────        │ │
│  │  최소 디스크   2        2        3         4         4              │ │
│  │  용량 효율    100%     50%      67-94%    50-87%    50%            │ │
│  │  고장 허용    0        1        1         2         1/그룹         │ │
│  │  읽기 성능    최고      높음      높음       높음      최고          │ │
│  │  쓰기 성능    최고      보통      보통       낮음      높음          │ │
│  │  복구 속도    -        빠름      느림       느림      빠름          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux mdadm RAID 10 생성】                                       │
│  ──────────────────                                                  │
│  // 기본 RAID 10 생성 (자동 레이아웃)                                  │
│  $ sudo mdadm --create /dev/md0 --level=10 --raid-devices=4 \       │
│        /dev/sd{b,c,d,e}1                                            │
│                                                                     │
│  // 레이아웃 지정 (far, near, offset)                                 │
│  $ sudo mdadm --create /dev/md0 --level=10 --raid-devices=4 \       │
│        --layout=f2 /dev/sd{b,c,d,e}1                                │
│  // f2 = far 레이아웃, 2개 복사본                                     │
│                                                                     │
│  // 상태 확인                                                        │
│  $ cat /proc/mdstat                                                 │
│  md0 : active raid10 sde1[3] sdd1[2] sdc1[1] sdb1[0]                │
│        2095104 blocks super 1.2 512K chunks 2 near-copies           │
│        [4/4] [UUUU]                                                 │
│                                                                     │
│  // 상세 정보                                                          │
│  $ sudo mdadm --detail /dev/md0                                     │
│    Raid Level : raid10                                              │
│    Layout : near=2                                                  │
│    Array Size : 2095104                                             │
│                                                                     │
│  【레이아웃 옵션】                                                      │
│  ──────────────────                                                  │
│  // near (기본): 같은 블록이 인접 디스크에 미러링                        │
│  // far: 같은 블록이 떨어진 디스크에 미러링 (읽기 성능 향상)              │
│  // offset: 스트라이프 단위로 오프셋                                    │
│                                                                     │
│  // 6디스크 RAID 10 (3 미러 그룹)                                     │
│  $ sudo mdadm --create /dev/md0 --level=10 --raid-devices=6 \       │
│        /dev/sd{b,c,d,e,f,g}1                                        │
│  // 3개 미러 그룹, 각 그룹 2디스크                                      │
│                                                                     │
│  【적합한 사용 사례】                                                  │
│  ──────────────────                                                  │
│  • 데이터베이스 서버 (높은 IOPS 필요)                                  │
│  • OLTP 시스템                                                        │
│  • 이메일 서버                                                        │
│  • 가상화 호스트                                                      │
│  • 쓰기 집약적 애플리케이션                                            │
│  • 고성능 컴퓨팅                                                       │
│                                                                     │
│  【성능 특성】                                                          │
│  ──────────────────                                                  │
│  // 읽기: RAID 0 수준 (모든 디스크 활용)                               │
│  // 쓰기: RAID 0 수준 (미러링 오버헤드만)                              │
│  // 복구: 미러에서 직접 복사, 매우 빠름                                 │
│  // 쓰기 페널티: 없음 (패리티 계산 없음)                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: RAID 1 미러링 + RAID 0 스트라이핑
• 장점: 최고 성능, 빠른 복구, 높은 가용성
• 단점: 50% 용량만 사용, 비용 높음
• 고장 허용: 각 미러 그룹에서 1개씩
• 용량: N/2, 최소 4디스크
• 성능: 읽기/쓰기 모두 우수
• 복구: 미러에서 직접 복사
• 용도: DB, OLTP, 가상화, 고성능
```
