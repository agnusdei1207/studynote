# 473. RAID 5 (Block-Level Striping with Distributed Parity)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스트라이핑 + 분산 패리티
> 2. **가치**: 성능, 용량 효율, 내결함성 균형
> 3. **융합**: RAID 0, RAID 1과 연관

---

## Ⅰ. 개요

### 개념 정의

RAID 5는 **블록 단위 스트라이핑과 분산 패리티를 결합하여 성능과 내결함성을 모두 제공하는 방식**이다.

### 💡 비유: 퍼즐과 여분 조각
RAID 5는 **퍼즐과 여분 조각**과 같다. 퍼즐 조각(데이터)을 여러 상자에 나누고, 각 상자마다 여분 조각(패리티)을 하나씩 둔다.

### RAID 5 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 5 분산 패리티                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조 (3디스크)】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  데이터: [A][B][C][D][E][F]                                         │ │
│  │  패리티: P = A⊕B (XOR)                                             │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                                                         │    │ │
│  │  │    Disk 0      Disk 1      Disk 2                       │    │ │
│  │  │   ┌──────┐    ┌──────┐    ┌──────┐                      │    │ │
│  │  │   │  A   │    │  B   │    │ P(AB)│  ← 스트라이프 0        │    │ │
│  │  │   ├──────┤    ├──────┤    ├──────┤                      │    │ │
│  │  │   │  C   │    │P(CD) │    │  D   │  ← 스트라이프 1        │    │ │
│  │  │   ├──────┤    ├──────┤    ├──────┤                      │    │ │
│  │  │   │P(EF) │    │  E   │    │  F   │  ← 스트라이프 2        │    │ │
│  │  │   └──────┘    └──────┘    └──────┘                      │    │ │
│  │  │                                                         │    │ │
│  │  │  패리티가 각 디스크에 분산됨                               │    │ │
│  │  │                                                         │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【패리티로 복구】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Disk 0 고장 시:                                                     │ │
│  │                                                             │ │
│  │  A 복구: B ⊕ P(AB) = A                                             │ │
│  │  C 복구: D ⊕ P(CD) = C                                             │ │
│  │                                                             │ │
│  │  P(EF)는 복구 필요 없음 (데이터 아님)                                 │ │
│  │                                                             │ │
│  │  XOR 특성: X ⊕ Y ⊕ Y = X                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                RAID 5 상세                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 균형 잡힌 성능:                                                   │ │
│  │     • 읽기: N-1개 디스크에서 분산 읽기                                  │ │
│  │     • 쓰기: 패리티 계산 오버헤드 있지만 양호                            │ │
│  │                                                             │ │
│  │  2. 용량 효율:                                                        │ │
│  │     • (N-1)/N 용량 사용 가능                                          │ │
│  │     • 3디스크: 67%, 4디스크: 75%, 5디스크: 80%                        │ │
│  │     • RAID 1보다 효율적                                               │ │
│  │                                                             │ │
│  │  3. 내결함성:                                                        │ │
│  │     • 1디스크 고장 허용                                               │ │
│  │     • 운영 중 디스크 교체 가능                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 쓰기 페널티 (Write Penalty):                                      │ │
│  │     • 작은 쓰기: 읽기 2번 + 쓰기 2번 = 4 IOPS                          │ │
│  │     • 기존 데이터 읽기 + 기존 패리티 읽기                               │ │
│  │     • 새 데이터 쓰기 + 새 패리티 쓰기                                   │ │
│  │                                                             │ │
│  │  2. 복구 시간:                                                        │ │
│  │     • 패리티로 모든 데이터 재계산 필요                                  │ │
│  │     • 대용량에서 복구 오래 걸림                                        │ │
│  │     • 복구 중 성능 저하                                                │ │
│  │                                                             │ │
│  │  3. 2디스크 동시 고장:                                                │ │
│  │     • 복구 불가능                                                     │ │
│  │     • 복구 중 다른 디스크 고장 위험                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【용량 계산】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  사용 가능 용량 = (N-1) × 디스크 크기                                  │ │
│  │                                                             │ │
│  │  3×1TB = 2TB 사용 가능                                              │ │
│  │  4×1TB = 3TB 사용 가능                                              │ │
│  │  5×1TB = 4TB 사용 가능                                              │ │
│  │                                                             │ │
│  │  최소 3디스크 필요                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux mdadm RAID 5 생성】                                        │
│  ──────────────────                                                  │
│  // 최소 3디스크 필요                                                  │
│  $ sudo mdadm --create /dev/md0 --level=5 --raid-devices=3 \        │
│        /dev/sdb1 /dev/sdc1 /dev/sdd1                                │
│                                                                     │
│  // 스트라이프 크기 지정                                                │
│  $ sudo mdadm --create /dev/md0 --level=5 --raid-devices=4 \        │
│        --chunk=64 /dev/sd{b,c,d,e}1                                 │
│                                                                     │
│  // 상태 확인                                                        │
│  $ cat /proc/mdstat                                                 │
│  md0 : active raid5 sdd1[3] sdc1[2] sdb1[1]                         │
│        2095104 blocks super 1.2 level 5, 64k chunk                  │
│        algorithm 2 [3/3] [UUU]                                      │
│  // [UUU] = 3디스크 모두 정상                                          │
│                                                                     │
│  // 초기 동기화 대기                                                   │
│  $ watch cat /proc/mdstat                                           │
│  // 100%까지 대기                                                     │
│                                                                     │
│  【디스크 추가 (확장)】                                                │
│  ──────────────────                                                  │
│  // 새 디스크 추가                                                      │
│  $ sudo mdadm --add /dev/md0 /dev/sde1                              │
│                                                                     │
│  // 어레이 확장                                                        │
│  $ sudo mdadm --grow /dev/md0 --raid-devices=4                      │
│                                                                     │
│  // 파일 시스템 확장                                                    │
│  $ sudo resize2fs /dev/md0                                          │
│                                                                     │
│  【성능 고려사항】                                                      │
│  ──────────────────                                                  │
│  // 읽기 성능: 양호                                                     │
│  // 쓰기 성능: 작은 쓰기에서 페널티                                       │
│                                                                      │
│  // 쓰기 성능 개선 팁                                                   │
│  • 큰 스트라이프 크기 사용                                               │
│  • SSD 사용 (랜덤 쓰기 빠름)                                            │
│  • 쓰기 집약적이면 RAID 10 고려                                         │
│  • 배터리 백업 캐시 (BBWC) 사용                                         │
│                                                                     │
│  【적합한 사용 사례】                                                  │
│  ──────────────────                                                  │
│  • 파일 서버                                                          │
│  • 웹 서버                                                            │
│  • 데이터 웨어하우스                                                    │
│  • 읽기 집약적 애플리케이션                                              │
│  • 일반 목적 스토리지                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 스트라이핑 + 분산 패리티
• 장점: 성능/용량/내결함성 균형
• 단점: 쓰기 페널티, 복구 시간 김
• 내결함성: 1디스크 고장 허용
• 용량: (N-1)/N, 최소 3디스크
• 패리티: XOR 연산으로 계산/복구
• 쓰기 페널티: 4 IOPS (작은 쓰기)
• 용도: 파일서버, 웹서버, 일반 스토리지
```
