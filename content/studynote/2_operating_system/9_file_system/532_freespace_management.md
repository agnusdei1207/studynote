# 532. 직접 I/O (Direct I/O)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 I/O 명령을 직접 수행
> 2. **가치**: 높은 성능과 낮은 지연 시스템 콜 감소
> 3. **융합**: 메모리 매핑, DMA, 버퍼링과 연관

---

## Ⅰ. 개요

### 개념 정의
**직접 I/O(Direct I/O)**는 **프로세스가 I/O 명령을 직접 실행하여 데이터를 전송하는 방식**입니다.

### 💡 비유: 직통 전화
직접 I/O는 **직통 전화**와 같습니다. 상대방과 바로 응답하는 것과 같습니다.

### 직접 I/O 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                직접 I/O 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【I/O 방식 비교】                                                │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                           │ │   │
│  │  1. 프로그램 I/O (Programmed I/O)                               │ │   │
│  │     ─── I/O 명령 → CPU 대기 → 장치 응답 → 데이터 전송           │ │   │
│  │     장점: 단순, 직관적                                      │ │   │
│  │     단점: CPU가 대기 (낮은 효율)                              │ │   │
│  │                                                             │ │   │
│  │  2. 인터럽트 구동 I/O (Interrupt-Driven I/O)                │ │   │
│  │     ─── I/O 명령 → 다른 작업 → 인터럽트 발생 → 데이터 전송           │ │   │
│  │     장점: CPU 대기 없음                                    │ │   │
│  │     단점: 인터럽트 오버헤드                              │ │   │
│  │                                                             │ │   │
│  │  3. 직접 메모리 액세 I/O (Direct Memory Access)               │ │   │
│  │     ─── I/O 명령 → DMA 컨트롤러 → 메모리 직접 전송               │ │   │
│  │     장점: CPU 관여 없음, 최고 성능                            │ │   │
│  │     단점: 하드웨어 복잡도 증가                              │ │   │
│  │                                                             │ │   │
│  │  4. 채널 I/O (Channel I/O)                                 │ │   │
│  │     ─── I/O 명령 → 채널 프로그램 → 복잡한 연산 수행             │ │   │
│  │     장점: 매우 높은 성능, 복잡한 연산 지원               │ │   │
│  │     단점: 매우 복잡한 프로그래밍                             │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【메모리 맵 방식】                                                  │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  방식              CPU 역할              장치 역할                  │ │   │
│  │  ────              ──────              ──────                  │ │   │
│  │  PIO              모든 작업 수행          단순 응답                   │ │   │
│  │  인터럽트          요청 후 대기           완료 시 인터럽트              │ │   │
│  │  DMA              설정만 수행            전체 작업 수행               │ │   │
│  │  채널             설정만 수행            복잡한 연산까지              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                직접 I/O 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프로그램 I/O 상세】                                                │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  // CPU가 직접 제어                                        │ │   │
│  │  void programmed_io(char* data, int size) {                   │ │   │
│  │      for (int i = 0; i < size; i++) {                        │ │   │
│  │          // 상태 레지스터 확인                              │ │   │
│  │          while ((*STATUS_REG & BUSY)) {                    │ │   │
│  │              // CPU가 대기                                   │ │   │
│  │          }                                                     │ │   │
│  │                                                             │ │   │
│  │          // 데이터 레지스터에 쓰기                           │ │   │
│  │          *DATA_REG = data[i];                               │ │   │
│  │      }                                                     │ │   │
│  │  }                                                         │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【인터럽트 구동 I/O 상세】                                            │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  // CPU가 요청만 하고 다른 작업 수행                          │ │   │
│  │  void interrupt_io(char* data, int size) {                  │ │   │
│  │      // 장치에 읽기 요청                                      │ │   │
│  │      *COMMAND_REG = READ_CMD;                               │ │   │
│  │      *SIZE_REG = size;                                       │ │   │
│  │                                                             │ │   │
│  │      // 다른 작업 수행...                                       │ │   │
│  │      do_other_work();                                        │ │   │
│  │  }                                                         │ │   │
│  │                                                             │ │   │
│  │  // 인터럽트 핸들러                                           │ │   │
│  │  void io_interrupt_handler() {                              │ │   │
│  │      // 완료된 데이터 처리                                      │ │   │
│  │      for (int i = 0; i < *SIZE_REG; i++) {                 │ │   │
│  │          data[i] = *DATA_REG;                              │ │   │
│  │      }                                                     │ │   │
│  │  }                                                         │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【DMA 상세】                                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  // DMA 설정                                                 │ │   │
│  │  void dma_io(char* src, char* dst, int size) {              │ │   │
│  │      // DMA 컨트롤러 설정                                      │ │   │
│  │      *DMA_SRC = src;          // 소스 주소                    │ │   │
│  │      *DMA_DST = dst;          // 목적지 주소                    │ │   │
│  │      *DMA_SIZE = size;        // 전송 크기                     │ │   │
│  │      *DMA_CMD = START;        // 시작 명령                    │ │   │
│  │                                                             │ │   │
│  │      // CPU는 다른 작업 수행...                                 │ │   │
│  │      do_other_work();                                        │ │   │
│  │                                                             │ │   │
│  │      // DMA 완료 대기                                          │ │   │
│  │      while (!(*DMA_STATUS & DONE)) {                          │ │   │
│  │          // 또는 인터럽트 대기                                   │ │   │
│  │      }                                                     │ │   │
│  │  }                                                         │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【장단점 비교】                                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  방식              CPU 사용            지연 시간            복잡도              │ │   │
│  │  ────              ──────              ──────              ──────              │ │   │
│  │  PIO              높음 (대기)          높음               낮음               │ │   │
│  │  인터럽트          중간                낮음               중간              │ │   │
│  │  DMA              낮음 (효율적)         낮음               높음               │ │   │
│  │  채널             매우 낮음              매우 낮음             매우 높음           │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86 IN/OUT 명령어】                                              │
│  ──────────────────                                                │
│  // 문자 출력 (프로그램 I/O)                                      │
│  out dx, al            ; dx = 포트 주소                          │
│       mov al, char        ; al = 출력할 문자                          │
│       int 0x80           ; 화면에 출력                               │
│                                                                     │
│  // 문자 입력 (프로그램 I/O)                                      │
│  mov dx, 0x60           ; dx = 포트 주소                          │
│       in al, 0x00          ; 대기                                        │
│       mov al, [buffer]   ; al = 입력된 문자                          │
│                                                                     │
│  【ARM GPIO (메모리 맵핍)】                                           │
│  ──────────────────                                                │
│  // GPIO 레지스터 구조 (BCM2835)                                │
│  #define GPIO_BASE 0x3F200000                                  │
│  volatile unsigned int *GPIO = (unsigned int *)GPIO_BASE;          │
│                                                                     │
│  void gpio_write(int pin, int value) {                              │
│      GPIO[0] = (1 << pin);         // 출력 설정                             │
│      if (value)                                                │
│          GPIO[7] |= (1 << pin);    // HIGH                                │
│      else                                                        │
│          GPIO[7] &= ~(1 << pin);   // LOW                                │
│  }                                                             │
│                                                                     │
│  int gpio_read(int pin) {                                      │
│      return (GPIO[13] >> pin) & 1;  // 레벨 읽기                      │
│  }                                                             │
│                                                                     │
│  【DMA 전송 예시】                                              │
│  ──────────────────                                                │
│  void dma_memcpy(void* dst, void* src, size_t size) {              │
│      // DMA 채널 설정                                          │
│      DMA_CH0_SRC = (uint32_t)src;                               │
│      DMA_CH0_DST = (uint32_t)dst;                               │
│      DMA_CH0_SIZE = size;                                     │
│      DMA_CH0_CTRL = DMA_START;                               │
│                                                                     │
│      // 완료 대기                                                  │
│      while (!(DMA_CH0_INT & DMA_COMPLETE)) {                  │
│          // 대기 또는 다른 작업                                   │
│      }                                                             │
│  }                                                             │
│                                                                     │
│  【성능 비교 (이론적)】                                             │
│  ──────────────────                                                │
│  // 1MB 데이터 전송 시 CPU 사이클 수                            │
│  PIO:     ~1,000,000 사이클 (1바이트씽 대기)                 │
│  인터럽트:   ~10,000 사이클 (버스트 전송)                     │
│  DMA:     ~1,000 사이클 (설정 + 완료 확인)                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: CPU가 I/O 명령을 직접 수행
• 방식: PIO, 인터럽트, DMA, 채널
• PIO: 단순하지만 CPU 대기
• 인터럽트: CPU 대기 없음, 오버헤드 있음
• DMA: CPU 관여 최소화, 높은 성능
• 채널: 가장 복잡하지만 최고 성능
• 메모리 맵: PIO/인터럽트는 포트 맵핑 DMA: 별도 주소
• 선택 기준: 데이터 크기, 성능 요구사항
```
