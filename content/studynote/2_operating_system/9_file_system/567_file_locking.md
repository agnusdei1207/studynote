# 567. 플러그 앤 플레이 (Plug and Play)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장치 자동 감지 및 구성 기술
> 2. **가치**: 수동 설정 없이 장치 사용
> 3. **융합**: ACPI, udev, 드라이버와 연관

---

## Ⅰ. 개요

### 개념 정의
**플러그 앤 플레이(PnP)**는 **장치 연결 시 자동으로 인식하고 구성하는 기술**입니다.

### 💡 비유: 스마트 콘센트
플러그 앤 플레이는 **스마트 콘센트**와 같습니다. 꽂으면 바로 인식합니다.

### 플러그 앤 플레이 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                플러그 앤 플레이 구조                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【PnP 동작 과정】                                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  [장치 연결]                                                    │ │   │
│  │       │                                                     │ │   │
│  │       ▼                                                     │ │   │
│  │  ┌─────────┐                                                │ │   │
│  │  │장치 감지 │  → 버스 스캔, ID 확인                            │ │   │
│  │  └─────────┘                                                │ │   │
│  │       │                                                     │ │   │
│  │       ▼                                                     │ │   │
│  │  ┌─────────┐                                                │ │   │
│  │  │리소스 요청│  → IRQ, I/O, DMA, 메모리                       │ │   │
│  │  └─────────┘                                                │ │   │
│  │       │                                                     │ │   │
│  │       ▼                                                     │ │   │
│  │  ┌─────────┐                                                │ │   │
│  │  │리소스 할당│  → 충돌 없이 자동 배정                          │ │   │
│  │  └─────────┘                                                │ │   │
│  │       │                                                     │ │   │
│  │       ▼                                                     │ │   │
│  │  ┌─────────┐                                                │ │   │
│  │  │드라이버 로드│  → 커널 모듈 자동 로드                        │ │   │
│  │  └─────────┘                                                │ │   │
│  │       │                                                     │ │   │
│  │       ▼                                                     │ │   │
│  │  [장치 사용 가능]                                               │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【PnP 리소스】                                                       │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  리소스            설명                    예시                   │ │   │
│  │  ──────            ────                    ────                   │ │   │
│  │  IRQ              인터럽트 요청             IRQ 5, 7, 10          │ │   │
│  │  I/O Port         입출력 포트              0x3F8-0x3FF           │ │   │
│  │  DMA              직접 메모리 액세스         DMA 1, 3, 5          │ │   │
│  │  Memory           메모리 주소               0xD0000-0xDFFFF       │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【PnP 지원 버스】                                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  버스              PnP 지원              특징                    │ │   │
│  │  ────              ─────────              ────                    │ │   │
│  │  PCI/PCIe         완전 지원              자동 구성               │ │   │
│  │  USB              완전 지원              클래스 드라이버          │ │   │
│  │  ISA PnP          부분 지원              레거시                  │ │   │
│  │  PCMCIA           완전 지원              카드 버스               │ │   │
│  │  Thunderbolt      완전 지원              PCIe 터널링             │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │

│  // PnP 장치 목록                                                     │

│                                                                     │

│                                                                     │
│  【udev (장치 관리자)】                                               │
│  ──────────────────                                                │
│  // udev 이벤트 모니터링                                              │
│  $ sudo udevadm monitor                                             │
│  KERNEL[123456.789] add /devices/pci0000:00/... (pci)              │
│  UDEV  [123456.790] add /devices/pci0000:00/... (pci)              │
│                                                                     │
│  // 장치 정보                                                         │
│  $ udevadm info -a -n /dev/sda                                     │
│  ATTRS{vendor}=="0x8086"                                           │
│  ATTRS{device}=="0x1e03"                                           │
│                                                                     │
│  // 장치 규칙                                                         │
│  /etc/udev/rules.d/99-mydevice.rules:                              │
│  SUBSYSTEM=="usb", ATTR{idVendor}=="046d", \                       │
│      ATTR{idProduct}=="c52b", MODE="0666"                          │
│                                                                     │
│  // 규칙 적용                                                        │

│                                                                     │

│                                                                     │
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 장치 자동 감지 및 구성
• 과정: 감지 → 리소스 요청 → 할당 → 드라이버 로드
• 리소스: IRQ, I/O Port, DMA, Memory
• 식별자: Vendor ID, Device ID, _HID
• 버스: PCI/PCIe, USB, ISA PnP
• USB 클래스: HID, Mass Storage, CDC
• Linux: udev, udevadm, /sys/bus/
• 모듈: modprobe, modules.alias
• 문제 해결: 리바인딩, 리셋, 재스캔
• 장점: 수동 설정 불필요
```
