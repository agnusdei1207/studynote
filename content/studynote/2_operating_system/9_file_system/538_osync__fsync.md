# 538. I/O 프로파일링 (I/O Profiling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 작업 패턴 분석
> 2. **가치**: 성능 병목 지점 및 최적화 기회 식별
> 3. **융합**: 벤치마크, 트레이싱, 디버깅과 연관

---

## Ⅰ. 개요

### 개념 정의
**I/O 프로파일링(I/O Profiling)**는 **시스템의 I/O 작업 패턴을 분석하여 성능 병목 지점을 찾는 기술**입니다.

### 💡 비유: 의사 진단
I/O 프로파일링은 **의사가 환자의 증상을 분석하는 것**과 같습니다. 어떤 부위에 문제가 있는지 확인합니다.

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                I/O 프로파일링 상세                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【strace 예시】                                                      │
│  ──────────────────                                                │
│  // read/write 호출 추적                                             │
│  $ strace -e trace=read,write -p <pid>                             │
│  $ strace -c -f ./my_program       // 호출 통계                        │
│                                                                     │
│  // 출력 예시                                                        │
│  read(3, 0x7fff1234, 4096)          = 4096                             │
│  write(4, 0x7fff5678, 1024)         = 1024                           │
│                                                                     │
│  【perf 예시】                                                        │
│  ──────────────────                                                │
│  // I/O 이벤트 기록                                                  │
│  $ perf record -e syscalls:sys_enter_read -p <pid>                │
│  $ perf record -e block:block_rq_issue -a                          │
│                                                                     │
│  // 보고서 생성                                                      │
│  $ perf report                                                      │
│  $ perf annotate                                                    │
│                                                                     │
│  【eBPF/bcc 예시】                                                    │
│  ──────────────────                                                │
│  // 파일 I/O 추적                                                    │
│  $ sudo bpftrace -e 'kprobe:vfs_read { printf("read: %s\n", comm); }' │
│                                                                     │
│  // I/O 대기 시간 측정                                                │
│  $ sudo bpftrace -e '                                               │
│    kprobe:blk_start_request { @req[tid] = nsecs; }                 │
│    kretprobe:blk_start_request {                                    │
│      @latency[tid] = nsecs - @req[tid];                              │
│      printf("I/O latency: %d us\n", @latency[tid] / 1000);          │
│    }                                                                │
│  '                                                                  │
│                                                                     │
│  【blktrace 예시】                                                   │
│  ──────────────────                                                │
│  // 블록 I/O 추적 시작                                                │
│  $ sudo blktrace -d /dev/sda -o - | blkparse -i -                  │
│                                                                     │
│  // 출력 예시                                                        │
│  8,0   0     0.000001  Q  A  0  + 4096 [read]                        │
│  8,0   0     0.000002  G  A  0  + 4096 [complete]                   │
│                                                                     │
│  【ftrace 예시】                                                      │
│  ──────────────────                                                │
│  // 함수 추적 활성화                                                  │
│  $ echo do_sys_open > /sys/kernel/tracing/set_ftrace_filter         │
│  $ echo 1 > /sys/kernel/tracing/tracing_on                        │
│  $ cat /sys/kernel/tracing/trace                                   │
│                                                                     │
│  // 이벤트 추적                                                      │
│  $ echo 1 > /sys/kernel/tracing/events/sched/sched_switch/enable   │
│                                                                     │
│  【SystemTap 예시】                                                  │
│  ──────────────────                                                │
│  // 파일 읽기 추적                                                    │
│  probe vfs.read {                                                    │
│      printf("read %s: %d bytes\n", execname(), $count);              │
│  }                                                                   │
│                                                                     │
│  // I/O 대기 시간                                                     │
│  probe scheduler.cpu_off {                                          │
│      if (task_state == 2)  // TASK_UNINTERRUPTIBLE                   │
│          printf("CPU off for I/O: %s\n", execname());                 │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

│  5. 결과 검증                                                        │

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: I/O 작업 패턴 분석 및 성능 병목 식별
• 분석 항목: 시스템 콜, 대기 시간, 처리량, 캐시 효율
• 도구: strace, perf, eBPF, blktrace, ftrace
• 수준: 사용자 공간 → 라이브러리 → 시스템 콜 → 커널 → 드라이버
• 병목 패턴: 동기 I/O, 작은 버퍼, 순차 읽기, 캐시 미스
• 워크플로우: 베이스라인 → 프로파일링 → 분석 → 최적화 → 검증
• 목적: 성능 최적화, 문제 해결, 용량 계획
```
