## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연결 할당(Linked Allocation)은 파일을 물리적으로 흩어진 블록들의 집합으로 보고, 각 블록이 다음 블록을 가리키는 포인터(Pointer)를 포함하여 리스트 형태로 연결하는 할당 방식이다.
> 2. **가치**: 외부 단편화(External Fragmentation) 문제를 완벽히 해결하며, 파일 생성 시 크기를 미리 알 필요가 없어 동적인 파일 확장에 매우 유리하다.
> 3. **융합**: MS-DOS의 FAT(File Allocation Table) 파일 시스템의 근간이 되었으며, 포인터 관리의 비효율성을 개선하기 위해 인덱싱 기법과 결합하여 발전하였다.

---

## Ⅰ. 연결 할당의 정의와 메커니즘 (Definition)

- **정의**: 각 데이터 블록은 실제 데이터와 다음 블록의 주소를 담은 포인터로 구성된다.
- **디렉터리 정보**: 파일 이름과 함께 **[시작 블록 주소, 끝 블록 주소]**만 저장한다.
- **할당 과정**: 디스크의 어떤 블록이든 비어 있기만 하면 할당할 수 있다. 블록들은 연결 리스트(Linked List) 구조를 형성한다.

📢 **섹션 요약 비유**: 연결 할당은 "보물찾기 게임"과 같다. 첫 번째 장소(블록)에 가면 보물(데이터)과 함께 다음 장소가 어디인지 적힌 쪽지(포인터)가 들어있는 것과 동일하다.

---

## Ⅱ. 연결 할당의 물리적 구조 (Architecture)

블록의 일부분이 포인터 공간으로 사용되므로, 실제 데이터를 담는 공간은 조금 줄어든다.

### 1. 블록 구성
- 만약 블록 크기가 512B이고 포인터가 4B라면, 실제 데이터는 508B만 담을 수 있다.

### 2. 연결 할당 개념도 (ASCII Diagram)

```text
 [ Directory Entry ]
 ┌───────────┬───────┬──────┐
 │ File Name │ Start │ End  │
 ├───────────┼───────┼──────┤
 │ jeep (J)  │   9   │  25  │
 └───────────┴───────┴──────┘

 [ Disk Blocks Layout ]
 ┌─────┐      ┌─────┐      ┌─────┐      ┌─────┐      ┌─────┐
 │  9  │────▶ │ 16  │────▶ │  1  │────▶ │ 10  │────▶ │ 25  │
 ├─────┤      ├─────┤      ├─────┤      ├─────┤      ├─────┤
 │(Data)      │(Data)      │(Data)      │(Data)      │(Data)│
 │Ptr:16      │Ptr: 1      │Ptr:10      │Ptr:25      │Ptr:-1│
 └─────┘      └─────┘      └─────┘      └─────┘      └─────┘
```

📢 **섹션 요약 비유**: 연결 할당 구조는 "줄줄이 사탕"과 같다. 사탕(데이터)들이 실(포인터)로 연결되어 있어, 실만 따라가면 사탕을 하나씩 계속 꺼낼 수 있는 원리다.

---

## Ⅲ. 연결 할당의 결정적 장점 (Advantages)

- **외부 단편화 제거**: 디스크 상에 흩어진 어떤 빈 블록이라도 활용 가능하므로 공간 낭비가 없다.
- **유연한 크기 조절**: 파일이 커질 때 빈 블록 하나를 찾아 기존 마지막 블록의 포인터만 수정하면 끝이다.
- **사전 정보 불필요**: 파일이 생성될 때 최종 크기를 알 필요가 없다.

📢 **섹션 요약 비유**: 유연함의 이점은 "무한히 늘어나는 기차"와 같다. 승객(데이터)이 늘어날 때마다 기차 칸(블록)을 하나씩 뒤에 더 붙이기만 하면 되므로 기차 길이를 미리 정할 필요가 없다.

---

## Ⅳ. 연결 할당의 대표적 변형: FAT (File Allocation Table)

- **원리**: 포인터들을 각 데이터 블록에 직접 두지 않고, 디스크의 특정 구역에 모아서 표(Table) 형태로 관리한다.
- **장점**: 
    - 실제 데이터 블록은 2의 제곱수 크기를 온전히 유지할 수 있다.
    - 테이블만 메모리에 올리면 검색 속도가 획기적으로 개선된다.
- **사용처**: USB 메모리, SD 카드 등 호환성이 중요한 매체에서 여전히 널리 쓰인다.

📢 **섹션 요약 비유**: FAT는 "목차 페이지"와 같다. 책장 곳곳에 쪽지를 숨기는 대신, 책의 맨 앞 페이지에 '1절은 5페이지, 2절은 20페이지'라고 모아서 적어둔 것과 같다.

---

## Ⅴ. 연결 할당의 실무적 한계 (Challenges)

비록 유연하지만, 다음과 같은 치명적인 단점들을 내포하고 있다.

- **직접 접근 불가 (No Direct Access)**: `i`번째 블록을 읽으려면 반드시 앞선 `i-1`개의 블록을 거쳐야만 한다. (순차 접근만 효율적)
- **포인터 오버헤드**: 수많은 블록의 포인터가 차지하는 공간이 전체 용량의 일정 부분을 소모한다.
- **신뢰성 문제**: 중간 블록의 포인터가 하나라도 손상되면, 그 뒤의 모든 데이터를 잃게 된다.

📢 **섹션 요약 비유**: 직접 접근의 한계는 "테이프 재생"과 같다. 5번째 곡을 듣고 싶으면 앞의 1~4번 곡을 다 감아서 지나쳐야만 도달할 수 있는 답답함과 같다.

## 👶 아이를 위한 비유 (Child Analogy)
> "연결 할당은 장난감들을 집안 여기저기에 숨겨두고, 장난감마다 **'다음 장난감은 어디 있어'**라고 적힌 작은 쪽지를 붙여두는 거야. 로봇 발바닥을 보면 자동차가 어디 있는지 적혀 있고, 자동차 바퀴를 보면 인형이 어디 있는지 적혀 있는 식이지. 집안이 아무리 어질러져 있어도(단편화) 쪽지만 잘 따라가면 모든 장난감을 다 찾을 수 있단다!"
