# 577. NTFS (New Technology File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 윈도우 기본 파일시스템
> 2. **가치**: 보안, 대용량 지원, 복구 기능
> 3. **융합**: ACL, 저널링, 암호화와 연관

---

## Ⅰ. 개요

### 개념 정의
**NTFS**는 **마이크로소프트 윈도우의 기본 파일시스템**입니다.

### 💡 비유: 보안이 강화된 금고
NTFS는 **보안 금고**와 같습니다. 접근 제어와 암호화를 제공합니다.

### NTFS 구조
```

│                NTFS 구조                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NTFS 특징】                                                       │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  항목              NTFS                FAT32                 │ │   │
│  │  ────              ────                ─────                 │ │   │
│  │  최대 파일 크기      16EB (이론)           4GB                   │ │   │
│  │  최대 볼륨 크기      256TB              32GB (기본)            │ │   │
│  │  보안              ACL                없음                   │ │   │
│  │  암호화             EFS                없음                   │ │   │
│  │  압축              지원                없음                   │ │   │
│  │  저널링             $LogFile           없음                   │ │   │
│  │  하드 링크          지원                없음                   │ │   │
│  │  심볼릭 링크         지원                없음                   │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【NTFS 구조】                                                       │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  ┌─────────────────────────────────────────────────────┐    │ │   │
│  │  │                  MFT (Master File Table)             │    │ │   │
│  │  ├─────────────────────────────────────────────────────┤    │ │   │
│  │  │  $MFT          MFT 자체                              │    │ │   │
│  │  │  $MFTMirr      MFT 백업                              │    │ │   │
│  │  │  $LogFile      저널 파일                              │    │ │   │
│  │  │  $Volume       볼륨 정보                              │    │ │   │
│  │  │  $AttrDef      속성 정의                              │    │ │   │
│  │  │  $Bitmap       할당 비트맵                             │    │ │   │
│  │  │  $Boot         부트 섹터                               │    │ │   │
│  │  │  $BadClus      불량 클러스터                           │    │ │   │
│  │  │  ...                                                 │    │ │   │
│  │  │  일반 파일 MFT 엔트리                                 │    │ │   │

│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【MFT 엔트리 구조】                                                  │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  MFT 엔트리 (1KB):                                             │ │   │
│  │  ┌─────────────────────────────────────────────────────┐    │ │   │
│  │  │  Standard Information  (생성/수정/접근 시간)           │    │ │   │
│  │  │  Attribute List        (속성 목록)                     │    │ │   │
│  │  │  File Name             (파일명)                        │    │ │   │
│  │  │  Security Descriptor   (ACL)                           │    │ │   │
│  │  │  Data                  (파일 내용 또는 포인터)           │    │ │   │
│  │  │  Object ID             (고유 식별자)                     │    │ │   │
│  │  │  ...                                                 │    │ │   │

│  │  작은 파일: MFT 내에 직접 저장                                 │ │   │

│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                NTFS 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ACL (Access Control List)】                                       │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  권한 항목:                                                    │ │   │
│  │  • Full Control    - 전체 제어                                │ │   │
│  │  • Modify          - 수정                                    │ │   │
│  │  • Read & Execute  - 읽기 및 실행                             │ │   │
│  │  • Read            - 읽기만                                   │ │   │
│  │  • Write           - 쓰기만                                   │ │   │
│  │                                                             │ │   │
│  │  상속: 부모 디렉토리 권한 자동 상속                              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【EFS (Encrypting File System)】                                    │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  특징:                                                         │ │   │
│  │  • 파일/폴더 단위 암호화                                        │ │   │
│  │  • 사용자 인증서 기반                                           │ │   │
│  │  • 투명한 암호화/복호화                                          │ │   │
│  │  • 속도 저하 최소화                                             │ │   │
│  │                                                             │ │   │
│  │  // 암호화                                                      │ │   │
│  │  cipher /e /s:directory                                         │ │   │
│  │  cipher /e filename                                             │ │   │
│  │                                                             │ │   │
│  │  // 복호화                                                      │ │   │
│  │  cipher /d filename                                             │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【주요 기능】                                                       │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  기능              설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  저널링            $LogFile로 메타데이터 보호                      │ │   │
│  │  압축              파일/폴더 단위 압축                             │ │   │
│  │  할당량            디스크 사용량 제한                              │ │   │
│  │  섀도 복사본        이전 버전 복원                                 │ │   │
│  │  스파스 파일        실제 사용 공간만 할당                           │ │   │
│  │  재분석 지점        마운트 포인트, 심볼릭 링크                       │ │   │
│  │  하드 링크          같은 파일의 여러 이름                           │ │   │
│  │  심볼릭 링크         다른 파일/디렉토리 참조                         │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Windows NTFS 명령어】                                             │
│  ──────────────────                                                │
│  // 파일시스템 확인                                                   │
│  > fsutil fsinfo volumeinfo C:                                      │
│  > chkdsk C: /f                                                     │
│                                                                     │
│  // 권한 관리                                                        │
│  > icacls filename                                                  │
│  > icacls filename /grant user:(F)    // Full control              │
│  > icacls filename /grant user:(M)    // Modify                    │
│  > icacls filename /grant user:(R)    // Read                      │
│  > icacls filename /deny user:(W)     // Deny Write                │
│                                                                     │
│  // 압축                                                            │
│  > compact /c filename                                               │
│  > compact /u filename     // 압축 해제                               │
│  > compact /c /s:directory // 디렉토리 압축                           │
│                                                                     │
│  // 암호화 (EFS)                                                     │
│  > cipher /e filename                                                │
│  > cipher /d filename                                                │
│  > cipher /e /s:directory                                           │
│                                                                     │
│  【Linux에서 NTFS 접근】                                             │
│  ──────────────────                                                │
│  // ntfs-3g 설치                                                     │
│  $ sudo apt install ntfs-3g                                         │
│                                                                     │
│  // 마운트                                                           │
│  $ sudo mount -t ntfs-3g /dev/sdb1 /mnt/windows                     │
│                                                                     │
│  // 읽기 전용 마운트                                                  │
│  $ sudo mount -t ntfs-3g -o ro /dev/sdb1 /mnt/windows               │
│                                                                     │
│  // /etc/fstab                                                       │
│  /dev/sdb1  /mnt/windows  ntfs-3g  defaults,uid=1000  0  0         │
│                                                                     │
│  // NTFS 파티션 생성                                                  │
│  $ sudo mkntfs /dev/sdb1                                            │
│  $ sudo mkntfs -L mylabel /dev/sdb1                                 │
│                                                                     │
│  // NTFS 정보 확인                                                    │
│  $ ntfsinfo /dev/sdb1                                               │
│  $ ntfsfix /dev/sdb1          // 기본 복구                            │
│                                                                     │
│  【하드 링크 / 심볼릭 링크】                                           │ |
│  ──────────────────                                                │
│  // Windows                                                          │
│  > mklink /D link target     // 심볼릭 링크 (디렉토리)                │
│  > mklink link target        // 심볼릭 링크 (파일)                    │
│  > mklink /H link target     // 하드 링크                             │
│  > mklink /J link target     // 접합 (Junction)                      │
│                                                                     │
│  // Linux (NTFS-3G)                                                  │
│  $ ln target hardlink                                                │
│  $ ln -s target symlink                                              │
│                                                                     │
│  【할당량 (Quota)】                                                   │
│  ──────────────────                                                │
│  // Windows                                                          │
│  > fsutil quota enable C:                                            │
│  > fsutil quota modify C: 1000 1000000000 username                   │
│  // 1GB 경고, 1GB 제한                                               │
│                                                                     │
│  // 또는 디스크 속성 > 할당량 탭에서 설정                               │
│                                                                     │
│  【섀도 복사본 (Shadow Copy)】                                        │ |
│  ──────────────────                                                │
│  // 이전 버전 복원                                                    │
│  // 파일 속성 > 이전 버전 탭                                          │
│                                                                     │
│  // vssadmin 명령어                                                  │
│  > vssadmin list shadows                                            │
│  > vssadmin create shadow /for=C:                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 윈도우 기본 파일시스템
• 최대 크기: 파일 16EB (이론), 볼륨 256TB
• MFT: Master File Table로 메타데이터 관리
• ACL: 세밀한 접근 제어
• EFS: 파일 시스템 수준 암호화
• 저널링: $LogFile으로 복구 지원
• 압축: 파일/폴더 단위 투명 압축
• 할당량: 사용자별 디스크 제한
• 섀도 복사본: 이전 버전 복원
• Linux: ntfs-3g로 접근
• 권한: icacls 명령어
```
