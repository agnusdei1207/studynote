# 477. NFS (Network File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크를 통한 원격 파일 시스템 접근
> 2. **가치**: 투명한 원격 파일 공유
> 3. **융합**: VFS, RPC, 네트워킹과 연관

---

## Ⅰ. 개요

### 개념 정의

NFS(Network File System)는 **네트워크를 통해 원격 서버의 파일 시스템을 로컬처럼 사용할 수 있게 하는 분산 파일 시스템**이다.

### 💡 비유: 클라우드 드라이브
NFS는 **클라우드 드라이브**와 같다. 내 컴퓨터에서 원격 저장소를 내 폴더처럼 사용한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                NFS 상세                                               │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RPC (Remote Procedure Call)】                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  NFS는 RPC 위에서 동작:                                               │ │
│  │  • portmapper/rpcbind: RPC 서비스 등록                                │ │
│  │  • rpc.mountd: 마운트 요청 처리                                        │ │
│  │  • rpc.nfsd: NFS 서비스 제공                                          │ │
│  │  • rpc.rquotad: 쿼터 정보                                             │ │
│  │  • rpc.lockd: 파일 락킹 (NFSv3)                                       │ │
│  │  • rpc.statd: 파일 락 상태 복구 (NFSv3)                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NFS 마운트 옵션】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  성능 옵션:                                                             │ │
│  │  • rsize/wsize: 읽기/쓰기 버퍼 크기 (기본 1MB)                         │ │
│  │  • hard/soft: 서버 장애 시 동작                                        │ │
│  │  • timeo: 타임아웃 (1/10초 단위)                                       │ │
│  │  • retrans: 재전송 횟수                                               │ │
│  │  • async/sync: 비동기/동기 쓰기                                        │ │
│  │                                                             │ │
│  │  보안 옵션:                                                             │ │
│  │  • sec=sys: UNIX 인증 (기본)                                          │ │
│  │  • sec=krb5: Kerberos 인증                                            │ │
│  │  • sec=krb5i: Kerberos + 무결성                                       │ │
│  │  • sec=krb5p: Kerberos + 암호화                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                               │
│  │  • 투명한 원격 파일 접근                                              │ │
│  │  • 중앙 집중식 파일 관리                                              │ │
│  │  • 디스크 공간 절약                                                   │ │
│  │  • 유닉스/리눅스 표준                                                 │ │
│  │                                                             │ │
│  │  단점:                                                               │
│  │  • 네트워크 의존성                                                    │ │
│  │  • 서버 장애 시 문제                                                  │ │
│  │  • 보안 고려 필요                                                     │ │
│  │  • 성능은 로컬보다 느림                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NFS 서버 설정】                                                    │
│  ──────────────────                                                  │
│  // 패키지 설치                                                       │
│  $ sudo apt install nfs-kernel-server                              │
│                                                                     │
│  // /etc/exports 설정                                                 │
│  /export/data    192.168.1.0/24(rw,sync,no_subtree_check)          │
│  /export/home    *(rw,async,no_root_squash)                        │
│  /export/ro      192.168.1.100(ro)                                  │
│                                                                     │
│  // exports 옵션                                                       │
│  // rw/ro: 읽기쓰기/읽기전용                                           │
│  // sync/async: 동기/비동기                                            │
│  // no_root_squash: root 권한 유지                                    │
│  // no_subtree_check: 서브트리 검사 안 함                              │
│                                                                     │
│  // 설정 적용                                                        │
│  $ sudo exportfs -ra                                                │
│  $ sudo systemctl restart nfs-kernel-server                        │
│                                                                     │
│  【NFS 클라이언트 설정】                                              │
│  ──────────────────                                                  │
│  // 패키지 설치                                                       │
│  $ sudo apt install nfs-common                                      │
│                                                                     │
│  // 마운트                                                           │
│  $ sudo mount -t nfs4 server:/export/data /mnt/nfs                 │
│                                                                     │
│  // 옵션 지정                                                         │
│  $ sudo mount -t nfs4 -o rw,hard,intr server:/data /mnt/nfs        │
│                                                                     │
│  // /etc/fstab에 영구 설정                                            │
│  server:/export/data  /mnt/nfs  nfs4  rw,hard  0  0                │
│                                                                     │
│  【NFS 상태 확인】                                                    │
│  ──────────────────                                                  │
│  // 서버에서 내보낸 목록 확인                                          │
│  $ sudo exportfs -v                                                 │
│  /export/data    192.168.1.0/24(rw,wdelay,no_root_squash)          │
│                                                                     │
│  // 클라이언트에서 마운트 정보                                          │
│  $ showmount -e server                                              │
│  Export list for server:                                            │
│  /export/data 192.168.1.0/24                                        │
│                                                                     │
│  // NFS 통계                                                          │
│  $ nfsstat -c  // 클라이언트                                          │
│  $ nfsstat -s  // 서버                                                │
│                                                                     │
│  // RPC 서비스 확인                                                   │
│  $ rpcinfo -p server                                                │
│  program vers proto   port                                          │
│  100003    4   tcp    2049  nfs                                     │
│  100005    3   udp    20048  mountd                                 │
│                                                                     │
│  【문제 해결】                                                          │
│  ──────────────────                                                  │
│  // 마운트 실패 시                                                      │
│  $ sudo mount -v -t nfs4 server:/data /mnt/nfs                     │
│  // 상세 에러 확인                                                     │
│                                                                     │
│  // 연결 테스트                                                        │
│  $ rpcinfo -u server nfs 4                                          │
│  program 100003 version 4 ready and waiting                        │
│                                                                     │
│  // 방화벽 설정 (NFSv4는 2049만 필요)                                 │
│  $ sudo ufw allow from 192.168.1.0/24 to any port nfs             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
