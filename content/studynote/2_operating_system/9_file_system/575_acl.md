# 575. 파일 시스템 (File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 저장 장치의 데이터를 관리하는 방법과 구조
> 2. **가치**: 파일 조직화, 접근 제어, 공간 관리
> 3. **융합**: inode, 디렉토리, 마운트와 연관

---

## Ⅰ. 개요

### 개념 정의
**파일 시스템(File System)**은 **저장 장치의 데이터를 체계적으로 관리하는 방법과 구조**입니다.

### 💡 비유: 도서관 분류 시스템
파일 시스템은 **도서관 분류 시스템**과 같습니다. 책을 찾고 정리하는 체계입니다.

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 상세                                       │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【VFS (Virtual File System)】                                       │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  목적:                                                           │ │   │
│  │  • 다양한 파일 시스템에 통합 인터페이스 제공                       │ │   │
│  │  • 애플리케이션이 파일 시스템 타입에 독립적                        │ │   │
│  │                                                             │ │   │
│  │  주요 객체:                                                       │ │   │
│  │  • superblock: 파일시스템 정보                                   │ │   │
│  │  • inode: 파일 메타데이터                                        │ │   │
│  │  • dentry: 디렉토리 엔트리                                       │ │   │
│  │  • file: 열린 파일                                              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【저널링 파일 시스템】                                                │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  저널링 모드:                                                     │ │   │
│  │  • data=journal: 모든 데이터 + 메타데이터 저널링                   │ │   │
│  │  • data=ordered: 메타데이터만 저널링, 데이터 순서 보장              │ │   │
│  │  • data=writeback: 메타데이터만 저널링 (빠름)                      │ │   │
│  │                                                             │ │   │
│  │  장점:                                                           │ │   │
│  │  • 빠른 복구 시간                                                 │ │   │
│  │  • 데이터 무결성 향상                                             │ │   │
│  │                                                             │ │   │
│  │  단점:                                                           │ │   │
│  │  • 오버헤드                                                       │ │   │
│  │  • 추가 디스크 공간                                               │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【블록 할당 방식】                                                    │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  방식              설명                    장점          단점          │ │   │
│  │  ────              ────                    ────          ────          │ │   │
│  │  연속 할당          연속된 블록             순차 접근 빠름   단편화        │ │   │
│  │  연결 할당          링크드 리스트           외부 단편화 없음 느림          │ │   │
│  │  인덱스 할당        inode 포인터            직접 접근      간접 블록      │ │   │
│  │  ext4 extent       연속 범위               효율적         복잡          │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파일 시스템 확인】                                                  │ |
│  ──────────────────                                                │
│  // 마운트된 파일 시스템                                               │
│  $ df -Th                                                           │
│  Filesystem     Type   Size  Used Avail Use% Mounted on            │
│  /dev/sda1      ext4   100G   15G   85G  15% /                      │
│  /dev/sda2      xfs    500G  200G  300G  40% /data                  │
│  tmpfs          tmpfs  7.8G     0  7.8G   0% /dev/shm               │
│                                                                     │
│  // 블록 장치 파일 시스템                                               │
│  $ lsblk -f                                                         │
│  NAME FSTYPE FSVER LABEL    UUID                                 │
│  sda                                                     │
│  ├─sda1 ext4   1.0   ROOT     12345678-1234-1234-1234-123456789012│
│  └─sda2 xfs          DATA     a1b2c3d4-e5f6-7890-abcd-ef1234567890│
│                                                                     │
│  【파일 시스템 생성】                                                  │ |
│  ──────────────────                                                │
│  // ext4 생성                                                        │
│  $ sudo mkfs.ext4 /dev/sdb1                                         │
│  $ sudo mkfs.ext4 -L mylabel /dev/sdb1   // 라벨 지정                 │
│  $ sudo mkfs.ext4 -b 4096 /dev/sdb1      // 블록 크기                 │
│  $ sudo mkfs.ext4 -N 1000000 /dev/sdb1   // inode 수                  │
│                                                                     │
│  // XFS 생성                                                         │
│  $ sudo mkfs.xfs /dev/sdb2                                          │
│  $ sudo mkfs.xfs -L mydata /dev/sdb2                                │
│                                                                     │
│  // FAT32 생성                                                       │
│  $ sudo mkfs.vfat -F 32 /dev/sdb3                                   │
│                                                                     │
│  // Btrfs 생성                                                       │
│  $ sudo mkfs.btrfs /dev/sdb4                                        │
│  $ sudo mkfs.btrfs -L mybtrfs -d raid1 -m raid1 /dev/sdb4 /dev/sdb5 │
│                                                                     │
│  【파일 시스템 마운트】                                                 │ |
│  ──────────────────                                                │
│  // 임시 마운트                                                        │
│  $ sudo mount /dev/sdb1 /mnt/data                                   │
│  $ sudo mount -t ext4 /dev/sdb1 /mnt/data                           │
│  $ sudo mount -o ro /dev/sdb1 /mnt/data    // 읽기 전용               │
│  $ sudo mount -o noexec /dev/sdb1 /mnt/data  // 실행 방지             │
│                                                                     │
│  // 영구 마운트 (/etc/fstab)                                          │
│  /dev/sdb1  /mnt/data  ext4  defaults  0  2                         │
│  UUID=xxx   /mnt/data  ext4  defaults  0  2                         │
│  // 필드: 장치, 마운트포인트, 타입, 옵션, dump, fsck 순서               │
│                                                                     │
│  // 마운트 해제                                                       │
│  $ sudo umount /mnt/data                                            │
│  $ sudo umount -l /mnt/data    // lazy unmount                       │
│                                                                     │
│  【파일 시스템 유지보수】                                               │ |
│  ──────────────────                                                │
│  // 공간 확보                                                        │
│  $ sudo fstrim /                  // SSD TRIM                        │
│  $ sudo fstrim -a                 // 모든 파일시스템                  │
│  $ sudo btrfs filesystem df /     // Btrfs 사용량                    │
│  $ sudo btrfs balance start /     // Btrfs 밸런싱                     │
│                                                                     │
│  // 온라인 확장                                                       │
│  $ sudo resize2fs /dev/sda1        // ext4 확장                       │
│  $ sudo xfs_growfs /data          // XFS 확장                        │
│                                                                     │
│  // 라벨 변경                                                        │
│  $ sudo e2label /dev/sda1 newlabel  // ext4                          │
│  $ sudo xfs_admin -L newlabel /dev/sda2  // XFS                     │
│                                                                     │
│  【파일 시스템 변환】                                                  │ |
│  ──────────────────                                                │
│  // ext2 → ext3 (저널 추가)                                           │
│  $ sudo tune2fs -j /dev/sda1                                        │
│                                                                     │
│  // ext3 → ext4                                                       │
│  $ sudo tune2fs -O extents,uninit_bg,dir_index /dev/sda1            │
│  $ sudo e2fsck -f /dev/sda1                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 저장 장치 데이터 관리 방법과 구조
• 계층: VFS → 파일시스템 → 블록 드라이버 → 장치
• 주요 파일시스템: ext4, XFS, Btrfs, NTFS, FAT32
• VFS: 통합 인터페이스, 파일시스템 독립성
• 저널링: data=journal, ordered, writeback
• 할당: 연속, 연결, 인덱스, extent
• 기능: 파일/디렉토리 관리, 공간, 접근 제어
• 생성: mkfs.ext4, mkfs.xfs, mkfs.btrfs
• 마운트: mount, umount, /etc/fstab
• 유지보수: resize2fs, fstrim, xfs_growfs
```
