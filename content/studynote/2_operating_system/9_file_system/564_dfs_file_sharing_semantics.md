## 핵심 인사이트 (3줄 요약)
> 1. **공유 세맨틱의 정의**: 공유 세맨틱(Sharing Semantics)은 다중 사용자 시스템에서 여러 클라이언트가 동시에 같은 파일에 접근할 때, 어떤 시점에 수정 사항이 다른 사용자에게 보이는지를 규정하는 규칙이다.
> 2. **일관성 vs 성능**: 즉각적인 업데이트를 보장하는 '유닉스 세맨틱'과 성능 최적화를 위해 지연 업데이트를 허용하는 '세션 세맨틱' 사이의 트레이드오프(Trade-off)가 발생한다.
> 3. **데이터 무결성**: 분산 환경에서는 네트워크 지연으로 인해 완벽한 실시간 동기화가 어렵기 때문에, 애플리케이션의 성격에 맞는 적절한 세맨틱 선택이 필수적이다.

---

## Ⅰ. 파일 공유 세맨틱 (File Sharing Semantics) 개요

- **개념**: 여러 사용자가 동시에 파일을 읽고 쓸 때 발생하는 데이터 일관성(Consistency) 문제를 해결하기 위한 시스템 설계의 규약이다.
- **주요 과제**: "사용자 A가 파일에 'X'를 썼을 때, 사용자 B가 즉시 'X'를 읽을 수 있어야 하는가?"에 대한 답을 결정한다.

📢 **섹션 요약 비유**: 공유 세맨틱은 "대화 규칙"과 같다. 말하자마자 바로 들려야 하는지(실시간), 아니면 편지처럼 다 쓰고 나서 보내야 하는지(세션)를 정하는 약속이다.

---

## Ⅱ. 주요 공유 모델 비교 (ASCII Diagram)

분산 환경에서 업데이트가 전파되는 시점을 시각화한 구조도다.

```text
 [ Time Axis ──────────────▶ ]

 1. Unix Semantics (Immediate)
 User A: ---[Write X]--------------------
 User B: --------------[Read X]---------- (B sees X immediately)

 2. Session Semantics (Close-to-Open)
 User A: ---[Write X]---[Close]----------
 User B: -----------------------[Open]---[Read X] (B sees X only after A closes)

 3. Immutable Semantics (Read-Only)
 File:   ---[Creation/Write]---[Seal]----
 User A: -------------------------------[Read only]
 User B: -------------------------------[Read only] (No one can change after seal)
```

📢 **섹션 요약 비유**: 유닉스 세맨틱은 "화상 통화"이고, 세션 세맨틱은 "이메일"이며, 불변 세맨틱은 "출판된 책"과 같다.

---

## Ⅲ. 유닉스 세맨틱 (Unix Semantics)

- **정의**: 단일 머신에서의 파일 시스템 동작을 분산 환경에서 그대로 구현한 모델이다.
- **특징**:
    - **즉각성**: 쓰기 연산의 결과가 다른 모든 클라이언트에게 즉시 가시화된다.
    - **절대적 일관성**: 데이터가 항상 최신 상태임을 보장한다.
- **한계**: 분산 환경에서는 네트워크 오버헤드가 너무 커서 구현이 어렵고 성능이 크게 저하된다.

📢 **섹션 요약 비유**: 유닉스 세맨틱은 "공동 작업용 화이트보드"와 같다. 누군가 보드에 한 획을 그으면 옆에 서 있는 모든 사람이 그 즉시 그 획을 볼 수 있는 것과 같다.

---

## Ⅳ. 세션 세맨틱 (Session Semantics)

- **정의**: 파일을 열고(Open) 닫는(Close) 과정을 하나의 세션(Session)으로 정의하고, 수정 사항은 파일이 닫힐 때 서버에 반영하는 방식이다.
- **특징**:
    - **지연 업데이트**: 세션 중간의 수정 사항은 다른 사용자에게 보이지 않는다.
    - **성능 우수**: 네트워크 통신 횟수를 최소화하여 대규모 시스템(예: AFS)에서 선호된다.
    - **충돌 위험**: 두 사용자가 동시에 수정하고 닫으면 나중에 닫은 사용자의 내용이 덮어씌워질 수 있다.

📢 **섹션 요약 비유**: 세션 세맨틱은 "숙제 검사"와 같다. 집에서 숙제를 고쳐도 선생님은 나중에 숙제 노트를 제출(Close)해야만 고친 내용을 알 수 있는 것과 같다.

---

## Ⅴ. 불변 파일 세맨틱 (Immutable Semantics)

- **정의**: 파일이 한 번 생성되어 공유되면 더 이상 수정할 수 없도록 하는 방식이다.
- **특징**:
    - **단순성**: 동시성 제어(Concurrency Control)나 잠금(Locking)이 필요 없다.
    - **안전성**: 읽기 전용으로 공유되므로 데이터 변조 위험이 없다.
    - **수정 방식**: 내용을 바꾸려면 새로운 이름의 파일을 생성해야 한다.
- **예시**: WORM(Write Once, Read Many) 미디어나 일부 정적 데이터 저장소.

📢 **섹션 요약 비유**: 불변 세맨틱은 "역사 책"과 같다. 이미 인쇄되어 배포된 책의 내용은 바꿀 수 없으며, 새로운 사실이 발견되면 '개정판'이라는 새 책을 내야 하는 것과 같다.
