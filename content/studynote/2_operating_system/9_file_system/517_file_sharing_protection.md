## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 공유 (File Sharing)는 다중 사용자 시스템에서 여러 사용자가 동일한 파일에 접근할 수 있게 하는 기능이며, 보호 (Protection)는 권한이 없는 접근으로부터 데이터를 지키는 메커니즘이다.
> 2. **가치**: 협업 효율성을 높이는 동시에, 사용자 식별 (UID/GID)과 접근 제어 행렬을 통해 시스템의 기밀성 (Confidentiality)과 무결성 (Integrity)을 유지한다.
> 3. **일관성**: 공유 파일에 대한 동시 수정 문제를 해결하기 위해 유닉스 시맨틱 (Unix Semantics)이나 세션 시맨틱 (Session Semantics)과 같은 일관성 모델을 적용한다.

---

## Ⅰ. 개요 (Context & Background)

- **정의**: 시스템 내의 자원을 여러 사용자가 공동으로 사용하도록 허용하면서도, 각 자원의 소유권과 접근 범위를 명확히 규정하여 안전하게 관리하는 일련의 과정이다.
- **배경**: 메인프레임이나 서버 환경처럼 여러 사람이 동시에 한 대의 컴퓨터를 사용하는 시분할 시스템이 발달하며 필수적인 기능이 되었다.
- **핵심 요소**:
  - **사용자 ID (UID, User ID)**: 각 사용자를 식별하는 고유 번호.
  - **그룹 ID (GID, Group ID)**: 여러 사용자를 하나의 권한 단위로 묶는 번호.

📢 **섹션 요약 비유**: 파일 공유와 보호는 "공유 오피스의 출입 통제"와 같다. 회의실(파일)을 다 같이 쓰기도 하지만, 아무나 남의 책상을 뒤지지 못하게 출입증(ID)과 열쇠(권한)로 관리하는 것과 같다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 보호 체계 시각화 (ASCII Diagram)

```text
 [ User Context ]         [ Access Control ]         [ File System ]
 ┌──────────────┐         ┌────────────────┐         ┌─────────────┐
 │ UID: 1001    ├───────▶ │ Check: rwx     ├───────▶ │ File: a.txt │
 │ GID: 500     │         │ Owner: 1001    │         │ Data Blocks │
 └──────────────┘         └────────────────┘         └─────────────┘
                                  ▲
                         [ Permission Bits ]
                         ┌───┬───┬───┐
                         │ r │ w │ x │ (Owner)
                         ├───┼───┼───┤
                         │ r │ - │ - │ (Group)
                         ├───┼───┼───┤
                         │ - │ - │ - │ (Others)
                         └───┴───┴───┘
```

### 2. 접근 제어 방식 (Access Control)
- **소유권 (Ownership)**: 파일을 생성한 사용자가 기본적으로 소유자가 되며, 권한 설정의 전권을 가진다.
- **접근 권한 비트 (Permission Bits)**:
  - **Read (r)**: 파일 내용 읽기 / 디렉터리 내 목록 보기.
  - **Write (w)**: 파일 내용 수정 / 디렉터리 내 파일 생성 및 삭제.
  - **Execute (x)**: 파일을 프로그램으로 실행 / 디렉터리 내로 진입(`cd`).

📢 **섹션 요약 비유**: 보호 방식은 "신분증 검사"와 같다. 문 앞의 관리자가 신분증(UID)을 확인하고, 명부(권한 비트)에 적힌 대로 '입장만 가능'할지 '물건 이동도 가능'할지 결정하는 것이다.

## Ⅲ. 공유 파일의 일관성 모델 (Consistency Semantics)

공유된 파일을 여러 명의 사용자가 동시에 읽고 쓸 때 발생하는 문제를 해결하기 위한 규칙이다.

1. **유닉스 시맨틱 (Unix Semantics)**: 한 사용자가 쓴 내용이 즉시 다른 모든 사용자에게 보이게 한다. (강력한 일관성)
2. **세션 시맨틱 (Session Semantics)**: 파일을 닫을(Close) 때만 변경 사항을 전체 공유 폴더에 반영한다. (NFS 등에서 사용)
3. **불변 공유 파일 (Immutable Shared Files)**: 파일이 한 번 공유되면 수정이 불가능하게 하여 동시성 문제를 원천 차단한다.

📢 **섹션 요약 비유**: 일관성 모델은 "공유 문서 편집 규칙"과 같다. '글자 하나 쓸 때마다 실시간 중계하기(유닉스)'와 '편집 다 끝내고 저장 버튼 누를 때만 공개하기(세션)'의 차이다.

## Ⅳ. 융합 비교 및 분석 (Comparison)

| 비교 항목 | 단일 사용자 시스템 | 다중 사용자 시스템 | 분산 파일 시스템 (DFS) |
|:---|:---|:---|:---|
| **식별자** | 필요 없음 | UID, GID 기반 | 네트워크 ID (Kerberos 등) |
| **보호 수준** | 낮음 (물리적 접근 통제) | 높음 (ACL 등 적용) | 매우 높음 (암호화 필수) |
| **공유 범위** | 로컬 시스템 내부 | 시스템 내 사용자 간 | 원격 컴퓨터 사용자 간 |
| **복잡도** | 단순 | 보통 | 높음 (네트워크 보안 필요) |

📢 **섹션 요약 비유**: 단일 사용자가 "나 혼자 사는 집"이라면, 다중 사용자는 "아파트 단지", 분산 시스템은 "도시 간을 잇는 거대한 유통망"과 같다.

## Ⅴ. 기술적 구현 및 실무 이슈 (Implementation)

- **원격 파일 공유**: NFS (Unix/Linux), SMB/CIFS (Windows) 등의 프로토콜을 사용하여 네트워크 너머의 파일을 로컬처럼 사용한다.
- **보안 위협**: 잘못된 권한 설정(예: `777` 권한)은 시스템 전체의 보안을 무너뜨리는 가장 큰 원인이다.
- **최소 권한의 원칙 (Principle of Least Privilege)**: 사용자에게 업무 수행에 필요한 최소한의 권한만 부여하는 것이 현대 보안의 기본 철학이다.

📢 **섹션 요약 비유**: 권한 설정은 "열쇠 관리"와 같다. 모든 사람에게 마스터키를 주면 편하겠지만, 도둑이 들었을 때 모든 방이 털리는 위험을 감수해야 하는 것과 같다.
