# 587. LVM (Logical Volume Manager)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리적 디스크를 논리적으로 추상화
> 2. **가치**: 유연한 저장소 관리, 동적 확장
> 3. **융합**: PV, VG, LV, 스냅샷과 연관

---

## Ⅰ. 개요

### 개념 정의
**LVM(Logical Volume Manager)**은 **물리적 저장 장치를 논리적 볼륨으로 추상화하여 관리하는 시스템**입니다.

### 💡 비유: 가변형 창고
LVM은 **가변형 창고**와 같습니다. 필요에 따라 공간을 늘리거나 줄일 수 있습니다.

### LVM 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                LVM 계층 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LVM 계층】                                                         │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │                    논리적 볼륨 (LV)                             │ │   │
│  │                    ┌─────┬─────┬─────┐                         │ │   │
│  │                    │ LV1 │ LV2 │ LV3 │                         │ │   │
│  │                    └──┬──┴──┬──┴──┬──┘                         │ │   │
│  │                       │      │      │                            │ │   │
│  │                    볼륨 그룹 (VG)                                │ │   │
│  │                    ┌────────────────┐                          │ │   │
│  │                    │      VG0       │                          │ │   │
│  │                    └───────┬────────┘                          │ │   │
│  │                            │                                    │ │   │
│  │                    물리적 볼륨 (PV)                              │ │   │
│  │              ┌─────────────┼─────────────┐                      │ │   │
│  │              │             │             │                       │ │   │
│  │           ┌──┴──┐      ┌──┴──┐      ┌──┴──┐                    │ │   │
│  │           │ PV1 │      │ PV2 │      │ PV3 │                    │ │   │
│  │           └──┬──┘      └──┬──┘      └──┬──┘                    │ │   │
│  │              │             │             │                       │ │   │
│  │           물리적        물리적        물리적                      │ │   │
│  │           디스크        디스크        디스크                       │ │   │
│  │           /dev/sda     /dev/sdb     /dev/sdc                    │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【용어 설명】                                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  용어              설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  PV (Physical Volume)   물리적 볼륨, 디스크/파티션                  │ │   │
│  │  VG (Volume Group)      볼륨 그룹, PV의 집합                       │ │   │
│  │  LV (Logical Volume)    논리적 볼륨, 실제 사용하는 공간               │ │   │
│  │  PE (Physical Extent)   물리적 익스텐트, PV의 최소 단위              │ │   │
│  │  LE (Logical Extent)    논리적 익스텐트, LV의 최소 단위              │ │   │
│  │                                                             │ │   │
│  │  기본 PE 크기: 4MB                                                 │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【LVM 장점】                                                         │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  기능              설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  동적 확장          시스템 중단 없이 볼륨 크기 증가                    │ │   │
│  │  동적 축소          온라인 상태에서 볼륨 크기 감소                     │ │   │
│  │  스냅샷             볼륨의 특정 시점 복사본                          │ │   │
│  │  스트라이핑          여러 디스크에 분산 저장 (성능)                    │ │   │
│  │  미러링             데이터 복제 (안정성)                            │ │   │
│  │  이동              PV 간 데이터 이동                              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                LVM 상세                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LVM 타입】                                                         │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  타입              설명                    특징                   │ │   │
│  │  ────              ────                    ────                   │ │   │
│  │  linear            기본, 순차 할당           기본값                 │ │   │
│  │  striped           스트라이핑              성능 향상               │ │   │
│  │  mirror            미러링                 데이터 보호              │ │   │
│  │  thin              씬 프로비저닝           실제 사용량만 할당         │ │   │
│  │  raid0~6           LVM RAID               mdadm 대체              │ │   │
│  │  cache             캐시                   SSD 캐시                │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【씬 프로비저닝 (Thin Provisioning)】                                  │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  개념:                                                           │ │   │
│  │  • 논리적으로 큰 볼륨 생성                                         │ │   │
│  │  • 실제 사용하는 만큼만 물리 공간 할당                               │ │   │
│  │                                                             │ │   │
│  │  예시:                                                           │ │   │
│  │  • 물리 공간: 100GB                                               │ │   │
│  │  • Thin Pool: 100GB                                              │ │   │
│  │  • Thin LV 1: 50GB (실제 사용: 10GB)                              │ │   │
│  │  • Thin LV 2: 80GB (실제 사용: 20GB)                              │ │   │
│  │  • 오버커밋 가능                                                   │ │   │
│  │                                                             │ │   │
│  │  주의:                                                           │ │   │
│  │  • 실제 공간 고갈 시 I/O 오류                                       │ │   │
│  │  • 모니터링 필수                                                   │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【스냅샷】                                                           │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  특징:                                                           │ │   │
│  │  • 특정 시점의 볼륨 상태 보존                                       │ │   │
│  │  • COW (Copy-On-Write) 방식                                      │ │   │
│  │  • 변경되는 블록만 복사                                             │ │   │
│  │  • 빠른 생성                                                      │ │   │
│  │                                                             │ │   │
│  │  용도:                                                           │ │   │
│  │  • 백업                                                          │ │   │
│  │  • 테스트 환경                                                    │ │   │
│  │  • 복구                                                          │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【PV 생성】                                                          │
│  ──────────────────                                                │
│  // 물리적 볼륨 생성                                                   │
│  $ sudo pvcreate /dev/sdb1                                         │
│  $ sudo pvcreate /dev/sdb /dev/sdc                                 │
│                                                                     │
│  // PV 확인                                                          │
│  $ sudo pvdisplay                                                   │
│  $ sudo pvs                         // 간단한 목록                    │
│  $ sudo pvscan                                                      │
│                                                                     │
│  // PV 삭제                                                          │
│  $ sudo pvremove /dev/sdb1                                         │
│                                                                     │
│  【VG 생성】                                                          │
│  ──────────────────                                                │
│  // 볼륨 그룹 생성                                                     │
│  $ sudo vgcreate vg0 /dev/sdb1 /dev/sdc1                           │
│  $ sudo vgcreate -s 16M vg0 /dev/sdb1    // PE 크기 지정             │
│                                                                     │
│  // VG 확인                                                          │
│  $ sudo vgdisplay                                                   │
│  $ sudo vgs                                                         │
│  $ sudo vgscan                                                      │
│                                                                     │
│  // VG 확장                                                          │
│  $ sudo vgextend vg0 /dev/sdd1                                     │
│                                                                     │
│  // VG 축소                                                          │
│  $ sudo vgreduce vg0 /dev/sdb1                                     │
│                                                                     │
│  // VG 삭제                                                          │
│  $ sudo vgremove vg0                                                │
│                                                                     │
│  【LV 생성】                                                          │
│  ──────────────────                                                │
│  // 기본 LV 생성                                                      │
│  $ sudo lvcreate -L 50G -n lv0 vg0                                 │
│  $ sudo lvcreate -l 100%FREE -n lv1 vg0    // 남은 공간 모두          │
│                                                                     │
│  // 스트라이프 LV                                                      │
│  $ sudo lvcreate -L 100G -i 2 -I 64K -n striped_lv vg0             │
│  // -i: 스트라이프 수, -I: 스트라이프 크기                              │
│                                                                     │
│  // 미러 LV                                                          │
│  $ sudo lvcreate -L 50G -m1 -n mirror_lv vg0                       │
│                                                                     │
│  // RAID LV                                                          │
│  $ sudo lvcreate --type raid1 -L 50G -n raid_lv vg0                │
│  $ sudo lvcreate --type raid5 -L 100G -n raid5_lv vg0              │
│                                                                     │
│  // LV 확인                                                          │
│  $ sudo lvdisplay                                                   │
│  $ sudo lvs                                                         │
│  $ sudo lvscan                                                      │
│                                                                     │
│  // 경로: /dev/vg0/lv0 또는 /dev/mapper/vg0-lv0                      │
│                                                                     │
│  【LV 확장/축소】                                                      │ |
│  ──────────────────                                                │
│  // 확장                                                             │
│  $ sudo lvextend -L +10G /dev/vg0/lv0                              │
│  $ sudo lvextend -l +100%FREE /dev/vg0/lv0                         │
│                                                                     │
│  // 파일시스템 확장 (함께)                                              │
│  $ sudo lvextend -L +10G -r /dev/vg0/lv0                           │
│  $ sudo lvextend -l +100%FREE -r /dev/vg0/lv0                      │
│                                                                     │
│  // 또는 별도 파일시스템 확장                                            │
│  $ sudo lvextend -L +10G /dev/vg0/lv0                              │
│  $ sudo resize2fs /dev/vg0/lv0           // ext4                    │
│  $ sudo xfs_growfs /mnt/data             // XFS (확장만)             │
│                                                                     │
│  // 축소 (주의!)                                                       │
│  $ sudo umount /dev/vg0/lv0                                        │
│  $ sudo e2fsck -f /dev/vg0/lv0                                     │
│  $ sudo resize2fs /dev/vg0/lv0 40G                                 │
│  $ sudo lvreduce -L 40G /dev/vg0/lv0                               │
│  $ sudo mount /dev/vg0/lv0 /mnt/data                               │
│                                                                     │
│  【Thin Provisioning】                                               │ |
│  ──────────────────                                                │
│  // Thin Pool 생성                                                   │
│  $ sudo lvcreate -L 100G -T vg0/thin_pool                          │
│                                                                     │
│  // Thin LV 생성                                                      │
│  $ sudo lvcreate -V 50G -T vg0/thin_pool -n thin_lv1               │
│  $ sudo lvcreate -V 80G -T vg0/thin_pool -n thin_lv2               │
│                                                                     │
│  // Thin Pool 확인                                                    │
│  $ sudo lvs -a                                                      │
│  $ sudo lvdisplay vg0/thin_pool                                    │
│                                                                     │
│  // Thin Pool 확장                                                    │
│  $ sudo lvextend -L +50G /dev/vg0/thin_pool                        │
│                                                                     │
│  【스냅샷】                                                           │ |
│  ──────────────────                                                │
│  // 스냅샷 생성                                                        │
│  $ sudo lvcreate -L 10G -s -n snap_lv0 /dev/vg0/lv0                │
│                                                                     │
│  // 스냅샷 확인                                                        │
│  $ sudo lvs -a                                                      │
│                                                                     │
│  // 스냅샷 마운트                                                       │
│  $ sudo mount /dev/vg0/snap_lv0 /mnt/snapshot                      │
│                                                                     │
│  // 스냅샷 복원                                                        │
│  $ sudo umount /dev/vg0/lv0                                        │
│  $ sudo lvconvert --merge /dev/vg0/snap_lv0                        │
│                                                                     │
│  // 스냅샷 삭제                                                        │
│  $ sudo lvremove /dev/vg0/snap_lv0                                 │
│                                                                     │
│  【LVM 상태 확인】                                                     │ |
│  ──────────────────                                                │
│  // 전체 요약                                                          │
│  $ sudo lvs -a -o +devices                                         │
│  $ sudo vgs -v                                                      │
│  $ sudo pvs -v                                                      │
│                                                                     │
│  // 사용량 확인                                                        │
│  $ df -h /dev/mapper/vg0-lv0                                       │
│                                                                     │
│  // 디바이스 맵퍼                                                      │
│  $ ls -la /dev/mapper/                                              │
│  $ sudo dmsetup ls                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 물리적 디스크를 논리적 볼륨으로 추상화
• 계층: PV → VG → LV
• PV: 물리적 볼륨 (디스크/파티션)
• VG: 볼륨 그룹 (PV의 집합)
• LV: 논리적 볼륨 (실제 사용 공간)
• PE/LE: 물리/논리 익스텐트 (기본 4MB)
• 타입: linear, striped, mirror, thin, raid
• Thin: 씬 프로비저닝, 오버커밋 가능
• 스냅샷: COW 방식, 특정 시점 보존
• 생성: pvcreate, vgcreate, lvcreate
• 확인: pvs, vgs, lvs
• 확장: lvextend -r (파일시스템 함께)
• 축소: lvreduce (주의 필요)
• 경로: /dev/vg0/lv0, /dev/mapper/vg0-lv0
```
