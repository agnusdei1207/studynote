# 561. CAN 버스 (Controller Area Network)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 차량용 다중 마스터 직렬 버스
> 2. **가치**: 신뢰성, 실시간, 오류 감지
> 3. **융합**: 자동차, 산업 자동화, 임베디드와 연관

---

## Ⅰ. 개요

### 개념 정의
**CAN 버스(Controller Area Network)**는 **차량 및 산업용 다중 마스터 직렬 통신 버스**입니다.

### 💡 비유: 자동차 신경망
CAN 버스는 **자동차의 신경망**과 같습니다. 모든 부품이 서로 통신합니다.

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                CAN 버스 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CAN 프레임 구조】                                                 │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  [SOF] [ID] [RTR] [IDE] [r0] [DLC] [DATA] [CRC] [ACK] [EOF]   │ │   │
│  │                                                             │ │   │
│  │  필드             비트              설명                        │ │   │
│  │  ────             ────              ────                        │ │   │
│  │  SOF             1                 Start of Frame             │ │   │
│  │  ID              11/29             식별자 (우선순위)             │ │   │
│  │  RTR             1                 Remote Transmission        │ │   │
│  │  IDE             1                 Identifier Extension       │ │   │
│  │  DLC             4                 Data Length Code (0-8)     │ │   │
│  │  DATA            0-64              데이터 (0-8 바이트)           │ │   │
│  │  CRC             15                순환 중복 검사               │ │   │
│  │  ACK             2                 Acknowledge                │ │   │
│  │  EOF             7                 End of Frame               │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【우선순위 (Arbitration)】                                          │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  • 낮은 ID가 높은 우선순위                                      │ │   │
│  │  • 비트 단위 중재 (Bit-wise Arbitration)                       │ │   │
│  │  • Dominant(0)이 Recessive(1)을 덮어씀                         │ │   │
│  │                                                             │ │   │
│  │  ID 0x100 (0001 0000 0000) → 높은 우선순위                      │ │   │
│  │  ID 0x200 (0010 0000 0000) → 낮은 우선순위                      │ │   │
│  │                                                             │ │   │
│  │  충돌 시 낮은 ID를 가진 메시지만 전송 계속                        │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【오류 감지】                                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  오류 타입         설명                                         │ │   │
│  │  ────────           ────                                         │ │   │
│  │  Bit Error         송신 비트 ≠ 수신 비트                         │ │   │
│  │  Stuff Error       6비트 연속 동일값                            │ │   │
│  │  CRC Error         CRC 불일치                                   │ │   │
│  │  Form Error        고정 필드 위반                               │ │   │
│  │  ACK Error         ACK 수신 없음                                │ │   │
│  │                                                             │ │   │
│  │  오류 프레임 전송 → 모든 노드 재전송                             │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux CAN (SocketCAN)】                                          │
│  ──────────────────                                                │
│  // CAN 인터페이스 확인                                               │
│  $ ip link show can0                                               │
│  3: can0: <NOARP,ECHO> mtu 16 qdisc noop state DOWN mode DEFAULT   │
│                                                                     │
│  // CAN 인터페이스 설정                                               │
│  $ sudo ip link set can0 type can bitrate 500000                   │
│  $ sudo ip link set can0 up                                        │
│                                                                     │
│  // 상태 확인                                                        │
│  $ cat /proc/net/can                                               │
│  can0: 0 transmitted, 0 received, 0 errors                         │
│                                                                     │
│  【cansend/candump】                                                 │
│  ──────────────────                                                │
│  // CAN 메시지 전송                                                   │
│  $ cansend can0 123#1122334455667788    // ID 0x123, 8바이트 데이터  │
│  $ cansend can0 123#R                  // Remote frame             │
│                                                                     │
│  // CAN 메시지 수신                                                   │
│  $ candump can0                                                     │
│  can0  123  [8]  11 22 33 44 55 66 77 88                           │
│  can0  456  [4]  AA BB CC DD                                        │
│                                                                     │
│  // 필터링                                                           │
│  $ candump can0,123:7FF   // ID 0x123만 수신                        │
│                                                                     │
│  // 로그 저장                                                        │
│  $ candump -l can0           // candump-2024-01-15_123456.log      │
│                                                                     │
│  【Python CAN (python-can)】                                        │
│  ──────────────────                                                │
│  import can                                                         │
│                                                                     │
│  # 인터페이스 열기                                                    │
│  bus = can.interface.Bus(channel='can0', bustype='socketcan')      │
│                                                                     │
│  # 메시지 생성                                                        │
│  msg = can.Message(                                                 │
│      arbitration_id=0x123,                                         │
│      data=[0x11, 0x22, 0x33, 0x44],                                │
│      is_extended_id=False                                          │
│  )                                                                  │
│                                                                     │
│  # 전송                                                             │
│  bus.send(msg)                                                      │
│                                                                     │
│  # 수신                                                             │
│  while True:                                                        │
│      msg = bus.recv()                                              │
│      print(f"ID: {msg.arbitration_id:X}, Data: {msg.data.hex()}")  │
│                                                                     │
│  bus.shutdown()                                                     │
│                                                                     │
│  【CAN DB (DBC 파일)】                                               │
│  ──────────────────                                                │
│  // DBC 파일 예시                                                    │
│  VERSION ""                                                         │
│                                                                     │
│  NS_ :                                                              │
│    BA_DEF_                                                          │
│    BA_DEF_DEF_                                                      │
│    BO_                                                              │
│    SG_                                                              │
│                                                                     │
│  BO_ 100 MSG1: 8 Vector__XXX                                        │
│   SG_ Signal1 : 0|8@1+ (1,0) [0|255] "Unit" Vector__XXX             │
│   SG_ Signal2 : 8|16@1- (0.1,-40) [-40|125] "°C" Vector__XXX        │
│                                                                     │
│  // cantools로 파싱                                                  │
│  $ pip install cantools                                            │
│  import cantools                                                    │
│  db = cantools.database.load_file('database.dbc')                  │
│  msg = db.get_message_by_name('MSG1')                              │
│  data = msg.decode(bytes([0x64, 0x02, 0x00, 0x00]))                │
│  print(data)  # {'Signal1': 100, 'Signal2': 10.0}                  │
│                                                                     │
│  【CAN 유틸리티】                                                     │
│  ──────────────────                                                │
│  // CAN 버스 통계                                                    │
│  $ canbusload can0@500000                                          │
│                                                                     │
│  // CAN 시퀀스 재생                                                   │
│  $ canplayer -I candump-2024-01-15.log                             │
│                                                                     │
│  // CAN 스니핑                                                       │
│  $ cansniffer can0                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 차량용 다중 마스터 직렬 버스
• 물리: CANH/CANL 차동 신호 (2선)
• 속도: 1 Mbps (HS-CAN), 125 kbps (LS-CAN)
• 프레임: SOF, ID, DATA, CRC, ACK, EOF
• ID: 11비트 (Base), 29비트 (Extended)
• 우선순위: 낮은 ID가 높음
• 오류 감지: Bit, Stuff, CRC, Form, ACK
• 종단: 120Ω 저항
• Linux: SocketCAN, cansend, candump
• 용도: 자동차, 산업 자동화, 로봇
```
