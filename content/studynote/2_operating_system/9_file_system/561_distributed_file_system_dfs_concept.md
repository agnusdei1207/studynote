## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 파일 시스템(DFS, Distributed File System)은 네트워크로 연결된 여러 서버에 데이터를 분산 저장하되, 사용자에게는 단일한 로컬 파일 시스템처럼 보이게 하는 서비스 모델이다.
> 2. **가치**: 데이터의 물리적 위치에 관계없이 일관된 인터페이스를 제공하며, 대규모 확장성(Scalability)과 높은 가용성(Availability)을 확보하는 데 목적이 있다.
> 3. **융합**: 현대 클라우드 컴퓨팅과 빅데이터 분석의 근간이 되며, 네트워크 지연(Latency)과 일관성(Consistency) 문제를 해결하기 위한 복잡한 분산 프로토콜이 적용된다.

---

## Ⅰ. DFS의 정의와 기본 철학 (Definition & Philosophy)

- **정의**: **분산 파일 시스템(DFS, Distributed File System)**은 물리적으로 떨어진 여러 컴퓨터의 저장 장치를 논리적으로 통합하여 공유하는 시스템이다. 클라이언트는 원격 서버에 저장된 파일에 접근할 때 로컬 디스크의 파일에 접근하는 것과 동일한 방식으로 연산을 수행한다.
- **핵심 철학**: "위치 투명성(Location Transparency)"을 통해 사용자가 데이터가 어느 노드에 있는지 알 필요가 없게 하며, 여러 노드 간의 데이터 동기화를 자동화한다.
- **주요 차이점**: 로컬 파일 시스템(LFS, Local File System)이 단일 머신의 자원을 관리한다면, DFS는 네트워크 지연, 부분적 장애(Partial Failure), 동시성 제어 등을 관리해야 한다.

📢 **섹션 요약 비유**: DFS는 "전 세계 지점들이 연결된 프랜차이즈 도서관"과 같다. 내가 어느 지점에 있든 도서관 전산망을 통해 책을 빌리고 반납할 수 있으며, 실제 책이 어느 서고에 있는지는 사서(DFS 서비스)가 관리한다.

---

## Ⅱ. DFS 시스템 아키텍처 (Architecture & Components)

DFS는 전형적으로 클라이언트-서버 구조 또는 피어 투 피어(P2P) 구조를 취하며, 메타데이터 관리 방식에 따라 중앙 집중형과 분산형으로 나뉜다.

### 1. 계층적 구조 및 논리도 (ASCII Diagram)

```text
      [ Client Side ]                 [ Network ]                 [ Server Side ]
 ┌────────────────────────┐      ┌──────────────────┐      ┌────────────────────────┐
 │ User Application       │      │                  │      │ File Server (Node A)   │
 ├────────────────────────┤      │   RPC / HTTP     │      ├────────────────────────┤
 │ VFS (Virtual FS) Interface    │◀─────────────────▶│      │ File Storage Service   │
 ├────────────────────────┤      │                  │      ├────────────────────────┤
 │ DFS Client Stub (Cache)│      │   Data Stream    │      │ Local File System      │
 └────────────────────────┘      └──────────────────┘      └────────────────────────┘
             ▲                                                         ▲
             │                                                         │
             └─────────────────────────────────────────────────────────┘
                       Network-wide File Namespace (Shared)
```

### 2. 주요 구성 요소
- **클라이언트 (Client)**: 파일 접근 요청을 보내고 로컬 캐시를 관리한다.
- **파일 서버 (File Server)**: 실제 데이터를 저장하고 동시성 제어 및 보안을 담당한다.
- **네임 서비스 (Name Service)**: 파일 이름과 실제 물리적 위치(주소) 간의 매핑을 관리한다.

📢 **섹션 요약 비유**: 아키텍처는 "주방(서버)과 홀(클라이언트)을 연결하는 주문 시스템"과 같다. 손님은 메뉴판(네임스페이스)만 보고 주문하면, 시스템이 실제 식재료(데이터)가 있는 냉장고(스토리지)를 찾아 요리를 배달한다.

## Ⅲ. DFS의 주요 설계 목표 및 투명성 (Design Goals & Transparencies)

DFS의 성공 여부는 사용자에게 분산 환경의 복잡성을 얼마나 잘 숨기는가에 달려 있다.

- **위치 투명성 (Location Transparency)**: 파일 이름에 서버의 위치가 포함되지 않으며, 파일 이동 시에도 이름이 변하지 않는다.
- **성능 및 확장성 (Performance & Scalability)**: 사용자 수가 늘어나도 시스템 성능이 급격히 저하되지 않아야 하며, 노드 추가가 용이해야 한다.
- **장애 허용 (Fault Tolerance)**: 특정 서버가 다운되어도 복제본(Replication)을 통해 서비스를 지속할 수 있어야 한다.
- **일관성 (Consistency)**: 여러 클라이언트가 동시에 파일을 수정할 때 데이터의 무결성을 보장해야 한다.

📢 **섹션 요약 비유**: 투명성은 "전화번호부"와 같다. 상대방이 이사를 가거나 통신사를 바꿔도 전화번호(파일명)만 알면 연결되는 것처럼, 사용자는 시스템 내부의 변화를 몰라도 된다.

---

## Ⅳ. DFS의 운영 메커니즘: 캐싱과 복제 (Caching & Replication)

네트워크 오버헤드를 줄이고 가용성을 높이기 위한 두 가지 핵심 기법이다.

- **클라이언트 사이드 캐싱 (Client-side Caching)**: 자주 사용하는 파일 데이터를 클라이언트 메모리나 디스크에 임시 저장하여 서버 접근 횟수를 줄인다. 일관성 유지가 최대 과제다.
- **데이터 복제 (Replication)**: 동일한 데이터를 여러 서버에 중복 저장한다.
    - **가용성 증대**: 한 대가 고장 나도 다른 노드에서 읽기 가능.
    - **부하 분산**: 여러 노드에서 동시에 데이터를 읽어 속도 향상.
- **동기화 프로토콜**: 수정 사항을 어떻게 전파할지(Write-through, Write-back) 결정한다.

📢 **섹션 요약 비유**: 캐싱은 "자주 쓰는 단어를 적어둔 메모지"이고, 복제는 "중요 문서를 여러 개 복사해서 다른 금고에 보관하는 것"이다. 메모지는 편리하지만 원본과 다를 수 있고, 복사본은 안전하지만 관리가 힘들다.

---

## Ⅴ. DFS의 보안 및 보안 위협 (Security & Threats)

분산 환경은 신뢰할 수 없는 네트워크를 통과하므로 로컬 시스템보다 강력한 보안이 필요하다.

- **인증 (Authentication)**: 클라이언트와 서버가 서로 신뢰할 수 있는지 확인 (예: Kerberos).
- **인가 (Authorization)**: 특정 사용자가 파일에 대한 접근 권한(Read/Write)이 있는지 검증.
- **데이터 암호화 (Encryption)**: 네트워크 전송 중 데이터 탈취를 방지하기 위해 SSL/TLS 등을 사용한다.
- **무결성 검사 (Integrity Check)**: 데이터가 전송 중 변조되지 않았는지 해시 함수 등을 통해 확인한다.

📢 **섹션 요약 비유**: 보안은 "국경 검문소"와 같다. 외부인이 들어올 때 신분증(인증)을 검사하고, 가방 안의 위험물(변조 데이터)을 수색하며, 허가된 구역(인가)에만 들어갈 수 있게 통제한다.

> "분산 파일 시스템은 **'마법의 그림판'**이야. 네 방에도 그림판이 있고, 친구 방에도 그림판이 있는데, 사실은 큰 도화지 하나를 나눠 쓰는 거야. 네가 내 방에서 그림을 그려도 친구 방에 있는 그림판에 똑같이 나타나지. 도화지가 어디에 있는지 몰라도, 친구랑 멀리 떨어져 있어도 같이 그림을 그릴 수 있게 해주는 아주 똑똑한 마법이란다!"
