# 258. 락 스트라이핑 (Lock Striping)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 분할하여 각기 다른 락으로 보호
> 2. **가치**: 핫스팟처럼 높은 동시성 확보
> 3. **융합**: 락 세분성, 파티셔닝, ConcurrentHashMap과 연관

---

## Ⅰ. 개요

### 개념 정의

락 스트라이핑(Lock Striping)은 **데이터를 여러 파티션(스트라이프)으로 분할하고, 각 파티션에 별도의 락을 사용하는 기법**이다. 핫스팟 같은 데이터를 분산시켜 락 경합을 줄인다.

### 💡 비유: 병렬 주차장
락 스트라이핑은 **병렬 주차장**과 같다. 전체 주차장을 막는 대신, 각 구역마다 따로 관리한다. 다른 구역에서 동시에 주차할 수 있다.

### 락 스트라이핑 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 스트라이핑 구조                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  단일 락 (No Striping):                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  전체 데이터                                            │ │ │
│  │  │  ┌─────────────────────────────────────────────────────┐ │ │ │
│  │  │  │                 단일 락                               │ │ │ │
│  │  │  └─────────────────────────────────────────────────────┘ │ │ │
│  │  │                    ↓                                      │ │ │
│  │  │  모든 접근이 직렬 대기                                     │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  락 스트라이핑:                                                │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  스트라이프 0     스트라이프 1     스트라이프 2     ...     │ │ │
│  │  │  ┌───────┐        ┌───────┐        ┌───────┐                        │ │ │
│  │  │  │ Lock 0│        │ Lock 1│        │ Lock 2│                        │ │ │
│  │  │  └───────┘        └───────┘        └───────┘                        │ │ │
│  │  │                    ↓              ↓              ↓                    │ │ │
│  │  │  병렬 접근 가능 (다른 스트라이프 동시 접근)                       │ │ │

│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해시 테이블 스트라이핑】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  버킷 0      버킷 1      버킷 2      버킷 3      버킷 4      ...   │ │
│  │  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐           │ │
│  │  │ Lock 0│  │ Lock 1│  │ Lock 2│  │ Lock 3│  │ Lock 4│           │ │
│  │  └───────┘  └───────┘  └───────┘  └───────┘  └───────┘           │ │
│  │                                                             │ │
│  │  key % N → 버킷 결정                                            │ │
│  │  해시(key) = 17 → 버킷 1 → Lock 1 사용                             │ │
│  │  해시(key) = 42 → 버킷 2 → Lock 2 사용                             │ │
│  │                                                             │ │
│  │  다른 버킷은 동시 접근 가능                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 단일 락】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단일 락:                                                          │ │
│  │  • 구현 단순                                                       │ │
│  │  • 모든 작업 직렬화                                                │ │
│  │  • 낮은 동시성                                                      │ │
│  │  • 낮은 메모리 사용                                                 │ │
│  │                                                             │ │
│  │  락 스트라이핑:                                                    │ │
│  │  • 구현 복잡                                                       │ │
│  │  • 스트라이프 간 병렬화                                              │ │
│  │  • 높은 동시성                                                      │ │
│  │  • 더 많은 메모리 사용 (락 N개)                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```

│  ┌───────────────────────────────────────────────────────────────┐ │

│  │  • 경합 패턴: 핫스팟이면 더 많은 스트라이프                          │ │

│  │                                                             │ │

│  │  • 2의 거듭제곱 (4, 8, 16, 32...)                                  │ │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특정 스트라이프에만 접근 집중                                       │ │
│  │                                                             │ │
│  │  해결:                                                              │ │
│  │  • 더 많은 스트라이프                                                │ │
│  │  • 스트라이프 분할                                                   │ │
│  │  • 일관된 해시 함수                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스레드 안전성】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스트라이프 수가 고정되면:                                           │ │
│  │  • 초기화 시 모든 락 생성 → 스레드 안전                               │ │
│  │  • 지연 초기화 → double-checked locking 필요                        │ │
│  │  • 동적 확장 → 추가 동기화 필요                                       │ │
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Java ConcurrentHashMap (Segment 기반 스트라이핑)】                  │
│  ──────────────────                                                  │
│  // Java 7: 16개 세그먼트, 각각 락                                    │
│  // Java 8+: 각 버킷별 락 (더 세밀)                                   │
│                                                                     │
│  ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>(); │
│                                                                     │
│  // 병렬로 다른 키에 접근 가능                                          │
│  map.put("key1", 1);  // 버킷 1 락                                   │
│  map.put("key2", 2);  // 버킷 2 락 (동시 실행 가능)                    │
│                                                                     │
│  【Go sync.Map (Striped Locking)】                                   │
│  ──────────────────                                                  │
│  var m sync.Map                                                      │
│                                                                     │
│  // 내부적으로 여러 락 사용                                            │
│  m.Store(key1, value1)  // 락 1                                     │
│  m.Store(key2, value2)  // 락 2 (병렬)                               │
│                                                                     │
│  【Go striped mutex 패턴】                                            │
│  ──────────────────                                                  │
│  type StripedMutex struct {                                         │
│      mu    []sync.Mutex                                              │
│      size int                                                        │
│  }                                                                  │
│                                                                     │
│  func NewStripedMutex(size int) *StripedMutex {                     │
│      s := &StripedMutex{                                             │
│          mu:   make([]sync.Mutex, size),                             │
│          size: size,                                                 │
│      }                                                              │
│      return s                                                        │
│  }                                                                  │
│                                                                     │
│  func (s *StripedMutex) Lock(key string) {                          │
│      idx := s.hash(key) % s.size                                    │
│      s.mu[idx].Lock()                                                │
│  }                                                                  │
│                                                                     │
│  func (s *StripedMutex) Unlock(key string) {                        │
│      idx := s.hash(key) % s.size                                    │
│      s.mu[idx].Unlock()                                              │
│  }                                                                  │
│                                                                     │
│  【C++ 스트라이핑 구현】                                               │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│  #include <vector>                                                  │
│  #include <functional>                                              │
│                                                                     │
│  template<typename Key, size_t N>                                   │
│  class StripedLock {                                                │
│      std::array<std::mutex, N> locks_;                              │
│      std::hash<Key> hasher_;                                        │
│  public:                                                            │
│      std::mutex& get_lock(const Key& key) {                          │
│          size_t idx = hasher_(key) % N;                             │
│          return locks_[idx];                                         │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  // 사용                                                              │
│  StripedLock<std::string, 16> striped_lock;                         │
│                                                                     │
│  void safe_access(const std::string& key) {                         │
│      std::lock_guard<std::mutex> lock(striped_lock.get_lock(key));  │
│      // ... 작업 ...                                                  │
│  }                                                                  │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구조              단일 락    16개 스트라이프    256개 스트라이프        │ │
│  │  ────              ──────        ──────────         ───────────         │ │
│  │  4 스레드          1x           4x                4x                 │ │
│  │  16 스레드         1x           4x                16x                │ │
│  │  64 스레드         1x           4x                16x                │ │
│  │                                                             │ │
│  │  (처리량, 1x = 기준 성능)                                          │ │
│  │  스트라이프 수 = CPU 코어 수 × 2~4 정도가 최적                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
