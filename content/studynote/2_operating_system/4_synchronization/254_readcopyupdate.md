# 254. 비관적 락 (Pessimistic Locking)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 충돌을 가정하고 미리 락 획득
> 2. **가치**: 데이터 무결성 보장, 구현 단순
> 3. **융합**: 뮤텍스, 세마포어, DB 락과 연관

---

## Ⅰ. 개요

### 개념 정의

비관적 락(Pessimistic Locking)은 **충돌이 발생할 것이라고 가정하고 데이터에 접근하기 전에 미리 락을 획득하는 동시성 제어 패턴**이다. 락을 가진 동안 다른 스레드는 접근할 수 없다.

### 💡 비유: 화장실 열쇠
비관적 락은 **화장실 열쇠**와 같다. 들어가기 전에 문을 잠근다. 안에서 작업하는 동안 아무도 들어올 수 없다. 안전하지만 기다릴 수 있다.

### 비관적 락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                비관적 락 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 패턴】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  // 락 획득                                                        │ │
│  │  lock();                                                            │ │
│  │                                                             │ │
│  │  // 데이터 읽기                                                     │ │
│  │  data = read();                                                     │ │
│  │                                                             │ │
│  │  // 데이터 수정                                                     │ │
│  │  new_data = modify(data);                                          │ │
│  │                                                             │ │
│  │  // 데이터 쓰기                                                     │ │
│  │  write(new_data);                                                   │ │
│  │                                                             │ │
│  │  // 락 해제                                                        │ │
│  │  unlock();                                                          │ │
│  │                                                             │ │
│  │  특징:                                                             │ │
│  │  • 전체 과정 동안 락 보유                                            │ │
│  │  • 다른 스레드 접근 차단                                              │ │
│  │  • 데이터 무결성 보장                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 낙관적 락】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  비관적 락:                                                           │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  lock() ── [락 보유] ── unlock()                         │ │ │
│  │  │         ↑              ↑                                  │ │ │
│  │  │    데이터 접근    데이터 수정                                 │ │ │
│  │  │                                                         │ │ │
│  │  │  장점: 충돌 없음, 일관성 보장                                 │ │ │
│  │  │  단점: 동시성 낮음, 대기 발생                                 │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  낙관적 락:                                                           │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  read() ── modify() ── [검사] ── write() 또는 retry       │ │ │
│  │  │         (락 없이)    (락 없이)   (커밋 시점)                   │ │ │
│  │  │                                                         │ │ │
│  │  │  장점: 락 오버헤드 없음, 높은 동시성                            │ │ │
│  │  │  단점: 충돌 시 롤백 비용, 기아 상태 가능                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락 종류】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 공유 락 (Shared Lock / Read Lock):                             │ │
│  │     • 읽기 전용                                                     │ │
│  │     • 여러 스레드 동시 획득 가능                                    │ │
│  │     • 쓰기 차단                                                    │ │
│  │                                                             │ │
│  │  2. 배타 락 (Exclusive Lock / Write Lock):                          │ │
│  │     • 읽기/쓰기 모두 차단                                            │ │
│  │     • 단일 스레드만 획득 가능                                        │ │
│  │                                                             │ │
│  │  3. 의도 락 (Intention Lock):                                      │ │
│  │     • IS: 의도 공유 (하위 레벨 공유 락 의도)                          │ │
│  │     • IX: 의도 배타 (하위 레벨 배타 락 의도)                          │ │
│  │     • SIX: 공유 + 의도 배타                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  │  // 테이블 레벨 락                                                    │ │
│  │  LOCK TABLE table IN EXCLUSIVE MODE;                              │ │

│  │  4. 교착상태 예방 가능                                                │ │

│                                                                     │
│  【단점】                                                              │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 동시성 저하                                                      │ │
│  │  2. 락 오버헤드                                                       │ │
│  │  3. 교착상태 가능성                                                   │ │

│  │  부적합:                                                              │ │
│  │  • 낮은 충돌률                                                        │ │

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ std::mutex】                                                   │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│  #include <shared_mutex>                                            │
│                                                                     │
│  std::mutex mtx;                                                     │
│  std::shared_mutex shared_mtx;  // C++17                             │
│                                                                     │
│  // 배타 락                                                            │
│  void exclusive_access() {                                          │
│      std::lock_guard<std::mutex> lock(mtx);                         │
│      // ... 임계 영역 ...                                             │
│  }                                                                  │
│                                                                     │
│  // 공유 락 (읽기)                                                    │
│  void shared_access() {                                             │
│      std::shared_lock<std::shared_mutex> lock(shared_mtx);          │
│      // ... 읽기 전용 ...                                              │
│  }                                                                  │
│                                                                     │
│  // 배타 락 (쓰기)                                                    │
│  void exclusive_access() {                                          │
│      std::unique_lock<std::shared_mutex> lock(shared_mtx);          │
│      // ... 쓰기 ...                                                  │
│  }                                                                  │
│                                                                     │
│  【Java synchronized】                                               │
│  ──────────────────                                                  │
│  // 메서드 레벨                                                        │
│  public synchronized void update() {                                │
│      // 배타 락 자동 획득                                              │
│  }                                                                  │
│                                                                     │
│  // 블록 레벨                                                          │
│  public void update() {                                             │
│      synchronized(this) {                                           │
│          // 배타 락                                                   │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  // ReentrantReadWriteLock                                          │
│  ReentrantReadWriteLock rwLock = new ReentrantReadWriteLock();     │
│                                                                     │
│  void read() {                                                      │
│      rwLock.readLock().lock();  // 공유 락                             │
│      // ... 읽기 ...                                                  │
│      rwLock.readLock().unlock();                                    │
│  }                                                                  │
│                                                                     │
│  void write() {                                                     │
│      rwLock.writeLock().lock();  // 배타 락                            │
│      // ... 쓰기 ...                                                  │
│      rwLock.writeLock().unlock();                                   │
│  }                                                                  │
│                                                                     │
│  【SQL (SELECT FOR UPDATE)】                                         │
│  ──────────────────                                                  │
│  BEGIN;                                                              │
│  -- 행 레벨 배타 락                                                   │
│  SELECT balance FROM accounts WHERE id = 1 FOR UPDATE;              │
│  UPDATE accounts SET balance = balance - 100 WHERE id = 1;          │
│  COMMIT;                                                             │
│                                                                     │
│  【Python threading.Lock】                                           │
│  ──────────────────                                                  │
│  import threading                                                   │
│                                                                     │
│  lock = threading.Lock()                                            │
│                                                                     │
│  def update():                                                      │
│      with lock:  # 비관적 락                                          │
│          # ... 임계 영역 ...                                          │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시나리오              비관적 락    낙관적 락    권장                  │ │
│  │  ────────              ────────     ────────     ────                │ │
│  │  높은 충돌률 (50%+)     O            X            비관적              │ │
│  │  중간 충돌률 (10-50%)  O            O            상황별              │ │
│  │  낮은 충돌률 (<10%)    X            O            낙관적              │ │
│  │  긴 트랜잭션            O            X            비관적              │ │
│  │  짧은 트랜잭션          X            O            낙관적              │ │
│  │  읽기 위주              X            O            낙관적              │ │
│  │  쓰기 위주              O            X            비관적              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 충돌 가정하고 미리 락 획득
• 특징: 전체 과정 동안 락 보유
• 종류: 공유 락, 배타 락, 의도 락
• 장점: 일관성 보장, 구현 단순
• 단점: 동시성 저하, 대기 발생
• 적합: 높은 충돌률, 긴 트랜잭션
```
