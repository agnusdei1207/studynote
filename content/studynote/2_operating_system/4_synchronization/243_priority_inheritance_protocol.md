# 243. 생산자-소비자 (Producer-Consumer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 생성하는 생산자와 소비하는 소비자의 협력 패턴
> 2. **가치**: 비동기 처리, 버퍼링, 파이프라인의 기반
> 3. **융합**: 세마포어, 조건 변수, 큐와 연관

---

## Ⅰ. 개요

### 개념 정의

생산자-소비자(Producer-Consumer) 패턴은 **데이터를 생성하는 생산자가 버퍼에 넣고, 소비자가 버퍼에서 가져가는 동시성 디자인 패턴**이다. 버퍼 크기 제한과 동기화가 핵심이다.

### 💡 비유: 식당 주방
생산자-소비자는 **식당 주방**과 같다. 요리사(생산자)가 요리를 만들어 내어놓으면, 서빙(소비자)이 가져간다. 창구(버퍼)가 꽉 차면 요리사는 기다리고, 비면 서빙은 기다린다.

### 생산자-소비자 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                생산자-소비자 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  생산자들                버퍼 (N 슬롯)              소비자들        │ │
│  │  ────────                ──────────              ────────       │ │
│  │                                                             │ │
│  │  P1 ──┐               ┌───┬───┬───┬───┬───┐        ┌── C1    │ │
│  │       │               │   │   │   │   │   │        │         │ │
│  │  P2 ──┼─── put() ───►│ 1 │ 2 │ 3 │ 4 │ 5 │── get()┼── C2    │ │
│  │       │               │   │   │   │   │   │        │         │ │
│  │  P3 ──┘               └───┴───┴───┴───┴───┘        └── C3    │ │
│  │                              ▲                                │ │
│  │                              │                                 │ │
│  │                         [버퍼 크기 = 5]                         │ │
│  │                                                             │ │
│  │  제약:                                                            │ │
│  │  • 버퍼 가득 차면 생산자 대기                                      │ │
│  │  • 버퍼 비면 소비자 대기                                           │ │
│  │  • 버퍼 접근은 상호 배제                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【세마포어 해법】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  semaphore empty = N;    // 빈 슬롯 수                           │ │
│  │  semaphore full = 0;     // 찬 슬롯 수                           │ │
│  │  semaphore mutex = 1;    // 버퍼 접근 보호                        │ │
│  │                                                             │ │
│  │  생산자:                    소비자:                               │ │
│  │  ─────────────────────────────────────────────────────            │ │
│  │  while (true) {            while (true) {                        │ │
│  │      item = produce();         item = NULL;                      │ │
│  │      P(empty);                 P(full);                          │ │
│  │      P(mutex);                 P(mutex);                          │ │
│  │      buffer[in] = item;        item = buffer[out];               │ │
│  │      in = (in + 1) % N;        out = (out + 1) % N;              │ │
│  │      V(mutex);                 V(mutex);                          │ │
│  │      V(full);                  V(empty);                          │ │
│  │      // produce next           // consume item                   │ │
│  │  }                            }                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【조건 변수 해법】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  mutex lock;                                                      │ │
│  │  condition not_full, not_empty;                                  │ │
│  │  int count = 0;                                                   │ │
│  │                                                             │ │
│  │  생산자:                    소비자:                               │ │
│  │  lock();                    lock();                               │ │
│  │  while (count == N)         while (count == 0)                   │ │
│  │      wait(not_full);            wait(not_empty);                  │ │
│  │  buffer[in] = item;         item = buffer[out];                  │ │
│  │  in = (in + 1) % N;         out = (out + 1) % N;                 │ │
│  │  count++;                    count--;                             │ │
│  │  signal(not_empty);          signal(not_full);                    │ │
│  │  unlock();                  unlock();                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                생산자-소비자 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【변형: 무제한 버퍼】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 버퍼 크기 무제한                                                │ │
│  │  • 생산자는 항상 진입 가능                                         │ │
│  │  • 소비자만 대기                                                   │ │
│  │  • 메모리 고갈 위험                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【변형: 단일 슬롯】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 버퍼 크기 = 1                                                   │ │
│  │  • 생산-소비 교대                                                  │ │
│  │  • 단순하지만 처리량 낮음                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【활용 사례】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 로그 시스템: 로그 생성 → 버퍼 → 파일 작성                      │ │
│  │  • 네트워크: 패킷 수신 → 버퍼 → 처리                               │ │
│  │  • 이벤트 시스템: 이벤트 발생 → 큐 → 핸들러                        │ │
│  │  • 스레드 풀: 작업 제출 → 큐 → 워커                               │ │
│  │  • 파이프라인: 단계 간 데이터 전달                                │ │
│  │  • 스트림 처리: 데이터 스트림 → 버퍼 → 처리                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 고려사항】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 버퍼 크기: 너무 작으면 대기, 너무 크면 메모리 낭비               │ │
│  │  • 생산/소비 속도 밸런스                                           │ │
│  │  • 다중 생산자/소비자 시 락 경합                                   │ │
│  │  • 락 프리 큐로 최적화 가능                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ std::condition_variable】                                       │
│  ──────────────────                                                  │
│  #include <queue>                                                    │
│  #include <mutex>                                                    │
│  #include <condition_variable>                                       │
│                                                                     │
│  template<typename T>                                               │
│  class BoundedQueue {                                               │
│      std::queue<T> queue;                                           │
│      std::mutex mtx;                                                │
│      std::condition_variable not_full, not_empty;                   │
│      size_t capacity;                                               │
│  public:                                                            │
│      BoundedQueue(size_t cap) : capacity(cap) {}                    │
│                                                                     │
│      void put(T item) {                                             │
│          std::unique_lock<std::mutex> lock(mtx);                    │
│          not_full.wait(lock, [this]{ return queue.size() < capacity; });
│          queue.push(item);                                          │
│          not_empty.notify_one();                                    │
│      }                                                              │
│                                                                     │
│      T take() {                                                     │
│          std::unique_lock<std::mutex> lock(mtx);                    │
│          not_empty.wait(lock, [this]{ return !queue.empty(); });    │
│          T item = queue.front();                                    │
│          queue.pop();                                               │
│          not_full.notify_one();                                     │
│          return item;                                               │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【Java BlockingQueue】                                                │
│  ──────────────────                                                  │
│  import java.util.concurrent.*;                                     │
│                                                                     │
│  BlockingQueue<String> queue = new ArrayBlockingQueue<>(100);       │
│                                                                     │
│  // 생산자                                                           │
│  void produce(String item) throws InterruptedException {            │
│      queue.put(item);  // 가득 차면 대기                             │
│  }                                                                  │
│                                                                     │
│  // 소비자                                                           │
│  String consume() throws InterruptedException {                      │
│      return queue.take();  // 비면 대기                              │
│  }                                                                  │
│                                                                     │
│  【Python queue.Queue】                                                │
│  ──────────────────                                                  │
│  import queue                                                        │
│  import threading                                                   │
│                                                                     │
│  q = queue.Queue(maxsize=10)                                        │
│                                                                     │
│  def producer():                                                    │
│      for i in range(100):                                           │
│          q.put(i)  # 가득 차면 대기                                   │
│                                                                     │
│  def consumer():                                                    │
│      while True:                                                    │
│          item = q.get()  # 비면 대기                                  │
│          # 처리                                                       │
│          q.task_done()                                               │
│                                                                     │
│  【Go channel】                                                        │
│  ──────────────────                                                  │
│  ch := make(chan int, 10)  // 버퍼 채널                              │
│                                                                     │
│  // 생산자                                                           │
│  go func() {                                                         │
│      for i := 0; ; i++ {                                            │
│          ch <- i  // 가득 차면 대기                                   │
│      }                                                               │
│  }()                                                                 │
│                                                                     │
│  // 소비자                                                           │
│  for item := range ch {  // 비면 대기                                │
│      // 처리                                                          │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 생산자 생성 → 버퍼 → 소비자 소비
• 제약: 버퍼 가득 참/비면 대기
• 세마포어: empty, full, mutex
• 조건 변수: not_full, not_empty
• 활용: 로그, 이벤트, 스레드 풀
• 구현: BlockingQueue, Channel
```
