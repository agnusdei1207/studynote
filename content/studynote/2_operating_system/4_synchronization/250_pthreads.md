# 250. 락 엘리션 (Lock Elision)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락을 획득하지 않고 임계 영역 실행
> 2. **가치**: 하드웨어 트랜잭션으로 락 오버헤드 제거
> 3. **융합**: TSX, 하드웨어 트랜잭셔널 메모리, 낙관적 락과 연관

---

## Ⅰ. 개요

### 개념 정의

락 엘리션(Lock Elision)은 **락을 실제로 획득하지 않고 하드웨어 트랜잭션으로 임계 영역을 실행하는 최적화 기법**이다. 충돌 없으면 락 오버헤드 없이 실행된다.

### 💡 비유: 예약 없이 식당
락 엘리션은 **예약 없이 식당 가기**와 같다. 자리가 있으면 바로 들어간다. 자리가 없으면 나와서 예약(락)하고 들어간다. 보통은 자리가 있다.

### 락 엘리션 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 엘리션 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 아이디어】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  일반적인 락:                                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  lock();                    // 락 획득 오버헤드                    │ │ │
│  │  │  // 임계 영역                                                 │ │ │
│  │  │  unlock();                  // 락 해제 오버헤드                    │ │ │
│  │  │                                                         │ │ │
│  │  │  문제: 충돌 없어도 매번 락 오버헤드                              │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  락 엘리션 (TSX):                                                    │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  xbegin();                 // 트랜잭션 시작                        │ │ │
│  │  │  if (success) {                                               │ │ │
│  │  │      // 임계 영역 실행 (락 없음!)                                 │ │ │
│  │  │      xend();             // 트랜잭션 커밋                          │ │ │
│  │  │  } else {                                                      │ │ │
│  │  │      // 충돌! 폴백                                               │ │ │
│  │  │      lock();            // 전통적 락                               │ │ │
│  │  │      // 임계 영역                                                 │ │ │
│  │  │      unlock();                                                 │ │ │
│  │  │  }                                                              │ │ │
│  │  │                                                         │ │ │
│  │  │  장점: 충돌 없으면 락 오버헤드 0                                  │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Intel TSX (Transactional Synchronization Extensions)】            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RTM (Restricted Transactional Memory):                           │ │
│  │  • xbegin: 트랜잭션 시작                                            │ │
│  │  • xend: 트랜잭션 커밋                                               │ │
│  │  • xabort: 트랜잭션 중단                                             │ │
│  │  • xtest: 트랜잭션 중인지 확인                                       │ │
│  │                                                             │ │
│  │  HLE (Hardware Lock Elision):                                     │ │
│  │  • XACQUIRE: 락 획득 (엘리션)                                       │ │
│  │  • XRELEASE: 락 해제 (엘리션)                                       │ │
│  │  • 기존 코드에 프리픽스만 추가                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 원리】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 메모리 읽기/쓰기를 캐시에 기록 (Read/Write Set)                │ │
│  │  2. 다른 코어의 쓰기 충돌 감지                                      │ │
│  │  3. 충돌 없으면 원자적 커밋                                          │ │
│  │  4. 충돌 있으면 트랜잭션 중단 → 폴백                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 엘리션 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【HLE (Hardware Lock Elision)】                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 기존 코드                                                      │ │
│  │  lock();                                                            │ │
│  │  critical_section();                                                │ │
│  │  unlock();                                                          │ │
│  │                                                             │ │
│  │  // HLE 적용 (어셈블리)                                              │ │
│  │  XACQUIRE lock;      // 락을 쓰지만 엘리션 시도                       │ │
│  │  critical_section();                                                │ │
│  │  XRELEASE unlock;    // 락을 해제하지만 엘리션 중이면 무시               │ │
│  │                                                             │ │
│  │  장점: 코드 수정 최소화                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RTM (Restricted Transactional Memory)】                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  unsigned status = xbegin();                                       │ │
│  │  if (status == _XBEGIN_STARTED) {                                  │ │
│  │      // 트랜잭션 중                                                  │ │
│  │      if (conflict_detected) {                                      │ │
│  │          xabort(_ABORT_CODE);                                      │ │
│  │      }                                                              │ │
│  │      // 임계 영역 실행                                               │ │
│  │      xend();  // 커밋                                                │ │
│  │  } else {                                                           │ │
│  │      // 폴백 경로                                                    │ │
│  │      lock();                                                         │ │
│  │      // 임계 영역                                                    │ │
│  │      unlock();                                                       │ │
│  │  }                                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【제약 사항】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 트랜잭션 버퍼 크기 제한 (L1 캐시 크기)                            │ │
│  │  • 중첩 트랜잭션 제한                                               │ │
│  │  • I/O, 시스템 콜 트랜잭션 중 불가                                  │ │
│  │  • 인터럽트 시 트랜잭션 중단                                         │ │
│  │  • 캐시 라인이 너무 많으면 중단                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【GCC RTM 인라인 함수】                                               │
│  ──────────────────                                                  │
│  #include <immintrin.h>                                              │
│  #include <pthread.h>                                                │
│                                                                     │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│                                                                     │
│  void critical_section_with_elision() {                              │
│      const int MAX_RETRIES = 3;                                      │
│      for (int i = 0; i < MAX_RETRIES; i++) {                        │
│          unsigned status = _xbegin();                                │
│          if (status == _XBEGIN_STARTED) {                            │
│              // 트랜잭션 실행                                          │
│              // ... 임계 영역 ...                                       │
│              _xend();                                                │
│              return;  // 성공                                          │
│          }                                                            │
│          // 재시도 또는 폴백                                           │
│      }                                                                │
│      // 폴백: 전통적 락                                                │
│      pthread_mutex_lock(&mutex);                                     │
│      // ... 임계 영역 ...                                             │
│      pthread_mutex_unlock(&mutex);                                   │
│  }                                                                   │
│                                                                     │
│  【glibc HLE 지원 pthread】                                           │
│  ──────────────────                                                  │
│  // glibc 2.18+에서 자동 HLE 지원                                      │
│  // 환경변수로 제어                                                     │
│  // GLIBC_TUNABLES=glibc.elision.enabled=1                          │
│                                                                     │
│  pthread_mutex_t mutex;                                             │
│  // 내부적으로 HLE 사용 시도                                           │
│  pthread_mutex_lock(&mutex);                                        │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시나리오           일반 락       HLE/RTM      개선율              │ │
│  │  ────────           ──────        ────────      ────                │ │
│  │  무충돌              100ns         20ns         80%                 │ │
│  │  저충돌 (1%)         150ns         50ns         66%                 │ │
│  │  중충돌 (10%)        500ns         400ns        20%                 │ │
│  │  고충돌 (50%)        2000ns        2000ns       0%                  │ │
│  │                                                             │ │
│  │  충돌이 낮을 때 큰 성능 향상                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【언제 사용할까?】                                                    │
│  ──────────────────                                                  │
│  적합:                                                                 │
│  • 읽기 위주 임계 영역                                                 │
│  • 짧은 임계 영역                                                      │
│  • 낮은 충돌률                                                          │
│  • 기존 락 기반 코드                                                    │
│                                                                     │
│  부적합:                                                               │
│  • 높은 충돌률                                                          │
│  • 긴 임계 영역                                                         │
│  • I/O 포함                                                            │
│  • 큰 메모리 접근                                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 락 없이 하드웨어 트랜잭션으로 임계 영역 실행
• Intel TSX: RTM, HLE 두 가지 방식
• HLE: 기존 코드에 프리픽스만 추가
• RTM: 명시적 트랜잭션 제어
• 장점: 무충돌 시 락 오버헤드 제거
• 제약: 캐시 크기, I/O 불가
• 활용: 읽기 위주, 낮은 충돌
```
