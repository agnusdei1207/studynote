# 244. 락 프리 (Lock-Free)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 없이 원자적 연산만으로 동기화
> 2. **가치**: 데드락 없음, 높은 동시성, 진행 보장
> 3. **융합**: CAS, 원자적 연산, ABA 문제와 연관

---

## Ⅰ. 개요

### 개념 정의

락 프리(Lock-Free)는 **락(뮤텍스, 스핀락)을 사용하지 않고 원자적 연산만으로 동시성을 제어하는 프로그래밍 방식**이다. 시스템 전체가 항상 진행됨을 보장한다.

### 💡 비유: 회전문
락 프리는 **회전문**과 같다. 한 사람이 들어가면 다음 사람이 바로 들어갈 수 있다. 문이 닫혀서 기다리는 일이 없다. 누군가는 항상 들어간다.

### 락 프리 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 프리 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락 기반 vs 락 프리】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  락 기반 (Blocking):                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  T1: lock() ────[임계 영역]──── unlock()                 │ │ │
│  │  │  T2:         lock() ────[대기...]──── [임계 영역]──►     │ │ │
│  │  │                    ↑                                    │ │ │
│  │  │                T1이 끝날 때까지 블록                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  락 프리 (Non-blocking):                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  T1: CAS() ────[성공]──── [작업]                          │ │ │
│  │  │  T2: CAS() ────[실패]──── CAS() ────[성공]──── [작업]     │ │ │
│  │  │                    ↑         ↑                           │ │ │
│  │  │              재시도    재시도 성공                          │ │ │
│  │  │                                                         │ │ │
│  │  │  특징: 어떤 스레드라도 항상 진행 가능                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【비블로킹 계층】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  계층              설명                    보장                    │ │
│  │  ────                  ────                        ────            │ │
│  │  Obstruction-Free   충돌 없으면 진행     개별 진행                  │ │
│  │  Lock-Free          어떤 스레드든 진행   시스템 전체 진행           │ │
│  │  Wait-Free          모든 스레드 진행     개별 유한 시간 완료        │ │
│  │                                                             │ │
│  │  강도: Obstruction-Free < Lock-Free < Wait-Free                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                           │ │
│  │  • 데드락 없음                                                    │ │
│  │  • 컨텍스트 스위치 최소화                                          │ │
│  │  • 우선순위 역전 없음                                              │ │
│  │  • 높은 동시성                                                    │ │
│  │  • 비동기 안전 (시그널 핸들러)                                     │ │
│  │                                                             │ │
│  │  단점:                                                           │ │
│  │  • 구현 복잡                                                      │ │
│  │  • ABA 문제                                                       │ │
│  │  • 메모리 관리 어려움                                              │ │
│  │  • 경합 시 높은 CPU 사용                                          │ │
│  │  • 디버깅 어려움                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 프리 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락 프리 카운터】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // CAS 기반 증가                                                 │ │
│  │  void lock_free_inc(atomic_int* counter) {                      │ │
│  │      int old;                                                    │ │
│  │      do {                                                        │ │
│  │          old = atomic_load(counter);                             │ │
│  │      } while (!atomic_compare_exchange_weak(counter,              │ │
│  │                                            &old, old + 1));        │ │
│  │  }                                                               │
│  │                                                             │ │
│  │  // 또는 fetch_add 사용                                           │ │
│  │  atomic_fetch_add(counter, 1);  // 더 간단하고 효율적              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락 프리 스택 (Treiber Stack)】                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct Node { T data; Node* next; };                           │ │
│  │  atomic<Node*> head;                                              │ │
│  │                                                             │ │
│  │  void push(T value) {                                            │ │
│  │      Node* n = new Node{value, nullptr};                         │ │
│  │      do {                                                        │ │
│  │          n->next = head.load();                                  │ │
│  │      } while (!head.compare_exchange_weak(n->next, n));          │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  │  bool pop(T& result) {                                           │ │
│  │      Node* old_head;                                              │ │
│  │      do {                                                        │ │
│  │          old_head = head.load();                                 │ │
│  │          if (!old_head) return false;                            │ │
│  │      } while (!head.compare_exchange_weak(old_head,               │ │
│  │                                          old_head->next));         │ │
│  │      result = old_head->data;                                    │ │
│  │      // delete old_head;  // ABA 문제 주의!                       │ │
│  │      return true;                                                 │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Wait-Free vs Lock-Free】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Lock-Free:                                                       │ │
│  │  • 어떤 스레드는 계속 진행                                        │ │
│  │  • 다른 스레드는 재시도할 수 있음                                  │ │
│  │  • CAS 루프                                                       │ │
│  │                                                             │ │
│  │  Wait-Free:                                                       │ │
│  │  • 모든 스레드가 유한 시간 내 완료                                 │ │
│  │  • 재시도 없음                                                    │ │
│  │  • 구현 더 어려움                                                  │ │
│  │  • fetch_add 같은 원자적 연산은 wait-free                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ 락 프리 큐 (MPSC)】                                              │
│  ──────────────────                                                  │
│  template<typename T>                                               │
│  class LockFreeQueue {                                              │
│      struct Node {                                                  │
│          T data;                                                      │
│          std::atomic<Node*> next{nullptr};                          │ │
│      };                                                              │
│      std::atomic<Node*> head;                                        │
│      std::atomic<Node*> tail;                                        │
│  public:                                                            │
│      void enqueue(T value) {                                        │
│          Node* node = new Node{value};                              │
│          Node* prev_tail = tail.exchange(node);                     │
│          prev_tail->next.store(node, std::memory_order_release);    │
│      }                                                              │
│      bool dequeue(T& result) {                                      │
│          Node* h = head.load();                                      │
│          Node* next = h->next.load(std::memory_order_acquire);      │
│          if (!next) return false;                                   │
│          result = std::move(next->data);                            │
│          head.store(next);                                           │
│          delete h;                                                    │
│          return true;                                                 │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【C++ atomic_fetch_add (Wait-Free)】                                   │
│  ──────────────────                                                  │
│  std::atomic<int> counter{0};                                        │
│                                                                     │
│  // Wait-Free: 항상 한 번에 완료                                       │
│  int old = counter.fetch_add(1, std::memory_order_relaxed);         │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  // 락 기반                                                           │
│  pthread_mutex_lock(&mutex);                                        │
│  counter++;                                                          │
│  pthread_mutex_unlock(&mutex);                                      │
│  // 오버헤드: lock/unlock syscall                                     │
│                                                                     │
│  // 락 프리                                                           │
│  atomic_fetch_add(&counter, 1);                                     │
│  // 오버헤드: 단일 원자적 명령어                                        │
│                                                                     │
│  【언제 락 프리를 사용할까?】                                            │
│  ──────────────────                                                  │
│  • 고성능 필요 시                                                     │
│  │  • 실시간 시스템                                                   │
│  │  • 신호 핸들러에서                                                 │
│  • 저지연 요구 시                                                     │
│  │  • 고빈도 카운터                                                   │
│  │  • 높은 경합 예상 시                                               │
│  │                                                             │ │
│  │  언제 락이 더 나을까?                                                │
│  │  • 복잡한 연산                                                      │
│  │  • 구현 단순함 중요                                                 │
│  │  • 디버깅 용이성 중요                                               │
│  │  • 메모리 관리 복잡                                                 │
│  │                                                             │ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 락 없이 원자적 연산만으로 동기화
• 계층: Obstruction-Free < Lock-Free < Wait-Free
• 핵심 연산: CAS, fetch_add
• 장점: 데드락 없음, 높은 동시성
• 단점: 복잡, ABA 문제, 디버깅 어려움
• 활용: 고성능, 실시간, 카운터
```
