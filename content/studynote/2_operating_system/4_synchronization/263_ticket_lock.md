# 267. 티켓 락 (Ticket Lock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 티켓 번호로 순서 보장하는 락
> 2. **가치**: FIFO 순서 보장, 공정성 확보
> 3. **융합**: 스핀락, NUMA, 공정 스케줄링과 연관

---

## Ⅰ. 개요

### 개념 정의

티켓 락(Ticket Lock)은 **각 스레드에 티켓 번호를 부여하고, 자신의 차례가 올 때까지 대기하는 FIFO 순서 보장 락**이다. 기아 상태(starvation)를 방지한다.

### 💡 비유: 대기표 번호표
티켓 락은 **대기표 번호표**와 같다. 번호표를 받는다. "현재 10번" 화면을 본다. 내 번호가 25번이면 10번이 25번이 될 때까지 기다린다. 순서대로 진행된다.

### 티켓 락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                티켓 락 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 구조】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct ticket_lock {                                              │ │
│  │      atomic_uint next_ticket;  // 다음 티켓 번호                    │ │
│  │      atomic_uint serving;      // 현재 서빙 중인 번호                │ │
│  │  };                                                              │ │
│  │                                                             │ │
│  │  lock():                                                          │ │
│  │    my_ticket = next_ticket.fetch_add(1);  // 티켓 받기              │ │
│  │    while (serving.load() != my_ticket) spin();  // 대기             │ │
│  │                                                             │ │
│  │  unlock():                                                        │ │
│  │    serving.fetch_add(1);  // 다음 서빙                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 예시】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  초기 상태: next_ticket = 0, serving = 0                            │ │
│  │                                                             │ │
│  │  T1: lock() → 티켓 0, serving == 0? 진입                           │ │
│  │  T2: lock() → 티켓 1, serving == 0? 대기 (0 != 1)                  │ │
│  │  T3: lock() → 티켓 2, serving == 0? 대기 (0 != 2)                  │ │
│  │  T4: lock() → 티켓 3, serving == 0? 대기 (0 != 3)                  │ │
│  │                                                             │ │
│  │  T1: unlock() → serving = 1                                       │ │
│  │  T2: serving == 1? 진입! (티켓 1 == serving 1)                       │ │
│  │                                                             │ │
│  │  T2: unlock() → serving = 2                                       │ │
│  │  T3: serving == 2? 진입!                                           │ │
│  │                                                             │ │
│  │  순서대로 진입 보장 (FIFO)                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 일반 스핀락】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  일반 스핀락:                                                        │ │
│  │  • 획득 순서 불확실 (CAS 경쟁)                                        │ │
│  │  • 기아 상태 발생 가능                                                │ │
│  │  • 구현 단순                                                        │ │
│  │                                                             │ │
│  │  티켓 락:                                                            │ │
│  │  • FIFO 순서 보장                                                   │ │
│  │  • 기아 상태 방지                                                    │ │
│  │  • 공정성 보장                                                       │ │
│  │  • 약간의 오버헤드                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ 티켓 락 구현】                                                   │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  #include <cstdint>                                                  │
│                                                                     │
│  class TicketLock {                                                 │
│      std::atomic<uint32_t> ticket{0};                               │
│      std::atomic<uint32_t> serving{0};                              │
│                                                                     │
│  public:                                                            │
│      void lock() {                                                  │
│          uint32_t my_ticket = ticket.fetch_add(1,                   │
│              std::memory_order_relaxed);                            │
│          while (serving.load(std::memory_order_acquire) != my_ticket) {│
│              // 스핀 (백오프 추가 가능)                                   │
│          }                                                           │
│      }                                                              │
│                                                                     │
│      void unlock() {                                                │
│          serving.fetch_add(1, std::memory_order_release);           │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【백오프 추가】                                                        │
│  ──────────────────                                                  │
│  class TicketLockWithBackoff {                                      │
│      // ... 위와 동일 ...                                              │
│                                                                     │
│      void lock() {                                                  │
│          uint32_t my_ticket = ticket.fetch_add(1,                   │
│              std::memory_order_relaxed);                            │
│                                                                     │
│          uint32_t backoff = 1;                                       │
│          while (serving.load(std::memory_order_acquire) != my_ticket) {│
│              for (uint32_t i = 0; i < backoff; i++) {                │
│                  cpu_relax();                                        │
│              }                                                       │
│              backoff = std::min(backoff * 2, 1024u);                 │
│          }                                                           │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【Rust parking_lot::FairMutex】                                      │
│  ──────────────────                                                  │
│  // parking_lot 크레이트: 티켓 락 기반 공정 락                            │
│  use parking_lot::FairMutex;                                        │
│                                                                     │
│  let mutex = FairMutex::new(0);                                      │
│  {                                                                  │
│      let _guard = mutex.lock();  // FIFO 순서                        │
│      // 작업                                                          │
│  }                                                                  │
│                                                                     │
│  【Linux 커널 ticket_spinlock】                                       │
│  ──────────────────                                                  │
│  // arch/x86/include/asm/spinlock.h                                 │
│  typedef struct {                                                    │
│      __ticket_t head;                                                │
│      __ticket_t tail;                                                │
│  } arch_spinlock_t;                                                  │
│                                                                     │
│  static __always_inline void arch_spin_lock(arch_spinlock_t *lock)  │
│  {                                                                  │
│      __ticket_t inc = __sync_fetch_and_add(&lock->tail, 1);          │
│      while (lock->head != inc)                                       │
│          cpu_relax();                                                │
│  }                                                                  │
│                                                                     │
│  static __always_inline void arch_spin_unlock(arch_spinlock_t *lock)│
│  {                                                                  │
│      lock->head++;                                                   │
│  }                                                                  │
│                                                                     │
│  // Linux 5.10 이후: qspinlock으로 대체 (MCS 기반)                      │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  32 코어, 높은 경합                                                  │ │
│  │  ──────────────────                                                │ │
│  │  락 타입              처리량 (ops)    공정성                          │ │
│  │  ────────              ─────────        ────                          │ │
│  │  스핀락 (CAS)           100,000         X (기아 가능)                  │ │
│  │  티켓 락                80,000          O (FIFO)                      │ │
│  │  티켓 락 + 백오프         150,000         O                            │ │
│  │  MCS 락                200,000         O                            │ │
│  │                                                             │ │
│  │  공정성 vs 성능 트레이드오프                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【언제 사용할까?】                                                      │
│  ──────────────────                                                  │
│  적합:                                                                 │
│  • 공정성이 중요한 경우                                                   │
│  • 기아 상태 방지 필요                                                   │
│  • 중간 수준 경합                                                       │
│  • 실시간 시스템 (예측 가능한 지연)                                        │
│                                                                     │
│  부적합:                                                               │
│  • 매우 높은 경합 (MCS 락 권장)                                          │
│  • NUMA 시스템 (NUMA 인식 락 권장)                                       │
│  • 공정성 불필요                                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
