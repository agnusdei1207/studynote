# 275. 락 그래프 (Lock Graph)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 간 의존 관계를 나타낸 방향 그래프
> 2. **가치**: 교착상태 분석, 순환 탐지
> 3. **융합**: 교착상태, 락 순서, 락dep와 연관

---

## Ⅰ. 개요

### 개념 정의

락 그래프(Lock Graph)는 **락 간의 의존 관계를 노드와 간선으로 표현한 방향 그래프**다. 순환이 있으면 교착상태 가능성을 나타낸다.

### 💡 비유: 지도의 원웨이
락 그래프는 **지도의 원웨이 도로**와 같다. A→B→C로 이어진다. 순환하면 계속 돌게 된다. 교착상태다.

### 락 그래프 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 그래프 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  노드 (Node): 락                                                 │ │
│  │  간선 (Edge): A → B (A 보유 중 B 요청)                             │ │
│  │                                                             │ │
│  │  예시:                                                             │ │
│  │  T1: lock(A), lock(B)  →  A → B                                   │ │
│  │  T2: lock(B), lock(C)  →  B → C                                   │ │
│  │  T3: lock(C), lock(A)  →  C → A                                   │ │
│  │                                                             │ │
│  │  그래프: A → B → C → A (순환! 교착상태)                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순환 없는 그래프】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │       [A] ────► [B] ────► [D]                                     │ │
│  │         │                    ▲                                     │ │
│  │         │                    │                                     │ │
│  │         ▼                    │                                     │ │
│  │       [C] ─────────────────┘                                      │ │
│  │                                                             │ │
│  │  DAG (방향성 비순환 그래프)                                          │ │
│  │  교착상태 없음                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순환 있는 그래프】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │       [A] ────► [B]                                                │ │
│  │         ▲          │                                              │ │
│  │         │          ▼                                              │ │
│  │       [D] ◄──── [C]                                               │ │
│  │                                                             │ │
│  │  순환: A → B → C → D → A                                          │ │
│  │  교착상태 발생 가능                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【리소스 할당 그래프】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 프로세스 노드 (원형)                                              │ │
│  │  • 리소스 노드 (사각형)                                              │ │
│  │  • 요청 간선: 프로세스 → 리소스                                       │ │
│  │  • 할당 간선: 리소스 → 프로세스                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 그래프 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【교착상태 조건】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  락 그래프에서 순환이 있다면 다음 4가지 조건 만족:                       │ │
│  │                                                             │ │
│  │  1. 상호 배제 (Mutual Exclusion)                                   │ │
│  │  2. 보유 및 대기 (Hold and Wait)                                   │ │
│  │  3. 비선점 (No Preemption)                                         │ │
│  │  4. 순환 대기 (Circular Wait)                                      │ │
│  │                                                             │ │
│  │  그래프에서 순환 = 순환 대기 조건                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순환 탐지 알고리즘】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. DFS (깊이 우선 탐색):                                          │ │
│  │     O(V + E)                                                       │ │
│  │                                                             │ │
│  │  2. Union-Find:                                                    │ │
│  │     O(E α(V))                                                       │ │
│  │                                                             │ │
│  │  3. Topological Sort:                                              │ │
│  │     O(V + E), DAG만 가능                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux lockdep】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 커널 락 의존성 추적                                               │ │
│  │  • 런타임에 락 그래프 구축                                           │ │
│  │  • 순환 탐지 시 경고                                                 │ │
│  │  • /proc/lockdep 로 확인                                            │ │
│  │                                                             │ │
│  │  사용: CONFIG_LOCKDEP=y                                             │ │
│  │  cat /proc/lockdep                                                  │ │
│  │  cat /proc/lockdep_stats                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Java Thread Dump 분석】                                             │
│  ──────────────────                                                  │
│  // jstack <pid>                                                     │
│                                                                     │
│  // 교착상태 감지 예시                                                  │
│  Found one Java-level deadlock:                                     │
│  =============================                                       │
│  "Thread-1":                                                        │
│    waiting to lock monitor 0x... (object 0x..., a LockA)            │
│    which is held by "Thread-2"                                      │
│  "Thread-2":                                                        │
│    waiting to lock monitor 0x... (object 0x..., a LockB)            │
│    which is held by "Thread-1"                                      │
│                                                                     │
│  // 순환: Thread-1 → LockA → Thread-2 → LockB → Thread-1            │
│                                                                     │
│  【Python 교착상태 탐지】                                               │
│  ──────────────────                                                  │
│  import threading                                                   │
│  import sys                                                         │
│                                                                     │
│  # threading 모듈의 교착상태 탐지                                       │
│  # sys.setswitchinterval()로 체크                                     │
│                                                                     │
│  # deadlock-detection 라이브러리                                       │
│  from deadlock_detection import detect_deadlocks                    │
│                                                                     │
│  【C++ AddressSanitizer (ASan) + Lockdep】                            │
│  ──────────────────                                                  │
│  // g++ -fsanitize=thread -g                                        │
│  // ThreadSanitizer가 교착상태 감지                                     │
│                                                                     │
│  // TSAN_OPTIONS=detect_deadlocks=1                                 │
│                                                                     │
│  【Linux lockdep 사용】                                                │
│  ──────────────────                                                  │
│  // 커널 설정                                                         │
│  CONFIG_LOCKDEP=y                                                    │
│  CONFIG_LOCK_STAT=y                                                  │
│                                                                     │
│  // 부팅 후                                                           │
│  cat /proc/lockdep                                                  │
│  cat /proc/lockdep_stats                                            │
│  cat /proc/lockdep_chains                                           │
│                                                                     │
│  // 예시 출력                                                         │
│  lockdep: .../lockA                                                 │
│    .../lockB                                                         │
│    .../lockC                                                         │
│                                                                     │
│  // 순환 경고                                                         │
│  ======================================================              │
│  [ INFO: possible circular locking dependency detected ]            │
│  ------------------------------------------------------              │
│                                                                     │
│  【락 그래프 시각화 도구】                                                │
│  ──────────────────                                                  │
│  • pstack + graphviz                                                 │
│  • jstack + TDA (Thread Dump Analyzer)                               │
│  • JConsole, VisualVM                                               │
│  • perf + flamegraph                                                 │
│                                                                     │
│  【순환 탐지 구현】                                                      │
│  ──────────────────                                                  │
│  class LockGraph {                                                   │
│      std::map<Lock*, std::set<Lock*>> edges;                        │
│                                                                     │
│      void add_edge(Lock* from, Lock* to) {                          │
│          edges[from].insert(to);                                    │
│      }                                                              │
│                                                                     │
│      bool has_cycle() {                                              │
│          std::set<Lock*> visited, rec_stack;                         │
│          for (auto& [lock, _] : edges) {                            │
│              if (dfs(lock, visited, rec_stack))                      │
│                  return true;                                        │
│          }                                                          │
│          return false;                                               │
│      }                                                              │
│                                                                     │
│      bool dfs(Lock* lock, std::set<Lock*>& visited,                 │
│               std::set<Lock*>& rec_stack) {                          │
│          if (rec_stack.count(lock)) return true;  // 순환!            │
│          if (visited.count(lock)) return false;                      │
│          visited.insert(lock);                                        │
│          rec_stack.insert(lock);                                     │
│          for (Lock* next : edges[lock]) {                            │
│              if (dfs(next, visited, rec_stack))                      │
│                  return true;                                        │
│          }                                                          │
│          rec_stack.erase(lock);                                      │
│          return false;                                               │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【성능 고려사항】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 그래프 분석: O(V + E)                                            │ │
│  │  • 락dep: 런타임 오버헤드 발생                                         │ │
│  │  • 프로덕션에서는 보통 비활성화                                         │ │
│  │  • 개발/테스트에서 활성화 권장                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
