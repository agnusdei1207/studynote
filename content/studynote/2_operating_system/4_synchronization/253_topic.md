# 253. 낙관적 락 (Optimistic Locking)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 충돌 가정 없이 실행, 충돌 시 롤백
> 2. **가치**: 높은 동시성, 낮은 경합 시 높은 성능
> 3. **융합**: MVCC, 트랜잭셔널 메모리, CAS와 연관

---

## Ⅰ. 개요

### 개념 정의

낙관적 락(Optimistic Locking)은 **충돌이 없을 것이라고 가정하고 락 없이 실행한 후, 커밋 시점에 충돌을 검사하는 동시성 제어 패턴**이다. 충돌 시에는 롤백하고 재시도한다.

### 💡 비유: 편의점 계산대기 없이 나가기
낙관적 락은 **편의점 계산대기 없이 나가기**와 같다. 물건을 담고, 계산대기로 간다. 앞 사람과 충돌 없으면 바로 계산한다. 충돌하면 다시 줄을 선다.

### 낙관적 락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                낙관적 락 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【vs 비관적 락】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  비관적 락 (Pessimistic Locking):                                 │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  lock();                    // 락 획득                            │ │ │
│  │  │  // 임계 영역                                                 │ │ │
│  │  │  data = read();             // 읽기                               │ │ │
│  │  │  data = modify(data);       // 수정                               │ │ │
│  │  │  write(data);               // 쓰기                                │ │ │
│  │  │  unlock();                  // 락 해제                            │ │ │
│  │  │                                                         │ │ │
│  │  │  특징: 충돌 방지, 락 오버헤드                                   │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  낙관적 락 (Optimistic Locking):                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  old_version = version;   // 버전 읽기                          │ │ │
│  │  │  // 락 없이 읽기                                               │ │ │
│  │  │  data = read();                                                │ │ │
│  │  │  // 락 없이 수정                                                │ │ │
│  │  │  new_data = modify(data);                                      │ │ │
│  │  │  // 커밋 시점 검사                                              │ │ │
│  │  │  if (version == old_version) {                                 │ │ │
│  │  │      version++;              // 버전 증가                         │ │ │
│  │  │      write(new_data);       // 쓰기                                │ │ │
│  │  │  } else {                                                      │ │ │
│  │  │      retry();                // 롤백, 재시도                       │ │ │
│  │  │  }                                                              │ │ │
│  │  │                                                         │ │ │
│  │  │  특징: 충돌 감지, 롤백 후 재시도                                │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버전 기반 낙관적 락】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스레드 A:                           스레드 B:                        │ │
│  │  ────────────────                    ────────────────            │ │
│  │  version = 1                                                   │ │
│  │  v1 = read(version) → 1       v2 = read(version) → 1             │ │
│  │  data1 = read(data)            data2 = read(data)                │ │
│  │  new1 = modify(data1)          new2 = modify(data2)              │ │
│  │                                                             │ │
│  │  // 커밋 시도                                                       │ │
│  │  CAS(version, 1, 2)           // 성공!                            │ │
│  │  write(data, new1)                                               │ │
│  │                                                             │ │
│  │                                CAS(version, 1, 2)  // 실패! (version=2) │ │
│  │                                // 롤백: 처음부터 재시도                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                             │ │
│  │  • 락 오버헤드 없음                                                 │ │
│  │  • 높은 동시성                                                      │ │
│  │  • 교착상태 없음                                                    │ │
│  │  • 읽기 작업에 특히 유리                                             │ │
│  │                                                             │ │
│  │  단점:                                                             │ │
│  │  • 충돌 시 롤백 비용                                                 │ │
│  │  • 기아 상태 (starvation) 가능                                      │ │
│  │  • 구현 복잡                                                        │ │
│  │  • 높은 경합 시 성능 저하                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                낙관적 락 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구현 방식】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 버전 번호 (Version Number):                                   │ │
│  │     struct Data {                                                 │ │
│  │         int version;                                              │ │
│  │         // 실제 데이터...                                            │ │
│  │     };                                                             │ │
│  │                                                             │ │
│  │  2. 타임스탬프 (Timestamp):                                        │ │
│  │     struct Data {                                                 │ │
│  │         timestamp last_modified;                                  │ │
│  │         // 실제 데이터...                                            │ │
│  │     };                                                             │ │
│  │                                                             │ │
│  │  3. 체크섬 (Checksum):                                             │ │
│  │     struct Data {                                                 │ │
│  │         uint32_t checksum;                                        │ │
│  │         // 실제 데이터...                                            │ │
│  │     };                                                             │ │
│  │                                                             │ │
│  │  4. CAS (Compare-And-Swap):                                       │ │
│  │     // 버전 번호 없이 직접 CAS                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【활용 분야】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 데이터베이스 (MVCC):                                            │ │
│  │     • PostgreSQL, MySQL InnoDB                                    │ │
│  │     • 읽기 차단 없음                                                 │ │
│  │                                                             │ │
│  │  2. 소스 제어 (Git):                                                │ │
│  │     • 충돌 감지 후 병합                                              │ │
│  │                                                             │ │
│  │  3. 웹 애플리케이션:                                                 │ │
│  │     • ETag 기반 동시성 제어                                          │ │
│  │     • If-Match 헤더                                                 │ │
│  │                                                             │ │
│  │  4. 캐시 시스템:                                                     │ │
│  │     • 캐시 무효화 버전                                               │ │
│  │                                                             │ │
│  │  5. 분산 시스템:                                                     │ │
│  │     • 벡터 시계                                                      │ │
│  │     • 분산 락                                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Java 낙관적 락 (JPA @Version)】                                      │
│  ──────────────────                                                  │
│  @Entity                                                             │
│  public class Account {                                              │
│      @Id                                                              │
│      private Long id;                                                 │
│                                                                     │
│      @Version  // 자동 버전 관리                                       │
│      private Long version;                                           │
│                                                                     │
│      private BigDecimal balance;                                     │
│                                                                     │
│      public void withdraw(BigDecimal amount) {                        │
│          // JPA가 자동으로 버전 확인                                    │
│          if (this.balance.compareTo(amount) >= 0) {                    │
│              this.balance = this.balance.subtract(amount);            │
│          }                                                              │
│      }                                                                  │
│  }                                                                   │
│                                                                     │
│  【C++ 버전 기반 낙관적 락】                                             │
│  ──────────────────                                                  │
│  struct Data {                                                        │
│      std::atomic<int> version{0};                                     │
│      int value;                                                        │
│  };                                                                   │
│                                                                     │
│  bool update(Data* d, int new_value) {                                │
│      int old_version = d->version.load();                             │
│      // ... 수정 작업 ...                                              │
│      return d->version.compare_exchange_strong(                       │
│          old_version, old_version + 1                                 │
│      );                                                               │
│  }                                                                   │
│                                                                     │
│  【SQL 낙관적 락】                                                      │
│  ──────────────────                                                  │
│  -- 버전 열 추가                                                       │
│  ALTER TABLE products ADD COLUMN version INT DEFAULT 1;              │
│                                                                     │
│  -- 업데이트 시 버전 확인                                               │
│  UPDATE products                                                     │
│  SET price = 100, version = version + 1                              │
│  WHERE id = 1 AND version = 5;                                       │
│                                                                     │
│  -- 영향받은 행이 0이면 충돌 → 재시도                                  │
│                                                                     │
│  【HTTP ETag 기반 낙관적 락】                                         │
│  ──────────────────                                                  │
│  // 클라이언트                                                          │
│  GET /api/resource/1                                                 │
│  Response:                                                            │
│    ETag: "v1"                                                         │
│    { data: "..." }                                                    │
│                                                                     │
│  // 업데이트                                                           │
│  PUT /api/resource/1                                                 │
│  Headers:                                                            │
│    If-Match: "v1"                                                    │
│  Body: { updated_data: "..." }                                       │
│                                                                     │
│  // 서버: 버전이 v1이면 성공, 아니면 409 Conflict                     │
│                                                                     │
│  【Git 낙관적 병합】                                                   │
│  ──────────────────                                                  │
│  git pull    // 원격 변경사항 가져오기                                │
│  // 충돌 없으면 자동 병합                                              │
│  // 충돌 있으면 수동 해결 필요                                          │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  경합률     비관적 락    낙관적 락        추천                  │ │
│  │  ──────     ────────        ────────        ────                   │ │
│  │  0%         좋음           매우 좋음        둘 다 좋음               │ │
│  │  1%         좋음           좋음            비슷함                 │ │
│  │  10%        느려짐           비슷함           비슷함                 │ │
│  │  50%        느림            느려짐           비관적이 나음           │ │
│  │                                                             │ │
│  │  낙관적 락: 낮은 경합 시 우수, 높은 경합 시 성능 저하            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 충돌 가정 없이 실행, 충돌 시 롤백
• 특징: 락 없이 읽기, 커밋 시 충돌 검사
• 구현: 버전 번호, 타임스탬프, CAS
• 장점: 높은 동시성, 교착상태 없음
• 단점: 높은 경합 시 성능 저하, 기아 상태
• 활용: DB MVCC, HTTP ETag, Git
```
