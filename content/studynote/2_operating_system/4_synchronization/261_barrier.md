# 261. 락 타임아웃 (Lock Timeout)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 획득 대기 시간 제한
> 2. **가치**: 무한 대기 방지, 응답성 보장
> 3. **융합**: 교착상태 방지, 타임아웃 정책과 연관

---

## Ⅰ. 개요

### 개념 정의

락 타임아웃(Lock Timeout)은 **락 획득을 위해 대기하는 최대 시간을 제한하는 메커니즘**이다. 지정된 시간 내에 락을 획득하지 못하면 실패를 반환하거나 예외를 발생시킨다.

### 💡 비유: 티켓 부스 대기
락 타임아웃은 **티켓 부스 대기**와 같다. 10분만 기다리기로 한다. 10분 내에 티켓을 못 사면 포기하고 다른 일을 한다. 무한정 기다리지 않는다.

### 락 타임아웃 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 타임아웃 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【무한 대기 vs 타임아웃】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  무한 대기 (기본 동작):                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  T1: lock() ────[락 보유]──────────────────────────────►│ │ │
│  │  │  T2: lock() ─X─[영원히 대기...]───────────────────────────►│ │ │
│  │  │                    ↑                                       │ │ │
│  │  │              T1이 unlock 안 하면 영원히 대기                   │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  타임아웃:                                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  T1: lock() ────[락 보유]──────────────────────────────────►│ │ │
│  │  │  T2: lock(timeout=5s)                                    │ │ │
│  │  │      ─X─[5초 대기]─X─ 타임아웃! → 실패 반환                    │ │ │
│  │  │                    ↑                                       │ │ │
│  │  │              지정된 시간 내 획득 못 하면 포기                    │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 옵션】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 즉시 실패 (try_lock):                                           │ │
│  │     • 타임아웃 = 0                                                  │ │
│  │     • 락 획득 시도만 하고 실패 시 즉시 반환                            │ │
│  │                                                             │ │
│  │  2. 지정 시간 대기 (timed lock):                                     │ │
│  │     • 타임아웃 = N 밀리초/초                                         │ │
│  │     • 지정된 시간 동안 대기 후 실패 시 반환                            │ │
│  │                                                             │ │
│  │  3. 무한 대기 (blocking lock):                                      │ │
│  │     • 타임아웃 = 무제한                                              │ │
│  │     • 락 획득까지 영원히 대기                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                              │ │
│  │  • 무한 대기 방지                                                    │ │
│  │  • 응답성 보장                                                       │ │
│  │  • 교착상태 복구 가능                                                │ │
│  │  • 사용자 경험 개선                                                  │ │
│  │                                                             │ │
│  │  단점:                                                               │ │
│  │  • 실패 처리 로직 필요                                               │ │
│  │  • 타임아웃 값 설정 어려움                                            │ │
│  │  • 너무 짧으면 불필요한 실패                                          │ │
│  │  • 너무 길면 응답성 저하                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  고려사항:                                                          │ │
│  │  • 작업 복잡도: 긴 작업일수록 긴 타임아웃                                │ │
│  │  • 시스템 부하: 높은 부하 시 더 긴 타임아웃                              │ │
│  │  • 사용자 기대: 사용자 대기 가능 시간                                  │ │

│  │                                                             │ │
│  │  일반적 가이드라인:                                                    │ │
│  │  • UI 스레드: 100ms ~ 1s                                            │ │
│  │  • 백엔드 서비스: 1s ~ 30s                                          │ │
│  │  • 배치 작업: 30s ~ 5min                                            │ │
│  │  • DB 연결: 30s (기본값)                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실패 시 대응 전략】                                                  │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 즉시 재시도:                                                     │ │
│  │     • 스피너 패턴                                                    │ │
│  │     • 짧은 작업에 적합                                                │ │
│  │                                                             │ │
│  │  2. 지수 백오프:                                                     │ │
│  │     wait = min(max_wait, base * 2^attempt)                        │ │
│  │     • 네트워크/분산 시스템에 적합                                      │ │
│  │                                                             │ │
│  │  3. 포기 및 대안:                                                    │ │
│  │     • 다른 작업 수행                                                  │ │
│  │     • 에러 반환                                                      │ │
│  │                                                             │ │
│  │  4. 로깅 및 모니터링:                                                 │ │
│  │     • 타임아웃 발생 추적                                              │ │

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ std::timed_mutex】                                             │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│  #include <chrono>                                                   │
│                                                                     │
│  std::timed_mutex mtx;                                              │
│                                                                     │
│  void try_with_timeout() {                                          │
│      // 100ms 동안 대기                                               │
│      if (mtx.try_lock_for(std::chrono::milliseconds(100))) {        │
│          // 락 획득 성공                                               │
│          // ... 작업 ...                                               │
│          mtx.unlock();                                               │
│      } else {                                                        │
│          // 타임아웃 - 락 획득 실패                                     │
│          handle_timeout();                                           │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  // 시점 기반 타임아웃                                                   │
│  void try_until_deadline() {                                        │
│      auto deadline = std::chrono::steady_clock::now() +             │
│                      std::chrono::seconds(5);                        │
│      if (mtx.try_lock_until(deadline)) {                            │
│          // ... 작업 ...                                               │
│          mtx.unlock();                                               │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【C++ std::shared_timed_mutex (읽기-쓰기 타임아웃)】                   │
│  ──────────────────                                                  │
│  std::shared_timed_mutex rw_mtx;                                    │
│                                                                     │
│  void read_with_timeout() {                                         │
│      std::shared_lock<std::shared_timed_mutex> lock;                │
│      if (lock.try_lock_for(std::chrono::milliseconds(50))) {        │
│          // 읽기 작업                                                  │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Java ReentrantLock with Timeout】                                  │
│  ──────────────────                                                  │
│  import java.util.concurrent.locks.*;                               │
│                                                                     │
│  ReentrantLock lock = new ReentrantLock();                          │
│                                                                     │
│  public void tryWithTimeout() {                                     │
│      try {                                                          │
│          if (lock.tryLock(100, TimeUnit.MILLISECONDS)) {            │
│              try {                                                   │
│                  // 작업                                               │
│              } finally {                                             │
│                  lock.unlock();                                      │
│              }                                                       │
│          } else {                                                    │
│              // 타임아웃                                               │
│          }                                                          │
│      } catch (InterruptedException e) {                             │
│          Thread.currentThread().interrupt();                        │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Python threading.Lock with timeout】                               │
│  ──────────────────                                                  │
│  import threading                                                   │
│                                                                     │
│  lock = threading.Lock()                                            │
│                                                                     │
│  def try_with_timeout():                                            │
│      if lock.acquire(timeout=1.0):  # 1초                           │
│          try:                                                        │
│              # 작업                                                   │
│              pass                                                    │
│          finally:                                                    │
│              lock.release()                                          │
│      else:                                                          │
│          # 타임아웃                                                   │
│          pass                                                        │
│                                                                     │
│  【Go context.WithTimeout】                                           │
│  ──────────────────                                                  │
│  import (                                                           │
│      "context"                                                       │
│      "time"                                                          │
│  )                                                                  │
│                                                                     │
│  func tryWithTimeout(ctx context.Context) error {                   │
│      ctx, cancel := context.WithTimeout(ctx, 5*time.Second)         │
│      defer cancel()                                                  │
│                                                                     │
│      select {                                                        │
│      case <-ctx.Done():                                              │
│          return ctx.Err()  // 타임아웃                                 │
│      case <-acquireLock():                                           │
│          defer releaseLock()                                         │
│          // 작업                                                      │
│          return nil                                                  │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【DB 락 타임아웃】                                                     │
│  ──────────────────                                                  │
│  -- SQL Server                                                       │
│  SET LOCK_TIMEOUT 5000;  -- 5초                                      │
│  SELECT * FROM table WITH (NOWAIT);  -- 즉시 실패                     │
│                                                                     │
│  -- Oracle                                                          │
│  SELECT * FROM table FOR UPDATE NOWAIT;  -- 즉시 실패                │
│  SELECT * FROM table FOR UPDATE WAIT 5;  -- 5초 대기                 │
│                                                                     │
│  -- MySQL                                                           │
│  SET innodb_lock_wait_timeout = 50;  -- 50초                        │
│                                                                     │
│  【지수 백오프 구현】                                                    │
│  ──────────────────                                                  │
│  func withExponentialBackoff(ctx context.Context,                   │
│      base time.Duration, max time.Duration,                          │
│      fn func() error) error {                                        │
│      wait := base                                                    │
│      for {                                                           │
│          err := fn()                                                 │
│          if err == nil {                                             │
│              return nil                                               │
│          }                                                           │
│          select {                                                     │
│          case <-ctx.Done():                                           │
│              return ctx.Err()                                         │
│          case <-time.After(wait):                                     │
│              wait = time.Duration(min(int64(wait)*2, int64(max)))    │
│          }                                                           │
│      }                                                              │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
