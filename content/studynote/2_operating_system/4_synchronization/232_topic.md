# 232. 뮤텍스 (Mutex)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 상호 배제를 위한 소유권 있는 락
> 2. **가치**: 임계 영역 보호, 소유자만 해제
> 3. **융합**: 세마포어, 스핀락, 동기화와 연관

---

## Ⅰ. 개요

### 개념 정의

뮤텍스(Mutex, Mutual Exclusion)는 **상호 배제를 구현하고, 락을 획득한 스레드만 해제할 수 있는 소유권 있는 동기화 객체**다. 임계 영역 보호에 가장 널리 사용된다.

### 💡 비유: 화장실 열쇠
뮤텍스는 **화장실 열쇠**와 같다. 열쇠를 가진 사람만 화장실을 쓰고, 그 사람만 열쇠를 반납한다. 다른 사람이 반납할 수 없다.

### 뮤텍스 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                뮤텍스 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【뮤텍스 상태】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  상태           소유자            대기 큐                      │ │
│  │  ────               ────                ────                    │ │
│  │  잠금            있음              비어있음                      │
│  │  잠금 대기        있음              스레드들                      │ │
│  │  해제            없음              비어있음                      │ │
│  │                                                             │ │
│  │  ┌─────────────┐                                          │ │
│  │  │   Mutex     │                                          │ │
│  │  │  ┌───────┐  │                                          │ │
│  │  │  │ Locked│  │ ← Owner: Thread A                        │ │
│  │  │  └───────┘  │                                          │ │
│  │  │  Waiters:   │                                          │ │
│  │  │  [B] [C] [D]│                                          │ │
│  │  └─────────────┘                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【뮤텍스 vs 세마포어】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              뮤텍스                세마포어                │ │
│  │  ────                  ────                    ────              │ │
│  │  카운터            0 또는 1           정수 (음수 가능)           │ │
│  │  소유권            있음                없음                      │ │
│  │  해제              소유자만            누구나                    │ │
│  │  재귀              가능 (일부)          불가능                   │ │
│  │  용도              상호 배제            자원 수 관리              │ │
│  │  의미론            락                  신호                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【뮤텍스 연산】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  lock(): 락 획득 (소유권 획득, 대기 시 슬립)                       │ │
│  │  unlock(): 락 해제 (소유자만 가능)                               │ │
│  │  trylock(): 락 획득 시도 (실패 시 즉시 반환)                       │ │
│  │  timedlock(): 타임아웃 있는 락 획득                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                뮤텍스 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【뮤텍스 종류】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 일반 뮤텍스 (Normal Mutex)                                   │ │
│  │     • 가장 기본적인 형태                                          │ │
│  │     • 재귀 획득 시 데드락 또는 정의되지 않은 동작                   │ │
│  │                                                             │ │
│  │  2. 재귀 뮤텍스 (Recursive Mutex)                                │ │
│  │     • 같은 스레드에서 여러 번 획득 가능                            │ │
│  │     • 획득 횟수만큼 해제 필요                                      │ │
│  │                                                             │ │
│  │  3. 에러 체크 뮤텍스 (Errorcheck Mutex)                          │ │
│  │     • 잘못된 사용 시 에러 반환                                    │ │
│  │     • 디버깅에 유용                                               │ │
│  │                                                             │ │
│  │  4. 적응형 뮤텍스 (Adaptive Mutex)                               │ │
│  │     • 짧은 대기 시 스핀                                           │ │
│  │     • 긴 대기 시 슬립                                             │ │
│  │                                                             │ │
│  │  5. 우선순위 상속 뮤텍스 (Priority Inheritance)                  │ │
│  │     • RT 애플리케이션용                                           │ │
│  │     • 우선순위 역전 방지                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAII 패턴】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // lock_guard: 단순한 스코프 락                                  │ │
│  │  {                                                               │ │
│  │      std::lock_guard<std::mutex> lock(mtx);                      │ │
│  │      // 임계 영역                                                 │ │
│  │  }  // 자동 unlock                                               │
│  │                                                             │ │
│  │  // unique_lock: 더 유연함                                        │ │
│  │  {                                                               │ │
│  │      std::unique_lock<std::mutex> lock(mtx);                     │ │
│  │      // 필요시 lock.unlock() / lock.lock()                       │ │
│  │  }  // 자동 unlock                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Futex 기반 구현】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 빠른 경로: 원자적 CAS로 락 획득 시도                          │ │
│  │  2. 느린 경로: 커널에 대기 요청 (futex_wait)                      │ │
│  │  3. 깨우기: 커널이 대기 중인 스레드 깨움 (futex_wake)              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【POSIX Mutex】                                                      │
│  ──────────────────                                                  │
│  #include <pthread.h>                                                │
│                                                                     │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│                                                                     │
│  void critical_section() {                                          │
│      pthread_mutex_lock(&mutex);                                    │
│      // 임계 영역                                                    │
│      pthread_mutex_unlock(&mutex);                                  │
│  }                                                                  │
│                                                                     │
│  // 재귀 뮤텍스                                                       │
│  pthread_mutexattr_t attr;                                          │
│  pthread_mutexattr_init(&attr);                                     │
│  pthread_mutexattr_settype(&attr, PTHREAD_MUTEX_RECURSIVE);         │
│  pthread_mutex_t rec_mutex;                                         │
│  pthread_mutex_init(&rec_mutex, &attr);                             │
│                                                                     │
│  【C++ std::mutex】                                                   │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│                                                                     │
│  std::mutex mtx;                                                    │
│  int shared_data = 0;                                               │
│                                                                     │
│  void safe_increment() {                                            │
│      std::lock_guard<std::mutex> lock(mtx);                         │
│      shared_data++;                                                 │
│  }                                                                  │
│                                                                     │
│  // 여러 뮤텍스 안전하게 잠그기                                        │
│  std::mutex mtx1, mtx2;                                             │
│  void safe_multi_lock() {                                           │
│      std::lock(mtx1, mtx2);  // 데드락 방지                          │
│      std::lock_guard<std::mutex> lock1(mtx1, std::adopt_lock);      │
│      std::lock_guard<std::mutex> lock2(mtx2, std::adopt_lock);      │
│  }                                                                  │
│                                                                     │
│  【Java synchronized】                                                │
│  ──────────────────                                                  │
│  public class Counter {                                             │
│      private int count = 0;                                         │
│      private final Object lock = new Object();                      │
│                                                                     │
│      public void increment() {                                      │
│          synchronized (lock) {                                      │
│              count++;                                               │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Python threading.Lock】                                            │
│  ──────────────────                                                  │
│  import threading                                                   │
│                                                                     │
│  lock = threading.Lock()                                            │
│                                                                     │
│  with lock:  # context manager                                      │
│      shared_data += 1                                               │
│                                                                     │
│  【성능 분석】                                                         │
│  ──────────────────                                                  │
│  # 뮤텍스 경합 측정                                                   │
│  $ perf lock record ./program                                       │
│  $ perf lock report                                                 │
│                                                                     │
│  # 뮤텍스 대기 시간                                                   │
│  $ perf record -e lock:lock_acquire ./program                      │
│  $ perf report                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
