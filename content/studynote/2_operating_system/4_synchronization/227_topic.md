# 227. happens-before (선행 관계)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 연산이 다른 연산보다 먼저 보임을 보장하는 관계
> 2. **가치**: 메모리 가시성과 순서 보장의 기반
> 3. **융합**: 메모리 모델, 동기화, 메모리 배리어와 연관

---

## Ⅰ. 개요

### 개념 정의

happens-before는 **한 메모리 연산의 결과가 다른 연산에게 보이는 것을 보장하는 순서 관계**다. A happens-before B이면, A의 쓰기가 B에게 보인다.

### 💡 비유: 우편 배달
happens-before는 **우편 배달**과 같다. 편지를 보낸 후(A) 상대방이 받기까지(B) 순서가 보장된다. 보내기 전에 받을 수는 없다.

### happens-before 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                happens-before 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【happens-before 규칙】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 프로그램 순서 (Program Order)                                │ │
│  │     • 같은 스레드 내에서 앞선 문장이 뒤따르는 문장에 선행           │ │
│  │     • A → B (같은 스레드)                                       │ │
│  │                                                             │ │
│  │  2. 모니터 락 (Monitor Lock)                                     │ │
│  │     • unlock() → 같은 락의 lock()                               │ │
│  │                                                             │ │
│  │  3. volatile/atomic                                            │ │
│  │     • volatile 쓰기 → 같은 변수의 volatile 읽기                  │ │
│  │                                                             │ │
│  │  4. 스레드 시작/종료                                              │ │
│  │     • thread.start() → 스레드의 첫 번째 액션                     │ │
│  │     • 스레드의 마지막 액션 → thread.join()                        │ │
│  │                                                             │ │
│  │  5. 전이성 (Transitivity)                                        │ │
│  │     • A → B 이고 B → C 이면 A → C                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시각화】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  스레드 A                    스레드 B                           │ │
│  │  ────────                    ────────                          │ │
│  │                                                             │ │
│  │  x = 1                     ┌───────────────┐                  │ │
│  │       │                    │  while (!flag)│                  │ │
│  │       ▼                    │    대기...     │                  │ │
│  │  flag = true───────────────►               │                  │ │
│  │       │                    └───────┬───────┘                  │ │
│  │       │                            │                          │ │
│  │       │  happens-before            ▼                          │ │
│  │       │                    print(x)  // x=1 보장               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【synchronizes-with】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  release 쓰기 ←────────────────── acquire 읽기                  │ │
│  │       │                                    │                   │
│  │       ▼                                    ▼                   │
│  │  happens-before                     happens-after               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                happens-before 상세                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Java Memory Model 규칙】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Program Order Rule                                          │ │
│  │  2. Monitor Lock Rule                                           │ │
│  │  3. Volatile Variable Rule                                      │ │
│  │  4. Thread Start Rule                                           │ │
│  │  5. Thread Termination Rule                                     │ │
│  │  6. Interruption Rule                                           │ │
│  │  7. Object Finalization Rule                                    │ │
│  │  8. Transitivity                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【C++ Memory Model 규칙】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • sequenced-before: 같은 스레드 내 순서                         │ │
│  │  • synchronizes-with: 원자적 연산 간 동기화                      │ │
│  │  • happens-before: sequenced-before + synchronizes-with         │ │
│  │                                                             │ │
│  │  release store ←── synchronizes-with ──→ acquire load          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【사용 예시】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 올바른 순서 보장                                              │ │
│  │  data = 42;                        // A                        │ │
│  │  ready.store(true, release);       // B (release)              │ │
│  │                                                             │ │
│  │  // 다른 스레드                                                   │ │
│  │  while (!ready.load(acquire));     // C (acquire)              │ │
│  │  use(data);                        // D                        │ │
│  │                                                             │ │
│  │  순서: A ─(sb)→ B ─(sw)→ C ─(sb)→ D                              │ │
│  │  따라서: A ─(hb)→ D  (data=42이 D에게 보임)                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【생산자-소비자 패턴】                                                │
│  ──────────────────                                                  │
│  // C++                                                              │
│  std::atomic<bool> ready{false};                                    │
│  int data;                                                          │
│                                                                     │
│  void producer() {                                                  │
│      data = 42;  // (1)                                             │
│      ready.store(true, std::memory_order_release);  // (2)         │
│  }                                                                  │
│                                                                     │
│  void consumer() {                                                  │
│      while (!ready.load(std::memory_order_acquire)) {}  // (3)     │
│      assert(data == 42);  // (4) 항상 참                             │
│  }                                                                  │
│                                                                     │
│  // happens-before: (1)→(2)→(3)→(4)                                 │
│  // 따라서 (4)에서 data==42 보장                                      │
│                                                                     │
│  【잘못된 예시】                                                       │
│  ──────────────────                                                  │
│  // 메모리 순서 없이 - 버그 가능                                       │
│  bool ready = false;                                                │
│  int data;                                                          │
│                                                                     │
│  void producer() {                                                  │
│      data = 42;                                                     │
│      ready = true;  // 재배치 가능!                                  │
│  }                                                                  │
│                                                                     │
│  void consumer() {                                                  │
│      if (ready) {                                                   │
│          // data가 42이 아닐 수 있음!                                 │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Java volatile】                                                    │
│  ──────────────────                                                  │
│  volatile boolean ready = false;                                    │
│  int data;                                                          │
│                                                                     │
│  // 스레드 A                                                          │
│  void write() {                                                     │
│      data = 42;        // (1)                                       │
│      ready = true;     // (2) volatile 쓰기                         │
│  }                                                                  │
│                                                                     │
│  // 스레드 B                                                          │
│  void read() {                                                      │
│      if (ready) {      // (3) volatile 읽기                         │
│          // data == 42 보장 (happens-before)                        │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Go sync 패키지】                                                    │
│  ──────────────────                                                  │
│  var (                                                              │
│      data  int                                                      │
│      ready bool                                                     │
│      mu    sync.Mutex                                               │
│  )                                                                  │
│                                                                     │
│  func write() {                                                     │
│      mu.Lock()                                                      │
│      data = 42                                                      │
│      ready = true                                                   │
│      mu.Unlock()  // happens-before 다음 Lock                       │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 연산 A의 결과가 연산 B에게 보이는 관계
• 규칙: 프로그램 순서, 락, volatile, 스레드 시작/종료
• 전이성: A→B, B→C 이면 A→C
• synchronizes-with: release-acquire 페어
• 활용: 생산자-소비자, 초기화 패턴
• 중요: 메모리 가시성 보장의 핵심
```
