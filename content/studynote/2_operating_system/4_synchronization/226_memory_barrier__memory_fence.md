# 226. 메모리 모델 (Memory Model)

#### 핵심 인사이트 (3줄 요약)

## Ⅰ. 개요

### 개념 정의

메모리 모델(Memory Model)은 **다중 스레드 환경에서 메모리 읽기/쓰기 연산의 순서와 가시성을 정의하는 규칙**이다. 프로그래머와 컴파일러/CPU 간의 계약이다.

### 💡 비유: 게임 규칙
메모리 모델은 **게임 규칙**과 같다. 모든 선수가 같은 규칙을 따르면 경기가 공정하게 진행된다. 규칙이 없으면 혼란이다.

### 메모리 모델 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 모델 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【순차적 일관성 (Sequential Consistency)】                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  스레드 A: W(x, 1) → W(y, 2)                                  │ │
│  │  스레드 B: R(y) → R(x)                                        │ │
│  │                                                             │ │
│  │  가능한 결과:                                                   │ │
│  │  R(y)=0, R(x)=0  (둘 다 실행 전)                               │ │
│  │  R(y)=0, R(x)=1  (A의 x 쓰기만 보임)                           │ │
│  │  R(y)=2, R(x)=1  (A의 둘 다 보임)                              │ │
│  │  R(y)=2, R(x)=0  (y는 보이고 x는 안 보임) ← 불가능!             │ │
│  │                                                             │ │
│  │  SC에서는 프로그램 순서가 보존됨                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【약한 메모리 모델 (Weak Memory Model)】                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  순서 재배치 (Reordering) 허용:                                  │ │
│  │  ┌─────────────────────────────────────────────────────┐     │ │
│  │  │  프로그램 순서:      W(x) → W(y)                       │     │ │
│  │  │  실제 실행 순서:     W(y) → W(x)  (재배치됨!)           │     │ │
│  │  └─────────────────────────────────────────────────────┘     │ │
│  │                                                             │ │
│  │  이유: 성능 최적화 (캐시, 파이프라인, 버퍼)                       │ │
│  │  문제: 버그 발생 가능                                             │ │
│  │  해결: 메모리 배리어                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【아키텍처별 메모리 모델】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  아키텍처        모델                특징                        │ │
│  │  ────                ────                    ────                │ │
│  │  x86/x86-64     TSO (강함)         스토어 순서만 재배치           │ │
│  │  ARM            약함                다양한 재배치 허용            │ │
│  │  POWER          매우 약함           더 많은 재배치 허용           │ │
│  │  SPARC          TSO                x86과 유사                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 모델 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【재배치 유형】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 컴파일러 재배치                                              │ │
│  │     • 최적화를 위해 명령어 순서 변경                              │ │
│  │     • -O2, -O3에서 적극적                                        │ │
│  │                                                             │ │
│  │  2. CPU 재배치                                                   │ │
│  │     • 아웃오브오더 실행                                           │ │
│  │     • 스토어 버퍼                                                │ │
│  │     • 무효화 큐                                                  │ │
│  │                                                             │ │
│  │  3. 캐시 일관성 지연                                              │ │
│  │     • 다른 코어에 변경 전파 지연                                  │ │
│  │     • MESI 프로토콜                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【C++ 메모리 순서】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  enum memory_order {                                             │ │
│  │      relaxed,    // 순서 보장 없음                                │ │
│  │      consume,    // 데이터 의존성만                               │ │
│  │      acquire,    // 읽기 획득                                    │ │
│  │      release,    // 쓰기 해제                                    │ │
│  │      acq_rel,    // 획득 + 해제                                  │ │
│  │      seq_cst     // 순차적 일관성 (기본)                          │ │
│  │  };                                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Java 메모리 모델】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • volatile: 읽기/쓰기가 메모리 배리어                            │ │
│  │  • final: 생성 후 변경 불가                                      │ │
│  │  • synchronized: 모니터 진입/퇴출이 배리어                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 사용 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ 원자적 연산과 메모리 순서】                                      │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  std::atomic<bool> ready{false};                                    │
│  int data = 0;                                                      │
│                                                                     │
│  // 생산자                                                           │
│  void producer() {                                                  │
│      data = 42;                                                     │
│      ready.store(true, std::memory_order_release);                  │
│  }                                                                  │
│                                                                     │
│  // 소비자                                                           │
│  void consumer() {                                                  │
│      while (!ready.load(std::memory_order_acquire)) {              │
│          // 대기                                                    │
│      }                                                              │
│      // data == 42 보장                                             │
│  }                                                                  │
│                                                                     │
│  【Double-Checked Locking (올바른 구현)】                               │
│  ──────────────────                                                  │
│  class Singleton {                                                  │
│      static std::atomic<Singleton*> instance;                       │
│  public:                                                            │
│      static Singleton* get() {                                      │
│          Singleton* tmp = instance.load(std::memory_order_acquire);│
│          if (!tmp) {                                                │
│              std::lock_guard<std::mutex> lock(mtx);                 │
│              tmp = instance.load(std::memory_order_relaxed);        │
│              if (!tmp) {                                            │
│                  tmp = new Singleton();                             │
│                  instance.store(tmp, std::memory_order_release);    │
│              }                                                      │
│          }                                                          │
│          return tmp;                                                │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【컴파일러 최적화 방지】                                               │
│  ──────────────────                                                  │
│  // 변수 최적화 방지                                                   │
│  volatile int flag;  // C/C++ (순서 보장 없음)                        │
│                                                                     │
│  // 컴파일러 배리어                                                   │
│  asm volatile("" ::: "memory");  // GCC                             │
│                                                                     │
│  // 전체 최적화 방지                                                   │
│  __sync_synchronize();  // GCC full barrier                         │
│                                                                     │
│  【문제 진단】                                                         │
│  ──────────────────                                                  │
│  # TSAN으로 데이터 경쟁 탐지                                          │
│  $ g++ -fsanitize=thread -g program.cpp                            │
│  $ ./a.out                                                          │
│                                                                     │
│  # 메모리 모델 검증 도구                                               │
│  # CDSChecker, GenMC, herd7                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
