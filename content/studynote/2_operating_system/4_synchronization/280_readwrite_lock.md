# 280. 락 통계 (Lock Statistics)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 사용 패턴과 성능 데이터 수집
> 2. **가치**: 병목 식별, 최적화 가이드
> 3. **융합**: 락 경합, 락 프로파일링과 연관

---

## Ⅰ. 개요

### 개념 정의

락 통계(Lock Statistics)는 **락의 획득/해제 횟수, 대기 시간, 경합률 등의 성능 데이터를 수집하고 분석하는 기능**이다. 병목 식별과 최적화에 필수적이다.

### 💡 비유: 교통 통계
락 통계는 **교통 통계**와 같다. 어느 교차로가 막히는지, 얼마나 기다리는지, 얼마나 자주 지나가는지 데이터로 알 수 있다.

### 락 통계 구조

```

│                락 통계 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【수집 항목】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  락 식별자 (Lock ID):                                              │ │
│  │  ┌────────────────────────────────────────────────────────┐ │ │
│  │  │  이름, 주소, 클래스, 파일:라인                              │ │ │
│  │  └────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  카운터:                                                           │ │
│  │  ┌────────────────────────────────────────────────────────┐ │ │
│  │  │  • 획득 횟수 (acquires): lock() 호출 횟수                  │ │ │
│  │  │  • 해제 횟수 (releases): unlock() 호출 횟수                │ │ │
│  │  │  • 경합 횟수 (contentions): 대기 발생 횟수                  │ │ │
│  │  │  • 대기 시간 (wait time): 총/평균/최대 대기 시간            │ │ │
│  │  │  • 보유 시간 (hold time): 총/평균/최대 보유 시간            │ │ │
│  │  └────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  파생 지표:                                                        │ │
│  │  ┌────────────────────────────────────────────────────────┐ │ │
│  │  │  • 경합률: contentions / acquires                        │ │ │
│  │  │  • 평균 대기: wait_time / contentions                    │ │ │
│  │  │  • 평균 보유: hold_time / acquires                       │ │ │
│  │  └────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시각화】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  락별 대기 시간 분포                                                │ │
│  │  ┌─────────────────────────────────────────────────────────┐│ │
│  │  │  Lock A: ████████████████████  1500ms                    ││ │
│  │  │  Lock B: ████████            800ms                       ││ │
│  │  │  Lock C: ████                400ms                       ││ │
│  │  │  Lock D: █                  100ms                        ││ │
│  │  └─────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 통계 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 지표 해석】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  높은 경합률 (> 30%):                                              │ │
│  │  • 락 세분화 필요                                                   │ │
│  │  • 락 스트라이핑 고려                                               │ │
│  │  • 임계 영역 단축                                                   │ │
│  │                                                             │ │
│  │  긴 대기 시간 (> 100μs):                                           │ │
│  │  • 임계 영역 최적화                                                  │ │
│  │  • I/O 제거                                                        │ │
│  │  • 락 분리 고려                                                     │ │
│  │                                                             │ │
│  │  긴 보유 시간 (> 1ms):                                              │ │
│  │  • 임계 영역 과대                                                   │ │
│  │  • 불필요한 작업 포함                                               │ │
│  │  • 비동기 처리 고려                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux lockdep + lock_stat】                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  커널 설정:                                                         │ │
│  │  CONFIG_LOCKDEP=y                                                  │ │
│  │  CONFIG_LOCK_STAT=y                                                │ │
│  │                                                             │ │
│  │  /proc/lockdep_stats:                                              │ │
│  │  • 락 클래스별 통계                                                 │ │
│  │  • 경합, 대기 시간                                                  │ │
│  │                                                             │ │
│  │  /proc/lock_stat:                                                  │ │
│  │  • 상세 성능 통계                                                   │ │
│  │  • con-bounces, waittime, holdtime                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【오버헤드】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 통계 수집: ~5-10% 성능 저하                                       │ │
│  │  • 메모리: 락당 수 KB                                               │ │
│  │  • 프로덕션: 보통 비활성화                                           │ │
│  │  • 개발/테스트: 활성화 권장                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux lock_stat 확인】                                             │
│  ──────────────────                                                  │
│  // 활성화                                                            │
│  echo 1 > /proc/sys/kernel/lock_stat                                │
│                                                                     │
│  // 통계 확인                                                          │
│  cat /proc/lock_stat                                                │
│                                                                     │
│  // 출력 예시                                                         │
│  //-------------------------------------------------------------------│
│  // class name    con-bounces  contentions  waittime-min  waittime-max│
│  //                                                                          │
│  // &inode_lock   12345        1234         0.10us        5000us      │
│  // &dentry_lock  8901         890          0.05us        2000us      │
│  //-------------------------------------------------------------------│
│                                                                     │
│  // 초기화                                                            │
│  echo 0 > /proc/sys/kernel/lock_stat                                │
│                                                                     │
│  【Java Lock Monitoring】                                             │
│  ──────────────────                                                  │
│  // JMX으로 모니터링                                                   │
│  ThreadMXBean threadBean = ManagementFactory.getThreadMXBean();     │
│                                                                     │
│  // ThreadInfo로 락 정보 확인                                         │
│  ThreadInfo info = threadBean.getThreadInfo(threadId);              │
│  String lockName = info.getLockName();                              │
│  String lockOwner = info.getLockOwnerName();                        │
│                                                                     │
│  // ReentrantLock 통계                                               │
│  ReentrantLock lock = new ReentrantLock();                          │
│  int queueLength = lock.getQueueLength();                           │
│  boolean fair = lock.isFair();                                      │
│                                                                     │
│  【BPF/BCC 락 분석】                                                   │
│  ──────────────────                                                  │
│  # bcc/tools/offcputime.py -p <pid>                                 │
│  # bcc/tools/lockstat.py                                            │
│                                                                     │
│  // 커스텀 BPF 스크립트                                                 │
│  // futex, mutex, rwlock 추적 가능                                    │
│                                                                     │
│  【perf 락 분석】                                                      │
│  ──────────────────                                                  │
│  // 락 이벤트 기록                                                     │
│  perf record -e 'syscalls:sys_enter_futex' -a sleep 10              │
│  perf report                                                         │
│                                                                     │
│  // 스케줄링 지연                                                      │
│  perf sched record -- sleep 10                                      │
│  perf sched latency                                                 │
│                                                                     │
│  【자체 통계 구현】                                                     │
│  ──────────────────                                                  │
│  class LockStats {                                                  │
│      std::atomic<uint64_t> acquires{0};                             │
│      std::atomic<uint64_t> contentions{0};                          │
│      std::atomic<uint64_t> total_wait_ns{0};                        │
│      std::atomic<uint64_t> max_wait_ns{0};                          │
│      std::string name;                                              │
│                                                                     │
│      void record_acquire(bool waited, uint64_t wait_ns) {           │
│          acquires.fetch_add(1);                                     │
│          if (waited) {                                              │
│              contentions.fetch_add(1);                              │
│              total_wait_ns.fetch_add(wait_ns);                      │
│              uint64_t current_max = max_wait_ns.load();             │
│              while (wait_ns > current_max) {                        │
│                  if (max_wait_ns.compare_exchange_weak(             │
│                          current_max, wait_ns)) break;              │
│              }                                                      │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      void print_stats() {                                           │
│          double contention_rate =                                   │
│              (double)contentions.load() / acquires.load() * 100;    │
│          double avg_wait = (double)total_wait_ns.load() /           │
│                           contentions.load() / 1000.0;              │
│          printf("%s: acquires=%lu contentions=%lu rate=%.1f%%\n",   │
│                 name.c_str(), acquires.load(), contentions.load(),  │
│                 contention_rate);                                   │
│          printf("  avg_wait=%.2fus max_wait=%.2fus\n",              │
│                 avg_wait, max_wait_ns.load() / 1000.0);             │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【성능 분석 보고서 예시】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  락 이름         획득 횟수    경합률    평균 대기   최대 대기      │ │
│  │  ────────         ──────      ──────    ──────     ──────       │ │
│  │  db_conn_lock     50,000     45%       150μs      5000μs       │ │
│  │  cache_lock       100,000    25%       50μs       500μs        │ │
│  │  log_lock         200,000    5%        10μs       100μs        │ │
│  │                                                             │ │
│  │  권장 사항:                                                        │ │
│  │  • db_conn_lock: 세분화 필요 (높은 경합)                            │ │
│  │  • cache_lock: 양호                                               │ │
│  │  • log_lock: 비동기 로깅 고려                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프로파일링 도구】                                                    │
│  ──────────────────                                                  │
│  • Linux: perf, eBPF, lockstat, lockdep                             │
│  • Java: JConsole, VisualVM, JProfiler, Java Mission Control        │
│  • C++: Intel VTune, perf, valgrind --tool=drd                      │
│  • Go: pprof, trace                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
