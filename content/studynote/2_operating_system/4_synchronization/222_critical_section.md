# 222. 임계 영역 (Critical Section)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공유 자원에 접근하는 코드 영역
> 2. **가치**: 한 번에 하나의 스레드만 실행 보장
> 3. **융합**: 상호 배제, 락, 동기화와 연관

---

## Ⅰ. 개요

### 개념 정의

임계 영역(Critical Section)은 **공유 자원에 접근하거나 수정하는 코드 영역으로, 한 번에 하나의 스레드만 실행할 수 있어야 한다**. 동기화의 핵심 개념이다.

### 💡 비유: 단일 선로
임계 영역은 **기차 단일 선로**와 같다. 한 기차가 지나가는 동안 다른 기차는 대기해야 한다. 충돌을 막기 위해서다.

### 임계 영역 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                임계 영역 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【코드 구조】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────┐    │ │
│  │  │                 스레드 실행 흐름                       │    │ │
│  │  └─────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │         ┌─────────────┐                                    │ │
│  │         │ 진입 섹션    │ lock 획득                          │ │
│  │         └──────┬──────┘                                    │ │
│  │                │                                            │ │
│  │                ▼                                            │ │
│  │    ┌──────────────────────┐                                │ │
│  │    │                      │                                │ │
│  │    │    임계 영역          │ 공유 자원 접근                   │ │
│  │    │    (Critical Section)│                                 │ │
│  │    │                      │                                │ │
│  │    └──────────┬───────────┘                                │ │
│  │                │                                            │ │
│  │                ▼                                            │ │
│  │         ┌─────────────┐                                    │ │
│  │         │ 퇴출 섹션    │ lock 해제                          │ │
│  │         └──────┬──────┘                                    │ │
│  │                │                                            │ │
│  │                ▼                                            │ │
│  │         ┌─────────────┐                                    │ │
│  │         │ 나머지 섹션  │ 공유 자원 미사용                    │ │
│  │         └─────────────┘                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【임계 영역 요구조건】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 상호 배제 (Mutual Exclusion)                                │ │
│  │     • 한 번에 하나의 스레드만 임계 영역 실행                     │ │
│  │                                                             │ │
│  │  2. 진행 (Progress)                                             │ │
│  │     • 임계 영역이 비면, 대기 중인 스레드가 진입                   │ │
│  │     • 무한 대기 없음                                            │ │
│  │                                                             │ │
│  │  3. 한계 대기 (Bounded Waiting)                                 │ │
│  │     • 스레드가 무한정 대기하지 않음                              │ │
│  │     • 공정한 진입 보장                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 상세 분석

│                임계 영역 상세                                          │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 인터럽트 비활성화 (단일 CPU)                                 │ │
│  │     cli(); // 인터럽트 끄기                                     │ │
│  │     // 임계 영역                                                │ │
│  │     sti(); // 인터럽트 켜기                                     │ │

│  │                                                             │ │
│  │  3. Test-and-Set (하드웨어)                                     │ │
│  │     원자적 명령어로 락 구현                                      │ │
│  │                                                             │ │
│  │  4. 뮤텍스/세마포어 (OS 지원)                                    │ │

│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【임계 영역 식별】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  질문: 다음이 임계 영역인가?                                     │ │
│  │                                                             │ │
│  │  1. 로컬 변수만 사용? → NO                                     │ │
│  │  2. 전역 변수 사용? → YES                                      │ │
│  │  3. 힙 메모리 공유? → YES                                      │ │
│  │  4. 파일 I/O? → YES (동일 파일)                                │ │
│  │  5. 정적 변수? → YES                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 가능한 짧게                                                   │ │
│  │  • 복잡한 연산 피하기                                             │ │
│  │  • I/O 작업 피하기                                               │ │

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C 언어 (pthread)】                                                 │
│  ──────────────────                                                  │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│  int shared_counter = 0;                                            │
│                                                                     │
│  void* thread_func(void* arg) {                                     │
│      // 진입 섹션                                                    │
│      pthread_mutex_lock(&mutex);                                    │
│                                                                     │
│      // 임계 영역                                                    │
│      shared_counter++;                                              │
│      printf("Counter: %d\n", shared_counter);                       │
│                                                                     │
│      // 퇴출 섹션                                                    │
│      pthread_mutex_unlock(&mutex);                                  │
│                                                                     │
│      // 나머지 섹션                                                  │
│      return NULL;                                                   │
│  }                                                                  │
│                                                                     │
│  【C++ (std::mutex)】                                                 │
│  ──────────────────                                                  │
│  std::mutex mtx;                                                    │
│  int shared_data = 0;                                               │
│                                                                     │
│  void critical_work() {                                             │
│      std::lock_guard<std::mutex> lock(mtx);  // RAII                │
│      // 임계 영역                                                    │
│      shared_data++;                                                 │
│  }                                                                  │
│                                                                     │
│  【Python】                                                           │
│  ──────────────────                                                  │
│  import threading                                                   │
│                                                                     │
│  lock = threading.Lock()                                            │
│  shared_list = []                                                   │
│                                                                     │
│  def add_item(item):                                                │
│      with lock:  # 임계 영역                                         │
│          shared_list.append(item)                                   │
│                                                                     │
│  【Linux 커널】                                                        │
│  ──────────────────                                                  │
│  spinlock_t my_lock;                                                │
│                                                                     │
│  void critical_section(void) {                                      │
│      spin_lock(&my_lock);                                           │
│      // 임계 영역                                                    │
│      shared_data++;                                                 │
│      spin_unlock(&my_lock);                                         │
│  }                                                                  │
│                                                                     │
│  【문제 진단】                                                         │
│  ──────────────────                                                  │
│  # 임계 영역 진입 시간 측정                                           │
│  $ perf record -e lock:lock_acquire ./program                       │
│  $ perf report                                                      │
│                                                                     │
│  # 락 경합 분석                                                       │
│  $ perf lock record ./program                                       │
│  $ perf lock report                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 공유 자원 접근 코드 영역
• 구조: 진입 섹션 → 임계 영역 → 퇴출 섹션
• 요구조건: 상호 배제, 진행, 한계 대기
• 구현: 인터럽트 제어, 뮤텍스, 스핀락
• 설계: 가능한 짧게, I/O 피하기
• 진단: perf lock, ThreadSanitizer
```
