# 287. 락 에스컬레이션 (Lock Escalation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 낮은 수준 락이 높은 수준 락으로 자동 승격
> 2. **가치**: 락 관리 단순화, 데드락 방지
> 3. **융합**: 락 세분성, 락 호환성과 연관

---

## Ⅰ. 개요

### 개념 정의

락 에스컬레이션(Lock Escalation)은 **시스템이 낮은 수준의 락(행 락)을 높은 수준의 락(테이블 락)으로 자동으로 승격하는 기법**이다. 주로 데이터베이스에서 사용된다.

### 💡 비유: 주차장
락 에스컬레이션은 **주차장**과 같다. 한 대씩 주차하다가 자리가 많이 필요하면 구역 전체를 빌린다.

### 락 에스컬레이션 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 에스컬레이션 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데이터베이스 락 계층】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  락 수준 (낮음 → 높음):                                           │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  1. 행 락 (Row Lock)                                       │ │ │
│  │  │     • 가장 세분화된 락                                       │ │ │
│  │  │     • 개별 행에 대한 락                                       │ │ │
│  │  │                                                         │ │ │
│  │  │  2. 페이지 락 (Page Lock)                                   │ │ │
│  │  │     • 페이지 단위 락                                          │ │ │
│  │  │                                                         │ │ │
│  │  │  3. 테이블 락 (Table Lock)                                  │ │ │
│  │  │     • 테이블 전체 락                                           │ │ │
│  │  │                                                         │ │ │
│  │  │  4. 데이터베이스 락 (Database Lock)                           │ │ │
│  │  │     • 전체 데이터베이스 락                                     │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【에스컬레이션 시나리오】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  트랜잭션이 1000개의 행 락을 획득하려고 시도                           │ │
│  │                                                             │ │
│  │  [시작] → 행 락 1 → 행 락 2 → ... → 행 락 N                     │ │
│  │     ↓                                                            │ │
│  │  임계값 도달 (예: 100개)                                           │ │
│  │     ↓                                                            │ │
│  │  [에스컬레이션] 행 락 → 테이블 락으로 승격                           │ │
│  │     ↓                                                            │ │
│  │  더 이상 행 락 없이 테이블 락만 사용                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【트리거 조건】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 락 개수 임계값 초과                                             │ │
│  │  • 메모리 사용량 초과                                              │ │
│  │  • 락 관리 오버헤드 감소 필요                                       │ │
│  │  • 시스템 설정에 따른 자동                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SQL Server 락 에스컬레이션】                                        │
│  ──────────────────                                                  │
│  -- 에스컬레이션 모니터링                                               │
│  SELECT * FROM sys.dm_tran_locks                                    │
│  WHERE resource_type = 'OBJECT'                                     │
│                                                                     │
│  -- 임계값 확인                                                        │
│  EXEC sp_configure 'locks'                                          │
│                                                                     │
│  -- 에스컬레이션 방지: 힌트 사용                                         │
│  SELECT * FROM large_table WITH (ROWLOCK) WHERE ...                 │
│                                                                     │
│  -- 명시적 테이블 락                                                   │
│  SELECT * FROM large_table WITH (TABLOCKX) WHERE ...                │
│                                                                     │
│  【Oracle 락 에스컬레이션】                                             │
│  ──────────────────                                                  │
│  -- 락 통계 확인                                                        │
│  SELECT * FROM v$lock                                                │
│                                                                     │
│  -- 세그먼트 설정                                                       │
│  ALTER TABLE my_table STORAGE (INITRANS 10 MAXTRANS 100);           │
│                                                                     │
│  -- 명시적 테이블 락                                                     │
│  LOCK TABLE my_table IN EXCLUSIVE MODE;                             │
│                                                                     │
│  【Java/Hibernate 배치 처리】                                           │
│  ──────────────────                                                  │
│  // 배치 크기 제한으로 에스컬레이션 방지                                  │
│  @Transactional                                                     │
│  public void batchInsert(List<Entity> entities) {                   │
│      int batchSize = 50;  // 적절한 크기                               │
│      for (int i = 0; i < entities.size(); i++) {                    │
│          entityManager.persist(entities.get(i));                    │
│          if (i % batchSize == 0) {                                  │
│              entityManager.flush();                                  │
│              entityManager.clear();  // 락 해제                       │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【에스컬레이션 감지 쿼리】                                               │
│  ──────────────────                                                  │
│  -- SQL Server: 에스컬레이션 이벤트                                      │
│  SELECT * FROM sys.dm_exec_requests                                │
│  WHERE blocking_session_id > 0                                      │
│                                                                     │
│  -- Extended Events로 추적                                            │
│  CREATE EVENT SESSION LockEscalation ON SERVER                      │
│  ADD EVENT sqlserver.lock_escalation                                │
│  ADD TARGET package0.event_file                                     │
│  WITH (MAX_FILE_SIZE = 100MB)                                       │
│                                                                     │
│  【성능 영향 분석】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  작업 유형           행 락          에스컬레이션 후               │ │
│  │  ────────           ──────          ─────────────               │ │
│  │  소량 읽기            O(1)           O(1)                        │ │
│  │  대량 읽기            O(n) 메모리     O(1) 메모리                  │ │
│  │  소량 수정            높은 동시성     높은 동시성                   │ │
│  │  대량 수정            낮은 동시성     매우 낮은 동시성               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예방 전략】                                                          │
│  ──────────────────                                                  │
│  • 배치 크기 제한                                                      │
│  • 주기적 커밋/플러시                                                   │
│  • 적절한 인덱스                                                        │
│  • 쿼리 최적화                                                          │
│  • 명시적 락 힌트 사용                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 낮은 락 → 높은 락 자동 승격
• 계층: 행 → 페이지 → 테이블 → DB
• 장점: 오버헤드 감소, 메모리 절약
• 단점: 동시성 감소, 블록 증가
• DBMS: SQL Server, Oracle 자동
• 예방: 배치 제한, 주기적 커밋
```
