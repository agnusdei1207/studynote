# 236. 순차적 락 (Seqlock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 버전 카운터로 읽기 무잠금 구현
> 2. **가치**: 읽기 매우 빠름, 락 오버헤드 없음
> 3. **융합**: 원자적 연산, 메모리 배리어, RCU와 연관

---

## Ⅰ. 개요

### 개념 정의

순차적 락(Sequential Lock, Seqlock)은 **버전 카운터를 사용하여 읽기는 락 없이, 쓰기는 배타적으로 수행하는 동기화 기법**이다. 읽기가 매우 많고 쓰기가 드문 경우에 최적이다.

### 💡 비유: 신문 읽기
순차적 락은 **신문을 읽는 것**과 같다. 기자가 기사를 쓰는 동안에도 읽을 수 있다. 하지만 기사가 바뀌면 다시 읽어야 한다. 버전이 바뀌면 재시도한다.

### 순차적 락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                순차적 락 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 원리】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  버전 카운터 (홀수 = 쓰기 중, 짝수 = 쓰기 없음)                   │ │
│  │                                                             │ │
│  │  읽기:                                                            │ │
│  │  1. start = version_read()  // 시작 버전 기록                   │ │
│  │  2. if (start & 1) goto 1;  // 쓰기 중이면 재시도                │ │
│  │  3. 데이터 읽기                                                   │ │
│  │  4. end = version_read()   // 종료 버전 확인                     │ │
│  │  5. if (start != end) goto 1;  // 버전 변경 시 재시도            │ │
│  │                                                             │ │
│  │  쓰기:                                                            │ │
│  │  1. version++  // 홀수로 변경 (쓰기 시작)                        │ │
│  │  2. 데이터 쓰기                                                   │ │
│  │  3. version++  // 짝수로 변경 (쓰기 종료)                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시각화】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  버전:     0      1      2      3      4                      │ │
│  │           ───┬───┬───┬───┬───┬───                            │ │
│  │              │   │   │   │   │                                 │ │
│  │  쓰기:        └─W─┘   └─W─┘                                    │ │
│  │               (홀수)   (홀수)                                    │ │
│  │                                                             │ │
│  │  읽기1: ┌─start(0)──읽기──end(2)─┐  OK (0→2)                    │ │
│  │        └─────────────────────────┘                             │ │
│  │                                                             │ │
│  │  읽기2: ┌─start(1)─┐                                          │ │
│  │        └─ 홀수! 재시도 ─┐                                       │ │
│  │                       └─start(2)──읽기──end(2)─┘  OK           │ │
│  │                                                             │ │
│  │  읽기3: ┌─start(2)──읽기──end(4)─┐  재시도 (2→4)                │ │
│  │        └─start(4)──읽기──end(4)─┘  OK                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 읽기-쓰기 락】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              읽기-쓰기 락           Seqlock                  │ │
│  │  ────                  ────                    ────                │ │
│  │  읽기              락 획득/해제         락 없음                    │ │
│  │  읽기 오버헤드       있음                 거의 없음                 │ │
│  │  읽기 재시도        없음                 가능                      │ │
│  │  쓰기              배타적               배타적 (짧음)              │ │
│  │  적합              읽기 많음             읽기 매우 많음             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                순차적 락 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 구현】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  typedef struct {                                                │
│  │      unsigned sequence;                                          │
│  │      spinlock_t lock;                                            │ │
│  │  } seqlock_t;                                                    │ │
│  │                                                             │ │
│  │  // 읽기                                                         │
│  │  unsigned do_read(seqlock_t *s, data_t *out) {                  │ │
│  │      unsigned seq;                                               │ │
│  │      do {                                                        │ │
│  │          seq = READ_ONCE(s->sequence);                          │
│  │          if (seq & 1)        // 쓰기 중                           │
│  │              continue;                                           │ │
│  │          *out = read_data();                                     │ │
│  │      } while (seq != READ_ONCE(s->sequence));                   │
│  │      return seq;                                                 │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  │  // 쓰기                                                         │
│  │  void write_lock(seqlock_t *s) {                                 │ │
│  │      spin_lock(&s->lock);                                        │ │
│  │      s->sequence++;  // 홀수                                     │ │
│  │      smp_wmb();          // 메모리 배리어                         │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  │  void write_unlock(seqlock_t *s) {                               │ │
│  │      smp_wmb();                                                  │ │
│  │      s->sequence++;  // 짝수                                     │ │
│  │      spin_unlock(&s->lock);                                      │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적합한 용도】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 시간 정보 (jiffies)                                           │ │
│  │  • 통계 카운터                                                    │ │
│  │  • 시스템 정보                                                    │ │
│  │  • 구성/설정 정보                                                 │ │
│  │  • 읽기 매우 많고 쓰기 드묾                                        │ │
│  │  • 데이터 크기 작음                                               │ │
│  │  • 포인터가 아닌 값                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【부적합한 용도】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 포인터 타입 (재시도 시 dangling pointer)                       │ │
│  │  • 쓰기 빈번한 데이터                                             │ │
│  │  • 읽기가 긴 연산                                                  │ │
│  │  • 읽기 실패 시 비용 큰 경우                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 사용】                                                   │
│  ──────────────────                                                  │
│  #include <linux/seqlock.h>                                          │
│                                                                     │
│  seqlock_t my_seqlock;                                              │
│  seqlock_init(&my_seqlock);                                         │
│                                                                     │
│  // 읽기 (락 없음)                                                    │
│  unsigned int seq;                                                  │
│  data_t data;                                                       │
│  do {                                                               │
│      seq = read_seqbegin(&my_seqlock);                              │
│      data = read_shared_data();                                     │
│  } while (read_seqretry(&my_seqlock, seq));                         │
│                                                                     │
│  // 쓰기                                                             │
│  write_seqlock(&my_seqlock);                                        │
│  update_shared_data(&new_data);                                     │
│  write_sequnlock(&my_seqlock);                                      │
│                                                                     │
│  【사용자 공간 구현】                                                  │
│  ──────────────────                                                  │
│  #include <stdatomic.h>                                             │
│                                                                     │
│  typedef struct {                                                   │
│      atomic_uint sequence;                                          │
│      pthread_mutex_t write_lock;                                    │
│  } seqlock_t;                                                       │
│                                                                     │
│  void seqlock_init(seqlock_t *s) {                                  │
│      atomic_init(&s->sequence, 0);                                  │
│      pthread_mutex_init(&s->write_lock, NULL);                      │
│  }                                                                  │
│                                                                     │
│  unsigned read_begin(seqlock_t *s) {                                │
│      return atomic_load_explicit(&s->sequence, memory_order_acquire);│
│  }                                                                  │
│                                                                     │
│  int read_retry(seqlock_t *s, unsigned start) {                     │
│      unsigned end = atomic_load_explicit(&s->sequence,              │
│                                          memory_order_acquire);     │
│      return (start != end) || (start & 1);                          │
│  }                                                                  │
│                                                                     │
│  void write_lock(seqlock_t *s) {                                    │
│      pthread_mutex_lock(&s->write_lock);                            │
│      atomic_fetch_add_explicit(&s->sequence, 1, memory_order_release);│
│  }                                                                  │
│                                                                     │
│  void write_unlock(seqlock_t *s) {                                  │
│      atomic_fetch_add_explicit(&s->sequence, 1, memory_order_release);│
│      pthread_mutex_unlock(&s->write_lock);                          │
│  }                                                                  │
│                                                                     │
│  【사용 예시】                                                         │
│  ──────────────────                                                  │
│  // 시스템 시간                                                       │
│  struct timespec get_time(void) {                                   │
│      unsigned seq;                                                  │
│      struct timespec ts;                                            │
│      do {                                                           │
│          seq = read_begin(&time_seqlock);                           │
│          ts = current_time;                                         │
│      } while (read_retry(&time_seqlock, seq));                      │
│      return ts;                                                     │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 버전 카운터 기반 읽기 무잠금
• 읽기: 락 없음, 재시도 가능
• 쓰기: 버전 홀수→짝수 변경
• 버전: 홀수=쓰기 중, 짝수=안정
• 적합: 읽기 매우 많음, 쓰기 드묾
• 주의: 포인터, 긴 읽기 부적합
```
