# 247. 퓌텍스 (Futex)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 공간 락 + 커널 대기 조합
> 2. **가치**: uncontended 시 시스템 콜 없음, 효율적 동기화
> 3. **융합**: 뮤텍스, 조건 변수, 실시간 시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

퓌텍스(Futex, Fast Userspace Mutex)는 **락이 충돌하지 않을 때는 사용자 공간에서만 처리하고, 충돌 시에만 커널 대기 큐를 사용하는 동기화 메커니즘**이다. Linux에서 널리 사용된다.

### 💡 비유: 입장권 자동발매기
퓌텍스는 **입장권 자동발매기**와 같다. 사람이 없으면 바로 꺼내간다. 줄이 생기면 직원(커널)이 와서 관리한다. 빈 경우 시스템 콜 없이 빠르다.

### 퓌텍스 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                퓌텍스 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 아이디어】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Uncontended (충돌 없음):                                         │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  사용자 공간에서 원자적 연산만으로 락 획득/해제                 │ │ │
│  │  │  시스템 콜 없음 → 매우 빠름                                 │ │ │
│  │  │                                                         │ │ │
│  │  │  T1: atomic_exchange(lock, 0 → 1)  // 성공!               │ │ │
│  │  │      ↓                                                   │ │ │
│  │  │  임계 영역 실행                                            │ │ │
│  │  │      ↓                                                   │ │ │
│  │  │  atomic_store(lock, 0)                                   │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  Contended (충돌 발생):                                           │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  락 획득 실패 시 커널 대기 큐에 등록                          │ │ │
│  │  │  시스템 콜 futex_wait() 호출                               │ │ │
│  │  │                                                         │ │ │
│  │  │  T1: [락 보유 중]                                          │ │ │
│  │  │  T2: atomic_exchange(lock, 0 → 1)  // 실패 (1 반환)        │ │ │
│  │  │      ↓                                                   │ │ │
│  │  │  futex_wait(&lock, 1)  // 커널에서 대기                     │ │ │
│  │  │      ...                                                 │ │ │
│  │  │  T1: atomic_store(lock, 0)                               │ │ │
│  │  │  T1: futex_wake(&lock, 1)  // T2 깨움                     │ │ │
│  │  │      ↓                                                   │ │ │
│  │  │  T2: 깨어남 → 재시도 → 락 획득                               │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【퓌텍스 시스템 콜】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  futex_wait(futex_addr, expected_val, timeout):                 │ │
│  │  • *futex_addr == expected_val이면 대기                          │ │
│  │  • 아니면 즉시 반환                                               │ │
│  │  • 커널이 스레드를 대기 큐에 등록                                  │ │
│  │                                                             │ │
│  │  futex_wake(futex_addr, count):                                  │ │
│  │  • count개의 대기 스레드 깨움                                     │ │
│  │  • FUTEX_WAKE_PRIVATE: 동일 주소 공간만                          │ │
│  │  • FUTEX_WAKE: 모든 주소 공간                                     │ │
│  │                                                             │ │
│  │  futex_requeue: 대기 스레드를 다른 큐로 이동                       │ │
│  │  futex_cmp_requeue: 비교 후 이동                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                퓌텍스 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락 구현 패턴】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 간단한 퓌텍스 락                                               │ │
│  │  void lock(int* futex) {                                         │ │
│  │      // 빠른 경로: 원자적 시도                                      │ │
│  │      if (atomic_exchange(futex, 1) == 0)                        │ │
│  │          return;  // 즉시 획득                                    │ │
│  │                                                             │ │
│  │      // 느린 경로: 커널 대기                                        │ │
│  │      while (atomic_exchange(futex, 2) != 0) {                   │ │
│  │          futex_wait(futex, 2);                                  │ │
│  │      }                                                           │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  │  void unlock(int* futex) {                                       │ │
│  │      int old = atomic_exchange(futex, 0);                       │ │
│  │      if (old == 2) {                                             │ │
│  │          futex_wake(futex, 1);  // 대기자 있으면 깨움              │ │
│  │      }                                                           │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  │  상태: 0 = 해제, 1 = 해제 중(빠른 경로), 2 = 대기 중                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【조건 변수 구현】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct condvar {                                                │ │
│  │      int seq;  // 순서 번호                                        │ │
│  │  };                                                              │ │
│  │                                                             │ │
│  │  void wait(condvar* cv, mutex* mtx) {                            │ │
│  │      int seq = cv->seq;                                          │ │
│  │      mutex_unlock(mtx);                                          │ │
│  │      futex_wait(&cv->seq, seq);                                  │ │
│  │      mutex_lock(mtx);                                            │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  │  void signal(condvar* cv) {                                      │ │
│  │      cv->seq++;                                                   │ │
│  │      futex_wake(&cv->seq, 1);                                    │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【adaptive spinning】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 잠시 스핀 후 퓌텍스 대기                                        │ │
│  │  • 락이 곧 풀릴 것 같으면 스핀                                      │ │
│  │  • 오래 걸릴 것 같으면 퓌텍스 대기                                  │ │
│  │  • Linux glibc pthread_mutex: adaptive mutex                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux futex 시스템 콜】                                           │
│  ──────────────────                                                  │
│  #include <linux/futex.h>                                           │
│  #include <sys/syscall.h>                                           │
│  #include <unistd.h>                                                │
│                                                                     │
│  // 직접 futex 호출                                                   │
│  long futex_wait(int* uaddr, int val, const struct timespec* timeout) {│
│      return syscall(SYS_futex, uaddr, FUTEX_WAIT_PRIVATE, val, timeout);│
│  }                                                                  │
│                                                                     │
│  long futex_wake(int* uaddr, int val) {                             │
│      return syscall(SYS_futex, uaddr, FUTEX_WAKE_PRIVATE, val);    │
│  }                                                                  │
│                                                                     │
│  【glibc pthread_mutex (내부적으로 futex 사용)】                      │
│  ──────────────────                                                  │
│  #include <pthread.h>                                               │
│                                                                     │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│                                                                     │
│  // Uncontended: 사용자 공간 원자적 연산만                             │
│  // Contended: futex_wait/wake 사용                                  │
│  pthread_mutex_lock(&mutex);                                        │
│  // ... 임계 영역 ...                                                 │
│  pthread_mutex_unlock(&mutex);                                      │
│                                                                     │
│  【C++ std::mutex (구현체에 따라 futex 사용)】                        │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│                                                                     │
│  std::mutex mtx;                                                     │
│  std::lock_guard<std::mutex> lock(mtx);  // 내부적으로 futex          │
│                                                                     │
│  【Python threading.Lock (glibc futex 간접 사용)】                   │
│  ──────────────────                                                  │
│  import threading                                                   │
│                                                                     │
│  lock = threading.Lock()                                            │
│  lock.acquire()  # 내부적으로 pthread_mutex → futex                  │
│  # ... 임계 영역 ...                                                 │
│  lock.release()                                                      │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시나리오              순수 스핀락    퓌텍스        뮤텍스          │ │
│  │  ────────              ────────      ──────        ──────          │ │
│  │  Uncontended          50ns          50ns          50ns            │ │
│  │  짧은 대기 (1μs)       1μs (CPU)      1μs (CPU)     1μs            │ │
│  │  긴 대기 (1ms)         1ms (CPU)      1ms (절전)    1ms            │ │
│  │  높은 경합              CPU 100%      CPU 낮음      CPU 낮음        │ │
│  │                                                             │ │
│  │  퓌텍스: 최고의 균형 (빠른 경로 + 효율적 대기)                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

