# 290. 락 프리 큐 (Lock-Free Queue)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 없이 동시성을 보장하는 큐
> 2. **가치**: 높은 동시성, 교착상태 면역
> 3. **융합**: CAS, ABA 문제, 메모리 배리어와 연관

---

## Ⅰ. 개요

### 개념 정의

락 프리 큐(Lock-Free Queue)는 **락을 사용하지 않고 CAS 등의 원자적 연산으로 동시성을 보장하는 큐 자료구조**다. 교착상태에 면역이고 높은 동시성을 제공한다.

### 💡 비유: 회전문
락 프리 큐는 **회전문**과 같다. 한 번에 한 사람이 지나가지만, 줄을 서서 기다릴 필요 없다. 차례가 되면 자연스럽게 지나간다.

### 락 프리 큐 구조

│  【기본 구조: 연결 리스트 기반】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Head ──► [Node1] ──► [Node2] ──► [Node3] ──► Tail           │ │
│  │              ▲                          ▲                          │ │
│  │              │                          │                          │ │
│  │          dummy node               real nodes                     │ │
│  │                                                             │ │
│  │  핵심: Head와 Tail을 CAS로 갱신                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Michael-Scott 큐 (MS Queue)】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  enqueue:                                                          │ │
│  │  1. 새 노드 생성                                                    │ │
│  │  2. Tail의 next를 null → new_node로 CAS                            │ │
│  │  3. 성공하면 Tail을 old_tail → new_node로 CAS                       │ │
│  │  4. 실패하면 2부터 재시도                                             │ │
│  │                                                             │ │
│  │  dequeue:                                                          │ │
│  │  1. Head 읽기                                                       │ │
│  │  2. Head.next 읽기                                                  │ │
│  │  3. Head를 old_head → next로 CAS                                     │ │
│  │  4. 성공하면 next의 데이터 반환                                        │ │
│  │  5. 실패하면 1부터 재시도                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 락 기반 큐】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  락 기반 큐:                                                         │ │
│  │  • mutex/lock으로 보호                                               │ │
│  │  • 단순한 구현                                                       │ │
│  │  • 교착상태 가능                                                      │ │
│  │  • 스레드가 블록될 수 있음                                              │ │
│  │                                                             │ │
│  │  락 프리 큐:                                                          │ │
│  │  • CAS로 보호                                                        │ │
│  │  • 복잡한 구현                                                       │ │
│  │  • 교착상태 면역                                                      │ │
│  │  • 스레드가 항상 진전                                                   │ │

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 프리 큐 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구현 난이도】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. ABA 문제:                                                       │ │
│  │     • 노드가 재사용되면 CAS가 잘못 성공                                  │ │
│  │     • 해결: 버전 번호, hazard pointer                                 │ │
│  │                                                             │ │
│  │  2. 메모리 관리:                                                     │ │
│  │     • 언제 노드를 해제할지 결정 어려움                                    │ │
│  │     • 해결: hazard pointer, RCU, epoch-based                        │ │
│  │                                                             │ │
│  │  3. 메모리 배리어:                                                    │ │
│  │     • 올바른 순서 보장 필요                                            │ │
│  │     • 해결: acquire/release semantics                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【변형】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. MPSC (Multi-Producer Single-Consumer):                        │ │
│  │     • 여러 생산자, 하나의 소비자                                          │ │
│  │     • 더 간단한 구현                                                  │ │
│  │                                                             │ │
│  │  2. SPSC (Single-Producer Single-Consumer):                       │ │
│  │     • 하나의 생산자, 하나의 소비자                                        │ │
│  │     • 가장 간단, 가장 빠름                                              │ │
│  │                                                             │ │
│  │  3. Bounded Lock-Free Queue:                                       │ │
│  │     • 고정 크기 배열 사용                                               │ │
│  │     • 메모리 관리 단순                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 특성】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 저경합: 락 기반보다 약간 느림 (CAS 오버헤드)                           │ │
│  │  • 고경합: 락 기반보다 훨씬 빠름 (블록 없음)                               │ │
│  │  • 확장성: 스레드 증가에도 성능 유지                                      │ │
│  │  • 지연: 예측 가능한 지연                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ Lock-Free Queue (간단 버전)】                                  │
│  ──────────────────                                                  │
│  template<typename T>                                               │
│  class LockFreeQueue {                                              │
│      struct Node {                                                  │
│          T data;                                                    │
│          std::atomic<Node*> next{nullptr};                          │
│          Node(const T& d) : data(d) {}                              │
│      };                                                             │
│                                                                     │
│      std::atomic<Node*> head;                                       │
│      std::atomic<Node*> tail;                                       │
│                                                                     │
│  public:                                                            │
│      LockFreeQueue() {                                              │
│          Node* dummy = new Node(T{});                               │
│          head.store(dummy);                                         │
│          tail.store(dummy);                                         │
│      }                                                              │
│                                                                     │
│      void enqueue(const T& data) {                                  │
│          Node* node = new Node(data);                               │
│          while (true) {                                             │
│              Node* t = tail.load();                                 │
│              Node* next = t->next.load();                           │
│              if (t == tail.load()) {                                │
│                  if (next == nullptr) {                              │
│                      if (t->next.compare_exchange_weak(next, node)) {│
│                          tail.compare_exchange_weak(t, node);        │
│                          return;                                     │
│                      }                                              │
│                  } else {                                            │
│                      tail.compare_exchange_weak(t, next);            │
│                  }                                                  │
│              }                                                      │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      bool dequeue(T& result) {                                      │
│          while (true) {                                             │
│              Node* h = head.load();                                 │
│              Node* t = tail.load();                                 │
│              Node* next = h->next.load();                           │
│              if (h == head.load()) {                                │
│                  if (h == t) {                                       │
│                      if (next == nullptr) return false;              │
│                      tail.compare_exchange_weak(t, next);            │
│                  } else {                                            │
│                      result = next->data;                            │
│                      if (head.compare_exchange_weak(h, next)) {      │
│                          delete h;  // 실제로는 위험!                    │
│                          return true;                                │
│                      }                                              │
│                  }                                                  │
│              }                                                      │
│          }                                                          │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【Java ConcurrentLinkedQueue】                                      │
│  ──────────────────                                                  │
│  import java.util.concurrent.ConcurrentLinkedQueue;                 │
│                                                                     │
│  ConcurrentLinkedQueue<String> queue = new ConcurrentLinkedQueue<>();│
│                                                                     │
│  // 생산자                                                            │
│  queue.offer("data");  // 락 프리                                     │
│                                                                     │
│  // 소비자                                                            │
│  String data = queue.poll();  // 락 프리                              │
│                                                                     │
│  【C++ boost::lockfree::queue】                                       │
│  ──────────────────                                                  │
│  #include <boost/lockfree/queue.hpp>                                │
│                                                                     │
│  boost::lockfree::queue<int> q{128};  // 고정 크기                     │
│                                                                     │
│  q.push(42);        // 락 프리                                        │
│  int value;                                                          │
│  q.pop(value);      // 락 프리                                        │
│                                                                     │
│  【Go Lock-Free Queue (SPSC)】                                        │
│  ──────────────────                                                  │
│  type Node struct {                                                 │
│      value interface{}                                              │
│      next  unsafe.Pointer                                           │
│  }                                                                  │
│                                                                     │
│  type Queue struct {                                                │
│      head unsafe.Pointer                                            │
│      tail unsafe.Pointer                                            │
│  }                                                                  │
│                                                                     │
│  func (q *Queue) Enqueue(v interface{}) {                           │
│      n := &Node{value: v}                                           │
│      for {                                                          │
│          tail := atomic.LoadPointer(&q.tail)                        │
│          next := atomic.LoadPointer(&(*Node)(tail).next)            │
│          if tail == atomic.LoadPointer(&q.tail) {                   │
│              if next == nil {                                        │
│                  if atomic.CompareAndSwapPointer(                   │
│                      &(*Node)(tail).next, nil, unsafe.Pointer(n)) { │
│                      atomic.CompareAndSwapPointer(&q.tail,          │
│                          tail, unsafe.Pointer(n))                   │
│                      return                                         │
│                  }                                                  │
│              }                                                      │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Disruptor (고성능 대안)】                                            │
│  ──────────────────                                                  │
│  // LMAX Disruptor: 사실상 락 프리                                      │
│  // Ring buffer + 시퀀스                                              │
│  // 매우 높은 처리량 (수백만 ops/sec)                                     │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  큐 타입                스레드 수    처리량 (ops/sec)              │ │
│  │  ────────                ──────      ─────────────              │ │
│  │  std::queue + mutex     1          5M                            │ │
│  │  std::queue + mutex     8          2M (경합)                     │ │
│  │  lock-free queue        1          4M                            │ │
│  │  lock-free queue        8          10M (확장성)                   │ │
│  │  Disruptor              8          50M+                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【사용 시나리오】                                                      │
│  ──────────────────                                                  │
│  • 고빈도 메시지 큐                                                      │
│  • 이벤트 시스템                                                        │
│  • 로깅 파이프라인                                                        │
│  • 네트워크 패킷 처리                                                    │
│  • 작업 큐                                                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
