# 245. 웨이트 프리 (Wait-Free)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 연산이 유한 시간 내 완료 보장
> 2. **가치**: 가장 강력한 비블로킹 보장
> 3. **융합**: 락 프리, 원자적 연산, 실시간 시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

웨이트 프리(Wait-Free)는 **모든 스레드가 유한 시간 내에 연산을 완료함을 보장하는 비블로킹 동기화**다. 어떤 스레드도 다른 스레드의 행동에 의해 무한 대기하지 않는다.

### 💡 비유: 회전문 개찰구
웨이트 프리는 **회전문 개찰구**와 같다. 누구나 지나가면 바로 통과한다. 앞사람이 멈춰도 내 회전문은 돌아간다. 모두가 제한 시간 내 통과 보장된다.

### 웨이트 프리 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                웨이트 프리 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【비블로킹 계층】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Obstruction-Free (장애물 프리)                                 │ │
│  │  ├─ 충돌 없으면 진행 보장                                         │ │
│  │  ├─ 다른 스레드가 멈추면 완료                                      │ │
│  │  └─ 가장 약한 보장                                               │ │
│  │                                                             │ │
│  │  Lock-Free (락 프리)                                            │ │
│  │  ├─ 어떤 스레드든 항상 진행                                        │ │
│  │  ├─ 시스템 전체 진행 보장                                          │ │
│  │  └─ CAS 루프로 구현                                              │ │
│  │                                                             │ │
│  │  Wait-Free (웨이트 프리)                                         │ │
│  │  ├─ 모든 스레드가 유한 시간 내 완료                                 │ │
│  │  ├─ 개별 스레드 진행 보장                                          │ │
│  │  └─ 가장 강한 보장                                                │ │
│  │                                                             │ │
│  │  강도: Obstruction < Lock-Free < Wait-Free                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Lock-Free vs Wait-Free】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Lock-Free:                                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │  T1: CAS() ─ 실패 ─ CAS() ─ 실패 ─ CAS() ─ 성공           │   │ │
│  │  │  T2: CAS() ─ 실패 ─ CAS() ─ 실패 ─ CAS() ─ 실패 ─ ...     │   │ │
│  │  │  T3: CAS() ─ 성공                                        │   │ │
│  │  │                                                         │   │ │
│  │  │  특징: 어떤 스레드는 계속 실패할 수 있음                    │   │ │
│  │  │        하지만 시스템 전체는 진행                           │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  │  Wait-Free:                                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │  T1: 연산 ─ 완료 (항상 k단계 내)                          │   │ │
│  │  │  T2: 연산 ─ 완료 (항상 k단계 내)                          │   │ │
│  │  │  T3: 연산 ─ 완료 (항상 k단계 내)                          │   │ │
│  │  │                                                         │   │ │
│  │  │  특징: 모든 스레드가 보장된 단계 내 완료                    │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                웨이트 프리 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Wait-Free의 종류】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Wait-Free (일반):                                                │ │
│  │  • 모든 연산이 유한 시간 내 완료                                   │ │
│  │  • 단계 수의 상한이 존재                                           │ │
│  │                                                             │ │
│  │  Wait-Free Bounded ( bounded):                                   │ │
│  │  • 단계 수가 미리 알려진 상한                                      │ │
│  │  • 더 강력한 보장                                                 │ │
│  │                                                             │ │
│  │  Wait-Free Population-Oblivious:                                  │ │
│  │  • 단계 수가 스레드 수와 무관                                      │ │
│  │  • 스레드 수를 모르는 경우에도 보장                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【구현 기법】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 원자적 읽기-수정-쓰기 (RMW)                                    │ │
│  │     • fetch_add, fetch_sub                                        │ │
│  │     • 단일 원자적 연산으로 완료                                    │ │
│  │                                                             │ │
│  │  2. 도움주기 (Helping)                                             │ │
│  │     • 느린 스레드의 작업을 다른 스레드가 대신 완료                  │ │
│  │     • 모든 스레드가 진행 상태 공유                                  │ │
│  │                                                             │ │
│  │  3. 원자적 스냅샷                                                   │ │
│  │     • 모든 상태를 원자적으로 읽기                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【활용 사례】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 실시간 시스템: 엄격한 시간 보장                                  │ │
│  │  • 카운터: fetch_add (wait-free)                                  │ │
│  │  • 큐: Wait-Free 큐 (MPSC는 쉬움, MPMC은 어려움)                  │ │
│  │  • 해시 테이블: Wait-Free 읽기                                     │ │
│  │  • 이벤트 카운터                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Wait-Free 카운터】                                                  │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│                                                                     │
│  std::atomic<int> counter{0};                                        │
│                                                                     │
│  // Wait-Free: 항상 한 번의 원자적 연산으로 완료                        │
│  int increment() {                                                  │
│      return counter.fetch_add(1, std::memory_order_relaxed) + 1;   │
│  }                                                                  │
│                                                                     │
│  int decrement() {                                                  │
│      return counter.fetch_sub(1, std::memory_order_relaxed) - 1;   │
│  }                                                                  │
│                                                                     │
│  int get() {                                                        │
│      return counter.load(std::memory_order_relaxed);               │
│  }                                                                  │
│                                                                     │
│  【Wait-Free MPSC 큐 (단일 소비자)】                                    │
│  ──────────────────                                                  │
│  template<typename T>                                               │
│  class WaitFreeMPSCQueue {                                          │
│      struct Node {                                                  │
│          T data;                                                      │
│          std::atomic<Node*> next{nullptr};                          │
│      };                                                              │
│      std::atomic<Node*> head;  // 생산자용                            │
│      Node* tail;  // 소비자 전용 (단일 스레드)                         │
│  public:                                                            │
│      void enqueue(T value) {                                        │
│          Node* node = new Node{value};                              │
│          // 원자적으로 head 교체                                       │
│          Node* prev = head.exchange(node, std::memory_order_acq_rel);│
│          prev->next.store(node, std::memory_order_release);         │
│      }                                                              │
│      bool dequeue(T& result) {                                      │
│          Node* next = tail->next.load(std::memory_order_acquire);   │
│          if (!next) return false;                                   │
│          result = std::move(next->data);                            │
│          delete tail;                                                │
│          tail = next;                                                │
│          return true;                                                 │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【Wait-Free vs Lock-Free 선택 기준】                                  │
│  ──────────────────                                                  │
│  • Wait-Free 선택:                                                   │
│    - 실시간 시스템                                                   │
│    - 엄격한 시간 보장 필요                                            │
│    - 단순 연산 (카운터, 플래그)                                       │
│                                                                     │
│  • Lock-Free 선택:                                                   │
│    - 일반 고성능 시스템                                               │
│    - 구현 복잡도 허용                                                 │
│    - 복잡한 자료구조                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
