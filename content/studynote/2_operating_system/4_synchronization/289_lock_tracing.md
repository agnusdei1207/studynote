# 289. 락 트레이싱 (Lock Tracing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 획득/해제 이벤트 기록
> 2. **가치**: 교착상태 분석, 성능 디버깅
> 3. **융합**: 락 통계, lockdep, ftrace와 연관

---

## Ⅰ. 개요

### 개념 정의

락 트레이싱(Lock Tracing)은 **락의 획득, 해제, 대기 등 모든 이벤트를 기록하는 기법**이다. 교착상태 분석, 성능 병목 식별, 디버깅에 사용된다.

### 💡 비유: 블랙박스
락 트레이싱은 **비행기 블랙박스**와 같다. 모든 조작을 기록한다. 문제 발생 시 기록을 분석해서 원인을 찾는다.

### 락 트레이싱 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 트레이싱 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기록 이벤트】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  이벤트 타입:                                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • lock_acquire: 락 획득 시도                               │ │ │
│  │  │  • lock_acquired: 락 획득 완료                              │ │ │
│  │  │  • lock_release: 락 해제                                   │ │ │
│  │  │  • lock_wait_start: 대기 시작                               │ │ │
│  │  │  • lock_wait_end: 대기 종료                                 │ │ │
│  │  │  • lock_contention: 경합 발생                               │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  기록 정보:                                                         │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 타임스탬프 (timestamp)                                   │ │ │
│  │  │  • 스레드 ID (thread_id)                                    │ │ │
│  │  │  • 락 식별자 (lock_id/address)                              │ │ │
│  │  │  • 락 이름/클래스 (lock_name/class)                          │ │ │
│  │  │  • 호출 스택 (call_stack)                                   │ │ │
│  │  │  • 대기 시간 (wait_time)                                    │ │ │
│  │  │  • 보유 시간 (hold_time)                                    │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【트레이스 출력 예시】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  timestamp           thread   event         lock      duration  │ │
│  │  ──────────────────  ──────   ────────────  ───────   ────────  │ │
│  │  12:34:56.123456    T1       lock_acquire   0x7fff...            │ │
│  │  12:34:56.123460    T1       lock_acquired  0x7fff...  4μs       │ │
│  │  12:34:56.125000    T2       lock_acquire   0x7fff...            │ │
│  │  12:34:56.125010    T2       lock_wait       0x7fff...            │ │
│  │  12:34:56.130000    T1       lock_release    0x7fff...  5.5ms    │ │
│  │  12:34:56.130005    T2       lock_acquired  0x7fff...  5ms       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux ftrace 사용】                                                │
│  ──────────────────                                                  │
│  # 모든 락 이벤트 추적                                                  │
│  echo 1 > /sys/kernel/debug/tracing/events/lock/enable              │
│  echo 1 > /sys/kernel/debug/tracing/tracing_on                      │
│                                                                     │
│  # 특정 락만 필터                                                      │
│  echo 'mutex.*' > /sys/kernel/debug/tracing/set_event               │
│                                                                     │
│  # 실시간 확인                                                         │
│  cat /sys/kernel/debug/tracing/trace_pipe                           │
│                                                                     │
│  【Linux perf】                                                       │
│  ──────────────────                                                  │
│  # 락 이벤트 기록                                                       │
│  perf record -e 'syscalls:sys_enter_futex' -a sleep 10              │
│  perf report                                                         │
│                                                                     │
│  # 락 대기 시간 분석                                                    │
│  perf lock record -- sleep 10                                       │
│  perf lock report                                                   │
│                                                                     │
│  【eBPF/BCC】                                                         │
│  ──────────────────                                                  │
│  # 락 분석 도구                                                        │
│  /usr/share/bcc/tools/lockstat                                     │
│  /usr/share/bcc/tools/offcputime                                   │
│                                                                     │
│  # 커스텀 eBPF                                                         │
│  // futex 추적                                                        │
│  bpftrace -e 'kprobe:do_futex { printf("futex: %s\n", comm); }'     │
│                                                                     │
│  【Java Flight Recorder】                                             │
│  ──────────────────                                                  │
│  # 시작                                                               │
│  jcmd <pid> JFR.start name=lock_trace settings=profile              │
│                                                                     │
│  # 덤프                                                               │
│  jcmd <pid> JFR.dump name=lock_trace filename=lock_trace.jfr        │
│                                                                     │
│  # 분석                                                               │
│  # JDK Mission Control으로 .jfr 파일 열기                             │
│                                                                     │
│  【커스텀 트레이싱 구현】                                                 │
│  ──────────────────                                                  │
│  class LockTracer {                                                 │
│      struct LockEvent {                                             │
│          uint64_t timestamp;                                        │
│          std::thread::id thread_id;                                 │
│          void* lock_addr;                                           │
│          const char* event_type;                                    │
│          uint64_t duration_ns;                                      │
│      };                                                             │
│                                                                     │
│      std::vector<LockEvent> events;                                 │
│      std::mutex events_mutex;                                       │
│                                                                     │
│      void record_acquire(void* lock) {                              │
│          LockEvent e;                                               │
│          e.timestamp = get_timestamp();                            │
│          e.thread_id = std::this_thread::get_id();                 │
│          e.lock_addr = lock;                                        │
│          e.event_type = "acquire";                                  │
│          std::lock_guard<std::mutex> lk(events_mutex);              │
│          events.push_back(e);                                       │
│      }                                                              │
│                                                                     │
│      void record_release(void* lock, uint64_t hold_time) {          │
│          LockEvent e;                                               │
│          e.timestamp = get_timestamp();                            │
│          e.thread_id = std::this_thread::get_id();                 │
│          e.lock_addr = lock;                                        │
│          e.event_type = "release";                                  │
│          e.duration_ns = hold_time;                                 │
│          std::lock_guard<std::mutex> lk(events_mutex);              │
│          events.push_back(e);                                       │
│      }                                                              │
│                                                                     │
│      void dump(const char* filename) {                              │
│          std::ofstream out(filename);                               │
│          for (const auto& e : events) {                             │
│              out << e.timestamp << "," << e.thread_id << ","        │
│                  << e.lock_addr << "," << e.event_type              │
│                  << "," << e.duration_ns << "\n";                   │
│          }                                                          │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【트레이스 분석】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분석 항목:                                                         │ │
│  │  • 핫 락: 가장 자주 사용되는 락                                       │ │
│  │  • 긴 대기: 평균/최대 대기 시간                                       │ │
│  │  • 긴 보유: 평균/최대 보유 시간                                       │ │
│  │  • 경합률: 대기 발생 비율                                            │ │
│  │  • 데드락 패턴: 순환 대기                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  도구            오버헤드    기능        프로덕션 적합               │ │
│  │  ────────        ──────    ──────      ─────────────            │ │
│  │  ftrace          ~5%       기본        주의                      │ │
│  │  lockdep         ~10%      종합        권장 안 함                 │ │
│  │  eBPF            ~1-2%     유연        가능                       │ │
│  │  커스텀           변동       제한적       가능                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【사용 가이드】                                                        │
│  ──────────────────                                                  │
│  • 개발: lockdep 활성화 권장                                           │
│  • 테스트: ftrace, eBPF 사용                                          │
│  • 프로덕션: eBPF로 최소 오버헤드                                        │
│  • 문제 발생 시: 상세 트레이싱 활성화                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
