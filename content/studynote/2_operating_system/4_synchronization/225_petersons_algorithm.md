# 피터슨의 해결책 (Peterson's Algorithm)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 피터슨의 알고리즘 (Peterson's Algorithm)은 두 개의 프로세스가 임계 구역 문제를 해결하기 위해 고안된 **소프트웨어 기반의 동기화 알고리즘**이다.
> 2. **가치**: 상호 배제, 진행, 한정된 대기의 3조건을 모두 완벽하게 만족하는 고전적이고 우아한 해결책으로, 동기화의 원리를 이해하는 데 결정적인 역할을 한다.
> 3. **융합**: 현대의 멀티코어 환경에서는 CPU의 비순차적 실행(Out-of-order Execution)으로 인해 직접 사용하기는 어렵지만, 모든 동기화 소프트웨어 설계의 근간이 된다.

+++

## Ⅰ. 피터슨 알고리즘의 개요

### 1. 정의
- 두 개의 프로세스($P_0, P_1$)가 공유 변수를 사용하여 임계 구역에 안전하게 진입할 수 있도록 설계된 알고리즘이다.

### 2. 공유 변수
- `boolean flag[2]`: "나 들어갈 준비 됐어!"를 나타냄. (초기값 false)
- `int turn`: "지금은 네 차례야."를 나타냄.

### 3. 💡 비유: '양보하는 두 친구'
- 복도가 좁아 한 명만 지나갈 수 있을 때, 철수는 "나 갈래(flag)"라고 말한 뒤 영희에게 "네가 먼저 가(turn)"라고 양보한다. 영희도 갈 준비가 되었다면 영희가 먼저 가고, 영희가 갈 생각이 없다면 철수가 바로 지나가는 지극히 예의 바른 소통 방식이다.

- **📢 섹션 요약 비유**: 두 명의 신사가 좁은 문 앞에서 서로의 의사를 확인하고 기꺼이 양보하며 질서를 지키는 모습입니다.

+++

## Ⅱ. 알고리즘 구조 및 작동 원리 (Deep Dive)

프로세스 $i$ (상대방은 $j$)의 관점에서 코드를 분석한다.

### 1. 알고리즘 코드 (C-like)
```c
do {
    flag[i] = true;           // 1. 진입 의사 표시
    turn = j;                 // 2. 상대방에게 우선권 양보
    
    // 3. 대기 조건: 상대방이 준비됐고(flag[j]), 아직 상대방 차례(turn == j)라면 대기
    while (flag[j] && turn == j); 

        /* 임계 구역 (Critical Section) */

    flag[i] = false;          // 4. 퇴장: 의사 표시 철회

    /* 나머지 구역 (Remainder Section) */
} while (true);
```

### 2. ASCII 작동 시나리오
```text
    P0 실행                       P1 실행                       상태 변화
    ───────────────────           ───────────────────           ──────────────────
    flag[0] = true;                                             flag=[T,F], turn=?
                                  flag[1] = true;               flag=[T,T], turn=?
    turn = 1;                                                   flag=[T,T], turn=1
                                  turn = 0;                     flag=[T,T], turn=0
    
    while(flag[1] && turn==1)     while(flag[0] && turn==0)
    => (T && F) = False           => (T && T) = True
    [ P0 진입!! ]                 [ P1 대기... ]               P0가 CS 수행 중
    
    flag[0] = false;                                            flag=[F,T], turn=0
                                  [ P1 진입!! ]                 P1이 CS 수행 중
```

### 3. 3조건 증명
- **상호 배제**: `flag[0] && flag[1]`이 true일 때 `turn`은 0이거나 1일 수밖에 없으므로 한 명은 반드시 `while` 문에 걸린다.
- **진행**: 한 프로세스가 끝나면 `flag`를 false로 바꾸므로 기다리던 상대방이 즉시 들어간다.
- **한정된 대기**: 진입하려는 프로세스는 상대방에게 한 번 양보(`turn = j`)하므로, 상대방이 들어갔다 나온 뒤 다시 들어가려 해도 결국은 내 차례가 오게 된다.

- **📢 섹션 요약 비유**: "가고 싶다"는 깃발을 들고, "네 차례야"라고 팻말을 넘겨주는 명확한 신호 체계입니다.

## Ⅲ. 현대 컴퓨팅에서의 한계점

### 1. 비순차적 실행 (Out-of-order Execution)
- 현대 CPU는 성능 최적화를 위해 명령어 순서를 바꾼다. `flag[i] = true`와 `turn = j`의 순서가 바뀌면 상호 배제가 깨질 수 있다.

### 2. 메모리 가시성 (Memory Visibility)
- 한 코어에서 바꾼 `flag` 값이 다른 코어의 캐시나 메모리에 즉시 반영되지 않을 수 있다.

### 3. 해결책: 메모리 장벽 (Memory Barrier)
- 순서 변경을 막고 가시성을 보장하기 위해 특수 명령어(Memory Fence)를 함께 사용해야 한다.

- **📢 섹션 요약 비유**: 규칙이 아무리 좋아도 선수가 멋대로 순서를 바꾸거나 심판이 못 보면 무용지물이 되는 것과 같습니다.

+++

## Ⅳ. 실무 적용 및 알고리즘 인사이트 (Strategy)

### 1. 실무 시나리오: 저사양 임베디드 동기화
- **상황**: 하드웨어적인 원자적 명령어(Test-and-Set 등)를 지원하지 않는 아주 단순한 8비트 MCU 환경.
- **판단**: 복잡한 뮤텍스 라이브러리를 올리기 부담스럽다.
- **해결**: 피터슨 알고리즘의 원리를 활용하여 두 태스크 간의 충돌을 방지하는 소프트웨어 로직을 직접 구현할 수 있다. (단, 컴파일러 최적화 방지 키워드 `volatile` 필수 사용)

### 2. 기술사적 인사이트: 알고리즘의 진화
- 피터슨 알고리즘은 2개의 프로세스만 지원한다는 한계가 있다. 이를 $N$개의 프로세스로 확장한 것이 **빵집 알고리즘 (Lamport's Bakery Algorithm)**이다. 모든 동기화 알고리즘의 뿌리를 이해하려면 피터슨 알고리즘의 상호 양보(Deference) 철학을 반드시 이해해야 한다.

- **📢 섹션 요약 비유**: 구구단을 알아야 고등 수학을 풀 수 있듯, 피터슨 알고리즘은 동기화라는 복잡한 학문의 구구단입니다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **동기화 원리 체득**: 락(Lock)이 내부적으로 어떻게 '의사 표시'와 '양보'를 조합하는지 이해.
- **논리적 사고 향상**: 병행 프로세스 간의 상태 전이를 추적하는 정교한 논리 훈련.

### 2. 미래 전망
직접적인 사용은 줄었지만, 피터슨 알고리즘의 정신은 현대의 **Lock-free 자료구조** 설계에 여전히 살아있다. 또한 소프트웨어 검증(Formal Verification) 분야에서 알고리즘의 무결성을 증명하는 대표적인 예제로 활용되며, 안전성이 극도로 중요한 시스템의 로직 검증에 중요한 지표가 되고 있다.

- **📢 섹션 요약 비유**: 클래식 음악이 현대 음악의 기초가 되듯, 피터슨 알고리즘은 운영체제의 영원한 고전입니다.

### 📌 관련 개념 맵 (Knowledge Graph)
- **임계 구역 문제 해결의 3조건**: 피터슨 알고리즘이 충족하는 3가지 목표.
- **메모리 장벽 (Memory Barrier)**: 피터슨 알고리즘을 현대 CPU에서 구동하기 위한 필수 장치.
- **빵집 알고리즘 (Bakery Algorithm)**: 피터슨 알고리즘의 $N$개 프로세스 확장판.
