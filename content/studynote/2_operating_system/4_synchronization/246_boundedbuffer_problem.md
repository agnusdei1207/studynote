# 246. 배리어 (Barrier)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스레드들이 동기화 지점에 도달할 때까지 대기
> 2. **가치**: 병렬 계산의 단계적 동기화
> 3. **융합**: 조건 변수, 메모리 펜스, 페이즈 배리어와 연관

---

## Ⅰ. 개요

### 개념 정의

배리어(Barrier)는 **여러 스레드가 특정 지점(배리어)에 도달할 때까지 대기한 동기화 메커니즘**이다. 모든 스레드가 도달해야만 다음 단계로 진행할 수 있다.

### 💡 비유: 마라톤 출발선
배리어는 **마라톤 출발선**과 같다. 모든 선수가 출발선에 서야 한다. 다 모이면 출발한다. 누구도 먼저 가지 못한다.

### 배리어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                배리어 구조                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 동작】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  스레드 1  ────────┐                                     │ │
│  │                    │                                     │ │
│  │  스레드 2  ────────┼───►  [배리어]  ───►  다음 단계         │ │
│  │                    │         ▲                              │ │
│  │  스레드 3  ────────┘         │                              │ │
│  │                              │                              │ │
│  │  스레드 4  ────────────────┘                              │ │
│  │                                                             │ │
│  │  특징:                                                          │ │
│  │  • 모든 스레드가 도달 시까지 대기                              │ │
│  │  • 마지막 스레드 도달 시 전원 해제                            │ │
│  │  • 동시에 다음 단계로 진행                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【배리어 vs 조건 변수】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  조건 변수:                                                       │ │
│  │  • 특정 조건 만족 시 대기 해제                                 │ │
│  │  • 순차적 알림 (signal)                                        │ │
│  │  • 일부 스레드만 대기에서 풀려날 수 있음                      │ │
│  │                                                             │ │
│  │  배리어:                                                          │ │
│  │  • 모든 스레드 도달 시까지 대기                                 │ │
│  │  • 동시 해제 (broadcast)                                         │ │
│  │  • 모든 스레드가 동시에 해제                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【활용: 병렬 계산 단계 동기화】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단계 1: 모든 스레드가 데이터 분할 완료까지 대기                  │ │
│  │  단계 2: 모든 스레드가 병렬 계산 완료까지 대기                  │ │
│  │  단계 3: 모든 스레드가 결과 집계 완료까지 대기                  │ │
│  │  단계 4: 최종 결과 출력                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                배리어 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【배리어 구현 방식】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 카운터 기반:                                                   │ │
│  │     atomic_int count = 0;                                          │ │
│  │     const int N = 스레드 수;                                        │ │
│  │     // 각 스레드:                                                  │ │
│  │     if (++count == N) {                                            │ │
│  │         // 마지막 스레드: 모두 깨움                                 │ │
│  │         broadcast_all();                                            │ │
│  │     } else {                                                       │ │
│  │         // 대기                                                     │ │
│  │         wait();                                                     │ │
│  │     }                                                               │ │
│  │                                                             │ │
│  │  2. 세마포어 기반:                                                │ │
│  │     semaphore barrier = 0;                                         │ │
│  │     mutex lock;                                                    │ │
│  │     int count = 0;                                                 │ │
│  │     // 진입:                                                       │ │
│  │     lock();                                                         │ │
│  │     if (++count == N) {                                            │ │
│  │         unlock();                                                   │ │
│  │         for (i=0; i<N-1; i++) V(barrier);                          │ │
│  │     } else {                                                       │ │
│  │         unlock();                                                   │ │
│  │         P(barrier);                                                 │ │
│  │     }                                                               │ │
│  │                                                             │ │
│  │  3. Sense-Reversing 배리어:                                         │ │
│  │     // 재사용 가능한 배리어                                         │ │
│  │     // 여러 번 사용 가능                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순환 배리어 (Cyclic Barrier)】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 여러 번 재사용 가능                                             │ │
│  │  • 각 라운드마다 모든 스레드 동기화                                 │ │
│  │  • 반복적 병렬 알고리즘에 유용                                      │ │
│  │                                                             │ │
│  │  예: 행렬 곱셈의 여러 번 반복                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【배리어 vs 래치 (Latch)】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  배리어:                                                            │ │
│  │  • 모든 스레드가 대기                                               │ │
│  │  • 동시 해제                                                        │ │
│  │  • 재사용 가능 (순환 배리어)                                        │ │
│  │                                                             │ │
│  │  래치:                                                              │ │
│  │  • 하나의 스레드가 대기                                             │ │
│  │  • 카운터 0 되면 해제                                               │ │
│  │  • 일회성 (CountDownLatch)                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【POSIX pthread_barrier】                                            │
│  ──────────────────                                                  │
│  #include <pthread.h>                                               │
│                                                                     │
│  pthread_barrier_t barrier;                                         │
│  pthread_barrierattr_t attr;                                        │
│                                                                     │
│  // 초기화                                                           │
│  pthread_barrier_init(&barrier, &attr, 4);  // 4개 스레드            │
│                                                                     │
│  // 스레드에서 사용                                                  │
│  void* thread_func(void* arg) {                                     │
│      // 작업...                                                       │
│      pthread_barrier_wait(&barrier);  // 모두 도달 시까지 대기        │
│      // 다음 단계...                                                   │
│  }                                                                  │
│                                                                     │
│  【C++ std::barrier (C++20)】                                         │
│  ──────────────────                                                  │
│  #include <barrier>                                                  │
│  #include <iostream>                                                 │
│  #include <vector>                                                  │
│  #include <thread>                                                   │
│                                                                     │
│  int main() {                                                        │
│      const int n = 4;                                                │
│      auto on_completion = []() noexcept {                            │
│          std::cout << "All threads reached barrier\n";               │
│      };                                                              │
│      std::barrier sync_point(n, on_completion);                     │
│                                                                     │
│      auto work = [&](int id) {                                      │
│          std::cout << "Thread " << id << " phase 1\n";              │
│          sync_point.arrive_and_wait();                               │
│          std::cout << "Thread " << id << " phase 2\n";              │
│      };                                                              │
│                                                                     │
│      std::vector<std::thread> threads;                              │
│      for (int i = 0; i < n; i++)                                    │
│          threads.emplace_back(work, i);                             │
│      for (auto& t : threads) t.join();                              │
│  }                                                                  │
│                                                                     │
│  【Java CyclicBarrier】                                              │
│  ──────────────────                                                  │
│  import java.util.concurrent.*;                                     │
│                                                                     │
│  CyclicBarrier barrier = new CyclicBarrier(4, () -> {               │
│      System.out.println("All threads reached barrier");              │
│  });                                                                 │
│                                                                     │
│  // 각 스레드에서                                                    │
│  barrier.await();  // InterruptedException, BrokenBarrierException  │
│                                                                     │
│  【Python threading.Barrier】                                        │
│  ──────────────────                                                  │
│  import threading                                                   │
│                                                                     │
│  barrier = threading.Barrier(4)                                     │
│                                                                     │
│  def worker(id):                                                    │
│      print(f"Worker {id} phase 1")                                  │
│      barrier.wait()                                                  │
│      print(f"Worker {id} phase 2")                                  │
│                                                                     │
│  【Go WaitGroup (유사한 패턴)】                                       │
│  ──────────────────                                                  │
│  var wg sync.WaitGroup                                              │
│  wg.Add(4)  // 4개 고루틴 대기                                     │
│                                                                     │
│  for i := 0; i < 4; i++ {                                          │
│      go func(id int) {                                              │
│          defer wg.Done()                                             │
│          // 작업...                                                   │
│      }(i)                                                            │
│  }                                                                  │
│  wg.Wait()  // 모두 완료 시까지 대기                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
