# 228. 메모리 배리어 (Memory Barrier)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU와 컴파일러의 메모리 연산 재배치 방지
> 2. **가치**: 메모리 가시성과 순서 보장
> 3. **융합**: happens-before, 원자적 연산, 캐시 일관성과 연관

---

## Ⅰ. 개요

### 개념 정의

메모리 배리어(Memory Barrier, Memory Fence)는 **CPU와 컴파일러가 메모리 연산의 순서를 재배치하지 못하도록 강제하는 명령어**다. happens-before 관계를 구현한다.

### 💡 비유: 출입 통제
메모리 배리어는 **건물 출입 통제**와 같다. 앞사람이 모두 들어간 후에 다음 사람이 들어갈 수 있다. 순서가 꼬이지 않는다.

### 메모리 배리어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 배리어 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【배리어 없이】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  프로그램 순서:    W(A) → W(B) → W(C)                          │ │
│  │                                                             │ │
│  │  CPU 실행 순서:   W(C) → W(A) → W(B)  (재배치됨!)              │ │
│  │                                                             │ │
│  │  다른 코어에서 보는 순서도 달라질 수 있음                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【배리어 있이】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  프로그램 순서:    W(A) → W(B) → [BARRIER] → W(C)              │ │
│  │                                                             │ │
│  │  CPU 실행 순서:   W(A) → W(B) → [BARRIER] → W(C)              │ │
│  │                   (배리어 전후 재배치 금지)                      │ │
│  │                                                             │ │
│  │  다른 코어에서도 A, B가 C보다 먼저 보임                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【배리어 유형】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  유형              효과                    용도                  │ │
│  │  ────                  ────                        ────          │ │
│  │  Load Load        읽기-읽기 순서 보장      읽기 일관성            │ │
│  │  Store Store      쓰기-쓰기 순서 보장      쓰기 순서              │ │
│  │  Load Store       읽기-쓰기 순서 보장      읽고 나서 쓰기          │ │
│  │  Store Load       쓰기-읽기 순서 보장      쓰고 나서 읽기          │ │
│  │  Full             모든 연산 순서 보장      완전 배리어            │ │
│  │                                                             │ │
│  │  Acquire          배리어 후 읽기가 위로 올라감 안 됨            │ │
│  │  Release          배리어 전 쓰기가 아래로 내려감 안 됨           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 배리어 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【아키텍처별 명령어】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86:                                                             │
│  │  • MFENCE: 전체 배리어                                           │ │
│  │  • LFENCE: 읽기 배리어                                            │ │
│  │  • SFENCE: 쓰기 배리어                                            │ │
│  │  • LOCK prefix: 원자적 연산 + 전체 배리어                         │ │
│  │                                                             │ │
│  │  ARM:                                                             │ │
│  │  • DMB: Data Memory Barrier                                      │ │
│  │  • DSB: Data Synchronization Barrier                             │ │
│  │  • ISB: Instruction Synchronization Barrier                      │ │
│  │                                                             │ │
│  │  POWER:                                                           │ │
│  │  • sync: heavyweight sync                                        │ │
│  │  • lwsync: lightweight sync                                      │ │
│  │  • eieio: ensure in-order I/O                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Acquire/Release 의미】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Acquire (읽기):                                                  │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  [이전 읽기/쓰기]                                        │ │ │
│  │  │         ↕ 이동 불가 (위로 못 올라감)                       │ │ │
│  │  │  [ACQUIRE 배리어]                                        │ │ │
│  │  │         ↓                                                │ │ │
│  │  │  [이후 읽기/쓰기]                                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  Release (쓰기):                                                  │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  [이전 읽기/쓰기]                                        │ │ │
│  │  │         ↓                                                │ │ │
│  │  │  [RELEASE 배리어]                                        │ │ │
│  │  │         ↕ 이동 불가 (아래로 못 내려감)                     │ │ │
│  │  │  [이후 읽기/쓰기]                                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널】                                                        │
│  ──────────────────                                                  │
│  // 전체 메모리 배리어                                                │
│  #define barrier() __asm__ __volatile__("":::"memory")             │
│  #define mb()    asm volatile("mfence":::"memory")                  │
│  #define rmb()   asm volatile("lfence":::"memory")                  │
│  #define wmb()   asm volatile("sfence":::"memory")                  │
│                                                                     │
│  // Acquire/Release                                                 │
│  #define smp_load_acquire(p) ...                                    │
│  #define smp_store_release(p, v) ...                                │
│                                                                     │
│  【C++ std::atomic】                                                  │
│  ──────────────────                                                  │
│  std::atomic<int> x{0};                                             │
│                                                                     │
│  // Release store                                                   │
│  x.store(42, std::memory_order_release);                            │
│                                                                     │
│  // Acquire load                                                    │
│  int v = x.load(std::memory_order_acquire);                         │
│                                                                     │
│  // 전체 배리어                                                       │
│  std::atomic_thread_fence(std::memory_order_seq_cst);               │
│                                                                     │
│  【GCC 내장 함수】                                                    │
│  ──────────────────                                                  │
│  // 전체 배리어                                                       │
│  __sync_synchronize();                                              │
│                                                                     │
│  // Acquire/Release                                                 │
│  __atomic_thread_fence(__ATOMIC_ACQUIRE);                           │
│  __atomic_thread_fence(__ATOMIC_RELEASE);                           │
│                                                                     │
│  【x86 어셈블리】                                                      │
│  ──────────────────                                                  │
│  ; 전체 배리어                                                        │
│  mfence                                                              │
│                                                                     │
│  ; LOCK 접두어로 원자적 + 배리어                                       │
│  lock addl $0, 0(%rsp)                                              │
│                                                                     │
│  【ARM 어셈블리】                                                      │
│  ──────────────────                                                  │
│  ; 데이터 메모리 배리어                                                │
│  dmb ish                                                             │
│                                                                     │
│  ; 데이터 동기화 배리어                                                │
│  dsb ish                                                             │
│                                                                     │
│  ; 명령어 동기화 배리어                                                │
│  isb                                                                 │
│                                                                     │
│  【성능 영향】                                                         │
│  ──────────────────                                                  │
│  • 배리어는 파이프라인을 플러시할 수 있음                               │
│  • 불필요한 배리어는 성능 저하                                         │
│  • x86은 강한 모델이라 배리어 필요 적음                                │
│  • ARM은 약한 모델이라 배리어 자주 필요                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
