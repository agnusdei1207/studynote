# 299. 비선점 (No Preemption)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다른 프로세스의 자원을 강제로 뺏을 수 없음
> 2. **가치**: 교착상태 조건 3
> 3. **융합**: 선점, 타임아웃, tryLock과 연관

---

## Ⅰ. 개요

### 개념 정의

비선점(No Preemption)은 **프로세스가 할당받은 자원을 스스로 해제하기 전까지 다른 프로세스가 강제로 뺏을 수 없는 속성**이다. 교착상태 발생의 4가지 필요조건 중 세 번째다.

### 💡 비유: 대여한 책
비선점은 **빌려준 책**과 같다. 친구에게 빌려준 책은 친구가 돌려줄 때까지 달라고 못 한다. 강제로 뺏을 수 없다.

### 비선점 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                비선점 구조                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【비선점 자원】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  [P1] ──► 자원 R 보유                                               │ │
│  │                                                             │ │
│  │  [P2] ──► R 요청 ──► 대기 (P1이 해제할 때까지)                         │ │
│  │                                                             │ │
│  │  P2는 R을 강제로 뺏을 수 없음!                                          │ │
│  │  P1만이 R을 해제할 수 있음                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【비선점 자원 예시】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 뮤텍스/락: 소유자만 해제 가능                                        │ │
│  │  • 프린터: 인쇄 중단 불가                                               │ │
│  │  • 파일 쓰기: 한 프로세스만 쓰기                                         │ │
│  │  • 테이프 드라이브: 전통적 비선점 자원                                    │ │
│  │  • 그래픽 카드: 특정 작업 수행 중                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【선점 가능 자원】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • CPU: 스케줄러가 선점 가능                                           │ │
│  │  • 메모리: 페이지 아웃 가능                                             │ │
│  │  • 캐시: 교체 가능                                                    │ │
│  │  • 네트워크 대역폭: 재할당 가능                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                비선점 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【무력화 방법】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 타임아웃 (Timeout):                                              │ │
│  │     • 일정 시간 후 자동 해제                                           │ │
│  │     • 무한 대기 방지                                                  │ │
│  │                                                             │ │
│  │  2. tryLock (Non-blocking):                                        │ │
│  │     • 락 획득 실패 시 즉시 반환                                         │ │
│  │     • 보유 락 해제 후 재시도                                            │ │
│  │                                                             │ │
│  │  3. 선점 가능 락 (Preemptible Locks):                               │ │
│  │     • rt_mutex: 우선순위 기반 선점                                    │ │
│  │     • 특수한 상황에서만                                                │ │
│  │                                                             │ │
│  │  4. 체크포인트와 롤백:                                                 │ │
│  │     • 상태 저장 후 선점                                                │ │
│  │     • 나중에 복구 가능                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【tryLock 패턴】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 비선점 무력화: 보유 락 해제 후 재시도                                 │ │
│  │  while (true) {                                                    │
│  │      lock(&A);                                                      │
│  │      if (tryLock(&B)) {                                            │
│  │          // A, B 모두 획득 성공                                        │ │
│  │          break;                                                      │
│  │      }                                                              │
│  │      // B 획득 실패: A를 해제하고 재시도                                  │ │
│  │      unlock(&A);                                                    │
│  │      sleep(backoff);                                               │
│  │  }                                                                  │
│  │                                                             │ │
│  │  // A를 보유하며 B를 대기하지 않음!                                       │ │
│  │  // (순간적으로만 보유)                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점 (무력화 시):                                                    │ │
│  │  • 교착상태 예방                                                      │ │
│  │  • 무한 대기 방지                                                     │ │
│  │                                                             │ │
│  │  단점 (무력화 시):                                                    │ │
│  │  • 작업 손실 가능 (롤백 필요)                                          │ │
│  │  • 오버헤드 (재시도)                                                   │ │
│  │  • 기아 가능성 (계속 실패)                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【tryLock으로 비선점 무력화】                                          │
│  ──────────────────                                                  │
│  // C++                                                              │
│  std::mutex mtxA, mtxB;                                              │
│                                                                     │
│  void safe_work() {                                                  │
│      while (true) {                                                  │
│          std::unique_lock<std::mutex> lockA(mtxA);                   │
│          if (std::try_lock(mtxB) == -1) {                            │
│              // 두 락 모두 획득 성공                                      │
│              break;                                                  │
│          }                                                          │
│          // mtxB 획득 실패: lockA 자동 해제 후 재시도                      │
│          std::this_thread::sleep_for(std::chrono::milliseconds(1)); │
│      }                                                              │
│      // 작업 수행                                                      │
│      mtxB.unlock();                                                  │
│  }                                                                  │
│                                                                     │
│  【Java ReentrantLock tryLock】                                        │
│  ──────────────────                                                  │
│  ReentrantLock lockA = new ReentrantLock();                         │
│  ReentrantLock lockB = new ReentrantLock();                         │
│                                                                     │
│  void safeWork() throws InterruptedException {                       │
│      while (true) {                                                  │
│          lockA.lock();                                               │
│          try {                                                       │
│              if (lockB.tryLock(100, TimeUnit.MILLISECONDS)) {        │
│                  try {                                               │
│                      // 두 락 모두 획득 성공                              │
│                      doWork();                                       │
│                      return;                                         │
│                  } finally {                                         │
│                      lockB.unlock();                                 │
│                  }                                                  │
│              }                                                      │
│          } finally {                                                 │
│              lockA.unlock();                                          │
│          }                                                          │
│          // 백오프 후 재시도                                              │
│          Thread.sleep(randomBackoff());                              │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【타임아웃으로 비선점 무력화】                                            │
│  ──────────────────                                                  │
│  // pthread mutex timedlock                                          │
│  struct timespec timeout;                                            │
│  timeout.tv_sec = 5;                                                 │
│  timeout.tv_nsec = 0;                                                │
│                                                                     │
│  int result = pthread_mutex_timedlock(&mutex, &timeout);             │
│  if (result == ETIMEDOUT) {                                          │
│      // 타임아웃: 락 획득 실패                                            │
│      // 복구 또는 재시도                                                 │
│  } else if (result == 0) {                                           │
│      // 락 획득 성공                                                    │
│      // 작업 수행                                                      │
│      pthread_mutex_unlock(&mutex);                                   │
│  }                                                                  │
│                                                                     │
│  【선점 가능한 자원: CPU】                                               │
│  ──────────────────                                                  │
│  // CPU는 선점 가능한 자원의 대표적인 예                                     │
│  // 스케줄러가 언제든지 프로세스에서 CPU를 뺏을 수 있음                          │
│                                                                     │
│  // 선점 vs 비선점 스케줄링                                                │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  비선점 스케줄링:                                                       │ │
│  │  • FCFS, SJF (non-preemptive)                                      │ │
│  │  • 프로세스가 자발적으로 CPU 양보                                         │ │
│  │                                                             │ │
│  │  선점 스케줄링:                                                         │ │
│  │  • Round Robin, Priority, SRTF                                      │ │
│  │  • 스케줄러가 강제로 CPU 회수                                            │ │
│  │  • 현대 OS의 표준                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【데이터베이스 타임아웃】                                                  │
│  ──────────────────                                                  │
│  // MySQL                                                            │
│  SET innodb_lock_wait_timeout = 50;  // 50초                          │
│                                                                     │
│  // PostgreSQL                                                       │
│  SET lock_timeout = '10s';                                           │
│  SET statement_timeout = '30s';                                      │
│                                                                     │
│  // 타임아웃 시 트랜잭션 롤백 → 비선점 조건 무력화                              │
│                                                                     │
│  【성능 비교】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전략             교착상태    오버헤드    작업 손실                     │ │
│  │  ────────         ──────     ──────     ──────                    │ │
│  │  비선점 허용        가능       없음        없음                       │ │
│  │  타임아웃          예방       낮음        가능                       │ │
│  │  tryLock          예방       중간        없음                       │ │
│  │  롤백              예방       높음        있음                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
