# 274. 공정성 (Fairness)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 스레드가 공평하게 락 획득 기회
> 2. **가치**: 기아 방지, 예측 가능한 동작
> 3. **융합**: 락 큐, FIFO, 스케줄링과 연관

---

## Ⅰ. 개요

### 개념 정의

공정성(Fairness)은 **모든 스레드가 락 획득 기회를 공평하게 갖는 속성**이다. 기아(starvation)를 방지하고 예측 가능한 동작을 보장한다.

### 💡 비유: 신호등
공정성은 **신호등**과 같다. 모든 방향의 차가 순서대로 지나간다. 한 방향만 계속 녹색이 아니다.

### 공정성 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                공정성 구조                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【공정 vs 불공정】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  불공정 (Unfair/Non-fair):                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  T1: [락 해제] ─ [락 획득] ─ [락 해제] ─ [락 획득] ...       │ │ │
│  │  │  T2:      [대기...] ─────[대기...] ─────[대기...]          │ │ │
│  │  │  T3:      [대기...] ─────[대기...] ─────[대기...]          │ │ │
│  │  │                                                         │ │ │
│  │  │  T1이 계속 획득, T2/T3 기아 가능                             │ │ │
│  │  │  처리량 높음                                               │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  공정 (Fair):                                                        │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  T1: [락 해제] ─ [대기] ─ [대기] ─ [락 획득] ─ [락 해제]    │ │ │
│  │  │  T2:      [락 획득] ─ [락 해제] ─ [대기] ─ [대기]          │ │ │
│  │  │  T3:                    [락 획득] ─ [락 해제]              │ │ │
│  │  │                                                         │ │ │
│  │  │  순서대로 진입, 기아 없음                                    │ │ │
│  │  │  처리량 낮을 수 있음                                         │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【공정성 유형】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 무기아 (Starvation-free):                                       │ │
│  │     • 모든 스레드가 결국 락 획득                                     │ │
│  │     • bounded waiting                                               │ │
│  │                                                             │ │
│  │  2. FIFO 공정성:                                                     │ │
│  │     • 도착 순서대로 락 획득                                          │ │
│  │     • 티켓 락, MCS 락                                               │ │
│  │                                                             │ │
│  │  3. 우선순위 공정성:                                                 │ │
│  │     • 우선순위에 따른 락 획득                                        │ │
│  │     • rt_mutex                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  처리량        ─────┐                                                │ │
│  │                    \                                                │ │
│  │                     \                                                │ │
│  │                      \______                                         │ │
│  │                              \                                       │ │
│  │                               \________ 공정성                       │ │
│  │                                                             │ │
│  │  불공정: 처리량 높음, 공정성 낮음                                      │ │
│  │  공정: 처리량 낮음, 공정성 높음                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【언어/라이브러리별 공정성】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Java ReentrantLock:                                                │ │
│  │  • new ReentrantLock() → 불공정 (기본)                               │ │
│  │  • new ReentrantLock(true) → 공정                                   │ │
│  │                                                             │ │
│  │  Java synchronized:                                                  │
│  │  • 불공정 (구현 의존)                                                 │ │
│  │                                                             │ │
│  │  C++ std::mutex:                                                     │ │
│  │  • 구현 정의 (보통 불공정)                                            │ │
│  │                                                             │ │
│  │  Linux mutex:                                                        │
│  │  • 불공정 (optimistic spinning)                                      │ │
│  │                                                             │ │
│  │  Go sync.Mutex:                                                      │
│  │  • 불공정                                                             │ │
│  │                                                             │ │
│  │  Rust std::sync::Mutex:                                             │
│  │  • 불공정                                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【공정성 보장 방법】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. FIFO 큐: 티켓 락, MCS 락, CLH 락                                 │ │
│  │  2. 타임스탬프: 들어온 순서 기록                                       │ │
│  │  3. 대기 횟수 제한: 일정 횟수 후 우선권                                 │ │
│  │  4. 우선순위 상속: rt_mutex                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Java ReentrantLock 공정성 선택】                                     │
│  ──────────────────                                                  │
│  import java.util.concurrent.locks.*;                               │
│                                                                     │
│  // 불공정 (기본, 높은 처리량)                                          │
│  ReentrantLock unfairLock = new ReentrantLock();                    │
│                                                                     │
│  // 공정 (기아 방지)                                                    │
│  ReentrantLock fairLock = new ReentrantLock(true);                  │
│                                                                     │
│  // 언제 공정 락 사용?                                                  │
│  // • 기아 방지 필요 시                                                 │
│  // • 예측 가능한 지연 필요                                              │
│  // • 공평한 리소스 분배                                                 │
│                                                                     │
│  【Java ReentrantReadWriteLock】                                      │
│  ──────────────────                                                  │
│  // 공정 RW 락                                                         │
│  ReentrantReadWriteLock rwLock = new ReentrantReadWriteLock(true);  │
│                                                                     │
│  // 공정 모드: writer 기아 방지                                         │
│  // 불공정 모드: reader 선호                                            │
│                                                                     │
│  【C++ 공정 락 구현 (티켓 락)】                                          │
│  ──────────────────                                                  │
│  class FairLock {                                                    │
│      std::atomic<uint32_t> ticket{0};                               │
│      std::atomic<uint32_t> serving{0};                              │
│                                                                     │
│  public:                                                            │
│      void lock() {                                                  │
│          uint32_t my_ticket = ticket.fetch_add(1);                  │
│          while (serving.load() != my_ticket) {                      │
│              std::this_thread::yield();                              │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      void unlock() {                                                │
│          serving.fetch_add(1);                                       │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【Go: 불공정 락의 예시】                                                │
│  ──────────────────                                                  │
│  var mu sync.Mutex                                                   │
│                                                                     │
│  // Go 뮤텍스는 불공정                                                  │
│  // 새 goroutine이 오래 기다린 것보다 먼저 획득 가능                       │
│                                                                      │
│  // 공정성이 필요하면 채널 사용                                          │
│  ch := make(chan struct{}, 1)                                        │
│  ch <- struct{}{}  // 락 획득                                         │
│  <-ch             // 락 해제                                          │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시나리오              불공정 락       공정 락                       │ │
│  │  ────                  ────────        ──────                      │ │
│  │  무경합                 1000 ops       980 ops                     │ │
│  │  낮은 경합              950 ops        850 ops                     │ │
│  │  높은 경합              500 ops        400 ops                     │ │
│  │  매우 높은 경합           200 ops        150 ops                     │ │
│  │                                                             │ │
│  │  불공정: 경합 높을수록 처리량 우위                                       │ │
│  │  공정: 모든 스레드 공평한 기회                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【공정성 선택 가이드】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  불공정 선택:                                                        │ │
│  │  • 높은 처리량 필요                                                   │ │
│  │  • 짧은 임계 영역                                                    │ │
│  │  • 기아 가능성 낮음                                                   │ │
│  │  • 일반적인 애플리케이션                                               │ │
│  │                                                             │ │
│  │  공정 선택:                                                          │ │
│  │  • 기아 방지 필수                                                    │ │
│  │  • 예측 가능한 지연 필요                                              │ │
│  │  • 실시간 시스템                                                      │ │
│  │  • 리소스 공평 분배                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【기아 감지】                                                          │
│  ──────────────────                                                  │
│  // Java: 대기 횟수 모니터링                                           │
│  if (lock.getQueueLength() > THRESHOLD) {                            │
│      log.warn("Lock queue too long, possible starvation");           │
│  }                                                                  │
│                                                                     │
│  // 대기 시간 모니터링                                                  │
│  long start = System.nanoTime();                                     │
│  lock.lock();                                                        │
│  long waitTime = System.nanoTime() - start;                         │
│  if (waitTime > MAX_WAIT) {                                          │
│      log.warn("Long lock wait: " + waitTime);                        │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 모든 스레드 공평한 락 획득 기회
• 유형: 무기아, FIFO, 우선순위
• 공정: 기아 방지, 예측 가능
• 불공정: 높은 처리량, 기아 가능
• 트레이드오프: 처리량 vs 공정성
• 선택: 상황에 따른 적절한 선택
```
