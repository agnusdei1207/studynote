# 웨이트-프리 알고리즘 (Wait-free Algorithm)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 웨이트-프리는 모든 스레드가 다른 스레드의 간섭이나 성공 여부와 상관없이, 유한한 단계(Bounded Steps) 내에 자신의 작업을 반드시 완료할 수 있도록 보장하는 가장 강력한 무결성 수준의 병렬 알고리즘이다.
> 2. **가치**: 락-프리가 '누군가는 성공한다'는 전체적 진행을 보장한다면, 웨이트-프리는 **'나도 무조건 성공한다'**는 개별적 진행을 보장하여, 실시간성(Real-time)이 극도로 중요한 시스템에서 결정론적 응답 시간을 제공한다.
> 3. **융합**: 고도의 수치 해석, 하드웨어 인터럽트 처리기, 미션 크리티컬 항공 제어 시스템 등 실패나 지연이 용납되지 않는 영역의 핵심 로직이다.

+++

## Ⅰ. 개요 (Context & Background)

- **개념**: 락-프리 알고리즘은 CAS 실패 시 재시도(Retry)를 하므로, 운이 나쁜 스레드는 이론적으로 무한히 재시도만 할 수 있다(Starvation 가능성). 웨이트-프리는 이를 방지하기 위해 '도움(Helping)' 메커니즘 등을 도입한다. 앞서가는 스레드가 뒤처진 스레드의 작업을 대신 처리해주거나, 각 스레드가 전용 슬롯을 가져 충돌 자체가 없게 설계한다.

- **💡 비유**: 웨이트-프리는 "전용 차선이 있는 마라톤"과 같다. 락-프리가 "누구든 먼저 들어오면 장땡이고 밀리면 다시 달려야 하는 경기"라면, 웨이트-프리는 "모든 주자에게 전용 레인을 배정하여, 옆 사람과 부딪힐 걱정 없이 자기 보폭대로만 뛰면 정해진 시간 안에 반드시 들어오는 경기"와 같다.

- **📢 섹션 요약 비유**: 남의 성공을 시기하거나 기다릴 필요 없이, 나에게 주어진 길을 묵묵히 가면 반드시 목적지에 도달하는 완벽한 자율 주행 시스템과 같습니다.

+++

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 병행성 알고리즘의 보장 수준 (Hierarchy)

1. **Blocking**: 락 점유자가 멈추면 모두가 멈춤.
2. **Lock-free**: 전체는 진행되나 개별 스레드는 무한 재시도 가능.
3. **Wait-free**: **모든 스레드가 유한 시간 내에 완료.**

### 웨이트-프리의 구현 기법

- **도움 (Helping) 메커니즘**: 스레드 A가 자기 일을 하려는데 B가 방해된다면, A는 단순히 기다리는 대신 B의 작업 정보(Descriptor)를 보고 B의 일을 대신 끝내버린 뒤 자기 일을 한다.
- **상태 배열 (State Array)**: 각 스레드마다 고유한 메모리 위치를 할당하여 쓰기 경합을 원천 차단한다. (예: Fast I/O Counters)

```text
  [ Thread A ]                   [ Shared Object ]                   [ Thread B ]
       |                                |                                |
  1. Register My Task            [ Task List ]                   1. Register My Task
       |                                |                                |
  2. See B's task is pending     [ B's Job ] ◀───────────────────┘
       |                                |
  3. Help B finish Job! ─────────▶ [ B's Job: Done ]
       |                                |
  4. Now finish My Job! ─────────▶ [ A's Job: Done ]
       |                                |                        2. Check My Task
       |                                |                        3. "Oh, it's already done!"
```

**[다이어그램 해설]** 스레드 A는 자신의 작업을 완료하기 위해 방해가 되는 B의 작업을 대신 처리(Helping)해 준다. 덕분에 B가 멈춰있더라도 A는 유한한 단계 내에 작업을 마칠 수 있다. B가 다시 깨어났을 때는 이미 일이 끝나 있어 즉시 다음 단계로 넘어간다.

- **📢 섹션 요약 비유**: 앞서가는 제설차가 눈을 치워주면(Helping), 뒤따라오는 모든 차가 멈추지 않고 제시간에 도착할 수 있는 원리와 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 락-프리 vs 웨이트-프리 심층 비교

| 비교 항목 | 락-프리 (Lock-free) | 웨이트-프리 (Wait-free) |
|:---|:---|:---|
| **진행 보장** | 시스템 전체 (System-wide) | 개별 스레드 모두 (Per-thread) |
| **최악의 시간** | 알 수 없음 (무한 재시도 가능) | 상한선이 존재함 (Bounded) |
| **구현 복잡도** | 매우 높음 | **극한으로 높음** |
| **성능 (Average)** | 매우 좋음 | 추가 로직(Helping)으로 인해 더 느릴 수 있음 |
| **기아 상태** | 발생 가능 | **절대 없음** |

웨이트-프리는 성능(평균 속도)을 희생하여 신뢰성(최악의 시간)을 얻는 트레이드오프의 정점이다.

- **📢 섹션 요약 비유**: 락-프리는 "자유 경쟁 시장"이고, 웨이트-프리는 "모든 시민에게 기본권을 보장하는 복지 국가"입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 항공기 플랩 제어 시스템
- **상황**: 조종사가 레버를 당기면 4개의 모터 제어 스레드가 동기화되어 움직여야 함. 한 스레드가 연산 지연으로 늦어지면 비행기가 기우뚱할 수 있음.
- **해결책**: 웨이트-프리 알고리즘을 적용한다. 어떤 스레드가 느려지더라도 다른 스레드가 그 상태값을 대신 갱신하거나, 각 스레드가 정해진 타임 슬롯 내에 반드시 결과를 내놓도록 설계하여 시스템의 결정론적(Deterministic) 동작을 보장한다.

### 도입 체크리스트
- **Criticality**: 지연 시간이 생명이 위태로울 만큼 치명적인가?
- **Step Bound**: 알고리즘의 최대 단계 수(N)를 수학적으로 증명할 수 있는가?

- **📢 섹션 요약 비유**: 0.1초의 오차도 허용하지 않는 올림픽 육상 경기 기록기처럼, 웨이트-프리는 시스템의 정밀도를 완성합니다.

+++

## Ⅴ. 기대효과 및 결론

- **기대효과**: 웨이트-프리는 병렬 프로그래밍이 도달할 수 있는 최고의 안전 등급이며, 하드웨어 결함이나 스케줄링 지연에도 굴하지 않는 강건한 시스템을 만든다.
- **미래 전망**: 멀티코어를 넘어 수만 개의 코어를 가진 가속기 환경에서는 개별 스레드의 진행 보장이 더욱 중요해질 것이며, 이를 자동화된 검증 도구(Formal Verification)로 설계하는 기술이 핵심 경쟁력이 될 것이다.

- **📢 섹션 요약 비유**: 모두를 포용하고 낙오자 없이 전진하는 웨이트-프리는 기술이 지향해야 할 완벽한 질서의 표본입니다.
