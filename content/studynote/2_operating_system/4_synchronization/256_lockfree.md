# 229. 락 (Lock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 상호 배제를 구현하는 동기화 객체
> 2. **가치**: 임계 영역 보호, 데이터 무결성 보장
> 3. **융합**: 뮤텍스, 스핀락, 읽기-쓰기 락과 연관

---

## Ⅰ. 개요

### 개념 정의

락(Lock)은 **상호 배제를 구현하여 한 번에 하나의 스레드만 임계 영역에 진입할 수 있게 하는 동기화 객체**다. 획득(acquire)과 해제(release) 연산을 제공한다.

### 💡 비유: 화장실 열쇠
락은 **화장실 열쇠**와 같다. 열쇠를 가진 사람만 들어갈 수 있다. 나오면 열쇠를 반납한다. 다음 사람이 들어간다.

### 락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 구조                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락의 기본 연산】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  lock.acquire(): 락 획득 (다른 스레드가 가지면 대기)             │ │
│  │  lock.release(): 락 해제 (대기 중인 스레드에게 기회)             │ │
│  │  lock.try_acquire(): 락 획득 시도 (실패 시 즉시 반환)            │ │
│  │                                                             │ │
│  │  스레드 A                      스레드 B                        │ │
│  │  ────────                      ────────                        │ │
│  │  acquire() ─────────────────► 대기...                          │ │
│  │  [임계 영역]                   대기...                          │ │
│  │  release() ─────────────────► acquire()                        │ │
│  │                                [임계 영역]                      │ │
│  │                                release()                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락의 종류】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  종류              특징                    용도                  │ │
│  │  ────                  ────                        ────          │ │
│  │  스핀락            대기하며 스핀           짧은 임계 영역          │ │
│  │  뮤텍스            대기하며 슬립           긴 임계 영역            │ │
│  │  읽기-쓰기 락       읽기 병렬, 쓰기 배제     읽기 많은 경우         │ │
│  │  재귀 락           같은 스레드 재진입       재귀 함수              │ │
│  │  적응형 락         스핀 후 슬립             하이브리드             │ │
│  │  순차적 락         읽기 무잠금              읽기 많은 경우         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락의 속성】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 상호 배제 (Mutual Exclusion)                                  │ │
│  │  • 공정성 (Fairness): FIFO, 공정한 기회                          │ │
│  │  • 재진입성 (Reentrancy): 같은 스레드 재획득 가능                 │ │
│  │  • 범위 (Scope): 프로세스 내, 프로세스 간                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락 구현 방식】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 스핀락 (Spinlock)                                            │ │
│  │     while (test_and_set(&lock)) { /* 스핀 */ }                 │ │
│  │     장점: 컨텍스트 스위치 없음                                    │ │
│  │     단점: CPU 낭비                                               │ │
│  │                                                             │ │
│  │  2. 뮤텍스 (Mutex)                                                │ │
│  │     if (try_lock()) { /* 성공 */ }                               │ │
│  │     else { sleep(); }  // 커널에 의한 대기                        │ │
│  │     장점: CPU 낭비 없음                                           │ │
│  │     단점: 컨텍스트 스위치 오버헤드                                 │ │
│  │                                                             │ │
│  │  3. 적응형 락 (Adaptive Lock)                                    │ │
│  │     if (try_lock()) { /* 성공 */ }                               │ │
│  │     else if (소유자가 실행 중) { 스핀(); }                        │ │
│  │     else { sleep(); }                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락 선택 기준】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  조건                      추천 락                               │ │
│  │  ────                          ────                               │ │
│  │  임계 영역 매우 짧음        스핀락                                 │ │
│  │  임계 영역 긺             뮤텍스                                  │ │
│  │  읽기 많고 쓰기 적음        읽기-쓰기 락                           │ │
│  │  재귀 호출 필요            재귀 락                                │ │
│  │  최고 성능 필요            순차적 락 / RCU                        │ │
│  │  실시간 보장              우선순위 상속 락                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락 문제점】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 데드락: 서로 상대방의 락을 기다림                              │ │
│  │  • 라이브락: 서로 양보하며 진행 못 함                             │ │
│  │  • 기아: 특정 스레드가 계속 대기                                  │ │
│  │  • 우선순위 역전: 낮은 우선순위가 높은 것을 블록                   │ │
│  │  • 컨보이 효과: 락 경합 시 성능 저하                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【POSIX Mutex】                                                      │
│  ──────────────────                                                  │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│                                                                     │
│  void critical_section() {                                          │
│      pthread_mutex_lock(&mutex);                                    │
│      // 임계 영역                                                    │
│      pthread_mutex_unlock(&mutex);                                  │
│  }                                                                  │
│                                                                     │
│  // trylock (논블로킹)                                               │
│  if (pthread_mutex_trylock(&mutex) == 0) {                          │
│      // 임계 영역                                                    │
│      pthread_mutex_unlock(&mutex);                                  │
│  }                                                                  │
│                                                                     │
│  【C++ std::mutex】                                                   │
│  ──────────────────                                                  │
│  std::mutex mtx;                                                    │
│                                                                     │
│  void safe_increment() {                                            │
│      std::lock_guard<std::mutex> lock(mtx);  // RAII                │
│      counter++;                                                     │
│  }                                                                  │
│                                                                     │
│  // unique_lock (더 유연함)                                          │
│  void flexible_lock() {                                             │
│      std::unique_lock<std::mutex> lock(mtx, std::defer_lock);       │
│      if (need_lock) {                                               │
│          lock.lock();                                               │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【재귀 뮤텍스】                                                       │
│  ──────────────────                                                  │
│  std::recursive_mutex rmtx;                                         │
│                                                                     │
│  void recursive_func(int depth) {                                   │
│      std::lock_guard<std::recursive_mutex> lock(rmtx);              │
│      if (depth > 0) {                                               │
│          recursive_func(depth - 1);  // 같은 스레드에서 재진입 가능   │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【성능 측정】                                                         │
│  ──────────────────                                                  │
│  # 락 경합 분석                                                       │
│  $ perf lock record ./program                                       │
│  $ perf lock report                                                 │
│                                                                     │
│  # 락 대기 시간                                                       │
│  $ perf record -e lock:lock_wait ./program                         │
│  $ perf report                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 상호 배제를 구현하는 동기화 객체
• 연산: acquire, release, try_acquire
• 종류: 스핀락, 뮤텍스, rwlock, 재귀락
• 선택: 임계 영역 길이에 따라 결정
• 문제: 데드락, 기아, 우선순위 역전
• RAII: lock_guard, unique_lock
```
