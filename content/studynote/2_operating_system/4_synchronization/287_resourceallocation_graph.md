# 301. 자원 할당 그래프 (Resource Allocation Graph)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스와 자원 간 관계를 그래프로 표현
> 2. **가치**: 교착상태 분석의 시각적 도구
> 3. **융합**: 대기 그래프, 순환 탐지와 연관

---

## Ⅰ. 개요

### 개념 정의

자원 할당 그래프(Resource Allocation Graph)는 **프로세스와 자원 간의 할당 및 요청 관계를 방향성 그래프로 표현한 것**이다. 교착상태 분석의 핵심 도구다.

### 💡 비유: 지하철 노선도
자원 할당 그래프는 **지하철 노선도**와 같다. 역(프로세스)과 노선(자원)의 연결을 보여준다. 순환선이 있으면 계속 돈다.

### 자원 할당 그래프 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 할당 그래프 구조                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【그래프 구성요소】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  노드 (Nodes):                                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  ● 프로세스 (원형): P1, P2, P3, ...                           │ │ │
│  │  │  □ 자원 (사각형): R1, R2, R3, ...                              │ │ │
│  │  │    자원 내부 ●: 인스턴스                                       │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  간선 (Edges):                                                        │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  P → R: 요청 간선 (프로세스가 자원 요청)                           │ │ │
│  │  │  R → P: 할당 간선 (자원이 프로세스에 할당됨)                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【그래프 예시】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │       ●P1 ──요청──► □R1 ──할당──► ●P2                          │ │
│  │         ▲                       │                              │ │
│  │         │                       ▼                              │ │
│  │       □R2 ◄──할당── ●P3 ◄──요청── □R3                          │ │
│  │         │                       ▲                              │ │
│  │         └──────요청──────────────┘                              │ │
│  │                                                             │ │
│  │  해석:                                                               │ │
│  │  • P1이 R1 요청                                                       │ │
│  │  • R1이 P2에 할당됨                                                   │ │
│  │  • P2가 R3 요청                                                       │ │
│  │  • R3이 P3에 할당됨                                                   │ │
│  │  • P3이 R2 요청                                                       │ │
│  │  • R2가 P1에 할당됨                                                   │ │
│  │  → 순환: P1 → R1 → P2 → R3 → P3 → R2 → P1                        │ │
│  │  → 교착상태!                                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 할당 그래프 상세                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【교착상태 판정 규칙】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 순환이 없으면: 교착상태 없음                                         │ │
│  │                                                             │ │
│  │  2. 순환이 있고, 각 자원이 1개의 인스턴스만 가지면:                        │ │
│  │     → 교착상태 발생                                                    │ │
│  │                                                             │ │
│  │  3. 순환이 있고, 일부 자원이 여러 인스턴스를 가지면:                       │ │
│  │     → 교착상태 가능성 (확정 아님)                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【다중 인스턴스 예시】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  R1: 2개 인스턴스 (□ 안에 ●●)                                         │ │
│  │  R2: 1개 인스턴스 (□ 안에 ●)                                          │ │
│  │                                                             │ │
│  │  순환이 있어도 교착상태 아닐 수 있음:                                     │ │
│  │  • P1이 R1의 1개 인스턴스 요청                                          │ │
│  │  • P2가 R1의 다른 1개 인스턴스 사용 중                                   │ │
│  │  → 순환처럼 보여도 다른 인스턴스 사용 가능                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【그래프 축소 (Reduction)】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 완전히 충족된 프로세스 제거 (간선과 함께)                             │ │
│  │  2. 할당된 자원을 가용 자원으로 복귀                                      │ │
│  │  3. 더 이상 축소 불가능할 때까지 반복                                     │ │
│  │  4. 모든 프로세스가 제거되면: 교착상태 없음                                │ │
│  │  5. 남은 프로세스가 있으면: 교착상태 발생                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【그래프 표현】                                                        │
│  ──────────────────                                                  │
│  class ResourceAllocationGraph {                                    │
│      Set<Process> processes;                                        │
│      Set<Resource> resources;                                       │
│      Map<Process, Set<Resource>> requestEdges;  // P → R            │
│      Map<Resource, Set<Process>> allocationEdges; // R → P          │
│                                                                     │
│      void requestResource(Process p, Resource r) {                  │
│          requestEdges.computeIfAbsent(p, k -> new HashSet<>())      │
│              .add(r);                                                │
│      }                                                              │
│                                                                     │
│      void allocateResource(Resource r, Process p) {                 │
│          allocationEdges.computeIfAbsent(r, k -> new HashSet<>())   │
│              .add(p);                                                │
│          requestEdges.get(p).remove(r);                              │
│      }                                                              │
│                                                                     │
│      void releaseResource(Resource r, Process p) {                  │
│          allocationEdges.get(r).remove(p);                           │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【교착상태 탐지 (순환 검사)】                                             │
│  ──────────────────                                                  │
│  // 대기 그래프로 변환 후 순환 탐지                                         │
│  WaitForGraph toWaitForGraph() {                                    │
│      WaitForGraph wfg = new WaitForForGraph();                      │
│                                                                     │
│      // P1 → R → P2 를 P1 → P2 로 변환                               │
│      for (Process p1 : processes) {                                 │
│          for (Resource r : requestEdges.getOrDefault(p1, Set.of())) {│
│              for (Process p2 : allocationEdges.getOrDefault(r, Set.of())) {│
│                  wfg.addEdge(p1, p2);                               │
│              }                                                      │
│          }                                                          │
│      }                                                              │
│      return wfg;                                                     │
│  }                                                                  │
│                                                                     │
│  boolean hasDeadlock() {                                            │
│      return toWaitForGraph().hasCycle();                            │
│  }                                                                  │
│                                                                     │
│  【시각화】                                                             │
│  ──────────────────                                                  │
│  // Graphviz DOT 형식으로 출력                                          │
│  String toDot() {                                                    │
│      StringBuilder sb = new StringBuilder();                        │
│      sb.append("digraph RAG {\n");                                  │
│      sb.append("  node [shape=circle];\n");  // 프로세스              │
│                                                                     │
│      for (Process p : processes) {                                  │
│          sb.append("  \"" + p + "\" [shape=circle];\n");            │
│      }                                                              │
│      for (Resource r : resources) {                                 │
│          sb.append("  \"" + r + "\" [shape=box];\n");               │
│      }                                                              │
│                                                                     │
│      // 요청 간선                                                     │
│      for (var entry : requestEdges.entrySet()) {                     │
│          for (Resource r : entry.getValue()) {                       │
│              sb.append("  \"" + entry.getKey() + "\" -> \"" +        │
│                  r + "\" [label=\"request\"];\n");                   │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      // 할당 간선                                                     │
│      for (var entry : allocationEdges.entrySet()) {                  │
│          for (Process p : entry.getValue()) {                        │
│              sb.append("  \"" + entry.getKey() + "\" -> \"" +        │
│                  p + "\" [label=\"allocate\"];\n");                  │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      sb.append("}\n");                                               │
│      return sb.toString();                                           │
│  }                                                                  │
│                                                                     │
│  【실제 사용 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템                도구                시각화                    │ │
│  │  ────────              ────────            ──────                    │ │
│  │  Linux 커널            lockdep             /proc/lockdep             │ │
│  │  Java                  JConsole            스레드 덤프                 │ │
│  │  데이터베이스            DB 모니터링          그래프 UI                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 고려사항】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  연산                    시간 복잡도                                  │ │
│  │  ────────                 ──────────                                │ │
│  │  그래프 구축              O(V + E)                                   │ │
│  │  순환 탐지                O(V + E)                                   │ │
│  │  그래프 축소              O(V × E)                                   │ │
│  │                                                             │ │
│  │  V: 프로세스 + 자원 노드 수                                            │ │
│  │  E: 요청 + 할당 간선 수                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 프로세스-자원 관계 그래프
• 구성: 프로세스(원), 자원(사각형), 간선
• 간선: 요청(P→R), 할당(R→P)
• 판정: 순환 + 단일 인스턴스 = 교착상태
• 축소: 프로세스 제거로 분석
• 활용: 교착상태 분석, 시각화
```
