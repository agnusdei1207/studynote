# 278. 우선순위 상한 (Priority Ceiling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 획득 시 미리 정해진 상한으로 승격
> 2. **가치**: 우선순위 역전 사전 예방
> 3. **융합**: 우선순위 상속, 데드락 방지와 연관

---

## Ⅰ. 개요

### 개념 정의

우선순위 상한(Priority Ceiling)은 **락을 획득할 때 미리 정해진 최대 우선순위로 즉시 승격하는 기법**이다. 우선순위 역전을 사전에 예방한다.

### 💡 비유: VIP 패스
우선순위 상한은 **VIP 패스**와 같다. 특정 구역에 들어가면 자동으로 VIP 대우를 받는다. 나올 때는 원래 신분으로 돌아온다.

### 우선순위 상한 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                우선순위 상한 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  각 락마다 "천장(ceiling)" 우선순위가 미리 지정됨                     │ │
│  │                                                             │ │
│  │  ceiling(LockA) = 이 락을 사용할 최고 우선순위                     │ │
│  │                                                             │ │
│  │  예시:                                                             │ │
│  │  ┌────────────────────────────────────────────────────────┐ │ │
│  │  │  Lock A: ceiling = 90 (High가 사용)                      │ │ │
│  │  │  Lock B: ceiling = 70 (Medium이 사용)                    │ │ │
│  │  │  Lock C: ceiling = 50 (Low가 사용)                       │ │ │
│  │  └────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 시나리오】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. L (Low, 우선순위 30)이 Lock A 획득 시도                       │ │
│  │                                                             │ │
│  │     [락 획득 전]              [락 획득 후]                         │ │
│  │     L (prio=30)     ───►     L (prio=90)                       │ │
│  │                           ceiling(Lock A)로 승격               │ │
│  │                                                             │ │
│  │  2. M (Medium, 우선순위 50)이 CPU 요청                             │ │
│  │     → L이 이미 90이므로 M이 선점 불가                               │ │
│  │                                                             │ │
│  │  3. L이 Lock A 해제                                               │ │
│  │     → 원래 우선순위 30으로 복귀                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 우선순위 상속】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  상속 (Inheritance):                                               │ │
│  │  • 높은 우선순위 웨이터가 생길 때 승격                                 │ │
│  │  • 반응형 (reactive)                                               │ │
│  │  • 동적                                                            │ │
│  │                                                             │ │
│  │  상한 (Ceiling):                                                    │ │
│  │  • 락 획득 시 즉시 승격                                              │ │
│  │  • 선제형 (proactive)                                               │ │
│  │  • 정적 (미리 지정)                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                우선순위 상한 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Ceiling 계산】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ceiling(lock) = max(이 락을 사용하는 모든 스레드의 우선순위)         │ │
│  │                                                             │ │
│  │  스레드 H (prio=90)이 Lock A 사용                                  │ │
│  │  스레드 M (prio=70)이 Lock A 사용                                  │ │
│  │  스레드 L (prio=30)이 Lock A 사용                                  │ │
│  │                                                             │ │
│  │  → ceiling(Lock A) = max(90, 70, 30) = 90                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Original Ceiling Protocol (OCP)】                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  규칙:                                                             │ │
│  │  1. 락 획득 시 ceiling으로 승격                                      │ │
│  │  2. 높은 우선순위 블록 방지                                          │ │
│  │  3. 교착상태 방지 (ceiling 기반 순서)                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                              │ │
│  │  • 역전 완전 방지                                                    │ │
│  │  • 교착상태 방지 (ceiling 순서)                                      │ │
│  │  • 연쇄 블록 방지                                                    │ │
│  │  • 구현 단순 (상속보다)                                              │ │
│  │                                                             │ │
│  │  단점:                                                              │ │
│  │  • ceiling 사전 지정 필요                                           │ │
│  │  • 낮은 우선순위도 높게 승격 (자원 낭비)                               │ │
│  │  • 유연성 낮음                                                       │ │
│  │  • 시스템 변경 시 재설정                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Immediate vs Original】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Immediate Ceiling Protocol:                                      │ │
│  │  • 락 획득 즉시 ceiling 적용                                         │ │
│  │  • 더 간단한 구현                                                    │ │
│  │                                                             │ │
│  │  Original Ceiling Protocol:                                       │ │
│  │  • 조건부 ceiling 적용                                              │ │
│  │  • 더 효율적이나 복잡                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【POSIX Priority Ceiling Mutex】                                     │
│  ──────────────────                                                  │
│  #include <pthread.h>                                               │
│                                                                     │
│  pthread_mutexattr_t attr;                                          │
│  pthread_mutexattr_init(&attr);                                     │
│                                                                     │
│  // Protocol 설정                                                    │
│  pthread_mutexattr_setprotocol(&attr, PTHREAD_PRIO_PROTECT);        │
│                                                                     │
│  // Ceiling 설정 (예: 우선순위 80)                                     │
│  pthread_mutexattr_setprioceiling(&attr, 80);                       │
│                                                                     │
│  pthread_mutex_t pc_mutex;                                          │
│  pthread_mutex_init(&pc_mutex, &attr);                              │
│                                                                     │
│  【Linux rt_mutex + Ceiling】                                         │
│  ──────────────────                                                  │
│  // 실제로는 PI를 더 많이 사용                                          │
│  // PC는 특수한 경우에만                                                │
│                                                                     │
│  // 수동 구현 예시                                                     │
│  struct ceiling_mutex {                                             │
│      pthread_mutex_t mutex;                                         │
│      int ceiling_prio;                                              │
│      int original_prio;                                             │
│  };                                                                 │
│                                                                     │
│  void ceiling_lock(struct ceiling_mutex *m) {                       │
│      m->original_prio = get_current_priority();                    │
│      if (m->original_prio < m->ceiling_prio) {                     │
│          set_priority(m->ceiling_prio);                            │
│      }                                                              │
│      pthread_mutex_lock(&m->mutex);                                 │
│  }                                                                  │
│                                                                     │
│  void ceiling_unlock(struct ceiling_mutex *m) {                     │
│      pthread_mutex_unlock(&m->mutex);                               │
│      set_priority(m->original_prio);                               │
│  }                                                                  │
│                                                                     │
│  【VxWorks Priority Ceiling】                                         │
│  ──────────────────                                                  │
│  // 세마포어 생성 시 ceiling 지정                                       │
│  SEM_ID sem = semMCreate(SEM_Q_PRIORITY |                           │
│                          SEM_PRIO_PROTECT,                          │
│                          100);  // ceiling = 100                    │
│                                                                     │
│  【QNX Priority Ceiling】                                             │
│  ──────────────────                                                  │
│  pthread_mutexattr_setprotocol(&attr, PTHREAD_PRIO_PROTECT);        │
│  pthread_mutexattr_setprioceiling(&attr, 63);  // QNX 최대 63        │
│                                                                     │
│  【Ceiling 결정 가이드】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 코드 분석으로 락 사용자 식별                                     │ │
│  │  2. 각 사용자의 우선순위 파악                                        │ │
│  │  3. 최대값을 ceiling으로 지정                                        │ │
│  │  4. 안전 마진 추가 고려                                             │ │
│  │                                                             │ │
│  │  예: Lock A를 H(99), M(70), L(30)이 사용                            │ │
│  │      → ceiling = 99 또는 99+1 = 100                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  프로토콜       락 획득    역전 방지    교착상태 방지                 │ │
│  │  ────────       ──────    ──────      ────────                   │ │
│  │  일반           O(1)      X          X                          │ │
│  │  PI             O(log n)  O          X                          │ │
│  │  PC             O(1)      O          O                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【사용 시나리오】                                                      │
│  ──────────────────                                                  │
│  • 미션 크리티컬 실시간 시스템                                          │
│  • 교착상태 방지가 중요한 경우                                           │
│  • 락 사용 패턴이 정적인 경우                                           │
│  • 우선순위가 사전에 알려진 경우                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 락 획득 시 ceiling으로 즉시 승격

```
