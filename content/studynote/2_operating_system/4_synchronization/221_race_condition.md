# 221. 동기화 개요 (Synchronization Overview)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 실행 흐름 간의 순서와 자원 접근 조율
> 2. **가치**: 데이터 무결성, 예측 가능한 동작 보장
> 3. **융합**: 임계 영역, 상호 배제, 락과 연관

---

## Ⅰ. 개요

### 개념 정의

동기화(Synchronization)는 **여러 프로세스나 스레드가 공유 자원에 접근할 때, 순서를 조율하여 일관성을 보장하는 메커니즘**이다. 경쟁 조건과 데이터 손상을 방지한다.

### 💡 비유: 화장실 열쇠
동기화는 **공용 화장실 열쇠**와 같다. 한 사람이 쓰고 있으면 다른 사람은 대기한다. 열쇠(락)가 동시 접근을 막는다.

### 동기화 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                동기화 개요 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동기화 없이】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  스레드 A: count = count + 1                                 │ │
│  │           (1. 읽기: 10)                                      │ │
│  │           (2. 증가: 11) ───────────────────┐                 │ │
│  │           (3. 쓰기: 11)               │                    │ │
│  │                                         ▼                    │ │
│  │  스레드 B: count = count + 1         (2. 증가: 11)           │ │
│  │           (1. 읽기: 10) ◄───────────────┘                   │ │
│  │           (3. 쓰기: 11)               │                    │ │
│  │                                                             │ │
│  │  결과: 11 (올바른 값: 12)                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동기화 있이】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  스레드 A: lock() → [1. 읽기: 10, 2. 증가: 11, 3. 쓰기: 11] → unlock()
│  │                                                             │ │
│  │  스레드 B:            대기... → lock() → [읽기: 11, 증가: 12, 쓰기: 12] → unlock()
│  │                                                             │ │
│  │  결과: 12 (올바른 값)                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동기화 도구】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  도구              특징                    용도                  │ │
│  │  ────                  ────                        ────          │ │
│  │  뮤텍스            상호 배제             임계 영역 보호            │ │
│  │  세마포어          카운팅 동기화          자원 수 관리             │ │
│  │  스핀락            대기하며 스핀           짧은 대기               │ │
│  │  조건 변수         이벤트 대기             생산자-소비자            │ │
│  │  모니터            고수준 추상화           언어 지원               │ │
│  │  RCU               읽기 최적화             읽기 많은 경우          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                동기화 개요 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동기화 문제 유형】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 경쟁 조건 (Race Condition)                                  │ │
│  │     • 여러 스레드가 동시에 공유 자원 접근                         │ │
│  │     • 실행 순서에 따라 결과 달라짐                                │ │
│  │                                                             │ │
│  │  2. 데드락 (Deadlock)                                           │ │
│  │     • 서로 상대방의 자원을 기다림                                 │ │
│  │     • 진행 불가 상태                                             │ │
│  │                                                             │ │
│  │  3. 기아 (Starvation)                                           │ │
│  │     • 특정 스레드가 계속 대기                                    │ │
│  │     • 자원 할당받지 못함                                         │ │
│  │                                                             │ │
│  │  4. 라이브락 (Livelock)                                          │ │
│  │     • 서로 양보하며 진행 못 함                                    │ │
│  │     • CPU 소모만 함                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동기화 수준】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 하드웨어 수준                                                │ │
│  │     • 원자적 명령어 (CAS, Test-and-Set)                          │ │
│  │     • 메모리 배리어                                              │ │
│  │                                                             │ │
│  │  2. 커널 수준                                                    │ │
│  │     • 스핀락, 뮤텍스                                              │ │
│  │     • 인터럽트 제어                                               │ │
│  │                                                             │ │
│  │  3. 사용자 수준                                                  │ │
│  │     • POSIX mutex, semaphore                                     │ │
│  │     • 조건 변수, rwlock                                          │ │
│  │                                                             │ │
│  │  4. 언어 수준                                                    │ │
│  │     • synchronized, lock                                         │ │
│  │     • atomic, volatile                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동기화 필요성 확인】                                               │
│  ──────────────────                                                  │
│  // 동기화 없이 (버그)                                                │
│  int counter = 0;                                                   │
│  void increment() {                                                 │
│      counter++;  // 읽기-수정-쓰기: 원자적 아님                       │
│  }                                                                  │
│                                                                     │
│  // 동기화 있이 (올바름)                                              │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│  int counter = 0;                                                   │
│                                                                     │
│  void increment() {                                                 │
│      pthread_mutex_lock(&mutex);                                    │
│      counter++;                                                     │
│      pthread_mutex_unlock(&mutex);                                  │
│  }                                                                  │
│                                                                     │
│  【C++11 atomic】                                                     │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  std::atomic<int> counter{0};                                       │
│                                                                     │
│  void increment() {                                                 │
│      counter.fetch_add(1, std::memory_order_relaxed);               │
│  }                                                                  │
│                                                                     │
│  【데이터 경쟁 탐지】                                                 │
│  ──────────────────                                                  │
│  # ThreadSanitizer 사용                                              │
│  $ gcc -fsanitize=thread -g program.c                              │
│  $ ./a.out                                                          │
│  # 경쟁 조건 보고                                                    │
│                                                                     │
│  # Helgrind (Valgrind)                                              │
│  $ valgrind --tool=helgrind ./program                              │
│                                                                     │
│  【성능 고려사항】                                                    │
│  ──────────────────                                                  │
│  • 임계 영역 최소화                                                  │
│  • 락 세분화 (fine-grained locking)                                 │
│  • 읽기-쓰기 락 활용                                                 │
│  • lock-free 자료구조 고려                                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
