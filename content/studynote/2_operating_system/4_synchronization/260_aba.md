# 260. 락 호환성 매트릭스 (Lock Compatibility Matrix)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서로 다른 락 모드 간의 호환성 정의
> 2. **가치**: 차단 여부 판단, 동시성 제어
> 3. **융합**: 공유 락, 배타 락, 의도 락과 연관

---

## Ⅰ. 개요

### 개념 정의

락 호환성 매트릭스(Lock Compatibility Matrix)는 **서로 다른 락 모드 간에 동시 존재가 가능한지를 정의한 표**다. 호환되면 동시 접근 가능하고, 호환되지 않으면 대기해야 한다.

### 💡 비유: 도서관 좌석
락 호환성 매트릭스는 **도서관 좌석**과 같다. 공유 좌석은 여러 사람이 앉을 수 있다. 전용 좌석은 한 사람만. 이미 앉은 사람과 새로 오는 사람의 관계를 정의한다.

### 락 호환성 매트릭스 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 호환성 매트릭스 구조                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SQL Server 락 모드】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  모드       이름              설명                                │ │
│  │  ────       ────              ────                                │ │
│  │  S          Shared            공유 (읽기)                          │ │
│  │  X          Exclusive         배타 (읽기/쓰기)                      │ │
│  │  U          Update            갱신 (S → X 전환 예정)               │ │
│  │  IS         Intent Shared     공유 의도                            │ │
│  │  IX         Intent Exclusive  배타 의도                            │ │
│  │  SIX        Shared+IX         공유+배타의도                        │ │
│  │  Sch-M      Schema Modify     스키마 수정                          │ │
│  │  Sch-S      Schema Stability  스키마 안정                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【호환성 매트릭스】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │           요청 락 →                                           │ │
│  │           S    X    U    IS   IX   SIX                        │ │
│  │       ┌────┬────┬────┬────┬────┬────┐                         │ │
│  │     S │ ✓  │ ✗  │ ✓  │ ✓  │ ✗  │ ✗  │   ✓ = 호환 (진입 가능)       │ │
│  │       ├────┼────┼────┼────┼────┼────┤   ✗ = 충돌 (대기)          │ │
│  │     X │ ✗  │ ✗  │ ✗  │ ✗  │ ✗  │ ✗  │                         │ │
│  │  보    ├────┼────┼────┼────┼────┼────┤                         │ │
│  │  유    U │ ✓  │ ✗  │ ✓  │ ✓  │ ✗  │ ✗  │                         │ │
│  │  락    ├────┼────┼────┼────┼────┼────┤                         │ │
│  │  ↓    IS│ ✓  │ ✗  │ ✓  │ ✓  │ ✓  │ ✓  │                         │ │
│  │       ├────┼────┼────┼────┼────┼────┤                         │ │
│  │     IX │ ✗  │ ✗  │ ✗  │ ✓  │ ✓  │ ✗  │                         │ │
│  │       ├────┼────┼────┼────┼────┼────┤                         │ │
│  │    SIX │ ✗  │ ✗  │ ✗  │ ✓  │ ✗  │ ✗  │                         │ │
│  │       └────┴────┴────┴────┴────┴────┘                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【핵심 규칙】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 공유(S) + 공유(S) = 호환 ✓                                      │ │
│  │     • 여러 읽기 동시 가능                                           │ │
│  │                                                             │ │
│  │  2. 공유(S) + 배타(X) = 충돌 ✗                                      │ │
│  │     • 읽기 중 쓰기 차단                                             │ │
│  │                                                             │ │
│  │  3. 배타(X) + 배타(X) = 충돌 ✗                                      │ │
│  │     • 동시 쓰기 차단                                                │ │
│  │                                                             │ │
│  │  4. 의도 락(IS, IX) = 계층적 호환성                                  │ │
│  │     • 상위 레벨 의도와 하위 레벨 락 호환성 확인                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 호환성 매트릭스 상세                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Oracle 락 모드】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  모드       이름              설명                                │ │
│  │  ────       ────              ────                                │ │
│  │  NULL       Null              락 없음                              │ │
│  │  SS         Sub-Shared        하위 공유                            │ │
│  │  S          Shared            공유                                │ │
│  │  SSX        Sub-Exclusive     하위 배타                            │ │
│  │  SX         Shared-Exclusive  공유 배타                           │ │
│  │  X          Exclusive         배타                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【의도 락(Intent Lock) 계층】                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  목적: 계층적 락에서 상위 레벨에 하위 락 의도 표시                     │ │
│  │                                                             │ │
│  │  예: 행 락(X) 획득 시                                               │ │
│  │  1. 테이블에 IX 락 획득                                              │ │
│  │  2. 페이지에 IX 락 획득                                              │ │
│  │  3. 행에 X 락 획득                                                  │ │
│  │                                                             │ │
│  │  다른 트랜잭션이 테이블 X 락 요청 시:                                  │ │
│  │  → IX 락 존재 확인 → 충돌 → 대기                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【업데이트 락(U)】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  목적: 교착상태 방지                                                 │ │
│  │                                                             │ │
│  │  문제: S → X 전환 시                                                │ │
│  │  T1: S 획득 → X 요청 (대기)                                         │ │
│  │  T2: S 획득 → X 요청 (대기)  → 교착상태                               │ │
│  │                                                             │ │
│  │  해결: U 락 사용                                                     │ │
│  │  • U는 S와 호환                                                     │ │
│  │  • U → X 전환은 단독만 가능                                          │ │
│  │  • U끼리는 호환되지 않음                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SIX (Shared with Intent Exclusive)】                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  목적: 읽기 + 일부 행 갱신                                           │ │
│  │  • 테이블 전체 읽기 (S)                                               │ │
│  │  • 일부 행 갱신 의도 (IX)                                            │ │
│  │  • S + IX = SIX                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SQL Server 락 힌트】                                               │
│  ──────────────────                                                  │
│  -- 공유 락 (읽기)                                                    │
│  SELECT * FROM table WITH (TABLOCK, HOLDLOCK) WHERE id = 1;        │
│                                                                     │
│  -- 배타 락 (쓰기)                                                    │
│  UPDATE table WITH (XLOCK) SET col = val WHERE id = 1;             │
│                                                                     │
│  -- 업데이트 락 (교착상태 방지)                                         │
│  SELECT * FROM table WITH (UPDLOCK) WHERE id = 1;                  │
│  UPDATE table SET col = val WHERE id = 1;                          │
│                                                                     │
│  【Oracle 락 모드】                                                   │
│  ──────────────────                                                  │
│  -- 행 배타 락                                                        │
│  SELECT * FROM table WHERE id = 1 FOR UPDATE;                      │
│                                                                     │
│  -- 행 공유 락                                                         │
│  SELECT * FROM table WHERE id = 1 FOR SHARE;                       │
│                                                                     │
│  -- NOWAIT (대기 없이 즉시 실패)                                       │
│  SELECT * FROM table WHERE id = 1 FOR UPDATE NOWAIT;               │
│                                                                     │
│  【MySQL InnoDB 락】                                                  │
│  ──────────────────                                                  │
│  -- 공유 락                                                            │
│  SELECT * FROM table WHERE id = 1 LOCK IN SHARE MODE;              │
│                                                                     │
│  -- 배타 락                                                            │
│  SELECT * FROM table WHERE id = 1 FOR UPDATE;                      │
│                                                                     │
│  -- 갭 락                                                             │
│  SELECT * FROM table WHERE id > 1 FOR UPDATE;  -- 갭도 잠금           │
│                                                                     │
│  【차단 분석 쿼리】                                                     │
│  ──────────────────                                                  │
│  -- SQL Server                                                       │
│  SELECT                                                              │
│      t1.resource_type,                                               │
│      t1.request_mode,                                                │
│      t2.request_mode AS blocking_mode                               │
│  FROM sys.dm_tran_locks t1                                          │
│  JOIN sys.dm_tran_locks t2                                          │
│      ON t1.resource_address = t2.resource_address                  │
│  WHERE t1.request_status = 'GRANT'                                  │
│  AND t2.request_status = 'WAIT';                                    │
│                                                                     │
│  -- Oracle                                                           │
│  SELECT                                                              │
│      l1.type,                                                        │
│      l1.lmode,                                                      │
│      l2.lmode AS blocking_lmode                                     │
│  FROM v$lock l1                                                      │
│  JOIN v$lock l2 ON l1.id1 = l2.id1                                  │
│  WHERE l1.block = 1;                                                │
│                                                                     │
│  【성능 고려사항】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  락 모드 선택:                                                       │ │
│  │  • 읽기만: S 락                                                     │ │
│  │  • 읽고 갱신: U 락 → X 락                                            │ │
│  │  • 대량 갱신: X 락                                                  │ │
│  │  • 읽기 + 일부 갱신: SIX                                             │ │
│  │                                                             │ │
│  │  호환성 이해:                                                        │ │
│  │  • S끼리: 호환 (높은 동시성)                                          │ │
│  │  • X: 모두와 충돌 (낮은 동시성)                                       │ │
│  │  • U: S와 호환, U와 충돌                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 락 모드 간 호환성 정의
• S: 공유, 읽기, 여럿이 동시 가능
• X: 배타, 쓰기, 단독만 가능
• U: 갱신, S→X 전환, 교착상태 방지
• IS/IX: 의도, 계층적 락
• SIX: 공유+배타의도, 읽기+일부갱신
• 활용: 차단 분석, 쿼리 힌트
```
