# 286. 락 컨보이 (Lock Convoy)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 해제 시 다수 스레드가 같은 락 획득 시도
> 2. **가치**: 처리량 급감, 높은 지연의 원인
> 3. **융합**: 락 스로틀링, 락 경합과 연관

---

## Ⅰ. 개요

### 개념 정의

락 컨보이(Lock Convoy)는 **락 해제 시 다수 스레드가 연이어 같은 락 획득을 시도하는 현상**이다. 처리량이 급격히 감소하고 지연이 증가한다.

### 💡 비유: 고속도로 정체
락 컨보이는 **고속도로로 정체**와 같다. 앞차가 천천히 가면 뒤차가 느리게 간다. 모두 느리게 가면 도로가 비효율적이다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 컨보이 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컨보이 vs 경합】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  락 경합 (Contention):                                            │ │
│  │  • 랜덤한 시점에 발생                                         │ │
│  │  • 일부 스레드만 영향                                         │ │
│  │  • 짧은 기간                                                │ │
│  │                                                             │ │
│  │  락 컨보이 (Convoy):                                              │ │
│  │  • 지속적인 패턴                                               │ │
│  │  • 모든 스레드가 영향                                          │ │
│  │  • 긴 기간                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【감지 방법】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 락 대기 시간 모니터링                                         │ │
│  │  2. 락 큐 길이 추적                                              │ │
│  │  3. 처리량 변화 감지                                              │ │
│  │  4. 락 보유 시간 분석                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결 방법】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 락 세분화 (Lock Granularity):                                 │ │
│  │     • 하나의 큰 락 → 여러 작은 락                               │ │
│  │                                                             │ │
│  │  2. 락 스트라이핑 (Lock Striping):                                 │ │
│  │     • 여러 락 인스턴스로 분산                                  │ │
│  │                                                             │ │
│  │  3. 타임아웃 (Timeout):                                          │ │
│  │     • 무한 대기 방지                                             │ │
│  │                                                             │ │
│  │  4. 짧은 임계 영역:                                               │ │
│  │     • 락 보유 시간 최소화                                         │ │
│  │                                                             │ │
│  │  5. 비동기 I/O:                                                   │ │
│  │     • 락 밖에서 I/O 수행                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컨보이 감지】                                                        │
│  ──────────────────                                                  │
│  // Java: 락 대기 시간 모니터링                                      │
│  long start = System.nanoTime();                                     │
│  lock.lock();                                                        │
│  long waitTime = System.nanoTime() - start;                         │
│  if (waitTime > THRESHOLD_NS) {                                      │
│      log.warn("Lock convoy detected: " + waitTime + "ns");           │
│  }                                                                  │
│                                                                     │
│  // Linux: perf로 분석                                                │
│  perf record -e lock:lock_acquire -e lock:lock_release -a sleep 10  │
│  perf script -g lock-convoys.py                                     │
│                                                                     │
│  【해결: 락 세분화】                                                    │
│  ──────────────────                                                  │
│  // Before: 하나의 큰 락                                              │
│  synchronized (bigLock) {                                            │
│      updateA();                                                       │
│      updateB();                                                       │
│      updateC();  // 모두 같은 락                                        │
│  }                                                                  │
│                                                                     │
│  // After: 여러 작은 락                                               │
│  synchronized (lockA) { updateA(); }                                 │
│  synchronized (lockB) { updateB(); }                                 │
│  synchronized (lockC) { updateC(); }  // 각각 다른 락                  │
│                                                                     │
│  【해결: 락 스트라이핑】                                                │
│  ──────────────────                                                  │
│  class StripedLock {                                                 │
│      private static final int STRIPES = 16;                           │
│      private final Object[] locks = new Object[STRIPES];             │
│                                                                     │
│      public Object getLock(Object key) {                             │
│          return locks[Math.abs(key.hashCode() % STRIPES)];           │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  // 사용                                                               │
│  StripedLock striped = new StripedLock();                            │
│  synchronized (striped.getLock(userId)) {                            │
│      // userId 기반 락                                                │
│  }                                                                  │
│                                                                     │
│  【해결: 짧은 임계 영역】                                               │
│  ──────────────────                                                  │
│  // Before: I/O 포함                                                   │
│  synchronized (lock) {                                               │
│      readDataFromFile();  // I/O - 락 보유 중!                          │
│      processData();                                                   │
│  }                                                                  │
│                                                                     │
│  // After: I/O 제외                                                   │
│  Data data;                                                           │
│  synchronized (lock) {                                               │
│      data = readFromCache();  // 캐시에서만 읽기                         │
│  }                                                                  │
│  readDataFromFile(data);  // 락 없이 I/O                              │
│  synchronized (lock) {                                               │
│      processData(data);                                              │
│  }                                                                  │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시나리오           처리량           평균 지연    최대 지연          │ │
│  │  ────────           ──────           ──────     ──────              │ │
│  │  컨보이 발생         100 ops/s        500ms       5000ms             │ │
│  │  세분화 후           800 ops/s        50ms        200ms              │ │
│  │  스트라이핑 후        900 ops/s        40ms        150ms              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예방 가이드라인】                                                    │
│  ──────────────────                                                  │
│  • 임계 영역: 가능한 짧게                                              │
│  • I/O: 락 밖에서 수행                                                 │
│  • 모니터링: 대기 시간 추적                                             │
│  • 세분화: 자주 변경되는 데이터 분리                                    │
│  • 알림: 컨보이 감지 시 즉시 조치                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

