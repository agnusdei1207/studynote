# 297. 상호 배제 (Mutual Exclusion)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 번에 하나만 자원 사용 가능
> 2. **가치**: 데이터 무결성 보장의 핵심
> 3. **융합**: 뮤텍스, 세마포어, 교착상태와 연관

---

## Ⅰ. 개요

### 개념 정의

상호 배제(Mutual Exclusion)는 **한 번에 오직 하나의 프로세스/스레드만 공유 자원에 접근할 수 있는 속성**이다. 동시성 제어의 가장 기본적인 요구사항이다.

### 💡 비유: 화장실 열쇠
상호 배제는 **화장실 열쇠**와 같다. 열쇠가 하나뿐이다. 한 사람이 사용 중이면 다른 사람은 기다려야 한다.

### 상호 배제 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                상호 배제 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  자원 R에 대해:                                                      │ │
│  │                                                             │ │
│  │  [P1] ──► R ──┐                                                     │ │
│  │                    │                                                │ │
│  │  [P2] ──► R ──┼──► 한 번에 하나만 접근                               │ │
│  │                    │                                                │ │
│  │  [P3] ──► R ──┘                                                     │ │
│  │                                                             │ │
│  │  T1이 R을 사용 중이면 T2, T3는 대기                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【임계 영역 (Critical Section)】                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  entry section: 락 획득                                              │ │
│  │  ┌───────────────────────────────────────────────────────────┐│ │
│  │  │  lock(&mutex);                                             ││ │
│  │  └───────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  │  critical section: 공유 자원 접근                                     │ │
│  │  ┌───────────────────────────────────────────────────────────┐│ │
│  │  │  // 한 번에 하나의 스레드만 실행                                ││ │
│  │  │  shared_var++;                                              ││ │
│  │  │  update_resource();                                         ││ │
│  │  └───────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  │  exit section: 락 해제                                               │ │
│  │  ┌───────────────────────────────────────────────────────────┐│ │
│  │  │  unlock(&mutex);                                           ││ │
│  │  └───────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                상호 배제 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【상호 배제가 필요한 이유】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터 무결성:                                                       │ │
│  │  // 상호 배제 없이                                                    │
│  │  counter++;  // read, modify, write                                 │
│  │  // 두 스레드가 동시에 실행하면丢失 업데이트 발생                          │ │
│  │                                                             │ │
│  │  리소스 보호:                                                         │ │
│  │  • 프린터: 한 번에 한 문서                                              │ │
│  │  • 파일 쓰기: 한 번에 한 프로세스                                        │ │
│  │  • 하드웨어 레지스터                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상호 배제 구현 방법】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  소프트웨어:                                                           │ │
│  │  • Peterson 알고리즘                                                 │ │
│  │  • Dekker 알고리즘                                                   │ │
│  │  • Lamport's Bakery 알고리즘                                          │ │
│  │                                                             │ │
│  │  하드웨어:                                                             │ │
│  │  • Test-and-Set (TAS)                                               │ │
│  │  • Compare-and-Swap (CAS)                                           │ │
│  │  • Load-Linked/Store-Conditional (LL/SC)                            │ │
│  │                                                             │ │
│  │  OS 지원:                                                             │ │
│  │  • Mutex (뮤텍스)                                                     │ │
│  │  • Semaphore (세마포어)                                               │ │
│  │  • Spinlock (스핀락)                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상호 배제 요구사항】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 안전성 (Safety):                                                 │ │
│  │     한 번에 최대 하나의 프로세스만 임계 영역에                              │ │
│  │                                                             │ │
│  │  2. 진행 (Progress):                                                 │ │
│  │     임계 영역이 비어있으면 누군가 진입해야                                  │ │
│  │                                                             │ │
│  │  3. 유한 대기 (Bounded Waiting):                                      │ │
│  │     무한정 기다리면 안 됨                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ std::mutex】                                                   │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│                                                                     │
│  std::mutex mtx;                                                     │
│  int shared_counter = 0;                                            │
│                                                                     │
│  void increment() {                                                  │
│      mtx.lock();                                                     │
│      shared_counter++;  // 임계 영역                                   │
│      mtx.unlock();                                                   │
│  }                                                                  │
│                                                                     │
│  // RAII 패턴 (권장)                                                   │
│  void safe_increment() {                                            │
│      std::lock_guard<std::mutex> lock(mtx);                         │
│      shared_counter++;  // 임계 영역                                   │
│  }  // 자동 해제                                                       │
│                                                                     │
│  【Java synchronized】                                                │
│  ──────────────────                                                  │
│  // 메서드 레벨                                                         │
│  public synchronized void increment() {                             │
│      counter++;  // 임계 영역                                          │
│  }                                                                  │
│                                                                     │
│  // 블록 레벨                                                          │
│  public void increment() {                                          │
│      synchronized(this) {                                           │
│          counter++;  // 임계 영역                                      │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  // ReentrantLock                                                   │
│  private final ReentrantLock lock = new ReentrantLock();            │
│  public void increment() {                                          │
│      lock.lock();                                                   │
│      try {                                                          │
│          counter++;                                                 │
│      } finally {                                                    │
│          lock.unlock();                                             │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Python threading.Lock】                                            │
│  ──────────────────                                                  │
│  import threading                                                   │
│                                                                     │
│  lock = threading.Lock()                                            │
│  counter = 0                                                        │
│                                                                     │
│  def increment():                                                   │
│      with lock:  # 상호 배제                                          │
│          global counter                                             │
│          counter += 1                                               │
│                                                                     │
│  【Go sync.Mutex】                                                     │
│  ──────────────────                                                  │
│  var (                                                               │
│      mu      sync.Mutex                                             │
│      counter int                                                    │
│  )                                                                  │
│                                                                     │
│  func increment() {                                                  │
│      mu.Lock()                                                       │
│      defer mu.Unlock()  // 상호 배제                                   │
│      counter++                                                       │
│  }                                                                  │
│                                                                     │
│  【상호 배제 없이 발생하는 문제】                                          │
│  ──────────────────                                                  │
│  // 두 스레드가 동시에 counter++ 실행                                     │
│  // counter = 0에서 시작                                               │
│                                                                     │
│  // 스레드 1:                          스레드 2:                        │
│  // read counter (0)                  read counter (0)               │
│  // increment (1)                     increment (1)                  │
│  // write counter (1)                 write counter (1)              │
│                                                                     │
│  // 결과: counter = 1 (2가 되어야 함!)                                   │
│  // Lost Update 발생                                                  │
│                                                                     │
│  【성능 영향】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스레드 수    상호 배제 있음    상호 배제 없음 (오류)                   │ │
│  │  ────────     ────────────     ────────────────                  │ │
│  │  1            정상             정상                              │ │
│  │  2            정상             ~50% 오류                          │ │
│  │  4            정상             ~75% 오류                          │ │
│  │  8            정상             ~87% 오류                          │ │
│  │                                                             │ │
│  │  상호 배제 = 정확성 보장                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 한 번에 하나만 자원 접근
• 목적: 데이터 무결성 보장
• 구현: 뮤텍스, 세마포어, synchronized
• 요구사항: 안전성, 진행, 유한 대기
• 임계 영역: entry → critical → exit
• 중요성: 동시성 제어의 기본
```
