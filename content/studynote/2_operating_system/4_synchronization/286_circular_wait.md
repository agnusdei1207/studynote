# 300. 순환 대기 (Circular Wait)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 대기 그래프에 순환이 존재
> 2. **가치**: 교착상태 조건 4 (필요충분)
> 3. **융합**: 락 순서, 대기 그래프와 연관

---

## Ⅰ. 개요

### 개념 정의

순환 대기(Circular Wait)는 **프로세스들의 대기 관계가 원형을 이루는 상태**다. P1이 P2를, P2가 P3를, ... Pn이 P1을 대기하는 순환이다. 교착상태 4가지 조건 중 마지막이자 가장 중요하다.

### 💡 비유: 원탁 회의
순환 대기는 **원탁 회의의 마이크**와 같다. 각 사람이 자기 옆 사람의 마이크가 필요하다. 모두가 옆 사람을 기다리면 아무도 말할 수 없다.

### 순환 대기 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                순환 대기 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【순환 대기 시나리오】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │       [P1] ──► R1 보유, R2 요청 ──► [P2]                        │ │
│  │         ▲                              │                          │ │
│  │         │                              ▼                          │ │
│  │       [P4] ◄── R4 요청, R3 보유 ── [P3]                          │ │
│  │                                                             │ │
│  │  대기 그래프: P1 → P2 → P3 → P4 → P1 (순환!)                       │ │
│  │                                                             │ │
│  │  P1: R1 보유, R2 요청 (P2가 R2 보유)                               │ │
│  │  P2: R2 보유, R3 요청 (P3이 R3 보유)                               │ │
│  │  P3: R3 보유, R4 요청 (P4가 R4 보유)                               │ │
│  │  P4: R4 보유, R1 요청 (P1이 R1 보유)                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대기 그래프】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  노드: 프로세스                                                       │ │
│  │  간선: P1 → P2 (P1이 P2가 보유한 자원을 대기)                          │ │
│  │                                                             │ │
│  │       [P1] ────────► [P2]                                          │ │
│  │         ▲                 │                                        │ │
│  │         │                 ▼                                        │ │
│  │       [P4] ◄─────── [P3]                                          │ │
│  │                                                             │ │
│  │  순환 존재 = 교착상태!                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【필요충분 조건】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  4가지 조건이 모두 만족 + 순환 대기 = 교착상태 확정                       │ │
│  │                                                             │ │
│  │  순환 대기는 다른 3가지가 만족될 때                                      │ │
│  │  교착상태의 필요충분조건이 됨                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                순환 대기 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【무력화 방법: 락 순서 규칙】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  가장 실용적인 교착상태 예방 방법:                                        │ │
│  │                                                             │ │
│  │  1. 모든 자원에 전역 순서 번호 부여                                       │ │
│  │     R1 < R2 < R3 < ... < Rn                                       │ │
│  │                                                             │ │
│  │  2. 항상 오름차순으로만 자원 요청                                          │ │
│  │     R_i를 보유하면 R_j (j > i)만 요청 가능                             │ │
│  │                                                             │ │
│  │  3. 순환 대기 불가능                                                   │ │
│  │     P1 → P2 → P3 → ... → Pn                                       │ │
│  │     각 단계에서 자원 번호가 증가해야 함                                      │ │
│  │     마지막에서 처음으로 돌아갈 수 없음 (번호 감소)                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【증명: 순서 규칙으로 순환 대기 불가】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  순환이 있다고 가정: P1 → P2 → P3 → ... → Pn → P1                    │ │
│  │                                                             │ │
│  │  P1이 R_a 보유, R_b 요청 (a < b)                                     │ │
│  │  P2가 R_b 보유, R_c 요청 (b < c)                                     │ │
│  │  ...                                                               │ │
│  │  Pn이 R_y 보유, R_z 요청 (y < z)                                     │ │
│  │  P1이 R_z 보유, R_a 요청 (z < a) ← 모순!                              │ │
│  │                                                             │ │
│  │  따라서 순환 불가능                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                               │ │
│  │  • 구현 용이                                                          │ │
│  │  • 성능 저하 적음                                                      │ │
│  │  • 교착상태 확실히 예방                                                  │ │
│  │  • 널리 사용됨                                                         │ │
│  │                                                             │ │
│  │  단점:                                                               │ │
│  │  • 모든 자원에 순서 부여 필요                                            │ │
│  │  • 동적 자원에는 적용 어려움                                              │ │
│  │  • 필요하지 않은 자원도 미리 알아야 할 수 있음                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락 순서 규칙 구현】                                                   │
│  ──────────────────                                                  │
│  // 전역 락 순서 정의                                                    │
│  #define LOCK_ORDER_DATABASE  1                                     │
│  #define LOCK_ORDER_FILE      2                                     │
│  #define LOCK_ORDER_NETWORK   3                                     │
│  #define LOCK_ORDER_CACHE     4                                     │
│                                                                     │
│  // 올바른 사용: 오름차순                                                 │
│  void correct_order() {                                              │
│      lock(LOCK_ORDER_DATABASE);   // 1                               │
│      lock(LOCK_ORDER_FILE);       // 2                               │
│      lock(LOCK_ORDER_NETWORK);    // 3                               │
│                                                                     │
│      // 작업 수행                                                      │
│                                                                     │
│      unlock(LOCK_ORDER_NETWORK);                                     │
│      unlock(LOCK_ORDER_FILE);                                        │
│      unlock(LOCK_ORDER_DATABASE);                                    │
│  }                                                                  │
│                                                                     │
│  // 잘못된 사용: 내림차순 (교착상태 가능!)                                   │
│  void wrong_order() {                                                │
│      lock(LOCK_ORDER_NETWORK);    // 3                               │
│      lock(LOCK_ORDER_DATABASE);   // 1 ← 위험!                        │
│  }                                                                  │
│                                                                     │
│  【계좌 이체 예시】                                                      │
│  ──────────────────                                                  │
│  // 계좌 ID로 순서 결정                                                  │
│  void transfer(Account* from, Account* to, int amount) {            │
│      // ID 순서로 락 획득                                                │
│      Account* first = from->id < to->id ? from : to;                │
│      Account* second = from->id < to->id ? to : from;               │
│                                                                     │
│      pthread_mutex_lock(&first->lock);                              │
│      pthread_mutex_lock(&second->lock);                             │
│                                                                     │
│      from->balance -= amount;                                        │
│      to->balance += amount;                                          │
│                                                                     │
│      pthread_mutex_unlock(&second->lock);                           │
│      pthread_mutex_unlock(&first->lock);                            │
│  }                                                                  │
│                                                                     │
│  // 이제 A→B, B→A 동시 실행해도 교착상태 없음                               │
│  // 항상 낮은 ID의 락을 먼저 획득하기 때문                                    │
│                                                                     │
│  【순환 탐지 알고리즘】                                                    │
│  ──────────────────                                                  │
│  bool hasCycle() {                                                   │
│      Set<Node> visited = new HashSet<>();                           │
│      Set<Node> recStack = new HashSet<>();                          │
│                                                                     │
│      for (Node node : graph.nodes()) {                              │
│          if (dfs(node, visited, recStack))                          │
│              return true;                                            │
│      }                                                              │
│      return false;                                                   │
│  }                                                                  │
│                                                                     │
│  bool dfs(Node node, Set<Node> visited, Set<Node> recStack) {       │
│      if (recStack.contains(node)) return true;  // 순환 발견!          │
│      if (visited.contains(node)) return false;                      │
│                                                                     │
│      visited.add(node);                                              │
│      recStack.add(node);                                             │
│                                                                     │
│      for (Node neighbor : node.neighbors()) {                        │
│          if (dfs(neighbor, visited, recStack))                      │
│              return true;                                            │
│      }                                                              │
│                                                                     │
│      recStack.remove(node);                                          │
│      return false;                                                   │
│  }                                                                  │
│                                                                     │
│  【Linux lockdep】                                                    │
│  ──────────────────                                                  │
│  // 커널 락 순서 검증                                                    │
│  CONFIG_LOCKDEP=y                                                    │
│                                                                     │
│  // 순환 대기 가능성 경고                                                 │
│  dmesg | grep "circular locking dependency"                         │
│                                                                     │
│  // lockdep는 런타임에 락 순서를 추적하고                                   │
│  // 순환 가능성을 감지                                                   │
│                                                                     │
│  【실제 사용 예시】                                                      │
│  ──────────────────                                                  │
│  • 데이터베이스: 테이블 락 순서                                             │
│  • 파일 시스템: inode 락 순서                                             │
│  • 네트워크: 연결 락 순서                                                 │
│  • 커널: lockdep로 자동 검증                                             │
│                                                                     │
│  【성능 분석】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전략              오버헤드    교착상태 예방    구현 난이도              │ │
│  │  ────────          ──────     ────────────     ────────          │ │
│  │  순서 규칙          매우 낮음    100%            낮음                  │ │
│  │  탐지 + 복구        중간       발생 후 처리      높음                  │ │
│  │  회피 (은행원)       높음       100%            매우 높음              │ │
│  │                                                             │ │
│  │  순서 규칙이 가장 실용적                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
