# 255. 순차 락 (Seqlock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 순차 카운터로 읽기-쓰기 동기화
> 2. **가치**: 읽기 락 없이 일관성 보장
> 3. **융합**: RCU, 버전 카운터, 읽기-쓰기 락과 연관

---

## Ⅰ. 개요

### 개념 정의

순차 락(Seqlock)은 **순차 카운터를 사용하여 읽기에서 락 없이 일관된 데이터를 읽는 동기화 메커니즘**이다. 쓰기는 배타적 락으로 보호하고, 읽기는 버전 확인으로 일관성을 보장한다.

### 💡 비유: 잡지 읽기
순차 락은 **잡지 읽기**와 같다. 읽기 시작할 때 페이지 번호를 확인한다. 읽기 끝날 때 다시 확인한다. 번호가 같으면 중간에 바뀌지 않은 것이다.

### 순차 락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                순차 락 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 구조】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct seqlock {                                                 │ │
│  │      unsigned sequence;  // 순차 번호                              │ │
│  │      spinlock_t lock;    // 쓰기 락                                │ │
│  │  };                                                              │ │
│  │                                                             │ │
│  │  쓰기: sequence++ (홀수) → 데이터 수정 → sequence++ (짝수)          │ │
│  │  읽기: seq1 읽기 → 데이터 읽기 → seq2 읽기 → seq1 == seq2 확인        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【읽기 동작】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  do {                                                              │ │
│  │      seq1 = sequence;  // 홀수면 쓰기 중                            │ │
│  │      if (seq1 & 1) continue;  // 쓰기 중이면 재시도                  │ │
│  │                                                             │ │
│  │      // 락 없이 데이터 읽기                                          │ │
│  │      data = read_data();                                          │ │
│  │                                                             │ │
│  │      seq2 = sequence;  // 다시 확인                                 │ │
│  │  } while (seq1 != seq2);  // 변경되었으면 재시도                     │ │
│  │                                                             │ │
│  │  // seq1 == seq2: 읽기 중 쓰기 없었음 → 일관된 데이터               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【쓰기 동작】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  spin_lock(&lock);                                                │ │
│  │                                                             │ │
│  │  sequence++;  // 홀수: 쓰기 시작                                    │ │
│  │  write_data(new_data);                                            │ │
│  │  sequence++;  // 짝수: 쓰기 완료                                    │ │
│  │                                                             │ │
│  │  spin_unlock(&lock);                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시퀀스 번호 상태】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  0 (짝수): 쓰기 없음, 안전하게 읽기 가능                            │ │
│  │  1 (홀수): 쓰기 진행 중, 읽기 대기                                  │ │
│  │  2 (짝수): 쓰기 완료, 안전하게 읽기 가능                            │ │
│  │  3 (홀수): 쓰기 진행 중, 읽기 대기                                  │ │
│  │  ...                                                              │ │
│  │                                                             │ │
│  │  홀수 = 쓰기 중, 짝수 = 쓰기 완료                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                순차 락 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【vs 다른 동기화 메커니즘】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  뮤텍스:                                                            │ │
│  │  • 읽기/쓰기 모두 락 필요                                             │ │
│  │  • 읽기끼리도 배타적                                                  │ │
│  │                                                             │ │
│  │  읽기-쓰기 락:                                                       │ │
│  │  • 읽기는 공유 락                                                    │ │
│  │  • 여전히 락 오버헤드 있음                                              │ │
│  │                                                             │ │
│  │  RCU:                                                              │ │
│  │  • 읽기는 완전 락 없음                                                 │ │
│  │  • 쓰기는 복사 필요                                                   │ │
│  │  • 메모리 오버헤드                                                    │ │
│  │                                                             │ │
│  │  Seqlock:                                                           │ │
│  │  • 읽기는 락 없음, 버전만 확인                                          │ │
│  │  • 쓰기는 스핀락                                                      │ │
│  │  • 쓰기 빈번하면 읽기 재시도 많음                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 배리어】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  읽기:                                                              │ │
│  │  • smp_rmb(): 읽기 배리어                                            │ │
│  │  • seq1 읽기 전에 데이터 읽기 방지                                       │ │
│  │  • seq2 읽기 전에 데이터 읽기 완료                                      │ │
│  │                                                             │ │
│  │  쓰기:                                                              │ │
│  │  • smp_wmb(): 쓰기 배리어                                            │ │
│  │  • sequence 증가 전에 데이터 쓰기 완료                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적합한 사용 사례】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 시간 정보 (jiffies)                                               │ │
│  │  • 통계 카운터                                                         │ │
│  │  • 읽기 빈번, 쓰기 드문 데이터                                          │ │
│  │  • 작은 데이터 구조                                                    │ │
│  │  • 쓰기가 짧은 경우                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 seqlock】                                               │
│  ──────────────────                                                  │
│  #include <linux/seqlock.h>                                         │
│                                                                     │
│  // 정의                                                              │
│  DEFINE_SEQLOCK(sl);                                                 │
│                                                                     │
│  // 쓰기                                                              │
│  write_seqlock(&sl);                                                 │
│  // ... 데이터 수정 ...                                                │
│  write_sequnlock(&sl);                                               │
│                                                                     │
│  // 읽기                                                              │
│  unsigned int seq;                                                   │
│  do {                                                                │
│      seq = read_seqbegin(&sl);                                       │
│      // ... 데이터 읽기 ...                                            │
│  } while (read_seqretry(&sl, seq));                                  │
│                                                                     │
│  【시간 읽기 (jiffies_64)】                                            │
│  ──────────────────                                                  │
│  u64 get_jiffies_64(void) {                                         │
│      unsigned int seq;                                               │
│      u64 val;                                                         │
│                                                                     │
│      do {                                                            │
│          seq = read_seqbegin(&jiffies_seqlock);                      │
│          val = jiffies_64;                                           │
│      } while (read_seqretry(&jiffies_seqlock, seq));                 │
│                                                                     │
│      return val;                                                      │
│  }                                                                   │
│                                                                     │
│  【C++ 구현 (간단 버전)】                                              │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  #include <mutex>                                                    │
│                                                                     │
│  template<typename T>                                                │
│  class SeqLock {                                                      │
│      std::atomic<unsigned> seq{0};                                   │
│      std::mutex write_mtx;                                            │
│      T data;                                                          │
│  public:                                                              │
│      T read() {                                                      │
│          unsigned s1, s2;                                             │
│          T val;                                                       │
│          do {                                                         │
│              s1 = seq.load(std::memory_order_acquire);                │
│              if (s1 & 1) continue;  // 쓰기 중                          │
│              val = data;  // 읽기 (실제로는 원자적 필요)                  │
│              s2 = seq.load(std::memory_order_acquire);                │
│          } while (s1 != s2);                                          │
│          return val;                                                   │
│      }                                                                │
│                                                                     │
│      void write(const T& val) {                                      │
│          std::lock_guard<std::mutex> lock(write_mtx);                │
│          seq.fetch_add(1, std::memory_order_release);  // 홀수         │
│          data = val;                                                  │
│          seq.fetch_add(1, std::memory_order_release);  // 짝수         │
│      }                                                                │
│  };                                                                   │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  메커니즘      읽기 비용    쓰기 비용    읽기 동시성              │ │
│  │  ────────      ──────       ──────        ────────                │ │
│  │  뮤텍스        락 획득     락 획득        배타적                  │ │
│  │  RW 락        공유 락      배타 락        공유                    │ │
│  │  Seqlock      버전 확인    스핀락        락 없음                  │ │
│  │  RCU          없음         복사+RCU      없음                    │ │
│  │                                                             │ │
│  │  Seqlock: 읽기 많을 때 우수                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 순차 카운터로 읽기 일관성 보장
• 구조: sequence + spinlock
• 읽기: seq1 → 데이터 읽기 → seq2 → 비교
• 쓰기: sequence++ → 수정 → sequence++
• 장점: 읽기 락 없음, 높은 읽기 동시성
• 단점: 쓰기 빈번하면 읽기 재시도 많음
• 적합: 읽기 빈번, 쓰기 드문 데이터
```
