# 271. 락 홀더 (Lock Holder)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 현재 락을 보유 중인 스레드
> 2. **가치**: 교착상태 분석, 락 디버깅의 핵심
> 3. **융합**: 교착상태, 락 순서, 락 그래프와 연관

---

## Ⅰ. 개요

### 개념 정의

락 홀더(Lock Holder)는 **현재 락을 보유 중인 스레드**를 의미한다. 교착상태 분석, 디버깅, 락 타임아웃 감지에 핵심 정보다.

### 💡 비유: 열쇠 보유자
락 홀더는 **열쇠 보유자**와 같다. 문을 열고 있는 사람. 누가 문을 잠갔는지 알면 대기할지 말지 결정할 수 있다.

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 락 내부에 owner 필드 유지                                        │ │
│  │  • 락 획득 시 owner = current_thread                                │ │
│  │  • 락 해제 시 owner = null                                          │ │
│  │  • 디버깅/분석 시 owner 확인                                        │ │

│  │  1. 교착상태 진단:                                                   │ │
│  │     • 누가 락을 보유 중인지 확인                                      │ │
│  │     • 대기 그래프 분석                                               │ │
│  │                                                             │ │
│  │  2. 락 타임아웃:                                                     │ │

│  │  3. 디버깅:                                                          │ │
│  │     • 락 보유자 스택 트레이스                                          │ │
│  │     • 락 보유 시간 측정                                               │ │
│  │                                                             │ │
│  │  4. 모니터링:                                                        │ │
│  │     • 락 보유 패턴 분석                                               │ │

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 홀더 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구현 방법】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. atomic 포인터:                                                 │ │
│  │     atomic<thread::id> owner;                                      │ │
│  │                                                             │ │
│  │  2. 커널 락 구조체:                                                 │ │
│  │     struct mutex {                                                  │ │
│  │         atomic_t count;                                             │ │
│  │         spinlock_t wait_lock;                                       │ │
│  │         struct list_head wait_list;                                 │ │
│  │         struct task_struct *owner;  // 락 홀더                       │ │
│  │     };                                                              │ │
│  │                                                             │ │
│  │  3. 디버깅 정보:                                                    │ │
│  │     • lockdep: 락 의존성 추적                                        │ │
│  │     • lock_stat: 락 통계                                            │ │
│  │     • ftrace: 락 이벤트 추적                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【교착상태 분석에서 활용】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 락 대기 그래프:                                                  │ │
│  │     T1 → [락A 대기] → T2 (락A 홀더)                                 │ │
│  │     T2 → [락B 대기] → T3 (락B 홀더)                                 │ │
│  │     T3 → [락A 대기] → T2 (락A 홀더)                                 │ │
│  │     → 순환! 교착상태                                                │ │
│  │                                                             │ │
│  │  2. 락 홀더 확인:                                                   │ │
│  │     /proc/lockdep                                                  │ │
│  │     sysrq-t (스택 덤프)                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ 락 홀더 추적】                                                  │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  #include <thread>                                                   │
│                                                                     │
│  class TrackedMutex {                                                │
│      std::atomic<std::thread::id> owner{};                           │
│      std::atomic<uint64_t> acquire_time{0};                          │
│                                                                     │
│  public:                                                            │
│      void lock() {                                                  │
│          internal_lock();                                            │
│          owner.store(std::this_thread::get_id());                    │
│          acquire_time.store(get_timestamp());                        │
│      }                                                              │
│                                                                     │
│      void unlock() {                                                │
│          owner.store(std::thread::id{});                             │
│          acquire_time.store(0);                                      │
│          internal_unlock();                                          │
│      }                                                              │
│                                                                     │
│      std::thread::id get_owner() const {                            │
│          return owner.load();                                        │
│      }                                                              │
│                                                                     │
│      uint64_t get_hold_duration() const {                           │
│          auto start = acquire_time.load();                           │
│          if (start == 0) return 0;                                   │
│          return get_timestamp() - start;                             │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【Linux 커널 mutex owner】                                           │
│  ──────────────────                                                  │
│  // kernel/locking/mutex.c                                          │
│  // mutex->owner 필드로 홀더 추적                                      │
│                                                                     │
│  // cat /proc/lockdep                                               │
│  // 또는 echo t > /proc/sysrq-trigger                                │
│                                                                     │
│  【Java ThreadMXBean】                                                │
│  ──────────────────                                                  │
│  ThreadMXBean threadBean = ManagementFactory.getThreadMXBean();     │
│  long ownerId = lock.getOwnerThreadId();                            │
│  ThreadInfo ownerInfo = threadBean.getThreadInfo(ownerId);          │
│                                                                     │
│  【교착상태 덤프】                                                      │
│  ──────────────────                                                  │
│  // Linux                                                            │
│  echo t > /proc/sysrq-trigger                                       │
│  // 또는 echo 1 > /proc/sys/kernel/sysrq                             │
│  // Ctrl+Break (Windows)                                            │
│                                                                     │
│  【락 홀더 정보 확인】                                                  │
│  ──────────────────                                                  │
│  // pstack (Solaris)                                                │
│  // jstack (Java)                                                    │
│  // gstack (GDB)                                                     │
│  // gdb: thread apply all bt                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
