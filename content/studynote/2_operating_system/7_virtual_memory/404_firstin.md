# 비밀번호 솔트 (Password Salting)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 해시 함수에 무작위 값을 추가(Salt)하여 레인보우 테이블 공격을 방어하는 기법
> 2. **가치**: 동일 비밀번호라도 다른 해시값을 생성하여 무차별 대입 공격 비용을 증가
> 3. **융합**: 해시 함수, 키 스트레칭, 안전한 저장소(bcrypt, Argon2)의 조합

---

### Ⅰ. 개요 (Context & Background)

**개념**
솔트(Salt)는 **비밀번호 해시에 추가되는 무작위 데이터**로, 동일한 비밀번호가 해시될 때마다 다른 결과값을 생성하게 한다. 이는 레인보우 테이블(Rainbow Table) 공격을 방지하고, 해시 미리 계산을 무력화한다. 솔트는 각 비밀번호마다 고유해야 하며, 해시값과 함께 저장된다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         솔트 없는 해시 vs 솔트 사용                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   문제: 솔트 없는 해시                                                      │
│   ───────────────────────────────────────────────────────────────────────   │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  Alice의 비밀번호: "password123"                                  │  │
│   │  Bob의 비밀번호:   "password123"                                  │  │
│   │                                                                     │  │
│   │  Hash("password123") = 5e884898da28047151d0e56f8dc6292773603d0d...  │  │
│   │  Hash("password123") = 5e884898da28047151d0e56f8dc6292773603d0d...  │  │
│   │                                                                     │  │
│   │  동일한 비밀번호 → 동일한 해시!                                    │  │
│   │  → 공격자가 레인보우 테이블로 미리 계산된 해시로 찾을 수 있음       │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   해결: 솔트 사용                                                          │
│   ───────────────────────────────────────────────────────────────────────   │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  Alice의 솔트:   "a7f3b2c1"                                        │  │
│   │  Bob의 솔트:     "e9d4c8f5"                                        │  │
│   │                                                                     │  │
│   │  Hash("password123" + "a7f3b2c1")                                   │  │
│   │    = 8f3d2a1e... (Alice의 해시)                                   │  │
│   │                                                                     │  │
│   │  Hash("password123" + "e9d4c8f5")                                   │  │
│   │    = 2b7e9c4a... (Bob의 해시)                                     │  │
│   │                                                                     │  │
│   │  동일한 비밀번호 → 다른 해시!                                      │  │
│   │  → 레인보우 테이블 무효화, 무차별 대입 필요                       │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
솔트는 **요리에 각 개인만의 비밀 양념**을 넣는 것과 같다.
- 같은 재료(비밀번호): 소금기름
- 각자 다른 양념(솔트): 천일염, 후추, 허브
- 결과: 같은 재료로 다른 맛

**등장 배경**
① **1970년**: UNIX 해싱(crypt) 도입
② **2000년대**: 레인보우 테이블 공격 대두
③ **2010년대**: bcrypt, Argon2 보급
④ **현재**: Argon2, scrypt 표준

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 기법 | 솔트 길이 | 저장 | 재사용 | 보안성 |
|:---|:---|:---|:---|:---|
| **단순 솔트** | 8-16바이트 | 평문 + 해시 | 금지 | 중간 |

**해설**
1. **솔트 생성**: CSPRNG로 고유한 난수 생성
2. **해싱**: 솔트 + 비밀번호를 해시
3. **저장**: 솔트와 해시를 함께 저장
4. **검증**: 저장된 솔트로 동일하게 해싱 후 비교

**심층 동작 원리: 키 스트레칭 (Key Stretching)**

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                       키 스트레칭 (PBKDF2, bcrypt, Argon2)                    │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   목적: 무차별 대입 공격 비용 증가                                           │
│   ┌───────────────────────────────────────────────────────────────────────┐ │
│   │  문제: 단일 해시는 너무 빠름                                         │ │
│   │  • SHA-256: 1초에 수천만 번 연산 가능                              │ │
│   │  • 공격자: 1초에 수천만 비밀번호 시도                              │ │
│   │                                                                      │ │
│   │  해결: 해시를 여러 번 반복 (반복 횟수를 조정)                        │ │
│   │                                                                      │ │
│   │  PBKDF2 (Password-Based Key Derivation Function 2):                  │ │
│   │  ┌────────────────────────────────────────────────────────────────┐ │ │
│   │  │  Hash = PBKDF2(password, salt, iterations)                    │ │ │
│   │  │                                                                │ │ │
│   │  │  iterations = 100,000 (현재 권장)                             │ │ │
│   │  │                                                                │ │ │
│   │  │  단일 해시: ~0.000001초                                       │ │ │
│   │  │  100,000번 반복: ~0.1초                                       │ │ │
│   │  │                                                                │ │ │
│   │  │  → 공격자는 1초에 10번만 시도 가능!                           │ │ │
│   │  │  → 공격 비용 1,000,000배 증가                                │ │ │
│   │  └────────────────────────────────────────────────────────────────┘ │ │
│   │                                                                      │ │
│   │  bcrypt:                                                           │ │
│   │  • Blowfish 기반                                                  │ │
│   │  • cost 파라미터로 연산량 조정                                    │ │
│   │  • 솔트 내장                                                      │ │
│   │  • 메모리 연산에 취약하지 않음                                    │ │
│   │                                                                      │ │
│   │  Argon2 (2015년 Password Hashing Competition 우승):               │ │
│   │  • 메모리 하드 기반 (GPU 공격 방지)                               │ │
│   │  • 시간 및 메모리 비용 조정 가능                                  │ │
│   │  • 병렬 처리 지원                                                 │ │
│   │  • 현재 최신 표준                                                │ │
│   │                                                                      │ │
│   └───────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│   예제 (Python):                                                            │
│   ┌───────────────────────────────────────────────────────────────────────┐ │
│   │  import hashlib                                                      │ │
│   │  import os                                                           │ │
│   │                                                                      │ │
│   │  # 솔트 생성                                                         │ │
│   │  salt = os.urandom(32)                                              │ │
│   │                                                                      │ │
│   │  # PBKDF2로 키 스트레칭                                            │ │
│   │  key = hashlib.pbkdf2_hmac(                                        │ │
│   │      'sha256',                                                      │ │
│   │      password.encode('utf-8'),                                      │ │
│   │      salt,                                                          │ │
│   │      100000  # 10만 번 반복                                        │ │
│   │  )                                                                  │ │
│   │                                                                      │ │
│   └───────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 해시 알고리즘**

| 알고리즘 | 솔트 지원 | 키 스트레칭 | 메모리 하드 | GPU 저항 | 속도 |
|:---|:---:|:---:|:---:|:---:|:---|
| **MD5** | ✗ | ✗ | ✗ | 낮음 | 매우 빠름 |
| **SHA-1/256** | ✓ | ✗ | ✗ | 낮음 | 빠름 |
| **PBKDF2** | ✓ | ✓ | ✗ | 낮음 | 느림 |
| **bcrypt** | ✓ | ✓ | 약함 | 중간 | 느림 |
| **scrypt** | ✓ | ✓ | ✓ | 높음 | 매우 느림 |
| **Argon2** | ✓ | ✓ | ✓ | 매우 높음 | 느림 |

**과목 융합 관점**

1. **암호학**: 해시 함수, 난수 생성, 키 파생
2. **운영체제**: /etc/shadow, 사용자 인증, PAM
3. **보안**: 패스워드 크래킹 방어, 저장소 보안

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: 비밀번호 저장 (Python)**
```python
import bcrypt

# 해싱
password = "password123".encode('utf-8')
salt = bcrypt.gensalt()
hashed = bcrypt.hashpw(password, salt)

# 검증
if bcrypt.checkpw(password, hashed):
    print("인증 성공")
```

**시나리오 2: /etc/shadow 구조**
```
alice:$6$salt$hash:18000:0:99999:7:::
```

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**

| 지표 | 평문 저장 | 솔트+해시 | 개선 효과 |
|:---|:---|:---|:---|
| **크래킹 시간** | 즉시 | 수년~수십년 | 10^9배 증가 |
| **레인보우 테이블** | 유효 | 무효 | 100% 방어 |
| **대량 크래킹** | 가능 | 불가능 | 1개씩만 |

**미래 전망**
1. **Argon2 표준화**: 메모리 하드 해싱
2. **하드웨어 보안**: TPM, Secure Enclave
3. **비밀번호 없는 인증**: FIDO2, 생체

**참고 표준**
- **NIST SP 800-63B**: 비밀번호 저장 가이드
- **RFC 2898**: PBKDF2
- **RFC 9106**: Argon2
