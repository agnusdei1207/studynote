# 431. 메모리 압축 (Memory Compression)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 페이지를 압축하여 공간 절약
> 2. **가치**: 스와핑 지연, 성능 유지
> 3. **융합**: 스와핑, zram과 연관

---

## Ⅰ. 개요

### 개념 정의

메모리 압축(Memory Compression)은 **메모리 페이지를 압축하여 물리 메모리 사용량을 줄이는 기법**이다. 스와핑 전에 압축을 먼저 시도한다.

### 💡 비유: 압축 주머니
메모리 압축은 **압축 주머니**와 같다. 옷을 압축하면 더 많이 들어간다. 꺼낼 때는 다시 펼친다.

### 메모리 압축 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 압축 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 동작】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  메모리 부족 시:                                                        │
│  │                                                             │ │
│  │  전통적 방식:                                                           │
│  │  페이지 → 스왑 디스크 → 매우 느림                                        │ │
│  │                                                             │ │
│  │  메모리 압축:                                                            │
│  │  페이지 → 압축 → 메모리 내 저장 → 빠름                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【zram/zswap 구조】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────┐        ┌─────────────────────────────┐      │ │
│  │  │ 일반 메모리   │        │      압축 메모리 (zram)      │      │ │
│  │  │  Pages      │   ───► │  Compressed Pages          │      │ │
│  │  │  (4KB)      │  압축  │  (평균 1-2KB)               │      │ │
│  │  └─────────────┘        └─────────────────────────────┘      │ │
│  │                                                             │ │
│  │  압축 비율: 일반적으로 2:1 ~ 3:1                                          │ │
│  │  4KB 페이지 → 1-2KB로 압축                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 과정】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 메모리 부족 감지                                                     │ │
│  │  2. 희생자 페이지 선택                                                   │ │
│  │  3. 페이지 압축 시도                                                     │ │
│  │  4. 압축 성공 → zram에 저장                                             │ │
│  │  5. 압축 실패 → 스왑 디스크로                                            │ │
│  │  6. 접근 시: 압축 해제하여 일반 메모리로                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 압축 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 스와핑 지연: 디스크 I/O보다 CPU 압축이 빠름                            │ │
│  │  • 유효 메모리 증가: 압축률만큼 메모리 확장                                │ │
│  │  • 반응성 향상: 페이지 폴트 처리 빠름                                     │ │
│  │  • 디스크 수명: SSD 쓰기 감소                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • CPU 오버헤드: 압축/해제에 CPU 사용                                    │ │
│  │  • 압축 불가 페이지: 이미 압축된 데이터는 효과 없음                        │ │
│  │  • 메모리 파편화: 압축된 페이지 크기 다양                                 │ │
│  │  • 복잡성: 구현과 관리 복잡                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【zram vs zswap】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  zram:                                                                  │ │
│  │  • 블록 장치로 동작                                                     │ │
│  │  • 스왑 공간으로 사용                                                   │ │
│  │  • 모든 스왑이 메모리 내 압축                                            │ │
│  │                                                             │ │
│  │  zswap:                                                                 │ │
│  │  • 스왑 캐시로 동작                                                     │ │
│  │  • 캐시 초과 시 디스크 스왑                                              │ │
│  │  • 하이브리드 방식                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【zram 설정 (Linux)】                                                 │
│  ──────────────────                                                  │
│  // zram 모듈 로드                                                      │
│  $ modprobe zram                                                     │
│                                                                     │
│  // zram 장치 생성                                                     │
│  $ zramctl --find --size 4G                                          │
│  /dev/zram0                                                          │
│                                                                     │
│  // 스왑으로 활성화                                                      │
│  $ mkswap /dev/zram0                                                 │
│  $ swapon /dev/zram0 -p 100  // 높은 우선순위                          │
│                                                                     │
│  // 상태 확인                                                          │
│  $ zramctl                                                          │
│  NAME       ALGORITHM DISKSIZE  DATA  COMPR  TOTAL  MOUNTPOINT       │
│  /dev/zram0 lzo        4G       512M  200M   210M  [SWAP]            │
│                                                                     │
│  【zswap 설정】                                                        │
│  ──────────────────                                                  │
│  // zswap 활성화                                                        │
│  $ echo 1 > /sys/module/zswap/parameters/enabled                    │
│                                                                     │
│  // 압축 알고리즘 선택                                                   │
│  $ echo zstd > /sys/module/zswap/parameters/compressor              │
│                                                                     │
│  // 최대 풀 크기 (전체 메모리의 %)                                        │
│  $ echo 20 > /sys/module/zswap/parameters/max_pool_percent          │
│                                                                     │
│  // 부팅 시 활성화                                                       │
│  // /etc/default/grub                                                 │
│  GRUB_CMDLINE_LINUX="zswap.enabled=1 zswap.compressor=zstd"         │
│                                                                     │
│  【Android의 zram】                                                    │
│  ──────────────────                                                  │
│  // Android는 기본적으로 zram 사용                                       │
│  // 보통 RAM의 50% 크기로 설정                                            │
│                                                                     │
│  $ cat /proc/swaps                                                   │
│  /dev/block/zram0                    2097148  123456  -2            │
│                                                                     │
│  // zram 통계                                                           │
│  $ cat /sys/block/zram0/mm_stat                                     │
│  // orig_data_size, compr_data_size, mem_used_total 등               │
│                                                                     │
│  【성능 모니터링】                                                        │
│  ──────────────────                                                  │
│  // 압축률 확인                                                          │
│  $ cat /sys/block/zram0/mm_stat | awk '{print $2/$1}'               │
│  0.4  // 40%로 압축됨                                                   │
│                                                                     │
│  // 스왑 활동                                                            │
│  $ cat /proc/vmstat | grep -E "pswpin|pswpout"                      │
│  pswpin 1234                                                         │
│  pswpout 5678                                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
