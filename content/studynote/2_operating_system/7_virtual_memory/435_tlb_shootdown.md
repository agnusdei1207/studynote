# 435. 고정 페이지 (Pinned Page)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스왑아웃되지 않는 페이지
> 2. **가치**: 실시간 성능, DMA 보장
> 3. **융합**: 스와핑, DMA와 연관

---

## Ⅰ. 개요

### 개념 정의

고정 페이지(Pinned Page)는 **물리 메모리에 고정되어 스왑아웃되지 않는 페이지**다. mlock()으로 설정한다. 페이지 폴트가 발생하지 않는다.

### 💡 비유: 액자에 넣은 사진
고정 페이지는 **액자에 넣은 사진**과 같다. 앨범에 넣지 않고 벽에 고정해 둔다. 절대 이동하지 않는다.

### 고정 페이지 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                고정 페이지 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【일반 페이지 vs 고정 페이지】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  일반 페이지:                                                            │
│  │  • 메모리 부족 시 스왑아웃 가능                                           │ │
│  │  • 페이지 교체 알고리즘 대상                                              │ │
│  │  • 페이지 폴트 발생 가능                                                  │ │
│  │                                                             │ │
│  │  고정 페이지 (Pinned):                                                   │ │
│  │  • 절대 스왑아웃 안 됨                                                   │ │
│  │  • 교체 알고리즘에서 제외                                                 │ │
│  │  • 페이지 폴트 없음 (항상 메모리에 있음)                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【고정이 필요한 상황】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 실시간 처리:                                                         │ │
│  │     • 페이지 폴트로 인한 지연 허용 안 됨                                   │ │
│  │     • 오디오/비디오 처리                                                 │ │
│  │                                                             │ │
│  │  2. DMA 전송:                                                            │ │
│  │     • 물리 주소가 변경되면 안 됨                                          │ │
│  │     • 장치 드라이버                                                      │ │
│  │                                                             │ │
│  │  3. 보안:                                                                 │ │
│  │     • 암호화 키가 스왑에 쓰이면 안 됨                                     │ │
│  │     • 민감 데이터 보호                                                   │ │
│  │                                                             │ │
│  │  4. 성능:                                                                 │ │
│  │     • 핫 코드/데이터                                                     │ │
│  │     • 자주 접근하는 데이터                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【mlock 예시】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  mlock(addr, size):   메모리 고정                                         │ │
│  │  munlock(addr, size): 고정 해제                                           │ │
│  │  mlockall(flags):     전체 주소 공간 고정                                  │ │
│  │  munlockall():        전체 고정 해제                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                고정 페이지 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【mlock 플래그】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  mlockall() 플래그:                                                      │ │
│  │                                                             │ │
│  │  MCL_CURRENT:                                                            │ │
│  │  • 현재 매핑된 모든 페이지 고정                                           │ │
│  │                                                             │ │
│  │  MCL_FUTURE:                                                              │ │
│  │  • 앞으로 매핑될 모든 페이지도 고정                                        │ │
│  │                                                             │ │
│  │  MCL_ONFAULT (Linux 4.4+):                                               │ │
│  │  • 실제로 접근할 때만 고정                                               │ │
│  │  • 지연 고정                                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주의사항】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 권한: CAP_IPC_LOCK 또는 RLIMIT_MEMLOCK 제한                          │ │
│  │  • 과용 위험: 너무 많은 고정은 시스템 성능 저하                            │ │
│  │  • 해제 필요: 사용 후 반드시 munlock                                      │ │
│  │  • fork: 자식은 고정 상태 상속 안 됨                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【고정 페이지와 스와핑】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  메모리 부족 시:                                                        │
│  │  • 일반 페이지 → 스왑아웃                                               │ │
│  │  • 고정 페이지 → 스왑아웃 안 됨                                          │ │
│  │  • 고정 페이지가 많으면 OOM 가능                                         │ │
│  │                                                             │ │
│  │  교체 알고리즘:                                                          │
│  │  • 고정 페이지는 후보에서 제외                                           │ │
│  │  • PAGEVEC_PINNED 플래그로 표시                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【mlock 사용 예시】                                                    │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│  #include <stdlib.h>                                                 │
│  #include <stdio.h>                                                  │
│                                                                     │
│  int main() {                                                        │
│      size_t size = 1024 * 1024;  // 1MB                              │
│      void *ptr = malloc(size);                                       │
│                                                                     │
│      // 메모리 고정                                                     │
│      if (mlock(ptr, size) != 0) {                                    │
│          perror("mlock failed");                                     │
│          return 1;                                                   │
│      }                                                               │
│                                                                     │
│      // 이제 이 메모리는 스왑아웃되지 않음                                 │
│      // 실시간 처리에 안전                                               │
│      memset(ptr, 0, size);                                           │
│                                                                     │
│      // 고정 해제                                                      │
│      munlock(ptr, size);                                             │
│      free(ptr);                                                      │
│      return 0;                                                       │
│  }                                                                   │
│                                                                     │
│  【mlockall 사용】                                                      │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│                                                                     │
│  // 현재와 미래의 모든 메모리 고정                                        │
│  if (mlockall(MCL_CURRENT | MCL_FUTURE) != 0) {                      │
│      perror("mlockall failed");                                      │
│  }                                                                   │
│                                                                     │
│  // 실시간 애플리케이션에서 사용                                          │
│  // ...                                                              │
│                                                                     │
│  // 종료 시 해제                                                        │
│  munlockall();                                                       │
│                                                                     │
│  【제한 확인 및 설정】                                                  │
│  ──────────────────                                                  │
│  // mlock 제한 확인                                                     │
│  $ ulimit -l                                                         │
│  65536  // 64KB (키로바이트)                                            │
│                                                                     │
│  // 제한 변경 (root)                                                    │
│  $ ulimit -l unlimited                                               │
│  // 또는                                                               │
│  $ prlimit --memlock=unlimited:unlimited ./program                  │
│                                                                     │
│  // /etc/security/limits.conf                                        │
│  *    hard    memlock    unlimited                                   │
│  *    soft    memlock    unlimited                                   │
│                                                                     │
│  【고정 페이지 확인】                                                    │
│  ──────────────────                                                  │
│  // 프로세스의 메모리 맵에서 VmLck 확인                                   │
│  $ cat /proc/self/status | grep VmLck                               │
│  VmLck:     1024 kB  // 고정된 메모리 양                                │
│                                                                     │
│  // flags 필드의 'l'이 고정됨을 의미                                     │
│  $ cat /proc/self/smaps | grep -A1 "^VmFlags"                       │
│  VmFlags: rd wr mr mw me ac sd                                       │
│  // 'lo' 플래그가 있으면 locked                                         │
│                                                                     │
│  【실시간 오디오 예시】                                                  │
│  ──────────────────                                                  │
│  // JACK 오디오 서버 같은 실시간 앱                                       │
│  void *audio_buffer = malloc(BUFFER_SIZE);                           │
│  mlock(audio_buffer, BUFFER_SIZE);                                   │
│  // 오디오 처리 중 페이지 폴트로 끊김 방지                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 스왑아웃되지 않는 페이지
• API: mlock(), mlockall()
• 용도: 실시간, DMA, 보안
• 특징: 페이지 폴트 없음
• 제한: RLIMIT_MEMLOCK
• 주의: 과용 시 시스템 성능 저하
• 확인: /proc/[pid]/status VmLck
```
