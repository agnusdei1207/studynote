## 💡 Insight
- 가상 메모리와 물리 메모리의 매핑 방식은 CPU 캐시(L1, L2, L3)의 효율성에 직접적인 영향을 미친다.
- 특히 데이터가 여러 캐시 세트에 골고루 분산되지 못하는 '캐시 색상화(Cache Coloring)' 현상을 해결하기 위해 페이지 배치 기법이 활용된다.
- 이를 통해 캐시 충돌(Cache Conflict)을 최소화하고 메모리 대역폭(Bandwidth)을 최대로 활용할 수 있다.

---

## Ⅰ. 캐시 친화적 메모리 배치의 배경
### 1. 하드웨어 캐시의 작동 (Set-associative)
- CPU 캐시는 메모리 주소의 특정 비트를 기반으로 저장될 위치(Set)를 결정함.
- 여러 가상 주소가 동일한 캐시 세트로 몰리면 기존 데이터가 계속 쫓겨나는 캐시 트래싱(Thrashing)이 발생함.

### 2. 가상 메모리와의 관계
- OS가 가상 페이지를 물리 프레임에 매핑할 때, 캐시 인덱스를 결정하는 비트 부분을 고려하지 않으면 성능이 급격히 저하될 수 있음.

📢 섹션 요약 비유: 서랍장(캐시)의 특정 칸(세트)만 계속 쓰지 말고, 골고루 나누어 담을 수 있도록 물건(데이터)에 색지(Coloring)를 붙여 관리하는 것입니다.

---

## Ⅱ. 캐시 색상화 (Cache Coloring / Page Coloring)
### 1. 개념
- 물리 주소의 특정 비트(캐시 인덱스 비트)가 겹치지 않도록 페이지를 할당하는 기법.
- 가상 주소의 페이지 번호와 물리 주소의 프레임 번호가 특정 '색상(Color)'을 공유하도록 강제함.

### 2. 작동 원리
- 캐시 크기가 페이지 크기보다 클 때 발생하는 'Alias' 문제를 해결하기 위해, 가상 주소의 일부 비트를 물리 주소 할당 시 그대로 유지함.

### 3. ASCII 다이어그램: Cache Color Distribution
```text
[ Physical Memory Frames ]
Frame 0 (Red) | Frame 1 (Blue) | Frame 2 (Green) | Frame 3 (Yellow)
Frame 4 (Red) | Frame 5 (Blue) | Frame 6 (Green) | Frame 7 (Yellow)
      ^               ^                ^                 ^
      +---------------+----------------+-----------------+
              (Mapped to different L2/L3 Cache Sets)
```

📢 섹션 요약 비유: 도서관 책을 주제별로 색깔 스티커(Color)를 붙여서, 같은 색깔끼리 한 선반에 너무 많이 몰리지 않게 분산 배치하는 시스템입니다.

---

## Ⅲ. 페이지 클러스터링 (Page Clustering)
### 1. 정의
- 관련성이 높은 가상 페이지들을 물리적으로도 인접하게 배치하거나, 한 번에 읽어 들이는 기법.
### 2. 성능 이점
- **공간 지역성(Spatial Locality)** 활용: CPU가 다음 데이터를 예측하여 미리 읽어오는 하드웨어 프리페칭(Prefetching)의 효율성을 극대화함.

📢 섹션 요약 비유: 마트 진열대에 라면(데이터 A) 옆에 만두(데이터 B)를 두어, 고객(CPU)이 한 번에 같이 집어갈 수 있게 편의성을 높이는 것과 같습니다.

---

## Ⅳ. 캐시 친화적 관리의 주요 도전 과제
### 1. 동적 파편화 (Dynamic Fragmentation)
- 메모리가 조각나면 원하는 색상의 페이지를 찾기 어려워짐.
### 2. 대형 페이지와의 충돌
- Hugepages(2MB)는 이미 많은 캐시 세트를 점유하므로 색상화의 의미가 퇴색되거나 관리가 복잡해질 수 있음.

📢 섹션 요약 비유: 색깔별로 옷을 정리하고 싶은데, 옷장이 너무 꽉 차서 빈자리에 아무 옷이나 쑤셔 넣어야 하는(파편화) 상황입니다.

---

## Ⅴ. 최적화 사례: 데이터 정렬 (Alignment)
### 1. 캐시 라인 정렬 (Cache Line Alignment)
- 구조체의 크기를 캐시 라인(주로 64바이트)의 배수로 맞추어 한 데이터가 두 개의 캐시 라인에 걸쳐 성능을 깎아 먹는 일을 방지함.
### 2. 폴스 셰어링 (False Sharing) 방지
- 서로 다른 쓰레드가 쓰는 데이터를 같은 캐시 라인에 두지 않도록 간격(Padding)을 둠.

📢 섹션 요약 비유: 서랍 칸막이(캐시 라인)에 물건이 딱 맞게 들어가게 크기를 조절하여, 옆 칸 물건을 꺼낼 때 방해하지 않게 하는 것입니다.
