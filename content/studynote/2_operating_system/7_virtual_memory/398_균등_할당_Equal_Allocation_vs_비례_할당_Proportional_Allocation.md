## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 균등 할당은 프로세스의 크기와 무관하게 모든 프로세스에 동일한 수의 프레임을 배분하는 방식이며, 비례 할당은 프로세스의 가상 메모리 크기에 맞춰 프레임을 차등 배분하는 방식이다.
> 2. **가치**: 균등 할당은 구현이 단순하고 소형 프로세스에 유리한 반면, 비례 할당은 대형 프로세스의 페이지 부재를 억제하여 시스템 자원 활용의 합리성을 높인다.
> 3. **융합**: 운영체제는 이 두 정적 할당 방식의 한계를 극복하기 위해, 프로세스의 우선순위나 실행 시점의 동적 메모리 요구량(Working Set)을 결합한 하이브리드 전략을 주로 사용한다.

---

### Ⅰ. 개요 (Context & Background)

- **概念**: 프레임 할당의 두 가지 기본 모델이다. **균등 할당 (Equal Allocation)**은 'n개의 프레임을 m개의 프로세스에 n/m개씩' 나누는 공산주의적 배분이며, **비례 할당 (Proportional Allocation)**은 '덩치가 큰 프로세스에게 더 많은 땅(프레임)을 주는' 자본주의적 배분이다.

- **💡 비유**: 균등 할당은 모든 아이에게 똑같이 **"사탕 3개씩"** 주는 것이고, 비례 할당은 덩치가 커서 배가 많이 고픈 아이에겐 사탕을 더 많이 주는 것과 같다.

- **등장 배경**:
  1. **단순성의 요구**: 초기 시스템에서는 계산 부하를 줄이기 위해 균등 할당을 사용했다.
  2. **성능 불균형**: 대형 프로세스가 균등 할당을 받으면 페이지 부재가 폭주하는 반면, 소형 프로세스는 프레임이 남는 자원 왜곡 현상이 발생했다.
  3. **효율적 배분 공식의 필요**: 가상 메모리 크기(`Si`)라는 객관적 지표를 활용하여 부재율을 평준화하려는 시도가 비례 할당으로 이어졌다.

- **📢 섹션 요약 비유**: 무조건 1/N로 나누는 '더치페이'와 먹은 만큼 내는 '계산법'의 차이와 같습니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

#### 할당 방식 비교 분석 (표)

| 비교 항목 | 균등 할당 (Equal) | 비례 할당 (Proportional) |
|:---|:---|:---|:---|
| **할당 공식** | `ai = m / n` | `ai = (si / S) * m` |
| **si / S 의미** | N/A | (프로세스 i 크기 / 총 프로세스 크기 합) |
| **주요 특징** | 모든 프로세스 동일 대우 | 가상 메모리 점유율에 비례 |
| **적합 환경** | 프로세스들의 크기가 비슷할 때 | 프로세스 크기 편차가 클 때 |
| **문제점** | 대형 프로그램의 실행 저하 | 실행 중 크기가 변할 때 대응 불가 |

#### 할당 모델 아키텍처 다이어그램

```text
  [ Physical RAM: 100 Frames ]
  
  (1) Equal Allocation                (2) Proportional Allocation
  ┌──────────────────────────┐        ┌──────────────────────────┐
  │ Process A (10MB): [50]   │        │ Process A (10MB): [20]   │
  ├──────────────────────────┤        ├──────────────────────────┤
  │ Process B (40MB): [50]   │        │ Process B (40MB): [80]   │
  └──────────────────────────┘        └──────────────────────────┘
       (Fair but Inefficient)              (Rational & Scalable)
```

**[다이어그램 해설]** 가용한 프레임이 100개일 때, (1)균등 할당은 A와 B에게 무조건 50개씩 준다. 이 경우 10MB인 A는 프레임이 남지만, 40MB인 B는 턱없이 부족하여 고생한다. (2)비례 할당은 전체 50MB 중 B가 80%를 차지하므로 프레임도 80개를 준다. 결과적으로 두 프로세스 모두 자신의 크기에 걸맞은 쾌적한 환경을 갖게 된다.

- **📢 섹션 요약 비유**: 발 사이즈가 300인 사람과 200인 사람에게 똑같이 250 사이즈 신발을 주는 것(균등)과, 각자 발에 맞는 사이즈를 주는 것(비례)의 차이입니다.

### Ⅲ. 융합 비교 및 다각도 분석

#### 시나리오별 유리한 방식

| 시나리오 | 선호 방식 | 이유 |
|:---|:---|:---|
| **임베디드 제어 시스템** | 균등 할당 | 각 작업의 크기가 고정되어 있고 예측 가능함 |
| **일반 목적 OS (Windows/Linux)** | 비례 할당 | 웹 브라우저(대형)와 메모장(소형)의 격차가 큼 |
| **보안 강화 시스템** | 균등 할당 | 자원 점유율을 통한 정보 유출(Side Channel) 차단 |

- **📢 섹션 요약 비유**: 똑같은 규격의 부품을 만드는 공장(임베디드)에서는 균등이 편하지만, 온갖 메뉴를 파는 푸드코트(범용 OS)에서는 비례가 합리적입니다.

### Ⅳ. 실무 적용 및 기술사적 판단

#### 실무 시나리오: 비례 할당의 맹점과 보완
비례 할당은 '가상 메모리 크기'를 기준으로 하지만, 실제로는 프로그램이 크다고 해서 항상 많은 메모리를 쓰는 것은 아니다. 예를 들어 1GB짜리 데이터베이스 프로그램이라도 지금 당장은 10MB의 인덱스만 조회 중일 수 있다. 기술사적 관점에서 비례 할당은 '잠재적 요구량'에 대응하기엔 좋지만, '실제 사용량'을 반영하지 못한다. 이를 보완하기 위해 현대 OS는 비례 할당을 베이스로 하되, 실제 페이지 부재 빈도(PFF)를 측정하여 프레임을 줬다 뺏는 동적 조절을 병행한다.

- **📢 섹션 요약 비유**: 덩치 큰 선수가 항상 많이 먹는 건 아니니, 일단 밥을 많이 퍼주되(비례) 남기면 뺏고 더 달라고 하면 더 주는(동적) 유연함이 필요합니다.

### Ⅴ. 기대효과 및 결론

#### 할당 전략의 정성적 기대효과
1. **자원 활용 최적화**: 비례 할당을 통해 물리 메모리의 '빈 공간'과 '부족 공간'의 불균형을 해소한다.
2. **시스템 처리량 증대**: 모든 프로세스가 각자의 임계치(Threshold) 이상의 프레임을 확보하게 하여 전체 성능을 높인다.
3. **예측 가능성**: 정적 할당 모델은 시스템 설계 단계에서 자원 배분 계획을 세우는 기초가 된다.

- **📢 섹션 요약 비유**: '무조건 반분'의 정(情)보다는 '필요한 만큼의 배려'라는 이성이 시스템을 더 건강하게 만듭니다.

### 👶 어린이를 위한 3줄 비유 설명
1. 균등 할당은 키 큰 형이랑 꼬마 동생에게 똑같은 양의 밥을 주는 거예요.
2. 비례 할당은 배가 많이 고픈 큰 형에게 밥을 더 많이 주는 거고요.
3. 형이 배고파서 울지 않게 하려면 형에게 밥을 더 많이 주는 비례 할당이 더 똑똑한 방법이랍니다!
