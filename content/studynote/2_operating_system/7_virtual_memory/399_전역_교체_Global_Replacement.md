## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전역 교체(Global Replacement)는 페이지 부재 발생 시, 해당 프로세스에 할당된 프레임뿐만 아니라 메모리 상의 '모든 프레임'을 대상으로 교체 대상(Victim)을 찾는 방식이다.
> 2. **가치**: 시스템 전체의 관점에서 가장 오랫동안 사용되지 않은 페이지를 내보낼 수 있어 물리 메모리 활용도를 극대화하고, 프로세스 간의 유연한 자원 이동을 가능하게 한다.
> 3. **융합**: 프레임 할당량(Allocation)이 고정되지 않고 실행 상황에 따라 동적으로 변하게 만들며, 현대 운영체제에서 가장 널리 채택되는 범용적인 교체 전략이다.

---

### Ⅰ. 개요 (Context & Background)

- **概念**: **전역 교체 (Global Replacement)**는 프로세스가 다른 프로세스의 프레임을 '뺏어올 수 있는' 교체 방식이다. 페이지 부재가 발생했을 때, OS는 현재 실행 중인 모든 프로세스의 페이지들을 검사하여 가장 적절한 희생자를 선정한다.

- **💡 비유**: 전역 교체는 **"자유석 도서관"**과 같다. 내가 자리가 필요할 때, 내 구역에서만 찾는 게 아니라 도서관 전체를 돌아다니며 가장 오래 비어 있는 자리를 차지하는 방식이다.

- **등장 배경**:
  1. **자원 활용의 불균형**: 지역 교체 방식에서는 특정 프로세스는 메모리가 남고 다른 프로세스는 부족한 상황이 빈번했다.
  2. **동적 요구 대응**: 프로그램의 실행 단계에 따라 필요한 메모리 양이 계속 변하므로, 이를 유연하게 조절할 메커니즘이 필요했다.
  3. **전체 최적화**: 개별 프로세스의 성능보다 시스템 전체의 처리량(Throughput)을 높이는 것이 더 중요하다는 판단이 작용했다.

- **📢 섹션 요약 비유**: 옆집에 남는 방이 있다면 우리 집 손님을 위해 잠시 빌려 쓸 수 있는 오픈 마인드의 자원 관리법입니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

#### 전역 교체의 주요 특징 (표)

| 항목 | 상세 내용 | 비유 |
|:---|:---|:---|
| **교체 범위** | 물리 메모리 내의 모든 프레임 | 도서관 전체 좌석 |
| **할당량 변화** | 실행 중 계속 변함 (Dynamic) | 빌려온 의자 수에 따라 변함 |
| **장점** | 메모리 활용 효율이 매우 높음 | 노는 자리가 없음 |
| **단점** | 다른 프로세스에 의해 내 성능이 영향받음 | 옆 사람이 내 자리를 뺏어감 |

#### 전역 교체 프로세스 구조

```text
  [ Physical RAM Frames ]
  ┌─────────┬─────────┬─────────┬─────────┐
  │ Proc A  │ Proc B  │ Proc A  │ Proc C  │
  │ Page 1  │ Page 5  │ Page 2  │ Page 8  │
  └─────────┴────┬────┴─────────┴─────────┘
                 │
      (Victim Selected by Global LRU)
                 │
  [ Process C Needs a New Page! ]
  ┌─────────┬─────────┬─────────┬─────────┐
  │ Proc A  │ Proc C  │ Proc A  │ Proc C  │
  │ Page 1  │ Page 12 │ Page 2  │ Page 8  │
  └─────────┴─────────┴─────────┴─────────┘
      (Proc C "Stole" a Frame from Proc B)
```

**[다이어그램 해설]** 프로세스 C에서 페이지 부재가 발생했다. 전역 교체 방식에서는 C의 프레임 중에서 고르는 게 아니라, 전체 메모리에서 가장 안 쓰인 페이지를 찾는다. 만약 프로세스 B의 Page 5가 가장 오래된 것이라면, OS는 B의 프레임을 뺏어서 C에게 준다. 결과적으로 B의 할당량은 줄어들고 C의 할당량은 늘어난다.

- **📢 섹션 요약 비유**: 우리 팀 선수만 교체하는 게 아니라, 컨디션 안 좋은 다른 팀 선수까지 포함해서 가장 못 뛰는 선수를 경기장 밖으로 내보내는 파격적인 교체 전술입니다.

### Ⅲ. 융합 비교 및 다각도 분석

#### 비교: 전역 교체 (Global) vs 지역 교체 (Local)

| 비교 항목 | 전역 교체 (Global Replacement) | 지역 교체 (Local Replacement) |
|:---|:---|:---|
| **희생자 선정 범위** | 모든 프로세스의 프레임 | 부재가 발생한 프로세스의 프레임만 |
| **성능 독립성** | 낮음 (남의 탓에 느려질 수 있음) | 높음 (자기 잘못에만 책임짐) |
| **메모리 효율** | 매우 높음 | 낮음 (자원 파편화 발생 가능) |
| **구현 난이도** | 비교적 단순함 | 복잡함 (프로세스별 할당량 유지 필요) |

- **📢 섹션 요약 비유**: 마을 전체의 우물을 같이 쓰는 것(전역)과, 집집마다 각자의 우물만 파서 쓰는 것(지역)의 차이입니다.

### Ⅳ. 실무 적용 및 기술사적 판단

#### 실무 시나리오: 현대 OS의 선택과 쓰래싱 위험
리눅스(Linux)나 윈도우(Windows) 같은 현대 OS는 기본적으로 전역 교체를 사용한다. 이는 특정 프로그램(예: 압축 프로그램)이 잠시 많은 메모리를 쓰고 싶어 할 때, 놀고 있는 다른 프로그램의 메모리를 즉시 가져올 수 있어 사용자 체감 성능이 좋기 때문이다. 하지만 기술사적으로 주의할 점은, 전역 교체 상황에서 쓰래싱(Thrashing)이 발생하면 그 여파가 시스템 전체로 순식간에 번진다는 것이다. 한 프로세스가 남의 프레임을 뺏기 시작하면, 뺏긴 쪽에서도 다시 남의 것을 뺏으려 하며 연쇄적인 페이지 부재 폭발이 일어날 수 있다.

- **📢 섹션 요약 비유**: 한 명의 빌런(메모리 과소비 프로세스)이 옆 사람 의자를 뺏기 시작하면, 결국 파티장 전체가 의자 뺏기 게임 아수라장이 되는 위험도 공존합니다.

### Ⅴ. 기대효과 및 결론

#### 전역 교체의 정성적 기대효과
1. **유연한 자원 배분**: 프로그램의 생애 주기에 따른 메모리 요구량 변화를 실시간으로 수용한다.
2. **평균 처리량 향상**: 시스템 전체의 관점에서 페이지 부재 횟수를 최소화하는 방향으로 동작한다.
3. **사용자 편의성**: 사용자가 일일이 메모리 할당량을 지정할 필요 없이 OS가 알아서 최적의 상태를 찾는다.

- **📢 섹션 요약 비유**: '내 것 네 것'을 가리지 않는 나눔을 통해, 시스템이라는 공동체의 이익을 극대화하는 합리적인 이타주의 모델입니다.

### 📌 관련 개념 맵
- **지역 교체 (Local Replacement)**: 대조되는 폐쇄적 교체 방식.
- **LRU (Least Recently Used)**: 전역 교체에서 희생자를 고르는 흔한 기준.
- **쓰래싱 (Thrashing)**: 전역 교체의 부작용이 극대화된 상태.

### 👶 어린이를 위한 3줄 비유 설명
1. 전역 교체는 친구들과 장난감 상자를 **"다 같이 한꺼번에"** 쓰는 거예요.
2. 내가 새 장난감을 넣고 싶은데 자리가 없으면, 내 칸이 아니라 친구 칸에서 제일 안 노는 장난감을 빼고 내 걸 넣을 수 있어요.
3. 상자 전체를 넓게 쓸 수 있어서 좋지만, 친구 장난감을 내가 빼버리면 친구가 슬퍼할 수도 있는 방법이랍니다!
