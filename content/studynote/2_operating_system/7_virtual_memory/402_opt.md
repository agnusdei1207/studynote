# 402. 작업 집합 (Working Set)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스가 집중 사용하는 페이지 집합
> 2. **가치**: 스래싱 방지와 메모리 할당 기준
> 3. **융합**: 지역성, 페이지 교체와 연관

---

## Ⅰ. 개요

### 개념 정의

작업 집합(Working Set)은 **일정 시간 윈도우(Δ) 내에 프로세스가 참조한 페이지의 집합**이다. 프로세스가 활발히 사용하는 페이지들을 나타낸다.

### 💡 비유: 책상 위 책
작업 집합은 **책상 위에 펼쳐둔 책**과 같다. 공부할 때 여러 책을 동시에 보는데, 이것이 작업 집합이다.

### 작업 집합 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                작업 집합 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정의】                                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  W(t, Δ) = 시간 t에서 Δ 시간 동안 참조된 페이지 집합                    │ │
│  │                                                             │ │
│  │  t: 현재 시간                                                          │ │
│  │  Δ: 작업 집합 윈도우 (Working Set Window)                               │ │
│  │  |W(t, Δ)|: 작업 집합 크기 (Working Set Size, WSS)                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시각화】                                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시간 ─────────────────────────────────────────────────────►       │ │
│  │                                                             │ │
│  │           ┌─────────────────Δ───────────────────┐                    │ │
│  │           │    작업 집합 윈도우                  │                    │ │
│  │           │  ┌─────────────────────────────┐   │                    │ │
│  │  ...  P1  P2  P3  P1  P4  P2  P1  P5  P3  P1 ...                   │ │
│  │           │  └─────────────────────────────┘   │                    │ │
│  │           │   현재 시간 t에서 작업 집합:            │                    │ │
│  │           │   W(t, Δ) = {P1, P2, P3, P4, P5}       │                    │ │
│  │           │   |W| = 5                           │                    │ │
│  │           └─────────────────────────────────────┘                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【작업 집합 크기 (WSS)】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  총 메모리 요구 = Σ |WSS_i| (모든 프로세스의 작업 집합 합)              │ │
│  │                                                             │ │
│  │  if 총 메모리 요구 > 물리 메모리:                                         │ │
│  │     → 스래싱 발생 가능                                                   │ │
│  │                                                             │ │
│  │  if 총 메모리 요구 ≤ 물리 메모리:                                         │ │
│  │     → 스래싱 없음                                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                작업 집합 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Δ 설정】                                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Δ가 너무 작음:                                                        │ │
│  │  • 작업 집합이 너무 작게 추정                                           │ │
│  │  • 불필요한 페이지 교체                                                 │ │
│  │  • 스래싱 위험                                                         │ │
│  │                                                             │ │
│  │  Δ가 너무 큼:                                                           │ │
│  │  • 작업 집합이 너무 크게 추정                                           │ │
│  │  • 메모리 낭비                                                          │ │
│  │  • 다른 프로세스에 영향                                                  │ │
│  │                                                             │ │
│  │  적절한 Δ:                                                               │ │
│  │  • 지역성을 포착할 정도                                                  │ │
│  │  • 일반적으로 수십 ~ 수백 ms                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지역성과 관계】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  작업 집합은 지역성(Locality)을 활용:                                   │ │
│  │                                                             │ │
│  │  시간 지역성:                                                           │ │
│  │  • 최근 접근한 데이터 재접근                                             │ │
│  │  • 작업 집합에 포함된 페이지 재참조                                      │ │
│  │                                                             │ │
│  │  공간 지역성:                                                           │ │
│  │  • 인접한 데이터 접근                                                   │ │
│  │  • 연속적인 페이지 접근                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【작업 집합 모델】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스래싱 방지:                                                           │ │
│  │  if Σ |WSS_i| > 총 프레임:                                            │ │
│  │     • 일부 프로세스를 스왑 아웃                                          │ │
│  │     • 나머지 프로세스의 작업 집합 유지                                   │ │
│  │                                                             │ │
│  │  프레임 할당:                                                           │ │
│  │  • 각 프로세스에 |WSS| 만큼 프레임 할당                                  │ │
│  │  • 작업 집합 변화에 따라 동적 조정                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【작업 집합 추적 (개념적)】                                              │
│  ──────────────────                                                  │
│  class WorkingSetTracker {                                         │
│      long windowSize = 100000000;  // 100ms (ns)                       │
│      Map<Integer, LinkedList<Long>> pageAccessTimes;               │
│      Map<Integer, Set<Integer>> workingSets;  // 프로세스별 작업 집합   │
│                                                                     │
│      void onAccess(int processId, int pageNumber) {                 │
│          long now = System.nanoTime();                               │
│          long cutoff = now - windowSize;                             │
│                                                                     │
│          // 접근 기록                                                   │ │
│          LinkedList<Long> times = pageAccessTimes                    │
│              .computeIfAbsent(processId * 1000 + pageNumber, k -> new LinkedList<>());│
│          times.add(now);                                             │
│                                                                     │
│          // 오래된 기록 제거                                               │
│          while (!times.isEmpty() && times.peekFirst() < cutoff) {    │
│              times.pollFirst();                                       │
│              if (times.isEmpty()) {                                   │
│                  // 작업 집합에서 제거                                    │ │
│                  workingSets.get(processId).remove(pageNumber);      │
│              }                                                      │
│          }                                                          │
│                                                                     │
│          // 작업 집합에 추가                                             │ │
│          workingSets.computeIfAbsent(processId, k -> new HashSet<>())│
│                       .add(pageNumber);                              │
│      }                                                              │
│                                                                     │
│      int getWorkingSetSize(int processId) {                         │
│          return workingSets.getOrDefault(processId, Set.of()).size();│
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Linux 작업 집합】                                                        │
│  ──────────────────                                                  │
│  // 프로세스의 작업 집합 확인                                               │
│  $ cat /proc/<pid>/stat | awk '{print $24}'                          │
│  // wchan: 현재 대기 중인 커널 함수                                          │
│                                                                     │
│  // 프로세스의 물리 메모리 사용                                             │
│  $ cat /proc/<pid>/statm | awk '{print $2}'                          │
│  // 두 번째 필드: 실제 물리 페이지 수                                       │
│                                                                     │
│  // smaps로 상세 분석                                                    │
│  $ grep -E "^Size|^Rss|^Pss" /proc/<pid>/smaps                       │
│  Size:               1024 kB                                         │
│  Rss:                 512 kB   ← 실제 사용 중                             │
│  Pss:                 400 kB   ← 프로포셔널 공유 후                       │
│                                                                     │
│  【메모리 압력 (Memory Pressure)】                                         │
│  ──────────────────                                                  │
│  // Linux PSI (Pressure Stall Information)                             │
│  $ cat /proc/pressure/memory                                         │
│  some avg10=0.00 avg60=0.00 avg300=0.00 total=0                     │
│  // 메모리 압력이 높으면 작업 집합 유지 어려움                                │
│                                                                     │
│  // OOM 점수와 작업 집합                                                   │
│  $ cat /proc/<pid>/oom_score                                        │
│  // 높은 점수 = 큰 작업 집합 = 킬 대상                                      │
│                                                                     │
│  【프로그래밍에서 작업 집합】                                                 │
│  ──────────────────                                                  │
│  // 캐시 크기 = 작업 집합 크기로 설정                                       │
│  Cache<String, Data> cache = Caffeine.newBuilder()                  │
│      .maximumSize(1000)  // 작업 집합 추정 크기                            │
│      .expireAfterAccess(10, TimeUnit.MINUTES)  // 윈도우               │
│      .build();                                                       │
│                                                                     │
│  // 데이터 구조 선택                                                      │
│  // 작업 집합이 작고 빈번한 접근: 메모리에 유지                                 │
│  // 작업 집합이 큼: 필요할 때만 로드                                         │
│                                                                     │
│  【성능 최적화】                                                            │
│  ──────────────────                                                  │
│  // 작업 집합을 고려한 메모리 할당                                           │
│  void optimizeMemoryUsage() {                                       │
│      long workingSetSize = estimateWorkingSet();                     │
│      long availableMemory = getAvailableMemory();                    │
│                                                                     │
│      if (workingSetSize > availableMemory * 0.8) {                   │
│          // 작업 집합이 메모리를 초과                                        │ │
│          // 데이터 스트리밍으로 전환                                        │ │
│          switchToStreaming();                                        │
│      } else {                                                       │
│          // 작업 집합을 메모리에 유지                                        │ │
│          keepInMemory();                                            │
│      }                                                              │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 일정 시간 내 참조된 페이지 집합
• 구성: W(t, Δ) - 시간 t, 윈도우 Δ
• 크기: WSS (Working Set Size)
• 활용: 스래싱 방지, 프레임 할당
• 관계: 지역성 활용
• 설정: Δ가 핵심 (너무 작/큼 문제)
```
