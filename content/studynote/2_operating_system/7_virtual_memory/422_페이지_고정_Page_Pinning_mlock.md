## 💡 Insight
- 페이지 고정은 특정 메모리 페이지가 스왑(Swap) 장치로 쫓겨나지 않도록 물리 메모리(RAM)에 강제로 상주시추는 기법이다.
- 실시간 시스템(Real-time System)의 응답성 보장과 보안에 민감한 데이터(비밀번호, 키)의 디스크 유출 방지를 위해 필수적이다.
- 시스템 전체의 가용 메모리를 제한할 수 있으므로, `RLIMIT_MEMLOCK` 설정을 통해 할당량을 엄격히 관리해야 한다.

---

## Ⅰ. 페이지 고정 (Page Pinning / Locking) 개요
### 1. 정의
- 특정 가상 메모리 페이지를 물리 메모리 프레임에 고정하여, 페이지 교체 알고리즘(Page Replacement Algorithm)의 대상에서 제외시키는 작업.
### 2. 주요 시스템 콜
- `mlock()`: 특정 주소 범위의 페이지를 고정함.
- `mlockall()`: 프로세스의 현재 및 미래 주소 공간 전체를 고정함.
- `munlock()`: 고정을 해제함.

📢 섹션 요약 비유: 도서관에서 "반드시 봐야 할 책"을 대출 금지 구역(RAM)에 묶어두어, 사서(OS)가 마음대로 창고(Disk)로 치우지 못하게 하는 것과 같습니다.

---

## Ⅱ. 페이지 고정의 작동 매커니즘
### 1. 페이지 테이블 플래그 설정
- OS는 해당 페이지의 비트맵이나 페이지 테이블 엔트리(PTE)에 `Locked` 플래그를 표시함.
- LRU 리스트(Least Recently Used List)에서 제거되어 스왑 아웃 후보군에 들지 않게 됨.

### 2. ASCII 다이어그램: Pinning Process
```text
[ Virtual Memory ]      [ MMU / OS Logic ]      [ Physical RAM ]
+---------------+       +----------------+      +---------------+
| Secret Key    | ----> | Check mlock()  |      | Frame #102    |
| (Page A)       |       | Set 'L' Bit    | <--- | [LOCKED]      |
+---------------+       +----------------+      +---------------+
        |                       |                       |
        | (Swap Req?)           X (Skip)                |
        V                       |                       V
[ Disk Swap Area ] <---------------------------- [ Other Pages ]
```

📢 섹션 요약 비유: 호텔 프런트(OS)에서 특정 투숙객(페이지)에게 "VIP(mlock)" 딱지를 붙여, 방이 부족해도 체크아웃(Swap out)시키지 못하게 보호하는 예약 시스템입니다.

---

## Ⅲ. 주요 활용 사례
### 1. 실시간 시스템 (RTOS / Real-time Apps)
- 페이지 폴트(Page Fault) 발생 시 디스크 I/O로 인한 비결정적 지연(Latency)이 발생하므로, 핵심 코드를 메모리에 고정하여 지연 시간을 최소화함.
### 2. 보안 (Security)
- 비밀번호, 개인키 등의 민감 정보가 디스크의 스왑 영역에 기록되면, 시스템 종료 후에도 유출될 위험이 있음. 이를 방지하기 위해 메모리에만 머물게 함.
### 3. 고성능 네트워킹 및 DB
- 대량의 I/O 버퍼를 RAM에 유지하여 데이터 처리 속도를 극대화함.

📢 섹션 요약 비유: 보안 요원이 "금고 열쇠"를 절대 주머니 밖(디스크)으로 꺼내지 않고 손에 꼭 쥐고 있는 것과 같습니다.

---

## Ⅳ. 부작용 및 주의사항
### 1. 시스템 메모리 부족 (Memory Pressure)
- 너무 많은 페이지를 고정하면 다른 프로세스가 사용할 RAM 공간이 부족해져, 시스템 전체가 스래싱(Thrashing)에 빠질 수 있음.
### 2. 권한 제한
- 일반 사용자는 일정량 이상 메모리를 고정할 수 없도록 설정(ulimit -l)되어 있으며, 대량 고정은 루트 권한이 필요할 수 있음.

📢 섹션 요약 비유: 모든 손님이 "내 방은 절대 못 나간다"고 버티면, 새로 온 손님은 로비(시스템)에서 하염없이 기다려야 하는 상황이 벌어집니다.

---

## Ⅴ. mlock과 요구 페이징 (Demand Paging)
### 1. 즉시 할당 (Immediate Faulting)
- `mlock()` 호출 시, 해당 영역에 아직 물리 프레임이 할당되지 않았다면 즉시 페이지 폴트를 유도하여 RAM에 적재함.
### 2. 미래의 페이지 고정 (MCL_FUTURE)
- `mlockall(MCL_FUTURE)`을 사용하면 앞으로 힙(Heap)이나 스택(Stack)이 늘어날 때 생성되는 페이지도 즉시 고정됨.

📢 섹션 요약 비유: mlock은 "예약뿐만 아니라 지금 당장 방으로 안내하라"는 특급 명령까지 포함된 골드 멤버십 서비스입니다.
