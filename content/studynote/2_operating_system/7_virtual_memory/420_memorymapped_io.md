# 420. FIFO 페이지 교체 (FIFO Page Replacement)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가장 먼저 들어온 페이지 교체
> 2. **가치**: 구현 간단, 오버헤드 작음
> 3. **융합**: 페이지 교체, 큐와 연관

---

## Ⅰ. 개요

### 개념 정의

FIFO 페이지 교체는 **가장 먼저 메모리에 들어온 페이지를 교체하는 알고리즘**이다. 선입선출(First-In First-Out) 방식이다.

### 💡 비유: 편의점 음료수
FIFO는 **편의점 음료수 진열**과 같다. 가장 오래된 음료수를 먼저 판다. 새 음료수는 뒤에 둔다.

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FIFO 시뮬레이션 코드】                                                │
│  ──────────────────                                                  │
│  // 간단한 FIFO 페이지 교체 시뮬레이션                                    │
│  #include <stdio.h>                                                  │
│  #include <stdlib.h>                                                 │
│                                                                     │
│  int fifo_replace(int frames, int *refs, int n) {                   │
│      int *memory = calloc(frames, sizeof(int));                      │
│      int queue[1000], front = 0, rear = 0;                           │
│      int faults = 0, count = 0;                                      │
│                                                                     │
│      for (int i = 0; i < frames; i++) memory[i] = -1;               │
│                                                                     │
│      for (int i = 0; i < n; i++) {                                  │
│          int page = refs[i];                                         │
│          int found = 0;                                              │
│                                                                     │
│          // 페이지 히트 확인                                            │
│          for (int j = 0; j < count; j++) {                          │
│              if (memory[j] == page) { found = 1; break; }            │
│          }                                                           │
│                                                                     │
│          if (!found) {                                               │
│              faults++;                                               │
│              if (count < frames) {                                   │
│                  memory[count++] = page;                             │
│              } else {                                                │
│                  // FIFO 교체                                         │
│                  int victim = queue[front++];                        │
│                  for (int j = 0; j < frames; j++) {                  │
│                      if (memory[j] == victim) {                      │
│                          memory[j] = page;                           │
│                          break;                                      │
│                      }                                               │
│                  }                                                   │
│              }                                                       │
│              queue[rear++] = page;                                   │
│          }                                                           │
│      }                                                               │
│      free(memory);                                                   │
│      return faults;                                                  │
│  }                                                                   │
│                                                                     │
│  【Linux의 FIFO 활용】                                                 │
│  ──────────────────                                                  │
│  // 실제 Linux는 FIFO 단독 사용 안 함                                    │
│  // 하지만 일부 상황에서 FIFO 유사 동작                                   │
│                                                                     │
│  // 페이지 교체 정책 확인                                                 │
│  $ cat /sys/kernel/mm/transparent_hugepage/enabled                  │
│  // always [madvise] never                                          │
│                                                                     │
│  【성능 측정】                                                            │
│  ──────────────────                                                  │
│  // 페이지 폴트 측정                                                      │
│  $ perf stat -e page-faults,faults ./fifo_program                   │
│                                                                     │
│  // Belady's Anomaly 확인                                               │
│  // 다양한 프레임 수로 테스트                                               │
│  for frames in 2 3 4 5 6; do                                        │
│      echo "Frames: $frames"                                          │
│      ./fifo_sim $frames < reference_string.txt                      │
│  done                                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 가장 먼저 들어온 페이지 교체
• 구조: FIFO 큐로 관리
• 장점: 구현 간단, 오버헤드 작음
• 단점: 지역성 무시, 성능 낮음
• 이상: Belady's Anomaly 발생 가능
• 실제: 단독 사용 드묾, 다른 알고리즘과 조합
• 비교: LRU, Clock보다 성능 낮음
```
