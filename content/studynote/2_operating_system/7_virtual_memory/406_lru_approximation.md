# 406. 메모리 보호 (Memory Protection)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 간 메모리 접근 제어
> 2. **가치**: 시스템 안정성과 보안 보장
> 3. **융합**: 가상 메모리, 페이지 테이블과 연관

---

## Ⅰ. 개요

### 개념 정의

메모리 보호(Memory Protection)는 **프로세스가 다른 프로세스나 커널의 메모리에 무단 접근하는 것을 방지하는 메커니즘**이다. 하드웨어와 OS가 협력하여 구현한다.

### 💡 비유: 집 문잠금
메모리 보호는 **집 문잠금**과 같다. 내 방은 내가 들어갈 수 있다. 남의 방은 못 들어간다. 문이 잠겨 있다.

### 메모리 보호 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 보호 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【보호 비트】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  페이지 테이블 엔트리의 보호 비트:                                        │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │ R │ W │ X │ U/S │                                        │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  │  R (Read):     읽기 권한                                              │ │
│  │  W (Write):    쓰기 권한                                              │ │
│  │  X (Execute):  실행 권한                                              │ │
│  │  U/S (User/Supervisor): 사용자/커널 모드                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【접근 위반 예시】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  사용자 모드에서:                                                       │ │
│  │  • 커널 메모리 접근 → Segmentation Fault                              │ │
│  │  • 다른 프로세스 메모리 접근 → Segmentation Fault                      │ │
│  │  • 쓰기 금지 영역에 쓰기 → Segmentation Fault                          │ │
│  │  • 실행 금지 영역 실행 → Segmentation Fault                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【보호 예시】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  코드 영역:     R-X (읽기, 실행)                                       │ │
│  │  데이터 영역:   RW- (읽기, 쓰기)                                       │ │
│  │  스택 영역:     RW- (읽기, 쓰기)                                       │ │
│  │  커널 영역:     RWX, S (커널만 접근)                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 보호 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드웨어 지원】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  MMU (Memory Management Unit):                                        │ │
│  │  • 주소 변환 시 보호 비트 검사                                          │ │
│  │  • 위반 시 CPU에 예외 발생                                             │ │
│  │                                                             │ │
│  │  x86:                                                                 │ │
│  │  • CR0.WP: 쓰기 보호 비트                                              │ │
│  │  • Page Table Entry의 R/W, U/S 비트                                  │ │
│  │                                                             │ │
│  │  ARM:                                                                 │ │
│  │  • AP (Access Permission) 비트                                        │ │
│  │  • XN (Execute Never) 비트                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DEP/NX 비트】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  DEP (Data Execution Prevention):                                     │ │
│  │  • 데이터 영역에서 코드 실행 방지                                       │ │
│  │  • 버퍼 오버플로우 공격 방어                                            │ │
│  │                                                             │ │
│  │  NX (No eXecute) 비트:                                                 │ │
│  │  • 페이지에 실행 권한 없음 표시                                         │ │
│  │  • 스택, 힙 등 데이터 영역에 설정                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ASLR】                                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Address Space Layout Randomization:                                  │ │
│  │  • 메모리 영역 위치 무작위화                                            │ │
│  │  • 공격자가 주소 예측 어려움                                            │ │
│  │  • 코드, 데이터, 스택, 힙 등                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 영역 보호 확인】                                                  │
│  ──────────────────                                                  │
│  // Linux: /proc/pid/maps                                             │
│  $ cat /proc/self/maps | head -5                                     │
│  555555554000-555555555000 r--p 00000000 08:01 12345 /bin/cat        │
│  555555555000-555555556000 r-xp 00001000 08:01 12345 /bin/cat        │
│  555555556000-555555557000 r--p 00002000 08:01 12345 /bin/cat        │
│  555555557000-555555558000 r--p 00002000 08:01 12345 /bin/cat        │
│  555555558000-555555559000 rw-p 00003000 08:01 12345 /bin/cat        │
│  //              ^^^                                                │
│  //              r=read, w=write, x=execute                          │
│                                                                     │
│  【mprotect로 보호 설정】                                                 │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│                                                                     │
│  void* mem = malloc(4096);                                           │
│                                                                     │
│  // 읽기 전용으로 변경                                                    │
│  mprotect(mem, 4096, PROT_READ);                                     │
│                                                                     │
│  // 실행 권한 추가                                                       │
│  mprotect(mem, 4096, PROT_READ | PROT_EXEC);                        │
│                                                                     │
│  // 모든 권한 부여                                                       │
│  mprotect(mem, 4096, PROT_READ | PROT_WRITE | PROT_EXEC);           │
│                                                                     │
│  【DEP/NX 확인】                                                            │
│  ──────────────────                                                  │
│  // 스택 실행 방지 확인                                                   │
│  $ readelf -l /bin/ls | grep GNU_STACK                              │
│  GNU_STACK      0x000000 0x00000000 0x00000000 0x00000 0x00000 RWE  │
│  //                                                         ^^^      │
│  //                                           RWE면 실행 가능            │
│  //                                           RW-면 실행 불가 (NX)        │
│                                                                     │
│  【ASLR 확인】                                                              │
│  ──────────────────                                                  │
│  $ cat /proc/sys/kernel/randomize_va_space                          │
│  2  // 0=비활성, 1=부분, 2=완전                                          │
│                                                                     │
│  // 여러 번 실행해서 주소 변화 확인                                        │
│  $ for i in 1 2 3; do cat /proc/self/maps | grep stack; done        │
│  7ffc12345000-7ffc12346000 rw-p ... [stack]                          │
│  7ffc67890000-7ffc67891000 rw-p ... [stack]  ← 주소 다름              │
│  7ffcabcd0000-7ffcabcd1000 rw-p ... [stack]                          │
│                                                                     │
│  【보안 관련 처리】                                                        │
│  ──────────────────                                                  │
│  // Segmentation Fault 처리                                             │
│  #include <signal.h>                                                 │
│                                                                     │
│  void handler(int sig) {                                             │
│      printf("Memory violation!\n");                                   │
│      exit(1);                                                        │
│  }                                                                  │
│                                                                     │
│  signal(SIGSEGV, handler);                                          │
│                                                                     │
│  // 의도적 위반 (테스트용)                                               │
│  int* ptr = NULL;                                                    │
│  *ptr = 1;  // SIGSEGV 발생                                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
