## 💡 Insight
- NUMA(Non-Uniform Memory Access) 구조에서는 CPU가 어떤 메모리 뱅크에 접근하느냐에 따라 성능 차이(Latency)가 발생한다.
- 가상 메모리 관리자는 프로세스가 실행되는 CPU와 가까운 물리 메모리(Local Node)를 우선 할당하는 'First-touch Policy'를 기본으로 한다.
- 성능 최적화를 위해 프로세스(Thread)와 메모리 페이지를 적절한 노드에 묶는 친화도(Affinity) 및 자동 NUMA 밸런싱이 필수적이다.

---

## Ⅰ. NUMA (Non-Uniform Memory Access) 아키텍처 개요
### 1. 정의
- 멀티 CPU 시스템에서, 각 CPU(또는 소켓)가 자신만의 로컬 메모리 컨트롤러를 가지고 독립된 메모리 영역을 관리하는 비균등 메모리 접근 방식.
### 2. 접근 속도 차이 (Latency Gap)
- **Local Access**: 자신의 노드에 연결된 RAM 접근 (매우 빠름).
- **Remote Access**: 상호 연결망(QPI/UPI 등)을 통해 다른 노드의 RAM 접근 (상대적으로 느림).

📢 섹션 요약 비유: 각 층(CPU)마다 전용 편의점(로컬 메모리)이 있는 건물에서, 자기 층 가게를 가면 빠르지만 다른 층(리모트 메모리)까지 가면 엘리베이터 기다리는 시간이 걸리는 것과 같습니다.

---

## Ⅱ. NUMA 환경의 메모리 할당 정책
### 1. First-touch Policy (기본 정책)
- 페이지가 처음 참조될 때(Page Fault 시점), 그 순간 해당 코드를 실행 중인 CPU의 로컬 노드에 실제 물리 프레임을 할당함.
### 2. Interleave Policy
- 메모리를 여러 노드에 번갈아가며 분산 배치. 특정 노드에 트래픽이 몰리는 병목 현상을 방지할 때 사용.
### 3. Preferred / Bind Policy
- 특정 노드를 우선시하거나, 오직 지정된 노드에서만 할당받도록 강제함.

### 4. ASCII 다이어그램: Local vs Remote Access
```text
[ Node 0 ]                     [ Node 1 ]
+-------+   (Fast)   +-----+   |   +-----+   (Fast)   +-------+
| CPU 0 | <--------> | RAM |---|---| RAM | <--------> | CPU 1 |
+-------+            +-----+   |   +-----+            +-------+
    |                          |                          |
    +--------------------------+--------------------------+
               (Interconnect: Remote Access - SLOW)
```

📢 섹션 요약 비유: 요리를 시작할 때(First-touch), 내 주방(Local Node)에 재료를 꺼내 놓아야 나중에 요리가 빠릅니다. 옆집 주방(Remote Node)에 두면 계속 왔다 갔다 해야 하죠.

---

## Ⅲ. 자동 NUMA 밸런싱 (Auto NUMA Balancing)
### 1. 매커니즘
- 커널이 주기적으로 프로세스의 메모리 접근 패턴을 감시함.
- 프로세스가 다른 노드의 메모리를 자꾸 쓴다면, **메모리 페이지를 해당 프로세스가 있는 노드로 이사(Migration)** 시키거나 프로세스를 메모리 근처로 옮김.
### 2. 페이지 폴트 힌트 (Page Fault Hint)
- 페이지를 일부러 'Unmapped' 상태로 두어 다시 접근할 때 트랩을 발생시켜 위치 정보를 갱신함.

📢 섹션 요약 비유: 이삿짐 센터 직원이 집주인이 자주 가는 동네를 지켜보다가, 알아서 직장 근처로 집(메모리)을 옮겨주는 스마트 이사 서비스입니다.

## Ⅳ. 가상 메모리와 NUMA 제어 툴
### 1. numactl
- 특정 정책을 적용하여 프로그램을 실행함. (예: `numactl --membind=0 ./app`)
### 2. libnuma
- 프로그램 소스 코드 내에서 메모리 노드 할당 로직을 직접 제어할 수 있는 라이브러리.

📢 섹션 요약 비유: "나는 이 방만 쓰겠다"거나 "방을 공평하게 나눠 쓰겠다"고 미리 규칙(numactl)을 정하는 가이드라인입니다.

## Ⅴ. NUMA 환경의 성능 고려사항
### 1. NUMA Miss (Remote Access)
- 리모트 메모리 접근 비율이 높을수록 시스템 전체 처리량(Throughput)이 급격히 하락함.
### 2. 쓰레드 친화성 (Thread Affinity)
- CPU 스케줄러가 쓰레드를 다른 노드로 옮겨버리면, 원래 로컬이었던 메모리가 갑자기 리모트가 되어 성능이 떨어질 수 있음. (스케줄러와의 협업 중요)

📢 섹션 요약 비유: 전용 요리사가 갑자기 다른 층 주방으로 불려가면, 원래 쓰던 도구(메모리)가 멀어져서 요리 시간이 길어지는 것과 같습니다.
