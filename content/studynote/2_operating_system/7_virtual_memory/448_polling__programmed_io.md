# 506. 폴링 (Polling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장치 상태를 주기적으로 확인
> 2. **가치**: 구현 단순, 낮은 지연
> 3. **융합**: 인터럽트, 스풀링, 버퍼링과 연관

---

## Ⅰ. 개요

### 개념 정의

폴링(Polling)은 **CPU가 I/O 장치의 상태를 주기적으로 확인하여 준비 여부를 검사하는 방식**이다.

### 💡 비유: 빈번한 확인
폴링은 **택시 기다리며 계속 창밖 보기**와 같다. 왔는지 안 왔는지 계속 확인한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                폴링 상세                                               │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【폴링 종류】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 스핀 대기 (Spin Wait / Busy Wait):                                   │ │
│  │     • while 루프로 계속 확인                                            │ │
│  │     • CPU 100% 사용                                                     │ │
│  │     • 응답 시간 최소                                                     │ │
│  │     • 짧은 대기에 적합                                                   │ │
│  │                                                             │ │
│  │  2. 인터벌 폴링 (Interval Polling):                                      │ │
│  │     • 일정 간격으로 확인                                                │ │
│  │     • sleep/usleep 사용                                                 │ │
│  │     • CPU 사용 절약                                                     │ │
│  │     • 응답 지연 증가                                                     │ │
│  │                                                             │ │
│  │  3. 하이브리드 (Hybrid):                                                 │ │
│  │     • 짧은 시간 스핀 후 인터벌 폴링                                       │ │
│  │     • 균형 잡힌 접근                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【폴링 적합 상황】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  상황              이유                                                  │ │
│  │  ────              ────                                                  │ │
│  │  고속 장치         지연 최소화로 처리량 증가                                │ │
│  │  고부하            인터럽트 폭주 방지                                     │ │
│  │  실시간 시스템     예측 가능한 지연                                        │ │
│  │  단순 임베디드    인터럽트 없이 구현                                       │ │
│  │  커널 초기화       인터럽트 비활성 상태                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                                 │ │
│  │  • 구현이 매우 단순                                                    │ │
│  │  • 인터럽트 오버헤드 없음                                               │ │
│  │  • 낮은 지연 시간                                                      │ │
│  │  • 예측 가능한 동작                                                    │ │
│  │                                                             │ │
│  │  단점:                                                                 │ │
│  │  • CPU 사이클 낭비                                                     │ │
│  │  • 전력 소모 증가                                                      │ │
│  │  • 다른 작업 지연                                                      │ │
│  │  • 멀티태스킹에 부적합                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C 언어 폴링 예시】                                                   │
│  ──────────────────                                                  │
│  // 스핀 대기 (Busy Wait)                                               │
│  #include <stdint.h>                                                  │
│                                                                     │
│  #define STATUS_REG  (*(volatile uint8_t*)0x1000)                    │
│  #define DATA_REG    (*(volatile uint8_t*)0x1001)                    │
│  #define READY_BIT   0x01                                             │
│                                                                     │
│  void poll_wait() {                                                   │
│      while (!(STATUS_REG & READY_BIT)) {                             │
│          // 계속 확인 (busy wait)                                       │
│      }                                                                │
│  }                                                                    │
│                                                                     │
│  uint8_t read_device() {                                              │
│      poll_wait();                                                     │
│      return DATA_REG;                                                 │
│  }                                                                    │
│                                                                     │
│  void write_device(uint8_t data) {                                    │
│      poll_wait();                                                     │
│      DATA_REG = data;                                                 │
│  }                                                                    │
│                                                                     │
│  【인터벌 폴링】                                                        │
│  ──────────────────                                                  │
│  #include <unistd.h>                                                  │
│                                                                     │
│  int poll_with_timeout(int timeout_ms) {                              │
│      int elapsed = 0;                                                 │
│      while (elapsed < timeout_ms) {                                   │
│          if (STATUS_REG & READY_BIT) {                                │
│              return 1;  // 준비됨                                       │
│          }                                                            │
│          usleep(1000);  // 1ms 대기                                    │
│          elapsed += 1;                                                │
│      }                                                                │
│      return 0;  // 타임아웃                                              │
│  }                                                                    │
│                                                                     │
│  【Linux 커널 폴링】                                                   │
│  ──────────────────                                                  │
│  // 조건 변수 대기                                                      │
│  #include <linux/wait.h>                                              │
│                                                                     │
│  wait_queue_head_t my_queue;                                          │
│  init_waitqueue_head(&my_queue);                                      │
│                                                                     │
│  // 폴링 대기                                                          │
│  wait_event_interruptible(my_queue, (STATUS_REG & READY_BIT));       │
│                                                                     │
│  // 타임아웃 있는 대기                                                   │
│  wait_event_interruptible_timeout(my_queue,                          │
│                                   (STATUS_REG & READY_BIT),          │
│                                   msecs_to_jiffies(1000));           │
│                                                                     │
│  【사용자 공간 장치 폴링】                                               │
│  ──────────────────                                                  │
│  // GPIO 폴링                                                          │
│  #include <gpiod.h>                                                   │
│                                                                     │
│  struct gpiod_line *line = gpiod_line_get("gpiochip0", 17);          │
│  while (1) {                                                          │
│      int value = gpiod_line_get_value(line);                          │
│      if (value == 1) {                                                │
│          // 버튼 눌림 처리                                               │
│      }                                                                │
│      usleep(10000);  // 10ms 폴링 간격                                  │
│  }                                                                    │
│                                                                     │
│  【select/poll/epoll】                                                │
│  ──────────────────                                                  │
│  // poll() 시스템 콜                                                    │
│  #include <poll.h>                                                    │
│                                                                     │
│  struct pollfd fds[1];                                                │
│  fds[0].fd = device_fd;                                               │
│  fds[0].events = POLLIN;                                              │
│                                                                     │
│  int ret = poll(fds, 1, 1000);  // 1초 타임아웃                         │
│  if (ret > 0 && (fds[0].revents & POLLIN)) {                          │
│      // 데이터 읽기                                                     │
│      read(device_fd, buffer, sizeof(buffer));                         │
│  }                                                                    │
│                                                                     │
│  // epoll (Linux 고성능)                                               │
│  int epfd = epoll_create1(0);                                         │
│  struct epoll_event ev;                                               │
│  ev.events = EPOLLIN;                                                 │
│  ev.data.fd = device_fd;                                              │
│  epoll_ctl(epfd, EPOLL_CTL_ADD, device_fd, &ev);                      │
│                                                                     │
│  int n = epoll_wait(epfd, events, MAX_EVENTS, -1);                    │
│                                                                     │
│  【네트워크 폴링】                                                      │
│  ──────────────────                                                  │
│  // 논블로킹 소켓                                                       │
│  int flags = fcntl(sock, F_GETFL, 0);                                 │
│  fcntl(sock, F_SETFL, flags | O_NONBLOCK);                            │
│                                                                     │
│  // 폴링으로 데이터 확인                                                 │
│  while (1) {                                                          │
│      ssize_t n = recv(sock, buffer, sizeof(buffer), 0);               │
│      if (n > 0) {                                                     │
│          // 데이터 처리                                                 │
│      } else if (n == -1 && errno == EAGAIN) {                         │
│          usleep(1000);  // 데이터 없음, 대기                              │
│      }                                                                │
│  }                                                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 장치 상태를 주기적으로 확인
• 종류: 스핀 대기, 인터벌 폴링, 하이브리드
• 장점: 구현 단순, 낮은 지연, 예측 가능
• 단점: CPU 낭비, 전력 소모, 멀티태스킹 부적합
• 적합: 고속 장치, 고부하, 실시간 시스템
• 구현: while 루프, select/poll/epoll
• 대안: 인터럽트 (저부하 시)
• 트레이드오프: 지연 ↔ CPU 사용량
```
