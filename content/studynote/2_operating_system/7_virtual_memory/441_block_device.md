# 441. 파일 시스템 (File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 저장하고 관리하는 체계
> 2. **가치**: 영속성, 조직화, 접근성 제공
> 3. **융합**: 운영체제, 저장장치와 연관

---

## Ⅰ. 개요

### 개념 정의

파일 시스템(File System)은 **저장장치에 데이터를 체계적으로 저장하고 관리하기 위한 방법과 자료 구조**다. 파일과 디렉토리를 조직화한다.

### 💡 비유: 도서관
파일 시스템은 **도서관**과 같다. 책(파일)을 선반(디스크)에 정리한다. 분류 시스템(디렉토리)으로 찾는다.

### 파일 시스템 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【계층 구조】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  응용 프로그램 (Application)                                            │ │
│  │       │                                                             │ │
│  │       ▼                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  가상 파일 시스템 (VFS - Virtual File System)              │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │       │                                                             │ │
│  │       ├─────────────┬─────────────┬─────────────┐                │ │
│  │       ▼             ▼             ▼             ▼                │ │
│  │  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐          │ │
│  │  │  EXT4   │   │  XFS    │   │  Btrfs  │   │  FAT32  │          │ │
│  │  │  파일시스템│   │  파일시스템│   │  파일시스템│   │  파일시스템│          │ │
│  │  └─────────┘   └─────────┘   └─────────┘   └─────────┘          │ │
│  │       │             │             │             │                │ │
│  │       └─────────────┴─────────────┴─────────────┘                │ │
│  │       │                                                             │ │
│  │       ▼                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  블록 장치 드라이버 (Block Device Driver)                   │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │       │                                                             │ │
│  │       ▼                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  물리 저장 장치 (HDD, SSD, etc.)                           │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파일 시스템 구성 요소】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 파일 (File): 논리적 데이터 단위                                     │ │
│  │  2. 디렉토리 (Directory): 파일을 계층적으로 조직                          │ │
│  │  3. 메타데이터 (Metadata): 파일 속성 정보                                │ │
│  │  4. 데이터 블록 (Data Block): 실제 데이터 저장                           │ │
│  │  5. 아이노드 (Inode): 유닉스 계열 메타데이터 구조                         │ │
│  │  6. 슈퍼블록 (Superblock): 파일 시스템 전체 정보                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파일 시스템 확인】                                                    │
│  ──────────────────                                                  │
│  // 마운트된 파일 시스템 확인                                             │
│  $ df -Th                                                            │
│  Filesystem     Type      Size  Used Avail Use% Mounted on          │
│  /dev/sda1      ext4      100G   50G   50G  50% /                    │
│  /dev/sda2      xfs       200G   80G  120G  40% /home                │
│                                                                     │
│  // 파일 시스템 상세 정보                                                  │
│  $ tune2fs -l /dev/sda1 | head -20                                  │
│  // 또는                                                                 │
│  $ xfs_info /dev/sda2                                                │
│                                                                     │
│  【파일 시스템 생성】                                                    │
│  ──────────────────                                                  │
│  // ext4 파일 시스템 생성                                                │
│  $ mkfs.ext4 /dev/sdb1                                               │
│                                                                     │
│  // XFS 파일 시스템 생성                                                 │
│  $ mkfs.xfs /dev/sdb2                                                │
│                                                                     │
│  // 마운트                                                                │
│  $ mount /dev/sdb1 /mnt/data                                         │
│  $ mount -t xfs /dev/sdb2 /mnt/backup                                │
│                                                                     │
│  // 영구 마운트 (/etc/fstab)                                            │
│  /dev/sdb1  /mnt/data  ext4  defaults  0  0                         │
│  /dev/sdb2  /mnt/backup xfs   defaults  0  0                        │
│                                                                     │
│  【파일 시스템 유지보수】                                                │
│  ──────────────────                                                  │
│  // 파일 시스템 검사                                                      │
│  $ fsck /dev/sda1                                                    │
│  // 또는 (ext4)                                                         │
│  $ e2fsck -f /dev/sda1                                               │
│                                                                     │
│  // 파일 시스템 정보 조회                                                  │
│  $ stat /etc/passwd                                                   │
│  File: /etc/passwd                                                    │
│  Size: 2847      Blocks: 8          IO Block: 4096                  │
│  Inode: 12345    Links: 1                                               │
│                                                                     │
│  // 디스크 사용량                                                          │
│  $ du -sh /home                                                      │
│  50G    /home                                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 저장장치에 데이터를 체계적으로 저장/관리
• 목적: 영속성, 조직화, 접근성, 보호
• 구조: VFS → 파일시스템 → 블록드라이버 → 저장장치
• 구성: 파일, 디렉토리, 메타데이터, 블록
• 유닉스: ext4, XFS, Btrfs, ZFS
• Windows: NTFS, FAT32
• 네트워크: NFS, SMB/CIFS
• 명령어: mkfs, mount, fsck, df, du
```
