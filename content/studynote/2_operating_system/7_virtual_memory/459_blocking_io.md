# 459. 탐색 시간 (Seek Time)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 헤드가 원하는 트랙으로 이동하는 시간
> 2. **가치**: 디스크 성능의 주요 병목
> 3. **융합**: 디스크 스케줄링, 회전 지연과 연관

---

## Ⅰ. 개요

### 개념 정의

탐색 시간(Seek Time)은 **디스크 헤드가 현재 위치에서 원하는 트랙으로 이동하는 데 걸리는 시간**이다.

### 💡 비유: 턴테이블
탐색 시간은 **턴테이블의 바늘**과 같다. 바늘이 원하는 트랙(노래)으로 이동하는 시간이다.

### 탐색 시간 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                탐색 시간                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【탐색 시간 구성】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  탐색 시간 = 가속 시간 + 등속 이동 시간 + 정착 시간                     │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  1. 가속 (Acceleration):                                  │    │ │
│  │  │     • 헤드 암이 움직이기 시작                                   │    │ │
│  │  │     • 0 → 최대 속도까지 가속                                   │    │ │
│  │  │     • 짧은 이동에서는 최대 속도 미달                             │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  2. 등속 이동 (Coast):                                    │    │ │
│  │  │     • 최대 속도로 이동                                        │    │ │
│  │  │     • 긴 이동에서만 발생                                        │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  3. 정착 (Settling):                                      │    │ │
│  │  │     • 목표 트랙에서 속도 감소                                   │    │ │
│  │  │     • 정확한 위치에 정착                                        │    │ │
│  │  │     • 미세 조정 포함                                          │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【탐색 시간 특성】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  트랙 수    1     10    50    100   200   400                   │ │
│  │  ──────    ───   ───   ───   ───   ───   ───                    │ │
│  │  시간(ms)  0.5   1.0   2.5   4.0   6.0   9.0                    │ │
│  │                                                             │ │
│  │  비선형적: 트랙 수에 정비례하지 않음                                      │ │
│  │  가속/감속 오버헤드 때문                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                탐색 시간 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【탐색 시간 분류】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 평균 탐색 시간 (Average Seek Time):                              │ │
│  │     • 모든 가능한 트랙 이동의 평균                                      │ │
│  │     • 일반 HDD: 5-10 ms                                             │ │
│  │     • 고성능 HDD: 3-5 ms                                            │ │
│  │     • 디스크 성능 비교의 핵심 지표                                      │ │
│  │                                                             │ │
│  │  2. 트랙간 탐색 (Track-to-Track Seek):                               │ │
│  │     • 인접 트랙 이동 시간                                             │ │
│  │     • 보통 0.5-1 ms                                                 │ │
│  │     • 순차 읽기 성능에 영향                                           │ │
│  │                                                             │ │
│  │  3. 전체 스트로크 (Full Stroke):                                      │ │
│  │     • 처음에서 끝 트랙까지 이동                                        │ │
│  │     • 보통 15-20 ms                                                 │ │
│  │     • 최악의 경우                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【탐색 시간 영향 요소】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  요소              영향                                               │ │
│  │  ────              ────                                              │ │
│  │  디스크 밀도       높을수록 탐색 정밀도 필요 → 시간 증가                   │ │
│  │  트랙 수           많을수록 이동 거리 증가                               │ │
│  │  RPM              직접 영향 없음                                       │ │
│  │  암 메커니즘       빠른 암 = 짧은 탐색 시간                              │ │
│  │  온도             극단적 온도에서 영향                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【접근 시간에서 비중】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  디스크 접근 시간 = 탐색 시간 + 회전 지연 + 전송 시간                     │ │
│  │                                                             │ │
│  │  일반적인 HDD (7200 RPM):                                            │ │
│  │  ┌───────────────────────────────────────────────────────┐        │ │
│  │  │  탐색 시간      │████████████│ 8 ms (53%)               │        │ │
│  │  │  회전 지연      │████████│ 4 ms (27%)                   │        │ │
│  │  │  전송 시간      │███│ 3 ms (20%)                       │        │ │
│  │  └───────────────────────────────────────────────────────┘        │ │
│  │  총 접근 시간: 약 15 ms                                               │ │
│  │                                                             │ │
│  │  탐색 시간이 가장 큰 비중 차지!                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【탐색 시간 확인】                                                    │
│  ──────────────────                                                  │
│  // hdparm으로 확인                                                   │
│  $ sudo hdparm -I /dev/sda | grep -i seek                           │
│  (일부 디스크에서는 제공하지 않음)                                      │
│                                                                     │
│  // hdparm으로 타이밍 측정                                             │
│  $ sudo hdparm -t /dev/sda                                          │
│  Timing buffered disk reads: 150 MB/s                              │
│                                                                     │
│  // iostat로 서비스 시간 확인 (탐색+회전+전송)                           │
│  $ iostat -x 1                                                      │
│  Device  svctm  await                                               │
│  sda    5.00   8.50                                                 │
│  // svctm = 평균 서비스 시간 (ms)                                       │
│                                                                     │
│  【탐색 시간 최소화】                                                  │
│  ──────────────────                                                  │
│  1. 디스크 스케줄링 사용:                                               │
│     • SSTF, SCAN, C-SCAN 등                                          │
│     • 연속 요청을 인접 위치로 모음                                        │
│                                                                     │
│  2. 데이터 배치 최적화:                                                │
│     • 자주 사용하는 데이터를 인접 위치에                                  │
│     • 조각 모음 (defragmentation)                                     │
│                                                                     │
│  3. 캐싱:                                                             │
│     • 버퍼 캐시, 페이지 캐시                                             │
│     • 디스크 접근 자체를 줄임                                            │
│                                                                     │
│  4. SSD 사용:                                                        │
│     • 탐색 시간 없음 (0 ms)                                            │
│     • 랜덤 I/O 성능 대폭 향상                                           │
│                                                                     │
│  【프로그래밍에서 고려사항】                                            │
│  ──────────────────                                                  │
│  // 순차 읽기 (탐색 최소)                                               │
│  char buffer[1024*1024];  // 큰 버퍼                                  │
│  read(fd, buffer, sizeof(buffer));  // 연속 읽기                       │
│                                                                     │
│  // 랜덤 읽기 (탐색 많이 발생)                                           │
│  for (int i = 0; i < 1000; i++) {                                  │
│      lseek(fd, random_offsets[i], SEEK_SET);  // 매번 탐색            │
│      read(fd, buf, 512);                                            │
│  }                                                                   │
│                                                                     │
│  // 성능 차이: 순차가 10-100배 빠름                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 헤드가 트랙으로 이동하는 시간
• 구성: 가속 + 등속 + 정착
• 평균: 5-10 ms (일반 HDD)
• 트랙간: 0.5-1 ms
• 전체 스트로크: 15-20 ms
• 비중: 접근 시간의 50% 이상
• 최소화: 스케줄링, 조각 모음, 캐싱, SSD
```
