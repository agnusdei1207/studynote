# 422. Optimal 페이지 교체 (Optimal Page Replacement)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 앞으로 가장 오랫동안 안 쓸 페이지 교체
> 2. **가치**: 이론적 최적 성능 기준
> 3. **융합**: 페이지 교체, 미래 지식과 연관

---

## Ⅰ. 개요

### 개념 정의

Optimal 페이지 교체는 **앞으로 가장 오랫동안 사용되지 않을 페이지를 교체하는 알고리즘**이다. 미래의 참조 패턴을 알고 있다고 가정한다.

### 💡 비유: 요정의 예언
Optimal은 **미래를 보는 요정**과 같다. 어떤 책이 제일 늦게 필요한지 안다. 그 책을 치운다.

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Optimal 시뮬레이션】                                                 │
│  ──────────────────                                                  │
│  #include <stdio.h>                                                  │
│  #include <limits.h>                                                 │
│                                                                     │
│  int find_optimal_victim(int *memory, int frames,                   │
│                           int *refs, int current, int n) {          │
│      int farthest = -1, victim = 0;                                  │
│                                                                     │
│      for (int i = 0; i < frames; i++) {                             │
│          int page = memory[i];                                       │
│          int next_use = INT_MAX;                                     │
│                                                                     │
│          // 다음 참조 위치 찾기                                          │
│          for (int j = current + 1; j < n; j++) {                    │
│              if (refs[j] == page) {                                  │
│                  next_use = j;                                       │
│                  break;                                              │
│              }                                                       │
│          }                                                           │
│                                                                     │
│          // 가장 먼 미래에 참조되는 페이지                                 │
│          if (next_use > farthest) {                                  │
│              farthest = next_use;                                    │
│              victim = i;                                             │
│          }                                                           │
│      }                                                               │
│      return victim;                                                  │
│  }                                                                   │
│                                                                     │
│  【알고리즘 성능 비교 실험】                                              │
│  ──────────────────                                                  │
│  // 참조 문자열로 성능 비교                                               │
│  $ ./page_sim -a optimal,lru,fifo,clock -r trace.txt                │
│                                                                     │
│  // 결과 예시                                                            │
│  Algorithm    Faults  Hit Rate                                      │
│  ──────────  ──────  ────────                                      │
│  Optimal       7      72%                                           │
│  LRU          10      60%                                           │
│  Clock         9      64%                                           │
│  FIFO          9      64%                                           │
│                                                                     │
│  【실제 시스템에서의 활용】                                              │
│  ──────────────────                                                  │
│  // Optimal은 직접 구현 안 됨                                            │
│  // 하지만 다음과 같이 간접 활용:                                          │
│                                                                     │
│  // 1. 사전 컴파일된 참조 패턴 분석                                        │
│  //    - 프로파일링으로 참조 패턴 수집                                      │
│  //    - 오프라인 분석으로 최적 페이지 배치                                  │
│                                                                     │
│  // 2. 힌트 기반 프리페칭                                                 │
│  //    - 프로그래머가 힌트 제공                                            │
│  //    - madvise() 시스템 콜                                            │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_WILLNEED);                      │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_SEQUENTIAL);                    │
│                                                                     │
│  【연구 및 교육 활용】                                                    │
│  ──────────────────                                                  │
│  // 페이지 교체 알고리즘 연구                                               │
│  // Optimal에 얼마나 가까운지 측정                                         │
│  optimality_ratio = optimal_faults / actual_faults                  │
│  // 1.0에 가까울수록 좋음                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
