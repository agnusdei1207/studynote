# 439. OOM 킬러 (OOM Killer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 부족 시 프로세스 종료
> 2. **가치**: 시스템 보호
> 3. **융합**: 메모리 과할당, 메모리 압력과 연관

---

## Ⅰ. 개요

### 개념 정의

OOM 킬러(Out of Memory Killer)는 **시스템의 메모리가 완전히 고갈되었을 때 프로세스를 종료하여 메모리를 확보하는 커널 메커니즘**이다.

### 💡 비유: 비상 탈출
OOM 킬러는 **비상 탈출**과 같다. 배가 가라앉을 때 짐을 버린다. 무거운 짐부터 버린다.

### OOM 킬러 구조

```

│  │  1. 페이지 할당 실패                                                      │ │

│  │  └─────────────────────────────────────────────────────────┘  │ │
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                OOM 킬러 상세                                           │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【oom_score_adj】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  사용자 조정 가능:                                                       │ │
│  │                                                             │ │
│  │  -1000: 절대 종료 안 됨 (보호)                                           │ │
│  │  0:      기본값                                                          │ │
│  │  +1000: 항상 종료 대상                                                   │ │
│  │                                                             │ │
│  │  예: 중요 서비스 보호                                                    │ │
│  │  echo -1000 > /proc/<pid>/oom_score_adj                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【OOM 동작 설정】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  /proc/sys/vm/panic_on_oom:                                             │ │
│  │                                                             │ │
│  │  0: OOM 킬러 동작 (기본)                                                 │ │
│  │  1: 커널 패닉 (시스템 재부팅)                                            │ │
│  │  2: 항상 커널 패닉                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【cgroup과 OOM】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  cgroup 메모리 제한 초과 시:                                             │ │
│  │                                                             │ │
│  │  • 해당 cgroup 내에서만 OOM 발생                                         │ │
│  │  • 다른 cgroup 영향 없음                                                 │ │
│  │  • memory.oom_control로 제어                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【OOM 상태 확인】                                                      │
│  ──────────────────                                                  │
│  // OOM 로그 확인                                                        │
│  $ dmesg | grep -i "out of memory"                                  │
│  Out of memory: Killed process 1234 (bigprogram) oom_score=800      │
│  Out of memory: Killed process 5678 (anotherprog) oom_score=600     │
│                                                                     │
│  // /var/log/syslog 또는 journalctl                                    │
│  $ journalctl -k | grep -i "out of memory"                          │
│                                                                     │
│  // 프로세스 OOM 점수 확인                                               │
│  $ cat /proc/<pid>/oom_score                                        │
│  800                                                                 │
│                                                                     │
│  $ cat /proc/<pid>/oom_score_adj                                    │
│  0                                                                   │
│                                                                     │
│  【프로세스 보호 설정】                                                  │
│  ──────────────────                                                  │
│  // 중요 프로세스 보호                                                   │
│  $ echo -1000 > /proc/<pid>/oom_score_adj                           │
│  // 예: 데이터베이스, 웹 서버                                            │
│                                                                     │
│  // 종료 대상으로 지정                                                   │
│  $ echo 1000 > /proc/<pid>/oom_score_adj                            │
│  // 예: 테스트 프로세스, 배치 작업                                        │
│                                                                     │
│  // systemd 서비스에서 설정                                              │
│  [Service]                                                            │
│  OOMScoreAdjust=-1000                                                 │
│                                                                     │
│  【OOM 비활성화 (위험!)】                                                 │
│  ──────────────────                                                  │
│  // cgroup에서 OOM 비활성화                                              │
│  $ echo 1 > /sys/fs/cgroup/memory/mygroup/memory.oom_control        │
│  // OOM 발생 시 프로세스 중단 (SIGSTOP)                                  │
│                                                                     │
│  // 전체 OOM 비활성화 (위험)                                              │
│  $ sysctl vm.overcommit_memory=2                                    │
│  $ sysctl vm.panic_on_oom=0                                         │
│                                                                     │
│  【OOM 방지 전략】                                                        │
│  ──────────────────                                                  │
│  // 1. 메모리 제한 설정                                                   │
│  $ ulimit -v 4194304  // 가상 메모리 4GB 제한                           │
│                                                                     │
│  // 2. cgroup 메모리 제한                                                │
│  $ echo 2G > /sys/fs/cgroup/memory/app/memory.limit_in_bytes        │
│                                                                     │
│  // 3. 스왑 활성화                                                       │
│  $ swapon -a                                                          │
│                                                                     │
│  // 4. 메모리 모니터링                                                    │
│  $ watch -n 1 'free -h'                                              │
│  $ cat /proc/meminfo                                                  │
│                                                                     │
│  【프로그래밍에서 OOM 처리】                                              │
│  ──────────────────                                                  │
│  #include <stdlib.h>                                                 │
│                                                                     │
│  // malloc 실패 처리                                                    │
│  void *ptr = malloc(huge_size);                                      │
│  if (ptr == NULL) {                                                   │
│      // OOM 상황                                                      │
│      fprintf(stderr, "Memory allocation failed\n");                  │
│      exit(1);                                                         │
│  }                                                                   │
│                                                                     │
│  // 또는 mmap 사용                                                      │
│  void *ptr = mmap(NULL, size, PROT_READ|PROT_WRITE,                 │
│                   MAP_PRIVATE|MAP_ANONYMOUS, -1, 0);                 │
│  if (ptr == MAP_FAILED) {                                             │
│      // 처리                                                           │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 메모리 고갈 시 프로세스 종료
• 기준: oom_score (메모리 사용량)
• 조정: oom_score_adj (-1000 ~ +1000)
• 보호: -1000으로 절대 종료 안 됨
• 로그: dmesg, journalctl
• 설정: panic_on_oom, overcommit
• 방지: 메모리 제한, 모니터링
```
