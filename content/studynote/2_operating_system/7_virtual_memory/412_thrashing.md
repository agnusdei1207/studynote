# 412. 역매핑 (Reverse Mapping)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 페이지에서 가상 페이지로의 역방향 매핑
> 2. **가치**: 페이지 공유 관리와 교체 지원
> 3. **융합**: 페이지 교체, 페이지 공유와 연관

---

## Ⅰ. 개요

### 개념 정의

역매핑(Reverse Mapping)은 **물리 프레임에서 이를 참조하는 가상 페이지로의 역방향 매핑**이다. 어떤 프로세스의 어떤 가상 페이지가 해당 프레임을 참조하는지 추적한다.

### 💡 비유: 발신자 추적
역매핑은 **발신자 추적**과 같다. 전화번호(물리 주소)만 알면 누가(프로세스) 이 번호를 쓰는지 알 수 있다.

### 역매핑 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                역매핑 구조                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정방향 매핑 vs 역매핑】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  정방향 매핑 (Forward Mapping):                                         │ │
│  │  가상 주소 → 페이지 테이블 → 물리 프레임                                  │ │
│  │  "내 물건이 어디 있나?"                                                  │ │
│  │                                                             │ │
│  │  역매핑 (Reverse Mapping):                                              │ │
│  │  물리 프레임 → 역매핑 테이블 → 가상 페이지                                 │ │
│  │  "이 물건은 누구의 것인가?"                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【역매핑 구조】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  물리 프레임 테이블 (Page Frame Table):                                  │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │ 프레임 │ 프로세스 │ 가상 페이지 │ 상태 │                        │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │   0   │   P1     │    VP5      │ 사용중│                        │  │ │
│  │  │   1   │   P2     │    VP3      │ 사용중│                        │  │ │
│  │  │   2   │   P1     │    VP2      │ 사용중│                        │  │ │
│  │  │   3   │    -     │     -       │  비어있음  │                    │  │ │
│  │  │  ...  │   ...    │    ...      │ ...  │                        │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  │  공유 페이지의 경우:                                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │ 프레임 5는 P1의 VP3와 P2의 VP7이 공유                              │  │ │
│  │  │                                                         │  │ │
│  │  │   Frame 5 ─────┬──► P1, VP3                               │  │ │
│  │  │                └──► P2, VP7                               │  │ │
│  │  │                                                         │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                역매핑 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【활용 사례】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 페이지 교체:                                                        │ │
│  │     • 희생자 프레임 선택 시                                              │ │
│  │     • 해당 프레임을 참조하는 PTE 찾기                                     │ │
│  │     • PTE 무효화                                                        │ │
│  │                                                             │ │
│  │  2. 페이지 공유 관리:                                                    │ │
│  │     • 공유 페이지의 모든 참조자 추적                                      │ │
│  │     • 참조 카운트 관리                                                   │ │
│  │     • 마지막 참조자 해제 시 페이지 해제                                    │ │
│  │                                                             │ │
│  │  3. Copy-on-Write:                                                     │ │
│  │     • COW 발생 시 공유자 모두에게 알림                                   │ │
│  │     • 페이지 테이블 갱신                                                 │ │
│  │                                                             │ │
│  │  4. 메모리 압축:                                                         │ │
│  │     • 페이지 이주 시 참조자 갱신                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux의 struct page】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct page {                                                         │ │
│  │      unsigned long flags;      // 상태 플래그                            │ │
│  │      struct list_head lru;     // LRU 리스트                             │ │
│  │      struct address_space *mapping;  // 파일 매핑                         │ │
│  │      pgoff_t index;            // 파일 내 오프셋                          │ │
│  │      atomic_t _mapcount;       // 매핑 수                                │ │
│  │      void *virtual;            // 커널 가상 주소                          │ │
│  │  };                                                                   │ │
│  │                                                             │ │
│  │  _mapcount: 이 페이지를 매핑하는 PTE 수                                  │ │
│  │  -1: 매핑 없음                                                          │ │
│  │  0: 1개 매핑                                                            │ │
│  │  >0: 여러 매핑 (공유)                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 페이지 프레임 정보】                                             │
│  ──────────────────                                                  │
│  // /proc/kpagecount: 각 페이지의 매핑 수                                │
│  $ cat /proc/kpagecount | head -10                                  │
│  // 각 8바이트: 매핑 카운트                                               │
│                                                                     │
│  // /proc/kpageflags: 페이지 플래그                                      │
│  $ cat /proc/kpageflags | head -10                                  │
│  // 비트맵: 동적, 이동 가능, etc.                                          │
│                                                                     │
│  // page-types 도구                                                     │
│  $ page-types -p <pid>                                                │
│  // 페이지 타입별 통계                                                     │
│                                                                     │
│  【역매핑을 통한 페이지 교체】                                             │
│  ──────────────────                                                  │
│  // 커널 코드 (개념적)                                                    │
│  void try_to_unmap(struct page *page) {                              │
│      struct rmap_walk_control rwc = {                                │
│          .invalid_vma = check_vma,                                   │
│          .pfn = page_to_pfn(page),                                   │
│      };                                                               │
│      rmap_walk(page, &rwc);                                          │
│  }                                                                  │
│  // page를 참조하는 모든 PTE를 무효화                                      │
│                                                                     │
│  【공유 메모리 추적】                                                      │
│  ──────────────────                                                  │
│  // 공유 메모리 페이지 확인                                                │
│  $ cat /proc/<pid>/smaps | grep -E "^Size|^Rss|^Pss|Shared"        │
│  Size:               1024 kB                                         │
│  Rss:                 512 kB   ← 실제 물리 메모리                        │
│  Pss:                 256 kB   ← 프로포셔널 공유                          │
│  Shared_Clean:        256 kB                                         │
│  Shared_Dirty:          0 kB                                         │
│                                                                     │
│  // Pss가 Shared_Clean보다 작으면 여러 프로세스가 공유                       │
│                                                                     │
│  【역매핑 오버헤드】                                                        │
│  ──────────────────                                                  │
│  // 각 물리 페이지마다 역매핑 정보 유지                                      │
│  // 16GB 메모리, 4KB 페이지                                              │
│  // 페이지 수 = 16GB / 4KB = 4M개                                        │
│  // struct page 크기 ≈ 64바이트                                          │
│  // 총 오버헤드 = 4M × 64B = 256MB                                        │
│                                                                     │
│  // 메모리 오버헤드가 크지만 필요함                                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
