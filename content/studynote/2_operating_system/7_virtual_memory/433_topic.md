# 433. Zero Fill (0으로 채우기)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이지를 0으로 초기화
> 2. **가치**: 보안, 초기화 보장
> 3. **융합**: 요구 페이징, 페이지 폴트와 연관

---

## Ⅰ. 개요

### 개념 정의

Zero Fill은 **새로 할당된 페이지를 0으로 채워 초기화하는 기법**이다. 이전 데이터가 남아있지 않도록 보장한다.

### 💡 비유: 새 공책
Zero Fill은 **새 공책**과 같다. 이전 학생의 글씨가 안 보이게 깨끗이 지운다.

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C 언어에서의 Zero Fill】                                            │
│  ──────────────────                                                  │
│  // malloc - zero fill 안 됨 (쓰레기 값)                               │
│  int *ptr1 = malloc(100 * sizeof(int));                              │
│  // ptr1[0]은 정의되지 않음                                             │
│                                                                     │
│  // calloc - zero fill 됨                                            │
│  int *ptr2 = calloc(100, sizeof(int));                               │
│  // ptr2[0] == 0 보장                                                  │
│                                                                     │
│  // malloc + memset                                                  │
│  int *ptr3 = malloc(100 * sizeof(int));                              │
│  memset(ptr3, 0, 100 * sizeof(int));                                 │
│                                                                     │
│  // mmap 익명 매핑 - zero fill 됨                                      │
│  void *ptr4 = mmap(NULL, 4096, PROT_READ|PROT_WRITE,                 │
│                    MAP_PRIVATE|MAP_ANONYMOUS, -1, 0);                 │
│  // ptr4[0] == 0 보장                                                  │
│                                                                     │
│  【Zero Page 확인】                                                    │
│  ──────────────────                                                  │
│  // zero page 물리 주소 (보통 0)                                        │
│  $ cat /proc/1/maps | grep "\[ anon zero \]"                        │
│                                                                     │
│  // pagemap으로 확인                                                    │
│  // zero-filled 페이지는 같은 프레임 번호를 가짐                          │
│                                                                     │
│  【BSS 영역 확인】                                                      │
│  ──────────────────                                                  │
│  // C 프로그램                                                           │
│  int global_uninit;      // BSS (zero fill)                          │
│  int global_init = 42;   // Data (초기화됨)                           │
│                                                                     │
│  $ size a.out                                                        │
│  text    data     bss     dec     hex   filename                    │
│  1234     456     100    1790     6fe   a.out                        │
│  // bss 크기 확인                                                        │
│                                                                     │
│  // objdump로 세그먼트 확인                                              │
│  $ objdump -h a.out | grep -E "bss|data"                            │
│                                                                     │
│  【성능 고려사항】                                                        │
│  ──────────────────                                                  │
│  // 큰 배열 할당 시                                                      │
│  // 안 좋음: 즉시 초기화                                                  │
│  int big[10000000] = {0};  // 명시적 zeroing                          │
│                                                                     │
│  // 좋음: 필요한 부분만 사용                                               │
│  int *big = malloc(10000000 * sizeof(int));                          │
│  // 실제 사용 시 zero fill (demand)                                     │
│  for (int i = 0; i < 1000; i++)  // 일부만 사용                        │
│      big[i] = i;                                                      │
│                                                                     │
│  // calloc이 더 효율적일 수 있음                                          │
│  int *big2 = calloc(10000000, sizeof(int));                          │
│  // 커널이 최적화된 zero fill 수행                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
