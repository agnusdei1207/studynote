# 403. 페이지 공유 (Page Sharing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 프로세스가 같은 물리 페이지 공유
> 2. **가치**: 메모리 절약과 IPC 구현
> 3. **융합**: 가상 메모리, 공유 메모리와 연관

---

## Ⅰ. 개요

### 개념 정의

페이지 공유(Page Sharing)는 **여러 프로세스가 동일한 물리 페이지를 공유하여 메모리를 절약하는 기법**이다. 각 프로세스의 페이지 테이블이 같은 프레임을 가리킨다.

### 💡 비유: 도서관 책
페이지 공유는 **도서관 책**과 같다. 여러 사람이 같은 책을 볼 수 있다. 책은 한 권이지만 여러 사람이 읽는다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 공유 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【코드 공유】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  라이브러리 공유:                                                       │ │
│  │  • glibc, libpthread 등은 여러 프로세스가 공유                          │ │
│  │  • 메모리 크게 절약                                                    │ │
│  │  • 페이지 보호: 읽기 + 실행                                           │ │
│  │                                                             │ │
│  │  예: 100개의 프로세스가 각각 2MB 라이브러리 사용                         │ │
│  │      공유 없음: 200MB 필요                                            │ │
│  │      공유 있음: 2MB 필요 (99% 절약!)                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【공유 메모리 IPC】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  POSIX 공유 메모리:                                                     │ │
│  │  • shm_open(): 공유 메모리 객체 생성                                   │ │
│  │  • mmap(): 프로세스 주소 공간에 매핑                                    │ │
│  │  • 매우 빠른 IPC                                                       │ │
│  │                                                             │ │
│  │  주의사항:                                                               │ │
│  │  • 동기화 필요 (세마포어, 뮤텍스)                                        │ │
│  │  • 동시 쓰기 문제                                                       │ │
│  │  • 일관성 유지                                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Copy-on-Write와 공유】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  fork() 직후:                                                           │ │
│  │  • 부모와 자식이 모든 페이지 공유                                       │ │
│  │  • 쓰기 보호 설정                                                       │ │
│  │                                                             │ │
│  │  쓰기 발생 시:                                                           │ │
│  │  • 페이지 복사                                                          │ │
│  │  • 각자 별도 페이지 소유                                                 │ │
│  │  • 쓰지 않은 페이지는 계속 공유                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【POSIX 공유 메모리】                                                     │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│  #include <fcntl.h>                                                   │
│  #include <unistd.h>                                                 │
│                                                                     │
│  // 공유 메모리 생성                                                      │
│  int fd = shm_open("/myshm", O_CREAT | O_RDWR, 0666);               │
│  ftruncate(fd, 4096);  // 크기 설정                                   │
│                                                                     │
│  // 매핑                                                                │
│  void* ptr = mmap(NULL, 4096, PROT_READ | PROT_WRITE,               │
│                      MAP_SHARED, fd, 0);                                │
│                                                                     │
│  // 사용                                                                │
│  sprintf(ptr, "Hello from process %d", getpid());                  │
│                                                                     │
│  // 해제                                                                │
│  munmap(ptr, 4096);                                                 │
│  shm_unlink("/myshm");                                              │
│                                                                     │
│  【System V 공유 메모리】                                                 │
│  ──────────────────                                                  │
│  #include <sys/shm.h>                                               │
│  #include <sys/ipc.h>                                               │
│                                                                     │
│  // 공유 메모리 생성                                                      │
│  key_t key = ftok("/tmp/shmfile", 'R');                                   │
│  int shmid = shmget(key, 4096, IPC_CREAT | 0666);              │
│                                                                     │
│  // 첨부                                                                │
│  char* ptr = shmat(shmid, NULL, 0);                                   │
│                                                                     │
│  // 사용                                                                │
│  strcpy(ptr, "Shared data");                                           │
│                                                                     │
│  // 분리                                                                │
│  shmdt(shmid);                                                        │
│  // 삭제                                                                │
│  shmctl(shmid, IPC_RMID, NULL);                                      │
│                                                                     │
│  【메모리 공유 확인】                                                      │
│  ──────────────────                                                  │
│  // 공유 메모리 세그먼트 확인                                              │
│  $ ipcs -m                                                            │
│  ------ Shared Memory Segments --------                              │
│  key        shmid      owner  perms  bytes  nattch status             │
│  0x00012345 12345     pf    600   4096  2      dest                 │
│                                                                     │
│  // 프로세스별 메모리 맵 확인                                            │
│  $ pmap -x <pid> | grep -E "s|r|w|x"                                    │
│  // 's' 플래그가 있으면 공유                                               │
│                                                                     │
│  【라이브러리 공유】                                                        │
│  ──────────────────                                                  │
│  // 공유 라이브러리 확인                                                   │
│  $ ldd /bin/ls | grep "shared"                                         │
│  linux-vdso.so.1 => libc.so.6                                         │
│  /lib64/libc.so.6 (0x00007f...)                                        │
│                                                                     │
│  // 메모리 맵에서 공유 라이브러리                                           │
│  $ cat /proc/<pid>/maps | grep ".so"                                   │
│  7f1234567000-7f1234568000 r-xp 00000000 08:02 1000 /lib/libc.so.6    │
│  //                    ^^^                                              │
│  //              읽기+실행, 공유(private 아님)                              │
│                                                                     │
│  【Java 공유 메모리】                                                       │
│  ──────────────────                                                  │
│  import java.nio.MappedByteBuffer;                                  │
│  import java.nio.channels.FileChannel;                              │
│  import java.io.RandomAccessFile;                                   │
│                                                                     │
│  RandomAccessFile file = new RandomAccessFile("shared.dat", "rw"); │
│  FileChannel channel = file.getChannel();                           │
│  MappedByteBuffer buffer = channel.map(                             │
│      FileChannel.MapMode.READ_WRITE, 0, 1024);                      │
│                                                                     │
│  buffer.putInt(12345);  // 다른 프로세스에서 읽기 가능                      │
│  buffer.force();  // 디스크에 동기화                                       │
│                                                                     │
│  【동기화 예시】                                                            │
│  ──────────────────                                                  │
│  // 공유 메모리에서 동기화                                                  │
│  sem_t* sem = sem_open("/mysem", O_CREAT, 0666, 1);                 │
│                                                                     │
│  // 쓰기 전 잠금                                                          │
│  sem_wait(sem);                                                     │
│  // 공유 메모리에 쓰기                                                    │
│  memcpy(ptr, data, size);                                           │
│  sem_post(sem);  // 잠금 해제                                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 여러 프로세스가 같은 물리 페이지 공유
• 종류: 코드 공유 (읽기 전용), 데이터 공유 (IPC)
• 장점: 메모리 절약, 빠른 IPC
• 구현: POSIX shm, System V shm, mmap
• 주의: 동기화 필요
• 활용: 라이브러리, 공유 메모리, COW
```
