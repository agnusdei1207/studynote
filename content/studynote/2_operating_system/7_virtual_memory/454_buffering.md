# 454. SSTF 스케줄링 (Shortest Seek Time First)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가장 가까운 요청 우선 처리
> 2. **가치**: 탐색 시간 최소화
> 3. **융합**: 디스크 스케줄링, 기아 현상과 연관

---

## Ⅰ. 개요

### 개념 정의

SSTF 스케줄링(Shortest Seek Time First)은 **현재 헤드 위치에서 가장 가까운 트랙의 요청을 먼저 처리하는 알고리즘**이다.

### 💡 비유: 가까운 배달부터
SSTF는 **배달 기사**와 같다. 현재 위치에서 가장 가까운 배달지부터 방문한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                SSTF 상세                                               │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 성능 향상:                                                      │ │
│  │     • 평균 탐색 시간 최소화                                            │ │
│  │     • FCFS보다 처리량 증가                                             │ │
│  │     • 총 헤드 이동 감소                                                │ │
│  │                                                             │ │
│  │  2. 효율적 자원 사용:                                                 │ │
│  │     • 디스크 기계적 움직임 최소화                                        │ │
│  │     • 전력 소모 감소                                                   │ │
│  │     • 디스크 수명 연장                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 기아 현상 (Starvation):                                        │ │
│  │     • 멀리 있는 요청은 계속 밀림                                        │ │
│  │     • 새 요청이 계속 가까이 오면 무한 대기                                │ │
│  │     • 외곽 트랙 요청 불리                                              │ │
│  │                                                             │ │
│  │  2. 오버헤드:                                                        │ │
│  │     • 매번 최소 거리 계산 필요                                          │ │
│  │     • O(n) 복잡도 (n = 요청 수)                                        │ │
│  │     • 요청이 많으면 계산 부담                                            │ │
│  │                                                             │ │
│  │  3. 불공정성:                                                        │ │
│  │     • 요청 순서 무시                                                   │ │
│  │     • 위치에 따른 편향                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【기아 현상 예시】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  현재 위치: 50                                                   │ │
│  │  요청 큐: 180 (먼 곳)                                             │ │
│  │  새 요청: 55, 52, 48, 60, 45, ... (가까운 곳 계속 도착)               │ │
│  │                                                             │ │
│  │  결과: 180 요청은 계속 밀림 → 기아 현상                                  │ │
│  │                                                             │ │
│  │  해결책: 에이징 (Aging) 적용                                          │ │
│  │  • 대기 시간에 따라 우선순위 조정                                         │ │
│  │  • 오래된 요청의 가중치 증가                                              │ │

│  │                                                             │ │
│  │  지표           FCFS         SSTF                                   │ │

│  │  평균 탐색       93.8         29.5                                   │ │
│  │  총 이동         469          236                                    │ │
│  │  공정성          높음         낮음                                    │ │
│  │  기아 현상       없음         있음                                    │ │

│  │                                                             │ │
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SSTF 구현 (의사코드)】                                              │
│  ──────────────────                                                  │
│  int findClosestRequest(Queue requests, int currentHead) {          │
│      int minDistance = MAX_INT;                                      │
│      int closestIndex = -1;                                          │
│                                                                     │
│      for (int i = 0; i < requests.size(); i++) {                    │
│          int distance = abs(requests[i].track - currentHead);       │
│          if (distance < minDistance) {                               │
│              minDistance = distance;                                 │
│              closestIndex = i;                                       │
│          }                                                            │
│      }                                                                │
│      return closestIndex;                                            │
│  }                                                                   │
│                                                                     │
│  void sstf_schedule(Queue requests, int head) {                     │
│      while (!requests.empty()) {                                     │
│          int idx = findClosestRequest(requests, head);              │
│          Request req = requests.remove(idx);                         │
│          processRequest(req);                                        │
│          head = req.track;                                           │
│      }                                                                │
│  }                                                                   │
│                                                                     │
│  【에이징 적용 SSTF】                                                  │
│  ──────────────────                                                  │
│  struct Request {                                                    │
│      int track;                                                      │
│      int waitTime;    // 대기 시간                                      │
│  };                                                                   │
│                                                                     │
│  int findBestRequest(Queue requests, int head) {                    │
│      int bestScore = MAX_INT;                                        │
│      int bestIndex = -1;                                             │
│                                                                     │
│      for (int i = 0; i < requests.size(); i++) {                    │
│          int distance = abs(requests[i].track - head);              │
│          int score = distance - requests[i].waitTime * AGING_FACTOR;│
│          if (score < bestScore) {                                    │
│              bestScore = score;                                      │
│              bestIndex = i;                                          │
│          }                                                            │
│      }                                                                │
│      return bestIndex;                                               │
│  }                                                                   │
│                                                                     │
│  【성능 측정】                                                          │
│  ──────────────────                                                  │
│  void measureSSTF(int requests[], int n, int head) {                │
│      int totalSeek = 0;                                              │
│      int currentHead = head;                                         │
│      bool visited[n] = {false};                                      │
│                                                                     │
│      for (int i = 0; i < n; i++) {                                  │
│          // 가장 가까운 미방문 요청 찾기                                    │
│          int minDist = MAX_INT, idx = -1;                            │
│          for (int j = 0; j < n; j++) {                              │
│              if (!visited[j]) {                                       │
│                  int dist = abs(requests[j] - currentHead);          │
│                  if (dist < minDist) {                                │
│                      minDist = dist;                                  │
│                      idx = j;                                         │
│                  }                                                    │
│              }                                                         │
│          }                                                             │
│          visited[idx] = true;                                         │
│          totalSeek += minDist;                                        │
│          currentHead = requests[idx];                                 │
│      }                                                                │
│      printf("Total seek: %d, Average: %.2f\n",                       │
│             totalSeek, (float)totalSeek / n);                         │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 가장 가까운 요청 우선 처리
• 목표: 탐색 시간 최소화
• 장점: 성능 향상, 자원 효율
• 단점: 기아 현상, 오버헤드, 불공정
• 비교: FCFS보다 2~3배 빠름
• 개선: 에이징으로 기아 완화
• 복잡도: O(n²) 전체, O(n) 선택
```
