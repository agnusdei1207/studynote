# 401. 페이지 폴트 빈도 (Page Fault Frequency)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이지 폴트 발생률로 프레임 할당 조절
> 2. **가치**: 스래싱 방지와 메모리 효율성 균형
> 3. **융합**: 스래싱, 페이지 교체와 연관

---

## Ⅰ. 개요

### 개념 정의

페이지 폴트 빈도(Page Fault Frequency, PFF)는 **페이지 폴트 발생률을 기준으로 프로세스의 프레임 할당을 동적으로 조절하는 기법**이다. 스래싱 방지를 위해 사용된다.

### 💡 비유: 온도 조절
페이지 폴트 빈도는 **에어컨 온도 조절**과 같다. 너무 더우면 온도를 낮추고, 너무 추우면 온도를 높인다. 적정 온도를 유지한다.

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【PFF 구현 (개념적)】                                                    │
│  ──────────────────                                                  │
│  class PageFaultFrequency {                                        │
│      int upperBound = 100;   // 초당 상한                               │ │
│      int lowerBound = 10;    // 초당 하한                                │ │
│      long windowSize = 1000;  // 측정 윈도우 (ms)                          │ │
│                                                                     │
│      Map<Integer, List<Long>> faultTimes = new HashMap<>();          │
│                                                                     │
│      void onPageFault(int processId) {                              │
│          long now = System.currentTimeMillis();                       │
│          faultTimes.computeIfAbsent(processId, k -> new ArrayList<>())│
│                         .add(now);                                    │
│                                                                     │
│          // 폴트율 계산                                                    │
│          int faults = countRecentFaults(processId, now - windowSize);│
│          double rate = faults * 1000.0 / windowSize;                 │
│                                                                     │
│          if (rate > upperBound) {                                    │
│              // 프레임 추가 할당                                          │ │
│              allocateMoreFrames(processId);                           │
│          } else if (rate < lowerBound) {                             │
│              // 프레임 회수                                               │ │
│              reclaimFrames(processId);                                │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Linux 페이지 폴트 처리】                                                │
│  ──────────────────                                                  │
│  // 커널은 PFF와 유사한 메커니즘 사용                                      │
│  // /proc/vmstat에서 페이지 폴트 통계 확인                                 │
│  $ cat /proc/vmstat | grep -E "pgfault|pgmajfault|pglazyfreed"      │
│  pgfault 1234567                                                    │
│  pgmajfault 1234                                                    │
│  pglazyfreed 5678                                                   │
│                                                                     │
│  // 프로세스별 폴트 확인                                                   │
│  $ cat /proc/<pid>/stat | awk '{print $10, $12}'                     │
│  // $10: minflt, $12: majflt                                          │
│                                                                     │
│  【메모리 부족 처리】                                                      │
│  ──────────────────                                                  │
│  // Linux 메모리 관리 파라미터                                              │
│  $ cat /proc/sys/vm/page-cluster                                    │
│  // 한 번에 읽을 페이지 수 (폴트 처리 최적화)                                │
│                                                                     │
│  $ cat /proc/sys/vm/stat_interval                                   │
│  // VM 통계 갱신 간격                                                    │
│                                                                     │
│  【성능 모니터링】                                                          │
│  ──────────────────                                                  │
│  // 실시간 페이지 폴트 모니터링                                             │
│  $ perf stat -e page-faults,minor-faults,major-faults ./program     │
│                                                                     │
│  // 시스템 전체 폴트율                                                      │
│  $ sar -B 1 5                                                       │
│  pgpgin/s pgpgout/s fault/s  majflt/s  pgfree/s pgscank/s            │
│      10.00     5.00  1234.56      0.00   1234.56      0.00           │
│                                                                     │
│  【애플리케이션 레벨 PFF】                                                   │
│  ──────────────────                                                  │
│  // 메모리 사용량과 폴트 관계 모니터링                                       │
│  class ApplicationMemoryMonitor {                                   │
│      MemoryMXBean memoryBean = ManagementFactory.getMemoryMXBean(); │
│                                                                     │
│      void monitor() {                                               │
│          MemoryUsage heap = memoryBean.getHeapMemoryUsage();        │
│          long used = heap.getUsed();                                │
│          long max = heap.getMax();                                  │
│          double usage = (double) used / max;                        │
│                                                                     │
│          // 메모리 사용이 높으면 GC 유발 전 조치                             │ │
│          if (usage > 0.8) {                                          │
│              // 캐시 축소 등                                            │ │
│              shrinkCaches();                                         │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 페이지 폴트 발생률로 프레임 할당 조절
• 구조: 상한, 하한, 적정 구간
• 동작: 폴트율 > 상한 → 프레임 추가
• 동작: 폴트율 < 하한 → 프레임 회수
• 장점: 스래싱 방지, 메모리 효율
• 비교: 작업 집합보다 구현 간단
```
