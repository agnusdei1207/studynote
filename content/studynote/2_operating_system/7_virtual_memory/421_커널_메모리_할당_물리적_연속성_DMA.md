## 💡 Insight
- 커널은 하드웨어 제어 및 입출력 효율성을 위해 가상 주소가 아닌 물리적으로 연속된 메모리 공간을 필요로 한다.
- 사용자 프로세스와 달리 페이징에 의한 분산 배치가 불가능한 DMA(Direct Memory Access) 장치를 지원하기 위한 필수 요건이다.
- 이를 관리하기 위해 SLAB/SLUB 할당자와 버디 시스템(Buddy System)을 활용하여 단편화를 최소화하며 연속성을 보장한다.

---

## Ⅰ. 커널 메모리 할당과 물리적 연속성 개요
### 1. 커널 메모리의 특수성
- **HW 접근성**: 커널은 CPU와 하드웨어 사이의 중재자로서, 장치 컨트롤러가 직접 메모리에 접근하는 DMA(Direct Memory Access)를 관리함.
- **성능 최적화**: 페이지 테이블 조회를 생략하거나 TLB(Translation Lookaside Buffer) 미스를 줄이기 위해 물리적 주소와 가상 주소가 일정한 오프셋으로 매핑된 영역을 사용함.

### 2. 물리적 연속성(Physical Contiguity)의 정의
- 가상 주소 공간에서 연속적일 뿐만 아니라, 실제 RAM 상의 물리 프레임(Physical Frame) 번호가 순차적으로 나열된 상태를 의미함.

📢 섹션 요약 비유: 사용자 메모리가 "흩어져 있어도 주소만 맞으면 배달되는 택배"라면, 커널 메모리는 "중장비가 한 번에 지나가야 하는 일직선의 전용 도로"와 같습니다.

---

## Ⅱ. 물리적 연속성이 필요한 이유와 매커니즘
### 1. DMA (Direct Memory Access) 지원
- 많은 하드웨어 장치는 페이징 유닛(MMU)을 거치지 않고 직접 물리 주소로 메모리를 읽고 씀.
- 장치가 "0x1000부터 1MB를 읽어라"라고 명령받았을 때, 메모리가 물리적으로 쪼개져 있으면 장치는 이를 처리할 수 없음.

### 2. 주소 변환 오버헤드 제거
- 커널의 핵심 코드와 데이터 구조는 빈번하게 참조되므로, 복잡한 다단계 페이지 테이블을 거치는 대신 단순 계산으로 물리 주소를 얻는 직접 매핑(Direct Mapping) 방식을 선호함.

### 3. 할당 매커니즘 (Buddy System)
- 커널은 2^n 크기의 블록 단위로 메모리를 관리하는 버디 시스템을 통해 큰 규모의 물리적 연속 공간을 할당함.

```text
[ Buddy System Allocation Concept ]
+-------------------------------------------------------+
|          Total Free Memory Block (Size: 2^4 = 16)     |
+-----------+-----------+-----------+-----------+-------+
|     8     |     4     |     2     |     1     |   1   |
+-----------+-----------+-----------+-----------+-------+
      ↓ (Request 4 units)
+-----------+-----------+-----------+-----------+-------+
|     8     | [Allocated]|     2     |     1     |   1   |
+-----------+-----------+-----------+-----------+-------+
(Adjacent free blocks can be merged back into 2^n sizes)
```

📢 섹션 요약 비유: DMA 장치는 "지도(MMU)를 볼 줄 모르는 기사"와 같아서, 길이 끊기지 않고 하나로 이어진 "직진 구간(물리적 연속 공간)"이 반드시 필요합니다.

---

## Ⅲ. 커널 메모리 할당자: SLAB 및 SLUB
### 1. SLAB (Slab Allocator)
- 물리적으로 연속된 페이지를 할당받아, 이를 동일한 크기의 객체(Object)들로 미리 쪼개놓고 재사용하는 방식.
- **커널 오브젝트 캐싱**: 프로세스 제어 블록(PCB), 파일 객체 등 자주 생성/삭제되는 데이터를 위한 전용 풀 제공.

### 2. 내부 단편화 해결
- 작은 크기의 메모리 요청에 대해 페이지 단위가 아닌 객체 단위로 할당하여 낭비를 줄임.

📢 섹션 요약 비유: SLAB은 "똑같은 규격의 방이 나열된 고시원"과 같아서, 거주자(객체)가 바뀌어도 방의 구조(메모리 연속성)를 유지하며 빠르게 입주자를 받습니다.

---

## Ⅳ. 연속 할당의 한계와 대안: vmalloc
### 1. 물리적 연속성의 한계
- 시스템이 오래 가동될수록 메모리 파편화(Fragmentation)가 발생하여, 큰 규모의 물리적 연속 공간을 찾기 어려워짐.

### 2. vmalloc (Virtual Memory Allocation)
- 물리적으로는 떨어져 있어도 **가상 주소 공간에서만 연속**되도록 만드는 할당 방식.
- 하드웨어 접근이 필요 없는 커널 모듈 로딩이나 큰 버퍼 확보 시 사용됨.

📢 섹션 요약 비유: `vmalloc`은 "여러 건물에 흩어진 사무실을 하나의 유선 전화망으로 묶은 가상 오피스"와 같아, 겉보기엔 하나지만 실제 발걸음은 여기저기 옮겨야 합니다.

---

## Ⅴ. 커널 메모리 관리의 성능 임팩트
### 1. TLB 효율성
- 물리적으로 연속된 거대 페이지를 사용하면 TLB 엔트리 하나가 커버하는 영역이 넓어져 캐시 적중률이 상승함.
### 2. 할당 지연 시간
- 버디 시스템과 SLAB은 가상 메모리의 요구 페이징(Demand Paging)보다 훨씬 빠르게 즉각적으로 물리 메모리를 할당함.

📢 섹션 요약 비유: 커널 메모리 관리는 "F1 경주용 트랙을 관리하는 것"과 같아서, 노면의 연속성과 정비 효율성이 전체 시스템의 최고 속도를 결정합니다.
