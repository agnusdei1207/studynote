# 408. Huge Page (거대 페이지)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 표준보다 큰 페이지 크기 (2MB, 1GB)
> 2. **가치**: TLB 효율 향상과 대용량 처리 최적화
> 3. **융합**: 페이지 크기, TLB와 연관

---

## Ⅰ. 개요

### 개념 정의

Huge Page는 **표준 페이지(4KB)보다 큰 페이지 크기(2MB, 1GB)를 사용하는 기법**이다. TLB 커버리지를 늘려 성능을 향상시킨다.

### 💡 비유: 큰 상자
Huge Page는 **큰 상자**와 같다. 작은 상자 여러 개 대신 큰 상자 하나를 쓴다. 더 많은 물건을 한 번에 담는다.

### Huge Page 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                Huge Page 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 크기 비교】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  4KB Standard Page:                                                  │ │
│  │  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐               │ │
│  │  │ 4K│ 4K│ 4K│ 4K│ 4K│ 4K│ 4K│ 4K│...│ 4K│ 4K│ 4K│               │ │
│  │  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘               │ │
│  │  (512개 필요 = 2MB)                                                   │ │
│  │                                                             │ │
│  │  2MB Huge Page:                                                        │ │
│  │  ┌─────────────────────────────────────────────────────────────┐ │ │
│  │  │                         2MB                                    │ │ │
│  │  └─────────────────────────────────────────────────────────────┘ │ │
│  │  (1개로 동일 커버)                                                    │ │
│  │                                                             │ │
│  │  1GB Giant Page:                                                        │ │
│  │  ┌─────────────────────────────────────────────────────────────────┐│ │
│  │  │                              1GB                                 ││ │
│  │  └─────────────────────────────────────────────────────────────────┘│ │
│  │  (매우 큰 메모리 한 번에)                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 커버리지 비교】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  64개 TLB 항목일 때:                                                   │ │
│  │                                                             │ │
│  │  4KB:  64 × 4KB   = 256KB  커버                                       │ │
│  │  2MB:  64 × 2MB   = 128MB  커버  (512배 증가!)                        │ │
│  │  1GB:  64 × 1GB   = 64GB   커버  (262,144배 증가!)                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                Huge Page 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. TLB 미스 감소:                                                     │ │
│  │     • 더 많은 메모리를 TLB로 커버                                      │ │
│  │     • 페이지 워크 감소                                                 │ │
│  │                                                             │ │
│  │  2. 페이지 테이블 크기 감소:                                            │ │
│  │     • 더 적은 페이지 테이블 항목                                        │ │
│  │     • 메모리 절약                                                      │ │
│  │                                                             │ │
│  │  3. 메모리 할당 오버헤드 감소:                                          │ │
│  │     • 더 적은 페이지 폴트                                              │ │
│  │     • 더 큰 I/O 단위                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 내부 단편화 증가:                                                   │ │
│  │     • 2MB 중 일부만 사용해도 2MB 전체 할당                              │ │
│  │     • 작은 프로세스에 비효율적                                          │ │
│  │                                                             │ │
│  │  2. 사전 할당 필요:                                                     │ │
│  │     • 부팅 시 또는 런타임에 미리 할당                                    │ │
│  │     • 동적 할당 어려움                                                  │ │
│  │                                                             │ │
│  │  3. 메모리 낭비:                                                        │ │
│  │     • 사용하지 않아도 예약됨                                            │ │
│  │     • 다른 용도로 사용 불가                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【종류】                                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Explicit Huge Pages:                                              │ │
│  │     • HugeTLB 파일 시스템                                              │ │
│  │     • 명시적 할당                                                      │ │
│  │     • 관리 복잡                                                        │ │
│  │                                                             │ │
│  │  2. Transparent Huge Pages (THP):                                     │ │
│  │     • 커널이 자동 관리                                                  │ │
│  │     • 애플리케이션 수정 불필요                                          │ │
│  │     • 더 쉬운 사용                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Huge Page 설정】                                                        │
│  ──────────────────                                                  │
│  // Huge Page 수 설정                                                   │
│  $ echo 1000 > /proc/sys/vm/nr_hugepages                             │
│                                                                     │
│  // 확인                                                                │
│  $ cat /proc/meminfo | grep Huge                                    │
│  HugePages_Total:    1000                                            │
│  HugePages_Free:      500                                            │
│  HugePages_Rsvd:      200                                            │
│  Hugepagesize:       2048 kB                                         │
│  Hugetlb:          2048000 kB                                        │
│                                                                     │
│  【Transparent Huge Page】                                              │
│  ──────────────────                                                  │
│  // THP 상태 확인                                                        │
│  $ cat /sys/kernel/mm/transparent_hugepage/enabled                   │
│  always [madvise] never                                               │
│                                                                     │
│  // THP 활성화                                                          │
│  $ echo always > /sys/kernel/mm/transparent_hugepage/enabled         │
│                                                                     │
│  // THP 사용 통계                                                        │
│  $ cat /sys/kernel/mm/transparent_hugepage/stats                     │
│  hugepage_anon    12345                                               │
│  hugepage_pmd     5678                                                │
│                                                                     │
│  【프로그램에서 사용】                                                      │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│                                                                     │
│  // MAP_HUGETLB 사용                                                   │
│  void* addr = mmap(NULL, 2*1024*1024,                               │
│                     PROT_READ | PROT_WRITE,                           │
│                     MAP_PRIVATE | MAP_ANONYMOUS | MAP_HUGETLB,        │
│                     -1, 0);                                          │
│  if (addr == MAP_FAILED) {                                            │
│      // Huge Page 없음, 일반 페이지 사용                                 │ │
│  }                                                                  │
│                                                                     │
│  // HugeTLB 파일 시스템                                                 │
│  int fd = open("/mnt/huge/file", O_CREAT | O_RDWR, 0755);           │
│  void* addr = mmap(NULL, 2*1024*1024,                               │
│                     PROT_READ | PROT_WRITE,                           │
│                     MAP_SHARED, fd, 0);                              │
│                                                                     │
│  【데이터베이스 설정】                                                      │
│  ──────────────────                                                  │
│  // PostgreSQL                                                         │
│  // postgresql.conf                                                    │
│  huge_pages = on                                                      │
│  shared_buffers = 16GB                                                │
│                                                                     │
│  // Huge Page 필요량 계산                                               │
│  $ head -1 /proc/meminfo | awk '{print $2}'                          │
│  // shared_buffers보다 약간 더 필요                                      │
│                                                                     │
│  // MySQL                                                              │
│  [mysqld]                                                             │
│  large-pages                                                          │
│  innodb_buffer_pool_size = 8G                                        │
│                                                                     │
│  // Redis                                                              │
│  $ redis-server --enable-hugepages                                   │
│                                                                     │
│  【성능 측정】                                                              │
│  ──────────────────                                                  │
│  // TLB 미스 측정                                                        │
│  $ perf stat -e dTLB-load-misses,iTLB-load-misses ./app              │
│                                                                     │
│  // 4KB vs 2MB 비교                                                     │
│  // 4KB: dTLB-load-misses: 1,234,567                                  │
│  // 2MB: dTLB-load-misses: 12,345   (99% 감소!)                        │
│                                                                     │
│  // 메모리 대역폭 측정                                                     │
│  $ numactl --pagesize=2M ./memory_benchmark                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 표준보다 큰 페이지 (2MB, 1GB)
• 장점: TLB 미스 감소, 페이지 테이블 축소
• 단점: 내부 단편화, 사전 할당 필요
• 종류: Explicit (HugeTLB), Transparent (THP)
• 활용: DB, VM, 메모리 집약적 앱
• API: MAP_HUGETLB, HugeTLB fs
```
