# 413. 페이지 캐시 (Page Cache)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 데이터를 메모리에 캐싱
> 2. **가치**: 파일 I/O 성능 대폭 향상
> 3. **융합**: 가상 메모리, 파일 시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

페이지 캐시(Page Cache)는 **파일 시스템의 데이터를 메모리에 캐싱하는 커널의 메커니즘**이다. 디스크 I/O를 줄여 성능을 향상시킨다.

### 💡 비유: 도서관 열람실
페이지 캐시는 **도서관 열람실**과 같다. 자주 읽는 책은 열람실에 둔다. 서가까지 갈 필요가 없다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 캐시 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Dirty 페이지】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  캐시에서 수정되었지만 아직 디스크에 안 쓰인 페이지:                      │ │
│  │                                                             │ │
│  │  • Dirty 플래그로 표시                                                  │ │
│  │  • 정기적으로 플러시 (writeback)                                         │ │
│  │  • 시스템 크래시 시 데이터 손실 가능                                      │ │
│  │                                                             │ │
│  │  장점: 쓰기 성능 향상, 쓰기 병합                                          │ │
│  │  단점: 데이터 손실 위험                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Writeback】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Dirty 페이지를 디스크에 쓰는 과정:                                       │ │
│  │                                                             │ │
│  │  • 백그라운드에서 수행                                                   │ │
│  │  • pdflush/kswapd 스레드가 담당                                          │ │
│  │  • 스로틀링으로 과부하 방지                                               │ │
│  │                                                             │ │
│  │  플러시 조건:                                                            │ │
│  │  • dirty 비율 임계값 초과                                                │ │
│  │  • sync() 호출                                                          │ │
│  │  • 메모리 압력                                                           │ │
│  │  • 타임아웃                                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 캐시와 메모리 관리】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  메모리 부족 시:                                                         │ │
│  │  • 페이지 캐시 해제 (clean 페이지)                                       │ │
│  │  • dirty 페이지 플러시 후 해제                                            │ │
│  │  • 활성 페이지보다 캐시 해제 우선                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 캐시 통계】                                                      │
│  ──────────────────                                                  │
│  $ cat /proc/meminfo | grep -E "Cached|Buffers|Dirty|Writeback"     │
│  Cached:          12345678 kB   ← 페이지 캐시                          │
│  Buffers:          1234567 kB   ← 버퍼 캐시                            │
│  Dirty:             123456 kB   ← dirty 페이지                         │
│  Writeback:          12345 kB   ← 쓰기 진행 중                          │
│                                                                     │
│  // 캐시 히트율 확인                                                       │
│  $ sar -r 1 5                                                        │
│  kbmemfree kbmemused  %memused kbbuffers  kbcached                    │
│                                                                     │
│  【Dirty 페이지 설정】                                                     │
│  ──────────────────                                                  │
│  // dirty 비율 임계값                                                      │
│  $ cat /proc/sys/vm/dirty_ratio                                      │
│  20  // 메모리의 20%가 dirty면 전체 플러시                                  │
│                                                                     │
│  $ cat /proc/sys/vm/dirty_background_ratio                           │
│  10  // 10%면 백그라운드 플러시 시작                                       │
│                                                                     │
│  // dirty 타임아웃                                                       │
│  $ cat /proc/sys/vm/dirty_expire_centisecs                          │
│  3000  // 30초 후에 dirty 페이지 플러시                                    │
│                                                                     │
│  【수동 플러시】                                                            │
│  ──────────────────                                                  │
│  // 모든 dirty 페이지 플러시                                               │
│  $ sync                                                              │
│                                                                     │
│  // 페이지 캐시 해제 (주의!)                                               │
│  $ echo 3 > /proc/sys/vm/drop_caches                                │
│  // 1: pagecache 해제                                                   │
│  // 2: dentries/inodes 해제                                             │
│  // 3: 모두 해제                                                          │
│                                                                     │
│  【프로그래밍에서 활용】                                                      │
│  ──────────────────                                                  │
│  #include <unistd.h>                                                 │
│  #include <fcntl.h>                                                   │
│                                                                     │
│  // 파일을 캐시에 프리로드                                                   │
│  int fd = open("largefile.bin", O_RDONLY);                           │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_WILLNEED);                      │
│  // 커널이 백그라운드에서 캐시에 로드                                        │
│                                                                     │
│  // 캐시에서 제거                                                          │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_DONTNEED);                       │
│                                                                     │
│  // 순차 접근 힌트                                                          │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_SEQUENTIAL);                     │
│                                                                     │
│  // 무작위 접근 힌트                                                        │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_RANDOM);                        │
│                                                                     │
│  【캐시 효과 측정】                                                        │
│  ──────────────────                                                  │
│  // 두 번째 읽기가 빠름 (캐시)                                              │
│  $ time cat bigfile.bin > /dev/null                                  │
│  real    0m5.234s   // 디스크에서 읽기                                    │
│                                                                     │
│  $ time cat bigfile.bin > /dev/null                                  │
│  real    0m0.123s   // 캐시에서 읽기 (42배 빠름!)                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 데이터를 메모리에 캐싱
• 장점: 디스크 I/O 감소, 읽기/쓰기 성능 향상
• Dirty: 수정되었지만 아직 디스크에 안 쓰인 페이지
• Writeback: 백그라운드에서 dirty 페이지 플러시
• 설정: dirty_ratio, dirty_background_ratio
• API: posix_fadvise(), sync()
```
