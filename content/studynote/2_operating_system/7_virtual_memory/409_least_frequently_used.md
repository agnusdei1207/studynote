# 409. 페이지 컬러링 (Page Coloring)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시 색상 기반 페이지 할당 기법
> 2. **가치**: 캐시 충돌 감소와 성능 향상
> 3. **융합**: 캐시 메모리, 페이지 할당과 연관

---

## Ⅰ. 개요

### 개념 정의

페이지 컬러링(Page Coloring)은 **물리 페이지를 캐시 색상으로 분류하여 캐시 충돌을 줄이는 기법**이다. 각 페이지에 색상을 부여하고 색상 기반으로 할당한다.

### 💡 비유: 주차 구역
페이지 컬러링은 **주차 구역**과 같다. 같은 색 구역에만 주차하면 혼잡해진다. 여러 색 구역에 나누면 원활하다.

### 페이지 컬러링 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 컬러링 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 색상 개념】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  캐시 크기 = 64KB, 페이지 크기 = 4KB, 4-way set associative        │ │
│  │  • 캐시 세트 수 = 64KB / 4KB / 4 = 4개 세트                         │ │
│  │  • 색상 수 = 4개                                                       │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │            Cache Sets                                    │   │ │
│  │  │  ┌─────┬─────┬─────┬─────┐                              │   │ │
│  │  │  │  0  │  1  │  2  │  3  │  ← Set Index                 │   │ │
│  │  │  ├─────┼─────┼─────┼─────┤                              │   │ │
│  │  │  │Color│Color│Color│Color│                              │   │ │
│  │  │  │  0  │  1  │  2  │  3  │  ← Page Color                │   │ │
│  │  │  └─────┴─────┴─────┴─────┘                              │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【컬러링 없이 할당】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  무작위 할당 시:                                                       │ │
│  │  • 같은 색상 페이지 여러 개 할당 가능                                   │ │
│  │  • 캐시 세트 충돌 증가                                                  │ │
│  │  • 캐시 미스 증가                                                      │ │
│  │                                                             │ │
│  │  프로세스 A: Page0(Color0), Page1(Color0), Page2(Color0)             │ │
│  │  → 모두 같은 캐시 세트 사용 → 충돌!                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【컬러링으로 할당】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  색상 기반 할당 시:                                                     │ │
│  │  • 여러 색상 페이지 균등 할당                                          │ │
│  │  • 캐시 세트 충돌 감소                                                  │ │
│  │  • 캐시 미스 감소                                                      │ │
│  │                                                             │ │
│  │  프로세스 A: Page0(Color0), Page1(Color1), Page2(Color2)             │ │
│  │  → 다른 캐시 세트 사용 → 효율적!                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 컬러링 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【색상 계산】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  페이지 색상 = (물리 주소 / 페이지 크기) % 색상 수                       │ │
│  │                                                             │ │
│  │  예: 물리 주소 0x8000, 페이지 4KB, 색상 4개                            │ │
│  │  색상 = (0x8000 / 0x1000) % 4 = 8 % 4 = 0 (Color 0)                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【할당 정책】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 라운드 로빈:                                                       │ │
│  │     • 색상 0, 1, 2, 3, 0, 1, ... 순환                                  │ │
│  │                                                             │ │
│  │  2. 최소 사용 색상:                                                     │ │
│  │     • 가장 적게 사용된 색상 선택                                        │ │
│  │                                                             │ │
│  │  3. 프로세스별 독립:                                                    │ │
│  │     • 각 프로세스가 모든 색상 사용 가능                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【효과】                                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 캐시 미스 감소: 5-20%                                               │ │
│  │  • 특히 대용량 작업에 효과적                                            │ │
│  │  • 멀티코어 시스템에서 더 중요                                          │ │
│  │  • 구현 오버헤드 작음                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 페이지 컬러링】                                                  │
│  ──────────────────                                                  │
│  // 커널이 자동으로 수행                                                 │
│  // /proc/sys/vm/page_coloring_order로 확인/설정                        │
│  $ cat /proc/sys/vm/page_coloring_order                              │
│                                                                     │
│  // ARM 아키텍처에서 특히 중요                                             │
│  // 일부 ARM SoC는 캐시 색상에 민감                                       │
│                                                                     │
│  【성능 측정】                                                              │
│  ──────────────────                                                  │
│  // 캐시 미스 측정                                                        │
│  $ perf stat -e cache-references,cache-misses ./program             │
│                                                                     │
│  // 컬러링 비교 (개념적)                                                   │
│  // Without: cache-misses: 1,234,567 (15%)                            │
│  // With:    cache-misses: 1,012,345 (12%)  ← 3% 개선                 │
│                                                                     │
│  【수동 구현 (개념적)】                                                    │
│  ──────────────────                                                  │
│  class PageColoringAllocator {                                     │
│      int numColors;  // 색상 수                                        │
│      int[] colorUsage;  // 각 색상 사용량                               │
│                                                                     │
│      int getColor(long physAddr, int pageSize) {                     │
│          return (int)((physAddr / pageSize) % numColors);            │
│      }                                                              │
│                                                                     │
│      long allocatePage(int desiredColor) {                           │
│          // 해당 색상의 프리 리스트에서 할당                               │ │
│          return freeLists[desiredColor].allocate();                   │
│      }                                                              │
│                                                                     │
│      long allocateBalanced() {                                       │
│          // 가장 적게 사용된 색상 선택                                     │ │
│          int minColor = findMinUsedColor();                          │
│          return allocatePage(minColor);                              │
│      }                                                              │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 캐시 색상 기반 페이지 할당
• 목적: 캐시 충돌 감소
• 계산: (물리주소/페이지크기) % 색상수
• 정책: 라운드로빈, 최소사용, 독립
• 효과: 캐시 미스 5-20% 감소
• 중요성: ARM, 멀티코어에서 중요
```
