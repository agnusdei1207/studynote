# 457. LOOK 스케줄링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SCAN에서 끝까지 이동 제거
> 2. **가치**: 불필요한 헤드 이동 감소
> 3. **융합**: SCAN, C-SCAN과 연관

---

## Ⅰ. 개요

### 개념 정의

LOOK 스케줄링은 **SCAN의 변형으로, 헤드가 요청이 있는 곳까지만 이동하고 끝까지 가지 않는 알고리즘**이다.

### 💡 비유: 스마트 엘리베이터
LOOK은 **스마트 엘리베이터**와 같다. 내리는 사람이 없으면 끝 층까지 안 가고 방향을 바꾼다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                LOOK 상세                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 방식】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 현재 방향으로 이동하며 요청 처리                                  │ │
│  │  2. 그 방향에 더 이상 요청 없으면 방향 전환                           │ │
│  │  3. 끝까지 이동하지 않음                                             │ │
│  │  4. 반대 방향으로 이동하며 요청 처리                                  │ │
│  │                                                             │ │
│  │  "LOOK": 헤드가 양쪽을 "look" 하고 요청 있으면만 이동                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【C-LOOK (Circular LOOK)】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  C-SCAN + LOOK = C-LOOK                                            │ │
│  │                                                             │ │
│  │  • 한 방향으로만 서비스                                              │ │
│  │  • 끝 요청까지만 이동                                                │ │
│  │  • 처음으로 점프                                                    │ │
│  │  • 가장 공정하고 효율적                                              │ │
│  │                                                             │ │
│  │  예시:                                                               │ │
│  │  요청: 37, 65, 98, 122, 183 (현재: 53)                              │ │
│  │  LOOK: 53→65→98→122→183→37                                        │ │
│  │  C-LOOK: 53→65→98→122→183→(점프)→37                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【알고리즘 비교】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  알고리즘   방향   끝 이동    기아   공정성   효율성                   │ │
│  │  ───────   ────   ──────    ────   ──────   ─────                   │ │
│  │  SCAN      양방   O         없음   중간     중간                    │ │
│  │  C-SCAN    단방   O         없음   높음     중간                    │ │
│  │  LOOK      양방   X         없음   중간     높음                    │ │
│  │  C-LOOK    단방   X         없음   높음     높음                    │ │
│  │                                                             │ │
│  │  효율성: 불필요한 이동 없음이 높음                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 분석】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  예시: 요청 98, 183, 37, 122, 14, 124, 65, 67 (현재: 53)           │ │
│  │                                                             │ │
│  │  SCAN:                                                             │ │
│  │  이동: 53→65→67→98→122→124→183→199→37→14→0                        │ │
│  │  총 이동: 199 + 199 = 398 (양 끝 포함)                              │ │
│  │                                                             │ │
│  │  LOOK:                                                             │ │
│  │  이동: 53→65→67→98→122→124→183→37→14                              │ │
│  │  총 이동: 130 + 169 = 299 (양 끝 제외)                              │ │
│  │                                                             │ │
│  │  LOOK이 약 25% 더 효율적                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LOOK 구현 (의사코드)】                                             │
│  ──────────────────                                                  │
│  void look_schedule(Queue requests, int head, int direction) {      │
│      sort(requests);                                                 │
│                                                                     │
│      while (!requests.empty()) {                                    │
│          // 현재 방향에서 다음 요청 찾기                                │
│          Request next = findNextInDirection(                        │
│              requests, head, direction);                            │
│                                                                     │
│          if (next == null) {                                        │
│              // 그 방향에 요청 없음 → 방향 전환                          │
│              direction = !direction;                                │
│              next = findNextInDirection(                            │
│                  requests, head, direction);                        │
│          }                                                           │
│                                                                     │
│          processRequest(next);                                      │
│          head = next.track;                                         │
│          removeFromQueue(requests, next);                           │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  Request findNextInDirection(Queue reqs, int head, int dir) {       │
│      Request closest = null;                                        │
│      for (r in reqs) {                                              │
│          if (dir == INCREASING && r.track >= head) {                │
│              if (closest == null || r.track < closest.track) {      │
│                  closest = r;                                        │
│              }                                                       │
│          } else if (dir == DECREASING && r.track <= head) {         │
│              if (closest == null || r.track > closest.track) {      │
│                  closest = r;                                        │
│              }                                                       │
│          }                                                           │
│      }                                                               │
│      return closest;                                                │
│  }                                                                   │
│                                                                     │
│  【C-LOOK 구현】                                                      │
│  ──────────────────                                                  │
│  void clook_schedule(Queue requests, int head) {                    │
│      sort(requests);                                                 │
│                                                                     │
│      while (!requests.empty()) {                                    │
│          // 증가 방향으로만 서비스                                      │
│          Request next = findNextGreater(requests, head);            │
│                                                                     │
│          if (next == null) {                                        │
│              // 없으면 처음으로 점프                                    │
│              next = findMin(requests);                              │
│          }                                                           │
│                                                                     │
│          processRequest(next);                                      │
│          head = next.track;                                         │
│          removeFromQueue(requests, next);                           │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  【실제 사용 예】                                                      │
│  ──────────────────                                                  │
│  // LOOK/C-LOOK는 많은 시스템에서 기본 스케줄러로 사용                  │
│  // 특히 데이터베이스 서버에서 선호                                     │
│                                                                     │
│  // Linux CFQ는 LOOK과 유사한 동작                                    │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  noop deadline [cfq]                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: SCAN에서 끝까지 이동 제거
• 장점: 불필요한 이동 없음, 효율성 향상
• LOOK: 양방향, 끝 안 감
• C-LOOK: 단방향, 점프, 끝 안 감
• 효율: SCAN보다 약 20-30% 개선
• 적합: 데이터베이스, 고하중
• 이름: 헤드가 "look"하고 이동
```
