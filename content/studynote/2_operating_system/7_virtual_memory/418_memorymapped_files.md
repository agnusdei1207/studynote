# 418. 주소 변환 (Address Translation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 논리 주소를 물리 주소로 변환
> 2. **가치**: 메모리 격리와 유연한 할당
> 3. **융합**: MMU, 페이지 테이블과 연관

---

## Ⅰ. 개요

### 개념 정의

주소 변환(Address Translation)은 **CPU가 생성한 논리 주소를 실제 물리 메모리 주소로 변환하는 과정**이다. MMU가 수행한다.

### 💡 비유: 우편물 배달
주소 변환은 **우편물 배달**과 같다. 적힌 주소(논리 주소)를 실제 위치(물리 주소)로 찾아간다.

### 주소 변환 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                주소 변환 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【변환 과정】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  CPU                                                            │ │
│  │   │                                                             │ │
│  │   │ 논리 주소 (Virtual Address)                                  │ │
│  │   │ 예: 0x12345678                                              │ │
│  │   ▼                                                             │ │
│  │  ┌─────────────────────────────────────────────────────┐       │ │
│  │  │                      MMU                              │       │ │
│  │  │                                                       │       │ │
│  │  │  1. 논리 주소 분해                                     │       │ │
│  │  │     페이지 번호: 0x12345                               │       │ │
│  │  │     오프셋: 0x678                                      │       │ │
│  │  │                                                       │       │ │
│  │  │  2. TLB 조회 (빠른 경로)                                │       │ │
│  │  │     TLB 히트 → 즉시 물리 주소                           │       │ │
│  │  │     TLB 미스 → 페이지 테이블 조회                       │       │ │
│  │  │                                                       │       │ │
│  │  │  3. 페이지 테이블 조회                                  │       │ │
│  │  │     페이지 번호 → 프레임 번호                           │       │ │
│  │  │     프레임 번호: 0xABCDE                               │       │ │
│  │  │                                                       │       │ │
│  │  │  4. 물리 주소 생성                                     │       │ │
│  │  │     물리 주소 = (프레임 번호 × 페이지크기) + 오프셋       │       │ │
│  │  │     = 0xABCDE678                                      │       │ │
│  │  │                                                       │       │ │
│  │  └─────────────────────────────────────────────────────┘       │ │
│  │   │                                                             │ │
│  │   │ 물리 주소 (Physical Address)                                 │ │
│  │   ▼                                                             │ │
│  │  ┌─────────────┐                                               │ │
│  │  │ 물리 메모리   │                                               │ │
│  │  └─────────────┘                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 분해 (32비트, 4KB 페이지)】                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  논리 주소: 0x12345678                                            │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────┬─────────────────────────┐    │ │
│  │  │      페이지 번호 (20비트)      │     오프셋 (12비트)      │    │ │
│  │  │         0x12345             │        0x678            │    │ │
│  │  └─────────────────────────────┴─────────────────────────┘    │ │
│  │                                                             │ │
│  │  페이지 번호: 페이지 테이블 인덱스                                        │ │
│  │  오프셋: 페이지 내 위치 (0~4095)                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                주소 변환 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLB의 역할】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Translation Lookaside Buffer:                                       │ │
│  │                                                             │ │
│  │  • 최근 변환 결과 캐시                                                  │ │
│  │  • 페이지 번호 → 프레임 번호 매핑                                       │ │
│  │  • 하드웨어 캐시 (매우 빠름)                                            │ │
│  │  • 64~1024 엔트리                                                      │ │
│  │                                                             │ │
│  │  TLB 히트: 1 클럭 사이클                                               │ │
│  │  TLB 미스: 10~100 클럭 사이클 (페이지 테이블 접근)                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【다단계 페이지 테이블】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  4단계 페이지 테이블 (x86_64):                                         │ │
│  │                                                             │ │
│  │  논리 주소 분해:                                                        │ │
│  │  ┌─────────┬─────────┬─────────┬─────────┬─────────────┐        │ │
│  │  │ PML4(9) │ PDPT(9) │  PD(9)  │  PT(9)  │ Offset(12)  │        │ │
│  │  └─────────┴─────────┴─────────┴─────────┴─────────────┘        │ │
│  │                                                             │ │
│  │  변환 과정:                                                              │
│  │  CR3 → PML4 → PDPT → PD → PT → 물리 주소                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【변환 실패】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  페이지 폴트 발생 조건:                                                 │ │
│  │                                                             │ │
│  │  • 페이지가 물리 메모리에 없음 (valid=0)                               │ │
│  │  • 접근 권한 위반 (읽기 전용에 쓰기)                                    │ │
│  │  • 페이지가 스왑 영역에 있음                                            │ │
│  │                                                             │ │
│  │  처리: OS가 페이지를 메모리에 로드                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLB 정보 확인】                                                        │
│  ──────────────────                                                  │
│  // CPU TLB 정보                                                         │
│  $ cpuid -1 | grep -i tlb                                            │
│  // 또는                                                                 │
│  $ cat /sys/devices/system/cpu/cpu0/cache/index*/size                │
│                                                                     │
│  // TLB 미스 통계 (perf)                                                  │
│  $ perf stat -e dTLB-load-misses,dTLB-loads ./program               │
│                                                                     │
│  【페이지 테이블 확인】                                                    │
│  ──────────────────                                                  │
│  // 페이지 테이블 엔트리 확인 (pagemap)                                    │
│  $ cat /proc/self/pagemap  // root 필요                               │
│                                                                     │
│  // 페이지 크기 확인                                                        │
│  $ getconf PAGESIZE                                                  │
│  4096                                                                │
│                                                                     │
│  【huge page로 TLB 효율 향상】                                            │
│  ──────────────────                                                  │
│  // 2MB huge page 사용                                                   │
│  $ echo 1024 > /proc/sys/vm/nr_hugepages                             │
│  // 1024개의 2MB 페이지 할당                                              │
│                                                                     │
│  // 프로그램에서 huge page 사용                                             │
│  void* ptr = mmap(NULL, 2*1024*1024, PROT_READ|PROT_WRITE,          │
│                   MAP_PRIVATE|MAP_ANONYMOUS|MAP_HUGETLB,            │
│                   -1, 0);                                           │
│                                                                     │
│  【주소 변환 비용 측정】                                                    │
│  ──────────────────                                                  │
│  // TLB 히트 vs 미스 성능 차이                                             │
│  $ perf stat -e cycles,instructions,dTLB-load-misses ./program      │
│                                                                     │
│  // 페이지 폴트 횟수                                                        │
│  $ perf stat -e page-faults ./program                               │
│     1,234 page-faults                                               │
│                                                                     │
│  【주소 변환 최적화】                                                        │
│  ──────────────────                                                  │
│  // 1. 페이지 크기 증가 (TLB 커버리지 향상)                                  │
│  // 2. 메모리 지역성 향상 (TLB 히트율 증가)                                  │
│  // 3. Huge page 사용                                                    │
│  // 4. 메모리 매핑 최소화                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 논리 주소를 물리 주소로 변환
• 수행: MMU 하드웨어
• 구조: 페이지 번호 + 오프셋
• 캐시: TLB로 변환 가속화
• 다단계: 4단계 페이지 테이블 (x86_64)
• 실패: 페이지 폴트 발생
• 최적화: huge page, 지역성 향상
```
