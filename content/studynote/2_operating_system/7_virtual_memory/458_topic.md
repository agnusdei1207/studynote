# 458. 회전 지연 (Rotational Latency)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 섹터가 헤드 아래 올 때까지 대기
> 2. **가치**: 디스크 접근 시간의 주요 구성요소
> 3. **융합**: 디스크 구조, 탐색 시간과 연관

---

## Ⅰ. 개요

### 개념 정의

회전 지연(Rotational Latency)은 **원하는 섹터가 디스크 헤드 아래에 올 때까지 기다리는 시간**이다.

### 💡 비유: 룰렛
회전 지연은 **룰렛**과 같다. 원하는 숫자가 구슬 앞에 올 때까지 기다린다.

### 회전 지연 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                회전 지연                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【디스크 회전 개념】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │         회전 방향 →                                                 │ │
│  │                    ┌─────────────────┐                          │ │
│  │                 ╭──┤                 ├──╮                        │ │
│  │                │   │   섹터 1        │   │                       │ │
│  │                │   ├─────────────────┤   │                       │ │
│  │      헤드 ──────┼──►│   섹터 2   ●    │   │ ←─ 원하는 섹터          │ │
│  │                │   ├─────────────────┤   │                       │ │
│  │                │   │   섹터 3        │   │                       │ │
│  │                 ╰──┤                 ├──╯                        │ │
│  │                    └─────────────────┘                          │ │
│  │                           │                                       │ │
│  │                     회전 지연: 섹터 2가 헤드 아래로 올 때까지         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【회전 지연 계산】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  RPM (Revolutions Per Minute): 분당 회전 수                         │ │
│  │                                                             │ │
│  │  7200 RPM:                                                          │ │
│  │  • 1회전 시간 = 60 / 7200 = 8.33 ms                                 │ │
│  │  • 평균 회전 지연 = 8.33 / 2 = 4.17 ms (반 회전)                      │ │
│  │                                                             │ │
│  │  10000 RPM:                                                         │ │
│  │  • 1회전 시간 = 60 / 10000 = 6 ms                                   │ │
│  │  • 평균 회전 지연 = 6 / 2 = 3 ms                                     │ │
│  │                                                             │ │
│  │  15000 RPM:                                                         │ │
│  │  • 1회전 시간 = 60 / 15000 = 4 ms                                   │ │
│  │  • 평균 회전 지연 = 4 / 2 = 2 ms                                     │ │
│  │                                                             │ │
│  │  최소 회전 지연: 0 ms (섹터가 바로 앞에 있음)                           │ │
│  │  최대 회전 지연: 1회전 시간 (섹터가 방금 지나감)                         │ │
│  │  평균 회전 지연: 1회전 시간의 절반                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                회전 지연 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RPM별 회전 지연】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  RPM      1회전 시간    평균 지연    용도                              │ │
│  │  ────     ─────────    ─────────    ────                            │ │
│  │  5400     11.11 ms     5.56 ms     저가형 노트북                     │ │
│  │  7200     8.33 ms      4.17 ms     일반 데스크탑                     │ │
│  │  10000    6.00 ms      3.00 ms     고성능 데스크탑                   │ │
│  │  15000    4.00 ms      2.00 ms     서버, 엔터프라이즈                 │ │
│  │                                                             │ │
│  │  높은 RPM = 낮은 회전 지연 = 빠른 성능                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【회전 지연 최적화】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 섹터 인터리빙 (Sector Interleaving):                             │ │
│  │     • 논리 섹터를 물리 섹터에 건너뛰어 배치                            │ │
│  │     • 연속 읽기 시 회전 지연 회피                                     │ │
│  │                                                             │ │
│  │  2. 섹터 슬립 (Sector Skewing):                                     │ │
│  │     • 트랙/실린더 경계에서 섹터 오프셋                                  │ │
│  │     • 헤드 이동 시간 고려                                             │ │
│  │                                                             │ │
│  │  3. 회전 최적화 스케줄링:                                             │ │
│  │     • SSTF + 회전 위치 고려                                          │ │
│  │     • 같은 트랙에서 회전 순서대로                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SSD와 회전 지연】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  SSD는 회전 부품 없음:                                               │ │
│  │  • 회전 지연 = 0                                                     │ │
│  │  • 탐색 시간 = 0                                                     │ │
│  │  • 실질적 접근 시간: 전송 시간만                                       │ │
│  │                                                             │ │
│  │  HDD → SSD 전환 시:                                                 │ │
│  │  • 회전 지연 + 탐색 시간 제거로 큰 성능 향상                            │ │
│  │  • 랜덤 I/O에서 특히 큰 차이                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【회전 지연 확인】                                                    │
│  ──────────────────                                                  │
│  // hdparm으로 디스크 정보 확인                                        │
│  $ sudo hdparm -I /dev/sda | grep RPM                               │
│  Nominal Media Rotation Rate: 7200                                  │
│                                                                     │
│  // /sys에서 회전 정보                                                │
│  $ cat /sys/block/sda/queue/rotational                              │
│  1  // 1 = 회전 디스크 (HDD), 0 = SSD                                 │
│                                                                     │
│  【접근 시간 측정】                                                    │
│  ──────────────────                                                  │
│  #include <time.h>                                                   │
│  #include <fcntl.h>                                                  │
│  #include <unistd.h>                                                 │
│                                                                     │
│  double measure_access_time(int fd, off_t offset) {                 │
│      struct timespec start, end;                                     │
│      char buf[512];                                                  │
│                                                                     │
│      clock_gettime(CLOCK_MONOTONIC, &start);                        │
│      lseek(fd, offset, SEEK_SET);                                   │
│      read(fd, buf, 512);                                            │
│      clock_gettime(CLOCK_MONOTONIC, &end);                          │
│                                                                     │
│      return (end.tv_sec - start.tv_sec) * 1000.0 +                  │
│             (end.tv_nsec - start.tv_nsec) / 1000000.0;              │
│  }                                                                   │
│                                                                     │
│  // 평균 접근 시간 측정                                                 │
│  double total = 0;                                                   │
│  for (int i = 0; i < 100; i++) {                                    │
│      off_t random_offset = rand() % disk_size;                      │
│      total += measure_access_time(fd, random_offset);               │
│  }                                                                   │
│  printf("Average access time: %.2f ms\n", total / 100);             │
│  // HDD: 약 10-15 ms (탐색 + 회전지연 + 전송)                            │
│  // SSD: 약 0.1-0.5 ms (전송만)                                        │
│                                                                     │
│  【회전 고려 스케줄링】                                                │
│  ──────────────────                                                  │
│  struct Request {                                                    │
│      int track;                                                      │
│      int sector;    // 회전 위치                                       │
│      int deadline;  // 처리 기한                                       │
│  };                                                                   │
│                                                                     │
│  // 트랙 + 섹터 위치로 정렬                                             │
│  int compare(Request a, Request b, int current_track, int current_sector) {│
│      int seek_a = abs(a.track - current_track);                     │
│      int seek_b = abs(b.track - current_track);                     │
│                                                                     │
│      // 같은 트랙이면 회전 위치 고려                                     │
│      if (seek_a == 0 && seek_b == 0) {                              │
│          return (a.sector - current_sector + SECTORS_PER_TRACK) %   │
│                 SECTORS_PER_TRACK -                                  │
│                 (b.sector - current_sector + SECTORS_PER_TRACK) %   │
│                 SECTORS_PER_TRACK;                                   │
│      }                                                               │
│      return seek_a - seek_b;                                         │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 섹터가 헤드 아래 올 때까지 대기
• 계산: 1회전 시간의 절반 (평균)
• RPM: 7200→4.17ms, 10000→3ms, 15000→2ms
• 최적화: 인터리빙, 슬립, 회전 고려 스케줄링
• SSD: 회전 지연 없음 (0 ms)
• 영향: 디스크 접근 시간의 30-40%
• 확인: hdparm -I, /sys/block/.../rotational
```
