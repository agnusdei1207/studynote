# 599. 문자 장치 (Character Device)

#### 핵심 인사이트 (3줄 요약)
> 1. **본술**: 바이트 스트림으로 접근하는 장치
> 2. **가치**: 순차 접근, 실시간, 하드웨어 제어
> 3. **융합**: 터미널, 시리얼, 디바이스 드라이버와 연관

---

## Ⅰ. 개요

### 개념 정의
**문자 장치(Character Device)**는 **바이트 단위로 순차적으로 데이터를 전송하는 장치**입니다.

### 💡 비유: 파이프
문자 장치는 **파이프**와 같습니다. 데이터가 순서대로 흐릅니다.

### 문자 장치 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                문자 장치 구조                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【문자 장치 특징】                                                     |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  특징              설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  바이트 단위        1바이트씩 전송                                  │ │   │
│  │  순차 접근          순서대로만 접근                                 │ │   │
│  │  버퍼링             없음 (또는 최소)                                │ │   │
│  │  랜덤 액세스         불가능                                         │ │   │
│  │  파일시스템         불가능                                         │ │   │
│  │  장치 파일         /dev/tty, /dev/null                            │ │   │
│  │  심볼              c (ls -l 출력)                                 │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【주요 문자 장치】                                                     │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  장치 파일          설명                    용도                   │ │   │
│  │  ────────            ────                    ────                   │ │   │
│  │  /dev/null          널 장치                입력 무시, 출력 버림      │ │   │
│  │  /dev/zero          0으로 채움              메모리 초기화            │ │   │

│  │  /dev/urandom       의사 난수                일반 용도               │ │   │

│  │  /dev/tty           제어 터미널              현재 터미널              │ │   │

│  │  /dev/ttyS0         시리얼 포트              COM1                   │ │   │

│  │  /dev/input/event0  입력 장치                키보드, 마우스           │ │   │
│  │  /dev/lp0           병렬 포트                프린터                  │ │   │

│  │                                                             │ │   │
│  │  항목            블록 장치           문자 장치                  │ │   │
│  │  ────            ─────────           ─────────                │ │   │
│  │  접근 단위        블록                 바이트                    │ │   │
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                문자 장치 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【특수 문자 장치】                                                     │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  /dev/null:                                                       │ │   │
│  │  • 모든 입력 무시                                                  │ │   │
│  │  • 읽기: EOF 반환                                                   │ │   │
│  │  $ cat /dev/null          // 아무것도 출력 안 함                    │ │   │
│  │  $ command > /dev/null    // 출력 버리기                           │ │   │
│  │  $ command 2> /dev/null   // 에러 버리기                           │ │   │
│  │                                                             │ │   │
│  │  /dev/zero:                                                       │ │   │
│  │  • 읽기: 무한한 0x00 반환                                           │ │   │
│  │  • 쓰기: 무시                                                       │ │   │
│  │  $ dd if=/dev/zero of=file bs=1M count=100                       │ │   │
│  │                                                             │ │   │
│  │  /dev/random:                                                      │ │   │
│  │  • 커널 엔트로피 풀에서 난수 생성                                    │ │   │
│  │  • 엔트로피 부족 시 블록                                             │ │   │
│  │  • 암호학적으로 안전                                                │ │   │
│  │  $ head -c 16 /dev/random | xxd                                   │ │   │
│  │                                                             │ │   │
│  │  /dev/urandom:                                                     │ │   │
│  │  • 엔트로피 부족해도 블록 안 함                                       │ │   │
│  │  • 일반 용도에 충분                                                  │ │   │
│  │  $ head -c 32 /dev/urandom | base64                               │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【문자 장치 드라이버】                                                 │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  필수 연산 (file_operations):                                      │ │   │
│  │  • open:    장치 열기                                              │ │   │
│  │  • read:    장치에서 읽기                                           │ │   │
│  │  • write:   장치에 쓰기                                            │ │   │
│  │  • release: 장치 닫기                                              │ │   │
│  │  • ioctl:   장치 제어                                              │ │   │
│  │                                                             │ │   │
│  │  등록:                                                             │ │   │
│  │  • register_chrdev() 또는                                          │ │   │
│  │  • cdev_init() + cdev_add()                                        │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【터미널 장치】                                                        │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  /dev/tty:   현재 프로세스의 제어 터미널                            │ │   │
│  │  /dev/tty0:  현재 가상 콘솔                                       │ │   │
│  │  /dev/tty1~: 가상 콘솔 1~                                         │ │   │
│  │  /dev/pts/*: 의사 터미널 (pty)                                     │ │   │
│  │                                                             │ │   │
│  │  // 현재 터미널 확인                                                  │ │   │
│  │  $ tty                                                            │ │   │
│  │  /dev/pts/0                                                       │ │   │
│  │                                                             │ │   │
│  │  // 제어 터미널에서 읽기                                              │ │   │
│  │  $ cat /dev/tty                                                   │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【문자 장치 확인】                                                    │ |
│  ──────────────────                                                │
│  // ls -l로 확인                                                      │
│  $ ls -l /dev/tty /dev/null /dev/zero                               │
│  crw-rw-rw- 1 root tty     5, 0 Jan 15 /dev/tty                      │
│  crw-rw-rw- 1 root root    1, 3 Jan 15 /dev/null                     │
│  crw-rw-rw- 1 root root    1, 5 Jan 15 /dev/zero                     │
│  //↑ c = 문자 장치        ↑ Major, Minor                               │
│                                                                     │
│  // 장치 통계                                                          │
│  $ cat /proc/devices                                                │
│  Character devices:                                                  │
│    1 mem                                                             │
│    4 ttyS                                                            │
│    5 /dev/tty                                                        │
│  ...                                                                 │
│                                                                     │
│  【/dev/null 활용】                                                   │ |
│  ──────────────────                                                │
│  // 출력 버리기                                                        │
│  $ command > /dev/null                                              │
│                                                                     │
│  // 표준 출력 + 에러 버리기                                              │
│  $ command > /dev/null 2>&1                                         │
│  $ command &> /dev/null                                             │
│                                                                     │
│  // cron에서 이메일 방지                                                │
│  0 * * * * /path/to/script > /dev/null 2>&1                         │
│                                                                     │
│  // 파일 비우기                                                        │
│  $ cat /dev/null > file.txt                                         │
│  $ > file.txt                                                        │
│                                                                     │
│  【/dev/zero 활용】                                                   │ |
│  ──────────────────                                                │
│  // 0으로 채운 파일 생성                                                │
│  $ dd if=/dev/zero of=zeros.bin bs=1M count=100                    │
│                                                                     │
│  // 파티션 0으로 초기화                                                 │
│  $ sudo dd if=/dev/zero of=/dev/sdb1 bs=1M                         │
│                                                                     │
│  // 메모리 테스트                                                       │
│  $ dd if=/dev/zero bs=1M count=1024 | md5sum                        │
│                                                                     │
│  【난수 생성】                                                         │ |
│  ──────────────────                                                │
│  // 16바이트 난수                                                      │
│  $ head -c 16 /dev/urandom | xxd                                    │
│  $ head -c 16 /dev/urandom | base64                                 │
│                                                                     │
│  // 난수 비밀번호                                                       │
│  $ head -c 32 /dev/urandom | base64                                 │
│  $ openssl rand -base64 32                                          │
│                                                                     │
│  // 엔트로피 확인                                                       │
│  $ cat /proc/sys/kernel/random/entropy_avail                        │
│                                                                     │
│  // 엔트로피 증가 (haveged)                                             │
│  $ sudo apt install haveged                                         │
│                                                                     │
│  【시리얼 포트】                                                        │ |
│  ──────────────────                                                │
│  // 시리얼 장치 확인                                                     │
│  $ ls -l /dev/ttyS* /dev/ttyUSB*                                    │
│  $ dmesg | grep tty                                                 │
│                                                                     │
│  // 시리얼 설정                                                        │
│  $ sudo stty -F /dev/ttyS0 9600 cs8 -cstopb -parenb                 │
│  $ sudo stty -F /dev/ttyS0 -a         // 현재 설정 확인               │
│                                                                     │
│  // 시리얼 통신                                                         │
│  $ sudo screen /dev/ttyUSB0 115200                                  │
│  $ sudo minicom -D /dev/ttyUSB0                                     │
│  $ sudo picocom /dev/ttyUSB0 -b 115200                              │
│                                                                     │
│  // cat으로 읽기                                                        │
│  $ cat < /dev/ttyUSB0                                               │
│  $ echo "AT" > /dev/ttyUSB0                                         │
│                                                                     │
│  【입력 장치】                                                          │ |
│  ──────────────────                                                │
│  // 입력 장치 목록                                                      │
│  $ ls -l /dev/input/                                                 │
│  $ cat /proc/bus/input/devices                                      │
│                                                                     │
│  // 입력 이벤트 확인                                                    │
│  $ sudo cat /dev/input/event0 | xxd                                 │
│  $ sudo evtest /dev/input/event0                                    │
│                                                                     │
│  【Python으로 문자 장치 접근】                                          │ |
│  ──────────────────                                                │
│  # /dev/urandom 읽기                                                  │
│  with open('/dev/urandom', 'rb') as f:                              │
│      random_bytes = f.read(16)                                      │
│      print(random_bytes.hex())                                       │
│                                                                     │
│  # /dev/null 쓰기                                                      │
│  with open('/dev/null', 'w') as f:                                  │
│      f.write("This is ignored\n")                                    │
│                                                                     │
│  # 시리얼 포트                                                          │
│  import serial                                                       │
│  ser = serial.Serial('/dev/ttyUSB0', 115200, timeout=1)             │
│  ser.write(b'Hello\n')                                              │
│  response = ser.readline()                                          │
│  ser.close()                                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 바이트 단위로 순차 접근하는 장치
• 특징: 순차 접근, 버퍼링 없음, 랜덤 액세스 불가
• 심볼: c (ls -l)
• /dev/null: 입력 무시, 읽기 시 EOF
• /dev/zero: 읽기 시 0x00 반환
• /dev/random: 진성 난수 (블록 가능)
• /dev/urandom: 의사 난수 (논블록)
• /dev/tty: 현재 제어 터미널
• /dev/ttyS0: 시리얼 포트
• Major/Minor: 드라이버/장치 식별
• 드라이버: open, read, write, ioctl
• 파일시스템: 문자 장치에는 불가능
• 확인: ls -l, /proc/devices
```
