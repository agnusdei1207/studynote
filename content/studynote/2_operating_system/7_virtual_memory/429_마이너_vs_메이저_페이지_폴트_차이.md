## 💡 Insight
- 페이지 폴트는 원인에 따라 디스크 I/O 발생 여부가 갈리며, 이는 시스템 성능에 천차만별의 영향을 미친다.
- 메이저 폴트는 물리적인 저장장치 접근이 수반되어 '밀리초(ms)' 단위의 지연을 발생시키는 반면, 마이너 폴트는 커널 내부의 주소 매핑 재설정만으로 '마이크로초(μs)' 이내에 처리된다.
- 고성능 시스템 튜닝 시 메이저 폴트 횟수를 최소화하기 위해 캐시 크기 최적화 및 Prepaging 기법이 동원된다.

---

## Ⅰ. 페이지 폴트의 두 얼굴 개요
### 1. 공통점
- 가상 주소 접근 시 하드웨어(MMU)가 해당 페이지 테이블 엔트리(PTE)에서 유효한 물리 주소를 찾지 못해 트랩(Trap)을 발생시킴.
### 2. 근본적 차이점 (I/O 여부)
- **메이저(Major)**: 물리 메모리에 데이터가 없어 디스크에서 가져와야 함. (비쌈)
- **마이너(Minor)**: 데이터는 이미 메모리 어딘가에 있지만, 현재 프로세스의 페이지 테이블에만 등록되지 않은 상태. (저렴)

📢 섹션 요약 비유: 도서관에 갔는데 내가 찾는 책이 대출 중이라 서고(Disk)에서 새로 꺼내 와야 하는 것이 메이저, 책은 옆 사람 책상(다른 프로세스 캐시)에 있는데 대출 목록만 안 바뀐 것이 마이너입니다.

---

## Ⅱ. 마이너 페이지 폴트 (Minor Page Fault / Soft Fault)
### 1. 주요 발생 상황
- **공유 메모리 (Shared Memory)**: 다른 프로세스가 이미 메모리에 올려둔 페이지에 처음 접근할 때.
- **페이지 재활용**: 할당 해제된 페이지가 아직 초기화(Zeroing)되지 않고 'Free List'에 남아 있어 즉시 회수할 때.
- **Copy-on-Write (COW)**: 읽기 전용으로 공유되던 페이지에 쓰기가 발생하여 새로운 페이지를 할당하고 매핑만 바꿀 때.

### 2. ASCII 다이어그램: Minor Page Fault Mechanism
```text
[ Process A PTE ]     [ OS Page Cache / RAM ]     [ Process B PTE ]
    (Invalid)  ---+   +-----------------------+   +--- (Valid)
                  |   |   Data Block "X"      | <--- Already Mapped
                  +-> | (Already in RAM)      |
                      +-----------------------+
                           | (Update mapping)
                           V
                      [ Process A PTE ] -> (Valid Now!)
```

📢 섹션 요약 비유: 새 아파트로 이사를 왔는데, 가구(데이터)는 전 주인이 그대로 두고 몸만 들어와서 전입 신고(매핑)만 새로 하는 것과 같습니다.

---

## Ⅲ. 메이저 페이지 폴트 (Major Page Fault / Hard Fault)
### 1. 주요 발생 상황
- **요구 페이징 (Demand Paging)**: 프로그램 실행 시 데이터가 아직 디스크(Executable File)에만 있을 때.
- **스왑 인 (Swap-in)**: 메모리 부족으로 디스크의 스왑 영역으로 쫓겨났던 페이지를 다시 불러올 때.
- **파일 I/O**: `mmap`된 파일의 데이터를 처음 읽을 때.

### 2. 성능 영향
- 디스크 접근은 RAM 접근보다 약 10,000배 ~ 1,000,000배 느리므로, 메이저 폴트가 잦으면 시스템이 멈춘 듯한 현상이 발생함.

📢 섹션 요약 비유: 집에 음식이 하나도 없어서 마트(Disk)까지 차를 타고 가서 장을 봐와야 하는(I/O 지연) 상황입니다.

---

## Ⅳ. 성능 분석과 모니터링
### 1. 확인 방법
- Linux: `ps -o min_flt,maj_flt` 또는 `top`, `vmstat` 명령어를 통해 확인 가능.
### 2. 튜닝 전략
- **메이저 폴트 과다**: RAM 증설, 스와피니스(Swappiness) 조절, Hugepages 사용 고려.
- **마이너 폴트 과다**: 라이브러리 로딩 구조 개선, 메모리 풀(Pool) 활용.

📢 섹션 요약 비유: 우리 집 요리사가 마트에 너무 자주 가면(메이저 폴트), 냉장고(RAM)를 더 큰 걸로 바꿔줘야 합니다.

## Ⅴ. 처리 절차 비교 (커널 내부)
| 단계 | 마이너 페이지 폴트 | 메이저 페이지 폴트 |
| :--- | :--- | :--- |
| **트랩 발생** | 예 (CPU -> OS) | 예 (CPU -> OS) |
| **VMA 확인** | 예 (유효성 검사) | 예 (유효성 검사) |
| **디스크 I/O** | **아니오** (RAM 내 검색) | **예** (저장장치 대기) |
| **프로세스 상태** | 실행 대기 (짧음) | 블록(Blocked) 상태로 전환 |
| **PTE 갱신** | 예 | 예 |

📢 섹션 요약 비유: 마이너 폴트는 "깜빡하고 안 적은 이름표 달기"이고, 메이저 폴트는 "빈 상자 채우러 원정 가기"입니다.
