# 453. FCFS 스케줄링 (First-Come, First-Served)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 요청 도착 순서대로 처리
> 2. **가치**: 구현 단순, 공정성 보장
> 3. **융합**: 디스크 스케줄링, 대기열과 연관

---

## Ⅰ. 개요

### 개념 정의

FCFS 스케줄링(First-Come, First-Served)은 **요청이 도착한 순서대로 디스크 I/O를 처리하는 가장 단순한 스케줄링 알고리즘**이다.

### 💡 비유: 줄 서기
FCFS는 **줄 서기**와 같다. 먼저 온 사람이 먼저 받는다. 순서가 바뀌지 않는다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                FCFS 상세                                               │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 공정성 (Fairness):                                            │ │
│  │     • 모든 요청이 도착 순서대로 처리                                    │ │
│  │     • 기아 현상 없음                                                  │ │
│  │     • 대기 시간 예측 가능                                              │ │
│  │                                                             │ │
│  │  2. 단순성 (Simplicity):                                          │ │
│  │     • 구현이 매우 간단                                                │ │
│  │     • 오버헤드 거의 없음                                               │ │
│  │     • FIFO 큐만 필요                                                 │ │
│  │                                                             │ │
│  │  3. 추가/삭제 용이:                                                   │ │
│  │     • 큐 연산만으로 관리                                                │ │
│  │     • 동적 요청 처리 가능                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 성능 저하:                                                      │ │
│  │     • 평균 탐색 시간이 길어질 수 있음                                    │ │
│  │     • 랜덤 요청 패턴에 취약                                            │ │
│  │     • 헤드 이동이 비효율적                                              │ │
│  │                                                             │ │
│  │  2. Convoy Effect:                                                 │ │
│  │     • 여러 요청이 한 곳에 몰리면 다른 요청 대기 길어짐                       │ │
│  │     • 불균형한 분포에 취약                                              │ │
│  │                                                             │ │
│  │  3. 스로틀링 (Thrashing) 위험:                                        │ │
│  │     • 양 끝단 이동 반복 시 성능 급감                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FCFS 성능 분석】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  최악의 경우:                                                         │ │
│  │  요청: 0, 199, 0, 199, 0, 199, ... (현재: 100)                        │ │
│  │  이동: 100 + 199 + 199 + 199 + ... = 매번 199씩 이동                   │ │
│  │                                                             │ │
│  │  최선의 경우:                                                         │ │
│  │  요청이 순차적이거나 인접 위치에 있음                                      │ │
│  │  이동 거리 최소화                                                        │ │
│  │                                                             │ │
│  │  평균 성능:                                                           │ │
│  │  • 이론적 평균 탐색 시간: 전체 트랙의 1/3                                 │ │
│  │  • 실제로는 SSTF, SCAN보다 느림                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FCFS 구현 (의사코드)】                                              │
│  ──────────────────                                                  │
│  struct Request {                                                    │
│      int track;                                                      │
│      // 기타 정보 (시간, 데이터 등)                                        │
│  };                                                                   │
│                                                                     │
│  Queue requestQueue;  // FIFO 큐                                      │
│  int currentHead = 0;                                                │
│                                                                     │
│  void fcfs_schedule() {                                              │
│      while (!requestQueue.empty()) {                                 │
│          Request req = requestQueue.dequeue();                       │
│          int seekDistance = abs(req.track - currentHead);            │
│          processRequest(req);                                        │
│          currentHead = req.track;                                    │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  【성능 측정】                                                          │
│  ──────────────────                                                  │
│  int totalSeek = 0;                                                  │
│  int requestCount = 0;                                               │
│                                                                     │
│  void measureFCFS(int requests[], int n, int head) {                │
│      totalSeek = 0;                                                  │
│      currentHead = head;                                             │
│      for (int i = 0; i < n; i++) {                                  │
│          totalSeek += abs(requests[i] - currentHead);               │
│          currentHead = requests[i];                                  │
│          requestCount++;                                             │
│      }                                                               │
│      printf("Total seek: %d\n", totalSeek);                          │
│      printf("Average seek: %.2f\n", (float)totalSeek / n);           │
│  }                                                                   │
│                                                                     │
│  【FCFS 사용 사례】                                                    │
│  ──────────────────                                                  │
│  적합한 경우:                                                           │
│  • 요청이 적고 랜덤하지 않을 때                                          │
│  • 구현 단순성이 중요할 때                                                │
│  • 공정성이 최우선일 때                                                   │
│  • SSD (탐색 시간 없음)                                                │
│                                                                     │
│  부적합한 경우:                                                         │
│  • 고하중 (high load) 환경                                              │
│  • 실시간 시스템                                                        │
│  • 랜덤 I/O 패턴                                                        │
│                                                                     │
│  【리눅스에서 FCFS 확인】                                               │
│  ──────────────────                                                  │
│  // noop 스케줄러가 FCFS와 유사                                         │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  [noop] deadline cfq                                                 │
│                                                                     │
│  // noop으로 변경                                                       │
│  $ echo noop > /sys/block/sda/queue/scheduler                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 요청 도착 순서대로 처리
• 구현: FIFO 큐 사용
• 장점: 공정, 단순, 기아 없음
• 단점: 평균 탐색 시간 김, 비효율
• 적합: 저하중, SSD, 단순 시스템
• 부적합: 고하중, 실시간, 랜덤 I/O
• 최악: 양 끝단 반복 이동
```
