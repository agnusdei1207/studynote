# 598. 블록 장치 (Block Device)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 블록 단위로 접근하는 장치
> 2. **가치**: 랜덤 액세스, 버퍼링, 파일시스템 지원
> 3. **융합**: HDD, SSD, 문자 장치와 연관

---

## Ⅰ. 개요

### 개념 정의
**블록 장치(Block Device)**는 **고정된 크기의 블록 단위로 데이터를 읽고 쓰는 장치**입니다.

### 💡 비유: 책장
블록 장치는 **책장**과 같습니다. 원하는 칸에 직접 접근할 수 있습니다.

### 블록 장치 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                블록 장치 구조                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【블록 장치 특징】                                                     │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  특징              설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  블록 단위         고정 크기 (보통 512B 또는 4KB)                   │ │   │
│  │  랜덤 액세스        임의 위치 직접 접근 가능                         │ │   │
│  │  버퍼링            커널 버퍼 캐시 사용                             │ │   │
│  │  파일시스템         파일시스템 구축 가능                            │ │   │
│  │  장치 파일         /dev/sda, /dev/nvme0n1                        │ │   │
│  │  심볼              b (ls -l 출력)                                 │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【블록 장치 vs 문자 장치】                                             │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  항목            블록 장치           문자 장치                  │ │   │
│  │  ────            ─────────           ─────────                │ │   │
│  │  접근 단위        블록 (512B~4KB)     바이트                    │ │   │
│  │  랜덤 액세스      가능                불가능                    │ │   │
│  │  버퍼링           있음                없음                      │ │   │
│  │  파일시스템       가능                불가능                    │ │   │
│  │  장치 파일        /dev/sda            /dev/tty                  │ │   │
│  │  심볼             b                   c                         │ │   │
│  │  예시             HDD, SSD, USB       터미널, 키보드              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【주요 블록 장치】                                                     │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  장치 파일          설명                    예시                  │ │   │
│  │  ────────            ────                    ────                  │ │   │
│  │  /dev/sda           첫 번째 SATA/SCSI        HDD, SSD              │ │   │
│  │  /dev/sdb           두 번째 SATA/SCSI                              │ │   │
│  │  /dev/nvme0n1       첫 번째 NVMe SSD                              │ │   │
│  │  /dev/mmcblk0       SD 카드, eMMC                                 │ │   │
│  │  /dev/loop0         루프백 장치                                     │ │   │
│  │  /dev/ram0          RAM 디스크                                     │ │   │
│  │  /dev/dm-0          Device Mapper (LVM)                           │ │   │
│  │  /dev/md0           Software RAID                                  │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                블록 장치 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【블록 크기】                                                         │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  섹터 (Sector):                                                   │ │   │
│  │  • 물리적 최소 단위                                               │ │   │
│  │  • 전통적으로 512바이트                                            │ │   │
│  │  • 최신 디스크: 4KB (Advanced Format)                             │ │   │
│  │                                                             │ │   │
│  │  블록 (Block):                                                     │ │   │
│  │  • 논리적/파일시스템 단위                                          │ │   │
│  │  • 보통 4KB (4096바이트)                                           │ │   │
│  │  • 파일시스템 생성 시 설정                                          │ │   │
│  │                                                             │ │   │
│  │  // 섹터 크기 확인                                                   │
│  │  $ sudo blockdev --getpbsz /dev/sda    // 물리                   │ │   │
│  │  $ sudo blockdev --getss /dev/sda      // 논리                   │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【버퍼 캐시】                                                          │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  페이지 캐시 (Page Cache):                                         │ │   │
│  │  • 파일 데이터 캐시                                               │ │   │
│  │  • 블록 장치 I/O 최적화                                            │ │   │
│  │  • 읽기: 캐시에서 반환 (히트 시)                                    │ │   │
│  │  • 쓰기: 지연 쓰기 (writeback)                                     │ │   │
│  │                                                             │ │   │
│  │  동기화:                                                           │ │   │
│  │  $ sync                         // 모든 캐시 플러시                │ │   │
│  │  $ sudo blockdev --flushbufs /dev/sda                            │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【장치 번호】                                                          │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  Major 번호: 장치 드라이버 식별                                     │ │   │
│  │  Minor 번호: 해당 드라이버의 개별 장치                              │ │   │
│  │                                                             │ │   │
│  │  // 장치 번호 확인                                                   │
│  │  $ ls -l /dev/sda                                                │
│  │  brw-rw---- 1 root disk 8, 0 Jan 15 /dev/sda                     │
│  │  //                        ↑ ↑                                    │
│  │  //                    Major Minor                                 │ │   │
│  │                                                             │ │   │
│  │  $ cat /proc/devices | grep -i block                              │ │   │
│  │  Block devices:                                                    │ │   │
│  │   8 sd                                                             │ │   │
│  │  259 nvme                                                          │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【블록 장치 확인】                                                    │ |
│  ──────────────────                                                │
│  // lsblk                                                            │
│  $ lsblk                                                             │
│  $ lsblk -f                     // 파일시스템 정보                   │
│  $ lsblk -o NAME,SIZE,TYPE,FSTYPE,MOUNTPOINT                        │
│  $ lsblk -d                     // 파티션 제외                       │
│                                                                     │
│  // fdisk                                                            │
│  $ sudo fdisk -l                                                     │
│  $ sudo fdisk -l /dev/sda                                           │
│                                                                     │
│  // /proc/partitions                                                  │
│  $ cat /proc/partitions                                              │
│                                                                     │
│  // blkid                                                            │
│  $ sudo blkid                                                        │
│  $ sudo blkid /dev/sda1                                             │
│                                                                     │
│  【블록 장치 정보】                                                    │ |
│  ──────────────────                                                │
│  // blockdev                                                          │
│  $ sudo blockdev --getsize64 /dev/sda      // 크기 (bytes)          │
│  $ sudo blockdev --getsz /dev/sda           // 크기 (512B 섹터)       │
│  $ sudo blockdev --getbsz /dev/sda          // 블록 크기              │
│  $ sudo blockdev --getss /dev/sda           // 섹터 크기              │
│  $ sudo blockdev --getpbsz /dev/sda         // 물리 섹터 크기         │
│  $ sudo blockdev --getro /dev/sda           // 읽기 전용 여부         │
│                                                                     │
│  // hdparm (SATA)                                                     │
│  $ sudo hdparm -I /dev/sda               // 상세 정보                  │
│  $ sudo hdparm -t /dev/sda               // 읽기 성능                  │
│                                                                     │
│  // nvme-cli (NVMe)                                                   │
│  $ sudo nvme list                                                   │
│  $ sudo nvme smart-log /dev/nvme0                                   │
│                                                                     │
│  【블록 장치 설정】                                                    │ |
│  ──────────────────                                                │
│  // 읽기 전용 설정                                                      │
│  $ sudo blockdev --setro /dev/sda                                   │
│  $ sudo blockdev --setrw /dev/sda                                   │
│                                                                     │
│  // read-ahead 설정                                                   │
│  $ sudo blockdev --setra 256 /dev/sda    // 256 * 512 = 128KB        │
│  $ sudo blockdev --getra /dev/sda                                   │
│                                                                     │
│  // udev 규칙 (/etc/udev/rules.d/)                                    │
│  // 블록 장치 속성 변경                                                 │
│                                                                     │
│  【루프백 장치】                                                       │ |
│  ──────────────────                                                │
│  // 이미지 파일을 블록 장치로 마운트                                      │
│  $ sudo losetup /dev/loop0 disk.img                                │
│  $ sudo losetup -a                 // 연결된 루프 장치 목록             │
│  $ sudo losetup -d /dev/loop0      // 해제                           │
│                                                                     │
│  // 직접 마운트                                                        │
│  $ sudo mount -o loop disk.img /mnt                                │
│                                                                     │
│  // 새 루프 장치 생성                                                   │
│  $ sudo losetup -f --show disk.img    // 자동 할당                    │
│  /dev/loop1                                                         │
│                                                                     │
│  【RAM 디스크】                                                        │ |
│  ──────────────────                                                │
│  // ramdisk 장치                                                      │
│  $ sudo mke2fs /dev/ram0                                           │
│  $ sudo mount /dev/ram0 /mnt/ram                                   │
│                                                                     │
│  // tmpfs (더 유연)                                                    │
│  $ sudo mount -t tmpfs -o size=100M tmpfs /mnt/tmp                 │
│                                                                     │
│  【dd로 블록 장치 복사】                                               │ |
│  ──────────────────                                                │
│  // 디스크 이미지 생성                                                  │
│  $ sudo dd if=/dev/sda of=disk.img bs=4M status=progress            │
│                                                                     │
│  // 디스크 간 복사                                                      │
│  $ sudo dd if=/dev/sda of=/dev/sdb bs=4M                           │
│                                                                     │
│  // 빠른 복사 (block size 최적화)                                       │
│  $ sudo dd if=/dev/sda of=/dev/sdb bs=64K conv=noerror,sync         │
│                                                                     │
│  【I/O 스케줄러】                                                      │ |
│  ──────────────────                                                │
│  // 현재 스케줄러 확인                                                   │
│  $ cat /sys/block/sda/queue/scheduler                              │
│  [mq-deadline] none bfq                                             │
│                                                                     │
│  // 스케줄러 변경                                                       │
│  $ echo bfq | sudo tee /sys/block/sda/queue/scheduler              │
│                                                                     │
│  // 종류:                                                             │
│  // none - NVMe용 (기본)                                              │
│  // mq-deadline - 일반                                                │
│  // bfq - 데스크탑 (공정성)                                            │
│  // kyber - 저지연 SSD                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 블록 단위로 접근하는 장치
• 특징: 랜덤 액세스, 버퍼링, 파일시스템 지원
• 심볼: b (ls -l)
• 섹터: 물리적 단위 (512B 또는 4KB)
• 블록: 논리적 단위 (보통 4KB)
• 장치 파일: /dev/sda, /dev/nvme0n1
• Major/Minor: 드라이버/장치 식별
• 확인: lsblk, fdisk -l, blkid
• 정보: blockdev --getsize64
• 루프백: losetup
• RAM 디스크: /dev/ram0, tmpfs
• I/O 스케줄러: none, mq-deadline, bfq
• 버퍼 캐시: 페이지 캐시, sync
```
