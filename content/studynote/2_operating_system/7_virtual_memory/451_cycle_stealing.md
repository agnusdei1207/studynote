# 451. 디스크 구조 (Disk Structure)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 회전식 저장 장치의 물리적 구조
> 2. **가치**: 데이터 저장과 접근 시간 결정
> 3. **융합**: 파일 시스템, I/O 스케줄링과 연관

---

## Ⅰ. 개요

### 개념 정의

디스크 구조(Disk Structure)는 **하드디스크의 물리적 구성 요소와 데이터 저장 방식**을 정의한다. 플래터, 트랙, 실린더, 섹터로 구성된다.

### 💡 비유: 자판기
디스크는 **회전하는 자판기**와 같다. 여러 층(플래터)이 있고, 각 층에는 원형 선(트랙)이 있다. 헤드가 원하는 위치로 이동해 물건(데이터)을 꺼낸다.

### 디스크 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                디스크 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【물리적 구성 요소】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │         ┌─────────────────────────────┐                       │ │
│  │         │      플래터 (Platter)        │                       │ │
│  │         │  ┌─────────────────────┐    │                       │ │
│  │         │  │    트랙 (Track)      │    │                       │ │
│  │         │  │  ┌─────────────┐    │    │                       │ │
│  │         │  │  │  섹터       │    │    │                       │ │
│  │         │  │  │  (Sector)   │    │    │                       │ │
│  │         │  │  └─────────────┘    │    │                       │ │
│  │         │  │                     │    │                       │ │
│  │         │  └─────────────────────┘    │                       │ │
│  │         │            │                │                       │ │
│  │         │      ┌─────┴─────┐          │                       │ │
│  │         │      │  스핀들   │          │                       │ │
│  │         │      │ (Spindle) │          │                       │ │
│  │         │      └───────────┘          │                       │ │
│  │         └─────────────────────────────┘                       │ │
│  │                    ▲                                          │ │
│  │                    │                                          │ │
│  │              ┌─────┴─────┐                                    │ │
│  │              │  헤드     │                                    │ │
│  │              │  (Head)   │                                    │ │
│  │              └───────────┘                                    │ │
│  │                    │                                          │ │
│  │              ┌─────┴─────┐                                    │ │
│  │              │   암      │                                    │ │
│  │              │  (Arm)    │                                    │ │
│  │              └───────────┘                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【용어 정리】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 플래터 (Platter): 데이터가 기록되는 원형 판                             │ │
│  │  • 트랙 (Track): 플래터上的 동심원                                         │ │
│  │  • 섹터 (Sector): 트랙을 나눈 최소 단위 (512B)                              │ │
│  │  • 실린더 (Cylinder): 같은 위치의 트랙 모음                                │ │
│  │  • 헤드 (Head): 데이터를 읽고 쓰는 장치                                    │ │
│  │  • 암 (Arm): 헤드를 이동시키는 장치                                        │ │
│  │  • 스핀들 (Spindle): 플래터를 회전시키는 축                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                디스크 구조 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【실린더 개념】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  플래터 0:  트랙 0 ───────────────────────────────            │ │
│  │  플래터 1:  트랙 0 ───────────────────────────────            │ │
│  │  플래터 2:  트랙 0 ───────────────────────────────            │ │
│  │  플래터 3:  트랙 0 ───────────────────────────────            │ │
│  │           ─────────────────────────────────                   │ │
│  │                    실린더 0                                   │ │
│  │                                                             │ │
│  │  같은 실린더의 모든 트랙은 암 이동 없이 접근 가능                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【디스크 용량 계산】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  용량 = 플래터 수 × 트랙 수/플래터 × 섹터 수/트랙 × 섹터 크기                 │ │
│  │                                                             │ │
│  │  예시:                                                                     │ │
│  │  • 플래터: 4개                                                             │ │
│  │  • 트랙: 10,000개/플래터                                                   │ │
│  │  • 섹터: 1,000개/트랙                                                      │ │
│  │  • 섹터 크기: 512B                                                         │ │
│  │                                                             │ │
│  │  용량 = 4 × 10,000 × 1,000 × 512B = 20GB                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【데이터 저장 방식】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ZBR (Zone Bit Recording):                                               │ │
│  │  • 외곽 트랙: 더 많은 섹터 (회전당 더 많은 데이터)                           │ │
│  │  • 내부 트랙: 더 적은 섹터                                                 │ │
│  │  • 용량 효율화                                                             │ │
│  │                                                             │ │
│  │  CHS 주소 지정:                                                            │ │
│  │  • Cylinder (실린더 번호)                                                 │ │
│  │  • Head (헤드 번호 = 플래터 면)                                            │ │
│  │  • Sector (섹터 번호)                                                     │ │
│  │                                                             │ │
│  │  LBA (Logical Block Addressing):                                         │ │
│  │  • 섹터를 선형 번호로 주소 지정                                             │ │
│  │  • LBA 0, 1, 2, ...                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  • RPM 중요                               • IOPS 중요                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 구성: 플래터, 트랙, 섹터, 실린더, 헤드, 암
• 저장: 자기 방식으로 데이터 기록
• 주소: CHS → LBA (논리 주소)
• 용량: 플래터×트랙×섹터×섹터크기
• 성능: RPM, 탐색시간, 회전지연
• 확인: fdisk, hdparm, smartctl
• 진화: HDD → SSD (물리적 구조 변경)
```
