# 426. 수정 비트 (Dirty Bit)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이지 내용 변경 여부 표시
> 2. **가치**: 불필요한 디스크 쓰기 방지
> 3. **융합**: 페이지 교체, 쓰기 정책과 연관

---

## Ⅰ. 개요

### 개념 정의

수정 비트(Dirty Bit)는 **페이지 내용이 메모리에서 수정되었는지 나타내는 비트**다. 페이지 테이블 엔트리에 있다. 수정되면 디스크와 다르다.

### 💡 비유: 오염된 옷
수정 비트는 **오염된 옷**과 같다. 입은 옷(수정)은 세탁(디스크 쓰기)이 필요하다. 안 입은 옷은 그대로 둔다.

### 수정 비트 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                수정 비트 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 테이블 엔트리】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │ 프레임 번호  │ R │ D │ X │ W │ U │ ... │                │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                         ↑                                      │ │
│  │                    수정 비트 (Dirty/Modify)                     │ │
│  │                                                             │ │
│  │  D=0: 페이지 내용이 디스크와 동일 (clean)                           │ │
│  │  D=1: 페이지 내용이 수정됨 (dirty)                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 방식】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  페이지 로드 시:                                                        │
│  │  • 수정 비트 = 0 (clean 상태)                                          │ │
│  │                                                             │ │
│  │  페이지 쓰기 시:                                                        │
│  │  • 하드웨어가 자동으로 수정 비트 = 1 (dirty 상태)                       │ │
│  │                                                             │ │
│  │  페이지 교체 시:                                                        │
│  │  • D=0: 그냥 버림 (디스크와 동일)                                       │ │
│  │  • D=1: 디스크에 쓰고 버림 (변경 사항 저장)                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【교체 비용 차이】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Clean 페이지 (D=0) 교체:                                              │ │
│  │  • 메모리에서 해제만 하면 됨                                             │ │
│  │  • 비용: 메모리 연산만                                                  │ │
│  │                                                             │ │
│  │  Dirty 페이지 (D=1) 교체:                                              │ │
│  │  • 디스크에 쓰기 → 메모리 해제                                          │ │
│  │  • 비용: 디스크 I/O + 메모리 연산                                       │ │
│  │  • 훨씬 느림!                                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                수정 비트 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【수정 비트의 이점】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 디스크 쓰기 감소                                                    │ │
│  │     • 수정 안 된 페이지는 디스크에 안 써도 됨                             │ │
│  │     • I/O 비용 절약                                                     │ │
│  │                                                             │ │
│  │  2. 페이지 교체 최적화                                                  │ │
│  │     • clean 페이지 우선 교체                                            │ │
│  │     • 교체 속도 향상                                                    │ │
│  │                                                             │ │
│  │  3. 정전 시 데이터 복구                                                 │ │
│  │     • dirty 페이지만 복구하면 됨                                        │ │
│  │     • clean은 디스크에서 다시 로드                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【참조 비트와 수정 비트 조합】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  (R, D)   상태                  교체 우선순위                    │ │
│  │  ──────  ─────────────────  ────────────────                   │ │
│  │  (0, 0)  사용 안 함, clean    1순위 (최우선)                      │ │
│  │  (0, 1)  사용 안 함, dirty    2순위                              │ │
│  │  (1, 0)  사용함, clean       3순위                              │ │
│  │  (1, 1)  사용함, dirty       4순위 (최하)                        │ │
│  │                                                             │ │
│  │  Enhanced Clock Algorithm에서 활용                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【수정 비트 리셋】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  디스크에 쓴 후:                                                        │ │
│  │  • 수정 비트 = 0으로 리셋                                               │ │
│  │  • 페이지 내용 = 디스크와 동일                                          │ │
│  │                                                             │ │
│  │  pdflush/kswapd가 백그라운드에서 수행                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【수정 페이지 통계】                                                    │
│  ──────────────────                                                  │
│  $ cat /proc/meminfo | grep -E "Dirty|Writeback"                    │
│  Dirty:             12345 kB  // 수정된 페이지                         │
│  Writeback:          1234 kB  // 디스크 쓰기 중                        │
│                                                                     │
│  // 수정 페이지 플러시                                                   │
│  $ sync  // 모든 dirty 페이지를 디스크에 쓰기                             │
│                                                                     │
│  【수정 비트 관련 설정】                                                 │
│  ──────────────────                                                  │
│  // dirty 페이지 비율 설정                                               │
│  $ cat /proc/sys/vm/dirty_ratio                                      │
│  20  // 메모리의 20%가 dirty면 프로세스 중단하고 플러시                     │
│                                                                     │
│  $ cat /proc/sys/vm/dirty_background_ratio                           │
│  10  // 10%면 백그라운드 플러시 시작                                       │
│                                                                     │
│  // dirty 페이지 만료 시간                                               │
│  $ cat /proc/sys/vm/dirty_expire_centisecs                          │
│  3000  // 30초 지난 dirty 페이지는 플러시                                  │
│                                                                     │
│  【프로그래밍에서 활용】                                                  │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│                                                                     │
│  // 메모리 동기화                                                        │
│  void *ptr = mmap(NULL, size, PROT_READ|PROT_WRITE,                 │
│                   MAP_SHARED, fd, 0);                                │
│  // 데이터 수정...                                                      │
│  ptr[0] = 'X';                                                       │
│                                                                     │
│  // 수정 내용을 디스크에 동기화                                           │
│  msync(ptr, size, MS_SYNC);  // 동기                                   │
│  msync(ptr, size, MS_ASYNC); // 비동기                                  │
│                                                                     │
│  // 파일 동기화                                                          │
│  #include <unistd.h>                                                 │
│  fsync(fd);   // 파일 + 메타데이터                                       │
│  fdatasync(fd); // 파일 데이터만                                         │
│  sync();      // 전체 시스템                                             │
│                                                                     │
│  【성능 모니터링】                                                        │
│  ──────────────────                                                  │
│  // writeback 통계                                                     │
│  $ cat /proc/vmstat | grep -E "nr_dirty|nr_writeback"               │
│  nr_dirty 1234                                                       │
│  nr_writeback 123                                                    │
│                                                                     │
│  // I/O 통계                                                            │
│  $ iostat -x 1                                                       │
│  // 디스크 쓰기 모니터링                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
