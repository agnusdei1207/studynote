# 181. 에이징 (Aging)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 대기 시간에 따른 우선순위 점진적 증가
> 2. **가치**: 기아 현상 방지의 핵심 기법
> 3. **융합**: 우선순위 스케줄링, 기아 현상, 공정성과 연관

---

## Ⅰ. 개요

### 개념 정의

에이징(Aging)은 **오랫동안 대기한 프로세스의 우선순위를 점진적으로 높여주는 기법**이다. 기아 현상을 방지하고 공정성을 보장한다.

### 💡 비유: 마일리지 적립
에이징은 **마일리지 적립**과 같다. 오래 기다릴수록(대기 시간) 포인트(우선순위)가 쌓여서, 결국에는 우대 혜택을 받는다.

### 에이징 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                에이징 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【에이징 동작 원리】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  유효 우선순위 = 기본 우선순위 + (대기 시간 / 에이징 상수)         │ │
│  │                                                             │ │
│  │  또는                                                         │ │
│  │                                                             │ │
│  │  유효 우선순위 = 기본 우선순위 × 에이징 팩터^대기시간              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【에이징 적용 예시】                                                    │
│  ──────────────────                                                  │

│  │                                                             │ │
│  │  프로세스    기본 우선순위    대기 시간    유효 우선순위         │ │

│  │  P1             10              0초           10                  │ │
│  │  P2             30              0초           30                  │ │
│  │  P3             50              0초           50                  │ │
│  │                                                             │ │
│  │  5초 후 (에이징: +2/초):                                        │ │
│  │  P1             10              5초           10                 │ │

│  │  → P3가 가장 높은 우선순위 획득 → 실행 기회!                     │ │

│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시간에 따른 우선순위 변화】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  우선순위                                                      │ │
│  │    ▲                                                           │ │
│  │    │  P_low (기아 위험)                                        │ │
│  │ 60│              ╱                                            │ │
│  │    │            ╱                                             │ │
│  │ 40│          ╱  P_med                                         │ │

│  │ 20│      ╱ ╱  P_high                                          │ │
│  │    │    ╱ ╱ ────────                                          │ │
│  │    │  ╱ ╱                                                     │ │
│  │  0├─────────────────────────────────────────────► 시간         │ │
│  │    0   5   10   15   20   25   30                             │ │
│  │                                                             │ │
│  │  → 낮은 우선순위가 점진적으로 상승                                │ │
│  │  → 결국 높은 우선순위를 넘어서 실행                               │ │
```

## Ⅱ. 상세 분석

```
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 선형 에이징                                                 │ │
│  │     priority += wait_time / constant                            │ │
│  │     - 구현 간단                                                 │ │
│  │     - 대기 시간에 비례                                           │ │
│  │                                                               │ │
│  │  2. 지수 에이징                                                 │ │
│  │     priority *= aging_factor^wait_time                          │ │

│  │     - 빠른 기아 해결                                             │ │
│  │                                                               │ │
│  │  3. 주기적 증가                                                  │ │
│  │     매 N초마다 priority += 1                                    │ │
│  │     - 이산적 증가                                                │ │
│  │     - 오버헤드 적음                                              │ │

│  │  ────                    ────                  ────              │ │

│  │  최대 상한           없음: 우선순위 과다  너무 낮음: 기아 가능      │ │
```

## Ⅲ. 실무 적용

### 구현 및 설정
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【에이징 구현 코드】                                                    │
│  ──────────────────                                                  │
│  #define AGING_INTERVAL 1000  // 1초                                │
│  #define AGING_BONUS 1                                              │
│                                                                     │
│  struct Process {                                                    │
│      int pid;                                                        │
│      int base_priority;                                              │
│      int effective_priority;                                         │
│      time_t arrival_time;                                            │
│  };                                                                  │
│                                                                     │
│  void apply_aging(Process *p, time_t current_time) {                 │
│      time_t wait_time = current_time - p->arrival_time;              │
│      int aging_bonus = (wait_time * 1000) / AGING_INTERVAL;          │
│      p->effective_priority = p->base_priority + aging_bonus;         │
│  }                                                                   │
│                                                                     │
│  // 스케줄링 결정 시                                                   │
│  Process* select_process(Process processes[], int n) {               │
│      time_t current = time(NULL);                                    │
│      Process *best = NULL;                                           │
│      int max_priority = -1;                                          │
│                                                                     │
│      for (int i = 0; i < n; i++) {                                   │
│          apply_aging(&processes[i], current);                        │
│          if (processes[i].effective_priority > max_priority) {       │
│              max_priority = processeses[i].effective_priority;       │
│              best = &processes[i];                                   │
│          }                                                           │
│      }                                                               │
│      return best;                                                    │
│  }                                                                   │
│                                                                     │
│  【Linux CFS의 에이징 유사 기능】                                        │
│  ──────────────────                                                  │
│  # CFS는 vruntime으로 기아 방지                                       │
│  # 적게 실행된 프로세스 = 낮은 vruntime = 우선 실행                     │
│                                                                     │
│  # vruntime 확인                                                     │
│  $ cat /proc/<pid>/sched | grep vruntime                            │
│                                                                     │
│  # nice 값에 따른 가중치                                               │
│  # 낮은 nice = 높은 가중치 = vruntime 천천히 증가                       │
│                                                                     │
│  【Windows 우선순위 부스트】                                             │
│  ──────────────────                                                  │
│  # 포그라운드 프로세스 우선순위 부스트                                   │
│  # 대기 후 우선순위 일시적 증가                                         │
│                                                                     │
│  // Windows API                                                      │
│  SetProcessPriorityBoost(GetCurrentProcess(), TRUE);                 │
│                                                                     │
│  【에이징 관련 설정 확인】                                                │
│  ──────────────────                                                  │
│  # Linux 스케줄러 관련                                                 │
│  $ cat /proc/sys/kernel/sched_min_granularity_ns                     │
│  $ cat /proc/sys/kernel/sched_wakeup_granularity_ns                  │
│                                                                     │
│  # starvation 감지 스크립트                                           │
│  $ while true; do                                                    │
│      ps -eo pid,stat,time,comm --sort=time | head -20                │
│      sleep 5                                                         │
│  done                                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 대기 시간에 따른 우선순위 증가
• 목적: 기아 현상 방지
• 방식: 선형, 지수, 주기적 증가
• 공식: 유효 우선순위 = 기본 + (대기시간/상수)
• 장점: 공정성 보장
• 단점: 오버헤드, 파라미터 튜닝
```
