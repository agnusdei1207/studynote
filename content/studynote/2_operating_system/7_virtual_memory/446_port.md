# 446. 파일 권한 (File Permission)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 접근 제어 메커니즘
> 2. **가치**: 보안과 보호
> 3. **융합**: 아이노드, 사용자 관리와 연관

---

## Ⅰ. 개요

### 개념 정의

파일 권한(File Permission)은 **누가 파일에 어떤 방식으로 접근할 수 있는지 제어하는 메커니즘**이다. 읽기, 쓰기, 실행 권한을 관리한다.

### 💡 비유: 열쇠
파일 권한은 **열쇠**와 같다. 주인은 모든 열쇠를 갖는다. 가족은 일부만 받는다. 낯선 사람은 못 들어간다.

### 파일 권한 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 권한 구조                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【유닉스 권한 모델】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  사용자 분류:                                                               │
│  │  • 소유자 (Owner/User, u): 파일 생성자                                    │ │
│  │  • 그룹 (Group, g): 파일이 속한 그룹                                      │ │
│  │  • 기타 (Other, o): 그 외 모든 사용자                                     │ │
│  │                                                             │ │
│  │  권한 종류:                                                               │
│  │  • 읽기 (Read, r): 내용 보기                                             │ │
│  │  • 쓰기 (Write, w): 내용 수정, 삭제                                      │ │
│  │  • 실행 (Execute, x): 프로그램 실행, 디렉토리 진입                        │ │
│  │                                                             │ │
│  │  권한 표현:                                                               │
│  │  -rw-r--r-- 1 user group 1234 Mar 8 10:00 file.txt                   │ │
│  │  │├┬┤├┬┤├┬┤                                                          │ │
│  │  ││ │ │ │ │ │                                                          │ │
│  │  ││ │ │ │ │ └── 기타(o): r-- (읽기만)                                    │ │
│  │  ││ │ │ │ └──── 그룹(g): r-- (읽기만)                                    │ │
│  │  ││ │ │ └────── 소유자(u): rw- (읽기, 쓰기)                              │ │
│  │  ││ │ └──────── 파일 타입: - (일반 파일)                                  │ │
│  │  ││ └────────── 파일 타입: d (디렉토리) 등                                │ │
│  │  │└──────────── 파일 타입 플래그                                          │ │
│  │  │                                                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【숫자 표기법 (8진수)】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  r = 4, w = 2, x = 1                                                   │ │
│  │                                                             │ │
│  │  rwx = 4 + 2 + 1 = 7                                                   │ │
│  │  rw- = 4 + 2 + 0 = 6                                                   │ │
│  │  r-x = 4 + 0 + 1 = 5                                                   │ │
│  │  r-- = 4 + 0 + 0 = 4                                                   │ │
│  │  -wx = 0 + 2 + 1 = 3                                                   │ │
│  │  -w- = 0 + 2 + 0 = 2                                                   │ │
│  │  --x = 0 + 0 + 1 = 1                                                   │ │
│  │  --- = 0 + 0 + 0 = 0                                                   │ │
│  │                                                             │ │
│  │  예: chmod 755 file.txt                                                │ │
│  │  7 (rwx) for owner, 5 (r-x) for group, 5 (r-x) for other              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 권한 상세                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【디렉토리 권한의 의미】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  r (읽기): 디렉토리 내용 나열 (ls)                                        │ │
│  │  w (쓰기): 디렉토리 내 파일 생성/삭제                                      │ │
│  │  x (실행): 디렉토리 진입 (cd), 파일 접근                                  │ │
│  │                                                             │ │
│  │  중요: 디렉토리에서 x 없이 r만 있으면 ls는 가능하지만                      │ │
│  │        파일 상세 정보(inode)는 볼 수 없음                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【특수 권한】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SUID (Set User ID, 4xxx):                                              │ │
│  │  • 실행 시 파일 소유자 권한으로 실행                                      │ │
│  │  • 예: /usr/bin/passwd (root 권한 필요)                                 │ │
│  │  • 표시: -rwS------ (s가 대문자면 x 없음)                                │ │
│  │                                                             │ │
│  │  SGID (Set Group ID, 2xxx):                                             │ │
│  │  • 실행 시 파일 그룹 권한으로 실행                                        │ │
│  │  • 디렉토리: 생성된 파일이 디렉토리 그룹 상속                             │ │
│  │  • 표시: -rw-rS---                                                      │ │
│  │                                                             │ │
│  │  Sticky Bit (1xxx):                                                     │ │
│  │  • /tmp 같은 공유 디렉토리에서 사용                                       │ │
│  │  • 파일 소유자만 삭제 가능                                               │ │
│  │  • 표시: drwxrwxrwt                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【umask】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  기본 권한:                                                               │ │
│  │  파일: 0666 - umask                                                     │ │
│  │  디렉토리: 0777 - umask                                                  │ │
│  │                                                             │ │
│  │  일반 umask: 022 → 파일: 644, 디렉토리: 755                              │ │
│  │  보안 umask: 077 → 파일: 600, 디렉토리: 700                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【권한 변경】                                                          │
│  ──────────────────                                                  │
│  // 숫자로 변경                                                          │
│  $ chmod 755 script.sh                                               │
│  $ chmod 644 file.txt                                                │
│  $ chmod 700 private_dir                                             │
│                                                                     │
│  // 심볼로 변경                                                          │
│  $ chmod u+x script.sh        // 소유자에 실행 권한 추가                │
│  $ chmod g-w file.txt         // 그룹에서 쓰기 권한 제거                │
│  $ chmod o+r file.txt         // 기타에 읽기 권한 추가                  │
│  $ chmod a+x script.sh        // 모두에게 실행 권한 추가                │
│  $ chmod ug=rw,o= file.txt    // 소유자,그룹 rw, 기타 없음              │
│                                                                     │
│  // 특수 권한                                                            │
│  $ chmod 4755 setuid_prog    // SUID + 755                             │
│  $ chmod 2755 shared_dir     // SGID + 755                             │
│  $ chmod 1777 /tmp          // Sticky + 777                            │
│                                                                     │
│  【소유권 변경】                                                        │
│  ──────────────────                                                  │
│  $ chown user file.txt        // 소유자 변경                            │
│  $ chown user:group file.txt  // 소유자, 그룹 변경                       │
│  $ chgrp group file.txt       // 그룹만 변경                            │
│  $ chown -R user:group dir/   // 재귀적 변경                            │
│                                                                     │
│  【umask 설정】                                                        │
│  ──────────────────                                                  │
│  $ umask                    // 현재 umask 확인                          │
│  0022                                                                 │
│  $ umask 077                 // 더 엄격하게 설정                         │
│                                                                     │
│  // /etc/profile 또는 ~/.bashrc에 설정                                  │
│  umask 027                                                            │
│                                                                     │
│  【프로그래밍에서 권한 다루기】                                            │
│  ──────────────────                                                  │
│  #include <sys/stat.h>                                               │
│  #include <sys/types.h>                                              │
│  #include <unistd.h>                                                 │
│                                                                     │
│  // 파일 생성 시 권한 설정                                               │
│  int fd = open("file.txt", O_CREAT | O_WRONLY, 0644);                │
│  creat("newfile.txt", 0644);                                          │
│  mkdir("newdir", 0755);                                              │
│                                                                     │
│  // 권한 변경                                                            │
│  chmod("file.txt", 0644);                                            │
│  chown("file.txt", uid, gid);                                        │
│                                                                     │
│  // umask 확인/설정                                                      │
│  mode_t old = umask(022);                                            │
│  // 파일 생성...                                                        │
│  umask(old);  // 복원                                                  │
│                                                                     │
│  // 권한 확인                                                            │
│  #include <sys/stat.h>                                               │
│  if (access("file.txt", R_OK) == 0)                                  │
│      printf("읽기 가능\n");                                            │
│  if (access("file.txt", W_OK) == 0)                                  │
│      printf("쓰기 가능\n");                                            │
│  if (access("file.txt", X_OK) == 0)                                  │
│      printf("실행 가능\n");                                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 접근 제어 메커니즘
• 분류: 소유자(u), 그룹(g), 기타(o)
• 권한: 읽기(r=4), 쓰기(w=2), 실행(x=1)
• 표기: 심볼(rwx), 8진수(755)
• 특수: SUID(4), SGID(2), Sticky(1)
• umask: 기본 권한 결정
• 명령: chmod, chown, chgrp
```
