# 447. 아이노드 (Inode)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 유닉스 파일의 메타데이터 구조
> 2. **가치**: 파일 식별과 데이터 분리
> 3. **융합**: 파일 시스템, 하드 링크와 연관

---

## Ⅰ. 개요

### 개념 정의

아이노드(Inode, Index Node)는 **유닉스 계열 파일 시스템에서 파일의 메타데이터를 저장하는 자료 구조**다. 파일 이름 외 모든 파일 정보를 포함한다.

### 💡 비유: 신분증
아이노드는 **신분증**과 같다. 이름(파일 이름)은 디렉토리에 있고, 신분증(아이노드)에 실제 정보가 있다.

### 아이노드 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                아이노드 구조                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【아이노드 vs 디렉토리 엔트리】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  디렉토리 엔트리:                                                        │ │
│  │  ┌───────────────────────────────────┐                              │ │
│  │  │ 파일 이름    │ 아이노드 번호         │                              │ │
│  │  │ document.txt│ 12345                │                              │ │
│  │  │ image.png   │ 12346                │                              │ │
│  │  └───────────────────────────────────┘                              │ │
│  │                                                             │ │
│  │  아이노드 (번호 12345):                                                   │ │
│  │  ┌───────────────────────────────────┐                              │ │
│  │  │ 파일 타입, 권한                      │                              │ │
│  │  │ 소유자, 그룹                         │                              │ │
│  │  │ 크기, 시간 정보                       │                              │ │
│  │  │ 데이터 블록 포인터들                  │                              │ │
│  │  │ 참조 수 (링크 수)                     │                              │ │
│  │  └───────────────────────────────────┘                              │ │
│  │                                                             │ │
│  │  핵심: 파일 이름은 아이노드에 없음!                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【아이노드 내부 구조】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct inode {                                                          │ │
│  │      mode_t    i_mode;     // 파일 타입 + 접근 권한                      │ │
│  │      uid_t     i_uid;      // 소유자 사용자 ID                            │ │
│  │      gid_t     i_gid;      // 소유자 그룹 ID                              │ │
│  │      off_t     i_size;     // 파일 크기 (바이트)                          │ │
│  │      time_t    i_atime;    // 마지막 접근 시간                           │ │
│  │      time_t    i_mtime;    // 마지막 수정 시간                           │ │
│  │      time_t    i_ctime;    // 마지막 상태 변경 시간                       │ │
│  │      nlink_t   i_nlink;    // 하드 링크 수                                │ │
│  │      blkcnt_t  i_blocks;   // 할당된 블록 수                                │ │
│  │      // 데이터 블록 포인터 (직접/간접/이중간접)                           │ │
│  │      u32       i_block[15];                                            │ │
│  │  };                                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【아이노드 테이블】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파일 시스템 생성 시 아이노드 테이블 생성:                                 │ │
│  │                                                             │ │
│  │  ┌─────┬─────┬─────┬─────┬─────┬─────┐                              │ │
│  │  │ #1  │ #2  │ #3  │ #4  │ #5  │ ... │                              │ │
│  │  │     │     │     │     │     │     │                              │ │
│  │  └─────┴─────┴─────┴─────┴─────┴─────┘                              │ │
│  │                                                             │ │
│  │  아이노드 번호로 아이노드 테이블 인덱싱                                     │ │
│  │  아이노드 번호는 파일 시스템 내에서 고유                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                아이노드 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【아이노드 특징】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                                   │ │
│  │  • 하드 링크 지원: 여러 이름이 같은 아이노드 참조                          │ │
│  │  • 파일 이동 효율: 디렉토리 엔트리만 변경                                   │ │
│  │  • 빠른 속성 접근: 아이노드에 모든 메타데이터                                │ │
│  │                                                             │ │
│  │  특징:                                                                   │ │
│  │  • 이름 없음: 파일 이름은 디렉토리 엔트리에만                               │ │
│  │  • 번호로 식별: 아이노드 번호가 파일의 진짜 식별자                          │ │
│  │  • 제한된 수: 파일 시스템 생성 시 개수 고정                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【데이터 블록 주소 지정】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  i_block 배열 (ext2/3/4):                                               │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │ [0-11]: 직접 블록 (Direct blocks)                        │    │ │
│  │  │ [12]: 단일 간접 블록 (Single indirect)                    │    │ │
│  │  │ [13]: 이중 간접 블록 (Double indirect)                    │    │ │
│  │  │ [14]: 삼중 간접 블록 (Triple indirect)                    │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  작은 파일: 직접 블록만 사용                                              │ │
│  │  큰 파일: 간접 블록으로 확장                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【아이노드 고갈】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제: 아이노드가 모두 사용 중인 경우                                        │ │
│  │                                                             │ │
│  │  • 디스크 공간 있어도 새 파일 생성 불가                                     │ │
│  │  • df -i로 아이노드 사용량 확인                                            │ │
│  │  • 파일 시스템 재생성으로 아이노드 수 늘림                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【아이노드 정보 확인】                                                  │
│  ──────────────────                                                  │
│  // ls -i로 아이노드 번호 확인                                            │
│  $ ls -la                                                            │
│  -rw-r--r-- 2 user group 100 Mar 8 10:00 file.txt                   │
│  // 링크 수=2 (하드 링크 2개)                                             │
│                                                                     │
│  // 아이노드 번호 확인                                                      │
│  $ ls -la                                                            │
│  12345 -rw-r--r-- 1 user group 100 Mar 8 10:00 file.txt              │
│  // 12345 = 아이노드 번호                                                  │
│                                                                     │
│  // stat으로 상세 확인                                                      │
│  $ stat file.txt                                                      │
│  Inode: 1234567                                                       │
│  Links: 1                                                             │
│                                                                     │
│  // 파일 시스템별 아이노드 사용량                                             │
│  $ df -iTh                                                           │
│  Filesystem     Type   Inodes  IUsed  IFree IUse% Mounted on         │
│  /dev/sda1      ext4   655360  50000 605360   8% /                  │
│  // IUse%가 높으면 아이노드 부족                                             │
│                                                                     │
│  【아이노드 관련 문제 해결】                                              │
│  ──────────────────                                                  │
│  // 아이노드 사용량 상세 확인                                                 │
│  $ sudo tune2fs -l /dev/sda1 | grep -i inode                       │
│  Inode count:              655360                                   │
│  Free inodes:              605360                                   │
│                                                                     │
│  // 많은 작은 파일 생성 → 아이노드 고갈                                     │
│  // 해결: 더 많은 아이노드로 파일시스템 재생성                                 │
│  $ mkfs.ext4 -N 1000000 /dev/sda1                                  │
│                                                                     │
│  【프로그래밍에서 아이노드】                                                │
│  ──────────────────                                                  │
│  #include <sys/stat.h>                                               │
│  #include <sys/types.h>                                              │
│                                                                     │
│  // 아이노드 정보 얻기                                                      │
│  struct stat st;                                                      │
│  if (stat("file.txt", &st) == 0) {                                   │
│      printf("Inode: %ld\n", st.st_ino);                              │
│      printf("Links: %ld\n", st.st_nlink);                            │
│      printf("Size: %ld\n", st.st_size);                              │
│  }                                                                   │
│                                                                     │
│  // 파일 타입 확인 (아이노드의 mode)                                        │
│  if (S_ISREG(st.st_mode))  printf("Regular file\n");                 │
│  if (S_ISDIR(st.st_mode))  printf("Directory\n");                    │
│  if (S_ISLNK(st.st_mode))  printf("Symbolic link\n");                │
│                                                                     │
│  【하드 링크와 아이노드】                                                  │
│  ──────────────────                                                  │
│  // 같은 파일, 다른 이름                                                   │
│  $ ln original.txt hardlink.txt                                     │
│  $ ls -la                                                            │
│  12345 -rw-r--r-- 2 user group 100 Mar 8 10:00 hardlink.txt         │
│  12345 -rw-r--r-- 2 user group 100 Mar 8 10:00 original.txt         │
│  // 같은 아이노드 번호 12345                                                │
│  // 링크 수 = 2                                                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
