# 460. 디스크 성능 (Disk Performance)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크 I/O 처리 능력
> 2. **가치**: 시스템 전체 성능의 병목
> 3. **융합**: 탐색 시간, 회전 지연, 대역폭과 연관

---

## Ⅰ. 개요

### 개념 정의

디스크 성능(Disk Performance)은 **디스크가 데이터를 얼마나 빠르게 읽고 쓸 수 있는지를 나타내는 지표들**이다.

### 💡 비유: 도로 용량
디스크 성능은 **도로 용량**과 같다. 차선 수(대역폭), 제한 속도(RPM), 교통 상황(큐)이 복합적으로 작용한다.

### 디스크 성능 지표

```
┌─────────────────────────────────────────────────────────────────────┐
│                디스크 성능 지표                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 성능 지표】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 접근 시간 (Access Time):                                         │ │
│  │     • 탐색 시간 + 회전 지연 + 전송 시간                                 │ │
│  │     • HDD: 5-15 ms                                                  │ │
│  │     • SSD: 0.1-0.5 ms                                               │ │
│  │                                                             │ │
│  │  2. 전송 속도 (Transfer Rate):                                       │ │
│  │     • 초당 전송 바이트 수                                              │ │
│  │     • 순차 읽기: 100-300 MB/s (HDD)                                  │ │
│  │     • SSD: 500-7000 MB/s                                            │ │
│  │                                                             │ │
│  │  3. IOPS (I/O Operations Per Second):                               │ │
│  │     • 초당 I/O 작업 수                                               │ │
│  │     • HDD: 80-160                                                   │ │
│  │     • SATA SSD: 80,000-100,000                                      │ │
│  │     • NVMe SSD: 500,000+                                            │ │
│  │                                                             │ │
│  │  4. 대기 시간 (Latency):                                             │ │
│  │     • 요청부터 완료까지 시간                                           │ │
│  │     • 큐 대기 + 접근 시간                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 계층 구조】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │              CPU / 메모리 (나노초)                          │    │ │
│  │  │              속도: 수십 GB/s                              │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │              캐시 (마이크로초)                              │    │ │
│  │  │              속도: 수 GB/s                                │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │              디스크 (밀리초) ★ 병목                          │    │ │
│  │  │              속도: 수백 MB/s                              │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  디스크가 가장 느림 → 시스템 병목                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  지표           HDD (7200)      SATA SSD      NVMe SSD         │ │

│  │  접근 시간      10-15 ms       0.1-0.2 ms    0.02-0.05 ms      │ │
│  │  순차 읽기      100-200 MB/s   500 MB/s      3500 MB/s         │ │
│  │  순차 쓰기      100-200 MB/s   500 MB/s      3300 MB/s         │ │
│  │  랜덤 읽기 IOPS  80-100        90,000        500,000           │ │
│  │  랜덤 쓰기 IOPS  80-100        80,000        500,000           │ │
│  │  지연 시간      5-10 ms        0.1 ms        0.02 ms           │ │
│  │                                                             │ │
│  │  SSD가 HDD 대비 100-1000배 빠름 (랜덤 I/O)                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 영향 요소】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 물리적 요소:                                                      │ │
│  │     • RPM (회전 속도)                                                 │ │
│  │     • 플래터 수, 밀도                                                 │ │
│  │     • 인터페이스 (SATA, SAS, NVMe)                                   │ │
│  │                                                             │ │
│  │  2. 워크로드:                                                         │ │
│  │     • 순차 vs 랜덤                                                    │ │
│  │     • 읽기 vs 쓰기 비율                                               │ │
│  │     • 블록 크기                                                       │ │
│  │                                                             │ │
│  │  3. 시스템 요소:                                                      │ │
│  │     • 큐 깊이 (Queue Depth)                                          │ │
│  │     • 캐시 크기                                                       │ │
│  │     • 스케줄러                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순차 vs 랜덤 I/O】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  순차 I/O:                                                            │ │
│  │  • 연속된 섹터 읽기/쓰기                                               │ │
│  │  • 탐색 시간 최소                                                     │ │
│  │  • HDD에서도 높은 성능                                                │ │
│  │                                                             │ │
│  │  랜덤 I/O:                                                            │ │
│  │  • 불연속 위치 접근                                                    │ │
│  │  • 매번 탐색 발생                                                     │ │
│  │  • HDD에서 성능 급감, SSD 유리                                        │ │
│  │                                                             │ │
│  │  예: HDD 순차 150 MB/s → 랜덤 1 MB/s (150배 차이)                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【성능 모니터링】                                                    │
│  ──────────────────                                                  │
│  // iostat로 I/O 통계 확인                                            │
│  $ iostat -x 1                                                      │
│  Device  r/s   w/s   rMB/s  wMB/s  await  svctm  %util              │
│  sda     100   50    10.0   5.0    5.00   2.00   30.0               │
│                                                                     │
│  // r/s, w/s: 초당 읽기/쓰기 IOPS                                      │
│  // await: 평균 대기 시간 (ms)                                         │
│  // svctm: 평균 서비스 시간 (ms)                                        │
│  // %util: 사용률                                                     │
│                                                                     │
│  // 실시간 I/O 확인                                                    │
│  $ iotop                                                            │
│                                                                     │
│  【벤치마크 도구】                                                      │
│  ──────────────────                                                  │
│  // fio - 유연한 I/O 테스터                                            │
│  $ fio --name=randread --ioengine=libaio --iodepth=16 \             │
│        --rw=randread --bs=4k --direct=1 --size=1G \                 │
│        --numjobs=4 --runtime=60 --filename=/dev/sda                 │
│                                                                     │
│  // 결과: IOPS, 대역폭, 지연 시간                                       │
│                                                                     │
│  // dd로 간단한 테스트                                                  │
│  $ dd if=/dev/zero of=testfile bs=1M count=1000                    │
│  1048576000 bytes (1.0 GB) copied, 5.2 s, 192 MB/s                  │
│                                                                     │
│  【성능 튜닝】                                                          │
│  ──────────────────                                                  │
│  // 1. I/O 스케줄러 변경                                               │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  noop deadline [cfq]                                                 │
│  $ echo deadline > /sys/block/sda/queue/scheduler                   │
│                                                                     │
│  // 2. Read-ahead 조정                                                │
│  $ cat /sys/block/sda/queue/read_ahead_kb                           │
│  128                                                                 │
│  $ echo 256 > /sys/block/sda/queue/read_ahead_kb                    │
│                                                                     │
│  // 3. 큐 깊이 조정                                                    │
│  $ cat /sys/block/sda/queue/nr_requests                             │
│  128                                                                 │
│                                                                     │
│  // 4. SSD인 경우 discard (TRIM) 활성화                                │
│  $ cat /sys/block/sda/queue/discard_granularity                     │
│                                                                     │
│  【프로그래밍에서 성능 고려】                                            │
│  ──────────────────                                                  │
│  // 버퍼 크기 최적화                                                    │
│  #define BUFFER_SIZE (64 * 1024)  // 64KB                            │
│                                                                     │
│  // 비동기 I/O                                                         │
│  #include <libaio.h>                                                 │
│  io_setup(128, &ctx);                                               │
│  io_submit(ctx, 1, &iocb);                                          │
│  io_getevents(ctx, 1, 1, &event, NULL);                             │
│                                                                     │
│  // O_DIRECT로 버퍼 캐시 우회                                           │
│  int fd = open("file", O_RDONLY | O_DIRECT);                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 지표: 접근 시간, 전송 속도, IOPS, 대기 시간
• HDD: 80-160 IOPS, 100-200 MB/s
• SSD: 80K-500K+ IOPS, 500-7000 MB/s
• 병목: 디스크가 가장 느린 구성 요소
• 순차: HDD에서도 빠름
• 랜덤: SSD가 압도적
• 모니터링: iostat, iotop, fio
• 튜닝: 스케줄러, read-ahead, 비동기 I/O
```
