# 430. 프리페칭 (Prefetching)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 필요하기 전에 미리 로드
> 2. **가치**: 페이지 폴트 감소, 성능 향상
> 3. **융합**: 요구 페이징, 지역성과 연관

---

## Ⅰ. 개요

### 개념 정의

프리페칭(Prefetching)은 **프로세스가 필요하기 전에 미리 페이지를 메모리에 로드하는 기법**이다. 페이지 폴트를 줄여 성능을 향상시킨다.

### 💡 비유: 미리 책상에 책 두기
프리페칭은 **미리 책상에 책 두기**와 같다. 필요할 것 같은 책을 미리 꺼내둔다. 나중에 바로 쓴다.

### 프리페칭 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                프리페칭 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Demand Paging:                                                         │
│  │  필요할 때만 로드 → 많은 페이지 폴트                                      │ │
│  │                                                             │ │
│  │  Prefetching:                                                            │
│  │  필요할 것 같은 페이지 미리 로드 → 페이지 폴트 감소                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순차 접근 프리페칭】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파일을 순차로 읽을 때:                                                   │
│  │                                                             │ │
│  │  페이지 N 접근 → 페이지 N+1, N+2, N+3 미리 로드                          │ │
│  │                                                             │ │
│  │  ┌─────┬─────┬─────┬─────┬─────┬─────┐                          │ │
│  │  │  0  │  1  │  2  │  3  │  4  │  5  │                          │ │
│  │  └─────┴─────┴─────┴─────┴─────┴─────┘                          │ │
│  │     ↑                    ↑                                          │ │
│  │   접근              프리페치                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프리페칭 종류】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 순차 프리페칭 (Sequential Prefetch):                                 │ │
│  │     • 파일 순차 읽기 시 다음 페이지 미리 로드                              │ │
│  │     • read-ahead                                                          │
│  │                                                             │ │
│  │  2. 클러스터 프리페칭 (Cluster Prefetch):                                │ │
│  │     • 접근한 페이지 주변 페이지들 로드                                     │ │
│  │     • 공간 지역성 활용                                                    │ │
│  │                                                             │ │
│  │  3. 스마트 프리페칭 (Smart Prefetch):                                    │
│  │     • 접근 패턴 분석                                                      │ │
│  │     • 머신러닝 기반 예측                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                프리페칭 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 페이지 폴트 감소: 필요할 때 이미 메모리에 있음                          │ │
│  │  • I/O 병렬화: 백그라운드에서 미리 로드                                   │ │
│  │  • 응답 시간 단축: 대기 시간 감소                                        │ │
│  │  • 지역성 활용: 공간 지역성 기반                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 메모리 낭비: 실제로 안 쓰는 페이지 로드                                 │ │
│  │  • I/O 대역폭: 불필요한 I/O 발생                                         │ │
│  │  • 캐시 오염: 필요한 페이지 밀려남                                        │ │
│  │  • 예측 오류: 잘못된 프리페치는 비용만 발생                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux Read-Ahead】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Linux의 프리페칭 구현:                                                  │ │
│  │                                                             │ │
│  │  • 순차 읽기 감지                                                        │ │
│  │  • 자동으로 read-ahead 수행                                             │ │
│  │  • 동적으로 read-ahead 크기 조정                                        │ │
│  │  • random 읽기는 프리페칭 안 함                                          │ │
│  │                                                             │ │
│  │  기본 read-ahead: 128KB                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux Read-Ahead 설정】                                            │
│  ──────────────────                                                  │
│  // read-ahead 크기 확인                                                │
│  $ blockdev --getra /dev/sda                                        │
│  256  // 256 × 512바이트 = 128KB                                       │
│                                                                     │
│  // read-ahead 크기 변경                                               │
│  $ blockdev --setra 512 /dev/sda  // 256KB                           │
│                                                                     │
│  // 장치별 설정                                                          │
│  $ cat /sys/block/sda/queue/read_ahead_kb                           │
│  128                                                                 │
│                                                                     │
│  $ echo 256 > /sys/block/sda/queue/read_ahead_kb                    │
│                                                                     │
│  【프로그래밍에서 프리페칭】                                            │
│  ──────────────────                                                  │
│  // posix_fadvise로 힌트 제공                                          │
│  #include <fcntl.h>                                                   │
│                                                                     │
│  int fd = open("bigfile.bin", O_RDONLY);                             │
│                                                                     │
│  // 순차 읽기 힌트 (프리페칭 활성화)                                      │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_SEQUENTIAL);                     │
│                                                                     │
│  // 미리 로드 요청                                                       │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_WILLNEED);                       │
│                                                                     │
│  // 랜덤 읽기 힌트 (프리페칭 비활성화)                                    │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_RANDOM);                        │
│                                                                     │
│  // 캐시에서 제거                                                        │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_DONTNEED);                       │
│                                                                     │
│  【수동 프리페칭】                                                      │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│                                                                     │
│  // mmap + madvise                                                     │
│  void *map = mmap(NULL, size, PROT_READ, MAP_PRIVATE, fd, 0);        │
│  madvise(map, size, MADV_SEQUENTIAL);  // 순차 접근 힌트               │
│  madvise(map, size, MADV_WILLNEED);    // 미리 로드                   │
│  madvise(map, size, MADV_DONTNEED);    // 캐시 해제                   │
│                                                                     │
│  // readahead 시스템 콜                                                  │
│  #include <linux/fadvise.h>                                          │
│  readahead(fd, offset, size);                                         │
│                                                                     │
│  【성능 측정】                                                          │
│  ──────────────────                                                  │
│  // 프리페칭 효과 비교                                                   │
│  $ time dd if=bigfile.bin of=/dev/null bs=4k                        │
│  // 프리페칭 없이                                                         │
│                                                                     │
│  $ time dd if=bigfile.bin of=/dev/null bs=4k iflag=sequential       │
│  // 프리페칭 있이                                                         │
│                                                                     │
│  // 페이지 폴트 측정                                                      │
│  $ perf stat -e page-faults,major-faults ./program                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 필요하기 전에 미리 페이지 로드
• 종류: 순차, 클러스터, 스마트 프리페칭
• 장점: 페이지 폴트 감소, 성능 향상
• 단점: 메모리 낭비, 예측 오류 시 비용
• Linux: read-ahead로 자동 수행
• API: posix_fadvise, madvise
• 힌트: SEQUENTIAL, WILLNEED, RANDOM
```
