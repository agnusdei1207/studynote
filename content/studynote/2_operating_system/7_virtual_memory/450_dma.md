# 450. 블록과 섹터 (Block and Sector)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 저장 장치의 데이터 단위
> 2. **가치**: I/O 효율성과 주소 지정
> 3. **융합**: 파일 시스템, 디스크와 연관

---

## Ⅰ. 개요

### 개념 정의

- **섹터(Sector)**: 디스크의 물리적 저장 단위 (보통 512B)
- **블록(Block)**: 파일 시스템의 논리적 저장 단위 (보통 4KB)

### 💡 비유: 책과 페이지
섹터는 **페이지**, 블록은 **장**과 같다. 페이지는 책의 물리적 단위, 장은 내용의 논리적 단위다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                블록과 섹터 상세                                         │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【섹터 상세】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  물리적 특성:                                                            │ │
│  │  • 디스크 드라이브의 최소 단위                                             │ │
│  │  • 하드디스크: 512B (전통) 또는 4KB (AF)                                  │ │
│  │  • SSD: 4KB 이상 (내부 관리 단위)                                         │ │
│  │                                                             │ │
│  │  LBA (Logical Block Addressing):                                        │ │
│  │  • 섹터 번호로 주소 지정                                                  │ │
│  │  • LBA 0, LBA 1, LBA 2, ...                                            │ │
│  │  • 48비트 LBA: 128PB까지 주소 지정                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【블록 상세】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  논리적 특성:                                                            │ │
│  │  • 파일 시스템의 최소 할당 단위                                             │ │
│  │  • ext4 기본: 4KB                                                       │ │
│  │  • mkfs 시 블록 크기 지정 가능                                            │ │
│  │                                                             │ │
│  │  블록 크기 선택 기준:                                                      │ │
│  │  • 작은 파일 많음 → 작은 블록 (1KB, 2KB)                                  │ │
│  │  • 큰 파일 많음 → 큰 블록 (8KB, 16KB)                                    │ │
│  │  • 트레이드오프: 내부 단편화 vs 메타데이터 크기                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【내부 단편화】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  블록 크기 4KB, 파일 크기 1KB인 경우:                                      │ │
│  │  ┌───────────────────────────────────┐                              │ │
│  │  │ 사용 (1KB)  │ 낭비 (3KB)           │                              │ │
│  │  │ ████████    │ ░░░░░░░░░░░░░░░░░░░ │                              │ │
│  │  └───────────────────────────────────┘                              │ │
│  │                                                             │ │
│  │  평균 낭비: 블록 크기의 절반                                               │ │
│  │  작은 파일 많으면 낭비 증가                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【블록 크기 확인】                                                      │
│  ──────────────────                                                  │
│  // 파일 시스템 블록 크기                                                 │
│  $ stat -f / | grep "Block size"                                    │
│  Block size: 4096                                                    │
│                                                                     │
│  // tune2fs로 상세 확인                                                   │
│  $ tune2fs -l /dev/sda1 | grep "Block size"                         │
│  Block size:               4096                                      │
│                                                                     │
│  // 물리적 섹터 크기                                                       │
│  $ cat /sys/block/sda/queue/logical_block_size                       │
│  512                                                                  │
│  $ cat /sys/block/sda/queue/physical_block_size                      │
│  4096                                                                 │
│                                                                     │
│  【파일 시스템 생성 시 블록 크기 지정】                                    │
│  ──────────────────                                                  │
│  // 1KB 블록으로 생성                                                     │
│  $ mkfs.ext4 -b 1024 /dev/sda1                                      │
│                                                                     │
│  // 4KB 블록으로 생성 (기본)                                               │
│  $ mkfs.ext4 -b 4096 /dev/sda1                                      │
│                                                                     │
│  // 64KB 스트라이프 (RAID용)                                              │
│  $ mkfs.ext4 -b 4096 -E stride=16,stripe-width=16 /dev/md0           │
│                                                                     │
│  【섹터 읽기/쓰기 (dd)】                                                  │
│  ──────────────────                                                  │
│  // 섹터 단위 읽기                                                          │
│  $ dd if=/dev/sda of=sector.bin bs=512 count=1 skip=0               │
│  // 섹터 0 읽기 (MBR)                                                      │
│                                                                     │
│  // 블록 단위 읽기                                                          │
│  $ dd if=/dev/sda of=block.bin bs=4096 count=1 skip=0               │
│  // 블록 0 읽기                                                            │
│                                                                     │
│  【프로그래밍에서 블록/섹터】                                              │
│  ──────────────────                                                  │
│  #include <sys/statvfs.h>                                            │
│                                                                     │
│  // 블록 크기 확인                                                        │
│  struct statvfs sv;                                                  │
│  statvfs("/", &sv);                                                  │
│  printf("Block size: %lu\n", sv.f_bsize);                            │
│  printf("Fragment size: %lu\n", sv.f_frsize);                        │
│                                                                     │
│  // 직접 섹터 접근                                                        │
│  #include <fcntl.h>                                                   │
│  #include <unistd.h>                                                 │
│                                                                     │
│  int fd = open("/dev/sda", O_RDONLY);                                │
│  char sector[512];                                                   │
│  lseek(fd, 0, SEEK_SET);  // 섹터 0                                    │
│  read(fd, sector, 512);                                              │
│  close(fd);                                                          │
│                                                                     │
│  【성능 고려사항】                                                        │
│  ──────────────────                                                  │
│  // 정렬된 I/O (성능 향상)                                                 │
│  // 블록 크기의 배수로 버퍼와 오프셋 정렬                                    │
│  void *buf;                                                          │
│  posix_memalign(&buf, 4096, 4096);  // 4KB 정렬                       │
│  // I/O 작업...                                                        │
│  free(buf);                                                          │
│                                                                     │
│  // 직접 I/O (버퍼 캐시 우회)                                              │
│  int fd = open("file", O_RDONLY | O_DIRECT);                         │
│  // 블록 크기 정렬 필요                                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 섹터: 디스크 물리적 저장 단위 (512B/4KB)
• 블록: 파일 시스템 논리적 저장 단위 (4KB)
• 관계: 1 블록 = 여러 섹터
• 주소: 섹터는 LBA, 블록은 블록 번호
• 단편화: 블록 크기 클수록 내부 단편화 증가
• 선택: 파일 크기 패턴에 따라 결정
• 확인: stat -f, tune2fs -l
```
