# 405. 메모리 사상 파일 (Memory-Mapped File)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일을 메모리처럼 접근하는 기법
> 2. **가치**: 파일 I/O 간소화와 성능 향상
> 3. **융합**: 가상 메모리, 페이지 공유와 연관

---

## Ⅰ. 개요

### 개념 정의

메모리 사상 파일(Memory-Mapped File)은 **파일의 내용을 프로세스의 가상 주소 공간에 매핑하는 기법**이다. 파일을 마치 메모리처럼 읽고 쓸 수 있다.

### 💡 비유: 투명 창
메모리 사상 파일은 **투명 창**과 같다. 창 밖 풍경(파일)을 창 안(메모리)에서 본다. 실제로는 밖에 있다.

### 메모리 사상 파일 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 사상 파일 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  프로세스 주소 공간                                                     │ │
│  │  ┌─────────────────────────────────────────────────────────────┐│ │
│  │  │                                                             │ │
│  │  │  ┌─────┬─────┬─────┬─────┬─────┐                            │ │
│  │  │  │ P0  │ P1  │ P2  │ P3  │ ... │  ← 페이지                   │ │
│  │  │  └─────┴─────┴─────┴─────┴─────┘                            │ │
│  │  │     │     │     │     │                                        │ │
│  │  │     │     │     │     │                                        │ │
│  │  └─────┼─────┼─────┼─────┼────────────────────────────────────────┘│ │
│  │        │     │     │     │                                        │ │
│  │        │     │     │     │  페이지 폴트 시                          │ │
│  │        │     │     │     │  파일에서 로드                            │ │
│  │        ▼     ▼     ▼     ▼                                        │ │
│  │  ┌─────────────────────────────────────────────────────────────┐│ │
│  │  │                    디스크 파일                                ││ │
│  │  │  ┌─────┬─────┬─────┬─────┬─────┐                              ││ │
│  │  │  │ B0  │ B1  │ B2  │ B3  │ ... │  ← 파일 블록                  ││ │
│  │  │  └─────┴─────┴─────┴─────┴─────┘                              ││ │
│  │  └─────────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【전통적 파일 I/O vs mmap】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전통적 파일 I/O:                                                       │ │
│  │  read(fd, buffer, size)  → 디스크 → 커널 버퍼 → 사용자 버퍼            │ │
│  │  write(fd, buffer, size) → 사용자 버퍼 → 커널 버퍼 → 디스크            │ │
│  │  (2번 복사!)                                                           │ │
│  │                                                             │ │
│  │  mmap:                                                                 │ │
│  │  ptr = mmap(file)        → 디스크 → 사용자 메모리                       │ │
│  │  ptr[0] = 'x'            → 사용자 메모리 (수정)                         │ │
│  │  msync(ptr)              → 사용자 메모리 → 디스크                       │ │
│  │  (1번 복사!)                                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 사상 파일 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【mmap 종류】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 파일 매핑 (File-backed mapping):                                   │ │
│  │     • 실제 파일과 연결                                                  │ │
│  │     • 수정 시 파일에 반영                                               │ │
│  │     • MAP_SHARED / MAP_PRIVATE                                        │ │
│  │                                                             │ │
│  │  2. 익명 매핑 (Anonymous mapping):                                     │ │
│  │     • 파일 없음                                                        │ │
│  │     • 힙 메모리처럼 사용                                                │ │
│  │     • MAP_ANONYMOUS                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【공유 vs 비공유 매핑】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  MAP_SHARED:                                                           │ │
│  │  • 수정이 파일과 다른 프로세스에 보임                                    │ │
│  │  • IPC에 활용                                                          │ │
│  │                                                             │ │
│  │  MAP_PRIVATE:                                                          │ │
│  │  • 수정이 이 프로세스에만 보임 (COW)                                    │ │
│  │  • 파일은 변경되지 않음                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                                 │ │
│  │  • 사용자 공간에서 파일 접근                                            │ │
│  │  • read/write 시스템 콜 오버헤드 없음                                  │ │
│  │  • 페이지 캐시 자동 활용                                               │ │
│  │  • 큰 파일 처리에 효율적                                               │ │
│  │                                                             │ │
│  │  단점:                                                                 │ │
│  │  • 매핑 오버헤드                                                       │ │
│  │  • 주소 공간 낭비 가능                                                 │ │
│  │  • 작은 파일에는 비효율적                                              │ │
│  │  • 동기화 주의 필요                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C mmap】                                                               │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│  #include <fcntl.h>                                                   │
│  #include <unistd.h>                                                 │
│                                                                     │
│  // 파일 열기                                                            │
│  int fd = open("data.bin", O_RDWR);                                 │
│  off_t size = lseek(fd, 0, SEEK_END);                               │
│                                                                     │
│  // 메모리 매핑                                                          │
│  void* map = mmap(NULL, size, PROT_READ | PROT_WRITE,               │
│                    MAP_SHARED, fd, 0);                              │
│  close(fd);  // 매핑 후에는 fd 닫아도 됨                                 │
│                                                                     │
│  // 사용 (마치 배열처럼)                                                  │
│  char* data = (char*)map;                                           │
│  data[0] = 'H';                                                     │
│  data[1] = 'i';                                                     │
│                                                                     │
│  // 동기화 (선택적)                                                       │
│  msync(map, size, MS_SYNC);                                         │
│                                                                     │
│  // 해제                                                                │
│  munmap(map, size);                                                 │
│                                                                     │
│  【Java MappedByteBuffer】                                                │
│  ──────────────────                                                  │
│  import java.nio.MappedByteBuffer;                                  │
│  import java.nio.channels.FileChannel;                              │
│  import java.io.RandomAccessFile;                                   │
│                                                                     │
│  RandomAccessFile file = new RandomAccessFile("data.bin", "rw");  │
│  FileChannel channel = file.getChannel();                           │
│                                                                     │
│  MappedByteBuffer buffer = channel.map(                             │
│      FileChannel.MapMode.READ_WRITE, 0, channel.size());           │
│                                                                     │
│  // 사용                                                                │
│  buffer.put(0, (byte)'H');                                          │
│  buffer.put(1, (byte)'i');                                          │
│                                                                     │
│  // 강제 동기화                                                          │
│  buffer.force();                                                    │
│                                                                     │
│  channel.close();                                                   │
│  file.close();                                                      │
│                                                                     │
│  【Python mmap】                                                           │
│  ──────────────────                                                  │
│  import mmap                                                         │
│                                                                     │
│  with open("data.bin", "r+b") as f:                                 │
│      mm = mmap.mmap(f.fileno(), 0)                                  │
│                                                                     │
│      # 사용                                                            │
│      mm[0:2] = b"Hi"                                                │
│                                                                     │
│      # 읽기                                                            │
│      print(mm[0:10])                                                │
│                                                                     │
│      mm.close()                                                     │
│                                                                     │
│  【메모리 매핑 확인】                                                      │
│  ──────────────────                                                  │
│  // 프로세스의 메모리 맵 확인                                               │
│  $ cat /proc/<pid>/maps | head -10                                  │
│  00400000-0040b000 r-xp 00000000 08:01 12345 /bin/ls                │
│  0060a000-0060b000 r--p 0000a000 08:01 12345 /bin/ls                │
│  0060b000-0060c000 rw-p 0000b000 08:01 12345 /bin/ls                │
│  //                      ^ 파일 매핑                                     │
│                                                                     │
│  // pmap으로 상세 확인                                                   │
│  $ pmap -x <pid>                                                    │
│  Address           Kbytes     RSS   Dirty Mode  Mapping             │
│  0000000000400000    44      44       0 r-x-- ls                    │
│  0000000000600000     4       4       4 r---- ls                    │
│  0000000000601000     4       4       4 rw--- ls                    │
│                                                                     │
│  【공유 메모리로 활용】                                                      │
│  ──────────────────                                                  │
│  // 여러 프로세스가 같은 파일을 MAP_SHARED로 매핑                            │
│  // 프로세스 A                                                           │
│  int fd = open("/tmp/shared", O_RDWR | O_CREAT, 0666);             │
│  ftruncate(fd, 4096);                                               │
│  void* ptr = mmap(NULL, 4096, PROT_READ | PROT_WRITE,               │
│                    MAP_SHARED, fd, 0);                              │
│  strcpy(ptr, "Hello from A");                                       │
│                                                                     │
│  // 프로세스 B (다른 프로세스)                                              │
│  int fd = open("/tmp/shared", O_RDWR);                              │
│  void* ptr = mmap(NULL, 4096, PROT_READ | PROT_WRITE,               │
│                    MAP_SHARED, fd, 0);                              │
│  printf("%s\n", ptr);  // "Hello from A" 출력                        │
│                                                                     │
│  【성능 비교】                                                              │
│  ──────────────────                                                  │
│  // 대용량 파일 읽기                                                       │
│  // read() 방식                                                         │
│  while (read(fd, buffer, 4096) > 0) {                               │
│      process(buffer);                                               │
│  }                                                                  │
│  // 시스템 콜 오버헤드 발생                                                 │
│                                                                     │
│  // mmap 방식                                                           │
│  void* map = mmap(NULL, size, PROT_READ, MAP_PRIVATE, fd, 0);      │
│  for (off_t i = 0; i < size; i += 4096) {                          │
│      process(map + i);                                               │
│  }                                                                  │
│  // 시스템 콜 없음, 페이지 폴트만 발생                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일을 메모리처럼 접근
• 종류: 파일 매핑, 익명 매핑
• 모드: MAP_SHARED, MAP_PRIVATE
• 장점: 시스템 콜 감소, 페이지 캐시 활용
• API: mmap(), munmap(), msync()
• 활용: 대용량 파일, 공유 메모리, IPC
```
