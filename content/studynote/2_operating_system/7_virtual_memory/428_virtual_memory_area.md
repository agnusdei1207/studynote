# 428. Aging 알고리즘 (Aging Algorithm)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 참조 비트를 누적하여 LRU 근사
> 2. **가치**: 정밀한 사용 빈도 추적
> 3. **융합**: 페이지 교체, 카운터와 연관

---

## Ⅰ. 개요

### 개념 정의

Aging 알고리즘은 **각 페이지에 카운터를 두고 참조 비트를 누적하여 LRU를 근사하는 알고리즘**이다. 정기적으로 카운터를 시프트하고 참조 비트를 추가한다.

### 💡 비유: 나이 먹기
Aging은 **나이 먹기**와 같다. 활동하면(참조) 나이가 젊어진다. 안 하면 나이를 먹는다. 가장 나이 많은 것을 교체한다.

│  │                                                             │ │
│  │  R: 참조 비트, C: 카운터                                               │ │

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Aging 알고리즘 구현】                                                │
│  ──────────────────                                                  │
│  #include <stdio.h>                                                  │
│  #include <stdint.h>                                                 │
│  #include <limits.h>                                                 │
│                                                                     │
│  #define COUNTER_BITS 8                                              │
│  #define NUM_FRAMES 4                                                │
│                                                                     │
│  typedef struct {                                                    │
│      int page;                                                       │
│      uint8_t counter;  // 8비트 카운터                                 │
│      int ref_bit;                                                    │
│  } Frame;                                                            │
│                                                                     │
│  Frame frames[NUM_FRAMES];                                           │
│  int aging_faults = 0;                                               │
│                                                                     │
│  // 클럭 인터럽트마다 호출                                                │
│  void update_counters() {                                            │
│      for (int i = 0; i < NUM_FRAMES; i++) {                         │
│          // 오른쪽으로 1비트 시프트                                        │
│          frames[i].counter >>= 1;                                    │
│          // 참조 비트를 최상위 비트에 추가                                  │
│          if (frames[i].ref_bit) {                                    │
│              frames[i].counter |= (1 << (COUNTER_BITS - 1));         │
│              frames[i].ref_bit = 0;  // 리셋                          │
│          }                                                           │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  // 가장 작은 카운터의 페이지 찾기                                        │
│  int find_victim() {                                                 │
│      int min_idx = 0;                                                │
│      uint8_t min_counter = UCHAR_MAX;                                │
│      for (int i = 0; i < NUM_FRAMES; i++) {                         │
│          if (frames[i].counter < min_counter) {                      │
│              min_counter = frames[i].counter;                        │
│              min_idx = i;                                            │
│          }                                                           │
│      }                                                               │
│      return min_idx;                                                 │
│  }                                                                   │
│                                                                     │
│  【실제 시스템에서의 Aging】                                             │
│  ──────────────────                                                  │
│  // Linux의 페이지 에이징                                               │
│  // 페이지가 참조되면 참조 비트 설정                                       │
│  // kswapd가 정기적으로 스캔하며 에이징 수행                               │
│                                                                     │
│  // 페이지 에이징 관련 설정                                               │
│  $ cat /proc/sys/vm/page-aging-ratio                                │
│  // 페이지 노화 비율                                                     │
│                                                                     │
│  // 활성/비활성 리스트 이동                                               │
│  $ cat /proc/vmstat | grep pgactivate                               │
│  pgactivate 12345                                                   │
│  // 비활성 → 활성으로 이동한 페이지 수                                     │
│                                                                     │
│  【성능 측정】                                                          │
│  ──────────────────                                                  │
│  // Aging vs 다른 알고리즘 비교                                          │
│  $ ./bench_page_replace -a aging,lru,clock -f trace.txt             │
│                                                                     │
│  Algorithm   Faults   Overhead                                      │
│  Clock         8      Low                                           │
│  Aging         7      Medium                                        │
│  LRU           7      High                                          │
│  Optimal       6      N/A                                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 참조 비트를 누적하여 LRU 근사
• 구조: n비트 카운터 + 정기적 시프트
• 동작: 시프트 후 참조 비트를 MSB에 추가
• 교체: 가장 작은 카운터 페이지 선택
• 장점: 정밀한 LRU 근사, 구현 용이
• 단점: 정밀도 제한, 오버플로우 가능
• 위치: Clock < Aging < LRU (정확도)
```
