# 436. 슬랩 할당자 (Slab Allocator)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 객체를 위한 메모리 할당자
> 2. **가치**: 빠른 할당, 단편화 최소화
> 3. **융합**: 메모리 풀, 캐시와 연관

---

## Ⅰ. 개요

### 개념 정의

슬랩 할당자(Slab Allocator)는 **커널에서 자주 사용하는 객체를 효율적으로 할당하기 위한 메모리 관리 기법**이다. 객체별 캐시를 사용한다.

### 💡 비유: 서랍 정리
슬랩 할당자는 **서랍 정리**와 같다. 양말 서랍, 옷 서랍처럼 종류별로 구분한다. 필요한 걸 바로 꺼낸다.

### 슬랩 할당자 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                슬랩 할당자 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Cache (객체 타입별):                                                   │ │
│  │  ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐ │ │
│  │  │ task_struct     │ │ inode           │ │ dentry          │ │ │
│  │  │ Cache           │ │ Cache           │ │ Cache           │ │ │
│  │  └────────┬────────┘ └────────┬────────┘ └────────┬────────┘ │ │
│  │           │                   │                   │           │ │
│  │           ▼                   ▼                   ▼           │ │
│  │  Slab (실제 메모리):                                                    │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │  [obj][obj][obj][obj] [obj][obj][obj][obj]              │  │ │
│  │  │      Slab 1                Slab 2                       │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【슬랩 상태】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Full Slabs:   모든 객체 사용 중 (free 없음)                             │ │
│  │  Partial Slabs: 일부 사용, 일부 free (할당/해제 대상)                     │ │
│  │  Empty Slabs:  모든 객체 free (메모리 회수 대상)                          │ │
│  │                                                             │ │
│  │  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐          │ │
│  │  │   Full       │→│   Partial    │→│   Empty      │          │ │
│  │  │ ████████████ │ │ ████░░░░░░░░ │ │ ░░░░░░░░░░░░ │          │ │
│  │  │ 모두 사용중   │ │ 일부 사용중   │ │ 모두 비었음   │          │ │
│  │  └──────────────┘ └──────────────┘ └──────────────┘          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 빠른 할당: Free 리스트에서 O(1)                                       │ │
│  │  • 단편화 최소화: 같은 크기 객체만                                       │ │
│  │  • 캐시 효율: 객체가 연속 배치                                            │ │
│  │  • 초기화 생략: 생성자/소멸자로 재사용                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                슬랩 할당자 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 슬랩 할당자 종류】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. SLAB:                                                               │ │
│  │     • 초기 구현                                                          │ │
│  │     • 복잡하지만 기능 풍부                                               │ │
│  │                                                             │ │
│  │  2. SLUB (기본):                                                         │ │
│  │     • 간소화된 구현                                                      │ │
│  │     • 대규모 시스템에 적합                                               │ │
│  │     • 현재 Linux 기본                                                   │ │
│  │                                                             │ │
│  │  3. SLOB:                                                                │ │
│  │     • 매우 간단한 구현                                                   │ │
│  │     • 임베디드/소규모 시스템용                                           │ │
│  │     • 메모리 효율 중심                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 캐시들】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  캐시 이름          객체 크기    용도                                 │ │
│  │  ────────────────  ─────────  ─────────────────                  │ │
│  │  task_struct       ~7KB      프로세스 정보                          │ │
│  │  inode             ~600B     파일 정보                              │ │
│  │  dentry            ~200B     디렉토리 엔트리                        │ │
│  │  filp              ~200B     열린 파일                              │ │
│  │  mm_struct         ~1KB      메모리 관리                            │ │
│  │  kmalloc-8~8192    8~8192    일반 할당                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【kmalloc 슬랩】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  2의 거듭제곱 크기별 캐시:                                               │ │
│  │  kmalloc-8, kmalloc-16, kmalloc-32, ..., kmalloc-8192               │ │
│  │                                                             │ │
│  │  요청 시: 가장 가까운 크기의 캐시에서 할당                                │ │
│  │  예: 100바이트 요청 → kmalloc-128에서 할당                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【슬랩 정보 확인】                                                      │
│  ──────────────────                                                  │
│  // 슬랩 캐시 목록                                                       │
│  $ cat /proc/slabinfo | head -20                                    │
│  slabinfo - version: 2.1                                            │
│  # name            <active_objs> <num_objs> <objsize> <objperslab>  │
│  task_struct          1234         1500      7168      4           │
│  inode_cache          5678         6000       600     54           │
│  dentry               8901        10000       192     42           │
│  kmalloc-8           12345        15000         8   512           │
│  kmalloc-16           8901        10000        16   256           │
│  kmalloc-32           4567         5000        32   128           │
│                                                                     │
│  // 슬랩 메모리 요약                                                      │
│  $ cat /proc/meminfo | grep -E "Slab|SReclaimable|SUnreclaim"      │
│  Slab:            123456 kB                                         │
│  SReclaimable:     67890 kB  // 회수 가능                             │
│  SUnreclaim:       55566 kB  // 회수 불가                             │
│                                                                     │
│  【슬랩 통계 상세】                                                      │
│  ──────────────────                                                  │
│  $ slabtop                                                          │
│  Active / Total Objects (% used)    : 123456 / 150000 (82.3%)      │
│  Active / Total Slabs (% used)      : 12345 / 15000 (82.3%)        │
│  Active / Total Caches (% used)     : 100 / 150 (66.7%)            │
│                                                                     │
│  OBJS ACTIVE  USE OBJ SIZE  SLABS OBJ/SLAB CACHE SIZE NAME         │
│  15000  12345  82%    0.19K   1000       15      8000K dentry      │
│  10000   8901  89%    0.10K    500       20      2000K buffer_head │
│                                                                     │
│  【커널 모듈에서 슬랩 사용】                                            │
│  ──────────────────                                                  │
│  // 슬랩 캐시 생성                                                       │
│  struct kmem_cache *my_cache;                                        │
│  my_cache = kmem_cache_create("my_objects",                         │
│                                sizeof(struct my_obj),               │
│                                0,                                   │
│                                SLAB_HWCACHE_ALIGN,                  │
│                                NULL);                               │
│                                                                     │
│  // 객체 할당                                                          │
│  struct my_obj *obj = kmem_cache_alloc(my_cache, GFP_KERNEL);        │
│                                                                     │
│  // 객체 해제                                                          │
│  kmem_cache_free(my_cache, obj);                                     │
│                                                                     │
│  // 캐시 삭제                                                          │
│  kmem_cache_destroy(my_cache);                                        │
│                                                                     │
│  【kmalloc 사용】                                                       │
│  ──────────────────                                                  │
│  // 커널에서 일반 메모리 할당                                            │
│  void *ptr = kmalloc(size, GFP_KERNEL);                              │
│  kfree(ptr);                                                         │
│                                                                     │
│  // 크기별 kmalloc 슬랩에서 할당됨                                       │
│  // size=100 → kmalloc-128                                           │
│  // size=200 → kmalloc-256                                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 커널 객체를 위한 메모리 할당자
• 구조: Cache → Slab → Object
• 상태: Full, Partial, Empty
• 종류: SLAB, SLUB(기본), SLOB
• 장점: 빠름, 단편화 최소, 캐시 효율
• kmalloc: 일반 커널 메모리 할당
• 확인: /proc/slabinfo, slabtop
```
