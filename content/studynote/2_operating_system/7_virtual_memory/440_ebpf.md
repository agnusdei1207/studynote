# 440. 메모리 벌루닝 (Memory Ballooning)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상머신 메모리 동적 조정
> 2. **가치**: 메모리 오버커밋 활성화
> 3. **융합**: 가상화, 가상 메모리와 연관

---

## Ⅰ. 개요

### 개념 정의

메모리 벌루닝(Memory Ballooning)은 **가상머신의 메모리를 동적으로 회수하거나 반환하는 가상화 기법**이다. 호스트가 게스트의 메모리를 조절한다.

### 💡 비유: 풍선 불기
메모리 벌루닝은 **풍선 불기**와 같다. 풍선을 부면(메모리 회수) 다른 곳의 공간이 늘어난다. 풍선을 빼면(메모리 반환) 다시 줄어든다.

### 메모리 벌루닝 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 벌루닝 구조                                       │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  가상화 환경에서:                                                        │ │
│  │  • 여러 VM이 같은 물리 메모리 공유                                        │ │
│  │  • VM이 사용 안 하는 메모리를 회수                                        │ │
│  │  • 필요한 VM에 메모리 재배분                                              │ │
│  │                                                             │ │
│  │  벌루닝:                                                                  │ │
│  │  • VM 내부의 "벌루운" 드라이버가 메모리 할당                               │ │
│  │  • 할당된 메모리는 VM이 사용 못 함                                        │ │
│  │  • 호스트는 해당 메모리를 다른 곳에 사용                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 과정】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  메모리 회수 (풍선 부풀리기):                                             │ │
│  │                                                             │ │
│  │  1. 호스트가 메모리 필요 감지                                             │ │
│  │  2. VM의 벌루운 드라이버에 요청                                           │ │
│  │  3. 벌루운 드라이버가 VM 내에서 메모리 할당                               │ │
│  │  4. 할당된 페이지를 호스트에 반환                                         │ │
│  │  5. VM의 가용 메모리 감소                                                │ │
│  │                                                             │ │
│  │  메모리 반환 (풍선 빼기):                                                 │ │
│  │                                                             │ │
│  │  1. VM이 더 많은 메모리 필요                                             │ │
│  │  2. 호스트가 벌루운에 메모리 반환                                         │ │
│  │  3. 벌루운이 할당했던 메모리 해제                                         │ │
│  │  4. VM의 가용 메모리 증가                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【다이어그램】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │                    호스트 물리 메모리                      │  │ │
│  │  │  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐       │  │ │
│  │  │  │    VM1      │ │    VM2      │ │   벌루운    │       │  │ │
│  │  │  │  ▓▓▓▓░░░░  │ │  ▓▓▓▓▓▓░░  │ │   ▓▓▓▓▓▓   │       │  │ │
│  │  │  │  사용 빈공간 │ │  사용 빈공간 │ │  (회수됨)   │       │  │ │
│  │  │  └─────────────┘ └─────────────┘ └─────────────┘       │  │ │
│  │  │        ↑                                     ↑           │  │ │
│  │  │        └───────────── 벌루닝 ─────────────────┘           │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 벌루닝 상세                                       │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 메모리 오버커밋: 물리 메모리보다 많은 VM 실행 가능                    │ │
│  │  • 동적 조정: VM 재부팅 없이 메모리 조절                                 │ │
│  │  • 효율성: 사용 안 하는 메모리 회수                                       │ │
│  │  • 비용 절감: 하드웨어 투자 감소                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 오버헤드: 벌루운 드라이버 실행 비용                                   │ │
│  │  • 지연: 메모리 회수/반환에 시간 소요                                     │ │
│  │  • 게스트 협력: VM에 드라이버 설치 필요                                   │ │
│  │  • 스와핑: 과도한 회수 시 VM 내 스와핑 발생                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【구현 예시】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  VMware: vmmemctl (벌루운 드라이버)                                      │ │
│  │  KVM/QEMU: virtio-balloon                                               │ │
│  │  Xen: balloon driver                                                    │ │
│  │  Hyper-V: Dynamic Memory                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【관련 기술】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • KSM (Kernel Same-page Merging): 같은 페이지 병합                      │ │
│  │  • Transparent Page Sharing: 메모리 중복 제거                            │ │
│  │  • Memory Hotplug: 메모리 동적 추가/제거                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【KVM/QEMU 벌루닝】                                                    │
│  ──────────────────                                                  │
│  // virtio-balloon 드라이버 확인                                        │
│  $ lsmod | grep virtio                                               │
│  virtio_balloon         16384  0                                      │
│                                                                     │
│  // 현재 벌루닝 상태                                                      │
│  $ cat /sys/devices/system/memory/memory*/state                     │
│  // online/offline 상태                                                │
│                                                                     │
│  // virsh로 벌루닝 조정                                                  │
│  $ virsh dommemstat vm1                                              │
│  actual 4194304                                                      │
│  balloon 1048576  // 현재 벌루닝으로 회수된 양                            │
│                                                                     │
│  // 벌루닝 크기 변경                                                      │
│  $ virsh setmem vm1 2G                                               │
│                                                                     │
│  【VMware 벌루닝】                                                       │
│  ──────────────────                                                  │
│  // vmmemctl 상태 확인                                                   │
│  $ vmware-toolbox-cmd stat balloon                                  │
│  512 MB ballooned                                                    │
│                                                                     │
│  // vSphere에서 설정                                                     │
│  // VM 설정 → 옵션 → 메모리/CPU → 메모리 핫플러그                          │
│                                                                     │
│  【벌루닝 모니터링】                                                      │
│  ──────────────────                                                  │
│  // VM 내 메모리 상태                                                     │
│  $ cat /proc/meminfo | grep -E "MemTotal|MemFree|MemAvailable"      │
│  MemTotal:        2097152 kB   // 벌루닝으로 줄어들 수 있음              │
│  MemFree:          524288 kB                                         │
│  MemAvailable:    1048576 kB                                         │
│                                                                     │
│  // 호스트에서 VM 메모리 사용                                              │
│  $ virsh dommemstat --list                                          │
│  Name         RSS    Max    Used                                    │
│  vm1          2G     4G     1.5G                                    │
│  vm2          1G     4G     800M                                    │
│                                                                     │
│  【벌루닝 설정 예시】                                                      │
│  ──────────────────                                                  │
│  // libvirt XML 설정                                                     │
│  <memory unit='KiB'>4194304</memory>                                 │
│  <currentMemory unit='KiB'>2097152</currentMemory>                   │
│  <memballoon model='virtio'>                                          │
│    <stats period='10'/>                                               │
│  </memballoon>                                                         │
│                                                                     │
│  // QEMU 직접 실행                                                       │
│  $ qemu-system-x86_64 -m 4G,slots=4,maxmem=8G \                      │
│      -device virtio-balloon-pci,bus=pci.0,addr=0x4                   │
│                                                                     │
│  【벌루닝 vs 다른 기술 비교】                                              │
│  ──────────────────                                                  │
│  // KSM: 페이지 병합                                                     │
│  $ cat /sys/kernel/mm/ksm/run                                        │
│  $ cat /sys/kernel/mm/ksm/pages_shared                               │
│                                                                     │
│  // 벌루닝 + KSM 조합이 최적                                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
