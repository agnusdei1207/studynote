# 432. 메모리 단편화 (Memory Fragmentation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 공간이 조각조각 나뉨
> 2. **가치**: 메모리 활용도 저하
> 3. **융합**: 메모리 할당, 페이징과 연관

---

## Ⅰ. 개요

### 개념 정의

메모리 단편화(Memory Fragmentation)는 **사용 가능한 메모리가 작은 조각들로 흩어져 있어 연속된 큰 공간을 할당할 수 없는 현상**이다.

### 💡 비유: 조각난 퍼즐
메모리 단편화는 **조각난 퍼즐**과 같다. 빈 공간이 많아도 연결되지 않아 큰 그림을 못 그린다.

### 메모리 단편화 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 단편화 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【내부 단편화 (Internal Fragmentation)】                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  할당된 블록 내부의 낭비:                                                 │ │
│  │                                                             │ │
│  │  ┌───────────────────────────────────┐                              │ │
│  │  │ 할당된 블록 (4KB)                    │                              │ │
│  │  │ ████████████░░░░░░░░░░░░░░░░░░░░░ │                              │ │
│  │  │  사용 (2KB)    낭비 (2KB)          │                              │ │
│  │  └───────────────────────────────────┘                              │ │
│  │                                                             │ │
│  │  원인: 고정 크기 블록 할당                                               │ │
│  │  예: 4KB 페이지에 2KB만 사용                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【외부 단편화 (External Fragmentation)】                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  할당된 블록 사이의 낭비:                                                 │ │
│  │                                                             │ │
│  │  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐                  │ │
│  │  │ A   │ 빈  │ B   │ 빈  │ C   │ 빈  │ D   │ 빈  │                  │ │
│  │  │ 2KB │1KB  │1KB  │1KB  │2KB  │1KB  │1KB  │1KB  │                  │ │
│  │  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘                  │ │
│  │                                                             │ │
│  │  총 빈 공간: 4KB, 하지만 연속 4KB 불가능!                               │ │
│  │                                                             │ │
│  │  원인: 가변 크기 블록 할당/해제 반복                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단편화 비교】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분          내부 단편화          외부 단편화                        │ │
│  │  ────────     ───────────────     ────────────────                 │ │
│  │  위치          블록 내부            블록 사이                         │ │
│  │  원인          고정 크기 할당        가변 크기 할당                    │ │
│  │  문제          공간 낭비             큰 블록 할당 실패                 │ │
│  │  해결          다양한 크기 블록       압축, 버디 시스템                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 단편화 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이징과 단편화】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  페이징:                                                                 │ │
│  │  • 외부 단편화 없음 (고정 크기 프레임)                                   │ │
│  │  • 내부 단편화만 발생 (마지막 페이지)                                    │ │
│  │  • 평균 내부 단편화: 페이지 크기의 절반                                   │ │
│  │                                                             │ │
│  │  세그먼테이션:                                                            │ │
│  │  • 외부 단편화 발생 (가변 크기)                                          │ │
│  │  • 내부 단편화 없음                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단편화 해결 방법】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 페이징 (Paging):                                                    │ │
│  │     • 외부 단편화 제거                                                  │ │
│  │     • 내부 단편화 최소화로 페이지 크기 조정                              │ │
│  │                                                             │ │
│  │  2. 메모리 압축 (Compaction):                                           │ │
│  │     • 사용 중인 블록을 한쪽으로 모음                                     │ │
│  │     • 외부 단편화 해결                                                  │ │
│  │                                                             │ │
│  │  3. 버디 시스템 (Buddy System):                                         │ │
│  │     • 2의 거듭제곱 크기 블록                                            │ │
│  │     • 분할/병합으로 단편화 관리                                          │ │
│  │                                                             │ │
│  │  4. 슬랩 할당자 (Slab Allocator):                                       │ │
│  │     • 객체 크기별 캐시                                                  │ │
│  │     • 내부 단편화 최소화                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단편화 측정】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단편화율 = (1 - (가장 큰 연속 블록 / 총 여유 공간)) × 100               │ │
│  │                                                             │ │
│  │  0%: 완전히 연속                                                        │ │
│  │  100%: 완전히 조각남                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 메모리 단편화 확인】                                          │
│  ──────────────────                                                  │
│  // 버디 시스템 상태                                                      │
│  $ cat /proc/buddyinfo                                                │
│  Node 0, zone      DMA      0      1      2      3      2 ...         │
│  Node 0, zone    Normal    123    456    234     12      5 ...       │
│  // 각 열은 2^n 페이지 블록 수                                            │
│                                                                     │
│  // 페이지 단편화 정보                                                    │
│  $ cat /proc/pagetypeinfo                                            │
│  // 페이지 타입별 (Unmovable, Reclaimable, Movable)                      │
│                                                                     │
│  // 메모리 압축 통계                                                      │
│  $ cat /proc/vmstat | grep compact                                  │
│  compact_migrate_scanned 12345                                      │
│  compact_free_scanned 67890                                         │
│  compact_isolated 1234                                              │
│  compact_stall 12                                                   │
│  // 메모리 압축 발생 횟수                                                  │
│                                                                     │
│  【메모리 압축 설정】                                                    │
│  ──────────────────                                                  │
│  // 메모리 압축 활성화                                                    │
│  $ echo 1 > /proc/sys/vm/compact_memory                             │
│  // 전체 메모리 압축                                                      │
│                                                                     │
│  // 압축 임계값                                                           │
│  $ cat /proc/sys/vm/compact_memory                                  │
│  // extfrag_threshold: 압축 시작 임계값                                 │
│                                                                     │
│  // transparent huge page와 압축                                       │
│  $ cat /sys/kernel/mm/transparent_hugepage/defrag                   │
│  // [always] madvise never                                           │
│                                                                     │
│  【단편화 최소화 프로그래밍】                                            │
│  ──────────────────                                                  │
│  // 큰 메모리 할당 시                                                     │
│  void *ptr = malloc(huge_size);                                      │
│  posix_memalign(&ptr, page_size, size);  // 페이지 정렬                │
│                                                                     │
│  // 메모리 풀 사용                                                        │
│  // 자주 할당/해제하는 크기는 풀로 관리                                     │ │
│  struct mem_pool *pool = mem_pool_create(1024, 100);                 │
│  void *obj = mem_pool_alloc(pool);                                   │
│  mem_pool_free(pool, obj);                                           │
│                                                                     │
│  // 연속 메모리 필요 시                                                   │
│  #include <linux/dma-mapping.h>                                      │
│  dma_addr_t dma_handle;                                              │
│  void *virt = dma_alloc_coherent(dev, size, &dma_handle, GFP_KERNEL);│
│  // 물리적으로 연속된 메모리                                               │
│                                                                     │
│  【단편화 모니터링】                                                      │
│  ──────────────────                                                  │
│  // 메모리 정보                                                           │
│  $ cat /proc/meminfo | grep -E "MemFree|MemAvailable|Fragmented"    │
│                                                                     │
│  // 시스템 메모리 통계                                                     │
│  $ vmstat -s | grep -E "fragment|compact"                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 메모리가 조각나 연속 할당 어려움
• 내부: 블록 내 낭비 (페이징)
• 외부: 블록 간 낭비 (세그먼테이션)
• 해결: 페이징, 압축, 버디, 슬랩
• 측정: 단편화율로 표현
• Linux: compaction으로 해결
• 확인: /proc/buddyinfo
```
