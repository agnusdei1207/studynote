# 포트 스캐닝

## 핵심 인사이트

**포트 스캐닝(Port Scanning)은 대상 시스템의 열린 포트를 식별하여 가능한 취약점이나 서비스를 찾아내는 정보 수집 기법이다.** 공격자는 스캔 결과를 바탕으로 **후속 공격(익스플로잇, 무차별 대입 시도)을 수행**하며, 방화벽 로그, 침입 탐지 시스템(IPS)이 이를 탐지할 수 있다. 방어를 위해서는 **불필요한 포트 차단**, **스캔 탐지**, **포트 미러링** 등이 필요하다.

## Ⅰ. 개요 및 정의

### 1. 정의

| 요소 | 설명 |
|:---|:---|
| **포트 스캐닝** | 시스템의 열린 포트 식별 |
| **서비스 식별** | 포트 번호로 실행 중인 서비스 추정 |
| **취약점 발견** | 특정 버전 서비스/OS 식별 |
| **네트워크 맵핑** | 대상 네트워크 구조 파악 |
| **탐지 가능성** | 로그/IDS에 남는 흔적 |

## Ⅱ. 스캔 기법

### 1. 스캔 유형

```
┌─────────────────────────────────────────────────────────────┐
│               포트 스캔 기법 분류                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. TCP Connect Scan (Full Connection Scan)                 │
│     ┌───────────────────────────────────────────────────┐  │
│     │ → SYN ────────▶ 서버                               │  │
│     │ ← SYN-ACK ────┤ (연결 성공: 포트 열림)            │  │
│     │ → ACK ────────┘                                    │  │
│     │ → RST ────────▶ 서버 (연결 종료)                  │  │
│     │                                                    │  │
│     │ 장점: 신뢰할 수 있는 결과                           │  │
│     │ 단점: 로그에 남음, 느림                            │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  2. SYN Scan (Half-Open Scan)                               │
│     ┌───────────────────────────────────────────────────┐  │
│     │ → SYN ─────────▶ 서버                              │  │
│     │ ← SYN-ACK ────┤ (포트 열림)                       │  │
│     │ → RST ─────────┘ (연결 안 완료, 바로 종료)         │  │
│     │                                                    │  │
│     │ 장점: 빠름, 로그에 안 남음                           │  │
│     │ 단점: Root 권한 필요                                │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  3. FIN Scan                                               │
│     ┌───────────────────────────────────────────────────┐  │
│     │ → FIN ────────▶ 서버                               │  │
│     │                                                    │  │
│     │ RFC 793: FIN을 받은 상태에 따라 응답 달라짐       │  │
│     │ • 열린 포트: 무시                                 │  │
│     │ • 닫힌 포트: RST                                   │  │
│     │                                                    │  │
│     │ 장점: 은밀, 방화벽 우회 가능                      │  │
│     │ 단점: 모든 OS에서 작동하지 않음                       │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  4. NULL Scan                                              │
│     ┌───────────────────────────────────────────────────┐  │
│     │ → (플래그 없음) ──▶ 서버                           │  │
│     │                                                    │  │
│     │ RFC 793: 플래그 없는 패킷은 거부해야 함          │  │
│     │ • 열린 포트: 무시                                 │  │
│     │ • 닫힌 포트: RST                                   │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  5. XMAS Scan                                              │
│     ┌───────────────────────────────────────────────────┐  │
│     │ → FIN+URG+PSH ──▶ 서버 ("크리스마스 트리")       │  │
│     │                                                    │  │
│     │ 특이한 플래그 조합으로 포트 상태 확인            │  │
│     │                                                    │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  6. UDP Scan                                               │
│     ┌───────────────────────────────────────────────────┐  │
│     │ → UDP 데이터그램 ──▶ 서버                         │  │
│     │                                                    │  │
│     │ • 열린 포트: 응답 (또는 ICMP Port Unreachable 없음) │
│     │ • 닫힌 포트: ICMP Port Unreachable               │  │
│     │                                                    │  │
│     │ 단점: 느림 (타임아웃 기다림)                        │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. Nmap 사용

```bash
# 1. TCP Connect 스캔
$ nmap -sT 192.168.1.1

# 2. SYN 스캔 (Root 필요)
$ sudo nmap -sS 192.168.1.1

# 3. UDP 스캔
$ sudo nmap -sU 192.168.1.1

# 4. 버전 스캔 (서비스/OS 식별)
$ nmap -sV 192.168.1.1

# 5. OS 식별
$ sudo nmap -O 192.168.1.1

# 6. 특정 포트 범위
$ nmap -p 1-1000 192.168.1.1

# 7. 빠른 스캔 (Top 100 포트)
$ nmap -F 192.168.1.1

# 8. 스크립트 스캔 (취약점 검사)
$ nmap --script vuln 192.168.1.1
```

## Ⅲ. 고급 스캐닝

### 1. 서비스/OS 식별

```
┌─────────────────────────────────────────────────────────────┐
│             OS/서비스 식별 기법                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. Passive OS Fingerprinting (p0f 등)                     │
│     ┌───────────────────────────────────────────────────┐  │
│     │ 패시브하게 SYN 패킷 분석                           │  │
│     │ • TTL, Window Size, MSS 등 분석                   │  │
│     │ • OS마다 고유한 패턴                               │  │
│     │ • 직접 스캔 없이 추정                               │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  2. Banner Grabbing                                        │
│     ┌───────────────────────────────────────────────────┐  │
│     │ nc 192.168.1.1 21                                  │  │
│     │ 220 vsFTPd (Version 2.3.4)                        │  │
│     │                                                    │  │
│     │ telnet 192.168.1.1 80                              │  │
│     │ HTTP/1.1 400 Bad Request                           │  │
│     │ Server: nginx/1.18.0                              │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  3. Service Enumeration                                    │
│     ┌───────────────────────────────────────────────────┐  │
│     │ nmap --script banner 192.168.1.1                  │  │
│     │ nmap -sV --version-intensity 9 192.168.1.1        │  │
│     │                                                    │  │
│     │ SMB, SNMP, FTP 등 서비스 정보 수집                │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 취약점 스캔

```bash
# NSE (Nmap Scripting Engine) 사용
$ nmap --script vuln 192.168.1.1
$ nmap --script auth 192.168.1.1
$ nmap --script brute 192.168.1.1

# 일반적인 취약점 검사
$ nmap --script=smb-vuln-ms17-010 192.168.1.0/24
$ nmap --script=http-vuln-cve2006-3392 192.168.1.1

# 보안 평가
$ nmap -sV --script vuln -oA scan_report 192.168.1.1
```

## Ⅳ. 방어 및 탐지

### 1. 방화벽 로그

```bash
# 방화벽 로그에서 스캔 탐지
$ sudo iptables -A INPUT -m limit --limit 1/s \
    -j LOG --log-prefix "[PORT_SCAN] "

# 포트 스캔 탐지 룰 (iptables recent 모듈)
# 60초 동안 10개 이상의 포트 연결 시도 시 차단
$ sudo iptables -A INPUT -m recent --set
$ sudo iptables -A INPUT -m recent --update --seconds 60 --hitcount 10 -j DROP

# fail2ban으로 자동 차단
[DEFAULT]
portscan = iptables-allports
maxretry = 1
findtime = 600
bantime = 86400
```

### 2. IDS/IPS 탐지

```
┌─────────────────────────────────────────────────────────────┐
│               IDS/IPS의 포트 스캔 탐지                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  탐지 패턴:                                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • 단일 IP에서 다수 포트 접속 시도                   │   │
│  │ • SYN 패킷이 SYN-ACK 없이 연속 전송                  │   │
│  │ • 짧은 시간 내에 다수 포트 시도                      │   │

│  • 알림 발생                                               │
│  • 자동 차단 (IPS)                                        │

│  • 포트 차단                                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3. 포트 미러링 (Port Mirroring)

```
┌─────────────────────────────────────────────────────────────┐
│               포트 미러링/허니팟                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Honeypot (덫이):                                           │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 모든 포트 "열린 것처럼" 가장                      │   │
│  │ • 실제 서비스 없음                                   │   │
│  │ • 연결 시도 기록                                    │   │
│  │ • 공격자 정보 수집                                   │   │
│  │                                                    │   │
│  │ 공격자 스캔 결과:                                   │   │
│  │ • 모든 포트 열림                                   │   │
│  │ → 하지만 전부 가짜!                                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Port Knocking:                                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 미리 정의된 포트 시퀀스로 포트 열기              │   │
│  │                                                    │   │
│  │ 예: 7000 → 8000 → 9000 순서로 Knock하면              │   │
│  │     22번 포트가 일시적으로 열림                      │   │
│  │                                                    │   │
│  │ 장점: 포트 스캔으로는 서비스 숨기                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅴ. 실무 모범 사례

### 1. 보안 강화 가이드

```bash
# 1. 불필요한 포트 확인
$ sudo netstat -tulpn | grep LISTEN

# 2. 불필요한 서비스 중지
$ sudo systemctl stop telnetd
$ sudo systemctl disable telnetd

# 3. 포트 필터링
# 외부에서 접근 불필요한 포트 차단
$ sudo iptables -A INPUT -p tcp --dport 23 -j DROP  # Telnet
$ sudo iptables -A INPUT -p tcp --dport 3306 -j DROP # MySQL

# 4. IP 제한 (신뢰하는 IP만)
$ sudo iptables -A INPUT -p tcp --dport 22 -s 192.168.1.0/24 -j ACCEPT
$ sudo iptables -A INPUT -p tcp --dport 22 -j DROP

# 5. Rate Limiting
$ sudo iptables -A INPUT -p tcp --dport 80 -m limit --limit 50/second --limit-burst 100 -j ACCEPT
$ sudo iptables -A INPUT -p tcp --dport 80 -j DROP
```

### 2. 모니터링

```bash
# 실시간 포트 모니터링
$ sudo watch -n 1 'netstat -tuna | tee netstat.log'

# 연결 추적
$ sudo lsof -i -P -n | grep ESTABLISHED

# 네트워크 연결 상태
$ ss -tuna
$ ss -tunap

# 포트 스캔 탐지
$ sudo tail -f /var/log/messages | grep PORT_SCAN
```

## 어린이를 위한 비유

**포트 스캐닝은 "문 두드리기"**와 같습니다.

상상해 보세요:
- 집에 **문이 여러 개** 있어요
- **현관문**, **창문**, **창고문**, **지하실 입구**...

**스캐닝**은:
- 어떤 문이 열려 있는지 확인하기
- "현관문 열렸어?", "창문 열렸어?" 확인
- 열려 있는 문 = **해커가 들어올 수 있는 곳!**

**스캔 기법**:
- **Connect**: 문 열어보고 닫기 (확실하지만 흔적 남아)
- **SYN**: 문콕으로만 확인 (빠르고 흔적 적음)
- **FIN**: "나 왔어?"라고 확인 (은밀)

**Nmap**은:
- "전문 문 조사 도구"
- "이 집 문 몇 개야?" 자동 확인
- "어떤 문이 잠겨 취약점이야?" 분석

**방어하는 방법**:
- **불필요한 문 잠그기**: "안 쓰는 창문은 막아"
- **킥킷이**: "이상한 사람이 문 두드리면 CCTV 체크"
- **Honeypot**: "가짜 문 만들어서 두드리는 사람 잡기"

포트 스캐닝은 **"집 구경하기"**로 첫 단계예요!
