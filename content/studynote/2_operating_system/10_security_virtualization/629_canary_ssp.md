# 스택 가디언 / 카나리

## 핵심 인사이트

**스택 가디언(Stack Guardian) 또는 카나리(Canary)는 컴파일러가 함수의 프롤로그/에필로그에 삽입하는 보안 코드로, 버퍼 오버플로우를 탐지하기 위해 스택에 **무작위 값(카나리)**를 배치한다.** 오버플로우 공격으로 리턴 주소를 덮어쓰려면 먼저 카나리를 지나야 하므로, 함수 리턴 전 **카나리 값이 변조되었는지 확인**하여 공격을 탐지한다.

---

## Ⅰ. 개요 및 원리

### 1. 정의

| 요소 | 설명 |
|:---|:---|
| **카나리** | 스택의 보호 변수, 석탄 광산의 카나리 새에서 유래 |
| **StackGuard** | 최초의 카나리 구현 (1997) |
| **SSP** | GCC의 Stack Smashing Protector |
| **ProGuard** | 윈도우의 /GS 보안 |
| **__stack_chk_fail** | 카나리 변조 시 호출되는 함수 |

### 2. 역사적 배경

```

│  │  1900년대 광산                                       │   │

│                                                             │
```

## Ⅱ. 동작 원리

### 1. 스택 레이아웃 (카나리 있음)

```
┌─────────────────────────────────────────────────────────────┐
│           카나리가 있는 스택 프레임                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  High Address                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ Return Address    ◀─────────────────────┐            │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ Saved EBP                                         │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ Canary            ◀─────────────────┐              │   │
│  │ [무작위 값]       │                   │              │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ Padding (정렬용)                                  │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ Buffer (64 bytes)                                │   │
│  │ [쓰레기 값 → 쉸코드 → NOP → 주소]                │   │
│  │   ▲                                                │   │
│  │   │                                                │   │
│  │   └── 버퍼에서 시작한 오버플로우                     │   │
│  │       카나리를 지나지 못하면 리턴 주소 덮기 불가  │   │
│  └─────────────────────────────────────────────────────┘   │
│  Low Address                                               │
│                                                             │
│  함수 리턴 시:                                             │
│  1. 카나리 값 확인                                        │
│  2. 변조되면 __stack_chk_fail() 호출 → 프로그램 종료       │
│  3. 정상이면 리턴 주소로 점프                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 카나리 검사 코드

```assembly
; GCC가 추가하는 카나리 코드 (개념적)
function_prologue:
    push    rbp
    mov     rbp, rsp
    sub     rsp, 0x50           ; 로컬 변수 공간 확보
    mov     rax, QWORD PTR fs:[40]    ; ★ 카나리 읽기
    mov     QWORD PTR [rbp-8], rax    ; ★ 카나리 저장

function_body:
    ; ... 사용자 코드 ...

function_epilogue:
    mov     rax, QWORD PTR [rbp-8]    ; ★ 저장된 카나리
    xor     rax, QWORD PTR fs:[40]    ; ★ 원본과 비교
    jne     __stack_chk_fail          ; ★ 다르면 탐지!
    ; ...
    leave
    ret

__stack_chk_fail:
    ; 카나리 변조 감지!
    call    abort                    ; 프로그램 종료
```

## Ⅲ. 카나리 유형

### 1. 카나리 생성 방식

```
┌─────────────────────────────────────────────────────────────┐
│                카나리 생성 방식 비교                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. null terminator (터미네이터)                           │
│     ┌───────────────────────────────────────────────────┐  │
│     │ 0x00000000aabbccdd                               │  │
│     │ ↑ NULL로 시작                                     │  │
│     │                                                    │  │
│     │ 장점: strcpy, printf(%s) 등이 카나리 복사 방지    │  │
│     │ 단점: 예측 가능, 취약                              │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  2. random canary (무작위 카나리)                          │
│     ┌───────────────────────────────────────────────────┐  │
│     │ 0x7f12ab34c5678900                               │  │
│     │ ↑ 매번 다른 무작위 값                              │  │
│     │                                                    │  │
│     │ 장점: 예측 불가능                                  │  │
│     │ 단점: fork() 시 자식 프로세스가 같은 값 사용     │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  3. random canary + NULL terminator                       │
│     ┌───────────────────────────────────────────────────┐  │
│     │ 0x00ab34cd56789eff                               │  │
│     │ ↑ NULL 바이트 포함                                 │  │
│     │                                                    │  │
│     │ 장점: 문자열 함수로 누출 방지 + 무작위 값        │  │
│     │ 현재: 리눅스 기본값                                │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. fs:[40] TLS 카나리

```c
// 리눅스 카나리 저장소
// arch/x86/include/asm/processor.h

struct thread_struct {
    ...
    unsigned long           stack_canary;  // ★ TLS에 저장
    ...
};

// fs 세그먼트 레지스터가 현재 스레드의 thread_struct 가리킴
// 카나리는 스택이 아닌 TLS(Thread Local Storage)에 저장
// → 스택 오버플로우로 카나리 원본 변조 불가!
```

## Ⅳ. OS별 구현

### 1. 리눅스 (GCC SSP)

```bash
# SSP(Stack Smashing Protector) 확인
$ gcc -fstack-protector -o test test.c

# 강도별 옵션
# -fstack-protector:     문자열 함수 사용 함수만
# -fstack-protector-strong: 대부분 함수 (권장)
# -fstack-protector-all:    모든 함수 (성능 저하 큼)

# 확인
$ objdump -d test | grep stack_chk_fail
0000000000000000 <__stack_chk_fail@GLIBC>

# 실행 시점 확인
$ ./test
*** stack smashing detected ***: terminated
# ↑ 카나리 변조 감지 메시지
```

## Ⅴ. 우회 기법

### 1. 정보 누출

```c
// 카나리 값 알아내기
void leak_canary() {
    char buffer[64];
    long canary;

    // 1. format string 공격으로 카나리 노출
    printf("%p %p %p %p\n", &buffer[64], &buffer[72], ...);

    // 2. 초기화되지 않은 변수 읽기
    char leak[128];
    printf("%.100s", leak);  // 이전 스택 프레임의 카나리 포함

    // 3. 미세 조정된 오버플로우
    // 한 바이트씩 늘려가며 카나리를 덮지 않는 최대 크기 찾기
    for (int i = 0; i < 200; i++) {
        send_payload(buffer_size + i);
        if (crash) break;
    }

    // 4. 카나리 브루트 포스 (fork server 활용)
    // 자식 프로세스는 같은 카나리 사용
    // 한 바이트씩 맞춰가기
}
```

### 2. 카나리 무시

```
┌─────────────────────────────────────────────────────────────┐
│          카나리를 우회하는 공격 기법                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 다른 공격 벡터                                          │
│     ├── 힙 오버플로우 (카나리 없음)                         │
│     ├── 정수 오버플로우                                     │
│     ├── Double Free                                        │

│                                                             │
│  2. 포인터 오버플로우                                       │

│     ┌───────────────────────────────────────────────────┐  │

│     │ → 카나리 검사 회피                                 │  │
│     └───────────────────────────────────────────────────┘  │
```

## Ⅵ. 성능 및 효과

### 1. 성능 오버헤드

| 함수 타입 | SSP 적용 | 성능 영향 |
|:---|:---:|:---:|
| **문자열 함수 사용** | ✓ | 약간 |
| **버퍼 없음** | ✗ | 없음 |
| **단순 함수** | ✓ | 매우 적음 |
| **복잡한 함수** | ✓ | 무시할 수 있음 |

```c
// 성능 벤치마크
// -fstack-protector-all 적용 시
// 평균 1~3% 성능 저하 (실제로는 더 적음)
// 보안 이점 >>> 성능 손실
```

```
│  │ → 거의 모든 버퍼 오버플로우 공격 차단                 │   │
```

## 어린이를 위한 비유

**카나리는 "경비견"**과 같습니다.

석탄 광산에서:
- **카나리 새**: 막다운 갱도에 가장 먼저 들어가는 새
- 가스가 새면 새가 먼저 쓰러져
- 광부들이 "위험하다!"라고 알 수 있어

컴퓨터에서도 똑같아요:
- **버퍼**: 우리 집
- **리턴 주소**: 내가 가야 할 "학교 주소"
- **카나리**: 현관 앞 **경비견**

**공격자(나쁜 친구)**가:
- 우리 집 벽을 뚫고 들어와서
- "학교 주소"를 "비밀번호 훔쳐!"로 바꾸려고 해
- 하지만 경비견(카나리)이 입구를 지키고 있어!

**카나리의 종류**:
- **고정된 경비견**: 항상 같은 모습 (예측 가능, 위험!)
- **매번 바뀌는 경비견**: 오늘은 A형, 내일은 B형 (안전!)
- **NULL 바이트 경비견**: "문자열 복사" 같은 공격도 막아

**우회하는 방법**:
- **정보 누출**: "경비견 모습이 어떻게 생겼어?"라고 물어보기
- **힙 공격**: "경비견 없는 뒷문으로 들어가기"

**더 강력하게**:
- **ASLR + 카나리**: "경비견 모습도 매번 바뀌고, 위치도 바뀌어!"
- **NX + ASLR + 카나리**: "3중 방어선 - 거의 뚫을 수 없어!"

카나리는 **"내 물건을 건드리면 짖는 경비견"**이에요!
