# IDS / IPS

## 핵심 인사이트

**IDS(Intrusion Detection System)는 네트워크/시스템의 이상 활동을 탐지하고, IPS(Intrusion Prevention System)는 이를 차단하는 보안 솔루션이다.** IDS는 경보만 발생시키는 **경찰관**이라면, IPS는 실제로 공격을 막는 **경비원** 역할을 한다. **시그니처 기반**과 **이상 탐지(Anomaly Detection)** 두 가지 방식이 있으며, 현대 보안에서는 **자동화된 위협 대응**의 핵심이다.

---

## Ⅰ. 개요 및 정의

### 1. 정의

| 용어 | 설명 |
|:---|:---|
| **IDS** | 침입 탐지 시스템 (탐지 + 경고) |
| **IPS** | 침입 방지 시스템 (탐지 + 차단) |
| **NIDS** | 네트워크 IDS (패킷 분석) |
| **HIDS** | 호스트 IDS (시스템 호출/로그 분석) |
| **False Positive** | 정상을 공격으로 오탐 (FP) |
| **False Negative** | 공격을 정상으로 누락 (FN) |

### 2. IDS vs IPS

```
┌─────────────────────────────────────────────────────────────┐
│               IDS vs IPS 비교                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  IDS (Intrusion Detection)          IPS (Intrusion Prevention) │
│  ┌─────────────────┐              ┌─────────────────┐       │
│  │ 경찰관 역할    │              │ 경비원 역할    │       │
│  │                 │              │                 │       │
│  │ 모니터링       │              │ 모니터링 + 차단 │       │
│  │ 경고 발생       │              │ 실시간 방어     │       │
│  │                 │              │                 │       │
│  │ 영향 없음       │              │ 트래픽 지연     │       │
│  │                 │              │ 오탐 시 서비스  │       │
│  │                 │              │ 중단 가능       │       │
│  │ (수동 대응)     │              │ (자동 대응)     │       │
│  └─────────────────┘              └─────────────────┘       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅲ. 탐지 방식

### 1. 시그니처 기반 (Signature-Based)

```
┌─────────────────────────────────────────────────────────────┐
│           시그니처 기반 탐지 (Signature-Based)                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  원리: 알려진 공격 패턴과 비교                               │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                     │   │
│  │  패턴 매칭                                          │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │ if (packet matches pattern) {               │   │   │
│  │  │     alert("Attack detected!");           │   │   │
│  │  │ }                                            │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  │                                                     │   │
│  │  예시 시그니처                                      │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │ alert tcp any any -> any 80 (              │   │   │
│  │  │     content:"../../../etc/passwd";          │   │   │
│  │  │     msg:"Directory traversal attempt";       │   │   │
│  │  │ )                                            │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  │                                                     │   │
│  │  장점:                                              │   │
│  │  • 정확도 높음 (알려진 공격만 탐지)                 │   │
│  │  • 신속한 탐지                                       │   │
│  │  • 관리가 쉬움                                       │   │
│  │                                                     │   │
│  │  단점:                                              │   │
│  │  • 제로데이 공격 탐지 불가                           │   │
│  │  • 변형 공격 우회 가능                               │   │
│  │  • 오탐(False Positive) 가능                           │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 이상 탐지 (Anomaly-Based)

```
┌─────────────────────────────────────────────────────────────┐
│           이상 탐지 (Anomaly-Based Detection)                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  원리: 정상 행동 기준(Baseline) 학습 후 이상 탐지          │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                     │   │
│  │  학습 단계 (학습 기간)                            │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │ 정상 트래픽 수집                             │   │   │
│  │  │ • 프로토콜 분포                                 │   │   │
│  │  │ • 연결 빈도                                     │   │   │
│  │  │ • 패킷 크기 분포                               │   │   │
│  │  │ • 시간대별 패턴                                 │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  │                                                     │   │
│  │  탐지 단계                                          │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │ if (activity deviates_from_baseline) {     │   │   │
│  │  │     alert("Anomaly detected!");           │   │   │
│  │  │ }                                            │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  │                                                     │   │
│  │  예시:                                              │   │
│  │  • 새로운 포트 접속                                   │   │
│  │  • 평소 3시에 대량 트래픽                             │   │
│  │  • 비정상적인 DNS 쿼리                                │   │
│  │  • 단일 IP에서 다수 연결                               │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  장점:                                                      │
│  • 제로데이 공격 탐지 가능                               │
│  • 변형 공격에 강함                                      │
│                                                             │
│  단점:                                                      │
│  • 높은 오탐(False Positive)                              │
│  • 학습 기간 필요                                         │
│  │  → 처음에는 정상이 비정상으로 탐지될 수 있음        │   │
│  • 기준 설정 어려움                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅴ. 규칙 언어 (Snort/Suricata)

### 1. 규칙 구조

```
┌─────────────────────────────────────────────────────────────┐
│               Snort/Suricata 규칙 구조                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  alert tcp any any -> any 80 (                              │
│    msg:"WEB-MISC webhpkit directory traversal";           │
│    flow:established,to_server;                             │
│    content:"../../../";                                     │
│    nocase;                                                  │
│    http_uri;                                                │
│    sid:1000001;                                             │
│    rev:1;                                                    │
│  )                                                           │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 1. 액션 (Action)                                     │   │
│  │    alert, log, pass, drop, reject                  │   │
│  │                                                     │   │
│  │ 2. 프로토콜 (Protocol)                                │   │
│  │    tcp, udp, icmp, ip, http, dns, tls...          │   │
│  │                                                     │   │
│  │ 3. 주소 (Source/Destination)                         │   │
│  │    any, $HOME_NET, $EXTERNAL_NET, 192.168.1.0/24  │   │
│  │                                                     │   │
│  │ 4. 포트 (Port)                                       │   │
│  │    any, 80, 53, !22                                  │   │
│  │                                                     │   │
│  │ 5. 옵션 (Options)                                     │   │
│  │    msg: 설명                                          │   │
│  │    flow: 연결 상태                                  │   │
│  │    content: 패턴                                     │   │
│  │    pcre: 정규식                                       │   │
│  │    sid: 규칙 ID                                     │   │
│  │    rev: 버전                                         │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 예시 규칙

```rule
# SQL Injection 탐지
alert tcp $EXTERNAL_NET any -> $HTTP_SERVERS $HTTP_PORTS (
    msg:"WEB-MISC SQL injection attempt";
    flow:to_server,established;
    content:"UNION SELECT";
    nocase;
    http_uri;
    sid:1000002;
    rev:1;
)

# SYN Flood 탐지
alert tcp any any -> any any (
    msg:"DDoS SYN Flood";
    detection_filter:track by_src and count 100, seconds 1;
    threshold:type both, track by_src, count 100, seconds 1;
    sid:1000003;
    rev:1;
)

# SSH 무차별 대입 시도 탐지
alert tcp $EXTERNAL_NET any -> $SSH_SERVERS 22 (
    msg:"ATTACK-RESPONSE SSH brute force";
    flow:to_server,established;
    threshold:type both, track by_src, count 5, seconds 60;
    sid:1000004;
    rev:1;
)
```

## Ⅴ. 운영 및 관리

### 1. 배포 전략

```
┌─────────────────────────────────────────────────────────────┐
│            IDS/IPS 배포 전략                                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 투명 모드 (Promiscuous Mode)                           │
│     ┌───────────────────────────────────────────────────┐  │
│     │ NIC가 모든 프레임 수신                             │  │
│     │ • 네트워크 스위치 포트 미러링                      │  │
│     │ • SPAN 포트 활성화                                 │  │
│     │ • TAP (Test Access Point) 사용                      │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  2. 배포 방식                                             │
│     ┌───────────────────────────────────────────────────┐  │
│     │ 인라인 (Inline): 트래픽을 직접 통과           │  │
│     │ ┌─────────────────────────────────────────────┐   │  │
│     │ │ 인터넷 ──▶ [IPS] ──▶ 방화벽 ──▶ 내부     │   │  │
│     │ └─────────────────────────────────────────────┘   │  │
│     │                                                    │  │
│     │ 터미널 (Tap/Promiscuous): 복사본만 검사         │  │
│     │ ┌─────────────────────────────────────────────┐   │  │
│     │ │ 인터넷 ──▶ 스위치 ──▶ 방화벽 ──▶ 내부     │   │  │
│     │ │                      ▲                        │   │  │
│     │ │                      │                        │   │  │
│     │ │                    [IDS]                      │   │  │
│     │ └─────────────────────────────────────────────┘   │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 어린이를 위한 비유

**IDS는 "경비탐지기", IPS는 "경비원"**과 같습니다.

**IDS (Intrusion Detection System)** = 경비탐지기
- "이상한 소리 들렸어!"라고 알림
- CCTV만 보고 경보
- 직접 잡지 못 해
- 관리자가 직접 대응

**IPS (Intrusion Prevention System)** = 경비원
- "이상한 행동 하면 바로 제지!"
- 직접 체포할 수 있어
- 실시간 대응
- 자동으로 침입자 막아

**시그니처 기반**:
- "나쁜 사람은 이렇게 행동해"
- "이런 문장 쓰면 해커야!"
- 미리 알려진 범죄 수첩부

**이상 탐지**:
- "평소엔 이렇게 안 왔는데"
- "오늘은 10시인데 왜 이렇게 붐적일까?"
- 평소와 다르면 이상해

**오탐(False Positive)**:
- "경비원이 착각해서 잡았어"
- 밤에 야근하는 친구를 도둑으로 잡은 것
- 튜닝으로 줄여야 해

**IDS/IPS는**
- **"집에 숨겨둔 CCTV와 경비원"**
- IDS가 "이상해!"라고 하면
- IPS가 "잡았다!"라고 해

요즘은 **자동화된 스마트 보안**이 핵심이에요!
