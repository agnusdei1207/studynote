# 버퍼 오버플로우

## 핵심 인사이트

**버퍼 오버플로우는 프로그램이 할당된 메모리 경계를 초과하여 데이터를 쓸 때 발생하는 취약점이다.** C/C++ 같은 low-level 언어에서 경계 검사 없는 메모리 조작 함수로 인해 발생하며, 공격자는 이를 이용해 **리턴 주소를 덮어쓰고 쉸코드를 실행**하여 시스템을 장악할 수 있다. 역사상 가장 흔하고 위험한 보안 취약점 중 하나다.

---

## Ⅰ. 개요 및 원리

### 1. 정의

| 요소 | 설명 |
|:---|:---|
| **버퍼** | 고정된 크기의 메모리 영역 |
| **오버플로우** | 버퍼 경계를 넘어선 데이터 쓰기 |
| **취약 함수** | `gets()`, `strcpy()`, `sprintf()` 등 경계 검사 없는 함수 |
| **영향** | 프로그램 충돌, 임의 코드 실행, 권한 상향 |

### 2. 스택 메모리 레이아웃

```

│  └─────────────────────────────────────────────────────┘   │

│                                                             │
│  스택은 아래로 자란다 (High → Low)                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅱ. 공격 원리

### 1. 기본 버퍼 오버플로우

```
┌─────────────────────────────────────────────────────────────┐
│            버퍼 오버플로우 공격 과정                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 취약한 코드                                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ void vulnerable() {                                │   │
│  │     char buffer[64];  // 64바이트 버퍼             │   │
│  │     gets(buffer);     // ★ 경계 검사 없음!          │   │
│  │     printf("Input: %s\n", buffer);                 │   │
│  │ }                                                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  2. 정상 입력: "Hello"                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ [H][e][l][l][o][\0][  ][  ]...[RET]                │   │
│  │  ↑buffer             ↑Saved EBP   ↑Return Addr     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  3. 공격 입력: "A"×80 + 쉸코드 주소                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ [A]×80 [쉸코드 주소]                               │   │
│  │  ↑buffer──────────────────↑Saved EBP  ↑Return Addr │   │
│  │                                   (덮어씀!)         │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  4. 함수 리턴 시                                            │
│     • EBP 복원 → 스택 포인터 조작                           │
│     • RET 명령 → 쉸코드 주소로 점프!                        │
│     • 쉸코드 실행 → 루트 쉘 획득                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 쉸코드 (Shellcode)

```nasm
; x86 리눅스 쉸코드 예시 (execve("/bin/sh"))
section .text
    global _start

_start:
    ; execve("/bin/sh", NULL, NULL)
    xor eax, eax        ; eax = 0
    push eax            ; 문자열 종료 NULL
    push 0x68732f2f     ; "//sh"
    push 0x6e69622f     ; "/bin"
    mov ebx, esp        ; ebx = "/bin//sh" 주소
    push eax            ; envp[] = NULL
    push eax            ; argv[] = NULL
    push ebx            ; argv[0] = 프로그램 이름
    mov ecx, esp        ; ecx = argv[]
    mov al, 11          ; syscall: execve (11)
    int 0x80            ; 커널 호출
```

## Ⅲ. 취약 함수

### 1. 위험한 함수 목록

| 함수 | 위험도 | 대체 함수 |
|:---|:---:|:---|
| `gets()` | 🔴极高 | `fgets()` |
| `strcpy()` | 🔴高 | `strncpy()`, `strlcpy()` |
| `strcat()` | 🔴高 | `strncat()`, `strlcat()` |
| `sprintf()` | 🔴高 | `snprintf()` |
| `scanf()` | 🟡中 | `fgets()` + `sscanf()` |
| `vsprintf()` | 🔴高 | `vsnprintf()` |
| `realpath()` | 🟡中 | 길이 제한 버전 |

### 2. 취약 코드 vs 안전한 코드

```c
// ❌ 취약한 코드
void vulnerable() {
    char buffer[64];
    gets(buffer);  // 경계 검사 없음
}

void also_vulnerable() {
    char buffer[64];
    strcpy(buffer, user_input);  // 길이 확인 안 함
}

// ✅ 안전한 코드
void secure() {
    char buffer[64];
    fgets(buffer, sizeof(buffer), stdin);  // 크기 제한
}

void also_secure() {
    char buffer[64];
    strncpy(buffer, user_input, sizeof(buffer) - 1);
    buffer[sizeof(buffer) - 1] = '\0';  // NULL 보장
}
```

### 3. 정수 오버플로우

```c
// 정수 오버플로우로 버퍼 오버플로우 유도
void integer_overflow_exploit() {
    int len = 0x7fffffff;  // 매운 큰 값
    char *buffer = malloc(len + 10);  // 오버플로우!

    // len + 10 = 음수 (정수 오버플로우)
    // malloc은 작은 메모리 할당
    // 하지만 memcpy는 큰 메모리 복사 시도
    memcpy(buffer, input, len + 10);  // 힙 오버플로우!
}
```

### 2. 안전한 코딩 관행

```c
// ✅ 안전한 코딩 규칙
void secure_coding() {
    // 1. 크기 명시적 관리
    char buffer[64];
    strncpy(buffer, input, sizeof(buffer) - 1);
    buffer[sizeof(buffer) - 1] = '\0';

    // 2. snprintf 사용
    char output[128];
    snprintf(output, sizeof(output), "Input: %s", input);

    // 3. fgets 사용
    char line[256];
    if (fgets(line, sizeof(line), stdin) != NULL) {
        // null 제거
        line[strcspn(line, "\n")] = '\0';
    }

    // 4. calloc으로 0 초기화
    struct data *d = calloc(1, sizeof(struct data));
    if (d == NULL) return;

    // 5. 길이 검증
    size_t len = strlen(input);
    if (len >= sizeof(buffer)) {
        errno = EMSGSIZE;
        return;
    }

    // 6. strlcpy, strlcat (BSD)
    strlcpy(buffer, input, sizeof(buffer));
}
```

## 어린이를 위한 비유

**버퍼 오버플로우는 "넘치는 물"**과 같습니다.

상상해 보세요:
- **컵(버퍼)**: 100ml만 담을 수 있어요
- **물(데이터)**: 150ml를 부으면 어떻게 될까요?
- **결과**: 50ml가 밖으로 넘쳐 흘러요

컴퓨터에서도 똑같이 일어나요:
```
컵(64바이트)에 "AAAA..."를 계속 부어
→ 64바이트가 넘으면?
→ 옆에 있는 중요한 정보(리턴 주소)가 덮여
→ 프로그램이 "원래 가야 할 곳" 말고
→ "나쁜 사람이 원하는 곳"으로 가게 돼요
```

**리턴 주소**는 "집 주소"와 같아요:
- 정상: 학교 → 집 → 내일 다시 학교
- 공격: 학교 → **나쁜 집** → 비밀번호 훔쳐!

**쉸코드**는 "나쁜 명령"이에요:
```
AAAAAA...AAAAA[나쁜 집 주소]
                    ↑
           여기에 가면 "비밀번호 훔쳐!"라는
           명령이 미리 준비되어 있어요
```

**방어하는 방법**:
- **Canary**: "경비견"을 세워두고, 누가 지나가면 짖어
- **NX**: "이 방에는 실행 못 해!"라는 표지판
- **ASLR**: "매번 집 주소를 바꿔서" 찾지 못하게

가장 좋은 방어는 **"컵 크기만큼만 따르는"** 안전한 코드예요!
