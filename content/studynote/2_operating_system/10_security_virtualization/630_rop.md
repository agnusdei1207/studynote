# ROP (Return-Oriented Programming)

## 핵심 인사이트

**ROP(Return-Oriented Programming)는 NX Bit로 쉸코드 실행이 불가능해진 환경에서, 실행 가능한 영역에 이미 존재하는 코드 조각(Gadget)들을 연결하여 공격하는 기법이다.** 각 Gadget는 "유용한 명령어 + RET"로 끝나며, 이들을 체인으로 연결해 **임의의 작업을 수행**한다. ASLR로 Gadget 주소를 무작위화하면 방어 가능하다.

## Ⅰ. 개요 및 배경

### 1. 정의

| 요소 | 설명 |
|:---|:---|
| **ROP** | Return-Oriented Programming, 리턴 지향 프로그래밍 |

| **ROP Chain** | 실행 흐름을 조작하기 위한 Gadget들의 연결 |

## Ⅱ. Gadget

### 1. Gadget의 구조

```
┌─────────────────────────────────────────────────────────────┐
│                  Gadget의 기본 구조                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Gadget = "유용한 명령어" + "RET"                           │
│                                                             │
│  예시 (x86-64):                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 0x7ffff7a2e1b: pop rdi; ret                         │   │
│  │                     ↑           ↑                    │   │
│  │                유용한 명령    반환                   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  분석:                                                      │
│  • pop rdi: 스택에서 값을 꺼내 rdi에 저장                 │
│  • ret: 다음 주소(스택에 있음)로 점프                     │
│  • 결과: rdi 제어 가능                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 일반적인 Gadget 유형

| 유형 | 예시 | 용도 |
|:---|:---|:---|
| **Load Gadget** | `pop rdi; ret` | 레지스터에 값 로드 |
| **Store Gadget** | `mov [rsi], rax; ret` | 메모리에 값 저장 |
| **Arithmetic Gadget** | `add rax, rbx; ret` | 연산 |
| **Syscall Gadget** | `syscall; ret` | 시스템 콜 |
| **JMP Gadget** | `jmp rax;` | 간접 점프 |

### 3. Gadget 예시

```nasm
; x86-64 libc.so에서 찾을 수 있는 일반적인 Gadget

; 1. 인자 설정 (pop 체인)
pop rdi; ret              ; 첫 번째 인자 (rdi)
pop rsi; ret              ; 두 번째 인자 (rsi)
pop rdx; ret              ; 세 번째 인자 (rdx)
pop rax; ret              ; 시스템 콜 번호 (rax)

; 2. 함수 호출
pop rbp; ret              ; 스택 정렬
mov rdi, rsi; ret        ; 인자 복사

; 3. 시스템 콜
syscall; ret             ; 또는 "int 0x80; ret"

; 4. 스택 조정
add rsp, 0x20; ret       ; 스택 포인터 조정
leave; ret               ; 프레임 해제

; 5. 데이터 이동
mov qword ptr [rdi], rsi; ret
mov rax, qword ptr [rdi]; ret
```

```bash
# 특정 레지스터 관련 Gadget만

```

## 어린이를 위한 비유

**ROP는 "레고 조립"**과 같습니다.

**전통적 공격**:
- 쉸코드 = **새로운 장난감 만들기**
- NX Bit = "새 장난감 가져오지 마!"

**ROP 공격**:
- "가져오지 말고 **있는 장난감으로만** 놀자!"
- 레고 상자에 있는 **작은 조각들**(Gadget)을
- 연결해서 **원하는 모양**만들기

**Gadget은 레고 조각**:
```
"몸통 조각" + "머리 조각" = 로봇 만들기
pop rdi; ret + syscall = 실행 권한 얻기
```

**Gadget Chain은 조립 설계서**:
```
1. 먼저 "rdi 조각" 꺼내기
2. 다음 "rsi 조각" 꺼내기
3. 그다음 "syscall 조각"으로 완성!
```

**문제는 조각 주소를 알아야 한다는 거**:
- **ASLR 없으면**: 상자가 항상 같은 곳에 있어 → 쉬움
- **ASLR 있으면**: 상자가 매번 다른 곳에 → 어려움

**더 막으려면**:
- **CFI**: "이 조각은 여기에만 연결 가능!"라고 규칙 정하기
- **CET**: "가짜 조각 못 쓰게!" 진품 확인기

ROP는 **"남의 레고 조각 훔쳐와서 내 로봇 만들기"**라고 할 수 있어요!
