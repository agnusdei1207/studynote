# DoS / DDoS

## 핵심 인사이트

**DoS(Denial of Service)는 시스템의 정상 서비스를 방해하는 공격으로, DDoS(Distributed DoS)는 다중 공격자(봇)가 동시에 공격하는 분산형이다.** 공격자는 **대역폭, 리소스, 취약점**을 노리어 서비스를 마비시키며, 최근 **증폭 공격(Amplification Attack)**과 **보안 장비 악용**이 심각한 문제다. 방어를 위해서는 **전문 DDoS 방어 서비스**나 **CDN** 활용이 필수적이다.

---

## Ⅰ. 개요 및 정의

### 1. 정의

| 용어 | 설명 |
|:---|:---|
| **DoS** | 단일 공격자에 의한 서비스 거부 |
| **DDoS** | 다중 공격자(봇넷)에 의한 분산 서비스 거부 |
| **봇(Bot)** | 악성코드에 감염되어 공격자 명령에 따르는 좀비 PC |
| **봇넷** | 다수의 봇으로 구성된 네트워크 |
| **Amplification** | 작은 공격으로 큰 반사를 유도하는 증폭 공격 |

### 2. DoS vs DDoS

```
┌─────────────────────────────────────────────────────────────┐
│               DoS vs DDoS 비교                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  DoS (Denial of Service)          DDoS (Distributed)         │
│  ┌─────────────────┐              ┌─────────────────┐       │
│  │ 단일 공격자     │              │ 다중 공격자     │       │
│  │                 │              │ (봇넷)          │       │
│  │ ┌───────────┐   │              │ ┌───┐ ┌───┐ ┌───┐│       │
│  │ │ 공격자    │   │              │ │봇1│ │봇2│ │봇3││       │
│  │ └─────┬─────┘   │              │ └─┬─┘ └─┬─┘ └─┬─┘│       │
│  │       │         │              │   │    │    │  │       │
│  │       ▼         │              │   └────┴────┘  │       │
│  │   ┌─────────┐  │              │       ▼         │       │
│  │   │ 타겟    │  │              │   ┌─────────┐   │       │
│  │   │ 서버    │  │              │   │ 타겟    │   │       │
│  │   │ 과부하  │  │              │   │ 서버    │   │       │
│  │   └─────────┘  │              │   │ 붕괴    │   │       │
│  └─────────────────┘              └─────────────────┘       │
│                                                             │
│  특징:                                                      │
│  • 단일 출발지 IP (차단 쉬움)                            │
│  • 대역폭 제한                                            │
│  • 비교적 방어 쉬움                                      │
│                                                             │
│  • 다중 출발지 IP (차단 어려움)                          │
│  • 압도적인 대역폭                                       │
│  • 전문 방어 서비스 필요                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅱ. DoS 공격 유형

### 1. 공격 분류

```
┌─────────────────────────────────────────────────────────────┐
│              DoS 공격 유형별 분류                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 대역폭 고갈 (Bandwidth Consumption)                     │
│     ├── UDP Flood                                          │
│     ├── ICMP Flood                                         │
│     └── 링크 대역폭 소진                                    │
│                                                             │
│  2. 리소스 고갈 (Resource Exhaustion)                        │
│     ├── SYN Flood (반 연결)                               │
│     ├── HTTP Flood                                        │
│     ├── Slowloris (느린 요청)                             │
│     └── 연결 테이블 소진                                    │
│                                                             │
│  3. 프로토콜 취약점 (Protocol Flaw)                          │
│     ├── Ping of Death                                      │
│     ├── Teardrop                                          │
│     └── Land Attack                                        │
│                                                             │
│  4. 증폭 공격 (Amplification Attack)                         │
│     ├── DNS Amplification                                  │
│     ├── NTP Amplification                                  │
│     └── SNMP Amplification                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 주요 공격 기법

| 공격 | 포트 | 증폭비 | 설명 |
|:---|:---:|:---:|:---|
| **DNS Amplification** | 53 | 28~54x | 짧은 쿼리 → 큰 응답 |
| **NTP Amplification** | 123 | 556x | monlist 요청 |
| **SNMPv2 Amplification** | 161 | 6.3x | GetBulk 요청 |
| **SSDP Amplification** | 1900 | 30x | UPnP 디스커버리 |
| **Memcached Amplification** | 11211 | 51,000x | 최대 증폭! |
| **CharGEN Amplification** | 19 | 358x | 문자 스트림 |

## Ⅲ. SYN Flood

### 1. 3-Way Handshake

```
┌─────────────────────────────────────────────────────────────┐
│              정상 TCP 3-Way Handshake                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Client                    Server                            │
│    │                          │                             │
│    │ ────── SYN ──────────────▶│                             │
│    │                          │ (SYN_RCVD 상태)              │
│    │                          │                             │
│    │ ◀─── SYN-ACK ─────────────│                             │
│    │ (ESTABLISHED 상태)       │                             │
│    │                          │                             │
│    │ ────── ACK ──────────────▶│                             │
│    │                          │ (연결 완료)                  │
│    │                          │                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. SYN Flood 공격

```
┌─────────────────────────────────────────────────────────────┐
│              SYN Flood 공격                                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Attacker                   Server                           │
│    │                          │                             │
│    │ ────── SYN(위조) ────────▶│                             │
│    │                          │ (반 연결 대기)              │
│    │ ────── SYN(위조) ────────▶│                             │
│    │                          │ (반 연결 대기)              │
│    │ ────── SYN(위조) ────────▶│                             │
│    │                          │ (반 연결 대기)              │
│    │ ... 수천만 개 SYN ...     │                             │
│    │                          │                             │
│  결과:                                                      │
│  • 연결 테이블 FULL                                      │
│  • 정상 SYN 처리 불가                                    │
│  • 서비스 거부                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3. 방어: SYN Cookies

```
┌─────────────────────────────────────────────────────────────┐
│              SYN Cookies 방어                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  SYN Cookies를 사용하면 서버가 SYN_RCVD 상태 유지 안 함  │
│                                                             │
│  1. 클라이언트: SYN 전송                                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ Server: SYN+ACK 발송                                  │   │
│  │         (ACK 번호 = 암호화된 연결 정보)             │   │
│  │         = "SYN Cookie"                               │   │
│  │                                                      │   │
│  │ SYN Cookie: hash(src_ip, src_port, dst_ip, dst_port, │   │
│  │              seq) + timestamp                           │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  2. 정상 클라이언트: ACK 전송 (SYN Cookie 포함)            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ Server: Cookie 검증                                  │   │
│  │         • 정상이면 연결 확립                         │   │
│  │         • 위조면 무시                                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  3. 공격자: ACK 없음                                        │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ Server: 리소스 낭비 없음 (상태 유지 안 함)          │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  활성화:                                                   │
│  $ echo 1 > /proc/sys/net/ipv4/tcp_syncookies           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅳ. 증폭 공격

### 1. DNS Amplification

```
┌─────────────────────────────────────────────────────────────┐
│            DNS 증폭 공격 원리                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  공격자의 전략:                                             │
│  "작은 요청으로 큰 응답을 받아서 타겟에 전송"             │
│                                                             │
│  1. 공격자가 DNS 리졸버에게 쿼리 전송 (발신지 위조)        │
│     ┌───────────────────────────────────────────────────┐  │
│     │ Query: "." (ANY 레코드)                          │  │
│     │ SRC: 타겟 IP (위조!)                             │  │
│     │ DST: 오픈 DNS 리졸버                              │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  2. DNS 리졸버가 응답 (타겟으로 전송!)                    │
│     ┌───────────────────────────────────────────────────┐  │
│     │ Response: ALL DNS root servers (수 KB)           │  │
│     │ SRC: DNS 리졸버                                  │  │
│     │ DST: 타겟 IP (위조된 발신지)                     │  │
│     │ ◀───────────────────────────────────────────────┐  │
│     │           (타겟으로 폭주!)                         │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  증폭비: 요청 50바이트 → 응답 3,000바이트 (60배)          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 방어: BCP 38 (원천지 필터링)

```
┌─────────────────────────────────────────────────────────────┐
│            증폭 공격 방어                                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 네트워크 에지 필터링                                   │
│     ┌───────────────────────────────────────────────────┐  │
│     │ Ingress Filter (입구):                            │  │
│     │ if (SRC_IP != MY_NETWORK) { drop; }               │  │
│     │ • 내부에서 나가는 패킷이 내부 IP가 아니면 드롭   │  │
│     │ → 스푸핑된 발신지 차단                              │  │
│     │                                                    │  │
│     │ Egress Filter (출구):                             │  │
│     │ if (DST_IP != MY_NETWORK) { check_SRC(); }       │  │
│     │ • 외부로 나가는 패킷의 발신지 확인               │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  2. DNS 리졸버 보안                                        │
│     ┌───────────────────────────────────────────────────┐  │
│     │ • Rate Limiting (쿼리 속도 제한)                 │  │
│     │ • Response Rate Limiting                           │  │
│     │ • Any 쿼리 금지                                     │  │
│     │ • IP-based ACL (신뢰하는 클라이언트만)            │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  3. RPKI (Resource Public Key Infrastructure)               │
│     ┌───────────────────────────────────────────────────┐  │
│     │ BGP Route Origin Validation                         │  │
│     │ • AS 경로 정당성 검증                             │  │
│     │ • IP 주소 할당 검증                               │  │
│     │ → 스푸핑된 IP 차단                                 │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅵ. 방어 및 완화

### 1. 다계층 방어

```
┌─────────────────────────────────────────────────────────────┐
│            DDoS 방어 다계층 접근                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Layer 3-4 (Network Layer)                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • 방화벽: ACL, Rate Limiting                        │   │
│  │ • ISP: uRPF, Blackhole Routing                      │   │
│  │ • Scrubbing Center: 트래픽 정화                     │   │
│  │ • Anycast: 트래픽 분산                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                           │                                 │
│                           ▼                                 │
│  Layer 7 (Application Layer)                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • WAF: 웹 공격 필터링                              │   │
│  │ • CDN: 콘텐츠 캐싱, 분산                            │   │
│  │ • Load Balancer: 트래픽 분산                      │   │
│  │ • Rate Limiting: 세션/요청 제한                     │   │
│  │ • CAPTCHA: Bot 확인                                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                           │                                 │
│                           ▼                                 │
│  Cloud-Based DDoS Protection                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ • Cloudflare, AWS Shield, Akamai Prolexic          │   │
│  │ • 대규모 대역폭 흡수                                 │   │
│  │ • 전문 mitigation                                  │   │
│  │ • 글로벌 PoP 분산                                   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 주요 방어 기술

| 기술 | 설명 | OSI 계층 |
|:---|:---|:---:|
| **Blackhole Routing** | 공격 트래픽을 null 라우팅 | L3 |
| **Rate Limiting** | 패킷/요청 속도 제한 | L3/L4/L7 |
| **SYN Cookies** | SYN Flood 방어 | L4 |
| **Anycast** | 트래픽을 가장 가까운 PoP로 분산 | L3 |
| **CDN** | 콘텐츠 캐싱, 분산 | L7 |
| **WAF** | 웹 공격 필터링 | L7 |
| **Challenge/Response** | CAPTCHA, JavaScript 챌린지 | L7 |

        │ SYN Flood   │          │ Dyn (2016)  │
        │ UDP Flood   │          │ GitHub (2018)│

## 어린이를 위한 비유

**DoS/DDoS는 "문을 막고 들어가기"**와 같습니다.

**DoS (단일 공격자)**:
- 혼자서 입구 막기
- "내가 혼자 문 막고 서 있으면 아무도 못 들어와!"
- 하지만 경비원이 한 명만 잡으면 돼

**DDoS (분산 공격)**:
- 친구들 수백 명이 각자 다른 입구에서 막기
- "우리 모두가 동시에 문 막자!"
- 경비원이 누구를 잡을지 몰라

**대역폭 소모형**:
- 도로에 트럭 수천 대 몰리기
- 진짜 손님들은 못 들어와

**증폭 공격**:
- "작은 말 크게 하기"
- 10명에게 "너희 친구 다 불러와!"라고 하면
- 수백 명이 몰려와

**방어하는 방법**:
- **Rate Limiting**: "한 사람은 1분에 1번만!"
- **WAF**: "이상한 사람은 입구 컷!"
- **CDN**: "우리 집이 여러 개니까 다 가봐!"
- **Cloud Protection**: "전문 방범 회사에 맡기기"

DDoS는 **"문 앞에 인산막을 쌓는 것"**과 같아요!
