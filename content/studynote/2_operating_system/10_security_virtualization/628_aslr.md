# ASLR

## 핵심 인사이트

**ASLR(Address Space Layout Randomization)는 프로그램 실행 시마다 메모리 영역(스택, 힙, 라이브러리)의 주소를 무작위로 배치하여 공격자가 주소를 예측하지 못하게 하는 보안 기법이다.** 버퍼 오버플로우, ROP(Return-Oriented Programming) 등의 공격에서 **정확한 주소를 알아야 하는** 공격자에게 큰 장애물이 되며, NX Bit와 결합하면 방어 효과가 더욱 강력해진다.

---

## Ⅰ. 개요 및 정의

### 1. 정의

| 요소 | 설명 |
|:---|:---|
| **ASLR** | 주소 공간 레이아웃 무작위화 |
| **목적** | 메모리 주소 예측 방지 |
| **적용 대상** | 스택, 힙, 라이브러리, 실행 파일 |
| **효과** | 버퍼 오버플로우, ROP 공격 난이도 상승 |

### 2. ASLR 전후 비교

```
┌─────────────────────────────────────────────────────────────┐
│               ASLR 전후 메모리 배포                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ASLR 비활성 (항상 같은 주소)                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 0x400000  [실행 파일 .text]                        │   │
│  │ 0x600000  [데이터 영역]                             │   │
│  │ 0x7ffff000 [스택]                                   │   │
│  │ 0x7ffff000 [libc.so]                                │   │
│  │           ↑                                         │   │
│  │           항상 같음 → 공격자가 주소 알기 쉬움       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ASLR 활성 (실행 시마다 다른 주소)                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 실행 1:                                             │   │
│  │ 0x5678000  [실행 파일 .text]                        │   │
│  │ 0x7678000  [스택]                                    │   │
│  │ 0x7ffff123 [libc.so]                                │   │
│  │                                                     │   │
│  │ 실행 2:                                             │   │
│  │ 0x5a1b000  [실행 파일 .text]                        │   │
│  │ 0x7a1b000  [스택]                                    │   │
│  │ 0x7ffff456 [libc.so]                                │   │
│  │           ↑                                         │   │
│  │           매번 바뀜 → 공격자가 주소 예측 불가       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅱ. ASLR 구성 요소

### 1. 랜덤화 대상

```
┌─────────────────────────────────────────────────────────────┐
│               ASLR 랜덤화 4가지 요소                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 스택 (Stack Base Address)                               │
│     ┌───────────────────────────────────────────────────┐  │
│     │ 무작위 스택 주소                                   │  │
│     │ 예: 0x7fffffffdc00                                │  │
│     │ → 16비트 (~65,536 조합)                          │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  2. 힙 (Heap Base Address)                                 │
│     ┌───────────────────────────────────────────────────┐  │
│     │ 무작위 힙 시작 주소                               │  │
│     │ 예: 0x555555555000                                │  │
│     │ → malloc() 반환 주소 매번 다름                   │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  3. 공유 라이브러리 (Shared Libraries)                     │
│     ┌───────────────────────────────────────────────────┐  │
│     │ libc, libm 등 로딩 주소 무작위화                  │  │
│     │ 예: libc.so @ 0x7ffff7a20000                       │  │
│     │ → system() 함수 주소 예측 불가                    │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  4. 실행 파일 (Executable Base Address)                    │
│     ┌───────────────────────────────────────────────────┐  │
│     │ PIE(Position-Independent Executable) 필요          │  │
│     │ 예: /bin/ls @ 0x555555554000                       │  │
│     │ → 코드 섹션 주소 무작위화                         │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 엔트로피 (Entropy) 크기

| 플랫폼 | 스택 | 힙 | 라이브러리 | 실행 파일 |
|:---|:---:|:---:|:---:|:---:|
| **Linux x86_64** | 28비트 | 32비트 | 28비트 | 32비트 (PIE) |
| **Linux x86_32** | 19비트 | 22비트 | 8비트 | 16비트 (PIE) |
| **Windows x64** | 13비트 | 18비트 | 25비트 | 16비트 (ASLR) |
| **macOS** | 매우 높음 | 매우 높음 | 매우 높음 | 매우 높음 |

## Ⅲ. OS별 구현

### 1. 리눅스

```bash
# ASLR 상태 확인
$ cat /proc/sys/kernel/randomize_va_space
2

# 값의 의미:
# 0 = 비활성화
# 1 = 부분 랜덤화 (mmap 베이스, 스택, VDSO)
# 2 = 완전 랜덤화 (힙 포함)

# ASLR 활성화
$ echo 2 | sudo tee /proc/sys/kernel/randomize_va_space

# 영구 설정 (/etc/sysctl.conf)
kernel.randomize_va_space = 2

# PIE 실행 파일 확인
$ file /bin/ls
/bin/ls: ELF 64-bit LSB pie executable ...  ← "pie" 확인

# 주소 확인
$ cat /proc/self/maps
555555554000-555555558000 r-xp 00000000 ... /bin/ls
# ↑ 매번 다름
```

### 2. 윈도우


## Ⅳ. PIE(Position-Independent Executable)

### 1. PIE와 ASLR의 관계

```
┌─────────────────────────────────────────────────────────────┐
│           PIE와 ASLR의 관계                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  비-PIE 실행 파일:                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 컴파일: gcc -o program program.c                   │   │
│  │                                                     │   │
│  │ 0x00400000  [실행 파일] ← 항상 고정                │   │
│  │ 0x7ffff000 [스택]     ← ASLR로 랜덤화             │   │
│  │ 0x7ffff000 [libc]     ← ASLR로 랜덤화             │   │
│  │                                                     │   │
│  │ 실행 파일 내 함수 주소 예측 가능!                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  PIE 실행 파일:                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 컴파일: gcc -fPIC -pie -o program program.c        │   │
│  │                                                     │   │
│  │ 0x56550000  [실행 파일] ← ASLR로 랜덤화!          │   │
│  │ 0x76550000  [스택]     ← ASLR로 랜덤화            │   │
│  │ 0x7f123000  [libc]     ← ASLR로 랜덤화            │   │
│  │                                                     │   │
│  │ 모든 주소 예측 불가!                                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. PIE 컴파일

```bash
# PIE 컴파일 옵션
$ gcc -fPIC -pie -o program program.c

# 확인
$ file program
program: ELF 64-bit LSB pie executable ...

# readelf로 확인
$ readelf -h program | grep Type
  Type:                              DYN (Shared object file)

# 보안 강화를 위한 컴파일 옵션
$ gcc -fPIE -pie -fstack-protector-all \
      -D_FORTIFY_SOURCE=2 -Wformat -Wformat-security \
      -o program program.c
```

## Ⅴ. 우회 기법

### 1. 정보 누출

```
┌─────────────────────────────────────────────────────────────┐
│            ASLR 우회를 위한 정보 누출 공격                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  문제: ASLR로 주소 무작위화                                 │
│                                                             │
│  해결: 다른 취약점을 통해 주소 누출                         │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                     │   │
│  │  1. Format String 취약점                           │   │
│  │     printf("%s %p %p %p\n", user_input, &a, &b, &c) │   │
│  │     → 스택 주소 노출                                  │   │
│  │                                                     │   │
│  │  2. 메모리 누출                                      │   │
│  │     read(0, buf, 0x100)  // 첫 0x100바이트 읽기    │   │
│  │     → 초기화되지 않은 스택 데이터                    │   │
│  │     → 이전 함수의 프레임 포인터(주소) 포함           │   │
│  │                                                     │   │
│  │  3. /proc/self/maps 읽기                           │   │
│  │     공격자가 이미 프로세스에 침투한 경우            │   │
│  │     → 메모리 맵 확인 가능                            │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  결과: 주소 하나 알면 → ASLR 부분 우회                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 힙 스프레이

```c
// 힙 스프레이로 ASLR 우회 시도
void heap_spray() {
    // 많은 양의 힙 할당으로 특정 주소에 데이터 배치
    for (int i = 0; i < 100000; i++) {
        char *buf = malloc(0x1000);
        // NOP 슬레드 + 쉸코드 패턴
        memset(buf, 0x90, 0x1000);  // 0x90 = NOP
        memcpy(buf + 0x1000 - sizeof(shellcode),
               shellcode, sizeof(shellcode));
    }

    // 많은 힙 영역 중 하나는 공격자가 원하는 주소에 가까울 확률 높음
    // 예: 0x0c0c0c0c와 같은 "자주 나오는" 주소 목표
}
```

## Ⅵ. 모범 사례

### 1. 완전한 ASLR 구성

```bash
# 1. 시스템 ASLR 활성화
$ sudo sysctl -w kernel.randomize_va_space=2

# 2. PIE 실행 파일 컴파일
$ gcc -fPIE -pie -o secure_app secure_app.c

# 3. PIE 라이브러리 컴파일
$ gcc -fPIC -shared -o libsecure.so secure_lib.c

# 4. 링커 강화 옵션
$ gcc -Wl,-z,relro -Wl,-z,now -o app app.c
#    ↑ RELRO         ↑ BIND_NOW
#    GOT 읽기전용    지연 로딩 비활성

# 5. 확인
$ ldd app
    linux-vdso.so.1 (0x00007ffc57f5b000)
    libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6 (0x00007f8a1f2e4000)
    ↑ 매번 다른 주소 확인
```

## 어린이를 위한 비유

**ASLR은 "집 위치 바꾸기"**와 같습니다.

**ASLR 없으면**:
- 학교가 항상 같은 주소에 있어요
- "서울시 강남구 OO로 123번지"
- 나쁜 사람이 **주소를 외우면** 언제든 찾아와

**ASLR 있으면**:
- 매일 아침마다 학교가 다른 곳에 있어요
- 월요일: "강남구 OO로 123번지"
- 화요일: "서초구 XX로 456번지"
- 수요일: "송파구 YY로 789번지"
- 나쁜 사람이 **오늘 학교가 어딘지 몰라**!

**PIE**는 "이사 전문가"와 같아요:
- ASLR이 "오늘 학교가 어딘지 알려줘"라고 하면
- PIE는 "학교 건물 자체를 분해해서 다른 곳에 다시 지어"
- 건물까지 매일 다른 곳에!

**우회하는 방법**:
- **정보 누출**: "오늘 학교 어디야?"라고 친구에게 물어보기
- **힙 스프레이**: "시내 모든 건물에 내 집을 한 번씩 만들기"

**더 막으려면**:
- **PIE + ASLR**: 건물도 매일 바꾸고 위치도 바꾸기
- **Canary**: "이상한 사람이 오면 알람"

ASLR은 **"매일 주소를 바꿔서** 나쁜 사람이 찾지 못하게 하는" 재미있는 보안 방법이에요!
