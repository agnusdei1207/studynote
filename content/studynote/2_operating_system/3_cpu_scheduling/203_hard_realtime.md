# 203. 스케줄러 튜닝 (Scheduler Tuning)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 워크로드에 맞게 스케줄러 파라미터 최적화
> 2. **가치**: 시스템 성능과 응답성 개선
> 3. **융합**: CFS, 타임 슬라이스, 우선순위와 연관

---

## Ⅰ. 개요

### 개념 정의

스케줄러 튜닝(Scheduler Tuning)은 **시스템의 워크로드 특성에 맞게 CPU 스케줄러의 파라미터를 조정하여 성능을 최적화하는 작업**이다.

### 💡 비유: 자동차 세팅
스케줄러 튜닝은 **자동차 세팅 조정**과 같다. 고속주행용, 연비용, 오프로드용 등 용도에 따라 서스펜션, 기어비, 타이어를 다르게 설정한다.

### 스케줄러 튜닝 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄러 튜닝 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 튜닝 파라미터】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  파라미터                    설명                    기본값      │ │
│  │  ────                            ────                        ────  │ │
│  │  sched_min_granularity    최소 타임 슬라이스       2.25ms       │ │
│  │  sched_wakeup_granularity 웨이크업 선점 기준       3ms          │ │
│  │  sched_migration_cost     CPU 이동 비용 임계값     0.5ms        │ │
│  │  sched_nr_migrate         마이그레이션 태스크 수    32           │ │
│  │  sched_rt_runtime_us      실시간 CPU 시간 한도     950000       │ │
│  │  sched_rt_period_us       실시간 기간              1000000      │ │
│  │  sched_cfs_bandwidth_slice CFS 대역폭 슬라이스     5ms          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【워크로드별 튜닝 방향】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  워크로드              튜닝 방향                                 │ │
│  │  ────                        ────                                  │ │
│  │  대화형 (데스크탑)      작은 min_granularity, 낮은 migration_cost  │ │
│  │  배치 (서버)           큰 min_granularity, 높은 migration_cost   │ │
│  │  실시간                낮은 rt_runtime, 높은 우선순위              │ │
│  │  HPC/계산             큰 타임 슬라이스, 높은 캐시 지역성           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【튜닝 효과】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 응답 시간 개선/저하                                            │ │
│  │  • 처리량 증가/감소                                               │ │
│  │  • 문맥 교환 빈도 변화                                             │ │
│  │  • 캐시 지역성 향상/악화                                           │ │
│  │  • 전력 소비 변화                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄러 튜닝 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【min_granularity 튜닝】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  작게 설정:                                                       │ │
│  │  • 더 많은 문맥 교환                                              │ │
│  │  • 더 나은 응답성                                                 │ │
│  │  • 더 나은 공정성                                                 │ │
│  │  • 오버헤드 증가                                                  │ │
│  │                                                             │ │
│  │  크게 설정:                                                       │ │
│  │  • 더 적은 문맥 교환                                              │ │
│  │  • 더 나은 처리량                                                 │ │
│  │  • 더 나은 캐시 효율                                              │ │
│  │  • 응답성 저하 가능                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【migration_cost 튜닝】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  높게 설정:                                                       │ │
│  │  • CPU 간 마이그레이션 감소                                        │ │
│  │  • 캐시 지역성 향상                                               │ │
│  │  • 로드 밸런싱 느려짐                                             │ │
│  │                                                             │ │
│  │  낮게 설정:                                                       │ │
│  │  • 더 적극적 로드 밸런싱                                          │ │
│  │  • 더 나은 CPU 활용률                                             │ │
│  │  • 캐시 지역성 저하 가능                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실시간 설정】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  rt_runtime / rt_period = 실시간 태스크 CPU 비율                  │ │
│  │  기본: 950000 / 1000000 = 95%                                    │ │
│  │                                                             │ │
│  │  -1로 설정: 실시간 태스크 제한 없음 (위험)                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 튜닝 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【현재 설정 확인】                                                      │
│  ──────────────────                                                  │
│  # 모든 스케줄러 파라미터 확인                                          │
│  $ ls /proc/sys/kernel/sched_*                                       │
│                                                                     │
│  # 주요 파라미터                                                       │
│  $ cat /proc/sys/kernel/sched_min_granularity_ns                     │
│  $ cat /proc/sys/kernel/sched_wakeup_granularity_ns                  │
│  $ cat /proc/sys/kernel/sched_migration_cost_ns                      │
│  $ cat /proc/sys/kernel/sched_rt_runtime_us                          │
│                                                                     │
│  【튜닝 적용】                                                            │
│  ──────────────────                                                  │
│  # 일시적 적용 (재부팅 시 초기화)                                       │
│  $ sysctl -w kernel.sched_min_granularity_ns=10000000                │
│  $ sysctl -w kernel.sched_wakeup_granularity_ns=15000000             │
│  $ sysctl -w kernel.sched_migration_cost_ns=5000000                  │
│                                                                     │
│  # 영구 적용 (/etc/sysctl.conf)                                        │
│  kernel.sched_min_granularity_ns = 10000000                          │
│  kernel.sched_wakeup_granularity_ns = 15000000                       │
│  kernel.sched_migration_cost_ns = 5000000                            │
│                                                                     │
│  【프리셋 예시】                                                          │
│  ──────────────────                                                  │
│  # 대화형/데스크탑용                                                    │
│  kernel.sched_min_granularity_ns = 1000000      # 1ms                │
│  kernel.sched_wakeup_granularity_ns = 1000000   # 1ms                │
│  kernel.sched_migration_cost_ns = 100000        # 0.1ms              │
│                                                                     │
│  # 서버/배치용                                                         │
│  kernel.sched_min_granularity_ns = 10000000     # 10ms               │
│  kernel.sched_wakeup_granularity_ns = 15000000  # 15ms               │
│  kernel.sched_migration_cost_ns = 5000000       # 5ms                │
│                                                                     │
│  # 실시간용                                                            │
│  kernel.sched_min_granularity_ns = 500000       # 0.5ms              │
│  kernel.sched_rt_runtime_us = -1                # RT 제한 없음         │
│                                                                     │
│  【튜닝 효과 측정】                                                      │
│  ──────────────────                                                  │
│  # 응답 시간 측정                                                       │
│  $ cyclictest -t1 -p 80 -n -i 10000 -l 10000                         │
│                                                                     │
│  # 문맥 교환 측정                                                       │
│  $ vmstat 1                                                          │
│                                                                     │
│  # 처리량 측정                                                          │
│  $ perf stat -e task-clock,context-switches ./benchmark             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 워크로드에 맞는 스케줄러 파라미터 조정
• 주요 파라미터: min_granularity, wakeup_granularity, migration_cost
• 대화형: 작은 슬라이스, 낮은 migration_cost
• 배치/서버: 큰 슬라이스, 높은 migration_cost
• 실시간: 낮은 rt_runtime 제한
• 적용: sysctl -w, /etc/sysctl.conf
```
