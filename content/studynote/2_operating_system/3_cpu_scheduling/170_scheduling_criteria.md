# 170. 스케줄링 기준 (Scheduling Criteria)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스케줄링 알고리즘 성능을 평가하는 척도
> 2. **가치**: 서로 상충하는 지표들의 균형 필요
> 3. **융합**: CPU 이용률, 처리량, 반환/대기/응답 시간과 연관

---

## Ⅰ. 개요

### 개념 정의

스케줄링 기준(Scheduling Criteria)은 **CPU 스케줄링 알고리즘의 성능을 평가하고 비교하기 위한 정량적 지표**다. 알고리즘 선택의 기준이 된다.

### 💡 비유: 식당 서비스 평가
스케줄링 기준은 **식당 서비스 평가 기준**과 같다. 회전율(처리량), 고객 대기 시간, 직원 효율성(CPU 이용률), 응답 속도 등 여러 지표로 평가한다.

### 스케줄링 기준 구조

```

│                스케줄링 기준 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【5대 핵심 기준】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  CPU 이용률          처리량            반환 시간              │ │
│  │  (Utilization)      (Throughput)     (Turnaround Time)       │ │
│  │  ┌─────────┐       ┌─────────┐       ┌─────────────────┐   │ │
│  │  │ CPU %   │       │ 작업/시간│       │ 도착 → 완료      │   │ │
│  │  │  ▲▲▲▲▲  │       │  ▲▲▲    │       │ ─────────────── │   │ │
│  │  │ 0-100%  │       │         │       │                 │   │ │
│  │  └─────────┘       └─────────┘       └─────────────────┘   │ │
│  │                                                             │ │
│  │  대기 시간            응답 시간                                 │ │
│  │  (Waiting Time)     (Response Time)                         │ │
│  │  ┌─────────────────┐ ┌─────────────────┐                   │ │
│  │  │ 준비 큐 대기     │ │ 요청 → 첫 응답   │                   │ │
│  │  │ ─────────────── │ │ ─────────────── │                   │ │
│  │  │                 │ │                 │                   │ │
│  │  └─────────────────┘ └─────────────────┘                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【기준 간 상충 관계】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU 이용률 ↑  ←→  응답 시간 ↓                                 │ │
│  │  처리량 ↑      ←→  공정성                                     │ │
│  │  반환 시간 ↓   ←→  문맥 교환 오버헤드                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄링 기준 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【각 기준 상세 정의】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. CPU 이용률 (CPU Utilization)                                │ │
│  │     - CPU가 실행 중인 시간 비율                                  │ │
│  │     - 0% ~ 100% (이상적: 40%-90%)                               │ │
│  │     - 너무 높으면 병목, 낮으면 자원 낭비                          │ │
│  │                                                               │ │
│  │  2. 처리량 (Throughput)                                         │ │
│  │     - 단위 시간당 완료된 프로세스 수                              │ │
│  │     - 예: 10 프로세스/초                                        │ │
│  │     - 짧은 프로세스에 유리하면 증가                               │ │
│  │                                                               │ │
│  │  3. 반환 시간 (Turnaround Time)                                 │ │
│  │     = 대기 시간 + CPU 버스트 + I/O 시간                          │ │
│  │     - 프로세스 도착부터 완료까지의 총 시간                        │ │
│  │     - 배치 시스템에서 중요                                       │ │
│  │                                                               │ │
│  │  4. 대기 시간 (Waiting Time)                                    │ │
│  │     - 준비 큐에서 대기한 총 시간                                 │ │
│  │     - CPU 버스트 사이의 대기 시간 합                             │ │
│  │     - 공정성 척도                                               │ │
│  │                                                               │ │
│  │  5. 응답 시간 (Response Time)                                   │ │
│  │     - 요청 제출부터 첫 응답까지의 시간                            │ │
│  │     - 대화형 시스템에서 중요                                      │ │
│  │     - 첫 출력까지의 시간                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시스템별 중요 기준】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템 유형            중요 기준                                │ │
│  │  ────                        ────                                │ │
│  │  배치 시스템            처리량, 반환 시간, CPU 이용률             │ │
│  │  대화형 시스템          응답 시간                                 │ │
│  │  실시간 시스템          데드라인 준수                             │ │
│  │  시분할 시스템          응답 시간, 공정성                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 측정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 성능 지표 확인】                                                │
│  ──────────────────                                                  │
│  # CPU 이용률                                                       │
│  $ top                                                              │
│  $ mpstat 1                                                         │
│  # %usr + %sys + %iowait + %idle                                    │
│                                                                     │
│  # 처리량 (프로세스 생성/완료)                                         │
│  $ vmstat 1                                                         │
│  # 프로세스 생성 (procs - r), 문맥 교환 (system - cs)                 │
│                                                                     │
│  # 대기 시간 분석                                                     │
│  $ iowait -x 1                                                      │
│  $ pidstat -w 1  # 문맥 교환                                         │
│                                                                     │
│  【프로세스별 성능 측정】                                                 │
│  ──────────────────                                                  │
│  # 특정 프로세스 CPU 시간                                              │
│  $ time ./program                                                   │
│  # real: 반환 시간, user: 사용자 CPU, sys: 커널 CPU                   │
│                                                                     │
│  # 프로세스 통계                                                       │
│  $ ps -p <pid> -o pid,vsz,rss,pcpu,pmem,time,comm                   │
│                                                                     │
│  【스케줄링 지연 측정】                                                   │
│  ──────────────────                                                  │
│  # 스케줄링 대기 시간                                                  │
│  $ perf sched record sleep 10                                       │
│  $ perf sched latency                                               │
│                                                                     │
│  # 응답 시간 측정                                                       │
│  $ perf stat -e task-clock,context-switches ./program               │
│                                                                     │
│  【알고리즘 비교 코드】                                                   │
│  ──────────────────                                                  │
│  // 스케줄링 시뮬레이션                                                 │
│  struct Process {                                                    │
│      int pid, arrival, burst;                                        │
│      int completion, turnaround, waiting, response;                  │
│  };                                                                  │
│                                                                     │
│  void calculate_metrics(Process p[], int n) {                        │
│      for (int i = 0; i < n; i++) {                                   │
│          p[i].turnaround = p[i].completion - p[i].arrival;           │
│          p[i].waiting = p[i].turnaround - p[i].burst;                │
│      }                                                               │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• CPU 이용률: CPU 사용 비율 (40-90% 이상적)
• 처리량: 단위 시간당 완료 프로세스
• 반환 시간: 도착부터 완료까지
• 대기 시간: 준비 큐 대기 총 시간
• 응답 시간: 요청부터 첫 응답까지
• 상충 관계: 이상적 알고리즘 없음
```
