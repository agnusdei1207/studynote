# 208. 스케줄링 도메인 (Scheduling Domains)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 계층 구조를 반영한 로드 밸런싱 범위
> 2. **가치**: 캐시 친화적 마이그레이션 결정
> 3. **융합**: NUMA, SMP, CPU 선호도와 연관

---

## Ⅰ. 개요

### 개념 정의

스케줄링 도메인(Scheduling Domains)은 **CPU의 물리적 계층 구조를 반영하여 로드 밸런싱을 수행하는 논리적 그룹**이다. 각 도메인은 스케줄링 그룹들의 집합으로 구성된다.

### 💡 비유: 조직 계층
스케줄링 도메인은 **회사 조직 계층**과 같다. 팀 → 부서 → 본부 식으로 계층이 있고, 업무 분배도 이 계층을 따른다.

### 스케줄링 도메인 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄링 도메인 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【계층 구조 예시 (8코어 시스템)】                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │                    ┌─────────────┐                          │ │
│  │                    │  DIE Level  │ (소켓)                    │ │
│  │                    │   도메인    │                          │ │
│  │                    └──────┬──────┘                          │ │
│  │                           │                                  │ │
│  │              ┌────────────┴────────────┐                    │ │
│  │              │                         │                    │ │
│  │        ┌─────┴─────┐           ┌─────┴─────┐               │ │
│  │        │ NUMA Node │           │ NUMA Node │               │ │
│  │        │    0      │           │    1      │               │ │
│  │        └─────┬─────┘           └─────┬─────┘               │ │
│  │              │                         │                    │ │
│  │         ┌────┴────┐              ┌────┴────┐               │ │
│  │         │         │              │         │               │ │
│  │    ┌────┴───┐ ┌───┴────┐   ┌────┴───┐ ┌───┴────┐          │ │
│  │    │MC L1   │ │MC L1   │   │MC L1   │ │MC L1   │          │ │
│  │    │CPU 0-1 │ │CPU 2-3 │   │CPU 4-5 │ │CPU 6-7 │          │ │
│  │    └────────┘ └────────┘   └────────┘ └────────┘          │ │
│  │                                                             │ │
│  │    MC = SMT (Hyperthreading) Level                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【도메인 레벨】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  레벨          설명                    마이그레이션 비용         │ │
│  │  ────                ────                        ────            │ │
│  │  SMT/MC      하이퍼스레딩 코어            낮음 (L1 공유)         │ │
│  │  MC_L2       L2 캐시 공유 코어            낮음                  │ │
│  │  MC_L3       L3 캐시 공유                 중간                  │ │
│  │  NUMA        NUMA 노드                   높음                   │
│  │  DIE         소켓                         매우 높음             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【로드 밸런싱 동작】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 낮은 레벨부터 밸런싱 시도 (낮은 비용)                         │ │
│  │  2. 실패 시 상위 레벨로 이동                                      │ │
│  │  3. 캐시 지역성 고려                                              │ │
│  │  4. 에너지 효율성 고려                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄링 도메인 상세                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【도메인 플래그】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  플래그                 설명                                    │ │
│  │  ────                       ────                                │ │
│  │  SD_LOAD_BALANCE      로드 밸런싱 활성화                        │ │
│  │  SD_BALANCE_NEWIDLE   idle 밸런싱 활성화                        │ │
│  │  SD_BALANCE_EXEC      exec 밸런싱 활성화                        │ │
│  │  SD_BALANCE_FORK      fork 밸런싱 활성화                        │ │
│  │  SD_WAKE_AFFINE       웨이크업 선호도                           │ │
│  │  SD_SHARE_CPUCAPACITY SMT 코어 공유                             │ │
│  │  SD_SHARE_PKG_RESOURCES L3 캐시 공유                            │ │
│  │  SD_NUMA              NUMA 도메인                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【밸런싱 intervals】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • busy_factor: 바쁠 때 interval 증가                           │ │
│  │  • cache_nice_tries: 캐시 웜업 시도 횟수                         │ │
│  │  • max_newidle_lb_cost: idle 밸런싱 최대 비용                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【그룹 밸런싱】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. busiest 그룹 찾기                                            │ │
│  │  2. busiest 그룹에서 busiest CPU 찾기                            │ │
│  │  3. 태스크 마이그레이션                                           │ │
│  │  4. 캐시 핫 태스크는 이동 보류                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 확인 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스케줄링 도메인 확인】                                              │
│  ──────────────────                                                  │
│  # 도메인 정보                                                        │
│  $ cat /proc/sys/kernel/sched_domain/cpu0/name                      │
│  $ cat /proc/sys/kernel/sched_domain/cpu0/domain0/name              │
│  $ cat /proc/sys/kernel/sched_domain/cpu0/domain0/flags             │
│                                                                     │
│  # 전체 도메인 계층                                                   │
│  $ ls -la /proc/sys/kernel/sched_domain/cpu0/                       │
│                                                                     │
│  # 각 도메인의 그룹                                                   │
│  $ cat /proc/sys/kernel/sched_domain/cpu0/domain0/groups            │
│                                                                     │
│  【로드 밸런싱 튜닝】                                                  │
│  ──────────────────                                                  │
│  # 밸런싱 간격                                                        │
│  $ cat /proc/sys/kernel/sched_domain/cpu0/domain0/interval          │
│                                                                     │
│  # busy factor                                                       │
│  $ cat /proc/sys/kernel/sched_domain/cpu0/domain0/busy_factor       │
│                                                                     │
│  # imbalance pct                                                     │
│  $ cat /proc/sys/kernel/sched_domain/cpu0/domain0/imbalance_pct     │
│                                                                     │
│  【NUMA 밸런싱 튜닝】                                                  │
│  ──────────────────                                                  │
│  # NUMA 밸런싱 활성화                                                 │
│  $ sysctl -w kernel.numa_balancing=1                                │
│                                                                     │
│  # NUMA 밸런싱 scan 간격                                              │
│  $ sysctl -w kernel.numa_balancing_scan_period_min_ms=1000          │
│  $ sysctl -w kernel.numa_balancing_scan_period_max_ms=60000         │
│                                                                     │
│  【디버깅】                                                            │
│  ──────────────────                                                  │
│  # 스케줄링 도메인 추적                                                │
│  $ trace-cmd record -e sched:sched_load_balance ./program          │
│  $ trace-cmd report                                                  │
│                                                                     │
│  # ftrace로 마이그레이션 추적                                         │
│  $ echo 1 > /sys/kernel/debug/tracing/events/sched/enable          │
│  $ cat /sys/kernel/debug/tracing/trace                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: CPU 계층 구조 기반 로드 밸런싱 범위
• 레벨: SMT → MC_L2 → MC_L3 → NUMA → DIE
• 마이그레이션: 낮은 레벨 우선
• 장점: 캐시 친화적, 비용 최적화
• 확인: /proc/sys/kernel/sched_domain/
• 튜닝: NUMA 밸런싱 파라미터
```
