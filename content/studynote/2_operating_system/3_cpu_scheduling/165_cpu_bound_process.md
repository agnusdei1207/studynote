# 165. CPU 바운드 프로세스 (CPU Bound Process)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 연산에 많은 시간을 보내는 프로세스
> 2. **가치**: CPU 집약적 작업, 긴 CPU 버스트
> 3. **융합**: I/O 바운드, 스케줄링 알고리즘과 연관

---

## Ⅰ. 개요

### 개념 정의

CPU 바운드 프로세스(CPU Bound Process)는 **CPU 연산에 많은 시간을 보내는 프로세스**다. 긴 CPU 버스트를 가지며 I/O 작업은 적다.

### 💡 비유: 수학 문제 푸는 학생
CPU 바운드 프로세스는 **수학 문제를 푸는 학생**과 같다. 한 자리에 앉아 계속 계산만 한다 (CPU). 책장에서 책을 찾으러(I/O) 가는 일은 거의 없다.

### 상세 분석
```

│  │  • 낮은 우선순위 → I/O 바운드가 우선                              │ │
│  │  • MLFQ에서 하위 큐로 강등                                        │ │
│  │  • SJF/FCFS에서 유리                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/O 바운드와의 상호작용】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  이상적 혼합:                                                    │ │
│  │  • I/O 바운드: 짧게 CPU 쓰고 I/O 대기                             │ │
│  │  • CPU 바운드: I/O 대기 중 CPU 활용                               │ │
│  │  • 결과: 높은 CPU 이용률 + 좋은 응답성                            │ │

│  │  • CPU 바운드만 많으면 I/O 바운드 기아                            │ │
```

## Ⅲ. 실무 적용

### 확인 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │\
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CPU 바운드 프로세스 확인】                                             │
│  ──────────────────                                                  │
│  # CPU 사용률이 높은 프로세스                                          │
│  $ top -o %CPU                                                       │
│  $ htop                                                              │
│                                                                     │
│  # 특정 프로세스 CPU 사용                                               │
│  $ pidstat -p <pid> 1                                                │
│  # %usr, %system 합이 높으면 CPU 바운드                                │
│                                                                     │
│  # I/O wait가 낮은지 확인                                              │
│  $ iostat -x 1                                                       │
│  # %iowait이 낮으면 CPU 바운드                                         │
│                                                                     │
│  【CPU 바운드 작업 예시】                                                 │
│  ──────────────────                                                  │
│  # CPU 스트레스 테스트                                                 │
│  $ stress --cpu 4 --timeout 60                                       │
│                                                                     │
│  # 암호화 (CPU 집약적)                                                 │
│  $ openssl speed                                                     │
│  $ gzip -9 largefile                                                 │
│                                                                     │
│  # 컴파일 (CPU 집약적)                                                 │
│  $ make -j4                                                          │
│                                                                     │
│  【우선순위 조정】                                                        │
│  ──────────────────                                                  │
│  # 백그라운드 CPU 작업 우선순위 낮춤                                     │
│  $ nice -n 19 ./cpu_intensive_task                                   │
│                                                                     │
│  # 실시간 스케줄링으로 CPU 보장                                         │
│  $ sudo chrt -f 50 ./cpu_intensive_task                              │
│                                                                     │
│  【CPU Affinity 설정】                                                   │
│  ──────────────────                                                  │
│  # 특정 CPU에서만 실행                                                 │
│  $ taskset -c 0,1 ./cpu_task                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: CPU 연산에 많은 시간을 보내는 프로세스
• 특징: 긴 CPU 버스트, 적은 I/O
• 예시: 과학 연산, 렌더링, 컴파일, ML
• 스케줄링: 낮은 우선순위, MLFQ 하위 큐
• 최적화: CPU Affinity, NUMA 인식
• 지표: 처리량, 반환 시간 중요
```
