# CPU 바운드 프로세스 (CPU Bound Process)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU Bound 프로세스는 전체 실행 시간 중 입출력(I/O) 대기 시간보다 **순수 산술 및 논리 연산에 사용하는 CPU 버스트(Burst) 시간이 압도적으로 긴** 연산 집약적 프로세스이다.
> 2. **가치**: 복잡한 수치 계산, 과학 기술 시뮬레이션, 동영상 인코딩 등이 대표적이며, 이러한 프로세스는 한 번 CPU를 잡으면 끝까지 쓰려 하므로 시스템의 전체 처리량(Throughput)을 결정하는 지표가 된다.
> 3. **융합**: 운영체제는 이들을 I/O 바운드 프로세스와 적절히 혼합 스케줄링하여 자원 가동률을 높이며, 대화형 응답성 확보를 위해 낮은 우선순위나 긴 시간 할당량을 부여하는 차별화 전략을 취한다.

+++

## Ⅰ. CPU Bound 프로세스의 개요

### 1. 정의
- 데이터의 입력이나 출력보다는 데이터의 '가공'과 '계산'에 대부분의 시간을 소모하는 프로세스이다.

### 2. 특징: '성실한 연산자'
- **긴 CPU Burst**: 한 번 CPU를 할당받으면 매우 오랫동안 연산을 수행한다.
- **짧은 I/O Burst**: 외부 장치와의 소통이 매우 드물다.
- **성능 결정 요인**: CPU의 클럭 속도, 코어 개수, 캐시 메모리 크기에 민감하게 반응한다.

### 3. 💡 비유: '하루 종일 수학 문제만 푸는 학생'
- CPU Bound 프로세스는 **'방 안에 틀어박혀서 밥도 안 먹고 화장실도 안 가고(I/O 없음) 오직 어려운 수학 난제만 푸는 학생'**과 같다.
- 공부하는 시간(연산)이 압도적으로 많아서, 선생님(스케줄러)이 그만하라고 말리기 전까지는 절대 펜을 놓지 않는다.

+++

## Ⅱ. 프로세스 특성 비교 분석 (Deep Dive)

| 비교 항목 | CPU Bound 프로세스 | I/O Bound 프로세스 |
|:---|:---|:---|
| **주요 작업** | 수치 계산, 암호화, 렌더링 | 키보드 입력, 파일 읽기, 통신 |
| **Burst 특징** | **긴 CPU Burst, 짧은 I/O Burst** | 짧은 CPU Burst, 긴 I/O Burst |
| **스케줄링 우선순위** | 상대적 낮음 (공평성 위주) | **상대적 높음 (응답성 위주)** |
| **장치 이용률** | CPU 이용률 극대화 | I/O 장치 이용률 극대화 |

### ASCII 구조 다이어그램 (Burst Cycle)
```text
    [ CPU Bound ]
    CPU: ■■■■■■■■■■■■■■■■■■■■■■■■■■■■ (Long Burst)
    I/O: □ (Very Short)
    
    [ I/O Bound ]
    CPU: ■ (Short)
    I/O: □□□□□□□□□□□□□□□□□□□□□□□□□□□□ (Long Wait)
```

- **📢 섹션 요약 비유**: 마라톤 선수(CPU Bound)는 한 번 뛰면 오래 달리고, 단거리 선수(I/O Bound)는 짧게 폭발적으로 뛰고 오래 쉬는 것과 같습니다.

## Ⅲ. CPU Bound를 위한 스케줄링 전략

운영체제는 시스템 전체 효율을 위해 CPU Bound 프로세스를 다음과 같이 다룬다.

### 1. 시간 할당량 (Time Quantum)의 조절
- CPU Bound 프로세스에게는 짧은 시간을 자주 주는 것보다, **긴 시간 할당량**을 주어 문맥 교환 오버헤드를 줄이는 것이 전체 처리량 향상에 유리하다.

### 2. 우선순위 강등
- 다단계 피드백 큐(MLFQ) 환경에서, 할당된 시간을 다 소모한 프로세스는 자동으로 하위 큐로 밀려난다. 연산 위주의 작업은 자연스럽게 낮은 우선순위로 이동하여 대화형 작업을 방해하지 않게 된다.

- **📢 섹션 요약 비유**: 오래 걸리는 요리를 주문한 손님은 느긋하게 기다릴 수 있는 안쪽 자리(하위 큐)로 안내하고, 금방 먹고 갈 손님은 카운터 앞(상위 큐)에 앉히는 것과 같습니다.

+++

## Ⅳ. 실무 적용 및 최적화 인사이트 (Strategy)

### 1. 실무 시나리오: 비트코인 채굴 및 동영상 인코딩
- **상황**: 서버에서 고성능 연산이 필요한 채굴 앱을 돌리고 있음.
- **판단**: 이 앱은 전형적인 **CPU Bound**이다. 다른 웹 서비스 프로세스의 응답성을 해치지 않도록 `nice -n 19` 명령을 통해 우선순위를 최하로 낮춘다. CPU가 놀 때만 연산을 수행하게 함으로써, 시스템의 가용성을 유지하면서 유휴 자원을 100% 활용한다.

### 2. 기술사적 인사이트: 자원 혼합 (Process Mix)의 묘미
- 기술사라면 CPU Bound만 가득한 시스템은 I/O 장치가 놀게 되고, I/O Bound만 가득한 시스템은 CPU가 놀게 되는 **'자원 불균형'** 문제를 지적해야 한다. 장기 스케줄러(Job Scheduler) 단계에서 두 부류를 적절히 섞어 메모리에 올리는 것이 시스템 전체 처리량(Throughput)을 높이는 최고의 아키텍처적 솔루션임을 강조해야 한다.

- **📢 섹션 요약 비조**: 편식하지 않고 골고루 먹어야 건강하듯, 운영체제도 연산 작업과 소통 작업을 골고루 섞어 실행해야 건강한 시스템이 됩니다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **처리량 극대화**: 연산 자원의 낭비 없는 활용으로 대규모 작업 완료 시간 단축.
- **공정성 확보**: 독점적 프로세스의 제어를 통한 다중 사용자 환경 안정화.

### 2. 미래 전망
최근에는 CPU Bound 작업들이 GPU나 TPU와 같은 전용 가속기로 이동하는 **오프로딩 (Offloading)** 기술이 일반화되고 있다. 운영체제는 이제 CPU뿐만 아니라 이기종 가속기의 버스트 특성까지 파악하여 작업을 분배하는 '헤테로지니어스 스케줄링' 능력을 요구받고 있다. CPU Bound에 대한 깊은 이해는 모든 고성능 컴퓨팅 최적화의 시작점이다.

- **📢 섹션 요약 비유**: 몸을 쓰는 일(CPU)과 머리를 쓰는 일(I/O)을 적절히 나누어 맡기는 유능한 팀장의 지혜가 담긴 기술입니다.
