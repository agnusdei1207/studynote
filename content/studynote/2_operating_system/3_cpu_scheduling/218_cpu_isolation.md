# 218. CPU 격리 (CPU Isolation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 특정 CPU를 일반 스케줄링에서 제외
> 2. **가치**: 실시간 성능 보장, 저지연 달성
> 3. **융합**: nohz_full, RCU, 인터럽트와 연관

---

## Ⅰ. 개요

### 개념 정의

CPU 격리(CPU Isolation)는 **특정 CPU 코어를 일반적인 스케줄링에서 제외하여, 지정된 태스크만 실행하도록 만드는 기법**이다. 실시간 애플리케이션과 고성능 컴퓨팅에 사용된다.

### 💡 비유: 전용 회의실
CPU 격리는 **전용 회의실 예약**과 같다. 다른 팀은 못 들어오고, 우리 팀만 쓴다. 방해받지 않고 집중한다.

### CPU 격리 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                CPU 격리 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【격리 전후 비교】                                                    │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  격리 전:                                                       │ │
│  │  ┌───────────────────────────────────────────────────────┐   │ │
│  │  │ CPU 0: 시스템 태스크, 사용자 태스크, 인터럽트            │   │ │
│  │  │ CPU 1: 시스템 태스크, 사용자 태스크, 인터럽트            │   │ │
│  │  │ CPU 2: 시스템 태스크, 사용자 태스크, 인터럽트            │   │ │
│  │  │ CPU 3: 시스템 태스크, 사용자 태스크, 인터럽트            │   │ │
│  │  └───────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  │  격리 후:                                                       │ │
│  │  ┌───────────────────────────────────────────────────────┐   │ │

│  │  │ CPU 1: 시스템 태스크, 인터럽트 (일반)                   │   │ │
│  │  │ CPU 2: 실시간 애플리케이션만 ★ 격리 ★                  │   │ │
│  │  │ CPU 3: 실시간 애플리케이션만 ★ 격리 ★                  │   │ │
│  │  └───────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【격리 수준】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  수준              설명                    효과                  │ │

│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【격리 효과】                                                         │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                           │ │
│  │  • 일관된 성능                                                   │ │
│  │  • 최소 지연                                                     │ │
│  │  • 예측 가능한 실행                                              │ │
│  │  • 캐시 지역성 극대화                                            │ │
│  │                                                             │ │
│  │  단점:                                                           │ │
│  │  • CPU 자원 낭비 가능                                            │ │
│  │  • 설정 복잡                                                     │ │
│  │  • 로드 밸런싱 제한                                              │ │
│  │                                                             │ │
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                CPU 격리 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【격리 파라미터】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. isolcpus=CPU 리스트                                          │ │
│  │     • 스케줄러가 해당 CPU 사용 안 함                             │ │
│  │     • 예: isolcpus=2,3 또는 isolcpus=2-3                       │ │
│  │                                                             │ │
│  │  2. nohz_full=CPU 리스트                                         │ │
│  │     • 해당 CPU에서 tick 중단                                     │ │
│  │     • 단일 태스크 실행 시                                        │ │
│  │                                                             │ │
│  │  3. rcu_nocbs=CPU 리스트                                         │ │
│  │     • RCU 콜백을 다른 CPU로 오프로드                             │ │
│  │     • nohz_full과 함께 사용                                     │ │
│  │                                                             │ │
│  │  4. irqaffinity=CPU 마스크                                       │ │
│  │     • 인터럽트 처리 CPU 지정                                     │ │
│  │     • 격리 CPU 제외                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【추가 튜닝】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  # mce를 격리 CPU에서 제외                                       │ │
│  │  mce=off                                                         │
│  │                                                             │ │
│  │  # 최대 성능을 위한 주파수 고정                                   │ │
│  │  cpufreq.default_governor=performance                           │ │
│  │                                                             │ │
│  │  # NUMA 밸런싱 비활성화                                          │ │
│  │  numa_balancing=disable                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 설정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【커널 파라미터 설정】                                                │
│  ──────────────────                                                  │
│  # /etc/default/grub                                                 │
│  GRUB_CMDLINE_LINUX="isolcpus=2,3 nohz_full=2,3 rcu_nocbs=2,3"     │
│                                                                     │
│  # 적용                                                              │
│  $ sudo update-grub                                                 │
│  $ sudo reboot                                                      │
│                                                                     │
│  【격리 확인】                                                         │
│  ──────────────────                                                  │
│  # 격리된 CPU 확인                                                    │
│  $ cat /sys/devices/system/cpu/isolated                             │
│  2,3                                                                │
│                                                                     │
│  # nohz_full 확인                                                    │
│  $ cat /sys/devices/system/cpu/nohz_full                            │
│  2,3                                                                │
│                                                                     │
│  # 현재 CPU에서 실행 중인 프로세스                                     │
│  $ ps -Leo pid,tid,psr,comm | grep -E "^\s*[0-9]+\s+[0-9]+\s+[23]" │
│                                                                     │
│  【격리 CPU에서 태스크 실행】                                          │
│  ──────────────────                                                  │
│  # taskset으로 CPU 지정                                              │
│  $ taskset -c 2,3 ./realtime_app                                    │
│                                                                     │
│  # cgroup으로 설정                                                   │
│  $ mkdir /sys/fs/cgroup/cpuset/realtime                             │
│  $ echo 2,3 > /sys/fs/cgroup/cpuset/realtime/cpuset.cpus           │
│  $ echo <pid> > /sys/fs/cgroup/cpuset/realtime/cgroup.procs        │
│                                                                     │
│  # 실시간 우선순위와 함께                                             │
│  $ chrt -f 80 taskset -c 2 ./realtime_app                           │
│                                                                     │
│  【인터럽트 격리】                                                     │
│  ──────────────────                                                  │
│  # 격리 CPU에서 인터럽트 제외                                         │
│  $ for irq in /proc/irq/*/smp_affinity; do                         │
│        echo 03 > $irq  # CPU 0, 1만                                │
│    done                                                             │
│                                                                     │
│  # 특정 인터럽트 확인                                                 │
│  $ cat /proc/irq/default_smp_affinity                              │
│  03                                                                 │
│                                                                     │
│  【성능 측정】                                                         │
│  ──────────────────                                                  │
│  # 격리 전후 cyclictest 비교                                         │
│  $ cyclictest -t1 -p 80 -n -i 10000 -l 10000                        │
│                                                                     │
│  # CPU 사용량 확인                                                   │
│  $ mpstat -P ALL 1                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
