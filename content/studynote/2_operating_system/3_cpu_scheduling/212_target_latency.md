# 212. 스탑 머신 (Stop Machine)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 CPU를 중지시키고 원자적 작업 수행
> 2. **가치**: 커널 핫플러그, 모듈 로딩에 필수
> 3. **융합**: 동기화, RCU, CPU 핫플러그와 연관

---

## Ⅰ. 개요

### 개념 정의

스탑 머신(Stop Machine)은 **시스템의 모든 CPU를 중지시키고 하나의 CPU에서만 원자적으로 작업을 수행하는 메커니즘**이다. 커널의 핵심 데이터 구조 변경에 사용된다.

### 💡 비유: 모두 멈춤
스탑 머신은 **체육관에서 전체 준비운동**과 같다. 모두가 자신의 운동을 멈추고, 하나의 지시를 따른다.

│  │  3. 인터럽트 비활성화                                             │ │

│  │                                                             │ │

│                                                                     │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 기본 stop_machine                                            │ │
│  │  int stop_machine(int (*fn)(void *), void *data,                │ │
│  │                    const struct cpumask *cpus);                 │ │
│  │                                                             │ │
│  │  // CPU 핫플러그용                                                │ │
│  │  int stop_machine_cpuslocked(...);                              │ │

│  │  // 대안: cpus_read_lock() 사용                                  │ │

│  │  • static_key: 조건부 코드 경로                                  │ │

## Ⅲ. 실무 적용

### 커널 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용 (커널 개발자용)                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【stop_machine 사용 예시】                                            │
│  ──────────────────                                                  │
│  #include <linux/stop_machine.h>                                    │
│                                                                     │
│  // 콜백 함수                                                         │
│  static int my_callback(void *data) {                               │
│      struct my_data *d = data;                                      │
│      // 원자적으로 수행할 작업                                         │ │
│      d->result = modify_global_state();                             │
│      return 0;                                                      │
│  }                                                                  │
│                                                                     │
│  // stop_machine 호출                                                │
│  int perform_atomic_update(void) {                                  │
│      struct my_data data;                                           │
│      int ret;                                                       │
│                                                                     │
│      // 모든 CPU 정지 후 작업 수행                                     │
│      ret = stop_machine(my_callback, &data, cpu_online_mask);       │
│                                                                     │
│      return ret;                                                    │
│  }                                                                  │
│                                                                     │
│  【모듈 로딩에서의 사용】                                               │
│  ──────────────────                                                  │
│  // kernel/module.c                                                 │
│  static int add_module(struct module *mod) {                        │
│      // 모듈 리스트에 추가 - stop_machine 사용                        │
│      return stop_machine(__add_module, mod, NULL);                  │
│  }                                                                  │
│                                                                     │
│  【ftrace에서의 사용】                                                 │
│  ──────────────────                                                  │
│  // ftrace 활성화 시                                                 │
│  int ftrace_modify_code(unsigned long ip, unsigned char *new_code)  │
│  {                                                                  │
│      // stop_machine으로 안전하게 코드 수정                           │
│      return stop_machine(__ftrace_modify_code, &fdata, NULL);       │
│  }                                                                  │
│                                                                     │
│  【대안 사용 고려】                                                    │
│  ──────────────────                                                  │
│  // stop_machine 대신 RCU 사용                                       │
│  rcu_read_lock();                                                   │
│  // 읽기 작업                                                        │
│  rcu_read_unlock();                                                 │
│                                                                     │
│  // 쓰기는 RCU로 보호                                                 │
│  synchronize_rcu();                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 모든 CPU 중지 후 원자적 작업
• 동작: IPI → 스케줄러 정지 → 콜백 → 재개
• 사용: 모듈 로딩, CPU 핫플러그, ftrace
• 위험: 시스템 전체 정지
• API: stop_machine()
• 대안: RCU, text_poke, static_key
```
