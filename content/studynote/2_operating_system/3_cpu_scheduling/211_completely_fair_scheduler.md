# 211. BFS 스케줄러 (Brain Fuck Scheduler)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데스크탑/게이밍 최적화 단순 스케줄러
> 2. **가치**: 낮은 지연, 높은 응답성
> 3. **융합**: CFS, O(1), MuQSS와 연관

---

## Ⅰ. 개요

### 개념 정의

BFS(Brain Fuck Scheduler)는 **Con Kolivas가 개발한 데스크탑 및 게이밍 워크로드에 최적화된 단순한 CPU 스케줄러**다. 낮은 지연 시간과 높은 응답성을 목표로 한다.

### 💡 비유: 게이밍 마우스
BFS 스케줄러는 **게이밍 마우스**와 같다. 복잡한 기능보다 빠른 반응속도에 집중한다.

### BFS 스케줄러 구조

```

│  │                                                             │ │
│  │  • 단일 runqueue (전역 큐)                                      │ │
│  │  • O(n) 검색 (단순하지만 예측 가능)                              │ │
│  │  • 낮은 지연 우선                                               │ │
│  │  • 적은 CPU 수에 최적화 (≤16 코어)                              │ │

│                                                                     │
│  【vs CFS 비교】                                                       │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  CFS                          BFS                           │ │

│  │  • O(log n)                   • O(n)                        │ │
│  │  • 완전 공정                   • 응답성 우선                   │ │
│  │  • 대규모 시스템               • 소규모 시스템                  │ │

│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  •Earliest Effective Time First (EEVDF) 기반                   │ │
│  │  • Iso priority (대화형 태스크)                                 │ │
│  │  • SMT-nice (하이퍼스레딩 고려)                                 │ │

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                BFS 스케줄러 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Iso Priority】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  대화형 태스크를 위한 특별한 우선순위:                            │ │
│  │                                                             │ │
│  │  • 일반 nice 범위: -20 ~ +19                                   │ │
│  │  • Iso priority: 0 ~ -20 (실시간과 유사)                       │ │
│  │  • CPU 독점 방지                                               │ │
│  │  • 빠른 응답 보장                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vruntime 대체】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  BFS는 virtual deadline 사용:                                   │ │
│  │                                                             │ │
│  │  deadline = jiffies + (nice_slice * penalty)                 │ │
│  │                                                             │ │
│  │  • 가장 빠른 deadline 선택                                      │ │
│  │  • 간단한 계산                                                  │ │
│  │  • 예측 가능한 동작                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 특성】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                           │ │
│  │  • 낮은 오디오/비디오 지연                                       │ │
│  │  • 게임에서 높은 FPS                                             │ │
│  │  • 빠른 인터랙티브 응답                                          │ │
│  │                                                             │ │
│  │  단점:                                                           │ │
│  │  • 많은 CPU에서 확장성 낮음                                      │ │
│  │  • 표준 커널에 포함 안 됨                                        │ │
│  │  • 서버 워크로드에 부적합                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
