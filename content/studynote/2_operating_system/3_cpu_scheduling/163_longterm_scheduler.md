# 163. 장기 스케줄러 (Long-term Scheduler)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 잡 스케줄러, 디스크에서 메모리로 프로세스 승인
> 2. **가치**: 다중프로그래밍 정도 제어
> 3. **융합**: 단기/중기 스케줄러, 잡 큐와 연관

---

## Ⅰ. 개요

### 개념 정의

장기 스케줄러(Long-term Scheduler)는 **디스크의 잡 풀에서 메모리로 적재할 프로세스를 선택하는 스케줄러**다. 잡 스케줄러(job scheduler)라고도 하며, 다중프로그래밍 정도(degree of multiprogramming)를 제어한다.

### 💡 비유: 공장 입구 관리자
장기 스케줄러는 **공장 입구 관리자**와 같다. 공장(메모리)에 들어올 주문(프로세스)을 승인하고, 공장이 너무 붐비지 않도록 관리한다.

### 장기 스케줄러 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                장기 스케줄러 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스케줄러 비교】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장기 스케줄러           중기 스케줄러           단기 스케줄러    │ │
│  │  ────────────           ─────────────          ────────────   │ │
│  │                                                             │ │
│  │  디스크 → 메모리         메모리 ↔ 디스크        준비 → 실행       │ │
│  │  (잡 승인)              (스와핑)              (CPU 할당)        │ │
│  │                                                             │ │
│  │  빈도: 분/시간           빈도: 초/분           빈도: ms          │ │
│  │  속도: 느림              속도: 중간            속도: 빠름        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【잡 스케줄링 과정】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  디스크                  메모리                              │ │
│  │  ┌─────────────────┐     ┌─────────────────────────────┐    │ │
│  │  │ 잡 풀            │     │                           │    │ │
│  │  │ (Job Pool)       │     │   메모리의 프로세스들       │    │ │
│  │  │ ┌─────┐┌─────┐  │     │   ┌─────┐┌─────┐┌─────┐  │    │ │
│  │  │ │ Job ││ Job │  │     │   │ P1  ││ P2  ││ P3  │  │    │ │
│  │  │ │  1  ││  2  │  │     │   └─────┘└─────┘└─────┘  │    │ │
│  │  │ └─────┘└─────┘  │     │                           │    │ │
│  │  │       │         │     │   다중프로그래밍 정도       │    │ │
│  │  │       │ 승인    │     │   (= 메모리 내 프로세스 수)  │    │ │
│  │  │       ▼         │     │                           │    │ │
│  │  │ ┌──────────────┐│     └─────────────────────────────┘    │ │
│  │  │ │ 장기 스케줄러 │├──────►                                  │ │
│  │  │ │ (잡 승인)    ││                                         │ │
│  │  │ └──────────────┘│                                         │ │
│  │  └─────────────────┘                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【다중프로그래밍 정도】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  낮음 ─────────────────────────────────────► 높음             │ │
│  │                                                             │ │
│  │  CPU 이용률: 낮음                    CPU 이용률: 높음          │ │
│  │  문맥 교환: 적음                     문맥 교환: 많음           │ │
│  │  메모리: 여유                        메모리: 부족 가능          │ │
│  │  스래싱: 없음                        스래싱: 발생 가능          │ │
│  │                                                             │ │
│  │  장기 스케줄러가 이 정도를 적절히 유지                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                장기 스케줄러 상세                                       │

│  【잡 선택 기준】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  기준                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  우선순위               사용자/시스템 중요도                     │ │
│  │  예상 실행 시간          짧은 작업 우선                         │ │
│  │  I/O vs CPU 바운드      I/O 바운드 우선 (CPU 활용)              │ │
│  │  메모리 요구량            메모리 가용성 고려                     │ │
│  │  자원 요구               필요한 자원 가용성                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/O 바운드 vs CPU 바운드 혼합】                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • I/O 바운드: CPU 짧게 사용, I/O 대기 많음                      │ │
│  │  • CPU 바운드: CPU 길게 사용, I/O 적음                           │ │
│  │  • 혼합하면 CPU 이용률 최적화                                    │ │
│  │  • I/O 바운드 프로세스가 많으면 CPU 낭비                         │ │
│  │  • CPU 바운드 프로세스가 많으면 대기 증가                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【현대 시스템에서의 장기 스케줄러】                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 시분할 시스템: 사실상 없음                                    │ │
│  │  • 사용자가 프로세스 시작하면 즉시 메모리 적재                    │ │
│  │  • 대신 중기 스케줄러가 메모리 관리                               │ │
│  │  • 배치 시스템: 여전히 중요                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 시스템 안정성            • 응답성 저하                        │ │
│  │  • 자원 효율성              • 대기 시간 증가                     │ │
│  │  • 스래싱 방지              • 시분할에서는 거의 사용 안 함         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 개념 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【배치 시스템에서의 잡 스케줄링】                                         │
│  ──────────────────                                                  │
│  # 잡 제출                                                           │
│  $ qsub my_job.sh                  # PBS/Torque                      │
│  $ sbatch my_job.sh                # Slurm                           │
│  $ condor_submit my_job.condor     # HTCondor                        │
│                                                                     │
│  # 큐 상태 확인                                                       │
│  $ qstat                                                            │
│  $ squeue                                                           │
│                                                                     │
│  【Slurm 잡 스케줄러 예시】                                              │
│  ──────────────────                                                  │
│  #!/bin/bash                                                         │
│  #SBATCH --job-name=my_job                                           │
│  #SBATCH --time=01:00:00                                             │
│  #SBATCH --mem=4G                                                    │
│  #SBATCH --cpus-per-task=2                                           │
│                                                                     │
│  ./my_program                                                        │
│                                                                     │
│  【Linux 프로세스 제한】                                                 │
│  ──────────────────                                                  │
│  # 사용자당 최대 프로세스 수                                           │
│  $ ulimit -u                                                         │
│  $ cat /etc/security/limits.conf                                     │
│                                                                     │
│  # cgroup으로 프로세스 수 제한                                         │
│  $ cat /sys/fs/cgroup/pids/max                                       │
│                                                                     │
│  【시스템 부하 확인】                                                    │
│  ──────────────────                                                  │
│  $ uptime                                                            │
│  load average: 0.50, 0.60, 0.55                                      │
│  # 실행 중 + 대기 중인 프로세스 평균                                    │
│                                                                     │
│  $ cat /proc/loadavg                                                 │
│  0.50 0.60 0.55 2/500 12345                                          │
│  #            실행/전체  마지막 PID                                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 잡 스케줄러, 디스크 → 메모리 승인
• 역할: 다중프로그래밍 정도 제어
• 빈도: 분~시간 단위 (느림)
• 기준: 우선순위, 실행 시간, 자원 요구
• 현대: 시분할에서는 거의 사용 안 함

```
