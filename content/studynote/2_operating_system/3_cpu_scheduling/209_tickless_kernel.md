# 209. 틱리스 커널 (Tickless Kernel)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 주기적 타이머 틱을 제거한 커널
> 2. **가치**: 전력 절약, 정밀 타이밍, 오버헤드 감소
> 3. **융합**: CFS, 전원 관리, 실시간 시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

틱리스 커널(Tickless Kernel)은 **주기적인 타이머 인터럽트(tick)를 제거하거나 최소화하여 전력 소모와 오버헤드를 줄이는 커널 설계**다. CONFIG_NO_HZ로 설정한다.

### 💡 비유: 깨우지 않기
틱리스 커널은 **자는 아이를 깨우지 않기**와 같다. 일정 시간마다 깨우면 잠을 못 자니, 필요할 때만 깨운다.

### 틱리스 커널 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                틱리스 커널 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전통적 Periodic Tick】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시간 ──────────────────────────────────────────────────►    │ │
│  │                                                             │ │
│  │  Tick:  ↓    ↓    ↓    ↓    ↓    ↓    ↓    ↓    ↓           │ │
│  │         │    │    │    │    │    │    │    │    │            │ │
│  │        0   10   20   30   40   50   60   70   80ms           │ │
│  │                                                             │ │
│  │  문제점:                                                       │ │
│  │  • 유휴 시에도 깨어남 (전력 소모)                                │ │
│  │  • 불필요한 오버헤드                                           │ │
│  │  • 정확한 타이밍 어려움                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Tickless (NO_HZ)】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시간 ──────────────────────────────────────────────────►    │ │
│  │                                                             │ │
│  │  Tick:  ↓                        ↓                          │ │
│  │         │                        │                           │ │
│  │        0ms                      75ms                         │ │
│  │         │←────── 75ms 슬립 ──────→│                          │ │
│  │                                                             │ │
│  │  장점:                                                         │ │
│  │  • 유휴 시 깊은 절전 모드 진입                                  │ │
│  │  • 오버헤드 감소                                               │ │
│  │  • 정밀한 타이밍                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NO_HZ 모드】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  모드                    설명                                  │ │
│  │  ────                        ────                              │ │
│  │  CONFIG_NO_HZ_IDLE    idle 시 틱 중단 (기본)                   │ │
│  │  CONFIG_NO_HZ_FULL    태스크 실행 시에도 틱 중단               │ │
│  │  CONFIG_NO_HZ         하위 호환용 (IDLE과 동일)                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                틱리스 커널 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NO_HZ_IDLE 동작】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. CPU가 idle 진입                                             │ │
│  │  2. 다음 예정된 이벤트까지 타이머 프로그램                        │ │
│  │  3. tick 멈춤                                                   │ │
│  │  4. 이벤트 발생 시 깨어남                                        │ │
│  │  5. tick 재시작                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NO_HZ_FULL 동작】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  목적: 실시간 애플리케이션을 위한 틱 제거                        │ │
│  │                                                             │ │
│  │  1. 특정 CPU를 adaptive-ticks에서 제외                         │ │
│  │  2. 해당 CPU에서 실행 중인 태스크가 1개면 tick 중단              │ │
│  │  3. 시스템 전체 tick 영향 제거                                  │ │
│  │                                                             │ │
│  │  사용 사례:                                                      │ │
│  │  • 고주파 트레이딩                                              │ │
│  │  • 실시간 오디오/비디오                                         │ │
│  │  • HPC 워크로드                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【High-Resolution Timers】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 나노초 정밀도 타이머                                          │ │
│  │  • tickless와 함께 사용                                         │ │
│  │  • CONFIG_HIGH_RES_TIMERS                                      │ │
│  │  • hrtimer 서브시스템                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 설정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【커널 설정 확인】                                                    │
│  ──────────────────                                                  │
│  # 현재 설정                                                          │
│  $ grep NO_HZ /boot/config-$(uname -r)                              │
│  CONFIG_NO_HZ=y                                                     │
│  CONFIG_NO_HZ_IDLE=y                                                │
│  # 또는                                                              │
│  CONFIG_NO_HZ_FULL=y                                                │
│                                                                     │
│  # High-Resolution Timers                                            │
│  $ grep HIGH_RES_TIMERS /boot/config-$(uname -r)                    │
│  CONFIG_HIGH_RES_TIMERS=y                                           │
│                                                                     │
│  【NO_HZ_FULL 설정】                                                  │
│  ──────────────────                                                  │
│  # 커널 부트 파라미터                                                 │
│  nohz_full=4-7    # CPU 4-7에서 full nohz                          │
│  rcu_nocbs=4-7    # RCU 콜백 오프로드                               │
│                                                                     │
│  # /etc/default/grub                                                │
│  GRUB_CMDLINE_LINUX="nohz_full=4-7 rcu_nocbs=4-7"                   │
│                                                                     │
│  $ sudo update-grub                                                 │
│  $ sudo reboot                                                      │
│                                                                     │
│  【현재 상태 확인】                                                    │
│  ──────────────────                                                  │
│  # tick 상태                                                         │
│  $ cat /sys/kernel/debug/tickless/                                  │
│                                                                     │
│  # nohz_full 확인                                                    │
│  $ cat /sys/devices/system/cpu/nohz_full                            │
│                                                                     │
│  # CPU별 통계                                                        │
│  $ grep . /sys/kernel/debug/tickless/broadcast/*                    │
│                                                                     │
│  【전력 절약 효과 측정】                                               │
│  ──────────────────                                                  │
│  # CPU idle 상태 확인                                                 │
│  $ cpupower idle-info                                               │
│                                                                     │
│  # idle 통계                                                         │
│  $ cat /sys/devices/system/cpu/cpu0/cpuidle/*/time                  │
│  $ cat /sys/devices/system/cpu/cpu0/cpuidle/*/usage                 │
│                                                                     │
│  # 전력 소모 측정 (Intel)                                             │
│  $ cat /sys/class/powercap/intel-rapl/intel-rapl:0/energy_uj        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 주기적 tick 제거/최소화
• NO_HZ_IDLE: idle 시 tick 중단
• NO_HZ_FULL: 실행 중 tick 중단
• 장점: 전력 절약, 정밀 타이밍
• High-Res Timers: 나노초 정밀도
• 설정: nohz_full= 커널 파라미터
```
