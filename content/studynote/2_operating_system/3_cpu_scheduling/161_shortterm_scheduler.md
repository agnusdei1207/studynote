# 161. 단기 스케줄러 (Short-term Scheduler)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 스케줄러, 준비 큐에서 실행할 프로세스 선택
> 2. **가치**: CPU 활용도 최적화, 빈번한 실행
> 3. **융합**: 디스패처, 문맥 교환, 스케줄링 알고리즘과 연관

---

## Ⅰ. 개요

### 개념 정의

단기 스케줄러(Short-term Scheduler)는 **준비 큐에서 실행할 프로세스를 선택하고 CPU를 할당하는 스케줄러**다. CPU 스케줄러라고도 하며, 매우 빈번하게 실행된다.

### 💡 비유: 식당의 주문 처리 담당
단기 스케줄러는 **식당의 주문 처리 담당자**와 같다. 대기 손님(준비 큐) 중에서 주방(CPU)이 처리할 주문을 선택한다.

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux CFS 스케줄러】                                                 │
│  ──────────────────                                                  │
│  // /proc/sched_debug으로 스케줄러 정보 확인                         │
│  $ cat /proc/sched_debug                                            │
│                                                                     │
│  // 스케줄러 통계                                                     │
│  $ cat /proc/stat | grep -E "^cpu|^ctxt"                            │
│  cpu  2255 34 2290 22625563 6290 0 176 0 0 0                        │
│  ctxt 43350000   // 문맥 교환 횟수                                    │
│                                                                     │
│  【스케줄링 정책 확인】                                                  │
│  ──────────────────                                                  │
│  // 프로세스의 스케줄링 정책                                           │
│  $ chrt -p $$                                                        │
│  pid 1234's current scheduling policy: SCHED_OTHER                  │
│  pid 1234's current scheduling priority: 0                          │
│                                                                     │
│  【nice 값 조정】                                                       │
│  ──────────────────                                                  │
│  // 우선순위 낮춤                                                     │
│  $ nice -n 10 ./cpu_intensive_task                                   │
│                                                                     │
│  // 실행 중인 프로세스 우선순위 변경                                    │
│  $ renice -n -5 -p 1234                                              │
│                                                                     │
│  【시스템 호출】                                                         │
│  ──────────────────                                                  │
│  #include <sched.h>                                                 │
│  #include <sys/resource.h>                                          │
│                                                                     │
│  // 우선순위 확인/설정                                                 │
│  int prio = getpriority(PRIO_PROCESS, 0);                           │
│  setpriority(PRIO_PROCESS, 0, -5);  // 더 높은 우선순위               │
│                                                                     │
│  // 스케줄링 정책 설정                                                 │
│  struct sched_param param;                                          │
│  param.sched_priority = 50;                                         │
│  sched_setscheduler(0, SCHED_FIFO, &param);                         │
│                                                                     │

```
