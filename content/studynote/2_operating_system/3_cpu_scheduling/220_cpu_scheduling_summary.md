# 220. CPU 스케줄링 총정리 (CPU Scheduling Summary)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 자원을 태스크에 공정하고 효율적으로 할당
> 2. **가치**: 시스템 성능, 응답성, 처리량 최적화
> 3. **융합**: 모든 CPU 스케줄링 개념의 통합

---

## Ⅰ. 개요

### 개념 정의

CPU 스케줄링 총정리는 **운영체제에서 CPU 자원을 관리하는 모든 기법과 알고리즘의 종합적인 정리**다. 기본 개념부터 고급 기법까지 체계적으로 이해한다.

### 💡 비유: 오케스트라 지휘
CPU 스케줄링은 **오케스트라 지휘자**와 같다. 많은 연주자(태스크)가 있고, 지휘자(스케줄러)가 누가 언제 연주할지 결정한다.

## Ⅲ. 실무 적용

### 종합 설정 가이드
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용 (종합)                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시스템 진단】                                                       │
│  ──────────────────                                                  │
│  # CPU 정보                                                          │
│  $ lscpu                                                             │
│  $ cat /proc/cpuinfo                                                 │
│                                                                     │
│  # 현재 스케줄러 설정                                                 │
│  $ cat /proc/sys/kernel/sched_*                                      │
│                                                                     │
│  # CPU 활용률                                                        │
│  $ top -H                                                            │
│  $ htop                                                              │
│  $ mpstat -P ALL 1                                                   │
│                                                                     │
│  【성능 튜닝】                                                         │
│  ──────────────────                                                  │
│  # 데스크탑용                                                         │
│  kernel.sched_min_granularity_ns = 1000000                          │
│  kernel.sched_wakeup_granularity_ns = 1000000                       │
│  kernel.sched_migration_cost_ns = 100000                            │
│                                                                     │
│  # 서버용                                                             │
│  kernel.sched_min_granularity_ns = 10000000                         │
│  kernel.sched_wakeup_granularity_ns = 15000000                      │
│  kernel.sched_migration_cost_ns = 5000000                           │
│                                                                     │
│  # 실시간용                                                           │
│  kernel.sched_rt_runtime_us = -1                                     │
│  # PREEMPT_RT 커널 권장                                               │
│                                                                     │
│  【문제 해결】                                                         │
│  ──────────────────                                                  │
│  # 높은 지연 분석                                                     │
│  $ cyclictest -t1 -p 80 -n -i 10000 -l 10000                        │
│  $ perf sched record -- sleep 60                                    │
│  $ perf sched latency                                               │
│                                                                     │
│  # 기아 현상 확인                                                     │
│  $ perf stat -e sched:sched_stat_sleep ./program                    │
│                                                                     │
│  # 로드 밸런싱 문제                                                   │
│  $ trace-cmd record -e sched:sched_migrate_task ./program          │
│                                                                     │
│  【모니터링】                                                          │
│  ──────────────────                                                  │
│  # 실시간                                                            │
│  $ watch -n 1 'cat /proc/stat | head -5'                            │
│  $ watch -n 1 'cat /proc/loadavg'                                   │
│                                                                     │
│  # 이력 분석                                                         │
│  $ perf record -g ./program                                         │
│  $ perf report                                                      │
│                                                                     │
```
