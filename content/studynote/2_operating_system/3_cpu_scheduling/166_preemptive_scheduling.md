# 166. 선점형 스케줄링 (Preemptive Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실행 중인 프로세스를 강제로 중단하고 다른 프로세스로 교체
> 2. **가치**: 빠른 응답성, 시분할 시스템 필수
> 3. **융합**: 비선점형, 문맥 교환, 인터럽트와 연관

---

## Ⅰ. 개요

### 개념 정의

선점형 스케줄링(Preemptive Scheduling)은 **실행 중인 프로세스를 강제로 중단하고 다른 프로세스에게 CPU를 할당하는 방식**이다. 운영체제가 CPU 제어권을 회수할 수 있다.

### 💡 비유: 퀴즈 쇼의 시간 제한
선점형 스케줄링은 **퀴즈 쇼의 시간 제한**과 같다. 시간이 다 되면(타임 슬라이스 만료) 현재 참가자가 답을 말하던 중이어도 다음 참가자로 넘어간다.

### 선점형 스케줄링 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                선점형 스케줄링 구조                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【선점형 vs 비선점형】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  비선점형                       선점형                         │ │
│  │  ────────────                  ────────────                   │ │
│  │                                                             │ │
│  │  P1 ──────┐                    P1 ────┐                      │ │
│  │           │ 자발적               │    │ 선점                   │ │
│  │           │ 양보                 │    ▼                        │ │
│  │           ▼                    P2 ────┐                      │ │
│  │  P2 ──────┘                        │    │ 선점                   │ │
│  │                                   │    ▼                        │ │
│  │  • CPU 자발적 반납               P3 ─────┘                      │ │
│  │  • 블로킹 I/O 시만 교환                                         │ │
│  │  • 구현 간단                    • 타임 슬라이스 강제 교환         │ │
│  │  • 응답성 낮음                   • 높은 우선순위 즉시 실행         │ │
│  │  • 반환 시간 길다                • 응답성 높음                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【선점 발생 시점】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  상황                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  타임 슬라이스 만료       정기적 CPU 회수                        │ │
│  │  높은 우선순위 도착       즉시 선점                              │ │
│  │  인터럽트 발생            하드웨어 이벤트                        │ │
│  │  I/O 요청                 블로킹 호출                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대표적 선점형 알고리즘】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  알고리즘                  설명                                │ │
│  │  ────                        ────                                │ │
│  │  Round Robin             타임 슬라이스 기반                     │ │
│  │  SRTF                    최단 잔여 시간 우선                     │ │
│  │  Priority (선점형)       높은 우선순위 즉시 실행                  │ │
│  │  MLFQ                    다단계 피드백 큐                        │ │
│  │  Linux CFS               완전 공정 스케줄러                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 확인 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 선점형 스케줄링】                                               │
│  ──────────────────                                                  │
│  # 커널 선점 설정 확인                                                 │
│  $ cat /boot/config-$(uname -r) | grep PREEMPT                       │
│  CONFIG_PREEMPT=y           # 선점형 커널                             │
│  CONFIG_PREEMPT_VOLUNTARY=y # 자발적 선점                             │
│                                                                     │
│  # voluntary vs full preemption                                      │
│  # Voluntary: 커널 코드에서 자발적 선점점 확인                          │ │
│  # Full: 언제든 선점 가능                                            │
│                                                                     │
│  【문맥 교환 횟수 확인】                                                  │
│  ──────────────────                                                  │
│  $ cat /proc/stat | grep ctxt                                         │
│  ctxt 12345678                                                        │
│                                                                     │
│  $ vmstat 1                                                           │
│  # cs 컬럼: 초당 문맥 교환 횟수                                        │
│                                                                     │
│  【타임 슬라이스 확인】                                                   │
│  ──────────────────                                                  │
│  # CFS에서의 타임 슬라이스 (동적)                                       │
│  $ cat /proc/sys/kernel/sched_min_granularity_ns                      │
│  2250000    # 2.25ms                                                 │
│                                                                     │
│  $ cat /proc/sys/kernel/sched_wakeup_granularity_ns                   │
│  # 선점 결정을 위한 기준                                               │
│                                                                     │
│  【실시간 스케줄링 (완전 선점형)】                                         │
│  ──────────────────                                                  │
│  # SCHED_FIFO: 타임 슬라이스 없음, 높은 우선순위까지 실행                │
│  # SCHED_RR: 타임 슬라이스 있음                                        │
│  $ chrt -f 50 ./realtime_app                                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
