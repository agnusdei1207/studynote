# 182. 다단계 큐 (Multilevel Queue)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 우선순위별로 여러 큐를 운영
> 2. **가치**: 작업 유형별 최적화된 스케줄링
> 3. **융합**: 우선순위 스케줄링, 라운드 로빈, FCFS와 연관

---

## Ⅰ. 개요

### 개념 정의

다단계 큐(Multilevel Queue) 스케줄링은 **프로세스를 우선순위나 특성에 따라 여러 개의 큐로 분리하고, 각 큐마다 다른 스케줄링 알고리즘을 적용하는 방식**이다.

### 💡 비유: 병원 진료 과
다단계 큐는 **병원 진료과**와 같다. 응급실, 내과, 외과 등 각각 다른 대기줄이 있고, 각 과마다 진료 방식이 다르다.

### 다단계 큐 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                다단계 큐 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【큐 구성 예시】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  우선순위 0 (높음) ──► [실시간 큐] ──► SCHED_FIFO/RR           │ │
│  │                          │                                   │ │
│  │  우선순위 1         ──► [시스템 큐] ──► Priority              │ │
│  │                          │                                   │ │
│  │  우선순위 2         ──► [대화형 큐] ──► Round Robin           │ │
│  │                          │                                   │ │
│  │  우선순위 3         ──► [배치 큐]   ──► FCFS                  │ │
│  │                          │                                   │ │
│  │  우선순위 4 (낮음) ──► [백그라운드] ──► FCFS                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【큐간 스케줄링】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  고정 우선순위 선점:                                            │ │
│  │  • 높은 우선순위 큐가 비어야 낮은 큐 실행                         │ │
│  │  • 높은 큐에 프로세스 있으면 낮은 큐는 대기                       │ │
│  │                                                             │ │
│  │  타임 슬라이스 분배:                                             │ │
│  │  • 각 큐에 CPU 시간 비율 할당                                    │ │
│  │  • 예: 높은 50%, 중간 30%, 낮은 20%                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【각 큐별 알고리즘】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  큐 유형              적합한 알고리즘              이유          │ │
│  │  ────                        ────                        ────    │ │
│  │  실시간              SCHED_FIFO/RR           응답성, 데드라인     │ │
│  │  시스템              Priority                중요도               │ │
│  │  대화형              Round Robin             응답성               │ │
│  │  배치                FCFS                    처리량               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                다단계 큐 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장단점】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 유연한 스케줄링          • 큐간 이동 없음                     │ │
│  │  • 작업 유형별 최적화        • 기아 현상 가능                    │ │
│  │  • 구현 상대적 간단          • 분류 기준 결정 어려움              │ │
│  │  • 낮은 오버헤드            • 유연성 부족                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프로세스 분류 기준】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  기준                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  프로세스 유형          실시간, 대화형, 배치                    │ │
│  │  메모리 요구량          작음, 보통, 큼                          │ │
│  │  I/O 특성               I/O 바운드, CPU 바운드                  │ │
│  │  우선순위               숫자로 표현                              │ │
│  │  사용자                  관리자, 일반 사용자                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【큐간 시간 분배 예시】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시간 할당 방식:                                                 │ │
│  │                                                             │ │
│  │  Q0 (실시간):     40% CPU 시간                                  │ │
│  │  Q1 (대화형):     30% CPU 시간                                  │ │
│  │  Q2 (배치):       20% CPU 시간                                  │ │
│  │  Q3 (백그라운드): 10% CPU 시간                                  │ │
│  │                                                             │ │
│  │  → 높은 우선순위 큐가 시간 독점 방지                              │ │
│  │  → 낮은 큐도 실행 기회 보장                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【다단계 큐 구현 구조】                                                 │
│  ──────────────────                                                  │
│  #define NUM_QUEUES 5                                               │
│                                                                     │
│  typedef enum {                                                      │
│      RR,      // Round Robin                                        │
│      FCFS,    // First Come First Served                            │
│      PRIORITY // Priority                                           │
│  } SchedulingPolicy;                                                 │
│                                                                     │
│  typedef struct {                                                    │
│      Queue queue;                                                    │
│      int priority;           // 큐 우선순위                          │
│      SchedulingPolicy policy; // 큐 내부 스케줄링                     │
│      int time_slice;        // RR인 경우 타임 슬라이스                │
│  } MultilevelQueue;                                                  │
│                                                                     │
│  MultilevelQueue queues[NUM_QUEUES];                                 │
│                                                                     │
│  Process* schedule() {                                               │
│      // 높은 우선순위 큐부터 검색                                      │
│      for (int i = 0; i < NUM_QUEUES; i++) {                          │
│          if (!queue_empty(&queues[i].queue)) {                       │
│              return dequeue_by_policy(&queues[i]);                   │
│          }                                                           │
│      }                                                               │
│      return NULL;  // 실행할 프로세스 없음                            │
│  }                                                                   │
│                                                                     │
│  【Linux 스케줄링 클래스】                                              │
│  ──────────────────                                                  │
│  # 스케줄링 클래스 (우선순위 순)                                        │
│  stop_sched_class     # 최고 우선순위                                 │
│  dl_sched_class       # 데드라인 스케줄링                             │
│  rt_sched_class       # 실시간 (SCHED_FIFO, SCHED_RR)               │
│  fair_sched_class     # CFS (SCHED_NORMAL, SCHED_BATCH)             │
│  idle_sched_class     # 최저 우선순위                                 │
│                                                                     │
│  # 각 클래스는 독립적인 큐와 알고리즘                                   │
│                                                                     │
│  【실시간 vs 일반 프로세스】                                             │
│  ──────────────────                                                  │
│  # 실시간 프로세스 확인                                                 │
│  $ ps -eo pid,rtprio,ni,comm                                        │
│                                                                     │
│  # RT 태스크 CPU 시간 제한                                             │
│  $ cat /proc/sys/kernel/sched_rt_runtime_us                         │
│  $ cat /proc/sys/kernel/sched_rt_period_us                          │
│                                                                     │
│  【프로세스 타입별 스케줄링 설정】                                        │
│  ──────────────────                                                  │
│  // 실시간 태스크                                                      │
│  struct sched_param param = { .sched_priority = 50 };               │
│  sched_setscheduler(pid, SCHED_FIFO, &param);                        │
│                                                                     │
│  // 배치 태스크                                                        │
│  sched_setscheduler(pid, SCHED_BATCH, &param);                       │
│                                                                     │
│  // 아이들 태스크                                                       │
│  sched_setscheduler(pid, SCHED_IDLE, &param);                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 우선순위별 여러 큐 운영
• 구성: 각 큐마다 다른 알고리즘
• 장점: 작업 유형별 최적화
• 단점: 큐간 이동 없음, 기아 가능
• 스케줄링: 고정 우선순위 또는 시간 분배
• 예시: 실시간(RR), 대화형(RR), 배치(FCFS)
```
