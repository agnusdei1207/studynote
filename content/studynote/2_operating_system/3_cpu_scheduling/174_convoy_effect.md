# 174. 호위 효과 (Convoy Effect)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 짧은 프로세스들이 긴 프로세스 뒤에서 대기
> 2. **가치**: FCFS의 주요 문제점
> 3. **융합**: FCFS, 비선점형, 평균 대기 시간과 연관

---

## Ⅰ. 개요

### 개념 정의

호위 효과(Convoy Effect)는 **긴 CPU 버스트를 가진 프로세스가 먼저 실행되면, 짧은 프로세스들이 긴 대기 시간을 갖는 현상**이다. FCFS 스케줄링의 주요 문제점이다.

### 💡 비유: 버스 대기줄
호위 효과는 **버스 대기줄에서 큰 짐을 가진 사람**과 같다. 큰 짐을 싣는 데 오래 걸리면 뒤의 가벼운 짐을 가진 사람들이 모두 기다려야 한다.

### 호위 효과 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                호위 효과 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【호위 효과 발생】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시간 ───────────────────────────────────────────────────►   │ │
│  │                                                             │ │
│  │  긴 프로세스가 먼저 도착한 경우:                                │ │
│  │  ┌─────────────────────────────────────────────────────────┐│ │
│  │  │ P1 (24ms)              │ P2 (3ms) │ P3 (3ms) │ P4 (3ms)││ │
│  │  │├───────────────────────┤├────┤├────┤├────┤              ││ │
│  │  │0                      24    27    30    33              ││ │
│  │  │                                                         ││ │
│  │  │ P2, P3, P4는 P1을 "호위"하듯 길게 대기                     ││ │
│  │  └─────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  │  대기 시간: P1=0, P2=24, P3=27, P4=30                         │ │
│  │  평균 대기 시간: (0+24+27+30)/4 = 20.25ms                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【호위 효과 없는 경우】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  짧은 프로세스가 먼저 도착한 경우:                              │ │
│  │  ┌─────────────────────────────────────────────────────────┐│ │
│  │  │ P2 (3ms) │ P3 (3ms) │ P4 (3ms) │ P1 (24ms)              ││ │
│  │  │├────┤├────┤├────┤├───────────────────────┤              ││ │
│  │  │0     3     6      9                       33             ││ │
│  │  │                                                         ││ │
│  │  └─────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  │  대기 시간: P2=0, P3=3, P4=6, P1=9                            │ │
│  │  평균 대기 시간: (0+3+6+9)/4 = 4.5ms                          │ │
│  │                                                             │ │
│  │  → 20.25ms vs 4.5ms: 호위 효과로 4.5배 차이!                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【호위 효과 이름 유래】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  짧은 프로세스들이 긴 프로세스 뒤에 줄지어 있는 모양이            │ │
│  │  마치 VIP의 호위 인원들이 따라다니는 모습과 같다고 하여           │ │
│  │  "Convoy Effect"라 부름                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 시뮬레이션
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【호위 효과 시뮬레이션 코드】                                            │
│  ──────────────────                                                  │
│  #include <stdio.h>                                                  │
│  #include <stdlib.h>                                                 │
│                                                                     │
│  typedef struct {                                                    │
│      int pid, burst, arrival, wait, turnaround;                      │
│  } Process;                                                          │
│                                                                     │
│  // 호위 효과 시뮬레이션                                                │
│  void simulate_convoy() {                                            │
│      // Case 1: 긴 프로세스 먼저 (호위 효과)                            │
│      Process p1[] = {                                                │
│          {1, 24, 0, 0, 0},  // 긴 프로세스                             │
│          {2, 3, 0, 0, 0},                                           │
│          {3, 3, 0, 0, 0},                                           │
│          {4, 3, 0, 0, 0}                                            │
│      };                                                              │
│                                                                     │
│      int time = 0;                                                   │
│      float total_wait = 0;                                           │
│      for (int i = 0; i < 4; i++) {                                   │
│          p1[i].wait = time;                                          │
│          time += p1[i].burst;                                        │
│          p1[i].turnaround = time;                                    │
│          total_wait += p1[i].wait;                                   │
│      }                                                               │
│      printf("호위 효과 있음: 평균 대기 = %.2f\n", total_wait/4);       │
│                                                                     │
│      // Case 2: 짧은 프로세스 먼저 (SJF)                               │
│      Process p2[] = {                                                │
│          {2, 3, 0, 0, 0},                                           │
│          {3, 3, 0, 0, 0},                                           │
│          {4, 3, 0, 0, 0},                                           │
│          {1, 24, 0, 0, 0}                                           │
│      };                                                              │
│                                                                     │
│      time = 0; total_wait = 0;                                       │
│      for (int i = 0; i < 4; i++) {                                   │
│          p2[i].wait = time;                                          │
│          time += p2[i].burst;                                        │
│          total_wait += p2[i].wait;                                   │
│      }                                                               │
│      printf("호위 효과 없음: 평균 대기 = %.2f\n", total_wait/4);       │
│  }                                                                   │
│                                                                     │
│  【실제 시스템에서의 관찰】                                               │
│  ──────────────────                                                  │
│  # 긴 작업과 짧은 작업 섞여 있는지 확인                                   │
│  $ ps -eo pid,etime,cmd --sort=etime                                │
│                                                                     │
│  # 프로세스별 CPU 시간 확인                                             │
│  $ top -o TIME+                                                     │
│                                                                     │
│  # I/O 대기가 많은지 확인 (호위 효과 원인)                               │
│  $ iostat -x 1                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 짧은 프로세스가 긴 프로세스 뒤에서 대기
• 원인: FCFS + 긴 프로세스 먼저 도착
• 영향: 평균 대기/반환 시간 증가
• 해결: SJF, RR, 선점형 스케줄링
• 이름: VIP 호위하듯 줄지어 대기
• 비선점형: 주로 발생
```
