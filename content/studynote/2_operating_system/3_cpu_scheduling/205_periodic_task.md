# 205. 스케줄링 데드락 (Scheduling Deadlock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스케줄링 결정으로 인한 진행 불가 상태
> 2. **가치**: 실시간 시스템에서 특히 위험
> 3. **융합**: 우선순위 역전, 데드락, 기아와 연관

---

## Ⅰ. 개요

### 개념 정의

스케줄링 데드락(Scheduling Deadlock)은 **스케줄링 알고리즘의 결정으로 인해 프로세스들이 서로를 기다리며 진행하지 못하는 상태**다. 우선순위 역전, 자원 경쟁 등으로 발생한다.

### 💡 비유: 일방통행 교차로
스케줄링 데드락은 **네 방향 모두 진입한 일방통행 교차로**와 같다. 모두가 "내가 먼저 가겠다" 하며 서로를 막아 아무도 못 간다.

### 스케줄링 데드락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄링 데드락 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【우선순위 역전 (Priority Inversion)】                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  정상: 높은 우선순위가 낮은 것을 선점                              │ │
│  │  역전: 낮은 우선순위가 높은 것을 간접적으로 블록                    │ │
│  │                                                             │ │
│  │  시나리오:                                                        │ │
│  │  1. L (낮은 우선순위)이 락 획득                                    │ │
│  │  2. H (높은 우선순위)이 L을 선점                                   │ │
│  │  3. H이 락 필요 → 블록됨                                          │ │
│  │  4. M (중간 우선순위)이 L을 선점                                   │ │
│  │  5. L이 실행 못 함 → H도 실행 못 함                                │ │
│  │  → H > L인데 M 때문에 H가 L을 기다림                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【데드락 시나리오】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  P1 (높은 우선순위)  ──────►  자원 A 필요                         │ │
│  │        ▲                          │                              │ │
│  │        │                          ▼                              │ │
│  │  자원 B 보유                P2 선점                               │ │
│  │        │                          │                              │ │
│  │        │                          ▼                              │ │
│  │        └─────────────────  P2 (낮은 우선순위)                      │ │
│  │                                   │                              │ │
│  │                                   ▼                              │ │
│  │                              자원 B 필요                          │ │
│  │                                                             │ │
│  │  → P1은 B를, P2는 A를 기다림 → 데드락                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【발생 조건】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 상호 배제: 자원은 한 번에 하나만 사용                          │ │
│  │  2. 점유 대기: 자원을 가진 채 다른 자원 대기                        │ │
│  │  3. 비선점: 자원을 강제로 뺏을 수 없음                             │ │
│  │  4. 순환 대기: 대기 사이클 형성                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄링 데드락 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【해결 방법】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 우선순위 상속 (Priority Inheritance)                          │ │
│  │     • 락 보유 중인 낮은 우선순위가 높은 우선순위로 승격              │ │
│  │     • 락 해제 시 원래 우선순위로 복귀                              │ │
│  │     • RT Mutex에서 구현                                           │ │
│  │                                                             │ │
│  │  2. 우선순위 천장 (Priority Ceiling)                               │ │
│  │     • 각 락에 최대 우선순위 지정                                   │ │
│  │     • 락 획득 시 해당 우선순위로 승격                               │ │
│  │                                                             │ │
│  │  3. 데드락 회피                                                     │ │
│  │     • 자원 할당 순서 규칙                                          │ │
│  │     • 은행원 알고리즘                                               │ │
│  │                                                             │ │
│  │  4. 데드락 탐지 및 복구                                             │ │
│  │     • 주기적 데드락 검사                                           │ │
│  │     • 희생자 선택 및 종료                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【우선순위 상속 동작】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. L이 락 획득                                                   │ │
│  │  2. H이 락 대기                                                   │ │
│  │  3. L의 우선순위가 H로 승격                                        │ │
│  │  4. M이 L(H)를 선점 못 함                                          │ │
│  │  5. L이 빨리 완료 → 락 해제                                        │ │
│  │  6. L 우선순위 원래대로 복귀                                        │ │
│  │  7. H이 락 획득 → 실행                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 해결 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux RT Mutex】                                                      │
│  ──────────────────                                                  │
│  // 우선순위 상속 지원 mutex                                           │
│  #include <pthread.h>                                                │
│                                                                     │
│  pthread_mutexattr_t attr;                                           │
│  pthread_mutexattr_init(&attr);                                      │
│  pthread_mutexattr_setprotocol(&attr, PTHREAD_PRIO_INHERIT);         │
│                                                                     │
│  pthread_mutex_t mutex;                                              │
│  pthread_mutex_init(&mutex, &attr);                                  │
│                                                                     │
│  // 사용                                                              │
│  pthread_mutex_lock(&mutex);                                         │
│  // 임계 영역                                                          │
│  pthread_mutex_unlock(&mutex);                                       │
│                                                                     │
│  【우선순위 천장 설정】                                                  │
│  ──────────────────                                                  │
│  // 우선순위 천장 프로토콜                                             │
│  pthread_mutexattr_setprotocol(&attr, PTHREAD_PRIO_PROTECT);         │
│  pthread_mutexattr_setprioceiling(&attr, 80);  // 천장 우선순위       │
│                                                                     │
│  【데드락 탐지】                                                          │
│  ──────────────────                                                  │
│  # 데드락 발생 시 추적                                                  │
│  $ perf record -e lock:* ./program                                  │
│  $ perf report                                                      │
│                                                                     │
│  # ftrace로 락 대기 분석                                               │
│  $ trace-cmd record -e irq:irq_handler_entry \                       │
│      -e sched:sched_switch ./program                                │
│                                                                     │
│  【데드락 회피 코딩 가이드】                                              │
│  ──────────────────                                                  │
│  // 1. 항상 같은 순서로 락 획득                                        │
│  void safe_operation() {                                             │
│      pthread_mutex_lock(&lock_a);  // 항상 A 먼저                     │
│      pthread_mutex_lock(&lock_b);  // 그 다음 B                       │
│      // 작업                                                          │
│      pthread_mutex_unlock(&lock_b);                                  │
│      pthread_mutex_unlock(&lock_a);                                  │
│  }                                                                   │
│                                                                     │
│  // 2. 타임아웃 있는 락                                                │
│  struct timespec ts;                                                 │
│  clock_gettime(CLOCK_REALTIME, &ts);                                 │
│  ts.tv_sec += 1;  // 1초 타임아웃                                     │
│  if (pthread_mutex_timedlock(&mutex, &ts) != 0) {                    │
│      // 타임아웃 - 데드락 가능성                                        │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
