# 동적 우선순위 승급 (Priority Boost)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스케줄러가 특정 이벤트(I/O 완료, 잠금 해제 등) 발생 시 태스크의 정적 우선순위를 일시적으로 높여줌으로써 시스템 반응성을 개선하고 기아 상태(Starvation)를 방지하는 동적 조정 메커니즘이다.
> 2. **가치**: 대화형(Interactive) 태스크가 CPU를 빠르게 점유하게 하여 사용자의 체감 지연 시간을 단축하며, 우선순위 역전(Priority Inversion)과 같은 스케줄링 모순 상황을 완화하는 안정 장치 역할을 한다.
> 3. **융합**: 윈도우 커널의 스케줄링 전략, 리눅스의 휴리스틱 우선순위 계산, 그리고 실시간 시스템의 우선순위 상속(Priority Inheritance) 프로토콜의 기반 아이디어이다.

+++

## Ⅰ. 개요 (Context & Background)

- **개념**: 운영체제는 태스크가 생성될 때 부여받은 '기본 우선순위'만으로 실행 순서를 정하지 않는다. 시스템의 효율을 위해 특정 상황에서 우선순위를 '임시 혜택'으로 올려주는데, 이를 **우선순위 승급 (Priority Boost)**이라고 한다. 혜택을 받은 태스크는 CPU를 사용함에 따라 우선순위가 서서히 다시 깎여 내려온다(Decay).

- **💡 비유**: 우선순위 승급은 "놀이공원의 패스트패스 발권"과 같다. 줄을 서 있던 사람이 어떤 미션을 성공하거나(I/O 완료) 너무 오래 기다렸을 때(Hunger), 일시적으로 맨 앞줄로 갈 수 있는 권한을 주어 전체적인 고객 만족도를 높이는 전략이다.

- **📢 섹션 요약 비유**: 평범한 시민(일반 태스크)이라도 긴급한 상황(이벤트)이 발생하면 일시적으로 경찰차처럼 사이렌(승급)을 달고 달릴 수 있게 허용하는 유연한 교통 법규와 같습니다.

+++

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 윈도우(Windows)의 대표적인 승급 시나리오

윈도우 커널은 다음 5가지 경우에 스레드에게 보너스 점수를 준다.

```text
  [ Priority Level ]
   ^
   |    (3) 전경 윈도우 부스트 (Quantumn 증가)
   |          ____
   |         /    \
   |   (1)  /      \ (4) 기아 방지 부스트 (Level 15로 점프!)
   |  ____/        \____
   | /                  \
   |/  (2) I/O 완료 부스트 \
   +-------------------------------------> 시간 (Time)
     (키보드/마우스 완료 시 큰 폭 승급)
```

**[다이어그램 해설]** 
1. **I/O 완료**: 디스크 I/O가 끝나면 약 1단계, 키보드/마우스 입력이 끝나면 약 6단계 정도 우선순위를 올려준다. 그래야 사용자가 타이핑할 때 화면에 글자가 즉시 나타난다.
2. **이벤트/세마포어 대기 해제**: 공유 자원을 기다리다 풀려난 경우, 락 경합을 빨리 해소하기 위해 승급시킨다.
3. **전경(Foreground) 창**: 사용자가 활성화한 창의 스레드는 퀀텀을 늘려주어 끊김 없는 조작감을 준다.
4. **기아 방지 (Priority Scan)**: 4초 이상 CPU를 못 쓴 가여운 스레드가 발견되면, 스케줄러가 강제로 최고 가변 우선순위(15)로 퀀텀 2배를 주어 한 번 실행시킨다.

### 우선순위 감쇄 (Priority Decay)
부스트를 받은 스레드는 CPU를 한 번 사용할 때마다 우선순위가 1단계씩 떨어진다. 이는 부스트를 받은 스레드가 계속 CPU를 독점하는 것을 막고, 다시 원래의 공정한 상태로 되돌리기 위함이다.

- **📢 섹션 요약 비유**: 보너스 포인트(승급)를 받았더라도 물건을 살 때마다 포인트가 차감(감쇄)되어, 결국 다시 일반 회원으로 돌아오는 합리적인 포인트 제도와 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 우선순위 승급 (Boost) vs 우선순위 상속 (Inheritance)

| 비교 항목 | 우선순위 승급 (Boost) | 우선순위 상속 (Inheritance) |
|:---|:---|:---|
| **목적** | 시스템 반응성 및 기아 방지 | 우선순위 역전 문제 해결 |
| **발생 조건** | I/O 완료, 오래 대기 등 시스템 이벤트 | 고순위 태스크가 저순위 태스크의 자원을 기다릴 때 |
| **상승 폭** | 정해진 규칙(보너스 점수)에 따름 | 기다리는 태스크의 우선순위까지 끌어올림 |
| **적용 범위** | 일반 시분할 운영체제 (Windows 등) | 실시간 운영체제 (RTOS) 및 커널 내부 |

- **📢 섹션 요약 비유**: 승급은 '성실한 사람에게 주는 보너스'이고, 상속은 '대장의 길을 막지 않게 쫄병을 빨리 밀어주는 조치'입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 특정 앱의 미세한 버벅임 (Lag) 분석
- **상황**: 백그라운드에서 데이터를 수집하는 앱이 가끔 사용자 입력을 가로채며 윈도우 창 전환이 매끄럽지 않음.
- **판단**: 백그라운드 앱이 I/O 완료 부스트를 너무 자주 받아서 발생할 수 있다. 윈도우에서는 `SetThreadPriorityBoost` API를 통해 특정 스레드의 부스트 기능을 끌 수 있다. 반응성이 중요한 전경 앱에는 부스트를 켜두되, 백그라운드 일괄 처리 스레드는 부스트를 명시적으로 비활성화하여 시스템 전체의 반응성 균형을 맞춰야 한다.

- **📢 섹션 요약 비유**: 모든 손님에게 VIP 대접을 해주면 진짜 VIP가 불편해지니, 조용히 자기 일만 하는 손님(백그라운드)에게는 일반 대접만 하도록 조정하는 기술입니다.

+++

## Ⅴ. 기대효과 및 결론

- **기대효과**: 정적 스케줄링의 경직성을 극복하고, 실제 사용자의 행위(입력, 대기)와 시스템 자원 상태에 반응하는 지능적인 CPU 배분을 실현한다.
- **미래 전망**: AI 스케줄러가 도입되면, 과거의 수치 기반 부스트를 넘어 태스크의 패턴을 학습하여 다음 부스트 시점과 폭을 예측하는 '예측형 우선순위 제어' 기술로 발전할 것이다.

- **📢 섹션 요약 비유**: 정해진 규정(정적 순위)보다 상황의 긴급성(동적 승급)을 이해하는 유연함이 OS의 지능을 완성합니다.
