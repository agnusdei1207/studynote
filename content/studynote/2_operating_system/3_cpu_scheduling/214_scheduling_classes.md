# 214. 스케줄링 클래스 (Scheduling Classes)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스케줄러 동작을 정의하는 모듈화된 계층
> 2. **가치**: 다양한 스케줄링 정책의 통합 관리
> 3. **융합**: CFS, RT, Deadline, Idle 스케줄러와 연관

---

## Ⅰ. 개요

### 개념 정의

스케줄링 클래스(Scheduling Classes)는 **서로 다른 스케줄링 정책을 구현하는 계층적 모듈**이다. Linux 스케줄러는 여러 클래스를 우선순위 순으로 확인하여 실행할 태스크를 선택한다.

### 💡 비유: 부서별 담당
스케줄링 클래스는 **회사 부서별 담당자**와 같다. 긴급한 건은 비상팀, 일반은 기획팀 식으로 우선순위가 있다.

### 스케줄링 클래스 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄링 클래스 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【클래스 계층 (높은 우선순위 → 낮은 우선순위)】                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────┐                 │ │
│  │  │  stop_sched_class                       │ 최우선          │ │
│  │  │  • CPU 중지용 특수 태스크                 │                 │ │
│  │  │  • migrate thread, stop_machine          │                 │ │
│  │  └─────────────────────────────────────────┘                 │ │
│  │                        ↓                                    │ │
│  │  ┌─────────────────────────────────────────┐                 │ │
│  │  │  dl_sched_class (Deadline)              │                 │ │
│  │  │  • SCHED_DEADLINE                       │                 │ │
│  │  │  • 데드라인 기반 스케줄링                │                 │ │
│  │  │  • EDF (Earliest Deadline First)        │                 │ │
│  │  └─────────────────────────────────────────┘                 │ │
│  │                        ↓                                    │ │
│  │  ┌─────────────────────────────────────────┐                 │ │
│  │  │  rt_sched_class (Real-Time)             │                 │ │
│  │  │  • SCHED_FIFO, SCHED_RR                 │                 │ │
│  │  │  • 고정 우선순위                        │                 │ │
│  │  │  • 100개 우선순위 (0-99)                │                 │ │
│  │  └─────────────────────────────────────────┘                 │ │
│  │                        ↓                                    │ │
│  │  ┌─────────────────────────────────────────┐                 │ │
│  │  │  fair_sched_class (CFS)                 │ 일반 태스크     │ │
│  │  │  • SCHED_NORMAL, SCHED_BATCH            │                 │ │
│  │  │  • 완전 공정 스케줄러                    │                 │ │
│  │  │  • vruntime 기반                        │                 │ │
│  │  └─────────────────────────────────────────┘                 │ │
│  │                        ↓                                    │ │
│  │  ┌─────────────────────────────────────────┐ 최하순위        │ │
│  │  │  idle_sched_class                       │                 │ │
│  │  │  • SCHED_IDLE                           │                 │ │
│  │  │  • 유휴 태스크                           │                 │ │
│  │  └─────────────────────────────────────────┘                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【pick_next_task 흐름】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  for each class in [stop, dl, rt, fair, idle]:              │ │
│  │      task = class->pick_next_task(rq)                         │ │
│  │      if task:                                                  │ │
│  │          return task                                           │ │
│  │  return idle_task  # 아무것도 없으면 idle                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄링 클래스 상세                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【sched_class 구조체】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct sched_class {                                          │ │
│  │      const struct sched_class *next;  // 다음 클래스            │ │
│  │                                                             │ │
│  │      // 핵심 연산                                               │ │
│  │      void (*enqueue_task)(...);    // 태스크 추가              │ │
│  │      void (*dequeue_task)(...);    // 태스크 제거              │ │
│  │      void (*yield_task)(...);      // 양보                     │ │
│  │                                                             │ │
│  │      // 태스크 선택                                              │ │
│  │      struct task_struct *(*pick_next_task)(...);              │ │
│  │      void (*put_prev_task)(...);    // 이전 태스크 저장        │ │
│  │                                                             │ │
│  │      // 선점                                                    │ │
│  │      void (*check_preempt_curr)(...);                         │ │
│  │                                                             │ │
│  │      // 로드 밸런싱                                              │ │
│  │      int (*select_task_rq)(...);    // CPU 선택               │ │
│  │      void (*migrate_task_rq)(...);  // 마이그레이션            │ │
│  │  };                                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【클래스별 특징】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  클래스         정책          자료구조       선점               │ │
│  │  ────               ────            ────           ────         │ │
│  │  stop          STOP         N/A          절대 안 됨            │ │
│  │  deadline      DEADLINE     RB-tree      데드라인 기준         │ │
│  │  rt            FIFO/RR      비트맵+리스트  우선순위 기준        │ │
│  │  fair          NORMAL/BATCH RB-tree      vruntime 기준        │ │
│  │  idle          IDLE         N/A          절대 선점 안 함       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 확인 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【태스크 스케줄링 정책 확인】                                         │
│  ──────────────────                                                  │
│  # 프로세스 정책 확인                                                 │
│  $ chrt -p <pid>                                                    │
│  pid 1234's current scheduling policy: SCHED_OTHER                  │
│  pid 1234's current scheduling priority: 0                          │
│                                                                     │
│  # 모든 정책 목록                                                     │
│  $ chrt -m                                                          │
│  SCHED_OTHER min/max priority   : 0/0                               │
│  SCHED_FIFO min/max priority    : 1/99                              │
│  SCHED_RR min/max priority      : 1/99                              │
│  SCHED_BATCH min/max priority   : 0/0                               │
│  SCHED_IDLE min/max priority    : 0/0                               │
│  SCHED_DEADLINE min/max priority: 0/0                               │
│                                                                     │
│  【정책 변경】                                                         │
│  ──────────────────                                                  │
│  # 실시간 FIFO 정책으로 변경                                          │
│  $ chrt -f -p 50 <pid>                                              │
│                                                                     │
│  # 실시간 RR 정책으로 변경                                            │
│  $ chrt -r -p 50 <pid>                                              │
│                                                                     │
│  # DEADLINE 정책으로 실행                                             │
│  $ schedtool -E -t 10000000 -p 100000000 -d 50000000 ./program      │
│                                                                     │
│  【C 코드에서 설정】                                                   │
│  ──────────────────                                                  │
│  #include <sched.h>                                                  │
│  #include <sys/syscall.h>                                           │
│                                                                     │
│  // SCHED_FIFO 설정                                                  │
│  struct sched_param param;                                          │
│  param.sched_priority = 50;                                         │
│  sched_setscheduler(0, SCHED_FIFO, &param);                         │
│                                                                     │
│  // SCHED_DEADLINE 설정                                              │
│  struct sched_attr attr;                                            │
│  attr.size = sizeof(attr);                                          │
│  attr.sched_policy = SCHED_DEADLINE;                                │
│  attr.sched_runtime = 10000000;   // 10 ms                         │
│  attr.sched_deadline = 50000000;  // 50 ms                         │
│  attr.sched_period = 100000000;   // 100 ms                        │
│  syscall(__NR_sched_setattr, 0, &attr, 0);                          │
│                                                                     │
│  【클래스별 통계】                                                     │
│  ──────────────────                                                  │
│  # /proc/sched_debug                                                 │
│  $ cat /proc/sched_debug | grep -A 10 "rt_rq"                       │
│  $ cat /proc/sched_debug | grep -A 10 "cfs_rq"                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 스케줄링 정책을 모듈화한 계층
• 순서: stop > dl > rt > fair > idle
• stop: CPU 중지용 특수
• dl: Deadline 스케줄링
• rt: 실시간 FIFO/RR
• fair: CFS 일반 태스크
• idle: 유휴 태스크
```
