# 162. 중기 스케줄러 (Medium-term Scheduler)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리와 디스크 간 프로세스 교체 (스와핑)
> 2. **가치**: 메모리 관리, 다중프로그래밍 정도 제어
> 3. **융합**: 단기/장기 스케줄러, 스와핑과 연관

---

## Ⅰ. 개요

### 개념 정의

중기 스케줄러(Medium-term Scheduler)는 **메모리에서 디스크로 프로세스를 내보내고(중단), 다시 불러오는(재개) 스케줄러**다. 스와핑(swapping)을 담당한다.

### 💡 비유: 창고 이동
중기 스케줄러는 **비수기 물건을 창고로 옮기는 작업**과 같다. 매장(메모리)이 꽉 차면 자주 안 쓰는 물건을 창고(디스크)로 옮긴다.

### 중기 스케줄러 구조

```

│                중기 스케줄러 구조                                      │

│  【메모리와 스왑 공간】                                                  │

│  │                                                             │ │
│  │  메인 메모리                    스왑 공간 (디스크)              │ │
│  │  ┌───────────────────────────┐  ┌─────────────────────────┐ │ │
│  │  │ 실행 중인 프로세스들        │  │ 중단된 프로세스들        │ │ │
│  │  │ ┌─────┐┌─────┐┌─────┐    │  │ ┌─────┐┌─────┐        │ │ │
│  │  │ │ P1  ││ P2  ││ P3  │    │  │ │ P4  ││ P5  │        │ │ │
│  │  │ │Active││Active││Active│   │  │ │Suspend││Suspend│      │ │ │
│  │  │ └─────┘└─────┘└─────┘    │  │ └─────┘└─────┘        │ │ │
│  │  │                         │  │                         │ │ │
│  │  │ ←──── swap in ────────│  │                         │ │ │
│  │  │                        │  │                         │ │ │
│  │  │ ────── swap out ──────→│  │                         │ │ │
│  │  │                         │  │                         │ │ │
│  │  └───────────────────────────┘  └─────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【중단(Suspend) vs 재개(Resume)】                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  동작                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  Swap Out (중단)         메모리 부족 시 디스크로 이동             │ │
│  │  Swap In (재개)          메모리 확보 시 다시 메모리로             │ │
│  │  Swap                    메모리 ↔ 디스크                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프로세스 상태 변화】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌──────────┐              ┌──────────────┐                  │ │
│  │  │ Running  │              │ Suspended    │                  │ │
│  │  │  Ready   │── swap out ──►│ Ready        │                  │ │
│  │  │ Waiting  │              │ Waiting      │                  │ │

│  │                                                             │ │
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스왑 공간 관리】                                                      │
│  ──────────────────                                                  │
│  # 스왑 공간 생성 (파일)                                               │
│  $ dd if=/dev/zero of=/swapfile bs=1M count=2048                     │
│  $ chmod 600 /swapfile                                               │
│  $ mkswap /swapfile                                                  │
│  $ swapon /swapfile                                                  │
│                                                                     │
│  # 스왑 공간 제거                                                      │
│  $ swapoff /swapfile                                                 │
│  $ rm /swapfile                                                      │
│                                                                     │
│  【swappiness 조정】                                                    │
│  ──────────────────                                                  │
│  # 일시적 변경                                                        │
│  $ sysctl vm.swappiness=10                                           │
│                                                                     │
│  # 영구적 변경 (/etc/sysctl.conf)                                     │
│  vm.swappiness=10                                                    │
│                                                                     │
│  【메모리 압박 확인】                                                    │
│  ──────────────────                                                  │
│  $ cat /proc/meminfo | head -10                                      │
│  $ vmstat 1                                                          │
│  # si/so 컬럼: swap in/swap out                                      │
│                                                                     │
│  $ free -h                                                           │
│                total        used        free                         │
│  Swap:          2.0Gi          0B       2.0Gi                        │
│                                                                     │
│  【OOM (Out of Memory) 확인】                                           │
│  ──────────────────                                                  │
│  $ dmesg | grep -i "out of memory"                                   │
│  $ cat /proc/sys/vm/overcommit_memory                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 메모리 ↔ 디스크 스와핑
• 동작: Swap Out (중단), Swap In (재개)
• 조건: 메모리 부족, 다중프로그래밍 제어
• 빈도: 초~분 단위
• Linux: swappiness, /proc/swaps
• 장점: 메모리 확보, 단점: I/O 오버헤드
```
