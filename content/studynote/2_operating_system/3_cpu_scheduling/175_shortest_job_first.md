# 175. SJF (최단 작업 우선, Shortest Job First)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 버스트가 가장 짧은 프로세스 먼저 실행
> 2. **가치**: 평균 대기 시간 최소화 (이론적 최적)
> 3. **융합**: 비선점형, FCFS, SRTF와 연관

---

## Ⅰ. 개요

### 개념 정의

SJF(Shortest Job First)는 **CPU 버스트 시간이 가장 짧은 프로세스를 먼저 실행하는 스케줄링 알고리즘**이다. 비선점형 버전으로 평균 대기 시간을 최소화한다.

### 💡 비유: 편의점 계산대
SJF는 **편의점 계산대**와 같다. 물건이 적은 손님 먼저 처리하면 전체 대기 시간이 줄어든다.

### SJF 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                SJF 구조                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 방식】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  도착한 프로세스          버스트 시간 정렬          CPU         │ │
│  │                                                             │ │
│  │  P1 (24ms)              ┌─────┐                           │ │
│  │  P2 (3ms)    ────►      │ P2  │ 3ms                       │ │
│  │  P3 (3ms)               │ P3  │ 3ms    ──►  가장 짧은 것    │ │
│  │  P4 (6ms)               │ P4  │ 6ms        먼저 실행       │ │
│  │                         │ P1  │ 24ms                      │ │
│  │                         └─────┘                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【간트 차트 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  FCFS (호위 효과 발생):                                         │ │
│  │  P1 (24)            P2 (3)  P3 (3)  P4 (6)                     │ │
│  │  ├──────────────────┤├───┤├───┤├────┤                         │ │
│  │  0                 24       27      30     36                  │ │
│  │  평균 대기: (0+24+27+30)/4 = 20.25ms                           │ │
│  │                                                             │ │
│  │  SJF (최적):                                                   │ │
│  │  P2 (3)  P3 (3)  P4 (6)  P1 (24)                               │ │
│  │  ├───┤├───┤├────┤├──────────────────┤                         │ │
│  │  0     3     6      12             36                          │ │
│  │  평균 대기: (0+3+6+12)/4 = 5.25ms                               │ │
│  │                                                             │ │
│  │  → SJF가 20.25ms vs 5.25ms: 4배 개선!                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SJF 최적성 증명】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  짧은 작업을 먼저 처리하면:                                       │ │
│  │  • 짧은 작업의 대기 시간 ↓                                        │ │
│  │  • 긴 작업의 대기 시간 ↑                                          │ │
│  │  • 하지만 짧은 작업이 더 많으므로                                  │ │
│  │  • 전체 평균 대기 시간 ↓                                          │ │
│  │                                                               │ │
│  │  ∴ SJF는 평균 대기 시간 최소화 (이론적 최적)                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                SJF 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장단점】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 평균 대기 시간 최소       • 버스트 시간 미리 알 수 없음        │ │
│  │  • 처리량 향상              • 긴 프로세스 기아                  │ │
│  │  • 호위 효과 해결            • 버스트 시간 추정 필요             │ │
│  │  • 이론적 최적              • 추정 오류 시 성능 저하             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버스트 시간 추정】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제: 다음 CPU 버스트 시간을 미리 알 수 없음                     │ │
│  │                                                             │ │
│  │  해결: 지수 평균 (Exponential Averaging) 추정                   │ │
│  │                                                             │ │
│  │  τ(n+1) = α × t(n) + (1-α) × τ(n)                           │ │
│  │                                                             │ │
│  │  τ(n+1): 예측 다음 버스트                                        │ │
│  │  t(n): 실제 이전 버스트                                          │ │
│  │  τ(n): 예측 이전 버스트                                          │ │
│  │  α: 가중치 (보통 0.5)                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SJF vs SRTF】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              SJF                 SRTF                    │ │
│  │  ────                    ────                     ────          │ │
│  │  선점 여부          비선점형              선점형                  │ │
│  │  선택 기준          전체 버스트            잔여 버스트             │ │
│  │  문맥 교환          적음                  많음                    │ │
│  │  성능              좋음                  더 좋음                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SJF 구현】                                                           │
│  ──────────────────                                                  │
│  #include <stdio.h>                                                  │
│  #include <stdlib.h>                                                 │
│                                                                     │
│  typedef struct {                                                    │
│      int pid, burst, arrival, wait, turnaround;                      │
│  } Process;                                                          │
│                                                                     │
│  int compare_burst(const void *a, const void *b) {                   │
│      return ((Process*)a)->burst - ((Process*)b)->burst;             │
│  }                                                                   │
│                                                                     │
│  void SJF(Process p[], int n) {                                      │
│      // 버스트 시간순 정렬                                             │
│      qsort(p, n, sizeof(Process), compare_burst);                    │
│                                                                     │
│      int time = 0;                                                   │
│      float total_wait = 0;                                           │
│                                                                     │
│      for (int i = 0; i < n; i++) {                                   │
│          p[i].wait = time;                                           │
│          time += p[i].burst;                                         │
│          p[i].turnaround = time;                                     │
│          total_wait += p[i].wait;                                    │
│      }                                                               │
│      printf("SJF 평균 대기 시간: %.2f\n", total_wait/n);              │
│  }                                                                   │
│                                                                     │
│  【버스트 시간 추정 코드】                                               │
│  ──────────────────                                                  │
│  float estimate_next_burst(float actual, float predicted, float α) { │
│      return α * actual + (1 - α) * predicted;                        │
│  }                                                                   │
│                                                                     │
│  // 사용 예                                                           │
│  float tau = 10.0;  // 초기 예측                                      │
│  float alpha = 0.5;                                                  │
│  float actuals[] = {6, 4, 6, 4, 13, 13, 13};                         │
│                                                                     │
│  for (int i = 0; i < 7; i++) {                                       │
│      printf("예측: %.1f, 실제: %.0f\n", tau, actuals[i]);             │
│      tau = estimate_next_burst(actuals[i], tau, alpha);              │
│  }                                                                   │
│                                                                     │
│  【실제 시스템에서의 활용】                                               │
│  ──────────────────                                                  │
│  # Linux CFS는 SJF 개념을 변형하여 사용                               │
│  # 짧은 작업(적은 runtime)이 우선                                     │
│  $ cat /proc/sys/kernel/sched_min_granularity_ns                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

