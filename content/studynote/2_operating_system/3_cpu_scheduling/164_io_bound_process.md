# 164. I/O 바운드 프로세스 (I/O Bound Process)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 작업에 많은 시간을 보내는 프로세스
> 2. **가치**: CPU 활용도 관점에서 스케줄링 고려
> 3. **융합**: CPU 바운드, 스케줄링 알고리즘과 연관

---

## Ⅰ. 개요

### 개념 정의

I/O 바운드 프로세스(I/O Bound Process)는 **입출력 작업에 많은 시간을 보내는 프로세스**다. CPU를 짧게 사용하고 긴 I/O 대기를 반복한다.

### 💡 비유: 도서관에서 책 찾는 사람
I/O 바운드 프로세스는 **도서관에서 책을 찾는 사람**과 같다. 읽기(CPU)보다 책 찾으러 다니는 시간(I/O)이 더 많다.

### I/O 바운드 프로세스 구조

```

│                I/O 바운드 프로세스 구조                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CPU 바운드 vs I/O 바운드】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  CPU 바운드                     I/O 바운드                     │ │
│  │  ────────────                  ────────────                   │ │
│  │                                                             │ │
│  │  시간 ─────────────────────►   시간 ─────────────────────►    │ │
│  │                                                             │ │
│  │  ┌──────────────────────┐   ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐       │ │
│  │  │                      │   │CPU││I/O││CPU││I/O││CPU││I/O│       │ │
│  │  │   CPU 사용 (길게)     │   │짧││길││짧││길││짧││길│       │ │ │
│  │  │                      │   │  ││  ││  ││  ││  ││  │       │ │
│  │  └──────────────────────┘   └──┘└──┘└──┘└──┘└──┘└──┘       │ │
│  │                                                             │ │
│  │  CPU 버스트: 길다             CPU 버스트: 짧다                │ │
│  │  I/O 버스트: 짧다/없다         I/O 버스트: 길다                │ │
│  │  빈도: 낮다                   빈도: 높다                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【특징 비교】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특징                  CPU 바운드          I/O 바운드           │ │
│  │  ────                      ────                  ────           │ │
│  │  CPU 버스트          길다                 짧다                  │ │
│  │  I/O 버스트          짧다/없다             길다                  │ │
│  │  문맥 교환           적다                 많다                  │ │
│  │  우선순위            낮음 (보통)           높음 (빠른 응답)       │ │
│  │  예시               연산, 렌더링           DB, 웹서버, 텍스트편집 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대표적 예시】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 텍스트 편집기 (사용자 입력 대기)                                │ │
│  │  • 데이터베이스 서버 (디스크 I/O)                                 │ │
│  │  • 웹 서버 (네트워크 I/O)                                        │ │
│  │  • 파일 복사/이동                                                │ │
│  │  • 비디오 스트리밍                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                I/O 바운드 프로세스 상세                                 │\
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스케줄링에 미치는 영향】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • I/O 바운드는 자주 CPU를 양보                                   │ │
│  │  • CPU 바운드와 혼합 시 시스템 효율 증가                           │ │
│  │  • 높은 우선순위 부여 시 대화형 응답성 향상                         │ │
│  │  • MLFQ에서 상위 큐에 머무름                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CPU 버스트 분포】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  빈도                                                        │ │
│  │    ▲                                                         │ │
│  │    │    ■■■■■■■■■                                             │ │
│  │    │   ■■■■■■■■■■■■■■                                          │ │
│  │    │  ■■■■■■■■■■■■■■■■■■                                        │ │
│  │    │ ■■■■■■■■■■■■■■■■■■■■■■                                     │ │
│  │    └──────────────────────────────────────► CPU 버스트 길이    │ │
│  │         짧음(대부분)              길음(소수)                    │ │
│  │                                                             │ │
│  │  → 대부분의 프로세스는 I/O 바운드                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【혼합 시스템에서의 이점】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  I/O 바운드가 대기 중 → CPU 바운드가 CPU 활용                     │ │
│  │  I/O 완료 시 → I/O 바운드가 다시 실행                            │ │
│  │  결과: 높은 CPU 이용률 + 좋은 응답성                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 확인 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프로세스 유형 확인】                                                  │
│  ──────────────────                                                  │
│  # CPU 사용률과 I/O 대기 확인                                         │
│  $ top                                                               │
│  # wa (I/O wait) 컬럼 확인                                           │
│                                                                     │
│  $ iostat -x 1                                                       │
│  # %iowait, %util 확인                                               │
│                                                                     │
│  $ pidstat -d 1                                                      │
│  # 프로세스별 디스크 I/O                                               │
│                                                                     │
│  【프로세스별 CPU/I/O 분석】                                             │
│  ──────────────────                                                  │
│  $ pidstat -p <pid> 1                                                │
│  # %usr, %system (CPU)                                              │
│  # 해당 프로세스의 CPU vs I/O 패턴                                    │
│                                                                     │
│  # I/O 많이 하는 프로세스 찾기                                         │
│  $ iotop                                                             │
│                                                                     │
│  【시스템 전체 I/O vs CPU 비율】                                          │
│  ──────────────────                                                  │
│  $ vmstat 1                                                          │
│  # r (실행 대기), b (I/O 대기)                                        │
│  # us/sy (CPU), wa (I/O wait)                                       │
│                                                                     │
│  【스트레스 테스트】                                                      │
│  ──────────────────                                                  │
│  # I/O 부하 생성                                                      │
│  $ fio --name=test --filename=testfile --rw=randread \               │
│        --bs=4k --size=1G --numjobs=4                                 │
│                                                                     │
│  # CPU 부하 생성                                                      │
│  $ stress --cpu 4                                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: I/O 작업에 많은 시간을 보내는 프로세스
• 특징: 짧은 CPU 버스트, 긴 I/O 버스트
• 예시: 텍스트 편집기, DB, 웹 서버
• 스케줄링: 높은 우선순위, MLFQ 상위 큐
• 혼합: CPU 바운드와 섞으면 효율 증가
• 분포: 대부분의 프로세스는 I/O 바운드
```
