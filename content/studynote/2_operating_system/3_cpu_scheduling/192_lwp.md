# 192. SMP 스케줄링 (Symmetric Multiprocessing Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 CPU 코어에서의 프로세스 분배
> 2. **가치**: 로드 밸런싱, 캐시 지역성, 확장성
> 3. **융합**: CPU Affinity, 로드 밸런서, NUMA와 연관

---

## Ⅰ. 개요

### 개념 정의

SMP 스케줄링(Symmetric Multiprocessing Scheduling)은 **여러 CPU 코어가 동등한 지위로 메모리를 공유하며 프로세스를 실행하는 시스템에서의 스케줄링**이다. 각 CPU는 독립적으로 스케줄링을 수행한다.

### 💡 비유: 여러 계산대
SMP 스케줄링은 **여러 계산대가 있는 마트**와 같다. 각 계산대(CPU)가 독립적으로 손님(프로세스)을 처리하지만, 전체 대기줄 관리(로드 밸런싱)도 필요하다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                SMP 스케줄링 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CPU Affinity (선호도)】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • Soft Affinity: 같은 CPU에서 실행하려는 경향                    │ │
│  │  • Hard Affinity: 특정 CPU에서만 실행 강제                        │ │
│  │                                                             │ │
│  │  장점:                                                         │ │
│  │  • 캐시 적중률 향상                                              │ │
│  │  • TLB 유지                                                    │ │
│  │  • 문맥 교환 오버헤드 감소                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【로드 밸런싱】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시점:                                                          │ │
│  │  • CPU가 유휴 상태일 때 (idle balancing)                          │ │
│  │  • 정기적 간격으로 (periodic balancing)                           │ │
│  │  • 새 프로세스 생성 시 (fork balancing)                           │ │
│  │                                                             │ │
│  │  방식:                                                          │ │
│  │  • Push: 바쁜 CPU에서 유휴 CPU로 태스크 이동                       │ │
│  │  • Pull: 유휴 CPU가 바쁜 CPU에서 태스크 가져옴                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SMP vs NUMA】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              SMP                  NUMA                    │ │
│  │  ────                    ────                     ────           │ │
│  │  메모리 접근       균등                비균등 (지역 메모리 빠름)   │ │
│  │  스케줄링          단순                메모리 지역성 고려          │ │
│  │  규모              소규모              대규모 서버                │ │
│  │  복잡도            낮음                높음                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 설정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CPU 정보 확인】                                                       │
│  ──────────────────                                                  │
│  # CPU 코어 수                                                        │
│  $ nproc                                                             │
│  $ lscpu                                                             │
│                                                                     │
│  # NUMA 노드                                                          │
│  $ numactl --hardware                                                │
│                                                                     │
│  【CPU Affinity 설정】                                                  │
│  ──────────────────                                                  │
│  # taskset으로 실행                                                   │
│  $ taskset -c 0,1 ./program        # CPU 0, 1에서만 실행              │
│  $ taskset -c 2-3 ./program        # CPU 2, 3에서만 실행              │
│                                                                     │
│  # 실행 중인 프로세스 Affinity 변경                                     │
│  $ taskset -pc 0,1 <pid>                                             │
│                                                                     │
│  【프로그래밍에서의 설정】                                                │
│  ──────────────────                                                  │
│  #define _GNU_SOURCE                                                  │
│  #include <sched.h>                                                  │
│                                                                     │
│  void set_affinity(int cpu) {                                        │
│      cpu_set_t cpuset;                                               │
│      CPU_ZERO(&cpuset);                                              │
│      CPU_SET(cpu, &cpuset);                                          │
│      sched_setaffinity(0, sizeof(cpuset), &cpuset);                  │
│  }                                                                   │
│                                                                     │
│  【CPU별 부하 확인】                                                     │
│  ──────────────────                                                  │
│  $ mpstat -P ALL 1                                                   │
│  $ top -1                                                            │
│  $ htop                                                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
