# 210. O(1) 스케줄러 (O(1) Scheduler)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 상수 시간에 스케줄링 결정을 내리는 알고리즘
> 2. **가치**: 확장성, 일정한 성능 보장
> 3. **융합**: CFS, 우선순위 큐, Linux 역사와 연관

---

## Ⅰ. 개요

### 개념 정의

O(1) 스케줄러는 **태스크 수와 관계없이 상수 시간 O(1)에 스케줄링 결정을 내리는 스케줄러**다. Linux 2.5~2.6 커널에서 사용되었으며, 현재는 CFS로 대체되었다.

### 💡 비유: 빠른 출구
O(1) 스케줄러는 **번호표 출구**와 같다. 줄이 길어도 내 번호만 보면 바로 나간다. 기다림이 줄 수와 무관하다.

### O(1) 스케줄러 구조

```

│                O(1) 스케줄러 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【우선순위 배열 구조】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │              Active 배열                                     │ │
│  │  ┌─────┬─────┬─────┬─────┬─────┐                            │ │
│  │  │  0  │  1  │  2  │ ... │139  │  우선순위 (140개)            │ │
│  │  └──┬──┴──┬──┴─────┴─────┴──┬──┘                            │ │
│  │     │     │                  │                                │ │
│  │   태스크  태스크              └── 태스크                       │ │
│  │   리스트  리스트                 리스트                        │ │
│  │                                                             │ │
│  │  bitmap: 1  0  1  0  0  1  0 ...                             │ │
│  │          │     │           │                                  │ │
│  │          ↓     ↓           ↓                                  │ │
│  │       실행   실행         실행                                 │ │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. Active 배열에서 태스크 선택                                │ │
│  │  2. 타임 슬라이스 소진 시 Expired 배열로 이동                   │ │
│  │  3. Active 배열 비면 배열 포인터 교체                          │ │
│  │                                                             │ │
│  │  ┌──────────┐         ┌──────────┐                          │ │
│  │  │  Active  │  swap   │ Expired  │                          │ │

│  【시간 복잡도】                                                       │

│  │  태스크 선택          O(1)  (bitmap 검색)                       │ │
```

## Ⅲ. 실무 적용

### 개념적 구현
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용 (개념적 이해)                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【O(1) 스케줄러 핵심 구조 (개념)】                                     │
│  ──────────────────                                                  │
│  #define PRIO_SIZE 140                                              │
│                                                                     │
│  struct prio_array {                                                │
│      int nr_active;           // 활성 태스크 수                       │
│      unsigned long bitmap[5]; // 우선순위 비트맵                      │
│      struct list_head queue[PRIO_SIZE]; // 우선순위별 리스트         │
│  };                                                                 │
│                                                                     │
│  struct runqueue {                                                  │
│      struct prio_array *active;   // 현재 실행 배열                  │
│      struct prio_array *expired;  // 만료된 태스크 배열               │
│      struct prio_array arrays[2]; // 실제 배열 저장소                │
│  };                                                                 │
│                                                                     │
│  // 태스크 선택 (O(1))                                               │
│  struct task_struct *pick_next_task(struct runqueue *rq) {          │
│      struct prio_array *array = rq->active;                         │
│                                                                     │
│      // 비트맵에서 가장 높은 우선순위 찾기                             │
│      int idx = sched_find_first_bit(array->bitmap);                 │
│                                                                     │
│      // 해당 우선순위 리스트에서 첫 태스크 반환                        │
│      struct list_head *head = &array->queue[idx];                   │
│      return list_entry(head->next, struct task_struct, run_list);   │
│  }                                                                  │
│                                                                     │
│  【현재 Linux 커널에서 확인】                                          │
│  ──────────────────                                                  │
│  # CFS가 O(1) 스케줄러를 대체                                         │
│  $ grep -r "O(1)" /usr/src/linux/Documentation/scheduler/           │
│                                                                     │
│  # 스케줄러 역사                                                      │
│  $ cat /usr/src/linux/Documentation/scheduler/sched-design-CFS.txt  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 상수 시간 스케줄링 알고리즘
• 구조: 140개 우선순위 배열 + bitmap
• 동작: Active ↔ Expired 배열 교체
• 시간복잡도: 모든 연산이 O(1)
• 역사: Linux 2.5~2.6 커널
• 현재: CFS로 대체됨
```
