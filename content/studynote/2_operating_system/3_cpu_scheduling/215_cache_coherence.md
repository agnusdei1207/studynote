# 215. 에너지 인식 스케줄링 (Energy-Aware Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전력 소모를 고려한 CPU 스케줄링
> 2. **가치**: 배터리 수명 연장, 전력 효율 최적화
> 3. **융합**: big.LITTLE, DVFS, EAS와 연관

---

## Ⅰ. 개요

### 개념 정의

에너지 인식 스케줄링(Energy-Aware Scheduling, EAS)은 **전력 소모를 최소화하면서 성능을 유지하는 스케줄링 기법**이다. 모바일 기기와 데이터 센터에서 중요하다.

### 💡 비유: 연비 운전
에너지 인식 스케줄링은 **연비 좋은 운전**과 같다. 급가속/급감속 피하고, 경제 속도로 달린다.

### 에너지 인식 스케줄링 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                에너지 인식 스케줄링 구조                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【big.LITTLE 아키텍처】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │      ┌─────────────────┐    ┌─────────────────┐             │ │
│  │      │   big 클러스터   │    │  LITTLE 클러스터 │             │ │
│  │      │                 │    │                 │             │ │
│  │      │  ┌───┐ ┌───┐   │    │  ┌───┐ ┌───┐ ┌───┐ ┌───┐    │ │
│  │      │  │C0 │ │C1 │   │    │  │C4 │ │C5 │ │C6 │ │C7 │    │ │
│  │      │  │성능│ │성능│   │    │  │절전│ │절전│ │절전│ │절전│    │ │
│  │      │  └───┘ └───┘   │    │  └───┘ └───┘ └───┘ └───┘    │ │
│  │      │   고성능        │    │    저전력                      │ │
│  │      │   고전력        │    │    저성능                      │ │
│  │      └─────────────────┘    └─────────────────┘             │ │
│  │                                                             │ │
│  │  EAS 결정:                                                     │ │
│  │  • 가벼운 태스크 → LITTLE 코어                                  │ │
│  │  • 무거운 태스크 → big 코어                                     │ │
│  │  • 전력 효율 계산                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Energy Model (EM)】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 각 CPU의 성능 상태별 전력 비용                                │ │
│  │  • idle 상태별 전력 비용                                        │ │
│  │  • 클러스터 공유 비용                                            │ │
│  │                                                             │ │
│  │  CPU    Frequency    Power (mW)                              │ │
│  │  ────               ────                ────                   │ │
│  │  big    2.0 GHz     1500                                  │ │
│  │  big    1.5 GHz     800                                   │ │
│  │  LITTLE 1.5 GHz     400                                   │ │
│  │  LITTLE 1.0 GHz     200                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【EAS 알고리즘】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 태스크의 CPU 사용량 추정                                     │ │
│  │  2. 각 후보 CPU의 에너지 비용 계산                               │ │
│  │  3. 최소 에너지 CPU 선택                                        │ │
│  │  4. 성능 저하 없이 전력 절약                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                에너지 인식 스케줄링 상세                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Schedutil Governor】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 스케줄러와 통합된 주파수 조절                                 │ │
│  │  • CPU 사용량에 따른 DVFS                                        │ │
│  │  • 빠른 응답 시간                                                │ │
│  │  • EAS와 함께 사용                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Energy 계산】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  energy = busy_energy + idle_energy                            │ │
│  │                                                             │ │
│  │  busy_energy = util * power_at_freq                          │ │
│  │  idle_energy = (1 - util) * idle_power                       │ │
│  │                                                             │ │
│  │  최적 CPU = argmin(energy)                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Overutilization】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 시스템가 과부하 시 EAS 비활성화                               │ │
│  │  • 성능 우선 모드로 전환                                        │ │
│  │  • 임계값: cpu_capacity * 80%                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 설정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【EAS 활성화 확인】                                                  │
│  ──────────────────                                                  │
│  # Energy Model 존재 확인                                            │
│  $ ls /sys/devices/system/cpu/cpu0/cpufreq/energy_performance_*    │
│                                                                     │
│  # schedutil governor 확인                                           │
│  $ cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_driver        │
│  schedutil                                                         │
│                                                                     │
│  # EAS 상태                                                          │
│  $ cat /proc/sys/kernel/sched_energy_aware                         │
│  1                                                                  │
│                                                                     │
│  【EAS 토글】                                                          │
│  ──────────────────                                                  │
│  # EAS 비활성화                                                       │
│  $ echo 0 > /proc/sys/kernel/sched_energy_aware                    │
│                                                                     │
│  # EAS 활성화                                                        │
│  $ echo 1 > /proc/sys/kernel/sched_energy_aware                    │
│                                                                     │
│  【CPU 주파수 설정】                                                  │
│  ──────────────────                                                  │
│  # schedutil governor 사용                                           │
│  $ echo schedutil > /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor
│                                                                     │
│  # powersave (최대 절전)                                             │
│  $ echo powersave > /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor
│                                                                     │
│  # performance (최대 성능)                                           │
│  $ echo performance > /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor
│                                                                     │
│  【전력 측정】                                                         │
│  ──────────────────                                                  │
│  # Intel RAPL                                                       │
│  $ cat /sys/class/powercap/intel-rapl/intel-rapl:0/energy_uj       │
│                                                                     │
│  # ARM (있는 경우)                                                   │
│  $ cat /sys/class/hwmon/hwmon0/power1_input                        │
│                                                                     │
│  # 배터리 (노트북)                                                   │
│  $ cat /sys/class/power_supply/BAT0/power_now                      │
│                                                                     │
│  【에너지 프로파일링】                                                 │
│  ──────────────────                                                  │
│  # perf로 에너지 이벤트 추적                                          │
│  $ perf stat -e power/energy-cores/ ./program                      │
│                                                                     │
│  # turbostat로 전력 모니터링                                          │
│  $ turbostat --show PkgWatt --interval 1                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
