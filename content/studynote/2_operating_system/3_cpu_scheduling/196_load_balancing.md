# 196. 실시간 스케줄링 개요

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 엄격한 시간 제약(데드라인)을 보장하는 스케줄링
> 2. **가치**: 산업 제어, 미사일 유도, 의료 기기에 필수
> 3. **융합**: 경성/연성 실시간, 데드라인, 우선순위와 연관

---

## Ⅰ. 개요

### 개념 정의

실시간 스케줄링(Real-time Scheduling)은 **작업이 반드시 지정된 시간 내에 완료되어야 하는 시스템을 위한 스케줄링**이다. 데드라인 준수가 최우선이다.

### 💡 비유: 심장 박동기
실시간 스케줄링은 **심장 박동기**와 같다. 정해진 간격으로 반드시 전기 자극을 줘야 한다. 1초만 늦어도 생명 위협이다.

### 실시간 스케줄링 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                실시간 스케줄링 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【실시간 시스템 분류】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  경성 실시간 (Hard Real-time):                                   │ │
│  │  • 데드라인 위반 = 치명적 실패                                    │ │
│  │  • 예: 항공기 제어, 원자로 제어, 제동 시스템                        │ │
│  │  • 반드시 보장 필요                                              │ │
│  │                                                             │ │
│  │  연성 실시간 (Soft Real-time):                                   │ │
│  │  • 데드라인 위반 = 성능 저하, 치명적 아님                           │ │
│  │  • 예: 비디오 스트리밍, 오디오 재생                                │ │
│  │  • 최선 노력 (best effort)                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실시간 태스크 특성】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  릴리스 시간 (r)         태스크가 실행 가능해지는 시간              │ │
│  │  실행 시간 (e)           최악의 경우 실행 시간 (WCET)              │ │
│  │  데드라인 (d)            반드시 완료해야 하는 시간                  │ │
│  │  주기 (p)               주기적 태스크의 반복 간격                   │ │
│  │  우선순위               태스크의 중요도                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【타임 라인】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │   릴리스        실행           데드라인                         │ │
│  │     r    ──────► e ──────►     d                              │ │
│  │     │                         │                               │ │
│  │     └─────────────────────────┘                               │ │
│  │              여유 시간 (slack)                                  │ │
│  │                                                             │ │
│  │  slack = d - r - e (여유 시간이 음수면 데드라인 위반)              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                실시간 스케줄링 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【실시간 스케줄링 알고리즘】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정적 우선순위:                                                   │ │
│  │  • Rate Monotonic (RM): 주기 짧은 것 = 높은 우선순위               │ │
│  │  • Deadline Monotonic (DM): 데드라인 짧은 것 = 높은 우선순위        │ │
│  │                                                             │ │
│  │  동적 우선순위:                                                   │ │
│  │  • EDF (Earliest Deadline First): 가장 가까운 데드라인 우선        │ │
│  │  • LST (Least Slack Time): 여유 시간 적은 것 우선                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스케줄 가능성 분석】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU 이용률 테스트 (n개의 주기적 태스크):                           │ │
│  │                                                             │ │
│  │  Rate Monotonic:                                                │ │
│  │  U = Σ(ei/pi) ≤ n(2^(1/n) - 1)                                  │ │
│  │  n=1: 100%, n=2: 82.8%, n=∞: 69.3%                             │ │
│  │                                                             │ │
│  │  EDF:                                                           │ │
│  │  U = Σ(ei/pi) ≤ 1 (100%까지 스케줄 가능)                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【일반 OS vs 실시간 OS】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              일반 OS             실시간 OS (RTOS)          │ │
│  │  ────                    ────                     ────           │ │
│  │  목표              처리량/공정성        데드라인 보장              │ │
│  │  선점              선택적               완전 선점형               │ │
│  │  지연              변동 큼               보장됨                    │ │
│  │  메모리            가상 메모리           정적 할당 선호             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### Linux 실시간 설정
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 실시간 스케줄링】                                               │
│  ──────────────────                                                  │
│  # 실시간 스케줄링 정책                                                 │
│  SCHED_FIFO  # 선입선출, 타임 슬라이스 없음                              │
│  SCHED_RR    # 라운드 로빈, 타임 슬라이스 있음                            │
│  SCHED_DEADLINE  # 데드라인 스케줄링 (최신)                             │
│                                                                     │
│  # 실시간 우선순위 범위                                                 │
│  # 1-99 (높을수록 높은 우선순위)                                        │
│  # 일반 태스크는 0                                                     │
│                                                                     │
│  【실시간 태스크 실행】                                                  │
│  ──────────────────                                                  │
│  # chrt으로 실행                                                      │
│  $ chrt -f 50 ./realtime_task       # SCHED_FIFO, 우선순위 50        │
│  $ chrt -r 30 ./realtime_task       # SCHED_RR, 우선순위 30          │
│                                                                     │
│  # 실행 중인 태스크 확인                                                │
│  $ chrt -p <pid>                                                     │
│                                                                     │
│  【프로그래밍에서의 설정】                                                │
│  ──────────────────                                                  │
│  #include <sched.h>                                                  │
│                                                                     │
│  void set_realtime_priority(int policy, int priority) {              │
│      struct sched_param param;                                       │
│      param.sched_priority = priority;                                │
│      sched_setscheduler(0, policy, &param);                          │
│  }                                                                   │
│                                                                     │
│  // SCHED_FIFO로 설정                                                  │
│  set_realtime_priority(SCHED_FIFO, 50);                              │
│                                                                     │
│  // SCHED_DEADLINE 설정                                               │
│  struct sched_attr attr;                                             │
│  attr.size = sizeof(attr);                                           │
│  attr.sched_policy = SCHED_DEADLINE;                                 │
│  attr.sched_runtime = 10000000;   // 10ms 실행 시간                   │
│  attr.sched_deadline = 30000000;  // 30ms 데드라인                    │
│  attr.sched_period = 30000000;    // 30ms 주기                        │
│  sched_setattr(0, &attr, 0);                                          │
│                                                                     │
│  【PREEMPT_RT 커널】                                                    │
│  ──────────────────                                                  │
│  # 실시간 커널 확인                                                     │
│  $ cat /boot/config-$(uname -r) | grep PREEMPT                       │
│  CONFIG_PREEMPT_RT=y                                                 │
│                                                                     │
│  # 실시간 지연 측정                                                     │
│  $ cyclictest -t1 -p 80 -n -i 10000 -l 10000                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 데드라인 보장 스케줄링
• 분류: 경성(치명적), 연성(성능저하)
• 특성: 릴리스 시간, 실행 시간, 데드라인
• 알고리즘: RM, DM, EDF, LST
• CPU 이용률: RM ≤ 69%, EDF ≤ 100%
• Linux: SCHED_FIFO, SCHED_RR, SCHED_DEADLINE
```
