# 190. 로드 평균 (Load Averages)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 부하를 나타내는 평균 실행 큐 길이
> 2. **가치**: 시스템 성능 모니터링의 핵심 지표
> 3. **융합**: CPU 코어 수, 스케줄링, 성능 분석과 연관

---

## Ⅰ. 개요

### 개념 정의

로드 평균(Load Averages)은 **지정된 시간 동안 실행 가능한 상태(Runnable)와 실행 불가능한 상태(Uninterruptible)에 있는 프로세스의 평균 개수**다. 보통 1분, 5분, 15분 간격으로 측정한다.

### 💡 비유: 은행 대기줄
로드 평균은 **은행 대기줄의 평균 길이**와 같다. 1분 평균 3명, 5분 평균 2명, 15분 평균 1.5명이 대기 중이라면 현재 상태와 추세를 알 수 있다.

### 로드 평균 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                로드 평균 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【로드 평균 출력 예시】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  $ uptime                                                        │
│  │  10:30:00 up 10 days,  3:45,  5 users,  load average: 2.50, 2.10, 1.80│
│  │                                                             │ │
│  │                     1분 평균  5분 평균  15분 평균                  │ │
│  │                        │        │         │                     │ │
│  │                        ▼        ▼         ▼                     │ │
│  │                       2.50     2.10      1.80                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【로드 평균의 의미】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  싱글 코어 시스템:                                               │ │
│  │  • 1.0 = CPU 100% 활용                                          │ │
│  │  • < 1.0 = CPU 유휴 시간 있음                                     │ │
│  │  • > 1.0 = 프로세스가 CPU 대기 중 (대기열 발생)                    │ │
│  │                                                             │ │
│  │  멀티 코어 시스템 (4코어):                                         │ │
│  │  • 4.0 = 모든 코어 100% 활용                                      │ │
│  │  • < 4.0 = 여유 있음                                             │ │
│  │  • > 4.0 = 대기 발생                                             │ │
│  │                                                             │ │
│  │  일반적 기준: 로드 평균 < CPU 코어 수 = 양호                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【로드 평균 구성 요소】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  로드 = 실행 중인 프로세스 + 실행 불가능한 프로세스                 │ │
│  │                                                             │ │
│  │  실행 중 (R): CPU에서 실제 실행 중                               │ │
│  │  실행 가능 (R): 실행 큐에서 대기 중                               │ │
│  │  실행 불가능 (D): I/O 대기 등 (uninterruptible sleep)            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                로드 평균 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【계산 방식】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  지수 평균 (Exponential Decay)                                   │ │
│  │                                                             │ │
│  │  load(t) = load(t-1) × e^(-5/60) + n × (1 - e^(-5/60))          │ │
│  │                                                             │ │
│  │  n: 현재 실행 가능/불가능 프로세스 수                             │ │
│  │  5/60: 5초 간격 / 60초 (1분 로드)                                │ │
│  │                                                             │ │
│  │  → 최근 값에 더 큰 가중치                                         │ │
│  │  → 과거 값은 서서히 감소                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해석 가이드】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  로드/코어              상태                    조치              │ │
│  │  ────                        ────                        ────      │ │
│  │  < 0.7               여유 있음              모니터링만           │ │
│  │  0.7 - 1.0           적절                   유지                 │ │
│  │  1.0 - 2.0           부하 있음              주의                 │ │
│  │  > 2.0               과부하                 조사/대응             │ │
│  │                                                               │ │
│  │  추세 분석:                                                       │ │
│  │  • 1분 > 5분 > 15분: 부하 증가 중                                 │ │
│  │  • 1분 < 5분 < 15분: 부하 감소 중                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【높은 로드의 원인】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. CPU 부족: 많은 CPU 바운드 프로세스                             │ │
│  │  2. I/O 대기: 디스크/네트워크 병목                                 │ │
│  │  3. 메모리 부족: 스와핑 발생                                       │ │
│  │  4. 잦은 문맥 교환: 너무 많은 프로세스                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 모니터링 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【로드 평균 확인】                                                      │
│  ──────────────────                                                  │
│  # uptime 명령어                                                      │
│  $ uptime                                                            │
│  10:30:00 up 10 days, load average: 0.50, 0.60, 0.55                │
│                                                                     │
│  # /proc/loadavg                                                     │
│  $ cat /proc/loadavg                                                 │
│  0.50 0.60 0.55 2/128 12345                                          │
│  # 1분 5분 15분 실행/전체 마지막 PID                                   │
│                                                                     │
│  # top 명령어                                                         │
│  $ top                                                               │
│  # 첫 줄에 load average 표시                                          │
│                                                                     │
│  # w 명령어                                                           │
│  $ w                                                                 │
│                                                                     │
│  【CPU 코어 수 확인】                                                    │
│  ──────────────────                                                  │
│  # CPU 코어 수                                                        │
│  $ nproc                                                             │
│  4                                                                   │
│                                                                     │
│  $ grep -c ^processor /proc/cpuinfo                                  │
│  4                                                                   │
│                                                                     │
│  $ lscpu | grep "^CPU(s):"                                           │
│  CPU(s):                4                                            │
│                                                                     │
│  【로드 평균 모니터링 스크립트】                                          │
│  ──────────────────                                                  │
│  #!/bin/bash                                                         │
│  CORES=$(nproc)                                                      │
│  while true; do                                                      │
│      LOAD=$(cat /proc/loadavg | awk '{print $1}')                    │
│      RATIO=$(echo "scale=2; $LOAD / $CORES" | bc)                    │
│      echo "$(date): Load $LOAD, Ratio $RATIO per core"               │
│      if (( $(echo "$RATIO > 1.5" | bc -l) )); then                   │
│          echo "WARNING: High load detected!"                         │
│      fi                                                              │
│      sleep 5                                                         │
│  done                                                                │
│                                                                     │
│  【sar로 분석】                                                          │
│  ──────────────────                                                  │
│  # 시스템 활동 리포트                                                   │
│  $ sar -q 1 5                                                        │
│  # runq-sz: 실행 큐 크기                                               │
│  # plist-sz: 전체 프로세스/스레드 수                                    │
│  # ldavg-1/5/15: 로드 평균                                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 실행 가능/불가능 프로세스의 평균 수
• 구간: 1분, 5분, 15분
• 기준: 로드 < 코어 수 = 양호
• 계산: 지수 평균 (최근 가중)
• 구성: 실행 중 + 실행 큐 + I/O 대기
• 활용: 시스템 부하 모니터링
```
