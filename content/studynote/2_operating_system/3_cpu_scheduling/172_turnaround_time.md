# 172. 처리량 (Throughput)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단위 시간당 완료된 프로세스 수
> 2. **가치**: 시스템 생산성의 핵심 지표
> 3. **융합**: CPU 이용률, 반환 시간, 스케줄링 알고리즘과 연관

---

## Ⅰ. 개요

### 개념 정의

처리량(Throughput)은 **단위 시간당 완료된 프로세스의 수**다. 시스템의 생산성을 나타내는 핵심 지표다.

### 💡 비유: 식당 주문 처리
처리량은 **식당에서 1시간당 처리한 주문 수**와 같다. 1시간에 100개 주문을 처리하면 처리량은 100주문/시간이다.

### 처리량 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                처리량 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【처리량 계산】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │           완료된 프로세스 수                                   │ │
│  │  처리량 = ─────────────────────                              │ │
│  │           단위 시간                                           │ │
│  │                                                             │ │
│  │  예: 10 프로세스 / 초                                        │ │
│  │      3600 프로세스 / 시간                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【처리량에 영향을 미치는 요소】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌──────────────────────────────────────────────────────┐   │ │
│  │  │                    높은 처리량                         │   │ │
│  │  │  ──────────────────────────────────────────────────    │   │ │
│  │  │  • 짧은 프로세스           • 낮은 문맥 교환            │   │ │
│  │  │  • 높은 CPU 이용률        • 효율적 스케줄링            │   │ │
│  │  │  • 충분한 메모리          • 적은 I/O 대기              │   │ │
│  │  └──────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  │  ┌──────────────────────────────────────────────────────┐   │ │
│  │  │                    낮은 처리량                         │   │ │
│  │  │  ──────────────────────────────────────────────────    │   │ │
│  │  │  • 긴 프로세스            • 잦은 문맥 교환             │   │ │
│  │  │  • 낮은 CPU 이용률        • 비효율적 스케줄링          │   │ │
│  │  │  • 메모리 부족            • 많은 I/O 대기              │   │ │
│  │  └──────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【알고리즘별 처리량 특성】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  알고리즘             처리량 특성                              │ │
│  │  ────                        ────                                │ │
│  │  FCFS               긴 프로세스에 의해 감소                      │ │
│  │  SJF                짧은 프로세스 우선 → 높음                    │ │
│  │  Round Robin        공정하지만 문맥 교환으로 감소 가능           │ │
│  │  Priority           높은 우선순위 위주 → 편향                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                처리량 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【처리량 vs 다른 기준】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 처리량 ↑ ←→ 반환 시간 ↓ (상충 가능)                         │ │
│  │  • 처리량 ↑ ←→ CPU 이용률 ↑ (비례)                             │ │
│  │  • 처리량 ↑ ←→ 응답 시간 ? (독립적)                            │ │
│  │                                                               │ │
│  │  짧은 프로세스 많이 완료 → 처리량 ↑, 반환 시간 ↓                │ │
│  │  긴 프로세스 적게 완료 → 처리량 ↓, 반환 시간 ↑                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최대 처리량 조건】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. CPU와 I/O 균형                                              │ │
│  │     - CPU 바운드와 I/O 바운드 혼합                               │ │
│  │                                                               │ │
│  │  2. 메모리 충분                                                 │ │
│  │     - 스와핑 최소화                                             │ │
│  │                                                               │ │
│  │  3. 문맥 교환 최적화                                             │ │
│  │     - 적절한 타임 퀀텀                                          │ │
│  │                                                               │ │
│  │  4. 병목 제거                                                   │ │
│  │     - I/O, 메모리, CPU 중 가장 느린 것                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시스템별 처리량 목표】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템              처리량 목표                                │ │
│  │  ────                        ────                                │ │
│  │  웹 서버             요청/초 (RPS)                              │ │
│  │  DB 서버            트랜잭션/초 (TPS)                           │ │
│  │  배치 시스템          작업/시간                                  │ │
│  │  메시지 큐           메시지/초                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 측정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 처리량 측정】                                                   │
│  ──────────────────                                                  │
│  # 프로세스 생성/종료 통계                                             │
│  $ vmstat 1                                                         │
│  # procs - r: 실행 대기, b: 블록됨                                    │
│                                                                     │
│  # 문맥 교환 횟수 (처리량 간접 지표)                                     │
│  $ cat /proc/stat | grep ctxt                                        │
│  ctxt 123456789                                                      │
│                                                                     │
│  # 초당 문맥 교환                                                      │
│  $ vmstat 1 | awk '{print $1}'                                       │
│  # cs 컬럼                                                           │
│                                                                     │
│  【웹 서버 처리량 측정】                                                  │
│  ──────────────────                                                  │
│  # Apache/Nginx 요청/초                                              │
│  $ ab -n 10000 -c 100 http://localhost/                             │
│  # Requests per second                                              │
│                                                                     │
│  # nginx 상태                                                        │
│  $ curl http://localhost/nginx_status                               │
│                                                                     │
│  【DB 처리량 측정】                                                      │
│  ──────────────────                                                  │
│  # MySQL                                                             │
│  mysql> SHOW STATUS LIKE 'Queries';                                 │
│  mysql> SHOW STATUS LIKE 'Questions';                               │
│                                                                     │
│  # PostgreSQL                                                        │
│  $ psql -c "SELECT xact_commit + xact_rollback FROM pg_stat_database"│
│                                                                     │
│  【처리량 계산 코드】                                                     │
│  ──────────────────                                                  │
│  #include <stdio.h>                                                  │
│  #include <time.h>                                                   │
│  #include <unistd.h>                                                 │
│                                                                     │
│  int main() {                                                        │
│      int completed = 0;                                              │
│      time_t start = time(NULL);                                      │
│      time_t duration = 60;  // 60초                                  │
│                                                                     │
│      while (time(NULL) - start < duration) {                         │
│          // 작업 수행...                                              │
│          completed++;                                                │
│      }                                                               │
│                                                                     │
│      double throughput = (double)completed / duration;               │
│      printf("Throughput: %.2f processes/sec\n", throughput);         │
│      return 0;                                                       │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
