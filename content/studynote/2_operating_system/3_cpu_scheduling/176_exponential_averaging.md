# [OS] 176. 지수 평균 (Exponential Averaging)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: **SJF (Shortest Job First)** 스케줄링의 단점을 극복하기 위해 과거 CPU **버스트 시간(Burst Time)**의 지수 가중 이동 평균을 활용하여 다음 수행 시간을 예측하는 확률적 기법.
> 2. **가치**: 별도의 큰 메모리 버퍼 없이 이전 예측값 하나만 유지하여 **O(1)**의 연산으로 최근 트렌드에 민감한 적응형 스케줄링을 가능하게 함.
> 3. **융합**: **OS (운영체제)**의 성능 최적화와 통계적 예측 알고리즘의 기초이며, **TCP (Transmission Control Protocol)**의 **RTT (Round-Trip Time)** 추정이나 **ML (Machine Learning)**의 SGD 최적화 등 광범위하게 응용됨.

+++

### Ⅰ. 개요 (Context & Background)

**1. 개념 정의**
**지수 평균법 (Exponential Averaging)** 또는 **지수 가중 이동 평균 (Exponentially Weighted Moving Average, EWMA)**은 시계열 데이터에서 과거 데이터에 지수적으로 감소하는 가중치를 부여하여 미래 값을 예측하는 통계적 기법입니다. 운영체제에서는 주로 프로세스의 다음 **CPU 버스트 시간 (CPU Burst Time)**을 예측하여 **SJF (Shortest Job First)** 스케줄링의 효율성을 높이는 데 사용됩니다.

**2. 작동 배경 및 철학**
기존의 단순 평균법(Simple Average)은 모든 과거 데이터를 동일한 비중으로 취급하여, 시스템의 부하 패턴이 변하는(예: I/O 중심에서 연산 중심으로 전환) 상황에 느리게 반응합니다. 반면, 지수 평균법은 **"최근의 관찰값은 미래를 예측하는 데 더 중요하다"**는 철학에 기초하여, 최근 데이터(`t_n`)에는 높은 가중치를, 먼 과거 데이터에는 낮은 가중치를 지수적으로 할당합니다.

**3. 비즈니스적 요구**
빅데이터 시대의 OS는 워크로드가 매우 가변적입니다. 사용자가 게임을 하다가 영상 편집을 전환하는 경우, OS는 즉시 CPU 연산 예측치를 조정해야 합니다. 이때 과거의 오래된 데이터에 묶여 있으면 스케줄링 병목이 발생하므로, 메모리 효율과 적응력을 모두 잡는 지수 평균법이 표준으로 자리 잡았습니다.

**4. 💡 비유**
날씨预报에서 "작년의 날씨 데이터"보다 "어제와 오늘의 날씨 변화"에 더 큰 가중치를 두어 내일의 날씨를 예측하는 것과 같습니다.

```text
[ Data Aging Concept ]
최신 데이터 (Hot) ----> 시간의 흐름 ----> 오래된 데이터 (Cold)
    ▲                                ▲
 High Weight (High Relevance)      Low Weight (Low Relevance)
   (α)                           ((1-α)^n)
```

> **📢 섹션 요약 비유**: 마치 자동차의 계기판이 과거 1년간의 평균 속도가 아니라, 지난 1분간의 가속 페달 변화를 즉시 반영하여 연비를 계산하는 것과 같습니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**1. 구성 요소 및 수식**
지수 평균법의 핵심은 재귀적 수식을 통해 무한한 과거의 역사를 하나의 변수로 압축하는 것입니다.

*   **τ (Tau)**: 예측된 CPU 버스트 시간 (Predicted Value)
*   **t**: 실제 측정된 CPU 버스트 시간 (Actual Measured Value)
*   **α (Alpha)**: 평활 계수 또는 감쇠 인자 (Smoothing Factor, 0 ≤ α ≤ 1)

**수식:**
```
τ_{n+1} = α · t_n + (1 - α) · τ_n
```

**2. ASCII 구조 다이어그램: 예측 루프**
아래 다이어그램은 시스템이 프로세스를 수행하고, 그 결과를 측정하여 다음 예측값을 갱신하는 **Feedback Loop**를 도식화한 것입니다.

```text
      +------------------+
      |   History (τ_n)  | <----[ Previous Prediction ]
      +--------+---------+
               |
               v
     +---------------------+
     |  Execute Process    |  (CPU Dispatch)
     +----------+----------+
                |
                v
     +---------------------+
     |   Measure Actual    |  (Hardware Timer Interrupt)
     |     Burst (t_n)     |
     +----------+----------+
                |
                v
      +-----------------------+
      |  Calculate New Pred.  |
      |  τ_{n+1} = αt_n +    |
      |  (1-α)τ_n             |
      +----------+------------+
                 |
                 v
      +------------------+
      | Update (τ_{n+1}) | ----> [ Used for Next SJF Decision ]
      +------------------+
```

**3. 다이어그램 해설**
1.  **Initialization (초기화)**: 시스템 부팅 시 첫 프로세스는 과거 데이터가 없으므로, 일반적으로 시스템 기본값 또는 첫 측정값(`t_0`)을 `τ_1`으로 설정합니다.
2.  **Measurement (측정)**: 프로세스가 CPU를 사용하고 **Interrupt (인터럽트)**가 발생하면, **TSC (Time Stamp Counter)** 등 하드웨어 타이머를 통해 실제 사용 시간인 `t_n`을 정확히 측정합니다.
3.  **Calculation (갱신)**: 측정된 `t_n`과 기존 예측값 `τ_n`을 가중 평균하여 `τ_{n+1`을 산출합니다. 이때 `α`의 설정에 따라 시스템의 반응 속도가 결정됩니다.
4.  **Storage (저장)**: 새로운 예측값 `τ_{n+1}`은 **PCB (Process Control Block)**의 스케줄링 관련 필드에 덮어쓰기(Overwrite)됩니다. 모든 과거 배열을 저장할 필요가 없어 메모리 효율이 극대화됩니다.

**4. 가중치 계수(α)에 따른 시스템 거동 분석**

| α 값 (Alpha) | 특징 (Characteristics) | 시스템 반응성 (Responsiveness) | 변동성 (Volatility) | 적합 상황 |
|:---:|:---|:---|:---|:---|
| **α → 1** | 가장 최근 값 `t_n`을 거의 그대로 반영 | 매우 빠름 (High) | 매우 높음 (Unstable) | 워크로드가 급격하게 변하거나 실시간 반응이 중요한 시스템 |
| **α → 0** | 과거 이력 `τ_n`을 강하게 유지 | 매우 느림 (Low) | 매우 낮음 (Stable) | 워크로드가 일정하거나 노이즈가 많은 환경 |
| **α = 0.5** | 최근과 과거의 균형 | 중간 (Moderate) | 중간 | 일반적인 범용 **OS (Operating System)** |

> **📢 섹션 요약 비유**: α의 조절은 마치 고성능 스포츠카의 서스펜션을 세팅하는 것과 같습니다. α를 높이면(단단하게) 노면의 울퉁불퉁한 변화(최근 데이터)를 그대로 느껴 빠르게 반응하지만 승차감(안정성)은 거칠어지고, α를 낮추면(부드럽게) 충격을 흡수해 편안하지만, 급회전(급격한 워크로드 변화)에 뒤늦게 반응할 수 있습니다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**1. 기술적 비교 분석: 단순 평균 vs 지수 평균**
과거 데이터를 처리하는 방식과 메모리 사용량에서 결정적인 차이가 있습니다.

| 비교 항목 | 단순 산술 평균 (Simple Average) | 지수 평균 (Exponential Average) |
|:---|:---|:---|
| **수식 예시** | `τ_{n+1} = (1/n) * Σ t_i` | `τ_{n+1} = αt_n + (1-α)τ_n` |
| **과거 가중치** | 모든 `t_i`에 동일한 가중치 `1/n` 부여 | 최근 `t_n`은 `α`, `t_{n-1}`은 `α(1-α)`, ... 지수적 감소 |
| **계산 복잡도** | **O(N)** - 과거 데이터 모두 필요 | **O(1)** - 이전 예측값만 필요 |
| **메모리 사용** | 버스트 길이 N만큼 배열 저장 필요 | 단 하나의 변수(`τ_n`)만 저장 |
| **트렌드 추종** | 느림 (Legacy Data Drag) | 빠름 (Recent Trend Reflection) |

**2. 타 영역과의 융합 (Synergy)**

*   **Network (네트워크)**: **TCP (Transmission Control Protocol)** 프로토콜의 혼잡 제어 및 재전송 타이머 설정 시, 네트워크 상황에 따라 **RTT (Round-Trip Time)**가 변하므로 이를 예측하기 위해 지수 평균법이 사용됩니다. 이를 통해 패킷 손실을 최소화하고 전송 속도를 동적으로 조절합니다.
*   **AI / Machine Learning**: **경사 하강법(Gradient Descent)** 최적화 기법 중 하나인 **Momentum**이나 **Adam Optimizer**의 기반 개념으로 쓰입니다. 과거의 기울기(Gradient) 정보를 지수 평균하여 현재의 파라미터 업데이트에 반영함으로써, 국소 최적해(Local Optima)에 빠지는 것을 방지하고 학습 속도를 높입니다.

```text
[ Expanded History Weight Visualization ]

     t_n (Latest)      t_{n-1}       t_{n-2}       t_{n-3} ... (Past)
       |                |             |             |
       v                v             v             v
      [α]         + [(1-α)α]     + [(1-α)^2 α] + [(1-α)^3 α] ...
       |                |             |             |
    High Weight    Moderate       Low Weight    Very Low Weight

Total Sum of Weights ≈ 1.0 (Normalization)
```

**3. 다이어그램 해설**
위 다이어그램은 시간이 거슬러 올라갈수록 가중치가 `(1-α)`의 거듭제곱만큼 줄어듦을 보여줍니다. 예를 들어 `α=0.5`라면, 가중치는 `1/2`, `1/4`, `1/8`, `1/16`...으로 줄어듭니다. 이는 수학적으로 무한한 과거의 합을 유한한 값으로 수렴하게 만드는 **기하 급수(Geometric Series)**의 성질을 이용한 것으로, **"과거는 현재를 보는 창이지만, 안개가 자욱하게 끼어 있다"**는 수리적 모델링입니다.

> **📢 섹션 요약 비유**: 단순 평균이 "10년 전 친구들과의 추억을 모두 동등하게 중요하게 여겨 일기장에 빼곡히 적어두는 것"이라면, 지수 평균은 "오늘 있었던 일은 뇌 깊이 새기고, 어제의 일은 희미하게, 1년 전의 일은 아주 연한 흔적만 남기는 뇌의 기억 메커니즘(Human Memory Decay)"과 같습니다.

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**1. 실무 시나리오 및 의사결정**

*   **Case A: 대용량 트랜잭션 처리 서버 (OLTP)**
    *   **상황**: 짧은 쿼리가 수만 건 발생하는 금융 서버.
    *   **결정**: 안정적인 처리량(Throughput)이 중요하므로, 급격한 예측 변동을 피하기 위해 **α = 0.3 ~ 0.4** 수준으로 낮춰 설정하여 노이즈를 필터링함.
*   **Case B: 실시간 스트리밍 인코딩 서버**
    *   **상황**: 화면 전환이 잦고 연산량이 급변하는 멀티미디어 서버.
    *   **결정**: 프레임 드랍을 막기 위해 워크로드 변화를 즉각 반영해야 하므로 **α = 0.7 ~ 0.9** 수준으로 높여 **SJF**가 빠르게 CPU를 할당하도록 유도함.

**2. 도입 체크리스트**

| 구분 | 체크 항목 |
|:---|:---|
| **기술적** | 시스템이 **Preemptive (선점형)** 환경인가? (비선점형 SJF에서는 Convoy 현상 방지를 위해 더욱 정확한 예측 필요) |
| **운영적** | **α** 값 동적 조정 기능이 포함되었는가? (런타임에 튜닝 가능 여부 확인) |
| **보안적** | 악의적인 프로세스가 예측값을 조작하여 CPU 시간을 독점하려는 시도에 대한 방어 로직이 있는가? (우선순위 역전 공격 방지) |

**3. 안티패턴 (Anti-Pattern)**
*   **과도한 낮은 α 사용**: 워크로드가 급격히 커지는 스파크(Spark) 상황에서 `α`가 너무 낮으면, 시스템이 여전히 "짧은 작업"으로 착각하여 다른 무거운 프로세스들을 기다리게 만드는 **Priority Inversion (우선순위 역전)** 현상을 유발할 수 있습니다.

> **📢 섹션 요약 비유**: 지수 평균법을 사용하는 스케줄러는 마치 "실시간 교통 상황 카메라"를 보는 내비게이션과 같습니다. 만약 카메라 영상(α)을 무시하고 10년 전 지도(과거 데이터)만 보고 길을 안내한다면, 도로 공사로 막힌 곳을 피하지 못해 엄청난 정체(병목)를 초래할 것입니다.

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**1. 정량/정성 기대효과**

| 항목 | 도입 전 (No Prediction) | 도입 후 (Exponential Avg) |
|:---|:---|:---|
| **Average Waiting Time** | Random Scheduling 대비 긴 대기 시간 발생 | 약 20~40% 개선 (문헌에 따라 상이) |
| **System Throughput** | Long Jobs가 Short Jobs를 방해 | 단위 시간당 처리량(T
