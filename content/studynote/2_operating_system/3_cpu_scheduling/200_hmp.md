# 200. 여유 시간 (Laxity/Slack)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데드라인까지 남은 시간에서 실행 시간을 뺀 값
> 2. **가치**: 스케줄링 결정과 데드라인 위반 예측에 활용
> 3. **융합**: 데드라인, EDF, LST 스케줄링과 연관

---

## Ⅰ. 개요

### 개념 정의

여유 시간(Laxity 또는 Slack)은 **데드라인까지 남은 시간에서 필요한 실행 시간을 뺀 값**이다. 태스크가 얼마나 여유 있는지를 나타낸다.

### 💡 비유: 비행기 탑승 여유
여유 시간은 **비행기 탑승 전 남은 시간에서 보안 검사+이동 시간을 뺀 것**과 같다. 30분 남았고 검사에 20분 걸리면 여유는 10분이다.

### 여유 시간 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                여유 시간 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【여유 시간 계산】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Laxity = (데드라인 - 현재시간) - 남은실행시간                     │ │
│  │                                                             │ │
│  │  L(t) = (d - t) - c_remaining                                │ │
│  │                                                             │ │
│  │  d: 절대 데드라인                                                │ │
│  │  t: 현재 시간                                                   │ │
│  │  c_remaining: 남은 실행 시간                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【여유 시간의 의미】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Laxity > 0: 여유 있음 (아직 늦지 않음)                           │ │
│  │  Laxity = 0: 지금 시작해야 정확히 맞춤                             │ │
│  │  Laxity < 0: 이미 데드라인 위반                                   │ │
│  │                                                             │ │
│  │  ┌──────────────────────────────────────────────────────┐   │ │
│  │  │                                                        │   │ │
│  │  │  현재     실행 필요    데드라인                          │   │ │
│  │  │   t    ────────────►    d                              │   │ │
│  │  │   │         │            │                             │   │ │
│  │  │   └─────────┴────────────┘                             │   │ │
│  │  │     남은 시간    실행 시간     여유 시간                   │   │ │
│  │  │                                                        │   │ │
│  │  └──────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예시】                                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  태스크 A: d=100, 현재=50, 남은 실행=30                           │ │
│  │  Laxity = (100-50) - 30 = 20 (여유 있음)                          │ │
│  │                                                             │ │
│  │  태스크 B: d=80, 현재=50, 남은 실행=35                            │ │
│  │  Laxity = (80-50) - 35 = -5 (이미 위반)                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                여유 시간 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LST 스케줄링 (Least Slack Time)】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  원칙: 여유 시간이 가장 적은 태스크 먼저 실행                        │ │
│  │                                                             │ │
│  │  특징:                                                           │ │
│  │  • 동적 우선순위                                                  │ │
│  │  • EDF와 유사하지만 실행 시간 고려                                 │ │
│  │  • 과부하 시 EDF보다 예측 가능한 동작                               │ │
│  │                                                             │ │
│  │  LST vs EDF:                                                     │ │
│  │  • EDF: 데드라인만 고려                                           │ │
│  │  • LST: 데드라인 - 실행시간 고려                                   │ │
│  │  • 실행 시간이 긴 태스크를 EDF보다 우대                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【여유 시간 활용】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 스케줄링 결정                                                  │ │
│  │     - 여유 시간 작은 태스크 우선                                    │ │
│  │                                                             │ │
│  │  2. 데드라인 위반 예측                                             │ │
│  │     - 여유 시간 음수 → 이미 위반                                   │ │
│  │     - 여유 시간 0에 가까움 → 위험                                   │ │
│  │                                                             │ │
│  │  3. 부하 분석                                                      │ │
│  │     - 평균 여유 시간으로 시스템 여유도 파악                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【0 여유 시점 (Zero Laxity Point)】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  여유 시간이 0이 되는 시점 = 반드시 실행 시작해야 하는 시점           │ │
│  │                                                             │ │
│  │  이 시점 이후에 시작하면 데드라인 위반 확정                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 계산 코드
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【여유 시간 계산 코드】                                                  │
│  ──────────────────                                                  │
│  #include <time.h>                                                   │
│  #include <stdio.h>                                                  │
│                                                                     │
│  typedef struct {                                                    │
│      int id;                                                         │
│      struct timespec deadline;                                       │
│      int remaining_ms;                                               │
│  } Task;                                                             │
│                                                                     │
│  long calculate_laxity(Task *task) {                                 │
│      struct timespec now;                                            │
│      clock_gettime(CLOCK_MONOTONIC, &now);                           │
│                                                                     │
│      long time_to_deadline_ms =                                      │
│          (task->deadline.tv_sec - now.tv_sec) * 1000 +               │
│          (task->deadline.tv_nsec - now.tv_nsec) / 1000000;           │
│                                                                     │
│      return time_to_deadline_ms - task->remaining_ms;                │
│  }                                                                   │
│                                                                     │
│  int is_deadline_at_risk(Task *task) {                               │
│      long laxity = calculate_laxity(task);                           │
│      return laxity < 0;  // 음수면 위반                               │
│  }                                                                   │
│                                                                     │
│  Task* select_lst_task(Task tasks[], int n) {                        │
│      Task *best = NULL;                                              │
│      long min_laxity = LONG_MAX;                                     │
│                                                                     │
│      for (int i = 0; i < n; i++) {                                   │
│          if (tasks[i].remaining_ms > 0) {                            │
│              long laxity = calculate_laxity(&tasks[i]);              │
│              if (laxity < min_laxity) {                              │
│                  min_laxity = laxity;                                │
│                  best = &tasks[i];                                   │
│              }                                                       │
│          }                                                           │
│      }                                                               │
│      return best;                                                    │
│  }                                                                   │
│                                                                     │
│  【모니터링 함수】                                                        │
│  ──────────────────                                                  │
│  void monitor_laxity(Task tasks[], int n) {                          │
│      printf("=== Laxity Monitor ===\n");                             │
│      for (int i = 0; i < n; i++) {                                   │
│          long laxity = calculate_laxity(&tasks[i]);                  │
│          printf("Task %d: Laxity = %ld ms", tasks[i].id, laxity);    │
│          if (laxity < 0) printf(" [DEADLINE MISSED]");               │
│          else if (laxity < 10) printf(" [CRITICAL]");                │
│          printf("\n");                                               │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  【실제 시스템에서의 활용】                                              │
│  ──────────────────                                                  │
│  # 스케줄링 통계 확인                                                   │
│  $ cat /proc/<pid>/sched                                             │
│                                                                     │
│  # 실시간 지연 확인                                                     │
│  $ cyclictest -t1 -p 80 -n -i 1000 -l 10000                          │
│  # Max 값이 작을수록 여유 시간 충분                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 데드라인까지 남은 시간 - 실행 시간
• 공식: L = (d - t) - c_remaining
• 의미: >0 여유, =0 긴급, <0 위반
• 활용: LST 스케줄링, 위반 예측
• 비교: EDF는 데드라인만, LST는 여유 시간
• 0 여유 시점: 반드시 실행 시작해야 하는 시점
```
