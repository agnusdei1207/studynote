# 168. 디스패처 (Dispatcher)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 제어권을 다음 프로세스에게 넘겨주는 모듈
> 2. **가치**: 문맥 교환 실제 수행, 스케줄러와 협력
> 3. **융합**: 문맥 교환, 단기 스케줄러, PCB와 연관

---

## Ⅰ. 개요

### 개념 정의

디스패처(Dispatcher)는 **단기 스케줄러가 선택한 프로세스에게 CPU 제어권을 넘겨주는 모듈**이다. 문맥 교환(context switch)을 실제로 수행한다.

### 💡 비유: 교대 근무 인수인계
디스패처는 **교대 근무 인수인계 담당자**와 같다. 스케줄러가 "다음은 김씨"라고 정하면, 디스패처가 이씨에서 김씨로 인수인계(문맥 교환)를 진행한다.

### 디스패처 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                디스패처 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스케줄러 vs 디스패처】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  단기 스케줄러                    디스패처                      │ │
│  │  ────────────                    ────────────                  │ │
│  │                                                             │ │
│  │  "누구를 실행할까?"               "어떻게 전환할까?"             │ │
│  │                                                             │ │
│  │  • 알고리즘 실행                   • 문맥 교환 수행               │ │
│  │  • 준비 큐 검색                   • PCB 저장/복원               │ │
│  │  • 프로세스 선택                   • 사용자 모드 전환             │
│  │  • 결정만 함                       • 실제 전환 수행               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【디스패처 동작 과정】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  실행 중인 프로세스            디스패처               다음 프로세스  │ │
│  │  ┌─────────────────┐        ┌─────────────────┐              │ │
│  │  │ PCB             │        │                 │              │ │
│  │  │ - 레지스터 저장   │───────►│ 1. 문맥 저장     │              │ │
│  │  │ - 상태 업데이트   │        │ 2. PCB 업데이트   │              │ │
│  │  │ - 메모리 정보    │        │ 3. 메모리 관리    │              │ │
│  │  │- 스케줄링 정보   │        │ 4. 문맥 복원     │───────►     │ │
│  │  └─────────────────┘        │ 5. 사용자 모드    │              │ │
│  │                             │ 6. 점프          │              │ │
│  │                             └─────────────────┘              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【디스패치 지연 (Dispatch Latency)】                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정의: 한 프로세스 정지 → 다른 프로세스 시작까지의 시간             │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │ P1 실행 │ 디스패치 지연 │ P2 실행                        │ │ │
│  │  │         │╌╌╌╌╌╌╌╌╌╌╌│                                │ │ │
│  │  │         │ 문맥 교환    │                                │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  구성:                                                         │ │
│  │  1. 현재 프로세스 문맥 저장                                     │ │
│  │  2. 새 프로세스 문맥 로드                                       │ │
│  │  3. 다른 오버헤드 (캐시 플러시 등)                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                디스패처 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【디스패처 작업 상세】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 현재 프로세스 문맥 저장                                       │ │
│  │     - 레지스터 값을 PCB에 저장                                    │ │
│  │     - 프로그램 카운터(PC) 저장                                    │ │
│  │     - 스택 포인터(SP) 저장                                        │ │
│  │                                                               │ │
│  │  2. PCB 상태 업데이트                                             │ │
│  │     - 실행 → 준비/대기/종료                                       │ │
│  │                                                               │ │
│  │  3. 다른 프로세스 PCB 선택                                        │ │
│  │     - 스케줄러가 선택한 프로세스                                   │ │
│  │                                                               │ │
│  │  4. 선택된 프로세스 문맥 복원                                     │ │
│  │     - PCB에서 레지스터 복원                                       │ │
│  │                                                               │ │
│  │  5. 사용자 모드로 전환                                            │ │
│  │     - 커널 모드 → 사용자 모드                                      │ │
│  │                                                               │ │
│  │  6. 선택된 프로세스로 점프                                        │ │
│  │     - PC가 가리키는 주소로 이동                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【디스패치 지연에 영향을 미치는 요소】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  요소                      영향                                │ │
│  │  ────                        ────                                │ │
│  │  레지스터 수             많을수록 느림                          │ │
│  │  TLB 크기                크면 플러시 비용 큼                     │ │
│  │  캐시 상태               콜드 캐시면 느림                        │ │
│  │  메모리 계층             깊을수록 느림                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최적화 기법】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 레지스터 윈도우 (SPARC)                                        │ │
│  │  • TLB 항목 유지 (PCID, ASID)                                     │ │
│  │  • 캐시 친화적 스케줄링                                            │ │
│  │  • CPU Affinity                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 성능 측정
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【문맥 교환 비용 측정】                                                  │
│  ──────────────────                                                  │
│  // 문맥 교환 시간 측정 프로그램                                         │
│  #include <stdio.h>                                                 │
│  #include <unistd.h>                                                │
│  #include <sys/time.h>                                              │
│                                                                     │
│  int main() {                                                        │
│      int pipe1[2], pipe2[2];                                         │
│      pipe(pipe1);                                                    │
│      pipe(pipe2);                                                    │
│                                                                     │
│      if (fork() == 0) {                                              │
│          // 자식                                                     │
│          for (int i = 0; i < 1000; i++) {                            │
│              write(pipe1[1], "x", 1);                                │
│              read(pipe2[0], &c, 1);                                  │
│          }                                                           │
│      } else {                                                        │
│          // 부모                                                     │
│          struct timeval start, end;                                  │
│          gettimeofday(&start, NULL);                                 │
│          for (int i = 0; i < 1000; i++) {                            │
│              read(pipe1[0], &c, 1);                                  │
│              write(pipe2[1], "x", 1);                                │
│          }                                                           │
│          gettimeofday(&end, NULL);                                   │
│          // 문맥 교환당 시간 계산                                       │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  【시스템 성능 지표】                                                    │
│  ──────────────────                                                  │
│  # 문맥 교환 횟수                                                     │
│  $ cat /proc/stat | grep ctxt                                         │
│                                                                     │
│  # 초당 문맥 교환                                                      │
│  $ vmstat 1                                                           │
│  # cs 컬럼 확인                                                       │
│                                                                     │
│  # 디스패치 지연 확인                                                  │
│  $ perf stat -e sched:sched_switch ./program                          │
│                                                                     │
│  【Linux 디스패처 튜닝】                                                 │
│  ──────────────────                                                  │
│  # 최소 그램규래리티 (타임 슬라이스 관련)                                │
│  $ cat /proc/sys/kernel/sched_min_granularity_ns                      │
│                                                                     │
│  # wakeup 그램규래리티 (선점 결정)                                       │
│  $ cat /proc/sys/kernel/sched_wakeup_granularity_ns                   │
│                                                                     │
│  # migration cost (CPU 이동 비용)                                      │
│  $ cat /proc/sys/kernel/sched_migration_cost_ns                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
