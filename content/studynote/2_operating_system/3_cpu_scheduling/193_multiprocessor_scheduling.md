# 193. 로드 밸런싱 (Load Balancing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 간 부하를 균등하게 분배
> 2. **가치**: SMP 시스템의 효율적 자원 활용
> 3. **융합**: SMP 스케줄링, CPU Affinity, Migration과 연관

---

## Ⅰ. 개요

### 개념 정의

로드 밸런싱(Load Balancing)은 **SMP 시스템에서 여러 CPU 간의 작업 부하를 균등하게 분배하는 메커니즘**이다. 일부 CPU가 유휴 상태인 동안 다른 CPU가 과부하되는 것을 방지한다.

### 💡 비유: 식당 서빙 분배
로드 밸런싱은 **식당에서 서빙 직원 간의 테이블 분배**와 같다. 한 서빙 직원이 바쁘고 다른 직원은 놀고 있으면, 매니저가 테이블을 재분배한다.

### 로드 밸런싱 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                로드 밸런싱 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【밸런싱 전】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  CPU0 (과부하)              CPU1 (유휴)                        │ │
│  │  ┌─────────────────┐       ┌─────────────────┐               │ │
│  │  │ [P1][P2][P3][P4]│       │                 │               │ │
│  │  │ [P5][P6][P7][P8]│       │                 │               │ │
│  │  │     Load: 8     │       │     Load: 0     │               │ │
│  │  └─────────────────┘       └─────────────────┘               │ │
│  │                                                             │ │
│  │  → CPU0 과부하, CPU1 유휴 → 비효율                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【밸런싱 후】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  CPU0 (균형)                CPU1 (균형)                        │ │
│  │  ┌─────────────────┐       ┌─────────────────┐               │ │
│  │  │ [P1][P2][P3][P4]│       │ [P5][P6][P7][P8]│               │ │
│  │  │                 │       │                 │               │ │
│  │  │     Load: 4     │       │     Load: 4     │               │ │
│  │  └─────────────────┘       └─────────────────┘               │ │
│  │                                                             │ │
│  │  → 로드 밸런싱으로 균등 분배 → 효율                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【밸런싱 시점】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시점                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  Idle Balancing          CPU가 유휴 상태 진입 시                 │ │
│  │  Periodic Balancing       정기적 간격 (수 ms)                    │ │
│  │  New Task Balancing       새 태스크 생성 시                      │ │
│  │  Wakeup Balancing         태스크 깨어날 때                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                로드 밸런싱 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Push vs Pull 마이그레이션】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Push Migration:                                                 │ │
│  │  • 과부하 CPU가 다른 CPU로 태스크 전송                             │ │
│  │  • 주기적 로드 밸런싱에서 사용                                     │ │
│  │                                                             │ │
│  │  Pull Migration:                                                 │ │
│  │  • 유휴 CPU가 다른 CPU에서 태스크 가져옴                            │ │
│  │  • Idle Balancing에서 사용                                        │ │
│  │                                                             │ │
│  │  Steal:                                                          │ │
│  │  • 유휴 CPU가 바쁜 CPU의 런큐에서 태스크 훔침                        │ │
│  │  • CFS에서 사용                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【로드 계산】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU 로드 = Σ(태스크 가중치 × 태스크 로드)                        │ │
│  │                                                             │ │
│  │  태스크 로드:                                                     │ │
│  │  • 실행 가능한 태스크의 가중치 합                                  │ │
│  │  • 지수 평균으로 계산                                             │ │
│  │  • nice 값 반영                                                   │ │
│  │                                                             │ │
│  │  균형 조건: |CPU0_load - CPU1_load| < 임계값                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【마이그레이션 비용】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 캐시 무효화                                                   │ │
│  │  • TLB 플러시 가능                                                │ │
│  │  • 문맥 교환 오버헤드                                              │ │
│  │                                                             │ │
│  │  비용 > 이득이면 밸런싱 안 함                                       │ │
│  │  sched_migration_cost_ns로 설정                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 설정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【로드 밸런싱 설정 확인】                                               │
│  ──────────────────                                                  │
│  # 마이그레이션 비용                                                    │
│  $ cat /proc/sys/kernel/sched_migration_cost_ns                      │
│  500000    # 0.5ms                                                   │
│                                                                     │
│  # 한 번에 마이그레이션할 태스크 수                                       │
│  $ cat /proc/sys/kernel/sched_nr_migrate                             │
│  32                                                                  │
│                                                                     │
│  # 스케줄러 도메인 확인                                                 │
│  $ cat /proc/sys/kernel/sched_domain/cpu*/domain*/name               │
│                                                                     │
│  【CPU별 부하 확인】                                                     │
│  ──────────────────                                                  │
│  # CPU별 사용률                                                       │
│  $ mpstat -P ALL 1                                                   │
│                                                                     │
│  # CPU별 문맥 교환                                                     │
│  $ mpstat -I SUM -P ALL 1                                            │
│                                                                     │
│  # CPU별 런큐 길이                                                      │
│  $ cat /proc/sched_debug | grep -A 5 "cpu#"                          │
│                                                                     │
│  【로드 밸런싱 튜닝】                                                    │
│  ──────────────────                                                  │
│  # 마이그레이션 비용 증가 (캐시 지역성 중시)                              │
│  $ sysctl -w kernel.sched_migration_cost_ns=1000000                  │
│                                                                     │
│  # 마이그레이션 비용 감소 (균등 분배 중시)                                │
│  $ sysctl -w kernel.sched_migration_cost_ns=100000                   │
│                                                                     │
│  # NUMA 밸런싱 활성화/비활성화                                          │
│  $ sysctl -w kernel.numa_balancing=1                                │
│  $ sysctl -w kernel.numa_balancing=0                                │
│                                                                     │
│  【수동 로드 밸런싱】                                                    │
│  ──────────────────                                                  │
│  # 특정 CPU로 마이그레이션                                              │
│  $ taskset -pc 1 <pid>              # PID를 CPU 1로 이동              │
│                                                                     │
│  # IRQ 밸런싱                                                         │
│  $ cat /proc/irq/*/smp_affinity                                      │
│  $ echo 2 > /proc/irq/24/smp_affinity  # IRQ 24를 CPU 1로            │
│                                                                     │
│  【프로그래밍에서의 활용】                                                │
│  ──────────────────                                                  │
│  // 특정 CPU로 마이그레이션                                             │
│  void migrate_to_cpu(int cpu) {                                      │
│      cpu_set_t cpuset;                                               │
│      CPU_ZERO(&cpuset);                                              │
│      CPU_SET(cpu, &cpuset);                                          │
│      sched_setaffinity(0, sizeof(cpuset), &cpuset);                  │
│  }                                                                   │
│                                                                     │
│  // 현재 CPU 확인                                                      │
│  int get_current_cpu() {                                             │
│      return sched_getcpu();                                          │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
