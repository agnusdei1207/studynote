# 204. CPU 그룹 스케줄링 (CPU Cgroup Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 그룹별로 CPU 자원 분배
> 2. **가치**: 공정한 자원 할당, 격리, QoS 보장
> 3. **융합**: cgroup, CFS, 컨테이너와 연관

---

## Ⅰ. 개요

### 개념 정의

CPU 그룹 스케줄링(CPU Cgroup Scheduling)은 **프로세스를 그룹으로 묶어 그룹별로 CPU 자원을 할당하고 제한하는 메커니즘**이다. cgroup (control group)을 통해 구현된다.

### 💡 비유: 회사 부서 예산
CPU 그룹 스케줄링은 **회사 부서별 예산 배정**과 같다. 영업팀 30%, 개발팀 40%, 관리팀 30% 식으로 부서별로 예산(CPU)을 배정한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                CPU 그룹 스케줄링 상세                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【shares vs quota】                                                     ▓
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  cpu.shares (상대적):                                            │ │
│  │  • 비율로 CPU 분배                                                │ │
│  │  • A=1024, B=512 → A이 B의 2배 CPU                               │ │
│  │  • 유휴 CPU 있으면 더 사용 가능                                    │ │
│  │  • soft limit                                                    │ │
│  │                                                             │ │
│  │  cpu.cfs_quota_us (절대적):                                      │ │
│  │  • 정확한 CPU 시간 제한                                            │ │
│  │  • quota=50000, period=100000 → 최대 50% CPU                     │ │
│  │  • 초과 시 대기                                                   │ │
│  │  • hard limit                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【활용 사례】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 컨테이너 격리                                                  │ │
│  │     Docker, Kubernetes에서 컨테이너별 CPU 제한                     │ │
│  │                                                             │ │
│  │  2. 멀티테넌시                                                      │ │
│  │     사용자/테넌트별 공정한 자원 분배                                │ │
│  │                                                             │ │
│  │  3. QoS 보장                                                       │ │
│  │     중요 서비스에 최소 CPU 보장                                     │ │
│  │                                                             │ │
│  │  4. 배치 작업 제한                                                  │ │
│  │     백그라운드 작업의 CPU 사용 제한                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CPU 샤 계산 예시】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  그룹      shares     전체 shares    CPU 비율                    │ │
│  │  ────                ────                  ────                │ │
│  │  A         2048         4096          50% (2048/4096)           │ │
│  │  B         1024         4096          25% (1024/4096)           │ │
│  │  C         1024         4096          25% (1024/4096)           │ │
│  │                                                             │ │
│  │  단, 유휴 CPU 있으면 더 사용 가능                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 설정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【cgroup v2 설정】                                                      │
│  ──────────────────                                                  │
│  # cgroup 마운트 확인                                                   │
│  $ mount | grep cgroup                                               │
│                                                                     │
│  # 그룹 생성                                                           │
│  $ mkdir /sys/fs/cgroup/mygroup                                      │
│                                                                     │
│  # CPU shares 설정 (상대적)                                             │
│  $ echo 512 > /sys/fs/cgroup/mygroup/cpu.weight                      │
│                                                                     │
│  # CPU 제한 설정 (절대적)                                               │
│  $ echo 50000 > /sys/fs/cgroup/mygroup/cpu.max                       │
│  $ echo 100000 > /sys/fs/cgroup/mygroup/cpu.max                      │
│  # quota 50000, period 100000 = 50% CPU                              │
│                                                                     │
│  # 프로세스 추가                                                        │
│  $ echo <pid> > /sys/fs/cgroup/mygroup/cgroup.procs                  │
│                                                                     │
│  【Docker에서의 활용】                                                   │
│  ──────────────────                                                  │
│  # CPU shares 설정                                                     │
│  $ docker run --cpu-shares=512 nginx                                 │
│                                                                     │
│  # CPU 할당량 설정                                                      │
│  $ docker run --cpus=0.5 nginx         # 0.5 CPU (50%)               │
│  $ docker run --cpus=2 nginx           # 2 CPU                       │
│                                                                     │
│  # CPU 세트 지정                                                        │
│  $ docker run --cpuset-cpus=0,1 nginx # CPU 0, 1만 사용              │
│                                                                     │
│  【Kubernetes에서의 활용】                                               │
│  ──────────────────                                                  │
│  # Pod 스펙에서 CPU 제한                                                 │
│  resources:                                                          │
│    requests:                                                         │
│      cpu: "500m"        # 0.5 CPU 요청                               │
│    limits:                                                           │
│      cpu: "1000m"       # 1 CPU 제한                                 │
│                                                                     │
│  【상태 확인】                                                            │
│  ──────────────────                                                  │
│  # cgroup 통계                                                          │
│  $ cat /sys/fs/cgroup/mygroup/cpu.stat                               │
│                                                                     │
│  # 프로세스 목록                                                        │
│  $ cat /sys/fs/cgroup/mygroup/cgroup.procs                           │
│                                                                     │
│  # 실시간 모니터링                                                      │
│  $ systemd-cgtop                                                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
