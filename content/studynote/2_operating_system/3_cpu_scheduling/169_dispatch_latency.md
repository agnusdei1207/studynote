# 169. 디스패치 지연 (Dispatch Latency)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 프로세스 정지에서 다른 시작까지의 시간
> 2. **가치**: 스케줄링 효율성의 핵심 지표
> 3. **융합**: 문맥 교환, 디스패처, 실시간 시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

디스패치 지연(Dispatch Latency)은 **한 프로세스의 실행을 중단하고 다른 프로세스의 실행을 시작하기까지 걸리는 시간**이다. 문맥 교환 비용의 핵심 구성 요소다.

### 💡 비유: 레이서 교체 시간
디스패치 지연은 **릴레이 경주의 바통 터치 시간**과 같다. 선수 교체(프로세스 전환)에 걸리는 시간이 짧을수록 좋다.

### 디스패치 지연 구조

```
│  【시간 흐름】                                                                             │
│  ──────────────────                                                  │

│  │                                                             │ │
│  │  P1 실행         디스패치 지연          P2 실행                 │ │
│  │  ┌────────┐     ┌─────────────────┐    ┌────────────────┐   │ │
│  │  │        │     │                 │    │                │   │ │
│  │  │ P1 작업 │     │ 1. 문맥 저장    │    │                │   │ │
│  │  │        │     │ 2. PCB 업데이트  │    │ P2 작업         │   │ │
│  │  │        │     │ 3. 문맥 복원    │    │                │   │ │
│  │  │        │     │ 4. 모드 전환    │    │                │   │ │
│  │  └────────┘     └─────────────────┘    └────────────────┘   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【구성 요소】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단계                      소요 시간 (일반적)                    │ │
│  │  ────                        ────                               │ │
│  │  문맥 저장               ~0.5-2 µs                              │ │
│  │  PCB 업데이트               ~0.1-0.5 µs                          │ │
│  │  문맥 복원               ~0.5-2 µs                              │ │
│  │  모드 전환               ~0.1-0.5 µs                            │ │
│  │  캐시/TLB 영향              ~0-10 µs (상황에 따라)                │ │
│  │  ─────────────────────────────────────                           │ │
│  │  전체 디스패치 지연          ~1-15 µs                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【영향을 미치는 요소】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  요소                      영향                                │ │
│  │  ────                        ────                                │ │
│  │  CPU 아키텍처            레지스터 수, 파이프라인                  │ │
│  │  메모리 계층              캐시 크기, TLB                         │ │
│  │  OS 구현                  스케줄러 복잡도                         │ │
│  │  시스템 부하              문맥 교환 빈도                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  │  • 일반적으로 10-100 µs 이하 필요                                  │ │
│  │  • 선점형 커널로 지연 최소화                                       │ │
│  │  • 실시간 OS (RTOS): 1-10 µs                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지연 구간 상세】                                                                             │

│  │  1. 인터럽트 핸들링                                               │ │

│  │                                                               │ │

│  │     - 다음 프로세스 선택                                          │ │

│  │  3. 문맥 교환                                                     │ │
│  │     - 레지스터, 메모리 상태 저장/복원                              │ │
│  │                                                               │ │
│  │  4. 캐시/TLB 복구                                                 │ │
│  │     - 콜드 미스 발생 가능                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최적화 기법】                                                                             │
│  ──────────────────                                                  │

│  │  기법                      효과                                │ │
│  │  ────                        ────                                │ │
```

## Ⅲ. 실무 적용

### 측정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 디스패치 지연 확인】                                             │
│  ──────────────────                                                  │
│  # 스케줄링 지연 측정                                                 │
│  $ perf sched record sleep 10                                        │
│  $ perf sched latency                                                │
│                                                                     │
│  # 실시간 지연 측정                                                    │
│  $ rt-tests/cyclictest -t1 -p 80 -n                                  │
│  # Min: 1 µs, Max: 50 µs, Avg: 5 µs                                 │
│                                                                     │
│  # ftrace로 스케줄링 이벤트 추적                                       │
│  $ trace-cmd record -e sched:sched_switch ./program                   │
│  $ trace-cmd report                                                  │
│                                                                     │
│  【실시간 커널 설정】                                                    │
│  ──────────────────                                                  │
│  # PREEMPT_RT 패치 확인                                               │
│  $ cat /boot/config-$(uname -r) | grep PREEMPT                        │
│  CONFIG_PREEMPT_RT=y        # 실시간 커널                              │ │
│                                                                     │
│  # 실시간 우선순위 스레드 실행                                          │
│  $ chrt -f 99 cyclictest -t1 -p 80 -n                                │
│                                                                     │
│  【지연에 영향을 주는 설정】                                              │
│  ──────────────────                                                  │
│  # 타임 슬라이스 관련                                                  │
│  $ cat /proc/sys/kernel/sched_min_granularity_ns                      │
│                                                                     │
│  # CPU 마이그레이션 비용                                               │
│  $ cat /proc/sys/kernel/sched_migration_cost_ns                       │
│                                                                     │
│  # 자동 그룹 스케줄링                                                  │
│  $ cat /proc/sys/kernel/sched_autogroup_enabled                       │
│                                                                     │
│  【문맥 교환 비용 측정 코드】                                             │
│  ──────────────────                                                  │
│  #include <time.h>                                                   │
│  #include <unistd.h>                                                 │
│  #include <sys/syscall.h>                                            │
│                                                                     │
│  // gettid()로 문맥 교환 측정                                          │
│  long measure_ctx_switch() {                                          │
│      struct timespec start, end;                                      │
│      clock_gettime(CLOCK_MONOTONIC, &start);                          │
│      sched_yield();  // 문맥 교환 강제                                  │
│      clock_gettime(CLOCK_MONOTONIC, &end);                            │
│      return (end.tv_sec - start.tv_sec) * 1000000000 +                │
│             (end.tv_nsec - start.tv_nsec);                            │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 프로세스 전환 시간
• 구성: 문맥 저장, 복원, 모드 전환
• 일반적: 1-15 µs
• 실시간: 10-100 µs 이하 필요
• 영향: CPU 아키텍처, 캐시, OS 구현
• 최적화: 선점형 커널, TLB 유지, Affinity
```
