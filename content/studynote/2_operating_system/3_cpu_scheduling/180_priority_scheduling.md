# 179. 우선순위 스케줄링 (Priority Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 우선순위가 높은 프로세스 먼저 실행
> 2. **가치**: 중요 작업 우선 처리, 실시간 시스템 필수
> 3. **융합**: 기아 현상, 에이징, 선점/비선점형과 연관

---

## Ⅰ. 개요

### 개념 정의

우선순위 스케줄링(Priority Scheduling)은 **각 프로세스에 우선순위를 부여하고, 우선순위가 가장 높은 프로세스를 먼저 실행하는 알고리즘**이다. 선점형과 비선점형 모두 가능하다.

### 💡 비유: 응급실 우선순위
우선순위 스케줄링은 **응급실 분류**와 같다. 위급한 환자(높은 우선순위)가 먼저 진료받고, 경증 환자는 나중에 진료받는다.

### 우선순위 스케줄링 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                우선순위 스케줄링 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 방식】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  우선순위 기준:                                                │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  내부적                    외부적                        │ │ │
│  │  │  ────                        ────                         │ │ │
│  │  │  • 메모리 요구량            • 프로세스 중요도             │ │ │
│  │  │  • CPU 버스트 시간          • 사용자 유형                 │ │ │
│  │  │  • I/O 요구량              • 정치적/상업적 요인           │ │ │
│  │  │  • 파일 열기 수                                           │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  우선순위 큐:                                                  │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  우선순위 0 (높음)  →  [P1]                              │ │ │
│  │  │  우선순위 1          →  [P2] [P5]                        │ │ │
│  │  │  우선순위 2          →  [P3]                             │ │ │
│  │  │  우선순위 3 (낮음)  →  [P4] [P6]                         │ │ │
│  │  │                                                        │ │ │
│  │  │  → 가장 높은 우선순위의 P1 먼저 실행                       │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【선점형 vs 비선점형】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  비선점형:                                                      │ │
│  │  • 실행 중인 프로세스 종료/I/O까지 계속                          │ │
│  │  • 새로운 높은 우선순위 도착해도 대기                            │ │
│  │                                                             │ │
│  │  선점형:                                                        │ │
│  │  • 더 높은 우선순위 도착 시 즉시 선점                            │ │
│  │  • 실시간 시스템에 적합                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【우선순위 표현】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템              높은 우선순위        낮은 우선순위          │ │
│  │  ────                        ────                  ────         │ │
│  │  일반적              0                  큰 수                   │ │
│  │  Linux               낮은 nice 값       높은 nice 값             │ │
│  │  Windows             높은 수            낮은 수                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 설정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 우선순위 조작】                                                │
│  ──────────────────                                                  │
│  # nice 값으로 실행                                                  │
│  $ nice -n 10 ./low_priority_task      # 낮은 우선순위               │
│  $ nice -n -10 ./high_priority_task    # 높은 우선순위               │
│                                                                     │
│  # 실행 중인 프로세스 우선순위 변경                                     │
│  $ renice -n 5 -p 1234                                            │
│                                                                     │
│  # 실시간 우선순위 설정                                                │
│  $ chrt -f 50 ./realtime_task        # SCHED_FIFO, 우선순위 50       │
│  $ chrt -r 30 ./realtime_task        # SCHED_RR, 우선순위 30         │
│                                                                     │
│  【프로그래밍에서의 우선순위 설정】                                       │
│  ──────────────────                                                  │
│  #include <sched.h>                                                  │
│  #include <sys/resource.h>                                           │
│  #include <unistd.h>                                                 │
│                                                                     │
│  // nice 값 변경                                                     │
│  void set_nice(int nice_value) {                                     │
│      setpriority(PRIO_PROCESS, 0, nice_value);                       │
│  }                                                                   │
│                                                                     │
│  // 실시간 스케줄링 설정                                               │
│  void set_realtime_priority(int policy, int priority) {              │
│      struct sched_param param;                                       │
│      param.sched_priority = priority;                                │
│      sched_setscheduler(0, policy, &param);                          │
│  }                                                                   │
│                                                                     │
│  // 사용 예                                                          │
│  set_nice(10);                     // 낮은 우선순위                   │
│  set_realtime_priority(SCHED_FIFO, 50);  // 실시간                   │
│                                                                     │
│  【우선순위 확인】                                                      │
│  ──────────────────                                                  │
│  # 프로세스별 우선순위                                                  │
│  $ ps -eo pid,ni,pri,comm                                           │
│  # ni: nice 값, pri: 커널 우선순위                                   │
│                                                                     │
│  # 실시간 우선순위                                                     │
│  $ ps -eo pid,rtprio,ni,comm                                        │
│                                                                     │
│  # top에서 확인                                                       │
│  $ top -o PR                                                        │
│                                                                     │
│  【Windows 우선순위】                                                   │
│  ──────────────────                                                  │
│  // Windows API                                                      │
│  SetPriorityClass(GetCurrentProcess(), HIGH_PRIORITY_CLASS);         │
│  SetThreadPriority(GetCurrentThread(), THREAD_PRIORITY_ABOVE_NORMAL);│
│                                                                     │
│  // 우선순위 클래스                                                    │
│  IDLE_PRIORITY_CLASS          // 64                                  │
│  BELOW_NORMAL_PRIORITY_CLASS  // 16384                               │
│  NORMAL_PRIORITY_CLASS        // 32 (기본)                           │
│  ABOVE_NORMAL_PRIORITY_CLASS  // 32768                               │
│  HIGH_PRIORITY_CLASS          // 128                                 │
│  REALTIME_PRIORITY_CLASS      // 256                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
