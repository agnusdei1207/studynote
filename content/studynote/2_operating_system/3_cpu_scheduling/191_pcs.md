# 191. 문맥 교환 오버헤드 (Context Switch Overhead)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 문맥 교환 중 발생하는 CPU 사이클 손실
> 2. **가치**: 스케줄링 효율성의 핵심 고려사항
> 3. **융합**: 문맥 교환, 캐시, TLB, 파이프라인과 연관

---

## Ⅰ. 개요

### 개념 정의

문맥 교환 오버헤드(Context Switch Overhead)는 **프로세스/스레드 전환 시 발생하는 모든 비용**이다. 레지스터 저장/복원, 캐시 갱신, TLB 플러시 등이 포함된다.

### 💡 비유: 작업 전환 비용
문맥 교환 오버헤드는 **책상에서 다른 과제로 바꿀 때의 준비 시간**과 같다. 이전 과제를 치우고, 새 과제 자료를 꺼내고, 생각을 전환하는 데 시간이 걸린다.

### 문맥 교환 오버헤드 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                문맥 교환 오버헤드 구조                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【오버헤드 구성 요소】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  직접 비용 (Direct Cost):                                        │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │ 레지스터 저장/복원        ~0.5-2 µs                       │ │ │
│  │  │ PCB 업데이트             ~0.1-0.5 µs                      │ │ │
│  │  │ 커널 모드 전환           ~0.1-0.5 µs                       │ │ │
│  │  │ 스케줄러 실행            ~0.5-2 µs                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  간접 비용 (Indirect Cost):                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │ 캐시 미스 (L1/L2/L3)      ~0-10 µs                        │ │ │
│  │  │ TLB 미스                 ~0.5-5 µs                        │ │ │
│  │  │ 파이프라인 플러시         ~0.1-1 µs                        │ │ │
│  │  │ 분기 예측 실패           ~0.1-0.5 µs                       │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  전체: ~1-20 µs (시스템에 따라 다름)                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【오버헤드 영향 시각화】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  P1 실행         문맥 교환          P2 실행                      │ │
│  │  ┌───────────┐   ┌────────────┐   ┌───────────┐              │ │
│  │  │ 유용한    │   │ 오버헤드   │   │ 유용한    │              │ │
│  │  │ 작업      │   │ 손실       │   │ 작업      │              │ │
│  │  └───────────┘   └────────────┘   └───────────┘              │ │
│  │                                                             │ │
│  │  ← 타임 슬라이스 →← CS 오버헤드 →← 타임 슬라이스 →                │ │
│  │                                                             │ │
│  │  효율 = 유용한 작업 / (유용한 작업 + 오버헤드)                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                문맥 교환 오버헤드 상세                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【타임 퀀텀과 오버헤드 비율】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  타임 퀀텀      CS 오버헤드      오버헤드 비율                    │ │
│  │  ────                ────              ────                      │ │
│  │  1 ms            10 µs           1%                             │ │
│  │  10 ms           10 µs           0.1%                           │ │
│  │  100 ms          10 µs           0.01%                          │ │
│  │                                                             │ │
│  │  → 타임 퀀텀이 작을수록 오버헤드 비율 증가                          │ │
│  │  → 너무 작은 타임 퀀텀 = 비효율                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 효과 (가장 큰 간접 비용)】                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  캐시 레벨          크기           접근 시간     미스 시간        │ │
│  │  ────                  ────              ────          ────        │ │
│  │  L1                ~64 KB          ~1 ns       ~10 ns              │ │
│  │  L2               ~256 KB          ~4 ns       ~40 ns              │
│  │  L3               ~8 MB            ~10 ns      ~100 ns             │
│  │  메모리            ~GB             ~100 ns     -                   │
│  │                                                             │ │
│  │  문맥 교환 후:                                                   │ │
│  │  • 이전 프로세스의 캐시 데이터 무효화                              │ │
│  │  • 새 프로세스의 콜드 캐시 미스 발생                               │ │
│  │  • 수백~수천 번의 캐시 미스 가능                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최적화 기법】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  기법                      효과                                │ │
│  │  ────                        ────                                │ │
│  │  CPU Affinity            캐시 지역성 유지                        │ │
│  │  ASID/PCID               TLB 플러시 방지                         │ │
│  │  레지스터 윈도우           저장/복원 최소화                        │ │
│  │  큰 타임 퀀텀             문맥 교환 빈도 감소                      │ │
│  │  스레드 풀               프로세스 생성/종료 감소                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 측정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【문맥 교환 횟수 확인】                                                 │
│  ──────────────────                                                  │
│  # 전체 문맥 교환 횟수                                                  │
│  $ cat /proc/stat | grep ctxt                                         │
│  ctxt 123456789                                                       │
│                                                                     │
│  # 초당 문맥 교환                                                      │
│  $ vmstat 1                                                          │
│  # cs 컬럼 확인                                                       │
│                                                                     │
│  # 프로세스별 문맥 교환                                                 │
│  $ pidstat -w 1                                                      │
│  # cswch/s: 자발적, nvcswch/s: 비자발적                               │
│                                                                     │
│  【문맥 교환 시간 측정】                                                 │
│  ──────────────────                                                  │
│  # perf로 측정                                                        │
│  $ perf stat -e context-switches,cpu-migrations ./program           │
│                                                                     │
│  # ftrace로 상세 추적                                                 │
│  $ trace-cmd record -e sched:sched_switch ./program                  │
│  $ trace-cmd report                                                  │
│                                                                     │
│  【프로그래밍에서 최소화】                                                │
│  ──────────────────                                                  │
│  #include <pthread.h>                                                │
│  #include <sched.h>                                                  │
│                                                                     │
│  // CPU Affinity 설정                                                 │
│  void set_cpu_affinity(int cpu) {                                    │
│      cpu_set_t cpuset;                                               │
│      CPU_ZERO(&cpuset);                                              │
│      CPU_SET(cpu, &cpuset);                                          │
│      pthread_setaffinity_np(pthread_self(), sizeof(cpuset), &cpuset);│
│  }                                                                   │
│                                                                     │
│  // 불필요한 yield 피하기                                               │
│  // 나쁨: 불필요한 문맥 교환 유발                                        │
│  while (!done) {                                                     │
│      sched_yield();  // 피해야 함                                     │
│  }                                                                   │
│                                                                     │
│  // 좋음: 조건 변수 사용                                               │
│  pthread_mutex_lock(&mutex);                                         │
│  while (!done) {                                                     │
│      pthread_cond_wait(&cond, &mutex);                               │
│  }                                                                   │
│  pthread_mutex_unlock(&mutex);                                       │
│                                                                     │
│  【문맥 교환 많은 프로세스 찾기】                                         │
│  ──────────────────                                                  │
│  $ pidstat -w | sort -k 3 -rn | head -10                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 문맥 교환 중 발생하는 CPU 손실
• 직접 비용: 레지스터, PCB, 모드 전환
• 간접 비용: 캐시, TLB, 파이프라인
• 범위: ~1-20 µs
• 영향: 타임 퀀텀 작을수록 비율 증가
• 최적화: CPU Affinity, ASID, 큰 타임 퀀텀
```
