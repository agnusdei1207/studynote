# 213. 선점 모델 (Preemption Models)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 코드가 선점될 수 있는 시점 결정
> 2. **가치**: 응답성과 처리량의 트레이드오프
> 3. **융합**: 실시간, 커널 설정, 스케줄링과 연관

---

## Ⅰ. 개요

### 개념 정의

선점 모델(Preemption Models)은 **커널 모드 코드가 언제 선점될 수 있는지를 결정하는 정책**이다. CONFIG_PREEMPT 설정에 따라 3가지 모델로 구분된다.

### 💡 비유: 회의 중 전화
선점 모델은 **회의 중 전화 받기**와 같다. 중요 전화는 즉시 받고, 아니면 나중에 받는다.

### 선점 모델 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                선점 모델 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【세 가지 선점 모델】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. CONFIG_PREEMPT_NONE (서버/볼륨틱)                          │ │
│  │     ┌─────────────────────────────────────────────────────┐ │ │
│  │     │ • 커널 모드에서 선점 안 함                             │ │ │
│  │     │ • 사용자 모드 전환 시에만 선점                          │ │ │
│  │     │ • 최대 처리량                                          │ │ │
│  │     │ • 최고 캐시 효율                                        │ │ │
│  │     │ • 서버/데이터 센터용                                    │ │ │
│  │     └─────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  2. CONFIG_PREEMPT_VOLUNTARY (데스크탑/볼런터리)                │ │
│  │     ┌─────────────────────────────────────────────────────┐ │ │
│  │     │ • cond_resched() 지점에서만 선점                       │ │ │
│  │     │ • 명시적 선점 지점                                      │ │ │
│  │     │ • 균형잡힌 응답성                                       │ │ │
│  │     │ • 데스크탑 배포판 기본                                  │ │ │
│  │     └─────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  3. CONFIG_PREEMPT (선점형/프리엠션)                            │ │
│  │     ┌─────────────────────────────────────────────────────┐ │ │
│  │     │ • 커널 모드에서도 언제든 선점 가능                      │ │ │
│  │     │ • spinlock 제외                                        │ │ │
│  │     │ • 최고 응답성                                          │ │ │
│  │     │ • 저지연 데스크탑                                       │ │ │
│  │     └─────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  4. CONFIG_PREEMPT_RT (실시간)                                 │ │
│  │     ┌─────────────────────────────────────────────────────┐ │ │
│  │     │ • 모든 spinlock을 mutex로 변환                        │ │ │
│  │     │ • 인터럽트 핸들러도 스레드화                            │ │ │
│  │     │ • 완전 선점형                                          │ │ │
│  │     │ • 실시간 시스템용                                       │ │ │
│  │     └─────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【응답성 vs 처리량】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  응답성        PREEMPT_RT  PREEMPT  VOLUNTARY  NONE           │ │
│  │    ▲            최고        높음     중간       낮음            │ │
│  │    │                                                        │ │
│  │    ▼                                                        │ │
│  │  처리량        낮음        중간     높음       최고             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                선점 모델 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【선점 포인트】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  모델              선점 발생 시점                                │ │
│  │  ────                    ────                                    │ │
│  │  NONE            사용자 모드 전환, 인터럽트 반환                  │ │
│  │  VOLUNTARY       cond_resched() 호출 시                         │ │
│  │  PREEMPT         인터럽트 반환, preemt_enable()                  │ │
│  │  PREEMPT_RT      거의 모든 지점                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최대 선점 지연】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  모델              최대 지연 (대략)                              │ │
│  │  ────                    ────                                    │ │
│  │  NONE            수백 ms                                        │ │
│  │  VOLUNTARY       수십 ms                                        │ │
│  │  PREEMPT         수 ms                                          │ │
│  │  PREEMPT_RT      수십 μs                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【PREEMPT_RT 변환】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • spinlock_t → rt_mutex                                        │ │
│  │  • local_irq_disable() → preempt_disable()                     │ │
│  │  • 인터럽트 핸들러 → 스레드                                       │ │
│  │  • preempt_count 추적                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 설정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【현재 선점 모델 확인】                                               │
│  ──────────────────                                                  │
│  # 커널 설정 확인                                                     │
│  $ grep PREEMPT /boot/config-$(uname -r)                            │
│  CONFIG_PREEMPT_VOLUNTARY=y                                         │
│  # 또는                                                              │
│  CONFIG_PREEMPT=y                                                   │
│  # 또는                                                              │
│  CONFIG_PREEMPT_NONE=y                                              │
│                                                                     │
│  # 커널 버전 확인 (RT 패치 포함 여부)                                  │
│  $ uname -v                                                         │
│  # ... PREEMPT RT ...                                               │
│                                                                     │
│  【실시간 커널 설치】                                                  │
│  ──────────────────                                                  │
│  # Ubuntu/Debian                                                    │
│  $ sudo apt install linux-image-rt-amd64                            │
│                                                                     │
│  # RHEL/CentOS                                                      │
│  $ sudo yum install kernel-rt                                       │
│                                                                     │
│  # 커스텀 커널 빌드                                                   │
│  $ make menuconfig                                                  │
│  # General setup -> Preemption Model -> Fully Preemptible Kernel    │
│                                                                     │
│  【선점 지연 측정】                                                    │
│  ──────────────────                                                  │
│  # cyclictest로 측정                                                 │
│  $ cyclictest -t1 -p 80 -n -i 10000 -l 10000                        │
│                                                                     │
│  # 결과 예시                                                         │
│  # Min: 5 us, Max: 123 us, Avg: 15 us                              │
│                                                                     │
│  # ftrace로 선점 추적                                                │
│  $ echo function > /sys/kernel/debug/tracing/current_tracer        │
│  $ echo sched_switch >> /sys/kernel/debug/tracing/set_event        │
│                                                                     │
│  【실시간 설정 튜닝】                                                  │
│  ──────────────────                                                  │
│  # CPU 주파수 고정                                                   │
│  $ cpupower frequency-set -g performance                            │
│                                                                     │
│  # RT throttling                                                    │
│  $ sysctl -w kernel.sched_rt_runtime_us=-1                          │
│                                                                     │
│  # CPU isolation                                                    │
│  # 커널 파라미터: isolcpus=2,3                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 커널 선점 가능 시점 결정
• NONE: 서버용, 최대 처리량
• VOLUNTARY: 데스크탑, 균형
• PREEMPT: 저지연, 높은 응답성
• PREEMPT_RT: 실시간, 최소 지연
• 설정: CONFIG_PREEMPT_*
```
