# 217. 스케줄링 지연 (Scheduling Latency)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 태스크가 실행 준비부터 실제 실행까지의 지연
> 2. **가치**: 시스템 응답성의 핵심 지표
> 3. **융합**: 웨이크업 지연, 선점 지연, 실시간과 연관

---

## Ⅰ. 개요

### 개념 정의

스케줄링 지연(Scheduling Latency)은 **태스크가 실행 준비 상태가 된 시점부터 실제 CPU를 할당받아 실행되기까지 걸리는 시간**이다. 웨이크업 지연(wakeup latency)이라고도 한다.

### 💡 비유: 식당 대기
스케줄링 지연은 **식당 대기 시간**과 같다. 주문 완료(실행 준비)부터 음식 도착(실제 실행)까지 걸리는 시간이다.

### 스케줄링 지연 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄링 지연 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【지연 구성 요소】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시간 ────────────────────────────────────────────────────►  │ │
│  │                                                             │ │
│  │  이벤트 발생         스케줄링          실행 시작               │ │
│  │       │                 │                  │                  │ │
│  │       ▼                 ▼                  ▼                  │ │
│  │       ├─────┬───────────┼──────────────────┤                  │ │
│  │       │     │           │                  │                  │ │
│  │       │ 인터럽트  │   스케줄링    │     문맥 교환        │                  │
│  │       │ 처리     │   지연       │                  │                  │
│  │       │     │           │                  │                  │ │
│  │       └─────┴───────────┴──────────────────┘                  │ │
│  │                                                             │ │
│  │                    전체 스케줄링 지연                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지연 원인】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 인터럽트 비활성화 구간                                       │ │
│  │     • spinlock 보유                                             │ │
│  │     • IRQ 핸들러 실행                                           │ │
│  │                                                             │ │
│  │  2. 선점 불가 구간                                               │ │
│  │     • preempt_count > 0                                         │ │
│  │     • 커널 비선점 구간                                          │ │
│  │                                                             │ │
│  │  3. 다른 태스크 실행 중                                          │ │
│  │     • 더 높은 우선순위 실행                                     │ │
│  │     • 타임 슬라이스 소진 대기                                   │ │
│  │                                                             │ │
│  │  4. 로드 밸런싱                                                  │ │
│  │     • CPU 간 마이그레이션                                       │ │
│  │     • 캐시 콜드                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지연 vs 응답 시간】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스케줄링 지연: 준비 → 실행 시작                                │ │
│  │  응답 시간:     요청 → 응답 완료                                │ │
│  │  처리 시간:     실행 시작 → 실행 완료                           │ │
│  │                                                             │ │
│  │  응답 시간 = 스케줄링 지연 + 처리 시간                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 측정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【cyclictest로 지연 측정】                                           │
│  ──────────────────                                                  │
│  # 기본 측정                                                          │
│  $ cyclictest -t1 -p 80 -n -i 10000 -l 10000                        │
│                                                                     │
│  # 결과 해석                                                          │
│  # Min: 최소 지연, Act: 실제, Max: 최대 지연                         │
│  # Policy: SCHED_OTHER, SCHED_FIFO 등                               │
│                                                                     │
│  # 멀티 스레드                                                        │
│  $ cyclictest -t5 -p 80 -n -i 1000 -l 10000                         │
│                                                                     │
│  【ftrace로 지연 추적】                                               │
│  ──────────────────                                                  │
│  # 웨이크업 지연 추적                                                 │
│  $ echo 0 > /sys/kernel/debug/tracing/tracing_on                  │
│  $ echo wakeup > /sys/kernel/debug/tracing/current_tracer         │
│  $ echo 1 > /sys/kernel/debug/tracing/tracing_on                  │
│  $ cat /sys/kernel/debug/tracing/trace                             │
│                                                                     │
│  # sched_switch 이벤트                                               │
│  $ trace-cmd record -e sched:sched_switch ./program                │
│  $ trace-cmd report                                                 │
│                                                                     │
│  【latency_top 사용】                                                 │
│  ──────────────────                                                  │
│  # 설치 및 실행                                                       │
│  $ sudo apt install latencytop                                      │
│  $ sudo latencytop                                                  │
│                                                                     │
│  # 지연 원인 확인                                                     │
│  # - CPU starvation                                                 │
│  # - Block device wait                                              │
│  # - Memory pressure                                                │
│                                                                     │
│  【지연 최소화 튜닝】                                                  │
│  ──────────────────                                                  │
│  # CPU 격리                                                          │
│  # 커널 파라미터: isolcpus=2,3 nohz_full=2,3 rcu_nocbs=2,3         │
│                                                                     │
│  # 실시간 우선순위                                                    │
│  $ chrt -f -p 80 <pid>                                             │
│                                                                     │
│  # 스케줄러 파라미터                                                  │
│  $ sysctl -w kernel.sched_min_granularity_ns=1000000               │
│  $ sysctl -w kernel.sched_rt_runtime_us=-1                         │
│                                                                     │
│  # 인터럽트 분산                                                      │
│  $ echo 0f > /proc/irq/24/smp_affinity                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
