# 184. CFS (완전 공정 스케줄러, Completely Fair Scheduler)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 런타임 기반 공정한 CPU 분배
> 2. **가치**: 현대 Linux의 기본 스케줄러
> 3. **융합**: vruntime, 레드블랙트리, nice 값과 연관

---

## Ⅰ. 개요

### 개념 정의

CFS(Completely Fair Scheduler)는 **모든 프로세스에게 공평하게 CPU 시간을 분배하기 위해 가상 런타임(vruntime)을 사용하는 리눅스의 기본 스케줄러**다. 2.6.23 커널부터 도입되었다.

### 💡 비유: 공평한 놀이터
CFS는 **공평한 놀이터**와 같다. 각 아이에게 똑같은 시간을 주고, 더 적게 놀은 아이가 먼저 기회를 갖는다.

### CFS 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                CFS 구조                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 개념: vruntime】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  vruntime = 실제 실행 시간 × 가중치                            │ │
│  │                                                             │ │
│  │  가중치: nice 값에 따라 결정                                   │ │
│  │  • nice -20: 가중치 높음 → vruntime 천천히 증가                 │ │
│  │  • nice   0: 기본 가중치                                      │ │
│  │  • nice +19: 가중치 낮음 → vruntime 빨리 증가                   │ │
│  │                                                             │ │
│  │  스케줄링: 가장 낮은 vruntime을 가진 태스크 선택                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【레드블랙트리 구조】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │                    ┌─────────┐                              │ │
│  │                    │ vrt=50  │                              │ │
│  │                    │  Task C │                              │ │
│  │                    └─────────┘                              │ │
│  │                    ╱         ╲                              │ │
│  │             ┌─────────┐   ┌─────────┐                       │ │
│  │             │ vrt=30  │   │ vrt=70  │                       │ │
│  │             │  Task A │   │  Task D │                       │ │
│  │             └─────────┘   └─────────┘                       │ │
│  │                ╱                                            │ │
│  │          ┌─────────┐                                       │ │
│  │          │ vrt=10  │  ← 가장 낮은 vruntime                  │ │
│  │          │  Task B │    → 다음에 실행                       │ │
│  │          └─────────┘                                       │ │
│  │                                                             │ │
│  │  O(log n)로 최소 vruntime 태스크 검색                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【타임 슬라이스 계산】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  타임 슬라이스 = (태스크 가중치 / 전체 가중치) × 스케줄링 주기     │ │
│  │                                                             │ │
│  │  스케줄링 주기: 모든 실행 가능한 태스크가 한 번씩 실행하는 시간    │ │
│  │                                                             │ │
│  │  nice가 낮을수록(높은 우선순위) 더 큰 타임 슬라이스               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                CFS 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CFS 특징】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • "완전 공정": 이상적으로 모든 태스크가 동등한 CPU 시간           │ │
│  │  • vruntime: 실행 시간을 정규화하여 비교                          │ │
│  │  • 레드블랙트리: 효율적인 태스크 관리                               │ │
│  │  • nice 지원: 우선순위를 가중치로 반영                             │ │
│  │  • 선점형: vruntime이 작은 태스크가 나타나면 선점                   │ │
│  │  • 그룹 스케줄링: 태스크 그룹별 공정성                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【nice 값과 가중치】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  nice 값          가중치              CPU 비율                  │ │
│  │  ────                  ────                   ────               │ │
│  │  -20 (높음)       88761              ~45%                      │ │
│  │  -10              26212              ~13%                       │ │
│  │    0 (기본)       1024               ~5%                        │ │
│  │  +10              40                 ~0.2%                      │ │
│  │  +19 (낮음)       15                 ~0.01%                     │ │
│  │                                                               │ │
│  │  nice -1 = CPU 시간 약 1.25배                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CFS vs 이전 스케줄러】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              O(1) 스케줄러         CFS                    │ │
│  │  ────                    ────                     ────           │ │
│  │  데이터 구조        우선순위 배열          레드블랙트리            │ │
│  │  타임 슬라이스      고정                   동적                   │ │
│  │  공정성             휴리스틱               수학적 모델             │ │
│  │  대화형 감지        복잡한 로직            자동                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 모니터링 및 설정
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CFS 설정 확인】                                                      │
│  ──────────────────                                                  │
│  # 최소 그랜규래리티                                                   │
│  $ cat /proc/sys/kernel/sched_min_granularity_ns                     │
│  2250000    # 2.25ms                                                 │
│                                                                     │
│  # wakeup 그랜규래리티                                                 │
│  $ cat /proc/sys/kernel/sched_wakeup_granularity_ns                  │
│                                                                     │
│  # 마이그레이션 비용                                                   │
│  $ cat /proc/sys/kernel/sched_migration_cost_ns                      │
│                                                                     │
│  # 자동 그룹 스케줄링                                                  │
│  $ cat /proc/sys/kernel/sched_autogroup_enabled                      │
│                                                                     │
│  【태스크 vruntime 확인】                                               │
│  ──────────────────                                                  │
│  # 특정 태스크 스케줄링 정보                                            │
│  $ cat /proc/<pid>/sched                                             │
│                                                                     │
│  # 주요 필드                                                         │
│  # se.vruntime           : 가상 런타임                               │
│  # se.sum_exec_runtime   : 총 실행 시간                              │
│  # se.load.weight        : 가중치                                    │
│  # nr_switches           : 문맥 교환 횟수                            │
│                                                                     │
│  【CFS 디버깅】                                                          │
│  ──────────────────                                                  │
│  # 스케줄러 디버그 정보                                                 │
│  $ cat /proc/sched_debug                                             │
│                                                                     │
│  # 런큐 정보                                                          │
│  $ cat /proc/sched_debug | grep -A 20 "cfs_rq"                       │
│                                                                     │
│  【성능 튜닝】                                                           │
│  ──────────────────                                                  │
│  # 대화형 워크로드                                                     │
│  $ sysctl -w kernel.sched_min_granularity_ns=1000000                 │
│                                                                     │
│  # 배치 워크로드                                                       │
│  $ sysctl -w kernel.sched_min_granularity_ns=10000000                │
│                                                                     │
│  # CFS 설정 영구 적용                                                  │
│  # /etc/sysctl.conf에 추가                                           │
│  kernel.sched_min_granularity_ns = 10000000                          │
│                                                                     │
│  【프로그래밍에서의 활용】                                                │
│  ──────────────────                                                  │
│  #include <sys/resource.h>                                           │
│                                                                     │
│  // nice 값 설정                                                      │
│  setpriority(PRIO_PROCESS, 0, -5);  // 높은 우선순위                   │
│                                                                     │
│  // nice 값 확인                                                      │
│  int nice = getpriority(PRIO_PROCESS, 0);                            │
│                                                                     │
│  // CPU亲和性 설정 (같은 CPU에서 실행되면 캐시 효율)                    │
│  cpu_set_t cpuset;                                                   │
│  CPU_ZERO(&cpuset);                                                  │
│  CPU_SET(0, &cpuset);                                                │
│  sched_setaffinity(0, sizeof(cpuset), &cpuset);                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 가상 런타임 기반 공정 스케줄링
• 데이터: 레드블랙트리 (O(log n))
• 선택: 가장 낮은 vruntime 태스크
• 가중치: nice 값에 따라 vruntime 증가 속도
• 장점: 수학적 공정성, 자동 I/O/CPU 구분
• 도입: Linux 2.6.23 (2007)
```
