# 167. 비선점형 스케줄링 (Non-preemptive Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실행 중인 프로세스가 자발적으로 CPU 반납
> 2. **가치**: 구현 간단, 문맥 교환 오버헤드 적음
> 3. **융합**: 선점형, 배치 시스템, FCFS와 연관

---

## Ⅰ. 개요

### 개념 정의

비선점형 스케줄링(Non-preemptive Scheduling)은 **실행 중인 프로세스가 자발적으로 CPU를 반납할 때까지 계속 실행하는 방식**이다. 운영체제가 강제로 CPU를 회수할 수 없다.

### 💡 비유: 자유 발언 시간
비선점형 스케줄링은 **자유 발언 시간**과 같다. 발언자가 자리에서 내려올 때까지(자발적 반납) 다음 사람은 기다려야 한다.

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Windows 스케줄링 우선순위 클래스】                                     │
│  ──────────────────                                                  │
│  // 스레드 우선순위 설정                                               │
│  SetThreadPriority(hThread, THREAD_PRIORITY_NORMAL);                  │
│  SetThreadPriority(hThread, THREAD_PRIORITY_ABOVE_NORMAL);            │
│                                                                     │
│  【자발적 CPU 양보 (cooperative multitasking)】                           │
│  ──────────────────                                                  │
│  // Windows                                                          │
│  Sleep(0);  // 다른 스레드에게 양보                                    │
│  SwitchToThread();                                                    │
│                                                                     │
│  // POSIX                                                            │
│  sched_yield();                                                       │
│                                                                     │
│  // Qt (이벤트 루프)                                                   │
│  QCoreApplication::processEvents();                                   │
│                                                                     │
│  【비선점형 코루틴/협력적 멀티태스킹】                                     │
│  ──────────────────                                                  │
│  // Python (비선점형)                                                 │
│  import asyncio                                                      │
│                                                                     │
│  async def task1():                                                   │
│      print("task1 시작")                                              │
│      await asyncio.sleep(0)  # 여기서 양보                            │
│      print("task1 계속")                                              │
│                                                                     │
│  async def task2():                                                   │
│      print("task2 시작")                                              │
│      await asyncio.sleep(0)  # 여기서 양보                            │
│      print("task2 계속")                                              │
│                                                                     │
│  【cooperative 스레드 풀】                                               │
│  ──────────────────                                                  │
│  // 작업이 길면 중간에 yield                                           │
│  void process_chunk(int start, int end) {                             │
│      for (int i = start; i < end; i++) {                              │
│          // 작업...                                                    │
│          if (i % 1000 == 0)                                           │
│              sched_yield();  // 다른 스레드에 양보                       │
│      }                                                               │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 핵심 요약

```

```
