# 176. SRTF (최단 잔여 시간 우선, Shortest Remaining Time First)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 남은 CPU 버스트가 가장 짧은 프로세스 먼저 실행
> 2. **가치**: SJF의 선점형 버전, 평균 대기 시간 최소
> 3. **융합**: SJF, 선점형, 문맥 교환과 연관

---

## Ⅰ. 개요

### 개념 정의

SRTF(Shortest Remaining Time First)는 **남은 CPU 버스트 시간이 가장 짧은 프로세스를 먼저 실행하는 선점형 스케줄링 알고리즘**이다. SJF의 선점형 버전이다.

### 💡 비유: 점심시간 식당
SRTF는 **점심시간 식당**과 같다. 먹는 속도가 빠른 손님에게 우선 자리를 주고, 새로운 빠른 손님이 오면 느린 손님을 잠시 대기시킨다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                SRTF 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장단점】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 평균 대기 시간 최소       • 잦은 문맥 교환                   │ │
│  │  • 새 작업 즉시 반영         • 버스트 시간 추정 필요             │ │
│  │  • 응답성 좋음              • 구현 복잡                        │ │
│  │  • 처리량 높음              • 오버헤드 가능                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【선점 발생 조건】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  새 프로세스 P_new 도착 시:                                      │ │
│  │                                                             │ │
│  │  if (P_new.잔여시간 < P_current.잔여시간) {                      │ │
│  │      P_current 선점;                                           │ │
│  │      P_new 실행;                                               │ │
│  │  } else {                                                      │ │
│  │      P_new를 준비 큐에 삽입 (잔여시간순);                        │ │
│  │  }                                                             │ │
│  │                                                             │ │
│  │  ※ 매 tick마다 잔여 시간 업데이트                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【문맥 교환 오버헤드】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SRTF는 새 프로세스 도착마다 선점 가능                            │ │
│  │  → 문맥 교환 빈도 높음                                          │ │
│  │  → 오버헤드 증가                                                │ │
│  │  → 실제 시스템에서는 타협 필요                                   │ │
│  │                                                               │ │
│  │  해결: 일정 임계값 이상 차이날 때만 선점                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SRTF 구현 (의사코드)】                                                │
│  ──────────────────                                                  │
│  struct Process {                                                    │
│      int pid, arrival, burst, remaining, completion;                 │
│      int waiting, turnaround;                                        │
│  };                                                                  │
│                                                                     │
│  void SRTF(Process p[], int n) {                                     │
│      int completed = 0, current_time = 0;                            │
│      int min_remaining = INT_MAX;                                    │
│      int shortest = 0, finish_time;                                  │
│      bool check = false;                                             │
│                                                                     │
│      // 모든 프로세스 완료까지                                         │
│      while (completed != n) {                                        │
│          // 현재 시간까지 도착한 프로세스 중 최소 잔여 찾기             │
│          for (int i = 0; i < n; i++) {                               │
│              if (p[i].arrival <= current_time &&                     │
│                  p[i].remaining < min_remaining &&                   │
│                  p[i].remaining > 0) {                               │
│                  min_remaining = p[i].remaining;                     │
│                  shortest = i;                                       │
│                  check = true;                                       │
│              }                                                       │
│          }                                                           │
│                                                                     │
│          if (!check) {                                               │
│              current_time++;                                         │
│              continue;                                               │
│          }                                                           │
│                                                                     │
│          // 1단위 시간 실행                                           │
│          p[shortest].remaining--;                                    │
│          min_remaining = p[shortest].remaining;                      │
│          current_time++;                                             │
│                                                                     │
│          // 완료 확인                                                 │
│          if (p[shortest].remaining == 0) {                           │
│              completed++;                                            │
│              p[shortest].completion = current_time;                  │
│              min_remaining = INT_MAX;                                │
│          }                                                           │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  【성능 비교】                                                           │
│  ──────────────────                                                  │
│  # FCFS, SJF, SRTF 비교 시뮬레이션                                    │
│  $ ./scheduling_simulator -compare fcfs,sjf,srtf                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
