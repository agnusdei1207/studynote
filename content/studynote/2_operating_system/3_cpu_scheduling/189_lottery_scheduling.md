# 189. 레드블랙트리 (Red-Black Tree)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 균형 이진 탐색 트리의 일종
> 2. **가치**: CFS에서 태스크를 vruntime 기준으로 정렬
> 3. **융합**: CFS, vruntime, O(log n) 연산과 연관

---

## Ⅰ. 개요

### 개념 정의

레드블랙트리(Red-Black Tree)는 **각 노드가 빨간색 또는 검은색인 균형 이진 탐색 트리**다. 삽입, 삭제, 검색 모두 O(log n) 시간을 보장한다. Linux CFS에서 태스크 관리에 사용한다.

### 💡 비유: 자동 정렬 책장
레드블랙트리는 **자동으로 정렬 상태를 유지하는 책장**과 같다. 책을 넣거나 빼도 항상 알파벳 순서가 유지되고, 원하는 책을 빠르게 찾을 수 있다.

### 레드블랙트리 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                레드블랙트리 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【레드블랙트리 속성】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 각 노드는 빨간 또는 검은색                                    │ │
│  │  2. 루트는 검은색                                                 │ │
│  │  3. 모든 리프(NIL)은 검은색                                       │ │
│  │  4. 빨간 노드의 자식은 모두 검은색 (빨간이 연속 불가)               │ │
│  │  5. 어떤 노드에서 리프까지의 검은 노드 수가 같음                    │ │
│  │                                                             │ │
│  │  → 이 속성들로 트리 균형 유지                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CFS에서의 사용】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │                    ┌─────────────┐                          │ │
│  │                    │ vruntime=50 │ (검은)                    │ │
│  │                    │    Task C   │                          │ │
│  │                    └─────────────┘                          │ │
│  │                   ╱               ╲                          │ │
│  │           ┌─────────────┐   ┌─────────────┐                 │ │
│  │           │ vruntime=30 │   │ vruntime=70 │ (빨간)           │ │
│  │           │    Task A   │   │    Task D   │                 │ │
│  │           └─────────────┘   └─────────────┘                 │ │
│  │              ╱                                                 │ │
│  │      ┌─────────────┐                                          │ │
│  │      │ vruntime=10 │ (빨간)                                   │ │
│  │      │    Task B   │ ← 최소 vruntime (가장 왼쪽)               │ │
│  │      └─────────────┘                                          │ │
│  │                                                             │ │
│  │  • 키: vruntime                                               │ │
│  │  • 값: 태스크                                                  │ │
│  │  • 최소값: 가장 왼쪽 노드                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시간 복잡도】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  연산              시간 복잡도                                   │ │
│  │  ────                        ────                                │ │
│  │  검색              O(log n)                                     │ │
│  │  삽입              O(log n)                                     │ │
│  │  삭제              O(log n)                                     │ │
│  │  최소값 찾기        O(log n)                                     │ │
│  │  순회              O(n)                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                레드블랙트리 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【리밸런싱 연산】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 회전 (Rotation)                                             │ │
│  │     - 좌회전: 오른쪽 자식을 부모로                                │ │
│  │     - 우회전: 왼쪽 자식을 부모로                                  │ │
│  │                                                             │ │
│  │  2. 재색칠 (Recoloring)                                         │ │
│  │     - 빨간 → 검은, 검은 → 빨간                                   │ │
│  │                                                             │ │
│  │  삽입/삭제 후 속성 위반 시 리밸런싱 수행                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CFS에서의 활용 이유】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 빠른 최소값 검색: 가장 낮은 vruntime 태스크 즉시 찾음           │ │
│  │  • 동적 업데이트: 태스크 추가/제거 시 O(log n)                    │ │
│  │  • 균형 보장: 최악의 경우도 O(log n) 보장                         │ │
│  │  • 효율적 순회: 모든 태스크 나열 가능                              │ │
│  │                                                             │ │
│  │  대안 비교:                                                       │ │
│  │  • 연결 리스트: O(n) 검색                                        │ │
│  │  • 힙: 삭제 후 최소값 갱신 복잡                                   │ │
│  │  • AVL 트리: 더 엄격한 균형, 더 많은 회전                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux 커널 구현】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct rb_node {                                               │ │
│  │      unsigned long __rb_parent_color;                           │ │
│  │      struct rb_node *rb_right;                                  │ │
│  │      struct rb_node *rb_left;                                   │ │
│  │  } __attribute__((aligned(sizeof(long))));                      │ │
│  │                                                             │ │
│  │  // 최하위 2비트: 색상 (나머지: 부모 포인터)                        │ │
│  │  // 2비트 활용: 4바이트 정렬 보장                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 커널 활용
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CFS 런큐의 레드블랙트리】                                             │
│  ──────────────────                                                  │
│  # 스케줄러 디버그 정보                                                │
│  $ cat /proc/sched_debug                                             │
│                                                                     │
│  # cfs_rq 섹션에서 rb_tree 정보 확인                                   │
│  $ cat /proc/sched_debug | grep -A 20 "cfs_rq\[0\]"                  │
│                                                                     │
│  # 태스크 수와 트리 깊이                                               │
│  $ cat /proc/sched_debug | grep "nr_running"                        │
│                                                                     │
│  【레드블랙트리 API (커널)】                                              │
│  ──────────────────                                                  │
│  // 삽입                                                             │
│  void rb_insert_color(struct rb_node *, struct rb_root *);           │
│                                                                     │
│  // 삭제                                                             │
│  void rb_erase(struct rb_node *, struct rb_root *);                  │
│                                                                     │
│  // 검색                                                             │
│  struct rb_node *rb_first(struct rb_root *);  // 최소                │
│  struct rb_node *rb_last(struct rb_root *);   // 최대                │
│  struct rb_node *rb_next(struct rb_node *);   // 다음                │
│  struct rb_node *rb_prev(struct rb_node *);   // 이전                │
│                                                                     │
│  【사용자 공간 레드블랙트리 라이브러리】                                   │
│  ──────────────────                                                  │
│  // C++ STL                                                          │
│  #include <map>                                                      │
│  #include <set>                                                      │
│  // 대부분 구현에서 레드블랙트리 사용                                    │
│                                                                     │
│  std::map<int, Task*> task_map;                                      │
│  task_map.insert({vruntime, task});                                  │
│  auto first = task_map.begin();  // 최소 vruntime                    │
│                                                                     │
│  【직접 구현 예시】                                                      │
│  ──────────────────                                                  │
│  typedef enum { RED, BLACK } Color;                                  │
│                                                                     │
│  typedef struct RBNode {                                             │
│      int key;                                                        │
│      Color color;                                                    │
│      struct RBNode *left, *right, *parent;                           │
│  } RBNode;                                                           │
│                                                                     │
│  RBNode* find_min(RBNode *root) {                                    │
│      while (root && root->left)                                      │
│          root = root->left;                                          │
│      return root;                                                    │
│  }                                                                   │
│                                                                     │
│  // CFS에서 최소 vruntime 태스크 찾기와 동일                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 빨간/검은 노드 규칙을 가진 균형 BST
• 속성: 5가지 규칙으로 균형 유지
• 연산: 모두 O(log n) 보장
• 용도: CFS에서 vruntime 기준 태스크 정렬
• 장점: 빠른 최소값 검색, 동적 업데이트
• 구현: Linux 커널 rb_node, rb_root
```
