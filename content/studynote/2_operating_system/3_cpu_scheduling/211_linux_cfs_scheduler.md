# 리눅스 CFS (Completely Fair Scheduler)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 실행 중인 태스크에 이상적인 하드웨어 성능을 공평하게 분배하는 것을 목표로 하며, 가상 실행 시간 (vruntime)을 기준으로 가장 적게 실행된 태스크를 먼저 선택하는 리눅스의 표준 스케줄러이다.
> 2. **가치**: 기존 O(1) 스케줄러의 복잡한 휴리스틱을 제거하고, 레드-블랙 트리 (Red-Black Tree) 데이터 구조를 도입하여 O(log n)의 효율적인 탐색 속도와 수학적으로 증명된 공정성을 제공한다.
> 3. **융합**: 가중치 기반 우선순위 제어 (Nice value), 그룹 스케줄링 (cgroups), 그리고 다중 코어 간의 부하 균등화 (Load Balancing) 엔진의 핵심 알고리즘이다.

+++

## Ⅰ. 개요 (Context & Background)

- **개념**: CFS는 "실제 CPU 시간 / (실행 가능한 태스크 수 * 가중치)" 만큼의 시간을 각 태스크에게 공평하게 할당하려고 노력한다. 태스크가 CPU를 사용한 시간(Runtime)을 가중치로 나누어 **가상 실행 시간 (vruntime)**을 계산하고, 이 vruntime이 가장 작은 태스크를 다음에 실행한다.

- **💡 비유**: CFS는 "정확한 n분의 1 정산 시스템"과 같다. 식당에서 모든 손님이 먹은 시간과 양을 초 단위로 기록하여, 가장 덜 먹은 손님에게 다음 음식을 먼저 서빙함으로써 모든 손님이 결국에는 똑같이 배부르게 만드는 시스템이다.

- **📢 섹션 요약 비유**: 주관적인 판단을 배제하고 오직 '누가 가장 적게 혜택을 받았는가'라는 숫자(vruntime)에만 기반하여 차례를 정하는 가장 민주적이고 투명한 행정관과 같습니다.

+++

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 레드-블랙 트리 (Red-Black Tree) 구조

CFS는 대기 중인 태스크들을 `vruntime` 순서로 정렬된 자가 균형 이진 탐색 트리인 레드-블랙 트리에 보관한다.

```text
  [ CFS Runqueue - Red-Black Tree ]
  
             (vruntime=50)
             /           \
      (vruntime=30)     (vruntime=80)
       /      \           /      \
    [T1:10]  [T2:40]    [T3:70]  [T4:90]
      ^
      |
  [ leftmost ] <-- 다음에 실행될 태스크 (vruntime이 가장 작음)
```

**[다이어그램 해설]** 
1. **Leftmost**: 트리의 가장 왼쪽 단말 노드는 항상 `vruntime`이 최소인 태스크다. 스케줄러는 O(1) 시간 내에 이 태스크를 꺼낼 수 있다 (캐싱된 경우).
2. **트리 갱신**: 현재 태스크가 실행을 멈추면 `vruntime`을 증가시켜 다시 트리에 삽입한다. 이때 삽입 복잡도는 O(log n)이다.
3. **vruntime**: 실제 실행 시간 (wall time)에 비례하며, Nice 값이 낮을수록 (우선순위가 높을수록) 천천히 증가하여 더 긴 CPU 점유 시간을 확보하게 된다.

### Nice 값과 가중치 (Weights)
CFS는 우선순위(Nice -20 ~ +19)를 가중치로 변환한다. Nice 값이 1 변할 때마다 CPU 할당량이 약 10%씩 차이나도록 설계되어 있다.

- **📢 섹션 요약 비유**: 키가 작은 아이(높은 우선순위)는 시계가 천천히 가도록 설정(vruntime 가중치)하여, 다른 아이들과 똑같은 시각을 가질 때까지 더 오랫동안 놀 수 있게 배려하는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### O(1) 스케줄러 vs CFS

| 비교 항목 | O(1) 스케줄러 | CFS (Completely Fair) |
|:---|:---|:---|
| **데이터 구조** | 비트맵 + 링크드 리스트 (140개) | 레드-블랙 트리 |
| **선택 기준** | 정적 우선순위 + 보너스/패널티 | 가상 실행 시간 (vruntime) |
| **반응성 조절** | 복잡한 수면 시간 예측 알고리즘 | 공정성 원칙에 따른 자연스러운 응답 |
| **시간 할당량** | 고정된 Time Slice 사용 | 가변적 (태스크 수에 따라 동적 결정) |

- **📢 섹션 요약 비유**: O(1)은 100개의 바구니를 미리 준비해두고 골라 담는 것이라면, CFS는 하나의 거대한 저울(트리)을 이용해 무게가 가장 가벼운 것부터 차례로 들어 올리는 것입니다.

+++

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: CPU 집약적 작업과 대화형 작업의 공존
- **상황**: 컴파일(CPU 집약) 중 마우스를 움직이면(대화형), CFS는 마우스 프로세스가 오랫동안 잠들어 있어 `vruntime`이 낮게 유지된 것을 확인하고 즉시 선점(Preemption)을 허용한다.
- **판단**: 개발자는 `sched_latency_ns` (스케줄링 주기)와 `sched_min_granularity_ns` (최소 점유 시간) 파라미터를 튜닝하여, 대화형 반응성을 높일지 아니면 문맥 교환을 줄여 전체 처리량을 높일지 결정해야 한다. 서버 환경에서는 주기를 늘려 처리량을, 데스크탑 환경에서는 줄여서 반응성을 챙기는 것이 정석이다.

- **📢 섹션 요약 비유**: 한 명에게 최소 1분(최소 입자)은 말을 시켜줘야 대화가 끊기지 않지만, 너무 오래 기다리게 하면(지연 시간) 손님들이 짜증을 내는 것과 같은 이치입니다.

+++

## Ⅴ. 기대효과 및 결론

- **기대효과**: 리눅스 시스템의 안정성과 공정성을 획기적으로 높였으며, 수천 개의 컨테이너가 실행되는 클라우드 환경에서도 예측 가능한 성능을 보장한다.
- **미래 전망**: 최근에는 EEVDF (Earliest Eligible Virtual Deadline First) 알고리즘이 CFS를 대체할 후보로 거론되고 있는데, 이는 공정성뿐만 아니라 '마감 시간' 개념을 더 정교하게 결합하여 지연 시간을 더욱 줄이려는 시도다.

- **📢 섹션 요약 비유**: 완벽하게 공정한 저울(CFS)을 넘어, 이제는 배고픈 사람(마감 시간)에게 더 빨리 음식을 내주는 똑똑한 저울(EEVDF)의 시대로 가고 있습니다.

## 👶 어린이를 위한 3줄 비유 설명
1. 컴퓨터가 여러 가지 일을 할 때, "누가 제일 적게 일했지?"를 찾아서 그 일을 먼저 시켜주는 아주 착하고 공평한 반장님이에요.
2. 'vruntime'이라는 특별한 시계를 가지고 있어서, 힘든 일을 하는 친구의 시간은 조금 천천히 가게 해서 도와준답니다.
3. 이렇게 하면 모든 일이 골고루 처리되어 컴퓨터가 멈추지 않고 기분 좋게 움직일 수 있어요!
