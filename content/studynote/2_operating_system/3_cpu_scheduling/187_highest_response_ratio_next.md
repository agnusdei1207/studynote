# 187. 가상 런타임 (vruntime)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가중치를 적용한 정규화된 실행 시간
> 2. **가치**: CFS에서 공정한 CPU 분배의 핵심
> 3. **융합**: CFS, nice 값, 레드블랙트리와 연관

---

## Ⅰ. 개요

### 개념 정의

가상 런타임(vruntime, Virtual Runtime)은 **각 태스크의 실제 실행 시간에 가중치를 적용하여 정규화한 값**이다. CFS에서 가장 적게 실행된 태스크를 선택하기 위해 사용한다.

### 💡 비유: 보정된 달리기 기록
vruntime은 **보정된 달리기 기록**과 같다. 다리가 짧은 사람은 실제 시간보다 적게 기록하고, 다리가 긴 사람은 더 많이 기록하여 공정하게 비교한다.

### vruntime 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                vruntime 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【vruntime 계산】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  vruntime += 실제 실행 시간 × (1024 / 태스크 가중치)            │ │
│  │                                                             │ │
│  │  1024: 기본 가중치 (nice 0)                                     │ │
│  │                                                             │ │
│  │  예시:                                                         │ │
│  │  • nice 0 (가중치 1024): 10ms 실행 → vruntime += 10ms           │ │
│  │  • nice -5 (가중치 ~3072): 10ms 실행 → vruntime += ~3.3ms       │ │
│  │  • nice +5 (가중치 ~341): 10ms 실행 → vruntime += ~30ms         │ │
│  │                                                             │ │
│  │  → 높은 우선순위는 vruntime이 천천히 증가                        │ │
│  │  → 낮은 우선순위는 vruntime이 빨리 증가                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스케줄링 결정】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  레드블랙트리에서 가장 낮은 vruntime을 가진 태스크 선택           │ │
│  │                                                             │ │
│  │  태스크        vruntime        실제 실행 시간                    │ │
│  │  ────              ────                ────                      │ │
│  │  A (nice 0)       50ms            50ms                         │ │
│  │  B (nice 0)       30ms            30ms    ← 선택됨              │ │
│  │  C (nice -5)      20ms            60ms                         │ │
│  │                                                             │ │
│  │  → B가 가장 낮은 vruntime → B 실행                              │ │
│  │  → C는 높은 우선순위라 실제로 더 실행했지만 vruntime은 낮음        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【공정성 의미】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  "공정" = 모든 태스크의 vruntime이 비슷하게 유지됨                │ │
│  │                                                             │ │
│  │  이상적: 모든 태스크의 vruntime이 같음                          │ │
│  │  현실: vruntime 차이가 sched_latency 이내                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                vruntime 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【nice 값과 가중치 매핑】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  nice 값          가중치            vruntime 증가율              │ │
│  │  ────                  ────                  ────                 │ │
│  │  -20 (높음)       88761           ~1.2% (매우 느림)              │ │
│  │  -10              26212           ~4%                            │ │
│  │    0 (기본)       1024            ~100% (기준)                   │ │
│  │  +10              40              ~2500%                         │ │
│  │  +19 (낮음)       15              ~6800%                         │ │
│  │                                                             │ │
│  │  nice -1마다 CPU 시간 약 1.25배 (가중치 1.25배)                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vruntime 특성】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 단조 증가: 실행할수록 계속 증가만 함                           │ │
│  │  • 정규화: 서로 다른 우선순위를 공정하게 비교                      │ │
│  │  • 정밀도: 나노초 단위                                          │ │
│  │  • 랩핑 방지: 64비트 값으로 사실상 무한                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【sleep 후 vruntime 조정】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제: 오래 sleep한 태스크가 깨면 vruntime이 너무 높음?           │ │
│  │                                                             │ │
│  │  해결: min_vruntime으로 조정                                    │ │
│  │  • 깨어날 때: vruntime = max(자신의 vruntime, min_vruntime)      │ │
│  │  • 너무 뒤쳐지지 않도록 보장                                      │ │
│  │  • 반면, I/O 바운드 태스크 보호                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 확인 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【태스크 vruntime 확인】                                               │
│  ──────────────────                                                  │
│  # 특정 태스크의 스케줄링 정보                                          │
│  $ cat /proc/<pid>/sched | grep vruntime                            │
│  se.vruntime                          : 123456789                   │
│                                                                     │
│  # 전체 스케줄링 정보                                                  │
│  $ cat /proc/<pid>/sched                                             │
│                                                                     │
│  # 주요 필드                                                         │
│  # se.exec_start          : 실행 시작 시간                           │
│  # se.sum_exec_runtime    : 총 실행 시간                             │
│  # se.vruntime            : 가상 런타임                               │
│  # se.load.weight         : 가중치 (nice 기반)                        │
│                                                                     │
│  【런큐 min_vruntime 확인】                                              │
│  ──────────────────                                                  │
│  # CPU별 런큐 정보                                                    │
│  $ cat /proc/sched_debug | grep -A 5 "min_vruntime"                 │
│                                                                     │
│  # min_vruntime: 새 태스크의 기준점                                    │
│                                                                     │
│  【vruntime 기반 스케줄링 시뮬레이션】                                    │
│  ──────────────────                                                  │
│  #include <stdio.h>                                                  │
│                                                                     │
│  // 가중치 테이블 (간소화)                                              │
│  int weight_table[] = {88761, ..., 1024, ..., 15};                   │
│                                                                     │
│  long calc_vruntime_increment(long runtime, int nice) {              │
│      int weight = weight_table[nice + 20];                           │
│      return runtime * 1024 / weight;                                 │
│  }                                                                   │
│                                                                     │
│  int main() {                                                        │
│      long runtime = 10000000;  // 10ms                               │
│      printf("nice -5: +%ld\n", calc_vruntime_increment(runtime, -5));│
│      printf("nice  0: +%ld\n", calc_vruntime_increment(runtime, 0)); │
│      printf("nice +5: +%ld\n", calc_vruntime_increment(runtime, 5)); │
│      return 0;                                                       │
│  }                                                                   │
│                                                                     │
│  【vruntime 모니터링 스크립트】                                          │
│  ──────────────────                                                  │
│  #!/bin/bash                                                         │
│  # vruntime 모니터링                                                  │
│  while true; do                                                      │
│      for pid in $(pgrep -f "my_program"); do                         │
│          vruntime=$(grep se.vruntime /proc/$pid/sched | awk '{print $3}')│
│          echo "PID $pid: vruntime = $vruntime"                       │
│      done                                                            │
│      sleep 1                                                         │
│  done                                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 가중치 적용된 정규화 실행 시간
• 공식: vruntime += 실제시간 × (1024/가중치)
• 스케줄링: 가장 낮은 vruntime 선택
• nice 영향: 낮은 nice = 느린 증가 = 더 많은 CPU
• 공정성: 모든 태스크 vruntime 비슷하게 유지
• 조정: sleep 후 min_vruntime으로 보정
```
