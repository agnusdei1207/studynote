# 171. CPU 이용률 (CPU Utilization)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 실제 작업을 수행하는 시간의 비율
> 2. **가치**: 시스템 자원 효율성의 핵심 지표
> 3. **융합**: 처리량, 문맥 교환, I/O 바운드와 연관

---

## Ⅰ. 개요

### 개념 정의

CPU 이용률(CPU Utilization)은 **전체 시간 중 CPU가 실제로 작업을 수행한 시간의 비율**이다. 시스템 효율성의 가장 기본적인 지표다.

### 💡 비유: 공장 가동률
CPU 이용률은 **공장 가동률**과 같다. 공장이 멈추지 않고 계속 생산하면 가동률이 높다. 유휴 시간이 많으면 가동률이 낮다.

├─────────────────────────────────────────────────────────────────────┤

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │              CPU가 작업한 시간                                │ │
│  │  CPU 이용률 = ───────────────────── × 100%                    │ │
│  │              전체 시간                                        │ │
│  │                                                             │ │
│  │  범위: 0% ~ 100%                                             │ │
│  │  이상적: 40% ~ 90% (시스템에 따라 다름)                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CPU 시간 분해】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  전체 CPU 시간                                                │ │
│  │  ┌──────────────────────────────────────────────────────┐   │ │
│  │  │                User          │     System    │ Idle │   │ │
│  │  │ 사용자 모드 실행              │ 커널 모드      │ 대기  │   │ │
│  │  │ ─────────────────────────────│───────────────│────── │   │ │
│  │  │ 응용 프로그램 코드            │ 시스템 콜     │       │   │ │
│  │  │                             │ 인터럽트      │       │   │ │
│  │  │                             │ I/O 처리      │       │   │
│  │  └──────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  │  CPU 이용률 = User + System / 전체                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【이용률 수준별 해석】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  수준              상태                    조치                  │ │
│  │  ────                    ────                        ────        │ │
│  │  0-30%            자원 낭비              작업 추가               │ │
│  │  30-70%           적절                   유지                    │ │
│  │  70-90%           높은 효율              모니터링                │ │
│  │  90-100%          병목 가능              CPU 추가/최적화         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅲ. 실무 적용

### 측정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux CPU 이용률 확인】                                               │
│  ──────────────────                                                  │
│  # 실시간 CPU 사용률                                                  │
│  $ top                                                              │
│  # %Cpu(s): us, sy, ni, id, wa, hi, si, st                         │
│                                                                     │
│  # CPU 통계                                                          │
│  $ mpstat 1                                                         │
│  # %usr: 사용자, %sys: 커널, %iowait: I/O 대기, %idle: 유휴          │
│                                                                     │
│  # 전체 CPU 이용률                                                    │
│  $ vmstat 1                                                         │
│  # cpu - us, sy, id 컬럼                                            │
│                                                                     │
│  【CPU 이용률 계산】                                                     │
│  ──────────────────                                                  │
│  # /proc/stat에서 계산                                               │
│  $ cat /proc/stat | head -1                                         │
│  cpu  user nice system idle iowait irq softirq steal               │
│                                                                     │
│  # 이용률 = 1 - (idle / total) × 100                                │
│                                                                     │
│  # sar 명령어                                                        │
│  $ sar -u 1 5                                                       │
│  # %user, %system, %iowait, %idle                                  │
│                                                                     │
│  【프로세스별 CPU 사용】                                                 │
│  ──────────────────                                                  │
│  # CPU 사용률 높은 프로세스                                             │
│  $ top -o %CPU                                                      │
│  $ htop                                                             │
│                                                                     │
│  # 특정 프로세스                                                       │
│  $ pidstat -p <pid> 1                                               │
│                                                                     │
│  【CPU 이용률 모니터링 스크립트】                                          │
│  ──────────────────                                                  │
│  #!/bin/bash                                                         │
│  # CPU 이용률 80% 초과 시 알림                                         │
│  while true; do                                                      │
│      CPU_IDLE=$(top -bn1 | grep "Cpu(s)" | awk '{print $8}')         │
│      CPU_USAGE=$(echo "100 - $CPU_IDLE" | bc)                        │
│      if (( $(echo "$CPU_USAGE > 80" | bc -l) )); then                │
│          echo "High CPU: $CPU_USAGE%"                                │
│      fi                                                              │
│      sleep 5                                                         │
│  done                                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: CPU 작업 시간 / 전체 시간 × 100%
• 구성: User + System / 전체
• 범위: 0% ~ 100%
• 이상적: 40-90% (시스템별 상이)
• 높은 이용률: 병목 가능성
• 낮은 이용률: 자원 낭비
```
