# 216. 인터럽트 스케줄링 (Interrupt Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 처리를 위한 CPU 시간 할당
> 2. **가치**: 시스템 응답성, 실시간 보장
> 3. **융합**: IRQ, 소프트irq, 인터럽트 스레드와 연관

---

## Ⅰ. 개요

### 개념 정의

인터럽트 스케줄링(Interrupt Scheduling)은 **인터럽트 처리가 CPU 시간을 언제, 얼마나 사용할지를 관리하는 메커니즘**이다. 하드웨어 인터럽트, 소프트IRQ, 인터럽트 스레드를 포함한다.

### 💡 비유: 응급실 우선
인터럽트 스케줄링은 **응급실 환자 처리**와 같다. 응급 환자는 바로 보고, 덜 급한 건 나중에 본다.

### 인터럽트 스케줍링 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 스케줄링 구조                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 처리 계층】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  하드웨어 인터럽트 (Hard IRQ)                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • CPU 즉시 선점                                          │ │ │
│  │  │  • 최소 작업만 수행                                        │ │ │
│  │  │  • 인터럽트 비활성화                                       │ │ │
│  │  │  • 빠른 응답 필수                                          │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                        ↓                                    │ │
│  │  소프트IRQ (SoftIRQ)                                            │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 인터럽트 활성화 상태                                    │ │ │
│  │  │  • 지연된 처리 수행                                        │ │ │
│  │  │  • NET_RX, TIMER, TASKLET 등                             │ │ │
│  │  │  • 우선순위 있음                                           │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                        ↓                                    │ │
│  │  인터럽트 스레드 (Threaded IRQ)                                 │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 일반 태스크처럼 스케줄됨                                 │ │ │
│  │  │  • 실시간 우선순위 가능                                    │ │ │
│  │  │  • PREEMPT_RT에서 사용                                    │ │ │
│  │  │  • RT 스케줄러로 관리                                      │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 유형별 스케줄링】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  유형             스케줄링 방식           우선순위              │ │
│  │  ────                   ────                    ────            │ │
│  │  Hard IRQ         즉시, 비선점           최우선                │ │
│  │  SoftIRQ          컨텍스트 허용           높음                  │ │
│  │  Tasklet          SoftIRQ 기반            중간                  │ │
│  │  Workqueue        태스크 컨텍스트          낮음                  │ │
│  │  Threaded IRQ     RT 스케줄러             설정 가능             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【IRQ Affinity】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 특정 CPU에서 인터럽트 처리                                   │ │
│  │  • /proc/irq/<irq>/smp_affinity                               │ │
│  │  • irqbalance 데몬                                             │ │
│  │  • NUMA 고려                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 설정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 통계】                                                     │
│  ──────────────────                                                  │
│  # 인터럽트 발생 횟수                                                  │
│  $ cat /proc/interrupts                                             │
│                                                                     │
│  # SoftIRQ 통계                                                      │
│  $ cat /proc/softirqs                                               │
│                                                                     │
│  【IRQ Affinity 설정】                                                │
│  ──────────────────                                                  │
│  # 특정 IRQ의 현재 affinity                                           │
│  $ cat /proc/irq/24/smp_affinity                                    │
│  000f                                                               │
│                                                                     │
│  # 특정 CPU로 설정                                                   │
│  $ echo 02 > /proc/irq/24/smp_affinity    # CPU 1만                │
│                                                                     │
│  # 모든 IRQ 확인                                                      │
│  $ for i in /proc/irq/*/smp_affinity; do                           │
│        echo "$i: $(cat $i)"                                        │
│    done                                                             │
│                                                                     │
│  【irqbalance 설정】                                                  │
│  ──────────────────                                                  │
│  # irqbalance 상태                                                   │
│  $ systemctl status irqbalance                                      │
│                                                                     │
│  # irqbalance 설정                                                   │
│  $ cat /etc/default/irqbalance                                      │
│                                                                     │
│  # 특정 IRQ 제외                                                     │
│  # IRQBALANCE_BANNED_IRQS="24,25"                                   │
│                                                                     │
│  【인터럽트 스레드 우선순위】                                          │
│  ──────────────────                                                  │
│  # 인터럽트 스레드 확인 (RT 커널)                                      │
│  $ ps -eLo pid,tid,class,rtprio,comm | grep irq                    │
│                                                                     │
│  # 우선순위 변경                                                      │
│  $ chrt -f -p 50 $(pidof irq/24-eth0)                              │
│                                                                     │
│  【인터럽트 추적】                                                     │
│  ──────────────────                                                  │
│  # ftrace로 인터럽트 추적                                             │
│  $ echo irq > /sys/kernel/debug/tracing/current_tracer            │
│  $ cat /sys/kernel/debug/tracing/trace                             │
│                                                                     │
│  # perf로 인터럽트 분석                                               │
│  $ perf record -e irq:* ./program                                  │
│  $ perf report                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 인터럽트 처리 CPU 시간 관리
• 계층: Hard IRQ → SoftIRQ → Threaded IRQ
• SoftIRQ: 지연된 처리, 우선순위 있음
• Threaded IRQ: 스케줄 가능, RT 커널
• IRQ Affinity: CPU 지정 처리
• 설정: /proc/irq/*/smp_affinity
```
