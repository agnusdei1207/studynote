# 201. WCET (최악 실행 시간, Worst-Case Execution Time)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 태스크가 완료까지 걸릴 수 있는 최대 시간
> 2. **가치**: 실시간 시스템 스케줄링 분석의 필수 입력
> 3. **융합**: 데드라인, 실시간 스케줄링, 스케줄 가능성 분석과 연관

---

## Ⅰ. 개요

### 개념 정의

WCET(Worst-Case Execution Time)는 **태스크가 어떤 상황에서도 완료까지 걸리는 최대 시간**이다. 실시간 시스템에서 데드라인 보장을 위해 필수적이다.

### 💡 비유: 통근 시간
WCET는 **최악의 통근 시간**과 같다. 평소 30분이지만, 비 오고 사고 나면 2시간 걸릴 수 있다. 2시간이 WCET다.

### WCET 구조

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  실행 시간                                                       │ │
│  │    ▲                                                           │ │
│  │    │     ┌────┐                                                │ │
│  │    │    ╱    ╲                                               │ │

│  │    └──────────────────────────────────────────────►          │ │
│  │     BCET    평균      최빈값     WCET                         │ │
│  │                                                             │ │
│  │  BCET: 최선 실행 시간 (Best-Case)                               │ │
│  │  WCET: 최악 실행 시간 (Worst-Case)                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【WCET 영향 요소】                                                      │
│  ──────────────────                                                  │

│  │  인터럽트               인터럽트 처리 시간                        │ │
│  │  선점                   다른 태스크에 의한 선점                   │ │
│  │  메모리 접근            TLB 미스, 페이지 폴트                     │ │

│  │                                                             │ │

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                WCET 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【WCET 분석 방법】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 정적 분석 (Static Analysis)                                  │ │
│  │     • 코드 분석으로 WCET 계산                                     │ │
│  │     • 도구: aiT, OTAWA, SWEET                                   │ │
│  │     • 안전하지만 과대평가 가능                                    │ │
│  │                                                             │ │
│  │  2. 측정 기반 (Measurement-Based)                                │ │
│  │     • 실제 실행으로 측정                                          │ │
│  │     • 최대값을 WCET로 사용                                        │ │
│  │     • 진짜 최악 케이스 놓칠 수 있음                                │ │
│  │                                                             │ │
│  │  3. 하이브리드                                                    │ │
│  │     • 정적 분석 + 측정 결합                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【WCET 분석 시 고려사항】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 루프 최대 반복 횟수                                            │ │
│  │  • 재귀 호출 깊이                                                 │ │
│  │  • 분기 경로                                                      │ │
│  │  • 함수 호출                                                      │ │
│  │  • 하드웨어 특성 (캐시, 파이프라인)                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【안전 마진 (Safety Margin)】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  WCET_analyzed × (1 + margin) = WCET_final                       │ │
│  │                                                             │ │
│  │  일반적 마진: 10-50%                                              │ │
│  │  • 측정 기반: 더 큰 마진 필요                                      │ │
│  │  • 정적 분석: 작은 마진 가능                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 측정 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【WCET 측정 코드】                                                      │
│  ──────────────────                                                  │
│  #include <time.h>                                                   │
│  #include <stdio.h>                                                  │
│                                                                     │
│  // 고해상도 타이머로 측정                                              │
│  long long measure_execution_time(void (*func)(), int iterations) {  │
│      struct timespec start, end;                                     │
│      long long max_time = 0;                                         │
│                                                                     │
│      for (int i = 0; i < iterations; i++) {                          │
│          // 캐시 워밍업 방지                                          │
│          __builtin___clear_cache();                                  │
│                                                                     │
│          clock_gettime(CLOCK_MONOTONIC, &start);                     │
│          func();                                                     │
│          clock_gettime(CLOCK_MONOTONIC, &end);                       │
│                                                                     │
│          long long elapsed =                                         │
│              (end.tv_sec - start.tv_sec) * 1000000000LL +            │
│              (end.tv_nsec - start.tv_nsec);                          │
│                                                                     │
│          if (elapsed > max_time)                                     │
│              max_time = elapsed;                                     │
│      }                                                               │
│      return max_time;                                                │
│  }                                                                   │
│                                                                     │
│  // WCET 추정 (안전 마진 적용)                                         │
│  long long estimate_wcet(long long measured_max, double margin) {    │
│      return measured_max * (1 + margin);                             │
│  }                                                                   │
│                                                                     │
│  【Linux perf로 분석】                                                  │
│  ──────────────────                                                  │
│  # 실행 시간 측정                                                      │
│  $ perf stat -e task-clock,cycles,instructions ./realtime_task       │
│                                                                     │
│  # 캐시 미스 분석                                                      │
│  $ perf stat -e cache-misses,cache-references ./realtime_task       │
│                                                                     │
│  # 분기 예측 실패                                                      │
│  $ perf stat -e branch-misses,branches ./realtime_task              │
│                                                                     │
│  【정적 분석 도구】                                                      │
│  ──────────────────                                                  │
│  # 오픈소스 도구                                                       │
│  # OTAWA (http://www.otawa.fr/)                                      │
│  # SWEET (https://www.mrtc.mdh.se/projects/wcet/)                    │
│  # Platin (https://github.com/tum-ei-esi/platin)                     │
│                                                                     │
│  【WCET 최적화】                                                          │
│  ──────────────────                                                  │
│  • 루프 언롤링                                                         │
│  • 분기 최소화                                                         │
│  • 캐시 친화적 코드                                                    │
│  • 인라인 함수                                                         │
│  • 동적 메모리 할당 회피                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 최악의 경우 실행 시간
• 분석: 정적 분석, 측정 기반, 하이브리드

• 용도: 스케줄 가능성 분석 입력
• 안전 마진: 10-50% 추가
• 최적화: 루프 언롤링, 분기 최소화
```
