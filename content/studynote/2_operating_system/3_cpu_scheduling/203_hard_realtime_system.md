# 경성 실시간 (Hard Real-time) 시스템

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 작업의 마감 시간 (Deadline) 준수가 시스템의 논리적 정확성을 결정하며, 단 한 번의 마감 시간 위반도 시스템의 완전한 파괴나 인명 사고 등 치명적인 결과(Catastrophic Failure)를 초래하는 엄격한 실시간 시스템이다.
> 2. **가치**: 어떤 최악의 상황 (Worst-case)에서도 예측 가능한 응답 시간 (Determinism)을 하드웨어와 소프트웨어 수준에서 100% 보장하여, 생명 유지 및 안전 필수 분야의 신뢰성을 담보한다.
> 3. **융합**: RTOS (Real-Time OS), 정적 우선순위 스케줄링 (RMS), 동적 우선순위 스케줄링 (EDF), 그리고 하드웨어의 인터럽트 지연 시간 통제 기술이 집약된다.

+++

## Ⅰ. 개요 (Context & Background)

- **개념**: 경성 실시간 시스템은 "정확한 결과물"뿐만 아니라 "정확한 시점"에 결과가 나오는 것이 필수적인 시스템이다. 시간적 제약 조건이 논리적 명세의 일부이며, 이를 위반하는 것은 곧 버그가 아니라 시스템의 붕괴를 의미한다.

- **💡 비유**: 경성 실시간은 "자동차의 에어백 시스템"과 같다. 사고가 발생하고 0.03초(마감 시간) 안에 터지지 않고, 1초 뒤에 터지는 에어백은 아무런 가치가 없을 뿐만 아니라 탑승자의 생명을 구하지 못하는 시스템 실패다.

- **📢 섹션 요약 비유**: 1초의 오차도 허용하지 않는 시한폭탄 해체 작업처럼, 정확한 타이밍이 곧 생존인 절박한 제어 환경과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 경성 실시간 시스템의 가치 함수 (Value Function)

경성 실시간에서 시간은 단순한 성능 지표가 아니라 생존 조건이다.

```text
  가치 (Value)
   ^
  1|----------| (Deadline)
   |          |
   |          |
   |          |
  0+----------+------------> 시간 (Time)
  -무한대      | (Immediate Failure)
```

**[다이어그램 해설]** 마감 시간 전까지는 최고의 가치를 유지하다가, 마감 시간을 단 1나노초라도 넘기는 순간 시스템의 가치는 급격히 수직 하강하여 마이너스(시스템 실패)가 된다. 따라서 평균 응답 시간보다 **최악 실행 시간 (WCET, Worst-Case Execution Time)** 분석이 가장 중요하다.

### 결정성 (Determinism) 확보를 위한 설계 원칙
1. **예측 가능한 스케줄링**: 우선순위가 높은 태스크는 반드시 정해진 시간 내에 CPU를 점유함을 수학적으로 증명(Schedulability Analysis)해야 한다.
2. **인터럽트 지연 통제**: 커널이 인터럽트를 금지하는 시간(Critical Section)을 극도로 짧게 유지하여, 외부 이벤트에 즉각 대응해야 한다.
3. **자원 경합 방지**: 우선순위 역전 (Priority Inversion) 방지 프로토콜을 반드시 적용하여 하위 태스크가 상위 태스크를 막지 못하게 한다.
4. **가상 메모리 지연 제거**: 페이지 폴트 (Page Fault)로 인한 불확실한 지연을 막기 위해 메모리 락킹 (Memory Locking)을 사용하여 모든 실시간 코드를 물리 메모리에 상주시킨다.

- **📢 섹션 요약 비유**: 모든 신호등이 구급차의 도착 시간에 맞춰 미리 초록불로 바뀌도록 설계된, 한 치의 오차도 없는 VIP 전용 이송 시스템과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 경성 실시간 vs 연성 실시간 vs 일반 시스템

| 비교 항목 | 경성 실시간 (Hard RT) | 연성 실시간 (Soft RT) | 일반 시스템 (Best-effort) |
|:---|:---|:---|:---|
| **마감 시간 준수** | 필수 (100%) | 목표 (최대한 노력) | 고려 안 함 |
| **실패의 결과** | 재앙 (Catastrophe) | 서비스 품질 저하 | 불편함 |
| **성능 지표** | 예측 가능성 (Predictability) | 평균 처리량 (Throughput) | 사용자 편의성 |
| **주요 운영체제** | VxWorks, QNX, RT-Linux | Linux, Windows, macOS | Windows, macOS |

- **📢 섹션 요약 비유**: 경성 실시간은 '낙하산'이고, 연성 실시간은 '우편물'이며, 일반 시스템은 '인터넷 서핑'입니다. 낙하산은 안 펴지면 끝이지만, 편지는 늦게 와도 되고, 인터넷은 느려도 다시 시도하면 됩니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 자율주행차의 장애물 급제동
- **상황**: 라이다 (LiDAR) 센서가 장애물을 감지하고 제동 명령을 내리는 태스크가 다른 데이터 로깅 태스크에 밀려 마감 시간을 놓침.
- **판단**: 제동 태스크는 최고 우선순위의 경성 실시간 태스크로 설정되어야 하며, 로깅 태스크가 공유 자원(버스 등)을 점유하고 있을 때 발생할 수 있는 지연 시간을 엄격히 계산 (WCET 분석)해야 한다. 만약 수학적으로 마감 시간 준수가 보장되지 않는다면 하드웨어 사양을 높이거나 로직을 단순화하여 '결정성'을 확보해야 한다.

- **📢 섹션 요약 비유**: 브레이크를 밟아야 하는 순간에 블랙박스 영상을 저장하느라 머뭇거리는 것은 자율주행차에게 허용되지 않는 치명적 결함입니다.

+++

## Ⅴ. 기대효과 및 결론

- **기대효과**: 기계적 제어의 신뢰성을 극대화하여 인간의 생명과 고가의 장비를 안전하게 보호한다.
- **미래 전망**: 로봇 수술, 스마트 팩토리, 도심 항공 모빌리티 (UAM) 등 초정밀 제어가 필요한 분야가 확대됨에 따라, 복잡한 연산을 수행하면서도 경성 실시간성을 유지하는 고성능 RTOS와 가상화 기술(Partitioning)이 핵심 산업이 될 것이다.

- **📢 섹션 요약 비유**: 세상이 아무리 복잡해져도 절대 변하지 말아야 할 '약속'을 하드웨어적으로 지켜내는 것이 경성 실시간의 가치입니다.

+++

## 📌 관련 개념 맵 (Knowledge Graph)
- **최악 실행 시간 (WCET)**: 경성 실시간 설계의 기초 데이터
- **우선순위 상속 프로토콜 (PIP)**: 지연 시간을 예측 가능하게 만드는 동기화 기술
- **RTOS (Real-Time Operating System)**: 경성 실시간 시스템을 위한 전용 OS

## 👶 어린이를 위한 3줄 비유 설명
1. 자동차 에어백처럼, 아주 위험한 순간에 1초도 늦지 않고 "당장!" 작동해야 하는 시스템이에요.
2. 약속 시간을 단 1초만 어겨도 아주 큰 사고가 날 수 있기 때문에, 컴퓨터가 다른 일은 다 제쳐두고 이 일만은 꼭 제시간에 끝내도록 약속되어 있어요.
3. 우리를 안전하게 지켜주기 위해 세상에서 가장 약속을 잘 지키는 컴퓨터 친구라고 생각하면 돼요!
