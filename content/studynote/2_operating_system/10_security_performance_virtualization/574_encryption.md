# 암호화 (Encryption)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 암호화(Encryption)는 평문을 알고리즘과 키를 통해 **읽을 수 없는 암호문(Ciphertext)으로 변환**하는 기술로, 기밍성을 보장하는 보안의 핵심이다.
> 2. **가치**: 데이터 유출 시에도 내용을 알 수 없게 하여 **개인정보 보호, 기업 비밀 보호, 통신 보안**을 실현하며, 규정 준수(GDPR, 개인정보보호법)에 필수적이다.
> 3. **융합**: 대칭키(AES)와 공개키(RSA, ECC) 방식이 상호 보완적으로 사용되며, TLS/SSL, VPN, 디스크 암호화(BitLocker, LUKS) 등 다양한 계층에서 적용된다.

+++

## Ⅰ. 암호화의 개요

### 1. 정의
- 암호화는 평문(Plaintext)을 암호키와 암호 알고리즘을 사용하여 암호문(Ciphertext)으로 변환하는 과정이다.
- **복호화(Decryption)**: 암호문을 다시 평문으로 변환

### 2. 등장 배경: 정보 보호의 필요성
- 전쟁 중 기밀 정보 보호에서 시작
- 디지털 시대의 데이터 유출, 도청 위험 방지

### 3. 💡 비유: '자물쇠와 열쇠'
- 암호화는 **'편지를 자물쇠가 달린 상자에 넣는 것'**과 같다.
- 열쇠(키) 없이는 상자를 열어 내용을 볼 수 없다.

- **📢 섹션 요약 비유**: 누구나 읽을 수 있는 편지(평문)를 암호로 써서, 열쇠 가진 사람만 읽을 수 있게 만드는 비밀 언어입니다.

## Ⅱ. 암호화의 기본 원리 (Deep Dive)

### 1. 암호 시스템의 구성 요소
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                  암호 시스템의 구성 요소                          │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [Plaintext] 평문                                               │
    │   → 암호화할 원본 데이터                                         │
    │                                                                 │
    │  [Ciphertext] 암호문                                            │
    │   → 암호화된 결과 (무의미한 데이터)                              │
    │                                                                 │
    │  [Encryption Algorithm] 암호 알고리즘                            │
    │   → 평문을 암호문으로 변환하는 수학적 함수                        │
    │                                                                 │
    │  [Decryption Algorithm] 복호화 알고리즘                           │
    │   → 암호문을 평문으로 변환하는 역함수                            │
    │                                                                 │
    │  [Key] 암호키                                                    │
    │   → 암호/복호화에 필요한 비밀 값                                 │
    │   → 키의 비밀성이 보안의 핵심                                    │
    │                                                                 │
    │  [Kerckhoffs의 원칙]                                            │
    │   → "알고리즘은 공개되어야 하고, 키만 비밀이어야 한다"           │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 암호화의 목표
| 목표 | 설명 | 예시 |
|:---|:---|:---|
| **기밀성 (Confidentiality)** | unauthorized 읽기 방지 | 이메일 암호화 |
| **무결성 (Integrity)** | unauthorized 수정 감지 | HMAC, Digital Signature |
| **인증 (Authentication)** | 메시지 출처 확인 | Digital Signature |
| **부인 방지 (Non-repudiation)** | 송수신 사실 증명 | Digital Signature |

## Ⅲ. 암호화 알고리즘 분류

### 1. 대칭키 암호화 (Symmetric Key)
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │               대칭키 암호화 (Symmetric Key)                      │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   송신자                              수신자                     │
    │      │                                   │                       │
    │      ├── 평문 "Hello" ────┐              │                       │
    │      │                    │              │                       │
    │      │              [암호화]             │                       │
    │      │                    │              │                       │
    │      │           암호문 "Xy7$"           │                       │
    │      │                    │              │                       │
    │   (공유 키 K) ══════════════════════════(공유 키 K)              │
    │      │                    │              │                       │
    │      │              [복호화]             │                       │
    │      │                    │              │                       │
    │      └───────── 평문 "Hello" ◀───────────┘                       │
    │                                                                 │
    │  [특징]                                                         │
    │   - 암호화/복호화에 동일한 키 사용                               │
    │   - 속도 빠름 (대용량 데이터 적합)                               │
    │   - 키 배송 문제 (Key Distribution Problem)                     │
    │                                                                 │
    │  [알고리즘]                                                     │
    │   - AES (Advanced Encryption Standard): 128/192/256비트          │
    │   - DES (Data Encryption Standard): 56비트 (취약)               │
    │   - 3DES (Triple DES): DES의 3번 반복                            │
    │   - ChaCha20: 모바일/임베디드 환경                               │
    │                                                                 │
    │  [운영 모드]                                                    │
    │   - ECB: 블록 독립 암호화 (취약)                                │
    │   - CBC: chaining block (널리 사용)                             │
    │   - GCM: 병렬 처리 + 무결성 (권장)                              │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 공개키 암호화 (Asymmetric Key)
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │              공개키 암호화 (Asymmetric Key)                      │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   송신자                              수신자(Bob)                │
    │      │                                   │                       │
    │      ├── 평문 "Hello" ────┐              │                       │
    │      │                    │              │                       │
    │      │              [암호화]             │                       │
    │      │         (Bob의 공개키)            │                       │
    │      │                    │              │                       │
    │      │           암호문 "Ab9$" ──────────┐                       │
    │      │                                  │   │                   │
    │      │                                  │ [복호화]              │
    │      │                                  │   │                   │
    │      │                                  │ (Bob의 개인키)        │
    │      │                                  │   │                   │
    │      │                                  └─ 평문 "Hello"        │
    │                                                                 │
    │  [특징]                                                         │
    │   - 암호화용 공개키, 복호화용 개인키 쌍                          │
    │   - 공개키는 공개 가능, 개인키는 비밀 유지                        │
    │   - 키 배송 문제 해결                                           │
    │   - 대칭키 대비 100~1000배 느림                                 │
    │                                                                 │
    │  [알고리즘]                                                     │
    │   - RSA: 소인수분해 기반 (2048~4096비트)                         │
    │   - ECC (Elliptic Curve): 타원 곡선 기반 (짧은 키로 강력한 보안)  │
    │   - Diffie-Hellman: 키 교환 프로토콜                            │
    │   - ElGamal: 이산대수 기반                                      │
    │                                                                 │
    │  [하이브리드 방식]                                               │
    │   - 대용량 데이터: 대칭키(AES) 암호화                            │
    │   - 대칭키: 공개키(RSA)로 암호화하여 전송                        │
    │   - 예: TLS/SSL, PGP, S/MIME                                    │
    └─────────────────────────────────────────────────────────────────┘
```

### 3. 해시 함수 (Hash Function)
- **목적**: 무결성 검증, 비밀번호 저장
- **특징**: 일방향 함수(역함수 불가), 고정 길이 출력, Avalanche 효과
- **알고리즘**:
  - SHA-256, SHA-3 (권장)
  - MD5 (취약, 충돌 발견)
  - SHA-1 (취약, 폐기)

- **📢 섹션 요약 비유**: 대칭키는 '주머니 열쇠'(같은 열쇠로 열고 잠금), 공개키는 '우편함'(누구나 투입 가능, 주인만 개봉)과 같습니다.

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. OS 수준 암호화
| 레벨 | 기술 | 설명 | 예시 |
|:---|:---|:---|:---|
| **전체 디스크** | BitLocker, LUKS | 부팅 전 전체 암호화 | Windows, Linux |
| **파일 시스템** | eCryptfs, EncFS | 특정 디렉토리 암호화 | ~/.Private |
| **파일 단위** | GPG, 7-Zip | 개별 파일 암호화 | 이메일, 문서 |
| **데이터베이스** | TDE, Column Encryption | DB 암호화 | Oracle, SQL Server |

### 2. 네트워크 암호화
- **TLS/SSL**: HTTPS, SMTPS, FTPS
- **VPN**: IPSec, OpenVPN, WireGuard
- **SSH**: 원격 접속 암호화

### 3. 안티패턴
- **"자체 암호 알고리즘 개발"** → 표준(AES, RSA) 사용
- **"암호화 키를 코드에 하드코딩"** → 키 관리 시스템 사용
- **"CBC 모드에서 IV 재사용"** → 랜덤 IV 사용
- **"MD5/SHA-1 사용"** → SHA-256 이상 사용

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **규정 준수**: GDPR, PCI-DSS, 개인정보보호법 만족
- **데이터 유출 방지**: 분실 도난 시에도 기밀 유지

### 2. 미래 전망
- **Quantum-safe 암호**: 양자 컴퓨터에 강한 포스트양자 암호 (Lattice-based 등)
- **Homomorphic Encryption**: 암호화된 상태로 연산 가능
