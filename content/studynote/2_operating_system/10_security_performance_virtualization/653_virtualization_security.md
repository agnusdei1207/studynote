# 가상화 보안 (Virtualization Security)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상화 보안은 **VM 간 격리, 하이퍼바이저 보안, VM Escape 방어**를 통해 가상화 환경의 보안을 보장하는 분야이다.
> 2. **가치**: "한 VM이 해킹되어도 다른 VM으로 확산 방지"가 핵심이며, **공격 표면 확장**을 방지한다.
> 3. **융합**: 하드웨어 지원(Intel VT-x, AMD-V), 보안 커널, TPM과 결합하여 **신뢰할 수 있는 가상화** 환경을 구축한다.

+++

## Ⅰ. 가상화 보안의 개요

### 1. 정의
- 가상화 보안은 가상화 환경의 특정 보안 위혙(VM Escape, 하이퍼바이저 공격)을 방어한다.

### 2. 등장 배경
- 가상화로 "하나의 하드웨어에 여러 시스템" → 새로운 공격 표면

### 3. 💡 비유: '아파트의 방隔音'
- 가상화 보안은 **"아파트의 각 세대 방음과 방범"**과 같다.
- 한 세대에서 문제가 발생해도 이웃에게 영향을 주지 않아야 한다.

- **📢 섹션 요약 비유**: 여러 명이 같은 버스를 타도 각자의 좌석에 앉듯, VM도 서로 격리되어야 안전합니다.

## Ⅱ. 주요 위협 (Deep Dive)

### 1. VM Escape
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                     VM Escape 공격                              │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   ┌──────────────┐       ┌──────────────┐                     │
    │   │ Attacker VM  │       │ Victim VM    │                     │
    │   │ (Guest OS)   │       │ (Guest OS)   │                     │
    │   └──────┬───────┘       └──────▲───────┘                     │
    │          │                       │                             │
    │          │   취약점 악용         │                             │
    │          │                      │                             │
    │          └──────────┬───────────┘                             │
    │                     ▼                                          │
    │            ┌───────────────────────┐                          │
    │            │    Hypervisor         │                          │
    │            │    (제어 획득)        │                          │
    │            └───────────────────────┘                          │
    │                                                                 │
    │  * VM이 하이퍼바이저를 탈출하여 호스트나 다른 VM에 접근        │
    │  * 가장 위험한 가상화 공격                                      │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 기타 위협
| 위협 | 설명 |
|:---|:---|
| **VM Sprawl** | 관리되지 않는 VM 방치로 공격 표면 증가 |
| **호스트 공격** | 하이퍼바이저/호스트 OS 공격 |
| **Side-channel** | 캐시, 타이밍으로 정보 유출 (Spectre/Meltdown) |
| **리소스 DoS** | 한 VM이 리소스 독점 |

## Ⅲ. 격리 기술

### 1. 하드웨어 지원 격리
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                   하드웨어 보안 기술                             │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [Intel VT-x / AMD-V]                                          │
    │   - CPU 가상화 확장, VMCS 보호                                   │
    │                                                                 │

    │  [VT-d / IOMMU]                                                │
    │   - 장치를 VM에 직접 할당, DMA 보호                             │
    │                                                                 │
    │  [SGX / SEV]                                                   │
    │   - 암호화된 VM (Intel SGX, AMD SEV)                            │
    │                                                                 │
    │  [TPM]                                                         │
    │   - 신뢰할 수 있는 플랫폼 (vTPM)                                │
    └─────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 하이퍼바이저 보안

### 1. 최소화 원칙
- **TCD (Trusted Computing Base)** 최소화
- 불필요한 드라이버/기능 비활성화

### 2. 보안 설정
```bash
# KVM 예시
# /etc/modprobe.d/kvm.conf
options kvm ignore_msrs=1

# libvirt 보안
/etc/libvirt/qemu.conf
  - user="root"
  - group="root"
  - dynamic_ownership=1
```

## Ⅴ. 컨테이너 보안

### 1. VM vs 컨테이너 보안
| 비교 항목 | VM | 컨테이너 |
|:---|:---|:---|
| **격리** | 강함 (하드웨어) | 약함 (프로세스) |
| **Attack Surface** | 큼 (전체 OS) | 작음 |
| **Escape 난이도** | 어려움 | 상대적으로 쉬움 |

### 2. 컨테이너 보안 기법
- **User Namespace**: root를 비권한 사용자로 매핑
- **Seccomp**: 시스템 콜 제한
- **AppArmor/SELinux**: 프로필 기반 접근 제어
- **Rootless 컨테이너**: root 권한 없이 실행
