# 가상화 (Virtualization)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상화(Virtualization)은 **단일 물리 리소스(CPU, 메모리, 디스크)를 여러 논리적 단위로 추상화**하여 각각이 독립된 시스템처럼 동작하게 하는 기술이다.
> 2. **가치**: 하드웨어 활용도 극대화, **리소스 격리**, **신속한 프로비저닝**, **마이그레이션** 용이성을 제공하여 클라우드 컴퓨팅의 기반이 되었다.
> 3. **융합**: Type 1(Bare Metal), Type 2(Hosted) 하이퍼바이저, 컨테이너(Docker), 파라가상화, 전가상화 등 다양한 기법이 하드웨어 지원(Intel VT-x, AMD-V)과 결합한다.

+++

## Ⅰ. 가상화의 개요

### 1. 정의
- 가상화는 하드웨어나 운영체제를 소프트웨어로 추상화하여 **여러 개의 가상 시스템(VM)을 생성 및 실행**하는 기술이다.
- **Hypervisor**: 가상머신을 생성하고 관리하는 소프트웨어 계층

### 2. 등장 배경: x86 하드웨어의 비효율
- 1960년대 IBM mainframe에서 시작 (CP/CMS)
- 1990년대 x86에서 가상화 어려움 (特权 명령이 많음)
- 2005년 Intel VT-x, AMD-V로 하드웨어 지원 시작 → VM웨어, Xen 등 등장

### 3. 💡 비유: '아파트 건물'
- 가상화는 **'하나의 대지(물리 서버)에 여러 세대(VM)를 가진 아파트'**와 같다.
- 각 세대는 독립된 공간(격리)을 가지지만, 수도/전기(하드웨어)은 공유한다.

- **📢 섹션 요약 비유**: 하나의 큰 부엌(물리 서버)을 칸막이로 나누어 여러 사람이 각자 요리(VM 실행)할 수 있게 하는 것입니다.

## Ⅱ. 하이퍼바이저 유형 (Deep Dive)

### 1. Type 1 (Bare Metal) vs Type 2 (Hosted)
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │           하이퍼바이저 유형 비교                                 │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [Type 1: Bare Metal / Native]                                 │
    │   ┌─────────────────────────────────────────────────────────┐  │
    │   │  VM1        │  VM2        │  VM3        │  VM4        │  │
    │   │  (Guest OS) │  (Guest OS) │  (Guest OS) │  (Guest OS) │  │
    │   ├────────────┼────────────┼────────────┼────────────┤  │
    │   │            Hypervisor (VMware ESXi, Xen, KVM)         │  │
    │   └─────────────────────────────────────────────────────────┘  │
    │   │                                                           │
    │   ▼                                                           │
    │  Hardware (CPU, RAM, Disk)                                    │
    │                                                                 │
    │   * 장점: 빠름, 안정적                                         │
    │   * 용도: 데이터센터, 클라우드                                  │
    │                                                                 │
    │  [Type 2: Hosted]                                             │
    │   ┌─────────────────────────────────────────────────────────┐  │
    │   │  VM1        │  VM2        │  VM3                        │  │
    │   │  (Guest OS) │  (Guest OS) │  (Guest OS)                 │  │
    │   ├────────────┼────────────┼────────────┤                 │  │
    │   │       Hypervisor (VMware Workstation, VirtualBox)       │  │
    │   ├─────────────────────────────────────────────────────────┤  │
    │   │       Host OS (Windows, Linux, macOS)                   │  │
    │   └─────────────────────────────────────────────────────────┘  │
    │   │                                                           │
    │   ▼                                                           │
    │  Hardware                                                      │
    │                                                                 │
    │   * 장점: 설치 쉬움, 개발용 적합                               │
    │   * 단점: Host OS 오버헤드                                    │
    │   * 용도: 개발, 테스트                                         │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 하이퍼바이저 제품 비교
| 유형 | 제품 | 특징 | 용도 |
|:---|:---|:---|:---|
| **Type 1** | VMWare ESXi | 상용, 안정적 | 엔터프라이즈 |
| **Type 1** | Microsoft Hyper-V | Windows 통합 | 윈도우 환경 |
| **Type 1** | Xen | 오픈소스, 파라가상화 | AWS, Citrix |
| **Type 1** | KVM | Linux 커널 내장, 모듈형 | Red Hat, OpenStack |
| **Type 2** | VMware Workstation | 개발자용 | 데스크톱 |
| **Type 2** | VirtualBox | 무료, 크로스플랫폼 | 개발/테스트 |
| **Type 2** | Parallels | macOS 최적화 | Mac 사용자 |

## Ⅲ. 가상화 기법

### 1. 전가상화 (Full Virtualization)
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                 전가상화 (Full Virtualization)                  │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   Guest OS                                                      │
    │      │                                                         │
    │      ├─特权 명령 (Ring 0)                                      │
    │      │                                                         │
    │      ▼                                                         │
    │   ┌───────────────────────────────────────────────┐            │
    │   │  Binary Translation (이진 번역)              │            │
    │   │  -特权 명령을 하이퍼바이저 호출로 변환         │            │
    │   │  - 하드웨어 지원 (Intel VT-x)으로 가속         │            │
    │   └───────────────────────────────────────────────┘            │
    │      │                                                         │
    │      ▼                                                         │
    │   Hypervisor                                                    │
    │      │                                                         │
    │      ▼                                                         │
    │   Hardware                                                      │
    │                                                                 │
    │  * Guest OS는 자신이 물리 머신에서 실행된다고 착각             │
    │  * 수정 불필요                                                   │
    │  * 예: VMware ESXi, KVM                                         │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 파라가상화 (Para-Virtualization)
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │               파라가상화 (Para-Virtualization)                   │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   Guest OS (수정됨)                                             │
    │      │                                                         │
    │      ├─ Hypercall (하이퍼바이저 직접 호출)                     │
    │      │  -特权 명령 대신 hypercall 사용                          │
    │      │                                                         │
    │      ▼                                                         │
    │   Hypervisor (가벼움)                                           │
    │      │                                                         │
    │      ▼                                                         │
    │   Hardware                                                      │
    │                                                                 │
    │  * Guest OS 수정 필요                                          │
    │  * 전가상화 대비 오버헤드 적음                                  │
    │  * 예: Xen (초기)                                              │
    └─────────────────────────────────────────────────────────────────┘
```

### 3. 하드웨어 지원 가상화 (HVM)
- **Intel VT-x / AMD-V**: CPU 가상화 지원
- **EPT (Extended Page Tables)**: 메모리 가상화 가속
- **VT-d**: I/O 장치 가상화 (Direct Device Assignment)

- **📢 섹션 요약 비유**: 전가상화는 "번역기 통해 대화", 파라가상화는 "직접 알아듣는 말로 대화", HVM은 "모국어로 대화"하는 차이입니다.

+++

## Ⅳ. 메모리 및 CPU 가상화

### 1. CPU 가상화
- **Time Slicing**: 각 VM에 CPU 시간 할당 (Scheduler)
- **vCPU**: 물리 코어를 논리적으로 분할
- **CPU Affinity**: VM을 특정 코어에 고정

### 2. 메모리 가상화
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │              메모리 가상화: 2단계 페이지 테이블                   │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   Guest OS (가상 주소)                                          │
    │      │                                                         │
    │      ▼ Guest Page Table                                        │
    │   Guest Physical Address (가상 물리 주소)                       │
    │      │                                                         │
    │      ▼ Shadow Page Table / EPT                                 │
    │   Host Physical Address (실제 물리 주소)                        │
    │      │                                                         │
    │      ▼                                                         │
    │   Physical RAM                                                 │
    │                                                                 │
    │  * VA → GPA → PA 변환                                         │
    │  * EPT/NPT로 하드웨어 가속                                     │
    └─────────────────────────────────────────────────────────────────┘
```

### 3. 메모리 오버커밋
- 하이퍼바이저는 VM에 할당된 총 메모리 > 실제 물리 메모리 가능
- **Ballooning**: Guest OS 내부 드라이버가 여유 메모리 반납
- **Swapping**: 메모리 부족 시 디스크 스왑

## Ⅴ. 가상화의 응용 및 이점

### 1. 서버 통합 (Server Consolidation)
- 물리 서버 10대 → 가상화 1대로 통합
- 전력, 공간, 관리 비용 절감

### 2. 라이브 마이그레이션 (Live Migration)
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │               라이브 마이그레이션 과정                           │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [Source Host]                  [Destination Host]              │
    │      │                                │                        │
    │      │ ① Pre-Copy (반복 전송)          │                        │
    │      ├───────────────▶                │                        │
    │      │ (변경된 페이지만 전송)           │                        │
    │      │                                │                        │
    │      │ ② Downtime (짧은 중지)          │                        │
    │      ├───────────────▶                │                        │
    │      │ (마지막 페이지 + 상태 전송)      │                        │
    │      │                                │                        │
    │      │ ③ Activation (활성화)           │                        │
    │      │                                │                        │
    │   VM ▶                                ◀ VM                      │
    │                                                                 │
    │  * 사용자는 중단 감지 못 함 (ms 단위)                           │
    └─────────────────────────────────────────────────────────────────┘
```

### 3. 스냅샷 및 복구
- **Snapshot**: 특정 시점의 VM 상태 보존
- **Rollback**: 스냅샷으로 복구
- **Cloning**: VM 복제

### 4. 안티패턴
- **"VM Sprawl"**: 사용하지 않는 VM 방치
- **"Over-allocation"**: 실제 필요 이상의 리소스 할당
- **"Type 2를 프로덕션에 사용"**

## Ⅵ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **비용 절감**: 서버 대수 50~80% 감소
- **유연성**: 새 서버 배포 시간 1주 → 5분
- **고가용성**: 라이브 마이그레이션으로 무중단 유지보수

### 2. 미래 전망
- **Containers**: Docker, Kubernetes (경량 가상화)
- **NFV**: 네트워크 기능 가상화 (5G)
- **Edge Computing**: 엣지에서 가상화
