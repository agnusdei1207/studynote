# 사용자 인증 (Authentication)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 사용자의 신원을 검증하는 보안의 1단계 절차
> 2. **가치**: 무단 접근 방지를 위한 첫 관문이자 추적 가능성의 기초
> 3. **융합**: 지식(Knowledge), 소유(Possession), 특성(Inherence) 요소의 다층적 조합

---

### Ⅰ. 개요 (Context & Background)

**개념**
인증(Authentication)은 **사용자나 프로세스의 신원을 확인하는 절차**이다. 이는 보안의 3요소(CIA) 중 첫 단계로, "누가(Access Control)"의 사전 확인 작업이다. 단일 요소 인증(SFA)에서 다중 요소 인증(MFA)으로 발전해 왔으며, 생체 인증, 하드웨어 토큰, 행동 기반 인증 등 다양한 기법이 존재한다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         인증의 기본 흐름                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   1단계: 신원 제시 (Identification)                                         │
│   ───────────────────────────────────────────────────────────────────────   │
│   사용자: "저는 alice입니다"                                                │
│   시스템: "ID 입력 받음"                                                    │
│                                                                             │
│   2단계: 인증 (Authentication)                                              │
│   ───────────────────────────────────────────────────────────────────────   │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  인증 요소 제시                                                     │  │
│   │                                                                     │  │
│   │  [지식] Something you KNOW                                         │  │
│   │   • 비밀번호                                                        │  │
│   │   • PIN                                                             │  │
│   │   • 보안 질문 답                                                    │  │
│   │                                                                     │  │
│   │  [소유] Something you HAVE                                         │  │
│   │   • 스마트폰 (TOTP 앱)                                             │  │
│   │   • 하드웨어 토큰 (YubiKey)                                        │  │
│   │   • OTP 카드                                                        │  │
│   │                                                                     │  │
│   │  [특성] Something you ARE                                          │  │
│   │   • 지문                                                            │  │
│   │   • 안면 인식                                                       │  │
│   │   • 홍채                                                            │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   3단계: 승인 (Authorization) - 인증 후 권한 확인                           │
│   ───────────────────────────────────────────────────────────────────────   │
│   시스템: "alice의 신원 확인됨, 접근 허가"                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
인증은 **건물 경비실 출입 확인**과 같다.
- 신분증: 신원 확인(Identification)
- 비밀번호: 사진 본인 확인(Authentication)
- 출입 카드: 소유물 확인(Possession)
- 얼굴: 생체 확인(Inherence)

**등장 배경**
① **1960년대**: 컴퓨터 비밀번호 도입 (MIT CTSS)
② **1980년대**: Kerberos 인증 시스템
③ **2000년대**: 2FA(2-Factor Authentication) 보급
④ **현재**: MFA, 생체 인증, FIDO2 표준

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 인증 요소 | 유형 | 장점 | 단점 | 예시 |
|:---|:---|:---|:---|:---|
| **지식(Knowledge)** | SFA | 구현 쉬움 | 도난, 피싱 | 비밀번호, PIN |
| **소유(Possession)** | 2FA | 재사용 어려움 | 분실 가능 | OTP, 토큰 |
| **특성(Inherence)** | 생체 | 항상 휴대 | 변형, 위조 가능 | 지문, 홍채 |
| **위치(Location)** | 컨텍스트 | 무단 접속 방지 | 우회 가능 | IP, GPS |
| **행동(Behavior)** | 행동 기반 | 동적 탐지 | 오탐지 | 타이핑 패턴 |

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 인증 프로토콜**

| 프로토콜 | 유형 | 장점 | 단점 | 사용 환경 |
|:---|:---|:---|:---|:---|
| **Password** | SFA | 간단 | 취약함 | 일반 |
| **Kerberos** | 티켓 기반 | SS0, 안전 | KDC 필요 | 엔터프라이즈 |
| **LDAP** | 디렉터리 | 중앙 관리 | 복잡함 | 기업 내부 |
| **SAML/OIDC** | FedID | SSO | 복잡함 | 웹 |
| **FIDO2** | 공개키 | 강력함 | 지원 필요 | 최신 웹 |

**과목 융합 관점**

1. **운영체제**: 사용자 관리, 세션 관리, PAM
2. **네트워크**: 프로토콜, 암호화, MITM 방어
3. **암호학**: 해시, HMAC, 공개키

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**

| 지표 | SFA만 | MFA 도입 | 개선 효과 |
|:---|:---|:---|:---|
| **계정 도난 위험** | 높음 | 매우 낮음 | 99.9% 감소 |
| **피싱 성공률** | 높음 | 낮음 | 95% 감소 |
| **사용자 불편** | 낮음 | 중간 | 허용 가능 |

**미래 전망**
1. **비밀번호 없는 인증**: FIDO2, WebAuthn
2. **연속 인증**: 행동 패턴 기반
3. **영지식 증명**: Privacy-preserving 인증

**참고 표준**
- **NIST SP 800-63B**: 디지털 아이덴티티 가이드
- **RFC 6238**: TOTP
- **FIDO2**: 공개키 기반 인증

### 👶 어린이를 위한 3줄 비유 설명
1. **개념**: 인증은 현관문 열 때 몇 가지 방법으로 내가 누군지 증명하는 거예요
2. **원리**: 비밀번호(아는 것), 열쇠카드(가진 것), 지문(내 몸의 특징)을 사용해요
3. **효과**: 여러 방법을 함께 사용하면 나쁜 사람이 우리 집에 들어오기 훨씬 어려워져요
