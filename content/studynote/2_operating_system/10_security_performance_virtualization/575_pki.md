# 공개키 기반 구조 (PKI)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: PKI(Public Key Infrastructure)는 **공개키 암호화 기반의 신뢰 가능한 디지털 생태계**를 구축하는 인프라로, 인증 기관(CA)이 발급한 인증서를 통해 공개키의 소유자를 보증한다.
> 2. **가치**: "이 공개키가 정말 해당 기관의 것인가?"라는 **신뢰 문제를 해결**하여 HTTPS, 코드 사이닝, 이메일 암호화 등 현대 인터넷 보안의 기반을 제공한다.
> 3. **융합**: X.509 인증서, CRL/OCSP(폐지 확인), HSM(하드웨어 보안 모듈)과 결합하여 **Enterprise Trust Architecture**를 구현한다.

+++

## Ⅰ. PKI의 개요

### 1. 정의
- PKI는 공개키 암호화를 기반으로 디지털 인증서를 발급, 관리, 폐기하는 시스템, 프로토콜, 정책의 집합이다.
- **목표**: 안전한 전자 거래, 통신, 신원 확인을 위한 신뢰 인프라 제공

### 2. 등장 배경: 공개키의 신뢰 문제
- 공개키 암호화는 강력하지만, "이 공개키가 정말 Google의 것인가?"를 확인할 방법이 필요
- 중간자 공격(Man-in-the-Middle) 방지를 위해 제3자(CA)가 보증 필요

### 3. 💡 비유: '주민등록등본 발급 시스템'
- PKI는 **'정부가 발급한 신분증 발급 시스템'**과 같다.
- 주민센터(CA)가 내 사진과 이름을 확인하고 공적으로 신원을 보증한다.

- **📢 섹션 요약 비유**: 은행에서 본인 확인 서류(인증서)를 발급받으면, 다른 은행에서도 그 서류를 믿고 거래할 수 있게 하는 시스템입니다.

## Ⅱ. PKI 구성 요소 (Deep Dive)

### 1. PKI의 핵심 구성 요소
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                    PKI 구성 요소                                │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [CA (Certificate Authority)] 인증 기관                          │
    │   ┌─────────────────────────────────────────────────────────┐  │
    │   │                                                         │  │
    │   │  Root CA ──────► Intermediate CA ──────► Issuing CA     │  │
    │   │      │                 │                     │          │  │
    │   │      └───── 신뢰의 뿌리  └───── 인증서 발급     │          │  │
    │   │                                              │          │  │
    │   │                                       [인증서 발급]       │  │
    │   └─────────────────────────────────────────────────────────┘  │
    │                                                                 │
    │  [RA (Registration Authority)] 등록 기관                         │
    │   - 신원 확인, 인증서 신청 접수 (CA의 대리기관)                   │
    │                                                                 │
    │  [Certificate] 디지털 인증서                                     │
    │   - 공개키 + 소유자 정보 + CA 서명                               │
    │   - X.509 표준                                                  │
    │                                                                 │
    │  [Repository] 저장소                                            │
    │   - 발급된 인증서, CRL(폐지 목록) 공개 저장소                    │
    │                                                                 │
    │  [Policy] 정책                                                   │
    │   - CP (Certificate Policy): 인증서 발급 정책                     │
    │   - CPS (Certification Practice Statement): 운영 지침서          │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. X.509 인증서 구조
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │              X.509 인증서의 주요 필드                            │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [Version]                                                       │
    │   → X.509 버전 (v3 일반적)                                      │
    │                                                                 │
    │  [Serial Number]                                                │
    │   → CA가 할당한 고유 번호                                       │
    │                                                                 │
    │  [Issuer]                                                       │
    │   → 발급한 CA의 식별자 (Distinguished Name)                     │
    │                                                                 │
    │  [Validity]                                                     │
    │   → 유효 기간 (Not Before ~ Not After)                         │
    │                                                                 │
    │  [Subject]                                                      │
    │   → 인증서 소유자의 정보 (CN, O, OU, C, L 등)                   │
    │                                                                 │
    │  [Subject Public Key Info]                                     │
    │   → 소유자의 공개키와 알고리즘                                  │
    │                                                                 │
    │  [Issuer Unique ID / Subject Unique ID]                         │
    │   → v2 이상에서 중복 방지용 (선택)                              │
    │                                                                 │
    │  [Extensions]                                                   │
    │   → Key Usage, Extended Key Usage, CRL Distribution Points 등   │
    │                                                                 │
    │  [Signature Algorithm]                                         │
    │   → CA의 서명 알고리즘 (SHA256withRSA 등)                       │
    │                                                                 │
    │  [Signature]                                                    │
    │   → CA의 개인키로 생성한 디지털 서명                            │
    └─────────────────────────────────────────────────────────────────┘
```

### 3. 인증서 검증 체인 (Certificate Chain)
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │               인증서 체인 검증 과정                               │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   End-Entity Certificate (서버 인증서)                          │
    │        │                                                        │
    │        │ ① 자신의 서명 검증                                     │
    │        ▼                                                        │
    │   Intermediate CA Certificate                                  │
    │        │                                                        │
    │        │ ② Intermediate CA의 서명 검증                         │
    │        ▼                                                        │
    │   Root CA Certificate                                          │
    │        │                                                        │
    │        │ ③ Root CA는 자서명(Self-signed)이며 신뢰 저장소에 사전등록│
    │        ▼                                                        │
    │   [신뢰 저장소] (OS/브라우저 내장)                              │
    │        │                                                        │
    │        ✓ 신뢰 체인 완성                                        │
    │                                                                 │
    │  * 브라우저는 Root CA 리스트(≈100개)를 내장                     │
    └─────────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 인증서 체인은 "정부 → 시청 → 동사무소"의 확인 서류와 같습니다. 최상위(정부)를 신뢰하면 아래(동사무소)까지 모두 신뢰할 수 있습니다.

## Ⅲ. PKI 동작 과정

### 1. 인증서 발급 과정
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                  인증서 발급 절차                                 │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [1단계] 키 생성 (Key Generation)                               │
    │   - 서버: 공개키/개인키 쌍 생성                                  │
    │                                                                 │
    │  [2단계] CSR 생성 (Certificate Signing Request)                 │
    │   - 서버: CSR(공개키 + 정보) 생성                               │
    │   - 개인키로 서명                                               │
    │                                                                 │
    │  [3단계] 신원 확인 (Identity Verification)                      │
    │   - RA/CA: 신청자 신원 확인 (도메인 소유권, 사업자등록 등)       │
    │                                                                 │
    │  [4단계] 인증서 발급 (Issuance)                                 │
    │   - CA: CSR의 공개키 + 정보 + CA 서명 → 인증서                  │
    │                                                                 │
    │  [5단계] 인증서 설치 (Installation)                             │
    │   - 서버: 인증서 + 개인키 설치                                  │
    │                                                                 │
    │  [6단계] 인증서 제시 (Presentation)                             │
    │   - 클라이언트 접속 시 인증서 전송                              │
    │                                                                 │
    │  [7단계] 검증 (Validation)                                     │
    │   - 클라이언트: 체인 검증, 유효기간 확인, CRL 확인              │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 인증서 폐지 (Revocation)
- **CRL (Certificate Revocation List)**: 폐기된 인증서 목록 주기적 발행
- **OCSP (Online Certificate Status Protocol)**: 실시간 폐지 확인
- **폐지 사유**: 키 유출, CA 해킹, 정보 변경, 만료

## Ⅳ. PKI 응용 분야

| 응용 분야 | 설명 | 예시 |
|:---|:---|:---|
| **TLS/SSL** | 웹 서버 인증, HTTPS | google.com 인증서 |
| **Code Signing** | 코드 서명, 무결성 보증 | Windows 드라이버, 모바일 앱 |
| **S/MIME** | 이메일 암호화 & 서명 | 기업 이메일 |
| **VPN** | VPN 게이트웨이 인증 | IPSec, OpenVPN |
| **Document Signing** | 전자 문서 서명 | 계약서, PDF |

## Ⅴ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 자체 PKI 구축 vs 공용 CA
| 항목 | 자체 PKI | 공용 CA (DigiCert, Let's Encrypt) |
|:---|:---|:---|
| **비용** | 초기 높음 | 무료~저렴 |
| **신뢰 범위** | 내부만 | 전 세계 |
| **관리** | 직접 | CA 담당 |
| **용도** | 내부 시스템, VPN | 공개 웹서비스 |

### 2. OpenSSL 명령어
```bash
# CSR 생성
openssl req -new -newkey rsa:2048 -nodes -keyout server.key -out server.csr

# 인증서 확인
openssl x509 -in server.crt -text -noout

# 인증서 체인 확인
openssl s_client -connect google.com:443 -showcerts
```

### 3. 안티패턴
- **"만료된 인증서 방치"**
- **"Root CA 개인키를 온라인에 보관"**
- **"모든 서버에 동일한 인증서 사용"**
- **"인증서 폐지 미처리"**
