# TPM (Trusted Platform Module)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드웨어 기반의 암호화 키 저장 및 신뢰 실행(Remote Attestation)을 위한 보안 칩
> 2. **가치**: 소프트웨어 공격으로부터 키를 보호하고 시스템 무결성을 검증
> 3. **융합**: BIOS/UEFI, OS, 애플리케이션 전 계층의 하드웨어 루트 오브 트러스트

---

### Ⅰ. 개요 (Context & Background)

**개념**
TPM(Trusted Platform Module)는 **마더보드에 장착된 보안 칩으로, 암호화 키를 안전하게 저장하고 시스템 무결성을 검증하는 하드웨어 모듈**이다. 1999년 Trusted Computing Group(TCG)이 표준화했으며, TPM 1.2, 2.0 버전이 있다. 키 생성, 저장, 서명, 암호화, 원격 검증(Remote Attestation) 등의 기능을 제공한다.

**💡 비유**
TPM은 **은행 금고**와 같다.
- 금고: 귀중품(키)을 안전하게 보관
- 본인 확인: 특정 조건에서만 열림
- 위변조 방지: 금고 상태를 원격으로 확인

**등장 배경**
① **1999년**: TCG 설립, TPM 1.0
② **2003년**: TPM 1.2 표준화
③ **2014년**: TPM 2.0 (알고리즘 독립적)
④ **현재**: Windows 11 필수 요건

**구성 요소**

| 구성 요소 | 역할 | 특징 |
|:---|:---|:---|
| **EK (Endorsement Key)** | TPM 고유 키 | 제조 시 생성, 변경 불가 |
| **SRK (Storage Root Key)** | 키 계층의 루트 | TPM 내부 생성, 불출 |
| **PCR (Platform Configuration Register)** | 무결성 측정 | 24개 레지스터, 확장 가능 |
| **NVRAM** | 비휘발성 메모리 | 키, 데이터 저장 |
| **RNG** | 난수 생성기 | 하드웨어 기반 |

**해설**
1. **EK**: TPM의 신원증명, 제조 시 생성
2. **SRK**: 키 계층의 루트, TPM에서 불출
3. **PCR**: 부팅 과정 측정, 무결성 검증
4. **키 봉인**: PCR 값에 따라 데이터 접근 제어

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: TPM vs 소프트웨어 보안**

| 특성 | TPM | 소프트웨어 |
|:---|:---|:---|
| **키 보호** | 하드웨어 격리 | 메모리 노출 |
| **물리 공격** | 내성성 있음 | 취약 |
| **무결성 검증** | PCR 기반 | 제한적 |
| **원격 검증** | 가능 | 어려움 |
| **비용** | 하드웨어 추가 | 무료 |
| **호환성** | TPM 필수 | 범용 |

**과목 융합 관점**

1. **하드웨어**: 마더보드, 칩, 보안 부팅
2. **운영체제**: 드라이버, 키 관리, 암호화 API
3. **암호학**: 공개키, 대칭키, 해시

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: TPM 확인 (Linux)**
```bash
# TPM 존재 확인
sudo dmesg | grep -i tpm

# TPM 정보
tpm2_getcap properties-fixed

# PCR 값 확인
tpm2_pcrread
```

**시나리오 2: BitLocker 사용 (Windows)**
- TPM + BitLocker로 드라이브 암호화
- TPM이 키를 안전하게 보관

**도입 체크리스트**

**기술적 측면**
- [ ] TPM 칩 존재 확인
- [ ] TPM 활성화 (BIOS 설정)
- [ ] TPM 소프트웨어 설치
- [ ] 키 봉인/해제 테스트
- [ ] 백업 키 저장

**운영·보안적 측면**
- [ ] TPM 복구 키 관리
- [ ] 하드웨어 교체 절차
- [ ] 정기적 무결성 검사
- [ ] 원격 검증 구성

**안티패턴**
1. **"TPM만 있으면 돼"**: 소프트웨어도 필요
2. **"복구 키 필요 없어"**: TPM 오류 시 복구 불가
3. **"구형 TPM도 OK"**: TPM 2.0 권장

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**

| 지표 | TPM 미사용 | TPM 사용 | 개선 효과 |
|:---|:---|:---|:---|
| **키 노출 위험** | 높음 | 매우 낮음 | 99% 감소 |
| **부팅 무결성** | 확인 어려움 | 자동 검증 | 100% 검증 |
