# 그린 컴퓨팅 (Green Computing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 그린 컴퓨팅은 **컴퓨팅의 환경 영향을 줄이기 위한 전략**으로, 에너지 효율, 전자 폐기물 감축, 친환경 설계를 포함한다.
> 2. **가치**: 데이터센터의 전력 소비는 전 세계 전력의 2~3%를 차지하며, **PUE 개선, 가상화, 저전력 장치**로 탄소 배출을 줄일 수 있다.
> 3. **융합**: 하드웨어(ARM, 칩렛), 소프트웨어(전력 인식 스케줄링), 데이터센터(자연 냉각)가 결합하여 **지속 가능한 IT**를 추구한다.

+++

## Ⅰ. 그린 컴퓨팅의 개요

### 1. 정의
- 그린 컴퓨팅은 환경에 미치는 컴퓨팅의 영향을 최소화하는 **설계, 제조, 사용, 폐기**의 전 과정이다.

### 2. 등장 배경
- IT 부문의 탄소 배출 증가
- 에너지 비용 상승, 규제 강화

### 3. 💡 비유: '친환경 주택'
- 그린 컴퓨팅은 **'단열, 태양광, 에너지 효율 가전을 갖춘 친환경 주택'**과 같다.
- 초기 비용은 더 들지만 장기적으로 에너지와 환경 비용을 절감한다.

- **📢 섹션 요약 비유**: 컴퓨터도 자동차처럼 연비(전력 효율)가 좋고, 배기가스(탄소)가 적은 친환경 모델로 바꿔야 해요.

+++

## Ⅱ. 에너지 효율 (Deep Dive)

### 1. 성능당 와트 (Performance per Watt)
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                성능당 와트 (Performance/Watt)                   │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [지표]                                                        │
    │   성능당 와트 = 성능 (MIPS/FLOPS) / 전력 소비 (W)              │
    │                                                                 │
    │  [예시]                                                        │
    │   CPU A: 1000 MIPS / 100W = 10 MIPS/W                         │
    │   CPU B: 800 MIPS  / 50W  = 16 MIPS/W  ← 더 친환경            │
    │                                                                 │
    │  [현황]                                                       │
    │   - x86: ~20-30 GFLOPS/W (HEDT)                               │
    │   - ARM: ~50-100 GFLOPS/W (모바일/서버)                       │
    │   - GPU: ~30-50 GFLOPS/W (HPC)                                │
    │                                                                 │
    │  * 전력 효율이 향상되고 있음                                   │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. SPECpower
- 서버의 전력 효율을 측정하는 벤치마크
- **전체 범위에서의 성능당 와트** 측정 (idle ~ 100% 부하)

+++

## Ⅲ. 데이터센터 최적화

### 1. PUE (Power Usage Effectiveness)
- **PUE = 총 전력 / IT 장비 전력**
- 목표: 1.0에 가까울수록 좋음
- 평균: 1.2~1.5 (최신), 2.0~3.0 (레거시)

### 2. 냉각 최적화
| 방식 | 설명 | PUE 개선 효과 |
|:---|:---|:---|
| **Hot/Cold Aisle** | 뜨겁고 찬 공간 분리 | 10~20% |
| **Free Cooling** | 외부 공기 활용 | 30~50% |
| **액체 냉각** | 물/유체로 냉각 | 40~60% |
| **치몰 기술** | 고산도에서 외기 활용 | 일부 지역에서 매우 효과적 |

+++

## Ⅳ. 하드웨어 친환경 설계

### 1. 저전력 아키텍처
- **ARM**: RISC 기반, 낮은 전력
- **칩렛(Chiplet)**: 필요한 만큼만 칩 사용
- **이종 컴퓨팅**: CPU + GPU + NPU 조합

### 2. 재활용 설계
- **모듈형 설계**: 부품 교체 용이
- **자재 선택**: 납, 수은 제거
- **표준화**: 부품 호환성

+++

## Ⅴ. 소프트웨어 최적화

### 1. 전력 인식 소프트웨어
- **유휴 상태 활용**: CPU, 디스크 절전
- **배치 처리**: 야간에만 실행
- **분산 컴퓨팅**: 여러 저전력 장치 활용

### 2. 알고리즘 효율
- 더 적은 연산으로 같은 결과
- **캐싱**: 반복 계산 방지

+++

## Ⅵ. 실무 적용

### 1. 데이터센터 현대화
- 구형 서버 → 최신 저전력 서버
- **가상화**: 서버 통합
- **자동화**: 불필요한 서버 off

### 2. 클라우드 활용
- 온-프레미스보다 **클라우드가 더 친환경**일 수 있음
- 규모의 경제, 최신 하드웨어

+++

## Ⅶ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **탄소 배출 감축**: 데이터센터의 탄소 발자국 줄이기
- **비용 절감**: 전력 비용 감소

### 2. 미래 전맹
- **재생 에너지**: 태양광, 풍력 기반 데이터센터
- **Grid-Interactive**: 전력망과 상호작용

## 👶 어린이를 위한 3줄 비유 설명
1. 그린 컴퓨팅은 **"환경을 생각하는 컴퓨터"**예요.
2. 전기를 적게 쓰고, 다 쓴 컴퓨터는 재활용해서 환경을 보호하죠.
3. 우리가 전기 절약하듯이, 컴퓨터도 전기를 아껴서 지구를 지켜요!
