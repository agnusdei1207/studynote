# 루트킷 (Rootkit)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템의 최고 권한(Root)을 획득한 후 존재 자체를 숨기는 은폐형 악성코드
> 2. **가치**: 공격자가 시스템을 지속적으로 장악하고 탐지를 회피하기 위한 최종 도구
> 3. **융합**: 커널 모듈, 시스템 콜 테이블 후킹, 프로세스 숨김 등 OS 깊은 지식 필요

---

### Ⅰ. 개요 (Context & Background)

**개념**
루트킷(Rootkit)은 **시스템의 최고 권한을 획득한 후 자신의 존재를 숨기고 활동을 은폐하는 스텔스형 악성코드**이다. 이름은 Root(최고 권한) + Kit(도구 모음)에서 유래했다. 사용자 수준(User-mode)과 커널 수준(Kernel-mode) 루트킷이 있으며, 커널 수준 루트킷은 탐지가 매우 어렵다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         루트킷 감염 및 은폐 과정                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   1단계: 초기 침투                                                          │
│   ───────────────────────────────────────────────────────────────────────   │
│   [취약점 악용] ──▶ [악성코드 실행] ──▶ [권한 상승]                        │
│                                                                             │
│   2단계: 루트킷 설치                                                         │
│   ───────────────────────────────────────────────────────────────────────   │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  일반 악성코드                      루트킷                         │  │
│   │  ┌─────────────┐                  ┌─────────────┐                 │  │
│   │  │ 프로세스 목록│     보임         │ 프로세스 목록│     숨김!       │  │
│   │  │ 파일 시스템 │     보임         │ 파일 시스템 │     숨김!       │  │
│   │  │ 네트워크    │     보임         │ 네트워크    │     숨김!       │  │
│   │  └─────────────┘                  └─────────────┘                 │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   3단계: 지속적인 활동 및 은폐                                               │
│   ───────────────────────────────────────────────────────────────────────   │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  [원격 접속] ◀───────────────────────────────────────────▶ [C2 서버] │  │
│   │       │                                                              │  │
│   │       │ 후킹: ps, ls, netstat, top 등이 루트킷을 표시하지 않음        │  │
│   │       │                                                              │  │
│   │       ▼                                                              │  │
│   │  [키로깅, 데이터 탈취, 공격 지원]                                      │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
루트킷은 **집에 숨어든 투명 인간**과 같다.
- 투명 인간: 집 안을 자유롭게 돌아다님
- CCTV: 인간이 보이지 않음
- 결과: 침입 사실을 모름, 지속적인 피해

**등장 배경**
① **1990년**: 처음으로 "rootkit" 용어 사용 (Sun OS)
② **2000년대**: 윈도우 루트킷 등장 (Hooking 기술 발전)
③ **2005년**: Sony BMG CD 루트킷 사건 (DRM 목적 루트킷)
④ **현재**: 부트킷, 펌웨어 루트킷으로 진화

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 유형 | 동작 레벨 | 난이도 | 탐지 어려움 | 대표 예시 |
|:---|:---|:---|:---|:---|
| **User-mode Rootkit** | 사용자 레벨 | 쉬움 | 쉬움 | Hacker Defender, Vanquish |
| **Kernel-mode Rootkit** | 커널 레벨 | 어려움 | 매우 어려움 | Adore-Ng, SuckIT |
| **Bootkit** | 부트로더 | 매우 어려움 | 극도로 어려움 | Stoned Bootkit |
| **Firmware Rootkit** | BIOS/UEFI | 극도로 어려움 | 거의 불가능 | LoJax (APT28) |

**해설**
1. **User-mode**: DLL 인젝션, API 후킹으로 은폐
2. **Kernel-mode**: 시스템 콜 테이블 후킹으로 커널 레벨 은폐
3. **Bootkit**: 부트로더 감염으로 OS 로드 전 실행

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 탐지 기법**

| 탐지 방법 | 원리 | 장점 | 단점 | 대표 도구 |
|:---|:---|:---|:---|:---|
| **Signature** | 알려진 루트킷 패턴 | 빠름 | 신변종 탐지 불가 | ClamAV |
| **Behavioral** | 이상 행동 패턴 | 신변종 탐지 | 오탐지 | OSSEC |
| **Integrity Check** | 시스템 콜 테이블 검증 | 확실 | 우회 가능 | rkhunter, chkrootkit |
| **Memory Dump** | 메모리 분석 | 커널 루트킷 탐지 | 복잡 | Volatility |
| **Boot Analysis** | 부트 섹터 검사 | 부트킷 탐지 | 시간 소요 | MBRCheck |

**과목 융합 관점**

1. **운영체제**: 커널 아키텍처, 시스템 콜, 모듈 로딩
2. **보안**: 침투 테스트, 포렌식, 탐지/대응
3. **네트워크**: 백도어 통신, C2 프로토콜

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: 루트킷 탐지**
```bash
# rkhunter 실행
sudo rkhunter --check --skip-keypress

# chkrootkit 실행
sudo chkrootkit

# 시스템 무결성 검사
sudo rpm -Va
```

**시나리오 2: 커널 모듈 검사**
```bash
# 로드된 모듈 확인
lsmod

# 의심스러운 모듈 확인
modinfo module_name
```

**도입 체크리스트**

**기술적 측면**
- [ ] 루트킷 탐지 도구 설치 (rkhunter, chkrootkit)
- [ ] 정기적 무결성 검사
- [ ] 커널 모듈 모니터링
- [ ] 부트 섹터 검사
- [ ] 네트워크 이상 탐지

**운영·보안적 측면**
- [ ] 최신 커널 유지
- [ ] 불필요한 모듈 비활성화
- [ ] 정기적 보안 업데이트
- [ ] 이벤트 로그 분석
- [ ] 침해 대응 절차 수립

**안티패턴**
1. **"관리자만 되면 돼"**: 루트킷도 관리자 권한 필요
2. **"백신으로 돼"**: 커널 루트킷 우회 가능
3. **"재설치하면 돼"**: 백업에서 재감염 가능

### 👶 어린이를 위한 3줄 비유 설명
1. **개념**: 루트킷은 투해 인간이 우리 집에 숨어드는 것과 같아요
2. **원리**: 컴퓨터의 모든 것을 보고, 조종하면서 자신은 아무도 못 보게 숨겨요
3. **효과**: 특별한 도구로 찾아내서 제거하지 않으면 계속 머물면서 정보를 훔쳐요
