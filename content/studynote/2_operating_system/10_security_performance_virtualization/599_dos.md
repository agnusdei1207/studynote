# DoS (Denial of Service) 및 DDoS (Distributed DoS)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템의 자원을 고갈시켜 정상 사용자의 서비스 이용을 불가능하게 만드는 공격
> 2. **가치**: 가용성(Availability) 침해 공격으로 랜섬웨와 결합하여 금전적 이득 추구
> 3. **융합**: 네트워크 대역폭, 시스템 자원, 애플리케이션 계층 등 다층면 공격 가능

---

### Ⅰ. 개요 (Context & Background)

**개념**
DoS(Denial of Service)는 **시스템의 자원(CPU, 메모리, 네트워크 대역폭 등)을 소진시켜 정상적인 서비스 제공을 불가능하게 만드는 공격**이다. DDoS(Distributed DoS)는 여러 대의 좀비 컴퓨터(Botnet)를 동원하여 분산적으로 공격하는 방식으로, 단일 DoS보다 훨씬 강력하고 탐지/방어가 어렵다.

**💡 비유**
DDoS는 **수도꼭(People)이 가게에 몰려드는 것**과 같다.
- 정상 손님: 물건 구매 (정상 트래픽)
- 몰려든 무리: 입구를 막음 (공격 트래픽)
- 결과: 진짜 손님 입장 불가

**등장 배경**
① **1990대**: 초기 DoS 공격 (Ping of Death, SYN Flood)
② **2000년대**: DDoS for Hire, 봇넷 비즈니스
③ **2010년대**: IoT 봇넷 (Mirai), 암호화폭탄
④ **현재**: AI 기반 공격, 1Tbps 이상 대규모 공격

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 공격 유형 | 대상 계층 | 공격 기법 | 영향 | 방어 기술 |
|:---|:---|:---|:---|:---|
| **Volumetric** | Network Layer | 대역폭 소진 | 회선 폭주 | Scrubbing, Mitigation |
| **Protocol** | Network/Transport | 프로토콜 취약점 | 연결 자원 고갈 | SYN Proxy, Rate Limit |
| **Application** | Application Layer | HTTP/SIPS 취약점 | 애플리케이션 장애 | WAF, API GW |
| **Amplification** | Network Layer | 반사 공격 | 소스 IP 위조 | uRPF, BCP 38 |

```
│  │                                                                     │    │
```

**해설**
1. **Volume-based**: 대역폭 포화
2. **Protocol-based**: 프로토콜 취약점 악용
3. **Application Layer**: 애플리케이션 취약점
4. **Amplification**: 반사 공격으로 증폭

**심층 동작 원리: SYN Flood 공격**

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                       SYN Flood 공격 메커니즘                                 │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  정상 TCP 3-way handshake:                                                 │
│  ┌───────────────────────────────────────────────────────────────────────┐ │

│  │     │                     │                                         │ │
│  │     │ ──── SYN ────▶     │ SYN_RCQ에 추가                     │ │
│  │     │                     │                                         │ │
│  │     │ ◀── SYN+ACK ──── │                                         │ │
│  │     │                     │                                         │ │
│  │     │ ──── ACK ────▶     │ 연결 확립                           │ │
│  │                                                                      │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  SYN Flood 공격 시:                                                        │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  공격자                서버                                      │ │
│  │     │                     │                                         │ │
│  │     │ ──── SYN ────▶     │ SYN_RCQ에 추가                     │ │

│  │     │                     │ (ACK를 보낼 곳 없음)                │ │
│  │     │ ──── SYN ────▶     │                                         │ │
│  │     │   (위조 IP)         │ SYN_RCQ 포화!                       │ │
│  │     │                     │                                         │ │
│  │     │ ──── SYN ────▶     │                                         │ │
│  │     │   (위조 IP)         │ → 연결 자원 고갈                     │ │
│  │                                                                      │ │

│  대응 기법:                                                                │
```

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나시오 1: DDoS 대응 절차**
```
1. 탐지: 트래픽 급증, 이상 패턴
2. 분석: 공격 유형, 소스 식별
3. 완화: 블랙홀, Rate Limit, Scrubbing
4. 모니터링: 추가 공격 감시
```

**시나리오 2: 예방 방법**
- **네트워크**: 여러 ISP, Anycast
- **시스템**: Auto-scaling, Load Balancing
- **애플리케이션**: Rate Limiting, Caching
