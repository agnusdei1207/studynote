# 스푸핑 (Spoofing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공격자가 합법적인 사용자/시스템인 것처럼 위장하여 신뢰를 획득하는 기법
> 2. **가치**: 피싱, MITM, DoS 등 다양한 공격의 기초이자 필수 요소
> 3. **융합**: IP, MAC, Email, DNS, Web 등 계층별 스푸핑과 방어 기술

---

### Ⅰ. 개요 (Context & Background)

**개념**
스푸핑(Spoofing)은 **데이터의 출처를 위조하여 다른 출처인 것처럼 위장하는 공격 기법**이다. 공격자는 합법적인 사용자, 신뢰할 수 있는 시스템, 또는 권한 있는 사용자인 것처럼 가장하여 보안 장치를 우회하거나 피해자의 신뢰를 얻는다. 유형에는 IP 스푸핑, MAC 스푸핑, 이메일 스푸핑, DNS 스푸핑, 웹 스푸핑 등이 있다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         스푸핑 기본 개념                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   정상 통신                                                                │
│   ────────────────                                                          │
│   ┌──────────┐                          ┌──────────┐                      │
│   │  사용자   │ ─────────────────────────▶│  서버   │                      │
│   │ IP: A     │   실제 IP: A                 │          │                      │
│   └──────────┘                          └──────────┘                      │
│                                                                             │
│   스푸핑 공격                                                               │
│   ────────────────                                                          │
│   ┌──────────┐                          ┌──────────┐                      │
│   │  공격자   │ ─────────────────────────▶│  서버   │                      │
│   │ IP: B     │   위조된 IP: A               │          │                      │
│   │(실제 IP)  │   (사용자인 척)             │          │                      │
│   └──────────┘                          └──────────┘                      │
│          │                                                                 │
│          ▼                                                                 │
│   서버는 사용자 A의 요청으로 인식                                       │
│   → 승인, 정보 노출, 공격 가능                                            │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│  │                        스푸핑 유형                                 │  │
│  ├─────────────────────────────────────────────────────────────────────┤  │
│  │                                                                     │  │
│  │  • IP 스푸핑: 발신자 IP 위조                                       │  │
│  │  • MAC 스푸핑: 네트워크 카드 주소 위조                             │  │
│  │  • 이메일 스푸핑: 발신자 이메일 위조                               │  │
│  │  • DNS 스푸핑: 도메인 이름 to IP 위조                              │  │
│  │  • 웹 스푸핑: 가짜 웹사이트                                     │  │
│  │  • ARP 스푸핑: MAC 주소 to IP 위조                                │  │
│  │                                                                     │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
스푸핑은 **가짜 신분증 위조**와 같다.
- 신분증: 진짜 신분 (합법 사용자)
- 위조: 가짜 신분증 제작 (스푸핑)
- 통과: 보안 초소 통과 (신뢰 획득)

**등장 배경**
① **1980대**: IP 스푸핑 등장
② **1990대**: Email 스푸핑, 피싱 시작
③ **2000대**: Pharming, DNS 스푸핑
④ **현재**: AI 기반 딥페이크

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 스푸핑 유형 | 대상 계층 | 위조 대상 | 주요 공격 | 방어 기술 |
|:---|:---|:---|:---|:---|
| **IP Spoofing** | Network Layer | 발신자 IP | DoS, MITM | Ingress Filtering |
| **MAC Spoofing** | Data Link | MAC 주소 | MITM | Port Security |
| **Email Spoofing** | Application | From 주소 | 피싱 | SPF/DKIM/DMARC |
| **DNS Spoofing** | Application | DNS 레코드 | Pharming | DNSSEC |
| **ARP Spoofing** | Data Link | ARP 테이블 | MITM | Static ARP |
| **Web Spoofing** | Application | URL | Pharming | HTTPS, EV SSL |

**해설**
1. **IP 스푸핑**: 네트워크 계층에서 발신자 IP 위조
2. **ARP 스푸핑**: MAC 주소로 IP 위조
3. **DNS 스푸핑**: 도메인 이름 resolution 위조
4. **이메일 스푸핑**: 발신자 이메일 위조

**심층 동작 원리: IP 스푸핑 구현**

│  ┌───────────────────────────────────────────────────────────────────────┐  │

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 스푸핑 방어 기술**

| 스푸핑 유형 | 방어 기술 | 구현 복잡도 | 효과 | 단점 |
|:---|:---|:---|:---|:---|
| **IP** | Ingress/Egress Filtering | 중간 | 높음 | 완전 차단 불가 |
| **MAC** | Port Security | 쉬움 | 높음 | 수동 설정 필요 |
| **Email** | SPF/DKIM/DMARC | 중간 | 중간 | 도입 부족 |
| **DNS** | DNSSEC | 어려움 | 높음 | 복잡함 |
| **ARP** | Static ARP | 쉬움 | 높음 | 확장성 문제 |

**과목 융합 관점**

1. **네트워크**: 프로토콜 스택, 라우팅
2. **보안**: 인증, 무결성
3. **애플리케이션**: 사용자 인터페이스

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: IP 스푸핑 방어**
- **Ingress Filtering**: 출구 필터링
- **uRPF (Unicast Reverse Path Forwarding)**: 라우팅 검증
- **BCP 38**: RFC 2827

**시나리오 2: Email 스푸핑 방어**
- **SPF(Sender Policy Framework)**
- **DKIM(DomainKeys Identified Mail)**
- **DMARC(Domain-based Message Authentication)**

**도입 체크리스트**

**기술적 측면**
- [ ] Ingress Filtering 구성
- [ ] Port Security 활성화
- [ ] SPF/DKIM/DMARC 도입
- [ ] DNSSEC 구현
- [ ] HTTPS 강제

**운영·보안적 측면**
- [ ] 사용자 교육
- [ ] 피싱 시뮬레이션
- [ ] 정책 수립

**안티패턴**
1. **"내부 네트워크는 안전"**: 내부 공격 가능
2. **"이메일 보낸 거 틀려"**: 위조 쉬움
3. **"암호화 있으면 됨"**: MITM 가능
