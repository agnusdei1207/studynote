# 신뢰 컴퓨팅 (Trusted Computing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 신뢰 컴퓨팅은 **하드웨어 루트 오브 트러스트(Root of Trust)**를 기반으로 시스템의 **부팅부터 실행 무결성**을 보장하고, **원격 증명**으로 상대방 시스템의 신뢰성을 검증한다.
> 2. **가치**: TPM(Trusted Platform Module) 칩으로 **디스크 암호화, 안전한 키 저장, 부팅 무결성**을 제공하며, 도난 방지, 데이터 보호에 활용된다.
> 3. **융합**: TCG(Trusted Computing Group) 표준, TPM 2.0, Secure Boot, Intel TXT, AMD SVM와 결합하여 **플랫폼 보안 체인**을 구축한다.

+++

## Ⅰ. 신뢰 컴퓨팅의 개요

### 1. 정의
- 신뢰 컴퓨팅은 시스템이 **신뢰할 수 있는 상태로 부팅**되고 실행되는 것을 보장하는 기술이다.

### 2. 등장 배경
- "내 PC는 정말 내가 통제하고 있나?"
- 맬웨어가 부팅 전부터 침투하는 위협

### 3. 💡 비유: '은행 금고'
- 신뢰 컴퓨팅은 **"은행 금고의 다중 잠금 장치"**와 같다.
- 여러 단계를 거쳐야만 열리며, 누가 언제 열었는지 기록된다.

- **📢 섹션 요약 비유**: 부동산 계약 때 공증인증을 받는 것처럼, 컴퓨터도 부팅할 때마다 "내가 진짜야"라고 증명하는 시스템입니다.

## Ⅱ. TPM (Trusted Platform Module) (Deep Dive)

### 1. TPM 구조
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                  TPM (Trusted Platform Module)                  │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [하드웨어]                                                     │
    │   - 별도 칩 (CPU와 독립)                                       │
    │   - 암호화 코프로세서                                          │
    │   - 보안 키 저장소                                             │
    │                                                                 │
    │  [주요 기능]                                                   │
    │   - 키 생성/저장 (추출 불가)                                   │
    │   - 디지털 서명                                                │
    │   - 난수 생성                                                 │
    │   - PCR (Platform Configuration Register)                      │
    │                                                                 │
    │  [용도]                                                        │
    │   - 디스크 암호화 (BitLocker, LUKS)                            │
    │   - 안전한 부팅 (Secure Boot)                                  │
    │   - 원격 증명 (Remote Attestation)                             │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. PCR (Platform Configuration Register)
- **부팅 과정의 해시를 저장하는 레지스터**
- 각 단계(UEFI, Bootloader, Kernel)가 실행될 때마다 해시 확장(Extend)
```text
PCR[0] = SHA256(PCR[0] || UEFI_Code_Hash)
PCR[1] = SHA256(PCR[1] || Bootloader_Hash)
...
```

## Ⅲ. 신뢰 부팅 체인

### 1. Measured Boot
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                  신뢰 부팅 체인                                 │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [CRTM (Core Root of Trust for Measurement)]                    │
    │   ──▶ UEFI Firmware (PCR[0])                                    │
    │       ──▶ Option ROMs (PCR[1~2])                               │
    │           ──▶ Bootloader (PCR[3~4])                            │
    │               ──▶ OS Kernel (PCR[5~8])                         │
    │                   ──▶ Application (PCR[9~23])                   │
    │                                                                 │
    │  * 각 단계마다 해시 측정(Measure)하여 PCR에 저장                │
    │  * PCR 값으로 무결성 검증 가능                                   │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. Secure Boot (UEFI)
- **서명된 부트로더만 실행 허용**
- 무단 수정된 부트로더 실행 방지

## Ⅳ. 원격 증명 (Remote Attestation)

### 1. 동작 과정
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                원격 증명 (Remote Attestation)                   │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   [Challenger]         [Attester]                               │
    │   (검증자)            (시스템)                                 │
    │        │                                                       │
    │        │ ① 증명 요청 + Nonce                                   │
    │        ├──────────────────────────────────────▶                │
    │        │                                                       │
    │        │                            │                           │
    │        │                            ▼                           │
    │        │                       [TPM]                           │
    │        │                       - PCR 값 읽기                   │
    │        │                       - TPM 키로 서명                │
    │        │                            │                           │
    │        │ ② PCR 값 + 서명 반환 ◀───────────────────────────┤     │
    │        │                                                       │
    │        │ ③ 검증 (PCR 값이 예상과 일치?)                        │
    │        │                                                       │
    │        ✅ 신뢰 가능 OR ❌ 의심                                 │
    │                                                                 │
    │  * 서버는 클라이언트가 무결 상태인지 확인                      │
    └─────────────────────────────────────────────────────────────────┘
```

## Ⅴ. TPM 활용

### 1. 디스크 암호화
- **BitLocker** (Windows): TPM + TPM 키
- **LUKS** (Linux): TPM으로 키 봉인

### 2. SSH 키 보호
```bash
# TPM으로 SSH 키 보호
ssh-keygen -t ecdsa-sk
```
