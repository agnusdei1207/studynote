# ROP (Return-Oriented Programming)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기존 실행 가능 코드의 작은 조각(Gadget)을 체이닝하여 새로운 악성 기능 수행
> 2. **가치**: NX/DEP로 인한 셸코드 실행 방지를 우회하는 가장 강력한 기법
> 3. **융합**: ASLR와 결합 시 정보 누출 필요, 정적 분석으로 탐지 어려움

---

### Ⅰ. 개요 (Context & Background)

**개념**
ROP(Return-Oriented Programming)는 **NX/DEP로 인해 스택/힙에서 코드 실행이 불가능할 때, 기존 실행 가능한 코드(라이브러리 등)의 작은 조각(Gadget)을 연결하여 원하는 공격을 수행하는 기법**이다. 각 Gadget는 "ret" 명령으로 끝나며, 이를 체이닝하여 마치 셸코드처럼 동작하게 만든다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         ROP 기본 개념                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   전통적인 셸코드 인젝션 (NX/DEP로 차단됨)                               │
│   ────────────────────────────────────────────────────────                 │
│   ┌─────────────────────────────────────────────────────────────────┐      │
│  │  [NOP Sled][Shellcode][NOP 주소][Return Address]                │      │
│  │   ↑                                                        │      │
│  │   └─ 스택에 있는 코드 실행 시도 → NX Bit에 의해 차단!           │      │
│  └───────────────────────────────────────────────────────────────────┘      │
│                                                                             │
│   ROP 공격 (NX/DEP 우회)                                                   │
│   ────────────────────────────────────────────────────────                 │
│   ┌─────────────────────────────────────────────────────────────────┐      │
│  │  [Gadget 1 주소][Gadget 2 주소][Gadget 3 주소]...[쉸 획득 Gadget]  │      │
│  │       │               │               │              │               │      │
│  │       ▼               ▼               ▼              ▼               │      │
│  │   ┌─────────┐    ┌─────────┐    ┌─────────┐   ┌─────────┐         │      │
│  │   │pop eax; │    │xor edx,  │    │mov [ecx]│   │system@  │         │      │
│  │   │ret      │    │edx; ret │    │eax; ret │   │plt      │         │      │
│  │   └─────────┘    └─────────┘    └─────────┘   └─────────┘         │      │
│  │                                                             │         │      │
│  │   기존 코드 조각들로                                           │         │      │
│  │   → 실행 가능 영역의 코드만 사용 (libc, executable 등)          │         │      │
│  │   → NX/DEP 우회!                                               │         │      │
│  │                                                             │         │      │
│  └───────────────────────────────────────────────────────────────────┘      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
ROP는 **레고 블록 조립**과 같다.
- 새로운 플라스틱(셸코드): 만들 수 없음 (NX)
- 기존 레고(Gadget): 해체했다가 재조립
- 원하는 형태(악성 기능): 조립해서 완성

**등장 배경**
① **2000년대 중반**: NX/DEP 보편으로 셸코드 차단
② **2007년 Hovav Shacham**: ROP 학술적 제시
③ **2010대**: 실제 공격에서 널리 사용
④ **현재**: 모던 익스플로잇의 필수 기법

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 요소 | 역할 | 위치 | 특징 | 비유 |
|:---|:---|:---|:---|:---|
| **Gadget** | 작은 코드 조각 | libc, executable | ret로 끝남 | 레고 블록 |
| **ROP Chain** | Gadget 연결 | 스택 | 순서대로 실행 | 조립 도면 |
| **Stack Pivot** | 스택 포인터 변경 | Gadget | ESP 조작 | 기반 작업 |
| **Gadget Finder** | Gadget 검색 도구 | rp++, ROPgadget | 자동화 | 부품 찾기 |

│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                    Gadget 예시 (x86-64)                              │    │
│  ├─────────────────────────────────────────────────────────────────────┤    │
│  │                                                                     │    │
│  │  libc.so에서 발견할 수 있는 일반적인 Gadget                           │    │
│  │                                                                     │    │
│  │  1. pop rdi; ret                                                  │    │
│  │     0x40058a: pop rdi; ret                                         │    │

│  │                                                                     │    │

│  │     0x4007b2: pop rsi; pop r15; ret                                 │    │

│  │                                                                     │    │
│  │  4. mov qword ptr [rax], rsi; ret                                  │    │
│  │     → 메모리 쓰기                                                    │    │

│  │  5. syscall; ret                                                   │    │

│  ├─────────────────────────────────────────────────────────────────────┤    │
│  │                                                                     │    │
│  │   목표: execve system call로 /bin/sh 실행                           │    │
│  │   rax = 59 (execve)                                                │    │
│  │   rdi = "/bin/sh" 주소                                             │    │
│  │   rsi = argv[] 주소 (NULL)                                          │    │
│  │   rdx = envp[] 주소 (NULL)                                          │    │
│  │                                                                     │    │
│  │   스택 구조:                                                        │    │
│  │   ┌───────────────────────────────────────────────────────────────┐ │    │
│  │   │ low                                                    high │ │    │
│  │   │                                                           │    │
│  │   │  ... (gadget chain)                                           │    │
│  │   │                                                           │    │
│  │   │  ├─ pop rdi; ret                                              │    │
│  │   │  │  "/bin/sh" 문자열 주소                                    │    │
│  │   │  │                                                           │    │
│  │   │  ├─ pop rsi; ret; pop r15; ret                                 │    │
│  │   │  │  argv 주소                                                  │    │
│  │   │  │  dummy (for r15)                                            │    │
│  │   │  │                                                           │    │
│  │   │  ├─ xor rdx, rdx; ret                                         │    │
│  │   │  │                                                           │    │
│  │   │  ├─ mov eax, 59; ret                                          │    │

│  │   │  │                                                           │    │

│  │   └───────────────────────────────────────────────────────────────┘ │    │
│  │                                                                     │    │
│  │   실행 흐름:                                                        │    │
│  │   1. 첫 Gadget("pop rdi; ret") 실행 → "/bin/sh" 주소를 rdi에 저장    │    │
│  │   2. 다음 Gadget("pop rsi; ...") 실행 → argv 주소를 rsi에 저장        │    │
│  │   3. ... 필요한 레지스터 설정                                       │    │
│  │   4. syscall Gadget 실행 → execve("/bin/sh", ...)                    │    │

│                                                                              │

**해설**
1. **Gadget**: ret로 끝나는 작은 코드 조각
2. **ROP Chain**: 스택에 순서대로 주소 배치
3. **실행**: 각 ret가 다음 Gadget로 점프

**심층 동작 원리: ROP 체이닝**

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                       ROP 공격 실행 과정                                      │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  ① Gadget 수집 (Gadget Finding)                                            │
│  ────────────────────────────────────────────────────────────────────────   │
│  • 대상 라이브러리 분석 (libc, ld-linux 등)                               │
│  • 도구: rp++, ROPgadget, ropeme                                          │
│  • 필요한 Gadget 식별: pop, mov, xor, syscall 등                           │
│                                                                              │
│  ② ROP Chain 구성                                                          │
│  ────────────────────────────────────────────────────────────────────────   │
│  • 목표 시스템 호출 구성                                                    │
│  • 레지스터 설정 순서 결정                                                │
│  • 스택 레이아웃 설계                                                        │
│                                                                              │
│  ③ 스택 피벗 (Stack Pivot)                                                 │
│  ────────────────────────────────────────────────────────────────────────   │
│  • ASLR가 있으면 libc의 정확한 주소 필요                                   │
│  • 정보 누출로 libc_base 주소 획득                                         │
│  • Stack Pivot Gadget로 esp 제어                                           │
│                                                                              │
│  ④ 공격 실행                                                                │
│  ────────────────────────────────────────────────────────────────────────   │
│  • 버퍼 오버플로우로 반환 주소를 첫 Gadget 주소로 덮어씀기                │
│  • 각 Gadget가 순차적으로 실행                                               │
│  • 최종적으로 원하는 시스템 호출 실행                                       │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 익스플로잇 기법 비교**

| 기법 | NX/DEP 환경 | ASLR 환경 | 난이도 | 탐지 용이성 |
|:---|:---|:---|:---|:---|
| **셸코드** | 차단됨 | 필요 없음 | 쉬움 | 쉬움 |
| **Return-to-libc** | 우회 가능 | 필요 | 중간 | 중간 |
| **ROP** | 우회 가능 | 필요 (정보 누출) | 어려움 | 어려움 |
| **JOP** | 우회 가능 | 필요 | 매우 어려움 | 매우 어려움 |

**과목 융합 관점**

1. **컴퓨터 구조**: 명령어 세트, 레지스터, 호출 규약

2. **역공학**: 디컴파일, Gadget 분석

3. **보안**: CFI(Control Flow Integrity)로 방어

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**

| 지표 | ROP 방어 전 | CFI 활성 | 개선 효과 |
|:---|:---|:---|:---|
