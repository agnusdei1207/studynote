# KVM (Kernel-based Virtual Machine)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: KVM은 **Linux 커널을 하이퍼바이저로 변환**하는 모듈로, Intel VT-x/AMD-V 하드웨어 가상화 지원을 활용하여 **전가상화**를 구현한다.
> 2. **가치**: Linux의 일부로 통합되어 커널 업데이트와 함께 발전하며, **QEMU, libvirt**와 결합하여 완전한 가상화 솔루션을 제공한다.
> 3. **융합**: KVM은 Linux 생태계(RHEL, Ubuntu, OpenStack)의 표준 가상화 기술이 되었으며, 클라우드(AWS, GCP)와 가상 데스크톱에 널리 사용된다.

+++

## Ⅰ. KVM의 개요

### 1. 정의
- KVM은 Linux 커널 모듈로, 커널 자체를 하이퍼바이저로 변환하여 가상머신을 실행한다.
- 2007년 Red Hat의 Avi Kivity 개발, Linux 2.6.20(2007)에 머지

### 2. 등장 배경
- 기존 가상화(VMware, Xen)는 별도 설치 필요
- Linux 커널에 내장된 가상화로 별도 하이퍼바이저 불필요

### 3. 💡 비유: '리눅스는 하이퍼바이저가 되다'
- KVM은 **"Linux 커널이 하이퍼바이저 역할까지 겸하는 것"**과 같다.
- OS인 동시에 다른 OS(VM)를 실행시키는 호스트가 된다.

- **📢 섹션 요약 비유**: 운전면허 시험장에 있는 교육용 차량처럼, 리눅스는 본인의 기능을 하면서도 다른 운전자(VM)를 훈련시킬 수 있는 이중 역할을 합니다.

### 1. KVM 아키텍처

    │                      KVM 아키텍처                               │

### 2. KVM 모듈
- **kvm.ko**: 코어 KVM 모듈 (Intel 또는 AMD 버전)
- **kvm-intel.ko / kvm-amd.ko**: CPU 벤더별 모듈
- **kvm-mmio**: MMIO 처리
```bash
# 모듈 확인
lsmod | grep kvm
# kvm_intel  327680  0
# kvm         860160  1 kvm_intel
```

### 3. /dev/kvm 디바이스
- KVM은 캐릭터 디바이스(/dev/kvm)로 인터페이스 제공
- ioctl 시스템 콜로 VM 생성, vCPU 실행, 메모리 매핑
```c
// 예시: KVM ioctls
kvm_fd = open("/dev/kvm", O_RDWR);
vm_fd = ioctl(kvm_fd, KVM_CREATE_VM, 0);
vcpu_fd = ioctl(vm_fd, KVM_CREATE_VCPU, 0);
```

- **📢 섹션 요약 비유**: KVM은 리눅스 커널 안에 숨겨진 "가상 머신 조종 장치"입니다. QEMU라는 조종사가 이 장치를 사용해 VM을 운전합니다.

## Ⅲ. KVM + QEMU 통합

### 1. QEMU의 역할
- **하드웨어 에뮬레이션**: BIOS, VGA, 네트워크 카드, 디스크 컨트롤러 등
- **사용자 인터페이스**: VM 생성/관리 CLI 제공
- **I/O 다중화**: 비동기 I/O 처리

### 2. virtio (가상 I/O 장치)
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                virtio: 반가상화 I/O 프레임워크                    │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   Guest OS                      Host                           │
    │  ┌─────────┐                  ┌─────────┐                     │
    │  │virtio   │  virtqueue      │  Host   │                     │
    │  │ Driver  │─────────────────▶│ Backend │                     │
    │  └────┬────┘  (shared memory) └────┬────┘                     │
    │       │                              │                         │
    │       │ ① 드라이버가 요청을 큐에 넣음   │                         │
    │       │ ② 백엔드가 처리                 │                         │
    │       │ ③ 인터럽트로 완료 알림          │                         │
    │       │                              │                         │
    │  [virtio-net]  네트워크 카드                                     │
    │  [virtio-blk]  블록 장치                                         │
    │  [virtio-scsi] SCSI 장치                                         │
    │  [virtio-balloon] 메모리 풍선                                     │
    │                                                                 │
    │  * 에뮬레이션 대비 가까운 네이티브 성능                          │
    └─────────────────────────────────────────────────────────────────┘
```

### 3. libvirt (관리 API)
- KVM/QEMU를 위한 **표준化管理 API** 및 CLI
- **virsh**: KVM 관리 CLI
- **XML 기반 설정**: VM을 XML로 정의

## Ⅳ. KVM 명령어 및 관리

### 1. virsh 명령어
```bash
# VM 목록
virsh list --all

# VM 시작
virsh start vm-name

# VM 중지
virsh shutdown vm-name

# VM 강제 중지
virsh destroy vm-name

# VM 콘솔 연결
virsh console vm-name

# VM 설정 편집
virsh edit vm-name

# 스냅샷
virsh snapshot-create-as vm-name --description "Before patch"
virsh snapshot-list vm-name
virsh snapshot-revert vm-name snapshot-name
```

### 2. qemu-kvm 직접 실행
```bash
# VM 생성 (간단 예시)
qemu-system-x86_64 \
  -m 2048 \
  -smp 2 \
  -drive file=ubuntu.qcow2,format=qcow2 \
  -net nic,model=virtio \
  -net user,hostfwd=tcp::2222-:22 \
  -enable-kvm
```

## Ⅴ. KVM 성능 최적화

### 1. vCPU Pinning
- vCPU를 특한 물리 코어에 고정 (CPU Cache 활용)
```xml
<vcpupin vcpu='0' cpuset='0-1'/>
<vcpupin vcpu='1' cpuset='2-3'/>
```

### 2. Huge Pages
- 2MB/1GB 페이지로 TLB Miss 감소
```bash
# Huge Page 할당
echo 1024 > /proc/sys/vm/nr_hugepages
```

### 3. CPU 모델 (host-passthrough)
```xml
<cpu mode='host-passthrough'/>
```
- 호스트 CPU 기능을 VM에 그대로 노출

## Ⅵ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. KVM 활용 분야
| 용도 | 설명 | 예시 |
|:---|:---|:---|
| **서버 가상화** | 데이터센터 서버 통합 | RHEL Virtualization |

| **VDI** | 가상 데스크톱 인프라 | Citrix VDI, VMware Horizon |
| **개발** | 로컬 개발 환경 | virt-manager, virsh |

### 2. 안티패턴
- **"오버커밋 과도"**: 물리 메모리 이상으로 VM 할당
- **"스냅샷 방치"**: 오래된 스냅샷으로 디스크 낭비
- **"CPU Passthrough 남용"**: 대부분의 경우 host-model 충분
