# 보안 부팅 (Secure Boot)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: UEFI firmware의 기능으로, 서명되지 않은 부트로더가 실행되는 것을 방지하는 하드웨어 보안 기술
> 2. **가치**: 부팅 단계에서 악성코드(Bootkit) 로딩을 방지하고 시스템 무결성 보장
> 3. **융합**: UEFI firmware, 인증서 체인, 서명된 부트로더, 커널의 연동 보안

---

### Ⅰ. 개요 (Context & Background)

**개념**
보안 부팅(Secure Boot)는 **UEFI(Unified Extensible Firmware Interface) 사양의 일부로, 부팅 과정에서 각 단계의 코드(부트로더, OS 커널, 드라이버)가 신뢰할 수 있는 인증서로 서명되었는지 검증하는 하드웨어 기반 보안 기술**이다. 2012년 Microsoft가 Windows 8 요구사항으로 도입했으며, 현재는 UEFI 표준의 필수 기능이다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         보안 부팅의 검증 체인                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   전원 ON                                                                   │
│     │                                                                       │
│     ▼                                                                       │
│   ┌─────────────────┐                                                      │
│   │   UEFI Firmware │                                                      │
│   │   (OEM 서명됨)  │                                                      │

│            │                                                               │
│            │ 1. 부트로더 서명 검증                                         │

│            │                                                               │
│            ▼                                                               │

│            │                                                               │
│            │ 2. 부트로더가 다음 단계 서명 검증                               │
│            │                                                               │
│            ▼                                                               │
│   ┌─────────────────┐                                                      │
│   │   OS Kernel     │                                                      │
│   │   (OS 서명됨)    │                                                      │
│   └────────┬────────┘                                                      │
│            │                                                               │
│            │ 3. 커널이 모듈 서명 검증                                       │
│            │   - (Linux에서 선택적)                                         │
│            │                                                               │
│            ▼                                                               │
│   ┌─────────────────┐                                                      │
│   │   Kernel Modules│                                                      │
│   │   (선택적 서명)  │                                                      │
│   └────────┬────────┘                                                      │
│            │                                                               │
│            ▼                                                               │
│        정상 부팅!                                                           │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  보안 부팅이 차단하는 것:                                           │  │
│   │  • 서명되지 않은 부트로더                                           │  │
│   │  • 변조된 UEFI firmware                                            │  │
│   │  • 악성 Bootloader (Bootkit)                                       │  │
│   │  • 위조된 커널                                                     │  │
│   │  • 악성 커널 모듈                                                   │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
보안 부팅은 **건물의 출입구 보안 검색**과 같다.
- 1층 경비: 신원증 확인 (UEFI firmware)
- 2층 경비: 사원증 확인 (부트로더)
- 3층 경비: 출입카드 확인 (커널)
- 목적: 무단 출입 방지

**등장 배경**
① **2010년**: UEFI 2.3.1에 보안 부팅 사양 추가
② **2012년**: Windows 8 하드웨어 요구사항
③ **2013년**: Linux shim 부트로더로 호환성 해결
④ **현재**: 대부분의 PC/서버 기본 활성화

**구성 요소**

| 구성 요소 | 역할 | 특징 |
|:---|:---|:---|
| **KEK (Key Exchange Key)** | DB/DBX 업데이트 키 | OEM/Microsoft 소유 |

**해설**
1. **PK**: 사용자가 플랫폼 제어, 보안 부팅 ON/OFF
2. **KEK**: DB/DBX 업데이트 키
3. **DB**: 신뢰하는 서명 (화이트리스트)
4. **DBX**: 거부된 서명 (블랙리스트)

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 보안 부팅 vs 레거시 BIOS**

| 특성 | 레거시 BIOS | UEFI 보안 부팅 |
|:---|:---|:---|
| **부트로더 검증** | 없음 | 서명 확인 |
| **악성 Bootkit** | 취약 | 방어 |
| **키 관리** | 없음 | PK/KEK/DB/DBX |
| **사용자 제어** | 제한적 | PK 소유 |
| **Linux 호환성** | shim 불필요 | shim 필요 |

**과목 융합 관점**

1. **하드웨어**: UEFI firmware, 마더보드
2. **운영체제**: 부트로더, 커널 서명
3. **암호학**: 공개키 인프라, X.509 인증서

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: 보안 부팅 확인**
```bash
# Linux에서 보안 부팅 상태 확인
sudo mokutil --sb-state

# EFI 변수 확인
sudo efivar -l | grep -i secure
```

**시나리오 2: 자체 서명된 커널 사용**
1. 자체 인증서 생성
2. MOK (Machine Owner Key) 등록
3. 커널/모듈 서명
4. 부팅 시 MOK 등록

**도입 체크리스트**

**기술적 측면**
- [ ] UEFI firmware 지원 확인
- [ ] 보안 부팅 활성화 (BIOS 설정)
- [ ] OS 지원 확인 (shim 등)
- [ ] 자체 서명 인프라 (필요시)
- [ ] DBX 업데이트 설정

**운영·보안적 측면**
- [ ] PK(Platform Key) 백업
- [ ] PK 소유자 교육
- [ ] MOK 관리 절차
- [ ] 인증서 만료 관리

**안티패턴**
1. **"항상 켜야 해"**: 개발 시 문제
2. **"기본값 OK"**: PK 소유권 필요
3. **"Linux 돼"**: shim, MOK 필요
