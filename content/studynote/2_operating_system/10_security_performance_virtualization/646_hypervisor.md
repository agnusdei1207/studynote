# 하이퍼바이저 (Hypervisor)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하이퍼바이저(Hypervisor, VMM)는 **물리 하드웨어 위에서 여러 개의 가상머신(VM)을 생성, 실행, 관리**하는 소프트웨어 계층이다.
> 2. **가치**: 하드웨어를 추상화하여 **하나의 물리 서버를 여러 논리 서버로 분리**하고, 리소스 격리, 마이그레이션, 리소스 공유를 가능하게 한다.
> 3. **융합**: Type 1(Bare Metal)과 Type 2(Hosted)로 분류되며, Intel VT-x/AMD-V 같은 하드웨어 지원과 결합하여 성능을 극대화한다.

+++

## Ⅰ. 하이퍼바이저의 개요

### 1. 정의
- 하이퍼바이저는 가상머신 모니터(VMM, Virtual Machine Monitor)라고도 하며, **여러 운영체제가 동시에 하나의 하드웨어에서 실행되도록 중개**한다.

### 2. 등장 배경
- 1960년 IBM CP/CMS에서 시작
- x86 하드웨어의 가상화 한계 극복 후 2000년대 대중화

### 3. 💡 비유: '건물 관리인'
- 하이퍼바이저는 **'대형 건물의 관리인'**과 같다.
- 각 세대(VM)에 자원을 배분하고, 문제 발생 시 수리하며, 새 세입자를 안내한다.

- **📢 섹션 요약 비유**: 하나의 큰 부엌(물리 하드웨어)에 여러 요리사(VM)가 동시에 요리를 할 수 있게 조율하는 주방장님입니다.

+++

## Ⅱ. 하이퍼바이저 유형 (Deep Dive)

### 1. Type 1 vs Type 2
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │           Type 1 vs Type 2 하이퍼바이저                          │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [Type 1: Bare Metal / Native]                                 │
    │                                                                 │
    │   ┌─────────────────────────────────────────────────────────┐  │
    │   │  VM1        │  VM2        │  VM3        │  VM4        │  │
    │   │  Guest OS   │  Guest OS   │  Guest OS   │  Guest OS   │  │
    │   └────────────┴────────────┴────────────┴────────────┘     │  │
    │                           │                                   │
    │   ┌───────────────────────┴───────────────────────────────┐  │
    │   │               Hypervisor (VMware ESXi, Xen)           │  │
    │   └───────────────────────┬───────────────────────────────┘  │
    │                           │                                   │
    │                           ▼                                   │
    │                      Hardware                                │
    │                                                                 │
    │   * 장점: 빠름, 안정적, 데이터센터용                              │
    │   * 예시: VMware ESXi, Microsoft Hyper-V, Xen, KVM             │
    │                                                                 │
    │  [Type 2: Hosted]                                             │
    │                                                                 │
    │   ┌─────────────────────────────────────────────────────────┐  │
    │   │  VM1        │  VM2        │  VM3                        │  │
    │   │  Guest OS   │  Guest OS   │  Guest OS                   │  │
    │   └────────────┴────────────┴────────────┘                 │  │
    │                           │                                   │
    │   ┌───────────────────────┴───────────────────────────────┐  │
    │   │            Hypervisor (VMware Workstation)            │  │
    │   ├───────────────────────────────────────────────────────┤  │
    │   │            Host OS (Windows, Linux, macOS)            │  │
    │   └───────────────────────┬───────────────────────────────┘  │
    │                           │                                   │
    │                           ▼                                   │
    │                      Hardware                                │
    │                                                                 │
    │   * 장점: 설치 쉬움, 개발용                                      │
    │   * 단점: Host OS 오버헤드                                    │
    │   * 예시: VMware Workstation, VirtualBox, Parallels           │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 상세 비교
| 특성 | Type 1 (Native) | Type 2 (Hosted) |
|:---|:---|:---|
| **설치 위치** | 하드웨어에 직접 | Host OS 위 |
| **성능** | 네이티브에 가까움 | Host OS 오버헤드 |
| **용도** | 서버, 데이터센터 | 개발, 테스트 |
| **관리** | 중앙 관리 도구 | 개인용 |
| **예시** | ESXi, Hyper-V, KVM | Workstation, VirtualBox |

- **📢 섹션 요약 비유**: Type 1은 "전문 주방장이 직접 주방을 관리", Type 2는 "가정 주부가 부엌에서 요리하며 아이도 돌보는" 형태입니다.

## Ⅲ. 하이퍼바이저의 핵심 기능

### 1. CPU 가상화
- **Time Slicing**: 각 VM에 CPU 시간 슬라이스 할당
- **vCPU**: 물리 코어를 논리적으로 분할하여 제공
- **Context Switching**: VM 간 전환 오버헤드 최소화

### 2. 메모리 가상화
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │              메모리 가상화: 2단계 매핑                            │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   Guest OS (가상 주소: VA)                                      │
    │      │                                                         │
    │      ▼ Guest Page Table                                        │
    │   Guest Physical Address (GPA): 가상 물리 주소                   │
    │      │                                                         │
    │      ▼ Shadow PT / EPT (Extended Page Tables)                  │
    │   Host Physical Address (HPA): 실제 물리 주소                    │
    │      │                                                         │
    │      ▼                                                         │
    │   Physical RAM                                                 │
    │                                                                 │
    │  * VA → GPA → PA 변환                                         │
    │  * 하드웨어 지원 (EPT/NPT)로 가속                               │
    └─────────────────────────────────────────────────────────────────┘
```

### 3. I/O 가상화
- **Emulation**: 소프트웨어로 장치 에뮬레이션 (느림)
- **Paravirtualization**: Guest OS가 하이퍼바이저에 I/O 요청 (빠름)
- **Passthrough**: 장치를 VM에 직접 할당 (가장 빠름)

## Ⅳ. 주요 하이퍼바이저 제품

### 1. VMware ESXi
- **Type 1**, 상용, 안정적
- vCenter로 중앙 관리
- **VMFS**: 클러스터 파일 시스템

### 2. Microsoft Hyper-V
- **Type 1** (Windows Server 역할)
- Windows生态系统 통합
- **Generation 1/2 VM**: BIOS/UEFI 지원

### 3. KVM (Kernel-based Virtual Machine)
- **Type 1**, Linux 커널 모듈
- QEMU와 함께 사용
- 오픈소스, Red Hat 기반

### 4. Xen
- **Type 1**, Cambridge 대학 개발
- 파라가상화 선구자
- AWS, Citrix XenServer 사용

## Ⅴ. 하이퍼바이저 vs 컨테이너

| 비교 항목 | 하이퍼바이저 (VM) | 컨테이너 |
|:---|:---|:---|
| **격리** | 전체 OS 격리 | 프로세스 격리 |
| **용량** | GB (OS 포함) | MB (앱만) |
| **시작 시간** | 분 | 초 |
| **성능** | 90~95% | 99%+ |
| **이식성** | 플랫폼 의존 | 플랫폼 독립 |
| **용도** | 완전한 격리 | 마이크로서비스 |

## Ⅵ. 실무 적용 및 결론

### 1. 선택 기준
- **Type 1**: 프로덕션 서버, 클라우드
- **Type 2**: 개발, 테스트, 데스크톱

### 2. 미래 전망
- **Hybrid**: VM + Container 병행
- **Nested Virtualization**: VM 안에서 VM

## 👶 어린이를 위한 3줄 비유 설명
1. 하이퍼바이저는 **"한 집에 여러 세입자(VM)를 모시는 집주인"** 같아요.
2. 각 세입자에게 방(자원)을 배분하고, 전기/수도 요금을 정리하고, 수리도 해줘요.
3. 집주님 덕분에 여러 세입자가 한 집에서 알차게 살 수 있답니다!
