# 시스템 보안 위협 유형 - 해커가 전 세계망을 타격하는 4가지 본질적 무기 (CIA Triad 파괴 렌더)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 해커가 아무리 화려한 코드를 짜도, 그 궁극적 목표는 결국 보안 3요소(CIA Triad) 중 하나를 부수는 것이다. 내 비밀을 훔쳐보는 **기밀성 침해(Interception 가로채기)**, 내 장부를 거짓으로 조작하는 **무결성 침해(Modification 변조 빔!)**, 그리고 서버 시스템을 뻗게 만들어 장사를 못 하게 하는 **가용성 침해(Interruption 중단/DDoS 랙!)** 가 3대 본질이다.
> 2. **가치**: 이 **분석 프레임워크(Security Threat Modeling 빔)** 덕분에, OS 커널 보안 관리자는 제로 데이(Zero-Day) 공격이 터져도 당황하지 않고 "이건 가용성 타격(DDoS) 이니 L4 로드밸런서로 막자!", "이건 기밀성 타격(스니핑) 이니 TLS 통신 암호화로 막자!" 며 $O(1)$ 즉각 대응 패러다임을 이륙시켰다 포팅.
> 3. **한계**: 가장 끔찍한 사회 공학(Social Engineering) 앞의 무력화 딜레마. 아무리 CIA 3원칙을 수학적(암호학) 으로 완벽히 지켜내는 서버를 구축해 놔도, 관리자가 보이스피싱에 속아 해커한테 2차 인증 OTP 번호를 카톡으로 넘겨줘 버리면(인증 위조 Fabrication 침해의 끝판왕!), 우주방어 커널이 정당한 주인님으로 착각해 문을 싹 다 열어버리는 아키텍처 외부의 휴먼 에러 수렁을 영원히 안고 있다 결착.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 
  - **보안의 파편화 늪 (무지성 방어 파단)**: 과거엔 "바이러스 막아!", "디도스 막아!" 처럼 땜질식 방어만 했다. 해커가 통신선 중간에서 패킷을 엿보기(스니핑) 만 하는데, 백신(수정 변조 방어) 만 미친 듯이 돌리며 정작 진짜 유출 구멍은 못 막던 맹목적 참사.
  - **CIA Triad 분류 모델 통달 (3원칙 프레임워크 빔!)**: 적의 무기를 3+1 등분으로 해체하라! 보안은 **1. 기밀성(Confidentiality: 못 보게), 2. 무결성(Integrity: 못 고치게), 3. 가용성(Availability: 안 죽게)** 이라는 3개의 기둥(Triad) 과, 덤으로 **4. 인증성(Authenticity: 너 진짜니?)** 까지 4개의 대들보로 나뉜다. 적의 공격이 어느 대들보를 부수려 하는지 논리적으로 진단하는 위협 조율이다 컷 스왑.
- **필요성**: 수백만 개의 분산 마이크로서비스(MSA) 와 쿠버네티스 아키텍처에서 방어벽 비용은 무한하지 않다. 은행은 [무결성] 공격 방어에 예산을 몰빵해야 하고, 유튜브는 [가용성] 방어에 목숨을 걸어야 한다. 이처럼 비즈니스 환경에 따라 타격점(Threat Surface) 의 우선순위를 배분하기 위해 "CIA 타격 분류법" 이 21세기에 필연적으로 멱살 부합 요구되었다 증명 록보장.

- **💡 비유**: 시스템 위협 4유형 뷰는 중세 시대 마차 금화 운송의 **"도적이 금화를 훔칠까(기밀성)? 훔쳐 내고 돌을 채워 넣을까(무결성)? 마차 바퀴를 부술까(가용성)? 아예 자기가 왕실 기사로 변장할까(인증 위조)? 에 따른 각각의 대비책 마스킹 락백!!"** 이랑 100% 동일 오류 제어율입니다!! 
  - (기밀성 유출): 마차가 숲을 지날 때 닌자가 몰래 숨어서 엑스레이로 박스 안의 금화 개수를 스캔합니다! 
  - (무결성 변조): 마부가 조는 사이 도적이 몰래 상자를 열어 금화 100개를 빼고 납덩이 100개를 채워 넣습니다!
  - (가용성 파괴): 도적 1만 명이 도로에 드러누워 마차가 통과를 못 하고 멈춰버립니다(DDoS 에러!)
  - **(인증 위조 조작 기전!)**: 똑똑한 해커는 힘을 쓰지 않습니다 스왑! 왕실 공식 인장 장부를 위조(가짜 신분증 빔!) 해서 마부에게 당당히 보여주면, 마부가 "오 장군님 오셨습니까" 하고 금화를 0.1초 만에 직접 수레로 옮겨 실어주는 무적 통달(Fabrication 파괴) 을 완성합니다 결속!

- **네트워크 흐름 기반 4가지 위협 벡터 ASCII 폭쇄 뷰**:
송신자(Sender) 와 수신자(Receiver) 파이프라인 정중앙에서, 해커(Attacker) 가 어떤 짓을 벌이는지 렌더 체계를 까보면 다음과 같다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────────────┐
  │                 "통신 파이프라인의 허리를 자를 것인가, 엿볼 것인가, 똥을 쌀 것인가!" │
  ├──────────────────────────────────────────────────────────────────────────────────────┤
  │                                                                                      │
  │  🚨 [ 정상 통신 (Normal) ] :  Sender ──────(데이터)──────▶ Receiver                  │
  │                                                                                      │
  │  =========================▼===================================                       │
  │                                                                                      │
  │  🔥 [ CIA + 인증 타격 4가지 렌더 록백 ❗ ]                                           │
  │                                                                                      │
  │   1. 가로채기 (Interception) ---> [ 💥 기밀성 (Confidentiality) 침해! ]              │
  │      Sender ────(데이터)─────▶ Receiver                                              │
  │                   └─몰래 훔쳐봄─▶ (해커)    (스니핑 Sniffing)                        │
  │                                                                                      │
  │   2. 흐름 차단 (Interruption) ---> [ 💥 가용성 (Availability) 침해! ]                │
  │      Sender ────(데이터를 쐈는데 길막!) ❌❌❌  Receiver (응답불능 랙!)              │
  │      (해커가 랜섬웨어로 파일 삭제 / 또는 DDoS 공격으로 입구 폭파)                    │
  │                                                                                      │
  │   3. 변조 조작 (Modification) ---> [ 💥 무결성 (Integrity) 침해! ]                   │
  │      Sender ──(정상 데이터)──▶ (해커가 스틸!) ──(거짓 데이터로 수정)──▶ Receiver     │
  │      (계좌 송금액 1억을 100억으로 중간 수정)                                         │
  │                                                                                      │
  │   4. 위조 창조 (Fabrication) ---> [ 💥 인증 (Authenticity) 침해! ]                   │
  │      Sender (진짜는 자고 있음)                                                       │
  │      (해커 변장) ──(가짜 신분증으로 내가 Sender요! 하며 데이터 살포!)─▶ Receiver     │
  └──────────────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 이 4가지 그림은 보안 기사뿐만 아니라 현대 클라우드 WAF(웹 방화벽) 설계의 코어 뼈대다. 각 화살표 끊김 현상마다 TLS(가로채기 방어), 데이터 해시(변조 방어), 다중화 LB(가용성 방어), IAM 2FA 인증(위조 방어) 이라는 카운터 아키텍처가 1:1 로 매핑되어 대응하는 논리적 대결 궤적이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 트레이드오프 전선 종결: 가용성(Availability) vs 기밀/무결성의 끝없는 제로섬 환각
보안을 세게 결속하면 유저가 못 쓰는(가용성 하락) 딜레마의 SRE 최전선 타결.

| CIA 우선순위 아키텍처 뷰 | ✨ 군대/정보 기관 (기밀성 몰빵 늪) | 🔥 넷플릭스/이커머스 (가용성 몰빵 록백) |
|:---|:---|:---|
| **가장 혐오하는 위협 벡터 빔** | 파일이 적국에 노출되는 **기밀성 침해(Interception 가로채기) 를 가장 두려워함.** | 고객 앱이 안 켜져서 서버가 터지는 **가용성 침해(Interruption DDoS 방해) 가 가장 큰 죄악.** |
| **방어 아키텍처 설정 구조 랙** | "조금이라도 이상한 패킷이 들어오면 서버 전체를 셧다운 시켜!" **(망분리 및 강제 차단 스왑)** | "해킹 좀 당해도 되니까 일단 결제 버튼은 무조건 띄워라!" **(지연없는 캐싱/다중화 라우팅 방패)** |
| **태생적 Trade-off 멸망 스왑** | 완벽한 보안을 얻었지만, 일상 업무 결재 한 번 하려면 **VPN 로그인 10번 하는 미친 사용성 불편 아크.** | 편하게 클릭클릭 하지만 결국 언젠가 털려 **천문학적 개인정보 유출 과징금을 두드려 맞는 파단.** |

### 2. 치명적 보안 모순: 분산 서비스 거부 공격(DDoS) 과 합법적 패킷의 구별 불가능 학살 사건
기밀성/무결성은 암호학(수학) 으로 100% 방어 수학 증명이 가능하지만, 왜 가용성(DDoS) 방관은 막을 수 없는 현상인지 해석한다.

- **안티패턴 오염 발생 미스터리 (가용성 Interruption 폭주 랙)**: 
  - (태생적 SYN Flood 파단 스왑): 해커가 10만 대의 좀비 PC 로 쇼핑몰 서버 문짝을 쉼 없이 두드린다! "나 손님인데 접속 좀~" 
  - (환상 브레이크 구분 불능 빔 발동!): 앗! 이 패킷(TCP SYN) 은 문법적으로 전혀! 1% 도 틀린 곳이 없는 '합법적이고 정상적인 고객 패킷' 이다! 해커가 비밀번호를 깬 것도, 암호를 푼 것도 아니다. 그냥 **"존나 많이" 보냈을 뿐이다!**
  - 파멸 결과: 수학적으로 이걸 쳐낼 명분(무결성 서명 오류 등) 이 0이기에 방화벽이 속수무책 다 받아주다가 서버 RAM 과 소켓 FD 자원이 100% 터져버려(OOM), 어떠한 손님도 클릭을 못하는 타임아웃 멸망(Denial of Service 가용성 박살 랙) 참사가 발생했다 증명 록보장.
- **SRE 극복 솔루션 패치 타결 조율 (CDN Anycast 스크러빙 센터 흡수 록백!!) / 자율 치유 방패**: 
  - 클라우드 아키텍트 타격!: "이건 소프트웨어로 못 막어! 하드웨어 인프라 물량전으로 받아내라 쾅!" 
  - 갓기능 마스킹 조합 스왑: Cloudflare 같은 CDN 업체가 전 세계망에 **초대형 패킷 흡수 진공청소기(Scrubbing Center)** 를 깐다. 좀비 PC 의 무한 클릭 패킷을 BGP Anycast 라우팅으로 전 세계 100개 노드로 찢어서($O(1/100)$ 방어!) 대신 맞아준다. 서버 본체 IP를 숨기고, 쓰레기 패킷과 정상 패킷을 AI 핑거프린팅으로 체에 걸러(Scrubbing) 맑은 물만 Origin 서버로 흘려보내는 인프라 스케일 레이저 도출해 낸다 확인. (가용성은 오직 자금력(Scalability) 으로 덮는 아키텍처!)

## Ⅲ. 실무 융합 적용 및 안티패턴 (ARP 스푸핑 맨인더미들 공격 통달과 TLS 인증 렌더)

### 해커가 "가로채기 + 변조 + 위조" 3콤보를 동시에 터트리는 최악의 사내망 해킹 폭파 뷰.
중간자 공격(Man in the Middle) 이 어떻게 무결성과 기밀성을 동시에 요리조리 박살 내는지 스왑 기전.

- **안티패턴 충돌 (카페 와이파이에서의 ARP Spoofing 스니핑/변조 타임아웃 랙)**: 
  - 상황: 철수가 스벅 와이파이에 접속해서 은행 사이트에 송금을 누른다. 그런데 옆자리에 있던 해커가 와이파이망에 ARP 속임수 라우팅(내가 공유기다!) 을 날렸다(위조 Fabrication 인증 침해!). 
  - 재앙 터짐: 철수의 핸드폰은 해커 노트북을 공유기로 착각하고 모든 송금 패킷을 해커한테 전달! 공격자는 그 패킷을 몰래 까서(가로채기 Interception 기밀성 털림!) 철수 통장의 남은 잔액을 보고 미소를 지은 뒤, 송금 계좌를 자기 통장으로 바꾼 뒤(변조 Modification 무결성 파단!) 진짜 은행 서버로 전송한다!
  - 충격 멸망: 철수는 자기가 정상 송금했다고 믿지만, CIA 3원칙 중 무려 3개가 박살 나는($O(N)$ 병목 퍼포먼스 붕괴) 퍼펙트 속임수에 계좌가 0원이 되는 빚 돌려 막기 파단.
- **SRE 엔지니어 도축 솔루션 (종단 간 암호화 TLS 1.3 과 인증서 서명 체인 빔!)**: 
  - 네트워크 리드 아키텍트 마스킹 발사!: 카페 와이파이는 원래 믿지 마라! 무조건 핸드폰 앱과 서버 단 양끝을 터널(Tunneling) 로 용접해라!
  - 운영 방검복 스왑: 철수가 송금을 누르는 순간 핸드폰에서 은행 서버의 공개키로 패킷을 $O(1)$ 초고속 암호화(RSA/ECC) 해버린다. (TLS 구간 암호화!) 해커가 중간에서 가로채도 복호화 키가 없어서 텍스트가 깨져서 못 보고(기밀성 복구!), 1비트라도 임의로 조작하면 도착지 서버에서 MAC 해시값 검증 실패로 패킷이 즉각 드랍(무결성 방어!) 되는 현대 인증 마스킹 생존 결속이다 증명 예고 컷.

## Ⅳ. 기대효과 및 결론
- '시스템 보안 위협 분해 (`CIA + Authenticity` 4대 분류 타격 렌더)' 프레임워크는 유닉스 50년의 역사에서 쏟아지는 수만 가지의 난해한 해킹 기법들(Ransomware, SQL Injection, XSS) 을 단 4개의 뼈대 상자(기밀/무결/가용/인증) 로 모조리 귀결시킴으로써 보안 대응 지침(Incident Response) 의 절대적 수학 나침반 뼈대다.

- 비록 "서버가 디도스를 맞아 뻗어버리는 가용성(Availability 파단 타임아웃 랙)" 은 소프트웨어의 로직(무결성/기밀성) 이 아무리 완벽해도 인터넷이라는 태생적 개방 인프라 앞에선 물량 공세를 막을 수 없다는 태생적 네트워크 한계 트레이드오프 레이턴시 파단을 낳았지만, 이를 클라우드 CDN 과 오토스케일링(Auto Scaling 물량 방어 마스킹) 우회망으로 완벽히 병합해내며 차세대 무결점 탄력적 방어구역 진화판으로 영원히 록백 보장.
