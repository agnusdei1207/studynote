# 임의적 접근 제어 (DAC) - 파일의 "소유자(Owner)" 가 신(God)이 되어 자기 맘대로 권력을 나눠주고 뺏는 자율 통치 렌더

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 앞선 장들(ACL, Capability) 이 "어떻게 권한을 적어둘까?" 라는 구현 방식이라면, **DAC (임의적 접근 제어 Discretionary 빔!)** 는 "도대체 누가 그 권한에 이름을 적을 자격을 갖는가?" 라는 철학이다. DAC 는 오직 **데이터의 소유자(Owner) 에게 자신의 파일을 보호하고 타인에게 열어줄 100% 의 자유재량(Discretion) 을 위임하는** 개인주의 방어 체제다.
> 2. **가치**: 이 **소유자 기반 자유 튜닝(Owner-based Security 빔)** 덕분에, A유저가 B유저에게 문서를 공유할 때마다 시스템 관리자(Root) 한테 결재 서류를 올리지 않고 자기가 직접 `chmod` 나 우클릭으로 권한을 즉시 양도하며, 전 세계 모든 윈도우/리눅스 개인용 PC 의 초고속 협업망을 이륙시켰다 포팅.
> 3. **한계**: 가장 끔찍한 트로이 목마(Trojan Horse) 와 바이러스 연쇄 폭파 딜레마. 유저가 파일의 유일한 신이므로, 유저가 미쳐서 자기 모든 파일 권한을 [777 빔] 으로 열어버리면 OS 는 그걸 막을 길이 없다. 특히 유저가 실수로 악성코드(트로이 목마) 를 실행하면, 그 악성코드는 유저의 권한을 고스란히 훔쳐서 시스템 내 유저의 모든 문서를 파괴하고 털어버리는(통제 상실 늪!) 치명적 책임 방기 수렁을 영원히 안고 있다 결착.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 
  - **중앙 통제의 스루풋 늪 (결재 대기 병목 파단)**: 파일이 1만 개일 때마다 보안 관리자 한 명이 "A가 이거 만들어도 돼?", "B가 이거 읽어도 돼?" 다 결재해주며 통제하는 원시 시대. 회사가 커지자 보안팀이 과로사로 죽고 협업 스피드가 0 에 수렴해버린 참사.
  - **DAC 통달 (개인 정보 통치 빔!)**: 너희들 파일은 너희들이 알아서 지켜라! 파일을 최초로 생성한(Create) 자를 **소유자(Owner)** 로 임명하고, 소유자는 자기 파일에 붙은 ACL 에 남의 이름표를 써주거나(Grant), 찢거나(Revoke) 할 무한대의 권력을 거머쥔다. 철저한 탈중앙(Decentralized) 자율 방어 조율이다 컷 스왑.
- **필요성**: 수천 명이 접속하는 유닉스(Unix) 다중 접속 환경에서, 연구생 A가 동기 B에게만 리포트를 공유하고 교수님한테 제출하려면 "이 파일의 공개 범위를 유저 스스로 마음대로(Discretionary) 바꿀 수 있어야" 했다. 중앙 보안의 개입 없이 물 흐르듯 유연한 파일 시스템 확장성을 유지하기 위해 21세기에 필연적으로 멱살 부합 요구되었다 증명 록보장.

- **💡 비유**: 임의적 제어(DAC) 뷰는 대학교 사물함 보안의 **"사물함 열쇠를 경비 아저씨가 다 들고 매번 허락받기 늪 VS 학생마다 자기 사물함 열쇠 들고 자기 맘대로 친구한테 빌려주기 락백!!"** 이랑 100% 동일 오류 제어율입니다!! 
  - (중앙 통제 늪): 철수가 자기 사물함에서 책을 영희에게 빌려주려 합니다. 철수와 영희는 손잡고 경비실(시스템 관리자 루트 빔!) 에 가서 사유서를 쓰고 서명한 뒤에야 경비 아저씨가 열쇠로 사물함을 열어줍니다. 귀찮아서 책을 안 빌립니다 멸망 에러!
  - **(DAC 소유자 자율 기전!)**: 똑똑한 대학교는 철수 사물함 열쇠(Owner 권력!) 를 그냥 철수 주머니에 줘버립니다 스왑! 철수는 자기 맘대로 열쇠 복사해서 영희 주고 동수 주고(임의적 권한 위임) 쌩쑈를 해도 학교는 관여 안 합니다! 학생들끼리의 거대한 노트 공유 인프라(자율 협업망) 가 무적 통달 파이프를 완성합니다 결속!

- **DAC 에서 소유권(Owner) 에 의해 권한이 임의로 흐르는 ASCII 폭쇄 뷰**:
관리자 개입 없이 소유자 스스로가 ACL 을 조작하며 권한을 전파하는 통치 렌더 체계를 까보면 다음과 같다.

```text
  ┌─────────────────────────────────────────────────────────────────┐
  │                 "내 파일은 내 맘대로! ACL 은 소유자인 내 손끝에서 바뀐다!" │
  ├─────────────────────────────────────────────────────────────────┤
  │                                                                 │
  │  🚨 [ 유저 A가 '비밀문서.txt' 를 최초 생성 (Owner 권력 획득!) ]                │
  │     [ 비밀문서.txt ACL 장부 ]                                            │
  │      => < User A (Owner) : Read, Write, 👑Grant(권한위임) >           │
  │                                                                 │
  │  =========================▼===================================  │
  │                                                                 │
  │  🔥 [ DAC 임의 방임 스왑 - 소유자의 마음대로 권력이 파생되는 록백 ❗ ]            │
  │                                                                 │
  │     (유저 A 가 유저 B 한테 같이 작업하자며 ACL 에 B이름을 추가함!)                 │
  │     [ 업데이트된 비밀문서.txt ACL 장부 ]                                  │
  │      => < User A (Owner) : Read, Write, Grant >                 │
  │      => < User B         : Read, Write >                        │
  │                                                                 │
  │  ✅ [ OS 커널 VFS 판결 심연 크래시 빔! ]                                  │
  │     (악성코드 터짐!) 유저 A가 무심코 해킹 프로그램을 더블클릭 실행!                 │
  │     -> 악성 프로그램은 [User A] 의 탈을 쓰고 작동하며, User A 가 Owner 인 모든     │
  │     문서를 지 맘대로 지우고 훔치고 777로 바꿔버림! (커널: "어 주인이 하네? 통과~" 멸망)│
  └─────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 전통적 리눅스의 `chmod`(파일 권한 변경) 명령어 체제가 바로 가장 완벽한 DAC 의 실사판이다. 자신이 소유자(Owner) 인 파일이라면 언제든지 `chmod 777` 로 세상 모든 사람에게 읽기/쓰기를 풀어버리는(Grant) 자살 행위도 OS 가 막지 않는다. 자율성 뒤에 숨은 무거운 책임의 뼈대다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 트레이드오프 전선 종결: 자율 분산 방어(DAC) vs 권력 통제 파탄 ви상 환각
DAC 이 현대 B2C 환경(윈도우 PC, 맥os) 을 영원히 평정했지만 군대 기밀망에서는 서지 못하는 최전선 타결.

| 접근 제어 철학 아키텍처 뷰 | ✨ 임의적 제어 (DAC 자유 통치 빔) | 🔥 강제적 제어 (MAC 직후 579장 대치 늪) |
|:---|:---|:---|
| **파일 권한 결정의 절대 권력자** | 파일의 **[생성자(Owner 유저)]** 가 파일에 대한 생사여탈권과 ACL 수정 권력을 100% 영구 독점함. | **[시스템(커널 보안 정책)]** 만이 룰을 갖고, 유저는 지가 만든 파일 지 맘대로 못 여는 철벽 통치. |
| **운영 유연성 (Flexibility) 랙** | "김대리, 이거 잠깐 봐봐" **1초 만에 유저 손끝 파단에서 실무 협업이 쏟아져 나오는 속도 아크.** | 김대리 보게 하려고 시스템 관리자한테 "권한 티어 상향 서류" 내느라 **업무가 마비되는 멸망.** |
| **보안 사고의 치명적 책임 스왑** | 멍청한 유저가 비번 털리거나 실수하면 **그 유저 파일과 시스템 일부분이 즉각 포맷 초토화 붕괴 늪.** | 유저가 멍청해도 **시스템이 강압 룰셋으로 틀어막아 악성코드를 모래상자 밖으로 못 나가게 결속 방패.** |

### 2. 치명적 보안 모순: 트로이 목마(Trojan Horse) 의 기생과 혼동된 대리인(Confused Deputy) 학살 사건
DAC 철학이 "프로그램이 유저의 탈을 쓰고" 공격할 때 커널이 완전히 장님이 되어버리는 파멸 현상을 해석한다.

- **안티패턴 오염 발생 미스터리 (Trojan Horse 기생 파괴 폭주 랙)**: 
  - (태생적 대리 실행 파단 스왑): DAC 의 핵심 룰! "프로그램을 실행한 자 의 권한으로 프로그램이 돈다!". 내가 크롬() 을 켜면 크롬 프로세스는 나의 [User Domain] 권력 리스트를 통째로 상속받아 내 파일을 다 읽을 수 있다. 
  - (환상 브레이크 기생충 빔 발동!): 앗! 회사 게시판에서 "귀여운 강아지.exe (트로이 목마!)" 를 무심코 눌렀다! 
  - 파멸 결과: 강아지 그림이 뜨는 동안, 백그라운드에서 이 악성코드는 "나를 누른 유저" 의 권한으로 작동한다! 즉 내가 내 소중한 파일들의 DAC Owner 권력을 쥐고 있으니, 이 악성 파리 새끼도 내 Owner 권력을 복사해서 들고 내 모든 기밀 문서를 압축해 해커 랜섬웨어 서버로 전송해 버린다. 커널은 "응? 주인이 자기 파일 읽는 거네? 허가 O" 라며 하이패스 범죄를 허용해 버리는 최악의 백도어 참사가 증명 록보장.
- **SRE 극복 솔루션 패치 타결 조율 (DAC 의 태생적 한계 인정과 샌드박스/MAC 수혈 록백!!) / 자율 치유 방패**: 
  - SRE 보안 아키텍트 타격!: "유저를 믿지 마라! 이놈의 자율 통제는 악성코드에 너무 무방비다!" 
  - 갓기능 마스킹 스왑: 결국 DAC 철학 하나만으로는 모바일과 기업 시스템을 지킬 수 없다 판단. 안드로이드와 윈도우는 DAC 바탕(내 파일은 내 것) 위에, MAC(프로세스는 유저 권력을 100% 다 가지지 못함 샌드박싱 빔!) 을 이중으로 덧발라 강아지.exe 가 내 폴더 밖으로 못 나가게 결박하는 융합(Hybrid) 체제로 도출해 낸다 확인.

## Ⅲ. 실무 융합 적용 및 안티패턴 (카카오톡 파일 전송과 chown 권력 양도 렌더)

### 리눅스에서 루트(Root) 조차 귀찮아하는 소유권(Owner) 변경 로직이 DAC 의 무서움을 증명한 뷰.
리눅스 서버에서 파일 주인(Ownership) 이 넘어갈 때 벌어지는 DAC 의 딜레마 스왑 기전.

- **안티패턴 충돌 (내가 만든 파일을 남한테 강제로 떠넘겨 용량 독박 씌우기 타임아웃 랙)**: 
  - 상황: 리눅스 서버에서 A유저가 1테라 용량의 `쓰레기_야동.mp4` 파일을 만들었다(A가 Owner). 그런데 하드 용량 쿼터(Quota) 할당량 초과로 관리자한테 욕먹게 생겼다. 
  - 재앙 터짐: A유저가 꾀를 부린다! "이 파일의 Owner 를 멍청한 B유저로 줘버리고 도망가야지!" 전통적인 DAC 에서는 `chown B유저 쓰레기.mp4` 를 쳐서 내 파일의 소유권을 B 동료한테 짬처리 시킬 수 있었다!
  - 충격 멸망: B유저는 다음 날 출근해 보니 자기도 모르는 1테라 파일의 '주인' 이 되어 있었고, 쿼터 위반으로 계정이 폭파당했다! 이처럼 소유권(Owner) 이라는 DAC 의 핵심 뼈대가 너무 자유롭게 이동하면 남을 암살하는 버그로 변질되는 빚 돌려 막기 파단.
- **SRE 엔지니어 도축 솔루션 (소유권 변경(chown) 의 엄격한 루트(Root) 통제 빔!)**: 
  - 리눅스 아키텍트 마스킹 발사!: 유저끼리 소유권(Owner) 떠넘기기를 일절 금지한다 쾅!
  - 운영 방검복 스왑: 최신 리눅스 / 최신 운영체제는 아무리 내가 파일의 주인(Owner) 이라도, 내 파일의 권한(ACL) 만 고칠 수 있을 뿐 내 파일의 "주인 자리(Ownership)" 를 타인에게 양도(`chown`) 하는 건 오직 시스템 최고 신(Root) 1명만 할 수 있도록 DAC 자유도에 거대한 족쇄($O(1)$ 강제 제어 락) 를 채워 공격을 틀어막는 생존 결속이다 증명 예고 컷.

## Ⅳ. 기대효과 및 결론
- '임의적 접근 제어 (`DAC` 파일 생성자 오너 자율 통치 렌더)' 아키텍처는 거대한 운영체제 생태계에서 "나의 정보 재산은 내가 온전히 통제한다" 는 철학을 박아넣어 중앙 집중형 결재 병목(Admin Bottleneck) 의 극악한 지연을 박살 내고, 즉각적이고 유연한 소규모 자율 협업의 속도를 $O(1)$ 으로 펌핑시킨 유닉스 세계의 자유민주주의 뼈대다.
- 이 극한의 분권화 탈중앙(Decentralized Object Control) 사기에 힘입어, 세상 99% 의 PC 사용자와 수많은 오픈소스 리눅스 데스크탑 환경은 관리자의 도움 전혀 없이도 매일매일 파일을 압축하고 이메일로 친구한테 문서 권한을 던지며 자유롭게 스루풋 궤적을 지탱했다 선고.
- 비록 "내 파일의 권한을 악성코드(Trojan) 에게 무의식적으로 고스란히 털려버려 해커의 놀이터가 되는 보안 무방비(No Forced Containment 파단 타임아웃 랙)" 라는 태생적 트레이드오프 파단을 낳았지만, 이를 스마트 강제적 접근 제어(MAC 의 모래상자 덧칠) 와 안티바이러스 휴리스틱 마스킹 우회망으로 완벽히 병합해내며 차세대 OS 방어 체계의 영원한 베이스캠프 진화판으로 록백 보장.

| **ACL (직전 575장 구현체 객체 중심 부착 장부 뷰)** | 578장 DAC 은 사상(Philosophy) 이고, 575장 ACL 은 그것을 만드는 도구(Tool) 다! DAC 철학("내 맘대로 친구 이름 써넣기") 이 없었다면 유저가 파일 뒤에 붙은 쪽지 ACL 에 자기 손으로 남의 이름을 적어 넣는 짓거리 자체가 성립하지 않는 상하위 융합 폭주 아크. |
| **파일 접근 제어 9비트 (9단원 547장 chmod 파편 렌더)** | 유눅스 콘솔에서 누구나 숨 쉬듯 치는 `chmod 644 파일.txt`. 이 1줄의 명령어가 바로 DAC 철학이 현실 커널 시스템으로 뿜어져 나오는 현현기다. "Owner 가 지 맘대로 파일 비트를 644 (나만 쓰고 남들은 읽기) 로 튜닝했다!" 완전무결한 연계 통달 구조화 파이프. |
| **강제적 접근 제어 MAC (직후 579장 보안 무결성 통제 대치 부합)** | DAC (니 맘대로 해라) 가 털리고 털려 정부 기관과 군대가 눈물을 흘리다가, 극대노해서 들고 온 게 직후의 579장 MAC (다 닥쳐! 시스템 라벨대로 쳐박혀 있어!) 이다. 두 철학은 자유 VS 통제 라는 영원한 대척점의 시소 게임 아크. |
| **신뢰받는 경로 (Trojan Horse 트로이 목마 586장 직결 타격)** | 578장에서 소개된 치명적 약점! 악성코드가 유저의 탈을 쓰는 트로이 목마(586장) 기법은 이 DAC 만이 가진 취약점의 영원한 먹잇감이다. 구조가 자유로울수록 사기꾼(기생충 프로세스) 의 침투가 O(1) 초고속으로 통과되는 쌍둥이 진화 로직 스왑 패치. |

## 👶 어린이를 위한 3줄 비유 설명
1. 멍청한 군대(일반 무식한 빡빡한 중앙 통치 늪!) 는 장병이 쓴 일기장 하나를 반장이 읽으려 해도, 매번 사단장님(루트 관리자!) 한테 결재 서류를 올려서 1주일 뒤에 허락(결재 병목 지연 랙!) 을 받아야만 볼 수 있었어요. 군대가 안 돌아가고 몽땅 굶어 죽으며 파산을 야기했어요 덜덜 에러!
2. 그래서 똑똑한 최신 대학교 봇이 **"무조건 내가 왕이다! 내 일기장은 내 맘대로! (DAC 임의적 제어 록백!)"** 마법을 결속해 줬어요! 대학교는 자기가 산 노트(파일) 는 자기가 주인(Owner 권력통치!) 임을 인정해요. 자기가 맘에 드는 친구한테 찢어주고, 싫은 애는 절대 못 보게 직접 이름을 엑스표(ACL 수정 빔!) 쳐요! 선생님 몰래 0.1초 만에 학생 자율로 척척 돌아가는 초고속 교실 방어망을 창조해요 도출!
3. 치명적 슬픔 내 노트 필기 권한을 나쁜 조폭 형한테 빼앗길 때 대참사 폭파 발생! 앗! 이 영원한 자유 Магия 에도 끔찍한 사기꾼 모순 단점이 있어요. 나쁜 조폭(트로이 목마 악성 바이러스!) 이 친한 친구인 척 내 주머니를 뒤져 내 노트 통제권을 얻어갔어요. 반장 선생님(커널 OS) 은 "응? 노트 주인이 조폭한테 주라고 허락했나 보네 통과!" 라며(Owner 의 무식한 맹신 탓에 보호 실패 파단!) 내 노트를 통째로 불태우는 끔찍한 무법지대 폭력 타협을 영원히 감당해야 하는 마법의 파이프 튜브랍니다. 진화 랙!
