# 515. I/O 캐싱 (I/O Caching)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자주 사용하는 I/O 데이터의 빠른 복사본
> 2. **가치**: 디스크 접근 감소와 응답 시간 단축
> 3. **융합**: 버퍼링, 페이지 캐시, 버퍼 캐시와 연관

---

## Ⅰ. 개요

### 개념 정의

I/O 캐싱(I/O Caching)은 **자주 접근하는 디스크 데이터를 메모리에 저장하여 디스크 I/O를 줄이는 기술**이다.

### 💡 비유: 책상 위 자주 쓰는 책
I/O 캐싱은 **책상 위에 자주 보는 책을 올려두는 것**과 같다. 책장까지 가지 않아도 바로 볼 수 있다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                I/O 캐싱 상세                                           │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 교체 알고리즘】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  알고리즘         설명                    특징                              │ │
│  │  ────────         ────                    ────                              │ │
│  │  LRU              가장 오래 사용되지 않은   일반적, 효과적                   │ │
│  │  LFU              가장 적게 사용된         빈도 기반                        │ │
│  │  FIFO             가장 먼저 들어온         단순                            │ │
│  │  Clock            순환 참조 비트           LRU 근사                        │ │
│  │  ARC              적응형 교체              LRU+LFU 하이브리드               │ │
│  │  LIRS             상호 참조               지역성 고려                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【쓰기 정책】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  정책              설명                    장단점                            │ │
│  │  ────              ────                    ──────                            │ │
│  │  Write-through    즉시 디스크에 쓰기        안전하지만 느림                   │ │
│  │  Write-back       나중에 디스크에 쓰기      빠르지만 위험                     │ │
│  │  Write-around     캐시 건너뛰고 쓰기        읽기 전용 데이터에 적합            │ │
│  │  No-write-allocate쓰기 시 캐시에 안 넣음   일회성 쓰기에 적합                │ │
│  │                                                             │ │
│  │  Linux: 기본적으로 Write-back 사용                                       │ │
│  │  sync() / fsync(): 명시적 디스크 동기화                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 적중률】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Hit Ratio = (Cache Hits) / (Total Accesses) * 100%                     │ │
│  │                                                             │ │
│  │  일반적인 적중률:                                                         │ │
│  │  • 파일 서버: 90-95%                                                     │ │
│  │  • 웹 서버: 80-90%                                                       │ │
│  │  • 데이터베이스: 70-95%                                                   │ │
│  │  • 데스크탑: 60-80%                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 상태 확인】                                                      │
│  ──────────────────                                                  │
│  // 메모리 사용량 (캐시 포함)                                            │
│  $ free -h                                                           │
│                total        used        free      shared  buff/cache │
│  Mem:           7.7G        2.1G        512M        256M        5.1G│
│  Swap:          2.0G          0B        2.0G                              │
│                                                                     │
│  // /proc/meminfo 상세                                                  │
│  $ cat /proc/meminfo | grep -i "cache\|buffer"                       │
│  Buffers:          123456 kB                                         │
│  Cached:          2345678 kB                                         │
│  Slab:             345678 kB                                         │
│  SReclaimable:     234567 kB                                         │
│                                                                     │
│  // vmstat 통계                                                         │
│  $ vmstat 1                                                          │
│  procs -----------memory---------- ---swap-- -----io----             │
│  r  b   swpd   free   buff  cache   si   so    bi    bo              │
│  1  0      0 512000 123456 2345678    0    0    10    20              │
│                                                                     │
│  【캐시 제어】                                                            │
│  ──────────────────                                                  │
│  // 캐시 플러시                                                          │
│  $ sync                            // 더티 페이지 디스크에 쓰기          │
│  $ echo 1 > /proc/sys/vm/drop_caches   // 페이지 캐시 해제             │
│  $ echo 2 > /proc/sys/vm/drop_caches   // dentry/inode 캐시 해제       │
│  $ echo 3 > /proc/sys/vm/drop_caches   // 모든 캐시 해제               │
│                                                                     │
│  // swappiness 설정                                                     │
│  $ cat /proc/sys/vm/swappiness                                       │
│  60                                                                  │
│  $ echo 10 > /proc/sys/vm/swappiness                                │
│                                                                     │
│  // vfs_cache_pressure                                                  │
│  $ cat /proc/sys/vm/vfs_cache_pressure                               │
│  100                                                                 │
│  $ echo 50 > /proc/sys/vm/vfs_cache_pressure                         │
│                                                                     │
│  【dirty 페이지 설정】                                                  │
│  ──────────────────                                                  │
│  // dirty 페이지 임계값                                                  │
│  $ cat /proc/sys/vm/dirty_ratio                                      │
│  20                                                                  │
│  $ cat /proc/sys/vm/dirty_background_ratio                           │
│  10                                                                  │
│  $ cat /proc/sys/vm/dirty_expire_centisecs                           │
│  3000                                                                │
│                                                                     │
│  // 튜닝 (데이터베이스 서버)                                              │
│  $ sysctl -w vm.dirty_ratio=40                                      │
│  $ sysctl -w vm.dirty_background_ratio=20                           │
│                                                                     │
│  【fadvise로 캐시 제어】                                                │
│  ──────────────────                                                  │
│  #include <fcntl.h>                                                  │
│                                                                     │
│  // 캐시 정책 설정                                                       │
│  int fd = open("file.txt", O_RDONLY);                                │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_NORMAL);      // 기본             │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_RANDOM);      // 랜덤 접근        │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_SEQUENTIAL);  // 순차 접근        │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_DONTNEED);    // 캐시 해제        │
│  posix_fadvise(fd, 0, 0, POSIX_FADV_NOREUSE);     // 재사용 안 함     │
│                                                                     │
│  【mmap과 페이지 캐시】                                                 │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│                                                                     │
│  int fd = open("file.txt", O_RDONLY);                                │
│  void *map = mmap(NULL, size, PROT_READ, MAP_PRIVATE, fd, 0);        │
│  // 페이지 캐시를 통해 파일 접근                                          │
│  munmap(map, size);                                                  │
│                                                                     │
│  // 캐시에서 제거                                                        │
│  madvise(map, size, MADV_DONTNEED);                                  │
│                                                                     │
│  【캐시 모니터링】                                                        │
│  ──────────────────                                                  │
│  // cachestat 추적                                                      │
│  $ perf stat -e cache-references,cache-misses ./program             │
│                                                                     │
│  // /proc/pid/smaps                                                     │
│  $ cat /proc/<pid>/smaps | grep -i "shared_clean\|file"              │
│                                                                     │
│  // BPF로 캐시 추적                                                      │
│  $ bpftrace -e 'kprobe:mark_page_accessed { @[comm] = count(); }'    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 자주 쓰는 디스크 데이터를 메모리에 저장
• 구조: 페이지 캐시 + 버퍼 캐시 (통합)
• 알고리즘: LRU, LFU, Clock, ARC
• 쓰기 정책: Write-through vs Write-back
• 확인: free, /proc/meminfo, vmstat
• 제어: drop_caches, swappiness, vfs_cache_pressure
• API: posix_fadvise, madvise, mmap
• 적중률: 높을수록 I/O 감소
```
