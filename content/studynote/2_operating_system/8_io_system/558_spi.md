# 558. SPI (Serial Peripheral Interface)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고속 전이중 직렬 통신 버스
> 2. **가치**: 마스터-슬레이브, 높은 전송 속도
> 3. **융합**: 플래시, 디스플레이, SD 카드와 연관

---

## Ⅰ. 개요

### 개념 정의
**SPI(Serial Peripheral Interface)**는 **4선 기반 고속 전이중 직렬 통신 버스**입니다.

### 💡 비유: 전용 고속도로
SPI는 **전용 고속도로**와 같습니다. 양방향으로 빠르게 이동합니다.

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux SPI】                                                       │
│  ──────────────────                                                │
│  // SPI 장치 확인                                                    │
│  $ ls /dev/spidev*                                                  │
│  /dev/spidev0.0  /dev/spidev0.1                                     │
│                                                                     │
│  // SPI 정보                                                         │
│  $ cat /sys/class/spidev/spidev0.0/dev                             │
│                                                                     │
│  // 커널 모듈                                                        │
│  $ lsmod | grep spi                                                 │
│  spidev                 8192  0                                     │
│                                                                     │
│  【Python SPI (spidev)】                                             │
│  ──────────────────                                                │
│  import spidev                                                      │
│                                                                     │
│  spi = spidev.SpiDev()                                              │
│  spi.open(0, 0)           # 버스 0, 장치 0                          │
│  spi.max_speed_hz = 1000000  # 1 MHz                               │
│  spi.mode = 0b00         # Mode 0                                  │
│  spi.bits_per_word = 8                                            │
│                                                                     │
│  # 바이트 전송 (단일)                                                │
│  resp = spi.xfer([0xAA])         # 1바이트 송수신                   │
│                                                                     │
│  # 여러 바이트 전송                                                   │
│  tx = [0x03, 0x00, 0x00, 0x00]   # 읽기 명령 + 주소                  │
│  rx = spi.xfer(tx)              # 전이중                           │
│  print(f"Received: {rx}")                                         │
│                                                                     │
│  # write만                                                          │
│  spi.writebytes([0x06])          # Write Enable                    │
│                                                                     │
│  # read만                                                           │
│  data = spi.readbytes(4)         # 4바이트 읽기                     │
│                                                                     │
│  spi.close()                                                        │
│                                                                     │
│  【C SPI (리눅스)】                                                   │
│  ──────────────────                                                │
│  #include <fcntl.h>                                                 │
│  #include <linux/spi/spidev.h>                                      │
│  #include <sys/ioctl.h>                                             │
│  #include <unistd.h>                                                │
│                                                                     │
│  int fd = open("/dev/spidev0.0", O_RDWR);                          │
│                                                                     │
│  // 모드 설정                                                        │
│  uint8_t mode = SPI_MODE_0;                                        │
│  ioctl(fd, SPI_IOC_WR_MODE, &mode);                                │
│                                                                     │
│  // 속도 설정                                                        │
│  uint32_t speed = 1000000;  // 1 MHz                                │
│  ioctl(fd, SPI_IOC_WR_MAX_SPEED_HZ, &speed);                       │
│                                                                     │
│  // 비트 per 워드                                                    │
│  uint8_t bits = 8;                                                 │
│  ioctl(fd, SPI_IOC_WR_BITS_PER_WORD, &bits);                       │
│                                                                     │
│  // 전송                                                            │
│  uint8_t tx[] = {0x03, 0x00, 0x00, 0x00};                          │
│  uint8_t rx[4];                                                    │
│                                                                     │
│  struct spi_ioc_transfer tr = {                                     │
│      .tx_buf = (unsigned long)tx,                                   │
│      .rx_buf = (unsigned long)rx,                                   │
│      .len = 4,                                                      │
│      .speed_hz = speed,                                             │
│      .bits_per_word = bits,                                         │
│  };                                                                 │
│                                                                     │
│  ioctl(fd, SPI_IOC_MESSAGE(1), &tr);                               │
│                                                                     │
│  close(fd);                                                         │
│                                                                     │
│  【SPI 플래시 읽기 예시】                                             │
│  ──────────────────                                                │
│  // JEDEC ID 읽기                                                    │
│  tx = [0x9F, 0x00, 0x00, 0x00]                                     │
│  rx = spi.xfer(tx)                                                 │
│  manufacturer = rx[1]                                              │
│  device_id = (rx[2] << 8) | rx[3]                                  │
│                                                                     │
│  // 데이터 읽기 (READ 0x03)                                          │
│  address = 0x1000                                                  │
│  tx = [0x03, (address >> 16) & 0xFF,                               │
│        (address >> 8) & 0xFF, address & 0xFF, 0x00]                │
│  rx = spi.xfer(tx)                                                 │
│  data = rx[4]  # 읽은 데이터                                         │
│                                                                     │
│  【SPI 문제 해결】                                                   │
│  ──────────────────                                                │
│  // SPI 활성화 (라즈베리파이)                                         │
│  $ sudo raspi-config  # Interface Options → SPI                    │
│                                                                     │
│  // 장치 파일 권한                                                    │
│  $ sudo usermod -aG spi $USER                                      │
│                                                                     │
│  // 핀 확인                                                          │
│  $ gpio readall                                                     │
│                                                                     │
│  // DT overlay                                                      │
│  /boot/config.txt:                                                 │
│  dtparam=spi=on                                                    │
│  dtoverlay=spi1-3cs  # SPI1 활성화                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 4선 고속 전이중 직렬 버스
• 신호: SCLK, MOSI, MISO, CS/SS
• 모드: Mode 0-3 (CPOL, CPHA 조합)
• 속도: 1 MHz - 50 MHz (일반)
• 전이중: 동시 송수신
• 마스터/슬레이브: 단일 마스터 기본
• 장치: Flash, SD, 디스플레이, ADC
• Linux: /dev/spidevX.Y
• Python: spidev 라이브러리
• 비교: I2C보다 빠르지만 선 많음
```
