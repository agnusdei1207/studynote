# 다단계 페이징 (Hierarchical Paging)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다단계 페이징(Hierarchical Paging)은 페이지 테이블 자체가 너무 커지는 문제를 해결하기 위해, 페이지 테이블을 다시 페이징하여 계층 구조(Tree Structure)로 관리하는 기법이다.
> 2. **가치**: 프로세스가 실제로 사용하는 가상 주소 영역에 대해서만 페이지 테이블을 생성하므로, 거대한 논리 주소 공간을 지원하면서도 물리 메모리 낭비를 극적으로 줄여준다.
> 3. **비용**: 주소 변환을 위해 메모리에 여러 번 접근해야 하므로(예: 2단계 시 3번 접근), TLB(Translation Look-aside Buffer)의 효율적인 활용이 성능의 핵심 전제 조건이 된다.

+++

## Ⅰ. 다단계 페이징 (Hierarchical Paging)의 정의 및 필요성

### 1. 정의
- **다단계 페이징 (Hierarchical Paging)**은 페이지 테이블을 여러 단계의 계층으로 나누어 관리하는 방식이다. 가장 바깥쪽 페이지 테이블(Outer Page Table/Page Directory)이 실제 주소 변환을 담당하는 하위 페이지 테이블들을 가리킨다.

### 2. 등장 배경: 거대해진 페이지 테이블
- 32비트 시스템에서 페이지 크기가 4KB일 때, 전체 주소 공간을 커버하려면 약 100만 개의 엔트리가 필요하며, 이는 단일 프로세스당 4MB의 연속된 메모리를 요구한다.
- 64비트 시스템에서는 이 수치가 기하급수적으로 늘어나 단일 단계 페이징은 사실상 불가능해졌다.

### 3. Sparse Address Space 지원
- 대부분의 프로세스는 주소 공간의 극히 일부(코드, 데이터, 힙, 스택 사이의 거대한 빈 공간)만 사용한다. 다단계 페이징은 사용하지 않는 영역의 하위 페이지 테이블을 아예 생성하지 않음으로써 메모리를 절약한다.

- **📢 섹션 요약 비유**: "백과사전 전체를 들고 다니는 대신, 'ㄱ~ㅎ' 색인집(페이지 디렉터리)만 들고 있다가 필요한 글자가 있는 권(페이지 테이블)만 서재에서 꺼내 쓰는 것과 같습니다."

+++

## Ⅱ. 2단계 페이징의 작동 메커니즘 및 ASCII 다이어그램

### 1. 논리 주소의 분할 (32비트 기준)
- 논리 주소는 **[P1 | P2 | d]** 세 부분으로 나뉜다.
    - **P1 (Outer Page Number)**: 페이지 디렉터리의 인덱스.
    - **P2 (Inner Page Number)**: 실제 페이지 테이블의 인덱스.
    - **d (Offset)**: 해당 페이지 내의 변위.

### 2. ASCII: 2단계 주소 변환 과정
```text
[ Logical Address ]
+------------+------------+-----------------+
| P1 (10bit) | P2 (10bit) | Offset (12bit)  |
+------------+------------+-----------------+
      |            |                |
      v            v                |
+-----------+      |                |
| Page Dir  |      |                |
+-----------+      v                |
|   P1      |--->+------------+     |
+-----------+    | Page Table |     |
                 +------------+     |
                 |    P2      |---->+-----------+
                 +------------+     |  Physical |
                                    |   Memory  |
                                    +-----------+
```

### 3. 단계별 접근
1. CPU가 제시한 P1으로 페이지 디렉터리에 접근하여 해당 페이지 테이블의 기본 주소를 얻는다.
2. 얻어낸 주소와 P2를 조합하여 실제 프레임 번호를 찾는다.
3. 프레임 번호와 Offset을 결합하여 실제 물리 주소에 도달한다.

- **📢 섹션 요약 비유**: "아파트 단지 입구에서 동 번호(P1)를 확인하고, 해당 동에 들어가서 호수(P2)를 찾는 2단계 확인 과정입니다."

+++

## Ⅲ. 다단계 페이징의 기술적 특징 및 장단점

### 1. 장점 (Pros)
- **메모리 효율성**: 사용되지 않는 가상 주소 범위에 대해서는 하위 페이지 테이블을 할당할 필요가 없어 메모리 오버헤드가 최소화된다.
- **불연속적 할당**: 페이지 테이블 자체가 페이징되므로, 페이지 테이블을 위한 거대한 연속 메모리 공간을 찾을 필요가 없다.

### 2. 단점 (Cons)
- **메모리 접근 횟수 증가**: 2단계 페이징은 2번의 테이블 조회 + 1번의 실제 데이터 접근으로 총 3번의 메모리 접근이 발생한다. (계층이 깊어질수록 심화)
- **TLB 의존성**: 메모리 접근 지연을 줄이기 위해 TLB 적중률에 극도로 의존하게 된다.

- **📢 섹션 요약 비유**: "방은 적게 차지하지만, 물건 하나를 찾으러 갈 때 여러 개의 문을 열고 들어가야 해서 시간이 더 걸리는 수납장과 같습니다."

+++

## Ⅳ. 현대 64비트 시스템에서의 계층 구조 (Strategy)

### 1. 계층의 심화 (Multi-level)
- 64비트 주소 체계에서는 주소 공간이 워낙 넓어 2단계만으로는 부족하다.
- **x86-64(Intel 64)**: 일반적으로 4단계 페이징(PML4 -> PDPT -> PD -> PT)을 사용하며, 최근에는 5단계 페이징까지 도입되었다.

### 2. 64비트 주소 분할 예시 (48비트 유효 주소 기준)
- [9비트 | 9비트 | 9비트 | 9비트 | 12비트 Offset] 구조로 총 4단계 계층을 형성한다.

### 3. 성능 최적화
- **Huge Pages**: 계층 구조의 하위 단계를 생략하고 상위 단계에서 직접 거대한 프레임을 가리키게 하여 TLB 부담을 줄이는 기술이 병행된다.

- **📢 섹션 요약 비유**: "전 세계 주소를 찾기 위해 [국가-도-시-구-동]으로 단계를 세분화하여, 내가 살지 않는 다른 나라의 세부 지도는 아예 가지고 있지 않는 전략입니다."

+++

## Ⅴ. 결론 및 요약

### 1. 핵심 요약
- 다단계 페이징은 **'공간 복잡도(메모리 점유)'를 낮추기 위해 '시간 복잡도(접근 횟수)'를 희생**한 전형적인 트레이드오프 사례다.

### 2. 시스템적 의의
- 현대 컴퓨팅의 거대한 가상 주소 공간을 물리적인 메모리 한계 내에서 구현 가능하게 만드는 주소 변환 아키텍처의 표준이다.

- **📢 섹션 요약 비유**: "결국 '필요한 것만 계층적으로 관리한다'는 원칙이 컴퓨터 메모리 관리에도 그대로 적용된 결과입니다."

### 📌 관련 개념 맵 (Knowledge Graph)
- **페이지 테이블 (Page Table)**: 다단계로 쪼개지는 기본 단위.
- **TLB (Translation Look-aside Buffer)**: 다단계 페이징의 속도 저하를 보완하는 필수 하드웨어.
- **Sparse Address Space**: 다단계 페이징이 존재해야 하는 근본적인 이유.
- **역 페이지 테이블 (Inverted Page Table)**: 계층 구조의 대안으로 제시되는 또 다른 방식.
