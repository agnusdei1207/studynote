# 394. TLB (Translation Lookaside Buffer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이지 테이블을 캐싱하는 고속 하드웨어
> 2. **가치**: 주소 변환 속도 대폭 향상
> 3. **융합**: 페이징, 페이지 테이블과 연관

---

## Ⅰ. 개요

### 개념 정의

TLB(Translation Lookaside Buffer)는 **페이지 테이블의 캐시로, 최근 사용된 페이지-프레임 매핑을 저장하는 고속 하드웨어**다. TLB 조회로 주소 변환 속도를 높인다.

### 💡 비유: 색인
TLB는 **책의 색인**과 같다. 자주 찾는 책의 위치를 별도로 적어둔다. 색인에서 바로 찾을 수 있다.

### TLB 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                TLB 구조                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLB의 역할】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  CPU ──논리 주소──► TLB ──(히트)──► 물리 주소 ──> 메모리 접근               │ │
│  │                    │                                           │ │
│  │                    └──(미스)──► 페이지 테이블 ──> TLB 갱신 ──> 메모리 접근       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 항목 구조】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │ 가상 페이지 번호 │ 물리 프레임 번호 │ 보호 비트 │ 유효 비트 │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │     VPN         │     PFN        │   R/W/X   │   V    │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  │  VPN: Virtual Page Number                                        │ │
│  │  PFN: Physical Frame Number                                      │ │
│  │  R/W/X: Read/Write/eXecute 권한                                    │ │
│  │  V: Valid (유효 항목인지)                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 히트와 미스】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  TLB Hit:                                                          │ │
│  │  • 원하는 매핑이 TLB에 있음                                            │ │
│  │  • 페이지 테이블 조회 불필요                                        │ │
│  │  • ~1-2 클럭 사이클                                                │ │
│  │                                                             │ │
│  │  TLB Miss:                                                          │ │
│  │  • 원하는 매핑이 TLB에 없음                                            │ │
│  │  • 페이지 테이블 조회 필요                                          │ │
│  │  • ~10-100 클럽 사이클                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                TLB 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLB 종류】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 단일 TLB:                                                         │ │
│  │     • 모든 프로세스가 공유                                             │ │
│  │     • 컨텍스트 스위치 시 플러시 필요                                    │ │
│  │                                                             │ │
│  │  2. 분리 TLB:                                                         │ │
│  │     • 데이터 TLB (DTLB) + 명령 TLB (ITLB)                            │ │
│  │     • 병렬 조회 가능                                                   │ │
│  │                                                             │ │
│  │  3. ASID 지원 TLB:                                                    │ │
│  │     • 프로세스별 TLB 항목 식별                                         │ │
│  │     • 컨텍스트 스위치 시 플러시 불필요                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 교체 정책】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Random: 무작위 교체                                               │ │
│  │  2. FIFO: 선입선출                                                    │ │
│  │  3. LRU: 최근 최소 사용 (Least Recently Used)                         │ │
│  │  4. NRU: 무작위 교체 (Not Recently Used)                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 슈트다운】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  프로세스 전환 시 TLB 내용 무효화                                        │ │
│  │                                                             │ │
│  │  문제:                                                                 │ │
│  │  • 새 프로세스 시작 시 많은 TLB 미스                                    │ │
│  │  • 성능 저하                                                          │ │
│  │                                                             │ │
│  │  해결:                                                                 │ │
│  │  • ASID (Address Space ID) 사용                                        │ │
│  │  • 각 항목에 프로세스 ID 포함                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TLB 정보 확인】                                                          │
│  ──────────────────                                                  │
│  // Linux: TLB 정보                                                       │
│  $ cat /proc/cpuinfo | grep -i tlb                               │
│  $ lscpu | grep -i tlb                                            │
│                                                                     │
│  // x86: TLB 크기                                                         │
│  // L1: 64 entries, L2: 512-1024 entries                              │
│  // 페이지 크기별로 분리 (4KB, 2MB, 1GB)                                    │
│                                                                     │
│  【TLB 미스 모니터링】                                                      │
│  ──────────────────                                                  │
│  // perf 도구                                                             │
│  $ perf stat -e dTLB-loads,dTLB-load-misses,iTLB-loads,iTLB-load-misses│
│                                                                     │
│  // /proc 통계                                                            │
│  $ cat /proc/vmstat | grep tlb                                    │
│  nr_tlb_global_flush_all 12345                                      │
│  nr_tlb_local_flush_all 67890                                      │
│                                                                     │
│  【TLB 플러시】                                                             │
│  ──────────────────                                                  │
│  // 컨텍스트 스위치 시 TLB 플러시                                          │
│  void switch_mm(struct mm_struct *prev, struct mm_struct *next) {    │
│      if (prev != next) {                                              │
│          // ASID가 없으면 전체 TLB 플러시                                   │ │
│          flush_tlb();                                                 │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  // 페이지 테이블 변경 시 부분 TLB 플러시                                   │ │
│  void flush_tlb_page(struct vm_area_struct *vma, unsigned long addr) {│
│      // 해당 페이지의 TLB 항목만 무효화                                    │ │
│  }                                                                  │
│                                                                     │
│  【TLB 일관성 유지】                                                        │
│  ──────────────────                                                  │
│  // 페이지 테이블 변경 시 TLB 갱신                                         │ │
│  void update_tlb(struct mm_struct *mm, unsigned long addr) {        │
│      pte_t *pte = get_pte(mm, addr);                                │
│      unsigned long pfn = pte_pfn(*pte);                             │
│      // TLB 항목 갱신                                                   │ │
│      tlb_update(addr, pfn, pte_flags(*pte));                        │
│  }                                                                  │
│                                                                     │
│  // mmap/munmap 시 TLB 갱신                                            │ │
│  void munmap_region(unsigned long addr, size_t len) {               │
│      // 페이지 테이블 해제                                                 │ │
│      free_page_tables(addr, len);                                   │
│      // 해당 범위의 TLB 항목 무효화                                       │ │
│      flush_tlb_range(current->mm, addr, addr + len);                │
│  }                                                                  │
│                                                                     │
│  【TLB 성능 최적화】                                                        │
│  ──────────────────                                                  │
│  // 큰 페이지 사용으로 TLB 커버리지 증가                                   │ │
│  // Huge Pages: 2MB 또는 1GB 페이지                                      │ │
│  $ echo 1000 > /proc/sys/vm/nr_hugepages                            │
│                                                                     │
│  // HugeTLB로 메모리 할당                                                 │
│  void* buf = mmap(NULL, size, PROT_READ | PROT_WRITE,              │
│                  MAP_PRIVATE | MAP_HUGETLB, -1, 0);                 │
│                                                                     │
│  // TLB 미스 감소 → 성능 향상                                              │ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 페이지 테이블 캐시
• 구조: VPN, PFN, 보호 비트, 유효 비트
• 히트: 페이지 테이블 조회 불필요, ~1-2 사이클
• 미스: 페이지 테이블 조회 필요, ~10-100 사이클
• 교체: Random, FIFO, LRU, NRU
• 슈트다운: 컨텍스트 스위치 시 플러시
```
