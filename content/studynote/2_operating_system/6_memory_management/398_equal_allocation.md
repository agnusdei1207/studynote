# 398. 세그먼테이션 (Segmentation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 논리적 단위로 메모리를 관리하는 기법
> 2. **가치**: 가변 크기 메모리 할당에 적합
> 3. **융합**: 페이징, 가상 메모리와 연관

---

## Ⅰ. 개요

### 개념 정의

세그먼테이션(Segmentation)은 **메모리를 논리적 단위(세그먼트)로 나누어 관리하는 기법**이다. 각 세그먼트는 의미 있는 단위(함수, 배열, 모듈 등)다.

### 💡 비유: 챕터
세그먼테이션은 **책의 챕터**와 같다. 각 챕터는 서로 다른 길이를 가질 수 있다. 목차에서 찾아간다.

### 세그먼테이션 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                세그먼테이션 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【세그먼트 구조】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  논리 주소                                                           │ │
│  │  ┌─────────────────────┬─────────────────────┐                    │ │
│  │  │    세그먼트 번호 (s)   │    오프셋 (d)        │                    │ │
│  │  └─────────────────────┴─────────────────────┘                    │ │
│  │           │                                                         │ │
│  │           │ 세그먼트 테이블 조회                                         │ │
│  │           ▼                                                         │ │
│  │  ┌─────────────────────────────────────────────────────────┐     │ │
│  │  │ 세그먼트 테이블                                            │     │ │
│  │  │ ┌───────────────────────────────────────────────┐       │     │ │
│  │  │ │ 번호 │ 기준 주소 │ 한계 │                        │       │     │ │
│  │  │ ├───────────────────────────────────────────────┤       │     │ │
│  │  │ │  0  │ 0x1000  │ 0x500 │ (코드 세그먼트)              │       │     │ │
│  │  │ │  1  │ 0x2000  │ 0x1000 │ (데이터 세그먼트)             │       │     │ │
│  │  │ │  2  │ 0x3000  │ 0x200 │ (스택 세그먼트)              │       │     │ │
│  │  │ │ ... │   ...   │  ...  │                             │       │     │ │
│  │  │ └───────────────────────────────────────────────┘       │     │ │
│  │  └─────────────────────────────────────────────────────────┘     │ │
│  │           │                                                         │ │
│  │           │ 기준 주소 + 오프셋                                           │ │
│  │           ▼                                                         │ │
│  │  물리 주소 = 기준 주소 + 오프셋                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【세그먼트 종류】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 코드 세그먼트 (Code Segment): 프로그램 코드                           │ │
│  │  • 데이터 세그먼트 (Data Segment): 전역 변수                            │ │
│  │  • 스택 세그먼트 (Stack Segment): 지역 변수, 호출 스택                  │ │
│  │  • 힙 세그먼트 (Heap Segment): 동적 할당 메모리                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 페이징】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           페이징              세그먼테이션                       │ │
│  │  ────────       ────────────        ────────────                   │ │
│  │  분할 단위       고정 크기            가변 크기                       │ │
│  │  분할 기준       페이지               논리적 단위                     │ │
│  │  내부 단편화      있음                 없음                          │ │
│  │  외부 단편화      없음                 있음                          │ │
│  │  주소 변환       1단계                1단계                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                세그먼테이션 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【외부 단편화】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  가변 크기 할당으로 인한 문제:                                          │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │ [세그먼트A][  빈공간  ][세그먼트B][빈공간][세그먼트C]         │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  │  전체 빈 공간은 충분하지만 연속적이지 않아 대형 세그먼트 할당 불가      │ │
│  │                                                             │ │
│  │  해결: 압축(Compaction) 또는 페이징과 결합                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【세그먼트 보호】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  각 세그먼트에 보호 비트:                                               │ │
│  │                                                             │ │
│  │  • 읽기 비트(R): 읽기 권한                                             │ │
│  │  • 쓰기 비트(W): 쓰기 권한                                             │ │
│  │  • 실행 비트(X): 실행 권한                                             │ │
│  │                                                             │ │
│  │  예: 코드 세그먼트 → R-X (읽기, 실행)                                  │ │
│  │      데이터 세그먼트 → RW- (읽기, 쓰기)                                │ │
│  │      스택 세그먼트 → RW- (읽기, 쓰기)                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【세그먼트 공유】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  여러 프로세스가 동일한 세그먼트 공유 가능:                               │ │
│  │                                                             │ │
│  │  • 코드 공유: 공통 라이브러리                                            │ │
│  │  • 데이터 공유: 공유 메모리                                              │ │
│  │                                                             │ │
│  │  세그먼트 테이블 항목이 같은 기준 주소를 가리킴                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86 세그먼테이션】                                                     │
│  ──────────────────                                                  │
│  // 세그먼트 레지스터                                                       │
│  // CS: Code Segment                                                  │
│  // DS: Data Segment                                                  │
│  // SS: Stack Segment                                                 │
│  // ES, FS, GS: Extra Segments                                       │
│                                                                     │
│  // 세그먼트 디스크립터 (GDT/LDT)                                         │
│  struct segment_descriptor {                                        │
│      uint64_t base;     // 기준 주소                                    │ │
│      uint64_t limit;    // 한계                                        │ │
│      uint32_t type;     // 타입 (코드, 데이터)                             │ │
│      uint32_t dpl;      // 권한 레벨                                     │ │
│      bool present;      // 메모리에 있는지                                 │ │
│  };                                                                 │
│                                                                     │
│  【세그먼트 폴트】                                                          │
│  ──────────────────                                                  │
│  // Linux 시그널: SIGSEGV                                              │
│  void handler(int sig) {                                             │
│      printf("Segmentation fault!\n");                               │
│      exit(1);                                                        │
│  }                                                                  │
│  signal(SIGSEGV, handler);                                          │
│                                                                     │
│  // 세그먼트 폴트 원인                                                     │
│  // 1. NULL 포인터 접근                                                  │
│  // 2. 해제된 메모리 접근                                                  │
│  // 3. 배열 인덱스 초과                                                   │
│  // 4. 스택 오버플로우                                                     │
│                                                                     │
│  【현대 OS에서의 세그먼테이션】                                             │
│  ──────────────────                                                  │
│  // 대부분 현대 OS는 페이징 사용                                            │
│  // 세그먼테이션은 하드웨어적으로만 사용 (x86)                               │
│  // "Flat" 메모리 모델: 세그먼트 = 전체 주소 공간                            │ │
│                                                                     │
│  // Linux: 세그먼트를 모두 0부터 시작으로 설정                               │ │
│  // 실제로는 페이징만 사용                                                  │ │
│                                                                     │
│  【세그먼테이션 with 페이징】                                               │
│  ──────────────────                                                  │
│  // 세그먼트: 논리적 단위                                                   │
│  // 각 세그먼트 내부: 페이징                                               │ │
│  // 장점: 논리적 단위 + 외부 단편화 없음                                     │ │
│                                                                     │
│  논리 주소 ──세그먼트──► 선형 주소 ──페이지──► 물리 주소                  │
│                                                                     │
│  // Intel x86-64: 세그먼테이션 + 페이징 결합                                │ │
│                                                                     │
│  【메모리 보호】                                                            │
│  ──────────────────                                                  │
│  // 세그먼트 한계 검사                                                      │
│  if (offset >= segment.limit) {                                     │
│      raise(SIGSEGV);  // 세그먼트 폴트                                   │ │
│  }                                                                  │
│                                                                     │
│  // 권한 검사                                                             │
│  if (operation == WRITE && !(segment.flags & WRITE_FLAG)) {         │
│      raise(SIGSEGV);                                                │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 논리적 단위로 메모리 관리
• 구조: 세그먼트 번호 + 오프셋
• 구성요소: 세그먼트 테이블 (기준 주소, 한계)
• 장점: 논리적 단위, 보호, 공유
• 단점: 외부 단편화
• 활용: x86 하드웨어, 세그먼테이션 with 페이징
```
