# 외부 단편화 (External Fragmentation)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 외부 단편화(External Fragmentation)는 메모리 내에 사용 가능한 총 공간은 충분하지만, 그 공간들이 연속적이지 않고 작은 조각들로 흩어져 있어 새로운 프로세스를 할당할 수 없는 상태이다.
> 2. **발생 원인**: 가변 분할(Variable Partitioning) 방식에서 프로세스들이 수시로 생성되고 소멸하며 서로 다른 크기의 구멍(Hole)들을 남기기 때문에 발생한다.
> 3. **해결 전략**: 메모리 압축(Compaction)을 통해 흩어진 조각들을 모으거나, 페이징(Paging)과 같은 비연속 할당 기법을 사용하여 논리적 연속성을 확보한다.

+++

## Ⅰ. 외부 단편화 (External Fragmentation)의 개념

### 1. 정의
- **외부 단편화 (External Fragmentation)**란 물리적 메모리 전체를 통틀어 남은 공간의 합계는 요청받은 프로세스의 크기보다 크지만, 할당 가능한 연속된(Contiguous) 단일 블록이 없어 할당이 실패하는 현상을 말한다.

### 2. 발생 배경
- 연속 메모리 할당 방식에서는 프로세스가 시작 주소부터 끝 주소까지 끊김 없이 배치되어야 한다. 프로세스가 종료되어 나간 자리에 더 작은 프로세스가 들어오면, 그 차이만큼 미세한 틈새가 남게 되는데 이것이 누적되면 외부 단편화가 심화된다.

### 3. 💡 비유: '주차장의 애매한 틈새'
- 주차장에 차 1대 분량의 빈 공간이 3군데 흩어져 있지만, 정작 새로 들어온 커다란 트럭 1대를 세울 자리는 없는 상황과 같다.

- **📢 섹션 요약 비유**: "돈은 충분히 있는데, 10원짜리 동전으로만 흩어져 있어서 1,000원짜리 지폐 한 장이 필요한 자판기를 이용하지 못하는 상태입니다."

+++

## Ⅱ. 외부 단편화 가시화 및 ASCII 다이어그램

### 1. 가변 분할 환경에서의 파편화 과정
- 프로세스가 들어오고 나감에 따라 메모리 맵에 빈 공간(Hole)들이 불규칙하게 형성된다.

### 2. ASCII 메모리 맵 (External Fragmentation)
```text
[ Physical Memory Layout ]
+----------------------------+
|  Operating System (OS)     |
+----------------------------+
|  Process A (Used)          |
+----------------------------+
|  [Hole 1: 20KB] <----------! (Too small for New Proc)
+----------------------------+
|  Process B (Used)          |
+----------------------------+
|  [Hole 2: 30KB] <----------! (Too small for New Proc)
+----------------------------+
|  Process C (Used)          |
+----------------------------+
|  [Hole 3: 10KB] <----------! (Too small for New Proc)
+----------------------------+
[ Total Free: 60KB ]
[ Requirement: Process D (50KB) ]
--> Result: Allocation Fails due to Fragmentation!
```

### 3. 기술적 특징
- **동적 할당의 부작용**: 메모리 관리를 유연하게 하려다 보니 발생하는 고질적인 문제다. 통계적으로 총 메모리의 약 1/3이 외부 단편화로 인해 낭비될 수 있다는 '50% 규칙'이 언급되기도 한다.

- **📢 섹션 요약 비유**: "퍼즐 조각들은 다 있는데, 모양이 안 맞아서 그림 하나를 완성하지 못하는 텅 빈 캔버스와 같습니다."

+++

## Ⅲ. 발생 원인 및 영향력 분석

### 1. 주된 발생 원인
- **연속 할당 조건**: 프로세스는 반드시 연속된 메모리 공간을 점유해야 한다는 제약 조건 때문이다.
- **불규칙한 프로세스 수명**: 프로세스들의 실행 시간이 다르고, 메모리 반납 시점이 제각각이기 때문에 공간이 파편화된다.

### 2. 시스템에 미치는 영향
- **메모리 자원 고갈**: 실제 빈 공간이 있음에도 불구하고 'Out of Memory' 오류가 발생할 수 있다.
- **성능 저하**: 할당 가능한 공간을 찾기 위해 검색 시간이 길어지거나, 해결을 위해 압축(Compaction) 작업을 수행할 때 시스템이 멈추는 오버헤드가 발생한다.

- **📢 섹션 요약 비유**: "창고 곳곳에 빈 자리는 많은데, 정작 큰 냉장고 하나를 들여놓을 자리가 없어서 이사를 못 가는 상황입니다."

+++

## Ⅳ. 외부 단편화 해결 전략 (Strategy)

### 1. 메모리 압축 (Compaction)
- 실행 중인 프로세스들을 메모리의 한쪽 끝으로 밀어 넣고, 흩어진 빈 공간들을 하나로 합치는 방법이다. (단, 프로세스의 재배치(Relocation)가 가능해야 하며 매우 높은 CPU 오버헤드를 동반한다.)

### 2. 통합(Coalescing) 및 버디 시스템
- 인접한 빈 공간들이 생기면 즉시 하나로 합치는 기법이다. 버디 시스템(Buddy System)은 메모리를 2의 거듭제곱 단위로 관리하여 통합을 효율화한다.

### 3. 페이징 (Paging) - 궁극적 해결책
- 프로세스를 고정된 크기의 페이지(Page)로 나누고, 물리 메모리의 빈 프레임(Frame)에 흩어져서 배치한다. 물리적으로는 조각나 있지만 논리적으로는 연속된 것처럼 보이게 하여 외부 단편화를 근본적으로 해결한다.

- **📢 섹션 요약 비유**: "책상 위의 잡동사니를 한데 모아 공간을 만드는 '압축'과, 물건을 분해해서 빈틈에 쏙쏙 집어넣는 '페이징' 전략입니다."

+++

## Ⅴ. 결론 및 요약

### 1. 핵심 요약
- 외부 단편화는 '조각난 빈 공간'의 문제다. 연속성이라는 제약이 자원 효율을 떨어뜨리는 대표적인 사례다.

### 2. 향후 전망
- 현대 OS는 페이징을 통해 물리 레벨의 외부 단편화를 해결했다. 그러나 페이징 내부에서 발생하는 내부 단편화와의 균형을 맞추기 위해 '대형 페이지(Huge Pages)' 기술 등을 도입하며 최적화 경로를 찾고 있다.

- **📢 섹션 요약 비유**: "조각난 땅을 합쳐서 큰 건물을 짓거나, 건물을 조립식으로 만들어 좁은 땅 여러 곳에 나누어 짓는 지혜가 필요합니다."

### 📌 관련 개념 맵 (Knowledge Graph)
- **내부 단편화 (Internal Fragmentation)**: 할당된 구역 내부에서 발생하는 공간 낭비.
- **압축 (Compaction)**: 외부 단편화를 해결하기 위한 물리적 데이터 이동 기법.
- **페이징 (Paging)**: 외부 단편화를 원천 차단하는 비연속 메모리 할당 기법.
- **배치 알고리즘 (Placement Algorithm)**: 단편화를 줄이기 위해 빈 공간을 고르는 전략.
