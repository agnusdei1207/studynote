# 표준 스와핑 (전체 프로세스) vs 페이징 시스템 스와핑

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 표준 스와핑은 프로세스 전체를 통째로 옮기지만, 페이징 시스템 스와핑은 프로세스를 구성하는 작은 단위인 '페이지'만을 선택적으로 옮긴다.
> 2. **가치**: 페이징 기반 방식은 디스크 I/O 전송 시간을 획기적으로 단축시키고, 필요한 부분만 메모리에 적재하여 자원 활용도를 극대화한다.
> 3. **융합**: 현대 운영체제는 표준 스와핑 대신 페이징 기법을 활용한 '페이지 스와핑(Page Swapping)' 혹은 '디맨드 페이징(Demand Paging)'을 표준으로 채택하고 있다.

+++

## Ⅰ. 개념 및 배경 비교

### 1. 정의
- **표준 스와핑 (Standard Swapping)**: 프로세스 주소 공간 전체를 메인 메모리와 보조 기억장치 사이에서 이동시키는 고전적 방식.
- **페이징 시스템 스와핑 (Paging Swapping)**: 프로세스의 일부분(페이지)만을 디스크로 내보내거나(Page-out), 가져오는(Page-in) 방식.

### 2. 등장 배경: '덩치의 한계'
- 프로세스의 크기가 수백 MB, 수 GB로 커지면서 전체를 옮기는 표준 스와핑은 시스템을 너무 오랫동안 멈추게 했다. 이에 "프로세스 전체가 아니라, 당장 안 쓰는 일부 페이지만 빼자"는 효율적인 생각이 대두되었다.

### 3. 💡 비유: '이사 vs 짐 일부 보관'
- 표준 스와핑은 집 전체의 가구(Process)를 다 빼서 이사 가는 것이라면, 페이징 스와핑은 집은 그대로 두고 지금 안 쓰는 **'여름 옷 상자(Page)'** 몇 개만 창고에 맡기는 것과 같다.

- **📢 섹션 요약 비유**: "통째로 움직이는 비효율을 버리고, 조각 단위로 실속을 챙기는 진화입니다."

+++

## Ⅱ. 작동 원리 (Deep Dive)

### 1. 매커니즘 비교
- **표준**: 링커에 의해 생성된 바이너리 전체를 스왑 영역에 기록/독출.
- **페이징**: 페이지 테이블의 `valid-invalid bit`를 활용하여 메모리에 없는 페이지만 디스크에서 읽어옴 (Demand Paging).

### 2. ASCII 다이어그램: 단위의 차이
```text
 [ Standard Swapping ]           [ Paging Swapping ]
 +-------------------+           +-------------------+
 | [  Whole Proc A  ]|           | [ Page 1 ] (In)   |
 |       1GB         |           | [ Page 2 ] (Out)  |
 +-------------------+           | [ Page 3 ] (In)   |
          │                      +-------------------+
          ▼                                │
  (Heavy I/O Load)               (Selective I/O Load)
```

### 3. 기술적 상세 해설
- **스와퍼(Swapper) vs 페이저(Pager)**: 표준 스와핑을 담당하는 하부 시스템을 스와퍼라 부르고, 페이징 시스템에서 페이지 단위 교체를 담당하는 시스템을 페이저(Pager)라고 구분하여 부르기도 한다.

- **📢 섹션 요약 비유**: "포크레인으로 산 전체를 옮기느냐(표준), 삽으로 흙 한 바구니씩만 옮기느냐(페이징)의 차이입니다."

+++

## Ⅲ. 주요 특징 및 분석 (Comparison)

### 1. 비교 분석 테이블
| 구분 | 표준 스와핑 | 페이징 스와핑 |
|:---|:---|:---|
| **이동 단위** | 프로세스 전체 | 페이지 (고정 크기 조각) |
| **I/O 비용** | 매우 높음 (느림) | 낮음 (빠름) |
| **메모리 활용** | 낮음 (사용 안 하는 코드도 적재) | 매우 높음 (필요한 것만 적재) |
| **현대적 관점** | 거의 사용 안 함 | 표준 기법 |

### 2. 기술적 특이점
- **지연 시간**: 표준 스와핑은 프로세스 전환 시 수 초의 지연이 발생할 수 있어 실시간 시스템에서 치명적이지만, 페이징 방식은 밀리초 단위로 제어가 가능하다.

- **📢 섹션 요약 비유**: "전체 결제만 가능한 쇼핑몰과 낱개 구매가 가능한 쇼핑몰의 차이입니다."

## Ⅳ. 실무 적용 및 시스템 전략 (Strategy)

### 1. 실제 사례
- **가상 메모리(Virtual Memory)**: 현대의 모든 범용 OS는 페이징 스와핑을 기반으로 가상 메모리를 구현한다.
- **초기 유닉스(Early UNIX)**: 메모리가 극히 작았던 초기 시절에는 표준 스와핑 방식을 사용했다.

### 2. 문제 해결 전략
- **워킹 셋(Working Set)**: 프로세스가 자주 사용하는 페이지 집합(Working Set)을 메모리에 유지함으로써, 페이징 스와핑으로 인한 성능 저하를 방지한다.

- **📢 섹션 요약 비유**: "자주 쓰는 도구는 항상 책상 위에 두어 서랍을 여는 횟수를 줄이는 전략입니다."

+++

## Ⅴ. 기대효과 및 결론

### 1. 기대효과
- **다중 프로그래밍의 비약적 향상**: 더 작은 단위로 자원을 쪼개어 관리하므로 훨씬 많은 프로세스를 동시에 수용할 수 있다.
- **응답 속도 개선**: 불필요한 데이터 전송이 사라져 시스템 전체의 반응성이 좋아진다.

### 2. 결론 및 향후 전망
- 하드웨어의 발전으로 메모리가 비약적으로 커졌음에도 불구하고, 클라우드와 고성능 컴퓨팅 환경에서 효율성을 극대화하기 위해 페이징 시스템 스와핑은 더욱 세밀하고 지능적으로(AI 기반 예측 등) 진화하고 있다.

- **📢 섹션 요약 비유**: "작은 조각이 모여 거대한 효율을 만드는 분할 정복의 승리입니다."
