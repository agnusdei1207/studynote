# 비연속 메모리 할당 (Non-contiguous Memory Allocation)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비연속 메모리 할당(Non-contiguous Memory Allocation)은 하나의 프로세스가 물리적 메모리의 여러 군데에 흩어져서 배치될 수 있도록 허용하는 메모리 관리 기법이다.
> 2. **가치**: 프로세스가 반드시 연속된 공간을 점유해야 한다는 제약을 제거함으로써, 외부 단편화 문제를 근본적으로 해결하고 메모리 이용 효율을 극대화한다.
> 3. **구현 기법**: 논리적 주소와 물리적 주소를 분리하는 매핑 기술이 필수적이며, 대표적으로 페이징(Paging)과 세그먼테이션(Segmentation)이 이에 해당한다.

+++

## Ⅰ. 비연속 메모리 할당 (Non-contiguous Memory Allocation)의 개념

### 1. 정의
- **비연속 메모리 할당 (Non-contiguous Memory Allocation)**이란 프로세스의 논리적인 주소 공간을 일정한 단위로 나누어, 물리적 메모리의 비어있는 임의의 위치들에 분산하여 할당하는 방식이다.

### 2. 등장 배경: 연속 할당의 한계 극복
- 연속 할당 방식은 외부 단편화로 인해 총 메모리가 충분해도 프로세스를 할당하지 못하는 경우가 빈번했다. 이를 해결하기 위해 '흩어져 있어도 하나처럼 쓸 수 있게 하자'는 아이디어가 도입되었다.

### 3. 💡 비유: '분할 납부 및 분산 저장'
- 커다란 가구 하나가 현관문으로 안 들어올 때, 가구를 부품별로 분해해서 집안 곳곳의 남는 틈새에 넣어두고 필요할 때 조립해서 쓰는 것과 같다.

- **📢 섹션 요약 비유**: "커다란 보물을 통째로 숨길 땅이 없어서, 보물을 여러 조각으로 나누어 마당 여기저기의 작은 구멍들에 나누어 묻어두는 전략입니다."

+++

## Ⅱ. 비연속 할당의 메커니즘 및 ASCII 다이어그램

### 1. 핵심 원리: 주소 변환 (Address Translation)
- CPU가 바라보는 **논리적 주소(Logical Address)**는 연속적이지만, 실제 **물리적 주소(Physical Address)**는 파편화되어 있다. 이 둘을 연결해주는 매핑 테이블(Page Table 등)이 핵심 역할을 한다.

### 2. ASCII 메모리 매핑 구조
```text
[ Logical Process View ]       [ Mapping Table ]       [ Physical Memory ]
+---------------------+        +-------+-------+       +---------------------+
| Page 0              | ------>| Index | Frame |       | Frame 5: Page 1     |
+---------------------+        +-------+-------+       +---------------------+
| Page 1              | ------>|   0   |   8   |------>| Frame 8: Page 0     |
+---------------------+        |   1   |   5   |       +---------------------+
| Page 2              | ------>|   2   |  12   |       | (Other Process)     |
+---------------------+        +-------+-------+       +---------------------+
                                                       | Frame 12: Page 2    |
                                                       +---------------------+
```

### 3. 주요 구현 방식
- **페이징 (Paging)**: 프로세스를 '고정된 크기'의 페이지로 나누어 관리.
- **세그먼테이션 (Segmentation)**: 프로세스를 '논리적 의미 단위(함수, 스택 등)'로 나누어 관리.

- **📢 섹션 요약 비유**: "지도(매핑 테이블)만 있으면 보물이 어디에 흩어져 있든 상관없이 언제든 찾아낼 수 있는 '보물지도 시스템'입니다."

+++

## Ⅲ. 기술적 특징 및 장단점 분석

### 1. 장점 (Pros)
- **외부 단편화 해결**: 물리 메모리에 남는 공간(Hole)이 프로세스 조각보다 크기만 하면 어디든 들어갈 수 있다.
- **메모리 공유 용이**: 여러 프로세스가 동일한 물리 메모리 조각을 각자의 매핑 테이블에 등록하여 공유 라이브러리 등을 효율적으로 쓸 수 있다.
- **유연한 확장**: 프로세스의 크기가 커지더라도 연속된 공간을 찾을 필요 없이 빈 프레임만 추가로 할당하면 된다.

### 2. 단점 (Cons)
- **주소 변환 오버헤드**: CPU가 메모리에 접근할 때마다 테이블을 참조해야 하므로 추가적인 시간이 소요된다. (이를 해결하기 위해 TLB 하드웨어 사용)
- **내부 단편화 발생**: 페이징의 경우, 마지막 조각이 페이지 크기보다 작으면 내부 단편화가 발생할 수 있다.

- **📢 섹션 요약 비유**: "물건을 찾기 위해 매번 장부를 확인해야 하는 번거로움(오버헤드)은 있지만, 창고 공간을 1%도 남기지 않고 쓸 수 있는 엄청난 이득이 있습니다."

+++

## Ⅳ. 현대 운영체제의 적용 사례 (Strategy)

### 1. 가상 메모리 (Virtual Memory)의 근간
- 비연속 할당은 가상 메모리 기술을 가능하게 하는 핵심 기반이다. 프로세스의 일부는 메모리에, 일부는 디스크(Swap)에 있어도 주소 변환을 통해 사용자는 이를 인식하지 못한다.

### 2. 하드웨어 지원: MMU (Memory Management Unit)
- 소프트웨어만으로는 주소 변환 속도를 감당할 수 없으므로, MMU라는 별도의 하드웨어 장치가 실시간으로 비연속 주소 매핑을 수행한다.

- **📢 섹션 요약 비유**: "현실의 땅(물리 메모리)은 좁고 조각나 있지만, 마법의 안경(MMU)을 쓰면 우리 집 마당이 끝없이 펼쳐진 것처럼 보이는 가상 현실 세계입니다."

+++

## Ⅴ. 결론 및 요약

### 1. 핵심 요약
- 비연속 메모리 할당은 '물리적 한계'를 '논리적 추상화'로 극복한 운영체제 설계의 위대한 승리다.

### 2. 향후 전망
- 64비트 컴퓨팅과 수백 GB의 대용량 메모리 시대가 오면서, 매핑 테이블의 크기를 줄이기 위한 다단계 페이징(Multi-level Paging)이나 대형 페이지(Huge Pages) 기술 등이 계속해서 발전하고 있다.

- **📢 섹션 요약 비유**: "결국 '어디에 있느냐'보다 '어떻게 찾느냐'가 더 중요하다는 정보 관리의 대원칙을 메모리 세계에 구현한 것입니다."
