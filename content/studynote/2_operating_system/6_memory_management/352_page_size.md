# 407. 페이지 크기 (Page Size)

#### 핵심 인사이트 (3줄 요약)

## Ⅰ. 개요

### 개념 정의

페이지 크기(Page Size)는 **가상 메모리와 물리 메모리를 나누는 고정 크기 블록의 크기**다. 페이지와 프레임은 같은 크기를 가진다.

### 💡 비유: 책 페이지
페이지 크기는 **책 페이지 크기**와 같다. 너무 작으면 많이 넘겨야 한다. 너무 크면 낭비가 많다.

### 페이지 크기 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 크기 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【일반적 페이지 크기】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  크기          이름              사용처                               │ │
│  │  ────────      ────────          ────────                             │ │
│  │  4 KB          Standard Page    대부분의 시스템                       │ │
│  │  2 MB          Huge Page        대용량 처리                          │ │
│  │  1 GB          Giant Page       데이터베이스, VM                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 크기 트레이드오프】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  특성           작은 페이지       큰 페이지                          │ │
│  │  ────────       ────────────       ────────────                     │ │
│  │  내부 단편화      적음               많음                              │ │
│  │  페이지 테이블    큼                 작음                              │ │
│  │  I/O 효율         낮음               높음                              │ │
│  │  TLB 커버리지     낮음               높음                              │ │
│  │  지역성 활용      좋음               나쁨                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 크기 계산 예시】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  가상 주소: 32비트, 페이지 크기: 4KB                                  │ │
│  │  • 오프셋 비트: log2(4096) = 12비트                                   │ │
│  │  • 페이지 번호 비트: 32 - 12 = 20비트                                │ │
│  │  • 페이지 수: 2^20 = 1M개                                            │ │
│  │                                                             │ │
│  │  가상 주소: 64비트, 페이지 크기: 4KB                                  │ │
│  │  • 오프셋 비트: 12비트                                                │ │
│  │  • 페이지 번호 비트: 64 - 12 = 52비트 (실제 사용은 48비트)            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 크기 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【내부 단편화】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  마지막 페이지의 낭비:                                                 │ │
│  │                                                             │ │
│  │  프로세스 크기: 10KB, 페이지 크기: 4KB                                 │ │
│  │  • 필요한 페이지: ceil(10/4) = 3개                                   │ │
│  │  • 실제 할당: 3 × 4KB = 12KB                                        │ │
│  │  • 내부 단편화: 12KB - 10KB = 2KB                                    │ │
│  │  • 평균 내부 단편화: 페이지 크기 / 2                                   │ │
│  │                                                             │ │
│  │  페이지가 클수록 내부 단편화 증가!                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TLB 커버리지】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  TLB 커버리지 = TLB 항목 수 × 페이지 크기                              │ │
│  │                                                             │ │
│  │  64개 TLB 항목일 때:                                                   │ │
│  │  • 4KB 페이지: 64 × 4KB = 256KB 커버                                  │ │
│  │  • 2MB 페이지: 64 × 2MB = 128MB 커버                                  │ │
│  │                                                             │ │
│  │  큰 페이지가 TLB 효율 더 좋음!                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Huge Page 사용 사례】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 데이터베이스: Oracle, PostgreSQL                                   │ │
│  │  • 가상 머신: KVM, Xen                                               │ │
│  │  • 빅데이터: Hadoop, Spark                                           │ │
│  │  • 메모리 집약적 애플리케이션                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 크기 확인】                                                      │
│  ──────────────────                                                  │
│  // Linux                                                             │
│  $ getconf PAGESIZE                                                   │
│  4096                                                                 │
│                                                                     │
│  // 또는                                                               │
│  $ cat /proc/sys/kernel/mm/hugepages/hugepages-2048kB/nr_hugepages  │
│  // 2MB Huge Page 수                                                   │
│                                                                     │
│  【Huge Page 설정】                                                        │
│  ──────────────────                                                  │
│  // 2MB Huge Page 1000개 할당                                          │
│  $ echo 1000 > /proc/sys/vm/nr_hugepages                             │
│                                                                     │
│  // 확인                                                               │
│  $ cat /proc/meminfo | grep Huge                                    │
│  HugePages_Total:    1000                                            │
│  HugePages_Free:     1000                                            │
│  HugePages_Rsvd:        0                                            │
│  Hugepagesize:       2048 kB                                         │
│                                                                     │
│  【Transparent Huge Pages】                                              │
│  ──────────────────                                                  │
│  // 자동으로 Huge Page 사용                                              │
│  $ cat /sys/kernel/mm/transparent_hugepage/enabled                   │
│  [always] madvise never                                               │
│                                                                     │
│  // 설정 변경                                                          │
│  $ echo always > /sys/kernel/mm/transparent_hugepage/enabled         │
│                                                                     │
│  【프로그램에서 Huge Page 사용】                                           │
│  ──────────────────                                                  │
│  #include <sys/mman.h>                                               │
│  #include <fcntl.h>                                                   │
│                                                                     │
│  // HugeTLB 파일 시스템 사용                                             │
│  int fd = open("/mnt/huge/page", O_CREAT | O_RDWR, 0755);           │
│  void* addr = mmap(NULL, 2*1024*1024, PROT_READ | PROT_WRITE,        │
│                     MAP_SHARED, fd, 0);                              │
│                                                                     │
│  // MAP_HUGETLB 플래그 사용                                              │
│  void* addr = mmap(NULL, 2*1024*1024, PROT_READ | PROT_WRITE,        │
│                     MAP_PRIVATE | MAP_ANONYMOUS | MAP_HUGETLB,        │
│                     -1, 0);                                          │
│                                                                     │
│  【데이터베이스 Huge Page 설정】                                           │
│  ──────────────────                                                  │
│  // PostgreSQL                                                        │
│  // postgresql.conf                                                   │
│  huge_pages = on            # or try                                 │
│  shared_buffers = 8GB       # Huge Page로 할당                         │
│                                                                     │
│  // Oracle                                                            │
│  // init.ora                                                          │
│  use_large_pages = ONLY                                               │
│                                                                     │
│  【성능 비교】                                                              │
│  ──────────────────                                                  │
│  // TLB 미스 비교                                                        │
│  $ perf stat -e dTLB-load-misses ./program_with_4k                  │
│  1,234,567 dTLB-load-misses                                           │
│                                                                     │
│  $ perf stat -e dTLB-load-misses ./program_with_2mb                  │
│  123,456 dTLB-load-misses    # 90% 감소!                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 메모리 관리 기본 단위 크기
• 일반: 4KB (표준), 2MB (Huge), 1GB (Giant)
• 트레이드오프: 내부 단편화 vs TLB 효율
• 작은 페이지: 단편화 적음, 테이블 큼
• 큰 페이지: 단편화 많음, TLB 효율 좋음
• 활용: DB, VM, 빅데이터
```
