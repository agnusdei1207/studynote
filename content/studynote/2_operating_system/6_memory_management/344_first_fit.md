# 최초 적합 (First-Fit)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 최초 적합(First-Fit)은 메모리의 시작 지점부터 탐색하여 요구 크기를 만족하는 '가장 먼저 발견된' 빈 공간에 즉시 프로세스를 할당하는 방식이다.
> 2. **강점**: 모든 빈 공간을 조사할 필요가 없으므로 탐색 속도가 매우 빠르며, 일반적으로 시간과 공간 효율성 면에서 최적 적합(Best-fit)에 뒤지지 않는 우수한 성능을 보인다.
> 3. **특징**: 메모리 앞부분에 할당이 집중되어 작은 단편화가 앞에 쌓이는 경향이 있으나, 뒷부분에는 큰 빈 공간을 유지할 가능성이 높다는 장점이 있다.

+++

## Ⅰ. 최초 적합 (First-Fit)의 개념 및 정의

### 1. 정의
- **최초 적합 (First-Fit)**은 동적 메모리 할당 시 가용 공간 리스트(Free List)를 처음부터 검색하다가, 프로세스의 크기보다 크거나 같은 첫 번째 빈 공간(Hole)을 찾으면 바로 할당을 수행하는 알고리즘이다.

### 2. 작동 원리
- 리스트 순회: `Hole_Size >= Request_Size` 조건을 만족하는 순간 검색 중단.
- 할당: 해당 Hole에 프로세스 배치 후, 남은 공간은 새로운 작은 Hole로 리스트에 유지.

### 3. 💡 비유: '가장 가까운 빈 자리 앉기'
- 버스에 탔을 때 빈 자리를 찾아 끝까지 걸어가지 않고, 입구에서 가장 가까운 첫 번째 빈 자리에 바로 앉는 것과 같다.

- **📢 섹션 요약 비유**: "배가 고플 때 맛집을 고르기 위해 온 동네를 돌아다니지 않고, 처음 눈에 띄는 식당에 바로 들어가는 속전속결 전략입니다."

+++

## Ⅱ. First-Fit의 메커니즘 및 ASCII 다이어그램

### 1. 할당 프로세스 가시화
- 검색은 항상 고정된 시작 지점(주로 저주소 영역)에서 출발한다.

### 2. ASCII 할당 시뮬레이션
```text
[ Memory Free List Status ]
Entry 1: Hole A (10KB)
Entry 2: Hole B (50KB)
Entry 3: Hole C (30KB)
Entry 4: Hole D (100KB)

[ Request: Process X (40KB) ]

Step 1: Check Hole A (10KB) -> Too Small (Skip)
Step 2: Check Hole B (50KB) -> 50 >= 40 (Match!)
Step 3: Allocate Process X in Hole B.

[ Resulting Memory Layout ]
... [Hole A: 10KB] | [Proc X: 40KB][Hole B': 10KB] | [Hole C: 30KB] ...
```

### 3. 기술적 특징: '메모리 전방 쏠림'
- 메모리의 저주소(앞부분)에 프로세스들이 밀집하게 된다. 이로 인해 앞부분은 파편화된 작은 구멍들이 많아지지만, 뒷부분(고주소)은 탐색이 덜 되어 커다란 빈 공간이 온전하게 보존되는 효과가 있다.

- **📢 섹션 요약 비유**: "책꽂이의 왼쪽부터 책을 꽂아 나가는 방식입니다. 왼쪽은 틈새가 많아지지만, 오른쪽은 넓은 빈 공간이 남게 됩니다."

+++

## Ⅲ. 성능 분석 및 장단점

### 1. 장점 (Pros)
- **최고의 속도**: 조건이 맞으면 탐색을 중단하므로 평균 탐색 시간이 알고리즘 중 가장 짧다.
- **공간 효율성**: 놀랍게도 전수조사를 하는 Best-Fit과 비교해도 전체적인 외부 단편화 발생 정도가 비슷하거나 때로는 더 좋다.
- **큰 공간 보존**: 고주소 영역의 큰 가용 공간을 건드리지 않고 보존하여 나중에 큰 프로세스가 들어올 때 유리하다.

### 2. 단점 (Cons)
- **전방 파편화**: 메모리 앞부분에 아주 작은 쓸모없는 공간(Tiny Holes)들이 누적되어, 나중에는 앞부분을 매번 건너뛰어야 하는 탐색 낭비가 생길 수 있다.

- **📢 섹션 요약 비유**: "고민 없이 빨리 결정해서 시간은 벌었지만, 앞쪽 자리에 작은 짐들이 너저분하게 쌓이는 부작용이 있습니다."

+++

## Ⅳ. 변형 및 최적화 전략 (Strategy)

### 1. 차기 적합 (Next-Fit)
- First-Fit의 단점인 '앞부분 탐색 낭비'를 해결하기 위해, 마지막으로 할당했던 지점부터 다음 탐색을 시작하는 방식이다. 메모리 전체에 할당을 균등하게 분산시킨다.

### 2. 리스트 정렬
- 주소 순으로 가용 리스트를 정렬해두면 First-Fit의 구현이 가장 효율적이다. 인접한 빈 공간을 합치는 '통합(Coalescing)' 작업도 주소 순 정렬 시 가장 빠르게 수행된다.

- **📢 섹션 요약 비유**: "어제 청소한 곳 다음부터 오늘 청소를 시작하는 식으로 부담을 나누는 지혜입니다."

+++

## Ⅴ. 결론 및 요약

### 1. 핵심 요약
- First-Fit은 '단순함의 미학'이 돋보이는 알고리즘이다. 복잡한 계산보다 빠른 반응성이 중요한 실제 운영체제 환경에서 가장 널리 선호되는 방식 중 하나다.

### 2. 실무적 의의
- 이론적으로는 Best-Fit이 좋아 보일 수 있으나, 탐색 시간 오버헤드와 미세 단편화 문제를 고려하면 First-Fit이 시스템 전체의 스루풋(Throughput) 면에서 더 나은 선택이 되는 경우가 많다.

- **📢 섹션 요약 비유**: "완벽한 자리를 찾느라 시간을 다 쓰는 것보다, 적당한 자리에 빨리 앉아 업무를 시작하는 것이 효율적이라는 교훈을 줍니다."

- **최악 적합 (Worst-Fit)**: 가장 큰 공간을 찾는 알고리즘.
- **외부 단편화 (External Fragmentation)**: 배치의 결과로 나타나는 증상.
