# 415. 가상 주소 공간 (Virtual Address Space)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스가 사용하는 논리적 주소 공간
> 2. **가치**: 격리와 보호, 큰 주소 공간 지원
> 3. **융합**: 가상 메모리, 메모리 보호와 연관

---

## Ⅰ. 개요

### 개념 정의

가상 주소 공간(Virtual Address Space)은 **프로세스가 사용하는 논리적 주소의 범위**다. 각 프로세스는 독립적인 가상 주소 공간을 가진다.

### 💡 비유: 개인 도서관
가상 주소 공간은 **개인 도서관**과 같다. 각자 자신만의 도서관이 있다. 다른 사람의 도서관은 볼 수 없다.

### 가상 주소 공간 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                가상 주소 공간 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【32비트 주소 공간 (4GB)】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  0xFFFFFFFF ┌─────────────────────────────────┐              │ │
│  │             │        커널 영역 (1GB)           │  ← 커널       │ │
│  │             │        (사용자 접근 불가)         │              │ │
│  │  0xC0000000 ├─────────────────────────────────┤              │ │
│  │             │                                   │              │ │
│  │             │        스택 ↓                    │  ← 사용자    │ │
│  │             │        (...)                     │              │ │
│  │             │                                   │              │ │
│  │             │        ↑ 힙                       │              │ │
│  │             │        (동적 할당)                 │              │ │
│  │             │                                   │              │ │
│  │             │        BSS (미초기화 데이터)       │              │ │
│  │             │        Data (초기화 데이터)        │              │ │
│  │             │        Text (코드)                │              │ │
│  │  0x00000000 └─────────────────────────────────┘              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【64비트 주소 공간】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  이론적: 2^64 = 16 EB (엑사바이트)                                     │ │
│  │  실제 사용: 48비트 = 256 TB (현재 구현)                                │ │
│  │                                                             │ │
│  │  0xFFFFFFFFFFFFFFFF ┌─────────────────────────────┐            │ │
│  │                      │      커널 영역               │            │ │
│  │  0xFFFF800000000000 ├─────────────────────────────┤            │ │
│  │                      │      (비할당)               │            │ │
│  │                      │                             │            │ │
│  │                      │      사용자 영역             │            │ │
│  │                      │      (코드, 데이터, 힙, 스택) │            │ │
│  │  0x0000000000000000 └─────────────────────────────┘            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프로세스 간 격리】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  프로세스 A               프로세스 B               프로세스 C      │ │
│  │  ┌─────────┐            ┌─────────┐            ┌─────────┐    │ │
│  │  │ 0x1000  │            │ 0x1000  │            │ 0x1000  │    │ │
│  │  │ 0x2000  │            │ 0x2000  │            │ 0x2000  │    │ │
│  │  │  ...    │            │  ...    │            │  ...    │    │ │
│  │  └─────────┘            └─────────┘            └─────────┘    │ │
│  │       │                       │                       │         │ │
│  │       │ 같은 가상 주소         │                       │         │ │
│  │       │ 다른 물리 주소!        │                       │         │ │
│  │       ▼                       ▼                       ▼         │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │                   물리 메모리                              │  │ │
│  │  │  [A의 데이터] [B의 데이터] [C의 데이터]                     │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                가상 주소 공간 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 영역 (VMA)】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  각 영역은 vm_area_struct로 관리:                                      │ │
│  │                                                             │ │
│  │  • vm_start: 시작 가상 주소                                             │ │
│  │  • vm_end: 끝 가상 주소                                                 │ │
│  │  • vm_flags: 권한 (읽기, 쓰기, 실행)                                    │ │
│  │  • vm_file: 매핑된 파일                                                 │ │
│  │                                                             │ │
│  │  영역 종류:                                                               │
│  │  • 코드 세그먼트 (Text)                                                 │ │
│  │  • 데이터 세그먼트 (Data, BSS)                                           │ │
│  │  • 힙 (Heap)                                                            │ │
│  │  • 메모리 매핑 (Memory Mapping)                                         │ │
│  │  • 스택 (Stack)                                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 공간 레이아웃】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ASLR (Address Space Layout Randomization):                           │ │
│  │  • 각 영역의 시작 주소 무작위화                                          │ │
│  │  • 보안 공격 방어                                                       │ │
│  │  • 코드, 데이터, 힙, 스택, 라이브러리                                   │ │
│  │                                                             │ │
│  │  PIE (Position Independent Executable):                                │ │
│  │  • 실행 파일 위치 독립                                                  │ │
│  │  • 어디서든 로드 가능                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【크기 제한】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ulimit -s: 스택 크기 제한                                               │ │
│  │  ulimit -v: 가상 메모리 제한                                             │ │
│  │  RLIMIT_AS: 주소 공간 크기 제한                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 공간 확인】                                                        │
│  ──────────────────                                                  │
│  // 프로세스의 메모리 맵 확인                                               │
│  $ cat /proc/self/maps                                               │
│  555555554000-555555555000 r--p 00000000 08:01 12345 /bin/cat        │
│  555555555000-555555556000 r-xp 00001000 08:01 12345 /bin/cat        │
│  555555556000-555555557000 r--p 00002000 08:01 12345 /bin/cat        │
│  555555557000-555555558000 r--p 00002000 08:01 12345 /bin/cat        │
│  555555558000-555555559000 rw-p 00003000 08:01 12345 /bin/cat        │
│  7ffff7dda000-7ffff7dfc000 r--p 00000000 08:01 54321 libc.so.6       │
│  7ffff7dfc000-7ffff7f51000 r-xp 00022000 08:01 54321 libc.so.6       │
│  ...                                                                  │
│  7ffffffde000-7ffffffff000 rw-p 00000000 00:00 0          [stack]   │
│  ffffffffff600000-ffffffffff601000 r-xp 00000000 00:00 0  [vsyscall] │
│                                                                     │
│  // 요약 정보                                                             │
│  $ cat /proc/self/statm                                              │
│  size resident shared text lib data dt                               │
│  1000   500     100   100  10  100  10                               │
│                                                                     │
│  【ASLR 확인】                                                              │
│  ──────────────────                                                  │
│  $ cat /proc/sys/kernel/randomize_va_space                          │
│  2  // 0=비활성, 1=부분, 2=완전                                           │
│                                                                     │
│  // 여러 실행에서 주소 변화 확인                                            │
│  $ for i in 1 2 3; do cat /proc/self/maps | grep stack; done        │
│  7ffc12345000-7ffc12346000 rw-p ... [stack]                          │
│  7ffc67890000-7ffc67891000 rw-p ... [stack]  // 주소 다름             │
│  7ffcabcd0000-7ffcabcd1000 rw-p ... [stack]                          │
│                                                                     │
│  【주소 공간 제한】                                                        │
│  ──────────────────                                                  │
│  // 현재 제한 확인                                                        │
│  $ ulimit -a | grep -E "stack|virtual"                              │
│  stack size              (kbytes, -s) 8192                            │
│  virtual memory          (kbytes, -v) unlimited                       │
│                                                                     │
│  // 제한 변경                                                            │
│  $ ulimit -s 16384  // 스택 크기 16MB                                    │
│  $ ulimit -v 4194304  // 가상 메모리 4GB                                 │
│                                                                     │
│  【프로그래밍에서 활용】                                                      │
│  ──────────────────                                                  │
│  #include <sys/resource.h>                                           │
│                                                                     │
│  // 주소 공간 제한 확인/설정                                               │
│  struct rlimit rl;                                                    │
│  getrlimit(RLIMIT_AS, &rl);                                          │
│  printf("Current: %lu MB\n", rl.rlim_cur / (1024*1024));             │
│  printf("Max: %lu MB\n", rl.rlim_max / (1024*1024));                 │
│                                                                     │
│  // 제한 설정                                                            │
│  rl.rlim_cur = 1024 * 1024 * 1024;  // 1GB                           │
│  setrlimit(RLIMIT_AS, &rl);                                          │
│                                                                     │
│  【메모리 영역 분석】                                                      │
│  ──────────────────                                                  │
│  // pmap으로 상세 분석                                                    │
│  $ pmap -x <pid>                                                      │
│  Address           Kbytes     RSS   Dirty Mode  Mapping              │
│  0000000000400000    44      44       0 r-x-- ls                    │
│  0000000000600000     4       4       4 r---- ls                    │
│  0000000000601000     4       4       4 rw--- ls                    │
│  00007f8b9a000000   1572     312       0 r-x-- libc-2.31.so         │
│  ...                                                                  │
│  00007ffce91f9000   132     132      12 rw---   [stack]             │
│  ---------------- ------- ------- -------                              │
│  total kB          12345    1234     123                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 프로세스가 사용하는 논리적 주소 공간
• 구조: 커널 영역 + 사용자 영역
• 구성: Text, Data, BSS, Heap, Stack
• 격리: 각 프로세스 독립적 주소 공간
• 보안: ASLR, PIE
• 제한: ulimit, RLIMIT_AS
```
