# 연속 메모리 할당 (Contiguous Memory Allocation)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연속 메모리 할당은 각 프로세스를 물리적 메모리의 단일한 연속된 공간(Contiguous Block)에 통째로 배치하는 가장 고전적인 메모리 관리 기법이다.
> 2. **가치**: 주소 변환 과정이 단순(Base + Offset)하여 하드웨어 구현이 용이하고 실행 속도가 빠르지만, 메모리 공간 낭비인 '단편화(Fragmentation)' 문제가 치명적이다.
> 3. **융합**: 고정 분할과 가변 분할 방식으로 나뉘며, 현대의 불연속 할당(페이징 등) 기술이 등장하기 전까지 운영체제 메모리 관리의 근간을 이루었다.

+++

## Ⅰ. 연속 메모리 할당의 개념 및 배경

### 1. 정의
- **연속 메모리 할당 (Contiguous Memory Allocation)**이란 프로세스의 논리적 주소 공간 전체가 물리적 메모리상에서도 중간에 끊김 없이 연속적으로 인접하여 존재하도록 할당하는 방식이다.

### 2. 등장 배경: '단순함의 미학'
- 컴퓨터 초기 시절, 메모리 관리 장치(MMU)는 매우 단순했다. 주소 변환을 복잡하게 할 성능이 부족했기에, 단순히 "프로세스 A는 100번지부터 200번지까지 써라"라고 지정해주는 방식이 가장 효율적이었다.

### 3. 💡 비유: '기차 좌석 예매'
- 연속 메모리 할당은 4명의 가족이 기차를 탈 때 반드시 **'옆으로 나란히 붙은 4자리'**를 예매하는 것과 같다. 가족이 흩어지지 않아 찾기(주소 변환)는 쉽지만, 중간중간 1~2자리씩 빈자리가 있어도 4자리가 연속되지 않으면 앉을 수 없는 문제가 생긴다.

- **📢 섹션 요약 비유**: "가족은 언제나 뭉쳐 있어야 한다는 단결력을 강조하지만, 그만큼 빈틈을 찾기 힘든 전략입니다."

+++

## Ⅱ. 연속 메모리 할당의 작동 원리 (Deep Dive)

### 1. 메모리 분할 구조
- 메모리는 크게 **운영체제 영역(Low Memory)**과 **사용자 프로세스 영역(High Memory)**으로 나뉜다.
- 사용자 영역은 다시 여러 개의 분할(Partition)로 쪼개어 프로세스를 수용한다.

### 2. ASCII 다이어그램: 연속 할당 구조
```text
 [ Physical Memory ]
 +-----------------+ 0
 | Operating System|
 +-----------------+ 100
 |    Process P1   | (Contiguous)
 +-----------------+ 250
 |    Process P2   | (Contiguous)
 +-----------------+ 400
 |      Hole       | (Unused Space)
 +-----------------+ 500
 |    Process P3   | (Contiguous)
 +-----------------+ Max
```

### 3. 기술적 상세 해설
- **재배치 레지스터(Relocation Register)**: 프로세스가 시작되는 물리 주소를 저장.
- **한계 레지스터(Limit Register)**: 프로세스의 크기를 저장하여 메모리 보호를 수행.
- **주소 변환**: 물리 주소 = 재배치 레지스터 값 + 논리 주소 (단, 논리 주소 < 한계 레지스터).

- **📢 섹션 요약 비유**: "시작점과 길이만 알면 바로 위치를 찾을 수 있는 '직선 거리' 기반의 길 찾기입니다."

+++

## Ⅲ. 주요 특징 및 분석 (Fragmentation)

### 1. 단편화 문제 (Fragmentation)
- **외부 단편화 (External Fragmentation)**: 빈 공간(Hole)들의 총합은 프로세스를 수용하기에 충분하지만, 공간들이 쪼개져 있어 연속된 큰 공간이 없는 상태.
- **내부 단편화 (Internal Fragmentation)**: 프로세스에게 할당된 공간이 프로세스 크기보다 커서 내부에서 낭비되는 공간.

### 2. 비교 분석 테이블
| 구분 | 고정 분할 (Fixed) | 가변 분할 (Variable) |
|:---|:---|:---|
| **분할 크기** | 모두 동일하거나 미리 정해짐 | 프로세스 크기에 맞춰 동적 생성 |
| **단편화** | 내부 단편화 발생 | 외부 단편화 발생 |
| **관리 용이성** | 매우 쉬움 | 복잡함 (Hole 관리 필요) |

- **📢 섹션 요약 비유**: "사이즈가 정해진 옷에 몸을 맞추느냐(고정), 몸에 맞춰 옷을 재단하느냐(가변)의 고민입니다."

+++

## Ⅳ. 실무 적용 및 문제 해결 전략 (Strategy)

### 1. 가변 분할 할당 전략 (Dynamic Storage-Allocation)
- **First-fit (최초 적합)**: 첫 번째로 발견한 충분히 큰 빈 공간에 할당 (가장 빠름).
- **Best-fit (최적 적합)**: 프로세스 크기와 차이가 가장 적은 공간에 할당 (공간 효율 좋지만 탐색 시간 소요).
- **Worst-fit (최악 적합)**: 가장 큰 공간에 할당 (남는 공간을 크게 만들어 다른 프로세스를 받으려 함, 효율 낮음).

### 2. 압축 (Compaction)
- 외부 단편화를 해결하기 위해 프로세스들을 메모리의 한쪽으로 몰아서 빈 공간들을 하나로 합치는 작업. 하지만 실행 중에 주소를 옮겨야 하므로 **Dynamic Relocation** 지원이 필수적이며 오버헤드가 크다.

- **📢 섹션 요약 비유**: "교실의 빈자리를 만들기 위해 모든 학생을 앞줄로 바짝 당겨 앉히는 대공사와 같습니다."

+++

## Ⅴ. 기대효과 및 결론

### 1. 기대효과
- **구현의 단순성**: 별도의 복잡한 매핑 테이블 없이 하드웨어 레지스터만으로 메모리 관리가 가능하다.
- **순차 접근 성능**: 데이터가 물리적으로 인접해 있어 캐시 효율이 좋고 순차적인 데이터 처리에 유리하다.

### 2. 결론 및 향후 전망
- 연속 메모리 할당의 한계(단편화)를 극복하기 위해 현대 운영체제는 **불연속 메모리 할당(Paging, Segmentation)** 방식을 사용한다. 하지만 임베디드 시스템이나 특수 목적의 고성능 커널 루틴에서는 여전히 연속 할당의 속도와 단순함이 가치를 발휘한다.

- **📢 섹션 요약 비유**: "흩어지면 살고 뭉치면 죽는다? 메모리에서는 흩어져야(불연속) 더 잘 살 수 있습니다."
