# 410. NUMA (Non-Uniform Memory Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU별 메모리 접근 속도 차이
> 2. **가치**: 대규모 시스템 성능 최적화
> 3. **융합**: 멀티프로세서, 메모리 관리와 연관

---

## Ⅰ. 개요

### 개념 정의

NUMA(Non-Uniform Memory Access)는 **각 CPU가 자신에게 가까운 메모리에 더 빠르게 접근하는 아키텍처**다. CPU-메모리 쌍이 여러 개 있는 시스템이다.

### 💡 비유: 지역 창고
NUMA는 **지역 창고**와 같다. 내 지역 창고는 가깝다. 다른 지역 창고는 멀다. 가까운 곳이 더 빠르다.

│                NUMA 구조                                               │
├─────────────────────────────────────────────────────────────────────┤

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────┐   ┌─────────────────────────┐    │ │
│  │  │    NUMA Node 0          │   │    NUMA Node 1          │    │ │

│  │  │  │ CPU │────│Memory │  │   │  │ CPU │────│Memory │  │    │ │
│  │  │  │  0  │    │  0    │  │   │  │  1  │    │  1    │  │    │ │
│  │  │  └─────┘    └───────┘  │   │  └─────┘    └───────┘  │    │ │
│  │  │      │            │    │   │      │            │    │    │ │
│  │  │      │   로컬     │    │   │      │   로컬     │    │    │ │
│  │  │      │   빠름     │    │   │      │   빠름     │    │    │ │
│  │  └──────┼────────────┼────┘   └──────┼────────────┼────┘    │ │
│  │         │            │               │            │          │ │
│  │         └────────────┼───────────────┘            │          │ │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  로컬 메모리 접근: 100ns                                               │ │
│  │  원격 메모리 접근: 150-300ns (1.5-3배 느림)                            │ │
│  │                                                             │ │
│  │  → 메모리 할당 시 로컬 메모리 우선!                                    │ │

│  │  • 모든 CPU가 메모리에 동일한 속도로 접근                              │ │

│  │  • CPU마다 메모리 접근 속도 다름                                       │ │

│  │                                                             │ │

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                NUMA 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NUMA 노드】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  각 NUMA 노드:                                                          │ │
│  │  • CPU(들)                                                            │ │
│  │  • 로컬 메모리                                                          │ │
│  │  • I/O 장치                                                             │ │
│  │                                                             │ │
│  │  노드 간 통신:                                                          │ │
│  │  • 인터커넥트 (Interconnect)                                           │ │
│  │  • QPI, HyperTransport, Infinity Fabric                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NUMA 스케줄링】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CPU 친화도 (CPU Affinity):                                            │ │
│  │  • 프로세스를 특정 노드에 바인딩                                        │ │
│  │  • 로컬 메모리 접근 보장                                                │ │
│  │                                                             │ │
│  │  메모리 정책 (Memory Policy):                                          │ │
│  │  • MPOL_DEFAULT: 기본 정책                                             │ │
│  │  • MPOL_BIND: 특정 노드만 사용                                          │ │
│  │  • MPOL_INTERLEAVE: 여러 노드에 분산                                    │ │
│  │  • MPOL_PREFERRED: 특정 노드 우선                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NUMA 문제점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 메모리 불균형:                                                      │ │
│  │     • 일부 노드만 과부하                                                │ │
│  │                                                             │ │
│  │  2. 원격 메모리 접근:                                                    │ │
│  │     • 잘못된 할당으로 성능 저하                                         │ │
│  │                                                             │ │
│  │  3. 마이그레이션 오버헤드:                                               │ │
│  │     • 페이지 이동 비용                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NUMA 정보 확인】                                                        │
│  ──────────────────                                                  │
│  // 노드 정보                                                            │
│  $ numactl --hardware                                                │
│  available: 2 nodes (0-1)                                            │
│  node 0 cpus: 0 1 2 3                                                │
│  node 0 size: 32768 MB                                               │
│  node 1 cpus: 4 5 6 7                                                │
│  node 1 size: 32768 MB                                               │
│  node distances:                                                       │
│  node   0   1                                                          │
│    0:  10  21                                                          │
│    1:  21  10                                                          │
│  // 로컬=10, 원격=21 (2.1배 느림)                                        │
│                                                                     │
│  // 노드별 메모리 사용                                                    │
│  $ numastat                                                          │
│  node0 node1                                                          │
│  numa_hit   1234567 1234567                                          │
│  numa_miss        0       0                                          │
│  numa_foreign     0       0                                          │
│  // numa_hit: 로컬 접근, numa_miss: 원격 접근                           │
│                                                                     │
│  【NUMA 정책 설정】                                                        │
│  ──────────────────                                                  │
│  // 특정 노드에서만 실행                                                   │
│  $ numactl --cpunodebind=0 --membind=0 ./program                     │
│                                                                     │
│  // 인터리브 메모리 할당                                                   │
│  $ numactl --interleave=all ./program                                │
│                                                                     │
│  // 선호 노드 지정                                                        │
│  $ numactl --preferred=1 ./program                                   │
│                                                                     │
│  【프로그래밍에서 NUMA】                                                    │
│  ──────────────────                                                  │
│  #include <numa.h>                                                   │
│  #include <numaif.h>                                                 │
│                                                                     │
│  // NUMA 사용 가능 확인                                                  │
│  if (numa_available() == -1) {                                        │
│      printf("NUMA not available\n");                                   │
│  }                                                                  │
│                                                                     │
│  // 노드 수 확인                                                         │
│  int numNodes = numa_num_configured_nodes();                          │
│                                                                     │
│  // 특정 노드에 메모리 할당                                                 │
│  void* mem = numa_alloc_onnode(1024*1024, 0);  // 노드 0에 1MB        │
│  numa_free(mem, 1024*1024);                                          │
│                                                                     │
│  // CPU 선호도 설정                                                      │
│  cpu_set_t cpuset;                                                   │
│  CPU_ZERO(&cpuset);                                                   │
│  CPU_SET(0, &cpuset);  // CPU 0에 바인딩                               │
│  sched_setaffinity(0, sizeof(cpuset), &cpuset);                      │
│                                                                     │
│  【데이터베이스 NUMA 설정】                                                 │
│  ──────────────────                                                  │
│  // PostgreSQL                                                         │
│  // postgresql.conf                                                    │
│  num_memnodes = 2                                                      │
│  // 각 노드에 워커 분산                                                    │
│                                                                     │
│  // MySQL                                                              │
│  $ numactl --interleave=all mysqld                                   │
│                                                                     │
│  // Oracle                                                             │
│  // NUMA 인식 활성화                                                     │
│  _enable_NUMA_interleave = TRUE                                       │
│                                                                     │
│  【NUMA 밸런싱】                                                            │
│  ──────────────────                                                  │
│  // 자동 NUMA 밸런싱 확인                                                 │
│  $ cat /proc/sys/kernel/numa_balancing                               │
│  1  // 1=활성, 0=비활성                                                   │
│                                                                     │
│  // 설정                                                                │
│  $ echo 1 > /proc/sys/kernel/numa_balancing                          │
│                                                                     │
│  // 밸런싱은 커널이 자동으로 페이지 마이그레이션                               │
│  // 단점: 오버헤드 발생 가능                                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
