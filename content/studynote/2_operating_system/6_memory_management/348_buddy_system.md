# 437. 버디 시스템 (Buddy System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 2의 거듭제곱 블록으로 메모리 관리
> 2. **가치**: 외부 단편화 최소화, 빠른 병합
> 3. **융합**: 물리 메모리 관리, 슬랩과 연관

---

## Ⅰ. 개요

### 개념 정의

버디 시스템(Buddy System)은 **메모리를 2의 거듭제곱 크기 블록으로 관리하는 할당 알고리즘**이다. 블록 분할과 병합으로 단편화를 줄인다.

### 💡 비유: 종이 접기
버디 시스템은 **종이 접기**와 같다. 큰 종이를 반으로 접고, 또 반으로 접는다. 다시 펼칠 때는 반대로 합친다.

### 버디 시스템 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                버디 시스템 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  • 메모리를 2^n 크기의 블록으로 관리                                        │ │
│  │  • 블록 크기: 1, 2, 4, 8, 16, 32, 64, ... 페이지                      │ │
│  │  • 같은 크기의 인접한 두 블록은 "버디"                                    │ │
│  │  • 할당: 큰 블록을 분할                                                   │ │
│  │  • 해제: 버디가 비어있으면 병합                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【블록 분할 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  초기: [        64KB 블록        ]                                     │ │
│  │                                                             │ │
│  │  16KB 요청 시:                                                         │ │
│  │  1. 64KB를 32KB + 32KB로 분할                                          │ │
│  │  2. 앞의 32KB를 16KB + 16KB로 분할                                      │ │
│  │  3. 앞의 16KB 할당                                                      │ │
│  │                                                             │ │
│  │  [16KB 할당][16KB][    32KB    ]                                     │ │
│  │       ↑       ↑         ↑                                            │ │
│  │     사용중   free      free                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버디 병합 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  16KB 해제 시:                                                          │ │
│  │  [free][16KB][    32KB    ]                                           │ │
│  │  → 옆 16KB가 free → 병합                                               │ │
│  │  [   32KB free   ][    32KB    ]                                      │ │
│  │  → 옆 32KB가 free → 병합                                               │ │
│  │  [        64KB free           ]                                       │ │
│  │                                                             │ │
│  │  버디 조건:                                                             │ │
│  │  • 같은 크기                                                              │ │
│  │  • 인접한 위치                                                           │ │
│  │  • 둘 다 free                                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                버디 시스템 상세                                         │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 외부 단편화 최소화: 병합으로 큰 블록 복원                             │ │
│  │  • 빠른 병합: O(1) - 주소 XOR로 버디 찾기                               │ │
│  │  • 간단한 구현: 2의 거듭제곱만 관리                                      │ │
│  │  • DMA 친화적: 연속된 물리 메모리 보장                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 내부 단편화: 2의 거듭제곱 크기만 할당                                 │ │
│  │    예: 17KB 요청 → 32KB 할당 (15KB 낭비)                               │ │
│  │  • 분할 비용: 큰 블록에서 작은 블록 분할 시                               │ │
│  │  • 작은 할당에 비효율적                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux 버디 시스템】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Order (2^n 페이지):                                                    │ │
│  │  Order 0: 1 페이지   (4KB)                                              │ │
│  │  Order 1: 2 페이지   (8KB)                                              │ │
│  │  Order 2: 4 페이지   (16KB)                                             │ │
│  │  ...                                                                   │ │
│  │  Order 10: 1024 페이지 (4MB)                                            │ │
│  │                                                             │ │
│  │  각 Order마다 Free List 유지                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【버디 찾기 알고리즘】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  버디 주소 = 블록 주소 XOR (1 << order)                                 │ │
│  │                                                             │ │
│  │  예: Order 2 (4페이지), 블록 주소 0x8000                                │ │
│  │  버디 = 0x8000 XOR 0x4000 = 0xC000                                     │ │
│  │                                                             │ │
│  │  매우 빠른 버디 찾기 (O(1))                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 버디 시스템 확인】                                            │
│  ──────────────────                                                  │
│  // 각 Order별 사용 가능한 블록 수                                      │
│  $ cat /proc/buddyinfo                                                │
│  Node 0, zone      DMA      0      1      2      3      2      1    │
│  Node 0, zone    Normal    123    456    234    123     45     12    │
│  //               Order:   0      1      2      3      4      5      │
│  //               Size:   4KB    8KB   16KB   32KB   64KB  128KB    │
│                                                                     │
│  // 상세 정보                                                            │
│  $ cat /proc/pagetypeinfo                                            │
│  Page block order: 9                                                 │
│  Pages per block:  512                                               │
│                                                                     │
│  Order      0      1      2      3      4      5      6 ...        │
│  DMA        5      3      2      1      1      0      0 ...        │
│  Normal   456    234    123     45     12      5      2 ...        │
│                                                                     │
│  【버디 시스템 동작 예시】                                              │ |
│  ──────────────────                                                  │
│  // 4KB 할당 요청                                                       │
│  // Order 0에서 free list 확인                                         │
│  // 있으면 반환, 없으면 Order 1에서 분할                                 │
│                                                                     │
│  // Order 0 비었을 때:                                                  │
│  // 1. Order 1의 8KB 블록을 두 개의 4KB로 분할                          │
│  // 2. 하나는 할당, 하나은 Order 0 free list에 추가                     │
│                                                                     │
│  【커널 메모리 할당】                                                    │
│  ──────────────────                                                  │
│  // 커널에서 페이지 단위 할당                                            │
│  #include <linux/gfp.h>                                               │
│                                                                     │
│  // 1페이지 (4KB) 할당                                                   │
│  struct page *page = alloc_pages(GFP_KERNEL, 0);                      │
│  void *addr = page_address(page);                                     │
│  __free_pages(page, 0);                                               │
│                                                                     │
│  // 4페이지 (16KB) 할당                                                  │
│  struct page *pages = alloc_pages(GFP_KERNEL, 2);  // Order 2        │
│  __free_pages(pages, 2);                                              │
│                                                                     │
│  // 연속 메모리 할당                                                     │
│  void *buf = kmalloc(8192, GFP_KERNEL);  // 버디에서 할당              │
│  kfree(buf);                                                           │
│                                                                     │
│  【버디 시스템 디버깅】                                                  │
│  ──────────────────                                                  │
│  // 버디 시스템 통계                                                      │
│  $ cat /proc/vmstat | grep buddy                                     │
│  buddy_alloc 12345                                                   │
│  buddy_free 12340                                                    │
│                                                                     │
│  // 페이지 할당 실패 추적                                                │
│  $ echo 1 > /proc/sys/vm/pagealloc_debug                             │
│  // 또는                                                               │
│  $ trace-cmd record -e kmem:mm_page_alloc ./program                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 2의 거듭제곱 블록으로 메모리 관리
• 구조: Order 0~10 (4KB~4MB)
• 할당: 필요한 크기로 분할
• 해제: 버디와 병합
• 장점: 외부 단편화 최소화, 빠른 병합
• 단점: 내부 단편화
• 확인: /proc/buddyinfo
```
