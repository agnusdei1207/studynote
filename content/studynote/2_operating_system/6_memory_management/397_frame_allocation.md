# 419. 프레임 할당 (Frame Allocation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 메모리 프레임을 프로세스에 배분
> 2. **가치**: 공정하고 효율적인 메모리 분배
> 3. **융합**: 페이지 교체, 스래싱과 연관

---

## Ⅰ. 개요

### 개념 정의

프레임 할당(Frame Allocation)은 **물리 메모리의 프레임을 각 프로세스에 배분하는 정책**이다. 얼마나 많은 프레임을 줄지 결정한다.

### 💡 비유: 도서관 자리 배정
프레임 할당은 **도서관 자리 배정**과 같다. 한 사람에게 자리를 몇 개 줄지 정한다. 너무 많이 주면 다른 사람이 불편하다.

### 프레임 할당 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                프레임 할당 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【할당 정책】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 균등 할당 (Equal Allocation):                                     │ │
│  │     • 모든 프로세스에 동일한 프레임 수                                   │ │
│  │     • 간단하지만 비효율적                                               │ │
│  │     • 예: 100프레임 ÷ 5프로세스 = 20프레임/프로세스                      │ │
│  │                                                             │ │
│  │  2. 비례 할당 (Proportional Allocation):                              │ │
│  │     • 프로세스 크기에 비례하여 할당                                      │ │
│  │     • 큰 프로세스 → 많은 프레임                                         │ │
│  │     • 작은 프로세스 → 적은 프레임                                       │ │
│  │                                                             │ │
│  │  3. 우선순위 할당 (Priority Allocation):                               │ │
│  │     • 높은 우선순위 프로세스에 더 많은 프레임                             │ │
│  │     • 중요 프로세스 성능 보장                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【비례 할당 계산】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  총 프레임: m개                                                         │ │
│  │  프로세스: P1(S1), P2(S2), P3(S3) (S=크기)                             │ │
│  │  총 크기: S = S1 + S2 + S3                                             │ │
│  │                                                             │ │
│  │  P1 할당 = m × (S1 / S)                                                │ │
│  │  P2 할당 = m × (S2 / S)                                                │ │
│  │  P3 할당 = m × (S3 / S)                                                │ │
│  │                                                             │ │
│  │  예: m=100, S1=20KB, S2=30KB, S3=50KB                                 │ │
│  │  P1 = 100 × (20/100) = 20 프레임                                       │ │
│  │  P2 = 100 × (30/100) = 30 프레임                                       │ │
│  │  P3 = 100 × (50/100) = 50 프레임                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최소 프레임 수】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  각 프로세스는 최소 프레임이 필요:                                        │ │
│  │                                                             │ │
│  │  • 명령어 실행에 필요한 페이지들                                          │ │
│  │  • 최소: 명령어 + 데이터 페이지                                          │ │
│  │  • 직접 주소 지정: 1 페이지                                               │ │
│  │  • 간접 주소 지정: 2 페이지                                               │ │
│  │  • 지역성 고려                                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                프레임 할당 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전역 vs 지역 교체】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전역 교체 (Global Replacement):                                      │ │
│  │  • 모든 프로세스의 프레임에서 교체 가능                                 │ │
│  │  • 프로세스가 다른 프로세스의 프레임 뺏을 수 있음                        │ │
│  │  • 처리량 높음                                                         │ │
│  │                                                             │ │
│  │  지역 교체 (Local Replacement):                                       │ │
│  │  • 자신에게 할당된 프레임에서만 교체                                     │ │
│  │  • 할당량 보장                                                         │ │
│  │  • 성능 예측 가능                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스래싱 방지】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  프레임이 너무 적으면 스래싱 발생:                                       │ │
│  │                                                             │ │
│  │  • 페이지 폴트 빈번                                                     │ │
│  │  • CPU 이용률 저하                                                      │ │
│  │  • 시스템 전체 느려짐                                                   │ │
│  │                                                             │ │
│  │  해결:                                                                  │ │
│  │  • 작업 집합 모델 사용                                                  │ │
│  │  • 페이지 폴트 빈도(PFF) 사용                                           │ │
│  │  • L=Σ(모든 프로세스의 지역성) ≤ 총 프레임                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동적 할당】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  실행 중 프레임 할당 조정:                                              │ │
│  │                                                             │ │
│  │  • 페이지 폴트율 모니터링                                               │ │
│  │  • 높으면: 프레임 추가                                                  │ │
│  │  • 낮으면: 프레임 회수                                                  │ │
│  │  • 작업 집합 크기에 맞춤                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프로세스별 메모리 확인】                                              │
│  ──────────────────                                                  │
│  // 물리 메모리 사용량                                                    │
│  $ cat /proc/<pid>/statm                                             │
│  size resident shared text lib data dt                               │
│  1000   500     100   100  10  100  10                               │
│  // resident = 실제 할당된 프레임 수 (페이지 단위)                         │
│                                                                     │
│  // 상세 메모리 정보                                                      │
│  $ cat /proc/<pid>/status | grep -E "VmRSS|VmSize"                  │
│  VmRSS:     2048 kB  // 실제 물리 메모리                                │
│  VmSize:    4096 kB  // 가상 메모리                                    │
│                                                                     │
│  【시스템 전체 프레임 정보】                                              │
│  ──────────────────                                                  │
│  $ cat /proc/meminfo | grep -E "MemTotal|MemFree|Active|Inactive"  │
│  MemTotal:       16383892 kB                                        │
│  MemFree:         1234567 kB                                        │
│  Active:          8192456 kB  // 활성 프레임                          │
│  Inactive:        4096123 kB  // 비활성 프레임                        │
│                                                                     │
│  【프레임 할당 제한】                                                    │
│  ──────────────────                                                  │
│  // cgroup으로 메모리 제한                                               │
│  $ echo 1G > /sys/fs/cgroup/memory/mygroup/memory.limit_in_bytes    │
│  $ echo <pid> > /sys/fs/cgroup/memory/mygroup/tasks                  │
│                                                                     │
│  // ulimit으로 제한                                                      │
│  $ ulimit -v 1048576  // 가상 메모리 1GB 제한                           │
│                                                                     │
│  【동적 프레임 할당 조정】                                                │
│  ──────────────────                                                  │
│  // swappiness (스왑 vs 페이지 회수 선호도)                               │
│  $ cat /proc/sys/vm/swappiness                                      │
│  60  // 높을수록 스왑 선호, 낮을수록 페이지 회수 선호                        │
│                                                                     │
│  // min_free_kbytes (최소 여유 메모리)                                   │
│  $ cat /proc/sys/vm/min_free_kbytes                                 │
│  67584  // 이 정도는 항상 확보                                            │
│                                                                     │
│  【프로그램에서 메모리 할당 확인】                                          │
│  ──────────────────                                                  │
│  #include <sys/resource.h>                                          │
│  #include <stdio.h>                                                 │
│                                                                     │
│  // 현재 메모리 사용량 확인                                                │
│  struct rusage usage;                                                │
│  getrusage(RUSAGE_SELF, &usage);                                    │
│  printf("Max RSS: %ld KB\n", usage.ru_maxrss);                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
