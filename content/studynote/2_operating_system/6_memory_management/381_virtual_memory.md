# 396. 가상 메모리 (Virtual Memory)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 메모리보다 큰 주소 공간 제공
> 2. **가치**: 메모리 확장, 프로세스 격리
> 3. **융합**: 페이징, 요구 페이징, 스와핑과 연관

---

## Ⅰ. 개요

### 개념 정의

가상 메모리(Virtual Memory)는 **물리 메모리보다 큰 논리적 주소 공간을 프로세스에 제공하는 기법**이다. 디스크를 메모리의 확장으로 사용한다.

### 💡 비유: 가상 도서관
가상 메모리는 **가상 도서관**과 같다. 실제 책장은 작지만, 전체 도서관 카드를 가진다. 필요할 때만 책을 가져온다.

│  │                                                             │ │
│  │  ┌───────────────────────────────────────────────────────┐   │ │
│  │  │ 가상 주소 공간 (Virtual Address Space)                    │   │ │
│  │  │ • 각 프로세스마다 독립적                                    │   │ │
│  │  │ • 연속적인 주소 공간                                        │   │ │
│  │  │ • 실제보다 클 수 있음                                       │   │ │

│  │                          │                                       │ │
│  │                          │ MMU 변환                               │ │
│  │                          ▼                                       │ │
│  │  ┌───────────────────────────────────────────────────────┐   │ │
│  │  │ 물리 메모리 (Physical Memory)                             │   │ │
│  │  │ • 실제 RAM                                                │   │ │
│  │  │ • 제한된 크기                                              │   │ │
│  │  │ • 프레임 단위 관리                                          │   │ │
│  │  └───────────────────────────────────────────────────────┘   │ │
│  │                          │                                       │ │
│  │                          │ 스와핑                                 │ │
│  │                          ▼                                       │ │
│  │  ┌───────────────────────────────────────────────────────┐   │ │
│  │  │ 스왑 공간 (Swap Space)                                     │   │ │
│  │  │ • 디스크의 일부                                             │   │ │
│  │  │ • 메모리 확장                                               │   │ │
│  │  │ • 느린 접근 속도                                            │   │ │
│  │  └───────────────────────────────────────────────────────┘   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 변환】                                                            │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  가상 주소                        물리 주소                                 │ │

│  │  │ 가상 페이지 │ 오프셋    │      │ 물리 프레임 │ 오프셋    │          │ │

│  └───────────────────────────────────────────────────────────────┘ │

│  │  1. 메모리 확장: 물리 메모리보다 큰 프로그램 실행 가능                 │ │
│  │  2. 프로세스 격리: 각 프로세스의 독립적 주소 공간                            │ │
│  │  3. 메모리 보호: 페이지 단위 접근 권한 관리                              │ │
│  │  4. 메모리 공유: 페이지 단위 메모리 공유                                   │ │

│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                가상 메모리 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구현 방법】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 요구 페이징 (Demand Paging):                                       │ │
│  │     • 필요할 때만 페이지 로드                                           │ │
│  │     • 페이지 폴트 기반                                                 │ │
│  │                                                             │ │
│  │  2. 스와핑 (Swapping):                                                │ │
│  │     • 전체 프로세스를 디스크로 이동                                     │ │
│  │     • 메모리 부족 시 사용                                               │ │
│  │                                                             │ │
│  │  3. 세그먼테이션 (Segmentation):                                      │ │
│  │     • 논리적 단위로 메모리 분할                                         │ │
│  │     • 가변 크기 세그먼트                                              │ │
│  │                                                             │ │
│  │  4. 세그먼트 페이징:                                                    │ │
│  │     • 세그먼테이션 + 페이징 결합                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【가상 주소 공간 구조】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Linux 프로세스 가상 주소 공간:                                         │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │ 0xFFFFFFFF │ 커널 공간 (Kernel Space)                   │    │ │
│  │  ├───────────────────────────────────────────────────────┤    │ │
│  │  │            │ 스택 (Stack) ↓                            │    │ │
│  │  ├───────────────────────────────────────────────────────┤    │ │
│  │  │            │ (빈 공간)                                 │    │ │
│  │  ├───────────────────────────────────────────────────────┤    │ │
│  │  │            │ 힙 (Heap) ↑                              │    │ │
│  │  ├───────────────────────────────────────────────────────┤    │ │
│  │  │            │ BSS (초기화 안 된 데이터)                   │    │ │
│  │  ├───────────────────────────────────────────────────────┤    │ │
│  │  │            │ 데이터 (초기화된 데이터)                    │    │ │
│  │  ├───────────────────────────────────────────────────────┤    │ │
│  │  │            │ 텍스트 (코드)                               │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【오버커밋】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  가상 메모리 합계 > 물리 메모리 + 스왑                                   │ │
│  │                                                             │ │
│  │  이유:                                                                 │ │
│  │  • 모든 프로세스가 할당된 메모리를 다 사용하지 않음                       │ │
│  │  • Copy-on-Write로 실제 복사 미루기                                    │ │
│  │                                                             │ │
│  │  제어:                                                                 │ │
│  │  • /proc/sys/vm/overcommit_memory                                     │ │
│  │  • 0: 휴리용 계산 (기본)                                               │ │
│  │  • 1: 오버커밋 허용                                                     │ │
│  │  • 2: 오버커밋 금지                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  $ echo 0 > /proc/sys/vm/overcommit_memory                          │

│  $ dd if=/dev/zero of=/swapfile bs=1G count=8                       │

│                                                                     │

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 물리 메모리보다 큰 논리적 주소 공간
• 계층: 가상 주소 → 물리 메모리 → 스왑 공간
• 구현: 요구 페이징, 스와핑, 세그먼테이션
• 이점: 메모리 확장, 격리, 보호, 공유
• 구조: 텍스트, 데이터, 힙, 스택
• 오버커밋: 가상 메모리 합 > 물리 메모리 + 스왑
```
