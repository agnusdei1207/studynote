# 434. 메모리 풀 (Memory Pool)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 미리 할당된 메모리 블록 관리
> 2. **가치**: 할당/해제 성능 향상
> 3. **융합**: 단편화, 슬랩 할당자와 연관

---

## Ⅰ. 개요

### 개념 정의

메모리 풀(Memory Pool)은 **미리 할당된 메모리 블록들을 풀로 관리하여 빠르게 할당/해제하는 기법**이다. 동적 할당 오버헤드를 줄인다.

### 💡 비유: 도서관 대출
메모리 풀은 **도서관 대출**과 같다. 책을 매번 사는 게 아니라 도서관에서 빌린다. 반납하면 다시 빌릴 수 있다.

### 메모리 풀 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 풀 구조                                           │|
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  전통적 malloc/free:                                                    │
│  │  요청 시 → 메모리 검색 → 할당 → 반환                                     │ │
│  │  해제 시 → 병합 → 리스트에 추가                                          │ │
│  │  (오버헤드 큼)                                                           │ │
│  │                                                             │ │
│  │  메모리 풀:                                                               │
│  │  미리 큰 블록 할당 → 고정 크기 블록으로 분할 → 리스트 관리                 │ │
│  │  할당: 리스트에서 꺼내기만 하면 됨 (빠름!)                                 │ │
│  │  해제: 리스트에 다시 넣기만 하면 됨 (빠름!)                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 풀 구조】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────┐          │ │
│  │  │              Memory Pool (32바이트 블록)             │          │ │
│  │  ├─────────────────────────────────────────────────────┤          │ │
│  │  │ ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐   │          │ │
│  │  │ │ Free  │→│ Free  │→│ Used  │→│ Free  │→│ Used  │   │          │ │
│  │  │ │ Block │ │ Block │ │ Block │ │ Block │ │ Block │   │          │ │
│  │  │ └───────┘ └───────┘ └───────┘ └───────┘ └───────┘   │          │ │
│  │  │     ↑           ↑                   ↑               │          │ │
│  │  │     └───────────┴───────────────────┘               │          │ │
│  │  │              Free List (연결 리스트)                 │          │ │
│  │  └─────────────────────────────────────────────────────┘          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 과정】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  초기화:                                                                 │ │
│  │  큰 메모리 블록 할당 → 고정 크기로 분할 → Free List 구성                 │ │
│  │                                                             │ │
│  │  할당 (alloc):                                                            │ │
│  │  Free List의 첫 번째 블록 반환 → Free List에서 제거                      │ │
│  │                                                             │ │
│  │  해제 (free):                                                             │ │
│  │  블록을 Free List 맨 앞에 추가                                           │ │
│  │                                                             │ │
│  │  O(1) 시간 복잡도!                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 풀 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 속도: O(1) 할당/해제                                                 │ │
│  │  • 단편화 없음: 고정 크기 블록                                           │ │
│  │  • 예측 가능: 할당 시간 일정                                             │ │
│  │  • 캐시 친화적: 연속된 메모리                                            │ │
│  │  • 디버깅: 메모리 누수 추적 용이                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 내부 단편화: 블록 크기보다 작은 요청도 블록 전체 사용                   │ │
│  │  • 유연성 부족: 다양한 크기에 비효율적                                    │ │
│  │  • 메모리 낭비: 사용하지 않는 블록도 메모리 점유                          │ │
│  │  • 크기 제한: 풀 크기 초과 시 처리 복잡                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 풀 종류】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 고정 크기 풀 (Fixed-size Pool):                                     │ │
│  │     • 모든 블록이 같은 크기                                             │ │
│  │     • 가장 간단하고 빠름                                                │ │
│  │                                                             │ │
│  │  2. 가변 크기 풀 (Variable-size Pool):                                  │ │
│  │     • 여러 크기의 블록 관리                                              │ │
│  │     • 크기별로 별도 풀                                                  │ │
│  │                                                             │ │
│  │  3. 계층적 풀 (Hierarchical Pool):                                      │ │
│  │     • 2의 거듭제곱 크기 풀들                                             │ │
│  │     • 버디 시스템과 유사                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【간단한 메모리 풀 구현】                                              │
│  ──────────────────                                                  │
│  #include <stdlib.h>                                                 │
│  #include <stddef.h>                                                 │
│                                                                     │
│  typedef struct Block {                                              │
│      struct Block *next;                                             │
│  } Block;                                                            │
│                                                                     │
│  typedef struct {                                                    │
│      Block *free_list;                                               │
│      size_t block_size;                                              │
│      size_t capacity;                                                │
│      void *memory;                                                   │
│  } MemPool;                                                          │
│                                                                     │
│  MemPool* mem_pool_create(size_t block_size, size_t count) {         │
│      MemPool *pool = malloc(sizeof(MemPool));                        │
│      pool->block_size = block_size < sizeof(Block)                   │
│                          ? sizeof(Block) : block_size;               │
│      pool->capacity = count;                                         │
│      pool->memory = malloc(pool->block_size * count);                │
│                                                                     │
│      // Free List 구성                                                 │
│      pool->free_list = NULL;                                         │
│      char *ptr = pool->memory;                                       │
│      for (size_t i = 0; i < count; i++) {                            │
│          Block *block = (Block*)ptr;                                 │
│          block->next = pool->free_list;                              │
│          pool->free_list = block;                                    │
│          ptr += pool->block_size;                                    │
│      }                                                               │
│      return pool;                                                    │
│  }                                                                   │
│                                                                     │
│  void* mem_pool_alloc(MemPool *pool) {                               │
│      if (!pool->free_list) return NULL;  // 풀 비었음                  │
│      Block *block = pool->free_list;                                 │
│      pool->free_list = block->next;                                  │
│      return block;                                                   │
│  }                                                                   │
│                                                                     │
│  void mem_pool_free(MemPool *pool, void *ptr) {                      │
│      Block *block = (Block*)ptr;                                     │
│      block->next = pool->free_list;                                  │
│      pool->free_list = block;                                        │
│  }                                                                   │
│                                                                     │
│  void mem_pool_destroy(MemPool *pool) {                              │
│      free(pool->memory);                                             │
│      free(pool);                                                     │
│  }                                                                   │
│                                                                     │
│  【사용 예시】                                                          │
│  ──────────────────                                                  │
│  // 32바이트 블록 100개 풀 생성                                          │
│  MemPool *pool = mem_pool_create(32, 100);                           │
│                                                                     │
│  // 할당                                                                │
│  void *obj1 = mem_pool_alloc(pool);                                  │
│  void *obj2 = mem_pool_alloc(pool);                                  │
│                                                                     │
│  // 해제                                                                │
│  mem_pool_free(pool, obj1);                                          │
│  mem_pool_free(pool, obj2);                                          │
│                                                                     │
│  // 풀 삭제                                                            │
│  mem_pool_destroy(pool);                                             │
│                                                                     │
│  【Linux 커널 메모리 풀】                                              │
│  ──────────────────                                                  │
│  // mempool_create (커널 API)                                          │
│  // 실패하지 않아야 하는 할당을 보장                                       │
│                                                                     │
│  // /proc/slabinfo로 슬랩 풀 확인                                       │
│  $ cat /proc/slabinfo | head -20                                    │
│  slabinfo - version: 2.1                                            │
│  # name    <active_objs> <num_objs> <objsize> ...                   │
│  task_struct  1234  1500  4096  ...                                 │
│  inode_cache  5678  6000  512   ...                                 │
│                                                                     │
│  // /proc/meminfo로 슬랩 메모리 확인                                    │
│  $ cat /proc/meminfo | grep Slab                                    │
│  Slab:            123456 kB                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 미리 할당된 메모리 블록 관리
• 구조: 고정 크기 블록 + Free List
• 할당: O(1) - 리스트에서 꺼내기
• 해제: O(1) - 리스트에 추가
• 장점: 빠름, 단편화 없음, 예측 가능
• 단점: 내부 단편화, 유연성 부족
• 활용: 커널, 게임, 실시간 시스템
```
