# 395. 페이지 테이블 (Page Table)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 주소를 물리 주소로 매핑하는 자료구조
> 2. **가치**: 페이징 시스템의 핵심 데이터
> 3. **융합**: 페이징, TLB, 가상 메모리와 연관

---

## Ⅰ. 개요

### 개념 정의

페이지 테이블(Page Table)은 **가상 페이지 번호를 물리 프레임 번호로 매핑하는 테이블**이다. 각 프로세스마다 자신의 페이지 테이블을 가진다.

### 💡 비유: 도서관 색인
페이지 테이블은 **도서관 색인 카드**와 같다. 책의 번호(가상 주소)를 보고 실제 위치(물리 주소)를 찾는다.

### 페이지 테이블 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 테이블 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  가상 주소                                                           │ │
│  │  ┌─────────────────────┬─────────────────────┐                    │ │
│  │  │    페이지 번호 (p)    │    오프셋 (d)       │                    │ │
│  │  └─────────────────────┴─────────────────────┘                    │ │
│  │           │                                                         │ │
│  │           │ 페이지 테이블 조회                                         │ │
│  │           ▼                                                         │ │
│  │  ┌─────────────────────────────────────────────────────────┐     │ │
│  │  │ 페이지 테이블                                            │     │ │
│  │  │ ┌───────────────────────────────────────────────┐       │     │ │
│  │  │ │ 인덱스 │ 프레임 │ 플래그 │                       │       │     │ │
│  │  │ ├───────────────────────────────────────────────┤       │     │ │
│  │  │ │   0   │   5   │ V=1,D=0,R=1 │                     │       │     │ │
│  │  │ │   1   │   2   │ V=1,D=0,R=1 │                     │       │     │ │
│  │  │ │   2   │   -   │ V=0       │ (메모리 없음)      │       │     │ │
│  │  │ │   3   │   8   │ V=1,D=1,R=1 │                     │       │     │ │
│  │  │ │  ...  │  ...  │ ...       │                     │       │     │ │
│  │  │ └───────────────────────────────────────────────┘       │     │ │
│  │  └─────────────────────────────────────────────────────────┘     │ │
│  │           │                                                         │ │
│  │           │ 프레임 번호 획득                                          │ │
│  │           ▼                                                         │ │
│  │  물리 주소                                                           │ │
│  │  ┌─────────────────────┬─────────────────────┐                    │ │
│  │  │   프레임 번호 (f)     │    오프셋 (d)       │                    │ │
│  │  └─────────────────────┴─────────────────────┘                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 테이블 엔트리 (PTE)】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │ 비트        │ 설명                                          │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │ 유효 비트(V)  │ 페이지가 메모리에 있는지                           │  │ │
│  │  │ 수정 비트(D)  │ 페이지가 수정되었는지 (더티)                       │  │ │
│  │  │ 참조 비트(R)  │ 최근에 참조되었는지                               │  │ │
│  │  │ 읽기 비트     │ 읽기 권한                                           │  │ │
│  │  │ 쓰기 비트     │ 쓰기 권한                                           │  │ │
│  │  │ 실행 비트     │ 실행 권한                                           │  │ │
│  │  │ 사용자/커널  │ 사용자 모드 또는 커널 모드                           │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                페이지 테이블 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 테이블 크기】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예시 계산:                                                           │ │
│  │  • 가상 주소 공간: 32비트 = 4GB                                        │ │
│  │  • 페이지 크기: 4KB = 2^12                                            │ │
│  │  • 페이지 수: 4GB / 4KB = 1M 개                                       │ │
│  │  • PTE 크기: 4바이트                                                 │ │
│  │  • 페이지 테이블 크기: 1M × 4B = 4MB                                  │ │
│  │                                                             │ │
│  │  각 프로세스마다 4MB의 페이지 테이블 필요!                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【다중 레벨 페이지 테이블】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  2단계 페이징 (Two-Level Paging):                                       │ │
│  │  • 외부 페이지 테이블 (Page Directory)                                 │ │
│  │  • 내부 페이지 테이블 (Page Table)                                     │ │
│  │                                                             │ │
│  │  3단계 페이징:                                                          │ │
│  │  • Page Directory Pointer Table (PDPT)                                │ │
│  │  • Page Directory (PD)                                                │ │
│  │  • Page Table (PT)                                                    │ │
│  │                                                             │ │
│  │  4단계 페이징 (x86_64):                                                 │ │
│  │  • PML4 → PDPT → PD → PT                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【역매핑 테이블】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  물리 프레임 → 가상 페이지 매핑                                          │ │
│  │                                                             │ │
│  │  용도:                                                                 │ │
│  │  • 페이지 교체 시 소유자 식별                                           │ │
│  │  • 공유 페이지 추적                                                     │ │
│  │  • 메모리 압축                                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 테이블 정보 확인】                                                │
│  ──────────────────                                                  │
│  // Linux: /proc/[pid]/maps                                           │
│  $ cat /proc/self/maps                                                │
│  00400000-00401000 r--p 00000000 fd=00                               │
│  7f8e8e50000-7f8e8e6000 r-xp 00000000 fd=00                           │
│                                                                     │
│  // 상세 정보: /proc/[pid]/smaps                                       │
│  $ cat /proc/self/smaps | head -20                                    │
│  Size:               4096 kB                                          │
│  KernelPageSize:        4 kB                                          │
│  MMUPageSize:           4 kB                                          │
│  Rss:                 640 kB                                          │
│  Pss:                 640 kB                                          │
│  Shared_Clean:          0 kB                                          │
│  Shared_Dirty:          0 kB                                          │
│  Private_Clean:         0 kB                                          │
│  Private_Dirty:       640 kB                                          │
│  Referenced:          640 kB                                          │
│  Anonymous:           640 kB                                          │
│  Locked:                0 kB                                          │
│                                                                     │
│  【페이지 테이블 구조체 (리눅스 커널)】                                      │
│  ──────────────────                                                  │
│  struct page_table_entry {                                            │
│      unsigned long present:1;    /* 페이지가 메모리에 있는지 */           │
│      unsigned long rw:2;         /* 읽기/쓰기 권한 */                    │
│      unsigned long user:1;       /* 사용자/커널 */                       │
│      unsigned long accessed:1;  /* 참조됨 */                             │
│      unsigned long dirty:1;      /* 수정됨 */                             │
│      unsigned long frame:20;     /* 프레임 번호 */                       │
│  };                                                                   │
│                                                                     │
│  /* 4바이트 = 32비트 PTE */                                            │
│                                                                     │
│  【페이지 테이블 조회】                                                      │
│  ──────────────────                                                  │
│  unsigned long get_physical_addr(unsigned long virtual_addr) {       │
│      unsigned long page = virtual_addr >> PAGE_SHIFT;                │
│      unsigned long offset = virtual_addr & PAGE_MASK;                │
│      struct page_table_entry *pte = &current->mm->pgd[page];          │
│                                                                     │
│      if (!pte->present) {                                            │
│          // 페이지 폴트 발생                                              │ │
│          handle_page_fault(virtual_addr);                            │
│          return 0;                                                    │
│      }                                                              │
│                                                                     │
│      pte->accessed = 1;  /* 참조 비트 설정 */                           │
│      return (pte->frame << PAGE_SHIFT) | offset;                      │
│  }                                                                  │
│                                                                     │
│  【페이지 테이블 수정】                                                      │
│  ──────────────────                                                  │
│  void update_page_table(unsigned long virtual_addr,                  │
│                         unsigned long frame, int flags) {             │
│      unsigned long page = virtual_addr >> PAGE_SHIFT;                │
│      struct page_table_entry *pte = &current->mm->pgd[page];          │
│                                                                     │
│      pte->frame = frame;                                             │
│      pte->present = 1;                                               │
│      pte->rw = (flags & PROT_WRITE) ? 1 : 0;                        │
│      pte->dirty = 0;                                                 │
│      pte->accessed = 0;                                              │
│                                                                     │
│      // TLB 무효화                                                     │ │
│      flush_tlb_page(current->mm, virtual_addr);                      │
│  }                                                                  │
│                                                                     │
│  【페이지 테이블 통계】                                                      │
│  ──────────────────                                                  │
│  // 시스템 전체 메모리 정보                                                │ │
│  $ cat /proc/meminfo                                                  │
│  MemTotal:       16383892 kB                                        │
│  MemFree:          8234512 kB                                        │
│  MemAvailable:    12345678 kB                                        │
│  PageTables:        123456 kB  // 페이지 테이블이 사용하는 메모리           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 가상 주소 → 물리 주소 매핑 테이블
• 구조: 페이지 번호 → 프레임 번호 + 플래그
• 항목(PTE): 프레임, V, D, R, R/W/X 비트
• 크기: 32비트 시스템에서 프로세스당 ~4MB
• 다중 레벨: 2단계, 3단계, 4단계 페이징
• 역매핑: 물리 → 가상 주소 매핑
```
