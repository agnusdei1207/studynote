# 페이지 테이블 (Page Table)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이지 테이블(Page Table)은 프로세스의 논리적 페이지가 물리적 메모리의 어떤 프레임에 저장되어 있는지 관리하는 일종의 '주소 매핑 지도'이다.
> 2. **역할**: CPU가 생성한 논리 주소를 물리 주소로 실시간 변환하기 위해 인덱스(페이지 번호)를 기반으로 해당 프레임의 시작 주소를 제공한다.
> 3. **저장 위치**: 프로세스마다 고유하게 존재하며, 그 크기가 방대하여 보통 메인 메모리(RAM)에 저장되고 하드웨어 레지스터를 통해 관리된다.

+++

## Ⅰ. 페이지 테이블 (Page Table)의 개념 및 역할

### 1. 정의
- **페이지 테이블 (Page Table)**은 페이징 시스템에서 가상 주소를 물리 주소로 매핑하기 위해 사용하는 데이터 구조이다.

### 2. 주요 기능
- **주소 매핑**: 페이지 번호를 입력받아 해당 페이지가 위치한 프레임 번호를 반환한다.
- **상태 관리**: 페이지의 유효성(Valid/Invalid), 접근 권한(Read/Write/Execute), 수정 여부(Dirty Bit) 등을 관리한다.

### 3. 구조적 특징
- 페이지 테이블은 **페이지 테이블 엔트리(PTE, Page Table Entry)**들의 배열 형태로 구성된다. 논리적 페이지 번호가 이 배열의 인덱스가 된다.

- **📢 섹션 요약 비유**: "수만 개의 아파트(프로세스 공간) 호수(페이지)가 실제 도시의 어떤 지번(프레임)에 있는지 적어둔 거대한 '주소록'과 같습니다."

+++

## Ⅱ. 페이지 테이블의 내부 구조 및 ASCII 다이어그램

### 1. 페이지 테이블 엔트리 (PTE)의 구성
- **Frame Number**: 실제 데이터가 위치한 물리 메모리 프레임의 시작 번호.
- **Valid-Invalid Bit**: 해당 페이지가 현재 메모리에 있는지(V), 아니면 디스크에 있는지(I) 표시.
- **Protection Bits**: 해당 페이지에 대한 읽기/쓰기/실행 권한 설정.
- **Reference/Dirty Bits**: 페이지 교체 알고리즘에서 사용하기 위한 참조/수정 기록.

### 2. ASCII: 페이지 테이블 참조 메커니즘
```text
[ Logical Address ]           [ Page Table ]              [ Physical Memory ]
+---------+---------+        Index | Frame | Bits         +-----------------+
| Page #  | Offset  |        ------+-------+------        | Frame 0         |
+---------+---------+          0   |   5   | V-R-W  ---+  +-----------------+
     |         |               1   |   2   | V-R--  ---|--| Frame 1         |
     |         |               2   |  ---  | I----     |  +-----------------+
     |         |               3   |   8   | V-R-W     +--| Frame 2: Page 1 |
     |         |             ...   |  ...  |  ...         +-----------------+
     |         +----------------------------------------> | Offset d        |
     +----------------------> [ Access Index 1 ]          +-----------------+
```

### 3. 데이터 저장 방식
- 페이지 테이블 자체도 메모리를 차지하므로, 프로세스가 커질수록 테이블 크기도 커진다. 예를 들어 4GB 주소 공간에서 4KB 페이지를 쓰면 약 100만 개의 엔트리가 필요하다.

- **📢 섹션 요약 비유**: "책의 맨 앞이나 뒤에 있는 '찾아보기(인덱스)'와 같습니다. 단어(페이지 번호)를 찾으면 해당 내용이 있는 페이지 번호(프레임 번호)를 알려줍니다."

+++

## Ⅲ. 페이지 테이블의 관리 기술 및 이슈

### 1. 위치 및 접근 속도 문제
- 페이지 테이블은 너무 커서 CPU 레지스터에 담을 수 없으므로 **메인 메모리**에 저장한다.
- 이로 인해 메모리 접근 시 '페이지 테이블 참조' 1회 + '실제 데이터 접근' 1회로 총 2회의 메모리 접근이 발생하여 속도가 저하된다.

### 2. 하드웨어 레지스터: PTBR
- **PTBR (Page-Table Base Register)**: 현재 실행 중인 프로세스의 페이지 테이블 시작 주소를 가리키는 CPU 내의 특수 레지스터다.

### 3. 해결책: TLB 가속
- 메모리 2번 접근 문제를 해결하기 위해 고속 하드웨어 캐시인 TLB를 사용하여 자주 쓰이는 PTE를 CPU 가까이에 보관한다.

- **📢 섹션 요약 비유**: "전화번호부가 너무 두꺼워 가방(메모리)에 넣어두고 필요할 때마다 꺼내보는 상황입니다. 자주 거는 번호는 메모지(TLB)에 적어 손에 들고 있는 것과 같죠."

+++

## Ⅳ. 페이지 테이블의 현대적 변형 (Strategy)

### 1. 계층적 페이징 (Hierarchical Paging)
- 페이지 테이블 자체가 너무 커지는 것을 방지하기 위해, 페이지 테이블을 다시 페이지로 나누는 '페이지 테이블의 페이지 테이블' 방식을 사용한다. (다단계 페이징)

### 2. 역 페이지 테이블 (Inverted Page Table)
- 프로세스마다 테이블을 만드는 대신, 시스템 전체에 단 하나의 테이블만 두고 '물리 프레임당 하나의 엔트리'를 갖게 하여 메모리 낭비를 극단적으로 줄인다.

- **📢 섹션 요약 비유**: "주소록이 너무 두꺼워지자, 구별로 주소록을 나누어 관리하거나(다단계), 아예 집집마다 누가 사는지 적어둔 명부(역 페이지 테이블)로 바꾸는 전략입니다."

+++

## Ⅴ. 결론 및 요약

### 1. 핵심 요약
- 페이지 테이블은 페이징 기법의 핵심 데이터 구조로, 논리적 추상화 공간을 물리적 실체 공간으로 연결해주는 가교 역할을 한다.

### 2. 설계의 중요성
- 페이지 테이블의 효율적인 설계(크기 최적화 및 접근 속도 향상)는 운영체제의 전반적인 메모리 관리 성능과 시스템 처리량(Throughput)에 결정적인 영향을 미친다.

- **📢 섹션 요약 비유**: "무질서하게 흩어진 물리적 메모리 조각들을 하나의 완벽한 논리적 질서로 엮어내는 운영체제의 '마법 지도'입니다."

+++

### 📌 관련 개념 맵 (Knowledge Graph)
- **PTBR / PTLR**: 페이지 테이블의 위치와 길이를 관리하는 하드웨어 레지스터.
- **유효-무효 비트 (Valid-Invalid Bit)**: 페이지 테이블 엔트리의 핵심 상태 비트.
- **TLB (Translation Look-aside Buffer)**: 페이지 테이블 접근 속도를 획기적으로 개선하는 장치.
- **다단계 페이지 테이블 (Multi-level Page Table)**: 거대한 페이지 테이블을 효율적으로 관리하는 기법.
