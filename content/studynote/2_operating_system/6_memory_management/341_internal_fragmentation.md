# 내부 단편화 (Internal Fragmentation)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 내부 단편화(Internal Fragmentation)는 할당된 메모리 블록이 프로세스의 실제 요구량보다 커서 블록 내부에서 발생하는 사용 불가능한 빈 공간이다.
> 2. **발생 원인**: 주로 고정 분할(Fixed Partitioning) 방식이나 페이징(Paging) 기법에서 할당 단위(Page Size 등)와 데이터 크기 간의 불일치로 인해 발생한다.
> 3. **해결 전략**: 가변 분할(Variable Partitioning) 방식을 채택하거나 페이지 크기를 최적화하여 낭비되는 잔여 공간을 최소화해야 한다.

+++

## Ⅰ. 내부 단편화 (Internal Fragmentation)의 정의와 메커니즘

### 1. 정의
- **내부 단편화 (Internal Fragmentation)**란 메모리를 분할할 때, 할당된 구역이 실제 필요한 크기보다 커서 남은 공간이 생기지만, 이 공간이 해당 프로세스에 할당되어 있어 다른 프로세스가 사용할 수 없는 낭비 상태를 의미한다.

### 2. 발생 메커니즘
- 시스템이 메모리를 100KB 단위로만 쪼개서 관리한다고 가정할 때, 프로세스가 70KB를 요청하면 100KB 블록을 통째로 준다. 이때 남는 30KB는 '내부'에 고립된 채 누구도 쓰지 못하는 죽은 공간이 된다.

### 3. 💡 비유: '1인 식탁에 앉은 혼밥족'
- 4인용 테이블만 있는 식당에 혼자 온 손님이 앉으면, 남은 3석은 비어있음에도 다른 손님을 앉힐 수 없는 상태와 같다. 테이블(메모리 블록) 내부의 낭비다.

- **📢 섹션 요약 비유**: "큰 상자에 작은 선물을 담고 남은 빈 공간에 솜을 채워 넣었지만, 그 빈 공간에 다른 선물은 넣을 수 없는 상태입니다."

+++

## Ⅱ. 내부 단편화의 가시적 분석 및 ASCII 다이어그램

### 1. 고정 분할 방식에서의 단편화
- 고정 분할(Fixed Partitioning)은 메모리를 미리 정해진 크기로 나누어 놓기 때문에 프로세스 크기가 딱 맞지 않으면 무조건 내부 단편화가 발생한다.

### 2. ASCII 메모리 맵 (Internal Fragmentation)
```text
[ Physical Memory Layout ]
+----------------------------+
|  Operating System (OS)     |
+----------------------------+ <--- Fixed Partition Start
|  Process A (80KB)          |
|  [Used: 70KB]              |
|  [Internal Frag: 10KB] <---!  (Wasted space inside block)
+----------------------------+
|  Process B (80KB)          |
|  [Used: 45KB]              |
|  [Internal Frag: 35KB] <---!  (Significant waste)
+----------------------------+
|  Free Partition (80KB)     |
+----------------------------+
```

### 3. 페이징(Paging)에서의 마지막 페이지 단편화
- 프로세스의 전체 크기가 페이지 크기의 배수가 아닐 경우, 마지막 페이지의 뒷부분은 항상 남게 된다. 이를 **페이지 내부 단편화**라고 부른다.

- **📢 섹션 요약 비유**: "일정한 크기의 타일을 바닥에 깔다가, 벽 끝부분에서 타일이 남아서 잘라버리고 버려지는 조각들과 같습니다."

+++

## Ⅲ. 발생 원인 및 기술적 영향성 분석

### 1. 주된 발생 원인
- **고정 크기 할당**: 관리의 편의성을 위해 메모리를 정해진 크기(Slot, Page)로 나누기 때문이다.
- **최소 할당 단위**: 하드웨어 제약이나 관리 오버헤드를 줄이기 위해 설정한 최소 단위가 프로세스 요구량보다 클 때 발생한다.

### 2. 시스템에 미치는 영향
- **메모리 이용률(Utilization) 저하**: 물리적 메모리는 충분해 보이지만 실제로는 조각난 빈 공간들 때문에 프로세스를 더 수용하지 못한다.
- **성능 오버헤드**: 낭비되는 공간이 많아지면 스와핑(Swapping)이 더 자주 발생하여 I/O 부하가 커진다.

- **📢 섹션 요약 비유**: "방은 넓은데 가구 배치를 잘못해서 정작 누울 자리가 좁아진 안타까운 인테리어 상황입니다."

+++

## Ⅳ. 내부 단편화 해결 전략 (Strategy)

### 1. 가변 분할 (Variable Partitioning) 도입
- 프로세스가 요구하는 크기만큼만 메모리를 동적으로 할당하여 내부 단편화를 근본적으로 차단한다. (단, 이 경우 외부 단편화가 발생할 수 있다.)

### 2. 페이지 크기 (Page Size) 최적화
- 페이지 크기를 작게 설정하면 마지막 페이지에서 남는 공간이 줄어들어 내부 단편화가 감소한다. 하지만 페이지 테이블의 크기가 커지는 트레이드오프(Trade-off)가 존재한다.

### 3. 세그먼테이션 (Segmentation) 활용
- 논리적 단위(함수, 데이터, 스택 등)로 메모리를 나누어 할당함으로써 고정된 틀에 억지로 맞추는 부작용을 줄인다.

- **📢 섹션 요약 비유**: "기성복 양복(고정 분할) 대신 맞춤 정장(가변 분할)을 입어 남는 품이 없게 만드는 수선 전략입니다."

+++

## Ⅴ. 결론 및 요약

### 1. 핵심 요약
- 내부 단편화는 '할당된 블록 내부'의 낭비다. 관리의 단순함(고정 분할)을 선택한 대가로 지불하는 자원 손실 비용이라 할 수 있다.

### 2. 향후 전망
- 현대 운영체제는 페이징을 기본으로 하되, 하위 레벨에서는 슬랩 할당기(Slab Allocator) 등을 사용하여 커널 객체 단위의 내부 단편화까지 정교하게 관리하며 효율을 극대화하고 있다.

- **📢 섹션 요약 비유**: "완벽한 정리를 위해 작은 물건은 작은 서랍에, 큰 물건은 큰 서랍에 넣는 맞춤형 수납 시스템으로 진화하고 있습니다."

+++

### 📌 관련 개념 맵 (Knowledge Graph)
- **외부 단편화 (External Fragmentation)**: 할당되지 않은 빈 공간들이 조각나서 발생하는 낭비.
- **고정 분할 (Fixed Partitioning)**: 내부 단편화의 주범인 메모리 할당 방식.
- **페이징 (Paging)**: 페이지 단위 할당으로 인해 미세한 내부 단편화가 발생하는 기법.
- **슬랩 할당기 (Slab Allocator)**: 내부 단편화를 최소화하는 커널 메모리 할당 기술.

### 👶 어린이를 위한 3줄 비유 설명
1. 커다란 피자 상자에 작은 만두 하나만 넣어두면, 상자 안에 빈 공간이 생기지만 다른 음식을 넣을 순 없어요. 이게 바로 **내부 단편화**예요.
2. 상자가 너무 커서 공간을 아깝게 낭비하는 것과 비슷하답니다.
3. 내 물건 크기에 딱 맞는 작은 상자를 준비하면 이 공간 낭비를 줄일 수 있어요!
