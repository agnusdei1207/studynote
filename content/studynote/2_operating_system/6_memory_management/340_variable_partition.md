# 가변 분할 방식 (Variable Partition)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가변 분할 방식은 고정된 칸막이 없이, 프로세스의 크기에 딱 맞춰서 메모리 공간을 동적으로 할당하는 연속 메모리 할당 기법이다.
> 2. **가치**: 내부 단편화를 완전히 제거하여 메모리 활용의 유연성을 높였으나, 프로세스의 종료와 할당이 반복되면서 작은 빈 공간들이 흩어지는 '외부 단편화(External Fragmentation)'를 야기한다.
> 3. **융합**: 효율적인 공간 할당을 위해 최초 적합(First-fit), 최적 적합(Best-fit) 등의 탐색 알고리즘과 압축(Compaction) 기술이 함께 사용된다.

+++

## Ⅰ. 가변 분할 방식의 개념 및 배경

### 1. 정의
- **가변 분할 방식 (Variable Partitioning)**이란 프로세스가 메모리에 적재될 때, 해당 프로세스가 요구하는 크기만큼의 구역을 동적으로 생성하여 할당하는 방식이다.

### 2. 등장 배경: '내부 단편화의 극복'
- 고정 분할 방식은 메모리 낭비(내부 단편화)가 너무 심했다. "그냥 오는 사람 몸무게에 딱 맞춰서 의자를 만들어주자"는 유연한 철학이 가변 분할 방식의 출발점이다.

### 3. 💡 비유: '백지장 주차장'
- 가변 분할은 주차장에 미리 선을 긋지 않고, 차가 들어오면 그 차의 크기만큼 즉석에서 **'분필로 선을 그어주는'** 것과 같다. 공간을 아주 알뜰하게 쓸 수 있다.

- **📢 섹션 요약 비유**: "틀에 사람을 맞추지 않고, 사람에 맞춰 틀을 짜는 맞춤형 서비스입니다."

+++

## Ⅱ. 가변 분할의 작동 원리 (Deep Dive)

### 1. Hole (빈 공간) 관리
- 운영체제는 현재 사용 가능한 메모리 블록(Hole)들의 리스트를 유지한다.
- 프로세스가 들어오면 충분히 큰 Hole을 찾아 분할해준 뒤, 남은 부분은 다시 새로운 Hole로 관리한다.

### 2. ASCII 다이어그램: 외부 단편화 진행 과정
```text
 [ Initial ]      [ Running ]      [ After P2 Exit ]
 +---------+      +---------+      +---------+
 |   OS    |      |   OS    |      |   OS    |
 +---------+      +---------+      +---------+
 |         |      |  P1(8)  |      |  P1(8)  |
 |         |      +---------+      +---------+
 |  Empty  |      |  P2(15) |      | Hole(15)| <--- 외부 단편화 후보
 |  (100)  |      +---------+      +---------+
 |         |      |  P3(20) |      |  P3(20) |
 |         |      +---------+      +---------+
 |         |      | Hole(57)|      | Hole(57)|
 +---------+      +---------+      +---------+
```

### 3. 기술적 상세 해설
- **통합 (Coalescing)**: 인접한 두 Hole이 생기면 이를 하나의 큰 Hole로 합쳐서 외부 단편화를 줄인다.
- **할당 전략**: 빈 공간 리스트를 탐색하는 방식에 따라 시스템의 성능과 공간 효율이 결정된다.

- **📢 섹션 요약 비유**: "쪼개진 조각들을 하나로 뭉쳐 큰 덩어리로 만드는 '점토 놀이' 같은 관리법입니다."

+++

## Ⅲ. 주요 특징 및 분석 (Strategy)

### 1. 할당 전략 비교
| 전략 | 방식 | 특징 |
|:---|:---|:---|
| **First-fit** | 첫 번째 발견한 공간에 할당 | 가장 빠름, 앞부분에 작은 Hole 양산. |
| **Best-fit** | 크기가 가장 비슷한 공간에 할당 | 공간 낭비 최소화, 아주 작은 Hole 양산. |
| **Worst-fit** | 가장 큰 공간에 할당 | 남은 공간을 활용 가능하게 함, 비효율적. |

### 2. 분석 결과
- **외부 단편화(External Fragmentation)**: 메모리 전체의 빈 공간 합계는 100MB인데, 연속된 100MB 자리가 없어서 80MB 프로세스를 못 받는 현상이 발생한다.

- **📢 섹션 요약 비유**: "돈은 충분한데 잔돈뿐이라 비싼 물건(큰 프로세스)을 못 사는 '거스름돈의 비극'입니다."

+++

## Ⅳ. 실무 적용 및 장애 분석 (Strategy)

### 1. 실제 사례
- **IBM OS/360 (MVT - Multiprogramming with a Variable number of Tasks)**: 가변 분할 방식의 선구적 운영체제다.
- **현대 힙(Heap) 메모리 관리**: 응용 프로그램 내부에서 `malloc()` 등으로 메모리를 할당할 때 이와 유사한 원리를 사용한다.

### 2. 문제 해결 전략: 압축 (Compaction)
- 흩어진 Hole들을 한곳으로 모으기 위해 현재 실행 중인 모든 프로세스를 메모리의 한쪽 끝으로 밀어 넣는다.
- **단점**: 시스템 전체가 일시 중지(Stop-the-world)되며, 막대한 CPU 연산이 필요하다.

- **📢 섹션 요약 비유**: "떨어져 있는 친구들을 한데 모으려고 모든 학생을 앞줄로 당겨 앉히는 대이동입니다."

+++

## Ⅴ. 기대효과 및 결론

### 1. 기대효과
- **내부 단편화 제거**: 프로세스 크기만큼만 정확히 할당하므로 개별 분할 내의 낭비가 전혀 없다.
- **유연한 다중 프로그래밍**: 메모리 크기가 허용하는 한 프로세스의 개수에 제한이 없다.

### 2. 결론 및 향후 전망
- 가변 분할 방식의 외부 단편화 문제는 결국 **페이징(Paging)**이라는 혁신적 기술을 낳는 계기가 되었다. 물리적으로는 흩어져 있어도 논리적으로는 이어진 것처럼 보이게 하는 페이징의 등장은 연속 할당의 시대에 마침표를 찍었다.

- **📢 섹션 요약 비유**: "조각난 땅도 내 땅이다! 흩어진 공간을 묶어 쓰는 가상화 시대를 향한 디딤돌입니다."
