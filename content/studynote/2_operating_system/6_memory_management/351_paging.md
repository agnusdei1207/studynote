# 391. 페이징 (Paging)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 메모리를 고정 크기 블록으로 관리
> 2. **가치**: 메모리 관리의 핵심 기법
> 3. **융합**: 가상 메모리, 페이지 교체와 연관

---

## Ⅰ. 개요

### 개념 정의

페이징(Paging)은 **물리 메모리를 고정 크기의 블록(페이지)로 나누어 관리하는 메모리 관리 기법**이다. 가상 메모리와 물리 메모리 모두 페이지 단위로 관리된다.

### 💡 비유: 책
페이징은 **책**과 같다. 책을 페이지 단위로 나누어 관리한다. 페이지는 고정된 크기다.

### 페이징 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                페이징 구조                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 계층】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  가상 메모리 (Logical Memory)                                       │ │
│  │  ┌───┬───┬───┬───┬───┬───┬───┬───┐                                    │ │
│  │  │ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │...│ n │  페이지 번호                           │ │
│  │  └───┴───┴───┴───┴───┴───┴───┴───┘                                    │ │
│  │                                                             │ │
│  │  페이지 테이블                                                    │ │
│  │  ┌───────────────────────────────────────┐                           │ │
│  │  │ 페이지 │ 프레임 │ 유효비트 │              │                           │ │
│  │  │   0   │   5   │   1    │              │                           │ │
│  │  │   1   │   2   │   1    │              │                           │ │
│  │  │   2   │   -   │   0    │ (메모리 없음)   │                           │ │
│  │  │   3   │   8   │   1    │              │                           │ │
│  │  │  ...  │ ...  │  ...  │              │                           │ │
│  │  └───────────────────────────────────────┘                           │ │
│  │                                                             │ │
│  │  물리 메모리 (Physical Memory)                                       │ │
│  │  ┌───┬───┬───┬───┬───┬───┬───┬───┐                                    │ │
│  │  │ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │...│ m │  프레임 번호                          │ │
│  │  └───┴───┴───┴───┴───┴───┴───┴───┘                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 변환】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  논리 주소                            물리 주소                         │ │
│  │  ────────────                      ────────────                         │ │
│  │  ┌─────────────────────┐          ┌─────────────────────┐          │ │
│  │  │ 페이지 번호 │ 오프셋    │   ───►   │ 프레임 번호 │ 오프셋    │          │ │
│  │  └─────────────────────┘          └─────────────────────┘          │ │
│  │                                                             │ │
│  │  논리 주소 = 페이지 번호 × 페이지 크기 + 오프셋                        │ │
│  │  물리 주소 = 프레임 번호 × 프레임 크기 + 오프셋                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 크기】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  일반적 크기: 4KB                                                   │ │
│  │  큰 페이지: 2MB, 1GB (Huge Pages)                                 │ │
│  │  페이지 크기 = 프레임 크기                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                페이징 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 테이블 항목】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  각 페이지마다 다음 정보 저장:                                          │ │
│  │                                                             │ │
│  │  • 프레임 번호: 매핑된 물리 프레임                                     │ │
│  │  • 유효 비트(Valid Bit): 메모리에 있는지 여부                           │ │
│  │  • 수정 비트(Dirty Bit): 변경되었는지 여부                            │ │
│  │  • 참조 비트(Reference Bit): 최근 접근 여부                            │ │
│  │  • 보호 비트: 읽기/쓰기/실행 권한                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【내부 단편화】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  마지막 페이지의 낭비:                                               │ │
│  │                                                             │ │
│  │  프로세스 크기 = 10KB, 페이지 크기 = 4KB                             │ │
│  │  필요한 페이지 = 3개 (12KB 중 10KB 사용)                              │ │
│  │  내부 단편화 = 12KB - 10KB = 2KB                                     │ │
│  │                                                             │ │
│  │  평균 내부 단편화 = 페이지 크기 / 2                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【외부 단편화 없음】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  페이징의 장점:                                                       │ │
│  │                                                             │ │
│  │  • 외부 단편화 없음 (단편화 문제 해결)                                │ │
│  │  • 메모리 할당이 연속적일 필요 없음                                   │ │
│  │  • 메모리 활용 효율 증가                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 페이징】                                                         │
│  ──────────────────                                                  │
│  // 페이지 크기 확인                                                    │
│  $ getconf PAGESIZE                                                   │
│  4096                                                                │
│                                                                     │
│  // 페이지 테이블 정보                                                   │
│  $ cat /proc/<pid>/maps                                               │
│  $ cat /proc/<pid>/smaps                                              │
│                                                                     │
│  // 시스템 메모리 정보                                                   │
│  $ cat /proc/meminfo                                                  │
│  $ free -h                                                             │
│                                                                     │
│  【TLB (Translation Lookaside Buffer)】                                 │
│  ──────────────────                                                  │
│  // 페이지 테이블 캐싱                                                   │
│  // TLB 크기 확인                                                       │
│  $ cat /sys/kernel/mm/tlb/flush_all                                   │
│                                                                     │
│  // TLB 미스 확인                                                       │
│  $ perf stat -e dTLB-load-misses,dTLB-store-misses                   │
│                                                                     │
│  【Huge Pages】                                                          │
│  ──────────────────                                                  │
│  // Huge Pages 확인                                                     │
│  $ cat /proc/meminfo | grep Huge                                      │
│  HugePages_Total:       0                                             │
│  HugePages_Free:        0                                             │
│  Hugepagesize:       2048 kB                                          │
│                                                                     │
│  // Huge Pages 활성화                                                   │
│  $ echo 100 > /proc/sys/vm/nr_hugepages                               │
│                                                                     │
│  【페이지 폴트 처리】                                                      │
│  ──────────────────                                                  │
│  // 페이지 폴트 통계                                                      │
│  $ cat /proc/vmstat | grep fault                                      │
│  pgfault 123456                                                       │
│  pgmaj_fault 789                                                      │
│                                                                     │
│  // 실시간 모니터링                                                       │
│  $ vmstat 1                                                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
