# 페이징 (Paging)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이징(Paging)은 프로세스의 논리적 주소 공간을 동일한 크기의 '페이지(Page)'로 나누고, 물리적 메모리를 동일한 크기의 '프레임(Frame)'으로 나누어 매핑하는 메모리 관리 기법이다.
> 2. **가치**: 프로세스를 물리 메모리의 비연속적인 공간에 배치할 수 있게 함으로써 외부 단편화(External Fragmentation) 문제를 근본적으로 해결한다.
> 3. **구현 핵심**: 주소 변환을 위한 페이지 테이블(Page Table)과 이를 지원하는 하드웨어(MMU)가 필수적이며, 현대 OS 가상 메모리 관리의 표준이다.

+++

## Ⅰ. 페이징 (Paging)의 개념 및 정의

### 1. 정의
- **페이징 (Paging)**이란 프로세스의 가상 주소 공간을 고정된 크기의 블록인 **페이지(Page)**로 나누고, 물리 메모리를 같은 크기의 블록인 **프레임(Frame)**으로 나누어 관리하는 기법이다.

### 2. 주요 구성 요소
- **페이지 (Page)**: 논리적 메모리(Logical Memory)를 나누는 단위.
- **프레임 (Frame)**: 물리적 메모리(Physical Memory)를 나누는 단위.
- **페이지 테이블 (Page Table)**: 논리적 페이지가 물리적 어느 프레임에 위치하는지 기록한 매핑 정보.

### 3. 기술적 의의
- 물리적 메모리의 연속적인 빈 공간을 찾을 필요가 없으므로 메모리 할당이 매우 유연해지며, 프로세스가 조각나서 저장되어도 CPU는 이를 연속된 공간으로 인식한다.

- **📢 섹션 요약 비유**: "두꺼운 책(프로세스)을 한 페이지씩 뜯어서 도서관의 빈 선반(프레임) 어디든 꽂아두고, 목차(페이지 테이블)만 보고 찾아내는 방식입니다."

+++

## Ⅱ. 페이징의 메커니즘 및 ASCII 다이어그램

### 1. 주소 변환 구조 (Address Translation)
- 논리적 주소는 **페이지 번호(p)**와 **페이지 오프셋(d)**으로 구성된다.
  - **p (Page Number)**: 페이지 테이블의 인덱스로 사용되어 물리 메모리의 베이스 주소(프레임 번호)를 찾는다.
  - **d (Page Offset)**: 해당 프레임 내에서의 상대적 위치(변위)를 나타낸다.

### 2. ASCII 메모리 매핑 구조
```text
[ CPU (Logical) ]        [ Page Table ]          [ Physical Memory ]
+---------------+        +-----+-----+           +-----------------+
|   p   |   d   | ---->  |  p  |  f  |           | Frame 0         |
+-------+-------+        +-----+-----+           +-----------------+
    |       |            |  0  |  3  | --------> | Frame 1         |
    |       +------------|  1  |  1  | ----+     +-----------------+
    |                    |  2  |  4  | --+ |     | Frame 2         |
    +------------------->|  3  |  0  | --|-|---> +-----------------+
                         +-----+-----+   | |     | Frame 3: Page 0 |
                                         | +---> +-----------------+
                                         |       | Frame 4: Page 2 |
                                         +-----> +-----------------+
```

### 3. 하드웨어의 역할
- **MMU (Memory Management Unit)**: CPU가 내보내는 논리 주소를 가로채 페이지 테이블을 참조하여 실시간으로 물리 주소로 변환한다.

- **📢 섹션 요약 비유**: "지도(페이지 테이블)를 들고 목적지(프레임)를 찾아가는 과정이며, 이 지도는 주소(페이지 번호)를 넣으면 실제 위치(프레임 번호)가 튀어나오는 마법의 지도입니다."

+++

## Ⅲ. 페이징의 특징 및 장단점

### 1. 장점 (Pros)
- **외부 단편화 완전 해결**: 비어있는 프레임이 프로세스의 페이지 수만큼만 있으면 어디든 할당 가능하다.
- **메모리 보호 및 공유**: 각 페이지별로 접근 권한을 설정할 수 있으며, 여러 프로세스가 동일한 프레임을 공유하기 쉽다.

### 2. 단점 (Cons)
- **내부 단편화 (Internal Fragmentation)**: 프로세스의 크기가 페이지 크기의 배수가 아닐 경우, 마지막 페이지에서 낭비되는 공간이 발생한다.
- **메모리 접근 오버헤드**: 페이지 테이블이 메모리에 있으므로, 실제 데이터에 접근하기 위해 메모리를 두 번(테이블 참조 + 실제 데이터 접근) 방문해야 한다.

- **📢 섹션 요약 비유**: "이사 갈 때 가구들을 박스에 나눠 담으면 틈새 공간까지 다 쓸 수 있지만, 박스 하나를 다 못 채우면 그만큼의 빈 공간(내부 단편화)이 생기는 것과 같습니다."

+++

## Ⅳ. 페이징 시스템의 성능 최적화 (Strategy)

### 1. 하드웨어 지원: TLB (Translation Look-aside Buffer)
- 메모리 접근 속도 저하를 막기 위해 최근에 사용된 페이지 매핑 정보를 고속 캐시 메모리인 TLB에 저장하여 변환 속도를 획기적으로 높인다.

### 2. 페이지 크기 결정
- 페이지가 작으면 내부 단편화는 줄어들지만 페이지 테이블이 커진다. 반대로 페이지가 크면 테이블은 작아지지만 내부 단편화가 심해질 수 있다. (현대 OS는 보통 4KB를 기본으로 사용)

- **📢 섹션 요약 비유**: "자주 가는 길은 지도(메모리)를 보지 않고 머릿속(TLB)에서 바로 떠올려 찾아가는 일종의 지름길 전략입니다."

+++

## Ⅴ. 결론 및 요약

### 1. 핵심 요약
- 페이징은 가상 메모리 시스템의 근간이 되는 기술로, 물리적 메모리의 불연속성을 논리적 연속성으로 치환하여 시스템 자원 활용도를 극대화한다.

### 2. 현대적 응용
- 현대의 64비트 시스템에서는 다단계 페이징(Multi-level Paging)과 대용량 페이지(Huge Pages) 기술을 통해 방대한 메모리 공간을 효율적으로 관리하고 있다.

- **📢 섹션 요약 비유**: "물리적 공간의 파편화를 '논리적 질서'로 정복한 현대 운영체제의 가장 효율적인 자원 관리 철학입니다."

+++

### 📌 관련 개념 맵 (Knowledge Graph)
- **페이지 크기 (Page Size)**: 페이징 효율을 결정하는 핵심 파라미터.
- **페이지 테이블 (Page Table)**: 페이징 매핑 정보의 집합체.
- **TLB (Translation Look-aside Buffer)**: 페이징 속도를 높이는 하드웨어 가속기.
- **내부 단편화 (Internal Fragmentation)**: 페이징의 유일한 공간적 단점.

2. 나중에 피자를 먹고 싶을 때 어느 칸에 어떤 조각을 넣었는지 적어둔 메모(페이지 테이블)를 보고 꺼내면 돼요.
3. 이렇게 하면 냉장고에 큰 빈 공간이 없어도 피자 한 판을 전부 보관할 수 있답니다!
