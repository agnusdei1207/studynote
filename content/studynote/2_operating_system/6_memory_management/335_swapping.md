# 스와핑 (Swapping)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스와핑(Swapping)은 실행 중인 프로세스를 일시적으로 메모리에서 보조 기억장치(Backing Store)로 내쫓고(Swap-out), 나중에 다시 메모리로 불러들이는(Swap-in) 기법이다.
> 2. **가치**: 실제 물리 메모리 크기보다 더 많은 수의 프로세스를 동시에 수용할 수 있게 하여, 다중 프로그래밍의 정도(Degree of Multiprogramming)를 극대화한다.
> 3. **융합**: 프로세스 전체를 교체하는 '표준 스와핑'에서 페이지 단위로 교체하는 '페이징 기반 스와핑'으로 진화하여 현대 가상 메모리 시스템의 근간이 되었다.

+++

## Ⅰ. 스와핑 (Swapping)의 개념 및 배경

### 1. 정의
- **스와핑 (Swapping)**이란 CPU 할당 시간이 끝난 프로세스의 메모리 이미지를 보조 기억장치로 이동시키고, 다른 프로세스를 메모리에 올리는 메모리 관리 기법이다.

### 2. 등장 배경: '메모리는 항상 부족하다'
- 다중 프로그래밍 환경에서 모든 실행 대기 프로세스를 메모리에 다 올릴 수 없는 상황이 발생한다. 이때 현재 실행되지 않는 프로세스를 잠시 디스크로 빼두어 빈 공간을 확보하고, CPU가 필요한 프로세스를 그 자리에 앉히는 유동적인 공간 관리 전략이 필요해졌다.

### 3. 💡 비유: '도서관 열람실의 사물함'
- 열람실 좌석(메모리)은 한정되어 있는데 공부하려는 학생(Process)은 많다. 자리가 없으면 공부를 다 한 학생이나 잠시 자리를 비운 학생의 책을 **'사물함(Backing Store)'**에 잠시 넣어두고(Swap-out), 새로운 학생을 앉힌다. 나중에 그 학생이 다시 오면 사물함에서 책을 꺼내 자리에 놓아준다(Swap-in).

- **📢 섹션 요약 비유**: "자리가 없으면 잠시 밖으로 나갔다가 순서가 되면 다시 들어오는 순환 시스템입니다."

+++

## Ⅱ. 스와핑의 작동 원리 (Deep Dive)

### 1. Swap-out & Swap-in 메커니즘
- **Swap-out**: 중기 스케줄러(Medium-term Scheduler)에 의해 선택된 프로세스를 메모리에서 제거하여 디스크의 스왑 영역(Swap Area)에 저장한다.
- **Swap-in**: CPU 호출이 필요한 프로세스를 디스크에서 읽어 메모리의 빈 공간에 적재한다.

### 2. ASCII 다이어그램: 스와핑 프로세스
```text
      [ Main Memory ]               [ Backing Store ]
    +-----------------+             +-----------------+
    | Operating System|             |                 |
    +-----------------+             |   Process P1    |
    |   Process P1    | ──Swap-out─▶|   (Swapped)     |
    +-----------------+             |                 |
    |                 |             |   Process P2    |
    |   Process P2    | ◀──Swap-in──|   (Waiting)     |
    +-----------------+             |                 |
    +-----------------+             +-----------------+
```

### 3. 기술적 상세 해설
- **Backing Store (Swap Area)**: 스와핑을 위해 할당된 고속 디스크 영역. 모든 사용자의 메모리 이미지를 수용할 수 있을 만큼 충분히 커야 한다.
- **문맥 교환 오버헤드**: 스와핑은 디스크 I/O를 수반하므로 매우 느리다. 따라서 프로세스 전체를 옮기는 표준 스와핑보다는 필요한 부분만 옮기는 방식이 현대에서 선호된다.

- **📢 섹션 요약 비유**: "짐을 싸서 나갔다가 다시 짐을 풀고 들어오는 과정에서 발생하는 '이사 비용(시간)'이 핵심입니다."

+++

## Ⅲ. 주요 특징 및 분석

### 1. 기능적 분석
| 구분 | 내용 |
|:---|:---|
| **목적** | 메모리 용량의 논리적 확장, 다중 프로그래밍 효율 향상. |
| **제약** | 실행 중인 I/O가 있는 프로세스는 함부로 Swap-out 할 수 없음 (데이터 유실 위험). |

### 2. 기술적 특이점: 재배치 (Relocation)
- Swap-out 된 프로세스가 다시 Swap-in 될 때, 반드시 원래 있던 메모리 주소로 돌아가야 하는가?
  - **Compile-time/Load-time binding**: 원래 자리로 가야 함 (제약 큼).
  - **Execution-time binding**: 빈 곳 아무 데나 갈 수 있음 (현대 방식, 유연함).

- **📢 섹션 요약 비유**: "다시 돌아올 때 꼭 그 자리에 앉아야 하는 식당과, 빈자리 어디든 앉아도 되는 식당의 차이입니다."

+++

## Ⅳ. 실무 적용 및 장애 분석 (Strategy)

### 1. 실제 사례
- **Linux Swap Partition**: 리눅스 설치 시 설정하는 스왑 파티션이 스와핑의 물리적 공간이다.
- **Mobile OS**: iOS나 초기 안드로이드는 성능 저하를 막기 위해 표준 스와핑 대신 프로세스를 아예 종료시키거나 압축하는 방식을 사용하기도 한다.

### 2. 문제 해결 전략
- **스왑 스래싱 (Swap Thrashing)**: 메모리 부족으로 스와핑이 너무 빈번하게 일어나 시스템 성능이 급격히 떨어지는 현상. 이를 막기 위해 물리 메모리를 증설하거나 프로세스 실행 개수를 조절해야 한다.

- **📢 섹션 요약 비유**: "짐 싸고 푸느라 공부는 하나도 못 하는 배보다 배꼽이 더 큰 상황을 경계해야 합니다."

+++

## Ⅴ. 기대효과 및 결론

### 1. 기대효과
- **시스템 처리량(Throughput) 증대**: 물리적 한계를 넘어 더 많은 작업을 시스템에 투입할 수 있다.
- **유연한 자원 배분**: 우선순위가 높은 프로세스에게 즉각적으로 메모리를 할당할 수 있는 기반을 제공한다.

### 2. 결론 및 향후 전망
- 전통적인 프로세스 단위 스와핑은 오늘날 **페이징(Paging)** 기술과 결합되어 '페이지 스와핑' 형태로 발전했다. 이는 현대 운영체제가 수기가바이트의 프로그램을 단 몇 메가바이트의 실제 메모리만으로도 실행할 수 있게 만드는 마법의 열쇠다.

- **📢 섹션 요약 비유**: "비우면 비울수록 더 많은 것을 담을 수 있는 메모리의 미학입니다."

### 👶 어린이를 위한 3줄 비유 설명
1. 스와핑은 놀이방에 친구들이 너무 많을 때, 잠시 안 노는 친구를 **'복도'**로 나가 있게 하는 거예요.
2. 새로운 친구가 들어와서 놀다가 나가면, 복도에 있던 친구가 다시 들어와서 놀 수 있죠.
3. 복도가 있어서 좁은 놀이방에서도 많은 친구가 번갈아 가며 즐겁게 놀 수 있답니다!
