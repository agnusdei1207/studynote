# 399. 스와핑 (Swapping)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리와 디스크 간 프로세스 교환
> 2. **가치**: 메모리 부족 해결과 멀티프로그래밍 지원
> 3. **융합**: 가상 메모리, 페이지 교체와 연관

---

## Ⅰ. 개요

### 개념 정의

스와핑(Swapping)은 **메모리가 부족할 때 프로세스를 통째로 디스크로 내보내고, 필요할 때 다시 메모리로 가져오는 기법**이다.

### 💡 비유: 여행 가방
스와핑은 **여행 가방**과 같다. 호텔 방(메모리)이 작으면 가방의 일부를 차(디스크)에 둔다. 필요할 때 다시 가져온다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                스와핑 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스왑 공간】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스왑 공간(Swap Space) 할당:                                           │ │
│  │  • 디스크의 일정 영역을 스왑용으로 예약                                  │ │
│  │  • 보통 물리 메모리 크기와 비슷하게 설정                                 │ │
│  │  • 연속적인 할당이 유리 (I/O 성능)                                      │ │
│  │                                                             │ │
│  │  Linux: /swapfile 또는 스왑 파티션                                      │ │
│  │  Windows: pagefile.sys                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스와핑 비용】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스왑 아웃 비용:                                                        │ │
│  │  • 메모리 → 디스크 쓰기: 수 ms ~ 수 초                                   │ │
│  │  • 프로세스 상태 저장                                                   │ │
│  │  • 컨텍스트 스위치 오버헤드                                             │ │
│  │                                                             │ │
│  │  스왑 인 비용:                                                          │ │
│  │  • 디스크 → 메모리 읽기: 수 ms ~ 수 초                                   │ │
│  │  • 메모리 할당                                                         │ │
│  │  • 프로세스 상태 복원                                                   │ │
│  │                                                             │ │
│  │  총 비용은 매우 큼 → 자주 발생하면 성능 심각 저하                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【현대 시스템의 스와핑】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  프로세스 전체 스와핑은 드묾:                                            │ │
│  │  • 메모리가 충분히 큼                                                   │ │
│  │  • 페이지 단위 스와핑(페이지 교체)이 더 효율적                            │ │
│  │                                                             │ │
│  │  하지만 여전히 사용됨:                                                   │ │
│  │  • 메모리 극도 부족 시                                                  │ │
│  │  • 절전 모드 (Hibernate)                                               │ │
│  │  • 모바일 기기 (iOS, Android)                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 스와핑】                                                         │
│  ──────────────────                                                  │
│  // 스왑 공간 확인                                                        │
│  $ swapon --show                                                      │
│  $ cat /proc/swaps                                                     │
│  $ free -h                                                             │
│                                                                     │
│  // 스왑 공간 생성                                                        │
│  $ fallocate -l 8G /swapfile                                         │
│  $ chmod 600 /swapfile                                               │
│  $ mkswap /swapfile                                                   │
│  $ swapon /swapfile                                                   │
│                                                                     │
│  // 스왑 사용 조정                                                        │
│  $ cat /proc/sys/vm/swappiness    // 0-100 (기본 60)                  │
│  $ echo 10 > /proc/sys/vm/swappiness                                 │
│                                                                     │
│  【스와핑 통계】                                                            │
│  ──────────────────                                                  │
│  // vmstat으로 모니터링                                                 │
│  $ vmstat 1                                                          │
│  procs -----------memory---------- ---swap--  -----io----             │
│   r  b   swpd   free   buff  cache   si   so    bi    bo              │
│  // si: swap in (페이지/초)                                             │
│  // so: swap out (페이지/초)                                            │
│                                                                     │
│  // 스왑 사용량 확인                                                      │
│  $ cat /proc/meminfo | grep -i swap                                  │
│  SwapCached:       12345 kB                                         │
│  SwapTotal:       8388608 kB                                        │
│  SwapFree:        8273612 kB                                        │
│                                                                     │
│  【프로세스 스왑 확인】                                                     │
│  ──────────────────                                                  │
│  // 각 프로세스의 스왑 사용량                                               │
│  $ grep VmSwap /proc/<pid>/status                                    │
│  VmSwap:     1234 kB                                                 │
│                                                                     │
│  // 모든 프로세스 스왑 사용량 정렬                                           │
│  $ for file in /proc/*/status ; do                                   │
│      grep VmSwap $file 2>/dev/null                                   │
│  done | sort -k 2 -n -r                                             │
│                                                                     │
│  【스와핑 방지】                                                            │
│  ──────────────────                                                  │
│  // mlock: 메모리 잠금 (스왑 방지)                                         │
│  #include <sys/mman.h>                                               │
│  void* addr = malloc(size);                                          │
│  mlock(addr, size);  // 이 영역은 스왑되지 않음                           │
│  munlock(addr, size);                                                │
│                                                                     │
│  // mlockall: 전체 주소 공간 잠금                                          │
│  mlockall(MCL_CURRENT | MCL_FUTURE);                                 │
│  munlockall();                                                       │
│                                                                     │
│  // 실시간 애플리케이션에서 사용                                            │ │
│                                                                     │
│  【Windows 스와핑】                                                        │
│  ──────────────────                                                  │
│  // 페이지 파일 설정                                                       │
│  // 제어판 > 시스템 > 고급 시스템 설정 > 성능 > 고급                         │
│  // > 가상 메모리                                                        │
│                                                                     │
│  // PowerShell로 확인                                                   │
│  Get-CimInstance -ClassName Win32_PageFileUsage                      │
│                                                                     │
│  【macOS 스와핑】                                                          │
│  ──────────────────                                                  │
│  // 스왑 파일 위치: /private/var/vm/swapfile*                          │
│  $ sysctl vm.swapusage                                               │
│  vm.swapusage: total = 2048.00M  used = 512.00M  free = 1536.00M    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 프로세스를 메모리와 디스크 간 교환
• 동작: Swap Out (메모리→디스크), Swap In (디스크→메모리)
• 공간: 스왑 공간, 스왑 파티션
• 비용: 수 ms~수 초 (매우 큼)
• 목적: 메모리 부족 해결, 멀티프로그래밍 지원
• 현대: 페이지 단위 스와핑이 일반적
```
