# 123. 파이프 (Pipe)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단방향 데이터 흐름을 제공하는 IPC 메커니즘
> 2. **가치**: 부모-자식 간 간단한 통신, 쉘 파이프라인
> 3. **융합**: 지명 파이프, 소켓, 스트림과 연관

---

## Ⅰ. 개요

### 개념 정의

파이프(Pipe)는 **단방향(Half-duplex) 데이터 흐름을 제공하는 IPC 메커니즘**이다. 주로 부모-자식 프로세스 간 통신에 사용되며, 쉘 파이프라인의 기반이다.

### 💡 비유: 물 파이프
파이프는 **물 파이프**와 같다. 한쪽에서 물을 넣으면(쓰기) 다른 쪽에서 나온다(읽기). 반대 방향으로는 흐르지 않는다.

### 파이프 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【익명 파이프 (Anonymous Pipe)】                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  부모 프로세스                      자식 프로세스              │ │
│  │  ┌─────────────────┐              ┌─────────────────┐        │ │
│  │  │ fd[0] (읽기)    │              │ fd[0] (읽기)    │        │ │
│  │  │ fd[1] (쓰기)    │              │ fd[1] (쓰기)    │        │ │
│  │  └────────┬────────┘              └────────┬────────┘        │ │
│  │           │                                 │                  │ │
│  │           │         파이프 버퍼             │                  │ │
│  │           │    ┌───────────────────┐       │                  │ │
│  │           └───►│  데이터 버퍼       ├───────┘                  │ │
│  │                │  (커널 공간)       │                          │ │
│  │                └───────────────────┘                          │ │
│  │                          ▲                                     │ │
│  │                          │ 단방향                              │ │
│  │                          ▼                                     │ │
│  │                                                             │ │
│  │  사용 패턴:                                                     │ │
│  │  • 부모는 fd[1]로 쓰고, fd[0] 닫기                               │ │
│  │  • 자식은 fd[0]으로 읽고, fd[1] 닫기                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파이프 특징】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특징                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  방향              단방향 (Half-duplex)                         │ │
│  │  범위              부모-자식 관계 프로세스만                      │ │
│  │  생명주기          프로세스 종료 시 소멸                          │ │
│  │  버퍼 크기         보통 64KB (PIPE_BUF)                         │ │
│  │  동기화            읽기/쓰기 자동 동기화                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【읽기/쓰기 동작】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  상황                      동작                                │ │
│  │  ────                        ────                                │ │
│  │  파이프 비었는데 읽기      쓰기 끝까지 블록                       │ │
│  │  파이프 가득 찼는데 쓰기   읽기 끝까지 블록                       │ │
│  │  쓰기 끝 닫힌 상태에서 읽기 EOF 반환                             │ │
│  │  읽기 끝 닫힌 상태에서 쓰기 SIGPIPE                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파이프 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  #include <unistd.h>                                           │ │
│  │  int pipe(int fd[2]);                                          │ │
│  │                                                               │ │
│  │  fd[0]: 읽기 전용 파일 디스크립터                                │ │
│  │  fd[1]: 쓰기 전용 파일 디스크립터                                │ │
│  │  반환: 0 (성공), -1 (실패)                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파이프 크기】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 파이프 버퍼 크기 확인                                        │ │
│  │  // PIPE_BUF: 원자적 쓰기 보장 크기 (보통 4096)                   │ │
│  │  #define PIPE_BUF  4096                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 구현 단순               • 단방향만                            │ │
│  │  • 자동 동기화             • 부모-자식 관계만                    │ │
│  │  • 바이트 스트림           • 버퍼 크기 제한                      │ │
│  │  • 쉘 연동 가능                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 파이프 사용】                                                   │
│  ──────────────────                                                  │
│  int fd[2];                                                         │
│  pipe(fd);                                                          │
│                                                                     │
│  pid_t pid = fork();                                                │
│  if (pid == 0) {                                                    │
│      // 자식: 읽기만                                                  │
│      close(fd[1]);                                                  │
│      char buf[100];                                                 │
│      read(fd[0], buf, sizeof(buf));                                 │
│      printf("수신: %s\n", buf);                                       │
│      close(fd[0]);                                                  │
│  } else {                                                           │
│      // 부모: 쓰기만                                                  │
│      close(fd[0]);                                                  │
│      write(fd[1], "Hello", 5);                                      │
│      close(fd[1]);                                                  │
│      wait(NULL);                                                    │
│  }                                                                  │
│                                                                     │
│  【쉘 파이프 구현 예시】                                                │
│  ──────────────────                                                  │
│  $ ls | grep ".txt" | wc -l                                         │
│                                                                     │
│  // 내부적으로:                                                       │
│  // pipe() → fork() → ls                                           │
│  // pipe() → fork() → grep                                         │
│  // pipe() → fork() → wc                                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 단방향 데이터 흐름 IPC
• 특징: 부모-자식, 자동 동기화
• API: pipe(fd[2]), fd[0] 읽기, fd[1] 쓰기
• 버퍼: 보통 64KB, PIPE_BUF 원자적 쓰기
• 장점: 단순, 동기화 자동
• 단점: 단방향, 부모-자식만
```
