# 127. XDR (External Data Representation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이기종 시스템 간 데이터 교환을 위한 표준 형식
> 2. **가치**: 플랫폼 독립적 데이터 표현
> 3. **융합**: RPC, 마샬링, 네트워크 바이트 순서와 연관

---

## Ⅰ. 개요

### 개념 정의

XDR(External Data Representation)은 **서로 다른 아키텍처의 시스템 간 데이터 교환을 위한 표준 데이터 표현 형식**이다. Sun RPC에서 개발되어 이기종 시스템 간 통신의 기반이다.

### 💡 비유: 에스페란토
XDR은 **에스페란토(국제어)**와 같다. 각 나라(시스템)가 다른 언어(데이터 표현)를 써도, 공통 언어(XDR)로 소통한다.

### XDR 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                XDR 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【이기종 시스템 간 데이터 교환 문제】                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  x86 (Little Endian)            SPARC (Big Endian)            │ │
│  │  ┌─────────────┐               ┌─────────────┐              │ │
│  │  │ 0x12345678  │               │ 0x78563412  │              │ │
│  │  │ 메모리:     │               │ 메모리:     │              │ │
│  │  │ 78 56 34 12 │               │ 12 34 56 78 │              │ │
│  │  └─────────────┘               └─────────────┘              │ │
│  │                                                             │ │
│  │  → 바이트 순서, 정렬, 데이터 크기가 다름                         │ │
│  │  → XDR로 표준화 필요                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【XDR 변환 과정】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  송신측                       XDR                        수신측 │
│  │  ┌─────────────┐           ┌─────────────┐           ┌─────────────┐ │
│  │  │ 내부 표현    │           │ 표준 형식   │           │ 내부 표현    │ │
│  │  │ (Little     │ ────────► │ (Big        │ ────────► │ (Big        │ │
│  │  │  Endian)    │   인코딩   │  Endian,    │   디코딩   │  Endian)    │ │
│  │  │             │           │ 4바이트 정렬)│           │             │ │
│  │  └─────────────┘           └─────────────┘           └─────────────┘ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【XDR 데이터 타입】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  타입                      크기                                │ │
│  │  ────                        ────                                │ │
│  │  int                     4바이트                                 │ │
│  │  unsigned int            4바이트                                 │ │
│  │  hyper (64비트)          8바이트                                 │ │
│  │  float                   4바이트 (IEEE 754)                      │ │
│  │  double                  8바이트                                 │ │
│  │  string                  가변 (패딩 포함)                        │ │
│  │  opaque                  가변                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                XDR 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【XDR 특징】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • Big Endian (네트워크 바이트 순서) 사용                        │ │
│  │  • 4바이트 경계 정렬                                            │ │
│  │  • 부호 있는 정수는 2의 보수 표현                                │ │
│  │  • 부동소수점은 IEEE 754                                        │ │
│  │  • 문자열은 ASCII (null 종료 아님)                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【XDR vs 다른 직렬화 형식】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  형식                      특징                                │ │
│  │  ────                        ────                                │ │
│  │  XDR                      바이너리, Sun RPC                     │ │
│  │  JSON                     텍스트, 웹                            │ │
│  │  Protocol Buffers         바이너리, gRPC                        │ │
│  │  MessagePack              바이너리, 경량                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 이기종 호환             • 바이너리 (가독성 낮음)              │ │
│  │  • 표준화                  • 패딩 오버헤드                      │ │
│  │  • RPC 연동                • 현대적 대안 존재                   │ │
│  │  • 효율적 전송                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【XDR 함수 예시】                                                      │
│  ──────────────────                                                  │
│  #include <rpc/xdr.h>                                               │
│                                                                     │
│  // 정수 인코딩/디코딩                                                │
│  XDR xdr;                                                           │
│  int value = 12345;                                                 │
│  xdr_int(&xdr, &value);                                             │
│                                                                     │
│  // 문자열 인코딩/디코딩                                               │
│  char *str = "Hello";                                               │
│  xdr_string(&xdr, &str, MAX_LEN);                                   │
│                                                                     │
│  【NFS에서의 XDR 사용】                                                 │
│  ──────────────────                                                  │
│  // NFS 파일 속성 구조체                                              │
│  struct nfs_fattr {                                                  │
│      ftype  type;         // 파일 타입                                │
│      u_int  mode;         // 권한                                    │
│      u_int  nlink;        // 링크 수                                  │
│      u_int  uid;          // 사용자 ID                               │
│      u_int  gid;          // 그룹 ID                                 │
│      u_int  size;         // 파일 크기                                │
│  };                                                                  │
│  // 모든 필드가 XDR로 직렬화됨                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
