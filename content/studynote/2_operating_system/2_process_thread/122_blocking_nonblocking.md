# 122. 동기식/비동기식 통신 (Blocking/Non-blocking)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메시지 전송/수신 시 대기 여부에 따른 분류
> 2. **가치**: 응답성 vs 효율성 트레이드오프
> 3. **융합**: IPC, 메시지 전달, I/O 모델과 연관

---

## Ⅰ. 개요

### 개념 정의

동기식 통신(Blocking/Synchronous)은 **메시지 전송/수신이 완료될 때까지 대기**하는 방식이다. 비동기식 통신(Non-blocking/Asynchronous)은 **즉시 반환하고 완료는 나중에 확인**하는 방식이다.

### 💡 비유: 전화 vs 문자
동기식은 **전화**와 같다. 상대가 받을 때까지 기다린다. 비동기식은 **문자 메시지**와 같다. 보내고 나서 다른 일을 한다.

│  │  │      ▼      │             │             │               │ │
│  │  │   대기...   │ ◄────────── │ receive()   │               │ │
│  │  │      │      │   완료       │             │               │ │
│  │  │      ▼      │             │             │               │ │
│  │  │   반환      │             │             │               │ │
│  │  └─────────────┘             └─────────────┘               │ │
│  │                                                             │ │

│  │                                                             │ │

│  │  ┌─────────────┐             ┌─────────────┐               │ │
│  │  │             │             │             │               │ │
│  │  │ send() 호출 │ ──────────► │             │               │ │
│  │  │      │      │             │             │               │ │
│  │  │      ▼      │             │             │               │ │

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동기식 메시지 큐】                                                   │
│  ──────────────────                                                  │
│  // 블로킹 송신                                                       │
│  msgsnd(msqid, &msg, size, 0);  // 완료까지 대기                     │
│                                                                     │
│  // 블로킹 수신                                                        │
│  msgrcv(msqid, &msg, size, type, 0);  // 메시지 올 때까지 대기        │
│                                                                     │
│  【비동기식 IPC (IPC_NOWAIT)】                                          │
│  ──────────────────                                                  │
│  // 비블로킹 송신                                                      │
│  int result = msgsnd(msqid, &msg, size, IPC_NOWAIT);                │
│  if (result == -1 && errno == EAGAIN) {                            │
│      // 큐가 가득 참, 나중에 재시도                                    │
│  }                                                                  │
│                                                                     │
│  // 비블로킹 수신                                                      │
│  ssize_t len = msgrcv(msqid, &msg, size, type, IPC_NOWAIT);         │
│  if (len == -1 && errno == ENOMSG) {                               │
│      // 메시지 없음, 다른 작업 수행                                    │
│  }                                                                  │
│                                                                     │
│  【소켓 비동기 I/O】                                                    │
│  ──────────────────                                                  │
│  // 비블로킹 소켓 설정                                                 │
│  int flags = fcntl(sock, F_GETFL, 0);                              │
│  fcntl(sock, F_SETFL, flags | O_NONBLOCK);                         │
│                                                                     │
│  // 비블로킹 읽기                                                      │
│  int n = recv(sock, buffer, len, 0);                               │
│  if (n == -1 && errno == EWOULDBLOCK) {                            │
│      // 데이터 없음, 나중에 다시 시도                                   │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 동기식: 완료까지 대기 (블로킹)
• 비동기식: 즉시 반환 (비블로킹)
• 동기식 장점: 단순, 자동 동기화
• 비동기식 장점: 응답성, 효율성
• 완료 확인: 폴링, 콜백, 이벤트
• 선택 기준: 응답성 vs 단순성
```
