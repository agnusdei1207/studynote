# 프로세스 메모리 구조 - Text(Code), Data, BSS, Heap, Stack

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 메모리 구조는 운영체제가 실행 중인 프로세스에게 할당하는 **논리적 주소 공간**으로, 데이터의 성격(상수, 변수, 동적 데이터)과 생명 주기에 따라 5개의 주요 세그먼트로 나뉘어 관리된다.
> 2. **가치**: 컴파일 타임에 결정되는 정적 영역(Text, Data, BSS)과 런타임에 결정되는 동적 영역(Heap, Stack)을 분리하여 메모리 관리의 효율성을 높이고, 영역별 접근 권한 설정을 통해 보안성과 안정성을 확보한다.
> 3. **융합**: 프로세스 간 메모리 공유(Text 영역)와 함수 호출 메커니즘(Stack), 대규모 데이터 관리(Heap)를 가능케 하는 현대 운영체제 자원 관리의 핵심 설계도이다.

+++

## Ⅰ. 프로세스 메모리 구조의 개요

### 1. 정의
- 운영체제가 프로세스를 실행하기 위해 메모리에 구획을 나누어 놓은 구조이다. 각 영역은 특정 용도의 데이터만 담도록 약속되어 있다.

### 2. 영역별 배치 특징
- 낮은 주소(Low Address)에는 정적 영역이, 높은 주소(High Address) 쪽으로 갈수록 동적 영역이 배치되는 것이 일반적인 구조이다.

### 3. 💡 비유: '셰프의 작업실 (주방)'
- **Text**: 주방에 비치된 **'요리 매뉴얼'**. 바뀌지 않는다.
- **Data/BSS**: 주방 선반에 미리 준비된 **'기본 양념'**. 언제든 쓸 수 있게 대기 중이다.
- **Heap**: 요리 도중 필요해서 시장에서 **'그때그때 사 오는 특별 재료'**. 다 쓰고 나면 치워야 한다.
- **Stack**: 요리하면서 잠시 메모하는 **'포스트잇'**. 순서대로 붙였다가 요리가 끝나면 떼어 버린다.

+++

## Ⅱ. 영역별 상세 분석 (Deep Dive)

### 1. Text (Code) 영역
- **내용**: 실행할 프로그램의 기계어 명령어가 저장되는 곳.
- **특징**: **Read-Only** 권한을 가지며, 동일 프로그램을 실행하는 여러 프로세스가 메모리에서 공유할 수 있다.

### 2. Data 영역 (GVAR / Initialized Data)
- **내용**: 초기화된 전역 변수와 정적 변수(static)가 저장된다.
- **특징**: 프로그램 시작 시 값이 할당되며 종료 시까지 유지된다. **Read-Write** 권한을 가진다.

### 3. BSS (Block Started by Symbol) 영역
- **내용**: 초기화되지 않은 전역 변수와 정적 변수가 저장된다.
- **특징**: 실제 파일 용량을 차지하지 않으며, 실행 시 0으로 자동 초기화된다.

### 4. Heap 영역
- **내용**: 런타임에 사용자가 직접 할당하는 메모리 공간(`malloc`, `new`).
- **특징**: 낮은 주소에서 높은 주소 방향으로 커진다 (LIFO가 아님). 사용 후 반드시 해제해야 하며, 그렇지 않으면 메모리 누수(Leak)가 발생한다.

### 5. Stack 영역
- **내용**: 함수 호출 시 생성되는 지역 변수, 매개변수, 리턴 주소 등이 저장된다.
- **특징**: 높은 주소에서 낮은 주소 방향으로 커진다. 함수가 종료되면 자동으로 반환(LIFO)된다.

### 6. ASCII 구조 다이어그램
```text
    [ High Address ]
    ┌──────────────────────────┐
    │      Stack               │ ◀── (지역 변수, 함수 호출) - 아래로 확장
    ├──────────────────────────┤
    │          ▼               │
    │      (Free Space)        │
    │          ▲               │
    ├──────────────────────────┤
    │      Heap                │ ◀── (동적 할당: malloc) - 위로 확장
    ├──────────────────────────┤
    │      BSS                 │ ◀── (초기화 안 된 전역 변수)
    ├──────────────────────────┤
    │      Data                │ ◀── (초기화 된 전역 변수)
    ├──────────────────────────┤
    │      Text (Code)         │ ◀── (명령어, 기계어)
    └──────────────────────────┘
    [ Low Address ]
```

- **📢 섹션 요약 비유**: 변하지 않는 뼈대(Text) 위에, 미리 준비한 도구(Data/BSS)를 챙기고, 상황에 따라 공간을 늘렸다 줄였다(Heap/Stack) 하며 작업을 수행하는 유연한 공간 설계입니다.

+++

## Ⅲ. Stack vs Heap 심층 비교

| 비교 항목 | Stack 영역 | Heap 영역 |
|:---|:---|:---|
| **관리 방식** | OS/컴파일러가 자동 관리 | **개발자가 직접 관리** |
| **할당 시점** | 컴파일 시 크기 결정 (정적) | **런타임 시 크기 결정 (동적)** |
| **속도** | 매우 빠름 (단순 포인터 이동) | 상대적 느림 (공간 탐색 오버헤드) |
| **한계** | 크기가 작고 제한적임 | 크기가 매우 큼 (가상 메모리 한계까지) |
| **에러** | Stack Overflow | Out of Memory (OOM) |

- **📢 섹션 요약 비유**: 서랍장에서 옷을 꺼내 입는 것처럼 순서가 명확하고 빠른 것(Stack)과, 넓은 창고에 짐을 쌓아두고 필요할 때마다 찾아 쓰는 것(Heap)의 차이입니다.

## Ⅳ. 실무 적용 및 최적화 인사이트 (Strategy)

### 1. 실무 시나리오: 재귀 함수와 스택 오버플로우
- **상황**: 복잡한 알고리즘을 재귀 함수로 구현했는데 특정 입력에서 프로그램이 강제 종료됨.
- **판단**: 함수의 깊이가 너무 깊어져 **Stack 영역**이 정해진 한계를 넘고 Heap 영역을 침범하거나 할당된 경계를 넘은 것이다. 해결을 위해 재귀 대신 반복문(Iteration)을 사용하거나, 큰 배열은 힙 영역(`malloc`)으로 옮겨 스택 부하를 줄여야 한다.

### 2. 기술사적 인사이트: 메모리 오염 공격과 방어
- 해커는 Stack의 리턴 주소를 변조하여 악성 코드를 실행하는 **Buffer Overflow** 공격을 시도한다. 현대 운영체제는 이를 막기 위해 **ASLR (Address Space Layout Randomization)** 기술로 메모리 주소를 매번 섞고, **NX (No-eXecute) 비트**로 데이터 영역(Stack, Heap)에서의 코드 실행을 금지하는 방어 체계를 갖추고 있다.

- **📢 섹션 요약 비유**: 구획이 정해진 건물이라도 테러리스트가 벽을 부술 수 있으므로, 매번 방 번호를 바꾸고(ASLR) 창고에는 불을 켤 수 없게(NX) 조치하는 보안 설계가 필요합니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **메모리 보호**: 영역별 권한 설정으로 프로세스 내부의 데이터 무결성 유지.
- **효율적 자원 공유**: Text 영역 공유를 통한 물리 메모리 절약.

### 2. 미래 전망
최근에는 멀티코어 환경에서 힙 할당 성능을 높이기 위해 **Thread Local Heap** 기술이 발전하고 있다. 또한, Rust와 같은 언어는 컴파일 타임에 Stack과 Heap의 생명 주기를 엄격히 관리하여 개발자의 실수(Memory Leak)를 원천 차단하는 방향으로 진화하고 있다. 운영체제의 메모리 구조는 이제 하드웨어의 물리적 한계를 넘어, 소프트웨어의 안전성을 담보하는 논리적 틀로서 그 역할이 더욱 공고해지고 있다.

- **📢 섹션 요약 비유**: 잘 정리된 서류 가방(프로세스 메모리)이 업무의 능률을 높이듯, 완벽한 메모리 구조가 시스템의 품격을 결정합니다.

### 📌 관련 개념 맵 (Knowledge Graph)
- **BSS 영역**: 초기화되지 않은 데이터의 특수 처리.
- **가상 메모리**: 메모리 구조가 실제로 매핑되는 기반 기술.
- **PCB**: 메모리 구조 정보를 포함하는 프로세스 신분증.
