# 139. 액터 모델 (Actor Model)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메시지 전달 기반의 동시성 모델
> 2. **가치**: 락 없는 동시성, 분산 시스템 적합
> 3. **융합**: 메시지 전달, 불변성, Erlang/Akka와 연관

---

## Ⅰ. 개요

### 개념 정의

액터 모델(Actor Model)은 **메시지 전달을 통해 통신하는 독립적인 액터로 구성된 동시성 모델**이다. 각 액터는 고유 상태를 가지며 락 없이 동시성을 처리한다.

### 💡 비유: 우편 배달원
액터 모델은 **우편 배달원들**과 같다. 각 배달원(액터)은 독립적으로 일하고, 편지(메시지)로만 소통한다.

### 액터 모델 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                액터 모델 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【액터 구성 요소】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  액터 A                        액터 B                        │ │
│  │  ┌───────────────────┐        ┌───────────────────┐         │ │
│  │  │                   │        │                   │         │ │
│  │  │  ┌─────────────┐  │        │  ┌─────────────┐  │         │ │
│  │  │  │ 상태        │  │        │  │ 상태        │  │         │ │
│  │  │  │ (State)     │  │        │  │ (State)     │  │         │ │
│  │  │  └─────────────┘  │        │  └─────────────┘  │         │ │
│  │  │                   │        │                   │         │ │
│  │  │  ┌─────────────┐  │        │  ┌─────────────┐  │         │ │
│  │  │  │ 메일박스     │◄─┼────────┼──│ 메일박스     │  │         │ │
│  │  │  │ (Mailbox)   │  │  메시지  │  │ (Mailbox)   │  │         │ │
│  │  │  └─────────────┘  │        │  └─────────────┘  │         │ │
│  │  │                   │        │                   │         │ │
│  │  │  ┌─────────────┐  │        │  ┌─────────────┐  │         │ │
│  │  │  │ 동작        │  │        │  │ 동작        │  │         │ │
│  │  │  │ (Behavior)  │  │        │  │ (Behavior)  │  │         │ │
│  │  │  └─────────────┘  │        │  └─────────────┘  │         │ │
│  │  │                   │        │                   │         │ │
│  │  └───────────────────┘        └───────────────────┘         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【액터의 동작】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  액터는 메시지를 받으면 다음 중 하나를 수행:                       │ │
│  │                                                               │ │
│  │  1. 새 액터 생성 (Create)                                       │ │
│  │  2. 다른 액터에게 메시지 전송 (Send)                              │ │
│  │  3. 다음 메시지 처리를 위한 동작 지정 (Become)                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【특징】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 공유 상태 없음 (No Shared State)                             │ │
│  │  • 비동기 메시지 전달                                            │ │
│  │  • 각 액터는 한 번에 하나의 메시지 처리                            │ │
│  │  • 위치 투명성 (분산 가능)                                       │ │
│  │  • 장애 격리 (Let It Crash)                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  │  구현체                    특징                                │ │

│  │  Erlang/OTP              통신 시스템, 고가용성                   │ │
│  │  Akka (Scala/Java)       JVM 기반, 분산 지원                    │ │
│  │  Orleans (.NET)          가상 액터, C#                          │ │
│  │  Elixir                  Erlang VM, 현대 문법                   │ │

│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Let It Crash 철학】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 액터는 격리되어 있어 충돌이 전파되지 않음                       │ │
│  │  • Supervisor가 액터를 감시하고 재시작                            │ │
│  │  • 방어적 프로그래밍 대신 복구 중심                               │ │
│  │  • 에러 로그 후 정상 상태로 복귀                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스레드 vs 액터】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              스레드 모델           액터 모델               │ │
│  │  ────                    ────                  ────              │ │
│  │  상태              공유                  격리                    │ │
│  │  통신              공유 메모리            메시지                  │ │
│  │  동기화            락 필요               없음                    │ │
│  │  장애              전파 가능             격리                    │ │
│  │  확장              어려움                쉬움                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 락 없는 동시성          • 학습 곡선                          │ │
│  │  • 분산 시스템 적합         • 메시지 순서 보장 어려움             │ │
│  │  • 장애 격리               • 디버깅 어려움                      │ │
│  │  • 높은 확장성                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Akka 액터 예시 (Scala)】                                             │
│  ──────────────────                                                  │
│  import akka.actor.{Actor, ActorSystem, Props}                      │
│                                                                     │
│  // 액터 정의                                                         │
│  class Counter extends Actor {                                      │
│    var count = 0                                                    │
│                                                                     │
│    def receive = {                                                  │
│      case "increment" =>                                            │
│        count += 1                                                   │
│        println(s"Count: $count")                                    │
│      case "get" =>                                                  │
│        sender() ! count  // 메시지 전송                              │
│    }                                                                │
│  }                                                                  │
│                                                                     │
│  // 액터 사용                                                         │
│  val system = ActorSystem("MySystem")                               │
│  val counter = system.actorOf(Props[Counter], "counter")            │
│                                                                     │
│  counter ! "increment"  // 비동기 메시지 전송                         │
│  counter ! "increment"                                               │
│  counter ! "get"                                                     │
│                                                                     │
│  【Erlang 액터 예시】                                                   │
│  ──────────────────                                                  │
│  % 액터 정의 (loop)                                                   │
│  counter(Count) ->                                                  │
│      receive                                                        │
│          {increment} ->                                             │
│              counter(Count + 1);                                    │
│          {get, Pid} ->                                              │
│              Pid ! {count, Count},                                  │
│              counter(Count)                                         │
│      end.                                                           │
│                                                                     │
│  % 액터 생성과 메시지 전송                                             │
│  Pid = spawn(fun() -> counter(0) end),                              │
│  Pid ! {increment},                                                 │
│  Pid ! {get, self()}.                                               │
│                                                                     │
│  【Supervisor 트리】                                                   │
│  ──────────────────                                                  │
│  // Akka Supervisor                                                  │
│  class Supervisor extends Actor {                                   │
│    override val supervisorStrategy =                                │
│      OneForOneStrategy() {                                          │
│        case _: Exception => Restart                                 │
│      }                                                              │
│    // 자식 액터 생성...                                               │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
