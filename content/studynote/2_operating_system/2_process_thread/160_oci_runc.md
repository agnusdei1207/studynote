# 160. OCI와 runc

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컨테이너 표준화 기구와 참조 구현
> 2. **가치**: 호환성, 이식성, 표준화
> 3. **융합**: 컨테이너, 런타임, 이미지 형식과 연관

---

## Ⅰ. 개요

### 개념 정의

OCI(Open Container Initiative)는 **컨테이너 이미지와 런타임의 표준을 정의하는 기구**다. runc는 OCI 런타임 사양의 참조 구현이다.

### 💡 비유: USB 표준
OCI는 **USB 표준**과 같다. 어떤 제조사(런타임)든 표준을 따르면 호환된다.

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【runc 직접 사용】                                                      │
│  ──────────────────                                                  │
│  # 번들 디렉토리 생성                                                  │
│  $ mkdir -p mycontainer/rootfs                                       │
│                                                                     │
│  # 루트 파일시스템 생성 (Docker 이미지에서 추출)                          │
│  $ cd mycontainer                                                    │
│  $ docker export $(docker create busybox) | tar -C rootfs -xvf -     │
│                                                                     │
│  # config.json 생성                                                   │
│  $ runc spec                                                         │
│                                                                     │
│  # 컨테이너 실행                                                       │
│  $ runc run mycontainer                                              │
│                                                                     │
│  【config.json 예시】                                                    │
│  ──────────────────                                                  │
│  {                                                                   │
│    "ociVersion": "1.0.2",                                            │
│    "process": {                                                      │
│      "terminal": true,                                               │
│      "user": { "uid": 0, "gid": 0 },                                 │
│      "args": ["sh"],                                                 │
│      "env": ["PATH=/bin"]                                            │
│    },                                                                │
│    "root": { "path": "rootfs", "readonly": true },                   │
│    "hostname": "runc",                                               │
│    "linux": {                                                        │
│      "namespaces": [                                                 │
│        { "type": "pid" },                                            │
│        { "type": "network" },                                        │
│        { "type": "ipc" },                                            │
│        { "type": "uts" },                                            │
│        { "type": "mount" }                                           │
│      ]                                                               │
│    }                                                                 │
│  }                                                                   │
│                                                                     │
│  【runc 명령어】                                                          │
│  ──────────────────                                                  │
│  $ runc create <container-id>      # 컨테이너 생성                     │
│  $ runc start <container-id>       # 컨테이너 시작                     │
│  $ runc run <container-id>         # 생성+시작                        │
│  $ runc list                       # 컨테이너 목록                     │
│  $ runc state <container-id>       # 상태 확인                        │
│  $ runc delete <container-id>      # 컨테이너 삭제                     │
│  $ runc spec                       # config.json 생성                 │
│                                                                     │
│  【crun 비교】                                                                             │
│  ──────────────────                                                  │
│  $ crun --version                                                    │
│  $ crun run mycontainer    # runc와 호환                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• OCI: 컨테이너 표준화 기구
• Runtime Spec: 런타임 동작 표준
• Image Spec: 이미지 형식 표준
• runc: OCI Runtime Spec 참조 구현
• 대안: crun, kata, gVisor
• 계층: 고수준(Docker) → 저수준(runc)
```
