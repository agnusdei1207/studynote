# 129. LPC (Local Procedure Call)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 같은 시스템 내 프로세스 간 원격 호출 방식
> 2. **가치**: RPC보다 빠른 로컬 통신
> 3. **융합**: RPC, ALPC, Windows IPC와 연관

---

## Ⅰ. 개요

### 개념 정의

LPC(Local Procedure Call)는 **같은 컴퓨터 내의 프로세스 간 함수 호출 메커니즘**이다. RPC의 로컬 버전으로, 네트워크 오버헤드 없이 더 빠른 통신을 제공한다.

### 💡 비유: 이웃집 전화
LPC는 **이웃집에 직접 가서 말하는 것**과 같다. 전화(RPC)로 이야기하는 것보다 더 빠르다.

## Ⅲ. 실무 적용

### 시스템 확인
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Windows LPC/ALPC 확인】                                              │
│  ──────────────────                                                  │
│  // Process Explorer로 LPC 포트 확인                                 │
│  // Handles 탭에서 "ALPC Port" 검색                                  │
│                                                                     │
│  // WinDbg로 LPC 확인                                                 │
│  !alpc                                                              │
│  !alpc /p [pid]                                                     │
│                                                                     │
│  【유닉스 대안: 유닉스 도메인 소켓】                                      │
│  ──────────────────                                                  │
│  // 로컬 통신용 소켓                                                  │
│  int sock = socket(AF_UNIX, SOCK_STREAM, 0);                       │
│                                                                     │
│  struct sockaddr_un addr;                                           │
│  addr.sun_family = AF_UNIX;                                         │
│  strcpy(addr.sun_path, "/tmp/socket");                             │
│                                                                     │
│  bind(sock, &addr, sizeof(addr));                                   │
│  listen(sock, 5);                                                   │
│                                                                     │
│  // 네트워크 스택 우회, 로컬만                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 로컬 시스템 내 프로시저 호출

• 용도: 서비스 간 통신
• 장점: 낮은 오버헤드
```
