# 126. RPC (Remote Procedure Call)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 원격 서버의 함수를 로컬처럼 호출하는 메커니즘
> 2. **가치**: 분산 시스템 투명성, 마이크로서비스 기반
> 3. **융합**: 마샬링, XDR, IDL, gRPC와 연관

---

## Ⅰ. 개요

### 개념 정의

RPC(Remote Procedure Call)는 **네트워크상의 다른 컴퓨터에 있는 함수를 로컬 함수처럼 호출하는 메커니즘**이다. 분산 시스템에서 프로세스 간 통신을 추상화한다.

### 💡 비유: 전화 주문
RPC는 **전화로 음식 주문**과 같다. 손님(클라이언트)이 전화로 주문(호출)하면, 식당(서버)이 요리해서(실행) 배달한다(반환).

### RPC 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                RPC 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RPC 동작 과정】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  클라이언트                      서버                         │ │
│  │  ┌─────────────┐                 ┌─────────────┐             │ │
│  │  │             │                 │             │             │ │
│  │  │ 1. 호출     │                 │ 5. 언마샬   │             │ │
│  │  │    ↓        │                 │    ↓        │             │ │
│  │  │ 2. 마샬링   │                 │ 6. 실제     │             │ │
│  │  │    ↓        │                 │    함수     │             │ │
│  │  │ 3. 전송     │ ──────────────► │    실행     │             │ │
│  │  │             │                 │    ↓        │             │ │
│  │  │             │ ◄───────────── │ 7. 마샬링   │             │ │
│  │  │ 8. 수신     │                 │    ↓        │             │ │
│  │  │    ↓        │                 │ 8. 전송     │             │ │
│  │  │ 9. 언마샬   │                 │             │             │ │
│  │  │    ↓        │                 │             │             │ │
│  │  │ 10. 반환    │                 │             │             │ │
│  │  └─────────────┘                 └─────────────┘             │ │
│  │                                                             │ │
│  │  Stub: 클라이언트/서버 측 프록시                                 │ │
│  │  Marshalling: 데이터를 네트워크 전송 형식으로 변환               │ │
│  │  Unmarshalling: 받은 데이터를 원래 형식으로 복원                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RPC 구성 요소】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구성 요소                  설명                                │ │
│  │  ────                        ────                                │ │
│  │  Client Stub            클라이언트 측 프록시                     │ │
│  │  Server Stub            서버 측 프록시                          │ │
│  │  IDL                    인터페이스 정의 언어                     │ │
│  │  Marshaller             데이터 직렬화                          │ │
│  │  Transport              네트워크 전송                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【gRPC 예시】                                                          │
│  ──────────────────                                                  │
│  // 1. Proto 파일 정의                                               │
│  // service.proto                                                    │
│  syntax = "proto3";                                                  │
│  service Greeting {                                                  │
│      rpc SayHello(HelloRequest) returns (HelloReply);               │
│  }                                                                   │
│  message HelloRequest {                                              │
│      string name = 1;                                                │
│  }                                                                   │
│  message HelloReply {                                                │
│      string message = 1;                                             │
│  }                                                                   │
│                                                                     │
│  // 2. 서버 구현                                                      │
│  class GreetingImpl : public Greeting::Service {                    │
│      Status SayHello(ServerContext* ctx,                            │
│                      const HelloRequest* req,                        │
│                      HelloReply* reply) override {                   │
│          reply->set_message("Hello " + req->name());                │
│          return Status::OK;                                          │
│      }                                                               │
│  };                                                                  │
│                                                                     │
│  // 3. 클라이언트 호출                                                 │
│  auto stub = Greeting::NewStub(channel);                            │
│  HelloRequest req;                                                   │
│  req.set_name("World");                                              │
│  HelloReply reply;                                                   │
│  stub->SayHello(&context, req, &reply);                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
