# 프로세스 상태 전이 모델 (New -> Ready -> Running -> Waiting -> Terminated)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 상태 전이 모델은 프로세스가 생존 기간 동안 하드웨어 인터럽트, 시스템 호출, 스케줄링 결정에 따라 **한 지위에서 다른 지위로 이동하는 논리적 경로**를 정의한 설계도이다.
> 2. **가치**: 상태 전이의 각 경로(Dispatch, Timeout, Block, Wakeup)를 명확히 함으로써, 운영체제는 CPU 유휴 시간을 최소화하고 여러 프로세스 간의 자원 경합을 질서 있게 조율할 수 있다.
> 3. **융합**: 하드웨어 타이머와 I/O 장치의 신호가 커널 스케줄러와 연동되는 지점이며, 다중 프로그래밍 환경에서 시스템 성능을 결정짓는 핵심 메커니즘이다.

+++

## Ⅰ. 프로세스 상태 전이의 주요 경로 (Deep Dive)

프로세스는 5가지 주요 상태 사이를 4가지 핵심 경로를 통해 이동한다.

### 1. Admitted: New ──▶ Ready
- **트리거**: 사용자가 프로그램을 실행하여 OS가 PCB를 생성하고 메모리 할당을 승인함.
- **의미**: 이제 시스템의 관리 대상이 되어 CPU 할당 경쟁에 참여할 자격을 얻음.

### 2. Dispatch: Ready ──▶ Running
- **트리거**: CPU 스케줄러가 준비 큐에 있는 프로세스 중 하나를 선택함.
- **의미**: 프로세스가 CPU 제어권을 얻어 실제 연산을 수행하기 시작함. (Context Restore 발생)

### 3. Timeout / Preemption: Running ──▶ Ready
- **트리거**: 할당된 시간(Time Slice)이 만료되거나, 더 높은 우선순위의 프로세스가 나타남.
- **의미**: 실행할 능력은 있으나 공평성을 위해 강제로 CPU를 반납함. (선점형 OS의 핵심)

### 4. Event Wait / Block: Running ──▶ Waiting
- **트리거**: 프로세스가 I/O 작업(디스크 읽기 등)을 요청하거나, 동기화 신호(세마포어 등)를 기다림.
- **의미**: 외부 이벤트가 완료될 때까지 CPU를 점유해봐야 소용없으므로 스스로 자리를 비켜줌.

### 5. Event Completion / Wakeup: Waiting ──▶ Ready
- **트리거**: 요청한 I/O 작업이 완료되어 하드웨어 인터럽트가 발생하거나, 기다리던 신호를 받음.
- **의미**: 다시 실행될 준비가 되었음을 알림. **주의: 즉시 Running으로 가지 않고 Ready Queue로 들어감.**

### 6. 💡 비유: '놀이공원 어트랙션 이용'
- **New ──▶ Ready**: 입장권을 사고 줄(Ready Queue)의 맨 끝에 서는 것.
- **Dispatch**: 내 차례가 되어 기구(CPU)에 올라타 신나게 즐기는 것.
- **Timeout**: 한 번 더 타고 싶지만 뒤에 사람이 많아 기구에서 내려 다시 줄 끝으로 가는 것.
- **Wait ──▶ Ready**: 중간에 신발 끈이 풀려 줄 밖으로 나갔다가(Waiting), 다 묶고 다시 줄을 서는 것.

## Ⅱ. 상태 전이 모델의 기술적 구조

### 1. 상태 전이의 핵심 관리 정보
- **상태 코드**: PCB 내부에 현재 상태(0: Ready, 1: Running 등)를 숫자로 기록한다.
- **큐 포인터**: 프로세스가 어느 큐(Ready Queue, Wait Queue)에 매달려 있는지 관리한다.

### 2. ASCII 상태 전이 상세 다이어그램
```text
    ┌───────────┐  [ Admitted ]  ┌───────────┐  [ Dispatch ]  ┌───────────┐
    │    New    │ ──────┬──────▶ │   Ready   │ ──────┬──────▶ │  Running  │
    └───────────┘       │        └─────┬─────┘       │        └─────┬─────┘
                        │              ▲             │              │
                        │              │ [ Timeout ] │              │ [ Exit ]
                        │              └─────────────┘              ▼
                        │                                     ┌───────────┐
                        │        ┌───────────┐ [ I/O Done ]   │ Terminated│
                        │        │  Waiting  │ ──────▶        └───────────┘
                        │        └─────▲─────┘
                        │              │ [ I/O Request ]
                        └──────────────┘
```

- **📢 섹션 요약 비유**: 복잡한 교차로(커널)에서 신호등(스케줄러)과 교통 표지판(상태 전이 규칙)에 따라 차들(프로세스)이 멈추고 가기를 반복하며 흐름을 만들어내는 과정입니다.

+++

## Ⅲ. 상태 전이 모델의 확장: 7-상태 모델

메모리 부족 시 발생하는 '보류(Suspended)' 상태를 포함한 모델이다.

| 전이 명칭 | 설명 | 필요성 |
|:---|:---|:---|
| **Suspend** | Ready/Waiting ──▶ Suspended | 메모리 공간을 확보하기 위해 프로세스를 디스크로 쫓아냄 (Swap-out) |
| **Activate** | Suspended ──▶ Ready/Waiting | 메모리 여유가 생겨 프로세스를 다시 메모리로 불러옴 (Swap-in) |

- **📢 섹션 요약 비유**: 은행 대기실이 너무 붐비면 "서류 다 쓰면 밖에서 기다리세요"라고 밖으로 내보내는(Suspended) 고육지책과 같습니다.

## Ⅳ. 실무 적용 및 스케줄링 인사이트 (Strategy)

### 1. 실무 시나리오: 높은 I/O 응답성 확보
- **상황**: 키보드 입력이나 마우스 클릭에 대한 반응이 느린 시스템.
- **판단**: Waiting 상태에서 Ready 상태로 Wakeup 되는 프로세스에게 **'우선순위 프리미엄'**을 주어야 한다. 이들은 CPU를 아주 짧게만 쓰고 다시 Waiting으로 갈 가능성이 높으므로, 즉시 Dispatch 해주면 사용자는 시스템이 매우 빠르다고 느끼게 된다.

### 2. 기술사적 인사이트: 상태 전이와 인터럽트 핸들링
- 상태 전이는 마법처럼 일어나는 것이 아니라, CPU의 **인터럽트 라인**에 신호가 들어오는 순간 커널 모드로 진입하여 일어나는 관리 행위이다. 기술사라면 인터럽트 발생 시 Context Switch가 수반됨을 이해하고, 불필요한 상태 전이를 줄여 '지터(Jitter)'를 억제하는 실시간 제어 전략을 논할 수 있어야 한다.

- **📢 섹션 요약 비유**: 신호등이 너무 자주 바뀌면(잦은 상태 전이) 오히려 교통 흐름이 방해되듯이, 적절한 시간 동안 차들을 보내주는(타임 슬라이스 조절) 정책이 중요합니다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **CPU 활용도 극대화**: I/O 대기 중인 프로세스를 제외하고 실행 가능한 프로세스에게만 CPU 집중.
- **시스템 예측 가능성**: 정해진 경로에 따라 프로세스를 관리함으로써 시스템의 안정적 운용 보장.

### 2. 미래 전망
최근에는 컨테이너 기반의 오케스트레이션에서 프로세스의 상태를 **Pause/Unpause** 하거나, 머신러닝 학습 중 메모리 체크포인트를 찍어 상태를 보존하는 기술이 활발히 연구되고 있다. 프로세스 상태 전이 모델은 이제 단일 노드를 넘어 분산 클라우드 인프라의 가용성을 조율하는 거시적 상태 관리로 확장되고 있다.

- **📢 섹션 요약 비유**: 태어나서 죽을 때까지의 인생 행로(상태 전이)를 가장 보람차고(효율적) 안전하게(안정적) 관리해 주는 운영체제의 지혜입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. 상태 전이 모델은 컴퓨터 친구들이 **'어디로 이동할지 정해진 지도'**와 같아요.
2. 줄을 서다가(Ready) 자기 차례가 되면 신나게 놀고(Running), 목이 마르면 물 마시러 가죠(Waiting).
3. 운영체제 선생님이 지도를 보고 친구들을 안내해 주니까, 복잡하지 않고 모두가 즐겁게 컴퓨터를 쓸 수 있답니다!
