# 147. 데몬 프로세스 (Daemon Process)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 백그라운드에서 실행되는 시스템 서비스 프로세스
> 2. **가치**: 지속적 서비스 제공, 사용자 개입 없음
> 3. **융합**: 세션, 제어 터미널, 서비스 관리와 연관

---

## Ⅰ. 개요

### 개념 정의

데몬 프로세스(Daemon Process)는 **백그라운드에서 실행되며 사용자와 직접 상호작용하지 않는 시스템 서비스 프로세스**다. 보통 시스템 시작 시 실행되어 종료까지 유지된다.

### 💡 비유: 보이지 않는 일꾼
데몬은 **보이지 않는 일꾼**과 같다. 항상 일하지만, 사용자는 그 존재를 직접 보지 못한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                데몬 프로세스 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데몬 생성 단계】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. fork() 후 부모 종료                                         │ │
│  │     → 셸이 완료로 인식, 백그라운드로 실행                         │ │
│  │                                                               │ │
│  │  2. setsid()로 새 세션 생성                                     │ │
│  │     → 프로세스 그룹 리더, 세션 리더                               │ │
│  │     → 제어 터미널 없음                                          │ │
│  │                                                               │ │
│  │  3. (선택) 다시 fork() 후 부모 종료                               │ │
│  │     → 세션 리더 아니게 함 (터미널 획득 방지)                       │ │
│  │                                                               │ │
│  │  4. chdir("/") 작업 디렉토리 변경                                │ │
│  │     → 파일 시스템 언마운트 방지                                   │ │
│  │                                                               │ │
│  │  5. umask(0) 파일 권한 마스크 초기화                              │ │
│  │                                                               │ │
│  │  6. 표준 입출력 닫기/리다이렉트                                   │ │
│  │     → /dev/null로 리다이렉트                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【systemd 데몬 관리】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 유닛 파일로 서비스 정의                                        │ │
│  │  • 의존성 관리                                                   │ │
│  │  • 자동 재시작                                                   │ │
│  │  • 로깅 통합 (journald)                                         │ │
│  │  • 소켓 활성화                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 지속적 서비스             • 디버깅 어려움                     │ │
│  │  • 사용자 개입 없음          • 보안 위험                         │ │
│  │  • 시스템 시작 시 실행        • 리소스 지속 사용                  │ │
│  │  • 독립적 실행                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전통적 데몬 생성】                                                    │
│  ──────────────────                                                  │
│  #include <unistd.h>                                                │
│  #include <stdlib.h>                                                │
│  #include <sys/stat.h>                                              │
│  #include <fcntl.h>                                                 │
│                                                                     │
│  int daemonize() {                                                  │
│      // 1. fork                                                     │
│      pid_t pid = fork();                                            │
│      if (pid < 0) return -1;                                        │
│      if (pid > 0) exit(0);  // 부모 종료                              │
│                                                                     │
│      // 2. 새 세션                                                   │
│      if (setsid() < 0) return -1;                                   │
│                                                                     │
│      // 3. 두 번째 fork (옵션)                                        │
│      pid = fork();                                                  │
│      if (pid < 0) return -1;                                        │
│      if (pid > 0) exit(0);                                          │
│                                                                     │
│      // 4. 작업 디렉토리                                              │
│      chdir("/");                                                    │
│                                                                     │
│      // 5. umask                                                    │
│      umask(0);                                                      │
│                                                                     │
│      // 6. 표준 입출력 닫기                                           │
│      for (int fd = 0; fd < 3; fd++)                                 │
│          close(fd);                                                 │
│                                                                     │
│      // /dev/null로 리다이렉트                                        │
│      open("/dev/null", O_RDWR);  // stdin                           │
│      dup(0);  // stdout                                             │
│      dup(0);  // stderr                                             │
│                                                                     │
│      return 0;                                                      │
│  }                                                                  │
│                                                                     │
│  【systemd 유닛 파일】                                                  │
│  ──────────────────                                                  │
│  # /etc/systemd/system/mydaemon.service                             │
│  [Unit]                                                              │
│  Description=My Daemon Service                                       │
│  After=network.target                                                │
│                                                                     │
│  [Service]                                                           │
│  Type=forking                                                        │
│  ExecStart=/usr/local/bin/mydaemon                                   │
│  ExecReload=/bin/kill -HUP $MAINPID                                  │
│  Restart=on-failure                                                  │
│  User=myuser                                                         │
│                                                                     │
│  [Install]                                                           │
│  WantedBy=multi-user.target                                          │
│                                                                     │
│  【데몬 관리 명령어】                                                    │
│  ──────────────────                                                  │
│  # 데몬 시작/중지/재시작                                               │
│  $ systemctl start mydaemon                                          │
│  $ systemctl stop mydaemon                                           │
│  $ systemctl restart mydaemon                                        │
│                                                                     │
│  # 상태 확인                                                          │
│  $ systemctl status mydaemon                                         │
│  $ journalctl -u mydaemon                                            │
│                                                                     │
│  # 부팅 시 자동 시작                                                   │
│  $ systemctl enable mydaemon                                         │
│  $ systemctl disable mydaemon                                        │
│                                                                     │
│  # 데몬 목록                                                          │
│  $ systemctl list-units --type=service                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 백그라운드 시스템 서비스

• 생성: fork, setsid, chdir, umask
• 관리: systemd (현대), init (전통)

• 이름: 서비스명 + d
```
