# 158. 네임스페이스 (Namespace)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 리소스를 격리하는 메커니즘
> 2. **가치**: 컨테이너 기술의 핵심, 가상화
> 3. **융합**: cgroup, 컨테이너, 가상화와 연관

---

## Ⅰ. 개요

### 개념 정의

네임스페이스(Namespace)는 **프로세스가 보는 시스템 리소스를 격리하는 리눅스 커널 기능**이다. 각 네임스페이스는 독립된 리소스 뷰를 제공한다.

### 💡 비유: 각자의 방
네임스페이스는 **각자의 방**과 같다. 각 방(네임스페이스)은 독립적이고 다른 방의 물건을 볼 수 없다.

### 네임스페이스 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                네임스페이스 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【네임스페이스 종류】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  네임스페이스             격리 대상                   Linux버전  │ │
│  │  ────                       ────                      ────      │ │
│  │  PID                  프로세스 ID                      2.6.24   │ │
│  │  Network (net)        네트워크 인터페이스               2.6.29   │ │
│  │  Mount (mnt)          마운트 포인트                     2.4.19   │ │
│  │  UTS                  호스트명, 도메인명                2.6.19   │ │
│  │  IPC                  System V IPC, POSIX 큐           2.6.19   │ │
│  │  User                 사용자/그룹 ID                    3.8      │ │
│  │  Cgroup               cgroup 루트                      4.6      │ │
│  │  Time                 부팅 시간, monotonic              5.6      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【격리 예시】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  호스트 시스템                   컨테이너                      │ │
│  │  ┌───────────────────┐        ┌───────────────────┐          │ │
│  │  │ PID 1: systemd    │        │ PID 1: init       │          │ │
│  │  │ PID 100: nginx    │        │ PID 2: nginx      │          │ │
│  │  │ PID 200: mysql    │        │ (PID 네임스페이스) │          │ │
│  │  └───────────────────┘        └───────────────────┘          │ │
│  │                                                             │ │
│  │  호스트: eth0                 컨테이너: eth0                   │ │
│  │  IP: 192.168.1.10            IP: 172.17.0.2                   │ │
│  │  (Network 네임스페이스)                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【clone 플래그】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  플래그                   네임스페이스                          │ │
│  │  ────                       ────                               │ │
│  │  CLONE_NEWPID            PID                                  │ │
│  │  CLONE_NEWNET            Network                              │ │
│  │  CLONE_NEWNS             Mount                                │ │
│  │  CLONE_NEWUTS            UTS                                  │ │
│  │  CLONE_NEWIPC            IPC                                  │ │
│  │  CLONE_NEWUSER           User                                 │ │
│  │  CLONE_NEWCGROUP         Cgroup                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【unshare 명령어】                                                      │
│  ──────────────────                                                  │
│  # 새 UTS 네임스페이스                                                 │
│  $ unshare -u /bin/bash                                              │
│  $ hostname mycontainer                                              │
│  $ hostname                                                          │
│  mycontainer  # 격리됨                                                │
│                                                                     │
│  # 새 PID 네임스페이스                                                 │
│  $ unshare -p -f --mount-proc /bin/bash                              │
│  $ ps aux                                                            │
│  # PID 1부터 시작                                                     │
│                                                                     │
│  # 새 네트워크 네임스페이스                                             │
│  $ unshare -n /bin/bash                                              │
│  $ ip link                                                           │
│  1: lo: <LOOPBACK> ...  # lo만 존재                                  │
│                                                                     │
│  【nsenter 명령어】                                                      │
│  ──────────────────                                                  │
│  # 프로세스 1234의 네임스페이스로 진입                                   │
│  $ nsenter -t 1234 -n -u -i /bin/bash                                │
│                                                                     │
│  # Docker 컨테이너 네임스페이스 진입                                    │
│  $ docker inspect --format '{{.State.Pid}}' mycontainer              │
│  5678                                                                │
│  $ nsenter -t 5678 -n /bin/bash                                      │
│                                                                     │
│  【ip netns】                                                            │
│  ──────────────────                                                  │
│  # 네트워크 네임스페이스 생성                                          │
│  $ ip netns add net1                                                 │
│  $ ip netns list                                                     │
│                                                                     │
│  # 네임스페이스에서 명령 실행                                          │
│  $ ip netns exec net1 ip link                                        │
│                                                                     │
│  【lsns 명령어】                                                          │
│  ──────────────────                                                  │
│  $ lsns                                                              │
│  NS TYPE  NPROCS   PID USER COMMAND                                  │
│  4026531835 cgroup    1     1 root /sbin/init                        │
│  4026531836 pid       1     1 root /sbin/init                        │
│  ...                                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
