# 155. 실시간 스케줄링 (Real-time Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데드라인 보장을 위한 스케줄링 정책
> 2. **가치**: 결정적 응답 시간, 우선순위 보장
> 3. **융합**: 스케줄링, 우선순위, POSIX와 연관

---

## Ⅰ. 개요

### 개념 정의

실시간 스케줄링(Real-time Scheduling)은 **엄격한 타이밍 요구사항을 만족하기 위한 스케줄링 정책**이다. 일반 스케줄링과 달리 데드라인 보장이 핵심이다.

### 💡 비유: 응급실 우선순위
실시간 스케줄링은 **응급실 환자 분류**와 같다. 위급한 환자(높은 우선순위)가 먼저 치료된다.

### 실시간 스케줄링 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                실시간 스케줄링 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【POSIX 스케줄링 정책】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  일반 스케줄링 (Non-Real-time)                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  SCHED_OTHER: 기본, CFS (Completely Fair Scheduler)     │ │ │
│  │  │  SCHED_BATCH: 배치 작업                                  │ │ │
│  │  │  SCHED_IDLE: 최저 우선순위                                │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  실시간 스케줄링 (Real-time)                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  SCHED_FIFO: 선입선출, 우선순위 유지                       │ │ │
│  │  │  SCHED_RR: 라운드 로빈, 타임 슬라이스                      │ │ │
│  │  │  (Linux) SCHED_DEADLINE: 데드라인 기반                    │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【우선순위 범위】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  높은 우선순위 ─────────────────────────────► 낮은 우선순위       │ │
│  │                                                             │ │
│  │  실시간 (1~99)               일반 (0)                          │ │
│  │  ┌─────────────────┐       ┌─────────────┐                   │ │
│  │  │ 99 (최고)        │       │             │                   │ │
│  │  │ ...             │       │  SCHED_OTHER│                   │ │
│  │  │ 1 (최저 실시간)   │       │  nice: -20~19                  │   │ │
│  │  └─────────────────┘       └─────────────┘                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SCHED_FIFO vs SCHED_RR】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              SCHED_FIFO          SCHED_RR                │ │
│  │  ────                    ────                  ────              │ │
│  │  선점              같은/높은 우선순위     타임 슬라이스 만료       │ │
│  │  타임 슬라이스       없음                있음                    │ │
│  │  같은 우선순위       실행 중인 것 계속      라운드 로빈           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                실시간 스케줄링 상세                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【sched_setscheduler API】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  #include <sched.h>                                             │ │
│  │                                                               │ │
│  │  struct sched_param {                                           │ │
│  │      int sched_priority;    // 우선순위                         │ │
│  │  };                                                             │ │
│  │                                                               │ │
│  │  int sched_setscheduler(pid_t pid, int policy,                  │ │
│  │                         const struct sched_param *param);       │ │
│  │                                                               │ │
│  │  int sched_getscheduler(pid_t pid);                             │ │
│  │  int sched_getparam(pid_t pid, struct sched_param *param);      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【우선순위 범위 확인】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  int min = sched_get_priority_min(SCHED_FIFO);  // 보통 1        │ │
│  │  int max = sched_get_priority_max(SCHED_FIFO);  // 보통 99       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CPU 선점 (Preemption)】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 실시간 프로세스는 일반 프로세스보다 항상 우선                    │ │
│  │  • 높은 우선순위 실시간 프로세스가 준비되면 즉시 선점               │ │
│  │  • 무한 루프 실시간 프로세스 주의 (시스템 멈춤)                     │ │
│  │  • CONFIG_PREEMPT 커널 옵션                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 결정적 응답              • CPU 독점 위험                      │ │
│  │  • 데드라인 보장             • 높은 우선순위 무한 루프 위험         │ │
│  │  • 우선순위 보장             • 권한 필요 (CAP_SYS_NICE)           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【실시간 스케줄링 설정】                                                │
│  ──────────────────                                                  │
│  #include <sched.h>                                                 │
│  #include <stdio.h>                                                 │
│  #include <unistd.h>                                                │
│  #include <sys/resource.h>                                          │
│                                                                     │
│  void set_realtime_priority(int policy, int priority) {             │
│      struct sched_param param;                                      │
│      param.sched_priority = priority;                               │
│                                                                     │
│      if (sched_setscheduler(0, policy, &param) < 0) {               │
│          perror("sched_setscheduler");                              │
│          return;                                                    │
│      }                                                              │
│      printf("실시간 스케줄링 설정: policy=%d, priority=%d\n",          │ │
│             policy, priority);                                      │
│  }                                                                  │
│                                                                     │
│  int main() {                                                        │
│      // SCHED_FIFO, 우선순위 50                                      │
│      set_realtime_priority(SCHED_FIFO, 50);                         │
│                                                                     │
│      // 실시간 작업                                                   │
│      while (1) {                                                    │
│          // ...                                                     │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【현재 스케줄링 정보 확인】                                             │
│  ──────────────────                                                  │
│  void print_sched_info() {                                          │
│      int policy = sched_getscheduler(0);                            │
│      struct sched_param param;                                      │
│      sched_getparam(0, &param);                                     │
│                                                                     │
│      const char *policy_name;                                       │
│      switch (policy) {                                              │
│          case SCHED_OTHER: policy_name = "OTHER"; break;            │
│          case SCHED_FIFO:  policy_name = "FIFO";  break;            │
│          case SCHED_RR:    policy_name = "RR";    break;            │
│          default: policy_name = "UNKNOWN";                          │
│      }                                                              │
│      printf("Policy: %s, Priority: %d\n",                           │
│             policy_name, param.sched_priority);                     │
│  }                                                                  │
│                                                                     │
│  【chrt 명령어】                                                         │
│  ──────────────────                                                  │
│  # 실시간 우선순위 확인                                                 │
│  $ chrt -p 1234                                                      │
│                                                                     │
│  # SCHED_FIFO로 실행                                                  │
│  $ sudo chrt -f 50 ./my_program                                      │
│                                                                     │
│  # SCHED_RR로 실행                                                    │
│  $ sudo chrt -r 30 ./my_program                                      │
│                                                                     │
│  # 현재 셸을 실시간으로                                                 │
│  $ sudo chrt -f 99 $$                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 데드라인 보장 스케줄링
• 정책: SCHED_FIFO, SCHED_RR
• 우선순위: 1~99 (높을수록 높음)
• API: sched_setscheduler
• 명령: chrt
• 권한: CAP_SYS_NICE 또는 root
```
