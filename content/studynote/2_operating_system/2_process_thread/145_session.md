# 145. 세션 (Session)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 그룹들의 모음
> 2. **가치**: 터미널 제어, 작업 제어 격리
> 3. **융합**: 프로세스 그룹, 제어 터미널, 데몬과 연관

---

## Ⅰ. 개요

### 개념 정의

세션(Session)은 **하나 이상의 프로세스 그룹으로 구성된 집합**이다. 주로 사용자 로그인 세션이나 제어 터미널과 연관된 프로세스들을 묶는다.

### 💡 비유: 로그인 세션
세션은 **사용자의 로그인 세션**과 같다. 로그인부터 로그아웃까지의 모든 작업이 포함된다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                세션 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【세션 관련 API】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  함수                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  getsid(pid)            세션 ID 반환                           │ │
│  │  setsid()               새 세션 생성 (호출자가 리더)              │ │
│  │                                                               │ │
│  │  setsid() 성공 조건:                                            │ │
│  │  • 호출 프로세스가 프로세스 그룹 리더가 아니어야 함                 │ │
│  │  • 새 세션, 새 프로세스 그룹 생성                                 │ │
│  │  • 제어 터미널 없음                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【제어 터미널 (Controlling Terminal)】                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 세션당 최대 하나의 제어 터미널                                  │ │
│  │  • 로그인 시 자동 연결                                            │ │
│  │  • Ctrl+C, Ctrl+Z는 제어 터미널에서 발생                          │ │
│  │  • /dev/tty로 접근                                               │ │
│  │  • 세션 리더 종료 시 SIGHUP 전송                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SIGHUP 동작】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 터미널 연결 종료 시 발생                                        │ │
│  │  • 세션 리더가 종료하면:                                          │ │
│  │    → 제어 터미널의 모든 프로세스에 SIGHUP                          │ │
│  │  • nohup으로 무시 가능                                            │ │
│  │  • 데몬 재시작 신호로도 사용                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 작업 격리              • 복잡한 계층                          │ │
│  │  • 터미널 제어             • 규칙 이해 필요                       │ │
│  │  • 데몬 생성              • SIGHUP 처리 필요                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데몬 생성 예시】                                                     │
│  ──────────────────                                                  │
│  #include <unistd.h>                                                │
│  #include <stdlib.h>                                                │
│  #include <sys/stat.h>                                              │
│  #include <fcntl.h>                                                 │
│                                                                     │
│  void daemonize() {                                                 │
│      // 1. fork                                                     │
│      pid_t pid = fork();                                            │
│      if (pid < 0) exit(1);                                          │
│      if (pid > 0) exit(0);  // 부모 종료                              │
│                                                                     │
│      // 2. 새 세션 생성                                               │
│      if (setsid() < 0) exit(1);                                     │
│                                                                     │
│      // 3. fork again (옵션, 제어 터미널 획득 방지)                    │
│      pid = fork();                                                  │
│      if (pid < 0) exit(1);                                          │
│      if (pid > 0) exit(0);                                          │
│                                                                     │
│      // 4. 파일 모드 마스크                                           │
│      umask(0);                                                      │
│                                                                     │
│      // 5. 작업 디렉토리                                              │
│      chdir("/");                                                    │
│                                                                     │
│      // 6. 표준 파일 디스크립터 닫기                                   │
│      close(STDIN_FILENO);                                           │
│      close(STDOUT_FILENO);                                          │
│      close(STDERR_FILENO);                                          │
│  }                                                                  │
│                                                                     │
│  【세션 정보 확인】                                                      │
│  ──────────────────                                                  │
│  # 세션, 프로세스 그룹 확인                                            │
│  $ ps -j                    # PGID, SID 표시                        │
│  $ ps -o pid,pgid,sid,comm  # 특정 필드                              │
│                                                                     │
│  # 세션 ID 확인                                                       │
│  $ ps -o sid= -p $$                                                  │
│                                                                     │
│  # 제어 터미널 확인                                                    │
│  $ tty                                                              │
│                                                                     │
│  【nohup 사용】                                                         │
│  ──────────────────                                                  │
│  # 터미널 종료 후에도 실행                                              │
│  $ nohup ./my_program &                                              │
│  $ nohup ./my_program > output.log 2>&1 &                           │
│                                                                     │
│  # disown으로 기존 작업 보호                                           │
│  $ sleep 100 &                                                       │
│  $ disown %1                                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
