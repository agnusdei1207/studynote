# 142. 그린 스레드 (Green Thread)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 공간에서 관리되는 경량 스레드
> 2. **가치**: 높은 동시성, 낮은 생성 비용
> 3. **융합**: 코루틴, M:N 스레딩, Go 스케줄러와 연관

---

## Ⅰ. 개요

### 개념 정의

그린 스레드(Green Thread)는 **사용자 공간 런타임에 의해 관리되는 경량 스레드**다. OS 커널이 아닌 가상 머신이나 런타임이 스케줄링을 담당한다.

### 💡 비유: 실내 자전거
그린 스레드는 **실내 자전거**와 같다. 페달은 밟지만(실행), 바퀴가 땅에 닿지 않는다(커널 스레드 아님).

```

│  │                                                             │ │
│  │  ┌─────────┐┌─────────┐     ┌──────────────────────────┐    │ │

│  │  │(OS 관리) ││(OS 관리) │     │                          │    │ │
│  │  └────┬────┘└────┬────┘     │  ┌────┐┌────┐┌────┐     │    │ │

│  │  │   스케줄러         │     │  ┌──────────────┐        │    │ │
│  │  └───────────────────┘     │  │  OS 스레드    │        │    │ │
│  │                            │  │  (M:N 매핑)  │        │    │ │
│  │                            │  └──────────────┘        │    │ │

│  │                                                             │ │

│  【주요 특징】                                                           │
│  ──────────────────                                                  │

│  │  특징                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  관리 주체              런타임/VM (커널 아님)                    │ │
│  │  스케줄링              사용자 공간                              │ │
│  │  컨텍스트 스위치        빠름 (커널 진입 없음)                     │ │
│  │  멀티코어              구현에 따라 다름                          │ │
│  │  블로킹 I/O            전체 스레드 블록 가능                     │ │
│  │                                                               │ │

│                                                                     │

│  │  • Java 1.2+: 네이티브 스레드로 전환                             │ │
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                그린 스레드 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구현 예시】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  환경                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  Java (초기)            Green threads in JDK 1.0               │ │
│  │  Go                     Goroutine (M:N 스케줄링)                │ │
│  │  Erlang                 Lightweight processes                  │ │
│  │  Haskell                Green threads (GHC)                    │ │
│  │  Python                 Greenlet, gevent                       │ │
│  │  Rust                   async/await + tokio                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 낮은 생성 비용          • 멀티코어 활용 어려움                │ │
│  │  • 빠른 컨텍스트 스위치     • 블로킹 I/O 문제                    │ │
│  │  • 높은 동시성             • 스케줄러 복잡성                     │ │
│  │  • 이식성                   • 디버깅 어려움                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【M:N 스레딩】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  M개의 그린 스레드 → N개의 OS 스레드에 매핑                       │ │
│  │                                                               │ │
│  │  • M >> N: 많은 그린 스레드를 적은 OS 스레드로 실행               │ │
│  │  • Work Stealing: 유휴 OS 스레드가 작업 훔침                     │ │
│  │  • Go의 GMP 모델이 대표적                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【블로킹 I/O 문제】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 동기 I/O 시 OS 스레드가 블록되면                              │ │
│  │    해당 스레드의 모든 그린 스레드가 블록됨                        │ │
│  │  • 해결: 비동기 I/O 사용                                        │ │
│  │  • Go: 네트워크 I/O를 비동기로 처리                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 비교 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Java 그린 스레드 (과거)】                                            │
│  ──────────────────                                                  │
│  // JDK 1.0-1.1: 그린 스레드                                        │
│  // 단일 OS 스레드에서 모든 그린 스레드 실행                            │
│                                                                     │
│  // JDK 1.2+: 네이티브 스레드로 전환                                  │
│  Thread t = new Thread(() -> {                                      │
│      // OS 스레드에서 실행                                            │
│  });                                                                │
│  t.start();                                                         │
│                                                                     │
│  【Go Goroutine (현대적 그린 스레드)】                                  │
│  ──────────────────                                                  │
│  // M:N 스케줄링으로 멀티코어 활용                                     │
│  for i := 0; i < 1000000; i++ {                                    │
│      go func(id int) {                                              │
│          // 그린 스레드에서 실행                                       │
│      }(i)                                                           │
│  }                                                                  │
│                                                                     │
│  【Python gevent】                                                     │
│  ──────────────────                                                  │
│  import gevent                                                      │
│  from gevent import monkey                                          │
│  monkey.patch_all()  # 표준 라이브러리 패치                           │
│                                                                     │
│  def task(n):                                                       │
│      gevent.sleep(1)  # 비동기 sleep                                 │
│      return n * 2                                                   │
│                                                                     │
│  jobs = [gevent.spawn(task, i) for i in range(10)]                  │
│  gevent.joinall(jobs)                                               │
│                                                                     │
│  【Rust async (Future 기반)】                                          │
│  ──────────────────                                                  │
│  use tokio;                                                         │
│                                                                     │
│  #[tokio::main]                                                     │
│  async fn main() {                                                  │
│      let tasks: Vec<_> = (0..10)                                    │
│          .map(|i| async move {                                      │
│              i * 2                                                  │
│          })                                                         │
│          .collect();                                                │
│                                                                     │
│      let results = futures::future::join_all(tasks).await;          │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
