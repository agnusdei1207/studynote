# 146. 작업 제어 (Job Control)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 포그라운드/백그라운드 프로세스 관리
> 2. **가치**: 멀티태스킹, 터미널 효율성
> 3. **융합**: 시그널, 세션, 프로세스 그룹과 연관

---

## Ⅰ. 개요

### 개념 정의

작업 제어(Job Control)는 **사용자가 포그라운드와 백그라운드 프로세스를 관리하는 기능**이다. 프로세스를 일시 중단, 재개, 전환할 수 있다.

### 💡 비유: 창 전환
작업 제어는 **컴퓨터의 창 전환**과 같다. 한 창을 최소화(백그라운드)하고 다른 창을 사용(포그라운드)한다.

### 작업 제어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                작업 제어 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Foreground vs Background】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Foreground 작업                Background 작업               │ │
│  │  ─────────────────             ─────────────────              │ │
│  │                                                             │ │
│  │  • 터미널 입력 수신              • 터미널 입력 불가              │ │
│  │  • 터미널 출력                  • 출력 가능 (리다이렉트 권장)     │ │
│  │  • Ctrl+C/Z 동작               • 시그널 영향 적음              │ │
│  │  • 셸 대기                     • 셸 즉시 사용 가능             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【작업 상태 전이】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │                    Ctrl+Z                                        │ │
│  │    ┌──────────────┐         ┌──────────────┐                  │ │
│  │    │   Running    │────────►│   Stopped    │                  │ │
│  │    │  (Foreground)│         │ (Background) │                  │ │
│  │    └──────────────┘         └──────┬───────┘                  │ │
│  │           ▲                        │                           │ │
│  │           │ fg                     │ bg                        │ │
│  │           │                        ▼                           │ │
│  │    ┌──────┴───────┐         ┌──────────────┐                  │ │
│  │    │   Running    │◄────────│   Running    │                  │ │
│  │    │  (Foreground)│   fg    │ (Background) │                  │ │
│  │    └──────────────┘         └──────────────┘                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 명령어】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어                    설명                                │ │
│  │  ────                        ────                                │ │
│  │  &                      백그라운드 실행                         │ │
│  │  Ctrl+Z                 일시 중단 (SIGTSTP)                     │ │
│  │  Ctrl+C                 종료 (SIGINT)                          │ │
│  │  jobs                   작업 목록                              │ │
│  │  fg %n                  작업 n을 포그라운드로                   │ │

└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                작업 제어 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【작업 상태】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  상태                      설명                                │ │
│  │  ────                        ────                                │ │
│  │  Running               실행 중                                  │ │
│  │  Stopped               일시 중단 (SIGTSTP, SIGSTOP)             │ │
│  │  Done                   완료                                    │ │
│  │  Terminated            종료됨                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시그널과 작업 제어】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시그널                    동작                                │ │
│  │  ────                        ────                                │ │
│  │  SIGINT (Ctrl+C)       포그라운드 프로세스 종료                  │ │
│  │  SIGTSTP (Ctrl+Z)      포그라운드 프로세스 일시 중단              │ │
│  │  SIGQUIT (Ctrl+\)      코어 덤프와 함께 종료                      │ │
│  │  SIGCONT               중단된 프로세스 재개                       │ │
│  │  SIGSTOP               강제 일시 중단                            │ │
│  │  SIGHUP               터미널 종료 시                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【터미널 제어】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • tcsetpgrp(): 포그라운드 프로세스 그룹 설정                    │ │
│  │  • tcgetpgrp(): 포그라운드 프로세스 그룹 조회                    │ │
│  │  • 백그라운드 프로세스가 터미널 읽기 시도 → SIGTTIN               │ │
│  │  • 백그라운드 프로세스가 터미널 쓰기 시도 → SIGTTOU (옵션)         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 멀티태스킹              • 터미널 필요                         │ │
│  │  • 유연한 작업 관리         • 백그라운드 입출력 복잡              │ │
│  │  • 프로세스 전환            • 일부 환경 미지원                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 사용법】                                                        │
│  ──────────────────                                                  │
│  # 백그라운드 실행                                                    │
│  $ sleep 100 &                                                       │
│  [1] 12345                                                          │
│                                                                     │
│  # 작업 목록                                                         │
│  $ jobs                                                              │
│  [1]+  Running    sleep 100 &                                        │
│  [2]-  Stopped    vim file                                          │
│                                                                     │
│  # 포그라운드로 전환                                                   │
│  $ fg %1                                                             │
│                                                                     │
│  # 백그라운드로 전환 (중단된 작업)                                       │
│  $ bg %2                                                             │
│                                                                     │
│  【복잡한 예시】                                                         │
│  ──────────────────                                                  │
│  # 파이프라인 백그라운드 실행                                           │
│  $ cat largefile | grep pattern | wc -l &                            │
│  [1] 23456                                                          │
│                                                                     │
│  # 백그라운드 작업 대기                                                 │
│  $ wait %1           # 작업 1 완료 대기                                │
│  $ wait              # 모든 백그라운드 대기                             │
│                                                                     │
│  # 작업 종료                                                          │
│  $ kill %2           # 작업 2 종료                                    │
│  $ kill -9 %3        # 강제 종료                                      │
│                                                                     │
│  【nohup과 disown】                                                     │
│  ──────────────────                                                  │
│  # 터미널 종료 후에도 실행                                               │
│  $ nohup ./server &                                                  │
│                                                                     │
│  # 실행 중인 작업 보호                                                  │
│  $ sleep 1000 &                                                      │
│  $ disown %1          # 셸이 종료해도 유지                              │
│                                                                     │
│  【bash 스크립트에서】                                                   │
│  ──────────────────                                                  │
│  #!/bin/bash                                                         │
│  # 백그라운드 작업들                                                    │
│  ./task1 &                                                           │
│  pid1=$!                                                             │
│  ./task2 &                                                           │
│  pid2=$!                                                             │
│                                                                     │
│  # 모두 완료 대기                                                       │
│  wait $pid1                                                          │
│  wait $pid2                                                          │
│                                                                     │
│  echo "모든 작업 완료"                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
