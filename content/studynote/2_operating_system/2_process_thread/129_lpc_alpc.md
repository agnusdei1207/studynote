# 로컬 프로시저 호출 (LPC) / ALPC (Windows)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: LPC(Local Procedure Call)는 윈도우(Windows) 운영체제에서 **동일한 시스템 내의 프로세스 간에 메시지를 주고받기 위해 사용하는 고속 IPC(Inter-Process Communication) 매커니즘**이다.
> 2. **가치**: 클라이언트-서버 모델을 기반으로 하며, 메시지 크기에 따라 공유 메모리나 포트(Port) 방식을 유동적으로 선택하여 **통신 성능을 최적화**한다.
> 3. **융합**: 현대 윈도우 비스타 이후부터는 비동기 성능을 대폭 강화한 **ALPC (Advanced LPC)**로 진화하였으며, 윈도우 서브시스템(Win32, RPC 등)의 핵심 통신 인프라로 작동한다.

+++

## Ⅰ. LPC의 정의 및 아키텍처

### 1. 정의
- 원격 프로시저 호출(RPC)의 로컬 버전으로, 네트워크가 아닌 동일 머신 내의 프로세스 간 통신을 위해 특화된 윈도우 전용 기술이다.

### 2. 기본 구조
- **커널 포트 (Connection Port)**: 서버가 클라이언트의 연결 요청을 기다리는 통로.
- **통신 포트 (Communication Port)**: 연결이 수립된 후 실제 메시지를 주고받는 전용 통로.

### 3. 💡 비유: '사내 인터폰 시스템'
- LPC는 **'한 건물(시스템) 안의 부서(프로세스)끼리만 사용하는 전용 인터폰'**과 같다.
- 외부 전화망(RPC/Network)을 거치지 않고 내부 선로(커널)만 사용하여 옆 부서와 매우 빠르게 소통하는 시스템이다.

+++

## II. LPC의 세 가지 메시지 전달 방식 (Deep Dive)

메시지의 크기에 따라 커널이 가장 효율적인 방법을 선택한다.

### 1. 포트 메시지 (Short Message)
- **대상**: 256바이트 이하의 작은 데이터.
- **방식**: 메시지를 커널 버퍼에 직접 복사하여 상대방에게 전달한다.

### 2. 공유 메모리 (LPC Section)
- **대상**: 대용량 데이터.
- **방식**: '섹션(Section)' 객체를 생성하여 두 프로세스가 메모리를 공유하고, 포트로는 해당 메모리의 포인터만 보낸다. 복사 오버헤드를 원천 차단한다.

### 3. 직접 쓰기 (Quick LPC)
- **대상**: 서버가 클라이언트의 주소 공간에 직접 접근 가능한 경우.
- **방식**: 데이터를 상대방의 메모리에 직접 쓴다.

+++

## Ⅲ. ALPC (Advanced LPC)의 등장

윈도우 비스타(Vista) 이후 도입된 개선 버전으로, 현대 윈도우의 표준이다.

| 항목 | 기존 LPC | ALPC (Advanced) |
|:---|:---|:---|
| **동기화** | 주로 동기식(Blocking) | **비동기식(Non-blocking) 강화** |
| **성능** | 단순 메시지 큐 방식 | 일괄 처리(Batching) 및 고속 전송 최적화 |
| **보안** | 기본적인 프로세스 격리 | 프로세스 신뢰 수준에 따른 세밀한 보안 제어 |

- **📢 섹션 요약 비유**: 일반 인터폰(LPC)에서 영상 통화와 다자간 회의가 가능한 최첨단 스마트 협업 시스템(ALPC)으로 업그레이드된 것과 같습니다.

+++

## Ⅳ. 실무 적용 및 아키텍처 인사이트 (Strategy)

### 1. 실무 시나리오: 윈도우 서비스 통신
- **상황**: 유저 모드 앱이 윈도우 보안 서브시스템(LSASS)에 인증 요청을 보냄.
- **판단**: 이때 내부적으로 **ALPC**가 작동한다. 앱은 라이브러리 함수를 호출할 뿐이지만, 실제로는 ALPC 포트를 통해 보안 서비스 프로세스로 요청이 전달되고 결과가 돌아온다. 기술사라면 윈도우의 수많은 '서비스'들이 이 ALPC라는 보이지 않는 실로 묶여 있음을 이해해야 한다.

### 2. 기술사적 인사이트: 마이크로커널 사상의 구현
- 기술사라면 LPC가 **마이크로커널(Microkernel)** 설계 사상의 유산임을 짚어야 한다. 기능을 커널에 다 넣지 않고 별도 프로세스로 분리하되, 그 사이의 성능 저하를 LPC라는 고속 통신망으로 극복하려 했던 윈도우 NT의 초기 철학을 설명할 수 있어야 한다.

- **📢 섹션 요약 비조**: "부서 간 거리를 멀리 두어(격리) 안전을 챙기되, 그 사이를 고속 열차(LPC)로 연결하여 속도를 챙기는 영리한 도시 설계와 같습니다."

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **시스템 전체 성능 향상**: 빈번한 OS 서비스 호출의 오버헤드 최소화.
- **견고한 보안 격리**: 직접 메모리 침범 없이 커널 중재 하에 안전한 데이터 교환.

### 2. 미래 전망
앞으로는 컨테이너 가상화 기술이 윈도우 커널 수준까지 깊숙이 침투함에 따라, 서로 다른 격리 환경(Silo) 간의 ALPC 통신 효율을 높이는 기술이 더욱 중요해질 것이다. ALPC는 윈도우라는 거대한 운영체제가 유기적으로 숨 쉬게 만드는 보이지 않는 신경망이다.

- **📢 섹션 요약 비유**: 수만 개의 세포(프로세스)가 한 몸(OS)처럼 움직이게 돕는, 가장 빠르고 정확한 디지털 신경 전달 물질입니다.
