# 138. 멀티스레드 아키텍처 오버헤드

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멀티스레드 환경의 성능 비용 요소
> 2. **가치**: 성능 최적화를 위한 오버헤드 분석
> 3. **융합**: 동기화, 컨텍스트 스위치, 캐시와 연관

---

## Ⅰ. 개요

### 개념 정의

멀티스레드 아키텍처 오버헤드는 **멀티스레드 환경에서 발생하는 추가 성능 비용**이다. 동기화, 컨텍스트 스위치, 캐시 미스 등이 주요 원인이다.

### 💡 비유: 회의실 조정
멀티스레드 오버헤드는 **여러 팀이 회의실을 쓰려고 조정하는 시간**과 같다. 혼자 쓸 때보다 대기하고 조율하는 시간이 든다.

### 멀티스레드 오버헤드 구조

```

│                                                                     │
│  【주요 오버헤드 요소】                                                 │

│  │  │  • 락 획득/해제 비용                                      │ │ │

│  │  │  • 세마포어, 뮤텍스 대기                                  │ │ │
│  │  │  • 메모리 배리어                                         │ │ │

│  │                                                             │ │

│  │  │  • 레지스터 저장/복원                                     │ │ │

│  │  │  • 스케줄링 결정                                          │ │ │

│  │  └─────────────────────────────────────────────────────────┘ │ │

│  │  3. 캐시 오버헤드                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │

│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │

│  【오버헤드 비교】                                                       │

│  │  작업                      오버헤드 (대략)                       │ │

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                멀티스레드 오버헤드 상세                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【False Sharing (거짓 공유)】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  CPU 0 캐시 라인         CPU 1 캐시 라인                      │ │
│  │  ┌───────────────────┐  ┌───────────────────┐               │ │
│  │  │ [A][B][C][D][E]...│  │ [A][B][C][D][E]...│               │ │
│  │  └─────────┬─────────┘  └─────────┬─────────┘               │ │
│  │            │                      │                          │ │
│  │      스레드 1이 A 수정      스레드 2가 E 수정                   │ │
│  │            │                      │                          │ │
│  │            └──────────┬───────────┘                          │ │
│  │                       ▼                                      │ │
│  │            같은 캐시 라인 → 캐시 일관성 트래픽 발생              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락 경합 비용】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  상황                      성능 영향                           │ │
│  │  ────                        ────                               │ │
│  │  락 없음                  최고 성능, 경쟁 조건                   │ │
│  │  낮은 경합                좋은 성능                             │ │
│  │  높은 경합                급격한 성능 저하                       │ │
│  │  Spinlock                짧은 대기에 효율                       │ │
│  │  Mutex                   긴 대기에 효융                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최적화 기법】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  기법                      효과                                │ │
│  │  ────                        ────                               │ │
│  │  Lock-free 자료구조       락 오버헤드 제거                      │ │
│  │  Read-Copy-Update (RCU)   읽기 최적화                          │ │
│  │  캐시 라인 패딩           False Sharing 방지                   │ │
│  │  스레드 로컬 저장         동기화 감소                          │ │
│  │  작업 스케줄링            캐시 지역성 활용                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【False Sharing 방지】                                                │
│  ──────────────────                                                  │
│  // 문제 코드                                                         │
│  struct Counter {                                                   │
│      long value1;  // 스레드 1 사용                                   │
│      long value2;  // 스레드 2 사용                                   │
│  };  // 같은 캐시 라인에 있을 수 있음                                   │
│                                                                     │
│  // 해결: 캐시 라인 패딩                                              │
│  struct Counter {                                                   │
│      long value1;                                                   │
│      char padding[56];  // 64 - 8 = 56바이트 패딩                    │
│      long value2;                                                   │
│  } __attribute__((aligned(64)));                                    │
│                                                                     │
│  // C++17 alignas 사용                                               │
│  struct alignas(64) Counter {                                       │
│      std::atomic<long> value1;                                      │
│      char padding[56];                                              │
│      std::atomic<long> value2;                                      │
│  };                                                                 │
│                                                                     │
│  【락 경합 측정】                                                        │
│  ──────────────────                                                  │
│  // perf로 락 경합 분석                                               │
│  $ perf record -e lock:contention_begin ./program                   │
│  $ perf report                                                      │
│                                                                     │
│  // ftrace로 컨텍스트 스위치 분석                                      │
│  $ trace-cmd record -e sched:sched_switch ./program                 │
│  $ trace-cmd report                                                 │
│                                                                     │
│  【스레드 수 최적화】                                                   │
│  ──────────────────                                                  │
│  // 최적 스레드 수 = CPU 코어 수 * (1 + 대기시간/계산시간)              │
│  int optimal_threads = std::thread::hardware_concurrency();         │
│                                                                     │
│  // I/O 바운드 작업은 더 많은 스레드                                   │
│  int io_threads = optimal_threads * 2;                              │
│                                                                     │
│  // CPU 바운드 작업은 코어 수                                          │
│  int cpu_threads = optimal_threads;                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 동기화: 락, 세마포어, 메모리 배리어
• 컨텍스트 스위치: 레지스터, TLB, 스케줄링
• 캐시: False Sharing, 일관성 트래픽
• 관리: 생성, 종료, TCB
• 최적화: Lock-free, RCU, 패딩, TLS
• 측정: perf, ftrace
```
