# 137. 멀티프로세스 아키텍처

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 프로세스를 활용한 소프트웨어 설계
> 2. **가치**: 격리된 실행 환경, 안정성, 확장성
> 3. **융합**: IPC, 병렬 처리, 마이크로서비스와 연관

---

## Ⅰ. 개요

### 개념 정의

멀티프로세스 아키텍처(Multiprocess Architecture)는 **여러 독립적인 프로세스로 구성된 소프트웨어 아키텍처**다. 각 프로세스는 격리된 메모리 공간에서 실행되며 IPC로 통신한다.

### 💡 비유: 독립된 사무실
멀티프로세스 아키텍처는 **독립된 사무실들**과 같다. 각 방은 따로 있고(격리), 전화로 통신한다(IPC).

│  ──────────────────                                                  │

│  │  • Apache Prefork MPM                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 높은 메모리 사용                                             │ │
│  │  • IPC 오버헤드                                                 │ │
│  │  • 프로세스 생성 비용                                            │ │
│  │  • 상태 공유 어려움                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
