# 136. 좀비 스레드 (Zombie Thread)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 종료했지만 자원이 회수되지 않은 스레드
> 2. **가치**: 스레드 누수 문제 진단
> 3. **융합**: 좀비 프로세스, 스레드 관리, 자원 해제와 연관

---

## Ⅰ. 개요

### 개념 정의

좀비 스레드(Zombie Thread)는 **실행을 완료했지만 스레드 관련 자원이 완전히 회수되지 않은 스레드**다. 조인되지 않은 스레드가 대표적이다.

### 💡 비유: 퇴근했지만 자리 정리 안 한 직원
좀비 스레드는 **퇴근했지만 자리에 짐이 남아있는 직원**과 같다. 일은 끝났지만 공간을 차지한다.

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【좀비 스레드 생성 (문제)】                                            │
│  ──────────────────                                                  │
│  void* thread_func(void* arg) {                                     │
│      // 스레드 작업                                                   │
│      return NULL;                                                   │
│  }                                                                  │
│                                                                     │
│  int main() {                                                       │
│      pthread_t tid;                                                 │
│      pthread_create(&tid, NULL, thread_func, NULL);                 │
│      // join 없음 → 좀비 스레드 발생                                   │
│      return 0;                                                      │
│  }                                                                  │
│                                                                     │
│  【해결 방법 1: pthread_join】                                          │
│  ──────────────────                                                  │
│  int main() {                                                       │
│      pthread_t tid;                                                 │
│      pthread_create(&tid, NULL, thread_func, NULL);                 │
│      pthread_join(tid, NULL);  // 명시적 조인                        │
│      return 0;                                                      │
│  }                                                                  │
│                                                                     │
│  【해결 방법 2: pthread_detach】                                        │
│  ──────────────────                                                  │
│  void* thread_func(void* arg) {                                     │
│      pthread_detach(pthread_self());  // 자신을 분리                  │
│      // 스레드 작업                                                   │
│      return NULL;                                                   │
│  }                                                                  │
│                                                                     │
│  【해결 방법 3: 생성 시 detached 속성】                                  │
│  ──────────────────                                                  │
│  pthread_t tid;                                                     │
│  pthread_attr_t attr;                                               │
│  pthread_attr_init(&attr);                                          │
│  pthread_attr_setdetachstate(&attr, PTHREAD_CREATE_DETACHED);       │
│  pthread_create(&tid, &attr, thread_func, NULL);                    │
│  pthread_attr_destroy(&attr);                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 종료했지만 자원 미회수 스레드
• 원인: join 누락, detached 미설정
• 문제: 메모리 누수, 자원 고갈
• 해결: join, detach, 속성 설정
• 감지: /proc, valgrind, tsan
• 예방: RAII, 스마트 포인터
```
