# 316. 롤백 (Rollback)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이전 상태로 되돌리는 복구 작업
> 2. **가치**: 교착상태 복구의 핵심 메커니즘
> 3. **융합**: 희생자 선택, 체크포인트와 연관

---

## Ⅰ. 개요

### 개념 정의

롤백(Rollback)은 **프로세스나 트랜잭션을 이전 안전한 상태로 되돌리는 작업**이다. 교착상태 복구 시 희생자 프로세스를 롤백하여 자원을 해제한다.

### 💡 비유: 게임 세이브
롤백은 **게임 세이브**와 같다. 죽으면 저장했던 지점으로 돌아간다. 진행 상황은 잃지만 다시 시작할 수 있다.

### 롤백 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                롤백 구조                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 원리】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시간 ─────────────────────────────────────────────►              │ │
│  │                                                             │ │
│  │  ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐           │ │
│  │  │CP1   │   │CP2   │   │CP3   │   │현재   │   │      │           │ │
│  │  │안전   │   │안전   │   │안전   │   │교착   │   │      │           │ │
│  │  └──────┘   └──────┘   └──────┘   └──────┘   └──────┘           │ │
│  │      ◄─────────────────────────── 롤백                           │ │
│  │                                                             │ │
│  │  CP = Checkpoint (체크포인트)                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【롤백 종류】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 전체 롤백:                                                       │ │
│  │     • 처음부터 다시 시작                                              │ │
│  │     • 모든 작업 취소                                                  │ │
│  │     • 가장 큰 손실                                                    │ │
│  │                                                             │ │
│  │  2. 부분 롤백:                                                       │ │
│  │     • 가장 최근 체크포인트로 복구                                      │ │
│  │     • 일부 작업만 취소                                                │ │
│  │     • 효율적이지만 구현 복잡                                          │ │
│  │                                                             │ │
│  │  3. 최소 롤백:                                                       │ │
│  │     • 교착상태 해소에 필요한 최소한만 롤백                              │ │
│  │     • 가장 효율적                                                     │ │
│  │     • 구현 매우 복잡                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【롤백 과정】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 롤백 대상 식별                                                    │ │
│  │  2. 체크포인트 위치 확인                                               │ │
│  │  3. 변경 사항 취소 (undo)                                            │ │
│  │  4. 자원 해제                                                        │ │
│  │  5. 상태 복원                                                        │ │
│  │  6. 재시작 또는 종료                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                롤백 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【체크포인트와 롤백】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  체크포인트: 정기적으로 저장하는 안전한 상태                             │ │
│  │                                                             │ │
│  │  저장 내용:                                                            │ │
│  │  • 프로세스 상태                                                       │ │
│  │  • 메모리 내용                                                         │ │
│  │  • 자원 할당 정보                                                      │ │
│  │  • 열린 파일 목록                                                      │ │
│  │                                                             │ │
│  │  롤백 비용:                                                            │ │
│  │  • 체크포인트 간격이 짧을수록 롤백 비용 낮음                             │ │
│  │  • 하지만 체크포인트 오버헤드 증가                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Undo 로그】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  변경 사항을 되돌리기 위한 로그:                                        │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │ TxID │ Operation │ Before │ After │ Timestamp │          │  │ │
│  │  │──────│───────────│────────│───────│───────────│          │  │ │
│  │  │ T1   │ Write     │ A=10   │ A=20  │ t1        │          │  │ │
│  │  │ T1   │ Write     │ B=30   │ B=40  │ t2        │          │  │ │
│  │  │ T1   │ Lock      │ R1     │ -     │ t3        │          │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  │  롤백 시 로그를 역순으로 읽어 undo 수행                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【부분 롤백 vs 전체 롤백】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성            전체 롤백           부분 롤백                    │ │
│  │  ────────        ────────────        ────────────               │ │
│  │  손실            모든 작업            일부 작업                   │ │
│  │  복구 시간        길음                짧음                       │ │
│  │  구현 복잡도      단순                복잡                       │ │
│  │  오버헤드         높음                낮음                       │ │
│  │  활용            트랜잭션             교착상태 복구               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【트랜잭션 롤백】                                                      │
│  ──────────────────                                                  │
│  // SQL                                                               │
│  BEGIN TRANSACTION;                                                   │
│  UPDATE accounts SET balance = balance - 100 WHERE id = 1;          │
│  UPDATE accounts SET balance = balance + 100 WHERE id = 2;          │
│  -- 문제 발생 시                                                       │
│  ROLLBACK;  -- 모든 변경 취소                                          │ │
│                                                                     │
│  -- 성공 시                                                           │
│  COMMIT;                                                              │
│                                                                     │
│  // JDBC                                                               │
│  Connection conn = dataSource.getConnection();                      │
│  conn.setAutoCommit(false);                                          │
│  try {                                                               │
│      // 작업                                                           │
│      conn.commit();                                                  │
│  } catch (Exception e) {                                             │
│      conn.rollback();  // 롤백                                        │ │
│  }                                                                   │
│                                                                     │
│  【체크포인트 기반 롤백】                                                │
│  ──────────────────                                                  │
│  class CheckpointManager {                                          │
│      List<Checkpoint> checkpoints = new ArrayList<>();              │
│                                                                     │
│      void saveCheckpoint(Process p) {                               │
│          Checkpoint cp = new Checkpoint(                            │
│              p.getId(),                                              │
│              p.getState(),                                          │
│              p.getMemory(),                                         │
│              p.getResources(),                                      │
│              System.currentTimeMillis()                             │
│          );                                                          │
│          checkpoints.add(cp);                                        │
│      }                                                              │
│                                                                     │
│      void rollback(Process p) {                                     │
│          Checkpoint cp = findLatestCheckpoint(p);                   │
│          if (cp != null) {                                          │
│              p.setState(cp.getState());                             │
│              p.restoreMemory(cp.getMemory());                       │
│              // 자원은 해제 후 재할당                                   │ │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Undo 로그 구현】                                                     │
│  ──────────────────                                                  │
│  class UndoLog {                                                    │
│      List<LogEntry> entries = new ArrayList<>();                    │
│                                                                     │
│      void logWrite(long txId, String key, Object oldValue) {        │
│          entries.add(new LogEntry(txId, "WRITE", key, oldValue));  │
│      }                                                              │
│                                                                     │
│      void logLock(long txId, Object resource) {                     │
│          entries.add(new LogEntry(txId, "LOCK", resource, null));  │
│      }                                                              │
│                                                                     │
│      void undo(long txId) {                                         │
│          // 역순으로 로그 처리                                          │ │
│          for (int i = entries.size() - 1; i >= 0; i--) {            │
│              LogEntry entry = entries.get(i);                       │
│              if (entry.txId == txId) {                              │
│                  switch (entry.operation) {                         │
│                      case "WRITE":                                  │
│                          restoreValue(entry.key, entry.oldValue);  │
│                          break;                                     │
│                      case "LOCK":                                   │
│                          releaseLock(entry.resource);               │
│                          break;                                     │
│                  }                                                  │
│              }                                                      │
│          }                                                          │
│          // 해당 트랜잭션 로그 제거                                     │ │
│          entries.removeIf(e -> e.txId == txId);                     │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【데이터베이스 롤백】                                                   │
│  ──────────────────                                                  │
│  // Spring @Transactional                                           │
│  @Transactional                                                     │
│  public void transferMoney(Long from, Long to, int amount) {        │
│      Account fromAccount = accountRepo.findById(from);              │
│      Account toAccount = accountRepo.findById(to);                  │
│                                                                     │
│      fromAccount.setBalance(fromAccount.getBalance() - amount);     │
│      toAccount.setBalance(toAccount.getBalance() + amount);         │
│                                                                     │
│      accountRepo.save(fromAccount);                                 │
│      accountRepo.save(toAccount);                                   │
│      // 예외 발생 시 자동 롤백                                          │ │
│  }                                                                  │
│                                                                     │
│  【분산 트랜잭션 롤백】                                                   │
│  ──────────────────                                                  │
│  class DistributedTransactionManager {                              │
│      void rollback(List<Participant> participants) {                │
│          for (Participant p : participants) {                       │
│              try {                                                   │
│                  p.rollback();  // 각 참여자에게 롤백 요청              │ │
│              } catch (Exception e) {                                │
│                  // 롤백 실패 시 재시도 또는 로깅                        │ │
│                  log.error("Rollback failed for participant: " + p);│
│              }                                                      │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【롤백 모니터링】                                                       │
│  ──────────────────                                                  │
│  class RollbackMetrics {                                            │
│      AtomicLong rollbackCount = new AtomicLong();                   │
│      AtomicLong rollbackTime = new AtomicLong();                    │
│                                                                     │
│      void recordRollback(long duration) {                           │
│          rollbackCount.incrementAndGet();                           │
│          rollbackTime.addAndGet(duration);                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 이전 상태로 되돌리는 복구 작업
• 종류: 전체, 부분, 최소 롤백
• 도구: 체크포인트, Undo 로그
• 과정: 식별 → 취소 → 해제 → 복원 → 재시작
• 비용: 체크포인트 간격과 반비례
• 활용: 트랜잭션, 교착상태 복구
```
