# 실전 교착 상태 트러블슈팅 가이드

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실전 데드락 트러블슈팅은 시스템이 멈춘 긴박한 상황에서 원인을 신속히 식별하고, 최소한의 서비스 중단으로 정상 상태를 복구하는 체계적인 대응 프로세스이다.
> 2. **단계**: 모니터링을 통한 징후 포착 -> 스레드/락 상태 데이터 수집 -> 순환 의존성 분석 -> 임시 조치(프로세스 재시작 등) -> 근본 원인(Lock Ordering 규칙 등) 해결 순으로 진행한다.
> 3. **가치**: 반복되는 동기화 문제를 단순히 '재시작'으로 회피하지 않고, 구조적 결함을 찾아내어 시스템의 장기적인 견고함을 확보하는 데 기여한다.

+++

## Ⅰ. 트러블슈팅의 시작: 징후 포착

### 1. 주요 감지 포인트
- **애플리케이션 로그**: 'Lock wait timeout exceeded' 또는 'Transaction Deadlock' 에러 발생.
- **모니터링 도구**: CPU 사용률 급락(정적 데드락) 또는 급등(라이브락), 특정 서비스의 응답 지연(Latency) 폭증.
- **리소스 대기열**: 자원 요청 큐(Queue)가 비정상적으로 길어짐.

- **📢 섹션 요약 비유**: "자동차가 갑자기 멈추거나 연기가 나면 본넷을 열어보기 전에 계기판의 경고등부터 확인하는 것과 같습니다."

+++

## Ⅱ. 상황 분석 매핑 (ASCII Diagram)

### 1. 트러블슈팅 워크플로우 (Workflow)
```text
 [ Detect ] ---> [ Analyze ] ---> [ Resolve ] ---> [ Prevent ]
     |              |                |                 |
     +-- Alarm      +-- Thread Dump  +-- Kill Process  +-- Code Fix
     +-- User Rep.  +-- Lock Graph   +-- Rollback      +-- Lock Order
```

### 2. 분석 매트릭스
- **DB 데드락**: DB 엔진이 제공하는 데드락 리포트(`SHOW ENGINE INNODB STATUS` 등) 확인.
- **App 데드락**: 스레드 덤프(`jstack`, `pstack`)로 락 소유권 추적.

- **📢 섹션 요약 비유**: "환자가 오면 어디가 아픈지 묻고(감지), 엑스레이를 찍고(분석), 약을 주거나 수술하고(해결), 생활 습관을 고치게 하는(예방) 병원 시스템과 같습니다."

+++

## Ⅲ. 긴급 복구 및 조치 전략

### 1. 최소 파괴적 해소 (Least Disruptive Termination)
- 교착 상태에 연루된 프로세스 중 가장 '저렴한' 하나만 골라 강제 종료(SIGKILL)한다.

### 2. 타임아웃 강제 적용
- 설정된 타임아웃 값이 너무 길다면, 임시로 값을 줄여 자가 해소를 유도한다.

### 3. 부하 분산 (Traffic Control)
- 특정 자원에 쏠린 트래픽을 차단(Circuit Breaker)하여 추가적인 데드락 발생을 막는다.

- **📢 섹션 요약 비유**: "불이 났을 때 건물 전체를 부수는 게 아니라, 불이 난 방 하나만 격리하거나 소화기를 뿌리는 것과 같습니다."

+++

## Ⅳ. 근본 원인 해결 (Root Cause Analysis)

| 문제 유형 | 원인 분석 | 해결 방안 |
|:---|:---|:---|
| **락 오더링 부재** | 개발자마다 락 획득 순서가 다름 | 전사적 락 획득 순서 표준 정의 |
| **중첩 락 (Nested Lock)** | 불필요하게 여러 락을 한꺼번에 잡음 | 락의 범위를 최소화하고 단일화 |
| **비결정적 실행** | 외부 이벤트에 따라 락 순서가 바뀜 | 락 획득 로직을 순차적/결정론적으로 수정 |

- **📢 섹션 요약 비유**: "넘어진 이유가 돌부리 때문이라면 돌부리를 치우고, 내 신발 끈 때문이라면 끈을 다시 꽉 묶는 근본적인 대책을 세우는 것입니다."

+++

## Ⅴ. 결론 및 실무적 교훈

### 1. 요약
- 트러블슈팅은 당장의 불을 끄는 기술(Emergency Action)과 다음 불을 막는 지혜(Prevention)가 결합된 활동이다.

### 2. 최종 결론
- 완벽한 데드락 방지란 불가능에 가깝다. 하지만 "감지하지 못하는 데드락은 있어도, 트러블슈팅 할 수 없는 데드락은 없다"는 마음가짐으로 모니터링과 분석 체계를 구축하는 것이 시니어 엔지니어의 핵심 덕목이다.

- **📢 섹션 요약 비유**: "사고는 예고 없이 찾아오지만, 잘 훈련된 대처법이 있다면 큰 피해를 막을 수 있다는 교훈을 줍니다."

+++

### 📌 관련 개념 맵 (Knowledge Graph)
- **교착 상태 탐지 (Detection)**: 분석 단계의 핵심 기술.
- **스레드 덤프 (Thread Dump)**: 실전 분석에서 가장 많이 쓰이는 무기.
- **서킷 브레이커 (Circuit Breaker)**: 과부하로 인한 데드락 확산 방지책.
