# 317. 교착상태 무시 (Deadlock Ignorance)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착상태를 처리하지 않고 무시하는 전략
> 2. **가치**: 현실적이고 실용적인 접근법
> 3. **융합**: 타조 알고리즘, 교착상태 예방과 연관

---

## Ⅰ. 개요

### 개념 정의

교착상태 무시(Deadlock Ignorance)는 **교착상태가 발생하더라도 무시하고, 발생 시 재부팅으로 해결하는 전략**이다. 교착상태를 예방, 회피, 탐지, 복구하지 않는다.

### 💡 비유: 모래사장 타조
교착상태 무시는 **모래사장에 머리를 묻는 타조**와 같다. 문제가 있어도 보지 않는다. 문제가 커지면 다시 시작한다.

### 교착상태 무시 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                교착상태 무시 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전략 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  교착상태 전략:                                                        │ │
│  │                                                             │ │
│  │  1. 예방 (Prevention): 조건 차단                                       │ │
│  │     • 교착상태 불가능                                                   │ │
│  │     • 비용 높음                                                        │ │
│  │                                                             │ │
│  │  2. 회피 (Avoidance): 안전한 할당만 허용                                │ │
│  │     • 교착상태 회피                                                     │ │
│  │     • 복잡한 계산                                                      │ │
│  │                                                             │ │
│  │  3. 탐지 및 복구 (Detection & Recovery): 감지 후 해결                   │ │
│  │     • 교착상태 발생 가능                                                 │ │
│  │     • 복구 비용                                                        │ │
│  │                                                             │ │
│  │  4. 무시 (Ignorance): 아무것도 안 함  ← 이것!                          │ │
│  │     • 교착상태 발생 가능                                                 │ │
│  │     • 발생 시 재부팅                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【무시 전략의 동작】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  정상 운영                                                            │ │
│  │       │                                                             │ │
│  │       ▼                                                             │ │
│  │  교착상태 발생?                                                       │ │
│  │       │                                                             │ │
│  │       │ 예                                                           │ │
│  │       ▼                                                             │ │
│  │  사용자가 감지 (시스템 멈춤)                                           │ │
│  │       │                                                             │ │
│  │       ▼                                                             │ │
│  │  재부팅                                                               │ │
│  │       │                                                             │ │
│  │       ▼                                                             │ │
│  │  정상 운영 재개                                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【사용 예시】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • Windows, Linux, macOS 등 대부분의 OS                             │ │
│  │  • 교착상태 처리 메커니즘 없음                                          │ │
│  │  • 사용자가 멈춤을 감지하면 재부팅                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                교착상태 무시 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【무시가 가능한 이유】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 교착상태 발생 빈도:                                                │ │
│  │     • 실제로는 드물게 발생                                             │ │
│  │     • 처리 비용 > 발생 비용                                            │ │
│  │                                                             │ │
│  │  2. 예방/회피 비용:                                                   │ │
│  │     • 성능 저하                                                       │ │
│  │     • 자원 활용률 감소                                                │ │
│  │     • 구현 복잡도 증가                                                │ │
│  │                                                             │ │
│  │  3. 사용자 수용:                                                      │ │
│  │     • 가끔 재부팅하는 것을 용인                                        │ │
│  │     • "멈췄으면 다시 켜라"는 상식적 해결                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                                │ │
│  │  • 구현 간단 (아무것도 안 함)                                          │ │
│  │  • 런타임 오버헤드 없음                                                │ │
│  │  • 성능 저하 없음                                                     │ │
│  │  • 대부분의 경우 잘 작동                                               │ │
│  │                                                             │ │
│  │  단점:                                                                │ │
│  │  • 교착상태 발생 시 시스템 멈춤                                         │ │
│  │  • 데이터 손실 가능                                                   │ │
│  │  • 사용자 개입 필요                                                   │ │
│  │  • 실시간 시스템에 부적합                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적용 기준】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  무시가 적합한 경우:                                                    │ │
│  │  • 교착상태 발생 빈도가 낮음                                            │ │
│  │  • 일시적 멈춤이 용인됨                                                │ │
│  │  • 성능이 중요함                                                      │ │
│  │  • 예: 일반 PC, 워크스테이션                                           │ │
│  │                                                             │ │
│  │  무시가 부적합한 경우:                                                  │ │
│  │  • 교착상태 발생 빈도가 높음                                            │ │
│  │  • 실시간 응답 필요                                                   │ │
│  │  • 데이터 손실 용인 불가                                               │ │
│  │  • 예: 실시간 시스템, 서버, 임베디드                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【운영체제에서의 무시】                                                 │
│  ──────────────────                                                  │
│  // Linux: 교착상태 처리 메커니즘 없음                                    │ │
│  // 커널 락 획득 코드                                                    │ │
│  void mutex_lock(struct mutex *lock) {                              │
│      // 무한 대기 (교착상태 가능)                                        │ │
│      while (!mutex_trylock(lock)) {                                 │
│          schedule();  // 다른 프로세스에게 CPU 양보                      │ │
│      }                                                              │
│  }                                                                  │
│  // 교착상태 발생 시 사용자가 감지하고 재부팅                               │ │
│                                                                     │
│  【응용 프로그램에서의 무시】                                             │
│  ──────────────────                                                  │
│  // 간단한 락 사용 (교착상태 처리 없음)                                   │ │
│  public void transfer(Account from, Account to, int amount) {       │
│      synchronized (from) {                                          │
│          synchronized (to) {  // 교착상태 가능!                        │ │
│              from.withdraw(amount);                                 │
│              to.deposit(amount);                                    │
│          }                                                          │
│      }                                                              │
│      // 교착상태 예방/회피 없음                                          │ │
│      // 발생하면 앱이 멈춤 → 사용자가 재시작                               │ │
│  }                                                                  │
│                                                                     │
│  【사용자 대응】                                                         │
│  ──────────────────                                                  │
│  교착상태 감지 시 사용자가 하는 일:                                       │
│  • Windows: 작업 관리자 → 프로세스 종료                                  │ │
│  • Linux: kill -9 <pid>                                             │
│  • macOS: 강제 종료                                                   │
│  • 모두 응답 없음: 재부팅                                                │
│                                                                     │
│  【서버 시스템의 경우】                                                   │
│  ──────────────────                                                  │
│  // 교착상태 감지는 DBMS가 처리                                          │ │
│  // OS 레벨에서는 무시                                                   │ │
│  // DBMS 예: MySQL                                                    │
│  // innodb_deadlock_detect = ON (DBMS 레벨 탐지)                      │
│  // OS 레벨에서는 무시                                                   │ │
│                                                                     │
│  【Watchdog로 간접 처리】                                               │
│  ──────────────────                                                  │
│  // 교착상태 "무시"하지만 Watchdog로 감지                                 │ │
│  class Watchdog {                                                   │
│      Map<Thread, Long> lastActivity = new ConcurrentHashMap<>();    │
│      long timeout = 60000;  // 60초                                  │ │
│                                                                     │
│      void registerActivity(Thread t) {                              │
│          lastActivity.put(t, System.currentTimeMillis());            │
│      }                                                              │
│                                                                     │
│      void checkDeadlockedThreads() {                                │
│          long now = System.currentTimeMillis();                      │
│          for (Map.Entry<Thread, Long> entry : lastActivity.entrySet()) {│
│              if (now - entry.getValue() > timeout) {                │
│                  // 응답 없음 → 교착상태 가능                             │ │
│                  log.warn("Thread {} appears deadlocked",           │
│                           entry.getKey().getName());                 │
│                  // 강제 종료 또는 알림                                   │ │
│              }                                                      │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【무시 전략의 문서화】                                                   │
│  ──────────────────                                                  │
│  // README 또는 사용자 가이드                                            │ │
│  /*                                                                  │
│   * Known Issues:                                                    │
│   * - 프로그램이 멈출 수 있음 (교착상태)                                    │ │
│   * - 해결 방법: 프로그램 재시작                                           │ │
│   * - 이 문제는 드물게 발생하며 수정 계획 없음                               │ │
│   */                                                                 │
│                                                                     │
│  【재시작 자동화】                                                        │
│  ──────────────────                                                  │
│  // Systemd 서비스 설정                                                │ │
│  [Unit]                                                              │
│  Description=My Application                                         │
│                                                                     │
│  [Service]                                                           │
│  ExecStart=/usr/bin/myapp                                           │
│  Restart=on-failure                                                  │
│  RestartSec=10s                                                      │
│  # 크래시(교착상태 포함) 시 자동 재시작                                     │ │
│                                                                     │
│  [Install]                                                           │
│  WantedBy=multi-user.target                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 교착상태를 처리하지 않고 무시
• 이유: 발생 빈도 낮음, 처리 비용 > 발생 비용
• 장점: 간단, 오버헤드 없음
• 단점: 멈춤, 데이터 손실, 사용자 개입
• 적용: 일반 PC, 워크스테이션
• 비적용: 실시간 시스템, 미션 크리티컬
```
