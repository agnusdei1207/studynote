# 417. 물리 주소 (Physical Address)

#### 핵심 인사이트 (3줄 요약)

---

## Ⅰ. 개요

### 개념 정의

물리 주소(Physical Address)는 **실제 물리 메모리(RAM)에서의 위치를 나타내는 주소**다. 메모리 컨트롤러가 사용하는 주소다.

### 💡 비유: 지도 좌표
물리 주소는 **지도 좌표**와 같다. GPS 좌표(37.5, 127.0)는 실제 위치다. 집 주소(논리 주소)와 다르다.

### 물리 주소 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                물리 주소 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【물리 메모리 구조】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  물리 메모리 (RAM):                                                     │ │
│  │  ┌─────────────────────────────────────────────────────────────┐│ │
│  │  │ 주소      내용                   용도                          ││ │
│  │  ├─────────────────────────────────────────────────────────────┤│ │
│  │  │ 0x0000    BIOS/UEFI             시스템 시작                   ││ │
│  │  │ 0x1000    커널 코드              운영체제                      ││ │
│  │  │ 0x2000    커널 데이터            운영체제 데이터                ││ │
│  │  │ ...                                                           ││ │
│  │  │ 0x8000    사용자 프로세스 A       응용 프로그램                  ││ │
│  │  │ 0x9000    사용자 프로세스 B       응용 프로그램                  ││ │
│  │  │ ...                                                           ││ │
│  │  │ 0xFFFF   (끝)                    메모리 끝                     ││ │
│  │  └─────────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【논리 → 물리 변환】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  논리 주소: 0x1234                                                    │ │
│  │       │                                                             │ │
│  │       │ MMU 변환                                                      │ │
│  │       ▼                                                             │ │
│  │  물리 주소: 0x5678                                                    │ │
│  │       │                                                             │ │
│  │       │ 메모리 접근                                                    │ │
│  │       ▼                                                             │ │
│  │  ┌─────────────┐                                                   │ │
│  │  │ 물리 메모리   │                                                   │ │
│  │  │  [0x5678]   │                                                   │ │
│  │  └─────────────┘                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【물리 주소 공간】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  물리 주소 공간 크기 = 설치된 RAM 크기                                   │ │
│  │                                                             │ │
│  │  예:                                                                   │ │
│  │  • 4GB RAM → 물리 주소 0x00000000 ~ 0xFFFFFFFF                        │ │
│  │  • 16GB RAM → 물리 주소 0x0 ~ 0x3FFFFFFFF                             │ │
│  │  • 64GB RAM → 물리 주소 0x0 ~ 0xFFFFFFFFF                             │ │
│  │                                                             │ │
│  │  가상 주소 공간 ≥ 물리 주소 공간 (보통)                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                물리 주소 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【물리 주소와 메모리 관리】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  프레임 (Frame):                                                       │ │
│  │  • 물리 메모리의 고정 크기 블록                                          │ │
│  │  • 페이지와 같은 크기 (보통 4KB)                                        │ │
│  │  • 프레임 번호로 식별                                                    │ │
│  │                                                             │ │
│  │  프레임 번호 (Frame Number):                                            │ │
│  │  • 물리 주소 / 페이지 크기                                               │ │
│  │  • 예: 물리 주소 0x5000 → 프레임 번호 5                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【물리 메모리 관리】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  커널의 물리 메모리 관리:                                                │ │
│  │                                                             │ │
│  │  • 프리 리스트: 사용 가능한 프레임 목록                                  │ │
│  │  • 버디 시스템: 크기별 블록 관리                                         │ │
│  │  • 페이지 할당자: 프레임 할당/해제                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【물리 주소와 DMA】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  DMA (Direct Memory Access):                                           │ │
│  │  • CPU 없이 직접 메모리 접근                                             │ │
│  │  • 물리 주소 사용                                                        │ │
│  │  • 장치가 물리 주소로 직접 전송                                          │ │
│  │                                                             │ │
│  │  주의:                                                                 │ │
│  │  • DMA는 물리 주소 사용                                                 │ │
│  │  • 가상 주소 이해 못함                                                   │ │
│  │  • 물리적으로 연속된 버퍼 필요                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【물리 메모리 정보 확인】                                                │
│  ──────────────────                                                  │
│  // 전체 물리 메모리                                                     │
│  $ cat /proc/meminfo | grep MemTotal                                │
│  MemTotal:       16383892 kB  // 16GB                                 │
│                                                                     │
│  // 물리 메모리 상세                                                      │
│  $ cat /proc/iomem | grep "System RAM"                               │
│  00001000-0009ffff : System RAM                                      │
│  00100000-7fffffff : System RAM                                      │
│  // 물리 주소 범위                                                       │
│                                                                     │
│  // 프레임 정보                                                           │
│  $ cat /proc/buddyinfo                                                │
│  Node 0, zone      DMA      0      2      1      1      2 ...         │
│  Node 0, zone    Normal    123    456    789   1234 ...              │
│  // 각 크기별 사용 가능 블록 수                                            │
│                                                                     │
│  【물리 주소 → 가상 주소 (커널)】                                          │
│  ──────────────────                                                  │
│  // 커널은 물리 주소에 직접 매핑 (일부)                                    │
│  // lowmem: 물리 메모리 0~896MB를 커널 가상 주소에 직접 매핑               │
│  // PAGE_OFFSET (0xC0000000) + 물리 주소 = 커널 가상 주소               │
│                                                                     │
│  // 물리 주소 → 가상 주소 매크로                                            │
│  #define __va(x) ((void *)((unsigned long)(x) + PAGE_OFFSET))        │
│  #define __pa(x) ((unsigned long)(x) - PAGE_OFFSET)                  │
│                                                                     │
│  【물리 페이지 상태】                                                      │
│  ──────────────────                                                  │
│  // /proc/kpageflags                                                    │
│  $ page-types -a                                                      │
│  // 각 물리 페이지의 상태 비트                                              │
│  // LOCKED, ERROR, REFERENCED, DIRTY, etc.                            │
│                                                                     │
│  // /proc/kpagecount                                                    │
│  $ cat /proc/kpagecount | head                                        │
│  // 각 물리 페이지의 매핑 수                                               │
│                                                                     │
│  【NUMA 물리 메모리】                                                      │
│  ──────────────────                                                  │
│  // NUMA 노드별 물리 메모리                                               │
│  $ numactl --hardware                                                │
│  available: 2 nodes (0-1)                                            │
│  node 0 size: 8192 MB                                                │
│  node 1 size: 8192 MB                                                │
│  // 각 노드의 물리 메모리 크기                                              │
│                                                                     │
│  【물리 메모리 압력】                                                      │
│  ──────────────────                                                  │
│  // PSI (Pressure Stall Information)                                   │
│  $ cat /proc/pressure/memory                                         │
│  some avg10=0.00 avg60=0.00 avg300=0.00 total=0                      │
│  // 물리 메모리 압력 지표                                                  │
│                                                                     │
│  // 메모리 부족 시                                                         │
│  $ dmesg | grep -i "out of memory"                                   │
│  // OOM 킬러 로그                                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
