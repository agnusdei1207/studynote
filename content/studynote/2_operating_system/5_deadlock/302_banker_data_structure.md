# 은행원 알고리즘의 자료구조 (Data Structures)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 은행원 알고리즘을 수행하기 위해서는 현재 시스템의 자원 가용량, 각 프로세스의 최대 요구량, 현재 할당량, 그리고 추가 필요량을 나타내는 벡터와 행렬 자료구조가 필수적이다.
> 2. **관계**: 핵심은 `Need[i, j] = Max[i, j] - Allocation[i, j]`라는 수식이며, 이 관계를 통해 프로세스가 향후 요청할 수 있는 최대치를 파악한다.
> 3. **가치**: 정형화된 데이터 구조를 통해 자원 할당 상태를 수치화함으로써, 시스템이 수학적으로 명확하게 안전성 여부를 판단할 수 있게 한다.

+++

## Ⅰ. 은행원 알고리즘의 4대 자료구조

### 1. 정의
- $n$개의 프로세스와 $m$개의 자원 유형이 있다고 가정할 때 다음의 구조를 사용한다.

- **Available**: 크기가 $m$인 벡터. 가용한 자원 인스턴스 수.
- **Max**: $n \times m$ 행렬. 각 프로세스가 요청할 수 있는 최대 자원 수.
- **Allocation**: $n \times m$ 행렬. 현재 각 프로세스에 할당된 자원 수.
- **Need**: $n \times m$ 행렬. 각 프로세스가 향후 추가로 요청할 수 있는 자원 수.

- **📢 섹션 요약 비유**: "가계부를 쓸 때 통장 잔액(Available), 사고 싶은 총 목록(Max), 이미 산 것(Allocation), 앞으로 사야 할 것(Need)을 꼼꼼히 적어두는 것과 같습니다."

+++

## Ⅱ. 데이터 구조 간의 관계 및 ASCII 시각화

### 1. 핵심 관계식
- $Need[i][j] = Max[i][j] - Allocation[i][j]$

### 2. ASCII 자료구조 예시
```text
 [ n=5 프로세스, m=3 자원(A,B,C) ]
 
   Allocation       Max          Need       Available
     A B C         A B C        A B C         A B C
 P0  0 1 0         7 5 3        7 4 3         3 3 2
 P1  2 0 0         3 2 2        1 2 2
 P2  3 0 2         9 0 2        6 0 0
 P3  2 1 1         2 2 2        0 1 1
 P4  0 0 2         4 3 3        4 3 1
 
 * Need = Max - Allocation (예: P0의 A자원 Need = 7 - 0 = 7)
```

- **📢 섹션 요약 비유**: "전체 그림을 완성하기 위해 이미 맞춘 퍼즐 조각과 아직 맞추지 못한 빈칸의 개수를 표로 정리해 둔 것과 같습니다."

+++

## Ⅲ. 자료구조별 상세 역할 (Technical Detail)

### 1. Available Vector
- `Available[j] = k`는 자원 $R_j$의 가용 인스턴스가 $k$개임을 의미한다.

### 2. Allocation Matrix
- `Allocation[i, j] = k`는 프로세스 $P_i$가 자원 $R_j$를 $k$개 점유 중임을 의미한다.

### 3. Max Matrix
- `Max[i, j] = k`는 $P_i$가 작업을 완료하기 위해 최대 $k$개의 $R_j$가 필요함을 선언한 것이다.

- **📢 섹션 요약 비유**: "창고지기가 현재 물건이 몇 개 남았는지(Available), 각 부서에서 얼마나 가져갔는지(Allocation), 최종적으로 얼마나 필요하다고 했는지(Max) 장부에 적는 것과 같습니다."

+++

## Ⅳ. 알고리즘 수행 시의 동적 변화 (Strategy)

| 단계 | 자료구조의 변화 |
|:---|:---|
| **자원 요청 승인 시** | Available 감소, Allocation 증가, Need 감소 |
| **자원 반납 시** | Available 증가, Allocation 감소, Need 소멸 |
| **안전성 검사 시** | Work(임시 Available) 변수를 사용하여 시뮬레이션 |

- **📢 섹션 요약 비유**: "물건을 빌려주면 장부의 숫자를 빼고, 물건을 돌려받으면 다시 더하며 창고 상황을 실시간으로 업데이트하는 것과 같습니다."
