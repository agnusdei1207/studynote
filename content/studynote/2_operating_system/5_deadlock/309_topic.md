# 309. 자원 인스턴스 (Resource Instances)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자원 유형의 구체적 단위
> 2. **가치**: 동시성 제어의 세분화
> 3. **융합**: 자원 유형, 자원 할당 그래프와 연관

---

## Ⅰ. 개요

### 개념 정의

자원 인스턴스(Resource Instances)는 **동일한 자원 유형 내의 개별 단위**다. 하나의 자원 유형이 여러 인스턴스를 가질 수 있다.

### 💡 비유: 프린터
자원 인스턴스는 **프린터**와 같다. "프린터"가 자원 유형이고, 3대의 프린터가 각각 하나의 인스턴스다.

### 자원 인스턴스 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 인스턴스 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【개념】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  자원 유형 R                                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  인스턴스 1  인스턴스 2  인스턴스 3  ...  인스턴스 n          │ │ │
│  │  │    ●          ●          ●              ●                │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  예: 프린터 유형 (R1) = {프린터1, 프린터2}                              │ │
│  │      테이프 드라이브 유형 (R2) = {테이프1, 테이프2, 테이프3}              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【자원 할당 그래프 표현】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  □ 자원 유형 R (사각형)                                              │ │
│  │  ● 인스턴스 (사각형 안의 점)                                           │ │
│  │                                                             │ │
│  │       ┌───●───┐                                                     │ │
│  │       │ R ●   │  ← 2개 인스턴스                                       │ │
│  │       └───────┘                                                     │ │
│  │                                                             │ │
│  │  P1 ──요청──► R●1 ──할당──► P2                                       │ │
│  │                    R●2 (가용)                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단일 vs 다중 인스턴스】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단일 인스턴스:                                                        │ │
│  │  • 자원 유형당 1개만 존재                                              │ │
│  │  • 예: 특정 파일, 특정 뮤텍스                                          │ │
│  │  • 순환 = 교착상태 확정                                                │ │
│  │                                                             │ │
│  │  다중 인스턴스:                                                        │ │
│  │  • 자원 유형당 여러 개 존재                                             │ │
│  │  • 예: 프린터 3대, 메모리 페이지                                        │ │
│  │  • 순환이 있어도 교착상태 아닐 수 있음                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 인스턴스 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인스턴스 수와 교착상태】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단일 인스턴스 자원:                                                   │ │
│  │  • 자원 할당 그래프에서 순환 = 교착상태                                 │ │
│  │  • 분석이 단순                                                        │ │
│  │                                                             │ │
│  │  다중 인스턴스 자원:                                                   │ │
│  │  • 순환이 있어도 다른 인스턴스 사용 가능                                 │ │
│  │  • 은행원 알고리즘 필요                                                 │ │
│  │  • 분석이 복잡                                                        │ │
│  │                                                             │ │
│  │  예시:                                                                 │
│  │  R1: 2개 인스턴스 (R1a, R1b)                                         │
│  │  P1 → R1a 보유, R1b 요청                                             │
│  │  P2 → R1b 보유, R1a 요청                                             │
│  │  → 순환처럼 보이지만 다른 인스턴스 사용 가능                            │ │
│  │  → 교착상태 아님!                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인스턴스 상태】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  각 인스턴스는 다음 상태 중 하나:                                       │ │
│  │                                                             │ │
│  │  • Free (가용): 어떤 프로세스도 사용하지 않음                           │ │
│  │  • Allocated (할당됨): 특정 프로세스에 할당                             │ │
│  │                                                             │ │
│  │  인스턴스별로 독립적 상태 관리                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실제 예시】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  자원 유형              인스턴스 수      단위                        │ │
│  │  ────────               ──────────        ────                       │ │
│  │  CPU 코어               4-64개           코어                        │ │
│  │  메모리                 수백만            페이지 (4KB)                 │ │
│  │  프린터                 1-10대            대                          │ │
│  │  디스크                 1-10개            개                          │ │
│  │  네트워크 포트           65535개          포트                        │ │
│  │  DB 연결                10-1000개        연결                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【자원 인스턴스 관리】                                                 │
│  ──────────────────                                                  │
│  class ResourceType {                                                │
│      String name;                                                     │
│      int totalInstances;                                              │
│      List<ResourceInstance> instances;                               │
│  }                                                                  │
│                                                                     │
│  class ResourceInstance {                                            │
│      int id;                                                          │
│      ResourceType type;                                              │
│      Process owner;  // null이면 가용                                  │
│      boolean isFree() { return owner == null; }                      │
│  }                                                                  │
│                                                                     │
│  【인스턴스 할당】                                                      │
│  ──────────────────                                                  │
│  class ResourceManager {                                            │
│      Map<ResourceType, List<ResourceInstance>> resources;            │
│                                                                     │
│      ResourceInstance allocate(ResourceType type, Process p) {        │
│          List<ResourceInstance> instances = resources.get(type);     │
│          for (ResourceInstance inst : instances) {                   │
│              if (inst.isFree()) {                                    │
│                  inst.owner = p;                                     │
│                  return inst;                                        │
│              }                                                      │
│          }                                                          │
│          return null;  // 가용 인스턴스 없음                            │
│      }                                                              │
│                                                                     │
│      void release(ResourceInstance inst) {                           │
│          inst.owner = null;                                          │
│      }                                                              │
│                                                                     │
│      int getAvailableCount(ResourceType type) {                      │
│          return (int) resources.get(type).stream()                   │
│              .filter(ResourceInstance::isFree)                       │
│              .count();                                                │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【데이터베이스 연결 풀】                                                │
│  ──────────────────                                                  │
│  // 연결 유형                                                          │
│  // Resource Type: DatabaseConnection                                │
│  // Instances: 20개의 연결                                             │
│                                                                     │
│  HikariConfig config = new HikariConfig();                           │
│  config.setMaximumPoolSize(20);  // 20개 인스턴스                      │
│  config.setMinimumIdle(5);       // 최소 5개 유지                      │
│                                                                     │
│  HikariDataSource ds = new HikariDataSource(config);                │
│                                                                     │
│  // 인스턴스 획득                                                       │
│  Connection conn = ds.getConnection();  // 가용 인스턴스 할당           │
│  // 사용                                                               │
│  conn.close();  // 인스턴스 반납                                        │
│                                                                     │
│  【스레드 풀】                                                           │
│  ──────────────────                                                  │
│  // Resource Type: WorkerThread                                      │
│  // Instances: 10개의 스레드                                            │
│                                                                     │
│  ExecutorService executor = Executors.newFixedThreadPool(10);        │
│                                                                     │
│  // 인스턴스 획득 (자동)                                                 │
│  executor.submit(() -> {                                             │
│      // 작업                                                           │
│  });  // 작업 완료 시 스레드 반환                                         │
│                                                                     │
│  【메모리 페이지】                                                       │
│  ──────────────────                                                  │
│  // Resource Type: MemoryPage (4KB)                                  │
│  // Instances: 수백만 개                                                │
│                                                                     │
│  // 페이지 할당                                                         │
│  void* page = mmap(NULL, 4096, PROT_READ | PROT_WRITE,              │
│                     MAP_PRIVATE | MAP_ANONYMOUS, -1, 0);             │
│  // 사용                                                               │
│  munmap(page, 4096);  // 페이지 반납                                    │
│                                                                     │
│  【인스턴스 모니터링】                                                   │
│  ──────────────────                                                  │
│  // DB 연결 풀                                                         │
│  HikariPoolMXBean pool = ds.getHikariPoolMXBean();                  │
│  System.out.println("Active: " + pool.getActiveConnections());       │
│  System.out.println("Idle: " + pool.getIdleConnections());           │
│  System.out.println("Total: " + pool.getTotalConnections());         │
│                                                                     │
│  // 스레드 풀                                                           │
│  ThreadPoolExecutor tpe = (ThreadPoolExecutor) executor;             │
│  System.out.println("Active: " + tpe.getActiveCount());              │
│  System.out.println("Pool size: " + tpe.getPoolSize());              │
│  System.out.println("Max: " + tpe.getMaximumPoolSize());             │
│                                                                     │
│  【성능 고려사항】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  인스턴스 수        대기 시간      처리량       리소스 사용             │ │
│  │  ──────────        ──────        ──────       ──────────           │ │
│  │  너무 적음           높음         낮음         효율적                  │ │
│  │  적절함              낮음         높음         효율적                  │ │
│  │  너무 많음           매우 낮음     최대         낭비                   │ │
│  │                                                             │ │
│  │  최적화: 부하에 따라 인스턴스 수 동적 조정                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
