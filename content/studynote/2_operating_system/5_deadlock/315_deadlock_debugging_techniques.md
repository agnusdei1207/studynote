# 동기화 결함 (순환 의존성) 코드 레벨 디버깅 기법

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동기화 결함 디버깅은 소스 코드 내에서 자원 획득(Lock Acquisition) 순서가 꼬여 발생하는 순환 의존성(Circular Dependency)을 추적하고 가시화하는 과정이다.
> 2. **도구**: 스레드 덤프(Thread Dump) 분석, 런타임 락 그래프 생성, 정적 코드 분석(Static Analysis) 도구를 활용하여 데드락 지점을 정밀 타격한다.
> 3. **가치**: 간헐적으로 발생하는 재현 불가능한 '하이젠버그(Heisenbug)' 성격의 버그를 논리적 분석을 통해 확정적 결함으로 식별하고 수정할 수 있게 한다.

+++

## Ⅰ. 디버깅의 첫걸음: 상태 파악

### 1. 전조 증상 식별
- CPU 사용률 0% 또는 100% 상태에서의 무응답.
- 특정 함수 호출 시점에서 멈추는 현상(Hang).

### 2. 환경 고립 (Environment Isolation)
- 최소한의 스레드와 자원만 사용하는 환경에서 데드락 상황을 재현한다.

- **📢 섹션 요약 비유**: "몸이 아플 때 어느 부위가 어떻게 아픈지 먼저 증상을 살피고 병원에 가는 것과 같습니다."

+++

## Ⅱ. 디버깅 매핑 기술 (ASCII Diagram)

### 1. 스레드 덤프 분석 (Thread Dump Analysis)
```text
 [ Thread Stack Trace Example ]
 
 Thread-1 (P1):
  - State: BLOCKED (on <0x01>)
  - at classA.method1 (waiting to lock <0x01>)
  - locked <0x02>
  
 Thread-2 (P2):
  - State: BLOCKED (on <0x02>)
  - at classB.method2 (waiting to lock <0x02>)
  - locked <0x01>
  
  Conclusion: P1(Lock 2, Want 1) <--> P2(Lock 1, Want 2) => CYCLE!
```

### 2. 락 그래프 가시화
- 수천 줄의 코드 사이에서 '누가 누구를 기다리는지' 화살표를 그려본다.

- **📢 섹션 요약 비유**: "누가 누구에게 돈을 빌렸는지 복잡한 장부를 보고 화살표로 돈의 흐름을 그려서 범인을 찾는 것과 같습니다."

+++

## Ⅲ. 실전 디버깅 기법 상세

### 1. 정적 분석 (Static Code Analysis)
- `Clang-Tidy`, `FindBugs` 등의 도구로 락 획득 순서의 모순을 미리 찾아낸다.

### 2. 런타임 추적 (Instrumentation)
- 모든 `lock()`과 `unlock()` 호출 시점에 로그를 남겨 자원 흐름을 기록한다.

### 3. 코어 덤프 분석 (Core Dump Analysis)
- 시스템이 멈춘 시점의 메모리 스냅샷을 찍어 각 스레드가 어떤 명령어를 수행 중이었는지 사후 분석한다.

- **📢 섹션 요약 비유**: "CCTV를 돌려보며 사고 직전에 누가 어디에 있었는지 확인하는 것과 같습니다."

+++

## Ⅳ. 디버깅 도구 비교 및 선택

| 도구 유형 | 대표 예시 | 주요 특징 | 추천 시나리오 |
|:---|:---|:---|:---|
| **언어 내장** | `jstack`, `pstack` | 추가 설치 없이 즉시 사용 | 서버 멈춤 현상 발생 시 |
| **동적 분석** | `Valgrind (Helgrind)` | 메모리 및 락 오류 정밀 탐지 | 개발 단계 성능 최적화 |
| **운영체제 레벨** | `DTrace`, `eBPF` | 커널 수준의 이벤트 추적 | 저수준 시스템 프로그래밍 |

- **📢 섹션 요약 비유**: "간단한 수리는 펜치로 하고, 정밀한 수리는 현미경으로 하는 것처럼 상황에 맞는 도구를 써야 합니다."

+++

## Ⅴ. 결론 및 실무적 교훈

### 1. 요약
- 코드 레벨 디버깅은 복잡한 락의 얽힘을 논리적인 선형 흐름으로 풀어내는 '추리 작업'이다.

### 2. 최종 결론
- 데드락은 발생한 후에 고치는 것보다, 디버깅 과정에서 얻은 교훈을 바탕으로 '락 오더링(Lock Ordering)' 규칙을 세워 예방하는 것이 훨씬 경제적이다.

- **📢 섹션 요약 비유**: "한 번 엉킨 실타래를 푸는 법을 배우면, 다음부터는 실이 안 엉키게 감는 법을 알게 된다는 교훈을 줍니다."

+++

### 📌 관련 개념 맵 (Knowledge Graph)
- **스레드 덤프 (Thread Dump)**: 디버깅의 가장 핵심적인 데이터 소스.
- **락 오더링 (Lock Ordering)**: 디버깅을 통해 도달해야 할 예방 규칙.
- **Heisenbug (하이젠버그)**: 관찰 시점에 사라지는 고난도 동기화 버그.
