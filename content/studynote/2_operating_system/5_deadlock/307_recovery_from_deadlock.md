# 307. 재사용 자원 (Reusable Resources)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 반납 후 다른 프로세스가 재사용 가능한 자원
> 2. **가치**: 교착상태의 주요 대상
> 3. **융합**: 소모성 자원, 자원 인스턴스와 연관

---

## Ⅰ. 개요

### 개념 정의

재사용 자원(Reusable Resources)은 **한 프로세스가 사용 후 반납하면 다른 프로세스가 재사용할 수 있는 자원**이다. 시스템 내에 고정된 개수로 존재한다.

### 💡 비유: 도서관 책
재사용 자원은 **도서관 책**과 같다. 한 사람이 읽고 반납하면 다른 사람이 빌릴 수 있다. 책의 개수는 정해져 있다.

### 재사용 자원 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                재사용 자원 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【특성】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 고정된 개수: 시스템 내에 일정 수 존재                                │ │
│  │  2. 반납 가능: 사용 후 반드시 반납해야 함                                │ │
│  │  3. 배타적 사용: 한 번에 한 프로세스만 사용                               │ │
│  │  4. 교착상태 가능: 여러 프로세스가 동시 요청 시                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대표적 재사용 자원】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  하드웨어:                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • CPU: 시간 슬라이스 단위 할당                              │ │ │
│  │  │  • 메모리 (RAM): 페이지/세그먼트 단위                         │ │ │
│  │  │  • 디스크: 블록 단위                                        │ │ │
│  │  │  • 프린터: 전체 또는 페이지 단위                              │ │ │
│  │  │  • 테이프 드라이브: 전체 장치                                 │ │ │
│  │  │  • 네트워크 카드: 대역폭 단위                                 │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  소프트웨어:                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 파일: 읽기/쓰기 잠금                                     │ │ │
│  │  │  • 뮤텍스/세마포어: 동기화 객체                              │ │ │
│  │  │  • 데이터베이스 연결: 연결 풀                                │ │ │
│  │  │  • 소켓: 네트워크 연결                                       │ │ │
│  │  │  • 스레드: 스레드 풀                                         │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │

│  【교착상태와의 관계】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  재사용 자원에서 교착상태가 더 자주 발생:                                │ │
│  │                                                             │ │
│  │  이유:                                                               │ │
│  │  1. 고정된 개수 → 경합 발생                                           │ │
│  │  2. 배타적 사용 → 대기 필요                                           │ │
│  │  3. 보유 및 대기 → 교착상태 조건                                       │ │
│  │                                                             │ │
│  │  전형적 시나리오:                                                       │ │
│  │  • P1이 R1 보유, R2 요청                                              │ │
│  │  • P2가 R2 보유, R1 요청                                              │ │
│  │  → 교착상태!                                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【할당 단위】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  자원별 할당 단위:                                                     │ │
│  │  • CPU: 시간 슬라이스 (ms)                                            │ │
│  │  • 메모리: 페이지 (4KB) 또는 KB 단위                                   │ │
│  │  • 디스크: 블록 (4KB)                                                 │ │
│  │  • 파일: 전체 또는 레코드                                              │ │
│  │  • 락: 전체 (나눌 수 없음)                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 소모성 자원】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  재사용 자원:                                                          │ │
│  │  • 사용 후 반납                                                       │ │
│  │  • 개수 고정                                                          │ │
│  │  • 교착상태 분석 용이                                                  │ │
│  │                                                             │ │
│  │  소모성 자원:                                                           │ │
│  │  • 사용 후 소멸                                                       │ │
│  │  • 생성 가능                                                          │ │
│  │  • 교착상태 분석 복잡                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 재사용】                                                      │
│  ──────────────────                                                  │
│  // C 메모리 할당/해제                                                  │
│  void* ptr = malloc(1024);  // 1KB 할당                               │
│  // 사용                                                               │
│  free(ptr);                  // 반납 → 재사용 가능                       │
│                                                                     │
│  // Java 가비지 컬렉션                                                  │
│  byte[] data = new byte[1024];                                       │
│  data = null;  // 참조 해제 → GC가 수거 후 재사용                        │
│                                                                     │
│  【파일 재사용】                                                         │
│  ──────────────────                                                  │
│  // C 파일 처리                                                        │
│  FILE* fp = fopen("data.txt", "r");                                  │
│  // 읽기                                                               │
│  fclose(fp);  // 반납 → 다른 프로세스가 열 수 있음                         │
│                                                                     │
│  // Java try-with-resources                                           │
│  try (FileInputStream fis = new FileInputStream("data.txt")) {       │
│      // 읽기                                                           │
│  }  // 자동 close() → 반납                                              │
│                                                                     │
│  【데이터베이스 연결 재사용】                                             │
│  ──────────────────                                                  │
│  // 연결 풀                                                            │
│  HikariDataSource pool = new HikariDataSource();                     │
│  pool.setMaximumPoolSize(10);  // 최대 10개 재사용                      │
│                                                                     │
│  Connection conn = pool.getConnection();  // 대기 가능                 │
│  // 사용                                                               │
│  conn.close();  // 풀로 반환 → 재사용                                    │
│                                                                     │
│  【락 재사용】                                                            │
│  ──────────────────                                                  │
│  // ReentrantLock                                                     │
│  ReentrantLock lock = new ReentrantLock();                           │
│                                                                     │
│  lock.lock();                                                         │
│  // 임계 영역                                                           │
│  lock.unlock();  // 반납 → 다른 스레드가 획득 가능                         │
│                                                                     │
│  // synchronized                                                      │
│  synchronized (lockObject) {                                          │
│      // 임계 영역                                                       │
│  }  // 자동 해제 → 재사용                                                │
│                                                                     │
│  【자원 누수 방지】                                                      │
│  ──────────────────                                                  │
│  // RAII 패턴 (C++)                                                    │
│  class ScopedLock {                                                   │
│      std::mutex& mtx;                                                │
│  public:                                                            │
│      ScopedLock(std::mutex& m) : mtx(m) { mtx.lock(); }              │
│      ~ScopedLock() { mtx.unlock(); }  // 자동 해제                     │
│  };                                                                 │
│                                                                     │
│  void safeFunction() {                                               │
│      ScopedLock lock(mutex);  // 생성 시 lock                         │
│      // 작업                                                           │
│  }  // 소멸 시 자동 unlock → 재사용 보장                                  │
│                                                                     │
│  【자원 모니터링】                                                        │
│  ──────────────────                                                  │
│  // Linux: 열린 파일 확인                                               │
│  lsof -p <pid> | wc -l                                               │
│                                                                     │
│  // Java: 메모리 사용                                                   │
│  Runtime runtime = Runtime.getRuntime();                             │
│  long used = runtime.totalMemory() - runtime.freeMemory();           │
│                                                                     │
│  // DB 연결 풀 모니터링                                                  │
│  HikariPoolMXBean poolProxy = pool.getHikariPoolMXBean();            │
│  int active = poolProxy.getActiveConnections();                      │
│  int idle = poolProxy.getIdleConnections();                          │
│  int waiting = poolProxy.getThreadsAwaitingConnection();             │
│                                                                     │
│  【재사용 자원 관리 모범 사례】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 항상 반납 (close, free, unlock)                                  │ │
│  │  2. RAII/try-with-resources 사용                                    │ │
│  │  3. 타임아웃 설정으로 무한 대기 방지                                    │ │
│  │  4. 자원 사용량 모니터링                                               │ │
│  │  5. 누수 감지 도구 사용 (valgrind, JProfiler)                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 반납 후 재사용 가능한 자원
• 특성: 고정 개수, 배타적 사용, 반납 필요
• 예시: CPU, 메모리, 파일, 락, DB 연결
• 교착상태: 재사용 자원에서 빈번
• 관리: 반납 필수, RAII 패턴
• 모니터링: 누수 감지, 사용량 추적
```
