# 308. 소모성 자원 (Consumable Resources)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용하면 사라지는 자원
> 2. **가치**: 생산자-소비자 패턴의 기반
> 3. **융합**: 재사용 자원, 신호와 연관

---

## Ⅰ. 개요

### 개념 정의

소모성 자원(Consumable Resources)은 **사용하면 소멸되어 더 이상 사용할 수 없는 자원**이다. 생산자가 생성하고 소비자가 소비한다.

### 💡 비유: 음식
소모성 자원은 **음식**과 같다. 먹으면 사라진다. 계속 요리하면(생산) 계속 먹을 수 있지만, 한 번 먹은 음식은 사라진다.

### 소모성 자원 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                소모성 자원 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【특성】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 소멸성: 사용하면 사라짐                                             │ │
│  │  2. 생성 가능: 생산자에 의해 무한히 생성 가능 (이론적)                     │ │
│  │  3. 반납 없음: 사용 후 반납하지 않음                                     │ │
│  │  4. 교착상태 가능: 복잡한 분석 필요                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대표적 소모성 자원】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  • 신호 (Signals/Interrupts):                                         │ │
│  │    - 프로세스 간 통신                                                   │ │
│  │    - 한 번 처리하면 사라짐                                              │ │
│  │                                                             │ │
│  │  • 메시지 (Messages):                                                  │ │
│  │    - 메시지 큐의 메시지                                                 │ │
│  │    - 수신하면 큐에서 제거                                               │ │
│  │                                                             │ │
│  │  • 이벤트 (Events):                                                    │ │
│  │    - 비동기 이벤트                                                     │ │
│  │    - 처리 후 소멸                                                      │ │
│  │                                                             │ │
│  │  • 버퍼의 데이터:                                                      │ │
│  │    - 읽으면 제거                                                       │ │
│  │    - 파이프, 소켓 데이터                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【생산자-소비자 모델】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  [생산자] ──생산──► [소모성 자원] ──소비──► [소비자]                       │ │
│  │                           │                                         │ │
│  │                           ▼                                         │ │
│  │                        소멸                                          │ │
│  │                                                             │ │
│  │  예: 생산자가 메시지 생성 → 큐에 저장 → 소비자가 읽음 → 소멸             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                소모성 자원 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【교착상태와의 관계】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  소모성 자원에서도 교착상태 가능:                                       │ │
│  │                                                             │ │
│  │  시나리오:                                                               │
│  │  • P1이 메시지 M1 생성 → P2가 M1 소비 대기                            │ │
│  │  • P2가 메시지 M2 생성 → P1이 M2 소비 대기                            │ │
│  │  → 서로가 서로의 메시지 대기 = 교착상태                                 │ │
│  │                                                             │ │
│  │  특징:                                                                   │
│  │  • 재사용 자원보다 분석 어려움                                          │ │
│  │  • 생성 가능성 때문에 최대치 예측 불가                                  │ │
│  │  • 은행원 알고리즘 적용 불가                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 재사용 자원】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성            재사용 자원           소모성 자원                   │ │
│  │  ────────        ────────────          ────────────               │ │
│  │  개수            고정                   무제한 (생산 가능)            │ │
│  │  사용 후         반납                   소멸                        │ │
│  │  반납            필수                   불필요                      │ │
│  │  교착상태 분석    상대적으로 쉬움         어려움                     │ │
│  │  예시            메모리, 파일            신호, 메시지                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【교착상태 예방】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  소모성 자원에서는 전통적 예방 방법 적용 어려움:                         │ │
│  │                                                             │ │
│  │  대안:                                                                   │
│  │  • 타임아웃: 무한 대기 방지                                             │ │
│  │  • 순환 감지: 대기 그래프 분석                                          │ │
│  │  • 설계 패턴: 단방향 통신만 허용                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메시지 큐 (소모성)】                                                 │
│  ──────────────────                                                  │
│  // POSIX 메시지 큐                                                    │
│  mqd_t mq = mq_open("/myqueue", O_CREAT | O_RDWR, 0644, NULL);      │
│                                                                     │
│  // 생산자                                                             │
│  mq_send(mq, "Hello", 6, 0);  // 메시지 생성                           │
│                                                                     │
│  // 소비자                                                             │
│  char buffer[256];                                                   │
│  mq_receive(mq, buffer, 256, NULL);  // 메시지 소비 → 소멸             │
│                                                                     │
│  【신호 (소모성)】                                                       │
│  ──────────────────                                                  │
│  // Unix 신호                                                         │
│  kill(pid, SIGUSR1);  // 신호 전송 (생산)                               │
│                                                                     │
│  // 신호 처리                                                          │
│  void handler(int sig) {                                              │
│      // 신호 소비 → 처리 후 소멸                                         │
│  }                                                                  │
│  signal(SIGUSR1, handler);                                           │
│                                                                     │
│  【파이프 (소모성)】                                                     │
│  ──────────────────                                                  │
│  int pipefd[2];                                                       │
│  pipe(pipefd);                                                        │
│                                                                     │
│  // 생산자                                                             │
│  write(pipefd[1], "data", 4);  // 데이터 생성                          │
│                                                                     │
│  // 소비자                                                             │
│  char buffer[4];                                                      │
│  read(pipefd[0], buffer, 4);  // 데이터 소비 → 소멸                     │
│                                                                     │
│  【이벤트 (소모성)】                                                      │
│  ──────────────────                                                  │
│  // JavaScript 이벤트                                                  │
│  EventEmitter emitter = new EventEmitter();                          │
│                                                                     │
│  emitter.emit('message', 'Hello');  // 이벤트 생성                     │
│  emitter.on('message', (data) => {                                   │
│      // 이벤트 소비 → 처리 후 소멸                                        │
│  });                                                                 │
│                                                                     │
│  【Kafka 메시지 (소모성)】                                               │
│  ──────────────────                                                  │
│  // 생산자                                                             │
│  Producer<String, String> producer = new KafkaProducer<>(props);    │
│  producer.send(new ProducerRecord<>("topic", "key", "value"));      │
│                                                                     │
│  // 소비자                                                             │
│  Consumer<String, String> consumer = new KafkaConsumer<>(props);    │
│  consumer.subscribe(Arrays.asList("topic"));                         │
│  ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));│
│  for (ConsumerRecord<String, String> record : records) {             │
│      // 메시지 소비 → 오프셋 커밋 시 재소비 불가                            │
│  }                                                                  │
│                                                                     │
│  【교착상태 예방: 설계 패턴】                                             │
│  ──────────────────                                                  │
│  // 나쁜 예: 양방향 대기                                                 │
│  void bad() {                                                        │
│      // P1: wait message from P2                                    │
│      // P2: wait message from P1                                    │
│      // → 교착상태 가능!                                                │
│  }                                                                  │
│                                                                     │
│  // 좋은 예: 단방향 + 타임아웃                                            │
│  void good() {                                                       │
│      // P1 → P2로만 메시지 전송                                         │
│      // P2는 응답하지 않음 (새로운 메시지로만)                             │
│      // 타임아웃으로 무한 대기 방지                                        │
│  }                                                                  │
│                                                                     │
│  【모니터링】                                                            │
│  ──────────────────                                                  │
│  // 메시지 큐 상태                                                       │
│  mq_getattr(mq, &attr);                                              │
│  printf("Messages in queue: %ld\n", attr.mq_curmsgs);               │
│                                                                     │
│  // 이벤트 루프 상태                                                     │
│  // Node.js: 이벤트 루프 지연 측정                                       │
│  const { performance } = require('perf_hooks');                      │
│  const start = performance.now();                                    │
│  setImmediate(() => {                                                │
│      const delay = performance.now() - start;                       │
│      console.log(`Event loop delay: ${delay}ms`);                   │
│  });                                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 사용하면 사라지는 자원
• 특성: 소멸성, 생성 가능, 반납 없음
• 예시: 신호, 메시지, 이벤트, 파이프 데이터
• 모델: 생산자-소비자
• 교착상태: 분석 어려움, 타임아웃으로 예방
• 설계: 단방향 통신 권장
```
