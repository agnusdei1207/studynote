# 희생자 선택 (Victim Selection) 최소 비용 기준

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 희생자 선택(Victim Selection) 최소 비용 기준은 교착 상태 해소를 위해 중단시킬 프로세스를 결정할 때, 시스템의 전체적인 처리량 저하와 자원 낭비를 수치화하여 가장 손실이 적은 대상을 고르는 전략이다.
> 2. **메커니즘**: 프로세스의 우선순위, 실행 경과 시간, 남은 작업 시간, 점유 자원 수 등을 복합적으로 고려하여 비용 함수(Cost Function)를 산출한다.
> 3. **목표**: 단순히 한 프로세스를 종료하는 것이 아니라, "가장 적은 대가로 시스템을 정상화"하는 경제적 최적화(Economic Optimization)를 실현하는 데 있다.

+++

## Ⅰ. 최소 비용 선택의 기본 원칙

### 1. 경제적 합리성 (Economic Rationality)
- 교착 상태를 풀기 위해 지불해야 하는 '수정 비용(Correction Cost)'을 최소화한다.
- 다시 시작했을 때 처음부터 다시 해야 하는 작업의 양을 고려한다.

### 2. 다각적 지표 분석
- 단일 지표가 아닌 다차원적인 지표를 결합하여 판단한다.

- **📢 섹션 요약 비유**: "이사 갈 때 버릴 물건을 정하면서, 산 지 얼마 안 된 비싼 가구는 남기고 오래된 낡은 의자를 먼저 버리는 것과 같습니다."

+++

## Ⅱ. 비용 산정 매핑 (ASCII Diagram)

### 1. 비용 함수 구조 (Cost Function Structure)
```text
 [ Cost Function Calculation ]
 
   Variable (V)      Impact (W)      Calculated Score
  ----------------------------------------------------
  Priority (P)       High (x10)  --> [ P * 10 ]
  Time Processed(T1) Med  (x5)   --> [ T1 * 5 ]
  Resources Held(R)  High (x8)   --> [ R * 8 ]
  Type (Batch/Int)   Low  (x3)   --> [ Type * 3 ]
  
  Total Cost = Sum(Scores)
  Decision: MIN(Total Cost) is the Victim!
```

### 2. 프로세스 상태별 비용 매핑
- **신규 프로세스**: 누적 작업량이 적으므로 비용이 낮음 (희생 1순위).
- **완료 직전 프로세스**: 거의 다 끝난 작업을 버리는 것은 낭비이므로 비용이 높음 (보호 대상).

- **📢 섹션 요약 비유**: "가게 메뉴판 옆에 원가와 조리 시간을 적어두고, 재료가 떨어졌을 때 어떤 메뉴를 품절 처리할지 계산기로 두드려보는 것과 같습니다."

+++

## Ⅲ. 주요 결정 요인 (Decision Factors)

### 1. 자원 점유의 희귀성
- 흔한 자원(CPU Time)보다는 구하기 힘든 자원(Specialized I/O)을 가진 프로세스를 종료할 때 해소 효과가 크다.

### 2. 프로세스 성격 (Process Type)
- **대화형(Interactive)**: 사용자 경험에 직결되므로 종료 비용이 매우 높음.
- **일괄 처리(Batch)**: 나중에 다시 돌려도 큰 지장이 없으므로 상대적으로 비용이 낮음.

- **📢 섹션 요약 비유**: "손님과 대화 중인 점원을 퇴근시키는 것보다, 창고에서 재고 정리 중인 점원을 퇴근시키는 게 서비스에 지장이 적은 것과 같습니다."

+++

## Ⅳ. 비용 기준의 한계와 실무적 고려

| 항목 | 상세 설명 | 대응 방안 |
|:---|:---|:---|
| **정적 비용의 위험** | 고정된 가중치는 상황 변화에 취약함 | 시스템 부하에 따른 가변 가중치 적용 |
| **중복 종료 (Over-termination)** | 너무 많은 프로세스를 한꺼번에 죽임 | 한 번에 하나씩 죽이고 상태 재확인 |
| **정보 부족** | 프로세스의 남은 시간을 정확히 알기 어려움 | 과거 실행 데이터를 기반으로 예측치 사용 |

- **📢 섹션 요약 비유**: "물건 가격만 보고 버렸는데, 알고 보니 그게 단종된 한정판이라 나중에 더 큰 손해를 보는 일을 방지하기 위해 꼼꼼히 체크해야 한다는 뜻입니다."

+++

## Ⅴ. 결론 및 향후 과제

### 1. 요약
- 최소 비용 기준은 '무엇을 죽일 것인가'를 '얼마만큼의 손해를 감수할 것인가'의 문제로 치환하여 해결한다.

### 2. 최종 결론
- 효율적인 교착 상태 복구는 단순히 알고리즘의 정확성뿐만 아니라, 시스템의 비즈니스 가치를 얼마나 잘 반영하는 정책(Policy)을 수립하느냐에 달려 있다.

- **📢 섹션 요약 비유**: "가장 적게 잃고 가장 빨리 일어나는 것이 진정한 복구의 핵심이라는 교훈을 남깁니다."
