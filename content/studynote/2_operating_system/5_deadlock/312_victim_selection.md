# 312. 자원 해제 (Resource Release)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 할당된 자원을 시스템에 반환
> 2. **가치**: 대기자의 기회, 교착상태 해소
> 3. **융합**: 자원 할당, 대기 큐와 연관

---

## Ⅰ. 개요

### 개념 정의

자원 해제(Resource Release)은 **프로세스가 사용을 마친 자원을 시스템에 반환하는 행위**다. 해제된 자원은 대기 중인 다른 프로세스에 할당될 수 있다.

### 💡 비유: 도서관 반납
자원 해제는 **도서관 반납**과 같다. 책을 다 읽으면 반납한다. 그러면 기다리던 다른 사람이 빌릴 수 있다.

### 자원 해제 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 해제 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【해제 전후 상태】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  해제 전:                                                            │ │
│  │  ┌───────────────────────────────────────────────────────────┐│ │
│  │  │  P1이 R1: 2개, R2: 1개 보유                                  ││ │
│  │  │  Available: [R1: 1, R2: 1]                                 ││ │
│  │  │  Wait Queue: [P3(R1), P4(R2)]                               ││ │
│  │  └───────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  │  P1이 R1: 2개, R2: 1개 해제:                                        │ │
│  │  ┌───────────────────────────────────────────────────────────┐│ │
│  │  │  P1이 모든 자원 반납                                            ││ │
│  │  │  Available: [R1: 3, R2: 2]                                 ││ │
│  │  │  Wait Queue: []  // P3, P4 할당 가능                          ││ │
│  │  └───────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해제 프로세스】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 해제 요청:                                                       │ │
│  │     release(pid, resourceType, count)                               │ │
│  │                                                             │ │
│  │  2. 할당 테이블 갱신:                                                 │ │
│  │     allocation[pid][resourceType] -= count                          │ │
│  │     available[resourceType] += count                                │ │
│  │                                                             │ │
│  │  3. 대기 큐 확인:                                                     │ │
│  │     for each waiter in waitQueue:                                   │ │
│  │         if waiter.need <= available:                                │ │
│  │             allocate(waiter)                                        │ │
│  │             remove from waitQueue                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대기자 깨우기】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  해제 전:                     해제 후:                                │ │
│  │  ┌───────────┐              ┌───────────┐                        │ │
│  │  │ P2 대기중   │              │ P2 실행중   │                        │ │
│  │  │ (R1 요청)   │     해제      │ (R1 획득)   │                        │ │
│  │  └───────────┘   ────►     └───────────┘                        │ │
│  │        ▲                                                             │ │
│  │        │                                                             │ │
│  │   [P1] ─── R1 해제                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 해제 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【해제 시점】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 정상 완료:                                                       │ │
│  │     • 작업 완료 후 자발적 해제                                          │ │
│  │                                                             │ │
│  │  2. 명시적 해제:                                                      │ │
│  │     • unlock(), close(), free() 호출                                  │ │
│  │                                                             │ │
│  │  3. 프로세스 종료:                                                     │ │
│  │     • 프로세스가 종료되면 모든 자원 해제                                   │ │
│  │                                                             │ │
│  │  4. 강제 해제:                                                        │ │
│  │     • 타임아웃, 교착상태 복구 시                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해제와 교착상태】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  교착상태에 있는 프로세스 중 하나가 해제하면:                              │ │
│  │  • 교착상태 해소                                                       │ │
│  │  • 다른 프로세스 진행 가능                                              │ │
│  │                                                             │ │
│  │  이것이 교착상태 복구의 기본 원리                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【부분 vs 전체 해제】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  부분 해제:                                                            │ │
│  │  • 일부 자원만 해제                                                    │ │
│  │  • 계속 작업 가능                                                       │ │
│  │                                                             │ │
│  │  전체 해제:                                                            │ │
│  │  • 모든 자원 해제                                                      │ │
│  │  • 프로세스 종료 또는 대기                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해제 순서】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  락 순서 규칙을 따르는 경우:                                             │ │
│  │  • 획득의 역순으로 해제                                                 │ │
│  │  • 마지막에 획득한 락을 먼저 해제                                         │ │
│  │                                                             │ │
│  │  예: lock(A), lock(B), lock(C)                                      │ │
│  │      unlock(C), unlock(B), unlock(A)                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락 해제】                                                              │
│  ──────────────────                                                  │
│  // C++ mutex                                                         │
│  std::mutex mtx;                                                      │
│  mtx.lock();                                                          │
│  // ... 작업 ...                                                       │
│  mtx.unlock();  // 해제                                                 │
│                                                                     │
│  // RAII 패턴 (자동 해제)                                               │
│  {                                                                     │
│      std::lock_guard<std::mutex> lock(mtx);                          │
│      // ... 작업 ...                                                   │
│  }  // 자동 해제                                                         │
│                                                                     │
│  // Java synchronized (자동 해제)                                      │
│  synchronized(obj) {                                                 │
│      // ... 작업 ...                                                   │
│  }  // 자동 해제                                                         │
│                                                                     │
│  // ReentrantLock                                                      │
│  lock.lock();                                                         │
│  try {                                                                 │
│      // ... 작업 ...                                                   │
│  } finally {                                                          │
│      lock.unlock();  // 해제                                            │
│  }                                                                  │
│                                                                     │
│  【메모리 해제】                                                         │
│  ──────────────────                                                  │
│  // C                                                                  │
│  void* p = malloc(1024);                                              │
│  free(p);  // 해제                                                      │
│                                                                     │
│  // C++                                                                │
│  int* p = new int[256];                                               │
│  delete[] p;  // 해제                                                   │
│                                                                     │
│  // C++ 스마트 포인터 (자동 해제)                                          │
│  {                                                                     │
│      std::unique_ptr<int[]> p(new int[256]);                         │
│      // ... 사용 ...                                                   │
│  }  // 자동 해제                                                         │
│                                                                     │
│  // Java (GC가 해제)                                                    │
│  int[] arr = new int[256];                                            │
│  arr = null;  // 참조 해제, GC가 메모리 해제                                │
│                                                                     │
│  // Rust (소유권 기반 자동 해제)                                           │
│  {                                                                     │
│      let v = vec![1, 2, 3];                                          │
│      // ... 사용 ...                                                   │
│  }  // 자동 해제                                                         │
│                                                                     │
│  【파일 해제】                                                             │
│  ──────────────────                                                  │
│  // 파일 핸들 해제                                                        │
│  int fd = open("file.txt", O_RDONLY);                                │
│  close(fd);  // 해제                                                    │
│                                                                     │
│  // 파일 레코드 락 해제                                                   │
│  struct flock fl;                                                     │
│  fl.l_type = F_UNLCK;                                                 │
│  fcntl(fd, F_SETLK, &fl);                                              │
│                                                                     │
│  // RAII 패턴                                                           │
│  {                                                                     │
│      std::ifstream file("file.txt");                                  │
│      // ... 사용 ...                                                   │
│  }  // 자동 해제                                                         │
│                                                                     │
│  // Java try-with-resources                                            │
│  try (FileInputStream fis = new FileInputStream("file.txt")) {        │
│      // ... 사용 ...                                                   │
│  }  // 자동 해제                                                         │
│                                                                     │
│  【데이터베이스 연결 해제】                                                  │
│  ──────────────────                                                  │
│  Connection conn = null;                                              │
│  try {                                                                 │
│      conn = dataSource.getConnection();                              │
│      // ... 사용 ...                                                   │
│  } finally {                                                          │
│      if (conn != null) {                                              │
│          conn.close();  // 해제                                         │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  // try-with-resources                                                 │
│  try (Connection conn = dataSource.getConnection()) {                 │
│      // ... 사용 ...                                                   │
│  }  // 자동 해제                                                         │
│                                                                     │
│  【프로세스 종료 시 자원 해제】                                             │
│  ──────────────────                                                  │
│  // Linux: 프로세스 종료 시 자동 해제                                        │
│  // • 열린 파일 닫힘                                                     │
│  // • 할당된 메모리 해제                                                   │
│  // • 보유한 락 해제 (일부 락은 해제 안 될 수 있음)                           │
│                                                                     │
│  // /proc로 확인                                                        │
│  cat /proc/<pid>/status  // 프로세스 상태                                │
│  cat /proc/<pid>/fd     // 열린 파일                                    │
│                                                                     │
│  【해제 모니터링】                                                         │
│  ──────────────────                                                  │
│  // 해제 로깅                                                             │
│  void logRelease(int pid, int resource) {                            │
│      log.debug("RELEASE: P{} -> R{} @ {}",                           │
│                pid, resource, System.currentTimeMillis());           │
│  }                                                                  │
│                                                                     │
│  // 해제 메트릭                                                          │
│  meterRegistry.counter("resource.release",                           │
│                        "type", resourceType).increment();            │
│                                                                     │
│  【해제 실패 처리】                                                       │
│  ──────────────────                                                  │
│  // 이미 해제된 자원                                                      │
│  try {                                                                 │
│      lock.unlock();                                                   │
│  } catch (IllegalMonitorStateException e) {                          │
│      log.warn("Attempted to unlock without holding lock");           │
│  }                                                                  │
│                                                                     │
│  // double free 방지                                                   │
│  if (ptr != NULL) {                                                   │
│      free(ptr);                                                        │
│      ptr = NULL;  // 다시 free 방지                                      │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
