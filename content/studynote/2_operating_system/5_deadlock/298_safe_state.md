# 303. 안전 상태 (Safe State)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 프로세스가 완료될 수 있는 상태
> 2. **가치**: 교착상태가 발생하지 않음을 보장
> 3. **융합**: 은행원 알고리즘, 교착상태 회피와 연관

---

## Ⅰ. 개요

### 개념 정의

안전 상태(Safe State)는 **시스템이 모든 프로세스를 순차적으로 완료시킬 수 있는 상태**다. 안전 순서(Safe Sequence)가 존재하면 안전 상태다.

### 💡 비유: 여유 자금
안전 상태는 **은행의 여유 자금**과 같다. 모든 고객이 최대 한도까지 대출을 요청해도 지급할 수 있는 상태다.

### 안전 상태 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                안전 상태 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정의】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  안전 순서 <P1, P2, ..., Pn>이 존재:                                 │ │
│  │                                                             │ │
│  │  각 Pi에 대해:                                                       │
│  │  • Pi가 요청할 수 있는 자원 ≤ 현재 가용 자원 +                           │ │
│  │    Pj (j < i)가 보유한 자원                                           │ │
│  │                                                             │ │
│  │  즉, Pi는 앞선 프로세스들이 완료되면 필요한 자원을 획득할 수 있음            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상태 다이어그램】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────┐                                              │ │
│  │  │ 안전 상태     │  ← 목표                                      │ │
│  │  │ (Safe)      │  교착상태 발생 불가                              │ │
│  │  └──────┬──────┘                                              │ │
│  │         │                                                        │
│  │         │ 자원 요청                                               │
│  │         ▼                                                        │
│  │  ┌─────────────┐                                              │ │
│  │  │ 불안전 상태   │  교착상태 "가능"                                │ │
│  │  │ (Unsafe)    │  (반드시 발생하는 것은 아님)                       │ │
│  │  └──────┬──────┘                                              │ │
│  │         │                                                        │
│  │         │ 조건 악화                                               │
│  │         ▼                                                        │
│  │  ┌─────────────┐                                              │ │
│  │  │ 교착상태     │  확정적 발생                                    │ │
│  │  │ (Deadlock)  │                                              │ │
│  │  └─────────────┘                                              │ │
│  │                                                             │ │
│  │  중요: 불안전 ≠ 교착상태, 가능성만 존재                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【안전 순서 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  프로세스   현재 할당   최대 필요   추가 필요                        │ │
│  │  ────────   ────────   ────────   ────────                       │ │
│  │  P0        0, 1, 0    7, 5, 3    7, 4, 3                         │ │
│  │  P1        2, 0, 0    3, 2, 2    1, 2, 2                         │ │
│  │  P2        3, 0, 2    9, 0, 2    6, 0, 0                         │ │
│  │                                                             │ │
│  │  Available = 3, 3, 2                                               │ │
│  │                                                             │ │
│  │  안전 순서: <P1, P0, P2>                                             │ │
│  │  1. P1: Need(1,2,2) ≤ Available(3,3,2) ✓                          │ │
│  │     → P1 완료 후 Available = (5,3,2)                               │ │
│  │  2. P0: Need(7,4,3) ≤ Available(5,3,2) ✗                          │ │
│  │     → 아직 대기...                                                   │ │
│  │  실제로는 다른 순서가 존재                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                안전 상태 상세                                           │`
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 원칙】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 안전 상태에서는 교착상태가 절대 발생하지 않음                            │ │
│  │  • 불안전 상태에서는 교착상태 "가능성"이 존재                              │ │
│  │  • 교착상태 회피 = 항상 안전 상태 유지                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【안전 상태 확인 알고리즘】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Work = Available                                                │
│  │  2. Finish[i] = false (모든 i)                                      │
│  │  3. 다음 조건을 만족하는 i 찾기:                                       │
│  │     !Finish[i] && Need[i] ≤ Work                                   │
│  │  4. 찾으면:                                                          │
│  │     Work += Allocation[i]                                          │
│  │     Finish[i] = true                                                │
│  │     3으로 돌아가기                                                     │
│  │  5. 모든 Finish[i] = true면 안전 상태                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【안전 vs 불안전 vs 교착상태】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  안전 상태:                                                            │ │
│  │  • 안전 순서 존재                                                      │ │
│  │  • 교착상태 불가능                                                      │ │
│  │  • 모든 요청이 결국 승인됨                                              │ │
│  │                                                             │ │
│  │  불안전 상태:                                                           │ │
│  │  • 안전 순서 없음                                                      │ │
│  │  • 교착상태 가능                                                       │ │
│  │  • 일부 요청이 무기한 대기 가능                                          │ │
│  │                                                             │ │
│  │  교착상태:                                                              │ │
│  │  • 순환 대기 발생                                                      │ │
│  │  • 일부 프로세스가 영원히 대기                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【안전 상태 검사 구현】                                                 │
│  ──────────────────                                                  │
│  class SafeStateChecker {                                           │
│      int[] available;                                                │
│      int[][] allocation;                                             │
│      int[][] need;                                                   │
│                                                                     │
│      boolean isSafeState() {                                        │
│          int n = allocation.length;                                 │
│          int m = available.length;                                  │
│                                                                     │
│          int[] work = available.clone();                            │
│          boolean[] finish = new boolean[n];                         │
│          List<Integer> safeSequence = new ArrayList<>();            │
│                                                                     │
│          int count = 0;                                              │
│          while (count < n) {                                         │
│              boolean found = false;                                  │
│              for (int i = 0; i < n; i++) {                           │
│                  if (!finish[i] && canSatisfy(need[i], work)) {      │
│                      // Pi의 요청을 만족할 수 있음                       │
│                      for (int j = 0; j < m; j++) {                    │
│                          work[j] += allocation[i][j];                │
│                      }                                              │
│                      finish[i] = true;                               │
│                      safeSequence.add(i);                            │
│                      found = true;                                   │
│                      count++;                                        │
│                  }                                                  │
│              }                                                      │
│              if (!found) break;  // 더 이상 진행 불가                   │
│          }                                                          │
│                                                                     │
│          if (count == n) {                                           │
│              System.out.println("Safe sequence: " + safeSequence);  │
│              return true;  // 안전 상태                                 │
│          }                                                          │
│          return false;  // 불안전 상태                                   │
│      }                                                              │
│                                                                     │
│      boolean canSatisfy(int[] need, int[] work) {                   │
│          for (int i = 0; i < need.length; i++) {                     │
│              if (need[i] > work[i]) return false;                   │
│          }                                                          │
│          return true;                                                │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【자원 요청과 안전 상태】                                                │
│  ──────────────────                                                  │
│  boolean requestResources(int pid, int[] request) {                 │
│      // 요청 가능한지 확인                                               │
│      if (!canSatisfy(request, need[pid])) {                         │
│          throw new Error("요청이 최대 필요량 초과");                     │
│      }                                                              │
│      if (!canSatisfy(request, available)) {                         │
│          return false;  // 자원 부족, 대기                               │
│      }                                                              │
│                                                                     │
│      // 시험적 할당                                                      │
│      for (int i = 0; i < m; i++) {                                  │
│          available[i] -= request[i];                                │
│          allocation[pid][i] += request[i];                          │
│          need[pid][i] -= request[i];                                │
│      }                                                              │
│                                                                     │
│      // 안전성 검사                                                      │
│      if (isSafeState()) {                                           │
│          return true;  // 승인                                        │
│      }                                                              │
│                                                                     │
│      // 불안전: 롤백                                                    │
│      for (int i = 0; i < m; i++) {                                  │
│          available[i] += request[i];                                │
│          allocation[pid][i] -= request[i];                          │
│          need[pid][i] += request[i];                                │
│      }                                                              │
│      return false;  // 거부                                           │
│  }                                                                  │
│                                                                     │
│  【실제 사용 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 자원 요청 전                                                     │ │
│  │  int[] request = {1, 0, 2};  // A=1, C=2 요청                       │ │
│  │                                                             │ │
│  │  if (checker.requestResources(pid, request)) {                     │ │
│  │      // 요청 승인됨                                                   │ │
│  │      useResources();                                               │ │
│  │  } else {                                                            │ │
│  │      // 요청 거부됨 (불안전 상태)                                       │ │
│  │      waitOrRetry();                                                │ │
│  │  }                                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 특성】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 안전 상태 검사: O(m × n²)                                          │ │
│  │  • 자원 요청마다 검사 필요                                              │ │
│  │  • 오버헤드로 인해 실제 시스템에서는 제한적 사용                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 모든 프로세스 완료 가능한 상태
• 조건: 안전 순서 존재
• 보장: 안전 상태에서는 교착상태 불가
• 불안전: 교착상태 "가능성"
• 검사: O(m × n²)
• 활용: 교착상태 회피의 기준
```
