# 291. 교착상태 (Deadlock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 두 개 이상의 작업이 서로의 자원을 기다리며 영원히 멈춤
> 2. **가치**: 동시성 시스템의 치명적 오류
> 3. **융합**: 교착상태 조건, 예방, 회피, 탐지와 연관

---

## Ⅰ. 개요

### 개념 정의

교착상태(Deadlock)는 **두 개 이상의 프로세스/스레드가 서로 상대방이 보유한 자원을 기다리며 영원히 블록된 상태**다. 아무도 진행할 수 없다.

### 💡 비유: 교차로
교착상태는 **네 방향 교차로의 차들**과 같다. 각 차가 앞 차를 기다린다. 아무도 움직일 수 없다.

### 교착상태 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                교착상태 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 시나리오】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  T1: 자원 A 보유 → 자원 B 요청 → 대기 (T2가 B 보유)               │ │
│  │                                                             │ │
│  │  T2: 자원 B 보유 → 자원 A 요청 → 대기 (T1이 A 보유)               │ │
│  │                                                             │ │
│  │  결과: T1 ↔ T2 서로 대기, 영원히 멈춤                              │ │
│  │                                                             │ │
│  │         [T1]                    [T2]                          │ │
│  │           │                        │                          │ │
│  │     보유 ▼        요청        ▼ 보유                        │ │
│  │       [자원 A] ◄──────────── [자원 B]                         │ │
│  │                  요청        ▲                                │ │
│  │                              │                                │ │
│  │  순환 대기: T1 → A → T2 → B → T1                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【필요 조건 4가지】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  모두 만족해야 교착상태 발생:                                          │ │
│  │                                                             │ │
│  │  1. 상호 배제 (Mutual Exclusion)                                   │ │
│  │     자원은 한 번에 하나만 사용 가능                                    │ │
│  │                                                             │ │
│  │  2. 보유 및 대기 (Hold and Wait)                                    │ │
│  │     자원을 보유하면서 다른 자원 대기                                    │ │
│  │                                                             │ │
│  │  3. 비선점 (No Preemption)                                         │ │
│  │     다른 프로세스의 자원을 강제로 뺏을 수 없음                            │ │
│  │                                                             │ │
│  │  4. 순환 대기 (Circular Wait)                                       │ │
│  │     대기 그래프에 순환이 존재                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                교착상태 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【자원 할당 그래프】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ● 프로세스 (원형)                                                  │ │
│  │  □ 자원 (사각형)                                                    │ │
│  │  → 요청 간선: 프로세스 → 자원                                          │ │
│  │  ← 할당 간선: 자원 → 프로세스                                          │ │
│  │                                                             │ │
│  │       ●P1 ──请求──► □R1 ──分配──► ●P2                           │ │
│  │        ▲                         │                              │ │
│  │        │                         ▼                              │ │
│  │       □R2 ◄──分配── ●P3 ◄──请求── □R3                          │ │
│  │                                                             │ │
│  │  순환이 있으면 교착상태 가능성                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결 전략】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 예방 (Prevention):                                             │ │
│  │     4가지 조건 중 하나를 무력화                                         │ │
│  │                                                             │ │
│  │  2. 회피 (Avoidance):                                               │ │
│  │     실행 중 안전하지 않은 요청 거부                                       │ │
│  │                                                             │ │
│  │  3. 탐지 및 복구 (Detection & Recovery):                            │ │
│  │     발생 후 탐지하고 해결                                               │ │
│  │                                                             │ │
│  │  4. 무시 (Ignorance):                                               │ │
│  │     발생 빈도가 낮으면 무시                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실제 사례】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 데이터베이스 락 교착상태                                             │ │
│  │  • 파일 시스템 락 교착상태                                              │ │
│  │  • 네트워크 통신 타임아웃                                               │ │
│  │  • 커널 자원 경합                                                     │ │
│  │  • 멀티스레드 프로그램                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【교착상태 발생 코드】                                                 │
│  ──────────────────                                                  │
│  // 스레드 1                                                          │
│  pthread_mutex_lock(&mutexA);                                        │
│  // ... 작업 ...                                                      │
│  pthread_mutex_lock(&mutexB);  // 대기!                               │
│                                                                     │
│  // 스레드 2 (동시 실행)                                                 │
│  pthread_mutex_lock(&mutexB);                                        │
│  // ... 작업 ...                                                      │
│  pthread_mutex_lock(&mutexA);  // 대기! → 교착상태                      │
│                                                                     │
│  【교착상태 예방: 순서 규칙】                                             │
│  ──────────────────                                                  │
│  // 항상 같은 순서로 락 획득                                               │
│  void transfer(Account* from, Account* to, int amount) {            │
│      Account* first = from->id < to->id ? from : to;                │
│      Account* second = from->id < to->id ? to : from;               │
│                                                                     │
│      pthread_mutex_lock(&first->lock);                              │
│      pthread_mutex_lock(&second->lock);                             │
│                                                                     │
│      from->balance -= amount;                                        │
│      to->balance += amount;                                          │
│                                                                     │
│      pthread_mutex_unlock(&second->lock);                           │
│      pthread_mutex_unlock(&first->lock);                            │
│  }                                                                  │
│                                                                     │
│  【데이터베이스 교착상태 감지】                                            │
│  ──────────────────                                                  │
│  -- MySQL                                                            │
│  SHOW ENGINE INNODB STATUS;  -- 교착상태 정보                           │
│                                                                     │
│  -- PostgreSQL                                                       │
│  SELECT * FROM pg_stat_activity WHERE wait_event = 'Lock';          │
│                                                                     │
│  -- Oracle                                                           │
│  SELECT * FROM V$LOCK WHERE BLOCK > 0;                              │
│                                                                     │
│  【Java 교착상태 감지】                                                 │
│  ──────────────────                                                  │
│  // Thread dump                                                      │
│  jstack <pid> | grep -A 20 "deadlock"                               │
│                                                                     │
│  // JMX으로 프로그래밍 감지                                               │
│  ThreadMXBean threadBean = ManagementFactory.getThreadMXBean();     │
│  long[] deadlockedThreads = threadBean.findDeadlockedThreads();     │
│  if (deadlockedThreads != null) {                                   │
│      ThreadInfo[] infos = threadBean.getThreadInfo(deadlockedThreads);│
│      // 로그 및 알림                                                   │
│  }                                                                  │
│                                                                     │
│  【교착상태 복구: 타임아웃】                                               │
│  ──────────────────                                                  │
│  // 타임아웃으로 무한 대기 방지                                             │
│  if (pthread_mutex_timedlock(&mutex, &timeout) == ETIMEDOUT) {      │
│      // 타임아웃 발생, 복구 로직                                           │
│      rollback();                                                    │
│      retry_or_abort();                                              │
│  }                                                                  │
│                                                                     │
│  【모니터링】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  지표:                                                              │ │
│  │  • 교착상태 발생 횟수                                                  │ │
│  │  • 교착상태 해결 시간                                                  │ │
│  │  • 롤백 횟수                                                        │ │
│  │  • 대기 시간                                                        │ │
│  │                                                             │ │
│  │  도구:                                                              │ │
│  │  • Linux: lockdep                                                  │ │
│  │  • Java: JConsole, JVisualVM                                       │ │
│  │  • DB: SHOW ENGINE INNODB STATUS                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
