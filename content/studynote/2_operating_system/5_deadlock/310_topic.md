# 310. 자원 요청 (Resource Request)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스가 자원을 요구하는 행위
> 2. **가치**: 교착상태 발생의 시작점
> 3. **융합**: 자원 할당, 자원 해제와 연관

---

## Ⅰ. 개요

### 개념 정의

자원 요청(Resource Request)은 **프로세스가 작업 수행을 위해 필요한 자원을 시스템에 요구하는 행위**다. 요청은 즉시 충족되거나 대기 상태가 될 수 있다.

### 💡 비유: 도서관 대출 신청
자원 요청은 **도서관 대출 신청**과 같다. 책을 빌리고 싶다고 신청한다. 책이 있으면 바로 받고, 없으면 대기한다.

### 자원 요청 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 요청 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【요청 시나리오】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  프로세스 P                                                            │ │
│  │       │                                                             │ │
│  │       │ request(R)                                                  │ │
│  │       ▼                                                             │ │
│  │  ┌─────────────┐                                                    │ │
│  │  │ 자원 관리자   │                                                    │ │
│  │  └──────┬──────┘                                                    │ │
│  │         │                                                             │
│  │    ┌────┴────┐                                                       │ │
│  │    │         │                                                       │ │
│  │   가용       불가용                                                   │
│  │    │         │                                                       │
│  │    ▼         ▼                                                       │
│  │  할당       대기                                                      │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【요청 결과】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 즉시 할당:                                                        │ │
│  │     • 요청한 자원이 가용 상태                                          │ │
│  │     • 프로세스가 자원 획득                                             │ │
│  │                                                             │ │
│  │  2. 대기:                                                             │ │
│  │     • 자원이 다른 프로세스에 할당됨                                     │ │
│  │     • 요청 큐에 등록                                                   │ │
│  │     • 자원 해제 시 재시도                                              │ │
│  │                                                             │ │
│  │  3. 거부:                                                             │ │
│  │     • 교착상태 회피를 위해 거부                                         │ │
│  │     • 최대 필요량 초과                                                 │ │
│  │     • 권한 없음                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【요청 정보】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 프로세스 ID (Process ID)                                          │ │
│  │  • 자원 유형 (Resource Type)                                         │ │
│  │  • 요청 수량 (Request Count)                                         │ │
│  │  • 요청 시간 (Request Time)                                          │ │
│  │  • 우선순위 (Priority) - 선택적                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 요청 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【은행원 알고리즘에서의 요청 처리】                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Request[i] = 프로세스 i의 자원 요청 벡터                              │ │
│  │                                                             │ │
│  │  1. Request[i] ≤ Need[i] 확인                                        │ │
│  │     아니면: "최대 필요량 초과" 에러                                     │ │
│  │                                                             │ │
│  │  2. Request[i] ≤ Available 확인                                       │ │
│  │     아니면: 대기 (가용 자원 부족)                                       │ │
│  │                                                             │ │
│  │  3. 시험적 할당:                                                       │ │
│  │     Available = Available - Request[i]                              │
│  │     Allocation[i] = Allocation[i] + Request[i]                      │
│  │     Need[i] = Need[i] - Request[i]                                  │
│  │                                                             │ │
│  │  4. 안전성 검사:                                                       │ │
│  │     안전하면: 승인                                                      │ │
│  │     불안전하면: 롤백하고 대기                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【요청 패턴】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 단일 요청:                                                        │ │
│  │     • 한 번에 하나의 자원 요청                                          │ │
│  │     • 간단하지만 비효율적                                              │ │
│  │                                                             │ │
│  │  2. 일괄 요청:                                                        │ │
│  │     • 필요한 모든 자원을 한 번에 요청                                   │ │
│  │     • 교착상태 예방 효과                                               │ │
│  │                                                             │ │
│  │  3. 점진적 요청:                                                      │ │
│  │     • 필요할 때마다 요청                                               │ │
│  │     • 효율적이지만 교착상태 위험                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【요청과 교착상태】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  교착상태는 요청에서 시작:                                              │ │
│  │  • P1이 R1 요청 (이미 P2가 보유)                                       │ │
│  │  • P1이 대기                                                          │ │
│  │  • P2가 R2 요청 (이미 P1이 보유)                                       │ │
│  │  • 순환 대기 형성 → 교착상태                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락 요청】                                                             │
│  ──────────────────                                                  │
│  // C++ mutex                                                         │
│  std::mutex mtx;                                                      │
│  mtx.lock();        // 요청 (블로킹)                                    │
│  mtx.try_lock();    // 요청 (논블로킹)                                   │
│  mtx.try_lock_for(std::chrono::seconds(5));  // 요청 (타임아웃)         │
│                                                                     │
│  // Java ReentrantLock                                                │
│  ReentrantLock lock = new ReentrantLock();                           │
│  lock.lock();                              // 요청 (블로킹)             │
│  lock.tryLock();                           // 요청 (논블로킹)           │
│  lock.tryLock(5, TimeUnit.SECONDS);        // 요청 (타임아웃)           │
│                                                                     │
│  【메모리 요청】                                                         │
│  ──────────────────                                                  │
│  // C                                                                  │
│  void* ptr = malloc(1024);  // 1KB 요청                               │
│  if (ptr == NULL) {                                                   │
│      // 요청 실패                                                       │
│  }                                                                  │
│                                                                     │
│  // Java                                                               │
│  byte[] data = new byte[1024];  // 1KB 요청                           │
│  // OutOfMemoryError 발생 가능                                         │
│                                                                     │
│  【파일 요청】                                                             │
│  ──────────────────                                                  │
│  // 파일 열기 요청                                                       │
│  int fd = open("file.txt", O_RDONLY);                                │
│  if (fd == -1) {                                                      │
│      // 요청 실패 (파일 없음, 권한 없음 등)                               │
│  }                                                                  │
│                                                                     │
│  // 파일 잠금 요청                                                       │
│  struct flock fl;                                                     │
│  fl.l_type = F_RDLCK;                                                 │
│  fcntl(fd, F_SETLKW, &fl);  // 블로킹 요청                              │
│  fcntl(fd, F_SETLK, &fl);   // 논블로킹 요청                             │
│                                                                     │
│  【데이터베이스 연결 요청】                                                │
│  ──────────────────                                                  │
│  // 연결 요청                                                           │
│  Connection conn = DriverManager.getConnection(                      │
│      "jdbc:mysql://localhost/db", "user", "pass");                   │
│                                                                     │
│  // 풀에서 연결 요청                                                      │
│  Connection conn = dataSource.getConnection();                       │
│  // 풀이 비어있으면 대기                                                  │
│                                                                     │
│  【세마포어 요청】                                                        │
│  ──────────────────                                                  │
│  // POSIX 세마포어                                                      │
│  sem_t sem;                                                           │
│  sem_init(&sem, 0, 5);  // 초기값 5                                    │
│                                                                     │
│  sem_wait(&sem);        // 요청 (블로킹)                                │
│  sem_trywait(&sem);     // 요청 (논블로킹)                               │
│  sem_timedwait(&sem, &timeout);  // 요청 (타임아웃)                      │
│                                                                     │
│  【요청 처리 흐름】                                                       │
│  ──────────────────                                                  │
│  class ResourceRequestHandler {                                      │
│      ResourceManager manager;                                        │
│                                                                     │
│      RequestResult request(int pid, int resourceType, int count) {   │
│          // 1. 요청 유효성 검사                                          │
│          if (!isValidRequest(pid, resourceType, count)) {             │
│              return RequestResult.INVALID;                           │
│          }                                                          │
│                                                                     │
│          // 2. 가용 자원 확인                                            │
│          if (manager.getAvailable(resourceType) < count) {           │
│              // 3. 대기 큐에 등록                                        │
│              manager.addToWaitQueue(pid, resourceType, count);        │
│              return RequestResult.WAITING;                           │
│          }                                                          │
│                                                                     │
│          // 4. 할당                                                    │
│          manager.allocate(pid, resourceType, count);                 │
│          return RequestResult.GRANTED;                              │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【요청 모니터링】                                                        │
│  ──────────────────                                                  │
│  // 요청 통계                                                           │
│  class RequestMetrics {                                              │
│      AtomicLong totalRequests = new AtomicLong();                    │
│      AtomicLong grantedRequests = new AtomicLong();                  │
│      AtomicLong waitingRequests = new AtomicLong();                  │
│      AtomicLong rejectedRequests = new AtomicLong();                 │
│                                                                     │
│      void recordRequest(RequestResult result) {                       │
│          totalRequests.incrementAndGet();                            │
│          switch (result) {                                            │
│              case GRANTED: grantedRequests.incrementAndGet(); break; │
│              case WAITING: waitingRequests.incrementAndGet(); break; │
│              case REJECTED: rejectedRequests.incrementAndGet(); break;│
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
