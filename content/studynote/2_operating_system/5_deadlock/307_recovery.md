# 교착 상태 복구 (Recovery from Deadlock)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착 상태 복구(Recovery from Deadlock)는 탐지 알고리즘에 의해 확인된 교착 상태를 해소하기 위해 시스템 자원을 강제로 조정하거나 프로세스를 중단시키는 사후 수습 과정이다.
> 2. **메커니즘**: 주로 프로세스를 강제 종료(Termination)하거나, 자원을 선점(Preemption)하여 다른 프로세스에 할당하는 두 가지 전략을 사용한다.
> 3. **가치**: 시스템이 멈춘 상태를 정상화하여 전체적인 가용성을 회복하지만, 희생되는 프로세스의 작업 손실과 복구 오버헤드 사이의 정밀한 계산이 필요하다.

+++

## Ⅰ. 교착 상태 복구의 정의 및 중요성

### 1. 정의
- 탐지 알고리즘이 "교착 상태임!"을 선언했을 때, 시스템을 다시 작동 가능하게 만드는 작업이다.

### 2. 복구의 필요성
- 복구하지 않으면 시스템은 영구적으로 응답하지 않으며, 자원은 낭비된다. 따라서 자동화된 복구 메커니즘은 고가용성 시스템의 필수 요소이다.

- **📢 섹션 요약 비유**: "길이 꽉 막혔을 때 견인차를 불러서 차 한 대를 끌어내거나, 경찰이 와서 강제로 차를 뒤로 빼게 하는 것과 같습니다."

+++

## Ⅱ. 복구 전략의 분류 (ASCII)

### 1. 프로세스 종료 (Process Termination)
- 교착 상태에 연루된 프로세스를 하나 이상 종료하여 자원을 회수한다.

### 2. 자원 선점 (Resource Preemption)
- 프로세스로부터 자원을 빼앗아 다른 프로세스에게 준다.

```text
  [ Detection Result ]
  P1 <-> R1 <-> P2 (Cycle Found!)
         |
  [ Recovery Action ]
         |
  +------+------+------+
  |             |      |
 [Abort All] [Seq Abort] [Preemption]
  (Reset)     (Careful)   (Take it!)
```

- **📢 섹션 요약 비유**: "싸우는 애들을 다 집으로 보내버릴지, 한 명만 불러서 훈육할지, 아니면 가지고 있는 장난감을 뺏어서 줄지 결정하는 것과 같습니다."

+++

## Ⅲ. 복구 알고리즘의 결정 요소 (Policy)

### 1. 종료 대상 선택
- 가장 적은 비용으로 데드락을 풀 수 있는 '희생자'를 골라야 한다.

### 2. 롤백(Rollback) 범위
- 프로세스를 어느 시점까지 되돌릴 것인가?
- 완전 롤백 vs 체크포인트 롤백.

- **📢 섹션 요약 비유**: "전쟁터에서 후퇴할 때, 처음 자리까지 도망칠지 아니면 아까 지어둔 기지까지만 도망칠지 정하는 것과 같습니다."

+++

## Ⅳ. 복구의 트레이드오프 (Analysis)

| 전략 | 장점 | 단점 |
|:---|:---|:---|
| **프로세스 종료** | 가장 확실하고 단순함 | 수행 중이던 작업 데이터가 소실됨 |
| **자원 선점** | 작업 손실을 최소화할 수 있음 | 롤백 로직이 복잡하고 기아 상태 위험 |

- **📢 섹션 요약 비유**: "컴퓨터를 껐다 켜는 게 가장 빠르지만 작업한 게 다 날아가는 것과, 하나씩 프로그램을 끄면서 살려보려고 노력하는 것의 차이입니다."

+++

## Ⅴ. 결론 및 실무적 교훈

### 1. 요약
- 교착 상태 복구는 '누군가는 희생해야 시스템이 산다'는 비정한 운영 원칙을 바탕으로 한다.

### 2. 최종 결론
- 효율적인 복구를 위해서는 프로세스의 우선순위, 소요 시간, 자원 소모량 등을 미리 관리해야 하며, 최악의 경우를 대비한 체크포인팅(Checkpointing) 기법이 병행되어야 한다.

- **📢 섹션 요약 비유**: "문제가 터진 후에 우왕좌왕하기보다, 누구를 먼저 도와줄지 미리 순서를 정해두는 것이 가장 중요하다는 교훈을 줍니다."

+++

- **프로세스 종료 방식**: 복구 시 사용하는 가장 일반적인 기법.
- **자원 선점 방식**: 좀 더 정교한 복구 기법.
- **체크포인트 (Checkpoint)**: 복구 시 손실을 최소화하기 위한 저장점.
