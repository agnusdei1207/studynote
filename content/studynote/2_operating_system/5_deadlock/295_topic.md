# 295. 교착상태 탐지 (Deadlock Detection)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착상태 발생 후 순환을 탐지
> 2. **가치**: 교착상태 복구의 전제 조건
> 3. **융합**: 대기 그래프, 자원 할당 그래프와 연관

---

## Ⅰ. 개요

### 개념 정의

교착상태 탐지(Deadlock Detection)는 **시스템에서 교착상태가 발생했는지 확인하고, 어떤 프로세스가 관련되었는지 식별하는 과정**이다. 탐지 후 복구가 이어진다.

### 💡 비유: 구조 요청
교착상태 탐지는 **조난 신호 수신**과 같다. 문제가 발생했는지 확인한다. 누가 위험에 처했는지 찾는다. 그 다음 구조가 이어진다.

### 교착상태 탐지 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                교착상태 탐지 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【탐지 방법】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 타임아웃 기반:                                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 일정 시간 이상 대기 시 교착상태로 간주                         │ │ │
│  │  │  • 간단하지만 오탐 가능성                                      │ │ │
│  │  │  • 가장 많이 사용                                             │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  2. 대기 그래프 분석:                                                 │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 프로세스 간 대기 관계 그래프 구축                           │ │ │
│  │  │  • 순환 탐지 알고리즘 (DFS, Union-Find)                       │ │ │
│  │  │  • 정확하지만 오버헤드 존재                                     │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  3. 자원 할당 그래프 분석:                                            │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 프로세스-자원 간 관계 그래프                                │ │ │
│  │  │  • 그래프 축소로 교착상태 확인                                 │ │ │
│  │  │  • 가장 정확                                                   │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대기 그래프】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  노드: 프로세스                                                       │ │
│  │  간선: P1 → P2 (P1이 P2가 보유한 자원을 대기)                          │ │
│  │                                                             │ │
│  │       [P1] ────► [P2]                                              │ │
│  │         ▲            │                                             │ │
│  │         │            ▼                                             │ │
│  │       [P4] ◄──── [P3]                                              │ │
│  │                                                             │ │
│  │  순환: P1 → P2 → P3 → P4 → P1 → 교착상태!                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                교착상태 탐지 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【탐지 알고리즘】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. DFS 기반 순환 탐지:                                             │ │
│  │     O(V + E)                                                       │ │
│  │     - 각 노드에서 DFS 시작                                           │ │
│  │     - 방문 중인 노드 재방문 시 순환                                     │ │
│  │                                                             │ │
│  │  2. Union-Find:                                                    │ │
│  │     O(E α(V))                                                       │ │
│  │     - 간선 추가 시 같은 집합이면 순환                                   │ │
│  │                                                             │ │
│  │  3. 그래프 축소 (Reduction):                                        │
│  │     - 요청을 만족할 수 있는 프로세스 제거                               │ │
│  │     - 남은 프로세스가 있으면 교착상태                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【탐지 시점】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 주기적 탐지:                                                     │ │
│  │     • 일정 간격으로 탐지                                              │ │
│  │     • 오버헤드 일정                                                   │ │
│  │                                                             │ │
│  │  2. 자원 요청 시 탐지:                                                │ │
│  │     • 요청이 블록될 때마다 탐지                                        │ │
│  │     • 빠른 감지, 높은 오버헤드                                         │ │
│  │                                                             │ │
│  │  3. 타임아웃 시 탐지:                                                 │ │
│  │     • 일정 시간 대기 시 탐지                                          │ │
│  │     • 균형잡힌 접근                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【복잡도 vs 정확도】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  방법            복잡도        정확도       오버헤드                │ │
│  │  ────────        ──────        ──────       ────────                │ │
│  │  타임아웃         O(1)         낮음         낮음                    │ │
│  │  대기 그래프      O(V + E)      높음         중간                    │ │
│  │  자원 그래프      O(V² × E)     매우 높음    높음                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【순환 탐지 구현 (DFS)】                                              │
│  ──────────────────                                                  │
│  class DeadlockDetector {                                           │
│      Map<Integer, Set<Integer>> waitGraph;  // P1 → {P2, P3, ...}   │
│                                                                     │
│      boolean hasCycle() {                                           │
│          Set<Integer> visited = new HashSet<>();                    │
│          Set<Integer> recStack = new HashSet<>();                   │
│                                                                     │
│          for (int node : waitGraph.keySet()) {                      │
│              if (dfs(node, visited, recStack))                      │
│                  return true;                                        │
│          }                                                          │
│          return false;                                               │
│      }                                                              │
│                                                                     │
│      boolean dfs(int node, Set<Integer> visited, Set<Integer> recStack) {│
│          if (recStack.contains(node)) return true;  // 순환!           │
│          if (visited.contains(node)) return false;                  │
│                                                                     │
│          visited.add(node);                                          │
│          recStack.add(node);                                         │
│                                                                     │
│          for (int neighbor : waitGraph.getOrDefault(node, Set.of())) {│
│              if (dfs(neighbor, visited, recStack))                  │
│                  return true;                                        │
│          }                                                          │
│                                                                     │
│          recStack.remove(node);                                      │
│          return false;                                               │
│      }                                                              │
│                                                                     │
│      Set<Integer> findDeadlockedProcesses() {                       │
│          // 교착상태에 연관된 모든 프로세스 반환                            │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Java 교착상태 탐지】                                                │
│  ──────────────────                                                  │
│  // JMX으로 프로그래밍 탐지                                              │
│  ThreadMXBean threadBean = ManagementFactory.getThreadMXBean();     │
│                                                                     │
│  // 모니터 교착상태 탐지                                                  │
│  long[] deadlocked = threadBean.findDeadlockedThreads();            │
│  if (deadlocked != null) {                                          │
│      ThreadInfo[] infos = threadBean.getThreadInfo(deadlocked);     │
│      for (ThreadInfo info : infos) {                                │
│          System.out.println("Deadlocked: " + info.getThreadName()); │
│          System.out.println("  Lock: " + info.getLockName());       │
│          System.out.println("  Owner: " + info.getLockOwnerName()); │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  // jstack 명령줄                                                     │
│  // jstack <pid> | grep -A 20 "Found one Java-level deadlock"       │
│                                                                     │
│  【데이터베이스 교착상태 탐지】                                            │
│  ──────────────────                                                  │
│  // MySQL InnoDB                                                     │
│  SHOW ENGINE INNODB STATUS;  // LATEST DETECTED DEADLOCK 섹션        │
│                                                                     │
│  // 자동 탐지 및 롤백                                                   │
│  // InnoDB는 교착상태 자동 탐지 후 한 트랜잭션 롤백                         │
│                                                                     │
│  // PostgreSQL                                                       │
│  SET deadlock_timeout = '1s';                                       │
│  // 교착상태 탐지 후 에러 반환                                             │
│                                                                     │
│  【Linux lockdep】                                                    │
│  ──────────────────                                                  │
│  // 커널 락 의존성 추적                                                  │
│  CONFIG_LOCKDEP=y                                                    │
│                                                                     │
│  // 교착상태 경고 확인                                                   │
│  dmesg | grep -i "possible circular locking dependency"             │
│                                                                     │
│  // /proc/lockdep로 상세 확인                                          │
│  cat /proc/lockdep                                                  │
│                                                                     │
│  【성능 고려사항】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  탐지 방법        탐지 주기    CPU 사용    응답 시간                 │ │
│  │  ────────        ──────      ──────     ────────                   │ │
│  │  요청 시마다       높음        높음        빠름                       │ │
│  │  주기적 (1초)      중간        중간        보통                       │ │
│  │  타임아웃 시        낮음        낮음        느림                       │ │
│  │                                                             │ │
│  │  권장: 시스템 부하와 요구사항에 따라 선택                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 교착상태 발생 후 탐지
• 방법: 타임아웃, 대기 그래프, 자원 그래프
• 알고리즘: DFS, Union-Find, 그래프 축소
• 시점: 주기적, 요청 시, 타임아웃 시
• 도구: jstack, lockdep, DB 모니터링
• 복잡도: O(V + E) ~ O(V² × E)
```
