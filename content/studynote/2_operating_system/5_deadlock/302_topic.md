# 302. 은행원 알고리즘 (Banker's Algorithm)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착상태 회피를 위한 고전 알고리즘
> 2. **가치**: 안전 상태 유지로 교착상태 방지
> 3. **융합**: 안전 상태, 교착상태 회피와 연관

---

## Ⅰ. 개요

### 개념 정의

은행원 알고리즘(Banker's Algorithm)은 **Dijkstra가 제안한 교착상태 회피 알고리즘**이다. 자원 할당 전 안전성을 검사하여, 불안전 상태로 이어지는 요청을 거부한다.

### 💡 비유: 은행 대출
은행원 알고리즘은 **은행의 대출 승인**과 같다. 은행은 모든 고객이 최대 대출 한도를 요청해도 지급 가능할 때만 승인한다.

### 은행원 알고리즘 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                은행원 알고리즘 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데이터 구조】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  n: 프로세스 수                                                      │ │
│  │  m: 자원 유형 수                                                     │ │
│  │                                                             │ │
│  │  Available[m]: 각 자원의 가용 인스턴스 수                              │ │
│  │    예: Available = [3, 3, 2]  // A=3, B=3, C=2                   │ │
│  │                                                             │ │
│  │  Max[n][m]: 각 프로세스의 최대 자원 필요량                              │ │
│  │    예: Max[0] = [7, 5, 3]  // P0는 최대 A=7, B=5, C=3 필요        │ │
│  │                                                             │ │
│  │  Allocation[n][m]: 현재 각 프로세스에 할당된 자원                       │ │
│  │    예: Allocation[0] = [0, 1, 0]  // P0에 A=0, B=1, C=0 할당      │ │
│  │                                                             │ │
│  │  Need[n][m]: 각 프로세스의 추가 필요량                                 │ │
│  │    Need[i][j] = Max[i][j] - Allocation[i][j]                       │ │
│  │    예: Need[0] = [7, 4, 3]  // P0는 A=7, B=4, C=3 추가 필요        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예제 상태】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │           Max          Allocation       Need          Available │ │
│  │         A  B  C        A  B  C         A  B  C       A  B  C    │ │
│  │  P0    7  5  3        0  1  0         7  4  3                    │ │
│  │  P1    3  2  2        2  0  0         1  2  2                    │ │
│  │  P2    9  0  2        3  0  2         6  0  0                    │ │
│  │  P3    2  2  2        2  1  1         0  1  1         3  3  2   │ │
│  │  P4    4  3  3        0  0  2         4  3  1                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                은행원 알고리즘 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【안전성 알고리즘 (Safety Algorithm)】                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Work[m] = Available[m]                                        │
│  │     Finish[i] = false (모든 i에 대해)                               │
│  │                                                             │ │
│  │  2. 다음 조건을 만족하는 i 찾기:                                       │
│  │     a) Finish[i] == false                                         │ │
│  │     b) Need[i] <= Work                                            │ │
│  │                                                             │ │
│  │  3. 찾으면:                                                          │
│  │     Work = Work + Allocation[i]                                   │
│  │     Finish[i] = true                                               │
│  │     2로 돌아가기                                                      │ │
│  │                                                             │ │
│  │  4. 못 찾으면:                                                       │
│  │     모든 Finish[i] == true면 안전 상태                                 │ │
│  │     아니면 불안전 상태                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【자원 요청 알고리즘 (Resource-Request Algorithm)】                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Request[i][j]: 프로세스 i의 자원 j 요청량                             │ │
│  │                                                             │ │
│  │  1. Request[i] <= Need[i] 확인                                      │ │
│  │     아니면: 최대 필요량 초과 에러                                        │ │
│  │                                                             │ │
│  │  2. Request[i] <= Available 확인                                     │ │
│  │     아니면: 대기                                                       │ │
│  │                                                             │ │
│  │  3. 시험적 할당:                                                       │ │
│  │     Available = Available - Request[i]                             │ │
│  │     Allocation[i] = Allocation[i] + Request[i]                     │ │
│  │     Need[i] = Need[i] - Request[i]                                 │ │
│  │                                                             │ │
│  │  4. 안전성 검사:                                                      │
│  │     안전하면: 승인                                                      │ │
│  │     불안전하면: 롤백하고 대기                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예제: 안전 순서 찾기】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  초기: Work = [3, 3, 2], 모든 Finish = false                        │ │
│  │                                                             │ │
│  │  1. P1: Need[1] = [1, 2, 2] <= [3, 3, 2] ✓                         │ │
│  │     Work = [3, 3, 2] + [2, 0, 0] = [5, 3, 2]                       │ │
│  │     Finish[1] = true                                                │ │
│  │                                                             │ │
│  │  2. P3: Need[3] = [0, 1, 1] <= [5, 3, 2] ✓                         │ │
│  │     Work = [5, 3, 2] + [2, 1, 1] = [7, 4, 3]                       │ │
│  │     Finish[3] = true                                                │ │
│  │                                                             │ │
│  │  ... (계속)                                                          │ │
│  │                                                             │ │
│  │  안전 순서: <P1, P3, P4, P0, P2>                                     │ │
│  │  → 안전 상태!                                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Java 구현】                                                         │
│  ──────────────────                                                  │
│  class BankersAlgorithm {                                           │
│      int nProcesses, nResources;                                    │
│      int[] available;                                                │
│      int[][] max;                                                    │
│      int[][] allocation;                                             │
│      int[][] need;                                                   │
│                                                                     │
│      public BankersAlgorithm(int[] available, int[][] max,          │
│                               int[][] allocation) {                 │
│          this.available = available.clone();                        │
│          this.max = max.clone();                                     │
│          this.allocation = allocation.clone();                       │
│          this.nProcesses = max.length;                              │
│          this.nResources = available.length;                        │
│                                                                     │
│          // Need 계산                                                │
│          this.need = new int[nProcesses][nResources];               │
│          for (int i = 0; i < nProcesses; i++) {                     │
│              for (int j = 0; j < nResources; j++) {                 │
│                  need[i][j] = max[i][j] - allocation[i][j];         │
│              }                                                      │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      public boolean isSafe() {                                       │
│          int[] work = available.clone();                            │
│          boolean[] finish = new boolean[nProcesses];                │
│                                                                     │
│          while (true) {                                              │
│              int found = -1;                                         │
│              for (int i = 0; i < nProcesses; i++) {                  │
│                  if (!finish[i] && lessOrEqual(need[i], work)) {     │
│                      found = i;                                      │
│                      break;                                          │
│                  }                                                  │
│              }                                                      │
│                                                                     │
│              if (found == -1) break;                                 │
│                                                                     │
│              for (int j = 0; j < nResources; j++) {                  │
│                  work[j] += allocation[found][j];                    │
│              }                                                      │
│              finish[found] = true;                                   │
│          }                                                          │
│                                                                     │
│          for (boolean f : finish)                                    │
│              if (!f) return false;                                   │
│          return true;                                                │
│      }                                                              │
│                                                                     │
│      public boolean requestResources(int pid, int[] request) {       │
│          // 1. 요청이 필요량 이하인지 확인                                │
│          if (!lessOrEqual(request, need[pid])) return false;        │
│                                                                     │
│          // 2. 요청이 가용량 이하인지 확인                                │
│          if (!lessOrEqual(request, available)) return false;        │
│                                                                     │
│          // 3. 시험적 할당                                              │
│          for (int j = 0; j < nResources; j++) {                      │
│              available[j] -= request[j];                             │
│              allocation[pid][j] += request[j];                       │
│              need[pid][j] -= request[j];                             │
│          }                                                          │
│                                                                     │
│          // 4. 안전성 검사                                              │
│          if (isSafe()) {                                             │
│              return true;  // 승인                                    │
│          } else {                                                    │
│              // 롤백                                                   │
│              for (int j = 0; j < nResources; j++) {                  │
│                  available[j] += request[j];                         │
│                  allocation[pid][j] -= request[j];                   │
│                  need[pid][j] += request[j];                         │
│              }                                                      │
│              return false;  // 거부                                   │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      private boolean lessOrEqual(int[] a, int[] b) {                │
│          for (int i = 0; i < a.length; i++) {                        │
│              if (a[i] > b[i]) return false;                         │
│          }                                                          │
│          return true;                                                │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【시간 복잡도】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  안전성 알고리즘: O(m × n²)                                            │ │
│  │  자원 요청 알고리즘: O(m × n²)                                         │ │
│  │                                                             │ │
│  │  m: 자원 유형 수, n: 프로세스 수                                        │ │
│  │                                                             │ │
│  │  → 오버헤드가 커서 실제 시스템에서는 잘 사용 안 됨                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실제 적용 사례】                                                      │
│  ──────────────────                                                  │
│  • 교육용: 교착상태 회피 개념 학습                                          │
│  • 특수 시스템: 자원 유형이 고정된 실시간 시스템                               │
│  • DBMS: 일부 고급 격리 수준                                             │
│                                                                     │
│  【현실적 한계】                                                         │
│  ──────────────────                                                  │
│  • 최대 자원 필요량을 미리 알아야 함                                         │
│  • 프로세스 수가 고정되어야 함                                              │
│  • 자원 유형이 고정되어야 함                                               │
│  • 오버헤드가 큼                                                         │
│  • 보수적 (불필요한 대기 발생)                                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 교착상태 회피 고전 알고리즘
• 구조: Available, Max, Allocation, Need
• 핵심: 안전성 검사로 불안전 요청 거부
• 안전 순서: 모든 프로세스 완료 가능 순서
• 복잡도: O(m × n²)
• 현실: 한계로 인해 실제 사용 제한적
```
