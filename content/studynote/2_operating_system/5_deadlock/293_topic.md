# 293. 교착상태 예방 (Deadlock Prevention)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 4가지 조건 중 하나를 무력화
> 2. **가치**: 교착상태 발생 자체를 차단
> 3. **융합**: 락 순서, 두 단계 락킹, 타임아웃과 연관

---

## Ⅰ. 개요

### 개념 정의

교착상태 예방(Deadlock Prevention)은 **교착상태 4가지 필요조건 중 적어도 하나를 무력화하여 교착상태 발생 자체를 차단하는 전략**이다.

### 💡 비유: 안전벨트
교착상태 예방은 **안전벨트**와 같다. 사고가 발생하기 전에 미리 착용한다. 사고를 예방하는 것이 아니라, 사고 시 피해를 예방한다.

```
│  │  │  • tryLock 사용                                            │ │ │
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【순환 대기 예방: 락 순서 규칙】                                        │
│  ──────────────────                                                  │
│  // 자원 번호 정의                                                     │
│  enum LockOrder {                                                    │
│      LOCK_DATABASE = 1,                                              │
│      LOCK_FILE = 2,                                                  │
│      LOCK_NETWORK = 3,                                               │
│      LOCK_CACHE = 4                                                  │
│  };                                                                 │
│                                                                     │
│  void safe_operation() {                                            │
│      // 항상 오름차순으로 획득                                            │
│      pthread_mutex_lock(&db_lock);      // 1                        │
│      pthread_mutex_lock(&file_lock);    // 2                        │
│      pthread_mutex_lock(&net_lock);     // 3                        │
│                                                                     │
│      // 작업 수행                                                      │
│                                                                     │
│      // 역순으로 해제                                                   │
│      pthread_mutex_unlock(&net_lock);                               │
│      pthread_mutex_unlock(&file_lock);                              │
│      pthread_mutex_unlock(&db_lock);                                │
│  }                                                                  │
│                                                                     │
│  【보유 및 대기 예방: 일괄 요청】                                        │
│  ──────────────────                                                  │
│  void hold_and_wait_prevention() {                                  │
│      // 1단계: 모든 락을 한 번에 획득                                     │
│      pthread_mutex_lock(&global_lock);  // 전역 락                    │
│      pthread_mutex_lock(&lockA);                                      │
│      pthread_mutex_lock(&lockB);                                      │
│      pthread_mutex_unlock(&global_lock);                             │
│                                                                     │
│      // 2단계: 작업 수행                                                │
│      do_work();                                                      │
│                                                                     │
│      // 3단계: 모든 락 해제                                             │
│      pthread_mutex_unlock(&lockB);                                   │
│      pthread_mutex_unlock(&lockA);                                   │
│  }                                                                  │
│                                                                     │
│  【비선점 예방: tryLock】                                               │
│  ──────────────────                                                  │
│  void no_preemption_prevention() {                                  │
│      while (true) {                                                  │
│          pthread_mutex_lock(&lockA);                                 │
│                                                                     │
│          // 락 B를 논블로킹으로 시도                                       │
│          if (pthread_mutex_trylock(&lockB) == 0) {                   │
│              // 성공: 두 락 모두 획득                                     │
│              break;                                                  │
│          }                                                          │
│                                                                     │
│          // 실패: 락 A 해제하고 재시도                                     │
│          pthread_mutex_unlock(&lockA);                               │
│          usleep(random_backoff());                                  │
│      }                                                              │
│                                                                     │
│      // 작업 수행                                                      │
│      do_work();                                                      │
│                                                                     │
│      pthread_mutex_unlock(&lockB);                                   │
│      pthread_mutex_unlock(&lockA);                                   │
│  }                                                                  │
│                                                                     │
│  【두 단계 락킹 (Two-Phase Locking)】                                   │
│  ──────────────────                                                  │
│  // 데이터베이스 트랜잭션에서 사용                                          │
│  void two_phase_locking() {                                         │
│      // 1단계 (Growing): 락 획득만, 해제 없음                              │
│      lock(&record1);                                                 │
│      lock(&record2);                                                 │
│      lock(&record3);                                                 │
│                                                                     │
│      // 2단계 (Shrinking): 락 해제만, 획득 없음                             │
│      update(record1);                                               │
│      update(record2);                                               │
│      update(record3);                                               │
│                                                                     │
│      unlock(&record1);                                               │
│      unlock(&record2);                                               │
│      unlock(&record3);                                               │
│  }                                                                  │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전략             안전성    성능    구현 난이도    자원 효율           │ │
│  │  ────────         ──────   ──────   ────────     ────────          │ │
│  │  일괄 요청          높음     낮음     낮음          낮음              │ │
│  │  순서 규칙          높음     높음     중간          높음              │ │
│  │  tryLock           높음     중간     중간          중간              │ │
│  │  두 단계 락킹        높음     중간     높음          중간              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

```
• 개념: 4가지 조건 중 하나 무력화
```
