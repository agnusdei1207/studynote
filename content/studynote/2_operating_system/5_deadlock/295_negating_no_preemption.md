# 비선점 부정 (Negating No Preemption)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비선점 부정은 어떤 프로세스도 자원을 독점할 수 없게 하여, 다른 프로세스가 긴급하게 자원을 필요로 할 때 이미 점유된 자원을 강제로 뺏어올 수 있게 하는 전략이다.
> 2. **작동**: 자원을 보유한 프로세스가 즉시 할당받을 수 없는 새로운 자원을 요청하면, 현재 보유 중인 모든 자원을 스스로 반납(Release)하고 대기 상태로 전환된다.
> 3. **가치**: 데드락의 연쇄 대기 상태를 원천 차단하지만, CPU 레지스터나 메모리처럼 상태 저장이 쉬운 자원이 아닌 경우에는 오버헤드와 데이터 일관성 파괴 문제가 발생한다.

+++

## Ⅰ. 비선점 부정의 개념

### 1. 정의
- "자원을 할당받은 프로세스는 그 작업을 마칠 때까지 자원을 반납하지 않는다"는 비선점(No Preemption) 규칙을 깨고, 시스템이나 다른 프로세스에 의해 자원을 회수당할 수 있게 하는 기법이다.

### 2. 기본 원칙
- 요청한 자원을 지금 당장 쓸 수 없다면, 내가 가진 자원을 일단 내놓고 나중에 다시 처음부터 시작하거나, 내 자원을 기다리는 다른 프로세스에게 양보한다.

- **📢 섹션 요약 비유**: "놀이터에서 장난감을 가지고 놀다가 새로운 게 필요해졌는데 그게 없으면, 지금 가진 걸 다 내려놓고 줄을 다시 서는 것과 같습니다."

+++

## Ⅱ. 기술적 구현 매커니즘 및 ASCII 다이어그램

### 1. 자원 선점 시나리오 (Resource Preemption)
- 프로세스 A가 R1을 가진 상태에서 R2를 요청함.
- R2가 프로세스 B에 의해 점유 중이라면:
    1. 프로세스 A는 R1을 반납하고 대기 큐로 들어감.
    2. 또는, 시스템이 프로세스 B가 대기 중인지 확인하여 B의 자원을 뺏어 A에게 줄 수도 있음.

### 2. ASCII 작동 흐름
```text
    [ Process A ] (Holds: R1)
          │
    [ Requests: R2 ] ──▶ [ Is R2 Free? ] ── NO ──┐
                                                 │
    ┌────────────────────────────────────────────┘
    ▼
    [ Force Preemption Action ]
    1. A releases R1 (R1 becomes Free)
    2. A enters Wait State
    3. R1 can be used by other processes
```

### 3. 상태 저장 및 복구 (Save & Restore)
- 자원을 뺏기기 직전의 작업 상태(Checkpoint)를 안전하게 저장해야 나중에 재개할 수 있다.

- **📢 섹션 요약 비유**: "요리하다가 양념이 없어서 마트로 사러 가야 한다면, 가스레인지를 끄고 냄비를 내려놓아 다른 사람이 쓸 수 있게 자리를 비워주는 것입니다."

+++

## Ⅲ. 기술적 트레이드오프 분석

| 비교 항목 | 비선점 부정 (Prevention) | 비선점 유지 (Non-preemptive) |
|:---|:---|:---|
| **데드락 방지** | **탁월함** | 데드락 위험 높음 |
| **적용 가능 자원** | CPU, Memory (상태 저장 용이) | Disk, Printer (상태 저장 불가) |
| **성능 오버헤드** | 높음 (반복적인 상태 저장/복구) | 낮음 |
| **작업 속도** | 지연 가능성 높음 | 일정함 |

- **📢 섹션 요약 비유**: "빌린 책을 다 읽지 못했어도 다른 사람이 예약하면 즉시 반납해야 하는 도서관 시스템과 같아서, 공부 흐름이 자주 끊길 수 있습니다."

+++

## Ⅳ. 실무적 한계 및 적용 분야 (Strategy)

### 1. 상태 저장의 어려움
- 프린터와 같이 한 번 출력을 시작하면 중간에 멈출 수 없는 자원에는 적용이 불가능하다. (중간에 뺏기면 출력물이 엉망이 됨)

### 2. 오버헤드 이슈
- 뺏고 뺏기는 과정이 빈번하면 시스템은 실제 업무보다 자원 재할당 작업에 더 많은 시간을 쓰게 되는 '쓰레싱(Thrashing)'과 유사한 현상을 겪을 수 있다.

- **📢 섹션 요약 비유**: "붓글씨를 쓰다가 먹이 떨어졌다고 붓을 뺏기면, 나중에 다시 쓸 때 앞의 글씨와 획이 이어지지 않아 작품을 망칠 수 있는 것과 같습니다."

+++

## Ⅴ. 결론 및 향후 전망

### 1. 요약
- 비선점 부정은 자원 할당의 유연성을 극대화하여 데드락을 방지하는 강력한 수단이다.

### 2. 현대적 응용
- 가상 메모리(Virtual Memory) 환경에서 페이지 교체(Page Replacement)나 가상 머신(VM)에서의 리소스 동적 배분이 이 철학을 계승하고 있다. 또한, 프리엠프티브 커널(Preemptive Kernel)의 기초가 된다.

- **📢 섹션 요약 비유**: "내가 지금 최고가 아니라면 기꺼이 자리를 내주고 다음 기회를 기약하는 것이, 전체 공동체의 마비를 막는 현명한 선택일 수 있습니다."
