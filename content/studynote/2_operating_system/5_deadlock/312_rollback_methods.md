# 후퇴 (Rollback) 방식

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 후퇴(Rollback)는 교착 상태가 발생했을 때, 프로세스를 이전의 안전한 상태(Safe State)로 되돌려 자원을 반납하게 함으로써 데드락을 해소하는 기법이다.
> 2. **메커니즘**: 정기적으로 프로세스의 상태(레지스터, 메모리, 변수 등)를 저장하는 체크포인팅(Checkpointing)을 기반으로 하며, 필요 시 해당 지점부터 재실행한다.
> 3. **가치**: 프로세스를 완전히 종료(Abort)하지 않고도 작업을 재개할 수 있게 하여, 작업 손실을 최소화하고 시스템의 가용성을 높인다.

+++

## Ⅰ. 후퇴 방식의 기본 개념

### 1. 후퇴의 정의
- 교착 상태에 연루된 프로세스를 특정 시점으로 '타임머신'처럼 되돌리는 프로세스이다.

### 2. 전제 조건: 체크포인팅 (Checkpointing)
- 실행 중에 주기적으로 현재의 컨텍스트를 비휘발성 저장 장치에 기록해야 한다.

- **📢 섹션 요약 비유**: "게임을 하다가 보스 몬스터에게 졌을 때, 게임을 아예 새로 시작하는 게 아니라 마지막으로 저장된 '세이브 포인트'에서 다시 시작하는 것과 같습니다."

+++

## Ⅱ. 후퇴 프로세스 매핑 (ASCII Diagram)

### 1. 체크포인트와 후퇴 흐름
```text
 [ Process Execution Timeline ]
 
 Start ---- CP1 ---- CP2 ---- CP3 ---- DEADLOCK!
           (Safe)   (Safe)   (Safe)     (Error)
             |        |        |          |
             |<-------|<-------|<---------|
                 [ Rollback to CP3 ]
```

### 2. 후퇴 범위에 따른 분류
- **전체 후퇴 (Total Rollback)**: 프로세스를 처음부터 다시 시작 (가장 단순함).
- **부분 후퇴 (Partial Rollback)**: 가장 최근의 체크포인트로 복귀 (효율적이지만 구현 복잡).

- **📢 섹션 요약 비유**: "시험을 보다가 틀린 걸 발견했을 때, 시험지를 다 찢고 처음부터 푸는 게 '전체 후퇴', 틀린 문제만 지우개로 지우고 다시 푸는 게 '부분 후퇴'입니다."

+++

## Ⅲ. 후퇴의 기술적 구현 단계

### 1. 안전 상태 탐색 (Search for Safe State)
- 시스템이 교착 상태에 빠지지 않았던 가장 최근의 전역적 체크포인트를 찾는다.

### 2. 자원 회수 (Resource Reclaiming)
- 후퇴 대상 프로세스가 해당 시점 이후에 획득했던 모든 자원을 시스템으로 강제 반납한다.

### 3. 상태 복원 (State Restoration)
- 저장된 레지스터 값, 메모리 이미지, 파일 포인터 등을 복구하여 실행 준비를 마친다.

- **📢 섹션 요약 비유**: "실수로 엎지른 물을 닦아내고(자원 회수), 엎지르기 직전의 깨끗한 상태로 되돌리는 것과 같습니다."

+++

## Ⅳ. 후퇴 방식의 장단점 및 한계

| 비교 항목 | 상세 설명 | 비고 |
|:---|:---|:---|
| **장점** | 작업 손실 최소화, 시스템 회복력 강화 | 종료 방식보다 효율적 |
| **단점** | 체크포인팅 오버헤드, 저장 공간 필요 | 실행 속도 저하 요인 |
| **위험 요소** | 연쇄 후퇴 (Cascading Rollback) | 분산 시스템에서 심각 |

- **📢 섹션 요약 비유**: "매번 세이브를 하느라 게임이 중간중간 끊길 수는 있지만(오버헤드), 나중에 캐릭터가 죽었을 때 처음부터 안 해도 되는 안심 보험을 드는 셈입니다."

+++

## Ⅴ. 결론 및 실무적 교훈

### 1. 요약
- 후퇴 방식은 '완전 파괴' 대신 '부분 복구'를 선택하여 시스템의 연속성을 보장하는 지능적인 복구 전략이다.

### 2. 최종 결론
- 현대의 고신뢰성 시스템(DBMS, 금융 시스템)에서는 후퇴 방식이 필수적이며, 이는 단순한 오류 복구를 넘어 시스템의 결함 허용(Fault Tolerance) 능력을 결정짓는 핵심 요소이다.

- **📢 섹션 요약 비유**: "잘못된 길로 들어섰을 때 포기하지 않고, 아는 길까지 돌아가서 다시 시작하는 용기가 시스템을 살린다는 교훈을 줍니다."

### 📌 관련 개념 맵 (Knowledge Graph)
- **체크포인트 (Checkpoint)**: 후퇴를 가능하게 하는 데이터 저장 기점.
- **교착 상태 탐지 (Detection)**: 후퇴를 실행할 타이밍을 결정.
- **연쇄 후퇴 (Cascading Rollback)**: 한 프로세스의 후퇴가 다른 프로세스에 미치는 영향.
