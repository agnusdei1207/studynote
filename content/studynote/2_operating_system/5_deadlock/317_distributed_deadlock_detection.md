# 분산 시스템에서의 교착 상태 탐지 - 분산 교착 상태 그래프

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 시스템에서의 교착 상태 탐지는 여러 노드에 걸쳐 흩어진 프로세스 간의 대기 관계를 통합 분석하여, 네트워크 단위의 순환 대기(Circular Wait)를 식별하는 고난도 기술이다.
> 2. **메커니즘**: 개별 노드의 로컬 대기 그래프(Local WFG)를 조합하여 글로벌 대기 그래프(Global WFG)를 생성하거나, 에이전트 간 메시지 교환을 통해 순환 경로를 추적한다.
> 3. **가치**: 단일 시스템의 시야를 넘어 전역적인 자원 효율성을 확보하고, 네트워크 지연이나 메시지 유실 상황에서도 신뢰할 수 있는 데드락 판정을 제공한다.

+++

## Ⅰ. 분산 교착 상태의 특수성

### 1. 전역 상태 파악의 어려움
- 모든 노드의 상태를 한순간에 '스냅샷' 찍는 것이 불가능하므로, 시차가 존재한다.

### 2. 가상 교착 상태 (Phantom Deadlock)
- 실제로는 데드락이 아니지만, 상태 정보 전송 지연으로 인해 데드락으로 오인되는 현상이다.

- **📢 섹션 요약 비유**: "여러 나라에 흩어진 친구들이 서로 물건을 빌려달라고 편지를 보냈는데, 배달이 늦어져서 누가 무엇을 가졌는지 헷갈리는 상황과 같습니다."

+++

## Ⅱ. 분산 탐지 매핑 (ASCII Diagram)

### 1. 분산 대기 그래프 (Wait-For Graph)
```text
 [ Node A ]           [ Network ]           [ Node B ]
 
   P1 (Holds R1) ----> (Remote Req) ----> P2 (Holds R2)
    ^                                        |
    |                                        |
    +---- (Remote Resp) <---- (Wait R2) <----+
    
  * Global Cycle: P1 -> Node B(P2) -> Node A(P1)
```

### 2. 중앙 집중형 vs 분산형 탐지
- **중앙형**: 모든 정보를 중앙 코디네이터 노드로 전송하여 분석.
- **분산형**: 인접 노드끼리 정보를 교환하여 스스로 감지.

- **📢 섹션 요약 비유**: "한 선생님이 전교생의 일기장을 다 걷어서 확인하는 게 '중앙형', 반장들끼리 모여서 서로의 반 정보를 맞추는 게 '분산형'입니다."

+++

## Ⅲ. 주요 탐지 알고리즘

### 1. Chandy-Misra-Haas 알고리즘 (Edge Chasing)
- 프로세스가 자원을 요청할 때 '프로브(Probe)' 메시지를 던지고, 이 메시지가 자신에게 돌아오면 순환 대기로 판정한다.

### 2. 오버마쉬(Obermarck) 알고리즘
- 분산된 그래프 조각들을 특정한 가상 노드로 합쳐서 전역적인 일관성을 유지한다.

- **📢 섹션 요약 비유**: "동굴 탐험을 할 때 실타래를 풀고 들어가서, 그 실의 끝이 다시 내 손으로 돌아오면 길이 연결되었다는 걸 아는 것과 같습니다."

+++

## Ⅳ. 분산 시스템에서의 난제와 대응

| 문제점 | 상세 설명 | 해결 방안 |
|:---|:---|:---|
| **메시지 오버헤드** | 탐지를 위한 네트워크 트래픽 과다 | 탐지 주기 조절 및 점진적 탐지 |
| **일관성 결여** | 노드 간 시계(Clock) 불일치 | 논리적 시계(Lamport Clock) 사용 |
| **장애 내성** | 탐지 중 특정 노드 다운 | 타임아웃 및 리더 선출 알고리즘 도입 |

- **📢 섹션 요약 비유**: "국제 전화를 할 때 신호가 끊기거나 시차가 달라서 생기는 오해를 줄이기 위해 전 세계 공통 시간을 정해두는 것과 같습니다."

## Ⅴ. 결론 및 실무적 교훈

### 1. 요약
- 분산 교착 상태 탐지는 '부분적인 정보'로 '전역적인 진실'을 추론해 나가는 복잡한 퍼즐 맞추기이다.

### 2. 최종 결론
- 클라우드와 마이크로서비스 아키텍처(MSA)가 확산됨에 따라, 단일 DB 수준을 넘어 서비스 간의 자원 경합과 데드락을 관리하는 기술은 시스템 안정성의 핵심 역량이 되고 있다.

- **📢 섹션 요약 비유**: "멀리 떨어져 있어도 서로의 상황을 잘 공유해야 큰 사고를 막을 수 있다는 소통의 중요성을 일깨워줍니다."

### 📌 관련 개념 맵 (Knowledge Graph)
- **대기 그래프 (Wait-For Graph)**: 분산 탐지의 기본 데이터 구조.
- **논리적 시계 (Logical Clock)**: 메시지 순서를 보장하기 위한 필수 기술.
- **MSA (Microservices Architecture)**: 분산 데드락이 빈번히 발생하는 현대적 환경.
