# 프로세스 종료 방식 (Process Termination)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 종료(Process Termination)는 교착 상태를 해소하기 위해 연루된 프로세스를 강제로 중단시켜 점유 중인 자원을 회수하는 가장 확실한 복구 방법이다.
> 2. **유형**: 모든 연루 프로세스를 한꺼번에 종료하여 깨끗하게 해결하는 '전체 종료(Abort All)'와, 데드락이 풀릴 때까지 하나씩 종료하는 '순차적 종료(Sequential Abort)'로 나뉜다.
> 3. **가치**: 전체 종료는 빠르고 확실하지만 작업 손실이 매우 크고, 순차적 종료는 복잡하지만 손실을 최소화하려는 노력을 담고 있다.

+++

## Ⅰ. 전체 종료 방식 (Abort All Processes)

### 1. 정의
- 교착 상태 사이클에 포함된 모든 프로세스를 한꺼번에 강제 종료하는 방식이다.

### 2. 특징
- 구현이 매우 단순하고 확실하다.
- 작업 중이던 모든 데이터가 소실되므로 비용이 매우 비싸다. (예: 10시간 작업한 것이 다 날아감)

- **📢 섹션 요약 비유**: "누가 싸우는지 일일이 따지지 않고, 싸우는 무리 전체를 다 교실 밖으로 쫓아내는 것과 같습니다."

+++

## Ⅱ. 순차적 종료 방식 (Sequential Abort)

### 1. 정의
- 교착 상태가 해소될 때까지 한 번에 하나씩 프로세스를 종료하는 방식이다.

### 2. ASCII 비교 도식
```text
  [ Process A, B, C Deadlock ]
  
  [ Abort All ]           [ Sequential Abort ]
  1. Kill A, B, C         1. Kill C (Lowest Priority)
  2. Free all resources   2. Run Detection Algorithm
  3. All Restart          3. Deadlock still? -> Yes
                          4. Kill B
                          5. Run Detection Algorithm
                          6. Deadlock still? -> No
                          7. Done! (A is saved)
```

- **📢 섹션 요약 비유**: "실타래를 풀 때 가위로 다 잘라버리는 게 아니라, 매듭 하나를 자르고 풀리는지 보고 다시 하나를 더 자르는 것과 같습니다."

+++

## Ⅲ. 방식별 오버헤드 분석 (Strategy)

### 1. 전체 종료의 오버헤드
- **재실행 비용**: 종료된 모든 프로세스를 처음부터 다시 시작해야 하는 막대한 시간 낭비.

### 2. 순차적 종료의 오버헤드
- **반복 검사 비용**: 프로세스를 하나 종료할 때마다 탐지 알고리즘을 다시 돌려야 하므로 탐지 오버헤드가 누적됨.

- **📢 섹션 요약 비유**: "가게 문을 닫고 청소를 다 할지, 손님을 한 명씩 내보내면서 장사를 계속할지 고민하는 가게 주인과 같습니다."

+++

## Ⅳ. 종료 방식의 기술적 선택 (Trade-off)

| 비교 항목 | 전체 종료 (Abort All) | 순차적 종료 (Sequential Abort) |
|:---|:---|:---|
| **복잡성** | 매우 낮음 | 높음 |
| **작업 보존** | 거의 불가능 | 부분적 가능 |
| **탐지 빈도** | 1회 | 매 종료 시마다 수행 |
| **확실성** | 100% | 100% (언젠가는 풀림) |

- **📢 섹션 요약 비유**: "전쟁터에서 후퇴할 때 전군이 동시에 물러나는 것과, 후방 부대부터 천천히 물러나는 전략적 차이입니다."

+++

## Ⅴ. 결론 및 실무적 교훈

### 1. 요약
- 전체 종료는 '단순함'의 가치를, 순차적 종료는 '효율성'의 가치를 추구한다.

### 2. 최종 결론
- 시스템의 성격에 따라 선택이 달라진다. 데이터 정합성이 중요한 시스템은 순차적 종료를 선호하지만, 응답성이 최우선인 실시간 시스템은 전체 종료 후 빠른 재부팅을 택하기도 한다.

- **📢 섹션 요약 비유**: "병을 치료할 때 몸 전체를 소독하느냐, 아픈 부위만 찾아내어 수술하느냐의 차이라는 교훈을 줍니다."
