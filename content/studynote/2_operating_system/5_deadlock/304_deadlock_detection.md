# 교착 상태 탐지 (Deadlock Detection)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착 상태 탐지(Deadlock Detection)는 시스템이 교착 상태에 빠지는 것을 막지 않고, 주기적으로 상태를 점검하여 교착 상태가 발생했는지를 찾아내는 사후 점검 기법이다.
> 2. **방법**: 자원 할당 그래프를 분석하여 '대기 사이클'이 존재하는지 확인하거나, 은행원 알고리즘과 유사한 행렬 계산을 통해 자원 요청을 만족시킬 수 없는 프로세스 집합을 추출한다.
> 3. **가치**: 회피 알고리즘보다 훨씬 완화된 조건에서 작동하여 시스템 자원 활용도를 높일 수 있지만, 탐지 시 발생하는 오버헤드와 복구 비용 사이의 트레이드오프가 중요하다.

+++

## Ⅰ. 교착 상태 탐지의 기본 개념

### 1. 정의
- 교착 상태가 일어날 가능성을 열어두고, 실제로 발생했을 때 이를 인지하기 위해 시스템 상태를 분석하는 과정이다.

### 2. 특징
- 예방이나 회피와 달리 자원 요청 시 특별한 제한을 두지 않는다. 따라서 시스템은 최고 성능으로 돌아가지만, 데드락 발생 시 처리 비용이 발생한다.

- **📢 섹션 요약 비유**: "사고가 날까 봐 차를 못 가게 막는 대신, 도처에 CCTV를 달아서 사고가 나면 즉시 출동하겠다고 지켜보는 것과 같습니다."

+++

## Ⅱ. 탐지 알고리즘의 작동 원리 (ASCII)

### 1. 단일 인스턴스 환경
- 대기 그래프(Wait-for Graph)에서 사이클을 찾는 것으로 충분하다.

### 2. 다중 인스턴스 환경
- 가용 자원(Available), 할당량(Allocation), 현재 요청량(Request)을 사용하여 더 이상 진행 가능한 프로세스가 없는지 계산한다.

```text
 [ Detection Algorithm ]
 
 Step 1: Work = Available (현재 남은 자원)
 Step 2: Allocation != 0 인 프로세스 중 
         Request <= Work 인 Pi 탐색
 Step 3: Pi를 찾으면 Finish[i] = true,
         Work = Work + Allocation (자원 회수 시뮬레이션)
 Step 4: Finish 가 false인 프로세스가 남았다면? 
         -> [ DEADLOCK DETECTED! ]
```

- **📢 섹션 요약 비유**: "가진 돈으로 물건을 살 수 있는 손님들을 한 명씩 내보내 보고, 돈은 있는데 아무도 물건을 못 사는 막다른 상황이 오는지 체크하는 것과 같습니다."

+++

## Ⅲ. 탐지 시점과 빈도 (Strategy)

### 1. 수행 빈도 결정 요소
- 얼마나 자주 교착 상태가 발생하는가?
- 교착 상태에 의해 영향을 받는 프로세스가 얼마나 많은가?

### 2. 실행 시점
- 자원 요청이 거부될 때마다 실행 (가장 철저하지만 오버헤드 큼).
- 주기적으로 실행 (예: 1시간마다).
- CPU 이용률이 특정 수준 이하로 떨어질 때 실행.

- **📢 섹션 요약 비유**: "병원에 매일 가서 검진을 받을지, 몸이 아플 때만 갈지 정하는 건강검진 주기 설정과 같습니다."

+++

## Ⅳ. 탐지 알고리즘의 비교 (Technical Comparison)

| 구분 | 단일 인스턴스 | 다중 인스턴스 |
|:---|:---|:---|
| **도구** | 대기 그래프 (Wait-for Graph) | 행렬 연산 (Detection Algorithm) |
| **판단 기준** | 사이클 형성 여부 | 안전 순서 존재 여부 |
| **복잡도** | $O(N^2)$ | $O(M \times N^2)$ |

- **📢 섹션 요약 비유**: "줄이 엉켰는지 눈으로 슥 보는 것과, 장부를 하나하나 대조해서 돈이 빈 곳을 찾는 정밀 회계 감사의 차이입니다."

## Ⅴ. 결론 및 실무적 교훈

### 1. 요약
- 탐지는 '자원 활용의 자유'를 주는 대신 '정기적인 감시'를 수행하는 합리적인 절충안이다.

### 2. 최종 결론
- 시스템 부하가 적고 데드락 발생 확률이 낮은 환경에서는 회피보다 탐지 전략이 훨씬 경제적이다. 단, 탐지된 데드락을 어떻게 복구할 것인가에 대한 대책이 반드시 수반되어야 한다.

- **📢 섹션 요약 비유**: "자유를 주는 대신 법과 경찰을 두어 문제가 생기면 해결하는 민주적인 관리 방식이 더 효율적일 때가 많다는 교훈을 줍니다."
