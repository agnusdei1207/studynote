# 305. 안전 순서 (Safe Sequence)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 프로세스를 완료시키는 순서
> 2. **가치**: 안전 상태의 증명
> 3. **융합**: 은행원 알고리즘, 안전 상태와 연관

---

## Ⅰ. 개요

### 개념 정의

안전 순서(Safe Sequence)는 **모든 프로세스가 순차적으로 완료될 수 있는 실행 순서**다. 이 순서가 존재하면 시스템은 안전 상태다.

### 💡 비유: 대출 상환 순서
안전 순서는 **대출 상환 순서**와 같다. 고객들을 특정 순서로 부르면, 모든 고객이 대출을 갚고 은행도 안전하다.

### 안전 순서 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                안전 순서 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정의】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  순서 <P1, P2, ..., Pn>이 안전 순서이려면:                              │ │
│  │                                                             │ │
│  │  각 Pi에 대해:                                                       │ │
│  │  Need[i] ≤ Available + Σ(Allocation[Pj]) where j < i              │ │
│  │                                                             │ │
│  │  즉: Pi의 필요 자원 ≤ 현재 가용 + 앞선 프로세스들이 보유한 자원             │ │
│  │                                                             │ │
│  │  의미: 앞선 프로세스들이 완료되면 Pi도 완료할 수 있음                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시각화】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  초기 Available: ████ (4)                                          │ │
│  │                                                             │ │
│  │  1단계: P1 완료                                                      │
│  │     Need[P1] ≤ 4 ✓                                                 │
│  │     P1의 Allocation 반환                                            │ │
│  │     Available: ████████ (8)                                        │ │
│  │                                                             │ │
│  │  2단계: P2 완료                                                      │
│  │     Need[P2] ≤ 8 ✓                                                 │
│  │     P2의 Allocation 반환                                            │ │
│  │     Available: ████████████ (12)                                   │ │
│  │                                                             │ │
│  │  3단계: P3 완료                                                      │
│  │     Need[P3] ≤ 12 ✓                                                │
│  │     모든 프로세스 완료!                                                │ │
│  │                                                             │ │
│  │  안전 순서: <P1, P2, P3>                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예제】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │           Allocation    Max       Need       Available         │ │
│  │           A  B          A  B     A  B       A  B               │ │
│  │  P0      1  2          4  4     3  2                          │ │
│  │  P1      0  1          2  2     2  1         2  1              │ │
│  │  P2      1  0          3  1     2  1                          │ │
│  │                                                             │ │
│  │  안전 순서 찾기:                                                       │ │
│  │  1. Available = [2, 1]                                              │ │
│  │  2. P1: Need[2,1] ≤ Available[2,1] ✓                               │ │
│  │     완료 → Available = [2,1] + [0,1] = [2,2]                        │ │
│  │  3. P2: Need[2,1] ≤ Available[2,2] ✓                               │ │
│  │     완료 → Available = [2,2] + [1,0] = [3,2]                        │ │
│  │  4. P0: Need[3,2] ≤ Available[3,2] ✓                               │ │
│  │     완료 → Available = [3,2] + [1,2] = [4,4]                        │ │
│  │                                                             │ │
│  │  안전 순서: <P1, P2, P0>                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                안전 순서 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【안전 순서 찾기 알고리즘】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Work = Available                                               │ │
│  │  2. Finish[i] = false (모든 i)                                      │ │
│  │  3. 반복:                                                            │ │
│  │     a. 다음 조건을 만족하는 i 찾기:                                    │ │
│  │        !Finish[i] && Need[i] ≤ Work                                │ │
│  │     b. 찾으면:                                                       │ │
│  │        Work = Work + Allocation[i]                                 │ │
│  │        Finish[i] = true                                             │ │
│  │        i를 안전 순서에 추가                                            │ │
│  │        a로 돌아가기                                                   │ │
│  │     c. 못 찾으면 종료                                                 │ │
│  │  4. 모든 Finish[i] = true면 안전 순서 존재                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【특성】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 유일하지 않음:                                                    │ │
│  │     • 여러 안전 순서가 존재할 수 있음                                    │ │
│  │     • 하나만 찾아도 충분                                               │ │
│  │                                                             │ │
│  │  2. 동적 변경:                                                        │ │
│  │     • 자원 할당/해제에 따라 안전 순서 변화                                │ │
│  │     • 매 요청마다 재계산 필요                                           │ │
│  │                                                             │ │
│  │  3. 존재 의미:                                                        │ │
│  │     • 안전 순서 존재 = 안전 상태                                        │ │
│  │     • 교착상태 불가능                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【복잡도】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시간 복잡도: O(m × n²)                                               │ │
│  │  • m: 자원 유형 수                                                    │ │
│  │  • n: 프로세스 수                                                     │ │
│  │  • 각 단계에서 모든 프로세스 검사                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【안전 순서 찾기 구현】                                                 │
│  ──────────────────                                                  │
│  class SafeSequenceFinder {                                         │
│      int[] available;                                                │
│      int[][] allocation;                                             │
│      int[][] need;                                                   │
│                                                                     │
│      List<Integer> findSafeSequence() {                             │
│          int n = allocation.length;                                 │
│          int m = available.length;                                  │
│                                                                     │
│          int[] work = available.clone();                            │
│          boolean[] finish = new boolean[n];                         │
│          List<Integer> sequence = new ArrayList<>();                │
│                                                                     │
│          int count = 0;                                              │
│          while (count < n) {                                         │
│              boolean found = false;                                  │
│                                                                     │
│              for (int i = 0; i < n; i++) {                           │
│                  if (!finish[i] && lessOrEqual(need[i], work)) {     │
│                      // Pi를 안전 순서에 추가                             │
│                      for (int j = 0; j < m; j++) {                    │
│                          work[j] += allocation[i][j];                │
│                      }                                              │
│                      finish[i] = true;                               │
│                      sequence.add(i);                                │
│                      found = true;                                   │
│                      count++;                                        │
│                  }                                                  │
│              }                                                      │
│                                                                     │
│              if (!found) {                                           │
│                  // 불안전 상태: 안전 순서 없음                             │
│                  return null;                                        │
│              }                                                      │
│          }                                                          │
│                                                                     │
│          return sequence;  // 안전 순서 반환                             │
│      }                                                              │
│                                                                     │
│      private boolean lessOrEqual(int[] a, int[] b) {                │
│          for (int i = 0; i < a.length; i++) {                        │
│              if (a[i] > b[i]) return false;                         │
│          }                                                          │
│          return true;                                                │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【사용 예시】                                                           │
│  ──────────────────                                                  │
│  SafeSequenceFinder finder = new SafeSequenceFinder(               │
│      available, allocation, need                                    │
│  );                                                                 │
│                                                                     │
│  List<Integer> safeSeq = finder.findSafeSequence();                 │
│  if (safeSeq != null) {                                              │
│      System.out.println("Safe sequence: " + safeSeq);               │
│      // 예: Safe sequence: [1, 3, 0, 4, 2]                          │
│  } else {                                                            │
│      System.out.println("System in unsafe state!");                 │
│  }                                                                  │
│                                                                     │
│  【자원 요청 처리】                                                      │
│  ──────────────────                                                  │
│  boolean processRequest(int pid, int[] request) {                   │
│      // 시험적 할당                                                      │
│      for (int i = 0; i < m; i++) {                                  │
│          available[i] -= request[i];                                │
│          allocation[pid][i] += request[i];                          │
│          need[pid][i] -= request[i];                                │
│      }                                                              │
│                                                                     │
│      // 안전 순서 존재 확인                                               │
│      List<Integer> seq = findSafeSequence();                        │
│      if (seq != null) {                                              │
│          System.out.println("Request granted. Safe seq: " + seq);   │
│          return true;  // 승인                                        │
│      }                                                              │
│                                                                     │
│      // 롤백                                                           │
│      for (int i = 0; i < m; i++) {                                  │
│          available[i] += request[i];                                │
│          allocation[pid][i] -= request[i];                          │
│          need[pid][i] += request[i];                                │
│      }                                                              │
│      return false;  // 거부                                           │
│  }                                                                  │
│                                                                     │
│  【다양한 안전 순서】                                                     │
│  ──────────────────                                                  │
│  // 같은 상태에서 여러 안전 순서 가능                                        │
│  // 예: <P1, P0, P2>, <P1, P2, P0>, <P2, P1, P0>                    │
│  // 하나만 존재해도 안전 상태임                                              │
│  // 실제 실행 순서와는 무관 (분석용)                                         │
│                                                                     │
│  【디버깅 활용】                                                          │
│  ──────────────────                                                  │
│  void debugSystemState() {                                          │
│      List<Integer> seq = findSafeSequence();                        │
│      if (seq == null) {                                              │
│          log.error("UNSAFE STATE DETECTED");                        │
│          logResourceState();                                        │
│          alerting.critical("System in unsafe state");               │
│      } else {                                                        │
│          log.info("Safe sequence available: " + seq);               │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【성능 고려사항】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  연산              복잡도          빈도                           │ │
│  │  ────────          ──────          ────                           │ │
│  │  안전 순서 찾기      O(m × n²)       모든 요청 시                     │ │
│  │  상태 갱신          O(m)            모든 요청/해제 시                  │ │
│  │                                                             │ │
│  │  고빈도 시스템에서는 오버헤드 문제                                        │ │
│  │  → 근사 알고리즘 또는 주기적 검사 사용                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 모든 프로세스 완료 가능한 순서
• 조건: Need[i] ≤ Available + 앞선 할당
• 의미: 존재 = 안전 상태
• 특성: 유일하지 않음, 동적 변경
• 복잡도: O(m × n²)
• 활용: 회피 알고리즘, 디버깅
```
