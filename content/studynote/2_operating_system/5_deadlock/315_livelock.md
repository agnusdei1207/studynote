# 288. 라이브락 (Livelock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스레드가 계속 실행되지만 진전 없음
> 2. **가치**: 데드락의 특수 형태, 감지 어려움
> 3. **융합**: 교착상태, 락 타임아웃, 백오프와 연관

---

## Ⅰ. 개요

### 개념 정의

라이브락(Livelock)은 **스레드들이 계속 실행되지만 실제 작업을 진전시키지 못하는 상태**다. 서로에게 양보하려 계속 반응하지만 아무도 진행하지 못한다.

### 💡 비유: 복도 좁은길
라이브락은 **복도에서 서로 양보하려는 두 사람**과 같다. 왼쪽으로 비키고, 오른쪽으로 비키고, 계속 반복. 둘 다 움직이지만 지나가지 못한다.

### 라이브락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                라이브락 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시나리오】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  T1: "T2에게 양보" → 뒤로 이동 → "T2에게 양보" → 뒤로 이동 ...     │ │
│  │                                                             │ │
│  │  T2: "T1에게 양보" → 뒤로 이동 → "T1에게 양보" → 뒤로 이동 ...     │ │
│  │                                                             │ │
│  │  두 스레드 모두 활성 상태!                                           │ │
│  │  CPU 사용 중!                                                      │ │
│  │  하지만 진전 없음!                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【코드 예시】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  // T1                                                            │ │
│  │  while (!resource1.available()) {                                 │
│  │      Thread.yield();  // 양보                                      │ │
│  │  }                                                              │ │
│  │                                                             │ │
│  │  // T2                                                            │ │
│  │  while (!resource2.available()) {                                 │ │
│  │      Thread.yield();  // 양보                                      │ │
│  │  }                                                              │ │
│  │                                                             │ │
│  │  // 서로가 서로의 리소스를 기다리며 계속 양보                           │ │
│  │  // CPU는 계속 사용하지만 진전 없음                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 데드락】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데드락:                                                            │ │
│  │  • 스레드가 멈춤 (blocked)                                          │ │
│  │  • CPU 사용 안 함                                                   │ │
│  │  • 감지 비교적 쉬움                                                  │ │
│  │                                                             │ │
│  │  라이브락:                                                           │ │
│  │  • 스레드가 계속 실행 (running)                                      │ │
│  │  • CPU 사용함                                                       │ │
│  │  • 감지 어려움                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                라이브락 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【발생 원인】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 과도한 양보 (Excessive Yielding):                              │ │
│  │     • 백오프 알고리즘에서 동기화된 대기                               │ │
│  │                                                             │ │
│  │  2. 메시지 패싱 시스템:                                              │ │
│  │     • 서로가 서로의 메시지를 계속 처리                                 │ │
│  │                                                             │ │
│  │  3. tryLock 루프:                                                  │ │
│  │     • 락 획득 실패 시 즉시 재시도                                     │ │
│  │                                                             │ │
│  │  4. 분산 시스템:                                                    │ │
│  │     • 코디네이터 실패 후 복구 시도                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【감지 방법】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 진전 없는 CPU 사용                                               │ │
│  │  • 상태 변화 없는 반복                                               │ │
│  │  • 타임아웃 후에도 변화 없음                                          │ │
│  │  • 로그에서 패턴 분석                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결 방법】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 랜덤 백오프:                                                    │ │
│  │     • 재시도 간격을 무작위화                                          │ │
│  │                                                             │ │
│  │  2. 타임아웃:                                                       │ │
│  │     • 무한 루프 방지                                                │ │
│  │                                                             │ │
│  │  3. 우선순위:                                                       │ │
│  │     • 한 스레드에 우선권 부여                                        │ │
│  │                                                             │ │
│  │  4. 상태 추적:                                                      │ │
│  │     • 진전 여부 확인                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【라이브락 발생 코드】                                                 │
│  ──────────────────                                                  │
│  // 나쁜 예: 라이브락 가능                                              │
│  while (true) {                                                      │
│      if (tryLock(lockA)) {                                           │
│          if (tryLock(lockB)) {                                       │
│              // 작업                                                  │
│              unlock(lockB);                                          │
│              unlock(lockA);                                          │
│              break;                                                  │
│          }                                                          │
│          unlock(lockA);  // B 실패 시 A 해제                           │
│      }                                                              │
│      // 즉시 재시도 → 라이브락 가능                                      │
│  }                                                                  │
│                                                                     │
│  【해결: 랜덤 백오프】                                                  │
│  ──────────────────                                                  │
│  Random random = new Random();                                      │
│  while (true) {                                                      │
│      if (tryLock(lockA)) {                                           │
│          if (tryLock(lockB)) {                                       │
│              // 작업                                                  │
│              unlock(lockB);                                          │
│              unlock(lockA);                                          │
│              break;                                                  │
│          }                                                          │
│          unlock(lockA);                                              │
│      }                                                              │
│      // 랜덤 대기로 라이브락 방지                                        │
│      Thread.sleep(random.nextInt(100));                             │
│  }                                                                  │
│                                                                     │
│  【해결: 지수 백오프】                                                  │
│  ──────────────────                                                  │
│  int backoff = 1;                                                    │
│  while (true) {                                                      │
│      if (tryLock(lockA)) {                                           │
│          if (tryLock(lockB)) {                                       │
│              // 작업                                                  │
│              unlock(lockB);                                          │
│              unlock(lockA);                                          │
│              break;                                                  │
│          }                                                          │
│          unlock(lockA);                                              │
│      }                                                              │
│      Thread.sleep(backoff);                                          │
│      backoff = Math.min(backoff * 2, MAX_BACKOFF);                   │
│  }                                                                  │
│                                                                     │
│  【분산 시스템: 코디네이터 선출】                                         │
│  ──────────────────                                                  │
│  // 라이브락 가능: 여러 노드가 동시에 선출 시도                             │
│  // 해결: 랜덤 대기 후 재시도                                            │
│                                                                     │
│  void tryBecomeLeader() {                                           │
│      while (!leaderElected) {                                        │
│          if (proposeSelfAsLeader()) {                                │
│              if (getAcceptance()) {                                  │
│                  becomeLeader();                                     │
│                  return;                                             │
│              }                                                      │
│          }                                                          │
│          // 랜덤 대기                                                  │
│          sleep(random.nextInt(1000));                               │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【라이브락 감지】                                                      │
│  ──────────────────                                                  │
│  class LivelockDetector {                                           │
│      int lastProgress = 0;                                          │
│      int checkCount = 0;                                            │
│                                                                     │
│      boolean checkProgress() {                                      │
│          int current = getWorkDone();                               │
│          if (current == lastProgress) {                             │
│              checkCount++;                                          │
│              if (checkCount > 10) {                                 │
│                  log.warn("Possible livelock detected!");            │
│                  return true;  // 라이브락 감지                          │
│              }                                                      │
│          } else {                                                    │
│              checkCount = 0;                                        │
│          }                                                          │
│          lastProgress = current;                                    │
│          return false;                                              │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전략           CPU 사용    라이브락 가능    해결 시간              │ │
│  │  ────────       ──────      ────────────    ────────            │ │
│  │  즉시 재시도      높음        높음           불가능                 │ │
│  │  고정 대기        중간        낮음           느림                   │ │
│  │  랜덤 백오프       중간        없음           빠름                   │ │
│  │  지수 백오프       낮음        없음           보통                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예방 가이드라인】                                                    │
│  ──────────────────                                                  │
│  • tryLock 루프: 항상 백오프 포함                                       │
│  • 양보: 랜덤 요소 추가                                                 │
│  • 타임아웃: 무한 루프 방지                                              │
│  • 진전 추적: 주기적으로 확인                                             │
│  • 로그: 상태 변화 기록                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 실행 중이지만 진전 없는 상태
• 특징: CPU 사용, 감지 어려움
• 원인: 동기화된 양보, tryLock 루프
• 해결: 랜덤 백오프, 타임아웃
• 감지: 진전 추적, 상태 변화
• 예방: 랜덤 요소, 지수 백오프
```
