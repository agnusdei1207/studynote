# 304. 불안전 상태 (Unsafe State)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 안전 순서가 존재하지 않는 상태
> 2. **가치**: 교착상태 가능성을 경고
> 3. **융합**: 안전 상태, 교착상태와 연관

---

## Ⅰ. 개요

### 개념 정의

불안전 상태(Unsafe State)는 **모든 프로세스를 완료시킬 수 있는 안전 순서가 존재하지 않는 상태**다. 교착상태가 발생할 "가능성"이 있는 상태다.

### 💡 비유: 위험한 자금 상태
불안전 상태는 **위험한 자금 상태**와 같다. 모든 고객이 최대 한도를 요청하면 지급할 수 없다. 아직 망하지 않았지만 위험하다.

### 불안전 상태 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                불안전 상태 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정의】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  불안전 상태 = 안전 순서가 존재하지 않는 상태                              │ │
│  │                                                             │ │
│  │  특징:                                                               │ │
│  │  • 어떤 순서로도 모든 프로세스를 완료시킬 수 없음                          │ │
│  │  • 일부 프로세스의 요청이 현재 가용 자원으로 충족 불가                       │ │
│  │  • 다른 프로세스가 완료되어도 해결되지 않을 수 있음                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상태 관계】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  안전 상태 ──요청──► 불안전 상태 ──악화──► 교착상태                        │ │
│  │     ↑                  │                     │                      │
│  │     │                  │                     │                      │
│  │     └── 자원 해제 ◄────┴─────── 복구 ◄────────┘                      │
│  │                                                             │ │
│  │  중요: 불안전 상태 ≠ 교착상태                                             │ │
│  │  • 불안전: 교착상태 "가능성"                                             │ │
│  │  • 교착상태: 실제로 발생한 상태                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예시】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템: A=12개, B=2개의 자원                                          │ │
│  │                                                             │ │
│  │  프로세스   최대(A)    할당(A)    Need(A)                            │ │
│  │  ────────   ──────     ──────     ──────                             │ │
│  │  P0         10         5           5                                │ │
│  │  P1         4          2           2                                │ │
│  │  P2         9          2           7                                │ │
│  │                                                             │ │
│  │  총 할당: 9, Available: 3                                          │ │
│  │                                                             │ │
│  │  안전 순서 시도:                                                       │ │
│  │  • P0: Need=5 > Available=3 ✗                                      │ │
│  │  • P1: Need=2 ≤ Available=3 ✓ → 완료 → Available=5                  │ │
│  │  • P0: Need=5 ≤ Available=5 ✓ → 완료 → Available=10                 │ │
│  │  • P2: Need=7 ≤ Available=10 ✓ → 완료                               │ │
│  │                                                             │ │
│  │  안전 순서 존재: <P1, P0, P2> → 안전 상태!                             │ │
│  │                                                             │ │
│  │  만약 P0가 추가로 1개 요청:                                              │ │
│  │  Available=2, P0 Need=4, P1 Need=2, P2 Need=7                       │ │
│  │  • P1: Need=2 ≤ Available=2 ✓ → 완료 → Available=4                  │ │
│  │  • P0: Need=4 ≤ Available=4 ✓ → 완료 → Available=9                  │ │
│  │  • P2: Need=7 ≤ Available=9 ✓                                       │ │
│  │  여전히 안전!                                                          │ │
│  │                                                             │ │
│  │  만약 P0가 추가로 2개 더 요청 (총 8개):                                  │ │
│  │  Available=1, P0 Need=2, P1 Need=2, P2 Need=7                       │ │
│  │  모든 Need > Available → 불안전 상태!                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                불안전 상태 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【불안전 상태의 의미】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 교착상태 가능성:                                                   │ │
│  │     • 불안전 상태에서는 교착상태가 "발생할 수 있음"                        │ │
│  │     • 반드시 발생하는 것은 아님                                          │ │
│  │                                                             │ │
│  │  2. 예측 불가:                                                        │ │
│  │     • 실제 요청 패턴에 따라 교착상태 발생 여부 결정                        │ │
│  │     • 모든 프로세스가 최대치를 요청하면 발생                                │ │
│  │                                                             │ │
│  │  3. 회피의 기준:                                                      │ │
│  │     • 불안전 상태로 가는 요청은 거부                                      │ │
│  │     • 항상 안전 상태 유지가 목표                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【안전 → 불안전 → 교착상태 전이】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 안전 상태에서 시작                                                 │ │
│  │  2. 자원 요청 수락 → 불안전 상태로 전이 가능                              │ │
│  │  3. 불안전 상태에서 추가 요청 → 교착상태 발생 가능                         │ │
│  │  4. 교착상태 → 복구 필요                                               │ │
│  │                                                             │ │
│  │  전이를 막는 것이 교착상태 회피                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실제 vs 이론적】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  이론적 (최대 요청 기준):                                              │ │
│  │  • 모든 프로세스가 최대 필요량을 요청한다고 가정                            │ │
│  │  • 매우 보수적                                                        │ │
│  │                                                             │ │
│  │  실제:                                                                │ │
│  │  • 프로세스는 보통 최대보다 적게 요청                                     │ │
│  │  • 불안전 상태에서도 교착상태가 안 발생할 수 있음                           │ │
│  │  • 하지만 보장을 위해 보수적으로 판단                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【불안전 상태 감지】                                                    │
│  ──────────────────                                                  │
│  boolean isUnsafeState(int[] available, int[][] allocation,         │
│                        int[][] need) {                              │
│      int n = allocation.length;                                     │
│      int m = available.length;                                      │
│                                                                     │
│      int[] work = available.clone();                                │
│      boolean[] finish = new boolean[n];                             │
│                                                                     │
│      boolean changed = true;                                        │
│      while (changed) {                                              │
│          changed = false;                                           │
│          for (int i = 0; i < n; i++) {                               │
│              if (!finish[i] && canSatisfy(need[i], work)) {          │
│                  for (int j = 0; j < m; j++) {                        │
│                      work[j] += allocation[i][j];                    │
│                  }                                                  │
│                  finish[i] = true;                                   │
│                  changed = true;                                     │
│              }                                                      │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      // 하나라도 false가 있으면 불안전                                    │
│      for (boolean f : finish) {                                      │
│          if (!f) return true;  // 불안전 상태                           │
│      }                                                              │
│      return false;  // 안전 상태                                       │
│  }                                                                  │
│                                                                     │
│  【불안전 상태에서의 결정】                                                │
│  ──────────────────                                                  │
│  // 옵션 1: 요청 거부 (회피)                                              │
│  if (wouldCauseUnsafe(request)) {                                   │
│      return REJECT;  // 대기                                          │
│  }                                                                  │
│                                                                     │
│  // 옵션 2: 요청 승인 (위험 감수)                                          │
│  if (wouldCauseUnsafe(request)) {                                   │
│      log.warn("Warning: Entering unsafe state");                    │
│      // 승인하지만 교착상태 가능성 존재                                      │
│  }                                                                  │
│                                                                     │
│  // 옵션 3: 타임아웃으로 복구                                              │
│  // 불안전 상태 진입 후 일정 시간 내 해제 없으면                                │
│  // 롤백 수행                                                           │
│                                                                     │
│  【실제 시스템에서의 접근】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템            접근 방식                                           │ │
│  │  ────────          ────────                                           │ │
│  │  일반 OS           불안전 상태 허용 + 교착상태 시 복구                       │ │
│  │  실시간 시스템       불안전 상태 완전 거부                                   │ │
│  │  데이터베이스        타임아웃으로 자동 복구                                  │ │
│  │  클라우드           자동 확장으로 자원 추가                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【모니터링】                                                            │
│  ──────────────────                                                  │
│  // 불안전 상태 감지 시 알림                                               │
│  if (isUnsafeState()) {                                             │
│      metrics.increment("unsafe_state_count");                       │
│      alerting.warning("System in unsafe state");                    │
│      // 로그에 관련 프로세스 정보 기록                                      │
│  }                                                                  │
│                                                                     │
│  【성능 vs 안전성 트레이드오프】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전략                안전성      처리량     지연                     │ │
│  │  ────────            ──────     ──────     ──────                   │ │
│  │  불안전 상태 완전 거부    높음       낮음       높음                     │ │
│  │  불안전 상태 허용       중간       높음       낮음                     │ │
│  │  탐지 + 복구          중간       중간       중간                     │ │
│  │                                                             │ │
│  │  대부분 시스템: 불안전 허용 + 교착상태 시 복구                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```
• 개념: 안전 순서가 없는 상태
```
