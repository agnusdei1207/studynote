# 296. 교착상태 복구 (Deadlock Recovery)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착상태를 해결하고 시스템을 정상화
> 2. **가치**: 시스템 가용성 복원
> 3. **융합**: 희생자 선택, 롤백, 프로세스 종료와 연관

---

## Ⅰ. 개요

### 개념 정의

교착상태 복구(Deadlock Recovery)는 **발생한 교착상태를 해결하고 시스템을 정상 상태로 되돌리는 과정**이다. 탐지 후에 수행된다.

### 💡 비유: 구조 작업
교착상태 복구는 **조난자 구조**와 같다. 문제를 발견했으니 이제 해결한다. 일부를 희생시키더라도 전체를 구한다.

### 복구 전략 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                교착상태 복구 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【복구 방법】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 프로세스 종료 (Process Termination):                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 교착상태 프로세스 모두 종료                                  │ │ │
│  │  │  • 하나씩 종료하며 교착상태 해제 확인                            │ │ │
│  │  │  • 가장 급진적, 단순함                                         │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  2. 자원 선점 (Resource Preemption):                                 │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 일부 프로세스에서 자원 강제 회수                              │ │ │
│  │  │  • 회수한 자원을 다른 프로세스에 할당                            │ │ │
│  │  │  • 더 정교함, 복잡함                                           │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프로세스 종료 전략】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 모두 종료:                                                      │ │
│  │     • 교착상태에 있는 모든 프로세스 종료                                │ │
│  │     • 확실하지만 비용 큼                                              │ │
│  │                                                             │ │
│  │  2. 하나씩 종료:                                                     │ │
│  │     • 하나를 종료하고 교착상태 확인                                    │ │
│  │     • 해제될 때까지 반복                                              │ │
│  │     • 더 효율적                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【자원 선점 시 고려사항】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 희생자 선택 (Victim Selection):                                  │ │
│  │     • 최소 비용으로 해결할 프로세스 선택                                │ │
│  │                                                             │ │
│  │  2. 롤백 (Rollback):                                                │ │
│  │     • 선점된 프로세스를 이전 상태로 복구                                │ │
│  │                                                             │ │
│  │  3. 기아 방지 (Starvation Prevention):                              │ │
│  │     • 같은 프로세스가 계속 희생되지 않도록                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                교착상태 복구 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【희생자 선택 기준】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  고려 요소:                                                          │ │
│  │  • 프로세스 우선순위                                                  │ │
│  │  • 실행 시간 (얼마나 했는가)                                          │ │
│  │  • 사용 자원 (얼마나 많이 쓰는가)                                      │ │
│  │  • 완료까지 남은 시간                                                  │ │
│  │  • 필요한 자원 수                                                    │ │
│  │  • 대화형 vs 일괄 처리                                                │ │
│  │  • 이전 롤백 횟수 (기아 방지)                                          │ │
│  │                                                             │ │
│  │  비용 함수 예시:                                                      │ │
│  │  cost = w1 × priority + w2 × elapsed + w3 × resources              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【롤백 전략】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 전체 롤백:                                                       │ │
│  │     • 처음부터 다시 시작                                               │ │
│  │     • 단순하지만 비용 큼                                              │ │
│  │                                                             │ │
│  │  2. 부분 롤백:                                                       │ │
│  │     • 체크포인트로 복구                                               │ │
│  │     • 효율적이지만 구현 복잡                                           │ │
│  │                                                             │ │
│  │  3. 안전 지점 복구:                                                  │ │
│  │     • 정의된 안전 지점으로 복구                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【기아 방지】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 롤백 횟수를 비용 함수에 포함                                        │ │
│  │  • 자주 롤백된 프로세스는 높은 비용                                     │ │
│  │  • 결과: 다른 프로세스가 선택될 확률 증가                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데이터베이스 복구】                                                  │
│  ──────────────────                                                  │
│  // MySQL InnoDB: 자동 교착상태 감지 및 롤백                              │
│  // 희생자 선택: 롤백 비용이 가장 적은 트랜잭션                               │
│  // 에러 코드: ER_LOCK_DEADLOCK (1213)                                  │
│                                                                     │
│  try {                                                               │
│      connection.setAutoCommit(false);                               │
│      // 트랜잭션 작업                                                   │
│      connection.commit();                                            │
│  } catch (SQLException e) {                                          │
│      if (e.getErrorCode() == 1213) {  // Deadlock                    │
│          // 롤백 후 재시도                                               │
│          connection.rollback();                                      │
│          retry_transaction();                                        │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【희생자 선택 구현】                                                   │
│  ──────────────────                                                  │
│  class VictimSelector {                                             │
│      Map<Integer, Integer> rollbackCount = new HashMap<>();         │
│                                                                     │
│      Process selectVictim(Set<Process> deadlocked) {                │
│          Process victim = null;                                      │
│          int minCost = Integer.MAX_VALUE;                           │
│                                                                     │
│          for (Process p : deadlocked) {                              │
│              int cost = calculateCost(p);                            │
│              if (cost < minCost) {                                   │
│                  minCost = cost;                                     │
│                  victim = p;                                          │
│              }                                                      │
│          }                                                          │
│                                                                     │
│          // 기아 방지: 선택된 victim의 롤백 횟수 증가                       │
│          rollbackCount.merge(victim.id, 1, Integer::sum);            │
│                                                                     │
│          return victim;                                              │
│      }                                                              │
│                                                                     │
│      int calculateCost(Process p) {                                 │
│          int priority_cost = (10 - p.priority) * 100;               │
│          int elapsed_cost = (int)(p.elapsedTime / 1000);            │
│          int resource_cost = p.heldResources.size() * 50;           │
│          int rollback_cost = rollbackCount.getOrDefault(p.id, 0) * 200;│
│                                                                     │
│          return priority_cost + elapsed_cost + resource_cost + rollback_cost;│
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【롤백 구현】                                                          │
│  ──────────────────                                                  │
│  class CheckpointManager {                                          │
│      Map<Transaction, List<Checkpoint>> checkpoints;                │
│                                                                     │
│      void saveCheckpoint(Transaction t) {                           │
│          Checkpoint cp = new Checkpoint(                            │
│              t.getState(),                                          │
│              t.getModifiedResources(),                              │
│              System.currentTimeMillis()                             │
│          );                                                          │
│          checkpoints.computeIfAbsent(t, k -> new ArrayList<>())     │
│              .add(cp);                                               │
│      }                                                              │
│                                                                     │
│      void rollback(Transaction t) {                                 │
│          List<Checkpoint> cps = checkpoints.get(t);                 │
│          if (cps != null && !cps.isEmpty()) {                        │
│              Checkpoint last = cps.get(cps.size() - 1);              │
│              t.restoreState(last.state);                            │
│              t.releaseModifiedResources(last.modifiedResources);    │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【시스템별 복구 방식】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템            복구 방식         희생자 선택                    │ │
│  │  ────────          ────────          ────────                    │ │
│  │  MySQL InnoDB      트랜잭션 롤백       최소 롤백 비용                 │ │
│  │  PostgreSQL        트랜잭션 롤백       랜덤                          │ │
│  │  Oracle            트랜잭션 롤백       대기 그래프 기반               │ │
│  │  Linux 커널        프로세스 종료       수동 (사용자)                  │ │
│  │  Java (JVM)        스레드 덤프         수동 (개발자)                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【모범 사례】                                                          │
│  ──────────────────                                                  │
│  • 애플리케이션: 재시도 로직 구현                                          │
│  • 데이터베이스: 짧은 트랜잭션                                            │
│  • 모니터링: 교착상태 발생 추적                                           │
│  • 설계: 교착상태 예방 전략 병행                                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 교착상태 해결 후 정상화

• 희생자: 최소 비용으로 선택
• 롤백: 체크포인트로 복구
• 기아: 롤백 횟수 고려
• 현실: DB는 자동, 앱은 재시도
```
