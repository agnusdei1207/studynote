# 306. 자원 유형 (Resource Types)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템에서 관리되는 자원의 종류
> 2. **가치**: 교착상태 분석의 기본 단위
> 3. **융합**: 재사용 자원, 소모성 자원과 연관

---

## Ⅰ. 개요

### 개념 정의

자원 유형(Resource Types)은 **시스템에서 프로세스가 요청하고 사용하는 자원의 종류**다. 교착상태 분석의 기본 단위다.

### 💡 비유: 도서관 자료
자원 유형은 **도서관 자료 종류**와 같다. 책, DVD, 잡지, 컴퓨터실 좌석 등 각각 다른 종류의 자원이다.

### 자원 유형 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 유형 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 분류】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 재사용 자원 (Reusable Resources):                                │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 한 번 사용 후 반납하면 다른 프로세스가 재사용 가능              │ │ │
│  │  │  • 고정된 개수 존재                                          │ │ │
│  │  │  • 예: CPU, 메모리, 파일, 락, 장치                             │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  2. 소모성 자원 (Consumable Resources):                              │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 사용하면 사라짐                                            │ │ │
│  │  │  • 무한히 생성 가능 (이론적)                                   │ │ │
│  │  │  • 예: 신호(signal), 메시지, 인터럽트                          │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시스템 자원 예시】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  하드웨어 자원:                                                       │ │
│  │  • CPU 시간                                                          │ │
│  │  • 메모리 (RAM)                                                       │ │
│  │  • 저장 장치 (디스크, SSD)                                            │ │
│  │  • I/O 장치 (프린터, 키보드, 네트워크)                                  │ │
│  │                                                             │ │
│  │  소프트웨어 자원:                                                     │
│  │  • 파일                                                              │ │
│  │  • 락 (뮤텍스, 세마포어)                                               │ │
│  │  • 소켓                                                              │ │
│  │  • 데이터베이스 연결                                                   │ │
│  │  • 메시지 큐                                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 유형 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【재사용 vs 소모성 비교】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성            재사용 자원           소모성 자원                   │ │
│  │  ────────        ────────────          ────────────               │ │
│  │  개수            고정                   무제한 (생산 가능)            │ │
│  │  사용 후         반납 (재사용)           소멸                        │ │
│  │  교착상태        가능                   가능                        │ │
│  │  예시            CPU, 메모리, 락         신호, 메시지                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【자원의 인스턴스】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  자원 유형 R이 여러 인스턴스를 가질 수 있음:                             │ │
│  │                                                             │ │
│  │  예:                                                                    │ │
│  │  • 프린터 (R1): 2대 → 2개 인스턴스                                    │ │
│  │  • CD-ROM (R2): 1대 → 1개 인스턴스                                    │ │
│  │  • 메모리 (R3): 100MB → 100개의 1MB 단위 인스턴스                       │ │
│  │                                                             │ │
│  │  인스턴스가 여러 개면 동일 유형을 여러 프로세스가 동시 사용 가능            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【자원 상태】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • Free (가용): 할당되지 않음                                         │ │
│  │  • Allocated (할당됨): 특정 프로세스에 할당                            │ │
│  │  • Requested (요청됨): 프로세스가 대기 중                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【교착상태와의 관계】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  재사용 자원에서 교착상태 더 흔함:                                      │ │
│  │  • 고정된 개수                                                        │ │
│  │  • 경합 발생                                                          │ │
│  │                                                             │ │
│  │  소모성 자원에서도 교착상태 가능:                                       │ │
│  │  • 생산자-소비자 문제                                                  │ │
│  │  • 신호 대기                                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【자원 유형 정의】                                                     │
│  ──────────────────                                                  │
│  // 시스템 자원 유형                                                    │
│  enum ResourceType {                                                 │
│      CPU,                                                             │
│      MEMORY,                                                          │
│      DISK,                                                            │
│      NETWORK,                                                         │
│      FILE,                                                            │
│      LOCK,                                                            │
│      DATABASE_CONNECTION                                              │
│  }                                                                  │
│                                                                     │
│  // 자원 인스턴스                                                       │
│  struct Resource {                                                   │
│      ResourceType type;                                              │
│      int id;                 // 인스턴스 ID                            │
│      int total_instances;     // 총 인스턴스 수                         │
│      int available_instances; // 가용 인스턴스 수                        │
│      List<Process> waiters;   // 대기 중인 프로세스                      │
│  }                                                                  │
│                                                                     │
│  【자원 관리자】                                                        │
│  ──────────────────                                                  │
│  class ResourceManager {                                            │
│      Map<ResourceType, Resource> resources;                         │
│                                                                     │
│      // 자원 요청                                                      │
│      boolean request(ResourceType type, int count) {                │
│          Resource res = resources.get(type);                        │
│          if (res.available_instances >= count) {                    │
│              res.available_instances -= count;                      │
│              return true;                                            │
│          }                                                          │
│          return false;  // 대기                                       │
│      }                                                              │
│                                                                     │
│      // 자원 해제                                                      │
│      void release(ResourceType type, int count) {                   │
│          Resource res = resources.get(type);                        │
│          res.available_instances += count;                          │
│          // 대기자 중 할당 가능한지 확인                                   │
│          notifyWaiters(res);                                        │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【실제 시스템 예시】                                                    │
│  ──────────────────                                                  │
│  // Linux: /proc 파일시스템으로 자원 확인                                │
│  cat /proc/meminfo      // 메모리                                     │
│  cat /proc/cpuinfo      // CPU                                       │
│  cat /proc/devices      // 장치                                       │
│                                                                     │
│  // 시스템 호출                                                         │
│  open()    // 파일 자원 요청                                            │
│  malloc()  // 메모리 자원 요청                                          │
│  socket()  // 소켓 자원 요청                                            │
│                                                                     │
│  【자원 제한 확인】                                                      │
│  ──────────────────                                                  │
│  // Linux ulimit                                                      │
│  ulimit -a   // 모든 제한 확인                                          │
│  ulimit -n   // 열린 파일 수 제한                                        │
│  ulimit -u   // 프로세스 수 제한                                         │
│                                                                     │
│  // 프로그래밍                                                           │
│  struct rlimit rl;                                                   │
│  getrlimit(RLIMIT_NOFILE, &rl);  // 파일 제한 확인                      │
│  getrlimit(RLIMIT_NPROC, &rl);   // 프로세스 제한 확인                   │
│                                                                     │
│  【데이터베이스 자원】                                                   │
│  ──────────────────                                                  │
│  // 연결 풀                                                            │
│  HikariDataSource ds = new HikariDataSource();                      │
│  ds.setMaximumPoolSize(20);  // 최대 연결 수                            │
│                                                                     │
│  // 연결 요청                                                          │
│  Connection conn = ds.getConnection();  // 대기 가능                   │
│  conn.close();  // 반납                                                │
│                                                                     │
│  【성능 모니터링】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  자원 유형           모니터링 명령/도구                              │ │
│  │  ────────            ────────────────────                         │ │
│  │  CPU                 top, htop, mpstat                            │ │
│  │  메모리              free, vmstat, /proc/meminfo                  │ │
│  │  디스크              df, iostat, dstat                            │ │
│  │  네트워크            ifstat, netstat, ss                          │ │
│  │  파일                lsof, /proc/<pid>/fd                         │ │
│  │  락                  lockdep, /proc/lockdep                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
