# 자원 선점 (Resource Preemption)을 통한 복구

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자원 선점(Resource Preemption)은 교착 상태를 해소하기 위해 특정 프로세스가 점유하고 있는 자원을 강제로 회수하여 다른 프로세스에 할당함으로써 대기 사이클을 끊는 복구 전략이다.
> 2. **과정**: 희생자 선택, 롤백(Rollback), 기아 상태 방지라는 3단계 프로세스를 거치며, 프로세스를 죽이지 않고 상태를 되돌리는 정교한 제어가 핵심이다.
> 3. **가치**: 프로세스 전체 종료보다 작업 손실을 줄일 수 있는 세밀한 복구 기법이지만, 복잡한 롤백 로직과 자원 일관성 유지라는 기술적 난제를 동반한다.

+++

## Ⅰ. 자원 선점의 기본 단계 (3-Step Process)

### 1. 희생자 선택 (Selecting a Victim)
- 어떤 자원을 어떤 프로세스로부터 빼앗을지 결정한다. 비용이 가장 낮은 프로세스를 선택하는 것이 원칙이다.

### 2. 롤백 (Rollback)
- 자원을 빼앗긴 프로세스는 정상적인 실행이 불가능하므로, 자원을 할당받기 전의 '안전한 상태'로 상태를 되돌려야 한다.

### 3. 기아 상태 방지 (Preventing Starvation)
- 한 프로세스가 계속해서 자원을 선점당하지 않도록 보장해야 한다.

- **📢 섹션 요약 비유**: "놀이터에서 친구가 가지고 있는 장난감을 선생님이 잠시 뺏어서 다른 친구에게 주고, 뺏긴 친구는 아까 놀던 곳으로 다시 가 있게 하는 것과 같습니다."

+++

## Ⅱ. 롤백 메커니즘의 종류 (ASCII)

### 1. 완전 롤백 vs 부분 롤백
```text
  [ Process Timeline ]
  
  Start ---- Checkpoint 1 ---- Checkpoint 2 ---- [ DEADLOCK ]
    |             |                |                |
    +-------------+----------------+----------------+
    
  1. Total Rollback: Back to [Start] (Safe but Expensive)
  2. Partial Rollback: Back to [Checkpoint 2] (Smart but Complex)
```

### 2. ASCII 선점 과정
```text
  P1(Has R1) <-> P2(Has R2)  --[ Detect ]--> Deadlock!
  
  --[ Action ]-->
  1. Take R2 from P2 (Preemption)
  2. Give R2 to P1
  3. P2 Rollback to Checkpoint
  4. P1 Completes -> Returns R1, R2
  5. P2 Resumes
```

- **📢 섹션 요약 비유**: "게임하다가 막혔을 때, 아예 처음부터 다시 할지 아니면 마지막에 저장된 '세이브 포인트'에서 다시 할지 정하는 것과 같습니다."

+++

## Ⅲ. 기아 상태(Starvation) 문제와 해결책 (Strategy)

### 1. 문제점
- 비용 기반의 희생자 선택을 고수하면, 항상 비용이 낮은 특정 프로세스만 계속 자원을 뺏기게 되어 영영 작업을 끝내지 못할 수 있다.

### 2. 해결책
- 비용 계산식에 '선점 횟수(Number of Preemptions)'를 포함시킨다. 선점당한 횟수가 많아질수록 희생자로 선정될 확률이 낮아지도록 우선순위를 보정한다.

- **📢 섹션 요약 비유**: "착한 애 장난감만 계속 뺏으면 그 아이는 울어버릴 테니, 뺏긴 횟수를 기억해 두었다가 다음에는 다른 아이 걸 뺏는 공평한 규칙을 만드는 것과 같습니다."

+++

## Ⅳ. 자원 선점의 기술적 난제 (Analysis)

| 고려 사항 | 상세 설명 |
|:---|:---|
| **상태 저장 (Checkpointing)** | 롤백을 위해 수시로 프로세스의 상태를 저장해야 하며, 이는 자체적인 오버헤드가 됨. |
| **자원 원자성 (Atomicity)** | 자원을 뺏는 순간 데이터가 깨지지 않도록 트랜잭션 처리가 필요함. |
| **복잡도** | 단순히 종료시키는 것보다 OS 커널 코드가 훨씬 복잡해짐. |

- **📢 섹션 요약 비유**: "요리 중간에 냄비를 뺏으면 음식이 망가지듯, 프로세스 중간에 자원을 뺏어도 문제가 없도록 완벽하게 덮개를 씌워두는 기술이 필요하다는 뜻입니다."

+++

## Ⅴ. 결론 및 실무적 교훈

### 1. 요약
- 자원 선점은 '살릴 수 있는 프로세스는 살린다'는 목표를 가진 고도화된 데드락 복구 기법이다.

### 2. 최종 결론
- 비록 구현은 어렵지만, 대규모 데이터베이스나 분산 시스템에서는 작업의 연속성이 매우 중요하기 때문에 롤백과 선점 기법을 적극적으로 활용한다. 이는 시스템의 회복 탄력성(Resilience)을 높이는 핵심 기술이다.

- **📢 섹션 요약 비유**: "넘어졌을 때 포기하지 않고 가장 최근에 서 있던 자리에서 다시 시작할 수 있게 도와주는 것이 진정한 복구 시스템이라는 교훈을 줍니다."
