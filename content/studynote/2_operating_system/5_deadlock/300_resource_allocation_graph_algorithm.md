# 단일 인스턴스 환경의 회피 (Resource Allocation Graph Algorithm)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자원 할당 그래프 알고리즘은 자원 유형별로 인스턴스가 하나뿐인 환경에서, 프로세스의 잠재적 자원 요청을 나타내는 '선언 간선(Claim Edge)'을 도입하여 사이클 형성을 미리 감시하는 회피 전략이다.
> 2. **작동**: 자원 요청 시 선언 간선을 할당 간선(Assignment Edge)으로 변환했을 때 그래프에 사이클(Cycle)이 생기는지 확인하고, 사이클이 발생한다면 해당 할당을 거부하여 불안전 상태 진입을 막는다.
> 3. **가치**: 복잡한 수치 계산(은행원 알고리즘) 없이 그래프의 기하학적 구조만으로 데드락 회피 여부를 판별할 수 있어 직관적이고 효율적인 리소스 관리가 가능하다.

+++

## Ⅰ. 자원 할당 그래프 알고리즘의 개요

### 1. 정의
- 기존의 자원 할당 그래프에 '선언 간선(Claim Edge)'이라는 새로운 유형의 간선을 추가하여, 프로세스가 향후 요청할 수 있는 자원을 미리 명시하고 관리하는 알고리즘이다.

### 2. 적용 범위
- 자원 유형(Resource Type) 당 **단 하나의 인스턴스**만 존재하는 시스템에 한정된다. (인스턴스가 여러 개일 때는 은행원 알고리즘을 사용해야 함)

- **📢 섹션 요약 비유**: "누가 어떤 장난감을 나중에 가지고 놀고 싶어 하는지 미리 '찜'해두고, 그 찜한 것들끼리 서로 꼬이지 않게 미리 살펴보는 것과 같습니다."

+++

## Ⅱ. 기술적 구조 및 ASCII 다이어그램

### 1. 간선의 종류
- **선언 간선 (Claim Edge, $P_i \rightarrow R_j$)**: 점선 화살표. 나중에 자원을 요청할 것임을 미리 선언.
- **요청 간선 (Request Edge, $P_i \rightarrow R_j$)**: 실선 화살표. 현재 자원을 요청 중.
- **할당 간선 (Assignment Edge, $R_j \rightarrow P_i$)**: 실선 화살표. 자원이 할당됨.

### 2. ASCII 작동 원리
```text
    [ Safe State ]             [ Unsafe State Check ]
    P1 --- 점선 ---▶ R1        P1 --- 실선 ---▶ R1 (Requested)
    ▲                │         ▲                │
    │              점선        │              실선 (Allocated)
    │                ▼         │                ▼
    R2 ◀--- 실선 --- P2        R2 ◀--- 실선 --- P2
    
    * P1이 R1을 요청했을 때 실선으로 바꿨더니
      P1 -> R1 -> P2 -> R2 -> P1 형태의 
      사이클이 생기면 할당 거부!
```

- **📢 섹션 요약 비유**: "길을 갈 때 미리 가고 싶은 길을 지도에 표시해 보고, 그렇게 가면 나중에 서로 막다른 길에서 마주칠 일이 없는지 확인하는 지도 그리기와 같습니다."

+++

## Ⅲ. 알고리즘 수행 단계 (Step-by-Step)

### 1. 초기 상태
- 모든 프로세스는 자원을 요청하기 전, 미래에 요청할 모든 자원에 대해 선언 간선($P_i \rightarrow R_j$)을 그래프에 추가해야 한다.

### 2. 자원 요청 시
- 선언 간선 $P_i \rightarrow R_j$를 요청 간선으로 전환한다.
- 요청 간선을 임시로 할당 간선 $R_j \rightarrow P_i$로 변환해 본다.
- **사이클 탐지**: 변환된 그래프에 사이클이 발생하는지 검사한다.
    - 사이클 없음: 실제 할당을 승인한다.
    - 사이클 있음: 할당을 거부하고 요청 간선 상태로 되돌리며 프로세스를 대기시킨다.

- **📢 섹션 요약 비유**: "가위바위보를 하기 전에 각자 내고 싶은 걸 미리 종이에 써서 내고, 그 패들이 무승부(데드락)가 날 것 같으면 다시 짜오라고 시키는 것과 같습니다."

+++

## Ⅳ. 기술적 한계 및 트레이드오프 (Strategy)

| 비교 항목 | 자원 할당 그래프 알고리즘 | 은행원 알고리즘 |
|:---|:---|:---|
| **대상** | 단일 인스턴스 | **다중 인스턴스** |
| **판단 기준** | **그래프 사이클 존재 여부** | 가용 자원량 계산 |
| **복잡도** | $O(N^2)$ (사이클 탐지 비용) | $O(M \times N^2)$ |
| **직관성** | 매우 높음 | 보통 |

- **📢 섹션 요약 비유**: "한 개뿐인 리모컨을 두고 싸울 때는 누가 가질지 미리 정하면 되지만, 여러 개 있는 사탕을 나눠줄 때는 복잡한 뺄셈 계산이 필요한 것과 같습니다."

+++

## Ⅴ. 결론 및 실무적 교훈

### 1. 요약
- 단일 인스턴스 환경에서 사이클 탐지는 교착 상태 회피를 위한 필요충분조건이다.

### 2. 최종 결론
- 이 알고리즘은 '미리 선언한다'는 개념이 얼마나 강력한 안전장치가 될 수 있는지 보여준다. 비록 다중 인스턴스에는 적용하지 못하지만, 락(Lock) 하나하나가 유니크한 프로그래밍 환경에서는 데드락 분석의 기초 이론이 된다.

- **📢 섹션 요약 비유**: "앞으로의 계획을 미리 투명하게 공개하면, 나중에 생길 수 있는 커다란 충돌을 아주 가볍게 막을 수 있다는 지혜를 줍니다."

+++

### 📌 관련 개념 맵 (Knowledge Graph)
- **자원 할당 그래프 (RAG)**: 알고리즘의 베이스가 되는 시각적 도구.
- **은행원 알고리즘 (Banker's Algorithm)**: 인스턴스가 여러 개일 때의 확장형 알고리즘.
- **순환 대기 (Circular Wait)**: 그래프 알고리즘이 찾아내고자 하는 핵심 징후.
