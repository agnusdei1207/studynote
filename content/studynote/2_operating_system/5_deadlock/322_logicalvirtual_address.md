# 416. 논리 주소 (Logical Address)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 생성하는 주소
> 2. **가치**: 가상 주소 공간에서의 위치
> 3. **융합**: 물리 주소, MMU와 연관

---

## Ⅰ. 개요

### 개념 정의

논리 주소(Logical Address)는 **프로그램 실행 중 CPU가 생성하는 주소**다. 가상 주소라고도 한다. 실제 물리 메모리 주소와 다를 수 있다.

### 💡 비유: 집 주소 vs 좌표
논리 주소는 **집 주소**와 같다. "123번지"라고 말한다. 실제 지도 좌표(물리 주소)는 다르다.

### 논리 주소 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                논리 주소 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【논리 주소 vs 물리 주소】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  논리 주소 (Logical/Virtual Address):                                    │ │
│  │  • CPU가 생성                                                           │ │
│  │  • 프로그램 관점                                                          │ │
│  │  • 0x1000, 0x2000, ...                                                  │ │
│  │                                                             │ │
│  │  물리 주소 (Physical Address):                                           │ │
│  │  • 실제 메모리 위치                                                       │ │
│  │  • 하드웨어 관점                                                          │ │
│  │  • 0x5000, 0x8000, ...                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 변환 과정】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  프로그램                                                            │ │
│  │     │                                                             │ │
│  │     │ 논리 주소 (0x1234)                                             │ │
│  │     ▼                                                             │ │
│  │  ┌─────┐                                                           │ │
│  │  │ CPU │                                                           │ │
│  │  └─────┘                                                           │ │
│  │     │                                                             │ │
│  │     │ 논리 주소                                                      │ │
│  │     ▼                                                             │ │
│  │  ┌─────┐                                                           │ │
│  │  │ MMU │ ──페이지 테이블 조회──► 물리 주소 (0x5678)                     │ │
│  │  └─────┘                                                           │ │
│  │     │                                                             │ │
│  │     │ 물리 주소                                                      │ │
│  │     ▼                                                             │ │
│  │  ┌─────────────┐                                                   │ │
│  │  │ 물리 메모리   │                                                   │ │
│  │  │  [0x5678]   │                                                   │ │
│  │  └─────────────┘                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【논리 주소 구성 (페이징)】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  32비트 논리 주소 (4KB 페이지):                                          │ │
│  │                                                             │ │
│  │  ┌─────────────────────┬─────────────────────┐                    │ │
│  │  │   페이지 번호 (20비트)   │   오프셋 (12비트)    │                    │ │
│  │  └─────────────────────┴─────────────────────┘                    │ │
│  │                                                             │ │
│  │  페이지 번호: 어떤 페이지인지                                              │ │
│  │  오프셋: 페이지 내에서의 위치                                             │ │
│  │                                                             │ │
│  │  예: 0x12345678                                                     │ │
│  │  페이지 번호 = 0x12345                                                │ │
│  │  오프셋 = 0x678                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                논리 주소 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 바인딩 시점】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 컴파일 시간 (Compile Time):                                        │ │
│  │     • 절대 코드 생성                                                    │ │
│  │     • 위치 고정                                                        │ │
│  │     • 현대 시스템에서 거의 사용 안 함                                    │ │
│  │                                                             │ │
│  │  2. 로드 시간 (Load Time):                                              │ │
│  │     • 로더가 주소 할당                                                   │ │
│  │     • 재배치 가능                                                       │ │
│  │                                                             │ │
│  │  3. 실행 시간 (Execution Time):                                        │ │
│  │     • 실행 중 주소 바인딩                                               │ │
│  │     • 동적 메모리 할당                                                   │ │
│  │     • 현대 시스템                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MMU의 역할】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Memory Management Unit:                                               │ │
│  │                                                             │ │
│  │  • 논리 주소 → 물리 주소 변환                                            │ │
│  │  • 페이지 테이블 사용                                                    │ │
│  │  • TLB 캐시 활용                                                        │ │
│  │  • 보호 비트 검사                                                        │ │
│  │  • 페이지 폴트 트리거                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【세그먼테이션에서의 논리 주소】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  논리 주소 = 세그먼트 번호 + 오프셋                                      │ │
│  │                                                             │ │
│  │  ┌─────────────────────┬─────────────────────┐                    │ │
│  │  │   세그먼트 번호        │       오프셋         │                    │ │
│  │  └─────────────────────┴─────────────────────┘                    │ │
│  │                                                             │ │
│  │  물리 주소 = 세그먼트 기준 + 오프셋                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【포인터와 논리 주소】                                                    │
│  ──────────────────                                                  │
│  // C 언어에서 포인터는 논리 주소                                          │
│  int x = 10;                                                         │
│  int* ptr = &x;  // ptr은 논리 주소                                     │
│  printf("%p\n", ptr);  // 예: 0x7ffce91f9000                         │
│                                                                     │
│  // 같은 논리 주소, 다른 물리 주소                                          │
│  // 프로세스 A의 0x1000 != 프로세스 B의 0x1000 (물리적)                  │
│                                                                     │
│  【주소 변환 확인】                                                        │
│  ──────────────────                                                  │
│  // 페이지 테이블을 통한 변환                                              │
│  // /proc/self/pagemap으로 확인 가능 (root 필요)                         │
│                                                                     │
│  // 가상 주소 → 페이지 번호                                               │
│  void* vaddr = malloc(4096);                                         │
│  unsigned long pfn = ((unsigned long)vaddr) >> 12;                   │
│                                                                     │
│  // pagemap에서 물리 프레임 번호 조회                                      │
│  // (CAP_SYS_ADMIN 권한 필요)                                           │
│                                                                     │
│  【디버깅에서 활용】                                                        │
│  ──────────────────                                                  │
│  // gdb에서 주소 확인                                                     │
│  (gdb) print &variable                                               │
│  $1 = (int *) 0x7fffffffe1fc                                        │
│  // 논리 주소                                                            │
│                                                                     │
│  // 메모리 맵 확인                                                        │
│  (gdb) info proc mappings                                            │
│  Start           End             Offset         File                 │
│  0x555555554000  0x555555555000  0x00000000     /bin/cat             │
│  // 논리 주소 범위                                                        │
│                                                                     │
│  【같은 논리 주소, 다른 물리 주소 예시】                                     │
│  ──────────────────                                                  │
│  // 두 프로세스 실행                                                      │
│  $ cat /proc/$(pgrep -n cat)/maps | head -1                         │
│  555555554000-555555555000 r--p ...                                  │
│                                                                     │
│  $ cat /proc/$(pgrep -n cat)/maps | head -1                         │
│  555555554000-555555555000 r--p ...                                  │
│  // 같은 논리 주소!                                                      │
│                                                                     │
│  // 하지만 실제 물리 메모리는 다름                                          │
│  // 각자 다른 페이지 테이블 사용                                           │
│                                                                     │
│  【논리 주소 공간 크기】                                                    │
│  ──────────────────                                                  │
│  // 32비트: 4GB (2^32)                                                  │
│  // 64비트: 16 EB (2^64) 이론적                                           │
│  // 실제: 48비트 구현 = 256 TB                                           │
│                                                                     │
│  // 프로그램에서 확인                                                      │
│  #include <stdint.h>                                                  │
│  printf("Pointer size: %zu bytes\n", sizeof(void*));                │
│  // 32비트: 4바이트, 64비트: 8바이트                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: CPU가 생성하는 주소
• 구성: 페이지 번호 + 오프셋
• 변환: MMU가 논리 → 물리 변환
• 특징: 프로세스마다 독립적
• 관계: 논리 주소 ≠ 물리 주소
• 활용: 포인터, 메모리 관리
```
