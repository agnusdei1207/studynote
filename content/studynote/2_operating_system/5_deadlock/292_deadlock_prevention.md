# 292. 교착상태 조건 (Deadlock Conditions)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착상태 발생에 필요한 4가지 필요조건
> 2. **가치**: 예방 전략의 기반
> 3. **융합**: 상호배제, 보유대기, 비선점, 순환대기

---

## Ⅰ. 개요

### 개념 정의

교착상태 조건(Deadlock Conditions)은 **교착상태가 발생하기 위해 동시에 만족해야 하는 4가지 필요조건**이다. 하나라도 만족하지 않으면 교착상태는 발생하지 않는다.

### 💡 비유: 4개의 열쇠
교착상태 조건은 **4개의 열쇠**와 같다. 4개 다 있어야 문이 열린다. 하나라도 없으면 교착상태 문이 안 열린다.

### 4가지 조건

```
┌─────────────────────────────────────────────────────────────────────┐
│                교착상태 4가지 조건                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 상호 배제 (Mutual Exclusion)】                                   │
│  ─────────────────────────────────────                               │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  정의: 자원은 한 번에 한 프로세스만 사용 가능                                 │ │
│  │                                                             │ │
│  │  예시:                                                             │ │
│  │  • 프린터: 한 번에 한 문서만 인쇄                                        │ │
│  │  • 뮤텍스: 한 스레드만 보유                                             │ │
│  │  • 파일 쓰기: 한 프로세스만 쓰기                                          │ │
│  │                                                             │ │
│  │  무력화 방법:                                                          │ │
│  │  • 공유 자원 사용 (읽기 전용)                                            │ │
│  │  • 락 프리 알고리즘                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【2. 보유 및 대기 (Hold and Wait)】                                    │
│  ─────────────────────────────────────                               │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  정의: 자원을 보유하면서 다른 자원을 대기                                    │ │
│  │                                                             │ │
│  │  예시:                                                             │ │
│  │  • 락 A 보유 + 락 B 요청                                              │ │
│  │  • 메모리 보유 + 디스크 대기                                             │ │
│  │                                                             │ │
│  │  무력화 방법:                                                          │ │
│  │  • 모든 자원을 한 번에 요청                                              │ │
│  │  • 자원을 보유하지 않고 요청                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【3. 비선점 (No Preemption)】                                         │
│  ─────────────────────────────────────                               │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  정의: 다른 프로세스의 자원을 강제로 뺏을 수 없음                              │ │
│  │                                                             │ │
│  │  예시:                                                             │ │
│  │  • 뮤텍스: 다른 스레드가 해제해야 함                                        │ │
│  │  • 프린터: 인쇄 중단 불가                                               │ │
│  │                                                             │ │
│  │  무력화 방법:                                                          │ │
│  │  • 선점 가능한 자원 사용                                                │ │
│  │  • 타임아웃으로 강제 해제                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【4. 순환 대기 (Circular Wait)】                                      │
│  ─────────────────────────────────────                               │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  정의: 대기 그래프에 순환이 존재                                           │ │
│  │                                                             │ │
│  │  예시:                                                             │ │
│  │  P1 → R1 → P2 → R2 → P1 (순환)                                     │ │
│  │                                                             │ │
│  │  무력화 방법:                                                          │ │
│  │  • 자원 순서 부여                                                       │ │
│  │  • 오름차순으로만 요청                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                조건별 예방 전략                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【조건 → 예방 매핑】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  조건              예방 전략                                        │ │
│  │  ────────          ────────                                        │ │
│  │  상호 배제          공유 자원, 락 프리                                   │ │
│  │  보유 및 대기        일괄 요청, 두 단계 락킹                               │ │
│  │  비선점            선점 가능 자원, 타임아웃                               │ │
│  │  순환 대기          자원 순서 규칙                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【필요조건 vs 충분조건】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  4가지 조건 모두 만족 → 교착상태 "가능"                                  │ │
│  │  (반드시 발생하는 것은 아님)                                            │ │
│  │                                                             │ │
│  │  • 필요조건: 교착상태가 있으면 4가지 모두 참                                │ │
│  │  • 충분조건: 4가지가 참이면 교착상태 발생 (아님!)                            │ │
│  │                                                             │ │
│  │  실제로는: 자원 할당 그래프에서 순환을 확인해야 함                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실제 적용 예시】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Linux 커널:                                                        │ │
│  │  • 상호 배제: spinlock, mutex                                        │ │
│  │  • 보유 및 대기: 락 획득 후 다른 락 요청 가능                              │ │
│  │  • 비선점: 락은 선점 불가                                               │ │
│  │  • 순환 대기: lockdep로 감지                                          │ │
│  │                                                             │ │
│  │  해결: lockdep로 순환 대기 감지                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【보유 및 대기 예방】                                                  │
│  ──────────────────                                                  │
│  // 방법 1: 모든 락을 한 번에 획득                                        │
│  void transfer(Account* from, Account* to, int amount) {            │
│      // 모든 락을 먼저 획득                                              │
│      lock(&all_accounts_lock);                                       │
│      lock(&from->lock);                                              │
│      lock(&to->lock);                                                │
│      unlock(&all_accounts_lock);                                     │
│                                                                     │
│      // 작업                                                          │
│      from->balance -= amount;                                        │
│      to->balance += amount;                                          │
│                                                                     │
│      unlock(&to->lock);                                              │
│      unlock(&from->lock);                                            │
│  }                                                                  │
│                                                                     │
│  【비선점 예방: tryLock】                                               │
│  ──────────────────                                                  │
│  // 타임아웃 또는 논블로킹 시도                                              │
│  while (true) {                                                      │
│      lock(&lockA);                                                   │
│      if (tryLock(&lockB)) {                                          │
│          // 성공: 두 락 모두 획득                                         │
│          break;                                                      │
│      }                                                              │
│      // 실패: lockA 해제하고 재시도                                        │
│      unlock(&lockA);                                                 │
│      sleep(random_time());  // 백오프                                  │
│  }                                                                  │
│                                                                     │
│  【순환 대기 예방: 순서 규칙】                                             │
│  ──────────────────                                                  │
│  // 자원에 번호 부여, 오름차순으로만 획득                                      │
│  #define LOCK_DB 1                                                   │
│  #define LOCK_FILE 2                                                 │
│  #define LOCK_NET 3                                                  │
│                                                                     │
│  void work() {                                                       │
│      // 항상 낮은 번호 먼저                                                │
│      lock(LOCK_DB);                                                  │
│      lock(LOCK_FILE);                                                │
│      lock(LOCK_NET);                                                 │
│                                                                     │
│      // 작업                                                          │
│                                                                     │
│      unlock(LOCK_NET);                                               │
│      unlock(LOCK_FILE);                                              │
│      unlock(LOCK_DB);                                                │
│  }                                                                  │
│                                                                     │
│  【조건 확인 함수】                                                      │
│  ──────────────────                                                  │
│  bool checkDeadlockConditions() {                                   │
│      bool mutualExclusion = hasExclusiveResources();               │
│      bool holdAndWait = hasProcessHoldingAndWaiting();              │
│      bool noPreemption = hasNonPreemptibleResources();             │
│      bool circularWait = hasCircularWait();                         │
│                                                                     │
│      return mutualExclusion && holdAndWait &&                        │
│             noPreemption && circularWait;                           │
│  }                                                                  │
│                                                                     │
│  【성능 vs 안전성 트레이드오프】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  예방 전략        안전성    성능    복잡도                                │ │
│  │  ────────        ──────   ──────   ──────                              │ │
│  │  일괄 요청         높음     낮음     낮음                                   │ │
│  │  순서 규칙         높음     중간     중간                                   │ │
│  │  타임아웃          중간     중간     낮음                                   │ │
│  │  탐지/복구         중간     높음     높음                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 조건 1: 상호 배제 - 자원은 한 번에 하나만
• 조건 2: 보유 및 대기 - 보유하면서 대기
• 조건 3: 비선점 - 강제 뺏기 불가
• 조건 4: 순환 대기 - 대기 그래프 순환
• 특성: 모두 만족해야 발생 가능
• 예방: 하나라도 무력화하면 예방
```
