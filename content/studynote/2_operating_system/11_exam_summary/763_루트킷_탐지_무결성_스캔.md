# 루트킷 탐지 / 무결성 스캔

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 수준에서 숨어 작동하는 악성코드 탐지 기술
> 2. **가치**: 보안 침해 탐지, 무결성 보장, APT 대응
> 3. **융합**: 시스템 콜 훅, 커널 모듈, 보안 감사와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

루트킷(Rootkit)은 커널 수준에서 숨어 작동하는 악성코드이고, 무결성 스캔은 시스템 변조를 탐지하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    루트킷 탐지 / 무결성 스캔                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              루트킷 유형                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  User-mode Rootkit:                                  │   │ │
│   │   │  • 사용자 공간에서 동작                               │   │ │
│   │   │  • 시스템 바이너리 교체 (ls, ps, netstat)             │   │ │
│   │   │  • LD_PRELOAD로 라이브러리 후킹                       │   │ │
│   │   │  • 탐지가 상대적으로 쉬움                              │   │ │
│   │   │                                                      │   │ │
│   │   │  Kernel-mode Rootkit:                                │   │ │
│   │   │  • 커널 공간에서 동작                                 │   │ │
│   │   │  • 시스템 콜 테이블 후킹                               │   │ │
│   │   │  • 커널 모듈로 로드 (LKM)                             │   │ │
│   │   │  • /proc, /sys 숨김                                  │   │ │
│   │   │  • 탐지가 매우 어려움                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  Hypervisor Rootkit:                                 │   │ │
│   │   │  • 하이퍼바이저 수준에서 동작                          │   │ │
│   │   │  • VM 아래에 숨음                                     │   │ │
│   │   │  • 가장 탐지 어려움                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              탐지 기술                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 무결성 검사 (Integrity Check):                    │   │ │
│   │   │     • 파일 해시 비교 (AIDE, Tripwire)                 │   │ │
│   │   │     • 바이너리 무결성 확인                             │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 행위 기반 탐지:                                   │   │ │
│   │   │     • 시스템 콜 패턴 분석                             │   │ │
│   │   │     • 이상 행위 탐지                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 커널 무결성 검사:                                 │   │ │
│   │   │     • 시스템 콜 테이블 검증                           │   │ │
│   │   │     • 커널 코드 무결성 (IMA, EVM)                    │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 신뢰 실행 환경:                                   │   │ │
│   │   │     • Secure Boot, TPM                               │   │ │
│   │   │     • 측정된 부팅 (Measured Boot)                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 루트킷은 깊이 숨을수록 탐지가 어렵습니다.

**💡 비유**: 루트킷은 건물 지하실에 숨은 침입자와 같습니다.

**📢 섹션 요약 비유**: 커널 루트킷이 가장 위험!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Syscall Hook** | 가로채기 | 테이블 교체 | 도청 |
| **AIDE** | 무결성 | 해시 비교 | 감사 |
| **IMA** | 측정 | 커널 통합 | 기록 |
| **TPM** | 하드웨어 | 증명 | 인장 |

**탐지 도구**

| 도구 | 용도 |
|:---|:---|
| **rkhunter** | 루트킷 탐지 |
| **chkrootkit** | 루트킷 확인 |
| **AIDE** | 파일 무결성 |
| **IMA/EVM** | 커널 무결성 |

**핵심 알고리즘**: 스캔 → 비교 → 변조 탐지

**📢 섹션 요약 비유**: AIDE가 무결성 표준!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 서버 보안** → AIDE + cron
**시나리오 2: 클라우드** → IMA + TPM
**시나리오 3: APT 대응** → 행위 기반 + EDR

**도입 체크리스트**: 정기 스캔, 변경 추적

**📢 섹션 요약 비유**: 정기 스캔이 핵심!
