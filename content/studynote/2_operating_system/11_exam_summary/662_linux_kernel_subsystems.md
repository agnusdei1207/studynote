> **[Insight]**
> 리눅스 커널(Linux Kernel)은 모놀리식 커널(Monolithic Kernel)의 고성능 구조와 모듈형 설계의 유연성을 결합하여 전 세계 서버 및 임베디드 시장을 지배하는 오픈소스 핵심 자산이다.
> 각 서브시스템은 독립적으로 작동하면서도 상호 긴밀하게 연결되어 있으며, 특히 VFS(Virtual File System)를 통한 추상화와 효율적인 메모리 관리 메커니즘이 시스템 전체의 성능과 안정성을 지탱한다.
> 현대 리눅스는 LKM(Loadable Kernel Module)을 통해 실행 중 커널 기능을 확장하고, 최신 하드웨어 변화에 기민하게 대응하는 진화적 아키텍처를 보여준다.

+++

### Ⅰ. 리눅스 커널의 기본 설계 철학

1. 모놀리식 커널(Monolithic Kernel) 기반의 통합 구조
   - 핵심 OS 서비스(스케줄링, 메모리, 파일 시스템 등)가 하나의 거대한 주소 공간에서 실행되어 시스템 호출 오버헤드를 최소화한다.
2. 계층적 추상화 및 모듈성
   - LKM(Loadable Kernel Module)을 통해 필요할 때만 드라이버나 기능을 로드하여 메모리 효율성을 높인다.
3. 모든 것은 파일이다(Everything is a File)
   - 일반 파일뿐만 아니라 디바이스, 프로세스 정보 등을 파일 인터페이스를 통해 일관되게 접근한다.

📢 섹션 요약 비유: 리눅스 커널은 '만능 맥가이버 칼'과 같아서, 핵심 도구가 하나로 뭉쳐 있어 빠르고 강력하면서도 필요한 부품을 끼웠다 뺐다 할 수 있는 유연함이 특징입니다.

+++

### Ⅱ. 리눅스 커널 서브시스템 통합 맵(Architecture Map)

1. 서브시스템 간 상호작용
   - 프로세스 관리(Process Management), 메모리 관리(Memory Management), 파일 시스템(File System), 네트워크 스택(Network Stack)이 상호 의존적인 관계를 형성한다.

```text
[ Linux Kernel Subsystem Map ]

    +-------------------------------------------------+
    |              User Applications                  |
    +-------------------------------------------------+
    |           Standard C Library (libc)             |
    +-------------------------------------------------+
    |             System Call Interface               |
    +-------------------------------------------------+
    |                                                 |
    |  +-------------+  +------------+  +-----------+ |
    |  |   Process   |  |   Memory   |  |    VFS    | |
    |  |  Scheduler  |--| Management |--|   (File)  | |
    |  +-------------+  +------------+  +-----------+ |
    |         |               |               |       |
    |  +-------------+  +------------+  +-----------+ |
    |  |   Network   |  |   Device   |  |    I/O    | |
    |  |    Stack    |  |   Drivers  |  | Scheduler | |
    |  +-------------+  +------------+  +-----------+ |
    |                                                 |
    +-------------------------------------------------+
    |                Hardware (CPU, RAM, I/O)         |
    +-------------------------------------------------+
```

2. 데이터 흐름의 특징
   - 시스템 콜(System Call)을 통해 사용자 모드에서 커널 모드로 진입하며, 인터럽트 핸들러가 하드웨어 이벤트를 비동기적으로 처리한다.

📢 섹션 요약 비유: 커널 서브시스템은 '오케스트라'와 같아서, 지휘자(스케줄러)가 리듬을 맞추고 각 악기 연주자(서브시스템)들이 서로의 소리를 들으며 완벽한 하모니를 만들어내는 것과 같습니다.

+++

### Ⅲ. 프로세스 및 CPU 스케줄링(Process Management)

1. 테스크 구조체(task_struct)
   - 리눅스에서 프로세스와 스레드를 모두 관리하는 핵심 자료구조이다.
2. CFS(Completely Fair Scheduler)
   - 레드-블랙 트리(Red-Black Tree)를 사용하여 프로세스에게 CPU 시간을 공평하게 분배하는 기본 스케줄러이다.
3. 실시간 스케줄링(Real-Time Scheduling)
   - 우선순위 기반의 FIFO 및 Round-Robin 스케줄링 정책을 지원하여 응답성을 보장한다.

📢 섹션 요약 비유: 프로세스 스케줄러는 '공정한 초등학교 선생님'과 같아서, 모든 아이가 놀이터 기구를 골고루 탈 수 있게 시간을 나눠주고 기록하는 역할을 합니다.

+++

### Ⅳ. 가상 파일 시스템(VFS, Virtual File System)과 I/O

1. 추상화 레이어(Abstraction Layer)
   - 사용자 영역에서 서로 다른 파일 시스템(ext4, NFS, Btrfs 등)을 동일한 시스템 콜로 접근할 수 있게 한다.
2. 아이노드(Inode)와 디엔트리(Dentry)
   - 아이노드는 파일의 메타데이터를 저장하고, 디엔트리는 파일 경로와 아이노드 사이의 매핑을 관리한다.
3. 페이지 캐시(Page Cache)
   - 디스크 I/O 성능 향상을 위해 메모리의 일부를 데이터 캐싱 영역으로 활용한다.

📢 섹션 요약 비유: VFS는 '다국어 번역기'와 같아서, 손님이 어떤 언어(파일 시스템)를 쓰든 웨이터(OS)가 다 알아듣고 서비스를 제공할 수 있게 해줍니다.

+++

### Ⅴ. 메모리 관리 서브시스템(Memory Management)

1. 가상 메모리 관리(Virtual Memory Management)
   - 페이징(Paging) 기법을 사용하여 각 프로세스에게 독립된 가상 주소 공간을 제공한다.
2. 슬랩 할당자(Slab Allocator)
   - 커널 내부 자료구조를 위해 메모리 할당 및 해제 속도를 극대화하는 객체 캐싱 메커니즘이다.
3. OOM(Out of Memory) 킬러
   - 메모리가 고갈되었을 때 시스템 전체의 붕괴를 막기 위해 특정 프로세스를 강제로 종료하는 최후의 수단이다.

📢 섹션 요약 비유: 메모리 관리는 '유능한 호텔 총지배인'과 같아서, 손님들에게 방(메모리)을 배정하고 방이 모자라면 안 쓰는 방을 정리하거나 손님을 내보내는 결단을 내립니다.
