# 클론 시스템 콜 (프로세스 생성)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 리눅스의 통합 프로세스/스레드 생성 시스템 콜
> 2. **가치**: 세밀한 자원 공유 제어, 경량 프로세스
> 3. **융합**: fork, pthread, namespace와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

clone()은 리눅스에서 프로세스나 스레드를 생성하는 저수준 시스템 콜로, 공유 수준을 세밀하게 제어합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    클론 시스템 콜                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              clone() 시그니처                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  int clone(int (*fn)(void *), void *child_stack,    │   │ │
│   │   │            int flags, void *arg, ...);              │   │ │
│   │   │                                                      │   │ │
│   │   │  flags로 공유 수준 결정:                              │   │ │
│   │   │  • CLONE_VM: 메모리 공유 (스레드)                     │   │ │
│   │   │  • CLONE_FS: 파일 시스템 정보 공유                    │   │ │
│   │   │  • CLONE_FILES: 파일 디스크립터 공유                  │   │ │
│   │   │  • CLONE_SIGHAND: 시그널 핸들러 공유                  │   │ │
│   │   │  • CLONE_THREAD: 스레드 그룹                         │   │ │
│   │   │  • CLONE_NEWNS: 새 namespace                         │   │ │
│   │   │  • CLONE_NEWPID: 새 PID namespace                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              fork vs pthread vs clone                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  fork():                                               │   │ │
│   │   │  • clone(SIGCHLD, 0)                                  │   │ │
│   │   │  • 모든 것 복사 (COW)                                  │   │ │
│   │   │  • 완전히 독립된 프로세스                               │   │ │
│   │   │                                                      │   │ │
│   │   │  pthread_create():                                     │   │ │
│   │   │  • clone(CLONE_VM | CLONE_FS | CLONE_FILES |          │   │ │
│   │   │         CLONE_SIGHAND | CLONE_THREAD | ...)           │   │ │
│   │   │  • 거의 모든 것 공유                                    │   │ │
│   │   │  • 사용자 수준 스레드                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  clone():                                              │   │ │
│   │   │  • flags로 세밀한 제어                                 │   │ │
│   │   │  • 컨테이너, sandbox 활용                               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: clone()은 fork와 pthread의 내부 구현입니다.

**💡 비유**: clone()은 레고 블록처럼 필요한 만큼만 공유합니다.

**📢 섹션 요약 비유**: flags가 공유 결정!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **fn** | 함수 | 자식 진입점 | 시작점 |
| **child_stack** | 스택 | 자식 스택 | 메모장 |
| **flags** | 플래그 | 공유 옵션 | 설정 |
| **ptid/ctid** | ID | TID 저장 | 식별자 |

**주요 flags**

| Flag | 효과 |
|:---|:---|
| **CLONE_VM** | 메모리 공유 |
| **CLONE_FILES** | fd 공유 |
| **CLONE_THREAD** | 스레드 그룹 |
| **CLONE_NEWPID** | 새 PID NS |

**핵심 알고리즘**: task_struct 복사 → flags 적용 → 자원 공유/복사 → 실행

**📢 섹션 요약 비유**: CLONE_VM이 스레드!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 생성 방식**

| 방식 | 메모리 | fd | 시그널 |
|:---|:---:|:---:|:---:|
| **fork** | 복사 | 복사 | 복사 |
| **pthread** | 공유 | 공유 | 공유 |
| **clone** | 선택 | 선택 | 선택 |

**📢 섹션 요약 비유**: clone이 최대 유연성!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 프로세스** → fork()
**시나리오 2: 스레드** → pthread_create()
**시나리오 3: 컨테이너** → clone(CLONE_NEW*)

**도입 체크리스트**: 격리 수준, 성능 요구

**📢 섹션 요약 비유**: 컨테이너는 clone!
