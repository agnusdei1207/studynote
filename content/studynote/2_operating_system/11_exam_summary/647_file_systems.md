> **[Insight]**
> 파일 시스템은 보조기억장치의 물리적인 저장 공간을 추상화하여 사용자가 데이터를 논리적인 파일과 디렉토리 구조로 저장, 관리, 검색할 수 있게 하는 핵심 서비스이다.
> 데이터의 효율적 할당(Allocation)과 빈 공간 관리(Free Space Management), 그리고 시스템 장애 시 데이터의 일관성을 보장하는 저널링(Journaling) 기법 등이 기술적 완성도의 핵심이다.
> 현대의 컴퓨팅 환경은 단일 시스템을 넘어 네트워크상에서 자원을 공유하는 분산 파일 시스템(DFS)과 대규모 데이터 처리를 위한 클라우드 스토리지 형태로 진화하고 있다.

+++

### Ⅰ. 파일과 디렉토리의 논리적 구조

1. 파일(File)의 정의와 속성
   - 관련 있는 데이터의 집합체로, OS에 의해 관리되는 논리적 저장 단위이다.
   - 이름, 유형, 위치, 크기, 보호 설정, 타임스탬프 등의 메타데이터(Metadata)를 가진다.
2. 디렉토리(Directory) 구조
   - 파일들을 체계적으로 분류하기 위한 구조 (1단계, 2단계, 트리 구조, 비순환 그래프 등).
   - 현대 OS는 트리 구조 디렉토리를 통해 효율적인 경로 관리를 제공한다.

📢 섹션 요약 비유: 도서관에서 책(File)들을 장르별(Directory)로 서가에 정리해두어 찾기 편하게 만드는 시스템과 같습니다.

+++

### Ⅱ. 파일 할당 기법(File Allocation Methods)

1. 연속 할당(Contiguous Allocation)
   - 파일을 디스크의 연속된 블록에 저장. 접근 속도가 빠르나 외부 파편화가 발생함.
2. 불연속 할당(Non-contiguous Allocation)
   - **연결 할당(Linked Allocation)**: 블록들을 포인터로 연결. 파편화는 없으나 직접 접근(Direct Access)이 불가능하고 신뢰성이 낮음.
   - **인덱스 할당(Indexed Allocation)**: 파일마다 인덱스 블록을 두어 각 블록의 위치를 저장. 직접 접근이 가능하고 파편화 문제가 없음.

```text
[ Indexed Allocation Diagram ]

   Index Block (File A)
   +---+-------+
   | 0 | block 1 | -> [ Data 1 ]
   | 1 | block 7 | -> [ Data 2 ]
   | 2 | block 3 | -> [ Data 3 ]
   +---+-------+
```

📢 섹션 요약 비유: 서류를 한 뭉치로 묶어 두거나(연속), 각 페이지 뒤에 다음 페이지 번호를 적어 두거나(연결), 첫 페이지에 전체 페이지 번호를 다 적어 두는(인덱스) 방식들입니다.

+++

### Ⅲ. 빈 공간 관리와 저널링(Journaling)

1. 빈 공간 관리(Free Space Management)
   - 비트맵(Bit Map)이나 연결 리스트(Linked List)를 사용하여 사용 가능한 블록 추적.
2. 저널링(Journaling) 기법
   - 데이터를 쓰기 전 변경 사항을 로그(Log) 파일에 먼저 기록함.
   - 갑작스러운 전원 차단이나 시스템 오류 시 로그를 기반으로 빠른 데이터 복구(Recovery) 및 일관성 보장.

📢 섹션 요약 비유: 일기를 먼저 쓰고 일을 처리하는 습관 덕분에, 중간에 무슨 일이 생겨도 어디까지 했는지 바로 알 수 있는 지혜로운 기록법입니다.

+++

### Ⅳ. 분산 파일 시스템(DFS, Distributed File System)

1. 개념
   - 네트워크로 연결된 여러 서버의 파일들을 마치 로컬 파일처럼 투명하게(Transparency) 접근할 수 있게 해주는 시스템.
2. 특징
   - 위치 투명성(Location Transparency): 파일의 실제 물리적 위치를 알 필요가 없음.
   - 복제 투명성(Replication Transparency): 파일이 여러 곳에 복제되어 있어도 하나의 파일처럼 보임.
3. 대표 사례
   - **NFS (Network File System)**: 유닉스 계열 표준 분산 파일 시스템.
   - **AFS (Andrew File System)**: 대규모 환경을 위한 캐싱 중심 분산 파일 시스템.

📢 섹션 요약 비유: 내 가방에 있는 책뿐만 아니라 친구 가방에 있는 책도 내 가방에 있는 것처럼 자연스럽게 꺼내서 볼 수 있게 해주는 마법 같은 기술입니다.

+++

### Ⅴ. 현대적 파일 시스템과 클라우드

1. 리눅스/윈도우 파일 시스템
   - Ext4, XFS (리눅스): 대용량 데이터 및 저널링 강화.
   - NTFS (윈도우): 보안, 압축, 할당량 관리 등 고성능 기능 제공.
2. 클라우드 및 객체 스토리지(Object Storage)
   - 데이터와 메타데이터를 하나의 객체로 관리하여 무한 확장성 제공 (S3 등).
   - 대규모 비정형 데이터 처리에 최적화.

📢 섹션 요약 비유: 집집마다 있는 작은 서재를 넘어, 전 세계 사람들이 함께 쓰는 거대한 디지털 도서관으로 확장된 오늘날의 모습입니다.
