# NFS (네트워크 파일 시스템 마운트)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크를 통해 원격 파일 시스템을 로컬처럼 접근
> 2. **가치**: 파일 공유, 중앙 관리, 투명한 접근
> 3. **융합**: RPC, VFS, 캐싱, ID 매핑과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NFS(Network File System)는 네트워크를 통해 원격 파일 시스템을 로컬에 마운트하여 투명하게 접근하는 프로토콜입니다.

> **해설**: NFS는 VFS를 통해 로컬 파일 시스템처럼 투명하게 접근합니다.

**💡 비유**: NFS는 원격 창고를 내 방처럼 쓰는 것과 같습니다.

**📢 섹션 요약 비유**: 원격 파일이 로컬처럼!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **RPC** | 원격 호출 | XDR 인코딩 | 전화 |
| **VFS** | 추상화 | 통일 인터페이스 | 번역기 |
| **Mountd** | 마운트 | 익스포트 확인 | 관리자 |
| **NFSd** | 서비스 | 파일 작업 | 직원 |

**주요 작업**

| 작업 | 설명 |
|:---|:---|
| **GETATTR** | 속성 조회 |
| **READ** | 읽기 |
| **WRITE** | 쓰기 |
| **LOOKUP** | 탐색 |

**핵심 알고리즘**: RPC → XDR 직렬화 → TCP 전송 → 처리 → 응답

**📢 섹션 요약 비유**: RPC가 핵심!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 파일 공유**

| 프로토콜 | 플랫폼 | 포트 | 인증 |
|:---|:---|:---:|:---|
| **NFS** | Unix/Linux | 2049 | Kerberos |
| **SMB** | Windows | 445 | NTLM/Kerb |
| **CIFS** | Windows | 445 | NTLM |

**📢 섹션 요약 비유**: NFS는 Unix 표준!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 홈 디렉토리** → NFS + LDAP
**시나리오 2: 컨테이너 스토리지** → NFS PV
**시나리오 3: HPC** → pNFS + RDMA

**도입 체크리스트**: 보안, 성능, 동시성

**📢 섹션 요약 비유**: NFSv4가 보안 강화!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 중앙 관리, 스토리지 공유

**미래 전망**: NVMe-oF, RDMA 통합

**참고 표준**: RFC 5661 (NFSv4.1)

**📢 섹션 요약 비유**: NFS는 네트워크 파일 공유의 클래식!
