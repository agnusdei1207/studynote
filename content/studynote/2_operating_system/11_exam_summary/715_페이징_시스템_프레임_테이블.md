# 페이징 시스템 (프레임 테이블)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 메모리를 고정 크기 프레임으로, 논리 메모리를 페이지로 분할하여 비연속 할당을 가능하게 하는 메모리 관리 기법
> 2. **가치**: 외부 단편화 해결, 가상 메모리 기반, 메모리 보호
> 3. **융합**: MMU, TLB, 페이지 테이블, 프레임 테이블과 통합된 메모리 관리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

페이징 시스템(Paging System)은 물리 메모리를 고정 크기 프레임으로, 논리 메모리를 페이지로 분할하여 비연속 할당을 가능하게 하는 메모리 관리 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    페이징 시스템 (Paging System)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              기본 개념                                        │ │
│   │                                                              │ │
│   │   페이지 (Page): 논리 메모리의 고정 크기 블록                   │ │
│   │   프레임 (Frame): 물리 메모리의 고정 크기 블록                  │ │
│   │   페이지 크기 = 프레임 크기 (보통 4KB)                         │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              논리 메모리 → 물리 메모리 매핑                     │ │
│   │                                                              │ │
│   │   논리 주소 공간 (프로세스)              물리 메모리            │ │
│   │   ┌─────────────────┐                ┌─────────────────┐    │ │
│   │   │ Page 0          │                │ Frame 2         │    │ │
│   │   │ Page 1          │ ──페이지 테이블──→│ Frame 5         │    │ │
│   │   │ Page 2          │                │ Frame 1         │    │ │
│   │   │ Page 3          │                │ Frame 8         │    │ │
│   │   └─────────────────┘                └─────────────────┘    │ │
│   │          │                                    ↑              │ │
│   │          │         페이지 테이블               │              │ │
│   │          │  ┌─────────────────────────┐      │              │ │
│   │          │  │ Page 0 → Frame 2        │──────┘              │ │
│   │          │  │ Page 1 → Frame 5        │                     │ │
│   │          │  │ Page 2 → Frame 1        │                     │ │
│   │          │  │ Page 3 → Frame 8        │                     │ │
│   │          │  └─────────────────────────┘                     │ │
│   │          │                                                   │ │
│   │   비연속 할당: 페이지들이 연속되지 않은 프레임에 매핑          │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              프레임 테이블 (Frame Table)                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │ Frame │ 상태  │ 프로세스 │ 페이지 │                  │   │ │
│   │   │───────┼───────┼─────────┼────────│                  │   │ │
│   │   │   0   │ 사용  │   P1    │   2    │                  │   │ │
│   │   │   1   │ 사용  │   P2    │   0    │                  │   │ │
│   │   │   2   │ 빈    │    -    │   -    │                  │   │ │
│   │   │   3   │ 사용  │   P1    │   0    │                  │   │ │
│   │   │  ...  │  ...  │   ...   │  ...   │                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 페이징은 외부 단편화를 완전히 해결합니다.

**💡 비유**: 페이징은 책의 페이지를 책장의 임의 위치에 꽂는 것과 같습니다.

**📢 섹션 요약 비유**: 페이지는 책장에 흩어져 있어요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **페이지** | 논리 블록 | 번호 + 오프셋 | 책 페이지 |
| **프레임** | 물리 블록 | 번호 + 오프셋 | 책장 칸 |
| **페이지 테이블** | 매핑 정보 | P→F 매핑 | 목차 |
| **프레임 테이블** | 할당 상태 | 사용/빈 추적 | 대장 |

**주소 변환 구조**

| 구성 | 비트 수 | 역할 |
|:---|:---|:---|
| **페이지 번호** | p비트 | 페이지 테이블 인덱스 |
| **페이지 오프셋** | d비트 | 페이지 내 위치 |
| **프레임 번호** | f비트 | 물리 프레임 |
| **물리 주소** | f+d | 최종 주소 |

**핵심 알고리즘**: 논리 주소 → 페이지 번호/오프셋 분리 → PT 조회 → 프레임 결합

**📢 섹션 요약 비유**: p비트 + d비트 = 논리 주소!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 페이징 vs 세그멘테이션**

| 비교 항목 | 페이징 | 세그멘테이션 |
|:---|:---:|:---:|
| **크기** | 고정 | 가변 |
| **외부 단편화** | 없음 | 있음 |
| **내부 단편화** | 있음 | 없음 |
| **사용자 인식** | 없음 | 있음 |

**📢 섹션 요약 비유**: 페이징이 단편화 문제를 해결해요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 4KB 페이지** → 표준 크기, 내부 단편화 평균 2KB
**시나리오 2: Huge Pages** → 2MB/1GB, TLB 효율 증가
**시나리오 3: 커널 메모리** → 연속 프레임 필요

**도입 체크리스트**: 페이지 크기 선택, PT 크기 관리, TLB 활용

**📢 섹션 요약 비유**: 4KB가 표준이에요!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 외부 단편화 제거, 가상 메모리 지원

**미래 전망**: 다양한 페이지 크기, Transparent Huge Pages

**참고 표준**: x86 페이지, ARM 페이지, 4KB 표준

**📢 섹션 요약 비유**: 페이징은 현대 OS의 기본이에요!
