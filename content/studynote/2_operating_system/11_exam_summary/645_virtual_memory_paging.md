> **[Insight]**
> 가상 메모리(Virtual Memory)는 프로세스 전체가 메모리에 있지 않아도 실행이 가능하도록 하여 물리적 메모리 한계를 극복하고 멀티프로그래밍의 정도를 비약적으로 높여주는 기술이다.
> 실제 필요한 부분만 메모리에 적재하는 요구 페이징(Demand Paging) 기법이 핵심이며, 한정된 프레임 자원을 효율적으로 순환시키기 위한 페이지 교체 알고리즘(Page Replacement Algorithm)의 최적화가 필수적이다.
> 빈번한 페이지 부재(Page Fault)로 인해 CPU 이용률이 급격히 저하되는 스래싱(Thrashing) 현상을 방지하기 위해 워킹 셋(Working Set) 모델 등 정교한 프레임 할당 전략이 수반된다.

+++

### Ⅰ. 가상 메모리와 요구 페이징(Demand Paging)

1. 가상 메모리의 의의
   - 프로세스마다 0부터 시작하는 가상의 주소 공간을 제공하여 물리 메모리 크기보다 큰 프로그램 실행을 가능하게 함.
2. 요구 페이징(Demand Paging)
   - 프로그램 실행 시 모든 페이지를 올리지 않고, 실제로 필요한 페이지만 메모리에 적재하는 방식.
   - 유효-무효 비트(Valid-Invalid Bit)를 사용하여 페이지 적재 여부 확인.
3. 페이지 부재(Page Fault)
   - 접근하려는 페이지가 메모리에 없을 때 하드웨어(MMU)가 OS에 보내는 트랩(Trap) 신호.
   - OS는 디스크에서 해당 페이지를 찾아 빈 프레임에 적재하고 페이지 테이블을 업데이트함.

📢 섹션 요약 비유: 서재의 모든 책을 책상 위에 다 꺼내 놓는 것이 아니라, 지금 읽을 '페이지'만 한 장씩 꺼내서 보는 것과 같습니다.

+++

### Ⅱ. 페이지 교체 알고리즘(Page Replacement)

1. 페이지 교체 알고리즘의 목표
   - 페이지 부재율(Page Fault Rate)을 최소화하여 I/O 비용을 줄임.
2. 주요 알고리즘
   - **FIFO(First-In-First-Out)**: 먼저 들어온 페이지를 먼저 교체. Belady의 모순(Belady's Anomaly) 발생 가능.
   - **Optimal Algorithm**: 미래에 가장 오랫동안 사용되지 않을 페이지 교체 (이론적 최적).
   - **LRU(Least Recently Used)**: 가장 오랫동안 사용되지 않은 페이지 교체 (성능 우수).
   - **LFU(Least Frequently Used)**: 참조 횟수가 가장 적은 페이지 교체.
   - **Second-Chance (Clock)**: 참조 비트(Reference Bit)를 사용하여 LRU를 근사화한 방식.

```text
[ Page Replacement (LRU Example) ]

   Reference Stream: 7, 0, 1, 2, 0, 3, 0, 4
   
   Frames: [ 7 ] -> [ 7, 0 ] -> [ 7, 0, 1 ] -> [ 2, 0, 1 ] (7 is replaced as LRU)
```

📢 섹션 요약 비유: 책상이 꽉 찼을 때, '가장 오래전에 본 책(LRU)'을 선반으로 다시 보내고 새 책을 놓는 영리한 정리법입니다.

+++

### Ⅲ. 스래싱(Thrashing)과 원인 분석

1. 스래싱(Thrashing)의 정의
   - 프로세스가 실제 실행 시간보다 페이지를 교체하는 데 더 많은 시간을 소비하여 CPU 이용률이 급격히 떨어지는 현상.
2. 발생 메커니즘
   - 프레임 부족 -> 잦은 페이지 부재 -> I/O 대기 증가 -> CPU 이용률 저하 -> OS가 멀티프로그래밍 정도를 높임 -> 악순환 반복.

📢 섹션 요약 비유: 공부방이 너무 좁아서 책을 한 장 넘길 때마다 다른 책을 창고에서 가져와야 하는 바람에 정작 공부는 한 글자도 못 하고 책만 옮기다 끝나는 상황입니다.

+++

### Ⅳ. 스래싱 방지 및 프레임 할당 전략

1. 워킹 셋(Working Set) 모델
   - 프로세스가 일정 시간 동안 집중적으로 참조하는 페이지들의 집합(Working Set)을 메모리에 동시에 유지하도록 보장함.
   - 지역성(Locality) 원리에 근거하여 스래싱을 예방.
2. 페이지 부재 빈도(PFF, Page Fault Frequency) 조절
   - 페이지 부재율의 상한선과 하한선을 정하여 프레임 수를 동적으로 조절함.
3. 전역 교체(Global Replacement) vs 지역 교체(Local Replacement)
   - 다른 프로세스의 프레임을 뺏어올지, 자신에게 할당된 프레임 내에서만 교체할지 결정.

📢 섹션 요약 비유: 시험 기간에 꼭 필요한 '핵심 요약 노트들(Working Set)'만큼은 항상 책상 위에 있도록 자리를 보장해주는 정책입니다.

+++

### Ⅴ. 기타 가상 메모리 관리 기술

1. 페이지 크기 결정(Page Size)
   - 작은 페이지: 내부 파편화 감소, 페이지 테이블 커짐.
   - 큰 페이지: I/O 효율 증가, 지역성 저하 및 내부 파편화 증가.
2. 쓰기 시 복사(COW, Copy-On-Write)
   - fork() 시 부모와 자식이 페이지를 공유하다가 수정이 발생할 때만 복사하여 메모리 절약.
3. 메모리 매핑 파일(Memory-Mapped Files)
   - 디스크 파일을 가상 메모리 공간에 매핑하여 파일 I/O 속도 향상.

📢 섹션 요약 비유: 원고지를 아주 잘게 나누면 빈칸이 적지만 관리가 힘들고, 크게 나누면 관리는 쉽지만 빈칸이 많이 생기는 고민과 같습니다.
