> **[Insight]**
> 프로세스 동기화는 병행 실행되는 프로세스들이 공유 자원에 접근할 때 데이터의 일관성(Consistency)을 유지하기 위한 필수적인 메커니즘이다.
> 동기화 문제를 완벽히 해결하지 못하면 경쟁 상태(Race Condition)가 발생하며, 반대로 과도하거나 잘못된 동기화 설계는 시스템을 영구적으로 중단시키는 교착상태(Deadlock)를 초래한다.
> 따라서 효율적인 상호 배제(Mutual Exclusion) 구현과 교착상태 방지/회피 전략 사이의 균형을 맞추는 것이 동시성 프로그래밍의 핵심 과제이다.

+++

### Ⅰ. 프로세스 동기화의 기초와 경쟁 상태

1. 경쟁 상태(Race Condition)
   - 여러 프로세스가 동시에 공유 자원에 접근하여 실행 결과가 접근 순서에 따라 달라지는 비결정적인 상태이다.
2. 임계 영역(Critical Section)
   - 공유 자원에 접근하는 코드 영역으로, 오직 하나의 프로세스만 진입 가능해야 한다.
3. 임계 영역 문제 해결의 3대 조건
   - **상호 배제(Mutual Exclusion)**: 한 프로세스가 임계 영역에 있으면 다른 프로세스는 진입 불가.
   - **진행(Progress)**: 임계 영역에 있는 프로세스가 없으면 진입하려는 프로세스 중 하나는 들어갈 수 있어야 함.
   - **한정 대기(Bounded Waiting)**: 대기 중인 프로세스는 언젠가는 임계 영역에 진입할 수 있어야 함 (기아 방지).

📢 섹션 요약 비유: 화장실(Critical Section)에 여러 사람이 동시에 들어가려고 하면 사고가 나므로, '문 잠금장치'를 통해 한 명씩만 이용하게 하는 것과 같습니다.

+++

### Ⅱ. 동기화 도구: Mutex, Semaphore, Monitor

1. 뮤텍스(Mutex - Mutual Exclusion)
   - Boolean 변수(Lock)를 사용하여 임계 영역을 잠그고 해제하는 이진 동기화 도구이다.
2. 세마포어(Semaphore)
   - 정수형 변수(S)를 사용하여 가용한 자원의 개수를 관리한다.
   - P(wait) 연산과 V(signal) 연산을 통해 자원 점유 및 반납을 제어한다.
3. 모니터(Monitor)
   - 프로그래밍 언어 차원에서 제공하는 고수준 동기화 도구로, 공유 자원과 접근 함수를 캡슐화하여 자동으로 상호 배제를 보장한다.

```text
[ Semaphore Mechanism ]

      Process A                Semaphore (S=1)               Process B
          |                          |                           |
    P(S) 요청 (S=1) ----------------> S=0                        |
    (Critical Section)               |                           |
          |                          |                     P(S) 요청 (S=0)
          |                          |                     (Wait in Queue)
    V(S) 신호 (S=0) ----------------> S=1                        |
          |                          | ----------------------> Wake up
          |                          |                     (Critical Section)
```

📢 섹션 요약 비유: 뮤텍스는 화장실 '열쇠 하나'를 주고받는 것이고, 세마포어는 '번호표'를 뽑아 빈 자리가 날 때까지 기다리는 것과 같습니다.

+++

### Ⅲ. 교착상태(Deadlock)의 정의와 발생 조건

1. 교착상태(Deadlock)의 정의
   - 둘 이상의 프로세스가 서로 상대방이 가진 자원을 기다리며 무한히 대기하는 상태이다.
2. 교착상태 발생의 4가지 필요충분조건 (Coffman 조건)
   - **상호 배제(Mutual Exclusion)**: 자원은 한 번에 한 프로세스만 사용 가능.
   - **점유 및 대기(Hold and Wait)**: 자원을 가진 채로 다른 자원을 기다림.
   - **비선점(No Preemption)**: 다른 프로세스의 자원을 강제로 뺏을 수 없음.
   - **환형 대기(Circular Wait)**: 대기 프로세스 간에 고리 모양의 대기 관계 형성.

📢 섹션 요약 비유: 좁은 골목길에서 두 대의 차가 마주 보고 서서 서로 '먼저 뒤로 빼라'며 한 발짝도 못 움직이는 상황과 같습니다.

+++

### Ⅳ. 교착상태 해결 전략

1. 예방(Prevention)
   - 4가지 발생 조건 중 하나를 제거한다. 자원 낭비가 심하다.
2. 회피(Avoidance)
   - 자원 할당 시 교착상태 가능성을 동적으로 검사한다. (은행원 알고리즘 - Banker's Algorithm). 안전 상태(Safe State) 유지 여부 확인.
3. 탐지 및 복구(Detection and Recovery)
   - 교착상태 발생을 허용하되, 주기적으로 탐지하여 프로세스 종료나 자원 선점을 통해 해결한다.
4. 무시(Ignorance)
   - 교착상태가 드물게 발생한다고 가정하고 아무 조치도 취하지 않는다 (타조 알고리즘 - Ostrich Algorithm). 현대 범용 OS(Windows, Unix 등)에서 주로 채택.

📢 섹션 요약 비유: 사고가 안 나게 신호등을 세우거나(예방), 차가 오는지 미리 확인하거나(회피), 사고가 나면 견인차로 치우는(복구) 방법들입니다.

+++

### Ⅴ. 전통적인 동기화 문제들 (Classic Problems)

1. 유한 버퍼 문제 (Bounded-Buffer Problem)
   - 생산자-소비자 문제. 버퍼가 꽉 차면 생산자 대기, 비어 있으면 소비자 대기.
2. 읽기-쓰기 문제 (Readers-Writers Problem)
   - 읽기 중에는 여러 명 허용, 쓰기 중에는 독점적 접근 필요.
3. 식사하는 철학자 문제 (Dining Philosophers Problem)
   - 인접한 자원을 동시에 확보해야 할 때 발생하는 교착상태와 기아 현상의 전형적 예시.

📢 섹션 요약 비유: 주방장(생산자)이 음식을 내놓는 속도와 손님(소비자)이 먹는 속도를 맞추는 문제, 도서관에서 책을 보거나 고치는 규칙을 정하는 문제들입니다.
