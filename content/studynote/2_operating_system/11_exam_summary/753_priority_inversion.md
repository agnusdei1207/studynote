# 276. 우선순위 역전 (Priority Inversion)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 낮은 우선순위가 높은 우선순위를 간접적으로 블록
> 2. **가치**: 실시간 시스템의 치명적 문제
> 3. **융합**: 우선순위 상속, 우선순위 상한과 연관

---

## Ⅰ. 개요

### 개념 정의

우선순위 역전(Priority Inversion)은 **낮은 우선순위 스레드가 높은 우선순위 스레드의 실행을 간접적으로 차단하는 현상**이다. 실시간 시스템에서 치명적인 문제다.

### 💡 비유: 고속도로
우선순위 역전은 **고속도로의 일반 차선**과 같다. 구급차(높은 우선순위)가 일반 차(낮은 우선순위) 뒤에 갇힌다.

### 우선순위 역전 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                우선순위 역전 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【문제 시나리오】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  우선순위: High (H) > Medium (M) > Low (L)                        │ │
│  │                                                             │ │
│  │  시간선:                                                         │ │
│  │  ─────────────────────────────────────────────────────────►      │ │
│  │                                                             │ │
│  │  1. L이 락 획득                              │ │
│  │     ┌──────────────────────────────────────────────────┐    │ │
│  │     │ L (Low)                                           │    │ │
│  │     │ 락 보유 중...                                       │    │ │
│  │     └──────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  2. H가 락 요청 (블록됨)                                          │ │
│  │                    ┌─────────────────────────────────────┐  │ │
│  │                    │ H (High) - 락 대기                   │  │ │
│  │                    │ (L이 락을 놓을 때까지)                 │  │ │
│  │                    └─────────────────────────────────────┘  │ │
│  │                                                             │ │
│  │  3. M가 실행 (H보다 먼저!)                                        │ │
│  │                              ┌──────────────────────────┐  │ │
│  │                              │ M (Medium)               │  │ │
│  │                              │ CPU 선점                 │  │ │
│  │                              └──────────────────────────┘  │ │
│  │                                                             │ │
│  │  결과: High > Medium > Low 순서지만                           │ │
│  │        H가 M보다 늦게 실행됨 (역전!)                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【영향】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 실시간 데드라인 위반                                             │ │
│  │  • 시스템 응답 시간 저하                                            │ │
│  │  • 예측 불가능한 동작                                               │ │
│  │  • 안전 시스템 오작동                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                우선순위 역전 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【역전 유형】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 단순 역전 (Simple Inversion):                                 │ │
│  │     • H가 L이 보유한 락 대기                                        │ │
│  │     • 짧은 기간                                                    │ │
│  │                                                             │ │
│  │  2. 연쇄 역전 (Chained Inversion):                                │ │
│  │     • H → L1 → L2 → ... → Ln                                     │ │
│  │     • 다중 락                                                      │ │
│  │                                                             │ │
│  │  3. 무기한 역전 (Unbounded Inversion):                            │ │
│  │     • 중간 우선순위 스레드들이 계속 개입                             │ │
│  │     • H가 무한정 대기                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결 방법】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 우선순위 상속 (Priority Inheritance):                          │ │
│  │     • L이 H의 우선순위로 승격                                       │ │
│  │     • 빠르게 락 해제 유도                                           │ │
│  │                                                             │ │
│  │  2. 우선순위 상한 (Priority Ceiling):                              │ │
│  │     • 락 획득 시 미리 정해진 상한으로 승격                            │ │
│  │                                                             │ │
│  │  3. 락 비선점:                                                     │ │
│  │     • 락 보유 중 선점 방지                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실제 사례】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Mars Pathfinder (1997):                                          │ │
│  │  • 우선순위 역전으로 재부팅 반복                                     │ │
│  │  • 지구에서 패치 업로드로 해결                                       │ │
│  │  • Priority Inheritance Protocol 적용                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux rt_mutex (Priority Inheritance)】                           │
│  ──────────────────                                                  │
│  // 커널 설정                                                         │
│  CONFIG_RT_MUTEXES=y                                                 │
│                                                                     │
│  // 사용                                                             │
│  #include <linux/rtmutex.h>                                         │
│                                                                     │
│  struct rt_mutex lock;                                              │
│  rt_mutex_init(&lock);                                              │
│                                                                     │
│  rt_mutex_lock(&lock);    // 우선순위 상속 자동                       │
│  // 임계 영역                                                         │
│  rt_mutex_unlock(&lock);                                            │
│                                                                     │
│  【POSIX Priority Inheritance Mutex】                                 │
│  ──────────────────                                                  │
│  #include <pthread.h>                                               │
│                                                                     │
│  pthread_mutexattr_t attr;                                          │
│  pthread_mutexattr_init(&attr);                                     │
│  pthread_mutexattr_setprotocol(&attr, PTHREAD_PRIO_INHERIT);        │
│                                                                     │
│  pthread_mutex_t mutex;                                             │
│  pthread_mutex_init(&mutex, &attr);                                 │
│                                                                     │
│  【Linux 우선순위 확인】                                               │
│  ──────────────────                                                  │
│  // 실시간 우선순위 설정                                               │
│  chrt -f -p 50 <pid>    // FIFO 스케줄링, 우선순위 50                 │
│  chrt -r -p 30 <pid>    // RR 스케줄링, 우선순위 30                   │
│                                                                     │
│  // 우선순위 확인                                                     │
│  chrt -p <pid>                                                       │
│                                                                     │
│  【우선순위 역전 감지】                                                 │
│  ──────────────────                                                  │
│  // ftrace로 추적                                                    │
│  echo 1 > /sys/kernel/debug/tracing/events/sched/sched_switch/enable│
│  cat /sys/kernel/debug/tracing/trace_pipe                           │
│                                                                     │
│  // eBPF로 모니터링                                                   │
│  // bcc/tools/pidpersec.c                                           │
│                                                                     │
│  【성능 분석】                                                          │

│  ┌───────────────────────────────────────────────────────────────┐ │

│  │  ────────              ──────        ──────        ──────      │ │

```

