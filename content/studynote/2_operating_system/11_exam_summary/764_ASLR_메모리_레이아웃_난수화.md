# ASLR (메모리 레이아웃 난수화)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 주소를 무작위로 배치하여 공격 난이도 증가
> 2. **가치**: 버퍼 오버플로우 방어, ROP 공격 완화
> 3. **융합**: DEP/NX, PIE, 보안 메커니즘과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ASLR(Address Space Layout Randomization)은 메모리 주소를 무작위로 배치하여 공격 난이도를 높이는 보안 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ASLR (Address Space Layout Randomization)         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ASLR 없이                                        │ │
│   │                                                              │ │
│   │   메모리 주소 공간 (항상 동일):                                │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  0x08048000: Code (항상 같음)                        │   │ │
│   │   │  0x08049000: Data (항상 같음)                        │   │ │
│   │   │  0xbfff0000: Stack (항상 같음)                       │   │ │
│   │   │  0xb7e00000: Heap (항상 같음)                        │   │ │
│   │   │  0xb7f00000: Library (항상 같음)                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   공격자가 정확한 주소를 알 수 있음 → 공격 쉬움               │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ASLR 적용                                        │ │
│   │                                                              │ │
│   │   메모리 주소 공간 (매번 다름):                                │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  실행 1회차:                                          │   │ │
│   │   │  0x5634a000: Code (무작위)                           │   │ │
│   │   │  0x5634b000: Data                                    │   │ │
│   │   │  0x7ffe8a00: Stack (무작위)                          │   │ │
│   │   │  0x55b3c000: Heap (무작위)                           │   │ │
│   │   │  0x7f4a2000: Library (무작위)                        │   │ │
│   │   │                                                      │   │ │
│   │   │  실행 2회차:                                          │   │ │
│   │   │  0x55a3e000: Code (다른 값)                          │   │ │
│   │   │  0x7ffd3c00: Stack (다른 값)                         │   │ │
│   │   │  ...                                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   공격자가 주소를 추측하기 어려움 → 공격 어려움               │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ASLR 적용 영역                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  • Stack Randomization: 스택 주소 난수화             │   │ │
│   │   │  • Heap Randomization: 힙 주소 난수화                │   │ │
│   │   │  • Library Randomization: 공유 라이브러리 난수화     │   │ │
│   │   │  • PIE (Position Independent Executable): 코드 난수화│   │ │
│   │   │  • VDSO Randomization: VDSO 주소 난수화              │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   확인: /proc/sys/kernel/randomize_va_space                │ │
│   │   0: 끔, 1: 부분, 2: 전체                                   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: ASLR은 주소를 숨겨 공격을 어렵게 합니다.

**💡 비유**: ASLR은 집 주소를 매일 바꾸는 것과 같습니다.

**📢 섹션 요약 비유**: 주소를 숨겨요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Stack ASLR** | 스택 난수화 | RSP offset | 층 |
| **Heap ASLR** | 힙 난수화 | brk offset | 방 |
| **PIE** | 코드 난수화 | 위치 독립 | 건물 |
| **mmap ASLR** | 라이브러리 | 랜덤 베이스 | 지도 |

**설정 값**

| 값 | 설명 |
|:---|:---|
| **0** | ASLR 끔 |
| **1** | 스택 + 라이브러리 |
| **2** | 전체 (기본값) |

**핵심 알고리즘**: exec → 랜덤 오프셋 → 주소 할당

**📢 섹션 요약 비유**: 2가 기본값!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 웹 서버** → ASLR + DEP + PIE
**시나리오 2: 레거시** → PIE로 재빌드
**시나리오 3: 컨테이너** → 호스트 ASLR 상속

**도입 체크리스트**: PIE 컴파일, randomize_va_space=2

**📢 섹션 요약 비유**: -fPIE -pie로 컴파일!
