# eBPF (확장 버클리 패킷 필터)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 공간에서 안전하게 샌드박스 프로그램 실행
> 2. **가치**: 관측성, 네트워킹, 보안, 성능 (커널 수정 없이)
> 3. **융합**: 네트워크, 트레이싱, 보안, Cilium과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

eBPF(Extended Berkeley Packet Filter)는 커널 공간에서 샌드박스된 프로그램을 안전하게 실행하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    eBPF (확장 버클리 패킷 필터)                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              eBPF 아키텍처                                     │ │
│   │                                                              │ │
│   │   사용자 공간                                                  │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  eBPF 프로그램 (C/Rust) → LLVM → BPF 바이트코드      │   │ │
│   │   └───────────────────────┬─────────────────────────────┘   │ │
│   │                           │ bpf() syscall                    │ │
│   │   ────────────────────────┼────────────────────────────────  │ │
│   │   커널 공간                ▼                                  │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Verifier (안전성 검증)                               │   │ │
│   │   │    ↓                                                  │   │ │
│   │   │  JIT 컴파일 (네이티브 코드)                            │   │ │
│   │   │    ↓                                                  │   │ │
│   │   │  Hook Points:                                         │   │ │
│   │   │  • kprobes (커널 함수 진입/종료)                       │   │ │
│   │   │  • tracepoints (정적 트레이싱)                         │   │ │
│   │   │  • XDP (네트워크 드라이버)                             │   │ │
│   │   │  • tc (트래픽 컨트롤)                                  │   │ │
│   │   │  • cgroups (컨테이너)                                  │   │ │
│   │   │  • sockets                                            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              eBPF 활용 사례                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  관측성 (Observability):                              │   │ │
│   │   │  • bpftrace, BCC, Cilium                             │   │ │
│   │   │  • 성능 분석, 트레이싱                                │   │ │
│   │   │                                                      │   │ │
│   │   │  네트워킹 (Networking):                               │   │ │
│   │   │  • Cilium, Katran, Cloudflare                        │   │ │
│   │   │  • 로드밸런싱, 방화벽, DDoS 방어                      │   │ │
│   │   │                                                      │   │ │
│   │   │  보안 (Security):                                     │   │ │
│   │   │  • Falco, Tetragon, Tracee                           │   │ │
│   │   │  • 런타임 보안, 감사                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: eBPF는 커널 수정 없이 확장 기능을 안전하게 추가합니다.

**💡 비유**: eBPF는 커널에 안전하게 주입하는 미니 프로그램입니다.

**📢 섹션 요약 비유**: 커널 확장 without 모듈!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Verifier** | 검증 | 안전성 확인 | 심사 |
| **JIT** | 컴파일 | 네이티브 변환 | 번역 |
| **Map** | 데이터 | 사용자-커널 통신 | 저장소 |
| **Helper** | 함수 | 커널 API | 도구 |

**Hook Point 종류**

| Hook | 위치 | 용도 |
|:---|:---|:---|
| **XDP** | 드라이버 | 패킷 처리 |
| **tc** | QoS | 트래픽 제어 |
| **kprobe** | 함수 | 트레이싱 |
| **tracepoint** | 정적 | 관측 |

**핵심 알고리즘**: 로드 → Verifier → JIT → Hook → 실행 → Map 통신

**📢 섹션 요약 비유**: XDP가 최고 속도!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 확장 방식**

| 방식 | 안전성 | 성능 | 유연성 |
|:---|:---:|:---:|:---:|
| **커널 모듈** | 낮음 | 높음 | 높음 |
| **eBPF** | 높음 | 높음 | 높음 |
| **iptables** | 높음 | 중간 | 낮음 |

**📢 섹션 요약 비유**: eBPF가 모두 만족!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 관측성** → bpftrace + BCC
**시나리오 2: 네트워크** → Cilium + XDP
**시나리오 3: 보안** → Falco + Tetragon

**도입 체크리스트**: 커널 버전 (4.4+), BTF 지원

**📢 섹션 요약 비유**: 5.10+ 커널 권장!
