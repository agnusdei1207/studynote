# 스핀락 (멀티 프로세서 전용 활용)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락을 얻을 때까지 루프를 돌며 대기하는 동기화 기법
> 2. **가치**: 짧은 대기, 멀티프로세서, 커널 동기화
> 3. **융합**: 인터럽트, 선점, MCS 락과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

스핀락(Spinlock)은 락을 얻을 때까지 루프를 돌며 대기(busy waiting)하는 동기화 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    스핀락 (Spinlock)                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스핀락 동작 원리                                  │ │
│   │                                                              │ │
│   │   CPU 0                      CPU 1                           │ │
│   │     │                          │                              │ │
│   │     │──lock() 시도────────────→│                              │ │
│   │     │   while(locked) { }     │                              │ │
│   │     │   스핀! 스핀! 스핀!      │   (락 보유 중)                │ │
│   │     │   ...                   │                              │ │
│   │     │                          │──unlock()──                 │ │
│   │     │←──────락 획득!───────────│                              │ │
│   │     │                          │                              │ │
│   │   특징:                                                       │ │
│   │   • 컨텍스트 스위치 없음                                      │ │
│   │   • CPU 사용 (busy waiting)                                  │ │
│   │   • 짧은 구간에 적합                                          │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스핀락 vs 뮤텍스                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Spinlock:                                            │   │ │
│   │   │  • 락을 얻을 때까지 CPU 사용 (busy wait)              │   │ │
│   │   │  • 컨텍스트 스위치 없음                                │   │ │
│   │   │  • 짧은 대기에 적합                                    │   │ │
│   │   │  • 단일 CPU에서는 비효율적                             │   │ │
│   │   │  • 인터럽트 컨텍스트에서 사용 가능                     │   │ │
│   │   │                                                      │   │ │
│   │   │  Mutex:                                               │   │ │
│   │   │  • 락을 얻지 못하면 슬립 (blocked)                    │   │ │
│   │   │  • 컨텍스트 스위치 발생                                │   │ │
│   │   │  • 긴 대기에 적합                                      │   │ │
│   │   │  • 단일 CPU에서도 효율적                               │   │ │
│   │   │  • 프로세스 컨텍스트에서만 사용                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   선택 기준: 대기 시간 < 컨텍스트 스위치 비용 → 스핀락       │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스핀락 변형                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  spin_lock(): 기본 스핀락                            │   │ │
│   │   │  spin_lock_irq(): 인터럽트 비활성화 + 락             │   │ │
│   │   │  spin_lock_irqsave(): 이전 IRQ 상태 저장 + 락        │   │ │
│   │   │  spin_lock_bh(): Bottom Half 비활성화 + 락           │   │ │
│   │   │                                                      │   │ │
│   │   │  Ticket Lock: FIFO 순서 보장                         │   │ │
│   │   │  MCS Lock: 캐시 친화적, 스케일러블                   │   │ │
│   │   │  RCU: 읽기 락 없음, 쓰기 지연                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스핀락은 짧은 구간, 멀티 CPU, 인터럽트 컨텍스트에 적합합니다.

**💡 비유**: 스핀락은 문 앞에서 계속 두드리는 것과 같습니다.

**📢 섹션 요약 비유**: 짧으면 스핀락!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **locked** | 상태 | 0/1 | 잠금 |
| **spin** | 대기 | while 루프 | 대기 |
| **irq** | 인터럽트 | 비활성화 | 차단 |
| **bh** | Bottom Half | 비활성화 | 지연 |

**함수 종류**

| 함수 | 설명 |
|:---|:---|
| **spin_lock()** | 기본 |
| **spin_lock_irq()** | IRQ 끔 |
| **spin_lock_irqsave()** | IRQ 저장 |
| **spin_lock_bh()** | BH 끔 |

**핵심 알고리즘**: while(test_and_set(&lock));

**📢 섹션 요약 비유**: irqsave가 안전!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 동기화**

| 기법 | 대기 | 컨텍스트 | 용도 |
|:---|:---|:---:|:---|
| **Spinlock** | Busy | X | 짧음 |
| **Mutex** | Sleep | O | 김 |
| **RCU** | 없음 | X | 읽기 많음 |
| **Seqlock** | 없음 | X | 읽기 많음 |

**📢 섹션 요약 비유**: RCU가 읽기에 최고!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: ISR** → spin_lock_irqsave
**시나리오 2: 커널 짧은 구간** → spin_lock
**시나리오 3: 사용자 공간** → Mutex

**도입 체크리스트**: 대기 시간, 인터럽트 컨텍스트

**📢 섹션 요약 비유**: ISR에선 스핀락만!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 짧은 구간 오버헤드 10배 감소

**미래 전망**: MCS Lock, qspinlock

**참고 표준**: Linux Spinlock API

**📢 섹션 요약 비유**: 스핀락은 커널의 기본 동기화!
