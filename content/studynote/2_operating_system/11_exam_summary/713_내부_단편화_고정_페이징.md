# 내부 단편화 (고정/페이징)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고정 분할/페이징에서 할당된 블록 내부에 발생하는 사용하지 않는 공간
> 2. **가치**: 메모리 낭비, 페이지 크기와 관련, 외부 단편화와 상충 관계
> 3. **융합**: 페이징, 고정 분할, 슬랩 할당자와 관련된 단편화 유형

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

내부 단편화(Internal Fragmentation)는 고정 분할/페이징에서 할당된 블록 내부에 발생하는 사용하지 않는 공간입니다.

> **해설**: 내부 단편화는 고정 크기 할당의 불가피한代价입니다.

**💡 비유**: 내부 단편화는 큰 상자에 작은 물건을 넣을 때 남는 공간과 같습니다.

**📢 섹션 요약 비유**: 상자 안에 빈 공간이 남아요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **고정 분할** | 블록 크기 고정 | 미리 정의 | 상자 |
| **내부 단편화** | 블록 내 낭비 | 할당-사용 차이 | 빈 공간 |
| **페이지 크기** | 단편화 영향 | 클수록 낭비 증가 | 큰 상자 |
| **슬랩** | 최소화 기법 | 객체 크기 맞춤 | 맞춤 상자 |

**단편화와 페이지 크기**

| 페이지 크기 | 내부 단편화 | 외부 단편화 |
|:---|:---:|:---:|
| **작음** | 적음 | 많음 |
| **큼** | 많음 | 없음 |
| **적절** | 균형 | 없음 |

**핵심 알고리즘**: 할당 크기 → 페이지 수 계산 → 마지막 페이지 단편화

**📢 섹션 요약 비유**: 페이지가 클수록 내부 단편화가 늘어요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 단편화와 메모리 기법**

| 기법 | 내부 | 외부 | 특징 |
|:---|:---:|:---:|:---|
| **고정 분할** | 있음 | 없음 | 단순 |
| **가변 분할** | 없음 | 있음 | 유연 |
| **페이징** | 있음 | 없음 | 표준 |
| **버디 시스템** | 있음 | 적음 | 절충 |

**📢 섹션 요약 비유**: 페이징은 외부를 없애고 내부를 감수해요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 4KB 페이지** → 평균 2KB 내부 단편화
**시나리오 2: Huge Pages** → 큰 내부 단편화, TLB 효율
**시나리오 3: Slab 할당** → 객체 크기 맞춤으로 최소화

**도입 체크리스트**: 페이지 크기 선택, Slab 할당자 활용

**📢 섹션 요약 비유**: Slab 할당자가 도와줘요!
