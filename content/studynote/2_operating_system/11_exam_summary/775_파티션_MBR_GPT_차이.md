# 파티션 (MBR vs GPT 차이)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크를 논리적 영역으로 나누는 구조
> 2. **가치**: 멀티부팅, 대용량 지원, 데이터 격리
> 3. **융합**: BIOS/UEFI, 부트로더, 파일 시스템과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

파티션은 물리적 디스크를 논리적으로 나눈 영역이며, MBR과 GPT는 서로 다른 파티션 테이블 형식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    파티션 (MBR vs GPT)                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MBR (Master Boot Record)                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  512바이트 부트 섹터                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────┐             │   │ │
│   │   │  │ Boot Code (446 bytes)               │             │   │ │
│   │   │  ├─────────────────────────────────────┤             │   │ │
│   │   │  │ Partition Table (64 bytes)          │             │   │ │
│   │   │  │   - 4개 파티션 엔트리 (각 16 bytes)  │             │   │ │
│   │   │  │   - 최대 4개 주 파티션              │             │   │ │
│   │   │  │   - 또는 3 주 + 1 확장              │             │   │ │
│   │   │  ├─────────────────────────────────────┤             │   │ │
│   │   │  │ Boot Signature (0x55AA, 2 bytes)    │             │   │ │
│   │   │  └─────────────────────────────────────┘             │   │ │
│   │   │                                                      │   │ │
│   │   │  제한:                                                │   │ │
│   │   │  • 최대 디스크 크기: 2TB (32-bit LBA)                │   │ │
│   │   │  • 최대 파티션: 4개 (또는 확장 사용)                 │   │ │
│   │   │  • 레거시 BIOS                                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              GPT (GUID Partition Table)                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  디스크 시작과 끝에 중복 저장                         │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────┐             │   │ │
│   │   │  │ Protective MBR (LBA 0)              │             │   │ │
│   │   │  ├─────────────────────────────────────┤             │   │ │
│   │   │  │ GPT Header (LBA 1)                  │             │   │ │
│   │   │  │   - 디스크 GUID                     │             │   │ │
│   │   │  │   - 파티션 엔트리 수                │             │   │ │
│   │   │  ├─────────────────────────────────────┤             │   │ │
│   │   │  │ Partition Entries (LBA 2-33)        │             │   │ │
│   │   │  │   - 최대 128개 파티션               │             │   │ │
│   │   │  │   - 각 128 bytes                    │             │   │ │
│   │   │  ├─────────────────────────────────────┤             │   │ │
│   │   │  │ Partitions...                       │             │   │ │
│   │   │  ├─────────────────────────────────────┤             │   │ │
│   │   │  │ Backup Partition Entries            │             │   │ │
│   │   │  ├─────────────────────────────────────┤             │   │ │
│   │   │  │ Backup GPT Header                   │             │   │ │
│   │   │  └─────────────────────────────────────┘             │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                │   │ │
│   │   │  • 최대 디스크: 9.4 ZB (64-bit LBA)                  │   │ │
│   │   │  • 최대 파티션: 128개 (Windows)                      │   │ │
│   │   │  • CRC32 체크섬                                       │   │ │
│   │   │  • UEFI                                               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: GPT는 UEFI와 함께 대용량 디스크를 지원합니다.

**💡 비유**: 파티션은 큰 책장을 칸막이로 나누는 것과 같습니다.

**📢 섹션 요약 비유**: GPT는 2TB 이상 필수!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **MBR** | 부트 코드 | 부트로더 | 시작 |
| **GPT Header** | 메타데이터 | GUID, CRC | 목차 |
| **Entry** | 파티션 | 시작/끝 LBA | 색인 |
| **Type GUID** | 유형 | 파일시스템 | 종류 |

**비교표**

| 항목 | MBR | GPT |
|:---|:---|:---|
| **최대 디스크** | 2TB | 9.4 ZB |
| **최대 파티션** | 4 | 128 |
| **부트 방식** | BIOS | UEFI |
| **중복** | 없음 | 있음 |
| **체크섬** | 없음 | CRC32 |

**핵심 알고리즘**: BIOS → MBR 읽기 → 부트로더 실행 vs UEFI → GPT → EFI 파티션

**📢 섹션 요약 비유**: UEFI + GPT 조합!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 부트 과정**

| 단계 | BIOS/MBR | UEFI/GPT |
|:---|:---|:---|
| **1** | POST | POST |
| **2** | MBR 읽기 | ESP 탐색 |
| **3** | 부트로더 | EFI 실행 |
| **4** | 커널 로드 | 커널 로드 |

**📢 섹션 요약 비유**: UEFI가 보안 부팅 지원!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 2TB 이하** → MBR 가능
**시나리오 2: 2TB 이상** → GPT 필수
**시나리오 3: 듀얼부팅** → 호환성 확인

**도입 체크리스트**: 디스크 크기, OS, BIOS/UEFI

**📢 섹션 요약 비유**: 신규는 GPT!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대용량 지원, 안정성 향상

**미래 전망**: GPT 표준화 완료

**참고 표준**: UEFI 2.10

**📢 섹션 요약 비유**: GPT는 현대 디스크의 표준!
