# 역 페이지 테이블 (전역 해시 매핑)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 페이지마다 하나의 엔트리를 가지는 역방향 매핑 테이블
> 2. **가치**: 메모리 절약, 큰 주소 공간 지원, PowerPC/IA-64
> 3. **융합**: 해시 테이블, TLB, 페이지 교체와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

역 페이지 테이블(Inverted Page Table)은 물리 페이지마다 하나의 엔트리를 가지며, 가상 주소를 해시로 매핑하는 테이블입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    역 페이지 테이블 (Inverted Page Table)             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              일반 페이지 테이블 vs 역 페이지 테이블             │ │
│   │                                                              │ │
│   │   일반 페이지 테이블:                                         │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  프로세스마다 페이지 테이블 존재                        │   │ │
│   │   │  가상 페이지 수 = 엔트리 수                             │   │ │
│   │   │                                                      │   │ │
│   │   │  64-bit 주소 공간, 4KB 페이지:                        │   │ │
│   │   │  2^64 / 4KB = 2^52 엔트리 (너무 큼!)                  │   │ │
│   │   │                                                      │   │ │
│   │   │  문제: 64-bit에서 메모리 과다 사용                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   역 페이지 테이블:                                           │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  시스템 전체에 하나의 테이블                            │   │ │
│   │   │  물리 페이지 수 = 엔트리 수                             │   │ │
│   │   │                                                      │   │ │
│   │   │  물리 메모리 16GB, 4KB 페이지:                        │   │ │
│   │   │  16GB / 4KB = 4M 엔트리 (작음!)                       │   │ │
│   │   │                                                      │   │ │
│   │   │  이점: 64-bit 시스템에서 메모리 효율적                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              역 페이지 테이블 구조                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  해시 함수: hash(PID, VPN) → IPT 인덱스              │   │ │
│   │   │                                                      │   │ │
│   │   │  IPT Entry:                                          │   │ │
│   │   │  ┌────────────────────────────────────────────────┐ │   │ │
│   │   │  │ PID | VPN | Flags | Chain Pointer             │ │   │ │
│   │   │  └────────────────────────────────────────────────┘ │   │ │
│   │   │                                                      │   │ │
│   │   │  물리 페이지 0: [PID=1, VPN=100, ...]               │   │ │
│   │   │  물리 페이지 1: [PID=2, VPN=200, ...]               │   │ │
│   │   │  물리 페이지 2: [PID=1, VPN=101, ...]               │   │ │
│   │   │  ...                                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  충돌 시: 체이닝으로 연결                             │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 역 페이지 테이블은 물리 메모리 크기에만 비례합니다.

**💡 비유**: 역 페이지 테이블은 전화번호부를 거꾸로 찾는 것과 같습니다.

**📢 섹션 요약 비유**: 물리 페이지 수만큼만!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Hash** | 매핑 | VPN → 인덱스 | 색인 |
| **PID** | 프로세스 | 식별자 | 이름 |
| **Chain** | 충돌 해결 | 연결 리스트 | 줄 |
| **TLB** | 캐시 | 최근 조회 | 기억 |

**변환 과정**

1. 해시(PID, VPN) → IPT 인덱스
2. 엔트리 검색 (체이닝)
3. PID/VPN 매칭 확인
4. PFN 반환

**핵심 알고리즘**: hash → chain 탐색 → PFN

**📢 섹션 요약 비유**: TLB가 필수!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 페이지 테이블**

| 유형 | 크기 | 검색 |
|:---|:---|:---:|
| **일반 PT** | 가상 페이지 수 | O(1) |
| **다단계 PT** | 작음 | O(n) |
| **역 PT** | 물리 페이지 수 | O(1+α) |

**📢 섹션 요약 비유**: 역 PT가 64-bit에 유리!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 64-bit 시스템** → 역 PT
**시나리오 2: PowerPC** → 역 PT 기본
**시나리오 3: x86** → 일반/다단계 PT

**도입 체크리스트**: 해시 충돌, TLB 미스

**📢 섹션 요약 비유**: PowerPC와 IA-64에서 사용!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 페이지 테이블 메모리 99% 절약

**미래 전망**: 5단계 페이지 워크, 확장

**참고 표준**: PowerPC MMU, IA-64

**📢 섹션 요약 비유**: 역 페이지 테이블은 큰 주소 공간의 해결책!
