# 페이지 폴트 (Page Fault ISR)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 접근한 페이지가 메모리에 없을 때 발생하는 인터럽트로 OS가 디스크에서 페이지를 로드
> 2. **가치**: 가상 메모리의 핵심, 요구 페이징의 구현 수단
> 3. **융합**: 인터럽트, 컨텍스트 스위치, 디스크 I/O와 통합된 페이지 처리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

페이지 폴트(Page Fault)는 접근한 페이지가 메모리에 없을 때 발생하는 인터럽트입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    페이지 폴트 (Page Fault ISR)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              페이지 폴트 처리 6단계                            │ │
│   │                                                              │ │
│   │   ① 폴트 발생                                                 │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  CPU: 논리 주소 접근 → MMU: valid bit = 0 → 트랩     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                          │                                   │ │
│   │                          ↓                                   │ │
│   │   ② 커널 모드 진입                                            │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  레지스터 저장, 페이지 폴트 핸들러 호출                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                          │                                   │ │
│   │                          ↓                                   │ │
│   │   ③ 원인 분석                                                 │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  잘못된 참조? → 종료 (Segmentation Fault)             │   │ │
│   │   │  정상 참조? → 디스크 읽기                             │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                          │                                   │ │
│   │                          ↓                                   │ │
│   │   ④ 빈 프레임 확보                                            │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  빈 프레임 있음 → 사용                                 │   │ │
│   │   │  빈 프레임 없음 → 페이지 교체 알고리즘 실행            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                          │                                   │ │
│   │                          ↓                                   │ │
│   │   ⑤ 디스크 읽기                                               │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  스왑 공간/파일에서 페이지 읽기                        │   │ │
│   │   │  I/O 완료 대기 (다른 프로세스 실행 가능)               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                          │                                   │ │
│   │                          ↓                                   │ │
│   │   ⑥ 테이블 갱신 및 재시작                                      │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  페이지 테이블 갱신 (valid=1, frame=할당된것)         │   │ │
│   │   │  TLB 갱신                                             │   │ │
│   │   │  문맥 복원, 명령어 재시작                              │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Minor vs Major Fault                             │ │
│   │                                                              │ │
│   │   Minor Fault: 페이지가 메모리에 있음 (다른 프로세스 것이나    │ │
│   │                메모리 매핑 파일 등)                           │ │
│   │   Major Fault: 디스크 I/O 필요                                │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 페이지 폴트 처리는 OS가 가상 메모리를 관리하는 핵심입니다.

**💡 비유**: 페이지 폴트는 도서관에서 없는 책을 주문하는 것과 같습니다.

**📢 섹션 요약 비유**: 없으면 주문해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **트랩** | 인터럽트 | 커널 진입 | 비상벨 |
| **핸들러** | 처리 루틴 | OS 코드 | 사서 |
| **프레임 확보** | 메모리 할당 | 교체 알고리즘 | 책장 정리 |
| **재시작** | 실행 계속 | 명령어 재실행 | 다시 읽기 |

**폴트 처리 시간**

| 유형 | 시간 | 원인 |
|:---|:---|:---|
| **Minor** | ~1μs | 메모리 내 |
| **Major** | ~10ms | 디스크 I/O |

**핵심 알고리즘**: 폴트 → 분석 → 확보 → 로드 → 갱신 → 재시작

**📢 섹션 요약 비유**: Major 폴트가 느려요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 폴트 유형**

| 유형 | 원인 | 처리 |
|:---|:---|:---|
| **Valid** | 정상 참조 | 로드 |
| **Invalid** | 잘못된 참조 | 종료 |
| **Protection** | 권한 위반 | 시그널 |

**📢 섹션 요약 비유**: Invalid는 에러예요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 메모리 부족** → 잦은 폴트 → 스래싱
**시나리오 2: SSD** → 폴트 비용 감소
**시나리오 3: 실시간** → 폴트 최소화 필요

**도입 체크리스트**: 스왑 공간, 프레임 풀, 교체 알고리즘

**📢 섹션 요약 비유**: 메모리가 부족하면 폴트가 늘어요!
