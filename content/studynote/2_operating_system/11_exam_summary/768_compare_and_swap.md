# 240. CAS (Compare-And-Swap)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비교 후 조건부 교환하는 원자적 연산
> 2. **가치**: 락 프리 동기화의 핵심 기반
> 3. **융합**: 원자적 연산, ABA 문제, 스핀락과 연관

---

## Ⅰ. 개요

### 개념 정의

CAS(Compare-And-Swap)는 **메모리 위치의 값을 예상값과 비교하여, 같으면 새 값으로 교체하는 원자적 연산**이다. 성공 여부를 반환하며, 락 프리 알고리즘의 기반이다.

### 💡 비유: 티켓 교환
CAS는 **티켓 교환**과 같다. "이 티켓이 100번이면 101번으로 바꿔주세요"라고 한다. 100번이면 바꾸고, 아니면 안 바꾼다. 한 번에 끝난다.

### CAS 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                CAS 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CAS 의사코드】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  boolean CAS(int* ptr, int expected, int new_value) {          │ │
│  │      // 전체가 원자적으로 실행                                   │ │
│  │      atomic {                                                   │ │
│  │          if (*ptr == expected) {                               │ │
│  │              *ptr = new_value;                                 │ │
│  │              return true;   // 성공                             │ │
│  │          } else {                                              │ │
│  │              return false;  // 실패                             │ │
│  │          }                                                      │ │
│  │      }                                                          │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CAS 기반 카운터】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  void atomic_increment(int* counter) {                          │ │
│  │      int old_val;                                               │ │
│  │      do {                                                        │ │
│  │          old_val = *counter;   // 현재 값 읽기                   │ │
│  │      } while (!CAS(counter, old_val, old_val + 1));             │ │
│  │      // CAS 실패 시 다른 스레드가 변경한 것, 재시도               │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  │  타임라인:                                                        │
│  │  ──────────────────────────────────────────────────────►        │ │
│  │  T1: read(10) ──────────────────CAS(10,11) OK ────►             │ │
│  │  T2:        read(10) ─CAS(10,11) FAIL─read(11)─CAS(11,12) OK   │ │
│  │                                    ↑                             │ │
│  │                            값이 바뀜, 재시도                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CAS 기반 락】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  void lock(int* lock_var) {                                     │ │
│  │      while (!CAS(lock_var, 0, 1)) {                             │ │
│  │          // 스핀 대기                                             │ │
│  │      }                                                           │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  │  void unlock(int* lock_var) {                                   │ │
│  │      *lock_var = 0;  // 단순 쓰기 (원자적 필요)                   │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                CAS 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【weak vs strong CAS】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  weak:                                                            │ │
│  │  • 값이 같아도 가끔 실패할 수 있음 (spurious failure)             │ │
│  │  • ARM 등 LL/SC 아키텍처에서 효율적                               │ │
│  │  • 항상 루프에서 사용해야                                         │ │
│  │                                                             │ │
│  │  strong:                                                          │ │
│  │  • 값이 같으면 항상 성공                                           │ │
│  │  • x86 등에서는 weak == strong                                     │ │
│  │  • 단일 시도에도 사용 가능                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ABA 문제】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시나리오 (연결 리스트 pop):                                      │ │
│  │  1. T1: head 읽음 → A                                            │ │
│  │  2. T1: A->next 읽음 → B                                         │ │
│  │  3. T2: pop(head) → A 제거                                        │ │
│  │  4. T2: push(head, A')  // A 재사용 또는 A와 같은 주소            │ │
│  │  5. T1: CAS(head, A, B) 성공!                                    │ │
│  │     문제: head가 A'이 아닌 B를 가리킴 (A' 손실)                    │ │
│  │                                                             │ │
│  │  해결 방법:                                                        │ │
│  │  • 더블 워드 CAS (값 + 버전)                                      │ │
│  │  • Hazard Pointer                                                │ │
│  │  • RCU                                                           │ │
│  │  • Epoch-based reclamation                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【아키텍처별 구현】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86: CMPXCHG (LOCK prefix와 함께)                               │ │
│  │  ARM: LDREX/STREX 또는 CAS (v8.1+)                              │ │
│  │  MIPS: LL/SC                                                     │ │
│  │  POWER: lwarx/stwcx                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ std::atomic CAS】                                              │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  std::atomic<int> counter{0};                                        │
│                                                                     │
│  // strong CAS                                                       │
│  int expected = 10;                                                 │
│  bool success = counter.compare_exchange_strong(                    │
│      expected, 20,                                                  │
│      std::memory_order_acq_rel,                                     │
│      std::memory_order_acquire                                      │
│  );                                                                 │
│                                                                     │
│  // weak CAS (루프에서 사용)                                          │
│  int old_val;                                                       │
│  do {                                                               │
│      old_val = counter.load();                                      │
│  } while (!counter.compare_exchange_weak(old_val, old_val + 1));    │
│                                                                     │
│  【락 프리 스택 (Treiber Stack)】                                       │
│  ──────────────────                                                  │
│  struct Node {                                                       │
│      int value;                                                      │
│      Node* next;                                                     │
│  };                                                                 │
│  std::atomic<Node*> head{nullptr};                                  │
│                                                                     │
│  void push(int value) {                                             │
│      Node* new_node = new Node{value, nullptr};                     │
│      new_node->next = head.load();                                  │
│      while (!head.compare_exchange_weak(new_node->next, new_node)); │
│  }                                                                  │
│                                                                     │
│  bool pop(int& value) {                                             │
│      Node* old_head = head.load();                                  │
│      while (old_head &&                                              │
│             !head.compare_exchange_weak(old_head, old_head->next)); │
│      if (old_head) {                                                │
│          value = old_head->value;                                   │
│          // 주의: ABA 문제! 메모리 해제 시 주의                        │
│          return true;                                               │
│      }                                                              │
│      return false;                                                  │
│  }                                                                  │
│                                                                     │
│  【GCC __sync CAS】                                                   │
│  ──────────────────                                                  │
│  int counter = 0;                                                   │
│  int expected = 0;                                                  │
│                                                                     │
│  // bool 반환                                                        │
│  if (__sync_bool_compare_and_swap(&counter, 0, 1)) {                │
│      // 성공                                                         │
│  }                                                                  │
│                                                                     │
│  // 이전 값 반환                                                      │
│  int old = __sync_val_compare_and_swap(&counter, 0, 1);             │
│  if (old == 0) {                                                    │
│      // 성공                                                         │
│  }                                                                  │
│                                                                     │
│  【x86 어셈블리】                                                      │
│  ──────────────────                                                  │
│  ; CMPXCHG dest, src                                                │
│  ; if (EAX == dest) { ZF=1; dest = src; }                          │
│  ; else { ZF=0; EAX = dest; }                                       │
│                                                                     │
│  lock cmpxchg [rdi], rsi    ; 원자적 CAS                             │
│  jz success                 ; ZF=1이면 성공                          │
│  ; 실패 처리                                                         │
│                                                                     │
│  【성능 고려사항】                                                     │
│  ──────────────────                                                  │
│  • 경합이 높으면 CAS 실패 잦음 → 스핀 많이 함                          │
│  • exponential backoff 고려                                         │
│  • 경우에 따라 뮤텍스가 더 나을 수 있음                                │
│  • CAS 루프 안에서 복잡한 연산 피하기                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

