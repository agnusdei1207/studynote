> **[Insight]**
> 메모리 관리는 CPU가 실행할 프로그램과 데이터를 저장하는 유한한 물리적 공간을 효율적으로 할당하고 보호하며, 프로세스 간 격리를 보장하는 운영체제의 핵심 기능이다.
> 논리 주소(Logical Address)와 물리 주소(Physical Address)의 분리를 통해 프로그램의 유연성을 확보하고, MMU(Memory Management Unit)를 통한 고속 주소 변환이 시스템 성능의 관건이 된다.
> 파편화(Fragmentation) 문제를 최소화하기 위해 고정 분할 방식에서 가변 분할, 나아가 페이징(Paging) 및 세그먼테이션(Segmentation)으로 발전해왔다.

+++

### Ⅰ. 메모리 관리의 배경과 주소 바인딩

1. 주소 바인딩(Address Binding)
   - 프로그램이 참조하는 논리 주소를 실제 물리 메모리 주소로 연결하는 과정이다.
   - **컴파일 시간(Compile time)**: 컴파일 시 물리 주소 확정.
   - **로드 시간(Load time)**: 메모리 적재 시 물리 주소 확정.
   - **실행 시간(Execution time)**: 실행 중 물리 주소 변환 (MMU 활용, 현대 OS 방식).
2. 논리 주소(Logical) vs 물리 주소(Physical)
   - 논리 주소: CPU가 생성하는 주소 (가상 주소).
   - 물리 주소: 메모리 하드웨어가 인식하는 실제 주소.
3. MMU(Memory Management Unit)
   - 논리 주소를 물리 주소로 실시간 변환해주는 하드웨어 장치.

📢 섹션 요약 비유: 도서관의 '청구기호(Logical)'와 실제 책이 꽂혀 있는 '서가 번호(Physical)'가 있고, 이를 연결해주는 '사서 선생님(MMU)'이 있는 것과 같습니다.

+++

### Ⅱ. 연속 메모리 할당과 파편화 문제

1. 연속 할당(Contiguous Allocation)
   - 프로세스를 메모리의 연속적인 공간에 통째로 올리는 방식이다.
2. 파편화(Fragmentation)
   - **내부 파편화(Internal Fragmentation)**: 프로세스에 할당된 공간이 실제 필요량보다 커서 남는 공간이 생기는 현상.
   - **외부 파편화(External Fragmentation)**: 할당되지 않은 작은 공간들이 흩어져 있어 전체 총합은 충분하지만 큰 프로세스를 수용하지 못하는 현상.
3. 압축(Compaction)
   - 외부 파편화 해결을 위해 프로세스들을 한쪽으로 몰아 큰 빈 공간을 만드는 작업 (오버헤드 큼).

```text
[ External Fragmentation ]

   [ P1 ] [ Free 2MB ] [ P2 ] [ Free 3MB ] [ P3 ]
   
   Total Free = 5MB, but cannot fit a 4MB Process.
```

📢 섹션 요약 비유: 주차장에 큰 차 한 대가 들어와야 하는데, 작은 빈자리들만 띄엄띄엄 있어서 주차를 못 하는 상황과 같습니다.

+++

### Ⅲ. 페이징(Paging) 기법

1. 페이징(Paging)의 정의
   - 물리 메모리를 고정 크기의 블록인 프레임(Frame)으로 나누고, 프로세스를 동일 크기의 페이지(Page)로 나누어 불연속적으로 할당하는 방식이다.
2. 특징
   - 외부 파편화 문제를 완전히 해결한다.
   - 내부 파편화는 발생할 수 있다 (마지막 페이지).
3. 페이지 테이블(Page Table)
   - 각 페이지가 어느 프레임에 있는지 정보를 저장하는 매핑 테이블.
   - 속도 향상을 위해 TLB(Translation Lookaside Buffer)라는 캐시 하드웨어 사용.

📢 섹션 요약 비유: 원고지를 칸(Frame)에 맞춰 한 글자(Page)씩 순서대로 적는 것이 아니라, 빈칸 어디든 남는 곳에 채워 넣고 '목차(Page Table)'를 만드는 방식입니다.

+++

### Ⅳ. 세그먼테이션(Segmentation) 기법

1. 세그먼테이션(Segmentation)의 정의
   - 프로세스를 의미 있는 단위(Main, Function, Stack 등)인 세그먼트(Segment)로 나누어 관리하는 방식이다.
2. 특징
   - 사용자 관점의 논리적 구분을 반영한다.
   - 세그먼트마다 크기가 다르므로 외부 파편화 문제가 발생할 수 있다.
3. 세그먼트 테이블(Segment Table)
   - 세그먼트 번호, 기준 주소(Base), 한계 주소(Limit)를 저장한다.

📢 섹션 요약 비유: 책을 똑같은 크기의 '페이지'로 나누는 것이 아니라, '서론, 본론, 결론'처럼 내용의 길이에 따라 '장(Chapter)'으로 나누어 보관하는 것과 같습니다.

+++

### Ⅴ. 페이징과 세그먼테이션의 혼용 (Paged Segmentation)

1. 개념
   - 세그먼테이션의 논리적 장점과 페이징의 물리적 관리 장점을 결합한다.
   - 먼저 프로세스를 세그먼트로 나누고, 각 세그먼트를 다시 페이지 단위로 나누어 관리한다.
2. 장점
   - 외부 파편화 제거와 보호/공유의 효율성을 동시에 달성한다.
   - 현대 대부분의 운영체제가 채택하는 방식이다.

📢 섹션 요약 비유: 책을 '장(Chapter)'별로 나누되, 각 장의 내용은 다시 '페이지' 단위로 인쇄하여 바인더에 꽂아 관리하는 정교한 방식입니다.
