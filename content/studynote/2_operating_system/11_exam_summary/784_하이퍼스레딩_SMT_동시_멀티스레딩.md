# 하이퍼스레딩 (SMT 동시 멀티스레딩)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하나의 물리 코어가 여러 스레드를 동시 실행
> 2. **가치**: 처리량 증가, 리소스 활용률 향상, 비용 효율
> 3. **융합**: 파이프라인, 캐시, 스케줄러와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

하이퍼스레딩(Hyper-Threading) 또는 SMT(Simultaneous Multi-Threading)는 하나의 물리 코어에서 여러 스레드를 동시에 실행하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    하이퍼스레딩 (SMT)                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SMT 원리                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  물리 코어 (1개):                                      │   │ │
│   │   │  ┌───────────────────────────────────────────────┐   │   │ │
│   │   │  │             파이프라인 + 실행 유닛              │   │   │ │
│   │   │  │  ┌─────────┐  ┌─────────┐  ┌─────────┐        │   │   │ │
│   │   │  │  │ Fetch   │  │ Decode  │  │ Execute │        │   │   │ │
│   │   │  │  └────┬────┘  └────┬────┘  └────┬────┘        │   │   │ │
│   │   │  │       │            │            │              │   │   │ │
│   │   │  │       │    공유 리소스 (ALU, FPU, Cache)       │   │   │ │
│   │   │  └───────┴────────────┴────────────┴──────────────┘   │   │ │
│   │   │                                                      │   │ │
│   │   │  논리 코어 (2개):                                     │   │ │
│   │   │  ┌─────────────┐    ┌─────────────┐                  │   │ │
│   │   │  │ Thread 0    │    │ Thread 1    │                  │   │ │
│   │   │  │ Registers   │    │ Registers   │ ← 개별           │   │ │
│   │   │  │ PC          │    │ PC          │                  │   │ │
│   │   │  └─────────────┘    └─────────────┘                  │   │ │
│   │   │                                                      │   │ │
│   │   │  공유: L1/L2 Cache, ALU, FPU, 메모리 컨트롤러         │   │ │
│   │   │  개별: 레지스터, PC, 일부 상태                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              이점과 제한                                       │ │
│   │                                                              │ │
│   │   이점:                                                       │ │
│   │   • 파이프라인 슬롯 활용률 증가                               │ │
│   │   • 메모리 대기 시간 숨김                                     │ │
│   │   • 처리량 20-40% 향상 (이상적)                               │ │
│   │                                                              │ │
│   │   제한:                                                       │ │
│   │   • 공유 리소스 경합                                          │ │
│   │   • 캐시 스래싱                                               │ │
│   │   • 단일 스레드 성능은 동일 또는 감소                         │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: SMT는 유휴 실행 유닛을 다른 스레드가 활용합니다.

**💡 비유**: SMT는 한 사람이 두 일을 번갈아 처리하는 것과 같습니다.

**📢 섹션 요약 비유**: 1코어 = 2스레드!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Thread Context** | 상태 | 레지스터 셋 | 작업 공간 |
| **Shared Resources** | 공유 | ALU, 캐시 | 공용 도구 |
| **Scheduler** | 분배 | 사이클 분할 | 관리자 |
| **SMT Level** | 수준 | 2-way, 4-way | 배수 |

**공유 vs 개별 리소스**

| 개별 | 공유 |
|:---|:---|
| 레지스터 | ALU |
| PC | FPU |
| 일부 TLB | L1/L2 캐시 |

**핵심 알고리즘**: 슬롯 가용성 → 스레드 선택 → 명령어 발행

**📢 섹션 요약 비유**: 레지스터만 개별!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 멀티스레딩**

| 유형 | 시점 | 성능 |
|:---|:---:|:---:|
| **Coarse-grained** | 블록 | 낮음 |
| **Fine-grained** | 사이클 | 중간 |
| **SMT** | 동시 | 높음 |

**📢 섹션 요약 비유**: SMT가 최고!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 서버** → SMT 활성화
**시나리오 2: HPC** → SMT 끄기 (단일 스레드)
**시나리오 3: 보안** → SMT 비활성화 (Spectre)

**도입 체크리스트**: 워크로드, 보안, 열

**📢 섹션 요약 비유**: 보안 문제시 끄기!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 처리량 20-40% 향상

**미래 전망**: 4-way SMT, Power SMT

**참고 표준**: Intel HT, IBM POWER, ARM SMT

**📢 섹션 요약 비유**: SMT는 코어 활용 극대화!
