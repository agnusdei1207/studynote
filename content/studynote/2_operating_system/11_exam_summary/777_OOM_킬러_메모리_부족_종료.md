# OOM 킬러 (메모리 부족 종료)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 부족 시 프로세스를 종료하여 시스템 보호
> 2. **가치**: 시스템 안정성, 데드락 방지, 자동 복구
> 3. **융합**: 메모리 관리, cgroups, 점수 계산과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

OOM Killer(Out-of-Memory Killer)는 시스템 메모리가 고갈될 때 프로세스를 종료하여 메모리를 확보하는 리눅스 커널 메커니즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OOM 킬러 (메모리 부족 종료)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OOM 발생 시나리오                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 메모리 할당 요청                                   │   │ │
│   │   │     ↓                                                  │   │ │
│   │   │  2. 페이지 폴트 발생                                    │   │ │
│   │   │     ↓                                                  │   │ │
│   │   │  3. 메모리 부족 (No free pages)                        │   │ │
│   │   │     ↓                                                  │   │ │
│   │   │  4. 회수 시도 (Cache, Buffer, Swap)                    │   │ │
│   │   │     ↓                                                  │   │ │
│   │   │  5. 여전히 부족 → OOM Killer 활성화                     │   │ │
│   │   │     ↓                                                  │   │ │
│   │   │  6. 점수 계산 (oom_score)                              │   │ │
│   │   │     ↓                                                  │   │ │
│   │   │  7. 최고 점수 프로세스 종료                              │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OOM 점수 계산                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  oom_score = 프로세스 메모리 사용량 × adj            │   │ │
│   │   │                                                      │   │ │
│   │   │  고려 요소:                                            │   │ │
│   │   │  • RSS (Resident Set Size)                           │   │ │
│   │   │  • Page Table 크기                                    │   │ │
│   │   │  • Swap 사용량                                        │   │ │
│   │   │  • oom_score_adj 조정값                               │   │ │
│   │   │                                                      │   │ │
│   │   │  adj 값:                                               │   │ │
│   │   │  -1000 ~ 1000 (음수 = 보호, 양수 = 희생)              │   │ │
│   │   │  -1000: 절대 종료 안 함 (oom_score_adj)               │   │ │
│   │   │  1000: 항상 종료 대상                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: OOM Killer는 "누굴 죽일지" 점수로 결정합니다.

**💡 비유**: OOM Killer는 비상시 가장 무거운 짐을 가진 사람을 내보내는 것과 같습니다.

**📢 섹션 요약 비유**: oom_score_adj로 보호!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **oom_score** | 점수 | 0-1000 | 위험도 |
| **oom_adj** | 조정 | 레거시 | 가중치 |
| **oom_score_adj** | 조정 | -1000~1000 | 가중치 |
| **panic_on_oom** | 동작 | 패닉 여부 | 대응 |

**파일 위치**

| 파일 | 경로 |
|:---|:---|
| **oom_score** | /proc/[pid]/oom_score |
| **oom_score_adj** | /proc/[pid]/oom_score_adj |
| **panic_on_oom** | /proc/sys/vm/panic_on_oom |

**핵심 알고리즘**: max(oom_score) 프로세스 선택 → SIGKILL → 메모리 회수

**📢 섹션 요약 비유**: RSS가 크면 위험!
