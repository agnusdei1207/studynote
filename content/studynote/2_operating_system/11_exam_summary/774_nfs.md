# 593. NFS (Network File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크를 통한 파일 공유 프로토콜
> 2. **가치**: 원격 파일시스템 투명 접근, 중앙 관리
> 3. **융합**: RPC, 마운트, 파일시스템과 연관

---

## Ⅰ. 개요

### 개념 정의
**NFS(Network File System)**은 **네트워크를 통해 원격 파일시스템에 접근하는 프로토콜**입니다.

### 💡 비유: 원격 파일 서랍
NFS는 **원격 파일 서랍**과 같습니다. 멀리 있는 서랍을 내 책상에서 열 수 있습니다.

### NFS 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                NFS 구조                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NFS 아키텍처】                                                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  NFS 클라이언트                    NFS 서버                    │ │   │
│  │  ┌─────────────────┐              ┌─────────────────┐        │ │   │
│  │  │                 │              │                 │        │ │   │
│  │  │   애플리케이션    │              │   파일시스템     │        │ │   │
│  │  │       │         │              │       ↑         │        │ │   │
│  │  │       ▼         │              │       │         │        │ │   │
│  │  │   VFS          │              │   NFS 데몬      │        │ │   │
│  │  │       │         │              │   (nfsd)       │        │ │   │
│  │  │       ▼         │              │       │         │        │ │   │
│  │  │   NFS 클라이언트 │◄────────────►│   RPC          │        │ │   │
│  │  │   (nfs.ko)      │    네트워크    │   (portmap)   │        │ │   │
│  │  │                 │              │                 │        │ │   │
│  │  └─────────────────┘              └─────────────────┘        │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【NFS 버전】                                                         │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  버전      특징                    포트                      │ │   │
│  │  ────      ────                    ────                      │ │   │
│  │  NFSv2    원본, UDP만              2049                      │ │   │
│  │  NFSv3    TCP 지원, 대용량          2049                      │ │   │
│  │  NFSv4    방화벽 친화적, 통합        2049만 사용               │ │   │
│  │  NFSv4.1  pNFS, 병렬               2049                      │ │   │
│  │  NFSv4.2  스파스 파일, 압축         2049                      │ │   │
│  │                                                             │ │   │
│  │  NFSv4 권장: 포트 2049만 사용, 방화벽 설정 간단                   │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【관련 데몬】                                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  데몬              역할                    포트                  │ │   │
│  │  ────              ────                    ────                  │ │   │
│  │  nfsd             NFS 서비스              2049                  │ │   │
│  │  mountd           마운트 요청 처리          NFSv4에서 불필요       │ │   │
│  │  rpcbind          RPC 포트 매퍼            111                   │ │   │
│  │  statd            파일 잠금 상태           NFSv4에서 불필요       │ │   │
│  │  lockd            파일 잠금                NFSv4에서 불필요       │ │   │
│  │  rquotad          할당량                   NFSv4에서 불필요       │ │   │
│  │  idmapd           UID/GID 매핑 (NFSv4)                             │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                NFS 상세                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NFS 마운트 옵션】                                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  옵션              설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  rw/ro            읽기쓰기 / 읽기전용                              │ │   │
│  │  sync/async       동기 / 비동기 쓰기                              │ │   │
│  │  hard/soft        서버 장애 시 무한 대기 / 에러 반환               │ │   │
│  │  intr             hard 마운트에서 인터럽트 가능                    │ │   │
│  │  noac             속성 캐시 비활성화                              │ │   │
│  │  rsize/wsize      읽기/쓰기 버퍼 크기 (기본 1MB)                    │ │   │
│  │  timeo            타임아웃 (1/10초)                               │ │   │
│  │  retrans          재전송 횟수                                     │ │   │
│  │  vers=4           NFS 버전 지정                                   │ │   │
│  │  sec=sys/krb5     보안 (시스템/kerberos)                          │ │   │
│  │  _netdev          네트워크 필요 표시                               │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【/etc/exports 형식】                                                │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  /export/share  client1(options) client2(options)              │ │   │
│  │                                                             │ │   │
│  │  클라이언트 표현:                                                  │ │   │
│  │  • hostname          호스트명                                    │ │   │
│  │  • 192.168.1.0/24    네트워크                                    │ │   │
│  │  • *.example.com     와일드카드                                   │ │   │
│  │  • *                 모든 호스트                                  │ │   │
│  │                                                             │ │   │
│  │  주요 옵션:                                                        │ │   │
│  │  • ro/rw            읽기전용/읽기쓰기                              │ │   │
│  │  • sync/async       동기/비동기                                   │ │   │
│  │  • root_squash      root → nfsnobody (기본)                      │ │   │
│  │  • no_root_squash   root 권한 유지                                │ │   │
│  │  • all_squash       모든 사용자 → nfsnobody                        │ │   │
│  │  • anonuid/anongid  익명 UID/GID 지정                             │ │   │
│  │  • no_subtree_check 서브트리 검사 생략                             │ │   │
│  │  • secure           1024 이하 포트만                              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【Root Squash】                                                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  기본 동작: root_squash                                          │ │   │
│  │  • 클라이언트 root → 서버에서 nfsnobody (65534)로 매핑              │ │   │
│  │  • 보안상 이유로 기본값                                            │ │   │
│  │                                                             │ │   │
│  │  no_root_squash:                                                 │ │   │
│  │  • root 권한 유지                                                 │ │   │
│  │  • 신뢰할 수 있는 환경에서만 사용                                    │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NFS 서버 설정】                                                    │ |
│  ──────────────────                                                │
│  // 패키지 설치                                                       │
│  $ sudo apt install nfs-kernel-server    // Ubuntu/Debian          │
│  $ sudo yum install nfs-utils            // CentOS/RHEL            │
│                                                                     │
│  // 공유 디렉토리 생성                                                  │
│  $ sudo mkdir -p /export/share                                     │
│  $ sudo chown nobody:nogroup /export/share                         │
│                                                                     │
│  // /etc/exports 설정                                                 │
│  /export/share  192.168.1.0/24(rw,sync,no_subtree_check)           │
│  /export/home   *.example.com(rw,async)                            │
│  /export/ro     10.0.0.0/8(ro,root_squash)                         │
│  /export/root   trusted.host.com(rw,no_root_squash)                │
│                                                                     │
│  // exports 적용                                                      │
│  $ sudo exportfs -a                   // 전체 적용                   │
│  $ sudo exportfs -r                   // 재로드                      │
│  $ sudo exportfs -v                   // 확인                       │
│                                                                     │
│  // 서비스 시작                                                        │
│  $ sudo systemctl start nfs-server                                  │
│  $ sudo systemctl enable nfs-server                                 │
│                                                                     │
│  // 방화벽 설정 (NFSv4)                                               │
│  $ sudo firewall-cmd --permanent --add-service=nfs                 │
│  $ sudo firewall-cmd --reload                                       │
│                                                                     │
│  【NFS 클라이언트 설정】                                               │ |
│  ──────────────────                                                │
│  // 패키지 설치                                                       │
│  $ sudo apt install nfs-common                                     │
│  $ sudo yum install nfs-utils                                      │
│                                                                     │
│  // 마운트                                                           │
│  $ sudo mount -t nfs server:/export/share /mnt/nfs                 │
│  $ sudo mount -t nfs4 server:/export/share /mnt/nfs                │
│                                                                     │
│  // 옵션 지정                                                         │
│  $ sudo mount -t nfs -o rw,hard,intr server:/share /mnt/nfs        │
│  $ sudo mount -t nfs -o rsize=8192,wsize=8192 server:/share /mnt   │
│                                                                     │
│  // /etc/fstab                                                        │
│  server:/export/share  /mnt/nfs  nfs  defaults,_netdev  0  0       │
│  server:/export/share  /mnt/nfs  nfs4 rw,hard,intr  0  0          │
│                                                                     │
│  // autofs 자동 마운트                                                 │
│  // /etc/auto.master                                                 │
│  /mnt    /etc/auto.nfs                                              │
│  // /etc/auto.nfs                                                    │
│  share   -fstype=nfs4,rw   server:/export/share                    │
│                                                                     │
│  【NFS 관리 명령어】                                                   │ |
│  ──────────────────                                                │
│  // 서버                                                              │
│  $ sudo exportfs -v                  // 내보낸 목록                   │
│  $ sudo exportfs -u client:/path     // 내보내기 해제                 │
│  $ sudo exportfs -ra                 // 재로드                       │
│                                                                     │
│  // 클라이언트                                                         │
│  $ showmount -e server              // 서버 공유 목록                 │
│  $ showmount -a server              // 마운트된 클라이언트             │
│  $ nfsstat                          // 통계                          │
│  $ nfsstat -c                       // 클라이언트 통계                │
│  $ nfsstat -s                       // 서버 통계                     │
│                                                                     │
│  // 마운트 확인                                                        │
│  $ mount | grep nfs                                                 │
│  $ df -h | grep nfs                                                 │
│  $ findmnt -t nfs                                                   │
│                                                                     │
│  【NFSv4 ID 매핑】                                                    │ |
│  ──────────────────                                                │
│  // /etc/idmapd.conf                                                 │
│  [General]                                                           │
│  Domain = example.com                                               │
│  [Mapping]                                                           │
│  Nobody-User = nobody                                               │
│  Nobody-Group = nogroup                                             │
│                                                                     │
│  // idmapd 재시작                                                     │
│  $ sudo systemctl restart nfs-idmapd                               │
│  $ sudo nfsidmap -c                  // 캐시 지우기                   │
│                                                                     │
│  【문제 해결】                                                         │ |
│  ──────────────────                                                │
│  // 연결 문제                                                          │
│  $ rpcinfo -p server                // RPC 서비스 확인               │
│  $ sudo tcpdump -i eth0 port 2049   // 패킷 캡처                    │
│                                                                     │
│  // 마운트 실패                                                        │
│  $ sudo mount -v -t nfs4 server:/share /mnt   // 상세 출력           │
│                                                                     │
│  // 성능 문제                                                          │
│  $ sudo mount -o remount,rsize=1048576,wsize=1048576 /mnt/nfs      │
│                                                                     │
│  // "stale file handle" 오류                                          │
│  $ sudo umount -f /mnt/nfs                                          │
│  $ sudo mount /mnt/nfs                                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 네트워크를 통한 파일 공유 프로토콜
• 버전: NFSv2, v3, v4 (권장), v4.1, v4.2
• 포트: 2049 (NFSv4는 이것만 사용)
• RPC: rpcbind (포트 111)
• 서버: nfs-kernel-server, nfs-utils
• 클라이언트: nfs-common, nfs-utils
• 설정: /etc/exports
• 적용: exportfs -a
• 마운트: mount -t nfs4
• fstab: _netdev 옵션
• 옵션: rw, ro, sync, async, hard, soft
• root_squash: root → nfsnobody (기본)
• no_root_squash: root 권한 유지
• 확인: showmount -e, exportfs -v
• 통계: nfsstat
```
