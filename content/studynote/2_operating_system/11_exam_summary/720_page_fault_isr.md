# 페이지 폴트 (Page Fault)의 발생과 ISR 처리 6단계

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 참조하려는 페이지가 현재 물리 메모리(RAM)에 없어 **유효 비트가 0인 경우 발생하는 하드웨어 트랩(Trap)**이자 예외 상황.
> 2. **가치**: 운영체제는 이 이벤트를 통해 부족한 메모리 자원을 동적으로 관리하며, 필요한 데이터만 디스크에서 읽어오는 **요구 페이징**의 실행 동력을 제공한다.
> 3. **융합**: 하드웨어의 MMU와 운영체제 커널의 페이지 폴트 핸들러(ISR)가 긴밀하게 협력하여, 사용자 프로그램 입장에서는 끊김 없는 주소 공간을 경험하게 한다.

---

### Ⅰ. 페이지 폴트 (Page Fault)의 정의

- **개념**: 프로세스가 자신의 가상 주소 공간에 정의된 페이지에 접근하려 할 때, 해당 페이지가 물리 메모리에 적재되어 있지 않아 CPU가 더 이상 명령을 수행할 수 없는 상태다.
- **발생 시점**: MMU가 페이지 테이블을 참조했는데 **유효 비트(Valid Bit)가 0(Invalid)**일 때 즉시 트랩을 발생시킨다.

---

### Ⅱ. 페이지 폴트 처리 6단계 (Procedure)

운영체제 커널은 아래 과정을 거쳐 장애를 복구한다.

1. **Trap 발생**: 하드웨어가 운영체제에 페이지 폴트가 발생했음을 알린다.
2. **상태 저장**: 현재 수행 중이던 프로세스의 레지스터와 프로그램 카운터를 저장한다.
3. **Internal Table 확인**: 운영체제는 이 접근이 정당한 메모리 접근인지, 아니면 정말 메모리에만 없는 것인지 확인한다. (부당한 접근이면 프로세스 종료)
4. **Disk I/O 수행**: 비어 있는 프레임(Free Frame)을 찾고, 디스크에서 해당 페이지를 메모리로 읽어온다. (이 동안 해당 프로세스는 대기 상태)
5. **Page Table 업데이트**: 읽기가 완료되면 페이지 테이블의 유효 비트를 1로 바꾸고 프레임 번호를 기록한다.
6. **명령 재시작**: 중단되었던 명령어를 **처음부터 다시** 실행한다.

---

### Ⅲ. 페이지 폴트 처리 아키텍처 (ASCII)

```ascii
    [ Step 1: Reference ]      [ Step 2: Trap ]        [ Step 3: Find Page ]
    CPU -> Logical Addr ----> MMU (Invalid!) --------> OS Kernel
                                     |                        |
    [ Step 6: Restart ]        [ Step 5: Update ]      [ Step 4: Swap In ]
    CPU <---- Restore State <--- Page Table <---------- Disk (Backing Store)
```

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 빈 프레임이 없을 때 (Page Replacement)
- **문제**: 디스크에서 페이지를 가져와야 하는데 RAM이 꽉 찬 경우.
- **기술사적 결단**: **페이지 교체 알고리즘(LRU, FIFO 등)**을 실행하여 희생자 페이지(Victim Page)를 선정한다. 
  - 희생된 페이지가 수정되었다면(Dirty Bit=1) 먼저 디스크에 쓰고 버려야 한다. (오버헤드 발생)

#### 2. 기술사적 인사이트
- **실행 시간 분석**: 페이지 폴트 처리는 디스크 I/O를 포함하므로 밀리초($ms$) 단위가 소요된다. 반면 메모리 접근은 나노초($ns$) 단위다. 
- **성능 목표**: 따라서 페이지 폴트가 발생하지 않도록 **Locality**를 고려한 프로그래밍과 충분한 **Working Set** 유지가 고성능 시스템 운영의 핵심이다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **가상 메모리 구현**: 실제 RAM 용량 제약으로부터 소프트웨어 독립성 확보.
- **안정적 다중 프로그래밍**: 여러 앱이 조각난 메모리에서도 원활히 수행 가능.

#### 2. 미래 전망
최근에는 **영속성 메모리(PMM)** 기술을 활용하여 디스크 I/O 단계를 메모리 복사 단계로 대체함으로써, 페이지 폴트 발생 시의 지연 시간을 혁신적으로 줄이고 있다. 또한, 예측 알고리즘을 통해 폴트가 나기 전에 미리 페이지를 가져오는 **지능형 프리페칭 (Intelligent Prefetching)** 기술이 OS 커널의 핵심 경쟁력이 되고 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. **페이지 폴트**는 공부하다가 모르는 단어가 나왔는데, 내 머릿속에 그 뜻이 없을 때 생기는 일이에요.
2. 그러면 공부를 멈추고 두꺼운 사전(디스크)을 펼쳐서 그 단어의 뜻을 찾아야 하죠.
3. 사전에서 뜻을 찾아 머릿속에 기억(메모리 적재)하고 나면, 다시 막혔던 부분부터 기분 좋게 공부를 계속할 수 있답니다!
