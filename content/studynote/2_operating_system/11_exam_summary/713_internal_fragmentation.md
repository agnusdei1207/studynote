# 내부 단편화 (Internal Fragmentation) 및 고정 분할/페이징 영향

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리를 고정된 크기로 나눈 구역에 그보다 작은 데이터를 할당했을 때, **구역 내부에서 사용되지 않고 남는 낭비 공간**.
> 2. **가치**: 외부 단편화를 해결하기 위해 도입된 **고정 분할(Fixed Partitioning)**이나 **페이징(Paging)** 기법에서 필연적으로 발생하는 Trade-off 비용이다.
> 3. **융합**: 페이지 크기가 클수록 내부 단편화는 증가하지만 관리 효율은 좋아지며, 페이지 크기가 작을수록 단편화는 줄어들지만 페이지 테이블이 비대해지는 **시스템 최적화의 균형점**을 시사한다.

---

### Ⅰ. 내부 단편화 (Internal Fragmentation)의 정의

- **개념**: 할당된 메모리 블록 내부에 프로세스가 사용하지 않는 빈 공간이 생기는 현상이다. 이 공간은 다른 프로세스가 사용할 수 없어 완전히 낭비된다.
- **💡 비유**: **4인용 식탁에 혼자 앉아 밥을 먹는 상황**과 같다. 나머지 3자리는 비어있지만, 식당 규칙상(고정 분할) 다른 모르는 사람을 그 자리에 앉힐 수 없어 공간이 낭비되는 것과 같다.

---

### Ⅱ. 고정 분할 및 페이징에서의 발생 원인 (ASCII)

#### 1. 고정 분할 (Fixed Partitioning)
- 메모리를 미리 일정한 크기(예: 100K)로 나눠두는 방식이다.

```ascii
    [ Memory Layout: 100K blocks ]
    +---------------------------+
    | Process A (80K)           |
    | [ Used ] [ 20K Empty ]    | <--- Internal Fragmentation (20K)
    +---------------------------+
    | Process B (50K)           |
    | [ Used ] [ 50K Empty ]    | <--- Internal Fragmentation (50K)
    +---------------------------+
```

#### 2. 페이징 (Paging)
- 프로그램을 고정 크기(Page)로 쪼갤 때, 마지막 조각이 페이지 크기보다 작을 경우 발생한다.

```ascii
    [ Page Size: 4KB ]
    - Program Size: 10KB
    
    Page 1: [ Full 4KB ]
    Page 2: [ Full 4KB ]
    Page 3: [ 2KB Used ] [ 2KB Waste ] <--- Internal Fragmentation
```

---

### Ⅲ. 단편화의 수학적 확률과 통계

- **평균 낭비 공간**: 페이징 시스템에서 프로세스당 평균적으로 **$\frac{1}{2} \times \text{Page Size}$** 만큼의 내부 단편화가 발생한다고 통계적으로 예측한다.
- **영향도**: 페이지 크기가 4KB라면 평균 2KB가 낭비되지만, 최근 2MB Huge Pages를 쓰면 평균 1MB가 낭비될 수 있어 영향력이 커진다.

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 페이지 크기 결정의 Trade-off
- **페이지가 작을 때**: 내부 단편화 감소, 메모리 정밀 활용 가능. 하지만 페이지 테이블이 너무 커져서 관리 비용(RAM 소모) 증가.
- **페이지가 클 때**: 페이지 테이블 크기 감소, I/O 효율성 증대(한 번에 많이 읽음). 하지만 내부 단편화 증가.
- **기술사적 결단**: 현대 OS는 기본적으로 4KB를 사용하되, 메모리 사용량이 극도로 큰 데이터베이스 서버 등에서는 성능을 위해 **Huge Pages**를 선택적으로 사용하여 TLB 히트율을 높이는 전략을 취한다.

#### 2. 슬랩 할당자 (Slab Allocator)와의 연계
- 커널 내부에서는 페이지 단위 할당으로 인한 내부 단편화를 막기 위해, 페이지를 더 잘게 쪼개어 객체 크기에 맞춰 관리하는 **슬랩 할당자**를 추가로 운영하여 낭비를 최소화한다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **메모리 할당 속도 향상**: 외부 단편화 고민 없이 고정 크기로 바로 할당 가능.
- **관리의 단순화**: 모든 조각의 크기가 같으므로 비어있는 곳 아무데나 넣어도 됨.

#### 2. 미래 전망
64비트 컴퓨팅과 수백 GB 단위의 RAM이 보편화되면서, 몇 KB 수준의 내부 단편화는 더 이상 치명적인 문제가 아니게 되었다. 오히려 **TLB 미스로 인한 성능 저하**가 더 큰 문제로 부각되고 있어, 내부 단편화를 조금 감수하더라도 페이지 크기를 키우거나(Huge Pages), 하드웨어 가속을 활용하는 방향으로 메모리 아키텍처가 진화하고 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. **내부 단편화**는 커다란 과자 봉지를 뜯었는데, 과자는 절반만 들어있고 나머지는 질소(공기)로 가득 찬 것과 같아요.
2. 봉지 크기(고정 분할)는 정해져 있는데 내용물이 작아서 생기는 빈자리죠.
3. 이 질소 낭비를 줄이려면 과자 봉지를 조금 더 작게 만들면 되지만, 그러면 봉지가 너무 많아져서 관리하기 힘들어질 수 있답니다!
