# 오브젝트 스토리지 (확장 아키텍처)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 객체 단위로 저장하는 수평 확장형 스토리지
> 2. **가치**: 무제한 확장, 내구성, REST API, 클라우드 네이티브
> 3. **융합**: S3 API, 분산 시스템, ERASURE 코딩과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

오브젝트 스토리지는 데이터를 파일 시스템 계층 대신 평면적 주소 공간(버킷)에 객체로 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    오브젝트 스토리지                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스토리지 유형 비교                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  블록 스토리지:                                       │   │ │
│   │   │  • 고정 크기 블록 (4KB 등)                            │   │ │
│   │   │  • LBA 주소 체계                                      │   │ │
│   │   │  • DB, VM에 적합                                      │   │ │
│   │   │  • 예: EBS, SAN                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  파일 스토리지:                                       │   │ │
│   │   │  • 계층적 디렉토리 구조                               │   │ │
│   │   │  • 경로 기반 접근                                     │   │ │
│   │   │  • 공유 파일에 적합                                   │   │ │
│   │   │  • 예: NFS, EFS                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  오브젝트 스토리지:                                    │   │ │
│   │   │  • 평면적 주소 공간 (Bucket)                          │   │ │
│   │   │  • Key-Value 접근                                     │   │ │
│   │   │  • 메타데이터 포함                                    │   │ │
│   │   │  • 예: S3, MinIO, GCS                                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              오브젝트 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Object = Data + Metadata + Unique ID               │   │ │
│   │   │                                                      │   │ │
│   │   │  Data: 바이너리 데이터 (크기 무제한)                  │   │ │
│   │   │  Metadata: 키-값 쌍 (사용자 정의)                     │   │ │
│   │   │  ID: 버킷 내 고유 키                                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 오브젝트 스토리지는 계층 없이 평면적으로 무제한 확장합니다.

**💡 비유**: 오브젝트 스토리지는 물건에 라벨을 붙여 창고에 보관하는 것과 같습니다.

**📢 섹션 요약 비유**: 계층 없이 무한 확장!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Bucket** | 컨테이너 | 객체 그룹 | 폴더 |
| **Object** | 데이터 | 데이터+메타 | 파일 |
| **Key** | 식별자 | 고유 ID | 이름 |
| **S3 API** | 인터페이스 | REST | 표준 |

**확장 아키텍처**

```
┌─────────────────────────────────────────────────────────┐
│                    Gateway/Proxy                        │
└───────────────────────┬─────────────────────────────────┘
                        │
    ┌───────────────────┼───────────────────┐
    │                   │                   │
┌───▼───┐           ┌───▼───┐           ┌───▼───┐
│ Node 1│           │ Node 2│           │ Node N│
│ ┌───┐ │           │ ┌───┐ │           │ ┌───┐ │
│ │OSD│ │           │ │OSD│ │           │ │OSD│ │
│ └───┘ │           │ └───┘ │           │ └───┘ │
│ ┌───┐ │           │ ┌───┐ │           │ ┌───┐ │
│ │MON│ │           │ │MON│ │           │ │MON│ │
│ └───┘ │           │ └───┘ │           │ └───┘ │
└───────┘           └───────┘           └───────┘
```

**핵심 알고리즘**: Consistent Hashing → 데이터 분산 → 복제

**📢 섹션 요약 비유**: Consistent Hash가 분산!
