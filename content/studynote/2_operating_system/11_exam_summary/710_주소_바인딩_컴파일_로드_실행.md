# 주소 바인딩 (컴파일/로드/실행)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로그램의 논리 주소를 물리 주소로 매핑하는 시점에 따른 분류 - 컴파일 시간, 적재 시간, 실행 시간
> 2. **가치**: 메모리 관리 유연성, 동적 할당 지원, 현대 OS의 기반
> 3. **융합**: MMU, 가상 메모리, 페이징, 세그멘테이션과 통합된 주소 변환

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

주소 바인딩(Address Binding)은 프로그램의 논리 주소를 물리 주소로 매핑하는 시점에 따른 분류입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    주소 바인딩 (Address Binding)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              3가지 바인딩 시점                                  │ │
│   │                                                              │ │
│   │   1. 컴파일 시간 (Compile Time)                               │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │   소스 코드 ──→ 컴파일러 ──→ 절대 코드              │   │ │
│   │   │                                                     │   │ │
│   │   │   장점: 실행 빠름                                    │   │ │
│   │   │   단점: 위치 고정, 재컴파일 필요                     │   │ │
│   │   │   예: 임베디드, 펌웨어                               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   2. 적재 시간 (Load Time)                                   │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │   재배치 가능 코드 ──→ 로더 ──→ 메모리 적재          │   │ │
│   │   │                                                     │   │ │
│   │   │   장점: 위치 유연                                    │   │ │
│   │   │   단점: 실행 중 변경 불가                             │   │ │
│   │   │   예: 정적 프로그램                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   3. 실행 시간 (Execution Time)                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │   논리 주소 ──→ MMU ──→ 물리 주소                    │   │ │
│   │   │                                                     │   │ │
│   │   │   장점: 스와핑, 가상 메모리 가능                     │   │ │
│   │   │   단점: 하드웨어(MMU) 필요                           │   │ │
│   │   │   예: 현대 OS (Linux, Windows)                       │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              바인딩 비교                                       │ │
│   │                                                              │ │
│   │   컴파일 시간 ────→ 적재 시간 ────→ 실행 시간                 │ │
│   │       ↑              ↑              ↑                        │ │
│   │    절대 코드      재배치 코드     MMU 필요                   │ │
│   │    위치 고정       위치 유연      동적 변경                   │ │
│   │    단순           중간           복잡                        │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 현대 OS는 실행 시간 바인딩을 사용하여 유연성을 확보합니다.

**💡 비유**: 주소 바인딩은 우편물 주소를 결정하는 시점과 같습니다.

**📢 섹션 요약 비유**: 언제 주소를 정하느냐가 핵심이에요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **컴파일 시간** | 미리 확정 | 절대 코드 | 확정 주소 |
| **적재 시간** | 로딩 시 결정 | 재배치 | 이사 주소 |
| **실행 시간** | 런타임 결정 | MMU | 유동 주소 |
| **MMU** | 주소 변환 | 하드웨어 | 우체국 |

**바인딩 비교**

| 시점 | 유연성 | 하드웨어 | 용도 |
|:---|:---:|:---:|:---|
| **컴파일** | 없음 | 불필요 | 임베디드 |
| **적재** | 낮음 | 불필요 | 정적 |
| **실행** | 높음 | MMU | 현대 OS |

**핵심 알고리즘**: 논리 주소 생성 → 바인딩 시점 결정 → 물리 주소 매핑

**📢 섹션 요약 비유**: 현대 OS는 실행 시간을 써요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 바인딩과 메모리 기법**

| 기법 | 바인딩 | 특징 |
|:---|:---:|:---|
| **절대 로딩** | 컴파일 | 고정 위치 |
| **재배치 로딩** | 적재 | 유연 배치 |
| **동적 로딩** | 실행 | 스와핑 가능 |
| **MMU** | 실행 | 가상 메모리 |
