# 교착 상태 복구 (프로세스 킬)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착 상태가 탐지된 후 프로세스를 종료하거나 자원을 선점하여 사이클을 해소하는 복구 기법
> 2. **가치**: 즉각적 해결, 시스템 복원, 선택적 종료
> 3. **융합**: 교착 상태 탐지, 자원 선점, 롤백과 통합된 복구 체계

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

교착 상태 복구(Deadlock Recovery)는 교착 상태가 탐지된 후 프로세스를 종료하거나 자원을 선점하여 사이클을 해소하는 복구 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    교착 상태 복구 (Deadlock Recovery)                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              복구 방법                                        │ │
│   │                                                              │ │
│   │   방법 1: 프로세스 종료                                       │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                     │   │ │
│   │   │   전체 종료: 교착 상태 프로세스 모두 종료             │   │ │
│   │   │   ┌───┐   ┌───┐   ┌───┐                            │   │ │
│   │   │   │ P1│   │ P2│   │ P3│   ──→  모두 KILL           │   │ │
│   │   │   └───┘   └───┘   └───┘                            │   │ │
│   │   │                                                     │   │ │
│   │   │   순차 종료: 하나씩 종료하며 교착 해제 확인          │   │ │
│   │   │   ┌───┐   ┌───┐   ┌───┐                            │   │ │
│   │   │   │ P1│ → │ P2│ → │ P3│   하나씩 종료 시도          │   │ │
│   │   │   └───┘   └───┘   └───┘                            │   │ │
│   │   │                                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   방법 2: 자원 선점                                          │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                     │   │ │
│   │   │   희생자 선택 → 자원 뺏기 → 롤백 → 재시작            │   │ │
│   │   │                                                     │   │ │
│   │   │   ┌───┐      ┌───┐      ┌───┐                      │   │ │
│   │   │   │ P1│ ──→  │ P2│ ──→  │ P3│                      │   │ │
│   │   │   │   │ 선점  │   │ 롤백 │   │ 재시작               │   │ │
│   │   │   └───┘      └───┘      └───┘                      │   │ │
│   │   │                                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              희생자 선택 기준                                  │ │
│   │                                                              │ │
│   │   • 프로세스 우선순위                                         │ │
│   │   • 이미 수행된 시간                                          │ │
│   │   • 앞으로 필요한 시간                                        │ │
│   │   • 보유한 자원 수                                            │ │
│   │   • 종료 비용                                                 │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 복구는 교착 상태를 물리적으로 해소하는 최후의 수단입니다.

**💡 비유**: 교착 상태 복구는 병목 구간의 차량을 견인하는 것과 같습니다.

**📢 섹션 요약 비유**: 문제 차량을 견인해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **전체 종료** | 확실한 해결 | 모두 KILL | 폭탄 |
| **순차 종료** | 최소 손실 | 하나씩 | 수술 |
| **자원 선점** | 선택적 회수 | 빼앗기 | 압수 |
| **롤백** | 상태 복원 | 체크포인트 | 되돌리기 |

**복구 비용 비교**

| 방법 | 비용 | 장점 | 단점 |
|:---|:---:|:---|:---|
| **전체 종료** | 높음 | 확실 | 낭비 |
| **순차 종료** | 중간 | 최소화 | 복잡 |
| **자원 선점** | 낮음 | 보존 | 롤백 필요 |

**핵심 알고리즘**: 탐지 → 희생자 선택 → 종료/선점 → 사이클 해제 확인

**📢 섹션 요약 비유**: 최소한의 희생으로 해결해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 종료 vs 선점**

| 비교 항목 | 프로세스 종료 | 자원 선점 |
|:---|:---:|:---:|
| **복구 속도** | 빠름 | 느림 |
| **작업 손실** | 큼 | 작음 |
| **복잡도** | 낮음 | 높음 |
| **롤백** | 불필요 | 필요 |

**📢 섹션 요약 비유**: 상황에 따라 다르게 해요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: DBMS** → 트랜잭션 롤백
**시나리오 2: OS** → 프로세스 종료

**📢 섹션 요약 비유**: DB는 롤백, OS는 종료!
