# 클라우드 컴퓨팅 (OS 가상화)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상화 기술로 컴퓨팅 자원을 풀로 관리하고 온디맨드 제공
> 2. **가치**: 탄력성, 비용 효율, 고가용성, 글로벌 확장
> 3. **융합**: IaaS/PaaS/SaaS, 컨테이너, 서버리스와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

클라우드 컴퓨팅은 가상화 기술을 기반으로 컴퓨팅 자원을 인터넷을 통해 서비스 형태로 제공합니다.

```

│                    클라우드 컴퓨팅 (OS 가상화)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              클라우드 서비스 모델                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  SaaS (Software as a Service)                       │   │ │
│   │   │  ├─ Gmail, Office 365, Salesforce                    │   │ │
│   │   │  └─ 사용자가 소프트웨어만 사용                         │   │ │
│   │   │                                                      │   │ │
│   │   │  PaaS (Platform as a Service)                        │   │ │
│   │   │  ├─ Heroku, GAE, AWS Elastic Beanstalk               │   │ │
```

> **해설**: 클라우드는 가상화로 자원을 추상화하고 풀링합니다.

**💡 비유**: 클라우드는 호텔처럼 방을 필요할 때만 빌려 쓰는 것과 같습니다.

**📢 섹션 요약 비유**: 가상화가 클라우드의 기반!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Hypervisor** | 가상화 | VCPU, VMEM | 매니저 |
| **VMM** | 관리 | 리소스 할당 | 관리자 |
| **Image** | 템플릿 | VM 스냅샷 | 설치판 |
| **Orchestrator** | 조율 | 스케줄링 | 지휘자 |

**주요 기술**

| 기술 | 설명 |
|:---|:---|
| **KVM** | Linux 커널 기반 하이퍼바이저 |
| **Xen** | Type-1 하이퍼바이저 |
| **Docker** | 컨테이너 런타임 |
| **Kubernetes** | 컨테이너 오케스트레이션 |

**핵심 알고리즘**: 요청 → 스케줄러 → 노드 선택 → 인스턴스 생성

**📢 섹션 요약 비유**: K8s가 컨테이너 표준!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: VM vs Container**

| 항목 | VM | Container |
|:---|:---:|:---:|
| **격리** | 강함 | 중간 |
| **크기** | GB | MB |
| **부팅** | 분 | 초 |
| **오버헤드** | 높음 | 낮음 |

**📢 섹션 요약 비유**: 컨테이너가 더 가볍다!
