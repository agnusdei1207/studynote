> **[Insight]**
> 마이크로소프트 윈도우(Microsoft Windows) 아키텍처는 하이브리드 커널(Hybrid Kernel) 구조를 채택하여 마이크로커널의 모듈성과 모놀리식 커널의 고성능을 절충적으로 결합한 것이 특징이다.
> 사용자 모드(User Mode)와 커널 모드(Kernel Mode)의 엄격한 분리를 유지하면서도, 시스템 서비스 간의 효율적인 통신을 위해 LPC(Local Procedure Call)와 객체 관리자(Object Manager)를 핵심 메커니즘으로 활용한다.
> 특히 하드웨어 추상화 계층인 HAL(Hardware Abstraction Layer)을 통해 다양한 하드웨어 플랫폼에서도 일관된 OS 인터페이스를 제공하는 이식성이 윈도우 생태계의 기반이 된다.

+++

### Ⅰ. 윈도우 아키텍처의 기본 구조 및 설계 원칙

1. 하이브리드 커널(Hybrid Kernel)
   - 순수 마이크로커널보다는 많은 서비스를 커널 모드에 포함하여 성능 손실을 줄이면서도 구조적 유연성을 확보한다.
2. 계층화된 구조(Layered Architecture)
   - 사용자 애플리케이션부터 하드웨어까지 명확한 계층 분리를 통해 보안과 안정성을 강화한다.
3. 객체 기반 관리(Object-based Management)
   - 프로세스, 스레드, 파일 등 시스템 자원을 객체(Object)로 추상화하여 보안 기술자(Security Descriptor)를 통한 일관된 접근 제어를 수행한다.

📢 섹션 요약 비유: 윈도우 아키텍처는 '거대한 백화점'과 같아서, 손님(사용자)이 이용하는 매장과 직원(커널)만 출입 가능한 창고가 엄격히 구분되어 관리되는 것과 같습니다.

+++

### Ⅱ. 윈도우 시스템 구성 요소 맵(Component Map)

1. 커널 모드와 사용자 모드의 구성
   - Executive Services, Kernel, HAL 등이 핵심을 이루며, 환경 서브시스템이 사용자 애플리케이션을 지원한다.

```text
[ Windows Architecture Map ]

  +---------------------------------------------------+
  | User Mode                                         |
  |  +------------+  +--------------+  +-----------+  |
  |  | User Apps  |  | System Svcs  |  | Env Subsys|  |
  |  +------------+  +--------------+  +-----------+  |
  +--------|------------------------------------------+
           | System Call (NTDLL.DLL)
  +--------v------------------------------------------+
  | Kernel Mode                                       |
  |  +---------------------------------------------+  |
  |  |              Executive Services             |  |
  |  | [Obj Mgr] [Proc Mgr] [Mem Mgr] [I/O Mgr]... |  |
  |  +---------------------------------------------+  |
  |  |                  Kernel                     |  |
  |  +---------------------------------------------+  |
  |  |        HAL (Hardware Abstraction Layer)     |  |
  |  +---------------------------------------------+  |
  +------------------------|--------------------------+
                           v
                     Hardware (CPU, RAM, I/O)
```

2. HAL(Hardware Abstraction Layer)의 역할
   - 메인보드 및 칩셋의 차이를 운영체제 상위 계층으로부터 숨겨서 동일한 바이너리가 다양한 하드웨어에서 실행되도록 한다.

📢 섹션 요약 비유: HAL은 '유니버설 어댑터'와 같아서, 어떤 모양의 하드웨어 콘센트(CPU, 보드)가 오더라도 운영체제라는 가전제품이 정상 작동할 수 있게 변환해주는 역할을 합니다.

+++

### Ⅲ. 윈도우 익제큐티브(Executive) 및 핵심 관리자

1. 객체 관리자(Object Manager)
   - 시스템 내의 모든 객체를 생성, 관리, 보호하며 객체 핸들(Handle)을 통해 자원 접근을 제어한다.
2. 메모리 관리자(Memory Manager)
   - 64비트 가상 주소 공간을 관리하며 페이징, 공유 메모리, 가상 메모리 할당 등을 담당한다.
3. I/O 관리자(I/O Manager)
   - I/O 요청 패킷(IRP, I/O Request Packet)을 사용하여 장치 독립적인 I/O 처리를 지원한다.

📢 섹션 요약 비유: 윈도우 익제큐티브는 '호텔의 각 부서 부장님들'과 같아서, 객실 관리(객체), 식재료 관리(메모리), 물류 배송(I/O) 등을 전문적으로 처리하는 팀입니다.

+++

### Ⅳ. 서브시스템 및 Win32 API 환경

1. 환경 서브시스템(Environment Subsystem)
   - Win32, POSIX, OS/2 등 다양한 운영 환경을 시뮬레이션하며, 현재는 Win32가 주력이다.
2. Win32 API (Application Programming Interface)
   - 애플리케이션이 OS 서비스에 접근하기 위한 표준 인터페이스로, DLL(Dynamic Link Library) 형태의 라이브러리를 통해 제공된다.
3. 레지스트리(Registry)
   - 시스템 구성 설정, 사용자 프로필, 설치된 프로그램 정보를 계층적 데이터베이스 형태로 통합 관리한다.

📢 섹션 요약 비유: Win32 서브시스템은 '백화점의 고객 서비스 데스크'와 같아서, 손님이 요청하는 각종 편의 기능을 친절하게 안내하고 커널에게 전달하는 창구 역할을 합니다.

+++

### Ⅴ. 윈도우 보안 모델 및 프로세스 격리

1. 보안 식별자(SID, Security Identifier)
   - 모든 사용자 및 그룹에 부여되는 고유 번호로, 접근 제어 목록(ACL, Access Control List) 검증에 사용된다.
2. 사용자 계정 컨트롤(UAC, User Account Control)
   - 일반 사용자 권한으로 프로그램을 실행하다가 관리자 권한이 필요할 때만 승인을 요청하여 보안 사고를 방지한다.
3. 프로세스 격리 및 ASLR/DEP
   - 메모리 주소 무작위화(ASLR) 및 데이터 실행 방지(DEP) 기술을 통해 익스플로잇 공격으로부터 시스템을 보호한다.

📢 섹션 요약 비유: 윈도우 보안은 '건물의 출입 통제 시스템'과 같아서, 신분증(SID)에 따라 들어갈 수 있는 방을 정해주고 위험한 행동을 할 때는 경비원(UAC)이 다시 한번 확인하는 것과 같습니다.
