# 393. 페이지 폴트 (Page Fault)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 필요한 페이지가 메모리에 없음
> 2. **가치**: 가상 메모리 구현의 핵심 메커니즘
> 3. **융합**: 페이징, 페이지 교체와 연관

---

## Ⅰ. 개요

### 개념 정의

페이지 폴트(Page Fault)는 **프로세스가 접근하려는 페이지가 물리 메모리에 없을 때 발생하는 인터럽트**다. 운영체제가 이를 처리하여 필요한 페이지를 메모리에 로드한다.

### 💡 비유: 책 빌리기
페이지 폴트는 **도서관에서 책 빌리기**와 같다. 책이 서가에 없으면 다른 곳에서 가져와야 한다. 가져올 때까지 기다려야 한다.

### 페이지 폴트 구조

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【페이지 폴트 모니터링】                                                   │
│  ──────────────────                                                  │
│  // Linux 페이지 폴트 통계                                               │
│  $ cat /proc/vmstat | grep pgfault                                    │
│  pgfault 12345678                                                     │
│                                                                     │
│  // 프로세스별 페이지 폴트                                                │
│  $ cat /proc/<pid>/stat | awk '{print $10, $12}'                      │
│  // minor faults, major faults                                        │
│                                                                     │
│  // 실시간 모니터링                                                       │
│  $ perf stat -e page-faults,page-faults:u                           │
│                                                                     │
│  【페이지 폴트 핸들러 (의사코드)】                                          │
│  ──────────────────                                                  │
│  void handlePageFault(PageTableEntry* pte, VirtualAddress addr) {  │
│      // 1. 메모리에서 빈 프레임 찾기                                       │ │
│      Frame frame = findFreeFrame();                                 │
│      if (frame == null) {                                            │
│          // 2. 없으면 교체 대상 선택                                       │ │
│          frame = selectVictimFrame();                                │
│          if (frame.dirty) {                                          │
│              // 3. 수정된 페이지는 디스크에 쓰기                             │ │
│              writeBackToDisk(frame);                                 │
│          }                                                          │
│          // 4. 페이지 테이블 갱신                                         │ │
│          invalidatePageTableEntry(frame);                            │
│      }                                                              │
│                                                                     │
│      // 5. 디스크에서 페이지 읽기                                         │ │
│      readPageFromDisk(addr.page, frame);                            │
│                                                                     │
│      // 6. 페이지 테이블 갱신                                             │ │
│      pte->frame = frame;                                             │
│      pte->valid = true;                                              │
│      pte->dirty = false;                                             │
│                                                                     │
│      // 7. TLB 갱신                                                    │ │
│      tlb_invalidate(addr);                                          │
│  }                                                                  │
│                                                                     │
│  【mmap과 페이지 폴트】                                                    │
│  ──────────────────                                                  │
│  // 메모리 매핑                                                          │
│  void* mapped = mmap(NULL, size, PROT_READ | PROT_WRITE,            │
│                       MAP_PRIVATE, fd, 0);                           │
│  // 실제 접근 시 페이지 폴트 발생 → 실제 메모리 할당                          │
│  // Demand Paging                                                     │
│                                                                     │
│  【Copy-on-Write와 페이지 폴트】                                          │
│  ──────────────────                                                  │
│  // fork() 시 페이지 공유                                                │
│  pid_t pid = fork();                                                  │
│  // 쓰기 시도 시 페이지 폴트 → 복사 후 쓰기                                │ │
│  // Copy-on-Write                                                     │
│                                                                     │
│  【페이지 폴트 성능 분석】                                                  │
│  ──────────────────                                                  │
│  // getrusage로 폴트 통계                                               │
│  #include <sys/resource.h>                                          │
│  struct rusage usage;                                                │
│  getrusage(RUSAGE_SELF, &usage);                                     │
│  printf("Minor faults: %ld\n", usage.ru_minflt);                     │
│  printf("Major faults: %ld\n", usage.ru_majflt);                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 필요한 페이지가 메모리에 없음
• 원인: 유효하지 않은 페이지, 접근 권한 없음
• 종류: Hard Fault, Soft Fault. Invalid Fault
• 처리: 디스크 읽기 → 페이지 테이블 갱신 → 재실행
• 비용: ~10ms (디스크 I/O)
• 최소화: 사전 페칭, 작업 집합 유지
```
