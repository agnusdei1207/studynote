# CAS (Compare And Swap) 명령어 기초

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 값을 비교하고 같으면 교체하는 원자적 명령어
> 2. **가치**: 락 프리(Lock-free), 원자성, 동시성
> 3. **융합**: 스핀락, 원자 변수, 락 프리 자료구조와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CAS(Compare-And-Swap)는 메모리 값을 비교하고, 예상과 같으면 새 값으로 교체하는 원자적 명령어입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CAS (Compare-And-Swap) 명령어                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CAS 동작 원리                                    │ │
│   │                                                              │ │
│   │   의사코드:                                                   │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  boolean CAS(int *ptr, int expected, int new) {      │   │ │
│   │   │      if (*ptr == expected) {                         │   │ │
│   │   │          *ptr = new;                                 │   │ │
│   │   │          return true;    // 성공                     │   │ │
│   │   │      } else {                                        │   │ │
│   │   │          return false;   // 실패                     │   │ │
│   │   │      }                                               │   │ │
│   │   │  }                                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  // 하드웨어에서 원자적으로 실행!                     │   │ │
│   │   │  // 중간에 끊기지 않음                                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CAS 활용: 카운터 증가                             │ │
│   │                                                              │ │
│   │   락 없이 원자적 증가:                                        │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  void atomic_increment(int *counter) {               │   │ │
│   │   │      int old_val;                                    │   │ │
│   │   │      do {                                            │   │ │
│   │   │          old_val = *counter;       // 읽기           │   │ │
│   │   │      } while (!CAS(counter, old_val, old_val + 1));   │ │ │
│   │   │      // CAS가 실패하면 다시 시도                      │   │ │
│   │   │  }                                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  // 여러 스레드가 동시에 호출해도 안전!               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ABA 문제                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  문제: 값이 A→B→A로 변했어도 CAS는 성공              │   │ │
│   │   │                                                      │   │ │
│   │   │  스레드 1: A 읽음 → 준비 → CAS(A, new)              │   │ │
│   │   │  스레드 2: A → B → A 로 변경 (중간에)                │   │ │
│   │   │  스레드 1: CAS 성공! (하지만 중간에 변했음)          │   │ │
│   │   │                                                      │   │ │
│   │   │  해결: 더블 워드 CAS, 버전 번호                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: CAS는 락 없이 원자적 연산을 가능하게 합니다.

**💡 비유**: CAS는 "이게 맞으면 바꿔"라는 조건부 교환입니다.

**📢 섹션 요약 비유**: 원자적 비교 후 교체!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **ptr** | 대상 주소 | 메모리 위치 | 금고 |
| **expected** | 예상 값 | 비교 대상 | 비밀번호 |
| **new** | 새 값 | 교체 값 | 새 내용 |
| **ZF** | 제로 플래그 | 성공 표시 | 결과 |

**아키텍처별 명령어**

| 아키텍처 | 명령어 |
|:---|:---|
| **x86** | cmpxchg |
| **ARM** | ldrex/strex |
| **MIPS** | ll/sc |

**핵심 알고리즘**: do { old = *ptr; } while (!CAS(ptr, old, new));

**📢 섹션 요약 비유**: cmpxchg가 x86 CAS!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 원자 카운터** → std::atomic
**시나리오 2: 락 프리 스택** → CAS 체인
**시나리오 3: 스핀락** → CAS 기반

**도입 체크리스트**: ABA 문제, 백오프

**📢 섹션 요약 비유**: std::atomic이 C++ 표준!
