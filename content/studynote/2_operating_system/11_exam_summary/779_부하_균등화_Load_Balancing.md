# 부하 균등화 (Load Balancing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 작업을 여러 리소스에 분산하여 효율성 극대화
> 2. **가치**: 고가용성, 확장성, 응답 시간 최소화
> 3. **융합**: 스케줄러, 네트워크 LB, 클러스터와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

부하 균등화(Load Balancing)는 작업이나 트래픽을 여러 서버/CPU/리소스에 분산하여 처리하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    부하 균등화 (Load Balancing)                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              로드 밸런싱 레벨                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  L4 (Transport Layer):                               │   │ │
│   │   │  • IP + Port 기반 분산                                │   │ │
│   │   │  • TCP/UDP 라우팅                                     │   │ │
│   │   │  • 예: HAProxy, LVS                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  L7 (Application Layer):                             │   │ │
│   │   │  • URL, Header, Cookie 기반                          │   │ │
│   │   │  • 콘텐츠 인식 분산                                   │   │ │
│   │   │  • 예: Nginx, Envoy, ALB                             │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스케줄링 알고리즘                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Round Robin: 순차 분산                               │   │ │
│   │   │  Weighted RR: 가중치 순차                             │   │ │
│   │   │  Least Connections: 연결 수 최소                      │   │ │
│   │   │  IP Hash: IP 기반 고정                                │   │ │
│   │   │  Least Response Time: 응답 시간 최소                  │   │ │
│   │   │                                                      │   │ │
│   │   │  CPU 스케줄러 LB:                                     │   │ │
│   │   │  • CFS Load Balancing                                 │   │ │
│   │   │  • CPU Runqueue 간 이동                               │   │ │
│   │   │  • Active/Idle 밸런싱                                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 로드 밸런싱은 OSI 레이어별로 다르게 동작합니다.

**💡 비유**: 로드 밸런서는 은행 창구 안내원처럼 고객을 분산합니다.

**📢 섹션 요약 비유**: L7이 더 스마트!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Scheduler** | 분배 | 알고리즘 | 안내 |
| **Health Check** | 상태 확인 | TCP/HTTP | 진찰 |
| **Backend** | 처리 | 실제 작업 | 창구 |
| **Sticky Session** | 세션 고정 | Cookie/IP | 단골 |

**알고리즘 비교**

| 알고리즘 | 분배 기준 | 사용 |
|:---|:---|:---|
| **RR** | 순서 | 균등 |
| **WRR** | 가중치 | 차등 |
| **LC** | 연결 수 | 불균등 |
| **Hash** | 키 | 고정 |

**핵심 알고리즘**: 부하 측정 → 대상 선택 → 요청 전달

**📢 섹션 요약 비유**: Health Check가 필수!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: LB 유형**

| 유형 | 위치 | 특징 |
|:---|:---|:---|
| **DNS LB** | DNS | 간단 |
| **HW LB** | 네트워크 | 고성능 |
| **SW LB** | 서버 | 유연 |
| **Cloud LB** | 클라우드 | 관리형 |

**📢 섹션 요약 비유**: Cloud LB가 운영 편함!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 웹 서비스** → L7 LB + WRR
**시나리오 2: DB** → Connection Pool + LB
**시나리오 3: 마이크로서비스** → Service Mesh

**도입 체크리스트**: 세션 관리, 헬스체크, 장애 조치

**📢 섹션 요약 비유**: Active-Active가 고가용성!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 가용성 99.99%, 응답 50% 개선

**미래 전망**: Service Mesh, Global LB

**참고 표준**: Proxy Protocol, xDS

**📢 섹션 요약 비유**: 로드 밸런싱은 확장의 핵심!
