# 400. 스래싱 (Thrashing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 과도한 페이지 폴트로 시스템 마비
> 2. **가치**: 메모리 관리의 경고 신호
> 3. **융합**: 페이지 교체, 작업 집합과 연관

---

## Ⅰ. 개요

### 개념 정의

스래싱(Thrashing)은 **프로세스가 작업을 수행하는 것보다 페이지 교체에 더 많은 시간을 소비하는 현상**이다. 시스템 처리량이 급격히 떨어진다.

### 💡 비유: 야생마
스래싱은 **야생마**와 같다. 말이 제멋대로 뛰어다닌다. 제어할 수 없다. 아무것도 할 수 없다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                스래싱 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스래싱 발생 원인】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 메모리 과할당:                                                    │ │
│  │     • 각 프로세스의 작업 집합 합 > 물리 메모리                          │ │
│  │                                                             │ │
│  │  2. 지역성 부족:                                                       │ │
│  │     • 프로세스가 여러 영역을 동시에 접근                                 │ │
│  │                                                             │ │
│  │  3. 부적절한 페이지 교체 알고리즘:                                       │ │
│  │     • 중요한 페이지 교체                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스래싱 방지 방법】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 작업 집합 모델 (Working Set Model):                               │ │
│  │     • 각 프로세스의 작업 집합을 메모리에 유지                              │ │
│  │     • 작업 집합 합 ≤ 물리 메모리 유지                                   │ │
│  │                                                             │ │
│  │  2. 페이지 폴트 빈도 (Page Fault Frequency):                           │ │
│  │     • 상한, 하한 설정                                                  │ │
│  │     • 상한 초과: 프레임 추가                                            │ │
│  │     • 하한 미달: 프레임 회수                                            │ │
│  │                                                             │ │
│  │  3. 지역 교체 (Local Replacement):                                    │ │
│  │     • 프로세스의 페이지만 교체                                           │ │
│  │     • 다른 프로세스에 영향 없음                                          │ │
│  │                                                             │ │
│  │  4. 프로세스 중단:                                                      │ │
│  │     • 메모리 부족 시 일부 프로세스 스왑 아웃                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【L=S 기준】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  L = 평균 페이지 폴트 간 서비스 시간                                     │ │
│  │  S = 평균 페이지 교체 서비스 시간                                        │ │
│  │                                                             │ │
│  │  • L > S: 페이지 폴트 빈도 낮음, 여유 있음                               │ │
│  │  • L = S: 최적 상태                                                    │ │
│  │  • L < S: 스래싱 발생                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스래싱 감지】                                                            │
│  ──────────────────                                                  │
│  // Linux: vmstat으로 모니터링                                          │
│  $ vmstat 1                                                          │
│  procs -----------memory---------- ---swap--  -----io----             │
│   r  b   swpd   free   buff  cache   si   so    bi    bo    in   cs  │
│   5  4  1024   1024    128   1024  100  100  1000  1000  500  200   │
│  //  ↑  ↑                                    ↑    ↑                  │
│  //  |  |                                    |    |                   │
│  //  |  └─ blocked 많음                     └── 스왑 in/out 많음       │
│  //  └─ running 많음 (실제로는 대기)                                    │
│                                                                     │
│  // 스왑 활동 급증 = 스래싱 가능성                                         │
│                                                                     │
│  【스래싱 방지 설정】                                                      │
│  ──────────────────                                                  │
│  // swappiness 조정                                                    │
│  $ cat /proc/sys/vm/swappiness    // 기본 60                          │
│  $ echo 10 > /proc/sys/vm/swappiness  // 스왑 자제                       │
│                                                                     │
│  // min_free_kbytes: 최소 여유 메모리                                    │
│  $ cat /proc/sys/vm/min_free_kbytes                                  │
│  $ echo 65536 > /proc/sys/vm/min_free_kbytes                         │
│                                                                     │
│  // vfs_cache_pressure: 캐시 압력                                       │
│  $ echo 50 > /proc/sys/vm/vfs_cache_pressure                         │
│                                                                     │
│  【작업 집합 모니터링】                                                     │
│  ──────────────────                                                  │
│  // 프로세스별 메모리 사용                                                  │
│  $ ps -o pid,vsz,rss,comm -p <pid>                                  │
│  // VSZ: 가상 메모리 크기                                                 │
│  // RSS: 실제 물리 메모리                                                  │
│                                                                     │
│  // 페이지 폴트 통계                                                       │
│  $ ps -o pid,min_flt,maj_flt,comm -p <pid>                          │
│  // min_flt: 소프트 폴트                                                 │
│  // maj_flt: 하드 폴트                                                   │
│                                                                     │
│  【OOM (Out of Memory) 킬러】                                            │
│  ──────────────────                                                  │
│  // 스래싱 극단적 상황에서 OOM 킬러 작동                                     │
│  $ dmesg | grep -i "Out of memory"                                  │
│  Out of memory: Kill process 1234 (firefox) score 500               │
│                                                                     │
│  // OOM 점수 확인                                                        │
│  $ cat /proc/<pid>/oom_score                                         │
│  // 점수가 높을수록 킬 대상                                                 │
│                                                                     │
│  // OOM 킬러 비활성화 (주의!)                                               │
│  $ echo -17 > /proc/<pid>/oom_adj                                   │
│                                                                     │
│  【애플리케이션 레벨 방지】                                                   │
│  ──────────────────                                                  │
│  // 메모리 사용 최적화                                                       │
│  class MemoryOptimizer {                                            │
│      // 데이터 스트리밍                                                    │
│      void processLargeFile(String path) {                            │
│          try (Stream<String> lines = Files.lines(Paths.get(path))) { │
│              // 전체 로드 대신 스트림 사용                                    │ │
│              lines.forEach(this::processLine);                        │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      // 캐시 크기 제한                                                    │
│      Cache<String, Object> cache = Caffeine.newBuilder()            │
│          .maximumSize(1000)  // 메모리 제한                              │ │
│          .build();                                                   │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
