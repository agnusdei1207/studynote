# 프로세스 친화성 (Affinity 설정)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스가 실행될 CPU 코어를 지정하는 메커니즘
> 2. **가치**: 캐시 지역성, NUMA 최적화, 실시간 성능
> 3. **융합**: 스케줄러, NUMA, 인터럽트와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CPU 친화성(Affinity)은 프로세스나 스레드가 실행될 CPU 코어를 제한하거나 지정하는 기능입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    프로세스 친화성 (Affinity)                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CPU 친화성 개념                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  CPU 비트마스크:                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  0b0001 → CPU 0만 사용                                │   │ │
│   │   │  0b0011 → CPU 0, 1 사용                               │   │ │
│   │   │  0b1111 → CPU 0, 1, 2, 3 사용                         │   │ │
│   │   │  0b11111111 → CPU 0-7 사용                            │   │ │
│   │   │                                                      │   │ │
│   │   │  기본: 모든 CPU 사용 가능                              │   │ │
│   │   │  지정: 특정 CPU만 사용                                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │

│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │

│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 캐시 지역성 (Cache Locality)                      │   │ │
│   │   │     • 동일 코어 실행 = 캐시 히트 증가                   │   │ │
│   │   │     • 컨텍스트 스위치 비용 감소                         │   │ │
│   │   │                                                      │   │ │
│   │   │  2. NUMA 최적화                                        │   │ │
│   │   │     • 로컬 메모리 접근                                 │   │ │
│   │   │     • 원격 메모리 접근 지양                             │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 실시간 성능                                        │   │ │
│   │   │     • 전용 코어 할당                                   │   │ │
│   │   │     • 선점 방지                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 인터럽트 격리                                      │   │ │
│   │   │     • IRQ를 특정 CPU로                                 │   │ │
│   │   │     • 워크로드 격리                                    │   │ │
```

> **해설**: 친화성은 캐시 지역성과 NUMA 최적화를 위해 사용합니다.

**💡 비유**: 친화성은 직원에게 전용 책상을 지정하는 것과 같습니다.

**📢 섹션 요약 비유**: 같은 코어 = 캐시 히트!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **cpu_mask** | 마스크 | 비트맵 | 배정 |
| **sched_setaffinity** | 시스템콜 | 설정 | 지정 |
| **taskset** | 명령어 | 런타임 | 도구 |
| **cgroups** | 그룹 | cpuset | 제어 |

**주요 API**

| API | 설명 |
|:---|:---|
| **sched_setaffinity()** | 친화성 설정 |
| **sched_getaffinity()** | 친화성 조회 |
| **pthread_setaffinity_np()** | 스레드 친화성 |

**핵심 알고리즘**: 비트마스크로 허용 CPU 지정 → 스케줄러가 해당 CPU만 사용

**📢 섹션 요약 비유**: taskset이 간편!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 격리 수준**

| 방식 | 범위 | 유연성 |
|:---|:---:|:---:|
| **taskset** | 프로세스 | 높음 |
| **cpuset cgroup** | 그룹 | 중간 |
| **isolcpus** | 부팅 | 낮음 |

**📢 섹션 요약 비유**: cgroup이 관리 편함!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: DB** → NUMA 노드에 고정
**시나리오 2: 실시간** → 전용 코어
**시나리오 3: NIC 처리** → RSS + IRQ

**도입 체크리스트**: NUMA 토폴로지, 캐시 크기

**📢 섹션 요약 비유**: NUMA는 로컬 메모리!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 캐시 히트 20-50% 향상

**미래 전망**: Hybrid Core (P/E) 스케줄링

**참고 표준**: POSIX, Linux sched

**📢 섹션 요약 비유**: 친화성은 성능 튜닝의 핵심!
