# 디바이스 드라이버 / 모듈 인터페이스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널과 하드웨어 장치 사이의 중간 계층 소프트웨어
> 2. **가치**: 하드웨어 추상화, 모듈화, 동적 로드
> 3. **융합**: 커널 API, 인터럽트, I/O 스케줄링과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

디바이스 드라이버(Device Driver)는 커널과 하드웨어 장치 사이의 인터페이스를 제공하는 소프트웨어입니다.

```

│   ┌──────────────────────────────────────────────────────────────┐ │

│   │                                                              │ │
```

**💡 비유**: 드라이버는 장치와 커널 사이의 통역사입니다.

**📢 섹션 요약 비유**: 커널 모듈로 로드!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **file_operations** | 인터페이스 | 함수 포인터 | 메뉴 |
| **module_init** | 초기화 | 로드 시 실행 | 시작 |
| **module_exit** | 정리 | 언로드 시 실행 | 종료 |
| **ioctl** | 제어 | 명령 전달 | 설정 |

**모듈 명령어**

| 명령 | 설명 |
|:---|:---|
| **insmod** | 모듈 로드 |
| **rmmod** | 모듈 언로드 |
| **modprobe** | 의존성 해결 로드 |
| **lsmod** | 모듈 목록 |

**핵심 알고리즘**: insmod → init → register → 사용 → exit → rmmod

**📢 섹션 요약 비유**: lsmod로 확인!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 장치 유형**

| 유형 | 접근 | 버퍼링 |
|:---|:---|:---|
| **문자** | 바이트 | 없음 |
| **블록** | 블록 | 있음 |
| **네트워크** | 패킷 | sk_buff |

**📢 섹션 요약 비유**: 블록 장치가 가장 복잡!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: GPU 드라이버** → 커널 모드 + 유저 모드
**시나리오 2: 네트워크 카드** → 인터럽트 + NAPI
**시나리오 3: SSD** -> NVMe 드라이버

**도입 체크리스트**: 커널 버전 호환, 라이선스 (GPL)

**📢 섹션 요약 비유**: GPL이어야 커널 심볼 접근!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 하드웨어 추상화

**미래 전망**: eBPF, 사용자 공간 드라이버 (DPDK, SPDK)

**참고 표준**: Linux Driver Model, WDM

**📢 섹션 요약 비유**: 드라이버는 하드웨어의 소프트웨어!
