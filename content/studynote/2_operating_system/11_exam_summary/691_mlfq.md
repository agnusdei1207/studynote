# 183. 다단계 피드백 큐 (Multilevel Feedback Queue)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 동작에 따라 큐간 이동
> 2. **가치**: 적응적 스케줄링, 기아 방지
> 3. **융합**: 다단계 큐, 에이징, 우선순위 스케줄링과 연관

---

## Ⅰ. 개요

### 개념 정의

다단계 피드백 큐(Multilevel Feedback Queue, MLFQ)는 **프로세스의 실행 패턴에 따라 동적으로 큐간 이동을 허용하는 스케줄링 알고리즘**이다. 다단계 큐의 확장 형태다.

### 💡 비유: 학급 성적 반 편성
MLFQ는 **학급 성적 반 편성**과 같다. 성적(동작)에 따라 상위 반으로 승격하거나 하위 반으로 강등된다.

### 다단계 피드백 큐 구조

│                                                                     │
│  【큐 구성과 이동】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Q0 (최고 우선순위)  타임퀀텀: 8ms                             │ │
│  │  ┌─────────────────────────────────────────┐                 │ │
│  │  │ 새 프로세스 진입점                       │                 │ │
│  │  │ [P1]→[P2]→[P3]  (Round Robin)           │                 │ │
│  │  └─────────────────────────────────────────┘                 │ │
│  │         │                                                    │ │
│  │         │ 타임퀀텀 소진 → 강등                                │ │
│  │         ▼                                                    │ │
│  │  Q1 (중간 우선순위)  타임퀀텀: 16ms                            │ │
│  │  ┌─────────────────────────────────────────┐                 │ │
│  │  │ [P4]→[P5]  (Round Robin)                │                 │ │
│  │  └─────────────────────────────────────────┘                 │ │
│  │         │                                                    │ │
│  │         │ 타임퀀텀 소진 → 강등                                │ │
│  │         ▼                                                    │ │
│  │  Q2 (최저 우선순위)  타임퀀텀: ∞ (FCFS)                        │ │
│  │  ┌─────────────────────────────────────────┐                 │ │
│  │  │ [P6]→[P7]→[P8]  (FCFS)                  │                 │ │
│  │  └─────────────────────────────────────────┘                 │ │
│  │         │                                                    │ │
│  │         │ 오래 대기 → 승격 (에이징)                           │ │
│  │         └─────────────────────────────────────┐               │ │
│  │                                               │               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【이동 규칙】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  상황                      동작                                │ │
│  │  ────                        ────                                │ │
│  │  새 프로세스              Q0 진입                               │ │
│  │  타임퀀텀 전부 사용        하위 큐로 강등                        │ │
│  │  I/O로 블록              현재 큐 유지 (I/O 바운드 감지)          │ │
│  │  오래 대기                상위 큐로 승격 (에이징)                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【특징】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 적응적: 프로세스 동작에 따라 우선순위 조정                     │ │
│  │  • I/O 바운드 우대: 짧게 쓰고 블록하면 높은 우선순위 유지          │ │
│  │  • CPU 바운드 하위: 길게 쓰면 낮은 우선순위로                     │ │
│  │  • 기아 방지: 에이징으로 오래 대기한 프로세스 승격                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MLFQ 구현 개요】                                                     │
│  ──────────────────                                                  │
│  #define NUM_QUEUES 8                                               │
│  #define BOOST_INTERVAL 5000  // 5초마다 boost                       │
│                                                                     │
│  struct MLFQ {                                                       │
│      Queue queues[NUM_QUEUES];                                       │
│      int time_slices[NUM_QUEUES];  // 각 큐의 타임퀀텀                │
│      int last_boost;                                                 │
│  };                                                                  │
│                                                                     │
│  void mlfq_schedule(MLFQ *mlfq, Process *p, int used_time) {         │
│      int current_q = p->queue_level;                                 │
│      int slice = mlfq->time_slices[current_q];                       │
│                                                                     │
│      if (used_time >= slice) {                                       │
│          // 타임퀀텀 모두 사용 → 강등                                 │
│          if (current_q < NUM_QUEUES - 1)                             │
│              p->queue_level++;                                       │
│      }                                                               │
│      // I/O로 블록 → 같은 큐 유지                                     │
│  }                                                                   │
│                                                                     │
│  void boost_all(MLFQ *mlfq) {                                        │
│      // 모든 프로세스를 최상위 큐로                                    │
│      for (int i = 1; i < NUM_QUEUES; i++) {                          │
│          while (!queue_empty(&mlfq->queues[i])) {                    │
│              Process *p = dequeue(&mlfq->queues[i]);                 │
│              p->queue_level = 0;                                     │
│              enqueue(&mlfq->queues[0], p);                           │
│          }                                                           │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  【BSD Unix MLFQ】                                                      │
│  ──────────────────                                                  │
│  # 전통적 Unix 스케줄러                                                │
│  # 0-127 우선순위 (낮을수록 높음)                                      │
│  # CPU 사용에 따라 우선순위 감소                                       │
│  # nice 값으로 조정                                                   │
│                                                                     │
│  $ nice -n 10 ./low_priority_task                                   │
│  $ nice -n -5 ./high_priority_task                                  │
│                                                                     │
│  【Linux CFS와의 비교】                                                 │
│  ──────────────────                                                  │
│  # MLFQ: 여러 큐 + 이동                                              │
│  # CFS: 단일 큐 + vruntime                                           │
│                                                                     │
│  # CFS도 MLFQ 개념 변형                                               │
│  # 적게 실행된 태스크가 우선 → I/O 바운드 우대                          │
│                                                                     │
│  $ cat /proc/sched_debug | head -50                                 │
│                                                                     │
│  【Windows 우선순위 부스트】                                             │
│  ──────────────────                                                  │
│  # 포그라운드 윈도우 우선순위 부스트                                    │
│  # I/O 완료 시 우선순위 일시적 증가                                     │
│  # CPU 굶기 방지를 위한 우선순위 상향                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 프로세스 동작에 따라 큐간 이동

```
