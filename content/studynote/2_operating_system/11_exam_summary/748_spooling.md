# 514. 스풀링 (Spooling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 느린 장치를 위한 데이터 큐잉
> 2. **가치**: CPU와 느린 장치의 병렬 작업
> 3. **융합**: 버퍼링, 인쇄, 배치 처리와 연관

---

## Ⅰ. 개요

### 개념 정의

스풀링(Spooling, Simultaneous Peripheral Operations On-Line)은 **느린 주변 장치를 위해 데이터를 디스크에 임시 저장했다가 순차적으로 처리하는 기술**이다.

### 💡 비유: 식당 주문 대기열
스풀링은 **식당 주문 대기열**과 같다. 주문을 받아 큐에 넣고, 요리사가 순서대로 처리한다.

├─────────────────────────────────────────────────────────────────────┤

│  │                                                             │ │

│  │          │    print()       │    print()       │    print()            │ │

│  │          ▼                  ▼                  ▼                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐            │ │

│  │  │  ┌─────────┐  ┌─────────┐  ┌─────────┐                 │            │ │
│  │  │  │파일 1   │  │파일 2   │  │파일 3   │                 │            │ │

│  │  구분        버퍼링               스풀링                                 │ │

│  │  처리 방식    즉시/짧은 지연          큐에 저장 후 순차 처리                │ │
│  │  장점        빠름                 대용량, 비동기                         │ │

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                스풀링 상세                                             │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스풀링 적용 분야】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  분야              설명                    예시                              │ │
│  │  ────              ────                    ────                              │ │
│  │  프린터            인쇄 작업 큐잉           CUPS, LPD                       │ │
│  │  이메일            메시지 큐잉              Mail Queue                      │ │
│  │  배치 처리         작업 대기열              Job Scheduler                   │ │
│  │  네트워크 전송      패킷 큐잉               MTA                             │ │
│  │  미디어 렌더링      렌더 큐잉               비디오 인코딩                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프린터 스풀링 시스템】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 사용자가 인쇄 명령                                                    │ │
│  │  2. 스풀러가 작업을 디스크에 저장                                          │ │
│  │  3. 프린터가 준비되면 스풀러가 데이터 전송                                  │ │
│  │  4. 인쇄 완료 후 스풀 파일 삭제                                            │ │
│  │                                                             │ │
│  │  장점:                                                                  │ │
│  │  • 사용자는 즉시 다른 작업 가능                                           │ │
│  │  • 여러 사용자의 인쇄 요청 큐잉                                           │ │
│  │  • 프린터 고장 시 작업 보존                                               │ │
│  │  • 인쇄 우선순위 관리 가능                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스풀링과 멀티프로그래밍】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  없는 경우:                                                              │ │
│  │  프로세스 → 인쇄 시작 → 대기 → 인쇄 완료 → 다음 작업                        │ │
│  │                    ↑                                                   │ │
│  │                CPU 유휴                                                 │ │
│  │                                                             │ │
│  │  있는 경우:                                                              │ │
│  │  프로세스 1 → 스풀에 저장 → 다른 작업 계속                                 │ │
│  │  프로세스 2 → 스풀에 저장 → 다른 작업 계속                                 │ │
│  │  스풀러 → 프린터 → 인쇄 (백그라운드)                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CUPS 프린터 스풀링】                                                 │
│  ──────────────────                                                  │
│  // 프린터 상태 확인                                                    │
│  $ lpstat -p                                                         │
│  $ lpstat -t                                                         │
│                                                                     │
│  // 인쇄 작업 목록                                                      │
│  $ lpq                                                               │
│  $ lpstat -o                                                         │
│                                                                     │
│  // 인쇄 작업 제출                                                      │
│  $ lpr file.txt                                                      │
│  $ lp -d printer_name file.txt                                       │
│                                                                     │
│  // 인쇄 작업 취소                                                      │
│  $ cancel job_id                                                     │
│  $ lprm job_id                                                       │
│                                                                     │
│  // 스풀 디렉토리                                                        │
│  $ ls /var/spool/cups                                                │
│                                                                     │
│  【메일 스풀링】                                                          │
│  ──────────────────                                                  │
│  // 메일 큐 확인                                                        │
│  $ mailq                                                             │
│  $ postqueue -p                                                      │
│                                                                     │
│  // 메일 스풀 디렉토리                                                    │
│  $ ls /var/spool/mail                                                │
│  $ ls /var/spool/postfix                                             │
│                                                                     │
│  // 큐 플러시                                                           │
│  $ postqueue -f                                                      │
│                                                                     │
│  // 특정 메일 삭제                                                      │
│  $ postsuper -d ALL                                                  │
│  $ postsuper -d <queue_id>                                           │
│                                                                     │
│  【cron/batch 작업 스풀링】                                             │
│  ──────────────────                                                  │
│  // at 작업 스풀                                                        │
│  $ at now + 1 hour                                                   │
│  at> ./script.sh                                                     │
│  at> <EOT>                                                           │
│                                                                     │
│  // at 작업 목록                                                        │
│  $ atq                                                               │
│                                                                     │
│  // batch 작업                                                         │
│  $ batch                                                             │
│  batch> ./heavy_script.sh                                            │
│  batch> <EOT>                                                        │
│                                                                     │
│  // 스풀 디렉토리                                                        │
│  $ ls /var/spool/cron                                                │
│  $ ls /var/spool/at                                                  │
│                                                                     │
│  【스풀링 구현 예시】                                                    │
│  ──────────────────                                                  │
│  #include <stdio.h>                                                  │
│  #include <stdlib.h>                                                 │
│  #include <string.h>                                                 │
│  #include <sys/stat.h>                                               │
│  #include <dirent.h>                                                 │
│                                                                     │
│  #define SPOOL_DIR "/var/spool/myapp"                                │
│                                                                     │
│  // 작업을 스풀에 추가                                                    │
│  int spool_job(const char *data, size_t len) {                       │
│      char path[256];                                                 │
│      snprintf(path, sizeof(path), "%s/job_%d",                       │
│               SPOOL_DIR, rand());                                    │
│      FILE *fp = fopen(path, "w");                                    │
│      if (!fp) return -1;                                             │
│      fwrite(data, 1, len, fp);                                       │
│      fclose(fp);                                                     │
│      return 0;                                                        │
│  }                                                                   │
│                                                                     │
│  // 스풀에서 작업 처리                                                    │
│  int process_spool(void) {                                           │
│      DIR *dir = opendir(SPOOL_DIR);                                  │
│      struct dirent *ent;                                             │
│      while ((ent = readdir(dir)) != NULL) {                          │
│          if (ent->d_name[0] == '.') continue;                        │
│          // 작업 처리                                                   │
│          char path[256];                                             │
│          snprintf(path, sizeof(path), "%s/%s",                       │
│                   SPOOL_DIR, ent->d_name);                           │
│          process_file(path);                                         │
│          unlink(path);  // 처리 후 삭제                                 │
│      }                                                               │
│      closedir(dir);                                                  │
│      return 0;                                                        │
│  }                                                                   │
│                                                                     │
│  【스풀 디렉토리 관리】                                                  │
│  ──────────────────                                                  │
│  // 스풀 디렉토리 구조                                                    │
│  /var/spool/                                                         │
│  ├── cron/         # cron 작업                                        │
│  ├── cups/         # 프린터                                            │
│  ├── mail/         # 사용자 메일                                        │
│  ├── mqueue/       # sendmail                                         │
│  ├── postfix/      # postfix                                          │
│  └── at/           # at 작업                                          │
│                                                                     │
│  // 스풀 정리                                                           │
│  $ sudo rm -rf /var/spool/cups/*   # 오래된 인쇄 작업                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 느린 장치를 위한 디스크 기반 큐잉
• 원어: Simultaneous Peripheral Operations On-Line
• 저장: 디스크 스풀 영역
• 분야: 프린터, 이메일, 배치 처리
• 장점: CPU와 I/O 병렬, 비동기 처리
• vs 버퍼링: 버퍼링(메모리), 스풀링(디스크)
• 프린터: CUPS, LPD, /var/spool/cups
• 메일: sendmail, postfix, /var/spool/mail
```
