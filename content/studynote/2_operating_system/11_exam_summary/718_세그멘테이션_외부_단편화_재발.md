# 세그멘테이션 (외부 단편화 재발)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 관점의 가변 크기 논리적 단위(코드, 데이터, 스택)로 메모리를 분할하는 기법
> 2. **가치**: 논리적 의미 보존, 공유/보호 용이, 외부 단편화 재발
> 3. **융합**: 페이징, 세그멘테이션-페이징 혼합, x86 아키텍처와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

세그멘테이션(Segmentation)은 사용자 관점의 가변 크기 논리적 단위로 메모리를 분할하는 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    세그멘테이션 (Segmentation)                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              논리적 세그먼트                                   │ │
│   │                                                              │ │
│   │   프로세스 주소 공간:                                          │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                     │   │ │
│   │   │   ┌─────────────┐                                  │   │ │
│   │   │   │   Code      │  세그먼트 0 (코드)                │   │ │
│   │   │   │   Segment   │  크기: 가변                       │   │ │
│   │   │   └─────────────┘                                  │   │ │
│   │   │                                                     │   │ │
│   │   │   ┌─────────────┐                                  │   │ │
│   │   │   │   Data      │  세그먼트 1 (데이터)              │   │ │
│   │   │   │   Segment   │  크기: 가변                       │   │ │
│   │   │   └─────────────┘                                  │   │ │
│   │   │                                                     │   │ │
│   │   │   ┌─────────────┐                                  │   │ │
│   │   │   │   Stack     │  세그먼트 2 (스택)                │   │ │
│   │   │   │   Segment   │  크기: 가변                       │   │ │
│   │   │   └─────────────┘                                  │   │ │
│   │   │                                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              세그먼트 테이블                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │ 세그먼트 │  Base   │  Limit  │  권한  │              │   │ │
│   │   │─────────┼─────────┼─────────┼────────│              │   │ │
│   │   │    0    │  0x1000 │  4KB    │  R-X   │  코드         │   │ │
│   │   │    1    │  0x5000 │  8KB    │  RW-   │  데이터       │   │ │
│   │   │    2    │  0xA000 │  2KB    │  RW-   │  스택         │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   주소 변환: 논리주소 = 세그먼트번호 + 오프셋                 │ │
│   │            물리주소 = Base[세그먼트번호] + 오프셋             │ │
│   │            조건: 오프셋 < Limit[세그먼트번호]                 │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              외부 단편화 재발                                  │ │
│   │                                                              │ │
│   │   ┌─────┬     ┬─────┬     ┬─────┐                          │ │
│   │   │Code │ 빈  │Data │ 빈  │Stack │  ← 가변 크기 = 외부 단편화│
│   │   └─────┴     ┴─────┴     ┴─────┘                          │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 세그멘테이션은 논리적 의미를 보존하지만 외부 단편화가 재발합니다.

**💡 비유**: 세그멘테이션은 책을 장(chapter)별로 나누는 것과 같습니다.

**📢 섹션 요약 비유**: 장별로 나누지만 파편이 생겨요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **세그먼트** | 논리 단위 | 가변 크기 | 장 |
| **Base** | 시작 주소 | 물리 위치 | 책장 위치 |
| **Limit** | 크기 | 세그먼트 길이 | 장 길이 |
| **세그먼트 테이블** | 매핑 정보 | Base/Limit | 목차 |

**페이징 vs 세그멘테이션**

| 특징 | 페이징 | 세그멘테이션 |
|:---|:---:|:---:|
| **크기** | 고정 | 가변 |
| **외부 단편화** | 없음 | 있음 |
| **내부 단편화** | 있음 | 없음 |
| **사용자 인식** | 없음 | 있음 |
| **공유/보호** | 페이지 단위 | 세그먼트 단위 |

**핵심 알고리즘**: 세그먼트 번호 → 테이블 조회 → Base + 오프셋 검증

**📢 섹션 요약 비유**: 논리적 보호가 쉬워요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 페이징과 세그멘테이션 혼합**

| 방식 | 특징 | 사용 |
|:---|:---|:---|
| **Pure Segmentation** | 외부 단편화 | 거의 안 씀 |
| **Pure Paging** | 내부 단편화 | 표준 |
| **Segmented Paging** | 세그먼트를 페이지로 | x86 |

**📢 섹션 요약 비유**: 혼합이 최고!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: x86 아키텍처** → 세그먼트 + 페이징
**시나리오 2: 보안 시스템** → 세그먼트 기반 보호
**시나리오 3: 현대 OS** → 주로 페이징만 사용

**도입 체크리스트**: 논리적 구조 필요성, 단편화 감수

**📢 섹션 요약 비유**: 현대 OS는 페이징을 더 좋아해요!
