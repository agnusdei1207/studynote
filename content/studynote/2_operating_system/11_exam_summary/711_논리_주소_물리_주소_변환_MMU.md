# 논리 주소 물리 주소 변환 MMU

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 생성하는 논리 주소를 메모리의 물리 주소로 동적 변환하는 하드웨어 장치
> 2. **가치**: 메모리 보호, 가상 메모리 지원, 프로세스 격리
> 3. **융합**: 페이징, 세그멘테이션, TLB, 페이지 테이블과 통합된 주소 변환

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MMU(Memory Management Unit)는 CPU가 생성하는 논리 주소를 메모리의 물리 주소로 동적 변환하는 하드웨어 장치입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MMU (Memory Management Unit)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주소 변환 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────┐      ┌─────────┐      ┌─────────┐            │ │
│   │   │   CPU   │      │   MMU   │      │  메모리  │            │ │
│   │   │         │      │         │      │         │            │ │
│   │   │ 논리주소 │ ───→ │  변환   │ ───→ │ 물리주소 │            │ │
│   │   │ (가상)  │      │         │      │ (실제)  │            │ │
│   │   └─────────┘      └─────────┘      └─────────┘            │ │
│   │                          │                                   │ │
│   │                          │                                   │ │
│   │                    ┌─────┴─────┐                            │ │
│   │                    │ 페이지 테이블│                           │ │
│   │                    │ 베이스 레지스터│                         │ │
│   │                    │ 한계 레지스터│                          │ │
│   │                    └───────────┘                            │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MMU 구성 요소                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  베이스 레지스터 (Base/Relocation Register)          │   │ │
│   │   │  • 프로세스의 물리 메모리 시작 주소                   │   │ │
│   │   │  • 논리 주소 + 베이스 = 물리 주소                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  한계 레지스터 (Limit Register)                       │   │ │
│   │   │  • 프로세스 메모리 크기                              │   │ │
│   │   │  • 메모리 보호: 논리 주소 < 한계 검사                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: MMU는 실행 시간 바인딩을 가능하게 하는 핵심 하드웨어입니다.

**💡 비유**: MMU는 우편물의 논리 주소를 실제 위치로 변환하는 우체국과 같습니다.

**📢 섹션 요약 비유**: MMU는 주소 번역기예요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **베이스 레지스터** | 시작 주소 | 오프셋 더하기 | 기준점 |
| **한계 레지스터** | 크기 제한 | 범위 검사 | 울타리 |
| **TLB** | 변환 캐시 | 고속 조회 | 메모장 |
| **페이지 테이블** | 매핑 정보 | 프레임 번호 | 지도 |

**변환 과정**

| 단계 | 동작 | 검사 |
|:---|:---|:---|
| **1** | 논리 주소 생성 | - |
| **2** | 한계 검사 | 주소 < 한계? |
| **3** | 베이스 더하기 | 논리 + 베이스 |
| **4** | 물리 주소 완성 | 메모리 접근 |

**핵심 알고리즘**: 논리 주소 → 한계 검사 → 베이스 더하기 → 물리 주소

**📢 섹션 요약 비유**: 범위를 확인하고 기준점을 더해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 주소 변환 방식**

| 방식 | MMU 역할 | 특징 |
|:---|:---|:---|
| **연속 할당** | 베이스+한계 | 단순 |
| **페이징** | 페이지 테이블 | 유연 |
| **세그멘테이션** | 세그먼트 테이블 | 논리적 |
| **혼합** | 다단계 테이블 | 복잡 |

**📢 섹션 요약 비유**: 페이징이 가장 널리 쓰여요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 가상 메모리** → MMU로 페이지 변환
**시나리오 2: 메모리 보호** → 한계 레지스터로 격리
**시나리오 3: 멀티태스킹** → 프로세스별 독립 주소 공간

**도입 체크리스트**: MMU 하드웨어, 페이지 테이블, TLB

**📢 섹션 요약 비유**: MMU가 있어야 가상 메모리가 가능해요!
