# RAID 레벨별 아키텍처 및 성능·신뢰성 비교 분석

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 개의 물리적 디스크를 하나의 논리적 장치로 묶어, **데이터를 분산 저장(Striping)**하거나 **중복 저장(Mirroring)**하여 성능과 안정성을 동시에 확보하는 기술.
> 2. **가치**: 단일 디스크의 성능 및 용량 한계를 극복하고, 디스크 장애 시에도 데이터 유실 없이 서비스를 지속할 수 있는 **결함 허용(Fault Tolerance)** 기능을 제공한다.
> 3. **융합**: 운영체제의 파일 시스템과 물리적 스토리지 하드웨어 사이의 중간 계층에서 작동하며, 현대 엔터프라이즈 서버와 클라우드 인프라의 데이터 저장 표준으로 자리 잡고 있다.

---

### Ⅰ. 주요 RAID 레벨별 아키텍처 분석

| RAID 레벨 | 핵심 기술 | 구조 설명 | 최소 디스크 |
|:---|:---|:---|:---:|
| **RAID 0** | **스트라이핑 (Striping)** | 데이터를 여러 디스크에 쪼개서 병렬 저장. | 2개 |
| **RAID 1** | **미러링 (Mirroring)** | 데이터를 똑같이 복사하여 두 디스크에 저장. | 2개 |
| **RAID 5** | **패리티 (Parity)** | 데이터와 오류 복구용 패리티를 분산 저장. | 3개 |
| **RAID 6** | **이중 패리티** | 패리티를 두 군데에 저장하여 디스크 2개 장애 견딤. | 4개 |

---

### Ⅱ. RAID 구성도 및 특징 (ASCII)

#### 1. RAID 0 (성능 최상, 안정성 최악)
- **특징**: 중복 없음. 하나만 고장 나도 전체 데이터 유실.

```ascii
    [ Disk 1 ]  [ Disk 2 ]
    | Chunk 1 |  | Chunk 2 |
    | Chunk 3 |  | Chunk 4 |
```

#### 2. RAID 1 (복구 최상, 용량 효율 최악)
- **특징**: 전체 용량의 50%만 사용 가능.

```ascii
    [ Disk 1 ]  [ Disk 2 ]
    | Data A  |  | Data A  |
    | Data B  |  | Data B  |
```

#### 3. RAID 5 (성능과 안정성의 타협점)
- **특징**: 디스크 1개 장애 복구 가능. 쓰기 시 패리티 계산 오버헤드 발생.

```ascii
    [ Disk 1 ]  [ Disk 2 ]  [ Disk 3 ]
    | Data A  |  | Data B  |  | Parity |
    | Data C  |  | Parity |  | Data D  |
    | Parity |  | Data E  |  | Data F  |
```

---

### Ⅲ. 성능 및 신뢰성 상세 비교

| 비교 항목 | RAID 0 | RAID 1 | RAID 5 | RAID 6 |
|:---|:---:|:---:|:---:|:---:|
| **읽기 성능** | **매우 우수** | 우수 | 우수 | 우수 |
| **쓰기 성능** | **매우 우수** | 보통 | 낮음 (패리티) | 매우 낮음 |
| **용량 효율** | **100% (n)** | 50% (n/2) | **(n-1)/n** | (n-2)/n |
| **신뢰성** | 없음 | 높음 | 보통 (1개 장애) | **매우 높음 (2개)** |
| **주요 용도** | 단순 고속 작업 | **OS 부팅용** | 일반 서버 데이터 | 핵심 DB, 대용량 스토리지 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 쓰기 페널티 (Write Penalty) 대응
- **문제**: RAID 5/6는 데이터 변경 시 기존 패리티를 읽고 새로 계산해서 써야 하는 '쓰기 페널티'가 발생한다.
- **기술사적 결단**: 
  - 쓰기 성능이 중요한 환경에서는 RAID 5 대신 **RAID 10 (1+0)**을 권고한다.
  - 스토리지 컨트롤러에 고속의 **NVRAM 캐시**를 탑재하여 쓰기 연산을 지연 처리(Write-back)함으로써 페널티를 완화한다.

#### 2. 기술사적 인사이트
- **Rebuild의 위험**: RAID 5에서 디스크 하나가 깨져 교체한 후, 데이터를 복구(Rebuild)하는 동안 나머지 디스크에 부하가 집중되어 또 다른 장애가 발생할 확률이 높다. 이를 위해 최신 시스템은 RAID 6나 분산 RAID 기술을 선호한다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **서비스 가용성(High Availability) 확보**: 물리적 하드웨어 장애 시에도 서비스 무중단 유지.
- **스토리지 투자 효율화**: RAID 5/6를 통한 저비용 고효율 대용량 저장소 구축.

#### 2. 미래 전망
최근에는 전통적인 하드웨어 RAID 카드 대신, 소프트웨어가 자원을 관리하는 **Erasure Coding (소거 코딩)** 기술이 클라우드 객체 스토리지(S3 등)의 표준이 되고 있다. 이는 RAID 6보다 훨씬 높은 수준의 결함 허용(예: 데이터 센터 3곳 동시 장애 견딤)을 제공하며, 소프트웨어 정의 스토리지(SDS)의 핵심으로 진화하고 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. **RAID 0**은 받아쓰기를 친구랑 나눠서 절반씩 쓰는 거예요. 엄청 빠르지만 한 명이라도 공책을 잃어버리면 내용을 다 알 수 없게 되죠.
2. **RAID 1**은 두 친구가 똑같은 내용을 각각 다 쓰는 거예요. 한 명이 잃어버려도 다른 친구 걸 보면 되니까 아주 안전하답니다.
3. **RAID 5**는 친구들끼리 힌트(패리티)를 나눠 가져서, 한 명의 공책이 없어져도 남은 친구들의 힌트를 모아 내용을 알아맞히는 마법 같은 방법이에요!
