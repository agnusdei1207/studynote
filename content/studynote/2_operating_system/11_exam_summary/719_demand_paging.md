# 요구 페이징 (Demand Paging)의 작동 원리 및 유효 비트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로그램 실행에 필요한 페이지를 미리 전부 메모리에 올리지 않고, **실제로 참조될 때(On Demand)**만 물리 메모리에 적재하는 지연 로딩(Lazy Loading) 전략.
> 2. **가치**: 물리 메모리보다 훨씬 큰 프로그램을 실행할 수 있게 하며, 불필요한 I/O를 줄여 시스템의 초기 부팅 속도와 다중 프로그래밍 정도(Degree of Multiprogramming)를 높인다.
> 3. **융합**: 페이지 테이블의 **유효/무효 비트(Valid-Invalid Bit)**를 사용하여 현재 해당 페이지가 메모리에 있는지 디스크에 있는지를 하드웨어적으로 판별한다.

---

### Ⅰ. 요구 페이징 (Demand Paging) 개요

- **정의**: 가상 메모리 시스템에서 실행 중인 프로세스가 특정 페이지를 요청할 때까지 그 페이지를 메모리에 올리지 않는 방식이다.
- **핵심 장치: 유효-무효 비트 (Valid-Invalid Bit)**:
  - **Valid (1)**: 해당 페이지가 현재 메모리에 있으며 유효함.
  - **Invalid (0)**: 해당 페이지가 가상 주소 공간에는 존재하지만, 현재 메모리에는 없음 (디스크의 Swap 영역에 존재).

---

### Ⅱ. 요구 페이징의 동작 아키텍처 (ASCII)

프로세스가 메모리에 없는 페이지에 접근하려 할 때 발생하는 흐름이다.

```ascii
    [ CPU ]                [ Page Table ]              [ Physical RAM ]
       |                          |                           |
    (1) Access Page 3             |                           |
       |---------------------> [ P3 | 0 (I) ]                 |
       |                          |                           |
    (2) Trap (Page Fault!) <------+                           |
       |                                                      |
    [ OS Kernel ] <-------------------------------------------+
       |
    (3) Find Page 3 on Disk [ Backup Store ]
    (4) Find Free Frame in RAM
    (5) Swap Page 3 into Frame 5 -----------------------> [ Frame 5 ]
    (6) Update Page Table: [ P3 | 1 (V) | Frame 5 ]
    (7) Restart Instruction
```

---

### Ⅲ. 요구 페이징의 주요 이점 (Benefits)

1. **메모리 절약**: 사용되지 않는 코드(예: 에러 처리 루틴)는 메모리를 차지하지 않음.
2. **빠른 응답**: 프로그램 시작 시 전체를 로드할 필요가 없어 실행 시작 속도가 빠름.
3. **더 많은 사용자**: 한정된 RAM에 더 많은 프로세스를 동시에 수용 가능.

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 성능 오버헤드: 페이지 폴트 (Page Fault)
- **문제**: 요구 페이징은 필연적으로 페이지 폴트를 유발하며, 이는 느린 디스크 I/O를 동반한다.
- **기술사적 결단**: 
  - **페이지 폴트 확률(p)**을 극단적으로 낮게 유지해야 한다.
  - 실질 메모리 접근 시간($EAT$) 공식: $EAT = (1-p) \times m + p \times \text{Page Fault Service Time}$
  - 디스크 접근 시간은 메모리보다 수천 배 느리므로, $p$가 0.1%만 되어도 시스템 성능은 심각하게 저하된다.

#### 2. 기술사적 인사이트
- **Pure Demand Paging**: 극단적으로 단 한 페이지도 없이 실행을 시작하는 방식이다. 초기 페이지 폴트가 집중되지만, 점차 필요한 페이지들이 채워지면서 국부성(Locality) 원리에 의해 안정화된다.
- **Pre-paging**: 성능 향상을 위해 예측 가능한 페이지들을 미리 로드하는 기법을 병행하기도 한다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **가상 메모리 완성**: 물리적 한계를 넘어서는 소프트웨어 실행 환경 제공.
- **I/O 효율화**: 필요한 데이터만 읽어 들여 전체 시스템 대역폭 최적화.

#### 2. 미래 전망
최근에는 SSD와 NVMe의 보급으로 페이지 폴트 시의 지연 시간이 급격히 줄어들고 있다. 또한, **영속성 메모리 (Persistent Memory)**의 등장으로 요구 페이징 과정에서 '디스크 읽기' 단계가 생략되거나 메모리 간 복사 수준으로 빨라지는 방향으로 메모리 아키텍처가 재편되고 있다.

1. **요구 페이징**은 공부할 때 책꽂이에 있는 수천 권의 책을 책상에 다 펴놓지 않는 것과 같아요.
