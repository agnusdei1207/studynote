# 397. 요구 페이징 (Demand Paging)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 필요할 때만 페이지를 메모리에 로드
> 2. **가치**: 메모리 효율성과 응답성 향상
> 3. **융합**: 페이징, 페이지 폴트와 연관

---

## Ⅰ. 개요

### 개념 정의

요구 페이징(Demand Paging)은 **프로세스가 실제로 필요로 하는 페이지만 메모리에 로드하는 기법**이다. 실행 전에 모든 페이지를 로드하지 않는다.

### 💡 비유: 도서관 대출
요구 페이징은 **도서관 대출**과 같다. 필요한 책만 빌린다. 모든 책을 한 번에 빌리지 않는다.

### 요구 페이징 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                요구 페이징 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 방식】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 프로세스 시작:                                                    │ │
│  │     • 페이지 테이블만 생성                                              │ │
│  │     • 유효 비트 = 0 (모든 페이지)                                      │ │
│  │     • 실제 메모리에 로드된 페이지 없음                                 │ │
│  │                                                             │ │
│  │  2. 첫 접근:                                                          │ │
│  │     • 페이지 폴트 발생                                                  │ │
│  │     • 필요한 페이지 로드                                                │ │
│  │     • 유효 비트 = 1                                                    │ │
│  │                                                             │ │
│  │  3. 이후 접근:                                                         │ │
│  │     • 필요한 페이지만 로드됨                                            │ │
│  │     • 불필요한 페이지는 메모리 낭비 없음                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 사전 로딩】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  사전 로딩 (Pre-paging):                                               │ │
│  │  • 실행 전 모든 페이지 로드                                             │ │
│  │  • 시작 시간 길어짐                                                     │ │
│  │  • 불필요한 페이지 로드 가능                                            │ │
│  │                                                             │ │
│  │  요구 페이징 (Demand Paging):                                          │ │
│  │  • 필요할 때만 로드                                                     │ │
│  │  • 시작 빠름                                                            │ │
│  │  • 메모리 효율적 사용                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【순수 요구 페이징】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  순수 요구 페이징:                                                       │ │
│  │  • 초기에 메모리에 단 하나의 페이지도 없음                              │ │
│  │  • 첫 번째 명령어부터 페이지 폴트                                       │ │
│  │  • 실행 가능하지만 초기 폴트 다발                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【요구 페이징 동작】                                                      │
│  ──────────────────                                                  │
│  // 페이지 폴트 처리                                                      │
│  void handle_demand_paging(Address vaddr) {                          │
│      // 1. 페이지 테이블 조회                                              │ │
│      PageTableEntry* pte = get_pte(vaddr);                           │
│                                                                     │
│      // 2. 유효하지 않은 경우                                                 │
│      if (!pte->valid) {                                               │
│          // 3. 빈 프레임 찾기                                             │ │
│          Frame frame = find_free_frame();                             │
│          if (frame == null) {                                          │
│              // 4. 없으면 교체                                              │ │
│              frame = select_victim_frame();                             │
│              if (frame.dirty) {                                          │
│                  // 5. 수정된 페이지 디스크에 쓰기                              │ │
│                  write_to_disk(frame);                                   │
│              }                                                              │
│          }                                                                  │
│          // 6. 디스크에서 페이지 읽기                                         │ │
│          read_from_disk(vaddr.page, frame);                            │
│          // 7. 페이지 테이블 갱신                                            │ │
│          pte->frame = frame;                                           │
│          pte->valid = true;                                            │
│          pte->dirty = false;                                           │
│      }                                                                  │
│  }                                                                  │
│                                                                     │
│  【mmap과 요구 페이징】                                                    │
│  ──────────────────                                                  │
│  // mmap은 실제 메모리를 할당하지 않음                                      │ │
│  void* addr = mmap(NULL, size, PROT_READ | PROT_WRITE,               │
│                    MAP_PRIVATE | MAP_ANONYMOUS, -1, 0);              │
│                                                                     │
│  // 실제 접근 시 페이지가 할당됨                                             │ │
│  memset(addr, 0, size);  // 여기서 실제 물리 페이지 할당                │
│                                                                     │
│  【사전 페칭 구현】                                                        │
│  ──────────────────                                                  │
│  void prefetch_pages(Process* proc, Address start, int count) {    │
│      for (int i = 0; i < count; i++) {                               │
│          Address addr = start + i * PAGE_SIZE;                       │
│          PageTableEntry* pte = get_pte(proc, addr);                 │
│          if (!pte->valid) {                                           │
│              // 백그라운드에서 페이지 로드                                  │ │
│              async_load_page(pte, addr);                             │
│          }                                                              │
│      }                                                                  │
│  }                                                                  │
│                                                                     │
│  【작업 집합 유지】                                                        │
│  ──────────────────                                                  │
│  void maintain_working_set(Process* proc) {                         │
│      // 최근 참조된 페이지 추적                                            │ │
│      Set<Page> working_set = proc->recently_referenced;             │
│                                                                     │
│      // 작업 집합이 메모리에 있도록 유지                                     │ │
│      for (Page p : working_set) {                                    │
│          if (!p.in_memory) {                                         │
│              load_page(p);                                            │
│          }                                                              │
│      }                                                                  │
│  }                                                                  │
│                                                                     │
│  【요구 페이징 모니터링】                                                  │
│  ──────────────────                                                  │
│  // 페이지 폴트 통계                                                        │
│  $ cat /proc/<pid>/stat | grep -E "min_flt|maj_flt"                 │
│  min_flt: 12345                                                      │
│  maj_flt: 678                                                          │
│                                                                     │
│  // 메모리 통계                                                           │
│  $ cat /proc/meminfo | grep -i "Demand"                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 필요할 때만 페이지 로드
• 특징: 초기에 페이지 없음, 접근 시 로드
• 장점: 메모리 효율, 빠른 시작, 큰 프로그램 실행
• 단점: 초기 폴트 오버헤드, 복잡한 관리
• 최적화: 사전 페칭, 작업 집합 유지
• 활용: 대부분의 현대 OS
```
