# 은행원 알고리즘 안전 상태

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 인스턴스 자원 환경에서 교착 상태를 회피하기 위해 안전 순서 존재 여부를 검사하는 알고리즘
> 2. **가치**: 사전 예방, 안전 순서 보장, 자원 효율성과 안정성 균형
> 3. **융합**: 교착 상태 회피, 자원 할당 그래프, Available/Max/Allocation/Need 행렬과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

은행원 알고리즘(Banker's Algorithm)은 다중 인스턴스 자원 환경에서 교착 상태를 회피하기 위해 안전 순서(Safe Sequence) 존재 여부를 검사하는 알고리즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    은행원 알고리즘 (Banker's Algorithm)               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              자료구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Available (가용 자원)                               │   │ │
│   │   │  [m개] - 현재 사용 가능한 각 자원 타입의 수           │   │ │
│   │   │  예: Available = [3, 3, 2] (A=3, B=3, C=2)          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Max (최대 요구)                                     │   │ │
│   │   │  [n × m] - 각 프로세스가 필요로 하는 최대 자원        │   │ │
│   │   │  예: P0 = [7, 5, 3]                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Allocation (할당됨)                                 │   │ │
│   │   │  [n × m] - 현재 각 프로세스에 할당된 자원             │   │ │
│   │   │  예: P0 = [0, 1, 0]                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Need (필요)                                         │   │ │
│   │   │  [n × m] - 추가로 필요한 자원 = Max - Allocation     │   │ │
│   │   │  예: P0 = [7, 4, 3]                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              안전 상태 판별                                    │ │
│   │                                                              │ │
│   │   Work = Available (초기화)                                  │ │
│   │   Finish[i] = false (모든 i)                                 │ │
│   │                                                              │ │
│   │   반복:                                                      │ │
│   │     Find i: Finish[i] = false AND Need[i] ≤ Work            │ │
│   │     if found:                                                │ │
│   │       Work = Work + Allocation[i]                           │ │
│   │       Finish[i] = true                                      │ │
│   │     else:                                                    │ │
│   │       break                                                  │ │
│   │                                                              │ │
│   │   if 모든 Finish[i] = true: 안전 상태                        │ │
│   │   else: 불안전 상태                                          │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 안전 상태면 모든 프로세스가 완료될 수 있는 순서가 존재합니다.

**💡 비유**: 은행원 알고리즘은 은행이 대출 승인 전 예금 인출 시뮬레이션을 하는 것과 같습니다.

**📢 섹션 요약 비유**: 은행처럼 신중하게 자원을 빌려줘요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Available** | 가용 자원 | 현재 사용 가능 | 현금 |
| **Max** | 최대 요구 | 총 필요량 | 대출 한도 |
| **Allocation** | 할당됨 | 현재 대출액 | 대출금 |
| **Need** | 추가 필요 | Max - Allocation | 추가 대출 |

**안전 vs 불안전 상태**

| 상태 | 의미 | 결과 |
|:---|:---|:---|
| **안전** | 안전 순서 존재 | 교착 없음 |
| **불안전** | 순서 없음 | 교착 가능 |

**핵심 알고리즘**: 요청 → Need ≤ Available? → 안전성 검사 → 승인/거부

**📢 섹션 요약 비유**: 안전 순서가 있어야 해요!
