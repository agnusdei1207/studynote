# 224. 경쟁 조건 (Race Condition)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실행 순서에 따라 결과가 달라지는 상황
> 2. **가치**: 동기화 없는 병렬 프로그래밍의 주요 버그
> 3. **융합**: 임계 영역, 상호 배제, 원자성과 연관

---

## Ⅰ. 개요

### 개념 정의

경쟁 조건(Race Condition)은 **여러 스레드가 공유 자원에 동시에 접근하고, 실행 순서에 따라 결과가 달라지는 버그 상황**이다. 동기화 없이 병렬 실행할 때 발생한다.

### 💡 비유: 동시 문 열기
경쟁 조건은 **두 사람이 동시에 문을 여는 것**과 같다. 누가 먼저 잡느냐에 따라 결과가 다르다. 충돌할 수도 있다.

### 경쟁 조건 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                경쟁 조건 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전형적인 예: 카운터 증가】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  int counter = 0;                                              │ │
│  │                                                             │ │
│  │  counter++ 는 실제로 3단계:                                     │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  1. LOAD:  register = counter    (메모리 → 레지스터)      │ │ │
│  │  │  2. ADD:   register = register + 1                      │ │ │
│  │  │  3. STORE: counter = register    (레지스터 → 메모리)      │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  스레드 A                      스레드 B                        │ │
│  │  ────────                      ────────                       │ │
│  │  LOAD counter (0)                                             │ │
│  │                               LOAD counter (0)  ← 같은 값!     │ │
│  │  ADD (1)                                                      │ │
│  │                               ADD (1)                          │ │
│  │  STORE counter (1)                                            │ │
│  │                               STORE counter (1)  ← 덮어쓰기!   │ │
│  │                                                             │ │
│  │  결과: counter = 1 (예상: 2)                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【실행 순서에 따른 결과】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  순서                   counter 값                              │ │
│  │  ────                       ────                                │ │
│  │  A→A→A → B→B→B         2 (정상)                                │ │
│  │  B→B→B → A→A→A         2 (정상)                                │ │
│  │  A→B → A→B → A→B       2 (정상)                                │ │
│  │  A→A→B→B→A→B           1 (버그!)                                │ │
│  │  A→B→B→A→A→B           1 (버그!)                                │ │
│  │                                                             │ │
│  │  실행 순서에 따라 결과가 다름 = 경쟁 조건                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【경쟁 조건 유형】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  유형                설명                    예시                │ │
│  │  ────                    ────                        ────        │ │
│  │  Read-Modify-Write  읽고 쓰기 사이 끼어듦      counter++         │ │
│  │  Check-Then-Act     조건 확인과 실행 사이      if (ptr) free()   │ │
│  │  Read-Read          두 읽기 사이 값 변경       double-checked    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                경쟁 조건 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Check-Then-Act 예시】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 위험한 코드: double-free 가능                               │ │
│  │  if (ptr != NULL) {                                            │ │
│  │      // ← 여기서 다른 스레드가 ptr을 해제하면?                   │ │
│  │      free(ptr);                                                 │ │
│  │      ptr = NULL;                                                │ │
│  │  }                                                             │ │
│  │                                                             │ │
│  │  // 해결: 락 사용                                                │ │
│  │  pthread_mutex_lock(&mutex);                                   │
│  │  if (ptr != NULL) {                                            │ │
│  │      free(ptr);                                                 │ │
│  │      ptr = NULL;                                                │ │
│  │  }                                                             │ │
│  │  pthread_mutex_unlock(&mutex);                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TOCTOU (Time-of-Check to Time-of-Use)】                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 파일 존재 확인 후 사용                                        │ │
│  │  if (access("/tmp/file", R_OK) == 0) {                         │ │
│  │      // ← 여기서 파일이 변경되면?                                │ │
│  │      fd = open("/tmp/file", O_RDONLY);                         │ │
│  │  }                                                             │ │
│  │                                                             │ │
│  │  // 보안 취약점의 흔한 원인                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【발견의 어려움】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 재현 어려움: 타이밍 의존                                       │ │
│  │  • 간헐적 발생: 특정 조건에서만                                   │ │
│  │  • 테스트 부족: 순차적 테스트로는 발견 안 됨                       │ │
│  │  • 디버깅 어려움: 디버거가 타이밍 변경                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 탐지 및 해결
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【정적 분석 도구】                                                    │
│  ──────────────────                                                  │
│  # ThreadSanitizer (GCC/Clang)                                      │
│  $ gcc -fsanitize=thread -g race.c                                  │
│  $ ./a.out                                                          │
│  ==================                                                  │
│  WARNING: ThreadSanitizer: data race (pid=1234)                     │
│    Write of size 4 at 0x... by thread T2:                           │
│    Previous read of size 4 at 0x... by thread T1:                   │
│                                                                     │
│  # Helgrind (Valgrind)                                              │
│  $ valgrind --tool=helgrind ./program                              │
│                                                                     │
│  # Clang Static Analyzer                                            │
│  $ scan-build make                                                  │
│                                                                     │
│  【해결 방법】                                                         │
│  ──────────────────                                                  │
│  // 1. Mutex 사용                                                    │
│  pthread_mutex_lock(&mutex);                                        │
│  counter++;                                                         │
│  pthread_mutex_unlock(&mutex);                                      │
│                                                                     │
│  // 2. Atomic 연산                                                   │
│  #include <stdatomic.h>                                             │
│  atomic_int counter = 0;                                            │
│  atomic_fetch_add(&counter, 1);                                     │
│                                                                     │
│  // 3. Thread-local storage                                         │
│  __thread int local_counter = 0;                                    │
│  local_counter++;  // 경쟁 없음                                      │
│                                                                     │
│  // 4. RCU (Read-Copy-Update)                                       │
│  rcu_read_lock();                                                   │
│  p = rcu_dereference(ptr);                                          │
│  // 읽기                                                             │
│  rcu_read_unlock();                                                 │
│                                                                     │
│  【테스트 기법】                                                       │
│  ──────────────────                                                  │
│  # 스트레스 테스트                                                    │
│  $ stress-ng --cpu 8 --io 4 --vm 2 --timeout 60s                   │
│                                                                     │
│  # 무작위 스케줄링                                                    │
│  $ perf record -e sched:sched_switch ./program                     │
│                                                                     │
│  # Concurrency testing                                              │
│  for i in {1..1000}; do ./program || echo "FAILED at $i"; done     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
