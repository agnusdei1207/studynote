# 456. C-SCAN 스케줄링 (Circular SCAN)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 방향으로만 서비스 후 처음으로 점프
> 2. **가치**: 균등한 대기 시간 보장
> 3. **융합**: SCAN 스케줄링, 디스크 성능과 연관

---

## Ⅰ. 개요

### 개념 정의

C-SCAN 스케줄링(Circular SCAN)은 **SCAN의 변형으로, 한 방향으로만 서비스하고 끝에 도달하면 처음으로 점프하는 알고리즘**이다.

### 💡 비유: 원형 트랙
C-SCAN은 **원형 레이스 트랙**과 같다. 한 방향으로 계속 달리며, 끝에 도달하면 처음으로 순간 이동한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                C-SCAN 상세                                            │ |
├─────────────────────────────────────────────────────────────────────┤

│  │  C-SCAN:                                                           │ │
│  │  0 ─────────────────────────────────────────→ 199               │ │

│  │  199 ════════════════════════════════════════→ 0                 │ │
│  │     (점프만, 서비스 없음)                                           │ │
│  │                                                             │ │

│                                                                     │

│  │     • 모든 위치에 동일한 패턴으로 서비스                                │ │
│  │     • SCAN의 양 끝단 불리 해결                                       │ │

│  │                                                             │ │
│  │  3. 예측 가능:                                                      │ │
│  │     • 요청 위치로 예상 처리 시간 계산 가능                              │ │

│  ┌───────────────────────────────────────────────────────────────┐ │
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C-SCAN 구현 (의사코드)】                                          │
│  ──────────────────                                                  │
│  void cscan_schedule(Queue requests, int head) {                    │
│      sort(requests);  // 오름차순 정렬                                 │
│                                                                     │
│      // 현재 위치 이후 요청 처리                                       │
│      for (req in requests where req.track >= head) {                │
│          processRequest(req);                                        │
│          head = req.track;                                           │
│      }                                                                │
│                                                                     │
│      // 끝까지 이동                                                    │
│      head = MAX_TRACK;  // 199                                       │
│                                                                     │
│      // 처음으로 점프                                                   │
│      head = 0;  // 점프 (서비스 없이)                                  │
│                                                                     │
│      // 처음부터 남은 요청 처리                                        │
│      for (req in requests where req.track < original_head) {        │
│          processRequest(req);                                        │
│          head = req.track;                                           │
│      }                                                                │
│  }                                                                   │
│                                                                     │
│  【C-LOOK 변형】                                                      │
│  ──────────────────                                                  │
│  void clook_schedule(Queue requests, int head) {                    │
│      sort(requests);                                                 │
│                                                                     │
│      // 현재 위치 이후 요청 처리                                       │
│      for (req in requests where req.track >= head) {                │
│          processRequest(req);                                        │
│          head = req.track;                                           │
│      }                                                                │
│                                                                     │
│      // 끝까지 안 가고 바로 처음으로                                    │
│      // 가장 낮은 트랙으로 점프                                         │
│      head = min(requests.track);                                     │
│                                                                     │
│      // 남은 요청 처리                                                │
│      for (req in remaining_requests) {                              │
│          processRequest(req);                                        │
│          head = req.track;                                           │
│      }                                                                │
│  }                                                                   │
│                                                                     │
│  【리눅스 Deadline 스케줄러】                                          │
│  ──────────────────                                                  │
│  // Deadline는 C-SCAN과 유사한 특성                                    │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  noop [deadline] cfq                                                 │
│                                                                     │
│  // Deadline 파라미터                                                  │
│  $ cat /sys/block/sda/queue/iosched/read_expire                     │
│  500  // 읽기 요청 만료 시간 (ms)                                       │
│                                                                     │
│  $ cat /sys/block/sda/queue/iosched/write_expire                    │
│  5000  // 쓰기 요청 만료 시간 (ms)                                      │
│                                                                     │
│  $ cat /sys/block/sda/queue/iosched/fifo_batch                      │
│  16  // 배치 크기                                                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 한 방향만 서비스, 끝에서 처음으로 점프
• 장점: 균등한 대기 시간, 기아 없음
• 단점: 점프 시간 낭비
• 변형: C-LOOK (끝까지 안 감)
• 비교: SCAN보다 공정
• 적합: 실시간, 공정성 중요
• 구현: 정렬 + 선형 스캔 + 점프
```
