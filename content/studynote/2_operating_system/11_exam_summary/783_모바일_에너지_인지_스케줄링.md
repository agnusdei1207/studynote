# 모바일 에너지 인지 스케줄링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 배터리 수명을 최적화하는 전력 관리 스케줄링
> 2. **가치**: 배터리 지속, 열 관리, 사용자 경험
> 3. **융합**: DVFS, big.LITTLE, EAS와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

에너지 인지 스케줄링(Energy-Aware Scheduling)은 배터리 수명과 전력 소비를 고려하여 작업을 할당하는 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    모바일 에너지 인지 스케줄링                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              모바일 전력 특성                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  전력 소비 요소:                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  CPU:                                                 │   │ │
│   │   │  • DVFS (동적 전압/주파수 스케일링)                     │   │ │
│   │   │  • C-state (유휴 상태)                                 │   │ │
│   │   │  • big.LITTLE (이종 코어)                              │   │ │
│   │   │                                                      │   │ │
│   │   │  Display:                                             │   │ │
│   │   │  • 백라이트 밝기                                       │   │ │
│   │   │  • OLED vs LCD                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Radio:                                               │   │ │
│   │   │  • 셀룰러, Wi-Fi, Bluetooth                           │   │ │
│   │   │  • 전송 주기 최적화                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  GPU, NPU, 기타:                                       │   │ │
│   │   │  • 그래픽, AI 가속기                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │

│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  성능 코어 (big):                                     │   │ │
│   │   │  • 높은 성능                                          │   │ │
│   │   │  • 높은 전력 소비                                      │   │ │
│   │   │  • 무거운 작업                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  효율 코어 (LITTLE):                                   │   │ │
│   │   │  • 낮은 성능                                          │   │ │
│   │   │  • 낮은 전력 소비                                      │   │ │
│   │   │  • 가벼운 작업                                        │   │ │
│   │   │                                                      │   │ │

│   │   │  • 에너지 모델 참조                                    │   │ │
│   │   │  • 적절한 코어 선택                                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
```

> **해설**: 이종 멀티코어는 성능과 전력의 균형을 달성합니다.

**💡 비유**: big.LITTLE은 경차와 스포츠카를 상황에 맞게 선택하는 것과 같습니다.

**📢 섹션 요약 비유**: 작은 일은 작은 코어!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **EAS** | 스케줄러 | 에너지 인식 | 관리자 |
| **EM** | 모델 | 전력 프로파일 | 지도 |
| **DVFS** | 주파수 | 전압/클럭 조절 | 속도 |
| **C-state** | 유휴 | 절전 모드 | 휴식 |

**주요 기술**

| 기술 | 설명 |
|:---|:---|
| **EAS** | Energy Aware Scheduling |
| **HMP** | Heterogeneous MP |
| **schedutil** | CPUFreq governor |
| **thermal** | 열 제어 |

**핵심 알고리즘**: 작업 분류 → 에너지 비용 계산 → 코어/주파수 선택

**📢 섹션 요약 비유**: EM(에너지 모델)이 필수!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 전력 관리**

| 기술 | 대상 | 오버헤드 |
|:---|:---:|:---:|
| **DVFS** | 주파수 | 낮음 |
| **C-state** | 유휴 | 낮음 |
| **big.LITTLE** | 코어 | 중간 |
| **Hotplug** | 코어 온/오프 | 높음 |

**📢 섹션 요약 비유**: DVFS + big.LITTLE 조합!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 백그라운드** → LITTLE 코어
**시나리오 2: 게임** → big 코어 + 높은 주파수
**시나리오 3: 동영상** → 균형

**도입 체크리스트**: EM 정확도, 열 설계

**📢 섹션 요약 비유**: Thermal이 제약 조건!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 배터리 수명 20-50% 연장

**미래 전망**: ARMv9, DynamiQ, CPU + NPU 통합

**참고 표준**: EAS, ACPI, PSCI

**📢 섹션 요약 비유**: 에너지 인지는 모바일 필수!
