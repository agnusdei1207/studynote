# ZFS 파일 시스템 (스냅샷)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Copy-on-Write 기반의 트랜잭션형 파일 시스템
> 2. **가치**: 데이터 무결성, 스냅샷, 중복 제거, 무제한 확장
> 3. **융합**: RAID, L2ARC, 복제, 압축과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ZFS는 Sun Microsystems에서 개발한 COW(Copy-on-Write) 기반의 통합 스토리지 시스템입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ZFS 파일 시스템                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ZFS 아키텍처                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  ZFS Stack:                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  ZPOOL (스토리지 풀)                                  │   │ │
│   │   │    ├─ vdev (가상 디바이스)                            │   │ │
│   │   │    │   ├─ mirror (미러)                               │   │ │
│   │   │    │   ├─ raidz1/2/3 (RAID)                          │   │ │
│   │   │    │   └─ disk (단일)                                 │   │ │
│   │   │    └─ spares, cache, log                             │   │ │
│   │   │                                                      │   │ │
│   │   │  ZFS (파일 시스템)                                    │   │ │
│   │   │    ├─ dataset (데이터셋)                              │   │ │
│   │   │    ├─ snapshot (스냅샷)                               │   │ │
│   │   │    ├─ clone (복제)                                    │   │ │
│   │   │    └─ volume (볼륨)                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              핵심 기능                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Copy-on-Write (COW):                                 │   │ │
│   │   │  • 데이터 덮어쓰기 안 함                               │   │ │
│   │   │  • 항상 새 블록에 쓰기                                 │   │ │
│   │   │  • 크래시 일관성 보장                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Checksum (체크섬):                                   │   │ │
│   │   │  • 모든 블록에 fletcher4/SHA256                       │   │ │
│   │   │  • 데이터 무결성 보장                                  │   │ │
│   │   │  • 자동 복구 (미러/RAIDZ)                             │   │ │
│   │   │                                                      │   │ │
│   │   │  Snapshot (스냅샷):                                    │   │ │
│   │   │  • 즉시 생성 (상수 시간)                               │   │ │
│   │   │  • 공간 효율적 (COW)                                   │   │ │
│   │   │  • 읽기 전용                                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: ZFS는 COW로 스냅샷, 무결성, 일관성을 동시에 달성합니다.

**💡 비유**: ZFS는 모든 변경을 새 페이지에 쓰는 영구 노트와 같습니다.

**📢 섹션 요약 비유**: COW가 ZFS의 핵심!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **zpool** | 풀 | 디스크 집합 | 창고 |
| **vdev** | 가상장치 | RAID 구성 | 선반 |
| **dataset** | 파일시스템 | 계층 구조 | 폴더 |
| **snapshot** | 스냅샷 | 읽기 전용 | 사진 |

**주요 기능**

| 기능 | 설명 |
|:---|:---|
| **RAIDZ** | 소프트웨어 RAID |
| **L2ARC** | 캐시 SSD |
| **ZIL** | 로그 SSD |
| **Dedup** | 중복 제거 |

**핵심 알고리즘**: 쓰기 → 새 블록 할당 → 체크섬 → 포인터 갱신

**📢 섹션 요약 비유**: zfs snapshot이 즉시!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 파일 시스템**

| 항목 | ext4 | ZFS |
|:---|:---:|:---:|
| **무결성** | 선택 | 필수 |
| **스냅샷** | LVM 필요 | 기본 |
| **RAID** | mdadm | 내장 |
| **압축** | 없음 | 기본 |

**📢 섹션 요약 비유**: ZFS가 올인원!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: NAS** → ZFS + RAIDZ2
**시나리오 2: VM 스토리지** → ZFS + 스냅샷
**시나리오 3: 백업** → ZFS send/recv

**도입 체크리스트**: ECC RAM, 디스크 크기

**📢 섹션 요약 비유**: ECC RAM 권장!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 무결성 100%, 스냅샷 즉시

**미래 전망**: OpenZFS 2.0+, dRAID

**참고 표준**: OpenZFS

**📢 섹션 요약 비유**: ZFS는 데이터 무결성의 표준!
