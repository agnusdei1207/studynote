# SELinux (보안 강제 접근 통제)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 수준에서 모든 접근을 정책으로 제어하는 보안 모듈
> 2. **가치**: 최소 권한, 권한 상승 방지, 컨테이너 보안
> 3. **융합**: LSM, DAC, 컨테이너, 정책 관리와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SELinux(Security-Enhanced Linux)는 커널 수준에서 모든 접근을 정책으로 제어하는 MAC(Mandatory Access Control) 구현체입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SELinux (Security-Enhanced Linux)                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DAC vs MAC                                       │ │
│   │                                                              │ │
│   │   DAC (Discretionary Access Control):                        │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  • 소유자가 권한 결정                                 │   │ │
│   │   │  • chmod, chown으로 제어                              │   │ │
│   │   │  • 유닉스 전통 방식                                   │   │ │
│   │   │  • 한계: 사용자 실수, 악성 프로그램                   │   │ │
│   │   │  • 예: 사용자가 파일을 777로 설정 가능                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   MAC (Mandatory Access Control):                            │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  • 시스템 정책이 권한 결정                            │   │ │
│   │   │  • 관리자만 정책 변경 가능                            │   │ │
│   │   │  • DAC 권한보다 우선                                  │   │ │
│   │   │  • 모든 접근이 명시적으로 허용되어야 함               │   │ │
│   │   │  • 루트도 정책 벗어날 수 없음                         │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SELinux 컨텍스트                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  user:role:type:level                                │   │ │
│   │   │                                                      │   │ │
│   │   │  예: unconfined_u:unconfined_r:unconfined_t:s0      │   │ │
│   │   │      system_u:system_r:httpd_t:s0                   │   │ │
│   │   │                                                      │   │ │
│   │   │  Type Enforcement (TE):                              │   │ │
│   │   │  • httpd_t 타입의 프로세스는 httpd_sys_content_t     │   │ │
│   │   │    타입의 파일만 읽을 수 있음                         │   │ │
│   │   │  • 타입 간 접근 규칙이 정책에 정의됨                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 명령어                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  getenforce: 현재 모드 확인                           │   │ │
│   │   │  setenforce 0/1: Permissive/Enforcing 전환           │   │ │
│   │   │  ls -Z, ps -Z: 컨텍스트 확인                          │   │ │
│   │   │  chcon: 컨텍스트 변경                                 │   │ │
│   │   │  restorecon: 기본 컨텍스트 복원                       │   │ │
│   │   │  audit2allow: 로그 기반 정책 생성                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: SELinux는 DAC 이후에 추가로 검사합니다.

**💡 비유**: SELinux는 은행 금고의 이중 잠금 장치와 같습니다.

**📢 섹션 요약 비유**: Enforcing이 기본!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **LSM** | 커널 훅 | 접근 제어 | 검문 |
| **Policy** | 규칙 | allow/deny | 법 |
| **Context** | 레이블 | user:role:type | 신분증 |
| **AVC** | 캐시 | 결정 저장 | 기억 |

**SELinux 모드**

| 모드 | 설명 |
|:---|:---|
| **Enforcing** | 정책 강제 |
| **Permissive** | 로그만 |
| **Disabled** | 끔 |

**핵심 알고리즘**: 접근 요청 → AVC 확인 → 정책 평가 → 허용/거부

**📢 섹션 요약 비유**: Permissive로 디버깅!
