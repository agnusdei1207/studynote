# 데드락 희생자 / 롤백 복구망

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착 상태를 희생자 선택과 롤백으로 복구하는 기법
> 2. **가치**: 데드락 탈출, 시스템 복원, 트랜잭션 안전성
> 3. **융합**: 교착 상태 탐지, 체크포인트, 로그와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

데드락 희생자(Victim) 선택과 롤백(Rollback)은 교착 상태를 해결하기 위해 희생 프로세스를 선택하고 이전 상태로 복구하는 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    데드락 희생자 / 롤백 복구                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              데드락 탐지 후 복구 과정                          │ │
│   │                                                              │ │
│   │   1. 데드락 탐지                                              │ │
│   │      ┌─────────────────────────────────────────────────────┐ │ │
│   │      │  자원 할당 그래프에서 사이클 존재 확인               │ │ │
│   │      │  P1 → R1 → P2 → R2 → P1 (사이클!)                  │ │ │
│   │      └─────────────────────────────────────────────────────┘ │ │
│   │                                                              │ │
│   │   2. 희생자 선택 (Victim Selection)                          │ │
│   │      ┌─────────────────────────────────────────────────────┐ │ │
│   │      │  기준:                                               │ │ │
│   │      │  • 우선순위가 낮은 프로세스                          │ │ │
│   │      │  • 작업 시간이 적게 남은 프로세스                    │ │ │
│   │      │  • 많은 자원을 사용하는 프로세스                     │ │ │
│   │      │  • 롤백 비용이 적은 프로세스                          │ │ │
│   │      │                                                      │ │ │
│   │      │  선택: P2가 희생자로 선정                            │ │ │
│   │      └─────────────────────────────────────────────────────┘ │ │
│   │                                                              │ │
│   │   3. 롤백 (Rollback)                                         │ │
│   │      ┌─────────────────────────────────────────────────────┐ │ │
│   │      │  P2를 이전 상태로 복원:                              │ │ │
│   │      │  • 보유 자원 R2 해제                                  │ │ │
│   │      │  • 체크포인트로 복원                                  │ │ │
│   │      │  • 로그를 사용한 Undo                                 │ │ │
│   │      │  • 프로세스 재시작 또는 종료                          │ │ │
│   │      └─────────────────────────────────────────────────────┘ │ │
│   │                                                              │ │
│   │   4. 대기 중인 프로세스 깨움                                  │ │
│   │      P1이 R2 획득 가능 → 실행 재개                           │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              롤백 기법                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Total Rollback:                                     │   │ │
│   │   │  • 프로세스를 처음부터 재시작                          │   │ │
│   │   │  • 간단하지만 비용이 큼                                │   │ │
│   │   │                                                      │   │ │
│   │   │  Partial Rollback:                                   │   │ │
│   │   │  • 가장 최근 체크포인트로 복원                         │   │ │
│   │   │  • 일부 작업만 다시 실행                               │   │ │
│   │   │  • 로그 기반 복구                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  Mini Rollback:                                      │   │ │
│   │   │  • 데드락에 관련된 연산만 취소                         │   │ │
│   │   │  • 정교하지만 복잡                                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 롤백은 체크포인트와 로그가 필수입니다.

**💡 비유**: 롤백은 게임에서 체크포인트로 돌아가는 것과 같습니다.

**📢 섹션 요약 비유**: 희생자는 우선순위 낮은 것!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Victim** | 희생자 | 선택 알고리즘 | 대타 |
| **Checkpoint** | 저장점 | 상태 스냅샷 | 세이브 |
| **Log** | 로그 | 연산 기록 | 기록 |
| **Undo** | 취소 | 역순 실행 | 되돌리기 |

**희생자 선택 기준**

| 기준 | 설명 |
|:---|:---|
| **우선순위** | 낮은 것 선택 |
| **진행 시간** | 짧은 것 선택 |
| **자원 사용** | 많은 것 선택 |
| **롤백 비용** | 적은 것 선택 |

**핵심 알고리즘**: 탐지 → 희생자 선택 → 자원 해제 → 롤백

**📢 섹션 요약 비유**: 기아 방지를 위해 선택 기록!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 복구 방식**

| 방식 | 복구 시간 | 복잡도 |
|:---|:---:|:---:|
| **프로세스 킬** | 즉시 | 낮음 |
| **전체 롤백** | 김 | 낮음 |
| **부분 롤백** | 중간 | 높음 |

**📢 섹션 요약 비유**: 킬이 가장 간단!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: DBMS** → 트랜잭션 롤백
**시나리오 2: OS** → 프로세스 킬
**시나리오 3: 분산 시스템** → 2PC + 타임아웃

**도입 체크리스트**: 체크포인트 주기, 로그 크기

**📢 섹션 요약 비유**: DB는 ARIES 복구!

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 데드락 자동 복구

**참고 표준**: ARIES, WAL
