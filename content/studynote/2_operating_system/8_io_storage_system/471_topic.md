# 455. SCAN 스케줄링 (Elevator Algorithm)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 헤드가 한 방향으로 끝까지 이동
> 2. **가치**: 기아 현상 해결과 공정성
> 3. **융합**: 디스크 스케줄링, C-SCAN과 연관

---

## Ⅰ. 개요

### 개념 정의

SCAN 스케줄링(Elevator Algorithm)은 **헤드가 한 방향으로 끝까지 이동하며 요청을 처리하고, 끝에 도달하면 방향을 바꾸는 알고리즘**이다.

### 💡 비유: 엘리베이터
SCAN은 **엘리베이터**와 같다. 엘리베이터는 한 방향으로 가면서 승객을 태우고, 끝 층에 도달하면 반대 방향으로 간다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                SCAN 상세                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 기아 현상 없음:                                                  │ │
│  │     • 모든 요청이 결국 처리됨                                          │ │
│  │     • 대기 시간 상한 존재                                              │ │
│  │     • SSTF의 단점 해결                                                │ │
│  │                                                             │ │
│  │  2. 균등한 서비스:                                                   │ │
│  │     • 모든 위치에 공정한 접근                                          │ │
│  │     • 내/외곽 트랙 편향 없음                                           │ │
│  │                                                             │ │
│  │  3. 예측 가능:                                                      │ │
│  │     • 최대 대기 시간 계산 가능                                         │ │
│  │     • 2번의 왕복 이내 처리 보장                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 양 끝단 불리:                                                    │ │
│  │     • 끝 부근 요청은 길게 대기                                         │ │

│  │                                                             │ │

│  │  3. 방향 의존:                                                      │ │
│  │     • 현재 방향에 따라 대기 시간 차이                                    │ │
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SCAN 구현 (의사코드)】                                             │
│  ──────────────────                                                  │
│  void scan_schedule(Queue requests, int head, int direction) {      │
│      // 요청을 트랙 번호로 정렬                                         │
│      sort(requests);                                                 │
│                                                                     │
│      // 현재 방향의 요청 처리                                          │
│      while (hasRequestsInDirection(requests, head, direction)) {    │
│          Request req = getNextInDirection(requests, head, direction);│
│          processRequest(req);                                        │
│          head = req.track;                                           │
│      }                                                                │
│                                                                     │
│      // 끝까지 이동                                                    │
│      if (direction == INCREASING) {                                  │
│          head = MAX_TRACK;  // 199                                   │
│      } else {                                                         │
│          head = 0;                                                   │
│      }                                                                │
│                                                                     │
│      // 방향 전환                                                      │
│      direction = !direction;                                         │
│                                                                     │
│      // 반대 방향 요청 처리                                            │
│      while (!requests.empty()) {                                     │
│          Request req = getNextInDirection(requests, head, direction);│
│          processRequest(req);                                        │
│          head = req.track;                                           │
│      }                                                                │
│  }                                                                   │
│                                                                     │
│  【LOOK 변형】                                                        │
│  ──────────────────                                                  │
│  void look_schedule(Queue requests, int head, int direction) {      │
│      sort(requests);                                                 │
│                                                                     │
│      while (hasRequestsInDirection(requests, head, direction)) {    │
│          Request req = getNextInDirection(requests, head, direction);│
│          processRequest(req);                                        │
│          head = req.track;                                           │
│      }                                                                │
│                                                                     │
│      // 끝까지 안 가고 바로 방향 전환                                    │
│      direction = !direction;                                         │
│                                                                     │
│      while (!requests.empty()) {                                     │
│          Request req = getNextInDirection(requests, head, direction);│
│          processRequest(req);                                        │
│          head = req.track;                                           │
│      }                                                                │
│  }                                                                   │
│                                                                     │
│  【리눅스 CFQ (Completely Fair Queuing)】                             │
│  ──────────────────                                                  │
│  // CFQ는 SCAN과 유사한 동작                                           │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  noop deadline [cfq]                                                 │
│                                                                     │
│  // CFQ 파라미터                                                       │
│  $ cat /sys/block/sda/queue/iosched/quantum                          │
│  4  // 동시 처리 요청 수                                               │
│                                                                     │
│  $ cat /sys/block/sda/queue/iosched/slice_idle                       │
│  8  // idle 시간 (ms)                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 엘리베이터 방식, 한 방향 끝까지
• 장점: 기아 없음, 균등 서비스, 예측 가능
• 단점: 끝단 불리, 불필요한 이동
• 변형: LOOK (끝까지 안 감)
• 개선: C-SCAN (한 방향만)
• 비교: SSTF보다 공정, FCFS보다 효율
• 적합: 다중 사용자, 고하중 환경
```
