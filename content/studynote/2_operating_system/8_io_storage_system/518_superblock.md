# 518. 블로킹/논블로킹 I/O (Blocking/Non-blocking I/O)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 작업 중 스레드 대기 여부
> 2. **가치**: 응답성과 자원 활용의 트레이드오프
> 3. **융합**: 동기, 비동기, 멀티플렉싱과 연관

---

## Ⅰ. 개요

### 개념 정의

- **블로킹 I/O**: I/O 작업이 완료될 때까지 스레드가 대기하는 방식
- **논블로킹 I/O**: I/O 작업이 완료되지 않아도 스레드가 즉시 반환받는 방식

### 💡 비유: 전화 vs 문자
- 블로킹 I/O는 **전화**와 같다. 상대방이 받을 때까지 기다린다.
- 논블로킹 I/O는 **문자 메시지**와 같다. 보내고 다른 일을 한다.

### 블로킹/논블로킹 I/O 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                블로킹 vs 논블로킹 I/O                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【I/O 모델 비교】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 블로킹 I/O (Blocking):                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  스레드    read() ──────▶ 대기 ──────▶ 반환                │    │ │
│  │  │                        (완료까지 멈춤)                      │    │ │
│  │  │                                                          │    │ │
│  │  │  장점: 단순, 직관적                                        │    │ │
│  │  │  단점: 대기 시간 낭비                                      │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  2. 논블로킹 I/O (Non-blocking):                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  스레드    read() ──────▶ 즉시 반환 (EAGAIN)              │    │ │
│  │  │           ↓                                              │    │ │
│  │  │           다른 작업 수행                                   │    │ │
│  │  │           ↓                                              │    │ │
│  │  │           read() 재시도 ──────▶ 성공/실패                  │    │ │
│  │  │                                                          │    │ │
│  │  │  장점: 대기 없이 다른 작업 가능                              │    │ │
│  │  │  단점: 폴링 오버헤드                                       │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  3. I/O 멀티플렉싱 (Multiplexing):                                        │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  스레드    select() ──────▶ 이벤트 대기 ──────▶ 반환        │    │ │
│  │  │                        (여러 FD 동시 감시)                  │    │ │
│  │  │                                                          │    │ │
│  │  │  장점: 여러 I/O를 효율적으로 관리                            │    │ │
│  │  │  단점: 구현 복잡                                          │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동기 vs 비동기, 블로킹 vs 논블로킹】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │              │  블로킹          │  논블로킹                                  │ │
│  │  ────────────┼──────────────────┼──────────────────                          │ │
│  │  동기        │ read(), recv()   │ read() + O_NONBLOCK                        │ │
│  │              │ (전통적)          │ (폴링)                                     │ │
│  │  ────────────┼──────────────────┼──────────────────                          │ │
│  │  비동기      │ (드묾)            │ aio_read(), io_uring                        │ │
│  │              │                   │ (진정한 비동기)                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                블로킹/논블로킹 I/O 상세                                  │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【논블로킹 설정 방법】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. open() 시 플래그:                                                   │ │
│  │     int fd = open("file", O_RDONLY | O_NONBLOCK);                       │ │
│  │                                                             │ │
│  │  2. fcntl()로 변경:                                                      │ │
│  │     int flags = fcntl(fd, F_GETFL, 0);                                  │ │
│  │     fcntl(fd, F_SETFL, flags | O_NONBLOCK);                             │ │
│  │                                                             │ │
│  │  3. 소켓 옵션:                                                           │ │
│  │     int flag = 1;                                                        │ │
│  │     ioctl(sock, FIONBIO, &flag);                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【논블로킹 동작】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  데이터 없을 때:                                                          │ │
│  │  • read() → -1 반환                                                      │ │
│  │  • errno = EAGAIN 또는 EWOULDBLOCK                                       │ │
│  │                                                             │ │
│  │  처리 패턴:                                                               │ │
│  │  while (1) {                                                              │ │
│  │      n = read(fd, buf, size);                                            │ │
│  │      if (n > 0) {                                                         │ │
│  │          // 데이터 처리                                                   │ │
│  │          break;                                                            │ │
│  │      } else if (n == -1) {                                               │ │
│  │          if (errno == EAGAIN) {                                           │ │
│  │              usleep(1000);  // 잠시 대기                                   │ │
│  │              continue;  // 재시도                                          │ │
│  │          } else {                                                         │ │
│  │              perror("read");  // 실제 에러                                 │ │
│  │              break;                                                        │ │
│  │          }                                                                 │ │
│  │      }                                                                     │ │
│  │  }                                                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【사용 시나리오】                                                        │**
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시나리오              블로킹          논블로킹                              │ │
│  │  ────────              ──────          ────────                              │ │
│  │  파일 읽기             적합            보통                                 │ │
│  │  네트워크 I/O          부적합*         적합                                 │ │
│  │  파이프/FIFO          부적합*         적합                                 │ │
│  │  시리얼 포트           부적합*         적합                                 │ │
│  │  터미널 I/O            부적합*         적합                                 │ │
│  │                                                             │ │
│  │  * 단일 연결, 간단한 프로그램에서는 블로킹도 사용 가능                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【블로킹 I/O 예시】                                                      │
│  ──────────────────                                                  │
│  #include <unistd.h>                                                 │
│  #include <fcntl.h>                                                  │
│                                                                     │
│  // 블로킹 읽기 (기본)                                                   │
│  int fd = open("file.txt", O_RDONLY);                                │
│  char buf[1024];                                                     │
│  ssize_t n = read(fd, buf, sizeof(buf));  // 완료까지 대기             │
│  close(fd);                                                          │
│                                                                     │
│  // 블로킹 소켓                                                          │
│  int sock = socket(AF_INET, SOCK_STREAM, 0);                         │
│  connect(sock, &addr, sizeof(addr));  // 연결까지 대기                 │
│  recv(sock, buf, sizeof(buf), 0);     // 데이터까지 대기               │
│  close(sock);                                                        │
│                                                                     │
│  【논블로킹 I/O 예시】                                                    │
│  ──────────────────                                                  │
│  #include <fcntl.h>                                                  │
│  #include <errno.h>                                                  │
│                                                                     │
│  // 논블로킹 소켓 설정                                                   │
│  int sock = socket(AF_INET, SOCK_STREAM, 0);                         │
│  int flags = fcntl(sock, F_GETFL, 0);                                │
│  fcntl(sock, F_SETFL, flags | O_NONBLOCK);                           │
│                                                                     │
│  // 논블로킹 연결                                                        │
│  int ret = connect(sock, &addr, sizeof(addr));                        │
│  if (ret == -1 && errno == EINPROGRESS) {                           │
│      // 연결 진행 중, 나중에 완료 확인 필요                               │
│  }                                                                   │
│                                                                     │
│  // 논블로킹 읽기                                                        │
│  while (1) {                                                          │
│      ssize_t n = recv(sock, buf, sizeof(buf), 0);                    │
│      if (n > 0) {                                                     │
│          // 데이터 수신 완료                                            │
│          process_data(buf, n);                                        │
│          break;                                                        │
│      } else if (n == -1) {                                           │
│          if (errno == EAGAIN || errno == EWOULDBLOCK) {              │
│              // 아직 데이터 없음                                         │
│              do_other_work();                                         │
│              continue;                                                │
│          } else {                                                     │
│              perror("recv");                                          │
│              break;                                                    │
│          }                                                             │
│      } else {  // n == 0                                              │
│          // 연결 종료                                                   │
│          break;                                                        │
│      }                                                                 │
│  }                                                                   │
│                                                                     │
│  【select()로 멀티플렉싱】                                               │
│  ──────────────────                                                  │
│  #include <sys/select.h>                                             │
│                                                                     │
│  fd_set readfds;                                                     │
│  struct timeval timeout;                                             │
│                                                                     │
│  FD_ZERO(&readfds);                                                  │
│  FD_SET(sock, &readfds);                                             │
│                                                                     │
│  timeout.tv_sec = 5;                                                 │
│  timeout.tv_usec = 0;                                                │
│                                                                     │
│  int ret = select(sock + 1, &readfds, NULL, NULL, &timeout);         │
│  if (ret > 0 && FD_ISSET(sock, &readfds)) {                         │
│      // 데이터 수신 준비 완료                                           │
│      recv(sock, buf, sizeof(buf), 0);                                │
│  }                                                                   │
│                                                                     │
│  【epoll로 효율적 멀티플렉싱 (Linux)】                                    │
│  ──────────────────                                                  │
│  #include <sys/epoll.h>                                              │
│                                                                     │
│  int epfd = epoll_create1(0);                                        │
│                                                                     │
│  struct epoll_event ev;                                              │
│  ev.events = EPOLLIN | EPOLLET;  // Edge-triggered                   │
│  ev.data.fd = sock;                                                  │
│  epoll_ctl(epfd, EPOLL_CTL_ADD, sock, &ev);                          │
│                                                                     │
│  #define MAX_EVENTS 10                                                │
│  struct epoll_event events[MAX_EVENTS];                              │
│                                                                     │
│  while (1) {                                                          │
│      int n = epoll_wait(epfd, events, MAX_EVENTS, -1);               │
│      for (int i = 0; i < n; i++) {                                   │
│          if (events[i].events & EPOLLIN) {                            │
│              // 데이터 수신                                             │
│              recv(events[i].data.fd, buf, sizeof(buf), 0);           │
│          }                                                             │
│      }                                                                 │
│  }                                                                   │
│                                                                     │
│  【Python에서 논블로킹 소켓】                                             │
│  ──────────────────                                                  │
│  import socket                                                       │
│  import select                                                        │
│                                                                     │
│  sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)            │
│  sock.setblocking(False)                                            │
│  sock.connect(('example.com', 80))                                   │
│                                                                     │
│  # select로 대기                                                       │
│  ready = select.select([sock], [], [], 5.0)                          │
│  if ready[0]:                                                        │
│      data = sock.recv(1024)                                          │
│                                                                     │
│  【소켓 플래그 확인】                                                    │
│  ──────────────────                                                  │
│  // 현재 플래그 확인                                                     │
│  int flags = fcntl(sock, F_GETFL, 0);                                │
│  if (flags & O_NONBLOCK) {                                           │
│      printf("Non-blocking mode\n");                                   │
│  } else {                                                             │
│      printf("Blocking mode\n");                                       │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 블로킹: 완료까지 스레드 대기
• 논블로킹: 즉시 반환 (EAGAIN)
• 설정: O_NONBLOCK 플래그, fcntl()
• 동기+논블로킹: 폴링 필요
• 비동기+논블로킹: 진정한 비동기
• 블로킹: 단순, 직관적
• 논블로킹: 응답성 높음
• 멀티플렉싱: 여러 I/O 동시 관리
```
