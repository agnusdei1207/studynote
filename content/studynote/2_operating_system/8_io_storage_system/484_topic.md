# 484. 파일 시스템 스냅샷 (File System Snapshot)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 특정 시점의 파일 시스템 상태 보존
> 2. **가치**: 빠른 백업과 즉시 복구
> 3. **융합**: COW, 백업, Btrfs와 연관

---

## Ⅰ. 개요

### 개념 정의

파일 시스템 스냅샷(File System Snapshot)은 **특정 시점의 파일 시스템 상태를 보존하여 빠르게 복구할 수 있는 기능**이다.

### 💡 비유: 사진 촬영
파일 시스템 스냅샷은 **사진 촬영**과 같다. 특정 순간의 모습을 찍어 두면 언제든 그 시점으로 돌아갈 수 있다.

### 파일 시스템 스냅샷 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 스냅샷                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스냅샷 동작 원리 (COW)】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Copy-on-Write (COW):                                                │ │
│  │                                                             │ │
│  │  스냅샷 생성 시:                                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  원본 파일 시스템                                         │    │ │
│  │  │  ┌───┬───┬───┬───┬───┐                                 │    │ │
│  │  │  │ A │ B │ C │ D │ E │                                 │    │ │
│  │  │  └───┴───┴───┴───┴───┘                                 │    │ │
│  │  │          │                                               │    │ │
│  │  │          ▼ 포인터만 복사 (데이터 복사 없음)                  │    │ │
│  │  │  ┌───────────────────────────────────────────────────────┐│    │ │
│  │  │  │ 스냅샷 (거의 즉시 생성, 공간 거의 안 씀)                ││    │ │
│  │  │  └───────────────────────────────────────────────────────┘│    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  데이터 수정 시:                                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  원본                스냅샷                              │    │ │
│  │  │  ┌───┐               ┌───┐                               │    │ │
│  │  │  │ A │               │ A │                               │    │ │
│  │  │  ├───┤               ├───┤                               │    │ │
│  │  │  │ B'│ ← 새 위치     │ B │ ← 원본 유지                    │    │ │
│  │  │  ├───┤               ├───┤                               │    │ │
│  │  │  │ C │               │ C │                               │    │ │
│  │  │  └───┘               └───┘                               │    │ │
│  │  │  B를 수정하면 새 위치에 쓰고, 스냅샷은 원본 B를 유지         │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 스냅샷 상세                                 │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스냅샷 장점】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 빠른 생성:                                                         │ │
│  │     • 메타데이터만 복사                                                  │ │
│  │     • 거의 즉시 완료                                                    │ │
│  │     • 서비스 중단 불필요                                                │ │
│  │                                                             │ │
│  │  2. 공간 효율:                                                        │ │
│  │     • 변경분만 추가 공간 사용                                            │ │
│  │     • 처음에는 거의 공간 안 씀                                           │ │
│  │                                                             │ │
│  │  3. 빠른 복구:                                                        │ │
│  │     • 스냅샷으로 즉시 되돌리기 가능                                      │ │
│  │     • 전체 백업보다 훨씬 빠름                                            │ │
│  │                                                             │ │
│  │  4. 일관성:                                                             │ │
│  │     • 특정 시점의 완전한 상태                                            │ │
│  │     • 크래시 일관성 보장                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스냅샷 지원 파일 시스템】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파일 시스템    스냅샷 지원    방식                                    │ │
│  │  ──────────    ──────────    ────                                    │ │
│  │  Btrfs          네이티브       COW                                     │ │
│  │  ZFS            네이티브       COW                                     │ │
│  │  LVM            볼륨 레벨      COW                                     │ │
│  │  ext4           간접 (LVM)    -                                        │ │
│  │  XFS            간접 (LVM)    -                                        │ │
│  │  NTFS           VSS           COW                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【사용 사례】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  • 업데이트 전 백업                                                    │ │
│  │  • 데이터베이스 백업 (일관성 있는 시점)                                  │ │
│  │  • 가상 머신 백업                                                      │ │
│  │  • 테스트 환경 생성                                                    │ │
│  │  • 랜섬웨어 복구                                                       │ │
│  │  • 실수로 삭제한 파일 복구                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LVM 스냅샷】                                                        │
│  ──────────────────                                                  │
│  // 스냅샷 생성                                                        │
│  $ sudo lvcreate -L 10G -s -n snap01 /dev/vg0/data                  │
│  // -L: 스냅샷 크기, -s: 스냅샷, -n: 이름                              │
│                                                                     │
│  // 스냅샷 확인                                                        │
│  $ sudo lvs                                                           │
│  LV     VG   Attr       LSize   Origin   Data%                       │
│  data   vg0  owi-aos--- 100.00g                                        │
│  snap01 vg0  swi-aos--- 10.00g  data     5.23                        │
│                                                                     │
│  // 스냅샷 마운트                                                      │
│  $ sudo mount /dev/vg0/snap01 /mnt/snapshot                         │
│                                                                     │
│  // 스냅샷에서 복구                                                    │
│  $ sudo cp /mnt/snapshot/important_file /data/                       │
│                                                                     │
│  // 스냅샷으로 되돌리기                                                 │
│  $ sudo lvconvert --merge /dev/vg0/snap01                            │
│                                                                     │
│  // 스냅샷 삭제                                                        │
│  $ sudo lvremove /dev/vg0/snap01                                     │
│                                                                     │
│  【Btrfs 스냅샷】                                                      │
│  ──────────────────                                                  │
│  // 스냅샷 생성                                                        │
│  $ sudo btrfs subvolume snapshot /data /data/snap_$(date +%Y%m%d)    │
│                                                                     │
│  // 읽기 전용 스냅샷                                                    │
│  $ sudo btrfs subvolume snapshot -r /data /data/snap_ro              │
│                                                                     │
│  // 스냅샷 목록                                                        │
│  $ sudo btrfs subvolume list /data                                   │
│  ID 256 gen 123 top level 5 path snap_20240101                       │
│  ID 257 gen 124 top level 5 path snap_20240102                       │
│                                                                     │
│  // 스냅샷으로 되돌리기                                                 │
│  $ sudo btrfs subvolume snapshot /data/snap_20240101 /data_restored  │
│  $ sudo btrfs subvolume set-default /data_restored                   │
│                                                                     │
│  // 스냅샷 삭제                                                        │
│  $ sudo btrfs subvolume delete /data/snap_20240101                   │
│                                                                     │
│  【ZFS 스냅샷】                                                        │
│  ──────────────────                                                  │
│  // 스냅샷 생성                                                        │
│  $ sudo zfs snapshot pool/data@snap_$(date +%Y%m%d)                  │
│                                                                     │
│  // 재귀적 스냅샷                                                      │
│  $ sudo zfs snapshot -r pool@daily                                   │
│                                                                     │
│  // 스냅샷 목록                                                        │
│  $ sudo zfs list -t snapshot                                          │
│  NAME                 USED  AVAIL  REFER  MOUNTPOINT                 │
│  pool/data@snap_20240101  0B   -    100G  -                          │
│                                                                     │
│  // 스냅샷으로 되돌리기                                                 │
│  $ sudo zfs rollback pool/data@snap_20240101                         │
│                                                                     │
│  // 스냅샷 복제                                                        │
│  $ sudo zfs clone pool/data@snap_20240101 pool/data_clone            │
│                                                                     │
│  // 스냅샷 삭제                                                        │
│  $ sudo zfs destroy pool/data@snap_20240101                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 특정 시점 파일 시스템 상태 보존
• 원리: Copy-on-Write (COW)
• 장점: 빠른 생성, 공간 효율, 빠른 복구
• 지원 FS: Btrfs, ZFS, LVM, NTFS VSS
• LVM: lvcreate -s, lvconvert --merge
• Btrfs: btrfs subvolume snapshot
• ZFS: zfs snapshot, zfs rollback
• 용도: 백업, 복구, 테스트, VM
```
