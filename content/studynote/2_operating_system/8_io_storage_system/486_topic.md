# 486. 파일 시스템 압축 (File System Compression)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 데이터를 실시간으로 압축하여 저장
> 2. **가치**: 저장 공간 절약, I/O 감소
> 3. **융합**: 중복 제거, 파일 시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

파일 시스템 압축(File System Compression)은 **파일 데이터를 압축하여 저장하고, 접근 시 자동으로 해제하는 투명한 압축 기술**이다.

### 💡 비유: 진공 압축 백
파일 시스템 압축은 **진공 압축 백**과 같다. 물건을 넣을 때 공기를 빼서 작게 만들고, 꺼낼 때 다시 펼친다.

### 파일 시스템 압축 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 압축                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【투명한 압축】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  애플리케이션                                                         │ │
│  │  write("AAAAABBBBBCCCCC")                                           │ │
│  │          │                                                             │ │
│  │          ▼                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐        │ │
│  │  │           파일 시스템 (자동 압축)                          │        │ │
│  │  │  "AAAAABBBBBCCCCC" → 압축 → "A5B5C5" (예시)              │        │ │
│  │  │  저장 공간: 15 bytes → 6 bytes                            │        │ │
│  │  └─────────────────────────────────────────────────────────┘        │ │
│  │          │                                                             │ │
│  │          ▼                                                             │ │
│  │  디스크: 압축된 데이터만 저장                                          │ │
│  │                                                             │ │
│  │  읽기 시: 자동 해제 → "AAAAABBBBBCCCCC" 반환                          │ │
│  │  애플리케이션은 압축 여부 모름                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【압축 알고리즘 비교】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  알고리즘   압축률    속도      용도                                  │ │
│  │  ────────   ──────    ────      ────                                  │ │
│  │  LZ4        보통      매우 빠름  실시간 압축                           │ │
│  │  ZSTD       높음      빠름      일반적                                 │ │
│  │  GZIP       높음      보통      호환성 좋음                            │ │
│  │  LZMA       매우 높음  느림      최대 압축                             │ │
│  │  ZLE        낮음      매우 빠름  0 데이터                              │ │
│  │                                                             │ │
│  │  트레이드오프: 압축률 ↔ 속도 ↔ CPU 사용량                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 압축 상세                                   │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【성능 영향】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                               │
│  │  • 디스크 공간 절약                                                   │ │
│  │  • 디스크 I/O 감소 (적은 데이터 읽기/쓰기)                             │ │
│  │  • 백업 크기 감소                                                      │ │
│  │  • 네트워크 전송량 감소                                                │ │
│  │                                                             │ │
│  │  단점:                                                               │
│  │  • CPU 사용량 증가                                                    │ │
│  │  • 쓰기 지연 (압축 시간)                                               │ │
│  │  • 읽기 시 일부 지연                                                   │ │
│  │  • 압축 불가 데이터에 오버헤드                                          │ │
│  │                                                             │ │
│  │  SSD에서 이점:                                                        │
│  │  • 쓰기 감소 → 수명 증가                                               │ │
│  │  • 적은 I/O → 성능 향상                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【지원 파일 시스템】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파일 시스템    압축 지원    알고리즘                                │ │
│  │  ──────────    ──────────    ────────                                │ │
│  │  NTFS          네이티브       LZNT1, XPRESS, LZX                    │ │
│  │  Btrfs          네이티브       ZLIB, LZO, ZSTD                       │ │
│  │  ZFS            네이티브       LZJB, ZLE, LZ4, GZIP, ZSTD            │ │
│  │  ReFS           네이티브       LZ4                                    │ │
│  │  F2FS           네이티브       LZ4                                    │ │
│  │  squashfs       네이티브       GZIP, LZMA, XZ, LZO, LZ4, ZSTD        │ │
│  │  ext4           없음          -                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【압축 효율 요인】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  높은 압축률:                                                         │ │
│  │  • 텍스트 파일 (로그, 소스코드)                                        │ │
│  │  • 문서 파일                                                          │ │
│  │  • 데이터베이스                                                        │ │
│  │  • 미사용 공간                                                         │ │
│  │                                                             │ │
│  │  낮은 압축률:                                                         │ │
│  │  • 이미 압축된 파일 (zip, jpg, png, mp4)                              │ │
│  │  • 암호화된 파일                                                       │ │
│  │  • 랜덤 데이터                                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Btrfs 압축 설정】                                                  │
│  ──────────────────                                                  │
│  // 파일 시스템 생성 시 압축 설정                                       │
│  $ sudo mkfs.btrfs -L data /dev/sdb1                                │
│  $ sudo mount -o compress=zstd /dev/sdb1 /mnt/data                  │
│                                                                     │
│  // 기존 파일 시스템에 압축 적용                                        │
│  $ sudo btrfs property set /mnt/data compression zstd               │
│                                                                     │
│  // 알고리즘 선택                                                       │
│  $ sudo mount -o compress=lzo /dev/sdb1 /mnt/data   // 빠름          │
│  $ sudo mount -o compress=zlib /dev/sdb1 /mnt/data  // 높은 압축률   │
│  $ sudo mount -o compress=zstd /dev/sdb1 /mnt/data  // 균형           │
│                                                                     │
│  // 압축 레벨 지정 (ZSTD)                                              │
│  $ sudo mount -o compress=zstd:15 /dev/sdb1 /mnt/data               │
│  // 레벨 1-15, 높을수록 압축률 높고 느림                                 │
│                                                                     │
│  【ZFS 압축 설정】                                                      │
│  ──────────────────                                                  │
│  // 압축 활성화                                                        │
│  $ sudo zfs set compression=lz4 pool/dataset                        │
│  $ sudo zfs set compression=gzip-9 pool/dataset  // 최대 압축         │
│  $ sudo zfs set compression=zstd pool/dataset                       │
│                                                                     │
│  // 압축 비활성화                                                       │
│  $ sudo zfs set compression=off pool/dataset                        │
│                                                                     │
│  // 압축률 확인                                                        │
│  $ sudo zfs get compressratio pool/dataset                         │
│  NAME         PROPERTY       VALUE  SOURCE                           │
│  pool/dataset compressratio  2.15x  -                                │
│  // 2.15x = 원래 크기의 46%로 저장                                      │
│                                                                     │
│  【NTFS 압축 (윈도우)】                                                │
│  ──────────────────                                                  │
│  // 폴더 압축                                                          │
│  > compact /c /s:C:\Data                                            │
│  // /c: 압축, /s: 하위 폴더 포함                                       │
│                                                                     │
│  // 압축 해제                                                          │
│  > compact /u /s:C:\Data                                            │
│                                                                     │
│  // 압축 상태 확인                                                      │
│  > compact C:\Data                                                   │
│                                                                     │
│  // LZX 압축 (Win10 이상, 더 높은 압축률)                               │
│  > compact /c /s:C:\Data /exe:lzx                                  │
│                                                                     │
│  【F2FS 압축】                                                          │
│  ──────────────────                                                  │
│  // LZ4 압축으로 마운트                                                 │
│  $ sudo mount -o compress_algorithm=lz4 /dev/sdb1 /mnt/f2fs        │
│  // 주로 안드로이드에서 사용                                            │
│                                                                     │
│  【압축 전/후 크기 비교】                                              │
│  ──────────────────                                                  │
│  // Btrfs                                                             │
│  $ sudo btrfs filesystem df /mnt/data                               │
│  Data, single: total=50.00GiB, used=35.00GiB                       │
│  // 35GB 사용 중, 압축 없으면 70GB였을 것                              │
│                                                                     │
│  // ZFS                                                               │
│  $ sudo zfs list                                                     │
│  NAME          USED   AVAIL   REFER  MOUNTPOINT                      │
│  pool/dataset  35G    65G     35G    /mnt/data                       │
│  $ sudo zfs get referenced,used pool/dataset                        │
│  // REFER: 논리적 크기, USED: 실제 사용                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 데이터 실시간 압축 저장
• 특징: 투명한 압축 (애플리케이션 모름)
• 알고리즘: LZ4, ZSTD, GZIP, LZMA
• 지원 FS: Btrfs, ZFS, NTFS, F2FS
• 장점: 공간 절약, I/O 감소, SSD 수명 증가
• 단점: CPU 사용량, 쓰기 지연
• 효율: 텍스트 높음, 미디어 낮음
• ZFS: compressratio로 압축률 확인
```
