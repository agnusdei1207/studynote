# 489. 클러스터 파일 시스템 (Cluster File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 서버가 동시에 같은 스토리지 접근
> 2. **가치**: 고가용성과 데이터 일관성
> 3. **융합**: 분산 파일 시스템, SAN과 연관

---

## Ⅰ. 개요

### 개념 정의

클러스터 파일 시스템(Cluster File System)은 **여러 서버가 공유 스토리지에 동시에 접근하여 일관된 파일 시스템을 제공하는 시스템**이다.

### 💡 비유: 공유 사무실
클러스터 파일 시스템은 **공유 사무실**과 같다. 여러 팀원이 같은 파일 캐비닛에 동시에 접근한다.

### 클러스터 파일 시스템 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                클러스터 파일 시스템                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SAN 기반 구조】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  서버 노드들                                                          │ │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐           │ │
│  │  │ Server1 │  │ Server2 │  │ Server3 │  │ Server4 │           │ │
│  │  │  GFS2   │  │  GFS2   │  │  GFS2   │  │  GFS2   │           │ │
│  │  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘           │ │
│  │       │            │            │            │                   │ │
│  │       └────────────┼────────────┼────────────┘                   │ │
│  │                    │            │                                 │ │
│  │                    ▼            ▼                                 │ │
│  │  ═══════════════════════════════════════════════════════════════│ │
│  │  ║                   SAN (Fibre Channel / iSCSI)              ║    │ │
│  │  ═══════════════════════════════════════════════════════════════│ │
│  │                    │            │                                 │ │
│  │                    ▼            ▼                                 │ │
│  │  ┌────────────────────────────────────────────────────────┐      │ │
│  │  │                   공유 스토리지                           │      │ │
│  │  │  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐      │      │ │
│  │  │  │ LUN │ LUN │ LUN │ LUN │ LUN │ LUN │ LUN │ LUN │      │      │ │
│  │  │  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘      │      │ │
│  │  └────────────────────────────────────────────────────────┘      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【분산 vs 클러스터】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  분산 파일 시스템:                                                     │ │
│  │  • 데이터가 여러 노드에 분산                                            │ │
│  │  • 각 노드가 자체 스토리지 보유                                          │ │
│  │  • HDFS, GlusterFS                                                    │ │
│  │                                                             │ │
│  │  클러스터 파일 시스템:                                                 │ │
│  │  • 공유 스토리지를 여러 노드가 동시 접근                                 │ │
│  │  • SAN/NAS 기반                                                       │ │
│  │  • GFS2, OCFS2, GPFS                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                클러스터 파일 시스템 상세                               │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 클러스터 파일 시스템】                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시스템      플랫폼        특징                                          │ │
│  │  ────────    ──────        ────                                          │ │
│  │  GFS2        Linux         Red Hat, 오픈소스                              │ │
│  │  OCFS2       Linux         Oracle, 오픈소스                              │ │
│  │  GPFS        AIX/Linux     IBM, 상용                                     │ │
│  │  VMFS        VMware        ESXi 전용                                     │ │
│  │  CSV         Windows       Windows Server                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【잠금 메커니즘】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  DLM (Distributed Lock Manager):                                      │ │
│  │  • 클러스터 전체 잠금 관리                                              │ │
│  │  • 메타데이터 일관성 유지                                               │ │
│  │  • 교착 상태 탐지                                                      │ │
│  │                                                             │ │
│  │  잠금 유형:                                                             │
│  │  • Shared Lock (읽기)                                                 │ │
│  │  • Exclusive Lock (쓰기)                                               │ │
│  │  • Delegated Lock (위임)                                               │ │
│  │                                                             │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【퀴럼 (Quorum)】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  과반수 이상의 노드가 온라인이어야 파일 시스템 작동                       │ │
│  │                                                             │ │
│  │  예: 5노드 클러스터 → 최소 3노드 필요                                  │ │
│  │                                                             │ │
│  │  스플릿 브레인 방지:                                                     │ │
│  │  • 네트워크 분할 시 양쪽 모두 작동하면 데이터 손상                        │ │
│  │  • 퀴럼이 없는 쪽은 읽기 전용 또는 중단                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【GFS2 설정】                                                          │
│  ──────────────────                                                  │
│  // 클러스터 소프트웨어 설치                                            │
│  $ sudo apt install pacemaker corosync dlm                           │
│                                                                     │
│  // 클러스터 구성                                                       │
│  $ sudo pcs cluster auth node1 node2                                │
│  $ sudo pcs cluster setup mycluster node1 node2                     │
│  $ sudo pcs cluster start --all                                      │
│                                                                     │
│  // DLM 리소스 생성                                                    │
│  $ sudo pcs resource create dlm ocf:pacemaker:controld \            │
│        op monitor interval=30s                                      │
│                                                                     │
│  // GFS2 파일 시스템 생성                                               │
│  $ sudo mkfs.gfs2 -p lock_dlm -t mycluster:gfsvol \                 │
│        -j 2 /dev/sdb1                                                │
│  // -j: 저널 수 (노드 수)                                              │
│                                                                     │
│  // 마운트                                                              │
│  $ sudo mount -t gfs2 /dev/sdb1 /mnt/gfs2                           │
│                                                                     │
│  【OCFS2 설정】                                                         │
│  ──────────────────                                                  │
│  // 설치                                                              │
│  $ sudo apt install ocfs2-tools                                      │
│                                                                     │
│  // 클러스터 설정 (/etc/ocfs2/cluster.conf)                            │
│  cluster:                                                             │
│    node_count = 2                                                    │
│    name = mycluster                                                  │
│  node:                                                               │
│    ip_port = 7777                                                    │
│    ip_address = 10.0.0.1                                             │
│    number = 1                                                        │
│    name = node1                                                      │
│    cluster = mycluster                                               │
│                                                                     │
│  // 파일 시스템 생성                                                    │
│  $ sudo mkfs.ocfs2 -L "ocfs2vol" -N 2 /dev/sdb1                      │
│                                                                     │
│  // 마운트                                                              │
│  $ sudo mount -t ocfs2 /dev/sdb1 /mnt/ocfs2                         │
│                                                                     │
│  【VMFS 설정 (VMware)】                                                │
│  ──────────────────                                                  │
│  // vSphere Client에서                                                 │
│  // 1. 데이터스토어 생성 마법사 실행                                    │
│  // 2. VMFS 버전 선택 (VMFS 6)                                         │
│  // 3. 공유 스토리지 LUN 선택                                           │
│  // 4. 파티션 구성                                                     │
│  // 5. 완료                                                            │
│                                                                     │
│  // ESXi 호스트에서 확인                                                │
│  ~ # vmkfstools -D /vmfs/volumes/datastore                           │
│  ~ # esxcli storage filesystem list                                  │
│                                                                     │
│  【모니터링】                                                            │
│  ──────────────────                                                  │
│  // GFS2 상태                                                          │
│  $ gfs2_tool -C -p lock_dlm /mnt/gfs2                               │
│  $ gfs2_tool journals /mnt/gfs2                                      │
│                                                                     │
│  // 클러스터 상태                                                       │
│  $ pcs status                                                         │
│  $ pcs status resources                                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 여러 서버가 공유 스토리지 동시 접근
• 기반: SAN (Fibre Channel, iSCSI)
• 종류: GFS2, OCFS2, GPFS, VMFS, CSV
• 메커니즘: DLM (분산 잠금 관리자)
• 퀴럼: 과반수 노드 필요 (스플릿 브레인 방지)
• 장점: 고가용성, 데이터 일관성
• 용도: 가상화, 데이터베이스, 고가용성 서버
• 비교: 분산 FS는 데이터 분산, 클러스터 FS는 스토리지 공유
```
