# 476. VFS (Virtual File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다양한 파일 시스템에 대한 통합 인터페이스
> 2. **가치**: 파일 시스템 독립적인 애플리케이션 개발
> 3. **융합**: 파일 시스템, 시스템 콜과 연관

---

## Ⅰ. 개요

### 개념 정의

VFS(Virtual File System)는 **다양한 파일 시스템에 대해 통일된 인터페이스를 제공하는 커널 추상화 계층**이다.

### 💡 비유: 통합 리모컨
VFS는 **통합 리모컨**과 같다. TV, 에어컨, 오디오가 달라도 같은 리모컨으로 조작한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                VFS 상세                                               │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【VFS 주요 객체】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. superblock: 파일 시스템 전체 정보                                 │ │
│  │     • 장치, 블록 크기, 파일 시스템 타입                               │ │
│  │     • 파일 시스템별 구현                                              │ │
│  │                                                             │ │
│  │  2. inode: 파일 메타데이터                                            │ │
│  │     • 권한, 소유자, 크기, 시간                                        │ │
│  │     • 파일 시스템 독립적                                              │ │
│  │                                                             │ │
│  │  3. dentry (directory entry): 디렉토리 엔트리                        │ │
│  │     • 파일명과 inode 연결                                             │ │
│  │     • 경로 탐색 캐시                                                  │ │
│  │                                                             │ │
│  │  4. file: 열린 파일                                                  │ │
│  │     • 파일 디스크립터                                                 │ │
│  │     • 읽기/쓰기 위치                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【VFS 연산】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  super_operations:                                                   │ │
│  │  • alloc_inode, destroy_inode, write_inode                          │ │
│  │  • put_super, sync_fs                                                │ │
│  │                                                             │ │
│  │  inode_operations:                                                   │ │
│  │  • create, lookup, link, unlink                                      │ │
│  │  • mkdir, rmdir, rename                                              │ │
│  │  • permission, getattr, setattr                                      │ │
│  │                                                             │ │
│  │  file_operations:                                                     │ │
│  │  • read, write, seek, mmap                                           │ │
│  │  • open, release, flush, fsync                                       │ │
│  │                                                             │ │
│  │  dentry_operations:                                                  │ │
│  │  • d_revalidate, d_hash, d_compare                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【VFS 지원 파일 시스템】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  로컬: ext4, XFS, Btrfs, NTFS, FAT, etc.                             │ │
│  │  네트워크: NFS, CIFS/SMB, Ceph, GlusterFS                            │ │
│  │  의사: procfs, sysfs, tmpfs, devtmpfs                                │ │
│  │  특수: cgroup, debugfs, tracefs                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【마운트된 파일 시스템 확인】                                        │
│  ──────────────────                                                  │
│  // 모든 마운트 확인                                                   │
│  $ cat /proc/mounts                                                  │
│  /dev/sda1 / ext4 rw,relatime 0 0                                   │
│  proc /proc proc rw,nosuid 0 0                                       │
│  sysfs /sys sysfs rw,nosuid 0 0                                      │
│  /dev/sdb1 /mnt/usb vfat rw 0 0                                      │
│                                                                     │
│  // 파일 시스템 타입 확인                                              │
│  $ df -Th                                                            │
│  Filesystem     Type   Size  Used Avail Use% Mounted on             │
│  /dev/sda1      ext4   50G   20G   28G  42% /                       │
│  proc           proc     0     0     0    - /proc                   │
│  sysfs          sysfs    0     0     0    - /sys                    │
│                                                                     │
│  【지원 파일 시스템 확인】                                            │
│  ──────────────────                                                  │
│  // 커널이 지원하는 파일 시스템                                        │
│  $ cat /proc/filesystems                                            │
│  nodev   sysfs                                                       │
│  nodev   tmpfs                                                       │
│  nodev   bdev                                                        │
│  nodev   proc                                                        │
│          ext3                                                        │
│          ext4                                                        │
│          xfs                                                         │
│          vfat                                                        │
│          ntfs                                                        │
│  nodev   nfs                                                         │
│  nodev   nfs4                                                        │
│                                                                     │
│  【VFS 캐시 정보】                                                    │
│  ──────────────────                                                  │
│  // 슬랩 캐시 확인                                                     │
│  $ cat /proc/slabinfo | grep -E "dentry|inode"                      │
│  dentry            XXXX  YYYY  ...                                   │
│  ext4_inode_cache  XXXX  YYYY  ...                                   │
│                                                                     │
│  // 메모리 사용                                                        │
│  $ cat /proc/meminfo | grep -E "Cached|Buffers|Slab"                │
│  Cached:          XXXXX kB                                           │
│  Buffers:         XXXXX kB                                           │
│  Slab:            XXXXX kB                                           │
│                                                                     │
│  【프로그래밍 예시】                                                    │
│  ──────────────────                                                  │
│  // 파일 시스템 독립적 코드 - VFS가 처리                               │
│  int fd = open("/path/to/file", O_RDONLY);  // VFS → 적절한 FS       │
│  read(fd, buffer, size);   // ext4든 NFS든 동일한 코드                │
│  close(fd);                                                          │
│                                                                     │
│  // stat으로 파일 정보 - VFS가 추상화                                 │
│  struct stat st;                                                      │
│  stat("/path/file", &st);  // 파일시스템 무관하게 동작                 │
│  printf("Size: %ld\n", st.st_size);                                 │
│                                                                     │
│  // 파일 시스템 정보                                                   │
│  struct statvfs sv;                                                  │
│  statvfs("/", &sv);  // VFS를 통해 파일시스템 정보                    │
│  printf("FS type: %lx\n", sv.f_type);                               │
│  printf("Block size: %ld\n", sv.f_bsize);                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 시스템 통합 인터페이스 계층

• 장점: 파일시스템 독립적 개발 가능
• 지원: ext4, XFS, NTFS, NFS, procfs 등
• 확인: /proc/filesystems, /proc/mounts

```
