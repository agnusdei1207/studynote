# 498. 오브젝트 스토리지 (Object Storage)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 + 메타데이터를 객체로 관리
> 2. **가치**: 무한 확장성과 비정형 데이터 최적화
> 3. **융합**: 클라우드, 분산 시스템, API와 연관

---

## Ⅰ. 개요

### 개념 정의

오브젝트 스토리지(Object Storage)는 **데이터와 메타데이터를 객체(Object) 단위로 저장하고 고유 식별자로 관리하는 스토리지 아키텍처**이다.

### 💡 비유: 창고 보관함
오브젝트 스토리지는 **창고 보관함**과 같다. 물건에 번호표를 붙여 보관하고, 번호로 찾는다.

### 오브젝트 스토리지 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                오브젝트 스토리지                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【객체 구조】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                     Object                               │    │ │
│  │  │  ┌─────────────────────────────────────────────────┐      │    │ │
│  │  │  │  Object ID (고유 식별자)                          │      │    │ │
│  │  │  │  예: /bucket/folder/file.txt                     │      │    │ │
│  │  │  └─────────────────────────────────────────────────┘      │    │ │
│  │  │  ┌─────────────────────────────────────────────────┐      │    │ │
│  │  │  │  Data (실제 데이터)                               │      │    │ │
│  │  │  │  • 파일 내용                                      │      │    │ │
│  │  │  │  • 크기 제한 없음                                 │      │    │ │
│  │  │  └─────────────────────────────────────────────────┘      │    │ │
│  │  │  ┌─────────────────────────────────────────────────┐      │    │ │
│  │  │  │  Metadata (메타데이터)                            │      │    │ │
│  │  │  │  • 사용자 정의: content-type, tags               │      │    │ │
│  │  │  │  • 시스템: 생성일, 크기, 해시                     │      │    │ │
│  │  │  └─────────────────────────────────────────────────┘      │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【비교: 블록 vs 파일 vs 오브젝트】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  특성        블록         파일          오브젝트                       │ │
│  │  ────        ────         ────          ────────                       │ │
│  │  단위        블록         파일          객체                          │ │
│  │  접근        LBA          경로          ID/Key                        │ │
│  │  메타데이터   없음         제한적         풍부                          │ │
│  │  계층        없음         디렉토리       평면                          │ │
│  │  확장성      제한         제한          무한                          │ │
│  │  성능        높음         보통          보통                          │ │
│  │  용도        DB, VM       공유 파일      백업, 미디어                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                오브젝트 스토리지 상세                                   │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 오브젝트 스토리지】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  제품              유형          특징                                   │ │
│  │  ────              ────          ────                                   │ │
│  │  Amazon S3         클라우드       가장 널리 사용, 99.999999999% 내구성    │ │
│  │  Azure Blob        클라우드       핫/쿨/아카이브 계층                     │ │
│  │  Google Cloud      클라우드       멀티 리전, Nearline/Coldline           │ │
│  │  IBM COS           클라우드       S3 호환,跨 리전 복제                    │ │
│  │  MinIO             온프레미스     S3 호환, 고성능, 오픈소스               │ │
│  │  Ceph RADOS        온프레미스     통합 스토리지, 분산                     │ │
│  │  OpenStack Swift   온프레미스     대규모, 분산                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【S3 스토리지 클래스】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  클래스              용도                    비용                          │ │
│  │  ──────              ────                    ────                          │ │
│  │  Standard           자주 접근                보통                          │ │
│  │  Intelligent-Tier   접근 패턴 불규칙          자동 조절                      │ │
│  │  Standard-IA        가끔 접근                낮은 저장, 높은 접근            │ │
│  │  One Zone-IA        가끔 접근, 단일 AZ        더 낮은 비용                  │ │
│  │  Glacier Instant    아카이브, 즉시 복구        매우 낮음                     │ │
│  │  Glacier Flexible   아카이브, 1~5시간 복구     최저                          │ │
│  │  Glacier Deep       장기 아카이브, 12시간      최저                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【핵심 기능】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  • 버전 관리 (Versioning): 객체의 모든 버전 보존                         │ │
│  │  • 수명 주기 (Lifecycle): 자동 계층 이동, 삭제                           │ │
│  │  • 복제 (Replication): 리전/계정 간 복제                                │ │
│  │  • 암호화 (Encryption): 서버/클라이언트 측                               │ │
│  │  • 액세스 제어 (ACL/IAM): 세밀한 권한 관리                               │ │
│  │  • 정적 웹 호스팅: S3만으로 웹사이트 운영                                │ │
│  │  • 이벤트 알림: 객체 생성/삭제 시 Lambda 트리거                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【AWS S3 기본 조작】                                                  │
│  ──────────────────                                                  │
│  // 버킷 생성                                                          │
│  $ aws s3api create-bucket --bucket my-bucket --region us-east-1   │
│  $ aws s3 mb s3://my-bucket                                         │
│                                                                     │
│  // 객체 업로드                                                        │
│  $ aws s3api put-object --bucket my-bucket --key file.txt \        │
│        --body file.txt                                              │
│  $ aws s3 cp file.txt s3://my-bucket/                               │
│                                                                     │
│  // 객체 다운로드                                                      │
│  $ aws s3api get-object --bucket my-bucket --key file.txt out.txt  │
│  $ aws s3 cp s3://my-bucket/file.txt .                              │
│                                                                     │
│  // 메타데이터와 함께 업로드                                             │
│  $ aws s3api put-object --bucket my-bucket --key doc.pdf \         │
│        --body doc.pdf --metadata type=report,year=2024              │
│                                                                     │
│  // presigned URL 생성 (임시 접근)                                     │
│  $ aws s3 presign s3://my-bucket/file.txt --expires-in 3600        │
│                                                                     │
│  【버전 관리】                                                          │
│  ──────────────────                                                  │
│  // 버전 관리 활성화                                                    │
│  $ aws s3api put-bucket-versioning --bucket my-bucket \            │
│        --versioning-configuration Status=Enabled                    │
│                                                                     │
│  // 버전 목록 확인                                                      │
│  $ aws s3api list-object-versions --bucket my-bucket               │
│                                                                     │
│  // 특정 버전 다운로드                                                   │
│  $ aws s3api get-object --bucket my-bucket --key file.txt \        │
│        --version-id VERSION_ID out.txt                              │
│                                                                     │
│  【수명 주기 정책】                                                      │
│  ──────────────────                                                  │
│  // 수명 주기 정책 적용                                                  │
│  $ aws s3api put-bucket-lifecycle-configuration \                   │
│        --bucket my-bucket \                                         │
│        --lifecycle-configuration file://lifecycle.json              │
│                                                                     │
│  // lifecycle.json 예시                                               │
│  {                                                                  │
│    "Rules": [{                                                      │
│      "ID": "ArchiveRule",                                           │
│      "Status": "Enabled",                                           │
│      "Filter": { "Prefix": "archive/" },                            │
│      "Transitions": [{                                              │
│        "Days": 30,                                                  │
│        "StorageClass": "STANDARD_IA"                                │
│      }, {                                                           │
│        "Days": 90,                                                  │
│        "StorageClass": "GLACIER"                                    │
│      }],                                                            │
│      "Expiration": { "Days": 365 }                                  │
│    }]                                                               │
│  }                                                                  │
│                                                                     │
│  【정적 웹 호스팅】                                                      │
│  ──────────────────                                                  │
│  // 웹 호스팅 활성화                                                    │
│  $ aws s3api put-bucket-website --bucket my-bucket \               │
│        --website-configuration file://website.json                  │
│                                                                     │
│  // 퍼블릭 액세스 설정                                                   │
│  $ aws s3api put-bucket-policy --bucket my-bucket \                │
│        --policy file://policy.json                                  │
│                                                                     │
│  // 웹사이트 URL                                                       │
│  http://my-bucket.s3-website-us-east-1.amazonaws.com               │
│                                                                     │
│  【MinIO 사용 (온프레미스)】                                            │
│  ──────────────────                                                  │
│  // Docker로 실행                                                     │
│  $ docker run -p 9000:9000 -p 9001:9001 \                          │
│        -v /data:/data \                                             │
│        minio/minio server /data --console-address ":9001"          │
│                                                                     │
│  // mc 클라이언트로 조작                                                  │
│  $ mc alias set local http://localhost:9000 minioadmin minioadmin  │
│  $ mc mb local/mybucket                                             │
│  $ mc cp file.txt local/mybucket/                                   │
│  $ mc ls local/mybucket                                             │
│                                                                     │
│  【대용량 파일 멀티파트 업로드】                                          │
│  ──────────────────                                                  │
│  // 큰 파일은 자동으로 멀티파트 업로드                                     │
│  $ aws s3 cp largefile.iso s3://my-bucket/                          │
│                                                                     │
│  // 수동 멀티파트 업로드                                                  │
│  $ aws s3api create-multipart-upload --bucket my-bucket \          │
│        --key largefile                                              │
│  // UploadId 반환                                                     │
│  $ aws s3api upload-part --bucket my-bucket --key largefile \      │
│        --part-number 1 --upload-id UPLOAD_ID --body part1.bin      │
│  $ aws s3api complete-multipart-upload ...                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 데이터 + 메타데이터를 객체로 관리
• 구조: Object ID + Data + Metadata
• 비교: 블록(고성능), 파일(공유), 객체(확장)
• 주요 서비스: S3, Azure Blob, GCS, MinIO
• 클래스: Standard, IA, Glacier
• 기능: 버전관리, 수명주기, 복제, 암호화
• 접근: REST API (HTTP/HTTPS)
• 용도: 백업, 미디어, 데이터 레이크, 웹호스팅
```
