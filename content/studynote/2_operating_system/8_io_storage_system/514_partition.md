# 495. 파티션 (Partition)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리적 디스크의 논리적 분할
> 2. **가치**: 데이터 격리, 다중 OS, 관리 편의성
> 3. **융합**: 파일 시스템, 부팅, LVM과 연관

---

## Ⅰ. 개요

### 개념 정의

파티션(Partition)은 **물리적 디스크를 논리적으로 나누어 독립적인 저장 영역으로 사용하는 기술**이다.

### 💡 비유: 방 분리
파티션은 **방 분리**와 같다. 하나의 큰 방을 벽으로 나누어 용도별로 사용한다.

### 파티션 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파티션 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MBR 파티션】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                    물리적 디스크                          │    │ │
│  │  │  ┌──────┬──────────────┬──────────────┬──────────────┐    │    │ │
│  │  │  │ MBR  │  파티션 1     │  파티션 2     │  파티션 3     │    │    │ │
│  │  │  │ 512B │              │              │              │    │    │ │
│  │  │  └──────┴──────────────┴──────────────┴──────────────┘    │    │ │
│  │  │     ↑                                                    │    │ │
│  │  │  부트 코드 + 파티션 테이블                                 │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  특징: 최대 4개 주 파티션, 2TB 제한                                     │ │
│  │        확장 파티션으로 논리 파티션 생성 가능                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【GPT 파티션】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                    물리적 디스크                          │    │ │
│  │  │  ┌─────┬─────┬────────────────────────────────┬─────┐    │    │ │
│  │  │  │Protective│GPT  │    파티션들 (최대 128개)      │GPT  │    │    │ │
│  │  │  │  MBR   │Header│                              │Backup│    │    │ │
│  │  │  └─────┴─────┴────────────────────────────────┴─────┘    │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  특징: 최대 128개 파티션, 9.4ZB 지원                                    │ │
│  │        UEFI 부팅 필수, 백업 헤더                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파티션 타입】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  타입            MBR 코드   용도                                          │ │
│  │  ────            ────────   ────                                          │ │
│  │  주(Primary)      0x07      독립 파티션                                    │ │
│  │  확장(Extended)   0x0F      논리 파티션 컨테이너                             │ │
│  │  논리(Logical)    -         확장 내부 파티션                                │ │
│  │  EFI 시스템       0xEF      UEFI 부팅                                     │ │
│  │  Linux            0x83      Linux 파일시스템                               │ │
│  │  Swap             0x82      스왑 파티션                                    │ │
│  │  LVM              0x8E      LVM 물리 볼륨                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파티션 상세                                             │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MBR vs GPT 비교】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  항목           MBR                    GPT                             │ │
│  │  ────           ────                    ────                             │ │
│  │  최대 용량       2TB                    9.4 ZB                          │ │
│  │  파티션 수       4개 (주) + 논리          128개 (기본)                     │ │
│  │  부팅 방식       BIOS                   UEFI                            │ │
│  │  데이터 보호     없음                    백업 헤더                        │ │
│  │  호환성          높음                    최신 시스템                      │ │
│  │  보안            없음                    CRC 체크                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【일반적인 파티션 구성】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파티션      마운트    크기          용도                                 │ │
│  │  ────────    ────     ────          ────                                 │ │
│  │  /boot/efi   없음     512MB        EFI 부팅                              │ │
│  │  /boot       /boot   1GB          부트로더                               │ │
│  │  /           /       50-100GB     루트 파일시스템                        │ │
│  │  swap        없음     RAM 1-2배    스왑                                  │ │
│  │  /home       /home   나머지        사용자 데이터                         │ │
│  │  /var        /var    20-50GB      로그, 가변 데이터                      │ │
│  │  /tmp        /tmp    10GB         임시 파일                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파티션 장단점】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                                 │ │
│  │  • 데이터 격리 (OS, 데이터 분리)                                        │ │
│  │  • 다중 OS 설치 가능                                                   │ │
│  │  • 백업/복구 용이                                                       │ │
│  │  • 성능 최적화 (별마운트 옵션)                                           │ │
│  │                                                             │ │
│  │  단점:                                                                 │ │
│  │  • 크기 변경 어려움                                                     │ │
│  │  • 공간 낭비 가능                                                       │ │
│  │  • 관리 복잡도 증가                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파티션 확인】                                                        │
│  ──────────────────                                                  │
│  // 파티션 테이블 확인                                                   │
│  $ sudo fdisk -l /dev/sda                                           │
│  Device     Boot   Start    End    Sectors  Size  Type               │
│  /dev/sda1  *      2048    1048575 1046528  511M  EFI System         │
│  /dev/sda2        1048576  20971519 19922944  9.5G  Linux filesystem │
│  /dev/sda3        20971520 41943039 20971520   10G   Linux swap      │
│                                                                     │
│  // parted로 확인                                                      │
│  $ sudo parted /dev/sda print                                       │
│  Number  Start   End     Size    Type      File system  Flags       │
│  1       1049kB  538MB   537MB   primary   fat32        boot, esp   │
│  2       538MB   10.7GB  10.2GB  primary   ext4                     │
│  3       10.7GB  21.5GB  10.7GB  primary   linux-swap(v1)           │
│                                                                     │
│  // lsblk로 확인                                                       │
│  $ lsblk -f                                                          │
│  NAME   FSTYPE   SIZE MOUNTPOINT                                    │
│  sda             100G                                               │
│  ├─sda1 vfat    512M /boot/efi                                      │
│  ├─sda2 ext4    9.5G /                                              │
│  └─sda3 swap    10G  [SWAP]                                         │
│                                                                     │
│  【GPT 파티션 생성 (fdisk)】                                           │
│  ──────────────────                                                  │
│  $ sudo fdisk /dev/sdb                                              │
│  Command (m for help): g     // GPT 라벨 생성                         │
│  Command (m for help): n     // 새 파티션                             │
│  Partition number (1-128): 1                                        │
│  First sector: <Enter>                                              │
│  Last sector: +50G                                                  │
│  Command (m for help): t     // 타입 변경                             │
│  Hex code: 20   // Linux filesystem                                 │
│  Command (m for help): w     // 저장                                 │
│                                                                     │
│  【parted로 파티션 생성】                                              │
│  ──────────────────                                                  │
│  // GPT 파티션 테이블 생성                                              │
│  $ sudo parted /dev/sdb mklabel gpt                                 │
│                                                                     │
│  // EFI 파티션 생성                                                    │
│  $ sudo parted /dev/sdb mkpart ESP fat32 1MiB 513MiB                │
│  $ sudo parted /dev/sdb set 1 esp on                                │
│                                                                     │
│  // 루트 파티션 생성                                                    │
│  $ sudo parted /dev/sdb mkpart primary ext4 513MiB 50GiB            │
│                                                                     │
│  // 스왑 파티션 생성                                                    │
│  $ sudo parted /dev/sdb mkpart primary linux-swap 50GiB 66GiB       │
│                                                                     │
│  // 확인                                                               │
│  $ sudo parted /dev/sdb print                                       │
│                                                                     │
│  【파티션 타입 변경】                                                   │
│  ──────────────────                                                  │
│  // fdisk로 타입 변경                                                  │
│  $ sudo fdisk /dev/sdb                                              │
│  Command: t                                                          │
│  Partition number: 1                                                │
│  Hex code (L to list codes): L                                      │
│  Hex code: 82   // Linux swap                                       │
│                                                                     │
│  // sgdisk로 타입 변경                                                 │
│  $ sudo sgdisk -t 1:8200 /dev/sdb   // 파티션 1을 스왑으로            │
│                                                                     │
│  【파티션 삭제】                                                        │
│  ──────────────────                                                  │
│  // fdisk로 삭제                                                       │
│  $ sudo fdisk /dev/sdb                                              │
│  Command: d                                                          │
│  Partition number: 3                                                │
│  Command: w                                                          │
│                                                                     │
│  // parted로 삭제                                                      │
│  $ sudo parted /dev/sdb rm 3                                        │
│                                                                     │
│  【파티션 크기 조절】                                                   │
│  ──────────────────                                                  │
│  // 삭제 후 재생성 (데이터 손실)                                        │
│  // 또는 LVM 사용 권장                                                  │
│                                                                     │
│  // growpart로 확장                                                    │
│  $ sudo growpart /dev/sda 2   // 파티션 2 확장                        │
│  $ sudo resize2fs /dev/sda2   // 파일시스템 확장                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 물리적 디스크의 논리적 분할
• MBR: 최대 4개 주 파티션, 2TB 제한
• GPT: 최대 128개, 9.4ZB, UEFI
• 도구: fdisk, parted, gdisk, sgdisk
• 구성: /boot, /, swap, /home, /var
• 장점: 격리, 다중 OS, 백업 용이
• 단점: 크기 변경 어려움
• 대안: LVM으로 유연성 확보
```
