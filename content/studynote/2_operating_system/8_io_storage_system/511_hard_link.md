# 511. 예외 처리 (Exception Handling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 명령어 실행 중 오류 발생 시 처리
> 2. **가치**: 시스템 안정성과 오류 복구
> 3. **융합**: 인터럽트, 시그널, 메모리 관리와 연관

---

## Ⅰ. 개요

### 개념 정의

예외 처리(Exception Handling)는 **CPU가 명령어를 실행하는 도중 발생하는 오류나 특수 상황을 처리하는 메커니즘**이다.

### 💡 비유: 응급 상황 대처
예외 처리는 **응급 상황 대처**와 같다. 정상 업무 중 갑작스런 사고가 발생하면 즉시 대응한다.

### 예외 처리 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                예외 처리 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【예외 발생 과정】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │       CPU                                                             │ │
│  │        │                                                              │ │
│  │        ▼                                                              │ │
│  │  ┌─────────────────────────────────────────────────────────┐           │ │
│  │  │  명령어 실행                                              │           │ │
│  │  │  mov eax, [ebx]                                          │           │ │
│  │  └─────────────────────────────────────────────────────────┘           │ │
│  │        │                                                              │ │
│  │        │ ← 예외 발생 (예: 페이지 폴트)                                   │ │
│  │        ▼                                                              │ │
│  │  ┌─────────────────────────────────────────────────────────┐           │ │
│  │  │  1. 현재 상태 저장 (PC, PSR, 레지스터)                     │           │ │
│  │  └─────────────────────────────────────────────────────────┘           │ │
│  │        │                                                              │ │
│  │        ▼                                                              │ │
│  │  ┌─────────────────────────────────────────────────────────┐           │ │
│  │  │  2. 예외 벡터로 점프                                       │           │ │
│  │  └─────────────────────────────────────────────────────────┘           │ │
│  │        │                                                              │ │
│  │        ▼                                                              │ │
│  │  ┌─────────────────────────────────────────────────────────┐           │ │
│  │  │  3. 예외 핸들러 실행                                       │           │ │
│  │  │     • 오류 원인 파악                                        │           │ │
│  │  │     • 복구 가능 여부 판단                                   │           │ │
│  │  │     • 복구 또는 프로세스 종료                               │           │ │
│  │  └─────────────────────────────────────────────────────────┘           │ │
│  │        │                                                              │ │
│  │        ▼                                                              │ │
│  │  ┌─────────────────────────────────────────────────────────┐           │ │
│  │  │  4. 복귀 또는 종료                                         │           │ │
│  │  └─────────────────────────────────────────────────────────┘           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예외 vs 인터럽트】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  구분        예외(Exception)      인터럽트(Interrupt)                    │ │
│  │  ────        ────────             ────────                              │ │
│  │  원인        명령어 실행 중 오류    외부 장치 이벤트                       │ │
│  │  발생 시점    동기                  비동기                                │ │
│  │  처리        현재 스레드            인터럽트 컨텍스트                      │ │
│  │  복귀        오류 후 명령어          다음 명령어                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                예외 처리 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86 예외 종류】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  번호   이름              원인                    복구 가능                   │ │
│  │  ────   ────              ────                    ────────                   │ │
│  │  0      #DE               0으로 나누기             아니오                      │ │
│  │  1      #DB               디버그                   예                          │ │
│  │  3      #BP               브레이크포인트            예                          │ │
│  │  6      #UD               잘못된 명령어            아니오                      │ │
│  │  7      #NM               FPU 없음                예                          │ │
│  │  10     #TS               잘못된 TSS               아니오                      │ │
│  │  11     #NP               세그먼트 없음            아니오                      │ │
│  │  12     #SS               스택 폴트                아니오                      │ │
│  │  13     #GP               일반 보호 오류           아니오                      │ │
│  │  14     #PF               페이지 폴트             예                          │ │
│  │  16     #MF               FPU 에러                예                          │ │
│  │  17     #AC               정렬 검사               예                          │ │
│  │  18     #MC               머신 체크               아니오                      │ │
│  │  19     #XF               SIMD 예외               예                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【페이지 폴트 상세】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  원인 코드 (error_code):                                                │ │
│  │  • Bit 0 (P): 0=페이지 없음, 1=보호 위반                                  │ │
│  │  • Bit 1 (W/R): 0=읽기, 1=쓰기                                          │ │
│  │  • Bit 2 (U/S): 0=커널, 1=사용자                                        │ │
│  │  • Bit 3 (RSVD): 예약 비트 위반                                          │ │
│  │  • Bit 4 (I/D): 명령 실행                                               │ │
│  │                                                             │ │
│  │  처리 방법:                                                              │ │
│  │  1. CR2 레지스터에서 폴트 주소 확인                                       │ │
│  │  2. VMA에서 해당 영역 확인                                                │ │
│  │  3. 페이지 할당 및 매핑                                                   │ │
│  │  4. 또는 SIGSEGV 전송                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux 시그널 변환】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  예외              시그널                 기본 동작                        │ │
│  │  ────              ────                 ────────                        │ │
│  │  #DE               SIGFPE               코어 덤프                      │ │
│  │  #GP               SIGSEGV              코어 덤프                      │ │
│  │  #PF (읽기)         SIGSEGV              코어 덤프                      │ │
│  │  #PF (쓰기)         SIGSEGV              코어 덤프                      │ │
│  │  #BP               SIGTRAP              코어 덤프                      │ │
│  │  #AC               SIGBUS               코어 덤프                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【예외 통계 확인】                                                      │
│  ──────────────────                                                  │
│  // 페이지 폴트 통계                                                     │
│  $ cat /proc/vmstat | grep -i fault                                 │
│  pgfault 12345678                                                    │
│  pgmajfault 1234                                                     │
│                                                                     │
│  // 프로세스별 페이지 폴트                                                │
│  $ ps -o pid,comm,min_flt,maj_flt -p <pid>                          │
│                                                                     │
│  // perf로 예외 추적                                                    │
│  $ perf record -e exceptions:* -a sleep 5                           │
│  $ perf report                                                      │
│                                                                     │
│  【예외 핸들러 (커널)】                                                   │
│  ──────────────────                                                  │
│  // 페이지 폴트 핸들러                                                     │
│  void do_page_fault(struct pt_regs *regs, unsigned long error_code) {│
│      unsigned long address = read_cr2();                            │
│      struct task_struct *tsk = current;                             │
│      struct vm_area_struct *vma;                                    │
│                                                                     │
│      // VMA 찾기                                                       │
│      vma = find_vma(mm, address);                                   │
│      if (!vma || address < vma->vm_start)                           │
│          goto bad_area;                                             │
│                                                                     │
│      // 페이지 핸들링                                                    │
│      handle_mm_fault(mm, vma, address, flags);                      │
│      return;                                                        │
│                                                                     │
│  bad_area:                                                          │
│      // 시그널 전송                                                     │
│      force_sig_info(SIGSEGV, &info, tsk);                           │
│  }                                                                   │
│                                                                     │
│  // 일반 보호 폴트 핸들러                                                  │
│  void do_general_protection(struct pt_regs *regs) {                  │
│      // 커널 모드면 패닉                                                  │
│      if (user_mode(regs))                                           │
│          force_sig(SIGSEGV);                                        │
│      else                                                           │
│          die("general protection fault", regs, 0);                  │
│  }                                                                   │
│                                                                     │
│  【사용자 공간 예외 처리】                                                │
│  ──────────────────                                                  │
│  // 시그널 핸들러 등록                                                     │
│  #include <signal.h>                                                 │
│                                                                     │
│  void segfault_handler(int sig, siginfo_t *info, void *ctx) {        │
│      printf("Segmentation fault at %p\n", info->si_addr);            │
│      exit(1);                                                        │
│  }                                                                   │
│                                                                     │
│  int main() {                                                        │
│      struct sigaction sa;                                            │
│      sa.sa_sigaction = segfault_handler;                            │
│      sa.sa_flags = SA_SIGINFO;                                       │
│      sigaction(SIGSEGV, &sa, NULL);                                 │
│                                                                     │
│      // ...                                                           │
│  }                                                                   │
│                                                                     │
│  【페이지 폴트 디버깅】                                                   │
│  ──────────────────                                                  │
│  // 코어 덤프 분석                                                        │
│  $ gdb ./program core                                               │
│  (gdb) bt                                                           │
│  (gdb) info registers                                               │
│  (gdb) x/i $pc                                                      │
│                                                                     │
│  // dmesg로 커널 로그 확인                                               │
│  $ dmesg | grep -i "segfault\|fault"                                │
│  segfault at 0 ip 0x... sp 0x... error 4                            │
│                                                                     │
│  // ftrace로 예외 추적                                                  │
│  $ trace-cmd record -e exceptions:page_fault_* -p <pid>             │
│  $ trace-cmd report                                                  │
│                                                                     │
│  【0으로 나누기 예외】                                                    │
│  ──────────────────                                                  │
│  // C 언어에서 0으로 나누기                                                │
│  #include <stdio.h>                                                  │
│  #include <signal.h>                                                 │
│                                                                     │
│  void fpe_handler(int sig) {                                         │
│      printf("Floating point exception caught!\n");                  │
│      exit(1);                                                        │
│  }                                                                   │
│                                                                     │
│  int main() {                                                        │
│      signal(SIGFPE, fpe_handler);                                   │
│      int x = 1 / 0;  // #DE 예외 발생                                 │
│      return 0;                                                       │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 명령어 실행 중 오류/특수 상황 처리
• 비교: 예외(동기) vs 인터럽트(비동기)
• 종류: #DE, #GP, #PF, #BP, #AC 등
• 페이지 폴트: 메모리 할당/접근 오류
• 처리: 상태 저장 → 핸들러 → 복귀/종료
• 시그널: SIGSEGV, SIGFPE, SIGBUS
• 확인: /proc/vmstat, dmesg, perf
• 디버깅: gdb 코어 덤프 분석
```
