# 499. 파일 시스템 보안 (File System Security)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일과 데이터의 무결성, 기밀성, 가용성 보호
> 2. **가치**: 데이터 유출 방지와 무단 접근 차단
> 3. **융합**: 권한, 암호화, 감사와 연관

---

## Ⅰ. 개요

### 개념 정의

파일 시스템 보안(File System Security)은 **파일 시스템의 데이터 무결성, 기밀성, 가용성을 보호하는 기술과 정책**이다.

### 💡 비유: 은행 금고실
파일 시스템 보안은 **금고실**과 같다. 여러 겹의 잠금장치와 감시 시스템으로 보호한다.

### 파일 시스템 보안 구조

```

├─────────────────────────────────────────────────────────────────────┤

│  ──────────────────                                                  │

│  │                          ▼                                       │ │

│  │  │  파일 시스템 보안                                         │    │ │
│  │  │  • 접근 제어 (DAC/MAC/ACL)                                │    │ │
│  │  │  • 파일 시스템 암호화 (eCryptfs, fscrypt)                  │    │ │
│  │  │  • immutability (변경 불가 속성)                          │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │

│  │                          ▼                                       │ │

│  │  │  네트워크 보안                                            │    │ │
│  │  │  • 전송 중 암호화 (SFTP, HTTPS)                           │    │ │
│  │  │  • 방화벽, 접근 제어                                      │    │ │

│  │  │  감사 및 모니터링                                         │    │ │

│  │  │  • 파일 무결성 검사 (AIDE, Tripwire)                       │    │ │

│  │  DAC        소유자가 권한 설정          Linux 권한, Windows ACL        │ │
│  │  MAC        정책 기반 강제 접근 제어     SELinux, AppArmor             │ │

│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 보안 상세                                    │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【접근 제어 메커니즘】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 기본 권한 (rwx):                                                    │ │
│  │     • 읽기(r): 파일 내용 보기, 디렉토리 목록                              │ │
│  │     • 쓰기(w): 파일 수정, 디렉토리 내 생성/삭제                           │ │
│  │     • 실행(x): 파일 실행, 디렉토리 진입                                   │ │
│  │                                                             │ │
│  │  2. 특수 권한:                                                         │ │
│  │     • SUID (4000): 실행 시 소유자 권한                                   │ │
│  │     • SGID (2000): 실행 시 그룹 권한                                    │ │
│  │     • Sticky bit (1000): 디렉토리에서 소유자만 삭제                       │ │
│  │                                                             │ │
│  │  3. ACL (Access Control List):                                         │ │
│  │     • 사용자/그룹별 세밀한 권한                                          │ │
│  │     • 기본 권한 이상의 제어                                              │ │
│  │                                                             │ │
│  │  4. SELinux/AppArmor:                                                  │ │
│  │     • 프로세스-파일 접근 정책                                            │ │
│  │     • 강제 접근 제어 (MAC)                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파일 속성 보안】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  속성        의미                    사용                             │ │
│  │  ────        ────                    ────                             │ │
│  │  immutable   변경/삭제 불가            로그, 설정 파일                   │ │
│  │  append-only 추가만 가능               로그 파일                        │ │
│  │  noatime     접근 시간 갱신 안 함        성능                           │ │
│  │  sync        동기 쓰기                  중요 데이터                      │ │
│  │  hidden      숨김                       민감 정보                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【보안 위협과 대응】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  위협                대응                                                  │ │
│  │  ────                ────                                                  │ │
│  │  무단 접근            권한 관리, ACL, MAC                                 │ │
│  │  데이터 유출          암호화, DLP                                         │ │
│  │  무결성 손상          체크섬, AIDE, Tripwire                              │ │
│  │  랜섬웨어            백업, 스냅샷, immutability                           │ │
│  │  권한 상승            SUID 점검, SELinux                                  │ │
│  │  숨김 파일            정기 스캔, 감사                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【권한 관리】                                                          │
│  ──────────────────                                                  │
│  // 기본 권한 변경                                                      │
│  $ chmod 750 /data/secret        // rwxr-x---                        │
│  $ chmod u+x script.sh           // 소유자 실행 권한 추가               │
│  $ chmod go-rwx secret/          // 그룹/타인 권한 제거                 │
│                                                                     │
│  // 소유권 변경                                                        │
│  $ chown user:group /data/file                                      │
│  $ chown -R user:group /data/    // 재귀적 변경                       │
│                                                                     │
│  // 특수 권한 설정                                                      │
│  $ chmod u+s /usr/bin/passwd     // SUID                             │
│  $ chmod g+s /shared/            // SGID                             │
│  $ chmod +t /tmp/                // Sticky bit                       │
│                                                                     │
│  【ACL 설정】                                                           │
│  ──────────────────                                                  │
│  // ACL 확인                                                           │
│  $ getfacl /data/file                                               │
│                                                                     │
│  // 사용자 권한 추가                                                    │
│  $ setfacl -m u:alice:rw /data/file                                 │
│                                                                     │
│  // 그룹 권한 추가                                                      │
│  $ setfacl -m g:developers:rwx /data/project                        │
│                                                                     │
│  // 기본 ACL 설정 (새 파일에 상속)                                       │
│  $ setfacl -d -m g:team:rwx /data/shared                            │
│                                                                     │
│  // ACL 제거                                                          │
│  $ setfacl -x u:alice /data/file                                    │
│  $ setfacl -b /data/file         // 모든 ACL 제거                     │
│                                                                     │
│  【파일 속성 설정】                                                      │
│  ──────────────────                                                  │
│  // immutable 설정 (root만 변경 가능)                                   │
│  $ sudo chattr +i /etc/resolv.conf                                  │
│  $ lsattr /etc/resolv.conf                                          │
│  ----i---------e------- /etc/resolv.conf                            │
│                                                                     │
│  // append-only 설정                                                   │
│  $ sudo chattr +a /var/log/secure.log                               │
│                                                                     │
│  // 속성 해제                                                          │
│  $ sudo chattr -i /etc/resolv.conf                                  │
│                                                                     │
│  【SELinux 관리】                                                       │
│  ──────────────────                                                  │
│  // SELinux 상태 확인                                                   │
│  $ getenforce                                                       │
│  $ sestatus                                                         │
│                                                                     │
│  // 컨텍스트 확인                                                       │
│  $ ls -Z /var/www/html                                              │
│  $ ps -Z | grep httpd                                               │
│                                                                     │
│  // 컨텍스트 변경                                                       │
│  $ chcon -t httpd_sys_content_t /var/www/html/file.html             │
│  $ restorecon -Rv /var/www/html                                     │
│                                                                     │
│  // 부울 설정                                                          │
│  $ getsebool -a | grep http                                         │
│  $ setsebool httpd_can_network_connect on                           │
│                                                                     │
│  【파일 무결성 검사】                                                    │
│  ──────────────────                                                  │
│  // AIDE 설치 및 초기화                                                 │
│  $ sudo apt install aide                                            │
│  $ sudo aideinit                                                    │
│  $ sudo cp /var/lib/aide/aide.db.new /var/lib/aide/aide.db          │
│                                                                     │
│  // 무결성 검사                                                        │
│  $ sudo aide --check                                                │
│                                                                     │
│  // 데이터베이스 업데이트                                                 │
│  $ sudo aide --update                                               │
│                                                                     │
│  【감사 로깅 (auditd)】                                                 │
│  ──────────────────                                                  │
│  // 파일 접근 감사 규칙 추가                                             │
│  $ auditctl -w /etc/passwd -p wa -k passwd_changes                  │
│                                                                     │
│  // 감사 로그 확인                                                      │
│  $ ausearch -k passwd_changes                                       │
│  $ aureport --file                                                  │
│                                                                     │
│  // 규칙 영구 저장 (/etc/audit/rules.d/)                               │
│  -w /etc/passwd -p wa -k passwd_changes                             │
│  -w /etc/shadow -p wa -k shadow_changes                             │
│                                                                     │
│  【SUID/SGID 점검】                                                     │
│  ──────────────────                                                  │
│  // SUID 파일 찾기                                                      │
│  $ find / -perm -4000 -type f 2>/dev/null                           │
│                                                                     │
│  // SGID 파일 찾기                                                      │
│  $ find / -perm -2000 -type f 2>/dev/null                           │
│                                                                     │
│  // world-writable 파일 찾기                                           │
│  $ find / -perm -0002 -type f 2>/dev/null                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일의 무결성, 기밀성, 가용성 보호
• 계층: 물리 → 파일시스템 → 네트워크 → 감사
• 권한: rwx, SUID, SGID, sticky bit
• ACL: 세밀한 사용자/그룹 권한
• MAC: SELinux, AppArmor
• 속성: immutable, append-only
• 감사: auditd, AIDE, Tripwire
• 위협: 무단접근, 유출, 랜섬웨어
```
