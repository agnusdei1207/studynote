# 492. 파일 시스템 성능 튜닝 (File System Performance Tuning)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 시스템 성능 최적화
> 2. **가치**: I/O 처리량 향상과 지연 감소
> 3. **융합**: 캐싱, 스케줄링, 레이아웃과 연관

---

## Ⅰ. 개요

### 개념 정의

파일 시스템 성능 튜닝(File System Performance Tuning)은 **파일 시스템의 설정과 구성을 최적화하여 I/O 성능을 향상시키는 기술**이다.

### 💡 비유: 자동차 튜닝
파일 시스템 성능 튜닝은 **자동차 튜닝**과 같다. 기본 설정을 용도에 맞게 조정한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 성능 튜닝 상세                               │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【마운트 옵션 튜닝】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  옵션            설명                    용도                           │ │
│  │  ────            ────                    ────                           │ │
│  │  noatime         접근 시간 갱신 안 함      모든 경우 (권장)               │ │
│  │  nodiratime      디렉토리 접근 시간 안 함   noatime와 함께                │ │
│  │  relatime        접근 시간 조건부 갱신      호환성 필요 시                │ │
│  │  data=writeback 저널링 느슨하게           속도 우선                      │ │
│  │  data=ordered   저널링 기본               균형                          │ │
│  │  data=journal   저널링 엄격               안전 우선                      │ │
│  │  barrier=0      쓰기 배리어 끔            속도↑ (위험)                   │ │
│  │  discard        TRIM 활성화              SSD                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/O 스케줄러 선택】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  스케줄러      특징                    적합한 용도                       │ │
│  │  ────────      ────                    ────────                       │ │
│  │  noop         순서 유지, 단순           SSD, VM                       │ │
│  │  deadline     읽기 우선, 지연 보장       데이터베이스, iSCSI             │ │
│  │  cfq          공정 분배                  데스크탑, 일반                 │ │
│  │  bfq          저지연, 대역폭 보장        데스크탑, 미디어               │ │
│  │  mq-deadline  멀티큐                    NVMe, 고속 SSD                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【워크로드별 최적화】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  워크로드       특성            최적화                            │ │
│  │  ────────       ────            ──────                            │ │
│  │  데이터베이스   랜덤 I/O, 동기   deadline, 큰 블록, 저널링 신중      │ │
│  │  웹서버         읽기 많음        readahead, 캐시                   │ │
│  │  미디어 스트림  순차 읽기        큰 readahead, noatime             │ │
│  │  로그 서버      순차 쓰기        쓰기 버퍼, async                  │ │
│  │  VM 스토리지    랜덤 I/O         noop, discard                    │ │
│  │  빅데이터       순차 대용량      큰 블록, 스트라이프                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【마운트 옵션 적용】                                                  │
│  ──────────────────                                                  │
│  // /etc/fstab 수정                                                   │
│  /dev/sdb1  /data  ext4  defaults,noatime,nodiratime  0 0           │
│                                                                     │
│  // 실시간 적용                                                        │
│  $ sudo mount -o remount,noatime /data                              │
│                                                                     │
│  // SSD용 마운트 옵션                                                  │
│  /dev/nvme0n1 /ssd ext4 defaults,noatime,discard 0 0                │
│                                                                     │
│  【I/O 스케줄러 변경】                                                 │
│  ──────────────────                                                  │
│  // 현재 스케줄러 확인                                                  │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  [cfq] deadline noop                                                │
│                                                                     │
│  // 스케줄러 변경                                                      │
│  $ echo deadline > /sys/block/sda/queue/scheduler                   │
│                                                                     │
│  // 영구 적용 (udev 규칙)                                              │
│  $ cat > /etc/udev/rules.d/60-scheduler.rules << 'EOF'              │
│  ACTION=="add|change", KERNEL=="sd[a-z]", ATTR{queue/scheduler}="deadline"│
│  ACTION=="add|change", KERNEL=="nvme[0-9]", ATTR{queue/scheduler}="none"│
│  EOF                                                                 │
│                                                                     │
│  【Read-ahead 조정】                                                  │
│  ──────────────────                                                  │
│  // 현재 값 확인                                                        │
│  $ blockdev --getra /dev/sda                                        │
│  256   // 256 * 512 = 128KB                                         │
│                                                                     │
│  // 값 변경                                                           │
│  $ sudo blockdev --setra 4096 /dev/sda   // 2MB                     │
│                                                                     │
│  // 순차 읽기 많은 경우                                                 │
│  $ sudo blockdev --setra 8192 /dev/sda   // 4MB                     │
│                                                                     │
│  【저널링 모드 변경】                                                  │
│  ──────────────────                                                  │
│  // ext4 저널링 모드 확인                                               │
│  $ tune2fs -l /dev/sdb1 | grep "Filesystem features"               │
│                                                                     │
│  // 저널링 모드 변경 (마운트 옵션)                                       │
│  $ sudo mount -o remount,data=writeback /data                       │
│                                                                     │
│  // XFS 마운트 옵션                                                    │
│  $ sudo mount -o noatime,nodiratime,logbufs=8 /dev/sdb1 /data       │
│                                                                     │
│  【커널 파라미터 튜닝】                                                │
│  ──────────────────                                                  │
│  // 더티 페이지 설정                                                    │
│  $ sysctl vm.dirty_ratio=20                                         │
│  $ sysctl vm.dirty_background_ratio=10                              │
│  $ sysctl vm.dirty_expire_centisecs=3000                            │
│                                                                     │
│  // /etc/sysctl.conf에 추가                                           │
│  vm.dirty_ratio = 20                                                │
│  vm.dirty_background_ratio = 10                                     │
│  vm.swappiness = 10                                                 │
│  vm.vfs_cache_pressure = 50                                         │
│                                                                     │
│  【성능 측정】                                                          │
│  ──────────────────                                                  │
│  // I/O 성능 측정                                                      │
│  $ sudo fio --name=randread --ioengine=libaio \                     │
│        --iodepth=16 --rw=randread --bs=4k \                         │
│        --direct=1 --size=1G --numjobs=4 /dev/sdb1                  │
│                                                                     │
│  // 순차 읽기/쓰기                                                      │
│  $ dd if=/dev/zero of=/data/test bs=1M count=1024 conv=fdatasync   │
│  $ dd if=/data/test of=/dev/null bs=1M                             │
│                                                                     │
│  // IOSTAT 모니터링                                                    │
│  $ iostat -x 1                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 시스템 성능 최적화
• 마운트 옵션: noatime, nodiratime, discard
• 저널링: writeback(속도) vs ordered(균형)
• I/O 스케줄러: noop(SSD), deadline(DB)
• readahead: 순차 읽기 성능 향상
• 더티 페이지: 쓰기 버퍼링 조정
• 워크로드별: DB, 웹, 미디어 최적화 다름
• 측정: fio, iostat, dd로 검증
```
