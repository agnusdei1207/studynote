# 491. 파일 시스템 복구 (File System Recovery)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 손상된 파일 시스템을 정상 상태로 복원
> 2. **가치**: 데이터 손실 방지와 서비스 연속성
> 3. **융합**: 백업, 스냅샷, 저널링과 연관

---

## Ⅰ. 개요

### 개념 정의

파일 시스템 복구(File System Recovery)는 **시스템 장애나 데이터 손상 시 파일 시스템을 정상 상태로 복원하는 기술**이다.

### 💡 비유: 의료 응급처치
파일 시스템 복구는 **응급처치**와 같다. 환자가 다쳤을 때 빠르게 진단하고 치료한다.

### 파일 시스템 복구 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 복구                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【복구 단계】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1단계: 장애 탐지                                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • 파일 시스템 오류 메시지                                 │    │ │
│  │  │  • I/O 에러, 읽기/쓰기 실패                               │    │ │
│  │  │  • 슈퍼블록 손상                                           │    │ │
│  │  │  • 마운트 실패                                             │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  2단계: 손상 분석                                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • fsck로 무결성 검사                                     │    │ │
│  │  │  • 메타데이터 일관성 확인                                  │    │ │
│  │  │  • 데이터 블록 손상 범위 파악                              │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  3단계: 복구 실행                                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • 자동 복구 (저널 리플레이)                               │    │ │
│  │  │  • 수동 복구 (fsck)                                        │    │ │
│  │  │  • 백업에서 복원                                           │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【복구 방법】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  방법            복구 시간    데이터 손실    용도                       │ │
│  │  ────            ────────    ─────────    ────                       │ │
│  │  fsck 수동 복구   길다        있을 수 있음  경미한 손상                 │ │
│  │  저널 리플레이     짧다        거의 없음    저널링 FS                   │ │
│  │  스냅샷 복원       짧다        스냅샷 이후   스냅샷 있는 경우            │ │
│  │  백업 복원         길다        백업 이후    심각한 손상                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 복구 상세                                   │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【저널링 파일 시스템 복구】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 저널 리플레이 (Journal Replay):                                    │ │
│  │     • 완료되지 않은 트랜잭션 식별                                       │ │
│  │     • 커밋된 트랜잭션 재적용                                            │ │
│  │     • 미완료 트랜잭션 롤백                                              │ │
│  │                                                             │ │
│  │  2. 복구 시간:                                                         │ │
│  │     • 저널 크기에 비례                                                  │ │
│  │     • 보통 수 초 ~ 수 분                                                │ │
│  │     • fsck보다 훨씬 빠름                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【fsck 복구 과정】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  단계      검사 항목                                                    │ │
│  │  ────      ────────                                                    │ │
│  │  Phase 1  아이노드 일관성                                               │ │
│  │  Phase 2  디렉토리 구조                                                 │ │
│  │  Phase 3  디렉토리 연결                                                 │ │
│  │  Phase 4  참조 카운트                                                   │ │
│  │  Phase 5  비트맵 일관성                                                 │ │
│  │  Phase 6  데이터 블록                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【손상 유형별 복구】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  손상 유형         증상              복구 방법                             │ │
│  │  ────────         ────              ────────                             │ │
│  │  슈퍼블록 손상      마운트 불가        예비 슈퍼블록 사용                    │ │
│  │  아이노드 손상      파일 접근 불가      fsck로 복구 또는 삭제                 │ │
│  │  디렉토리 손상      경로 탐색 불가      fsck로 재구성                        │ │
│  │  데이터 블록 손상   파일 내용 깨짐      백업에서 복원                        │ │
│  │  저널 손상          부팅 실패          저널 재생성                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ext4 파일 시스템 복구】                                            │
│  ──────────────────                                                  │
│  // 읽기 전용으로 마운트 시도                                           │
│  $ sudo mount -o ro /dev/sdb1 /mnt                                  │
│                                                                     │
│  // 파일 시스템 검사                                                   │
│  $ sudo e2fsck -n /dev/sdb1     // 읽기 전용 검사                     │
│  $ sudo e2fsck -p /dev/sdb1     // 자동 수정                         │
│  $ sudo e2fsck -y /dev/sdb1     // 모든 질문에 yes                    │
│  $ sudo e2fsck -f /dev/sdb1     // 강제 검사                         │
│                                                                     │
│  // 슈퍼블록 복구                                                       │
│  $ sudo mke2fs -n /dev/sdb1     // 예비 슈퍼블록 위치 확인             │
│  $ sudo e2fsck -b 32768 /dev/sdb1  // 예비 슈퍼블록 사용              │
│                                                                     │
│  【XFS 파일 시스템 복구】                                              │
│  ──────────────────                                                  │
│  // XFS 복구                                                          │
│  $ sudo xfs_repair -n /dev/sdb1  // 읽기 전용 검사                     │
│  $ sudo xfs_repair /dev/sdb1     // 복구 실행                         │
│                                                                     │
│  // 로그 재생성                                                        │
│  $ sudo xfs_repair -L /dev/sdb1 // 로그 zeroing (주의)               │
│                                                                     │
│  【ZFS 복구】                                                          │
│  ──────────────────                                                  │
│  // 풀 상태 확인                                                        │
│  $ sudo zpool status                                                 │
│  $ sudo zpool scrub poolname    // 무결성 검사                        │
│                                                                     │
│  // 복구                                                              │
│  $ sudo zpool clear poolname    // 에러 클리어                        │
│  $ sudo zpool import -F poolname   // 복구 모드로 import             │
│                                                                     │
│  // 스냅샷에서 복원                                                     │
│  $ sudo zfs rollback pool/dataset@snap1                             │
│                                                                     │
│  【백업에서 복원】                                                      │
│  ──────────────────                                                  │
│  // rsync 복원                                                        │
│  $ sudo rsync -avz /backup/data/ /mnt/recovered/                    │
│                                                                     │
│  // tar 복원                                                          │
│  $ sudo tar -xzvf backup.tar.gz -C /mnt/recovered/                  │
│                                                                     │
│  // dd 이미지 복원                                                     │
│  $ sudo dd if=/backup/disk.img of=/dev/sdb1 bs=4M                  │
│                                                                     │
│  【복구 후 검증】                                                      │
│  ──────────────────                                                  │
│  // 파일 시스템 무결성 확인                                             │
│  $ sudo e2fsck -fn /dev/sdb1                                        │
│                                                                     │
│  // 파일 개수 확인                                                      │
│  $ find /mnt -type f | wc -l                                        │
│                                                                     │
│  // 디스크 사용량 확인                                                  │
│  $ df -h /mnt                                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 손상된 파일 시스템 정상 복원
• 방법: fsck, 저널 리플레이, 백업 복원
• 도구: e2fsck, xfs_repair, zpool
• 우선순위: 저널 → fsck → 스냅샷 → 백업
• 예방: 정기 백업, 스냅샷, UPS
• 주의: 복구 전 이미지 백업 권장
• 저널링 FS: 빠른 복구, 데이터 보호
• 모니터링: SMART, dmesg, 로그
```
