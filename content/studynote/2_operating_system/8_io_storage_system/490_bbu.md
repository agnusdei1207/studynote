# 490. 로그 구조 파일 시스템 (Log-Structured File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 변경을 로그처럼 순차 기록
> 2. **가치**: 쓰기 성능 최적화와 빠른 복구
> 3. **융합**: 저널링, SSD, NoSQL과 연관

---

## Ⅰ. 개요

### 개념 정의

로그 구조 파일 시스템(Log-Structured File System, LFS)은 **모든 데이터와 메타데이터 변경을 로그 형태로 순차적으로 기록하는 파일 시스템**이다.

### 💡 비유: 일기장
로그 구조 파일 시스템은 **일기장**과 같다. 모든 일을 순서대로 적어 나간다. 지우는 대신 새 내용을 계속 추가한다.

### 로그 구조 파일 시스템 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                로그 구조 파일 시스템                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기존 파일 시스템 vs LFS】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  기존 파일 시스템 (In-place Update):                                    │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  시간 순서:                                               │    │ │
│  │  │  [A1][B1][C1][D1][E1]   ← 초기                             │    │ │
│  │  │  [A2][B1][C1][D1][E1]   ← A 수정 (같은 위치)               │    │ │
│  │  │  [A2][B2][C1][D1][E1]   ← B 수정 (같은 위치)               │    │ │
│  │  │         ↑ 랜덤 쓰기 발생                                     │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  로그 구조 파일 시스템 (Append-Only):                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  시간 순서:                                               │    │ │
│  │  │  [A1][B1][C1][D1][E1]   ← 초기                             │    │ │
│  │  │  [A1][B1][C1][D1][E1][A2][B2]  ← 새 위치에 추가            │    │ │
│  │  │                         ↑ 순차 쓰기                         │    │ │
│  │  │                                                             │    │ │
│  │  │  세그먼트 단위로 순차 기록                                      │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LFS 구조】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  세그먼트 0    세그먼트 1    세그먼트 2    세그먼트 3       │    │ │
│  │  │  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐        │    │ │
│  │  │  │데이터   │  │데이터   │  │데이터   │  │데이터   │        │    │ │
│  │  │  │+메타   │  │+메타   │  │+메타   │  │+메타   │        │    │ │
│  │  │  │+i노드  │  │+i노드  │  │+i노드  │  │+i노드  │        │    │ │
│  │  │  └────────┘  └────────┘  └────────┘  └────────┘        │    │ │
│  │  │       ↑                                             │    │ │
│  │  │   순차 기록                                         │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  아이노드 맵: 아이노드 번호 → 디스크 위치                               │ │
│  │  세그먼트 요약: 각 세그먼트의 내용 정보                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                로그 구조 파일 시스템 상세                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 메커니즘】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 순차 쓰기 (Sequential Write):                                      │ │
│  │     • 모든 쓰기가 로그 끝에 추가                                         │ │
│  │     • 랜덤 쓰기를 순차 쓰기로 변환                                       │ │
│  │     • HDD에서 큰 성능 향상                                              │ │
│  │                                                             │ │
│  │  2. 세그먼트 세척 (Segment Cleaning):                                  │ │
│  │     • 오래된 데이터가 있는 세그먼트 정리                                 │ │
│  │     • 유효 데이터만 새 세그먼트로 이동                                   │ │
│  │     • GC (Garbage Collection)의 일종                                   │ │
│  │                                                             │ │
│  │  3. 아이노드 맵 (Inode Map):                                           │ │
│  │     • 아이노드 번호로 현재 위치 조회                                     │ │
│  │     • 아이노드가 로그에 계속 추가되므로 필요                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                               │
│  │  • 뛰어난 쓰기 성능 (순차 I/O)                                         │ │
│  │  • 빠른 복구 (로그 끝에서 재생)                                         │ │
│  │  • SSD에 적합 (랜덤 쓰기 감소)                                          │ │
│  │  • 스냅샷 구현 용이                                                    │ │
│  │                                                             │ │
│  │  단점:                                                               │
│  │  • 세그먼트 세척 오버헤드                                              │ │
│  │  • 읽기 성능 (간접 참조)                                               │ │
│  │  • 구현 복잡도                                                         │ │
│  │  • 공간 오버헤드                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LFS 기반 파일 시스템】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • BSD LFS: 초기 LFS 구현                                             │ │
│  │  • ZFS: Copy-on-Write + 트랜잭션                                       │ │
│  │  • Btrfs: COW + B-트리                                                 │ │
│  │  • F2FS: 플래시 친화적 LFS (안드로이드)                                 │ │
│  │  • NILFS: 지속적인 스냅샷                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【F2FS (Flash-Friendly File System)】                               │
│  ──────────────────                                                  │
│  // F2FS 파일 시스템 생성                                              │
│  $ sudo mkfs.f2fs /dev/sdb1                                          │
│                                                                     │
│  // 마운트                                                              │
│  $ sudo mount -t f2fs /dev/sdb1 /mnt/f2fs                           │
│                                                                     │
│  // 상태 확인                                                          │
│  $ dump.f2fs /dev/sdb1                                               │
│  $ cat /sys/fs/f2fs/sdb1/status                                      │
│                                                                     │
│  // GC 설정                                                            │
│  $ cat /sys/fs/f2fs/sdb1/gc_urgent_sleep_time                        │
│  $ echo 500 > /sys/fs/f2fs/sdb1/gc_urgent_sleep_time                │
│                                                                     │
│  【NILFS2 (Continuous Snapshotting)】                                 │
│  ──────────────────                                                  │
│  // 파일 시스템 생성                                                    │
│  $ sudo mkfs.nilfs2 /dev/sdb1                                        │
│                                                                     │
│  // 마운트                                                              │
│  $ sudo mount -t nilfs2 /dev/sdb1 /mnt/nilfs                        │
│                                                                     │
│  // 자동 스냅샷 확인                                                    │
│  $ lscp                                                               │
│  CNO DATE     TIME     MODE  FLG   NBLKINC   ICNT                   │
│  1   2024-01-01 10:00:00  cp    -     1000   500                    │
│  2   2024-01-01 11:00:00  cp    ss    500     520                   │
│                                                                     │
│  // 과거 시점 마운트                                                    │
│  $ sudo mount -t nilfs2 -r -o cp=1 /dev/sdb1 /mnt/snap              │
│                                                                     │
│  【ZFS (LFS 특성 포함)】                                               │
│  ──────────────────                                                  │
│  // ZFS 풀 생성                                                        │
│  $ sudo zpool create mypool /dev/sdb1                               │
│                                                                     │
│  // Copy-on-Write 확인                                                 │
│  $ sudo zfs list -t all                                              │
│  $ sudo zfs get copies mypool                                        │
│                                                                     │
│  // 스냅샷 생성                                                        │
│  $ sudo zfs snapshot mypool@snap1                                    │
│                                                                     │
│  // 스냅샷 목록                                                        │
│  $ sudo zfs list -t snapshot                                         │
│                                                                     │
│  【Btrfs (LFS 특성)】                                                  │
│  ──────────────────                                                  │
│  // 생성                                                               │
│  $ sudo mkfs.btrfs /dev/sdb1                                         │
│  $ sudo mount /dev/sdb1 /mnt/btrfs                                  │
│                                                                     │
│  // 스냅샷                                                             │
│  $ sudo btrfs subvolume snapshot /mnt/btrfs /mnt/btrfs/snap1         │
│                                                                     │
│  // COW 속성 확인                                                      │
│  $ sudo btrfs property get /mnt/btrfs compression                    │
│                                                                     │
│  【LFS 특성 활용 사례】                                                │
│  ──────────────────                                                  │
│  • SSD: 랜덤 쓰기 감소로 수명 증가                                      │
│  • 데이터베이스: WAL (Write-Ahead Logging)                             │
│  • NoSQL: LSM Tree (Log-Structured Merge Tree)                       │
│  • 백업: 중복 제거와 스냅샷                                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 모든 변경을 로그처럼 순차 기록
• 특징: Append-Only, In-place Update 없음
• 구조: 세그먼트, 아이노드 맵, 세그먼트 요약
• 핵심: 순차 쓰기, 세그먼트 세척 (GC)
• 장점: 쓰기 성능, 빠른 복구, SSD 수명
• 단점: 읽기 간접 참조, GC 오버헤드
• 구현: F2FS, NILFS2, ZFS, Btrfs
• 응용: WAL, LSM Tree, NoSQL
```
