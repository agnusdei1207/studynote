# 504. I/O 포트 (I/O Port)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 장치 접근을 위한 주소 공간
> 2. **가치**: CPU-장치 간 통신 채널
> 3. **융합**: I/O 매핑, 인터페이스와 연관

---

## Ⅰ. 개요

### 개념 정의

I/O 포트(I/O Port)는 **CPU가 I/O 장치의 레지스터에 접근하기 위해 사용하는 주소 공간**이다.

### 💡 비유: 우편함 번호
I/O 포트는 **우편함 번호**와 같다. 각 우편함에 번호가 있어 특정 번지로 편지를 보낸다.

### I/O 포트 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                I/O 포트 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【I/O 주소 공간】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  I/O 매핑 (Port-Mapped I/O):                                           │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  메모리 주소 공간                    I/O 주소 공간         │    │ │
│  │  │  ┌─────────────────────┐        ┌─────────────────────┐ │    │ │
│  │  │  │ 0x00000000          │        │ 0x0000              │ │    │ │
│  │  │  │    RAM              │        │   타이머 (0x0040)    │ │    │ │
│  │  │  │                     │        │   키보드 (0x0060)    │ │    │ │
│  │  │  │                     │        │   PIC (0x0020)      │ │    │ │
│  │  │  │ 0xFFFFFFFF          │        │   DMA (0x0000)      │ │    │ │
│  │  │  └─────────────────────┘        │   ...               │ │    │ │
│  │  │         ↑                        │ 0xFFFF              │ │    │ │
│  │  │     MOV [addr]                   └─────────────────────┘ │    │ │
│  │  │         │                                ↑                │    │ │
│  │  │    메모리 접근                          IN/OUT            │    │ │
│  │  │                                         │                │    │ │
│  │  │                                    I/O 포트 접근           │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【x86 I/O 포트 범위】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  포트 범위          장치/용도                                            │ │
│  │  ────────           ────────                                            │ │
│  │  0x0000-0x000F     DMA 컨트롤러 #1                                       │ │
│  │  0x0020-0x0021     PIC (Programmable Interrupt Controller) #1            │ │
│  │  0x0040-0x0043     시스템 타이머                                          │ │
│  │  0x0060-0x0063     키보드 컨트롤러                                        │ │
│  │  0x0070-0x0071     CMOS RTC                                              │ │
│  │  0x0080-0x008F     DMA 페이지 레지스터                                    │ │
│  │  0x00A0-0x00A1     PIC #2                                                │ │
│  │  0x00C0-0x00DF     DMA 컨트롤러 #2                                        │ │
│  │  0x00F0-0x00FF     수학 보조프로세서                                       │ │
│  │  0x0170-0x0177     IDE #1                                                │ │
│  │  0x01F0-0x01F7     IDE #0                                                │ │
│  │  0x0278-0x027F     병렬 포트 #2                                          │ │
│  │  0x02F8-0x02FF     직렬 포트 #2                                          │ │
│  │  0x0378-0x037F     병렬 포트 #1                                          │ │
│  │  0x03F0-0x03F7     플로피 디스크                                          │ │
│  │  0x03F8-0x03FF     직렬 포트 #1                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                I/O 포트 상세                                           │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【I/O 매핑 vs 메모리 매핑】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  특성            I/O 매핑              메모리 매핑                        │ │
│  │  ────            ────────              ────────                        │ │
│  │  주소 공간        분리                   공유                            │ │
│  │  명령어          IN/OUT                 MOV                             │ │
│  │  보호            별도                   메모리 보호 사용                   │ │
│  │  캐싱            없음                   제어 필요                        │ │
│  │  접근 속도        느림                   빠름                            │ │
│  │  주소 크기        16비트 (64K)           32/64비트                        │ │
│  │  예시            x86                    ARM, RISC-V                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/O 명령어 (x86)】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  명령어          설명                    예시                              │ │
│  │  ──────          ────                    ────                              │ │
│  │  IN AL, port     포트에서 1바이트 읽기      IN AL, 60H                      │ │
│  │  IN AX, port     포트에서 2바이트 읽기      IN AX, 60H                      │ │
│  │  IN EAX, port    포트에서 4바이트 읽기      IN EAX, 60H                     │ │
│  │  OUT port, AL    포트에 1바이트 쓰기        OUT 60H, AL                     │ │
│  │  OUT port, AX    포트에 2바이트 쓰기        OUT 60H, AX                     │ │
│  │  OUT port, EAX   포트에 4바이트 쓰기        OUT 60H, EAX                    │ │
│  │  INS             문자열 입력               REP INSB                        │ │
│  │  OUTS            문자열 출력               REP OUTSB                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【포트 접근 권한】                                                      │**
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  권한 레벨        접근 가능                                                 │ │
│  │  ────────        ────────                                                 │ │
│  │  Ring 0 (커널)   모든 I/O 포트                                             │ │
│  │  Ring 1-2       I/O 허용 비트맵에 따라                                      │ │
│  │  Ring 3 (사용자) I/O 허용 비트맵에서만                                       │ │
│  │                                                             │ │
│  │  TSS (Task State Segment)의 IOPM으로 제어                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【I/O 포트 확인】                                                      │
│  ──────────────────                                                  │
│  // 시스템 I/O 포트 목록                                                 │
│  $ cat /proc/ioports                                                │
│  0000-0cf7 : PCI Bus 0000:00                                        │
│    0000-001f : dma1                                                 │
│    0020-0021 : pic1                                                 │
│    0040-0043 : timer0                                               │
│    0060-0060 : keyboard                                             │
│    ...                                                              │
│                                                                     │
│  // 특정 장치 포트 찾기                                                 │
│  $ cat /proc/ioports | grep -i serial                              │
│  $ cat /proc/ioports | grep -i keyboard                            │
│                                                                     │
│  【커널에서 I/O 포트 사용】                                              │
│  ──────────────────                                                  │
│  // I/O 포트 예약                                                      │
│  #include <linux/ioport.h>                                          │
│  struct resource *request_region(unsigned long start,               │
│                                   unsigned long n, const char *name);│
│  void release_region(unsigned long start, unsigned long n);         │
│                                                                     │
│  // I/O 포트 읽기/쓰기                                                  │
│  #include <asm/io.h>                                                │
│  unsigned char inb(unsigned short port);                            │
│  unsigned short inw(unsigned short port);                           │
│  unsigned int inl(unsigned short port);                             │
│  void outb(unsigned char value, unsigned short port);               │
│  void outw(unsigned short value, unsigned short port);               │
│  void outl(unsigned int value, unsigned short port);                │
│                                                                     │
│  【사용자 공간 I/O 포트 접근】                                           │
│  ──────────────────                                                  │
│  // iopl()로 권한 획득                                                  │
│  #include <sys/io.h>                                                │
│  int iopl(int level);     // level 3 = 모든 포트 접근                 │
│  int ioperm(unsigned long from, unsigned long num, int turn_on);    │
│                                                                     │
│  // 예: 키보드 LED 제어                                                 │
│  #include <sys/io.h>                                                │
│  ioperm(0x60, 8, 1);      // 0x60-0x67 포트 접근 권한                 │
│  outb(0xED, 0x60);        // LED 제어 명령                            │
│  outb(0x07, 0x60);        // 모든 LED 켜기                            │
│                                                                     │
│  【I/O 포트 디버깅】                                                    │
│  ──────────────────                                                  │
│  // I/O 포트 접근 추적                                                  │
│  $ trace-cmd record -e irq:irq_handler_entry -e irq:irq_handler_exit│
│                                                                     │
│  // /proc/ioports 갱신                                                │
│  $ cat /proc/ioports | wc -l                                        │
│                                                                     │
│  // 커널 메시지                                                        │
│  $ dmesg | grep -i "io port\|ioport"                               │
│                                                                     │
│  【QEMU에서 I/O 포트 테스트】                                           │
│  ──────────────────                                                  │
│  // QEMU 모니터에서                                                    │
│  (qemu) info ioports                                                │
│  (qemu) pio_read 0x60                                               │
│  (qemu) pio_write 0x60 0xED                                        │
│                                                                     │
│  【병렬 포트 제어 예시】                                                 │
│  ──────────────────                                                  │
│  // 병렬 포트 (0x378) 제어                                              │
│  #include <sys/io.h>                                                │
│  #include <unistd.h>                                                │
│                                                                     │
│  int main() {                                                       │
│      ioperm(0x378, 3, 1);  // 0x378-0x37A 접근                       │
│      outb(0xFF, 0x378);    // 데이터 핀 모두 HIGH                      │
│      usleep(1000000);                                               │
│      outb(0x00, 0x378);    // 데이터 핀 모두 LOW                       │
│      ioperm(0x378, 3, 0);  // 권한 해제                              │
│      return 0;                                                      │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: I/O 장치 접근을 위한 주소 공간
• 방식: I/O 매핑 (분리) vs 메모리 매핑 (공유)
• 범위: x86에서 0x0000-0xFFFF (64K)
• 명령: IN/OUT (x86), MOV (메모리 매핑)
• 권한: Ring 레벨과 IOPM으로 제어
• 확인: cat /proc/ioports
• 프로그래밍: inb/inw/inl, outb/outw/outl
• 사용자 접근: ioperm(), iopl()
```
