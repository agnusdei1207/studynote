# 605. SSD (Solid State Drive)
Solid State Drive)**는 **반도체 메모리를 사용하여 데이터를 저장하는 저장 장치**입니다.
### 💡 비유: 디지털 도서관
SSD는 **디지털 도서관**과 같습니다. 책장 없고 빠르게 찾을 수 있습니다.
### SSD 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                SSD 구조                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SSD 유형】                                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │

│  │  유형              설명                    용도                       │ │   │

│  │  SATA SSD         SATA 인터페이스          소비자/서버                 │ │   │
│  │  NVMe SSD         PCIe 인터페이스           고성능/서버                 │ │   │
│  │  M.2 SSD          M.2 슬롯 (소형)           노트북/소형 장치          │ │   │
│  │  mSATA SSD        mSATA 인터페이스         모바일/임베디드             │ │   │

│  └─────────────────────────────────────────────────────────────┘   │

│  │  ────              ─────────            ────                    │ │   │
│  │  SLC              1,000~3,000           좋음                  │ │   │

│  │  QLC             1,000~5,000,000        읽기/쓰기 혼합           │ │   │
│  │  PLC (3D)         150,000+              높은 용량               │ │   │

│                                                                     │

│  │  ────              ────                  ────                      │ │   │

│  │  랜덤 접근          5-10ms               0.1ms                  │ │   │
│  │  소음              높음                   낮음 (무음)                 │ │   │
│  │  전력 소비          5-10W                0.5-3W                  │ │   │
│  │  내구성             3-5년                 5-10년                  │ │   │
│  │  충격               높음                 낮음                    │ │   │
│  │  발열              있음                 없음                    │ │   │
│  │  IOPS             50-150               10,000-100,000           │ │   │
│  │  가격/GB           낮음                 높음                    │ │   │
│  │                                                             │ │   │
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                SSD 상세                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SSD 핵심 기술】                                                 │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  • TRIM (Trim): 사용하지 않은 블록 정리 (가비지 회수)                    │ │   │
│  │  • GC (Garbage Collection): 삭제된 데이터 회수 (쓰기 증가)                  │ │   │
│  │  • Wear Leveling: 쓰기 증가 균일 분산 (SSD 수명 단축)               │ │   │
│  │  • Over-Provisioning: 여유 공간 미리 할당 (성능 저하 방지)                │ │   │
│  │  • ECC (Error Correction): 오류 정정 코드                          │ │   │
│  │  • 암호화: 하드웨어 암호화 (자기 암호화, 전체 디스크)                │ │   │
│  │  • SED (Single-Level Cell): 1비트 셀  저장 (MLC)                │ │   │
│  │  • 3D NAND (TLC/QLC/PLC): 용량 증가, 비용 절감                  │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【SSD 수명 관리】                                                  │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  수명 단계             설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  신규 (Fresh)          쓰기 횟수 낮음, 성능 최고                         │ │   │
│  │  중고 (Used)          쓰기 횟수 중간, 성능 중간                        │ │   │
│  │  고갈 (Old)           쓰기 횟수 높음, 성능 저하                         │ │   │
│  │  사망 (Dead)          쓰기 불가, 교체 필요                              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【SSD 성능 지표】                                                  │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  지표                설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  순차 읽기/쓰기    연속된 데이터 전송 속도                         │ │   │
│  │  랜덤 읽기/쓰기    4KB 랜덤 접근 속도 (IOPS)                     │ │   │
│  │  지연 시간           읽기/쓰기 요청의 대기 시간                           │ │   │
│  │  대역폭             데이터 전송 속도 (MB/s)                          │ │   │
│  │  IOPS              초당 입출력 연산 (Input/Output Operations Per Second)  │ │   │
│  │  TBW               총 기록된 바이트 수 (Terabytes Written)               │ │   │
│  │  DWPD              일일 쓰기량 (Drive Writes Per Day)                  │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【NVMe SSD 특징】                                                  │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  • PCIe 버스 직접 사용 (AHCI 제거)                                │ │   │
│  │  • 다중 큐 (Submission Queue, Completion Queue)                       │ │   │
│  │  • MSI/MSI-X 인터럽트 지원 (벡터 I/O)                                │ │   │
│  │  • 낮은 지연 시간 (< 10μs)                                           │ │   │
│  │  • 높은 IOPS (수십만/십만 IOPS)                                       │ │   │
│  │  • 핫 스왑 지원 (선택적)                                           │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SSD 정보 확인】                                                    │ |
│  ──────────────────                                                │
│  // lsblk                                                            │
│  $ lsblk                                                            │
│  $ lsblk -d                  // 디바이스만                               │
│  $ lsblk -f                  // 파일시스템 정보                       │
│  $ lsblk -o +ROTA,TYPE,FSTYPE                                      │
│                                                                     │
│  // nvme list (NVMe SSD)                                             │
│  $ sudo nvme list                                                   │
│  $ sudo nvme list -v                                               │
│                                                                     │
│  // smartctl (SMART)                                                 │
│  $ sudo smartctl -a /dev/nvme0n1                                     │
│  $ sudo smartctl -H /dev/nvme0n1 | grep -i "SMART"              │
│  $ sudo smartctl -l error /dev/nvme0n1                               │
│                                                                     │
│  // hdparm (SATA SSD)                                                │
│  $ sudo hdparm -I /dev/sda                // 상세 정보                  │
│  $ sudo hdparm -t /dev/sda                // 읽기 성능                  │
│                                                                     │
│  【SSD 성능 테스트】                                                 │ |
│  ──────────────────                                                │
│  // 순차 읽기                                                          │
│  $ sudo hdparm -t /dev/nvme0n1                                       │
│  $ sudo hdparm -tT /dev/nvme0n1                                      │
│                                                                     │
│  // dd로 테스트                                                        │
│  $ sudo dd if=/dev/zero of=/tmp/test bs=1M count=100              │
│  $ sudo dd if=/tmp/test of=/dev/null bs=1M                        │
│                                                                     │
│  // fio (유연한 I/O 테스터)                                            │
│  $ sudo fio --name=/dev/nvme0n1 --rw=randread --bs=4k \               │
│    --iodepth=32 --numjobs=4 --size=1G --direct=1                  │
│                                                                     │
│  // io_uring (liburing)                                              │
│  $ sudo io_uring -b 4096 -d 1G -r /dev/nvme0n1                    │
│                                                                     │
│  【TRIM 실행】                                                        │
│  ──────────────────                                                │
│  // 수동 TRIM                                                         │
│  $ sudo fstrim -v /dev/nvme0n1p1                                    │
│  $ sudo fstrim -a /dev/nvme0n1p1                                   │
│                                                                     │
│  // 자동 TRIM (마운트 옵션)                                          │
│  $ sudo mount -o discard /dev/nvme0n1p1 /mnt                       │
│                                                                     │
│  // 주기적 TRIM (cron)                                               │
│  $ sudo systemctl enable fstrim.timer                              │
│  $ sudo fstrim -v /                                                 │
│                                                                     │
│  // TRIM 상태 확인                                                    │
│  $ lsblk -D /dev/nvme0n1                                            │
│  $ sudo hdparm -I /dev/sda | grep -i "TRIM"                    │
│                                                                     │
│  【SSD 건강 상태 확인】                                              │ |
│  ──────────────────                                                │
│  // SMART 속성 확인                                                   │
│  $ sudo smartctl -a /dev/nvme0n1                                    │
│  Available Spare: 100%                                            │
│  Percentage Used: 5%                                              │
│  Data Units Read: 1234567                                         │
│  Media and Data Integrity Errors: 0                               │
│                                                                     │
│  // 수명 확인                                                        │
│  $ sudo smartctl -a /dev/nvme0n1 | grep -i "Percentage"           │
│  Percentage used endurance indicator: 5%                          │
│                                                                     │
│  // 온도 확인                                                        │
│  $ sudo smartctl -a /dev/nvme0n1 | grep -i "Temperature"          │
│  Current Drive Temperature: 35 Celsius                           │
│                                                                     │
│  // 오류 확인                                                        │
│  $ sudo smartctl -l error /dev/nvme0n1                             │
│  No Errors Logged                                                 │
│                                                                     │
│  【NVMe 설정】                                                        │
│  ──────────────────                                                │
│  // NVMe 컨트롤러 정보                                                │
│  $ sudo nvme id-ctrl /dev/nvme0 --output-format=json               │
│                                                                     │
│  // NVMe 네임스페이스 정보                                            │
│  $ sudo nvme id-ns /dev/nvme0n1 --output-format=json               │
│                                                                     │
│  // NVMe 기능 확인                                                    │
│  $ sudo nvme get-feature /dev/nvme0 -f 0                           │
│  $ sudo nvme get-feature /dev/nvme0 -f 1                           │
│                                                                     │
│  // NVMe 기능 설정                                                    │
│  $ sudo nvme set-feature /dev/nvme0 -f 0 -v 1                      │
│                                                                     │
│  // NVMe 포맷                                                        │
│  $ sudo nvme format /dev/nvme0n1                                   │
│                                                                     │
│  // APST (Autonomous Power State)                                   │
│  $ sudo nvme get-apst /dev/nvme0                                   │
│  $ sudo nvme set-apst /dev/nvme0                                   │
│                                                                     │
│  【SSD 최적화】                                                        │
│  ──────────────────                                                │
│  // I/O 스케줄러 (SSD용)                                             │
│  $ cat /sys/block/nvme0n1/queue/scheduler                          │
│  [none] mq-deadline                                               │
│  $ echo none | sudo tee /sys/block/nvme0n1/queue/scheduler        │
│                                                                     │
│  // 마운트 옵션 (discard)                                            │
│  $ sudo mount -o discard /dev/nvme0n1p1 /mnt                       │
│                                                                     │
│  // 스왑 사용                                                        │
│  $ sudo swapon /dev/nvme0n1p2                                      │
│                                                                     │
│  // 파일시스템 선택 (SSD에 적합)                                     │
│  // ext4: 일반적                                                      │
│  // XFS: 대용량                                                       │
│  // Btrfs: CoW, 스냅샷                                                │
│  // F2FS: 플래시 친화 (모바일)                                         │
│                                                                     │
│  // 파티션 정렬 (1MB 경계)                                            │
│  $ sudo parted /dev/nvme0n1 mkpart primary 1MiB 100%               │
│                                                                     │
│  【SSD 백업】                                                          │
│  ──────────────────                                                │
│  // 이미지 백업                                                        │
│  $ sudo dd if=/dev/nvme0n1 of=ssd.img bs=4M status=progress            │
│                                                                     │
│  // 파일시스템 백업                                                  │
│  $ sudo rsync -avx /mnt/data/ /backup/data/                         │
│                                                                     │
│  // 증분 백업 (rsync)                                                │
│  $ sudo rsync -avz --link-dest=/backup/current /data/ /backup/new/   │
│                                                                     │
│  // 암호화 백업                                                        │
│  $ sudo dd if=/dev/nvme0n1 | openssl enc -aes-256-cbc > ssd.enc   │
│                                                                     │
│  【SSD 보안 삭제】                                                    │
│  ──────────────────                                                │
│  // ATA Secure Erase (SATA SSD)                                     │
│  $ sudo hdparm --security-erase /dev/sda                            │
│  $ sudo hdparm --security-erase-enhanced /dev/sda                     │
│                                                                     │
│  // NVMe Format (NVMe SSD)                                          │
│  $ sudo nvme format /dev/nvme0n1 --ses=1 --pil=0                       │
│                                                                     │
│  // shred                                                             │
│  $ sudo shred -vfz -n 3 /dev/nvme0n1                                │
│                                                                     │
│  // blkdiscard (파티션)                                              │
│  $ sudo blkdiscard /dev/nvme0n1p1                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 반도체 메모리를 사용하는 저장 장치
• 유형: SATA SSD, NVMe SSD, M.2 SSD
• NAND 유형: SLC, MLC, TLC, QLC, PLC (3D)
• 인터페이스: SATA (600-750 MB/s), PCIe (4-32 GB/s)
• 장점: 빠른 속도, 낮은 지연, 무음, 저전력
• 단점: 높은 가격, 쓰기 제한, 데이터 복구 어려움
• 수명: 5-10년 (TBW, DWPD 기반)
• 기술: TRIM, GC, Wear Leveling, ECC
 SED
 3D NAND
 확인: lsblk, smartctl, nvme list
• 성능: fio, hdparm, dd
• 최적화: I/O 스케줄러, TRIM, discard
 마운트
• 보안: Secure Erase, shred, blkdiscard
 백업: dd, rsync
 NVMe: nvme-cli 패키지
```
