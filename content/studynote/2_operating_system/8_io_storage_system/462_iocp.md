# 462. 파일 시스템 손상 (File System Corruption)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 시스템 메타데이터 오류
> 2. **가치**: 데이터 무결성 위협
> 3. **융합**: fsck, 저널링과 연관

---

## Ⅰ. 개요

### 개념 정의

파일 시스템 손상(File System Corruption)은 **파일 시스템의 메타데이터나 데이터가 일관성을 잃는 상황**이다.

### 💡 비유: 도서관 카탈로그 오류
파일 시스템 손상은 **도서관 카탈로그 오류**와 같다. 책 위치가 틀리거나, 없는 책이 등록되어 있다.

### 파일 시스템 손상 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 손상                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【손상 유형】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 메타데이터 손상:                                                  │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • 슈퍼블록 손상: 파일 시스템 전체 정보 오류                       │    │ │
│  │  │  • 아이노드 손상: 파일 메타데이터 오류                             │    │ │
│  │  │  • 디렉토리 손상: 경로 정보 오류                                   │    │ │
│  │  │  • 비트맵 손상: 블록 할당 정보 오류                                │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  2. 데이터 손상:                                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • 파일 내용 손상: 실제 데이터 변경                               │    │ │
│  │  │  • 블록 오류: 디스크 배드 섹터                                    │    │ │
│  │  │  • 부분 쓰기: 트랜잭션 미완료                                     │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  3. 구조적 손상:                                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • 고아 아이노드: 참조되지 않는 아이노드                           │    │ │
│  │  │  • 중복 할당: 같은 블록이 여러 파일에 할당됨                        │    │ │
│  │  │  • 순환 참조: 디렉토리가 서로를 무한히 참조                         │    │ │
│  │  │  • 잘못된 링크 수: 실제와 불일치                                   │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【손상 탐지】                                                          │
│  ──────────────────                                                  │
│  // dmesg로 커널 메시지 확인                                           │
│  $ dmesg | grep -i "error\|corrupt\|fail"                          │
│  EXT4-fs error (device sda1): ext4_find_entry: ...                 │
│                                                                     │
│  // SMART 상태 확인                                                   │
│  $ sudo smartctl -a /dev/sda                                        │
│  Reallocated_Sector_Ct: 10  // 배드 섹터                              │
│  Current_Pending_Sector: 5  // 대기 중                                │
│                                                                     │
│  // 파일 시스템 상태 확인                                                │
│  $ sudo tune2fs -l /dev/sda1 | grep "Filesystem state"             │
│  Filesystem state: clean   // 또는 "with errors"                     │
│                                                                     │
│  【손상 복구】                                                          │
│  ──────────────────                                                  │
│  // 1. 파일 시스템 언마운트                                              │
│  $ sudo umount /dev/sda1                                            │
│                                                                     │
│  // 2. 읽기 전용 검사                                                   │
│  $ sudo fsck -n /dev/sda1                                           │
│                                                                     │
│  // 3. 자동 복구                                                       │
│  $ sudo fsck -y /dev/sda1                                           │
│  // 고아 파일은 lost+found로 이동                                        │
│                                                                     │
│  // 4. 복구된 파일 확인                                                 │
│  $ ls -la /lost+found/                                              │
│  // 숫자로 된 파일명 (아이노드 번호)                                       │
│                                                                     │
│  // 5. 심각한 손상 시 백업에서 복원                                        │
│  $ sudo mkfs.ext4 /dev/sda1                                         │
│  $ sudo mount /dev/sda1 /mnt                                        │
│  $ sudo rsync -av /backup/ /mnt/                                   │
│                                                                     │
│  【읽기 전용 마운트로 복구】                                              │
│  ──────────────────                                                  │
│  // 손상된 파일시스템을 읽기 전용으로 마운트                                  │
│  $ sudo mount -o ro /dev/sda1 /mnt                                 │
│                                                                     │
│  // 데이터 백업                                                         │
│  $ sudo rsync -av /mnt/ /backup/                                   │
│                                                                     │
│  // 파일시스템 재생성 후 복원                                               │
│  $ sudo umount /mnt                                                 │
│  $ sudo mkfs.ext4 /dev/sda1                                         │
│  $ sudo mount /dev/sda1 /mnt                                        │
│  $ sudo rsync -av /backup/ /mnt/                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
