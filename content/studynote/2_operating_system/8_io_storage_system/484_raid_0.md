## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RAID 0는 데이터를 여러 디스크에 블록 단위로 분산하여 동시에 읽고 쓰는 '스트라이핑(Striping)' 기술을 통해 입출력 성능을 극대화하는 방식이다.
> 2. **성능**: 디스크 개수에 비례하여 대역폭이 확장되지만, 중복성(Redundancy)이 전혀 없어 단 하나의 디스크만 고장 나도 전체 데이터를 소실한다.
> 3. **용도**: 데이터 안정성보다는 초고속 처리가 필요한 임시 작업 공간(Scratch Space), 영상 편집 캐시, 비중요 데이터 처리에 주로 사용된다.

---

## Ⅰ. 개요 (Context & Background)

- **정의**: 중복성 없이 순수하게 '성능'과 '용량'만을 위해 구성된 RAID 레벨이다.
- **배경**: 단일 디스크의 전송 속도가 CPU나 메모리의 처리 속도를 따라가지 못할 때, 여러 개의 디스크를 병렬로 연결하여 병목 현상을 해결하기 위해 등장했다.
- **💡 비유**: **"이어달리기 대신 동시 달리기"**와 같다. 한 사람이 100m를 뛰는 것보다, 10명이 동시에 10m씩 나눠서 뛰면 전체 거리를 훨씬 빨리 완주할 수 있는 것과 같다. 다만, 한 명이라도 넘어지면 프로젝트(데이터) 자체가 실패한다는 점이 특징이다.

- **📢 섹션 요약 비유**: "안전장치 없이 속도에만 모든 것을 건 레이싱카"와 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### RAID 0 스트라이핑 메커니즘 (ASCII Diagram)

```text
  [ 호스트 쓰기 요청: 데이터 A, B, C, D ]
           │
  ┌────────▼────────┐
  │  RAID 컨트롤러  │ (데이터 분할)
  └────────┬────────┘
           │
  ┌────────┴──────────────────┐
  │         [ Striping ]      │
  │   Disk 1       Disk 2     │
  │  ┌──────┐     ┌──────┐    │
  │  │ 블록 A │     │ 블록 B │    │  (동시 쓰기)
  │  ├──────┤     ├──────┤    │
  │  │ 블록 C │     │ 블록 D │    │  (동시 쓰기)
  │  └──────┘     └──────┘    │
  │                           │
  │  (논리적 용량 = D1 + D2)   │
  └───────────────────────────┘
```

**[다이어그램 해설]** 호스트가 보낸 데이터를 일정 크기(Stripe Size)로 쪼개어 각 디스크에 순차적으로 분산 저장한다. 읽을 때도 각 디스크에서 동시에 데이터를 가져오므로 이론적으로 전송 속도는 `단일 디스크 속도 × 디스크 개수`에 수렴한다.

- **📢 섹션 요약 비유**: "책 한 권을 낱장으로 찢어 여러 사람이 동시에 읽게 함으로써 전체 읽는 시간을 줄이는 전략"과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### RAID 0의 특징 상세

| 항목 | 특성 | 비고 |
|:---|:---|:---|
| **최소 디스크 수** | 2개 | |
| **읽기/쓰기 성능** | 최고 (n배 향상) | n은 디스크 개수 |
| **공간 효율성** | 100% (n개 모두 사용) | 낭비되는 용량 없음 |
| **장애 허용 (Fault Tolerance)** | **0 (없음)** | 디스크 1개 장애 = 전체 데이터 파손 |
| **신뢰성 (MTBF)** | 낮음 | 디스크가 많아질수록 고장 확률은 산술적으로 증가 |

### 신뢰성 분석 (위험성)
- 단일 디스크의 고장 확률을 $p$라고 할 때, $n$개 디스크로 구성된 RAID 0의 고장 확률은 $1 - (1-p)^n$이 된다. 즉, 디스크를 4개 묶으면 단일 디스크보다 고장 날 확률이 약 4배 높아진다.

- **📢 섹션 요약 비유**: "엔진을 여러 개 달아 속도는 빨라졌지만, 엔진 하나만 꺼져도 추락하는 비행기"입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **영상 편집 워크스테이션**: 고해상도 4K/8K 영상을 실시간으로 프리뷰하거나 렌더링할 때 필요한 대역폭을 확보하기 위해 SSD 여러 개를 RAID 0로 묶어 사용한다.
2. **게임 로딩 최적화**: 게임 데이터는 언제든 다시 다운로드할 수 있으므로, 로딩 속도만을 위해 RAID 0를 구성하는 하이엔드 게이머들이 존재한다.
3. **SWAP 공간**: 서버의 가상 메모리(Swap) 공간을 RAID 0로 구성하여 메모리 부족 시의 성능 저하를 최소화한다.

### 도입 체크리스트
- **데이터 중요도**: 사라져도 금방 복구 가능한 데이터인가? (Yes인 경우에만 도입)
- **백업 전략**: RAID 0 외부에 실시간 혹은 주기적 백업 장치가 별도로 있는가?
- **Stripe Size 튜닝**: 워크로드의 평균 I/O 크기에 맞춰 스트라이프 크기(64KB, 128KB 등)를 최적화했는가?

### 안티패턴
- **중요 DB 저장**: 성능을 위해 DB 데이터 파일을 RAID 0에 두는 행위는 '데이터 자살 행위'와 같다. DB 로그나 데이터는 반드시 RAID 1, 5, 10 등 보호 기능이 있는 레벨에 두어야 한다.

- **📢 섹션 요약 비유**: "언제든 다시 채울 수 있는 물통(임시 데이터)은 큰 게 좋지만, 하나뿐인 보물상자(중요 데이터)를 구멍 숭숭 뚫린 바구니에 담아서는 안 됩니다."

## Ⅴ. 기대효과 및 결론

### 기대효과
- **가성비 최고 성능**: 별도의 패리티 연산이나 미러링 비용 없이 가장 저렴하게 최고 속도 구현.
- **대용량 단일 볼륨**: 여러 개의 작은 디스크를 모아 하나의 거대한 단일 드라이브처럼 사용 가능.

### 미래 전망
- **NVMe RAID**: 이미 개별 NVMe SSD의 속도가 매우 빠르기 때문에, 일반적인 환경에서 RAID 0의 필요성은 줄어들고 있다. 그러나 수십 GB/s 급의 초고대역폭이 필요한 특수 분야(과학 연산, AI 모델 학습 등)에서는 여전히 핵심 기술로 쓰인다.

- **📢 섹션 요약 비유**: "오직 앞만 보고 달리는 속도의 광신도, 하지만 그 뒤에는 철저한 백업이라는 그림자가 동행해야 합니다."

## 📌 관련 개념 맵 (Knowledge Graph)
- **RAID 1 (미러링)**: RAID 0와 정반대의 철학(안정성)을 가진 모델.
- **RAID 10 (1+0)**: RAID 0의 속도와 RAID 1의 안정성을 결합한 하이브리드 모델.
- **Striping (스트라이핑)**: 데이터베이스 샤딩 등 분산 시스템 전반에서 쓰이는 보편적 개념.
