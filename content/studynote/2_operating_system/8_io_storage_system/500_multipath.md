# 500. 파일 시스템 트러블슈팅 (File System Troubleshooting)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 시스템 문제 진단과 해결
> 2. **가치**: 서비스 복구와 데이터 보호
> 3. **융합**: 로그 분석, 복구, 모니터링과 연관

---

## Ⅰ. 개요

### 개념 정의

파일 시스템 트러블슈팅(File System Troubleshooting)은 **파일 시스템 관련 문제를 체계적으로 진단하고 해결하는 과정**이다.

### 💡 비유: 의료 진단
파일 시스템 트러블슈팅은 **의료 진단**과 같다. 증상을 보고 원인을 찾아 치료한다.

### 파일 시스템 트러블슈팅 구조

```

│                                                                     │
│  【문제 해결 프로세스】                                                  │

│  ┌───────────────────────────────────────────────────────────────┐ │

│  │  │  • 하드웨어 점검 (SMART)                                  │    │ │
```

```
│  【진단 도구】                                                          │

│  │  /var/log/*     시스템 로그                                              │ │
│  │  df             파일시스템 사용량                                         │ │
│  │  du             디렉토리 크기                                            │ │

│  【에러 메시지별 대응】                                                 │

│  │  에러                    원인              해결                             │ │
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【공간 부족 문제】                                                      │
│  ──────────────────                                                  │
│  // 사용량 확인                                                        │
│  $ df -h                                                             │
│  $ df -i    // 아이노드 확인                                          │
│                                                                     │
│  // 큰 파일/디렉토리 찾기                                               │
│  $ du -sh /* | sort -rh | head -10                                  │
│  $ du -sh /var/* | sort -rh | head -10                              │
│                                                                     │
│  // 로그 정리                                                          │
│  $ journalctl --vacuum-size=100M                                    │
│  $ find /var/log -name "*.log" -size +100M -exec truncate -s 0 {} \;│
│                                                                     │
│  // 오래된 파일 삭제                                                    │
│  $ find /tmp -type f -mtime +7 -delete                              │
│  $ find /var/log -name "*.gz" -mtime +30 -delete                    │
│                                                                     │
│  // 패키지 캐시 정리                                                    │
│  $ apt clean                                                        │
│  $ yum clean all                                                    │
│                                                                     │
│  【마운트 문제】                                                        │
│  ──────────────────                                                  │
│  // 마운트 상태 확인                                                    │
│  $ mount | grep /data                                               │
│  $ findmnt /data                                                    │
│                                                                     │
│  // 마운트 실패 시                                                      │
│  $ dmesg | tail -20                                                 │
│  $ journalctl -xe                                                   │
│                                                                     │
│  // 강제 언마운트                                                       │
│  $ umount -l /data    // lazy unmount                               │
│  $ umount -f /data    // force (NFS)                                │
│                                                                     │
│  // 읽기 전용 마운트 해제                                                │
│  $ mount -o remount,rw /data                                        │
│                                                                     │
│  【파일 시스템 손상】                                                    │
│  ──────────────────                                                  │
│  // 손상 확인                                                          │
│  $ dmesg | grep -i "error\|corrupt\|fail"                          │
│                                                                     │
│  // 언마운트 후 복구                                                    │
│  $ umount /dev/sdb1                                                 │
│  $ fsck -y /dev/sdb1                                                │
│                                                                     │
│  // 라이브 시스템에서                                                    │
│  $ fsck -n /dev/sdb1   // 읽기 전용 검사                               │
│                                                                     │
│  // XFS 복구                                                          │
│  $ xfs_repair /dev/sdb1                                             │
│                                                                     │
│  【권한 문제】                                                          │
│  ──────────────────                                                  │
│  // 현재 권한 확인                                                      │
│  $ ls -la /data/file                                                │
│  $ namei -l /data/file                                              │
│                                                                     │
│  // 권한 수정                                                          │
│  $ chmod 755 /data/file                                             │
│  $ chown user:group /data/file                                      │
│                                                                     │
│  // ACL 확인                                                          │
│  $ getfacl /data/file                                               │
│                                                                     │
│  // SELinux 컨텍스트                                                   │
│  $ ls -Z /data/file                                                 │
│  $ restorecon -Rv /data                                             │
│                                                                     │
│  【I/O 문제】                                                           │
│  ──────────────────                                                  │
│  // I/O 대기 확인                                                      │
│  $ iostat -x 1                                                      │
│  $ iotop                                                            │
│                                                                     │
│  // 프로세스별 I/O                                                       │
│  $ pidstat -d 1                                                     │
│                                                                     │
│  // 높은 I/O 대기 프로세스                                                │
│  $ ps -eo pid,user,state,wchan:30 | grep D                         │
│                                                                     │
│  // 디스크 상태                                                         │
│  $ smartctl -a /dev/sda                                             │
│  $ smartctl -H /dev/sda                                             │
│                                                                     │
│  【파일 잠금 문제】                                                      │
│  ──────────────────                                                  │
│  // 열린 파일 확인                                                      │
│  $ lsof /data/file                                                  │
│  $ lsof +D /data/                                                   │
│                                                                     │
│  // 파일 사용 프로세스                                                    │
│  $ fuser -v /data/file                                              │
│  $ fuser -k /data/file   // 강제 종료                                 │
│                                                                     │
│  // NFS 잠금 문제                                                      │
│  $ cat /proc/locks                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 시스템 문제 체계적 진단/해결
• 프로세스: 식별 → 분석 → 해결 → 예방
• 진단도구: dmesg, journalctl, df, iostat
• 일반문제: 공간부족, 마운트실패, I/O에러
• 복구도구: fsck, xfs_repair, smartctl
• 공간부족: du, 로그정리, 오래된 파일 삭제
• 권한문제: chmod, chown, ACL, SELinux
• 예방: 모니터링, 백업, UPS, RAID
```
