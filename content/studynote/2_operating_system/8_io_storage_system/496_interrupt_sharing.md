# 496. 디스크 포맷팅 (Disk Formatting)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 저장 매체에 파일 시스템 생성
> 2. **가치**: 데이터 저장 준비와 구조화
> 3. **융합**: 파티션, 파일 시스템, 부팅과 연관

---

## Ⅰ. 개요

### 개념 정의

디스크 포맷팅(Disk Formatting)은 **저장 매체에 데이터를 체계적으로 저장할 수 있도록 파일 시스템 구조를 생성하는 과정**이다.

### 💡 비유: 공책 줄 긋기
디스크 포맷팅은 **공책에 줄 긋기**와 같다. 빈 페이지에 글을 쓸 수 있도록 줄을 그어 구조를 만든다.

### 디스크 포맷팅 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                디스크 포맷팅                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【포맷팅 단계】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  저수준 포맷팅 (Low-Level Format)                                       │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • 물리적 섹터 정의 (하드웨어 수준)                        │    │ │
│  │  │  • 제조사에서 수행                                       │    │ │
│  │  │  • 사용자는 거의 수행하지 않음                             │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  파티셔닝 (Partitioning)                                               │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • 논리적 영역 분할                                       │    │ │
│  │  │  • MBR 또는 GPT 파티션 테이블 생성                         │    │ │
│  │  │  • fdisk, parted 사용                                    │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  고수준 포맷팅 (High-Level Format)                                     │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • 파일 시스템 생성                                       │    │ │
│  │  │  • 메타데이터 구조 초기화                                  │    │ │
│  │  │  • mkfs 명령어 사용                                       │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파일 시스템 생성 구조】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  포맷팅 후 디스크 구조:                                                  │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  [슈퍼블록][아이노드 테이블][데이터 블록][저널]            │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  초기화되는 내용:                                                        │ │
│  │  • 슈퍼블록: 파일시스템 전체 정보                                        │ │
│  │  • 아이노드: 파일 메타데이터 공간                                        │ │
│  │  • 비트맵: 블록/아이노드 사용 상태                                        │ │
│  │  • 루트 디렉토리 생성                                                   │ │
│  │  • 저널 영역 (저널링 FS)                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                디스크 포맷팅 상세                                       │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파일 시스템별 포맷팅】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파일 시스템    명령어           특징                                   │ │
│  │  ──────────    ────────         ────                                   │ │
│  │  ext4          mkfs.ext4        Linux 기본, 저널링                      │ │
│  │  XFS           mkfs.xfs         고성능, 대용량                          │ │
│  │  Btrfs         mkfs.btrfs       COW, 스냅샷                            │ │
│  │  NTFS          mkfs.ntfs        Windows 호환                           │ │
│  │  FAT32         mkfs.vfat        호환성 좋음, 4GB 제한                   │ │
│  │  exFAT         mkfs.exfat       대용량 이동식                          │ │
│  │  ZFS           zpool create     통합 스토리지                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【포맷팅 옵션】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  옵션             설명                    파일시스템                        │ │
│  │  ────             ────                    ────────                        │ │
│  │  -L               볼륨 라벨               전체                           │ │
│  │  -b               블록 크기               ext4, XFS                     │ │
│  │  -I               아이노드 크기            ext4                          │ │
│  │  -j               저널 생성               ext4                          │ │
│  │  -N               아이노드 개수            ext4                          │ │
│  │  -m               예약 블록 비율           ext4                          │ │
│  │  -f               강제 포맷               전체                           │ │
│  │  -c               불량 블록 검사           ext4                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【빠른 포맷 vs 일반 포맷】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  구분            빠른 포맷              일반 포맷                        │ │
│  │  ────            ────────              ────────                        │ │
│  │  시간            빠름                   느림                            │ │
│  │  동작            메타데이터만 초기화      전체 검사 후 초기화              │ │
│  │  불량블록        검사 안 함              검사                           │ │
│  │  데이터 복구     가능성 있음             어려움                          │ │
│  │  안전성          낮음                    높음                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ext4 파일 시스템 생성】                                             │
│  ──────────────────                                                  │
│  // 기본 포맷팅                                                        │
│  $ sudo mkfs.ext4 /dev/sdb1                                         │
│                                                                     │
│  // 라벨 지정                                                          │
│  $ sudo mkfs.ext4 -L "data" /dev/sdb1                               │
│                                                                     │
│  // 블록 크기 지정                                                      │
│  $ sudo mkfs.ext4 -b 4096 /dev/sdb1                                 │
│                                                                     │
│  // 아이노드 크기 지정 (파일이 많은 경우)                                │
│  $ sudo mkfs.ext4 -I 256 /dev/sdb1                                  │
│                                                                     │
│  // 예약 블록 비율 변경                                                 │
│  $ sudo mkfs.ext4 -m 1 /dev/sdb1    // 1% 예약                      │
│                                                                     │
│  // 불량 블록 검사                                                      │
│  $ sudo mkfs.ext4 -c /dev/sdb1                                      │
│                                                                     │
│  【XFS 파일 시스템 생성】                                               │
│  ──────────────────                                                  │
│  // 기본 생성                                                          │
│  $ sudo mkfs.xfs /dev/sdb1                                          │
│                                                                     │
│  // 강제 포맷 (기존 파일시스템 있을 때)                                  │
│  $ sudo mkfs.xfs -f /dev/sdb1                                       │
│                                                                     │
│  // 라벨 지정                                                          │
│  $ sudo mkfs.xfs -L "xfsdata" /dev/sdb1                             │
│                                                                     │
│  // 블록 크기 지정                                                      │
│  $ sudo mkfs.xfs -b size=4k /dev/sdb1                               │
│                                                                     │
│  【Btrfs 파일 시스템 생성】                                             │
│  ──────────────────                                                  │
│  // 기본 생성                                                          │
│  $ sudo mkfs.btrfs /dev/sdb1                                        │
│                                                                     │
│  // 라벨 지정                                                          │
│  $ sudo mkfs.btrfs -L "btrfsdata" /dev/sdb1                         │
│                                                                     │
│  // 여러 장치로 생성 (RAID)                                             │
│  $ sudo mkfs.btrfs -d raid1 -m raid1 /dev/sdb1 /dev/sdc1            │
│                                                                     │
│  【NTFS/FAT/exFAT 생성】                                               │
│  ──────────────────                                                  │
│  // NTFS                                                              │
│  $ sudo mkfs.ntfs -L "ntfsdata" /dev/sdb1                           │
│  $ sudo mkfs.ntfs -Q /dev/sdb1   // 빠른 포맷                        │
│                                                                     │
│  // FAT32                                                             │
│  $ sudo mkfs.vfat -F 32 -n "usb" /dev/sdb1                          │
│                                                                     │
│  // exFAT                                                             │
│  $ sudo mkfs.exfat -n "exfat" /dev/sdb1                             │
│                                                                     │
│  【파일 시스템 정보 확인】                                              │
│  ──────────────────                                                  │
│  // ext4 정보                                                          │
│  $ sudo tune2fs -l /dev/sdb1                                        │
│  $ sudo dumpe2fs /dev/sdb1 | less                                   │
│                                                                     │
│  // XFS 정보                                                           │
│  $ sudo xfs_info /dev/sdb1                                          │
│  $ sudo xfs_admin -l /dev/sdb1   // 라벨 확인                        │
│                                                                     │
│  // Btrfs 정보                                                         │
│  $ sudo btrfs filesystem show /dev/sdb1                             │
│  $ sudo btrfs filesystem df /mnt/data                               │
│                                                                     │
│  // blkid로 확인                                                       │
│  $ sudo blkid /dev/sdb1                                             │
│  /dev/sdb1: UUID="xxx" TYPE="ext4" LABEL="data"                     │
│                                                                     │
│  【라벨/UUID 변경】                                                     │
│  ──────────────────                                                  │
│  // ext4 라벨 변경                                                      │
│  $ sudo e2label /dev/sdb1 newlabel                                  │
│                                                                     │
│  // XFS 라벨 변경                                                       │
│  $ sudo xfs_admin -L newlabel /dev/sdb1                             │
│                                                                     │
│  // UUID 변경                                                          │
│  $ sudo tune2fs -U random /dev/sdb1                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 저장 매체에 파일 시스템 생성
• 단계: 저수준 → 파티셔닝 → 고수준
• 명령어: mkfs.ext4, mkfs.xfs, mkfs.btrfs
• 옵션: -L(라벨), -b(블록), -m(예약)
• 빠른 포맷: 메타데이터만 초기화
• 일반 포맷: 불량 블록 검사 포함
• 주의: 포맷팅 시 데이터 모두 삭제
• 확인: blkid, tune2fs -l, xfs_info
```
