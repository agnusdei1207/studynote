# 505. 메모리 맵드 I/O (Memory-Mapped I/O)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 장치를 메모리 주소 공간에 매핑
> 2. **가치**: 단일 주소 공간과 효율적 접근
> 3. **융합**: 가상 메모리, 캐시, I/O 포트와 연관

---

## Ⅰ. 개요

### 개념 정의

메모리 맵드 I/O(Memory-Mapped I/O, MMIO)는 **I/O 장치의 레지스터를 메모리 주소 공간에 매핑하여 메모리 접근 명령으로 제어하는 방식**이다.

### 💡 비유: 같은 서랍장 사용
메모리 맵드 I/O는 **같은 서랍장**에 옷과 서류를 함께 넣는 것과 같다. 같은 방식으로 꺼내 쓴다.

### 메모리 맵드 I/O 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 맵드 I/O                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주소 공간 구조】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  통합 주소 공간 (Memory-Mapped I/O):                                     │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  0x00000000                                              │    │ │
│  │  │  ┌───────────────────────────────────────────────────┐   │    │ │
│  │  │  │  RAM (일반 메모리)                                  │   │    │ │
│  │  │  │                                                     │   │    │ │
│  │  │  └───────────────────────────────────────────────────┘   │    │ │
│  │  │  ...                                                      │    │ │
│  │  │  ┌───────────────────────────────────────────────────┐   │    │ │
│  │  │  │  ROM (BIOS/UEFI)                                  │   │    │ │
│  │  │  └───────────────────────────────────────────────────┘   │    │ │
│  │  │  ...                                                      │    │ │
│  │  │  ┌───────────────────────────────────────────────────┐   │    │ │
│  │  │  │  MMIO 영역 (I/O 장치 레지스터)                       │   │    │ │
│  │  │  │  • GPU 레지스터                                     │   │    │ │
│  │  │  │  • NIC 레지스터                                     │   │    │ │
│  │  │  │  • NVMe 레지스터                                    │   │    │ │
│  │  │  │  • USB 컨트롤러                                     │   │    │ │
│  │  │  └───────────────────────────────────────────────────┘   │    │ │
│  │  │  0xFFFFFFFF (32비트) / 0xFFFFFFFFFFFFFFFF (64비트)          │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  모든 접근: MOV, LD, ST 등 메모리 명령어 사용                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/O 매핑 vs 메모리 매핑】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  I/O 매핑                      메모리 매핑                               │ │
│  │  ┌─────────────────────┐      ┌─────────────────────┐           │ │
│  │  │ 메모리 공간          │      │ 통합 주소 공간        │           │ │
│  │  │ ┌─────────────────┐ │      │ ┌─────────────────┐ │           │ │
│  │  │ │ RAM             │ │      │ │ RAM             │ │           │ │
│  │  │ └─────────────────┘ │      │ ├─────────────────┤ │           │ │
│  │  ├─────────────────────┤      │ │ MMIO (장치)     │ │           │ │
│  │  │ I/O 공간            │      │ └─────────────────┘ │           │ │
│  │  │ ┌─────────────────┐ │      │                     │           │ │
│  │  │ │ IN/OUT          │ │      │ │ MOV, LD, ST       │           │ │
│  │  │ └─────────────────┘ │      │ │ (메모리 명령)     │           │ │
│  │  └─────────────────────┘      └─────────────────────┘           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 맵드 I/O 상세                                     │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MMIO 장단점】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                                 │ │
│  │  • 단일 주소 공간으로 단순한 설계                                         │ │
│  │  • 모든 메모리 명령어 사용 가능 (MOV, LD, ST 등)                          │ │
│  │  • C 언어에서 포인터로 직접 접근 가능                                     │ │
│  │  • 더 효율적인 주소 지정 모드                                            │ │
│  │  • CPU 설계 단순화                                                      │ │
│  │                                                             │ │
│  │  단점:                                                                 │ │
│  │  • 메모리 주소 공간 일부 소모                                            │ │
│  │  • 캐시 문제 (MMIO는 캐시하면 안 됨)                                     │ │
│  │  • 메모리 보호와 연동 필요                                               │ │
│  │  • 사이드 이펙트 (읽기/쓰기가 상태 변경)                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MMIO 구현 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ARM의 경우:                                                            │ │
│  │  0x00000000 - 0x7FFFFFFF : 일반 메모리                                   │ │
│  │  0x80000000 - 0xFFFFFFFF : MMIO 영역                                    │ │
│  │                                                             │ │
│  │  x86의 경우:                                                            │ │
│  │  I/O 매핑과 메모리 매핑 모두 지원                                         │ │
│  │  PCIe 장치는 MMIO 사용                                                  │ │
│  │  BAR (Base Address Register)로 MMIO 영역 할당                           │ │
│  │                                                             │ │
│  │  RISC-V의 경우:                                                         │ │
│  │  메모리 맵드 I/O만 사용                                                  │ │
│  │  PMA (Physical Memory Attributes)로 MMIO 구분                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 문제와 해결】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  문제:                                                                  │ │
│  │  • MMIO 영역이 캐시되면 하드웨어 상태와 불일치                             │ │
│  │  • 읽기가 실제 장치에서 수행되지 않음                                     │ │
│  │  • 쓰기가 지연되거나 병합됨                                              │ │
│  │                                                             │ │
│  │  해결:                                                                  │ │
│  │  • MMIO 영역은 캐시 비활성화 (Uncacheable)                               │ │
│  │  • 페이지 테이블에서 캐시 속성 설정                                       │ │
│  │  • volatile 키워드로 컴파일러 최적화 방지                                 │ │
│  │  • 메모리 배리어로 순서 보장                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MMIO 영역 확인】                                                      │
│  ──────────────────                                                  │
│  // 메모리 맵핑 확인                                                     │
│  $ cat /proc/iomem                                                   │
│  00000000-00000fff : Reserved                                        │
│  00001000-0009efff : System RAM                                      │
│  000a0000-000bffff : PCI Bus 0000:00                                 │
│  000c0000-000cffff : Video ROM                                       │
│  ...                                                                 │
│  fd000000-fdffffff : 0000:00:02.0   // GPU MMIO                      │
│  fe000000-fe010fff : 0000:00:1f.2   // SATA MMIO                     │
│                                                                     │
│  // 특정 장치 MMIO 영역                                                  │
│  $ lspci -vv | grep -i "Memory at"                                  │
│                                                                     │
│  【Linux 커널 MMIO 사용】                                               │
│  ──────────────────                                                  │
│  // MMIO 영역 매핑                                                      │
│  #include <linux/io.h>                                               │
│  void __iomem *ioremap(phys_addr_t addr, size_t size);              │
│  void iounmap(void __iomem *addr);                                  │
│                                                                     │
│  // MMIO 읽기/쓰기                                                      │
│  unsigned char readb(const void __iomem *addr);                     │
│  unsigned short readw(const void __iomem *addr);                    │
│  unsigned int readl(const void __iomem *addr);                      │
│  void writeb(unsigned char value, void __iomem *addr);              │
│  void writew(unsigned short value, void __iomem *addr);              │
│  void writel(unsigned int value, void __iomem *addr);               │
│                                                                     │
│  // 메모리 배리어                                                       │
│  #define ioread8(addr)        readb(addr)                           │
│  #define iowrite8(val, addr)  writeb(val, addr)                     │
│  #define mmiowb()             // MMIO 쓰기 배리어                      │
│                                                                     │
│  【사용자 공간 MMIO 접근】                                               │
│  ──────────────────                                                  │
│  // /dev/mem을 통한 접근                                                │
│  #include <fcntl.h>                                                  │
│  #include <sys/mman.h>                                               │
│  #include <unistd.h>                                                 │
│                                                                     │
│  int fd = open("/dev/mem", O_RDWR);                                 │
│  void *mmio = mmap(NULL, size, PROT_READ|PROT_WRITE,                │
│                    MAP_SHARED, fd, phys_addr);                       │
│  // volatile로 접근                                                   │
│  volatile uint32_t *reg = (volatile uint32_t *)mmio;                 │
│  uint32_t value = *reg;                                              │
│  *reg = new_value;                                                   │
│  munmap(mmio, size);                                                 │
│  close(fd);                                                          │
│                                                                     │
│  // UIO (Userspace I/O) 드라이버 사용                                    │
│  $ cat /sys/class/uio/uio0/maps/map0/addr                           │
│  $ cat /sys/class/uio/uio0/maps/map0/size                           │
│                                                                     │
│  【PCIe 장치 MMIO 확인】                                                │
│  ──────────────────                                                  │
│  // PCIe BAR 정보                                                      │
│  $ lspci -vv -s 00:02.0                                              │
│  Region 0: Memory at fd000000 (32-bit, non-prefetchable) [size=16M] │
│  Region 2: Memory at d0000000 (64-bit, prefetchable) [size=256M]    │
│                                                                     │
│  // BAR 읽기                                                          │
│  $ setpci -s 00:02.0 BASE_ADDRESS_0                                 │
│                                                                     │
│  【GPIO MMIO 예시 (Raspberry Pi)】                                     │
│  ──────────────────                                                  │
│  // BCM2835 GPIO MMIO                                                 │
│  #define GPIO_BASE 0x3F200000                                        │
│  #define GPIO_SIZE 0xB4                                              │
│                                                                     │
│  volatile uint32_t *gpio = mmap(..., GPIO_BASE);                     │
│  gpio[GPIO_GPFSEL0] |= (1 << 12);  // 출력 모드 설정                   │
│  gpio[GPIO_GPSET0] = (1 << 4);     // GPIO 4 HIGH                    │
│  gpio[GPIO_GPCLR0] = (1 << 4);     // GPIO 4 LOW                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: I/O 장치를 메모리 주소 공간에 매핑
• 특징: 단일 주소 공간, 메모리 명령어 사용
• 장점: 단순한 설계, 풍부한 명령어, 포인터 접근
• 단점: 주소 소모, 캐시 문제, 사이드 이펙트
• 구현: ARM (전용), x86 (혼합), RISC-V (전용)
• 캐시: Uncacheable 속성 필수
• 확인: cat /proc/iomem, lspci -vv
• API: ioremap, readb/readw/readl, writeb/writew/writel
```
