# 485. 중복 제거 (Deduplication)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동일한 데이터의 중복 저장 방지
> 2. **가치**: 저장 공간 절약
> 3. **융합**: 파일 시스템, 백업, 압축과 연관

---

## Ⅰ. 개요

### 개념 정의

중복 제거(Deduplication)는 **동일한 데이터 블록을 한 번만 저장하고 참조만 늘려 저장 공간을 절약하는 기술**이다.

### 💡 비유: 도서관 책
중복 제거는 **도서관의 같은 책**과 같다. 같은 책을 여러 권 두는 대신 한 권만 두고 여러 사람이 대출한다.

### 중복 제거 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                중복 제거                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【중복 제거 없이】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파일 1: [AAAA][BBBB][CCCC]    →  3 블록                            │ │
│  │  파일 2: [AAAA][DDDD][EEEE]    →  3 블록                            │ │
│  │  파일 3: [AAAA][BBBB][FFFF]    →  3 블록                            │ │
│  │                                                             │ │
│  │  총 9 블록 사용                                                       │ │
│  │  AAAA 3번, BBBB 2번 중복                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【중복 제거 후】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  고유 블록 저장:                                                       │ │
│  │  [AAAA] [BBBB] [CCCC] [DDDD] [EEEE] [FFFF] → 6 블록                 │ │
│  │                                                             │ │
│  │  파일 1 → AAAA → BBBB → CCCC                                         │ │
│  │  파일 2 → AAAA → DDDD → EEEE   (AAAA는 참조만)                       │ │
│  │  파일 3 → AAAA → BBBB → FFFF   (AAAA, BBBB는 참조만)                 │ │
│  │                                                             │ │
│  │  총 6 블록 사용 (33% 절약)                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【중복 제거 방식】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 파일 레벨:                                                         │ │
│  │     • 동일한 파일 전체를 비교                                           │ │
│  │     • 구현 간단                                                        │ │
│  │     • 효율 낮음                                                         │ │
│  │                                                             │ │
│  │  2. 블록 레벨 (고정 크기):                                             │ │
│  │     • 고정 크기 블록 (4KB, 8KB)                                        │ │
│  │     • 효율 중간                                                         │ │
│  │                                                             │ │
│  │  3. 가변 길이 (Variable-Length):                                      │ │
│  │     • 콘텐츠 기반 청킹                                                  │ │
│  │     • 효율 높음                                                         │ │
│  │     • 오버헤드 큼                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                중복 제거 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【해시 기반 중복 탐지】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 블록 해시 계산 (SHA-256, MD5 등)                                   │ │
│  │  2. 해시 인덱스 조회                                                   │ │
│  │  3. 동일 해시 → 참조만 생성                                             │ │
│  │  4. 새 해시 → 실제 저장                                                │ │
│  │                                                             │ │
│  │  충돌 가능성: SHA-256은 사실상 없음                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적용 위치】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 인라인 (Inline):                                                   │ │
│  │     • 쓰기 시 실시간 중복 제거                                          │ │
│  │     • 성능 오버헤드                                                     │ │
│  │     • ZFS, Btrfs                                                       │ │
│  │                                                             │ │
│  │  2. 포스트 프로세스 (Post-Process):                                    │ │
│  │     • 쓰기 후 백그라운드에서 처리                                        │ │
│  │     • 쓰기 성능 영향 없음                                               │ │
│  │     • 일시적 중복 존재                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【효율 요인】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  높은 중복 제거 효율:                                                 │ │
│  │  • 백업 데이터 (90%+ 절약)                                             │ │
│  │  • 가상 머신 이미지 (80%+ 절약)                                        │ │
│  │  • 이메일 시스템                                                       │ │
│  │  • 사용자 홈 디렉토리                                                  │ │
│  │                                                             │ │
│  │  낮은 효율:                                                             │ │
│  │  • 암호화된 데이터                                                     │ │
│  │  • 압축된 미디어                                                       │ │
│  │  • 이미 중복 제거된 데이터                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Btrfs 중복 제거】                                                  │
│  ──────────────────                                                  │
│  // 파일 범위 중복 제거                                                 │
│  $ sudo btrfs filesystem dedupe /mnt/btrfs                          │
│                                                                     │
│  // duperemove 도구 사용                                              │
│  $ sudo apt install duperemove                                       │
│  $ sudo duperemove -r /mnt/btrfs                                    │
│  // -r: 재귀적                                                         │
│                                                                     │
│  // 건식 실행 (변경 없이 확인만)                                        │
│  $ sudo duperemove -rn /mnt/btrfs                                   │
│  // -n: dry-run                                                        │
│                                                                     │
│  【ZFS 중복 제거】                                                      │
│  ──────────────────                                                  │
│  // 중복 제거 활성화                                                    │
│  $ sudo zfs set dedup=on pool/dataset                               │
│                                                                     │
│  // 상태 확인                                                          │
│  $ sudo zfs get dedup pool/dataset                                  │
│  NAME         PROPERTY  VALUE  SOURCE                               │
│  pool/dataset dedup     on     local                                │
│                                                                     │
│  // 중복 제거율 확인                                                    │
│  $ sudo zpool list                                                   │
│  NAME   SIZE  ALLOC   FREE  DEDUP                                   │
│  pool   100G  30G    70G   1.50x                                    │
│  // DEDUP 1.50x = 50% 공간 절약                                        │
│                                                                     │
│  // 주의: 메모리 사용량 증가                                            │
│  // 대략 1TB당 1-5GB RAM 필요                                          │
│                                                                     │
│  【VDO (Virtual Data Optimizer)】                                     │
│  ──────────────────                                                  │
│  // RHEL/CentOS에서 사용                                               │
│  $ sudo yum install vdo                                              │
│  $ sudo vdo create --name=vdo1 --device=/dev/sdb                    │
│  $ sudo vdo status --name=vdo1                                       │
│  // 중복 제거 + 압축 자동 적용                                          │
│                                                                     │
│  【fdupes로 중복 파일 찾기】                                           │
│  ──────────────────                                                  │
│  // 설치                                                              │
│  $ sudo apt install fdupes                                           │
│                                                                     │
│  // 중복 파일 검색                                                      │
│  $ fdupes -r /home/user                                             │
│                                                                     │
│  // 중복 파일 삭제 (확인 포함)                                          │
│  $ fdupes -rd /home/user                                            │
│  // -r: 재귀, -d: 삭제                                                 │
│                                                                     │
│  // 요약만 표시                                                        │
│  $ fdupes -rs /home/user                                            │
│                                                                     │
│  【백업에서 중복 제거】                                                │
│  ──────────────────                                                  │
│  // Borg Backup                                                       │
│  $ borg create /backup::$(date +%Y%m%d) /home                       │
│  // 자동으로 중복 제거                                                  │
│                                                                     │
│  // Restic                                                            │
│  $ restic backup /home --repo /backup                               │
│  // 자동 중복 제거                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 동일 데이터 중복 저장 방지
• 원리: 해시 기반 비교, 참조만 생성
• 방식: 파일 레벨, 블록 레벨, 가변 길이
• 위치: 인라인, 포스트 프로세스
• 지원: ZFS, Btrfs, VDO
• 효율: 백업 90%+, VM 80%+
• 주의: 메모리 사용량, CPU 오버헤드
• 도구: duperemove, fdupes, borg, restic
```
