# 597. 심볼릭 링크 (Symbolic Link / Soft Link)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다른 파일의 경로를 가리키는 특수 파일
> 2. **가치**: 유연한 참조, 크로스 파일시스템 지원
> 3. **융합**: 경로, inode, Windows 바로가기와 연관

---

## Ⅰ. 개요

### 개념 정의
**심볼릭 링크(Symbolic Link)**는 **다른 파일이나 디렉토리의 경로를 가리키는 특수 파일**입니다.

### 💡 비유: 바로가기
심볼릭 링크는 **바로가기**와 같습니다. 원본을 가리키는 포인터입니다.

### 심볼릭 링크 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                심볼릭 링크 구조                                        │ |

│                                                                     │
│  【심볼릭 링크 개념】                                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  디렉토리 엔트리                 inode                      │ │   │
│  │  ┌─────────────────┐         ┌─────────────────┐           │ │   │
│  │  │ symlink.txt     │─────────▶│ inode #12346    │           │ │   │
│  │  │ (l)             │         │ • 타입: symlink  │           │ │   │
│  │  │                 │         │ • 크기: 13       │           │ │   │
│  │  │                 │         │ • 데이터: 경로   │           │ │   │
│  │  └─────────────────┘         └────────┬────────┘           │ │   │
│  │                                        │ "original.txt"     │ │   │
│  │                                        ▼                       │ │   │
│  │  ┌─────────────────┐         ┌─────────────────┐              │ │   │
│  │  │ original.txt    │─────────▶│ inode #12345    │              │ │   │
│  │  │ (-)             │         │ • 실제 파일       │              │ │   │
│  │  │                 │         │ • 데이터 블록     │              │ │   │
│  │  └─────────────────┘         └─────────────────┘              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【심볼릭 링크 특징】                                                   │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  특징              설명                                          │ │   │
│  │  ────              ────                                          │ │   │
│  │  새로운 inode      원본과 다른 inode 번호                          │ │   │
│  │  경로 저장         원본 파일의 경로를 데이터로 저장                   │ │   │
│  │  파일 크기         가리키는 경로의 길이                              │ │   │
│  │  권한              777 (실제 권한은 원본 따름)                        │ │   │
│  │  파일시스템         다른 파일시스템 가능                              │ │   │
│  │  디렉토리          디렉토리도 링크 가능                               │ │   │

│  │                                                             │ │   │

│  │  ────            ─────────           ─────────               │ │   │
│  │  inode           동일                다름                     │ │   │

│  │  디렉토리        불가                가능                      │ │   │
│  │  크기            원본과 같음          경로 길이                  │ │   │
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                심볼릭 링크 상세                                        │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【절대 경로 vs 상대 경로】                                             │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  절대 경로:                                                       │ │   │
│  │  $ ln -s /home/user/file.txt symlink.txt                        │ │   │
│  │  장점: 원본 이동해도 링크 유효                                      │ │   │
│  │  단점: 파일시스템 구조 변경 시 문제                                  │ │   │
│  │                                                             │ │   │
│  │  상대 경로:                                                       │ │   │
│  │  $ ln -s ../file.txt symlink.txt                                │ │   │
│  │  장점: 디렉토리 구조 유지 시 유연                                    │ │   │
│  │  단점: 링크 이동 시 끊김                                            │ │   │
│  │                                                             │ │   │
│  │  권장: 상황에 따라 선택                                              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【끊어진 링크 (Dangling Link)】                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  원본 삭제 시:                                                     │ │   │
│  │  $ rm original.txt                                               │ │   │
│  │  $ cat symlink.txt                                               │ │   │
│  │  cat: symlink.txt: No such file or directory                    │ │   │
│  │                                                             │ │   │
│  │  끊어진 링크 찾기:                                                  │ │   │
│  │  $ find /path -xtype l                                          │ │   │
│  │  $ symlinks -d /path    // 끊어진 링크 삭제                        │ │   │
│  │                                                             │ │   │
│  │  ls -l로 확인:                                                    │ │   │
│  │  $ ls -l symlink.txt                                             │ │   │
│  │  lrwxrwxrwx 1 user user 12 Jan 15 symlink.txt -> original.txt   │ │   │
│  │  // 빨간색으로 표시 (원본 없음)                                      │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【심볼릭 링크 권한】                                                   │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  링크 자체 권한: 777 (항상)                                         │ │   │
│  │  실제 접근 권한: 원본 파일의 권한                                    │ │   │
│  │                                                             │ │   │
│  │  $ ls -l symlink.txt                                             │ │   │
│  │  lrwxrwxrwx 1 user user 12 Jan 15 symlink.txt -> file.txt       │ │   │
│  │  //      ↑ 링크 권한 (항상 777)                                     │ │   │
│  │                                                             │ │   │
│  │  $ chmod 600 file.txt    // 원본 권한 변경                          │ │   │
│  │  $ cat symlink.txt       // 원본 권한으로 접근                       │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【심볼릭 링크 생성】                                                  │ |
│  ──────────────────                                                │
│  // 기본 생성                                                         │
│  $ ln -s original.txt symlink.txt                                  │
│                                                                     │
│  // 절대 경로로 생성                                                   │
│  $ ln -s /home/user/file.txt /tmp/link.txt                         │
│                                                                     │
│  // 디렉토리 링크                                                      │
│  $ ln -s /path/to/directory /mnt/link                              │
│                                                                     │
│  // 덮어쓰기                                                          │
│  $ ln -sf original.txt symlink.txt                                 │
│                                                                     │
│  // 확인                                                             │
│  $ ls -la symlink.txt                                               │
│  lrwxrwxrwx 1 user user 12 Jan 15 symlink.txt -> original.txt      │
│  //  ↑ 파일 타입 l (link)        ↑ 가리키는 경로                        │
│                                                                     │
│  $ readlink symlink.txt                                             │
│  original.txt                                                       │
│                                                                     │
│  $ readlink -f symlink.txt    // 절대 경로로 변환                      │
│  /home/user/original.txt                                            │
│                                                                     │
│  【심볼릭 링크 삭제】                                                  │ |
│  ──────────────────                                                │
│  // 링크만 삭제                                                        │
│  $ rm symlink.txt                                                   │
│  $ unlink symlink.txt                                               │
│                                                                     │
│  // 원본은 유지됨                                                      │
│  $ ls original.txt                                                  │
│                                                                     │
│  // 주의: 슬래시 포함 시 디렉토리 내용 삭제 가능                         │
│  $ rm -r symlink/     // 링크가 가리키는 디렉토리 내용 삭제             │
│  $ rm -r symlink      // 링크만 삭제 (권장)                            │
│                                                                     │
│  【실무 활용 예시】                                                    │ |
│  ──────────────────                                                │
│  // 버전 관리                                                          │
│  $ ln -s /opt/app-v2.0 /opt/app                                    │
│  $ ln -s /opt/app/current /opt/app/active                          │
│                                                                     │
│  // 라이브러리 버전                                                    │
│  $ ln -s libfoo.so.1.2.3 libfoo.so.1                               │
│  $ ln -s libfoo.so.1 libfoo.so                                      │
│                                                                     │
│  // 로그 파일                                                         │
│  $ ln -s /var/log/app/current.log /tmp/app.log                     │
│                                                                     │
│  // 설정 파일                                                         │
│  $ ln -s ~/dotfiles/.bashrc ~/.bashrc                              │
│  $ ln -s ~/dotfiles/.vimrc ~/.vimrc                                │
│                                                                     │
│  // chroot 환경                                                       │
│  $ ln -s /lib64 /chroot/lib64                                      │
│                                                                     │
│  【끊어진 링크 관리】                                                   │ |
│  ──────────────────                                                │
│  // 끊어진 링크 찾기                                                    │
│  $ find /path -xtype l                                             │
│  $ find /path -type l ! -exec test -e {} \; -print                 │
│                                                                     │
│  // 끊어진 링크 삭제                                                    │
│  $ find /path -xtype l -delete                                     │
│                                                                     │
│  // symlinks 유틸리티                                                  │
│  $ sudo apt install symlinks                                        │
│  $ symlinks -r /path      // 재귀적 확인                              │
│  $ symlinks -d /path      // 끊어진 링크 삭제                          │
│  $ symlinks -c /path      // 절대 경로로 변환                          │
│                                                                     │
│  【심볼릭 링크 복사】                                                   │ |
│  ──────────────────                                                │
│  // cp -P: 링크 자체 복사                                              │
│  $ cp -P symlink.txt copy.txt                                      │
│  // copy.txt도 같은 원본을 가리키는 링크                                │
│                                                                     │
│  // cp -L: 원본 파일 복사                                              │
│  $ cp -L symlink.txt copy.txt                                      │
│  // copy.txt는 원본 내용의 복사본                                       │
│                                                                     │
│  // cp -d: 링크 유지 (-P --preserve=links)                            │
│  $ cp -d symlink.txt copy.txt                                      │
│                                                                     │
│  // rsync                                                            │
│  $ rsync -l src/ dest/    // 링크로 복사                               │
│  $ rsync -L src/ dest/    // 원본으로 복사                              │
│                                                                     │
│  【Python으로 심볼릭 링크】                                            │ |
│  ──────────────────                                                │
│  import os                                                           │
│                                                                     │
│  # 심볼릭 링크 생성                                                    │
│  os.symlink('original.txt', 'symlink.txt')                          │
│                                                                     │
│  # 링크 읽기                                                          │
│  target = os.readlink('symlink.txt')                                │
│  print(f"Points to: {target}")                                       │
│                                                                     │
│  # 링크 확인                                                          │
│  os.path.islink('symlink.txt')  # True                              │
│  os.path.isfile('symlink.txt')  # True (원본이 파일이면)               │
│                                                                     │
│  # realpath                                                          │
│  real = os.path.realpath('symlink.txt')                             │
│                                                                     │
│  【Windows 심볼릭 링크】                                               │ |
│  ──────────────────                                                │
│  // 관리자 권한 필요                                                    │
│  > mklink link.txt target.txt           // 파일                      │
│  > mklink /D link_dir target_dir        // 디렉토리                  │
│  > mklink /J junction_dir target_dir    // 접합점                    │
│                                                                     │
│  // PowerShell                                                        │
│  > New-Item -ItemType SymbolicLink -Path link -Target target        │
│  > New-Item -ItemType Junction -Path junction -Target target        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 다른 파일의 경로를 가리키는 특수 파일
• 생성: ln -s 원본 심볼릭링크
• inode: 원본과 다름
• 파일 타입: l (link)
• 권한: 777 (실제는 원본 권한)
• 크기: 경로 문자열 길이
• 장점: 크로스 파일시스템, 디렉토리 링크 가능
• 단점: 원본 삭제 시 끊김
• 끊어진 링크: find -xtype l
• 삭제: rm, unlink
• 확인: ls -l, readlink
• 절대 경로: readlink -f
• cp -P: 링크 복사
• cp -L: 원본 복사
• Windows: mklink
```
