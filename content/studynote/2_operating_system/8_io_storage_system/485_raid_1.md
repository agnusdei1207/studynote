## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RAID 1은 두 개 이상의 디스크에 동일한 데이터를 동시에 기록하는 '미러링(Mirroring)' 기술을 통해 데이터의 완전한 중복성을 확보하는 방식이다.
> 2. **안정성**: 하나의 디스크만 살아있어도 데이터 접근이 가능하므로 가용성이 매우 높으며, 디스크 장애 시 복구 과정(Rebuilding)이 매우 단순하고 빠르다.
> 3. **비율**: 전체 물리 용량의 50%만 사용할 수 있다는 비용적 단점이 있으나, 쓰기 성능 저하를 감수하고라도 데이터 보호가 최우선인 시스템에 필수적이다.

---

## Ⅰ. 개요 (Context & Background)

- **정의**: '복제(Copy)'를 통해 데이터 신뢰성을 극대화한 RAID 레벨이다.
- **배경**: 하드웨어는 언제든 고장 날 수 있다는 가정 하에, 가장 확실한 보호 방법인 '똑같은 복사본 실시간 유지'를 구현하기 위해 탄생했다.
- **💡 비유**: **"두 권의 일기장"**과 같다. 일기를 쓸 때마다 똑같은 내용을 두 권의 일기장에 동시에 적는 것과 같다. 한 권을 잃어버려도 다른 한 권이 완벽하게 남아있으므로 내 기억(데이터)은 안전하다. 하지만 종이(디스크 용량)를 두 배로 써야 한다는 점이 특징이다.

- **📢 섹션 요약 비유**: "가장 고전적이면서도 가장 확실한 데이터 보험"입니다.

---

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### RAID 1 미러링 메커니즘 (ASCII Diagram)

```text
  [ 호스트 쓰기 요청: 데이터 A ]
           │
  ┌────────▼────────┐
  │  RAID 컨트롤러  │ (데이터 복제)
  └────────┬────────┘
           │
  ┌────────┴──────────────────┐
  │         [ Mirroring ]     │
  │   Disk 1       Disk 2     │
  │  ┌──────┐     ┌──────┐    │
  │  │ 블록 A │     │ 블록 A │    │  (동일 데이터 동시 기록)
  │  ├──────┤     ├──────┤    │
  │  │ 블록 B │     │ 블록 B │    │  (동일 데이터 동시 기록)
  │  └──────┘     └──────┘    │
  │                           │
  │  (논리적 용량 = D1 or D2)  │
  └───────────────────────────┘
```

**[다이어그램 해설]** 호스트가 1개의 데이터를 보내면 컨트롤러가 이를 복제하여 두 디스크의 동일한 위치에 쓴다. 읽기 시에는 두 디스크 중 더 빨리 응답하는 쪽에서 가져오거나, 번갈아 가져오는 방식으로 성능을 소폭 향상시킬 수 있다.

- **📢 섹션 요약 비유**: "왼손과 오른손에 똑같은 보물을 하나씩 쥐어주는 쌍둥이 전략"과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### RAID 1의 특징 상세

| 항목 | 특성 | 비고 |
|:---|:---|:---|
| **최소 디스크 수** | 2개 | 통상 2개로 구성 |
| **읽기 성능** | 우수 (병렬 읽기 가능) | 이론적으로 단일 디스크의 2배 가능 |
| **쓰기 성능** | 단일 디스크와 유사/낮음 | 두 번 써야 하므로 약간의 지연 발생 |
| **공간 효율성** | **50% (1/n)** | 디스크 2개 중 1개 분량만 사용 가능 |
| **장애 허용** | n-1개까지 가능 | 하나만 살아있어도 무중단 서비스 가능 |

### 성능 및 안정성 균형
- **읽기 최적화**: 컨트롤러가 똑똑하다면 요청을 분산시켜 RAID 0에 근접한 읽기 속도를 낼 수 있다.
- **복구 용이성**: 패리티 연산이 필요한 RAID 5/6와 달리, 단순히 살아있는 디스크의 내용을 새 디스크로 복사만 하면 되므로 시스템 부하가 적다.

- **📢 섹션 요약 비유**: "연산(패리티)의 복잡함을 버리고 물량(복제)으로 밀어붙이는 정직한 안정성"입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **OS 부팅 볼륨**: 서버가 켜지지 않으면 아무것도 못 하므로, 부팅용 드라이브(C드라이브나 Root Partition)는 무조건 RAID 1로 구성하는 것이 업계 표준이다.
2. **데이터베이스 로그 (WAL)**: DB의 트랜잭션 로그는 데이터 복구의 최후 보루이므로, 성능보다는 안정성이 극대화된 RAID 1 공간에 저장한다.
3. **소규모 오피스 서버**: 관리가 어려운 환경에서는 복구가 가장 쉬운 RAID 1이 선호된다.

### 도입 체크리스트
- **비용 감수**: 용량 절반을 포기할 만큼 데이터가 소중한가?
- **디스크 이종 구성**: 동일 제조사, 동일 주차의 디스크를 쓰면 비슷한 시기에 같이 고장 날 수 있으므로, 제조사를 섞거나 시차를 두고 구매하는 전략도 고려할 만하다.
- **모니터링**: 디스크 하나가 고장 나도 시스템은 멀쩡히 돌아가므로, 고장 사실을 즉시 알 수 있는 알림 설정이 필수다.

### 안티패턴
- **대용량 미디어 저장**: 영화나 단순 백업 데이터처럼 용량이 크고 중요도가 상대적으로 낮은 데이터를 RAID 1로 구성하는 것은 예산 낭비다. 이럴 때는 RAID 5/6가 더 적합하다.

- **📢 섹션 요약 비유**: "가장 중요한 열쇠는 두 개를 만들어 서로 다른 주머니에 넣어야 하지만, 모든 소지품을 두 개씩 들고 다닐 필요는 없습니다."

## Ⅴ. 기대효과 및 결론

### 기대효과
- **심리적 안정감**: 하드웨어 고장에 대한 공포로부터 해방.
- **무중단 운영 (Business Continuity)**: 서비스 중단 없이 즉각적인 장애 대응 가능.

### 미래 전망
- **클라우드 복제 (Replication)**: 로컬 RAID 1을 넘어, 지리적으로 떨어진 데이터센터 간에 데이터를 복제하는 'Geo-Replication' 기술로 개념이 확장되고 있다.
- **VROC (Virtual RAID on CPU)**: 별도 RAID 카드 없이 CPU 직결 NVMe를 RAID 1로 묶는 기술이 보편화되면서 지연 시간까지 잡고 있다.

- **📢 섹션 요약 비유**: "공간을 내주고 시간을 번다, 그리고 그 시간은 곧 시스템의 생명줄이 된다."

## 📌 관련 개념 맵 (Knowledge Graph)
- **RAID 0 (스트라이핑)**: 성능을 위해 안정성을 버린 모델.
- **RAID 10 (1+0)**: RAID 1의 안정성과 RAID 0의 성능을 모두 잡으려는 확장 모델.
- **Duplication (이중화)**: IT 인프라 전반의 고가용성을 위한 기본 설계 원칙.

## 👶 어린이를 위한 3줄 비유 설명
1. **RAID 1**은 소중한 보물을 똑같은 상자 두 개에 나눠 담아두는 거예요.
2. 한 상자를 도둑맞거나 잃어버려도, 다른 상자가 남아있으니 전혀 걱정할 필요가 없죠.
3. 비록 상자가 두 개 필요해서 돈은 좀 더 들지만, 절대로 잃어버리면 안 되는 소중한 추억을 지키기에 최고랍니다!
