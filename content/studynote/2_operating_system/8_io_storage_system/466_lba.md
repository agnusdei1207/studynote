# 466. FAT 파일 시스템 (File Allocation Table)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단순한 파일 할당 테이블 기반 시스템
> 2. **가치**: 호환성과 단순성
> 3. **융합**: NTFS, USB, 플래시 메모리와 연관

---

## Ⅰ. 개요

### 개념 정의

FAT 파일 시스템(File Allocation Table)은 **파일 할당 테이블을 사용하여 클러스터 체인으로 파일을 관리하는 단순한 파일 시스템**이다.

### 💡 비유: 체인 링크
FAT는 **체인 링크**와 같다. 각 클러스터가 다음 클러스터를 가리키며 파일 전체를 연결한다.

### FAT 파일 시스템 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                FAT 파일 시스템                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FAT 구조】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │              부트 섹터 (Boot Sector)                      │    │ │
│  │  │  • BPB (BIOS Parameter Block)                            │    │ │
│  │  │  • 파일 시스템 정보                                        │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │           FAT (File Allocation Table)                    │    │ │
│  │  │  • FAT1: 메인 할당 테이블                                  │    │ │
│  │  │  • FAT2: 백업 (FAT1의 복사본)                              │    │ │
│  │  │  • 각 엔트리: 다음 클러스터 번호                            │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │              루트 디렉토리 (Root Directory)                │    │ │
│  │  │  • 디렉토리 엔트리 배열                                    │    │ │
│  │  │  • FAT12/16: 고정 위치                                    │    │ │
│  │  │  • FAT32: 데이터 영역 내                                   │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │              데이터 영역 (Data Area)                       │    │ │
│  │  │  • 파일 데이터와 서브디렉토리                               │    │ │
│  │  │  • 클러스터 단위 할당                                       │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FAT 클러스터 체인】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  디렉토리 엔트리:                                                        │
│  │  ┌───────────────────────────────────────┐                       │ │
│  │  │ 파일명 | 시작 클러스터 | 크기 | ...     │                       │ │
│  │  │ FILE.TXT |    5      | 3 클러스터 |    │                       │ │
│  │  └───────────────────────────────────────┘                       │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  FAT 테이블:                                                           │
│  │  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐                       │ │
│  │  │ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │...│                       │ │
│  │  ├───┼───┼───┼───┼───┼───┼───┼───┼───┼───┤                       │ │
│  │  │ - │ - │ - │ - │ - │ 6 │ 8 │ - │EOF│   │ ← 체인: 5→6→8→EOF     │ │
│  │  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘                       │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  데이터 영역:                                                           │
│  │  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐          │ │
│  │  │ ... │ ... │ ... │ ... │ ... │DATA1│DATA2│ ... │DATA3│          │ │
│  │  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘          │ │
│  │     0     1     2     3     4     5     6     7     8             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                FAT 상세                                               │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FAT 종류】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  종류      엔트리 크기  최대 볼륨      용도                              │ │
│  │  ────      ─────────  ─────────      ────                              │ │
│  │  FAT12     12 bit     16 MB         플로피 디스크                     │ │
│  │  FAT16     16 bit     2 GB (4KB 클러스터)  USB, 구형 시스템            │ │
│  │  FAT32     32 bit*    2 TB (4KB 클러스터)  USB, SD 카드               │ │
│  │  exFAT     64 bit     128 PB*        대용량 플래시 메모리              │ │
│  │                                                             │ │
│  │  * FAT32: 실제로는 28비트 사용                                         │ │
│  │  * exFAT: 64비트 클러스터 번호 지원                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【FAT 특수 값】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  값 (FAT32)          의미                                               │ │
│  │  ──────────          ────                                               │ │
│  │  0x00000000          사용 가능 (빈 클러스터)                              │ │
│  │  0x0FFFFFF7          배드 클러스터                                       │ │
│  │  0x0FFFFFF8-0x0FFFFFFF  예약                                            │ │
│  │  0x0FFFFFFF          파일 끝 (EOF)                                       │ │
│  │  기타                 다음 클러스터 번호                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                               │
│  │  • 단순한 구조, 구현 쉬움                                              │ │
│  │  • 높은 호환성 (모든 OS 지원)                                          │ │
│  │  • 적은 오버헤드                                                      │ │
│  │  • 플래시 메모리에 적합                                                │ │
│  │                                                             │ │
│  │  단점:                                                               │
│  │  • 단편화 심각                                                        │ │
│  │  • 파일 크기 제한 (FAT32: 4GB)                                        │ │
│  │  • 보안 기능 없음 (권한 없음)                                          │ │
│  │  • 저널링 없음 (복구 느림)                                             │ │
│  │  • 대용량에서 성능 저하                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FAT 파일 시스템 생성】                                            │
│  ──────────────────                                                  │
│  // FAT32 생성                                                        │
│  $ sudo mkfs.vfat -F 32 /dev/sdb1                                   │
│                                                                     │
│  // exFAT 생성                                                        │
│  $ sudo mkfs.exfat /dev/sdb1                                        │
│                                                                     │
│  // FAT16 생성                                                        │
│  $ sudo mkfs.vfat -F 16 /dev/sdb1                                   │
│                                                                     │
│  // 볼륨 라벨 설정                                                      │
│  $ sudo mkfs.vfat -F 32 -n MYUSB /dev/sdb1                         │
│                                                                     │
│  【FAT 파일 시스템 마운트】                                          │
│  ──────────────────                                                  │
│  // 기본 마운트                                                        │
│  $ sudo mount /dev/sdb1 /mnt/usb                                    │
│                                                                     │
│  // 읽기 전용 마운트                                                    │
│  $ sudo mount -o ro /dev/sdb1 /mnt/usb                             │
│                                                                     │
│  // 코드페이지 지정 (한글 파일명)                                        │
│  $ sudo mount -o iocharset=utf8 /dev/sdb1 /mnt/usb                  │
│                                                                     │
│  【FAT 정보 확인】                                                    │
│  ──────────────────                                                  │
│  // 파일 시스템 타입 확인                                               │
│  $ sudo blkid /dev/sdb1                                             │
│  /dev/sdb1: UUID="..." TYPE="vfat"                                  │
│                                                                     │
│  // 상세 정보                                                          │
│  $ sudo dosfsck -v /dev/sdb1                                        │
│  dosfsck 3.0.20 (Fat File System)                                   │
│  Checking we can access the last sector of the filesystem           │
│  FAT32, 3999 MB, 4 KB clusters                                     │
│                                                                     │
│  // 사용량 확인                                                        │
│  $ df -Th /mnt/usb                                                   │
│  Filesystem     Type  Size  Used Avail Use% Mounted on              │
│  /dev/sdb1      vfat   4G   500M   3.5G  13% /mnt/usb               │
│                                                                     │
│  【FAT 복구】                                                          │
│  ──────────────────                                                  │
│  // 검사만 (수정 없음)                                                 │
│  $ sudo dosfsck -n /dev/sdb1                                        │
│                                                                     │
│  // 자동 복구                                                          │
│  $ sudo dosfsck -a /dev/sdb1                                        │
│                                                                     │
│  // 전체 검사                                                          │
│  $ sudo dosfsck -r /dev/sdb1                                        │
│                                                                     │
│  【윈도우에서 FAT 관리】                                              │
│  ──────────────────                                                  │
│  // 포맷                                                              │
│  > format E: /FS:FAT32 /Q                                          │
│  > format E: /FS:exFAT /Q                                          │
│                                                                     │
│  // 검사                                                              │
│  > chkdsk E: /F                                                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 할당 테이블 기반 파일 시스템
• 구조: 부트섹터 + FAT + 루트디렉토리 + 데이터영역
• 종류: FAT12, FAT16, FAT32, exFAT
• 체인: 클러스터가 다음 클러스터 가리킴
• 장점: 단순, 호환성 높음
• 단점: 단편화, 4GB 제한(FAT32), 보안 없음
• 용도: USB, SD카드, 임베디드
• 관리: mkfs.vfat, dosfsck
```
