# 586. 단편화 (Fragmentation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일이 분산되어 저장되는 현상
> 2. **가치**: 성능 저하 원인, 최적화 필요
> 3. **융합**: 파일시스템, 블록 할당, 조각 모음과 연관

---

## Ⅰ. 개요

### 개념 정의
**단편화(Fragmentation)**는 **파일이 연속되지 않은 블록에 분산 저장되는 현상**입니다.

### 💡 비유: 찢어진 문서
단편화는 **찢어진 문서**와 같습니다. 한 문서가 여러 조각으로 흩어져 있습니다.

## Ⅱ. 상세 분석
### 상세 분석
```

│                단편화 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【단편화 영향】                                                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  영향                설명                                          │ │   │
│  │  ────                ────                                          │ │   │
│  │  읽기 성능 저하        헤드 이동 증가 (HDD)                          │ │   │
│  │  쓰기 성능 저하        블록 검색 시간 증가                           │ │   │
│  │  디스크 수명           SSD wear leveling 영향                       │ │   │
│  │  응답 시간            I/O 지연 증가                                │ │   │
│  │                                                             │ │   │
│  │  SSD는 단편화 영향 적음:                                          │ │   │
│  │  • 랜덤 액세스 성능이 HDD와 다름                                    │ │   │
│  │  • TRIM으로 공간 관리                                              │ │   │
│  │  • Wear leveling이 내장                                            │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【단편화 방지 기법】                                                  │

│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │

│  │  ────                ────                    ────────────        │ │   │

│  │  지연 할당           쓰기 시점 할당           ext4, XFS           │ │   │
│  │  사전 할당           공간 미리 예약           XFS                 │ │   │
│  │  블록 그룹           관련 데이터 그룹화        ext4                │ │   │
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【단편화 확인】                                                      │
│  ──────────────────                                                │
│  // ext4 단편화 확인                                                  │
│  $ sudo e4defrag -c /dev/sda1                                      │
│  Filesystem Type: ext4                                             │
│  Fragmentation score: 5                                            │
│  [0-30: good, 31-55: OK, 56-: bad]                                 │
│                                                                     │
│  // 특정 디렉토리                                                      │
│  $ sudo e4defrag -c /home/user                                     │
│                                                                     │
│  // XFS 단편화 확인                                                    │
│  $ sudo xfs_db -c "frag -f" /dev/sda1                              │
│                                                                     │
│  // 파일 단편화 상세                                                    │
│  $ sudo filefrag -v /path/to/file                                  │
│  $ sudo filefrag /path/to/file                                     │
│                                                                     │
│  // 디렉토리 전체                                                       │
│  $ sudo find /path -type f -exec filefrag {} \; | awk '{sum+=$2} END{print sum/NR}'│
│                                                                     │
│  【단편화 해결 - 조각 모음】                                            │ |
│  ──────────────────                                                │
│  // ext4 조각 모음                                                    │
│  $ sudo e4defrag /dev/sda1                                         │
│  $ sudo e4defrag /home/user                                        │
│  $ sudo e4defrag -v /dev/sda1          // 상세 출력                  │
│                                                                     │
│  // XFS 조각 모음                                                      │
│  $ sudo xfs_fsr /dev/sda1              // 전체                       │
│  $ sudo xfs_fsr /path/to/file          // 단일 파일                  │
│  $ sudo xfs_fsr -v /dev/sda1           // 상세 출력                  │
│                                                                     │
│  // Btrfs 밸런스                                                       │
│  $ sudo btrfs balance start /mnt/btrfs                             │
│  $ sudo btrfs balance start -dusage=50 /mnt/btrfs                  │
│                                                                     │
│  【Windows 조각 모음】                                                │ |
│  ──────────────────                                                │
│  // 명령줄                                                            │
│  > defrag C:                                                        │
│  > defrag C: /O                     // 최적화 포함                    │
│  > defrag C: /A                     // 분석만                        │
│  > defrag C: /V                     // 상세 출력                      │
│                                                                     │
│  // PowerShell                                                       │
│  > Optimize-Volume -DriveLetter C -Defrag                          │
│  > Optimize-Volume -DriveLetter C -Analyze                         │
│                                                                     │
│  // SSD (TRIM)                                                       │
│  > defrag C: /L                     // TRIM 실행                    │
│  > Optimize-Volume -DriveLetter C -Trim                            │
│                                                                     │
│  【단편화 방지 설정】                                                  │ |
│  ──────────────────                                                │
│  // ext4 지연 할당 (기본 활성화)                                        │
│  $ sudo mount -o delalloc /dev/sda1 /mnt                           │
│                                                                     │
│  // XFS 사전 할당                                                      │
│  // fallocate 사용                                                    │
│  $ fallocate -l 1G /path/to/file                                   │
│                                                                     │
│  // XFS allocsize 옵션                                                │
│  $ sudo mount -o allocsize=64m /dev/sda1 /mnt                      │
│                                                                     │
│  【정기적 조각 모음 설정】                                              │ |
│  ──────────────────                                                │
│  // cron으로 주간 실행                                                 │
│  $ sudo crontab -e                                                  │
│  0 3 * * 0 /usr/sbin/e4defrag /home > /var/log/defrag.log 2>&1    │
│                                                                     │
│  // systemd timer                                                    │
│  // /etc/systemd/system/defrag.service                              │
│  [Unit]                                                             │
│  Description=Weekly defragmentation                                 │
│                                                                     │
│  [Service]                                                          │
│  Type=oneshot                                                       │
│  ExecStart=/usr/sbin/e4defrag /home                                 │
│                                                                     │
│  // /etc/systemd/system/defrag.timer                                │
│  [Unit]                                                             │
│  Description=Weekly defragmentation timer                           │
│                                                                     │
│  [Timer]                                                            │
│  OnCalendar=weekly                                                  │
│                                                                     │
│  [Install]                                                          │
│  WantedBy=timers.target                                             │
│                                                                     │
│  $ sudo systemctl enable defrag.timer                               │
│                                                                     │
│  【SSD 특별 고려사항】                                                 │ |
│  ──────────────────                                                │
│  // SSD는 조각 모음 필요 없음 (또는 최소화)                              │
│  // TRIM 실행                                                         │
│  $ sudo fstrim /                                                    │
│  $ sudo fstrim -a                   // 모든 파일시스템                 │
│  $ sudo fstrim -v /                 // 상세 출력                      │
│                                                                     │
│  // 주기적 TRIM 설정 (fstrim.timer)                                   │
│  $ sudo systemctl enable fstrim.timer                              │
│  $ sudo systemctl start fstrim.timer                               │
│                                                                     │
│  // 연속 TRIM (discard 옵션)                                          │
│  $ sudo mount -o discard /dev/sda1 /mnt                            │
│  // 또는 /etc/fstab에 discard 옵션 추가                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 파일이 분산 저장되는 현상
• 종류: 내부 단편화, 외부 단편화, 파일 단편화
• 영향: 읽기/쓰기 성능 저하
• FAT32: 단편화 높음
• ext4/XFS: 단편화 낮음 (extent, 지연 할당)
• Btrfs/ZFS: 단편화 매우 낮음 (COW)
• 방지: extent, 지연 할당, 사전 할당
• ext4 조각 모음: e4defrag
• XFS 조각 모음: xfs_fsr
• 확인: e4defrag -c, filefrag -v
• SSD: TRIM, 조각 모음 불필요
• Windows: defrag, Optimize-Volume
```
