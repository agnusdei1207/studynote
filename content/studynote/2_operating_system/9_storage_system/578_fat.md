# 578. FAT (File Allocation Table)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 간단한 파일시스템, 높은 호환성
> 2. **가치**: 광범위한 지원, 임베디드 적합
> 3. **융합**: 클러스터, 체인, exFAT와 연관

---

## Ⅰ. 개요

### 개념 정의
**FAT(File Allocation Table)**는 **클러스터 체인으로 파일을 관리하는 간단한 파일시스템**입니다.

### 💡 비유: 연결된 문서
FAT는 **연결된 문서**와 같습니다. 각 페이지가 다음 위치를 가리킵니다.

### FAT 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                FAT 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FAT 종류】                                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  종류       엔트리 크기   최대 볼륨      용도                   │ │   │
│  │  ────       ─────────   ─────────      ────                   │ │   │
│  │  FAT12      12-bit      32 MB         플로피                  │ │   │
│  │  FAT16      16-bit      2 GB (기본)    소형 플래시              │ │   │
│  │  FAT32      32-bit      32 GB (기본)    USB, SD카드            │ │   │
│  │  exFAT      64-bit      128 PB        대용량 이동식             │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【FAT 디스크 구조】                                                  │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  ┌──────────────────────────────────────────────────────┐   │ │   │
│  │  │            FAT 볼륨 구조                               │   │ │   │
│  │  ├──────────────────────────────────────────────────────┤   │ │   │
│  │  │  [예약 영역] [FAT1] [FAT2] [루트 디렉토리] [데이터 영역]   │   │ │   │
│  │  │      ↑          ↑      ↑         ↑              ↑        │   │ │   │
│  │  │   부트섹터    FAT복사   백업     고정위치        가변      │   │ │   │
│  │  └──────────────────────────────────────────────────────┘   │ │   │
│  │                                                             │ │   │
│  │  • FAT1, FAT2: 중복 저장 (복구용)                              │ │   │
│  │  • 루트 디렉토리: FAT12/16은 고정 크기, FAT32는 가변             │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【FAT 엔트리】                                                       │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  FAT 테이블 예시:                                               │ │   │
│  │  ┌────────────────────────────────────┐                       │ │   │
│  │  │ 클러스터 │ 값      │ 의미            │                       │ │   │
│  │  │    0    │ -      │ 예약             │                       │ │   │
│  │  │    1    │ -      │ 예약             │                       │ │   │
│  │  │    2    │ 5      │ 다음 클러스터: 5   │                       │ │   │
│  │  │    3    │ 0xFFF  │ 파일 끝 (EOF)     │                       │ │   │
│  │  │    4    │ 0      │ 사용 안 함        │                       │ │   │
│  │  │    5    │ 6      │ 다음 클러스터: 6   │                       │ │   │
│  │  │    6    │ 0xFFF  │ 파일 끝 (EOF)     │                       │ │   │
│  │  └────────────────────────────────────┘                       │ │   │
│  │                                                             │ │   │
│  │  파일 A: 클러스터 2 → 5 → 6 (3개 클러스터 사용)                  │ │   │
│  │  파일 B: 클러스터 3 (1개 클러스터 사용)                          │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                FAT 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【디렉토리 엔트리】                                                   │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  FAT 디렉토리 엔트리 (32 bytes):                                 │ │   │
│  │  ┌────────────────────────────────────────────────────┐      │ │   │
│  │  │ 필드             크기       설명                       │      │ │   │
│  │  │ ─────             ────       ────                       │      │ │   │
│  │  │ 파일명            8 bytes    파일 이름                   │      │ │   │
│  │  │ 확장자            3 bytes    파일 확장자                 │      │ │   │
│  │  │ 속성             1 byte     읽기전용, 숨김, 시스템       │      │ │   │
│  │  │ 예약             10 bytes   예약                        │      │ │   │
│  │  │ 시간             2 bytes    마지막 수정 시간             │      │ │   │
│  │  │ 날짜             2 bytes    마지막 수정 날짜             │      │ │   │
│  │  │ 시작 클러스터      2 bytes    첫 번째 클러스터 번호        │      │ │   │
│  │  │ 파일 크기         4 bytes    파일 크기 (bytes)           │      │ │   │
│  │  └────────────────────────────────────────────────────┘      │ │   │
│  │                                                             │ │   │
│  │  Long Filename (LFN):                                          │ │   │
│  │  • 유니코드 파일명 지원                                         │ │   │
│  │  • 255자까지 가능                                               │ │   │
│  │  • 여러 디렉토리 엔트리 사용                                     │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【FAT32 제한】                                                       │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  제한              값              비고                        │ │   │
│  │  ────              ────              ────                     │ │   │
│  │  최대 파일 크기      4 GB - 1          32-bit 파일 크기          │ │   │
│  │  최대 볼륨          32 GB (기본)        포맷 도구에 따라 다름       │ │   │
│  │  최대 클러스터       ~268 million                                   │ │   │
│  │  최대 파일 수        ~65,517 (루트)     FAT32는 무제한             │ │   │
│  │                                                             │ │   │
│  │  단점:                                                         │ │   │
│  │  • 보안 기능 없음 (ACL 미지원)                                   │ │   │
│  │  • 저널링 없음                                                  │ │   │
│  │  • 단편화 문제                                                   │ │   │
│  │  • 4GB 파일 크기 제한                                            │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【exFAT (Extended FAT)】                                            │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  특징:                                                         │ │   │
│  │  • 64-bit 파일 크기 (최대 16EB)                                 │ │   │
│  │  • 최대 볼륨 128 PB                                             │ │   │
│  │  • 단일 파일 크기 제한 없음                                       │ │   │
│  │  • FAT32 대비 적은 오버헤드                                      │ │   │
│  │  • 윈도우, macOS, Linux 지원                                     │ │   │
│  │                                                             │ │   │
│  │  용도:                                                         │ │   │
│  │  • 대용량 SD카드 (64GB 이상)                                      │ │   │
│  │  • USB 드라이브                                                 │ │   │
│  │  • 이동식 저장 장치                                              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FAT32 파일시스템 생성】                                            │
│  ──────────────────                                                │
│  // Linux                                                            │
│  $ sudo mkfs.vfat -F 32 /dev/sdb1                                   │
│  $ sudo mkfs.vfat -F 32 -n MYUSB /dev/sdb1    // 라벨 지정            │
│                                                                     │
│  // FAT16                                                            │
│  $ sudo mkfs.vfat -F 16 /dev/sdb1                                   │
│                                                                     │
│  // exFAT                                                            │
│  $ sudo mkfs.exfat /dev/sdb1                                        │
│  $ sudo mkexfatfs -n MYUSB /dev/sdb1                                │
│                                                                     │
│  【FAT 마운트】                                                       │
│  ──────────────────                                                │
│  // Linux                                                            │
│  $ sudo mount /dev/sdb1 /mnt/usb                                    │
│  $ sudo mount -t vfat /dev/sdb1 /mnt/usb                            │
│  $ sudo mount -t vfat -o uid=1000,gid=1000 /dev/sdb1 /mnt/usb       │
│                                                                     │
│  // /etc/fstab                                                       │
│  /dev/sdb1  /mnt/usb  vfat  defaults,uid=1000  0  0                 │
│                                                                     │
│  // exFAT 마운트                                                      │
│  $ sudo mount -t exfat /dev/sdb1 /mnt/usb                           │
│  $ sudo apt install exfat-fuse exfat-utils    // 패키지 필요          │
│                                                                     │
│  【FAT 정보 확인】                                                    │
│  ──────────────────                                                │
│  // 파일시스템 정보                                                    │
│  $ sudo fdisk -l /dev/sdb1                                          │
│  $ sudo blkid /dev/sdb1                                             │
│  $ sudo file -s /dev/sdb1                                           │
│                                                                     │
│  // FAT 상세 정보                                                     │
│  $ sudo dosfsck -v /dev/sdb1                                        │
│  $ sudo minfo -i /dev/sdb1                                          │
│                                                                     │
│  // 디스크 사용량                                                     │
│  $ df -h /mnt/usb                                                   │
│                                                                     │
│  【FAT 복구】                                                         │
│  ──────────────────                                                │
│  // 검사 및 복구                                                      │
│  $ sudo dosfsck -a /dev/sdb1      // 자동 복구                        │
│  $ sudo dosfsck -r /dev/sdb1      // 대화형 복구                      │
│  $ sudo dosfsck -v /dev/sdb1      // 상세 출력                        │
│                                                                     │
│  // Windows                                                          │
│  > chkdsk E: /f                                                     │
│  > chkdsk E: /f /r                                                  │
│                                                                     │
│  【FAT 파일 복구】                                                    │
│  ──────────────────                                                │
│  // testdisk                                                         │
│  $ sudo testdisk /dev/sdb1                                          │
│                                                                     │
│  // photorec                                                         │
│  $ sudo photorec /dev/sdb1                                          │
│                                                                     │
│  // foremost                                                         │
│  $ foremost -t all -i /dev/sdb1 -o /recovery                        │
│                                                                     │
│  【클러스터 크기】                                                     │
│  ──────────────────                                                │
│  // FAT32 클러스터 크기 (기본)                                         │
│  // 32MB ~ 8GB: 4KB                                                 │
│  // 8GB ~ 16GB: 8KB                                                 │
│  // 16GB ~ 32GB: 16KB                                               │
│  // 32GB ~: 32KB                                                    │
│                                                                     │
│  // 클러스터 크기 지정                                                  │
│  $ sudo mkfs.vfat -F 32 -s 8 /dev/sdb1    // 4KB 클러스터            │
│                                                                     │
│  【Windows FAT 명령어】                                               │
│  ──────────────────                                                │
│  // 포맷                                                             │
│  > format E: /FS:FAT32                                              │
│  > format E: /FS:exFAT                                              │
│  > format E: /FS:FAT32 /V:MYUSB /Q                                  │
│                                                                     │
│  // 파일시스템 변환                                                    │
│  > convert E: /FS:NTFS    // FAT → NTFS                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 클러스터 체인으로 파일 관리
• 종류: FAT12, FAT16, FAT32, exFAT
• 구조: 예약영역, FAT, 루트디렉토리, 데이터영역
• FAT32 최대: 파일 4GB-1, 볼륨 32GB (기본)
• exFAT 최대: 파일 16EB, 볼륨 128PB
• 장점: 광범위한 호환성, 간단한 구조
• 단점: 보안 없음, 저널링 없음, 단편화
• 생성: mkfs.vfat -F 32
• 마운트: mount -t vfat
• 복구: dosfsck, chkdsk
• 용도: USB, SD카드, 임베디드
```
