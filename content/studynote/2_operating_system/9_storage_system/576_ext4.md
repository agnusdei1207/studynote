# 576. ext4 (Fourth Extended Filesystem)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 리눅스 기본 저널링 파일시스템
> 2. **가치**: 대용량 지원, 호환성, 안정성
> 3. **융합**: ext3, 저널링, extent와 연관

---

## Ⅰ. 개요

### 개념 정의
**ext4**는 **리눅스의 기본 저널링 파일시스템으로 ext3의 후속 버전**입니다.

### 💡 비유: 업그레이드된 도서관
ext4는 **확장된 도서관**과 같습니다. 더 크고 빠르고 효율적입니다.

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ext4 파일시스템 생성】                                             │
│  ──────────────────                                                │
│  // 기본 생성                                                        │
│  $ sudo mkfs.ext4 /dev/sdb1                                         │
│                                                                     │
│  // 옵션 지정                                                        │
│  $ sudo mkfs.ext4 -L mylabel /dev/sdb1      // 라벨                  │
│  $ sudo mkfs.ext4 -b 4096 /dev/sdb1         // 블록 크기              │
│  $ sudo mkfs.ext4 -N 1000000 /dev/sdb1      // inode 수               │
│  $ sudo mkfs.ext4 -I 256 /dev/sdb1          // inode 크기             │
│  $ sudo mkfs.ext4 -O ^has_journal /dev/sdb1 // 저널 비활성화          │
│                                                                     │
│  【ext4 정보 확인】                                                   │
│  ──────────────────                                                │
│  // 슈퍼블록 정보                                                    │
│  $ sudo tune2fs -l /dev/sda1                                        │
│  $ sudo dumpe2fs /dev/sda1 | head -30                               │
│                                                                     │
│  // 파일시스템 특징                                                   │
│  $ sudo tune2fs -l /dev/sda1 | grep features                        │
│  Filesystem features:      has_journal ext_attr resize_inode        │
│                                                                     │
│  // 사용량                                                           │
│  $ df -h /dev/sda1                                                  │
│  $ df -iT /dev/sda1                  // inode 사용량                 │
│                                                                     │
│  【ext4 마운트 옵션】                                                 │ |
│  ──────────────────                                                │
│  // 기본 마운트                                                       │
│  $ sudo mount /dev/sda1 /mnt/data                                   │
│                                                                     │
│  // 성능 옵션                                                        │
│  $ sudo mount -o noatime /dev/sda1 /mnt/data   // atime 미갱신       │
│  $ sudo mount -o data=writeback /dev/sda1 /mnt/data                 │
│  $ sudo mount -o barrier=0 /dev/sda1 /mnt/data  // 쓰기 배리어 해제   │
│                                                                     │
│  // /etc/fstab 등록                                                  │
│  /dev/sda1  /mnt/data  ext4  defaults,noatime  0  2                 │
│  UUID=xxx   /mnt/data  ext4  defaults,noatime  0  2                 │
│                                                                     │
│  【ext4 유지보수】                                                    │
│  ──────────────────                                                │
│  // 파일시스템 검사                                                   │
│  $ sudo e2fsck -f /dev/sda1                                         │
│  $ sudo e2fsck -p /dev/sda1            // 자동 복구                  │
│  $ sudo e2fsck -y /dev/sda1            // 모든 질문 yes              │
│                                                                     │
│  // 크기 확장                                                        │
│  $ sudo resize2fs /dev/sda1             // 전체 크기로                │
│  $ sudo resize2fs /dev/sda1 100G        // 100GB로                   │
│                                                                     │
│  // 파라미터 변경                                                    │
│  $ sudo tune2fs -L newlabel /dev/sda1   // 라벨 변경                  │
│  $ sudo tune2fs -m 1 /dev/sda1          // 예약 블록 1%              │
│  $ sudo tune2fs -c 50 /dev/sda1         // 최대 마운트 횟수           │
│  $ sudo tune2fs -i 30d /dev/sda1        // fsck 간격 30일            │
│                                                                     │
│  【ext3에서 ext4 변환】                                               │
│  ──────────────────                                                │
│  // 오프라인 변환 (권장)                                              │
│  $ sudo tune2fs -O extents,uninit_bg,dir_index /dev/sda1            │
│  $ sudo e2fsck -f /dev/sda1                                         │
│                                                                     │
│  // 마운트 옵션으로만 (기능 제한)                                      │
│  $ sudo mount -t ext4 /dev/sda1 /mnt                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 리눅스 기본 저널링 파일시스템
• 최대 크기: 파일 16TB, 볼륨 1EB
• Extent: 연속 블록 범위 효율적 관리
• 지연 할당: 쓰기 시점까지 블록 할당 지연
• 저널링: journal, ordered, writeback
• 블록 크기: 1KB~64KB (기본 4KB)
• inode: 크기 256 bytes (ext3: 128 bytes)
• 타임스탬프: 나노초 단위
• 생성: mkfs.ext4
• 확인: tune2fs -l, dumpe2fs
• 확장: resize2fs
• 변환: tune2fs -O extents
```
