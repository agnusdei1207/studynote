# 014. API (Application Programming Interface)
#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 애플리케이션이 운영체제 서비스에 접근하는 프로그래밍 인터페이스
> 2. **가치**: 이식성, 추상화, 표준화, 생산성
> 3. **융합**: POSIX, Win32, 시스템 콜, 라이브러리와 연관
---

## Ⅰ. 개요
### 개념 정의
API(Application Programming Interface)는 **애플리케이션이 운영체제나 라이브러리의 서비스에 접근하기 위해 사용하는 프로그래밍 인터페이스**다. 시스템 콜의 상위 수준 추상화를 제공한다.
### 💡 비유: 식당 메뉴판
API는 **식당 메뉴판**과 같다. 손님(애플리케이션)이 메뉴판(API)에서 원하는 요리(함수)를 주문한다. 주방(커널)이 어떻게 요리하는지 몰라도 된다.
