# 003. 다중 프로그래밍 (Multiprogramming)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리에 여러 프로그램을 상주시켜 CPU 유휴 시간을 최소화하는 기법
> 2. **가치**: CPU 활용도 극대화, 처리량 향상, 시스템 효율성 증대
> 3. **융합**: 인터럽트, DMA, 메모리 관리, I/O 버퍼링과 연계

---

## Ⅰ. 개요

### 개념 정의

다중 프로그래밍(Multiprogramming)은 **메모리에 여러 개의 프로그램을 동시에 적재하여, 하나의 프로그램이 I/O 대기 상태가 되면 다른 프로그램에게 CPU를 할당함으로써 CPU 활용도를 극대화하는 기법**이다.

### 💡 비유: 식당 주방
다중 프로그래밍은 **식당 주방의 요리사**와 같다. 요리사(한 명의 CPU)가 A 주문의 파스타를 끓이는 동안(30분 대기) 가만히 있지 않고, B 주문의 샐러드를 만들고(다른 프로그램 실행), C 주문의 디저트를 준비한다(또 다른 프로그램). 이렇게 하면 요리사의 가동률이 높아진다.

### 다중 프로그래밍 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                다중 프로그래밍 개요                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 원리】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  [단일 프로그래밍]                                             │ │
│  │  Program A: ████░░░░░░░░░░░░████████░░░░░░░░░░░░████         │ │
│  │              CPU   I/O 대기    CPU      I/O 대기    CPU        │ │
│  │              ↑                  ↑                    ↑         │ │
│  │            작동              대기                 작동          │ │
│  │                                                               │ │
│  │  [다중 프로그래밍]                                             │ │
│  │  Program A: ████░░░░░░░░░░░░████████░░░░░░░░░░░░████         │ │
│  │  Program B: ░░░░████████████░░░░░░░░████████████░░░░░        │ │
│  │  Program C: ░░░░░░░░░░░░░░░░████████████████░░░░░░░░░        │ │
│  │              ↑  ↑           ↑          ↑                      │ │
│  │            A   B           A,B,C      C                       │ │
│  │          실행 실행         혼합       실행                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【핵심 구성요소】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  요소              역할                    기술                 │ │
│  │  ────              ────                    ────                 │ │
│  │  메모리 분할       여러 프로그램 적재        파티셔닝            │ │
│  │  인터럽트          I/O 완료 감지           하드웨어/소프트웨어    │ │
│  │  문맥 교환         프로세스 전환           PCB 저장/복구         │ │
│  │  스케줄러          다음 실행 프로그램 선정   단기 스케줄러        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 다중 프로그래밍 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                다중 프로그래밍 상세                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【다중 프로그래밍 정도 (Degree)】                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정도              메모리 내 프로그램 수      효과               │ │
│  │  ────              ────────────────────      ────               │ │
│  │  낮음 (1~2)       제한적                    CPU 유휴 많음        │ │
│  │  적정 (3~10)      균형                      효율적 활용          │ │
│  │  높음 (10+)       과다                      스래싱 위험          │ │
│  │                                                               │ │
│  │  CPU 이용률 ∝ 다중 프로그래밍 정도                             │ │
│  │  (임계점까지)                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CPU 이용률 공식】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  CPU 이용률 = 1 - p^n                                         │ │
│  │                                                               │ │
│  │  p: 프로세스가 I/O 대기할 확률                                 │ │
│  │  n: 메모리 내 프로세스 수 (다중 프로그래밍 정도)                 │ │
│  │                                                               │ │
│  │  예시: p = 0.5, n = 4일 때                                    │ │
│  │  CPU 이용률 = 1 - 0.5^4 = 1 - 0.0625 = 93.75%                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                          │ │
│  │  • CPU 활용도 극대화                                            │ │
│  │  • 처리량 증가                                                  │ │
│  │  • 자원 효율적 사용                                              │ │
│  │                                                               │ │
│  │  단점:                                                          │ │
│  │  • 메모리 요구량 증가                                            │ │
│  │  • 구현 복잡성                                                  │ │
│  │  • 문맥 교환 오버헤드                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 다중 프로그래밍 × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                다중 프로그래밍 × 타 과목 융합                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 인터럽트: I/O 완료 시 CPU 알림                                     │
│  • DMA: CPU 개입 없이 데이터 전송                                     │
│  • 메모리: 다중 파티션 관리                                           │
│  • 캐시: 문맥 교환 시 캐시 오염                                       │
│                                                                     │
│  【네트워크】                                                        │
│  • 서버: 다중 클라이언트 요청 처리                                    │
│  • 소켓: 비동기 I/O 활용                                              │
│  • 프로토콜: 멀티플렉싱                                               │
│                                                                     │
│  【데이터베이스】                                                      │
│  • 트랜잭션: 동시성 제어                                              │
│  • 버퍼: I/O 대기 최소화                                              │
│  • 락: 자원 경합 관리                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  비교: 다중 프로그래밍 vs 시분할 시스템                         │ │
│  │  ─────────────────────────────────────────────                 │ │
│  │                                                               │ │
│  │  구분          다중 프로그래밍         시분할 시스템             │ │
│  │  ────          ───────────────         ────────────             │ │
│  │  목적          CPU 활용도              응답 시간                 │ │
│  │  전환 시점     I/O 대기 시             시간 할당량 만료          │ │
│  │  사용자        단일/일괄               다중 인터랙티브            │ │
│  │  적용          일괄 처리               대화형 시스템              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 다중 프로그래밍 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                다중 프로그래밍 분석                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시스템 정보 확인 (Linux)】                                         │
│  ──────────────────                                                  │
│  # 실행 중인 프로세스 수                                              │
│  $ ps -e | wc -l                                                    │
│                                                                     │
│  # CPU 이용률 확인                                                    │
│  $ top -n 1 | grep "Cpu(s)"                                         │
│                                                                     │
│  # 평균 부하 (Load Average)                                          │
│  $ uptime                                                           │
│                                                                     │
│  【성능 지표】                                                        │
│  ──────────────────                                                  │
│  • CPU 이용률: 80~90% 목표                                           │
│  • 문맥 교환: 초당 1000회 이하 권장                                   │
│  • 대기 큐: 적정 길이 유지                                            │
│                                                                     │
│  【최적화 팁】                                                        │
│  ──────────────────                                                  │
│  1. I/O 바운드와 CPU 바운드 프로세스 균형                             │
│  2. 메모리 충분 확보 (스와핑 방지)                                     │
│  3. 적정 프로세스 수 유지                                             │
│  4. 우선순위 스케줄링 활용                                            │
│  5. 백그라운드 작업 관리                                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 다중 프로그래밍: 메모리에 여러 프로그램 상주
• 핵심 원리: I/O 대기 시 다른 프로그램 실행
• CPU 이용률: 1 - p^n 공식
• 장점: CPU 활용도, 처리량 증가
• 단점: 메모리 요구, 복잡성 증가
• 관련 기술: 인터럽트, 문맥 교환, 스케줄링
• 발전: 시분할, 다중 처리, 분산 시스템
```
