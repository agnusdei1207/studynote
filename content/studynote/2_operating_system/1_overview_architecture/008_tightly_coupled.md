# 008. 강결합 시스템 (Tightly Coupled System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 CPU가 공유 메모리를 통해 직접 통신하는 시스템
> 2. **가치**: 빠른 통신, 낮은 지연 시간, 높은 데이터 일관성
> 3. **융합**: SMP, NUMA, 공유 메모리 병렬 처리와 연관

---

## Ⅰ. 개요

### 개념 정의

강결합 시스템(Tightly Coupled System)은 **여러 프로세서가 공유 메모리를 통해 직접 통신하고 데이터를 교환하는 시스템**으로, 프로세서 간의 상호 의존도가 높다.

### 💡 비유: 한 부엌에서 요리하는 여러 요리사
강결합 시스템은 **하나의 큰 부엌에서 일하는 요리사들**과 같다. 모든 요리사(CPU)가 같은 냉장고(공유 메모리)를 사용한다. 요리사 A가 재료를 꺼내면 요리사 B가 바로 볼 수 있다. 소통이 매우 빠르지만, 냉장고 정리(동기화)를 잘해야 한다.

## Ⅱ. 강결합 시스템 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                강결합 시스템 상세                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 빠른 통신                                                   │ │
│  │     • 메모리 접근 속도로 통신                                    │ │
│  │     • 메시지 전달 오버헤드 없음                                  │ │
│  │                                                               │ │
│  │  2. 낮은 지연 시간                                               │ │
│  │     • 마이크로초 단위                                            │ │
│  │     • 실시간 처리에 적합                                         │ │
│  │                                                               │ │
│  │  3. 데이터 일관성                                                │ │
│  │     • 단일 메모리 공간                                           │ │
│  │     • 캐시 일관성 프로토콜 적용                                   │ │
│  │                                                               │ │
│  │  4. 자원 공유                                                    │ │
│  │     • 메모리, I/O 장치 공유                                      │ │
│  │     • 효율적 자원 활용                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 확장성 제한                                                  │ │
│  │     • 메모리 버스 병목                                           │ │
│  │     • CPU 추가 시 성능 향상 둔화                                  │ │
│  │                                                               │ │
│  │  2. 장애 전파                                                    │ │
│  │     • 하나의 CPU 장애가 전체 영향                                 │ │
│  │     • 공유 자원 손상 시 치명적                                    │ │
│  │                                                               │ │
│  │  3. 동기화 복잡성                                                │ │
│  │     • 정교한 락 메커니즘 필요                                     │ │
│  │     • 데드락 위험                                                │ │
│  │                                                               │ │
│  │  4. 비용                                                         │ │
│  │     • 고속 공유 메모리 비용                                       │ │
│  │     • 복잡한 하드웨어                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【강결합 시스템 예시】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시스템              특징                                       │ │
│  │  ────              ────                                        │ │
│  │  SMP 서버          다중 CPU, 공유 메모리                        │ │
│  │  NUMA 시스템        분산 메모리, 일관성 유지                      │ │
│  │  멀티코어 CPU       칩 내 공유 캐시                              │ │
│  │  메인프레임        대규모 공유 메모리                            │ │
│  │                                                               │ │
```

## Ⅳ. 실무 적용

### 강결합 시스템 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                강결합 시스템 분석                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【강결합 시스템 확인】                                               │
│  ──────────────────                                                  │
│  # CPU 및 메모리 정보                                                │
│  $ lscpu                                                            │
│  $ numactl --hardware                                               │
│                                                                     │
│  # 공유 메모리 IPC 확인                                              │
│  $ ipcs -m                                                          │
│                                                                     │
│  # SMP/NUMA 확인                                                    │
│  $ cat /proc/cpuinfo | grep "physical id" | sort -u                 │
│                                                                     │
│  【프로그래밍 모델】                                                  │
│  ──────────────────                                                  │
│  1. POSIX Threads (pthreads)                                        │
│  2. OpenMP                                                          │
│  3. Pthreads + 공유 메모리                                          │
│  4. MPI (공유 메모리 모드)                                           │
│                                                                     │
│  【동기화 기법】                                                      │
│  ──────────────────                                                  │
│  1. 뮤텍스 (Mutex)                                                   │
│  2. 세마포어 (Semaphore)                                            │
│  3. 스핀락 (Spinlock)                                                │
│  4. 원자 변수 (Atomic)                                              │
│  5. RCU (Read-Copy-Update)                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 강결합: 공유 메모리 기반 직접 통신
• 장점: 빠른 통신, 낮은 지연, 데이터 일관성
• 단점: 확장성 제한, 장애 전파, 동기화 복잡
• 비교: 약결합은 메시지 패싱, 확장성 우수
• 적용: SMP, NUMA, 멀티코어, HPC
• 프로그래밍: pthreads, OpenMP, 공유 메모리
```
