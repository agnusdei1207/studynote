# 027. 부트로더 (Bootloader)
 관련 인사이트 (3줄 요약)
> 1. **본질**: 운영체제 커널을 메모리에 로드하고 실행하는 프로그램
> 2. **가치**: 멀티부팅, 커널 선택, 초기 설정
> 3. **융합**: GRUB, systemd-boot, U-Boot와 연관

---

## Ⅰ. 개요

 관련 정의

부트로더(Bootloader)는 **컴퓨터 시작 시 운영체제 커널을 메모리에 로드하고 실행하는 프로그램**으로, BIOS/UEFI와 운영체제 사이의 중간 역할을 담당한다. 부트로더는 디스크의 MBR이나 EFI 파티션에서 커널 이미지를 찾아 메모리에 적재한 후 커널로 제어권을 넘긴다.

### 💡 비유: 호텔 안내 데스크
 부트로더는 **호텔 안내 데스크**와 같다. 투숙객(BIOS/UEFI)이 오면 안내 데스크(부트로더)에게 안내를 전달하고, 안내 데스크는 투숙객에게 방(key)을 배정하고 객실(커널)로 안내합니다. 객실이 준비되면 투숙객이 체크인합니다.### 부트로더 구조

## Ⅲ. 부트로더 × 타 과목 융합

 관련 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                부트로더 × 타 과목 융합                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【저장장치】                                                        │
│  • MBR: 부트 섹터(512B) + 파티션 테이블                   │
│  • GPT: EFI 파티션 테이블 + 보호 파티션                 │
│  • 활성 플래그: 부트 가능 파티션 식별                    │
│                                                                     │
│  【파일시스템】                                                      │
│  • 지원 파일시스템: ext4, XFS, Btrfs, FAT, NTFS            │
│  • 설정 파일 위치: /boot/grub/grub.cfg                   │
│  • 모듈 로딩: 파일시스템 모듈 필요                          │
│                                                                     │
│  【보안】                                                            │
│  • Secure Boot: 부트로더 검증                              │
│  • 비밀번호: GRUB 비밀번호 보호                           │
│  • 체인 로딩: 부트로더 서명 검증                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  【부트로더 최적화】                                             │
│  ──────────────────                                                  │
│  1. 빠른 부팅                                                 │
│  • UEFI 네이티브 부트로더 사용                             │
│  • 커널 프리로딩                             │
│                                                                     │
│  2. 네트워크 부팅                                             │
│  • PXE 부트로더 지원                              │
│  • iPXE (HTTP 기반)                                 │
│                                                                     │
│  3. 커스터마이짱된 부트로더                                          │
│  • 불필요한 기능 제거                             │
│  • 최소한 설정만 유지                              │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

 관련 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                부트로더 분석                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【GRUB 설정 관리】                                              │
│  ──────────────────                                                  │
│  # 설정 파일 편집                                                   │
│  $ sudo nano /etc/default/grub                        │
│  $ sudo update-grub                                        │
│                                                                     │
│  # 부트로더 설치                                               │
│  $ grub-install /dev/sda /boot/grub                    │
│  $ grub-install /dev/loop0 --root-directory=/boot    │
│                                                                     │
│  # 부트로더 재설치                                               │
│  $ grub-install --recheck /dev/sda                        │
│  $ grub-install /dev/sda --no-floppy           │
│                                                                     │
│  # 부트로더 제거                                                 │
│  $ apt remove 'apparmor'                                  │
│  $ yum remove abrt                              │
│  │                                                               │ │
│  【부팅 문제 해결】                                              │
│  ──────────────────                                                  │
│  # 부트로더 복구 (rescue mode)                                  │
│  $ chroot /mnt/sysimage                                │
│  $ grub-install /root=/dev/sdaX --root-directory=/mnt    │
│                                                                     │
│  # 부트로더 재설치                                                │
│  $ grub-install /dev/sda                               │
│  $ grub2-mkconfig                                         │
│  $ update-grub                                            │
│                                                                     │
│  【UEFI 부트로더 관리】                                            │
│  ──────────────────                                                  │
│  # 부트 숭목 추가                                                 │
│  $ efibootmgr -c -l                                             │
│                                                                     │
│  # 기본 부트 숭목 변경                                              │
│  $ efibootmgr -o 0000,Ubuntu                          │
│                                                                     │
│  # 부트 숭목 삭제                                                 │
│  $ efibootmgr -b 0000                                        │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

 관련 요약

```
• 부트로더: 커널 로딩, 멀티부팅, 초기 설정
• GRUB 2: 모듈화, 멀티부팅, 그래픽 메뉴
• systemd-boot: UEFI 네이티브, 빠른 부팅
• Windows Boot Manager: BCD 저장소, 하이버네이션 지원
• U-Boot: 임베디드, 다양한 아키텍처
• 관리: grub-install, efibootmgr
```
