# 040. Hot/Cold Standby

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장애 발생 시 대체 운영할 수 있는 예비 시스템 상태
> 2. **가치**: 고가용성, 재해 복구, 비즈니스 연속성
> 3. **융합**: HA 클러스터, 페일오버, DR 사이트와 연관

---

## Ⅰ. 개요

### 개념 정의

Hot/Cold Standby는 **장애 발생 시 서비스를 계속 제공하기 위해 준비된 예비 시스템의 운영 상태**를 의미한다. Hot Standby는 실시간으로 대기하며 즉시 전환 가능하고, Cold Standby는 필요 시 시작해야 한다.

### 💡 비유: 예비 타이어
Standby 시스템은 **자동차의 예비 타이어**와 같다. Hot Standby는 트렁크에 바로 꺼내 쓸 수 있게 준비된 스페어 타이어다. Cold Standby는 타이어 가게에 있는 예비 타이어다. 준비된 타이어(Hot)는 즉시 교체할 수 있지만, 가게 것(Cold)은 가져와야 한다.


## Ⅳ. 실무 적용

### Hot/Cold Standby 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                Hot/Cold Standby 분석                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데이터베이스 Hot Standby】                                         │
│  ──────────────────                                                  │
│  # MySQL 복제 상태                                                   │
│  mysql> SHOW SLAVE STATUS\G                                      │
│  mysql> SHOW MASTER STATUS\G                                      │
│                                                                     │
│  # PostgreSQL 복제 상태                                              │
│  $ pg_isready -h standby_host                                    │
│  $ psql -c "SELECT * FROM pg_stat_replication;"                  │
│                                                                     │
│  【Pacemaker 클러스터 상태】                                          │
│  ──────────────────                                                  │
│  # 클러스터 상태                                                     │
│  $ pcs status                                                     │
│  $ crm_mon -1                                                     │
│                                                                     │
│  # 리소스 상태                                                       │
│  $ pcs status resources                                           │
│                                                                     │
│  # 페일오버 테스트                                                   │
│  $ pcs cluster standby <node>                                     │
│  $ pcs cluster unstandby <node>                                   │
│                                                                     │
│  【HAProxy 헬스 체크】                                               │
│  ──────────────────                                                  │
│  # 백엔드 서버 상태                                                  │
│  $ echo "show stat" | socat stdio /var/run/haproxy.sock          │
│                                                                     │
│  # 서버 활성화/비활성화                                              │
│  $ echo "disable server backend/server" | \                      │
│      socat stdio /var/run/haproxy.sock                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• Hot Standby: 실시간 대기, 즉시 전환, 높은 비용
• Cold Standby: 전원 OFF, 수동 복구, 낮은 비용
• Warm Standby: 중간 상태, 부분 동기화
• RTO: 복구 시간 목표
• RPO: 복구 시점 목표 (데이터 손실)
• 페일오버: 감지 → 검증 → 전환 → 확인
```
