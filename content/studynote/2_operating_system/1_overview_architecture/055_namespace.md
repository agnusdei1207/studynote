# 055. 네임스페이스 (Namespace)
#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 리눅스 커널 기능으로 프로세스가 보는 자원을 격리
> 2. **가치**: 보안, 멀티테넌시, 컨테이너 격리 기반
> 3. **융합**: PID, NET, MNT, UTS, IPC, cgroups와 연관
---

## Ⅰ. 개요
### 개념 정의
네임스페이스(Namespace)는 **리눅스 커널 기능으로, 프로세스가 보는 시스템 자원을 격리하여 마치 독립된 환경인 것처럼 보이게 하는 기술**이다. 컨테이너 격리의 핵심 기반이다.
### 💡 비유: 방 번호표
네임스페이스는 **방 번호표**와 같다. 같은 건물(커널)에 여러 방(프로세스)이 있지만, 각 방은 번호(네임스페이스)로 구분된다. 101호에서 일어난 일이 102호에 영향을 주지 않는다.
### 네임스페이스 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                네임스페이스 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【리눅스 네임스페이스 종류】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  네임스페이스        설명                    예시              │ │
│  │  ────                ────                    ────              │ │
│  │  PID                프로세스 ID 격리             PID 1, 2, 3...   │ │
│  │  NET                네트워크 장치 격리             eth0, lo...     │ │
│  │  MNT                마운트 포인트 격리           /mnt1, /mnt2  │ │
│  │  UTS                UTS 네임스페이스 격리          호스트명         │ │
│  │  IPC                System V IPC 격리             세마포어, MQ     │ │
│  │  USER               사용자/그룹 ID 격리           uid, gid        │ │
│  │  CGROUP             cgroup 루트 격리             /sys/fs/cgroup │ │
│  │  TIME               부팅/단조 시계 격리             CLOCK_MONOTONIC │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【PID 네임스페이스 예시】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Host OS                          Container                     │ │
│  │  ┌─────────────────────┐      │  ┌─────────────────────┐       │ │
│  │  │ PID 1: init         │      │  │ PID 1: init (bash)  │       │ │
│  │  │ PID 100: sshd        │      │  │ PID 2: app1         │       │ │
│  │  │ PID 200: nginx       │      │  │ PID 3: app2         │       │ │
│  │  └─────────────────────┘      │  └─────────────────────┘       │ │
│  │                                                               │ │
│  │  컨테이너 내에서 PID 1은 init처럼 보임              │ │
│  │  호스트의 PID와 겹치지 않음                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
