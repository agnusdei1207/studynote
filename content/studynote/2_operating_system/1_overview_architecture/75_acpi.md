# ACPI (Advanced Configuration and Power Interface)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ACPI (Advanced Configuration and Power Interface)는 운영체제가 컴퓨터 하드웨어의 **전원 관리 및 구성 설정**을 직접 제어할 수 있도록 정의된 개방형 산업 표준 인터페이스이다.
> 2. **가치**: 기존의 하드웨어 종속적인 BIOS 전원 관리(APM)를 대체하여, 운영체제(OS)가 시스템 전반의 에너지 사용량을 워크로드에 맞춰 지능적으로 최적화하는 **OSPM (OS-directed Power Management)**을 가능케 한다.
> 3. **융합**: 하드웨어 제조사(인텔, MS 등) 간의 약속된 규격이며, 현대 OS가 절전 모드, 최대 절전 모드, 하이버네이션 등을 안전하게 구현하는 기술적 근간이 된다.

+++

## Ⅰ. ACPI의 개요

### 1. 정의
- PC 시스템의 하드웨어 자원을 발견하고 전원 상태를 관리하기 위해 운영체제와 펌웨어(BIOS/UEFI) 간에 주고받는 데이터 테이블과 메커니즘의 집합이다.

### 2. 등장 배경: 'OS에게 주도권을'
- 과거 APM(Advanced Power Management) 방식은 BIOS가 전권을 가졌다. OS는 하드웨어가 언제 꺼지는지 정확히 알기 어려워 데이터 손실 위험이 있었다. ACPI는 이 권한을 OS에게 넘겨주어, OS가 현재 실행 중인 앱의 상태를 고려해 전원을 정교하게 끄고 켤 수 있게 했다.

### 3. 💡 비유: '건물의 통합 스마트 전력 제어 시스템'
- 기존 방식(APM)이 방마다 설치된 수동 스위치라면, ACPI는 **'건물 전체를 관리하는 통합 스마트 제어실'**과 같다.
- 관리실(운영체제)은 어떤 방(장치)에 사람이 있는지, 작업이 끝났는지 실시간으로 파악하여 건물 전체의 냉난방과 조명(전력)을 가장 효율적으로 조절한다.

+++

## Ⅱ. ACPI 아키텍처 및 주요 구성 (Deep Dive)

### 1. OSPM (OS-directed Power Management)
- ACPI의 핵심 철학이다. 운영체제 내부에 상주하며 ACPI 테이블을 읽고 하드웨어 상태를 직접 변경하는 소프트웨어 모듈이다.

### 2. 핵심 구성 요소
- **ACPI Tables (DSDT, FADT 등)**: 하드웨어 장치의 사양과 제어 방법이 적힌 설계도이다.
- **ACPI Registers**: OS가 하드웨어 상태를 바꾸기 위해 값을 쓰는 물리적 저장소이다.
- **ACPI BIOS / UEFI**: OS가 로드되기 전 하드웨어를 초기화하고 ACPI 테이블을 메모리에 올리는 역할을 한다.
- **AML (ACPI Machine Language)**: 플랫폼 독립적인 가상 기계어로 작성된 제어 스크립트이다. OS는 AML 인터프리터를 통해 장치를 제어한다.

### 3. ASCII 구조 다이어그램
```text
    ┌──────────────────────────────────┐
    │      User Applications           │
    ├──────────────────────────────────┤
    │      Operating System            │
    │  ┌────────────────────────────┐  │
    │  │   OSPM (Power Manager)     │  │ ◀── 전원 제어 결정
    │  └─────────────┬──────────────┘  │
    └────────────────│─────────────────┘
                     ▼ (ACPI Interface)
    ┌────────────────┴─────────────────┐
    │  ACPI Tables & AML Interpreter   │ ◀── 하드웨어 정보 해석
    ├──────────────────────────────────┤
    │  ACPI Hardware / Firmware        │ ◀── 실제 전원 차단/전압 조절
    └──────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 복잡한 기계 장치를 직접 만지는 대신, 잘 정리된 설명서(ACPI Tables)와 버튼(Registers)을 통해 기계를 조작하는 '표준화된 컨트롤 패널'과 같습니다.

## Ⅲ. ACPI 기반 상태 관리 (States)

ACPI는 시스템 전체부터 개별 부품까지 정교한 상태 코드를 정의한다.

| 구분 | 명칭 | 설명 |
|:---|:---|:---|
| **G-States** | Global States | 시스템 전체 상태 (G0: Working, G1: Sleeping, G3: Mechanical Off) |
| **S-States** | Sleep States | 절전 단계 (S1~S4, 하이버네이션 포함) |
| **C-States** | CPU States | CPU 전력 상태 (C0: 활성, C1~C3: 저전력 대기) |
| **P-States** | Performance States | CPU 성능 상태 (전압 및 주파수 조절, DVFS) |
| **D-States** | Device States | 개별 장치 상태 (D0: Full On, D3: Off) |

- **📢 섹션 요약 비유**: 조명 밝기를 끄기(Off)와 켜기(On)로만 나누지 않고, 1단계부터 10단계까지 미세하게 조절(P-states)하거나 외출 모드(S-states)를 두어 에너지를 아끼는 것과 같습니다.

## Ⅳ. 실무 적용 및 하드웨어 연동 (Strategy)

### 1. 실무 시나리오: 노트북 덮개 닫기 (Lid Close)
- **상황**: 사용자가 작업 중에 노트북 덮개를 닫음.
- **판단**: 하드웨어가 인터럽트를 발생시키면, OSPM은 현재 열린 파일들을 저장하고 메모리 상태를 보존한 뒤, ACPI를 통해 시스템을 **S3 (Suspend to RAM)** 상태로 전이시킨다. 이 일련의 과정이 ACPI 규격 덕분에 제조사와 상관없이 일관되게 수행된다.

### 2. 기술사적 인사이트: 배터리 수명의 기술적 지표
- ACPI는 단순한 규격이 아니라 모바일 기기의 경쟁력이다. OS 기술사라면 **DSDT (Differentiated System Description Table)** 버그로 인한 전력 누수 문제를 진단하거나, 리눅스 커널의 ACPI 드라이버 튜닝을 통해 서버의 전력-성능 트레이드오프를 최적화하는 전략을 제시할 수 있어야 한다.

- **📢 섹션 요약 비유**: 설계도가 잘못되면(DSDT 오류) 건물의 전기가 샐 수 있듯이, ACPI 테이블의 무결성을 검증하는 것이 안정적인 시스템 운영의 시작입니다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **에너지 효율 극대화**: 사용하지 않는 하드웨어 부품을 즉시 꺼서 전력 낭비 방지.
- **플랫폼 독립성**: 특정 메인보드 칩셋에 상관없이 동일한 OS 전원 정책 적용 가능.

### 2. 미래 전망
최근에는 클라우드 서버의 탄소 배출 절감을 위해 ACPI의 하드웨어 원격 제어 기능이 더욱 고도화되고 있다. 또한 인공지능이 시스템 부하를 예측하여 선제적으로 ACPI 상태를 변경하는 **AI 기반 전력 관리** 기술이 도입되면서, ACPI는 운영체제의 '지능형 절전 도구'로 진화하고 있다.

- **📢 섹션 요약 비유**: 단순히 전기를 아끼는 단계를 넘어, 시스템의 컨디션을 최상으로 유지하면서도 환경까지 생각하는 '그린 운영체제'의 수호자입니다.

- **HAL (Hardware Abstraction Layer)**: ACPI가 구현되는 하위 추상화 계층.
