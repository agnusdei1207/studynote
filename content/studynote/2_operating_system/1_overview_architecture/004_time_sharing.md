# 004. 시분할 시스템 (Time-sharing System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 시간을 작은 단위로 분할하여 여러 사용자/프로세스에 순차적 할당
> 2. **가치**: 빠른 응답 시간, 대화형 처리, 사용자 편의성
> 3. **융합**: 인터럽트, 타이머, 스케줄링, 문맥 교환과 연계

---

## Ⅰ. 개요

### 개념 정의

시분할 시스템(Time-sharing System)은 **CPU 시간을 아주 작은 단위(Time Slice/Quantum)로 나누어 여러 사용자나 프로세스가 순차적으로 CPU를 사용하게 함으로써, 각자 독립된 컴퓨터를 사용하는 것처럼 느끼게 하는 시스템**이다.

### 💡 비유: 회의실
시분할 시스템은 **하나의 회의실을 여러 팀이 나눠 쓰는 것**과 같다. 회의실(한 대의 CPU)이 있고, A팀이 10분 쓰고, B팀이 10분 쓰고, C팀이 10분 쓴다. 각 팀은 회의실을 독점하는 것처럼 느끼지만, 실제로는 시간을 나눠 쓰는 것이다.

## Ⅳ. 실무 적용

### 시분할 시스템 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                시분할 시스템 분석                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시스템 정보 확인 (Linux)】                                         │
│  ──────────────────                                                  │
│  # 타이머 인터럽트 빈도 확인                                          │
│  $ grep "CONFIG_HZ" /boot/config-$(uname -r)                        │
│                                                                     │
│  # 현재 프로세스 시간 확인                                            │
│  $ ps -eo pid,comm,time,cputime                                     │
│                                                                     │
│  # 스케줄링 관련 정보                                                 │
│  $ cat /proc/<pid>/sched                                            │
│                                                                     │
│  【성능 튜닝】                                                        │
│  ──────────────────                                                  │
│  1. Time Quantum 조정 (kernel 설정)                                  │
│  2. 우선순위 관리 (nice, renice)                                     │
│  3. CPU 친화성 설정 (taskset)                                        │
│  4. 실시간 스케줄링 (chrt)                                           │
│  5. cgroups 자원 제한                                                │
│                                                                     │
│  【모니터링 지표】                                                    │
│  ──────────────────                                                  │
│  • 응답 시간: < 100ms 목표                                           │
│  • 대기 시간: 최소화                                                  │
│  • 처리량: 사용자당 충분한 CPU 시간                                   │
│  • 공정성: 모든 사용자에게 균등 할당                                   │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 시분할 시스템: CPU 시간을 분할하여 다중 사용자 지원
• 핵심 원리: Time Quantum 단위로 선점형 스케줄링
• 목표: 빠른 응답 시간 (인터랙티브)
• Quantum: 일반적으로 10~100ms
• 스케줄링: 라운드 로빈 (Round Robin) 기반
• 장점: 응답성, 다중 사용자, 자원 공유
• 단점: 오버헤드, 복잡성, 보안 이슈
```
