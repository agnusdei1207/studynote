# 034. 패닉 (Panic)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 운영체제가 복구 불가능한 오류를 만나 시스템을 중지하는 상황
> 2. **가치**: 시스템 보호, 데이터 손실 방지, 오류 원인 기록
> 3. **융합**: 커널 크래시, BSOD, 디버깅과 연관

---

## Ⅰ. 개요

### 개념 정의

패닉(Panic)은 **운영체제 커널이 복구할 수 없는 치명적인 오류를 감지하여 시스템 실행을 안전하게 중지하는 상황**이다. Unix 계열에서는 "kernel panic", Windows에서는 "Blue Screen of Death (BSOD)"라고 한다.

### 💡 비유: 비상 정지 버튼
패닉은 **공장의 비상 정지 버튼**과 같다. 기계(커널)가 심각한 오작동을 일으키면 작업자(커널)가 비상 정지 버튼을 눌러 모든 것을 멈춘다. 계속 돌아가면 더 큰 사고(데이터 손실)가 발생할 수 있어서다.

## Ⅳ. 실무 적용

### 패닉 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                패닉 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【패닉 설정 (Linux)】                                               │
│  ──────────────────                                                  │
│  # 패닉 후 동작                                                      │
│  $ cat /proc/sys/kernel/panic                                    │
│  $ echo 10 > /proc/sys/kernel/panic   # 10초 후 재부팅             │
│                                                                     │
│  # 패닉 시 kdump 실행                                                │
│  $ cat /proc/sys/kernel/kexec_load                              │
│                                                                     │
│  # 패닉 시 Oops 전송                                                 │
│  $ cat /proc/sys/kernel/panic_on_oops                            │
│                                                                     │
│  【kdump 설정】                                                      │
│  ──────────────────                                                  │
│  # 크래시 커널 예약                                                  │
│  $ cat /proc/cmdline | grep crashkernel                          │
│                                                                     │
│  # kdump 서비스 활성화                                               │
│  $ systemctl enable kdump                                        │
│  $ systemctl start kdump                                         │
│                                                                     │
│  # 덤프 분석                                                         │
│  $ crash /usr/lib/debug/vmlinux /var/crash/vmcore                │
│  crash> bt           # 백트레이스                                   │
│  crash> log          # 커널 로그                                    │
│  crash> ps           # 프로세스 목록                                │
│                                                                     │
│  【패닉 로그 확인】                                                  │
│  ──────────────────                                                  │
│  # 커널 로그                                                         │
│  $ dmesg | grep -i panic                                         │
│  $ journalctl -k | grep -i panic                                 │
│                                                                     │
│  # /var/log/messages                                                │
│  $ grep -i "kernel panic" /var/log/messages                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 패닉: 커널의 복구 불가능한 오류 시 안전한 중지
• 원인: 버그, 드라이버, 하드웨어, 파일시스템
• Linux: Kernel Panic, Windows: BSOD
• 분석: kdump, crash, dmesg
• 대응: 자동 재부팅, 덤프 수집
• 예방: 드라이버 검증, 하드웨어 점검
```
