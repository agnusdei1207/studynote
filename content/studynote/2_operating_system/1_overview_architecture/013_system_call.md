# 013. 시스템 콜 (System Call)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 프로그램이 커널 서비스를 요청하는 인터페이스
> 2. **가치**: 안전한 하드웨어/자원 접근, 권한 제어
> 3. **융합**: 트랩, 모드 전환, API, POSIX와 연관

---

## Ⅰ. 개요

### 개념 정의

시스템 콜(System Call)은 **사용자 프로그램이 운영체제 커널의 서비스를 요청하기 위해 사용하는 프로그래밍 인터페이스**로, 사용자 모드에서 커널 모드로 전환하여 특권 연산을 수행한다.

### 💡 비유: 호텔 프론트 데스크
시스템 콜은 **호텔 프론트 데스크**와 같다. 투숙객(사용자 프로그램)이 객실 청소나 룸서비스(커널 서비스)를 요청하려면 프론트 데스크(시스템 콜)에 요청해야 한다. 직접 청소하거나 주방에 들어갈 수 없다.

### 시스템 콜 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 콜 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시스템 콜 과정】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌────────────────────────────────────────────────────────┐  │ │
│  │  │              User Application                          │  │ │
│  │  │  printf("Hello");  // C 라이브러리 함수                  │  │ │
│  │  └───────────────────────────┬────────────────────────────┘  │ │
│  │                              │                                │ │
│  │                              ▼                                │ │
│  │  ┌────────────────────────────────────────────────────────┐  │ │
│  │  │              C Library (libc)                          │  │ │
│  │  │  write(fd, "Hello", 5);  // 라이브러리 내부 호출         │  │ │
│  │  │  syscall(SYS_write, ...);  // 시스템 콜 트리거          │  │ │
│  │  └───────────────────────────┬────────────────────────────┘  │ │
│  │                              │                                │ │
│  │                    ──────────┴───────────                    │ │
│  │                    │   Trap (syscall)    │                    │ │
│  │                    │   Mode: User→Kernel │                    │ │
│  │                    ──────────┬───────────                    │ │
│  │                              │                                │ │
│  │                              ▼                                │ │
│  │  ┌────────────────────────────────────────────────────────┐  │ │
│  │  │              Kernel (Kernel Mode)                      │  │ │
│  │  │  syscall_handler()                                      │  │ │
│  │  │    → sys_write()                                        │  │ │
│  │  │    → 파일 시스템, 디바이스 드라이버                      │  │ │
│  │  │    → 실제 I/O 수행                                       │  │ │
│  │  └───────────────────────────┬────────────────────────────┘  │ │
│  │                              │                                │ │
│  │                    ──────────┴───────────                    │ │
│  │                    │   Return (iret)     │                    │ │
│  │                    │   Mode: Kernel→User │                    │ │
│  │                    ─────────────────────────                 │ │
│  │                              │                                │ │
│  │                              ▼                                │ │
│  │  ┌────────────────────────────────────────────────────────┐  │ │
│  │  │              User Application                          │  │ │
│  │  │  // 결과 반환, 실행 계속                                │  │ │
│  │  └────────────────────────────────────────────────────────┘  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 시스템 콜 × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 콜 × 타 과목 융합                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【API vs 시스템 콜】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  API (Application Programming Interface)                       │ │
│  │  • 프로그래머에게 노출되는 인터페이스                           │ │
│  │  • 예: POSIX API, Win32 API                                   │ │
│  │                                                               │ │
│  │  시스템 콜                                                     │ │
│  │  • 커널에게 노출되는 인터페이스                                 │ │
│  │  • API 함수가 내부적으로 호출                                   │ │
│  │                                                               │ │
│  │  예시:                                                         │ │
│  │  printf() → write() → syscall(SYS_write)                      │ │
│  │  fopen()  → open()  → syscall(SYS_open)                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 트랩 명령: syscall (x86_64), svc (ARM)                           │
│  • 레지스터: 시스템 콜 번호, 인자 전달                                 │
│  • 컨텍스트: 사용자 스택 → 커널 스택                                   │
│  • 복귀: iret (x86), eret (ARM)                                     │
│                                                                     │
│  【보안】                                                            │
│  • seccomp: 시스템 콜 필터링                                         │
│  • SELinux: 시스템 콜 권한 제어                                      │
│  • 감사: auditd로 시스템 콜 로깅                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템 콜 오버헤드                                            │ │
│  │  ───────────────────                                          │ │
│  │                                                               │ │
│  │  구성 요소:                                                    │ │
│  │  1. 트랩 명령 실행 (수백 사이클)                                │ │
│  │  2. 컨텍스트 저장 (레지스터)                                    │ │
│  │  3. 커널 진입 및 핸들러 찾기                                    │ │
│  │  4. 인자 복사 (사용자→커널)                                    │ │
│  │  5. 실제 작업 수행                                             │ │
│  │  6. 결과 복사 (커널→사용자)                                    │ │
│  │  7. 컨텍스트 복원                                              │ │
│  │  8. 사용자 모드 복귀                                           │ │
│  │                                                               │ │
│  │  최적화: 버퍼링, 배칭, vDSO                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 시스템 콜 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 콜 분석                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시스템 콜 추적】                                                   │
│  ──────────────────                                                  │
│  # 프로세스별 시스템 콜 추적                                          │
│  $ strace -p <pid>                                                  │
│                                                                     │
│  # 시스템 콜 통계                                                    │
│  $ strace -c ls                                                    │
│                                                                     │
│  # 특정 시스템 콜 필터                                               │
│  $ strace -e trace=open,read,write ./program                       │
│                                                                     │
│  # 시스템 콜 번호 확인                                               │
│  $ ausyscall --dump                                                 │
│                                                                     │
│  【성능 분석】                                                        │
│  ──────────────────                                                  │
│  # 시스템 콜 발생 횟수                                                │
│  $ perf stat -e syscalls:sys_enter_* ./program                     │
│                                                                     │
│  # 시스템 콜 지연 시간                                                │
│  $ perf trace -s ./program                                         │
│                                                                     │
│  【최적화 팁】                                                        │
│  ──────────────────                                                  │
│  1. 버퍼링: 여러 데이터를 한 번에 처리                                │
│  2. 배칭: 여러 요청을 묶어서 처리                                     │
│  3. 캐싱: 자주 사용하는 데이터 메모리 보관                             │
│  4. vDSO: 빠른 시스템 콜 (gettimeofday 등)                           │
│  5. io_uring: 비동기 I/O로 시스템 콜 감소                            │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 시스템 콜: 사용자→커널 서비스 요청 인터페이스
• 카테고리: 프로세스, 파일, 장치, 통신, 보안
• 과정: 트랩 → 모드 전환 → 커널 실행 → 복귀
• 오버헤드: 트랩, 컨텍스트 저장/복원, 인자 복사
• API vs 콜: API는 프로그래머용, 콜은 커널용
• 최적화: 버퍼링, 배칭, vDSO, io_uring
```
