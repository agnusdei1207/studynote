# 026. 부트스트랩 (Bootstrap) 관련 인사이트 (3줄 요약)
> 1. **본질**: 컴퓨터 시작 시 하드웨어 초기화부터 OS 로딩까지의 과정
> 2. **가치**: 시스템 시작, 하드웨어 인식, OS 로딩
> 3. **융합**: BIOS, UEFI, MBR, GPT와 연관

---

## Ⅰ. 개요

 관련 정의

부트스트랩(Bootstrap)은 **컴퓨터 전원이 켜진 후 하드웨어를 초기화하고 운영체제를 메모리에 로딩하여 실행하는 시작 과정**이다. POST(POST-power-on state)에서 운영체제가 실행 가능한 상태로 만드는 일련의 단계다.

 전원을 켜고 POST(POST) 상태에서 정상(normal) 상태로 진입합니다

### 💡 비유: 건물 기공
부트스트랩은 **건물의 기공 절차**와 같다. 전기가 들어오면(전원 on) 조명이 켜지고(하드웨어 초기화), 엘리베이터가 운행되고(부트로더), 입주자가 도착합니다(OS 로딩).

 건물이 준비되면 정상 운영됩니다.

## Ⅴ. 기대효과 및 결론

 관련 요약

```
• 부트스트랩: 전원 on → OS 실행까지의 시작 과정
• BIOS: 16-bit, MBR, 1MB 제한
 구형
• UEFI: 32/64-bit, GPT, Secure Boot 지원
• 단계: POST → BIOS/UEFI → 부트로더 → 커널 → init
• 부트로더: GRUB, systemd-boot, Windows Boot Manager
• 최적화: 병렬 부팅, 네트워크 부팅, 빠른 부팅
```
