# 033. 코어 덤프 (Core Dump)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로그램 비정상 종료 시 메모리 상태를 저장한 파일
> 2. **가치**: 사후 분석, 디버깅, 장애 원인 파악
> 3. **융합**: 디버거, 시그널, 크래시 분석과 연관

---

## Ⅰ. 개요

### 개념 정의

코어 덤프(Core Dump)는 **프로그램이 비정상적으로 종료할 때 당시 메모리 내용을 파일로 저장한 것**으로, 메모리 이미지, 레지스터 상태, 스택 내용 등을 포함한다. "코어"는 초기 컴퓨터의 자기테이프 메모리 코어에서 유래했으며, 현대에는 메모리 덤프라고도 한다.

### 💡 비유: 사고 현장 사진
코어 덤프는 **사고 현장을 찍은 사진**과 같다. 사고(크래시)가 발생하면 경찰(개발자)이 현장에 와서 사진(코어 덤프)을 찍는다. 나중에 이 사진을 분석해서 사고 원인을 파악한다. 어느 시간에 어디서 무슨 일이 있었는지 알 수 있다.

```
│  │         │                                                     │ │

│  │         ▼                                                     │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │              백트레이스, 변수 검사                       │   │ │

│  │  └─────────────────────────────────────────────────────────┘   │ │

│                                                                     │
│  【코어 덤프 내용】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 실행 파일 정보 (ELF 헤더)                                     │ │

│  │  • CPU 레지스터 상태                                             │ │

│  │  • 힙 메모리 내용                                                 │ │
│  │  • 전역 변수 값                                                   │ │
│  │  • 스레드 정보                                                    │ │
│  │  • 열린 파일 디스크립터                                          │ │
│  │  • 신호 정보                                                      │ │
```

## Ⅱ. 코어 덤프 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                코어 덤프 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【코어 덤프 생성 시그널】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시그널            원인                    기본 동작             │ │
│  │  ────              ────                    ────                  │ │
│  │  SIGSEGV          세그먼트 오류           코어 덤프 + 종료      │ │
│  │  SIGABRT          abort() 호출            코어 덤프 + 종료      │ │
│  │  SIGBUS           버스 오류               코어 덤프 + 종료      │ │
│  │  SIGFPE           부동소수점 예외         코어 덤프 + 종료      │ │
│  │  SIGILL           잘못된 명령어           코어 덤프 + 종료      │ │
│  │  SIGQUIT          키보드 종료             코어 덤프 + 종료      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【코어 덤프 설정 (Linux)】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  # 코어 덤프 허용 크기 확인                                       │ │
│  │  $ ulimit -c                                                   │ │
│  │                                                               │ │
│  │  # 코어 덤프 허용 (무제한)                                        │ │
│  │  $ ulimit -c unlimited                                        │ │
│  │                                                               │ │
│  │  # 코어 덤프 패턴 설정                                            │ │
│  │  $ cat /proc/sys/kernel/core_pattern                           │ │
│  │  $ echo "/var/core/core.%e.%p" > /proc/sys/kernel/core_pattern  │ │
│  │                                                               │ │
│  │  # 패턴 지시자                                                    │ │
│  │  %%e = 실행 파일 이름    %%p = PID                             │ │
│  │  %%t = 시간           %%h = 호스트명                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ELF 코어 덤프 형식】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  섹션              내용                                        │ │
│  │  ────              ────                                        │ │
│  │  ELF Header       매직 넘버, 아키텍처                           │ │
│  │  Program Headers  세그먼트 정보                                  │ │
│  │  Notes            보조 정보 (프로세스 상태)                     │ │
│  │  Memory Segments 실제 메모리 내용                               │ │
│  │  Thread Info      스레드 레지스터, 스택                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────┐

│  【디버깅】                                                            │

│                                                                     │

│  │  (gdb) bt                    # 백트레이스                       │ │
```

## Ⅳ. 실무 적용

### 코어 덤프 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                코어 덤프 분석                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【코어 덤프 설정】                                                  │
│  ──────────────────                                                  │
│  # 영구 설정 (/etc/security/limits.conf)                            │
│  * soft core unlimited                                              │
│  * hard core unlimited                                              │
│                                                                     │
│  # systemd 서비스 설정                                               │
│  [Service]                                                          │
│  LimitCORE=infinite                                                 │
│                                                                     │
│  【코어 덤프 분석 명령】                                             │
│  ──────────────────                                                  │
│  # 코어 덤프 파일 찾기                                               │
│  $ find /var -name "core*" 2>/dev/null                           │
│  $ locate core | grep core.                                       │
│                                                                     │
│  # 간단한 분석                                                       │
│  $ file core.1234                                                │
│  $ readelf -h core.1234                                          │
│  $ objdump -s core.1234                                          │
│                                                                     │
│  【커널 코어 덤프 (kdump)】                                          │
│  ──────────────────                                                  │
│  # kdump 설정                                                       │
│  $ systemctl enable kdump                                        │
│  $ systemctl start kdump                                         │
│                                                                     │
│  # 크래시 덤프 분석                                                  │
│  $ crash /usr/lib/debug/lib/modules/$(uname -r)/vmlinux \        │
│        /var/crash/vmcore                                          │
│                                                                     │
│  【자동화된 분석 (apport, abrt)】                                    │
│  ──────────────────                                                  │
│  # Ubuntu apport                                                    │
│  $ apport-cli /var/crash/_path.1000.crash                        │
│                                                                     │
│  # RHEL abrt                                                        │
│  $ abrt-cli list                                                 │
│  $ abrt-cli info <id>                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 코어 덤프: 프로그램 크래시 시 메모리 상태 저장
• 생성 시그널: SIGSEGV, SIGABRT, SIGBUS, SIGFPE
• 내용: 레지스터, 스택, 힙, 전역변수, 스레드 정보
• 분석 도구: GDB, crash, readelf
• 설정: ulimit, core_pattern
• 커널: kdump, vmcore
```
