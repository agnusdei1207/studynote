# 032. 문맥 교환 (Context Switch)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 한 프로세스에서 다른 프로세스로 실행을 전환하는 과정
> 2. **가치**: 멀티태스킹 구현, CPU 공유, 동시성 제공
> 3. **융합**: 스케줄링, 인터럽트, 성능 비용과 연관

---

## Ⅰ. 개요

### 개념 정의

문맥 교환(Context Switch)은 **CPU가 실행 중인 프로세스의 문맥을 저장하고 다른 프로세스의 문맥을 복원하여 실행을 전환하는 과정**이다. 이를 통해 단일 CPU에서 여러 프로세스가 동시에 실행되는 것처럼 보이는 멀티태스킹을 구현한다.

### 💡 비유: TV 채널 변경
문맥 교환은 **TV 채널을 변경하는 것**과 같다. 한 채널(프로세스 A)을 보다가 리모컨(CPU)으로 다른 채널(프로세스 B)로 변경한다. 이전 채널에서 어디까지 봤는지(문맥 저장) 기억하고, 새 채널로 가서 계속 본다. 채널을 바꾸는 데는 약간의 시간(오버헤드)이 걸린다.

```
│  【문맥 교환 과정】                                                  │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────────┐                                         │ │
│  │  │ Process A       │                                         │ │
│  │  │ Running         │                                         │ │
│  │  │                 │                                         │ │
│  │  │  PC: 0x1234     │                                         │ │
│  │  │  R1: 100        │                                         │ │
│  │  │  SP: 0x8000     │                                         │ │
│  │  └────────┬────────┘                                         │ │
│  │           │                                                     │ │
│  │           │ 1. 문맥 저장                                       │ │
│  │           ▼                                                     │ │
│  │  ┌─────────────────┐     ┌─────────────────┐                  │ │
│  │  │ PCB A           │     │ PCB B           │                  │ │

│  │                                    │                            │ │
│  │                                    │ 2. 문맥 복원              │ │
│  │                                    ▼                            │ │
│  │                           ┌─────────────────┐                  │ │
│  │                           │ Process B       │                  │ │
│  │                           │ Running         │                  │ │
│  │                           │                 │                  │ │
│  │                           │  PC: 0x5678     │                  │ │
│  │                           │  R1: 200        │                  │ │
│  │                           │  SP: 0x9000     │                  │ │
│  │                           └─────────────────┘                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【문맥 교환 단계】                                                  │
│  ──────────────────                                                  │
│  1. 인터럽트/트랩 발생                                                │
│  2. 커널 모드 진입                                                   │
│  3. 현재 프로세스 문맥 저장 (PCB)                                    │
│  4. 스케줄러 호출 → 다음 프로세스 선택                               │
│  5. 선택된 프로세스 문맥 복원                                        │
│  6. 사용자 모드 복귀                                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 문맥 교환 × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                문맥 교환 × 타 과목 융합                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CPU 스케줄링】                                                    │
│  • 타임 퀀텀: 문맥 교환 주기 결정                                   │
│  • 스케줄링 알고리즘: 문맥 교환 빈도 영향                           │
│  • 선점형 vs 비선점형: 문맥 교환 발생 조건                          │
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 레지스터: 저장/복원 대상                                          │
│  • 캐시 계층: 문맥 교환 시 오염                                      │
│  • TLB: 주소 공간 변경 시 플러시                                    │
│  • 파이프라인: 문맥 교환 시 플러시                                   │
│                                                                     │
│  【성능 분석】                                                        │
│  • 문맥 교환 오버헤드: CPU 시간 소비                                │
│  • 최적화: 문맥 교환 최소화 전략                                    │
│  • 측정: perf, vmstat 활용                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  【문맥 교환 최적화 기법】                                             │
│  ──────────────────                                                  │
│  1. PCID (Process-Context Identifier)                              │
│     • TLB 엔트리에 프로세스 ID 태그                                  │
│     • TLB 전체 플러시 회피                                          │
│                                                                     │
│  2. ASID (Address Space Identifier)                                │
│     • ARM, MIPS 등에서 사용                                         │
│     • TLB 플러시 없이 문맥 교환                                     │
│                                                                     │
│  3. 경량 문맥 교환                                                   │
│     • 필수 레지스터만 저장                                           │
│     • lazy FPU 저장                                                  │
│                                                                     │
│  4. 스레드 풀                                                        │
│     • 문맥 교환 최소화                                               │
│     • 작업 큐 사용                                                   │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 문맥 교환 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                문맥 교환 분석                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【문맥 교환 통계 확인】                                              │
│  ──────────────────                                                  │
│  # 시스템 전체 문맥 교환                                             │
│  $ cat /proc/stat | grep ctxt                                  │
│  $ vmstat 1                                                     │
│                                                                     │
│  # 프로세스별 문맥 교환                                              │
│  $ cat /proc/<pid>/status | grep ctxt                          │
│                                                                     │
│  # CPU별 문맥 교환                                                   │
│  $ mpstat -P ALL 1                                              │
│                                                                     │
│  【문맥 교환 성능 분석】                                             │
│  ──────────────────                                                  │
│  # perf로 측정                                                       │
│  $ perf stat -e context-switches ./program                      │
│  $ perf record -e context-switches ./program                    │
│                                                                     │
│  # 문맥 교환 추적                                                    │
│  $ perf trace -e context-switches                               │
│                                                                     │
│  【문맥 교환 최적화】                                                 │
│  ──────────────────                                                  │
│  1. 스레드 사용 (프로세스 대신)                                      │
│  2. CPU 선호도 (affinity) 설정                                      │
│  3. 적절한 타임 퀀텀 설정                                            │
│  4. 비선점형 섹션 사용                                               │
│  5. I/O 멀티플렉싱 활용                                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 문맥 교환: CPU 실행 프로세스 전환 과정
• 단계: 저장 → 스케줄링 → 복원
• 발생: 타임슬라이스, I/O, 인터럽트, 우선순위
• 비용: 직접비용 + 캐시오염 + TLB플러시
• 프로세스 > 스레드: 스레드 문맥 교환 더 빠름
• 최적화: PCID, ASID, 경량 교환, 스레드 풀
```
