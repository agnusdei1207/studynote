## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 강결합 시스템(Tightly Coupled System)은 여러 개의 프로세서(CPU)가 메인 메모리, 시스템 클럭, 고속 버스 등 핵심 하드웨어 자원을 물리적으로 공유하는 다중 처리기 시스템이다.
> 2. **가치**: 프로세서 간 통신이 공유 메모리를 통해 이루어지므로 지연 시간(Latency)이 매우 짧고, 데이터 교환 속도가 극도로 빨라 높은 대역폭과 병렬 처리 성능을 제공한다.
> 3. **융합**: 우리가 흔히 사용하는 멀티코어 PC와 서버 시스템(SMP 등)이 대표적인 강결합 시스템이며, 프로세스 동기화와 캐시 일관성 유지가 시스템 성능을 좌우하는 핵심 기술이다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 강결합 시스템은 여러 프로세서가 하나의 운영체제 하에서 밀접하게 묶여(Tightly) 작동하는 시스템 구조다. 프로세서들은 하나의 물리적인 공유 메모리 공간(Global Shared Memory)을 가지며, 이를 통해 서로 데이터를 주고받는다.
- **💡 비유**: 강결합 시스템은 **하나의 커다란 칠판을 함께 보며 문제를 푸는 학생들**과 같다. 모든 학생(프로세서)이 같은 교실(메인보드) 안에서 같은 칠판(공유 메모리)을 보고 쓰기 때문에, 누군가 식을 고치면 즉시 다른 학생도 그 변화를 알 수 있고 소통이 매우 빠르다.
- **특징**:
  - 단일 운영체제가 시스템 전체 자원을 제어한다.
  - 프로세서 간 거리가 매우 짧아(보통 같은 칩 또는 보드 내) 고속 버스로 연결된다.
  - 메모리 충돌을 막기 위한 동기화(Synchronization) 작업이 필수적이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 공유 메모리 기반의 통신 아키텍처

강결합 시스템의 가장 큰 물리적 특징은 메인 메모리의 '공유'다. 프로세서 A가 메모리 특정 주소에 값을 쓰면, 프로세서 B는 즉시 그 주소를 읽어 통신을 완료한다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │               강결합 시스템 (Tightly Coupled System) 구조          │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │          ┌───────┐        ┌───────┐        ┌───────┐          │
 │          │ CPU A │        │ CPU B │        │ CPU C │          │
 │          └───┬───┘        └───┬───┘        └───┬───┘          │
 │              │                │                │              │
 │  ════════════╧════════════════╧════════════════╧═══════════   │
 │              고속 시스템 버스 (High-Speed System Bus)              │
 │  ═════════════════════════════╤════════════════════════════   │
 │                               │                               │
 │                       ┌───────┴───────┐                       │
 │                       │  Shared Memory│                       │
 │                       │  (공유 메모리)   │                       │
 │                       └───────────────┘                       │
 │                                                               │
 │  통신 방식: CPU A가 메모리 [0x1000]에 기록 → CPU B가 [0x1000] 읽음 │
 │  특징: 지연 시간 최소화 (Nano~Micro 초 단위), 높은 캐시 일관성 요구  │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 강결합 시스템에서는 프로세서 간의 물리적 장벽이 없다. 별도의 네트워크 프로토콜(TCP/IP 등) 없이 메모리 읽기/쓰기 명령어(Load/Store)만으로 즉각적인 정보 교환이 일어난다. 운영체제는 이 거대한 공유 메모리 풀을 논리적으로 분할하여 각 프로세스에 할당하고, 코어들이 동시에 접근할 때 발생할 수 있는 락(Lock) 경합을 제어한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 강결합 시스템 vs 약결합 시스템

컴퓨터 시스템의 연결 정도(Coupling)에 따라 강결합과 약결합(Loosely Coupled)으로 명확히 대비된다.

| 비교 항목 | 강결합 시스템 (Tightly Coupled) | 약결합 시스템 (Loosely Coupled) |
|:---|:---|:---|
| **물리적 구조** | 단일 보드/칩 내 멀티 프로세서 (SMP, 멀티코어) | 네트워크로 연결된 독립된 컴퓨터(노드) 들 (클러스터) |
| **메모리** | **공유 메모리 (Shared Memory)** | 분산 메모리 (각 노드가 독자적 메모리 소유) |
| **운영체제** | 단일 운영체제 (하나의 OS가 전체 통제) | 각 노드별로 독립된 운영체제 (분산 OS) |
| **통신 방식** | 메모리 읽기/쓰기 (고속 버스) | **메시지 전달 (Message Passing, 네트워크)** |
| **확장성** | 하드웨어 버스 한계로 확장성 제한 (Scale-up) | 노드 추가가 자유로워 확장성 무한 (Scale-out) |
| **속도/지연** | 초고속 통신, 지연 시간 극히 낮음 | 네트워크 오버헤드로 인한 높은 지연 시간 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 강결합 시스템의 확장성 한계 (Scalability Limit)
- **상황**: 대규모 데이터베이스(RDBMS) 서버를 운영 중이다. 데이터량이 늘어나자 관리자는 강결합 시스템인 서버의 CPU 코어를 32개에서 128개로 늘렸다(Scale-up). 그러나 기대와 달리 코어 64개 이상부터는 성능 향상 폭이 급격히 둔화되었다.
- **원인 분석**: 강결합 시스템은 모든 코어가 동일한 공유 메모리와 시스템 버스를 사용한다. 코어가 너무 많아지면 메모리에 접근하기 위한 버스 경합(Bus Contention)이 극심해지고, 캐시 일관성을 맞추기 위한 스누핑(Snooping) 트래픽이 폭주하여 대역폭이 포화상태에 이른다.
- **판단**: 강결합 시스템만으로는 성장에 한계가 있다. 이를 극복하기 위해 물리적 메모리를 논리적으로 분산시키는 NUMA 아키텍처를 적용하거나, 아예 강결합 시스템 여러 대를 네트워크로 묶는 **약결합 기반의 클러스터링(Scale-out)** 아키텍처로 데이터베이스를 분산(Sharding)시켜야 한다.

### 안티패턴
- 코어 간 잦은 데이터 교환이 없는 완전히 독립적인 배치 작업(Batch Job) 수천 개를 돌리기 위해 초고가의 대형 강결합 서버(Mainframe)를 구매하는 것. 독립적인 작업은 통신 오버헤드가 적으므로, 저렴한 컴퓨터 수십 대를 묶은 약결합(클러스터) 시스템을 구축하는 것이 비용(ROI) 측면에서 훨씬 유리하다.

---

## Ⅴ. 기대효과 및 결론

- 강결합 시스템은 복잡한 물리 시뮬레이션, 대용량 트랜잭션 처리(OLTP) 등 코어 간 데이터 교환이 빈번하고 즉각적인 처리가 필요한 환경에서 압도적인 위력을 발휘한다.
- 운영체제는 이러한 강결합 환경에서 자원 접근의 동기화와 병목 해소를 책임지는 가장 핵심적인 조율자 역할을 수행한다. 우리가 사용하는 스마트폰의 AP부터 클라우드 데이터센터의 물리 서버 한 대 한 대가 모두 완벽한 강결합 시스템의 산물이다.
