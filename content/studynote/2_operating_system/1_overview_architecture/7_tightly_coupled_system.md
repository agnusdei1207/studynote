# 강결합 시스템 (Tightly Coupled System)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 강결합 시스템은 여러 개의 프로세서가 **단일 주메모리(Shared Memory)**와 시스템 버스를 물리적으로 공유하며, 하나의 운영체제(Single OS)에 의해 전체 자원이 관리되는 병렬 컴퓨팅 구조다.
> 2. **가치**: 프로세서 간의 데이터 교환이 메모리를 통해 직접 이루어지므로 **통신 지연(Latency)이 극도로 낮으며**, 대규모 연산 자원을 하나의 논리적 개체로 활용하여 연산 밀도를 극대화할 수 있다.
> 3. **융합**: 현대 멀티코어 서버와 고성능 HPC(High Performance Computing)의 근간이 되며, 자원 공유에 따른 버스 경합(Contention)과 **동기화(Synchronization)** 오버헤드를 줄이는 것이 시스템 성능의 핵심 변수다.

+++

## Ⅰ. 강결합 시스템 (Tightly Coupled System)의 개요

### 1. 정의
- 동일한 메모리와 입출력 채널을 공유하는 두 개 이상의 프로세서로 구성된 시스템으로, 하드웨어적 연결성이 매우 긴밀한 '다중 처리기(Multiprocessor)' 시스템이다.

### 2. 특징
- **공유 메모리**: 모든 프로세서가 주소 공간을 공유하며 통신한다.
- **단일 운영체제**: 하나의 OS가 시스템 전체를 제어하고 부하를 분산한다.
- **고속 통신**: 프로세서 간 통신 속도가 프로세서 처리 속도와 대등하다.

### 3. 💡 비유: 한솥밥을 먹는 가족
- 강결합 시스템은 **"한 식탁(메모리)에서 같이 밥을 먹는 가족"**과 같다.
- 반찬(데이터)이 식탁 중앙에 있으므로 누구나 즉시 집어 먹을 수 있고, 서로 대화(통신)하기도 매우 쉽다. 하지만 식탁이 좁으면(버스 경합) 서로 팔꿈치가 부딪혀 불편해질 수 있다.

+++

## Ⅱ. 강결합 시스템의 아키텍처 및 통신 원리 (ASCII)

프로세서들이 공통 자원을 향해 집중된 구조를 띤다.

```text
    [ Processor 1 ]  [ Processor 2 ]  [ Processor n ]
           │                │                │
    ┌──────┴────────────────┴────────────────┴──────┐
    │           [ 고속 시스템 버스 / 스위치 ]       │
    └──────┬────────────────┬────────────────┬──────┘
           │                │                │
    ┌──────┴──────┐  ┌──────┴──────┐  ┌──────┴──────┐
    │  Shared RAM │  │  Shared I/O │  │  Global OS  │
    └─────────────┘  └─────────────┘  └─────────────┘
```

**[핵심 메커니즘]**
1. **메모리 기반 통신**: P1이 데이터를 특정 주소에 쓰면, P2는 별도의 네트워크 전송 없이 해당 주소를 읽음으로써 즉시 정보를 획득한다.
2. **락(Locking) 메커니즘**: 공유 메모리의 데이터 일관성을 지키기 위해 하드웨어적 `Test-and-Set`이나 소프트웨어적 `Semaphore` 등을 사용하여 접근을 제어한다.
3. **버스 중재**: 여러 CPU가 동시에 메모리에 접근할 때 순서를 정해주는 중재기(Arbiter)가 필수적이다.

+++

## Ⅲ. 강결합 vs 약결합 시스템 비교

| 비교 항목 | 강결합 (Tightly Coupled) | 약결합 (Loosely Coupled) |
|:---|:---|:---|
| **공유 자원** | **주메모리, I/O, OS 공유** | 각자 독립된 메모리, OS 보유 |
| **통신 방법** | 공유 메모리 (Shared Memory) | 메시지 전달 (Message Passing) |
| **결합 형태** | 다중 처리기 (Multiprocessor) | 다중 컴퓨터 (Multicomputer) |
| **통신 속도** | **매우 빠름 (Low Latency)** | 상대적으로 느림 (Network Latency) |
| **운영체제** | **단일 OS (Single OS)** | 독립 OS (Distributed OS) |
| **사례** | 메인프레임, 멀티코어 서버 | 분산 시스템, 클러스터 (L4 스위치 결합) |

- **📢 섹션 요약 비유**: 한 사무실에 모여 포스트잇(메모리)으로 정보를 주고받는 팀이 강결합이라면, 각자 집에서 메일(네트워크)로 소통하는 팀이 약결합입니다.

+++

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy)

### 1. 확장성(Scalability)의 한계와 임계점
- **기술사적 인사이트**: 강결합 시스템은 버스 경합 때문에 프로세서 수를 무한정 늘릴 수 없다. 통상 16~32개 이상의 프로세서가 붙으면 성능 향상보다 관리 비용(Snooping, Locking)이 더 커지는 **확장성 저하(Scalability Wall)**가 발생한다.
- **결단**: 이를 해결하기 위해 물리적으로는 메모리를 나누되 논리적으로는 공유하는 **NUMA** 기술을 도입하여 강결합의 장점과 확장성을 동시에 잡아야 한다.

### 2. 실시간성 요구 시스템에서의 선택
- **판단**: 마이크로초($\mu s$) 단위의 정밀한 제어가 필요한 미사일 제어 시스템이나 로봇 팔 제어기에서는 네트워크 지연이 치명적이다. 
- **인사이트**: 따라서 이러한 분야는 반드시 **강결합 아키텍처**를 기반으로 설계하여 결정론적 응답(Deterministic Response)을 보장해야 한다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **응답 성능 최적화**: 프로세서 간 통신 오버헤드 최소화.
- **프로그래밍 편의성**: 개발자가 데이터를 주고받기 위한 복잡한 네트워크 로직을 짤 필요가 없음.

### 2. 미래 전망
최근의 칩렛(Chiplet) 아키텍처는 개별 다이(Die)들을 강결합하여 하나의 거대한 프로세서처럼 동작하게 만든다. 또한 하드웨어 가속기(GPU, NPU)와 CPU 간의 **CXL (Compute Express Link)** 기술은 이질적인 장치들 사이를 '강결합' 수준의 메모리 공유 환경으로 묶어주고 있다. 결국 시스템의 성능은 '얼마나 긴밀하게 자원을 나눌 수 있는가'라는 강결합의 원리로 귀결될 것이다.

- **📢 섹션 요약 비유**: 혼자 하면 10시간 걸릴 일을 열 명이 한마음으로 1시간 만에 끝내는 '팀워크의 정점'이 바로 강결합 시스템입니다.

+++

### 📌 관련 개념 맵 (Knowledge Graph)
- **SMP (Symmetric Multiprocessing)**: 강결합 시스템의 가장 대표적인 소프트웨어 운영 방식.
- **NUMA (Non-Uniform Memory Access)**: 강결합 아키텍처의 확장성 대안.
- **공유 메모리 (Shared Memory)**: 강결합 시스템의 핵심 하드웨어 요건.

+++

### 👶 어린이를 위한 3줄 비유 설명
1. 강결합 시스템은 여러 명의 친구가 **'하나의 커다란 도화지'에 같이 그림을 그리는 것**과 같아요.
2. 내 옆 친구가 무엇을 그리는지 바로 볼 수 있어서(메모리 공유) 서로 돕기가 아주 쉽고 빠르죠.
3. 하지만 도화지가 하나라 너무 많은 친구가 모이면 서로 팔이 부딪혀서 방해될 수도 있답니다!
