# 021. 모놀리식 커널 (Monolithic Kernel)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 커널 서비스가 단일 주소 공간에서 실행되는 구조
> 2. **가치**: 높은 성능, 직접 함수 호출, 단순한 구조
> 3. **융합**: Linux, BSD, 초기 Unix와 연관

---

## Ⅰ. 개요

### 개념 정의

모놀리식 커널(Monolithic Kernel)은 **모든 커널 서비스(스케줄링, 메모리 관리, 파일 시스템, 장치 드라이버)가 단일 커널 주소 공간에서 실행되는 커널 아키텍처**다.

### 💡 비유: 단일 큰 건물
모놀리식 커널은 **모든 부서가 하나의 큰 건물에 있는 회사**와 같다. 모든 직원(커널 서비스)이 같은 층에서 일하므로 소통(함수 호출)이 빠르다. 하지만 한 부서에 불이 나면(버그) 전체 건물이 위험하다.

## Ⅱ. 모놀리식 커널 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                모놀리식 커널 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 높은 성능                                                   │ │
│  │     • 직접 함수 호출 (IPC 오버헤드 없음)                         │ │
│  │     • 컨텍스트 스위치 최소화                                     │ │
│  │     • 메모리 접근 효율                                          │ │
│  │                                                               │ │
│  │  2. 구현 단순                                                   │ │
│  │     • 단일 주소 공간                                            │ │
│  │     • 직관적인 설계                                             │ │
│  │     • 디버깅 용이                                               │ │
│  │                                                               │ │
│  │  3. 풍부한 기능                                                 │ │
│  │     • 모든 서비스 내장                                          │ │
│  │     • 광범위한 하드웨어 지원                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 크기                                                        │ │
│  │     • 커널 이미지 크기 큼                                        │ │
│  │     • 메모리 사용량 많음                                         │ │
│  │                                                               │ │
│  │  2. 신뢰성                                                      │ │
│  │     • 드라이버 버그가 전체 커널 영향                              │ │
│  │     • 커널 패닉 가능성                                           │ │
│  │                                                               │ │
│  │  3. 유지보수                                                     │ │
│  │     • 코드베이스 크기 방대                                        │ │
│  │     • 수정 시 전체 재컴파일 필요                                  │ │
│  │                                                               │ │
│  │  4. 확장성                                                      │ │
│  │     • 새 기능 추가 복잡                                          │ │
│  │     • 모듈화 노력 필요                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【모놀리식 커널 예시】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  커널              특징                                        │ │
│  │  ────              ────                                        │ │
│  │  Linux            모듈 지원, 현대적 모놀리식                    │ │
│  │  BSD (FreeBSD)    모놀리식, 모듈화 우수                         │ │
│  │  Unix (전통)      초기 모놀리식 구조                            │ │
│  │  MS-DOS           단순 모놀리식                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  │                                                               │ │

│  │  장점:                                                         │ │
│  │  • 런타임에 드라이버 추가/제거                                  │ │
│  │  • 커널 크기 최소화                                             │ │
│  │  • 개발/디버깅 용이                                             │ │
│  │                                                               │ │
│  │  단점:                                                         │ │

│  │  $ rmmod          # 모듈 언로드                                 │ │

└─────────────────────────────────────────────────────────────────────┘

## Ⅳ. 실무 적용

### 모놀리식 커널 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                모놀리식 커널 분석                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 확인】                                                  │
│  ──────────────────                                                  │
│  # 커널 버전                                                         │
│  $ uname -r                                                         │
│                                                                     │
│  # 커널 이미지 크기                                                   │
│  $ ls -lh /boot/vmlinuz-*                                           │
│                                                                     │
│  # 로드된 모듈                                                       │
│  $ lsmod | head -20                                                 │
│                                                                     │
│  # 커널 설정                                                         │
│  $ cat /boot/config-$(uname -r) | grep -i module                   │
│                                                                     │
│  【커널 모듈 개발】                                                    │
│  ──────────────────                                                  │
│  // 간단한 커널 모듈 예시                                             │
│  #include <linux/module.h>                                          │
│  #include <linux/kernel.h>                                          │
│                                                                     │
│  static int __init hello_init(void) {                              │
│      printk(KERN_INFO "Hello Kernel!\n");                          │
│      return 0;                                                      │
│  }                                                                  │
│                                                                     │
│  static void __exit hello_exit(void) {                             │
│      printk(KERN_INFO "Goodbye Kernel!\n");                        │
│  }                                                                  │
│                                                                     │
│  module_init(hello_init);                                           │
│  module_exit(hello_exit);                                           │
│                                                                     │
│  【모놀리식 커널 최적화】                                              │
│  ──────────────────                                                  │
│  1. 필요한 드라이버만 컴파일                                          │
│  2. 모듈로 분리 가능한 것은 모듈화                                     │
│  3. 디버깅 심볼 제거                                                 │
│  4. LKM (Loadable Kernel Module) 활용                               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 모놀리식: 모든 서비스가 단일 커널 공간 실행
• 장점: 높은 성능, 직접 호출, 구현 단순
• 단점: 크기 큼, 신뢰성 낮음, 유지보수 어려움
• 예시: Linux, BSD, 전통 Unix
• Linux: 모듈 지원으로 유연성 확보
• 선택: 성능 중시 → 모놀리식, 신뢰성 중시 → 마이크로
```
