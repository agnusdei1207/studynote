# 회계 및 로깅 (Accounting & Logging)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 회계(Accounting)는 시스템 자원 사용량을 추적하여 비용 산정과 용량 계획을 지원하고, 로깅(Logging)은 시스템 이벤트를 기록하여 장애 진단과 보안 감사를 가능하게 한다.
> 2. **가치**: 정확한 회계 데이터는 **비용 최적화**와 **과금 모델**의 기반이 되며, 체계적인 로깅은 **MTTR(평균 복구 시간) 단축**과 **규정 준수(Compliance)**를 보장한다.
> 3. **융합**: 클라우드 FinOps, SIEM(Security Information and Event Management), APM(Application Performance Monitoring)은 회계와 로깅 기술의 현대적 진화 형태다.

+++

## Ⅰ. 개요 (Context & Background)

### 1. 정의
- **회계(Accounting)**: 사용자별, 프로세스별 자원 사용량(CPU 시간, 메모리, 디스크 I/O, 네트워크)을 수집, 저장, 분석하여 비용 산정과 자원 관리를 지원하는 OS 기능이다.
- **로깅(Logging)**: 시스템 이벤트, 오류, 사용자 활동, 보안 관련 사항을 시계열로 기록하여 추적 및 분석을 가능하게 하는 메커니즘이다.

### 2. 💡 비유: 가계부와 CCTV
- 회계는 **"가계부"**와 같다. 언제, 무엇을, 얼마나 사용했는지 기록하여 나중에 지출을 분석하고 예산을 계획할 수 있다.
- 로깅은 **"CCTV"**와 같다. 건물 내 모든 활동을 시간 순서대로 녹화하여 문제 발생 시 무슨 일이 있었는지 정확히 파악할 수 있다.

### 3. 등장 배경
- **멀티유저 시스템**: 여러 사용자가 자원을 공유하면서 사용량 추적 필요
- **비용 정산**: 기업 환경에서 부서별/프로젝트별 IT 비용 배분
- **보안 감사**: 규정 준수(PCI-DSS, SOX, GDPR) 이력 보관 의무
- **장애 진단**: 시스템 문제 발생 시 원인 분석을 위한 이력 필요

### 4. 회계 vs 로깅 비교

| 구분 | 회계 (Accounting) | 로깅 (Logging) |
|:---|:---|:---|
| **목적** | 자원 사용량 측정, 비용 산정 | 이벤트 기록, 장애 진단, 감사 |
| **데이터** | 정량적 (수치, 시간, 양) | 정성적 (메시지, 상태, 이벤트) |
| **주요 소비자** | 관리자, 재무팀, FinOps | 개발자, 보안팀, SOC |
| **보관 기간** | 과금 주기별 (월/분기) | 규정에 따름 (1-7년) |
| **대표 도구** | sar, psacct, CloudWatch | syslog, journald, ELK |

- **📢 섹션 요약 비유**: 회계는 은행 거래 내역처럼 '얼마나'를 추적하고, 로깅은 블랙박스처럼 '무슨 일이' 일어났는지를 기록합니다.

### 2. 구성 요소 상세

| 구성 요소 | Full Name | 역할 | 대표 구현 |
|:---|:---|:---|:---|
| **Instrumentation** | 계측 | 커널/애플리케이션에서 데이터 수집 | eBPF, DTrace, Auditd |
| **Collector** | 수집기 | 데이터 중계 및 전처리 | rsyslog, fluentd, vector |
| **Storage** | 저장소 | 시계열/로그 데이터 영속화 | Prometheus, Elasticsearch |
| **Analyzer** | 분석기 | 패턴 탐지, 이상 징후 식별 | ML 기반, Rule 기반 |
| **Visualizer** | 시각화 | 대시보드, 리포트 생성 | Grafana, Kibana |

### 4. 회계 데이터 수집 메커니즘

```text
    ┌─────────────────────────────────────────────────────────────────────────┐
    │                   Process Accounting Flow                               │
    │                                                                         │
    │  [프로세스 종료 시]                                                     │
    │  ┌───────────────┐                                                      │
    │  │ Process       │                                                      │
    │  │ Termination   │                                                      │
    │  └───────┬───────┘                                                      │
    │          │                                                              │
    │          ▼                                                              │
    │  ┌───────────────────────────────────────────────────────────────┐     │
    │  │                    Kernel Accounting Hook                      │     │
    │  │                                                                │     │
    │  │  struct acct {                                                 │     │
    │  │    char      ac_flag;      // 종료 상태 플래그                 │     │
    │  │    uid_t     ac_uid;       // 실제 사용자 ID                   │     │
    │  │    gid_t     ac_gid;       // 그룹 ID                          │     │
    │  │    time_t    ac_btime;     // 프로세스 시작 시간               │     │
    │  │    comp_t    ac_utime;     // 사용자 CPU 시간                  │     │
    │  │    comp_t    ac_stime;     // 시스템 CPU 시간                  │     │
    │  │    comp_t    ac_mem;       // 평균 메모리 사용량               │     │
    │  │    comp_t    ac_io;        // I/O 블록 수                      │     │
    │  │    ...                                                          │     │
    │  │  }                                                             │     │
    │  └──────────────────────────┬────────────────────────────────────┘     │
    │                             │                                          │
    │                             ▼                                          │
    │  ┌───────────────────────────────────────────────────────────────┐     │
    │  │              /var/account/pacct (Accounting File)              │     │
    │  │              이진 형식으로 프로세스별 사용량 기록              │     │
    │  └───────────────────────────────────────────────────────────────┘     │
    │                                                                         │
    │  [분석 도구]                                                            │
    │  - lastcomm: 사용자/명령별 실행 이력                                    │
    │  - sa: 명령별 자원 사용량 요약                                          │
    │  - accton: 회계 기능 on/off                                             │
    └─────────────────────────────────────────────────────────────────────────┘
```

**[회계 메커니즘 해설]**
1. **커널 훅**: 프로세스 종료 시 `do_acct_process()`가 호출되어 회계 레코드 생성
2. **데이터 수집**: CPU 시간, 메모리, I/O 등 자원 사용량을 구조체에 패킹
3. **영속화**: 이진 파일로 저장하여 추후 분석 도구가 읽을 수 있게 함
4. **확장**: 현대 시스템은 cgroups + Prometheus로 실시간 메트릭 수집

- **📢 섹션 요약 비유**: 회계 데이터는 영수증처럼 '무엇을 얼마나 샀는지'를 증명하고, 로그는 일기처럼 '무슨 일이 있었는지'를 서술합니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 1. 로깅 프레임워크 비교

| 프레임워크 | 언어/플랫폼 | 특징 | 구조화 지원 |
|:---|:---|:---|:---|
| **syslog** | Unix 표준 | 중앙 집중식, facility/priority | 아니요 |
| **journald** | systemd | 바이너리, 메타데이터 풍부 | 예 (JSON) |
| **Log4j/SLF4J** | Java | 유연한 설정, MDC | 예 |
| **Winston** | Node.js | 다중 전송(transport) | 예 |
| **structlog** | Python | 컨텍스트 바인딩 | 예 |

### 2. 회계 vs 모니터링 vs 추적(Tracing)

| 구분 | Accounting | Monitoring | Tracing |
|:---|:---|:---|:---|
| **질문** | 얼마나 썼나? | 지금 어떤 상태인가? | 요청이 어떻게 흘렀나? |
| **시간성** | 사후 집계 | 실시간 | 요청 단위 |
| **데이터** | 집계된 수치 | 현재 값 + 시계열 | 분산 컨텍스트 |
| **용도** | 비용 정산, 용량 계획 | 알림, 대시보드 | 장애 원인 분석 |
| **도구** | psacct, CloudWatch | Prometheus, Datadog | Jaeger, Zipkin |

### 3. 과목 융합 관점

| 융합 과목 | 연관 개념 | 융합 포인트 |
|:---|:---|:---|
| **데이터베이스** | WAL, 트랜잭션 로그 | 장애 복구, 시점 복구 |
| **정보 보안** | 감사 로그, SIEM | 침입 탐지, 포렌식 |
| **네트워크** | NetFlow, sFlow | 트래픽 회계, 대역폭 관리 |
| **소프트웨어 공학** | APM, Observability | 3 pillars (Logs, Metrics, Traces) |
| **클라우드 컴퓨팅** | FinOps, Showback | 클라우드 비용 최적화 |

- **📢 섹션 요약 비유**: 회계는 나라의 세금 징수 기록관처럼 자원 사용을 측정하고, 로깅은 역사가처럼 시스템의 모든 사건을 기록하여 미래를 위한 교훈을 남깁니다.

```text
    │  - Chargeback: 실제 부서 비용 전가                                  │
    └─────────────────────────────────────────────────────────────────────┘
```

### 2. 도입 체크리스트

| 항목 | 체크 포인트 | 권장 사항 |
|:---|:---|:---|
| 로그 표준 | 포맷, 레벨, 필드 정의 | JSON 구조화, ISO 8601 타임스탬프 |
| 보관 정책 | 규정 준수 요구사항 | hot-warm-cold 아카이빙 |

| 알림 | 임계값 기반 경보 | ERROR+ 레벨 즉시 알림 |
