# 031. 문맥 (Context)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 실행 상태를 나타내는 모든 정보의 집합
> 2. **가치**: 실행 재개, 문맥 교환, 멀티태스킹 기반
> 3. **융합**: PCB, 레지스터, 스택과 연관

---

## Ⅰ. 개요

### 개념 정의

문맥(Context)은 **프로세스의 실행 상태를 나타내는 모든 정보의 집합**으로, CPU 레지스터 값, 프로그램 카운터, 스택 포인터, 메모리 상태 등을 포함한다. 문맥이 저장되면 프로세스를 중단했다가 나중에 정확히 같은 지점에서 재개할 수 있다.

### 💡 비유: 책갈피가 있는 독서
문맥은 **책을 읽다가 책갈피를 꽂아두는 것**과 같다. 책(프로그램)을 읽다가 잠시 멈추고 책갈피(문맥)를 꽂아둔다. 나중에 다시 읽을 때 책갈피 위치에서 계속 읽을 수 있다. 어떤 페이지(메모리), 어느 줄(PC), 무슨 생각을 하고 있었는지(레지스터) 모두 기록된다.

## Ⅲ. 문맥 × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                문맥 × 타 과목 융합                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 레지스터 파일: 문맥 저장 위치                                    │
│  • PC/IR: 명령어 실행 위치                                          │
│  • 파이프라인: 문맥 저장 시 플러시                                   │
│  • 캐시: 문맥 교환 시 캐시 오염                                     │
│                                                                     │
│  【CPU 스케줄링】                                                    │
│  • 문맥 교환: 프로세스 전환 비용                                    │
│  • 타임 퀀텀: 문맥 교환 주기                                        │
│  • 선점: 문맥 저장/복원                                            │
│                                                                     │
│  【메모리 관리】                                                      │
│  • TLB: 문맥 교환 시 TLB 플러시                                     │
│  • 페이지 테이블: 주소 공간 문맥                                    │
│  • 캐시 일관성: 문맥 교환 영향                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  【문맥 교환 비용】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  비용 요소            영향                    크기               │ │
│  │  ────                ────                    ────               │ │
│  │  직접 비용            레지스터 저장/복원       수백 ns~수 μs      │ │
│  │  캐시 오염            L1/L2 캐시 미스         수 μs~수십 μs      │ │
│  │  TLB 플러시           페이지 테이블 재로드     수 μs              │ │
│  │  파이프라인 플러시    파이프라인 재시작       수십 ns            │ │
│  │                                                               │ │
│  │  총 비용: 수 μs ~ 수백 μs (시스템에 따라 다름)                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 문맥 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                문맥 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프로세스 문맥 확인 (Linux)】                                      │
│  ──────────────────                                                  │
│  # 프로세스 정보                                                    │
│  $ cat /proc/<pid>/status                                      │
│  $ cat /proc/<pid>/stat                                        │
│                                                                     │
│  # 레지스터 상태 (디버거)                                             │
│  $ gdb -p <pid>                                                │
│  (gdb) info registers                                          │
│                                                                     │
│  # 메모리 맵                                                        │
│  $ cat /proc/<pid>/maps                                        │
│                                                                     │
│  【문맥 교환 통계】                                                  │
│  ──────────────────                                                  │
│  # 문맥 교환 횟수                                                   │
│  $ cat /proc/<pid>/status | grep ctxt                          │
│                                                                     │
│  # 시스템 전체 문맥 교환                                             │
│  $ cat /proc/stat | grep ctxt                                  │
│  $ vmstat -s | grep context                                    │
│                                                                     │
│  # 문맥 교환 모니터링                                                │
│  $ perf stat -e context-switches -a sleep 10                   │
│                                                                     │
│  【스레드 문맥】                                                      │
│  ──────────────────                                                  │
│  • 공유: 주소 공간, 파일, 신호                                       │
│  • 개별: 레지스터, 스택, PC                                         │
│                                                                     │
│  # 스레드별 문맥 교환                                                │
│  $ pidstat -t -p <pid>                                         │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 문맥: 프로세스 실행 상태의 모든 정보
• 구성: 레지스터, PC, 스택, 메모리, PCB
• 저장: 인터럽트, 시스템 콜, 문맥 교환 시
• 비용: 직접 비용 + 캐시 오염 + TLB 플러시
• PCB: 문맥 정보 저장 구조체
• 스레드: 주소 공간 공유, 레지스터/스택 개별
```
