# 018. ISR (인터럽트 서비스 루틴)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 발생 시 실행되는 특수 함수
> 2. **가치**: 빠른 응답, 하드웨어 이벤트 처리
> 3. **융합**: Top Half, Bottom Half, 동기화와 연관

---

## Ⅰ. 개요

### 개념 정의

ISR(Interrupt Service Routine)은 **인터럽트가 발생했을 때 실행되는 커널 모드의 특수 처리 함수**로, 하드웨어 이벤트에 대한 응답을 처리한다.

### 💡 비유: 119 응급 구조대
ISR은 **119 응급 구조대**와 같다. 신고(인터럽트)가 들어오면 즉시 출동한다. 현장에서 긴급 조치(Top Half)만 하고, 나머지 병원 이송(Bottom Half)은 나중에 한다. 다른 신고를 받기 위해 빨리 복귀해야 한다.

### ISR 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                ISR 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ISR 실행 흐름】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Hardware                                                     │ │
│  │    │                                                          │ │
│  │    │ 인터럽트 신호                                              │ │
│  │    ▼                                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │           CPU (자동 하드웨어 처리)                        │ │ │
│  │  │  1. 현재 명령 완료 대기                                    │ │ │
│  │  │  2. EFLAGS, CS, EIP 저장 (스택)                           │ │ │
│  │  │  3. 인터럽트 비활성화 (IF=0)                                │ │ │
│  │  │  4. IDT에서 ISR 주소 찾기                                   │ │ │
│  │  └───────────────────────────┬─────────────────────────────┘ │ │
│  │                              │                                  │ │
│  │                              ▼                                  │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │           ISR (Top Half)                                 │ │ │
│  │  │  1. 레지스터 저장 (save_all)                               │ │ │
│  │  │  2. 인터럽트 원인 확인                                     │ │ │
│  │  │  3. 하드웨어 처리 (최소한)                                  │ │ │
│  │  │  4. Bottom Half 스케줄링 (필요시)                          │ │ │
│  │  │  5. 레지스터 복원 (restore_all)                            │ │ │
│  │  │  6. iret (인터럽트 복귀)                                   │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ISR 제약사항】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 인터럽트 컨텍스트에서 실행                                  │ │
│  │     • 프로세스 컨텍스트 없음                                    │ │
│  │     • sleep() 불가                                             │ │
│  │                                                               │ │
│  │  2. 실행 시간 최소화                                            │ │
│  │     • 다른 인터럽트 지연                                        │ │
│  │     • 시스템 응답성 저하                                        │ │
│  │                                                               │ │
│  │  3. 동기화 주의                                                 │ │
│  │     • 스핀락 사용 가능                                         │ │
│  │     • 뮤텍스 사용 불가 (sleep 가능성)                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. ISR 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ISR 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Top Half vs Bottom Half】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  구분              Top Half (ISR)      Bottom Half            │ │
│  │  ────              ───────────────      ────────────            │ │
│  │  실행 시점         인터럽트 즉시        나중에 스케줄            │ │
│  │  컨텍스트          인터럽트 컨텍스트    프로세스 컨텍스트         │ │
│  │  인터럽트          비활성화 가능        활성화                   │ │
│  │  선점              불가                 가능                     │ │
│  │  sleep            불가                 가능                     │ │
│  │  처리 내용         하드웨어 ACK만      데이터 처리              │ │
│  │  예시              NIC ACK             패킷 프로세싱             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Bottom Half 메커니즘】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  메커니즘          설명                    특징                  │ │
│  │  ────────          ────                    ────                  │ │
│  │  SoftIRQ          소프트 인터럽트        정적, 고성능            │ │
│  │  Tasklet          SoftIRQ 기반           동적, 간편             │ │
│  │  Workqueue        커널 스레드             sleep 가능            │ │
│  │  Threaded IRQ     IRQ 스레드             per-IRQ 스레드         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ISR 작성 가이드라인】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 최소화: 필수 작업만 수행                                    │ │
│  │  2. 비동기: sleep/wait 금지                                     │ │
│  │  3. 안전: 공유 데이터 보호                                       │ │
│  │  4. ACK: 하드웨어 인지                                           │ │
│  │  5. 위임: 긴 작업은 Bottom Half로                               │ │
│  │  6. 재진입: ISR 재진입 고려                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. ISR × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ISR × 타 과목 융합                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【실시간 시스템】                                                    │
│  • ISR 지연 최소화                                                   │
│  • 우선순위 인터럽트                                                 │
│  • ISR 내 락 사용 제한                                               │
│                                                                     │
│  【디바이스 드라이버】                                                │
│  • ISR 등록: request_irq()                                          │
│  • ISR 해제: free_irq()                                             │
│  • 데이터 전달: Bottom Half로                                        │
│                                                                     │
│  【네트워크】                                                        │
│  • NAPI: ISR + 폴링 혼합                                             │
│  • 패킷 처리: SoftIRQ (NET_RX_SOFTIRQ)                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  네트워크 패킷 처리 예시                                        │ │
│  │  ────────────────────────                                     │ │
│  │                                                               │ │
│  │  1. NIC 인터럽트 발생 (패킷 도착)                               │ │
│  │  │                                                            │ │
│  │  ▼                                                            │ │
│  │  2. ISR (Top Half):                                           │ │
│  │     • NIC 레지스터 읽기                                        │ │
│  │     • DMA로 패킷 수신 버퍼에 복사                              │ │
│  │     • NIC ACK                                                 │ │
│  │     • NET_RX_SOFTIRQ 스케줄                                   │ │
│  │     • iret                                                    │ │
│  │  │                                                            │ │
│  │  ▼                                                            │ │
│  │  3. SoftIRQ (Bottom Half):                                    │ │
│  │     • 패킷 헤더 파싱                                           │ │
│  │     • IP, TCP 처리                                            │ │
│  │     • 소켓 버퍼에 데이터 넣기                                   │ │
│  │     • 프로세스 깨우기                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### ISR 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ISR 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 처리 통계 (Linux)】                                        │
│  ──────────────────                                                  │
│  # 인터럽트 발생 횟수                                                 │
│  $ cat /proc/interrupts                                             │
│                                                                     │
│  # 소프트IRQ 통계                                                    │
│  $ cat /proc/softirqs                                               │
│                                                                     │
│  # 인터럽트 처리 시간                                                 │
│  $ perf stat -e irq:*                                               │
│                                                                     │
│  【ISR 디버깅】                                                       │
│  ──────────────────                                                  │
│  1. ftrace: 인터럽트 핸들러 추적                                      │
│  2. eBPF: 인터럽트 처리 분석                                          │
│  3. /proc/interrupts: 통계 확인                                      │
│                                                                     │
│  【ISR 최적화 팁】                                                    │
│  ──────────────────                                                  │
│  1. 하드웨어 ACK만 Top Half에서                                       │
│  2. 데이터 처리는 Bottom Half로                                       │
│  3. 인터럽트 코일레싱 (여러 이벤트 묶기)                                │
│  4. NAPI (고속 네트워크용 ISR+폴링)                                   │
│  5. Threaded IRQ (높은 응답성 필요 시)                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• ISR: 인터럽트 발생 시 실행되는 처리 함수
• 제약: sleep 불가, 최소 실행 시간
• Top Half: ISR, 하드웨어 처리만
• Bottom Half: SoftIRQ, Tasklet, Workqueue
• 원칙: 빠른 ACK, 긴 작업은 위임
• 최적화: 코일레싱, NAPI, Threaded IRQ
```
