# 043. 클라이언트-서버 (Client-Server)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서비스 요청자(클라이언트)와 제공자(서버)의 분리 구조
> 2. **가치**: 분산 처리, 확장성, 중앙 관리, 자원 공유
> 3. **융합**: RPC, API, N-tier, 마이크로서비스와 연관

---

## Ⅰ. 개요

### 개념 정의

클라이언트-서버(Client-Server)는 **서비스를 요청하는 클라이언트와 서비스를 제공하는 서버로 역할을 분리한 분산 컴퓨팅 아키텍처**다. 클라이언트가 요청(Request)을 보내면 서버가 처리 후 응답(Response)을 반환한다.

### 💡 비유: 식당의 주문 시스템
클라이언트-서버는 **식당의 주문 시스템**과 같다. 손님(클라이언트)이 메뉴를 주문(요청)하면 주방(서버)에서 요리를 만들어 제공(응답)한다. 여러 손님이 주문해도 주방에서는 순서대로 처리한다.

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 클라이언트-서버: 요청자와 제공자 분리 구조
• 2-Tier: 클라이언트 - 서버
• 3-Tier: 프레젠테이션 - 애플리케이션 - 데이터
• N-Tier: 마이크로서비스 아키텍처
• 프로토콜: HTTP, WebSocket, gRPC, REST
• 장점: 확장성, 분산 처리, 중앙 관리
• 단점: 단일 장애점, 네트워크 의존
```
