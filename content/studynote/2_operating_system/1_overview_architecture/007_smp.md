# 007. 대칭 다중 처리 (SMP, Symmetric Multiprocessing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 CPU가 동등한 권한으로 OS 실행 및 작업 처리
> 2. **가치**: 높은 확장성, 장애 허용력, 균등한 부하 분산
> 3. **융합**: 캐시 일관성, 동기화, NUMA와 밀접 연관

---

## Ⅰ. 개요

### 개념 정의

대칭 다중 처리(SMP)는 **여러 CPU가 동등한 권한을 가지고 단일 공유 메모리를 접근하며, 모든 CPU가 OS를 실행하고 모든 작업을 처리할 수 있는 다중 처리 구조**다.

### 💡 비유: 민주적 식당 주방
SMP는 **민주적인 식당 주방**과 같다. 모든 요리사(CPU)가 동등한 권한을 가진다. 누구나 주문을 받고, 메뉴를 결정하고, 요리를 만들 수 있다. 한 요리사가 아파도 다른 요리사가 대신할 수 있다.

## Ⅱ. SMP 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                SMP 상세 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SMP 장점】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 높은 확장성                                                 │ │
│  │     • CPU 추가 시 선형적 성능 향상 (이상적)                      │ │
│  │     • 비용 대비 효율적                                           │ │
│  │                                                               │ │
│  │  2. 장애 허용력                                                  │ │
│  │     • CPU 장애 시 다른 CPU가 대체                               │ │
│  │     • SPOF (단일 실패점) 없음                                   │ │
│  │                                                               │ │
│  │  3. 균등한 부하 분산                                             │ │
│  │     • 모든 CPU가 작업 처리                                       │ │
│  │     • 유휴 자원 최소화                                           │ │
│  │                                                               │ │
│  │  4. 유연성                                                       │ │
│  │     • 프로세스가 어떤 CPU에서든 실행                              │ │
│  │     • 동적 부하 분산                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SMP 도전 과제】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 캐시 일관성 (Cache Coherence)                               │ │
│  │     • 각 CPU 캐시 간 데이터 동기화                               │ │
│  │     • MESI 프로토콜 등 필요                                      │ │
│  │                                                               │ │
│  │  2. 동기화 오버헤드                                              │ │
│  │     • 공유 자원 접근 시 락 필요                                   │ │
│  │     • 락 경합으로 성능 저하                                       │ │
│  │                                                               │ │
│  │  3. 메모리 대역폭 병목                                           │ │
│  │     • 모든 CPU가 동일 메모리 버스 사용                            │ │
│  │     • CPU 증가 시 병목 심화                                      │ │
│  │                                                               │ │
│  │  4. 복잡한 OS 설계                                               │ │
│  │     • 멀티프로세서 안전한 커널                                    │ │
│  │     • 정교한 동기화 메커니즘                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 일관성 프로토콜 (MESI)】                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  상태            의미                    허용 연산              │ │
│  │  ────            ────                    ────                   │ │
│  │  Modified (M)    이 캐시만 수정           읽기, 쓰기            │ │
│  │  Exclusive (E)   이 캐시만 소유           읽기, 쓰기            │ │
│  │  Shared (S)      여러 캐시가 공유         읽기만                │ │
│  │  Invalid (I)     유효하지 않음            없음                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. SMP × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                SMP × 타 과목 융합                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 버스: 공유 버스 아키텍처                                           │
│  • 캐시: 일관성 프로토콜 (MESI, MOESI)                                │
│  • 메모리: UMA (Uniform Memory Access)                              │
│  • 인터커넥트: 크로스바, 멀티포트                                     │
│                                                                     │
│  【동시성】                                                          │
│  • 락: 스핀락, 뮤텍스                                                │
│  • 원자 연산: CAS, Test-and-Set                                     │
│  • 메모리 배리어: 일관성 보장                                         │
│                                                                     │
│  【데이터베이스】                                                      │
│  • 병렬 쿼리: 다중 CPU 활용                                           │
│  • 래치: 메모리 내 동기화                                             │
│  • 버퍼 풀: 캐시 일관성 고려                                          │
│                                                                     │
│  【네트워크】                                                        │
│  • 멀티코어 서버: SMP 기반 웹 서버                                    │
│  • 부하 분산: CPU 간 작업 분배                                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SMP 확장 한계 (Amdahl's Law)                                 │ │
│  │  ─────────────────────────────                                │ │
│  │                                                               │ │
│  │  성능 향상 = 1 / ((1 - P) + P/N)                              │ │
│  │                                                               │ │
│  │  P: 병렬화 가능 비율                                           │ │
│  │  N: 프로세서 수                                                │ │
│  │                                                               │ │
│  │  ┌────────────────────────────────────────────┐              │ │
│  │  │                                            │              │ │
│  │  │  Speedup                                   │              │ │
│  │  │     ^                                      │              │ │
│  │  │  8 ──┬────────────────────                 │              │ │
│  │  │     │ P=95%                               │              │ │
│  │  │  6 ──┼───────────────                      │              │ │
│  │  │     │ P=90%                               │              │ │
│  │  │  4 ──┼───────────                          │              │ │
│  │  │     │ P=80%                               │              │ │
│  │  │  2 ──┼───────                              │              │ │
│  │  │     └────┬────┬────┬────┬────> N          │              │ │
│  │  │          2    4    8   16   32            │              │ │
│  │  │                                            │              │ │
│  │  └────────────────────────────────────────────┘              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```

│                SMP 분석                                              │

│                                                                     │

│  ──────────────────                                                  │
│  # CPU 정보                                                          │
│  $ lscpu                                                            │
│                                                                     │
│  # CPU 수                                                            │
│  $ nproc                                                            │
│                                                                     │
│  # CPU 온라인 상태                                                   │
│  $ cat /sys/devices/system/cpu/online                              │
│                                                                     │
│  # 부하 분산 확인                                                    │
│  $ mpstat -P ALL 1                                                  │
│                                                                     │
│  【SMP 최적화 팁】                                                    │
│  ──────────────────                                                  │
│  1. CPU 친화성 설정                                                   │
│     taskset -c 0,1 ./my_program                                    │
│                                                                     │
│  2. IRQ 밸런싱                                                       │
│     /proc/irq/*/smp_affinity                                       │
│                                                                     │
│  3. 스케줄러 튜닝                                                    │
│     /proc/sys/kernel/sched_*                                       │
│                                                                     │
│  4. cgroups CPU 제한                                                 │
│     cpu.cfs_quota_us, cpu.cfs_period_us                           │
│                                                                     │
│  【SMP 성능 지표】                                                    │
│  ──────────────────                                                  │
│  • CPU 이용률: 균등 분배 확인                                        │
│  • 캐시 미스율: 일관성 오버헤드                                       │
│  • 락 경합: 동기화 병목                                               │
│  • 컨텍스트 스위치: 오버헤드                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SMP: 모든 CPU가 동등한 권한으로 OS 실행
• 장점: 확장성, 장애 허용력, 균등 부하 분산
• 도전: 캐시 일관성, 동기화, 메모리 병목
• 프로토콜: MESI (Modified, Exclusive, Shared, Invalid)
• 한계: Amdahl's Law에 의한 확장 한계
• 적용: 현대 서버, 워크스테이션 표준
• 진화: NUMA로 확장 (대규모 시스템)
```
