# 012. 듀얼 모드 (Dual Mode)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 모드와 커널 모드를 분리하여 시스템 보호
> 2. **가치**: 사용자 프로그램의 직접 하드웨어 접근 차단
> 3. **융합**: 모드 비트, 시스템 콜, 인터럽트, Ring Protection

---

## Ⅰ. 개요

### 개념 정의

듀얼 모드(Dual Mode)는 **CPU가 사용자 모드(User Mode)와 커널 모드(Kernel Mode) 두 가지 권한 수준으로 동작하여, 사용자 프로그램이 시스템 자원에 직접 접근하는 것을 방지하는 보호 메커니즘**이다.

### 💡 비유: 일반인과 관계자
듀얼 모드는 **박물관의 일반인과 관계자** 구분과 같다. 일반인(사용자 모드)은 전시물을 볼 수만 있고, 관계자(커널 모드)만 유물을 옮기거나 보관실에 들어갈 수 있다. 전시물이 파손되는 것을 방지한다.

### 듀얼 모드 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                듀얼 모드 구조                                         │

│                                                                     │
│  【모드 분리】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │                   사용자 모드 (User Mode)                │ │ │
│  │  │                      Ring 3                             │ │ │
│  │  │  ┌─────────┐  ┌─────────┐  ┌─────────┐                 │ │ │
│  │  │  │ Browser │  │   App   │  │  Game   │                 │ │ │
│  │  │  └─────────┘  └─────────┘  └─────────┘                 │ │ │
│  │  │                                                         │ │ │
│  │  │  • 제한된 명령어만 실행                                  │ │ │
│  │  │  • 하드웨어 직접 접근 불가                               │ │ │
│  │  │  • I/O 요청 시 시스템 콜 필요                            │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                            │                                  │ │
│  │                     시스템 콜 (Trap)                          │ │

│  │  │                   커널 모드 (Kernel Mode)                │ │ │
│  │  │                      Ring 0                             │ │ │
│  │  │  ┌─────────────────────────────────────────────────┐   │ │ │
│  │  │  │                    Kernel                        │   │ │ │
│  │  │  │  Process | Memory | File | Device | Network     │   │ │ │
│  │  │  └─────────────────────────────────────────────────┘   │ │ │
│  │  │                                                         │ │ │
│  │  │  • 모든 명령어 실행 가능                                │ │ │
│  │  │  • 하드웨어 직접 접근 가능                              │ │ │
│  │  │  • 메모리 전체 접근 가능                                │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │

│  【모드 전환】                                                        │
```

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                듀얼 모드 상세                                         │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  금지 명령:                                                    │ │
│  │  • I/O 명령 (IN, OUT)                                         │ │
│  │  • 메모리 관리 레지스터 변경                                    │ │
│  │  • 인터럽트 제어                                               │ │
│  │  • halt 명령                                                  │ │

│  │                                                               │ │

│                                                                     │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  가능 작업:                                                    │ │
│  │  • 모든 하드웨어 접근                                          │ │
│  │  • 메모리 전체 접근                                            │ │
│  │  • 인터럽트 활성화/비활성화                                     │ │
│  │  • 페이지 테이블 수정                                          │ │
│  │  • I/O 포트 접근                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【x86 Ring Protection】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Ring 0: 커널 모드 (완전 권한)                                  │ │
│  │  Ring 1: 장치 드라이버 (거의 사용 안함)                         │ │
│  │  Ring 2: 장치 드라이버 (거의 사용 안함)                         │ │

│  │  ┌──────────────────────┐                                    │ │
│  │  │      Ring 0          │  커널                               │ │

│  │  │      Ring 1          │  (사용 안함)                        │ │
│  │  ├──────────────────────┤                                    │ │
│  │  │      Ring 2          │  (사용 안함)                        │ │
│  │  ├──────────────────────┤                                    │ │
│  │  │      Ring 3          │  애플리케이션                        │ │

│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 듀얼 모드 × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                듀얼 모드 × 타 과목 융합                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 모드 비트: PSW 레지스터 내 위치                                    │
│  • CPL (Current Privilege Level): 현재 권한 수준                     │
│  • DPL (Descriptor Privilege Level): 세그먼트/게이트 권한            │
│  • 보호 링: Ring 0~3                                                │
│                                                                     │
│  【가상화】                                                          │
│  • Ring -1: VMX Root Mode (하이퍼바이저)                             │
│  • Ring 0: VMX Non-Root Mode (게스트 OS)                            │
│  • VM Exit/Entry: 모드 전환                                         │
│                                                                     │
│  【보안】                                                            │
│  • 권한 상승 방지: 사용자→커널 전환 검증                              │
│  • 시스템 콜 필터링: seccomp                                        │
│  • 커널 보호: KPTI (Kernel Page Table Isolation)                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시스템 콜 과정                                               │ │
│  │  ────────────────                                            │ │
│  │                                                               │ │
│  │  1. 사용자 프로그램: 시스템 콜 호출                             │ │
│  │     (라이브러리 함수 → syscall 명령)                           │ │
│  │                                                               │ │
│  │  2. CPU: User Mode → Kernel Mode 전환                         │ │
│  │     (모드 비트 1 → 0)                                          │ │
│  │                                                               │ │
│  │  3. 커널: 시스템 콜 핸들러 실행                                 │ │
│  │     (번호 확인 → 적절한 함수 호출)                              │ │
│  │                                                               │ │
│  │  4. 커널: 작업 완료 후 결과 반환                                 │ │
│  │                                                               │ │
│  │  5. CPU: Kernel Mode → User Mode 복귀                         │ │
│  │     (모드 비트 0 → 1)                                          │ │
│  │                                                               │ │
│  │  6. 사용자 프로그램: 계속 실행                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 듀얼 모드 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                듀얼 모드 분석                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【모드 확인 (Linux)】                                               │
│  ──────────────────                                                  │
│  # 커널 모드 vs 사용자 모드 시간                                      │
│  $ cat /proc/stat | grep cpu                                        │
│  # cpu user nice system idle ...                                   │
│  # user: 사용자 모드, system: 커널 모드                               │
│                                                                     │
│  # 시스템 콜 추적                                                    │
│  $ strace -c ls                                                    │
│                                                                     │
│  # 인터럽트 확인                                                     │
│  $ cat /proc/interrupts                                             │
│                                                                     │
│  【성능 영향】                                                        │
│  ──────────────────                                                  │
│  • 시스템 콜 오버헤드: 수백~수천 사이클                               │
│  • 컨텍스트 스위치: 모드 전환 포함                                    │
│  • 최소화: 버퍼링, 배칭, 캐싱                                         │
│                                                                     │
│  【보안 고려사항】                                                    │
│  ──────────────────                                                  │
│  1. 시스템 콜 필터링 (seccomp)                                       │
│  2. 최소 권한 원칙                                                   │
│  3. 커널 진입점 제한                                                 │
│  4. 권한 상승 공격 방지                                               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 듀얼 모드: 사용자 모드와 커널 모드 분리
• 사용자 모드: 제한된 권한, 하드웨어 접근 불가
• 커널 모드: 완전한 권한, 모든 자원 접근 가능
• 전환: 시스템 콜, 인터럽트로만 가능
• x86: Ring 0~3 계층적 보호
• 가상화: Ring -1 (VMX Root) 추가
```
