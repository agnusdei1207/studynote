# 028. UEFI vs BIOS

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컴퓨터 부팅 시 하드웨어 초기화를 담당하는 펌웨어 인터페이스
> 2. **가치**: 하드웨어 추상화, 플랫폼 독립성, 시스템 초기화
> 3. **융합**: MBR/GPT, Secure Boot, 펌웨어와 연관

---

## Ⅰ. 개요

### 개념 정의

UEFI(Unified Extensible Firmware Interface)와 BIOS(Basic Input/Output System)은 **컴퓨터 부팅 시 하드웨어를 초기화하고 운영체제를 로드하는 펌웨어 인터페이스**다. BIOS는 구형 16-bit 펌웨어이고, UEFI는 현대적 32/64-bit 펌웨어다.

### 💡 비유: 건물 관리 사무실
UEFI/BIOS는 **건물 관리 사무실**과 같다. 입주자(OS)가 건물(컴퓨터)에 들어오기 전에 사무실에서 건물 상태를 점검하고 입주자에게 공간을 배정한다. BIOS는 구형 사무실, UEFI는 최신 사무실이다.

```
│                UEFI vs BIOS 구조                                    │
├─────────────────────────────────────────────────────────────────────┤

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  구분              BIOS                 UEFI                         │ │

│  │  개발 시기          1980년대            2000년대 중반~                 │ │
│  │  비트              16-bit               32/64-bit                     │ │
│  │  주소 공간         1MB 제한             4GB+                           │ │
│  │  파티션             MBR                 GPT                           │ │
│  │  최대 디스크        2.2TB               9.4ZB (ZB)                      │ │
│  │  부팅 속도         느림                   빠름                           │ │
│  │  GUI               텍스트 기반            그래픽 지원                     │ │
│  │  네트워크 부팅     제한적               지원                           │ │
│  │  보안              Secure Boot 미지원    Secure Boot 지원              │ │

│                                                                     │
│  【BIOS 부팅 과정】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 전원 on → BIOS 시작                                 │ │
│  │  2. POST (Power-On Self-Test) 실행                           │ │
│  │  3. MBR 읽기 → 부트로더 로딩                            │ │
│  │  4. 부트로더 → 커널 로딩                               │ │
│  │  5. 커널 실행                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【UEFI 부팅 과정】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 전원 on → UEFI 펌웨어 실행                          │ │
│  │  2. 하드웨어 초기화 (POST)                                │ │
│  │  3. ESP (EFI System Partition) 읽기                        │ │

│  │  5. 부트로더 → 커널 로딩                               │ │
│  │  6. 커널 실행                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
```

## Ⅱ. UEFI 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                UEFI 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【UEFI 주요 기능】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  기능              설명                                        │ │
│  │  ────              ────                                        │ │
│  │  GPT 지원         2TB 이상 디스크, 128개 파티션               │ │
│  │  Secure Boot      부트킷/rootkit 방지                        │ │
│  │  네트워크 부팅     PXE, HTTP 부팅 지원                        │ │
│  │  드라이버 모델     64-bit 드라이버, 플러그인                  │ │
│  │  쉘                UEFI Shell (명령어 인터페이스)             │ │
│  │  변수 저장        NVRAM에 부트 변수 저장                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【UEFI 파티션 구조】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ESP (EFI System Partition)                                   │ │
│  │  ─────────────────────                                         │ │
│  │  • FAT32 파일시스템                                          │ │
│  │  • 부트로더 (.efi 파일) 저장                                  │ │
│  │  • 크기: 100MB~500MB                                         │ │
│  │  • 위치: 디스크 시작 부분 권장                                │ │
│  │                                                               │ │
│  │  GPT 파티션 테이블                                            │ │
│  │  ─────────────────────                                         │ │
│  │  • 보호 MBR (Protective MBR)                                 │ │
│  │  • GPT 헤더                                                   │ │
│  │  • 파티션 엔트리 배열                                         │ │
│  │  • 파티션들                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Secure Boot】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  목적: 부트킷, 루트킷, 변조된 부트로더 방지                   │ │
│  │                                                               │ │
│  │  메커니즘:                                                     │ │
│  │  1. 서명된 부트로더만 실행                                    │ │
│  │  2. 키 교환 방식 (KEK, db, dbx)                               │ │

│  │                                                               │ │
│  │  문제점:                                                      │ │
│  │  • 리눅스 설치 어려움 (shim 필요)                             │ │
│  │  • 듀얼 부팅 복잡성                                           │ │
│  │  • 키 관리 부담                                               │ │
```

## Ⅳ. 실무 적용

### UEFI 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                UEFI 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【UEFI 정보 확인】                                                │
│  ──────────────────                                                  │
│  # UEFI 모드 확인                                                   │
│  $ [ -d /sys/firmware/efi ] && echo "UEFI mode"                │
│                                                                     │
│  # Secure Boot 상태                                                 │
│  $ mokutil --sb-state                                          │
│  $ mokutil --validation                                        │
│                                                                     │
│  # UEFI 변수                                                        │
│  $ efivar -l                                                   │
│  $ efivar -p                                                   │
│                                                                     │
│  # 부트 순서                                                         │
│  $ efibootmgr                                                 │
│                                                                     │
│  【UEFI 부트 항목 관리】                                             │
│  ──────────────────                                                  │
│  # 부트 항목 목록                                                   │
│  $ efibootmgr -v                                               │
│                                                                     │
│  # 새 부트 항목 추가                                                 │
│  $ efibootmgr -c -L "Ubuntu" -l \EFI\ubuntu\grubx64.efi       │
│                                                                     │
│  # 부트 순서 변경                                                   │
│  $ efibootmgr -o 0001,0002,0003                                │
│                                                                     │
│  【Secure Boot 관리】                                               │
│  ──────────────────                                                  │
│  # Secure Boot 활성화/비활성화                                      │
│  $ mokutil --enable-validation                                │
│  $ mokutil --disable-validation                               │
│                                                                     │
│  # 키 등록 (관리자 권한)                                             │
│  $ mokutil --import                                              │
│  $ mokutil --export                                              │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• UEFI: 현대적 32/64-bit 펌웨어 인터페이스
• BIOS: 구형 16-bit 펌웨어
• UEFI 장점: GPT, Secure Boot, 네트워크 부팅, 빠른 부팅
• BIOS 장점: 호환성, 단순성
• UEFI 구조: ESP, GPT, .efi 파일
• Secure Boot: 부트킷 방지, 서명 검증
• 전환: BIOS → UEFI (2010년대 후반부터)
```
