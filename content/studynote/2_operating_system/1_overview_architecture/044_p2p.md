# 044. P2P (Peer-to-Peer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 중앙 서버 없이 노드들이 직접 통신하는 분산 구조
> 2. **가치**: 확장성, 탈중앙화, 단일 장애점 없음
> 3. **융합**: BitTorrent, 블록체인, DHT와 연관

---

## Ⅰ. 개요

### 개념 정의

P2P(Peer-to-Peer)는 **중앙 서버 없이 모든 노드가 동등한(peer) 지위에서 서로 직접 통신하는 분산 네트워크 구조**다. 각 노드는 클라이언트이면서 동시에 서버 역할을 수행한다.

### 💡 비유: 바자회
P2P는 **동네 바자회**와 같다. 상점(서버)이 따로 없고, 이웃들(peer)이 서로 물건을 주고받는다. 누구나 물건을 팔 수도, 살 수도 있다. 한 사람이 빠져도 거래는 계속된다.

## Ⅱ. P2P 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                P2P 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【P2P 장단점】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점                                                         │ │
│  │  ────                                                         │ │
│  │  • 확장성: 노드 추가 → 용량 증가                               │ │
│  │  • 탈중앙화: 단일 통제점 없음                                  │ │
│  │  • 내결함성: 일부 노드 장애 영향 제한                           │ │
│  │  • 비용: 인프라 비용 분산                                      │ │
│  │                                                               │ │
│  │  단점                                                         │ │
│  │  ────                                                         │ │
│  │  • 검색 효율: 전체 검색 어려움                                 │ │
│  │  • 보안: 악의적 노드 위험                                      │ │
│  │  • 신뢰성: 데이터 무결성 보장 어려움                           │ │
│  │  • 법적 문제: 저작권 침해 가능성                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 P2P 애플리케이션】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  분야              애플리케이션              특징                │ │
│  │  ────              ────                      ────              │ │
│  │  파일 공유         BitTorrent               분할 다운로드       │ │
│  │                   eMule/eDonkey             DHT                │ │
│  │                                                               │ │
│  │  통신             Skype (과거)              슈퍼노드           │ │
│  │                   WebRTC                    브라우저 P2P       │ │
│  │                                                               │ │
│  │  블록체인         Bitcoin, Ethereum         합의 알고리즘      │ │
│  │                   IPFS                      분산 파일시스템    │ │
│  │                                                               │ │
│  │  스트리밍          P2P 스트리밍              WebTorrent        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DHT (Distributed Hash Table)】                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 분산 키-값 저장소                                           │ │
│  │  • O(log N) 검색 복잡도                                        │ │
│  │  • 구현: Kademlia, Chord, Pastry                              │ │
│  │  • 사용: BitTorrent, IPFS                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. P2P × 타 과목 융합

│                                                                     │

│  • 청킹: 파일 분할                                                  │

│  2. 트래커                                                           │
│     • 피어 목록 제공                                                 │

│                                                                     │
│  3. 피어 통신                                                        │
│     • 청크 교환                                                      │
│     • 희귀 조각 우선                                                 │
│     • choked/unchoked 알고리즘                                      │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │

## Ⅳ. 실무 적용

### P2P 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                P2P 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【BitTorrent 사용】                                                 │
│  ──────────────────                                                  │
│  # 토렌트 클라이언트                                                 │
│  $ transmission-cli file.torrent                                  │
│  $ rtorrent                                                       │
│  $ aria2c file.torrent                                            │
│                                                                     │
│  # 토렌트 파일 생성                                                   │
│  $ transmission-create -o output.torrent -t http://tracker/ file  │
│                                                                     │
│  【IPFS 사용】                                                      │
│  ──────────────────                                                  │
│  # IPFS 시작                                                        │
│  $ ipfs init                                                       │
│  $ ipfs daemon                                                     │
│                                                                     │
│  # 파일 추가                                                        │
│  $ ipfs add file.txt                                               │
│                                                                     │
│  # 파일 조회                                                        │
│  $ ipfs cat <hash>                                                 │
│                                                                     │
│  【WebRTC】                                                          │
│  ──────────────────                                                  │
│  # 브라우저 P2P 통신                                                 │
│  • RTCPeerConnection                                               │
│  • RTCDataChannel                                                  │
│  • Signaling Server 필요                                           │
│                                                                     │
│  【네트워크 상태】                                                   │
│  ──────────────────                                                  │
│  # 연결 확인                                                        │
│  $ netstat -an | grep ESTABLISHED                                 │
│  $ ss -s                                                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• P2P: 노드 간 직접 통신, 중앙 서버 없음
• 장점: 확장성, 탈중앙화, 내결함성
• 단점: 검색 효율, 보안, 신뢰성
• 유형: Pure, Hybrid, Super-node
• DHT: 분산 키-값 저장소
• 적용: BitTorrent, IPFS, 블록체인
```
