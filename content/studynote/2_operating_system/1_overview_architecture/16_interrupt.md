# 507. 인터럽트 (Interrupt)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 외부/내부 이벤트에 의한 CPU 제어권 전환
> 2. **가치**: 비동기 이벤트 처리와 CPU 효율성
> 3. **융합**: 폴링, 예외, 시그널과 연관

---

## Ⅰ. 개요

### 개념 정의

인터럽트(Interrupt)는 **CPU가 현재 작업을 중단하고 이벤트 발생을 알리는 신호를 처리하는 메커니즘**이다.

### 💡 비유: 비상벨
인터럽트는 **비상벨**과 같다. 일하다가 벨이 울리면 즉시 멈추고 비상 상황을 처리한다.

│  │           │                                                            │ │

│  │  ┌─────────────────────────────────────────────────────────┐            │ │

│  │  └─────────────────────────────────────────────────────────┘            │ │
│  │           │                                                            │ │

│  │  ┌─────────────────────────────────────────────────────────┐            │ │

│  │  └─────────────────────────────────────────────────────────┘            │ │
│  │           │                                                            │ │
│  │           ▼                                                            │ │

│  │           │                                                            │ │

│  │           │                                                            │ │
│  │           ▼                                                            │ │

│  │  소프트웨어 인터럽트 INT 명령어             시스템 콜, 브레이크포인트        │ │

│  │  14          #PF               Page Fault                              │ │
│  │  18          #MC               Machine Check                           │ │
│  │  32-255      IRQ               Hardware Interrupts                     │ │

│  │                                                             │ │

│  │  • CLI: 인터럽트 비활성화                                                │ │
│  │  • STI: 인터럽트 활성화                                                  │ │
│  │  • 대부분의 하드웨어 인터럽트                                            │ │
│  │                                                             │ │
│  │  논마스커블 인터럽트 (NMI):                                               │ │
│  │  • 소프트웨어로 차단 불가능                                              │ │
│  │  • 치명적 하드웨어 오류                                                  │ │

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |

│                                                                     │
│  【인터럽트 정보 확인】                                                  │
│  ──────────────────                                                  │
│  // 인터럽트 통계                                                        │
│  $ cat /proc/interrupts                                              │
│           CPU0       CPU1       CPU2       CPU3                      │
│  0:         34          0          0          0  IR-IO-APIC   2-edge  timer│
│  1:          0          0          0          0  IR-IO-APIC   1-edge  i8042│
│  NMI:        0          0          0          0  Non-maskable interrupts  │
│  LOC:  12345678  12345678  12345678  12345678  Local timer interrupts     │
│                                                                     │
│  // 실시간 인터럽트 변화                                                  │
│  $ watch -n 1 "cat /proc/interrupts | head -20"                      │
│                                                                     │
│  // 인터럽트 당 CPU 사용                                                  │
│  $ cat /proc/softirqs                                                │
│                                                                     │
│  【인터럽트 선호도 설정】                                                │
│  ──────────────────                                                  │
│  // 특정 CPU로 인터럽트 바인딩                                           │
│  $ cat /proc/irq/24/smp_affinity                                    │
│  f                                                                   │
│  $ echo 1 > /proc/irq/24/smp_affinity   // CPU 0만 처리               │
│  $ echo 2 > /proc/irq/24/smp_affinity   // CPU 1만 처리               │
│  $ echo f > /proc/irq/24/smp_affinity   // 모든 CPU 처리              │
│                                                                     │
│  // 선호도 리스트 확인                                                   │
│  $ cat /proc/irq/24/smp_affinity_list                               │
│  0-3                                                                 │
│                                                                     │
│  【인터럽트 밸런싱】                                                      │
│  ──────────────────                                                  │
│  // irqbalance 서비스                                                   │
│  $ systemctl status irqbalance                                      │
│  $ systemctl start irqbalance                                       │
│  $ systemctl stop irqbalance                                        │
│                                                                     │
│  // 수동 밸런싱                                                         │
│  $ irqbalance --debug                                               │
│                                                                     │
│  【인터럽트 핸들러 등록 (커널)】                                          │
│  ──────────────────                                                  │
│  #include <linux/interrupt.h>                                        │
│                                                                     │
│  // 인터럽트 핸들러 함수                                                  │
│  irqreturn_t my_handler(int irq, void *dev_id) {                     │
│      // 인터럽트 처리                                                    │
│      return IRQ_HANDLED;                                             │
│  }                                                                   │
│                                                                     │
│  // 인터럽트 요청                                                        │
│  int ret = request_irq(irq, my_handler,                              │
│                        IRQF_SHARED, "my_device", dev_id);            │
│  if (ret) {                                                          │
│      printk("Failed to request IRQ\n");                              │
│  }                                                                   │
│                                                                     │
│  // 해제                                                               │
│  free_irq(irq, dev_id);                                              │
│                                                                     │
│  【인터럽트 금지 구역】                                                  │
│  ──────────────────                                                  │
│  // 로컬 CPU 인터럽트 비활성화                                           │
│  unsigned long flags;                                                │
│  local_irq_save(flags);                                              │
│  // 임계 영역                                                          │
│  local_irq_restore(flags);                                           │
│                                                                     │
│  // 스핀락과 함께                                                        │
│  spin_lock_irq(&lock);                                               │
│  // 임계 영역                                                          │
│  spin_unlock_irq(&lock);                                             │
│                                                                     │
│  【지연된 작업 (Bottom Half)】                                          │
│  ──────────────────                                                  │
│  // Tasklet                                                           │
│  DECLARE_TASKLET(my_tasklet, tasklet_func, data);                    │
│  tasklet_schedule(&my_tasklet);                                      │
│                                                                     │
│  // Workqueue                                                         │
│  INIT_WORK(&my_work, work_func);                                     │
│  schedule_work(&my_work);                                            │
│                                                                     │
│  // SoftIRQ                                                           │
│  open_softirq(MY_SOFTIRQ, softirq_handler);                          │
│  raise_softirq(MY_SOFTIRQ);                                          │
│                                                                     │
│  【인터럽트 지연 측정】                                                  │
│  ──────────────────                                                  │
│  // /proc/interrupts 타임스탬프                                         │
│  $ cat /proc/interrupts; sleep 1; cat /proc/interrupts              │
│                                                                     │
│  // ftrace로 인터럽트 추적                                               │
│  $ trace-cmd record -e irq:irq_handler_entry -e irq:irq_handler_exit│
│  $ trace-cmd report                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 이벤트에 의한 CPU 제어권 전환
• 처리: 상태저장 → 핸들러실행 → 상태복원
• 종류: 하드웨어, 소프트웨어, 예외, IPI
• 우선순위: Reset > MCE > NMI > IRQ > Exception
• 마스킹: IF 플래그로 제어 (NMI 제외)
• 확인: /proc/interrupts, /proc/softirqs
• 설정: /proc/irq/N/smp_affinity
• 최적화: IRQ 밸런싱, Bottom Half 분리
```
