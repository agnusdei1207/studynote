# 016. 인터럽트 (Interrupt)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU에 비동기적 이벤트 알림, 실행 흐름 전환
> 2. **가치**: 폴링 대비 효율적 이벤트 처리, 실시간 응답
> 3. **융합**: ISR, 인터럽트 벡터, 우선순위, 마스킹과 연관

---

## Ⅰ. 개요

### 개념 정의

**인터럽트(Interrupt)**는 하드웨어나 소프트웨어 이벤트가 발생했을 때 CPU에게 알려 현재 실행 중인 작업을 중단하고 해당 이벤트를 처리하도록 하는 메커니즘이다.

### 💡 비유: 수업 중 손 들기
인터럽트는 **수업 중에 학생이 손을 드는 것**과 같다. 선생님(CPU)이 강의(작업)를 하던 중, 학생(장치)이 손을 들면(인터럽트), 선생님은 강의를 잠시 멈추고 학생의 질문(이벤트)을 처리한 후 다시 강의를 계속한다.

### 인터럽트 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 처리 과정】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 장치에서 인터럽트 신호 발생                                  │ │
│  │     ┌─────────┐                                               │ │
│  │     │ Device  │───────▶ IRQ 신호                              │ │
│  │     └─────────┘                                               │ │
│  │                                                               │ │
│  │  2. CPU가 현재 명령 완료 후 인터럽트 확인                        │ │
│  │     ┌───────────────────────────────────────────────────┐    │ │
│  │     │ Current Process: [Inst1] [Inst2] [Inst3] ...      │    │ │
│  │     │                                          ↑         │    │ │
│  │     │                                    인터럽트 확인     │    │ │
│  │     └───────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  3. 컨텍스트 저장 (PC, Flags, Registers)                       │ │
│  │                                                               │ │
│  │  4. 인터럽트 벡터로 ISR 주소 찾기                               │ │
│  │     ┌────────────────────────────────────────────────────┐   │ │
│  │     │  Interrupt Vector Table (IVT)                      │   │ │
│  │     │  ┌──────┬───────────────────────┐                 │   │ │
│  │     │  │ IRQ  │  ISR Address           │                 │   │ │
│  │     │  ├──────┼───────────────────────┤                 │   │ │
│  │     │  │  0   │  timer_handler         │                 │   │ │
│  │     │  │  1   │  keyboard_handler      │                 │   │ │
│  │     │  │  ... │  ...                   │                 │   │ │
│  │     │  └──────┴───────────────────────┘                 │   │ │
│  │     └────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  │  5. ISR (Interrupt Service Routine) 실행                      │ │
│  │                                                               │ │
│  │  6. 컨텍스트 복원 후 원래 작업 재개                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 종류】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  종류              원인                    특징                  │ │
│  │  ────              ────                    ────                  │ │
│  │  하드웨어 인터럽트   장치 (키보드, 디스크)     비동기적            │ │
│  │  소프트웨어 인터럽트 시스템 콜, 예외         동기적/비동기적      │ │
│  │  트랩 (Trap)       의도적 명령 (int)         동기적              │ │
│  │  예외 (Exception)  오류 (0 나누기, 페이지 폴트) 동기적           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 인터럽트 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 처리 시간】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  구간              소요 시간           설명                     │ │
│  │  ────              ──────────           ────                     │ │
│  │  하드웨어 지연      ~100ns             신호 전파                 │ │
│  │  컨텍스트 저장      ~1μs               레지스터 저장             │ │
│  │  ISR 실행          ~10μs~1ms          실제 처리                 │ │
│  │  컨텍스트 복원      ~1μs               레지스터 복구             │ │
│  │                                                               │ │
│  │  총 인터럽트 지연 = 하드웨어 + 소프트웨어 오버헤드               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 우선순위】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  우선순위          인터럽트 유형                                │ │
│  │  ────────          ────────────                                │ │
│  │  최고               NMI (Non-Maskable Interrupt)               │ │
│  │  높음               하드웨어 오류, 전원                         │ │
│  │  보통               타이머, 디스크, 네트워크                     │ │
│  │  낮음               키보드, 마우스                              │ │
│  │                                                               │ │
│  │  우선순위 인터럽트 컨트롤러 (PIC/APIC)가 관리                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 마스킹】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  마스크 가능 인터럽트 (Maskable):                               │ │
│  │  • IF (Interrupt Flag)로 제어                                  │ │
│  │  • cli: 인터럽트 비활성화                                       │ │
│  │  • sti: 인터럽트 활성화                                         │ │
│  │                                                               │ │
│  │  마스크 불가 인터럽트 (Non-Maskable, NMI):                       │ │
│  │  • 하드웨어 오류, 정전                                          │ │
│  │  • 마스킹 불가, 즉시 처리                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 인터럽트 × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 × 타 과목 융합                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • PIC/APIC: 인터럽트 컨트롤러                                        │
│  • IVT/IDT: 인터럽트 벡터 테이블                                      │
│  • IRQ: 인터럽트 요청 라인                                            │
│  • NMI: 마스크 불가 인터럽트                                          │
│                                                                     │
│  【실시간 시스템】                                                    │
│  • 인터럽트 지연 최소화                                               │
│  • 우선순위 역전 방지                                                 │
│  • ISR 최적화                                                        │
│                                                                     │
│  【I/O 시스템】                                                       │
│  • 인터럽트 구동 I/O                                                  │
│  • DMA 완료 인터럽트                                                 │
│  • 블로킹 vs 논블로킹                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Top Half vs Bottom Half                                      │ │
│  │  ─────────────────────────                                    │ │
│  │                                                               │ │
│  │  Top Half (ISR):                                               │ │
│  │  • 인터럽트 컨텍스트에서 실행                                   │ │
│  │  • 하드웨어 처리만 수행                                         │ │
│  │  • 최대한 빠르게 완료                                           │ │
│  │  • 인터럽트 비활성화 상태                                        │ │
│  │                                                               │ │
│  │  Bottom Half (SoftIRQ, Tasklet, Workqueue):                    │ │
│  │  • 프로세스 컨텍스트에서 실행                                   │ │
│  │  • 시간이 오래 걸리는 작업                                      │ │
│  │  • 인터럽트 활성화 상태                                          │ │
│  │  • 선점 가능                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 인터럽트 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 정보 확인 (Linux)】                                       │
│  ──────────────────                                                  │
│  # 인터럽트 통계                                                     │
│  $ cat /proc/interrupts                                             │
│                                                                     │
│  # 인터럽트 발생 빈도                                                 │
│  $ watch -n 1 "cat /proc/interrupts"                                │
│                                                                     │
│  # 소프트 인터럽트 확인                                               │
│  $ cat /proc/softirqs                                               │
│                                                                     │
│  # IRQ 밸런싱 확인                                                   │
│  $ cat /proc/irq/*/smp_affinity                                     │
│                                                                     │
│  【인터럽트 최적화】                                                   │
│  ──────────────────                                                  │
│  1. ISR 최소화: 하드웨어 처리만                                      │
│  2. Bottom Half 활용: 지연 처리                                      │
│  3. IRQ 밸런싱: CPU 간 분산                                          │
│  4. 인터럽트 코일레싱: 여러 이벤트 묶기                               │
│  5. 폴링 혼합: 고부하 시 폴링 전환                                    │
│                                                                     │
│  【인터럽트 관련 명령】                                                │
│  ──────────────────                                                  │
│  # IRQ 친화성 설정                                                   │
│  $ echo 2 > /proc/irq/24/smp_affinity                              │
│                                                                     │
│  # 인터럽트 통계 초기화                                               │
│  $ echo 1 > /proc/irq/default_smp_affinity                         │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 인터럽트: 비동기 이벤트 알림 메커니즘
• 종류: 하드웨어, 소프트웨어, 트랩, 예외
• 처리: 신호 → 확인 → 저장 → ISR → 복원
• 우선순위: NMI > 하드웨어 > 일반 장치
• 마스킹: IF 플래그로 제어 (NMI 제외)
• 최적화: Top Half/Bottom Half 분리
```
