# 017. 인터럽트 벡터 (Interrupt Vector)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 번호를 ISR 주소로 매핑하는 테이블
> 2. **가치**: 빠른 ISR 찾기, 일관된 인터럽트 처리
> 3. **융합**: IDT, IVT, PIC/APIC와 연관

---

## Ⅰ. 개요

### 개념 정의

인터럽트 벡터(Interrupt Vector)는 **인터럽트 번호(IRQ)를 해당 인터럽트 서비스 루틴(ISR)의 시작 주소로 매핑하는 테이블 또는 배열**이다.

### 💡 비유: 전화번호부
인터럽트 벡터는 **전화번호부**와 같다. 이름(인터럽트 번호)을 찾으면 전화번호(ISR 주소)가 나온다. 키보드가 눌리면(인터럽트 33) 전화번호부에서 키보드 담당자(ISR)를 찾아 연결한다.

### 인터럽트 벡터 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 벡터 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 벡터 테이블 (IVT/IDT)】                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │              ┌─────────────────────────────────┐              │ │
│  │              │   Interrupt Vector Table (IVT)  │              │ │
│  │              ├─────────────────────────────────┤              │ │
│  │              │ Vector 0  │ ISR Address for NMI │              │ │
│  │              ├───────────┼────────────────────┤              │ │
│  │              │ Vector 1  │ ISR Address for ... │              │ │
│  │              ├───────────┼────────────────────┤              │ │
│  │              │ Vector 2  │ ISR Address for ... │              │ │
│  │              ├───────────┼────────────────────┤              │ │
│  │              │   ...     │       ...           │              │ │
│  │              ├───────────┼────────────────────┤              │ │
│  │              │ Vector 32 │ ISR Address for IRQ0│              │ │
│  │              ├───────────┼────────────────────┤              │ │
│  │              │ Vector 33 │ ISR Address for IRQ1│              │ │
│  │              ├───────────┼────────────────────┤              │ │
│  │              │   ...     │       ...           │              │ │
│  │              ├───────────┼────────────────────┤              │ │
│  │              │ Vector 255│ ISR for System Call │              │ │
│  │              └─────────────────────────────────┘              │ │
│  │                          │                                    │ │
│  │                          │ 주소 조회                          │ │
│  │                          ▼                                    │ │
│  │              ┌─────────────────────────────────┐              │ │
│  │              │   ISR Code (커널 메모리)         │              │ │
│  │              │   0xFFFF0000: push registers    │              │ │
│  │              │   0xFFFF0004: ...               │              │ │
│  │              │   0xFFFF0008: iret              │              │ │
│  │              └─────────────────────────────────┘              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【x86 IDT (Interrupt Descriptor Table)】                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  IDT Entry (Descriptor) - 16 bytes (64-bit):                   │ │
│  │                                                               │ │
│  │  ┌──────────┬──────────┬──────────┬──────────┬──────────┐   │ │
│  │  │Offset    │Selector  │Attr      │Reserved  │Offset    │   │ │
│  │  │[63:32]   │[31:16]   │[15:0]    │          │[15:0]    │   │ │
│  │  └──────────┴──────────┴──────────┴──────────┴──────────┘   │ │
│  │                                                               │ │
│  │  Offset: ISR 시작 주소                                         │ │
│  │  Selector: 코드 세그먼트 선택자                                 │ │
│  │  Attr: 타입, DPL, Present 플래그                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 인터럽트 벡터 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 벡터 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86 인터럽트 벡터 할당】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  벡터 번호        유형                    설명                   │ │
│  │  ────            ────                    ────                   │ │
│  │  0               Divide Error            0으로 나누기           │ │
│  │  1               Debug                   디버그                  │ │
│  │  2               NMI                     마스크 불가 인터럽트     │ │
│  │  3               Breakpoint              중단점                  │ │
│  │  6               Invalid Opcode          잘못된 명령어           │ │
│  │  13              General Protection      보호 위반                │ │
│  │  14              Page Fault             페이지 부재              │ │
│  │  0x80 (128)      System Call            리눅스 시스템 콜        │ │
│  │  32-255          External IRQ           외부 인터럽트            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 벡터 vs IRQ】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  구분              설명                                        │ │
│  │  ────              ────                                        │ │
│  │  IRQ              하드웨어 인터럽트 요청 라인                     │ │
│  │  Vector           IDT 내 인덱스                                │ │
│  │                                                               │ │
│  │  매핑: IRQ n → Vector (32 + n)                                 │ │
│  │  예: IRQ 0 (Timer) → Vector 32                                 │ │
│  │      IRQ 1 (Keyboard) → Vector 33                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【APIC 인터럽트 벡터】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  범위              용도                                        │ │
│  │  ────              ────                                        │ │
│  │  0-15             예외 (Exception)                             │ │
│  │  16-31           예약됨                                        │ │
│  │  32-255          사용자 정의 가능                               │ │
│  │                                                               │ │
│  │  Local APIC: IPI (Inter-Processor Interrupt)                   │ │
│  │  IO APIC: 외부 장치 인터럽트                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 인터럽트 벡터 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 벡터 분석                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 벡터 확인 (Linux)】                                        │
│  ──────────────────                                                  │
│  # 인터럽트 번호와 핸들러 확인                                        │

│  request_irq(irq, handler, flags, name, dev);                       │
│                                                                     │
│  // 인터럽트 핸들러 해제                                              │

│  $ cyclictest                                                       │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 인터럽트 벡터: IRQ → ISR 주소 매핑 테이블
• x86: IDT (Interrupt Descriptor Table)
• 구조: Offset + Selector + Attributes
• 예외: 0-31 (Divide, GPF, Page Fault 등)
• 외부 IRQ: 32-255
• 보안: DPL 검사, IDT 후킹 방지
```
