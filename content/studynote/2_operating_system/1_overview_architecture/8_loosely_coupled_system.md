# 약결합 시스템 (Loosely Coupled System)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 약결합 시스템은 두 개 이상의 컴퓨터 시스템이 각각 **독립된 주메모리와 운영체제**를 보유하고, 통신망(Network)을 통해 메시지를 주고받으며 협력하는 분산 컴퓨팅 구조다.
> 2. **가치**: 강결합 시스템의 물리적 확장성 한계를 극복하여 **수천 대 이상의 노드를 연결**할 수 있으며, 특정 노드 고장이 전체 시스템으로 전파되지 않는 높은 고립성과 유연성을 제공한다.
> 3. **융합**: 현대의 분산 시스템, 클라우드 컴퓨팅, 서버 클러스터링의 핵심 아키텍처이며, 노드 간의 통신 지연(Latency)과 데이터 일관성을 소프트웨어 수준에서 관리하는 것이 최대 과제다.

+++

## Ⅰ. 약결합 시스템 (Loosely Coupled System)의 개요

### 1. 정의
- 프로세서마다 자신의 독자적인 메모리를 가지고 있으며, 각 프로세서가 독립적으로 운영체제를 실행하는 '다중 컴퓨터(Multicomputer)' 시스템이다.

### 2. 특징
- **분산 메모리**: 각 노드는 자신만의 로컬 주소 공간을 가지며 타 노드의 메모리에 직접 접근할 수 없다.
- **메시지 전달**: 데이터 교환을 위해 입출력 채널이나 네트워크 프로토콜(TCP/IP 등)을 통한 **메시지 패싱(Message Passing)** 방식을 사용한다.
- **자치성**: 각 노드는 독립적인 개체로서 스스로를 관리한다.

### 3. 💡 비유: 각자 집에서 일하는 프리랜서 팀
- 약결합 시스템은 **"각자의 집(메모리)에서 자기 도구로 일하고 메일(네트워크)로 결과를 주고받는 팀"**과 같다.
- 서로 부딪힐 일(버스 경합)은 없어서 팀원을 무한정 늘릴 수 있지만, 정보를 주고받을 때 메일을 쓰고 기다리는 시간(통신 지연)이 걸린다. 한 명이 컴퓨터가 고장 나도 다른 팀원들은 자기 일을 계속할 수 있다.

+++

## Ⅱ. 약결합 시스템의 아키텍처 및 통신 원리 (ASCII)

노드들이 통신 인프라를 통해 느슨하게 연결된 구조다.

```text
    [ Computer Node A ]      [ Computer Node B ]      [ Computer Node C ]
    ┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
    │  [ Local OS ]   │      │  [ Local OS ]   │      │  [ Local OS ]   │
    │  [ Local RAM ]  │      │  [ Local RAM ]  │      │  [ Local RAM ]  │
    │  [ CPU ]        │      │  [ CPU ]        │      │  [ CPU ]        │
    └───────┬─────────┘      └───────┬─────────┘      └───────┬─────────┘
            │                        │                        │
    ────────┴────────────────────────┴────────────────────────┴────────
                    [ 통신망 (LAN, WAN, Switch) ]
                    (Message Passing Interface)
```

**[핵심 동작 메커니즘]**
1. **메시지 전달**: 한 노드가 데이터를 보내려면 통신 인터페이스를 통해 패킷을 캡슐화하여 네트워크로 송출해야 한다.
2. **독립 스케줄링**: 각 노드의 OS는 자기 노드 내의 작업만 관리하므로 동기화 복잡도가 낮다.
3. **결합 해제**: 특정 노드를 시스템에서 제거하거나 추가할 때 전체 시스템을 멈출 필요가 없다.

+++

## Ⅲ. 강결합 vs 약결합 시스템 상세 비교

| 비교 항목 | 강결합 (Tightly Coupled) | 약결합 (Loosely Coupled) |
|:---|:---|:---|
| **기본 단위** | 프로세서 (Processor) | **컴퓨터 (Computer/Node)** |
| **메모리** | **주메모리 공유 (Shared)** | **분산 메모리 (Distributed/Local)** |
| **통신 매체** | 시스템 버스 (Bus) | 통신망 (LAN/WAN) |
| **결합 수준** | 물리적, 밀접함 | 논리적, 느슨함 |
| **확장성** | 낮음 (버스 경합 발생) | **매우 높음 (수평적 확장)** |
| **오류 격리** | 어려움 (메모리 오염 공유) | **용이함 (독립된 OS)** |

- **📢 섹션 요약 비유**: 한 권의 노트를 여러 명이 같이 쓰는 것이 강결합이라면, 각자 노트를 쓰다가 필요한 부분만 복사해서 전달하는 것이 약결합입니다.

+++

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy)

### 1. 수평적 확장(Scale-out) 전략
- **기술사적 결단**: 현대 클라우드 서비스(Netflix, Google 등)는 트래픽 폭증에 대비해 **약결합 아키텍처 기반의 Scale-out**을 기본 전략으로 삼는다. 
- **인사이트**: 서버 1대를 수천만 원짜리로 바꾸는 것(Scale-up)보다, 수십만 원짜리 서버 100대를 약결합으로 묶는 것이 가성비와 안정성 면에서 압도적이다.

### 2. 분산 데이터 일관성 (CAP 정리)
- **판단**: 약결합 시스템에서는 모든 노드가 똑같은 최신 데이터를 갖게 하는 것이 매우 어렵다.
- **결단**: 실무 아키텍트는 **일관성(Consistency), 가용성(Availability), 분단 허용성(Partition Tolerance)** 중 서비스 특성에 맞는 2가지를 선택하는 전략적 타협이 필요하다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **무한 확장성**: 노드를 추가하는 것만으로 시스템 전체의 처리 용량을 거의 선형적으로 증대 가능.
- **고가용성**: 특정 지역의 데이터센터가 마비되어도 다른 지역 노드가 서비스를 이어가는 재해 복구(DR) 용이.

### 2. 미래 전망
앞으로의 약결합 시스템은 엣지 컴퓨팅(Edge Computing)과 만나 더욱 분산될 것이다. 전 세계 수십억 개의 기기가 약결합으로 연결되어 하나의 거대한 지능형 네트워크를 형성하게 되며, 이를 효율적으로 관리하기 위한 **서버리스(Serverless)**와 **서비스 메시(Service Mesh)** 기술이 약결합의 복잡성을 은닉해주는 핵심 인프라 기술로 발전할 것이다.

- **📢 섹션 요약 비유**: 거대한 유조선 한 척보다 작은 보트 수백 척이 모인 선단이 폭풍우(장애) 속에서도 더 유연하고 안전하게 항해할 수 있습니다.

+++

### 📌 관련 개념 맵 (Knowledge Graph)
- **분산 운영체제 (Distributed OS)**: 약결합 시스템을 제어하는 소프트웨어.
- **강결합 시스템**: 약결합의 대척점에 있는 고속 공유 구조.
- **L4/L7 로드밸런서**: 약결합 노드들에게 작업을 분배하는 장치.

### 👶 어린이를 위한 3줄 비유 설명
1. 약결합 시스템은 친구들이 **'각자의 방'에서 따로 공부하다가 모르는 것만 전화로 물어보는 것**과 같아요.
2. 내 책상이 어지러워도 친구 공부에는 방해가 안 되고(오류 격리), 친구가 100명으로 늘어나도 방만 더 만들면 되죠(확장성).
3. 가끔 전화가 끊기거나 느릴 때도 있지만, 친구들끼리 서로 도와주기 때문에 아주 큰 숙제도 척척 해낼 수 있답니다!
