# 509. 인터럽트 벡터 (Interrupt Vector)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 번호와 핸들러 주소 매핑
> 2. **가치**: 빠른 핸들러 검색과 실행
> 3. **융합**: IDT, 인터럽트 핸들러, 예외와 연관

---

## Ⅰ. 개요

### 개념 정의

인터럽트 벡터(Interrupt Vector)는 **인터럽트 번호에 대응하는 인터럽트 핸들러의 주소를 저장하는 테이블**이다.

### 💡 비유: 전화번호부
인터럽트 벡터는 **전화번호부**와 같다. 이름(인터럽트 번호)으로 전화번호(핸들러 주소)를 찾는다.

### 인터럽트 벡터 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 벡터 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【인터럽트 벡터 테이블】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  인터럽트 벡터 테이블 (IVT) / 인터럽트 디스크립터 테이블 (IDT):            │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  번호   핸들러 주소        설명                              │    │ │
│  │  │  ────   ────────────       ────                              │    │ │
│  │  │   0     0x00100000        Divide Error (#DE)               │    │ │
│  │  │   1     0x00100100        Debug Exception (#DB)            │    │ │
│  │  │   2     0x00100200        NMI Interrupt                    │    │ │
│  │  │   3     0x00100300        Breakpoint (#BP)                 │    │ │
│  │  │   ...   ...              ...                               │    │ │
│  │  │   14    0x00100E00        Page Fault (#PF)                 │    │ │
│  │  │   ...   ...              ...                               │    │ │
│  │  │  32     0x00102000        IRQ0 (Timer)                     │    │ │
│  │  │  33     0x00102100        IRQ1 (Keyboard)                  │    │ │
│  │  │   ...   ...              ...                               │    │ │
│  │  │  255    0x0010FF00        (사용자 정의)                      │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터럽트 처리 흐름】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │       인터럽트 발생                                                      │ │
│  │           │                                                            │ │
│  │           ▼                                                            │ │
│  │  ┌─────────────────────────────────────────────────────────┐            │ │
│  │  │  1. 인터럽트 번호 확인 (IRQ n)                             │            │ │
│  │  └─────────────────────────────────────────────────────────┘            │ │
│  │           │                                                            │ │
│  │           ▼                                                            │ │
│  │  ┌─────────────────────────────────────────────────────────┐            │ │
│  │  │  2. 벡터 테이블에서 n번 엔트리 조회                        │            │ │
│  │  │     handler_addr = IDT[n].handler                        │            │ │
│  │  └─────────────────────────────────────────────────────────┘            │ │
│  │           │                                                            │ │
│  │           ▼                                                            │ │
│  │  ┌─────────────────────────────────────────────────────────┐            │ │
│  │  │  3. 핸들러 주소로 점프                                     │            │ │
│  │  │     JMP handler_addr                                      │            │ │
│  │  └─────────────────────────────────────────────────────────┘            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 벡터 상세                                       │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86 IDT 구조】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  IDT 디스크립터 (8바이트 또는 16바이트):                                   │ │
│  │  ┌────────────────────────────────────────────────────────┐           │ │
│  │  │  오프셋[15:0]    │ 세그먼트 셀렉터 │ 예약 │ P│DPL│0│타입 │           │ │
│  │  │  ─────────────────────────────────────────────────────────           │ │
│  │  │  오프셋[31:16]   │          예약          │          │           │ │
│  │  │  ─────────────────────────────────────────────────────────           │ │
│  │  │  (64비트 모드: 오프셋[63:32] 추가)                                  │           │ │
│  │  └────────────────────────────────────────────────────────┘           │ │
│  │                                                             │ │
│  │  필드 설명:                                                              │ │
│  │  • 오프셋: 핸들러 주소                                                   │ │
│  │  • 세그먼트 셀렉터: 코드 세그먼트                                        │ │
│  │  • P (Present): 유효 여부                                               │ │
│  │  • DPL: Descriptor Privilege Level                                     │ │
│  │  • 타입: 인터럽트 게이트 / 트랩 게이트                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【IRQ 매핑】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  IRQ        벡터 번호        장치                                         │ │
│  │  ────       ─────────       ────                                         │ │
│  │  IRQ0       32              타이머                                        │ │
│  │  IRQ1       33              키보드                                        │ │
│  │  IRQ2       34              캐스케이드                                    │ │
│  │  IRQ3       35              COM2                                         │ │
│  │  IRQ4       36              COM1                                         │ │
│  │  IRQ14      46              IDE Primary                                  │ │
│  │  IRQ15      47              IDE Secondary                                │ │
│  │                                                             │ │
│  │  벡터 0-31: CPU 예외용 예약                                              │ │
│  │  벡터 32-255: 외부 인터럽트용 사용 가능                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【게이트 타입】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  게이트          IF 플래그      용도                                      │ │
│  │  ────            ────────      ────                                      │ │
│  │  인터럽트 게이트   자동 클리어     하드웨어 인터럽트                        │ │
│  │  트랩 게이트       변경 없음      소프트웨어 예외, 시스템 콜                │ │
│  │  태스크 게이트     -            태스크 전환 (거의 사용 안 함)              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【IDT 정보 확인】                                                      │
│  ──────────────────                                                  │
│  // IDTR 레지스터 확인                                                   │
│  $ dmesg | grep -i idt                                              │
│                                                                     │
│  // IDT 베이스 주소                                                    │
│  $ cat /proc/kallsyms | grep idt_table                              │
│                                                                     │
│  // QEMU에서 IDT 덤프                                                  │
│  (qemu) info idt                                                    │
│  (qemu) info registers                                              │
│                                                                     │
│  【인터럽트 벡터 매핑 확인】                                             │
│  ──────────────────                                                  │
│  // 인터럽트 핸들러 심볼                                                  │
│  $ cat /proc/kallsyms | grep -i "irq\|exception" | head -20         │
│                                                                     │
│  // IRQ 번호 확인                                                       │
│  $ cat /proc/interrupts                                              │
│                                                                     │
│  // 특정 IRQ의 핸들러                                                   │
│  $ cat /proc/irq/*/smp_affinity                                      │
│                                                                     │
│  【커널에서 IDT 설정】                                                   │
│  ──────────────────                                                  │
│  // IDT 엔트리 구조체                                                    │
│  struct idt_entry {                                                  │
│      u16 offset_low;    // 핸들러 주소 하위 16비트                        │
│      u16 selector;      // 세그먼트 셀렉터                               │
│      u8  zero;          // 예약                                        │
│      u8  type_attr;     // 타입과 속성                                   │
│      u16 offset_high;   // 핸들러 주소 상위 16비트                         │
│  } __attribute__((packed));                                          │
│                                                                     │
│  // IDT 설정 함수                                                       │
│  void set_idt_gate(int n, u32 handler) {                             │
│      idt[n].offset_low = handler & 0xFFFF;                           │
│      idt[n].offset_high = (handler >> 16) & 0xFFFF;                  │
│      idt[n].selector = KERNEL_CS;                                    │
│      idt[n].zero = 0;                                                │
│      idt[n].type_attr = 0x8E;  // 인터럽트 게이트, P=1, DPL=0           │
│  }                                                                   │
│                                                                     │
│  // IDT 로드                                                           │
│  void load_idt() {                                                   │
│      struct idt_ptr idtp;                                            │
│      idtp.limit = sizeof(idt) - 1;                                   │
│      idtp.base = (u32)&idt;                                          │
│      asm volatile("lidt %0" : : "m"(idtp));                          │
│  }                                                                   │
│                                                                     │
│  【인터럽트 게이트 설정 (Linux 커널)】                                    │
│  ──────────────────                                                  │
│  // 게이트 설정                                                         │
│  void set_intr_gate(unsigned int n, void *addr) {                    │
│      struct idt_data data;                                           │
│      init_idt_data(&data, n, addr);                                  │
│      idt_setup_from_table(idt_table, &data, 1, false);               │
│  }                                                                   │
│                                                                     │
│  // 시스템 게이트 (사용자 호출 가능)                                       │
│  void set_system_intr_gate(unsigned int n, void *addr) {             │
│      // DPL = 3으로 설정                                               │
│  }                                                                   │
│                                                                     │
│  【ARM 인터럽트 벡터】                                                   │
│  ──────────────────                                                  │
│  // ARM 벡터 테이블 (고정 주소)                                          │
│  0x00000000 또는 0xFFFF0000:                                          │
│  0x00: Reset                                                         │
│  0x04: Undefined Instruction                                         │
│  0x08: Software Interrupt (SWI)                                      │
│ 0x0C: Prefetch Abort                                                 │
│ 0x10: Data Abort                                                     │
│ 0x14: Reserved                                                       │
│ 0x18: IRQ (Normal Interrupt)                                         │
│ 0x1C: FIQ (Fast Interrupt)                                           │
│                                                                     │
│  // 벡터 테이블 설정                                                      │
│  void set_vector_base(void *base) {                                  │
│      asm volatile("mcr p15, 0, %0, c12, c0, 0" :: "r"(base));        │
│  }                                                                   │
│                                                                     │
│  【인터럽트 추적】                                                        │
│  ──────────────────                                                  │
│  // ftrace로 IDT 접근 추적                                              │
│  $ trace-cmd record -e irq:irq_handler_entry -e irq:irq_handler_exit│
│  $ trace-cmd report                                                  │
│                                                                     │
│  // perf로 인터럽트 분석                                                  │
│  $ perf record -e irq:* -a sleep 5                                  │
│  $ perf script                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 인터럽트 번호와 핸들러 주소 매핑 테이블
• x86: IDT (Interrupt Descriptor Table)
• ARM: 벡터 테이블 (고정 주소)
• 구조: 오프셋, 세그먼트, 타입, DPL
• 범위: x86 - 0~255 (0-31 예외용 예약)
• 게이트: 인터럽트 게이트 (IF 클리어), 트랩 게이트
• 설정: LIDT 명령어로 로드
• 확인: /proc/interrupts, dmesg
```
