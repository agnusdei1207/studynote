# BPF (Berkeley Packet Filter) / eBPF (Extended BPF)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: eBPF (Extended Berkeley Packet Filter)는 리눅스 커널 소스 코드를 수정하거나 커널 모듈을 로드하지 않고도, 커널 내부의 특정 이벤트(시스템 콜, 네트워크 패킷 등) 발생 시 **사용자 정의 프로그램을 안전하게 실행**할 수 있게 해주는 커널 내 샌드박스 기술이다.
> 2. **가치**: 커널 패닉 위험 없이 고성능 네트워킹, 세밀한 보안 필터링, 시스템 전반의 관찰성(Observability)을 제공하며, 커널 모드와 사용자 모드 간의 빈번한 데이터 복사 오버헤드를 획기적으로 줄인다.
> 3. **융합**: '운영체제의 자바스크립트'라고 불리며, 클라우드 네이티브 환경(Cilium, Falco 등)에서 네트워크 서비스 메쉬와 런타임 보안의 핵심 엔진으로 급부상하고 있다.

+++

## Ⅰ. BPF에서 eBPF로의 진화

### 1. Classic BPF (cBPF)
- 1992년 탄생. 주로 네트워크 패킷을 필터링하여 사용자 공간으로 넘겨줄 데이터를 선별하는 단순한 가상 머신이었다. (예: `tcpdump`)

### 2. Extended BPF (eBPF)
- 리눅스 커널 3.18부터 도입. 레지스터 개수 증가, 메모리 접근(Maps) 허용, 다양한 커널 이벤트 훅(Hook) 연결 등 기능을 대폭 확장하여 **커널 전체를 위한 범용 실행 엔진**으로 진화했다.

### 3. 💡 비유: '커널이라는 거대 공장에 설치한 스마트 CCTV'
- 기존 커널 모듈(LKM)이 공장 기계를 직접 개조하는 위험한 작업이라면, eBPF는 **'공장 곳곳에 설치한 지능형 CCTV와 센서'**와 같다.
- 기계를 멈추거나 뜯지 않고도 물건이 지나갈 때마다(이벤트 발생) 즉시 무게를 재거나 불량품을 골라내는(사용자 정의 로직 실행) 작업을 아주 안전하고 빠르게 수행한다.

## Ⅱ. eBPF의 동작 메커니즘 (Deep Dive)

eBPF의 가장 큰 특징은 **'안전성'**과 **'성능'**의 공존이다.

### 1. 실행 프로세스
1. **작성**: 사용자가 C나 Rust로 eBPF 프로그램을 작성한다.
2. **컴파일**: LLVM/Clang을 통해 eBPF 바이트코드로 컴파일한다.
3. **로드 및 검증 (Verifier)**: 커널에 로드할 때, **검증기 (Verifier)**가 무한 루프, 메모리 침범, 권한 유무 등을 깐깐하게 검사한다. 실패 시 로드가 거부된다.
4. **JIT 컴파일**: 검증된 바이트코드를 해당 CPU의 네이티브 기계어로 즉시 변환(Just-In-Time)하여 실행 속도를 최적화한다.
5. **이벤트 훅**: 지정된 지점(kprobes, tracepoints 등)에서 프로그램이 실행된다.

### 2. eBPF Maps
- 커널에서 실행 중인 eBPF 프로그램과 사용자 공간의 앱이 데이터를 주고받기 위한 공유 저장소(Key-Value Store)이다. 이를 통해 커널의 활동을 실시간으로 집계하고 분석할 수 있다.

### 3. ASCII 구조 다이어그램
```text
    [ User Space ]              [ Kernel Space ]
    ┌──────────────┐            ┌────────────────────────────────┐
    │ eBPF Program │            │  eBPF Runtime (Sandbox)        │
    │ (C / Rust)   │            │  ┌──────────┐   ┌───────────┐  │
    └──────┬───────┘            │  │ Verifier │ ▶ │ JIT Comp. │  │
           ▼ (Bytecode)         │  └────┬─────┘   └─────┬─────┘  │
    ┌──────┴───────┐            │       ▼               ▼        │
    │ bpf() Syscall│ ──────────▶│ [ eBPF Program Execution ]     │
    └──────────────┘            └───────┬───────────────┬────────┘
                                        ▲               │
    ┌──────────────┐            ┌───────┴──────┐        │ (Maps)
    │ Control App  │◀──(Data)──▶│  eBPF Maps   │◀───────┘
    └──────────────┘            └──────────────┘
```

- **📢 섹션 요약 비유**: 외부 전문가(eBPF 프로그램)를 초빙하되, 공장에 들어오기 전 철저한 보안 검색(Verifier)을 거치고 전용 작업복(샌드박스)을 입혀 사고를 원천 차단하는 것과 같습니다.

## Ⅲ. 주요 활용 분야 분석

| 분야 | 설명 | 주요 도구 |
|:---|:---|:---|
| **네트워킹** | 고속 패킷 처리, 부하 분산, 서비스 메쉬 구현 | Cilium, Katran |
| **관찰성** | 시스템 콜 추적, CPU/메모리 부하 프로파일링 | bcc, bpftrace, Pixie |
| **보안** | 비정상 행동 탐지, 시스템 콜 차단, 컨테이너 보안 | Falco, Tetragon |

- **📢 섹션 요약 비유**: 의사가 환자를 수술하지 않고도 혈관 속을 돌아다니는 나노 로봇(eBPF)을 통해 실시간으로 진단하고 치료하는 것과 같습니다.

## Ⅳ. 실무 적용 및 아키텍처적 가치 (Strategy)

### 1. 실무 시나리오: 마이크로서비스 가시성 확보
- **상황**: 수백 개의 컨테이너가 서로 통신하는데, 어디서 병목이 생기는지 알 수 없음.
- **판단**: 애플리케이션 코드를 수정(Instrumentation)하지 않고, 커널 레벨에서 **eBPF 기반의 관찰 도구**를 실행한다. 모든 네트워크 패킷의 지연 시간을 커널에서 직접 측정하여 지도로 시각화함으로써 인프라 부하 없이 병목 지점을 즉시 찾아낸다.

### 2. 기술사적 인사이트: 커널의 플랫폼화
- eBPF는 커널을 단순한 '자원 관리자'에서 '사용자가 기능을 확장할 수 있는 플랫폼'으로 탈바꿈시켰다. 이는 과거 마이크로 커널이 꿈꿨던 유연성을 모놀리식 커널의 성능 손실 없이 달성한 것으로, 운영체제 발전사의 기념비적인 사건이다.

- **📢 섹션 요약 비유**: 딱딱한 기성복(고정된 커널) 시대가 가고, 언제든 주머니를 달거나 길이를 조절할 수 있는 맞춤형 스마트 의류(eBPF 커널) 시대가 온 것입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **성능 최적화**: 사용자 모드 전환 없이 커널 내에서 로직을 처리하여 처리량 향상.
- **안정성 강화**: 잘못된 코드로 인한 시스템 다운(Kernel Panic) 방지.

### 2. 미래 전망
eBPF는 리눅스를 넘어 윈도우(eBPF for Windows)로도 확산되고 있다. 앞으로 인공지능 추론 엔진을 커널 내 eBPF로 구현하여 시스템 레벨의 지능형 스케줄링을 수행하거나, 하드웨어 장치(SmartNIC)와 직접 연동되어 '소프트웨어 정의 인프라'의 완성을 이끌 기술적 중심축이 될 것이다.

- **📢 섹션 요약 비유**: 운영체제의 뇌(커널) 속에 지능형 칩을 심어, 생각하는 대로 신체(시스템)를 진화시키는 사이보그 운영체제의 서막입니다.

### 📌 관련 개념 맵 (Knowledge Graph)
- **모듈 적재 (LKM)**: eBPF가 대체하거나 보완하는 전통적 방식.
- **시스템 호출 (Seccomp)**: eBPF를 활용한 보안 필터링의 대상.
- **JIT 컴파일**: eBPF의 고성능을 보장하는 기술.

### 👶 어린이를 위한 3줄 비유 설명
1. eBPF는 컴퓨터의 대장님인 '커널'에게 **'새로운 마법 주문을 안전하게 가르쳐주는 것'**과 같아요.
2. 예전에는 주문을 잘못 외우면 컴퓨터가 아예 뻗어버렸지만, eBPF는 미리 주문이 안전한지 확인하고 실행해서 안심할 수 있죠.
3. 덕분에 컴퓨터는 끄지 않고도 순식간에 네트워크 요정도 되고, 보안 경비원도 될 수 있는 멋진 능력이 생긴답니다!
