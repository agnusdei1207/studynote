# 029. 시스템 생성 (SYSGEN)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 특정 하드웨어에 맞춰 운영체제를 구성하고 생성하는 과정
> 2. **가치**: 하드웨어 최적화, 자원 효율성, 시스템 맞춤화
> 3. **융합**: 커널 컴파일, 드라이버 구성, 디바이스 트리와 연관

---

## Ⅰ. 개요

### 개념 정의

시스템 생성(System Generation, SYSGEN)은 **특정 하드웨어 구성에 맞춰 운영체제를 생성하고 구성하는 과정**으로, 시스템이 실행될 하드웨어 환경에 최적화된 OS 이미지를 만드는 작업이다. Unix에서는 `config` 프로그램을 사용하여 커널 구성을 생성한다.

### 💡 비유: 주문 제작 가구
시스템 생성은 **주문 제작 가구**와 같다. 집(하드웨어)의 크기와 구조에 맞춰 가구(OS)를 제작한다. 벽면의 높이, 방의 크기, 창문 위치 등을 측정(하드웨어 탐지)한 후 맞춤형 가구를 만든다.

## Ⅱ. 시스템 생성 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 생성 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SYSGEN 입력 정보】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  정보              설명                                        │ │
│  │  ────              ────                                        │ │
│  │  CPU 정보          아키텍처, 코어 수, 속도                      │ │
│  │  메모리 크기        RAM 용량, 주소 범위                          │ │
│  │  저장장치          디스크 타입, 크기, 인터페이스                │ │
│  │  I/O 장치          콘솔, 터미널, 프린터                         │ │
│  │  네트워크          NIC 타입, 프로토콜                           │ │
│  │  드라이버          필요한 디바이스 드라이버                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Unix config 프로그램】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  # 구성 파일 예시 (BSD)                                        │ │
│  │  machine     i386                                            │ │
│  │  cpu         I686_CPU                                        │ │
│  │  ident       MYKERNEL                                        │ │
│  │                                                               │ │
│  │  options     SCHED_ULE        # 스케줄러                       │ │
│  │  options     PREEMPTION       # 선점                          │ │
│  │  options     SMP              # 멀티프로세서                   │ │
│  │                                                               │ │
│  │  device      pci              # PCI 버스                       │ │
│  │  device      ata              # ATA 디스크                     │ │
│  │  device      em               # Intel 이더넷                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux 커널 구성】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  # 구성 도구                                                   │ │
│  │  $ make config          # 텍스트 기반                          │ │
│  │  $ make menuconfig      # 메뉴 기반 (ncurses)                  │ │
│  │  $ make xconfig         # GUI 기반                            │ │
│  │  $ make defconfig       # 기본 설정                            │ │
│  │                                                               │ │
│  │  # 구성 파일                                                   │ │
│  │  /usr/src/linux/.config                                       │ │
│  │                                                               │ │
│  │  # 빌드                                                        │ │
│  │  $ make                                               │ │
│  │  $ make modules_install                                │ │
│  │  $ make install                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 시스템 생성 × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 생성 × 타 과목 융합                              │

│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • CPU 아키텍처: x86, ARM, RISC-V 선택                             │
│  • 메모리 맵: 물리 주소, MMU 설정                                  │
│  • 인터럽트 컨트롤러: APIC, GIC 구성                               │
│                                                                     │
│  【임베디드 시스템】                                                  │
│  • Device Tree: 하드웨어 정보 기술 (ARM)                            │
│  • BSP (Board Support Package): 보드별 설정                        │
│  • 크로스 컴파일: 타겟 보드용 빌드                                  │
│                                                                     │
│  【커널 아키텍처】                                                    │
│  • 모놀리식: 전체 재컴파일 필요                                      │
│  • 모듈화: 동적 모듈 로드 가능                                      │
│  • 마이크로커널: 서버 구성 필요                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  【시스템 생성 옵션】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  옵션              설명                    예시                │ │
│  │  ────              ────                    ────                │ │
│  │  SMP              대칭 다중 처리           CONFIG_SMP         │ │
│  │  PREEMPT          선점형 커널             CONFIG_PREEMPT     │ │
│  │  DEBUG            디버깅 심볼             CONFIG_DEBUG       │ │
│  │  MODULES          모듈 지원               CONFIG_MODULES     │ │
```

## Ⅳ. 실무 적용

### 시스템 생성 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 생성 분석                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 빌드】                                                │
│  ──────────────────                                                  │
│  # 소스 다운로드                                                   │
│  $ git clone https://github.com/torvalds/linux.git                │
│  $ cd linux                                                      │
│                                                                     │
│  # 현재 시스템 설정 복사                                           │
│  $ cp /boot/config-$(uname -r) .config                            │
│                                                                     │
│  # 설정 메뉴 실행                                                   │
│  $ make menuconfig                                               │
│                                                                     │
│  # 커널 빌드                                                        │
│  $ make -j$(nproc)                                               │
│                                                                     │
│  # 모듈 설치                                                        │
│  $ sudo make modules_install                                      │
│                                                                     │
│  # 커널 설치                                                        │
│  $ sudo make install                                              │
│                                                                     │
│  【BSD 커널 빌드】                                                  │
│  ──────────────────                                                  │
│  # 설정 파일 편집                                                   │
│  $ cd /usr/src/sys/amd64/conf                                    │
│  $ cp GENERIC MYKERNEL                                           │
│  $ vi MYKERNEL                                                   │
│                                                                     │
│  # 커널 빌드                                                        │
│  $ config MYKERNEL                                               │
│  $ cd ../compile/MYKERNEL                                        │
│  $ make depend                                                   │
│  $ make                                                          │
│  $ make install                                                  │
│                                                                     │
│  【임베디드 리눅스 빌드】                                           │
│  ──────────────────                                                  │
│  # Buildroot 사용 예                                               │
│  $ make menuconfig         # 구성                                 │
│  $ make                    # 전체 빌드                              │
│  $ ls output/images/       # 이미지 확인                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

 관련 요약

```
• 시스템 생성: 하드웨어에 맞춘 OS 구성 과정
• 입력: CPU, 메모리, 장치, 드라이버 정보
• Unix: config 프로그램, 구성 파일
• Linux: make menuconfig, .config
• 목적: 최적화, 자원 효율성, 맞춤화
• 임베디드: Device Tree, BSP, 크로스 컴파일
• 결과: 커널 이미지, 모듈, initramfs
```
