# 005. 다중 처리 시스템 (Multiprocessing System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 개의 CPU(프로세서)가 메모리를 공유하며 병렬로 작업 수행
> 2. **가치**: 처리량 증대, 신뢰성 향상, 확장성 확보
> 3. **융합**: 캐시 일관성, NUMA, 스핀락, 로드 밸런싱과 연계

---

## Ⅰ. 개요

### 개념 정의

다중 처리 시스템(Multiprocessing System)은 **둘 이상의 CPU(프로세서)를 장착하여 여러 프로세스를 동시에 병렬로 처리할 수 있는 컴퓨터 시스템**이다. 단일 CPU가 빠르게 전환하는 시분할과 달리, 실제 물리적 병렬 처리가 이루어진다.

### 💡 비유: 주방
다중 처리 시스템은 **여러 명의 요리사가 있는 주방**과 같다. 요리사가 한 명이면(단일 CPU) 요리를 순서대로 해야 하지만, 요리사가 네 명이면(쿼드코어) 네 가지 요리를 동시에 만들 수 있다. 싱크대, 냉장고(공유 메모리)는 함께 쓴다.

## Ⅱ. 다중 처리 시스템 상세

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  구분              ASMP                    SMP                 │ │

│  │  장애 대응         마스터 장애 시 치명적    견고함                │ │

│  │                                                               │ │
│  │  과제              설명                    해결책               │ │

## Ⅲ. 다중 처리 시스템 × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                다중 처리 시스템 × 타 과목 융합                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • 멀티코어: 칩 내 다중 CPU 코어                                      │
│  • 캐시 일관성: MESI, MOESI 프로토콜                                  │
│  • 버스: 공유 버스, 크로스바, NoC                                     │
│  • 인터커넥트: QPI, HyperTransport, Infinity Fabric                 │
│                                                                     │
│  【병렬 처리】                                                        │
│  • 스레드: 멀티스레딩 활용                                            │
│  • 동기화: 락, 배리어, 원자 연산                                      │
│  • 병렬 알고리즘: 분할 정복, 맵리듀스                                  │
│                                                                     │
│  【데이터베이스】                                                      │
│  • 병렬 쿼리: 다중 CPU 활용                                           │
│  • 래치: 메모리 내 동기화                                             │
│  • SMP DBMS: Oracle RAC, PostgreSQL                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  성능 향상 (암달의 법칙)                                        │ │
│  │  ─────────────────────────                                    │ │
│  │                                                               │ │
│  │  Speedup = 1 / ((1 - P) + P/N)                                │ │
│  │                                                               │ │
│  │  P: 병렬화 가능한 비율                                         │ │
│  │  N: 프로세서 수                                                │ │
│  │                                                               │ │
│  │  예: P = 0.8 (80% 병렬), N = 4                                │ │
│  │  Speedup = 1 / (0.2 + 0.8/4) = 1 / 0.4 = 2.5배                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 다중 처리: 여러 CPU가 메모리 공유하며 병렬 처리
• ASMP: 마스터-슬레이브 구조, OS는 마스터만
• SMP: 모든 CPU 동등, 현대 시스템 표준
• NUMA: CPU별 로컬 메모리, 비균등 접근
• 도전 과제: 캐시 일관성, 동기화, 로드 밸런싱
• 성능: 암달의 법칙, 병렬화 한계 존재
• 실제: 멀티코어 CPU, SMP 서버, NUMA 시스템
```
