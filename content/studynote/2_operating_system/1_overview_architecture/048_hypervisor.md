# 048. 하이퍼바이저 (Hypervisor) / VMM

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 머신을 생성하고 관리하는 소프트웨어 계층
> 2. **가치**: 자원 추상화, 격리, 멀티테넌시 활용
> 3. **융합**: Type 1/Type 2, 전가상화/반가상화 연관

---

## Ⅰ. 개요
### 개념 정의
하이퍼바이저(Hypervisor) 또는 VMM(Virtual Machine Monitor)은 **가상 머신을 생성, 실행, 관리하는 소프트웨어 계층**으로, 물리 하드웨어와 가상 머신 사이의 추상화 계층을 제공한다. Type 1(베어메탈)과 Type 2(호스트형)으로 구분된다.
### 💡 비유: 건물 관리자
하이퍼바이저는 **건물 관리자**와 같다. 하나의 건물(물리 서버)에 여러 입주자(VM)를 관리한다. 각 입주자는 독립된 공간을 쓰지만, 모든 입주자는 관리자를 통해 시설을 공유한다. 관리자가 없으면 입주자 간 간섭도 없다.
