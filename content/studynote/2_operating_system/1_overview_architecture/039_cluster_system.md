# 039. 클러스터 시스템 (Cluster System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 컴퓨터가 하나의 시스템처럼 동작하는 연결 구조
> 2. **가치**: 고가용성, 부하 분산, 확장성, 병렬 처리
> 3. **융합**: HA, HPC, 로드 밸런싱, 분산 시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

클러스터 시스템(Cluster System)은 **여러 대의 컴퓨터(노드)가 네트워크로 연결되어 하나의 통합된 시스템처럼 동작하는 구조**로, 고가용성(HA), 부하 분산(Load Balancing), 고성능 컴퓨팅(HPC) 등을 위해 사용된다.

### 💡 비유: 협동 작업 팀
클러스터는 **협동 작업 팀**과 같다. 각 팀원(노드)이 개별적으로 일하지만, 하나의 목표를 위해 협력한다. 한 명이 아파도(장애) 다른 팀원이 업무를 대신하고(장애 조치), 일이 많으면(부하) 팀원을 더 투입한다(확장).

## Ⅳ. 실무 적용

### 클러스터 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                클러스터 분석                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Pacemaker/Corosync 관리 (Linux)】                                │
│  ──────────────────                                                  │
│  # 클러스터 상태                                                    │
│  $ pcs status                                                     │
│  $ pcs status nodes                                               │
│  $ pcs status resources                                           │
│                                                                     │
│  # 리소스 관리                                                      │
│  $ pcs resource create vip ocf:heartbeat:IPaddr2 \               │
│      ip=192.168.1.100 cidr_netmask=24 op monitor interval=30s    │
│                                                                     │
│  # 리소스 시작/중지                                                 │
│  $ pcs resource enable <resource>                                 │
│  $ pcs resource disable <resource>                                │
│                                                                     │
│  # 노드 대기 모드                                                   │
│  $ pcs cluster standby <node>                                     │
│  $ pcs cluster unstandby <node>                                   │
│                                                                     │
│  【HAProxy 로드 밸런서】                                             │
│  ──────────────────                                                  │
│  # 상태 확인                                                        │
│  $ systemctl status haproxy                                       │
│  $ haproxy -vv                                                    │
│                                                                     │
│  # 통계 확인                                                        │
│  $ echo "show stat" | socat stdio /var/run/haproxy.sock          │
│                                                                     │
│  【Kubernetes 클러스터】                                             │
│  ──────────────────                                                  │
│  # 노드 상태                                                        │
│  $ kubectl get nodes                                              │
│  $ kubectl describe node <node>                                   │
│                                                                     │
│  # 파드 상태                                                        │
│  $ kubectl get pods -o wide                                       │
│  $ kubectl get services                                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 클러스터: 여러 노드가 하나의 시스템처럼 동작

• HA: Active-Active, Active-Passive
• 구성요소: 노드, 하트비트, Quorum, VIP, Fence
• 소프트웨어: Pacemaker, HAProxy, Kubernetes
• 파일시스템: GFS2, OCFS2, Ceph
```
