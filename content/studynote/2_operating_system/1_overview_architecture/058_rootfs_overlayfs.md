# 058. 루트 파일시스템 / 오버레이 파일시스템
#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컨테이너의 기본 파일 계층과 변경 사항을 레이어링하는 기술
> 2. **가치**: 이미지 크기 절감, 빠른 배포, 레이어 재사용
> 3. **융합**: UnionFS, Copy-on-Write, Docker와 연관
---

## Ⅰ. 개요
### 개념 정의
루트 파일시스템(Root Filesystem)은 **컨테이너의 기본 파일 계층 구조**이며, 오버레이 파일시스템(OverlayFS)은 **여러 파일시스템을 레이어로 겹쳐 통합 뷰를 제공하는 기술**이다. 컨테이너 이미지의 효율적인 저장과 배포를 가능하게 한다.
### 💡 비유: 투명지 올리기
오버레이 파일시스템은 **투명지 올리기**와 같다. 기본 문서(베이스) 위에 투명지(오버레이)를 올려 내용을 추가하거나 수정한다. 아래 문서는 그대로 두고, 변경분만 위에 표시된다.
### 루트/오버레이 파일시스템 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                루트/오버레이 파일시스템 구조                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【오버레이 파일시스템 구조】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Upper (Upperdir)                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │  읽기/쓰기 (컨테이너 수정 사항)                            │   │ │
│  │  │  • 새 파일 생성                                          │   │ │
│  │  │  • 기존 파일 수정                                        │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                           │                                   │                  │ │
│  │                           ▼ (Merge)                            ▼                  │ │
│  │  ┌─────────────────────────────────────────────────────────────┐   │ │
│  │  │  Merged View (통합 뷰)                                   │   │ │
│  │  │  • Upper + Lower 통합                                 │   │ │
│  │  │  • 컨테이너에 노출                                    │   │ │
│  │  └─────────────────────────────────────────────────────────────┘   │ │
│  │                           │                                   │                  │ │
│  │                           ▼                                   ▼                  │ │
│  │  ┌─────────────────────────────────────────────────────────────┐   │ │
│  │  │  Lower (Lowerdir)                                          │   │ │
│  │  │  ┌─────────────────────────────────────────────────────────┐   │ │ │
│  │  │  │  Base Layer (이미지 읽기 전용)                       │   │ │ │
│  │  │  │  • OS 파일, 라이브러리, 앱 바이너리                 │   │ │ │
│  │  │  └─────────────────────────────────────────────────────────┘   │ │ │
│  │  │  • 변경 불가능                                        │   │ │
│  │  │  • 여러 컨테이너에서 공유                            │   │ │
│  │  └─────────────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【OverlayFS 특징】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • Copy-on-Write: 수정 시 상위 레이어에 복사본 생성        │ │
│  │  • Whiteout: 삭제 시 상위 레이어에 whiteout 마커               │ │
│  │  • 성능: 페이지 단위 병합                                   │ │
│  │  • 스토리지 절약: 베이스 이미지 재사용                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
