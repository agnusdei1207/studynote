# 020. 커널 (Kernel)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 운영체제의 핵심, 하드웨어와 소프트웨어 간 중개자
> 2. **가치**: 자원 관리, 추상화, 보호, 서비스 제공
> 3. **융합**: 모놀리식, 마이크로, 하이브리드 커널과 연관

---

## Ⅰ. 개요

### 개념 정의

커널(Kernel)은 **운영체제의 핵심 구성요소로, 하드웨어 자원을 관리하고 응용 프로그램에 서비스를 제공하는 특권 모드에서 실행되는 소프트웨어**다.

### 💡 비유: 국가의 정부
커널은 **국가의 정부**와 같다. 시민(애플리케이션)이 자원(도로, 전기, 물)을 사용하려면 정부(커널)의 허가가 필요하다. 정부는 모든 자원을 관리하고, 시민 간 분쟁을 해결하고, 규칙을 집행한다.

## Ⅳ. 실무 적용

### 커널 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                커널 분석                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【커널 정보 확인 (Linux)】                                            │
│  ──────────────────                                                  │
│  # 커널 버전                                                         │
│  $ uname -r                                                         │
│  $ cat /proc/version                                                │
│                                                                     │
│  # 커널 설정                                                         │
│  $ cat /boot/config-$(uname -r)                                     │
│  $ zcat /proc/config.gz                                             │
│                                                                     │
│  # 커널 모듈                                                         │
│  $ lsmod                                                            │
│  $ modinfo <module>                                                 │
│                                                                     │
│  【커널 파라미터 튜닝】                                                │
│  ──────────────────                                                  │
│  # 현재 값 확인                                                      │
│  $ sysctl -a                                                        │
│                                                                     │
│  # 값 변경                                                          │
│  $ sysctl -w kernel.hostname=myhost                                 │
│                                                                     │
│  # 영구 설정                                                        │
│  $ echo "kernel.hostname=myhost" >> /etc/sysctl.conf                │
│                                                                     │
│  【커널 개발 도구】                                                    │
│  ──────────────────                                                  │
│  1. 컴파일: make, gcc                                               │
│  2. 디버깅: kgdb, kdump, ftrace                                     │
│  3. 추적: eBPF, perf, strace                                        │
│  4. 문서: kernel.org, Documentation/                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 커널: OS 핵심, 커널 모드에서 실행
• 기능: 프로세스, 메모리, 파일, 장치, 네트워크, 보안
• 모놀리식: 성능 우수, 크기 큼, Linux
• 마이크로: 신뢰성, 확장성, IPC 오버헤드, Mach
• 하이브리드: 균형, Windows, macOS
• 특징: 특권 모드, 직접 하드웨어 접근
```
