# 015. ABI (Application Binary Interface)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컴파일된 바이너리 코드 간의 인터페이스 규약
> 2. **가치**: 바이너리 호환성, 라이브러리 교체 가능성
> 3. **융합**: 호출 규약, 이름 맹글링, 데이터 정렬과 연관

---

## Ⅰ. 개요

### 개념 정의

**ABI (Application Binary Interface)**는 컴파일된 바이너리 코드가 서로 상호작용하는 방식을 정의하는 규약으로, 함수 호출 규약, 데이터 정렬, 이름 맹글링 등을 포함한다.

### 💡 비유: 자동차 부품 규격
ABI는 **자동차 부품 규격**과 같다. 엔진(라이브러리)을 교체할 때 나사 간격, 볼트 크기(ABI)가 같아야 장착된다. API는 "시동 걸기" 버튼(기능), ABI는 "볼트 구멍 위치"(물리 규격)이다.

### ABI 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                ABI 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【API vs ABI】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  구분              API                     ABI                  │ │
│  │  ────              ────                     ────                 │ │
│  │  수준              소스 코드                바이너리             │ │
│  │  대상              프로그래머               컴파일러/링커        │ │
│  │  호환성            소스 호환성              바이너리 호환성      │ │
│  │  변경 영향         재컴파일 필요            재컴파일 불필요      │ │
│  │  예시              함수 원형                호출 규약            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ABI 구성 요소】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  요소              설명                                        │ │
│  │  ────              ────                                        │ │
│  │  호출 규약          인자 전달 방식 (레지스터/스택)               │ │
│  │  (Calling Conv.)                                                │ │
│  │                                                               │ │
│  │  데이터 정렬       구조체 패딩, 정렬 요구사항                    │ │
│  │  (Data Alignment)                                               │ │
│  │                                                               │ │
│  │  이름 맹글링       함수 이름 변환 규칙                          │ │
│  │  (Name Mangling)                                                │ │
│  │                                                               │ │
│  │  타입 크기         int, long, 포인터 크기                       │ │
│  │  (Type Sizes)                                                   │ │
│  │                                                               │ │
│  │  레지스터 사용     호출자/피호출자 저장 레지스터                 │ │
│  │  (Register Usage)                                               │ │
│  │                                                               │ │
│  │  예외 처리         예외 전파 방식                               │ │
│  │  (Exception Handling)                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. ABI 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ABI 상세 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 호출 규약 (x86_64)】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  규약              OS/컴파일러        인자 전달                 │ │
│  │  ────              ────────────        ────────                  │ │
│  │  System V AMD64   Linux, Unix       RDI, RSI, RDX, RCX, R8, R9│ │
│  │  Microsoft x64    Windows           RCX, RDX, R8, R9          │ │
│  │  cdecl            32bit x86         스택                        │ │
│  │  stdcall          Windows API       스택 (피호출자 정리)        │ │
│  │  fastcall         최적화            레지스터 + 스택             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【System V AMD64 호출 규약 상세】                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  // 함수: int foo(int a, int b, int c, int d, int e, int f)  │ │
│  │  //                                          RDI RSI RDX RCX  │ │
│  │  //                                               R8  R9  스택  │ │
│  │                                                               │ │
│  │  정수 인자: RDI, RSI, RDX, RCX, R8, R9 (순서)                  │ │
│  │  부동소수점: XMM0-XMM7                                         │ │
│  │  반환값: RAX (정수), XMM0 (부동소수점)                          │ │
│  │  호출자 저장: RAX, RCX, RDX, RSI, RDI, R8-R11                  │ │
│  │  피호출자 저장: RBX, RBP, R12-R15                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【이름 맹글링 (Name Mangling)】                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  언어              원본 함수              맹글링 결과            │ │
│  │  ────              ────                     ────                 │ │
│  │  C                foo()                  foo                   │ │
│  │  C++              foo(int)               _Z3fooi               │ │
│  │  C++              foo(double)            _Z3food               │ │
│  │  C++              Namespace::func()      _ZN9Namespace4funcEv │ │
│  │                                                               │ │
│  │  extern "C"로 C 스타일 맹글링 사용 가능                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. ABI × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ABI × 타 과목 융합                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ABI 호환성 문제】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  상황              문제                    해결책               │ │
│  │  ────              ────                    ────                  │ │
│  │  컴파일러 변경     맹글링 차이             동일 컴파일러 사용    │ │
│  │  OS 변경          호출 규약 차이          OS별 조건부 컴파일    │ │
│  │  아키텍처 변경     레지스터/정렬 차이      재컴파일              │ │
│  │  라이브러리 버전   심볼 변경              버전 관리 (soname)    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【컴퓨터 구조】                                                      │
│  • x86_64: System V AMD64 ABI                                      │
│  • ARM64: AAPCS64 (ARM Architecture Procedure Call Standard)       │
│  • RISC-V: RISC-V ABI                                              │
│  • ILP32 vs LP64: 데이터 모델 차이                                   │
│                                                                     │
│  【동적 링킹】                                                        │
│  • 심볼 해석: 런타임에 함수 주소 찾기                                  │
│  • PLT/GOT: Procedure Linkage Table, Global Offset Table           │
│  • soname: 라이브러리 버전 관리                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터 모델 (Data Models)                                     │ │
│  │  ───────────────────────                                       │ │
│  │                                                               │ │
│  │  모델      int    long    pointer    예시                       │ │
│  │  ────      ────    ────    ──────     ────                       │ │
│  │  ILP32     32     32      32         32bit 시스템              │ │
│  │  LP64      32     64      64         64bit Unix/Linux          │ │
│  │  LLP64     32     32      64         64bit Windows             │ │
│  │                                                               │ │
│  │  주의: long 크기가 OS마다 다름!                                 │ │
│  │  Linux: long = 8바이트, Windows: long = 4바이트                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### ABI 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ABI 분석                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【심볼 확인】                                                        │
│  ──────────────────                                                  │
│  # 심볼 테이블 확인                                                  │
│  $ nm -C library.so                                                │
│                                                                     │
│  # 내보내기 심볼 확인                                                │
│  $ objdump -T library.so                                           │
│                                                                     │
│  # 동적 의존성 확인                                                  │
│  $ ldd executable                                                   │
│                                                                     │
│  # ABI 정보 확인                                                    │
│  $ readelf -h executable | grep Class                              │
│                                                                     │
│  【호환성 유지 팁】                                                   │
│  ──────────────────                                                  │
│  1. PIMPL (Pointer to Implementation) 패턴 사용                     │
│  2. 가상 함수만 사용 (구현 숨김)                                      │
│  3. extern "C"로 C 인터페이스 제공                                   │
│  4. ABI 안정적인 타입만 사용                                          │
│  5. 버전 관리 (symbol versioning)                                   │
│                                                                     │
│  【C++ ABI 주의사항】                                                 │
│  ──────────────────                                                  │
│  • 가상 함수 테이블 구조 변경 금지                                     │
│  • 인라인 함수 변경 금지                                              │
│  • 템플릿 인스턴스화 호환성                                           │
│  • STL 컨테이너 ABI 호환성                                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• ABI: 바이너리 코드 간 인터페이스 규약
• 구성: 호출 규약, 정렬, 맹글링, 타입 크기
• API vs ABI: 소스 호환성 vs 바이너리 호환성
• 호출 규약: System V AMD64 (Linux), Microsoft x64 (Windows)
• 데이터 모델: LP64 (Linux), LLP64 (Windows)
• 유지: PIMPL, extern "C", 심볼 버저닝
```
