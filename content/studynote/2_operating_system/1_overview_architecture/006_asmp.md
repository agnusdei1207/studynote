# 006. 비대칭 다중 처리 (ASMP, Asymmetric Multiprocessing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 마스터 CPU가 OS를 실행하고 슬레이브 CPU는 I/O 등 특정 작업만 수행
> 2. **가치**: 구현 단순, 마스터 CPU 부하 집중, 슬레이브 CPU 활용도 낮음
> 3. **융합**: 초기 멀티프로세서, I/O 프로세서, SMP로 진화

---

## Ⅰ. 개요

### 개념 정의

비대칭 다중 처리(ASMP)는 **하나의 마스터 CPU가 운영체제와 모든 시스템 제어를 담당하고, 나머지 슬레이브 CPU들은 마스터의 지시에 따라 특정 작업만 수행하는 다중 처리 구조**다.

### 💡 비유: 식당 주방
ASMP는 **주방장과 조수들**과 같다. 주방장(마스터 CPU)이 모든 주문을 받고 메뉴를 결정한다. 조수들(슬레이브 CPU)은 주방장의 지시에 따라 야채 썰기나 설거지만 담당한다. 조수들은 독자적으로 메뉴를 결정할 수 없다.

## Ⅱ. ASMP 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ASMP 상세 분석                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 구현 단순                                                  │ │
│  │     • 단일 OS 인스턴스                                         │ │
│  │     • 동기화 복잡도 낮음                                        │ │
│  │     • 기존 OS 수정 최소화                                       │ │
│  │                                                               │ │
│  │  2. 비용 효율                                                   │ │
│  │     • 저렴한 하드웨어                                           │ │
│  │     • 단순한 소프트웨어                                         │ │
│  │                                                               │ │
│  │  3. 디버깅 용이                                                  │ │
│  │     • 중앙 집중 제어                                            │ │
│  │     • 단일 실행 경로                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 마스터 병목                                                 │ │
│  │     • 모든 OS 작업이 마스터에 집중                               │ │
│  │     • 슬레이브 대기 시간 발생                                    │ │
│  │                                                               │ │
│  │  2. 확장성 제한                                                  │ │
│  │     • CPU 추가 시 마스터 부하 증가                               │ │
│  │     • 성능 향상 한계                                            │ │
│  │                                                               │ │
│  │  3. 신뢰성 문제                                                  │ │
│  │     • 마스터 장애 = 시스템 전체 장애                             │ │
│  │     • 단일 실패점 (SPOF)                                        │ │
│  │                                                               │ │
│  │  4. 슬레이브 활용도 저하                                         │ │
│  │     • 대기 시간 많음                                            │ │
│  │     • 자원 낭비                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ASMP → SMP 진화 이유】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ASMP 문제               SMP 해결                              │ │
│  │  ──────────               ──────────                           │ │
│  │  마스터 병목             모든 CPU 동등한 권한                    │ │
│  │  확장성 제한             선형적 확장 가능                        │ │
│  │  신뢰성 취약             장애 허용력 향상                        │ │
│  │  비효율                  균등한 부하 분산                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ASMP × 타 과목 융합                                    │

│  • 1990년대 이후: SMP로 대체                                          │

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• ASMP: 마스터 CPU가 OS 실행, 슬레이브는 지시 수행
• 장점: 구현 단순, 비용 저렴, 디버깅 용이
• 단점: 마스터 병목, 확장성 제한, 신뢰성 취약
• 진화: SMP로 대체, 특수 목적에만 활용
• 현대: I/O 프로세서, GPU, DSP 등에 활용
• 교훈: 확장성과 신뢰성을 위해 SMP 선호
```
