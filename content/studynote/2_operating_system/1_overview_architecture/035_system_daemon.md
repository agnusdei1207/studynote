# 035. 시스템 데몬 (System Daemon)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 백그라운드에서 실행되는 시스템 서비스 프로세스
> 2. **가치**: 시스템 관리, 서비스 제공, 지속적 운영
> 3. **융합**: init, systemd, cron, syslog와 연관

---

## Ⅰ. 개요

### 개념 정의

시스템 데몬(System Daemon)은 **백그라운드에서 실행되어 시스템 서비스를 제공하는 프로세스**로, 사용자와 직접 상호작용하지 않고 지속적으로 실행된다. 유닉스에서는 프로세스 이름 끝에 'd'를 붙여 데몬임을 표시한다(sshd, httpd, crond 등).

### 💡 비유: 건물 관리 직원
데몬은 **건물의 관리 직원**과 같다. 경비원(sshd), 청소부(cron), 기계실 담당자(syslogd)가 24시간 건물(시스템)을 관리한다. 입주자(사용자)가 직접 보지 못해도 건물은 원활하게 돌아간다.

## Ⅱ. 시스템 데몬 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 데몬 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【데몬 특징】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  특징              설명                                        │ │
│  │  ────              ────                                        │ │
│  │  백그라운드        터미널 없이 실행                             │ │
│  │  제어 터미널 없음   터미널에서 분리                             │ │
│  │  PPID = 1         init의 자식                                  │ │
│  │  무한 루프         지속적 실행                                  │ │
│  │  이벤트 대기       요청 시 응답                                 │ │
│  │  로그 기록         /var/log에 기록                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【데몬화 과정 (Daemonization)】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. fork() 실행                                                │ │
│  │     • 부모 프로세스 종료                                        │ │
│  │     • 자식이 백그라운드로 실행                                  │ │
│  │                                                               │ │
│  │  2. setsid() 실행                                              │ │
│  │     • 새 세션 생성                                              │ │
│  │     • 제어 터미널 분리                                          │ │
│  │                                                               │ │
│  │  3. chdir("/")                                                 │ │
│  │     • 루트 디렉터리로 이동                                      │ │
│  │                                                               │ │
│  │  4. umask(0)                                                   │ │
│  │     • 파일 생성 마스크 초기화                                   │ │
│  │                                                               │ │
│  │  5. 파일 디스크립터 닫기                                        │ │
│  │     • stdin, stdout, stderr 닫기                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 데몬 목록】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  데몬              기능                    포트/파일            │ │
│  │  ────              ────                    ────                │ │
│  │  sshd             원격 접속               22/tcp              │ │
│  │  httpd/apache2    웹 서버                 80, 443/tcp         │ │
│  │  nginx            웹 서버/프록시           80, 443/tcp         │ │
│  │  cron/crond       작업 스케줄러           -                   │ │
│  │  rsyslogd         시스템 로그             /var/log/           │ │
│  │  systemd-journald 저널 로그               /run/log/journal/   │ │
│  │  dbus-daemon      IPC 메시지              /var/run/dbus/      │ │
│  │  NetworkManager   네트워크 관리           D-Bus              │ │
│  │  systemd-resolved DNS 해석               53/udp, tcp         │ │
│  │  ntpd/chronyd     시간 동기화             123/udp             │ │
│  │  cupsd            인쇄 서버               631/tcp             │ │
│  │  mysqld           데이터베이스            3306/tcp            │ │
│  │  postgresql       데이터베이스            5432/tcp            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 시스템 데몬 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                시스템 데몬 분석                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【실행 중인 데몬 확인】                                              │
│  ──────────────────                                                  │
│  # systemd 서비스 목록                                               │
│  $ systemctl list-units --type=service                           │
│                                                                     │
│  # 프로세스 목록에서 데몬                                            │
│  $ ps aux | grep -E '\wd$'   # 이름이 d로 끝나는 프로세스           │
│  $ ps -eo pid,ppid,cmd | grep ']$'   # 대괄호 없는 프로세스        │
│                                                                     │
│  # 열린 포트와 데몬                                                  │
│  $ ss -tulnp                                                     │
│  $ netstat -tulnp                                                │
│                                                                     │
│  【데몬 로그 확인】                                                  │
│  ──────────────────                                                  │
│  # systemd 저널                                                    │
│  $ journalctl -u sshd                                            │
│  $ journalctl -u nginx                                           │
│                                                                     │
│  # 전통적 로그                                                      │
│  $ tail -f /var/log/syslog                                       │
│  $ tail -f /var/log/apache2/access.log                          │
│                                                                     │
│  【데몬 관리】                                                      │
│  ──────────────────                                                  │
│  # 서비스 시작/중지/재시작                                           │
│  $ sudo systemctl start sshd                                     │
│  $ sudo systemctl stop nginx                                     │
│  $ sudo systemctl restart mysql                                  │
│                                                                     │
│  # 서비스 상태                                                       │
│  $ systemctl status sshd                                         │
│  $ systemctl is-active sshd                                      │
│  $ systemctl is-enabled sshd                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 데몬: 백그라운드 시스템 서비스 프로세스
• 특징: 제어 터미널 없음, PPID=1, 지속 실행
• 이름: 끝이 'd' (sshd, httpd, crond)
• 관리: systemctl (systemd), service (SysVinit)
• 로그: journald, /var/log/
• 생성: fork → setsid → chdir → umask → close fd
```
