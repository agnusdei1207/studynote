# 508. 인터럽트 핸들러 (Interrupt Handler)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 발생 시 실행되는 처리 함수
> 2. **가치**: 이벤트에 대한 신속하고 안전한 대응
> 3. **융합**: 인터럽트, 컨텍스트 스위치, 지연 작업과 연관

---

## Ⅰ. 개요

### 개념 정의

인터럽트 핸들러(Interrupt Handler) 또는 ISR(Interrupt Service Routine)은 **인터럽트 발생 시 실행되어 해당 이벤트를 처리하는 커널 함수**이다.

### 💡 비유: 비상 대응팀
인터럽트 핸들러는 **비상 대응팀**과 같다. 비상벨이 울리면 즉시 출동해서 상황을 처리한다.

### 인터럽트 핸들러 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 핸들러 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핸들러 실행 과정】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Top Half (인터럽트 핸들러):                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  1. 하드웨어 인터럽트 비활성화                              │    │ │
│  │  │  2. 컨텍스트 저장                                          │    │ │
│  │  │  3. 인터럽트 원인 확인                                      │    │ │
│  │  │  4. 최소한의 필수 작업만 수행                               │    │ │
│  │  │     • 장치 상태 읽기                                        │    │ │
│  │  │     • 버퍼에서 데이터 복사                                  │    │ │
│  │  │     • ACK 전송                                              │    │ │
│  │  │  5. Bottom Half 스케줄링                                   │    │ │
│  │  │  6. 컨텍스트 복원                                           │    │ │
│  │  │  7. IRET (복귀)                                            │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  Bottom Half (지연된 작업):                                             │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  • SoftIRQ                                               │    │ │
│  │  │  • Tasklet                                               │    │ │
│  │  │  • Workqueue                                             │    │ │
│  │  │  • 시간이 걸리는 작업 수행                                  │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Top Half vs Bottom Half】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  구분        Top Half              Bottom Half                          │ │
│  │  ────        ────────              ───────────                          │ │
│  │  실행 시점    인터럽트 발생 즉시      나중에 스케줄됨                      │ │
│  │  실행 컨텍스트 인터럽트 컨텍스트      프로세스 컨텍스트                    │ │
│  │  인터럽트     비활성화됨              활성화 가능                        │ │
│  │  수행 작업    최소한의 필수 작업      시간이 걸리는 작업                  │ │
│  │  블로킹       불가능                 가능                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                인터럽트 핸들러 상세                                     │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핸들러 제약사항】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  금지 사항:                                                              │ │
│  │  • 수면 (sleep) 또는 블로킹 호출                                           │ │
│  │  • 사용자 공간 메모리 접근                                                 │ │
│  │  • 페이지 폴트 유발                                                        │ │
│  │  • 긴 연산 수행                                                          │ │
│  │  • 세마포어 획득 (뮤텍스 대신 스핀락 사용)                                  │ │
│  │  • 스케줄러 호출                                                         │ │
│  │                                                             │ │
│  │  권장 사항:                                                              │ │
│  │  • 최대한 짧게 실행                                                       │ │
│  │  • 복잡한 작업은 Bottom Half로 이동                                       │ │
│  │  • 하드웨어 ACK 빠르게 수행                                               │ │
│  │  • 공유 데이터는 적절히 보호                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Bottom Half 종류】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  방식          실행 컨텍스트    병렬성        용도                           │ │
│  │  ────          ────────────    ────        ────                           │ │
│  │  SoftIRQ       인터럽트        높음         네트워킹, 블록 I/O              │ │
│  │  Tasklet       인터럽트        낮음(직렬)    일반 목적                       │ │
│  │  Workqueue     프로세스        높음         수면 가능한 작업                │ │
│  │  Threaded IRQ  커널 스레드     높음         실시간성 중요                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【반환값】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  irqreturn_t:                                                            │ │
│  │  • IRQ_NONE: 인터럽트가 이 장치가 아님                                      │ │
│  │  • IRQ_HANDLED: 인터럽트 처리 완료                                         │ │
│  │  • IRQ_WAKE_THREAD: 스레드 핸들러 실행 요청                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 인터럽트 핸들러】                                               │
│  ──────────────────                                                  │
│  #include <linux/interrupt.h>                                        │
│  #include <linux/module.h>                                           │
│                                                                     │
│  // 인터럽트 핸들러 (Top Half)                                          │
│  irqreturn_t my_interrupt_handler(int irq, void *dev_id) {           │
│      struct my_device *dev = (struct my_device *)dev_id;             │
│                                                                     │
│      // 1. 장치 상태 확인                                               │
│      u32 status = readl(dev->mmio + REG_STATUS);                     │
│      if (!(status & IRQ_PENDING))                                    │
│          return IRQ_NONE;                                            │
│                                                                     │
│      // 2. 최소한의 작업 수행                                            │
│      writel(IRQ_ACK, dev->mmio + REG_IRQ_ACK);                       │
│                                                                     │
│      // 3. Bottom Half 스케줄링                                        │
│      tasklet_schedule(&dev->tasklet);                                │
│                                                                     │
│      return IRQ_HANDLED;                                             │
│  }                                                                   │
│                                                                     │
│  // Tasklet 핸들러 (Bottom Half)                                       │
│  void my_tasklet_func(unsigned long data) {                          │
│      struct my_device *dev = (struct my_device *)data;               │
│      // 시간이 걸리는 작업 수행                                           │
│      process_data(dev);                                              │
│  }                                                                   │
│                                                                     │
│  【인터럽트 등록/해제】                                                  │
│  ──────────────────                                                  │
│  // 등록                                                               │
│  int ret = request_irq(irq,                                          │
│                        my_interrupt_handler,                         │
│                        IRQF_SHARED,    // 공유 가능                    │
│                        "my_driver",                                  │
│                        &my_device);                                  │
│  if (ret) {                                                          │
│      dev_err(dev, "Failed to request IRQ %d\n", irq);                │
│      return ret;                                                     │
│  }                                                                   │
│                                                                     │
│  // 해제                                                               │
│  free_irq(irq, &my_device);                                          │
│                                                                     │
│  // Devm (자동 해제)                                                   │
│  ret = devm_request_irq(dev, irq, handler, 0, "name", dev_id);       │
│                                                                     │
│  【Workqueue 사용】                                                     │
│  ──────────────────                                                  │
│  #include <linux/workqueue.h>                                        │
│                                                                     │
│  // 정적 Workqueue                                                     │
│  DECLARE_WORK(my_work, work_handler);                                │
│                                                                     │
│  // 동적 Workqueue                                                     │
│  struct work_struct my_work;                                         │
│  INIT_WORK(&my_work, work_handler);                                  │
│                                                                     │
│  // 인터럽트 핸들러에서 스케줄                                            │
│  schedule_work(&my_work);                                            │
│                                                                     │
│  // 지연된 Workqueue                                                   │
│  DECLARE_DELAYED_WORK(delayed_work, delayed_handler);                │
│  schedule_delayed_work(&delayed_work, msecs_to_jiffies(100));        │
│                                                                     │
│  // Workqueue 핸들러                                                    │
│  void work_handler(struct work_struct *work) {                       │
│      // 수면 가능, 블로킹 가능                                           │
│      msleep(100);                                                    │
│      mutex_lock(&my_mutex);                                          │
│      // ... 작업 수행                                                  │
│      mutex_unlock(&my_mutex);                                        │
│  }                                                                   │
│                                                                     │
│  【Threaded IRQ】                                                      │
│  ──────────────────                                                  │
│  // Threaded IRQ 등록                                                  │
│  int ret = request_threaded_irq(irq,                                 │
│                                  my_hard_handler,   // Top Half       │
│                                  my_thread_fn,      // Bottom Half    │
│                                  IRQF_ONESHOT,                        │
│                                  "my_driver",                         │
│                                  &my_device);                         │
│                                                                     │
│  // 스레드 함수 (수면 가능)                                              │
│  irqreturn_t my_thread_fn(int irq, void *dev_id) {                   │
│      // 시간이 걸리는 작업                                                │
│      msleep(10);                                                     │
│      return IRQ_HANDLED;                                             │
│  }                                                                   │
│                                                                     │
│  【인터럽트 통계 확인】                                                  │
│  ──────────────────                                                  │
│  // 핸들러 실행 시간 측정                                                │
│  $ cat /proc/interrupts                                              │
│  $ cat /proc/softirqs                                                │
│                                                                     │
│  // ftrace로 추적                                                      │
│  $ echo 1 > /sys/kernel/debug/tracing/events/irq/enable             │
│  $ cat /sys/kernel/debug/tracing/trace                              │
│                                                                     │
│  // perf로 분석                                                        │
│  $ perf record -e irq:* -a sleep 10                                 │
│  $ perf report                                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 인터럽트 발생 시 실행되는 처리 함수
• 구조: Top Half (필수) + Bottom Half (지연)
• 제약: 수면/블로킹 금지, 빠른 실행
• Bottom Half: SoftIRQ, Tasklet, Workqueue
• 반환값: IRQ_NONE, IRQ_HANDLED, IRQ_WAKE_THREAD
• 등록: request_irq, request_threaded_irq
• 해제: free_irq
• 최적화: 짧은 Top Half, 긴 작업은 Bottom Half
```
