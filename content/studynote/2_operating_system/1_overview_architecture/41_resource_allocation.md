# 311. 자원 할당 (Resource Allocation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자원을 프로세스에 배정하는 행위
> 2. **가치**: 생산자의 작업 진행을 의미
> 3. **융합**: 자원 요청, 자원 해제와 연관

---

## Ⅰ. 개요

### 개념 정의

자원 할당(Resource Allocation)은 **요청된 자원을 특정 프로세스에 배정하는 행위**다. 할당이 일어나면 자원은 해당 프로세스에 귀속된다.

### 💡 비유: 도서관 대출 완료
자원 할당은 **도서관 대출 완료**와 같다. 신청이 승인되어 책을 실제로 빌린 상태다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                자원 할당 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【할당 정책】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. FIFO (선입선출):                                                 │ │
│  │     • 대기 순서대로 할당                                                │ │
│  │                                                             │ │
│  │  2. 우선순위 기반:                                                     │ │
│  │     • 높은 우선순위 프로세스 먼저                                         │ │
│  │                                                             │ │
│  │  3. 최소 요청 기반:                                                    │ │
│  │     • 가장 적게 요청한 프로세스 먼저                                      │ │
│  │                                                             │ │
│  │  4. 랜덤:                                                             │ │
│  │     • 무작위 선택                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【할당과 교착상태】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  할당은 생산자의 작업 진행을 의미:                                        │ │
│  │                                                             │ │
│  │  그러나 부적절한 할당은 교착상태로 이어질 수 있음:                         │ │
│  │  • P1이 R1 할당, R2 요청                                              │ │
│  │  • P2이 R2 할당, R1 요청                                              │ │
│  │  → 교착상태                                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【할당 테이블】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │           R1    R2    R3    ...                                     │ │
│  │  P1       2     0     1                                             │ │
│  │  P2       0     1     0                                             │ │
│  │  P3       1     2     0                                             │ │
│  │  ...                                                                │ │
│  │  Allocation[i][j] = 프로세스 i에 할당된 자원 j의 개수                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【할당 시간과 보유 시간】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  할당 시간: 자원이 프로세스에 배정된 시점                                  │ │
│  │  보유 시간: 현재까지 자원을 보유한 시간                                    │ │
│  │                                                             │ │
│  │  긴 보유 시간 = 잠재적 교착상태 위험                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【할당 관리 구조】                                                      │
│  ──────────────────                                                  │
│  class AllocationManager {                                          │
│      int[][] allocation;     // [프로세스][자원]                        │
│      int[] available;         // 각 자원의 가용 개수                     │
│                                                                     │
│      boolean allocate(int pid, int resourceType, int count) {       │
│          // 가용 자원 확인                                               │
│          if (available[resourceType] < count) {                       │
│              return false;                                            │
│          }                                                          │
│                                                                     │
│          // 할당                                                       │
│          available[resourceType] -= count;                            │
│          allocation[pid][resourceType] += count;                     │
│          return true;                                                 │
│      }                                                              │
│                                                                     │
│      void release(int pid, int resourceType, int count) {            │
│          allocation[pid][resourceType] -= count;                     │
│          available[resourceType] += count;                            │
│          // 대기자 확인 및 할당                                           │
│          checkWaiters(resourceType);                                 │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【락 할당】                                                             │
│  ──────────────────                                                  │
│  // pthread mutex                                                    │
│  pthread_mutex_t mutex;                                              │
│  pthread_mutex_lock(&mutex);    // 할당 요청                            │
│  // ... 할당됨 ...                                                     │
│  pthread_mutex_unlock(&mutex);  // 해제                                │
│                                                                     │
│  // Java synchronized                                                │
│  synchronized(obj) {                                                 │
│      // obj의 모니터 락 할당됨                                           │
│  }                                                                  │
│                                                                     │
│  // ReentrantLock                                                    │
│  lock.lock();      // 할당 요청                                         │
│  lock.unlock();    // 해제                                              │
│                                                                     │
│  【메모리 할당】                                                         │
│  ──────────────────                                                  │
│  // malloc (C)                                                       │
│  void* p = malloc(1024);    // 1KB 할당 요청                           │
│  free(p);                    // 해제                                    │
│                                                                     │
│  // new (C++)                                                        │
│  int* p = new int[256];      // 256 int 할당                          │
│  delete[] p;                  // 해제                                   │
│                                                                     │
│  // Java                                                             │
│  int[] arr = new int[256];   // 힙 할당                                │
│  arr = null;                  // GC가 해제                              │
│                                                                     │
│  【파일 할당】                                                           │
│  ──────────────────                                                  │
│  // 파일 열기 (파일 핸들 할당)                                           │
│  int fd = open("file.txt", O_RDWR);                                 │
│  close(fd);        // 파일 핸들 해제                                     │
│                                                                     │
│  // 파일 레코드 락                                                      │
│  struct flock fl;                                                     │
│  fcntl(fd, F_SETLKW, &fl);  // 레코드 락 할당                           │
│  fl.l_type = F_UNLCK;                                                │
│  fcntl(fd, F_SETLK, &fl);   // 레코드 락 해제                            │
│                                                                     │
│  【데이터베이스 연결 할당】                                                │
│  ──────────────────                                                  │
│  // 커넥션 풀에서 할당                                                    │
│  Connection conn = dataSource.getConnection();                       │
│  // ... 사용 ...                                                      │
│  conn.close();      // 풀로 반환 (해제)                                  │
│                                                                     │
│  【할당 상태 확인】                                                       │
│  ──────────────────                                                  │
│  // Linux: /proc로 확인                                                │
│  cat /proc/<pid>/fd        // 열린 파일                                │
│  cat /proc/<pid>/maps      // 메모리 맵                                 │
│  cat /proc/locks           // 파일 락                                   │
│                                                                     │
│  // Java: JMX로 확인                                                   │
│  ThreadMXBean threadBean = ManagementFactory.getThreadMXBean();     │
│  ThreadInfo info = threadBean.getThreadInfo(threadId);              │
│  String lockName = info.getLockName();  // 보유 중인 락                 │
│                                                                     │
│  【할당 통계】                                                            │
│  ──────────────────                                                  │
│  class AllocationStats {                                            │
│      Map<Integer, Long> allocationTime = new HashMap<>();            │
│                                                                     │
│      void recordAllocation(int pid, int resource) {                  │
│          allocationTime.put(pid * 1000 + resource,                  │
│                            System.currentTimeMillis());              │
│      }                                                              │
│                                                                     │
│      long getHoldTime(int pid, int resource) {                       │
│          Long start = allocationTime.get(pid * 1000 + resource);    │
│          return start != null ? System.currentTimeMillis() - start : 0;│
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【할당 로깅】                                                            │
│  ──────────────────                                                  │
│  void logAllocation(int pid, int resource, int count) {             │
│      log.info("ALLOCATE: P{} <- R{} x {} @ {}",                     │
│               pid, resource, count, System.currentTimeMillis());    │
│  }                                                                  │
│                                                                     │
│  void logRelease(int pid, int resource, int count) {                │
│      log.info("RELEASE: P{} -> R{} x {} @ {}",                      │
│               pid, resource, count, System.currentTimeMillis());    │
│  }                                                                  │
│                                                                     │
│  // 로그 분석으로 교착상태 패턴 식별                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 자원을 프로세스에 배정
• 상태: 가용 → 할당됨
• 정보: 프로세스 ID, 자원 유형, 시간
• 정책: FIFO, 우선순위, 랜덤
• 위험: 부적절한 할당 → 교착상태
• 관리: 할당 테이블, 로깅, 통계
```
