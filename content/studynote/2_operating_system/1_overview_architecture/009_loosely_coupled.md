# 009. 약결합 시스템 (Loosely Coupled System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 독립적인 노드들이 메시지 패싱을 통해 통신하는 분산 시스템
> 2. **가치**: 높은 확장성, 장애 격리, 지리적 분산 가능
> 3. **융합**: 클러스터, 그리드, 클라우드, 마이크로서비스와 연관

---

## Ⅰ. 개요

### 개념 정의

약결합 시스템(Loosely Coupled System)은 **독립적인 프로세서(노드)들이 각자의 메모리를 가지고, 네트워크를 통한 메시지 패싱으로 통신하는 분산 시스템**이다.

### 💡 비유: 여러 식당의 요리사들
약결합 시스템은 **각자 다른 식당에서 일하는 요리사들**과 같다. 각 식당(노드)은 자기만의 냉장고(로컬 메모리)가 있다. 요리사들 간 통신은 전화나 메신저(메시지 패싱)로 한다. 한 식당이 문제생겨도 다른 식당은 영업 가능하다.

## Ⅲ. 약결합 × 타 과목 융합

### 과목 융합 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                약결합 × 타 과목 융합                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【네트워크】                                                        │
│  • 프로토콜: TCP/IP, UDP, RDMA                                      │
│  • 토폴로지: 스타, 메시, 트리                                        │
│  • 스위칭: 패킷 스위칭, 회선 교환                                    │
│  • 지연: 대역폭, 지연 시간, 지터                                     │
│                                                                     │
│  【분산 시스템】                                                      │
│  • CAP 정리: 일관성, 가용성, 분할 내성                                │
│  • 합의: Paxos, Raft                                                │
│  • 분산 락: ZooKeeper, etcd                                         │
│  • 복제: 주-부, 다중 마스터                                          │
│                                                                     │
│  【클라우드】                                                        │
│  • 가상화: VM, 컨테이너                                              │
│  • 오케스트레이션: Kubernetes                                        │
│  • 서비스: IaaS, PaaS, SaaS                                         │
│  • 마이크로서비스: 분산 애플리케이션                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CAP 정리 (Brewer's Theorem)                                  │ │
│  │  ───────────────────────────                                  │ │
│  │                                                               │ │
│  │  분산 시스템에서 세 가지 중 두 가지만 보장 가능:                  │ │
│  │                                                               │ │
│  │  C (Consistency): 모든 노드가 동시에 같은 데이터 조회            │ │
│  │  A (Availability): 모든 요청이 응답을 받음                       │ │
│  │  P (Partition Tolerance): 네트워크 분할 시에도 동작              │ │
│  │                                                               │ │
│  │  ┌────────────┐                                               │ │
│  │  │    CA     │  네트워크 분할 불가 → 실제로 불가능              │ │
│  │  │ (RDBMS)   │                                               │ │
│  │  └────────────┘                                               │ │
│  │  ┌────────────┐  ┌────────────┐                               │ │
│  │  │    CP     │  │    AP     │                                 │ │
│  │  │ (HBase,   │  │ (Cassandra│                                 │ │
│  │  │  MongoDB) │  │  DynamoDB)│                                 │ │
│  │  └────────────┘  └────────────┘                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 약결합: 독립 노드가 메시지 패싱으로 통신
• 장점: 확장성, 장애 격리, 지리적 분산, 비용 효율
• 단점: 통신 지연, 프로그래밍 복잡, 부분 실패
• 기술: TCP/IP, RDMA, 메시지 큐, 분산 스토리지
• 패턴: CAP 정리, 사가, CQRS, 서킷 브레이커
• 적용: 클러스터, 그리드, 클라우드, CDN
```
