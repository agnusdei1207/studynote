# 다중 처리 시스템 (Multiprocessing System)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 처리 시스템(Multiprocessing System)은 한 대의 컴퓨터 안에 **두 개 이상의 CPU(프로세서)**를 장착하여 여러 개의 작업을 병렬로 처리하거나, 하나의 복잡한 작업을 나누어 처리하는 시스템이다.
> 2. **가치**: 단일 프로세서의 물리적 한계를 극복하여 **처리량(Throughput)을 획기적으로 증대**시키며, 특정 CPU 고장 시에도 다른 CPU가 업무를 대행하는 **신뢰성(Reliability)**과 결함 허용(Fault Tolerance) 능력을 제공한다.
> 3. **융합**: 현대의 멀티코어 프로세서 아키텍처의 모태가 되었으며, 이를 위해 프로세서 간의 작업 배분(Load Balancing)과 공유 메모리에 대한 동기화 제어가 운영체제의 핵심 난제로 부각되었다.

---

## Ⅰ. 다중 처리 시스템 (Multiprocessing System)의 개요

### 1. 정의
- 여러 개의 프로세서가 메모리와 입출력 장치 등 시스템 자원을 공유하면서 협력하여 프로그램을 실행하는 방식이다.

### 2. 다중 프로그래밍 vs 다중 처리 (혼동 주의)
- **다중 프로그래밍 (Multiprogramming)**: CPU는 **1개**이나, 메모리에 여러 프로그램을 올려 번갈아 실행 (소프트웨어적 기법).
- **다중 처리 (Multiprocessing)**: CPU가 **2개 이상**으로, 실제로 물리적인 병렬 실행이 일어남 (하드웨어적 기반).

### 3. 💡 비유: 식당의 주방
- **단일 처리**: 주방장 한 명이 주문을 받아 국 끓이고 생선 굽고 혼자 바쁘게 움직임.
- **다중 처리**: 주방장이 두 명 이상임. 한 명은 국을 끓이고, 동시에 다른 명은 생선을 구움. 음식이 나오는 속도가 훨씬 빠르고, 한 명이 아파도 식당 문을 닫지 않아도 됨.

---

## Ⅱ. 다중 처리 시스템의 분류 및 구조 (ASCII)

프로세서 간의 관계와 자원 공유 방식에 따라 크게 두 가지로 나뉜다.

### 1. 대칭형 다중 처리 (SMP, Symmetric Multiprocessing)
모든 프로세서가 대등한 권한을 가지며 하나의 운영체제에 의해 관리된다.
```text
    [ Shared Memory ] <──┐
           ▲             │ (System Bus)
    ┌──────┴──────┐      │
    │     CPU 1   │ <────┤
    ├─────────────┤      │
    │     CPU 2   │ <────┤ [ Operating System ]
    ├─────────────┤      │ (모든 CPU가 OS 커널을 
    │     CPU 3   │ <────┘  공유 실행 가능)
    └─────────────┘
```
- **특징**: 가장 대중적인 구조. 부하 분산이 효율적이지만 동기화 복잡도가 높다.

### 2. 비대칭형 다중 처리 (ASMP, Asymmetric Multiprocessing)
주 프로세서(Master)가 OS를 실행하고 작업을 할당하며, 종 프로세서(Slave)는 지시받은 업무만 수행한다.
```text
    [ Master CPU ]  ───────▶ [ Slave CPU 1 ] (Task X)
    (OS, Schedule)  ───────▶ [ Slave CPU 2 ] (Task Y)
```
- **특징**: 구조가 단순하고 설계가 쉽지만, Master CPU가 병목(Bottleneck)이 될 위험이 있다.

---

## Ⅲ. SMP vs ASMP 비교 분석

| 비교 항목 | SMP (대칭형) | ASMP (비대칭형) |
|:---|:---|:---|
| **프로세서 권한** | 모든 프로세서가 동등함 | 주(Master)/종(Slave) 구분 명확 |
| **운영체제 실행** | 모든 프로세서가 실행 가능 | 오직 Master 프로세서만 실행 |
| **작업 할당** | 스스로 대기 큐에서 가져옴 | Master가 Slave에게 할당함 |
| **자원 효율성** | 매우 높음 (유연함) | 보통 (Slave 유휴 발생 가능) |
| **복잡도** | 높음 (동기화 이슈) | 낮음 |

- **📢 섹션 요약 비유**: 모든 팀원이 알아서 일을 찾아서 하는 '민주적인 팀'이 SMP라면, 팀장님이 일감을 나눠주는 '수직적인 팀'이 ASMP입니다.

---

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy)

### 1. 프로세서 간 결합도 (Coupling)
- **강결합 (Tightly-coupled)**: 프로세서들이 메모리와 클럭을 공유하는 방식 (일반적인 다중 처리 시스템).
- **약결합 (Loosely-coupled)**: 독립된 메모리를 가진 컴퓨터들이 네트워크로 연결된 방식 (분산 처리 시스템, 클러스터링).
- **기술사적 인사이트**: 고성능 연산이 필요하면 **강결합**을, 시스템의 확장성과 지리적 분산이 필요하면 **약결합** 시스템을 설계하는 것이 공학적 정석이다.

### 2. 멀티코어 시대의 OS 과제
- **판단**: 이제는 물리적 CPU 칩 개수보다 하나의 칩 내에 코어가 여러 개인 멀티코어 환경이 일반적이다.
- **결단**: OS 설계자나 시스템 엔지니어는 공유 자원에 대한 **Race Condition**을 방지하기 위해 Spinlock, Mutex 등 정교한 동기화 메커니즘을 사용해야 하며, 캐시 일관성(Cache Coherency) 문제 해결에 집중해야 한다.

---

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **성능 가용성**: $N$개의 프로세서가 이론적으로 $N$배의 성능을 낼 수 있는 확장성 제공.
- **고가용성 (HA)**: 하드웨어 결함 발생 시에도 중단 없는 서비스(Graceful Degradation) 가능.

### 2. 미래 전망
최근에는 CPU뿐만 아니라 NPU, GPU 등을 혼합하여 사용하는 **이기종 다중 처리 (Heterogeneous Multiprocessing)**가 주류가 되고 있다. ARM의 big.LITTLE 아키텍처처럼 고성능 코어와 저전력 코어를 섞어 효율을 극대화하는 방식은 스마트폰부터 데이터센터까지 현대 컴퓨팅의 핵심 생존 전략이 될 것이다.

- **📢 섹션 요약 비유**: 머리가 두 개 달린 거인이 더 빨리 생각하고, 한쪽 눈을 다쳐도 다른 쪽 눈으로 세상을 볼 수 있는 것과 같은 강력한 진화입니다.

### 👶 어린이를 위한 3줄 비유 설명
1. 다중 처리 시스템은 **'머리가 여러 개 달린 로봇'**과 같아요.
2. 한쪽 머리가 수학 숙제를 하는 동안, 다른 쪽 머리는 영어 단어를 외울 수 있어서 공부를 엄청 빨리 끝내죠.
3. 한쪽 머리가 졸려도 다른 머리가 깨어있으면 로봇은 계속 움직일 수 있는 아주 튼튼하고 똑똑한 친구랍니다!
