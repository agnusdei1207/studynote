# 041. DLM (분산 락 관리자)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 환경에서 여러 노드 간 동기화를 위한 락 관리 시스템
> 2. **가치**: 동시성 제어, 데이터 일관성, 클러스터 동기화
> 3. **융합**: GFS2, OCFS2, 클러스터 파일시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

DLM(Distributed Lock Manager)은 **분산 환경에서 여러 노드가 공유 자원에 접근할 때 동시성을 제어하기 위한 락 관리 시스템**이다. 클러스터 파일시스템, 분산 데이터베이스 등에서 데이터 일관성을 보장하기 위해 사용된다.

### 💡 비유: 다중 식당의 주방 예약 시스템
DLM은 **체인 식당의 주방 예약 시스템**과 같다. 여러 지점(노드)의 요리사가 중앙 창고(공유 자원)에서 재료를 꺼내려고 한다. 재료를 꺼내기 전에 예약 시스템(DLM)에 예약(락)해야 한다. 다른 지점에서 이미 예약한 재료는 대기해야 한다.

## Ⅱ. DLM 상세

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                DLM 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【락 모드】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  모드              호환성              설명                    │ │
│  │  ────              ────                ────                    │ │
│  │  NL (Null)         모두               락 없음                  │ │
│  │  CR (Concurrent Read) NL, CR, PR, CW   동시 읽기              │ │
│  │  CW (Concurrent Write) NL, CR, CW      동시 쓰기              │ │
│  │  PR (Protected Read) NL, CR, PR        보호 읽기              │ │
│  │  PW (Protected Write) NL, CW           보호 쓰기              │ │
│  │  EX (Exclusive)    NL                  배타적                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락 호환성 매트릭스】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │           NL   CR   CW   PR   PW   EX                         │ │
│  │         ──────────────────────────────                        │ │
│  │  NL    │  Y    Y    Y    Y    Y    Y  │                       │ │
│  │  CR    │  Y    Y    Y    Y    Y    N  │                       │ │
│  │  CW    │  Y    Y    Y    N    Y    N  │                       │ │
│  │  PR    │  Y    Y    N    Y    N    N  │                       │ │
│  │  PW    │  Y    Y    Y    N    N    N  │                       │ │
│  │  EX    │  Y    N    N    N    N    N  │                       │ │
│  │                                                               │ │
│  │  Y = 호환, N = 충돌                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【DLM 구현 예시】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시스템             DLM 구현              용도                  │ │
│  │  ────               ────                  ────                  │ │
│  │  GFS2              DLM                   클러스터 FS           │ │
│  │  OCFS2             DLM                   Oracle 클러스터 FS    │ │
│  │  GPFS              DLM                   IBM 병렬 FS          │ │
│  │  Oracle RAC        DLM                   분산 DB              │ │
│  │  PostgreSQL        없음 (MVCC)           일반 DB              │ │
│  │  etcd              Raft                  분산 KV              │ │
│  │  ZooKeeper         ZAB                   분산 코디네이터       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

┌─────────────────────────────────────────────────────────────────────┐
│                DLM × 타 과목 융합                                    │

│                                                                     │
│  【동시성 제어】                                                      │

│  • 락 강등: 높은 모드 → 낮은 모드                                   │

│  │                                                               │ │

│  └───────────────────────────────────────────────────────────────┘ │

## Ⅳ. 실무 적용

### DLM 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                DLM 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【GFS2 DLM 확인】                                                  │
│  ──────────────────                                                  │
│  # DLM 모듈 확인                                                    │
│  $ lsmod | grep dlm                                               │
│                                                                     │
│  # DLM 상태                                                         │
│  $ dlm_tool ls                                                    │
│  $ dlm_tool dump                                                  │
│                                                                     │
│  # 락스페이스 정보                                                   │
│  $ dlm_tool lockdump <lockspace>                                  │
│                                                                     │
│  【OCFS2 DLM 확인】                                                 │
│  ──────────────────                                                  │
│  # OCFS2 상태                                                       │
│  $ cat /proc/fs/ocfs2/<uuid>/lock_stats                          │
│  $ debugfs.ocfs2 -R "stats" /dev/sdX                             │
│                                                                     │
│  【ZooKeeper 락 구현】                                               │
│  ──────────────────                                                  │
│  # 락 생성 (임시 노드)                                               │
│  create -e /locks/lock-                                           │
│                                                                     │
│  # 락 획득 확인                                                      │
│  ls /locks                                                        │
│                                                                     │
│  # 가장 작은 시퀀스 확인                                             │
│  # 가장 작은 번호를 가진 노드가 락 보유                               │
│                                                                     │
│  【etcd 분산 락】                                                    │
│  ──────────────────                                                  │
│  # 락 획득                                                          │
│  etcdctl lock mylock                                              │
│                                                                     │
│  # 락 해제                                                          │
│  etcdctl unlock mylock                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• DLM: 분산 환경 락 관리자
• 락 모드: NL, CR, CW, PR, PW, EX
• 적용: GFS2, OCFS2, Oracle RAC
• 기능: 동시성 제어, 데드락 감지, 장애 복구
• 대안: Paxos/Raft, 2PC, MVCC
• 비교: 단일 락(뮤텍스) vs 분산 락(DLM)
```
