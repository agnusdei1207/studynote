# 062. 동적 커널 패치 (Live Patching)
#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 재부팅 없이 실행 중인 커널에 보안 패치 적용
> 2. **가치**: 가용성 향상, 패치 윈도우 단축, 보안 강화
> 3. **융합**: kpatch, kGraft, Ksplice, livepatch와 연관
---

## Ⅰ. 개요
### 개념 정의
동적 커널 패치(Live Patching)은 **시스템 재부팅 없이 실행 중인 리눅스 커널에 보안 패치나 버그 수정을 적용하는 기술**이다. 가용성이 중요한 서버 환경에서 사용된다.
### 💡 비유: 비행 중인 비행기 수리
동적 커널 패치는 **비행 중인 비행기 수리**와 같다. 착륙(재부팅)하지 않고도 비행 중에 고장 난 부품(버그)을 교체한다. 승객(사용자)은 모르는 사이에 수리가 완료된다.
### 동적 커널 패치 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                동적 커널 패치 구조                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Live Patching 과정】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 패치 생성                                                 │ │
│  │  • 소스 코드 수정                                            │ │
│  │  • 컴파일 → 새 함수 생성                                      │ │
│  │  • 패치 모듈 (.ko) 빌드                                      │ │
│  │                           │                                   │                  │ │
│  │                           ▼                                   ▼                  │ │
│  │  2. 패치 적재                                                 │ │
│  │  • modprobe로 패치 모듈 적재                                  │ │
│  │  • ftrace 후크 등록                                          │ │
│  │                           │                                   │                  │ │
│  │                           ▼                                   ▼                  │ │
│  │  3. 함수 교체                                                 │ │
│  │  ┌─────────────────────────────────────────────────────────┐   │ │
│  │  │  Original Function    →    Patched Function            │   │ │
│  │  │  (기존 함수)              (패치된 함수)                  │   │ │
│  │  │                                                         │   │ │
│  │  │  호출 → ftrace → 새 함수 실행                          │   │ │
│  │  └─────────────────────────────────────────────────────────┘   │ │
│  │                           │                                   │                  │ │
│  │                           ▼                                   ▼                  │ │
│  │  4. 활성 스택 처리                                             │ │
│  │  • 실행 중인 스레드가 기존 함수에서 반환                        │ │
│  │  • 모든 스레드가 새 함수 사용 확인                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【구현 도구】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  도구              설명                    특징                │ │
│  │  ────                ────                    ────              │ │
│  │  kpatch            Red Hat                ftrace 기반          │ │
│  │  kGraft            SUSE                   작업 전환 기반        │ │
│  │  Ksplice           Oracle                 stop_machine 사용   │ │
│  │  livepatch         커널 내장              표준 API              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
