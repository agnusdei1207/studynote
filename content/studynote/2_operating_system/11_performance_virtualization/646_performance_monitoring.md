# 성능 모니터링 (Performance Monitoring)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 리소스(CPU, 메모리, I/O, 네트워크) 사용량을 실시간으로 측정하고 병목을 식별하는 활동
> 2. **가치**: 성능 저하 원인 파악, 용량 계획, 튜닝 효과 검증의 기초 데이터 제공
> 3. **융합**: 커널 카운터, /proc 파일시스템, 트레이싱, 벤치마킹의 통합

---

### Ⅰ. 개요 (Context & Background)

**개념**
성능 모니터링(Performance Monitoring)는 **시스템의 리소스 사용 현황을 수집, 분석하여 병목 지점을 찾아내고 최적화 방안을 도출하는 프로세스**이다. CPU 사용률, 메모리 사용량, 디스크 I/O, 네트워크 트래픽, 프로세스 상태 등 다양한 메트릭을 모니터링하며 top, vmstat, iostat, sar, perf 등의 도구가 사용된다.

**💡 비유**
성능 모니터링은 **자동차 대시보드**와 같다.
- 속도계: 처리 속도
- 온도계: 과부하 상태
- 연료 게이지: 리소스 남용량
- 경고등: 이상 상태

**등장 배경**
① **1960년**: 메인프레임 모니터링 (SMF)
② **1980년**: UNIX 도구 (vmstat, iostat)
③ **2000년**: APM (Application Performance Management)
④ **현재**: 클라우드, 컨테이너, AIOps

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 모니터링 도구**

| 도구 | 유형 | 장점 | 단점 | 사용 사례 |
|:---|:---|:---|:---|:---|
| **top/htop** | 실시간 | 간단, 직관 | 상세 정보 부족 | 빠른 확인 |
| **vmstat** | 실시간 | 시스템 전반 | 프로세스 정보 없음 | 병판 파악 |
| **iostat** | I/O 전용 | 디스크 상세 | I/O만 | I/O 병판 |
| **perf** | 프로파일링 | 커널 레벨 | 복잡함 | 성능 분석 |
| **eBPF** | 트레이싱 | 안전, 강력 | 최신 기술 | AIOps |

**과목 융합 관점**

1. **운영체제**: 커널, 스케줄러, 메모리 관리
2. **시스템 프로그래밍**: 시스템 콜, /proc 인터페이스
3. **네트워크**: 소켓, 프로토콜 스택

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: CPU 병판**
```bash
# CPU 사용률 확인
top -bn1 | grep "Cpu(s)"

# 프로세스별 CPU 사용
ps aux --sort=-%cpu | head -10

# 컨텍스트 스위치 확인
vmstat 1 5
```

**시나리오 2: 메모리 병판**
```bash
# 메모리 사용 확인
free -h

# 프로세스별 메모리
ps aux --sort=-%mem | head -10

# 스왑 활동
vmstat 1 5  # si/so 확인
```
