# 518-521. 보안 DNS와 로컬 이름 해석 (DNSSEC, DoH, mDNS)

> **핵심 인사이트**: DNS는 정보를 암호화하지 않고 보내는 약점이 있었다. 이를 보완하기 위해 응답의 '진위'를 확인하는 DNSSEC이 등장했고, 이제는 질의 내용 자체를 숨겨서 개인의 프라이버시를 지키는 DoH/DoT 기술이 웹 브라우저의 기본 사양으로 자리 잡고 있다.

---

## Ⅰ. DNSSEC (DNS Security Extensions)
DNS 응답 패킷이 해커에 의해 위조되지 않았음을 증명하기 위해 **디지털 서명**을 추가한 기술입니다.

* **목적**: DNS 캐시 포이즈닝(Cache Poisoning) 방어.
* **원리**: 상위 DNS 서버가 하위 서버의 정보를 보증하는 '신뢰의 사슬(Chain of Trust)' 구조를 형성합니다. (RRSET에 대한 RRSIG 서명)

---

## Ⅱ. DNS 암호화 기술 (DoH vs DoT)
사용자가 어느 사이트에 접속하려는지 중간에서 엿보지 못하게 질의 과정을 암호화합니다.

### 1. DoT (DNS over TLS)
* **포트**: TCP **853**
* **특징**: 전용 포트를 사용하여 DNS 패킷 전체를 TLS로 감쌉니다. 네트워크 관리자가 차단하기 쉽습니다.

### 2. DoH (DNS over HTTPS)
* **포트**: TCP **443** (표준 HTTPS 포트)
* **특징**: 웹 트래픽(HTTPS) 속에 DNS 질의를 숨겨 보냅니다. 일반 웹 서핑과 구분이 안 되어 검열을 회피하기 좋습니다. (구글 크롬, 파이어폭스 기본 지원)

---

## Ⅲ. 로컬 네트워크 이름 해석

### 1. mDNS (Multicast DNS)
* **개념**: 별도의 DNS 서버가 없는 작은 망에서 장치들끼리 이름을 찾는 기술입니다.
* **작동**: "나는 `myprinter.local` 인데 누구 내 IP 아는 사람?"이라고 **멀티캐스트(`224.0.0.251`)**로 물어봅니다.
* **사례**: 애플의 **Bonjour(봉쥬르)** 서비스가 대표적입니다.

### 2. LLMNR (Link-Local Multicast Name Resolution)
* **특징**: mDNS와 유사한 윈도우(Windows) 전용 로컬 이름 해석 프로토콜입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[DNS 보안 기술의 발전]

 [ DNS ] (평문)  ──>  [ DNSSEC ] (디지털 서명) ──> [ DoH / DoT ] (전체 암호화)
   "누가 봐도 됨"       "위조는 안 됨"              "아무도 못 봄"
```

📢 **섹션 요약 비유**: **DNSSEC**은 전단지를 돌릴 때 믿을 수 있는 기관의 '직인'을 찍어서 가짜 뉴스(위조 주소)가 아님을 증명하는 것입니다. **DoH/DoT**는 전단지를 내용이 안 보이는 '검은 봉투'에 담아 전달하여, 길거리의 구경꾼(ISP나 해커)이 내가 무슨 정보를 찾는지 모르게 하는 프라이버시 커튼입니다. **mDNS**는 이름표 없는 파티장에서 "철수야 어디 있니?"라고 크게 소리쳐서 철수가 손을 들게 만드는 친목 도모형 조회 방식입니다.
