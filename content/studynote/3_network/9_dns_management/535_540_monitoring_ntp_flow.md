# 535-540. 네트워크 모니터링 및 시간 동기화 (Syslog, NTP, Flow)

> **핵심 인사이트**: 네트워크 운영의 기본은 '기록'과 '일치'다. 무슨 일이 일어났는지 중앙 서버에 적어두고(Syslog), 모든 장비의 시계를 똑같이 맞춰야만(NTP) 장애 발생 시 정확한 선후 관계를 파악할 수 있다. 또한 NetFlow는 단순한 통계를 넘어 데이터의 흐름을 꿰뚫어 보게 해 준다.

---

## Ⅰ. Syslog (시스템 로그 프로토콜)
장비에서 발생하는 각종 이벤트 메시지를 중앙 로그 서버로 전송하는 표준 프로토콜입니다. (UDP **514** 포트 사용)

* **로그 레벨 (Severity Level)**: 0(Emergency)부터 7(Debug)까지 나뉩니다.
  - 보통 4(Warning)나 5(Notice) 이상을 중요하게 감시합니다.

---

## Ⅱ. NTP (Network Time Protocol)
네트워크에 연결된 장비들의 시간을 동기화하는 프로토콜입니다. (UDP **123** 포트 사용)

* **Stratum (계층) 구조**:
  - **Stratum 0**: 원자시계, GPS 등 정밀 시간원 그 자체.
  - **Stratum 1**: Stratum 0에 직접 연결된 서버.
  - **Stratum 2, 3...**: 하위로 내려갈수록 정확도가 조금씩 떨어집니다. (보통 Stratum 15까지만 유효)
* **PTP (Precision Time Protocol)**: 마이크로초 단위의 정밀도가 필요한 산업용/금융용 특수 환경에서 사용하는 상위 기술입니다.

---

## Ⅲ. 트래픽 흐름 분석 기술

### 1. NetFlow (Cisco) / IPFIX (표준)
* **개념**: 패킷 전체를 보지 않고, **흐름(Flow)** 단위의 통계를 수집합니다.
* **Flow 정의**: 동일한 {Source IP, Dest IP, Source Port, Dest Port, Protocol}을 가진 패킷들의 묶음.
* **장점**: 어떤 사용자가 어떤 앱을 얼마나 썼는지 일목요연하게 파악 가능합니다.

### 2. sFlow (Sampled Flow)
* **개념**: 모든 흐름을 보지 않고 일정 확률로 패킷을 추출(**Sampling**)하여 분석합니다.
* **장점**: 장비의 CPU 부하가 매우 적어 초고속 백본 스위치에 유리합니다.

### 3. RMON (Remote Monitoring)
* **개념**: OSI 1~2계층 수준의 통계(에러 패킷 수, 충돌 횟수 등)를 MIB 형태로 제공하는 모니터링 표준입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[네트워크 통합 관리 흐름]

  [ 장비 A ] ────(Syslog)───> [ 중앙 로그 서버 ] (무슨 일?)
  [ 장비 A ] <───(NTP)────── [ 타임 서버 ] (지금 몇 시?)
  [ 장비 A ] ────(NetFlow)──> [ 트래픽 분석기 ] (누가 많이 써?)
```

📢 **섹션 요약 비유**: **Syslog**는 학교 게시판에 붙는 '사건 사고 보고서'입니다. **NTP**는 시험 시작 전 "자, 모두 내 시계에 맞춰서 9시 정각!"이라고 외치는 선생님의 구령입니다. **NetFlow**는 통행 차량의 번호판 전체를 일일이 적는 대신 "승용차 몇 대, 트럭 몇 대 지나감"이라고 기록하는 스마트한 '교통량 조사' 시스템입니다.
