# 522-527. DHCP(Dynamic Host Configuration Protocol)의 원리

> **핵심 인사이트**: 컴퓨터가 네트워크에 접속할 때 일일이 IP를 수동으로 넣는 것은 고통스러운 일이다. DHCP는 새로운 장치가 들어올 때마다 남는 IP를 빌려주고 관리하는 '네트워크 자동 관리자'이며, 보안을 위해 가짜 관리자(Rogue DHCP)를 걸러내는 기술까지 갖추고 있다.

---

## Ⅰ. DHCP 개요
UDP 포트 **67**(Server)와 **68**(Client)을 사용하여 호스트에게 IP 주소, 서브넷 마스크, 게이트웨이, DNS 주소를 자동으로 할당하는 프로토콜입니다.

---

## Ⅱ. DHCP 동작 4단계 (DORA 과정)
네트워크에 처음 연결된 단말은 다음 과정을 거칩니다. (모든 패킷은 브로드캐스트로 시작함)

1. **Discover (찾기)**: 클라이언트가 "여기 IP 줄 수 있는 서버 있니?"라고 외칩니다.
2. **Offer (제안)**: 서버가 "나 여기 있어! `192.168.1.10` 주소 빌려줄까?"라고 제안합니다.
3. **Request (요청)**: 클라이언트가 "좋아! 그 주소로 나 쓸게. 확정해줘!"라고 요청합니다.
4. **Acknowledgment (확정)**: 서버가 "오케이, 너가 주인이야. 8시간 동안 써!"라고 최종 승인합니다.

---

## Ⅲ. DHCP 핵심 기술 및 파라미터

### 1. 임대 시간 (Lease Time)
* IP 주소는 영구 소유가 아니라 **빌려 쓰는(Lease)** 개념입니다. 
* **갱신 (Renewal)**: 임대 시간의 50%(T1) 시점에 서버에게 연장 신청을 하고, 87.5%(T2) 시점에 마지막 확인을 합니다.

### 2. DHCP Relay Agent (DHCP 릴레이)
* **문제**: DHCP Discover는 브로드캐스트라 라우터를 넘지 못합니다. 
* **해결**: 라우터가 이 요청을 가로채서 멀리 떨어진 DHCP 서버에게 **유니캐스트**로 대신 전달해 주는 기능입니다.

### 3. DHCP Snooping (보안)
* **문제**: 아무나 가짜 DHCP 서버를 띄워 사람들에게 가짜 게이트웨이 IP를 주어 정보를 빼갈 수 있습니다. (Rogue DHCP)
* **해결**: L2 스위치에서 진짜 서버가 연결된 포트만 **Trusted**로 설정하고, 그 외 포트에서 오는 DHCP 응답은 모두 차단하는 기술입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[DHCP DORA 통신 흐름]

 [ Client ] ────── (Discover: 누구 있니?) ─────> [ Network ]
                                                   │
 [ Client ] <───── (Offer: 이거 빌려줄까?) ───── [ Server ]
                                                   │
 [ Client ] ────── (Request: 이거 쓸게!) ──────> [ Server ]
                                                   │
 [ Client ] <───── (Ack: 확정! 잘 써~) ──────── [ Server ]
```

📢 **섹션 요약 비유**: **DHCP**는 '렌터카 업체'입니다. **Discover**는 주차장에서 "차 빌려주실 분?" 하고 찾는 소리고, **Offer**는 직원이 "아반떼 한 대 남았는데 어때요?" 하는 제안입니다. **Request**는 계약서에 싸인하는 것이고, **Ack**는 차 키를 넘겨받는 최종 단계입니다. **Relay Agent**는 옆 동네 지점에 차가 있는지 물어봐 주는 연결 대리점이며, **DHCP Snooping**은 무허가 짝퉁 렌터카 업자를 쫓아내는 주차장 보안 요원입니다.
