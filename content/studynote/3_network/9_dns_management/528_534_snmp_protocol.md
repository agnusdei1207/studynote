# 528-534. SNMP(Simple Network Management Protocol) 분석

> **핵심 인사이트**: 수만 대의 라우터와 스위치를 관리자가 일일이 돌아다니며 점검할 수는 없다. SNMP는 중앙 관리자(NMS)가 각 장비에게 "너 요즘 어떠니?"라고 묻거나, 장비가 알아서 "나 지금 열받았어!"라고 고함을 치는(Trap) 표준화된 관리 언어이다.

---

## Ⅰ. SNMP 구성 요소 3총사
1. **SNMP Manager (NMS)**: 관리를 수행하는 중앙 서버. (예: PRTG, Zabbix 등)
2. **SNMP Agent**: 관리를 받는 장비(라우터, 스위치, 서버) 내에 탑재된 소프트웨어.
3. **MIB (Management Information Base)**: 장비가 가진 정보(CPU 사용률, 인터페이스 상태 등)를 계층적으로 정리해둔 '데이터 사전'입니다.
   - **OID (Object Identifier)**: MIB 내의 특정 정보를 가리키는 고유 주소 숫자. (예: `1.3.6.1.2.1...`)

---

## Ⅱ. SNMP 버전별 차이점 (보안의 진화)

### 1. SNMP v1
* **특징**: 최초의 표준.
* **단점**: 보안이 매우 취약합니다. 'Community String'이라는 비밀번호를 암호화 없이 평문으로 주고받습니다.

### 2. SNMP v2c
* **특징**: 전송 효율이 개선되었습니다. (GetBulk 명령 추가 등)
* **단점**: 보안 방식은 여전히 v1과 똑같은 평문 Community String 방식입니다.

### 3. SNMP v3 (현대의 표준)
* **혁신**: **사용자 기반 보안 모델 (USM)**을 도입했습니다.
* **기능**:
  - **인증 (Authentication)**: 사용자가 누구인지 확인 (MD5, SHA).
  - **암호화 (Privacy)**: 전송되는 메시지 자체를 암호화 (DES, AES).
* **결론**: 실무에서는 보안을 위해 반드시 v3 사용을 권장합니다.

---

## Ⅲ. 주요 SNMP 동작 방식

* **Get**: 매니저가 에이전트에게 특정 정보를 요청.
* **Set**: 매니저가 에이전트의 설정값을 변경.
* **Trap**: **에이전트가 매니저에게 자발적으로 보내는 알림.** 장비에 장애(인터페이스 Down 등)가 생겼을 때 즉시 보고하기 위해 씁니다. (UDP 162 포트 사용)

---

## Ⅳ. 개념 맵 및 요약

```ascii
[SNMP 관리 아키텍처]

 [ SNMP Manager ]  <──(UDP 161: Get/Set)──>  [ SNMP Agent ]
 (중앙 NMS 서버)    <──(UDP 162: Trap)────   (라우터/스위치)
                                                  │
                                            [ MIB / OID ]
                                            (상태 데이터 사전)
```

📢 **섹션 요약 비유**: **SNMP**는 병원 관리 시스템과 같습니다. **매니저**는 의사이고, **에이전트**는 환자 몸에 부착된 센서입니다. **MIB**는 환자의 차트(심박수, 혈압 항목 등)입니다. **Get**은 의사가 회진하며 혈압을 체크하는 것이고, **Trap**은 환자의 심박수가 갑자기 떨어졌을 때 센서가 자동으로 "삐-삐-" 비상벨을 울리는 알람 서비스입니다. **v1/v2**는 보호 장비 없는 무방비 상태이고, **v3**는 방진복과 보안 카드로 철저히 무장한 VIP 병동입니다.
