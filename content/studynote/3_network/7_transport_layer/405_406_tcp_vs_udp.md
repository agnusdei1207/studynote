# 405-406. TCP와 UDP의 특징 비교

> **핵심 인사이트**: 전송 계층에는 '안전함'을 택할 것인가 '속도'를 택할 것인가에 따라 두 가지 길이 있다. TCP는 꼼꼼한 확인 과정을 거쳐 완벽한 배달을 보장하고, UDP는 확인 없이 일단 빠르게 던지는 데 집중한다.

---

## Ⅰ. TCP (Transmission Control Protocol)
신뢰성 있는 연결 지향형 서비스를 제공하는 프로토콜입니다.

* **연결 지향 (Connection-oriented)**: 데이터를 보내기 전 3-Way Handshake를 통해 상대방과 합을 맞춥니다.
* **신뢰성 보장**: 패킷 유실 시 재전송(ARQ)을 수행하며, 순서 번호(Sequence Number)를 통해 뒤섞인 패킷을 정렬합니다.
* **흐름 및 혼잡 제어**: 네트워크 상황에 맞춰 데이터 전송량을 지능적으로 조절합니다.
* **스트림 기반**: 데이터를 경계가 없는 연속된 흐름(Stream)으로 취급합니다.
* **용도**: 웹 브라우징(HTTP), 이메일(SMTP), 파일 전송(FTP), 원격 접속(SSH).

---

## Ⅱ. UDP (User Datagram Protocol)
오버헤드를 최소화한 비연결형 프로토콜입니다.

* **비연결형 (Connectionless)**: 상대방이 받을 준비가 됐는지 확인하지 않고 그냥 던집니다. (일방적 통보)
* **비신뢰성**: 데이터가 가다가 사라지거나 순서가 바뀌어도 UDP 자체는 신경 쓰지 않습니다.
* **빠른 속도**: 복잡한 제어 로직(Handshake, Retransmission)이 없어 지연 시간이 극도로 짧습니다.
* **메시지 기반**: 데이터의 경계가 뚜렷한 데이터그램(Datagram) 단위로 전송합니다.
* **유연성**: 일대다(Multicast), 일대전체(Broadcast) 전송이 가능합니다. (TCP는 오직 1:1만 가능)
* **용도**: 실시간 스트리밍, 온라인 게임, DNS 질의, VoIP(인터넷 전화).

---

## Ⅲ. TCP vs UDP 비교 요약

| 구분 | TCP | UDP |
| :--- | :--- | :--- |
| **연결 방식** | 연결 지향 (3-Way Handshake) | 비연결형 |
| **신뢰성** | 높음 (확인응답, 재전송) | 낮음 (확인 안 함) |
| **전송 순서** | 보장됨 | 보장 안 됨 |
| **제어 기능** | 흐름 제어, 혼잡 제어 있음 | 없음 (단순함) |
| **속도** | 상대적으로 느림 | **매우 빠름** |
| **데이터 단위** | 바이트 스트림 (Segment) | 메시지 (Datagram) |

---

## Ⅳ. 개념 맵 및 요약

```ascii
[전송 방식 비유]

1. TCP (신뢰의 등기 우편)
   [송신자] --- "받을 수 있니?" ---> [수신자]
   [송신자] <--- "응, 보내줘!" --- [수신자]
   [송신자] --- (물건 1 배달) ---> [수신자]
   [송신자] <--- "1번 잘 받았어" -- [수신자]

2. UDP (무자비한 물총 쏘기)
   [송신자] ==(물세례 다다다다!)==> [수신자]
   (상대방이 젖든 말든, 피하든 말든 일단 쏘고 봄)
```

📢 **섹션 요약 비유**: **TCP**는 중요한 서류를 보낼 때 사용하는 '영수증이 포함된 등기 우편'입니다. 받는 사람이 집에 있는지 확인하고, 도장을 찍어야 배달이 완료됩니다. **UDP**는 아파트 현관문에 전단지를 던져놓고 가는 '신문 배달'이나 '실시간 TV 중계'와 같습니다. 한 장쯤 잃어버려도(화질이 잠시 깨져도) 멈추지 않고 계속해서 다음 내용을 전달하는 것이 더 중요하기 때문입니다.
