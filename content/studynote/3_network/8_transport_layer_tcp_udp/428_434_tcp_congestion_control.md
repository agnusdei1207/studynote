# 428-434. TCP 혼잡 제어 (Congestion Control)

> **핵심 인사이트**: 흐름 제어가 상대방의 상태를 보는 것이라면, 혼잡 제어는 '도로 상황(네트워크 망)'을 보는 것이다. TCP는 패킷이 하나라도 유실되면 "길이 막혔구나!"라고 판단하고 즉시 전송량을 줄여 전체 네트워크의 붕괴(Congestive Collapse)를 막는다.

---

## Ⅰ. 혼잡 윈도우 (CWND, Congestion Window)
송신자가 네트워크의 혼잡 정도를 감안하여 **스스로 설정하는** 윈도우 크기입니다. 
* 실제 전송량은 **`Min(RWND, CWND)`** 즉, 수신자가 허용한 크기와 내가 판단한 혼잡 상황 중 더 작은 값을 기준으로 결정됩니다.

---

## Ⅱ. 혼잡 제어의 4단계 알고리즘

### 1. Slow Start (슬로우 스타트)
* **원리**: 처음에는 아주 작게(1 MSS) 시작하지만, ACK를 받을 때마다 CWND를 **2배씩 지수 함수적으로** 늘려나갑니다.
* **목적**: 네트워크의 가용 대역폭을 빠르게 탐색하기 위함입니다.

### 2. Congestion Avoidance (혼잡 회피)
* **시점**: CWND가 미리 정해둔 임계치(**ssthresh**)에 도달하면 발동합니다.
* **원리**: 조심스럽게 CWND를 **매 주기마다 1개씩 선형적으로** 늘립니다. (AIMD 방식의 정석)

### 3. Fast Retransmit (빠른 재전송)
* **시점**: 타임아웃이 나기 전이라도, 수신자로부터 **동일한 번호의 ACK가 3번 연속(3 Duplicate ACKs)** 오면 패킷 유실로 간주합니다.
* **원리**: 타이머가 만료될 때까지 기다리지 않고 즉시 해당 패킷을 재전송합니다.

### 4. Fast Recovery (빠른 회복)
* **원리**: 빠른 재전송 이후, 망이 완전히 마비된 건 아니라고 판단하여 CWND를 1로 줄이지 않고 **절반으로만 줄인 뒤** 다시 혼잡 회피 단계부터 시작합니다.

---

## Ⅲ. TCP 혼잡 제어의 철학: AIMD
* **AIMD (Additive Increase / Multiplicative Decrease)**
  - **합의 증가**: 망이 여유 있으면 조금씩 늘리고(선형 증가)
  - **곱의 감소**: 망이 막히면 반토막을 낸다(지수 감소)
* 이 보수적인 전략 덕분에 전 세계 인터넷이 공평하게 대역폭을 나눠 쓰며 유지되고 있습니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[TCP 혼잡 제어 윈도우 변화 그래프]

CWND 크기
  ^
  |          /| (혼잡 감지: 3 Dup-ACK)
  |         / |  \  (Fast Recovery: 절반으로 하락 후 다시 증가)
  |        /  |   \____/
  |  ____/    |  
  | / (ssthresh 도달)
  |/ (Slow Start: 지수 증가)
  └──────────────────────────────> 시간(Time)
```

📢 **섹션 요약 비유**: **혼잡 제어**는 안개 낀 고속도로를 운전하는 것과 같습니다. **슬로우 스타트**는 정지 상태에서 속도를 20, 40, 80km로 빠르게 올리는 것이고, **혼잡 회피**는 규정 속도 근처에서 1km씩 조심조심 올리는 것입니다. 그러다 앞차의 급정거(패킷 유실)를 발견하면, **빠른 회복**은 속도를 절반으로 줄여 다시 흐름을 타는 것이고, 최악의 사고(타임아웃)가 나면 기어를 다시 1단(CWND=1)으로 내리고 처음부터 시작하는 것입니다.
