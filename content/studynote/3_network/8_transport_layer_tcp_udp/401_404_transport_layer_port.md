# 401-404. 전송 계층의 역할과 포트 번호

> **핵심 인사이트**: 네트워크 계층(IP)이 '컴퓨터 집 주소'까지 배달하는 역할을 한다면, 전송 계층(Transport Layer)은 그 집 안에 있는 '특정 거주자(애플리케이션)'를 정확히 찾아내어 데이터를 전달하는 역할을 한다. 이를 위한 번호표가 바로 포트(Port) 번호이다.

---

## Ⅰ. 전송 계층(Layer 4)의 본질적 역할
IP 계층의 불안정한 배달 서비스를 보완하여, 상위 애플리케이션에게 신뢰성 있는 통신 서비스를 제공합니다.

1. **프로세스 간 통신 (Process-to-Process)**: 컴퓨터 내부에서 동시에 돌아가는 여러 프로그램(웹, 메일, 메신저 등)을 구별하여 데이터를 배달합니다.
2. **신뢰성 보장 (Reliability)**: 패킷 유실 시 재전송을 요청하고, 순서가 뒤섞인 데이터를 원래대로 정렬합니다.
3. **흐름 제어 (Flow Control)**: 받는 사람의 처리 속도에 맞춰 보내는 속도를 조절합니다.
4. **혼잡 제어 (Congestion Control)**: 전체 네트워크의 막힘 정도에 따라 전송량을 조절합니다.

---

## Ⅱ. 포트 번호 (Port Number)
전송 계층에서 사용하는 **16비트** 길이의 식별자입니다. (0 ~ 65,535)

### 1. 포트 번호의 대역별 분류
* **잘 알려진 포트 (Well-Known Ports, 0 ~ 1023)**:
  - 전 세계적으로 약속된 공통 서비스용입니다. (관리자 권한 필요)
  - HTTP(80), HTTPS(443), FTP(21), DNS(53), SSH(22) 등.
* **등록된 포트 (Registered Ports, 1024 ~ 49151)**:
  - 특정 회사가 자사 서비스를 위해 등록하여 사용하는 범위입니다.
  - MS-SQL(1433), MySQL(3306) 등.
* **동적 포트 (Dynamic/Private Ports, 49152 ~ 65535)**:
  - 클라이언트 PC가 임시로(Ephemerally) 사용하는 포트입니다. 사용자가 웹 서버에 접속할 때 자동으로 할당받습니다.

---

## Ⅲ. 소켓 주소 (Socket Address)
네트워크 통신을 하기 위해 필요한 최종적인 결합 주소입니다.

* **정의**: **`IP 주소 + 포트 번호`** 의 쌍을 의미합니다.
* **예시**: `1.2.3.4 : 80` (웹 서버의 소켓)
* **논리적 의미**: 전 세계 인터넷상에서 실행 중인 수억 개의 프로그램 중 딱 하나를 유일하게 가리키는 최종 종착지 주소입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[인터넷 주소의 계층적 구조]

 [ 물리 주소 ] : MAC Address (옆집 장비 찾기)
      │
 [ 논리 주소 ] : IP Address (목적지 컴퓨터 집 찾기)
      │
 [ 포트 번호 ] : Port Number (집 안의 특정 프로그램 찾기)
      │
 [ 소켓 주소 ] : IP + Port (최종 통신 창구 완성)
```

📢 **섹션 요약 비유**: **IP 주소**는 아파트 단지와 동호수(집 주소)입니다. 택배기사는 이 주소만 보고 경비실(라우터)을 거쳐 집 앞에 물건을 둡니다. **포트 번호**는 그 집 식구들의 '이름'입니다. 아빠(HTTP), 엄마(SSH), 아들(Game) 중 누구의 짐인지 확인하는 과정입니다. **소켓**은 물건을 주고받는 '현관문' 그 자체를 의미합니다.
