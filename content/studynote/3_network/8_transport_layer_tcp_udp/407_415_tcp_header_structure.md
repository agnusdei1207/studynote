# 407-415. TCP 세그먼트 헤더 구조 분석

> **핵심 인사이트**: TCP 헤더는 신뢰성 보장을 위한 온갖 제어 장치들의 집합이다. 특히 '내가 어디까지 보냈고(Sequence)', '너가 어디까지 받았는지(Ack)'를 32비트 숫자로 끊임없이 대조하며 단 하나의 바이트 유실도 허용하지 않는다.

---

## Ⅰ. TCP 헤더 개요
기본 **20바이트** 고정이며, 옵션 추가 시 최대 60바이트까지 늘어납니다.

---

## Ⅱ. 주요 필드 상세 분석

### 1. 포트 번호 (각 16bit)
* **Source Port / Destination Port**: 출발지와 목적지의 애플리케이션 식별 번호입니다.

### 2. 신뢰성 보장 필드 (각 32bit)
* **Sequence Number (순서 번호)**: 송신자가 바이트 단위로 부여하는 번호입니다. "이번에 보내는 데이터의 첫 번째 바이트는 전체 데이터 중 몇 번째인가?"를 나타냅니다.
* **Acknowledgment Number (확인 응답 번호)**: 수신자가 송신자에게 보내는 번호입니다. **"다음에 내가 받아야 할 바이트 번호"**를 의미합니다. (누적 ACK 방식)
  - 예: 송신자가 100번부터 500바이트를 보냈다면, 수신자는 600번을 보낼 차례임을 뜻하는 `ACK 600`을 보냅니다.

### 3. 제어 플래그 (6bit)
TCP의 상태를 제어하는 '스위치'들입니다.
* **URG (Urgent)**: 긴급 데이터 포함 여부.
* **ACK (Acknowledgment)**: 확인 응답 번호 필드가 유효함. (연결 설정 시 첫 SYN 패킷 제외하고 항상 1)
* **PSH (Push)**: 수신 측 버퍼에 데이터를 쌓아두지 말고 즉시 애플리케이션으로 올려보내라!
* **RST (Reset)**: 연결 강제 종료. (문제가 생겨서 연결을 확 끊어버림)
* **SYN (Synchronize)**: 연결 설정 요청. 순서 번호를 동기화하자!
* **FIN (Finish)**: 연결 종료 요청. "나 이제 보낼 데이터 다 보냈어."

### 4. 흐름 및 오류 제어 필드
* **Window Size (16bit)**: 현재 내가 받을 수 있는 여유 버퍼 크기를 상대방에게 알립니다. (흐름 제어용)
* **Checksum (16bit)**: 헤더와 데이터 전체, 그리고 가상 헤더(IP 정보 포함)까지 포함하여 에러를 검출합니다.
* **Urgent Pointer**: URG 플래그가 1일 때, 어디까지가 긴급 데이터인지 위치를 가리킵니다.

---

## Ⅲ. TCP 가상 헤더 (Pseudo Header)
TCP 체크섬 계산 시에는 실제 TCP 헤더뿐만 아니라, 하위 계층(IP)의 정보 중 일부(출발지/목적지 IP 주소, 프로토콜 번호 등)를 가져와 임시로 결합하여 계산합니다. 이는 패킷이 엉뚱한 IP로 배달되는 사고를 이중으로 방지하기 위함입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[TCP 세그먼트 헤더 구조]

 0                   15 16                  31
 ┌─────────────────────┬──────────────────────┐
 │     Source Port     │   Destination Port   │
 ├─────────────────────┴──────────────────────┤
 │              Sequence Number               │
 ├────────────────────────────────────────────┤
 │           Acknowledgment Number            │
 ├──────┬──────────────┬──────────────────────┤
 │Offset│ Flags(UAPRSF)│     Window Size      │
 ├──────┴──────────────┼──────────────────────┤
 │      Checksum       │    Urgent Pointer    │
 ├─────────────────────┴──────────────────────┤
 │          Options (가변, 최대 40B)           │
 └────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: **TCP 헤더**는 아주 꼼꼼한 관리자가 작성하는 '운송장'입니다. 이 운송장에는 상품의 바코드(Sequence), 고객의 수령 확인 싸인(Ack), "급함(URG)", "가게 문 닫음(FIN)" 같은 온갖 메모가 빼곡히 적혀 있습니다. 이 메모들을 통해 택배 기사(네트워크)가 물건을 잃어버리거나 순서를 바꿔도 끝내 주인에게 올바르게 전달할 수 있게 됩니다.
