# 446-447. 차세대 전송 프로토콜: MPTCP와 SCTP

> **핵심 인사이트**: 전통적인 TCP는 두 컴퓨터 사이의 '단 한 줄기 선'에 의존했다. 하지만 현대의 스마트폰은 LTE와 Wi-Fi를 동시에 가지고 있다. 이 여러 개의 통로를 하나로 묶어 속도를 높이고 끊김 없는 통신을 구현하기 위해 탄생한 것이 MPTCP와 SCTP이다.

---

## Ⅰ. MPTCP (Multipath TCP)
기존 TCP를 확장하여 하나의 세션 안에서 **여러 개의 하위 경로(Subflow)**를 동시에 사용할 수 있게 해주는 프로토콜입니다. (RFC 6824)

* **핵심 기능**:
  - **대역폭 결합 (Aggregation)**: Wi-Fi(100Mbps)와 LTE(100Mbps)를 동시에 써서 200Mbps의 속도를 냅니다.
  - **무단절 핸드오버 (Resilience)**: 집 안에서 Wi-Fi로 동영상을 보다가 밖으로 나가서 Wi-Fi가 끊겨도, 연결된 LTE 경로가 살아있으므로 끊김 없이 영상을 계속 볼 수 있습니다.
* **호환성**: 기존 애플리케이션은 수정 없이 그대로 쓸 수 있으며, 오직 운영체제 커널의 TCP 스택만 MPTCP를 지원하면 됩니다. (아이폰의 시리, 애플 뮤직 등에서 이미 활발히 사용 중)

---

## Ⅱ. SCTP (Stream Control Transmission Protocol)
TCP의 신뢰성과 UDP의 메시지 지향성을 모두 갖춘 차세대 전송 프로토콜입니다. 주로 4G/5G 이동통신망의 제어 신호(Signaling) 전송에 쓰입니다.

### 1. 주요 특징
* **멀티호밍 (Multi-homing)**: 하나의 연결(Association)에 **여러 개의 IP 주소**를 맵핑할 수 있습니다. 메인 IP가 죽으면 즉시 백업 IP로 경로를 바꿉니다. (고가용성)
* **멀티 스트리밍 (Multi-streaming)**: 하나의 연결 안에 여러 개의 독립적인 데이터 흐름(Stream)을 둡니다. 1번 스트림에서 패킷이 유실되어도 2번 스트림은 영향을 받지 않고 계속 진행됩니다. (Head-of-Line Blocking 해결)
* **4-Way Handshake (Cookie 방식)**: TCP의 3-Way 방식보다 보안이 강화되었습니다. 연결 요청 시 서버가 쿠키를 발행하여 클라이언트에게 다시 돌려받아야만 자원을 할당하므로, **SYN Flooding 공격에 매우 강합니다.**

---

## Ⅲ. 프로토콜 비교 요약

| 구분 | TCP | MPTCP | SCTP |
| :--- | :--- | :--- | :--- |
| **경로 수** | 단일 경로 | **다중 경로** | **다중 경로 (멀티호밍)** |
| **데이터 단위** | 바이트 스트림 | 바이트 스트림 | **메시지** |
| **신뢰성** | 높음 | 높음 | 높음 |
| **주요 장점** | 범용성 | **속도/이동성** | **안정성/보안** |

---

## Ⅳ. 개념 맵 및 요약

```ascii
[MPTCP vs SCTP 구조 차이]

1. MPTCP (통로를 합쳐서 속도 UP!)
   [ App ] ───> [ MPTCP Session ]
                    /      \
               [ Wi-Fi ]  [ LTE ]  (동시 전송)

2. SCTP (독립된 흐름으로 정체 ZERO!)
   [ App ] ───> [ SCTP Association ]
                    /      \
               [Stream 1] [Stream 2] (1번 막혀도 2번은 잘 감)
```

📢 **섹션 요약 비유**: **MPTCP**는 강남에서 분당까지 가는데 경부고속도로와 국도 두 길로 트럭 2대를 동시에 보내서 짐을 빨리 옮기는 전략입니다. **SCTP**는 기차 한 대에 칸을 여러 개(멀티 스트림) 나눠서, 1번 칸 손님이 짐을 찾느라 내려도 2번 칸 손님들은 지장 없이 내리는 방식이며, 선로가 고장 나면 즉시 옆 선로로 기차를 옮기는(멀티호밍) 안전 시스템까지 갖춘 특급 열차입니다.
