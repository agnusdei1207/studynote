# 281-284. 레거시 통신망 기술과 광통신(PON/AON)

> **핵심 인사이트**: 1980~90년대 이더넷(CSMA/CD)이 잦은 충돌 문제로 고전할 때, 충돌 없는 통신을 무기로 내세운 토큰 링(Token Ring)과 FDDI 같은 기술들이 치열하게 경쟁했으나, 스위치(Switch)의 등장으로 이더넷이 최종 승리하면서 역사 속으로 사라졌다.

---

## Ⅰ. 토큰 링과 토큰 버스
이더넷의 임의 접근 방식(경쟁)에 대항하여 IBM이 주도한 '통제된 접근' 방식의 네트워크 표준입니다.

### 1. 토큰 링 (Token Ring, IEEE 802.5)
* **방식**: 컴퓨터들이 논리적/물리적인 둥근 링(Ring) 형태로 연결되어 있습니다. 네트워크에는 '토큰(Token)'이라는 특수한 3바이트짜리 짧은 프레임이 계속 돕니다.
* **통신**: 데이터를 보낼 컴퓨터는 이 토큰을 잡아(Seize) 데이터 프레임으로 바꾼 뒤 보냅니다. 전송이 끝나면 다시 빈 토큰을 링에 뱉어냅니다. 
* **장점**: 충돌이 완벽하게 방지되며, 네트워크 부하가 100%일 때도 예측 가능한 전송 지연을 보장합니다.

### 2. 토큰 버스 (Token Bus, IEEE 802.4)
* **방식**: 케이블은 이더넷처럼 일자형 버스(Bus) 구조로 깔려있지만, 그 안에서 통신할 때는 각 단말이 논리적인 링 구조를 만들어 토큰을 주고받습니다.
* **용도**: 주로 공장 자동화(FA) 망에서 사용되었습니다.

---

## Ⅱ. 광통신 기반의 초기 고속망 기술

### 1. FDDI (Fiber Distributed Data Interface)
* **특징**: 1980년대 후반에 나온 100Mbps 급의 초고속 광케이블 기반 근거리/광역 통신망(LAN/MAN) 표준입니다. 토큰 링 방식을 광케이블에 적용했습니다.
* **이중 링 (Dual Ring) 구조**: 하나의 링(Primary)이 끊어지면 보조 링(Secondary)이 역방향으로 돌며 자동으로 끊어진 부분을 치유(Self-healing)하는 강력한 장애 복구 능력을 가졌습니다. (과거 대학 캠퍼스망 등 백본망으로 활약)

### 2. DQDB (Distributed Queue Dual Bus, IEEE 802.6)
* **특징**: 도시 규모의 통신망인 MAN(Metropolitan Area Network)을 구축하기 위해 만들어진 초기 프로토콜입니다.
* **구조**: 서로 반대 방향으로 흐르는 두 개의 버스(선)에 노드들이 물려 있으며, 슬롯(Slot)이라는 고정된 데이터 조각을 빈자리에 싣고 내리는 분산 큐(Queue) 방식을 사용했습니다. 나중에 ATM 기술의 모태가 됩니다.

---

## Ⅲ. 현대의 광가입자망 (PON vs AON)
집 앞까지 광케이블을 끌고 오는 FTTH(Fiber To The Home)를 구현하는 두 가지 방식입니다.

### 1. AON (Active Optical Network)
* **방식**: 전화국과 각 집 사이에 **전기가 필요한 스위치/라우터(Active Node)**를 배치하여 광신호를 분배합니다.
* **특징**: 각 가입자마다 독립적인 대역폭을 전용으로 보장할 수 있어 속도가 빠르지만, 중간 기지국에 전원을 공급해야 하고 장비가 비싸 유지보수 비용이 높습니다.

### 2. PON (Passive Optical Network)
* **방식**: 전기가 전혀 필요 없는 **수동 광분배기(Passive Splitter, 거울/프리즘 역할)**를 사용하여 하나의 광가닥을 32~64개로 쪼개 각 집으로 보냅니다.
* **특징**: 중간에 전기가 필요 없어 고장이 적고 구축 비용이 압도적으로 저렴합니다. (현재 대부분의 기가 인터넷 망, EPON/GPON 이 이 방식을 씁니다.)

## Ⅳ. 개념 맵 및 요약

```ascii
[LAN 토폴로지와 프로토콜의 경쟁의 역사]

(1980년대)
  Ethernet (버스형, CSMA/CD) vs Token Ring (링형, 토큰 제어)
    └─ 충돌 때문에 불안정함       └─ 충돌은 없지만 장비가 비쌈

(1990년대)
  Fast Ethernet (100Mbps)    vs FDDI (이중 링 광통신 100Mbps)
    └─ "스위치" 발명으로 충돌 극복! └─ 너무 비싸서 도태됨

=> 최종 승리자: Ethernet (스위치 기반의 Star 토폴로지)

[FTTH 구현 방식 비교]
        (전기가 필요함)                (전기 불필요, 빛 반사)
[ 전화국 ] ---> [ AON 스위치 ]             [ PON 스플리터 ] ---> [ 집 1 ]
                   |                       /                 ---> [ 집 2 ]
                 [ 집 1 ]                 /                  ---> [ 집 3 ]
```

📢 **섹션 요약 비유**: **토큰 링**은 회의실에서 "말하기 막대기"를 가진 사람만 말할 수 있는 아주 질서 정연한 회의 방식입니다. 반면 **이더넷**은 눈치껏 말하는 방식이었죠. 초기엔 이더넷이 시끄러웠지만, **스위치**라는 완벽한 진행자(사회자)가 등장하면서 이더넷이 압승을 거두었습니다.
**PON** 망은 커다란 수도관 하나를 동네 어귀까지 끌고 와서 전기가 필요 없는 Y자 배관 이음새(스플리터)로 쪼개어 여러 집에 싸게 물을 공급하는 지혜로운 방식입니다.
