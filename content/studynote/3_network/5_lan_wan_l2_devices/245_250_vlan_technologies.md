# 245-250. 가상 랜(VLAN)과 트렁킹 기술

### # [가상 랜(VLAN) 및 트렁킹 기술]
#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하나의 물리적 스위치를 논리적으로 분할하여 브로드캐스트 도메인을 격리하는 L2 계층의 가상화 기술이며, 802.1Q 프로토콜을 통해 프레임에 식별자를 삽입한다.
> 2. **가치**: 물리적 재배치 없이 보안 그룹을 유연하게 구성할 수 있으며, 불필요한 브로드캐스트 스톰을 억제하여 네트워크 대역폭 효율성을 극대화한다.
> 3. **융합**: 라우팅(Routing) 없이는 통신이 불가능하므로 L3 스위칭 기술과 필수적으로 결합되며, 가상화(Virtualization) 및 컨테이너 네트워킹의 기반이 된다.

---

## Ⅰ. 개요 (Context & Background)

### 1. 개념 및 정의
**VLAN (Virtual Local Area Network)**은 물리적인 배치와 상관없이 네트워크 장비들을 논리적으로 그룹화하여, 별도의 브로드캐스트 도메인을 구축하는 기술이다. 전통적인 LAN (Local Area Network)은 물리적 세그먼트(동일한 허브나 스위치)에 연결된 모든 장비가 동일한 브로드캐스트 도메인을 공유하지만, VLAN은 스위치의 소프트웨어 로직을 통해 이를 분할한다. 이는 하나의 스위치가 마치 여러 개의 독립된 스위치처럼 동작하게 만드는 논리적 파티셔닝(Logical Partitioning) 기술이다.

### 2. 등장 배경: 스위칭의 효율성과 보안의 딜레마
**① 기존 한계 (Collision & Broadcast)**:
공유 미디어(Hub) 환경의 충돌(Collision) 문제는 스위치(Switch)의 도입으로 해결되었다(각 포트가 독립적인 충돌 도메인). 하지만 스위치는 기본적으로 **Flooding** 메커니즘을 사용하여, 브로드캐스트 프레임(ARP, DHCP 등)을 동일 VLAN 내 모든 포트로 전송한다. 네트워크 규모가 커지면 브로드캐스트 트래픽이 기하급수적으로 증가하여 **Broadcast Storm** 발생으로 전체 성능이 저하되는 문제가 발생한다.

**② 혁신적 패러다임 (Logical Segmentation)**:
물리적으로 장비를 분리하거나 추가 라인을 깔지 않고, 하나의 스위치 내에서 트래픽을 격리할 필요성이 대두되었다. VLAN은 이러한 수요를 충족시켜 물리적 재배치 비용을 제로(Zero)로 만들었다.

**③ 현재의 비즈니스 요구 (Security & Flexibility)**:
재택근무와 핫데스킹(Hot-desking) 환경에서 사용자의 물리적 위치는 수시로 변하지만, 보안 그룹(부서)은 유지되어야 한다. VLAN은 포트 기반 또는 MAC 기반 할당을 통해 이러한 유연성과 보안(Isolation)을 동시에 제공한다.

> **💡 비유**: 일반적인 사무실 건물(스위치)에 벽을 쳐서 회계팀(Admin VLAN)과 개발팀(Dev VLAN)을 분리하는 것과 같다. 비록 같은 건물(Physical Device)에 있지만, 서로 다른 출입문(VLAN ID)을 사용하므로 업무 혼선이 발생하지 않는다.

### 3. 브로드캐스트 도메인의 분리 메커니즘
스위치 내부의 **VLAN Database**는 포트와 VLAN ID를 매핑한다. 특정 VLAN에 속하지 않은 포트로는 다른 VLAN의 프레임이 절대 전송되지 않는다. 즉, L2 스위치에서는 VLAN 간 통신이 원천적으로 차단된다.

```ascii
[물리적 구조 vs 논리적 구조]

   물리적 연결 (Physical Layout)          논리적 구조 (Logical Layout)
   ┌─────────────────────────┐
   │   Physical Switch 1     │           ┌───────┐ (VLAN 10)      ┌───────┐ (VLAN 20)
   │ ┌───┐┌───┐┌───┐┌───┐   │           │ PC A  │ <-----------> │ PC B  │
   │ │ 1 ││ 2 ││ 3 ││ 4 │   │           └───────┘               └───────┘
   │ └─┬─┘└─┬─┘└─┬─┘└─┬─┘   │                 ▲                       ▲
   │   │    │    │    │     │                 │                       │
   └───┼────┼────┼────┼─────┘                 │                       │
       │    │    │    │                       │                       │
     [A]  [B]  [C]  [D]                (Isolated)              (Isolated)
       
    [A, B]: VLAN 10 (Engineering)    → 논리적으로 완전히 분리된 스위치처럼 작동
    [C, D]: VLAN 20 (Sales)
```

### 4. 아키텍처적 핵심
스위치는 각 포트의 PVID (Port VLAN ID)를 확인하여 수신 프레임을 해당 VLAN의 MAC Address Table에만 등록한다. 이로 인해 브로드캐스트 프레임이 동일한 VLAN ID를 가진 포트로만 플러딩(Flooding)된다.

**📢 섹션 요약 비유**: VLAN은 마치 하나의 큰 회의실에 가벽(Partition)을 설치하여, 영업팀과 개발팀이 물리적으로는 같은 공간에 있지만 소리(트래픽)가 서로 섞이지 않도록 격리하는 '논리적인 방' 구축 기술입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 구성 요소 상세 분석

| 요소명 (Component) | 역할 (Role) | 내부 동작 (Internal Behavior) | 비유 (Analogy) |
|:---|:---|:---|:---|
| **VLAN ID** | VLAN 식별자 (12-bit) | 0~4095 범위의 식별자. 1~4094 사용 가능. 스위치가 프레임의 소속을 판단하는 기준 | 부서 번호 (예: 10번은 인사팀) |
| **Access Port** | 단말기 연결 포트 | **Untagged** 프레임 수신 → 내부적으로 태그 삽입(PVID 할당). 송신 시 태그 제거(Unstrip) 후 전달. 단일 VLAN만 허용. | 일반 사무실 입구 (명찰 없이 출입) |
| **Trunk Port** | 스위치 간 연결 포트 | **Tagged** 프레임만 처리. 다중 VLAN 트래픽을 하나의 물리 링크로_multiplexing_. 태그 유지하며 전달. | 고속 도로 (여러 지역차량이 혼재) |
| **Native VLAN** | 태그 미부여 트래픽 처리 | Trunk 포트에서 Untagged 프레임 수신 시, 이를 처리할 기본 VLAN ID. (Default: VLAN 1) | 특별 입국장 (별도 검사 없이 입국) |
| **802.1Q Header** | 태깅 프로토콜 | 이더넷 프레임 헤더에 4바이트 삽입. TPID(0x8100)와 TCI(VLAN ID 등) 포함. | 택배 송장 (어디로 갈지 정보) |

### 2. 포트 모드별 트래픽 처리 프로세스 (ASCII Flow)

스위치 포트는 Access Mode와 Trunk Mode에 따라 프레임을 처리하는 방식이 결정적으로 다르다.

```ascii
[Trunk Port 동작 메커니즘 (Ingress & Egress)]

   송신 (Egress) - 스위치 내부에서 외부로 나갈 때
   ┌────────────────────────────────────────────────────────────────┐
   │                        Switch Internal Fabric                 │
   └───────┬───────────────────────┬───────────────────────┬───────┘
           │                       │                       │
     [VLAN 10 Tag]          [VLAN 20 Tag]          [Native VLAN Tag]
           │                       │                       │
    ┌──────▼──────┐           ┌────▼─────┐          ┌─────▼─────┐
    │ 802.1Q Tag  │           │ 802.1Q Tag│          │ Strip Tag │
    │   Remain    │           │   Remain  │          │ (Remove)  │
    └──────┬──────┘           └────┬─────┘          └─────┬─────┘
           │                       │                       │
   ────────┴───────────────────────┴───────────────────────┴────────▶  (Wire)
    (Tagged Frame)           (Tagged Frame)           (Untagged Frame)
       
   수신 (Ingress) - 외부에서 스위치 내부로 들어올 때
   ◀───────────────────────────────────────────────────────────────
   ┌───────────────────────────────────────────────────────────────┐
   │  1. Tagged Frame    → Tag 확인 -> VLAN Table Lookup (Switching) │
   │  2. Untagged Frame  -> PVID(Native VLAN)로 할당 -> Switching    │
   └───────────────────────────────────────────────────────────────┘
```

### 3. 심층 동작 원리: 태깅 (Tagging)과 언태깅 (Untagging)

**① Ingress Process (수신)**:
*   **Access Port**: 태그가 없는 프레임을 수신하면, 포트에 설정된 **PVID (Port VLAN ID)**를 프레임에 할당하여 스위치 내부버스로 전달한다. 만약 태그가 있는 프레임이 들어오면 보안 설정(Discard)에 따라 드롭할 수 있다.
*   **Trunk Port**: 태그가 있는 프레임은 유효성을 검사(Allowed VLAN List) 후 내부로 전달한다. 태그가 없는 프레임이 수신되면 이를 **Native VLAN**으로 간주하여 해당 VLAN의 PVID를 할당한다.

**② Egress Process (송신)**:
*   **Access Port**: 프레임을 포트로 내보낼 때, 태그를 제거(**Untag/Strip**)하고 순수 이더넷 프레임으로 단말기에 전달한다. (단말기는 태그를 인식하지 못하므로)
*   **Trunk Port**: 프레임이 나가는 포트의 Native VLAN과 일치하면 태그를 제거하고, 나머지 VLAN은 태그를 유지한 채(**Tagged**) 전송한다.

### 4. IEEE 802.1Q 헤더 구조 및 수식

태깅은 기존 이더넷 헤더를 수정하지 않고, **Source MAC Address**와 **EtherType** 필드 사이에 **4바이트(32비트)**를 삽입(Insertion)하는 방식으로 작동한다.

*   **MTU (Maximum Transmission Unit) 변화**: 기존 1500바이트에서 1522바이트(Baby Giant)로 증가하므로, 모든 네트워크 장비가 이를 지원해야 한다.

```text
+-------+-------+--------+-------+--------+
| Dest  | Src   | 802.1Q | Type  | Data   |
| MAC   | MAC   |  Tag   |       |        |
+-------+-------+--------+-------+--------+

[802.1Q Tag 상세 구조 (4 Bytes)]
  16비트      3비트   1비트      12비트
+-----------+--------+-----+---------------+
| TPID      | PCP    | DEI | VLAN ID (VID) |
| 0x8100    | Priority| CFI | (1 ~ 4094)   |
+-----------+--------+-----+---------------+
```

*   **TPID (Tag Protocol Identifier)**: `0x8100` 고정. 이더넷 프레임임에도 802.1Q 태그가 있음을 식별.
*   **PCP (Priority Code Point)**: 802.1p QoS 우선순위 (0~7).
*   **VID (VLAN ID)**: $0$ (Null) 부터 $2^{12}-1$ (4095) 까지 가능.
    *   $0$: Null VID (우선순위만 표시)
    *   $1$: Default VLAN (보안상 사용 권장하지 않음)
    *   $4095$: Reserved

**📢 섹션 요약 비유**: Trunk 포트는 마치 고속도로 톨게이트와 같습니다. 진입할 때(Trunk In) 모든 차량이 혼재되지만, 목적지를 확인하기 위해 요금소(Tagging)에서 고지서를 붙입니다. 반대로 일반 도로(Access Port)로 나갈 때는 고지서를 제거(Untagging)하고 깔끔한 상태로 내보내줍니다. 이때 '고지서 면제 차선'이 바로 Native VLAN입니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 심층 기술 비교: 802.1Q vs ISL (Legacy)

| 비교 항목 | **IEEE 802.1Q (Standard)** | **ISL (Inter-Switch Link, Cisco Legacy)** |
|:---|:---|:---|
| **표준 여부** | **IEEE 공식 표준** (벤더 무관) | 시스코(Cisco) 전용 규격 (Legacy) |
| **캡슐화 방식** | **Tagging (삽입)**: 기존 헤더 중간에 태그 삽입 | **Encapsulation (캡슐화)**: 원본 프레임을 새 헤더/트레일러로 감쌈 |
| **프레임 크기** | **1522 Bytes** (원본 1500 + 4) | **1548 Bytes** (원본 + 26 Header + 4 Trailer) |
| **Native VLAN** | 지원 (태그 없는 트래픽 허용) | 미지원 (모든 프레임에 태그/캡슐 필수) |
| **호환성** | 모든 현대적인 스위치/L3 장비 지원 | 구형 시스코 장비에서만 작동 |

### 2. L2 스위칭과 L3 라우팅의 융합 (Inter-VLAN Routing)

VLAN으로 나누어진 브로드캐스트 도메인 간에는 L2 통신이 불가능하다. 이를 해결하기 위해 **Router on a Stick** 또는 **L3 Switch**가 필요하다.

*   **L3 Switch (Layer 3 Switch)**: 스위치 내부에 **VLAN Interface (SVI: Switched Virtual Interface)**를 생성하여, 스위치가 라우팅 기능까지 수행한다.
    *   **구조**: `VLAN 10 (Gatew
