# 272-275. ATM (비동기 전송 모드) 기술

> **핵심 인사이트**: 인터넷 데이터(가변 길이)와 전화 음성(실시간)을 하나의 망에서 완벽하게 처리하기 위해, 모든 데이터를 아주 작고 똑같은 크기(53바이트)의 조각(Cell)으로 잘라서 하드웨어적으로 빠르게 스위칭하는 기술이 ATM이다.

---

## Ⅰ. ATM (Asynchronous Transfer Mode) 개요
광대역 ISDN(B-ISDN)을 구현하기 위한 핵심 전송 기술로, 패킷 교환망과 회선 교환망의 장점만을 결합한 기술입니다.

### 1. 53바이트 고정 길이 셀 (Cell)
* 이더넷처럼 데이터 길이가 들쭉날쭉한 '프레임(Frame)'을 쓰지 않고, 무조건 **5바이트의 헤더 + 48바이트의 페이로드 = 53바이트** 길이를 가진 '셀(Cell)'로 데이터를 쪼개어 보냅니다.
* **이유**: 길이가 고정되어 있으면 스위치가 소프트웨어(CPU)로 길이를 계산할 필요 없이, 반도체 칩(하드웨어) 레벨에서 엄청나게 빠른 속도로 셀을 목적지로 던져줄 수 있기 때문입니다. 또한 크기가 작아 실시간 음성 트래픽의 지연(Delay)을 최소화할 수 있습니다.

### 2. 비동기식 (Asynchronous) 전송
* 송신자가 보낼 데이터가 있을 때만 셀을 전송합니다. 빈 시간에는 다른 단말이 그 대역폭을 쓸 수 있어, 전통적인 TDM(시분할 다중화) 방식보다 회선 효율이 매우 높습니다.

---

## Ⅱ. ATM의 핵심 구조와 주소 체계

### 1. ATM 동기화 (HEC)
ATM 셀은 너무 작아서 이더넷처럼 Preamble 같은 시작/끝 표시를 따로 두지 않습니다.
대신 헤더에 있는 **HEC (Header Error Control)** 필드를 이용해 지속적으로 수학적 검사를 수행하다가, 검사가 연속으로 통과하면 "아, 지금부터 셀이 시작되는구나" 하고 동기를 맞춥니다.

### 2. VPI와 VCI (가상 경로/채널 식별자)
ATM은 철저한 **가상 회선 (Virtual Circuit)** 기반 통신입니다. 목적지 IP 주소를 보지 않고, 셀 헤더에 적힌 가상 회선 번호만 보고 스위칭합니다.
* **VPI (Virtual Path Identifier)**: 큰 파이프(경로)의 번호입니다. 여러 개의 채널을 묶어서 한 번에 라우팅할 때 씁니다.
* **VCI (Virtual Channel Identifier)**: 파이프 안의 작은 개별 통신 채널 번호입니다.
* 스위치는 들어온 셀의 VPI/VCI 값을 자신의 테이블과 대조하여, 나갈 포트와 새로운 VPI/VCI 값으로 라벨을 바꿔치기(Label Swapping)하며 전송합니다.

---

## Ⅲ. AAL (ATM Adaptation Layer)
IP 패킷(1500바이트)이나 음성 데이터의 특성은 제각각인데, ATM 망은 무조건 48바이트로 쪼개야 합니다. 이 사이에서 데이터를 조각내고 다시 조립해 주는 '적응(Adaptation)' 계층이 AAL입니다.

### 트래픽 종류에 따른 AAL 분류
* **AAL 1**: 실시간이고 전송 속도가 일정한 트래픽 (예: 64Kbps 무압축 음성 전화)
* **AAL 2**: 실시간이지만 전송 속도가 가변적인 트래픽 (예: 압축된 오디오, 동영상 스트리밍)
* **AAL 5**: 시간에 쫓기지 않는 비실시간 데이터 트래픽. 구조가 가장 단순하여 IP 패킷이나 이더넷 프레임을 ATM 망에 태울 때 가장 널리 쓰입니다. (SEAL 이라고도 부름)

---

## Ⅳ. 개념 맵 및 요약

```ascii
[ATM 셀 구조 및 라우팅]

   [ 5 Bytes Header ] + [ 48 Bytes Payload ] = Total 53 Bytes
   (VPI/VCI, HEC 포함)     (잘게 쪼개진 IP 패킷이나 음성 데이터)

[계층별 역할]
 [ IP 계층 ] : 1500바이트 짜리 거대한 패킷 
     ↓
 [ AAL 계층 ]: 패킷을 48바이트 단위로 깍둑썰기 (Segmentation)
     ↓
 [ ATM 계층 ]: 썰어진 조각에 5바이트 헤더(VPI/VCI)를 붙여 53바이트 셀 완성 후 하드웨어 스위칭!
```

📢 **섹션 요약 비유**: 이더넷이 택배 트럭에 크기가 제각각인 상자(프레임)들을 싣고 가서 차가 막히면 한참을 기다려야 하는 방식이라면, ATM은 모든 화물을 아주 작은 규격의 '레고 블록(53바이트 셀)'으로 똑같이 쪼갠 뒤, 초고속 컨베이어 벨트를 통해 딜레이 없이 목적지로 쏘아버리는 물류 자동화 시스템입니다.
