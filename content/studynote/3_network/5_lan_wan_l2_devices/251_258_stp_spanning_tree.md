# 251-258. L2 루프 문제와 스패닝 트리 프로토콜(STP)

> **핵심 인사이트**: 네트워크 장애를 막기 위해 스위치를 여러 개 달아 선을 두 갈래(이중화)로 연결하면, 프레임이 무한히 맴도는 '루프(Loop)'라는 재앙이 발생한다. 이를 막기 위해 논리적으로 하나의 길만 남기고 나머지를 막아버리는 지능형 알고리즘이 바로 STP다.

---

## Ⅰ. L2 스위치 루프(Loop) 현상의 문제점
스위치 간의 연결이 원형(Ring) 구조를 이룰 때 발생합니다. IP 패킷(3계층)은 TTL(수명) 값이 있어 루프를 돌아도 언젠가 죽지만, 이더넷 프레임(2계층)은 수명 제한이 없어 영원히 네트워크를 떠돕니다.

### 1. 브로드캐스트 스톰 (Broadcast Storm)
* PC가 브로드캐스트 프레임을 보내면, 스위치는 이를 복사하여 다른 포트로 플러딩(Flooding)합니다.
* 루프 구조에서는 스위치 A가 B로, B가 C로, C가 다시 A로 플러딩을 무한 반복합니다. 순식간에 수십만 개의 프레임이 증식하여 네트워크 대역폭을 100% 잡아먹고 장비가 다운됩니다.

### 2. MAC 주소 호핑 (MAC Flapping)
* 프레임이 양쪽 길로 번갈아 들어오기 때문에, 스위치는 동일한 PC의 MAC 주소가 1번 포트에 있다가 2번 포트에 있다가 계속 헷갈리게 됩니다.
* MAC 테이블이 초당 수천 번씩 업데이트되며 스위치의 CPU가 과부하에 걸립니다.

---

## Ⅱ. 스패닝 트리 프로토콜 (STP, IEEE 802.1D)
물리적으로는 이중화(루프)로 연결해 두되, 소프트웨어적으로 한쪽 포트를 **논리적으로 차단(Blocking)**하여 전체 네트워크를 가지(Tree) 모양의 단일 경로로 만들어주는 프로토콜입니다.

### 1. BPDU (Bridge Protocol Data Unit)
* 스위치들이 서로 인사를 나누고 누가 대장(Root)을 할지, 어느 길을 막을지 결정하기 위해 2초마다 주고받는 특별한 제어 프레임입니다.

### 2. STP 포트 선정 규칙 (Root -> RP -> DP -> Block)
1. **루트 브리지 (Root Bridge) 선출**
   - 네트워크의 중심(뿌리)이 될 대장 스위치를 하나 뽑습니다.
   - **Bridge ID (우선순위 값 + MAC 주소)**가 가장 **낮은** 스위치가 대장이 됩니다.
2. **루트 포트 (RP, Root Port) 선정**
   - 나머지 부하 스위치(Non-Root)들은 대장에게 가는 "가장 빠르고 가까운 길(포트)"을 하나씩 뽑습니다. 이것이 RP입니다.
   - **경로 비용 (Path Cost)** 이 가장 낮은 포트를 우선으로 고릅니다. (속도가 1Gbps면 비용이 낮고, 10Mbps면 비용이 높음)
3. **지정 포트 (DP, Designated Port) 선정**
   - 스위치와 스위치가 마주 보는 하나의 선(세그먼트)마다, 트래픽을 전달할 대표 포트 1개를 뽑습니다. 이것이 DP입니다.
4. **차단 포트 (BP, Blocking / Non-Designated Port)**
   - 위의 RP도 아니고 DP도 아닌 남는 포트들은 무조건 **데이터가 통과하지 못하게 막아버립니다(Blocking).** 여기서 루프가 끊어집니다!

---

## Ⅲ. STP의 상태 변화와 컨버전스 시간
스위치의 포트가 켜진다고 해서 바로 데이터를 쏴버리면 루프가 돌 수 있으므로, STP는 아주 조심스럽게 단계를 밟아 포트를 엽니다.

### 4단계 상태 전이 (Port States)
1. **차단 (Blocking, 20초)**: 데이터 안 받음. BPDU(인사말)만 듣고 대기.
2. **청취 (Listening, 15초)**: BPDU를 주고받으며 누가 대장인지, 어느 포트를 막을지(RP, DP 계산) 고민하는 시기.
3. **학습 (Learning, 15초)**: 경로는 정해졌고, 이제 MAC 주소를 테이블에 등록(학습)하기 시작함. (아직 데이터 전달은 안 함)
4. **전송 (Forwarding)**: 비로소 정상적으로 데이터를 주고받는 최종 상태.

* **컨버전스(Convergence) 시간**: 끊어졌던 네트워크가 우회 경로(Blocking 포트)를 열어서 다시 정상화되는 데 걸리는 총 시간. 오리지널 STP는 약 **30~50초**나 걸립니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[STP 포트 선정과 루프 방지 예시]

       (비용: 낮음)
      ┌─────────── [Root Switch (대장)] ────────┐ (비용: 낮음)
      │                                         │
 [ RP (루트포트) ]                          [ RP (루트포트) ]
[ Switch B ] ────── (100Mbps, 비용: 높음) ───── [ Switch C ]
           [ DP (지정포트) ]            [ Block (막힘!) ]

-> B와 C를 잇는 선은 느리기 때문에 C쪽 포트를 Block 하여 물리적 루프를 제거함.
```

📢 **섹션 요약 비유**: 스위치 3대를 삼각형으로 연결하면 길 잃은 자동차(프레임)가 영원히 뺑글뺑글 도는 회전교차로(루프)가 됩니다. 경찰(STP)이 와서 1짱(루트 브리지)을 정하고, 1짱에게 가는 가장 빠른 길(루트 포트)만 남겨둔 뒤, 쓸데없는 샛길(차단 포트)에는 콘을 세워 막아버리는 것이 STP의 역할입니다.
