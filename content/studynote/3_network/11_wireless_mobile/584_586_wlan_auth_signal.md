# 584-586. 무선 네트워크 보안 및 인프라 관리 (802.1X, dBm)

> **핵심 인사이트**: 무선망은 눈에 보이지 않기 때문에 '누가 들어오는지'를 철저히 검사(802.1X)하고, '얼마나 멀리 나가는지'를 정확한 숫자(dBm)로 측정하는 것이 운영의 핵심이다. 특히 802.1X는 기업망 보안의 문지기 역할을 수행한다.

---

## Ⅰ. 802.1X (포트 기반 네트워크 접근 제어)
사용자가 네트워크에 접속하기 전, 정당한 권한이 있는지 검증하는 표준 인증 체계입니다. (PNAC)

### 1. 구성 요소 (3총사)
1. **Supplicant (신청자)**: 인증을 받고자 하는 단말(노트북, 스마트폰).
2. **Authenticator (인증자)**: 문지기 역할을 하는 스위치나 무선 AP.
3. **Authentication Server (인증 서버)**: 실제 DB를 대조하는 **RADIUS** 서버.

### 2. EAP (Extensible Authentication Protocol)
* 802.1X 환경에서 다양한 인증 방식(ID/PW, 인증서, 토큰 등)을 수용하기 위한 유연한 통신 규격입니다.

---

## Ⅱ. 캡티브 포털 (Captive Portal)
호텔, 카페, 공항 Wi-Fi에 연결했을 때 자동으로 뜨는 **웹 기반 인증 페이지**입니다.

* **동작**: 사용자가 인터넷을 하려 하면 게이트웨이가 강제로 가로채서 로그인 페이지로 리다이렉트시킵니다.
* **장점**: 별도의 보안 프로그램(Supplicant) 설치 없이 웹 브라우저만으로 인증이 가능하여 게스트용 망에 적합합니다.

---

## Ⅲ. 안테나 및 신호 측정 지표 (dBm)
무선 전력의 세기를 나타내는 단위입니다.

* **dBm**: 1mW(밀리와트)를 기준으로 한 전력의 상대적 크기입니다. (로그 스케일)
  - **0 dBm** = 1mW
  - **3 dBm 증가** = 전력이 2배 증가
  - **10 dBm 증가** = 전력이 10배 증가
* **반값 전력각 (Half-power Beamwidth)**: 안테나 신호가 가장 센 방향의 절반 전력이 되는 지점 사이의 각도입니다. 안테나의 지향성을 판단하는 기준이 됩니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[802.1X 인증 프로세스]

 [ Supplicant ] <───( EAPOL )───> [ Authenticator ] <───( RADIUS )───> [ Auth Server ]
   (사용자)                         (무선 AP)                           (서버)
       │                                │                                 │
       └───── "저 들어가도 되나요?" ─────>  (문 잠금)                          │
                                        └───── "누군지 확인해줘!" ───────> (DB 대조)
                                        <───── "응, 들여보내줘"  ─────────┘
       <───── (문 열림, 통신 시작) ───────┘
```

📢 **섹션 요약 비유**: **802.1X**는 아파트 공동 현관에서 입주민 카드를 찍어야 엘리베이터를 탈 수 있게 하는 시스템입니다. **캡티브 포털**은 모델하우스 구경 갈 때 입구에서 이름과 전화번호를 적어야 입장시켜 주는 방명록과 같습니다. **dBm**은 목소리의 크기를 '데시벨'로 재는 것과 같아서, 거리가 멀어질수록 소리가 작아지는 것을 숫자로 꼼꼼히 기록하는 단위입니다.
