# 592-600. 위성 및 근거리 특수 통신 (GPS, UWB, RFID)

### # [위성 및 근거리 특수 통신]
#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 통신의 영역을 '공간(Space)'과 '거리(Distance)'로 확장하여, 전 지구적 위치 측위(GPS)와 센티미터급 초정밀 감지(UWB), 그리고 비접촉 식별(RFID/NFC)를 가능하게 하는 무선 기술 집합체.
> 2. **가치**: GPS는 절대 좌표를 제공하여 물류/모빌리티의 인프라가 되며, UWB는 정밀한 ToF(Time of Flight) 계산으로 공장 자동화와 보안 키의 신뢰성을 극대화함.
> 3. **융합**: 위성 통신과 5G NTN(Non-Terrestrial Network)의 융합, UWB와 IoT/RTLS(Real-time Locating System)의 결합으로 6G 시대의 '디지털 트윈' 기반을 구축함.

+++

### Ⅰ. 개요 (Context & Background)

**개념**
위성 및 근거리 특수 통신은 전통적인 '데이터 전달'을 넘어 사물이나 사람의 **위치(Location)**, **존재(Identification)**, **근접성(Proximity)**을 인식하는 기술이다. 이는 단순한 연결성을 넘어 Context-Aware(상황 인식형) 서비스의 핵심 감각 기관 역할을 한다.

**💡 비유**
만약 통신망이 거미줄이라면, 이 기술들은 거미줄에 달린 '예민한 털'과 같다. 끈(네트워크)은 연결되어 있지만, 털(특수 통신)을 통해 바람의 방향(위치)과 먹이의 접촉(식별)을 감지한다.

**등장 배경**
① **기존 한계**: Cell-based 기반의 기지국 위치 추신은 도심에서는 오차가 크고, 실내(Indoor)에서는 GPS 신호가 도달하지 않는 'Location Gap'이 존재함.
② **혁신적 패러다임**: 위성을 직접 활용한 삼각 측량법과, 초광대역 주파수를 활용한 레이더류 기술, 그리고 전자기 유도를 이용한 비접촉 ID 인식이 등장하여 공간의 모든 지점을 디지털화함.
③ **현재 비즈니스 요구**: 자율주행차의 센티미터급 위치 정확도, 공장 내 물류 추적, 그리고 간편 결제와 보안 키(Passkey)에 이르기까지 '정밀함'과 '보안'이 요구됨.

**📢 섹션 요약 비유**
GPS와 근거리 통신은 마치 **우주에서 내려다보는 '위성 카메라'와 눈앞의 사물을 만지는 '촉각'을 결합한 것과 같습니다.** 카메라가 전체 맵을 보여주는 동안, 손끝의 촉각(UWB/RFID)이 정확한 위치와 물체의 정체를 확인합니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

본 섹션에서는 GPS(위성), UWB(초정밀), RFID(식별) 세 가지 핵심 기술의 구조를 심층 분석한다.

#### 1. 구성 요소 및 비교

| 구분 | GPS (GNSS) | UWB (Ultra-Wideband) | RFID/NFC |
|:---:|:---|:---|:---|
| **핵심 요소** | 위성(Constellation), 수신기, 제어국 | Anchor(기준점), Tag(태그), TOF 엔진 | Tag(Inlay), Reader, Antenna |
| **동작 원리** | Trilateration (거리 차이에 의한 위치 계산) | ToF (Time of Flight) 및 TDoA | Inductive Coupling (유도 결합) / Backscatter |
| **핵심 프로토콜** | L1 C/A, L2C, L5 (CDMA 방식) | IEEE 802.15.4z (HRP) | ISO/IEC 14443 (NFC), EPCglobal (RFID) |
| **거리/정밀도** | 전 지구, 오차 5~10m (RTK 시 <1cm) | 10~50m, 오차 5~10cm (3차원 측위 가능) | 수cm~수m, 식별 ID 기반 |
| **전력 특성** | 수신기만 수신 (Downlink Only) | 양방향 통신, 낮은 전력 (Impulse Radio) | Tag는 무전원(Passive) 혹은 배터리형 |

#### 2. GPS 및 GNSS 항법 원리

GPS (Global Positioning System)는 여러 위성으로부터 송신된 신호의 도달 시간(Time of Arrival)을 측정하여 위치를 계산한다. 3개 위성으로는 2차원 위치(위도, 경도)를, 4개 위성으로는 고도까지 포함한 3차원 위치를 확인할 수 있다.

```ascii
   [위성 A]       [위성 B]       [위성 C]
      •             •             •
      | \           | \           |
      |   \         |   \         |  <-- 거리(R) = 시간(t) × 빛의 속도(c)
   R_A|     \   R_B|     \   R_C |
      |       \     |       \     |
      v         x   v         x   v
      ~~~~~~~~~~~~~~[수신기(나)]~~~~~~~~~~~~~~
      
   수학적 모델:
   (x-x1)² + (y-y1)² + (z-z1)² = c²(t-t1)²
   (x-x2)² + (y-y2)² + (z-z2)² = c²(t-t2)²
   ...
   => 연립 방정식 풀이를 통해 (x, y, z) 좌표 추출
```

**해설**:
위성은 자신의 궤도 위치와 정밀한 시간 정보(Ephemeris)를 실어서 L1(1575.42MHz) 대역의 신호를 지속적으로 송신한다. 수신기는 최소 4개 위성의 신호를 수신하여 각 위성까지의 '의사거리(Pseudorange)'를 계산하고, 이 교차점을 통해 절대 좌표를 도출한다. 실무에서는 건물 반사 등에 의한 Multipath 오차를 줄이기 위해 SK(Selectivity Availability) 기술이나 DGPS(Differential GPS) 보정 데이터를 활용한다.

#### 3. UWB (Ultra-Wideband) 정밀 측위

UWB는 3.1GHz~10.6GHz의 초광대역 대역을 사용하여 나노초(ns) 단위의 매우 짧은 펄스를 송수신한다.

```ascii
      [ Time Domain Signal ]
   ___|   |_____|   |_____|   |_____  <-- 일반 무선 (연속파)
   
   |_|_|_|_|_|_|_|_|_|_|_|_|_|_|_|_|  <-- UWB (짧은 펄스)
   
   [ 측위 방식: Two-Way Ranging (TWR) ]
   
   Device A (Poll) ----------> Device B
          ( t0 )                  ( t1 수신 )
   Device A (Resp) <---------- Device B
          ( t3 수신 )             ( t2 송신 )
          
   Flight Time = ( (t3 - t0) - (t2 - t1) ) / 2
   Distance = Flight Time * c
```

**해설**:
UWB는 넓은 대역폭을 활용해 벽이나 장애물을 통과하면서도 신호 형태가 급격히 변하는 특성을 이용한다. ToF(Time of Flight) 계산 시 매우 짧은 펄스 폭 덕분에 직접 경로(Direct Path)와 반사 경로(Reflected Path)를 구분 능력이 뛰어나 다중 경로 효과(Multipath Fading)에 강하다. 이를 통해 Wi-Fi나 Bluetooth보다 훨씬 높은 정밀도로 거리와 각도(AoA, Angle of Arrival)를 측정한다.

#### 4. RFID 및 NFC 동작 메커니즘

RFID는 무선 주파수를 이용해 정보를 비접촉으로 읽는 기술이며, NFC는 RFID 중 13.56MHz 대역을 사용하는 근거리 통신 표준이다.

```ascii
   [ RFID 작동 구조 (Inductive Coupling) ]
   
   <판독기(Reader)>             <태그(Tag)>
   [--- 안테나 코일 ---]       [--- 칩 + 코일 ---]
      |      ^  (자기장)           |      ^
      +------+--------------------+------+
       (교류 자계 생성)     (전자기 유도 기전력)
       
   1. Reader Antenna -> 교류 흐름 -> 자기장 형성
   2. Tag 코일이 자기장을 통과 -> 전기 유도 -> Tag 기동 (Passive)
   3. Tag -> Load Modulation으로 데이터 응답
```

**해설**:
태그는 내부 전원이 없는 수동형(Passive)이 대부분이며, 판독기가 방출하는 전파의 자기장 에너지를 흡수하여 회로를 구동한다. 이때 태그는 자신의 임피던스를 변화시키는 '부하 변조(Load Modulation)' 방식으로 판독기에 신호를 보낸다. NFC는 이 방식을 기반으로 P2P(Peer-to-Peer) 통신을 지원하여 스마트폰끼리 데이터를 주고받거나 결제를 수행한다.

**📢 섹션 요약 비유**
이 시스템들은 마치 **오케스트라의 지휘자**와 같습니다. **GPS**는 콘서트홀 전체의 설계도(좌표)를 보여주고, **UWB**는 각 악기의 센티미터 단위 이동을 실시간으로 추적하며, **RFID/NFC**는 악기마다 붙은 이름표를 확인하여 누가 무엇을 연주하는지 순식간에 파악합니다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

#### 1. 심층 기술 비교표

| 분석 지표 | GPS (GNSS) | Wi-Fi RTT | Bluetooth 5.x | UWB (IEEE 802.15.4z) |
|:---:|:---|:---|:---|:---|
| **정밀도** | 5~10m (정밀 보정 시 cm급) | 1~5m | 1~5m | **5~10cm** |
| **지연 시간** | 수초 (TTFF Cold Start) | 수십 ms | 수 ms | **수 ms 이하** |
| **벽 투과력** | 약함 (실내 불가) | 중간 | 낮음 | **높음** (급준한 펄스) |
| **보안성** | 스푸핑 가능성 있음 | 일반 | 일반 | **높음** (Distance Bounding) |
| **주요 용도** | 외부 내비게이션, 항공 | 실내 측위 (IPS) | 추적기, 비콘 | **자동차 키, 자산 추적, AR** |

#### 2. 기술 융합 시너지 (Convergence)

**① GPS + 5G NTN (Non-Terrestrial Network)**
전통적인 GPS는 데이터 통신이 불가능하다. 그러나 5G NTN 기술은 위성을 기지국처럼 활용하여 위치 추적과 동시에 고속 데이터를 제공한다. 이는 고평가 사용자의 미래 교통 시스템(HAPS, Drone)의 핵심이다.

**② UWB + AIoT (Artificial Intelligence of Things)**
단순히 위치를 찾는 것을 넘어, UWB 센서가 수집한 미세한 움직임(Motion Signature)을 AI가 학습하면 사용자의 행동 패턴(예: 낙상 감지, 도보 보행 인식)까지 인식하여 안전 서비스를 제공할 수 있다.

**③ RFID + 블록체인 (Supply Chain)**
RFID로 스캔된 물품의 이동 기록을 블록체인에 해시(Hash)값으로 저장하면, 물류 과정에서 위변조가 불가능한 '디지털 공급망 투명성'을 확보할 수 있다.

**📢 섹션 요약 비유**
각 기술은 **운송 수단의 계층과 같습니다.** **GPS**는 대륙을 건너는 '여객기'로 전체 경로를 잡고, **Wi-Fi/BLE**는 시내를 달리는 '버스'로 대략적인 구역을 커버하며, **UWB**는 건물 안에서 문 앞까지 데려다주는 '정밀 택배'입니다.

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 실무 시나리오 및 의사결정 매트릭스

| 시나리오 | 요구 사항 | 최적 기술 선정 | 판단 근거 |
|:---|:---|:---|:---|
| **① 자동차 스마트 키** | 원격 시동 도난 방지, 정밀도 | **UWB** | RSSI(신호 세기) 기반의 Relay 중계 공격 방지를 위해 Distance Bounding(물리적 거리 계산)이 필수적임. |
| **② 터미널 컨테이너 관리** | 대량, 수 m 거리, 낮은 단가 | **Passive RFID** | 충전이 필요 없는 수동형 태그가 비용 효율적이며, 게이트 통과 시 다중 인식(Anti-collision)이 우수함. |
| **③ 지하 매설물 탐색** | 비가시선, 지하 위치 | **UWB + Magnetic** | 지하에서는 GPS 신호 차단으로 인해 UWB의 지하 투과력을 활용하거나 자기장 변조를 이용해야 함. |

#### 2. 도입 체크리스트 (Technical & Operational)

*   **기술적 측면**
    *   [ ] **전파 환경 분석**: UWB/RFID 설치 시 금속 재질(Metal Interference)이 주변에 있는지 확인. (금속은 RF 차단)
    *   [ ] **Multipath 환경**: 좁은 실내에서 신호 반사로 인한 'Ghost 위치' 발생 가능성 검토.
    *   [ ] **Anchor 배치**: UWB 기준국의 배치 밀도(NLoS 환경 고려)가 3차원 측위 기준을 충족하는가?

*   **운영 및 보안적 측면**
    *   [ ] **스푸핑 방지**: 단순 신호 세기(RSSI)를 신뢰하여 잠금 해제를 하지 않도록 설계.
    *   [ ] **개인정보 이슈**: 위치 추적 데이터의 암호화 저장 및 전송 여부 확인.

#### 3. 안티패턴 (Anti-Patterns)
