# 561-563. 모바일 IP (Mobile IP)와 위치 관리

> **핵심 인사이트**: 인터넷의 기본 주소(IP)는 원래 한자리에 고정된 장비를 위해 만들어졌다. 단말이 여기저기 돌아다녀도(Mobility) 연결을 유지하려면, 본가(Home)에서 우편물을 받아 전달해 주는 대리인(Agent)과 새로운 주소(CoA)를 관리하는 특별한 체계가 필요하다.

---

## Ⅰ. 위치 관리 시스템 (HLR / VLR)
이동통신망 내부에서 사용자의 현재 위치를 기록하는 데이터베이스입니다.

1. **HLR (Home Location Register)**: 사용자가 가입한 본래 지역의 중앙 주소록. (영구적 정보 저장)
2. **VLR (Visitor Location Register)**: 사용자가 현재 여행 중인(방문한) 지역 기지국이 관리하는 임시 주소록.

---

## Ⅱ. Mobile IPv4 (MIPv4)
IP 주소를 바꾸지 않고 네트워크를 이동할 때 사용하는 기술입니다.

### 1. 주요 구성 요소
* **HA (Home Agent)**: 원래 내 네트워크(집)에 있는 라우터.
* **FA (Foreign Agent)**: 내가 지금 가 있는 타지 네트워크의 라우터.
* **CoA (Care-of-Address)**: 타지에서 임시로 부여받은 주소.

### 2. 삼각형 라우팅 (Triangular Routing) 문제
* **현상**: 외부 사람이 나에게 데이터를 보내면 [송신자 $\rightarrow$ HA $\rightarrow$ FA $\rightarrow$ 나] 순으로 뱅 돌아갑니다. 반면 나는 외부 사람에게 직접 쏩니다.
* **단점**: 경로가 비효율적이고 지연이 발생합니다.

---

## Ⅲ. Mobile IPv6 (MIPv6)
IPv6 환경에서는 MIPv4의 단점을 획기적으로 개선했습니다.

* **FA 불필요**: IPv6의 자동 주소 설정(SLAAC) 기능 덕분에 FA 도움 없이도 스스로 CoA를 만듭니다.
* **경로 최적화 (Route Optimization, RO)**: 송신자에게 내 현재 CoA를 직접 알려줍니다. 이후부터는 HA를 거치지 않고 **1:1로 직접 통신**하여 삼각형 라우팅 문제를 해결합니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[Mobile IP의 동작 원리]

  (1) 편지 도착 ───> [ Home Agent ] (본가)
                           │
                           │ (2) "얘 지금 부산(CoA) 가 있어!" (Tunneling)
                           ▼
                     [ Foreign Agent ] (부산 지점)
                           │
                           │ (3) 최종 전달
                           ▼
                        [ Mobile Node ] (나)
```

📢 **섹션 요약 비유**: **Mobile IP**는 '우편물 전송 서비스'와 같습니다. 내가 고향(Home)을 떠나 제주도(Foreign)로 한 달 살기를 갔을 때, 고향 우체국(HA)에 "내 앞으로 오는 편지는 제주도 펜션(CoA)으로 보내줘"라고 신청하는 것입니다. **삼각형 라우팅**은 답장은 제주도에서 바로 쓰는데 받는 편지는 서울을 거쳐 와야 하는 비효율이고, **MIPv6**는 친구에게 "나 이제 제주도에 있으니 이리로 바로 편지 써"라고 새 주소를 직접 알려주는 스마트한 방식입니다.
