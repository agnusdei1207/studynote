# 113-115. 통제된 접근(Controlled Access) 매체 제어 방식

> **핵심 인사이트**: 임의 접근(CSMA/CD 등) 방식은 부하가 적을 때는 빠르지만, 트래픽이 몰리면 충돌로 인해 네트워크가 마비됩니다. 이를 해결하기 위해 단말들이 서로 '순서를 정해서' 전송하는 통제된 접근(Controlled Access) 방식이 등장했습니다.

---

## Ⅰ. 예약 방식 접속 (Reservation Access)
단말이 데이터를 전송하기 전에 "나 데이터를 보낼 거야!" 라고 미리 자리를 맡아두는 방식입니다.

* **동작 원리**: 
  1. 시간축은 **예약 프레임(Reservation Frame)**과 **데이터 프레임**으로 나뉩니다.
  2. $N$개의 단말이 있다면, 예약 프레임 안에는 $N$개의 미니 슬롯(Mini-slot)이 존재합니다.
  3. 데이터를 보내고 싶은 단말은 자신의 순번에 해당하는 미니 슬롯에 '1'을 채워 예약을 겁니다.
  4. 예약 기간이 끝나면, 예약을 걸어둔 단말들만 순서대로 데이터 프레임을 전송합니다.
* **장점**: 충돌이 전혀 발생하지 않아 네트워크 효율이 높습니다.
* **단점**: 보낼 데이터가 없어도 예약 프레임을 주고받는 데 대역폭이 낭비될 수 있습니다.

---

## Ⅱ. 폴링 접속 (Polling Access)
네트워크에 **주 제어기(Primary Station)**가 존재하여, 다른 모든 종속 단말(Secondary Stations)의 송수신 권한을 중앙 통제하는 방식입니다.

### 1. 폴(Poll) 기능
* 주 제어기가 종속 단말에게 **"너 보낼 데이터 있니?"** 하고 물어보고 송신 권한을 넘겨주는 기능입니다.
* 종속 단말은 주 제어기의 Poll 메시지를 받아야만 데이터를 전송할 수 있습니다. 보낼 게 없으면 NAK(부정 응답)를 보냅니다.

### 2. 셀렉트(Select) 기능
* 주 제어기가 종속 단말에게 **"내가 너한테 보낼 데이터가 있는데, 받을 준비 됐니?"** 하고 물어보는 기능입니다.
* 단말이 ACK를 보내면 주 제어기가 데이터를 전송합니다.

* **장점**: 충돌이 완벽히 방지되고, 우선순위 부여가 쉽습니다.
* **단점**: 주 제어기가 고장 나면 전체 네트워크가 마비되며(SPOF), 폴링 메시지 자체가 상당한 오버헤드(Overhead)를 유발합니다. 블루투스(Bluetooth)의 PCF(Point Coordination Function)가 이 방식을 사용합니다.

---

## Ⅲ. 토큰 패싱 (Token Passing)
"토큰(Token)"이라는 특별한 형태의 작은 제어 프레임을 가진 단말만이 데이터를 전송할 수 있는 분산 제어 방식입니다.

### 1. 토큰 링 (Token Ring, IEEE 802.5)
* **구조**: 단말들이 물리적/논리적으로 원형(Ring)으로 연결되어 있습니다.
* **동작**: 
  - 평소에는 빈 토큰이 링을 빙글빙글 돕니다.
  - 전송할 데이터가 있는 단말은 토큰을 붙잡아(Seize) 데이터 프레임으로 바꾼 뒤 목적지로 보냅니다.
  - 목적지 단말은 데이터를 복사하고 프레임에 '수신 완료' 표시를 한 뒤 링에 다시 흘려보냅니다.
  - 송신 단말은 한 바퀴 돌아온 프레임을 회수하고 다시 빈 토큰을 링에 방출합니다.

### 2. 토큰 버스 (Token Bus, IEEE 802.4)
* **구조**: 물리적으로는 일자형 버스(Bus) 토폴로지지만, 내부적으로는 단말끼리 논리적 링(Logical Ring) 순서를 만들어 토큰을 주고받습니다.
* 주로 공장 자동화(MAP) 등에서 전송 시간의 예측 가능성(지연 시간 보장)이 필요할 때 사용되었습니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[통제된 접근 방식(Controlled Access) 비교]

1. Reservation (예약)
   [ R1 | R2 | R3 ] -> [ Data 1 ] -> [ Data 3 ]
   (1번, 3번 단말이 미니 슬롯에 예약을 걸어 순서대로 전송)

2. Polling (폴링)
   [ Primary Controller ]
     ├── (Poll: 보낼거 있니?) ──> [ Station A ] -> (Data 응답)
     └── (Select: 받을래?) ───> [ Station B ] -> (ACK 후 수신)

3. Token Passing (토큰 패싱)
   [ Station A ] --(Token)--> [ Station B ] --(Token+Data)--> [ Station C ]
   (토큰을 가진 자만이 발언권을 얻는 구조)
```

📢 **섹션 요약 비유**:
* **예약 방식**: 식당 입구 대기 명단에 이름을 적은 사람만 순서대로 밥을 먹는 시스템입니다.
* **폴링 방식**: 선생님(주 제어기)이 학생들(단말)의 이름을 한 명씩 부르며 발표할 기회를 주는 시스템입니다.
* **토큰 패싱**: "발언권 마이크(토큰)"를 옆 사람에게 계속 넘기다가, 할 말이 있는 사람만 마이크를 쥐고 말한 뒤 다시 마이크를 넘기는 둥글게 앉은 회의 방식입니다.
