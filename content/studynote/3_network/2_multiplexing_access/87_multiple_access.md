## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 접속은 여러 사용자가 하나의 공유 전송 자원을 충돌 없이 나눠 쓰도록 시간, 주파수, 코드, 공간 차원을 분할하는 MAC (Medium Access Control) 설계 원리다.
> 2. **가치**: 동일한 무선 스펙트럼과 유선 매체 위에서 더 많은 사용자를 수용하고, 충돌·지연·공정성 문제를 제어함으로써 네트워크의 수용량과 서비스 품질을 결정한다.
> 3. **융합**: 다중화가 "한 회선에 여러 흐름을 싣는 방법"이라면, 다중 접속은 "여러 사용자가 그 회선 사용권을 어떻게 나눌지"를 정의하는 운영 규칙이다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 다중 접속은 하나의 공유 채널에 여러 단말이 동시에 접근하려 할 때, 누가 언제 어떤 자원 조각을 사용할지를 규칙화한 접근 제어 방식이다. 전송 매체가 한정되어 있기 때문에, 접속 제어가 없으면 충돌과 재전송이 반복되며 시스템 효율이 급격히 떨어진다.
- **💡 비유**: 한 개의 회의실을 여러 팀이 쓰는 상황과 같다. 시간표를 짜서 쓰면 TDMA, 층을 나눠 쓰면 FDMA, 각 팀이 다른 암호를 알고 같은 시간에 말하면 CDMA에 가깝다.
- **등장 배경**:
  1. 통신망이 단일 단말 전용에서 다수 사용자 공유형으로 바뀌었다.
  2. 무선 채널은 대역폭이 비싸고 희소해, 스펙트럼 효율 극대화가 필수 과제가 됐다.
  3. 모바일 세대가 진화할수록 사용자 수, 이동성, 트래픽 유형이 다양해져 접속 방식도 복합화됐다.
- **📢 섹션 요약 비유**: 한정된 운동장을 여러 반이 같이 써야 할 때, 질서 있는 사용 규칙이 없으면 결국 아무도 제대로 뛰지 못하는 것과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

다중 접속은 공유 자원을 어떤 축으로 잘라내는지에 따라 구조가 달라진다.

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **공유 매체** | 전체 자원 풀 | 채널, 주파수, 시간 슬롯 제공 | RF Channel, Ethernet | 공동 운동장 |
| **접속 규칙** | 사용자 분배 | 시간/주파수/코드/공간 분할 | TDMA, FDMA, CDMA, SDMA | 사용 시간표 |
| **스케줄러** | 우선순위 제어 | QoS와 혼잡 상태 반영 | MAC Scheduler | 경기 진행 요원 |
| **동기화 기능** | 충돌 감소 | 슬롯 경계·프레임 타이밍 정렬 | Timing Advance | 시작 총 |
| **피드백 경로** | 품질 보정 | 전력 제어, 재전송, 링크 적응 | HARQ, Power Control | 심판 판정 |

```text
┌──────────────────────────────────────────────────────────────┐
│             다중 접속의 자원 분할 관점                        │
├──────────────────────────────────────────────────────────────┤
│  공유 채널                                                   │
│   ├─ 주파수로 분할 -> FDMA                                   │
│   ├─ 시간으로 분할   -> TDMA                                 │
│   ├─ 코드로 분할   -> CDMA                                   │
│   ├─ 부반송파+시간   -> OFDMA                                │
│   └─ 공간 빔으로 분할 -> SDMA / Massive MIMO                 │
└──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 다중 접속의 핵심은 "한 자원을 여러 사용자가 동시에 쓰게 하되, 수신기가 서로를 구분할 수 있게 만드는 것"이다. FDMA는 서로 다른 주파수 대역을 주므로 간단하지만 유휴 대역 낭비가 생기고, TDMA는 시간 동기화가 중요하다. CDMA는 같은 시간과 주파수를 쓰되 서로 다른 확산 코드를 사용해 사용자 신호를 분리하고, OFDMA는 부반송파 묶음과 시간 슬롯을 함께 할당해 현대 셀룰러 시스템의 유연성을 확보한다.

```text
┌──────────────────────────────────────────────────────────────┐
│          MAC 계층에서 본 다중 접속 제어 흐름                  │
├──────────────────────────────────────────────────────────────┤
│ User Requests -> Scheduler -> Resource Allocation -> TX/RX   │
│        ^                   |                       |          │
│        |                   v                       v          │
│   Buffer Status      QoS Priority           ACK / CQI         │
└──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 실제 네트워크에서는 사용자에게 고정 자원을 영구 배정하지 않는다. 버퍼 상태, 채널 품질 지표인 CQI (Channel Quality Indicator), 서비스 우선순위가 스케줄러로 들어오고, MAC은 매 프레임 또는 슬롯마다 적절한 자원 조각을 다시 할당한다. 그래서 다중 접속은 단순한 이론 분류가 아니라, 실시간 스케줄링 문제와 직접 연결된다.

1. 단말이 접속 요청 또는 전송 요구를 올린다.
2. 기지국 또는 스위치가 현재 채널 상태와 우선순위를 평가한다.
3. 자원을 주파수·시간·코드·공간 중 하나 또는 복합 형태로 배정한다.
4. 수신기는 배정 규칙에 맞춰 복조·복호를 수행한다.
5. 충돌, 오류, 혼잡 상태에 따라 다음 자원 배분 정책을 보정한다.

- **📢 섹션 요약 비유**: 공연장에서 좌석, 입장 시간, 입장권 색깔을 함께 관리해야 관객이 몰려도 질서가 유지되는 것처럼, 다중 접속도 여러 차원의 표식을 조합해 혼잡을 다스립니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

| 방식 | 분할 축 | 장점 | 약점 | 대표 적용 |
|:---|:---|:---|:---|:---|
| **FDMA** | 주파수 | 구현 단순, 지속적 전송 유리 | 가드밴드 오버헤드 | 초기 셀룰러 |
| **TDMA** | 시간 | 자원 활용 제어 쉬움 | 정밀 동기화 필요 | GSM |
| **CDMA** | 코드 | 주파수 재사용도 높음 | 전력 제어 복잡 | 3G |
| **OFDMA** | 시간+부반송파 | 유연한 스케줄링 | PAPR, 복잡도 증가 | LTE, 5G |

| 관점 | 다중화 (Multiplexing) | 다중 접속 (Multiple Access) |
|:---|:---|:---|
| **질문** | 여러 흐름을 한 매체에 어떻게 실을까 | 여러 사용자가 매체를 어떻게 나눌까 |
| **주체** | 시스템 설계자 | 사용자/단말/기지국 |
| **대표 예시** | FDM, TDM, WDM | FDMA, TDMA, CDMA, OFDMA |
| **연결 지점** | 물리 계층 중심 | MAC 계층 중심 |

- **과목 융합 관점**:
  1. **운영체제 스케줄링**: CPU 스케줄러가 프로세스에 시간 조각을 나누듯, 다중 접속도 채널 자원을 사용 목적에 따라 배분한다.
  2. **보안**: 무선 다중 접속은 접속 제어 실패 시 재밍, 스푸핑, DoS 공격에 취약해져 인증과 무선 자원 보호가 함께 설계돼야 한다.
- **📢 섹션 요약 비유**: 버스 좌석 배치, 탑승 순서, 승차권 종류를 각각 비교해보면 어느 방식이 출퇴근 시간에 더 강한지 바로 보이는 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

- **시나리오 1: 도심 LTE/5G 셀 혼잡**
  - 문제: 고밀도 트래픽으로 사용자 체감 속도가 급감한다.
  - 판단: OFDMA 기반 스케줄링과 Massive MIMO를 결합해 주파수뿐 아니라 공간축까지 동시에 분할해야 한다.
- **시나리오 2: 산업용 사설망**
  - 문제: 지연이 민감한 제어 트래픽과 대용량 영상 트래픽이 공존한다.
  - 판단: 정시성이 중요한 제어 채널에는 예약형 자원 배정을 우선 적용하고, 나머지 트래픽은 가변 스케줄링으로 처리한다.
- **시나리오 3: IoT 대량 접속**
  - 문제: 저전력 단말 수천 개가 간헐적으로 짧은 패킷을 보낸다.
  - 판단: 전통적 인간 중심 접속 방식 대신 경량 랜덤 액세스와 그룹 기반 스케줄링을 고려해야 한다.

체크리스트:

1. 시스템의 핵심 제약이 대역폭, 지연, 단말 수 중 무엇인가.
2. 동기화 비용을 감당할 수 있는가.
3. 채널 품질 편차가 큰 환경인가.
4. 공정성보다 총 처리량이 더 중요한가.

- **안티패턴**: 단순 평균 처리량만 보고 다중 접속 방식을 고르면, 셀 가장자리 사용자나 저전력 단말이 계속 밀려 실사용 품질이 붕괴한다.
- **📢 섹션 요약 비유**: 운동장 가운데 학생들만 빠르게 뛰게 하면 기록은 좋아 보여도, 바깥쪽 학생은 계속 출발도 못 하는 상황과 같습니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

| 항목 | 기대 효과 |
|:---|:---|
| **수용량 확대** | 동일 스펙트럼에서 더 많은 사용자 지원 |
| **품질 제어** | 우선순위 기반 서비스 품질 보장 |
| **유연성 향상** | 트래픽 성격에 따라 자원 동적 배정 |

- **미래 전망**:
  1. 6G는 NOMA, RIS (Reconfigurable Intelligent Surface), AI 스케줄링을 결합해 초고밀도 접속을 목표로 한다.
  2. 위성-지상 통합망은 이동성과 빔포밍을 반영한 공간 기반 다중 접속 비중이 커질 가능성이 높다.
  3. 머신 타입 통신 증대로 "사람 중심 접속"에서 "기계 군집 접속"으로 최적화 축이 이동한다.
- **표준 연결**: 3GPP LTE/5G NR, IEEE 802.11 OFDMA, ITU-R 스펙트럼 효율 가이드.
- **📢 섹션 요약 비유**: 앞으로의 다중 접속은 단순 줄 세우기가 아니라, 사람마다 이동 속도와 목적지를 읽고 자동으로 가장 빠른 문을 열어주는 공항 운영 시스템에 가까워집니다.
