# 다중화 (Multiplexing) 개념 및 특징

> 1. **본질**: 고속의 전송로 대역폭을 저속의 여러 독립적인 채널로 분할하여 다수의 신호를 동시에 전송함으로써, 물리적 회선 자원의 이용 효율을 극대화하는 자원 공유 기술이다.
> 2. **가치**: 회선 가설 비용을 획기적으로 절감하고 통신 인프라의 수용량을 증대시키며, 현대 대규모 통신망에서 경제성과 서비스 범용성을 담보하는 핵심 아키텍처로 작동한다.
> 3. **융합**: 전송로 공유의 '다중화(Multiplexing)' 개념은 다수 사용자의 접속 권한을 제어하는 '다중 접속(Multiple Access)' 기술로 확장되며, 클라우드 컴퓨팅의 가상화(Virtualization) 자원 분배 원리와 일맥상통한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: **다중화(Multiplexing)**는 하나의 넓은 도로(고속 회선)를 여러 대의 차(저속 채널)가 동시에 달릴 수 있도록 차선을 나누거나 시간을 쪼개어 사용하는 기술이다. 송신단에서는 **Multiplexer(MUX)**가 여러 신호를 하나로 합치고, 수신단에서는 **Demultiplexer(DEMUX)**가 이를 다시 각각의 신호로 분리한다.

- **💡 비유**: 한 개의 큰 피자를 여러 조각으로 나누어 여러 명에게 나눠주는 것과 같습니다. 피자(대역폭)는 하나지만, 자르는 방법(시간, 주파수 등)에 따라 수많은 사람이 굶주림(통신 요구)을 해결할 수 있습니다.

- **등장 배경**:
  1. **경제적 한계**: 사용자마다 개별적인 통신 선로를 깔아주는 것은 비용 면에서 불가능하다.
  2. **자원 활용 불균형**: 개별 사용자가 차지하는 대역폭은 매우 작지만, 전송로(광섬유 등)의 잠재 용량은 매우 크다. 이 차이를 메워야 했다.
  3. **통합 서비스 요구**: 음성, 데이터, 영상 등 서로 다른 성격의 정보를 하나의 망에서 동시에 처리해야 할 필요성이 대두되었다.

- **📢 섹션 요약 비유**: 좁은 입구를 통과하기 위해 줄을 서거나 어깨를 비껴가는 질서를 만들어, 제한된 공간을 100% 활용하는 "공간과 시간의 질서 정연한 재배치"입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 다중화 시스템의 기본 구조

송수신 양단에 MUX와 DEMUX가 짝을 이루어 동작한다.

```text
 [ 다중화 기본 구성도 ]
 
 저속 채널들                                           저속 채널들
  (n개)                                                 (n개)
   ──▶ [ 1 ] ──┐                                 ┌──▶ [ 1 ]
   ──▶ [ 2 ] ──┼──▶ [ Multiplexer ] ──▶ [ DEMUX ] ──┼──▶ [ 2 ]
   ──▶ [ ... ] ──┘      (MUX)             (복구)    └──▶ [ ... ]
   ──▶ [ n ] ──┘                                 └──▶ [ n ]
                      <── 고속 공유 회선 ──▶
```

**[다이어그램 해설]** 다수의 저속 장치들이 내보내는 신호를 **MUX**가 하나의 고속 전송로에 싣는다. 이때 어떤 사용자의 데이터인지 구분하기 위해 주파수, 시간, 파장, 코드 등의 기준을 사용한다. 수신측의 **DEMUX**는 송신측과 동일한 규칙(동기화)을 적용하여 신호를 분리해 각 목적지로 정확히 배달한다.

---

### 다중화 기술의 4대 분류

| 방식 | 분할 기준 | 주요 특징 | 비유 |
|:---|:---|:---|:---|
| **FDM (주파수 분할)** | 주파수 (Hz) | 아날로그 방식, 보호 대역 필요 | 라디오 채널 나누기 |
| **TDM (시분할)** | 시간 (sec) | 디지털 방식, 타임슬롯 할당 | 차례대로 말하기 |
| **WDM (파장 분할)** | 빛의 색깔 ($\lambda$) | 광통신 전용, 초고속 대용량 | 프리즘으로 무지개 만들기 |
| **CDM (코드 분할)** | 암호 코드 (Code) | 보안성 우수, 같은 주파수/시간 공유 | 서로 다른 언어로 대화하기 |

- **📢 섹션 요약 비유**: 목적지(수신)는 하나지만 가는 방법은 다양합니다. 차선을 나누든(FDM), 출발 시간을 맞추든(TDM), 자동차 색깔을 바꾸든(WDM) 목적은 "도로 점유 최적화"로 수렴합니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 다중화(Multiplexing) vs 다중 접속(Multiple Access)

두 용어는 혼용되기 쉬우나 계층적·운영적 차이가 존재한다.

| 비교 항목 | 다중화 (Multiplexing) | 다중 접속 (Multiple Access) |
|:---|:---|:---|
| **주요 관점** | **물리적 전송로 효율** (Point-to-Point) | **매체 접근 제어** (Multi-point) |
| **작동 계층** | 물리 계층 (L1) 위주 | 데이터 링크 계층 (L2/MAC) 위주 |
| **제어 주체** | 고정된 장비 (MUX) | 분산된 다수의 사용자 (Node) |
| **예시** | T1/E1 전송 장비, WDM | LTE OFDMA, Wi-Fi CSMA/CA |

### 과목 융합 관점

- **운영체제 (OS)**: 입출력 다중화(I/O Multiplexing) 기술(예: `select`, `poll`, `epoll`)은 하나의 프로세스가 수천 개의 소켓 연결을 동시에 관리할 수 있게 하여 웹 서버의 성능을 비약적으로 향상시킨다.
- **컴퓨터 구조 (CA)**: CPU 내부의 버스 다중화기는 여러 레지스터의 출력을 ALU로 전달하거나, 주소 버스와 데이터 버스를 시간차로 공유(Multiplexed Bus)하여 핀 개수를 줄이고 칩 크기를 최적화한다.

- **📢 섹션 요약 비유**: 다중화가 "하나의 선로를 나눠 쓰는 기계적 장치"라면, 다중 접속은 "누가 먼저 선로를 쓸지 결정하는 민주적 규칙"입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 데이터센터 간 전용선 비용 절감**: 두 데이터센터 사이에 전송해야 할 데이터가 늘어나 신규 광케이블 포설이 필요하다. 기술사는 케이블을 새로 까는 대신 기존 선로에 **DWDM(고밀도 파장 분할 다중화)** 장비를 도입한다. 하나의 광섬유에 80개 이상의 서로 다른 파장을 실어 보냄으로써, 물리적 공사 없이 회선 용량을 80배 증설하는 마법을 부린다.

2. **시나리오 — 사내 전화망(PBX) 채널 부족**: 사무실 내 통화자가 늘어 회선이 부족하다. 기술사는 아날로그 교환기를 **디지털 TDM 방식**으로 전환한다. 125$\mu$s의 짧은 시간을 24개 또는 30개로 쪼개어(Time Slot), 하나의 구리선 쌍으로 30명이 동시에 통화할 수 있는 인프라를 구축한다.

### 도입 체크리스트
- **매체의 물리적 특성**: 사용하려는 케이블이 광섬유인가? (WDM), 구리선인가? (TDM/FDM), 무선인가? (CDM/OFDM)
- **정적 vs 동적 할당**: 트래픽이 일정한가(Synchronous TDM), 아니면 들쭉날쭉한가(Statistical TDM)?
- **간섭 방지(Guard Band/Interval)**: 채널 간의 충돌을 막기 위한 여유 공간을 충분히 확보했는가?

### 안티패턴
- **비효율적 자원 할당**: 트래픽이 거의 없는 사용자에게 고정된 타임슬롯(Sync TDM)이나 주파수(FDM)를 할당하여 아까운 자원을 놀리는 것. (동적 할당 방식 고려 필수)
- **보호 대역(Guard Band) 누락**: 대역폭을 아끼겠다고 주파수 채널을 너무 붙여 설계하면, 인접 채널 간섭(ACI)으로 인해 통신 전체가 오염된다.

- **📢 섹션 요약 비유**: 기술의 선택은 곧 경제의 선택입니다. 내 지갑(예산)과 도로(매체) 상황에 맞는 최적의 차선 나누기 전략이 필요합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 전 (Single Channel) | 도입 후 (Multiplexing) | 개선 효과 |
|:---|:---|:---|:---|
| **설비 비용** | n개 회선 설치 (천문학적) | 1개 회선으로 대체 | CAPEX **90% 이상** 절감 |
| **운영 편의** | n개 경로 각각 관리 | 단일 고속 경로 통합 관리 | OPEX 및 장애 포인트 감소 |
| **서비스 확장** | 회선 증설 시마다 공사 | 논리적 채널 추가로 해결 | 서비스 배포 속도(Agility) 향상 |

### 미래 전망
미래의 다중화는 단순한 2차원(시간, 주파수) 분할을 넘어, 공간의 차원까지 활용하는 **SDM(Space Division Multiplexing)**과 빛의 궤도 각운동량을 이용하는 **OAM(Orbital Angular Momentum)** 다중화로 진화하고 있다. 또한 AI가 네트워크 부하를 예측하여 다중화 자원을 실시간으로 재배치하는 **초지능형 동적 다중화**가 6G 통신의 핵심이 될 것이다.

### 참고 표준
- **ITU-T G.700 시리즈**: 디지털 다중화 계층(TDM) 표준
- **ITU-T G.694.1**: DWDM 파장 그리드 규격

- **📢 섹션 요약 비유**: 작게 나누어 크게 쓰는 지혜. 다중화는 한정된 지구의 자원을 인류의 무한한 정보 욕구와 연결하는 가장 아름다운 타협점입니다.

## 👶 어린이를 위한 3줄 비유 설명
1. **다중화**는 좁은 복도에서 여러 명이 부딪히지 않고 지나가도록 규칙을 정하는 거예요.
2. 1번 친구가 먼저 지나가고 그다음 2번 친구가 가는 식으로 시간을 나누거나(TDM), 복도에 왼쪽 길과 오른쪽 길을 그려서 공간을 나누는 것(FDM)과 같죠.
3. 이렇게 약속을 잘 정하면, 한 개의 복도로도 수많은 친구가 동시에 안전하게 이동할 수 있답니다!
