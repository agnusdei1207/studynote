# 108-110. 무선 LAN 매체 제어: CSMA/CA와 히든 노드 문제

> **핵심 인사이트**: 무선(Wireless) 환경에서는 신호 감쇠가 심해 충돌을 직접 탐지(CD)할 수 없으므로, 데이터 전송 전 사전에 충돌을 회피(CA)하는 전략을 사용하며, 보이지 않는 단말 간의 간섭 문제는 RTS/CTS 교환으로 해결한다.

---

## Ⅰ. CSMA/CA (Collision Avoidance)
**CSMA/CA (충돌 회피 반송파 감지 다중 접속)**는 IEEE 802.11 무선 LAN(Wi-Fi)에서 사용하는 표준 매체 접근 제어 방식입니다.
* **등장 배경**: 무선 환경에서는 신호의 송신 전력이 수신 전력보다 압도적으로 커서, 자신이 송신하는 동안 남의 신호(충돌)를 감지할 수 없습니다. (Collision Detection 불가)
* **기본 원리**: 충돌이 발생한 "후"에 처리하는 것이 아니라, 충돌이 발생하기 "전"에 미리 대기하고 회피하는 전략을 취합니다.

### CSMA/CA의 3가지 회피 전략
1. **IFS (InterFrame Space) 대기**: 채널이 비어 있다고 판단되더라도 즉시 전송하지 않고, 짧은 시간(IFS) 동안 추가로 대기하여 우선순위가 높은 프레임이 먼저 지나가도록 양보합니다.
2. **백오프 알고리즘 (Contention Window)**: IFS 대기 후에도 무작위 시간(Random Backoff)을 더 기다려, 동시에 여러 단말이 전송을 시작하는 것을 막습니다.
3. **ACK (Acknowledge) 수신**: 충돌을 감지할 수 없으므로, 수신 측으로부터 잘 받았다는 응답(ACK)을 받아야만 전송 성공으로 간주합니다.

---

## Ⅱ. 은닉 노드 문제 (Hidden Node Problem)
무선 통신 커버리지의 한계로 인해 발생하는 대표적인 충돌 문제입니다.

* **상황**: 단말 A와 단말 C는 AP(Access Point)인 B와 통신할 수 있지만, A와 C는 서로의 전파가 닿지 않는 거리에 있습니다.
* **문제 발생**: A가 B로 데이터를 보내는 중일 때, C는 채널이 비어 있다고(Carrier Sense) 착각하여 B로 데이터를 보냅니다. 결과적으로 B 지점에서 A와 C의 데이터가 충돌하게 됩니다.
* **요약**: "서로의 존재를 모르는(Hidden) 두 노드가 하나의 타겟(AP)에 동시에 전송하여 충돌하는 현상"

---

## Ⅲ. RTS / CTS 교환 메커니즘
은닉 노드 문제를 해결하기 위해 채널을 사전에 "예약"하는 방법입니다. CSMA/CA의 선택적 확장 기능으로 사용됩니다.

### 동작 절차
1. **RTS (Request To Send)**: 송신자(A)가 데이터를 보내기 전, 수신자(B)에게 "나 데이터 보낼게!"라는 짧은 제어 프레임을 보냅니다.
2. **CTS (Clear To Send)**: B가 A에게 "알겠어, 보내!"라는 응답을 브로드캐스트(모두에게 전송)합니다.
3. **NAV (Network Allocation Vector) 설정**: B의 CTS 신호를 들은 C(은닉 노드)는 "아, A가 데이터를 보낼 예정이구나" 하고 자신이 전송을 멈출 타이머(NAV)를 설정합니다.
4. **데이터 전송 및 ACK**: 채널이 예약된 상태에서 A가 데이터를 안전하게 전송하고 B로부터 ACK를 받습니다.

---

## Ⅳ. 노출 노드 문제 (Exposed Node Problem)
은닉 노드와 반대로, 불필요하게 전송을 양보하여 대역폭이 낭비되는 문제입니다.

* **상황**: B가 A에게 데이터를 전송하고 있습니다. 이때 B 옆에 있는 C가 D에게 데이터를 보내려 합니다.
* **문제 발생**: C는 B의 신호를 감지하고 "아, 채널이 사용 중이구나" 하고 대기합니다. 하지만 B의 전파는 D에게 닿지 않으므로, C가 D에게 보내더라도 아무런 충돌이 발생하지 않습니다. 
* **결과**: 전송해도 되는 상황임에도, 다른 노드의 통신에 노출(Exposed)되어 스스로 전송을 억제하는 효율성 저하 현상입니다. RTS/CTS를 통해 어느 정도 완화할 수 있습니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[CSMA/CA와 RTS/CTS 동작 흐름]

[Sender]                                [Receiver]                             [Hidden Node]
   |                                        |                                        |
   |----(1) RTS (Request To Send) --------->|                                        |
   |                                        |----(2) CTS (Clear To Send) ----------->|
   |                                        |                                [NAV 설정 / 대기!]
   |<---(2) CTS (Clear To Send) ------------|                                        |
   |                                        |                                        |
   |----(3) Data Transmission ------------->|                                        |
   |                                        |                                        |
   |<---(4) ACK (Acknowledge) --------------|                                        |
```

📢 **섹션 요약 비유**: 무선 환경은 눈을 가리고 대화하는 것과 같아서 서로의 목소리가 안 들릴 수 있습니다(은닉 노드). 그래서 진짜 말을 시작하기 전에 "나 말할게!"(RTS)라고 외치고, 방장이 "그래 말해!"(CTS)라고 마이크를 잡아주면, 그때 다른 사람들은 조용히 기다려주는(NAV) 것이 CSMA/CA의 핵심입니다.
