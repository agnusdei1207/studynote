## 핵심 인사이트 (3줄 요약)
> 1. **본질**: FDMA는 전체 주파수 대역을 여러 개의 좁은 채널로 나누고, 각 사용자에게 서로 다른 주파수 슬롯을 배정하는 다중 접속 방식이다.
> 2. **가치**: 사용자 간 시간 동기화 부담이 낮고 지속적 전송에 유리하지만, 가드밴드와 유휴 채널 때문에 스펙트럼 효율은 상대적으로 낮다.
> 3. **융합**: 초기 이동통신, 위성 통신, 일부 무선 백홀 구조에서 여전히 기본 개념으로 남아 있으며 OFDMA 이해의 출발점이 된다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: FDMA는 각 사용자에게 고정된 주파수 대역을 나눠줘 동시에 전송하게 하는 방식이다. 사용자는 같은 시간에 송신해도 다른 주파수에 있으므로 수신기가 분리할 수 있다.
- **💡 비유**: 여러 라디오 방송국이 각자 다른 주파수에서 방송하는 구조와 같다.
- **등장 배경**:
  1. 초기 무선 시스템은 구현 복잡도가 낮은 방식이 필요했다.
  2. 지속적 음성 통화처럼 일정 대역을 오래 쓰는 트래픽에 적합했다.
  3. 디지털 스케줄링이 발전하기 전에는 고정 채널 방식이 운영과 관리에 유리했다.
- **📢 섹션 요약 비유**: 친구마다 각자 전용 확성기 채널을 하나씩 주는 단순하지만 이해하기 쉬운 방식입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

| 요소명 | 역할 | 내부 동작 | 관련 기술 | 비유 |
|:---|:---|:---|:---|:---|
| **전체 대역폭** | 공유 자원 풀 | 여러 하위 채널로 분리 | RF Spectrum | 큰 도로 |
| **채널 슬롯** | 사용자별 대역 | 고정 폭 할당 | Narrowband Channel | 차선 |
| **가드밴드** | 간섭 방지 | 채널 사이 빈 주파수 | Guard Band | 차선 사이 안전거리 |
| **필터** | 채널 분리 | 원하는 대역만 통과 | Band-pass Filter | 검문소 |
| **주파수 계획** | 채널 재사용 | 셀 간 간섭 최소화 | Frequency Reuse | 도시 교통 계획 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  FDMA 주파수 분할 구조                        │
├──────────────────────────────────────────────────────────────┤
│ |User A|GB|User B|GB|User C|GB|User D|                       │
│ <---------------- Total Spectrum --------------------------> │
│ GB = Guard Band                                              │
└──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** FDMA에서는 사용자가 시간을 두고 번갈아 쓰는 것이 아니라, 서로 다른 주파수 대역을 동시에 점유한다. 따라서 시간 동기화는 단순하지만 인접 채널 간 간섭을 막기 위한 가드밴드가 필요하다. 이 가드밴드는 실제 데이터가 흐르지 않는 낭비 구간이므로 전체 스펙트럼 효율을 떨어뜨린다.

```text
┌──────────────────────────────────────────────────────────────┐
│             FDMA 운용 흐름                                    │
├──────────────────────────────────────────────────────────────┤
│ User Request -> Channel Assignment -> RF Filter -> TX/RX     │
│                         -> Release / Reuse                    │
└──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 네트워크는 사용자가 접속하면 빈 주파수 채널을 할당하고, 사용이 끝나면 이를 회수해 다른 사용자에게 재배정한다. 채널 폭과 주파수 계획은 서비스 품질과 셀 용량을 좌우한다. 음성 중심 시스템에서는 이 단순성이 강점이었지만, 트래픽이 bursty해질수록 고정 배정 방식의 비효율이 커졌다.

1. 전체 스펙트럼을 채널 단위로 분할한다.
2. 사용자에게 특정 주파수 채널을 할당한다.
3. 송수신기는 해당 대역만 필터링해 통신한다.
4. 인접 채널 간 간섭을 막기 위해 가드밴드를 유지한다.
5. 세션 종료 시 채널을 회수해 재사용한다.

- **📢 섹션 요약 비유**: 넓은 도로를 차선별로 나눠 차마다 한 차선씩 고정 배정하는 것과 같아서 단순하지만, 빈 차선이 생겨도 다른 차가 바로 활용하긴 어렵습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

| 방식 | 장점 | 단점 | 대표 용도 |
|:---|:---|:---|:---|
| **FDMA** | 구현 단순, 동시 전송 용이 | 가드밴드 낭비, 유휴 자원 발생 | 초기 이동통신 |
| **TDMA** | 시간 재활용 용이 | 동기화 복잡 | GSM |
| **CDMA** | 주파수 재사용 효율 높음 | 전력 제어 복잡 | 3G |
| **OFDMA** | 세밀한 자원 배정 | 시스템 복잡 | LTE/5G |

| 관점 | FDMA | FDM |
|:---|:---|:---|
| **정의** | 여러 사용자가 대역을 나눠 접근 | 여러 신호를 한 링크에 실어 전송 |
| **계층 관점** | 사용자 접속 제어 | 신호 다중화 |
| **연결성** | 다중 접속의 한 방식 | 물리 계층 전달 기술 |

- **과목 융합 관점**:
  1. **무선 공학**: 필터 성능과 주파수 안정도가 FDMA 품질에 직접 영향한다.
  2. **네트워크 관리**: 주파수 계획과 셀 재사용 전략이 용량을 좌우한다.
- **📢 섹션 요약 비유**: 각자 자기 차선을 쓰는 방식은 단순하고 안전하지만, 붐비는 시간에는 더 영리한 교통 제어가 필요해집니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

- **시나리오 1: 협대역 무전 시스템**
  - 판단: 지속 음성 통신과 장비 단순성이 중요하면 FDMA가 여전히 유효하다.
- **시나리오 2: 고밀도 모바일 데이터**
  - 판단: 사용자 수와 burst traffic이 많다면 FDMA 단독은 비효율적이므로 OFDMA 계열이 적합하다.
- **시나리오 3: 위성 링크**
  - 판단: 여러 독립 채널을 안정적으로 유지해야 할 때 FDMA 개념이 자주 활용된다.

체크리스트:

1. 지속형 트래픽인가, 버스트형 트래픽인가.
2. 인접 채널 간섭 관리가 가능한가.
3. 스펙트럼 효율보다 운영 단순성이 더 중요한가.

- **안티패턴**: 데이터 트래픽이 시간에 따라 크게 변하는 환경에서 고정 FDMA 채널만 쓰면 빈 대역이 많아져 자원 낭비가 심해진다.
- **📢 섹션 요약 비유**: 손님이 들쭉날쭉한 식당에 테이블을 손님별로 영구 예약해두면 자리가 비어도 다른 손님이 못 쓰는 것과 같습니다.

  2. 협대역 IoT, 특수 무선, 위성 통신에서 제한적으로 지속 활용된다.
  3. 동적 스펙트럼 공유와 결합해 더 유연한 형태로 진화할 수 있다.
