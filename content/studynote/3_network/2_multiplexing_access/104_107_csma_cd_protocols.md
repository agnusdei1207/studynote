# 104-107. 임의 접근 매체 제어: CSMA와 CSMA/CD

> **핵심 인사이트**: 다수의 단말이 하나의 공유 매체를 사용할 때, 데이터를 보내기 전 먼저 채널의 상태를 "듣고(Sense)" 빈 상태일 때만 전송하는 것이 CSMA의 기본 원리이며, 유선 환경에서는 충돌을 직접 "감지(Detect)"하여 처리하는 CSMA/CD가 사용된다.

---

## Ⅰ. CSMA (Carrier Sense Multiple Access)
**CSMA (반송파 감지 다중 접속)**는 "말하기 전에 먼저 듣는다(Listen Before Talk)"는 원칙을 적용한 임의 접근(Random Access) 제어 방식입니다.
* **작동 원리**: 단말이 데이터를 전송하기 전, 매체(채널)에 신호(반송파, Carrier)가 있는지 감지합니다.
* **목적**: 채널이 비어 있을 때만 전송을 시도하여, 알로하(ALOHA) 방식 대비 충돌 확률을 획기적으로 낮춥니다.
* **한계점**: 충돌 자체를 완전히 막을 수는 없습니다. 전파 지연(Propagation Delay) 때문에 A가 보낸 신호가 B에게 도달하기 직전, B가 채널이 비어 있다고 판단하여 전송할 경우 충돌이 발생합니다.

---

## Ⅱ. CSMA의 지속(Persistent) 전송 방식
채널이 사용 중일 때 단말이 어떻게 행동할 것인지 결정하는 방식입니다.

### 1. 1-Persistent CSMA (1-지속 방식)
* **동작**: 매체가 사용 중이면 채널이 빌 때까지 **계속 감지**하다가, 비는 즉시(100% 확률로) 전송합니다.
* **특징**: 이더넷(Ethernet)과 CSMA/CD에서 기본적으로 사용하는 방식입니다.
* **문제점**: 여러 단말이 채널이 비기를 기다리다가 동시에 전송을 시작하면 충돌 확률이 매우 높습니다.

### 2. Non-Persistent CSMA (비지속 방식)
* **동작**: 매체가 사용 중이면 감지를 멈추고, 임의의 무작위 시간(Random Time)을 기다린 후 다시 감지합니다.
* **특징**: 충돌 확률은 크게 감소하지만, 채널이 비어 있어도 즉시 전송하지 않아 네트워크 효율(Throughput)과 지연 시간(Delay) 측면에서 불리합니다.

### 3. p-Persistent CSMA (p-지속 방식)
* **동작**: 매체가 비어 있을 때, `p`의 확률로 즉시 전송하고, `1-p`의 확률로 다음 타임 슬롯까지 대기합니다. (슬롯화된 채널에서 사용)
* **특징**: 1-Persistent와 Non-Persistent의 장충돌률과 지연 시간을 절충한 방식입니다.

---

## Ⅲ. CSMA/CD (Collision Detection)
**CSMA/CD (충돌 탐지 반송파 감지 다중 접속)**는 주로 유선 LAN(IEEE 802.3 이더넷) 환경에서 사용되는 통신 규약입니다.

### 1. 주요 특징
유선 케이블에서는 신호의 송수신이 동시에 가능하여 송신 중에도 채널의 전압 변화를 통해 **충돌 발생 여부를 즉각적으로 감지(Detection)**할 수 있습니다.
* **최소 프레임 길이**: 충돌을 감지하기 위해 데이터 전송 시간은 최소한 네트워크의 왕복 전파 지연 시간(RTT)보다 길어야 합니다. (이더넷의 경우 최소 64 Bytes)

### 2. 동작 절차 (Listen & Detect)
1. **감지**: 전송 전 채널을 Listen 합니다.
2. **전송**: 채널이 비면 전송을 시작합니다. (1-Persistent)
3. **충돌 감지**: 전송과 동시에 수신을 수행하여 충돌 신호를 감시합니다.
4. **중단**: 충돌이 감지되면 즉시 데이터 전송을 중단합니다.

---

## Ⅳ. 잼 신호와 백오프 알고리즘
CSMA/CD에서 충돌이 감지된 직후 네트워크를 정상화하고 재전송을 조율하기 위한 매커니즘입니다.

### 1. 잼 신호 (Jam Signal)
* **역할**: 한 단말이 충돌을 감지하면, 즉시 전송을 멈추고 32비트 길이의 잼 신호(Jam Signal)를 전체 네트워크에 브로드캐스트합니다.
* **목적**: 네트워크 상의 다른 모든 노드들에게 충돌이 발생했음을 확실히 알려, 불필요한 데이터 전송을 중단시키고 모두 재전송 준비 단계로 들어가게 만듭니다.

### 2. 이진 지수 백오프 알고리즘 (Binary Exponential Backoff Algorithm)
* **역할**: 충돌 후 언제 다시 전송할지를 결정하는 대기 시간 계산 알고리즘입니다.
* **동작 방식**: 충돌 횟수(n)가 증가할수록 대기 시간의 범위를 지수 함수적으로 늘립니다. ($0 \sim 2^n - 1$ 사이의 슬롯 시간 선택)
* **특징**: 네트워크가 혼잡할수록 대기 시간을 크게 벌려 노드들을 분산시킵니다. (최대 16번 충돌 시 전송을 완전히 포기)

## Ⅴ. 개념 맵 및 요약

```ascii
[CSMA 계열 프로토콜 진화 흐름]

1. CSMA (Listen Before Talk) 
   ├── 1-Persistent : 빌 때까지 기다렸다가 100% 쏜다 (충돌 위험 높음)
   ├── Non-Persistent : 무작위 대기 후 다시 확인 (지연 발생)
   └── p-Persistent : 빌 때 p 확률로 쏜다 (슬롯 기반)
          │
          ▼
2. CSMA/CD (Collision Detection) -> 유선 LAN (Ethernet)
   ├── 전송 중 충돌 감지 (Voltage Level 체크)
   ├── 감지 시 전송 중지 & 잼 신호(Jam Signal) 발송
   └── 이진 지수 백오프(BEB) 알고리즘으로 무작위 지연 후 재전송
```

📢 **섹션 요약 비유**: CSMA는 회의실에서 남이 말하는지 먼저 "듣고" 말하는 예절입니다. CSMA/CD는 말을 하다가 다른 사람과 목소리가 겹치면(충돌 감지) "앗 죄송!" 하고 모두에게 신호를 보낸 뒤(잼 신호), 서로 다른 임의의 시간을 세고(백오프) 다시 말문을 여는 규칙과 같습니다.
