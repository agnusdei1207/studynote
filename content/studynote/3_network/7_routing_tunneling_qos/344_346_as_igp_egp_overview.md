# 344-346. AS(자율 시스템)와 라우팅 프로토콜 분류

> **핵심 인사이트**: 거대한 인터넷은 수만 개의 독립적인 '국가(AS)'들이 모여 만들어진 연합체다. 한 국가 내부의 길을 찾는 규칙(IGP)과 국가 간의 외교 경로를 설정하는 규칙(EGP)은 서로 다른 프로토콜을 사용한다.

---

## Ⅰ. AS (Autonomous System, 자율 시스템)
동일한 라우팅 정책(Routing Policy)을 공유하며 단일 관리 주체(예: KT, SKT, Google 등 ISP)에 의해 운영되는 네트워크 집합입니다.

* **AS 번호 (ASN)**: 전 세계적으로 AS를 식별하기 위해 부여된 고유 번호입니다. (예: AS 1234)
* **인터넷의 구조**: 인터넷은 이러한 AS들이 거미줄처럼 얽혀서 만들어진 거대한 네트워크의 네트워크(Network of Networks)입니다.

---

## Ⅱ. 라우팅 프로토콜의 영역별 분류

### 1. IGP (Interior Gateway Protocol, 내부 관문 프로토콜)
* **범위**: **하나의 AS 내부**에서 최적 경로를 찾기 위해 사용합니다.
* **특징**: 속도가 빠르고 메트릭(Metric) 계산이 정교합니다. 
* **대표 종류**:
  - **RIP** (거리 벡터 방식)
  - **OSPF** (링크 상태 방식)
  - **EIGRP** (Cisco 전용 하이브리드 방식)

### 2. EGP (Exterior Gateway Protocol, 외부 관문 프로토콜)
* **범위**: **서로 다른 AS들 사이**를 연결하고 데이터를 교환하기 위해 사용합니다.
* **특징**: 단순히 '빠른 길'보다는 '보안', '비용', '정책'에 따른 경로 설정을 중시합니다. (경로 벡터 방식)
* **대표 종류**: **BGP (Border Gateway Protocol)** ➔ 현재 인터넷 백본에서 유일하게 쓰이는 EGP 표준입니다.

---

## Ⅲ. 라우팅 알고리즘 방식별 분류

### 1. 거리 벡터 (Distance Vector)
* **원리**: 인접한 이웃 라우터와만 정보를 주고받습니다. "어느 방향(Vector)으로 얼마나 멀리(Distance) 있는가"만 관리합니다.
* **특징**: 구현이 쉽지만, 네트워크 전체 지도를 모르기 때문에 루프(Loop)가 생길 수 있고 수렴 속도가 느립니다. (예: RIP)

### 2. 링크 상태 (Link State)
* **원리**: 네트워크 전체의 지도(Topology Map)를 모든 라우터가 공유합니다. 
* **특징**: 수렴 속도가 매우 빠르고 루프가 거의 없지만, 지도를 그리느라 CPU와 메모리 소모가 큽니다. (예: OSPF)

---

## Ⅳ. 개념 맵 및 요약

```ascii
[인터넷 라우팅의 계층적 구조]

      [ AS 100 (KT) ] <────────── EGP (BGP) ──────────> [ AS 200 (Google) ]
     ┌───────────────┐                                 ┌──────────────────┐
     │   IGP (OSPF)  │                                 │    IGP (IS-IS)   │
     │  라우터 ── 라우터 │                                 │   라우터 ── 라우터  │
     └───────────────┘                                 └──────────────────┘
```

📢 **섹션 요약 비유**: **AS**는 하나의 '국가'입니다. 국가 내부의 내비게이션(도로 찾기)은 **IGP**이고, 다른 국가로 넘어가기 위한 국경 검문소와 외교 경로 설정은 **EGP**입니다. **거리 벡터**는 이정표만 보고 길을 찾는 초행길 운전자이고, **링크 상태**는 머릿속에 전체 지도와 실시간 교통 상황을 다 꿰고 있는 베테랑 택시기사입니다.
