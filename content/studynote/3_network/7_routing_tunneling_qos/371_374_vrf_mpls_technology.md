# 371-374. 가상 라우팅(VRF)과 MPLS 기술

> **핵심 인사이트**: 네트워크 인프라는 점차 '가상화'되고 있다. 라우터 한 대를 논리적으로 쪼개어 여러 고객이 독립적으로 쓰는 VRF 기술과, IP 주소 대신 짧은 '라벨'만 보고 데이터를 광속으로 전달하는 MPLS는 현대 기업용 통신망(VPN)의 핵심 근간이다.

---

## Ⅰ. VRF (Virtual Routing and Forwarding)
물리적인 라우터 한 대 내부에 **독립적인 여러 개의 라우팅 테이블**을 생성하는 기술입니다.

* **원리**: PC의 가상 머신(VM)과 비슷합니다. 라우터 포트별로 서로 다른 VRF를 할당하면, 같은 라우터 안에서도 A팀과 B팀의 통신을 완벽하게 분리할 수 있습니다. 
* **장점**: 주소가 겹쳐도(예: 두 팀 모두 사설 IP `192.168.1.1` 사용) 라우팅 테이블이 따로 존재하므로 충돌이 발생하지 않습니다.

---

## Ⅱ. MPLS (Multiprotocol Label Switching)
L2(스위칭)의 속도와 L3(라우팅)의 지능을 결합한 2.5계층 전송 기술입니다.

### 1. 작동 원리 (Label Switching)
* 기존 라우터는 복잡한 IP 헤더(32비트)를 읽고 최장 일치(Longest Match) 계산을 하느라 시간이 걸립니다.
* **MPLS**는 패킷 앞에 **20비트의 짧은 '라벨(Label)'**을 붙입니다. 중간 라우터들은 IP 주소는 쳐다보지도 않고, 이 숫자 라벨만 보고 하드웨어적으로 빠르게 패킷을 스왑(Swap)하여 던져줍니다.

### 2. 주요 장비 용어
* **LER (Label Edge Router)**: MPLS 망의 입구와 출구 라우터입니다. IP 패킷에 라벨을 붙이거나(Push), 떼는(Pop) 역할을 합니다.
* **LSR (Label Switch Router)**: MPLS 망 내부에서 라벨만 보고 고속으로 패킷을 전달(Swap)하는 라우터입니다.

---

## Ⅲ. MPLS VPN의 가치
통신사들이 기업 고객들에게 'MPLS VPN' 서비스를 제공하는 이유입니다.
* **보안성**: 전용선이 아니지만, MPLS 라벨과 VRF 기술을 통해 고객별 트래픽을 완벽하게 격리하여 전용선과 같은 보안을 제공합니다.
* **QoS**: 특정 라벨이 붙은 트래픽(보이스, 화상회의)에 대해 우선순위를 부여하기가 매우 쉽습니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[MPLS 망 통신 흐름]

[고객 A사] ──> [ LER (입구) ] ─── [ LSR ] ─── [ LSR ] ─── [ LER (출구) ] ──> [고객 A사]
 (IP 패킷)      (Label 삽입)      (Label 교체)   (Label 교체)   (Label 제거)      (IP 패킷)
                [Push]            [Swap]         [Swap]         [Pop]
```

📢 **섹션 요약 비유**: **VRF**는 라우터라는 건물 안에 '방음벽'을 세워 여러 회사가 서로 소리가 안 들리게 방을 나눠 쓰는 것입니다. **MPLS**는 택배 상자(패킷)의 복잡한 주소를 일일이 읽지 않고, 상자 위에 '빨간색 1번'이라는 커다란 스티커(라벨)를 붙여서 자동 분류기가 색깔만 보고 0.1초 만에 다음 칸으로 휙휙 던져주는 초고속 물류 센터 시스템입니다.
