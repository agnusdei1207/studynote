# 398-400. 기타 네트워크 계층 기술: IP SLA, Anycast, LISP

> **핵심 인사이트**: 현대 네트워크는 단순히 패킷을 보내는 것을 넘어, 회선의 상태를 실시간으로 감시하고(IP SLA), 지리적으로 가장 가까운 곳으로 응답을 유도하며(Anycast), 단말의 주소와 위치를 분리하여 이동성을 보장(LISP)하는 지능형 인프라로 진화하고 있다.

---

## Ⅰ. IP SLA (Service Level Agreement)
네트워크 장비(라우터 등)가 스스로 가상의 트래픽을 발생시켜 **네트워크의 성능(지연 시간, 지터, 패킷 유실 등)을 측정**하는 기술입니다.

* **동작**: 라우터가 특정 목적지로 주기적으로 Ping이나 HTTP 요청을 보내 응답 시간을 기록합니다.
* **활용 (Dynamic Routing 연동)**: 만약 1번 회선의 지연 시간이 500ms 이상으로 나빠지면, 자동으로 2번 회선(백업)으로 경로를 전환하도록 정적 라우팅과 연계할 수 있습니다.

---

## Ⅱ. Anycast (애니캐스트) 라우팅
똑같은 IP 주소를 가진 서버들을 전 세계 여러 곳에 배치하고, 사용자가 접속할 때 **네트워크상에서 가장 가까운(Shortest Path) 서버**로 패킷을 전달하는 방식입니다.

* **원리**: 여러 대의 서버(AS)가 동일한 IP 주소를 BGP를 통해 광고합니다. 라우터는 홉 수가 가장 적은 쪽으로 패킷을 보냅니다.
* **주요 용도**: 
  - **Root DNS 서버**: 전 세계 어디서든 빠른 이름 풀이를 가능케 함.
  - **CDN (Content Delivery Network)**: 사용자 근처의 캐시 서버로 접속 유도.
  - **DDoS 방어**: 공격 트래픽을 전 세계 여러 노드로 분산시켜 개별 서버의 부하를 줄임.

---

## Ⅲ. LISP (Locator/ID Separation Protocol)
전통적인 IP 주소가 가진 두 가지 역할(누구인가? + 어디에 있는가?)을 강제로 분리하는 차세대 네트워크 아키텍처입니다. (RFC 6830)

* **분리 구조**:
  - **EID (Endpoint Identifier)**: 단말기의 고유 '아이디'. 단말이 이동해도 변하지 않습니다.
  - **RLOC (Routing Locator)**: 단말이 현재 접속해 있는 라우터의 '위치'. 단말이 이동하면 바뀝니다.
* **장점**: IP 주소를 바꾸지 않고도 다른 네트워크로 이동(로밍)하며 통신을 유지할 수 있습니다. 클라우드 데이터센터 간의 가상 머신 이동(VM Mobility)에 필수적입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[Anycast 라우팅의 동작]

  [ User (한국) ] ──────> [ Router ] ──────> [ DNS Server (서울 노드) : 1.1.1.1 ]
                                             (가까우므로 당첨!)

  [ User (미국) ] ──────> [ Router ] ──────> [ DNS Server (LA 노드) : 1.1.1.1 ]
                                             (가까우므로 당첨!)

[LISP의 개념]
  "나는 누구인가(EID)" vs "나는 어디에 있는가(RLOC)"
  -> 위치가 바뀌어도 이름표는 그대로 유지!
```

📢 **섹션 요약 비유**: **IP SLA**는 택배 기사가 배송 전 미리 길 상황을 체크하기 위해 오토바이를 타고 한 바퀴 돌아보는 '사전 정찰'입니다. **Anycast**는 전 세계에 있는 '맥도날드 배달 서비스'와 같습니다. 어디서 전화를 걸든(1.1.1.1) 우리 집에서 가장 가까운 매장에서 햄버거가 옵니다. **LISP**는 휴대폰 번호(ID)는 그대로인데 내가 여행을 다니면서 계속 접속하는 기지국(위치)이 바뀌어도 전화를 끊김 없이 받을 수 있는 '스마트 로밍 시스템'입니다.
