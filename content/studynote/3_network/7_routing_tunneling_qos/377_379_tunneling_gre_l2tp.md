# 377-379. 터널링 기술: GRE와 L2TP

> **핵심 인사이트**: 터널링은 인터넷이라는 공용망 위에 '우리만 통과할 수 있는 가상의 파이프'를 까는 기술이다. GRE는 무엇이든 담을 수 있는 범용 바구니이고, L2TP는 먼 곳에서 사내망에 접속할 때 사용하는 정교한 가상 연결 통로이다.

---

## Ⅰ. 터널링 (Tunneling) 메커니즘 개요
하나의 네트워크 프로토콜(승객)을 다른 네트워크 프로토콜(운송수단)의 데이터 영역에 실어 보내는 기술입니다.

* **동작 원리 (Encapsulation)**: 원본 패킷(예: 사설 IP 패킷) 앞에 터널용 새 헤더(예: 공인 IP 헤더)를 덧붙여서 전송합니다.
* **목적**:
  - 서로 다른 종류의 네트워크 연결 (예: IPv6 over IPv4)
  - 공인망을 통한 사설망 연결 (VPN의 기초)
  - 네트워크 구조 은닉

---

## Ⅱ. GRE (Generic Routing Encapsulation)
시스코에서 개발하고 표준화된 **가장 대표적인 범용 터널링 프로토콜**입니다.

* **특징**: 
  - **유연성**: IP뿐만 아니라 IPX, AppleTalk 등 어떤 3계층 프로토콜도 다 담을 수 있습니다.
  - **멀티캐스트 지원**: 라우팅 프로토콜(OSPF, RIP 등)의 업데이트 패킷(멀티캐스트)을 터널을 통해 보낼 수 있습니다. (IPsec과의 결정적 차이점)
* **단점 (보안 부재)**: GRE 자체에는 암호화 기능이 전혀 없습니다. 데이터가 평문으로 노출되므로, 실제 운영 시에는 반드시 **IPsec**과 함께 사용(GRE over IPsec)해야 안전합니다.

---

## Ⅲ. L2TP (Layer 2 Tunneling Protocol)
주로 원격지 사용자가 사내망에 접속하는 **재택근무(Remote Access) VPN**에서 사용하는 기술입니다.

* **발전**: 시스코의 L2F와 마이크로소프트의 PPTP 기술을 합쳐서 표준화했습니다.
* **동작**: 2계층 데이터(PPP 프레임)를 3계층 IP 패킷에 담아서 터널링합니다.
* **특징**: 
  - 인증(Authentication) 기능이 강력합니다.
  - GRE와 마찬가지로 자체 암호화가 없어서, 보통 **IPsec**과 결합하여 **L2TP/IPsec** 형태로 가장 많이 쓰입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[터널링 패킷 포장 구조]

  [ Delivery Header ] [ Tunnel Header ] [ Original Packet ]
  (공인 IP 헤더)       (GRE / L2TP)      (사설 IP / 데이터)
         │                  │                  │
         └─ "운송 트럭"      └─ "특수 컨테이너"   └─ "승객(진짜 내용)"
```

📢 **섹션 요약 비유**: **터널링**은 지하철 노선(공용망) 위에 우리 회사 직원만 탈 수 있는 '전용 칸'을 만드는 것과 같습니다. **GRE**는 어떤 짐이든 가리지 않고 다 실을 수 있는 커다란 '무지개색 컨테이너'입니다. 하지만 컨테이너 벽이 유리(평문)라서 밖에서 내용물이 다 보입니다. **L2TP**는 사원증(인증)을 찍어야만 들어갈 수 있는 좀 더 꼼꼼한 전용 칸입니다. 둘 다 도둑(해커)을 막으려면 겉에 검은색 암막 커튼(**IPsec**)을 쳐야 비로소 완벽한 VPN이 됩니다.
