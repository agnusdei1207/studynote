# 337-338. 라우터의 내부 구조와 CEF 스위칭

> **핵심 인사이트**: 과거의 라우터는 패킷이 들어올 때마다 CPU가 경로를 계산하느라 느렸다. 현대의 라우터는 경로 계산(RIB)과 실제 전송(FIB)을 완전히 분리하고, 전용 하드웨어 칩셋(ASIC)을 활용하는 CEF 기술을 통해 지연 없는 초고속 포워딩을 구현한다.

---

## Ⅰ. 라우터의 논리적 평면 분리 (Planes)
라우터 내부의 역할 분담 체계입니다.

1. **제어 평면 (Control Plane)**
   * **역할**: 라우팅 프로토콜(OSPF, BGP 등)을 돌려 네트워크 지도를 그리고 **어디로 갈지 결정**하는 두뇌 역할입니다.
   * **결과물**: **RIB (Routing Information Base, 라우팅 테이블)**를 생성합니다.
   * **특징**: 소프트웨어(CPU) 기반으로 동작하며, 상대적으로 느립니다.

2. **데이터 평면 (Data Plane / Forwarding Plane)**
   * **역할**: 들어온 패킷을 제어 평면이 결정해 놓은 규칙에 따라 **실제로 옆 포트로 던지는** 팔다리 역할입니다.
   * **결과물**: **FIB (Forwarding Information Base)** 테이블을 사용합니다.
   * **특징**: 하드웨어(ASIC 칩셋) 기반으로 동작하며, 엄청나게 빠릅니다.

---

## Ⅱ. 라우터의 3가지 스위칭 방식의 진화

### 1. 프로세스 스위칭 (Process Switching) - 과거
* 패킷이 들어올 때마다 CPU가 라우팅 테이블을 뒤져서 경로를 찾습니다. CPU 부하가 엄청나고 매우 느립니다.

### 2. 패스트 스위칭 (Fast Switching) - 과도기
* 첫 번째 패킷만 CPU가 계산하고, 그 결과를 **캐시(Cache)**에 저장해 둡니다. 두 번째 패킷부터는 캐시를 보고 바로 보냅니다. (First-packet Process, Subsequent-packets Fast)

### 3. CEF (Cisco Express Forwarding) - 현대 표준
* **원리 (Topology-based)**: 패킷이 들어오기 전, 라우팅 테이블(RIB)을 미리 스캔하여 하드웨어가 읽기 편한 형태인 **FIB**와 **Adjacency Table(인접 테이블)**로 변환해 둡니다.
* **FIB (Forwarding Information Base)**: 목적지 IP에 따른 출력 인터페이스 정보를 담은 2계층 전용 테이블입니다.
* **Adjacency Table**: 목적지까지 가기 위해 필요한 상대방의 MAC 주소(L2 헤더 정보)를 미리 계산해 둔 테이블입니다.
* **장점**: 패킷이 들어오는 즉시 CPU의 개입 없이 하드웨어가 FIB와 인접 테이블을 결합하여 L2 헤더를 갈아 끼우고 바로 쏴버립니다. 캐시가 깨질 염려도 없고 가장 빠릅니다.

---

## Ⅲ. 개념 맵 및 요약

```ascii
[라우터 내부 정보 처리 흐름]

  (Control Plane - 두뇌)
  [ Routing Protocols ] ──> [ RIB (Routing Table) ]
                                     │
                                     ▼ (CEF 엔진이 미리 변환)
  (Data Plane - 하드웨어)
  [ 패킷 유입 ] ───> [ FIB ] + [ Adjacency Table ] ───> [ 패킷 유출 ]
                     (ASIC 칩셋이 고속 검색)
```

📢 **섹션 요약 비유**: **프로세스 스위칭**은 손님이 올 때마다 주방장(CPU)이 레시피 북을 뒤져 요리하는 식당입니다. **패스트 스위칭**은 한 번 만든 요리는 기억해 뒀다 주문하면 바로 주는 방식입니다. **CEF**는 손님이 오기도 전에 이미 모든 요리를 종류별로 다 만들어놓고 진열대에 올려둔 뒤, 손님이 집어가기만 하면 되는 '초고속 뷔페' 시스템입니다.
