# 369-370. 멀티캐스트 라우팅과 PIM

> **핵심 인사이트**: 멀티캐스트 라우팅은 전체에 뿌리는 브로드캐스트와 한 명에게 주는 유니캐스트의 장점만 섞은 기술이다. '역경로 전달(RPF)'을 통해 루프를 방지하고, PIM 프로토콜을 이용해 필요한 사람에게만 데이터를 복사해 전달하는 '배달 나무(Tree)'를 그린다.

---

## Ⅰ. 멀티캐스트 라우팅의 기초
단일 송신자가 특정 그룹에 가입된 다수의 수신자에게 데이터를 한 번에 보내는 기술입니다.

* **가장 중요한 원리: RPF (Reverse Path Forwarding)**
  - **개념**: "내가 이 패킷을 받은 포트가, 패킷의 출발지까지 가는 최단 경로인가?"를 확인하는 것입니다.
  - **목적**: 만약 돌아가는 길로 패킷이 들어왔다면 루프가 발생한 것으로 간주하고 즉시 드랍합니다. 멀티캐스트 루프 방지의 핵심입니다.

---

## Ⅱ. PIM (Protocol Independent Multicast)
유니캐스트 라우팅 프로토콜(OSPF, RIP 등)의 종류와 상관없이 독립적으로 동작하는 멀티캐스트 제어 프로토콜입니다.

### 1. PIM-DM (Dense Mode, 조밀 모드)
* **전략 (Push)**: 일단 네트워크 전체에 패킷을 다 뿌려보고(Flooding), "나 안 들을래"라고 요청(Prune)하는 사람만 목록에서 뺍니다.
* **특징**: 수신자가 아주 많을 때 유리하지만, 초기 플러딩으로 인해 대역폭 낭비가 큽니다.

### 2. PIM-SM (Sparse Mode, 희박 모드)
* **전략 (Pull)**: 네트워크 중간에 **RP (Rendezvous Point, 만남의 광장)** 라우터를 둡니다. 데이터를 받고 싶은 사람만 RP에게 "나 가입할래(Join)"라고 명시적으로 요청해야만 데이터를 보내줍니다.
* **특징**: 수신자가 듬성듬성 있을 때 효율적이며, 현대 멀티캐스트망(IPTV 등)의 표준 방식입니다.

---

## Ⅲ. 멀티캐스트 배달 트리 (Tree)

1. **Source Tree (Shortest Path Tree, SPT)**: 
   - 송신자(Source)를 뿌리로 하여 각 수신자까지의 최단 경로를 그립니다. 지연 시간이 짧지만 라우터의 메모리 소모가 큽니다. 표기: `(S, G)`
2. **Shared Tree (RP Tree)**: 
   - 중간 관리자인 RP를 뿌리로 하여 데이터를 공유합니다. 관리가 쉽지만 경로는 약간 돌아갈 수 있습니다. 표기: `(*, G)`

---

## Ⅳ. 개념 맵 및 요약

```ascii
[PIM-SM 동작 프로세스]

 [ Source ] ───> [ First-Hop Router ] ─── (Register) ───> [ RP (만남의 광장) ]
                                                              │
                                                              ▼ (Join 요청한 쪽으로만)
                                                        [ Last-Hop Router ]
                                                              │
                                                        [ Receiver (가입자) ]
```

📢 **섹션 요약 비유**: **멀티캐스트**는 '유료 유선 방송'입니다. **RPF**는 셋톱박스가 거꾸로 신호를 쏴봐서 우리 집 선로가 맞는지 확인해 가짜 신호를 차단하는 보안관입니다. **PIM-DM**은 일단 온 동네 집집마다 신호를 다 넣어준 뒤 "안 볼 사람 전화해!" 하는 막무가내 방식이고, **PIM-SM**은 "신청한 집만 전용 채널을 열어주는" 세련된 가입형 서비스입니다. 여기서 **RP**는 모든 채널 가입 신청을 접수하는 통합 콜센터 역할을 합니다.
