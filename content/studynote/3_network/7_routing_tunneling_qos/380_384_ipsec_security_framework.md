# 380-384. IPsec 보안 프레임워크

> **핵심 인사이트**: IPsec은 인터넷이라는 무법지대에서 데이터를 안전하게 옮기기 위한 '무장 호송 차량'과 같다. 데이터가 누구 것인지 확인(AH)하고, 내용물을 암호화 상자에 담아 잠그며(ESP), 열쇠를 안전하게 주고받는 규칙(IKE)을 모두 포함하는 거대한 보안 세트이다.

---

## Ⅰ. IPsec (IP Security) 개요
네트워크 계층(L3)에서 IP 패킷의 기밀성(Confidentiality), 무결성(Integrity), 인증(Authentication)을 제공하는 프로토콜 모음입니다.

* **동작 모드**:
  - **전송 모드 (Transport Mode)**: IP 페이로드(데이터)만 암호화합니다. 호스트 간 직접 연결에 씁니다.
  - **터널 모드 (Tunnel Mode)**: **IP 헤더를 포함한 전체 패킷을 암호화**하고 새로운 IP 헤더를 붙입니다. **VPN(지사-본사)** 구축 시 필수입니다.

---

## Ⅱ. 핵심 프로토콜 (AH와 ESP)

### 1. AH (Authentication Header)
* **역할**: "이 패킷은 진짜 우리 본사에서 보낸 게 맞다"라는 **인증**과 "오다가 한 비트도 안 변했다"라는 **무결성**만 보장합니다.
* **특징**: **암호화 기능이 없습니다.** 내용물이 훤히 보입니다. 
* **한계**: IP 헤더까지 무결성 검사를 하므로, 주소를 바꾸는 **NAT 구간을 통과할 수 없습니다.** (체크섬 오류 발생)

### 2. ESP (Encapsulating Security Payload)
* **역할**: 인증과 무결성은 물론, 가장 중요한 **기밀성(암호화)**을 제공합니다.
* **현대의 표준**: 현재 거의 모든 IPsec VPN은 ESP를 단독으로 사용하거나 AH와 섞어 씁니다.

---

## Ⅲ. 키 교환 및 설정 (IKE와 SA)

### 1. IKE (Internet Key Exchange)
* **역할**: 통신 시작 전, "어떤 암호 알고리즘을 쓸까?" 협상하고 **암호키를 안전하게 생성/교환**하는 프로토콜입니다.
* **ISAKMP**: IKE가 동작하기 위한 메시지 교환 규칙(프레임워크)입니다.

### 2. SA (Security Association)
* **개념**: 두 라우터 간에 맺어진 "보안 약속" 그 자체입니다. (암호 알고리즘, 키 유효기간 등 정보 저장)

---

## Ⅳ. NAT-T (NAT Traversal)
IPsec(특히 ESP)은 포트 정보가 없어서 포트를 주소로 쓰는 NAT 공유기를 통과하기 어렵습니다.

* **해결책**: IPsec 패킷을 한 번 더 **UDP 4500번 포트**로 감싸서(Encapsulation) 보냅니다. 이렇게 하면 공유기(NAT)가 일반 UDP 패킷인 줄 알고 길을 열어줍니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[IPsec 터널 모드 패킷 구조 (ESP)]

  [ New IP Header ] [ ESP Header ] [ Original IP Header ] [ Data ] [ ESP Trailer ] [ ESP Auth ]
  └──────┬────────┘ └──────┬─────┘ └───────────────────┬──────────┘ └──────┬──────┘ └────┬────┘
         │                 │                          │                    │             │
    (공인 IP 주소)     (암호화 정보)             (전체 암호화 구간)           (패딩 등)      (무결성 서명)
```

📢 **섹션 요약 비유**: **AH**는 상자 겉면에 떼어낼 수 없는 '봉인 씰'을 붙이는 것입니다. 뜯었는지 안 뜯었는지는 알 수 있지만 상자 안은 보입니다. **ESP**는 상자 안의 물건을 튼튼한 '금고'에 넣어 잠그는 것입니다. **IKE**는 금고를 잠그기 전, 상대방과 어떤 비밀번호를 쓸지 아무도 모르게 귓속말로 약속하는 과정입니다. **NAT-T**는 금고를 일반 '종이박스(UDP)'에 한 번 더 담아 경비실(NAT)을 자연스럽게 통과시키는 요령입니다.
