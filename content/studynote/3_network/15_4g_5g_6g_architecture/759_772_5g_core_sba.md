# 759-772. 5G 코어 아키텍처 (5GC & SBA)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 5G Core (5GC)는 하드웨어 중심의 전통적인 교환기(CS, PS)를 지우고, 모든 네트워크 기능(NF)을 소프트웨어(S/W)화하여 **SBA (Service Based Architecture)** 기반의 클라우드 서비스처럼 재설계한 패러다임이다.
> 2. **가치**: 이기종 장비 간 결합도를 제거하여 **C/U (Control/User) Plane 분리** 및 **Network Slicing (네트워크 슬라이싱)**을 통해 논리적으로 독립된 망을 하나의 인프라에서 제공함으로써, 운영 비용(OPEX)을 절감하고 서비스 제공 유연성을 극대화한다.
> 3. **융합**: **Cloud Native (CNF)** 기술과 융합하여 **MEC (Multi-Access Edge Computing)** 환경에서의 초저지연(Ultra-Low Latency) 처리를 가능하게 하며, AI 기반 자동화 네트워크의 기반이 된다.

+++

### Ⅰ. 개요 (Context & Background)

#### 1. 개념 및 정의
5G 코어(5GC, 5G Core Network)는 3GPP Release 15 이상에서 정의된 차세대 이동통신망의 핵심망으로, 기존 4G(LTE)의 EPC (Evolved Packet Core)가 가진 **'장비 기반의 독립적 인터페이스'** 한계를 극복하기 위해 **SBA (Service Based Architecture)**라는 완전히 새로운 소프트웨어 정의 아키텍처를 도입했다. 5GC는 **CP(Control Plane)**와 **UP(User Plane)**의 철저한 분리, **Stateless(무상태)** 설계, 그리고 **HTTP/2** 기반의 RESTful API 통신을 특징으로 한다.

#### 2. 등장 배경 및 필연성
① **기존 한계**: 4G EPC는 네트워크 기능(NF) 간의 통신이 물리적 망 구성에 종속되어 있어, 새로운 서비스(예: V2X, URLLC) 추가 시 전체 망의 장비 교체나 프로토콜 업그레이드가 불가피했다.
② **혁신적 패러다임**: **SDN (Software Defined Networking)**과 **NFV (Network Function Virtualization)** 기술을 만개하여, 네트워크를 '관로(Pipe)'가 아닌 '서비스(Service)'의 집합으로 정의한다. 이는 IT 시스템의 **Microservices Architecture (마이크로서비스 아키텍처)**를 통신망에 그대로 이식한 것으로 볼 수 있다.
③ **현재의 비즈니스 요구**: 사물인터넷(IoT) 폭발과 더불어 망 사업자는 **Network Slicing**을 통해 고객(자동차, 병원, 게임사 등)별로 차별화된 SLA (Service Level Agreement)를 제공해야만 하는 시장적 요구를 충족시킨다.

#### 3. 핵심 변화 포인트
```ascii
[4G EPC vs 5GC 아키텍처 비교]

+-----------------------------+          +-------------------------------+
|         4G LTE (EPC)         |          |         5G Core (SBA)          |
+-----------------------------+          +-------------------------------+
| (Point-to-Point Model)      |          | (Service Based Model)          |
|                             |          |                               |
| [ HSS ]-----(S6a)-----[ MME ]|          | [ UDM ]   [ PCF ]   [ AMF ]    |
|   |                      |  |          |   ^          ^         ^        |
| (S6d)                 (S11) |          |   |          |         |        |
|   v                      v  |          | [Nudm]     [Npcf]    [Namf]... |
| [ PCRF ]-----(Gx)-----[ PGW/C-GW]|      |   |          |         |        |
|                             |          |   |          |         |        |
| (Hardwired Dependencies)    |          | [  Service Bus (HTTP/2)  ]      |
+-----------------------------+          +-------------------------------+
```
*해설: 4G는 장비 간 1:1 연결이 고정된 '점대점(Point-to-Point)' 구조라 변경 시 영향도가 크지만, 5G는 모든 기능이 버스에 연결된 '버스형' 구조로, 필요한 서비스를 API만으로 탈부착할 수 있다.*

> **📢 섹션 요약 비유**:
> 5GC 아키텍처의 변화는 마치 **'전통식 식당'에서 '푸드코트'로 바뀌는 것**과 같습니다. 과거(4G)에는 주방에서 식당까지 모든 시스템이 일렬로 연결된 고정된 틀에서 주문을 처리해야 했다면, 이제(5G)는 각 매장(네트워크 기능)이 독립적으로 운영되면서, 중앙 결제 시스템(버스)을 통해 필요할 때마다 유연하게 주문을 연결하고 메뉴(서비스)를 바꿀 수 있게 되었습니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

#### 1. 주요 네트워크 기능 (NF, Network Functions) 상세
SBA에서 각 기능은 서비스 형태로 제공되며, 아래는 핵심적인 5가지 구성 요소이다.

| 요소 (NF) | 전체 명칭 (Full Name) | 핵심 역할 | 내부 동작 및 프로토콜 | 비유 |
|:---:|:---|:---|:---|:---|
| **AMF** | Access and Mobility Management Function | **접속 및 이동성 관리** <br> - UE의 등록, 연결 관리, 도달 가능성 확인 | - **N1, N2** 인터페이스 처리 <br> - **NAS (Non-Access Stratum)** 보안 | **호텔 프론트 데스크** <br> (손님 입장/퇴장 및 방 배정) |
| **SMF** | Session Management Function | **세션 관리** <br> - PDU Session 생성/수정/삭제 <br> - UE IP 할당, UPF 선택 | - **N4** 인터페이스로 UPF 제어 <br> - **PCC Rule** 적용 | **항공 관제사** <br> (비행 경로 설정 및 이륙 허가) |
| **UPF** | User Plane Function | **사용자 평면 처리** <br> - 패킷 라우팅/포워딩 <br> - QoS 강제, 트래픽 라우팅 | - **GTP-U (GPRS Tunneling Protocol)** 프로토콜 사용 <br> - Anchor 역할 수행 | **화물 트럭** <br> (실제 짐(데이터)을 나르는 차량) |
| **PCF** | Policy Control Function | **정책 제어** <br> - QoS 정책, 과금 정책 결정 <br> - 네트워크 슬라이스 선택 지원 | - **N7, N15** 인터페이스 <br> - UDR로부터 구독자 정보 조회 | **회사 경영진** <br> (예산, 우선순위 결정) |
| **UDM** | Unified Data Management | **데이터 관리** <br> - 가입자 프로필, 가입 데이터 저장 <br> - 인증 자원 관리 | - **UDR (Unified Data Repository)**와 연동 <br> - 인증 크레덴셜 발행 | **인사부 데이터베이스** <br> (직원/고객 정보 원본) |

#### 2. 5G Core 프로시저 동작 원리
사용자(UE)가 네트워크에 접속하고 데이터를 주고받는 과정은 다음과 같다.

```ascii
[5G Registration & Session Setup Flow]

UE (User Equipment)                           RAN (gNodeB)                    5G Core (AMF/SMF/UPF)
+----------------+                            +------------+                  +------------------+
| 1. RRC Setup   | -------------------------> |            |                  |                  |
| (접속 요청)    |                            |            |                  |                  |
+----------------+                            +------------+                  |                  |
      v                                                                        v
+----------------+  2. Initial UE Message (NAS) -------------------------> +------------------+
| 3. Authentication| <---------------------------------------------------- |     [ AMF ]      |
| Request/Response |  (Identity Request)    (Identity Response)              |                  |
+----------------+ ------------------------------------------------------------+                  |
      v                                                                                         v
+----------------+  4. PDU Session Request ---------------------------------> +------------------+
| "데이터 쓸게요"| ------------------------------------------------------------>|     [ SMF ]      |
+----------------+                                                             |  1. PCF에 정책 문의|
                                                                             |  2. UPF 선택 및 설정|
                                                                                 v
                                                                             +------------------+
                                                                             | [ UPF ]          |
                                                                             | (데이터 패킷 처리)|
                                                                             +------------------+
```
*해설:*
1. **UE → RAN**: 사용자가 기지국(gNodeB)에 연결을 요청(RRC Setup).
2. **RAN → AMF**: 기지국은 이 요청을 **AMF**에 전달. AMF는 가입자 인증(UDM/UDR 통해) 및 보안 절차 수행.
3. **AMF → SMF**: 인증 완료 후, AMF는 세션 관리를 위해 **SMF**에게 메시지를 전달.
4. **SMF → PCF/UPF**: **SMF**는 **PCF**로부터 QoS 정책을 받아 **UPF**(User Plane)의 터널(GTP-U) 설정.
5. **Data Flow**: 이후 트래픽은 UPF를 거쳐 인터넷으로 나간다.

> **📢 섹션 요약 비유**:
> 5G 코어의 동작은 **'고속 도로 통행료 징수 시스템'**과 같습니다. **AMF**는 진입로에서 차량을 검사하고 입구를 개방하는 '진입 관제소'이고, **SMF**는 차량의 종류나 목적지에 따라 '하이패스 차로'나 '일반 차로'를 배정하는 '운영 센터'입니다. **UPF**는 실제 차량이 달리는 '도로 본선'입니다. 이 시스템 덕분에 화물차(게임 트래픽)와 승용차(메신저 트래픽)를 다른 도로(슬라이스)로 완벽히 분리하여 운영할 수 있습니다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

#### 1. NSA vs SA (Standalone) 심층 비교
5G 초기 구축 단계인 NSA와 진정한 5G인 SA를 비교 분석한다.

| 구분 | **NSA (Non-Standalone)** | **SA (Standalone)** |
|:---:|:---|:---|
| **정의** | 4G LTE 망(EPC)을 제어부로 활용하고 5G 기지국만 추가하여 데이터 속도만 높이는 방식. | 5G 코어(5GC)와 5G 기지국(gNB)만으로 독립적으로 구성하는 완전한 형태. |
| **핵심 인터페이스** | **X2**: eNB(LTE)와 gNB(5G) 간 연결 <br> **S1-U**: 데이터는 4G EPC를 거쳐 인터넷으로 나감. | **N1, N2, N3**: 모든 인터페이스가 5GC 기반. <br> **N3** 인터페이스를 통해 UPF로 직접 연결. |
| **주요 특징** | 5G 코어 투자 없이 4G 기지국(S/W 업그레이드)만으로 서비스 가능. | **Network Slicing**, **URLLC**, **mMTC** 등 5G 고유 기능 모두 지원 가능. |
| **단점** | 4G 코어망 병목 현상으로 지연(Latency) 감액에 한계 존재. | 초기 설치 비용(CAPEX)이 높고 커버리지 확보에 시간 소요. |
| **적합한 서비스** | 일반적인 모바일 광대역(eMBB) 서비스 위주. | 자율주행, 원격제어, 스마트팩토리 등 초저지연 서비스. |

#### 2. 과제 융합 관점: C/U Plane 분리와 MEC
5G 아키텍처의 핵심인 **C/U Plane Separation (제어부/사용자부 분리)**은 **컴퓨터 구조(Computer Architecture)**의 **Harvard Architecture**와 맞닿아 있다. 제어 명령어(Control)와 데이터(Data)의 버스를 분리하여 병목 현상을 해소하는 원리다.

* **융합 효과**: UPF(사용자부)를 코어 센터가 아닌 **Edge(단말 인접 지역)**에 분산 배치(**Distributed UPF**)함으로써, **MEC (Multi-Access Edge Computing)**와 결합하여 데이터가 왕복하는 거리를 물리적으로 줄인다.
* **성능 지표**: 이를 통해 **Latency (지연 시간)**를 4G의 20~30ms에서 1ms 이하(Uplink)로 획기적으로 단축할 수 있다.

```ascii
[Centralized UPF vs Distributed UPF (MEC Scenario)]

[A. 4G/NSA 구조 (Centralized)]             [B. 5G SA 구조 (Distributed/Edge)]

   [Core DC]                                  [Edge DC]           [Core DC]
 +----------+                               +----------+        +----------+
 |   EPC    |                               | [ UPF ]   |        | [ SMF ]  |
 |  [ PGW ] | <---- 30~50ms 이동 ---------> | (Local)   | <----  |  [ AMF ] |
 +----------+                               +----------+        +----------+
      ^                                          ^                 ^
      |                                          |                 |
   [Internet]                                [Application]      [Internet]
                                              (AI Server)
```
*해설: A 구조에서는 모든 트래픽이 중앙(Central DC)까지 왕복해야 하므로 지연이 필연적이다. B 구조(5G SA)에서는 SMF(제어부)의 지시를 받아 UPF(데이터부)를 사용자 바로 옆(Edge)에 둠으로써, 로컬에서 데이터를 처리하고 다시 인터넷으로 내보낼 수 있다.*

> **📢 섹션 요약 비유**:
> NSA와 SA의 차이는 **'개조된 자동차와 신차'**의 차이입니다. NSA는 기존 티코(4G) 차체에 F1 엔진(5G 무선주파수)만 얹어 속도는 빠르지만 코너링이나 핸들링(초저지연 제어)이 불안정합니다. 반면 SA는 차체와 엔진, 핸들 시스템(5GC)을 모두 새로 설계한 F1 레이싱카라 극한의 속도와 정교한 제어가 모두 가능합니다. 또한 C/U 분리는 **'도로 설계의 패러다임 전환'**으로, 차량 흐름(제어)을 중앙통제실에서 보되, 차량 진입로(데이터)
