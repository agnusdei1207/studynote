# 759-772. 5G 코어 아키텍처 (5GC & SBA)

> **핵심 인사이트**: 5G 코어망은 더 이상 물리적인 장비의 집합이 아니다. 모든 기능이 소프트웨어(컨테이너)로 쪼개져 API로 대화하는 '클라우드 네이티브' 시스템(SBA)으로 변모했다. 덕분에 전용 차선을 만드는 슬라이싱이 가능해졌고, 데이터 고속도로(UPF)를 사용자 코앞까지 전진 배치할 수 있게 되었다.

---

## Ⅰ. 5G SBA (Service Based Architecture)
네트워크 기능(NF)들이 서로 미리 정해진 선으로 연결되는 대신, **공통된 버스(Bus)에 접속하여 REST API로 필요한 서비스를 호출**하는 방식입니다.

* **장점**: 새로운 기능을 추가하거나 확장하는 것이 클라우드 앱처럼 매우 유연하고 빠릅니다.

---

## Ⅱ. 5G 코어망 주요 기능 (NF, Network Functions)

1. **AMF (Access and Mobility Management Function)**: LTE의 MME 역할을 계승. 접속 제어와 이동성 관리를 담당하는 관제탑입니다.
2. **SMF (Session Management Function)**: 세션(통로)을 만들고 관리하며 IP 주소를 할당합니다.
3. **UPF (User Plane Function)**: **가장 중요한 데이터 통로.** 실제 사용자 패킷을 고속으로 전달하며, MEC 실현을 위해 기지국 근처로 전진 배치될 수 있습니다.
4. **PCF (Policy Control Function)**: 가입자별 요금제나 서비스 우선순위(QoS) 정책을 결정합니다.
5. **UDM (Unified Data Management)**: LTE의 HSS와 유사하게 가입자 마스터 정보를 관리합니다.

---

## Ⅲ. 구축 모드: NSA vs SA

### 1. NSA (Non-Standalone)
* **방식**: 제어(Control)는 기존 LTE 망을 쓰고, 데이터(User)만 5G 안테나를 쓰는 방식.
* **장점**: 초기 구축 비용이 저렴하고 빠르게 상용화 가능.

### 2. SA (Standalone)
* **방식**: 코어망부터 기지국까지 모두 5G 전용 장비(5GC)를 쓰는 방식.
* **장점**: 5G의 진정한 가치인 **네트워크 슬라이싱**과 **초저지연**을 완벽히 구현 가능.

---

## Ⅳ. 5G 주파수 대역 (NR, New Radio)
* **FR1 (Sub-6GHz)**: 6GHz 이하 대역. 커버리지가 넓어 보편적인 5G 서비스에 사용.
* **FR2 (mmWave)**: 24GHz 이상 초고주파. 직진성이 강하고 속도가 매우 빠르지만 도달 거리가 짧음. (진정한 20Gbps 속도의 열쇠)

---

## Ⅴ. 개념 맵 및 요약

```ascii
[5G SBA 구조도]

  [ UDM ] [ PCF ] [ AMF ] [ SMF ]  <─── ( NF 서비스 버스: REST API )
     │       │       │       │
     └───────┴───────┴───────┘
                     │
  [ gNodeB (5G기지국) ] <───(N3 인터페이스)───> [ UPF (데이터평면) ] ──> [ Internet ]
```

📢 **섹션 요약 비유**: **SBA**는 레고 블록으로 성을 쌓는 것과 같습니다. 필요한 기능(인증, 과금 등) 블록을 언제든 뗐다 붙였다 할 수 있는 유연한 구조입니다. **AMF**는 공항 입국 심사대이고, **SMF**는 활주로를 배정하는 관제실이며, **UPF**는 승객과 짐을 실어 나르는 실제 '비행기' 그 자체입니다. **SA 방식**은 공항 터미널부터 비행기까지 모두 최신식으로 새로 지은 5성급 공항 시스템과 같습니다.
