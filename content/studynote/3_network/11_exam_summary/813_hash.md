# Hash - 해시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고정 길이 출력
> 2. **가치**: 무결성 검증
> 3. **융합:** SHA-256, MD5, Bloom Filter

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Hash는 임의 길이 입력을 고정 길이 출력으로 변환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Hash                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              해시 특성                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  입력 → 해시 함수 → 출력 (Digest)                      │   │ │
│   │   │                                                      │   │ │
│   │   │  "Hello"     ───→  SHA-256  ───→  a591a6...         │   │ │
│   │   │  "Hello!"    ───→  SHA-256  ───→  2cf24d...         │   │ │
│   │   │  "Hello!!"   ───→  SHA-256  ───→  f4d5be...         │   │ │
│   │   │                                                      │   │ │
│   │   │  특성:                                                │   │ │
│   │   │  - 단방향: 역변환 불가                                  │   │ │
│   │   │  - 결정론적: 동일 입력 → 동일 출력                       │   │ │
│   │   │  - 균등 분포: 고르게 분산                                │   │ │
│   │   │  - 충돌 저항: 다른 입력 → 다른 출력                      │   │ │
│   │   │                                                      │   │ │
│   │   │  용도:                                                │   │ │
│   │   │  - 무결성 검증                                        │   │ │
│   │   │  - 비밀번호 저장                                       │   │ │
│   │   │  - 디지털 서명                                        │   │ │
│   │   │  - 해시 테이블                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 해시는 데이터 식별과 무결성 검증에 필수입니다.

**📢 섹션 요약 비유:** Hash = 디지털 지문!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **MD5** | 128비트, 취약 |
| **SHA-1** | 160비트, deprecated |
| **SHA-256** | 256비트, 표준 |
| **SHA-3** | 최신 표준 |
| **bcrypt** | 비밀번호용 |

**핵심 알고리즸:** 입력 → 해시함수 → 다이제스트

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**해시 알고리즘 비교**

| 알고리즘 | 길이 | 보안 |
|:---|:---|:---|
| **MD5** | 128bit | 취약 |
| **SHA-1** | 160bit | 취약 |
| **SHA-256** | 256bit | 안전 |
| **SHA-512** | 512bit | 안전 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 무결성 → SHA-256
**시나리오 2:** 비밀번호 → bcrypt
**시나리오 3:** 캐시 키 → MurmurHash
