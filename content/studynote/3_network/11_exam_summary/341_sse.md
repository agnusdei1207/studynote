# SSE - 서버 전송 이벤트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서버에서 클라이언트로 단방향 실시간
> 2. **가치**: HTTP 기반, 자동 재연결
> 3. **융합**: 알림, 피드, 모니터링

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SSE(Server-Sent Events)는 서버에서 클라이언트로 실시간 이벤트를 푸시합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SSE                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SSE 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ HTTP 요청 (Accept: text/event-stream)          │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           SSE Server                        │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  Content-Type: text/event-stream            │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  data: 이벤트 1                              │    │   │ │
│   │   │  │  data: 이벤트 2                              │    │   │ │
│   │   │  │  data: 이벤트 3                              │    │   │ │
│   │   │  │  ...                                         │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ 단방향 스트림                                 │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [Client 수신]                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SSE vs WebSocket                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ SSE      │ WebSocket│               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 방향       │ 단방향   │ 양방향   │               │   │ │
│   │   │  │ 프로토콜   │ HTTP     │ WS       │               │   │ │
│   │   │  | 재연결     │ 자동     │ 수동     │               │   │ │
│   │   │  │ 형식       │ 텍스트   │ 바이너리 │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SSE는 HTTP 기반으로 서버에서 클라이언트로 실시간 이벤트를 전송합니다.

**💡 비유**: SSE는 라디오 방송입니다. 서버가 계속 보냅니다.

**📢 섹션 요약 비유:** SSE = 서버 푸시!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Event** | 이벤트 |
| **Data** | 데이터 필드 |
| **ID** | 이벤트 ID |
| **Retry** | 재연결 간격 |

**비교표: SSE 이벤트 형식**

```
+------------+----------+----------+
| 필드       │ 설명     │ 예시     │
+------------+----------+----------+
| event      │ 이벤트명 │ message  │
| data       │ 데이터   │ {"x":1}  │
| id         │ ID       │ 123      │
| retry      │ 재시도   │ 3000     │
+------------+----------+----------+
```

**핵심 알고리즸:** 스트리밍, 자동 재연결

**📢 섹션 요약 비유:** text/event-stream = MIME!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 실시간 기술**

| 항목 | SSE | WebSocket | Long Polling |
|:---|:---|:---|:---|
| **방향** | 단방향 | 양방향 | 단방향 |
| **복잡도** | 낮음 | 중간 | 낮음 |
| **재연결** | 자동 | 수동 | 자동 |

**📢 섹션 요약 비유:** 알림 = SSE!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 알림** → SSE
**시나리오 2: 뉴스 피드** → SSE
**시나리오 3: 채팅** → WebSocket

**도입 체크리스트:** 단방향 여부, 연결 수, 캐싱

**📢 섹션 요약 비유:** 단방향 = SSE!
