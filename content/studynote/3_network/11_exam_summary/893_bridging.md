# Bridging - 브리징

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 세그먼트 연결
> 2. **가치**: 충돌 도메인 분리
> 3. **융합:** Transparent Bridge, STP, Learning

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Bridging은 두 네트워크 세그먼트를 연결합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Bridging                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              브리지 동작                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  네트워크 구조:                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  Segment A              Bridge            Segment B  │   │ │
│   │   │  ┌─────────┐          ┌───────┐          ┌─────────┐│   │ │
│   │   │  │ Host 1  │          │       │          │ Host 3  ││   │ │
│   │   │  │ Host 2  │ ──────── │Bridge │ ──────── │ Host 4  ││   │ │
│   │   │  └─────────┘          └───────┘          └─────────┘│   │ │
│   │   │  충돌 도메인 1         MAC Table          충돌 도메인 2│   │ │
│   │   │                                                      │   │ │
│   │   │  브리지 유형:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Transparent Bridge: MAC 학습, 투명              ││   │ │
│   │   │  │ Source Route Bridge: 경로 정보 사용             ││   │ │
│   │   │  │ Translation Bridge: 다른 매체 변환               ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  기능:                                                   │   │ │
│   │   │  - Learning: MAC 주소 학습                               │   │ │
│   │   │  - Forwarding: 프레임 전달                                │   │ │
│   │   │  - Filtering: 불필요 전송 차단                             │   │ │
│   │   │  - Flooding: 알 수 없는 MAC은 브로드캐스트                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 브리지는 충돌 도메인을 분리합니다.

**📢 섹션 요약 비유:** Bridging = 다리!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Segment** | 세그먼트 |
| **Forwarding** | 전달 |
| **Filtering** | 필터링 |
| **Learning** | 학습 |
| **Aging** | 에이징 |

**핵심 알고리즸:** 수신 → MAC 학습 → 전달/필터링

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Bridge vs Switch vs Router**

| 항목 | Bridge | Switch | Router |
|:---|:---|:---|:---|
| **계층** | L2 | L2/L3 | L3 |
| **포트** | 2-4 | 다수 | 다수 |
| **속도** | 느림 | 빠름 | 느림 |
| **도메인** | 충돌 분리 | 충돌 분리 | 브로드캐스트 분리 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 네트워크 확장 → Bridge
**시나리오 2:** VLAN → Switch
**시나리오 3:** WAN 연결 → Router
