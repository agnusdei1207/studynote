# Traceroute - 경로 추적

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 경로 추적 도구
> 2. **가치**: 네트워크 경로 확인
> 3. **융합:** TTL, ICMP Time Exceeded, Hops

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Traceroute는 패킷의 경로를 추적합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Traceroute                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Traceroute 동작                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  TTL 조작으로 경로 확인:                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  Hop 1 (TTL=1):                                        │   │ │
│   │   │  ┌─────┐        ┌─────┐                              │   │ │
│   │   │  │ Src │ ─────→ │ R1  │ ──X──→ (TTL 만료)            │   │ │
│   │   │  └─────┘        └─────┘                              │   │ │
│   │   │                   ↓                                   │   │ │
│   │   │              ICMP Time Exceeded                       │   │ │
│   │   │                                                      │   │ │
│   │   │  Hop 2 (TTL=2):                                        │   │ │
│   │   │  ┌─────┐        ┌─────┐        ┌─────┐                │   │ │
│   │   │  │ Src │ ─────→ │ R1  │ ─────→ │ R2  │ ──X──→         │   │ │
│   │   │  └─────┘        └─────┘        └─────┘                │   │ │
│   │   │                                     ↓                  │   │ │
│   │   │                              ICMP Time Exceeded         │   │ │
│   │   │                                                      │   │ │
│   │   │  Hop 3 (TTL=3): ... 목적지 도달                         │   │ │
│   │   │                                                      │   │ │
│   │   │  출력 예시:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ 1  192.168.1.1    1.2 ms   1.0 ms   1.1 ms      ││   │ │
│   │   │  │ 2  10.0.0.1       5.3 ms   5.1 ms   5.2 ms      ││   │ │
│   │   │  │ 3  142.250.x.x    15.2 ms  14.8 ms  15.0 ms     ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Traceroute는 각 홉의 지연과 경로를 보여줍니다.

**📢 섹션 요약 비유:** Traceroute = 경로 지도!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **TTL** | Time to Live |
| **Hop** | 홉 |
| **Time Exceeded** | 시간 초과 |
| **RTT** | 왕복 시간 |
| **Probe** | 프로브 |

**핵심 알고리즸:** TTL 증가 → 각 라우터 응답 → 경로 확인

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Traceroute vs Ping**

| 항목 | Ping | Traceroute |
|:---|:---|:---|
| **정보** | 연결 여부 | 전체 경로 |
| **용도** | 빠른 확인 | 문제 분석 |
| **시간** | 빠름 | 느림 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 연결 문제 → Traceroute
**시나리오 2:** 지연 원인 → 홉별 확인
**시나리오 3:** 경로 변경 → 모니터링
