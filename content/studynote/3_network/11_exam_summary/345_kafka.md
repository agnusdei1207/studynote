# Kafka - 분산 이벤트 스트리밍

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 이벤트 스트리밍 플랫폼
> 2. **가치**: 고처리량, 내구성, 확장성
> 3. **융합**: 빅데이터, 실시간 분석

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Kafka는 분산 이벤트 스트리밍 플랫폼입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Kafka                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Kafka 아키텍처                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Producer]                                         │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Produce (Topic)                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           Kafka Cluster                     │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌───────────────────────────────────────┐  │    │   │ │
│   │   │  │  │         Topic (파티션)                 │  │    │   │ │
│   │   │  │  │  [0] [1] [2] ... [N]                  │  │    │   │ │
│   │   │  │  └───────────────────────────────────────┘  │    │   │ │
│   │   │  │       │                                      │    │   │ │
│   │   │  │       │ Zookeeper/KRaft                     │    │   │ │
│   │   │  │       ▼                                      │    │   │ │
│   │   │  │  [Broker 1] [Broker 2] [Broker 3]           │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ Consume                                      │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [Consumer Group]                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Kafka 특징                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 특징       │ 설명     │ 가치     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 고처리량   │ 초당 백만│ 확장성   │               │   │ │
│   │   │  │ 내구성     │ 디스크   │ 영속성   │               │   │ │
│   │   │  | 파티셔닝   │ 분산     │ 병렬성   │               │   │ │
│   │   │  │ 복제       │ Replica  │ 가용성   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Kafka는 파티션 기반으로 고처리량 이벤트 스트리밍을 제공합니다.

**💡 비유**: Kafka는 분산 커밋 로그입니다. 모든 이벤트를 순서대로 저장합니다.

**📢 섹션 요약 비유:** Kafka = 이벤트 스트리밍!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Broker** | 브로커 |
| **Topic** | 토픽 |
| **Partition** | 파티션 |
| **Consumer Group** | 컨슈머 그룹 |

**비교표: Kafka vs RabbitMQ**

```
+------------+----------+----------+
| 항목       │ Kafka    │ RabbitMQ │
+------------+----------+----------+
| 모델       │ 로그     │ 큐       │
| 처리량     │ 높음     │ 중간     │
| 순서       │ 파티션   │ 큐       │
| 재생       │ 가능     │ 불가능   │
+------------+----------+----------+
```

**핵심 알고리즸:** 파티셔닝, 복제, 컨슈머 그룹

**📢 섹션 요약 비유:** Partition = 병렬성!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 스트리밍**
