# Error Detection - 오류 감지

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 오류 발견
> 2. **가치**: 데이터 무결성
> 3. **융합:** Checksum, CRC, Parity

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Error Detection은 전송 오류를 감지합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Error Detection                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              오류 감지 방법                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Parity Bit:                                          │   │ │
│   │   │  데이터 + 1비트 (홀수/짝수)                             │   │ │
│   │   │  홀수 개의 오류만 감지                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Checksum:                                            │   │ │
│   │   │  데이터 합의 보수                                       │   │ │
│   │   │  IP, TCP, UDP 헤더                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  CRC (Cyclic Redundancy Check):                        │   │ │
│   │   │  다항식 기반 나머지 계산                                 │   │ │
│   │   │  이더넷, HDLC                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  Hash:                                                │   │ │
│   │   │  MD5, SHA-1, SHA-256                                 │   │ │
│   │   │  파일 무결성, 디지털 서명                               │   │ │
│   │   │                                                      │   │ │
│   │   │  오류 감지 후:                                          │   │ │
│   │   │  - 재전송 요청 (ARQ)                                   │   │ │
│   │   │  - 폐기                                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 오류 감지는 데이터 무결성을 보장합니다.

**📢 섹션 요약 비유:** Error Detection = 무결성 인장!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Parity** | 패리티 |
| **Checksum** | 체크섬 |
| **CRC** | 순환 중복 검사 |
| **Hash** | 해시 |
| **MAC** | 메시지 인증 코드 |

**핵심 알고리즸:** 데이터 → 계산 → 부가 → 수신 → 검증

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**방법 비교**

| 방법 | 오버헤드 | 검출력 |
|:---|:---|:---|
| **Parity** | 1비트 | 낮음 |
| **Checksum** | 16비트 | 중간 |
| **CRC-32** | 32비트 | 높음 |
| **SHA-256** | 256비트 | 매우 높음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 네트워크 → CRC
**시나리오 2:** 프로토콜 → Checksum
**시나리오 3:** 보안 → Hash/MAC
