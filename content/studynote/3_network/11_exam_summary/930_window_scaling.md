# Window Scaling - 윈도우 스케일링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 윈도우 크기 확장
> 2. **가치**: 고속 네트워크 대응
> 3. **융합:** TCP Option, Shift Count, 1GB

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Window Scaling은 TCP 윈도우 크기를 확장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Window Scaling                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              윈도우 스케일링                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  기본 TCP 윈도우:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ 16-bit field = 최대 65,535 bytes (64KB)         ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  문제: 고속 네트워크에서 부족                        ││   │ │
│   │   │  │  예: 1Gbps, 100ms RTT                           ││   │ │
│   │   │  │      Bandwidth-Delay Product = 12.5MB           ││   │ │
│   │   │  │      → 64KB로는 부족                              ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  스케일링 옵션:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Scale Factor: 0-14                              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ 실제 윈도우 = Header Window × 2^Scale            ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ Scale 0: 64KB                                   ││   │ │
│   │   │  │ Scale 7: 8MB                                    ││   │ │
│   │   │  │ Scale 14: 1GB (최대)                             ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Window Scaling은 고속 네트워크에서 필수적입니다.

**📢 섹션 요약 비유:** Window Scaling = 고속도로 차선 확장!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Scale Factor** | 스케일 팩터 |
| **SYN** | 연결 설정 |
| **Option** | TCP 옵션 |
| **BDP** | 대역폭-지연 곱 |
| **Max Window** | 최대 윈도우 |

**핵심 알고리듬:** SYN에서 협상 → 2^Scale 곱셈
