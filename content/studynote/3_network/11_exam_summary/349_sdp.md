# SDP - 세션 설명 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멀티미디어 세션 설명 형식
> 2. **가치**: 미디어 협상, 코덱 설명
> 3. **융합**: WebRTC, SIP, VoIP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SDP(Session Description Protocol)는 멀티미디어 세션을 설명합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SDP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SDP 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  v=0                              # 버전             │   │ │
│   │   │  o=- 123456 2 IN IP4 192.168.1.1  # Origin          │   │ │
│   │   │  s=Session                        # 세션명           │   │ │
│   │   │  t=0 0                            # 시간             │   │ │
│   │   │  m=audio 49170 RTP/AVP 0          # 미디어           │   │ │
│   │   │  a=rtpmap:0 PCMU/8000            # 속성             │   │ │
│   │   │  m=video 49172 RTP/AVP 96        # 미디어           │   │ │
│   │   │  a=rtpmap:96 H264/90000          # 속성             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SDP 구성 요소                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 필드       │ 설명     │ 예시     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ v          │ 버전     │ 0        │               │   │ │
│   │   │  │ o          │ Origin   │ 세션ID   │               │   │ │
│   │   │  | m          │ 미디어   │ audio/video│              │   │ │
│   │   │  │ a          │ 속성     │ 코덱     │               │   │ │
│   │   │  │ c          │ 연결     │ IP       │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SDP는 미디어 세션의 코덱, 주소, 포트를 설명합니다.

**💡 비유**: SDP는 음악회 프로그램입니다. 무엇을 연주할지 알려줍니다.

**📢 섹션 요약 비유:** SDP = 세션 설명!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Session** | 세션 레벨 |
| **Media** | 미디어 레벨 |
| **Attribute** | 속성 |
| **Offer/Answer** | 제안/응답 |

**비교표: WebRTC SDP 확장**

```
+------------+----------+----------+
| 항목       │ 기본 SDP │ WebRTC   │
+------------+----------+----------+
| ICE        │ 없음     │ 있음     │
| DTLS       │ 없음     │ 있음     │
| BUNDLE     │ 없음     │ 있음     │
+------------+----------+----------+
```

**핵심 알고리즸:** Offer/Answer 모델

**📢 섹션 요약 비유:** Offer/Answer = 협상!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 세션 설명**

| 항목 | SDP | H.245 | SIP |
|:---|:---|:---|:---|
| **용도** | 설명 | 제어 | 시그널링 |
| **형식** | 텍스트 | ASN.1 | 텍스트 |
| **복잡도** | 낮음 | 높음 | 중간 |

**📢 섹션 요약 비유:** WebRTC = SDP!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: WebRTC** → SDP Offer/Answer
**시나리오 2: SIP** → SDP 포함
**시나리오 3: 스트리밍** → SDP

**도입 체크리스트:** 코덱 협상, ICE, DTLS

**📢 섹션 요약 비유:** 미디어 협상 = SDP!

### Ⅴ. 기대효과 및 결론 (Future & Standard)
