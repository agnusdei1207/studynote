# Slotted ALOHA (슬롯 알로하)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시간을 슬롯으로 나누어 슬롯 시작에만 전송
> 2. **가치**: Pure ALOHA 대비 2배 효율
> 3. **융합**: 위성, RFID, IoT와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Slotted ALOHA는 시간을 고정된 슬롯으로 나누어 슬롯 시작 시점에만 전송할 수 있게 한 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Slotted ALOHA (슬롯 알로하)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Slotted ALOHA 동작                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  알고리즘:                                              │   │ │
│   │   │  1. 시간을 고정된 슬롯으로 분할                          │   │ │
│   │   │  2. 프레임은 슬롯 시작에만 전송 가능                      │   │ │
│   │   │  3. 한 슬롯에 두 프레임 = 충돌                           │   │ │
│   │   │  4. 충돌 시 랜덤 슬롯 선택 후 재전송                      │   │ │
│   │   │                                                      │   │ │
│   │   │  타임라인:                                                │   │ │
│   │   │  슬롯: │  1  │  2  │  3  │  4  │  5  │  6  │          │   │ │
│   │   │        ├─────┼─────┼─────┼─────┼─────┼─────┤          │   │ │
│   │   │  노드A:│█████│     │     │     │     │     │          │   │ │
│   │   │  노드B:│█████│     │     │     │█████│     │          │   │ │
│   │   │        ↑ 충돌              ↑ 성공                      │   │ │
│   │   │  노드C:│     │█████│     │     │     │     │          │   │ │
│   │   │                      ↑ 성공                             │   │ │

│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │

│   ┌──────────────────────────────────────────────────────────────┐ │

│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  처리량:                                                 │   │ │
│   │   │  S = G × e^(-G)                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  최대 처리량:                                            │   │ │
│   │   │  G = 1.0일 때 S = 0.368 (36.8%)                         │   │ │
│   │   │                                                      │   │ │

│   │   │    ▲         * (최대 36.8%)                              │   │ │
│   │   │    │        / \                                          │   │ │
│   │   │    │       /   \                                         │   │ │
│   │   │    │      /     \                                        │   │ │
```

> **해설**: Slotted ALOHA는 동기화 비용을 감수하고 효율을 2배 높였습니다.

**💡 비유**: Slotted ALOHA는 버스 시간표에 맞춰 타는 것입니다.

**📢 섹션 요약 비유**: Slotted ALOHA 최대 효율 = 36.8%!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **슬롯** | 시간 단위 | 프레임 길이 | 틱 |
| **동기** | 정렬 | 클럭 | 시계 |
| **T** | 취약 구간 | 1 슬롯 | 위험 |

**비교표**

```
+------------+----------+----------+
| 항목       | Pure     | Slotted  |
+------------+----------+----------+
| 취약 구간  | 2T       | T        |
| 최대 S     | 0.184    | 0.368    |
| 최적 G     | 0.5      | 1.0      |
| 동기       | 불필요   | 필수     |
| 복잡도     | 낮음     | 중간     |
+------------+----------+----------+
```

**핵심 알고리즸**: 슬롯 대기 → 슬롯 시작에 전송 → ACK 확인 → 실패 시 재전송

**📢 섹션 요약 비유**: 동기화 = 2배 효율!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 동기 가능** → Slotted ALOHA
**시나리오 2: 동기 불가** → Pure ALOHA
**시나리오 3: 고부하** → CSMA

**도입 체크리스트**: 동기 능력, 부하, 복잡도

**📢 섹션 요약 비유**: 동기화 비용 vs 효율!
