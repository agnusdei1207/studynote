# OID - 객체 식별자

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: MIB 객체의 고유 식별자
> 2. **가치**: 정확한 객체 참조
> 3. **융합**: SNMP, MIB와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

OID(Object Identifier)는 MIB 트리에서 각 객체를 고유하게 식별하는 점 표기법 숫자입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OID                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OID 구조                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  OID 예시: 1.3.6.1.2.1.1.1.0                        │   │ │
│   │   │              │ │ │ │ │ │ │ │                        │   │ │
│   │   │              │ │ │ │ │ │ │ └─ Instance (0)          │   │ │
│   │   │              │ │ │ │ │ │ └─── sysDescr (1)          │   │ │
│   │   │              │ │ │ │ │ └───── system (1)            │   │ │
│   │   │              │ │ │ │ └─────── mib-2 (1)             │   │ │
│   │   │              │ │ │ └───────── mgmt (2)              │   │ │
│   │   │              │ │ └─────────── internet (1)          │   │ │
│   │   │              │ └───────────── dod (6)               │   │ │
│   │   │              └─────────────── org (3)               │   │ │
│   │   │               iso (1)                               │   │ │
│   │   │                                                      │   │ │
│   │   │  = iso.org.dod.internet.mgmt.mib-2.system.sysDescr.0│   │ │
│   │   │  = 시스템 설명 (예: "Cisco IOS...")                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 OID 예시                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  +--------------------------+----------------------+ │   │ │
│   │   │  │ OID                      │ 설명                 │ │   │ │
│   │   │  +--------------------------+----------------------+ │   │ │
│   │   │  │ 1.3.6.1.2.1.1.1.0        │ sysDescr (시스템)    │ │   │ │
│   │   │  │ 1.3.6.1.2.1.1.3.0        │ sysUpTime (가동시간) │ │   │ │
│   │   │  │ 1.3.6.1.2.1.1.5.0        │ sysName (호스트명)   │ │   │ │
│   │   │  │ 1.3.6.1.2.1.2.2.1.10.X   │ ifInOctets (수신)    │ │   │ │
│   │   │  │ 1.3.6.1.2.1.2.2.1.16.X   │ ifOutOctets (송신)   │ │   │ │
│   │   │  │ 1.3.6.1.2.1.2.2.1.8.X    │ ifOperStatus (상태)  │ │   │ │
│   │   │  +--------------------------+----------------------+ │   │ │
│   │   │  (X는 인터페이스 인덱스)                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: OID는 점으로 구분된 숫자로 MIB 객체를 식별합니다.

**💡 비유**: OID는 우편번호입니다. 정확한 위치를 가리킵니다.

**📢 섹션 요약 비유:** OID = 객체 주소!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **숫자 OID** | 1.3.6.1... |
| **이름 OID** | iso.org... |
| **Instance** | .0, .X |
| **Wildcard** | .X |

**비교표: OID 표기법**

```
+------------+----------+----------+
| 표기법     │ 예시     │ 용도     │
+------------+----------+----------+
| 숫자       │ 1.3.6... │ 프로그램 │
| 이름       │ iso.org..│ 사람     │
+------------+----------+----------+
```

**핵심 알고리즸:** 트리 순회, 인스턴스

**📢 섹션 요약 비유:** Instance = 실제 값!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: OID 타입**

| 타입 | 설명 |
|:---|:---|
| **Scalar** | 단일 (.0) |
| **Tabular** | 테이블 (.X) |
| **Walk** | 순회 |

**📢 섹션 요약 비유:** SNMP Walk = 트리 순회!
