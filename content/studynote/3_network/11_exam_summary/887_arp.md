# ARP - 주소 결정 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP → MAC 변환
> 2. **가치**: 로컬 통신 가능
> 3. **융합:** ARP Table, Cache, Spoofing

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ARP (Address Resolution Protocol)는 IP를 MAC으로 변환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ARP                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ARP 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  과정:                                                 │   │ │
│   │   │  Host A (IP: 192.168.1.10)                            │   │ │
│   │   │  "192.168.1.20의 MAC은?"                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ 1. ARP Request (Broadcast)                      ││   │ │
│   │   │  │    "Who has 192.168.1.20? Tell 192.168.1.10"   ││   │ │
│   │   │  │              ↓                                   ││   │ │
│   │   │  │ 2. ARP Reply (Unicast)                          ││   │ │
│   │   │  │    "192.168.1.20 is at aa:bb:cc:dd:ee:ff"      ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  ARP Table (Cache):                                    │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ IP Address      MAC Address         TTL         ││   │ │
│   │   │  │ 192.168.1.1     aa:bb:cc:00:00:01   300s        ││   │ │
│   │   │  │ 192.168.1.20    aa:bb:cc:dd:ee:ff   300s        ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  보안 문제:                                              │   │ │
│   │   │  - ARP Spoofing: 가짜 ARP 응답                          │   │ │
│   │   │  - ARP Cache Poisoning: 캐시 오염                       │   │ │
│   │   │                                                      │   │ │
│   │   │  관련 프로토콜:                                            │   │ │
│   │   │  - RARP: MAC → IP (역방향, 사용 안함)                    │   │ │
│   │   │  - Gratuitous ARP: 자신의 IP 알림                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ARP는 L3와 L2를 연결합니다.

**📢 섹션 요약 비유:** ARP = 전화번호부 조회!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Request** | 요청 |
| **Reply** | 응답 |
| **Cache** | 캐시 |
| **TTL** | 유효 시간 |
| **Broadcast** | 브로드캐스트 |

**핵심 알고리즸:** 캐시 확인 → 없으면 Request → Reply → 저장

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ARP vs RARP**

| 항목 | ARP | RARP |
|:---|:---|:---|
| **방향** | IP→MAC | MAC→IP |
| **용도** | 통신 | 부팅 |
| **현재** | 사용 | 미사용 |
| **대체** | - | DHCP |
