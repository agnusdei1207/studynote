# LLDP - 링크 레이어 디스커버리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: L2 장비 자동 발견
> 2. **가치**: 토폴로지 자동 매핑
> 3. **융합**: VoIP, PoE와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

LLDP(Link Layer Discovery Protocol)는 IEEE 802.1AB 표준으로 이웃 장비 정보를 교환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LLDP                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LLDP 동작                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Switch A] ◄──── LLDP ────▶ [Switch B]             │   │ │
│   │   │     │                                │                │   │ │
│   │   │     │ 정보:                          │ 정보:          │   │ │
│   │   │     │ • Chassis ID (MAC)            │ • Chassis ID   │   │ │
│   │   │     │ • Port ID                     │ • Port ID      │   │ │
│   │   │     │ • TTL                         │ • TTL          │   │ │
│   │   │     │ • System Name                 │ • System Name  │   │ │
│   │   │     │ • Capabilities                │ • Capabilities │   │ │
│   │   │     │ • Management IP               │ • Management IP│   │ │
│   │   │     │                                │                │   │ │
│   │   │     ▼                                ▼                │   │ │
│   │   │  ┌──────────────────────────────────────────────┐    │   │ │
│   │   │  │            LLDP Neighbor Table               │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  Switch A:                                   │    │   │ │
│   │   │  │  Neighbor: Switch B, Port: Gi0/1             │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  Switch B:                                   │    │   │ │
│   │   │  │  Neighbor: Switch A, Port: Gi0/24            │    │   │ │
│   │   │  └──────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LLDP TLV                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ TLV        │ 필수     │ 설명     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Chassis ID │ O        │ 장비 식별│               │   │ │
│   │   │  │ Port ID    │ O        │ 포트 식별│               │   │ │
│   │   │  │ TTL        │ O        │ 유효 시간│               │   │ │
│   │   │  │ Port Desc  │ X        │ 포트 설명│               │   │ │
│   │   │  │ System Name│ X        │ 시스템명 │               │   │ │
│   │   │  │ System Desc│ X        │ 설명     │               │   │ │
│   │   │  │ Capabilities│ X       │ 기능     │               │   │ │
│   │   │  │ Mgmt Addr  │ X        │ 관리 IP  │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  (O: 필수, X: 선택)                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: LLDP는 이웃 장비에게 자신의 정보를 광고합니다.

**💡 비유**: LLDP는 명함 교환입니다. 이웃에게 자신을 소개합니다.

**📢 섹션 요약 비유:** LLDP = 이웃 발견!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Chassis ID** | 장비 식별 |
| **Port ID** | 포트 식별 |
| **TTL** | 유효 시간 |
| **Capability** | 기능 |

**비교표: LLDP vs CDP**

```
+------------+----------+----------+
| 항목       │ LLDP     │ CDP      │
+------------+----------+----------+
| 표준       │ IEEE     │ Cisco    │
| 범용성     │ 모든 벤더│ Cisco만  │
| TLV        │ 표준     │ Cisco    │
| VoIP       │ LLDP-MED │ CDP      │
+------------+----------+----------+
```

**핵심 알고리즸:** TLV, 멀티캐스트

**📢 섹션 요약 비유:** LLDP = 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: LLDP-MED**

| TLV | 설명 |
|:---|:---|
| **LLDP-MED** | VoIP 확장 |
| **Power** | PoE 협상 |
| **Location** | 위치 정보 |

**📢 섹션 요약 비유:** LLDP-MED = VoIP!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 토폴로지 매핑** → LLDP
**시나리오 2: VoIP** → LLDP-MED
**시나리오 3: Cisco 환경** → CDP + LLDP

**도입 체크리스트:** 벤더, 토폴로지, VoIP

**📢 섹션 요약 비유:** LLDP는 표준 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 자동 발견

**미래 전망:** 데이터센터 확장

**참고 표준:** IEEE 802.1AB
