# NTP - Network Time Protocol

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 시간 동기화 프로토콜
> 2. **가치**: 로그 정확성, 인증 신뢰
> 3. **융합**: Stratum, 인증, 보안

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NTP는 네트워크로 시간을 동기화합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NTP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NTP Stratum 계층                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Stratum 0: GPS/원자시계                             │   │ │
│   │   │        │                                             │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  Stratum 1: 1차 타임서버                             │   │ │
│   │   │        │                                             │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  Stratum 2: 2차 타임서버                             │   │ │
│   │   │        │                                             │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  Stratum 3: 로컬 서버                                │   │ │
│   │   │        │                                             │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  Stratum 4+: 클라이언트                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NTP 특징                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ 설명     │ 비고     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 포트       │ UDP 123  │          │               │   │ │
│   │   │  │ 정확도     │ ms~μs    │ 환경별   │               │   │ │
│   │   │  | 인증       │ 대칭키   │ 보안     │               │   │ │
│   │   │  │ 계층       │ Stratum  │ 0-15     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: NTP는 계층적 구조로 시간을 전파합니다.

**💡 비유**: NTP는 모든 시계를 맞추는 것입니다. 모두가 같은 시간을 봅니다.

**📢 섹션 요약 비유:** NTP = 시간 동기화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Stratum** | 시간 계층 |
| **Reference Clock** | 기준 시계 |
| **Drift** | 시간 오차 |
| **Jitter** | 지터 |

**비교표: 시간 프로토콜**

```
+------------+----------+----------+
| 항목       │ NTP      │ PTP      │
+------------+----------+----------+
| 정확도     │ ms       │ ns       │
| 용도       │ 일반     │ 금융/산업│
| 복잡도     │ 낮음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** RTT 보정, 계층 선택

**📢 섹션 요약 비유:** Kerberos = NTP 필수!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: NTP 보안**

| 항목 | NTP 인증 | NTS | chrony |
|:---|:---|:---|:---|
| **보안** │ 대칭키 │ TLS | 인증 |
| **현대성** │ 레거시 │ 현대 | 현대 |

**📢 섹션 요약 비유:** 로그 = NTP 필수!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일반 서버 → 공용 NTP
**시나리오 2:** 기업 → 내부 NTP 서버
**시나리오 3:** 금융 → PTP

**도입 체크리스트:** 계층 설계, 인증, 이중화

**📢 섹션 요약 비유:** 시간 = 보안 기반!
