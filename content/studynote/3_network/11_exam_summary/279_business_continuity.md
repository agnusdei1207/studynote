# 비즈니스 연속성 (Business Continuity) - BC

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 중단 없는 비즈니스 운영
> 2. **가치**: 위험 관리, 신뢰
> 3. **융합**: DR, HA와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

비즈니스 연속성(BC)은 장애나 재해 시에도 비즈니스 운영을 지속하는 전략입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    비즈니스 연속성 (Business Continuity)              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              BC 프레임워크                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           BIA (영향평가)                     │    │   │ │
│   │   │  │  • 업무 프로세스 분석                        │    │   │ │
│   │   │  │  • RTO/RPO 정의                             │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           위험 분석                          │    │   │ │
│   │   │  │  • 위험 식별                                 │    │   │ │
│   │   │  │  • 영향도 평가                               │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           대응 전략                          │    │   │ │
│   │   │  │  • HA, DR, 백업                             │    │   │ │
│   │   │  │  • 복구 절차                                │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           테스트 및 개선                      │    │   │ │
│   │   │  │  • 정기 훈련                                 │    │   │ │
│   │   │  │  • 계획 업데이트                             │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              BC vs DR                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  BC (비즈니스 연속성):                                │   │ │
│   │   │     • 전체 비즈니스 관점                              │   │ │
│   │   │     • 프로세스, 인력, 시설                            │   │ │
│   │   │                                                      │   │ │
│   │   │  DR (재해 복구):                                      │   │ │
│   │   │     • IT 시스템 복구                                  │   │ │
│   │   │     • BC의 하위 집합                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: BC는 IT뿐 아니라 전체 비즈니스 연속성을 다룹니다.

**💡 비유**: BC는 회사의 비상 계획입니다. 모든 상황에 대비합니다.

**📢 섹션 요약 비유:** BC = 전체 비즈니스!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **BIA** | 영향 평가 |
| **RA** | 위험 분석 |
| **BCP** | 계획 |
| **DRP** | IT 복구 |

**비교표: BC 구성요소**

```
+------------+----------+----------+
| 구성요소   │ 범위     │ 내용     │
+------------+----------+----------+
| BIA        │ 전사     │ 영향 분석│
| BCP        │ 전사     │ 계획     │
| DRP        │ IT       │ 복구     │
| CM         │ 위기     │ 커뮤니케이션│
+------------+----------+----------+
```

**핵심 알고리즸:** BIA, 위험 분석

**📢 섹션 요약 비유:** BIA = 무엇이 중요한가!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: RTO 등급**

| 등급 | RTO | 설명 |
|:---|:---|:---|
| **Tier 1** | 0 | 무중단 |
| **Tier 2** | < 1시간 | 매우 빠름 |
| **Tier 3** | < 4시간 | 빠름 |
| **Tier 4** | < 24시간 | 보통 |

**📢 섹션 요약 비유:** Tier 1 = 무중단!
