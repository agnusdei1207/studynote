# Subnet - 서브넷

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 분할
> 2. **가치**: IP 주소 효율성
> 3. **융합:** Subnet Mask, CIDR, VLSM

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Subnet은 큰 네트워크를 작은 네트워크로 분할합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Subnet                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              서브넷팅                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IP 주소 구조:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ 192.168.1.0/24                                   ││   │ │
│   │   │  │ Network    Host                                  ││   │ │
│   │   │  │ (24 bits)  (8 bits = 256 주소)                    ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  서브넷팅 예시 (/24 → /26):                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ 192.168.1.0/26   → 62 hosts                     ││   │ │
│   │   │  │ 192.168.1.64/26  → 62 hosts                     ││   │ │
│   │   │  │ 192.168.1.128/26 → 62 hosts                     ││   │ │
│   │   │  │ 192.168.1.192/26 → 62 hosts                     ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  CIDR 표기법:                                            │   │ │
│   │   │  192.168.1.0/24 = 255.255.255.0                        │   │ │
│   │   │  10.0.0.0/8 = 255.0.0.0                                │   │ │
│   │   │                                                      │   │ │
│   │   │  VLSM (Variable Length Subnet Mask):                    │   │ │
│   │   │  필요에 따라 다양한 크기의 서브넷 사용                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 서브넷팅은 IP 주소 공간을 효율적으로 사용합니다.

**📢 섹션 요약 비유:** Subnet = 구역 나누기!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Network ID** | 네트워크 식별자 |
| **Host ID** | 호스트 식별자 |
| **Subnet Mask** | 서브넷 마스크 |
| **CIDR** | Classless 표기 |
| **Broadcast** | 브로드캐스트 주소 |

**핵심 알고리즸:** IP AND Mask = Network ID

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**서브넷 크기 비교**

| CIDR | 마스크 | 호스트 수 |
|:---|:---|:---|
| /24 | 255.255.255.0 | 254 |
| /25 | 255.255.255.128 | 126 |
| /26 | 255.255.255.192 | 62 |
| /27 | 255.255.255.224 | 30 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 사무실 → /24
**시나리오 2:** 부서별 → /26
**시나리오 3:** 점 대점 → /30
