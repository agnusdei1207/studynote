# Link Utilization - 링크 활용률

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 대역폭 사용 효율
> 2. **가치**: 자원 최적화
> 3. **융합:** Efficiency, Congestion, QoS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Link Utilization은 링크 대역폭 활용 정도입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Link Utilization                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              활용률 계산                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  공식:                                                │   │ │
│   │   │  Utilization = (실제 사용량 / 총 대역폭) × 100%        │   │ │
│   │   │                                                      │   │ │
│   │   │  시각화:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ 100% ┌────────────────────────────────────┐    ││   │ │
│   │   │  │      │████████████████████████████████│ ← 80% ││   │ │
│   │   │  │  50% │███████████████░░░░░░░░░░░░░░░░░│ ← 50% ││   │ │
│   │   │  │   0% └────────────────────────────────────┘    ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  목표:                                                │   │ │
│   │   │  - 너무 낮음: 자원 낭비                               │   │ │
│   │   │  - 너무 높음: 혼잡 위험                               │   │ │
│   │   │  - 이상적: 60-80%                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  Stop-and-Wait 효율:                                  │   │ │
│   │   │  U = (Frame Time) / (Frame Time + 2 × Prop. Delay)   │   │ │
│   │   │                                                      │   │ │
│   │   │  Sliding Window 효율:                                 │   │ │
│   │   │  U = min(1, N × Frame Time / RTT)                    │   │ │
│   │   │  N: 윈도우 크기                                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 링크 활용률은 네트워크 효율성의 핵심 지표입니다.

**📢 섹션 요약 비유:** Link Utilization = 도로 점유율!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Utilization** | 활용률 |

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**활용률 vs 성능**

| 활용률 | 상태 | 권장 |
|:---|:---|:---|
| **0-30%** | 저활용 | 증설 검토 |

| **80-100%** | 과부하 | 증설 |
