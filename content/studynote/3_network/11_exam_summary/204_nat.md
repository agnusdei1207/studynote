# NAT (Network Address Translation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 주소 변환
> 2. **가치**: IPv4 주소 절약, 보안
> 3. **융합**: 방화벽, 로드밸런서와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NAT(Network Address Translation)는 패킷의 IP 주소를 변환하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NAT (Network Address Translation)                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NAT 종류                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Static NAT (정적 NAT):                             │   │ │
│   │   │     • 1:1 매핑                                        │   │ │
│   │   │     • 사설 IP ↔ 공인 IP                                │   │ │
│   │   │     • 서버 공개용                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Dynamic NAT (동적 NAT):                            │   │ │
│   │   │     • N:M 매핑                                        │   │ │
│   │   │     • 풀에서 할당                                      │   │ │
│   │   │     • 사설 IP → 공인 IP 풀                             │   │ │
│   │   │                                                      │   │ │
│   │   │  3. PAT/NAPT (Port Address Translation):               │   │ │
│   │   │     • N:1 매핑                                        │   │ │
│   │   │     • 포트 기반 구분                                   │   │ │
│   │   │     • 가장 일반적                                      │   │ │
│   │   │     • 공인 IP 1개로 여러 호스트                        │   │ │
│   │   │                                                      │   │ │
│   │   │  4. NAT Overload (= PAT):                              │   │ │
│   │   │     • Dynamic NAT + PAT                               │   │ │
│   │   │     • 가정/소호용                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NAT 동작 예시                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [내부]              [NAT]               [외부]      │   │ │
│   │   │                                                      │   │ │
│   │   │  192.168.1.10    →  203.0.113.1:20001  →  8.8.8.8   │   │ │
│   │   │  :5000                (공인 IP:포트)                 │   │ │
│   │   │                                                      │   │ │
│   │   │  NAT 테이블:                                           │   │ │
│   │   │  내부 IP      내부 포트  외부 IP       외부 포트      │   │ │
│   │   │  192.168.1.10  5000      203.0.113.1   20001        │   │ │
│   │   │  192.168.1.11  5000      203.0.113.1   20002        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: PAT는 하나의 공인 IP로 여러 사설 IP를 지원합니다.

**💡 비유**: NAT는 아파트 우편함입니다. 하나의 주소로 여러 호수를 구분합니다.

**📢 섹션 요약 비유:** PAT = 포트 기반 N:1 NAT!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 용어 | 정의 | 예시 |
|:---|:---|:---|
| **Inside Local** | 내부 사설 IP | 192.168.1.10 |
| **Inside Global** | 내부 공인 IP | 203.0.113.1 |
| **Outside Local** | 외부 사설 IP | N/A |
| **Outside Global** | 외부 공인 IP | 8.8.8.8 |

**비교표**

```
+------------+----------+----------+
| 항목       │ Static   │ PAT      │
+------------+----------+----------+
| 매핑       │ 1:1      │ N:1      │
│ 공인 IP    │ 여러 개   │ 1개      │
│ 포트       │ 유지     │ 변경     │
│ 용도       │ 서버     │ 인터넷   │
+------------+----------+----------+
```

**핵심 알고리즸:** NAT 테이블, 타임아웃

**📢 섹션 요약 비유:** NAT 테이블 = 매핑 DB!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 사설 IP 대역**

| 클래스 | 대역 | 용도 |
|:---|:---|:---|
| **A** | 10.0.0.0/8 | 대규모 |
| **B** | 172.16.0.0/12 | 중규모 |
| **C** | 192.168.0.0/16 | 소규모 |
| **CGN** | 100.64.0.0/10 | ISP |

**📢 섹션 요약 비유:** RFC 1918 = 사설 IP 표준!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 서버 공개** → Static NAT
**시나리오 2: 인터넷 접속** → PAT
**시나리오 3: 대규모** → Dynamic NAT + PAT

**도입 체크리스트:** 공인 IP 수, 서버, 사용자 수

**📢 섹션 요약 비유:** 공인 IP 부족 = PAT 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: IPv4 주소 절약

**미래 전망:** IPv6로 NAT 불필요 (하지만 여전히 사용)

**참고 표준:** RFC 3022, RFC 1918

**📢 섹션 요약 비유:** NAT는 IPv4의 구원투수!
