# Routing - 라우팅

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 경로 결정
> 2. **가치**: 패킷 전달
> 3. **융합:** Static, Dynamic, Routing Table

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Routing은 패킷의 최적 경로를 결정합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Routing                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              라우팅 유형                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Static Routing:                                        │   │ │
│   │   │  - 관리자가 수동으로 경로 설정                            │   │ │
│   │   │  - 간단하지만 유연성 없음                                 │   │ │
│   │   │  - 소규모 네트워크에 적합                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  Dynamic Routing:                                       │   │ │
│   │   │  - 라우팅 프로토콜로 자동 학습                             │   │ │
│   │   │  - RIP, OSPF, EIGRP, BGP                               │   │ │
│   │   │  - 대규모 네트워크에 적합                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  Routing Table:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Destination    Gateway      Interface  Metric   ││   │ │
│   │   │  │ 10.0.0.0/8     192.168.1.1  eth0      1         ││   │ │
│   │   │  │ 0.0.0.0/0      192.168.1.1  eth0      1 (Default)││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  경로 선택 기준:                                          │   │ │
│   │   │  1. Longest Prefix Match                               │   │ │
│   │   │  2. Administrative Distance                             │   │ │
│   │   │  3. Metric                                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 라우팅은 네트워크 통신의 핵심입니다.

**📢 섹션 요약 비유:** Routing = 내비게이션!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Route** | 경로 |
| **Gateway** | 게이트웨이 |
| **Metric** | 비용 |
| **AD** | 관리 거리 |
| **Next Hop** | 다음 홉 |

**핵심 알고리즸:** 목적지 확인 → 테이블 조회 → 최적 경로 선택

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**라우팅 프로토콜 비교**

| 프로토콜 | 유형 | AD | 용도 |
|:---|:---|:---|:---|
| **Static** | - | 1 | 기본 |
| **RIP** | Distance | 120 | 소규모 |
| **OSPF** | Link State | 110 | 기업 |
| **BGP** | Path Vector | 20/200 | 인터넷 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 소규모 → Static
**시나리오 2:** 기업 → OSPF
**시나리오 3:** ISP → BGP
