# Istio - 서비스 메시 플랫폼

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오픈소스 서비스 메시
> 2. **가치**: 트래픽, 보안, 관찰
> 3. **융합**: K8s, Envoy와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Istio는 Kubernetes 환경에서 서비스 메시 기능을 제공하는 오픈소스 플랫폼입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Istio                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Istio 아키텍처                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Control Plane] - Istiod                           │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ 구성요소:                                    │   │ │
│   │   │       │ • Pilot (트래픽 관리)                        │   │ │
│   │   │       │ • Citadel (인증서)                           │   │ │
│   │   │       │ • Galley (설정 검증)                         │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           [Data Plane]                       │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌─────────────────────────────────────┐    │    │   │ │
│   │   │  │  │  Pod                                │    │    │   │ │
│   │   │  │  │  ┌──────────┐  ┌──────────┐        │    │    │   │ │
│   │   │  │  │  │ App      │  │ Envoy    │        │    │    │   │ │
│   │   │  │  │  │ Container│  │ Sidecar  │        │    │    │   │ │
│   │   │  │  │  └──────────┘  └──────────┘        │    │    │   │ │
│   │   │  │  └─────────────────────────────────────┘    │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Istio 리소스                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. VirtualService:                                  │   │ │
│   │   │     • 라우팅 규칙                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  2. DestinationRule:                                 │   │ │
│   │   │     • 서브셋, 로드밸런싱                             │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Gateway:                                         │   │ │
│   │   │     • Ingress/Egress                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  4. AuthorizationPolicy:                             │   │ │
│   │   │     • 접근 제어                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Istio는 Envoy Sidecar로 트래픽을 제어합니다.

**💡 비유**: Istio는 마이크로서비스의 교통경찰입니다.

**📢 섹션 요약 비유:** Istio = 트래픽 제어!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Istiod** | 통합 컨트롤 플레인 |
| **Envoy** | 데이터 플레인 |
| **VirtualService** | 라우팅 |
| **DestinationRule** | 서브셋 |

**비교표: Istio 버전**

```
+------------+----------+----------+
| 버전       │ 구조     │ 특징     │
+------------+----------+----------+
| 1.5 이전   │ 분산     │ 복잡     │
| 1.5+       │ 통합     │ Istiod   │
| 1.10+      │ 안정     │ Ambient  │
+------------+----------+----------+
```

**핵심 알고리즸:** 라우팅, mTLS

**📢 섹션 요약 비유:** Istiod = 단일 컨트롤!
