# ALOHA (순수 알로하)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 무작위 채널 접근, 충돌 허용
> 2. **가치**: 최초의 무선 패킷 네트워크
> 3. **융합**: 위성 통신, IoT와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ALOHA는 하와이 대학에서 개발된 최초의 무선 패킷 네트워크로, 노드가 전송하고 싶을 때 즉시 전송하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ALOHA (순수 알로하)                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Pure ALOHA 동작                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  알고리즘:                                              │   │ │
│   │   │  1. 데이터 있으면 즉시 전송                              │   │ │
│   │   │  2. 충돌 여부 모름 (ACK 대기)                            │   │ │
│   │   │  3. ACK 오면 성공                                         │   │ │
│   │   │  4. ACK 없으면 랜덤 대기 후 재전송                        │   │ │
│   │   │                                                      │   │ │
│   │   │  타임라인:                                                │   │ │
│   │   │  노드 A: ────[프레임]────                                │   │ │
│   │   │  노드 B:      ────[프레임]────                           │   │ │
│   │   │                 ↑ 충돌                                   │   │ │
│   │   │  노드 C: ────────────[프레임]────                        │   │ │
│   │   │                                                      │   │ │
│   │   │  충돌 조건:                                               │   │ │
│   │   │  • 두 프레임이 겹치면 충돌                                │   │ │
│   │   │  • 전체 전송 시간 동안 다른 전송 없어야 성공               │   │ │
│   │   │  • 취약 구간: 2 × 프레임 시간                             │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ALOHA 성능                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  처리량 (Throughput):                                   │   │ │
│   │   │  S = G × e^(-2G)                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  G: 제공 부하 (프레임/프레임 시간)                        │   │ │
│   │   │  S: 성공 처리량                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  최대 처리량:                                            │   │ │
│   │   │  G = 0.5일 때 S = 0.184 (18.4%)                         │   │ │
│   │   │                                                      │   │ │
│   │   │  S                                          │   │ │
│   │   │    ▲    * (최대 18.4%)                                   │   │ │
│   │   │    │   / \                                               │   │ │
│   │   │    │  /   \                                              │   │ │
│   │   │    │ /     \                                             │   │ │
│   │   │    └──────────────→ G                                    │   │ │
│   │   │       0.5                                                │   │ │
│   │   │                                                      │   │ │
│   │   │  한계:                                                   │   │ │
│   │   │  • 낮은 효율 (최대 18.4%)                                │   │ │
│   │   │  • 높은 충돌 확률                                        │   │ │
│   │   │  • 불안정 (G > 0.5에서 급격히 악화)                       │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: ALOHA는 1971년 하와이 대학 Norman Abramson이 개발했습니다.

**💡 비유**: ALOHA는 회의에서 말하고 싶을 때 바로 말하는 것입니다.

**📢 섹션 요약 비유**: Pure ALOHA 최대 효율 = 18.4%!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **G** | 제공 부하 | 프레임/시간 | 요청 |
| **S** | 처리량 | 성공 전송 | 완료 |
| **2T** | 취약 구간 | 충돌 구간 | 위험 |

**비교표**

```
+------------+----------+----------+
| 항목       | Pure     | Slotted  |
+------------+----------+----------+
| 취약 구간  | 2T       | T        |
| 최대 효율  | 18.4%    | 36.8%    |
| 최적 G     | 0.5      | 1.0      |
| 동기       | 불필요   | 필요     |
+------------+----------+----------+
```

**핵심 알고리즸**: 전송 → ACK 대기 → 타임아웃 → 랜덤 대기 → 재전송

**📢 섹션 요약 비유**: 취약 구간 2T가 문제!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: ALOHA 계보**

| 기술 | 효율 | 특징 |
|:---|:---:|:---|

**📢 섹션 요약 비유**: ALOHA → Slotted → CSMA 진화!
