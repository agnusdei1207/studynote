# NAT Traversal - NAT 통과

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: NAT 환경에서 P2P 연결
> 2. **가치**: 홈 네트워크 P2P 가능
> 3. **융합**: WebRTC, VPN, 게임

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NAT Traversal은 NAT 뒤의 장비 간 연결을 가능하게 합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NAT Traversal                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NAT 문제                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client A]                                         │   │ │
│   │   │  192.168.1.10 (사설IP)                              │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ NAT (공인IP: 1.1.1.1)                          │   │ │
│   │   │     │                                                │   │ │
│   │   │     │        ┌─────────────┐                         │   │ │
│   │   │     │        │   인터넷    │                         │   │ │
│   │   │     │        └─────────────┘                         │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ NAT (공인IP: 2.2.2.2)                          │   │ │
│   │   │     │                                                │   │ │
│   │   │  [Client B]                                         │   │ │
│   │   │  192.168.2.20 (사설IP)                              │   │ │
│   │   │                                                      │   │ │
│   │   │  문제: 서로 사설IP로 직접 연결 불가                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              해결 기술                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 기술       │ 설명     │ 특징     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ STUN       │ 주소 획득 │ 가벼움   │               │   │ │
│   │   │  │ TURN      │ 릴레이   │ 확실     │               │   │ │
│   │   │  | ICE        │ 프레임워크│ 조합     │               │   │ │
│   │   │  │ Hole Punching│ 구멍   │ P2P     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: NAT Traversal은 공인 IP 없는 장비 간 연결을 가능하게 합니다.

**💡 비유**: NAT Traversal은 우편함이 없는 집에 배달하는 방법입니다.

**📢 섹션 요약 비유:** NAT Traversal = P2P 연결!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **STUN** | 주소 획득 |
| **TURN** | 릴레이 |
| **ICE** | 연결 프레임워크 |
| **Hole Punching** | 홀 펀칭 |

**비교표: NAT 유형별 대응**

```
+------------+----------+----------+
| NAT 유형   │ 방법     │ 성공률   │
+------------+----------+----------+
| Full Cone  │ 쉬움     │ 높음     │
| Restricted │ 중간     │ 중간     │
| Symmetric  │ 어려움   │ 낮음     │
+------------+----------+----------+
```

**핵심 알고리즸:** 홀 펀칭, 릴레이

**📢 섹션 요약 비유:** Symmetric NAT = 어려움!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: P2P 연결**

|:---|:---|:---|:---|
| **P2P** | 예 | 아니오 | 예 |
| **릴레이** | 아니오 | 예 | 아니오 |

**📢 섹션 요약 비유:** WebRTC = NAT Traversal 필수!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: WebRTC** → STUN + TURN
**시나리오 2: P2P 게임** → Hole Punching

**도입 체크리스트:** NAT 유형, STUN/TURN 서버, 지연

**📢 섹션 요약 비유:** 공인 IP 없음 = NAT Traversal!
