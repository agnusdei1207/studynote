# MIB - 관리 정보 베이스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SNMP 관리 객체 정의
> 2. **가치**: 표준화된 데이터 구조
> 3. **융합**: SNMP, OID와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MIB(Management Information Base)는 SNMP로 관리할 수 있는 객체들의 계층적 데이터베이스입니다.


> **해설**: MIB는 OID 트리 구조로 관리 객체를 정의합니다.

**💡 비유**: MIB는 도서관의 책 분류 체계입니다. 모든 정보가 체계적으로 정리됩니다.

**📢 섹션 요약 비유:** MIB = 관리 객체 DB!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Scalar** | 단일 값 |
| **Tabular** | 테이블 |
| **OID** | 객체 식별자 |
| **SMI** | 구조 정의 |

**비교표: 표준 vs 벤더 MIB**

```
+------------+----------+----------+
| 유형       │ OID      │ 설명     │
+------------+----------+----------+
| 표준 MIB   │ 1.3.6.1.2│ RFC      │
| 벤더 MIB   │ 1.3.6.1.4│ Cisco 등 │
+------------+----------+----------+
```

**핵심 알고리즸:** OID 트리 순회

**📢 섹션 요약 비유:** MIB-II = 표준!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 주요 MIB 그룹**

| 그룹 | OID | 설명 |
|:---|:---|:---|
| **system** | 1.3.6.1.2.1.1 | 시스템 정보 |

| **ip** | 1.3.6.1.2.1.4 | IP |
| **tcp** | 1.3.6.1.2.1.6 | TCP |

**📢 섹션 요약 비유:** sysDescr = 시스템 설명!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 장비 모니터링** → 표준 MIB
**시나리오 2: 벤더 특화** → 벤더 MIB
**시나리오 3: 커스텀** → Private MIB

**도입 체크리스트:** MIB 파일, OID, NMS 지원

**📢 섹션 요약 비유:** 벤더 MIB = 상세 정보!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 표준화, 상호운용

**미래 전망:** YANG 모델로 전환

**참고 표준:** RFC 1213 (MIB-II)

**📢 섹션 요약 비유:** MIB는 SNMP 핵심!
