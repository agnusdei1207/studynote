# PRMA (Packet Reservation Multiple Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 패킷 단위 예약 기반 다중 접속
> 2. **가치**: 음성/데이터 통합, 효율적 자원 활용
> 3. **융합**: 위성, 무선 네트워크와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

PRMA(Packet Reservation Multiple Access)는 음성과 데이터를 통합 전송하기 위해 패킷 단위로 슬롯을 예약하는 다중 접속 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PRMA (Packet Reservation Multiple Access)        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PRMA 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  특징:                                                 │   │ │
│   │   │  • Slotted ALOHA 기반                                 │   │ │
│   │   │  • 음성 패킷: 슬롯 예약                               │   │ │
│   │   │  • 데이터 패킷: 경쟁 기반                             │   │ │
│   │   │                                                      │   │ │
│   │   │  프레임 구조:                                          │   │ │
│   │   │  ┌─────┬─────┬─────┬─────┬─────┬─────┐              │   │ │
│   │   │  │슬롯1│슬롯2│슬롯3│슬롯4│슬롯5│슬롯6│              │   │ │
│   │   │  │예약 │경쟁 │예약 │경쟁 │예약 │경쟁 │              │   │ │
│   │   │  └─────┴─────┴─────┴─────┴─────┴─────┘              │   │ │
│   │   │    ↑           ↑           ↑                        │   │ │
│   │   │  음성A      음성B       음성C                        │   │ │
│   │   │                                                      │   │ │
│   │   │  알고리즘:                                              │   │ │
│   │   │  1. 음성 패킷: 빈 슬롯에 전송 → 성공 시 예약 유지       │   │ │

│   │   │                                                      │   │ │
│   │   │  장점:                                                   │   │ │
│   │   │  • 음성 QoS 보장                                       │   │ │
│   │   │  • 데이터 효율적 전송                                  │   │ │
│   │   │  • 동적 자원 할당                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  단점:                                                   │   │ │
│   │   │  • 음성 활동 감지 필요                                 │   │ │

│   │   │  • Slotted ALOHA: 모든 패킷 경쟁                      │   │ │
│   │   │  • PRMA: 음성은 예약, 데이터는 경쟁                    │   │ │
│   │   │  • 예약 방식: 모든 트래픽 예약                         │   │ │
```

> **해설**: PRMA는 음성과 데이터를 통합 전송하는 무선 네트워크에서 사용됩니다.

**💡 비유**: PRMA는 식당에서 단골 손님은 예약석, 일반 손님은 빈 자리 경쟁입니다.

**📢 섹션 요약 비유**: 음성 = 예약, 데이터 = 경쟁!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **슬롯** | 전송 단위 | 고정 길이 | 자리 |
| **예약** | 음성 우선 | 지속 유지 | VIP석 |
| **경쟁** | 데이터 | 충돌 허용 | 일반석 |

**비교표**

```
+------------+----------+----------+
| 항목       | Slotted  | PRMA     |
+------------+----------+----------+
| 음성       | 경쟁     | 예약     |
| 데이터     | 경쟁     | 경쟁     |
| QoS        | 없음     | 부분     |
| 효율       | 낮음     | 높음     |
+------------+----------+----------+
```

**핵심 알고리즸**: 음성 활동 감지 → 빈 슬롯 전송 → 성공 시 예약 유지

**📢 섹션 요약 비유**: PRMA = 음성 우선!
