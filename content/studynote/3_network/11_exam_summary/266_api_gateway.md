# API Gateway - API 게이트웨이

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: API 진입점 관리
> 2. **가치**: 인증, 라우팅, 보안
> 3. **융합**: 마이크로서비스, K8s와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

API Gateway는 마이크로서비스 아키텍처에서 모든 API 요청의 진입점 역할을 합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    API Gateway                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              API Gateway 구조                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           API Gateway                        │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌─────────────────────────────────────┐    │    │   │ │
│   │   │  │  │  기능:                               │    │    │   │ │
│   │   │  │  │  • 인증/인가 (Auth)                  │    │    │   │ │
│   │   │  │  │  • 라우팅 (Routing)                 │    │    │   │ │
│   │   │  │  │  • 속도 제한 (Rate Limiting)        │    │    │   │ │
│   │   │  │  │  • 로드밸런싱                       │    │    │   │ │
│   │   │  │  │  • 로깅/모니터링                    │    │    │   │ │
│   │   │  │  │  • 캐싱                             │    │    │   │ │
│   │   │  │  │  • 프로토콜 변환                    │    │    │   │ │
│   │   │  │  └─────────────────────────────────────┘    │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │          │          │                        │   │ │
│   │   │       ↓          ↓          ↓                        │   │ │
│   │   │  [Service A] [Service B] [Service C]                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              API Gateway 패턴                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Backend for Frontend (BFF):                     │   │ │
│   │   │     • 클라이언트별 게이트웨이                        │   │ │
│   │   │                                                      │   │ │
│   │   │  2. API Aggregation:                                 │   │ │
│   │   │     • 여러 API 통합                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Protocol Translation:                           │   │ │
│   │   │     • REST → gRPC, GraphQL                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: API Gateway는 단일 진입점으로 모든 API 요청을 관리합니다.

**💡 비유**: API Gateway는 호텔 프론트 데스크입니다. 모든 요청을 처리합니다.

**📢 섹션 요약 비유:** API Gateway = 단일 진입점!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **라우팅** | 경로 매핑 |
| **인증** | JWT, OAuth |
| **속도 제한** | 트래픽 제어 |
| **캐싱** | 응답 저장 |

**비교표: API Gateway 솔루션**

```
+------------+----------+----------+
| 솔루션     │ 특징     │ 용도     │
+------------+----------+----------+
| Kong       │ Lua      │ 범용     │
| AWS API GW │ 관리형   │ 클라우드 │
| Apigee     │ 엔터프라이즈│ 기업   │
| Envoy      │ 프록시   │ Mesh    │
+------------+----------+----------+
```

**핵심 알고리즸:** 라우팅, 속도 제한

**📢 섹션 요약 비유:** BFF = 클라이언트별!
