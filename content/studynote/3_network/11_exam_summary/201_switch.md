# 스위치 (Switch) - L2/L3/포트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: MAC/IP 기반 프레임 전달
> 2. **가치**: 충돌 도메인 분리, 대역폭 독점
> 3. **융합**: VLAN, STP, 포트 미러링과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

스위치(Switch)는 MAC 주소 기반으로 프레임을 전달하는 2계층 장비입니다. L3 스위치는 IP 라우팅도 지원합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    스위치 (Switch)                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스위치 종류                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. L2 스위치 (Layer 2 Switch):                         │   │ │
│   │   │     • MAC 주소 기반 포워딩                              │   │ │
│   │   │     • VLAN 지원                                        │   │ │
│   │   │     • STP (Spanning Tree)                              │   │ │
│   │   │     • 동일 서브넷 내 통신                                │   │ │
│   │   │                                                      │   │ │
│   │   │  2. L3 스위치 (Layer 3 Switch / Multilayer):            │   │ │
│   │   │     • IP 라우팅 지원                                    │   │ │
│   │   │     • VLAN 간 라우팅 (Inter-VLAN)                       │   │ │
│   │   │     • 정적/동적 라우팅                                   │   │ │
│   │   │     • 코어/배포 계층                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 스위치 유형:                                        │   │ │
│   │   │     • 관리형 (Managed): SNMP, CLI, 웹                   │   │ │
│   │   │     • 비관리형 (Unmanaged): 플러그 앤 플레이             │   │ │
│   │   │     • 스마트 (Smart): 웹 기반 설정                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스위치 동작 원리                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. MAC 학습 (Learning):                              │   │ │
│   │   │     • 수신 프레임의 Source MAC + Port → MAC 테이블     │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 포워딩/필터링 (Forwarding/Filtering):               │   │ │
│   │   │     • Destination MAC 조회                            │   │ │
│   │   │     • Known: 해당 포트로 전송                          │   │ │
│   │   │     • Unknown: 플러딩 (Flooding)                       │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 에이징 (Aging):                                    │   │ │
│   │   │     • MAC 테이블 엔트리 만료 (기본 300초)               │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 브로드캐스트/멀티캐스트:                            │   │ │
│   │   │     • 브로드캐스트: 모든 포트로 플러딩                   │   │ │
│   │   │     • 멀티캐스트: IGMP 스누핑 또는 플러딩                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스위치는 MAC 주소 테이블을 사용하여 프레임을 지능적으로 전달합니다.

**💡 비유**: 스위치는 우체국입니다. 주소를 보고 정확한 목적지로 배달합니다.

**📢 섹션 요약 비유:** 스위치 = MAC 기반 포워딩!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 포트 속도**

| 속도 | 표준 | 케이블 | 거리 |
|:---:|:---|:---|:---:|
| **100M** | 100BASE-TX | Cat5 | 100m |
| **1G** | 1000BASE-T | Cat5e/6 | 100m |
| **10G** | 10GBASE-T | Cat6a/7 | 100m |
| **25G** | 25GBASE-T | Cat8 | 30m |

**📢 섹션 요약 비유:** 1G = 기업 표준!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 액세스 계층** → L2 스위치, PoE
**시나리오 2: 코어 계층** → L3 스위치, 고속
**시나리오 3: 데이터센터** → ToR 스위치, 10/25G

**도입 체크리스트:** 포트 수, 속도, PoE, 관리

**📢 섹션 요약 비유:** 액세스는 L2, 코어는 L3!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 충돌 도메인 분리, 대역폭 독점

**미래 전망:** 100G/400G 액세스, 자동화

**참고 표준:** IEEE 802.1D (STP), IEEE 802.1Q (VLAN)

**📢 섹션 요약 비유:** 스위치는 LAN의 핵심!
