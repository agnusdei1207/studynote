# 토큰 패싱 (Token Passing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 토큰을 가진 노드만 전송
> 2. **가치**: 충돌 없음, 결정적 접근
> 3. **융합**: 토큰 링, FDDI, 산업용 네트워크와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

토큰 패싱(Token Passing)은 특수한 프레임(토큰)을 순환시키며, 토큰을 가진 노드만 전송할 수 있는 제어 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    토큰 패싱 (Token Passing)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              토큰 링 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │            ┌─────┐                                  │   │ │
│   │   │           ╱       ╲                                 │   │ │
│   │   │      ┌───┐         ┌───┐                            │   │ │
│   │   │      │노드│         │노드│                            │   │ │
│   │   │      │ 1 │         │ 2 │                            │   │ │
│   │   │      └───┘         └───┘                            │   │ │
│   │   │        │    ◄── 토큰 ──►  │                          │   │ │
│   │   │      ┌───┐         ┌───┐                            │   │ │
│   │   │      │노드│         │노드│                            │   │ │
│   │   │      │ 4 │         │ 3 │                            │   │ │
│   │   │      └───┘         └───┘                            │   │ │
│   │   │           ╲       ╱                                 │   │ │
│   │   │            └─────┘                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  토큰 순환: 1 → 2 → 3 → 4 → 1 ...                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              토큰 패싱 동작                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  알고리즘:                                              │   │ │
│   │   │  1. 토큰 수신                                            │   │ │
│   │   │  2. 전송 데이터 있으면:                                   │   │ │
│   │   │     - 토큰 → 데이터 프레임 변경                           │   │ │
│   │   │     - 데이터 전송                                         │   │ │
│   │   │     - 수신자가 ACK 설정                                   │   │ │
│   │   │     - 프레임이 돌아오면 토큰으로 변경                     │   │ │
│   │   │  3. 전송 데이터 없으면:                                   │   │ │
│   │   │     - 토큰을 다음 노드로 전달                             │   │ │
│   │   │  4. 반복                                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                   │   │ │
│   │   │  • 충돌 없음                                             │   │ │
│   │   │  • 결정적 접근 (최대 대기 시간 보장)                       │   │ │
│   │   │  • 공정한 기회                                           │   │ │
│   │   │  • 토큰 손실 시 복구 필요                                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 토큰 패싱은 IBM 토큰 링(802.5)과 FDDI에서 사용됩니다.

**💡 비유**: 토큰 패싱은 마이크를 돌려가며 발언하는 것입니다.

**📢 섹션 요약 비유**: 토큰 = 전송 권한!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **토큰** | 전송 권한 | 3바이트 | 마이크 |
| **링** | 토폴로지 | 논리/물리 | 원형 |
| **모니터** | 토큰 관리 | 복구 | 관리자 |

**비교표**

```
+------------+----------+----------+
| 항목       | CSMA/CD  | Token    |
+------------+----------+----------+
| 충돌       | 있음     | 없음     |
| 결정성     | 없음     | 있음     |
| 고부하     | 비효율   | 효율     |
| 저부하     | 효율     | 비효율   |
| 복잡도     | 낮음     | 높음     |
+------------+----------+----------+
```

**핵심 알고리즸**: 토큰 수신 → 데이터 전송/토큰 전달 → 순환

**📢 섹션 요약 비유**: 토큰 링 최대 대기 = (N-1) × T!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 토큰 기반 시스템**

| 시스템 | 속도 | 특징 |
|:---|:---:|:---|
| **토큰 링** | 16Mbps | IBM |

**📢 섹션 요약 비유**: FDDI는 이중 링으로 복구!
