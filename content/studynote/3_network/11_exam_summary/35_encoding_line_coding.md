# 부호화 (Encoding - Line Coding, Block Coding)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디지털 데이터를 전송 가능한 신호로 변환
> 2. **가치**: 동기화, 대역폭 효율, 오류 검출
> 3. **융합**: NRZ, 맨체스터, 4B/5B, 8B/10B와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

부호화(Encoding)는 디지털 데이터를 전송 매체에 적합한 신호 형태로 변환하는 과정입니다. Line Coding과 Block Coding이 있습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    부호화 (Encoding)                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Line Coding 종류                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 단극성 (Unipolar):                                │   │ │
│   │   │     • 0: 무전압, 1: +V                                │   │ │
│   │   │     • 간단하지만 DC 성분 존재                          │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 극성 (Polar):                                     │   │ │
│   │   │     • 0: -V, 1: +V                                   │   │ │
│   │   │     • NRZ-L, NRZ-I                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 양극성 (Bipolar):                                 │   │ │
│   │   │     • 0: 무전압, 1: +V/-V 교대                        │   │ │
│   │   │     • AMI, Pseudoternary                             │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 맨체스터 (Manchester):                            │   │ │
│   │   │     • 0: High→Low, 1: Low→High                       │   │ │
│   │   │     • 동기화 내장                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  5. 차분 맨체스터 (Differential Manchester):           │   │ │
│   │   │     • 비트 시작에서 전이 유무로 구분                   │   │ │
│   │   │     • 토큰링에서 사용                                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Block Coding                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  4B/5B: 4비트 → 5비트 (FDDI, 100BASE-TX)              │   │ │
│   │   │  8B/10B: 8비트 → 10비트 (기가비트 이더넷)               │   │ │
│   │   │  64B/66B: 64비트 → 66비트 (10G 이더넷)                 │   │ │
│   │   │                                                      │   │ │
│   │   │  목적:                                               │   │ │
│   │   │  • 동기화 개선 (연속 0 방지)                           │   │ │
│   │   │  • 오류 검출 기능                                     │   │ │
│   │   │  • DC 밸런스 유지                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: Line Coding은 물리 계층의 핵심 기술입니다.

**💡 비유**: 부호화는 모스 부호처럼 데이터를 전송 가능한 신호로 바꾸는 것입니다.

**📢 섹션 요약 비유**: 맨체스터는 동기화 내장!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **NRZ** | 비복귀 | 레벨 유지 | 직선 |
| **RZ** | 복귀 | 0 복귀 | 펄스 |
| **Manchester** | 전이 | 중앙 전이 | 천이 |
| **Block** | 블록 | m비트→n비트 | 확장 |

**Line Coding 파형 비교**

```
데이터:  1   0   1   1   0   0   1   0

NRZ-L:  ─┐   ┌───┐       ┌───┐
          └───┘   └───────┘   └───

NRZ-I:  ─┐       ┌───┐       ┌───
   (1에서 전이)───┘   └───────┘

Manchester:
         ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
       ──┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘
         (중앙 전이로 동기화)

AMI:    ─┐       ┌───┐       ┌───
          └───┘   └───────┘   └───
         (+V,-V 교대)
```

**핵심 알고리즘**: 데이터 → 인코딩 → 전송 → 디코딩

**📢 섹션 요약 비유**: NRZ는 연속 0에서 동기화 문제!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: Line Coding**

| 방식 | 대역폭 | 동기화 | DC |
|:---|:---:|:---:|:---:|
| **NRZ-L** | 낮음 | 어려움 | 있음 |
| **AMI** | 낮음 | 보통 | 없음 |
| **Manchester** | 높음 | 좋음 | 없음 |
| **4B/5B+NRZ** | 중간 | 좋음 | 없음 |

**📢 섹션 요약 비유**: 100Mbps는 4B/5B + NRZ-I!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 10BASE-T** → 맨체스터
**시나리오 2: 100BASE-TX** → 4B/5B + MLT-3
**시나리오 3: 1000BASE-T** → 8B/10B + PAM-5

**도입 체크리스트**: 속도, 거리, 동기화

**📢 섹션 요약 비유**: 속도 높을수록 복잡한 부호화!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대역폭 효율, 동기화

**미래 전망**: PAM-4, PAM-8 (400G 이더넷)

**참고 표준**: IEEE 802.3

**📢 섹션 요약 비유**: 부호화는 디지털 통신의 기초!
