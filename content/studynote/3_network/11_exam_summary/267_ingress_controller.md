# Ingress Controller - 인그레스 컨트롤러

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: K8s 클러스터 진입점
> 2. **가치**: L7 라우팅, TLS 종료
> 3. **융합**: K8s, Service와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Ingress Controller는 Kubernetes 클러스터 외부 트래픽을 내부 서비스로 라우팅합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Ingress Controller                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Ingress Controller 구조                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [외부 트래픽]                                       │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           Ingress Controller                 │    │   │ │
│   │   │  │  (Nginx, Traefik, Kong, Istio Gateway)       │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌─────────────────────────────────────┐    │    │   │ │
│   │   │  │  │  Ingress 리소스:                     │    │    │   │ │
│   │   │  │  │  • Host: api.example.com            │    │    │   │ │
│   │   │  │  │  • Path: /api/*                     │    │    │   │ │
│   │   │  │  │  • Backend: api-service:8080        │    │    │   │ │
│   │   │  │  └─────────────────────────────────────┘    │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │          │          │                        │   │ │
│   │   │       ↓          ↓          ↓                        │   │ │
│   │   │  [Service]   [Service]   [Service]                   │   │ │
│   │   │     │           │           │                        │   │ │
│   │   │     ↓           ↓           ↓                        │   │ │
│   │   │  [Pod]        [Pod]       [Pod]                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Ingress 기능                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. L7 라우팅:                                        │   │ │
│   │   │     • Host 기반, Path 기반                           │   │ │
│   │   │                                                      │   │ │
│   │   │  2. TLS 종료:                                         │   │ │
│   │   │     • 인증서 관리                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 로드밸런싱:                                       │   │ │
│   │   │     • Service로 분산                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 이름 기반 가상 호스팅:                            │   │ │
│   │   │     • 여러 도메인 처리                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Ingress Controller는 L7 계층에서 트래픽을 라우팅합니다.

**💡 비유**: Ingress Controller는 건물 출입구 안내데스크입니다.

**📢 섹션 요약 비유:** Ingress = K8s 진입점!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Ingress** | K8s 리소스 |
| **IngressClass** | 컨트롤러 선택 |
| **TLS Secret** | 인증서 |
| **Backend** | 서비스 |

**비교표: Ingress Controller 종류**

```
+------------+----------+----------+
| 컨트롤러   │ 특징     │ 용도     │
+------------+----------+----------+
| Nginx      │ 표준     │ 범용     │
| Traefik    │ 자동     │ 동적     │
| Kong       │ API GW   │ 기업     │
| Istio      │ Mesh     │ 서비스   │
+------------+----------+----------+
```

**핵심 알고리즸:** 라우팅, TLS

**📢 섹션 요약 비유:** IngressClass = 선택!
