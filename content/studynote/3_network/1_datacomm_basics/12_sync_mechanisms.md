# 동기식 전송: 문자 및 비트 동기 방식 (SYN, BSC, SDLC, HDLC)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동기식 전송은 송수신 측이 공통의 클럭을 공유하여 대량의 데이터를 블록(프레임) 단위로 쏟아내는 방식이며, 데이터의 경계를 나누는 기준에 따라 **문자 동기**와 **비트 동기**로 구분된다.
> 2. **가치**: 시작/정지 비트가 없는 연속 전송을 통해 오버헤드를 최소화하여 전송 효율을 극대화하며, 고도의 에러 제어(CRC)와 흐름 제어 기능을 프레임 구조 내에 포함한다.
> 3. **융합**: 초기의 문자 중심 프로토콜(BSC)에서 현대의 강력한 비트 중심 프로토콜(HDLC)로 진화했으며, 이는 인터넷의 근간인 데이터 링크 계층 표준들의 아키텍처적 모태가 되었다.

+++

## Ⅰ. 동기식 전송의 두 가지 핵심 방식

### 1. 문자 동기 방식 (Character-Oriented / Byte-Oriented)
- **정의**: 특정 제어 문자(예: `SYN`, `STX`, `ETX`)를 사용하여 프레임의 시작과 끝을 식별하는 방식이다.
- **대표 프로토콜**: IBM의 **BSC (Binary Synchronous Communication)**.
- **특징**: 전송되는 데이터가 반드시 특정 문자 집합(ASCII 등) 단위여야 하므로 유연성이 떨어진다.

### 2. 비트 동기 방식 (Bit-Oriented)
- **정의**: 문자가 아닌 **특수한 비트 패턴(Flag, 01111110)**을 사용하여 프레임의 경계를 나누는 방식이다.
- **대표 프로토콜**: **SDLC, HDLC, LAPB**.
- **특징**: 데이터의 내용에 관계없이 임의의 비트 열을 전송할 수 있는 **투명성(Transparency)**을 보장한다.

### 3. 💡 비유: 기차의 칸막이
- **문자 동기**: 기차 칸마다 "여기가 칸의 시작입니다"라는 **문구가 적힌 팻말**을 붙여놓은 것과 같다. (팻말 자체도 글자로 구성됨)
- **비트 동기**: 기차 칸 사이를 **특수한 모양의 연결고리(Flag)**로 묶어놓은 것과 같다. 글자가 무엇이든 상관없이 연결고리만 보면 칸이 나뉘는 것을 알 수 있다.

## Ⅱ. 비트 동기 방식과 비트 스터핑 (ASCII)

비트 동기 방식의 핵심은 데이터 내에 플래그와 똑같은 패턴이 나타날 때 이를 처리하는 **비트 스터핑(Bit Stuffing)**이다.

### 1. 비트 중심 프레임 구조 (HDLC 예시)
```text
    ┌────────┬────────┬────────┬──────────────────┬────────┬────────┐
    │  Flag  │ Address│ Control│     Information  │  FCS   │  Flag  │
    │01111110│ 8 bits │ 8 bits │ (Variable Data)  │ 16 bits│01111110│
    └────────┴────────┴────────┴──────────────────┴────────┴────────┘
```

### 2. 비트 스터핑 (Bit Stuffing) 메커니즘
- **문제**: 데이터 중간에 `01111110`이 우연히 나타나면 수신기가 프레임의 끝으로 오해한다.
- **해결**: 송신기는 데이터에 '1'이 연속해서 5개 나오면 무조건 뒤에 **'0'을 강제로 삽입**한다. 수신기는 '1'이 5개 뒤에 오는 '0'을 무조건 제거하여 원본을 복원한다.

```text
    [ 송신 측 과정 ]
    Original: 0 1 1 1 1 1 1 0
    Stuffed : 0 1 1 1 1 1 (0) 1 0  <-- '0' 삽입
    
    [ 수신 측 과정 ]
    Received: 0 1 1 1 1 1 (0) 1 0
    Original: 0 1 1 1 1 1 1 0      <-- '0' 제거 및 복원
```

## Ⅲ. 문자 동기(BSC) vs 비트 동기(HDLC) 비교

| 비교 항목 | 문자 동기 (Character) | 비트 동기 (Bit) |
|:---|:---|:---|
| **동기 식별** | `SYN` 문자 (00010110) | **Flag 비트 (01111110)** |
| **투명성 보장** | `DLE` 문자 삽입 (복잡) | **비트 스터핑 (단순/강력)** |
| **전송 단위** | 8비트(바이트) 고정 | **임의의 비트 길이 가능** |
| **에러 제어** | 블록 체크 (VRC/LRC) | **강력한 16/32비트 CRC** |
| **전송 효율** | 보통 | **매우 높음** |

- **📢 섹션 요약 비유**: 정해진 규격의 상자만 실을 수 있는 컨테이너선(문자 동기)과, 어떤 모양의 짐이든 그물로 묶어 실을 수 있는 다목적 화물선(비트 동기)의 차이입니다.

+++

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy)

### 1. HDLC(High-level Data Link Control)의 위상
- **기술사적 인사이트**: HDLC는 단순히 하나의 프로토콜이 아니라, 현대 네트워크의 **데이터 링크 계층 아키텍처를 정립**한 표준이다. 
- **인사이트**: 우리가 현재 쓰는 **PPP(Point-to-Point Protocol)**나 이더넷 프레임 구조의 에러 체크 방식 등은 모두 HDLC의 비트 동기 메커니즘을 계승한 것이다.

### 2. 고속 전송에서의 오버헤드 최적화
- **판단**: 전송 속도가 수 Gbps로 올라갈수록 비트 단위로 '0'을 일일이 검사하여 삽입/제거하는 소프트웨어적 방식은 성능 병목을 일으킨다. 
- **결단**: 실무 고속 통신 장비 설계 시에는 이를 전용 하드웨어(FPGA/ASIC) 로직으로 구현하여 라인 속도(Line Rate)를 보장해야 한다.

+++

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **데이터 투명성**: 어떤 종류의 바이너리 데이터도 에러 없이 완벽 전송 가능.
- **채널 효율성**: 최소한의 제어 정보만으로 대용량 데이터를 고속 처리.

### 2. 미래 전망
최근의 초고속 광통신이나 위성 통신에서도 프레임의 경계를 나누고 동기를 맞추는 비트 동기 원리는 여전히 유효하다. 다만, 비트 스터핑 대신 고정된 크기의 슬롯을 사용하는 **GFP (Generic Framing Procedure)** 등으로 진화하고 있으며, 이러한 '동기화의 예술'은 통신의 속도가 빨라질수록 더욱 정밀한 제어 기술로 발전할 것이다.

- **📢 섹션 요약 비유**: 서로의 호흡을 맞추는 지휘자와 연주자처럼, 정교한 동기 방식은 무질서한 비트 열에 '의미 있는 메시지'라는 생명력을 불어넣습니다.
