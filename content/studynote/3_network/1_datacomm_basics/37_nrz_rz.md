# NRZ (Non-Return to Zero) 코딩 방식 분석

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비트 주기 동안 전압이 0V로 돌아오지 않고 유지되는 가장 단순한 라인 코딩 방식으로, 레벨 자체로 비트를 구분(NRZ-L)하거나 전압의 변화 유무로 구분(NRZ-I)한다.
> 2. **가치**: 전압 변화 횟수가 적어 대역폭 효율이 매우 높고 구현이 저렴하지만, 동일한 비트(특히 0)가 연속될 경우 동기화가 깨지는 물리적 한계를 가진다.
> 3. **융합**: USB나 과거 디스크 드라이브 인터페이스에서 동기화 문제를 해결한 **NRZ-I** 방식이 널리 쓰였으며, 현대 고속 통신에서는 블록 코딩(4B/5B 등)과 결합하여 단점을 보완한다.

---

### Ⅰ. NRZ 방식의 종류 및 매커니즘

| 구분 | NRZ-L (Level) | NRZ-I (Inversion) |
|:---|:---|:---|
| **비트 결정** | 전압의 **절대적 레벨**에 따라 결정 | 전압의 **변화(Transition)** 유무에 따라 결정 |
| **규칙 (보통)** | 0: High (+V), 1: Low (-V) | **1: 반전 발생**, 0: 상태 유지 |
| **장점** | 구현이 직관적이고 단순함 | 연속된 '1'이 올 때 전압이 계속 바뀌어 동기화 유리 |
| **단점** | 연속된 '0' 또는 '1'에서 동기화 상실 | **연속된 '0'**에서는 여전히 동기화 문제 발생 |

---

### Ⅱ. NRZ-L vs NRZ-I 파형 비교 (ASCII)

데이터 '0 1 0 0 1 1'을 보낼 때의 차이다.

```ascii
    Data  :    0      1      0      0      1      1
    
    [NRZ-L]: +---+  - - -  +---+  +---+  - - -  - - -
             |   |  |   |  |   |  |   |  |   |  |   |
    ---------+---+--+---+--+---+--+---+--+---+--+---+--- 0V
    
    [NRZ-I]: +---+  - - -  - - -  - - -  +---+  - - -
             |   |  |   |  |   |  |   |  |   |  |   |
    ---------+---+--+---+--+---+--+---+--+---+--+---+--- 0V
             (Start) (Inv) (Stay) (Stay) (Inv)  (Inv)
```

---

### Ⅲ. NRZ 방식의 공학적 한계 및 해결책

1. **동기화 문제 (Baseline Wander)**: 
   - 긴 '0'이나 '1'이 지속되면 평균 전압이 한쪽으로 쏠리고, 수신측 클럭이 비트의 경계를 놓침.
2. **해결책 (Block Coding 연계)**: 
   - 전송 전 데이터를 **4B/5B**나 **8B/10B**로 변환하여 연속된 동일 비트의 길이를 강제로 제한함.
   - 변환된 데이터를 NRZ-I로 보내면 최소한의 변화를 보장할 수 있음.

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 고속 인터페이스(USB 2.0) 설계
- **현상**: USB 케이블을 통해 데이터를 보낼 때 수신측에서 비트 타이밍을 자꾸 놓침.
- **기술사적 결단**: 
  - NRZ-L 대신 **NRZ-I**를 기본 코딩으로 채택한다. 
  - 여기에 연속된 0이 오면 강제로 1을 끼워넣는 **Bit Stuffing** 기술을 병행하여, 어떤 데이터가 오더라도 전압 변화가 일정 주기 내에 반드시 일어나도록 강제한다.

#### 2. 기술사적 인사이트: 대역폭 가성비
- 맨체스터 코딩은 비트당 2번의 전압 변화가 있어 대역폭 효율이 50%인 반면, NRZ는 최악의 경우에도 비트당 1번만 변하므로 **대역폭 효율이 100%**에 가깝다. 비용이 중요한 하드웨어 설계에서는 NRZ + 가공(Stuffing) 조합이 경제적임을 강조해야 한다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **전송 용량 극대화**: 낮은 주파수 대역에서도 높은 비트레이트 달성.
- **장치 소형화**: 단순한 회로 설계를 통한 저전력/저비용 컨트롤러 제작.

#### 2. 미래 전망
6G나 초고속 서버 인터페이스에서는 NRZ의 2진 레벨을 넘어선 **PAM-4 (4레벨 NRZ 확장형)** 기술이 표준으로 자리 잡고 있다. NRZ의 단순함과 대역폭 효율을 계승하면서도 한 번에 2비트씩 보내는 이 방식은, 현대 물리 계층 기술의 정점이자 NRZ 철학의 화려한 부활이라 할 수 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. **NRZ-L**은 친구에게 "1은 손들기, 0은 손 내리기"라고 약속하고 계속 그 자세를 유지하는 것과 같아요.
2. **NRZ-I**는 "1이 나올 때마다 자세를 반대로 바꿔!"라고 약속하는 거죠. (손 들고 있었으면 내리고, 내리고 있었으면 들고)
3. 1이 계속 나오면 춤추듯 자세를 계속 바꾸니까 친구가 내가 살아있는지(동기화) 금방 알 수 있어서 훨씬 좋답니다!
