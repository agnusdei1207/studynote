# RZ (Return to Zero) 코딩 방식 분석

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 각 비트 주기(Bit Interval)의 절반 지점에서 전압이 항상 **0V(Zero)로 되돌아오는(Return)** 라인 코딩 방식으로, 신호 변화를 통해 박자(클럭) 정보를 전달한다.
> 2. **가치**: 신호 자체에 클럭 정보가 포함된 **자기 동기화(Self-clocking)** 능력을 갖추어, 연속된 비트가 와도 수신측이 비트의 경계를 정확히 판별할 수 있게 한다.
> 3. **융합**: NRZ의 동기화 문제를 물리적으로 해결한 방식이며, 대역폭 효율은 낮지만 동기화가 중요한 초기 디지털 통신 시스템의 기술적 기반이 되었다.

---

### Ⅰ. RZ 방식의 정의 및 매커니즘

- **정의**: '1' 또는 '0'을 표현하는 펄스가 비트 주기의 절반($T/2$)만 유지되고, 나머지 절반은 무조건 0V로 복귀하는 방식.
- **규칙**: 
  - 1: 전반기에 +V 유지, 후반기에 0V로 복귀.
  - 0: 전반기에 -V 유지, 후반기에 0V로 복귀 (또는 0V 유지, 방식에 따라 다름).
- **특징**: 매 비트마다 전압 변화가 반드시 발생하므로 수신측 클럭 동기화가 매우 용이함.

---

### Ⅱ. NRZ vs RZ 파형 비교 (ASCII)

데이터 '1 0 1'을 보낼 때의 차이다.

```ascii
    Data:      1            0            1
    
    [NRZ]: +-------+    +-------+    +-------+
           |       |    |       |    |       |
    -------+       +----+       +----+       +--- 0V
    
    [RZ] : +---+        +---+        +---+
           |   |        |   |        |   |
    -------+   +---+----+   +---+----+   +--- 0V
               |   |        |   |        |   |
               +---+        +---+        +---+
```

---

### Ⅲ. RZ 방식의 상세 장단점 분석

| 구분 | 장점 (Pros) | 단점 (Cons) |
|:---|:---|:---|
| **동기화** | **자기 동기화 가능**. 별도의 클럭선 불필요. | - |
| **대역폭** | - | **대역폭 2배 소모**. 신호 변화가 잦아 전송 효율 낮음. |
| **복잡도** | 구현이 비교적 단순함. | 3개의 전압 레벨(+V, 0, -V) 제어 필요. |
| **신뢰성** | 비트 경계 판별이 확실하여 에러 적음. | 고속 전송 시 파형 뭉개짐에 취약. |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 동기화 우선 순위 시스템 설계
- **현상**: 초정밀 타이밍이 필요한 저속 계측 장비 간 통신.
- **기술사적 결단**: 
  - 대역폭 자원이 풍부하고 동기화 안정성이 최우선인 환경이라면 **RZ 코딩**을 적용한다. 
  - 하지만 최근의 고속 네트워크 환경에서는 RZ의 낮은 효율성을 극복하기 위해 **Manchester**나 **8B/10B**와 같은 더 세련된 동기화 기법으로 대체할 것을 권고한다.

#### 2. 기술사적 인사이트: 대역폭 효율의 대가
- RZ는 비트 하나를 표현하기 위해 두 번의 신호 상태 변화를 요구한다. 이는 나이퀴스트 이론에 따라 동일 대역폭에서 NRZ보다 절반의 속도밖에 낼 수 없음을 의미하며, 인프라 비용과 품질 사이의 철저한 기회비용 계산이 필요함을 기술해야 한다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **클럭 드리프트(Drift) 방지**: 장시간 통신 시에도 송수신 타이밍 일치 보장.
- **시스템 간편화**: 데이터 선 하나로 정보와 박자를 동시 전달하여 배선 간소화.

#### 2. 미래 전망
순수 RZ 방식은 현대 광통신이나 이더넷에서는 비효율성 때문에 잘 쓰이지 않는다. 하지만 박자 정보를 신호에 심는 RZ의 **'복귀(Return)' 철학**은 양자 통신이나 초고속 펄스 통신에서 신호의 간섭을 최소화하기 위한 **Narrow Pulse** 기술로 계승되어 연구되고 있다.
