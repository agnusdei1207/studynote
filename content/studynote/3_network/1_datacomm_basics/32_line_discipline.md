# 회선 제어 규약 (Line Discipline)의 매커니즘 분석

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 두 통신 장치 사이에서 물리적 회선을 누가, 언제 사용할 것인지를 결정하는 **전송 권한 제어** 규칙.
> 2. **가치**: 데이터 전송 전후의 연결 확립(Connection Setup) 및 해제 과정을 규정하여, 충돌 없이 안정적인 정보 교환을 가능케 한다.
> 3. **융합**: 점대점(Point-to-Point) 환경의 **ENQ/ACK** 방식과 다중점(Multi-point) 환경의 **Polling/Selection** 방식이 상황에 맞게 적용되어 전체 네트워크 효율을 관리한다.

---

### Ⅰ. 회선 제어 규약의 주요 방식

| 방식 분류 | 적용 환경 | 핵심 매커니즘 |
|:---|:---|:---|
| **ENQ/ACK** | **점대점 (1:1)** | 송신측이 "보내도 돼?"(ENQ) 묻고 수신측이 "응"(ACK) 하면 전송 시작. |
| **Polling** | **다중점 (Primary:Secondary)** | 주스테이션(Host)이 단말기들에게 "보낼 거 있어?"라고 순차적으로 물음. |
| **Selection** | **다중점 (Primary:Secondary)** | 주스테이션이 특정 단말기에게 "너한테 줄 거 있으니까 준비해!"라고 알림. |

---

### Ⅱ. 회선 제어 흐름도 (ASCII)

#### 1. ENQ/ACK 방식 (1:1 관계)
```ascii
    [ Station A ]                      [ Station B ]
          | -------- ENQ (보낼게!) --------> |
          | <------- ACK (알았어!) -------- |
          | -------- Text (데이터) --------> |
          | <------- ACK (잘 받았어) ------- |
          | -------- EOT (끝낼게) --------> |
```

#### 2. Polling/Selection 방식 (1:N 주종 관계)
- **Polling**: Host가 데이터 수집.
- **Selection**: Host가 데이터 분배.

---

### Ⅲ. 회선 제어의 5단계 과정

1. **회선 접속 (Circuit Connection)**: 물리적 통로 개설 (다이얼업 등).
2. **링크 확립 (Data Link Establishment)**: 송수신측 간의 논리적 동기화 (ENQ/ACK).
3. **정보 전송 (Information Transfer)**: 실제 데이터 패킷 교환.
4. **링크 종결 (Data Link Termination)**: 논리적 연결 해제 (EOT).
5. **회선 절단 (Circuit Termination)**: 물리적 통로 폐쇄.

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 네트워크 토폴로지에 따른 제어 선정
- **현상**: 버스(Bus)형 구조에서 여러 단말이 동시에 데이터를 보내려 함.
- **기술사적 결단**: 
  - 주스테이션의 통제가 가능한 환경이면 오버헤드가 적은 **Polling** 방식을 권고한다. 
  - 하지만 단말이 자율적인 지능형 장비라면 Polling 지연을 없애기 위해 **CSMA/CD**와 같은 경쟁 기반 제어 방식으로 전환을 검토해야 한다.

#### 2. 기술사적 인사이트: 타임아웃 (Time-out) 설정
- ENQ를 보냈는데 ACK가 오지 않을 경우, 무한정 기다리면 시스템이 멈춘다. 
- 따라서 적절한 **Time-out** 및 **재시도 횟수(Retry Limit)**를 설정하여 회선 점유를 강제로 해제하는 예외 처리 설계가 필수적임을 명시해야 한다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **충돌 방지**: 질서 있는 회선 사용을 통한 데이터 파손 예방.
- **자원 공정 분배**: Polling 주기 조절을 통한 단말기 간 공정성 확보.

#### 2. 미래 전망
현대 인터넷(Ethernet/IP)은 장비들이 대등한 관계에서 데이터를 주고받는 비연결형(Connectionless) 방식이 주류지만, 산업용 IoT나 위성 통신과 같이 확정적 전송이 필요한 분야에서는 여전히 **중앙 집중형 회선 제어** 기술이 핵심이다. 앞으로 6G에서는 AI가 트래픽을 예측하여 Polling 순서를 실시간으로 바꾸는 **지능형 스케줄링 제어**로 진화할 것이다.
