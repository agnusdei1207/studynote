# 부호화 (Encoding) - Line Coding, Block Coding

> 1. **본질**: 디지털 데이터를 전송 매체가 수용 가능한 전기적·광학적 신호로 변환(Line Coding)하거나, 에러 제어를 위해 논리적인 비트 패턴으로 재구성(Block Coding)하는 과정이다.
> 2. **가치**: 신호의 동기화(Synchronization) 유지, 직류 성분(DC Component) 제거, 대역폭 효율화 및 에러 검출 능력을 부여하여 통신 시스템의 물리적·논리적 신뢰성을 확보한다.
> 3. **융합**: 물리 계층(L1)의 변조 기술과 데이터 링크 계층(L2)의 에러 제어 기술을 연결하는 가교 역할을 하며, 현대의 고속 유무선 인터페이스(USB 4.0, PCIe 6.0, 5G 등)의 성능 한계를 결정짓는 핵심 기술이다.

+++

## Ⅰ. 개요 (Context & Background)

- **개념**: 부호화(Encoding)는 정보를 전송하기 위해 약속된 규칙에 따라 신호로 바꾸는 작업이다. 크게 전송 매체에 직접 흐르는 전압 파형을 결정하는 **선로 부호화(Line Coding)**와, 데이터 전송 전 에러 검출 비트를 섞어 비트 뭉치로 만드는 **블록 부호화(Block Coding)**로 나뉜다.

- **💡 비유**: 선로 부호화는 목소리를 얼마나 크게, 어떤 높낮이로 낼 것인지 정하는 "발성법"과 같고, 블록 부호화는 상대방이 내 말을 잘 이해했는지 확인하기 위해 중간중간 추임새나 요약을 넣는 "대화법"과 같습니다.

- **등장 배경**:
  1. **동기화 문제**: '0'이나 '1'이 연속해서 오면 수신측에서 비트의 경계를 놓치기 쉽다(Self-Clocking 필요).
  2. **에러 제어**: 전송 중 비트가 뒤집혀도 이를 찾아내거나 고칠 수 있는 논리적 구조가 필요했다.
  3. **물리적 제약**: 통신 선로(케이블, 변압기 등)는 직류(DC) 성분을 통과시키지 못하는 경우가 많아 신호의 평균 전압을 0으로 맞추어야 했다.

- **📢 섹션 요약 비유**: 복잡한 정보를 전송 가능한 '패키지'로 포장하고, 그 패키지를 운송 수단(매체)에 맞는 '형태'로 가공하는 종합 물류 공정과 같습니다.

+++

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 부호화 체계의 이원 구조 (Line vs Block)

| 구분 | 선로 부호화 (Line Coding) | 블록 부호화 (Block Coding) |
|:---|:---|:---|
| **주요 목적** | 물리적 신호 전송, 동기화, DC 제거 | 에러 탐지 및 교정, 런 길이(Run-length) 제한 |
| **작동 계층** | 물리 계층 (L1) | 데이터 링크 계층 (L2) |
| **처리 단위** | 비트(Bit) → 신호 레벨(Voltage) | 비트 묶음(Data) → 부호어(Codeword) |
| **대표 방식** | NRZ, RZ, Manchester, AMI | mBnB (4B/5B, 8B/10B), Hamming, CRC |
| **비유** | 전선에 흐르는 전기의 물결 모양 | 단어 끝에 붙이는 '끝'이라는 마침표 |

+++

### 부호화 프로세스 및 데이터 흐름

데이터는 전송 전 블록 부호화를 거쳐 논리적 견고함을 갖춘 뒤, 선로 부호화를 통해 실제 전기 신호로 변환된다.

```text
 [ 송신측 부호화 단계 ]
 
  원본 데이터 (m 비트) ──▶ [ 블록 부호화 (mBnB) ] ──▶ 부호화된 비트 (n 비트)
                                    │                     (n > m)
                                    ▼
                          [ 선로 부호화 (Line Coding) ] ──▶ 전기/광 신호 (Waveform)
                                    │
                                    ▼
                          [ 물리적 전송 매체 (Media) ]
```

**[다이어그램 해설]** 먼저 **블록 부호화**(예: 4B/5B)를 통해 4비트 데이터를 5비트 부호어로 변환한다. 이때 남는 1비트는 에러 검출이나 특정 제어 신호로 활용되며, 0이나 1이 너무 길게 이어지지 않도록 패턴을 제한한다. 이후 **선로 부호화**(예: Manchester)를 통해 이 비트들을 실제 전압의 높낮이나 변화(Edge)로 바꾸어 선로에 흘려보낸다. 수신측은 이 역순으로 복호화(Decoding)를 진행한다.

+++

### 부호화 성능의 4대 평가 지표

성공적인 부호화 설계를 위해 다음 지표들이 최적화되어야 한다.

```text
  1. 자기 동기화 (Self-Clocking): 데이터 신호 자체에 타이밍 정보를 포함하여 별도의 클럭선 없이 비트 추출 가능 여부
  2. DC 성분 (DC Component): 신호의 평균 전압이 0인지 여부 (변압기 결합 회로에서 필수)
  3. 대역폭 효율 (Bandwidth Efficiency): 주어진 주파수 대역에서 얼마나 많은 비트를 보낼 수 있는가
  4. 에러 검출 능력: 부호 규칙을 위반한 신호를 수신측에서 감지할 수 있는가
```

**[다이어그램 해설]** 예를 들어 NRZ(Non-Return to Zero) 방식은 대역폭 효율은 좋으나 자기 동기화 능력이 없어 별도의 동기화 기법이 필요하다. 반면 맨체스터(Manchester) 방식은 매 비트 중간에 반드시 신호 변화를 주어 자기 동기화가 완벽하지만, 대역폭을 NRZ의 2배나 사용한다. 이러한 트레이드오프(Trade-off)를 고려하여 통신 환경에 맞는 부호화 방식을 선택하는 것이 설계의 핵심이다.

- **📢 섹션 요약 비유**: 트럭에 짐을 실을 때, 짐이 쏟아지지 않게 묶는 것(블록 부호화)과 트럭이 고속도로 규정에 맞는 속도와 모양으로 달리는 것(선로 부호화)의 조화입니다.

+++

## Ⅲ. 융합 비교 및 다각도 분석

### 블록 부호화 기법 상세 비교 (4B/5B vs 8B/10B)

| 항목 | 4B/5B 부호화 | 8B/10B 부호화 |
|:---|:---|:---|
| **매핑 규칙** | 4비트 → 5비트 | 8비트 → 10비트 |
| **오버헤드** | 25% ($1/4$) | 25% ($2/8$) |
| **특징** | 고속 이더넷(100Base-FX)에서 사용 | 기가비트 이더넷, USB 3.0, PCIe에서 사용 |
| **장점** | 구현이 매우 단순함 | DC 밸런스 유지 능력이 탁월함 (Disparity 제어) |
| **단점** | 0의 연속을 완벽히 막지 못함 | 매핑 테이블이 복잡함 |

### 과목 융합 관점

- **컴퓨터 구조 (CA)**: CPU와 메모리 사이의 고속 버스(예: DDR5)나 저장장치 인터페이스(NVMe)에서 전송 신뢰성을 위해 128B/132B와 같은 고효율 블록 부호화를 적용한다.
- **보안 (Security)**: 부호화 규칙을 교묘하게 위반하는 신호를 보내 시스템의 오동작을 유도하는 'Physical Layer Attack'에 대비하여, 비정상 부호 패턴을 탐지하는 로직이 필요하다.

- **📢 섹션 요약 비유**: 4B/5B가 5명 중 1명을 보초로 세우는 것이라면, 8B/10B는 10명 중 2명을 보초로 세워 더 철저하게 감시하고 대형을 유지하는 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 기가비트 이더넷 광케이블 전송**: 광섬유를 통해 1Gbps 데이터를 보낼 때, 직류 성분이 누적되면 수신단 광센서의 임계값이 틀어져 에러가 발생한다. 이를 해결하기 위해 **8B/10B 블록 부호화**로 DC 균형을 맞춘 뒤, **NRZ 선로 부호화**로 전송하여 고속성과 안정성을 동시에 확보한다.

2. **시나리오 — 노이즈가 심한 공장 자동화 현장**: 전자기 노이즈가 심한 환경에서는 신호의 레벨(High/Low)보다는 변화(Edge)를 감지하는 것이 유리하다. 이때는 대역폭 손해를 감수하더라도 **맨체스터(Manchester) 부호화**를 선택하여 노이즈 내성을 키우고 동기화 안정성을 높이는 의사결정을 내린다.

### 도입 체크리스트
- **매체 특성 확인**: 사용하는 케이블이 DC 성분을 허용하는가? (허용 안 되면 AMI나 Manchester 필수)
- **클럭 동기화 방식**: 별도의 클럭 신호를 보내는가, 아니면 데이터에서 추출해야 하는가? (추출해야 하면 Self-Clocking 필수)
- **전송 효율 목표**: 오버헤드(Overhead)를 몇 %까지 허용할 수 있는가? (효율 중시 시 mBnB의 m 값을 키움)

### 안티패턴
- **부호화 방식 혼선**: 송신측은 NRZ로 보내는데 수신측이 RZ로 해석하도록 설정하면 전송은 되지만 데이터가 완전히 깨지는 현상이 발생한다.
- **Scrambling 생략**: 블록 부호화만 믿고 데이터를 뒤섞는 스크램블링을 생략하면, 특정 데이터 패턴에서 전자기 간섭(EMI)이 급증할 수 있다.

- **📢 섹션 요약 비유**: 무조건 빠른 차를 고르는 것이 아니라, 비포장도로(노이즈 심한 선로)인지 고속도로(광섬유)인지에 따라 SUV나 스포츠카를 골라 타는 지혜가 필요합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 전 | 도입 후 | 개선 효과 |
|:---|:---|:---|:---|
| **동기화** | 장시간 데이터 수신 시 틀어짐 | 신호 변화를 통한 자동 보정 | 비트 동기 유지율 **100%** 근접 |
| **신뢰성** | 에러 탐지 수단 없음 | 부호 규칙 위반 탐지 | L1/L2 레벨 에러 감지 능력 확보 |
| **물리** | 장치 소손 및 신호 왜곡 | DC 제거로 회로 보호 | 하드웨어 수명 연장 및 전송 거리 증대 |

### 미래 전망
데이터 전송 속도가 급격히 올라가면서(100Gbps+) 기존의 단순 부호화는 한계에 달했다. 최근에는 0과 1의 두 단계 전압이 아닌, 4단계 전압을 사용하는 **PAM-4(Pulse Amplitude Modulation 4-level)** 부호화가 PCIe 6.0 및 800G 이더넷의 표준으로 자리 잡고 있다. 이는 한 번에 2비트를 보낼 수 있어 대역폭 효율을 극대화한다.

### 참고 표준
- **IEEE 802.3**: 이더넷의 계층별 부호화 규격 (10Base-T: Manchester, 100Base-TX: 4B/5B + MLT-3 등)
- **ANSI X3.230**: FC(Fibre Channel)에서의 8B/10B 사용 표준

- **📢 섹션 요약 비유**: 흑백(0, 1)으로만 그리던 그림을 이제는 천연색(PAM-4 등)으로 그려내어, 같은 도화지(대역폭)에 훨씬 풍부한 정보를 담아내는 방향으로 진화하고 있습니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- **NRZ / RZ / Manchester**: 대표적인 선로 부호화 기법들의 세부 동작.
- **mBnB (Block Coding)**: 논리적 비트 패턴 변환 기술의 상세 메커니즘.
- **자기 동기화 (Self-Clocking)**: 데이터 신호에서 클럭을 추출하는 원리와 필요성.
- **PAM-4 (Pulse Amplitude Modulation)**: 차세대 초고속 전송을 위한 다치(Multi-level) 부호화 기술.

## 👶 어린이를 위한 3줄 비유 설명
1. 컴퓨터가 보내는 숫자(0, 1)를 전선이 이해할 수 있게 전기를 "세게" 보내거나 "약하게" 보내는 규칙으로 바꾸는 것이 **부호화**예요.
2. 0이 너무 길게 나오면 전기가 계속 흐르는 건지 멈춘 건지 헷갈릴 수 있는데, 이를 방지하기 위해 비트들을 예쁘게 섞어주는 마법도 부린답니다.
3. 이렇게 약속을 잘 정해두어야 멀리 있는 친구 컴퓨터도 "아, 지금 1을 보냈구나!"라고 정확하게 알아들을 수 있어요!
