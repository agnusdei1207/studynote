# PCM (Pulse Code Modulation) - 표본화, 양자화, 부호화

> 1. **본질**: 아날로그 정보의 진폭을 이산적인 시간 간격으로 추출하고(표본화), 이를 유한한 개수의 대표값으로 매핑한 뒤(양자화), 최종적으로 0과 1의 이진 비트열로 변환하는(부호화) **표준 디지털 전송 방식**이다.
> 2. **가치**: 잡음과 왜곡에 강한 디지털 전송의 핵심 인프라 기술이며, 재생 중계(Regeneration)를 통해 신호 감쇠 없이 무한 거리 전송을 가능하게 하여 현대 정보통신 혁명의 토대가 되었다.
> 3. **융합**: 유선 전화망(PSTN), 디지털 오디오(CD/WAV), 멀티미디어 코덱의 기반 기술이며, 대역폭 효율화를 위한 고도화된 예측 부호화(DPCM, ADPCM) 기술의 모태가 된다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: **PCM(Pulse Code Modulation)**은 파형의 높낮이(아날로그)를 숫자의 나열(디지털)로 바꾸는 3대 공정(표본화 $\to$ 양자화 $\to$ 부호화)을 의미한다. 1937년 알렉 リーブ스에 의해 고안된 이후, 현재 모든 디지털 기기의 음성 처리 표준으로 자리 잡았다.

- **💡 비유**: 복잡한 산맥의 능선(아날로그 신호)을 지도로 만드는 것과 같습니다. 일정한 거리마다 멈춰 서서(표본화), 고도를 미터 단위로 반올림하여 측정하고(양자화), 그 숫자를 이진수 암호로 적어두는(부호화) 과정입니다.

- **등장 배경**:
  1. **잡음 누적 문제**: 아날로그 신호는 증폭기를 거칠 때마다 잡음도 함께 커져 장거리 전송 시 소리가 뭉개진다.
  2. **재생 중계의 필요**: 디지털은 0과 1만 구분하면 되므로, 신호가 약해져도 원래의 0과 1로 '다시 찍어낼 수(Regenerate)' 있어 잡음 없는 깨끗한 전송이 가능하다.
  3. **TDM(시분할 다중화)**: 데이터를 비트 단위로 쪼개면, 여러 명의 통화를 한 선로에 교대로 실어 보내기 훨씬 유리해진다.

- **📢 섹션 요약 비유**: 연속적인 흐름을 딱딱 끊어지는 숫자로 바꾸어, 세월이 흘러도 변치 않는 "불멸의 데이터"를 만드는 연금술과 같은 공정입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### PCM 3단계 변환 프로세스

송신측의 변환 과정은 크게 3단계를 거치며, 전단에 LPF(저역 통과 필터)가 필수적이다.

```text
 [ PCM 송신기 아키텍처 ]
 
 아날로그 ──▶ [ LPF ] ──▶ [ 표본화 ] ──▶ [ 양자화 ] ──▶ [ 부호화 ] ──▶ 디지털
 신호 In      (필터링)     (Sampling)    (Quantize)    (Encoding)     비트열
 
 1. LPF: 에일리어싱 방지를 위해 4kHz 이상의 고주파 차단
 2. 표본화: 8,000Hz(125μs 간격)로 신호 추출 (PAM 신호 생성)
 3. 양자화: 8비트(256단계) 비선형 레벨로 매핑
 4. 부호화: 양자화된 값을 0과 1의 8자리 이진수로 변환
```

---

### 상세 동작 메커니즘

| 단계 | 주요 역할 | 핵심 원리/수식 | 비유 |
|:---|:---|:---|:---|
| **표본화 (Sampling)** | 연속 시간 $\to$ 이산 시간 | 나이퀴스트 정리 ($f_s \ge 2f_m$) | 필름 한 프레임 찍기 |
| **양자화 (Quantizing)** | 연속 진폭 $\to$ 이산 진폭 | 비선형 컴팬딩 ($\mu$-law, A-law) | 숫자로 반올림하기 |
| **부호화 (Encoding)** | 수치 $\to$ 이진 비트열 | $n$-bit Binary Code 생성 | 암호문으로 바꾸기 |

---

### 재생 중계 (Regeneration) 원리

디지털 통신이 아날로그보다 압도적으로 우수한 이유이다.

```text
 [ 아날로그 증폭 vs 디지털 재생 ]
 
 1. 아날로그: [신호+잡음] ──▶ (증폭기) ──▶ [신호*G + 잡음*G] (잡음도 커짐!)
 
 2. 디지털:   [약해진 0/1] ──▶ (재생중계기) ──▶ [새로운 0/1] (잡음 완벽 제거!)
              ~~~~|___|~~~~                   ____|---|____
```

**[다이어그램 해설]** 디지털 신호는 전압이 조금 흔들리거나 낮아져도 일정 임계값(Threshold)만 넘으면 다시 완벽한 사각형 펄스로 복원할 수 있다. 이 성질 덕분에 PCM 데이터는 전 세계를 돌아 수만 개의 중계기를 거쳐도 원본과 똑같은 품질을 유지한다.

- **📢 섹션 요약 비유**: 낡은 사진을 복사기(증폭기)로 복사하면 점점 흐려지지만, 사진의 내용을 숫자로 적어서 다시 그리면(재생 중계) 언제나 새 사진처럼 선명한 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### PCM의 전송 속도 계산 (표준 규격)

전 세계 표준 전화망(PSTN)의 전송 속도인 **64kbps**는 다음과 같이 도출된다.

- **표본화 주파수 ($f_s$)**: 8,000 Hz (음성 대역 4kHz의 2배)
- **양자화 비트 수 ($n$)**: 8 bits (256단계)
- **전송 속도 (Bit Rate)**: $8,000 \times 8 = 64,000 \text{ bps} = 64 \text{ kbps}$
- 이것이 1개의 채널(Time Slot)을 차지하며, T1(24채널)이나 E1(30채널) 다중화 전송의 기초 단위가 된다.

### 과목 융합 관점

- **운영체제 (OS)**: PCM 데이터는 WAV 파일의 원시 포맷(Raw PCM)으로 저장된다. OS의 오디오 믹서는 여러 개의 PCM 스트림을 소프트웨어적으로 더하여(Adding) 동시 출력을 구현한다.
- **전자 회로**: ADC(Analog-to-Digital Converter) 칩 내부의 샘플 앤 홀드(S/H) 회로가 표본화를 수행하고, 비교기(Comparator) 회로가 양자화를 수행한다.

- **📢 섹션 요약 비유**: 64kbps라는 숫자는 정보통신의 "황금비율"과 같습니다. 인간의 목소리를 가장 효율적으로 디지털 상자에 담아내는 완벽한 크기입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 사내 VoIP 교환기 대역폭 설계**: 본사와 지사 간 100명이 동시에 통화해야 한다. 기술사는 PCM(G.711) 사용 시 필요한 순수 대역폭을 $64\text{kbps} \times 100 = 6.4\text{Mbps}$로 산정한다. 여기에 IP 패킷 오버헤드(약 20~40kbps 추가)를 고려하여 실제로는 채널당 80~100kbps의 대역폭을 예약(QoS)해야 함을 판단한다.

2. **시나리오 — 원격 검침(Smart Meter) 통신 선로**: 전력선 통신(PLC) 노이즈가 심해 데이터가 자꾸 깨진다. 기술사는 아날로그 변조 대신 **PCM 디지털 전송**을 제안한다. 재생 중계기를 선로 중간에 배치함으로써, 노이즈가 누적되기 전에 신호를 매번 깨끗하게 새로 고쳐(Regeneration) 전송 신뢰성을 확보한다.

### 도입 체크리스트
- **필요 음질 수준**: 단순 음성 통화(8kHz/8bit)인가, 고음질 음악(44.1kHz/16bit)인가?
- **전송 효율**: 대역폭이 부족하다면 순수 PCM 대신 압축률이 높은 DPCM이나 ADPCM으로 전환할 준비가 되었는가?
- **지연 시간(Latency)**: 표본화와 부호화 과정에서 발생하는 프로세싱 지연이 실시간 서비스 허용치(보통 150ms) 이내인가?

### 안티패턴
- **과도한 오버샘플링**: 필요 이상의 샘플링 레이트는 데이터 양만 늘리고 시스템 부하를 가중시킨다.
- **컴팬딩 불일치**: 송신은 $\mu$-law(북미), 수신은 A-law(유럽)로 설정되면 소리가 찌그러지고 잡음이 섞인다. 국제 구간 연동 시 표준 변환기(Transcoder) 배치가 필수적이다.

- **📢 섹션 요약 비유**: 튼튼한 집을 지으려면 기초(PCM)가 중요합니다. 기초가 부실하면 아무리 화려한 인테리어(압축 알고리즘)를 해도 결국 무너지고 맙니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 전 (Analog) | 도입 후 (PCM Digital) | 개선 효과 |
|:---|:---|:---|:---|
| **잡음 내성** | 거리 비례 잡음 누적 | 재생 중계로 잡음 차단 | 신호 품질 극대화 |
| **운영 효율** | 점대점 물리 회선 위주 | TDM 통한 다중화 용이 | 회선 활용도 **20~30배 향상** |
| **서비스 융합** | 음성/데이터 분리 전송 | 모든 정보를 비트로 통합 | 통합 서비스 디지털망(ISDN) 구현 |

### 미래 전망
순수 PCM은 대역폭을 많이 차지한다는 단점 때문에 현대의 무선망(LTE/5G)에서는 잘 쓰이지 않지만, 그 변형 기술인 **VoLTE(AMR-WB)** 코덱 등은 여전히 PCM의 원리에 기반한다. 또한 비압축 원음(Hi-Res PCM)은 여전히 오디오 애호가들과 전문가들 사이에서 최고의 가치로 인정받고 있으며, 차세대 인터페이스 기술에서도 '무손실'의 기준점으로서 그 권위를 유지할 것이다.

### 참고 표준
- **ITU-T G.711**: 64kbps PCM 음성 부호화 국제 표준
- **IETF RFC 3551**: RTP 프로토콜에서의 PCM 데이터 페이로드 규격

- **📢 섹션 요약 비유**: 0과 1이라는 두 개의 벽돌로 쌓아 올린 정보의 성벽은, 물리적인 세월과 잡음의 공격에도 무너지지 않는 현대 통신의 철옹성입니다.

## 👶 어린이를 위한 3줄 비유 설명
1. **PCM**은 말소리를 컴퓨터가 이해할 수 있게 숫자로 바꾸는 세 가지 마법(표본화, 양자화, 부호화)이에요.
2. 소리를 아주 잘게 쪼개서 점을 찍고, 그 점의 높이를 숫자로 적은 다음, 0과 1이라는 비밀번호로 바꾸는 거죠.
3. 이렇게 숫자로 바꿔두면 멀리 보내도 소리가 변하지 않고 언제나 처음처럼 깨끗하게 들린답니다!
