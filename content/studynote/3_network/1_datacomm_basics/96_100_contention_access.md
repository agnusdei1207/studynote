# 경쟁 기반 매체 접근 제어 (ALOHA, CSMA/CD, CSMA/CA)

> 1. **본질**: 중앙의 통제 없이 다수의 노드가 공통 전송 매체에 자율적으로 접근하여 데이터를 전송하며, 충돌(Collision)이 발생할 경우 이를 탐지하거나 회피하여 재전송하는 **경쟁 기반(Contention-based)** 다중 접속 기술 체계이다.
> 2. **가치**: 복잡한 스케줄링 없이도 유연한 망 구성이 가능하며, 특히 **CSMA/CD**는 유선 이더넷의, **CSMA/CA**는 무선 Wi-Fi의 표준 접근 제어 로직으로서 전 세계 로컬 네트워크의 폭발적 성장을 견인했다.
> 3. **융합**: 확률론적 재전송 알고리즘(BEB)과 물리적 신호 감지 기술이 결합된 형태이며, 최근 고밀도 IoT 환경에서의 충돌 최소화를 위한 지능형 백오프(Backoff) 기술로 진화하고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: **ALOHA**는 충돌을 신경 쓰지 않고 일단 보내는 가장 원시적인 방식이다. **CSMA/CD**는 보내기 전에 남이 쓰는지 확인(Carrier Sense)하고, 보내는 도중에 충돌이 나면 즉시 멈추는 방식이다. **CSMA/CA**는 무선 환경 특성상 충돌 감지가 어려워, 미리 '조심조심' 기다렸다가 보내는 방식이다.

- **💡 비유**: 토론회에 비유할 수 있습니다. ALOHA는 아무 때나 끼어들어 말하는 무질서한 토론입니다. CSMA/CD는 상대방이 말을 멈췄을 때 끼어들되, 동시에 말이 겹치면 즉시 사과하고 멈추는 예의 바른 토론입니다. CSMA/CA는 말을 하기 전에 미리 "저 말해도 될까요?"라고 수줍게 손을 드는 조심스러운 토론과 같습니다.

- **등장 배경**:
  1. **중앙 통제의 비효율**: 모든 단말의 접속 순서를 기지국이 정해주기에는 노드 수가 너무 많고 트래픽이 불규칙했다.
  2. **유선 대역폭 공유**: 하나의 동축 케이블에 여러 PC를 연결했을 때 발생하는 충돌을 해결해야 했다. (이더넷의 탄생)
  3. **무선 환경의 한계**: 무선은 자기가 쏘는 신호가 너무 세서 남이 쏘는 신호를 동시에 듣는(Collision Detection) 것이 물리적으로 불가능했다. (Wi-Fi의 탄생)

- **📢 섹션 요약 비유**: 약속된 순서 없이 "눈치껏" 대화에 끼어드는 기술이지만, 그 눈치(Sense)를 얼마나 과학적으로 시스템화하느냐가 성능의 핵심입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. ALOHA (Pure ALOHA, Slotted ALOHA)

- **Pure ALOHA**: 데이터가 생기면 즉시 전송. 충돌 시 일정 시간 후 재전송. 최대 효율 18.4%.
- **Slotted ALOHA**: 시간을 슬롯으로 나누어 슬롯 시작점에서만 전송. 충돌 확률을 절반으로 줄여 최대 효율 36.8% 달성.

---

### 2. CSMA/CD (Collision Detection) — 이더넷(IEEE 802.3)

- **Listen Before Talk**: 전송 전 매체가 비었는지 확인.
- **Collision Detection**: 전송 중 전압 변화를 감시하여 충돌 감지 시 즉시 전송 중단.
- **Jam Signal**: 충돌 사실을 다른 노드들에게 알리기 위해 짧은 잡음 신호를 방출.
- **BEB (Binary Exponential Backoff)**: 충돌 횟수가 늘어날수록 재전송 대기 시간을 지수함수적으로 늘려 혼잡을 해소.

---

### 3. CSMA/CA (Collision Avoidance) — 무선랜(IEEE 802.11)

- **LBT (Listen Before Talk)**: 매체 상태 확인.
- **IFS (Inter Frame Space)**: 전송 전 의무적으로 기다려야 하는 최소 시간.
- **Random Backoff**: 충돌을 피하기 위해 각자 무작위의 대기 시간을 추가로 가짐.
- **RTS/CTS**: '숨은 터미널 문제(Hidden Terminal)' 해결을 위해 송신권 요청/허가 과정을 거침.

---

### 재전송 알고리즘: BEB (Binary Exponential Backoff)

충돌 발생 시 재전송 대기 시간($T$)을 결정하는 핵심 로직이다.

```text
 [ BEB 알고리즘 공식 ]
 
 T = Random(0, 2^k - 1) * SlotTime
 
 * k: 충돌 횟수 (최대 10~15회)
 * 충돌이 반복될수록 대기 시간의 범위가 2, 4, 8, 16...으로 넓어짐.
```

**[다이어그램 해설]** 한 번 충돌하면 금방 다시 시도하지만, 계속 충돌한다는 것은 망이 혼잡하다는 뜻이므로 대기 범위를 크게 넓혀 노드들이 서로 부딪힐 확률을 통계적으로 낮춘다. **하이퍼 익스포넨셜 백오프**는 이보다 더 급격하게 대기 시간을 늘려 극심한 혼잡 상황을 빠르게 정리하는 변종 기술이다.

- **📢 섹션 요약 비유**: 사고가 난 사거리에서 모든 차가 동시에 다시 출발하려 하면 또 사고가 납니다. 각자 머릿속으로 주사위를 굴려 나온 숫자만큼 기다렸다가 한 대씩 조심스레 출발하는 지능형 양보 시스템입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### CSMA/CD vs CSMA/CA 정밀 비교

| 비교 항목 | CSMA/CD (유선) | CSMA/CA (무선) |
|:---|:---|:---|
| **매체 특성** | 전압 감지 용이 (Cable) | 전압 감지 불가 (Air) |
| **핵심 전략** | **충돌 시 즉시 중단** (효율적) | **충돌 사전 회피** (보수적) |
| **주요 메커니즘** | Jam Signal, BEB | **IFS, RTS/CTS, ACK** |
| **효율성** | 높음 (전송 중 중단 가능) | 보통 (예약 과정 오버헤드) |
| **표준** | IEEE 802.3 (Ethernet) | IEEE 802.11 (Wi-Fi) |

### 과목 융합 관점

- **운영체제 (OS)**: 네트워크 드라이버 레벨에서 BEB 알고리즘의 `k` 값을 관리하며, 재전송 횟수 초과 시 시스템에 'Network Unreachable' 에러를 전달한다.
- **확률 및 통계**: 경쟁 기반 방식의 성능은 '포아송 분포(Poisson Distribution)'를 따르는 트래픽 모델로 분석된다. 사용자 수가 늘어날수록 충돌 확률이 기하급수적으로 올라가 성능이 급락하는 'Througput-Delay' 상관관계가 설계의 핵심이다.

- **📢 섹션 요약 비유**: CD가 "싸우다가 빨리 멈추는 법"을 가르친다면, CA는 "싸움이 날 것 같으면 아예 양보하는 법"을 가르칩니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 사무실 내 Wi-Fi 속도 저하**: 사용자 수는 적은데 속도가 안 나온다. 기술사는 **CSMA/CA의 RTS/CTS** 설정을 점검한다. 파티션 등 장애물로 인해 서로의 존재를 모르는 단말기들이 동시에 기지국에 신호를 쏴서 충돌이 잦을 수 있기 때문이다. 기술사는 RTS 임계값을 낮추어 강제로 예약 전송을 하게 함으로써 '숨은 터미널 문제'를 해결하고 처리량을 회복시킨다.

2. **시나리오 — 이더넷 더미 허브(Dummy Hub) 교체**: 100명이 연결된 허브 기반 망에서 충돌이 너무 잦아 업무가 불가능하다. 기술사는 **CSMA/CD의 경쟁 도메인(Collision Domain)**을 분석한다. 허브는 모든 노드가 하나의 경쟁 권역에 묶여 효율이 바닥임을 진단하고, 포트별로 경쟁 도메인을 분리해주는 **L2 스위치**로 전량 교체하여 충돌 확률을 0으로 만드는 아키텍처 고도화를 수행한다.

### 도입 체크리스트
- **망의 규모**: 노드 수가 수천 개 이상인가? (경쟁 기반은 부적합, 토큰이나 스케줄링 방식 고려)
- **전파 환경**: 장애물이 많은가? (RTS/CTS 필수 활성화)
- **실시간성 요구**: 지연 시간이 일정해야 하는가? (경쟁 기반은 가변 지연이 심하므로 전용 대역폭 예약 기능 필요)

### 안티패턴
- **Full-Duplex 스위치에서 CSMA/CD 활성화**: 현대의 스위칭 이더넷은 송수신 경로가 분리되어 충돌이 나지 않는다. 여기서 굳이 CD 로직을 켜두는 것은 성능만 갉아먹는 행위다.
- **무조건적인 짧은 백오프**: 전송 성공률을 높이겠다고 대기 시간을 짧게 고정하면, 혼잡 시 모든 노드가 일제히 재전송을 시도하여 전체 망이 마비(Congestion Collapse)된다.

- **📢 섹션 요약 비유**: 눈치 게임(경쟁)도 인원이 적을 때나 재밌지, 수천 명이 모이면 아수라장이 됩니다. 사람이 많아지면 결국 법(스케줄링)과 칸막이(스위칭)가 필요합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 전 (ALOHA) | 도입 후 (CSMA/CD, CA) | 개선 효과 |
|:---|:---|:---|:---|
| **채널 효율** | 18.4% | **80~90% 이상** | 전송 용량 **5배 이상** 증대 |
| **망 확장성** | 노드 추가 시 붕괴 위험 | 자율 분산 제어로 용이 | 유연한 네트워크 확장 가능 |
| **사용자 경험** | 잦은 데이터 유실 | 안정적인 무선 연결 | 서비스 신뢰도 향상 |

### 미래 전망
현대의 Wi-Fi 6(802.11ax)는 경쟁 방식의 한계를 넘기 위해 경쟁과 예약이 결합된 **OFDMA** 기술을 도입했다. 하지만 단말이 처음 망에 접속하거나 짧은 패킷을 보낼 때는 여전히 CSMA/CA 방식이 가장 효율적이다. 미래의 6G에서는 인공지능이 채널의 혼잡도를 딥러닝으로 학습하여, 최적의 백오프 시간과 전송 타이밍을 노드별로 다르게 부여하는 **AI 기반 MAC**이 등장할 것이다.

### 참고 표준
- **IEEE 802.3**: Carrier Sense Multiple Access with Collision Detection (Ethernet)
- **IEEE 802.11**: Wireless LAN Medium Access Control (MAC) and Physical Layer (PHY)

- **📢 섹션 요약 비유**: 보이지 않는 전파의 무질서 속에서 "듣고, 기다리고, 양보하는" 인간의 사회적 지능을 기계 언어로 구현해낸, 인터넷 시대의 가장 위대한 약속입니다.
