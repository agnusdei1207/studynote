# 926-939. 네트워크 보안 및 운용 심화 (RPKI, Sinkhole, Hybrid)

> **핵심 인사이트**: 인터넷의 근간이 되는 라우팅(BGP)과 주소 찾기(DNS)는 설계 당시 보안을 고려하지 않았다. 이를 보완하기 위해 '디지털 인감 장부(RPKI)'로 가짜 경로를 차단하고, '블랙홀(Sinkhole)'로 좀비 PC의 명령을 가로채며, 성능과 보안을 다 잡은 '하이브리드 암호'가 통신망의 마지막 방어선이 된다.

---

## Ⅰ. 라우팅 보안: RPKI (Resource PKI)
인터넷의 이정표인 BGP 라우팅 정보가 위조되지 않았음을 확인하는 **디지털 서명 인프라**입니다.

* **목적**: BGP 하이재킹(남의 네트워크 경로를 가로채기) 방어.
* **원리**: IP 주소 소유자가 자신의 주소 대역(Prefix)과 이를 광고할 수 있는 AS 번호를 **디지털 인증서**로 등록합니다. 라우터는 이 정보를 대조하여 가짜 광고를 무시합니다.

---

## Ⅱ. DNS 싱크홀 (DNS Sinkhole)
감염된 좀비 PC가 공격자의 C&C 서버로 접속하려 할 때, 이를 **가짜 IP(싱크홀 서버)로 우회**시켜 차단하는 기술입니다.

* **동작**: KISA와 같은 기관에서 관리하는 악성 도메인 목록을 DNS 서버에 미리 등록해 둡니다.
* **효과**: DDoS 공격을 미연에 방지하고, 어떤 PC가 감염되었는지 파악하여 치료를 유도할 수 있습니다.

---

## Ⅲ. 하이브리드 암호 시스템
대칭키 암호(속도)와 비대칭키 암호(안전한 키 배포)의 장점만 결합한 현대 보안의 표준입니다.

* **동작 단계**:
  1. 실제 데이터는 무작위로 생성한 **대칭키(세션키)**로 암호화합니다.
  2. 이 대칭키만 상대방의 **공개키**로 암호화하여 전송합니다.
* **사례**: SSL/TLS, PGP(이메일 보안) 등 거의 모든 현대 암호 시스템.

---

## Ⅳ. 기타 운용 및 분석 기술
* **EMP 방호**: 전자기 펄스 공격으로부터 장비를 보호하기 위해 금속 차폐(Faraday Cage)와 특수 쉴딩 케이블을 사용하는 물리 보안입니다.
* **파일 카빙 (File Carving)**: 네트워크 패킷 덤프(PCAP) 파일에서 헤더와 푸터 정보를 분석하여, 조각난 파일들을 원래대로 복구해내는 디지털 포렌식 기술입니다.
* **허니팟 (Honeypot)**: 해커를 유인하기 위해 일부러 취약하게 만든 가짜 서버입니다. 공격 기법을 분석하고 침투 경로를 파악하는 용도로 씁니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[하이브리드 암호화 흐름]

 [ DATA ] ──( 대칭키 )──> [ 암호 데이터 ] ┐
                                         │  ( 전송 )
 [대칭키] ──( 공개키 )──> [ 암호화된 키 ] ┘
```

📢 **섹션 요약 비유**: **RPKI**는 가짜 이정표를 세워 트럭을 뺏는 도둑을 막기 위해, 모든 이정표에 국가 공인 도장을 찍는 것입니다. **DNS 싱크홀**은 보이스피싱 전화를 걸었을 때 자동으로 경찰서로 연결되게 만드는 '착한 가로채기' 서비스입니다. **하이브리드 암호**는 무거운 금고(비대칭키)를 통째로 보내는 대신, 내용물은 가벼운 가방(대칭키)에 담고 가방 열쇠만 작은 금고에 넣어 전달하는 효율적인 배달법입니다. **허니팟**은 도둑을 잡기 위해 대문에 돈다발이 든 가짜 가방을 놓아두는 함정입니다.
