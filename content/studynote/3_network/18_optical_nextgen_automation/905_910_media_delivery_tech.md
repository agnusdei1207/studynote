# 905-910. 미디어 전송 고도화 (HLS, CMAF, FEC)

> **핵심 인사이트**: 전 세계가 유튜브와 넷플릭스를 본다. 미디어 전송의 핵심은 '끊김 없음'과 '낮은 지연'이다. 파일을 잘게 쪼개어 HTTP로 실어 나르는 HLS 방식부터, 파편화된 규격을 하나로 합친 CMAF, 그리고 깨진 데이터를 스스로 복구하는 FEC 기술이 최고의 시청 경험을 만든다.

---

## Ⅰ. HTTP 기반 스트리밍 (HLS)
애플에서 만든 가장 대중적인 비디오 스트리밍 프로토콜입니다.

* **원리**: 전체 영상을 몇 초 단위의 작은 조각(Segment, `.ts`)으로 나누고, 이 조각들의 목록인 인덱스 파일(`.m3u8`)을 제공합니다.
* **장점**: 일반 웹 서버(HTTP)를 그대로 쓸 수 있어 방화벽에 강하고 CDN 배포가 매우 쉽습니다.

---

## Ⅱ. 미디어 포맷의 통합: CMAF
* **CMAF (Common Media Application Format)**:
  - **배경**: 이전에는 애플용(HLS)과 구글/MS용(DASH) 파일을 각각 따로 만들어야 해서 저장 공간 낭비가 심했습니다.
  - **혁신**: 두 규격에서 모두 읽을 수 있는 **단일 미디어 파일 구조**를 정의했습니다. 한 번만 인코딩하면 모든 기기에서 볼 수 있습니다. (Chunked Transfer를 통한 초저지연 구현 포함)

---

## Ⅲ. 전송 품질 보상 및 측정

### 1. FEC (Forward Error Correction)
* **개념**: 데이터를 보낼 때 여분의 정보(패리티)를 미리 섞어서 보내는 기술입니다.
* **효과**: 수신 측에서 패킷 일부가 사라져도, 재전송 요청 없이 **남은 조각들로 원본을 스스로 복구**할 수 있습니다. 실시간 화상 회의 지연 감소에 필수입니다.

### 2. 지터 버퍼 (Jitter Buffer)
* 네트워크 상황에 따라 패킷이 들쭉날쭉하게 도착할 때, 이를 잠시 모아두었다가 일정한 간격으로 내보내는 완충 장치입니다. 소리가 끊기거나 떨리는 현상을 방지합니다.

### 3. MOS (Mean Opinion Score)
* 음성 및 영상 통화의 품질을 측정하는 5점 만점의 주관적 지표입니다. (5: 아주 좋음 ~ 1: 매우 나쁨)

---

## Ⅳ. 개념 맵 및 요약

```ascii
[미디어 스트리밍 전달 구조]

  [ 원본 영상 ] ──> [ Chunk 분할 ] ──> [ CMAF 포장 ] ──> [ CDN ] ──> [ Player ]
                                                                      │
                                            <── (속도에 맞춰 화질 선택) ──┘
```

📢 **섹션 요약 비유**: **HLS**는 두꺼운 책 한 권을 한 페이지씩 낱장으로 찢어서 전달하는 것입니다. 중간에 한 장 잃어버려도 다음 장을 바로 읽을 수 있죠. **CMAF**는 안드로이드 폰과 아이폰 모두에서 열리는 '만능 파일 포맷'입니다. **FEC**는 택배를 보낼 때 물건이 파손될까 봐 부품 여유분을 상자에 같이 넣어주는 센스이고, **MOS**는 영화를 보고 난 뒤 별점 5개를 주는 '시청 만족도 조사'와 같습니다.
