# 833-843. 클라우드 로드밸런싱과 인프라 연결 (DSR, VPC, DX)

> **핵심 인사이트**: 클라우드는 '가상'이지만 그 밑바닥은 철저한 '물리'의 법칙을 따른다. 부하 분산 장비의 병목을 없애는 DSR 기술, 나만의 격리된 영토를 만드는 VPC, 그리고 클라우드와 내 사무실을 고속도로로 잇는 Direct Connect는 하이브리드 클라우드 구축의 필수 3요소다.

---

## Ⅰ. 로드 밸런싱의 고도화: DSR (Direct Server Return)
* **문제**: 일반적인 로드밸런서는 들어오는 패킷과 나가는 패킷을 모두 처리해야 하므로, 응답 데이터가 큰 서비스에서는 로드밸런서 자체가 병목이 됩니다.
* **해결 (DSR)**: 요청은 로드밸런서가 받지만, **응답은 서버가 클라이언트에게 직접(Direct)** 쏩니다.
* **장점**: 로드밸런서의 부하가 획기적으로 줄어들어 대규모 트래픽 처리에 유리합니다.

---

## Ⅱ. VPC (Virtual Private Cloud)
퍼블릭 클라우드 내에 구축하는 사용자 전용의 **논리적으로 격리된 가상 네트워크**입니다.

* **구성 요소**: 
  - **Subnet**: 공인 IP를 가진 퍼블릭 서브넷과 사설 IP만 가진 프라이빗 서브넷으로 구분.
  - **Internet Gateway (IGW)**: 외부 인터넷과의 연결 통로.
  - **NAT Gateway**: 프라이빗 서브넷의 기기가 밖으로만 나갈 수 있게 해주는 장치.

---

## Ⅲ. 클라우드-온프레미스 연결 (Hybrid Cloud)
기업의 데이터센터와 클라우드를 연결하는 방식입니다.

1. **Direct Connect (AWS) / ExpressRoute (Azure)**: 인터넷이 아닌 **전용선**을 통해 클라우드와 직접 연결하는 기술입니다. 보안이 강력하고 속도가 일정합니다.
2. **IPsec VPN**: 기존 인터넷망을 터널링으로 묶어 저렴하게 연결하는 방식입니다.

---

## Ⅳ. 가상 머신 네트워크 최적화
* **SR-IOV**: 하이퍼바이저를 거치지 않고 가상 머신이 랜카드(NIC)에 직접 접근하게 하여 성능을 높이는 하드웨어 가상화 기술입니다.
* **DPDK**: 리눅스 커널의 무거운 네트워크 스택을 우회하여, 사용자 공간(User-space)에서 초고속으로 패킷을 처리하는 라이브러리입니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[하이브리드 클라우드 연결도]

 [ On-premise IDC ] <──────( Direct Connect )──────> [ AWS VPC ]
 (우리 회사 전산실)          (전용선: 고속/안전)          (클라우드 영토)
                                                        │
                                            [ Public Subnet ] -- [ IGW ] -- (Internet)
                                            [ Private Subnet ] -- [ NAT ]
```

📢 **섹션 요약 비유**: **DSR**은 맛집 대기 줄에서 주문은 점원(LB)이 받지만, 음식 배달은 주방장(Server)이 손님에게 직접 가져다주는 신속한 서비스입니다. **VPC**는 거대한 클라우드라는 광장에 나만 들어갈 수 있는 울타리를 치고 '비밀 기지'를 만드는 것이고, **Direct Connect**는 기지국까지 가는 일반 도로(인터넷) 대신 우리 집 대문에서 기지국까지 뚫어놓은 '전용 지하도로'와 같습니다.
