# 815-820. 가상화 네트워크와 오버레이(VXLAN)

> **핵심 인사이트**: 클라우드 세상에서 서버는 수시로 이사(VM Migration)를 다닌다. 하지만 물리적인 선을 매번 다시 깔 수는 없다. VXLAN은 물리적인 바닥(Underlay) 위에 소프트웨어로 그린 가상의 통로(Overlay)를 깔아, 서버가 어디로 가든 똑같은 네트워크 환경을 유지하게 해 준다.

---

## Ⅰ. 오버레이(Overlay)와 언더레이(Underlay)
1. **언더레이 (Underlay)**: 실제로 깔린 물리적인 케이블과 스위치들. 목적지까지 패킷을 전달하는 '바닥' 역할을 합니다. (주로 IP 라우팅 기반)
2. **오버레이 (Overlay)**: 물리망 위에 터널링 기술을 사용하여 만든 **논리적인 가상 네트워크**입니다. 물리 구조와 상관없이 자유롭게 L2/L3 네트워크를 구성할 수 있습니다.

---

## Ⅱ. VXLAN (Virtual eXtensible LAN)
가장 널리 쓰이는 네트워크 가상화 캡슐화 기술입니다. (RFC 7348)

* **배경**: 기존 VLAN은 개수가 4,096개로 제한되어 있어 수백만 개의 테넌트를 수용하는 클라우드 환경에 부적합했습니다.
* **특징**:
  - **VNI (VXLAN Network ID)**: 24비트 ID를 사용하여 최대 **1,600만 개**의 가상망을 만듭니다.
  - **L2 over L3**: 2계층 이더넷 프레임을 **UDP 패킷(포트 4789)**으로 감싸서 3계층 망을 가로질러 전달합니다.
* **VTEP (VXLAN Tunnel End Point)**: VXLAN 터널의 입구와 출구 역할을 하는 장치(스위치나 서버)입니다.

---

## Ⅲ. EVPN (Ethernet VPN)
VXLAN 터널을 더 똑똑하게 관리하기 위한 제어 평면(Control Plane) 기술입니다.

* **동작**: **BGP** 프로토콜을 사용하여 어떤 MAC 주소가 어느 VTEP에 있는지 정보를 공유합니다. 
* **장점**: 불필요한 브로드캐스트 트래픽(BUM 트래픽)을 억제하고, 멀티호밍을 지원하여 네트워크 효율을 극대화합니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[VXLAN 캡슐화 구조]

  [ Outer IP Header ] [ UDP Header ] [ VXLAN Header ] [ Original L2 Frame ]
         │                  │               │                 │
         └─ (물리망 주소)    └─ (포트 4789)   └─ (VNI ID)       └─ (가상머신 데이터)
```

📢 **섹션 요약 비유**: **언더레이**는 실제 땅 위에 닦인 도로이고, **오버레이**는 그 도로 위를 달리는 컨테이너 트럭 안에 실린 '미니어처 마을'과 같습니다. **VXLAN**은 마을의 집(L2 프레임)을 커다란 UDP 상자에 담아 고속도로(L3)를 통해 옆 도시로 옮기는 포장 기술입니다. **EVPN**은 어느 상자에 어느 집이 들어있는지 낱낱이 기록해둔 '중앙 물류 관리 장부'입니다.
