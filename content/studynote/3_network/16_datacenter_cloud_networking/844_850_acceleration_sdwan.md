# 844-850. 인프라 가속과 SD-WAN (DPU, SD-WAN)

> **핵심 인사이트**: 서버의 CPU가 네트워크 패킷을 처리하느라 정작 중요한 애플리케이션 연산을 못 하는 상황이 벌어지고 있다. 이를 해결하기 위해 네트워크 전용 프로세서인 DPU가 등장했으며, 본사와 지사 간의 복잡한 연결 또한 소프트웨어가 알아서 최적 경로를 찾아주는 SD-WAN 시대로 접어들었다.

---

## Ⅰ. 인프라 가속 기술: SmartNIC과 DPU
서버의 메인 CPU 부하를 덜어주기 위해 네트워크 카드가 스스로 지능을 갖게 된 기술입니다.

1. **SmartNIC**: 단순 전송을 넘어 암호화, 가상 스위칭 기능을 하드웨어적으로 처리합니다.
2. **DPU (Data Processing Unit)**: SmartNIC에서 한 단계 더 나아가, 아예 독립적인 CPU와 운영체제를 갖춘 '서버 속의 서버'입니다. 네트워크, 스토리지 제어, 보안 기능을 통째로 CPU에서 뺏어와 직접 처리합니다. (인프라 오프로딩)

---

## Ⅱ. SD-WAN (Software-Defined WAN)
전용선, 인터넷, 5G 등 다양한 광역 회선들을 소프트웨어가 중앙에서 통합 관리하고 제어하는 기술입니다.

* **핵심 기능**:
  - **다중 경로 최적화**: 트래픽의 종류(Zoom, Email 등)에 따라 가장 상태가 좋은 회선을 실시간으로 선택합니다.
  - **제로 터치 프로비저닝 (ZTP)**: 지사에 장비만 갖다 꽂으면 본사 설정이 자동으로 내려받아져 즉시 가동됩니다.
* **장점**: 비싼 전용선 대신 저렴한 인터넷 회선을 섞어 써도 품질(QoS) 보장이 가능하여 비용을 획기적으로 줄입니다.

---

## Ⅲ. 무손실 이더넷 (Lossless Ethernet)
스토리지나 AI 연산 트래픽은 패킷 하나만 사라져도 전체 성능이 급감합니다.
* **PFC (Priority Flow Control)**: 버퍼가 꽉 차기 전, 상대방에게 잠시 멈추라고 신호를 보내 패킷 유실을 원천 차단하는 기술입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[SD-WAN의 지능형 라우팅]

  [ 본사 ] <─── ( 중앙 컨트롤러: "지금 전용선 막히니 인터넷망 써!" )
      │
  ┌───┴───┬────────┬───────┐
  ▼       ▼        ▼       ▼
 [ MPLS ] [ Internet ] [ LTE/5G ]  <─── ( 다양한 회선 혼합 사용 )
  └───────┬────────┴───────┘
          ▼
       [ 지사 ]
```

📢 **섹션 요약 비유**: **DPU**는 사장님(CPU)이 직접 하던 영수증 정리(네트워크 처리)를 대신해주기 위해 새로 채용한 '유능한 경리 과장님'입니다. 사장님은 이제 사업 구상(애플리케이션)에만 집중할 수 있죠. **SD-WAN**은 본사와 지사를 잇는 길을 찾을 때, 국도, 고속도로, 지름길 상황을 실시간으로 파악해서 가장 빠른 길로 안내하는 '스마트 내비게이션'과 같습니다.
