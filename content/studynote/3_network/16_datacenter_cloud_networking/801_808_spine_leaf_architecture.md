# 801-808. 데이터센터 네트워크 아키텍처 (Spine-Leaf)

> **핵심 인사이트**: 데이터센터의 주인공은 더 이상 '사용자'가 아니라 '서버'들이다. 서버 간의 폭발적인 대화(East-West 트래픽)를 감당하기 위해, 데이터센터는 수직적인 계층 구조(3-Tier)를 버리고, 모든 서버가 평등한 거리에 있는 수평적인 'Spine-Leaf' 구조로 진화했다.

---

## Ⅰ. 전통적인 3-Tier 아키텍처와 한계
* **구조**: Core ── Aggregation ── Access (서버 연결)
* **특징**: 외부 사용자가 서버에 접속하는 **North-South** 트래픽 위주로 설계되었습니다.
* **한계**: 마이크로서비스(MSA) 환경에서 서버끼리 대화(East-West)할 때, 패킷이 상위 계층까지 올라갔다 내려와야 하므로 지연 시간(Latency)이 길고 병목 현상이 발생합니다.

---

## Ⅱ. 현대의 Spine-Leaf 아키텍처
모든 리프(Leaf) 스위치가 모든 스파인(Spine) 스위치와 연결되는 2계층 구조입니다.

1. **Leaf Switch**: 서버들이 직접 연결되는 스위치. (ToR 스위치 역할)
2. **Spine Switch**: 리프 스위치들을 서로 연결해 주는 고속 백본.
* **장점**:
  - **예측 가능한 지연**: 어떤 서버끼리 통신하든 무조건 'Leaf-Spine-Leaf' 3단계(Hop)만 거치면 됩니다.
  - **확장성 (Scale-out)**: 대역폭이 부족하면 스파인 스위치를, 서버 포트가 부족하면 리프 스위치를 옆으로 늘리기만 하면 됩니다.

---

## Ⅲ. 핵심 구현 기술

### 1. ECMP (Equal-Cost Multi-Path)
* Spine-Leaf 구조에서 리프와 스파인 사이에는 수많은 동일 비용 경로가 존재합니다. **ECMP**는 이 모든 길을 동시에 활성화하여 트래픽을 분산 전송함으로써 대역폭을 극대화합니다.

### 2. Clos 네트워크
* 여러 개의 소형 스위치를 조합하여 거대한 하나의 비폐쇄형(Non-blocking) 스위칭 구조를 만드는 수학적 모델입니다. Spine-Leaf의 근간이 되는 이론입니다.

---

## Ⅳ. 트래픽 흐름의 구분
* **North-South 트래픽**: 데이터센터 외부(인터넷)와 내부 서버 간의 흐름.
* **East-West 트래픽**: 데이터센터 내부의 서버와 서버, 혹은 컨테이너 간의 흐름. (현대 트래픽의 80% 이상)

---

## Ⅴ. 개념 맵 및 요약

```ascii
[Spine-Leaf 아키텍처 구조]

      [ Spine 1 ]      [ Spine 2 ]      [ Spine 3 ]  (고속 코어)
          │  ╲        ╱  │  ╲        ╱  │
          │    ╲    ╱    │    ╲    ╱    │
      [ Leaf 1 ]      [ Leaf 2 ]      [ Leaf 3 ]  (서버 접속)
          │               │               │
      [ Server ]      [ Server ]      [ Server ]
```

📢 **섹션 요약 비유**: **3-Tier**는 회장님(Core)부터 말단 사원(Access)까지 결재를 올리고 내려받는 수직적 관료 조직입니다. **Spine-Leaf**는 모든 부서장(Leaf)이 공용 회의실(Spine)을 통해 즉각 대화하는 수평적 애자일 조직입니다. **East-West 트래픽**은 옆 부서 동료와 수시로 메신저를 주고받는 협업 과정이며, **ECMP**는 목적지까지 가는 8차선 고속도로의 모든 차선을 빈틈없이 꽉 채워 달리는 지혜입니다.
