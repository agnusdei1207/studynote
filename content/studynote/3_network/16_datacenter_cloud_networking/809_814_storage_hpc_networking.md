# 809-814. 스토리지 및 HPC 네트워킹 (RDMA, InfiniBand)

> **핵심 인사이트**: 데이터 전송의 가장 큰 병목은 '네트워크 선'이 아니라 'CPU'다. CPU를 거치지 않고 메모리에서 메모리로 직접 데이터를 쏘는 RDMA 기술과, 이더넷의 한계를 넘는 인피니밴드는 현대 AI 클러스터와 대규모 스토리지(SAN)의 생존 전략이다.

---

## Ⅰ. 스토리지 네트워킹의 진화
서버와 거대 저장 장치(Storage)를 연결하는 전용망 기술입니다.

1. **FCoE (Fibre Channel over Ethernet)**: 고가의 전용 광채널(FC) 장비 없이, 일반 이더넷 위에서 FC 프로토콜을 돌리는 기술입니다. 물리적 선을 하나로 합칠 수 있습니다.
2. **iSCSI**: IP 네트워크 위에서 스토리지 명령어를 주고받는 방식입니다. 저렴하고 호환성이 좋지만 CPU 부하가 있습니다.

---

## Ⅱ. RDMA (Remote Direct Memory Access)
원격지에 있는 컴퓨터의 메모리에 **CPU의 개입 없이** 직접 접근하여 데이터를 읽고 쓰는 기술입니다.

* **효과**: OS 커널 스택을 거치는 복사(Zero-copy) 과정을 생략하여, CPU 부하를 0에 가깝게 줄이고 지연 시간(Latency)을 획기적으로 단축합니다.
* **구현 방식**:
  - **InfiniBand**: RDMA를 위해 태어난 전용 하드웨어.
  - **RoCE (RDMA over Converged Ethernet)**: 일반 이더넷 위에서 RDMA를 구현한 기술. (현재 AI 데이터센터의 주류)
  - **iWARP**: TCP/IP 기반의 RDMA. 호환성은 좋으나 성능은 RoCE보다 낮음.

---

## Ⅲ. 인피니밴드 (InfiniBand)
초고성능 컴퓨팅(HPC) 및 AI 학습 클러스터에서 사용하는 **초저지연 인터커넥트** 표준입니다.

* **특징**:
  - **Cut-through** 스위칭으로 나노초 단위 지연 보장.
  - 하드웨어 기반의 완벽한 흐름 제어(Credit-based).
  - TCP/IP의 무거운 오버헤드를 아예 제거함.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[데이터 전송 방식의 비교]

1. 일반적인 방식 (TCP/IP)
   [ App ] ──> [ Kernel Stack ] ──> [ CPU 복사 ] ──> [ NIC ] ──> (Network)

2. RDMA 방식 (Zero-copy)
   [ App Memory ] ───────────────────────────────> [ NIC ] ──> (Network)
                   (CPU는 다른 일 하는 중!)
```

📢 **섹션 요약 비유**: **iSCSI**는 고속도로(IP)를 이용해 택배를 나르는 트럭이고, **FCoE**는 전용 철길을 이더넷이라는 도로 위에 깐 트램과 같습니다. **RDMA**는 물건을 옮길 때 굳이 본사(CPU)에 결재 서류(커널 복사)를 올리지 않고, 창고 직원이 옆 동네 창고로 바로 물건을 던져넣는 '다이렉트 통로'입니다. **인피니밴드**는 시속 300km로 달리는 전용 자기부상열차와 같아서, 일반 이더넷 도로의 신호등이나 정체 걱정 없이 AI 연산 데이터를 광속으로 실어 나릅니다.
