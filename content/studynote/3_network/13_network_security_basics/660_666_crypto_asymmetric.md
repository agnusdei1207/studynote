# 660-666. 공개키 암호화와 키 교환 (RSA, ECC, DH)

> **핵심 인사이트**: 모르는 사람과 안전하게 비밀 대화를 하려면 어떻게 해야 할까? 공개키 암호화는 '누구나 잠글 수 있는 자물쇠(공개키)'와 '나만 열 수 있는 열쇠(개인키)'를 분리하여 키 전달의 난제를 해결했다. 특히 타원 곡선(ECC)은 더 짧은 키로도 강력한 보안을 제공하며 현대 모바일 보안의 표준이 되었다.

---

## Ⅰ. 공개키/비대칭키 암호화 (Asymmetric Key)
암호화 키와 복호화 키가 서로 다른 방식입니다.

* **원리**: 
  - **공개키 (Public Key)**: 세상 모든 사람에게 공개. 데이터를 잠글 때(암호화) 사용.
  - **개인키 (Private Key)**: 나만 간직함. 잠긴 데이터를 열 때(복호화) 사용.
* **용도**:
  - **기밀성**: 상대방의 공개키로 암호화하여 전송.
  - **전자서명**: 나의 개인키로 서명하여 내가 보냈음을 증명.

---

## Ⅱ. 주요 알고리즘

### 1. RSA
* **원리**: 아주 큰 수의 **소인수분해**가 어렵다는 수학적 원리를 이용합니다.
* **특징**: 가장 널리 쓰이지만, 보안을 위해 키 길이를 점점 늘려야 하는(2048비트 이상) 단점이 있습니다.

### 2. ECC (Elliptical Curve Cryptography)
* **원리**: **타원 곡선 이산 대수 문제**를 이용합니다.
* **장점**: RSA보다 훨씬 짧은 키(256비트)로 동일한 보안 수준을 냅니다. 연산 속도가 빠르고 메모리를 적게 먹어 스마트폰, IoT, 코인 지갑 등에 필수적입니다.

---

## Ⅲ. 키 교환 기술: 디피-헬만 (Diffie-Hellman)
직접 키를 주고받지 않고도, 공개된 통로에서 양측이 **동일한 대칭키(세션키)를 생성**해내는 혁신적인 방식입니다.

* **동작**: 각자 비밀 숫자를 정하고 공통 숫자를 섞어 서로 교환한 뒤, 다시 자신의 비밀 숫자를 섞으면 마법처럼 똑같은 결과값이 나옵니다.
* **취약점**: 중간에 해커가 끼어들어 가짜 숫자를 넘겨주는 **중간자 공격(MITM)**에 취약하여, 반드시 인증 절차와 함께 사용해야 합니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[공개키 암호화 동작 예시]

  [ 송신자 (Alice) ] ───( Bob의 공개키로 암호화 )───> [ 암호문 ]
                                                         │
  [ 수신자 (Bob) ]   <──( Bob의 개인키로 복호화 )───┘
```

📢 **섹션 요약 비유**: **비대칭키 암호**는 '우체통'과 같습니다. 누구나 편지를 넣을(암호화) 수는 있지만, 편지함을 열어 내용을 볼(복호화) 수 있는 열쇠는 우체부 아저씨(개인키 소유자)만 가지고 있습니다. **RSA**는 아주 거대한 퍼즐을 맞추는 것이고, **ECC**는 복잡한 곡선 위에서 특정한 점을 찾는 보물찾기입니다. **디피-헬만**은 두 사람이 노란색과 파란색 물감을 섞어 초록색을 만드는 법을 멀리서 서로 지켜보며 연구하여, 끝내 똑같은 색깔의 키를 만들어내는 연금술과 같습니다.
