# 651-659. 정보보안 기초와 대칭키 암호화

> **핵심 인사이트**: 보안은 자물쇠를 잠그는 기술(Confidentiality), 자물쇠가 뜯겼는지 확인하는 기술(Integrity), 그리고 필요할 때 언제든 문을 열 수 있는 상태(Availability)를 유지하는 종합 예술이다. 특히 대칭키 암호화는 하나의 열쇠로 광속의 암복호화를 수행하며 현대 통신의 기밀성을 책임진다.

---

## Ⅰ. 정보보안 3대 요소 (CIA Triad)
1. **Confidentiality (기밀성)**: 오직 인가된 사람만 정보를 볼 수 있음. (암호화)
2. **Integrity (무결성)**: 정보가 중간에 변조되지 않았음을 보장. (해시, 전자서명)
3. **Availability (가용성)**: 필요한 시점에 지체 없이 정보에 접근 가능함. (DDoS 방어, 이중화)
* + **Non-repudiation (부인 방지)**: 행위자가 자신의 행위를 나중에 부정할 수 없게 함.

---

## Ⅱ. 대칭키 암호화 (Symmetric Key Cryptography)
암호화할 때와 복호화할 때 사용하는 **키가 동일**한 방식입니다.

* **특징**: 속도가 매우 빠르지만, 키를 안전하게 전달하는 것이 어렵습니다. (키 분배 문제)
* **주요 알고리즘**:
  - **AES (Advanced Encryption Standard)**: 현대의 국제 표준. 128/192/256비트 키 사용.
  - **DES / 3DES**: 과거 표준이나 키 길이가 짧아 현재는 사용 권장 안 함.
  - **SEED / ARIA / LEA**: 대한민국 국가 표준 알고리즘.

---

## Ⅲ. 블록 암호 운영 모드 (Block Cipher Modes)
고정된 크기의 블록(예: 128비트)을 반복해서 암호화할 때 보안성을 높이는 방법입니다.

1. **ECB (Electronic Codebook)**: 가장 단순함. 같은 평문은 같은 암호문으로 나와 패턴이 노출됨. (사용 금지 권장)
2. **CBC (Cipher Block Chaining)**: 이전 블록의 결과와 XOR 연산 후 암호화. **IV (Initial Vector)**가 필수적임.
3. **GCM (Galois/Counter Mode)**: 현대의 대세. 암호화와 동시에 **메시지 인증(Integrity)**까지 수행하는 **AEAD** 방식입니다. (TLS 1.3 핵심 모드)

---

## Ⅳ. 스트림 암호 (Stream Cipher)
데이터를 블록 단위가 아닌 비트/바이트 단위로 실시간 암호화하는 방식입니다.
* **ChaCha20**: 모바일 환경에서 매우 빠르고 안전하여 현대 인터넷(QUIC 등)에서 널리 쓰입니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[대칭키 암호화의 핵심]

  [ 평문 (P) ] ────( Key )────> [ 암호문 (C) ] ────( Key )────> [ 평문 (P) ]
      │           (암호화)           │           (복호화)           │
      └──────────────────────────────┴──────────────────────────────┘
                         (같은 키를 안전하게 공유해야 함!)
```

📢 **섹션 요약 비유**: **기밀성**은 편지를 금고에 넣는 것이고, **무결성**은 편지 봉투의 봉인 씰이 뜯겼는지 확인하는 것이며, **가용성**은 우체국이 24시간 문을 열어두는 것입니다. **대칭키 암호**는 현관문 열쇠 하나를 집주인과 가족이 똑같이 나눠 갖는 방식입니다. **ECB 모드**는 모든 방의 열쇠 구멍이 똑같아 도둑이 패턴을 읽기 쉬운 것이고, **GCM 모드**는 문을 잠그는 동시에 누가 문을 땄는지 감지하는 센서까지 달린 스마트 도어락입니다.
