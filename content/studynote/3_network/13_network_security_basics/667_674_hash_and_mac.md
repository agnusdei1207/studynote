# 667-674. 해시 함수와 메시지 인증(MAC)

> **핵심 인사이트**: 데이터가 1비트라도 바뀌었는지 어떻게 알 수 있을까? 해시 함수는 데이터의 '지문'을 찍어 무결성을 증명한다. 여기에 비밀키까지 섞으면(HMAC), 지문의 주인까지 확인하는 강력한 출처 인증 수단이 된다.

---

## Ⅰ. 해시 함수 (Hash Function)
임의의 길이를 가진 데이터를 고정된 길이의 고유한 값(해시값)으로 변환하는 함수입니다.

* **특징**:
  - **일방향성**: 해시값으로 원래 데이터를 알아낼 수 없습니다.
  - **무결성 확인**: 원본이 조금만 바뀌어도 해시값이 완전히 달라집니다. (눈사태 효과)
  - **충돌 저항성**: 서로 다른 데이터가 같은 해시값을 가질 확률이 극도로 낮아야 합니다.
* **주요 알고리즘**:
  - **MD5 / SHA-1**: 현재는 취약점이 발견되어 사용을 금지합니다.
  - **SHA-2 (SHA-256/512)**: 현재 가장 널리 쓰이는 표준입니다.
  - **SHA-3**: 구조적으로 SHA-2와 달라 보안성이 보강된 차세대 알고리즘.

---

## Ⅱ. 패스워드 해시와 솔트 (Salt)
* **솔트**: 비밀번호를 해시하기 전, 앞에 붙이는 무작위 문자열입니다.
* **목적**: 해커가 미리 계산해둔 해시표인 **레인보우 테이블(Rainbow Table)** 공격을 무력화하기 위해 사용합니다.

---

## Ⅲ. 메시지 인증 코드 (MAC / HMAC)
해시 함수에 **비밀키**를 결합하여 데이터의 **무결성**과 **인증(출처 확인)**을 동시에 보장하는 기술입니다.

* **HMAC (Hash-based MAC)**: 
  - **원리**: `Hash(Key + Message)`
  - **작동**: 송신자가 키를 섞은 해시값을 보내면, 수신자도 똑같은 키를 넣어 해시를 계산해 봅니다. 두 값이 같으면 "내용이 안 변했고, 진짜 그 사람이 보낸 게 맞구나"라고 확신합니다.
* **용도**: IPsec, 금융 거래 데이터 검증 등.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[해시 함수의 기본 동작]

  [ 아주 긴 데이터 ] ───( SHA-256 )───> [ 256bit 고정값 ]
  [ 짧은 한 문장  ] ───( SHA-256 )───> [ 256bit 고정값 ]
                                          (지문 생성)

[HMAC의 구성]
  Hash ( [ Message ] + [ Secret Key ] ) = HMAC Tag
```

📢 **섹션 요약 비유**: **해시 함수**는 복잡한 책 한 권을 읽고 그 내용을 딱 한 줄로 요약하는 '독후감'과 같습니다. 책 내용이 한 글자만 바뀌어도 독후감 내용은 완전히 달라지죠. **솔트**는 흔한 독후감을 방지하기 위해 제목 뒤에 "나만의 비밀 코드"를 덧붙이는 것이고, **HMAC**은 독후감 끝에 미리 약속한 '비밀 도장'을 찍어 진짜 내가 쓴 것임을 증명하는 장치입니다.
