# 192-197. 오류 검출 기법 (Parity, Checksum, CRC)

> **핵심 인사이트**: 전송된 데이터에 에러가 있는지 찾아내는 방법은 수학 퍼즐과 같다. 1의 개수를 맞추는 단순한 패리티 검사부터, 덧셈을 이용한 체크섬, 그리고 나눗셈의 나머지를 이용해 연속적인 에러(Burst Error)까지 완벽하게 잡아내는 CRC 기술로 발전해 왔다.

---

## Ⅰ. 패리티 검사 (Parity Check)
가장 단순하고 저렴한 오류 검출 방식입니다. 데이터 끝에 딱 1비트(Parity Bit)만 추가합니다.

* **짝수 패리티 (Even Parity)**: 전체 데이터에서 `1`의 개수가 항상 **짝수**가 되도록 패리티 비트를 0이나 1로 설정합니다.
  - 데이터가 `1011` (1이 3개)라면 패리티 비트는 `1`을 붙여 총 `10111` (1이 4개)로 만듭니다.
* **홀수 패리티 (Odd Parity)**: 전체 `1`의 개수가 항상 **홀수**가 되도록 만듭니다.
* **한계**: 노이즈 때문에 2개의 비트가 동시에 뒤집혀버리면(예: 1011 $\rightarrow$ 0111), 전체 1의 개수는 짝수 개 그대로 유지되므로 **에러를 절대 찾아낼 수 없습니다.** (짝수 개의 에러 검출 불가)
* **이차원 패리티 (블록 패리티)**: 이 한계를 극복하기 위해 데이터를 2차원 행렬(가로 세로)로 배열하고 가로, 세로 양방향으로 패리티를 계산하여 덧붙이는 방식입니다. 이 경우 여러 비트의 에러를 잡을 수 있고, 1비트 에러는 위치를 찾아내어 직접 고칠(Correction) 수도 있습니다.

---

## Ⅱ. 검사합 (Checksum)
주로 네트워크 계층(IP 헤더)이나 전송 계층(TCP/UDP)에서 사용하는 소프트웨어적인 오류 검출 방식입니다.

* **작동 원리**: 
  1. 전송할 데이터를 16비트 단위로 모두 자릅니다.
  2. 자른 데이터들을 모두 **1의 보수 연산(1's Complement Arithmetic)**으로 더합니다. (덧셈 중 발생한 캐리(Carry)는 다시 더해줍니다.)
  3. 이 최종 합계의 반대값(1의 보수)을 헤더의 Checksum 필드에 꽂아 넣습니다.
  4. 수신자는 데이터와 Checksum 값을 모두 더해봅니다. 결과가 전부 `1111...1` (0) 이 나오면 에러가 없는 것으로 간주합니다.
* **특징**: 구현이 쉽고 속도가 빠르지만, 데이터의 순서가 뒤바뀌는 에러에는 취약합니다.

---

## Ⅲ. CRC (Cyclic Redundancy Check)
데이터 링크 계층(이더넷, Wi-Fi 등)에서 가장 널리 쓰이는 하드웨어 친화적이고 가장 강력한 오류 검출 기법입니다.

### 1. 다항식 연산 (Polynomial) 기반
* CRC는 덧셈이 아니라 **이진 나눗셈(Modulo-2 Division, XOR 연산)**을 사용합니다.
* 송신자와 수신자는 사전에 **생성 다항식(Generator Polynomial)** 이라는 특별한 나누는 값(제수)을 약속해 둡니다. (예: $x^3 + x + 1$ 이면 이진수로 `1011`)

### 2. 작동 원리
1. 송신자는 원본 데이터 끝에 생성 다항식 길이보다 1개 적은 수만큼 0을 붙입니다.
2. 이 데이터를 **생성 다항식으로 나눕니다.** (XOR 나눗셈)
3. 이때 나온 **나머지(FCS, Frame Check Sequence)**를 원본 데이터 끝에 붙여서 전송합니다.
4. 수신자는 받은 전체 데이터를 생성 다항식으로 나눠봅니다. **나머지가 0으로 딱 떨어지면 에러가 없는 것**이고, 0이 아니면 전송 중 에러가 발생한 것입니다.

### 3. CRC의 강력한 성능 (버스트 에러 검출)
* 번개나 스파크 등에 의해 수십 개의 비트가 연속적으로 깨지는 것을 **버스트 에러(Burst Error)**라고 합니다.
* 생성 다항식을 수학적으로 잘 설계하면(예: `CRC-32`), 32비트 이하 길이의 버스트 에러를 **100%** 검출해내며, 그보다 긴 에러도 99.999% 이상의 확률로 잡아냅니다.

## Ⅳ. 개념 맵 및 요약

```ascii
[오류 검출(Error Detection) 기법 요약]

                [오류 검출 기법]
               /      |       \
          패리티     체크섬      CRC
         (Parity) (Checksum)   (CRC)
             |        |          |
   원리: 1의 개수 맞춤   단순 덧셈    다항식 나눗셈(XOR)
   특징: 구현 초간단     IP/TCP용   이더넷 프레임 꼬리표(FCS)
   한계: 짝수개 에러     순서 바뀜    버스트 에러 완벽 차단!
        못 잡음        취약함     (가장 강력함)
```

📢 **섹션 요약 비유**: **패리티**는 상자에 담긴 사과가 항상 짝수 개가 되도록 끼워 넣는 규칙입니다. 2개를 동시에 훔쳐 가면 티가 안 납니다. **체크섬**은 영수증의 합계 금액입니다. 만약 500원짜리 두 개의 순서를 바꿔서 찍어도 합계는 같아 모를 수 있습니다. **CRC**는 아주 복잡한 비밀 소수(Prime number)로 나눈 나머지를 적어두는 암호학적 영수증입니다. 중간에 데이터가 어떻게 망가지든 나머지가 절대 0으로 떨어지지 않으므로 무조건 조작(에러)을 잡아냅니다.
