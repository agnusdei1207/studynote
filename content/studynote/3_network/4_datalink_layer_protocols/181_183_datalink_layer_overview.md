# 181-183. 데이터 링크 계층(Data Link Layer)의 역할과 하위 계층

> **핵심 인사이트**: 1계층(물리 계층)이 단순히 비트(0,1)를 전기 신호로 쏘아내는 '무식한 파이프'라면, 2계층(데이터 링크 계층)은 이 비트들을 의미 있는 덩어리(프레임)로 묶고 에러를 고치며, 누구에게 보낼지(MAC 주소)를 결정하는 '우체국' 역할을 한다.

---

## Ⅰ. 데이터 링크 계층의 4대 핵심 역할
노드 대 노드(Node-to-Node, 인접한 장비 간)의 신뢰성 있는 데이터 전송을 보장하기 위해 다음 네 가지 기능을 수행합니다.

1. **프레이밍 (Framing)**
   * 네트워크 계층에서 내려온 패킷을 감싸고, 앞뒤에 머리표(Header)와 꼬리표(Trailer)를 붙여 **프레임(Frame)**이라는 전송 단위를 만듭니다. 수신자는 이 꼬리표를 보고 데이터의 끝을 알아냅니다.
2. **물리적 주소 지정 (Addressing)**
   * 프레임 헤더에 출발지와 목적지의 물리적 주소(MAC Address)를 기록하여, 공유 매체에서 정확한 수신자를 찾아가게 합니다.
3. **흐름 제어 (Flow Control)**
   * 수신자가 처리할 수 있는 속도보다 송신자가 데이터를 너무 빨리 보내서 데이터가 넘쳐(Overflow) 유실되지 않도록, 송신 속도를 조절(Stop-and-Wait, Sliding Window 등)합니다.
4. **오류 제어 (Error Control)**
   * 물리 계층을 통과하며 발생한 신호의 왜곡이나 노이즈로 인한 비트 에러를 프레임 트레일러(FCS)를 통해 찾아내고(Error Detection), 필요시 재전송을 요청(ARQ)하여 고칩니다.

---

## Ⅱ. 데이터 링크 계층의 분할 (LLC와 MAC)
IEEE 802 위원회는 다양한 물리 매체(유선, 무선 등)를 하나로 묶기 위해, OSI 2계층을 다시 두 개의 부계층(Sub-layer)으로 쪼개어 정의했습니다.

### 1. LLC (Logical Link Control) 부계층 - IEEE 802.2
* **위치**: 데이터 링크 계층의 위쪽 절반. 네트워크 계층(IP)과 맞닿아 있습니다.
* **역할**: 하드웨어(물리 매체)가 유선(이더넷)이든 무선(Wi-Fi)이든 상관없이, 상위 계층(IP)에게 **동일한 논리적 인터페이스를 제공**하는 '소프트웨어적 접착제' 역할을 합니다.
* 다중화(Multiplexing)와 기본적인 흐름/오류 제어 기능을 담당합니다.

### 2. MAC (Media Access Control) 부계층 - IEEE 802.3 ~ 802.11
* **위치**: 데이터 링크 계층의 아래쪽 절반. 물리 계층(Physical Layer)과 직접 맞닿아 있습니다.
* **역할**: "이 물리적인 선(Media)을 언제, 누가 사용할 것인가?"를 조율하는 **신호등(Access Control)** 역할을 합니다.
* 각 매체의 특성에 따라 철저히 하드웨어(랜카드 칩셋) 종속적으로 동작합니다.
  - IEEE 802.3: 유선 이더넷의 MAC (CSMA/CD 사용)
  - IEEE 802.11: 무선 LAN의 MAC (CSMA/CA 사용)
* 고유의 하드웨어 주소인 MAC 주소(48비트)를 부여하고 식별합니다.

## Ⅲ. 개념 맵 및 요약

```ascii
[데이터 링크 계층의 계층적 구조 (IEEE 802 표준)]

[ 3계층 : Network Layer (IP, ICMP) ]
======================================================
[ 2계층 : Data Link Layer ]
  ┌────────────────────────────────────────────────┐
  │ LLC (Logical Link Control) - IEEE 802.2        │ -> 상위 계층과의 소프트웨어적 연결
  ├──────────────────────┬─────────────────────────┤
  │ MAC (Ethernet)       │ MAC (Wi-Fi)             │ -> 매체 접근 제어 및 물리 주소 (하드웨어 종속)
  │ IEEE 802.3           │ IEEE 802.11             │
  └──────────────────────┴─────────────────────────┘
======================================================
[ 1계층 : Physical Layer (꼬임쌍선, 광섬유, 전파) ]
```

📢 **섹션 요약 비유**: 택배 시스템에 비유하자면, 1계층은 그냥 도로와 트럭입니다. 2계층은 상자에 물건을 담고 테이프를 붙인 뒤(프레이밍), 찢어지진 않았는지 검사하고(오류 제어), 받는 사람 집 호수를 적는(MAC 주소) 택배 분류소입니다. 여기서 LLC는 전산망 접수처이고, MAC은 짐을 트럭에 싣기 위해 상하차 순서를 조율하는 현장 반장님입니다.
