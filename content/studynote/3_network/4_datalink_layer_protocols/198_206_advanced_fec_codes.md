# 198-206. 고급 오류 정정 기술 (FEC와 HARQ)

> **핵심 인사이트**: 에러를 발견하는 것을 넘어 "스스로 고치는(FEC)" 기술은 우주 통신에서 시작되어 현재 5G 초고속 무선망의 핵심이 되었다. 또한 현대의 통신은 에러 정정과 재전송을 합친 하이브리드 방식(HARQ)을 사용한다.

---

## Ⅰ. 기초 블록 에러 정정 코드
데이터를 일정한 크기의 블록 단위로 자른 뒤, 수학적인 연산을 통해 검사 비트를 추가하는 고전적인 FEC 방식입니다.

### 1. 해밍 코드 (Hamming Code)
* **특징**: 가장 유명하고 기본적인 에러 정정 코드입니다.
* **성능**: 1비트의 에러는 스스로 고칠 수 있고(1-bit Correction), 2비트의 에러는 찾아낼 수 있습니다(2-bit Detection).
* **용도**: 주로 컴퓨터 내부의 메모리(ECC RAM)에서 발생하는 미세한 오류를 잡는 데 사용됩니다. (네트워크 전송용으로는 너무 약함)

### 2. 리드-솔로몬 코드 (Reed-Solomon Code)
* **특징**: 여러 비트가 뭉텅이로 깨지는 **버스트 에러(Burst Error)**를 정정하는 데 매우 탁월한 수학적 코드입니다.
* **용도**: CD, DVD의 스크래치 복원, QR 코드의 얼룩 복원, 그리고 과거 ADSL이나 위성 방송 등에서 광범위하게 쓰였습니다.

### 3. BCH / 골레이 (Golay) 코드
* **BCH 코드**: 리드-솔로몬 코드의 수학적 기반이 되는 강력한 블록 에러 정정 코드입니다. 다수의 에러를 교정할 수 있습니다.
* **골레이 코드**: 1970년대 보이저(Voyager) 우주선이 목성과 토성 사진을 지구로 보낼 때 사용했던 전설적인 에러 정정 코드입니다.

---

## Ⅱ. 스트림 및 현대 통신의 에러 정정 코드
현대의 무선 통신(이동통신, Wi-Fi)은 블록 단위가 아니라 끊임없이 쏟아지는 연속된 데이터(Stream)의 에러를 빛의 속도로 고쳐내야 합니다.

### 1. 길쌈 코드 (Convolutional Code)
* **특징**: 블록 단위로 자르지 않고, 들어오는 데이터 스트림을 이전 데이터와 연속적으로 엮어서(Convolution) 잉여 비트를 만들어냅니다.
* **비터비 알고리즘 (Viterbi Algorithm)**: 수신 측에서 꼬여있는 길쌈 코드를 풀어서 원래 데이터를 가장 확률이 높은 값으로 추적하여 복원해 내는 천재적인 알고리즘입니다. (3G 통신까지 주로 사용)

### 2. 터보 코드 (Turbo Code)
* **특징**: 두 개의 길쌈 코드를 평행하게 엮어서 성능을 극대화한 코드로, 이론상 최대 한계치인 **샤논의 한계(Shannon Limit)**에 거의 근접한 완벽한 에러 정정 능력을 보여주었습니다.
* **용도**: 3G와 **4G LTE**의 핵심 데이터 전송 채널에 사용되었습니다.

### 3. LDPC (Low Density Parity Check)
* **특징**: 1960년대에 발명되었으나 당시엔 컴퓨터가 너무 느려 쓰지 못하다가, 최근 부활한 코드입니다. 터보 코드만큼 성능이 좋으면서도 **초고속 병렬 처리**가 가능합니다.
* **용도**: 10Gbps 이상을 처리해야 하는 **5G 이동통신(데이터 채널)**과 Wi-Fi 6의 핵심 표준 기술입니다.

### 4. 폴라 코드 (Polar Code)
* **특징**: 수학적으로 샤논 한계에 도달할 수 있음이 100% 증명된 인류 최초의 채널 코딩 기술입니다. 짧은 길이의 데이터 전송에서 오류율 0%에 가까운 신뢰성을 보여줍니다.
* **용도**: **5G 이동통신(제어 채널)**에서 명령어를 완벽하게 주고받기 위해 사용됩니다.

---

## Ⅲ. HARQ (Hybrid ARQ)
FEC(스스로 고치기)와 ARQ(다시 보내달라고 하기)의 장점만 섞은 궁극의 에러 제어 기술입니다. LTE와 5G에서 필수적으로 사용됩니다.

* **개념**: 일단 에러가 나면 먼저 FEC(터보/LDPC 코드)로 스스로 고쳐봅니다. 도저히 못 고칠 정도로 심하게 망가졌을 때만 ARQ(재전송 요청)를 보냅니다.
* **Chase Combining (체이스 컴바이닝)**: 에러가 나서 버려야 할 프레임을 안 버리고 킵해둡니다. 재전송된 프레임이 도착하면, 아까 버리려던 쓰레기 프레임과 합쳐서(Combine) 퍼즐을 맞춰 복원율을 극대화합니다.
* **IR (Incremental Redundancy)**: 재전송을 요청할 때, 똑같은 데이터를 또 받는 게 아니라 처음에는 안 줬던 '추가 힌트(Redundancy)'만 찔끔찔끔 더 받아서 기존 데이터와 합쳐서 복원하는 고효율 방식입니다.

## Ⅳ. 개념 맵 및 요약

```ascii
[오류 정정 코드(FEC)의 세대별 진화]

과거 (근거리/저속)  : 해밍 코드 (1비트 복원)
      ↓
중기 (버스트 에러) : 리드 솔로몬 코드 (CD, QR코드)
      ↓
3G / 4G (LTE)      : 터보 코드 (샤논 한계 근접)
      ↓
5G / Wi-Fi 6       : LDPC (데이터 채널 - 병렬 초고속 처리)
                     폴라 코드 (제어 채널 - 100% 신뢰성)

[HARQ 기술 개념]
수신: (망가진 데이터 도착) -> FEC로 복구 시도 -> 실패! 
      -> (안 버리고 메모리에 저장) -> NAK 전송
재수신: (추가 힌트 IR 도착) -> 아까 저장한 쓰레기 데이터와 합체! -> 100% 복원 완료!
```

📢 **섹션 요약 비유**: 해밍 코드가 "스펠링 하나" 틀린 것을 문맥상 눈치껏 고치는 수준이라면, 리드 솔로몬 코드는 "물감이 엎질러져 지워진 한 줄"을 복원하는 기술입니다. 현대 5G의 LDPC와 폴라 코드는 초당 수십억 장의 훼손된 문서를 여러 명이 동시에 투입되어 순식간에 복원해 내는 초정밀 AI 번역기 팀과 같습니다.
