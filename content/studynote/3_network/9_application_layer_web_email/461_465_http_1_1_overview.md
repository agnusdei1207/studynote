# 461-465. HTTP 프로토콜의 진화 (HTTP/1.0 to 1.1)

> **핵심 인사이트**: HTTP는 '무심한' 프로토콜이다. 상대방이 누구인지, 이전에 무슨 대화를 했는지 기억하지 않는(Stateless) 성질 덕분에 단순하고 확장성이 좋았으나, 웹이 복잡해짐에 따라 매번 새로 통성명(TCP 연결)을 해야 하는 비효율을 극복하기 위해 '지속 연결' 기술을 도입하며 발전해 왔다.

---

## Ⅰ. HTTP (HyperText Transfer Protocol)의 기본 철학
웹 브라우저(클라이언트)와 웹 서버 간에 HTML 문서나 멀티미디어 파일을 주고받기 위한 통신 규약입니다.

1. **Stateless (상태 비저장)**: 서버는 클라이언트의 이전 요청을 기억하지 않습니다. (이를 해결하기 위해 쿠키와 세션이 등장함)
2. **Request / Response 구조**: 클라이언트가 먼저 말을 걸어야(요청) 서버가 대답(응답)하는 수동적인 구조입니다.

---

## Ⅱ. HTTP 메서드 (Method)
서버에 무엇을 해달라고 할지 결정하는 동사입니다.
* **GET**: 서버로부터 자원을 **가져오기** (조회)
* **POST**: 서버에 데이터를 전달하여 **새로운 자원 생성** (글쓰기)
* **PUT**: 자원을 통째로 **교체**
* **PATCH**: 자원의 **일부만 수정**
* **DELETE**: 자원 **삭제**
* **OPTIONS**: 서버가 지원하는 메서드 목록 확인 (CORS 등에서 활용)

---

## Ⅲ. HTTP/1.0 vs HTTP/1.1 발전사

### 1. HTTP/1.0 (비지속 연결)
* **방식**: 이미지 하나를 받을 때마다 **매번 3-Way Handshake**를 통해 TCP 연결을 새로 맺고 끊습니다.
* **문제점**: 웹 페이지에 사진이 100장 있으면 100번의 연결 과정이 필요하여 네트워크 오버헤드가 극심하고 매우 느립니다.

### 2. HTTP/1.1 (지속 연결 및 효율성 개선)
1997년에 등장하여 여전히 가장 많이 쓰이는 표준입니다.
* **지속 연결 (Persistent Connection)**: 한 번 맺은 TCP 연결을 끊지 않고 **Keep-Alive** 헤더를 통해 재사용합니다. 이미지 100장을 한 번의 통성명으로 다 받아올 수 있습니다.
* **파이프라이닝 (Pipelining)**: 첫 번째 요청의 응답이 오기 전에 두 번째, 세 번째 요청을 연속해서 보내는 기술입니다.

---

## Ⅳ. HTTP/1.1의 고질적 문제: HOL (Head-of-Line) 블로킹
* **현상**: 파이프라이닝을 통해 요청을 3개 보냈을 때, 서버는 반드시 **요청받은 순서대로 응답**해야 합니다. 만약 1번 요청 처리가 서버에서 너무 오래 걸리면, 이미 준비가 끝난 2번, 3번 응답도 1번 때문에 줄줄이 대기하며 전송되지 못하는 병목 현상이 발생합니다.
* **결과**: 이 문제 때문에 HTTP/2라는 새로운 패러다임이 등장하게 됩니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[HTTP 연결 방식 비교]

1. HTTP/1.0 (Short-lived)
   [ Client ] --- SYN ---> [ Server ]
   [ Client ] <--- ACK --- [ Server ]
   [ Client ] --- GET ---> [ Server ] (데이터 전송)
   [ Client ] <--- Res --- [ Server ] (연결 즉시 종료!)
   (이미지 하나 더 받으려면 위 과정 반복)

2. HTTP/1.1 (Persistent)
   [ Client ] --- SYN ---> [ Server ] (한 번만 연결)
   [ Client ] --- GET 1 --> [ Server ]
   [ Client ] <--- Res 1 -- [ Server ]
   [ Client ] --- GET 2 --> [ Server ]
   [ Client ] <--- Res 2 -- [ Server ] (연결 유지)
```

📢 **섹션 요약 비유**: **HTTP/1.0**은 상점에 가서 사과 하나 사고 집에 갔다가, 다시 상점에 가서 포도 하나 사고 집에 오는 비효율적인 쇼핑입니다. **HTTP/1.1**은 바구니(지속 연결)를 들고 가서 사과, 포도를 한 번에 골라 담는 방식입니다. 하지만 **HOL 블로킹**은 계산대 줄(응답 순서)이 하나뿐이라, 앞사람이 환불 문제로 직원을 붙잡고 있으면 내 물건은 이미 가방에 다 담았는데도 나가지 못하고 계속 기다려야 하는 상황과 같습니다.
