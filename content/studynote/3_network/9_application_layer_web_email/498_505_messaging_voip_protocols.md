# 498-505. 실시간 메시징 및 VoIP 프로토콜

> **핵심 인사이트**: 인터넷은 정적인 문서 전달을 넘어 '실시간 대화'의 장으로 진화했다. 서버가 먼저 알려주는 웹훅(Webhook)부터, 전 세계 메신저의 표준 언어인 XMPP, 그리고 전화를 데이터 패킷으로 바꾸는 SIP와 브라우저만으로 화상회의가 가능한 WebRTC가 실시간 세상을 지탱한다.

---

## Ⅰ. 실시간 알림 기술: 웹훅 (Webhook)
* **개념**: 서버에서 어떤 이벤트가 발생했을 때, 미리 지정된 URL로 HTTP POST 요청을 보내어 실시간으로 알리는 기술입니다.
* **비유**: "다 되면 전화해"라고 번호를 남겨두고 오는 '콜백(Callback)' 서비스와 같습니다.

---

## Ⅱ. 메시징 프로토콜: XMPP
* **개념**: XML을 기반으로 하는 개방형 실시간 메시징 및 존재 알림(Presence) 프로토콜입니다.
* **특징**: 표준이 공개되어 있어 서로 다른 회사 메신저끼리도 연동이 가능합니다. (구글 토크, 초기 카카오톡 등에서 활용)

---

## Ⅲ. 인터넷 전화 (VoIP) 핵심 프로토콜

### 1. SIP (Session Initiation Protocol)
* **역할**: 전화 연결을 **시작하고, 수정하고, 종료**하는 "신호(Signaling)" 담당입니다.
* **특징**: 텍스트 기반이며 HTTP와 구조가 비슷해 확장성이 좋습니다. 실제 음성 데이터는 전송하지 않고, 오직 '전화 연결'만 관리합니다.

### 2. H.323
* **개념**: ITU-T에서 만든 좀 더 복잡하고 엄격한 화상회의 표준입니다. 바이너리 기반이라 무겁고 수정이 어려워 최근에는 SIP에 밀리는 추세입니다.

---

## Ⅳ. 웹 기반 실시간 통신: WebRTC
* **개념**: 웹 브라우저 간에 별도의 플러그인(ActiveX 등) 없이 **P2P로 음성, 영상, 데이터를 주고받는** 오픈 소스 프로젝트입니다.
* **핵심 기술**:
  - **ICE / STUN / TURN**: 방화벽(NAT)을 뚫고 상대방과 직접 연결하기 위한 기술.
  - **SDP**: 미디어 정보(코덱, 해상도 등)를 교환하는 규격.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[VoIP 통신 전체 구조]

  [ 송신자 ] ───── (SIP: "전화 받아!") ─────> [ 수신자 ]
      │                                          │
      └────── (RTP: 실제 음성/영상 데이터) ──────┘
```

📢 **섹션 요약 비유**: **웹훅**은 배달 앱에서 "주문하신 음식이 출발했습니다"라고 폰으로 알림을 쏴주는 고마운 비서입니다. **SIP**는 전화기 버튼을 눌러 벨을 울리게 하는 '교환원'이고, **RTP**는 연결된 전화선을 통해 실제로 오가는 '목소리'입니다. **WebRTC**는 복잡한 설치 과정 없이 인터넷 창만 열면 바로 서로 얼굴을 보며 대화할 수 있는 '투명한 유리창'과 같습니다.
