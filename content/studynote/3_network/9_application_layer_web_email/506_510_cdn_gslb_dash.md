# 506-510. 콘텐츠 전송 및 스트리밍 기술 (CDN, GSLB, DASH)

> **핵심 인사이트**: 전 세계 모든 사용자가 미국에 있는 단 하나의 서버에 접속하면 인터넷은 마비될 것이다. 이를 막기 위해 콘텐츠를 전 세계 집 근처 창고(CDN)에 미리 가져다 놓고, 교통 상황에 맞춰 가장 가까운 길로 안내하며(GSLB), 사용자의 인터넷 속도에 맞춰 화질을 자동으로 조절하는(DASH) 기술이 현대 미디어 환경을 지탱한다.

---

## Ⅰ. CDN (Content Delivery Network)
지리적으로 분산된 서버 네트워크를 구축하여 콘텐츠를 사용자에게 더 빠르고 안정적으로 전달하는 기술입니다.

* **원리**: 원본 서버(Origin)의 데이터를 전 세계 거점(PoP)에 있는 **엣지 서버(Edge Server)**에 복사(Caching)해 둡니다.
* **장점**: 응답 시간(Latency) 단축, 원본 서버 부하 분산, 대규모 트래픽 처리 가능.

---

## Ⅱ. GSLB (Global Server Load Balancing)
전 세계 여러 데이터센터에 분산된 서버들 중, 현재 사용자가 접속하기에 **가장 최적의 서버 IP를 DNS 수준에서 결정**해 주는 지능형 부하 분산 기술입니다.

* **판단 기준**: 
  - 사용자의 지리적 위치 (IP 기반)
  - 서버의 현재 상태 (Health Check)
  - 네트워크 지연 시간 및 홉(Hop) 수
* **차이점**: 일반 L4/L7 스위치가 한 데이터센터 내부의 부하를 분산한다면, GSLB는 **데이터센터 간**의 부하를 분산합니다.

---

## Ⅲ. 적응형 비디오 스트리밍 (DASH)
* **DASH (Dynamic Adaptive Streaming over HTTP)**:
  - 사용자의 **네트워크 대역폭 상황에 맞춰 영상 화질을 실시간으로 변경**하며 전송하는 기술입니다. (유튜브, 넷플릭스의 핵심 기술)
  - **원리**: 영상을 아주 짧은 조각(Segment)으로 나누고, 각 조각을 다양한 화질(4K, 1080p, 720p 등)로 미리 만들어둡니다. 플레이어는 속도가 빠르면 4K 조각을, 느려지면 720p 조각을 요청합니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[CDN 및 GSLB 동작 흐름]

1. [ User ] ── "google.com 어디야?" ──> [ GSLB ]
2. [ GSLB ] ── "너 한국이네? 서울 노드 가!" ──> [ User ]
3. [ User ] ─── (콘텐츠 요청) ───> [ 서울 엣지 서버 (CDN) ]
4. [ 서울 엣지 ] ── (캐싱된 데이터 전송) ──> [ User ]
```

📢 **섹션 요약 비유**: **CDN**은 미국에 본사가 있는 프랜차이즈 햄버거집이 우리 동네에 '체인점'을 내는 것과 같습니다. **GSLB**는 배달 앱이 내 위치를 파악해서 가장 배달비가 싸고 가까운 체인점을 자동으로 골라주는 시스템입니다. **DASH**는 유튜브를 볼 때 지하철 터널에 들어가서 인터넷이 느려지면, 영상이 멈추는 대신 화질을 조금 낮춰서라도 끊김 없이 보여주는 '눈치 빠른 전송 요령'입니다.
