# 482-486. 파일 전송 프로토콜: FTP와 변종들

> **핵심 인사이트**: FTP는 데이터를 보낼 때 '제어'용 통로와 '데이터'용 통로 두 개를 따로 쓰는 독특한 프로토콜이다. 하지만 이 독특함 때문에 방화벽과 자주 충돌하며(Active vs Passive), 보안 기능이 전혀 없어 현대에는 암호화된 SFTP가 그 자리를 대신하고 있다.

---

## Ⅰ. FTP (File Transfer Protocol) 개요
인터넷에서 파일을 주고받기 위한 고전적인 프로토콜입니다.

* **두 개의 포트 사용**:
  - **포트 21 (Control)**: 명령어를 주고받는 제어 채널. (로그인, 파일 목록 조회 등)
  - **포트 20 (Data)**: 실제 파일 데이터가 오가는 데이터 채널.

---

## Ⅱ. FTP의 두 가지 동작 모드

### 1. 액티브 모드 (Active Mode)
* **방식**: 클라이언트가 서버에 접속하면, **서버가 클라이언트에게** 데이터를 보내기 위해 역으로 접속을 시도하는 방식입니다.
* **문제점**: 클라이언트 쪽에 방화벽(공유기 등)이 있으면, 서버가 들어오는 요청을 차단해 버려 전송이 실패합니다.

### 2. 패시브 모드 (Passive Mode)
* **방식**: 서버가 데이터용 임시 포트를 열어두고 클라이언트에게 알려주면, **클라이언트가 서버로** 접속하는 방식입니다.
* **특징**: 대부분의 웹 브라우저와 현대 클라이언트는 방화벽 문제를 피하기 위해 이 방식을 기본으로 사용합니다.

---

## Ⅲ. 보안이 강화된 파일 전송

### 1. TFTP (Trivial FTP)
* **특징**: UDP 포트 **69**번 사용. 확인 절차 없이 아주 가볍게 동작합니다.
* **용도**: 라우터 부팅 시 설정 파일을 내려받는 등 아주 단순한 용도에만 씁니다.

### 2. FTPS (FTP over SSL/TLS)
* **특징**: 기존 FTP에 SSL 암호화를 입힌 것입니다. 인증서가 필요합니다.

### 3. SFTP (SSH File Transfer Protocol)
* **특징**: FTP와 이름은 비슷하지만 아예 다른 프로토콜입니다. **SSH(포트 22)** 통로를 통해 파일을 안전하게 전송합니다.
* **장점**: 포트 하나만 쓰므로 방화벽 설정이 쉽고, 보안성이 매우 강력하여 현대 실무의 표준입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[Active vs Passive FTP 흐름]

1. Active Mode: "서버가 나(클라이언트)에게 접속"
   Client (포트 21) ───> Server (제어)
   Client (차단!) <─── Server (포트 20, 데이터 전송 시도)

2. Passive Mode: "내가 서버에게 접속"
   Client (포트 21) ───> Server (제어)
   Client ──(나 갈게!)──> Server (임시 포트, 데이터)
```

📢 **섹션 요약 비유**: **FTP**는 '택배 배달'입니다. **액티브 모드**는 택배 기사가 집 벨을 누르고 들어오려는데 아파트 경비실(방화벽)에서 모르는 사람이라고 막는 상황입니다. **패시브 모드**는 기사가 "무인 택배함 105호에 넣어둘게"라고 하면 주인이 직접 가서 찾아오는 방식입니다. **SFTP**는 믿을 수 있는 보안 요원(SSH)이 튼튼한 금고 가방에 넣어 물건을 직접 가져다주는 가장 안전한 배달 방식입니다.
