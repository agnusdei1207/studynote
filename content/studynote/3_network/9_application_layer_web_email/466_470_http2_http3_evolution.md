# 466-470. 차세대 웹 표준: HTTP/2와 HTTP/3

> **핵심 인사이트**: 웹의 성능을 갉아먹던 순차적 통신(HOL Blocking)을 해결하기 위해 HTTP/2는 하나의 통로에 여러 데이터를 섞어 보내는 '멀티플렉싱'을 도입했고, HTTP/3는 아예 TCP를 버리고 UDP 기반의 QUIC 프로토콜로 갈아타며 무선 환경에서의 끊김 없는 통신을 완성했다.

---

## Ⅰ. HTTP/2 (성능의 극대화)
2015년에 승인된 표준으로, 구글의 SPDY 프로토콜을 기반으로 합니다.

### 1. 주요 특징
* **바이너리 프레이밍 (Binary Framing)**: 텍스트 대신 0과 1의 바이너리 데이터로 패킷을 쪼개어 보냅니다. 파싱이 훨씬 빨라지고 오류가 적습니다.
* **멀티플렉싱 (Multiplexing)**: **가장 핵심적인 혁신**입니다. 하나의 TCP 연결 안에 여러 개의 **스트림(Stream)**을 만들어, 여러 요청과 응답을 순서에 상관없이 동시에 주고받습니다. (HTTP HOL 블로킹 완벽 해결)
* **헤더 압축 (HPACK)**: 중복되는 헤더 정보를 압축하여 전송량을 획기적으로 줄입니다.
* **서버 푸시 (Server Push)**: 클라이언트가 요청하지 않아도, 서버가 필요할 것 같은 자원(CSS, JS 등)을 미리 보내줍니다.

---

## Ⅱ. HTTP/3 (속도와 이동성의 완성)
2022년에 표준화된 최신 규격으로, TCP 대신 **QUIC(Quick UDP Internet Connections)** 프로토콜을 사용합니다.

### 1. 주요 특징
* **UDP 기반 (QUIC)**: TCP의 느린 핸드셰이크 과정을 생략하고, 연결 설정과 암호화 설정을 한 번에 끝냅니다. (0-RTT/1-RTT)
* **TCP HOL 블로킹 해결**: HTTP/2는 하나의 패킷이 유실되면 전체 TCP 연결이 멈췄지만, HTTP/3는 각 스트림이 독립적이라 패킷 하나가 사라져도 다른 데이터는 멈추지 않고 계속 흐릅니다.
* **연결 마이그레이션**: IP가 바뀌어도(Wi-Fi $\rightarrow$ LTE) 연결이 끊기지 않고 세션이 유지됩니다.

---

## Ⅲ. HTTP 버전별 비교 요약

| 구분 | HTTP/1.1 | HTTP/2 | HTTP/3 |
| :--- | :--- | :--- | :--- |
| **전송 프로토콜** | TCP | TCP | **UDP (QUIC)** |
| **데이터 형식** | 텍스트 (Plain Text) | **바이너리 (Binary)** | **바이너리 (Binary)** |
| **통신 방식** | 순차적 (Pipelining) | **동시적 (Multiplexing)** | **동시적 (Multiplexing)** |
| **보안 (TLS)** | 선택 사항 | 거의 필수 (브라우저 제약) | **기본 내장 (필수)** |
| **주요 개선** | 지속 연결 | **HTTP HOL 해결**, 압축 | **TCP HOL 해결**, 빠른 연결 |

---

## Ⅳ. 개념 맵 및 요약

```ascii
[웹 통신 방식의 진화]

HTTP/1.1 : [ GET 1 ] [ Res 1 ] [ GET 2 ] [ Res 2 ]  (줄서기, 앞이 막히면 다 막힘)

HTTP/2   : [ Stream 1: GET ] [ Stream 2: GET ]      (한 통로에 섞어 보내기)
           [ Stream 2: Res ] [ Stream 1: Res ]      (먼저 준비된 놈부터 대답)

HTTP/3   : (QUIC) [ Str 1 ] [ Str 2 ] [ Str 3 ]     (통로 자체를 여러 개로 쪼개기)
           -> 1번 통로(Packet) 사고 나도 2, 3번은 쌩쌩 달림
```

📢 **섹션 요약 비유**: **HTTP/1.1**은 계산대 줄이 하나인 마트입니다. 앞사람이 오래 걸리면 뒷사람은 무조건 기다려야 합니다. **HTTP/2**는 계산대 직원이 1명인데 손이 10개라서 여러 손님의 물건을 동시에 바코드로 찍는 신공을 발휘하는 것입니다. **HTTP/3**는 마트 입구부터 계산대까지 손님마다 전용 무빙워크(UDP 스트림)를 깔아주고, 옆 손님이 넘어져도 내 길은 방해받지 않고 광속으로 지나가는 최첨단 미래형 마트입니다.
