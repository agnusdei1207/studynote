# Packet Scheduling - 패킷 스케줄링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 순서 결정
> 2. **가치**: QoS 보장
> 3. **융합:** FIFO, WFQ, Priority Queuing

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Packet Scheduling은 패킷 전송 순서를 결정합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Packet Scheduling                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스케줄링 알고리즘                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  FIFO (First-In First-Out):                           │   │ │
│   │   │  ┌───┬───┬───┬───┐                                  │   │ │
│   │   │  │ 1 │ 2 │ 3 │ 4 │ → 전송                           │   │ │
│   │   │  └───┴───┴───┴───┘                                  │   │ │
│   │   │  단순, 공정하지 않음                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Priority Queuing (PQ):                               │   │ │
│   │   │  ┌───────┐ High                                       │   │ │
│   │   │  │ H1 H2 │ → 우선 전송                                │   │ │
│   │   │  └───────┘                                            │   │ │
│   │   │  ┌───────┐ Medium                                     │   │ │
│   │   │  │ M1 M2 │ → High 비면                                │   │ │
│   │   │  └───────┘                                            │   │ │
│   │   │  ┌───────┐ Low                                        │   │ │
│   │   │  │ L1 L2 │ → 기아 가능                                │   │ │
│   │   │  └───────┘                                            │   │ │
│   │   │                                                      │   │ │
│   │   │  WFQ (Weighted Fair Queuing):                         │   │ │
│   │   │  ┌─────┐  ┌─────┐  ┌─────┐                           │   │ │
│   │   │  │Q1 50%│  │Q2 30%│  │Q3 20%│                        │   │ │
│   │   │  └──┬──┘  └──┬──┘  └──┬──┘                           │   │ │
│   │   │     └────────┴────────┘                              │   │ │
│   │   │            ↓                                          │   │ │
│   │   │         전송                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 패킷 스케줄링은 QoS 구현의 핵심입니다.

**📢 섹션 요약 비유:** Packet Scheduling = 교통 정리!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **FIFO** | 선입선출 |
| **PQ** | 우선순위 큐 |
| **WFQ** | 가중 공정 큐 |
| **CBQ** | 클래스 기반 |
| **RED** | Random Early Detection |

**핵심 알고리즸:** 분류 → 큐 할당 → 스케줄링 → 전송

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**스케줄링 알고리즘 비교**

| 알고리즘 | 공정성 | 복잡도 | QoS |
|:---|:---|:---|:---|
| **FIFO** | 없음 | 낮음 | 없음 |
| **PQ** | 없음 | 낮음 | 있음 |
| **WFQ** | 있음 | 높음 | 있음 |
| **DRR** | 있음 | 중간 | 있음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일반 → FIFO
**시나리오 2:** 음성/영상 → PQ
**시나리오 3:** ISP → WFQ
