# Paging - 페이징

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 고정 크기 분할
> 2. **가치**: 단편화 감소, 유연성
> 3. **융합:** Virtual Memory, Page Table, TLB

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Paging은 메모리를 고정 크기 블록으로 분할합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Paging                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              페이징 구조                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Virtual Memory          Physical Memory            │   │ │
│   │   │  ┌─────┬─────┬─────┐    ┌─────┬─────┬─────┐         │   │ │
│   │   │  │Page │Page │Page │    │Frame│Frame│Frame│         │   │ │
│   │   │  │  0  │  1  │  2  │    │  0  │  1  │  2  │         │   │ │
│   │   │  └──┬──┴──┬──┴──┬──┘    └─────┴─────┴─────┘         │   │ │
│   │   │     │     │     │                                   │   │ │
│   │   │     │     └─────┼──────────────────→ Frame 1         │   │ │
│   │   │     │           │                                   │   │ │
│   │   │     └───────────┼──────────────────→ Frame 0         │   │ │
│   │   │                 │                                   │   │ │
│   │   │                 └──────────────────→ Frame 2         │   │ │
│   │   │                                                      │   │ │
│   │   │  Page Table: Page → Frame 매핑                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 페이징은 가변 분할의 단편화 문제를 해결합니다.

**📢 섹션 요약 비유:** Paging = 책 페이지!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Page** | 가상 메모리 블록 |
| **Frame** | 물리 메모리 블록 |
| **Page Table** | 매핑 테이블 |
| **Page Fault** | 페이지 부재 |
| **Page Size** | 페이지 크기 (4KB 등) |

**핵심 알고리즸:** 가상 주소 → (Page#, Offset) → Page Table → Frame#

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Paging vs Segmentation**

| 항목 | Paging | Segmentation |
|:---|:---|:---|
| **크기** | 고정 | 가변 |
| **단편화** | 내부 | 외부 |
| **논리성** | 없음 | 있음 |
| **복잡도** | 낮음 | 높음 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일반 OS → Paging
**시나리오 2:** 메모리 보호 → Paging + Protection Bits
**시나리오 3:** 대용량 → Huge Pages
