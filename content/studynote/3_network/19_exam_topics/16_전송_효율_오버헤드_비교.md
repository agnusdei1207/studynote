# 전송 효율 (오버헤드 비교)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실제 데이터 대비 제어 정보의 비율
> 2. **가치**: 대역폭 활용률, 전송 속도, 비용 결정
> 3. **융합**: 프로토콜 선택, 설계 최적화와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

전송 효율(Transmission Efficiency)은 전체 전송 비트 중 실제 데이터가 차지하는 비율입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    전송 효율 (오버헤드 비교)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              전송 효율 계산                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  전송 효율 = (데이터 비트 / 전체 비트) × 100%          │   │ │
│   │   │                                                      │   │ │
│   │   │  오버헤드 = 전체 비트 - 데이터 비트                    │   │ │
│   │   │                                                      │   │ │
│   │   │  예: 8비트 데이터, 2비트 시작/정지                     │   │ │
│   │   │  효율 = 8 / (1 + 8 + 1) = 8/10 = 80%                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              전송 방식별 오버헤드 비교                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  비동기 전송 (8N1):                                    │   │ │
│   │   │  Start(1) + Data(8) + Stop(1) = 10비트               │   │ │
│   │   │  효율: 8/10 = 80%                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  비동기 전송 (7E1):                                    │   │ │
│   │   │  Start(1) + Data(7) + Parity(1) + Stop(1) = 10비트   │   │ │
│   │   │  효율: 7/10 = 70%                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  동기 전송 (1000바이트 블록):                           │   │ │
│   │   │  SYN(2) + STX(1) + Data(1000) + ETX(1) + BCC(2)      │   │ │
│   │   │  효율: 1000/1006 = 99.4%                              │   │ │
│   │   │                                                      │   │ │
│   │   │  Ethernet 프레임 (1500바이트):                          │   │ │
│   │   │  Header(14) + Data(1500) + FCS(4) + IFG(12)           │   │ │
│   │   │  효율: 1500/1530 = 98%                                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 동기 전송과 긴 블록이 높은 전송 효율을 보입니다.

**💡 비유**: 전송 효율은 트럭의 화물 대 연료 비율입니다.

**📢 섹션 요약 비유**: 블록이 길수록 효율!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Header** | 제어 | 주소, 제어 | 송장 |
| **Payload** | 데이터 | 실제 정보 | 화물 |
| **Trailer** | 검사 | FCS, CRC | 확인서 |
| **Gap** | 여유 | IFG, Idle | 대기 |

**오버헤드 유형**

| 유형 | 예시 | 영향 |
|:---|:---|:---:|
| **프레이밍** | Start/Stop | 높음 |
| **헤더** | 주소 | 중간 |
| **오류검출** | CRC | 낮음 |
| **프로토콜** | ACK | 낮음 |

**핵심 알고리즘**: 데이터 크기 증가 → 오버헤드 비율 감소 → 효율 증가

**📢 섹션 요약 비유**: Jumbo Frame이 99%!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 프로토콜 효율**

| 프로토콜 | 최대 효율 |
|:---|:---:|
| **비동기** | ~80% |
| **HDLC** | ~98% |
| **Ethernet** | ~98% |
| **Jumbo Frame** | ~99% |

**📢 섹션 요약 비유**: Jumbo Frame이 최고!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 저속** → 비동기, 짧은 프레임
**시나리오 2: 고속 LAN** → Ethernet, 큰 프레임
**시나리오 3: WAN** → 압축, 효율화

**도입 체크리스트**: 데이터 크기, 오류율, 대역폭

**📢 섹션 요약 비유**: 효율과 신뢰성의 균형!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대역폭 절약, 비용 감소
