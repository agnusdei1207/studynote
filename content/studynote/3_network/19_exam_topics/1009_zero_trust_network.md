# Zero Trust Network - 제로 트러스트 네트워크

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 신뢰하지 않는 보안 모델
> 2. **가치**: 내부 위협 방지
> 3. **융합:** Microsegmentation, IAM, MFA

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Zero Trust Network는 모든 접근을 검증하는 보안 모델입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Zero Trust Network                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Zero Trust 원칙                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  핵심 원칙:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  1. Never Trust (절대 신뢰하지 않음)               ││   │ │
│   │   │  │     - 내부/외부 구분 없음                          ││   │ │
│   │   │  │     - 모든 접근 검증                              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  2. Always Verify (항상 검증)                     ││   │ │
│   │   │  │     - 사용자 인증                                 ││   │ │
│   │   │  │     - 디바이스 인증                               ││   │ │
│   │   │  │     - 컨텍스트 검증                               ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  3. Least Privilege (최소 권한)                   ││   │ │
│   │   │  │     - 필요한 만큼만 접근                           ││   │ │
│   │   │  │     - JIT (Just-In-Time) 권한                     ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  구성 요소:                                        ││   │ │
│   │   │  │  - IAM (Identity & Access Management)             ││   │ │
│   │   │  │  - MFA (Multi-Factor Authentication)              ││   │ │
│   │   │  │  - Microsegmentation                              ││   │ │
│   │   │  │  - Policy Engine                                  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Zero Trust는 경계 기반 보안의 한계를 극복합니다.

**📢 섹션 요약 비유:** Zero Trust = 의심 많은 문지기!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Policy Engine** | 정책 엔진 |
| **Policy Admin** | 정책 관리 |
| **PEP** | Policy Enforcement Point |
| **IAM** | ID 및 접근 관리 |
| **Trust Algorithm** | 신뢰 알고리즘 |

**핵심 알고리듬:** 요청 → 인증 → 컨텍스트 평가 → 정책 적용 → 접근

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Zero Trust vs 전통적 보안**

| 항목 | Zero Trust | 전통적 |
|:---|:---|:---|
| **신뢰** | 없음 | 내부 신뢰 |
| **경계** | 없음 | 명확 |
| **접근** | 검증됨 | 신뢰됨 |
