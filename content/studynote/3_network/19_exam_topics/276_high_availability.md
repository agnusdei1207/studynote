# 고가용성 (High Availability) - HA

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서비스 중단 최소화
> 2. **가치**: 99.99% 가용성
> 3. **융합**: 클러스터링, 이중화와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

고가용성(HA)은 시스템이 지속적으로 운영될 수 있도록 하는 설계 원칙입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    고가용성 (High Availability)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HA 아키텍처                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Load Balancer]                                    │   │ │
│   │   │       │          │                                   │   │ │
│   │   │       ↓          ↓                                   │   │ │
│   │   │  ┌──────────┐ ┌──────────┐                          │   │ │
│   │   │  │ Server 1 │ │ Server 2 │  ← Active-Active         │   │ │
│   │   │  │ (Active) │ │ (Active) │                          │   │ │
│   │   │  └──────────┘ └──────────┘                          │   │ │
│   │   │       │          │                                   │   │ │
│   │   │       ↓          ↓                                   │   │ │
│   │   │  ┌─────────────────────────────┐                    │   │ │
│   │   │  │      Shared Storage         │                    │   │ │
│   │   │  │   (Replicated/Clustered)    │                    │   │ │
│   │   │  └─────────────────────────────┘                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HA 구성 방식                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Active-Active:                                   │   │ │
│   │   │     • 모든 노드가 서비스                              │   │ │
│   │   │     • 부하 분산                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Active-Standby:                                  │   │ │
│   │   │     • 하나만 활성, 나머지 대기                        │   │ │
│   │   │     • 자원 낭비 있음                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  3. N+1:                                             │   │ │
│   │   │     • N개 활성, 1개 예비                             │   │ │
│   │   │     • 비용 효율                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HA는 이중화와 페일오버로 서비스 연속성을 보장합니다.

**💡 비유**: HA는 비상발전기 시스템입니다. 정전이 되면 즉시 전환됩니다.

**📢 섹션 요약 비유:** HA = 99.99% 가용성!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **이중화** | 복제 |
| **페일오버** | 자동 전환 |
| **헬스체크** | 상태 모니터링 |
| **쿼럼** | 합의 |

**비교표: 가용성 등급**

```
+------------+----------+----------+
| 등급       │ 가용성   │ 연간 중단│
+------------+----------+----------+
| 99%        │ 2 nines  │ 3.65일   │
| 99.9%      │ 3 nines  │ 8.76시간 │
| 99.99%     │ 4 nines  │ 52.6분   │
| 99.999%    │ 5 nines  │ 5.26분   │
+------------+----------+----------+
```

**핵심 알고리즸:** 하트비트, 페일오버

**📢 섹션 요약 비유:** 5 nines = 5분 중단!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 데이터베이스** → Active-Standby 복제
**시나리오 2: 웹 서버** → Active-Active + LB
**시나리오 3: 스토리지** → RAID + 복제

**도입 체크리스트:** RTO, RPO, 비용

**📢 섹션 요약 비유:** 비용 vs 가용성!
