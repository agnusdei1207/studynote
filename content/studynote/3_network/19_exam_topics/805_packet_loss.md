# Packet Loss - 패킷 손실

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 중 패킷 분실
> 2. **가치**: 신뢰성 측정
> 3. **융합:** TCP, Retransmission, QoS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Packet Loss는 패킷이 손실되는 현상입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Packet Loss                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              패킷 손실 원인 및 영향                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  원인:                                                │   │ │
│   │   │  - 네트워크 혼잡                                       │   │ │
│   │   │  - 하드웨어 오류                                       │   │ │
│   │   │  - 소프트웨어 버그                                     │   │ │
│   │   │  - 신호 간섭                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  패킷 흐름:                                            │   │ │
│   │   │  Sender: [1][2][3][4][5]                              │   │ │
│   │   │  Network: [1][2][X][4][5]  ← 3 손실                   │   │ │
│   │   │  Receiver: [1][2][4][5]                               │   │ │
│   │   │                                                      │   │ │
│   │   │  대응:                                                │   │ │
│   │   │  - TCP: 재전송                                        │   │ │
│   │   │  - UDP: 무시/Forward Error Correction                 │   │ │
│   │   │  - FEC: 전방 오류 수정                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  측정:                                                │   │ │
│   │   │  Packet Loss Rate = (손실 패킷 / 전체 패킷) × 100%    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 패킷 손실은 1% 이하여야 좋은 품질입니다.

**📢 섹션 요약 비유:** Packet Loss = 우편물 분실!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Loss Rate** | 손실률 |
| **Retransmission** | 재전송 |
| **FEC** | 전방 오류 수정 |
| **Duplicate ACK** | 중복 ACK |
| **Timeout** | 타임아웃 |

**핵심 알고리즸:** 손실 감지 → 재전송/무시

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**손실 허용 범위**

| 용도 | 허용 손실률 |
|:---|:---|
| **VoIP** | < 1% |
| **Video** | < 0.1% |
| **Data** | 0% |
