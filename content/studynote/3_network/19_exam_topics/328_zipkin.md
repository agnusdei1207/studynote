# Zipkin - 분산 추적

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오픈소스 분산 추적 시스템
> 2. **가치**: 지연 분석, 의존성 파악
> 3. **융합**: Spring Cloud, OpenTelemetry와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Zipkin은 분산 시스템의 지연 문제를 분석하는 추적 시스템입니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Collector** | 수집기 |
| **Storage** | 저장소 |
| **Query** | 조회 |
| **UI** | 웹 인터페이스 |

**비교표: 저장소 옵션**

```
+------------+----------+----------+
| 저장소     │ 성능     │ 용도     │
+------------+----------+----------+
| In-Memory  │ 빠름     │ 개발     │
| MySQL      │ 중간     │ 중소     │
| Cassandra  │ 높음     │ 대규모   │
| Elasticsearch│ 높음   │ 검색     │
+------------+----------+----------+
```

**핵심 알고리즸:** B3 전파, 샘플링

**📢 섹션 요약 비유:** B3 = 전파 헤더!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 추적 시스템**
