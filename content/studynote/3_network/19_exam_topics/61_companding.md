# 컴팬딩 (Companding)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 작은 신호 확장, 큰 신호 압축
> 2. **가치**: 동적 범위 확보, SNR 균등화
> 3. **융합**: μ-law, A-law, 전화망과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

컴팬딩(Companding)은 COMpressing + exPANDING의 합성어로, 작은 신호는 확장하고 큰 신호는 압축하는 비선형 양자화 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    컴팬딩 (Companding)                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              컴팬딩 원리                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  송신측 (압축):                                         │   │ │
│   │   │  • 작은 신호: 크게 확장                                 │   │ │
│   │   │  • 큰 신호: 작게 압축                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  수신측 (확장):                                         │   │ │
│   │   │  • 압축의 역변환                                       │   │ │
│   │   │  • 원래 신호로 복원                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  입력-출력 곡선:                                        │   │ │
│   │   │  출력                      ●                          │   │ │
│   │   │   ↑                      ╱                            │   │ │
│   │   │   │                    ╱  (비선형)                    │   │ │
│   │   │   │                  ╱                                │   │ │
│   │   │   │                ╱                                  │   │ │
│   │   │   │──────────────╱──────────→ 입력                    │   │ │
│   │   │                ●                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  목적:                                                 │   │ │
│   │   │  • 동적 범위 확장                                      │   │ │
│   │   │  • 전체 레벨에서 균등 SNR                               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              μ-law / A-law                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  μ-law (북미, 일본):                                    │   │ │
│   │   │  • μ = 255                                             │   │ │
│   │   │  • y = (ln(1+μ|x|))/(ln(1+μ))                         │   │ │
│   │   │  • T1 회선                                             │   │ │
│   │   │                                                      │   │ │
│   │   │  A-law (유럽, 국제):                                    │   │ │
│   │   │  • A = 87.6                                            │   │ │
│   │   │  • 선형 + 로그 혼합                                     │   │ │
│   │   │  • E1 회선                                             │   │ │
│   │   │                                                      │   │ │
│   │   │  비교:                                                 │   │ │
│   │   │  • μ-law: 작은 신호 SNR 약간 우수                       │   │ │
│   │   │  • A-law: 구현 약간 간단                                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 컴팬딩으로 8비트로 13~14비트 선형 양자화 효과를 얻습니다.

**💡 비유**: 컴팬딩은 소리를 들을 때 작은 소리는 키우고 큰 소리는 줄이는 것입니다.

**📢 섹션 요약 비유**: 북미 μ-law, 유럽 A-law!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | μ-law | A-law | 비고 |
|:---|:---:|:---:|:---|
| **파라미터** | μ=255 | A=87.6 | - |
| **지역** | 북미/일본 | 유럽/국제 | 표준 |
| **회선** | T1 | E1 | 적용 |
| **SNR** | 약간 우수 | 좋음 | 성능 |

**비교표**

```
+------------+----------+----------+
| 항목       | μ-law    | A-law    |
+------------+----------+----------+
| 지역       | 북미/일본| 유럽     |
| 회선       | T1       | E1       |
| 파라미터   | 255      | 87.6     |
| 저레벨 SNR | 더 좋음  | 좋음     |
+------------+----------+----------+
```

**핵심 알고리즘**: 입력 → 비선형 압축 → 전송 → 비선형 확장 → 복원

**📢 섹션 요약 비유**: 8비트 컴팬딩 = 13비트 선형!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 선형 vs 비선형**

| 방식 | 비트 | 동적 범위 |
|:---|:---:|:---:|
| **선형 8비트** | 8 | 48dB |
| **μ-law 8비트** | 8 | 72dB |
| **A-law 8비트** | 8 | 72dB |

**📢 섹션 요약 비유**: 컴팬딩으로 24dB 이득!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 한국 T1** → μ-law
**시나리오 2: 유럽 E1** → A-law
**시나리오 3: VoIP** → G.711 (μ/A-law)

**도입 체크리스트**: 지역 표준, 호환성, 음질

**📢 섹션 요약 비유**: 전화는 8비트 컴팬딩!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 동적 범위 72dB

**미래 전망**: VoIP로 확장

**참고 표준**: ITU-T G.711

**📢 섹션 요약 비유**: 컴팬딩은 전화 음질의 비밀!
