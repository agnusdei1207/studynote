# API Design - API 설계

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터페이스 설계
> 2. **가치**: 사용성, 유지보수성
> 3. **융합:** REST, GraphQL, Versioning

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

API Design은 API 인터페이스를 설계합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    API Design                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RESTful API 설계 원칙                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Resource-based:                                     │   │ │
│   │   │  GET    /users           → 사용자 목록               │   │ │
│   │   │  GET    /users/{id}      → 사용자 조회               │   │ │
│   │   │  POST   /users           → 사용자 생성               │   │ │
│   │   │  PUT    /users/{id}      → 사용자 수정               │   │ │
│   │   │  DELETE /users/{id}      → 사용자 삭제               │   │ │
│   │   │                                                      │   │ │
│   │   │  설계 원칙:                                            │   │ │
│   │   │  - 일관성 (Consistency)                              │   │ │
│   │   │  - 직관성 (Intuitiveness)                            │   │ │
│   │   │  - 버전 관리 (Versioning)                            │   │ │
│   │   │  - 문서화 (Documentation)                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 좋은 API 설계는 직관적이고 일관적입니다.

**📢 섹션 요약 비유:** API Design = 사용자 매뉴얼!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Endpoint** | 엔드포인트 |
| **Method** | HTTP 메서드 |
| **Request/Response** | 요청/응답 |
| **Status Code** | 상태 코드 |
| **Versioning** | 버전 관리 |

**핵심 알고리즸:** 리소스 식별 → 메서드 매핑 → 응답 설계

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**API 스타일**
