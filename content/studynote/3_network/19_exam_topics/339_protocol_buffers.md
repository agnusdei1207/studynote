# Protocol Buffers - 직렬화 형식

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 구조화된 데이터 직렬화 형식

> 3. **융합**: gRPC, 다국어 지원

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Protocol Buffers는 Google이 개발한 구조화된 데이터 직렬화 형식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Protocol Buffers                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Protobuf 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  .proto 파일                                         │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ message Person {                              │   │ │
│   │   │     │   string name = 1;                            │   │ │
│   │   │     │   int32 age = 2;                              │   │ │
│   │   │     │   string email = 3;                           │   │ │
│   │   │     │ }                                              │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           Protobuf Compiler (protoc)        │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • Java, Python, Go, C++, ...               │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [직렬화된 바이너리 데이터]                          │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 크기: 작음, 속도: 빠름                         │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [네트워크 전송]                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              데이터 유형                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 유형       │ 설명     │ 예시     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ scalar     │ 기본     │ int32    │               │   │ │
│   │   │  │ enum       │ 열거     │ STATUS   │               │   │ │
│   │   │  | message    │ 복합     │ Person   │               │   │ │
│   │   │  │ repeated   │ 배열     │ list     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Protocol Buffers는 바이너리 직렬화로 작고 빠릅니다.

**💡 비유**: Protobuf는 데이터의 압축 파일입니다. 작고 빠릅니다.

**📢 섹션 요약 비유:** Protobuf = 작고 빠름!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **.proto** | 스키마 파일 |
| **protoc** | 컴파일러 |
| **Message** | 메시지 타입 |
| **Field** | 필드 |

**비교표: Protobuf vs JSON**

```
+------------+----------+----------+
| 항목       │ Protobuf │ JSON     │
+------------+----------+----------+
| 크기       │ 작음     │ 큼       │
| 속도       │ 빠름     │ 느림     │
| 가독성     │ 없음     │ 있음     │
| 스키마     │ 필수     │ 선택     │
+------------+----------+----------+
```

**핵심 알고리즸:** 가변 길이 인코딩, 필드 번호

**📢 섹션 요약 비유:** Varint = 작은 크기!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 직렬화**

| 항목 | Protobuf | JSON | XML |
|:---|:---|:---|:---|
| **크기** | 작음 | 중간 | 큼 |
| **성능** | 높음 | 중간 | 낮음 |
| **디버깅** | 어려움 | 쉬움 | 쉬움 |

**📢 섹션 요약 비유:** 고성능 = Protobuf!
