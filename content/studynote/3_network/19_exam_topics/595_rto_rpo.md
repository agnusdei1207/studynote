# RTO/RPO - 복구 시간/시점 목표

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 복구 목표 지표
> 2. **가치**: DR 전략 수립 기준
> 3. **융합:** DR, Backup, HA, BC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RTO/RPO는 복구 목표를 정의합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RTO/RPO                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RTO vs RPO                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Timeline:                                            │   │ │
│   │   │  ─────────────────────────────────────────────→     │   │ │
│   │   │                                                      │   │ │
│   │   │  [Disaster]----RTO----[Recovered]                   │   │ │
│   │   │      │                           │                   │   │ │
│   │   │      │                           │                   │   │ │
│   │   │  Last Good Backup:     │                          │   │ │
│   │   │      └──────RPO──────→ │                          │   │ │
│   │   │                                                      │   │ │
│   │   │  RTO (Recovery Time Objective):                     │   │ │
│   │   │  복구까지 허용 시간                                  │   │ │
│   │   │  예: 4시간, 1일                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  RPO (Recovery Point Objective):                    │   │ │
│   │   │  복구할 데이터 시점                                  │   │ │
│   │   │  예: 1시간 전, 1일 전                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RTO는 시간, RPO는 데이터 손실 허용치입니다.

**📢 섹션 요약 비유:** RTO/RPO = SLA 목표!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RTO** | 복구 시간 목표 |
| **RPO** | 복구 시점 목표 |
| **MTTR** | Mean Time To Repair |
| **MTBF** | Mean Time Between Failures |

**핵심 알고리즸:** 장애 → RTO 내 복구 → RPO 내 데이터

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**RTO/RPO와 비용**

| RTO/RPO | 솔루션 | 비용 |
|:---|:---|:---|
| **높음** | 테이프 백업 | 낮음 |
| **중간** | 디스크 백업 | 중간 |
| **낮음** | 실시간 복제 | 높음 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
