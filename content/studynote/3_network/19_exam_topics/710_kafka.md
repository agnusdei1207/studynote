# Kafka - 카프카

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 이벤트 스트리밍 플랫폼
> 2. **가치**: 높은 처리량, 내구성
> 3. **융합:** Pub/Sub, Stream Processing, Event Sourcing

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Kafka는 분산 이벤트 스트리밍 플랫폼입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Kafka                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Kafka 아키텍처                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Producers                     Consumers             │   │ │
│   │   │  ┌─────────┐                   ┌─────────────────┐   │   │ │
│   │   │  │ App 1   │                   │ Consumer Group 1│   │   │ │
│   │   │  │ App 2   │    Kafka          └─────────────────┘   │   │ │
│   │   │  │ App 3   │    Cluster        ┌─────────────────┐   │   │ │
│   │   │  └────┬────┘                   │ Consumer Group 2│   │   │ │
│   │   │       │        ┌─────────────┐ └─────────────────┘   │   │ │
│   │   │       └───────→│ Topic A     │                       │   │ │
│   │   │                │ Partition 0 │                       │   │ │
│   │   │                │ Partition 1 │                       │   │ │
│   │   │                ├─────────────┤                       │   │ │
│   │   │                │ Topic B     │                       │   │ │
│   │   │                │ Partition 0 │                       │   │ │
│   │   │                └─────────────┘                       │   │ │
│   │   │                                                      │   │ │
│   │   │  특징: 높은 처리량, 내구성, 분산                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Kafka는 Topic에 메시지를 저장하고 Consumer가 소비합니다.

**📢 섹션 요약 비유:** Kafka = 분산 메시지 버스!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Topic** | 토픽 |
| **Partition** | 파티션 |
| **Producer** | 생산자 |
| **Consumer** | 소비자 |
| **Broker** | 브로커 |

**핵심 알고리즸:** Produce → Partition → Consume

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Kafka vs RabbitMQ**

| 항목 | Kafka | RabbitMQ |
|:---|:---|:---|
| **모델** | 로그 | 큐 |
| **순서** | 파티션 내 보장 | 큐 내 보장 |
| **재생** | 가능 | 불가능 |
| **처리량** | 매우 높음 | 높음 |
