# Network Segment - 네트워크 세그먼트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 구분 단위
> 2. **가치**: 충돌/브로드캐스트 도메인 분리
> 3. **융합:** Collision Domain, Broadcast Domain, VLAN

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Network Segment는 네트워크의 구분된 구역입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Network Segment                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              세그먼트 구분                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  충돌 도메인 vs 브로드캐스트 도메인:                       │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Hub (리피터):                                   ││   │ │
│   │   │  │  ┌───┐  ┌───┐  ┌───┐  ┌───┐                    ││   │ │
│   │   │  │  │ A │──│ B │──│ C │──│ D │                    ││   │ │
│   │   │  │  └───┘  └───┘  └───┘  └───┘                    ││   │ │
│   │   │  │     하나의 충돌 도메인 = 하나의 브로드캐스트 도메인   ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Switch:                                        ││   │ │
│   │   │  │  ┌───┐  ┌───┐  ┌───┐  ┌───┐                    ││   │ │
│   │   │  │  │ A │  │ B │  │ C │  │ D │                    ││   │ │
│   │   │  │  └─┬─┘  └─┬─┘  └─┬─┘  └─┬─┘                    ││   │ │
│   │   │  │    └──────┴──────┴──────┘                      ││   │ │
│   │   │  │  각 포트 = 별도 충돌 도메인                       ││   │ │
│   │   │  │  전체 = 하나의 브로드캐스트 도메인                  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Router:                                        ││   │ │
│   │   │  │  각 인터페이스 = 별도 브로드캐스트 도메인            ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 네트워크 세그먼트는 성능과 보안을 위해 분리합니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Collision Domain** | 충돌 도메인 |
| **Broadcast Domain** | 브로드캐스트 도메인 |
| **Segment** | 세그먼트 |
| **Bridge** | 브리지 |
| **Router** | 라우터 |

**핵심 알고리듬:** 장비 유형별 도메인 분리

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**장비별 도메인 구분**

| 장비 | 충돌 도메인 | 브로드캐스트 도메인 |
|:---|:---|:---|
| **Hub** | 1개 | 1개 |
| **Switch** | 포트별 | 1개 |
| **Router** | 포트별 | 포트별 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 성능 → 스위치로 충돌 도메인 분리
**시나리오 2:** 보안 → VLAN으로 브로드캐스트 분리
**시나리오 3:** 확장 → 라우터로 네트워크 분리
