# SDN (Software-Defined Networking)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제어 평면과 데이터 평면 분리
> 2. **가치**: 중앙 집중 제어, 프로그래밍 가능
> 3. **융합**: 클라우드, 데이터센터와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SDN(Software-Defined Networking)은 네트워크의 제어 평면과 데이터 평면을 분리하여 중앙에서 프로그래밍 방식으로 제어하는 기술입니다.

> **해설**: SDN은 네트워크를 소프트웨어로 제어 가능하게 만듭니다.

**💡 비유**: SDN은 각 교차로에 신호등이 따로 있는 대신, 교통 상황을 한 곳에서 모니터링하여 제어하는 것입니다.

**📢 섹션 요약 비유:** SDN = 중앙 제어!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소 | 역할 | 예시 |
|:---|:---|:---|
| **컨트롤러** | 제어 평면 | OpenDaylight |
| **스위치** | 데이터 평면 | OpenFlow |
| **NBI** | 앱 인터페이스 | REST API |
| **SBI** | 스위치 인터페이스 | OpenFlow |

**비교표**

```
+------------+----------+----------+
| 항목       │ 기존     │ SDN      │
+------------+----------+----------+
| 제어       │ 분산     │ 중앙     │
│ 설정       │ CLI      │ API      │
│ 유연성     │ 낮음     │ 높음     │
│ 자동화     │ 제한     │ 가능     │
+------------+----------+----------+
```

**핵심 알고리즸:** OpenFlow, 최단 경로, 트래픽 엔지니어링

**📢 섹션 요약 비유:** OpenFlow = SDN 표준!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 컨트롤러**

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 데이터센터** → SDN, 트래픽 엔지니어링
**시나리오 2: WAN** → SD-WAN
**시나리오 3: 통신사** → SDN + NFV

**도입 체크리스트:** 규모, 자동화 요구, 역량

**📢 섹션 요약 비유:** SDN은 네트워크 프로그래밍!
