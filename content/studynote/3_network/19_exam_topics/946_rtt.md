# RTT - 왕복 시간

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 왕복 지연 시간
> 2. **가치**: 네트워크 성능 지표
> 3. **융합:** Ping, RTO, Bandwidth-Delay

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RTT (Round-Trip Time)는 패킷 왕복 소요 시간입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RTT                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RTT 측정                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RTT 측정 과정:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Client                              Server      ││   │ │
│   │   │  │    │                                   │         ││   │ │
│   │   │  │    │──── Request (t1) ─────────────→  │         ││   │ │
│   │   │  │    │                                   │         ││   │ │
│   │   │  │    │                                   │         ││   │ │
│   │   │  │    │  ←─── Response (t2) ───────────  │         ││   │ │
│   │   │  │    │                                   │         ││   │ │
│   │   │  │    │                                   │         ││   │ │
│   │   │  │    │  RTT = t2 - t1                              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  RTT 활용:                                              │   │ │
│   │   │  - RTO 계산: 재전송 타임아웃                             │   │ │
│   │   │  - BDP 계산: 대역폭-지연 곱                              │   │ │
│   │   │  - 성능 측정: 지연 시간 평가                              │   │ │
│   │   │                                                      │   │ │
│   │   │  일반적인 RTT:                                           │   │ │
│   │   │  - LAN: < 1ms                                          │   │ │
│   │   │  - 인터넷: 10-100ms                                     │   │ │
│   │   │  - 위성: 500-800ms                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RTT는 네트워크 성능의 핵심 지표입니다.

**📢 섹션 요약 비유:** RTT = 왕복 여행 시간!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **One-way** | 단방향 지연 |
| **RTT** | 왕복 시간 |
| **RTO** | 재전송 타임아웃 |
| **BDP** | 대역폭-지연 곱 |
| **SRTT** | 평균 RTT |

**핵심 알고리듬:** RTO = SRTT + 4*RTTVAR

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**RTT vs Latency**

| 항목 | Latency | RTT |
|:---|:---|:---|
| **방향** | 단방향 | 왕복 |
| **측정** | 어려움 | 쉬움 |
| **값** | RTT/2 | 측정값 |
