# AMI (Alternate Mark Inversion)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 1 비트를 +V, -V로 교대로 표현
> 2. **가치**: DC 성분 제거, 오류 검출 가능
> 3. **융합**: T1/E1, 전화망, B8ZS, HDB3와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

AMI(Alternate Mark Inversion)는 1(Mark)을 +V와 -V로 교대로 표현하는 양극성(Bipolar) 부호화 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    AMI (Alternate Mark Inversion)                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              AMI 부호화 규칙                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  규칙:                                                 │   │ │
│   │   │  • 0 (Space) = 무전압 (0V)                             │   │ │
│   │   │  • 1 (Mark)  = +V, -V 교대 (Alternate)                │   │ │
│   │   │                                                      │   │ │
│   │   │  파형:                                                 │   │ │
│   │   │  데이터:    1   0   1   1   0   0   1   0             │   │ │
│   │   │             ┌┐       ┌┐┌┐       ┌┐                  │   │ │
│   │   │           ──┘└───────┘└┘└───────┘└───                │   │ │
│   │   │            +V      -V +V       -V                     │   │ │
│   │   │             ↑       ↑ ↑         ↑                     │   │ │
│   │   │            교대     교대 교대    교대                  │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                               │   │ │
│   │   │  • DC 성분 없음 (±V 평균 = 0)                         │   │ │
│   │   │  • AMI 위반 = 오류 검출                               │   │ │
│   │   │  • 연속 0에서 동기화 어려움                            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              AMI 위반 (Bipolar Violation)                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  정상:                                                 │   │ │
│   │   │  데이터:  1   0   1                                    │   │ │
│   │   │           +V      -V   (교대 정상)                     │   │ │
│   │   │                                                      │   │ │
│   │   │  위반:                                                 │   │ │
│   │   │  데이터:  1   0   1   0   1                           │   │ │
│   │   │           +V      -V      +V  ← 위반!                  │   │ │
│   │   │                          (같은 극성 연속 = 에러)         │   │ │
│   │   │                                                      │   │ │
│   │   │  오류 검출: 연속 같은 극성 = 전송 에러                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: AMI는 T1/E1 디지털 전화망의 표준 부호화입니다.

**💡 비유**: AMI는 왔다 갔다 하는 진자 운동과 같습니다.

**📢 섹션 요약 비유**: AMI = 1은 교대로, 0은 없음!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Mark** | 1 비트 | ±V 교대 | 표시 |
| **Space** | 0 비트 | 0V | 공백 |
| **위반** | 에러 | 같은 극성 | 오류 |
| **DC** | 성분 | 없음 | 평균 0 |

**AMI 문제점과 해결**

```
문제: 연속 0에서 동기화 어려움

해결책:
1. B8ZS (Bipolar with 8-Zero Substitution)
   - 8개 연속 0 → 특수 패턴으로 대체
   - 북미 T1 표준

2. HDB3 (High-Density Bipolar 3 zeros)
   - 4개 연속 0 → 특수 패턴으로 대체
   - 유럽 E1 표준
```

**핵심 알고리즘**: 0→0V, 1→±V 교대 → 위반 검출

**📢 섹션 요약 비유**: B8ZS/HDB3로 연속 0 해결!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: AMI 변형**

| 방식 | 연속 0 허용 | 지역 |
|:---|:---:|:---|
| **AMI** | 제한 | 기본 |
| **B8ZS** | 8개 | 북미 |
| **HDB3** | 3개 | 유럽 |

**📢 섹션 요약 비유**: 한국은 T1/B8ZS, 유럽은 E1/HDB3!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: T1 회선** → AMI + B8ZS
**시나리오 2: E1 회선** → AMI + HDB3
**시나리오 3: ISDN** → AMI

**도입 체크리스트**: 지역, 연속 0, 동기화

**📢 섹션 요약 비유**: 전화망은 AMI 기반!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: DC 제거, 에러 검출

**미래 전망**: 광통신으로 이동

**참고 표준**: ITU-T G.703, ANSI T1.403

**📢 섹션 요약 비유**: AMI는 디지털 전화망의 기본!
