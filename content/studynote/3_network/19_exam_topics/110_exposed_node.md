# 노출 노드 문제 (Exposed Node Problem)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 불필요한 전송 자제로 효율 저하
> 2. **가치**: RTS/CTS의 부작용 이해
> 3. **융합**: WiFi, 무선 네트워크와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

노출 노드 문제(Exposed Node Problem)는 인접한 노드의 전송을 감지하여 불필요하게 전송을 자제하는 현상입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    노출 노드 문제 (Exposed Node Problem)            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              노출 노드 상황                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │   A ←────────→ B ←────────→ C ←────────→ D           │   │ │
│   │   │   │   범위    │   범위    │   범위    │               │   │ │
│   │   │                                                      │   │ │
│   │   │  상황:                                                   │   │ │
│   │   │  • B가 A에게 RTS 전송                                   │   │ │
│   │   │  • C는 B의 RTS를 감지                                   │   │ │
│   │   │  • C는 NAV 동안 전송 금지                                │   │ │
│   │   │                                                      │   │ │
│   │   │  문제:                                                   │   │ │
│   │   │  • C는 D에게 전송 가능 (간섭 없음)                        │   │ │
│   │   │  • 그러나 C는 B의 전송 때문에 자제                        │   │ │
│   │   │  → 불필요한 전송 지연                                    │   │ │
│   │   │  → 스펙트럼 효율 저하                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  핵심:                                                   │   │ │
│   │   │  숨은 노드: 충돌 발생 (안 보여서)                         │   │ │
│   │   │  노출 노드: 불필요한 자제 (보여서)                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              숨은 노드 vs 노출 노드                            │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  숨은 노드 (Hidden Node):                              │   │ │
│   │   │  • 상대방이 안 보여서 충돌                              │   │ │
│   │   │  • RTS/CTS로 해결                                      │   │ │
│   │   │  • 성능 저하 (충돌)                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  노출 노드 (Exposed Node):                              │   │ │
│   │   │  • 상대방이 보여서 불필요한 대기                         │   │ │
│   │   │  • RTS/CTS로 인해 악화                                  │   │ │
│   │   │  • 성능 저하 (비효율)                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  트레이드오프:                                           │   │ │
│   │   │  RTS/CTS 사용:                                         │   │ │
│   │   │  • 숨은 노드 문제 해결                                  │   │ │
│   │   │  • 노출 노드 문제 악화                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 노출 노드 문제는 RTS/CTS의 부작용입니다.

**💡 비유**: 노출 노드는 옆 방에서 말하는 걸 듣고 내 방에서도 말하면 안 된다고 생각하는 것입니다.

**📢 섹션 요약 비유**: 노출 노드 = 과도한 예절!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 문제 | 원인 | 결과 | 해결 |
|:---|:---|:---|:---|
| **숨은 노드** | 안 보임 | 충돌 | RTS/CTS |
| **노출 노드** | 보임 | 비효율 | 영역 기반 |

**비교표**

```
+------------+----------+----------+
| 항목       | 숨은 노드| 노출 노드|
+------------+----------+----------+
| 감지       | X        | O        |
| 충돌       | O        | X        |
| 효율       | 낮음     | 낮음     |
| RTS/CTS    | 해결     | 악화     |
+------------+----------+----------+
```

**핵심 알고리즸**: NAV 설정 → 불필요한 자제 → 전송 기회 손실

**📢 섹션 요약 비유**: RTS/CTS는 양날의 검!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 숨은 노드 많음** → RTS/CTS 사용
**시나리오 2: 노출 노드 많음** → RTS/CTS 끄기
**시나리오 3: 혼합** → 임계값 조절

**도입 체크리스트**: 네트워크 토폴로지, 노드 밀도

**📢 섹션 요약 비유**: 상황에 맞는 트레이드오프!
