# Fragmentation - 단편화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 큰 패킷 분할
> 2. **가치**: MTU 적응
> 3. **융합:** IP Fragmentation, MTU, DF

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Fragmentation은 큰 패킷을 작은 조각으로 나눕니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Fragmentation                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IP 단편화 과정                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  원본 패킷 (4000 bytes):                              │   │ │
│   │   │  ┌────────────────────────────────────────────────┐ │   │ │
│   │   │  │ IP Header (20B) │ Data (3980B)                 │ │   │ │
│   │   │  └────────────────────────────────────────────────┘ │   │ │
│   │   │                      ↓                               │   │ │
│   │   │              MTU = 1500 bytes                        │   │ │
│   │   │                      ↓                               │   │ │
│   │   │  단편화 결과:                                          │   │ │
│   │   │  ┌────────────────────────────────────────────────┐ │   │ │
│   │   │  │ Fragment 1: IP Hdr (20B) + Data (1480B)        │ │   │ │
│   │   │  │   Offset: 0, More: 1                           │ │   │ │
│   │   │  ├────────────────────────────────────────────────┤ │   │ │
│   │   │  │ Fragment 2: IP Hdr (20B) + Data (1480B)        │ │   │ │
│   │   │  │   Offset: 1480, More: 1                        │ │   │ │
│   │   │  ├────────────────────────────────────────────────┤ │   │ │
│   │   │  │ Fragment 3: IP Hdr (20B) + Data (1020B)        │ │   │ │
│   │   │  │   Offset: 2960, More: 0                        │ │   │ │
│   │   │  └────────────────────────────────────────────────┘ │   │ │
│   │   │                                                      │   │ │
│   │   │  IP 헤더 필드:                                         │   │ │
│   │   │  - Identification: 같은 패킷의 단편 식별               │   │ │
│   │   │  - Fragment Offset: 데이터 오프셋/8                    │   │ │
│   │   │  - More Fragments (MF): 추가 단편 있음                 │   │ │
│   │   │  - Don't Fragment (DF): 단편화 금지                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 단편화는 다양한 MTU를 가진 네트워크에서 필수적입니다.

**📢 섹션 요약 비유:** Fragmentation = 짐 나누기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **MTU** | 최대 전송 단위 |
| **Offset** | 오프셋 |
| **MF** | More Fragments |
| **DF** | Don't Fragment |
| **Reassembly** | 재조립 |

**핵심 알고리즸:** 원본 → 분할 → 전송 → 재조립
