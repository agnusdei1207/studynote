# DNS Record - DNS 레코드

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: DNS 데이터 항목
> 2. **가치**: 다양한 리소스 매핑
> 3. **융합:** A, AAAA, CNAME, MX, TXT

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DNS Record는 도메인 정보를 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DNS Record                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 DNS 레코드                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  레코드 유형:                                             │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  A:      도메인 → IPv4                           ││   │ │
│   │   │  │          example.com → 93.184.216.34            ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  AAAA:   도메인 → IPv6                           ││   │ │
│   │   │  │          example.com → 2606:2800:220:1:248:1893 ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  CNAME:  별칭 → 정식 이름                         ││   │ │
│   │   │  │          www.example.com → example.com          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  MX:     메일 서버                               ││   │ │
│   │   │  │          example.com → mail.example.com (10)    ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  TXT:    텍스트 정보                              ││   │ │
│   │   │  │          SPF, DKIM, 도메인 소유 확인              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  NS:     네임서버                                ││   │ │
│   │   │  │  SOA:    권한 시작                                ││   │ │
│   │   │  │  PTR:    IP → 도메인 (역방향)                      ││   │ │
│   │   │  │  SRV:    서비스 위치                              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DNS 레코드는 도메인 시스템의 핵심입니다.

**📢 섹션 요약 비유:** DNS Record = 전화번호부 항목!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Name** | 이름 |
| **Type** | 유형 |
| **TTL** | 캐시 시간 |
| **Class** | 클래스 |
| **Data** | 데이터 |

**핵심 알고리듬:** 질의 → 레코드 검색 → 응답

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**레코드 용도 비교**

| 레코드 | 용도 |
|:---|:---|
| **A/AAAA** | IP 주소 |
| **CNAME** | 별칭 |
| **MX** | 이메일 |
| **TXT** | 텍스트 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹사이트 → A 레코드
**시나리오 2:** 이메일 → MX 레코드
**시나리오 3:** 보안 → TXT (SPF/DKIM)
