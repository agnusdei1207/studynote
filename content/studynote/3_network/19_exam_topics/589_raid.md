# RAID (Redundant Array of Independent Disks) - 디스크 배열

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 디스크를 논리적 하나로
> 2. **가치**: 성능, 신뢰성
> 3. **융합:** Storage, HA, Backup

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RAID는 여러 디스크를 조합해 성능/신뢰성을 높입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RAID                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RAID 레벨                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 0 (Striping):                                  │   │ │
│   │   │  [D1][D2][D3][D4] → 성능 ↑, 신뢰성 ↓                 │   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 1 (Mirroring):                                 │   │ │
│   │   │  [D1][D1][D2][D2] → 성능 -, 신뢰성 ↑                 │   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 5 (Distributed Parity):                        │   │ │
│   │   │  [D1][D2][P1][D3][P2][P3] → 성능 ↑, 신뢰성 ↑         │   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 6 (Double Parity):                             │   │ │
│   │   │  2개 디스크 동시 장애 허용                            │   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 10 (1+0):                                      │   │ │
│   │   │  미러링 + 스트라이핑                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RAID 레벨에 따라 성능/용량/신뢰성이 다릅니다.

**📢 섹션 요약 비유:** RAID = 분업 체계!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 레벨 | 최소 디스크 | 장애 허용 | 용량 효율 |
|:---|:---|:---|:---|
| **RAID 0** | 2 | 0 | 100% |
| **RAID 1** | 2 | 1 | 50% |
| **RAID 5** | 3 | 1 | (n-1)/n |
| **RAID 6** | 4 | 2 | (n-2)/n |
| **RAID 10** | 4 | 1/미러 | 50% |

**핵심 알고리즸:** Striping/Mirroring/Parity

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**RAID 선택 기준**

| 용도 | 추천 |
|:---|:---|
| **성능** | RAID 0 |
| **신뢰성** | RAID 1 |
| **균형** | RAID 5/6 |
| **최고** | RAID 10 |
