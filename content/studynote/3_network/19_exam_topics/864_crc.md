# CRC - 순환 중복 검사

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다항식 기반 검출
> 2. **가치**: 높은 오류 검출률
> 3. **융합:** Ethernet, HDLC, CRC-32

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CRC (Cyclic Redundancy Check)는 다항식 나눗셈으로 오류를 검출합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CRC (Cyclic Redundancy Check)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CRC 계산                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  과정:                                                 │   │ │
│   │   │  1. 데이터를 다항식으로 표현                            │   │ │
│   │   │  2. 생성 다항식(G)로 나눔                               │   │ │
│   │   │  3. 나머지가 CRC                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  예시 (CRC-8):                                          │   │ │
│   │   │  Data:    11010011101100                              │   │ │
│   │   │  Padding: 11010011101100 00000000                     │   │ │
│   │   │  Generator (G): x^8 + x^2 + x^1 + 1 = 100000111       │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │     11010011101100 00000000                     ││   │ │
│   │   │  │   ÷ 100000111                                   ││   │ │
│   │   │  │   ─────────────                                 ││   │ │
│   │   │  │     나머지 = CRC (8비트)                         ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  일반적인 CRC:                                          │   │ │
│   │   │  - CRC-8: 8비트, 단순                                  │   │ │
│   │   │  - CRC-16: 16비트, 일반적                              │   │ │
│   │   │  - CRC-32: 32비트, Ethernet                           │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                 │   │ │
│   │   │  - 버스트 오류 검출 우수                                │   │ │
│   │   │  - 하드웨어 구현 쉬움                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: CRC는 데이터링크 계층의 표준 오류 검출 방법입니다.

**📢 섹션 요약 비유:** CRC = 나머지 연산!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Generator** | 생성 다항식 |
| **Divisor** | 제수 |
| **Remainder** | 나머지 |
| **FCS** | Frame Check Sequence |
| **XOR** | 배타적 논리합 |

**핵심 알고리즸:** 데이터 ÷ G = 나머지 (CRC)

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**CRC 다항식 비교**

| CRC | 다항식 | 비트 |
|:---|:---|:---|
| **CRC-8** | x^8+x^2+x+1 | 8 |
| **CRC-16** | x^16+x^15+x^2+1 | 16 |
| **CRC-32** | x^32+x^26+x^23+...+1 | 32 |
| **CRC-CCITT** | x^16+x^12+x^5+1 | 16 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** Ethernet → CRC-32
**시나리오 2:** HDLC → CRC-16
**시나리오 3:** ZIP → CRC-32
