# Saga Pattern - 사가 패턴

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 트랜잭션 관리
> 2. **가치**: 일관성, 보상 트랜잭션
> 3. **융합:** Microservices, Event Sourcing, Choreography

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Saga Pattern은 분산 트랜잭션을 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Saga Pattern                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              사가 패턴 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Choreography (코레오그래피):                         │   │ │
│   │   │  Order Service ──→ Payment Service ──→ Inventory    │   │ │
│   │   │        ↑                  │                  │       │   │ │
│   │   │        │                  ↓                  ↓       │   │ │
│   │   │    [이벤트 기반, 분산 조정]                          │   │ │
│   │   │                                                      │   │ │
│   │   │  Orchestration (오케스트레이션):                      │   │ │
│   │   │  ┌─────────────┐                                    │   │ │
│   │   │  │ Orchestrator│ ──→ Order ──→ Payment ──→ Inventory│   │ │
│   │   │  └─────────────┘                                    │   │ │
│   │   │        [중앙 조정자]                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  실패 시 보상 트랜잭션:                               │   │ │
│   │   │  Order → Payment ✗ → Refund → Cancel Order          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 사가 패턴은 분산 환경에서 일관성을 보장합니다.

**📢 섹션 요약 비유:** Saga Pattern = 여행 일정!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

| 항목 | 설명 |
|:---|:---|
| **Step** | 단계 |
| **Compensation** | 보상 |

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**코레오그래피 vs 오케스트레이션**

| 항목 | Choreography | Orchestration |
|:---|:---|:---|
| **조정** | 분산 | 중앙 |
| **복잡성** | 낮음 | 높음 |
| **결합** | 느슨함 | 강함 |
| **모니터링** | 어려움 | 쉬움 |
