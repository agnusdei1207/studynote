# Stream Processing - 스트림 처리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실시간 데이터 처리
> 2. **가치**: 낮은 지연, 즉시 반응
> 3. **융합:** Kafka, Flink, Real-time

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Stream Processing은 데이터를 실시간으로 처리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Stream Processing                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스트림 처리 아키텍처                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Producers          Stream            Consumers      │   │ │
│   │   │  ┌─────────┐       ┌───────────────────────┐ ┌────────┐│   │ │
│   │   │  │ IoT     │       │ Event 1 │ Event 2 │   │ │ Alert  ││   │ │
│   │   │  │ Sensor  │──────→│ Event 3 │ Event 4 │──→│ System ││   │ │
│   │   │  └─────────┘       │ Event 5 │ ...     │   │ └────────┘│   │ │
│   │   │  ┌─────────┐       └───────────────────────┘ ┌────────┐│   │ │
│   │   │  │ App     │──────→     Processing          │ DB     ││   │ │
│   │   │  │ Logs    │            Engine              └────────┘│   │ │
│   │   │  └─────────┘       (Kafka, Flink, Spark)              │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  - 무한 데이터 스트림                                 │   │ │
│   │   │  - 낮은 지연 (ms 단위)                                │   │ │
│   │   │  - 상태 유지 처리                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스트림 처리는 데이터가 도착하는 즉시 처리합니다.

**📢 섹션 요약 비유:** Stream Processing = 실시간 컨베이어!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Batch vs Stream**

| 항목 | Batch | Stream |
|:---|:---|:---|
| **지연** | 높음 | 낮음 |
| **데이터** | 한정 | 무한 |
| **복잡도** | 낮음 | 높음 |
| **용도** | 분석 | 실시간 |
