# Service Mesh - 서비스 메시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 마이크로서비스 통신 인프라
> 2. **가치**: 관찰, 보안, 트래픽 제어
> 3. **융합**: K8s, Istio와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Service Mesh는 마이크로서비스 간 통신을 처리하는 인프라 계층입니다.


> **해설**: Service Mesh는 Sidecar 프록시로 서비스 간 통신을 제어합니다.

**💡 비유**: Service Mesh는 도시의 교통 관제 시스템입니다.

**📢 섹션 요약 비유:** Service Mesh = 마이크로서비스 통신!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Data Plane** | 프록시 |
| **Control Plane** | 관리 |
| **Sidecar** | 컨테이너 |
| **mTLS** | 보안 |

**비교표: Service Mesh 구현**

```
+------------+----------+----------+
| 항목       │ Istio    │ Linkerd │
+------------+----------+----------+
| 프록시     │ Envoy    │ linkerd2│
| 복잡도     │ 높음     │ 낮음    │
| 기능       │ 풍부     │ 기본    │
| 성능       │ 중간     │ 높음    │
+------------+----------+----------+
```

**핵심 알고리즸:** 라우팅, mTLS

**📢 섹션 요약 비유:** Sidecar = 옆에 있어요!
