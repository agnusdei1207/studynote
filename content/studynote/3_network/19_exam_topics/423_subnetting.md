# 서브넷팅 - Subnetting

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크를 논리적으로 분할
> 2. **가치**: 브로드캐스트 도메인 축소
> 3. **융합**: VLSM, CIDR, 보안

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

서브넷팅은 하나의 네트워크를 여러 서브넷으로 나눕니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    서브넷팅                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              서브넷팅 예시                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  원본: 192.168.1.0/24 (256 주소)                    │   │ │
│   │   │        │                                             │   │ │
│   │   │        │ 서브넷팅 (/26)                               │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ 192.168.1.0/26   (0-63)      → 서브넷 1     │    │   │ │
│   │   │  │ 192.168.1.64/26  (64-127)    → 서브넷 2     │    │   │ │
│   │   │  │ 192.168.1.128/26 (128-191)  → 서브넷 3     │    │   │ │
│   │   │  │ 192.168.1.192/26 (192-255)  → 서브넷 4     │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              서브넷 구성요소                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 요소       │ /24 예시│ /26 예시│               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 네트워크   │ 0        │ 0        │               │   │ │
│   │   │  │ 첫 번째    │ 1        │ 1        │               │   │ │
│   │   │  | 마지막     │ 254      │ 62       │               │   │ │
│   │   │  │ 브로드캐스트│ 255     │ 63       │               │   │ │
│   │   │  │ 사용 가능  │ 254      │ 62       │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 서브넷팅은 브로드캐스트 도메인을 분리합니다.

**💡 비유**: 서브넷팅은 큰 방을 작은 방으로 나누는 것입니다.

**📢 섹션 요약 비유:** 서브넷팅 = 분할!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Network Address** | 네트워크 주소 |
| **Broadcast Address** | 브로드캐스트 주소 |
| **Host Range** | 호스트 범위 |
| **Subnet Mask** | 서브넷 마스크 |

**비교표: 서브넷 크기**

```
+------------+----------+----------+
| 항목       │ /24      │ /25      │
+------------+----------+----------+
| 서브넷수   │ 1        │ 2        │
| 호스트수   │ 254      │ 126      │
| 브로드캐스트│ 1        │ 2        │
+------------+----------+----------+
```

**핵심 알고리즸:** 2^n 서브넷, 2^n-2 호스트

**📢 섹션 요약 비유:** 2의 거듭제곱 = 서브넷팅!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 서브넷팅 전/후**

| 항목 | 서브넷팅 전 | 서브넷팅 후 |
|:---|:---|:---|
| **브로드캐스트** │ 큼 | 작음 |
| **보안** │ 낮음 | 높음 |
| **관리** │ 단순 | 복잡 |

**📢 섹션 요약 비유:** 분할 = 보안!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 부서별 분리 → 서브넷
**시나리오 2:** 보안 구역 → 서브넷
**시나리오 3:** 효율적 할당 → VLSM

**도입 체크리스트:** 호스트 수, 성장, 라우팅

**📢 섹션 요약 비유:** 계획 = 서브넷팅!
