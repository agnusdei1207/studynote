# WebSocket - 웹소켓

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 양방향 실시간 통신
> 2. **가치**: 낮은 오버헤드, 실시간
> 3. **융합:** HTTP, Real-time, Chat

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

WebSocket은 단일 TCP로 양방향 통신을 합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WebSocket                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              WebSocket 동작                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Client                              Server          │   │ │
│   │   │     │                                   │            │   │ │
│   │   │     │ ── HTTP Upgrade Request ───────→ │            │   │ │
│   │   │     │    GET /chat HTTP/1.1            │            │   │ │
│   │   │     │    Upgrade: websocket            │            │   │ │
│   │   │     │                                   │            │   │ │
│   │   │     │ ←── HTTP 101 Switching ───────── │            │   │ │
│   │   │     │                                   │            │   │ │
│   │   │     │ ←════ WebSocket Connection ════→ │            │   │ │
│   │   │     │     (Full-Duplex, Real-time)     │            │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                               │   │ │
│   │   │  - ws:// or wss:// (TLS)                             │   │ │
│   │   │  - 핸드셰이크 후 지속 연결                            │   │ │
│   │   │  - 낮은 오버헤드                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: WebSocket은 HTTP Upgrade로 시작 후 양방향 통신합니다.

**📢 섹션 요약 비유:** WebSocket = 직통 전화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Handshake** | HTTP Upgrade |
| **Frame** | 데이터 프레임 |
| **Ping/Pong** | 연결 유지 |
| **Close** | 연결 종료 |

**핵심 알고리즸:** Upgrade → 연결 → Frame 송수신

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**HTTP Polling vs WebSocket**

| 항목 | Polling | WebSocket |
|:---|:---|:---|
| **방향** | 단방향 | 양방향 |
| **오버헤드** | 높음 | 낮음 |
| **실시간** | 지연 | 즉시 |
