# RTO/RPO - 복구 시간/시점 목표

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 복구 목표 지표
> 2. **가치**: DR 전략 기준
> 3. **융합**: BC, HA와 통합

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RTO(Recovery Time Objective)는 복구 시간 목표, RPO(Recovery Point Objective)는 복구 시점 목표입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RTO/RPO                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RTO/RPO 개념                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  시간축                                              │   │ │
│   │   │  ─────────────────────────────────────────────▶     │   │ │
│   │   │       │                    │                    │    │   │ │
│   │   │       │                    │                    │    │   │ │
│   │   │    [재해]              [RPO]                [RTO]   │   │ │
│   │   │       │                    │                    │    │   │ │
│   │   │       │◀── 데이터 손실 ──▶│                    │    │   │ │
│   │   │       │                    │◀── 복구 시간 ────▶│    │   │ │

│   │   │  ┌──────────┐        ┌──────────┐        ┌──────────┐│   │ │
│   │   │  │ 장애 발생 │        │ 마지막   │        │ 복구 완료 ││   │ │
│   │   │  │          │        │ 백업 시점 │        │          ││   │ │
│   │   │  └──────────┘        └──────────┘        └──────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RTO/RPO 정의                                      │ │
│   │                                                              │ │

│   │   │     • 서비스 복구까지 최대 허용 시간                  │   │ │
│   │   │     • "얼마나 빨리 복구해야 하는가?"                 │   │ │
│   │   │                                                      │   │ │
│   │   │  RPO (Recovery Point Objective):                     │   │ │
│   │   │     • 허용 가능한 최대 데이터 손실량                  │   │ │
│   │   │     • "어느 시점까지 복구해야 하는가?"               │   │ │

└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RTO는 시간, RPO는 데이터 기준입니다.

**💡 비유**: RTO는 "몇 분 안에 돌아와야 하나", RPO는 "몇 분치 데이터를 잃어도 되나"입니다.

**📢 섹션 요약 비유:** RTO = 시간, RPO = 데이터!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RTO** | 복구 시간 |
| **RPO** | 데이터 손실 |
| **MTTR** | 평균 복구 시간 |
| **MTBF** | 평균 고장 간격 |

**비교표: RTO/RPO vs 비용**

```
+------------+----------+----------+
| RTO/RPO    │ 기술     │ 비용     │
+------------+----------+----------+
| 분 단위    │ 동기 복제│ 매우 높음│
| 시간 단위  │ 비동기   │ 높음     │
| 일 단위    │ 백업     │ 낮음     │
+------------+----------+----------+
```

**핵심 알고리즸:** BIA, 비용 분석

**📢 섹션 요약 비유:** RTO/RPO가 짧을수록 비용 증가!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: RPO별 기술**

| RPO | 기술 |
|:---|:---|
| **0** | 동기 복제 |
| **분** | 비동기 복제 |
| **시간** | 주기적 복제 |
| **일** | 백업 |

**📢 섹션 요약 비유:** RPO 0 = 무손실!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 금융 거래** → RTO 0, RPO 0
**시나리오 2: ERP** → RTO 4시간, RPO 1시간
**시나리오 3: 파일 서버** → RTO 24시간, RPO 24시간

**도입 체크리스트:** BIA, 비용, 기술

**📢 섹션 요약 비유:** 비즈니스 영향도로 결정!
