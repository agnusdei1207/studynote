# CIFS/SMB - 윈도우 파일 공유

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 윈도우 파일 공유 프로토콜
> 2. **가치**: 윈도우 환경 호환
> 3. **융합**: AD, NAS와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SMB(Server Message Block)/CIFS(Common Internet File System)는 윈도우 파일 공유 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CIFS/SMB                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SMB 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Windows Client]    [Linux Client]                 │   │ │
│   │   │       │                    │                         │   │ │
│   │   │       │ UNC Path           │ mount -t cifs           │   │ │
│   │   │       │ \\server\share     │ //server/share          │   │ │
│   │   │       │                    │                         │   │ │
│   │   │       └────────────────────┼─────────────────┘       │   │ │
│   │   │                            │                          │   │ │
│   │   │                            │ SMB (TCP 445)            │   │ │
│   │   │                            │                          │   │ │
│   │   │                            ▼                          │   │ │
│   │   │                     ┌──────────┐                      │   │ │
│   │   │                     │ SMB      │                      │   │ │
│   │   │                     │ Server   │                      │   │ │
│   │   │                     │ (Share)  │                      │   │ │
│   │   │                     └──────────┘                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SMB 버전                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 버전       │ 특징     │ 보안     │               │   │ │

│   │   │                                                      │   │ │
```

> **해설**: SMB는 윈도우 환경에서 파일, 프린터, IPC를 공유합니다.

**💡 비유**: SMB는 윈도우 네트워크 드라이브입니다.

**📢 섹션 요약 비유:** SMB = 윈도우 파일 공유!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Share** | 공유 폴더 |
| **UNC** | 경로 형식 |
| **NTLM/Kerberos** | 인증 |
| **DFS** | 분산 파일 시스템 |

**비교표: SMB vs NFS**

```
+------------+----------+----------+
| 항목       │ SMB      │ NFS      │
+------------+----------+----------+
| OS         │ 윈도우   │ 유닉스   │
| 인증       │ NTLM/Kerb│ UID/GID  │
| 권한       │ ACL      │ rwx      │
| 잠금       │ Oplock   │ NLM      │
+------------+----------+----------+
```

**핵심 알고리즸:** Oplock, 캐싱

**📢 섹션 요약 비유:** SMB 3.0 = 암호화!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: SMB 기능**

| 기능 | 설명 |
|:---|:---|
| **DFS** | 분산 파일 시스템 |
| **BranchCache** | 캐싱 |
| **SMB Direct** | RDMA |
| **암호화** | SMB 3.0+ |

**📢 섹션 요약 비유:** SMB Direct = RDMA 성능!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 윈도우 환경** → SMB 3.0
**시나리오 2: 혼합 환경** → SMB + NFS
**시나리오 3: 보안** → SMB 3.0 암호화

**도입 체크리스트:** 버전, AD 통합, 보안

**📢 섹션 요약 비유:** SMB 1.0 = 비활성화 권장!
