# Demultiplexing - 디멀티플렉싱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 신호 분리
> 2. **가치**: 목적지 식별
> 3. **융합:** Port-based, Protocol, Socket

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Demultiplexing은 결합된 신호를 분리합니다.

**📢 섹션 요약 비유:** Demultiplexing = 우편물 분류!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Header** | 헤더 정보 |
| **Port** | 포트 번호 |
| **Socket** | 소켓 |
| **Protocol** | 프로토콜 |
| **Process** | 프로세스 |

**핵심 알고리듬:** 헤더 검사 → 분류 → 해당 소켓 전달

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Mux vs Demux**

| 항목 | Multiplexing | Demultiplexing |
|:---|:---|:---|
| **방향** | 결합 | 분리 |
| **송신** | 클라이언트 | 서버 |
| **수신** | 서버 | 클라이언트 |
| **기준** | 주소/포트 | 주소/포트 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹서버 → 포트 기반 분리
**시나리오 2:** OS → 프로세스별 전달
**시나리오 3:** 라우터 → 프로토콜 분리
