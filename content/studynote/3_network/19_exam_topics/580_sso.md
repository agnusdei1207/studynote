# SSO (Single Sign-On) - 단일 로그인

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 번 로그인으로 여러 서비스 이용
> 2. **가치**: 편의성, 보안
> 3. **융합:** SAML, OIDC, LDAP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SSO는 한 번의 인증으로 여러 시스템에 접근합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SSO                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SSO 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 사용자 → App A 접근                              │   │ │
│   │   │  2. App A → SSO 서버로 리다이렉트                    │   │ │
│   │   │  3. SSO 서버에서 로그인                              │   │ │
│   │   │  4. App A로 토큰 전달                                 │   │ │
│   │   │  5. App B 접근 시 이미 로그인됨                       │   │ │
│   │   │                                                      │   │ │
│   │   │       ┌─────────────┐                                │   │ │
│   │   │       │   SSO 서버   │                                │   │ │
│   │   │       └──────┬──────┘                                │   │ │
│   │   │          /   |   \                                   │   │ │
│   │   │        App A App B App C                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SSO는 중앙 인증 서버가 모든 인증을 처리합니다.

**📢 섹션 요약 비유:** SSO = 마스터 키!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 프로토콜 | 설명 |
|:---|:---|
| **SAML** | XML 기반, 엔터프라이즈 |
| **OIDC** | JSON/JWT 기반, 웹/모바일 |
| **Kerberos** | 티켓 기반, 윈도우 |
| **CAS** | 대학/기관 |

**핵심 알고리즸:** 중앙 인증 → 토큰 발급 → 서비스 접근
