# NRZ (Non-Return to Zero)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비트 간 전압이 0으로 복귀하지 않는 부호화
> 2. **가치**: 대역폭 효율, 간단한 구현
> 3. **융합**: NRZ-L, NRZ-I, 직렬 통신과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NRZ(Non-Return to Zero)는 비트 전송 중 전압 레벨이 0으로 복귀하지 않고 유지되는 Line Coding 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NRZ (Non-Return to Zero)                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NRZ-L (Level)                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  규칙:                                                 │   │ │
│   │   │  • 0 = 낮은 레벨 (-V)                                  │   │ │
│   │   │  • 1 = 높은 레벨 (+V)                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  데이터:    1   0   1   1   0   0   1                │   │ │
│   │   │             ┌┐       ┌┐┌┐       ┌┐                  │   │ │
│   │   │           ──┘└───────┘└┘└───────┘└───                │   │ │
│   │   │             +V       -V          +V                   │   │ │
│   │   │                                                      │   │ │
│   │   │  문제: 연속 0이나 1에서 동기화 어려움                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NRZ-I (Invert)                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  규칙:                                                 │   │ │
│   │   │  • 1 = 레벨 전이 (반전)                                │   │ │
│   │   │  • 0 = 레벨 유지 (변화 없음)                           │   │ │
│   │   │                                                      │   │ │
│   │   │  데이터:    1   0   1   1   0   0   1                │   │ │
│   │   │             ┌┐       ┌┐   ┌───────┐                  │   │ │
│   │   │           ──┘└───────┘└───┘        └──                │   │ │
│   │   │             ↑       ↑ ↑             ↑                  │   │ │
│   │   │            전이     전이 전이       전이               │   │ │
│   │   │                                                      │   │ │
│   │   │  장점: 차등 부호화, 극성 반전에 강함                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: NRZ는 가장 간단하지만 동기화 문제가 있습니다.

**💡 비유**: NRZ-L은 스위치 상태, NRZ-I는 변화 횟수입니다.

**📢 섹션 요약 비유**: NRZ-L은 레벨, NRZ-I는 전이!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | NRZ-L | NRZ-I | 비고 |
|:---|:---|:---|:---|
| **0 표현** | -V | 유지 | 차등 |
| **1 표현** | +V | 전이 | 차등 |
| **동기화** | 어려움 | 어려움 | 연속 0 |
| **극성 반전** | 민감 | 강함 | NRZ-I 장점 |

**장단점 비교**

```
+------------+------------------+------------------+
| 항목       | NRZ-L            | NRZ-I            |
+------------+------------------+------------------+
| 구현       | 간단             | 약간 복잡        |
| 대역폭     | 낮음 (효율적)    | 낮음 (효율적)    |
| 동기화     | 어려움           | 어려움           |
| 극성 반전  | 영향 받음        | 영향 없음        |
| DC 성분    | 있음             | 있음             |
+------------+------------------+------------------+
```

**핵심 알고리즘**: 데이터 → 레벨/전이 매핑 → 전송

**📢 섹션 요약 비유**: USB는 NRZ-I 사용!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: Line Coding**

| 방식 | 대역폭 | 동기화 | 응용 |
|:---|:---:|:---:|:---|
| **NRZ** | 낮음 | 어려움 | 직렬 |
| **Manchester** | 높음 | 좋음 | 이더넷 |
| **4B/5B+NRZ** | 중간 | 좋음 | FDDI |

**📢 섹션 요약 비유**: 4B/5B로 NRZ 동기화 해결!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 직렬 포트** → NRZ-L
**시나리오 2: USB** → NRZ-I
**시나리오 3: 100BASE-TX** → 4B/5B + NRZ-I

**도입 체크리스트**: 동기화, 극성 반전, 속도

**📢 섹션 요약 비유**: 연속 비트 동기화엔 scrambling!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대역폭 효율 최고

**미래 전망**: PAM 방식으로 확장

**참고 표준**: USB, RS-232

**📢 섹션 요약 비유**: NRZ는 대역폭 효율의 기본!
