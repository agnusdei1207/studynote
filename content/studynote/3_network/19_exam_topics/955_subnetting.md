# Subnetting - 서브네팅

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 분할
> 2. **가치**: 효율적 주소 관리
> 3. **융합:** VLSM, CIDR, Network Design

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Subnetting은 네트워크를 더 작은 서브넷으로 분할합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Subnetting                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              서브네팅 과정                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  네트워크 분할:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  192.168.0.0/24 (256개 주소)                     ││   │ │
│   │   │  │           │                                     ││   │ │
│   │   │  │    ┌──────┴──────┐                              ││   │ │
│   │   │  │    │             │                              ││   │ │
│   │   │  │    ▼             ▼                              ││   │ │
│   │   │  │  192.168.0.0/25  192.168.0.128/25                ││   │ │
│   │   │  │  (128개)         (128개)                         ││   │ │
│   │   │  │    │             │                              ││   │ │
│   │   │  │  ┌─┴─┐        ┌─┴─┐                            ││   │ │
│   │   │  │  ▼   ▼        ▼   ▼                            ││   │ │
│   │   │  │ /26  /26     /26  /26                          ││   │ │
│   │   │  │ 64개 64개     64개 64개                          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  계산 공식:                                              │   │ │
│   │   │  - 서브넷 수 = 2^n (n = 빌린 비트)                       │   │ │
│   │   │  - 호스트 수 = 2^h - 2 (h = 호스트 비트)                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 서브네팅은 IP 주소를 효율적으로 관리합니다.

**📢 섹션 요약 비유:** Subnetting = 건물 층별 구분!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Network Bits** | 네트워크 비트 |
| **Subnet Bits** | 서브넷 비트 |
| **Host Bits** | 호스트 비트 |
| **Block Size** | 블록 크기 |
| **Magic Number** | 매직 넘버 |

**핵심 알고리듬:** 비트 차용 → 서브넷 계산 → 할당
