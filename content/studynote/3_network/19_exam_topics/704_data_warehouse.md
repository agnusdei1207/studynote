# Data Warehouse - 데이터 웨어하우스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분석 중앙 저장소
> 2. **가치**: 의사결정, BI
> 3. **융합:** ETL, OLAP, Data Lake

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Data Warehouse는 분석을 위한 중앙 저장소입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Data Warehouse                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DW 아키텍처                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Source Systems                                      │   │ │
│   │   │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                   │   │ │
│   │   │  │ ERP │ │ CRM │ │ Web │ │ IoT │                   │   │ │
│   │   │  └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘                   │   │ │
│   │   │     │       │       │       │                        │   │ │
│   │   │     └───────┼───────┼───────┘                        │   │ │
│   │   │             ↓                                       │   │ │
│   │   │  ┌─────────────────────────────────────┐            │   │ │
│   │   │  │         ETL (Extract-Transform-Load)│            │   │ │
│   │   │  └─────────────────────────────────────┘            │   │ │
│   │   │             ↓                                       │   │ │
│   │   │  ┌─────────────────────────────────────┐            │   │ │
│   │   │  │         Data Warehouse              │            │   │ │
│   │   │  │  ┌────────────────────────────────┐ │            │   │ │
│   │   │  │  │ Dimensional Model (Star/Snowflake)│           │   │ │
│   │   │  │  └────────────────────────────────┘ │            │   │ │
│   │   │  └─────────────────────────────────────┘            │   │ │
│   │   │             ↓                                       │   │ │
│   │   │  ┌─────────────────────────────────────┐            │   │ │
│   │   │  │    BI Tools │ Analytics │ Reports   │            │   │ │
│   │   │  └─────────────────────────────────────┘            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DW는 운영 데이터를 분석 가능한 형태로 저장합니다.

**📢 섹션 요약 비유:** Data Warehouse = 데이터 도서관!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **ETL** | 추출-변환-적재 |
| **Dimension** | 차원 |
| **Fact** | 팩트 |
| **Star Schema** | 성형 스키마 |
| **Snowflake Schema** | 눈송이 스키마 |

**핵심 알고리즸:** 추출 → 변환 → 적재 → 분석

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**DW vs OLTP**

| 항목 | DW | OLTP |
|:---|:---|:---|
| **목적** | 분석 | 운영 |
| **데이터** | 요약 | 상세 |
| **쓰기** | 적음 | 많음 |
| **읽기** | 복잡 | 단순 |
