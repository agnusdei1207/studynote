# NFV - 네트워크 기능 가상화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 기능의 가상화
> 2. **가치**: 하드웨어 독립성
> 3. **융합:** VNF, MANO, SDN

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NFV (Network Function Virtualization)는 네트워크 기능을 소프트웨어로 구현합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NFV                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NFV 아키텍처                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  전통적 네트워크 vs NFV:                                 │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  [전통적]              [NFV]                     ││   │ │
│   │   │  │  ┌─────┐ ┌─────┐      ┌─────────────────────┐  ││   │ │
│   │   │  │  │방화벽│ │  LB │      │    가상화 플랫폼      │  ││   │ │
│   │   │  │  │HW장비│ │HW장비│      │  ┌───┐ ┌───┐ ┌───┐ │  ││   │ │
│   │   │  │  └─────┘ └─────┘      │  │vFW│ │vLB│ │vR │ │  ││   │ │
│   │   │  │                       │  └───┘ └───┘ └───┘ │  ││   │ │
│   │   │  │  전용 하드웨어          │     VNF (가상 네트워크   ││   │ │
│   │   │  │                        │         기능)         ││   │ │
│   │   │  │                        └─────────────────────┘  ││   │ │
│   │   │  │  ┌─────┐ ┌─────┐      ┌─────────────────────┐  ││   │ │
│   │   │  │  │라우터│ │ IPS │      │    범용 서버 (x86)    │  ││   │ │
│   │   │  │  │HW장비│ │HW장비│      │                     │  ││   │ │
│   │   │  │  └─────┘ └─────┘      └─────────────────────┘  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  NFV MANO:                                             │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │  NFVO (오케스트레이터) → VNFM (VNF 관리자)        ││   │ │
│   │   │  │           ↓                                      ││   │ │
│   │   │  │       VIM (가상 인프라 관리자)                    ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: NFV는 전용 하드웨어를 범용 서버로 대체합니다.

**📢 섹션 요약 비유:** NFV = 네트워크 장비의 가상머신!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **VNF** | Virtual Network Function |
| **NFVI** | NFV Infrastructure |
| **MANO** | Management and Orchestration |
| **VIM** | Virtual Infrastructure Manager |
| **VNFM** | VNF Manager |

**핵심 알고리듬:** VNF 배포 → 스케일링 → 모니터링 → 복구

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**NFV vs SDN**

| 항목 | NFV | SDN |
|:---|:---|:---|
| **목표** | 기능 가상화 | 제어 중앙화 |
| **대상** | 네트워크 기능 | 네트워크 플로우 |
| **의존성** | 독립 가능 | 독립 가능 |
