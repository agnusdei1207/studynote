# Tunneling - 터널링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로토콜 캡슐화
> 2. **가치**: 네트워크 연결, 보안
> 3. **융합:** VPN, GRE, IP-in-IP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Tunneling은 한 프로토콜을 다른 프로토콜로 캡슐화합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Tunneling                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              터널링 구조                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Original Packet:                                    │   │ │
│   │   │  ┌─────────────────────────────────────┐            │   │ │
│   │   │  │ IP Header │ Payload                 │            │   │ │
│   │   │  └─────────────────────────────────────┘            │   │ │
│   │   │                          ↓                           │   │ │
│   │   │  Encapsulated Packet:                                 │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Outer IP │ Tunnel Header │ Inner IP │ Payload │   │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  터널링 프로토콜:                                     │   │ │
│   │   │  - GRE (Generic Routing Encapsulation)               │   │ │
│   │   │  - IP-in-IP                                          │   │ │
│   │   │  - 6to4, Teredo (IPv6)                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 터널링은 원래 패킷을 새 헤더로 감쌉니다.

**📢 섹션 요약 비유:** Tunneling = 봉투 속 봉투!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 프로토콜 | 설명 |
|:---|:---|
| **GRE** | Generic Routing Encapsulation |
| **IP-in-IP** | IP 안에 IP |
| **6to4** | IPv6 in IPv4 |
| **Teredo** | NAT 통과 IPv6 |

**핵심 알고리즸:** 캡슐화 → 전송 → 역캡슐화

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**터널링 용도**

| 용도 | 프로토콜 |
|:---|:---|
| **VPN** | IPsec, OpenVPN |
| **IPv6 전환** | 6to4, Teredo |
| **오버레이** | VXLAN, GRE |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
