# ping - 핑 (Packet InterNet Groper)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: ICMP Echo를 이용한 연결 테스트
> 2. **가치**: 네트워크 가용성 확인
> 3. **융합:** ICMP, RTT, TTL

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ping은 ICMP Echo로 상대방 도달 여부를 확인합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ping                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ping 동작                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  $ ping 8.8.8.8                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  Request:  Host A ── ICMP Echo ──→ Host B            │   │ │
│   │   │  Reply:    Host A ←── Echo Reply ── Host B           │   │ │
│   │   │                                                      │   │ │
│   │   │  64 bytes from 8.8.8.8: icmp_seq=0 ttl=117 time=5ms  │   │ │
│   │   │  64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=4ms  │   │ │
│   │   │  ...                                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  RTT, TTL, 패킷 손실 측정                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ping은 가장 기본적인 네트워크 진단 도구입니다.

**📢 섹션 요약 비유:** ping = 네트워크 하이!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **ICMP Type 8** | Echo Request |
| **ICMP Type 0** | Echo Reply |
| **icmp_seq** | 순서 번호 |
| **ttl** | 남은 홉 수 |
| **time** | RTT (ms) |

**핵심 알고리즸:** Echo Request → Echo Reply → RTT 계산

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ping 옵션**

| 옵션 | 설명 |
|:---|:---|
| **-c** | 횟수 지정 |
| **-i** | 간격 지정 |
| **-t** | TTL 지정 |
| **-s** | 패킷 크기 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 연결 확인 → ping
**시나리오 2:** 방화벽 → ICMP 차단될 수 있음
**시나리오 3:** MTU → ping -s 크기 지정
