# Flow Control - 흐름 제어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 송수신 속도 조절
> 2. **가치**: 수신자 보호
> 3. **융합:** TCP, Sliding Window, Backpressure

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Flow Control은 송수신 속도를 조절합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Flow Control                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              흐름 제어 메커니즘                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Stop-and-Wait:                                       │   │ │
│   │   │  Send → Wait ACK → Send → Wait ACK                   │   │ │
│   │   │  (비효율적)                                            │   │ │
│   │   │                                                      │   │ │
│   │   │  Sliding Window:                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Window = 4                                       ││   │ │
│   │   │  │ [1][2][3][4] 5  6  7  8                         ││   │ │
│   │   │  │  ↑  ↑  ↑  ↑                                     ││   │ │
│   │   │  │ 전송 가능                                        ││   │ │
│   │   │  │                                                  ││   │ │
│   │   │  │ ACK 1 수신 후:                                   ││   │ │
│   │   │  │ 1 [2][3][4][5] 6  7  8                          ││   │ │
│   │   │  │    ↑  ↑  ↑  ↑                                   ││   │ │
│   │   │  │    윈도우 이동                                    ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  TCP Flow Control:                                    │   │ │
│   │   │  - rwnd (Receiver Window): 수신자 버퍼 여유           │   │ │
│   │   │  - 송신자는 rwnd만큼만 전송                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 흐름 제어는 수신자 버퍼 오버플로우를 방지합니다.

**📢 섹션 요약 비유:** Flow Control = 물 조절!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Window** | 윈도우 |
| **ACK** | 확인 응답 |
| **rwnd** | 수신 윈도우 |
| **Buffer** | 버퍼 |
| **Zero Window** | 영 윈도우 |

**핵심 알고리즸:** rwnd 확인 → 윈도우 내 전송 → ACK 대기

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Flow Control vs Congestion Control**

| 항목 | Flow Control | Congestion Control |
|:---|:---|:---|
| **목적** | 수신자 보호 | 네트워크 보호 |
| **기준** | rwnd | cwnd |
| **주체** | 수신자 | 송신자 |
