# ACL (Access Control List) - 접근 제어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 패킷 필터링 규칙
> 2. **가치**: 보안, 트래픽 제어
> 3. **융합**: 방화벽, NAT, QoS와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ACL(Access Control List)은 패킷을 필터링하여 네트워크 접근을 제어하는 규칙 목록입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ACL (Access Control List)                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ACL 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  패킷 ───> [ACL 규칙 1] ───> Match? ───> Permit/Deny │   │ │
│   │   │              │               │                      │   │ │
│   │   │              │               No                     │   │ │
│   │   │              │               ↓                      │   │ │
│   │   │              │         [ACL 규칙 2] ───> Match?     │   │ │
│   │   │              │               │                      │   │ │
│   │   │              │               ...                    │   │ │
│   │   │              │               ↓                      │   │ │
│   │   │              │         [Implicit Deny]              │   │ │
│   │   │              │         (암시적 거부)                 │   │ │
│   │   │              │                                      │   │ │
│   │   │  순서대로 평가, 첫 번째 Match에서 종료              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ACL 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  유형    │ 번호 범위   │ 필터링 기준                 │   │ │
│   │   │  ───────┼────────────┼───────────────────────────│   │ │
│   │   │  Standard│ 1-99      │ 출발지 IP만                │   │ │
│   │   │  Extended│ 100-199   │ 출발지/목적지 IP, Port     │   │ │
│   │   │  Named   │ 이름       │ 확장 기능                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ACL은 위에서 아래로 순차적으로 평가되며, 암시적 Deny가 마지막에 있습니다.

**💡 비유**: ACL은 출입 명단입니다. 이름이 있으면 통과, 없으면 거부됩니다.

**📢 섹션 요약 비유:** ACL = 순서 중요!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | Standard | Extended |
|:---|:---|:---|
| **출발지** | O | O |
| **목적지** | X | O |
| **포트** | X | O |
| **프로토콜** | X | O |
| **번호** | 1-99 | 100-199 |

**비교표: 적용 방향**

```
+------------+----------+----------+
| 방향       │ 필터링   │ 용도     │
+------------+----------+----------+
| Inbound    │ 진입 전  │ 보안     │
| Outbound   │ 진출 전  │ 제어     │
+------------+----------+----------+
```

**핵심 알고리즸:** 순차 평가, 첫 번째 Match

**📢 섹션 요약 비유:** Implicit Deny = 마지막에 자동 거부!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: ACL vs 방화벽**

| 항목 | ACL | 방화벽 |
|:---|:---|:---|
| **상태** | 무상태 | 유상태 |
| **계층** | L3/L4 | L3-L7 |
| **성능** | 높음 | 중간 |

**📢 섹션 요약 비유:** ACL = 무상태, 방화벽 = 유상태!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 기본 보안** → Extended ACL
**시나리오 2: 라우팅 필터** → Standard ACL
**시나리오 3: 고급 보안** → 방화벽

**도입 체크리스트:** 순서, 방향, 로깅

**📢 섹션 요약 비유:** 구체적인 규칙 = 먼저!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 보안, 트래픽 제어

**미래 전망:** SDN 기반 정책

**참고 표준:** Cisco 전용

**📢 섹션 요약 비유:** ACL은 네트워크 보안의 기본!
