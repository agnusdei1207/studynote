# TIME_WAIT - 타임 웨이트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연결 종료 후 대기 상태
> 2. **가치**: 지연 패킷 처리, 신뢰성
> 3. **융합:** TCP Close, 2MSL, FIN

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TIME_WAIT는 연결 종료 후 2MSL 동안 대기합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TIME_WAIT                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TCP 연결 종료                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  클라이언트 (Active Close)                          │   │ │
│   │   │     │                                                 │   │ │
│   │   │     │ ─── FIN ──────────────→                        │   │ │
│   │   │     │ ←── ACK ───────────────                        │   │ │
│   │   │     │ ←── FIN ───────────────                        │   │ │
│   │   │     │ ─── ACK ──────────────→                        │   │ │
│   │   │     │                                                 │   │ │
│   │   │  TIME_WAIT (2MSL 대기)                               │   │ │
│   │   │     │                                                 │   │ │
│   │   │  CLOSED                                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: TIME_WAIT는 지연된 패킷이 새 연결에 영향주지 않도록 합니다.

**📢 섹션 요약 비유:** TIME_WAIT = 안전한 종료!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 값 | 설명 |
|:---|:---|:---|
| **2MSL** | 60초 | 2 × Maximum Segment Lifetime |
| **목적 1** | 지연 패킷 소멸 | 네트워크에서 사라짐 |
| **목적 2** | FIN ACK 보장 | 마지막 ACK 전달 |
| **문제** | 포트 고갈 | 많은 연결 시 |

**핵심 알고리즸:** active close → TIME_WAIT → 2MSL → CLOSED

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**TCP 종료 상태**

| 상태 | 위치 | 다음 |
|:---|:---|:---|
| **FIN_WAIT_1** | Active | ACK 대기 |
| **FIN_WAIT_2** | Active | FIN 대기 |
| **TIME_WAIT** | Active | 2MSL 대기 |
| **CLOSE_WAIT** | Passive | FIN 대기 |
| **LAST_ACK** | Passive | ACK 대기 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 서버 → Passive Close 권장
**시나리오 2:** 높은 동시 연결 → SO_REUSEADDR
**시나리오 3:** 짧은 MSL → 조정
