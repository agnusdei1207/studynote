# Anycast - 애니캐스트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 1:Nearest 통신
> 2. **가치**: 부하 분산과 고가용성
> 3. **융합:** DNS, CDN, BGP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Anycast는 가장 가까운 서버로 전달합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Anycast                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              애니캐스트 통신                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1:Nearest 통신:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │   Server A (Seoul)      Server B (Tokyo)        ││   │ │
│   │   │  │   203.0.113.10          203.0.113.10            ││   │ │
│   │   │  │        ▲                      ▲                 ││   │ │
│   │   │  │        │                      │                 ││   │ │
│   │   │  │        │    ┌─────────┐       │                 ││   │ │
│   │   │  │        └────│ Client  │───────┘                 ││   │ │
│   │   │  │             │ (Korea) │                         ││   │ │
│   │   │  │             └─────────┘                         ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  → 같은 IP, 여러 위치                              ││   │ │
│   │   │  │  → BGP가 가장 가까운 서버로 라우팅                    ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  용도:                                                  │   │ │
│   │   │  - DNS: 루트 서버, 권한 서버                              │   │ │
│   │   │  - CDN: 콘텐츠 배포                                     │   │ │
│   │   │  - DDoS 방어: 트래픽 분산                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
```

> **해설**: Anycast는 글로벌 서비스의 핵심입니다.

**📢 섹션 요약 비유:** Anycast = 가장 가까운 지점!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Same IP** | 동일 IP |
| **BGP** | 라우팅 |
| **Location** | 위치 |
| **Metric** | 메트릭 |
| **Failover** | 장애 조치 |

**핵심 알고리듬:** BGP 메트릭 → 가장 가까운 서버 선택

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Anycast vs Unicast**

| 항목 | Anycast | Unicast |
|:---|:---|:---|
| **IP** | 공유 | 고유 |
| **서버** | 여러 개 | 하나 |
| **선택** | 네트워크 | 클라이언트 |
