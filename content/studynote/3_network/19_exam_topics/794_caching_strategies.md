# Caching Strategies - 캐싱 전략

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 저장 전략
> 2. **가치**: 성능 최적화
> 3. **융합:** Cache-aside, Write-through, Write-back

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Caching Strategies는 캐시 사용 방식을 정의합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Caching Strategies                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              캐싱 전략 비교                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Cache-aside (Lazy Loading):                          │   │ │
│   │   │  App → Cache (miss) → DB → Cache → App              │   │ │
│   │   │  장점: 필요시만 로드 / 단점: 첫 조회 느림              │   │ │
│   │   │                                                      │   │ │
│   │   │  Write-through:                                       │   │ │
│   │   │  App → Cache + DB (동시) → 완료                       │   │ │
│   │   │  장점: 일관성 / 단점: 쓰기 느림                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Write-back (Write-behind):                           │   │ │
│   │   │  App → Cache → (나중에) DB                            │   │ │
│   │   │  장점: 빠른 쓰기 / 단점: 데이터 손실 가능              │   │ │
│   │   │                                                      │   │ │
│   │   │  Write-around:                                         │   │ │
│   │   │  App → DB (직접) / Cache (읽을 때)                    │   │ │
│   │   │  장점: 캐시 오염 방지 / 단점: 첫 조회 느림              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 캐싱 전략은 일관성과 성능의 균형입니다.

**📢 섹션 요약 비유:** Caching Strategy = 재고 관리!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Read-through** | 읽기 시 캐시 확인 |
| **Write-through** | 쓰기 시 캐시+DB 동시 |
| **Write-back** | 쓰기 시 캐시만 |
| **Write-around** | 쓰기 시 DB만 |
| **Invalidation** | 무효화 |

**핵심 알고리즸:** 읽기/쓰기 → 전략 선택 → 실행

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**전략 비교**

| 전략 | 일관성 | 성능 |
|:---|:---|:---|
| **Cache-aside** | 낮음 | 높음 |
| **Write-through** | 높음 | 중간 |
| **Write-back** | 낮음 | 높음 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 읽기 많음 → Cache-aside
**시나리오 2:** 일관성 중요 → Write-through
**시나리오 3:** 쓰기 많음 → Write-back
