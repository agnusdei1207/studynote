# Pipeline - 파이프라인

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어 처리 병렬화
> 2. **가치**: 처리량 향상
> 3. **융합:** CPU, Superscalar, Hazard

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Pipeline은 명령어 처리를 여러 단계로 나눕니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Pipeline                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              파이프라인 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Clock Cycle:                                        │   │ │
│   │   │  ┌─────┬─────┬─────┬─────┬─────┐                    │   │ │
│   │   │  │  1  │  2  │  3  │  4  │  5  │                    │   │ │
│   │   │  ├─────┼─────┼─────┼─────┼─────┤                    │   │ │
│   │   │  │ IF  │ ID  │ EX  │ MEM │ WB  │ Instruction 1       │   │ │
│   │   │  │     │ IF  │ ID  │ EX  │ MEM │ Instruction 2       │   │ │
│   │   │  │     │     │ IF  │ ID  │ EX  │ Instruction 3       │   │ │
│   │   │  │     │     │     │ IF  │ ID  │ Instruction 4       │   │ │
│   │   │  │     │     │     │     │ IF  │ Instruction 5       │   │ │
│   │   │  └─────┴─────┴─────┴─────┴─────┘                    │   │ │
│   │   │                                                      │   │ │
│   │   │  IF: Instruction Fetch                               │   │ │
│   │   │  ID: Instruction Decode                              │   │ │
│   │   │  EX: Execute                                         │   │ │
│   │   │  MEM: Memory Access                                  │   │ │
│   │   │  WB: Write Back                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 파이프라인은 각 단계를 병렬로 처리합니다.

**📢 섹션 요약 비유:** Pipeline = 조립 라인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **IF** | 명령어 인출 |
| **ID** | 명령어 해독 |
| **EX** | 실행 |
| **MEM** | 메모리 접근 |
| **WB** | 결과 저장 |

**핵심 알고리즸:** 명령어 → 단계별 처리 → 완료

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Pipeline Hazard**

| Hazard | 설명 |
|:---|:---|
| **Data Hazard** | 데이터 의존성 |
| **Control Hazard** | 분기 |
| **Structural Hazard** | 자원 충돌 |
