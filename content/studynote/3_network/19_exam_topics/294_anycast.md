### Ⅰ. 개요 (Context & Background)

**개념 정의**

Anycast는 동일한 IP 주소를 여러 위치에 할당하고, 라우팅 프로토콜이 가장 가까운 노드로 연결합니다.

**💡 비유**: Anycast는 하나의 전화번호에 여러 상담원이 응답합니다.

**📢 섹션 요약 비유:** Anycast = 라우팅 최적화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **동일 IP** | 여러 노드 |
| **BGP** | 라우팅 |
| **ECMP** | 다중 경로 |
| **Health** | 상태 확인 |

**비교표: Anycast vs Unicast**

```
+------------+----------+----------+
| 항목       │ Unicast  │ Anycast  │
+------------+----------+----------+
| IP         │ 1:1      │ 1:N      │
| 라우팅     │ 고정     │ 동적     │
| 장애 조치  │ 수동     │ 자동     │
| 확장       │ 제한     │ 용이     │
+------------+----------+----------+
```

**핵심 알고리즸:** BGP 최단 경로, ECMP

**📢 섹션 요약 비유:** BGP = 경로 선택!
