# SRv6 (세그먼트 라우팅 IPv6) - Segment Routing over IPv6

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPv6 기반 세그먼트 라우팅
> 2. **가치**: 프로그래밍 가능, 서비스 체이닝
> 3. **융합**: SRH, End SID, SFC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SRv6는 IPv6를 사용한 세그먼트 라우팅입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SRv6 아키텍처                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SRv6 패킷 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ IPv6 Header                                  │    │   │ │
│   │   │  │ SA: Source IPv6                              │    │   │ │
│   │   │  │ DA: First Segment                            │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ SRH (Segment Routing Header)               │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  Segments: [SID3, SID2, SID1]              │    │   │ │
│   │   │  │  (역순으로 저장)                            │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  Next Header │ Hdr Ext Len │ ...          │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SRv6 SID 유형                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ SID        │ 동작     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ End        │ 라우팅   │ 노드     │               │   │ │
│   │   │  │ End.X      │ 인접     │ 링크     │               │   │ │
│   │   │  | End.T      │ 조회     │ 테이블   │               │   │ │
│   │   │  │ End.DX2    │ L2       │ VPN      │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SRv6는 IPv6 확장 헤더로 세그먼트를 전달합니다.

**💡 비유**: SRv6는 IPv6의 프로그래밍 가능 네트워크입니다.

**📢 섹션 요약 비유:** SRv6 = IPv6 + SR!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SRH** | Segment Routing Header |
| **SID** | 128비트 식별자 |
| **End Function** | 노드 동작 |
| **Transit Function** | 전달 동작 |

**비교표: SRv6 vs SR-MPLS**

```
+------------+----------+----------+----------+
| 항목       │ SRv6     │ SR-MPLS  │ 비고     │
+------------+----------+----------+----------+
| 데이터     │ IPv6     │ MPLS     │ 캡슐화   │
| SID        │ 128bit   │ 20bit    │ 크기     │
| 확장       │ 높음     │ 낮음     │ 유연성   │
| 오버헤드   │ 높음     │ 낮음     │ 헤더     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** SID 처리, SL 업데이트

**📢 섹션 요약 비유:** End SID = 노드!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 활용 사례**

| 사례 | 설명 | 이점 |
|:---|:---|:---|
| **SFC** | 서비스 체이닝 | 유연성 |
| **VPN** | L2/L3 VPN | 단순 |
| **TE** | 트래픽 엔지니어링 | 정책 |

**📢 섹션 요약 비유:** SFC = 서비스 체이닝!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 백본 → SRv6 TE
**시나리오 2:** 데이터센터 → SRv6 SFC

**도입 체크리스트:** IPv6, IGP, 컨트롤러

**📢 섹션 요약 비유:** IPv6 필수!
