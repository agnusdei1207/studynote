# ARQ - 자동 재전송 요청

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오류 시 재전송
> 2. **가치**: 신뢰성 확보
> 3. **융합:** Stop-and-Wait, Go-Back-N, Selective Repeat

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ARQ (Automatic Repeat reQuest)는 오류 감지 시 재전송을 요청합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ARQ (Automatic Repeat reQuest)                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ARQ 방식 비교                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Stop-and-Wait:                                        │   │ │
│   │   │  Sender                    Receiver                  │   │ │
│   │   │  ┌─────┐                  ┌─────┐                   │   │ │
│   │   │  │     │ ── Frame 1 ────→ │     │                   │   │ │
│   │   │  │     │ ←── ACK 1 ────── │     │                   │   │ │
│   │   │  │     │ ── Frame 2 ────→ │     │                   │   │ │
│   │   │  │     │ ←── ACK 2 ────── │     │                   │   │ │
│   │   │  └─────┘                  └─────┘                   │   │ │
│   │   │  효율 낮음                                            │   │ │
│   │   │                                                      │   │ │
│   │   │  Go-Back-N:                                            │   │ │
│   │   │  ┌─────┐                  ┌─────┐                   │   │ │
│   │   │  │     │ ── 1,2,3,4,5 ──→ │     │                   │   │ │
│   │   │  │     │ ←── ACK 1,2 ──── │     │                   │   │ │
│   │   │  │     │ (3 손실)         │     │                   │   │ │
│   │   │  │     │ ── 3,4,5 ──────→ │     │ ← 3부터 재전송    │   │ │
│   │   │  └─────┘                  └─────┘                   │   │ │
│   │   │                                                      │   │ │
│   │   │  Selective Repeat:                                     │   │ │
│   │   │  ┌─────┐                  ┌─────┐                   │   │ │
│   │   │  │     │ ── 1,2,3,4,5 ──→ │     │                   │   │ │
│   │   │  │     │ (3 손실)         │     │                   │   │ │
│   │   │  │     │ ←── NAK 3 ────── │     │                   │   │ │
│   │   │  │     │ ── 3 ──────────→ │     │ ← 3만 재전송      │   │ │
│   │   │  └─────┘                  └─────┘                   │   │ │
│   │   │                                                      │   │ │
│   │   │  효율 비교:                                            │   │ │
│   │   │  Stop-and-Wait < Go-Back-N < Selective Repeat         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ARQ는 신뢰적 데이터 전송의 기본 메커니즘입니다.

**📢 섹션 요약 비유:** ARQ = 재주문!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **ACK** | 확인 응답 |
| **NAK** | 부정 확인 |
| **Timeout** | 타임아웃 |
| **Sequence** | 순서 번호 |
| **Window** | 윈도우 |

**핵심 알고리즸:** 전송 → ACK 대기 → 재전송/다음

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ARQ 방식 특성**

| 방식 | 윈도우 | 수신 버퍼 | 재전송 |
|:---|:---|:---|:---|
| **Stop-Wait** | 1 | 1 | 전체 |
| **Go-Back-N** | N | 1 | 이후 전체 |
| **Selective** | N | N | 선택적 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 단순 → Stop-and-Wait
**시나리오 2:** 일반 → Go-Back-N
**시나리오 3:** 고속 → Selective Repeat
