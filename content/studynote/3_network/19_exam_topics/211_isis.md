# IS-IS (Intermediate System to Intermediate System) - ISP 라우팅

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: OSI 링크 상태 라우팅 프로토콜
> 2. **가치**: ISP 대규모 네트워크에 적합
> 3. **융합**: MPLS, Segment Routing과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IS-IS(Intermediate System to Intermediate System)는 OSI 참조 모델의 링크 상태 라우팅 프로토콜로, ISP에서 널리 사용됩니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IS-IS (Intermediate System)                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IS-IS 특징                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Link State Protocol:                             │   │ │
│   │   │     • Dijkstra SPF 알고리즘                          │   │ │
│   │   │     • LSP (Link State PDU)                           │   │ │
│   │   │                                                      │   │ │
│   │   │  2. OSI 계층:                                        │   │ │
│   │   │     • L2 (Data Link) 직접 동작                       │   │ │
│   │   │     • IP 독립적                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 계층 구조:                                        │   │ │
│   │   │     • Level 1: 영역 내                               │   │ │
│   │   │     • Level 2: 영역 간                               │   │ │
│   │   │     • Level 1-2: 둘 다                               │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 용어:                                             │   │ │
│   │   │     • IS: Intermediate System (라우터)               │   │ │
│   │   │     • ES: End System (호스트)                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IS-IS vs OSPF 용어                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IS-IS              │ OSPF                          │   │ │
│   │   │  ──────────────────┼────────────────────────────│   │ │
│   │   │  IS (Router)       │ Router                        │   │ │
│   │   │  LSP               │ LSA                           │   │ │
│   │   │  Area              │ Area                          │   │ │
│   │   │  Level 1           │ Intra-Area                    │   │ │
│   │   │  Level 2           │ Inter-Area                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IS-IS는 L2에서 직접 동작하여 IP와 무관하게 라우팅합니다.

**💡 비유**: IS-IS는 OSPF의 ISP 버전입니다. 더 확장 가능합니다.

**📢 섹션 요약 비유:** IS-IS = ISP 표준!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 라우터 유형 | 역할 | 비고 |
|:---|:---|:---|
| **L1** | 영역 내 | Stub |
| **L2** | 영역 간 | Backbone |
| **L1-L2** | 둘 다 | ABR 역할 |

**비교표: IS-IS vs OSPF**

```
+------------+----------+----------+
| 항목       │ IS-IS    │ OSPF     │
+------------+----------+----------+
| 계층       │ L2       │ IP       │
| 프로토콜   │ CLNP     │ IP 89    │
| 확장성     │ 높음     │ 중간     │
| TLV       │ 지원     │ 제한     │
| 용도       │ ISP      │ 기업     │
+------------+----------+----------+
```

**핵심 알고리즸:** SPF, LSP Flooding

**📢 섹션 요약 비유:** TLV = 확장성 핵심!
