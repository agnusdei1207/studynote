# TCP Options - TCP 옵션

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: TCP 기능 확장
> 2. **가치**: 성능 최적화, 고급 기능
> 3. **융합:** MSS, SACK, Window Scaling, Timestamp

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TCP Options는 SYN 단계에서 협상됩니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TCP Options                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 TCP 옵션                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  옵션 2: MSS (Maximum Segment Size)                 │   │ │
│   │   │  옵션 3: Window Scaling                             │   │ │
│   │   │  옵션 4: SACK Permitted                             │   │ │
│   │   │  옵션 5: SACK                                       │   │ │
│   │   │  옵션 8: Timestamp                                  │   │ │
│   │   │  옵션 0: End of Options                             │   │ │
│   │   │  옵션 1: No Operation (NOP)                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SYN 패킷에서 옵션을 교환하여 기능을 협상합니다.

**📢 섹션 요약 비유:** TCP Options = 추가 기능!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 옵션 | 번호 | 설명 |
|:---|:---|:---|
| **MSS** | 2 | 최대 세그먼트 크기 |
| **Window Scale** | 3 | 윈도우 확장 |
| **SACK-Permitted** | 4 | SACK 지원 |
| **SACK** | 5 | SACK 블록 |
| **Timestamp** | 8 | RTT 측정 |

**핵심 알고리즸:** SYN에서 협상 → 연결 전체 적용
