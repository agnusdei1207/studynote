# IPv4 vs IPv6 - 인터넷 프로토콜 비교

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 32비트 vs 128비트 주소
> 2. **가치**: 주소 공간, 헤더 효율
> 3. **융합**: 듀얼 스택, 터널링, NAT

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IPv4와 IPv6는 인터넷 프로토콜의 주요 버전입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IPv4 vs IPv6                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주소 비교                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IPv4: 192.168.1.1                                   │   │ │
│   │   │        32비트, 43억 주소                             │   │ │
│   │   │                                                      │   │ │
│   │   │  IPv6: 2001:0db8:85a3:0000:0000:8a2e:0370:7334     │   │ │
│   │   │        128비트, 340간 주소                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 차이점                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ IPv4     │ IPv6     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 주소길이   │ 32비트   │ 128비트  │               │   │ │
│   │   │  │ 헤더       │ 복잡     │ 단순     │               │   │ │
│   │   │  | NAT        │ 필수     │ 불필요   │               │   │ │
│   │   │  │ 브로드캐스트│ 있음     │ 없음     │               │   │ │
│   │   │  │ 보안       │ 선택     │ 기본     │               │   │ │
│   │   │  │ 자동설정   │ DHCP     │ SLAAC    │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IPv6는 IPv4의 주소 고갈 문제를 해결합니다.

**💡 비유**: IPv4는 2억 자리 전화번호, IPv6는 무제한 전화번호입니다.

**📢 섹션 요약 비유:** IPv6 = 미래!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | IPv4 | IPv6 |
|:---|:---|:---|
| **헤더** | 가변 옵션 | 고정 40바이트 |
| **Checksum** | 있음 | 없음 |
| **Fragmentation** | 라우터 | 송신자만 |
| **QoS** | ToS | Traffic Class |

**비교표: 전환 기술**

```
+------------+----------+----------+
| 항목       │ 듀얼스택 │ 터널링   │
+------------+----------+----------+
| 방식       │ 병행     │ 캡슐화   │
| 복잡도     │ 낮음     │ 중간     │
| 호환성     │ 높음     │ 제한     │
+------------+----------+----------+
```

**핵심 알고리즸:** SLAAC, Neighbor Discovery

**📢 섹션 요약 비유:** 듀얼 스택 = 전환!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 주소 유형**

| 항목 | IPv4 | IPv6 |
|:---|:---|:---|
| **유니캐스트** │ 있음 │ GUA/ULA/LLA |
| **멀티캐스트** │ 있음 │ 있음 |
| **브로드캐스트** │ 있음 │ 없음 |
| **애니캐스트** │ 제한 | 있음 |

**📢 섹션 요약 비유:** IPv6 주소 = 풍부!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 신규 서비스 → IPv6 우선
**시나리오 2:** 기존 인프라 → 듀얼 스택
**시나리오 3:** IoT → IPv6 (주소 풍부)

**도입 체크리스트:** 듀얼 스택, DNS AAAA, 보안

**📢 섹션 요약 비유:** 천천히 = IPv6 전환!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 주소 풍부, 헤더 효율

**미래 전망:** IPv6 Only 네트워크 확대

**참고 표준:** RFC 791 (v4), RFC 8200 (v6)

**📢 섹션 요약 비유:** IPv6는 인터넷의 미래!
