# 대칭키 암호 - Symmetric Encryption

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동일한 키로 암호화/복호화
> 2. **가치**: 빠른 속도, 효율성
> 3. **융합**: TLS, VPN, 디스크 암호화

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

대칭키 암호는 동일한 키로 암호화와 복호화를 수행합니다.


> **해설**: 대칭키 암호는 빠르지만 키 분배가 문제입니다.

**💡 비유**: 대칭키는 자물쇠 키입니다. 같은 키로 잠그고 엽니다.

**📢 섹션 요약 비유:** 대칭키 = 같은 키!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Block Cipher** | 블록 단위 암호화 |
| **Stream Cipher** | 비트 단위 암호화 |
| **Mode of Operation** | 운영 모드 |
| **IV** | 초기화 벡터 |

**비교표: 블록 vs 스트림**

```
+------------+----------+----------+
| 항목       │ 블록     │ 스트림   │
+------------+----------+----------+
| 단위       │ 블록     │ 비트     │
| 패딩       │ 필요     │ 불필요   │
| 속도       │ 중간     │ 빠름     │
| 예시       │ AES      │ ChaCha20 │
+------------+----------+----------+
```

**핵심 알고리즸:** 치환, 전치, XOR

**📢 섹션 요약 비유:** AES = 대칭키 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 암호 모드**

| 항목 | ECB | CBC | GCM |
|:---|:---|:---|:---|
| **병렬** │ 가능 │ 불가 │ 가능 |
| **인증** │ 없음 │ 없음 │ 있음 |
| **보안** │ 낮음 │ 중간 │ 높음 |

**📢 섹션 요약 비유:** GCM = 권장!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** TLS 통신 → AES-GCM
**시나리오 2:** 파일 암호화 → AES-256
**시나리오 3:** 모바일 → ChaCha20

**도입 체크리스트:** 키 길이, 모드 선택, IV 관리

**📢 섹션 요약 비유:** 대용량 = 대칭키!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 빠른 암호화
