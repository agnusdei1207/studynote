# ARQ (자동 재전송 요청)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오류 검출 시 송신측에 재전송 요청
> 2. **가치**: 신뢰성, 구현 용이, 양방향 채널
> 3. **융합**: TCP, HDLC, 무선 프로토콜과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ARQ(Automatic Repeat reQuest)는 수신측이 오류를 검출하면 송신측에 재전송을 요청하는 오류 제어 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ARQ (자동 재전송 요청)                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ARQ 종류                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. Stop-and-Wait ARQ:                                 │   │ │
│   │   │     • 프레임 전송 → ACK 대기 → 다음 프레임             │   │ │
│   │   │     • 단순하지만 비효율                                 │   │ │
│   │   │     • 채널 이용률 낮음                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Go-Back-N ARQ:                                     │   │ │
│   │   │     • 윈도우 크기 N 연속 전송                           │   │ │
│   │   │     • 오류 발생 시 N개 모두 재전송                      │   │ │
│   │   │     • 수신 버퍼 불필요                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Selective Repeat ARQ:                              │   │ │
│   │   │     • 오류 프레임만 선택적 재전송                        │   │ │
│   │   │     • 수신 버퍼로 순서 재조립                            │   │ │
│   │   │     • 최고 효율                                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Stop-and-Wait 타이밍                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  송신측: [Frame 0] ──────── [Frame 1] ────────       │   │ │
│   │   │               │              │                       │   │ │
│   │   │               ▼              ▼                       │   │ │
│   │   │  수신측: [ACK 0] ─────────── [ACK 1] ─────────       │   │ │
│   │   │                                                      │   │ │
│   │   │  전송 시간 + 왕복 지연 + ACK 시간 = 한 사이클          │   │ │
│   │   │                                                      │   │ │
│   │   │  채널 효율 = (프레임 크기) / (프레임 + RTT + ACK)     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: Selective Repeat이 효율은 최고지만 구현이 복잡합니다.

**💡 비유**: ARQ는 택배 분실 시 재배송 요청과 같습니다.

**📢 섹션 요약 비유**: Selective Repeat이 효율 최고!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **ACK** | 긍정 확인 | 정상 수신 | 영수증 |
| **NAK** | 부정 확인 | 오류 통보 | 반품 |
| **Window** | 윈도우 | 전송 개수 | 한도 |
| **Timer** | 타이머 | 재전송 | 알람 |

**윈도우 크기 제한**

| ARQ | 윈도우 | 재전송 |
|:---|:---:|:---|
| **Stop-Wait** | 1 | 전체 |
| **Go-Back-N** | 2^n-1 | N개 |
| **Selective** | 2^(n-1) | 1개 |

**핵심 알고리즘**: 전송 → 타이머 → ACK/Timeout → 재전송/다음

**📢 섹션 요약 비유**: Timer가 만료되면 재전송!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: ARQ 효율**

| ARQ | 대역폭 효율 | 구현 |
|:---|:---:|:---:|
| **Stop-Wait** | 낮음 | 쉬움 |
| **Go-Back-N** | 중간 | 중간 |
| **Selective** | 높음 | 어려움 |

**📢 섹션 요약 비유**: TCP는 Selective Repeat 기반!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 위성** → Stop-Wait (RTT 큼)
**시나리오 2: LAN** → Selective Repeat
**시나리오 3: 무선** → HARQ (FEC + ARQ)

**도입 체크리스트**: RTT, 오류율, 버퍼

**📢 섹션 요약 비유**: HARQ가 무선 표준!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 신뢰성 99.9%+

**미래 전망**: Cross-Layer ARQ

**참고 표준**: TCP, HDLC, LTE

**📢 섹션 요약 비유**: ARQ는 신뢰성의 기본!
