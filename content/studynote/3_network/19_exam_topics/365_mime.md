# MIME - 인터넷 미디어 타입

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 미디어 콘텐츠 형식 표준
> 2. **가치**: 이메일, HTTP, 파일 식별
> 3. **융합**: 이메일, 웹, API

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MIME(Multipurpose Internet Mail Extensions)는 미디어 콘텐츠 형식을 정의합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MIME                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MIME 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  MIME 헤더                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  Content-Type: text/html; charset=utf-8             │   │ │
│   │   │  Content-Transfer-Encoding: base64                  │   │ │
│   │   │  Content-Disposition: attachment; filename="x.pdf"  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MIME 타입 예시                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 타입       │ 서브타입 │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ text       │ html     │ 웹       │               │   │ │
│   │   │  │ image      │ jpeg     │ 이미지   │               │   │ │
│   │   │  | video      │ mp4      │ 비디오   │               │   │ │
│   │   │  │ application│ json     │ API      │               │   │ │
│   │   │  │ multipart  │ mixed    │ 복합     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: MIME은 타입/서브타입으로 미디어 형식을 식별합니다.

**💡 비유**: MIME은 파일의 신분증입니다. 무슨 파일인지 알려줍니다.

**📢 섹션 요약 비유:** MIME = 콘텐츠 타입!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Type** | 주 타입 |
| **Subtype** | 부 타입 |
| **Charset** | 문자 인코딩 |
| **Boundary** | 멀티파트 구분 |

**비교표: 전송 인코딩**

```
+------------+----------+----------+
| 인코딩     │ 설명     │ 용도     │
+------------+----------+----------+
| 7bit       │ ASCII    │ 기본     │
| base64     │ 이진     │ 첨부     │
| quoted-printable│ 텍스트│ 특수문자│
+------------+----------+----------+
```

**핵심 알고리즸:** Base64, Quoted-Printable

**📢 섹션 요약 비유:** application/json = API!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: MIME 사용처**

| 항목 | 이메일 | HTTP | 파일 |
|:---|:---|:---|:---|
| **MIME** │ 필수 │ 필수 │ 선택 |
| **인코딩** │ 필요 │ 없음 │ 없음 |

**📢 섹션 요약 비유:** 웹 = MIME!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 이메일 첨부 → multipart/mixed
**시나리오 2:** API → application/json
**시나리오 3:** 웹 → text/html

**도입 체크리스트:** 타입 지정, 인코딩, 보안

**📢 섹션 요약 비유:** Content-Type = 필수!
