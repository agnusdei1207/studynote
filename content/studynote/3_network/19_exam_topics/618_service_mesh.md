# Service Mesh - 서비스 메시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 마이크로서비스 간 통신 인프라
> 2. **가치**: 관측성, 보안, 트래픽 제어
> 3. **융합:** Istio, Envoy, Sidecar

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Service Mesh는 서비스 간 통신을 투명하게 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Service Mesh                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Service Mesh 구성                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │           Control Plane (Istiod)                     │   │ │
│   │   │           ┌─────────────────────┐                    │   │ │
│   │   │           │ Config │ CA │ Discovery │                 │   │ │
│   │   │           └─────────────────────┘                    │   │ │
│   │   │                    ↓                                 │   │ │
│   │   │           Data Plane (Envoy Sidecar)                  │   │ │
│   │   │  ┌─────────┐    ┌─────────┐    ┌─────────┐          │   │ │
│   │   │  │ Service │    │ Service │    │ Service │          │   │ │
│   │   │  │   +     │    │   +     │    │   +     │          │   │ │
│   │   │  │ Envoy   │←──→│ Envoy   │←──→│ Envoy   │          │   │ │
│   │   │  └─────────┘    └─────────┘    └─────────┘          │   │ │
│   │   │                                                      │   │ │
│   │   │  기능:                                               │   │ │
│   │   │  - mTLS (보안)                                       │   │ │
│   │   │  - Load Balancing                                    │   │ │
│   │   │  - Circuit Breaking                                  │   │ │
│   │   │  - Observability                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Service Mesh는 Sidecar 패턴으로 통신을 제어합니다.

**📢 섹션 요약 비유:** Service Mesh = 교통 관제!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Control Plane** | 정책 관리 |
| **Data Plane** | 트래픽 처리 |
| **Sidecar** | 프록시 |
| **mTLS** | 상호 TLS |

**핵심 알고리즸:** 정책 배포 → Sidecar 적용 → 트래픽 제어
