# Timeout - 타임아웃

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 대기 시간 제한
> 2. **가치**: 무한 대기 방지
> 3. **융합:** RTO, Connection Timeout, Retry

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Timeout은 대기 시간의 상한입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Timeout                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              타임아웃 유형                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  TCP RTO (Retransmission Timeout):                    │   │ │
│   │   │  - ACK 미수신 시 재전송 대기 시간                       │   │ │
│   │   │  - RTO = SRTT + 4×RTTVAR                              │   │ │
│   │   │  - Exponential Backoff                                │   │ │
│   │   │                                                      │   │ │
│   │   │  Connection Timeout:                                   │   │ │
│   │   │  - 연결 수립 대기 시간                                  │   │ │
│   │   │  - 기본값: OS마다 다름 (보통 60-120초)                   │   │ │
│   │   │                                                      │   │ │
│   │   │  Read/Write Timeout:                                   │   │ │
│   │   │  - 데이터 송수신 대기 시간                              │   │ │
│   │   │                                                      │   │ │
│   │   │  Keep-Alive Timeout:                                   │   │ │
│   │   │  - 유휴 연결 유지 시간                                  │   │ │
│   │   │  - HTTP: 기본 60초                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  타임아웃 발생 시:                                       │   │ │
│   │   │  - 재시도 (Retry)                                     │   │ │
│   │   │  - 연결 종료                                          │   │ │
│   │   │  - 에러 반환                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 타임아웃은 시스템 안정성을 위한 필수 메커니즘입니다.

**📢 섹션 요약 비유:** Timeout = 알람 시계!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RTO** | 재전송 타임아웃 |
| **SRTT** | 평균 RTT |
| **RTTVAR** | RTT 분산 |
| **Backoff** | 백오프 |
| **Retry** | 재시도 |

**핵심 알고리즸:** RTO = SRTT + 4×RTTVAR

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**타임아웃 설정값**

| 유형 | 일반값 |
|:---|:---|
| **TCP RTO** | 1초~120초 |
| **Connect** | 30-60초 |
| **Read** | 30-60초 |
| **Keep-Alive** | 60-7200초 |
