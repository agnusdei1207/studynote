# Checksum - 체크섬

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 합계 기반 검증
> 2. **가치**: 간단한 오류 감지
> 3. **융합:** IP Checksum, TCP Checksum, UDP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Checksum은 데이터 합계로 오류를 감지합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Checksum                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              체크섬 계산                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IP Checksum 계산:                                     │   │ │
│   │   │  1. 헤더를 16비트 단위로 분할                           │   │ │
│   │   │  2. 모두 더함 (캐리 포함)                               │   │ │
│   │   │  3. 캐리를 다시 더함                                    │   │ │
│   │   │  4. 1의 보수 취함                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  예시:                                                 │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ 4500 0073 0000 4000 4011                        ││   │ │
│   │   │  │ + b861 0a0a 0a0a                                 ││   │ │
│   │   │  │ ─────────────                                    ││   │ │
│   │   │  │ = xxxxx (합계)                                   ││   │ │
│   │   │  │ → 캐리 추가                                       ││   │ │
│   │   │  │ → 1의 보수 = Checksum                            ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  검증:                                                 │   │ │
│   │   │  모든 워드 합 + Checksum = 0xFFFF                      │   │ │
│   │   │                                                      │   │ │
│   │   │  용도:                                                 │   │ │
│   │   │  - IP Header Checksum                                 │   │ │
│   │   │  - TCP Checksum (의사 헤더 포함)                        │   │ │
│   │   │  - UDP Checksum (선택적)                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 체크섬은 IP/TCP/UDP에서 기본 오류 감지 방법입니다.

**📢 섹션 요약 비유:** Checksum = 숫자 합계 확인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **16-bit Sum** | 16비트 합 |
| **Carry** | 캐리 |
| **1's Complement** | 1의 보수 |
| **Pseudo Header** | 의사 헤더 |
| **Verification** | 검증 |

**핵심 알고리즸:** 합계 → 캐리 처리 → 1의 보수

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**오류 감지 방식 비교**

| 방식 | 크기 | 검출력 | 속도 |
|:---|:---|:---|:---|
| **Checksum** | 16비트 | 중간 | 빠름 |
| **CRC-32** | 32비트 | 높음 | 중간 |
| **MD5** | 128비트 | 매우 높음 | 느림 |
| **SHA-256** | 256비트 | 최고 | 느림 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** IP → Header Checksum
**시나리오 2:** TCP → Full Checksum
**시나리오 3:** UDP → 선택적 Checksum
