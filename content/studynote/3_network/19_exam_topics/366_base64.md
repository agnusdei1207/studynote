# Base64 - 이진 인코딩

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이진 데이터를 텍스트로 인코딩
> 2. **가치**: 이메일, JSON, URL 안전 전송
> 3. **융합**: MIME, JWT, 인증

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Base64는 이진 데이터를 ASCII 텍스트로 인코딩합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Base64                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Base64 인코딩                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  원본: "Hello"                                      │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 이진: 01001000 01100101 01101100 01101100 01101111│   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  6비트 그룹화: 010010 000110 010101 101100 011011 000110...│   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  Base64: "SGVsbG8="                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Base64 변환표                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 값         │ 문자     │ 예시     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 0-25       │ A-Z      │ 대문자   │               │   │ │
│   │   │  │ 26-51      │ a-z      │ 소문자   │               │   │ │
│   │   │  | 52-61      │ 0-9      │ 숫자     │               │   │ │
│   │   │  │ 62         │ +        │ 기호     │               │   │ │
│   │   │  │ 63         │ /        │ 기호     │               │   │ │
│   │   │  │ padding    │ =        │ 패딩     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Base64는 6비트 단위로 문자를 매핑합니다.

**💡 비유**: Base64는 번역기입니다. 이진을 텍스트로 번역합니다.

**📢 섹션 요약 비유:** Base64 = 이진→텍스트!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Character Set** | A-Z, a-z, 0-9, +, / |
| **Padding** | = |
| **URL Safe** | -, _ |

**비교표: Base64 변형**

```
+------------+----------+----------+
| 변형       │ 문자     │ 용도     │
+------------+----------+----------+
| Standard   │ +/       │ 일반     │
| URL Safe   │ -_       │ URL      │
| Base64url  │ -_       │ JWT      │
+------------+----------+----------+
```

**핵심 알고리즸:** 3바이트 → 4문자

**📢 섹션 요약 비유:** 33% 크기 증가!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 인코딩**

| 항목 | Base64 | Hex | ASCII |
|:---|:---|:---|:---|
| **크기** │ +33% │ +100% │ 0% |
| **범위** │ 전체 │ 전체 │ 제한 |

**📢 섹션 요약 비유:** 이진 전송 = Base64!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 이메일 첨부 → Base64
**시나리오 2:** JWT → Base64url
**시나리오 3:** 이미지 인라인 → Data URL + Base64

**도입 체크리스트:** 크기 증가, URL 안전성

**📢 섹션 요약 비유:** 텍스트 전송 = Base64!
