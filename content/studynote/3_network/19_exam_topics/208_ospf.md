# OSPF (Open Shortest Path First) - 링크 상태 라우팅

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 링크 상태 기반 최단 경로 계산
> 2. **가치**: 빠른 컨버전스, 확장성
> 3. **융합**: 영역 분할, BGP와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

OSPF(Open Shortest Path First)는 링크 상태 알고리즘을 사용하여 최단 경로를 계산하는 IGP입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OSPF (Open Shortest Path First)                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OSPF 특징                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Link State Algorithm:                            │   │ │
│   │   │     • Dijkstra SPF 알고리즸                          │   │ │
│   │   │     • 전체 토폴로지 파악                              │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 계층적 구조:                                      │   │ │
│   │   │     • Area 0 (Backbone) 필수                         │   │ │
│   │   │     • 영역 간 라우팅                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 빠른 컨버전스:                                    │   │ │
│   │   │     • 이벤트 기반 업데이트                            │   │ │
│   │   │     • SPF 지연                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 메트릭: Cost = 10^8 / 대역폭(bps)               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OSPF 영역 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │              [ Area 0 - Backbone ]                   │   │ │
│   │   │                    ↑                                 │   │ │
│   │   │         ┌─────────┼─────────┐                       │   │ │
│   │   │         ↓         ↓         ↓                       │   │ │
│   │   │    [Area 1]   [Area 2]   [Area 3]                    │   │ │
│   │   │    Standard   Stub      NSSA                        │   │ │
│   │   │                                                      │   │ │
│   │   │  • ABR: Area Border Router                          │   │ │
│   │   │  • ASBR: Autonomous System Boundary Router          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: OSPF는 Area 0을 중심으로 계층적으로 구성됩니다.

**💡 비유**: OSPF는 도시 지도입니다. 모든 도로 정보를 알고 최단 경로를 찾습니다.

**📢 섹션 요약 비유:** OSPF = Area 0 중심!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 라우터 유형 | 역할 | 위치 |
|:---|:---|:---|
| **Internal** | 영역 내 | 일반 |
| **ABR** | 영역 경계 | Area 간 |
| **ASBR** | AS 경계 | 외부 연결 |
| **Backbone** | Area 0 | 핵심 |

**비교표: LSA 타입**

```
+------------+----------+----------+
| LSA        │ 타입     │ 설명     │
+------------+----------+----------+
| Router     │ 1        │ 라우터   │
| Network    │ 2        │ DR       │
| Summary    │ 3, 4     │ ABR      │
| External   │ 5        │ ASBR     │
| NSSA       │ 7        │ NSSA     │
+------------+----------+----------+
```

**핵심 알고리즸:** Dijkstra SPF, Flooding

**📢 섹션 요약 비유:** LSA Type 1 = 라우터 정보!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: OSPF vs IS-IS**

| 항목 | OSPF | IS-IS |
|:---|:---|:---|
| **계층** | IP | L2 |
| **확장** | 제한 | 유연 |
| **용도** | 기업 | ISP |

**📢 섹션 요약 비유:** 기업은 OSPF, ISP는 IS-IS!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 소규모** → 단일 영역
**시나리오 2: 대규모** → 다중 영역
**시나리오 3: 스텁** → Stub/NSSA

**도입 체크리스트:** Area 0 설계, DR/BDR, Cost 조정

**📢 섹션 요약 비유:** 50대 이상 = 다중 영역!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 빠른 컨버전스, 확장성

**미래 전망:** Segment Routing, OSPFv3

**참고 표준:** RFC 2328 (OSPFv2), RFC 5340 (OSPFv3)

**📢 섹션 요약 비유:** OSPF는 기업 네트워크 표준!
