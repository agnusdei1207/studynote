# HSRP - 핫 스탠바이 라우터 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Cisco 라우터 이중화
> 2. **가치**: 게이트웨이 고가용성
> 3. **융합:** Active, Standby, Virtual IP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HSRP (Hot Standby Router Protocol)는 Cisco 전용 라우터 이중화 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    HSRP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HSRP 동작                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  HSRP 상태 머신:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Initial → Listen → Speak → Standby → Active    ││   │ │
│   │   │  │                    ↑__________________|          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  상태 설명:                                          ││   │ │
│   │   │  │  - Initial: 비활성                                 ││   │ │
│   │   │  │  - Listen: 대기 중                                 ││   │ │
│   │   │  │  - Speak: 참여 중 (Hello 송신)                       ││   │ │
│   │   │  │  - Standby: 대기 라우터                             ││   │ │
│   │   │  │  - Active: 활성 라우터                              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  HSRP 그룹:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │  Active (Priority 110)    Standby (Priority 100)││   │ │
│   │   │  │       │                        │                 ││   │ │
│   │   │  │       └────────────┬───────────┘                 ││   │ │
│   │   │  │                    │                             ││   │ │
│   │   │  │            Virtual IP (10.0.0.1)                 ││   │ │
│   │   │  │            Virtual MAC (0000.0C07.ACXX)          ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HSRP는 Cisco 환경에서 널리 사용되는 이중화 프로토콜입니다.

**📢 섹션 요약 비유:** HSRP = Cisco의 예비 시스템!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Active** | 활성 라우터 |
| **Standby** | 대기 라우터 |
| **Priority** | 우선순위 (기본 100) |
| **Preempt** | 선점 |
| **Hello Time** | 헬로 간격 (기본 3초) |
| **Hold Time** | 홀드 타임 (기본 10초) |

**핵심 알고리듬:** 우선순위 → Active 선출 → Standby 선출 → 장애 시 전환
