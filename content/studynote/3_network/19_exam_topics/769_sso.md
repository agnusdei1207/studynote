# SSO (Single Sign-On) - 단일 로그인

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 번 로그인으로 모든 서비스 접근
> 2. **가치**: 사용자 편의성, 중앙 관리
> 3. **융합:** OAuth 2.0, SAML, OpenID Connect

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SSO는 한 번의 로그인으로 여러 서비스에 접근합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SSO (Single Sign-On)                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SSO 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  User                                                │   │ │
│   │   │  ┌─────────┐                                        │   │ │
│   │   │  │         │ 1. 로그인 요청                          │   │ │
│   │   │  │         │ ───────────────→ ┌─────────────┐       │   │ │
│   │   │  │         │                   │ Identity    │       │   │ │
│   │   │  │         │ 2. 인증           │ Provider    │       │   │ │
│   │   │  │         │ ←─────────────── │ (IdP)       │       │   │ │
│   │   │  │         │                   │             │       │   │ │
│   │   │  └─────────┘                   └──────┬──────┘       │   │ │
│   │   │                                      │               │   │ │
│   │   │       3. 토큰/세션으로 서비스 접근      │               │   │ │
│   │   │       ┌──────────┬──────────┬───────┘               │   │ │
│   │   │       ↓          ↓          ↓                        │   │ │
│   │   │  ┌─────────┐ ┌─────────┐ ┌─────────┐               │   │ │
│   │   │  │ App A   │ │ App B   │ │ App C   │               │   │ │
│   │   │  │ (로그인 │ │ (로그인 │ │ (로그인 │               │   │ │
│   │   │  │  됨)   │ │  됨)   │ │  됨)   │               │   │ │
│   │   │  └─────────┘ └─────────┘ └─────────┘               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SSO는 중앙 인증으로 모든 서비스 접근을 허용합니다.

**📢 섹션 요약 비유:** SSO = 만능 열쇠!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **IdP** | 인증 제공자 |
| **SP** | 서비스 제공자 |
| **Token** | 토큰 |
| **Session** | 세션 |
| **Federation** | 연합 |

**핵심 알고리즸:** 인증 → 토큰 발급 → 서비스 접근

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SSO 프로토콜**

| 프로토콜 | 용도 |
|:---|:---|
| **SAML** | 기업, XML |
| **OAuth 2.0/OIDC** | 웹, 모바일 |
| **CAS** | 대학 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 3:** 레거시 → CAS
