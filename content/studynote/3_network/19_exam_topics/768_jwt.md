# JWT (JSON Web Token) - 제이슨 웹 토큰

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자가 수용형 토큰
> 2. **가치**: 무상태, 서명 검증
> 3. **융합:** OAuth 2.0, API Security, Microservices

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

JWT는 JSON 기반의 자가 수용형 토큰입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    JWT (JSON Web Token)                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              JWT 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Header.Payload.Signature                            │   │ │
│   │   │                                                      │   │ │
│   │   │  Header (Base64):                                    │   │ │
│   │   │  {                                                   │   │ │
│   │   │    "alg": "HS256",                                   │   │ │
│   │   │    "typ": "JWT"                                      │   │ │
│   │   │  }                                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Payload (Base64):                                   │   │ │
│   │   │  {                                                   │   │ │
│   │   │    "sub": "user123",                                 │   │ │
│   │   │    "name": "Kim",                                    │   │ │
│   │   │    "iat": 1516239022,                                │   │ │
│   │   │    "exp": 1516242622                                 │   │ │
│   │   │  }                                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Signature:                                          │   │ │
│   │   │  HMACSHA256(                                         │   │ │
│   │   │    base64(header) + "." + base64(payload),          │   │ │
│   │   │    secret                                            │   │ │
│   │   │  )                                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: JWT는 헤더, 페이로드, 서명으로 구성됩니다.

**📢 섹션 요약 비유:** JWT = 디지털 여권!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Header** | 알고리즘 정보 |
| **Payload** | 클레임 |
| **Signature** | 서명 |
| **Claim** | 주장 (sub, iat, exp) |
| **Secret** | 비밀키 |

**핵심 알고리즸:** 생성 → 서명 → 전송 → 검증

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**서명 알고리즘**

| 알고리즘 | 설명 |
|:---|:---|
| **HS256** | 대칭키 |
| **RS256** | 비대칭키 (RSA) |
| **ES256** | 타원곡선 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 간단한 → HS256
**시나리오 2:** 분산 → RS256
**시나리오 3:** 보안 강화 → 짧은 만료 + Refresh
