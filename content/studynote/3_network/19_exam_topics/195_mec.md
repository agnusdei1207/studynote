# MEC (Multi-access Edge Computing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 엣지에서 컴퓨팅
> 2. **가치**: 저지연, 대역폭 절감
> 3. **융합**: 5G, IoT, AR/VR과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MEC(Multi-access Edge Computing)는 네트워크 엣지(기지국 근처)에 컴퓨팅 자원을 배치하여 저지연 서비스를 제공하는 기술입니다.

> **해설**: MEC는 5G의 저지연 서비스를 가능하게 하는 핵심 기술입니다.

**💡 비유**: MEC는 편의점입니다. 큰 마트까지 가지 않아도 가까운 곳에서 바로 살 수 있습니다.

**📢 섹션 요약 비유:** MEC = 엣지에서 처리!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소 | 역할 | 설명 |
|:---|:---|:---|
| **MEC 서버** | 컴퓨팅 | 엣지 데이터센터 |
| **MEP** | 플랫폼 | 서비스 등록/발견 |
| **MEAO** | 오케스트레이션 | 앱 관리 |
| **MEPM** | 플랫폼 관리 | 라이프사이클 |

**비교표**

```
+------------+----------+----------+
| 항목       │ 클라우드 │ MEC      │
+------------+----------+----------+
| 지연       │ 50-100ms │ 5-15ms   │
│ 대역폭     │ 높음     │ 절약     │
│ 가용성     │ 높음     │ 제한     │
│ 확장성     │ 높음     │ 제한     │
+------------+----------+----------+
```

**핵심 알고리즸:** 트래픽 스틼어링, DNS 기반 라우팅

**📢 섹션 요약 비유:** 트래픽 스틼어링 = MEC 핵심!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: MEC vs CDN**

| 항목 | MEC | CDN |
|:---|:---|:---|
| **위치** | 기지국 | POP |
| **지연** | 매우 낮음 | 낮음 |
| **동적** | 가능 | 제한 |
| **컴퓨팅** | 있음 | 제한 |

**📢 섹션 요약 비유:** MEC = CDN + 컴퓨팅!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: AR/VR** → MEC, <10ms 지연
**시나리오 2: 자율주행** → MEC + 5G uRLLC
**시나리오 3: IoT** → MEC 데이터 집계

**도입 체크리스트:** 지연 요구, 트래픽 패턴, 비용

**📢 섹션 요약 비유:** 지연이 중요하면 MEC!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 지연 80% 감소, 백홀 트래픽 50% 절감

**미래 전망:** 분산 클라우드로 확장

**참고 표준:** ETSI MEC, 3GPP TS 23.501

**📢 섹션 요약 비유:** MEC는 5G의 엣지!
