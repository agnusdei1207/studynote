# ICMP Type - ICMP 유형

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: ICMP 메시지 분류
> 2. **가치**: 네트워크 진단
> 3. **융합:** Echo, Destination Unreachable, Time Exceeded

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ICMP Type은 ICMP 메시지의 유형을 나타냅니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ICMP Type                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 ICMP 타입                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Type  Description                    용도             │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  0     Echo Reply               Ping 응답         ││   │ │
│   │   │  │  3     Destination Unreachable  도달 불가         ││   │ │
│   │   │  │  4     Source Quench            송신 제어 (사용 안함)││   │ │
│   │   │  │  5     Redirect                 경로 변경         ││   │ │
│   │   │  │  8     Echo Request             Ping 요청         ││   │ │
│   │   │  │  9     Router Advertisement     라우터 광고        ││   │ │
│   │   │  │  10    Router Solicitation      라우터 요청        ││   │ │
│   │   │  │  11    Time Exceeded            TTL 초과          ││   │ │
│   │   │  │  12    Parameter Problem        파라미터 오류      ││   │ │
│   │   │  │  13    Timestamp Request        시간 요청         ││   │ │
│   │   │  │  14    Timestamp Reply          시간 응답         ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  용도:                                                  │   │ │
│   │   │  - 네트워크 진단 (Ping)                                  │   │ │
│   │   │  - 오류 보고 (도달 불가, TTL 초과)                        │   │ │
│   │   │  - 경로 최적화 (Redirect)                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ICMP Type은 네트워크 진단과 오류 보고에 사용됩니다.

**📢 섹션 요약 비유:** ICMP Type = 우편 배송 상태!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Type** | 유형 |
| **Code** | 상세 코드 |
| **Checksum** | 체크섬 |
| **Query** | 질의 |
| **Error** | 오류 |

**핵심 알고리듬:** 이벤트 발생 → Type/Code 결정 → 메시지 생성

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Query vs Error 메시지**

| 카테고리 | Type | 예시 |
|:---|:---|:---|
| **Query** | 0, 8, 13, 14 | Ping |
| **Error** | 3, 5, 11, 12 | 오류 보고 |
