# CSMA (반송파 감지 다중 접속)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 채널이 비었는지 확인 후 전송
> 2. **가치**: 충돌 감소, 이더넷/WiFi 기반
> 3. **융합**: CSMA/CD, CSMA/CA로 확장

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CSMA(Carrier Sense Multiple Access)는 전송 전에 채널이 사용 중인지 감지(Carrier Sense)하여 유휴 시에만 전송하는 다중 접속 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CSMA (반송파 감지 다중 접속)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CSMA 기본 원리                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  동작 순서:                                              │   │ │
│   │   │                                                      │   │ │
│   │   │  1. Carrier Sense (채널 감지):                         │   │ │
│   │   │     • 채널이 비어있는지 확인                            │   │ │
│   │   │     • 신호 에너지 감지                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 판단:                                                │   │ │
│   │   │     • 채널 비어있음 → 전송 시작                          │   │ │
│   │   │     • 채널 사용 중 → 대기 또는 백오프                    │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 전송:                                                │   │ │
│   │   │     • 데이터 프레임 전송                                │   │ │
│   │   │                                                      │   │ │
│   │   │  문제:                                                   │   │ │
│   │   │  • 두 노드가 동시에 감지 → 동시 전송 → 충돌              │   │ │
│   │   │  • 전파 지연으로 인한 숨은 노드                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CSMA 변형                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1-Persistent:                                           │   │ │
│   │   │  • 채널 비면 즉시 전송                                    │   │ │
│   │   │  • 충돌 확률 높음                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Non-Persistent:                                         │   │ │
│   │   │  • 채널 사용 중이면 랜덤 대기 후 다시 감지                 │   │ │
│   │   │  • 충돌 감소, 지연 증가                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  p-Persistent:                                           │   │ │
│   │   │  • 채널 비면 확률 p로 전송, (1-p)로 1슬롯 대기            │   │ │
│   │   │  • 균형 있는 접근                                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: CSMA는 유선 이더넷과 무선 LAN의 기반 기술입니다.

**💡 비유**: CSMA는 회의에서 말하기 전에 조용한지 듣는 것입니다.

**📢 섹션 요약 비유**: Listen Before Talk!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 방식 | 전송 조건 | 충돌 확률 | 지연 |
|:---|:---|:---:|:---:|
| **1-Persistent** | 즉시 | 높음 | 낮음 |
| **Non-Persistent** | 랜덤 대기 | 낮음 | 높음 |
| **p-Persistent** | 확률 p | 중간 | 중간 |

**비교표**

```
+------------+----------+----------+----------+
| 상황       | 1-P      | Non-P    | p-P      |
+------------+----------+----------+----------+
| 채널 비면  | 즉시 전송 | 즉시 전송 | p로 전송 |
| 채널 사용중| 계속 감지 | 랜덤 대기 | -        |
| 충돌 시    | 백오프   | 백오프   | 백오프   |
+------------+----------+----------+----------+
```

**핵심 알고리즸**: 감지 → 판단 → 전송/대기 → 충돌 시 백오프

**📢 섹션 요약 비유**: p-Persistent가 가장 균형!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: CSMA 진화**

| 기술 | 특징 | 응용 |
|:---|:---|:---|
| **CSMA** | 감지만 | 기본 |
| **CSMA/CD** | 충돌 감지 | 이더넷 |
| **CSMA/CA** | 충돌 회피 | WiFi |

**📢 섹션 요약 비유**: 유선은 CD, 무선은 CA!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 유선 이더넷** → CSMA/CD
**시나리오 2: WiFi** → CSMA/CA
**시나리오 3: 고부하** → Non-Persistent

**도입 체크리스트**: 매체 유형, 부하, 충돌 환경

**📢 섹션 요약 비유**: 매체에 맞는 CSMA!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 충돌 감소, 공정한 접근

**미래 전망**: 스위칭으로 CSMA/CD 감소

**참고 표준**: IEEE 802.3, 802.11

**📢 섹션 요약 비유**: CSMA는 Listen First!
