# Observability - 관측성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 상태 파악
> 2. **가치**: 문제 진단, 성능 최적화
> 3. **융합:** Logging, Metrics, Tracing

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Observability는 시스템 내부 상태를 파악합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Observability                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              관측성 3대 요소                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐   │   │ │
│   │   │  │   Metrics   │ │   Logs      │ │   Traces    │   │   │ │
│   │   │  │   메트릭    │ │   로그      │ │   추적      │   │   │ │
│   │   │  ├─────────────┤ ├─────────────┤ ├─────────────┤   │   │ │
│   │   │  │ - CPU/메모리│ │ - 이벤트    │ │ - 요청 경로 │   │   │ │
│   │   │  │ - 요청 수   │ │ - 에러     │ │ - 지연시간  │   │   │ │
│   │   │  │ - 에러율    │ │ - 디버그   │ │ - 의존성    │   │   │ │
│   │   │  │ - 지연시간  │ │ - 감사     │ │ - 서비스맵  │   │   │ │
│   │   │  └─────────────┘ └─────────────┘ └─────────────┘   │   │ │
│   │   │                                                      │   │ │
│   │   │  What happened? ──→ Why? ──→ Where?                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 관측성은 "무엇이 왜 어디서"를 파악합니다.

**📢 섹션 요약 비유:** Observability = 의료 검진!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Metrics** | 메트릭 |
| **Logs** | 로그 |
| **Traces** | 추적 |
| **Correlation** | 상관관계 |
| **Alerting** | 알림 |

**핵심 알고리즸:** 데이터 수집 → 저장 → 분석 → 시각화

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**관측성 vs 모니터링**

| 항목 | Monitoring | Observability |
|:---|:---|:---|
| **질문** | 알려진 문제 | 알려지지 않은 문제 |
| **방식** | 미리 정의 | 탐색적 |
| **데이터** | 메트릭 | 3요소 통합 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 마이크로서비스 → 3요소 통합
**시나리오 2:** 문제 진단 → 로그 + 추적 상관
**시나리오 3:** 용량 계획 → 메트릭 분석
