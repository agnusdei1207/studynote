# OIDC (OpenID Connect) - 오픈아이디 커넥트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: OAuth 2.0 기반 인증 프로토콜
> 2. **가치**: 사용자 정보 표준화
> 3. **융합:** OAuth 2.0, JWT, ID Token

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

OIDC는 OAuth 2.0 위에 인증 계층을 추가합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OIDC                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OIDC vs OAuth 2.0                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  OAuth 2.0: 인가 (Authorization)                     │   │ │
│   │   │  - Access Token으로 리소스 접근                      │   │ │
│   │   │  - 사용자 정보 포함 안함                             │   │ │
│   │   │                                                      │   │ │
│   │   │  OIDC: 인가 + 인증 (Authentication)                  │   │ │
│   │   │  - ID Token (JWT)으로 사용자 정보 제공               │   │ │
│   │   │  - sub, name, email, picture 등                      │   │ │
│   │   │                                                      │   │ │
│   │   │  Token 종류:                                         │   │ │
│   │   │  - Access Token: 리소스 접근                         │   │ │
│   │   │  - ID Token: 사용자 정보 (JWT)                       │   │ │
│   │   │  - Refresh Token: 갱신                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: OIDC는 OAuth 2.0에 사용자 인증을 표준화합니다.

**📢 섹션 요약 비유:** OIDC = 여권 + 비자!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **ID Token** | 사용자 정보 JWT |
| **UserInfo Endpoint** | 사용자 정보 API |
| **Discovery** | 메타데이터 자동 발견 |
| **sub** | 고유 식별자 |
| **Claims** | 사용자 속성 |

**핵심 알고리즸:** OAuth + ID Token

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ID Token Claims**

| Claim | 설명 |
|:---|:---|
| **iss** | 발급자 |
| **sub** | 주체 |
| **aud** | 대상 |
| **exp** | 만료 |
| **iat** | 발급 시간 |
| **email** | 이메일 |
