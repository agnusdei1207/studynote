# IMS (IP 멀티미디어 서브시스템) - IP Multimedia Subsystem

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 기반 멀티미디어 서비스 플랫폼
> 2. **가치**: VoLTE, RCS, 멀티미디어
> 3. **융합**: SIP, VoLTE, VoWiFi

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

**💡 비유**: IMS는 통신사의 앱스토어입니다.

**📢 섹션 요약 비유:** IMS = 멀티미디어 플랫폼!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **P-CSCF** | Proxy CSCF |
| **S-CSCF** | Serving CSCF |
| **I-CSCF** | Interrogating CSCF |
| **HSS** | Home Subscriber Server |

**비교표: CSCF 역할**

```
+------------+----------+----------+----------+
| CSCF       │ 역할     │ 위치     │ 기능     │
+------------+----------+----------+----------+
| P-CSCF     │ 프록시   │ 단말근처 │ 진입점   │
| I-CSCF     │ 조회     │ 홈       │ 라우팅   │
| S-CSCF     │ 서빙     │ 홈       │ 제어     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** SIP 세션 관리

**📢 섹션 요약 비유:** SIP = IMS 프로토콜!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 음성 서비스**

| 서비스 | 기술 | 특징 |
|:---|:---|:---|
| **CS Voice** | 2G/3G | 회선 |
| **VoLTE** | IMS | 패킷 |
| **VoWiFi** | IMS+WiFi | 실내 |

**📢 섹션 요약 비유:** VoLTE = HD 음성!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 통신사 → VoLTE 배포
**시나리오 2:** 실내 커버리지 → VoWiFi
**시나리오 3:** 메시징 → RCS

**도입 체크리스트:** EPC, IMS, 단말

**📢 섹션 요약 비유:** SRVCC = VoLTE → CS!
