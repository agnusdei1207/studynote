# DNS (Domain Name System) - 이름 해석

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 도메인 이름을 IP 주소로 변환
> 2. **가치**: 사용자 친화적, 분산 데이터베이스
> 3. **융합**: CDN, 로드밸런싱, 보안과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DNS(Domain Name System)는 도메인 이름을 IP 주소로 변환하는 분산 데이터베이스 시스템입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DNS (Domain Name System)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DNS 계층 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │                    . (Root)                          │   │ │
│   │   │                     │                                │   │ │
│   │   │        ┌────────────┼────────────┐                  │   │ │
│   │   │        ↓            ↓            ↓                  │   │ │
│   │   │      .com         .net        .org  (TLD)            │   │ │
│   │   │        │            │            │                  │   │ │
│   │   │        ↓            ↓            ↓                  │   │ │
│   │   │    google.com   example.net  example.org            │   │ │
│   │   │        │                                          │   │ │
│   │   │        ↓                                          │   │ │
│   │   │   www.google.com                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DNS 쿼리 과정                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 클라이언트 → 로컬 DNS (Resolver)                  │   │ │
│   │   │  2. 로컬 DNS → Root 서버                             │   │ │
│   │   │  3. Root → TLD 서버 안내                             │   │ │
│   │   │  4. TLD → 권한 서버 안내                             │   │ │
│   │   │  5. 권한 서버 → IP 응답                              │   │ │
│   │   │                                                      │   │ │
│   │   │  캐싱: 각 단계에서 캐시                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DNS는 계층적 구조로 분산된 데이터베이스입니다.

**💡 비유**: DNS는 전화번호부입니다. 이름으로 전화번호를 찾습니다.

**📢 섹션 요약 비유:** DNS = 이름 → IP!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 레코드 | 용도 | 예시 |
|:---|:---|:---|
| **A** | IPv4 주소 | 192.168.1.1 |
| **AAAA** | IPv6 주소 | 2001:db8::1 |
| **CNAME** | 별칭 | www → example.com |
| **MX** | 메일 서버 | 10 mail.example.com |
| **NS** | 네임 서버 | ns1.example.com |
| **PTR** | 역방향 조회 | IP → 이름 |
| **SOA** | 권한 시작 | 영역 정보 |
| **TXT** | 텍스트 | SPF, DKIM |

**비교표: 쿼리 타입**

```
+------------+----------+----------+
| 타입       │ 설명     │ 용도     │
+------------+----------+----------+
| Recursive  │ 완전 조회│ 클라이언트│
| Iterative  │ 단계별   │ 서버     │
+------------+----------+----------+
```

**핵심 알고리즸:** 재귀/반복 쿼리, 캐싱

**📢 섹션 요약 비유:** TTL = 캐시 시간!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: DNS 보안**

| 기술 | 목적 |
|:---|:---|
| **DNSSEC** | 무결성/인증 |
| **DoH** | 암호화 (HTTPS) |
| **DoT** | 암호화 (TLS) |
| **DNS filtering** | 차단 |

**📢 섹션 요약 비유:** DNSSEC = 신뢰!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 기업** → 내부 DNS + 외부 DNS
**시나리오 2: CDN** → GeoDNS, Anycast
**시나리오 3: 로드밸런싱** → Round Robin DNS

**도입 체크리스트:** TTL, 이중화, 보안

**📢 섹션 요약 비유:** DNS 이중화 = 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 사용자 편의성

**미래 전망:** DNS over HTTPS, DNSSEC 확대

**참고 표준:** RFC 1035, RFC 4035 (DNSSEC)

**📢 섹션 요약 비유:** DNS는 인터넷의 전화번호부!
