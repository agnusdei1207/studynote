# Encryption - 암호화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 기밀성 보호
> 2. **가치**: 무단 접근 방지
> 3. **융합:** 대칭키, 공개키, 하이브리드

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Encryption은 데이터를 읽기 어려운 형태로 변환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Encryption                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              암호화 유형                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  대칭키 암호화 (Symmetric):                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  평문 ──[동일 키]──→ 암호문                      ││   │ │
│   │   │  │  암호문 ──[동일 키]──→ 평문                      ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  알고리즘: AES, DES, 3DES, ChaCha20             ││   │ │
│   │   │  │  장점: 빠름                                     ││   │ │
│   │   │  │  단점: 키 분배 문제                              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  공개키 암호화 (Asymmetric):                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  평문 ──[공개키]──→ 암호문                       ││   │ │
│   │   │  │  암호문 ──[개인키]──→ 평문                       ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  알고리즘: RSA, ECC, ElGamal                    ││   │ │
│   │   │  │  장점: 키 분배 용이                              ││   │ │
│   │   │  │  단점: 느림                                     ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 암호화는 보안의 핵심 기술입니다.

**📢 섹션 요약 비유:** Encryption = 비밀 금고!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Plaintext** | 평문 |
| **Ciphertext** | 암호문 |
| **Key** | 키 |
| **Algorithm** | 알고리즘 |
| **IV** | 초기화 벡터 |

**핵심 알고리듬:** 평문 + 키 → 암호화 → 암호문

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**대칭키 vs 공개키**

| 항목 | 대칭키 | 공개키 |
|:---|:---|:---|
| **속도** | 빠름 | 느림 |
| **키 길이** | 128-256bit | 2048-4096bit |
| **키 분배** | 어려움 | 쉬움 |
| **용도** | 대량 데이터 | 키 교환, 서명 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** HTTPS → 하이브리드
**시나리오 2:** 파일 → AES
**시나리오 3:** 서명 → RSA/ECC
