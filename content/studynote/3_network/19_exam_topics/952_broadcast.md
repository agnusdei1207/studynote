# Broadcast - 브로드캐스트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 1:All 통신
> 2. **가치**: 네트워크 전체 전달
> 3. **융합:** Broadcast Domain, ARP, DHCP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Broadcast는 하나의 송신자에서 네트워크 전체로 전달합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Broadcast                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              브로드캐스트 통신                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1:All 통신:                                             │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │           ┌─────────┐                          ││   │ │
│   │   │  │           │Receiver │                          ││   │ │
│   │   │  │           └────▲────┘                          ││   │ │
│   │   │  │                │                               ││   │ │
│   │   │  │    ┌───────────┼───────────┐                   ││   │ │
│   │   │  │    │           │           │                   ││   │ │
│   │   │  │ ┌──┴──┐    ┌───┴───┐   ┌───┴───┐               ││   │ │
│   │   │  │ │Sender│    │Receiver│  │Receiver│              ││   │ │
│   │   │  │ └──┬──┘    └───────┘   └───────┘               ││   │ │
│   │   │  │    │                                         ││   │ │
│   │   │  │    └─────── Broadcast ──────────────────────→││   │ │
│   │   │  │           (FF:FF:FF:FF:FF:FF)                 ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │

│   │   │  - L3: 255.255.255.255 (제한)                         │   │ │

│   │   │                                                      │   │ │

│   │                                                              │ │
```

> **해설**: 브로드캐스트는 네트워크 전체에 정보를 전달합니다.

**📢 섹션 요약 비유:** Broadcast = 마을 확성기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Domain** | 브로드캐스트 도메인 |
| **Address** | 브로드캐스트 주소 |
| **Storm** | 브로드캐스트 스톰 |
| **Flooding** | 플러딩 |
| **Segment** | 세그먼트 |

**핵심 알고리듬:** 브로드캐스트 주소로 전송 → 전체 수신

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**L2 vs L3 브로드캐스트**

| 계층 | 주소 | 범위 |
|:---|:---|:---|
| **L2** | FF:FF:FF:FF:FF:FF | LAN |
| **L3 제한** | 255.255.255.255 | 로컬 |
| **L3 서브넷** | 네트워크.255 | 서브넷 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** ARP → L2 브로드캐스트
**시나리오 2:** DHCP → L3 브로드캐스트
**시나리오 3:** 스톰 방지 → VLAN 분리
