# ICMP - 인터넷 제어 메시지 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오류 및 진단 메시지
> 2. **가치**: 네트워크 문제 보고
> 3. **융합:** Ping, Traceroute, Error

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ICMP (Internet Control Message Protocol)는 오류와 진단 정보를 전달합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ICMP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ICMP 메시지 유형                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  메시지 타입:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Type 0: Echo Reply (Ping 응답)                   ││   │ │
│   │   │  │ Type 3: Destination Unreachable (도달 불가)       ││   │ │
│   │   │  │ Type 4: Source Quench (사용 안함)                 ││   │ │
│   │   │  │ Type 5: Redirect (경로 변경)                      ││   │ │
│   │   │  │ Type 8: Echo Request (Ping 요청)                  ││   │ │
│   │   │  │ Type 11: Time Exceeded (TTL 초과)                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  Ping 동작:                                             │   │ │
│   │   │  Client                    Server                      │   │ │
│   │   │  ┌─────┐                  ┌─────┐                     │   │ │
│   │   │  │     │ ── Echo Req ───→ │     │                     │   │ │
│   │   │  │     │ ←─ Echo Reply ── │     │                     │   │ │
│   │   │  └─────┘                  └─────┘                     │   │ │
│   │   │                                                      │   │ │
│   │   │  Traceroute:                                             │   │ │
│   │   │  TTL=1 → 첫 번째 라우터 응답                              │   │ │
│   │   │  TTL=2 → 두 번째 라우터 응답                              │   │ │
│   │   │  ...                                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ICMP는 네트워크 진단의 핵심입니다.

**📢 섹션 요약 비유:** ICMP = 네트워크 우편 배달 실패 통지!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Type** | 메시지 유형 |
| **Code** | 세부 코드 |
| **Checksum** | 체크섬 |
| **Echo** | 에코 |
| **TTL Exceeded** | TTL 초과 |

**핵심 알고리즸:** 문제 발생 → ICMP 메시지 → 원본 송신자

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ICMP 메시지 분류**

| 분류 | 유형 | 용도 |
|:---|:---|:---|
| **Error** | 3, 11, 12 | 오류 보고 |
| **Query** | 0, 8 | 진단 |
| **Redirect** | 5 | 경로 변경 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 2:** 경로 추적 → Traceroute
**시나리오 3:** 방화벽 → ICMP 차단/허용
