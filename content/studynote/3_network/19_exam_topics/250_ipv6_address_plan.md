### Ⅰ. 개요 (Context & Background)

**개념 정의**

IPv6 주소 계획은 IPv6 네트워크를 위한 주소 체계를 설계하는 것입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IPv6 주소 계획                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IPv6 주소 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  /48 할당 (ISP → 고객)                              │   │ │
│   │   │  ┌──────────────────┬────────────────┬─────────┐   │   │ │
│   │   │  │   48비트         │   16비트       │  64비트  │   │   │ │
│   │   │  │   Prefix         │   Subnet       │  Host    │   │   │ │
│   │   │  │  (네트워크)       │   (부서)       │ (장비)   │   │   │ │
│   │   │  └──────────────────┴────────────────┴─────────┘   │   │ │
│   │   │                                                      │   │ │
│   │   │  예: 2001:db8:abcd::/48                             │   │ │
│   │   │      2001:db8:abcd:0001::/64 (서브넷 1)             │   │ │
│   │   │      2001:db8:abcd:0002::/64 (서브넷 2)             │   │ │
│   │   │                                                      │   │ │
│   │   │  서브넷 수: 65,536개 (/48 → /64)                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주소 할당 체계                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  계층      │ 크기    │ 용도                         │   │ │
│   │   │  ─────────┼────────┼───────────────────────────│   │ │
│   │   │  /32      │ ISP    │ 대형 ISP                     │   │ │
│   │   │  /48      │ Site   │ 기업/기관                    │   │ │
│   │   │  /56      │ Site   │ 소규모                       │   │ │
│   │   │  /64      │ Subnet │ LAN                          │   │ │
│   │   │                                                      │   │ │
│   │   │  권장: 단일 서브넷 = /64                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IPv6는 /64를 서브넷 단위로 사용합니다.

**💡 비유**: IPv6 주소는 도시 계획입니다. 구역별로 체계적으로 할당합니다.

**📢 섹션 요약 비유:** /64 = 서브넷!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **/32** | ISP 할당 |
| **/48** | 사이트 할당 |
| **/56** | 소규모 사이트 |
| **/64** | 서브넷 |
| **/127** | P2P 링크 |

**비교표: IPv4 vs IPv6 주소 계획**

```
+------------+----------+----------+
| 항목       │ IPv4     │ IPv6     │
+------------+----------+----------+
| 서브넷     │ VLSM     │ /64 고정 │
| 주소 수    │ 제한     │ 사실상 무제한│
| NAT        │ 필수     │ 불필요   │
| 계획       │ 복잡     │ 단순     │
+------------+----------+----------+
```

**핵심 알고리즸:** 계층적 할당

**📢 섹션 요약 비유:** IPv6 = 서브넷 /64로 단순화!
