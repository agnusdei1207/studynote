# DNS Caching - DNS 캐싱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: DNS 응답 저장
> 2. **가치**: 조회 속도 향상
> 3. **융합:** TTL, Resolver, Negative Cache

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DNS Caching은 DNS 응답을 로컬에 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DNS Caching                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DNS 캐시 계층                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  캐시 계층:                                               │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  1. Browser Cache (브라우저)                      ││   │ │
│   │   │  │     수 분 ~ 수 시간                              ││   │ │
│   │   │  │           │                                     ││   │ │
│   │   │  │           ▼                                     ││   │ │
│   │   │  │  2. OS Cache (운영체제)                           ││   │ │
│   │   │  │     TTL 따름                                     ││   │ │
│   │   │  │           │                                     ││   │ │
│   │   │  │           ▼                                     ││   │ │
│   │   │  │  3. Resolver Cache (ISP/공용)                     ││   │ │
│   │   │  │     TTL 따름                                     ││   │ │
│   │   │  │           │                                     ││   │ │
│   │   │  │           ▼                                     ││   │ │
│   │   │  │  4. Authoritative DNS (권한 서버)                  ││   │ │
│   │   │  │     원본 데이터                                   ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  TTL (Time To Live):                                   │   │ │
│   │   │  - 캐시 유효 시간                                       │   │ │
│   │   │  - 높은 TTL: 캐시 오래, 변경 어려움                       │   │ │
│   │   │  - 낮은 TTL: 캐시 짧음, 변경 쉬움                        │   │ │

│   │   └─────────────────────────────────────────────────────┘   │ │
```

> **해설**: DNS 캐싱은 인터넷 성능의 핵심입니다.

**📢 섹션 요약 비유:** DNS Caching = 기억해 두기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **TTL** | 캐시 시간 |
| **Resolver** | 해석기 |
| **Cache Hit** | 캐시 적중 |
| **Negative** | 부정 캐시 |
| **Flush** | 캐시 제거 |

**핵심 알고리듬:** 조회 → 캐시 확인 → 없으면 질의 → 캐시 저장

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 안정 서비스 → 높은 TTL (1일)
**시나리오 2:** 변경 잦음 → 낮은 TTL (5분)
**시나리오 3:** 장애 → 캐시 플러시
