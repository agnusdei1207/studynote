# Silly Window Syndrome (SWS) - 멍청한 윈도우 증후군

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 작은 윈도우 반복 전송
> 2. **가치**: 대역폭 낭비 방지
> 3. **융합:** Nagle, Clark, TCP 최적화

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SWS는 수신측 윈도우가 작아 효율이 떨어지는 현상입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Silly Window Syndrome                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SWS 문제                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  수신측 버퍼: [XXXX    ] (4바이트 공간)              │   │ │
│   │   │                                                      │   │ │
│   │   │  송신측: 1바이트씩 4번 전송                          │   │ │
│   │   │  → 4개 패킷 (각 41+ 바이트)                          │   │ │
│   │   │  → 비효율!                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  해결:                                               │   │ │
│   │   │  - 송신측: Nagle (모아서 전송)                       │   │ │
│   │   │  - 수신측: Clark (윈도우 업데이트 지연)              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 송/수신측 모두 협력하여 SWS를 방지합니다.

**📢 섹션 요약 비유:** SWS = 낭비 방지!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **송신측 해결** | Nagle Algorithm |
| **수신측 해결** | Clark Solution |
