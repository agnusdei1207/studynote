# Link Local - 링크 로컬

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 로컬 통신용 주소
> 2. **가치**: DHCP 없는 통신
> 3. **융합:** APIPA, 169.254.x.x, ZeroConf

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Link Local은 같은 네트워크 내 통신용 주소입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Link Local                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              링크 로컬 주소                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IPv4 링크 로컬 (APIPA):                                │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  169.254.0.0/16                                 ││   │ │
│   │   │  │  169.254.0.1 - 169.254.255.254                  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  DHCP 서버 없을 때 자동 할당                        ││   │ │
│   │   │  │  로컬 통신만 가능                                  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  IPv6 링크 로컬:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ fe80::/10                                       ││   │ │
│   │   │  │ fe80::xxxx:xxxx:xxxx:xxxx                       ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ 모든 IPv6 인터페이스에 필수                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                  │   │ │
│   │   │  - 라우터 거치지 않음                                    │   │ │
│   │   │  - 로컬 세그먼트 내에서만 유효                            │   │ │
│   │   │  - 자동 구성                                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Link Local은 DHCP 없는 환경에서 통신을 가능하게 합니다.

**📢 섹션 요약 비유:** Link Local = 같은 방 내 통신!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **APIPA** | 자동 개인 IP |
| **ZeroConf** | 제로 설정 |
| **Scope** | 범위 |
| **Auto-config** | 자동 설정 |
| **Neighbor** | 이웃 |

**핵심 알고리듬:** DHCP 실패 → 랜덤 생성 → 충돌 검사 → 사용

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**IPv4 vs IPv6 Link Local**

| 항목 | IPv4 | IPv6 |
|:---|:---|:---|
| **주소** | 169.254.x.x | fe80:: |
| **필수** | 아니오 | 예 |
| **할당** | DHCP 실패 시 | 항상 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** DHCP 서버 다운 → APIPA
**시나리오 2:** Ad-hoc 네트워크 → Link Local
**시나리오 3:** IPv6 → fe80:: 사용
