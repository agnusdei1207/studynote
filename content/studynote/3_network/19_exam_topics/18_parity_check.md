# 패리티 검사 (Parity Check)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 비트 수의 홀짝으로 오류 검출
> 2. **가치**: 간단한 구현, 1비트 오류 검출
> 3. **융합**: 비동기 전송, 메모리, 버스와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

패리티 검사(Parity Check)는 데이터 비트에 1비트를 추가하여 1의 개수를 홀수 또는 짝수로 만들어 오류를 검출하는 방법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    패리티 검사 (Parity Check)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              패리티 종류                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  짝수 패리티 (Even Parity):                            │   │ │
│   │   │  1의 개수가 짝수가 되도록 패리티 비트 설정              │   │ │
│   │   │                                                      │   │ │
│   │   │  데이터: 1011001 (1의 개수: 4개, 이미 짝수)           │   │ │
│   │   │  패리티: 0                                           │   │ │
│   │   │  전송: 10110010                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  홀수 패리티 (Odd Parity):                             │   │ │
│   │   │  1의 개수가 홀수가 되도록 패리티 비트 설정              │   │ │
│   │   │                                                      │   │ │
│   │   │  데이터: 1011001 (1의 개수: 4개, 짝수)                │   │ │
│   │   │  패리티: 1                                           │   │ │
│   │   │  전송: 10110011                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              패리티 검사 과정                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  송신측:                                                │   │ │
│   │   │  1. 데이터 비트 수신                                    │   │ │
│   │   │  2. 1의 개수 계산                                       │   │ │
│   │   │  3. 패리티 비트 생성                                    │   │ │
│   │   │  4. 데이터 + 패리티 전송                                │   │ │
│   │   │                                                      │   │ │
│   │   │  수신측:                                                │   │ │
│   │   │  1. 데이터 + 패리티 수신                                │   │ │
│   │   │  2. 1의 개수 계산                                       │   │ │
│   │   │  3. 홀짝 확인                                           │   │ │
│   │   │  4. 불일치 시 오류 검출                                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 패리티는 1비트 오류만 검출 가능, 2비트 오류는 검출 못함.

**💡 비유**: 패리티는 물건 개수가 홀짝인지 확인하는 것입니다.

**📢 섹션 요약 비유**: 짝수 패리티가 일반적!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Data Bits** | 데이터 | 5-8 비트 | 본문 |
| **Parity Bit** | 검사 | 홀/짝 | 인증 |
| **XOR** | 연산 | 배타적 논리합 | 계산 |

**XOR 연산으로 패리티 계산**

```
데이터: 1 0 1 1 0 0 1
XOR:   1⊕0⊕1⊕1⊕0⊕0⊕1 = 0
Even Parity: 0 (이미 짝수)
Odd Parity: 1 (홀수로 만듦)
```

**핵심 알고리즘**: XOR 연산 → 패리티 생성 → 수신측 XOR → 0이면 정상

**📢 섹션 요약 비유**: XOR 결과가 0이면 정상!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 패리티 방식**

| 방식 | 1의 개수 | 장점 |
|:---|:---:|:---|
| **Even** | 짝수 | 일반적 |
| **Odd** | 홀수 | 전체 0 방지 |
| **Mark** | 항상 1 | 테스트용 |
| **Space** | 항상 0 | 테스트용 |

**📢 섹션 요약 비유**: Odd Parity는 전체 0 오류 방지!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 시리얼 통신** → 7E1 (Even)
**시나리오 2: 메모리** → ECC로 대체
**시나리오 3: 버스** → 패리티 + 재시도

**도입 체크리스트**: 오류율, 정정 필요성

**📢 섹션 요약 비유**: 메모리는 ECC!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 1비트 오류 100% 검출

**미래 전망**: ECC, CRC로 대체 추세

**참고 표준**: RS-232, UART

**📢 섹션 요약 비유**: 패리티는 가장 단순한 오류 검출!
