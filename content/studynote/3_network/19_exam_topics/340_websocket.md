# WebSocket - 실시간 통신

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 양방향 실시간 통신 프로토콜
> 2. **가치**: 낮은 지연, 지속 연결
> 3. **융합**: 채팅, 게임, IoT와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

WebSocket은 단일 TCP 연결로 양방향 실시간 통신을 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WebSocket                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              WebSocket 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ HTTP Upgrade 요청                              │   │ │
│   │   │     │ GET /chat HTTP/1.1                             │   │ │
│   │   │     │ Upgrade: websocket                             │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           WebSocket Server                  │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • HTTP 핸드셰이크                           │    │   │ │
│   │   │  │  • 지속 연결                                 │    │   │ │
│   │   │  │  • 양방향 메시지                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ WebSocket 연결 (ws:// 또는 wss://)          │   │ │
│   │   │       │                                              │   │ │
│   │   │       │  ◄────── 양방향 메시지 ──────►               │   │ │
│   │   │       │                                              │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              WebSocket vs HTTP                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ HTTP     │ WebSocket│               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 연결       │ 단발성   │ 지속     │               │   │ │
│   │   │  │ 방향       │ 단방향   │ 양방향   │               │   │ │
│   │   │  | 오버헤드   │ 높음     │ 낮음     │               │   │ │
│   │   │  │ 지연       │ 높음     │ 낮음     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: WebSocket은 HTTP 핸드셰이크 후 지속 연결로 양방향 통신합니다.

**💡 비유**: WebSocket은 전화 통화입니다. 연결 후 계속 대화합니다.

**📢 섹션 요약 비유:** WebSocket = 실시간 양방향!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Handshake** | 핸드셰이크 |
| **Frame** | 프레임 |
| **Message** | 메시지 |
| **Ping/Pong** | 하트비트 |

**비교표: 실시간 기술**

```
+------------+----------+----------+
| 기술       │ 특징     │ 용도     │
+------------+----------+----------+
| WebSocket  │ 양방향   │ 채팅     │
| SSE        │ 단방향   │ 알림     │
| Long Polling│ 폴링    │ 호환성   │
+------------+----------+----------+
```

**핵심 알고리즸:** 핸드셰이크, 프레이밍

**📢 섹션 요약 비유:** Upgrade = 웹소켓 시작!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 실시간 통신**

| 항목 | WebSocket | SSE | Long Polling |
|:---|:---|:---|:---|
