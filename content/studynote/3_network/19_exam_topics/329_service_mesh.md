# Service Mesh - 서비스 메시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 마이크로서비스 통신 인프라
> 2. **가치**: 관측성, 보안, 트래픽 제어
> 3. **융합**: Istio, Envoy, Kubernetes와 통합

### Ⅰ. 개요 (Context & Background)

**개념 정의**

> **해설**: Service Mesh는 사이드카 프록시로 서비스 간 통신을 관리합니다.

**💡 비유**: Service Mesh는 마이크로서비스의 교통경찰입니다. 모든 통신을 제어합니다.

**📢 섹션 요약 비유:** Service Mesh = 통신 인프라!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Data Plane** | 사이드카 프록시 |
| **Control Plane** | 제어 플레인 |
| **Sidecar** | Envoy |
| **mTLS** | 상호 TLS |

**비교표: Service Mesh 구현**

```
+------------+----------+----------+
| 항목       │ Istio    │ Linkerd  │
+------------+----------+----------+
| 프록시     │ Envoy    │ linkerd2 │
| 복잡도     │ 높음     │ 낮음     │
| 기능       │ 풍부     │ 기본     │
| 성능       │ 중간     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** 사이드카 패턴, mTLS

**📢 섹션 요약 비유:** Sidecar = 프록시!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 통신 패턴**

| 항목 | Service Mesh | API Gateway | 직접 호출 |
|:---|:---|:---|:---|
| **관측성** | 자동 | 수동 | 없음 |
| **보안** | mTLS | TLS | 수동 |
| **복잡도** | 높음 | 중간 | 낮음 |

**📢 섹션 요약 비유:** 마이크로서비스 = Service Mesh!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 대규모 K8s** → Istio
**시나리오 2: 간단한 보안** → Linkerd
**시나리오 3: 소규모** → 직접 구현

**도입 체크리스트:** 규모, 팀 역량, 오버헤드

**📢 섹션 요약 비유:** 대규모 = Istio!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 통합 관측성, 자동 보안
