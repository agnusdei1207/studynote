# NDP (Neighbor Discovery Protocol) - IPv6 이웃 발견

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPv6용 ARP + 기능
> 2. **가치**: 자동 설정, 경로 발견
> 3. **융합**: SLAAC, RA와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NDP(Neighbor Discovery Protocol)는 IPv6에서 이웃 발견, 자동 설정, 경로 발견을 수행하는 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NDP (Neighbor Discovery Protocol)                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NDP 기능                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Router Discovery (RA):                           │   │ │
│   │   │     • 라우터가 네트워크 정보 광고                    │   │ │
│   │   │     • 접두사, MTU, 기본 게이트웨이                  │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Prefix Discovery:                                │   │ │
│   │   │     • 네트워크 접두사 학습                           │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Address Autoconfiguration (SLAAC):               │   │ │
│   │   │     • 자동 주소 생성                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  4. Neighbor Discovery (NS/NA):                      │   │ │
│   │   │     • ARP 대체                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  5. Duplicate Address Detection (DAD):               │   │ │
│   │   │     • IP 충돌 감지                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  6. Redirect:                                        │   │ │
│   │   │     • 더 나은 경로 안내                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ICMPv6 메시지                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  타입 │ 이름              │ 용도                    │   │ │
│   │   │  ─────┼──────────────────┼───────────────────────│   │ │
│   │   │  133  │ RS               │ 라우터 요청             │   │ │
│   │   │  134  │ RA               │ 라우터 광고             │   │ │
│   │   │  135  │ NS               │ 이웃 요청               │   │ │
│   │   │  136  │ NA               │ 이웃 광고               │   │ │
│   │   │  137  │ Redirect         │ 경로 변경               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: NDP는 ICMPv6를 사용하여 ARP 이상의 기능을 제공합니다.

**💡 비유**: NDP는 IPv6의 만능 도구입니다. ARP, DHCP, ICMP를 통합했습니다.

**📢 섹션 요약 비유:** NDP = IPv6의 ARP!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | IPv4 | IPv6 |
|:---|:---|:---|
| **주소 해석** | ARP | NDP (NS/NA) |
| **라우터 발견** | 없음 | NDP (RA) |
| **자동 설정** | DHCP | SLAAC |
| **중복 감지** | Gratuitous ARP | DAD |

**비교표: NS vs NA**

```
+------------+----------+----------+
| 메시지     │ 방향     │ 용도     │
+------------+----------+----------+
| NS         │ 요청     │ MAC 확인 │
| NA         │ 응답     │ MAC 광고 │
+------------+----------+----------+
```

**핵심 알고리즸:** SLAAC, DAD

**📢 섹션 요약 비유:** DAD = IP 충돌 방지!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: RA 플래그**

| 플래그 | 의미 |
|:---|:---|
| **A** | SLAAC 가능 |
| **L** | 로컬 사용 가능 |
| **M** | DHCPv6 사용 |
| **O** | DHCPv6 기타 정보 |

**📢 섹션 요약 비유:** A=1 = SLAAC!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: SLAAC** → RA (A=1)
**시나리오 2: DHCPv6** → RA (M=1)
**시나리오 3: 혼합** → RA (A=1, O=1)

**도입 체크리스트:** RA 설정, 보안 (RA Guard)

**📢 섹션 요약 비유:** RA Guard = 보안 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 자동 설정, 단순화

**미래 전망:** 보안 강화

**참고 표준:** RFC 4861
