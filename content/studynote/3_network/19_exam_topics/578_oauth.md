# OAuth 2.0 - 오픈 인증 2.0

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 위임 인증 프레임워크
> 2. **가치**: 비밀번호 노출 없는 권한 위임
> 3. **융합:** Access Token, Refresh Token, OIDC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

OAuth 2.0은 제3자에게 권한을 위임합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OAuth 2.0                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OAuth 2.0 흐름                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  User    Client    Auth Server    Resource Server   │   │ │
│   │   │    │        │            │              │            │   │ │
│   │   │    │──(1)──→│            │              │            │   │ │
│   │   │    │   Authorize        │              │            │   │ │
│   │   │    │        │──(2)──────→│              │            │   │ │
│   │   │    │        │  Auth Code │              │            │   │ │
│   │   │    │        │←──(3)──────│              │            │   │ │
│   │   │    │        │   Token    │              │            │   │ │
│   │   │    │        │──(4)───────────────────→ │            │   │ │
│   │   │    │        │        Access Token      │            │   │ │
│   │   │    │        │←──(5)─────────────────── │            │   │ │
│   │   │    │        │        Resource          │            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: OAuth 2.0은 Access Token으로 리소스에 접근합니다.

**📢 섹션 요약 비유:** OAuth 2.0 = 대리 키!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Resource Owner** | 사용자 |
| **Client** | 애플리케이션 |
| **Authorization Server** | 인증 서버 |
| **Resource Server** | 리소스 서버 |
| **Access Token** | 접근 토큰 |
| **Refresh Token** | 갱신 토큰 |

**핵심 알고리즸:** 권한 위임 → 토큰 발급 → 리소스 접근

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Grant Types**

| 타입 | 용도 |
|:---|:---|
| **Authorization Code** | 웹 앱 |
| **Implicit** | SPA (비권장) |
| **Client Credentials** | 서비스 간 |
| **Refresh Token** | 토큰 갱신 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹 앱 → Authorization Code
**시나리오 2:** 모바일 → PKCE
**시나리오 3:** 서비스 간 → Client Credentials
