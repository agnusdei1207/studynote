# 해밍 코드 (Hamming Distance)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 최소 거리로 오류 검출/정정 능력 결정
> 2. **가치**: 1비트 정정, 2비트 검출, 메모리 ECC
> 3. **융합**: ECC 메모리, 통신 채널, 코딩 이론과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

해밍 거리(Hamming Distance)는 두 코드워드 간 다른 비트의 개수이며, 해밍 코드는 이를 이용해 오류를 정정하는 코드입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    해밍 코드 (Hamming Distance)                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              해밍 거리 (Hamming Distance)                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  정의: 두 코드워드 간 다른 비트의 개수                   │   │ │
│   │   │                                                      │   │ │
│   │   │  예:                                                   │   │ │
│   │   │  코드 A: 10110                                        │   │ │
│   │   │  코드 B: 11010                                        │   │ │
│   │   │            ↑↑                                         │   │ │
│   │   │  해밍 거리 = 2 (두 비트가 다름)                        │   │ │
│   │   │                                                      │   │ │
│   │   │  최소 해밍 거리 (d_min):                               │   │ │
│   │   │  코드 집합 내 모든 쌍의 최소 거리                       │   │ │
│   │   │                                                      │   │ │
│   │   │  능력:                                                 │   │ │
│   │   │  • d_min >= t+1 : t개 오류 검출                       │   │ │
│   │   │  • d_min >= 2t+1 : t개 오류 정정                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              해밍 코드 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Hamming(7,4) 코드:                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  4비트 데이터 → 7비트 코드워드                        │   │ │
│   │   │                                                      │   │ │
│   │   │  위치: 1 2 3 4 5 6 7                                  │   │ │
│   │   │  비트: P1 P2 D1 P4 D2 D3 D4                           │   │ │
│   │   │       ↓  ↓      ↓                                    │   │ │
│   │   │      패리티 비트들 (2의 거듭제곱 위치)                 │   │ │
│   │   │                                                      │   │ │
│   │   │  패리티 계산:                                          │   │ │
│   │   │  P1: 위치 1,3,5,7 (1,3,5,7의 1번 비트=1)              │   │ │
│   │   │  P2: 위치 2,3,6,7 (2,3,6,7의 2번 비트=1)              │   │ │
│   │   │  P4: 위치 4,5,6,7 (4,5,6,7의 4번 비트=1)              │   │ │
│   │   │                                                      │   │ │
│   │   │  d_min = 3 → 1비트 정정, 2비트 검출                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 해밍 거리 3이면 1비트 정정, 2비트 검출 가능합니다.

**💡 비유**: 해밍 코드는 이웃 간 최소 거리가 3이면 1칸 이탈은 복구 가능합니다.

**📢 섹션 요약 비유**: d=3 → 1비트 정정!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Parity Bits** | 검사 | 2^n 위치 | 이정표 |
| **Data Bits** | 데이터 | 나머지 위치 | 화물 |
| **Syndrome** | 진단 | XOR 결과 | 진료 |
| **d_min** | 능력 | 최소 거리 | 내구성 |

**정정 과정**

1. 수신된 코드워드
2. 각 패리티 검사
3. 신드롬 계산 (XOR 결과)
4. 신드롬 = 0 → 정상
5. 신드롬 ≠ 0 → 오류 위치

**핵심 알고리즘**: Syndrome = 오류 위치 → 해당 비트 플립

**📢 섹션 요약 비유**: Syndrome이 오류 위치!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 오류 제어 코드**

| 코드 | d_min | 검출 | 정정 |
|:---|:---:|:---:|:---:|
| **Parity** | 2 | 1비트 | X |
| **Hamming** | 3 | 2비트 | 1비트 |
| **SECDED** | 4 | 3비트 | 1비트 |

**📢 섹션 요약 비유**: SECDED가 메모리 표준!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: DRAM** → ECC (SECDED)
**시나리오 2: 통신** → Hamming(7,4)
**시나리오 3: 저장** → Reed-Solomon

**도입 체크리스트**: 오류율, 정정 필요성

**📢 섹션 요약 비유**: 서버는 ECC 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 1비트 오류 100% 정정

**미래 전망**: 더 강력한 ECC, LDPC

**참고 표준**: Hamming(7,4), SECDED

**📢 섹션 요약 비유**: 해밍 코드는 단일 비트 정정의 기초!
