# Docker - 도커

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컨테이너 플랫폼
> 2. **가치**: Build once, Run anywhere
> 3. **융합:** Container Registry, CI/CD, Kubernetes

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Docker는 컨테이너를 빌드, 실행, 배포합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Docker                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Docker 아키텍처                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Client                   Server                     │   │ │
│   │   │  ┌─────────┐            ┌─────────────────────┐     │   │ │
│   │   │  │ docker  │ ── REST ──→│ dockerd (daemon)    │     │   │ │
│   │   │  │  CLI    │            │                     │     │   │ │
│   │   │  └─────────┘            │ ┌─────────────────┐ │     │   │ │
│   │   │                         │ │    Images       │ │     │   │ │
│   │   │                         │ ├─────────────────┤ │     │   │ │
│   │   │                         │ │   Containers    │ │     │   │ │
│   │   │                         │ ├─────────────────┤ │     │   │ │
│   │   │                         │ │   Networks      │ │     │   │ │
│   │   │                         │ ├─────────────────┤ │     │   │ │
│   │   │                         │ │   Volumes       │ │     │   │ │
│   │   │                         │ └─────────────────┘ │     │   │ │
│   │   │                         └─────────────────────┘     │   │ │
│   │   │                                                      │   │ │
│   │   │  Registry:                                            │   │ │
│   │   │  Docker Hub, ECR, GCR, Harbor                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Docker는 컨테이너 생애주기를 관리합니다.

**📢 섹션 요약 비유:** Docker = 표준 컨테이너!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Dockerfile** | 이미지 정의 |
| **Image** | 이미지 |
| **Container** | 실행 인스턴스 |
| **Volume** | 영구 저장 |
| **Network** | 네트워크 |

**핵심 알고리즸:** Dockerfile → Build → Image → Run → Container

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Docker 명령어**

| 명령 | 설명 |
|:---|:---|
| **build** | 이미지 빌드 |
| **run** | 컨테이너 실행 |
| **push** | 레지스트리 업로드 |
| **pull** | 레지스트리 다운로드 |
| **compose** | 다중 컨테이너 |
