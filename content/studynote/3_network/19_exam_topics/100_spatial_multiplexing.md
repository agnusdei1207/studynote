# 공간 다중화 (Spatial Multiplexing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 독립 데이터 스트림을 공간으로 병렬 전송
> 2. **가치**: 전송률 선형 증가 (안테나 수)
> 3. **융합**: MIMO, LTE, WiFi와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

공간 다중화(Spatial Multiplexing)는 MIMO 시스템에서 서로 다른 데이터 스트림을 각 안테나에서 동시에 전송하여 전송률을 안테나 수만큼 증가시키는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    공간 다중화 (Spatial Multiplexing)               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              공간 다중화 원리                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  송신 (2×2 예):                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  데이터 스트림                                         │   │ │
│   │   │  ┌─────┐    ┌─────┐                                  │   │ │
│   │   │  │S1   │    │S2   │                                  │   │ │
│   │   │  └──┬──┘    └──┬──┘                                  │   │ │
│   │   │     ↓          ↓                                      │   │ │
│   │   │  ┌─────┐    ┌─────┐                                  │   │ │
│   │   │  │TX1  │    │TX2  │  ← 동시 전송                      │   │ │
│   │   │  └─────┘    └─────┘                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  채널:                                                  │   │ │
│   │   │  TX1 → RX1: h11    TX1 → RX2: h12                    │   │ │
│   │   │  TX2 → RX1: h21    TX2 → RX2: h22                    │   │ │
│   │   │                                                      │   │ │
│   │   │  수신:                                                  │   │ │
│   │   │  y1 = h11·s1 + h21·s2 + n1                           │   │ │
│   │   │  y2 = h12·s1 + h22·s2 + n2                           │   │ │
│   │   │                                                      │   │ │
│   │   │  분리:                                                  │   │ │
│   │   │  [s1, s2] = H⁻¹ × [y1, y2]                           │   │ │
│   │   │  (ZF, MMSE 등)                                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              공간 다중화 vs 다이버시티                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  공간 다중화:                                          │   │ │
│   │   │  목적: 전송률 ↑                                        │   │ │
│   │   │  방식: 다른 데이터 동시 전송                            │   │ │
│   │   │  이득: min(N_tx, N_rx) ×                              │   │ │
│   │   │                                                      │   │ │
│   │   │  다이버시티:                                            │   │ │
│   │   │  목적: 신뢰성 ↑                                        │   │ │
│   │   │  방식: 같은 데이터 중복 전송                            │   │ │
│   │   │  이득: SNR 이득                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  트레이드오프: 다중화 ↔ 다이버시티                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 공간 다중화는 MIMO의 핵심 이득 원천입니다.

**💡 비유**: 공간 다중화는 여러 차선 도로에서 각각 다른 화물을 나르는 것입니다.

**📢 섹션 요약 비유**: 안테나 수 = 스트림 수 = 전송률!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **스트림** | 독립 데이터 | layer | 차선 |
| **프리코딩** | 송신 최적화 | 가중치 | 조향 |
| **디코딩** | 분리 | ZF/MMSE | 해독 |

**비교표**

```
+------------+----------+----------+
| MIMO       | 다중화   | 다이버시티|
+------------+----------+----------+
| 2×2        | 2 스트림 | 2-order  |
| 4×4        | 4 스트림 | 4-order  |
| 이득       | 속도     | 신뢰성   |
+------------+----------+----------+
```

**핵심 알고리즸**: S/P → 프리코딩 → 다중 전송 → 채널 → 디코딩 → P/S

**📢 섹션 요약 비유**: 채널 품질이 좋아야 다중화 효과!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 전송률 향상**

| MIMO | SNR 높음 | SNR 낮음 |
|:---|:---|:---|
| **2×2** | 다중화 | 다이버시티 |
| **4×4** | 4× 속도 | 4× 신뢰성 |

**📢 섹션 요약 비유**: SNR에 따른 적응적 선택!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 높은 SNR** → 공간 다중화
**시나리오 2: 낮은 SNR** → 다이버시티
**시나리오 3: 혼합** → 적응적 전환

**도입 체크리스트**: SNR, 채널 랭크, 복잡도

**📢 섹션 요약 비유**: 상황에 맞는 전략!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 안테나 수만큼 전송률 증가

**미래 전망**: Massive MIMO로 확장

**참고 표준**: IEEE 802.11n, 3GPP

**📢 섹션 요약 비유**: 공간 다중화는 MIMO의 엔진!
