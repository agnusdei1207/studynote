# Reverse Proxy - 리버스 프록시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서버 앞단 중계
> 2. **가치**: 보안, 로드밸런싱, 캐시
> 3. **융합:** Nginx, HAProxy, SSL Termination

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Reverse Proxy는 서버 앞에서 요청을 중계합니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Upstream** | 백엔드 서버 |
| **Downstream** | 클라이언트 |
| **Health Check** | 상태 확인 |
| **Session** | 세션 유지 |
| **SSL** | 암호화 |

**핵심 알고리듬:** 요청 수신 → 라우팅 → 백엔드 전달

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Reverse Proxy 구현체**

| 소프트웨어 | 특징 |
|:---|:---|
| **Nginx** | 고성능, 설정 용이 |

| **Envoy** | 서비스 메시 |
| **Traefik** | 클라우드 네이티브 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
