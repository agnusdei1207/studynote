# 맨체스터 / 차분 맨체스터 (Manchester / Differential Manchester)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비트 중앙에서 반드시 전이 발생
> 2. **가치**: 자체 동기화, DC 성분 없음
> 3. **융합**: 10BASE-T, 토큰링, 이더넷과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

맨체스터 부호화는 각 비트 구간 중앙에서 반드시 전이가 발생하여 동기화를 보장하는 Line Coding 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    맨체스터 부호화                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              맨체스터 (Manchester)                            │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  규칙:                                                 │   │ │
│   │   │  • 1 = Low→High (상승 전이)                            │   │ │
│   │   │  • 0 = High→Low (하강 전이)                            │   │ │
│   │   │  • 항상 비트 중앙에서 전이                              │   │ │
│   │   │                                                      │   │ │
│   │   │  파형:                                                 │   │ │
│   │   │  데이터:    1     0     1     1     0                 │   │ │
│   │   │             ┌─┐   ┌─┐   ┌─┐   ┌─┐   ┌─┐             │   │ │
│   │   │           ──┘ └───┘ └───┘ └───┘ └───┘ └───           │   │ │
│   │   │             ↑     ↑     ↑     ↑     ↑                 │   │ │
│   │   │            중앙   중앙   중앙   중앙   중앙             │   │ │
│   │   │            전이    전이   전이   전이   전이            │   │ │
│   │   │                                                      │   │ │
│   │   │  장점: 자체 동기화, DC 성분 없음                       │   │ │
│   │   │  단점: 2배 대역폭 필요                                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              차분 맨체스터 (Differential Manchester)           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  규칙:                                                 │   │ │
│   │   │  • 비트 중앙: 항상 전이 (동기화)                        │   │ │
│   │   │  • 비트 시작:                                         │   │ │
│   │   │    - 1 = 전이 없음                                    │   │ │
│   │   │    - 0 = 전이 발생                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  파형:                                                 │   │ │
│   │   │  데이터:    1     0     1     1     0                 │   │ │
│   │   │             ┌─┐ ┌─┐   ┌─┐   ┌─┐ ┌─┐   ┌─┐           │   │ │
│   │   │           ──┘ └─┘ └───┘ └───┘ └───┘ └───┘ └─         │   │ │
│   │   │             ↑   ↑     ↑     ↑   ↑     ↑               │   │ │
│   │   │            중앙 시작  중앙  중앙 시작  중앙             │   │ │
│   │   │            전이  전이  전이  전이  전이  전이           │   │ │
│   │   │                                                      │   │ │
│   │   │  장점: 극성 반전에 강함                                │   │ │
│   │   │  응용: 토큰링 (IEEE 802.5)                             │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 맨체스터는 10BASE-T 이더넷의 표준 부호화입니다.

**💡 비유**: 맨체스터는 매 비트마다 박자를 치는 것과 같습니다.

**📢 섹션 요약 비유**: 맨체스터 = 10BASE-T 표준!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 맨체스터 | 차분 맨체스터 | 비고 |
|:---|:---|:---|:---|
| **중앙 전이** | 항상 | 항상 | 동기화 |
| **시작 전이** | 없음 | 0에서만 | 차등 |
| **극성 반전** | 민감 | 강함 | 차등 장점 |
| **응용** | 이더넷 | 토큰링 | IEEE 802 |

**비교표**

```
+------------+------------------+----------------------+
| 항목       | Manchester       | Differential         |
+------------+------------------+----------------------+
| 동기화     | 중앙 전이        | 중앙 전이            |
| 1 표현     | Low→High         | 시작 전이 없음       |
| 0 표현     | High→Low         | 시작 전이 있음       |
| 극성 반전  | 영향 받음        | 영향 없음            |
| 응용       | 10BASE-T         | 토큰링               |
+------------+------------------+----------------------+
```

**핵심 알고리즘**: 비트 → 중앙 전이 → 동기화

**📢 섹션 요약 비유**: 차분 맨체스터는 극성 반전에 강함!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 이더넷 부호화**

| 속도 | 부호화 | 동기화 |
|:---|:---|:---|
| **10Mbps** | Manchester | 자체 |
| **100Mbps** | 4B/5B + MLT-3 | 블록 |
| **1Gbps** | 8B/10B | 블록 |

**📢 섹션 요약 비유**: 100Mbps부터는 효율 위해 블록 부호화!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 10BASE-T** → Manchester
**시나리오 2: 토큰링** → Differential Manchester
**시나리오 3: 100BASE-TX** → 4B/5B + NRZ-I

**도입 체크리스트**: 속도, 거리, 동기화

**📢 섹션 요약 비유**: 고속은 Manchester 안 써요!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 자체 동기화, DC 없음

**미래 전망**: PAM-4, PAM-8 (고속)

**참고 표준**: IEEE 802.3 (10BASE-T), IEEE 802.5 (토큰링)

**📢 섹션 요약 비유**: 맨체스터는 동기화의 정석!
