# Codec - 코덱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 부호화/복호화 알고리즘
> 2. **가치**: 압축, 품질, 대역폭 최적화
> 3. **융합**: 오디오, 비디오, 스트리밍

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Codec(Coder-Decoder)은 미디어를 압축하고 해제합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Codec                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Codec 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [원본 미디어]                                       │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 인코딩 (Encoding)                              │   │ │
│   │   │     │ - 압축                                         │   │ │
│   │   │     │ - 변환                                         │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [압축된 데이터]                                     │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 전송/저장                                      │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [디코딩 (Decoding)]                                 │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ - 압축 해제                                     │   │ │
│   │   │     │ - 재생                                         │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [재생 미디어]                                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              오디오 코덱                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 코덱       │ 대역폭   │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ G.711      │ 64kbps   │ 전통     │               │   │ │
│   │   │  │ G.729      │ 8kbps    │ VoIP     │               │   │ │
│   │   │  | Opus       │ 6-510kbps│ WebRTC   │               │   │ │
│   │   │  │ AAC        │ 가변     │ 스트리밍 │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Codec은 압축률과 품질의 균형을 맞춥니다.

**💡 비유**: Codec은 짐 싸기입니다. 작게 싸서 크게 씁니다.

**📢 섹션 요약 비유:** Codec = 압축!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Encoder** | 인코더 |
| **Decoder** | 디코더 |
| **Bitrate** | 비트레이트 |
| **Lossy/Lossless** | 손실/무손실 |

**비교표: 비디오 코덱**

```
+------------+----------+----------+
| 코덱       │ 압축률   │ 용도     │
+------------+----------+----------+
| H.264      │ 중간     │ 범용     │
| H.265      │ 높음     │ 4K       │
| AV1        │ 높음     │ 차세대   │
| VP9        │ 높음     │ YouTube  │
+------------+----------+----------+
```

**핵심 알고리즸:** DCT, 움직임 추정

**📢 섹션 요약 비유:** H.265 = 고효율!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 오디오 품질**

| 항목 | G.711 | Opus | AAC |
|:---|:---|:---|:---|
| **품질** | 높음 | 높음 | 높음 |
| **대역폭** | 높음 | 가변 | 가변 |
| **지연** | 낮음 | 낮음 | 중간 |

**📢 섹션 요약 비유:** WebRTC = Opus!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: VoIP** → G.711/G.729
**시나리오 2: 스트리밍** → H.265/AAC
**시나리오 3: WebRTC** → Opus/VP9

**도입 체크리스트:** 품질, 대역폭, 지연

**📢 섹션 요약 비유:** 4K = H.265!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대역폭 절약, 품질 유지

**미래 전망:** AV1, VVC 확대

**참고 표준:** ITU-T, MPEG

**📢 섹션 요약 비유:** Codec은 미디어 필수!
