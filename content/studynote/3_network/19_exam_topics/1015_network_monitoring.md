# Network Monitoring - 네트워크 모니터링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 상태 관찰
> 2. **가치**: 장애 사전 감지
> 3. **융합:** SNMP, Syslog, NetFlow

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Network Monitoring은 네트워크 상태를 지속적으로 관찰합니다.

```

├─────────────────────────────────────────────────────────────────────────┤
```

**📢 섹션 요약 비유:** Network Monitoring = CCTV!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **NMS** | Network Management System |
| **Agent** | 에이전트 |
| **MIB** | Management Information Base |
| **Trap** | 트랩 (이벤트) |
| **Threshold** | 임계값 |

**핵심 알고리듬:** 데이터 수집 → 분석 → 임계값 비교 → 알림

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**모니터링 도구**

| 도구 | 용도 |
|:---|:---|
| **SNMP** | 장비 상태 |
| **Syslog** | 로그 수집 |
| **NetFlow** | 트래픽 분석 |
| **Ping** | 가용성 |
| **Grafana** | 시각화 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 기본 → SNMP + Syslog
**시나리오 2:** 트래픽 분석 → NetFlow
**시나리오 3:** 대시보드 → Grafana + Prometheus
