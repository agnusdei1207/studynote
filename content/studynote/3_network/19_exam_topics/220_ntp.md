# NTP (Network Time Protocol) - 시간 동기화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 시간 동기화
> 2. **가치**: 로그 정확성, 보안 인증
> 3. **융합**: DNS, 인증, PTP와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NTP(Network Time Protocol)는 네트워크로 연결된 장비들의 시간을 동기화하는 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NTP (Network Time Protocol)                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NTP 계층 구조 (Stratum)                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Stratum 0: GPS, 원자시계 (참조 소스)                │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  Stratum 1: 1차 NTP 서버 (GPS 직접 연결)             │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  Stratum 2: 2차 NTP 서버 (Stratum 1에서 동기화)      │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  Stratum 3: 3차 NTP 서버                             │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ...                                                 │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  Stratum 15: 최하위                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NTP 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  클라이언트                서버                      │   │ │
│   │   │     │                         │                      │   │ │
│   │   │     │  t1: 요청 전송          │                      │   │ │
│   │   │     │ ───────────────────────>│                      │   │ │
│   │   │     │                         │ t2: 요청 수신        │   │ │
│   │   │     │                         │ t3: 응답 전송        │   │ │
│   │   │     │  t4: 응답 수신          │                      │   │ │
│   │   │     │ <───────────────────────│                      │   │ │
│   │   │     │                         │                      │   │ │
│   │   │  Offset = ((t2-t1)+(t3-t4))/2                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: NTP는 왕복 지연을 계산하여 정확한 시간을 동기화합니다.

**💡 비유**: NTP는 모든 시계를 똑같은 시간으로 맞추는 것입니다.

**📢 섹션 요약 비유:** Stratum = 품질!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **포트** | UDP 123 |
| **정확도** | ms (LAN), ~100ms (WAN) |
| **참조 소스** | GPS, 원자시계, 라디오 |
| **Mode** | Client, Server, Peer |

**비교표: NTP vs PTP**

```
+------------+----------+----------+
| 항목       │ NTP      │ PTP      │
+------------+----------+----------+
| 정확도     │ ms       │ μs       │
| 용도       │ 일반     │ 정밀     │
| 계층       │ 인터넷   │ LAN      │
+------------+----------+----------+
```

**핵심 알고리즸:** 왕복 지연 계산, Stratum

**📢 섹션 요약 비유:** PTP = 정밀 동기화!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 시간 소스**

| 소스 | 정확도 | 비용 |
|:---|:---|:---|
| **GPS** | <1μs | 중간 |
| **원자시계** | <1ns | 높음 |
| **인터넷** | ~10ms | 무료 |

**📢 섹션 요약 비유:** GPS = 최고 정확도!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 기업** → 내부 NTP 서버
**시나리오 2: 금융** → GPS + PTP
**시나리오 3: 일반** → pool.ntp.org

**도입 체크리스트:** Stratum, 보안(인증), 이중화

**📢 섹션 요약 비유:** NTP 인증 = 보안!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 로그 정확성, 보안 인증

**미래 전망:** NTS (NTP over TLS), PTP 확대

**참고 표준:** RFC 5905 (NTPv4), IEEE 1588 (PTP)

**📢 섹션 요약 비유:** NTP는 시간 동기화의 표준!
