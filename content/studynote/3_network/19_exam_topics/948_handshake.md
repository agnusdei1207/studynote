# Handshake - 핸드셰이크

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연결 설정 절차
> 2. **가치**: 양방향 통신 준비
> 3. **융합:** 3-way, 4-way, SYN/ACK/FIN

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Handshake는 통신 전 연결을 설정하는 절차입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Handshake                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TCP 3-way Handshake                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  연결 설정:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Client                              Server      ││   │ │
│   │   │  │    │                                   │         ││   │ │
│   │   │  │    │─── SYN ───────────────────────→  │         ││   │ │
│   │   │  │    │     (Seq=x)                       │         ││   │ │
│   │   │  │    │                                   │         ││   │ │
│   │   │  │    │  ←── SYN-ACK ──────────────────   │         ││   │ │
│   │   │  │    │     (Seq=y, Ack=x+1)              │         ││   │ │
│   │   │  │    │                                   │         ││   │ │
│   │   │  │    │─── ACK ───────────────────────→  │         ││   │ │
│   │   │  │    │     (Ack=y+1)                     │         ││   │ │
│   │   │  │    │                                   │         ││   │ │
│   │   │  │    │          연결 완료                  │         ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  TCP 4-way Handshake (종료):                           │   │ │
│   │   │  FIN → ACK → FIN → ACK                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 핸드셰이크는 신뢰성 있는 연결의 시작입니다.

**📢 섹션 요약 비유:** Handshake = 악수!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SYN** | 연결 요청 |
| **ACK** | 확인 응답 |
| **FIN** | 연결 종료 |
| **Sequence** | 시퀀스 번호 |
| **State** | 상태 |

**핵심 알고리듬:** SYN → SYN-ACK → ACK

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**핸드셰이크 유형**

| 유형 | 단계 | 용도 |
|:---|:---|:---|
| **3-way** | 3 | 연결 설정 |
| **4-way** | 4 | 연결 종료 |
