# Context Switch - 문맥 교환

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 제어권 전환
> 2. **가치**: 멀티태스킹 지원
> 3. **융합:** Process, Thread, Scheduler

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Context Switch는 CPU를 다른 프로세스/스레드로 전환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Context Switch                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              문맥 교환 흐름                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Process A              Context Switch               │   │ │
│   │   │  ┌─────────┐            ┌─────────┐                  │   │ │
│   │   │  │ Running │ ─────────→ │  Save   │                  │   │ │
│   │   │  │ Context │            │ Context │                  │   │ │
│   │   │  └─────────┘            └────┬────┘                  │   │ │
│   │   │                              │                        │   │ │
│   │   │                              ↓                        │   │ │
│   │   │  Process B              ┌─────────┐                  │   │ │
│   │   │  ┌─────────┐            │ Restore │                  │   │ │
│   │   │  │ Running │ ←───────── │ Context │                  │   │ │
│   │   │  │ Context │            └─────────┘                  │   │ │
│   │   │  └─────────┘                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  저장 내용:                                          │   │ │
│   │   │  - Register (PC, SP, General)                        │   │ │
│   │   │  - Stack Pointer                                     │   │ │
│   │   │  - Memory Map                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Context Switch는 멀티태스킹의 핵심입니다.

**📢 섹션 요약 비유:** Context Switch = 작업 전환!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **PCB** | Process Control Block |
| **Register** | 레지스터 |
| **PC** | Program Counter |
| **SP** | Stack Pointer |
| **Memory Map** | 메모리 맵 |

**핵심 알고리즸:** 인터럽트 → 상태 저장 → 다음 선택 → 상태 복원
