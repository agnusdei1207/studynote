### Ⅰ. 개요 (Context & Background)

**개념 정의**

Event-Driven Architecture는 이벤트로 통신합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Event-Driven Architecture                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              이벤트 기반 구조                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Producer          Event Bus           Consumer      │   │ │
│   │   │  ┌─────────┐      ┌─────────┐        ┌─────────┐   │   │ │
│   │   │  │         │      │         │        │         │   │   │ │
│   │   │  │ Service │ ───→ │  Event  │ ───→ │ Service │   │   │ │
│   │   │  │    A    │      │  Broker │        │    B    │   │   │ │
│   │   │  │         │      │         │        │         │   │   │ │
│   │   │  └─────────┘      └─────────┘        └─────────┘   │   │ │
│   │   │                         │                           │   │ │
│   │   │                         ↓                           │   │ │
│   │   │                    ┌─────────┐                      │   │ │
│   │   │                    │ Service │                      │   │ │
│   │   │                    │    C    │                      │   │ │
│   │   │                    └─────────┘                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 이벤트 기반은 비동기, 느슨한 결합을 제공합니다.

**📢 섹션 요약 비유:** Event-Driven = 알림 서비스!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Event** | 이벤트 |
| **Producer** | 생산자 |
| **Consumer** | 소비자 |
| **Channel** | 채널 |
| **Broker** | 브로커 |

**핵심 알고리즸:** 이벤트 발생 → 브로커 전송 → 구독자 처리
