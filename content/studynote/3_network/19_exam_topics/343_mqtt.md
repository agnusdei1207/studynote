# MQTT - 경량 메시징 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 발행-구독 경량 메시징
> 2. **가치**: 저대역폭, 신뢰성, IoT 최적
> 3. **융합**: IoT, 센서, 모바일

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MQTT는 IoT를 위한 경량 발행-구독 메시징 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MQTT                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MQTT 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Publisher]                                        │   │ │
│   │   │  (센서, 디바이스)                                    │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Publish (토픽: sensor/temp)                    │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           MQTT Broker                       │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • Topic 관리                                 │    │   │ │
│   │   │  │  • QoS 보장                                   │    │   │ │
│   │   │  │  • 세션 관리                                   │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ Subscribe (토픽: sensor/temp)               │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [Subscriber]                                       │   │ │
│   │   │  (앱, 서버)                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              QoS 레벨                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ QoS       │ 보장     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 0          │ 없음     │ 센서     │               │   │ │
│   │   │  │ 1          │ 최소 1회 │ 일반     │               │   │ │
│   │   │  | 2          │ 정확 1회 │ 금융     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: MQTT는 브로커 기반 발행-구독으로 IoT에 최적화되어 있습니다.

**💡 비유**: MQTT는 구독 신문입니다. 원하는 주제만 받습니다.

**📢 섹션 요약 비유:** MQTT = IoT 메시징!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Broker** | 브로커 |
| **Topic** | 토픽 |
| **QoS** | 서비스 품질 |
| **Retain** | 보관 메시지 |

**비교표: MQTT vs 다른 프로토콜**

```
+------------+----------+----------+
| 항목       │ MQTT     │ AMQP     │
+------------+----------+----------+
| 모델       │ Pub/Sub  │ 큐       │
| 오버헤드   │ 낮음     │ 높음     │
| QoS        │ 3단계    │ 복잡     │
| 용도       │ IoT      │ 엔터프라이즈│
+------------+----------+----------+
```

**핵심 알고리즸:** 브로커 라우팅, QoS

**📢 섹션 요약 비유:** Topic = 주소!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: IoT 프로토콜**

| 항목 | MQTT | CoAP | HTTP |
|:---|:---|:---|:---|
| **모델** | Pub/Sub | Request/Response | Request/Response |
| **오버헤드** | 낮음 | 낮음 | 높음 |
| **신뢰성** | QoS | 확인 | TCP |

**📢 섹션 요약 비유:** IoT = MQTT!
