# JWT (JSON Web Token) - JSON 웹 토큰

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자가 수반형 토큰
> 2. **가치**: 무상태 인증
> 3. **융합:** OAuth, Session, Base64

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

JWT는 JSON 기반의 자가 수반형 토큰입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    JWT                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              JWT 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Header.Payload.Signature                            │   │ │
│   │   │  ↓       ↓        ↓                                  │   │ │
│   │   │  eyJhb...eyJzd...SflKx                                │   │ │
│   │   │                                                      │   │ │
│   │   │  Header: {"alg":"HS256","typ":"JWT"}                 │   │ │
│   │   │  Payload: {"sub":"123","name":"John","exp":...}      │   │ │
│   │   │  Signature: HMACSHA256(base64(Header.Payload), key)  │   │ │
│   │   │                                                      │   │ │
│   │   │  → Stateless, Self-contained                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: JWT는 서명으로 무결성을 보장합니다.

**📢 섹션 요약 비유:** JWT = 여권!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Header** | 알고리즘, 타입 |
| **Payload** | Claims (데이터) |
| **Signature** | 서명 |
| **iss** | 발급자 |
| **sub** | 주체 |
| **exp** | 만료 |

**핵심 알고리즸:** 생성 → 서명 → 검증

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**JWT vs Session**

| 항목 | JWT | Session |
|:---|:---|:---|
| **저장** | 클라이언트 | 서버 |
| **상태** | Stateless | Stateful |
| **확장** | 용이 | 어려움 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** API → JWT
**시나리오 2:** SSO → JWT
**시나리오 3:** 민감 데이터 → 서버 Session
