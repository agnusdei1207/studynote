# Kernel Mode - 커널 모드

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 특권 실행 모드
> 2. **가치**: 하드웨어 접근, 보호
> 3. **융합:** User Mode, Ring, Protection Ring

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Kernel Mode는 모든 하드웨어에 접근 가능한 특권 모드입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Kernel Mode                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Protection Ring                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │           Ring 0 (Kernel Mode)                       │   │ │
│   │   │           ┌─────────────────────────────────────┐    │   │ │
│   │   │           │ Kernel                               │    │   │ │
│   │   │           │ - 모든 하드웨어 접근                 │    │   │ │
│   │   │           │ - 모든 명령어 실행                   │    │   │ │
│   │   │           └─────────────────────────────────────┘    │   │ │
│   │   │                         ↑                             │   │ │
│   │   │           Ring 1, 2 (Device Drivers)                 │   │ │
│   │   │           ┌─────────────────────────────────────┐    │   │ │
│   │   │           │ Device Drivers                       │    │   │ │
│   │   │           └─────────────────────────────────────┘    │   │ │
│   │   │                         ↑                             │   │ │
│   │   │           Ring 3 (User Mode)                         │   │ │
│   │   │           ┌─────────────────────────────────────┐    │   │ │
│   │   │           │ Applications                         │    │   │ │
│   │   │           │ - 제한된 명령어만                    │    │   │ │
│   │   │           │ - 커널 서비스는 syscall로           │    │   │ │
│   │   │           └─────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Ring 구조로 권한을 계층화합니다.

**📢 섹션 요약 비유:** Kernel Mode = 관리자 권한!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Ring 0** | 커널 모드 |
| **Ring 3** | 사용자 모드 |
| **CPL** | Current Privilege Level |
| **DPL** | Descriptor Privilege Level |
| **IOPL** | I/O Privilege Level |

**핵심 알고리즸:** User Mode → System Call → Kernel Mode → 복귀

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Kernel Mode vs User Mode**

| 항목 | Kernel Mode | User Mode |
|:---|:---|:---|
| **하드웨어 접근** | 전체 | 제한 |
| **명령어** | 전체 | 안전한 명령만 |
| **메모리** | 전체 | 할당된 영역만 |
| **충돌 영향** | 시스템 전체 | 해당 프로세스 |
