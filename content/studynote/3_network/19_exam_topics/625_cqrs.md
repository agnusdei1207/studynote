# CQRS - 명령 조회 책임 분리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 읽기/쓰기 분리
> 2. **가치**: 성능 최적화, 확장성
> 3. **융합:** Event Sourcing, Microservices

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CQRS는 Command와 Query를 분리하는 패턴입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CQRS                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CQRS 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Command Side              Query Side                │   │ │
│   │   │  ┌─────────────┐          ┌─────────────┐           │   │ │
│   │   │  │   Command   │          │   Query     │           │   │ │
│   │   │  │   Handler   │          │   Handler   │           │   │ │
│   │   │  └──────┬──────┘          └──────┬──────┘           │   │ │
│   │   │         │                        │                   │   │ │
│   │   │         ↓                        ↓                   │   │ │
│   │   │  ┌─────────────┐          ┌─────────────┐           │   │ │
│   │   │  │  Write DB   │──────────│  Read DB    │           │   │ │
│   │   │  │ (Normalize) │  Sync    │ (Denorm)    │           │   │ │
│   │   │  └─────────────┘          └─────────────┘           │   │ │
│   │   │                                                      │   │ │
│   │   │  Command: Create, Update, Delete                     │   │ │
│   │   │  Query: Read, Search, List                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 읽기와 쓰기를 독립적으로 최적화합니다.

**📢 섹션 요약 비유:** CQRS = 입구/출구 분리!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Command** | 상태 변경 요청 |
| **Query** | 상태 조회 요청 |
| **Command Handler** | 명령 처리 |
| **Query Handler** | 조회 처리 |
| **Read Model** | 조회 모델 |
| **Write Model** | 쓰기 모델 |

**핵심 알고리즸:** Command → Write DB → (동기화) → Read DB ← Query

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**CQRS 장단점**

| 항목 | 장점 | 단점 |
|:---|:---|:---|
| **성능** | 최적화 가능 | 복잡도 증가 |
| **확장** | 독립 확장 | 동기화 필요 |
| **복잡도** | - | 운영 난이도 상승 |
