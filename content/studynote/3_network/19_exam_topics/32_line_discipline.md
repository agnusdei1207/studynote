# 회선 제어 규약 (Line Discipline)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 통신 회선의 흐름과 순서를 제어하는 규칙
> 2. **가치**: 신뢰성, 효율성, 장비 간 호환성
> 3. **융합**: ENQ/ACK, 폴링, HDLC와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

회선 제어 규약(Line Discipline)은 데이터 통신에서 회선의 사용 권한, 전송 순서, 오류 제어 등을 관리하는 규칙입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    회선 제어 규약 (Line Discipline)                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              회선 제어 기능                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 회선 제어 (Line Control):                          │   │ │
│   │   │     • 누가 전송할 것인가?                               │   │ │
│   │   │     • 전송 시작/종료                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 흐름 제어 (Flow Control):                          │   │ │
│   │   │     • 얼마나 보낼 것인가?                               │   │ │
│   │   │     • 수신측 처리 능력                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 에러 제어 (Error Control):                         │   │ │
│   │   │     • 오류 검출                                        │   │ │
│   │   │     • 재전송 요청                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 프레이밍 (Framing):                                 │   │ │
│   │   │     • 데이터 시작/끝 표시                               │   │ │
│   │   │     • 동기 유지                                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              제어 문자                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  ENQ (Enquiry): 통신 개시 요청                         │   │ │
│   │   │  ACK (Acknowledge): 긍정 응답                          │   │ │
│   │   │  NAK (Negative Acknowledge): 부정 응답                 │   │ │
│   │   │  EOT (End of Transmission): 전송 종료                  │   │ │
│   │   │  SYN (Synchronous Idle): 동기 문자                     │   │ │
│   │   │  DLE (Data Link Escape): 제어 문자 확장                │   │ │
│   │   │  STX (Start of Text): 데이터 시작                      │   │ │
│   │   │  ETX (End of Text): 데이터 종료                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 회선 제어는 데이터 링크 계층의 핵심 기능입니다.

**💡 비유**: 회선 제어는 교통 신호등처럼 데이터 흐름을 제어합니다.

**📢 섹션 요약 비유**: ENQ → ACK → 데이터 → EOT!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **ENQ** | 요청 | 통신 개시 | 노크 |
| **ACK** | 긍정 | 수락 | 네 |
| **NAK** | 부정 | 거부 | 아니오 |
| **EOT** | 종료 | 완료 | 끝 |

**기본 통신 시퀀스**

```
Point-to-Point 통신:

A국                             B국
 │                               │
 │──── ENQ (통신 요청) ─────────→│
 │                               │
 │←─── ACK (수락) ───────────────│
 │                               │
 │──── STX 데이터 ETX ──────────→│
 │                               │
 │←─── ACK (수신 확인) ──────────│
 │                               │
 │──── EOT (종료) ──────────────→│
 │                               │
```

**핵심 알고리즘**: 요청 → 응답 → 전송 → 확인 → 종료

**📢 섹션 요약 비유**: BSC가 대표적인 문자 지향 프로토콜!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 회선 제어 방식**

| 방식 | 제어 | 용도 |
|:---|:---|:---|
| **문자 지향** | 제어 문자 | BSC |
| **비트 지향** | 플래그 | HDLC |
| **바이트 카운트** | 길이 | DDCMP |

**📢 섹션 요약 비유**: HDLC가 현대 표준!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 레거시** → BSC (IBM 메인프레임)
**시나리오 2: WAN** → HDLC, PPP
**시나리오 3: LAN** → Ethernet (프레이밍)

**도입 체크리스트**: 동기 방식, 오류 제어, 효율성

**📢 섹션 요약 비유**: 제어 문자는 ASCII 0-31!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 신뢰성, 호환성 확보

**미래 전망**: 소프트웨어 정의 프로토콜

**참고 표준**: ISO 1745 (BSC), ISO 3309 (HDLC)

**📢 섹션 요약 비유**: 회선 제어는 통신의 교통 규칙!
