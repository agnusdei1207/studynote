# SOAP - 단순 객체 접근 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: XML 기반 웹 서비스 프로토콜
> 2. **가치**: WS-*, 엔터프라이즈
> 3. **융합**: XML, HTTP, WSDL

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SOAP(Simple Object Access Protocol)은 XML 기반 웹 서비스 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SOAP                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SOAP 메시지 구조                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  <Envelope>                                          │   │ │
│   │   │    <Header>                                          │   │ │
│   │   │      <!-- 메타데이터, 인증 등 -->                     │   │ │
│   │   │    </Header>                                         │   │ │
│   │   │    <Body>                                            │   │ │
│   │   │      <!-- 실제 데이터 -->                             │   │ │
│   │   │      <GetPrice>                                      │   │ │
│   │   │        <Item>Apple</Item>                            │   │ │
│   │   │      </GetPrice>                                     │   │ │
│   │   │    </Body>                                           │   │ │
│   │   │  </Envelope>                                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              WS-* 표준                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 표준       │ 기능     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ WS-Security│ 보안     │ 암호화   │               │   │ │
│   │   │  │ WS-Addressing│ 주소  │ 라우팅   │               │   │ │
│   │   │  | WS-ReliableMessaging│ 신뢰성│ 보장     │               │   │ │
│   │   │  │ WS-AtomicTransaction│ 트랜잭션│ 분산     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SOAP은 Envelope/Header/Body로 구조화된 메시지를 전송합니다.

**💡 비유**: SOAP은 견적서입니다. 정해진 형식이 있습니다.

**📢 섹션 요약 비유:** SOAP = 엔터프라이즈!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Envelope** | 봉투 |
| **Header** | 헤더 |
| **Body** | 본문 |
| **Fault** | 오류 |

**비교표: SOAP vs REST**

```
+------------+----------+----------+
| 항목       │ SOAP     │ REST     │
+------------+----------+----------+
| 포맷       │ XML      │ JSON/XML │
| 프로토콜   │ HTTP/SMTP│ HTTP     │
| 복잡도     │ 높음     │ 낮음     │
| WS-*       │ 지원     │ 미지원   │
+------------+----------+----------+
```

**핵심 알고리즸:** XML, WSDL

**📢 섹션 요약 비유:** WSDL = 계약서!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 웹 서비스**

| 항목 | SOAP | REST | GraphQL |
|:---|:---|:---|:---|
| **스타일** │ RPC │ 리소스 │ 쿼리 |
| **형식** │ XML │ JSON │ JSON |
| **복잡도** │ 높음 │ 낮음 │ 중간 |

**📢 섹션 요약 비유:** 금융 = SOAP!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 금융 → SOAP + WS-Security
**시나리오 2:** 레거시 → SOAP
**시나리오 3:** 신규 → REST/GraphQL

**도입 체크리스트:** WSDL, WS-*, 엔터프라이즈

**📢 섹션 요약 비유:** WS-* = 기업용!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 엔터프라이즈 기능
