# Process - 프로세스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실행 중인 프로그램
> 2. **가치**: 격리, 보호
> 3. **융합:** OS, Memory, Scheduling

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Process는 실행 중인 프로그램 인스턴스입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Process                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              프로세스 구조                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │              Process Memory                  │    │   │ │
│   │   │  │  ┌─────────────────────────────────────┐    │    │   │ │
│   │   │  │  │ Stack (지역 변수, 함수 호출)          │    │    │   │ │
│   │   │  │  │         ↓                           │    │    │   │ │
│   │   │  │  │         ↑                           │    │    │   │ │
│   │   │  │  │ Heap (동적 할당)                    │    │    │   │ │
│   │   │  │  │ BSS (초기화 안 된 전역)             │    │    │   │ │
│   │   │  │  │ Data (초기화된 전역)                │    │    │   │ │
│   │   │  │  │ Text (코드)                         │    │    │   │ │
│   │   │  │  └─────────────────────────────────────┘    │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  PCB (Process Control Block):                        │   │ │
│   │   │  PID, State, PC, Register, Priority, Memory Info     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 프로세스는 독립된 메모리 공간을 갖습니다.

**📢 섹션 요약 비유:** Process = 공장!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **PID** | 프로세스 ID |
| **PCB** | 프로세스 제어 블록 |
| **Text** | 코드 영역 |
| **Data** | 데이터 영역 |
| **Heap** | 힙 영역 |
| **Stack** | 스택 영역 |

**핵심 알고리즸:** 생성 → 준비 → 실행 → 대기 → 종료

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**프로세스 상태**

| 상태 | 설명 |
|:---|:---|
| **New** | 생성 |
| **Ready** | 실행 대기 |
| **Running** | 실행 중 |
| **Waiting** | I/O 대기 |
| **Terminated** | 종료 |
