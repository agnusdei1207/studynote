# NFV (네트워크 기능 가상화) - Network Function Virtualization

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 기능의 가상화
> 2. **가치**: 하드웨어 독립, 비용 절감
> 3. **융합**: VNF, MANO, 클라우드

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NFV는 네트워크 기능을 가상화하여 범용 서버에서 실행합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NFV 아키텍처                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NFV 프레임워크                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ VNF (Virtual Network Function)              │    │   │ │
│   │   │  │ - 방화벽, 라우터, 로드밸런서                │    │   │ │
│   │   │  │ - 가상화된 네트워크 기능                    │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ NFVI (NFV Infrastructure)                   │    │   │ │
│   │   │  │ - 컴퓨트, 스토리지, 네트워크                │    │   │ │
│   │   │  │ - 가상화 플랫폼                             │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ MANO (Management & Orchestration)           │    │   │ │
│   │   │  │ - VNFM, NFVO, VIM                           │    │   │ │
│   │   │  │ - 자동화된 관리                             │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NFV vs 전통적 네트워크                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ 전통적   │ NFV      │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 하드웨어   │ 전용     │ 범용     │               │   │ │
│   │   │  │ 배포       │ 물리     │ 가상     │               │   │ │
│   │   │  | 확장       │ 어려움   │ 쉬움     │               │   │ │
│   │   │  │ 비용       │ CAPEX    │ OPEX     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: NFV는 전용 하드웨어를 가상화로 대체합니다.

**💡 비유**: NFV는 가상 머신으로 네트워크 장비를 대체합니다.

**📢 섹션 요약 비유:** NFV = 가상 네트워크 기능!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **VNF** | Virtual Network Function |
| **NFVI** | NFV Infrastructure |
| **MANO** | Management and Orchestration |
| **VIM** | Virtual Infrastructure Manager |

**비교표: VNF 유형**

```
+------------+----------+----------+----------+
| VNF 유형   │ 기능     │ 예시     │ 용도     │
+------------+----------+----------+----------+
| vEPC       │ 코어     │ EPC      │ 4G/5G    │
| vCPE       │ 엣지     │ 라우터   │ 기업     │
| vFW        │ 보안     │ 방화벽   │ 보안     │
| vLB        │ 트래픽   │ 로드밸런서│ 분산    │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 오케스트레이션, 스케일링

**📢 섹션 요약 비유:** MANO = NFV 관리!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: NFV vs SDN**

| 항목 | NFV | SDN |
|:---|:---|:---|
| **목적** | 기능 가상화 | 제어 분리 |
| **방법** | VM/컨테이너 | 중앙 제어 |
| **결합** | 상호 보완 | 상호 보완 |

**📢 섹션 요약 비유:** NFV + SDN = 완벽!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 통신사 → vEPC + vIMS
**시나리오 2:** 기업 → vCPE + vFW
**시나리오 3:** 클라우드 → vLB + vRouter

**도입 체크리스트:** 성능, HA, 관리

**📢 섹션 요약 비유:** 범용 하드웨어 = 비용 절감!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: TCO 절감

**미래 전망:** CNF (Containerized NF)

**참고 표준:** ETSI NFV

**📢 섹션 요약 비유:** NFV는 네트워크 가상화의 핵심!
