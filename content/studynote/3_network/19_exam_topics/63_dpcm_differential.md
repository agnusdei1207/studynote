# DPCM (차분 펄스 부호 변조)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이전 샘플과의 차이만 전송
> 2. **가치**: PCM 대역폭 절감, 효율 향상
> 3. **융합**: 예측 부호화, 음성 통신과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DPCM(Differential PCM)은 현재 샘플과 이전 샘플의 차이(차분)를 전송하여 비트율을 줄이는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DPCM (차분 펄스 부호 변조)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DPCM 원리                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  PCM:                                                    │   │ │
│   │   │  샘플: 100, 105, 108, 110, 112                         │   │ │
│   │   │  전송: 100, 105, 108, 110, 112 (각각 8비트)             │   │ │
│   │   │                                                      │   │ │
│   │   │  DPCM:                                                  │   │ │
│   │   │  샘플: 100, 105, 108, 110, 112                         │   │ │
│   │   │  차분:  100,  +5,  +3,  +2,  +2                        │   │ │
│   │   │  전송: 100, +5, +3, +2, +2 (차분은 4비트)               │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                 │   │ │
│   │   │  • 인접 샘플 간 상관성 높음                             │   │ │
│   │   │  • 차분 값이 원래 값보다 작음                           │   │ │
│   │   │  • 비트율 감소                                         │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DPCM 송수신 구조                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  송신측:                                                 │   │ │
│   │   │  x[n] → (+) ─→ e[n] → 양자화 → 전송                   │   │ │
│   │   │         ↑                                │              │   │ │
│   │   │         └────────── 예측기 ←── x̂[n-1] ──┘              │   │ │
│   │   │                                                      │   │ │
│   │   │  수신측:                                                 │   │ │
│   │   │  수신 → 역양자화 → (+) → x̂[n]                         │   │ │
│   │   │                      ↑       │                        │   │ │
│   │   │                      └──예측기←┘                        │   │ │
│   │   │                                                      │   │ │
│   │   │  e[n] = x[n] - x̂[n-1]  (예측 오차)                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: DPCM은 음성/영상의 인접 샘플 간 상관성을 이용합니다.

**💡 비유**: DPCM은 "어제보다 2도 더워"라고 말하는 것입니다.

**📢 섹션 요약 비유**: PCM 64kbps → DPCM 32kbps!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **예측기** | 이전값 예측 | P(z) | 추측 |
| **차분** | 오차 계산 | e[n] | 차이 |
| **양자화** | 차분 양자화 | Q(·) | 근사 |
| **복원** | 예측+오차 | x̂[n] | 합성 |

**비교표**

```
+------------+----------+----------+
| 항목       | PCM      | DPCM     |
+------------+----------+----------+
| 전송량     | 전체값   | 차분만   |
| 비트율     | 64kbps   | 32kbps   |
| 복잡도     | 낮음     | 중간     |
| SNR        | 높음     | 비슷     |
+------------+----------+----------+
```

**핵심 알고리즘**: x[n] - 예측값 → 차분 양자화 → 전송

**📢 섹션 요약 비유**: 50% 대역폭 절감!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 부호화 효율**

| 방식 | 비트율 | 압축률 |
|:---|:---:|:---:|
| **PCM** | 64kbps | 1:1 |
| **DPCM** | 32kbps | 2:1 |
| **ADPCM** | 24kbps | 2.7:1 |

**📢 섹션 요약 비유**: DPCM은 기본 예측 부호화!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 음성 통신** → DPCM 32kbps
**시나리오 2: 영상** → 차분 프레임
**시나리오 3: 저장** → 무손실 압축

**도입 체크리스트**: 상관성, 지연, 복잡도

**📢 섹션 요약 비유**: 음성은 샘플 간 변화가 작아 DPCM 효과적!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 50% 대역폭 절감

**미래 전망**: ADPCM, CELP로 진화

**참고 표준**: ITU-T G.726

**📢 섹션 요약 비유**: DPCM은 예측 부호화의 시작!
