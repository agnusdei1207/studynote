# AIMD (가산 증가 승산 감소) - Additive Increase Multiplicative Decrease

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: TCP 혼잡 제어 핵심 알고리즘
> 2. **가치**: 공정성, 안정성
> 3. **융합:** TCP Reno, Congestion Avoidance

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

AIMD는 선형 증가, 반감소 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    AIMD 동작                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              AIMD 알고리즸                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  AI (Additive Increase):                            │   │ │
│   │   │  - 매 RTT마다 cwnd += 1 MSS                         │   │ │
│   │   │  - 선형 증가                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  MD (Multiplicative Decrease):                      │   │ │
│   │   │  - 손실 시 cwnd = cwnd / 2                          │   │ │
│   │   │  - 반감소                                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: AIMD는 공정성과 효율성을 동시에 달성합니다.

**📢 섹션 요약 비유:** AIMD = 조심스런 증가!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **AI** | Additive Increase |

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**TCP 혼잡 제어 단계**

| 단계 | 조건 | 동작 |
|:---|:---|:---|
| **Slow Start** | cwnd < ssthresh | 지수 증가 |
| **CA** | cwnd ≥ ssthresh | AIMD |
| **Recovery** | 손실 | MD |
