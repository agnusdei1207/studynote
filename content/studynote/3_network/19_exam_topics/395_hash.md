# 해시 함수 - Hash Function

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 임의 길이 입력 → 고정 길이 출력
> 2. **가치**: 무결성, 패스워드 저장
> 3. **융합**: 디지털 서명, 블록체인

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

해시 함수는 임의 길이 데이터를 고정 길이 값으로 변환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    해시 함수                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              해시 함수 특성                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [임의 길이 입력]                                    │   │ │
│   │   │        │                                             │   │ │
│   │   │        │ Hash 함수                                   │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  [고정 길이 출력]                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  "Hello" → SHA-256 → 185f8db3...                    │   │ │
│   │   │  "Hello!" → SHA-256 → 9b71d224...                   │   │ │
│   │   │                                                      │   │ │
│   │   │  특성:                                               │   │ │
│   │   │  • 역상 저항성: 해시 → 원본 복구 불가                │   │ │
│   │   │  • 제2 역상 저항성: 같은 해시값 찾기 어려움          │   │ │
│   │   │  • 충돌 저항성: 같은 해시 쌍 찾기 어려움             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              해시 함수 종류                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 종류       │ 출력길이  │ 상태     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ MD5        │ 128비트  │ 취약     │               │   │ │
│   │   │  │ SHA-1      │ 160비트  │ 취약     │               │   │ │
│   │   │  | SHA-256    │ 256비트  │ 안전     │               │   │ │
│   │   │  │ SHA-3      │ 가변     │ 안전     │               │   │ │
│   │   │  │ BLAKE3     │ 256비트  │ 안전     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 해시 함수는 단방향 함수입니다.

**💡 비유**: 해시 함수는 고기 분쇄기입니다. 다시 원래대로 될 수 없습니다.

**📢 섹션 요약 비유:** 해시 = 단방향!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Preimage Resistance** | 역상 저항성 |
| **Second Preimage** | 제2 역상 저항성 |
| **Collision Resistance** | 충돌 저항성 |
| **Avalanche Effect** | 눈사태 효과 |

**비교표: SHA-2 변종**

```
+------------+----------+----------+
| 항목       │ SHA-256  │ SHA-512  │
+------------+----------+----------+
| 출력       │ 256비트  │ 512비트  │
| 블록       │ 512비트  │ 1024비트 │
| 라운드     │ 64       │ 80       │
| 용도       │ 범용     │ 고보안   │
+------------+----------+----------+
```

**핵심 알고리즸:** Merkle-Damgård 구조, 스펀지 구조

**📢 섹션 요약 비유:** SHA-256 = 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 용도별 해시**

| 항목 | 무결성 | 패스워드 | 블록체인 |
|:---|:---|:---|:---|
| **알고리즘** │ SHA-256 │ bcrypt/Argon2 | SHA-256 |
| **특징** │ 빠름 │ 느림 | 빠름 |

**📢 섹션 요약 비유:** 패스워드 = 전용 해시!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 무결성 검증 → SHA-256
**시나리오 2:** 패스워드 저장 → bcrypt/Argon2
**시나리오 3:** 디지털 서명 → SHA-256/SHA-3

**도입 체크리스트:** 용도별 선택, 솔트 사용, 키 스트레칭

**📢 섹션 요약 비유:** MD5/SHA-1 = 사용 금지!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 무결성 보장

**미래 전망:** SHA-3, 경량 해시

**참고 표준:** FIPS 180-4 (SHA-2), FIPS 202 (SHA-3)

**📢 섹션 요약 비유:** 해시는 무결성의 핵심!
