# Exception - 예외

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동기 이벤트/오류
> 2. **가치**: 오류 처리, 보호
> 3. **융합:** Interrupt, Trap, Signal

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Exception은 프로그램 실행 중 발생하는 특수 상황입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Exception                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Exception 종류                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Fault (복구 가능)                           │    │   │ │
│   │   │  │ - Page Fault: 메모리 로딩 후 재실행         │    │   │ │
│   │   │  │ - General Protection: 보안 위반             │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Trap (의도적)                               │    │   │ │
│   │   │  │ - System Call: 커널 진입                    │    │   │ │
│   │   │  │ - Breakpoint: 디버깅                       │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Abort (치명적)                              │    │   │ │
│   │   │  │ - Double Fault: 복구 불가                  │    │   │ │
│   │   │  │ - Machine Check: 하드웨어 오류             │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Exception은 동기적으로 발생하며 처리 방식이 다릅니다.

**📢 섹션 요약 비유:** Exception = 비상구!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Fault** | 복구 가능, 재실행 |
| **Trap** | 의도적, 다음 명령 |
| **Abort** | 치명적, 종료 |
| **Exception Handler** | 처리 루틴 |
| **Exception Code** | 예외 코드 |

**핵심 알고리즸:** 예외 발생 → 핸들러 호출 → 처리/종료

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Exception vs Interrupt**

| 항목 | Exception | Interrupt |
|:---|:---|:---|
| **발생 시점** | 동기 | 비동기 |
| **원인** | 프로그램 내부 | 외부 이벤트 |
| **복귀** | 오류 위치/다음 | 인터럽트 위치 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 페이지 폴트 → Fault 처리
**시나리오 2:** 0 나누기 → Fault/Abort
**시나리오 3:** 시스템 콜 → Trap
