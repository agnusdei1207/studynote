# VXLAN - 가상 확장 LAN

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: L2 오버레이 네트워크
> 2. **가치**: 16M 세그먼트, 데이터센터 확장
> 3. **융합**: EVPN, Spine-Leaf와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

VXLAN(Virtual Extensible LAN)은 UDP 오버레이를 사용하여 L2 네트워크를 확장하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    VXLAN                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VXLAN 구조                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [VM 1]              [Underlay]              [VM 2]│   │ │
│   │   │  VLAN 10                                     VLAN 10│   │ │
│   │   │     │                                           │    │   │ │
│   │   │     ↓                                           ↓    │   │ │
│   │   │  [VTEP] ──────── VXLAN Tunnel ────────── [VTEP]  │   │ │
│   │   │     │             (UDP 4789)               │       │   │ │
│   │   │     │              VNI: 10010              │       │   │ │
│   │   │     ↓                                           ↓    │   │ │
│   │   │  IP 네트워크 (Underlay)                           │   │ │
│   │   │                                                      │   │ │
│   │   │  캡슐화:                                             │   │ │
│   │   │  [원본 L2 프레임] + VXLAN 헤더 + UDP + IP          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VXLAN vs VLAN                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  항목     │ VLAN        │ VXLAN                     │   │ │
│   │   │  ────────┼─────────────┼──────────────────────────│   │ │
│   │   │  ID 공간 │ 12비트      │ 24비트                    │   │ │
│   │   │  세그먼트 │ 4,096       │ 16,777,216                │   │ │
│   │   │  범위     │ L2          │ L3 오버레이               │   │ │
│   │   │  확장     │ 제한        │ IP 네트워크 전체          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: VXLAN은 24비트 VNI로 1600만 개 이상의 세그먼트를 지원합니다.

**💡 비유**: VXLAN은 L2 네트워크의 VPN입니다. IP 네트워크 위로 L2를 확장합니다.

**📢 섹션 요약 비유:** VXLAN = L2 over UDP!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **VNI** | VXLAN Network ID (24비트) |
| **VTEP** | VXLAN Tunnel Endpoint |
| **UDP 포트** | 4789 (IANA) |
| **멀티캐스트** | BUM 트래픽 |

**비교표: 오버레이 기술**

```
+------------+----------+----------+
| 기술       │ 터널     │ ID 공간  │
+------------+----------+----------+
| VLAN       │ 없음     │ 4K       │
| VXLAN      │ UDP      │ 16M      │
| NVGRE      │ GRE      │ 16M      │
| Geneve     │ UDP      │ 가변     │
+------------+----------+----------+
```

**핵심 알고리즸:** VNI 매핑, VTEP 발견

**📢 섹션 요약 비유:** VTEP = VXLAN 게이트웨이!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: BUM 처리**

| 방식 | 설명 |
|:---|:---|
| **멀티캐스트** | Underlay 멀티캐스트 |
| **유니캐스트** | Head-end 복제 |
| **EVPN** | Type 3 경로 |

**📢 섹션 요약 비유:** EVPN = BUM 처리 최적화!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 데이터센터** → VXLAN + EVPN
**시나리오 2: 멀티테넌시** → VNI per Tenant
**시나리오 3: DCI** → VXLAN over WAN

**도입 체크리스트:** Underlay 설계, VTEP, MTU

**📢 섹션 요약 비유:** MTU = 9000 (점보 프레임)!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 16M 세그먼트, 확장성

**미래 전망:** EVPN-VXLAN 표준

**참고 표준:** RFC 7348

**📢 섹션 요약 비유:** VXLAN은 데이터센터 오버레이의 표준!
