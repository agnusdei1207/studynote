# Microservices Architecture - 마이크로서비스 아키텍처

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 작은 독립 서비스들
> 2. **가치**: 확장성, 유연성
> 3. **융합:** API Gateway, Container, Service Mesh

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Microservices는 애플리케이션을 작은 서비스로 분리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Microservices                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Microservices Architecture                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Monolith:              Microservices:               │   │ │
│   │   │  ┌───────────┐         ┌─────┐ ┌─────┐ ┌─────┐     │   │ │
│   │   │  │           │         │User │ │Order│ │Pay  │     │   │ │
│   │   │  │   App     │   →     │ Svc │ │ Svc │ │ Svc │     │   │ │
│   │   │  │           │         └─────┘ └─────┘ └─────┘     │   │ │
│   │   │  └───────────┘                                     │   │ │
│   │   │                          ↑                          │   │ │
│   │   │                    API Gateway                      │   │ │
│   │   │                          ↑                          │   │ │
│   │   │                        Client                        │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                               │   │ │
│   │   │  - 독립 배포                                         │   │ │
│   │   │  - 기술 스택 다양성                                  │   │ │
│   │   │  - 장애 격리                                         │   │ │
│   │   │  - 확장성                                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 각 서비스는 독립적으로 개발, 배포, 확장됩니다.

**📢 섹션 요약 비유:** Microservices = 전문 부서!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Service** | 독립 서비스 |
| **API Gateway** | 진입점 |
| **Service Discovery** | 서비스 탐색 |
| **Message Queue** | 비동기 통신 |
| **Circuit Breaker** | 장애 격리 |

**핵심 알고리즸:** 요청 → Gateway → Service → 응답

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Monolith vs Microservices**

| 항목 | Monolith | Microservices |
|:---|:---|:---|
| **복잡도** | 낮음 | 높음 |
| **확장** | 전체 | 개별 |
| **배포** | 전체 | 개별 |
