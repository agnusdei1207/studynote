# NAK - 부정 확인 응답

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 수신 실패 통보
> 2. **가치**: 빠른 재전송
> 3. **융합:** ARQ, Selective Repeat, NACK

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NAK (Negative Acknowledgment)는 수신 실패를 통보합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NAK (Negative Acknowledgment)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NAK 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  기본 동작:                                            │   │ │
│   │   │  Sender                     Receiver                 │   │ │
│   │   │  ┌───────┐                  ┌───────┐               │   │ │
│   │   │  │       │ ── Seg 1 ──────→ │       │               │   │ │
│   │   │  │       │ ── Seg 2 (X) ──→ │       │ ← 손실        │   │ │
│   │   │  │       │ ── Seg 3 ──────→ │       │               │   │ │
│   │   │  │       │                  │       │               │   │ │
│   │   │  │       │ ←── NAK 2 ────── │       │               │   │ │
│   │   │  │       │   (2번 재전송 요청) │     │               │   │ │
│   │   │  │       │                  │       │               │   │ │
│   │   │  │       │ ── Seg 2 ──────→ │       │               │   │ │
│   │   │  └───────┘                  └───────┘               │   │ │
│   │   │                                                      │   │ │
│   │   │  TCP와 NAK:                                           │   │ │
│   │   │  - TCP은 NAK 없음                                      │   │ │
│   │   │  - Duplicate ACK로 대체                                │   │ │
│   │   │  - 3 Duplicate ACK = Fast Retransmit                  │   │ │
│   │   │                                                      │   │ │
│   │   │  NAK 사용 프로토콜:                                     │   │ │
│   │   │  - HDLC                                               │   │ │
│   │   │  - Selective Repeat ARQ                                │   │ │
│   │   │  - SCTP                                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: NAK는 타임아웃 없이 빠른 재전송을 가능하게 합니다.

**📢 섹션 요약 비유:** NAK = 반품 요청!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **NAK** | 부정 확인 응답 |
| **Selective** | 선택적 |
| **Gap** | 갭 |
| **Timeout** | 타임아웃 |
| **Fast Retransmit** | 빠른 재전송 |

**핵심 알고리즸:** 손실 감지 → NAK 전송 → 재전송

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ACK vs NAK**

| 항목 | ACK | NAK |
|:---|:---|:---|
| **의미** | 수신 성공 | 수신 실패 |
| **TCP** | 있음 | 없음 (Dup ACK) |
| **지연** | 타임아웃 없음 | 타임아웃 없음 |
| **효율** | 양호 | 빠름 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** HDLC → NAK 사용
**시나리오 2:** TCP → Duplicate ACK
**시나리오 3:** SCTP → SACK + NAK
