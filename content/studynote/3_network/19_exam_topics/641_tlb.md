# TLB (Translation Lookaside Buffer) - TLB

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 주소 변환 캐시
> 2. **가치**: 메모리 접근 속도 향상
> 3. **융합:** Paging, MMU, Cache

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TLB는 페이지 테이블 엔트리를 캐싱합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TLB                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TLB 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  CPU                                                  │   │ │
│   │   │    │                                                 │   │ │
│   │   │    ↓ (Virtual Address)                               │   │ │
│   │   │  ┌─────────────────────────────────────┐            │   │ │
│   │   │  │              TLB                    │            │   │ │
│   │   │  │  ┌───────────────────────────────┐ │            │   │ │
│   │   │  │  │ VPN │ PFN │ Valid │ Protection│ │            │   │ │
│   │   │  │  │  0  │  5  │   1   │    RW     │ │            │   │ │
│   │   │  │  │  1  │  3  │   1   │    R      │ │            │   │ │
│   │   │  │  │ ... │ ... │  ...  │   ...     │ │            │   │ │
│   │   │  │  └───────────────────────────────┘ │            │   │ │
│   │   │  └─────────────────────────────────────┘            │   │ │
│   │   │    │ Hit                               │ Miss        │   │ │
│   │   │    ↓                                   ↓             │   │ │
│   │   │  (빠름)                          Page Table          │   │ │
│   │   │                                   (느림)              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: TLB는 자주 사용하는 주소 변환을 캐싱합니다.

**📢 섹션 요약 비유:** TLB = 자주 찾는 페이지 책갈피!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **VPN** | Virtual Page Number |
| **PFN** | Physical Frame Number |
| **Valid** | 유효 비트 |
| **Protection** | 보호 비트 |
| **ASID** | 주소 공간 ID |

**핵심 알고리즸:** VPN → TLB 조회 → (Hit) PFN / (Miss) Page Table

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**TLB 성능**

| 항목 | 설명 |
|:---|:---|
| **TLB Hit** | 캐시 적중 (빠름) |
| **TLB Miss** | 캐시 미스 (느림) |
| **Hit Rate** | 적중률 |
| **TLB Size** | 엔트리 수 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 높은 지역성 → TLB 효과적
**시나리오 2:** 큰 메모리 → Larger TLB
**시나리오 3:** 가상화 → Nested TLB / EPT
