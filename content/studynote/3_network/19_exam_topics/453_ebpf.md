# eBPF (확장 버클리 패킷 필터) - Extended Berkeley Packet Filter

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널에서 안전하게 프로그램 실행
> 2. **가치**: 고성능, 유연성, 관찰 가능성
> 3. **융합**: XDP, Cilium, Observability

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

eBPF는 리눅스 커널에서 샌드박스 프로그램을 실행합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    eBPF 아키텍처                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              eBPF 동작 방식                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ User Space                                   │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  ┌─────────┐                              │    │   │ │
│   │   │  │  │ eBPF    │                              │    │   │ │
│   │   │  │  │ Program │                              │    │   │ │
│   │   │  │  │ (C/Rust)│                              │    │   │ │
│   │   │  │  └────┬────┘                              │    │   │ │
│   │   │  └───────┼───────────────────────────────────┘    │   │ │
│   │   │          │ bpf() syscall                          │   │ │
│   │   │          ▼                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Kernel Space                                 │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  Verifier → JIT → Hook Points              │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  Hook: XDP, TC, Tracepoint, Socket...      │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Hook Point                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Hook       │ 위치     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ XDP        │ 드라이버 │ DDoS     │               │   │ │
│   │   │  │ TC         │ Qdisc    │ 라우팅   │               │   │ │
│   │   │  | Socket     │ 소켓     │ 필터     │               │   │ │
│   │   │  │ Tracepoint │ 커널     │ 추적     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: eBPF는 커널을 수정하지 않고 확장합니다.

**💡 비유**: eBPF는 커널의 플러그인 시스템입니다.

**📢 섹션 요약 비유:** eBPF = 커널 확장!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Verifier** | 안전성 검증 |
| **JIT** | Just-In-Time 컴파일 |
| **Maps** | 데이터 저장소 |
| **Helper** | 커널 함수 호출 |

**비교표: eBPF 활용**

```
+------------+----------+----------+----------+
| 영역       │ 도구     │ 용도     │ 특징     │
+------------+----------+----------+----------+
| 네트워크   │ Cilium   │ K8s      │ L7       │
| 보안       │ Falco    │ 모니터링 │ 런타임   │
| 관찰       │ bpftrace │ 추적     │ 동적     │
| 성능       │ XDP      │ DDoS     │ 라인레이트│
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 검증, JIT 컴파일

**📢 섹션 요약 비유:** Verifier = 안전!
