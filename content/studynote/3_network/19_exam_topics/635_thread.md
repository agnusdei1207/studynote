# Thread - 스레드

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 실행 단위
> 2. **가치**: 자원 공유, 경량 프로세스
> 3. **융합:** Process, Multi-threading, Concurrency

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Thread는 프로세스 내의 실행 단위입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Thread                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Process vs Thread                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Process:                    Threads in Process:     │   │ │
│   │   │  ┌───────────────────┐      ┌───────────────────┐   │   │ │
│   │   │  │ Code │ Data │ Heap│      │ Thread 1 │ Thread 2│   │   │ │
│   │   │  │                   │      │ Reg │ PC │ Reg │ PC│   │   │ │
│   │   │  │ Stack             │      │ Stack   │ Stack   │   │   │ │
│   │   │  │ (per Thread)      │      │         │         │   │   │ │
│   │   │  └───────────────────┘      └───────────────────┘   │   │ │
│   │   │                                                      │   │ │
│   │   │  공유: Code, Data, Heap                              │   │ │
│   │   │  독립: Register, PC, Stack                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스레드는 같은 프로세스 내 자원을 공유합니다.

**📢 섹션 요약 비유:** Thread = 일꾼!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Thread ID** | 스레드 식별자 |
| **Register** | 레지스터 |
| **PC** | 프로그램 카운터 |
| **Stack** | 스택 |
| **TLS** | Thread Local Storage |

**핵심 알고리즸:** 생성 → 실행 → 차단 → 종료

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Thread vs Process**

| 항목 | Thread | Process |
|:---|:---|:---|
| **생성 비용** | 낮음 | 높음 |
| **자원 공유** | 쉬움 | 어려움 |
| **안정성** | 낮음 | 높음 |
| **컨텍스트 스위치** | 빠름 | 느림 |
