# System Call - 시스템 콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 서비스 요청
> 2. **가치**: 하드웨어 추상화, 보호
> 3. **융합:** API, Kernel Mode, Trap

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

System Call은 사용자 모드에서 커널 서비스를 요청합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    System Call                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              시스템 콜 흐름                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  User Mode                          Kernel Mode       │   │ │
│   │   │  ┌─────────────────┐               ┌─────────────────┐│   │ │
│   │   │  │ Application     │               │ Kernel          ││   │ │
│   │   │  │ read(fd, buf, n)│               │                 ││   │ │
│   │   │  └────────┬────────┘               │                 ││   │ │
│   │   │           │ syscall                │                 ││   │ │
│   │   │           ↓                        │                 ││   │ │
│   │   │  ┌─────────────────┐               │ ┌─────────────┐ ││   │ │
│   │   │  │ Library (libc)  │   Trap        │ │ Syscall     │ ││   │ │
│   │   │  │                 │──────────────→│ │ Handler     │ ││   │ │
│   │   │  └─────────────────┘               │ └──────┬──────┘ ││   │ │
│   │   │                                    │        ↓        ││   │ │
│   │   │                                    │ ┌─────────────┐ ││   │ │
│   │   │                                    │ │ File System │ ││   │ │
│   │   │                                    │ │ Device      │ ││   │ │
│   │   │  ←────────────────────────────────────└─────────────┘ ││   │ │
│   │   │                                    └─────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 시스템 콜은 사용자-커널 모드 전환을 발생시킵니다.

**📢 섹션 요약 비유:** System Call = 민원 신청!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **User Mode** | 사용자 모드 |
| **Kernel Mode** | 커널 모드 |
| **Trap** | 모드 전환 |
| **Syscall Number** | 시스템 콜 번호 |
| **Parameters** | 인자 전달 |

**핵심 알고리즸:** 라이브러리 호출 → Trap → 커널 처리 → 복귀
