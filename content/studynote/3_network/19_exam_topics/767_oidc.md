---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

OpenID Connect는 OAuth 2.0에 인증을 추가합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OpenID Connect                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OIDC vs OAuth 2.0                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  OAuth 2.0:                                           │   │ │
│   │   │  - 목적: 권한 위임 (Authorization)                     │   │ │
│   │   │  - 토큰: Access Token                                │   │ │
│   │   │                                                      │   │ │
│   │   │  OpenID Connect:                                      │   │ │
│   │   │  - 목적: 인증 + 권한 위임 (Authentication + Auth)      │   │ │
│   │   │  - 토큰: Access Token + ID Token                     │   │ │
│   │   │                                                      │   │ │
│   │   │  ID Token (JWT):                                      │   │ │
│   │   │  {                                                   │   │ │
│   │   │    "iss": "https://auth.example.com",                │   │ │
│   │   │    "sub": "user123",                                 │   │ │
│   │   │    "aud": "client_id",                               │   │ │
│   │   │    "exp": 1234567890,                                │   │ │
│   │   │    "name": "Kim Cheol-su",                            │   │ │
│   │   │    "email": "kim@example.com"                         │   │ │
│   │   │  }                                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: OIDC는 사용자 신원을 확인합니다.

**📢 섹션 요약 비유:** OIDC = 여권!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **ID Token** | 신원 토큰 |
| **UserInfo** | 사용자 정보 |
| **Discovery** | 자동 설정 |
| **JWKS** | 키 세트 |
| **Scope** | openid, profile, email |

**핵심 알고리즸:** 인증 → ID Token → 검증 → 사용자 정보
