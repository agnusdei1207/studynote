# 비동기 전송 (시작/정지 비트)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시작 비트로 수신 타이밍 동기화
> 2. **가치**: 간단한 구현, 저비용, 문자 단위 독립성
> 3. **융합**: UART, RS-232, 모뎀과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

비동기 전송(Asynchronous Transmission)은 각 문자 앞뒤에 시작/정지 비트를 추가하여 독립적으로 전송하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    비동기 전송 (시작/정지 비트)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              프레임 구조                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Idle │ Start │ D0 │ D1 │ D2 │ D3 │ D4 │ D5 │ D6 │ D7 │ P │ Stop │   │ │
│   │   │  (1)  │  (0)  │ ←─── Data Bits (5-8) ───→ │(0/1)│(1) │   │ │
│   │   │                                                      │   │ │
│   │   │  시간 순서: ←─────────────────────────────────────────   │   │ │
│   │   │                                                      │   │ │
│   │   │  Start Bit: 0 (High → Low 천이)                      │   │ │
│   │   │  Data Bits: 5, 6, 7, 8 중 선택                       │   │ │
│   │   │  Parity: Even, Odd, None                             │   │ │
│   │   │  Stop Bit: 1, 1.5, 2 비트                            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              타이밍 다이어그램                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  신호:                                                 │   │ │
│   │   │  1 ─────┐                                       ┌─────│   │ │
│   │   │         │                                       │     │   │ │
│   │   │ 0       └───┬───┬───┬───┬───┬───┬───┬───┬───┬───┘     │   │ │
│   │   │             │ S │ D0│ D1│ D2│ D3│ D4│ D5│ D6│ D7│     │   │ │
│   │   │             │   │   │   │   │   │   │   │   │   │     │   │ │
│   │   │             └─────────────────────────────────┘ Stop   │   │ │
│   │   │                    ↑                                  ↑  │   │ │
│   │   │              샘플링 시작                          샘플링 끝 │   │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 시작 비트의 High→Low 천이로 수신측이 타이밍을 맞춥니다.

**💡 비유**: 비동기 전송은 문장마다 "시작"과 "끝"을 말하는 것과 같습니다.

**📢 섹션 요약 비유**: Start=0, Stop=1!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Start Bit** | 동기화 | 1→0 천이 감지 | 시작 신호 |
| **Data Bits** | 데이터 | 5-8 비트 | 본문 |
| **Parity** | 오류 검출 | 홀/짝 | 검사 |
| **Stop Bit** | 종료 | 1 유지 | 마침표 |

**오버헤드 계산**

| 설정 | 오버헤드 |
|:---|:---:|
| 8N1 (8 data, no parity, 1 stop) | 2/10 = 20% |
| 7E1 (7 data, even parity, 1 stop) | 3/10 = 30% |
| 8N2 (8 data, no parity, 2 stop) | 3/11 = 27% |

**핵심 알고리즘**: Idle 감지 → Start Edge → 중앙 샘플링 → Stop 확인

**📢 섹션 요약 비유**: 8N1이 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 패리티**

| 패리티 | 검출 | 정정 |
|:---|:---:|:---:|
| **None** | X | X |
| **Even** | 1비트 | X |
| **Odd** | 1비트 | X |

**📢 섹션 요약 비유**: 패리티는 1비트 오류만 검출!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: PC 시리얼** → 115200 8N1
**시나리오 2: 모뎀** → 7E1 (오류 검출)
**시나리오 3: 산업용** → 8E1 (신뢰성)

**도입 체크리스트**: 보레이트, 데이터 비트, 패리티

**📢 섹션 요약 비유**: 설정이 맞아야 통신!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 간단한 구현, 저비용

**미래 전망**: USB-C, 무선으로 대체

**참고 표준**: RS-232, UART

**📢 섹션 요약 비유**: 비동기는 간단하지만 느리다!
