# LLQ (Low Latency Queuing) - 저지연 큐잉

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Priority Queue + CBWFQ
> 2. **가치**: 실시간 트래픽 지연 최소화
> 3. **융합**: VoIP, 화상회의에 필수

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

LLQ(Low Latency Queuing)는 CBWFQ에 Priority Queue를 추가하여 실시간 트래픽의 지연을 최소화하는 스케줄링입니다.

> **해설**: LLQ는 PQ와 CBWFQ를 결합하여 실시간 트래픽 지연을 최소화합니다.

**💡 비유**: LLQ는 응급실 우선 진료입니다. 응급 환자가 항상 먼저 치료받습니다.

**📢 섹션 요약 비유:** LLQ = VoIP 필수!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Priority** | PQ 대역폭 |
| **Policing** | PQ 제한 |
| **CBWFQ** | 클래스별 대역폭 |

**비교표: PQ vs LLQ**

```
+------------+----------+----------+
| 항목       │ PQ       │ LLQ      │
+------------+----------+----------+
| 기아       │ O        │ X        │
| 제한       │ 없음     │ 폴리싱   │
| 다른 큐    │ 굶주림   │ 보장     │
+------------+----------+----------+
```

**핵심 알고리즸:** 엄격 우선, 폴리싱

**📢 섹션 요약 비유:** LLQ = PQ + Policing!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 지연 요구사항**

| 애플리케이션 | 지연 | 권장 큐 |
|:---|:---:|:---|
| **VoIP** | <150ms | LLQ |
| **화상회의** | <200ms | LLQ/CBWFQ |
| **스트리밍** | <5s | CBWFQ |
| **데이터** | 무제한 | CBWFQ |

**📢 섹션 요약 비유:** <150ms = LLQ!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: VoIP** → LLQ (10% 대역폭)
**시나리오 2: 화상회의** → LLQ 또는 CBWFQ
**시나리오 3: 데이터** → CBWFQ

**도입 체크리스트:** PQ 대역폭, Policing, 전체 대역폭

**📢 섹션 요약 비유:** PQ 대역폭 = 실제 필요량!
