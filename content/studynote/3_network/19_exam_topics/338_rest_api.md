# REST API - 웹 API 설계

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Representational State Transfer
> 2. **가치**: 표준화, 확장성, 캐싱
> 3. **융합**: HTTP, JSON, OAuth와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

REST API는 HTTP 프로토콜 기반의 웹 API 설계 아키텍처입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    REST API                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              REST API 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ HTTP Request                                   │   │ │
│   │   │     │ GET /users/123                                 │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           REST API Server                   │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • Resource (자원)                           │    │   │ │
│   │   │  │  • Verb (HTTP 메서드)                        │    │   │ │
│   │   │  │  • Representation (표현)                     │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ HTTP Response                                │   │ │
│   │   │       │ 200 OK + JSON                                │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HTTP 메서드                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 메서드     │ 작업     │ 멱등성   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ GET        │ 조회     │ O        │               │   │ │
│   │   │  │ POST       │ 생성     │ X        │               │   │ │
│   │   │  | PUT        │ 전체 수정│ O        │               │   │ │
│   │   │  │ PATCH      │ 부분 수정│ X        │               │   │ │
│   │   │  │ DELETE     │ 삭제     │ O        │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: REST API는 자원 중심의 HTTP 기반 API 설계입니다.

**💡 비유**: REST API는 웹의 언어입니다. HTTP로 대화합니다.

**📢 섹션 요약 비유:** REST = HTTP + 자원!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Resource** | 자원 (URI) |
| **Verb** | HTTP 메서드 |
| **Representation** | 표현 (JSON/XML) |
| **Stateless** | 무상태 |

**비교표: REST 제약 조건**

```
+------------+----------+----------+
| 제약       │ 설명     │ 가치     │
+------------+----------+----------+
| Stateless  │ 무상태   │ 확장성   │
| Cacheable  │ 캐시     │ 성능     │
| Uniform    │ 통일     │ 단순성   │
| Layered    │ 계층     │ 유연성   │
+------------+----------+----------+
```

**핵심 알고리즸:** URI 설계, 상태 코드

**📢 섹션 요약 비유:** Stateless = 확장성!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: API 스타일**

| 항목 | REST | GraphQL | gRPC |
|:---|:---|:---|:---|
| **표준** | HTTP | Schema | Protobuf |
| **캐싱** | 쉬움 | 복잡 | 없음 |
| **학습** | 낮음 | 중간 | 높음 |

**📢 섹션 요약 비유:** 웹 API = REST!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 공개 API** → REST
**시나리오 2: CRUD** → REST
**시나리오 3: 복잡한 쿼리** → GraphQL

**도입 체크리스트:** URI 설계, 버전관리, 인증

**📢 섹션 요약 비유:** 표준 = REST!
