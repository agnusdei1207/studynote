# SIP - 세션 초기화 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: VoIP 시그널링 프로토콜
> 2. **가치**: 통화 설정, 종료, 관리
> 3. **융합**: SDP, RTP, VoIP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SIP(Session Initiation Protocol)는 VoIP 통화 세션을 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SIP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SIP 통화 흐름                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Caller]                [Callee]                   │   │ │
│   │   │     │                        │                       │   │ │
│   │   │     │ INVITE                 │                       │   │ │
│   │   │     │────────────────────────▶│                       │   │ │
│   │   │     │                        │                       │   │ │
│   │   │     │ 100 Trying             │                       │   │ │
│   │   │     │◀────────────────────────│                       │   │ │
│   │   │     │                        │                       │   │ │
│   │   │     │ 180 Ringing            │                       │   │ │
│   │   │     │◀────────────────────────│                       │   │ │
│   │   │     │                        │                       │   │ │
│   │   │     │ 200 OK (SDP)           │                       │   │ │
│   │   │     │◀────────────────────────│                       │   │ │
│   │   │     │                        │                       │   │ │
│   │   │     │ ACK                    │                       │   │ │
│   │   │     │────────────────────────▶│                       │   │ │
│   │   │     │                        │                       │   │ │
│   │   │     │ RTP 미디어              │                       │   │ │
│   │   │     │◄───────────────────────▶│                       │   │ │
│   │   │     │                        │                       │   │ │
│   │   │     │ BYE                    │                       │   │ │
│   │   │     │────────────────────────▶│                       │   │ │
│   │   │     │                        │                       │   │ │
│   │   │     │ 200 OK                 │                       │   │ │
│   │   │     │◀────────────────────────│                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SIP 메서드                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 메서드     │ 기능     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ INVITE     │ 통화 시작│ 연결     │               │   │ │
│   │   │  │ ACK        │ 확인     │ 완료     │               │   │ │
│   │   │  | BYE        │ 종료     │ 끊기     │               │   │ │
│   │   │  │ CANCEL     │ 취소     │ 중단     │               │   │ │
│   │   │  │ REGISTER   │ 등록     │ 위치     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SIP는 INVITE/ACK/BYE로 통화 세션을 관리합니다.

**💡 비유**: SIP는 전화의 다이얼입니다. 번호를 누르고 연결합니다.

**📢 섹션 요약 비유:** SIP = VoIP 시그널링!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **User Agent** | 사용자 에이전트 |
| **Proxy** | 프록시 서버 |
| **Registrar** | 등록 서버 |
| **Redirect** | 리다이렉트 서버 |

**비교표: SIP vs WebRTC 시그널링**

```
+------------+----------+----------+
| 항목       │ SIP      │ WebRTC   │
+------------+----------+----------+
| 표준       │ IETF     │ W3C      │
| 프로토콜   │ UDP/TCP  │ WebSocket│
| 미디어     │ SDP      │ SDP      │
+------------+----------+----------+
```

**핵심 알고리즸:** INVITE/ACK/BYE, 트랜잭션

**📢 섹션 요약 비유:** INVITE = 통화 시작!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: VoIP 시그널링**

| 항목 | SIP | H.323 | WebRTC |
|:---|:---|:---|:---|
| **복잡도** | 중간 | 높음 | 낮음 |
| **확장성** | 높음 | 낮음 | 높음 |
| **브라우저** | 아니오 | 아니오 | 예 |

**📢 섹션 요약 비유:** VoIP = SIP!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 기업 전화** → SIP PBX
**시나리오 2: 통신사** → SIP 네트워크
**시나리오 3: 웹 통화** → WebRTC + SIP 게이트웨이

**도입 체크리스트:** SIP 서버, NAT, 보안

**📢 섹션 요약 비유:** 기업 VoIP = SIP!
