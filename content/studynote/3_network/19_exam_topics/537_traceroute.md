# traceroute - 트레이스라우트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 패킷 경로 추적
> 2. **가치**: 네트워크 문제 진단
> 3. **융합:** TTL, ICMP, RTT

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

traceroute는 패킷이 거치는 경로를 추적합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    traceroute                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              traceroute 동작                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Hop 1: TTL=1 ──→ Router1 (TTL=0, ICMP Time Exceed) │   │ │
│   │   │  Hop 2: TTL=2 ──→ Router2 (TTL=0, ICMP Time Exceed) │   │ │
│   │   │  Hop 3: TTL=3 ──→ Dest   (ICMP Echo Reply)          │   │ │
│   │   │                                                      │   │ │
│   │   │  $ traceroute 8.8.8.8                                │   │ │
│   │   │  1  192.168.1.1   1ms   1ms   1ms                    │   │ │
│   │   │  2  10.0.0.1      5ms   4ms   5ms                    │   │ │
│   │   │  3  8.8.8.8       10ms  9ms   10ms                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: traceroute는 TTL을 1씩 증가시키며 경로를 탐색합니다.

**📢 섹션 요약 비유:** traceroute = 네비게이션 경로!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **TTL=1** | 첫 번째 라우터 |
| **ICMP 11** | Time Exceeded |
| **UDP/TCP/ICMP** | 프로브 방식 |
| **RTT** | 각 홉 지연 |

**핵심 알고리즸:** TTL 증가 → 각 라우터 응답 수집
