# ETL (Extract-Transform-Load) - ETL

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 파이프라인
> 2. **가치**: 데이터 통합
> 3. **융합:** Data Warehouse, Integration, Batch

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ETL은 데이터를 추출, 변환, 적재합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ETL                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ETL 파이프라인                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Extract              Transform              Load   │   │ │
│   │   │  ┌─────────┐         ┌─────────┐         ┌─────────┐│   │ │
│   │   │  │ Source  │         │Cleanse  │         │  DW     ││   │ │
│   │   │  │ Systems │────────→│Format   │────────→│  Target ││   │ │
│   │   │  │         │         │Validate │         │         ││   │ │
│   │   │  └─────────┘         └─────────┘         └─────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  Extract:                                            │   │ │
│   │   │  - 원본 시스템에서 데이터 읽기                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Transform:                                          │   │ │
│   │   │  - 정제, 변환, 매핑, 집계                             │   │ │
│   │   │                                                      │   │ │
│   │   │  Load:                                               │   │ │
│   │   │  - 대상 시스템에 데이터 적재                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ETL은 데이터 웨어하우스 구축의 핵심 과정입니다.

**📢 섹션 요약 비유:** ETL = 데이터 이사!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Extract** | 추출 |
| **Transform** | 변환 |
| **Load** | 적재 |
| **Staging** | 스테이징 |
| **Scheduling** | 스케줄링 |

**핵심 알고리즸:** 추출 → 변환 → 적재

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ETL vs ELT**

| 항목 | ETL | ELT |
|:---|:---|:---|
| **변환 위치** | ETL 서버 | DB |
| **확장성** | 제한 | 높음 |
| **복잡도** | 높음 | 낮음 |
