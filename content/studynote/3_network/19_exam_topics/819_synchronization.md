# Synchronization - 동기화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 타이밍 일치
> 2. **가치**: 정확한 데이터 수신
> 3. **융합:** Clock Recovery, PTP, NTP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Synchronization은 송수신 타이밍을 일치시킵니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Synchronization                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              동기화 유형                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  비트 동기 (Bit Sync):                                │   │ │
│   │   │  클럭 복구, 샘플링 타이밍                               │   │ │
│   │   │                                                      │   │ │
│   │   │  프레임 동기 (Frame Sync):                            │   │ │
│   │   │  ┌──────┬─────────────────┬──────┐                  │   │ │
│   │   │  │Flag │      Data       │Flag  │                  │   │ │
│   │   │  └──────┴─────────────────┴──────┘                  │   │ │
│   │   │  HDLC, PPP                                            │   │ │
│   │   │                                                      │   │ │
│   │   │  네트워크 동기:                                        │   │ │
│   │   │  - NTP: 네트워크 시간 프로토콜 (ms)                     │   │ │
│   │   │  - PTP: 정밀 시간 프로토콜 (μs)                         │   │ │
│   │   │  - GPS: 위성 시간 (ns)                                │   │ │
│   │   │                                                      │   │ │
│   │   │  동기/비동기 전송:                                     │   │ │
│   │   │  - 동기: 클럭 공유                                     │   │ │
│   │   │  - 비동기: Start/Stop Bit                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 동기화는 디지털 통신의 필수 요소입니다.

**📢 섹션 요약 비유:** Synchronization = 오케스트라 지휘!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Clock** | 클럭 |
| **PLL** | 위상 고정 루프 |
| **Preamble** | 프리앰블 |
| **Sync Word** | 동기 워드 |
| **Timestamp** | 타임스탬프 |

**핵심 알고리즸:** 신호 수신 → 타이밍 추출 → 동기 달성

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**시간 동기화 프로토콜**

| 프로토콜 | 정밀도 | 용도 |
|:---|:---|:---|
| **NTP** | ~ms | 인터넷 |
| **PTP** | ~μs | 산업 |
| **GPS** | ~ns | 위치/시간 |
| **Atomic** | ~ns | 과학 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 인터넷 → NTP
**시나리오 2:** 금융 → PTP
**시나리오 3:** 통신망 → GPS
