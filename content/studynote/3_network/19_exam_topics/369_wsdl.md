# WSDL - 웹 서비스 기술 언어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SOAP 웹 서비스 인터페이스 정의
> 2. **가치**: 자동 코드 생성, 계약
> 3. **융합**: SOAP, XML, UDDI

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

WSDL(Web Services Description Language)은 SOAP 서비스 인터페이스를 정의합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WSDL                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              WSDL 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  <definitions>                                       │   │ │
│   │   │    <types>          <!-- 데이터 타입 -->              │   │ │
│   │   │    <message>        <!-- 메시지 -->                   │   │ │
│   │   │    <portType>       <!-- 인터페이스 -->               │   │ │
│   │   │    <binding>        <!-- 바인딩 -->                   │   │ │
│   │   │    <service>        <!-- 서비스 -->                   │   │ │
│   │   │  </definitions>                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              WSDL 구성 요소                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 요소       │ 설명     │ 역할     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ types      │ 타입     │ 데이터   │               │   │ │
│   │   │  │ message    │ 메시지   │ 입출력   │               │   │ │
│   │   │  | portType   │ 포트     │ 작업     │               │   │ │
│   │   │  │ binding    │ 바인딩   │ 프로토콜 │               │   │ │
│   │   │  │ service    │ 서비스   │ 엔드포인트│               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: WSDL은 SOAP 서비스의 인터페이스, 바인딩, 엔드포인트를 정의합니다.

**💡 비유**: WSDL은 계약서입니다. 무엇을 어떻게 제공하는지 명시합니다.

**📢 섹션 요약 비유:** WSDL = 서비스 계약!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Types** | 데이터 타입 (XSD) |
| **Message** | 요청/응답 메시지 |
| **PortType** | 인터페이스 |
| **Binding** | 프로토콜 바인딩 |

**비교표: WSDL vs OpenAPI**

```
+------------+----------+----------+
| 항목       │ WSDL     │ OpenAPI  │
+------------+----------+----------+
| 포맷       │ XML      │ YAML/JSON│
| 프로토콜   │ SOAP     │ REST     │
| 생성       │ 가능     │ 가능     │
+------------+----------+----------+
```

**핵심 알고리즸:** XSD, 바인딩

**📢 섹션 요약 비유:** portType = 인터페이스!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: API 명세**

| 항목 | WSDL | OpenAPI | GraphQL Schema |
|:---|:---|:---|:---|
| **스타일** │ SOAP │ REST │ GraphQL |
| **형식** │ XML │ YAML │ SDL |
| **복잡도** │ 높음 │ 중간 │ 중간 |

**📢 섹션 요약 비유:** SOAP = WSDL!
