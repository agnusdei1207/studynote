# Eventual Consistency - 결과적 일관성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시간 경과 후 일관성
> 2. **가치**: 높은 가용성
> 3. **융합:** BASE, NoSQL, Replication

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Eventual Consistency는 시간이 지나면 모든 노드가 일치합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Eventual Consistency                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              결과적 일관성 흐름                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  T0: Write X=1 to Node A                            │   │ │
│   │   │  ┌───────────────────────────────────────────────┐  │   │ │
│   │   │  │ Node A: X=1    Node B: X=0    Node C: X=0    │  │   │ │
│   │   │  └───────────────────────────────────────────────┘  │   │ │
│   │   │                                                      │   │ │
│   │   │  T1: Replication to Node B                          │   │ │
│   │   │  ┌───────────────────────────────────────────────┐  │   │ │
│   │   │  │ Node A: X=1    Node B: X=1    Node C: X=0    │  │   │ │
│   │   │  └───────────────────────────────────────────────┘  │   │ │
│   │   │                                                      │   │ │
│   │   │  T2: Replication to Node C                          │   │ │
│   │   │  ┌───────────────────────────────────────────────┐  │   │ │
│   │   │  │ Node A: X=1    Node B: X=1    Node C: X=1    │  │   │ │
│   │   │  └───────────────────────────────────────────────┘  │   │ │
│   │   │  → 모든 노드 일치 (Eventually Consistent)           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 쓰기가 전파되기까지 시간이 걸리지만 결국 일치합니다.

**📢 섹션 요약 비유:** Eventual Consistency = 소문!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Replication Lag** | 복제 지연 |
| **Quorum** | 정족수 |
| **Vector Clock** | 벡터 시계 |
| **Conflict Resolution** | 충돌 해결 |

**핵심 알고리즸:** 쓰기 → 비동기 복제 → 결과적 일치
