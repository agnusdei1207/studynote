# Snapshot - 스냅샷

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 특정 시점 상태 저장
> 2. **가치**: 빠른 복구, 백업 최적화
> 3. **융합:** VM, Storage, Database

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Snapshot은 특정 시점의 상태를 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Snapshot                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스냅샷 방식                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Copy-on-Write (COW):                                 │   │ │
│   │   │  원본 블록 변경 시 스냅샷에 복사                         │   │ │
│   │   │                                                      │   │ │
│   │   │  Redirect-on-Write (ROW):                             │   │ │
│   │   │  변경을 새 위치에 쓰고 포인터 변경                       │   │ │
│   │   │                                                      │   │ │
│   │   │  Timeline:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ T0: Snapshot 1                                   ││   │ │
│   │   │  │     ↓                                            ││   │ │
│   │   │  │ T1: Snapshot 2                                   ││   │ │
│   │   │  │     ↓                                            ││   │ │
│   │   │  │ T2: Snapshot 3                                   ││   │ │
│   │   │  │     ↓                                            ││   │ │
│   │   │  │ T3: Current                                      ││   │ │
│   │   │  │     ↓ 복원                                        ││   │ │
│   │   │  │ T1: Restored from Snapshot 2                     ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스냅샷은 순간 복구를 가능하게 합니다.

**📢 섹션 요약 비유:** Snapshot = 게임 저장!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **COW** | Copy-on-Write |
| **ROW** | Redirect-on-Write |
| **Metadata** | 메타데이터 |
| **Retention** | 보존 기간 |
| **Consistency** | 일관성 |

**핵심 알고리즸:** 스냅샷 생성 → 변경 추적 → 복원 시 포인터 변경
