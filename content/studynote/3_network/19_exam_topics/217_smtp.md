# SMTP (Simple Mail Transfer Protocol) - 메일 전송

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이메일 전송 프로토콜
> 2. **가치**: 글로벌 메일 시스템
> 3. **융합**: POP3/IMAP, SPF/DKIM/DMARC와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SMTP(Simple Mail Transfer Protocol)는 이메일을 전송하는 표준 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SMTP (Simple Mail Transfer Protocol)              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              이메일 시스템 구조                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [송신자]         [SMTP]          [수신자]           │   │ │
│   │   │                                                      │   │ │
│   │   │   MUA → MTA → MTA → ... → MTA → MDA → MUA           │   │ │
│   │   │                                                      │   │ │
│   │   │  용어:                                                │   │ │
│   │   │  • MUA: Mail User Agent (클라이언트)                 │   │ │
│   │   │  • MTA: Mail Transfer Agent (서버)                   │   │ │
│   │   │  • MDA: Mail Delivery Agent (배달)                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SMTP 명령어                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  명령어    │ 설명                                   │   │ │
│   │   │  ─────────┼────────────────────────────────────│   │ │
│   │   │  HELO     │ 연결 시작                              │   │ │
│   │   │  EHLO     │ 확장 연결                              │   │ │
│   │   │  MAIL FROM│ 발신자                                 │   │ │
│   │   │  RCPT TO  │ 수신자                                 │   │ │
│   │   │  DATA     │ 본문 시작                              │   │ │
│   │   │  QUIT     │ 연결 종료                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SMTP는 MTA 간 메일 전송을 담당합니다.

**💡 비유**: SMTP는 우체국입니다. 편지를 배달합니다.

**📢 섹션 요약 비유:** SMTP = 포트 25!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 포트 | 용도 | 보안 |
|:---:|:---|:---|
| **25** | MTA 간 | STARTTLS |
| **587** | 클라이언트 | STARTTLS |
| **465** | SMTPS | SSL/TLS |

**비교표: 메일 프로토콜**

```
+------------+----------+----------+
| 프로토콜   │ 용도     │ 방향     │
+------------+----------+----------+
| SMTP       │ 전송     │ 송신     │
| POP3       │ 수신     │ 다운로드 │
| IMAP       │ 수신     │ 동기화   │
+------------+----------+----------+
```

**핵심 알고리즸:** MX 레코드 조회, 릴레이

**📢 섹션 요약 비유:** 587 = 클라이언트 전용!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 스팸 방지**

| 기술 | 목적 |
|:---|:---|
| **SPF** | 발신자 검증 |
| **DKIM** | 서명 검증 |
| **DMARC** | 정책 통합 |
| **RBL** | 블랙리스트 |

**📢 섹션 요약 비유:** SPF + DKIM + DMARC = 스팸 방지!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 기업 메일** → SMTP + SPF + DKIM
**시나리오 2: 대량 메일** → 릴레이 서버
**시나리오 3: 보안** → TLS 필수

**도입 체크리스트:** MX 레코드, SPF/DKIM, 릴레이 제한

**📢 섹션 요약 비유:** 릴레이 제한 = 스팸 방지!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 글로벌 메일 전송

**미래 전망:** DMARC 강제화

**참고 표준:** RFC 5321, RFC 7208 (SPF), RFC 6376 (DKIM)

**📢 섹션 요약 비유:** SMTP는 이메일의 핵심!
