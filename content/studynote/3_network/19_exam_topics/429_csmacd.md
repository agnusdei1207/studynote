# CSMA/CD - Carrier Sense Multiple Access with Collision Detection

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이더넷 매체 접근 제어
> 2. **가치**: 충돌 감지 및 재전송
> 3. **융합**: Half-duplex, 허브, 1G 이하

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CSMA/CD는 이더넷에서 매체 접근을 제어하고 충돌을 감지합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CSMA/CD                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CSMA/CD 동작                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Carrier Sense (매체 감지)                        │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 사용 중이면 대기                               │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  2. Multiple Access (다중 접근)                      │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 유휴 시 전송 시작                               │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  3. Collision Detection (충돌 감지)                  │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 충돌 시 Jam 신호 전송                           │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  4. Backoff (대기 후 재전송)                         │   │ │
│   │   │     │                                                │   │ │
│   │   │     └──▶ Binary Exponential Backoff                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Backoff 알고리즘                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  충돌 횟수: 1 → 대기: 0~1 슬롯                       │   │ │
│   │   │  충돌 횟수: 2 → 대기: 0~3 슬롯                       │   │ │
│   │   │  충돌 횟수: 3 → 대기: 0~7 슬롯                       │   │ │
│   │   │  ...                                                   │   │ │
│   │   │  충돌 횟수: 10 → 대기: 0~1023 슬롯                    │   │ │
│   │   │  16회 충돌 시 폐기                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: CSMA/CD는 Half-duplex 환경에서만 필요합니다.

**💡 비유**: CSMA/CD는 누가 먼저 말하나 경쟁하는 회의입니다.

**📢 섹션 요약 비유:** CSMA/CD = 충돌 관리!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Carrier Sense** | 매체 사용 감지 |
| **Jam Signal** | 충돌 알림 신호 |
| **Slot Time** | 슬롯 시간 (512 비트) |
| **Backoff** | 대기 시간 계산 |

**비교표: 이더넷 슬롯 타임**

```
+------------+----------+----------+
| 속도       │ 슬롯타임 │ 최소프레임│
+------------+----------+----------+
| 10Mbps     │ 51.2μs   │ 64바이트 │
| 100Mbps    │ 5.12μs   │ 64바이트 │
| 1000Mbps   │ 5.12μs   │ 512바이트│
+------------+----------+----------+
```

**핵심 알고리즸:** 지수 백오프, 충돌 감지

**📢 섹션 요약 비유:** Full-duplex = CSMA/CD 불필요!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 매체 접근 방식**

| 항목 | CSMA/CD | CSMA/CA | TDMA |
|:---|:---|:---|:---|
| **용도** │ 유선 │ 무선 | 셀룰러 |
| **충돌** │ 감지 | 회피 | 예방 |

**📢 섹션 요약 비유:** 무선 = CSMA/CA!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 허브 네트워크 → CSMA/CD
**시나리오 2:** 스위치 Full-duplex → CSMA/CD 불필요
**시나리오 3:** 10G 이상 → CSMA/CD 미지원

**도입 체크리스트:** 듀플렉스 모드, 장비 선택

**📢 섹션 요약 비유:** 스위치 = CSMA/CD 제거!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 공정한 매체 접근

**미래 전망:** Full-duplex로 사실상 사용 안 함

**참고 표준:** IEEE 802.3

**📢 섹션 요약 비유:** CSMA/CD는 레거시!
