# IntServ (Integrated Services) - 통합 서비스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자원 예약 기반 QoS
> 2. **가치**: 하드 보장
> 3. **융합**: RSVP와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IntServ(Integrated Services)는 RSVP를 사용하여 네트워크 자원을 미리 예약하는 QoS 모델입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IntServ (Integrated Services)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IntServ 구조                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [송신자]                                        │   │ │
│   │   │     │                                             │   │ │
│   │   │     │ RSVP PATH                                   │   │ │
│   │   │     ↓                                             │   │ │
│   │   │  [라우터 1] ───> [라우터 2] ───> [라우터 3]        │   │ │
│   │   │     │               │               │              │   │ │
│   │   │     │ 자원 예약     │ 자원 예약     │              │   │ │
│   │   │     │               │               │              │   │ │
│   │   │     │ <─────────────────────────── │              │   │ │
│   │   │     │           RSVP RESV          │              │   │ │
│   │   │     │                                             │   │ │
│   │   │  [수신자]                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              서비스 클래스                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Guaranteed Service:                              │   │ │
│   │   │     • 대역폭, 지연 보장                              │   │ │
│   │   │     • 실시간 애플리케이션                            │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Controlled Load Service:                         │   │ │
│   │   │     • 혼잡 없는 네트워크 수준                        │   │ │
│   │   │     • Best Effort 개선                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IntServ는 RSVP로 각 흐름마다 자원을 예약합니다.

**💡 비유**: IntServ는 식당 예약입니다. 자리를 미리 확보합니다.

**📢 섹션 요약 비유:** IntServ = 자원 예약!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RSVP** | 자원 예약 프로토콜 |
| **PATH** | 송신→수신 경로 |
| **RESV** | 수신→송신 예약 |
| **FlowSpec** | 트래픽 특성 |

**비교표: IntServ 한계**

```
+------------+----------+----------+
| 문제       │ 설명     │ 결과     │
+------------+----------+----------+
| 확장성     │ 상태유지 │ 코어부하 │
| 복잡도     │ 높음     │ 관리어려움│
| 호환성     │ 제한     │ 제한적   │
+------------+----------+----------+
```

**핵심 알고리즸:** RSVP PATH/RESV, 상태 유지

**📢 섹션 요약 비유:** IntServ = 소규모만!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: IntServ vs DiffServ**

| 항목 | IntServ | DiffServ |
|:---|:---|:---|
| **보장** | Hard | Soft |
| **스케일** | 낮음 | 높음 |
| **상태** | 유지 | 무상태 |
| **용도** | 소규모 | 대규모 |

**📢 섹션 요약 비유:** 현대 = DiffServ!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 소규모 실시간** → IntServ
**시나리오 2: 대규모** → DiffServ
**시나리오 3: 혼합** → IntServ Edge + DiffServ Core

**도입 체크리스트:** 규모, 복잡도, 요구사항

**📢 섹션 요약 비유:** 대부분 = DiffServ!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)
