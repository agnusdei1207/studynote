# 네트워크 문제 해결 - Troubleshooting

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 체계적 문제 해결
> 2. **가치**: MTTR 감소
> 3. **융합**: 모니터링, 분석과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

네트워크 트러블슈팅은 체계적인 방법으로 네트워크 문제를 해결합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    네트워크 문제 해결                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              트러블슈팅 프로세스                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 문제 정의 (Define)                                │   │ │
│   │   │     • 증상 파악                                       │   │ │
│   │   │     • 영향 범위                                       │   │ │
│   │   │     │                                              │   │ │
│   │   │     ▼                                              │   │ │
│   │   │  2. 정보 수집 (Gather)                               │   │ │
│   │   │     • 로그, 메트릭                                   │   │ │
│   │   │     • 사용자 인터뷰                                  │   │ │
│   │   │     │                                              │   │ │
│   │   │     ▼                                              │   │ │
│   │   │  3. 원인 분석 (Analyze)                              │   │ │
│   │   │     • 가능한 원인 나열                               │   │ │
│   │   │     • 우선순위                                       │   │ │
│   │   │     │                                              │   │ │
│   │   │     ▼                                              │   │ │
│   │   │  4. 해결 (Resolve)                                   │   │ │
│   │   │     • 조치 적용                                      │   │ │
│   │   │     • 검증                                          │   │ │
│   │   │     │                                              │   │ │
│   │   │     ▼                                              │   │ │
│   │   │  5. 문서화 (Document)                                │   │ │
│   │   │     • 원인, 해결 방법                                │   │ │
│   │   │     • 재발 방지                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              계층별 접근 (OSI)                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  L1 Physical: 케이블, 포트, 전원                     │   │ │
│   │   │  L2 Data Link: MAC, VLAN, STP                       │   │ │
│   │   │  L3 Network: IP, 라우팅, ARP                         │   │ │
│   │   │  L4 Transport: TCP/UDP, 포트                        │   │ │
│   │   │  L7 Application: DNS, HTTP, 서비스                   │   │ │
│   │   │                                                      │   │ │
│   │   │  바텀업: L1 → L7                                     │   │ │
│   │   │  탑다운: L7 → L1                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 체계적인 접근으로 문제를 빠르게 해결합니다.

**💡 비유**: 트러블슈팅은 의사의 진단 과정입니다. 증상 → 원인 → 치료.

**📢 섹션 요약 비유:** 트러블슈팅 = 체계적!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **ping** | 연결성 |
| **traceroute** | 경로 |
| **nslookup** | DNS |
| **netstat** | 연결 |

**비교표: 진단 도구**

```
+------------+----------+----------+
| 도구       │ 계층     │ 용도     │
+------------+----------+----------+
| ping       │ L3       │ 연결성   │
| traceroute │ L3       │ 경로     │
| nslookup   │ L7       │ DNS      │
| telnet     │ L4-L7    │ 포트     │
| tcpdump    │ L2-L7    │ 패킷     │
+------------+----------+----------+
```

**핵심 알고리즸:** 바텀업, 탑다운

**📢 섹션 요약 비유:** ping 8.8.8.8 = 기본!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 일반 문제**

| 문제 | 원인 | 해결 |
|:---|:---|:---|
| **IP 충돌** | 중복 IP | DHCP |
| **DNS** | 캐시, 서버 | flush |
| **VLAN** | 태그 불일치 | 설정 |
| **라우팅** | 경로 없음 | 라우팅 테이블 |

**📢 섹션 요약 비유:** L1 = 케이블부터!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 연결 안 됨** → L1 → L2 → L3
**시나리오 2: 느림** → 대역폭, 지연
**시나리오 3: 간헐적** → 로그, 패턴

**도입 체크리스트:** 도구, 로그, 문서

**📢 섹션 요약 비유:** 문서화 = 재발 방지!
