# 지연 시간 vs 처리량 - Latency vs Throughput

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 반응 속도 vs 전송량
> 2. **가치**: 용도별 최적화
> 3. **융합**: 대역폭-지연 곱, 윈도우

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Latency는 반응 속도, Throughput은 단위 시간 전송량입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Latency vs Throughput                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              개념 비교                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Latency (지연 시간):                                │   │ │
│   │   │  - A에서 B까지 패킷 이동 시간                         │   │ │
│   │   │  - 단위: ms (밀리초)                                  │   │ │
│   │   │  - 낮을수록 좋음                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  Throughput (처리량):                                │   │ │
│   │   │  - 단위 시간당 전송 데이터 양                         │   │ │
│   │   │  - 단위: Mbps/Gbps                                    │   │ │
│   │   │  - 높을수록 좋음                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              관계                                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 상황       │ Latency  │ Throughput│              │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 게임       │ 중요     │ 덜 중요  │               │   │ │
│   │   │  │ 백업       │ 덜 중요  │ 중요     │               │   │ │
│   │   │  | VoIP      │ 중요     │ 중간     │               │   │ │
│   │   │  │ 스트리밍   │ 중간     │ 중요     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Latency와 Throughput은 독립적이지만 관련이 있습니다.

**💡 비유**: Latency는 반응 속도, Throughput은 파이프 굵기입니다.

**📢 섹션 요약 비유:** Latency = 반응, Throughput = 양!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RTT** | Round Trip Time |
| **One-way Delay** | 단방향 지연 |
| **Goodput** | 유용한 처리량 |
| **BDP** | Bandwidth-Delay Product |

**비교표: 지연 원인**

```
+------------+----------+----------+
| 원인       │ 영향     │ 해결     │
+------------+----------+----------+
| 거리       │ 전파지연 │ CDN      │
| 큐잉       │ 대기     │ QoS      │
| 처리       │ 라우팅   │ 가속화   │
+------------+----------+----------+
```

**핵심 알고리즸:** 윈도우 크기 = BDP

**📢 섹션 요약 비유:** BDP = 윈도우 계산!
