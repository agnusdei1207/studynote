# GLBP - 게이트웨이 로드 밸런싱 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Cisco 게이트웨이 로드밸런싱
> 2. **가치**: 대역폭 효율성
> 3. **융합:** AVG, AVF, Weighted Load Balancing

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

GLBP (Gateway Load Balancing Protocol)는 게이트웨이 로드밸런싱을 제공합니다.

```

│                    GLBP                                              │
├─────────────────────────────────────────────────────────────────────────┤

│   │   │                                                      │   │ │
│   │   │  GLBP 구조:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  AVG (Active Virtual Gateway)                   ││   │ │
│   │   │  │       │                                         ││   │ │
```

> **해설**: GLBP는 여러 게이트웨이의 대역폭을 모두 활용합니다.

**📢 섹션 요약 비유:** GLBP = 택배 분배 센터!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **AVG** | Active Virtual Gateway |
| **AVF** | Active Virtual Forwarder |
| **Weight** | 가중치 |
| **Redirect** | 리다이렉트 타이머 |

**핵심 알고리듬:** AVG 선출 → AVF 할당 → MAC 분배 → 로드밸런싱

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**GLBP vs HSRP vs VRRP**

| 항목 | GLBP | HSRP | VRRP |
|:---|:---|:---|:---|
| **로드밸런싱** | 지원 | 미지원 | 미지원 |
| **활성 라우터** | 다중 | 1개 | 1개 |
| **표준** | Cisco | Cisco | RFC |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 대역폭 활용 → GLBP

**시나리오 3:** 멀티벤더 → VRRP
