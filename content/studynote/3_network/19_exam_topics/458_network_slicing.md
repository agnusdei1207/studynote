# 네트워크 슬라이싱 - Network Slicing

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 네트워크를 논리적 슬라이스로 분할
> 2. **가치**: 격리, QoS 보장, 자원 효율
> 3. **융합**: 5G, SDN, NFV

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

네트워크 슬라이싱은 네트워크를 논리적 슬라이스로 나눕니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    네트워크 슬라이싱                                 │
```

> **해설**: 각 슬라이스는 독립적인 QoS를 보장받습니다.

**💡 비유**: 네트워크 슬라이싱은 도로의 전용차선입니다.

**📢 섹션 요약 비유:** Slicing = 논리적 분할!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **NSSMF** | 슬라이스 관리 |
| **RAN Slice** | 무선 슬라이스 |
| **Core Slice** | 코어 슬라이스 |
| **Transport Slice** | 전송 슬라이스 |

**비교표: 5G 슬라이스 유형**

```
+------------+----------+----------+----------+
| 유형       │ 특징     │ 예시     │ 요구사항 │
+------------+----------+----------+----------+
| eMBB       │ 고속     │ 스트리밍 │ 대역폭   │
| URLLC      │ 저지연   │ 원격수술 │ 지연     │
| mMTC       │ 대규모   │ 스마트시티│ 연결    │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 자원 할당, 격리

**📢 섹션 요약 비유:** URLLC = 1ms 이하!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 구현 기술**

| 기술 | 역할 | 설명 |
|:---|:---|:---|
| **SDN** | 제어 | 중앙 관리 |
| **NFV** | 기능 | 가상화 |
| **VLAN/VXLAN** | 격리 | L2 |

**📢 섹션 요약 비유:** e2e 슬라이스!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 통신사 5G → eMBB + URLLC
**시나리오 2:** 기업 VPN → 전용 슬라이스
**시나리오 3:** IoT → mMTC 슬라이스

**도입 체크리스트:** 요구사항, 격리, 관리

**📢 섹션 요약 비유:** SLA = 슬라이스 보장!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 자원 효율 50% 향상
