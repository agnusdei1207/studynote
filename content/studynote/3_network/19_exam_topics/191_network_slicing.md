# 네트워크 슬라이싱 (Network Slicing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하나의 물리 네트워크를 논리적 분할
> 2. **가치**: 서비스별 맞춤형 QoS
> 3. **융합**: 5G, 클라우드와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

네트워크 슬라이싱(Network Slicing)은 하나의 물리 네트워크 인프라를 여러 개의 논리적 가상 네트워크로 분할하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    네트워크 슬라이싱                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              슬라이스 종류                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. eMBB 슬라이스:                                    │   │ │
│   │   │     • 고속 데이터                                     │   │ │
│   │   │     • 대역폭: 100Mbps+                                │   │ │
│   │   │     • 용도: 스트리밍, 다운로드                         │   │ │
│   │   │                                                      │   │ │
│   │   │  2. uRLLC 슬라이스:                                   │   │ │
│   │   │     • 저지연, 고신뢰                                  │   │ │
│   │   │     • 지연: <1ms                                     │   │ │
│   │   │     • 용도: 자율주행, 원격 수술                        │   │ │
│   │   │                                                      │   │ │
│   │   │  3. mMTC 슬라이스:                                    │   │ │
│   │   │     • 대규모 IoT                                      │   │ │
│   │   │     • 연결: 100만/km²                                │   │ │
│   │   │     • 용도: 스마트시티, 스마트팜                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              슬라이싱 아키텍처                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌────────────────────────────────────────┐         │   │ │
│   │   │  │        서비스 슬라이스 (Service Slice)   │         │   │ │
│   │   │  │  eMBB  │ uRLLC │ mMTC  │ 기업 │         │   │ │
│   │   │  └────────┴───────┴──────┴─────┘         │   │ │
│   │   │  ┌────────────────────────────────────────┐         │   │ │
│   │   │  │        네트워크 슬라이스 (Network Slice)  │         │   │ │
│   │   │  │  RAN   │  전송망  │  코어망  │             │   │ │
│   │   │  └────────┴─────────┴────────┘             │   │ │
│   │   │  ┌────────────────────────────────────────┐         │   │ │
│   │   │  │        인프라 (Infrastructure)            │         │   │ │
│   │   │  │       물리 네트워크 자원                   │         │   │ │
│   │   │  └────────────────────────────────────────┘         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 슬라이싱은 5G의 핵심 기능으로 서비스별 맞춤형 네트워크를 제공합니다.

**💡 비유**: 슬라이싱은 하나의 도로를 버스전용차선, 자율주행차선으로 나누는 것입니다.

**📢 섹션 요약 비유:** 슬라이싱 = 서비스 맞춤형!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 계층 | 기술 | 역할 |
|:---|:---|:---|
| **서비스** | eMBB/uRLLC/mMTC | QoS 요구 |
| **네트워크** | RAN/전송/코어 | 논리 분할 |
| **인프라** | 가상화/NFV | 자원 풀 |

**비교표**

```
+------------+----------+----------+
| 슬라이스   │ 대역폭   │ 지연     │
+------------+----------+----------+
| eMBB       │ 높음     │ 보통     │
│ uRLLC      │ 보통     │ 낮음     │
│ mMTC       │ 낮음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** 자원 예약, 격리, 오케스트레이션

**📢 섹션 요약 비유:** E2E 슬라이싱 = 서비스 품질 보장!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 슬라이싱 vs VPN**

| 항목 | 슬라이싱 | VPN |
|:---|:---|:---|
| **격리** | 강함 | 약함 |
| **QoS** | 보장 | 미보장 |
| **자원** | 전용 | 공유 |

**📢 섹션 요약 비유:** 슬라이싱은 QoS 보장!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 자율주행** → uRLLC 슬라이스, 전용 자원
**시나리오 2: IoT 도시** → mMTC 슬라이스
**시나리오 3: 일반 인터넷** → eMBB 슬라이스

**도입 체크리스트:** 서비스 요구사항, 자원 가용성, 격리 수준

**📢 섹션 요약 비유:** 서비스에 맞는 슬라이스!
