# CRC (순환 중복 검사)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다항식 연산으로 오류 검출 코드 생성
> 2. **가치**: 높은 검출 능력, 버스트 오류 검출
> 3. **융합**: Ethernet, HDLC, ZIP과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CRC(Cyclic Redundancy Check)는 다항식 나눗셈으로 FCS(Frame Check Sequence)를 생성하여 오류를 검출하는 방법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CRC (순환 중복 검사)                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CRC 계산 원리                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 데이터를 다항식으로 표현                          │   │ │
│   │   │     M(x) = 데이터 비트                                │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 생성 다항식 G(x) 선택                             │   │ │
│   │   │     CRC-8, CRC-16, CRC-32 등                         │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 나눗셈 연산 (XOR)                                  │   │ │
│   │   │     M(x) × x^n ÷ G(x) → 나머지 R(x)                  │   │ │
│   │   │                                                      │   │ │
│   │   │  4. FCS = R(x)                                       │   │ │
│   │   │     데이터 + FCS 전송                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  5. 수신측 검사                                        │   │ │
│   │   │     (데이터 + FCS) ÷ G(x) → 0이면 정상               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 생성 다항식                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  CRC-8:    x^8 + x^2 + x + 1                         │   │ │
│   │   │  CRC-16:   x^16 + x^15 + x^2 + 1 (IBM)               │   │ │
│   │   │  CRC-CCITT: x^16 + x^12 + x^5 + 1                    │   │ │
│   │   │  CRC-32:   x^32 + x^26 + x^23 + ... + 1 (Ethernet)   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: CRC는 버스트 오류를 높은 확률로 검출합니다.

**💡 비유**: CRC는 데이터에 대한 디지털 지문입니다.

**📢 섹션 요약 비유**: CRC-32가 Ethernet!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **G(x)** | 생성 다항식 | 기준 | 공식 |
| **M(x)** | 메시지 | 데이터 | 입력 |
| **R(x)** | 나머지 | FCS | 결과 |
| **XOR** | 연산 | 모듈로 2 | 계산 |

**하드웨어 구현**

```
CRC-16 시프트 레지스터:
┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐
│D15│D14│D13│D12│D11│D10│D9 │D8 │D7 │D6 │D5 │D4 │D3 │D2 │D1 │D0 │
└─┬─┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴─┬─┴───┴─┬─┴─┬─┘
  │                                               │       │   │
  └──────────────────XOR──────────────────────────┘       │   │
                          ↑                                │   │
                      입력 + D15                           │   │
                                      └──────XOR───────────┘   │
```

**핵심 알고리즘**: 시프트 → XOR → 반복 → FCS 출력

**📢 섹션 요약 비유**: 하드웨어로 고속 구현!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: CRC 종류**

| CRC | 비트수 | 검출 능력 |
|:---|:---:|:---|
| **CRC-8** | 8 | 버스트 8 |
| **CRC-16** | 16 | 버스트 16 |
| **CRC-32** | 32 | 버스트 32 |

**📢 섹션 요약 비유**: CRC-32가 99.99% 검출!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: Ethernet** → CRC-32
**시나리오 2: HDLC** → CRC-CCITT
**시나리오 3: ZIP** → CRC-32

**도입 체크리스트**: 데이터 길이, 오류 패턴

**📢 섹션 요약 비유**: 통신망은 CRC-32!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 99.998% 오류 검출

**미래 전망**: 더 긴 CRC, 병렬 계산

**참고 표준**: IEEE 802.3, ISO 3309

**📢 섹션 요약 비유**: CRC는 신뢰성의 핵심!
