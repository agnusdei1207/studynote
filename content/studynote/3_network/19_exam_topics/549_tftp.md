# TFTP (Trivial FTP) - 단순 파일 전송 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: UDP 기반 단순 파일 전송
> 2. **가치**: 경량, 부팅 지원
> 3. **융합:** BOOTP, PXE, Config Backup

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TFTP는 UDP로 간단한 파일 전송을 합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TFTP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TFTP 동작                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Client                              TFTP Server     │   │ │
│   │   │     │                                      │          │   │ │
│   │   │     │ ── RRQ (Read Request) ────────────→ │          │   │ │
│   │   │     │    파일명, 모드                       │          │   │ │
│   │   │     │                                      │          │   │ │
│   │   │     │ ←── DATA (512바이트 블록) ────────── │          │   │ │
│   │   │     │                                      │          │   │ │
│   │   │     │ ── ACK ───────────────────────────→ │          │   │ │
│   │   │     │                                      │          │   │ │
│   │   │     │ ←── DATA (마지막 < 512바이트) ────── │          │   │ │
│   │   │     │                                      │          │   │ │
│   │   │     │ ── ACK ───────────────────────────→ │          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: TFTP는 Stop-and-Wait 방식으로 신뢰성을 보장합니다.

**📢 섹션 요약 비유:** TFTP = 미니 FTP!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **UDP 69** | 포트 |
| **RRQ** | Read Request |
| **WRQ** | Write Request |
| **DATA** | 데이터 블록 (최대 512바이트) |
| **ACK** | 승인 |

**핵심 알고리즸:** 요청 → 데이터 → ACK 반복

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**TFTP vs FTP**

| 항목 | TFTP | FTP |
|:---|:---|:---|
| **전송** | UDP | TCP |
| **인증** | 없음 | 있음 |
| **크기** | 작음 | 큼 |
| **용도** | 부팅 | 파일 전송 |
