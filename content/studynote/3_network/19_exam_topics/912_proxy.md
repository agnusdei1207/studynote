# Proxy - 프록시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 중개 서버
> 2. **가치**: 보안, 캐시, 필터링
> 3. **융합:** Forward Proxy, Reverse Proxy, Transparent

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Proxy는 클라이언트와 서버 사이의 중개 서버입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Proxy                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              프록시 유형                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Forward Proxy:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Client → Proxy → Server                         ││   │ │
│   │   │  │ ┌─────┐   ┌───────┐   ┌──────┐                 ││   │ │
│   │   │  │ │User │──→│Forward│──→│Web   │                 ││   │ │
│   │   │  │ │     │   │Proxy  │   │Server│                 ││   │ │
│   │   │  │ └─────┘   └───────┘   └──────┘                 ││   │ │
│   │   │  │           용도: 필터링, 캐시, 익명성              ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  Reverse Proxy:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Client → Proxy → Backend Servers                ││   │ │
│   │   │  │ ┌─────┐   ┌───────┐   ┌──────────┐             ││   │ │
│   │   │  │ │User │──→│Reverse│──→│Server 1/2│             ││   │ │
│   │   │  │ │     │   │Proxy  │   │          │             ││   │ │
│   │   │  │ └─────┘   └───────┘   └──────────┘             ││   │ │
│   │   │  │           용도: 로드밸런싱, 보안, 캐시            ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 프록시는 네트워크 중계의 핵심입니다.

**📢 섹션 요약 비유:** Proxy = 대리인!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Cache** | 캐시 저장 |
| **Filter** | 필터링 |
| **ACL** | 접근 제어 |
| **Logging** | 로깅 |
| **Anonymity** | 익명성 |

**핵심 알고리듬:** 요청 수신 → 정책 확인 → 중계/차단

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**프록시 유형 비교**

| 유형 | 방향 | 주 용도 |
|:---|:---|:---|
| **Forward** | 아웃바운드 | 필터링 |
| **Reverse** | 인바운드 | 로드밸런싱 |
| **Transparent** | 양방향 | 모니터링 |
| **SOCKS** | 아웃바운드 | 터널링 |
