# Traffic Shaping - 트래픽 쉐이핑

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트래픽 속도 조절
> 2. **가치**: 버스트 완화
> 3. **융합:** Token Bucket, Leaky Bucket, Policing

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Traffic Shaping은 트래픽 전송 속도를 조절합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Traffic Shaping                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              쉐이핑 알고리즘                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Leaky Bucket:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │            ╭───╮                                ││   │ │
│   │   │  │  버스트 ──→│   │──→ 일정 속도 ──→              ││   │ │
│   │   │  │            │   │    (물방울)                    ││   │ │
│   │   │  │            ╰───╯                                ││   │ │
│   │   │  │           양동이                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │  - 일정 속도로 출력                                   │   │ │
│   │   │  - 버스트 불가                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Token Bucket:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │              토큰 생성 (r/sec)                   ││   │ │
│   │   │  │                  ↓                               ││   │ │
│   │   │  │            ┌─────────┐                          ││   │ │
│   │   │  │  패킷 ────→│토큰 버킷│────→ 전송 (토큰 있으면)   ││   │ │
│   │   │  │            │ [●●●○○]│  대기 (토큰 없으면)       ││   │ │
│   │   │  │            └─────────┘                          ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │  - 버스트 허용 (버킷 크기만큼)                        │   │ │
│   │   │  - 평균 속도 제한                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  Shaping vs Policing:                                 │   │ │
│   │   │  - Shaping: 버퍼링 후 전송                            │   │ │
│   │   │  - Policing: 초과 시 폐기                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 트래픽 쉐이핑은 네트워크 안정성을 유지합니다.

**📢 섹션 요약 비유:** Traffic Shaping = 댐!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Token Bucket** | 토큰 버킷 |
| **Leaky Bucket** | 리키 버킷 |
| **Rate** | 속도 |
| **Burst** | 버스트 |
| **Committed** | 약속된 |

**핵심 알고리즸:** 패킷 → 토큰 확인 → 전송/대기

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Shaping vs Policing**

| 항목 | Shaping | Policing |
|:---|:---|:---|
| **동작** | 버퍼링 | 폐기 |
| **지연** | 있음 | 없음 |
| **손실** | 없음 | 있음 |
| **용도** | 송신측 | 수신측 |

**시나리오 1:** ISP → Token Bucket
**시나리오 2:** 데이터센터 → Shaping
**시나리오 3:** QoS → Policing
