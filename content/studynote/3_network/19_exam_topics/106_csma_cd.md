# CSMA/CD (충돌 감지)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 중 충돌 감지 시 즉시 중단
> 2. **가치**: 유선 이더넷의 효율적 충돌 처리
> 3. **융합**: 유선 LAN, 허브/스위치와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CSMA/CD(Carrier Sense Multiple Access with Collision Detection)는 CSMA에 충돌 감지 기능을 추가하여, 전송 중 충돌을 감지하면 즉시 전송을 중단하고 재전송하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CSMA/CD (충돌 감지)                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CSMA/CD 동작 과정                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  알고리즘:                                              │   │ │
│   │   │  1. 채널 감지 (Carrier Sense)                           │   │ │
│   │   │  2. 채널 비면 전송 시작                                  │   │ │
│   │   │  3. 전송 중 충돌 감지 (Collision Detection)              │   │ │
│   │   │  4. 충돌 시:                                            │   │ │
│   │   │     - Jam 신호 전송                                     │   │ │
│   │   │     - 백오프 알고리즘 대기                               │   │ │
│   │   │     - 재전송                                            │   │ │
│   │   │  5. 충돌 없이 완료 시 종료                               │   │ │
│   │   │                                                      │   │ │
│   │   │  충돌 감지 조건:                                         │   │ │
│   │   │  • 전송 중 신호 레벨 모니터링                             │   │ │
│   │   │  • 예상 레벨과 다르면 충돌로 판단                        │   │ │
│   │   │  • 2τ (왕복 지연) 내에 감지 가능해야                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              충돌 감지 원리                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  시나리오:                                               │   │ │
│   │   │                                                      │   │ │
│   │   │  노드A ──────────────────────────────────────→       │   │ │
│   │   │          │                                              │   │ │
│   │   │          │← 충돌 지점                                   │   │ │
│   │   │          ↓                                              │   │ │
│   │   │  노드B ──────────────────────────────────────→        │   │ │
│   │   │                                                      │   │ │
│   │   │  조건:                                                  │   │ │
│   │   │  • 프레임 길이 ≥ 2τ (왕복 전파 지연)                    │   │ │
│   │   │  • 10Mbps 이더넷: 64바이트 최소 프레임                  │   │ │
│   │   │  • 10BASE-T: 51.2μs 충돌 도메인                        │   │ │
│   │   │                                                      │   │ │
│   │   │  최소 프레임 크기 계산:                                  │   │ │
│   │   │  64 bytes × 8 bits / 10Mbps = 51.2μs                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: CSMA/CD는 유선 이더넷(802.3)의 핵심 매체 접속 방식입니다.

**💡 비유**: CSMA/CD는 말하다가 다른 사람 목소리가 들리면 멈추는 것입니다.

**📢 섹션 요약 비유**: CSMA + 충돌 감지 = CSMA/CD!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Jam 신호** | 충돌 알림 | 32비트 | 외침 |
| **백오프** | 재전송 지연 | Binary Exp | 대기 |
| **2τ** | 충돌 도메인 | 왕복 지연 | 시간 |

**비교표**

```
+------------+----------+----------+
| 항목       | CSMA     | CSMA/CD  |
+------------+----------+----------+
| 충돌 감지  | X        | O        |
| 즉시 중단  | X        | O        |
| 유선/무선  | 둘 다    | 유선     |
| 효율       | 낮음     | 높음     |
+------------+----------+----------+
```

**핵심 알고리즸**: 감지 → 전송 → 충돌감지 → Jam → 백오프 → 재전송

**📢 섹션 요약 비유**: 유선은 CD, 무선은 CA!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 이더넷 진화**

| 세대 | 속도 | 매체 접속 |
|:---|:---:|:---|
| **10BASE-T** | 10Mbps | CSMA/CD |
| **100BASE-TX** | 100Mbps | CSMA/CD |
| **1000BASE-T** | 1Gbps | CSMA/CD |
| **스위치** | - | Full Duplex |

**📢 섹션 요약 비유**: 스위치는 CSMA/CD 불필요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 허브** → CSMA/CD 필수
**시나리오 2: 스위치** → CSMA/CD 불필요 (Full Duplex)
**시나리오 3: 무선** → CSMA/CA 사용

**도입 체크리스트**: 매체 유형, 허브/스위치, 전이중

**📢 섹션 요약 비유**: 현대 이더넷은 스위치!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 충돌 시 빠른 복구

**미래 전망**: 스위칭으로 CSMA/CD 감소

**참고 표준**: IEEE 802.3

**📢 섹션 요약 비유**: CSMA/CD는 유선의遗産!
