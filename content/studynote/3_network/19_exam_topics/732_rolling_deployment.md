# Rolling Deployment - 롤링 배포

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 순차적 인스턴스 교체

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Max Surge** | 최대 추가 |
| **Max Unavailable** | 최대 불가용 |
| **Health Check** | 상태 확인 |
| **Progress** | 진행률 |
| **Rollback** | 롤백 |

**핵심 알고리즸:** 종료 → 배포 → Health Check → 반복

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**배포 전략 비교**

| 전략 | 장점 | 단점 |
|:---|:---|:---|
| **Rolling** | 리소스 효율 | 느린 롤백 |
| **Blue-Green** | 빠른 롤백 | 리소스 2배 |
| **Canary** | 위험 최소화 | 복잡 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** Kubernetes → Rolling (기본)
**시나리오 2:** 빠른 롤백 필요 → Blue-Green
**시나리오 3:** 위험 최소화 → Canary
