# IPv6 - 인터넷 프로토콜 v6

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 128비트 주소 체계
> 2. **가치**: 사실상 무제한 주소
> 3. **융합:** Header, Extension, Auto-configuration

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IPv6는 128비트 주소를 사용하는 차세대 인터넷 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IPv6                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IPv6 헤더                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Header 구조 (40 Bytes 고정):                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Version (4)  Traffic Class (8)  Flow Label (20)││   │ │
│   │   │  │ Payload Length (16)  Next Header (8)  Hop Limit││   │ │
│   │   │  │ Source Address (128 bits)                      ││   │ │
│   │   │  │ Destination Address (128 bits)                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  주소 형식:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ 128비트 = 16 octets                              ││   │ │
│   │   │  │ 예: 2001:0db8:85a3:0000:0000:8a2e:0370:7334     ││   │ │
│   │   │  │ 축약: 2001:db8:85a3::8a2e:370:7334              ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  주소 공간: 2^128 ≈ 340간 (3.4×10^38)                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IPv6는 주소 고갈 문제를 해결합니다.

**📢 섹션 요약 비유:** IPv6 = 미래의 주소 체계!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Flow Label** | 흐름 레이블 |
| **Next Header** | 다음 헤더 |
| **Hop Limit** | 홉 제한 |
| **Extension** | 확장 헤더 |
| **SLAAC** | 자동 설정 |

**핵심 알고리듬:** 단순화된 헤더로 빠른 처리

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**IPv4 vs IPv6**

| 항목 | IPv4 | IPv6 |
|:---|:---|:---|
| **주소** | 32비트 | 128비트 |
| **헤더** | 가변 | 고정 |
| **NAT** | 필수 | 불필요 |
| **보안** | 옵션 | 기본 (IPsec) |
