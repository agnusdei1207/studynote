# 서브네팅 (Subnetting) - FLSM/VLSM/CIDR

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크를 논리적으로 분할
> 2. **가치**: IP 주소 효율, 브로드캐스트 억제
> 3. **융합**: VLSM, CIDR와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

서브네팅(Subnetting)은 IP 네트워크를 논리적으로 분할하여 효율적으로 사용하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    서브네팅 (Subnetting)                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              서브네팅 개념                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IP 주소 구조:                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  192.168.1.100 / 255.255.255.0                       │   │ │
│   │   │  └──────┬──────┘   └──────┬──────┘                   │   │ │
│   │   │     네트워크            서브넷 마스크                  │   │ │
│   │   │                                                      │   │ │
│   │   │  CIDR 표기: 192.168.1.100/24                         │   │ │
│   │   │              └──┬──┘                                  │   │ │
│   │   │             비트 수                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  IP 주소:   192.168.1.100                            │   │ │
│   │   │  서브넷:    255.255.255.0  (/24)                     │   │ │
│   │   │  ───────────────────────                             │   │ │
│   │   │  네트워크:  192.168.1.0                              │   │ │
│   │   │  브로드캐스트: 192.168.1.255                          │   │ │
│   │   │  호스트 범위: 192.168.1.1 - 254                       │   │ │
│   │   │  가용 호스트: 254 (2^8 - 2)                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              서브네팅 방식                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. FLSM (Fixed Length Subnet Mask):                   │   │ │
│   │   │     • 모든 서브넷 동일 크기                              │   │ │
│   │   │     • 단순하지만 낭비                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  2. VLSM (Variable Length Subnet Mask):                │   │ │
│   │   │     • 서브넷별 다른 크기                                │   │ │
│   │   │     • 효율적                                           │   │ │
│   │   │     • 큰 것부터 할당                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  3. CIDR (Classless Inter-Domain Routing):             │   │ │
│   │   │     • 클래스 없는 라우팅                                │   │ │
│   │   │     • 슈퍼네팅 (Supernetting) 가능                       │   │ │
│   │   │     • 인터넷 표준                                       │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: VLSM은 큰 서브넷부터 할당하여 IP 주소를 효율적으로 사용합니다.

**💡 비유**: 서브네팅은 큰 방을 칸막이로 나누는 것입니다.

**📢 섹션 요약 비유:** /24 = 256개, /25 = 128개!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| CIDR | 마스크 | 호스트 수 | 용도 |
|:---:|:---|:---:|:---|
| **/24** | 255.255.255.0 | 254 | 소규모 LAN |
| **/25** | 255.255.255.128 | 126 | 중간 |
| **/26** | 255.255.255.192 | 62 | 부서 |
| **/27** | 255.255.255.224 | 30 | 소규모 |
| **/28** | 255.255.255.240 | 14 | VLAN |
| **/30** | 255.255.255.252 | 2 | 링크 |

**비교표**

```
+------------+----------+----------+
| 항목       │ FLSM     │ VLSM     │
+------------+----------+----------+
| 서브넷 크기│ 동일     │ 다름     │
│ 효율성     │ 낮음     │ 높음     │
│ 복잡도     │ 낮음     │ 높음     │
│ 낭비       │ 있음     │ 최소     │
+------------+----------+----------+
```

**핵심 알고리즸:** 2^n - 2, 서브넷 계산

**📢 섹션 요약 비유:** VLSM = IP 절약!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 소규모** → /24 고정
**시나리오 2: 다양한 크기** → VLSM
**시나리오 3: 링크** → /30

**도입 체크리스트:** 호스트 수, 성장률, 라우팅

**📢 섹션 요약 비유:** 30% 여유분 계획!
