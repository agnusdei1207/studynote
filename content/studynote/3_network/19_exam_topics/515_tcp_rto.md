# TCP RTO (Retransmission Timeout) - 재전송 타임아웃

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 재전송 대기 시간
> 2. **가치**: 손실 탐지, 혼잡 반응
> 3. **융합:** RTT, Karn's Algorithm, Exponential Backoff

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RTO는 재전송을 기다리는 시간입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TCP RTO                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RTO 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  송신측                    수신측                    │   │ │
│   │   │     │                         │                      │   │ │
│   │   │     │ ─── Data ─────────────→ │ (손실)              │   │ │
│   │   │     X                         │                      │   │ │
│   │   │     │                         │                      │   │ │
│   │   │     │ ─── Data (재전송) ────→ │ RTO 경과            │   │ │
│   │   │     │                         │                      │   │ │
│   │   │  RTO 초과 → 재전송 + 혼잡 반응                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RTO는 RTT를 기반으로 동적으로 계산됩니다.

**📢 섹션 요약 비유:** RTO = 재전송 타이머!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 공식 |
|:---|:---|
| **RTO** | EstimatedRTT + 4 × DevRTT |
| **Min RTO** | 1초 |
| **Max RTO** | 120초 |
| **Backoff** | × 2 |

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**RTO vs Fast Retransmit**

| 항목 | RTO | Fast Retransmit |
|:---|:---|:---|
| **트리거** | 타이머 | 3 DupACK |
| **시간** | 길음 | 짧음 |
| **혼잡** | 큼 | 작음 |
