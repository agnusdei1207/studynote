# Collision Domain vs Broadcast Domain

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 충돌/브로드캐스트 범위
> 2. **가치**: 성능 최적화, 세그먼트
> 3. **융합**: 허브, 스위치, 라우터

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Collision Domain은 충돌이 발생할 수 있는 범위, Broadcast Domain은 브로드캐스트가 전파되는 범위입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Collision vs Broadcast Domain                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              장비별 도메인                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  허브 (Hub):                                        │   │ │
│   │   │  - Collision Domain: 1개 (전체)                     │   │ │
│   │   │  - Broadcast Domain: 1개                            │   │ │
│   │   │                                                      │   │ │
│   │   │  스위치 (Switch):                                   │   │ │
│   │   │  - Collision Domain: 포트별 (N개)                   │   │ │
│   │   │  - Broadcast Domain: 1개                            │   │ │
│   │   │                                                      │   │ │
│   │   │  라우터 (Router):                                   │   │ │
│   │   │  - Collision Domain: 포트별                         │   │ │
│   │   │  - Broadcast Domain: 포트별 (N개)                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              비교표                                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+----------+     │   │ │
│   │   │  │ 장비       │ 허브     │ 스위치   │ 라우터   │     │   │ │
│   │   │  +------------+----------+----------+----------+     │   │ │
│   │   │  │ Collision  │ 1개      │ N개      │ N개      │     │   │ │
│   │   │  │ Broadcast  │ 1개      │ 1개      │ N개      │     │   │ │
│   │   │  | 계층       │ L1       │ L2       │ L3       │     │   │ │
│   │   │  +------------+----------+----------+----------+     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스위치는 Collision Domain을 분리하고, 라우터는 Broadcast Domain을 분리합니다.

**💡 비유**: Collision Domain은 좁은 길, Broadcast Domain은 확성기 범위입니다.

**📢 섹션 요약 비유:** 스위치 = Collision 분리!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Collision Domain** | CSMA/CD 영역 |
| **Broadcast Domain** | 브로드캐스트 전파 영역 |
| **Segment** | 네트워크 구간 |
| **Microsegmentation** | 마이크로 세그멘테이션 |

**비교표: 도메인 분리**

```
+------------+----------+----------+
| 항목       │ Collision│ Broadcast│
+------------+----------+----------+
| 분리장비   │ 스위치   │ 라우터   │
| L2 VLAN    │ X        │ O        │
| 성능영향   │ 충돌     │ 브로드캐스트│
+------------+----------+----------+
```

**핵심 알고리즸:** MAC 학습, 브로드캐스트 필터링

**📢 섹션 요약 비유:** 라우터 = Broadcast 분리!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 도메인 축소**

| 항목 | 허브 | 스위치 | VLAN |
|:---|:---|:---|:---|
| **Collision** │ 1 | N | N |
| **Broadcast** │ 1 | 1 | N |

**📢 섹션 요약 비유:** VLAN = Broadcast 분리!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 충돌 감소 → 스위치
**시나리오 2:** 브로드캐스트 감소 → VLAN/라우터
**시나리오 3:** 대역폭 증가 → 전이중 스위치

**도입 체크리스트:** 장비 선택, VLAN 설계, 라우팅

**📢 섹션 요약 비유:** 설계 = 도메인 분리!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 성능 향상

**미래 전망:** 전이상 전환으로 충돌 없음

**참고 표준:** IEEE 802.3

**📢 섹션 요약 비유:** 도메인 분리는 성능의 핵심!
