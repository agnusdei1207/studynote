# Authorization - 인가

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 권한 부여
> 2. **가치**: 접근 제어
> 3. **융합:** RBAC, ACL, OAuth Scopes

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Authorization은 사용자의 권한을 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Authorization                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              권한 부여 모델                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  인증 vs 인가:                                          │   │ │
│   │   │  - Authentication: "누구인가?"                         │   │ │
│   │   │  - Authorization: "무엇을 할 수 있는가?"                 │   │ │
│   │   │                                                      │   │ │
│   │   │  RBAC (Role-Based Access Control):                     │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ User ──→ Role ──→ Permission ──→ Resource        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ 예:                                             ││   │ │
│   │   │  │ Admin → 모든 권한                                ││   │ │
│   │   │  │ Editor → 읽기/쓰기                               ││   │ │
│   │   │  │ Viewer → 읽기만                                  ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  ACL (Access Control List):                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Resource: /file.txt                             ││   │ │
│   │   │  │ ACL: user1: rw, user2: r, group1: rwx           ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  OAuth Scopes:                                         │   │ │
│   │   │  scope=read,write,admin                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 인가는 리소스 접근 제어의 핵심입니다.

**📢 섹션 요약 비유:** Authorization = 출입카드 권한!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Role** | 역할 |
| **Permission** | 권한 |
| **Scope** | 범위 |
| **Policy** | 정책 |
| **Resource** | 리소스 |

**핵심 알고리즸:** 사용자 → 역할 → 권한 → 접근 허용/거부

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**인가 모델 비교**

| 모델 | 유연성 | 복잡도 | 용도 |
|:---|:---|:---|:---|
| **RBAC** | 중간 | 낮음 | 기업 |
| **ABAC** | 높음 | 높음 | 세밀 제어 |
| **ACL** | 낮음 | 낮음 | 파일 |
| **PBAC** | 높음 | 중간 | 클라우드 |
