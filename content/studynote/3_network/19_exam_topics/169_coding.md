# 부호화 (Coding) - 채널/소스/라인

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 효율적/신뢰성 있게 표현
> 2. **가치**: 전송 효율, 오류 정정
> 3. **융합**: 모뎀, 코덱과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

부호화(Coding)는 데이터를 특정 목적에 맞게 변환하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    부호화 (Coding)                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              부호화 종류                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 소스 부호화 (Source Coding):                        │   │ │
│   │   │     • 데이터 압축                                     │   │ │
│   │   │     • 무손실: Huffman, LZ77, ARITHMETIC               │   │ │
│   │   │     • 손실: JPEG, MP3, H.264                          │   │ │
│   │   │     • 목적: 대역폭 절약                                │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 채널 부호화 (Channel Coding):                       │   │ │
│   │   │     • 오류 정정/검출                                   │   │ │
│   │   │     • 블록: Hamming, BCH, Reed-Solomon, LDPC          │   │ │
│   │   │     • 컨볼루션: 비터비, 터보                           │   │ │
│   │   │     • 목적: 신뢰성 향상                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 라인 부호화 (Line Coding):                          │   │ │
│   │   │     • 디지털→전송 신호 변환                            │   │ │
│   │   │     • NRZ, NRZI, Manchester, 4B/5B, 8B/10B           │   │ │
│   │   │     • 목적: 클럭 복구, DC 균형                         │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 암호화 (Encryption):                                │   │ │
│   │   │     • 보안                                            │   │ │
│   │   │     • 대칭: AES, DES                                  │   │ │
│   │   │     • 비대칭: RSA, ECC                                │   │ │
│   │   │     • 목적: 기밀성                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              통신 시스템에서의 위치                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [소스]→암호화→소스부호→채널부호→라인부호→[채널]      │   │ │
│   │   │                    ↓                                 │   │ │
│   │   │                [수신]←복호←복호←복호←복호             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 부호화는 통신 시스템의 핵심 요소입니다.

**💡 비유**: 부호화는 편지를 암호문으로 바꾸고 봉투에 넣는 것입니다.

**📢 섹션 요약 비유:** 채널 부호화 = 오류 정정!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 부호화 | 목적 | 대표 기법 | 효과 |
|:---|:---|:---|:---|
| **소스** | 압축 | Huffman, LZ | 대역폭 절약 |
| **채널** | 오류 정정 | LDPC, Turbo | 신뢰성 |
| **라인** | 전송 적응 | 8B/10B | 클럭 복구 |
| **암호** | 보안 | AES, RSA | 기밀성 |

**비교표**

```
+------------+----------+----------+
| 채널 부호   │ 블록     │ 컨볼루션 │
+------------+----------+----------+
| 구조       │ 블록 단위 │ 연속     │
│ 메모리     │ 없음     │ 있음     │
│ 복잡도     │ 중간     │ 높음     │
│ 예         │ LDPC     │ Turbo    │
+------------+----------+----------+
```

**핵심 알고리즸:** Huffman, Viterbi, BCJR

**📢 섹션 요약 비유:** LDPC/Turbo = 5G 부호화!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 오류 정정 성능**

| 부호 | 코드율 | BER 개선 | 용도 |
|:---|:---:|:---:|:---|
| **Hamming** | 4/7 | 10배 | 메모리 |
| **RS** | 가변 | 100배 | CD/DVD |
| **Turbo** | 1/2-1/3 | 1000배 | 3G/4G |
| **LDPC** | 가변 | 10000배 | 5G/WiFi6 |

**📢 섹션 요약 비유:** LDPC = 최고 성능!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 5G 데이터** → LDPC
**시나리오 2: 5G 제어** → Polar
**시나리오 3: WiFi 6** → LDPC + BCC

**도입 체크리스트:** BER 요구사항, 지연, 복잡도

**📢 섹션 요약 비유:** 용도에 맞는 부호화!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 신뢰성 10000배 향상

**미래 전망:** 양자 내성 암호, AI 기반 부호화

**참고 표준:** 3GPP TS 38.212, IEEE 802.11

**📢 섹션 요약 비유:** 부호화는 통신의 안전망!
