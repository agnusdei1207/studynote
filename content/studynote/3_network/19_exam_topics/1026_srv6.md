# SRv6 - 세그먼트 라우팅 IPv6

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPv6 기반 세그먼트 라우팅
> 2. **가치**: 프로그래밍 가능한 네트워크
> 3. **융합:** SRH, SID, End Functions

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SRv6는 IPv6 확장 헤더를 이용한 세그먼트 라우팅입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SRv6                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SRv6 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IPv6 패킷 구조:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  ┌─────────────────────────────────────────────┐││   │ │
│   │   │  │  │ IPv6 Header                                 │││   │ │
│   │   │  │  │ Dest: SID1                                  │││   │ │
│   │   │  │  └─────────────────────────────────────────────┘││   │ │
│   │   │  │  ┌─────────────────────────────────────────────┐││   │ │
│   │   │  │  │ SRH (Segment Routing Header)                │││   │ │
│   │   │  │  │ ┌────────────────────────────────────────┐ │││   │ │
│   │   │  │  │ │ Segment List: [SID3][SID2][SID1]       │ │││   │ │
│   │   │  │  │ │ Segments Left: 2 → 1 → 0               │ │││   │ │
│   │   │  │  │ └────────────────────────────────────────┘ │││   │ │
│   │   │  │  └─────────────────────────────────────────────┘││   │ │
│   │   │  │  ┌─────────────────────────────────────────────┐││   │ │
│   │   │  │  │ Payload                                     │││   │ │
│   │   │  │  └─────────────────────────────────────────────┘││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  End Functions:                                   ││   │ │
│   │   │  │  - End: SID 처리                                   ││   │ │
│   │   │  │  - End.X: 인접으로 전달                             ││   │ │
│   │   │  │  - End.T: 특정 테이블 조회                          ││   │ │
│   │   │  │  - End.DX6: IPv6 디캡슐레이션                       ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SRv6는 IPv6 네이티브로 프로그래밍 가능한 네트워크를 제공합니다.

**📢 섹션 요약 비유:** SRv6 = IPv6 프로그래밍!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SRH** | Segment Routing Header |
| **SID** | 128-bit Segment Identifier |
| **Segments Left** | 남은 세그먼트 |
| **End Function** | 종단 함수 |
| **Behavior** | 동작 |

**핵심 알고리듬:** SID 활성 → SL 감소 → 다음 SID로 이동

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SRv6 vs SR-MPLS**

| 항목 | SRv6 | SR-MPLS |
|:---|:---|:---|
| **데이터 평면** | IPv6 | MPLS |
| **SID 크기** | 128-bit | 32-bit |
| **호환성** | IPv6 필요 | MPLS 필요 |
| **프로그래밍** | 강력 | 제한 |
