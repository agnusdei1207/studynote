# Canary Deployment - 카나리 배포

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 점진적 트래픽 이동
> 2. **가치**: 위험 최소화, A/B 테스트
> 3. **융합:** Feature Flags, Monitoring, CI/CD

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Canary Deployment는 점진적으로 트래픽을 이동합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Canary Deployment                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              카나리 배포 구조                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  단계별 트래픽 이동:                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  Step 1:          Step 2:          Step 3:          │   │ │
│   │   │  ┌────────┐       ┌────────┐       ┌────────┐       │   │ │
│   │   │  │ 95%    │       │ 80%    │       │ 50%    │       │   │ │
│   │   │  │ Stable │       │ Stable │       │ Stable │       │   │ │
│   │   │  └────────┘       └────────┘       └────────┘       │   │ │
│   │   │  ┌────────┐       ┌────────┐       ┌────────┐       │   │ │
│   │   │  │ 5%     │ ───→ │ 20%    │ ───→ │ 50%    │       │   │ │
│   │   │  │ Canary │       │ Canary │       │ Canary │       │   │ │
│   │   │  └────────┘       └────────┘       └────────┘       │   │ │
│   │   │                                                      │   │ │
│   │   │  모니터링: 에러율, 지연시간, 비즈니스 메트릭             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 카나리 배포는 소수 사용자에게 먼저 테스트합니다.

**📢 섹션 요약 비유:** Canary = 광산의 카나리아!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Stable** | 안정 버전 |
| **Canary** | 카나리 버전 |
| **Traffic Split** | 트래픽 분할 |
| **Metrics** | 메트릭 |
| **Auto Rollback** | 자동 롤백 |

**핵심 알고리즸:** 카나리 배포 → 모니터링 → (증가/롤백)

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹 → 1% → 5% → 10% → 100%
**시나리오 2:** 내부 사용자 → 먼저 테스트
**시나리오 3:** A/B 테스트 → 카나리 활용
