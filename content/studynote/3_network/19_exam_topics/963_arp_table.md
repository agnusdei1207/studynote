# ARP Table - ARP 테이블

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP-MAC 매핑 캐시
> 2. **가치**: 반복 조회 방지
> 3. **융합:** Cache, Timeout, Entry

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ARP Table은 IP와 MAC 주소 매핑을 캐시합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ARP Table                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ARP 테이블 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ARP 테이블 예시:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ IP Address      MAC Address        Type         ││   │ │
│   │   │  │ ──────────      ──────────        ─────         ││   │ │
│   │   │  │ 192.168.1.1     00:11:22:33:44:55  Dynamic      ││   │ │
│   │   │  │ 192.168.1.10    AA:BB:CC:DD:EE:FF  Dynamic      ││   │ │
│   │   │  │ 192.168.1.254   11:22:33:44:55:66  Dynamic      ││   │ │
│   │   │  │ 224.0.0.22      01:00:5e:00:00:16  Static       ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Windows: arp -a                                ││   │ │
│   │   │  │  Linux: ip neigh show                           ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  Entry 유형:                                            │   │ │
│   │   │  - Dynamic: ARP로 학습 (일반적으로 30-60분 만료)          │   │ │
│   │   │  - Static: 수동 등록 (영구)                              │   │ │
│   │   │                                                      │   │ │
│   │   │  ARP 동작:                                               │   │ │
│   │   │  1. ARP Table 조회                                      │   │ │
│   │   │  2. 있으면 사용                                           │   │ │
│   │   │  3. 없으면 ARP Request 브로드캐스트                        │   │ │
│   │   │  4. 응답으로 Table 갱신                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ARP Table은 네트워크 성능을 향상시킵니다.

**📢 섹션 요약 비유:** ARP Table = 주소록!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Entry** | 항목 |
| **Dynamic** | 동적 학습 |
| **Static** | 정적 등록 |
| **Timeout** | 만료 시간 |
| **Cache** | 캐시 |

**핵심 알고리듬:** 조회 → 있으면 사용 → 없으면 ARP

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ARP vs RARP**

| 항목 | ARP | RARP |
|:---|:---|:---|
| **질문** | IP가 뭐야? | MAC이 뭐야? |
| **목적** | MAC 찾기 | IP 찾기 |
| **용도** | 통신 | 부팅 |
