# XSS - 크로스 사이트 스크립팅

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 악성 스크립트 주입 공격
> 2. **가치**: 세션 탈취, 쿠키 도난
> 3. **융합**: WAF, 입력 검증, CSP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

XSS(Cross-Site Scripting)는 웹페이지에 악성 스크립트를 주입합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    XSS                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              XSS 공격 과정                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [공격자]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ <script>악성코드</script>                      │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [서버] ──── 저장/반환 ────▶                         │   │ │
│   │   │                            │                          │   │ │
│   │   │                            ▼                          │   │ │
│   │   │  [피해자 브라우저]                                    │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 스크립트 실행                                   │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [쿠키/세션 탈취] ────▶ [공격자]                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              XSS 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 유형       │ 특징     │ 지속성   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Stored     │ 서버저장 │ 영구     │               │   │ │
│   │   │  │ Reflected  │ URL반영  │ 일시     │               │   │ │
│   │   │  | DOM-based  │ 클라이언트│ 일시    │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: XSS는 사용자 브라우저에서 악성 스크립트를 실행합니다.

**💡 비유**: XSS는 웹페이지에 숨겨진 도청기입니다.

**📢 섹션 요약 비유:** XSS = 스크립트 주입!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Stored XSS** | 저장형 XSS |
| **Reflected XSS** | 반사형 XSS |
| **DOM-based XSS** | DOM 기반 XSS |
| **Payload** | 공격 코드 |

**비교표: XSS 방어**

```
+------------+----------+----------+
| 항목       │ 입력검증 │ 출력인코딩│
+------------+----------+----------+
| Stored     │ O        │ O        │
| Reflected  │ O        │ O        │
| DOM-based  │ X        │ O        │
+------------+----------+----------+
```

**핵심 알고리즸:** 입력 검증, 출력 인코딩

**📢 섹션 요약 비유:** 인코딩 = XSS 방어!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹 앱 → 입력 검증 + 출력 인코딩
**시나리오 2:** 관리자 페이지 → CSP + HttpOnly
**시나리오 3:** 사용자 입력 → sanitize

**도입 체크리스트:** 입력/출력 처리, CSP, 쿠키 보안

**📢 섹션 요약 비유:** 방어는 다층으로!
