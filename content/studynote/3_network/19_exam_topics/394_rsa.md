# RSA - Rivest-Shamir-Adleman

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소인수분해 기반 공개키 암호
> 2. **가치**: 암호화와 서명 모두 가능
> 3. **융합**: TLS, PGP, PKI

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RSA는 큰 수의 소인수분해 난이도에 기반한 암호 시스템입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RSA                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RSA 키 생성                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 두 소수 p, q 선택                                │   │ │
│   │   │     p = 61, q = 53                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  2. n = p × q 계산                                   │   │ │
│   │   │     n = 3233                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  3. φ(n) = (p-1)(q-1) 계산                          │   │ │
│   │   │     φ(n) = 3120                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 공개키 e 선택 (φ(n)과 서로소)                    │   │ │
│   │   │     e = 17                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  5. 개인키 d 계산 (e × d ≡ 1 mod φ(n))              │   │ │
│   │   │     d = 2753                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  공개키: (n=3233, e=17)                              │   │ │
│   │   │  개인키: (n=3233, d=2753)                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RSA 암호화/복호화                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  암호화: C = M^e mod n                              │   │ │
│   │   │  복호화: M = C^d mod n                              │   │ │
│   │   │  서명:   S = M^d mod n                              │   │ │
│   │   │  검증:   M = S^e mod n                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RSA는 암호화와 디지털 서명에 모두 사용됩니다.

**💡 비유**: RSA는 두 개의 열쇠입니다. 하나는 잠그고, 다른 하나는 엽니다.

**📢 섹션 요약 비유:** RSA = 공개키 암호!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **n** | 모듈러스 (p × q) |
| **e** | 공개 지수 |
| **d** | 개인 지수 |
| **PKCS** | 패딩 표준 |

**비교표: RSA 키 길이**

```
+------------+----------+----------+
| 항목       │ 1024비트 │ 2048비트 │
+------------+----------+----------+
| 보안       │ 깨짐     │ 안전     │
| 권장       │ 아니오   │ 예       │
| 속도       │ 빠름     │ 느림     │
+------------+----------+----------+
```

**핵심 알고리즸:** 모듈러 거듭제곱, 중국인 나머지 정리

**📢 섹션 요약 비유:** 2048비트 = 최소!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 공개키 알고리즘**

| 항목 | RSA | ECDSA | Ed25519 |
|:---|:---|:---|:---|
| **키 크기** │ 2048+ │ 256 | 256 |
| **속도** │ 느림 | 중간 | 빠름 |
| **서명 크기** │ 256바이트 | 64바이트 | 64바이트 |

**📢 섹션 요약 비유:** TLS = RSA 또는 ECC!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** TLS 인증서 → RSA-2048/4096
**시나리오 2:** 키 교환 → RSA-KEM
**시나리오 3:** 신규 프로젝트 → ECC 권장

**도입 체크리스트:** 키 길이, 패딩(OAEP), 난수 생성

**📢 섹션 요약 비유:** 레거시 = RSA!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)
