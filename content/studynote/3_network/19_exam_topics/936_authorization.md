# Authorization - 인가

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 권한 확인
> 2. **가치**: 접근 제어
> 3. **융합:** RBAC, ACL, ABAC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Authorization은 사용자 권한을 확인합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Authorization                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인가 모델                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RBAC (Role-Based Access Control):                      │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ User → Role → Permission → Resource             ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Admin     → 모든 권한                           ││   │ │
│   │   │  │  Editor    → 읽기, 쓰기                          ││   │ │
│   │   │  │  Viewer    → 읽기만                              ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  ABAC (Attribute-Based Access Control):                 │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Subject + Resource + Action + Context           ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ 예: 부서=재무 AND 시간=업무시간 AND IP=내부       ││   │ │
│   │   │  │     → 재무 문서 읽기 허용                         ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  ACL (Access Control List):                             │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Resource: /admin                                ││   │ │
│   │   │  │   Allow: admin, superuser                       ││   │ │
│   │   │  │   Deny: guest                                   ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 인가는 리소스 보호의 핵심입니다.

**📢 섹션 요약 비유:** Authorization = 출입 카드 권한!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Subject** | 주체 (사용자) |
| **Resource** | 자원 |
| **Action** | 행위 |
| **Policy** | 정책 |
| **Permission** | 권한 |

**핵심 알고리듬:** 인증 후 → 정책 확인 → 허용/거부
