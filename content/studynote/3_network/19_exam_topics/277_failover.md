# 페일오버 (Failover) - 자동 전환

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장애 시 자동 전환
> 2. **가치**: 무중단 서비스
> 3. **융합**: HA, 클러스터링과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

페일오버는 장애 발생 시 자동으로 대기 시스템으로 전환하는 기능입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    페일오버 (Failover)                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              페일오버 동작                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Normal]                                           │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌──────────┐     ┌──────────┐                      │   │ │
│   │   │  │ Primary  │────▶│ Secondary│                      │   │ │
│   │   │  │ (Active) │ Rep │ (Standby)│                      │   │ │
│   │   │  └──────────┘     └──────────┘                      │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ 장애 감지                                    │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌──────────┐     ┌──────────┐                      │   │ │
│   │   │  │ Primary  │  ✕  │ Secondary│                      │   │ │
│   │   │  │ (Failed) │────▶│ (Active) │  ← Failover          │   │ │
│   │   │  └──────────┘     └──────────┘                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              페일오버 단계                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 감지 (Detection):                                │   │ │
│   │   │     • 하트비트, 헬스체크                              │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 확인 (Confirmation):                             │   │ │
│   │   │     • 중복 확인, 쿼럼                                │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 전환 (Switch):                                   │   │ │
│   │   │     • IP 이동, 서비스 시작                           │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 복구 (Recovery):                                 │   │ │
│   │   │     • 원복 또는 재동기화                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 페일오버는 감지 → 확인 → 전환 → 복구 순서로 진행됩니다.

**💡 비유**: 페일오버는 비상 전원 스위치입니다. 자동으로 전환됩니다.

**📢 섹션 요약 비유:** Failover = 자동 전환!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Heartbeat** | 생존 신호 |
| **Health Check** | 상태 확인 |
| **Quorum** | 합의 |
| **Split-brain** | 분할 장애 |

**비교표: 페일오버 유형**

```
+------------+----------+----------+
| 유형       │ 전환 시간│ 용도     │
+------------+----------+----------+
| Hot        │ 즉시     | 미션 크리티컬│
| Warm       │ 분 단위  │ 일반     │
| Cold       │ 시간     │ 저비용   │
+------------+----------+----------+
```

**핵심 알고리즸:** 하트비트, 쿼럼

**📢 섹션 요약 비유:** Hot Standby = 즉시 전환!
