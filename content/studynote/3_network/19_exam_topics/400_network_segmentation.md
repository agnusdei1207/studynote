# 네트워크 분할 - Network Segmentation

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크를 논리적 영역으로 분리
> 2. **가치**: 공격 확산 방지, 접근 제어
> 3. **융합**: VLAN, 방화벽, 마이크로세그멘테이션

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

네트워크 분할은 네트워크를 보안 영역으로 나눕니다.

> **해설**: 네트워크 분할은 공격 확산(Lateral Movement)을 방지합니다.

**💡 비유**: 네트워크 분할은 격벽입니다. 한 구역의 화재가 번지지 않습니다.

**📢 섹션 요약 비유:** 분할 = 격리!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **VLAN** | 가상 LAN 분할 |
| **Subnet** | IP 서브넷 분할 |
| **Firewall** | 구간 간 필터링 |
| **ACL** | 접근 제어 목록 |

**비교표: 분할 기술**

```
+------------+----------+----------+
| 항목       │ VLAN     │ Microseg │
+------------+----------+----------+
| 계층       │ L2       │ L4-7     │
| 단위       │ 네트워크 │ 워크로드 │
| 동적       │ 정적     │ 동적     │
| 복잡도     │ 낮음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** ACL, 정책 기반 라우팅

**📢 섹션 요약 비유:** 마이크로세그멘테이션 = 세분화!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 보안 구역**

| 항목 | DMZ | 내부망 | 관리망 |
|:---|:---|:---|:---|
| **신뢰** │ 낮음 │ 중간 │ 높음 |
| **접근** │ 제한 | 내부만 | 관리자 |
| **서버** │ 공개 | 업무 | 관리 |

**📢 섹션 요약 비유:** PCI-DSS = 분할 필수!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** DMZ 구성 → 방화벽 + VLAN
**시나리오 2:** 데이터센터 → 마이크로세그멘테이션
**시나리오 3:** PCI-DSS → 카드 데이터 격리

**도입 체크리스트:** 구역 정의, ACL 설계, 모니터링

**📢 섹션 요약 비유:** 규정 준수 = 분할!
