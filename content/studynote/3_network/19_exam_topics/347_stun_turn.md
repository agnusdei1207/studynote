# STUN/TURN - NAT 통과 서버

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: NAT 환경 주소 획득 및 릴레이
> 2. **가치**: P2P 연결 지원
> 3. **융합**: WebRTC, VoIP, 게임

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

STUN은 공인 주소를, TURN은 릴레이 서버를 제공합니다.


> **해설**: STUN은 주소를 알려주고, TURN은 릴레이합니다.

**💡 비유**: STUN은 주소 확인, TURN은 대리 배달입니다.

**📢 섹션 요약 비유:** STUN = 주소, TURN = 릴레이!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Binding Request** | 바인딩 요청 |
| **Mapped Address** | 매핑 주소 |
| **Relay Address** | 릴레이 주소 |
| **Permission** | 권한 |

**비교표: STUN vs TURN**

```
+------------+----------+----------+
| 항목       │ STUN     │ TURN     │
+------------+----------+----------+
| RFC        │ 5389     │ 5766     │
| 포트       │ 3478     │ 3478     │
| 전송       │ UDP/TCP  │ UDP/TCP  │
| 인증       │ 선택     │ 필수     │
+------------+----------+----------+
```

**핵심 알고리즸:** 매핑 발견, 릴레이

**📢 섹션 요약 비유:** TURN = 마지막 수단!
