# Redis Pub/Sub - 레디스 펍섭

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인메모리 메시징
> 2. **가치**: 낮은 지연, 단순함
> 3. **융합:** Real-time, Notifications, Chat

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Redis Pub/Sub은 인메모리 메시징을 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Redis Pub/Sub                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Redis Pub/Sub 구조                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Publisher                                            │   │ │
│   │   │     │                                                 │   │ │
│   │   │     │ PUBLISH channel "message"                       │   │ │
│   │   │     ↓                                                 │   │ │
│   │   │  ┌─────────┐                                         │   │ │
│   │   │  │  Redis  │                                         │   │ │
│   │   │  │  Server │                                         │   │ │
│   │   │  └────┬────┘                                         │   │ │
│   │   │       │                                              │   │ │
│   │   │       ├──────────────┬──────────────┐                │   │ │
│   │   │       ↓              ↓              ↓                │   │ │
│   │   │  ┌─────────┐   ┌─────────┐   ┌─────────┐            │   │ │
│   │   │  │Subscriber│   │Subscriber│   │Subscriber│            │   │ │
│   │   │  │   1     │   │   2     │   │   3     │            │   │ │
│   │   │  └─────────┘   └─────────┘   └─────────┘            │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  - 실시간 (인메모리)                                   │   │ │
│   │   │  - 지속성 없음 (구독자 없으면 손실)                      │   │ │
│   │   │  - 패턴 구독 (PSUBSCRIBE)                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Redis Pub/Sub은 메시지를 저장하지 않고 즉시 전달합니다.

**📢 섹션 요약 비유:** Redis Pub/Sub = 라디오!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Channel** | 채널 |
| **Publisher** | 발행자 |
| **Subscriber** | 구독자 |
| **Pattern** | 패턴 |
| **PMessage** | 패턴 메시지 |

**핵심 알고리즸:** 발행 → 즉시 전달 → 구독자 수신

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Pub/Sub vs Stream**

| 항목 | Pub/Sub | Stream |
|:---|:---|:---|
| **지속성** | 없음 | 있음 |
| **소비** | 푸시 | 풀 |
| **용도** | 실시간 | 로그 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 알림 → Pub/Sub
**시나리오 2:** 채팅 → Pub/Sub
**시나리오 3:** 이벤트 로그 → Stream
