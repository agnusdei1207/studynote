# DSU / CSU (Digital/Channel Service Unit)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디지털 회선 종단 장치
> 2. **가치**: 전용선, WAN 연결
> 3. **융합**: T1/E1, 프레임 릴레이와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DSU(Digital Service Unit)와 CSU(Channel Service Unit)는 디지털 전용선에서 고객 장비와 통신사 네트워크를 연결하는 장치입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DSU / CSU                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DSU/CSU 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [고객 장비]    [DSU/CSU]          [통신사 망]        │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌───────┐    ┌──────────────┐    ┌─────────┐       │   │ │
│   │   │  │라우터 │────│  DSU  │ CSU  │────│ 전용선  │       │   │ │
│   │   │  │(DTE)  │    │       │      │    │(T1/E1)  │       │   │ │
│   │   │  └───────┘    └──────────────┘    └─────────┘       │   │ │
│   │   │                   │                                 │   │ │
│   │   │              ┌────┴────┐                            │   │ │
│   │   │              │         │                            │   │ │
│   │   │            DSU        CSU                            │   │ │
│   │   │         (데이터)    (채널)                            │   │ │
│   │   │                                                      │   │ │
│   │   │  역할:                                                   │   │ │
│   │   │  • DSU: 데이터 신호 변환, 타이밍 동기                    │   │ │
│   │   │  • CSU: 회선 보호, 루프백 테스트, 신호 재생              │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                   │   │ │
│   │   │  • 회선 보호 (과전압, 서지)                              │   │ │
│   │   │  • 신호 재생 및 증폭                                     │   │ │
│   │   │  • 진단 및 모니터링                                      │   │ │
│   │   │  • 타이밍 동기                                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              T1/E1 프레이밍                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  T1 (미국/일본):                                         │   │ │
│   │   │  • 1.544Mbps, 24채널 × 64kbps                          │   │ │
│   │   │  • 프레임: 193비트 (24×8 + 1)                           │   │ │
│   │   │                                                      │   │ │
│   │   │  E1 (유럽/한국):                                          │   │ │
│   │   │  • 2.048Mbps, 32채널 × 64kbps                          │   │ │
│   │   │  • 프레임: 256비트 (32×8)                               │   │ │
│   │   │  • 채널 0: 동기, 채널 16: 신호                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: DSU/CSU는 전용선 연결의 필수 장비로, 통상 하나의 장치로 통합됩니다.

**💡 비유**: DSU/CSU는 전용선의 변압기와 같습니다.

**📢 섹션 요약 비유:** DSU = 데이터, CSU = 회선!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **DSU** | 데이터 처리 | 인코딩/디코딩 | 어댑터 |
| **CSU** | 회선 보호 | 루프백, 재생 | 보호기 |
| **인터페이스** | 연결 | V.35, RS-449 | 포트 |

**비교표**

```
+------------+----------+----------+
| 항목       | T1       | E1       |
+------------+----------+----------+
| 속도       | 1.544Mbps| 2.048Mbps|
| 채널       | 24       | 32       |
| 프레임     | 193비트  | 256비트  |
| 지역       | 미국     | 유럽/한국|
+------------+----------+----------+
```

**핵심 알고리즸**: Bipolar 코딩, HDB3

**📢 섹션 요약 비유:** E1 = 2.048Mbps!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 인터페이스**

| 인터페이스 | 속도 | 용도 |
|:---|:---:|:---|
| **V.35** | 2Mbps | 구형 |
| **RS-449** | 2Mbps | 구형 |
| **G.703** | 2Mbps | E1 |

**📢 섹션 요약 비유:** DSU/CSU는 레거시!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: T1 전용선** → T1 DSU/CSU
**시나리오 2: E1 전용선** → E1 DSU/CSU
**시나리오 3: 신규** → 광섬유/Metro Ethernet

**도입 체크리스트:** 회선 타입, 속도, 인터페이스

**📢 섹션 요약 비유:** DSU/CSU는 전용선 시대!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 전용선 안정 연결

**미래 전망:** 광섬유/Metro Ethernet으로 대체

**참고 표준:** ANSI T1.107, ITU-T G.703

**📢 섹션 요약 비유:** DSU/CSU는 통신의 유산!
