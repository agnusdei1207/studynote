# SRE (Site Reliability Engineering) - 사이트 신뢰성 엔지니어링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 신뢰성을 엔지니어링
> 2. **가치**: 안정성과 속도의 균형
> 3. **융합:** DevOps, SLA/SLO, Error Budget

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SRE는 소프트웨어 엔지니어링으로 신뢰성을 확보합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SRE (Site Reliability Engineering)                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SRE 핵심 원칙                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. SLO 설정:                                         │   │ │
│   │   │     - 목표: 99.9% 가용성                              │   │ │
│   │   │     - 측정 가능해야 함                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Error Budget:                                     │   │ │
│   │   │     - 예산 내: 새 기능 개발                           │   │ │
│   │   │     - 예산 초과: 안정성 개선                          │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Toil 감소:                                        │   │ │
│   │   │     - 반복 작업 자동화                                │   │ │
│   │   │     - 50% 이상 엔지니어링                             │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 장애 대응:                                        │   │ │
│   │   │     - Blameless Postmortem                           │   │ │
│   │   │     - 시스템 개선                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  5. Automation:                                       │   │ │
│   │   │     - 자동화 우선                                     │   │ │
│   │   │     - 코드로 인프라 관리                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SRE는 운영을 엔지니어링 문제로 접근합니다.

**📢 섹션 요약 비유:** SRE = 안전 관리자!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SLO** | 서비스 수준 목표 |
| **Error Budget** | 에러 예산 |
| **Toil** | 반복 업무 |
| **Automation** | 자동화 |
| **On-call** | 당직 |

**핵심 알고리즸:** 측정 → 목표 설정 → 예산 관리 → 개선

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SRE vs DevOps**

| **메트릭** | SLO | 배포 빈도 |
