# Private IP - 사설 IP

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 내부 네트워크용 주소
> 2. **가치**: 공인 IP 절약
> 3. **융합:** NAT, RFC 1918, Reserved

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Private IP는 내부 네트워크에서만 사용하는 IP 주소입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Private IP                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              사설 IP 범위                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RFC 1918 사설 IP 범위:                                  │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Class A: 10.0.0.0/8                            ││   │ │
│   │   │  │    10.0.0.0 - 10.255.255.255                     ││   │ │
│   │   │  │    약 1,600만 개 (대규모)                          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Class B: 172.16.0.0/12                         ││   │ │
│   │   │  │    172.16.0.0 - 172.31.255.255                   ││   │ │
│   │   │  │    약 100만 개 (중규모)                            ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Class C: 192.168.0.0/16                        ││   │ │
│   │   │  │    192.168.0.0 - 192.168.255.255                 ││   │ │
│   │   │  │    약 65,000개 (소규모/가정)                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                  │   │ │
│   │   │  - 인터넷에서 라우팅 안 됨                               │   │ │
│   │   │  - 중복 사용 가능                                       │   │ │
│   │   │  - NAT으로 인터넷 접속                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 사설 IP는 IPv4 주소 고갈의 해결책입니다.

**📢 섹션 요약 비유:** Private IP = 내선 전화번호!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Range** | 범위 |
| **Non-routable** | 비라우팅 |
| **NAT** | 주소 변환 |
| **Reserved** | 예약됨 |
| **Local** | 로컬 |

**핵심 알고리듬:** 내부 사용 → NAT 변환 → 인터넷

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**공인 IP vs 사설 IP**

| 항목 | 공인 IP | 사설 IP |
|:---|:---|:---|
| **라우팅** | 인터넷 | 내부만 |
| **유일성** | 전역 | 로컬 |
| **비용** | 유료 | 무료 |
| **관리** | ISP | 자체 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 3:** 클라우드 → 172.16-31.x.x
