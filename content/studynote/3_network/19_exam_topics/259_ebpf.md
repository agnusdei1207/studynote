# eBPF - 확장 버클리 패킷 필터

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 내 프로그래밍
> 2. **가치**: 안전, 확장성
> 3. **융합**: Cilium, 관찰 가능성과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

eBPF(extended Berkeley Packet Filter)는 커널 공간에서 안전하게 프로그램을 실행하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    eBPF                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              eBPF 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [사용자 공간]                                       │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ eBPF 프로그램 로드                           │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           커널 공간                          │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  Hook Points:                                │    │   │ │
│   │   │  │  • kprobe (커널 함수 진입)                  │    │   │ │
│   │   │  │  • tracepoint (커널 이벤트)                 │    │   │ │
│   │   │  │  • XDP (네트워크 수신)                      │    │   │ │
│   │   │  │  • tc (트래픽 제어)                         │    │   │ │
│   │   │  │  • socket (소켓 이벤트)                     │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌───────────────────────────────────────┐  │    │   │ │
│   │   │  │  │           eBPF VM                    │  │    │   │ │
│   │   │  │  │  (안전한 실행, 검증된 프로그램)       │  │    │   │ │
│   │   │  │  └───────────────────────────────────────┘  │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              eBPF 용도                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 네트워킹:                                         │   │ │
│   │   │     • XDP, tc, Cilium                               │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 관찰 가능성:                                      │   │ │
│   │   │     • 추적, 모니터링                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 보안:                                             │   │ │
│   │   │     • 감사, 필터링                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 디버깅:                                           │   │ │
│   │   │     • 커널 분석                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: eBPF는 커널을 수정하지 않고 기능을 확장합니다.

**💡 비유**: eBPF는 커널의 플러그인 시스템입니다.

**📢 섹션 요약 비유:** eBPF = 커널 프로그래밍!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Hook** | 부착 지점 |
| **Map** | 데이터 저장 |
| **Helper** | 커널 함수 |
| **Verifier** | 안전 검증 |

**비교표: Hook 타입**

```
+------------+----------+----------+
| Hook       │ 위치     │ 용도     │
+------------+----------+----------+
| XDP        │ NIC 드라이버│ DDoS   │
| tc         │ qdisc    │ 라우팅  │
| kprobe     │ 커널 함수 │ 추적    │
| tracepoint │ 이벤트   │ 모니터링│
+------------+----------+----------+
```

**핵심 알고리즸:** JIT, 검증

**📢 섹션 요약 비유:** XDP = 최고 속도!
