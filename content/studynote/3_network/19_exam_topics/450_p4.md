# P4 (프로그래밍 가능 데이터 플레인) - Programming Protocol-independent Packet Processors

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 플레인 프로그래밍 언어
> 2. **가치**: 프로토콜 독립, 유연성
> 3. **융합**: Tofino, SmartNIC, P4 Runtime

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

P4는 스위치 데이터 플레인을 프로그래밍하는 언어입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    P4 아키텍처                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              P4 프로그래밍 모델                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ P4 Program                                   │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  parser → match-action → deparser           │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  1. Parser: 패킷 헤더 추출                   │    │   │ │
│   │   │  │  2. Match-Action: 처리                      │    │   │ │
│   │   │  │  3. Deparser: 패킷 재구성                    │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                     │                              │   │ │
│   │   │                     ▼ P4 Runtime                   │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Target (P4 호환 장비)                       │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  - Tofino ASIC                             │    │   │ │
│   │   │  │  - SmartNIC                                │    │   │ │
│   │   │  │  - BMv2 (소프트웨어 스위치)                  │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              P4 vs OpenFlow                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ OpenFlow │ P4       │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 유연성     │ 제한     │ 무제한   │               │   │ │
│   │   │  │ 프로토콜   │ 고정     │ 프로그래밍│              │   │ │
│   │   │  | 타겟      │ 스위치   │ 다양     │               │   │ │
│   │   │  │ 추상화     │ 낮음     │ 높음     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: P4는 데이터 플레인의 동작을 직접 프로그래밍합니다.

**💡 비유**: P4는 스위치의 펌웨어를 직접 작성합니다.

**📢 섹션 요약 비유:** P4 = 프로그래밍 가능 데이터 플레인!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Parser** | 패킷 파싱 |
| **Match-Action** | 매칭 및 액션 |
| **Deparser** | 패킷 직렬화 |
| **P4 Runtime** | 제어 플레인 인터페이스 |

**비교표: P4 타겟**

```
+------------+----------+----------+----------+
| 타겟       │ 유형     │ 속도     │ 용도     │
+------------+----------+----------+----------+
| Tofino     │ ASIC     │ Tbps     │ 프로덕션 │
| SmartNIC   │ FPGA     │ Gbps     │ 서버     │
| BMv2       │ 소프트웨어│ Mbps     │ 개발     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 상태 머신, 파이프라인

**📢 섹션 요약 비유:** Parser = 헤더 추출!
