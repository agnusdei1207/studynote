# DHCP (Dynamic Host Configuration Protocol) - 자동 IP 할당

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 주소 자동 할당
> 2. **가치**: 관리 편의성, 중앙 제어
> 3. **융합**: DNS, PXE, NTP와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DHCP(Dynamic Host Configuration Protocol)는 네트워크 장비에 IP 주소와 네트워크 설정을 자동으로 할당하는 프로토콜입니다.

> **해설**: DHCP는 DORA 과정을 통해 IP 주소를 자동 할당합니다.

**💡 비유**: DHCP는 호텔 체크인입니다. 방 번호(IP)를 자동으로 배정받습니다.

**📢 섹션 요약 비유:** DORA = DHCP 핵심!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 | 기본값 |
|:---|:---|:---|
| **Port** | UDP | 67 (서버), 68 (클라) |
| **Lease** | 임대 시간 | 86400초 (1일) |
| **Scope** | 할당 범위 | 풀 |
| **Reservation** | 고정 할당 | MAC 기반 |

**비교표: 할당 방식**

```
+------------+----------+----------+
| 방식       │ 특징     │ 용도     │
+------------+----------+----------+
| Dynamic    │ 자동     │ 일반     │
| Static     │ 예약     │ 서버     │
| Automatic  │ 영구     │ 특수     │
+------------+----------+----------+
```

**핵심 알고리즸:** DORA, Lease 갱신

**📢 섹션 요약 비유:** Lease 50% = 갱신 시작!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: DHCP vs SLAAC**

| 항목 | DHCP | SLAAC |
|:---|:---|:---|
| **IPv** | v4/v6 | v6 |
| **상태** | 유상태 | 무상태 |
| **관리** | 중앙 | 분산 |

**📢 섹션 요약 비유:** IPv4 = DHCP, IPv6 = SLAAC!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 기업** → DHCP + Reservation
**시나리오 2: 게스트** → DHCP + 짧은 Lease
**시나리오 3: PXE** → DHCP Option 66/67

**도입 체크리스트:** Scope, Exclusion, Lease, Option

**📢 섹션 요약 비유:** 서버는 Reservation!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 관리 편의성, IP 효율

**미래 전망:** DHCPv6, IPv6 RA

**참고 표준:** RFC 2131 (DHCPv4), RFC 8415 (DHCPv6)

**📢 섹션 요약 비유:** DHCP는 네트워크 자동화의 시작!
