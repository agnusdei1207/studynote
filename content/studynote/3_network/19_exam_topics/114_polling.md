# 폴링 / 선택 (Polling / Selection)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 중앙 제어기가 노드 순차 호출
> 2. **가치**: 충돌 없음, 공정한 접근
> 3. **융합**: 블루투스, 산업용 네트워크와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

폴링(Polling)은 중앙 제어기가 각 노드를 순차적으로 호출하여 전송 기회를 부여하고, 선택(Selection)은 노드가 중앙에 전송 의사를 알리는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    폴링 / 선택 (Polling / Selection)               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              폴링 (Polling)                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  구조:                                                  │   │ │
│   │   │                                                      │   │ │
│   │   │                    중앙 제어기                         │   │ │
│   │   │                       │                                │   │ │
│   │   │         ┌─────────────┼─────────────┐                 │   │ │
│   │   │         ↓             ↓             ↓                 │   │ │
│   │   │       노드1         노드2         노드3                │   │ │
│   │   │                                                      │   │ │
│   │   │  동작 순서:                                               │   │ │
│   │   │  1. 제어기 → 노드1: Poll (전송 있나?)                    │   │ │
│   │   │  2. 노드1 → 제어기: Data 또는 No Data                    │   │ │
│   │   │  3. 제어기 → 노드2: Poll                                 │   │ │
│   │   │  4. 노드2 → 제어기: Data 또는 No Data                    │   │ │
│   │   │  5. ... 반복                                             │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                   │   │ │
│   │   │  • 충돌 없음                                             │   │ │
│   │   │  • 공정한 접근                                           │   │ │
│   │   │  • 우선순위 가능                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  단점:                                                   │   │ │
│   │   │  • 폴링 오버헤드                                         │   │ │
│   │   │  • 제어기 의존                                           │   │ │
│   │   │  • 유휴 노드도 폴링                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              선택 (Selection / Probe)                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  구조:                                                  │   │ │
│   │   │  노드가 중앙에 전송 의사를 먼저 알림                      │   │ │
│   │   │                                                      │   │ │
│   │   │  동작 순서:                                               │   │ │
│   │   │  1. 노드 → 제어기: Probe (전송하고 싶어요)                │   │ │
│   │   │  2. 제어기: 전송 승인                                    │   │ │
│   │   │  3. 노드 → 제어기: Data                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  vs 폴링:                                                 │   │ │
│   │   │  폴링: 제어기 → 노드 (풀)                                 │   │ │
│   │   │  선택: 노드 → 제어기 (푸시)                               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 폴링은 블루투스와 산업용 네트워크에서 사용됩니다.

**💡 비유**: 폴링은 선생님이 학생을 차례로 지명하는 것입니다.

**📢 섹션 요약 비유**: 폴링 = 중앙 제어 순차 호출!

### Ⅱ. 아키티철 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **제어기** | 중앙 관리 | 마스터 | 선생님 |
| **Poll** | 호출 | 질의 | 지명 |
| **Response** | 응답 | 데이터/없음 | 대답 |

**비교표**

```
+------------+----------+----------+
| 항목       | 폴링     | 선택     |
+------------+----------+----------+
| 주도권     | 제어기   | 노드     |
| 방식       | Pull     | Push     |
| 오버헤드   | 높음     | 중간     |
| 응답성     | 낮음     | 높음     |
+------------+----------+----------+
```

**핵심 알고리즸**: Poll → Response → Poll → Response (반복)

**📢 섹션 요약 비유**: 블루투스는 마스터-슬레이브 폴링!
