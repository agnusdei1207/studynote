# Handshake - 핸드셰이크

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연결 협상
> 2. **가치**: 매개변수 교환
> 3. **융합:** TCP 3-way, TLS, Signal Protocol

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Handshake는 통신 전 매개변수를 협상합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Handshake                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              핸드셰이크 유형                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  TCP 3-Way Handshake:                                 │   │ │
│   │   │  Client                    Server                    │   │ │
│   │   │  CLOSED                   LISTEN                     │   │ │
│   │   │  ── SYN ─────────────────→                           │   │ │
│   │   │  SYN_SENT                SYN_RCVD                    │   │ │
│   │   │  ←── SYN+ACK ───────────                            │   │ │
│   │   │  ── ACK ─────────────────→                           │   │ │
│   │   │  ESTABLISHED             ESTABLISHED                 │   │ │
│   │   │                                                      │   │ │
│   │   │  TLS Handshake:                                         │   │ │
│   │   │  ── ClientHello ──────────→                           │   │ │
│   │   │  ←── ServerHello + Certificate ───                   │   │ │
│   │   │  ── KeyExchange ──────────→                           │   │ │
│   │   │  ←── Finished ───────────                            │   │ │
│   │   │  ── Finished ────────────→                           │   │ │
│   │   │                                                      │   │ │
│   │   │  협상 내용:                                            │   │ │
│   │   │  - 시퀀스 번호                                        │   │ │
│   │   │  - 윈도우 크기                                        │   │ │
│   │   │  - MSS                                                │   │ │
│   │   │  - 암호화 매개변수 (TLS)                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 핸드셰이크는 양측이 통신 준비를 확인합니다.

**📢 섹션 요약 비유:** Handshake = 악수!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SYN** | 동기화 |
| **ACK** | 확인 |
| **FIN** | 종료 |
| **Hello** | TLS 시작 |
| **Finished** | 완료 |

**핵심 알고리즸:** 요청 → 응답 → 확인 → 연결

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**핸드셰이크 비교**

| 유형 | 단계 | 용도 |
|:---|:---|:---|
| **TCP 3-way** | 3 | 연결 설정 |

| **TLS 1.3** | 1-2 | 보안 설정 (0-RTT) |
