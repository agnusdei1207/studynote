# 등화 (Equalization) - 선형/비선형/적응형

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 채널 왜곡 보상
> 2. **가치**: ISI 제거, 신호 복원
> 3. **융합**: 수신기, 모뎀과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

등화(Equalization)는 채널의 주파수 특성을 보상하여 ISI(심볼간 간섭)를 제거하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    등화 (Equalization)                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              등화기 구조                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [송신] ──► [채널] ──► [등화기] ──► [수신]            │   │ │
│   │   │              H(z)       1/H(z)                       │   │ │
│   │   │                                                      │   │ │
│   │   │  원리:                                                 │   │ │
│   │   │  • 채널의 역특성 적용                                  │   │ │
│   │   │  • H(z) × 1/H(z) = 1 (이상적)                         │   │ │
│   │   │  • ISI 제거                                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              등화기 종류                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 선형 등화기 (Linear Equalizer):                     │   │ │
│   │   │     • ZFE (Zero-Forcing Equalizer)                    │   │ │
│   │   │       - ISI 완전 제거 목표                             │   │ │
│   │   │       - 잡음 증폭 문제                                 │   │ │
│   │   │     • MMSE (Minimum Mean Square Error)                │   │ │
│   │   │       - ISI와 잡음 균형                                │   │ │
│   │   │       - 성능 우수                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 비선형 등화기 (Non-linear Equalizer):               │   │ │
│   │   │     • DFE (Decision Feedback Equalizer)               │   │ │
│   │   │       - 피드포워드 + 피드백 필터                       │   │ │
│   │   │       - 잡음 증폭 없음                                  │   │ │
│   │   │     • MLSE (Maximum Likelihood Sequence Estimator)   │   │ │
│   │   │       - 비터비 알고리즘                                 │   │ │
│   │   │       - 최적 성능, 복잡도 높음                          │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 적응형 등화기 (Adaptive Equalizer):                 │   │ │
│   │   │     • LMS (Least Mean Square)                         │   │ │
│   │   │     • RLS (Recursive Least Square)                    │   │ │
│   │   │     • 실시간 채널 추적                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 등화는 주파수 선택적 페이딩 환경에서 필수적인 기술입니다.

**💡 비유**: 등화는 안경을 쓰는 것과 같습니다. 왜곡된 시력을 교정합니다.

**📢 섹션 요약 비유:** DFE = 성능과 복잡도 균형!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 종류 | 원리 | 장점 | 단점 |
|:---|:---|:---|:---|
| **ZFE** | ISI=0 강제 | 단순 | 잡음 증폭 |
| **MMSE** | MSE 최소화 | 잡음 고려 | 복잡 |
| **DFE** | 결정 피드백 | 잡음 증폭 없음 | 오류 전파 |
| **MLSE** | 최적 추정 | 최고 성능 | 매우 복잡 |

**비교표**

```
+------------+----------+----------+
| 항목       │ 선형     │ 비선형   │
+------------+----------+----------+
| 잡음 증폭   │ 있음     │ 없음     │
│ 성능       │ 낮음     │ 높음     │
│ 복잡도     │ 낮음     │ 높음     │
│ 오류 전파   │ 없음     │ 있음     │
+------------+----------+----------+
```

**핵심 알고리즸:** LMS, RLS, Viterbi

**📢 섹션 요약 비유:** MMSE = 실용적 선택!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 적응 알고리즘**

| 알고리즘 | 수렴 속도 | 복잡도 | 용도 |
|:---|:---:|:---:|:---|
| **LMS** | 느림 | 낮음 | 일반 |
| **RLS** | 빠름 | 높음 | 빠른 변화 |
| **CMA** | - | 중간 | 블라인드 |

**📢 섹션 요약 비유:** LMS = 단순, RLS = 빠름!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: GSM** → MLSE (Viterbi)
**시나리오 2: WiFi** → DFE + MMSE
**시나리오 3: 고속 이동** → RLS 적응 등화

**도입 체크리스트:** 채널 변화 속도, 지연 확산, 복잡도

**📢 섹션 요약 비유:** OFDM = 등화 단순화!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: ISI 제거, 전송 속도 향상

**미래 전망:** OFDM으로 단순화 (1-tap 등화)

**참고 표준:** 3GPP, IEEE 802.11

**📢 섹션 요약 비유:** 등화는 채널의 보정!
