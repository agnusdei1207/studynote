# SMS (단문 메시지 서비스) - Short Message Service

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 160자 텍스트 메시지
> 2. **가치**: 범용성, 높은 도달률
> 3. **융합**: 2FA, 마케팅, 알림

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SMS는 휴대전화로 짧은 메시지를 전송합니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SMSC** | SMS Center |
| **MS** | Mobile Station |
| **HLR** | Home Location Register |
| **MAP** | Mobile Application Part |

**비교표: SMS vs MMS vs RCS**

```
+------------+----------+----------+----------+
| 항목       │ SMS      │ MMS      │ RCS      │
+------------+----------+----------+----------+
| 길이       │ 160자    │ 300KB    │ 무제한   │
| 미디어     │ 텍스트   │ 이미지   │ 모든것   │
| 비용       │ 낮음     │ 중간     │ 데이터   │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 저장-전달, 라우팅

**📢 섹션 요약 비유:** SMSC = 우체국!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 메시징 프로토콜**

| 프로토콜 | 네트워크 | 특징 |
|:---|:---|:---|
| **SMS** | SS7 | 신호망 |
| **SMS over IP** | LTE/IMS | SIP MESSAGE |

**📢 섹션 요약 비유:** SMS over IP = IMS 기반!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 2FA → SMS OTP
**시나리오 2:** 마케팅 → 대량 SMS
**시나리오 3:** IoT → SMS 알림

**도입 체크리스트:** 도달률, 비용, 보안

**📢 섹션 요약 비유:** 98% 도달률!
