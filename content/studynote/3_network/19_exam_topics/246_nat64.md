# NAT64 - IPv6/IPv4 변환

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPv6 클라이언트가 IPv4 서버에 접속
> 2. **가치**: IPv4 리소스 공유
> 3. **융합**: DNS64와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NAT64는 IPv6 전용 네트워크에서 IPv4 서버에 접속할 수 있게 하는 변환 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NAT64                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NAT64 구조                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [IPv6 클라이언트]      [NAT64]      [IPv4 서버]      │   │ │
│   │   │                                                      │   │ │
│   │   │  2001:db8::1         변환          192.0.2.1         │   │ │
│   │   │        │               │               │              │   │ │
│   │   │        │ IPv6 패킷     │               │              │   │ │
│   │   │        │ ─────────────>│               │              │   │ │
│   │   │        │               │ IPv4 패킷     │              │   │ │
│   │   │        │               │ ──────────────>│              │   │ │
│   │   │        │               │               │              │   │ │
│   │   │        │               │ IPv4 응답     │              │   │ │
│   │   │        │               │ <─────────────│              │   │ │
│   │   │        │ IPv6 응답     │               │              │   │ │
│   │   │        │ <────────────│               │              │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  • IPv6 → IPv4 단방향                                │   │ │
│   │   │  • IPv4 주소 절약                                    │   │ │
│   │   │  • DNS64 필요                                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │

│   │              NAT64 vs NAT44                                   │ │

│   │   ┌─────────────────────────────────────────────────────┐   │ │

│   │   │  항목     │ NAT44           │ NAT64                 │   │ │
│   │   │  ────────┼─────────────────┼─────────────────────│   │ │
│   │   │  입력     │ IPv4            │ IPv6                  │   │ │
│   │   │  출력     │ IPv4            │ IPv4                  │   │ │
│   │   │  목적     │ 주소 절약       │ IPv4 접속             │   │ │
│   │   │  프로토콜 │ RFC 3022        │ RFC 6146              │   │ │
│   │   │                                                      │   │ │
```

> **해설**: NAT64는 IPv6 네트워크가 IPv4 인터넷에 접속할 수 있게 합니다.

**💡 비유**: NAT64는 통역사입니다. 두 언어를 사용하는 사람들을 연결합니다.

**📢 섹션 요약 비유:** NAT64 = IPv6 → IPv4!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **DNS64** | AAAA 합성 |
| **Prefix** | 64:ff9b::/96 |
| **Stateful** | 상태 유지 |
| **Stateless** | 상태 없음 |

**비교표: NAT64 유형**

```
+------------+----------+----------+
| 유형       │ 상태     │ 특징     │
+------------+----------+----------+
| Stateless  │ 없음     │ 주소 매핑│
| Stateful   │ 유지     │ NAT 테이블│
+------------+----------+----------+
```

**핵심 알고리즸:** 주소 변환, 포트 매핑

**📢 섹션 요약 비유:** DNS64 = AAAA 합성!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 464XLAT**

| 항목 | NAT64 | 464XLAT |
|:---|:---|:---|
| **방향** | v6→v4 | v4→v6→v4 |
| **클라이언트** | IPv6 | IPv4 |
| **복잡도** | 낮음 | 높음 |

**📢 섹션 요약 비유:** 464XLAT = IPv4 앱 지원!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: IPv6 Only ISP** → NAT64
**시나리오 2: 모바일** → 464XLAT
**시나리오 3: 기업** → Dual Stack

**도입 체크리스트:** DNS64, 클라이언트 호환성

**📢 섹션 요약 비유:** DNS64 = 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: IPv4 접속, 주소 절약

**미래 전망:** IPv6 Only 네트워크 확대

**참고 표준:** RFC 6146 (NAT64), RFC 6052 (주소 매핑)

**📢 섹션 요약 비유:** NAT64는 IPv6 전환의 핵심!
