# Kerberos - 네트워크 인증 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 티켓 기반 인증 프로토콜
> 2. **가치**: SSO, 강력한 보안
> 3. **융합**: Active Directory, Windows

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Kerberos는 티켓을 사용하여 안전한 인증을 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Kerberos                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Kerberos 흐름                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 1. TGT 요청                                    │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [KDC/AS] ───▶ TGT 발급                              │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 2. 서비스 티켓 요청 (TGT 포함)                  │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [KDC/TGS] ───▶ 서비스 티켓 발급                      │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 3. 서비스 접근 (서비스 티켓)                    │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [Server] ───▶ 인증 완료                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Kerberos 구성요소                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 구성요소   │ 역할     │ 설명     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ KDC        │ 키 배포  │ 중앙서버 │               │   │ │
│   │   │  │ AS         │ 인증     │ 초기인증 │               │   │ │
│   │   │  | TGS        │ 티켓발급 │ 서비스   │               │   │ │
│   │   │  │ TGT        │ 티켓     │ 인증티켓 │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Kerberos는 비밀번호를 네트워크로 전송하지 않습니다.

**💡 비유**: Kerberos는 입장권입니다. 티켓으로 입장합니다.

**📢 섹션 요약 비유:** Kerberos = 티켓 인증!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Realm** | Kerberos 도메인 |
| **Principal** | 사용자/서비스 식별자 |
| **Ticket** | 인증 티켓 |
| **Session Key** | 세션 암호화 키 |

**비교표: Kerberos vs NTLM**

```
+------------+----------+----------+
| 항목       │ Kerberos │ NTLM     │
+------------+----------+----------+
| 보안       │ 높음     │ 낮음     │
| SSO        │ 지원     │ 제한     │
| 티켓       │ 있음     │ 없음     │
| 최신성     │ 현대적   │ 레거시   │
+------------+----------+----------+
```

**핵심 알고리즸:** 대칭키 암호화, 티켓 검증

**📢 섹션 요약 비유:** AD = Kerberos!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 인증 프로토콜**

| 항목 | Kerberos | SAML | OIDC |
|:---|:---|:---|:---|
| **형식** │ 티켓 │ XML │ JSON |
| **플랫폼** │ Windows │ 웹 │ 웹/모바일 |
| **SSO** │ 지원 │ 지원 │ 지원 |

**📢 섹션 요약 비유:** Windows = Kerberos!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** Windows 도메인 → Kerberos
**시나리오 2:** 기업 내부 SSO → Kerberos
**시나리오 3:** 웹 애플리케이션 → SAML/OIDC

**도입 체크리스트:** KDC 구성, 시간 동기화, Realm 설정

**📢 섹션 요약 비유:** 기업 내부 = Kerberos!
