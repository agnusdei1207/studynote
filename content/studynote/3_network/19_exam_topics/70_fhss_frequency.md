# FHSS (주파수 도약 확산 스펙트럼)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: carrier 주파수를 빠르게 변경하며 전송
> 2. **가치**: 간섭 회피, 주파수 다이버시티
> 3. **융합**: 블루투스, WiFi, 군사 통신과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

FHSS(Frequency Hopping Spread Spectrum)는 carrier 주파수를 의사난수 패턴에 따라 빠르게 변경하며 전송하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FHSS (주파수 도약 확산 스펙트럼)                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FHSS 원리                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  주파수 도약:                                             │   │ │
│   │   │                                                      │   │ │
│   │   │  f1 ──┐                                              │   │ │
│   │   │       │  ┌── f2                                      │   │ │
│   │   │       │  │     ┌── f3                                │   │ │
│   │   │       │  │     │     ┌── f1                           │   │ │
│   │   │       └──┴─────┴─────┴──→ 시간                        │   │ │
│   │   │                                                      │   │ │
│   │   │  도약 패턴: f1 → f2 → f3 → f1 → f5 → f2 → ...        │   │ │
│   │   │  (PN 시퀀스로 결정)                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  시간-주파수 그래프:                                    │   │ │
│   │   │  f │     ●           ●                                │   │ │
│   │   │    │   ●   ●       ●   ●                              │   │ │
│   │   │    │ ●       ●   ●     ●                              │   │ │
│   │   │    │●         ●-●       ●                             │   │ │
│   │   │    └──────────────────────→ t                         │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FHSS 특징                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  장점:                                                 │   │ │
│   │   │  • 협대역 간섭 회피 (다른 채널로 도약)                   │   │ │
│   │   │  • 다중 경로 내성                                       │   │ │
│   │   │  • 다중 접속 (다른 도약 패턴)                            │   │ │
│   │   │  • 저비용 구현                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  단점:                                                 │   │ │
│   │   │  • 동기화 필요                                          │   │ │
│   │   │  • 도약 시간 오버헤드                                    │   │ │
│   │   │  • 처리 이득 제한                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  도약 속도:                                              │   │ │
│   │   │  • Slow Hopping: 여러 비트 per hop                      │   │ │
│   │   │  • Fast Hopping: 여러 hop per 비트                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: FHSS는 블루투스의 핵심 기술이며 WiFi 초기 버전에도 사용되었습니다.

**💡 비유**: FHSS는 라디오 채널을 계속 바꾸며 방송하는 것입니다.

**📢 섹션 요약 비유**: 블루투스 = 79채널 FHSS!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **도약 패턴** | 주파수 순서 | PN 생성 | 암호 |
| **채널 수** | N | 79 (블루투스) | 개수 |
| **도약율** | hop/초 | 1600 (블루투스) | 속도 |
| **동기화** | 정렬 | 클럭 | 타이밍 |

**비교표**

```
+------------+----------+----------+
| 항목       | 블루투스 | WiFi FHSS|
+------------+----------+----------+
| 채널 수    | 79       | 75       |
| 대역폭     | 1MHz/ch  | 1MHz/ch  |
| 도약율     | 1600hop/s| 2.5hop/s |
| 대역       | 2.4GHz   | 2.4GHz   |
+------------+----------+----------+
```

**핵심 알고리즸**: 데이터 → PN 기반 채널 선택 → 변조 → 도약 → 수신 동기 → 복원

**📢 섹션 요약 비유**: Slow hop = 1hop에 여러 비트!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: DSSS vs FHSS**

| 항목 | DSSS | FHSS |
|:---|:---:|:---:|
| 확산 방식 | 곱셈 | 도약 |
| 대역폭 | 연속 | 이산 |
| 간섭 내성 | 좋음 | 좋음 |
| 구현 | 복잡 | 단순 |

**📢 섹션 요약 비유**: FHSS는 채널 도약!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 블루투스** → FHSS (79채널)
**시나리오 2: WiFi 802.11** → FHSS (초기)
**시나리오 3: 군사** → FHSS (저탐지)

**도입 체크리스트**: 채널 수, 도약율, 동기화

**📢 섹션 요약 비유**: 블루투스는 초당 1600회 도약!
