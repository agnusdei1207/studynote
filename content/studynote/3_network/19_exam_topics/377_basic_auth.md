# Basic Auth - 기본 인증

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자명/비밀번호 Base64 인코딩
> 2. **가치**: 구현 단순, 광범위 지원
> 3. **융합**: HTTPS, API 인증

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Basic Auth는 사용자명과 비밀번호를 Base64로 인코딩합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Basic Auth                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Basic Auth 흐름                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ username:password                              │   │ │
│   │   │     │     │                                          │   │ │
│   │   │     │     ▼ Base64 인코딩                            │   │ │
│   │   │     │ Authorization: Basic dXNlcjpwYXNz              │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [Server] ──────▶ [Base64 디코딩]                    │   │ │
│   │   │                          │                            │   │ │
│   │   │                          │ 검증                       │   │ │
│   │   │                          ▼                            │   │ │
│   │   │                      [인증 완료]                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Basic Auth 형식                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Authorization: Basic <base64(username:password)>   │   │ │
│   │   │                                                      │   │ │
│   │   │  예시:                                               │   │ │
│   │   │  username: admin                                     │   │ │
│   │   │  password: secret123                                 │   │ │
│   │   │  → Base64(admin:secret123) = YWRtaW46c2VjcmV0MTIz  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Basic Auth는 간단하지만 HTTPS 없이는 안전하지 않습니다.

**💡 비유**: Basic Auth는 ID카드입니다. 이름과 사진을 보여줍니다.

**📢 섹션 요약 비유:** Basic Auth = 간단!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Username** | 사용자명 |
| **Password** | 비밀번호 |
| **Base64** | 인코딩 |
| **Header** | Authorization |

**비교표: Basic Auth vs Bearer Token**

```
+------------+----------+----------+
| 항목       │ Basic    │ Bearer   │
+------------+----------+----------+
| 형식       │ Base64   │ 토큰     │
| 만료       │ 없음     │ 있음     │
| 세션       │ 없음     │ 있음     │
| 보안       │ 낮음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** Base64 인코딩, 자격증명 검증

**📢 섹션 요약 비유:** HTTPS 필수!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: HTTP 인증**

| 항목 | Basic | Digest | Bearer |
|:---|:---|:---|:---|
| **보안** │ 낮음 │ 중간 │ 높음 |
| **복잡도** │ 낮음 │ 중간 │ 중간 |
| **HTTPS** │ 필수 │ 선택 │ 필수 |

**📢 섹션 요약 비유:** 레거시 = Basic!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 내부 API → Basic Auth + HTTPS
**시나리오 2:** 테스트 환경 → Basic Auth
**시나리오 3:** 프로덕션 → OAuth/JWT

**도입 체크리스트:** HTTPS, 자격증명 저장, 로테이션

**📢 섹션 요약 비유:** 테스트 = Basic!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 간단한 인증

**미래 전망:** 레거시 시스템 유지

**참고 표준:** RFC 7617

**📢 섹션 요약 비유:** Basic Auth는 레거시!
