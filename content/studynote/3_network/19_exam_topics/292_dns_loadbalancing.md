# DNS 로드밸런싱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: DNS로 트래픽 분산

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DNS 로드밸런싱은 DNS 응답으로 여러 서버 IP를 반환하여 트래픽을 분산합니다.

> **해설**: DNS 로드밸런싱은 간단하지만 세밀한 제어가 어렵습니다.

**💡 비유**: DNS 로드밸런싱은 전화번호부에서 번호를 돌려가며 알려줍니다.

**📢 섹션 요약 비유:** DNS LB = 간단한 분산!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **A Record** | IPv4 |
| **AAAA** | IPv6 |
| **TTL** | 캐시 시간 |
| **Health Check** | 상태 확인 |

**비교표: DNS LB vs L4 LB**

```
+------------+----------+----------+
| 항목       │ DNS LB   │ L4 LB    │
+------------+----------+----------+
| 계층       │ L7 (DNS) │ L4       │
| 세밀도     │ 낮음     │ 높음     │
| 장애 감지  │ 느림     │ 빠름     │
| 비용       │ 낮음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** Round Robin, 가중치

**📢 섹션 요약 비유:** TTL = 변경 반영 속도!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 글로벌 분산** → Geo DNS
**시나리오 2: 간단한 분산** → Round Robin
**시나리오 3: 장애 조치** → DNS Failover

**도입 체크리스트:** TTL, Health Check, 캐시

**📢 섹션 요약 비유:** 낮은 TTL = 빠른 전환!
