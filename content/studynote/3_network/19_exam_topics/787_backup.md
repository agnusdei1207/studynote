# Backup - 백업

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 복사본 보관
> 2. **가치**: 데이터 손실 방지
> 3. **융합:** Disaster Recovery, RPO, Storage

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Backup은 데이터 복사본을 보관합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Backup                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              백업 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Full Backup:                                         │   │ │
│   │   │  모든 데이터 복사                                      │   │ │
│   │   │  장점: 완전한 복구 / 단점: 느림, 큼                      │   │ │
│   │   │                                                      │   │ │
│   │   │  Incremental Backup:                                  │   │ │
│   │   │  마지막 백업 이후 변경분만                               │   │ │
│   │   │  장점: 빠름, 작음 / 단점: 복구 복잡                      │   │ │
│   │   │                                                      │   │ │
│   │   │  Differential Backup:                                 │   │ │
│   │   │  마지막 Full 백업 이후 변경분                           │   │ │
│   │   │  장점: 복구 쉬움 / 단점: 점점 커짐                       │   │ │
│   │   │                                                      │   │ │
│   │   │  3-2-1 Rule:                                           │   │ │
│   │   │  - 3 copies                                           │   │ │
│   │   │  - 2 different media                                  │   │ │
│   │   │  - 1 offsite                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 백업은 3-2-1 규칙을 따르는 것이 좋습니다.

**📢 섹션 요약 비유:** Backup = 예비 저장!
