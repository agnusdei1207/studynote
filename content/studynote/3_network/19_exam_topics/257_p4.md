# P4 - 프로그래밍 가능한 패킷 처리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 평면 프로그래밍 언어
> 2. **가치**: 유연한 패킷 처리
> 3. **융합**: SDN, SmartNIC와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

P4(Programming Protocol-independent Packet Processors)는 데이터 평면의 패킷 처리 동작을 프로그래밍하는 언어입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    P4                                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              P4 특징                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Protocol-Independent:                            │   │ │
│   │   │     • 프로토콜에 독립적                               │   │ │
│   │   │     • 새로운 프로토콜 구현 가능                       │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Target-Independent:                               │   │ │
│   │   │     • 다양한 하드웨어 지원                            │   │ │
│   │   │     • ASIC, FPGA, SmartNIC                           │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Programmable:                                     │   │ │
│   │   │     • 파서, 매치-액션 파이프라인                      │   │ │
│   │   │     • 유연한 패킷 처리                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              P4 파이프라인                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Ingress]                                          │   │ │
│   │   │     │                                                │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  Parser → Match-Action → Match-Action               │   │ │
│   │   │     │                                                │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  [Egress]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  Deparser → Output                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: P4는 파서, 매치-액션 테이블, 디파서를 프로그래밍합니다.

**💡 비유**: P4는 네트워크의 어셈블리어입니다. 패킷 처리를 직접 코딩합니다.

**📢 섹션 요약 비유:** P4 = 데이터 평면 프로그래밍!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Parser** | 헤더 추출 |
| **Match** | 테이블 조회 |
| **Action** | 패킷 처리 |
| **Deparser** | 헤더 직렬화 |

**비교표: P4 vs OpenFlow**

```
+------------+----------+----------+
| 항목       │ OpenFlow │ P4       │
+------------+----------+----------+
| 유연성     │ 제한     │ 무제한   │
| 프로토콜   │ 고정     │ 프로그래밍│
| 대상       │ 스위치   │ 모든 장비│
+------------+----------+----------+
```

**핵심 알고리즸:** 파싱, 매치-액션

**📢 섹션 요약 비유:** Parser = 헤더 분해!
