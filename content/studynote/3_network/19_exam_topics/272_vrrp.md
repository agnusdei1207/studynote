# VRRP - 가상 라우터 이중화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 라우터 프로토콜
> 2. **가치**: 게이트웨이 이중화
> 3. **융합**: FHRP, Keepalived와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

VRRP(Virtual Router Redundancy Protocol)는 여러 라우터를 하나의 가상 라우터로 그룹화합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    VRRP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VRRP 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Host]                                             │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ 기본 게이트웨이: 10.0.0.1 (VIP)              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           VRRP 그룹 (VRID 1)                 │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌───────────┐        ┌───────────┐         │    │   │ │
│   │   │  │  │  Master   │        │  Backup   │         │    │   │ │
│   │   │  │  │ Router    │        │ Router    │         │    │   │ │
│   │   │  │  │ Priority  │        │ Priority  │         │    │   │ │
│   │   │  │  │   100     │        │    90     │         │    │   │ │
│   │   │  │  │10.0.0.2   │        │10.0.0.3   │         │    │   │ │
│   │   │  │  │           │◀──────▶│           │         │    │   │ │
│   │   │  │  │ VRRP Adv  │  VRRP  │           │         │    │   │ │
│   │   │  │  │ (224.0.0.18)     │           │         │    │   │ │
│   │   │  │  └───────────┘        └───────────┘         │    │   │ │
│   │   │  │       │                                      │    │   │ │
│   │   │  │       │ VIP: 10.0.0.1                       │    │   │ │
│   │   │  │       ↓                                      │    │   │ │
│   │   │  │  [네트워크]                                  │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VRRP 특징                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. VRID:                                            │   │ │
│   │   │     • 가상 라우터 ID (1-255)                         │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Priority:                                        │   │ │
│   │   │     • 0: Master 양도                                 │   │ │
│   │   │     • 1-254: 우선순위 (높을수록 Master)             │   │ │
│   │   │     • 255: IP 주소 소유자                            │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Advertisement:                                   │   │ │
│   │   │     • 멀티캐스트 224.0.0.18                          │   │ │
│   │   │     • 프로토콜 번호 112                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: VRRP는 Master 라우터가 VIP를 소유하고 백업이 대기합니다.

**💡 비유**: VRRP는 비행기의 조종사/부조종사 시스템입니다.

**📢 섹션 요약 비유:** VRRP = 게이트웨이 이중화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **VRID** | 가상 라우터 ID |
| **Master** | 활성 라우터 |
| **Backup** | 대기 라우터 |
| **Virtual MAC** | 00-00-5E-00-01-XX |

**비교표: VRRP 상태**

```
+------------+----------+----------+
| 상태       │ 동작     │ 전이     │
+------------+----------+----------+
| Initialize │ 시작     │ 부팅     │
| Master     │ 광고     │ 우선순위 │
| Backup     │ 수신     │ 타임아웃 │
+------------+----------+----------+
```

**핵심 알고리즸:** 우선순위, 광고

**📢 섹션 요약 비유:** Master Down = Backup 승격!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FHRP 프로토콜**

| 프로토콜 | 표준 | 특징 |
|:---|:---|:---|
| **VRRP** | RFC 5798 | 표준 |
| **HSRP** | 시코 전용 | Cisco |
| **GLBP** | 시스코 전용 | 로드밸런싱 |

**📢 섹션 요약 비유:** VRRP = 표준!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 게이트웨이 이중화** → VRRP
**시나리오 2: 멀티벤더** → VRRP (HSRP 대신)
**시나리오 3: 로드밸런싱** → GLBP 또는 VRRP 멀티그룹

**도입 체크리스트:** VRID, Priority, 인증

**📢 섹션 요약 비유:** Priority 100 = 기본값!

**정량/정성 기대효과**: 고가용성, 무중단

**미래 전망:** IPv6 VRRP 확대

**참고 표준:** RFC 5798

**📢 섹션 요약 비유:** VRRP는 게이트웨이 이중화 표준!
