# DNS (Domain Name System) - 도메인 이름 시스템

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이름-IP 주소 변환
> 2. **가치**: 사용자 친화적 주소
> 3. **융합:** UDP/TCP, Zone, Caching

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DNS는 도메인 이름을 IP 주소로 변환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DNS                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DNS 계층 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │           . (Root)                                   │   │ │
│   │   │          /     \                                     │   │ │
│   │   │       .com      .kr                                  │   │ │
│   │   │        |         |                                   │   │ │
│   │   │     google     naver                                 │   │ │
│   │   │        |         |                                   │   │ │
│   │   │      www        www                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  FQDN: www.google.com.                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DNS는 계층적 분산 데이터베이스입니다.

**📢 섹션 요약 비유:** DNS = 인터넷 전화번호부!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Resolver** | DNS 클라이언트 |
| **Recursive** | 전체 쿼리 수행 |
| **Authoritative** | 최종 답변 서버 |
| **Zone** | 관리 영역 |
| **TTL** | 캐시 시간 |

**핵심 알고리즸:** 계층적 쿼리 → 캐싱

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**DNS 레코드 타입**

| 타입 | 설명 |
|:---|:---|
| **A** | IPv4 주소 |
| **AAAA** | IPv6 주소 |
| **CNAME** | 별칭 |
| **MX** | 메일 서버 |
| **NS** | 네임서버 |
| **PTR** | 역방향 |
| **TXT** | 텍스트 |
