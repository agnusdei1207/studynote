# DiffServ (Differentiated Services) - 차등화 서비스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클래스 기반 QoS 모델
> 2. **가치**: 확장성, 대규모 네트워크 적용
> 3. **융합**: DSCP, PHB와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DiffServ(Differentiated Services)는 트래픽을 클래스별로 구분하여 차등화된 서비스를 제공하는 QoS 모델입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DiffServ (Differentiated Services)                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DiffServ 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Ingress]         [Core]           [Egress]        │   │ │
│   │   │                                                      │   │ │
│   │   │  분류/마킹         PHB              스케줄링         │   │ │
│   │   │     │                │                 │            │   │ │
│   │   │     ↓                ↓                 ↓            │   │ │
│   │   │  ┌─────┐         ┌─────┐          ┌─────┐          │   │ │
│   │   │  │DSCP │  ───>   │PHB  │  ───>    │큐잉 │          │   │ │
│   │   │  └─────┘         └─────┘          └─────┘          │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  • 스케일 가능 (확장성)                              │   │ │
│   │   │  • 코어에서 상태 유지 불필요                         │   │ │
│   │   │  • DSCP 필드 사용                                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PHB (Per-Hop Behavior)                            │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  PHB     │ DSCP │ 용도                               │   │ │
│   │   │  ────────┼──────┼────────────────────────────────│   │ │
│   │   │  EF      │ 46   │ Expedited Forwarding (VoIP)      │   │ │
│   │   │  AF11    │ 10   │ Assured Forwarding Class 1       │   │ │
│   │   │  AF21    │ 18   │ Assured Forwarding Class 2       │   │ │
│   │   │  AF31    │ 26   │ Assured Forwarding Class 3       │   │ │
│   │   │  AF41    │ 34   │ Assured Forwarding Class 4       │   │ │
│   │   │  BE      │ 0    │ Best Effort                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DiffServ는 DSCP 필드를 사용하여 트래픽을 분류합니다.

**💡 비유**: DiffServ는 항공 좌석 등급입니다. 비즈니스석이 이코노미석보다 우선합니다.

**📢 섹션 요약 비유:** DiffServ = 확장성!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **DSCP** | 6비트 (0-63) |
| **PHB** | 홉별 동작 |
| **SLA** | 서비스 수준 협약 |

**비교표: DiffServ vs IntServ**

```
+------------+----------+----------+
| 항목       │ DiffServ │ IntServ  │
+------------+----------+----------+
| 상태       │ 무상태   │ 상태유지 │
| 스케일     │ 높음     │ 낮음     │
| 복잡도     │ 낮음     │ 높음     │
| 예약       │ X        │ O (RSVP) │
+------------+----------+----------+
```

**핵심 알고리즸:** 분류, 마킹, PHB

**📢 섹션 요약 비유:** 코어 = 상태 없이 처리!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: AF 클래스**

| 클래스 | Drop 우선순위 | 용도 |
|:---:|:---|:---|
| **AF1** | Low/Med/High | Best Effort |
| **AF2** | Low/Med/High | 데이터 |
| **AF3** | Low/Med/High | 비디오 |
| **AF4** | Low/Med/High | 실시간 |

**📢 섹션 요약 비유:** EF = 최우선, AF = 보장!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: VoIP** → EF 마킹
**시나리오 2: 비디오** → AF41 마킹
**시나리오 3: 데이터** → AF21 마킹

**도입 체크리스트:** 클래스 정의, DSCP 매핑, PHB 설정

**📢 섹션 요약 비유:** 마킹 = 네트워크 경계!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 확장성, 서비스 차등화

**미래 전망:** SDN 기반 동적 QoS

**참고 표준:** RFC 2475, RFC 3246 (EF), RFC 2597 (AF)

**📢 섹션 요약 비유:** DiffServ는 대규모 QoS의 표준!
