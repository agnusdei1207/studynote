# 오버레이 네트워크 - Overlay Network

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 네트워크 위 가상 네트워크
> 2. **가치**: 유연성, 격리, 확장성
> 3. **융합**: VXLAN, GRE, GENEVE

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

오버레이 네트워크는 물리 네트워크 위에 논리적 네트워크를 구성합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    오버레이 네트워크                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              오버레이 vs 언더레이                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Overlay Network (가상)                       │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  VM1 ◄──────────────► VM2                   │    │   │ │
│   │   │  │  (10.0.0.1)          (10.0.0.2)             │    │   │ │
│   │   │  │       │                    │                 │    │   │ │
│   │   │  │       │    VXLAN Tunnel    │                 │    │   │ │

│   │   │  │       ▼                    ▼                 │    │ │

│   │   │  │ Underlay Network (물리)                     │    │   │ │

│   │   │  │  Host1 ◄──────────────► Host2               │    │   │ │
│   │   │  │  (192.168.1.1)        (192.168.1.2)         │    │   │ │

│   │   │  │ GRE        │ Key     │ 터널     │               │   │ │
```

> **해설**: 오버레이는 물리 네트워크와 논리 네트워크를 분리합니다.

**💡 비유**: 오버레이 네트워크는 터널입니다.

**📢 섹션 요약 비유:** Overlay = 가상 네트워크!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **VTEP** | VXLAN Tunnel Endpoint |
| **VNI** | VXLAN Network Identifier |
| **Encap/Decap** | 캡슐화/역캡슐화 |
| **Tunnel** | 터널 |

**비교표: 오버레이 기술**

```
+------------+----------+----------+----------+
| 기술       │ 오버헤드 │ 확장성   │ 용도     │
+------------+----------+----------+----------+
| VXLAN      │ 50B      │ 16M      │ 표준     │
| GRE        │ 4B+      │ 제한     │ 레거시   │
| GENEVE     │ 가변     │ 무제한   │ 확장     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 캡슐화, 플러딩

**📢 섹션 요약 비유:** VXLAN = 오버레이 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 활용 사례**

| 사례 | 기술 | 이점 |
|:---|:---|:---|
| **데이터센터** | VXLAN | VM 이동성 |
| **클라우드** | VXLAN/Geneve | 멀티테넌시 |
| **SDN** | 여러 | 유연성 |

**📢 섹션 요약 비유:** VTEP = 터널 끝점!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 데이터센터 → VXLAN + EVPN
**시나리오 2:** 컨테이너 → VXLAN/IPIP
**시나리오 3:** 클라우드 → GENEVE

**도입 체크리스트:** MTU, 멀티캐스트, 컨트롤러

**📢 섹션 요약 비유:** MTU = 오버레이 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 네트워크 유연성

**미래 전망:** SRv6, eBPF

**참고 표준:** RFC 7348 (VXLAN)

**📢 섹션 요약 비유:** 오버레이는 가상화의 핵심!
