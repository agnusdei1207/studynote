# Hyper-threading - 하이퍼스레딩

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SMT (동시 멀티스레딩)
> 2. **가치**: 코어 활용률 향상
> 3. **융합:** Multi-core, Thread, Pipeline

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Hyper-threading은 한 코어에서 두 스레드를 동시 실행합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Hyper-threading                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              하이퍼스레딩 구조                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Without HT:               With HT:                  │   │ │
│   │   │  ┌─────────────────┐      ┌───────────────────────┐ │   │ │
│   │   │  │      Core       │      │        Core           │ │   │ │
│   │   │  │  ┌───────────┐  │      │  ┌─────────┬────────┐ │ │   │ │
│   │   │  │  │Thread 1   │  │      │  │Thread 1 │Thread 2│ │ │   │ │
│   │   │  │  │           │  │      │  │ (ALU)   │ (MEM)  │ │ │   │ │
│   │   │  │  └───────────┘  │      │  └─────────┴────────┘ │ │   │ │
│   │   │  │   활용률: 50%   │      │   활용률: ~80%        │ │   │ │
│   │   │  └─────────────────┘      └───────────────────────┘ │   │ │
│   │   │                                                      │   │ │
│   │   │  공유: L1/L2 Cache, Execution Units                  │   │ │
│   │   │  독립: Register, PC                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 하이퍼스레딩은 유휴 자원을 활용합니다.

**📢 섹션 요약 비유:** Hyper-threading = 듀얼 태스킹!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SMT** | Simultaneous Multi-Threading |
| **Logical Core** | 논리 코어 |
| **Shared Resources** | 공유 자원 |
| **Thread State** | 스레드 상태 |
| **Pipeline** | 파이프라인 |

**핵심 알고리즸:** 스레드 A 대기 → 스레드 B 실행

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Physical vs Logical Core**

| 항목 | Physical | Logical (HT) |
|:---|:---|:---|
| **ALU** | 독립 | 공유 |
| **Cache** | 독립 | 공유 |
| **성능** | 100% | 30-40% |
