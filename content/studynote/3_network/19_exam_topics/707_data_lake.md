# Data Lake - 데이터 레이크

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 원시 데이터 저장소
> 2. **가치**: 유연성, 저렴한 비용
> 3. **융합:** Big Data, S3, Schema-on-Read

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Data Lake는 원시 데이터를 그대로 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Data Lake                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Data Lake 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Data Sources                                        │   │ │
│   │   │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐                   │   │ │
│   │   │  │Logs │ │ JSON│ │ CSV │ │ Img │                   │   │ │
│   │   │  └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘                   │   │ │
│   │   │     └───────┴───────┴───────┘                        │   │ │
│   │   │                     ↓                                 │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │              Data Lake                       │    │   │ │
│   │   │  │  ┌───────────────────────────────────────┐  │    │   │ │
│   │   │  │  │ Raw Zone (Bronze) - 원시 데이터        │  │    │   │ │
│   │   │  │  │ Cleansed Zone (Silver) - 정제 데이터   │  │    │   │ │
│   │   │  │  │ Curated Zone (Gold) - 가공 데이터      │  │    │   │ │
│   │   │  │  └───────────────────────────────────────┘  │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  Schema-on-Read (읽을 때 스키마 적용)        │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Data Lake는 스키마를 읽을 때 적용합니다.

**📢 섹션 요약 비유:** Data Lake = 원시 데이터 바다!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Raw Zone** | 원시 영역 |
| **Cleansed Zone** | 정제 영역 |
| **Curated Zone** | 가공 영역 |
| **Schema-on-Read** | 읽기 시 스키마 |
| **Object Storage** | 객체 스토리지 |

**핵심 알고리즸:** 수집 → 저장 → 변환 → 분석

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Data Lake vs Data Warehouse**

| 항목 | Data Lake | Data Warehouse |
|:---|:---|:---|
| **데이터** | 비정형/정형 | 정형 |
| **스키마** | 읽을 때 | 쓸 때 |
| **비용** | 낮음 | 높음 |
| **용도** | 탐색 | 분석 |
