### Ⅰ. 개요 (Context & Background)

**개념 정의**

Thread Pool은 스레드를 재사용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Thread Pool                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스레드 풀 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Task Queue        Thread Pool                       │   │ │
│   │   │  ┌─────────┐       ┌─────────────────────────────┐  │   │ │
│   │   │  │ Task 1  │       │ ┌─────┐ ┌─────┐ ┌─────┐    │  │   │ │
│   │   │  │ Task 2  │ ────→ │ │ T1  │ │ T2  │ │ T3  │    │  │   │ │
│   │   │  │ Task 3  │       │ └─────┘ └─────┘ └─────┘    │  │   │ │
│   │   │  │ Task 4  │       │     Worker Threads         │  │   │ │
│   │   │  │ ...     │       └─────────────────────────────┘  │   │ │
│   │   │  └─────────┘                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  구성:                                                │   │ │
│   │   │  - Core Pool Size: 기본 스레드 수                      │   │ │
│   │   │  - Max Pool Size: 최대 스레드 수                       │   │ │
│   │   │  - Queue Capacity: 대기열 크기                        │   │ │
│   │   │  - Keep Alive: 유휴 스레드 유지 시간                    │   │ │
│   │   │  - Rejection Policy: 거부 정책                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │

│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스레드 풀은 스레드 생성 비용을 줄입니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Core Size** | 기본 스레드 수 |
| **Max Size** | 최대 스레드 수 |
| **Queue** | 작업 대기열 |
| **Rejected** | 거부 정책 |
| **Hook** | 훅 메서드 |

**핵심 알고리즸:** 작업 제출 → 큐 → 스레드 할당 → 실행

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**거부 정책**

| 정책 | 설명 |
|:---|:---|
| **Abort** | 예외 발생 |
| **Caller Runs** | 호출자 실행 |
| **Discard** | 조용히 폐기 |
| **Discard Oldest** | 오래된 것 폐기 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** CPU 작업 → Core 수
**시나리오 2:** I/O 작업 → Core * 2
**시나리오 3:** 무제한 → 위험
