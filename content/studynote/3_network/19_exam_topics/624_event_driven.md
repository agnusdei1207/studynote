### Ⅰ. 개요 (Context & Background)

**개념 정의**

Event Driven Architecture는 이벤트로 통신하는 아키텍처입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Event Driven Architecture                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              EDA 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Event Producer                                      │   │ │
│   │   │  ┌─────────┐                                        │   │ │
│   │   │  │ Service │                                        │   │ │
│   │   │  │    A    │                                        │   │ │
│   │   │  └────┬────┘                                        │   │ │
│   │   │       │ Event                                       │   │ │
│   │   │       ↓                                             │   │ │
│   │   │  ┌─────────────────────────────────────┐            │   │ │
│   │   │  │          Event Bus                   │            │   │ │
│   │   │  └─────────────────────────────────────┘            │   │ │
│   │   │       │           │           │                      │   │ │
│   │   │       ↓           ↓           ↓                      │   │ │
│   │   │  ┌─────┐    ┌─────┐    ┌─────┐                     │   │ │
│   │   │  │ Svc │    │ Svc │    │ Svc │                     │   │ │
│   │   │  │  B  │    │  C  │    │  D  │                     │   │ │
│   │   │  └─────┘    └─────┘    └─────┘                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

**📢 섹션 요약 비유:** EDA = 방송!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Event** | 상태 변화 알림 |
| **Producer** | 이벤트 생성자 |
| **Consumer** | 이벤트 처리자 |
| **Event Bus** | 이벤트 전달 채널 |
| **Event Store** | 이벤트 저장소 |

**핵심 알고리즸:** 상태 변화 → 이벤트 발행 → 구독자 처리

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**EDA 패턴**

| 패턴 | 설명 |
|:---|:---|
| **Pub/Sub** | 게시-구독 |
| **Event Sourcing** | 이벤트 저장 |
| **CQRS** | 명령/조회 분리 |
| **Saga** | 분산 트랜잭션 |
