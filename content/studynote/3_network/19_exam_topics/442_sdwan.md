# SD-WAN (소프트웨어 정의 WAN) - Software-Defined WAN

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: WAN을 소프트웨어로 제어
> 2. **가치**: 비용 절감, 성능 최적화
> 3. **융합**: MPLS 대체, 하이브리드 WAN

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SD-WAN은 WAN 연결을 소프트웨어로 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SD-WAN 아키텍처                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SD-WAN 구성                                       │ │
│   │                                                              │ │

│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ SD-WAN Controller (중앙 관리)               │    │   │ │
│   │   │  │ - 정책 관리                                 │    │   │ │
│   │   │  │ - 경로 최적화                               │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                     │                              │   │ │
│   │   │     ┌───────────────┼───────────────┐             │   │ │
│   │   │     ▼               ▼               ▼             │   │ │

│   │   │  │지사1 │       │지사2 │       │본사  │          │   │ │
│   │   │  └──────┘       └──────┘       └──────┘          │   │ │
│   │   │                                                      │   │ │
│   │   │  링크: MPLS + 인터넷 + LTE/5G                        │   │ │

│   │   │  | 유연성     │ 낮음     │ 높음     │               │   │ │
```

> **해설**: SD-WAN은 여러 링크를 지능적으로 관리합니다.

**💡 비유**: SD-WAN은 트래픽의 내비게이션입니다.

**📢 섹션 요약 비유:** SD-WAN = WAN 혁신!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **CPE** | Customer Premises Equipment |
| **Controller** | 중앙 제어 |
| **Overlay** | 가상 네트워크 |
| **Underlay** | 물리 링크 |

**비교표: 링크 유형**

```
+------------+----------+----------+----------+
| 링크       │ 비용     │ 성능     │ 용도     │
+------------+----------+----------+----------+
| MPLS       │ 높음     │ 안정     │ 미션     │
| 인터넷     │ 낮음     │ 가변     │ 일반     │
| LTE/5G     │ 중간     │ 무선     │ 백업     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 경로 선택, QoS

**📢 섹션 요약 비유:** 하이브리드 = 최적!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 트래픽 제어**

| 기능 | 설명 | 이점 |
|:---|:---|:---|
| **애플리케이션 인식** | 앱별 라우팅 | 최적 경로 |
| **동적 경로** | 실시간 전환 | 고가용성 |
| **보안** | 암호화 | 데이터 보호 |

**📢 섹션 요약 비유:** 애플리케이션 기반 = 스마트!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 지사 연결 → SD-WAN
**시나리오 2:** MPLS 대체 → 하이브리드
**시나리오 3:** 클라우드 → Direct Connect

**도입 체크리스트:** 링크 평가, 정책, 보안

**📢 섹션 요약 비유:** MPLS + 인터넷 = 최적!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: WAN 비용 50% 절감

**미래 전망:** SASE (Secure Access Service Edge)

**참고 표준:** MEF 70

**📢 섹션 요약 비유:** SD-WAN은 WAN의 미래!
