# Sliding Window - 슬라이딩 윈도우

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연속 전송 메커니즘
> 2. **가치**: 효율적 전송
> 3. **융합:** TCP, Go-Back-N, Selective Repeat

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Sliding Window는 윈도우 내 연속 전송을 허용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Sliding Window                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              슬라이딩 윈도우 동작                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  윈도우 크기 = 4                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  초기:                                                 │   │ │
│   │   │  ┌───┬───┬───┬───┬───┬───┬───┬───┐                  │   │ │
│   │   │  │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │                  │   │ │
│   │   │  └───┴───┴───┴───┴───┴───┴───┴───┘                  │   │ │
│   │   │    ↑─── Window ───↑   ↑──── 대기 ────↑               │   │ │
│   │   │                                                      │   │ │
│   │   │  ACK 1 수신 후:                                        │   │ │
│   │   │  ┌───┬───┬───┬───┬───┬───┬───┬───┐                  │   │ │
│   │   │  │ ✓ │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │                  │   │ │
│   │   │  └───┴───┴───┴───┴───┴───┴───┴───┘                  │   │ │
│   │   │        ↑─── Window ───↑   ↑── 대기 ──↑               │   │ │
│   │   │        → 윈도우 슬라이드                               │   │ │
│   │   │                                                      │   │ │
│   │   │  ARQ 방식:                                             │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Go-Back-N: 손실 시 이후 모두 재전송               ││   │ │
│   │   │  │ Selective Repeat: 손실 패킷만 재전송             ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  효율:                                                 │   │ │
│   │   │  U = N / (1 + 2a)  (N: 윈도우, a: 전파/전송 비율)       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 슬라이딩 윈도우는 대역폭 활용도를 극대화합니다.

**📢 섹션 요약 비유:** Sliding Window = 컨베이어 벨트!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Window** | 윈도우 크기 |
| **Base** | 윈도우 시작 |
| **NextSeq** | 다음 전송 |
| **ACK** | 확인 응답 |
| **Slide** | 슬라이드 |

**핵심 알고리즸:** 전송 → ACK → 윈도우 이동 → 전송

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ARQ 방식 비교**

| 방식 | 재전송 | 효율 | 복잡도 |
|:---|:---|:---|:---|
| **Stop-Wait** | 전체 | 낮음 | 낮음 |
| **Go-Back-N** | 이후 전체 | 중간 | 중간 |
| **Selective Repeat** | 선택적 | 높음 | 높음 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** TCP → Sliding Window
**시나리오 2:** 높은 오류 → Selective Repeat
**시나리오 3:** 단순 구현 → Go-Back-N
