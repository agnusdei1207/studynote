# IPsec - IP 보안

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 계층 보안
> 2. **가치**: 투명한 암호화
> 3. **융합:** AH, ESP, IKE, SA

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IPsec (IP Security)은 IP 계층에서 보안을 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IPsec                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IPsec 프로토콜                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  AH (Authentication Header):                           │   │ │
│   │   │  - 인증만 제공 (무결성)                                 │   │ │
│   │   │  - 암호화 없음                                          │   │ │
│   │   │  - 프로토콜 번호: 51                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  ESP (Encapsulating Security Payload):                  │   │ │
│   │   │  - 인증 + 암호화                                        │   │ │
│   │   │  - 기밀성 + 무결성                                      │   │ │
│   │   │  - 프로토콜 번호: 50                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  모드:                                                   │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Transport Mode: 상위 계층만 보호                 ││   │ │
│   │   │  │   [IP][ESP][TCP][Data]                          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ Tunnel Mode: 전체 패킷 보호                      ││   │ │
│   │   │  │   [New IP][ESP][Orig IP][TCP][Data]             ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  IKE (Internet Key Exchange):                           │   │ │
│   │   │  - 키 교환 프로토콜                                     │   │ │
│   │   │  - SA (Security Association) 생성                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IPsec은 VPN 구현의 표준입니다.

**📢 섹션 요약 비유:** IPsec = IP 계층 경호원!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **AH** | 인증 헤더 |
| **ESP** | 캡슐화 보안 페이로드 |
| **SA** | 보안 연결 |
| **SPI** | 보안 매개변수 인덱스 |
| **IKE** | 인터넷 키 교환 |

**핵심 알고리즸:** IKE 협상 → SA 생성 → ESP/AH 적용

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**AH vs ESP 비교**

| 항목 | AH | ESP |
|:---|:---|:---|
| **인증** | 있음 | 있음 |
| **암호화** | 없음 | 있음 |
| **NAT 통과** | 불가 | 가능 |
| **용도** | 무결성 | 기밀성 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** Site-to-Site VPN → IPsec Tunnel
**시나리오 2:** NAT 환경 → ESP + NAT-T
**시나리오 3:** 무결성만 → AH
