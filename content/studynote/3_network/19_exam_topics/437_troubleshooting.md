# 네트워크 문제 해결 - Network Troubleshooting

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 체계적 문제 해결 방법론
> 2. **가치**: 빠른 복구, 원인 식별
> 3. **융합**: OSI 계층별 접근

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

네트워크 문제 해결은 체계적인 방법으로 문제를 해결합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    네트워크 문제 해결                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              문제 해결 프로세스                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 문제 정의                                        │   │ │
│   │   │     - 증상 파악                                      │   │ │
│   │   │     - 영향 범위                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 정보 수집                                        │   │ │
│   │   │     - 로그 분석                                      │   │ │
│   │   │     - 사용자 인터뷰                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 원인 분석                                        │   │ │
│   │   │     - OSI 계층별 접근                                │   │ │
│   │   │     - 하향식/상향식                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 해결 적용                                        │   │ │
│   │   │     - 변경 문서화                                    │   │ │
│   │   │     - 테스트                                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              계층별 접근법                                     │ │

│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 계층       │ 문제     │ 도구     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ L1 물리    │ 케이블   │ 케이블테스터│            │   │ │
│   │   │  │ L2 데이터  │ 스위치   │ show mac │               │   │ │
```

> **해설**: 계층별로 체계적으로 접근하면 효율적입니다.

**💡 비유**: 문제 해결은 의사의 진단 과정입니다.

**📢 섹션 요약 비유:** 체계적 접근 = 해결!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Top-Down** | 애플리케이션부터 시작 |
| **Bottom-Up** | 물리 계층부터 시작 |
| **Divide-Conquer** | 중간 계층부터 시작 |

**비교표: 접근법 선택**

```
+------------+----------+----------+----------+
| 접근법     │ 장점     │ 단점     │ 용도     │
+------------+----------+----------+----------+
| 하향식     │ 사용자   │ 계층    │ 앱 문제  │
| 상향식     │ 기본    │ 시간    │ 인프라   │
| 분할정복   │ 빠름    │ 경험    │ 일반     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 격리, 테스트

**📢 섹션 요약 비유:** OSI 계층 = 로드맵!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 일반적 문제**

| 문제 | 계층 | 해결 |
|:---|:---|:---|
| **케이블** | L1 | 교체 |
| **VLAN** | L2 | 설정 확인 |
| **IP** | L3 | DHCP/라우팅 |
| **방화벽** | L4 | 규칙 확인 |

**📢 섹션 요약 비유:** 물리 계층부터 확인!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 인터넷 안 됨 → L1 → L2 → L3
**시나리오 2:** 특정 사이트만 → DNS/방화벽
**시나리오 3:** 간헐적 → 로그/모니터링

**도입 체크리스트:** 문서화, 백업, 롤백

**📢 섹션 요약 비유:** 문서화 = 재발 방지!
