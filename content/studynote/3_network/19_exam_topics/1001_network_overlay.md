# Network Overlay - 네트워크 오버레이

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 논리적 네트워크 추상화
> 2. **가치**: 물리적 제약 극복
> 3. **융합:** VXLAN, NVGRE, Geneve

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Network Overlay는 물리 네트워크 위에 논리적 네트워크를 구성합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Network Overlay                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Overlay vs Underlay                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Overlay Network (논리):                               │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │  VM1 ←───────→ VM2 ←───────→ VM3               ││   │ │
│   │   │  │       L2 연결 (가상)                             ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  오버레이 기술:                                   ││   │ │
│   │   │  │  - VXLAN, NVGRE, Geneve                         ││   │ │
│   │   │  │  - STT, GENEVE                                  ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                        │                             │   │ │
│   │   │                        │ 캡슐화                       │   │ │
│   │   │                        ▼                             │   │ │
│   │   │  Underlay Network (물리):                             │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  ┌──────┐   ┌──────┐   ┌──────┐                ││   │ │
│   │   │  │  │Router│───│Router│───│Router│                ││   │ │
│   │   │  │  └──────┘   └──────┘   └──────┘                ││   │ │
│   │   │  │       L3 라우팅 (물리)                           ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Overlay는 물리적 제약 없이 유연한 네트워크를 제공합니다.

**📢 섹션 요약 비유:** Overlay = 가상 터널!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Encapsulation** | 캡슐화 |
| **VTEP** | 터널 종단 |
| **VNI/VSID** | 가상 네트워크 ID |
| **Tunnel** | 터널 |
| **Mapping** | 주소 매핑 |

**핵심 알고리듬:** 캡슐화 → 언더레이 전송 → 디캡슐레이션 → 전달

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**오버레이 기술 비교**

| 기술 | 캡슐화 | 용도 |
|:---|:---|:---|
| **VXLAN** | UDP | 데이터센터 |
| **NVGRE** | GRE | Hyper-V |
| **Geneve** | UDP | 범용 |
| **STT** | TCP | OVS |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 데이터센터 → VXLAN Overlay
**시나리오 2:** 멀티클라우드 → Overlay + IPsec
**시나리오 3:** 컨테이너 → Overlay (CNI)
