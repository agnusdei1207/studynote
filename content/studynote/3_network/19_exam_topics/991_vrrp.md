# VRRP - 가상 라우터 이중화 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 라우터 이중화 표준
> 2. **가치**: 게이트웨이 고가용성
> 3. **융합:** Master, Backup, Virtual Router

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

VRRP (Virtual Router Redundancy Protocol)는 라우터 이중화를 위한 표준 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    VRRP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VRRP 동작                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  VRRP 구성:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │         ┌─────────────────────┐                 ││   │ │
│   │   │  │         │   Virtual Router    │                 ││   │ │
│   │   │  │         │   VIP: 10.0.0.1     │                 ││   │ │
│   │   │  │         │   VMAC: 00-00-5E-00-01-XX            ││   │ │
│   │   │  │         └──────────┬──────────┘                 ││   │ │
│   │   │  │                    │                            ││   │ │

│   │   │  │         │                     │                 ││   │ │
│   │   │  │  ┌──────┴──────┐       ┌──────┴──────┐         ││   │ │
│   │   │  │  │   Master    │       │   Backup    │         ││   │ │
│   │   │  │  │  Priority   │       │  Priority   │         ││   │ │
│   │   │  │  │   100       │       │    90       │         ││   │ │
│   │   │  │  │ Real:       │       │ Real:       │         ││   │ │

│   │   │                                                      │   │ │
```

> **해설**: VRRP는 게이트웨이 장애 시 자동 전환을 제공합니다.

**📢 섹션 요약 비유:** VRRP = 예비 운전사!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Master** | 활성 라우터 |
| **Backup** | 대기 라우터 |
| **Priority** | 우선순위 |
| **VRID** | 가상 라우터 ID |
| **Preempt** | 선점 모드 |

**핵심 알고리듬:** 우선순위 비교 → 마스터 선출 → 광고 → 장애 감지

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**VRRP vs HSRP**

| 항목 | VRRP | HSRP |
|:---|:---|:---|
| **표준** | RFC 5798 | Cisco |
| **활성** | Master | Active |
| **대기** | Backup | Standby |
| **멀티캐스트** | 224.0.0.18 | 224.0.0.2 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 기본 게이트웨이 → VRRP

**시나리오 3:** Cisco 환경 → HSRP 또는 VRRP
