# Deadlock - 교착 상태

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 상호 대기 상태
> 2. **가치**: 시스템 정지 방지
> 3. **융합:** Lock, Resource, Prevention

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Deadlock은 두 프로세스가 서로의 자원을 기다리는 상태입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Deadlock                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Deadlock 상황                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │      Process A              Process B                │   │ │
│   │   │      ┌─────┐               ┌─────┐                  │   │ │
│   │   │      │     │               │     │                  │   │ │
│   │   │      │     │    Wait for   │     │                  │   │ │
│   │   │      └──┬──┘←──────────────└──┬──┘                  │   │ │
│   │   │         │        Resource B    │                     │   │ │
│   │   │         │                      │                     │   │ │
│   │   │   Holds │                      │ Holds               │   │ │
│   │   │   Res A │                      │ Res B               │   │ │
│   │   │         │                      │                     │   │ │
│   │   │         └──────────────────────┘                     │   │ │
│   │   │           Wait for Resource A                         │   │ │
│   │   │                                                      │   │ │
│   │   │  결과: 서로가 서로를 기다리며 멈춤                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 교착 상태는 상호 대기로 발생합니다.

**📢 섹션 요약 비유:** Deadlock = 일방통행 맞은편 차!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 조건 | 설명 |
|:---|:---|
| **상호 배제** | 자원은 하나만 사용 |
| **점유 대기** | 자원을 가진 채 대기 |
| **비선점** | 강제 회수 불가 |
| **순환 대기** | 원형 대기 |

**핵심 알고리즸:** 4조건 충족 시 Deadlock 발생
