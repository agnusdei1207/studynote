# RSVP (자원 예약 프로토콜) - Resource Reservation Protocol

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자원 예약 신호 프로토콜
> 2. **가치**: QoS 보장 경로 설정
> 3. **융합:** IntServ, MPLS TE

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RSVP는 경로를 따라 자원을 예약합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RSVP 동작                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RSVP 메시지 흐름                                  │ │
│   │                                                              │ │
│   │   Sender ──► Router1 ──► Router2 ──► Receiver              │   │ │
│   │              │              │              │                 │   │ │
│   │              │  PATH 메시지 (하행)          │                 │   │ │
│   │              │──────────────►──────────────►│                 │   │ │
│   │              │                              │                 │   │ │
│   │              │  RESV 메시지 (상행)          │                 │   │ │
│   │              │◄──────────────◄──────────────│                 │   │ │
│   │              │                              │                 │   │ │
│   │              └──────────────────────────────┘                 │   │ │
│   │                      자원 예약 완료                           │   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RSVP는 PATH로 경로 탐색, RESV로 자원 예약합니다.

**📢 섹션 요약 비유:** RSVP = 자원 예약!
