### Ⅰ. 개요 (Context & Background)

**개념 정의**

CAP는 분산 시스템의 세 가지 속성 중 두 가지만 보장됩니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CAP Theorem                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CAP 삼각형                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │                  Consistency                         │   │ │
│   │   │                     (C)                              │   │ │
│   │   │                    ╱ ╲                               │   │ │
│   │   │                   ╱   ╲                              │   │ │
│   │   │                  ╱     ╲                             │   │ │
│   │   │                 ╱       ╲                            │   │ │
│   │   │                ╱         ╲                           │   │ │
│   │   │               ╱           ╲                          │   │ │
│   │   │  Availability ───────────── Partition                │   │ │
│   │   │       (A)         (P)      Tolerance                 │   │ │
│   │   │                                                      │   │ │
│   │   │  CA: RDBMS (Partition 없을 때)                       │   │ │
│   │   │  CP: MongoDB, HBase                                 │   │ │
│   │   │  AP: Cassandra, DynamoDB                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 네트워크 분할 시 일관성 또는 가용성을 선택해야 합니다.

**📢 섹션 요약 비유:** CAP 이론 = 세 가지 중 두 가지만!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Consistency** | 모든 노드 동일 데이터 |
| **Availability** | 항상 응답 |
| **Partition Tolerance** | 분할 허용 |
| **Trade-off** | 선택 필요 |

**핵심 알고리즸:** 분할 발생 → C 또는 A 선택

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**CAP 조합 시스템**

| 조합 | 시스템 |
|:---|:---|
| **CA** | RDBMS (단일 노드) |

| **AP** | Cassandra, DynamoDB |
