# STP (Spanning Tree Protocol) - 스패닝 트리 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 루프 방지 트리 구성
> 2. **가치**: 브로드캐스트 스톰 방지
> 3. **융합:** Bridge ID, Root Bridge, Port State

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

STP는 루프가 없는 트리 토폴로지를 구성합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    STP                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              STP 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  루프 토폴로지:           STP 적용 후:               │   │ │
│   │   │                                                      │   │ │
│   │   │    A ──── B                A ──── B                  │   │ │
│   │   │    │      │                │      │                  │   │ │
│   │   │    │      │         →      │      │ (Blocked)        │   │ │
│   │   │    │      │                │      │                  │   │ │
│   │   │    C ──── D                C ──── D                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Root Bridge: 가장 낮은 Bridge ID                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: STP는 루프를 차단하여 브로드캐스트 스톰을 방지합니다.

**📢 섹션 요약 비유:** STP = 교통 정리!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Bridge ID** | Priority + MAC |
| **Root Bridge** | 최소 Bridge ID |
| **Root Port** | Root로 가는 포트 |
| **Designated Port** | 세그먼트당 전달 포트 |
| **Blocked Port** | 차단 포트 |

**핵심 알고리즸:** Root 선출 → 경로 계산 → 포트 상태 결정

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**STP 포트 상태**

| 상태 | 설명 |
|:---|:---|
| **Blocking** | 프레임 차단 |
| **Listening** | BPDU 수신 |
| **Learning** | MAC 학습 |
| **Forwarding** | 정상 전달 |
| **Disabled** | 비활성 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 루프 → STP 활성화
**시나리오 2:** 빠른 복구 → RSTP
**시나리오 3:** VLAN별 → PVST+
