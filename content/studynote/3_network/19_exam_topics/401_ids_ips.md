# IDS/IPS - 침입 탐지/방지 시스템

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 악성 트래픽 탐지/차단
> 2. **가치**: 실시간 위협 대응
> 3. **융합**: SIEM, EDR, 방화벽

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IDS는 침입을 탐지하고, IPS는 탐지 후 차단합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IDS/IPS                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IDS vs IPS                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [IDS - 침입 탐지]                                  │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 트래픽 미러링 (Inline 아님)                    │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [분석] ────▶ [알림만]                              │   │ │
│   │   │                                                      │   │ │
│   │   │  [IPS - 침입 방지]                                  │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Inline 배치                                    │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [분석] ────▶ [차단 + 알림]                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              탐지 방식                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 방식       │ 설명     │ 장단점   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 시그니처   │ 패턴 매칭│ 정확/0-dayX│             │   │ │
│   │   │  │ 이상       │ 행동분석│ 0-dayO/오탐│              │   │ │
│   │   │  | 하이브리드 │ 혼합     │ 균형     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IDS는 모니터링, IPS는 실제 차단을 수행합니다.

**💡 비유**: IDS는 CCTV, IPS는 경비원입니다.

**📢 섹션 요약 비유:** IPS = 탐지 + 차단!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Sensor** | 트래픽 수집 |
| **Analysis Engine** | 분석 엔진 |
| **Signature DB** | 시그니처 데이터베이스 |
| **Alert System** | 알림 시스템 |

**비교표: 배치 위치**

```
+------------+----------+----------+
| 항목       │ NIDS/NIPS│ HIDS/HIPS│
+------------+----------+----------+
| 위치       │ 네트워크 │ 호스트   │
| 대상       │ 트래픽   │ 시스템   │
| 범위       │ 전체     │ 단일     │
| 리소스     │ 낮음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** 패턴 매칭, 통계 분석, 머신러닝

**📢 섹션 요약 비유:** NGIPS = 차세대!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 보안 장비**

| 항목 | 방화벽 | IPS | WAF |
|:---|:---|:---|:---|
| **계층** │ L3/4 │ L3-7 │ L7 |
| **대상** │ IP/Port | 공격패턴 | 웹공격 |
| **행동** │ 허용/차단 | 차단 | 차단 |

**📢 섹션 요약 비유:** 네트워크 보안 = IPS!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 네트워크 경계 → NIPS
**시나리오 2:** 중요 서버 → HIDS/HIPS
**시나리오 3:** 웹 애플리케이션 → WAF

**도입 체크리스트:** 배치 위치, 시그니처 업데이트, 튜닝

**📢 섹션 요약 비유:** 실시간 차단 = IPS!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 실시간 위협 차단

**미래 전망:** AI/ML 기반 탐지

**참고 표준:** NIST SP 800-94

**📢 섹션 요약 비유:** IDS/IPS는 침입 방지 핵심!
