# XDP (eXpress Data Path) - 고속 패킷 처리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 네트워킹 스택 우회
> 2. **가치**: 최고 속도 패킷 처리
> 3. **융합**: eBPF, DDoS 방어와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

XDP(eXpress Data Path)는 NIC 드라이버에서 eBPF를 실행하여 초고속 패킷 처리를 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    XDP (eXpress Data Path)                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              XDP 위치                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [NIC]                                               │   │ │
│   │   │     │                                                │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │  XDP (eBPF)  ← 최고 속도                    │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │     │                                                │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │  커널 네트워킹 스택                          │    │   │ │
│   │   │  │  (TCP/IP, Socket)                           │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │     │                                                │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  [애플리케이션]                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              XDP 액션                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  액션          │ 설명                                │   │ │
│   │   │  ─────────────┼───────────────────────────────────│   │ │
│   │   │  XDP_PASS     │ 커널 스택으로 전달                   │   │ │
│   │   │  XDP_DROP     │ 패킷 폐기                           │   │ │
│   │   │  XDP_TX       │ 같은 NIC으로 반송                   │   │ │
│   │   │  XDP_REDIRECT │ 다른 인터페이스로 전달               │   │ │
│   │   │  XDP_ABORTED  │ 오류                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: XDP는 커널 스택 전에 패킷을 처리합니다.

**💡 비유**: XDP는 정문 경비원입니다. 들어오기 전에 확인합니다.

**📢 섹션 요약 비유:** XDP = NIC 수준 처리!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **XDP_DROP** | DDoS 방어 |
| **XDP_PASS** | 정상 전달 |
| **XDP_TX** | 반송 |
| **XDP_REDIRECT** | AF_XDP |

**비교표: XDP 모드**

```
+------------+----------+----------+
| 모드       │ 위치     │ 속도     │
+------------+----------+----------+
| Native     │ 드라이버 │ 빠름     │
| Offload    │ NIC      │ 최고     │
| Generic    │ 커널     │ 느림     │
+------------+----------+----------+
```

**핵심 알고리즸:** eBPF, 액션 결정

**📢 섹션 요약 비유:** Offload = NIC에서 실행!
