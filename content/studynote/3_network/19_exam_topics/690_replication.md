# Replication - 복제

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 복사
> 2. **가치**: 가용성, 내결함성
> 3. **융합:** Master-Slave, Multi-Master, Consensus

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Replication은 데이터를 여러 노드에 복사합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Replication                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              복제 아키텍처                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Master-Slave:                                       │   │ │
│   │   │  ┌───────────────────────────────────────────────┐  │   │ │
│   │   │  │                Master                         │  │   │ │
│   │   │  │           ┌─────────┐                        │  │   │ │
│   │   │  │           │  Write  │                        │  │   │ │
│   │   │  │           └────┬────┘                        │  │   │ │
│   │   │  │                │ Replicate                   │  │   │ │
│   │   │  │     ┌──────────┼──────────┐                  │  │   │ │
│   │   │  │     ↓          ↓          ↓                  │  │   │ │
│   │   │  │  Slave 1   Slave 2   Slave 3                │  │   │ │
│   │   │  │  (Read)    (Read)    (Read)                  │  │   │ │
│   │   │  └───────────────────────────────────────────────┘  │   │ │
│   │   │                                                      │   │ │
│   │   │  방식:                                                │   │ │
│   │   │  - Synchronous: 즉시 복제 (느림, 안전)               │   │ │
│   │   │  - Asynchronous: 지연 복제 (빠름, 위험)              │   │ │
│   │   │  - Semi-synchronous: 혼합                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 복제는 가용성과 성능을 위해 사용됩니다.

**📢 섹션 요약 비유:** Replication = 사본!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Master-Slave** | 주-종 구조 |
| **Multi-Master** | 다중 주 구조 |
| **Synchronous** | 동기 복제 |
| **Asynchronous** | 비동기 복제 |
| **Replication Lag** | 복제 지연 |

**핵심 알고리즸:** 쓰기 → 복제 → 확인 (동기/비동기)

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**복제 방식 비교**

| 방식 | 장점 | 단점 |
|:---|:---|:---|
| **Sync** | 데이터 손실 없음 | 느림 |
| **Async** | 빠름 | 손실 가능 |
| **Semi-sync** | 균형 | 복잡 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 금융 → Synchronous
**시나리오 2:** 소셜 → Asynchronous
**시나리오 3:** 읽기 많음 → Read Replica
