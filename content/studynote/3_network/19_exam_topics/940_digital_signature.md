# Digital Signature - 디지털 서명

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전자 서명
> 2. **가치**: 무결성, 인증, 부인방지
> 3. **융합:** RSA, ECDSA, PKI

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Digital Signature는 문서의 진위를 증명합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Digital Signature                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              디지털 서명 과정                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  서명 생성:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  메시지 → [Hash] → 해시값                        ││   │ │
│   │   │  │     ↓                                           ││   │ │
│   │   │  │  해시값 → [개인키 암호화] → 서명                   ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  전송: 메시지 + 서명                              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  서명 검증:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  수신: 메시지 + 서명                              ││   │ │
│   │   │  │     ↓                                           ││   │ │
│   │   │  │  메시지 → [Hash] → 해시값 A                       ││   │ │
│   │   │  │  서명 → [공개키 복호화] → 해시값 B                 ││   │ │
│   │   │  │     ↓                                           ││   │ │
│   │   │  │  A == B ? 유효 : 무효                             ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 디지털 서명은 전자 문서의 법적 효력을 부여합니다.

**📢 섹션 요약 비유:** Digital Signature = 공증!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Private Key** | 개인키 |
| **Public Key** | 공개키 |
| **Hash** | 해시 함수 |
| **Signature** | 서명값 |
| **Certificate** | 인증서 |

**핵심 알고리듬:** Hash → 개인키 암호화 → 서명

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**서명 알고리즘 비교**

| 알고리즘 | 키 크기 | 속도 |
|:---|:---|:---|
| **RSA** | 2048-4096 | 느림 |
| **ECDSA** | 256-521 | 빠름 |
| **EdDSA** | 25519/448 | 매우 빠름 |
| **DSA** | 1024-3072 | 중간 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 코드 서명 → RSA/ECDSA
**시나리오 2:** 문서 → 디지털 서명
**시나리오 3:** 블록체인 → ECDSA
