# Authentication - 인증

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 신원 확인
> 2. **가치**: 접근 제어 기반
> 3. **융합:** MFA, SSO, OAuth, JWT

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Authentication은 "누구인가"를 확인합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Authentication                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인증 요소 (Factor)                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Something you know:  비밀번호, PIN                  │   │ │
│   │   │  Something you have:   OTP, 스마트카드              │   │ │
│   │   │  Something you are:    지문, 안면 인식              │   │ │
│   │   │                                                      │   │ │
│   │   │  MFA (Multi-Factor Authentication):                  │   │ │
│   │   │  2가지 이상 요소 조합                                │   │ │
│   │   │                                                      │   │ │
│   │   │  Authentication vs Authorization:                    │   │ │
│   │   │  - Authentication: 누구인가?                         │   │ │
│   │   │  - Authorization: 무엇을 할 수 있는가?               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Authentication은 Authorization의 선행 조건입니다.

**📢 섹션 요약 비유:** Authentication = 신분증 확인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 방식 | 설명 |
|:---|:---|
| **Password** | 지식 기반 |
| **OTP** | 소유 기반 |
| **Biometric** | 생체 기반 |
| **Certificate** | 인증서 기반 |
| **SSO** | 통합 인증 |

**핵심 알고리즸:** 검증 → 토큰 발급

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**인증 프로토콜**

| 프로토콜 | 용도 |
|:---|:---|
| **OAuth 2.0** | 위임 인증 |
| **OIDC** | 인증 + 프로필 |
| **SAML** | 엔터프라이즈 SSO |
| **Kerberos** | 윈도우 도메인 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹 → Password + MFA
**시나리오 2:** API → OAuth 2.0
**시나리오 3:** 기업 → SAML SSO
