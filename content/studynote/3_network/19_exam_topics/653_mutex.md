# Mutex - 뮤텍스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 상호 배제 기법
> 2. **가치**: 데이터 보호
> 3. **융합:** Lock, Critical Section, Thread

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Mutex는 한 번에 하나의 스레드만 접근을 허용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Mutex                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              뮤텍스 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Thread A               Mutex                Thread B│   │ │
│   │   │  ┌─────────┐           ┌─────┐              ┌─────────┐│   │ │
│   │   │  │         │           │     │              │         ││   │ │
│   │   │  │ lock()  │──────────→│Locked│←────────────│  wait   ││   │ │
│   │   │  │         │           │     │              │ (block) ││   │ │
│   │   │  └─────────┘           └─────┘              └─────────┘│   │ │
│   │   │       │                                            ↑    │   │ │
│   │   │       ↓                                            │    │   │ │
│   │   │  ┌─────────┐                                     │    │   │ │
│   │   │  │Critical │                                     │    │   │ │
│   │   │  │ Section│                                     │    │   │ │
│   │   │  └─────────┘                                     │    │   │ │
│   │   │       │                                            │    │   │ │
│   │   │       ↓                                            │    │   │ │
│   │   │  ┌─────────┐           ┌───────┐                   │    │   │ │
│   │   │  │unlock() │──────────→│Unlocked│──────────────────┘    │   │ │
│   │   │  └─────────┘           └───────┘                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Mutex는 소유권 개념이 있어 소유자만 해제 가능합니다.

**📢 섹션 요약 비유:** Mutex = 화장실 열쇠!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **lock** | 잠금 획득 |
| **unlock** | 잠금 해제 |
| **trylock** | 비블로킹 시도 |
| **Owner** | 소유자 |
| **Recursive** | 재진입 가능 |

**핵심 알고리즸:** Lock → Critical Section → Unlock

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Mutex 종류**

| 종류 | 설명 |
|:---|:---|
| **Normal** | 기본 뮤텍스 |
| **Recursive** | 재진입 가능 |
| **Error Check** | 오류 검사 |
| **Spin** | 스핀 대기 |
