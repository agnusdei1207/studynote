# MTU (Maximum Transmission Unit) - 최대 전송 단위

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 링크 계층 최대 프레임 크기
> 2. **가치**: 단편화 결정, 성능 최적화
> 3. **융합:** MSS, Path MTU, Fragmentation

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MTU는 데이터링크 계층의 최대 프레임 크기입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MTU (Maximum Transmission Unit)                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              다양한 MTU                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  이더넷:        1500바이트                           │   │ │
│   │   │  PPPoE:        1492바이트                            │   │ │
│   │   │  VPN/IPsec:    ~1400바이트                           │   │ │
│   │   │  Jumbo Frame:  9000바이트                            │   │ │
│   │   │                                                      │   │ │
│   │   │  IP 패킷 > MTU → 단편화 (Fragmentation)              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: MTU를 초과하면 IP 단편화가 발생합니다.

**📢 섹션 요약 비유:** MTU = 트럭 적재 한계!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 네트워크 | MTU |
|:---|:---|
| **Ethernet** | 1500 |
| **PPPoE** | 1492 |
| **WiFi** | 1500 |
| **FDDI** | 4352 |
| **Jumbo** | 9000 |

**핵심 알고리즸:** packet > MTU → fragment

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**MTU vs MSS vs Window**

| 항목 | 계층 | 설명 |
|:---|:---|:---|
| **MTU** | L2 | 프레임 최대 |
| **MSS** | L4 | TCP 데이터 최대 |
| **Window** | L4 | 흐름 제어 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 표준 → 1500
**시나리오 2:** 데이터센터 → Jumbo Frame
**시나리오 3:** VPN → 낮은 MTU
