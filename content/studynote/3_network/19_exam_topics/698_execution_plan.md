# Execution Plan - 실행 계획

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 쿼리 실행 전략
> 2. **가치**: 성능 분석
> 3. **융합:** Query Optimization, EXPLAIN, Database

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Execution Plan은 쿼리 실행 전략입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Execution Plan                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              실행 계획 예시                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Query: SELECT * FROM users WHERE id = 1            │   │ │
│   │   │                                                      │   │ │
│   │   │  Execution Plan:                                     │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Id │ Operation       │ Rows │ Cost │        │    │   │ │
│   │   │  ├────┼─────────────────┼──────┼──────┤        │    │   │ │
│   │   │  │ 0  │ SELECT STATEMENT│   1  │   2  │        │    │   │ │
│   │   │  │ 1  │ INDEX UNIQUE   │   1  │   1  │        │    │   │ │
│   │   │  │    │ SCAN (PK)      │      │      │        │    │   │ │
│   │   │  │ 2  │ TABLE ACCESS   │   1  │   1  │        │    │   │ │
│   │   │  │    │ BY INDEX ROWID │      │      │        │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  분석: 인덱스 사용 → 빠른 조회                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 실행 계획은 쿼리가 어떻게 실행되는지 보여줍니다.

**📢 섹션 요약 비유:** Execution Plan = 지도!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Operation** | 연산 |
| **Rows** | 예상 행 수 |
| **Cost** | 비용 |
| **Access Method** | 접근 방법 |
| **Join Method** | 조인 방법 |

**핵심 알고리즸:** EXPLAIN → 계획 확인 → 분석

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**주요 연산**

| 연산 | 설명 |
|:---|:---|
| **Full Scan** | 전체 스캔 |
| **Index Scan** | 인덱스 스캔 |
| **Nested Loop** | 중첩 루프 조인 |
| **Hash Join** | 해시 조인 |
| **Sort Merge** | 정렬 병합 조인 |
