# POP3/IMAP - 메일 수신

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이메일 수신 프로토콜
> 2. **가치**: 다운로드/동기화
> 3. **융합**: SMTP와 함께 메일 시스템 구성

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

POP3(Post Office Protocol 3)와 IMAP(Internet Message Access Protocol)은 이메일을 수신하는 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    POP3 / IMAP                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              POP3 vs IMAP                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  POP3 (Post Office Protocol 3):                      │   │ │
│   │   │  • 다운로드 후 삭제                                  │   │ │
│   │   │  • 오프라인 읽기                                     │   │ │
│   │   │  • 단일 기기                                        │   │ │
│   │   │  • 단순함                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  IMAP (Internet Message Access Protocol):             │   │ │
│   │   │  • 서버에 보관                                      │   │ │
│   │   │  • 다중 기기 동기화                                  │   │ │
│   │   │  • 폴더 관리                                        │   │ │
│   │   │  • 현대적                                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              포트 및 보안                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  프로토콜 │ 기본  │ 보안   │ SSL/TLS               │   │ │
│   │   │  ────────┼──────┼───────┼────────────│   │ │
│   │   │  POP3    │ 110  │ 995   │ STLS                  │   │ │
│   │   │  IMAP    │ 143  │ 993   │ STARTTLS              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IMAP은 다중 기기 환경에 적합합니다.

**💡 비유**: POP3는 우편물을 가져가는 것, IMAP은 우편함을 확인하는 것입니다.

**📢 섹션 요약 비유:** IMAP = 다중 기기!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | POP3 | IMAP |
|:---|:---|:---|
| **저장** | 로컬 | 서버 |
| **동기화** | X | O |
| **폴더** | X | O |
| **플래그** | 제한 | 풍부 |
| **검색** | X | 서버 |

**비교표: IMAP 명령어**

```
+------------+----------+----------+
| 명령어     │ 기능     │ 설명     │
+------------+----------+----------+
| LOGIN      │ 인증     │ 사용자   │
| SELECT     │ 선택     │ 폴더     │
| FETCH      │ 조회     │ 메일     │
| SEARCH     │ 검색     │ 조건     │
| STORE      │ 저장     │ 플래그   │
+------------+----------+----------+
```

**핵심 알고리즸:** 메일 동기화, 플래그 관리

**📢 섹션 요약 비유:** IMAP = 서버 중심!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 메일 시스템**

**📢 섹션 요약 비유:** SMTP + IMAP = 완전한 메일!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 단일 기기** → POP3
**시나리오 2: 다중 기기** → IMAP
**시나리오 3: 기업** → IMAP + Exchange

**도입 체크리스트:** 기기 수, 오프라인 필요, 저장 공간

**📢 섹션 요약 비유:** 스마트폰 = IMAP!
