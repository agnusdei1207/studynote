# CIDR - Classless Inter-Domain Routing

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클래스 없는 주소 할당
> 2. **가치**: 주소 효율, 라우팅 축소
> 3. **융합**: VLSM, 서브넷팅, 라우팅

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CIDR은 클래스 경계 없이 IP 주소를 할당합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CIDR                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              클래스 vs CIDR                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  클래스 기반:                                        │   │ │
│   │   │  A: /8   (16M 주소)                                  │   │ │
│   │   │  B: /16  (65K 주소)                                  │   │ │
│   │   │  C: /24  (256 주소)                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  CIDR:                                               │   │ │
│   │   │  /20 (4,096 주소)                                    │   │ │
│   │   │  /23 (512 주소)                                      │   │ │
│   │   │  /27 (32 주소)                                       │   │ │
│   │   │  → 필요한 만큼만 할당                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CIDR 표기법                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ CIDR       │ 마스크   │ 주소수   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ /24        │ 255.255.255.0│ 256  │               │   │ │
│   │   │  │ /25        │ 255.255.255.128│ 128│               │   │ │
│   │   │  | /26        │ 255.255.255.192│ 64 │               │   │ │
│   │   │  │ /27        │ 255.255.255.224│ 32 │               │   │ │
│   │   │  │ /28        │ 255.255.255.240│ 16 │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: CIDR은 주소 낭비를 줄이고 라우팅을 효율화합니다.

**💡 비유**: CIDR은 필요한 만큼만 피자를 주문하는 것입니다.

**📢 섹션 요약 비유:** CIDR = 효율적 할당!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Prefix** | 네트워크 비트 수 |
| **Mask** | 서브넷 마스크 |
| **Block** | 주소 블록 |
| **Aggregation** | 라우팅 축약 |

**비교표: 클래스 vs CIDR**

```
+------------+----------+----------+
| 항목       │ 클래스   │ CIDR     │
+------------+----------+----------+
| 유연성     │ 낮음     │ 높음     │
| 낭비       │ 많음     │ 적음     │
| 라우팅     │ 큰 테이블│ 축약     │
+------------+----------+----------+
```

**핵심 알고리즸:** 2^(32-n) 주소 계산

**📢 섹션 요약 비유:** /24 = 256개!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: CIDR 활용**

| 항목 | /8 | /16 | /24 | /32 |
|:---|:---|:---|:---|:---|
| **용도** │ ISP | 대기업 | 소규모 | 단일 |
| **주소수** │ 16M | 65K | 256 | 1 |

**📢 섹션 요약 비유:** 라우팅 축약 = CIDR!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 서브넷 설계 → VLSM
**시나리오 2:** 라우팅 축약 → Supernetting
**시나리오 3:** 주소 할당 → CIDR 블록

**도입 체크리스트:** 주소 요구, 성장, 라우팅

**📢 섹션 요약 비유:** 계획 = CIDR!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 주소 효율, 라우팅 축소
