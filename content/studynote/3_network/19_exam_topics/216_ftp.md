# FTP (File Transfer Protocol) - 파일 전송

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 전송 프로토콜
> 2. **가치**: 대용량 파일 전송, 재개 지원
> 3. **융합**: SFTP, FTPS로 보안 강화

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

FTP(File Transfer Protocol)는 클라이언트와 서버 간에 파일을 전송하는 표준 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FTP (File Transfer Protocol)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FTP 동작 모드                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Active Mode:                                     │   │ │
│   │   │                                                      │   │ │
│   │   │     클라이언트                서버                    │   │ │
│   │   │        │                         │                    │   │ │
│   │   │        │  Command (Port 21)      │                    │   │ │
│   │   │        │ ──────────────────────> │                    │   │ │
│   │   │        │                         │                    │   │ │
│   │   │        │  Data (Port 20 → Client)│                    │   │ │
│   │   │        │ <────────────────────── │                    │   │ │
│   │   │        │     (서버가 연결 시도)   │                    │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Passive Mode:                                    │   │ │
│   │   │                                                      │   │ │
│   │   │     클라이언트                서버                    │   │ │
│   │   │        │                         │                    │   │ │
│   │   │        │  Command (Port 21)      │                    │   │ │
│   │   │        │ ──────────────────────> │                    │   │ │
│   │   │        │                         │                    │   │ │
│   │   │        │  Data (Client → Server) │                    │   │ │
│   │   │        │ ──────────────────────> │                    │   │ │
│   │   │        │     (클라이언트가 연결)  │                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FTP 포트                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  • Port 21: 제어 채널 (Command)                      │   │ │
│   │   │  • Port 20: 데이터 채널 (Active Mode)                │   │ │
│   │   │  • Random: 데이터 채널 (Passive Mode)                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Passive Mode는 방화벽 환경에서 선호됩니다.

**💡 비유**: FTP는 택배 배송입니다. 물건(파일)을 보내고 받습니다.

**📢 섹션 요약 비유:** Passive = 방화벽 친화적!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 명령어 | 기능 |
|:---|:---|
| **USER** | 사용자 이름 |
| **PASS** | 비밀번호 |
| **LIST** | 디렉토리 목록 |
| **RETR** | 다운로드 |
| **STOR** | 업로드 |
| **DELE** | 삭제 |
| **QUIT** | 종료 |

**비교표: 전송 모드**

```
+------------+----------+----------+
| 모드       │ 특징     │ 용도     │
+------------+----------+----------+
| ASCII      │ 텍스트   │ 텍스트   │
| Binary     │ 원본     │ 바이너리 │
+------------+----------+----------+
```

**핵심 알고리즸:** 이중 채널, ASCII/Binary 변환

**📢 섹션 요약 비유:** Binary = 원본 그대로!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 보안 FTP**

| 프로토콜 | 암호화 | 포트 |
|:---|:---|:---|
| **FTP** | X | 21, 20 |
| **FTPS** | SSL/TLS | 990, 989 |
| **SFTP** | SSH | 22 |

**📢 섹션 요약 비유:** SFTP = SSH 기반, FTPS = SSL 기반!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 내부 전송** → FTP
**시나리오 2: 외부 전송** → SFTP/FTPS
**시나리오 3: 대용량** → 이어올리기 (Resume)

**도입 체크리스트:** 보안 요구, 방화벽, 전송 크기

**📢 섹션 요약 비유:** 보안 = SFTP!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대용량 파일 전송

**미래 전망:** SFTP 확대, HTTPS 기반 전송

**참고 표준:** RFC 959

**📢 섹션 요약 비유:** FTP는 파일 전송의 원조!
