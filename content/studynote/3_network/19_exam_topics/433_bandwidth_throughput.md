# 대역폭 vs 처리량 - Bandwidth vs Throughput

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 최대 용량 vs 실제 전송량
> 2. **가치**: 용량 계획, 병목 식별
> 3. **융합**: 오버프로비저닝, Goodput

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Bandwidth는 최대 용량, Throughput은 실제 전송량입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Bandwidth vs Throughput                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              개념 비교                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Bandwidth (대역폭):                                 │   │ │
│   │   │  - 링크의 최대 전송 용량                              │   │ │
│   │   │  - "파이프 굵기"                                      │   │ │
│   │   │  - 예: 1Gbps 링크                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  Throughput (처리량):                                │   │ │
│   │   │  - 실제 달성되는 전송 속도                            │   │ │
│   │   │  - "실제 흐르는 물"                                   │   │ │
│   │   │  - 예: 850Mbps 실제                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Throughput ≤ Bandwidth (항상)                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Throughput 감소 요인                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 요인       │ 영향     │ 비율     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 오버헤드   │ 헤더     │ 5-10%    │               │   │ │
│   │   │  │ 프로토콜   │ TCP/ACK  │ 5-15%    │               │   │ │
│   │   │  | 혼잡       │ 큐잉     │ 가변     │               │   │ │
│   │   │  │ 오류       │ 재전송   │ 가변     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Throughput은 항상 Bandwidth보다 작거나 같습니다.

**💡 비유**: Bandwidth는 도로 폭, Throughput은 실제 지나가는 차량 수입니다.

**📢 섹션 요약 비유:** Throughput = 실제 속도!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Bandwidth** | 최대 용량 |
| **Throughput** | 실제 전송량 |
| **Goodput** | 유용한 데이터 |
| **Utilization** | 활용률 |

**비교표: Goodput vs Throughput**

```
+------------+----------+----------+
| 항목       │ Throughput│ Goodput │
+------------+----------+----------+
| 헤더       │ 포함     │ 제외     │
| 재전송     │ 포함     │ 제외     │
| 의미       │ 전체     │ 유용     │
+------------+----------+----------+
```

**핵심 알고리즸:** 유틸리제이션 계산

**📢 섹션 요약 비유:** Goodput = 진짜 성능!
