# 2PC (Two-Phase Commit) - 2단계 커밋

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 트랜잭션 프로토콜
> 2. **가치**: 원자성 보장
> 3. **융합:** XA, Database, JTA

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

2PC는 분산 트랜잭션의 원자성을 보장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    2PC (Two-Phase Commit)                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              2PC 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │           Coordinator                                │   │ │
│   │   │               │                                      │   │ │
│   │   │    ┌──────────┼──────────┐                          │   │ │
│   │   │    │          │          │                          │   │ │
│   │   │    ↓          ↓          ↓                          │   │ │
│   │   │ ┌─────┐   ┌─────┐   ┌─────┐                        │   │ │
│   │   │ │ DB1 │   │ DB2 │   │ DB3 │                        │   │ │
│   │   │ └─────┘   └─────┘   └─────┘                        │   │ │
│   │   │                                                      │   │ │
│   │   │ Phase 1: Prepare                                    │   │ │
│   │   │  → Coordinator: "준비되었나?"                        │   │ │
│   │   │  → Participants: "예/아니오"                         │   │ │
│   │   │                                                      │   │ │
│   │   │ Phase 2: Commit/Abort                               │   │ │
│   │   │  → 모두 Yes: Commit                                 │   │ │
│   │   │  → 하나라도 No: Abort                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 준비 단계와 커밋 단계로 나뉩니다.

**📢 섹션 요약 비유:** 2PC = 만장일치 투표!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Coordinator** | 트랜잭션 조정자 |
| **Participant** | 참여자 |
| **Prepare** | 준비 단계 |
| **Commit** | 커밋 단계 |
| **Abort** | 중단 |

**핵심 알고리즸:** Prepare → (모두 OK?) → Commit/Abort

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**2PC vs 3PC vs Saga**

| 항목 | 2PC | 3PC | Saga |
|:---|:---|:---|:---|
| **단계** | 2 | 3 | N |
| **차단** | 있음 | 없음 | 없음 |
| **복구** | 복잡 | 복잡 | 단순 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 단일 DB 클러스터 → 2PC
**시나리오 2:** 마이크로서비스 → Saga
**시나리오 3:** 고가용성 필요 → 3PC
