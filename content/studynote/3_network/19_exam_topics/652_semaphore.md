# Semaphore - 세마포어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동기화 기법
> 2. **가치**: 자원 관리, 상호 배제
> 3. **융합:** Mutex, Critical Section, IPC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Semaphore는 자원 접근을 제어하는 동기화 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Semaphore                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              세마포어 동작                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Binary Semaphore (0, 1):      Counting Semaphore:   │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────┐      ┌─────────────────────┐│   │ │
│   │   │  │ S = 1 (Available)   │      │ S = 3 (3 resources) ││   │ │
│   │   │  │ S = 0 (Locked)      │      │ S = 0~N             ││   │ │
│   │   │  └─────────────────────┘      └─────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  Operations:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ P (wait): S--; if S < 0 then block         │    │   │ │
│   │   │  │ V (signal): S++; if S <= 0 then wakeup     │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 세마포어는 P/V 연산으로 자원을 관리합니다.

**📢 섹션 요약 비유:** Semaphore = 입장권!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Wait (P)** | 자원 요청 |
| **Signal (V)** | 자원 해제 |
| **Count** | 카운터 |
| **Queue** | 대기 큐 |
| **Atomic** | 원자적 연산 |

**핵심 알고리즸:** Wait → (자원 있음?) → 진입 / 대기

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Semaphore vs Mutex**

| 항목 | Semaphore | Mutex |
|:---|:---|:---|
| **값** | 0 이상 정수 | 0, 1 |
| **소유권** | 없음 | 있음 |
| **용도** | 자원 관리 | 상호 배제 |
| **해제** | 누구나 | 소유자만 |
