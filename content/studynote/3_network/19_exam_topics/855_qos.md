# QoS - 서비스 품질

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트래픽 우선순위
> 2. **가치**: 성능 보장
> 3. **융합:** DiffServ, IntServ, Traffic Shaping

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

QoS (Quality of Service)는 트래픽에 우선순위를 부여합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    QoS (Quality of Service)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              QoS 모델                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  QoS 계층:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Voice (EF): 최고 우선순위, 지연 <150ms          ││   │ │
│   │   │  │  Video (AF41): 높은 우선순위, 지연 <400ms        ││   │ │
│   │   │  │  Business (AF21): 중간 우선순위                  ││   │ │
│   │   │  │  Best Effort (BE): 기본                          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  DiffServ (DSCP):                                     │   │ │
│   │   │  - IP 헤더 ToS 필드 사용 (6 bits)                      │   │ │
│   │   │  - PHB (Per-Hop Behavior)                             │   │ │
│   │   │  - EF, AF, BE 클래스                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  IntServ:                                              │   │ │
│   │   │  - RSVP (Resource Reservation Protocol)               │   │ │
│   │   │  - 종단간 자원 예약                                    │   │ │
│   │   │  - 확장성 문제                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  메트릭:                                                │   │ │
│   │   │  - 지연 (Latency)                                     │   │ │
│   │   │  - 지터 (Jitter)                                      │   │ │
│   │   │  - 대역폭 (Bandwidth)                                 │   │ │
│   │   │  - 손실률 (Loss Rate)                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: QoS는 실시간 트래픽 품질을 보장합니다.

**📢 섹션 요약 비유:** QoS = 우선 통행권!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**DiffServ vs IntServ**

| 항목 | DiffServ | IntServ |
|:---|:---|:---|
| **방식** | 코어 기반 | 종단간 |
| **확장성** | 높음 | 낮음 |
| **복잡도** | 낮음 | 높음 |
| **보장** | 클래스별 | 흐름별 |
