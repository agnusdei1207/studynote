# Memory Barrier - 메모리 장벽

> 1. **본질**: 메모리 순서 제어

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Memory Barrier는 메모리 연산 순서를 제어합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Memory Barrier                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              메모리 장벽                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  CPU 1:                      CPU 2:                  │   │ │
│   │   │  Store A = 1                 Load B                  │   │ │
│   │   │  ┌─────────────────┐         ┌─────────────────┐    │   │ │
│   │   │  │ Memory Barrier  │         │ Memory Barrier  │    │   │ │
│   │   │  └─────────────────┘         └─────────────────┘    │   │ │
│   │   │  Store B = 1                 Load A                  │   │ │
│   │   │                                                      │   │ │
│   │   │  보장:                                                │   │ │
│   │   │  - Store-Store: 쓰기 순서                            │   │ │
│   │   │  - Load-Load: 읽기 순서                              │   │ │
│   │   │  - Full: 전체 순서                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 메모리 장벽은 CPU와 컴파일러 최적화를 제어합니다.

**📢 섹션 요약 비유:** Memory Barrier = 순서 지킴이!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Acquire** | 읽기 장벽 |
| **Release** | 쓰기 장벽 |
| **Full** | 전체 장벽 |
| **Compiler Barrier** | 컴파일러 장벽 |

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)
