# 적응형 델타 변조 (ADM)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 신호 변화에 따라 스텝 크기 동적 조절
> 2. **가치**: 경사 과부하/과립러 잡음 해결
> 3. **융합**: 음성 통신, 센서 데이터와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ADM(Adaptive Delta Modulation)은 신호의 변화율에 따라 스텝 크기를 동적으로 조절하는 개선된 델타 변조 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ADM (적응형 델타 변조)                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ADM 원리                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  DM 문제:                                                 │   │ │
│   │   │  • 고정 스텝 Δ                                           │   │ │
│   │   │  • 급변: 경사 과부하 (Δ 너무 작음)                        │   │ │
│   │   │  • 완변: 과립러 잡음 (Δ 너무 큼)                          │   │ │
│   │   │                                                      │   │ │
│   │   │  ADM 해결:                                                │   │ │
│   │   │  • 가변 스텝 Δ[n]                                        │   │ │
│   │   │  • 급변 시: Δ 증가 (빠른 추적)                            │   │ │
│   │   │  • 완변 시: Δ 감소 (잡음 감소)                            │   │ │
│   │   │                                                      │   │ │
│   │   │  스텝 적응 알고리즘:                                       │   │ │
│   │   │  • 연속 같은 극성: Δ[n+1] = Δ[n] × P (증가)             │   │ │
│   │   │  • 극성 변화: Δ[n+1] = Δ[n] × Q (감소)                   │   │ │
│   │   │  • 일반적으로 P > 1, Q < 1                               │   │ │
│   │   │                                                      │   │ │
│   │   │  예: P = 1.5, Q = 0.8                                   │   │ │
│   │   │  1→1→1: Δ × 1.5 × 1.5 (증가)                             │   │ │
│   │   │  1→0: Δ × 0.8 (감소)                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              파형 비교                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  원본:     ╱╱╱╱╱───────╱╱╱                          │   │ │
│   │   │            (급변) (완만) (급변)                         │   │ │
│   │   │                                                      │   │ │
│   │   │  DM:      ┌┌┌┌┌┐┐┐┐┌┌┌                              │   │ │
│   │   │           (과부하)(과립러)                             │   │ │
│   │   │                                                      │   │ │
│   │   │  ADM:    ┌─┌─┌───┐──┌─┌─                             │   │ │
│   │   │           (적응)(적응)(적응)                             │   │ │
│   │   │           큰Δ   작은Δ  큰Δ                              │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: ADM은 DM의 두 가지 문제를 동시에 해결합니다.

**💡 비유**: ADM은 운전할 때 커브에서 속도를 조절하는 것입니다.

**📢 섹션 요약 비유**: 같은 방향 연속 = Δ 증가!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **적응기** | 스텝 조절 | P, Q 곱셈 | 가변 |
| **누적기** | 값 복원 | ±Δ[n] | 합산 |
| **메모리** | 이전 비트 | 1비트 저장 | 기억 |
| **스텝** | Δ[n] | 범위 제한 | 크기 |

**비교표**

```
+------------+----------+----------+
| 항목       | DM       | ADM      |
+------------+----------+----------+
| 스텝       | 고정     | 가변     |
| 과부하     | 심함     | 완화     |
| 과립러     | 심함     | 완화     |
| SNR        | 낮음     | 높음     |
| 복잡도     | 낮음     | 중간     |
+------------+----------+----------+
```

**핵심 알고리즘**: 비교 → 이전 비트 확인 → 스텝 조절 → 출력

**📢 섹션 요약 비유**: P > 1, Q < 1!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 음성 부호화**

| 방식 | 비트율 | 품질 |
|:---|:---:|:---:|
| **DM** | 16-32kbps | 낮음 |
| **ADM** | 16-32kbps | 중간 |
| **ADPCM** | 32kbps | 높음 |

**📢 섹션 요약 비유**: ADM은 DM과 ADPCM 중간!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 단순 음성** → ADM
**시나리오 2: 센서** → ADM (저전력)
**시나리오 3: 고품질** → ADPCM

**도입 체크리스트**: 품질, 복잡도, 전력

**📢 섹션 요약 비유**: ADM은 품질/복잡도 균형!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: DM 대비 6-10dB SNR 개선

**미래 전망**: IoT 센서 응용

**참고 표준**: 간단한 음성 시스템

**📢 섹션 요약 비유**: ADM은 적응의 지혜!
