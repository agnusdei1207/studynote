# FEC - 전방 오류 수정

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오류 수정 정보 포함
> 2. **가치**: 재전송 없이 수정
> 3. **융합:** Reed-Solomon, Hamming, LDPC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

FEC (Forward Error Correction)는 미리 오류 수정 정보를 포함합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FEC (Forward Error Correction)                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FEC 방식                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  개념:                                                 │   │ │
│   │   │  원본 데이터 + 중복 비트 → 전송 → 수신측에서 수정        │   │ │
│   │   │                                                      │   │ │
│   │   │  Hamming Code (7,4):                                   │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Data: 4 bit (d1 d2 d3 d4)                       ││   │ │
│   │   │  │ Parity: 3 bit (p1 p2 p3)                        ││   │ │
│   │   │  │ Codeword: 7 bit                                 ││   │ │
│   │   │  │ 1비트 오류 수정 가능                              ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  Reed-Solomon:                                         │   │ │
│   │   │  - 블록 단위 오류 수정                                  │   │ │
│   │   │  - CD, DVD, QR Code                                  │   │ │
│   │   │  - n개 심볼 중 k개 데이터, n-k개 패리티                │   │ │
│   │   │                                                      │   │ │
│   │   │  LDPC (Low-Density Parity-Check):                      │   │ │
│   │   │  - 5G, WiFi 6                                        │   │ │
│   │   │  - 높은 효율                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                 │   │ │
│   │   │  - 재전송 없음 (지연 감소)                              │   │ │
│   │   │  - 단방향 통신 가능                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  단점:                                                 │   │ │
│   │   │  - 오버헤드                                           │   │ │
│   │   │  - 수정 능력 한계                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: FEC는 실시간 통신에서 재전송 지연을 피합니다.

**📢 섹션 요약 비유:** FEC = 예비 타이어!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Parity** | 패리티 |
| **Redundancy** | 중복도 |
| **Code Rate** | 부호율 |
| **Distance** | 거리 |
| **Syndrome** | 신드롬 |

**핵심 알고리즸:** 데이터 → 인코딩 → 전송 → 디코딩 → 수정

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**FEC 방식 비교**

| 방식 | 수정 능력 | 효율 | 용도 |
|:---|:---|:---|:---|
| **Hamming** | 1비트 | 높음 | 메모리 |
| **RS** | 버스트 | 중간 | 미디어 |
| **LDPC** | 높음 | 높음 | 5G |
| **Turbo** | 높음 | 높음 | 4G |

**시나리오 3:** 무선 → LDPC
