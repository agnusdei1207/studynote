# SD-WAN (Software-Defined WAN)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소프트웨어 정의 광역 네트워크
> 2. **가치**: 비용 절감, 유연성, 가시성
> 3. **융합**: MPLS, 인터넷, 클라우드와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SD-WAN은 소프트웨어 정의 네트워킹을 WAN에 적용하여 트래픽을 지능적으로 라우팅하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SD-WAN (Software-Defined WAN)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SD-WAN 구조                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │   [지사 A]                     [지사 B]              │   │ │

│   │   │   │SD-WAN│                     │SD-WAN│               │   │ │
│   │   │   │Edge  │                     │Edge  │               │   │ │
│   │   │   └──┬──┘                     └──┬──┘               │   │ │
│   │   │      │                            │                  │   │ │

│   │   • 애플리케이션 인식 라우팅                                │   │
│   │   • 중앙 집중 관리                                          │   │
│   │   • 빠른 배포 (Zero-Touch Provisioning)                     │   │

│   │   • 품질 보장 (MPLS 대비)                                   │   │
│   │   • 벤더 종속 가능성                                        │   │
│   │                                                      │   │
```

> **해설**: SD-WAN은 기업 WAN의 비용과 복잡성을 줄이는 핵심 기술입니다.

**💡 비유**: SD-WAN은 똑똑한 내비게이션입니다. 교통 상황을 보고 최적 경로를 선택합니다.

**📢 섹션 요약 비유:** SD-WAN = WAN의 지능화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소 | 역할 | 설명 |
|:---|:---|:---|
| **SD-WAN Edge** | CPE | 지사 장비 |
| **Controller** | 제어 | 중앙 관리 |
| **Gateway** | 연결 | 클라우드 게이트웨이 |
| **Orchestrator** | 오케스트레이션 | 정책 배포 |

**비교표**

```
+------------+----------+----------+
| 항목       │ MPLS     │ SD-WAN   │
+------------+----------+----------+
| 비용       │ 높음     │ 낮음     │
│ 배포       │ 느림     │ 빠름     │
│ 유연성     │ 낮음     │ 높음     │
│ 품질       │ 보장     │ Best Effort│
+------------+----------+----------+
```

**핵심 알고리즸:** 애플리케이션 인식 라우팅, 동적 경로 선택

**📢 섹션 요약 비유:** 앱 인식 = SD-WAN 핵심!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 경로 선택**

| 트래픽 | 우선 경로 | 백업 |

| **백업** | 인터넷 | - |

**📢 섹션 요약 비유:** 트래픽에 따른 최적 경로!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 소규모 지사** → 인터넷 전용 SD-WAN
**시나리오 2: 대규모 지사** → MPLS + 인터넷 하이브리드
**시나리오 3: 클라우드 중심** → 클라우드 게이트웨이

**도입 체크리스트:** MPLS 비용, 클라우드 사용, 보안 요구

**📢 섹션 요약 비유:** 하이브리드 = 최적!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 비용 30-50% 절감, 배포 90% 단축

**미래 전망:** SASE (Secure Access Service Edge)로 진화

**참고 표준:** MEF 70, RFC 8300

**📢 섹션 요약 비유:** SD-WAN은 WAN의 미래!
