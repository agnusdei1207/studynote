# STP (Spanning Tree Protocol) - 루프 방지

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 루프 없는 트리 토폴로지 구성
> 2. **가치**: 브로드캐스트 스톰 방지
> 3. **융합**: RSTP, MSTP로 진화

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

STP(Spanning Tree Protocol)는 스위치 네트워크에서 루프를 방지하기 위해 논리적으로 포트를 차단하는 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    STP (Spanning Tree Protocol)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              STP 동작 원리                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Root Bridge 선출:                                │   │ │
│   │   │     • 가장 낮은 Bridge ID 우선                       │   │ │
│   │   │     • Bridge ID = Priority + MAC                    │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Root Port 선출:                                  │   │ │
│   │   │     • Root Bridge까지 최소 비용 포트                 │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Designated Port 선출:                            │   │ │
│   │   │     • 각 세그먼트당 하나                              │   │ │
│   │   │                                                      │   │ │
│   │   │  4. Non-Designated Port 차단:                        │   │ │
│   │   │     • 루프 방지                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              포트 상태                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Blocking → Listening → Learning → Forwarding       │   │ │
│   │   │     ↓          20s         15s        15s           │   │ │
│   │   │                                                      │   │ │
│   │   │  • Blocking: BPDU 수신만                            │   │ │
│   │   │  • Listening: BPDU 송수신                            │   │ │
│   │   │  • Learning: MAC 학습                                │   │ │
│   │   │  • Forwarding: 정상 동작                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: STP는 BPDU를 교환하여 루프 없는 토폴로지를 구성합니다.

**💡 비유**: STP는 교차로 신호등입니다. 충돌을 방지하기 위해 한쪽을 정지시킵니다.

**📢 섹션 요약 비유:** STP = 루프 방지!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 | 기본값 |
|:---|:---|:---|
| **Bridge ID** | Priority + MAC | 32768 |
| **Path Cost** | 대역폭 기반 | 19 (100M) |
| **Hello Time** | BPDU 주기 | 2초 |
| **Max Age** | BPDU 만료 | 20초 |
| **Forward Delay** | 상태 전이 | 15초 |

**비교표: STP 버전**

```
+------------+----------+----------+
| 버전       │ 표준     │ 특징     │
+------------+----------+----------+
| STP        │ 802.1D   │ 느림     │
| RSTP       │ 802.1w   │ 빠름     │
| MSTP       │ 802.1s   │ 다중     │
+------------+----------+----------+
```

**핵심 알고리즸:** BPDU, Root Bridge 선출

**📢 섹션 요약 비유:** RSTP = 1초 컨버전스!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 이중화** → STP + 포트 채널
**시나리오 2: 빠른 복구** → RSTP
**시나리오 3: 다중 VLAN** → MSTP

**도입 체크리스트:** Root Bridge 수동 설정, PortFast, BPDU Guard

**📢 섹션 요약 비유:** RSTP + PortFast = 빠른 복구!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)
