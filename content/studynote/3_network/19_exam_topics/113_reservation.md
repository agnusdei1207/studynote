# 예약 방식 접속 (Reservation Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 전 채널 예약으로 충돌 방지
> 2. **가치**: 높은 효율, QoS 보장
> 3. **융합**: 위성, LTE, 5G와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

예약 방식 접속(Reservation Access)은 데이터 전송 전에 채널을 예약하여 충돌 없이 전송하는 제어 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    예약 방식 접속 (Reservation Access)              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              예약 방식 동작                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  기본 구조:                                              │   │ │
│   │   │                                                      │   │ │
│   │   │  시간                                                 │   │ │
│   │   │  ──────────────────────────────────────────────→      │   │ │
│   │   │  ├─────┼─────┼─────┼─────┼─────┼─────┤             │   │ │
│   │   │  │예약  │예약  │ DATA │ DATA │ DATA │ DATA │             │   │ │
│   │   │  │슬롯  │슬롯  │ CH1 │ CH2 │ CH3 │ CH4 │             │   │ │
│   │   │  └─────┴─────┴─────┴─────┴─────┴─────┘             │   │ │
│   │   │         ↑                          ↑                  │   │ │
│   │   │    예약 프레임              데이터 전송                 │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                 │   │ │
│   │   │  • 예약 프레임으로 슬롯 할당 요청                        │   │ │
│   │   │  • 중앙 제어 또는 분산 제어                              │   │ │
│   │   │  • 충돌 없이 데이터 전송                                 │   │ │
│   │   │  • 오버헤드 존재 (예약 프레임)                           │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                   │   │ │

│   │   │  • 예약 오버헤드                                         │   │ │

│   │   │  • 제어 복잡도                                           │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │

│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 중앙 집중형:                                         │   │ │
│   │   │     • 중앙 스케줄러가 예약 관리                           │   │ │
│   │   │     • 위성, LTE, 5G                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 분산형:                                               │   │ │
│   │   │     • 노드 간 협력으로 예약                                │   │ │

│   │   │                                                      │   │ │
│   │   │  3. 하이브리드:                                           │   │ │
```

> **해설**: 예약 방식은 높은 부하 환경에서 효율적입니다.

**💡 비유**: 예약 방식은 식당에서 미리 자리를 예약하는 것입니다.

**📢 섹션 요약 비유**: 예약 = 충돌 없는 전송!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **예약 프레임** | 요청 | 작은 패킷 | 예약 |
| **스케줄러** | 할당 | 중앙/분산 | 관리자 |
| **슬롯** | 전송 단위 | 고정/가변 | 자리 |

**비교표**

```
+------------+----------+----------+
| 항목       | 경쟁 방식| 예약 방식|
+------------+----------+----------+
| 충돌       | 있음     | 없음     |
| 효율       | 낮음     | 높음     |
| 지연       | 낮음     | 높음     |
| QoS        | 어려움   | 가능     |
| 복잡도     | 낮음     | 높음     |
+------------+----------+----------+
```

**핵심 알고리즸**: 예약 요청 → 스케줄링 → 슬롯 할당 → 데이터 전송

**📢 섹션 요약 비유**: LTE/5G는 예약 기반!
