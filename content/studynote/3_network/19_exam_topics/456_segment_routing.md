# 세그먼트 라우팅 - Segment Routing

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소스 라우팅 기반 트래픽 엔지니어링
> 2. **가치**: 단순성, 확장성, 유연성
> 3. **융합**: MPLS, SRv6, TI-LFA

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

세그먼트 라우팅은 소스에서 경로를 지정하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    세그먼트 라우팅                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              세그먼트 라우팅 개념                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │

│   │   │  │ Src │───►│ S1  │───►│ S2  │───►│ Dst │         │   │ │
│   │   │  │     │    │     │    │     │    │     │         │   │ │
│   │   │  └─────┘    └─────┘    └─────┘    └─────┘         │   │ │

│   │   │  Segment List: [S1, S2, Dst]                        │   │ │
│   │   │                                                      │   │ │

│   │   │  - S2: 인접 세그먼트 (링크)                          │   │ │
│   │   │  - Dst: 최종 목적지                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              세그먼트 유형                                     │ │

│   │   │  | Service    │ 서비스   │ Function │               │   │ │
```

> **해설**: 세그먼트 라우팅은 소스가 경로를 결정합니다.

**💡 비유**: 세그먼트 라우팅은 GPS 내비게이션입니다.

**📢 섹션 요약 비유:** Segment Routing = 소스 라우팅!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SID** | Segment Identifier |
| **SRGB** | Segment Routing Global Block |
| **Segment List** | 경로 목록 |
| **TI-LFA** | Topology Independent LFA |

**비교표: SR-MPLS vs SRv6**

```
+------------+----------+----------+----------+
| 항목       │ SR-MPLS  │ SRv6     │ 비고     │
+------------+----------+----------+----------+
| 캡슐화     │ 레이블   │ IPv6     │ 헤더     │
| SID 크기   │ 20bit    │ 128bit   │ 식별자   │
| 호환성     │ 높음     │ 낮음     │ 기존     │
| 확장성     │ 중간     │ 높음     │ 미래     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 최단 경로, FRR

**📢 섹션 요약 비유:** TI-LFA = 50ms 복구!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 기존 TE vs SR**

| 항목 | RSVP-TE | Segment Routing |
|:---|:---|:---|
| **상태** | 경로별 | 소스만 |
| **확장성** | 낮음 | 높음 |
| **복잡성** | 높음 | 낮음 |

**📢 섹션 요약 비유:** 상태 없는 TE!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 백본 → SR-MPLS
**시나리오 2:** 데이터센터 → SRv6
**시나리오 3:** 5G → SR 정책

**도입 체크리스트:** IGP 지원, SRGB, 정책

**📢 섹션 요약 비유:** 중앙 정책 = 트래픽 엔지니어링!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 네트워크 단순화

**미래 전망:** SRv6, SFC

**참고 표준:** RFC 8402, RFC 8754

**📢 섹션 요약 비유:** 세그먼트 라우팅은 네트워크의 미래!
