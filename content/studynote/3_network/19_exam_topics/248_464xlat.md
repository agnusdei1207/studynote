# 464XLAT - IPv4 over IPv6 변환

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPv4 앱이 IPv6 네트워크에서 동작
> 2. **가치**: 앱 수정 없이 IPv6 전환
> 3. **융합**: CLAT + PLAT 구조

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

464XLAT는 IPv6 전용 네트워크에서 IPv4 애플리케이션이 동작할 수 있게 하는 변환 기술입니다.

> **해설**: 464XLAT는 IPv4 앱이 IPv6 네트워크에서 투명하게 동작합니다.

**💡 비유**: 464XLAT는 어댑터입니다. 다른 플러그를 사용할 수 있게 합니다.

**📢 섹션 요약 비유:** 464XLAT = 레거시 앱 구원!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **CLAT** | 클라이언트 측 변환 |
| **PLAT** | 제공자 측 NAT64 |
| **IPv4 Prefix** | 로컬 할당 |
| **IPv6 Prefix** | 네트워크 제공 |

**비교표: 트래픽 흐름**

```
+------------+----------+----------+
| 단계       │ 프로토콜 │ 장비     │
+------------+----------+----------+
| 앱→CLAT    │ IPv4     │ 로컬     │
| CLAT→PLAT  │ IPv6     │ 터널     │
| PLAT→서버  │ IPv4     │ NAT64    │
+------------+----------+----------+
```

**핵심 알고리즸:** SIIT (Stateless IP/ICMP Translation)

**📢 섹션 요약 비유:** CLAT = 클라이언트, PLAT = 제공자!
