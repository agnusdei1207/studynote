# HSRP (Hot Standby Router Protocol) - Cisco 게이트웨이 이중화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Cisco 전용 게이트웨이 이중화
> 2. **가치**: Active/Standby 구조
> 3. **융합**: VRRP, GLBP와 비교

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HSRP(Hot Standby Router Protocol)는 Cisco 전용 FHRP로, 여러 라우터를 하나의 가상 라우터로 구성합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    HSRP (Hot Standby Router Protocol)                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HSRP 구성                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │         Virtual IP: 192.168.1.1                     │   │ │
│   │   │         Virtual MAC: 00:00:0c:07:ac:XX             │   │ │
│   │   │                    ↑                                │   │ │
│   │   │         ┌─────────┴─────────┐                      │   │ │
│   │   │         │                   │                      │   │ │
│   │   │    [Router A]          [Router B]                  │   │ │
│   │   │    Active              Standby                     │   │ │
│   │   │    Priority: 110      Priority: 100                │   │ │
│   │   │    Real IP: .2        Real IP: .3                  │   │ │
│   │   │                                                      │   │ │
│   │   │  상태:                                                │   │ │
│   │   │  • Initial → Listen → Speak → Standby → Active     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HSRP 상태 전이                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  상태     │ 설명                                    │   │ │
│   │   │  ────────┼─────────────────────────────────────│   │ │
│   │   │  Initial │ 비활성                                  │   │ │
│   │   │  Learn   │ Virtual IP 학습                         │   │ │
│   │   │  Listen  │ Hello 수신 대기                         │   │ │
│   │   │  Speak   │ Active/Standby 선출 참여                │   │ │
│   │   │  Standby │ 백업                                    │   │ │
│   │   │  Active  │ 활성 (가상 IP/MAC 소유)                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HSRP는 Priority가 높은 라우터가 Active가 됩니다.

**💡 비유**: HSRP는 경호원 교대입니다. 한 명이 활동하고 다른 한 명이 대기합니다.

**📢 섹션 요약 비유:** HSRP = Cisco 전용!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 값 |
|:---|:---|
| **멀티캐스트** | 224.0.0.2 |
| **UDP 포트** | 1985 |
| **Virtual MAC** | 00:00:0c:07:ac:XX |
| **기본 Priority** | 100 |
| **Hello** | 3초 |
| **Hold** | 10초 |

**비교표: HSRP 버전**

```
+------------+----------+----------+
| 항목       │ HSRPv1   │ HSRPv2   │
+------------+----------+----------+
| 그룹 수    │ 0-255    │ 0-4095   │
| MD5 인증   │ X        │ O        │
| IPv6       │ X        │ O        │
+------------+----------+----------+
```

**핵심 알고리즸:** Priority 선출, Preempt, Tracking

**📢 섹션 요약 비유:** Tracking = 인터페이스 장애 시 Priority 감소!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FHRP**

| 항목 | HSRP | VRRP | GLBP |
|:---|:---|:---|:---|
| **표준** | Cisco | IEEE | Cisco |
| **로드밸런싱** | X | X | O |
| **상태** | Active/Standby | Master/Backup | AVG/AVF |

**📢 섹션 요약 비유:** 로드밸런싱 = GLBP!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: Cisco 환경** → HSRP
**시나리오 2: 멀티벤더** → VRRP
**시나리오 3: 로드밸런싱** → GLBP

**도입 체크리스트:** Priority, Preempt, Tracking, 인증

**📢 섹션 요약 비유:** MD5 인증 = 권장!
