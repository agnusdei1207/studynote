# Out-of-order Execution - 비순차 실행

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어 순서 변경 실행
> 2. **가치**: 파이프라인 활용률
> 3. **융합:** ROB, Reservation Station, ILP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Out-of-order Execution은 의존성 없는 명령어를 먼저 실행합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Out-of-order Execution                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              비순차 실행 구조                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  In-order:     I1 → I2 → I3 → I4 → I5              │   │ │
│   │   │                                                      │   │ │
│   │   │  Dependencies: I2 waits for I1                      │   │ │
│   │   │                 I4 waits for I3                      │   │ │
│   │   │                 I3, I5 independent                   │   │ │
│   │   │                                                      │   │ │
│   │   │  Out-of-order: I1 → I3 → I2 → I5 → I4              │   │ │
│   │   │                 (I3, I5을 먼저 실행)                  │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌───────────────────────────────────────────────┐  │   │ │
│   │   │  │ ROB (Reorder Buffer) - 순서 보장              │  │   │ │
│   │   │  │ In-order Commit                               │  │   │ │
│   │   │  └───────────────────────────────────────────────┘  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 비순차 실행은 순차적 커밋으로 결과를 보장합니다.

**📢 섹션 요약 비유:** Out-of-order = 작업 재배치!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **ROB** | Reorder Buffer |
| **Reservation Station** | 예약 스테이션 |
| **Register Renaming** | 레지스터 재명명 |
| **Load/Store Queue** | 로드/스토어 큐 |
| **Commit** | 순차 커밋 |

**핵심 알고리즸:** 인출 → 재명명 → 스케줄 → 실행 → 커밋

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**In-order vs Out-of-order**

| 항목 | In-order | Out-of-order |
|:---|:---|:---|
| **복잡도** | 낮음 | 높음 |
| **전력** | 낮음 | 높음 |
| **성능** | 낮음 | 높음 |
| **용도** | 임베디드 | 고성능 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 고성능 → Out-of-order
**시나리오 2:** 전력 효율 → In-order
**시나리오 3:** 실시간 → In-order
