# AES - Advanced Encryption Standard

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 블록 암호 표준 (128비트 블록)
> 2. **가치**: 보안성, 효율성, 하드웨어 지원
> 3. **융합**: TLS, WiFi, 디스크 암호화

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

AES는 128비트 블록을 암호화하는 대칭키 블록 암호입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    AES                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              AES 암호화 과정                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [128비트 평문]                                     │   │ │
│   │   │        │                                             │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  ┌─────────────────────────────────┐                │   │ │
│   │   │  │ Round 1: SubBytes → ShiftRows  │                │   │ │
│   │   │  │          → MixCols → AddKey    │                │   │ │
│   │   │  └─────────────────────────────────┘                │   │ │
│   │   │        │                                             │   │ │
│   │   │        ▼ (10/12/14 라운드 반복)                       │   │ │
│   │   │  ┌─────────────────────────────────┐                │   │ │
│   │   │  │ Final Round (MixCols 없음)      │                │   │ │
│   │   │  └─────────────────────────────────┘                │   │ │
│   │   │        │                                             │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  [128비트 암호문]                                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              AES 변종                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 변종       │ 키 길이   │ 라운드   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ AES-128    │ 128비트  │ 10       │               │   │ │
│   │   │  │ AES-192    │ 192비트  │ 12       │               │   │ │
│   │   │  | AES-256    │ 256비트  │ 14       │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: AES는 10-14 라운드의 치환-전치 네트워크를 사용합니다.

**💡 비유**: AES는 강력한 금고입니다. 키가 없으면 열 수 없습니다.

**📢 섹션 요약 비유:** AES = 암호화 표준!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SubBytes** | 바이트 치환 (S-box) |
| **ShiftRows** | 행 이동 |
| **MixColumns** | 열 혼합 |
| **AddRoundKey** | 라운드 키 XOR |

**비교표: AES 모드**

```
+------------+----------+----------+
| 항목       │ ECB      │ CBC      │
+------------+----------+----------+
| 패턴       │ 보임     │ 숨김     │
| IV         │ 없음     │ 필요     │
| 병렬       │ 가능     │ 불가     │
| 보안       │ 낮음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** 갈루아 필드 연산, S-box

**📢 섹션 요약 비유:** GCM = 권장 모드!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 블록 암호**

| 항목 | AES | DES | ChaCha20 |
|:---|:---|:---|:---|
| **블록** │ 128비트 │ 64비트 | 스트림 |
| **키** │ 128-256 │ 56 | 256 |
| **상태** │ 표준 | 레거시 | 현대적 |

**📢 섹션 요약 비유:** WPA2/WPA3 = AES!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** TLS → AES-256-GCM
**시나리오 2:** 디스크 암호화 → AES-256-XTS
**시나리오 3:** WiFi → AES-CCMP

**도입 체크리스트:** 키 길이, 모드 선택, IV/Nonce 관리

**📢 섹션 요약 비유:** 현대 암호 = AES!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 강력한 기밀성

**미래 전망:** AES-NI 하드웨어 가속

**참고 표준:** FIPS 197

**📢 섹션 요약 비유:** AES는 대칭키의 왕!
