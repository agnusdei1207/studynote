# RADIUS - 원격 인증 다이얼인 사용자 서비스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: AAA 프로토콜
> 2. **가치**: 중앙 집중식 인증
> 3. **융합:** UDP 1812/1813, 802.1X

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RADIUS (Remote Authentication Dial-In User Service)는 AAA 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RADIUS                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RADIUS 구조                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  AAA (Authentication, Authorization, Accounting):       │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  ┌────────────┐         ┌────────────┐          ││   │ │
│   │   │  │  │   NAS      │         │  RADIUS    │          ││   │ │
│   │   │  │  │(Network    │  UDP    │  Server    │          ││   │ │
│   │   │  │  │Access      │◄───────►│            │          ││   │ │
│   │   │  │  │Server)     │ 1812/3  │            │          ││   │ │
│   │   │  │  └────────────┘         └────────────┘          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  AAA 기능:                                        ││   │ │
│   │   │  │  ┌─────────────────────────────────────────┐    ││   │ │
│   │   │  │  │ Authentication: 사용자 인증              │    ││   │ │
│   │   │  │  │ Authorization: 권한 부여                 │    ││   │ │
│   │   │  │  │ Accounting: 사용 기록                   │    ││   │ │
│   │   │  │  └─────────────────────────────────────────┘    ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  패킷 유형:                                        ││   │ │
│   │   │  │  - Access-Request (인증 요청)                     ││   │ │
│   │   │  │  - Access-Accept (인증 성공)                      ││   │ │
│   │   │  │  - Access-Reject (인증 실패)                      ││   │ │
│   │   │  │  - Accounting-Request/Response                    ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RADIUS는 네트워크 접근 제어의 핵심 프로토콜입니다.

**📢 섹션 요약 비유:** RADIUS = 신분 확인소!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **NAS** | Network Access Server |
| **RADIUS Server** | 인증 서버 |
| **Shared Secret** | 공유 비밀키 |
| **Attributes** | 속성 (AVP) |
| **Vendor-Specific** | 벤더 확장 |

**핵심 알고리듬:** 요청 → 공유 비밀키 검증 → 사용자 인증 → 응답

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**RADIUS vs TACACS+**

| 항목 | RADIUS | TACACS+ |
|:---|:---|:---|
| **전송** | UDP | TCP |
| **포트** | 1812/1813 | 49 |
| **암호화** | 비밀번호만 | 전체 |
| **AAA 분리** | 없음 | 있음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 무선 → RADIUS + 802.1X
**시나리오 2:** VPN → RADIUS 인증
**시나리오 3:** 장비 관리 → TACACS+
