# SAML - Security Assertion Markup Language

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: XML 기반 인증/인가 프레임워크
> 2. **가치**: 기업 SSO 표준
> 3. **융합**: 페더레이션, IdP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SAML은 XML 기반의 인증 및 인가 프레임워크입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SAML                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SAML SSO 흐름                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [User]                                             │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 1. SP 접근                                     │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [SP] ──────▶ [IdP] 리다이렉트                        │   │ │
│   │   │                   │                                  │   │ │
│   │   │                   │ 2. 인증                          │   │ │
│   │   │                   ▼                                  │   │ │
│   │   │  [User] ◀─────────┘                                  │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 3. SAML Assertion                              │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [SP] ──────▶ [인증 완료]                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SAML Assertion                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 구성요소   │ 내용     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Subject    │ 사용자   │ 식별     │               │   │ │
│   │   │  │ Conditions │ 조건     │ 유효성   │               │   │ │
│   │   │  | AuthnStmt  │ 인증정보 │ 방식     │               │   │ │
│   │   │  │ AttrStmt   │ 속성     │ 권한     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SAML은 IdP와 SP 간의 신뢰 관계를 기반으로 합니다.

**💡 비유**: SAML은 보증서입니다. 신뢰할 수 있는 기관이 보증합니다.

**📢 섹션 요약 비유:** SAML = 기업 SSO!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **IdP** | Identity Provider |
| **SP** | Service Provider |
| **Assertion** | 인증/인가 정보 |
| **Metadata** | 설정 정보 |

**비교표: SAML vs OIDC**

```
+------------+----------+----------+
| 항목       │ SAML     │ OIDC     │
+------------+----------+----------+
| 형식       │ XML      │ JSON/JWT │
| 용도       │ 기업 SSO │ 웹/모바일│
| 복잡도     │ 높음     │ 낮음     │
| 바인딩     │ 다양     │ HTTP     │
+------------+----------+----------+
```

**핵심 알고리즸:** XML 서명 검증, Assertion 파싱

**📢 섹션 요약 비유:** 기업 = SAML!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: SSO 프로토콜**

| 항목 | SAML | OIDC | Kerberos |
|:---|:---|:---|:---|
| **형식** │ XML │ JSON │ 티켓 |
| **기업** │ 적합 │ 중간 │ 적합 |
| **웹** │ 적합 │ 적합 │ 제한 |

**📢 섹션 요약 비유:** 페더레이션 = SAML!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 기업 SSO → SAML 2.0
**시나리오 2:** 클라우드 서비스 → SAML + OIDC
**시나리오 3:** 레거시 시스템 → SAML Bridge

**도입 체크리스트:** IdP 선택, 메타데이터 교환, 속성 매핑

**📢 섹션 요약 비유:** 엔터프라이즈 = SAML!
