# Fibre Channel - FC 스토리지

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고속 스토리지 네트워크
> 2. **가치**: 높은 성능, 안정성
> 3. **융합**: SAN, 가상화와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Fibre Channel(FC)은 고속 스토리지 네트워크를 위한 전용 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Fibre Channel                                     │
├─────────────────────────────────────────────────────────────────────────┤

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Server]      [Server]      [Server]               │   │ │

│   │   │     │             │             │                   │   │ │

│   │   │            │                                        │   │ │
│   │   │     ┌──────▼──────┐                                 │   │ │
```

> **해설**: FC는 전용 네트워크로 높은 성능과 안정성을 제공합니다.

**💡 비유**: FC는 스토리지 전용 고속도로입니다.

**📢 섹션 요약 비유:** FC = 고성능 SAN!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **HBA** | 호스트 버스 어댑터 |
| **WWPN** | 포트 식별자 |
| **WWNN** | 노드 식별자 |
| **Fabric** | 스위치 네트워크 |

**비교표: FC 토폴로지**

```
+------------+----------+----------+
| 토폴로지   │ 설명     │ 용도     │
+------------+----------+----------+
| Point-to-Point│ 직접  │ 소규모   │
| FC-AL      │ 루프     │ 레거시   │
| Fabric     │ 스위치   │ 엔터프라이즈│
+------------+----------+----------+
```

**핵심 알고리즸:** FSPF, Zoning

**📢 섹션 요약 비유:** Zoning = 접근 제어!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FC vs iSCSI**

| 항목 | FC | iSCSI |
|:---|:---|:---|
| **성능** | 높음 | 중간 |
| **비용** | 높음 | 낮음 |
| **관리** | 복잡 | 쉬움 |

**📢 섹션 요약 비유:** FC = 엔터프라이즈!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 미션 크리티컬 DB** → FC
**시나리오 2: 대규모 가상화** → FC
**시나리오 3: 일반 스토리지** → iSCSI

**도입 체크리스트:** 성능, 비용, 관리

**📢 섹션 요약 비유:** FCoE = FC over Ethernet!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 고성능, 안정성

**미래 전망:** NVMe over FC

**참고 표준:** ANSI INCITS

**📢 섹션 요약 비유:** FC는 고성능 SAN 표준!
