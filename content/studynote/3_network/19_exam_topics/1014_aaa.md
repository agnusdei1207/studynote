# AAA - 인증 인가 계정

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 보안 프레임워크
> 2. **가치**: 접근 제어 체계
> 3. **융합:** RADIUS, TACACS+, LDAP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

AAA (Authentication, Authorization, Accounting)는 접근 제어 프레임워크입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    AAA                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              AAA 프레임워크                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  3가지 요소:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  1. Authentication (인증)                        ││   │ │
│   │   │  │     "누구인가?"                                  ││   │ │
│   │   │  │     - 사용자 ID, 비밀번호                         ││   │ │
│   │   │  │     - 인증서, 토큰                               ││   │ │
│   │   │  │     - 생체인식                                  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  2. Authorization (인가)                         ││   │ │
│   │   │  │     "무엇을 할 수 있는가?"                         ││   │ │
│   │   │  │     - 권한 부여                                  ││   │ │
│   │   │  │     - 역할 기반 (RBAC)                           ││   │ │
│   │   │  │     - 접근 범위 제한                              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  3. Accounting (계정)                            ││   │ │
│   │   │  │     "무엇을 했는가?"                              ││   │ │
│   │   │  │     - 접속 기록                                  ││   │ │
│   │   │  │     - 명령 기록                                  ││   │ │
│   │   │  │     - 사용량 기록                                ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: AAA는 보안 접근 제어의 기본 원칙입니다.

**📢 섹션 요약 비유:** AAA = 신분증 + 권한증 + 활동일지!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Authentication** | 신원 확인 |
| **Authorization** | 권한 부여 |
| **Accounting** | 활동 기록 |
| **Method List** | 인증 방법 목록 |
| **Server** | AAA 서버 |

**핵심 알고리듬:** 인증 → 인가 → 접근 → 계정 기록

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**AAA 프로토콜 비교**

| 프로토콜 | 용도 | 특징 |
|:---|:---|:---|
| **RADIUS** | 사용자 인증 | UDP |
| **TACACS+** | 장비 관리 | TCP |
| **LDAP** | 디렉토리 | TCP |
| **Kerberos** | SSO | 티켓 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 사용자 VPN → RADIUS
**시나리오 2:** 장비 접속 → TACACS+
**시나리오 3:** 무선 LAN → RADIUS + 802.1X
