# Saga Pattern - 사가 패턴

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 트랜잭션 패턴
> 2. **가치**: 데이터 일관성, 보상
> 3. **융합:** Microservices, Event Driven, Choreography

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Saga는 분산 트랜잭션을 보상 트랜잭션으로 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Saga Pattern                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Saga 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  정상 흐름:                                          │   │ │
│   │   │  ┌─────┐    ┌─────┐    ┌─────┐                     │   │ │
│   │   │  │Order│───→│Pay  │───→│Ship │ = 성공              │   │ │
│   │   │  │ T1  │    │ T2  │    │ T3  │                     │   │ │
│   │   │  └─────┘    └─────┘    └─────┘                     │   │ │
│   │   │                                                      │   │ │
│   │   │  실패 시 보상:                                       │   │ │
│   │   │  ┌─────┐    ┌─────┐    ┌─────┐                     │   │ │
│   │   │  │Order│←───│Pay  │←───│Ship │ = 실패              │   │ │
│   │   │  │ C1  │    │ C2  │    │ Err │                     │   │ │
│   │   │  │(보상)│    │(보상)│    │     │                     │   │ │
│   │   │  └─────┘    └─────┘    └─────┘                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 각 단계에 보상 트랜잭션을 정의합니다.

**📢 섹션 요약 비유:** Saga = 되돌리기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Local Transaction** | 로컬 트랜잭션 |
| **Compensating Tx** | 보상 트랜잭션 |
| **Choreography** | 분산 조정 |
| **Orchestration** | 중앙 조정 |
| **Saga Coordinator** | 사가 코디네이터 |

**핵심 알고리즸:** T1 → T2 → ... → Tn (실패 시 보상 실행)

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Choreography vs Orchestration**

| 항목 | Choreography | Orchestration |
|:---|:---|:---|
| **조정** | 분산 | 중앙 |
| **복잡도** | 단순 | 복잡 |
| **의존성** | 낮음 | 높음 |
| **가시성** | 낮음 | 높음 |
