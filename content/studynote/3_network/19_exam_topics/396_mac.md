# MAC - 메시지 인증 코드

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 키 기반 메시지 무결성/인증
> 2. **가치**: 변조 탐지, 송신자 인증
> 3. **융합**: TLS, IPsec, 파일 무결성

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MAC(Message Authentication Code)는 키를 사용하여 메시지 무결성을 보장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MAC                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MAC 생성 및 검증                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [송신자]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 메시지 + 비밀키                                 │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [MAC 함수] ────▶ [MAC 값]                          │   │ │
│   │   │     │                    │                           │   │ │
│   │   │     │ 메시지 + MAC      │                           │   │ │
│   │   │     └────────────────────┼─── 전송 ───▶             │   │ │
│   │   │                          │                           │   │ │
│   │   │  [수신자]                 │                           │   │ │
│   │   │     │                    │                           │   │ │
│   │   │     │ 같은 키로 MAC 재계산                            │   │ │
│   │   │     ▼                    ▼                           │   │ │
│   │   │  [MAC 검증] ◀───── [수신된 MAC]                      │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [일치/불일치]                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MAC 종류                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 종류       │ 기반     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ HMAC-SHA256│ 해시     │ TLS/IPsec│               │   │ │
│   │   │  │ CMAC       │ 블록암호 │ 파일     │               │   │ │
│   │   │  | GMAC       │ GCM      │ TLS 1.3  │               │   │ │
│   │   │  │ Poly1305   │ 다항식   │ ChaCha20 │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: MAC은 무결성과 인증을 동시에 제공합니다.

**💡 비유**: MAC은 봉투의 밀랍입니다. 열었는지 알 수 있습니다.

**📢 섹션 요약 비유:** MAC = 무결성 + 인증!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Message** | 원본 메시지 |
| **Key** | 공유 비밀키 |
| **MAC Algorithm** | MAC 함수 |
| **Tag** | MAC 값 |

**비교표: MAC vs Hash vs 서명**

```
+------------+----------+----------+
| 항목       │ MAC      │ 서명     │
+------------+----------+----------+
| 키         │ 대칭     │ 비대칭   │
| 인증       │ 공유자   │ 공개     │
| 부인방지   │ 불가     │ 가능     │
| 속도       │ 빠름     │ 느림     │
+------------+----------+----------+
```

**핵심 알고리즸:** HMAC: H(K⊕opad || H(K⊕ipad || m))

**📢 섹션 요약 비유:** TLS = MAC!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: MAC 알고리즘**

| 항목 | HMAC-SHA256 | GMAC | Poly1305 |
|:---|:---|:---|:---|
| **기반** │ SHA-256 | GCM | 다항식 |
| **속도** │ 중간 | 빠름 | 매우 빠름 |
| **TLS** │ 광범위 | 1.3 | 1.3 |

**📢 섹션 요약 비유:** 인증 암호 = MAC!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** TLS 통신 → HMAC 또는 AEAD
**시나리오 2:** API 인증 → HMAC
**시나리오 3:** 파일 무결성 → HMAC-SHA256

**도입 체크리스트:** 알고리즘 선택, 키 관리, 타이밍 공격 방지

**📢 섹션 요약 비유:** 메시지 인증 = MAC!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 무결성 + 인증

**미래 전망:** AEAD로 통합

**참고 표준:** FIPS 198-1 (HMAC), RFC 2104

**📢 섹션 요약 비유:** MAC은 메시지 보안!
