# OpenAPI - API 명세 표준

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: RESTful API 명세 표준
> 2. **가치**: 문서화, 코드 생성, 테스트
> 3. **융합**: Swagger, REST, JSON

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

OpenAPI(구 Swagger)는 RESTful API를 명세하는 표준입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OpenAPI                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OpenAPI 명세 구조                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  openapi: 3.0.0                                      │   │ │
│   │   │  info:                                                │   │ │
│   │   │    title: Sample API                                 │   │ │
│   │   │    version: 1.0.0                                    │   │ │
│   │   │  paths:                                               │   │ │
│   │   │    /users:                                            │   │ │
│   │   │      get:                                             │   │ │
│   │   │        summary: Get all users                         │   │ │
│   │   │        responses:                                     │   │ │
│   │   │          '200':                                       │   │ │
│   │   │            description: Success                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OpenAPI 구성 요소                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 요소       │ 설명     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ openapi    │ 버전     │ 명세     │               │   │ │
│   │   │  │ info       │ 정보     │ 메타     │               │   │ │
│   │   │  | paths      │ 경로     │ 엔드포인트│               │   │ │
│   │   │  │ components │ 구성요소 │ 스키마   │               │   │ │
│   │   │  │ security   │ 보안     │ 인증     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: OpenAPI는 YAML/JSON으로 REST API를 명세합니다.

**💡 비유**: OpenAPI는 API 설명서입니다. 어떻게 쓰는지 알려줍니다.

**📢 섹션 요약 비유:** OpenAPI = REST 명세!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Paths** | 경로 정의 |
| **Operations** | HTTP 메서드 |
| **Parameters** | 파라미터 |
| **Responses** | 응답 |

**비교표: OpenAPI vs WSDL**

```
+------------+----------+----------+
| 항목       │ OpenAPI  │ WSDL     │
+------------+----------+----------+
| 포맷       │ YAML/JSON│ XML      │
| 프로토콜   │ REST     │ SOAP     │
| 도구       │ Swagger UI│_wsimport │
+------------+----------+----------+
```

**핵심 알고리즸:** YAML, JSON Schema

**📢 섹션 요약 비유:** Swagger UI = 대화형 문서!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: API 문서화**

| 항목 | OpenAPI | GraphQL Schema | RAML |
|:---|:---|:---|:---|
| **스타일** │ REST │ GraphQL │ REST |
| **형식** │ YAML/JSON │ SDL │ YAML |
| **도구** │ 풍부 │ Apollo │ 제한 |

**📢 섹션 요약 비유:** REST = OpenAPI!
