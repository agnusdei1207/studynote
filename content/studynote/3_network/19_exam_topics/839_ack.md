# ACK - 확인 응답

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 수신 확인
> 2. **가치**: 신뢰성 보장
> 3. **융합:** TCP ACK, Cumulative, Selective

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ACK (Acknowledgment)는 데이터 수신을 확인합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ACK (Acknowledgment)                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ACK 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  기본 동작:                                            │   │ │
│   │   │  Sender                     Receiver                 │   │ │
│   │   │  ┌───────┐                  ┌───────┐               │   │ │
│   │   │  │       │ ── Seg 1,2,3 ──→ │       │               │   │ │
│   │   │  │       │                  │       │               │   │ │
│   │   │  │       │ ←── ACK 4 ────── │       │               │   │ │
│   │   │  │       │   (4번 기대)      │       │               │   │ │
│   │   │  └───────┘                  └───────┘               │   │ │
│   │   │                                                      │   │ │
│   │   │  ACK 유형:                                            │   │ │
│   │   │  - Cumulative ACK: 다음 기대 시퀀스                   │   │ │
│   │   │  - Selective ACK (SACK): 수신된 블록 정보             │   │ │
│   │   │  - Duplicate ACK: 같은 ACK 반복                       │   │ │
│   │   │  - Delayed ACK: 지연 후 한 번에                       │   │ │
│   │   │                                                      │   │ │
│   │   │  ACK 번호 의미:                                        │   │ │
│   │   │  ACK = N → N-1까지 모두 수신 완료                      │   │ │
│   │   │                                                      │   │ │
│   │   │  TCP 플래그: ACK 비트                                  │   │ │
│   │   │  연결 수립 후 모든 세그먼트에 설정                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ACK는 신뢰적 전송의 핵심 메커니즘입니다.

**📢 섹션 요약 비유:** ACK = 수령증!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **ACK Number** | 확인 번호 |
| **ACK Flag** | ACK 플래그 |
| **Cumulative** | 누적 ACK |
| **SACK** | 선택적 ACK |
| **Delayed** | 지연 ACK |

**핵심 알고리즸:** 수신 → ACK 생성 → 전송

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ACK 방식 비교**

| 방식 | 특징 | 효율 |
|:---|:---|:---|
| **Cumulative** | 순차적 | 낮음 |
| **SACK** | 선택적 | 높음 |
| **Delayed** | 지연 | 중간 |
| **Immediate** | 즉시 | 낮음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일반 → Cumulative ACK
**시나리오 2:** 손실 많음 → SACK
**시나리오 3:** 효율 → Delayed ACK
