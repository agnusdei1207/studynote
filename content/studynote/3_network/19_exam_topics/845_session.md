# Session - 세션

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 지속적 통신 맥락
> 2. **가치**: 상태 유지
> 3. **융합:** HTTP Session, TCP Session, Stateful

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Session은 지속적인 통신 상태입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Session                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              세션 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  TCP 세션:                                             │   │ │
│   │   │  - 연결 수립부터 종료까지                               │   │ │
│   │   │  - State: SYN_SENT, ESTABLISHED, FIN_WAIT            │   │ │
│   │   │                                                      │   │ │
│   │   │  HTTP 세션:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Client              Server                      ││   │ │
│   │   │  │   │                   │                         ││   │ │
│   │   │  │   │ ── Request 1 ────→│                         ││   │ │
│   │   │  │   │ ←─ Response + Session ID ─                  ││   │ │
│   │   │  │   │                   │                         ││   │ │
│   │   │  │   │ ── Request + Cookie ──→                     ││   │ │
│   │   │  │   │ ←─ Response (Stateful) ─                    ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  세션 관리:                                            │   │ │
│   │   │  - Cookie: 클라이언트 저장                             │   │ │
│   │   │  - Session ID: 서버 저장                               │   │ │
│   │   │  - Token: JWT, OAuth                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 세션은 무상태 프로토콜에 상태를 부여합니다.

**📢 섹션 요약 비유:** Session = 방문 카드!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Session ID** | 세션 식별자 |
| **Cookie** | 쿠키 |
| **Token** | 토큰 |
| **State** | 상태 |
| **Timeout** | 타임아웃 |

**핵심 알고리즸:** 로그인 → 세션 생성 → 유지 → 로그아웃

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**세션 저장 방식**

| 방식 | 저장 위치 | 보안 |
|:---|:---|:---|
| **Cookie** | 클라이언트 | 낮음 |
| **Server Session** | 서버 | 높음 |
| **JWT** | 클라이언트 | 중간 |
| **Redis** | 외부 | 높음 |
