# VRRP (Virtual Router Redundancy Protocol) - 라우터 이중화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 라우터로 게이트웨이 이중화
> 2. **가치**: 고가용성, 무중단 서비스
> 3. **융합**: HSRP, GLBP와 비교

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

VRRP(Virtual Router Redundancy Protocol)는 여러 라우터를 하나의 가상 라우터로 구성하여 게이트웨이 이중화를 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    VRRP (Virtual Router Redundancy Protocol)         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VRRP 구성                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │         Virtual IP: 192.168.1.1                     │   │ │
│   │   │         Virtual MAC: 00:00:5e:00:01:XX             │   │ │
│   │   │                    ↑                                │   │ │
│   │   │         ┌─────────┴─────────┐                      │   │ │
│   │   │         │                   │                      │   │ │
│   │   │    [Router A]          [Router B]                  │   │ │
│   │   │    Master              Backup                      │   │ │
│   │   │    Priority: 100      Priority: 90                 │   │ │
│   │   │    Real IP: .2        Real IP: .3                  │   │ │
│   │   │         │                   │                      │   │ │
│   │   │         └─────────┬─────────┘                      │   │ │
│   │   │                   ↓                                │   │ │
│   │   │              [Hosts]                               │   │ │
│   │   │           GW: 192.168.1.1                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VRRP 상태                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  상태       │ 설명                                  │   │ │
│   │   │  ──────────┼───────────────────────────────────│   │ │
│   │   │  Initialize│ 시작                                  │   │ │
│   │   │  Backup    │ 대기                                  │   │ │
│   │   │  Master    │ 활성 (가상 IP/MAC 소유)               │   │ │
│   │   │                                                      │   │ │
│   │   │  광고: VRRP Advertisement (멀티캐스트 224.0.0.18)   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: VRRP는 Priority가 높은 라우터가 Master가 됩니다.

**💡 비유**: VRRP는 대리인 제도입니다. 주인이 없으면 대리인이 역할을 수행합니다.

**📢 섹션 요약 비유:** Priority 높음 = Master!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 값 |
|:---|:---|
| **멀티캐스트** | 224.0.0.18 |
| **프로토콜** | IP 112 |
| **Virtual MAC** | 00:00:5e:00:01:VRID |
| **기본 Priority** | 100 |
| **광고 간격** | 1초 |

**비교표: FHRP (First Hop Redundancy Protocol)**

```
+------------+----------+----------+
| 프로토콜   │ 표준     │ 특징     │
+------------+----------+----------+
| VRRP       │ IEEE     │ 표준     │
| HSRP       │ Cisco    │ Cisco    │
| GLBP       │ Cisco    │ 로드밸런싱│
+------------+----------+----------+
```

**핵심 알고리즸:** Priority 기반 선출, Preempt

**📢 섹션 요약 비유:** Preempt = 우선권 자동 복귀!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: VRRP vs HSRP**

| 항목 | VRRP | HSRP |
|:---|:---|:---|
| **표준** | IEEE | Cisco |
| **용어** | Master/Backup | Active/Standby |
| **Virtual IP** | 실제 IP 가능 | 별도 필요 |
| **멀티캐스트** | 224.0.0.18 | 224.0.0.2 |

**📢 섹션 요약 비유:** VRRP = 표준, HSRP = Cisco!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 기업 게이트웨이** → VRRP
**시나리오 2: Cisco 환경** → HSRP
**시나리오 3: 로드밸런싱** → GLBP

**도입 체크리스트:** Priority, Preempt, 인증

**📢 섹션 요약 비유:** Preempt = 권장!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 고가용성, 무중단

**미래 전망:** IPv6 VRRP (VRRPv3)

**참고 표준:** RFC 5798

**📢 섹션 요약 비유:** VRRP는 게이트웨이 이중화의 표준!
