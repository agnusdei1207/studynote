# Lock-free - 락 프리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 없는 동시성
> 2. **가치**: 블로킹 없음, 성능
> 3. **융합:** CAS, Atomic, Wait-free

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Lock-free는 락 없이 동시성을 달성합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Lock-free                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Lock-free 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Lock-based:              Lock-free:                 │   │ │
│   │   │  ┌─────────────────┐     ┌─────────────────────┐    │   │ │
│   │   │  │ Thread A        │     │ Thread A            │    │   │ │
│   │   │  │ Lock → Wait →   │     │ CAS (retry) →       │    │   │ │
│   │   │  │ Critical →      │     │ Success             │    │   │ │
│   │   │  │ Unlock          │     │                     │    │   │ │
│   │   │  └─────────────────┘     └─────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                │   │ │
│   │   │  - Deadlock 없음                                     │   │ │
│   │   │  - 스레드 장애 영향 없음                              │   │ │
│   │   │  - 높은 동시성                                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Lock-free는 CAS 등으로 락 없이 동시성을 보장합니다.

**📢 섹션 요약 비유:** Lock-free = 예약 없이!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Lock-free vs Wait-free**

| 항목 | Lock-free | Wait-free |
|:---|:---|:---|
| **보장** | 전체 진행 | 개별 완료 |
| **복잡도** | 보통 | 높음 |
| **성능** | 좋음 | 더 좋음 |
| **구현 난이도** | 보통 | 높음 |
