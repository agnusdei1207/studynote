# tcpdump - CLI 패킷 캡처

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CLI 패킷 캡처 도구
> 2. **가치**: 심층 패킷 분석
> 3. **융합**: Wireshark, 분석과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

tcpdump는 명령줄 기반 패킷 캡처 도구입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    tcpdump                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              tcpdump 사용법                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  $ tcpdump -i eth0                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  옵션:                                              │   │ │
│   │   │  • -i: 인터페이스                                │   │ │
│   │   │  • -n: 패킷 수                                │   │ │
│   │   │  • -w: 쓰기 (pcap)                               │   │ │
│   │   │  • -r: 읽기 (pcap)                               │   │ │
│   │   │  • -v: 상세 출력                                │   │ │
│   │   │  • -A: 모든 패킷                               │   │ │
│   │   │  • host, port, proto 필터                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 옵션                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +--------------------+------------------------+           │   │ │
│   │   │  │ 옵션                │ 설명                   │           │   │ │
│   │   │  +--------------------+------------------------+           │   │ │
│   │   │  │ -i eth0            │ eth0 인터페이스          │           │   │ │
│   │   │  │ -n 100             │ 100개 패킷만              │           │   │ │
│   │   │  │ -w capture.pcap     │ 파일로 저장               │           │   │ │
│   │   │  │ -r capture.pcap     │ 파일에서 읽기              │           │   │ │
│   │   │  │ -v                 │ 상세 출력                │           │   │ │
│   │   │  │ -A                 │ ARP 해상만              │           │   │ │
│   │   │  │ host 192.168.1.1 │ 호스트 필터              │           │   │ │
│   │   │  │ port 80            │ 포트 필터                │           │   │ │
│   │   │  │ tcp                │ TCP만                  │           │   │ │
│   │   │  +--------------------+------------------------+           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: tcpdump는 BPF 필터를 사용하여 패킷을 캡처합니다.

**💡 비유**: tcpdump는 CLI 현미경입니다. 터미널에서 패킷을 봅니다.

**📢 섹션 요약 비유:** tcpdump = CLI 캡처!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **libpcap** | 라이브러리 |
| **BPF** | 필터 |
| **Filter** | 필터 표현식 |
| **Snapshot** | 길이 제한 |

**비교표: 필터 표현식**

```
+------------------------+------------------------+
| 필터                   │ 설명                   |
+------------------------+------------------------+
| host 192.168.1.1       │ 호스트 필터              |
| net 192.168.1.0/24      │ 네트워크 필터              |
| port 80               │ 포트 필터                |
| tcp                 │ TCP만                  |
| udp                 │ UDP만                  |
| icmp                │ ICMP만                 |
| arp                 │ ARP만                 |
+------------------------+------------------------+
```

**핵심 알고리즸:** BPF, promiscuous mode

**📢 섹션 요약 비유:** host 192.168.1.1 = IP 필터!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: tcpdump vs Wireshark**

| 항목 | tcpdump | Wireshark |
|:---|:---|:---|
| **인터페이스** | CLI | GUI |
| **실시간** | 빠름 | 느림 |
| **분석** | 기본 | 상세 |

**📢 섹션 요약 비유:** tcpdump + Wireshark = 최강 조합!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 서버 캡처** → tcpdump -w
**시나리오 2: 실시간 분석** → tcpdump -i eth0
**시나리오 3: 사후 분석** → tcpdump -w → Wireshark

**도입 체크리스트:** root 권한, 인터페이스, 저장

**📢 섹션 요약 비유:** -w = pcap 저장!
