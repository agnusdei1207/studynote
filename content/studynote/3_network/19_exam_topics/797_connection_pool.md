# Connection Pool - 커넥션 풀

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연결 재사용
> 2. **가치**: 성능, 자원 관리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Connection Pool은 연결을 재사용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Connection Pool                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              커넥션 풀 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Without Pool:                                        │   │ │
│   │   │  Request 1: Connect → Query → Close                  │   │ │
│   │   │  Request 2: Connect → Query → Close  (느림)           │   │ │
│   │   │                                                      │   │ │
│   │   │  With Pool:                                           │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │          Connection Pool                     │    │   │ │
│   │   │  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐   │    │   │ │
│   │   │  │  │ C1  │ │ C2  │ │ C3  │ │ C4  │ │ C5  │   │    │   │ │
│   │   │  │  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘   │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       ↑ Borrow      │ Return                       │   │ │
│   │   │       │              ↓                              │   │ │
│   │   │  Request 1: Get → Query → Return                    │   │ │
│   │   │  Request 2: Get → Query → Return  (빠름)             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 커넥션 풀은 연결 생성 비용을 줄입니다.

**📢 섹션 요약 비유:** Connection Pool = 택시 대기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Min Size** | 최소 연결 |
| **Max Size** | 최대 연결 |
| **Idle Timeout** | 유휴 타임아웃 |
| **Wait Timeout** | 대기 타임아웃 |
| **Validation** | 연결 검증 |

**핵심 알고리즸:** 요청 → 풀에서 대여 → 사용 → 반납
