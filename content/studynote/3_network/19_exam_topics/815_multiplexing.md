# Multiplexing - 멀티플렉싱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 신호 전송
> 2. **가치**: 자원 효율성
> 3. **융합:** FDM, TDM, HTTP/2

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Multiplexing은 하나의 채널에 여러 신호를 전송합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Multiplexing                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              멀티플렉싱 기법                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  FDM (Frequency Division):                           │   │ │
│   │   │  주파수 분할                                          │   │ │
│   │   │  ┌─────┬─────┬─────┐                                │   │ │
│   │   │  │ f1  │ f2  │ f3  │  ← 서로 다른 주파수              │   │ │
│   │   │  └─────┴─────┴─────┘                                │   │ │
│   │   │  라디오, TV                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  TDM (Time Division):                                │   │ │
│   │   │  시간 분할                                            │   │ │
│   │   │  ┌───┬───┬───┬───┐                                  │   │ │
│   │   │  │t1 │t2 │t3 │t1 │  ← 시간 슬롯                      │   │ │
│   │   │  └───┴───┴───┴───┘                                  │   │ │
│   │   │  디지털 통신                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  WDM (Wavelength Division):                          │   │ │
│   │   │  파장 분할 (광섬유)                                    │   │ │
│   │   │  CWDM, DWDM                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 멀티플렉싱은 대역폭 효율을 극대화합니다.

**📢 섹션 요약 비유:** Multiplexing = 고속도로 차선!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Mux** | 멀티플렉서 |
| **Demux** | 디멀티플렉서 |
| **Channel** | 채널 |
| **Slot** | 슬롯 |

**핵심 알고리즸:** 다중 입력 → 결합 → 전송 → 분리 → 다중 출력
