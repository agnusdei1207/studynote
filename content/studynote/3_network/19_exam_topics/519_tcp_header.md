### Ⅰ. 개요 (Context & Background)

**개념 정의**

TCP Header는 20바이트 기본 + 옵션 구조입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TCP Header (20+ bytes)                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TCP 헤더 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  0         15        16            31               │   │ │
│   │   │  ├──────────────────┼─────────────────────────────┤ │   │ │
│   │   │  │  Source Port     │  Destination Port            │ │   │ │
│   │   │  ├──────────────────────────────────────────────┤ │   │ │
│   │   │  │            Sequence Number                     │ │   │ │
│   │   │  ├──────────────────────────────────────────────┤ │   │ │
│   │   │  │            Acknowledgment Number              │ │   │ │
│   │   │  ├────┬────┬─────────────────────────────────┤ │   │ │
│   │   │  │Offset│Res│ Flags │ Window Size            │ │   │ │
│   │   │  ├──────────────────────────────────────────────┤ │   │ │
│   │   │  │  Checksum       │  Urgent Pointer            │ │   │ │
│   │   │  ├──────────────────────────────────────────────┤ │   │ │
│   │   │  │            Options (if present)               │ │   │ │
│   │   │  └─────────────────────────────────────────────┘   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: TCP Header는 신뢰적 전송을 위한 제어 정보를 담습니다.

**📢 섹션 요약 비유:** TCP Header = 패킷 명세서!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 필드 | 크기 | 설명 |
|:---|:---|:---|
| **Source Port** | 16bit | 송신 포트 |
| **Dest Port** | 16bit | 수신 포트 |
| **Seq** | 32bit | 시퀀스 번호 |
| **ACK** | 32bit | 승인 번호 |
| **Offset** | 4bit | 헤더 길이 |
| **Flags** | 6bit | 제어 플래그 |
| **Window** | 16bit | 윈도우 크기 |
| **Checksum** | 16bit | 오류 검사 |
| **Urgent** | 16bit | 긴급 포인터 |

**핵심 알고리즸:** 헤더 파싱 → 제어 수행
