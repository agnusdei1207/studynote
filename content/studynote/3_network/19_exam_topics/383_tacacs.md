# TACACS+ - 터미널 액세스 제어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Cisco AAA 프로토콜
> 2. **가치**: 장비 관리, 권한 분리
> 3. **융합**: 네트워크 장비, 관리자 인증

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TACACS+(Terminal Access Controller Access-Control System Plus)는 AAA 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TACACS+                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TACACS+ AAA                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Admin]                                             │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 1. 장비 접속                                   │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [Network Device] ────▶ [TACACS+ Server]            │   │ │
│   │   │                           │                          │   │ │
│   │   │                           │ Authentication           │   │ │
│   │   │                           │ Authorization            │   │ │
│   │   │                           │ Accounting               │   │ │
│   │   │                           ▼                          │   │ │
│   │   │                      [접속 허용/거부]                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TACACS+ 패킷 타입                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 패킷 타입  │ 용도     │ 방향     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Authen Start│ 인증시작│ 클→서   │               │   │ │

```

> **해설**: TACACS+는 전체 패킷을 암호화합니다.

**💡 비유**: TACACS+는 관리자 출입증입니다. 누가 언제 무엇을 했는지 기록합니다.

**📢 섹션 요약 비유:** TACACS+ = 장비 관리!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Authentication** | 사용자 인증 |
| **Authorization** | 권한 부여 |
| **Accounting** | 활동 기록 |
| **TCP 49** | 프로토콜 포트 |

**비교표: TACACS+ vs RADIUS**

```
+------------+----------+----------+
| 항목       │ TACACS+  │ RADIUS   │
+------------+----------+----------+
| 프로토콜    │ TCP      │ UDP      │
| 암호화     │ 전체     │ 비밀번호만│
| AAA 분리   │ 지원     │ 제한     │
| 장비관리   │ 적합     │ 제한     │
+------------+----------+----------+
```

**핵심 알고리즸:** MD5 암호화, 세션 관리

**📢 섹션 요약 비유:** Cisco = TACACS+!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: AAA 프로토콜**

| 항목 | TACACS+ | RADIUS | LDAP |
|:---|:---|:---|:---|
| **용도** │ 장비 │ 사용자 │ 디렉터리 |
| **프로토콜** │ TCP │ UDP │ TCP |
| **암호화** │ 전체 │ 부분 │ SSL/TLS |

**📢 섹션 요약 비유:** 장비 관리 = TACACS+!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
