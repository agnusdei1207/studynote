# Blue-Green Deployment - 블루-그린 배포

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 두 환경 교체 배포
> 2. **가치**: 무중단, 빠른 롤백
> 3. **융합:** Load Balancer, CI/CD, Infrastructure

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Blue-Green Deployment는 두 환경을 교체합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Blue-Green Deployment                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              블루-그린 배포 구조                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Before:                                              │   │ │
│   │   │                                                       │   │ │
│   │   │       Users                                          │   │ │
│   │   │         │                                            │   │ │
│   │   │         ↓                                            │   │ │
│   │   │  ┌─────────────┐                                     │   │ │
│   │   │  │   Router    │                                     │   │ │
│   │   │  └──────┬──────┘                                     │   │ │
│   │   │         │ 100%                                       │   │ │
│   │   │         ↓                                            │   │ │
│   │   │  ┌─────────────┐    ┌─────────────┐                  │   │ │
│   │   │  │    Blue     │    │    Green    │                  │   │ │
│   │   │  │  (v1 Live)  │    │  (v2 Idle)  │                  │   │ │
│   │   │  └─────────────┘    └─────────────┘                  │   │ │
│   │   │                                                      │   │ │
│   │   │  After:                                               │   │ │
│   │   │         │ 0%          │ 100%                          │   │ │
│   │   │         ↓             ↓                               │   │ │
│   │   │  ┌─────────────┐    ┌─────────────┐                  │   │ │
│   │   │  │    Blue     │    │    Green    │                  │   │ │
│   │   │  │  (v1 Idle)  │    │  (v2 Live)  │                  │   │ │
│   │   │  └─────────────┘    └─────────────┘                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 블루-그린 배포는 스위치만으로 즉시 전환합니다.

**📢 섹션 요약 비유:** Blue-Green = 교대 근무!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Blue** | 현재 버전 |
| **Green** | 새 버전 |
| **Router** | 라우터 |
| **Switch** | 전환 |
| **Rollback** | 롤백 |

**핵심 알고리즸:** Green 배포 → 테스트 → 스위치 → Blue → 롤백 가능

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**배포 전략 비교**

| 전략 | 중단 | 롤백 | 리소스 |
|:---|:---|:---|:---|
| **Blue-Green** | 없음 | 빠름 | 2배 |
| **Rolling** | 없음 | 느림 | 1배 |
| **Canary** | 없음 | 중간 | 1배+ |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
