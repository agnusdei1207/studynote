# EIGRP (Enhanced IGRP) - 고급 거리 벡터

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고급 거리 벡터 + 링크 상태 특성
> 2. **가치**: 매우 빠른 컨버전스, 부분 업데이트
> 3. **융합**: Cisco 전용 → 표준화

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

EIGRP(Enhanced Interior Gateway Routing Protocol)는 Cisco가 개발한 고급 거리 벡터 라우팅 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    EIGRP (Enhanced IGRP)                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              EIGRP 특징                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Advanced Distance Vector:                        │   │ │
│   │   │     • Distance Vector + Link State 특성             │   │ │
│   │   │     • 이웃 관계 형성                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  2. DUAL (Diffusing Update Algorithm):               │   │ │
│   │   │     • Loop-Free 경로 보장                            │   │ │
│   │   │     • Feasible Successor                             │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 복합 메트릭:                                      │   │ │
│   │   │     • K1: Bandwidth                                  │   │ │
│   │   │     • K3: Delay                                      │   │ │
│   │   │     • 기본: BW + Delay                               │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 부분 업데이트:                                    │   │ │
│   │   │     • 변경 시에만 전송                               │   │ │
│   │   │     • 대역폭 절약                                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DUAL 용어                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  • FD (Feasible Distance): 목적지까지 총 비용       │   │ │
│   │   │  • AD (Advertised Distance): 이웃이 광고한 비용     │   │ │
│   │   │  • Successor: 현재 최적 경로                        │   │ │
│   │   │  • FS (Feasible Successor): 백업 경로               │   │ │
│   │   │                                                      │   │ │
│   │   │  FS 조건: AD < FD(Successor)                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: EIGRP는 Feasible Successor를 미리 계산하여 매우 빠른 컨버전스를 제공합니다.

**💡 비유**: EIGRP는 예비 경로를 미리 준비하는 내비게이션입니다.

**📢 섹션 요약 비유:** EIGRP = Cisco 전용, 빠름!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 | 기본값 |
|:---|:---|:---|
| **AD** | 신뢰도 | 90 (내부) |
| **Hello** | 이웃 확인 | 5초 |
| **Hold** | 이웃 만료 | 15초 |
| **K 값** | 메트릭 가중치 | K1=1, K3=1 |

**비교표: 메트릭 구성**

```
+------------+----------+----------+
| K 값       │ 구성 요소 │ 기본    │
+------------+----------+----------+
| K1         │ BW       │ 1       │
| K2         │ Load     │ 0       │
| K3         │ Delay    │ 1       │
| K4         │ Reliability│ 0     │
| K5         │ MTU      │ 0       │
+------------+----------+----------+
```

**핵심 알고리즸:** DUAL, Feasible Condition

**📢 섹션 요약 비유:** FS = 백업 경로!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: EIGRP vs OSPF**

| 항목 | EIGRP | OSPF |
|:---|:---|:---|
| **유형** | Adv DV | Link State |
| **계층** | 선택 | 필수 |
| **컨버전스** | 매우 빠름 | 빠름 |
| **표준** | 부분 | 완전 |

**📢 섹션 요약 비유:** Cisco 환경 = EIGRP!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: Cisco 환경** → EIGRP
**시나리오 2: 멀티벤더** → OSPF
**시나리오 3: WAN** → EIGRP 스터브

**도입 체크리스트:** Cisco 장비, K 값 일치, 자동 요약

**📢 섹션 요약 비유:** K 값 = 반드시 일치!
