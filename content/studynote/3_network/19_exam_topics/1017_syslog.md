# Syslog - 시스템 로그

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 로그 전송 프로토콜
> 2. **가치**: 중앙 집중식 로그
> 3. **융합:** Facility, Severity, UDP 514

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Syslog는 시스템 로그를 중앙 서버로 전송합니다.

**📢 섹션 요약 비유:** Syslog = 블랙박스!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Facility** | 로그 출처 |
| **Severity** | 심각도 |
| **Priority** | 우선순위 |
| **Timestamp** | 시간 |
| **Message** | 메시지 |

**핵심 알고리듬:** 이벤트 발생 → Priority 계산 → UDP 전송 → 저장

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Severity 레벨**

| Level | Severity | 용도 |
|:---|:---|:---|
| **0** | Emergency | 시스템 사용 불가 |
| **1** | Alert | 즉시 조치 필요 |
| **2** | Critical | 심각한 상황 |
| **3** | Error | 에러 |
| **4** | Warning | 경고 |
| **5** | Notice | 정상적 중요 이벤트 |
| **6** | Informational | 정보 |
| **7** | Debug | 디버깅 |
