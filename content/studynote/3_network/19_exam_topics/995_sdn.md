# SDN - 소프트웨어 정의 네트워킹

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제어부와 데이터부 분리
> 2. **가치**: 프로그래밍 가능한 네트워크
> 3. **융합:** OpenFlow, Controller, NFV

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SDN (Software-Defined Networking)은 네트워크 제어를 중앙화합니다.

**📢 섹션 요약 비유:** SDN = 네트워크의 운영체제!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Controller** | 중앙 제어기 |
| **OpenFlow** | 남향 API |
| **Northbound API** | 북향 API |
| **Data Plane** | 데이터 평면 |
| **Control Plane** | 제어 평면 |

**핵심 알고리듬:** 패킷 수신 → 컨트롤러 문의 → 플로우 규칙 설치 → 전달
