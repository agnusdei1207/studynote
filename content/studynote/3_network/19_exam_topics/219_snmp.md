# SNMP (Simple Network Management Protocol) - 네트워크 관리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 장비 모니터링/관리
> 2. **가치**: 중앙 관리, 자동 경보
> 3. **융합**: MIB, OID, Trap과 통합

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SNMP(Simple Network Management Protocol)는 네트워크 장비를 모니터링하고 관리하는 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SNMP (Simple Network Management)                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SNMP 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Manager]                    [Agent]                │   │ │
│   │   │  (NMS)                        (장비)                 │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────┐                ┌─────────┐             │   │ │
│   │   │  │         │  GET/SET       │         │             │   │ │
│   │   │  │ Manager │ ─────────────> │  Agent  │             │   │ │
│   │   │  │         │                │         │             │   │ │
│   │   │  │         │ <───────────── │         │             │   │ │
│   │   │  │         │   RESPONSE     │   MIB   │             │   │ │
│   │   │  │         │                │         │             │   │ │
│   │   │  │         │ <───────────── │         │             │   │ │
│   │   │  │         │   TRAP         │         │             │   │ │
│   │   │  └─────────┘                └─────────┘             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SNMP 버전                                         │ │

│   └──────────────────────────────────────────────────────────────┘ │

└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SNMP v3는 인증과 암호화를 모두 제공합니다.

**💡 비유**: SNMP는 원격 관리 시스템입니다. 모든 장비를 한곳에서 감시합니다.

**📢 섹션 요약 비유:** SNMP v3 = 보안!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| PDU | 방향 | 용도 |
|:---|:---|:---|
| **GET** | Manager→Agent | 정보 요청 |
| **GET-NEXT** | Manager→Agent | 다음 OID |
| **SET** | Manager→Agent | 설정 변경 |
| **RESPONSE** | Agent→Manager | 응답 |
| **TRAP** | Agent→Manager | 이벤트 |

**비교표: 포트**

```
+------------+----------+----------+
| 항목       │ 포트     │ 프로토콜 │
+------------+----------+----------+
| 요청       │ 161      │ UDP      │
| Trap       │ 162      │ UDP      │
+------------+----------+----------+
```

**핵심 알고리즸:** MIB 탐색, OID 계층

**📢 섹션 요약 비유:** OID = 1.3.6.1...!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: MIB/OID**

| 항목 | 설명 |
|:---|:---|
| **MIB** | 관리 정보 베이스 |
| **OID** | 객체 식별자 |
| **Community** | 비밀번호 (v1/v2c) |

**📢 섹션 요약 비유:** Community = v1/v2c 비밀번호!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 모니터링** → SNMP v2c (내부)
**시나리오 2: 보안** → SNMP v3
**시나리오 3: 경보** → TRAP 설정

**도입 체크리스트:** 버전, Community, MIB

**📢 섹션 요약 비유:** v3 = 보안 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 중앙 관리, 자동 경보

**미래 전망:** NETCONF/YANG, RESTCONF

**참고 표준:** RFC 1157 (v1), RFC 3416 (v2c), RFC 3414 (v3)

**📢 섹션 요약 비유:** SNMP는 네트워크 관리의 표준!
