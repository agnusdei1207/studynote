# TCP PSH (Push) - 푸시 플래그

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 수신 애플리케이션에 즉시 전달
> 2. **가치**: 지연 없는 데이터 처리
> 3. **융합:** TCP Flags, 버퍼링

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

PSH 플래그는 수신 데이터를 즉시 애플리케이션에 전달합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TCP PSH (Push)                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PSH 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  송신측                    수신측                    │   │ │
│   │   │     │                         │                      │   │ │
│   │   │     │ ─── Data (PSH=1) ────→ │                      │   │ │
│   │   │     │                         │                      │   │ │
│   │   │     │                         │ TCP 버퍼             │   │ │
│   │   │     │                         │   ↓                  │   │ │
│   │   │     │                         │ 즉시 App 전달        │   │ │
│   │   │     │                         │ (버퍼링 안함)        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: PSH는 버퍼링을 우회하여 즉시 전달합니다.

**📢 섹션 요약 비유:** PSH = 즉시 배달!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **PSH=1** | 즉시 전달 요청 |
| **버퍼** | 무시 |
| **용도** | 실시간 데이터 |
| **결합** | 보통 마지막 세그먼트 |

**핵심 알고리즸:** PSH 수신 → 즉시 App push

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**TCP Flags**

| 플래그 | 용도 |
|:---|:---|
| **SYN** | 연결 시작 |
| **ACK** | 승인 |
| **FIN** | 연결 종료 |
| **RST** | 연결 리셋 |
| **PSH** | 즉시 전달 |
| **URG** | 긴급 데이터 |
