# DNS Record Types - DNS 레코드 타입

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: DNS 데이터 유형
> 2. **가치**: 다양한 리소스 매핑
> 3. **융합:** A, AAAA, CNAME, MX, TXT

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DNS 레코드는 도메인 정보를 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DNS Record Types                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 레코드 예시                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  A      www.example.com.  IN A     93.184.216.34     │   │ │
│   │   │  AAAA   www.example.com.  IN AAAA  2606:2800:220:1:: │   │ │
│   │   │  CNAME  mail.example.com IN CNAME www.example.com    │   │ │
│   │   │  MX     example.com     IN MX  10 mail.example.com   │   │ │
│   │   │  NS     example.com     IN NS  ns1.example.com       │   │ │
│   │   │  TXT    example.com     IN TXT "v=spf1 -all"         │   │ │
│   │   │  PTR    1.2.3.4         IN PTR host.example.com      │   │ │
│   │   │  SOA    example.com     IN SOA ns1.admin. 1 3600 ... │   │ │
│   │   │  SRV    _sip._tcp       IN SRV 10 60 5060 sip_srv    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 각 레코드 타입은 특정 목적을 가집니다.

**📢 섹션 요약 비유:** DNS 레코드 = 명함!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 타입 | 용도 |
|:---|:---|
| **A** | IPv4 주소 |
| **AAAA** | IPv6 주소 |
| **CNAME** | 별칭 |
| **MX** | 메일 서버 |
| **NS** | 네임서버 |
| **PTR** | 역방향 조회 |
| **SOA** | 권한 시작 |
| **SRV** | 서비스 위치 |
| **TXT** | 텍스트 정보 |

**핵심 알고리즸:** 레코드 타입별 처리

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**레코드 용도별 분류**

| 분류 | 레코드 |
|:---|:---|
| **주소** | A, AAAA |
| **별칭** | CNAME |
| **서비스** | MX, SRV |
| **관리** | NS, SOA |
| **확인** | TXT, PTR |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹사이트 → A 레코드
**시나리오 2:** 이메일 → MX 레코드
**시나리오 3:** SPF/DKIM → TXT 레코드
