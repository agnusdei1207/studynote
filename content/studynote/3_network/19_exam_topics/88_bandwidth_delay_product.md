# Bandwidth-Delay Product - 대역폭-지연 곱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파이프 용량
> 2. **가치**: 최적 윈도우 크기
> 3. **융합:** TCP Window, Long Fat Network

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Bandwidth-Delay Product는 링크에 채울 수 있는 데이터 양입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Bandwidth-Delay Product (BDP)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              BDP 계산                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  공식:                                                │   │ │
│   │   │  BDP = Bandwidth × RTT                               │   │ │
│   │   │                                                      │   │ │
│   │   │  예시:                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Bandwidth: 100 Mbps                             ││   │ │
│   │   │  │ RTT: 100 ms                                     ││   │ │
│   │   │  │ BDP = 100 Mbps × 0.1 s = 10 Mb = 1.25 MB       ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  의미:                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Sender ══════════════════════════════ Receiver ││   │ │
│   │   │  │         │←────── RTT ──────→│                   ││   │ │
│   │   │  │         │  1.25 MB 데이터    │                   ││   │ │
│   │   │  │         │  파이프에 채워짐   │                   ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  TCP 적용:                                            │   │ │
│   │   │  - Window Size ≥ BDP 필요                             │   │ │
│   │   │  - LFN (Long Fat Network) 문제                        │   │ │
│   │   │  - Window Scaling 필요                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: BDP는 TCP 성능 최적화의 핵심입니다.

**📢 섹션 요약 비유:** BDP = 파이프 용량!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **BDP** | 대역폭-지연 곱 |
| **RTT** | 왕복 시간 |
| **Window** | 윈도우 크기 |
| **Pipe** | 파이프 |
| **LFN** | Long Fat Network |

**핵심 알고리즸:** 대역폭 × 지연 = 파이프 용량

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**네트워크별 BDP**

| 네트워크 | 대역폭 | RTT | BDP |
