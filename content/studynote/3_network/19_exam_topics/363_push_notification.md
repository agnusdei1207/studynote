# Push Notification - 푸시 알림

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서버에서 클라이언트로 실시간 알림
> 2. **가치**: 사용자 참여, 실시간 업데이트
> 3. **융합**: 모바일, 웹, IoT

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Push Notification은 서버에서 클라이언트로 실시간 알림을 전송합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Push Notification                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Push 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [App Server]                                       │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Push 메시지                                     │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           Push Service                      │    │   │ │
│   │   │  │  (APNs / FCM / Web Push)                    │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • 디바이스 토큰 관리                         │    │   │ │
│   │   │  │  • 메시지 전달                                │    │   │ │
│   │   │  │  • 배달 확인                                  │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ 시스템 푸시                                   │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [Client Device]                                    │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 알림 표시                                       │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [사용자]                                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Push 서비스                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 플랫폼     │ 서비스   │ 프로토콜 │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ iOS        │ APNs     │ HTTP/2   │               │   │ │
│   │   │  │ Android    │ FCM      │ HTTP     │               │   │ │
│   │   │  | Web        │ Web Push │ VAPID    │               │   │ │
│   │   │  │ Cross      │ OneSignal│ 통합     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Push는 OS 벤더의 푸시 서비스를 통해 알림을 전달합니다.

**💡 비유**: Push는 우편 배달부입니다. 문 앞에 편지를 놓고 갑니다.

**📢 섹션 요약 비유:** Push = 실시간 알림!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Device Token** | 기기 식별자 |
| **Payload** | 메시지 내용 |
| **Silent Push** | 무음 푸시 |
| **Rich Push** | 미디어 푸시 |

**비교표: Push vs Polling**

```
+------------+----------+----------+
| 항목       │ Push     │ Polling  │
+------------+----------+----------+
| 방향       │ 서버→클라| 클라→서버│
| 배터리     │ 절약     │ 소모     │
| 지연       │ 낮음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** 장기 연결, 토큰 관리

**📢 섹션 요약 비유:** FCM = 구글 푸시!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 알림 방식**

| 항목 | Push | SMS | Email |
|:---|:---|:---|:---|
| **지연** | 초단위 | 초단위 | 분단위 |
| **비용** | 무료 | 유료 | 무료 |
| **참여** | 높음 | 중간 | 낮음 |

**📢 섹션 요약 비유:** 앱 알림 = Push!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 앱 알림 → Push
**시나리오 2:** 마케팅 → Push + SMS
**시나리오 3:** 보안 → 2FA Push

**도입 체크리스트:** 플랫폼, 토큰 관리, 권한

**📢 섹션 요약 비유:** 사용자 참여 = Push!
