# MFA (Multi-Factor Authentication) - 다중 인증

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 인증 요소 결합
> 2. **가치**: 보안 강화
> 3. **융합:** Password, OTP, Biometric

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MFA는 여러 인증 요소를 결합합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MFA (Multi-Factor Authentication)                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MFA 요소                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  인증 요소 (3가지):                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  1. Something You Know (지식)                         │   │ │
│   │   │     - 비밀번호                                        │   │ │
│   │   │     - PIN                                            │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Something You Have (소유)                         │   │ │
│   │   │     - 스마트폰                                        │   │ │
│   │   │     - OTP 토큰                                        │   │ │
│   │   │     - 보안 키                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Something You Are (생체)                          │   │ │
│   │   │     - 지문                                           │   │ │
│   │   │     - 얼굴 인식                                       │   │ │
│   │   │     - 홍채                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  MFA = 2가지 이상 조합                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: MFA는 하나의 요소가 유출되어도 안전합니다.

**📢 섹션 요약 비유:** MFA = 이중 잠금!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Factor** | 인증 요소 |
| **TOTP** | 시간 기반 OTP |
| **HOTP** | 카운터 기반 OTP |
| **Push** | 푸시 알림 |
| **FIDO** | 하드웨어 키 |

**핵심 알고리즸:** 1차 인증 → 2차 인증 → 접근

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**MFA 방식**

| 방식 | 편의성 | 보안성 |
|:---|:---|:---|
| **SMS OTP** | 높음 | 중간 |
| **TOTP** | 높음 | 높음 |
| **Push** | 높음 | 높음 |
| **FIDO2** | 높음 | 매우 높음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일반 → 비밀번호 + OTP
**시나리오 2:** 관리자 → FIDO2
**시나리오 3:** 사용자 친화적 → Push
