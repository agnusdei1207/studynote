# Cache Invalidation - 캐시 무효화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시 데이터를 최신으로 유지
> 2. **가치**: 데이터 일관성
> 3. **융합:** TTL, Event-based, Versioning

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Cache Invalidation은 캐시를 최신 상태로 유지합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Cache Invalidation                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              무효화 방법                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Time-based (TTL):                                    │   │ │
│   │   │  캐시 생성 후 N초 후 자동 만료                          │   │ │
│   │   │  Cache.set(key, value, ttl=60)                       │   │ │
│   │   │                                                      │   │ │
│   │   │  Event-based:                                         │   │ │
│   │   │  데이터 변경 시 캐시 삭제                               │   │ │
│   │   │  DB Update → Cache.delete(key)                       │   │ │
│   │   │                                                      │   │ │
│   │   │  Write-through:                                       │   │ │
│   │   │  쓰기 시 캐시와 DB 동시 업데이트                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Versioning:                                          │   │ │
│   │   │  키에 버전 포함                                        │   │ │
│   │   │  user:123:v1 → user:123:v2                           │   │ │
│   │   │                                                      │   │ │
│   │   │  Cache Stampede 방지:                                 │   │ │
│   │   │  - Mutex/Lock                                        │   │ │
│   │   │  - Early refresh                                     │   │ │
│   │   │  - Probabilistic refresh                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 무효화는 "캐시의 두 가지 어려운 문제" 중 하나입니다.

**📢 섹션 요약 비유:** Cache Invalidation = 유통기한!
