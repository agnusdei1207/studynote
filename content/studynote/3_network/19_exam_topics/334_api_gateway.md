# API Gateway - API 관문

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: API 진입점 및 관리 플랫폼

### Ⅰ. 개요 (Context & Background)

**개념 정의**

API Gateway는 마이크로서비스의 단일 진입점을 제공하는 프록시입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    API Gateway                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              API Gateway 구조                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ HTTPS                                          │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           API Gateway                       │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • 인증/인가 (Auth)                          │    │   │ │
```

> **해설**: API Gateway는 단일 진입점으로 보안과 라우팅을 관리합니다.

**💡 비유**: API Gateway는 건물의 출입구입니다. 모든 방문자가 여기를 통과합니다.

**📢 섹션 요약 비유:** API Gateway = 단일 진입점!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Router** | 라우터 |
| **Auth** | 인증 |
| **Rate Limiter** | 속도 제한 |
| **Transformer** | 변환기 |

**비교표: API Gateway 구현**

```
+------------+----------+----------+
| 항목       │ Kong     │ AWS API GW│
+------------+----------+----------+
| 타입       │ 오픈소스 │ 매니지드 │
| 확장성     │ 높음     │ 자동     │
| 비용       │ 낮음     │ 사용량   │
| 커스텀     │ 가능     │ 제한     │
+------------+----------+----------+
```

**핵심 알고리즸:** 라우팅, 토큰 버킷

**📢 섹션 요약 비유:** Rate Limit = DDoS 방어!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: Gateway 구현**
