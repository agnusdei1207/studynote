# Authorization - 인가

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 리소스 접근 권한 부여
> 2. **가치**: 최소 권한 원칙
> 3. **융합:** RBAC, ABAC, ACL

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Authorization은 "무엇을 할 수 있는가"를 결정합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Authorization                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인가 모델                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RBAC (Role-Based):                                  │   │ │
│   │   │  User → Role → Permission                            │   │ │
│   │   │  예: 관리자, 사용자, 게스트                           │   │ │
│   │   │                                                      │   │ │
│   │   │  ABAC (Attribute-Based):                             │   │ │
│   │   │  속성 기반 (사용자, 리소스, 환경)                     │   │ │
│   │   │  예: 부서=영업 AND 시간=업무시간                      │   │ │
│   │   │                                                      │   │ │
│   │   │  Principle of Least Privilege:                       │   │ │
│   │   │  필요한 최소 권한만 부여                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Authorization은 Authentication 이후에 수행됩니다.

**📢 섹션 요약 비유:** Authorization = 출입권한!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 모델 | 설명 |
|:---|:---|
| **RBAC** | 역할 기반 |
| **ABAC** | 속성 기반 |
| **ACL** | 목록 기반 |
| **PBAC** | 정책 기반 |

**핵심 알고리즸:** 사용자 → 권한 확인 → 접근 허용/거부

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**RBAC vs ABAC**

| 항목 | RBAC | ABAC |
|:---|:---|:---|
| **복잡도** | 낮음 | 높음 |
| **유연성** | 낮음 | 높음 |
| **관리** | 역할 관리 | 정책 관리 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일반 → RBAC
**시나리오 2:** 세밀한 제어 → ABAC
**시나리오 3:** 클라우드 → IAM
