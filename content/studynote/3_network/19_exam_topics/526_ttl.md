# TTL (Time To Live) - 생존 시간

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 패킷 수명 제한
> 2. **가치**: 루프 방지
> 3. **융합:** IP Header, traceroute, ICMP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TTL은 패킷이 거칠 수 있는 홉 수를 제한합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TTL (Time To Live)                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TTL 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  송신측 (TTL=64)                                     │   │ │
│   │   │      │                                               │   │ │
│   │   │      │ ── Hop 1 ──→ TTL=63                           │   │ │
│   │   │      │                │                              │   │ │
│   │   │      │                │ ── Hop 2 ──→ TTL=62          │   │ │
│   │   │      │                               │               │   │ │
│   │   │      │                               │ ── Hop 3 ──→  │   │ │
│   │   │      │                                              │   │ │
│   │   │  TTL=0 → 패킷 폐기 + ICMP Time Exceeded             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: TTL은 라우팅 루프를 방지하는 핵심 메커니즘입니다.

**📢 섹션 요약 비유:** TTL = 패킷 수명!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| OS | 기본 TTL |
|:---|:---|
| **Linux** | 64 |
| **Windows** | 128 |
| **Cisco** | 255 |
| **최대** | 255 |

**핵심 알고리즸:** TTL-- at each hop, discard if TTL=0

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**TTL 활용**

| 용도 | 설명 |
|:---|:---|
| **루프 방지** | 무한 순환 방지 |
| **Traceroute** | 경로 탐색 |
| **OS 탐지** | 기본값 차이 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 정상 → TTL 감소하며 전달
**시나리오 2:** 루프 → TTL=0 폐기
**시나리오 3:** traceroute → TTL=1부터 증가
