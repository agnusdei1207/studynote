# PAT - 포트 주소 변환

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 사설 IP → 단일 공인 IP
> 2. **가치**: 주소 효율 극대화
> 3. **융합:** Port Mapping, NAPT, Connection Tracking

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

PAT (Port Address Translation)는 포트를 이용한 주소 변환입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PAT                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PAT 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  변환 과정:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  내부                    PAT              외부   ││   │ │
│   │   │  │  192.168.1.10:5000  →  203.0.113.1:20001 → Web ││   │ │
│   │   │  │  192.168.1.11:5000  →  203.0.113.1:20002 → Web ││   │ │
│   │   │  │  192.168.1.12:5000  →  203.0.113.1:20003 → Web ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  하나의 공인 IP로 여러 기기 인터넷 접속            ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  PAT 테이블:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Inside IP      Inside Port  Outside Port        ││   │ │
│   │   │  │ 192.168.1.10   5000         20001               ││   │ │
│   │   │  │ 192.168.1.11   5000         20002               ││   │ │
│   │   │  │ 192.168.1.12   5000         20003               ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: PAT는 가정용 공유기의 핵심 기술입니다.

**📢 섹션 요약 비유:** PAT = 아파트 우편함 번호!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Source Port** | 원본 포트 |
| **Translated Port** | 변환 포트 |
| **Connection** | 연결 추적 |
| **Timeout** | 타임아웃 |
| **Port Pool** | 포트 풀 |

**핵심 알고리듬:** IP:Port → 공인IP:고유Port 매핑

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**NAT vs PAT**

| 항목 | NAT | PAT |
|:---|:---|:---|
| **매핑** | IP만 | IP+Port |
| **주소 효율** | 낮음 | 높음 |
| **동시 연결** | IP당 1개 | 65535개 |
| **복잡도** | 낮음 | 높음 |
