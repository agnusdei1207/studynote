# XDP (eXpress Data Path) - 고속 패킷 처리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 드라이버 레벨 패킷 처리
> 2. **가치**: 라인레이트 성능, DDoS 방어
> 3. **융합**: eBPF, AF_XDP, Offload

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

XDP는 드라이버에서 패킷을 처리하는 고속 경로입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    XDP 아키텍처                                      │
├─────────────────────────────────────────────────────────────────────────┤

│   │   │     ▼                                               │   │ │
```

> **해설**: XDP는 커널 네트워크 스택 전에 패킷을 처리합니다.

**💡 비유**: XDP는 빠른 출입구입니다.

**📢 섹션 요약 비유:** XDP = 드라이버 레벨!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **XDP Hook** | 드라이버 진입점 |
| **eBPF Program** | 패킷 처리 로직 |
| **XDP Maps** | 상태 저장 |
| **AF_XDP** | 유저스페이스 전달 |

**비교표: XDP 모드**

```
+------------+----------+----------+----------+
| 모드       │ 실행     │ 성능     │ 유연성   │
+------------+----------+----------+----------+
| Native     │ 드라이버 │ 최고     │ 중간     │
| Offload    │ NIC      │ 최고     │ 낮음     │
| Generic    │ 커널     │ 낮음     │ 높음     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 패킷 파싱, 액션 결정

**📢 섹션 요약 비유:** Native = 최고 성능!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)
