# 4B/5B, 8B/10B 부호화 (Block Coding)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: m비트 데이터를 n비트(n>m) 코드로 매핑
> 2. **가치**: 동기화 보장, DC 밸런스, 오류 검출
> 3. **융합**: FDDI, 100BASE-TX, 기가비트 이더넷과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Block Coding은 m비트 데이터 블록을 n비트(n>m) 코드 블록으로 변환하여 동기화와 DC 밸런스를 개선하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Block Coding (4B/5B, 8B/10B)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              4B/5B 부호화                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  원리: 4비트 데이터 → 5비트 코드                       │   │ │
│   │   │                                                      │   │ │
│   │   │  4비트 데이터 = 16가지                               │   │ │
│   │   │  5비트 코드  = 32가지                                │   │ │
│   │   │  사용 코드 = 16 + 제어용                             │   │ │
│   │   │                                                      │   │ │
│   │   │  선택 기준:                                          │   │ │
│   │   │  • 연속 1 최소화 (동기화)                             │   │ │
│   │   │  • 연속 0 최대 3개까지만                              │   │ │
│   │   │  • DC 밸런스 고려                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  예시:                                               │   │ │
│   │   │  0000 → 11110    1000 → 10010                        │   │ │
│   │   │  0001 → 01001    1001 → 10011                        │   │ │
│   │   │  0010 → 10100    1010 → 10110                        │   │ │
│   │   │  0011 → 10101    1011 → 10111                        │   │ │
│   │   │                                                      │   │ │
│   │   │  응용: FDDI, 100BASE-TX                              │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              8B/10B 부호화                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  원리: 8비트 데이터 → 10비트 코드                      │   │ │
│   │   │                                                      │   │ │
│   │   │  구조: 8비트 = 5비트(D) + 3비트(K)                    │   │ │
│   │   │       10비트 = 6비트 + 4비트                          │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                               │   │ │
│   │   │  • 완벽한 DC 밸런스 (Running Disparity)               │   │ │
│   │   │  • 연속 동일 비트 최대 5개                            │   │ │
│   │   │  • 제어 코드 (K문자) 지원                             │   │ │
│   │   │                                                      │   │ │
│   │   │  Running Disparity:                                  │   │ │
│   │   │  • 1의 개수가 많으면 다음 코드는 0 많은 것 선택        │   │ │
│   │   │  • DC 밸런스 동적 유지                                │   │ │
│   │   │                                                      │   │ │
│   │   │  응용: 기가비트 이더넷, PCIe, SATA                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: Block Coding은 NRZ의 동기화 문제를 오버헤드를 추가하여 해결합니다.

**💡 비유**: Block Coding은 4글자 단어를 5글자 암호로 바꾸는 것입니다.

**📢 섹션 요약 비유**: 4B/5B는 25% 오버헤드!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 4B/5B | 8B/10B | 비고 |
|:---|:---:|:---:|:---|
| **입력** | 4비트 | 8비트 | 데이터 |
| **출력** | 5비트 | 10비트 | 코드 |
| **오버헤드** | 25% | 25% | 대역폭 |
| **DC 밸런스** | 부분 | 완벽 | 8B/10B 우수 |

**비교표**

```
+------------+----------+----------+----------+
| 항목       | 4B/5B    | 8B/10B   | 64B/66B  |
+------------+----------+----------+----------+
| 효율       | 80%      | 80%      | 97%      |
| DC 밸런스  | 보통     | 완벽     | 좋음     |
| 복잡도     | 낮음     | 높음     | 높음     |
| 응용       | 100M     | 1G       | 10G+     |
+------------+----------+----------+----------+
```

**핵심 알고리즘**: m비트 → 코드 테이블 조회 → n비트

**📢 섹션 요약 비유**: 10G 이더넷은 64B/66B!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 이더넷 속도별 부호화**

| 속도 | 부호화 | 효율 |
|:---|:---|:---:|
| **10M** | Manchester | 50% |
| **100M** | 4B/5B + NRZ-I | 80% |
| **1G** | 8B/10B | 80% |
| **10G** | 64B/66B | 97% |

**📢 섹션 요약 비유**: 속도 높을수록 효율 높은 부호화!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 100BASE-TX** → 4B/5B + NRZ-I + MLT-3
**시나리오 2: 1000BASE-X** → 8B/10B
**시나리오 3: 10GBASE-R** → 64B/66B

**도입 체크리스트**: 속도, DC 밸런스, 구현 복잡도

**📢 섹션 요약 비유**: 4B/5B + NRZ-I가 FDDI/100M 표준!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 동기화 + 80% 효율

**미래 전망**: 64B/66B, 256B/257B (400G)

**참고 표준**: ANSI X3.263 (4B/5B), IEEE 802.3z (8B/10B)

**📢 섹션 요약 비유**: Block Coding은 동기화와 효율의 절충!
