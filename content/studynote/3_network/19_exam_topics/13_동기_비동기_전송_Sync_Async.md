# 동기/비동기 전송 (Sync/Async)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 송수신 간 타이밍 동기화 방식의 차이
> 2. **가치**: 오류율, 효율성, 구현 복잡도 결정
> 3. **융합**: 클럭 복구, 프레이밍, HDLC와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

동기식 전송은 송수신측이 공통 클럭으로 동기화하며, 비동기식은 시작/정지 비트로 프레임을 구분합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    동기/비동기 전송                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              비동기식 전송 (Asynchronous)                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Start Bit │ Data Bits │ Parity │ Stop Bit(s)       │   │ │
│   │   │     1       │   5-8     │  0/1   │   1-2            │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  • 문자 단위 전송                                     │   │ │
│   │   │  • 시작/정지 비트로 프레임 구분                        │   │ │
│   │   │  • 클럭 동기화 불필요                                  │   │ │
│   │   │  • 오버헤드 20-30%                                    │   │ │
│   │   │  • 예: RS-232, UART                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              동기식 전송 (Synchronous)                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  SYN │ SYN | Data Block | CRC | SYN | SYN | ...     │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  • 블록 단위 전송                                     │   │ │
│   │   │  • 공통 클럭 또는 클럭 복구                            │   │ │
│   │   │  • SYN 문자로 동기 유지                               │   │ │
│   │   │  • 오버헤드 5% 미만                                    │   │ │
│   │   │  • 예: HDLC, SDLC, Ethernet                           │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 비동기는 구현 쉽지만 오버헤드가 크고, 동기는 효율적이지만 동기화가 복잡합니다.

**💡 비유**: 비동기는 각 문장마다 시작/끝을 표시하는 것, 동기는 일정한 리듬으로 읽는 것입니다.

**📢 섹션 요약 비유**: 비동기 = 시작/정지 비트!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Start Bit** | 시작 | 1 → 0 천이 | 문 열기 |
| **Stop Bit** | 종료 | 1 유지 | 문 닫기 |
| **Parity** | 오류 | 홀/짝 검사 | 검수 |
| **SYN** | 동기 | 0x16 문자 | 신호 |

**비교표**

| 항목 | 비동기 | 동기 |
|:---|:---:|:---:|
| **전송 단위** | 문자 | 블록 |
| **오버헤드** | 높음 | 낮음 |
| **동기화** | 프레임 | 클럭 |
| **속도** | 낮음 | 높음 |

**핵심 알고리즘**: Start 감지 → 샘플링 → Stop 확인 → 문자 완성

**📢 섹션 요약 비유**: 동기식이 고속 전송에 유리!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 동기화 방식**

| 방식 | 클럭 | 복잡도 | 효율 |
|:---|:---:|:---:|:---:|
| **비동기** | 없음 | 낮음 | 낮음 |
| **동기(문자)** | 있음 | 중간 | 높음 |
| **동기(비트)** | 있음 | 높음 | 최고 |

**📢 섹션 요약 비유**: Ethernet은 동기식!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 키보드** → 비동기 (저속)
**시나리오 2: LAN** → 동기 (고속)
**시나리오 3: 직렬 포트** → 선택적

**도입 체크리스트**: 거리, 속도, 비용

**📢 섹션 요약 비유**: 원격은 비동기, 근거리는 동기!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)
