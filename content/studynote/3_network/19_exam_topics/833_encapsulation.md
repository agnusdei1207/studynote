# Encapsulation - 캡슐화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 감싸기
> 2. **가치**: 계층별 독립성
> 3. **융합:** Header, Trailer, Tunneling

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Encapsulation은 데이터에 헤더/트레일러를 추가합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Encapsulation                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              캡슐화 과정 (송신)                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  L7 Data                                             │   │ │
│   │   │  ┌──────────────────────────────────────────┐       │   │ │
│   │   │  │            Application Data              │       │   │ │
│   │   │  └──────────────────────────────────────────┘       │   │ │
│   │   │                      ↓ + L4 Header                   │   │ │
│   │   │  L4 Segment                                          │   │ │
│   │   │  ┌──────────────────────────────────────────┐       │   │ │
│   │   │  │ TCP Header │      Application Data      │       │   │ │
│   │   │  └──────────────────────────────────────────┘       │   │ │
│   │   │                      ↓ + L3 Header                   │   │ │
│   │   │  L3 Packet                                           │   │ │
│   │   │  ┌──────────────────────────────────────────┐       │   │ │
│   │   │  │ IP Hdr │ TCP Hdr │    App Data         │       │   │ │
│   │   │  └──────────────────────────────────────────┘       │   │ │
│   │   │                      ↓ + L2 Header/Trailer           │   │ │
│   │   │  L2 Frame                                            │   │ │
│   │   │  ┌──────────────────────────────────────────┐       │   │ │
│   │   │  │Eth Hdr│IP Hdr│TCP Hdr│App Data│FCS      │       │   │ │
│   │   │  └──────────────────────────────────────────┘       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 캡슐화는 각 계층이 독립적으로 동작하게 합니다.

**📢 섹션 요약 비유:** Encapsulation = 선물 포장!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Header** | 헤더 (제어 정보) |
| **Payload** | 페이로드 (실제 데이터) |
| **Trailer** | 트레일러 (오류 검출) |
| **Decapsulation** | 역캡슐화 |
| **Tunneling** | 터널링 |

**핵심 알고리즸:** 데이터 + Header → 전송 → Header 제거 → 데이터
