# XML - 확장 마크업 언어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 구조화된 데이터 마크업 언어
> 2. **가치**: 플랫폼 독립, 확장성
> 3. **융합**: SOAP, 설정, 문서

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

XML(eXtensible Markup Language)은 구조화된 데이터를 표현합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    XML                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              XML 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  <?xml version="1.0" encoding="UTF-8"?>             │   │ │
│   │   │  <person>                                            │   │ │
│   │   │    <name>홍길동</name>                               │   │ │
│   │   │    <age>30</age>                                     │   │ │
│   │   │    <address>                                         │   │ │
│   │   │      <city>서울</city>                               │   │ │
│   │   │      <zip>12345</zip>                                │   │ │
│   │   │    </address>                                        │   │ │
│   │   │  </person>                                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              XML 특징                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 특징       │ 설명     │ 가치     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 확장성     │ 태그     │ 유연성   │               │   │ │
│   │   │  │ 계층       │ 트리     │ 구조     │               │   │ │
│   │   │  | 검증      │ DTD/XSD  │ 무결성   │               │   │ │
│   │   │  │ 플랫폼     │ 독립     │ 호환성   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: XML은 태그로 데이터를 구조화합니다.

**💡 비유**: XML은 라벨이 붙은 상자입니다. 각 상자에 이름이 있습니다.

**📢 섹션 요약 비유:** XML = 구조화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Element** | 요소 |
| **Attribute** | 속성 |
| **Namespace** | 네임스페이스 |
| **Schema** | 스키마 |

**비교표: XML vs JSON**

```
+------------+----------+----------+
| 항목       │ XML      │ JSON     │
+------------+----------+----------+
| 크기       │ 큼       │ 작음     │
| 가독성     │ 높음     │ 높음     │
| 스키마     │ 강함     │ 약함     │
| 파싱       │ 복잡     │ 간단     │
+------------+----------+----------+
```

**핵심 알고리즸:** DOM, SAX, XPath

**📢 섹션 요약 비유:** XPath = 쿼리!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 데이터 포맷**

| 항목 | XML | JSON | YAML |
|:---|:---|:---|:---|
| **복잡도** | 높음 | 낮음 | 낮음 |
| **검증** | 강함 | 약함 | 없음 |
| **용도** | 문서 | API | 설정 |

**📢 섹션 요약 비유:** 설정 = XML/YAML!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** SOAP API → XML
**시나리오 2:** 설정 파일 → XML
**시나리오 3:** REST API → JSON

**도입 체크리스트:** 스키마, 파서, 크기

**📢 섹션 요약 비유:** 복잡한 문서 = XML!
