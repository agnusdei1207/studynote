# CDN - 콘텐츠 전송 네트워크

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 콘텐츠 캐싱
> 2. **가치**: 지연 감소, 성능 향상
> 3. **융합**: DNS, 클라우드와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CDN(Content Delivery Network)은 전 세계에 분산된 서버에서 콘텐츠를 캐싱하여 빠르게 전송합니다.

```

│   │   │                                                      │   │ │
```

> **해설**: CDN은 사용자와 가까운 Edge 서버에서 콘텐츠를 제공합니다.

**💡 비유**: CDN은 전 세계 편의점 체인입니다. 가까운 곳에서 상품을 받습니다.

**📢 섹션 요약 비유:** CDN = 지연 감소!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Origin** | 원본 서버 |
| **Edge PoP** | 엣지 서버 |
| **Cache** | 캐시 저장 |
| **TTL** | 캐시 수명 |

**비교표: CDN 제공자**

```
+------------+----------+----------+
| 제공자     │ 특징     │ 용도     │
+------------+----------+----------+
| Cloudflare │ 보안     │ 웹       │
| Akamai     │ 규모     │ 엔터프라이즈│
| AWS CloudFront│ AWS   │ 클라우드 │
| Azure CDN  │ Azure    │ 클라우드 │
+------------+----------+----------+
```

**핵심 알고리즸:** 캐시 무효화, 라우팅

**📢 섹션 요약 비유:** TTL = 캐시 수명!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 캐시 전략**

| 전략 | 설명 |
|:---|:---|
| **Push** | 미리 캐시 |
| **Pull** | 요청 시 캐시 |
| **Invalidation** | 무효화 |

**📢 섹션 요약 비유:** Pull = 요청 기반!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 웹사이트** → Cloudflare
**시나리오 2: 미디어** → Akamai
**시나리오 3: AWS** → CloudFront

**도입 체크리스트:** PoP 위치, 캐시 정책, 비용

**📢 섹션 요약 비유:** Cache-Control 헤더 = 캐시 제어!
