# Proxy - 프록시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 대리 통신
> 2. **가치**: 중개 및 제어
> 3. **융합:** Forward Proxy, Reverse Proxy, Transparent

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Proxy는 클라이언트를 대신해 서버와 통신합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Proxy                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              프록시 유형                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Forward Proxy:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Client ──→ Proxy ──→ Server                   ││   │ │
│   │   │  │           ↓                                     ││   │ │
│   │   │  │       캐싱, 필터링                               ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ 용도: 익명성, 콘텐츠 필터, 캐싱                   ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  Reverse Proxy:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Client ──→ Proxy ──→ Backend Server            ││   │ │
│   │   │  │           ↓                                     ││   │ │
│   │   │  │       로드밸런싱, SSL 종료                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ 용도: 로드밸런싱, 보안, SSL 오프로딩               ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  기능:                                                 │   │ │
│   │   │  - 캐싱 (Caching)                                     │   │ │
│   │   │  - 필터링 (Filtering)                                  │   │ │
│   │   │  - 로드밸런싱 (Load Balancing)                         │   │ │
│   │   │  - SSL 종료 (SSL Termination)                         │   │ │
│   │   │  - 익명화 (Anonymization)                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 프록시는 네트워크 중개 및 제어에 핵심적입니다.

**📢 섹션 요약 비유:** Proxy = 대리인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Forward** | 포워드 프록시 |
| **Reverse** | 리버스 프록시 |
| **Cache** | 캐시 |
| **Filter** | 필터 |
| **Anonymize** | 익명화 |

**핵심 알고리즸:** 요청 수신 → 처리/전달 → 응답 반환

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**프록시 유형 비교**

| 유형 | 방향 | 대표 | 용도 |
|:---|:---|:---|:---|
| **Forward** | 아웃바운드 | 클라이언트 | 필터, 캐시 |
| **Reverse** | 인바운드 | 서버 | LB, SSL |
| **Transparent** | 양방향 | 없음 | 모니터링 |
