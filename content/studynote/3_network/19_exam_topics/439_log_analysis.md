# 로그 분석 - Log Analysis

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 로그 데이터 분석으로 인사이트 도출
> 2. **가치**: 문제 진단, 보안 분석, 트렌드 파악
> 3. **융합**: SIEM, ELK Stack, Splunk

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

로그 분석은 시스템 로그에서 유용한 정보를 추출합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    로그 분석                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              로그 분석 파이프라인                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  수집 → 파싱 → 저장 → 분석 → 시각화 → 알림          │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │

│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │

│   │   │  │    - Elasticsearch, Splunk                  │    │   │ │
```

**📢 섹션 요약 비유:** 로그 = 네트워크 기록!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Syslog** | 표준 로그 프로토콜 |
| **ELK Stack** | Elasticsearch, Logstash, Kibana |
| **Splunk** | 상용 로그 분석 |
| **SIEM** | 보안 이벤트 관리 |

**비교표: 로그 분석 도구**

```
+------------+----------+----------+----------+
| 도구       │ 유형     │ 규모     │ 특징     │
+------------+----------+----------+----------+
| ELK        │ 오픈소스 │ 대규모   │ 확장성   │
| Splunk     │ 상용     │ 엔터프라이즈│ 강력  │
| Graylog    │ 오픈소스 │ 중규모   │ 단순     │
| Loki       │ 오픈소스 │ 클라우드 │ 경량     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 정규식, 집계

**📢 섹션 요약 비유:** ELK = 로그 분석 표준!

**기술 비교: 분석 기법**

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 문제 진단 → 키워드 검색
**시나리오 2:** 보안 분석 → 상관관계 분석
**시나리오 3:** 용량 계획 → 트렌드 분석

**도입 체크리스트:** 수집, 보관, 검색

**📢 섹션 요약 비유:** 보관 기간 = 규정 준수!
