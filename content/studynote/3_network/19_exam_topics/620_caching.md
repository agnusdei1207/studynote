# Caching - 캐싱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 임시 저장
> 2. **가치**: 지연 감소, 부하 감소
> 3. **융합:** CDN, Redis, Browser Cache

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Caching은 자주 쓰는 데이터를 빠른 저장소에 둡니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Caching                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              캐시 계층                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  L1 Cache (CPU)                                      │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  L2/L3 Cache (CPU)                                   │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  Browser Cache                                       │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  CDN / Reverse Proxy                                 │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  Application Cache (Redis/Memcached)                  │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  Database                                            │   │ │
│   │   │     ↓                                                │   │ │
│   │   │  Disk Storage                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 캐시는 속도와 비용의 균형을 맞춥니다.

**📢 섹션 요약 비유:** Caching = 자주 쓰는 물건!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Hit** | 캐시 적중 |
| **Miss** | 캐시 미스 |
| **TTL** | Time To Live |
| **Eviction** | 제거 정책 |

**핵심 알고리즸:** 조회 → (Hit) 반환 / (Miss) 원본 → 캐시

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**캐시 제거 정책**

| 정책 | 설명 |
|:---|:---|
| **LRU** | Least Recently Used |
| **LFU** | Least Frequently Used |
| **FIFO** | First In First Out |
| **TTL** | 시간 기반 |
