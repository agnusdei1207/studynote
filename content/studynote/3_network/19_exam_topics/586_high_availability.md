# HA (High Availability) - 고가용성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서비스 중단 최소화
> 2. **가치**: 비즈니스 연속성
> 3. **융합:** Failover, Redundancy, Cluster

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HA는 시스템 중단 없이 지속적 서비스를 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    HA (High Availability)                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HA 구성                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │            Load Balancer                             │   │ │
│   │   │                 │                                     │   │ │
│   │   │        ┌───────┴───────┐                             │   │ │
│   │   │        │               │                             │   │ │
│   │   │   ┌────┴────┐     ┌────┴────┐                       │   │ │
│   │   │   │ Active  │     │ Standby │                       │   │ │
│   │   │   │ Server  │     │ Server  │                       │   │ │
│   │   │   └─────────┘     └─────────┘                       │   │ │
│   │   │        ↑                                            │   │ │
│   │   │    Failover                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  목표: 99.99% (연간 52분 다운타임)                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HA는 이중화와 페일오버로 가용성을 보장합니다.

**📢 섹션 요약 비유:** HA = 비상 발전기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Redundancy** | 이중화 |
| **Failover** | 장애 조치 |
| **Heartbeat** | 상태 확인 |
| **Quorum** | 정족수 |

**핵심 알고리즸:** 장애 감지 → Failover → 복구
