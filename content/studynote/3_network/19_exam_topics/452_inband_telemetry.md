# 인밴드 텔레메트리 - In-band Telemetry

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 패킷에 메트릭 삽입
> 2. **가치**: 실시간 가시성, 지연 측정
> 3. **융합**: INT, IOAM, P4

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

인밴드 텔레메트리는 데이터 패킷으로 네트워크 상태를 수집합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    인밴드 텔레메트리                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              INT (In-band Network Telemetry)                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────┐    ┌─────────┐    ┌─────────┐         │   │ │
│   │   │  │ Source  │───►│ Switch1 │───►│ Switch2 │───► ... │   │ │
│   │   │  │         │    │         │    │         │         │   │ │
│   │   │  └─────────┘    └────┬────┘    └────┬────┘         │   │ │
│   │   │                      │              │               │   │ │
│   │   │                      ▼              ▼               │   │ │
│   │   │               ┌─────────────────────────┐          │   │ │
│   │   │               │ Packet Header           │          │   │ │
│   │   │               │ ┌─────┬─────┬─────┐    │          │   │ │
│   │   │               │ │ Hop1│ Hop2│ ... │    │          │   │ │
│   │   │               │ │Info │Info │     │    │          │   │ │
│   │   │               │ └─────┴─────┴─────┘    │          │   │ │
│   │   │               │ - 지연, 큐, 포트       │          │   │ │
│   │   │               └─────────────────────────┘          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              수집 정보                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 정보       │ 크기     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 타임스탬프 │ 8B       │ 지연     │               │   │ │
│   │   │  │ 스위치 ID  │ 4B       │ 경로     │               │   │ │
│   │   │  | 포트 정보  │ 4B       │ 병목     │               │   │ │
│   │   │  │ 큐 정보    │ 4B       │ 혼잡     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: INT는 각 홉에서 패킷에 정보를 추가합니다.

**💡 비유**: 인밴드 텔레메트리는 여행 일지입니다.

**📢 섹션 요약 비유:** INT = 실시간 가시성!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **INT Source** | 텔레메트리 시작점 |
| **INT Transit** | 정보 추가 스위치 |
| **INT Sink** | 텔레메트리 종료점 |
| **Collector** | 데이터 수집 |

**비교표: 텔레메트리 방식**

```
+------------+----------+----------+----------+
| 방식       │ 오버헤드 │ 실시간   │ 정확도   │
+------------+----------+----------+----------+
| SNMP       │ 낮음     │ 낮음     │ 중간     │
| NetFlow    │ 중간     │ 중간     │ 높음     │
| INT        │ 높음     │ 높음     │ 매우높음 │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 헤더 삽입, 메타데이터 추출

**📢 섹션 요약 비유:** Sink = 데이터 수집!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 데이터센터 → INT

**시나리오 3:** 디버깅 → 샘플링

**도입 체크리스트:** P4 지원, 오버헤드, 분석

**📢 섹션 요약 비유:** 샘플링 = 오버헤드 관리!
