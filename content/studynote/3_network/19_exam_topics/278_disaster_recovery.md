# 재해 복구 (Disaster Recovery) - DR

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 재해 후 서비스 복구
> 2. **가치**: 비즈니스 연속성
> 3. **융합**: 백업, HA와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

재해 복구(DR)는 자연재해, 장애 등으로부터 IT 서비스를 복구하는 전략입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    재해 복구 (Disaster Recovery)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DR 아키텍처                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Primary Site]            [DR Site]                │   │ │
│   │   │       │                         │                    │   │ │
│   │   │       │  Replication            │                    │   │ │
│   │   │       │───────────────────────▶│                    │   │ │
│   │   │       │                         │                    │   │ │
│   │   │  ┌──────────┐             ┌──────────┐              │   │ │
│   │   │  │ Server   │             │ Server   │              │   │ │
│   │   │  │ Storage  │             │ Storage  │              │   │ │
│   │   │  │ Network  │             │ Network  │              │   │ │

│   │   │       │                         │                    │   │ │
│   │   │       │ 재해 발생               │                    │   │ │
│   │   │       ↓                         ↓                    │   │ │
│   │   │  ┌──────────┐             ┌──────────┐              │   │ │
│   │   │  │  장애    │             │  Active  │ ← Failover   │   │ │
│   │   │  └──────────┘             └──────────┘              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │

│   │   │  1. Hot Site:                                        │   │ │
│   │   │     • 실시간 복제, 즉시 전환                          │   │ │
│   │   │                                                      │   │ │

│   │   │     • 백업만, 느린 전환                               │   │ │
```

> **해설**: DR은 원격 사이트에 백업하여 재해 시 복구합니다.

**💡 비유**: DR은 보험입니다. 만약의 사태에 대비합니다.

**📢 섹션 요약 비유:** DR = 재해 대비!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RTO** | 복구 시간 목표 |
| **RPO** | 복구 시점 목표 |
| **DR Site** | 복구 사이트 |
| **Replication** | 복제 |

**비교표: DR 사이트 유형**

```
+------------+----------+----------+
| 유형       │ RTO      │ 비용     │
+------------+----------+----------+
| Hot Site   │ 분 단위  │ 높음     │
| Warm Site  │ 시간     │ 중간     │
| Cold Site  │ 일       │ 낮음     │
+------------+----------+----------+
```

**핵심 알고리즸:** 복제, 페일오버

**📢 섹션 요약 비유:** Hot Site = 즉시 복구!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 복제 방식**

| 방식 | 설명 |
|:---|:---|
| **동기** | 데이터 손실 없음 |
| **비동기** | 약간 손실 가능 |
| **하이브리드** | 혼합 |

**📢 섹션 요약 비유:** 동기 복제 = 무손실!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 금융** → Hot Site + 동기 복제
**시나리오 2: 일반 기업** → Warm Site + 비동기
**시나리오 3: 소규모** → Cold Site + 백업

**도입 체크리스트:** RTO, RPO, 비용

**📢 섹션 요약 비유:** 비용 vs 복구 시간!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 비즈니스 연속성

**미래 전망:** 클라우드 DR 확대

**참고 표준:** ISO 22301

**📢 섹션 요약 비유:** DR은 필수 보험!
