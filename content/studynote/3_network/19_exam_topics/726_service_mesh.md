# Service Mesh - 서비스 메시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서비스 간 통신 인프라
> 2. **가치**: 관측성, 보안, 트래픽 제어
> 3. **융합:** Istio, Envoy, Sidecar Pattern

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Service Mesh는 서비스 간 통신을 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Service Mesh                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              서비스 메시 구조                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Service A           Service B           Service C  │   │ │
│   │   │  ┌─────────┐        ┌─────────┐        ┌─────────┐ │   │ │
│   │   │  │ Service │        │ Service │        │ Service │ │   │ │
│   │   │  │    A    │        │    B    │        │    C    │ │   │ │
│   │   │  ├─────────┤        ├─────────┤        ├─────────┤ │   │ │
│   │   │  │  Envoy  │ ←────→ │  Envoy  │ ←────→ │  Envoy  │ │   │ │
│   │   │  │(Sidecar)│        │(Sidecar)│        │(Sidecar)│ │   │ │
│   │   │  └─────────┘        └─────────┘        └─────────┘ │   │ │
│   │   │       ↓                 ↓                  ↓       │   │ │
│   │   │  ┌───────────────────────────────────────────────┐│   │ │
│   │   │  │            Control Plane (Istiod)              ││   │ │
│   │   │  │  - 설정 관리                                    ││   │ │
│   │   │  │  - 인증서 발급                                  ││   │ │
│   │   │  │  - 서비스 디스커버리                            ││   │ │
│   │   │  └───────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 서비스 메시는 통신, 보안, 관측성을 제공합니다.

**📢 섹션 요약 비유:** Service Mesh = 교통 관제!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Data Plane** | 데이터 평면 |
| **Control Plane** | 제어 평면 |
| **Sidecar** | 사이드카 |
| **Proxy** | 프록시 |
| **mTLS** | 상호 TLS |

**핵심 알고리즸:** 요청 → Sidecar → 라우팅 → 대상
