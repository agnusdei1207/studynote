# Disaster Recovery (DR) - 재해 복구

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 재해 시 서비스 복구
> 2. **가치**: 비즈니스 연속성
> 3. **융합:** RTO, RPO, Backup, HA

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DR은 재해 발생 시 서비스를 복구합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Disaster Recovery                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DR 구성                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Primary Site              DR Site                   │   │ │
│   │   │  ┌─────────────┐          ┌─────────────┐           │   │ │
│   │   │  │   Server    │ ─Sync──→ │   Server    │           │   │ │
│   │   │  │   Storage   │          │   Storage   │           │   │ │
│   │   │  │   Network   │          │   Network   │           │   │ │
│   │   │  └─────────────┘          └─────────────┘           │   │ │
│   │   │       ↓ Disaster               ↑ Failover           │   │ │
│   │   │                                                      │   │ │
│   │   │  복구 목표:                                           │   │ │
│   │   │  - RTO (Recovery Time Objective)                     │   │ │
│   │   │  - RPO (Recovery Point Objective)                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DR은 재해 시 RTO/RPO 내로 복구합니다.

**📢 섹션 요약 비유:** DR = 비상 대피소!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RTO** | 복구 시간 목표 |
| **RPO** | 복구 시점 목표 |
| **Hot Standby** | 실시간 복제 |
| **Cold Standby** | 필요 시 시작 |

**핵심 알고리즸:** 재해 감지 → 전환 → 복구

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**DR 전략**

| 전략 | RTO | 비용 |
|:---|:---|:---|
| **Hot** | 즉시 | 높음 |
| **Warm** | 시간 | 중간 |
| **Cold** | 일 | 낮음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 금융 → Hot Standby
**시나리오 2:** 일반 → Warm Standby
**시나리오 3:** 백오피스 → Cold Standby
