# 라우트 요약 - Route Summarization

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 경로를 하나로 축약
> 2. **가치**: 라우팅 테이블 축소
> 3. **융합**: CIDR, VLSM, 계층 설계

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

라우트 요약은 여러 네트워크를 하나의 경로로 표현합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    라우트 요약                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              요약 예시                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  개별 경로:                                          │   │ │
│   │   │  192.168.0.0/24                                      │   │ │
│   │   │  192.168.1.0/24                                      │   │ │
│   │   │  192.168.2.0/24                                      │   │ │
│   │   │  192.168.3.0/24                                      │   │ │
│   │   │        │                                             │   │ │
│   │   │        │ 요약                                        │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  192.168.0.0/22 (4개 네트워크 요약)                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              요약 효과                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ 요약 전  │ 요약 후  │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 경로 수    │ 4        │ 1        │               │   │ │
│   │   │  │ 메모리     │ 많음     │ 적음     │               │   │ │
│   │   │  | CPU       │ 높음     │ 낮음     │               │   │ │
│   │   │  │ 수렴속도   │ 느림     │ 빠름     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 라우트 요약은 라우팅 테이블 크기를 줄입니다.

**💡 비유**: 라우트 요약은 전화번호부를 지역별로 묶는 것입니다.

**📢 섹션 요약 비유:** 요약 = 라우팅 효율!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Supernetting** | 슈퍼넷팅 |
| **Summary Route** | 요약 경로 |
| **Common Bits** | 공통 비트 |
| **Longest Match** | 최장 일치 |

**비교표: 요약 가능 여부**

```
+------------+----------+----------+
| 네트워크   │ 요약 가능│ 요약 결과│
+------------+----------+----------+
| 0-3        │ O        │ /22      │
| 0-2        │ X        │ -        │
| 0,2,4,6    │ X        │ -        │
+------------+----------+----------+
```

**핵심 알고리즸:** 공통 비트 찾기, 2의 거듭제곱

**📢 섹션 요약 비유:** 2^n 개 = 요약 가능!
