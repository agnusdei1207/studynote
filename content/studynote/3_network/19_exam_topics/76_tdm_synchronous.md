# 동기식 시분할 다중화 (Synchronous TDM)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고정된 타임슬롯을 각 채널에 정적 할당
> 2. **가치**: 단순, 예측 가능, 저지연
> 3. **융합**: T1/E1, 전화망, 셀룰러와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

동기식 TDM(Synchronous TDM)은 각 입력 채널에 고정된 타임슬롯을 순차적으로 할당하여 전송하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    동기식 TDM (Synchronous TDM)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              동기식 TDM 구조                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  입력 vs 전송:                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  입력 CH1: ████████                    (데이터 있음)  │   │ │
│   │   │  입력 CH2: ░░░░░░░░░░░░░░░░░░░░░░░░░░░░ (데이터 없음) │   │ │
│   │   │  입력 CH3: ████████                    (데이터 있음)  │   │ │
│   │   │  입력 CH4: ░░░░░░░░░░░░░░░░░░░░░░░░░░░░ (데이터 없음) │   │ │
│   │   │                                                      │   │ │
│   │   │  TDM 출력:                                            │   │ │
│   │   │  ├────┼────┼────┼────┼────┼────┼────┼────┤          │   │ │
│   │   │  │CH1 │CH2 │CH3 │CH4 │CH1 │CH2 │CH3 │CH4 │          │   │ │
│   │   │  │데이터│공백│데이터│공백│데이터│공백│데이터│공백│          │   │ │
│   │   │  └────┴────┴────┴────┴────┴────┴────┴────┘          │   │ │
│   │   │         ↑    ↑    ↑    ↑                            │   │ │

│   │   │                                                      │   │ │
│   │   │  문제점: 유휴 슬롯 발생 → 대역폭 낭비                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              동기식 TDM 특징                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  장점:                                                 │   │ │
│   │   │  • 단순한 구현                                         │   │ │
│   │   │  • 일정한 지연 (예측 가능)                              │   │ │
│   │   │  • 낮은 복잡도                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  단점:                                                 │   │ │
│   │   │  • 유휴 슬롯 낭비                                      │   │ │
│   │   │  • 버스트 트래픽 비효율                                 │   │ │
│   │   │  • 고정 대역폭 할당                                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │

│   └──────────────────────────────────────────────────────────────┘ │
```

> **해설**: 동기식 TDM은 전화망의 기본 기술입니다.

**💡 비유**: 동기식 TDM은 회의에서 발언권이 돌아가는데 말할 게 없어도 차례는 옵니다.

**📢 섹션 요약 비유**: 고정 슬롯 = 예측 가능!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **프레임** | 슬롯 세트 | N+1 슬롯 | 주기 |
| **동기 비트** | 프레임 시작 | F 비트 | 표시 |
| **슬롯** | 채널 데이터 | 8비트 | 단위 |

**비교표**

```
+------------+----------+----------+
| T1 프레임  │ 내용     │ 비트     |
+------------+----------+----------+
| 동기 비트  | F        | 1        |
| CH1 슬롯   | 음성     | 8        |
| CH2 슬롯   | 음성     | 8        |
| ...        | ...      | ...      |
| CH24 슬롯  | 음성     | 8        |
| 합계       | 1프레임  | 193      |
+------------+----------+----------+
```

**핵심 알고리즸**: N 채널 → 각 1슬롯 → 순차 전송 → 반복

**📢 섹션 요약 비유**: T1 = 193비트 프레임!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: T1 vs E1**

| 항목 | T1 (북미) | E1 (유럽) |
|:---|:---:|:---:|
| 채널 수 | 24 | 32 |
| 전송률 | 1.544Mbps | 2.048Mbps |
| 프레임 | 193비트 | 256비트 |
| 동기 | 1비트 | 1채널 |

**📢 섹션 요약 비유**: 한국은 E1 기반!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 전화망** → 동기식 TDM
**시나리오 2: PBX** → T1/E1
**시나리오 3: 데이터망** → 통계적 TDM (비추천)

**도입 체크리스트**: 일정 트래픽, 낮은 지연, 단순성

**📢 섹션 요약 비유**: 음성은 동기식 TDM!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 일정 지연, 단순 구현

**미래 전망**: SDH/SONET로 진화

**참고 표준**: ITU-T G.703, G.704

**📢 섹션 요약 비유**: 동기식 TDM은 전화망의 기반!
