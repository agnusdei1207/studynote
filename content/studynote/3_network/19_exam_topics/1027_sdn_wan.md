# SD-WAN - 소프트웨어 정의 WAN

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: WAN 가상화 및 자동화
> 2. **가치**: 비용 절감 및 성능 최적화
> 3. **융합:** Overlay, Controller, Policy

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SD-WAN (Software-Defined Wide Area Network)은 소프트웨어로 WAN을 관리합니다.

```

│   │   │  │          SD-WAN Controller (Orchestrator)        ││   │ │

│   │   │  └─────────────────────┬───────────────────────────┘│   │ │
│   │   │                        │                             │   │ │
```

**📢 섹션 요약 비유:** SD-WAN = WAN의 Uber!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Controller** | 중앙 제어기 |
| **CPE/Edge** | 지사 장비 |
| **Overlay** | 가상 네트워크 |
| **Underlay** | 물리 회선 |
| **DPI** | Deep Packet Inspection |

**핵심 알고리듬:** 트래픽 식별 → 정책 매칭 → 경로 선택 → 전송

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SD-WAN vs 전통적 WAN**

| 항목 | SD-WAN | 전통적 WAN |
|:---|:---|:---|
| **회선** | 하이브리드 | MPLS |
| **관리** | 중앙 집중 | 분산 |
| **비용** | 낮음 | 높음 |
| **배포** | 빠름 | 느림 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 지사 연결 → SD-WAN
**시나리오 2:** 클라우드 → SD-WAN + Direct Connect
**시나리오 3:** 하이브리드 → MPLS + SD-WAN
