# Trunk - 트렁크 (VLAN Trunking)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 VLAN 전송 링크
> 2. **가치**: VLAN 간 연결 효율
> 3. **융합:** 802.1Q, ISL, Native VLAN

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Trunk는 여러 VLAN의 트래픽을 하나의 링크로 전송합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Trunk                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Trunk 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Switch A                         Switch B           │   │ │
│   │   │  ┌─────────┐                      ┌─────────┐       │   │ │
│   │   │  │VLAN 10  │                      │VLAN 10  │       │   │ │
│   │   │  │VLAN 20  │ ───── Trunk ───────→ │VLAN 20  │       │   │ │
│   │   │  │VLAN 30  │    (802.1Q Tag)      │VLAN 30  │       │   │ │
│   │   │  └─────────┘                      └─────────┘       │   │ │
│   │   │                                                      │   │ │
│   │   │  프레임 구조:                                        │   │ │
│   │   │  ┌──────────────────────────────────────┐            │   │ │
│   │   │  │ DA │ SA │ Tag │ Type │ Data │ FCS  │            │   │ │
│   │   │  │         └─ VLAN ID ─┘                │            │   │ │
│   │   │  └──────────────────────────────────────┘            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Trunk는 VLAN 태그로 여러 VLAN을 구분하여 전송합니다.

**📢 섹션 요약 비유:** Trunk = 고속도로!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **802.1Q** | 표준 VLAN 태깅 |
| **ISL** | Cisco 전용 (구형) |
| **Native VLAN** | 태그 없는 프레임 |
| **Tag** | 4바이트 추가 |

**핵심 알고리즸:** 송신 시 Tag 추가, 수신 시 Tag 제거

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**802.1Q vs ISL**

| 항목 | 802.1Q | ISL |
|:---|:---|:---|
| **표준** | IEEE | Cisco |
| **오버헤드** | 4바이트 | 30바이트 |
| **Native** | 지원 | 미지원 |
