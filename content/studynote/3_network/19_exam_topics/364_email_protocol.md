# Email Protocol - 이메일 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전자우편 송수신 프로토콜
> 2. **가치**: 비동기 통신, 보편성
> 3. **융합**: SMTP, IMAP, POP3

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

이메일 프로토콜은 전자우편 송수신을 담당합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Email Protocol                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              이메일 프로토콜 구조                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Sender]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ SMTP (발송)                                    │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           SMTP Server                       │    │   │ │
│   │   │  │  (Outgoing Mail Server)                     │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ 인터넷 전송                                   │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           Mail Server                       │    │   │ │
│   │   │  │  (수신 서버)                                  │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ IMAP/POP3 (수신)                             │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [Recipient]                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │

│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 프로토콜   │ 용도     │ 포트     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ SMTP       │ 발송     │ 25/587   │               │   │ │
│   │   │  │ IMAP       │ 수신     │ 143/993  │               │   │ │
│   │   │  | POP3       │ 수신     │ 110/995  │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SMTP는 발송, IMAP/POP3는 수신을 담당합니다.

**💡 비유**: SMTP는 우체국 접수, IMAP는 우편함입니다.

**📢 섹션 요약 비유:** SMTP = 발송, IMAP = 수신!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SMTP** | Simple Mail Transfer |
| **IMAP** | Internet Message Access |
| **POP3** | Post Office Protocol |
| **MIME** | 미디어 형식 |

**비교표: IMAP vs POP3**

```
+------------+----------+----------+
| 항목       │ IMAP     │ POP3     │
+------------+----------+----------+
| 동기화     │ 지원     │ 미지원   │
| 서버 저장  │ 지원     │ 선택     │
| 멀티 기기  │ 지원     │ 제한     │
| 오프라인   │ 제한     │ 지원     │
+------------+----------+----------+
```

**핵심 알고리즸:** MX 레코드, MIME 인코딩

**📢 섹션 요약 비유:** IMAP = 클라우드!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 이메일 보안**

| 항목 | SSL/TLS | STARTTLS | SPF/DKIM |
|:---|:---|:---|:---|
| **암호화** │ 전송 │ 전송 │ 없음 |
| **인증** │ 서버 │ 서버 │ 도메인 |

**📢 섹션 요약 비유:** 이메일 = SMTP + IMAP!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 기업 → IMAP
**시나리오 2:** 개인 → POP3
**시나리오 3:** 보안 → TLS

**도입 체크리스트:** MX 레코드, 보안, 스팸 필터

**📢 섹션 요약 비유:** 멀티 기기 = IMAP!
