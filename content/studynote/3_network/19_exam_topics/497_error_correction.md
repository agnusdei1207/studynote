# 에러 정정 - Error Correction

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오류 자동 수정
> 2. **가치**: 재전송 없는 신뢰성
> 3. **융합:** FEC, ARQ, HEC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

에러 정정은 오류를 자동으로 수정합니다.

```

│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ ARQ        │ 재전송   │ TCP      │               │   │ │
```

**📢 섹션 요약 비유:** Error Correction = 자동 수정!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **FEC** | Forward Error Correction |
| **ARQ** | Automatic Repeat reQuest |
| **Hamming** | 해밍 코드 |
| **Reed-Solomon** | RS 코드 |

**핵심 알고리즸:** 패리티, 다항식

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**FEC vs ARQ**

| 항목 | FEC | ARQ |
|:---|:---|:---|
| **지연** | 낮음 | 높음 |
| **오버헤드** | 항상 | 오류 시 |
| **용도** | 실시간 | 신뢰성 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** TCP → ARQ
**시나리오 2:** VoIP → FEC
**시나리오 3:** 위성 → HEC
