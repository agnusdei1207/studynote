# 서비스 메시 - Service Mesh

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 마이크로서비스 간 통신 인프라
> 2. **가치**: 관찰 가능성, 트래픽 제어, 보안
> 3. **융합**: Istio, Envoy, Sidecar 패턴

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

서비스 메시는 마이크로서비스 간 통신을 관리합니다.

**📢 섹션 요약 비유:** Service Mesh = 마이크로서비스 통신!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Data Plane** | Envoy 프록시 |
| **Control Plane** | Istiod |
| **Sidecar** | 프록시 컨테이너 |
| **Gateway** | 인그레스/이그레스 |

**비교표: 서비스 메시 구현**

```
+------------+----------+----------+----------+
| 구현       │ 프록시   │ 제어     │ 특징     │
+------------+----------+----------+----------+
| Istio      │ Envoy    │ Istiod   │ 풀기능   │
| Linkerd    │ Linkerd  │ 제어평면 │ 경량     │
| Consul     │ Envoy    │ Consul   │ 서비스   │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 로드밸런싱, 서킷브레이커

**📢 섹션 요약 비유:** Istio = 서비스 메시 표준!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 핵심 기능**

| 기능 | 설명 | 설정 |
|:---|:---|:---|

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 마이크로서비스 → Istio

**도입 체크리스트:** 성능, 복잡성, 운영

**📢 섹션 요약 비유:** 관찰 가능성 = 디버깅!
