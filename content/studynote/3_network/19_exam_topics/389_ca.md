# CA - 인증 기관

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디지털 인증서 발급 기관
> 2. **가치**: 신뢰 앵커, 신원 보증
> 3. **융합**: PKI, TLS, 코드 서명

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CA(Certificate Authority)는 디지털 인증서를 발급하는 신뢰 기관입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CA                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CA 계층 구조                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │              [Root CA]                              │   │ │
│   │   │              자체 서명                               │   │ │
│   │   │                   │                                  │   │ │
│   │   │         ┌─────────┴─────────┐                       │   │ │
│   │   │         │                   │                       │   │ │
│   │   │         ▼                   ▼                       │   │ │
│   │   │    [Intermediate CA]  [Intermediate CA]             │   │ │
│   │   │         │                   │                       │   │ │
│   │   │         ▼                   ▼                       │   │ │
│   │   │    [End Entity]       [End Entity]                 │   │ │
│   │   │    (서버 인증서)       (클라이언트)                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CA 종류                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ CA 종류    │ 신뢰     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Root CA    │ 최상위   │ 정책     │               │   │ │
│   │   │  │ Intermediate│ 중간    │ 발급     │               │   │ │
│   │   │  | Public CA  │ 공인     │ 인터넷   │               │   │ │
│   │   │  │ Private CA │ 사설     │ 내부     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: CA는 신뢰 체인의 시작점입니다.

**💡 비유**: CA는 공증인입니다. 문서의 진위를 보증합니다.

**📢 섹션 요약 비유:** CA = 신뢰 앵커!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Root CA** | 최상위 CA |
| **Intermediate CA** | 중간 CA |
| **Issuing CA** | 발급 CA |
| **Trust Store** | 신뢰 저장소 |

**비교표: 공인 CA vs 사설 CA**

```
+------------+----------+----------+
| 항목       │ 공인 CA  │ 사설 CA  │
+------------+----------+----------+
| 비용       │ 유료     │ 무료     │
| 신뢰       │ 공인     │ 내부     │
| 용도       │ 인터넷   │ 기업내부 │
| 관리       │ CA       │ 자체     │
+------------+----------+----------+
```

**핵심 알고리즸:** 키 쌍 생성, 서명, 폐기

**📢 섹션 요약 비유:** HTTPS = 공인 CA!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 공개 웹사이트 → 공인 CA
**시나리오 2:** 기업 내부 → 사설 CA
**시나리오 3:** 개발/테스트 → 자체 서명

**도입 체크리스트:** CA 선택, 검증 수준, 자동화

**📢 섹션 요약 비유:** 신뢰 = CA!
