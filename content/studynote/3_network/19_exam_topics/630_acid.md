# ACID - ACID 속성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트랜잭션 4대 속성
> 2. **가치**: 데이터 무결성 보장
> 3. **융합:** Database, Transaction, Isolation

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ACID는 트랜잭션이 갖춰야 할 4가지 속성입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ACID                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ACID 속성                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  A - Atomicity (원자성)                      │    │   │ │
│   │   │  │      전부 실행 또는 전부 미실행              │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  C - Consistency (일관성)                    │    │   │ │
│   │   │  │      트랜잭션 후 일관된 상태 유지            │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  I - Isolation (격리성)                      │    │   │ │
│   │   │  │      동시 실행 트랜잭션 간 간섭 없음         │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  D - Durability (지속성)                     │    │   │ │
│   │   │  │      완료된 트랜잭션은 영구 저장             │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ACID는 신뢰할 수 있는 트랜잭션의 기반입니다.

**📢 섹션 요약 비유:** ACID = 은행 거래 원칙!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 속성 | 설명 |
|:---|:---|
| **Atomicity** | 전부 또는 전무 |
| **Consistency** | 제약 조건 유지 |
| **Isolation** | 동시성 제어 |
| **Durability** | 영속성 보장 |

**핵심 알고리즸:** 트랜잭션 → ACID 검증 → Commit

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ACID vs BASE**

| 항목 | ACID | BASE |
|:---|:---|:---|
| **일관성** | 강함 | 약함 |
| **가용성** | 낮음 | 높음 |
| **복잡도** | 높음 | 낮음 |
| **용도** | RDBMS | NoSQL |
