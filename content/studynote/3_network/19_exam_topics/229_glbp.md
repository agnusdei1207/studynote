# GLBP (Gateway Load Balancing Protocol) - Cisco 로드밸런싱 이중화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Cisco 전용 로드밸런싱 FHRP
> 2. **가치**: 다중 Active 게이트웨이
> 3. **융합**: HSRP, VRRP보다 효율적

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

GLBP(Gateway Load Balancing Protocol)는 Cisco 전용 FHRP로, 여러 라우터가 동시에 트래픽을 분산 처리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    GLBP (Gateway Load Balancing Protocol)            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              GLBP 구성                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │         Virtual IP: 192.168.1.1 (AVG)               │   │ │
│   │   │                    ↑                                │   │ │
│   │   │         ┌─────────┼─────────┐                      │   │ │
│   │   │         │         │         │                      │   │ │
│   │   │    [R1]      [R2]      [R3]                        │   │ │
│   │   │    AVF       AVF       AVF                         │   │ │
│   │   │    MAC: .1   MAC: .2   MAC: .3                      │   │ │
│   │   │                                                      │   │ │
│   │   │  역할:                                                │   │ │
│   │   │  • AVG (Active Virtual Gateway): VIP 관리           │   │ │
│   │   │  • AVF (Active Virtual Forwarder): 실제 포워딩     │   │ │
│   │   │                                                      │   │ │
│   │   │  로드밸런싱: 클라이언트별 다른 AVF 할당              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              로드밸런싱 방식                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  방식        │ 설명                                 │   │ │
│   │   │  ───────────┼──────────────────────────────────│   │ │
│   │   │  Round-robin│ 순차 분배                           │   │ │
│   │   │  Weighted   │ 가중치 분배                         │   │ │
│   │   │  Host-dependent│ 같은 호스트는 항상 같은 AVF      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: GLBP는 여러 라우터가 동시에 트래픽을 처리합니다.

**💡 비유**: GLBP는 다중 창구 은행입니다. 여러 창구가 동시에 업무를 봅니다.

**📢 섹션 요약 비유:** GLBP = 로드밸런싱 + 이중화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **AVG** | Active Virtual Gateway |
| **AVF** | Active Virtual Forwarder |
| **멀티캐스트** | 224.0.0.102 |
| **UDP** | 3222 |

**비교표: FHRP 비교**

```
+------------+----------+----------+----------+
| 항목       │ HSRP     │ VRRP     │ GLBP     │
+------------+----------+----------+----------+
| Active 수  │ 1        │ 1        │ 다중     │
| 로드밸런싱 │ X        │ X        │ O        │
| 표준       │ Cisco    │ IEEE     │ Cisco    │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** AVG 선출, AVF 할당

**📢 섹션 요약 비유:** AVG = 관리자, AVF = 작업자!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 로드밸런싱**

| 방식 | 특징 |
|:---|:---|
| **Round-robin** | 균등 분배 |
| **Weighted** | 성능별 분배 |
| **Host-dependent** | 세션 유지 |

**📢 섹션 요약 비유:** Weighted = 성능 차이 반영!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: Cisco + 로드밸런싱** → GLBP
**시나리오 2: Cisco + 단순** → HSRP
**시나리오 3: 멀티벤더** → VRRP

**도입 체크리스트:** 로드밸런싱 방식, Weight, Preempt

**📢 섹션 요약 비유:** 대역폭 활용 = GLBP!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대역폭 활용, 고가용성

**미래 전망:** VRRP + ECMP로 대체 가능

**참고 표준:** Cisco 전용

**📢 섹션 요약 비유:** GLBP는 Cisco의 고급 FHRP!
