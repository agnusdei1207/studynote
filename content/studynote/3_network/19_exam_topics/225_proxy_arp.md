# Proxy ARP - 대리 ARP 응답

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다른 네트워크의 ARP 요청 대신 응답
> 2. **가치**: 서브넷 마스크 설정 오류 해결
> 3. **융합**: 라우터, 방화벽에서 활용

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Proxy ARP는 라우터가 다른 네트워크에 있는 호스트의 ARP 요청을 대신 응답하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Proxy ARP                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Proxy ARP 동작                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [네트워크 A]          [라우터]          [네트워크 B] │   │ │
│   │   │  192.168.1.0/24                       192.168.2.0/24 │   │ │
│   │   │                                                      │   │ │
│   │   │  Host A                eth0        eth1              │   │ │
│   │   │  192.168.1.10          │            │    Host B      │   │ │
│   │   │  /16 (잘못된 마스크)   │            │   192.168.2.10 │   │ │
│   │   │        │               │            │        │       │   │ │

│   │   │        │               │            │        │       │   │ │

│   │   │        │ "192.168.2.10 │            │        │       │   │ │
│   │   │        │  is at Router │            │        │       │   │ │
│   │   │        │  MAC"         │            │        │       │   │ │
│   │   │        │ <──────────── │            │        │       │   │ │
│   │   │        │               │            │        │       │   │ │
│   │   │  라우터가 자신의 MAC으로 응답                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │

│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              사용 사례                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 서브넷 마스크 설정 오류                          │   │ │
│   │   │  2. 게이트웨이 설정 없음                             │   │ │
│   │   │  3. 서브넷 확장                                      │   │ │
│   │   │  4. VPN/터널링                                       │   │ │

│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Proxy ARP는 라우터가 다른 네트워크 호스트의 ARP 요청에 대신 응답합니다.

**💡 비유**: Proxy ARP는 비서가 대신 전화를 받는 것입니다.

**📢 섹션 요약 비유:** Proxy ARP = 라우터 대리 응답!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **동작** | 라우터가 자신의 MAC으로 응답 |
| **조건** | 목적지가 다른 네트워크 |
| **장점** | 클라이언트 설정 불필요 |
| **단점** | ARP 테이블 증가 |

**비교표: 일반 ARP vs Proxy ARP**

```
+------------+----------+----------+
| 항목       │ 일반 ARP │ Proxy ARP│
+------------+----------+----------+
| 응답자     │ 본인     │ 라우터   │
| MAC        │ 실제 MAC │ 라우터   │
| 용도       │ 일반     │ 특수     │
+------------+----------+----------+
```

**핵심 알고리즸:** 라우팅 테이블 확인, 대리 응답

**📢 섹션 요약 비유:** 라우터 MAC = 중간 경유!
