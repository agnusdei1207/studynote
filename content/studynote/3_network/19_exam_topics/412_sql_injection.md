# SQL Injection - SQL 주입 공격

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 악의적 SQL 문장 주입
> 2. **가치**: 데이터 탈취, 인증 우회
> 3. **융합**: Prepared Statement, ORM, WAF

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SQL Injection은 악의적인 SQL을 주입해 데이터베이스를 조작합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SQL Injection                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SQL Injection 공격 예시                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  정상 쿼리:                                          │   │ │
│   │   │  SELECT * FROM users                                 │   │ │
│   │   │  WHERE id = 'user' AND pw = 'pass'                   │   │ │
│   │   │                                                      │   │ │
│   │   │  공격 입력:                                          │   │ │
│   │   │  id: admin' --                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  주입된 쿼리:                                        │   │ │
│   │   │  SELECT * FROM users                                 │   │ │
│   │   │  WHERE id = 'admin' --' AND pw = 'pass'              │   │ │
│   │   │  (비밀번호 검증 우회)                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SQL Injection 유형                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 유형       │ 설명     │ 결과     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ In-band    │ 결과직접 │ 데이터   │               │   │ │
│   │   │  │ Blind      │ 참/거짓 │ 추론     │               │   │ │
│   │   │  | Out-of-band│ 외부채널 │ DNS/HTTP │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SQL Injection은 입력값 검증 부재로 발생합니다.

**💡 비유**: SQL Injection은 문서 위조입니다. SQL 문장을 조작합니다.

**📢 섹션 요약 비유:** SQL Injection = DB 해킹!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **In-band** | 결과 직접 확인 |
| **Blind** | 참/거짓으로 추론 |
| **Time-based** | 응답 시간으로 추론 |
| **Union** | UNION으로 데이터 결합 |

**비교표: 방어 방안**

```
+------------+----------+----------+
| 항목       │ Prepared │ ORM      │
+------------+----------+----------+
| 파라미터화 │ O        │ O        │
| 편의성     │ 중간     │ 높음     │
| 보안       │ 높음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** 파라미터 바인딩, 입력 검증

**📢 섹션 요약 비유:** Prepared Statement = SQL Injection 방어!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 레거시 코드 → Prepared Statement
**시나리오 2:** 신규 개발 → ORM
**시나리오 3:** 추가 보안 → WAF

**도입 체크리스트:** 파라미터 바인딩, 입력 검증, 최소 권한

**📢 섹션 요약 비유:** 사용자 입력 = 의심!
