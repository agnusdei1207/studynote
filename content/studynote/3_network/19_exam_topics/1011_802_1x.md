# 802.1X - 포트 기반 네트워크 접근 제어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 포트 인증 프로토콜
> 2. **가치**: 인증된 접근만 허용
> 3. **융합:** EAP, RADIUS, NAC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IEEE 802.1X는 포트 기반 네트워크 접근 제어 표준입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    802.1X                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              802.1X 구조                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  3개의 주체 (AAA):                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  ┌────────────┐         ┌────────────┐          ││   │ │
│   │   │  │  │Supplicant  │         │Authenticator│          ││   │ │
│   │   │  │  │  (단말)     │◄─EAPOL─►│  (스위치)   │          ││   │ │
│   │   │  │  │            │         │            │          ││   │ │

│   │   │  │                        │Auth Server │            ││   │ │
```

> **해설**: 802.1X는 인증 성공 시에만 포트를 개방합니다.

**📢 섹션 요약 비유:** 802.1X = 포트 자물쇠!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Supplicant** | 인증 요청자 |
| **Authenticator** | 인증자 |
| **Auth Server** | 인증 서버 |
| **EAP** | Extensible Authentication Protocol |
| **EAPOL** | EAP over LAN |

**핵심 알고리듬:** EAPOL 시작 → EAP 인증 → RADIUS 전달 → 결과

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**EAP 방식 비교**

| EAP 방식 | 특징 |
|:---|:---|
| **EAP-TLS** | 인증서 기반 |
| **EAP-PEAP** | 암호화 터널 + MS-CHAPv2 |
| **EAP-TTLS** | 터널 + 내부 인증 |
| **EAP-MD5** | 비권장 (약함) |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 유선 → 802.1X + RADIUS
**시나리오 2:** 무선 → 802.1X + WPA2-Enterprise
**시나리오 3:** IoT → MAB (MAC Authentication Bypass)
