# Indexing - 인덱싱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 빠른 검색을 위한 자료구조
> 2. **가치**: 조회 성능 향상
> 3. **융합:** Database, Search Engine, B-Tree

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Indexing은 빠른 검색을 위한 자료구조입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Indexing                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인덱스 구조                                        │ │

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  B-Tree Index:                                        │   │ │
│   │   │           ┌─────┐                                    │   │ │
│   │   │           │ 50  │                                    │   │ │
│   │   │           └──┬──┘                                    │   │ │
│   │   │         ┌────┴────┐                                  │   │ │
│   │   │      ┌──┴──┐   ┌──┴──┐                               │   │ │
│   │   │      │ 25  │   │ 75  │                               │   │ │
│   │   │      └──┬──┘   └──┬──┘                               │   │ │
│   │   │    ┌────┼────┐ ┌──┼───┐                              │   │ │
│   │   │  ┌─┴─┐┌─┴─┐┌─┴┐┌─┴─┐┌─┴─┐                             │   │ │
```

**📢 섹션 요약 비유:** Indexing = 책 목차!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **B-Tree** | 균형 트리 |
| **Hash** | 해시 테이블 |
| **Composite** | 복합 인덱스 |
| **Covering** | 커버링 인덱스 |
| **Partial** | 부분 인덱스 |

**핵심 알고리즸:** 키 → 트리 탐색 → 레코드 위치
