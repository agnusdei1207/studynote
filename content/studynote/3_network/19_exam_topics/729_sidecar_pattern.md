# Sidecar Pattern - 사이드카 패턴

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 보조 컨테이너
> 2. **가치**: 관심사 분리, 재사용성
> 3. **융합:** Service Mesh, Kubernetes, Envoy

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Sidecar Pattern은 보조 컨테이너를 배치합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Sidecar Pattern                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              사이드카 패턴 구조                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Pod (Kubernetes)                                    │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌─────────────┐    ┌─────────────┐        │    │   │ │
│   │   │  │  │   Main      │    │   Sidecar   │        │    │   │ │
│   │   │  │  │ Application │ ←→ │  Container  │        │    │   │ │
│   │   │  │  │             │    │             │        │    │   │ │
│   │   │  │  │ - 비즈니스   │    │ - 로깅      │        │    │   │ │
│   │   │  │  │   로직      │    │ - 프록시    │        │    │   │ │
│   │   │  │  │             │    │ - 모니터링  │        │    │   │ │
│   │   │  │  └─────────────┘    └─────────────┘        │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │         공유: 네트워크, 스토리지              │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 사이드카는 메인 애플리케이션을 수정하지 않고 기능을 추가합니다.

**📢 섹션 요약 비유:** Sidecar = 옆자리 조수!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Main Container** | 메인 컨테이너 |
| **Sidecar** | 사이드카 |
| **Shared Volume** | 공유 볼륨 |
| **Shared Network** | 공유 네트워크 |
| **Lifecycle** | 생명주기 |

**핵심 알고리즸:** 요청 → Main ↔ Sidecar → 처리

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**사이드카 용도**
