# ICE - 연결 설정 프레임워크

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: NAT 통과 연결 설정 프레임워크
> 2. **가치**: 최적 경로 자동 선택
> 3. **융합**: WebRTC, VoIP, P2P

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ICE(Interactive Connectivity Establishment)는 NAT 통과 연결을 설정합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ICE                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ICE 프로세스                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Candidate 수집                                   │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Host (로컬 IP)                                 │   │ │
│   │   │     │ Server Reflexive (STUN 공인 IP)                │   │ │
│   │   │     │ Relayed (TURN 릴레이)                          │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  2. Candidate 교환 (시그널링)                         │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ SDP 교환                                       │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  3. Connectivity Check                               │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ STUN Binding Request/Response                  │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  4. Candidate Pair 선택                              │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 최우선순위 쌍 선택                              │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  5. 연결 완료                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Candidate 유형                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 유형       │ 설명     │ 우선순위 │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Host       │ 로컬     │ 최고     │               │   │ │
│   │   │  │ srflx      │ STUN     │ 중간     │               │   │ │
│   │   │  | relay      │ TURN     │ 낮음     │               │   │ │
│   │   │  │ prflx      │ 피어     │ 중간     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ICE는 후보 수집부터 최적 연결 선택까지 자동화합니다.

**💡 비유**: ICE는 여러 경로를 시도해 가장 좋은 길을 찾습니다.

**📢 섹션 요약 비유:** ICE = 최적 경로!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Candidate** | 후보 주소 |
| **Check** | 연결 확인 |
| **Nomination** | 후보 지명 |
| **Keepalive** | 연결 유지 |

**비교표: ICE 라이트 vs Full**

```
+------------+----------+----------+
| 항목       │ Lite     │ Full     │
+------------+----------+----------+
| 후보       │ Host만   │ 전체     │
| 체크       │ 없음     │ 있음     │
| 복잡도     │ 낮음     │ 높음     │
| 용도       │ 서버     │ 클라이언트│
+------------+----------+----------+
```

**핵심 알고리즸:** 후보 수집, 우선순위, 연결 확인

**📢 섹션 요약 비유:** Host Candidate = 최고!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: NAT 통과**

| 항목 | ICE | 수동 설정 | UPnP |
|:---|:---|:---|:---|
| **자동** | 예 | 아니오 | 예 |
| **신뢰성** | 높음 | 중간 | 낮음 |
| **복잡도** | 높음 | 낮음 | 낮음 |

**📢 섹션 요약 비유:** WebRTC = ICE 필수!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: WebRTC** → ICE Full
**시나리오 2: 미디어 서버** → ICE Lite
**시나리오 3: P2P 게임** → ICE

**도입 체크리스트:** STUN/TURN, 시그널링, 타임아웃

**📢 섹션 요약 비유:** 자동 연결 = ICE!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 자동 최적 연결

**📢 섹션 요약 비유:** ICE는 NAT 통과 표준!
