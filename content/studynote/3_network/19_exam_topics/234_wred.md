# WRED (Weighted RED) - 가중치 기반 혼잡 회피

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 혼잡 회피를 위한 패킷 폐기
> 2. **가치**: 테일 드롭 방지, TCP 동기화 방지
> 3. **융합**: QoS, DiffServ와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

WRED(Weighted Random Early Detection)는 혼잡 발생 전에 확률적으로 패킷을 폐기하여 혼잡을 회피하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WRED (Weighted RED)                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RED/WRED 동작                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  큐 사용률                                           │   │ │
│   │   │  100%│                           /──────────        │   │ │
│   │   │      │                         /                    │   │ │
│   │   │   Max│───────────────────────/ (100% 폐기)          │   │ │
│   │   │      │                     /                        │   │ │
│   │   │   Min│────────────────────/ (확률적 폐기)           │   │ │
│   │   │      │                   /                          │   │ │
│   │   │    0%├──────────────────/────────────────────       │   │ │
│   │   │      │  폐기 없음    │ 확률 폐기 │ 전체 폐기        │   │ │
│   │   │      └─────────────────────────────────────         │   │ │
│   │   │               Min      Max                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              WRED vs RED                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RED:                                                 │   │ │
│   │   │  • 단일 임계값                                        │   │ │
│   │   │  • 모든 트래픽 동일 처리                              │   │ │
│   │   │                                                      │   │ │
│   │   │  WRED:                                                │   │ │
│   │   │  • 여러 임계값 (IP Precedence/DSCP별)                │   │ │
│   │   │  • 우선순위별 다른 폐기 확률                          │   │ │
│   │   │  • 높은 우선순위 = 낮은 폐기 확률                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: WRED는 IP Precedence/DSCP에 따라 다른 임계값을 적용합니다.

**💡 비유**: WRED는 도로 정체 시 차별화된 진입 제한입니다.

**📢 섹션 요약 비유:** WRED = 우선순위 기반 폐기!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Min Threshold** | 폐기 시작 |
| **Max Threshold** | 전체 폐기 |
| **Mark Probability** | 폐기 확률 |
| **MPD** | Mark Probability Denominator |

**비교표: 혼잡 관리**

```
+------------+----------+----------+
| 방식       │ 시점     │ 방법     │
+------------+----------+----------+
| Tail Drop  │ 큐 만석  │ 무조건   │
| RED        │ 혼잡 전  │ 확률적   │
| WRED       │ 혼잡 전  │ 가중치   │
+------------+----------+----------+
```

**핵심 알고리즸:** 평균 큐 길이 계산, 확률적 폐기

**📢 섹션 요약 비유:** Tail Drop = TCP 동기화 문제!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: DSCP별 임계값**

| DSCP | Min | Max | MPD |
|:---:|:---:|:---:|:---:|
| **EF** | 높음 | 높음 | 높음 |
| **AF** | 중간 | 중간 | 중간 |
| **BE** | 낮음 | 낮음 | 낮음 |

**📢 섹션 요약 비유:** 높은 DSCP = 높은 임계값!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: VoIP** → WRED 비권장 (LLQ 사용)
**시나리오 2: 데이터** → WRED 권장
**시나리오 3: TCP 트래픽** → WRED 필수

**도입 체크리스트:** Min/Max 임계값, MPD, DSCP 매핑

**📢 섹션 요약 비유:** TCP = WRED, UDP = LLQ!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**미래 전망:** AQM (Active Queue Management)
