# iSCSI - IP 기반 스토리지

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 네트워크로 SCSI 전송
> 2. **가치**: 저비용 SAN
> 3. **융합**: 이더넷, 스토리지와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

iSCSI(Internet Small Computer System Interface)는 IP 네트워크를 통해 SCSI 명령을 전송합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    iSCSI                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              iSCSI 구조                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [iSCSI Initiator]          [iSCSI Target]          │   │ │
│   │   │       │                           │                   │   │ │
│   │   │       │  SCSI over IP (TCP 3260) │                   │   │ │
│   │   │       │  ─────────────────────▶  │                   │   │ │
│   │   │       │                           │                   │   │ │
│   │   │  ┌────▼────┐                 ┌────▼────┐             │   │ │
│   │   │  │ Server  │                 │ Storage │             │   │ │
│   │   │  │         │                 │  Array  │             │   │ │

│   │   │  └─────────┘                 └─────────┘             │   │ │
│   │   │       │                           │                   │   │ │
│   │   │       │   IP Network (Ethernet)   │                   │   │ │
│   │   │       │◀─────────────────────────▶│                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
```

> **해설**: iSCSI는 IP 네트워크를 사용하여 저비용 SAN을 구현합니다.

**💡 비유**: iSCSI는 인터넷으로 하드디스크를 연결합니다.

**📢 섹션 요약 비유:** iSCSI = IP 기반 SAN!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Initiator** | 클라이언트 |
| **Target** | 스토리지 |
| **LUN** | 논리 유닛 |
| **IQN** | 식별자 |

**비교표: iSCSI vs FC**

```
+------------+----------+----------+
| 항목       │ iSCSI    │ FC       │
+------------+----------+----------+
| 네트워크   │ IP       │ 전용     │
| 비용       │ 낮음     │ 높음     │
| 성능       │ 중간     │ 높음     │
| 관리       │ 쉬움     │ 어려움   │
+------------+----------+----------+
```

**핵심 알고리즸:** TCP 캡슐화, CHAP 인증

**📢 섹션 요약 비유:** IQN = 고유 이름!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: iSCSI 최적화**

| 기술 | 설명 |
|:---|:---|
| **Jumbo Frame** | MTU 9000 |
| **TOE** | TCP 오프로드 |
| **iSER** | RDMA |
| **Multipath** | 다중 경로 |

**📢 섹션 요약 비유:** Jumbo Frame = 성능 향상!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: SMB 스토리지** → iSCSI
**시나리오 2: VM 스토리지** → iSCSI + Multipath
**시나리오 3: 고성능** → FC 또는 iSER

**도입 체크리스트:** 네트워크, 보안, 성능

**📢 섹션 요약 비유:** 전용 VLAN = 보안!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 저비용 SAN

**미래 전망:** NVMe-oF 확대

**참고 표준:** RFC 3720

**📢 섹션 요약 비유:** iSCSI는 IP SAN 표준!
