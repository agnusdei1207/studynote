# 네트워크 보안 (Network Security) - 방화벽/IDS/IPS

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 위협 방어
> 2. **가치**: 기밀성, 무결성, 가용성
> 3. **융합**: Zero Trust, AI와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

네트워크 보안은 네트워크 인프라와 데이터를 위협으로부터 보호하는 기술과 정책입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    네트워크 보안                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              보안 장비                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 방화벽 (Firewall):                                 │   │ │
│   │   │     • 패킷 필터링 (L3/L4)                              │   │ │
│   │   │     • 상태 검사 (Stateful)                             │   │ │
│   │   │     • 애플리케이션 필터 (L7)                            │   │ │
│   │   │     • 차세대 방화벽 (NGFW)                             │   │ │
│   │   │                                                      │   │ │
│   │   │  2. IDS/IPS:                                          │   │ │
│   │   │     • IDS (침입 탐지): 모니터링만                      │   │ │
│   │   │     • IPS (침입 방지): 차단                            │   │ │
│   │   │     • 시그니처/행위 기반                               │   │ │
│   │   │                                                      │   │ │
│   │   │  3. WAF (Web Application Firewall):                   │   │ │
│   │   │     • 웹 공격 방어 (SQL Injection, XSS)               │   │ │
│   │   │     • L7 필터링                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 기타:                                             │   │ │
│   │   │     • DDoS 방어                                       │   │ │
│   │   │     • VPN 게이트웨이                                   │   │ │
│   │   │     • NAC (Network Access Control)                    │   │ │
│   │   │     • SIEM (보안 이벤트 관리)                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              보안 모델                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  CIA Triad:                                            │   │ │
│   │   │  • Confidentiality (기밀성)                            │   │ │
│   │   │  • Integrity (무결성)                                  │   │ │
│   │   │  • Availability (가용성)                               │   │ │
│   │   │                                                      │   │ │
│   │   │  Defense in Depth:                                     │   │ │
│   │   │  • 다층 방어                                           │   │ │
│   │   │  • 경계 → 내부 → 호스트 → 앱                           │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Defense in Depth는 여러 계층에서 보안을 적용하는 전략입니다.

**💡 비유**: 네트워크 보안은 성벽입니다. 여러 겹의 벽으로 보호합니다.

**📢 섹션 요약 비유:** NGFW = L7 + IPS!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 장비 | 계층 | 기능 | 탐지 방식 |
|:---|:---:|:---|:---|
| **패킷 필터** | L3/L4 | ACL | IP/Port |
| **Stateful** | L3/L4 | 상태 추적 | 연결 |
| **NGFW** | L3-L7 | 앱 인식 | DPI |
| **IPS** | L3-L7 | 침입 방지 | 시그니처/행위 |

**비교표**

```
+------------+----------+----------+
| 항목       │ IDS      │ IPS      │
+------------+----------+----------+
| 탐지       │ O        │ O        │
│ 차단       │ X        │ O        │
│ 위치       │ SPAN     │ 인라인   │
│ 영향       │ 없음     │ 있음     │
+------------+----------+----------+
```

**핵심 알고리즸:** DPI, 시그니처 매칭, 행위 분석

**📢 섹션 요약 비유:** IDS = 경보, IPS = 경비!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 위협 유형**

| 위협 | 설명 | 대응 |
|:---|:---|:---|
| **Malware** | 악성코드 | AV, NGFW |
| **Phishing** | 사칭 | 교육, 필터 |
| **DDoS** | 서비스 거부 | DDoS 장비 |
| **SQL Injection** | DB 공격 | WAF |
| **XSS** | 스크립트 삽입 | WAF |

**📢 섹션 요약 비유:** 다층 방어 = Defense in Depth!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 인터넷 경계** → NGFW + IPS
**시나리오 2: 데이터센터** → WAF + IDS
**시나리오 3: 내부망** → NAC + EDR

**도입 체크리스트:** 자산, 위협, 규정, 예산

**📢 섹션 요약 비유:** 보안은 투자!
