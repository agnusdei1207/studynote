### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **veth pair** | 가상 이더넷 페어 |
| **Bridge** | Linux 브리지 |
| **CNI** | Container Network Interface |
| **CNM** | Container Network Model |

**비교표: CNI vs CNM**

```
+------------+----------+----------+----------+
| 항목       │ CNI      │ CNM      │ 용도     │
+------------+----------+----------+----------+
| 주체       │ CNCF     │ Docker   │ 표준     │
| 플랫폼     │ K8s      │ Docker   │ 플랫폼   │
| 복잡성     │ 단순     │ 복잡     │ 구현     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** NAT, 포트 매핑

**📢 섹션 요약 비유:** CNI = 쿠버네티스 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: CNI 플러그인**

| 플러그인 | 설명 | 용도 |
|:---|:---|:---|
| **Flannel** | Overlay | 간단 |
| **Calico** | L3 라우팅 | 성능 |
| **Weave** | Overlay | 쉬운 설정 |
| **Cilium** | eBPF | 보안 |

**📢 섹션 요약 비유:** Calico = L3 라우팅!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 단일 호스트 → Bridge
**시나리오 2:** 클러스터 → Calico/Flannel
**시나리오 3:** 서비스 메시 → Cilium + Istio

**도입 체크리스트:** 규모, 성능, 보안

**📢 섹션 요약 비유:** 오버레이 = 멀티호스트!
