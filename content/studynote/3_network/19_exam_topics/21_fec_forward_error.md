# FEC (순방향 오류 정정)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 재전송 없이 수신측에서 오류 직접 정정
> 2. **가치**: 지연 감소, 실시간, 무선 통신 필수
> 3. **융합**: 해밍, RS, LDPC, 터보 코드와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

FEC(Forward Error Correction)는 송신측에서 미리 오류 정정 정보를 추가하여 수신측이 재전송 없이 오류를 정정하는 기술입니다.

```

│                    FEC (순방향 오류 정정)                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FEC vs ARQ                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  ARQ (Automatic Repeat reQuest):                      │   │ │
│   │   │  • 오류 검출 → 재전송 요청 → 재전송                   │   │ │
│   │   │  • 왕복 지연 발생                                      │   │ │
│   │   │  • 채널이 양방향이어야 함                               │   │ │
│   │   │                                                      │   │ │

│   │   │  • 수신측에서 직접 정정                                 │   │ │
│   │   │  • 단방향 통신 가능                                     │   │ │
```

> **해설**: FEC는 지연이 중요한 실시간/무선 통신에 필수입니다.

**💡 비유**: FEC는 예비 부품을 미리 보내 현장에서 수리하는 것입니다.

**📢 섹션 요약 비유**: FEC = 재전송 없이 정정!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Encoder** | 부호화 | 패리티 추가 | 포장 |
| **Channel** | 채널 | 오류 발생 | 운송 |
| **Decoder** | 복호화 | 정정 | 개봉 |
| **Code Rate** | 효율 | k/n | 압축률 |

**코드 레이트**

| 레이트 | 설명 |
|:---|:---|
| **1/2** | 데이터 1, 패리티 1 |
| **2/3** | 데이터 2, 패리티 1 |
| **3/4** | 데이터 3, 패리티 1 |

**핵심 알고리즘**: k비트 → n비트 부호화 → 전송 → 복호화 → 정정

**📢 섹션 요약 비유**: Code Rate 높으면 효율↑ 신뢰성↓!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FEC 코드**

| 코드 | 응용 | 성능 |
|:---|:---|:---:|
| **Hamming** | 메모리 | 낮음 |
| **RS** | CD, DVD | 중간 |
| **Turbo** | 3G/4G | 높음 |
| **LDPC** | 5G, WiFi | 최고 |

**📢 섹션 요약 비유**: LDPC가 Shannon 한계 근접!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 위성** → FEC 강함 (지연 큼)
**시나리오 2: LTE** → Turbo + HARQ
**시나리오 3: 5G NR** → LDPC + Polar

**도입 체크리스트**: 지연 허용, 채널 상태, 대역폭

**📢 섹션 요약 비유**: HARQ = FEC + ARQ!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 재전송 90% 감소

**미래 전망**: Quantum FEC, AI 기반

**참고 표준**: 3GPP, DVB, IEEE 802.11

**📢 섹션 요약 비유**: FEC는 무선 통신의 필수!
