# Sequence Number - 순서 번호

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 패킷 순서 식별
> 2. **가치**: 순서 보장 및 중복 제거
> 3. **융합:** TCP, ISN, Wraparound

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Sequence Number는 각 바이트의 순서를 식별합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Sequence Number                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TCP 순서 번호                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  바이트 단위 번호 부여:                                 │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Data:    H    e    l    l    o                 ││   │ │
│   │   │  │ Seq:    100  101  102  103  104                ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  세그먼트 전송:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Seg 1: Seq=1000, Data=100 bytes                 ││   │ │
│   │   │  │ Seg 2: Seq=1100, Data=50 bytes                  ││   │ │
│   │   │  │ Seg 3: Seq=1150, Data=80 bytes                  ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  ISN (Initial Sequence Number):                       │   │ │
│   │   │  - 연결 시 무작위 값으로 시작                           │   │ │
│   │   │  - 보안: 연결 혼란 방지                                │   │ │
│   │   │  - RFC 1948: 해시 기반 생성                           │   │ │
│   │   │                                                      │   │ │
│   │   │  Wraparound:                                           │   │ │
│   │   │  - 32비트: 4GB 후 순환                                 │   │ │
│   │   │  - 고속 네트워크: PAWS (Protection Against Wrapped)   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 순서 번호는 신뢰적 전송의 기반입니다.

**📢 섹션 요약 비유:** Sequence Number = 페이지 번호!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **ISN** | 초기 순서 번호 |
| **Seq** | 순서 번호 |
| **Ack** | 확인 번호 |
| **Wraparound** | 랩어라운드 |
| **PAWS** | 랩어라운드 방지 |

**핵심 알고리즸:** ISN 설정 → 증가 → Wraparound

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**순서 번호 크기**

| 프로토콜 | 비트 | 범위 |
|:---|:---|:---|
| **TCP** | 32 | 4GB |
| **UDP** | 없음 | - |
| **SCTP** | 32 | 4GB |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
