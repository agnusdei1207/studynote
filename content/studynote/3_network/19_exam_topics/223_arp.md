# ARP (Address Resolution Protocol) - 주소 해석

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 주소 → MAC 주소 변환
> 2. **가치**: L2 통신 가능
> 3. **융합**: Gratuitous ARP, Proxy ARP와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ARP(Address Resolution Protocol)는 IP 주소를 MAC 주소로 변환하는 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ARP (Address Resolution Protocol)                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ARP 동작 과정                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Host A (IP: 192.168.1.10)                          │   │ │
│   │   │  알고 싶음: 192.168.1.20의 MAC                       │   │ │
│   │   │                                                      │   │ │
│   │   │  1. ARP Request (Broadcast):                        │   │ │
│   │   │     "Who has 192.168.1.20? Tell 192.168.1.10"       │   │ │
│   │   │     ───────────────────────────────────>            │   │ │
│   │   │     (모든 호스트에 브로드캐스트)                      │   │ │
│   │   │                                                      │   │ │
│   │   │  2. ARP Reply (Unicast):                            │   │ │
│   │   │     "192.168.1.20 is at aa:bb:cc:dd:ee:ff"         │   │ │
│   │   │     <──────────────────────────────────             │   │ │
│   │   │     (해당 호스트만 응답)                              │   │ │
│   │   │                                                      │   │ │
│   │   │  3. ARP Cache 저장                                   │   │ │
│   │   │     192.168.1.20 → aa:bb:cc:dd:ee:ff               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ARP 캐시                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IP 주소        MAC 주소           상태              │   │ │
│   │   │  ────────────  ──────────────────  ────────          │   │ │
│   │   │  192.168.1.1   00:11:22:33:44:55   Complete          │   │ │
│   │   │  192.168.1.20  aa:bb:cc:dd:ee:ff   Complete          │   │ │
│   │   │                                                      │   │ │
│   │   │  캐시 만료: 일반적으로 15-20분                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ARP는 브로드캐스트로 MAC 주소를 찾고 캐시에 저장합니다.

**💡 비유**: ARP는 전화번호부입니다. 이름(IP)으로 전화번호(MAC)를 찾습니다.

**📢 섹션 요약 비유:** ARP = IP → MAC!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Request** | 브로드캐스트 |
| **Reply** | 유니캐스트 |
| **Cache** | 15-20분 |
| **Gratuitous** | 자기 자신 광고 |

**비교표: ARP vs RARP vs NDP**

```
+------------+----------+----------+
| 프로토콜   │ 변환     │ 계층     │
+------------+----------+----------+
| ARP        │ IP→MAC   │ L2       │
| RARP       │ MAC→IP   │ L2       │
| NDP        │ IP→MAC   │ IPv6     │
+------------+----------+----------+
```

**핵심 알고리즸:** 브로드캐스트 요청, 캐싱

**📢 섹션 요약 비유:** NDP = IPv6의 ARP!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: ARP 보안**

| 공격 | 설명 |
|:---|:---|
| **ARP 스푸핑** | 가짜 MAC 광고 |
| **ARP Poisoning** | 캐시 오염 |
| **MITM** | 중간자 공격 |

**📢 섹션 요약 비유:** DAI = ARP 보안!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 일반 통신** → ARP 자동
**시나리오 2: 고정 MAC** → Static ARP
**시나리오 3: 보안** → DAI (Dynamic ARP Inspection)

**도입 체크리스트:** 캐시 타임아웃, Static ARP, DAI

**📢 섹션 요약 비유:** Static ARP = 보안 강화!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: L2 통신 가능

**미래 전망:** IPv6 NDP로 대체

**참고 표준:** RFC 826

**📢 섹션 요약 비유:** ARP는 이더넷 통신의 핵심!
