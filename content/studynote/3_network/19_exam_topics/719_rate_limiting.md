# Rate Limiting - 속도 제한

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 요청 빈도 제어
> 2. **가치**: 서비스 보호, 공정성
> 3. **융합:** API Gateway, Throttling, DDoS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Rate Limiting은 요청 빈도를 제한합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Rate Limiting                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              속도 제한 알고리즘                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Token Bucket:                                       │   │ │
│   │   │  [토큰] [토큰] [토큰] ──→ 요청당 1개 소모             │   │ │
│   │   │  (토큰이 없으면 대기)                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Leaky Bucket:                                       │   │ │
│   │   │  요청 ──→ [버킷] ──→ 일정 속도로 처리                  │   │ │
│   │   │  (버킷이 가득 차면 폐기)                               │   │ │
│   │   │                                                      │   │ │
│   │   │  Fixed Window:                                       │   │ │
│   │   │  [0-1분: 100회] [1-2분: 100회]                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Sliding Window:                                     │   │ │
│   │   │  지난 N초간 요청 수 카운트                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 속도 제한은 서비스 과부하를 방지합니다.

**📢 섹션 요약 비유:** Rate Limiting = 입장 인원 제한!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Limit** | 제한 횟수 |
| **Window** | 시간 창 |
| **Key** | 식별자 (IP, User) |
| **Action** | 제한 시 동작 |
| **Response** | 429 Too Many Requests |

**핵심 알고리즸:** 요청 → 카운트 → (허용/거부)

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**알고리즘 비교**

| 알고리즘 | 장점 | 단점 |
|:---|:---|:---|
| **Token Bucket** | 버스트 허용 | 메모리 |
| **Leaky Bucket** | 일정 처리 | 지연 |
| **Fixed Window** | 단순 | 경계 문제 |
| **Sliding Window** | 정확 | 복잡 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** API → 사용자별 100회/분
**시나리오 2:** 로그인 → IP별 5회/분
**시나리오 3:** DDoS → 전역 제한
