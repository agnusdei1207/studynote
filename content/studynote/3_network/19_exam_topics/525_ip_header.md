# IP Header - IP 헤더

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 패킷 제어 정보
> 2. **가치**: 라우팅, 단편화
> 3. **융합:** TTL, Protocol, Checksum

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IP Header는 20바이트 기본 구조입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IP Header (IPv4, 20 bytes)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IPv4 헤더 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  0         15        16            31               │   │ │
│   │   │  ├──────────────────┼─────────────────────────────┤ │   │ │
│   │   │  │Ver│IHL│TOS      │      Total Length            │ │   │ │
│   │   │  ├──────────────────────────────────────────────┤ │   │ │
│   │   │  │   Identification  │Flags│ Fragment Offset    │ │   │ │
│   │   │  ├──────────────────┼─────────────────────────────┤ │   │ │
│   │   │  │    TTL           │ Protocol │ Header Checksum │ │   │ │
│   │   │  ├──────────────────────────────────────────────┤ │   │ │
│   │   │  │            Source IP Address                  │ │   │ │
│   │   │  ├──────────────────────────────────────────────┤ │   │ │
│   │   │  │         Destination IP Address                │ │   │ │
│   │   │  ├──────────────────────────────────────────────┤ │   │ │
│   │   │  │            Options (if present)               │ │   │ │
│   │   │  └─────────────────────────────────────────────┘   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IP Header는 패킷 전달에 필요한 정보를 담습니다.

**📢 섹션 요약 비유:** IP Header = 패킷 주소!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 필드 | 크기 | 설명 |
|:---|:---|:---|
| **Version** | 4bit | IPv4=4 |
| **IHL** | 4bit | 헤더 길이 |
| **TOS** | 8bit | 서비스 타입 |
| **Total Length** | 16bit | 전체 길이 |
| **ID** | 16bit | 식별자 |
| **Flags** | 3bit | DF, MF |
| **Fragment Offset** | 13bit | 조각 위치 |
| **TTL** | 8bit | 생존 시간 |
| **Protocol** | 8bit | 상위 프로토콜 |
| **Checksum** | 16bit | 오류 검사 |
| **Src IP** | 32bit | 출발지 |
| **Dst IP** | 32bit | 목적지 |

**핵심 알고리즸:** 헤더 파싱 → 라우팅
