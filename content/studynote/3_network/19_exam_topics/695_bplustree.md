---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

B+Tree는 리프에만 데이터를 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    B+Tree                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              B+Tree 구조                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │              [30 | 70]  (내부 노드: 키만)            │   │ │
│   │   │             ╱    |    ╲                              │   │ │
│   │   │          [10|20] [50|60] [80|90]  (키만)             │   │ │
│   │   │          ╱  |  ╲   ╱  |  ╲   ╱  |  ╲                │   │ │
│   │   │        [1]→[15]→[25]→[40]→[55]→[65]→[75]→[85]→[95]  │   │ │
│   │   │         ↑                                         ↑   │   │ │
│   │   │         └───────── Linked List ────────────────────┘   │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  - 데이터는 리프에만                                  │   │ │
│   │   │  - 리프가 연결 리스트로 연결                          │   │ │
│   │   │  - 범위 검색에 효율적                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: B+Tree는 범위 검색과 순차 접근에 최적화됩니다.

**📢 섹션 요약 비유:** B+Tree = 연결 색인!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Internal Node** | 내부 노드 (키만) |
| **Leaf Node** | 리프 노드 (데이터) |
| **Linked List** | 연결 리스트 |
| **Duplicate Keys** | 키 중복 |
| **Fan-out** | 팬아웃 |

**핵심 알고리즸:** 검색 → 리프 도달 → 범위 스캔
