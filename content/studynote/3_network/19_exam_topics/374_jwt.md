# JWT - JSON 웹 토큰

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자가 포함 인증 토큰
> 2. **가치**: 무상태, 분산 인증
> 3. **융합**: OAuth, SSO, API

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

JWT(JSON Web Token)는 자가 포함된 인증 토큰입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    JWT                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              JWT 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Header.Payload.Signature                           │   │ │
│   │   │                                                      │   │ │
│   │   │  eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxMjM0NTY3ODkwIn0.│   │ │
│   │   │  dQw4W9vXk6QjI2N3l5t8u                               │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────┐  ┌─────────┐  ┌─────────┐             │   │ │
│   │   │  │ Header  │  │ Payload │  │Signature│             │   │ │
│   │   │  │{alg,typ}│  │{sub,exp}│  │  HMAC   │             │   │ │
│   │   │  └─────────┘  └─────────┘  └─────────┘             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              JWT 클레임                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 클레임     │ 설명     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ iss        │ 발급자   │ 식별     │               │   │ │
│   │   │  │ sub        │ 주체     │ 사용자   │               │   │ │
│   │   │  | aud        │ 수신자   │ 대상     │               │   │ │
│   │   │  │ exp        │ 만료     │ 보안     │               │   │ │
│   │   │  │ iat        │ 발급시간 │ 기록     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: JWT는 Header, Payload, Signature로 구성됩니다.

**💡 비유**: JWT는 여권입니다. 신원과 권한을 증명합니다.

**📢 섹션 요약 비유:** JWT = 자가 포함 토큰!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Header** | 알고리즘, 타입 |
| **Payload** | 클레임 |
| **Signature** | 서명 |
| **Base64url** | 인코딩 |

**비교표: JWT vs 세션**

```
+------------+----------+----------+
| 항목       │ JWT      │ 세션     │
+------------+----------+----------+
| 저장       │ 클라이언트│ 서버     │
| 확장성     │ 높음     │ 낮음     │
| 취소       │ 어려움   │ 쉬움     │
+------------+----------+----------+
```

**핵심 알고리즸:** HMAC, RSA, ECDSA

**📢 섹션 요약 비유:** HS256 = 대칭키!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 토큰 타입**

| 항목 | JWT | Opaque Token | Paseto |
|:---|:---|:---|:---|
| **자가포함** │ 예 │ 아니오 │ 예 |
| **보안** │ 중간 │ 높음 │ 높음 |
| **복잡도** │ 낮음 │ 낮음 │ 중간 |

**📢 섹션 요약 비유:** 분산 = JWT!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** SPA 인증 → JWT
**시나리오 2:** 마이크로서비스 → JWT
**시나리오 3:** 높은 보안 → 세션 + JWT

**도입 체크리스트:** 만료, 갱신, 취소

**📢 섹션 요약 비유:** API 인증 = JWT!
