# 스토리지 복제 (Storage Replication)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스토리지 실시간 복사
> 2. **가치**: 데이터 가용성, DR
> 3. **융합**: HA, DR과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

스토리지 복제는 데이터를 실시간 또는 주기적으로 다른 스토리지로 복사합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    스토리지 복제 (Storage Replication)               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              복제 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 동기 복제 (Synchronous):                          │   │ │
│   │   │                                                      │   │ │
│   │   │     [Primary] ──────▶ [Secondary]                   │   │ │
│   │   │         │       Write       │                        │   │ │
│   │   │         │◀────── Ack ───────│                        │   │ │
│   │   │         │                    │                        │   │ │
│   │   │     • RPO = 0                                        │   │ │
│   │   │     • 지연 발생 가능                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 비동기 복제 (Asynchronous):                       │   │ │
│   │   │                                                      │   │ │
│   │   │     [Primary] ──────▶ [Secondary]                   │   │ │
│   │   │         │       Queue       │                        │   │ │
│   │   │         │◀────── Later ─────│                        │   │ │
│   │   │         │                    │                        │   │ │
│   │   │     • RPO > 0                                        │   │ │
│   │   │     • 지연 최소                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              복제 방식                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 로컬 복제:                                        │   │ │
│   │   │     • 같은 데이터센터 내                              │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 원격 복제:                                        │   │ │
│   │   │     • 다른 데이터센터                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 저장소 수준:                                      │   │ │
│   │   │     • SAN 복제                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 소프트웨어 수준:                                  │   │ │
│   │   │     • DRBD, Ceph                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 동기 복제는 RPO 0, 비동기는 RPO > 0입니다.

**💡 비유**: 복제는 라이브 방송(동기)과 녹화(비동기)입니다.

**📢 섹션 요약 비유:** 동기 = 무손실!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Synchronous** | 동기 |
| **Asynchronous** | 비동기 |
| **Mirror** | 미러링 |
| **Journal** | 저널링 |

**비교표: 복제 방식 비교**

```
+------------+----------+----------+----------+
| 방식       │ RPO      │ 지연     │ 비용     │
+------------+----------+----------+----------+
| 동기       │ 0        │ 높음     │ 높음     │
| 비동기     │ > 0      │ 낮음     │ 중간     │
| 하이브리드 │ 낮음      │ 중간     │ 중간     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 쓰기 순서, 일관성

**📢 섹션 요약 비유:** 동기 복제 = 거리 제한!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 금융** → 동기 복제
**시나리오 2: 일반 기업** → 비동기 복제
**시나리오 3: 원격 DR** → 비동기 + 압축

**도입 체크리스트:** RPO, 대역폭, 거리

**📢 섹션 요약 비유:** 거리 = 지연!
