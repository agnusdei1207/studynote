# MQTT (Message Queuing Telemetry Transport) - 메시지 큐 텔레메트리 전송

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 경량 Pub/Sub 메시징
> 2. **가치**: 저대역폭, 불안정 네트워크
> 3. **융합:** IoT, Broker, QoS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MQTT는 경량 발행/구독 메시징 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MQTT                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MQTT 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Publisher                    Broker                 │   │ │
│   │   │  (Sensor)                     ┌─────┐                │   │ │
│   │   │  ┌─────┐  Publish             │     │                │   │ │
│   │   │  │ IoT │────────────────────→ │ MQTT│                │   │ │
│   │   │  └─────┘  Topic: temp         │Broker│                │   │ │
│   │   │                               │     │←── Subscribe   │   │ │
│   │   │                               └─────┘   Topic: temp  │   │ │
│   │   │                                   │                   │   │ │
│   │   │                                   ↓                   │   │ │
│   │   │                              Subscriber               │   │ │
│   │   │                              (App/Monitor)            │   │ │
│   │   │                                                      │   │ │
│   │   │  QoS 레벨:                                           │   │ │
│   │   │  0: At most once (Fire & Forget)                     │   │ │
│   │   │  1: At least once (Acknowledged)                     │   │ │
│   │   │  2: Exactly once (Assured)                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: MQTT는 Broker를 통해 Topic 기반 메시징을 제공합니다.

**📢 섹션 요약 비유:** MQTT = 경량 라디오!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Broker** | 메시지 중계 |
| **Publisher** | 메시지 발행 |
| **Subscriber** | 메시지 구독 |
| **Topic** | 주제 |
| **QoS** | 서비스 품질 |

**핵심 알고리즸:** Publish → Broker → Subscribe

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**MQTT vs HTTP**

| 항목 | MQTT | HTTP |
|:---|:---|:---|
| **모델** | Pub/Sub | Req/Res |
| **오버헤드** | 낮음 | 높음 |
| **실시간** | 좋음 | Polling |
