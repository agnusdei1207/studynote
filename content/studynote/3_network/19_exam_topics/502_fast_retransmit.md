# Fast Retransmit - 빠른 재전송

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 중복 ACK로 빠른 손실 탐지
> 2. **가치**: 타임아웃 대기 없는 재전송
> 3. **융합:** TCP Reno, Fast Recovery

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Fast Retransmit는 중복 ACK로 손실을 탐지합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Fast Retransmit                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Fast Retransmit 동작                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  송신: 1 2 3 4 5 6 7 8                              │   │ │
│   │   │              ↑ 손실                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  수신: 1 2 3    5 6 7 8                             │   │ │
│   │   │                                                      │   │ │
│   │   │  ACK:  2  3  4  4  4  4  (중복 ACK 3개)             │   │ │
│   │   │              ↑                                     │   │ │
│   │   │         Fast Retransmit 4                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 3개의 중복 ACK 시 즉시 재전송합니다.

**📢 섹션 요약 비유:** Fast Retransmit = 빠른 대응!
