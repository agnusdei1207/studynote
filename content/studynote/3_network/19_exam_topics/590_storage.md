# Storage - 스토리지

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 영구 저장
> 2. **가치**: 데이터 보존, 접근성
> 3. **융합:** DAS, NAS, SAN, Cloud

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Storage는 데이터를 영구적으로 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Storage                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Storage 유형                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  DAS (Direct Attached Storage):                      │   │ │
│   │   │  서버에 직접 연결 (내장 HDD, 외장)                    │   │ │
│   │   │                                                      │   │ │
│   │   │  NAS (Network Attached Storage):                     │   │ │
│   │   │  네트워크로 연결된 파일 서버                          │   │ │
│   │   │  프로토콜: NFS, SMB/CIFS                             │   │ │
│   │   │                                                      │   │ │
│   │   │  SAN (Storage Area Network):                         │   │ │
│   │   │  전용 네트워크로 연결된 블록 스토리지                  │   │ │
│   │   │  프로토콜: FC, iSCSI, FCoE                           │   │ │
│   │   │                                                      │   │ │
│   │   │  Cloud Storage:                                      │   │ │
│   │   │  S3, Blob, GCS 등 객체 스토리지                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Storage 유형에 따라 성능, 확장성, 비용이 다릅니다.

**📢 섹션 요약 비유:** Storage = 창고!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 유형 | 접근 | 프로토콜 |
|:---|:---|:---|
| **DAS** | 직접 | SATA/SAS |
| **NAS** | 파일 | NFS/SMB |
| **SAN** | 블록 | FC/iSCSI |
| **Object** | 객체 | REST API |

**핵심 알고리즸:** I/O 처리 → 저장 → 읽기

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Storage 비교**

|:---|:---|:---|:---|

| **공유** | 어려움 | 쉬움 | 쉬움 |
| **비용** | 낮음 | 중간 | 높음 |
