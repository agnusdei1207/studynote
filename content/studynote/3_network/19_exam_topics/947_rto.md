# RTO - 재전송 타임아웃

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 재전송 대기 시간
> 2. **가치**: 손실 감지와 복구
> 3. **융합:** RTT, Exponential Backoff, Karn's Algorithm

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RTO (Retransmission Timeout)는 재전송을 기다리는 시간입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RTO                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RTO 동작                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RTO 동작 과정:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  전송 ────────→ RTO 대기 ────────→ ACK 없음?      ││   │ │
│   │   │  │                   │                             ││   │ │
│   │   │  │                   ↓                             ││   │ │
│   │   │  │                재전송                            ││   │ │
│   │   │  │                   │                             ││   │ │
│   │   │  │                   ↓                             ││   │ │
│   │   │  │           RTO × 2 (지수 백오프)                    ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  RTO 계산:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ RTO = SRTT + max(G, 4×RTTVAR)                   ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ SRTT: Smoothed RTT (평균 RTT)                    ││   │ │
│   │   │  │ RTTVAR: RTT Variance (RTT 분산)                   ││   │ │
│   │   │  │ G: Clock Granularity (클럭 정밀도)                ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ 초기 RTO: 1초                                    ││   │ │
│   │   │  │ 최소 RTO: 0.2초                                  ││   │ │
│   │   │  │ 최대 RTO: 120초                                  ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RTO는 TCP 신뢰성의 핵심입니다.

**📢 섹션 요약 비유:** RTO = 응답 대기 시간!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SRTT** | 평활화 RTT |
| **RTTVAR** | RTT 분산 |
| **Backoff** | 백오프 |
| **Karn** | Karn 알고리즘 |
| **Timeout** | 타임아웃 |

**핵심 알고리듬:** RTT 측정 → SRTT 갱신 → RTO 계산

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**RTO 백오프**

| 재전송 | RTO 배수 |
|:---|:---|
| **1차** | 1x |
| **2차** | 2x |
| **3차** | 4x |
| **n차** | 2^(n-1)x |
