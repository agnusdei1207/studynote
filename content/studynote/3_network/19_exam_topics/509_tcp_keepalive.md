# TCP Keepalive - TCP 연결 유지

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 유휴 연결 활성 확인
> 2. **가치**: 좀비 연결 탐지
> 3. **융합:** TCP, Heartbeat, Timeout

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TCP Keepalive는 연결이 살아있는지 주기적으로 확인합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TCP Keepalive                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Keepalive 동작                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  클라이언트                    서버                   │   │ │
│   │   │     │                            │                    │   │ │
│   │   │     │ ←── Keepalive Probe ──────│ (idle 시간 경과)   │   │ │
│   │   │     │ ─── ACK ────────────────→ │                    │   │ │
│   │   │     │                            │                    │   │ │
│   │   │  → 연결 유지 확인                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  응답 없으면: 연결 종료                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Keepalive는 유휴 연결의 상대방 생존을 확인합니다.

**📢 섹션 요약 비유:** Keepalive = 생존 확인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 기본값 | 설명 |
|:---|:---|:---|
| **Idle Time** | 2시간 | 프로브 시작 |
| **Interval** | 75초 | 재시도 간격 |
| **Probes** | 9회 | 최대 시도 |
| **Probe** | 0바이트 | ACK 유도 |

**핵심 알고리즸:** idle > threshold → probe

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Keepalive vs Application Heartbeat**

| 항목 | Keepalive | Heartbeat |
|:---|:---|:---|
| **계층** | TCP | App |
| **간격** | 길음 | 짧음 |
| **용도** | 연결 확인 | 서비스 확인 |
