# Encoding - 인코딩

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 표현 변환
> 2. **가치**: 호환성 보장
> 3. **융합:** Base64, UTF-8, URL Encoding

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Encoding은 데이터를 다른 형식으로 변환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Encoding                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인코딩 유형                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Base64:                                              │   │ │
│   │   │  바이너리 → ASCII 문자                                 │   │ │
│   │   │  "Hello" → "SGVsbG8="                                │   │ │
│   │   │  이메일, JSON 바이너리                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  URL Encoding:                                        │   │ │
│   │   │  특수문자 → %XX                                       │   │ │
│   │   │  "hello world" → "hello%20world"                     │   │ │
│   │   │                                                      │   │ │
│   │   │  UTF-8:                                               │   │ │
│   │   │  유니코드 인코딩                                       │   │ │
│   │   │  가변 길이 (1~4 bytes)                                 │   │ │
│   │   │  한글: 3 bytes                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Hex:                                                 │   │ │
│   │   │  바이너리 → 16진수                                     │   │ │
│   │   │  0xFF → "FF"                                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 인코딩은 데이터 전송 호환성을 보장합니다.

**📢 섹션 요약 비유:** Encoding = 번역!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Base64** | 바이너리→텍스트 |
| **UTF-8** | 유니코드 |
| **URL Encode** | URL 안전 |
| **Hex** | 16진수 |
| **Percent** | 퍼센트 인코딩 |

**핵심 알고리즸:** 원본 → 규칙 → 인코딩 결과

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**인코딩 용도 비교**

| 인코딩 | 용도 | 오버헤드 |
|:---|:---|:---|
| **Base64** | 바이너리 전송 | 33% |
| **URL** | URL 안전 | 가변 |
| **UTF-8** | 국제화 | 없음 |
| **Hex** | 디버깅 | 100% |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 3:** 다국어 → UTF-8
