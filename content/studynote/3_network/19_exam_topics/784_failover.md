# Failover - 페일오버

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장애 시 서비스 이전
> 2. **가치**: 서비스 연속성
> 3. **융합:** HA, Heartbeat, Replication

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Failover는 장애 시 서비스를 이전합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Failover                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Failover 유형                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Active-Active:                                       │   │ │
│   │   │  ┌─────────┐ ┌─────────┐                            │   │ │
│   │   │  │ Active  │ │ Active  │ ── 부하 분산                │   │ │
│   │   │  └─────────┘ └─────────┘                            │   │ │
│   │   │                                                      │   │ │
│   │   │  Active-Passive:                                      │   │ │
│   │   │  ┌─────────┐ ┌─────────┐                            │   │ │
│   │   │  │ Active  │ │ Standby │ ── 장애 시 인계              │   │ │
│   │   │  └─────────┘ └─────────┘                            │   │ │
│   │   │                                                      │   │ │
│   │   │  Failover 과정:                                       │   │ │
│   │   │  1. 장애 감지 (Heartbeat Timeout)                     │   │ │
│   │   │  2. 리소스 확보 (Fencing)                             │   │ │
│   │   │  3. 서비스 시작                                        │   │ │
│   │   │  4. 클라이언트 리다이렉트                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Failover는 장애 시 서비스를 유지합니다.

**📢 섹션 요약 비유:** Failover = 교체 투수!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Primary** | 주 서버 |
| **Secondary** | 보조 서버 |
| **Detection** | 감지 |
| **Switchover** | 계획된 전환 |

**핵심 알고리즸:** 감지 → 전환 → 서비스 → 복구
