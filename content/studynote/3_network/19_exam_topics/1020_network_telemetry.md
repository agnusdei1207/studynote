# Network Telemetry - 네트워크 텔레메트리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실시간 데이터 수집
> 2. **가치**: 세밀한 가시성
> 3. **융합:** Streaming, gRPC, Model-Driven

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **YANG** | 데이터 모델링 언어 |
| **gRPC** | 원격 프로시저 호출 |
| **Streaming** | 스트리밍 |
| **Encoder** | 인코더 (JSON/Protobuf) |
| **Collector** | 수집기 |

**핵심 알고리듬:** 구독 → 스트리밍 → 디코딩 → 저장/분석

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Telemetry vs SNMP**

| 항목 | Telemetry | SNMP |
|:---|:---|:---|
| **방식** | Push | Pull |
| **지연** | 밀리초 | 초 |
| **오버헤드** | 낮음 | 높음 |
| **데이터** | 구조화 | 단순 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 데이터센터 → Model-Driven Telemetry
**시나리오 2:** 클라우드 → gRPC Telemetry
**시나리오 3:** 레거시 → SNMP + Telemetry 하이브리드
