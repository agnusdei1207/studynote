# Retry Pattern - 재시도 패턴

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 일시적 오류 재시도
> 2. **가치**: 복원력 향상
> 3. **융합:** Circuit Breaker, Backoff, Resilience

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Retry Pattern은 일시적 오류 시 재시도합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Retry Pattern                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              재시도 전략                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Fixed Interval:                                     │   │ │
│   │   │  Attempt 1 ──→ Wait 1s ──→ Attempt 2 ──→ Wait 1s    │   │ │
│   │   │                                                      │   │ │
│   │   │  Exponential Backoff:                                │   │ │
│   │   │  Attempt 1 ──→ Wait 1s ──→ Attempt 2 ──→ Wait 2s    │   │ │
│   │   │                               ──→ Attempt 3 ──→ Wait 4s│   │ │
│   │   │                                                      │   │ │
│   │   │  Jitter (지터):                                      │   │ │
│   │   │  Wait + random(0, jitter)                            │   │ │
│   │   │  (동시 재시도 방지)                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  최대 재시도 횟수 + 최대 대기 시간                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 재시도는 점진적 대기로 서버 부하를 줄입니다.

**📢 섹션 요약 비유:** Retry Pattern = 다시 걸기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Max Retries** | 최대 재시도 |
| **Backoff** | 백오프 |
| **Jitter** | 지터 |
| **Retryable** | 재시도 가능 오류 |
| **Circuit Breaker** | 서킷 브레이커 |

**핵심 알고리즸:** 실패 → 대기 → 재시도 → (성공/실패)

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)
