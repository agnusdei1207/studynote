# CARP - 공통 주소 이중화 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오픈소스 VRRP 대안
> 2. **가치**: 특허 회피 이중화
> 3. **융합:** Master, Backup, VHID

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CARP (Common Address Redundancy Protocol)는 오픈소스 이중화 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CARP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CARP 동작                                         │ │

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  CARP 특징:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  OpenBSD 개발 (VRRP 특허 회피)                    ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  ┌───────────────┐      ┌───────────────┐       ││   │ │
│   │   │  │  │    Master     │      │    Backup     │       ││   │ │

│   │   │  │              │ Shared IP   │                     ││   │ │
│   │   │  │              │ VHID: 1     │                     ││   │ │
```

> **해설**: CARP는 BSD 기반 시스템의 대표적인 이중화 프로토콜입니다.

**📢 섹션 요약 비유:** CARP = 오픈소스 대체재!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **VHID** | Virtual Host ID |
| **Advskew** | 광고 지연값 |
| **Advbase** | 광고 기본 간격 |
| **Passphrase** | 인증 암호 |
| **Demote** | 우선순위 감소 |

**핵심 알고리듬:** Advskew 비교 → 마스터 선출 → 광고 → HMAC 인증

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**CARP vs VRRP**

| 항목 | CARP | VRRP |
|:---|:---|:---|
| **개발** | OpenBSD | IETF |
| **특허** | 없음 | 존재 |
| **인증** | HMAC | 선택적 |
| **프로토콜 번호** | 112 | 112 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** BSD 환경 → CARP
**시나리오 2:** Linux → Keepalived
**시나리오 3:** 상용 장비 → VRRP
