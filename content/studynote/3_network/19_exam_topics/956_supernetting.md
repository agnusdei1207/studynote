# Supernetting - 슈퍼네팅

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 결합
> 2. **가치**: 라우팅 테이블 축소
> 3. **융합:** Route Aggregation, CIDR, Summarization

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Supernetting은 여러 네트워크를 하나로 결합합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Supernetting                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              슈퍼네팅 과정                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  네트워크 결합:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  개별 네트워크:                                   ││   │ │
│   │   │  │  192.168.0.0/24                                 ││   │ │
│   │   │  │  192.168.1.0/24                                 ││   │ │
│   │   │  │  192.168.2.0/24                                 ││   │ │
│   │   │  │  192.168.3.0/24                                 ││   │ │
│   │   │  │           │                                     ││   │ │
│   │   │  │           ▼                                     ││   │ │
│   │   │  │           ↓ Supernetting                        ││   │ │
│   │   │  │           │                                     ││   │ │
│   │   │  │  192.168.0.0/22 (집약된 네트워크)                  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  조건:                                                  │   │ │
│   │   │  - 연속된 네트워크                                       │   │ │
│   │   │  - 2의 거듭제곱 개수                                     │   │ │
│   │   │  - 첫 번째 네트워크가 집약 크기에 정렬                     │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                  │   │ │
│   │   │  - 라우팅 테이블 크기 감소                                │   │ │
│   │   │  - 라우팅 속도 향상                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 슈퍼네팅은 인터넷 라우팅의 핵심입니다.

**📢 섹션 요약 비유:** Supernetting = 우편번호 통합!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Aggregation** | 집약 |
| **Summary** | 요약 |
| **Prefix** | 프리픽스 |
| **Contiguous** | 연속성 |
| **Alignment** | 정렬 |

**핵심 알고리듬:** 공통 비트 확인 → 프리픽스 단축

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Subnetting vs Supernetting**

| 항목 | Subnetting | Supernetting |
|:---|:---|:---|
| **방향** | 분할 | 결합 |
| **프리픽스** | 증가 | 감소 |
| **용도** | 내부 | 외부 |
| **목적** | 주소 관리 | 라우팅 축소 |
