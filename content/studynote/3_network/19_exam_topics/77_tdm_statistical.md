# 비동기식/통계적 시분할 다중화 (Statistical TDM)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터가 있는 채널에만 동적 슬롯 할당
> 2. **가치**: 대역폭 효율, 버스트 트래픽 대응
> 3. **융합**: 인터넷, 패킷망, ATM과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

통계적 TDM(Statistical TDM) 또는 비동기식 TDM은 데이터가 있는 채널에만 타임슬롯을 동적으로 할당하여 대역폭 효율을 높이는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    통계적 TDM (Statistical TDM)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              통계적 TDM 구조                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  입력 vs 전송:                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  입력 CH1: ████████                    (데이터 있음)  │   │ │
│   │   │  입력 CH2: ░░░░░░░░░░░░░░░░░░░░░░░░░░░░ (데이터 없음) │   │ │
│   │   │  입력 CH3: ████████                    (데이터 있음)  │   │ │
│   │   │  입력 CH4: ████████                    (데이터 있음)  │   │ │
│   │   │                                                      │   │ │
│   │   │  동기식 TDM:                                           │   │ │
│   │   │  ├────┼────┼────┼────┤                               │   │ │
│   │   │  │CH1 │CH2 │CH3 │CH4 │  ← 낭비 발생!                 │   │ │
│   │   │  │ D  │ ID │ D  │ D  │                               │   │ │
│   │   │  └────┴────┴────┴────┘                               │   │ │
│   │   │                                                      │   │ │
│   │   │  통계적 TDM:                                           │   │ │
│   │   │  ├────┬────┬────┬────┬────┤                          │   │ │
│   │   │  │CH1 │CH3 │CH4 │CH1 │... │  ← 효율적!               │   │ │
│   │   │  │ D  │ D  │ D  │ D  │    │                          │   │ │
│   │   │  └────┴────┴────┴────┴────┘                          │   │ │
│   │   │         ↑                                            │   │ │
│   │   │   데이터 있는 채널만 전송                              │   │ │
│   │   │                                                      │   │ │
│   │   │  주소 필요: 어떤 채널인지 식별 위해                    │   │ │
│   │   │  ┌────┬────┬────┐                                   │   │ │
│   │   │  │ADDR│DATA │CRC │                                   │   │ │
│   │   │  └────┴────┴────┘                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              통계적 TDM 특징                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  장점:                                                 │   │ │
│   │   │  • 높은 대역폭 효율                                    │   │ │
│   │   │  • 버스트 트래픽 적응                                  │   │ │
│   │   │  • 유휴 슬롯 없음                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  단점:                                                 │   │ │
│   │   │  • 주소 오버헤드                                       │   │ │
│   │   │  • 가변 지연                                          │   │ │
│   │   │  • 버퍼링 필요                                        │   │ │
│   │   │  • 복잡한 구현                                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 통계적 TDM은 패킷 스위칭의 기반이 됩니다.

**💡 비유**: 통계적 TDM은 발언하고 싶은 사람만 발언권을 갖는 것입니다.

**📢 섹션 요약 비유**: 데이터 있을 때만 전송!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **주소** | 채널 식별 | 4-8비트 | 이름표 |
| **버퍼** | 대기열 | 큐 | 대기실 |
| **스케줄러** | 할당 | 알고리즘 | 중재자 |
| **헤더** | 제어 정보 | ADDR+CRC | 주소 |

**비교표**

```
+------------+----------+----------+
| 항목       | 동기식   | 통계적   |
+------------+----------+----------+
| 슬롯 할당  | 고정     | 동적     |
| 대역폭     | 낭비     | 효율     |
| 지연       | 고정     | 가변     |
| 오버헤드   | 없음     | 있음     |
| 복잡도     | 낮음     | 높음     |
+------------+----------+----------+
```

**핵심 알고리즸**: 입력 → 버퍼링 → 스케줄링 → ADDR+DATA 전송

**📢 섹션 요약 비유**: ADDR = 채널 식별자!

**📢 섹션 요약 비유**: 저활용 시 통계적 TDM 유리!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 데이터망** → 통계적 TDM
**시나리오 2: 인터넷** → 패킷 스위칭 (통계적 TDM 기반)
**시나리오 3: ATM** → 셀 스위칭

**도입 체크리스트**: 버스트 트래픽, 버퍼, 지연 허용

**📢 섹션 요약 비유**: 데이터는 통계적 TDM!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 2-4배 대역폭 효율

**미래 전망**: 패킷 기반 네트워크 확장

**참고 표준**: Frame Relay, ATM

**📢 섹션 요약 비유**: 통계적 TDM은 인터넷의 기초!
