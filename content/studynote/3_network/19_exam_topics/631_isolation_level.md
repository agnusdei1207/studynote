# Isolation Level - 격리 수준

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동시성 제어 수준
> 2. **가치**: 일관성 vs 성능 균형
> 3. **융합:** Transaction, ACID, Lock

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Isolation Level은 트랜잭션 간 격리 정도를 정의합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Isolation Level                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              격리 수준 계층                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │                  Serializable                        │   │ │
│   │   │                    (완전 격리)                        │   │ │
│   │   │                        ↑                             │   │ │
│   │   │                  Repeatable Read                      │   │ │
│   │   │                    (반복 읽기)                         │   │ │
│   │   │                        ↑                             │   │ │
│   │   │                  Read Committed                        │   │ │
│   │   │                    (커밋된 읽기)                       │   │ │
│   │   │                        ↑                             │   │ │
│   │   │                  Read Uncommitted                      │   │ │
│   │   │                    (커밋 안 된 읽기)                   │   │ │
│   │   │                                                      │   │ │
│   │   │  ←─ 격리 강도 ↑ / 동시성 ↓ / 성능 ↓ ─→                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 격리 수준이 높을수록 일관성은 높지만 성능은 떨어집니다.

**📢 섹션 요약 비유:** Isolation Level = 방음벽!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | Dirty Read | Non-Repeatable | Phantom |
|:---|:---|:---|:---|
| **Read Uncommitted** | 가능 | 가능 | 가능 |
| **Read Committed** | 방지 | 가능 | 가능 |
| **Repeatable Read** | 방지 | 방지 | 가능 |
| **Serializable** | 방지 | 방지 | 방지 |

**핵심 알고리즸:** 격리 수준 설정 → Lock 적용 → 트랜잭션 실행

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**현상 설명**

| 현상 | 설명 |
|:---|:---|
| **Dirty Read** | 커밋되지 않은 데이터 읽기 |
| **Non-Repeatable Read** | 같은 데이터가 다르게 읽힘 |
| **Phantom Read** | 새로운 행이 나타남/사라짐 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 금융 → Serializable
**시나리오 2:** 웹 서비스 → Read Committed
**시나리오 3:** 대용량 → Read Uncommitted
