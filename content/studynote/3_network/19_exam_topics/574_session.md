# Session - 세션

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서버 측 상태 저장
> 2. **가치**: 사용자 인증, 상태 유지
> 3. **융합:** Cookie, Session ID, Redis

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Session은 서버에 사용자 상태를 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Session                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Session 동작                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 로그인 → 서버가 세션 생성                         │   │ │
│   │   │  2. Session ID를 쿠키로 클라이언트에 전달             │   │ │
│   │   │  3. 이후 요청 시 쿠키에 Session ID 포함               │   │ │
│   │   │  4. 서버는 Session ID로 세션 조회                     │   │ │
│   │   │                                                      │   │ │
│   │   │  서버 저장소:                                         │   │ │
│   │   │  - Memory (기본)                                     │   │ │
│   │   │  - Redis/Memcached (분산)                            │   │ │
│   │   │  - Database                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Session ID만 쿠키로 전송하고 데이터는 서버에 저장합니다.

**📢 섹션 요약 비유:** Session = 사물함 키!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Session ID** | 고유 식별자 |
| **Session Store** | 저장소 |
| **Timeout** | 만료 시간 |
| **Cookie** | ID 전송 |

**핵심 알고리즸:** ID 생성 → 저장 → 검증

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Session vs Token**

| 항목 | Session | Token |
|:---|:---|:---|
| **저장** | 서버 | 클라이언트 |
| **확장** | 어려움 | 쉬움 |
| **보안** | 서버 관리 | 클라이언트 관리 |
