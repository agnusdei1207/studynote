# Cache Eviction - 캐시 제거

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 캐시에서 항목 제거
> 2. **가치**: 공간 확보, 데이터 신선도
> 3. **융합:** LRU, LFU, TTL

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Cache Eviction은 캐시에서 항목을 제거합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Cache Eviction                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              제거 정책                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  LRU (Least Recently Used):                           │   │ │
│   │   │  가장 오래 사용하지 않은 것 제거                        │   │ │
│   │   │  [A][B][C][D][E] → 접근 C → [A][B][D][E][C]          │   │ │
│   │   │  가득 차면 A 제거                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  LFU (Least Frequently Used):                         │   │ │
│   │   │  사용 빈도가 가장 낮은 것 제거                          │   │ │
│   │   │  A:5, B:3, C:1 → C 제거                               │   │ │
│   │   │                                                      │   │ │
│   │   │  FIFO (First In First Out):                           │   │ │
│   │   │  가장 먼저 들어온 것 제거                               │   │ │
│   │   │                                                      │   │ │
│   │   │  TTL (Time To Live):                                  │   │ │
│   │   │  만료된 항목 제거                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  Random:                                              │   │ │
│   │   │  무작위 제거                                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 제거 정책은 캐시 효율성을 결정합니다.

**📢 섹션 요약 비유:** Cache Eviction = 냉장고 정리!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **LRU** | 최근 미사용 |
| **LFU** | 빈도 낮음 |
| **FIFO** | 선입선출 |
| **TTL** | 시간 만료 |
| **Size-based** | 크기 기반 |

**핵심 알고리즸:** 공간 부족 → 정책 적용 → 항목 제거

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**정책 비교**

| 정책 | 장점 | 단점 |
|:---|:---|:---|
| **LRU** | 직관적 | 순차 스캔에 취약 |
| **LFU** | 빈도 반영 | 새 항목 불리 |
| **TTL** | 단순 | 만료 전 제거 가능 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일반 → LRU
**시나리오 2:** 순차 → LFU
**시나리오 3:** 시간 민감 → TTL
