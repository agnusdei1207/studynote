# Ping - 핑

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Ping은 ICMP Echo로 호스트 연결을 확인합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Ping                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Ping 동작                                         │ │

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ICMP Echo Request/Reply:                              │   │ │
│   │   │  ┌─────┐                      ┌─────┐                │   │ │
│   │   │  │     │ ── Echo Req (TTL) ─→ │     │                │   │ │
│   │   │  │     │ ←─ Echo Reply ────── │     │                │   │ │
│   │   │  │ Src │     (시간 측정)       │ Dst │                │   │ │
│   │   │  └─────┘                      └─────┘                │   │ │
│   │   │                                                      │   │ │

│   │   │  │   time=15.2 ms                                  ││   │ │
```

> **해설**: Ping은 가장 기본적인 네트워크 진단 도구입니다.

**📢 섹션 요약 비유:** Ping = 네트워크 초인종!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Echo Request** | 요청 |
| **Echo Reply** | 응답 |
| **RTT** | 왕복 시간 |
| **TTL** | Time to Live |
| **Loss** | 손실률 |

**핵심 알고리즸:** Echo Request → 응답 대기 → RTT 계산

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Ping 결과 해석**

| 결과 | 의미 |
|:---|:---|
| **Reply** | 연결됨 |
| **Timeout** | 방화벽/도달 불가 |
| **Unreachable** | 경로 없음 |
| **High Latency** | 혼잡 |
