# RabbitMQ - 래빗엠큐

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메시지 브로커
> 2. **가치**: 신뢰성, 유연한 라우팅
> 3. **융합:** AMQP, Queue, Exchange

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RabbitMQ는 메시지 브로커입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RabbitMQ                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RabbitMQ 아키텍처                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Producer                                             │   │ │
│   │   │     │                                                 │   │ │
│   │   │     ↓                                                 │   │ │
│   │   │  ┌─────────┐                                         │   │ │
│   │   │  │ Exchange │ ── Routing ──→ Queue ──→ Consumer     │   │ │
│   │   │  │         │    Key         │                       │   │ │
│   │   │  └─────────┘               │                       │   │ │
│   │   │                            ↓                       │   │ │
│   │   │                       ┌─────────┐                  │   │ │
│   │   │                       │ Queue 2 │ ──→ Consumer     │   │ │
│   │   │                       └─────────┘                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Exchange Types:                                      │   │ │
│   │   │  - Direct: 정확한 라우팅 키 매칭                       │   │ │
│   │   │  - Fanout: 모든 큐에 브로드캐스트                       │   │ │
│   │   │  - Topic: 패턴 매칭                                   │   │ │
│   │   │  - Headers: 헤더 기반                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RabbitMQ는 Exchange와 Queue로 메시지를 라우팅합니다.

**📢 섹션 요약 비유:** RabbitMQ = 우체국!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Exchange** | 교환기 |
| **Queue** | 큐 |
| **Binding** | 바인딩 |
| **Routing Key** | 라우팅 키 |
| **VHost** | 가상 호스트 |

**핵심 알고리즸:** 발행 → Exchange → 라우팅 → Queue → 소비

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**RabbitMQ vs Kafka**

| 항목 | RabbitMQ | Kafka |
|:---|:---|:---|
| **모델** | Smart Broker | Smart Consumer |
| **지속성** | 큐 기반 | 로그 기반 |
| **순서** | 큐 내 보장 | 파티션 내 보장 |
| **재생** | 제한적 | 가능 |
