# TOML - 설정 형식

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명확한 설정 데이터 형식
> 2. **가치**: 간단한 문법, 타입 명확
> 3. **융합**: Rust, Python, Go 설정

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TOML(Tom's Obvious Minimal Language)은 명확한 설정 형식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TOML                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TOML 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  # 사용자 정보                                       │   │ │
│   │   │  name = "홍길동"                                     │   │ │
│   │   │  age = 30                                            │   │ │
│   │   │                                                      │   │ │
│   │   │  [address]                                           │   │ │
│   │   │  city = "서울"                                       │   │ │
│   │   │  zip = "12345"                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  hobbies = ["독서", "운동"]                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TOML 특징                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 특징       │ 설명     │ 가치     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 명확성     │ 타입     │ 안전     │               │   │ │
│   │   │  │ 섹션       │ []       │ 구조     │               │   │ │
│   │   │  | 주석       │ # 지원   │ 문서화   │               │   │ │
│   │   │  │ 파싱       │ 간단     │ 속도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: TOML은 key = "value" 형식으로 명확하게 표현합니다.

**💡 비유**: TOML은 양식입니다. 항목을 명확히 채웁니다.

**📢 섹션 요약 비유:** TOML = 명확한 설정!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Table** | 섹션 [section] |
| **Key/Value** | 키 = 값 |
| **Array** | 배열 [...] |
| **Inline Table** | 인라인 { } |

**비교표: TOML vs YAML**

```
+------------+----------+----------+
| 항목       │ TOML     │ YAML     │
+------------+----------+----------+
| 문법       │ 명확     │ 유연     │
| 들여쓰기   │ 불필요   │ 필수     │
| 파싱       │ 간단     │ 복잡     │
| 타입       │ 명확     │ 추론     │
+------------+----------+----------+
```

**핵심 알고리즸:** 키-값 파싱, 섹션

**📢 섹션 요약 비유:** key = value = 명확!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 설정 형식**

| 항목 | TOML | YAML | INI |
|:---|:---|:---|:---|
| **타입** │ 명확 │ 추론 │ 문자열 |
| **복잡도** │ 낮음 │ 중간 │ 낮음 |
| **Rust** │ 인기 │ 제한 │ 제한 |

**📢 섹션 요약 비유:** Rust = TOML!
