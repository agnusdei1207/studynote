# ADPCM (적응형 차분 펄스 부호 변조)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 신호 변화에 따라 양자화 스텝 동적 조절
> 2. **가치**: DPCM 대비 더 높은 압축 효율
> 3. **융합**: VoIP, 음성 메일, 게임 오디오와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ADPCM(Adaptive DPCM)은 신호의 변화율에 따라 양자화 스텝 크기를 동적으로 조절하는 DPCM의 개선판입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ADPCM (적응형 차분 펄스 부호 변조)                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ADPCM 원리                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  DPCM:                                                   │   │ │
│   │   │  • 고정 양자화 스텝                                      │   │ │
│   │   │  • 급격한 변화에 대응 어려움                             │   │ │
│   │   │                                                      │   │ │
│   │   │  ADPCM:                                                  │   │ │
│   │   │  • 적응형 양자화 스텝                                    │   │ │
│   │   │  • 큰 변화: 큰 스텝                                     │   │ │
│   │   │  • 작은 변화: 작은 스텝                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  스텝 크기 적응:                                         │   │ │
│   │   │  Δ[n+1] = Δ[n] × M[|I[n]|]                             │   │ │
│   │   │  (M: 곱셈기 테이블, I: 양자화 인덱스)                    │   │ │
│   │   │                                                      │   │ │
│   │   │  예:                                                   │   │ │
│   │   │  • 작은 차분: Δ 유지 또는 감소                          │   │ │
│   │   │  • 큰 차분: Δ 증가 (급변 대응)                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ADPCM 장단점                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┘   │ │
│   │   │  장점:                                                 │   │ │
│   │   │  • DPCM 대비 높은 압축률                                │   │ │
│   │   │  • 32kbps에서 좋은 음질                                 │   │ │
│   │   │  • 동적 범위 개선                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  단점:                                                 │   │ │
│   │   │  • 구현 복잡도 증가                                     │   │ │
│   │   │  • 에러 전파 가능                                      │   │ │
│   │   │  • 지연 발생                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  응용:                                                 │   │ │
│   │   │  • VoIP (G.726)                                        │   │ │
│   │   │  • 음성 메일                                           │   │ │
│   │   │  • 게임 오디오 (IMA ADPCM)                              │   │ │
│   │   │  • DECT 무선전화                                       │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: ADPCM은 PCM 대비 50% 대역폭 절감과 비슷한 품질을 제공합니다.

**💡 비유**: ADPCM은 급커브길에서 속도를 조절하는 것입니다.

**📢 섹션 요약 비유**: 64kbps → 32kbps = ADPCM!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **적응 양자화** | 스텝 조절 | Δ 동적 | 가변 |
| **예측기** | 이전값 예측 | 알고리즘 | 추정 |
| **곱셈기** | 스텝 배율 | M 테이블 | 조절 |
| **인덱스** | 양자화 레벨 | 4비트 | 값 |

**비교표**

```
+------------+----------+----------+----------+
| 항목       | PCM      | DPCM     | ADPCM    |
+------------+----------+----------+----------+
| 비트율     | 64kbps   | 32kbps   | 16-40kbps|
| 적응성     | 없음     | 없음     | 있음     |
| 품질       | 최고     | 좋음     | 좋음     |
| 복잡도     | 낮음     | 중간     | 높음     |
+------------+----------+----------+----------+
```

**핵심 알고리즘**: 차분 → 적응 양자화 → 인덱스 전송 → 적응 복원

**📢 섹션 요약 비유**: G.726이 ADPCM 표준!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: VoIP** → G.726 32kbps
**시나리오 2: 게임** → IMA ADPCM
**시나리오 3: DECT** → G.726 32kbps

**도입 체크리스트**: 대역폭, 품질, 지연

**📢 섹션 요약 비유**: 대역폭 제한 시 ADPCM!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 50% 대역폭 절감

**미래 전망**: Opus, AAC로 이동

**참고 표준**: ITU-T G.726, G.722

**📢 섹션 요약 비유**: ADPCM은 적응의 지혜!
