# Socket - 소켓

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 통신 종단점
> 2. **가치**: API 추상화
> 3. **융합:** TCP Socket, UDP Socket, Unix Socket

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Socket은 네트워크 통신의 종단점입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Socket                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              소켓 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Socket 주소:                                         │   │ │
│   │   │  IP Address + Port Number                             │   │ │
│   │   │  예: 192.168.1.100:8080                               │   │ │
│   │   │                                                      │   │ │
│   │   │  유형:                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ SOCK_STREAM: TCP (신뢰적, 연결형)                ││   │ │
│   │   │  │ SOCK_DGRAM:  UDP (비신뢰적, 비연결형)            ││   │ │
│   │   │  │ SOCK_RAW:   Raw (직접 접근)                      ││   │ │
│   │   │  │ UNIX Socket: 로컬 통신                           ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  TCP 연결 식별:                                       │   │ │
│   │   │  (Src IP, Src Port, Dst IP, Dst Port)               │   │ │
│   │   │  → 4-tuple                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  API:                                                 │   │ │
│   │   │  socket() → bind() → listen() → accept()             │   │ │
│   │   │         connect() → send()/recv() → close()          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 소켓은 애플리케이션과 네트워크의 인터페이스입니다.

**📢 섹션 요약 비유:** Socket = 전화기!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **socket()** | 소켓 생성 |
| **bind()** | 주소 바인딩 |
| **listen()** | 대기 상태 |
| **accept()** | 연결 수락 |
| **connect()** | 연결 요청 |

**핵심 알고리즸:** 생성 → 바인딩 → 연결 → 통신 → 종료

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**소켓 유형 비교**

| 유형 | 신뢰성 | 연결 | 용도 |
|:---|:---|:---|:---|
| **TCP** | 있음 | 필요 | 웹, 이메일 |
| **UDP** | 없음 | 불필요 | 스트리밍 |
| **Raw** | 없음 | 불필요 | 패킷 캡처 |
| **Unix** | 있음 | 선택 | 로컬 IPC |
