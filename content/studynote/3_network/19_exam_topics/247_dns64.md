# DNS64 - IPv6용 DNS 합성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: A 레코드를 AAAA로 합성
> 2. **가치**: NAT64와 함께 IPv4 서버 접속
> 3. **융합**: NAT64, 클라이언트와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DNS64는 IPv4 전용 서버의 A 레코드를 IPv6 AAAA 레코드로 합성하는 DNS 서비스입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DNS64                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DNS64 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 클라이언트 → DNS64: AAAA 쿼리                   │   │ │
│   │   │     예: www.example.com AAAA?                       │   │ │
│   │   │                                                      │   │ │
│   │   │  2. DNS64 → 권한 서버: AAAA 쿼리                    │   │ │
│   │   │     결과 없음                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  3. DNS64 → 권한 서버: A 쿼리                        │   │ │
│   │   │     결과: 192.0.2.1                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  4. DNS64: AAAA 레코드 합성                          │   │ │
│   │   │     64:ff9b::192.0.2.1 (NAT64 Prefix + IPv4)        │   │ │
│   │   │                                                      │   │ │
│   │   │  5. 클라이언트: 합성된 IPv6 주소 사용                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              합성된 IPv6 주소                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  합성 주소: Prefix + IPv4                            │   │ │
│   │   │                                                      │   │ │
│   │   │  Well-Known Prefix: 64:ff9b::/96                    │   │ │
│   │   │  예: 64:ff9b::192.0.2.1                             │   │ │
│   │   │  = 64:ff9b::c000:201                                │   │ │
│   │   │                                                      │   │ │
│   │   │  Network-Specific Prefix: 관리자 정의                │   │ │
│   │   │  예: 2001:db8:64::/96                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DNS64는 IPv4 전용 서버의 A 레코드를 IPv6 주소로 합성합니다.

**💡 비유**: DNS64는 번역기입니다. IPv4 주소를 IPv6로 변환합니다.

**📢 섹션 요약 비유:** DNS64 + NAT64 = 완벽!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Prefix** | 64:ff9b::/96 (Well-Known) |
| **AAAA 쿼리** | 먼저 시도 |
| **A 쿼리** | AAAA 없을 때 |
| **합성** | Prefix + IPv4 |

**비교표: DNS64 설정**

```
+------------+----------+----------+
| 항목       │ 기본     │ 커스텀   │
+------------+----------+----------+
| Prefix     │ 64:ff9b  │ 사용자   │
| 제외       │ 없음     │ 가능     │
| 매핑       │ 모든 A   │ 선택적   │
+------------+----------+----------+
```

**핵심 알고리즸:** AAAA → A → 합성

**📢 섹션 요약 비유:** 합성 = Prefix + IPv4!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: DNS64 예외**

| 예외 | 설명 |
|:---|:---|
| **DNSSEC** | 검증 실패 시 |
| **제외 목록** | 특정 도메인 |
| **로컬 AAAA** | 이미 있는 경우 |

**📢 섹션 요약 비유:** DNSSEC = 주의 필요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: IPv6 Only** → DNS64 + NAT64
**시나리오 2: Dual Stack** → DNS64 불필요
**시나리오 3: 하이브리드** → 선택적 DNS64

**도입 체크리스트:** Prefix, NAT64 연동, DNSSEC

**📢 섹션 요약 비유:** DNS64는 NAT64의 파트너!
