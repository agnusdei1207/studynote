# GLBP - 게이트웨이 로드밸런싱 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스코 로드밸런싱 FHRP
> 2. **가치**: 대역폭 활용 최적화
> 3. **융합**: HSRP, VRRP와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

GLBP(Gateway Load Balancing Protocol)는 시스코의 게이트웨이 이중화 및 로드밸런싱 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    GLBP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │

│   │              GLBP 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Host] [Host] [Host] [Host]                        │   │ │
│   │   │     │      │      │      │                          │   │ │
│   │   │     │      │      │      │                          │   │ │
│   │   │     ↓      ↓      ↓      ↓                          │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           GLBP 그룹                          │    │   │ │

│   │   │  │  │Virtual IP │        │Virtual IP │         │    │   │ │
```

> **해설**: GLBP는 여러 라우터가 동시에 트래픽을 처리합니다.

**💡 비유**: GLBP는 다중 출구 건물입니다. 여러 문에서 동시에 출입합니다.

**📢 섹션 요약 비유:** GLBP = 로드밸런싱!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **AVG** | 활성 게이트웨이 |
| **AVF** | 활성 포워더 |
| **Weight** | 부하 분산 비율 |
| **VMAC** | 가상 MAC |

**비교표: 로드밸런싱 방식**

```
+------------+----------+----------+
| 방식       │ 설명     │ 특징     │
+------------+----------+----------+
| Round-robin│ 순차     │ 기본     │
| Weighted   │ 가중치   │ 비율 조절│
| Host-dependent│ 호스트 │ 고정     │
+------------+----------+----------+
```

**핵심 알고리즸:** AVG 선출, AVF 할당

**📢 섹션 요약 비유:** Weight = 분산 비율!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FHRP**

| 프로토콜 | 로드밸런싱 | 라우터 수 |
|:---|:---:|:---|
| **HSRP** | 없음 | 1 Active |
| **VRRP** | 없음 | 1 Master |
| **GLBP** | 있음 | 최대 4 AVF |

**📢 섹션 요약 비유:** GLBP = 대역폭 최적!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 대역폭 활용** → GLBP
**시나리오 2: 단순 이중화** → HSRP
**시나리오 3: 멀티벤더** → VRRP

**도입 체크리스트:** Weight, Priority, Preempt

**📢 섹션 요약 비유:** 최대 4 AVF!

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대역폭 활용, 고가용성

**미래 전망:** VRRP + 로드밸런싱
