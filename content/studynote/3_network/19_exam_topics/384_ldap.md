# LDAP - 경량 디렉터리 액세스 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디렉터리 서비스 프로토콜
> 2. **가치**: 중앙 집중식 사용자/자원 관리
> 3. **융합**: Active Directory, OpenLDAP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

LDAP(Lightweight Directory Access Protocol)는 디렉터리 서비스에 접근합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LDAP                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LDAP 디렉터리 구조                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  dc=example,dc=com (Root)                           │   │ │
│   │   │       │                                              │   │ │
│   │   │       ├── ou=users                                   │   │ │
│   │   │       │       ├── cn=John Doe                        │   │ │
│   │   │       │       └── cn=Jane Smith                      │   │ │
│   │   │       │                                              │   │ │
│   │   │       └── ou=groups                                  │   │ │
│   │   │               ├── cn=admins                          │   │ │
│   │   │               └── cn=users                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LDAP 작업                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 작업       │ 설명     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Bind       │ 인증     │ 접속     │               │   │ │
│   │   │  │ Search     │ 검색     │ 조회     │               │   │ │
│   │   │  | Add        │ 추가     │ 생성     │               │   │ │
│   │   │  │ Modify     │ 수정     │ 변경     │               │   │ │
│   │   │  │ Delete     │ 삭제     │ 제거     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: LDAP은 트리 구조로 정보를 계층적으로 저장합니다.

**💡 비유**: LDAP은 전화번호부입니다. 사람과 자원을 찾습니다.

**📢 섹션 요약 비유:** LDAP = 디렉터리!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **DN** | Distinguished Name |
| **OU** | Organizational Unit |
| **CN** | Common Name |
| **DC** | Domain Component |

**비교표: LDAP vs RDBMS**

```
+------------+----------+----------+
| 항목       │ LDAP     │ RDBMS    │
+------------+----------+----------+
| 데이터     │ 트리     │ 테이블   │
| 읽기       │ 빠름     │ 중간     │
| 쓰기       │ 느림     │ 빠름     │
| 용도       │ 조회     │ 트랜잭션 │
+------------+----------+----------+
```

**핵심 알고리즸:** DN 검색, 필터링

**📢 섹션 요약 비유:** 조회 = LDAP!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

| 항목 | LDAP | AD | NIS |
|:---|:---|:---|:---|
