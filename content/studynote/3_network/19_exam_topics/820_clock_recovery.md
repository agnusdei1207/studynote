# Clock Recovery - 클럭 복구

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 타이밍 신호 추출
> 2. **가치**: 정확한 데이터 샘플링
> 3. **융합:** PLL, CDR, Manchester

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Clock Recovery는 수신 신호에서 클럭을 추출합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Clock Recovery                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              클럭 복구 메커니즘                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  문제:                                                 │   │ │
│   │   │  송신측 클럭과 수신측 클럭이 다름                        │   │ │
│   │   │  → 데이터 샘플링 오류                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  해결:                                                 │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  수신 데이터                                     ││   │ │
│   │   │  │      │                                          ││   │ │
│   │   │  │      ↓                                          ││   │ │
│   │   │  │  ┌───────────┐                                  ││   │ │
│   │   │  │  │   CDR     │  ← Clock Data Recovery           ││   │ │
│   │   │  │  │  (PLL)    │                                  ││   │ │
│   │   │  │  └───────────┘                                  ││   │ │
│   │   │  │      │                                          ││   │ │
│   │   │  │      ↓                                          ││   │ │
│   │   │  │  복구된 클럭 → 데이터 샘플링                       ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  방법:                                                │   │ │
│   │   │  - PLL (Phase Locked Loop)                           │   │ │
│   │   │  - Manchester 코딩                                   │   │ │
│   │   │  - 8b/10b 코딩                                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 클럭 복구는 고속 직렬 통신의 핵심입니다.

**📢 섹션 요약 비유:** Clock Recovery = 박자 맞추기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **PLL** | 위상 고정 루프 |
| **VCO** | 전압 제어 발진기 |
| **Phase Detector** | 위상 검출기 |
| **CDR** | 클럭 데이터 복구 |
| **Jitter** | 지터 |

**핵심 알고리즸:** 데이터 → 전이 검출 → 클럭 생성 → 동기

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**클럭 복구 방식**

| 방식 | 장점 | 단점 |
|:---|:---|:---|
| **PLL** | 정확 | 복잡 |
| **Manchester** | 단순 | 비효율 |
| **8b/10b** | 균형 | 오버헤드 |
| **Scrambling** | 효율 | 복잡 |
