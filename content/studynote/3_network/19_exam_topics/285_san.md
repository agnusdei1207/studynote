# SAN/NAS - 스토리지 네트워크

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 스토리지
> 2. **가치**: 중앙화, 공유
> 3. **융합**: 가상화, DR과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SAN(Storage Area Network)과 NAS(Network Attached Storage)는 네트워크 기반 스토리지입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SAN/NAS                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SAN (Storage Area Network)                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Server]      [Server]      [Server]               │   │ │
│   │   │      │             │             │                   │   │ │
│   │   │      │  FC / iSCSI │             │                   │   │ │
│   │   │      │      │      │             │                   │   │ │

│   │   │             │                                        │   │ │
│   │   │      ┌──────▼──────┐                                 │   │ │
│   │   │      │   FC Switch │                                 │   │ │

│   │   │             │                                        │   │ │

│   │   │      │  SAN Storage│                                 │   │ │
│   │   │      │  (LUN, Block)│                                │   │ │
│   │   │      └─────────────┘                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │

│   ┌──────────────────────────────────────────────────────────────┐ │
```

> **해설**: SAN은 블록 레벨, NAS는 파일 레벨 접근을 제공합니다.

**💡 비유**: SAN은 전용 도로, NAS는 공유 도로입니다.

**📢 섹션 요약 비유:** SAN = 블록, NAS = 파일!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **LUN** | 논리 유닛 |
| **FC** | 파이버 채널 |
| **iSCSI** | IP 기반 SCSI |
| **NFS/CIFS** | 파일 프로토콜 |

**비교표: SAN vs NAS**

```
+------------+----------+----------+
| 항목       │ SAN      │ NAS      │
+------------+----------+----------+
| 접근       │ 블록     │ 파일     │
| 프로토콜   │ FC,iSCSI │ NFS,CIFS │
| 성능       │ 높음     │ 중간     │
| 용도       │ DB, VM   │ 파일 공유│
+------------+----------+----------+
```

**핵심 알고리즸:** LUN 매핑, 파일 시스템

**📢 섹션 요약 비유:** FC = 고속 전용!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: SAN 프로토콜**

| 프로토콜 | 속도 | 특징 |
|:---|:---|:---|
| **FC** | 8-32Gbps | 전용 |
| **iSCSI** | 1-100Gbps | IP 기반 |
| **FCoE** | 10-100Gbps | 이더넷 |

**📢 섹션 요약 비유:** iSCSI = 저렴한 SAN!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: DB** → SAN (FC)
**시나리오 2: VM 스토리지** → SAN (iSCSI)
**시나리오 3: 파일 공유** → NAS

**도입 체크리스트:** 성능, 비용, 관리

**📢 섹션 요약 비유:** DB = SAN 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 중앙화, 공유

**미래 전망:** NVMe-oF 확대

**참고 표준:** SNIA, INCITS

**📢 섹션 요약 비유:** SAN/NAS는 엔터프라이즈 필수!
