# 토큰 버킷 - Token Bucket

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트래픽 계량 알고리즘
> 2. **가치**: 속도 제어, 버스트 허용
> 3. **융합**: 폴리싱, 쉐이핑, 스케줄링

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

토큰 버킷은 토큰으로 전송 속도를 제어합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    토큰 버킷 알고리즘                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              토큰 버킷 동작                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │    토큰 생성                                         │   │ │
│   │   │       │                                             │   │ │
│   │   │       ▼                                             │   │ │
│   │   │   ┌─────────┐      패킷 도착                        │   │ │
│   │   │   │ Bucket │◄────────────────                      │   │ │
│   │   │   │ (토큰) │                                       │   │ │
│   │   │   └────┬────┘                                       │   │ │
│   │   │        │                                            │   │ │
│   │   │        ▼                                            │   │ │
│   │   │   토큰 ≥ 패킷크기?                                  │   │ │
│   │   │     Yes → 전송, 토큰 차감                           │   │ │
│   │   │     No → 대기 or 폐기                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 토큰 버킷은 속도 제어와 버스트 허용을 동시에 제공합니다.

**📢 섹션 요약 비유:** Token Bucket = 티켓 시스템!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Rate** | 토큰 생성 속도 |
| **Bucket Size** | 버킷 용량 |
| **Tokens** | 사용 가능 토큰 |
| **Burst** | 최대 버스트 |

**핵심 알고리즸:** 토큰 생성, 소비

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Single vs Dual Token Bucket**

| 항목 | Single | Dual |
|:---|:---|:---|
| **버킷** | 1개 | 2개 |
| **색상** | 2색 | 3색 |
| **복잡성** | 낮음 | 높음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 폴리싱 → Single Bucket
**시나리오 2:** 쉐이핑 → Token Bucket
**시나리오 3:** 세분화 → Dual Bucket
