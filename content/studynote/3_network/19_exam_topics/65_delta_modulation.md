# 델타 변조 (Delta Modulation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 1비트만 전송하는 가장 단순한 DPCM
> 2. **가치**: 구현 간단, 저비용
> 3. **융합**: ADM, 음성 통신, 센서와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

델타 변조(Delta Modulation, DM)는 현재 샘플이 이전 샘플보다 크면 1, 작으면 0을 전송하는 가장 단순한 형태의 DPCM입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    델타 변조 (Delta Modulation)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DM 원리                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  규칙:                                                   │   │ │
│   │   │  • x[n] > x̂[n-1] → 1 (스텝 Δ 증가)                     │   │ │
│   │   │  • x[n] < x̂[n-1] → 0 (스텝 Δ 감소)                     │   │ │
│   │   │                                                      │   │ │
│   │   │  파형:                                                   │   │ │
│   │   │  원본:     ╱╲    ╱╲                                     │   │ │
│   │   │            ╱  ╲  ╱  ╲                                    │   │ │
│   │   │  계단:    ┌───┐┌───┐┌───                                │   │ │
│   │   │          ─┘   └┘   └┘                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  비트열: 1 1 1 0 0 1 1 1 0 0                           │   │ │
│   │   │         ↑         ↑                                   │   │ │
│   │   │        증가       감소                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                 │   │ │
│   │   │  • 1비트만 전송                                        │   │ │
│   │   │  • 매우 단순                                          │   │ │
│   │   │  • 과부하/과립ular 잡음 문제                           │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DM 문제점                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 경사 과부하 잡음 (Slope Overload Noise):           │   │ │
│   │   │     • 신호가 급격히 변할 때                             │   │ │
│   │   │     • 스텝이 너무 작아 따라가지 못함                     │   │ │
│   │   │                                                      │   │ │
│   │   │     원본: ╱╱╱ (급경사)                                 │   │ │
│   │   │     DM:  ┌┌┌ (따라가기 실패)                           │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 과립ular 잡음 (Granular Noise):                    │   │ │
│   │   │     • 신호가 천천히 변할 때                             │   │ │
│   │   │     • 스텝이 너무 커서 진동                             │   │ │
│   │   │                                                      │   │ │
│   │   │     원본: ───── (평탄)                                 │   │ │
│   │   │     DM:  ┌┐┌┐┌┐ (계단 진동)                           │   │ │
│   │   │                                                      │   │ │
│   │   │  해결: ADM (적응형 델타 변조)                           │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: DM은 단순하지만 스텝 크기 선택이 어렵습니다.

**💡 비유**: DM은 "올라" 또는 "내려" 두 단어만으로 위치를 설명하는 것입니다.

**📢 섹션 요약 비유**: 1비트 = 최단순!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Δ** | 스텝 크기 | 고정/가변 | 계단 |
| **비교기** | 크기 비교 | > or < | 판정 |
| **누적기** | 값 복원 | ±Δ | 합산 |
| **1비트** | 출력 | 0 or 1 | 신호 |

**비교표**

```
+------------+----------+----------+
| 항목       | DM       | ADM      |
+------------+----------+----------+
| 스텝       | 고정     | 가변     |
| 과부하     | 문제     | 완화     |
| 과립러     | 문제     | 완화     |
| 복잡도     | 낮음     | 중간     |
+------------+----------+----------+
```

**핵심 알고리즘**: 비교 → 1비트 출력 → ±Δ 누적 → 복원

**📢 섹션 요약 비유**: ADM이 실용적!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 비트율**

| 방식 | 비트/샘플 | 비트율 |
|:---|:---:|:---|
| **PCM** | 8 | 64kbps |
| **DPCM** | 4 | 32kbps |
| **DM** | 1 | 8-32kbps |

**📢 섹션 요약 비유**: DM은 오버샘플링 필요!
