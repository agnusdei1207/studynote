# Serverless - 서버리스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서버 관리 없는 실행
> 2. **가치**: 사용량 기반 과금, 자동 확장
> 3. **융합:** FaaS, BaaS, Event-Driven

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Serverless는 서버 관리 없이 코드를 실행합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Serverless                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              서버리스 아키텍처                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Triggers:                                            │   │ │
│   │   │  ┌─────────┐ ┌─────────┐ ┌─────────┐               │   │ │
│   │   │  │  HTTP   │ │  Queue  │ │ Schedule│               │   │ │
│   │   │  └────┬────┘ └────┬────┘ └────┬────┘               │   │ │
│   │   │       │           │           │                     │   │ │
│   │   │       └───────────┼───────────┘                     │   │ │
│   │   │                   ↓                                 │   │ │
│   │   │            ┌─────────────┐                          │   │ │
│   │   │            │  Function   │                          │   │ │
│   │   │            │ (FaaS)      │                          │   │ │
│   │   │            │ ─────────────                          │   │ │
│   │   │            │ - 코드만 작성                            │   │ │
│   │   │            │ - 자동 스케일링                          │   │ │
│   │   │            │ - 사용량 기반 과금                       │   │ │
│   │   │            └─────────────┘                          │   │ │
│   │   │                   │                                 │   │ │
│   │   │       ┌───────────┼───────────┐                     │   │ │
│   │   │       ↓           ↓           ↓                     │   │ │
│   │   │  ┌─────────┐ ┌─────────┐ ┌─────────┐               │   │ │
│   │   │  │Database │ │ Storage │ │  API    │               │   │ │
│   │   │  │ (BaaS)  │ │ (BaaS)  │ │ Gateway │               │   │ │
│   │   │  └─────────┘ └─────────┘ └─────────┘               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 서버리스는 인프라 관리를 클라우드 제공자에게 위임합니다.

**📢 섹션 요약 비유:** Serverless = 택시!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **FaaS** | 함수형 서비스 |
| **BaaS** | 백엔드형 서비스 |
| **Trigger** | 트리거 |
| **Cold Start** | 콜드 스타트 |
| **Execution** | 실행 환경 |

**핵심 알고리즸:** 이벤트 → 함수 실행 → 결과 → 종료
