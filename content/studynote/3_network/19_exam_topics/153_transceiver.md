# 트랜시버 (Transceiver) / MAU

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 송신기+수신기 통합 장치
> 2. **가치**: 매체 독립적 연결
> 3. **융합**: 이더넷, 광섬유, 무선과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

트랜시버(Transceiver)는 송신기(Transmitter)와 수신기(Receiver)를 결합한 장치로, MAU는 이더넷의 매체 연결 장치입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    트랜시버 / MAU                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              트랜시버 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [네트워크 장비]                                     │   │ │
│   │   │        │                                             │   │ │
│   │   │   ┌────┴────┐                                        │   │ │
│   │   │   │트랜시버 │                                        │   │ │
│   │   │   │ ┌───┬───┐│                                       │   │ │
│   │   │   │ │TX │RX ││  ← 송신기 + 수신기                    │   │ │
│   │   │   │ └───┴───┘│                                       │   │ │
│   │   │   └────┬────┘                                        │   │ │
│   │   │        │                                             │   │ │
│   │   │   [전송 매체]                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  역할:                                                   │   │ │
│   │   │  • 디지털 신호 ↔ 물리 신호 변환                          │   │ │
│   │   │  • 송수신 동시 지원 (전이중)                             │   │ │
│   │   │  • 매체 독립적 인터페이스                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MAU (Medium Attachment Unit)                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  정의: 이더넷에서 MAC 계층과 물리 매체를 연결           │   │ │
│   │   │                                                      │   │ │
│   │   │  종류:                                                   │   │ │
│   │   │  • AUI (Attachment Unit Interface):                    │   │ │
│   │   │    - 10BASE5 외부 트랜시버 연결                         │   │ │
│   │   │    - 15핀 D-sub 커넥터                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  • MII (Media Independent Interface):                  │   │ │
│   │   │    - 100Mbps 이더넷                                    │   │ │
│   │   │    - 40핀 커넥터                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  • GMII/RGMII/SGMII:                                   │   │ │
│   │   │    - 기가비트 이더넷                                    │   │ │
│   │   │    - R=Reduced, S=Serial                              │   │ │
│   │   │                                                      │   │ │
│   │   │  • SFP/SFP+/QSFP:                                      │   │ │
│   │   │    - 광섬유/동축 트랜시버 모듈                           │   │ │
│   │   │    - 핫 스왑 가능                                       │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 트랜시버는 송수신 기능을 통합하고, MAU는 매체 독립적 연결을 제공합니다.

**💡 비유**: 트랜시버는 무전기입니다. 말하기와 듣기를 동시에 할 수 있습니다.

**📢 섹션 요약 비유:** SFP = 광트랜시버!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 속도 | 용도 |
|:---|:---|:---|:---|
| **AUI** | 외부 MAU | 10M | 10BASE5 |
| **MII** | 매체 독립 | 100M | 100M |
| **GMII** | 기가비트 | 1G | 1G |
| **SFP** | 소형 | 1G | 광/동축 |
| **SFP+** | 고속 | 10G | 10G 광 |
| **QSFP** | 쿼드 | 40G+ | 40/100G |

**비교표**

```
+------------+----------+----------+
| 인터페이스 │ 속도     │ 핀 수    |
+------------+----------+----------+
| AUI        │ 10M      │ 15       |
| MII        │ 100M     │ 40       |
| RGMII      │ 1G       │ 24       |
| SGMII      │ 1G       │ 시리얼   |
+------------+----------+----------+
```

**핵심 알고리즸:** 직렬화/병렬화, 신호 변환

**📢 섹션 요약 비유:** SFP = 핫 스왑 트랜시버!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 광트랜시버**

| 모듈 | 속도 | 거리 |
|:---|:---:|:---:|
| **SFP** | 1G | 550m/10km |
| **SFP+** | 10G | 400m/10km |
| **QSFP28** | 100G | 100m/10km |

**📢 섹션 요약 비유:** QSFP28 = 100G!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 1G 광섬유** → SFP
**시나리오 2: 10G 광섬유** → SFP+
**시나리오 3: 100G** → QSFP28

**도입 체크리스트:** 속도, 거리, 매체 타입

**📢 섹션 요약 비유:** 트랜시버는 매체의 다리!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 매체 독립적 연결

**미래 전망:** QSFP-DD, OSFP로 진화

**참고 표준:** IEEE 802.3, SFF-8472

**📢 섹션 요약 비유:** 트랜시버는 인터페이스의 핵심!
