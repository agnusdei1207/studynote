# Scrambling - 스크램블링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 무작위화
> 2. **가치**: DC 밸런스 및 동기화
> 3. **융합:** LFSR, XOR, Self-Sync

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Scrambling은 데이터를 의사 난수로 변환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Scrambling                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스크램블링 메커니즘                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  목적:                                                 │   │ │
│   │   │  - 연속 0/1 방지                                       │   │ │
│   │   │  - DC 밸런스 유지                                      │   │ │
│   │   │  - 클럭 복구 용이                                       │   │ │
│   │   │  - EMI 감소                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  송신측:                                               │   │ │
│   │   │  ┌───────────┐    XOR    ┌───────────┐              │   │ │
│   │   │  │  데이터   │ ────────→ │스크램블링 │              │   │ │
│   │   │  └───────────┘           └───────────┘              │   │ │
│   │   │        +                       +                     │   │ │
│   │   │  ┌───────────┐           ┌───────────┐              │   │ │
│   │   │  │  LFSR     │           │  PRBS     │              │   │ │
│   │   │  └───────────┘           └───────────┘              │   │ │
│   │   │                                                      │   │ │
│   │   │  수신측:                                               │   │ │
│   │   │  동일한 LFSR로 역변환                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  유형:                                                 │   │ │
│   │   │  - Frame Synchronous: 프레임 단위                     │   │ │
│   │   │  - Self Synchronous: 자기 동기                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스크램블링은 고속 직렬 전송의 필수 요소입니다.

**📢 섹션 요약 비유:** Scrambling = 카드 섞기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **LFSR** | 선형 피드백 시프트 레지스터 |
| **PRBS** | 의사 난수 비트 열 |
| **Polynomial** | 다항식 |
| **Seed** | 시드 값 |
| **Descrambler** | 디스크램블러 |

**핵심 알고리즸:** 데이터 XOR PRBS → 스크램블드 데이터

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**스크램블링 유형 비교**

| 유형 | 동기화 | 오류 전파 |
|:---|:---|:---|
| **Frame Sync** | 프레임 | 없음 |
| **Self Sync** | 자동 | 있음 |
| **Additive** | 필요 | 없음 |
| **Multiplicative** | 자동 | 있음 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** SONET/SDH → 프레임 동기
**시나리오 2:** 이더넷 → self-sync
**시나리오 3:** SATA → LFSR
