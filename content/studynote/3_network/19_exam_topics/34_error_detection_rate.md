# 에러 검출율 (Error Detection Rate)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오류 검출 기법이 실제 오류를 얼마나 정확히 찾아내는지
> 2. **가치**: 통신 신뢰성 평가, 기법 선택 기준
> 3. **융합**: 패리티, CRC, 체크섬과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

에러 검출율(Error Detection Rate)은 오류 검출 기법이 실제 발생한 오류 중 얼마나 많은 비율을 검출하는지를 나타내는 지표입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    에러 검출율 (Error Detection Rate)               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              검출율 정의                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  에러 검출율 = 검출된 에러 / 총 발생 에러 × 100%       │   │ │
│   │   │                                                      │   │ │
│   │   │  관련 지표:                                           │   │ │
│   │   │  • 미검출율: 검출하지 못한 에러 비율                    │   │ │
│   │   │  • 오탐율: 정상을 에러로 잘못 판단                      │   │ │
│   │   │  • 검출 확률: 기법별 이론적 성능                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              기법별 검출 능력                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 패리티 비트 (Parity Bit):                          │   │ │
│   │   │     • 홀수/짝수 오류 검출                              │   │ │
│   │   │     • 검출율: ~50% (1비트 오류만)                      │   │ │
│   │   │     • 짝수 개 오류는 검출 불가                          │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 체크섬 (Checksum):                                 │   │ │
│   │   │     • 합계 기반                                        │   │ │
│   │   │     • 검출율: ~95%                                    │   │ │
│   │   │     • 순서 변경에 약함                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  3. CRC (Cyclic Redundancy Check):                     │   │ │
│   │   │     • 다항식 기반                                      │   │ │
│   │   │     • 검출율: 99.99%+                                 │   │ │
│   │   │     • 버스트 에러에 강함                               │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 해밍 코드 (Hamming Code):                          │   │ │
│   │   │     • 검출 + 정정                                      │   │ │
│   │   │     • 1비트 정정, 2비트 검출                           │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 검출율이 높을수록 좋지만, 오버헤드도 증가합니다.

**💡 비유**: 에러 검출율은 보안 검색대가 위험 물품을 얼마나 잘 찾아내는지와 같습니다.

**📢 섹션 요약 비유**: CRC가 검출율 최고!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **검출율** | 성능 | 백분율 | 정확도 |
| **오버헤드** | 비용 | 추가 비트 | 비용 |
| **버스트** | 패턴 | 연속 에러 | 뭉치 |
| **다항식** | CRC | 생성 다항식 | 수식 |

**CRC 검출 확률**

```
CRC-32 검출 능력:
• 모든 1비트 오류: 100%
• 모든 2비트 오류: 100%
• 홀수 개 오류: 100%
• 32비트 이하 버스트: 100%
• 33비트 버스트: 99.99999998%
• 긴 버스트: 99.9984%
```

**핵심 알고리즘**: 데이터 → 검출 코드 → 비교 → 판정

**📢 섹션 요약 비유**: CRC-32가 이더넷 표준!
