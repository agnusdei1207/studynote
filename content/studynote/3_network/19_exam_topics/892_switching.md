# Switching - 스위칭

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프레임 전달
> 2. **가치**: 효율적 로컬 통신
> 3. **융합:** MAC Table, VLAN, STP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Switching은 MAC 주소 기반 프레임 전달입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Switching                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스위치 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  MAC Address Table:                                     │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ MAC Address        Port    VLAN   Age           ││   │ │
│   │   │  │ aa:bb:cc:00:00:01  1       10     300s          ││   │ │
│   │   │  │ aa:bb:cc:00:00:02  2       10     300s          ││   │ │
│   │   │  │ aa:bb:cc:00:00:03  3       20     300s          ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  프레임 처리:                                            │   │ │
│   │   │  1. 프레임 수신                                          │   │ │
│   │   │  2. Source MAC 학습                                      │   │ │
│   │   │  3. Destination MAC 조회                                 │   │ │
│   │   │  4. Unicast: 해당 포트로 전송                            │   │ │
│   │   │  5. Unknown/Broadcast: 플러딩                            │   │ │
│   │   │                                                      │   │ │
│   │   │  스위치 유형:                                              │   │ │
│   │   │  - L2 Switch: MAC 기반                                   │   │ │
│   │   │  - L3 Switch: IP 라우팅 가능                              │   │ │
│   │   │  - L4-L7 Switch: 애플리케이션 인식                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스위칭은 LAN 통신의 핵심입니다.

**📢 섹션 요약 비유:** Switching = 스마트 우편 분류!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **MAC Table** | MAC 주소 테이블 |
| **Port** | 포트 |
| **VLAN** | 가상 LAN |
| **Flooding** | 플러딩 |
| **Learning** | 학습 |

**핵심 알고리즸:** 수신 → 학습 → 조회 → 전송

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Hub vs Switch**

| 항목 | Hub | Switch |
|:---|:---|:---|
| **동작** | 브로드캐스트 | 유니캐스트 |
| **속도** | 반복문 | 전이중 |
| **충돌** | 있음 | 없음 |
| **지능** | 없음 | MAC 학습 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** LAN → L2 Switch
**시나리오 2:** VLAN 간 → L3 Switch
**시나리오 3:** 데이터센터 → L4-L7 Switch
