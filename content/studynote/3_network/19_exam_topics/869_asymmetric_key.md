# Asymmetric Key - 비대칭키

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공개키/개인키 쌍
> 2. **가치**: 안전한 키 교환
> 3. **융합:** RSA, ECC, Digital Signature

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Asymmetric Key는 공개키와 개인키 쌍을 사용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Asymmetric Key                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              비대칭키 암호화                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  암호화 (기밀성):                                       │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Sender                    Receiver              ││   │ │
│   │   │  │  ┌─────┐                  ┌─────┐               ││   │ │
│   │   │  │  │Plain│ ── Public Key ─→ │Ciph │               ││   │ │
│   │   │  │  │     │    Encrypt       │     │               ││   │ │
│   │   │  │  └─────┘                  └──┬──┘               ││   │ │
│   │   │  │                              │                  ││   │ │
│   │   │  │                              │ Private Key      ││   │ │
│   │   │  │                              │ Decrypt          ││   │ │
│   │   │  │                              ↓                  ││   │ │
│   │   │  │                           ┌─────┐               ││   │ │
│   │   │  │                           │Plain│               ││   │ │
│   │   │  │                           └─────┘               ││   │ │

│   │   │                                                      │   │ │
│   │   │  서명 (인증):                                           │   │ │
│   │   │  Private Key로 서명 → Public Key로 검증                 │   │ │

│   │   │  알고리즘:                                              │   │ │

│   │   │  - ECC: 타원곡선, 더 짧은 키                             │   │ │
```

> **해설**: 비대칭키는 키 분배 문제를 해결합니다.

**📢 섹션 요약 비유:** Asymmetric Key = 우편함과 열쇠!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Public Key** | 공개키 |
| **Private Key** | 개인키 |
| **Key Pair** | 키 쌍 |
| **Certificate** | 인증서 |
| **PKI** | 공개키 기반 구조 |

**핵심 알고리즸:** 공개키로 암호화 → 개인키로 복호화

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**대칭키 vs 비대칭키**

| 항목 | 대칭키 | 비대칭키 |
|:---|:---|:---|
| **키** | 1개 | 2개 |
| **속도** | 빠름 | 느림 |
| **키 분배** | 어려움 | 쉬움 |
| **용도** | 데이터 | 키 교환, 서명 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 키 교환 → RSA/ECDH
**시나리오 2:** 서명 → RSA/ECDSA
**시나리오 3:** 하이브리드 → RSA로 AES 키 전송

---
