# Window - 윈도우

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 가능 데이터 양
> 2. **가치**: 흐름 제어
> 3. **융합:** rwnd, cwnd, Sliding Window

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Window는 ACK 없이 전송 가능한 데이터 양입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Window                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Sliding Window                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  윈도우 크기 = 4                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  초기 상태:                                           │   │ │
│   │   │  ┌───┬───┬───┬───┬───┬───┬───┬───┐                  │   │ │
│   │   │  │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │                  │   │ │
│   │   │  └───┴───┴───┴───┴───┴───┴───┴───┘                  │   │ │
│   │   │    ↑   ↑   ↑   ↑   ↑   ↑                            │   │ │
│   │   │   ────────┴───┴───┴───                              │   │ │
│   │   │   Window   전송 대기                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  ACK 1, 2 수신 후:                                    │   │ │
│   │   │  ┌───┬───┬───┬───┬───┬───┬───┬───┐                  │   │ │
│   │   │  │ ✓ │ ✓ │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │                  │   │ │
│   │   │  └───┴───┴───┴───┴───┴───┴───┴───┘                  │   │ │
│   │   │            ↑   ↑   ↑   ↑                            │   │ │
│   │   │           ────────윈도우 이동──                      │   │ │
│   │   │                                                      │   │ │
│   │   │  TCP 윈도우:                                          │   │ │
│   │   │  - rwnd: 수신자 버퍼 여유 (Flow Control)               │   │ │
│   │   │  - cwnd: 혼잡 윈도우 (Congestion Control)              │   │ │
│   │   │  - 전송 가능 = min(rwnd, cwnd)                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 윈도우는 네트워크 효율과 혼잡 제어의 핵심입니다.

**📢 섹션 요약 비유:** Window = 전송 허용증!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **rwnd** | 수신 윈도우 |
| **cwnd** | 혼잡 윈도우 |
| **ssthresh** | 임계값 |
| **Sliding** | 슬라이딩 |
| **Scaling** | 스케일링 |

**핵심 알고리즸:** 전송 → ACK → 윈도우 슬라이드

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**윈도우 유형 비교**

| 유형 | 목적 | 기준 |
|:---|:---|:---|
| **rwnd** | 수신자 보호 | 버퍼 |
| **cwnd** | 네트워크 보호 | 혼잡 |
| **Send** | 전송 제한 | min(rwnd,cwnd) |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 높은 대역폭 → Window Scaling
**시나리오 2:** 혼잡 → cwnd 감소
**시나리오 3:** 버퍼 부족 → rwnd=0
