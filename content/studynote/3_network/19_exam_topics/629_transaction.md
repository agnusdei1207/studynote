# Transaction - 트랜잭션

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 논리적 작업 단위
> 2. **가치**: 데이터 무결성, 일관성
> 3. **융합:** ACID, 2PC, Isolation Level

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Transaction은 하나의 논리적 작업 단위입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Transaction                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              트랜잭션 생명주기                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────┐                                        │   │ │
│   │   │  │  Begin  │                                        │   │ │
│   │   │  └────┬────┘                                        │   │ │
│   │   │       │                                             │   │ │
│   │   │       ↓                                             │   │ │
│   │   │  ┌─────────┐    ┌─────────┐                        │   │ │
│   │   │  │  Work   │───→│  Work   │                        │   │ │
│   │   │  │   #1    │    │   #2    │                        │   │ │
│   │   │  └─────────┘    └─────────┘                        │   │ │
│   │   │       │                                             │   │ │
│   │   │       ↓                                             │   │ │
│   │   │  ┌─────────┐    ┌─────────┐                        │   │ │
│   │   │  │ Commit  │ OR │ Rollback│                        │   │ │
│   │   │  │  (성공) │    │ (실패)  │                        │   │ │
│   │   │  └─────────┘    └─────────┘                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 트랜잭션은 전부 실행되거나 전부 실행되지 않습니다.

**📢 섹션 요약 비유:** Transaction = 은행 이체!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Begin** | 트랜잭션 시작 |
| **Commit** | 트랜잭션 확정 |
| **Rollback** | 트랜잭션 취소 |
| **Savepoint** | 저장점 |
| **Isolation** | 격리 수준 |

**핵심 알고리즸:** Begin → Operations → (Commit | Rollback)

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**격리 수준 (Isolation Level)**

| 수준 | Dirty Read | Non-Repeatable | Phantom |
|:---|:---|:---|:---|
| **Read Uncommitted** | O | O | O |
| **Read Committed** | X | O | O |
| **Repeatable Read** | X | X | O |
| **Serializable** | X | X | X |
