# 네트워크 프로그래밍 - Network Programming

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 코드로 네트워크 제어
> 2. **가치**: 자동화, 일관성, 버전 관리
> 3. **융합**: API, Netconf, RESTCONF

### Ⅰ. 개요 (Context & Background)

**개념 정의**

**📢 섹션 요약 비유:** IaC = 코드로 관리!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **NETCONF** | XML 기반 설정 프로토콜 |
| **RESTCONF** | RESTful API |
| **YANG** | 데이터 모델링 언어 |
| **gNMI** | gRPC 기반 관리 |

**비교표: 자동화 도구**

```
+------------+----------+----------+----------+
| 도구       │ 언어     │ 특징     │ 용도     │
+------------+----------+----------+----------+
| Ansible    │ YAML     │ 에이전트X│ 설정     │
| Terraform  │ HCL      │ 상태     │ 인프라   │
| NAPALM     │ Python   │ 추상화   │ 멀티벤더 │
| PyEZ       │ Python   │ 주니퍼   │ 전용     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 멱등성, 상태 관리

**📢 섹션 요약 비유:** Ansible = 에이전트 없는 자동화!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 데이터 형식**

| 형식 | 설명 | 용도 |
|:---|:---|:---|
| **XML** | NETCONF | 표준 |
| **JSON** | RESTCONF | 웹 |
| **YAML** | Ansible | 가독성 |

**📢 섹션 요약 비유:** YANG = 데이터 모델!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 설정 자동화 → Ansible
**시나리오 2:** 인프라 관리 → Terraform
**시나리오 3:** 멀티벤더 → NAPALM

**도입 체크리스트:** API 지원, 모델, 테스트

**📢 섹션 요약 비유:** 버전 관리 = 변경 추적!

### Ⅴ. 기대효과 및 결론 (Future & Standard)
