# RDBMS (Relational Database Management System) - 관계형 데이터베이스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 테이블 기반 데이터베이스
> 2. **가치**: ACID, SQL, 정형 데이터
> 3. **융합:** SQL, Normalization, Transaction

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RDBMS는 테이블로 데이터를 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RDBMS                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              관계형 모델                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Users Table              Orders Table               │   │ │
│   │   │  ┌─────────────────┐      ┌─────────────────────┐    │   │ │
│   │   │  │ ID │ Name │Email│      │ ID │ UserID│ Product│    │   │ │
│   │   │  ├────┼──────┼─────┤      ├────┼───────┼────────┤    │   │ │
│   │   │  │  1 │ John │j@x │      │  1 │   1   │ Laptop │    │   │ │
│   │   │  │  2 │ Jane │j@y │      │  2 │   1   │ Mouse  │    │   │ │
│   │   │  └─────────────────┘      │  3 │   2   │ Phone  │    │   │ │
│   │   │         ↑                  └─────────────────────┘    │   │ │
│   │   │         │ Foreign Key (ID = UserID)                   │   │ │
│   │   │         └─────────────────────────────────────────────┘   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  - 테이블 간 관계 (Relationship)                       │   │ │
│   │   │  - SQL로 질의                                         │   │ │
│   │   │  - ACID 트랜잭션                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RDBMS는 테이블 간 관계로 데이터를 구조화합니다.

**📢 섹션 요약 비유:** RDBMS = 스프레드시트!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Table** | 테이블 |
| **Row** | 행 (레코드) |
| **Column** | 열 (속성) |
| **Primary Key** | 기본 키 |
| **Foreign Key** | 외래 키 |

**핵심 알고리즸:** SQL → 파싱 → 실행 → 결과
