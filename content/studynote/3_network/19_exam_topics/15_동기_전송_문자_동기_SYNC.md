# 동기 전송 (문자 동기 SYNC)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SYN 문자로 문자 단위 동기화 달성
> 2. **가치**: 비동기보다 효율, 블록 전송 가능
> 3. **융합**: BISYNC, 클럭 복구, HDLC와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

동기 전송(Synchronous Transmission)은 SYN 문자(0x16)를 사용하여 송수신 간 동기를 맞추고 연속적인 문자열을 전송하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    동기 전송 (문자 동기 SYNC)                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              문자 동기 방식 (BISYNC)                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  SYN │ SYN │ STX │ Data │ ETX │ BCC │ SYN │ SYN │   │   │ │
│   │   │ 0x16 │0x16 │0x02 │      │0x03 │     │     │     │   │   │ │
│   │   │                                                      │   │ │
│   │   │  SYN (Synchronous Idle): 동기 문자                    │   │ │
│   │   │  STX (Start of Text): 본문 시작                       │   │ │
│   │   │  ETX (End of Text): 본문 종료                         │   │ │
│   │   │  BCC (Block Check Character): 블록 검사               │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  • 최소 2개 SYN로 동기 확보                           │   │ │
│   │   │  • 제어 문자로 프레임 구분                             │   │ │
│   │   │  • 오버헤드 낮음                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              클럭 복구                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  송신측                    수신측                     │   │ │
│   │   │  ┌─────────┐              ┌─────────┐                │   │ │
│   │   │  │Data+Clock│────────────→│Clock    │                │   │ │
│   │   │  │         │              │Recovery │                │   │ │
│   │   │  └─────────┘              └────┬────┘                │   │ │
│   │   │                                │                      │   │ │
│   │   │  방식:                          ▼                      │   │ │
│   │   │  • 별도 클럭 선                                        │   │ │
│   │   │  • 클럭 복구 (Manchester, NRZI)                        │   │ │
│   │   │  • PLL (Phase Locked Loop)                             │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 동기 전송은 SYN 문자로 동기를 맞춘 후 연속 전송합니다.

**💡 비유**: 동기 전송은 오케스트라 지휘자에 맞춰 연주하는 것과 같습니다.

**📢 섹션 요약 비유**: SYN SYN = 동기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **SYN** | 동기 | 0x16 | 튜닝 |
| **STX** | 시작 | 0x02 | 시작 |
| **ETX** | 종료 | 0x03 | 끝 |
| **BCC** | 검사 | LRC/CRC | 검수 |

**제어 문자표**

| 문자 | 코드 | 기능 |
|:---|:---:|:---|
| **SYN** | 0x16 | 동기 |
| **SOH** | 0x01 | 헤더 시작 |
| **STX** | 0x02 | 본문 시작 |
| **ETX** | 0x03 | 본문 종료 |
| **EOT** | 0x04 | 전송 종료 |

**핵심 알고리즘**: SYN 탐지 → 문자 경계 설정 → STX→ETX 수신 → BCC 검사

**📢 섹션 요약 비유**: 2개 SYN 권장!
