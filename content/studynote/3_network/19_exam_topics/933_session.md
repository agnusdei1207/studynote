# Session - 세션

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 통신 상태 유지
> 2. **가치**: 상태 기반 서비스
> 3. **융합:** Session ID, Cookie, Stateful

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Session은 클라이언트와 서버 간 상태를 유지합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Session                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              세션 관리                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  세션 수립 과정:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Client                Server                   ││   │ │
│   │   │  │    │                     │                      ││   │ │
│   │   │  │    │  1. Request          │                      ││   │ │
│   │   │  │    │ ─────────────────→  │                      ││   │ │
│   │   │  │    │                     │                      ││   │ │
│   │   │  │    │  2. Response + Session ID                  ││   │ │
│   │   │  │    │ ←─────────────────  │                      ││   │ │
│   │   │  │    │                     │                      ││   │ │
│   │   │  │    │  3. Request + Session ID                   ││   │ │
│   │   │  │    │ ─────────────────→  │                      ││   │ │
│   │   │  │    │                     │                      ││   │ │
│   │   │  │    │  4. Response (상태 유지)                     ││   │ │
│   │   │  │    │ ←─────────────────  │                      ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  세션 저장:                                              │   │ │
│   │   │  - In-Memory: 메모리 저장                               │   │ │
│   │   │  - Database: DB 저장                                    │   │ │
│   │   │  - Redis: 캐시 저장                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 세션은 Stateless HTTP를 Stateful하게 만듭니다.

**📢 섹션 요약 비유:** Session = 입장권!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Session ID** | 세션 식별자 |
| **Cookie** | 쿠키 저장 |
| **Timeout** | 만료 시간 |
| **Store** | 저장소 |
| **State** | 상태 데이터 |

**핵심 알고리듬:** ID 생성 → 상태 저장 → 요청 시 조회

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**세션 vs 쿠키**

| 항목 | Session | Cookie |
|:---|:---|:---|
| **저장** | 서버 | 클라이언트 |
| **보안** | 높음 | 낮음 |
| **용량** | 무제한 | 4KB |
| **속도** | 느림 | 빠름 |
