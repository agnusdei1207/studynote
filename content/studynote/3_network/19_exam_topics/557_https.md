# HTTPS (HTTP Secure) - 보안 HTTP

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: HTTP + TLS/SSL 암호화
> 2. **가치**: 기밀성, 무결성, 인증
> 3. **융합:** TLS, 인증서, 공개키

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HTTPS는 TLS로 암호화된 HTTP입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    HTTPS                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HTTPS Handshake                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Client                              Server          │   │ │
│   │   │     │                                   │            │   │ │
│   │   │     │ ── ClientHello ────────────────→ │            │   │ │
│   │   │     │    (지원 암호, 난수)              │            │   │ │
│   │   │     │                                   │            │   │ │
│   │   │     │ ←── ServerHello + Certificate ── │            │   │ │
│   │   │     │     (선택 암호, 인증서)           │            │   │ │
│   │   │     │                                   │            │   │ │
│   │   │     │ ── Key Exchange ───────────────→ │            │   │ │
│   │   │     │                                   │            │   │ │
│   │   │     │ ←── Finished ─────────────────── │            │   │ │
│   │   │     │                                   │            │   │ │
│   │   │  → 암호화 통신 시작                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HTTPS는 TLS Handshake 후 암호화 통신을 합니다.

**📢 섹션 요약 비유:** HTTPS = 보안 택배!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **TLS 1.3** | 최신 버전 |
| **인증서** | X.509 |
| **CA** | 인증 기관 |
| **공개키** | 암호화 |
| **대칭키** | 세션 암호화 |
| **Port 443** | 기본 포트 |

**핵심 알고리즸:** Handshake → 키 교환 → 암호화 통신

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**HTTP vs HTTPS**

| 항목 | HTTP | HTTPS |
|:---|:---|:---|
| **암호화** | 없음 | TLS |
| **포트** | 80 | 443 |
| **인증** | 없음 | 인증서 |
| **SEO** | 낮음 | 높음 |
