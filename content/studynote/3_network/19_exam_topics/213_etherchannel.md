# 이더채널 (EtherChannel) - 링크 집합

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 물리 링크를 논리적 단일 링크로 묶음
> 2. **가치**: 대역폭 증가, 이중화
> 3. **융합**: LACP, PAgP, 수동 모드

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

이더채널(EtherChannel)은 여러 개의 물리적 이더넷 링크를 하나의 논리적 링크로 묶어 대역폭을 증가시키고 이중화를 제공하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    이더채널 (EtherChannel)                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              이더채널 개념                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [스위치 A]                    [스위치 B]             │   │ │
│   │   │                                                      │   │ │
│   │   │    ┌─────────────────────────────────┐              │   │ │
│   │   │    │     EtherChannel (Port-channel 1)│              │   │ │
│   │   │    │                                  │              │   │ │
│   │   │    │  Link 1 (1G) ────────────────    │              │   │ │
│   │   │    │  Link 2 (1G) ────────────────    │  = 4G       │   │ │
│   │   │    │  Link 3 (1G) ────────────────    │              │   │ │

│   │   │    │                                  │              │   │ │

│   │   │  • 대역폭 증가 (8개까지 = 80G)                        │   │ │
│   │   │  • 이중화 (1개 장애 시 계속 동작)                     │   │ │
│   │   │  • STP 1개 포트로 인식                               │   │ │
│   │   │  • 부하 분산                                         │   │ │

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. LACP (Link Aggregation Control Protocol):        │   │ │
│   │   │     • IEEE 802.3ad                                  │   │ │
│   │   │     • 표준                                          │   │ │
│   │   │     • Active/Passive 모드                           │   │ │
│   │   │                                                      │   │ │
│   │   │  2. PAgP (Port Aggregation Protocol):                 │   │ │
│   │   │     • Cisco 전용                                     │   │ │
│   │   │     • Desirable/Auto 모드                           │   │ │

│   │   │  3. Static (수동):                                    │   │ │
│   │   │     • 프로토콜 없음                                  │   │ │
│   │   │     • On 모드                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │

└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 이더채널은 최대 8개 링크를 묶어 대역폭을 증가시킵니다.

**💡 비유**: 이더채널은 차선입니다. 여러 차선이 하나의 도로를 구성합니다.

**📢 섹션 요약 비유:** LACP = 표준!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | LACP | PAgP |
|:---|:---|:---|
| **표준** | IEEE 802.3ad | Cisco |
| **모드** | Active/Passive | Desirable/Auto |
| **최대 링크** | 8 (Active) | 8 |
| **멀티벤더** | O | X |

**비교표: 부하 분산**

```
+------------+----------+----------+
| 방식       │ 기준     │ 설명     │
+------------+----------+----------+
| src-mac    │ 출발지   │ MAC      │
| dst-mac    │ 목적지   │ MAC      │
| src-dst-mac│ 양방향   │ MAC      │
| src-ip     │ 출발지   │ IP       │
| dst-ip     │ 목적지   │ IP       │
| src-dst-ip │ 양방향   │ IP       │
+------------+----------+----------+
```

**핵심 알고리즸:** 해시 기반 부하 분산

**📢 섹션 요약 비유:** src-dst-ip = 기본!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 모드 조합**

| LACP | LACP | 결과 |
|:---|:---|:---|
| Active | Active | O |
| Active | Passive | O |
| Passive | Passive | X |

**📢 섹션 요약 비유:** Active + Active = 최고!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 코어 스위치** → LACP Active
**시나리오 2: 서버** → LACP Active-Active
**시나리오 3: Cisco만** → PAgP 또는 LACP

**도입 체크리스트:** 링크 수, 속도 일치, VLAN 일치

**📢 섹션 요약 비유:** 속도/VLAN = 반드시 일치!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대역폭 증가, 이중화

**미래 전망:** 100G/400G 포트, MLAG

**참고 표준:** IEEE 802.3ad, IEEE 802.1AX

**📢 섹션 요약 비유:** 이더채널은 이중화의 핵심!
