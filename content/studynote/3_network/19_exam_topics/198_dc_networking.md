# 데이터센터 네트워킹 (DC Networking) - Spine-Leaf

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고속, 저지연 데이터센터 네트워크
> 2. **가치**: 확장성, 예측 가능한 지연
> 3. **융합**: VXLAN, EVPN과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

데이터센터 네트워킹은 서버, 스토리지, 네트워크를 고속으로 연결하는 인프라입니다.

> **해설**: Spine-Leaf는 데이터센터의 표준 아키텍처입니다.

**💡 비유**: Spine-Leaf는 고속도로입니다. 모든 교차로에서 같은 거리로 갈 수 있습니다.

**📢 섹션 요약 비유:** Spine-Leaf = 2홉!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소 | 역할 | 속도 |
|:---|:---|:---|
| **Spine** | 코어 | 100-400G |
| **Leaf** | 액세스 | 10-100G |
| **ToR** | Top of Rack | 10-100G |
| **EoR** | End of Row | 100-400G |

**비교표**

```
+------------+----------+----------+
| 항목       │ 3-Tier   │ Spine-Leaf│
+------------+----------+----------+
| 홉 수      │ 1-3      │ 2        │
│ 확장성     │ 제한     │ 선형     │
│ 지연       │ 가변     │ 일정     │
│ 오버서브스크립션│ 높음│ 낮음     │
+------------+----------+----------+
```

**핵심 알고리즸:** ECMP, VXLAN, EVPN

**📢 섹션 요약 비유:** EVPN = BGP + VXLAN!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 고속 인터페이스**

| 속도 | 표준 | 용도 |
|:---:|:---|:---|
| **10G** | 10GBASE-SR | 서버 |
| **25G** | 25GBASE-SR | 서버 |
| **40G** | 40GBASE-SR4 | 업링크 |
| **100G** | 100GBASE-SR4 | Spine |
| **400G** | 400GBASE-SR8 | Spine |

**📢 섹션 요약 비유:** 100/400G = DC 표준!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 소규모 DC** → 2-Spine, 4-Leaf
**시나리오 2: 대규모 DC** → 4-Spine, 수십 Leaf
**시나리오 3: 클라우드** → EVPN-VXLAN

**도입 체크리스트:** 서버 수, 대역폭, 지연 요구

**📢 섹션 요약 비유:** 서버 수에 비례하는 Leaf!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 확장성, 예측 가능한 성능

**미래 전망:** 800G, Co-Packaged Optics

**참고 표준:** IEEE 802.3, RFC 7432 (EVPN)

**📢 섹션 요약 비유:** Spine-Leaf는 DC의 표준!
