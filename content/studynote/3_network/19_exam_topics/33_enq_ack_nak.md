# 엔트/애크/나크/EOT (ENQ/ACK/NAK/EOT)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 통신의 기본 제어 문자
> 2. **가치**: 통신 세션 관리, 오류 제어, 흐름 제어
> 3. **융합**: BSC, HDLC, XMODEM과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

제어 문자(Control Characters)는 데이터 통신에서 장비 간 통신 세션을 관리하고 오류를 제어하는 ASCII 코드입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    제어 문자 (Control Characters)                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              주요 제어 문자                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  ENQ (Enquiry) - 0x05:                                │   │ │
│   │   │  • 의미: "통신 가능한가?"                              │   │ │
│   │   │  • 용도: 통신 세션 시작 요청                           │   │ │
│   │   │  • 응답: ACK 또는 NAK                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  ACK (Acknowledge) - 0x06:                             │   │ │
│   │   │  • 의미: "예, 가능합니다"                              │   │ │
│   │   │  • 용도: 긍정 응답, 정상 수신 확인                      │   │ │
│   │   │  • 다음: 데이터 전송 계속                              │   │ │
│   │   │                                                      │   │ │
│   │   │  NAK (Negative Acknowledge) - 0x15:                    │   │ │
│   │   │  • 의미: "아니오, 문제가 있습니다"                      │   │ │
│   │   │  • 용도: 부정 응답, 오류 통보                          │   │ │
│   │   │  • 다음: 재전송 요청                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  EOT (End of Transmission) - 0x04:                     │   │ │
│   │   │  • 의미: "통신 종료합니다"                              │   │ │
│   │   │  • 용도: 세션 종료, 회선 해제                          │   │ │
│   │   │  • 다음: 대기 상태 복귀                                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              추가 제어 문자                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  STX (Start of Text) - 0x02: 데이터 시작              │   │ │
│   │   │  ETX (End of Text) - 0x03: 데이터 종료                │   │ │
│   │   │  SYN (Synchronous Idle) - 0x16: 동기 유지             │   │ │
│   │   │  DLE (Data Link Escape) - 0x10: 제어 확장             │   │ │
│   │   │  SOH (Start of Header) - 0x01: 헤더 시작              │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 이 제어 문자들은 주로 BSC(Binary Synchronous Communication) 프로토콜에서 사용됩니다.

**💡 비유**: 제어 문자는 전화 통화의 "여보세요", "네", "다시 말씀해주세요"와 같습니다.

**📢 섹션 요약 비유**: ENQ=요청, ACK=수락, NAK=거부, EOT=종료!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 문자 | ASCII | 역할 | 비유 |
|:---|:---:|:---|:---|
| **ENQ** | 0x05 | 요청 | 노크 |
| **ACK** | 0x06 | 긍정 | 네 |
| **NAK** | 0x15 | 부정 | 아니오 |
| **EOT** | 0x04 | 종료 | 끝 |

**통신 시퀀스 예시**

```
파일 전송 시퀀스:

송신측                          수신측
  │                               │
  │── ENQ ("준비되었나요?") ─────→│
  │                               │
  │←─ ACK ("네, 준비되었습니다") ─│
  │                               │
  │── STX 데이터 ETX ────────────→│
  │                               │
  │←─ ACK ("잘 받았습니다") ──────│
  │                               │
  │── STX 데이터 ETX ────────────→│
  │                               │
  │←─ NAK ("오류 발생, 재전송") ──│
  │                               │
  │── STX 데이터 ETX ────────────→│ (재전송)
  │                               │
  │←─ ACK ("확인") ───────────────│
  │                               │
  │── EOT ("종료합니다") ─────────→│
  │                               │
```

**핵심 알고리즘**: ENQ → ACK/NAK → 데이터 → ACK/NAK → EOT

**📢 섹션 요약 비유**: NAK는 재전송 트리거!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 프로토콜별 제어 문자**

| 프로토콜 | 제어 방식 | 문자 |
|:---|:---|:---|
| **BSC** | 문자 지향 | ENQ/ACK/NAK |
| **HDLC** | 비트 지향 | 프래그 |
| **XMODEM** | 블록 전송 | SOH/ACK/NAK |

**📢 섹션 요약 비유**: XMODEM은 파일 전송에 사용!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 레거시 통신** → BSC
**시나리오 2: 파일 전송** → XMODEM/YMODEM
**시나리오 3: 현대 네트워크** → TCP (비트 지향)

**도입 체크리스트**: 프로토콜, 오류율, 전송량

**📢 섹션 요약 비유**: 현대는 HDLC/TCP 기반!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 통신 세션 관리 표준화

**미래 전망**: 레거시 유지보수 용도

**참고 표준**: ISO 1745 (BSC), ASCII

**📢 섹션 요약 비유**: 제어 문자는 통신의 신호등!
