# Keepalived - VRRP 기반 고가용성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: VRRP 기반 IP 페일오버
> 2. **가치**: 서비스 연속성
> 3. **융합**: HAProxy, LVS와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Keepalived는 VRRP 프로토콜을 사용하여 IP 주소 페일오버를 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Keepalived                                        │
├─────────────────────────────────────────────────────────────────────────┤

│   │   │  [Client]                                           │   │ │
```

> **해설**: Keepalived는 Master/Backup 구조로 VIP를 관리합니다.

**💡 비유**: Keepalived는 당직 교대 시스템입니다. 한 명이 쉬면 다른 사람이 대신합니다.

**📢 섹션 요약 비유:** Keepalived = VIP 페일오버!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **VRRP Instance** | 가상 라우터 |
| **Virtual IP** | 공유 IP |
| **Priority** | 우선순위 |
| **Preempt** | 선점 모드 |

**비교표: Keepalived 상태**

```
+------------+----------+----------+
| 상태       │ 동작     │ 조건     │
+------------+----------+----------+
| MASTER     │ VIP 소유 │ 높은 우선순위│
| BACKUP     │ 대기     │ 낮은 우선순위│
| FAULT      │ 비활성   │ 헬스체크 실패│
+------------+----------+----------+
```

**핵심 알고리즸:** VRRP 광고, 우선순위

**📢 섹션 요약 비유:** Priority = 높으면 마스터!
