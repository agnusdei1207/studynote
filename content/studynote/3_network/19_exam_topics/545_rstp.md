# RSTP (Rapid STP) - 고속 스패닝 트리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 빠른 수렴 STP
> 2. **가치**: 초고속 장애 복구
> 3. **융합:** STP, Edge Port, BPDU

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RSTP는 STP의 수렴 시간을 대폭 단축합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RSTP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              STP vs RSTP 수렴 시간                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  STP:  30-50초 (Listening + Learning)               │   │ │
│   │   │  RSTP: 1-2초  (즉시 전환)                            │   │ │
│   │   │                                                      │   │ │
│   │   │  RSTP 포트 역할:                                     │   │ │
│   │   │  - Root Port: Root로 가는 포트                       │   │ │
│   │   │  - Designated Port: 전달 포트                        │   │ │
│   │   │  - Alternate Port: Root 포트 백업                    │   │ │
│   │   │  - Backup Port: Designated 포트 백업                 │   │ │
│   │   │  - Edge Port: 호스트 연결 (PortFast)                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RSTP는 Alternate/Backup 포트로 즉시 전환합니다.

**📢 섹션 요약 비유:** RSTP = 비상구!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Alternate** | Root Port 대체 |
| **Backup** | Designated 대체 |
| **Edge** | 호스트 포트 |
| **Proposal/Agree** | 빠른 협상 |

**핵심 알고리즸:** 미리 계산된 대체 경로로 즉시 전환

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**STP vs RSTP**

| 항목 | STP | RSTP |
|:---|:---|:---|
| **수렴** | 30-50초 | 1-2초 |
| **상태** | 5개 | 3개 |
| **BPDU** | 정기 | 연속 |

**시나리오 3:** 레거시 → STP 호환
