# MAC - 메시지 인증 코드

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메시지 무결성과 인증
> 2. **가치**: 변조 탐지
> 3. **융합:** HMAC, CMAC, Keyed Hash

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MAC (Message Authentication Code)는 메시지 인증을 위한 코드입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MAC                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MAC 생성 과정                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  MAC 생성:                                               │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  메시지 + 비밀키                                 ││   │ │
│   │   │  │       │                                         ││   │ │
│   │   │  │       ▼                                         ││   │ │
│   │   │  │  ┌─────────┐                                    ││   │ │
│   │   │  │  │MAC 함수 │                                    ││   │ │
│   │   │  │  └────┬────┘                                    ││   │ │
│   │   │  │       │                                         ││   │ │
│   │   │  │       ▼                                         ││   │ │
│   │   │  │  MAC 값 (고정 길이)                               ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  HMAC (Hash-based MAC):                                │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ HMAC(K, m) = H((K⊕opad) || H((K⊕ipad) || m))   ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │ H: 해시 함수 (SHA-256 등)                        ││   │ │
│   │   │  │ K: 비밀키                                        ││   │ │
│   │   │  │ m: 메시지                                        ││   │ │
│   │   │  │ opad/ipad: 패딩                                  ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: MAC은 메시지 출처와 무결성을 보장합니다.

**📢 섹션 요약 비유:** MAC = 봉투 실링!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Key** | 비밀키 |
| **Message** | 메시지 |
| **Tag** | MAC 태그 |
| **Verify** | 검증 |
| **Algorithm** | 알고리즘 |

**핵심 알고리듬:** 메시지 + 키 → MAC 생성 → 검증

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**MAC vs Hash vs Signature**

| 항목 | Hash | MAC | Signature |
|:---|:---|:---|:---|
| **키** | 없음 | 대칭키 | 비대칭키 |
| **인증** | 불가 | 가능 | 가능 |
| **부인방지** | 불가 | 불가 | 가능 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** TLS → HMAC
**시나리오 2:** API → HMAC 인증
**시나리오 3:** JWT → HMAC-SHA256
