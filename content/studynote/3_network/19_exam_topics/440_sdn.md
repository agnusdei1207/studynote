# SDN (소프트웨어 정의 네트워킹) - Software Defined Networking

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제어부와 데이터부 분리
> 2. **가치**: 중앙 관리, 프로그래밍 가능
> 3. **융합**: OpenFlow, SDN Controller

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SDN은 네트워크 제어를 소프트웨어로 추상화합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SDN 아키텍처                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SDN 3계층 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Application Layer (애플리케이션 계층)       │    │   │ │
│   │   │  │ - 네트워크 앱                               │    │   │ │
│   │   │  │ - 정책 관리                                 │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                     │ NBI                           │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Control Layer (제어 계층)                   │    │   │ │
│   │   │  │ - SDN Controller                           │    │   │ │
│   │   │  │ - 중앙 집중 제어                           │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                     │ SBI (OpenFlow)                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Infrastructure Layer (인프라 계층)          │    │   │ │
│   │   │  │ - SDN 스위치                               │    │   │ │
│   │   │  │ - 데이터 플레인                            │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SDN vs 전통적 네트워크                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ 전통적   │ SDN      │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 제어       │ 분산     │ 중앙     │               │   │ │
│   │   │  │ 관리       │ 개별     │ 통합     │               │   │ │
│   │   │  | 프로그래밍 │ CLI      │ API      │               │   │ │
│   │   │  │ 유연성     │ 낮음     │ 높음     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SDN은 제어와 데이터를 분리하여 유연성을 높입니다.

**💡 비유**: SDN은 네트워크의 운영체제입니다.

**📢 섹션 요약 비유:** SDN = 소프트웨어 제어!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **OpenFlow** | SDN 표준 프로토콜 |
| **SDN Controller** | 중앙 제어 소프트웨어 |
| **NBI** | Northbound Interface |
| **SBI** | Southbound Interface |

**비교표: SDN Controller**

```
+------------+----------+----------+----------+
| Controller │ 언어     │ 특징     │ 용도     │
+------------+----------+----------+----------+
| OpenDaylight│ Java    │ 오픈소스 │ 엔터프라이즈│
| ONOS       │ Java     │ 분산     │ 통신사   │
| Floodlight │ Java     │ 경량     │ 교육     │
| Ryu        │ Python   │ 단순     │ 개발     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 플로우 매칭, 규칙 설치

**📢 섹션 요약 비유:** OpenFlow = SDN 통신 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: SDN 활용 사례**

| 사례 | 설명 | 이점 |
|:---|:---|:---|
| **데이터센터** | VM 이동성 | 자동화 |
| **WAN** | SD-WAN | 비용 절감 |
| **캠퍼스** | 정책 관리 | 단순화 |
| **클라우드** | 멀티테넌시 | 격리 |

**📢 섹션 요약 비유:** SD-WAN = WAN 혁신!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 데이터센터 → OpenDaylight
**시나리오 2:** WAN 최적화 → SD-WAN
**시나리오 3:** 클라우드 → Neutron

**도입 체크리스트:** 호환성, 스킬, ROI

**📢 섹션 요약 비유:** 중앙 제어 = 효율!
