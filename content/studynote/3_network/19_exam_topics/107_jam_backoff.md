# 잼 신호 / 백오프 알고리즘

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 충돌 시 모든 노드에 알리고 재전송 지연
> 2. **가치**: 충돌 후 질서 있는 복구
> 3. **융합**: 이더넷, CSMA/CD와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

잼 신호(Jam Signal)는 충돌 감지 시 모든 노드에 충돌을 알리는 신호이고, 백오프 알고리즘(Backoff Algorithm)은 충돌 후 재전송까지의 대기 시간을 결정합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    잼 신호 / 백오프 알고리즘                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              잼 신호 (Jam Signal)                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  목적:                                                   │   │ │
│   │   │  • 충돌 발생을 모든 노드에 알림                          │   │ │
│   │   │  • 진행 중인 전송 중단                                   │   │ │
│   │   │  • 프레임 폐기 유도                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                   │   │ │
│   │   │  • 32비트 (또는 48비트) 패턴                             │   │ │
│   │   │  • 모든 노드가 수신 가능                                  │   │ │
│   │   │  • 충돌 감지 후 즉시 전송                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  동작:                                                   │   │ │
│   │   │  노드A 전송 ────┐                                        │   │ │
│   │   │                  ├── 충돌! ──→ Jam 전송                 │   │ │
│   │   │  노드B 전송 ────┘                                        │   │ │
│   │   │                           ↓                              │   │ │
│   │   │                  모든 노드 충돌 인지                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Binary Exponential Backoff                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  알고리즘:                                               │   │ │
│   │   │                                                      │   │ │
│   │   │  1. 충돌 횟수 n 확인                                    │   │ │
│   │   │  2. 범위 계산: [0, 2^min(n, 10) - 1]                   │   │ │
│   │   │  3. 랜덤 값 r 선택                                       │   │ │
│   │   │  4. 대기 시간: r × 512 비트 시간                        │   │ │
│   │   │  5. 재전송                                              │   │ │
│   │   │  6. 16회 실패 시 폐기                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  충돌 횟수별 범위:                                        │   │ │
│   │   │  n=0: [0, 1]       → 2 슬롯                             │   │ │
│   │   │  n=1: [0, 3]       → 4 슬롯                             │   │ │
│   │   │  n=2: [0, 7]       → 8 슬롯                             │   │ │
│   │   │  n=3: [0, 15]      → 16 슬롯                            │   │ │
│   │   │  ...                                                     │   │ │
│   │   │  n=10: [0, 1023]   → 1024 슬롯 (최대)                   │   │ │
│   │   │                                                      │   │ │
│   │   │  슬롯 시간: 512 비트 전송 시간                            │   │ │
│   │   │  10Mbps: 51.2μs                                        │   │ │
│   │   │  100Mbps: 5.12μs                                       │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 백오프 알고리즘은 충돌이 반복될수록 대기 범위를 지수적으로 늘립니다.

**💡 비유**: 백오프는 처음엔 1분, 다음엔 2분, 그 다음엔 4분 기다리는 것입니다.

**📢 섹션 요약 비유**: Binary Exponential Backoff = 2^n!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Jam** | 충돌 알림 | 32비트 | 비상 |
| **슬롯 시간** | 대기 단위 | 512 비트 | 틱 |
| **최대 시도** | 포기 | 16회 | 한계 |

**비교표**

```
+------------+----------+----------+
| 충돌 횟수  | 범위     | 최대 대기|
+------------+----------+----------+
| 1          | 0-1      | 1 슬롯   |
| 2          | 0-3      | 3 슬롯   |
| 3          | 0-7      | 7 슬롯   |
| 10+        | 0-1023   | 1023 슬롯|
+------------+----------+----------+
```

**핵심 알고리즸**: 충돌 → Jam → n++ → r 선택 → r×슬롯 대기 → 재전송

**📢 섹션 요약 비유**: 16회 실패 = 프레임 폐기!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 백오프 특성**

| 특성 | 의미 |
|:---|:---|
| **적응성** | 부하에 따른 자동 조절 |
| **공정성** | 랜덤 선택으로 기회 균등 |
| **안정성** | 최대 1024 슬롯으로 제한 |

**📢 섹션 요약 비유**: 고부하에서 자동 백오프!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 저부하** → 빠른 재전송
**시나리오 2: 고부하** → 긴 백오프
**시나리오 3: 극단적 부하** → 프레임 폐기

**도입 체크리스트**: 네트워크 규모, 부하, 타임아웃

**📢 섹션 요약 비유**: 백오프는 자동 조절!
