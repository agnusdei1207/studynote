# YAML - 설정 데이터 형식

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사람이 읽기 쉬운 데이터 직렬화
> 2. **가치**: 가독성, 주석, 계층 구조
> 3. **융합**: Kubernetes, CI/CD, 설정

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

YAML(YAML Ain't Markup Language)은 사람이 읽기 쉬운 데이터 형식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    YAML                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              YAML 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  # 사용자 정보                                       │   │ │
│   │   │  name: 홍길동                                        │   │ │
│   │   │  age: 30                                             │   │ │
│   │   │  address:                                            │   │ │
│   │   │    city: 서울                                         │   │ │
│   │   │    zip: "12345"                                       │   │ │
│   │   │  hobbies:                                             │   │ │
│   │   │    - 독서                                             │   │ │
│   │   │    - 운동                                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              YAML 특징                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 특징       │ 설명     │ 가치     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 가독성     │ 높음     │ 이해     │               │   │ │
│   │   │  │ 주석       │ # 지원   │ 문서화   │               │   │ │
│   │   │  | 계층       │ 들여쓰기 │ 구조     │               │   │ │
│   │   │  │ 타입       │ 자동     │ 유연성   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: YAML은 들여쓰기로 계층을 표현합니다.

**💡 비유**: YAML은 깔끔한 메모입니다. 읽기 쉽게 정리합니다.

**📢 섹션 요약 비유:** YAML = 가독성!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Indentation** | 들여쓰기 (공백) |
| **Key-Value** | 키: 값 |
| **List** | 대시 (-) |
| **Anchor** | 앵커 (&/*) |

**비교표: YAML vs JSON**

```
+------------+----------+----------+
| 항목       │ YAML     │ JSON     │
+------------+----------+----------+
| 가독성     │ 높음     │ 중간     │
| 주석       │ 지원     │ 미지원   │
| 파싱       │ 복잡     │ 간단     │
| 용도       │ 설정     │ 데이터   │
+------------+----------+----------+
```

**핵심 알고리즸:** 들여쓰기 파싱, 타입 추론

**📢 섹션 요약 비유:** 들여쓰기 = 구조!
