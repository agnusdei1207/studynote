# Socket - 소켓

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 통신 끝점
> 2. **가치**: 프로세스 간 통신 인터페이스
> 3. **융합:** TCP/UDP, Port, File Descriptor

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Socket은 네트워크 통신의 끝점입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Socket                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              소켓 통신                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  소켓 주소:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  IP Address + Port Number                       ││   │ │
│   │   │  │  192.168.1.100 : 8080                           ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  소켓 유형:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ SOCK_STREAM: TCP (연결형, 신뢰성)                  ││   │ │
│   │   │  │ SOCK_DGRAM: UDP (비연결형, 빠름)                    ││   │ │
│   │   │  │ SOCK_RAW: Raw (직접 접근)                          ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  소켓 API:                                              │   │ │
│   │   │  socket() → bind() → listen() → accept()              │   │ │
│   │   │                    ↓                                  │   │ │
│   │   │              connect() → send()/recv()                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 소켓은 네트워크 프로그래밍의 기본입니다.

**📢 섹션 요약 비유:** Socket = 통신용 전화기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **FD** | 파일 디스크립터 |
| **Bind** | 주소 바인딩 |
| **Listen** | 대기 상태 |
| **Accept** | 연결 수락 |
| **Connect** | 연결 요청 |

**핵심 알고리듬:** 생성 → 바인딩 → 연결 → 통신 → 종료
