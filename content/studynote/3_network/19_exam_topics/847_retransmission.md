# Retransmission - 재전송

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 손실 패킷 재전송
> 2. **가치**: 신뢰성 보장
> 3. **융합:** RTO, Fast Retransmit, Exponential Backoff

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Retransmission은 손실된 패킷을 다시 보냅니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Retransmission                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              재전송 메커니즘                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Timeout 기반 재전송:                               │   │ │
│   │   │  Sender                    Receiver                  │   │ │
│   │   │  ┌─────┐                  ┌─────┐                   │   │ │
│   │   │  │     │ ── Seg 1 ──────→ │     │                   │   │ │
│   │   │  │     │ ── Seg 2 ──(X)── │     │ ← 손실            │   │ │
│   │   │  │     │     RTO 만료      │     │                   │   │ │
│   │   │  │     │ ── Seg 2 ──────→ │     │                   │   │ │
│   │   │  └─────┘                  └─────┘                   │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Fast Retransmit (3 Duplicate ACK):                │   │ │
│   │   │  ┌─────┐                  ┌─────┐                   │   │ │
│   │   │  │     │ ── Seg 1 ──────→ │     │                   │   │ │
│   │   │  │     │ ── Seg 2 ──(X)── │     │                   │   │ │
│   │   │  │     │ ── Seg 3 ──────→ │     │                   │   │ │
│   │   │  │     │ ←─ Dup ACK 2 ─── │     │                   │   │ │
│   │   │  │     │ ── Seg 4 ──────→ │     │                   │   │ │
│   │   │  │     │ ←─ Dup ACK 2 ─── │     │                   │   │ │
│   │   │  │     │ ── Seg 5 ──────→ │     │                   │   │ │
│   │   │  │     │ ←─ Dup ACK 2 ─── │     │                   │   │ │
│   │   │  │     │ ── Seg 2 ──────→ │     │ ← 3 Dup ACK       │   │ │
│   │   │  └─────┘                  └─────┘                   │   │ │
│   │   │                                                      │   │ │
│   │   │  Exponential Backoff:                                 │   │ │
│   │   │  RTO_new = RTO_old × 2 (최대 120초)                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 재전송은 신뢰적 통신의 핵심입니다.

**📢 섹션 요약 비유:** Retransmission = 재발송!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RTO** | 재전송 타임아웃 |
| **Fast Retransmit** | 빠른 재전송 |
| **Dup ACK** | 중복 ACK |
| **Backoff** | 백오프 |
| **Retry Limit** | 재시도 한도 |

**핵심 알고리즸:** 손실 감지 → 재전송 → ACK 대기

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**재전송 방식 비교**

| 방식 | 트리거 | 지연 |
|:---|:---|:---|
| **Timeout** | RTO 만료 | 높음 |
| **Fast Retransmit** | 3 Dup ACK | 낮음 |
| **SACK** | SACK 정보 | 낮음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일반 → Timeout + Fast Retransmit
**시나리오 2:** 고속 → SACK
**시나리오 3:** 무선 → 더 공격적
