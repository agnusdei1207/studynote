# DHCP Lease - DHCP 임대

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 주소 임대 시간
> 2. **가치**: 주소 풀 관리
> 3. **융합:** Renewal, Rebinding, Expiration

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DHCP Lease는 IP 주소의 임대 기간입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DHCP Lease                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              임대 갱신 과정                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  임대 타임라인:                                           │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  0%        50%        87.5%       100%          ││   │ │
│   │   │  │  │          │           │           │            ││   │ │
│   │   │  │  │          │           │           │            ││   │ │
│   │   │  │  │    T1 (Renewal)    T2 (Rebind)  만료          ││   │ │
│   │   │  │  │          │           │           │            ││   │ │
│   │   │  │  시작       갱신 시도    재바인딩     임대 종료     ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  예시 (임대 8시간):                               ││   │ │
│   │   │  │  - T1 (50%): 4시간 후 원래 서버로 갱신 시도         ││   │ │
│   │   │  │  - T2 (87.5%): 7시간 후 모든 서버로 갱신 시도       ││   │ │
│   │   │  │  - 100%: 8시간 후 임대 종료                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  갱신 과정:                                              │   │ │
│   │   │  - T1: DHCPREQUEST (유니캐스트)                         │   │ │
│   │   │  - T2: DHCPREQUEST (브로드캐스트)                        │   │ │
│   │   │  - 만료: DHCP Discover (처음부터)                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DHCP Lease는 IP 주소 풀 관리의 핵심입니다.

**📢 섹션 요약 비유:** DHCP Lease = 임대 계약!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Lease Time** | 임대 시간 |
| **T1** | 갱신 타이머 (50%) |
| **T2** | 재바인딩 타이머 (87.5%) |
| **Expiration** | 만료 |
| **Pool** | 주소 풀 |

**핵심 알고리듬:** T1 → T2 → 만료 → 재할당
