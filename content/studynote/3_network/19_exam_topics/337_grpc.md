# gRPC - 고성능 RPC

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고성능 원격 프로시저 호출

> 3. **융합**: 마이크로서비스, 다국어 지원

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

gRPC는 Google이 개발한 고성능 RPC 프레임워크입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    gRPC                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              gRPC 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ gRPC Call (Protocol Buffers + HTTP/2)          │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           gRPC Channel                      │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • Stub (클라이언트 프록시)                   │    │   │ │
│   │   │  │  • Serialization (직렬화)                    │    │   │ │
│   │   │  │  • HTTP/2 Transport                          │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ Network                                      │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [Server]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Service Implementation                         │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [Business Logic]                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              gRPC 통신 패턴                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 패턴       │ 설명     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Unary      │ 1:1      │ 일반     │               │   │ │
│   │   │  │ Server Streaming │ 1:N │ 대용량  │               │   │ │
│   │   │  | Client Streaming │ N:1 │ 업로드  │               │   │ │
│   │   │  │ Bidirectional │ N:N │ 채팅    │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: gRPC는 Protocol Buffers와 HTTP/2로 고성능 통신을 제공합니다.

**💡 비유**: gRPC는 서비스 간 전용선입니다. 빠르고 효율적입니다.

**📢 섹션 요약 비유:** gRPC = 고성능 RPC!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Protocol Buffers** | 직렬화 |
| **HTTP/2** | 전송 계층 |
| **Stub** | 프록시 |
| **Channel** | 연결 |

**비교표: gRPC vs REST**

```
+------------+----------+----------+
| 항목       │ gRPC     │ REST     │
+------------+----------+----------+
| 포맷       │ Protobuf │ JSON     │
| 전송       │ HTTP/2   │ HTTP/1.1 │
| 성능       │ 높음     │ 중간     │
| 브라우저   │ 제한     │ 지원     │
+------------+----------+----------+
```

**핵심 알고리즸:** Protobuf 직렬화, HTTP/2 멀티플렉싱

**📢 섹션 요약 비유:** Protobuf = 작고 빠름!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: IPC**

| 항목 | gRPC | REST | GraphQL |
|:---|:---|:---|:---|
| **성능** | 높음 | 중간 | 중간 |
| **타입** | 강함 | 약함 | 강함 |
| **브라우저** | 제한 | 지원 | 지원 |

**📢 섹션 요약 비유:** 마이크로서비스 내부 = gRPC!
