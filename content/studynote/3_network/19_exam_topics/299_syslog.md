# syslog - 시스템 로그

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 로그 메시지 표준
> 2. **가치**: 중앙 로그 관리
> 3. **융합**: SIEM, 모니터링과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

syslog는 시스템 로그 메시지를 전송하는 표준 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    syslog                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              syslog 구조                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Devices]                                          │   │ │
│   │   │   Router   Switch   Server   Firewall               │   │ │
│   │   │     │         │         │         │                  │   │ │
│   │   │     │ syslog  │ syslog  │ syslog  │                 │   │ │
│   │   │     │ UDP 514 │         │         │                 │   │ │
│   │   │     └─────────┴─────────┴─────────┘                 │   │ │
│   │   │                 │                                    │   │ │
│   │   │                 ▼                                    │   │ │
│   │   │        ┌──────────────────┐                          │   │ │
│   │   │        │  Syslog Server   │                          │   │ │
│   │   │        │  (Central Log)   │                          │   │ │
│   │   │        │                  │                          │   │ │
│   │   │        │  • 수집          │                          │   │ │
│   │   │        │  • 저장          │                          │   │ │
│   │   │        │  • 분석          │                          │   │ │
│   │   │        │  • 알림          │                          │   │ │
│   │   │        └──────────────────┘                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              syslog 메시지 형식                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  PRI: Priority (Facility × 8 + Severity)            │   │ │
│   │   │  HEADER: Timestamp + Hostname                        │   │ │
│   │   │  MSG: 로그 메시지                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  예: <134>Oct 11 22:14:15 router sshd[1234]:        │   │ │
│   │   │      Accepted password for user from 10.0.0.1        │   │ │
│   │   │                                                      │   │ │
│   │   │  PRI = Facility(16) × 8 + Severity(6) = 134         │   │ │
│   │   │  (Local0 = 16, Info = 6)                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: syslog는 UDP 514 포트로 로그를 전송합니다.

**💡 비유**: syslog는 회사 게시판에 모든 공지를 모아놓는 것입니다.

**📢 섹션 요약 비유:** syslog = 중앙 로그!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Facility** | 출처 |
| **Severity** | 심각도 |
| **PRI** | 우선순위 |
| **RFC 5424** | 최신 표준 |

**비교표: Severity 레벨**

```
+------------+----------+----------+
| 레벨       │ 값       │ 설명     │
+------------+----------+----------+
| Emergency  │ 0        │ 시스템 중단│
| Alert      │ 1        │ 즉시 조치 │
| Critical   │ 2        │ 심각     │
| Error      │ 3        │ 오류     │
| Warning    │ 4        │ 경고     │
| Notice     │ 5        │ 알림     │
| Info       │ 6        │ 정보     │
| Debug      │ 7        │ 디버그   │
+------------+----------+----------+
```

**핵심 알고리즸:** Facility × 8 + Severity

**📢 섹션 요약 비유:** Emergency = 최고 심각!
