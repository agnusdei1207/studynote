# TLS - 전송 계층 보안

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 계층 암호화
> 2. **가치**: 안전한 통신
> 3. **융합:** HTTPS, Certificate, Handshake

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TLS (Transport Layer Security)는 전송 계층에서 보안을 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TLS                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TLS 핸드셰이크                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  TLS 1.2 Handshake:                                    │   │ │
│   │   │  Client                    Server                    │   │ │
│   │   │  ┌─────┐                  ┌─────┐                   │   │ │
│   │   │  │     │ ── ClientHello ─→│     │                   │   │ │
│   │   │  │     │←─ ServerHello ── │     │                   │   │ │
│   │   │  │     │←─ Certificate ── │     │                   │   │ │
│   │   │  │     │←─ ServerHelloDone │   │                   │   │ │
│   │   │  │     │ ── ClientKeyExchange ─                     │   │ │
│   │   │  │     │ ── ChangeCipherSpec ──                     │   │ │
│   │   │  │     │ ── Finished ────→│     │                   │   │ │
│   │   │  │     │←─ ChangeCipherSpec ──                      │   │ │
│   │   │  │     │←─ Finished ───── │     │                   │   │ │
│   │   │  └─────┘                  └─────┘                   │   │ │
│   │   │                                                      │   │ │
│   │   │  TLS 1.3 (간소화):                                     │   │ │
│   │   │  - 1-RTT 핸드셰이크                                     │   │ │
│   │   │  - 0-RTT 재연결                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  제공 기능:                                              │   │ │
│   │   │  - 기밀성 (Confidentiality)                             │   │ │
│   │   │  - 무결성 (Integrity)                                   │   │ │
│   │   │  - 인증 (Authentication)                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: TLS는 HTTPS의 보안 기반입니다.

**📢 섹션 요약 비유:** TLS = 안전한 파이프!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Handshake** | 핸드셰이크 |
| **Record** | 레코드 프로토콜 |
| **Cipher Suite** | 암호화 스위트 |
| **Certificate** | 인증서 |
| **Session Key** | 세션 키 |

**핵심 알고리즸:** 핸드셰이크 → 키 교환 → 암호화 통신

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**TLS 버전 비교**
