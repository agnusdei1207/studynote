# Quorum - 쿼럼

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 합의에 필요한 최소 수
> 2. **가치**: 일관성 보장, 내결함성
> 3. **융합:** Distributed Systems, Raft, Paxos

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Quorum은 합의에 필요한 최소 노드 수입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Quorum                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Quorum 계산                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Majority Quorum (과반수):                            │   │ │
│   │   │  Quorum = (N / 2) + 1                                │   │ │
│   │   │                                                      │   │ │
│   │   │  예시 (5 nodes):                                      │   │ │
│   │   │  Quorum = (5 / 2) + 1 = 3                            │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐           │   │ │
│   │   │  │ N1  │ │ N2  │ │ N3  │ │ N4  │ │ N5  │           │   │ │
│   │   │  │  ✓  │ │  ✓  │ │  ✓  │ │     │ │     │           │   │ │
│   │   │  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘           │   │ │
│   │   │           ↑ Quorum = 3                              │   │ │
│   │   │                                                      │   │ │
│   │   │  내결함성:                                            │   │ │
│   │   │  장애 허용 = (N - 1) / 2                             │   │ │
│   │   │  5 nodes → 2 장애 허용                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Quorum은 분산 시스템에서 합의를 보장합니다.

**📢 섹션 요약 비유:** Quorum = 투표 정족수!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **R** | 읽기 쿼럼 |
| **W** | 쓰기 쿼럼 |
| **N** | 복제본 수 |
| **R + W > N** | 일관성 조건 |
| **Split-brain** | 분할 뇌 |

**핵심 알고리즸:** 쓰기 → W개 응답 → 읽기 → R개 응답

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Quorum 전략**

| 전략 | R | W | 특징 |
|:---|:---|:---|:---|
| **Strict** | N | N | 강한 일관성 |
| **Majority** | N/2+1 | N/2+1 | 균형 |
| **Eventual** | 1 | 1 | 높은 가용성 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 금융 → Strict Quorum
**시나리오 2:** 일반 → Majority
**시나리오 3:** 캐시 → Eventual
