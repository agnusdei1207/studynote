# Hash - 해시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단방향 변환
> 2. **가치**: 무결성 검증
> 3. **융합:** MD5, SHA, Digital Signature

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Hash는 임의 길이 데이터를 고정 길이로 변환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Hash                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              해시 특성                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  해시 특성:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  1. 단방향성: 역변환 불가                          ││   │ │
│   │   │  │     Hash(x) → y  ✅                             ││   │ │
│   │   │  │     Hash^-1(y) → x  ❌                          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  2. 충돌 저항성: 같은 출력 불가                    ││   │ │
│   │   │  │     Hash(x) ≠ Hash(y) when x ≠ y                ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  3. 고정 출력: 입력 크기 무관                      ││   │ │
│   │   │  │     SHA-256: 항상 256-bit                       ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  4. 빠른 계산                                    ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  주요 알고리즘:                                          │   │ │
│   │   │  - MD5: 128-bit (취약, 사용 권장 안 함)                 │   │ │
│   │   │  - SHA-1: 160-bit (취약)                               │   │ │
│   │   │  - SHA-256: 256-bit (안전)                              │   │ │
│   │   │  - SHA-3: 최신 표준                                      │   │ │
│   │   │  - BLAKE2: 고속                                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 해시는 무결성 검증의 핵심입니다.

**📢 섹션 요약 비유:** Hash = 디지털 지문!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Input** | 입력 데이터 |
| **Digest** | 해시값 |
| **Collision** | 충돌 |
| **Salt** | 솔트값 |
| **Avalanche** | 눈사태 효과 |

**핵심 알고리듬:** 데이터 → 해시 함수 → 고정 길이 출력

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**해시 알고리즘 비교**

| 알고리즘 | 출력 | 상태 |
|:---|:---|:---|
| **MD5** | 128-bit | 취약 |
| **SHA-1** | 160-bit | 취약 |
| **SHA-256** | 256-bit | 안전 |
| **SHA-3** | 가변 | 안전 |
