# WebRTC - 실시간 통신

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 브라우저 간 실시간 통신
> 2. **가치**: P2P, 오디오/비디오/데이터
> 3. **융합**: 화상회의, 파일 공유, 게임

### Ⅰ. 개요 (Context & Background)

**개념 정의**

WebRTC는 브라우저 간 P2P 실시간 통신을 제공합니다.


> **해설**: WebRTC는 시그널링 서버를 통해 연결 후 P2P로 통신합니다.

**💡 비유**: WebRTC는 브라우저 간 직통 전화입니다. 중계 없이 연결됩니다.

**📢 섹션 요약 비유:** WebRTC = P2P 실시간!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SDP** | 세션 설명 |
| **ICE** | 연결 설정 |
| **STUN** | NAT 통과 |
| **TURN** | 릴레이 서버 |

**비교표: NAT 통과 기술**

```
+------------+----------+----------+
| 기술       │ 설명     │ 특징     │
+------------+----------+----------+
| STUN       │ 주소 획득 │ 가벼움   │
| TURN       │ 릴레이   │ 확실     │
| ICE        │ 프레임워크│ 조합     │
+------------+----------+----------+
```

**핵심 알고리즸:** ICE, SRTP 암호화

**📢 섹션 요약 비유:** STUN = NAT 통과!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 실시간 통신**

| 항목 | WebRTC | WebSocket | HLS |
|:---|:---|:---|:---|
| **지연** | 초저지연 | 낮음 | 높음 |
| **P2P** | 지원 | 아니오 | 아니오 |
| **미디어** | 네이티브 | 수동 | 스트리밍 |

**📢 섹션 요약 비유:** 화상회의 = WebRTC!
