# Syslog - 로그 수집

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 중앙 로그 수집 프로토콜
> 2. **가치**: 장애 분석, 보안 감사
> 3. **융합**: SIEM, ELK와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Syslog는 네트워크 장비와 시스템의 로그를 중앙 서버로 수집하는 표준 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Syslog                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Syslog 구조                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [네트워크 장비]     [서버]      [Syslog 서버]       │   │ │
│   │   │                                                      │   │ │
│   │   │   Router ─────┐                                     │   │ │
│   │   │   Switch ─────┼───────────> UDP 514 ──> 중앙 수집  │   │ │
│   │   │   Firewall ───┘                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  포맷: PRI + HEADER + MSG                           │   │ │
│   │   │  <PRI>Timestamp Hostname Program: Message           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Syslog Facility                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Facility │ 용도                                     │   │ │
│   │   │  ─────────┼────────────────────────────────────│   │ │
│   │   │  0        │ Kernel                                 │   │ │
│   │   │  1        │ User                                   │   │ │
│   │   │  2        │ Mail                                   │   │ │
│   │   │  3        │ Daemon                                 │   │ │
│   │   │  4        │ Auth                                   │   │ │
│   │   │  16-23    │ Local0-7 (사용자 정의)                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Syslog는 Facility와 Severity로 로그를 분류합니다.

**💡 비유**: Syslog는 CCTV 녹화 시스템입니다. 모든 활동을 기록합니다.

**📢 섹션 요약 비유:** UDP 514 = Syslog!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| Severity | 레벨 | 설명 |
|:---:|:---:|:---|
| **0** | Emergency | 시스템 사용 불가 |
| **1** | Alert | 즉시 조치 필요 |
| **2** | Critical | 심각 |
| **3** | Error | 오류 |
| **4** | Warning | 경고 |
| **5** | Notice | 통지 |
| **6** | Informational | 정보 |
| **7** | Debug | 디버그 |

**비교표: RFC 5424 포맷**

```
+------------+----------+----------+
| 필드       │ 설명     │ 예시     │
+------------+----------+----------+
| PRI        │ 우선순위 │ <134>    │
| VERSION    │ 버전     │ 1        │
| TIMESTAMP  │ 시간     │ ISO 8601 │
| HOSTNAME   │ 호스트   │ router1  │
| APP-NAME   │ 프로그램 │ kernel   │
+------------+----------+----------+
```

**핵심 알고리즸:** Facility * 8 + Severity = PRI

**📢 섹션 요약 비유:** 0-2 = 즉시 조치!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 로그 수집**

| 도구 | 용도 |
|:---|:---|
| **Syslog** | 표준 |
| **ELK** | 분석 |
| **Splunk** | SIEM |
| **Graylog** | 오픈소스 |

**📢 섹션 요약 비유:** Syslog + ELK = 완벽한 분석!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 네트워크** → Syslog 서버 + Severity 0-4
**시나리오 2: 보안** → SIEM 연동
**시나리오 3: 장애 분석** → ELK Stack

**도입 체크리스트:** 저장 공간, 로테이션, 보안(TLS)

**📢 섹션 요약 비유:** 로그 보관 = 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 장애 분석, 보안 감사

**미래 전망:** Syslog over TLS, 구조화 로그

**참고 표준:** RFC 5424

**📢 섹션 요약 비유:** Syslog는 장애 분석의 기본!
