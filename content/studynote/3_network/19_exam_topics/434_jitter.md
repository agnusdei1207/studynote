# 지터 - Jitter

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 지연 시간의 변동성
> 2. **가치**: 실시간 품질 영향
> 3. **융합**: VoIP, 화상회의, QoS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Jitter는 패킷 지연 시간의 변동을 의미합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Jitter                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Jitter 개념                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  지연 시간 변화:                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  패킷1: 20ms                                         │   │ │
│   │   │  패킷2: 25ms  ─┐                                     │   │ │
│   │   │  패킷3: 15ms    │ Jitter = 10ms                      │   │ │
│   │   │  패킷4: 35ms  ─┘                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  Jitter = 최대지연 - 최소지연                         │   │ │
│   │   │  (또는 평균 변동)                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              영향 및 완화                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 영향       │ 증상     │ 완화     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ VoIP       │ 끊김    │ Jitter Buffer│            │   │ │
│   │   │  │ 화상회의   │ 화면끊김│ QoS      │               │   │ │
│   │   │  | 게임       │ 랙      │ 예측     │               │   │ │
│   │   │  │ 스트리밍   │ 버퍼링  │ Buffer   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Jitter는 실시간 애플리케이션에 큰 영향을 줍니다.

**💡 비유**: Jitter는 교통 상황의 변화입니다. 예측할 수 없는 지연입니다.

**📢 섹션 요약 비유:** Jitter = 지연 변동!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Jitter Buffer** | 지터 완화 버퍼 |
| **Packet Delay Variation** | PDV |
| **RTP Jitter** | RTP 지터 측정 |
| **Dejitter Buffer** | 디지터 버퍼 |

**비교표: 목표 Jitter**

```
+------------+----------+----------+
| 애플리케이션│ 목표     │ 최대     │
+------------+----------+----------+
| VoIP       │ <30ms    │ <50ms    │
| 화상회의   │ <50ms    │ <100ms   │
| 게임       │ <10ms    │ <30ms    │
+------------+----------+----------+
```

**핵심 알고리즸:** 버퍼링, 재정렬

**📢 섹션 요약 비유:** 버퍼 = Jitter 완화!
