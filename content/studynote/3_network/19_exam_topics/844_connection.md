# Connection - 연결

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 통신 경로 설정
> 2. **가치**: 상태 유지
> 3. **융합:** TCP 3-way, 4-way, Stateful

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Connection은 두 호스트 간 통신 경로를 설정합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Connection                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TCP 연결 설정/해제                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  3-Way Handshake (연결 설정):                          │   │ │
│   │   │                                                      │   │ │
│   │   │  Client                    Server                    │   │ │
│   │   │  ┌─────┐                  ┌─────┐                   │   │ │
│   │   │  │     │ ── SYN ────────→ │     │                   │   │ │
│   │   │  │     │     (ISN=C)      │     │                   │   │ │
│   │   │  │     │ ←─ SYN+ACK ───── │     │                   │   │ │
│   │   │  │     │   (ISN=S,ACK=C+1)│     │                   │   │ │
│   │   │  │     │ ── ACK ────────→ │     │                   │   │ │
│   │   │  │     │     (ACK=S+1)    │     │                   │   │ │
│   │   │  └─────┘                  └─────┘                   │   │ │
│   │   │                                                      │   │ │
│   │   │  4-Way Handshake (연결 해제):                          │   │ │
│   │   │                                                      │   │ │
│   │   │  Client                    Server                    │   │ │
│   │   │  ┌─────┐                  ┌─────┐                   │   │ │
│   │   │  │     │ ── FIN ────────→ │     │                   │   │ │
│   │   │  │     │ ←─ ACK ───────── │     │                   │   │ │
│   │   │  │     │ ←─ FIN ───────── │     │                   │   │ │
│   │   │  │     │ ── ACK ────────→ │     │                   │   │ │
│   │   │  └─────┘                  └─────┘                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 연결은 신뢰적 통신의 기반입니다.

**📢 섹션 요약 비유:** Connection = 전화 통화!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SYN** | 동기화 |
| **ACK** | 확인 응답 |
| **FIN** | 종료 |
| **RST** | 재설정 |
| **State** | 상태 |

**핵심 알고리즸:** SYN → SYN-ACK → ACK → 연결

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**연결형 vs 비연결형**

| 항목 | 연결형 | 비연결형 |
|:---|:---|:---|
| **예시** | TCP | UDP |
| **신뢰성** | 있음 | 없음 |
| **오버헤드** | 높음 | 낮음 |
| **용도** | 데이터 | 스트리밍 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹 → TCP 연결
**시나리오 2:** DNS → UDP 비연결
**시나리오 3:** 긴 전송 → Keep-alive
