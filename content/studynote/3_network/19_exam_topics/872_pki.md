# PKI - 공개키 기반 구조

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공개키 관리 체계
> 2. **가치**: 신뢰 구축
> 3. **융합:** CA, Certificate, CRL, OCSP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

PKI (Public Key Infrastructure)는 공개키를 관리하는 체계입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PKI (Public Key Infrastructure)                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PKI 구성 요소                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  구성 요소:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  CA (Certificate Authority)                     ││   │ │
│   │   │  │  - 인증서 발급/관리                               ││   │ │
│   │   │  │  - Root CA, Intermediate CA                      ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  RA (Registration Authority)                    ││   │ │
│   │   │  │  - 신원 확인, 등록                                ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Repository                                     ││   │ │
│   │   │  │  - 인증서 저장소                                  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  CRL/OCSP                                       ││   │ │
│   │   │  │  - 인증서 폐기 목록                               ││   │ │
│   │   │  │  - 온라인 폐기 확인                               ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  인증서 폐기:                                            │   │ │
│   │   │  - CRL: 목록 다운로드                                   │   │ │
│   │   │  - OCSP: 실시간 조회                                    │   │ │
│   │   │  - OCSP Stapling: 서버가 대신 확인                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: PKI는 공개키 암호의 신뢰 기반입니다.

**📢 섹션 요약 비유:** PKI = 신원 확인 시스템!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **CA** | 인증 기관 |
| **RA** | 등록 기관 |
| **CRL** | 폐기 목록 |
| **OCSP** | 온라인 확인 |
| **Trust Store** | 신뢰 저장소 |

**핵심 알고리즸:** 등록 → 검증 → 발급 → 관리 → 폐기

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**CA 유형 비교**

| 유형 | 신뢰 수준 | 비용 |
|:---|:---|:---|
| **Public CA** | 높음 | 유료 |
| **Private CA** | 내부 | 무료 |
| **Self-signed** | 낮음 | 무료 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 인터넷 → Public CA
**시나리오 2:** 기업 내부 → Private CA
**시나리오 3:** 개발 → Self-signed
