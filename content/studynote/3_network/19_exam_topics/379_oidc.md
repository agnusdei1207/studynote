# OpenID Connect - 인증 계층

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: OAuth 2.0 기반 인증 프로토콜
> 2. **가치**: 표준화된 사용자 정보
> 3. **융합**: SSO, 소셜 로그인

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

OpenID Connect(OIDC)는 OAuth 2.0 위에 인증 계층을 추가합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OpenID Connect                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OIDC 흐름                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [User]                                             │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 1. 인증 요청 (scope=openid)                    │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [RP] ──────▶ [OP]                                   │   │ │
│   │   │                   │                                  │   │ │
│   │   │                   │ 2. 로그인 & 동의                  │   │ │
│   │   │                   ▼                                  │   │ │
│   │   │  [User] ◀─────────┘                                  │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 3. 인가 코드                                   │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [RP] ──────▶ [OP]                                   │   │ │
│   │   │     │            │                                   │   │ │
│   │   │     │ ID Token   │                                   │   │ │
│   │   │     │ UserInfo   │                                   │   │ │
│   │   │     ▼            ▼                                   │   │ │
│   │   │  [인증 완료]                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OIDC 토큰                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 토큰       │ 용도     │ 내용     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ ID Token   │ 인증     │ 사용자   │               │   │ │
│   │   │  │ Access     │ 인가     │ 접근권한 │               │   │ │
│   │   │  | Refresh    │ 갱신     │ 장기     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: OIDC는 ID Token으로 사용자 인증을 제공합니다.

**💡 비유**: OIDC는 신분증입니다. 누구인지 증명합니다.

**📢 섹션 요약 비유:** OIDC = 인증!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RP** | Relying Party |
| **OP** | OpenID Provider |
| **ID Token** | JWT 인증 토큰 |
| **UserInfo** | 사용자 정보 엔드포인트 |

**비교표: OAuth vs OIDC**

```
+------------+----------+----------+
| 항목       │ OAuth    │ OIDC     │
+------------+----------+----------+
| 목적       │ 인가     │ 인증     │
| ID Token   │ 없음     │ 있음     │
| 사용자     │ 모름     │ 식별     │
| 표준화     │ 제한     │ 완전     │
+------------+----------+----------+
```

**핵심 알고리즸:** JWT 검증, UserInfo 조회

**📢 섹션 요약 비유:** 인증 = OIDC!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 인증 프로토콜**

| 항목 | OIDC | SAML | Kerberos |
|:---|:---|:---|:---|
| **형식** │ JSON │ XML │ 티켓 |
| **웹** │ 적합 │ 적합 │ 제한 |
| **모바일** │ 적합 │ 제한 │ 제한 |

**📢 섹션 요약 비유:** 소셜 로그인 = OIDC!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 소셜 로그인 → OIDC (Google, Facebook)
**시나리오 2:** 기업 SSO → OIDC + SAML
**시나리오 3:** 모바일 앱 → OIDC + PKCE

**도입 체크리스트:** OP 선택, 스코프, 세션 관리

**📢 섹션 요약 비유:** 웹/모바일 = OIDC!
