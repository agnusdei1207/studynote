# SIP (세션 개시 프로토콜) - Session Initiation Protocol

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멀티미디어 세션 제어 프로토콜
> 2. **가치**: 음성/영상 통화, IM, 현재 상태
> 3. **융합**: VoLTE, WebRTC, Softphone

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SIP는 멀티미디어 세션을 설정, 수정, 종료합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SIP 아키텍처                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SIP 호출 흐름                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Caller (UA)                  Callee (UA)           │   │ │
│   │   │       │                              │               │   │ │
│   │   │       │─────── INVITE ────────────►│               │   │ │
│   │   │       │                              │               │   │ │
│   │   │       │◄────── 100 Trying ─────────│               │   │ │
│   │   │       │◄────── 180 Ringing ────────│               │   │ │
│   │   │       │                              │               │   │ │
│   │   │       │◄────── 200 OK ─────────────│               │   │ │
│   │   │       │                              │               │   │ │
│   │   │       │─────── ACK ────────────────►│               │   │ │
│   │   │       │                              │               │   │ │
│   │   │       │◄══════ RTP Media ════════►│               │   │ │
│   │   │       │                              │               │   │ │
│   │   │       │─────── BYE ─────────────────►│               │   │ │
│   │   │       │◄────── 200 OK ─────────────│               │   │ │
│   │   │       │                              │               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SIP 메서드                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 메서드     │ 설명     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ INVITE     │ 세션시작 │ 통화     │               │   │ │
│   │   │  │ ACK        │ 확인     │ 완료     │               │   │ │
│   │   │  | BYE        │ 종료     │ 통화종료 │               │   │ │
│   │   │  │ CANCEL     │ 취소     │ 취소     │               │   │ │
│   │   │  │ REGISTER   │ 등록     │ 위치     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SIP는 신호 프로토콜이며 미디어는 RTP를 사용합니다.

**💡 비유**: SIP는 전화번호와 다이얼입니다.

**📢 섹션 요약 비유:** SIP = 세션 제어!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **UA** | User Agent |
| **Proxy** | SIP 프록시 |
| **Registrar** | 등록 서버 |
| **Redirect** | 리다이렉트 서버 |

**비교표: SIP 응답 코드**

```
+------------+----------+----------+----------+
| 코드       │ 클래스    │ 설명     │ 예시     │
+------------+----------+----------+----------+
| 1xx        │ 정보     │ 진행     │ 100/180  │
| 2xx        │ 성공     │ 완료     │ 200 OK   │
| 3xx        │ 리다이렉트│ 이동     │ 302      │
| 4xx        │ 클라이언트│ 오류     │ 404/486  │
| 5xx        │ 서버     │ 오류     │ 500      │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** SDP 협상, 라우팅

**📢 섹션 요약 비유:** SDP = 미디어 협상!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: SIP vs H.323**

| 항목 | SIP | H.323 |
|:---|:---|:---|
| **기반** | 텍스트 | 바이너리 |
| **복잡성** | 단순 | 복잡 |
| **확장성** | 높음 | 낮음 |

**📢 섹션 요약 비유:** SIP = 웹 방식!
