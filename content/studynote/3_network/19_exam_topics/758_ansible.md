# Ansible - 앤서블

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 에이전트 없는 자동화
> 2. **가치**: 단순함, SSH 기반
> 3. **융합:** Config Management, Orchestration, Deployment

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Ansible은 SSH로 자동화를 수행합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Ansible                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Ansible 아키텍처                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Control Node                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Inventory      Playbooks                     │    │   │ │
│   │   │  │ [web]          - hosts: web                  │    │   │ │
│   │   │  │ server1        tasks:                        │    │   │ │
│   │   │  │ server2        - name: Install nginx         │    │   │ │
│   │   │  │ [db]             apt:                        │    │   │ │
│   │   │  │ server3            name: nginx               │    │   │ │
│   │   │  │              - name: Start nginx             │    │   │ │
│   │   │  │              service:                        │    │   │ │
│   │   │  │                name: nginx                   │    │   │ │
│   │   │  │                state: started                │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │              │                                       │   │ │
│   │   │              │ SSH                                   │   │ │
│   │   │       ┌──────┴──────┐                               │   │ │
│   │   │       ↓             ↓                               │   │ │
│   │   │  ┌─────────┐   ┌─────────┐                         │   │ │
│   │   │  │ Target  │   │ Target  │                         │   │ │
│   │   │  │ (No     │   │ (No     │                         │   │ │
│   │   │  │ Agent)  │   │ Agent)  │                         │   │ │
│   │   │  └─────────┘   └─────────┘                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Ansible은 에이전트 없이 SSH로 관리합니다.

**📢 섹션 요약 비유:** Ansible = 원격 조종!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Inventory** | 대상 목록 |
| **Playbook** | 실행 스크립트 |
| **Task** | 작업 |
| **Module** | 모듈 |
| **Role** | 역할 |

**핵심 알고리즸:** Inventory → Playbook → SSH → 실행
