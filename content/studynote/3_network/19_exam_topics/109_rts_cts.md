# RTS/CTS (요청 전송 / 전송 허가)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 송신 전 예약으로 충돌 방지
> 2. **가치**: 숨은 노드 문제 해결
> 3. **융합**: WiFi, 무선 센서 네트워크와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RTS/CTS(Request To Send/Clear To Send)는 송신 전에 작은 제어 프레임으로 채널을 예약하여 숨은 노드 문제를 해결하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RTS/CTS (요청 전송 / 전송 허가)                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              숨은 노드 문제                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  상황:                                                   │   │ │
│   │   │                                                      │   │ │
│   │   │        A        AP        B                          │   │ │
│   │   │        │←─────→│←─────→│                             │   │ │
│   │   │        └ 범위  ┘  범위  ┘                              │   │ │
│   │   │                                                      │   │ │
│   │   │  문제:                                                   │   │ │
│   │   │  A와 B는 서로의 신호를 감지 못함                         │   │ │
│   │   │  → 동시에 AP로 전송                                     │   │ │
│   │   │  → AP에서 충돌 발생                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  결과:                                                   │   │ │
│   │   │  A와 B는 전송 성공했다고 착각                            │   │ │
│   │   │  (ACK 못 받고 나서야 실패 인지)                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RTS/CTS 해결책                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  과정:                                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  1. A → AP: RTS (Request To Send)                     │   │ │
│   │   │     "데이터 보내도 되나요?"                             │   │ │
│   │   │                                                      │   │ │
│   │   │  2. AP → A (및 B): CTS (Clear To Send)                │   │ │
│   │   │     "네, 보내세요"                                     │   │ │
│   │   │     → B도 CTS를 들음!                                  │   │ │
│   │   │     → B는 NAV 동안 전송 안 함                           │   │ │
│   │   │                                                      │   │ │
│   │   │  3. A → AP: DATA                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  4. AP → A: ACK                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  NAV (Network Allocation Vector):                      │   │ │
│   │   │  • CTS에 포함된 예약 시간                               │   │ │
│   │   │  • 주변 노드가 이 시간 동안 전송 금지                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: RTS/CTS는 긴 데이터 프레임 전송 시 충돌 비용을 줄입니다.

**💡 비유**: RTS/CTS는 발언권을 요청하고 허가받는 것입니다.

**📢 섹션 요약 비유**: RTS → CTS → DATA → ACK!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 프레임 | 방향 | 목적 | 크기 |
|:---|:---|:---|:---:|
| **RTS** | STA→AP | 전송 요청 | 20B |
| **CTS** | AP→STA | 전송 허가 | 14B |
| **NAV** | - | 예약 시간 | 필드 |

**비교표**

```
+------------+----------+----------+
| 항목       | RTS 없음 | RTS 있음 |
+------------+----------+----------+
| 숨은 노드  | 충돌     | 해결     |
| 오버헤드   | 없음     | 있음     |
| 짧은 데이터| 유리     | 불리     |
| 긴 데이터  | 불리     | 유리     |
+------------+----------+----------+
```

**핵심 알고리즸**: RTS → SIFS → CTS → SIFS → DATA → SIFS → ACK

**📢 섹션 요약 비유**: RTS/CTS 임계값 = 2347바이트!
