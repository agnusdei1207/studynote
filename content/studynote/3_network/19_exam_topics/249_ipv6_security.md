# IPv6 보안 - 보안 고려사항

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPv6 고유 보안 이슈
> 2. **가치**: 확장 헤더, NDP 보안
> 3. **융합**: IPSec, RA Guard와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IPv6 보안은 IPv6 프로토콜의 고유한 보안 특성과 취약점을 다룹니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IPv6 보안                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IPv6 보안 이슈                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. NDP 공격:                                        │   │ │
│   │   │     • RA 스푸핑                                      │   │ │
│   │   │     • NS/NA 스푸핑                                   │   │ │
│   │   │     • DAD 공격                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 확장 헤더 악용:                                  │   │ │
│   │   │     • 과도한 체인                                    │   │ │
│   │   │     • 우회                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 스캔:                                            │   │ │
│   │   │     • 거대 주소 공간                                 │   │ │
│   │   │     • 난수화로 완화                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 터널링:                                          │   │ │
│   │   │     • 캡슐화 우회                                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              보안 대책                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  대책           │ 대상              │ 표준          │   │ │
│   │   │  ─────────────┼─────────────────┼──────────────│   │ │
│   │   │  RA Guard     │ RA 스푸핑        │ RFC 6105     │   │ │
│   │   │  DHCPv6 Guard │ DHCPv6 스푸핑    │ RFC 7610     │   │ │
│   │   │  SEND         │ NDP 공격         │ RFC 6494     │   │ │
│   │   │  IPSec        │ 암호화           │ RFC 4301     │   │ │
│   │   │  Privacy Ext  │ 주소 추적        │ RFC 4941     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IPv6는 스캐닝이 어렵지만 NDP 공격에 취약합니다.

**💡 비유**: IPv6 보안은 넓은 집에 살면 도둑이 집을 못 찾지만, 편지는 위조될 수 있습니다.

**📢 섹션 요약 비유:** RA Guard = 필수!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RA Guard** | 스위치에서 RA 차단 |
| **DHCPv6 Guard** | DHCPv6 서버 제한 |
| **SEND** | NDP 보안 |
| **IPSec** | 암호화 (필수) |

**비교표: IPv4 vs IPv6 보안**

```
+------------+----------+----------+
| 항목       │ IPv4     │ IPv6     │
+------------+----------+----------+
| ARP 스푸핑 │ 취약     │ 해당없음 │
| NDP 스푸핑 │ 없음     │ 취약     │
| 스캔       │ 쉬움     │ 어려움   │
| NAT        │ 보안     │ 없음     │
| IPSec      │ 선택     │ 필수     │
+------------+----------+----------+
```

**핵심 알고리즸:** RA Guard, SEND

**📢 섹션 요약 비유:** IPv6 = 스캔 어려움, NDP 취약!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 주소 프라이버시**

| 유형 | 설명 |
|:---|:---|
| **EUI-64** | MAC 기반 (추적 가능) |
| **Privacy Ext** | 랜덤 (RFC 4941) |
| **Stable Privacy** | 안정적 랜덤 |

**📢 섹션 요약 비유:** Privacy Ext = 추적 방지!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 기업** → RA Guard + DHCPv6 Guard
**시나리오 2: 데이터센터** → IPSec + 방화벽
**시나리오 3: ISP** → SEND

**도입 체크리스트:** RA Guard, 방화벽 IPv6 지원, IPSec

**📢 섹션 요약 비유:** IPv6 보안 = 별도 구성!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: NDP 공격 방지

**미래 전망:** SEND 확대

**참고 표준:** RFC 4941, RFC 6105, RFC 6494

**📢 섹션 요약 비유:** IPv6 보안은 IPv4와 다릅니다!
