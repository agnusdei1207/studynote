# Container Registry - 컨테이너 레지스트리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컨테이너 이미지 저장소
> 2. **가치**: 버전 관리, 배포 자동화
> 3. **융합:** Docker, CI/CD, Security Scanning

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Container Registry는 컨테이너 이미지를 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Container Registry                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              레지스트리 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Developer                    Registry               │   │ │
│   │   │  ┌─────────┐                 ┌─────────────┐        │   │ │
│   │   │  │ Docker  │ ── push ──────→│   Images    │        │   │ │
│   │   │  │ Build   │                 │ ┌─────────┐ │        │   │ │
│   │   │  └─────────┘                 │ │ v1.0    │ │        │   │ │
│   │   │                              │ │ v1.1    │ │        │   │ │
│   │   │  Production                  │ │ latest  │ │        │   │ │
│   │   │  ┌─────────┐                 │ └─────────┘ │        │   │ │
│   │   │  │  K8s    │ ←── pull ──────│             │        │   │ │
│   │   │  │ Cluster │                 │ + Scanning  │        │   │ │
│   │   │  └─────────┘                 │ + Signing   │        │   │ │
│   │   │                              └─────────────┘        │   │ │
│   │   │                                                      │   │ │

│   │                                                              │ │

│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 레지스트리는 이미지의 중앙 저장소입니다.

**📢 섹션 요약 비유:** Container Registry = 앱 스토어!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Repository** | 저장소 |
| **Tag** | 태그 |
| **Manifest** | 매니페스트 |
| **Layer** | 레이어 |
| **Digest** | 다이제스트 |

**핵심 알고리즸:** 빌드 → 태그 → 푸시 → 저장 → 풀

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**레지스트리 종류**

| 레지스트리 | 특징 |
|:---|:---|
| **Docker Hub** | 공개, 무료/유료 |
| **ECR** | AWS 통합 |
| **GCR/Artifact Registry** | GCP 통합 |
| **Harbor** | 오픈소스, 사설 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 공개 → Docker Hub
**시나리오 2:** AWS → ECR
**시나리오 3:** 사설 → Harbor
