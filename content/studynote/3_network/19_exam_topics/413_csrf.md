# CSRF - 크로스 사이트 요청 위조

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 몰래 요청 전송
> 2. **가치**: 권한 도용, 무단 작업
> 3. **융합**: CSRF Token, SameSite, CORS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CSRF(Cross-Site Request Forgery)는 사용자 권한을 도용해 요청을 전송합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CSRF                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CSRF 공격 과정                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [공격자 사이트]                                    │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 자동 실행 스크립트                              │   │ │
│   │   │     │ <img src="bank.com/transfer?to=attacker">      │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [피해자] (은행 로그인 상태)                         │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 쿠키 자동 포함                                  │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [은행 서버]                                        │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 정상 요청으로 처리                              │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [송금 완료] ────▶ [공격자]                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CSRF 방어                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 방어       │ 설명     │ 효과     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ CSRF Token │ 난수검증 │ 높음     │               │   │ │
│   │   │  │ SameSite   │ 쿠키정책 │ 높음     │               │   │ │
│   │   │  | Referer    │ 출처확인 │ 중간     │               │   │ │
│   │   │  │ Double Submit│ 이중전송│ 높음    │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: CSRF는 사용자의 인증 상태를 악용합니다.

**💡 비유**: CSRF는 서명 도용입니다. 사용자 이름으로 문서를 보냅니다.

**📢 섹션 요약 비유:** CSRF = 권한 도용!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **CSRF Token** | 세션별 난수 토큰 |
| **SameSite** | 쿠키 속성 |
| **Referer Check** | 요청 출처 확인 |
| **Double Submit** | 이중 전송 검증 |

**비교표: SameSite 속성**

```
+------------+----------+----------+
| 항목       │ Strict   │ Lax      │
+------------+----------+----------+
| 외부요청   │ 차단     │ GET허용  │
| 링크이동   │ 차단     │ 허용     │
| 보안       │ 최고     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** 토큰 검증, 출처 확인

**📢 섹션 요약 비유:** CSRF Token = 필수!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 방어 계층**

| 항목 | Token | SameSite | Referer |
|:---|:---|:---|:---|
| **보안** │ 높음 | 높음 | 중간 |
| **호환성** │ 높음 | 높음 | 중간 |
| **구현** │ 쉬움 | 쉬움 | 쉬움 |

**📢 섹션 요약 비유:** 다층 방어 = CSRF 방지!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 상태 변경 요청 → CSRF Token
**시나리오 2:** 쿠키 기반 인증 → SameSite=Strict
**시나리오 3:** API → CORS + CSRF 검증

**도입 체크리스트:** 토큰, SameSite, 상태 변경 확인

**📢 섹션 요약 비유:** GET = 조회만!
