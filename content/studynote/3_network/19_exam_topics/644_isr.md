# ISR (Interrupt Service Routine) - 인터럽트 서비스 루틴

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터럽트 처리 함수
> 2. **가치**: 빠른 이벤트 응답
> 3. **융합:** Interrupt, Device Driver, RTOS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

ISR은 인터럽트 발생 시 실행되는 처리 루틴입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ISR                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ISR 실행 흐름                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Interrupt 발생                                      │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────┐            │   │ │
│   │   │  │ ISR 진입                            │            │   │ │
│   │   │  │ 1. Register 저장                   │            │   │ │
│   │   │  │ 2. 인터럽트 비활성화                │            │   │ │
│   │   │  │ 3. 빠른 처리 (Top Half)            │            │   │ │
│   │   │  │ 4. 인터럽트 활성화                  │            │   │ │
│   │   │  │ 5. Register 복원                   │            │   │ │
│   │   │  └─────────────────────────────────────┘            │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓ (필요시)                                     │   │ │
│   │   │  ┌─────────────────────────────────────┐            │   │ │
│   │   │  │ Bottom Half (지연 처리)             │            │   │ │
│   │   │  │ - Tasklet                          │            │   │ │
│   │   │  │ - Workqueue                        │            │   │ │
│   │   │  └─────────────────────────────────────┘            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ISR은 최대한 빠르게 실행되어야 합니다.

**📢 섹션 요약 비유:** ISR = 응급 처치!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Top Half** | 즉시 처리 |
| **Bottom Half** | 지연 처리 |
| **Context Save** | 컨텍스트 저장 |
| **IRET** | 인터럽트 복귀 |
| **Interrupt Mask** | 인터럽트 마스크 |

**핵심 알고리즸:** 진입 → 저장 → 처리 → 복원 → 복귀

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ISR 제약사항**

| 제약 | 이유 |
|:---|:---|
| **블로킹 금지** | 시스템 정지 방지 |
| **짧은 실행** | 다른 인터럽트 지연 방지 |
| **재진입 주의** | 데이터 무결성 |
