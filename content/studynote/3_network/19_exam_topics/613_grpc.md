# gRPC (Google Remote Procedure Call) - 구글 RPC

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고성능 RPC 프레임워크
> 2. **가치**: 효율적 마이크로서비스 통신
> 3. **융합:** HTTP/2, Protobuf, Streaming

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

gRPC는 Protocol Buffers와 HTTP/2를 사용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    gRPC                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              gRPC 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Client                    gRPC                    Server │   │ │
│   │   │  ┌────────┐               Stack                 ┌────────┐│   │ │
│   │   │  │Service │─── Stub ────→ HTTP/2 ────→ Stub ───→│Service ││   │ │
│   │   │  │  Stub  │               Protobuf              │ Impl   ││   │ │
│   │   │  └────────┘                                     └────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  통신 패턴:                                           │   │ │
│   │   │  - Unary: 1 Request → 1 Response                    │   │ │
│   │   │  - Server Streaming: 1 Req → N Res                  │   │ │
│   │   │  - Client Streaming: N Req → 1 Res                  │   │ │
│   │   │  - Bidirectional Streaming: N Req ↔ N Res           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: gRPC는 Protobuf로 직렬화하고 HTTP/2로 전송합니다.

**📢 섹션 요약 비유:** gRPC = 고속 원격 호출!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Protobuf** | 직렬화 포맷 |
| **HTTP/2** | 전송 프로토콜 |
| **Stub** | 클라이언트/서버 코드 |
| **Channel** | 연결 |

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**gRPC vs REST**

| 항목 | gRPC | REST |
|:---|:---|:---|
| **포맷** | Protobuf | JSON |
| **성능** | 높음 | 중간 |
| **스트리밍** | 지원 | 제한 |
| **브라우저** | 제한 | 지원 |
