# HSRP - 핫 스탠바이 라우터 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스코 FHRP
> 2. **가치**: 게이트웨이 이중화
> 3. **융합**: FHRP, GLBP와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HSRP(Hot Standby Router Protocol)는 시스코 전용 게이트웨이 이중화 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    HSRP                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HSRP 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Host]                                             │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ 기본 게이트웨이: 10.0.0.1 (VIP)              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           HSRP 그룹                          │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌───────────┐        ┌───────────┐         │    │   │ │
│   │   │  │  │  Active   │        │  Standby  │         │    │   │ │
│   │   │  │  │ Router    │        │ Router    │         │    │   │ │
│   │   │  │  │ Priority  │        │ Priority  │         │    │   │ │
│   │   │  │  │   110     │        │   100     │         │    │   │ │
│   │   │  │  │10.0.0.2   │        │10.0.0.3   │         │    │   │ │
│   │   │  │  │           │◀──────▶│           │         │    │   │ │
│   │   │  │  │ Hello     │  HSRP  │           │         │    │   │ │
│   │   │  │  │ (224.0.0.2)      │           │         │    │   │ │
│   │   │  │  └───────────┘        └───────────┘         │    │   │ │
│   │   │  │       │                                      │    │   │ │
│   │   │  │       │ VIP: 10.0.0.1                       │    │   │ │
│   │   │  │       ↓                                      │    │   │ │
│   │   │  │  [네트워크]                                  │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HSRP 상태                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Initial:                                         │   │ │
│   │   │     • 시작 상태                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Learn:                                           │   │ │
│   │   │     • VIP 대기                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Listen:                                          │   │ │
│   │   │     • Hello 수신                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  4. Speak:                                           │   │ │
│   │   │     • Active/Standby 선출 참여                      │   │ │
│   │   │                                                      │   │ │
│   │   │  5. Standby:                                         │   │ │
│   │   │     • 대기 라우터                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  6. Active:                                          │   │ │
│   │   │     • 활성 라우터                                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HSRP는 Active/Standby 구조로 게이트웨이 이중화합니다.

**💡 비유**: HSRP는 당직 시스템입니다. 당직자가 없으면 대기자가 승격합니다.

**📢 섹션 요약 비유:** HSRP = Cisco 게이트웨이 이중화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Active** | 활성 라우터 |
| **Standby** | 대기 라우터 |
| **Virtual IP** | 공유 IP |
| **Virtual MAC** | 0000.0C07.ACXX |

**비교표: HSRP 버전**

```
+------------+----------+----------+
| 항목       │ HSRP v1  │ HSRP v2  │
+------------+----------+----------+
| 그룹 번호  │ 0-255    │ 0-4095   │
| 멀티캐스트 │ 224.0.0.2│ 224.0.0.102│
| IPv6       │ 미지원   │ 지원     │
+------------+----------+----------+
```

**핵심 알고리즸:** 우선순위, Preempt

**📢 섹션 요약 비유:** Preempt = 선점!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FHRP**

| 프로토콜 | 벤더 | 로드밸런싱 |
|:---|:---|:---|
| **HSRP** | Cisco | 없음 |
| **VRRP** | 표준 | 없음 |
| **GLBP** | Cisco | 있음 |

**📢 섹션 요약 비유:** HSRP = 단순 이중화!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: Cisco 환경** → HSRP
**시나리오 2: 멀티벤더** → VRRP
**시나리오 3: 로드밸런싱** → GLBP

**도입 체크리스트:** 그룹 번호, Priority, Preempt

**📢 섹션 요약 비유:** Priority 100 = 기본값!
