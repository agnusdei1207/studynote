# Reverse DNS (rDNS) - 역방향 DNS

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP → 도메인 이름 변환
> 2. **가치**: 이메일 검증, 로깅
> 3. **융합:** PTR, in-addr.arpa, SPF

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Reverse DNS는 IP 주소로 도메인 이름을 찾습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Reverse DNS                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              역방향 DNS 조회                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  정방향: www.google.com → 142.250.x.x               │   │ │
│   │   │  역방향: 142.250.x.x → www.google.com               │   │ │
│   │   │                                                      │   │ │
│   │   │  PTR 레코드:                                         │   │ │
│   │   │  IP: 192.168.1.10                                    │   │ │
│   │   │  PTR: 10.1.168.192.in-addr.arpa → host.example.com   │   │ │
│   │   │                                                      │   │ │
│   │   │  조회 과정:                                          │   │ │
│   │   │  1. IP를 역순으로 변환                                │   │ │
│   │   │  2. in-addr.arpa 추가                                │   │ │
│   │   │  3. PTR 쿼리                                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: rDNS는 PTR 레코드와 in-addr.arpa 도메인을 사용합니다.

**📢 섹션 요약 비유:** Reverse DNS = 발신자 확인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **PTR** | 역방향 레코드 |
| **in-addr.arpa** | IPv4 역방향 도메인 |
| **ip6.arpa** | IPv6 역방향 도메인 |
| **Zone Delegation** | IP 대역 관리 |

**핵심 알고리즸:** IP 역순 → in-addr.arpa → PTR 조회

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**정방향 vs 역방향**

| 항목 | 정방향 | 역방향 |
|:---|:---|:---|
| **레코드** | A/AAAA | PTR |
| **도메인** | 일반 | in-addr.arpa |
| **용도** | 접속 | 검증 |
