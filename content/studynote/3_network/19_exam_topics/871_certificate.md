# Certificate - 인증서

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공개키 신원 증명
> 2. **가치**: 신뢰 체인
> 3. **융합:** X.509, PKI, CA

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Certificate는 공개키와 소유자 정보를 바인딩합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Certificate                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              X.509 인증서 구조                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  인증서 내용:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Version: 3                                      ││   │ │
│   │   │  │ Serial Number: 01:23:45:...                     ││   │ │
│   │   │  │ Issuer: CN=DigiCert CA                          ││   │ │
│   │   │  │ Validity: 2024-01-01 to 2025-01-01              ││   │ │
│   │   │  │ Subject: CN=example.com                         ││   │ │
│   │   │  │ Public Key: RSA 2048 bits                       ││   │ │
│   │   │  │ Extensions: SAN, Key Usage...                   ││   │ │
│   │   │  │ Signature: CA의 서명                             ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  신뢰 체인:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Root CA (자체 서명)                             ││   │ │
│   │   │  │       │                                         ││   │ │
│   │   │  │       ↓ 서명                                    ││   │ │
│   │   │  │  Intermediate CA                                ││   │ │
│   │   │  │       │                                         ││   │ │
│   │   │  │       ↓ 서명                                    ││   │ │
│   │   │  │  End Entity Certificate (서버/클라이언트)         ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  PKI (Public Key Infrastructure):                     │   │ │
│   │   │  - CA: 인증 기관                                       │   │ │
│   │   │  - RA: 등록 기관                                       │   │ │
│   │   │  - CRL/OCSP: 폐기 확인                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 인증서는 공개키의 신뢰성을 보장합니다.

**📢 섹션 요약 비유:** Certificate = 신분증!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Subject** | 소유자 |
| **Issuer** | 발급자 |
| **Public Key** | 공개키 |
| **Validity** | 유효기간 |
| **Signature** | 서명 |

**핵심 알고리즸:** CA 검증 → 서명 → 발급 → 검증

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**인증서 유형 비교**

| 유형 | 검증 | 용도 |
|:---|:---|:---|
| **DV** | 도메인 | 일반 |
| **OV** | 조직 | 기업 |
| **EV** | 확장 | 금융 |
| **Code Signing** | 코드 | 소프트웨어 |
