# Race Condition - 경쟁 상태

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동시 접근으로 인한 오류
> 2. **가치**: 동기화 필요성
> 3. **융합:** Critical Section, Atomic, Lock

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Race Condition은 동시 접근 시 결과가 순서에 따라 달라집니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Race Condition                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              경쟁 상태 발생                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Counter = 0                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  Thread A              Thread B                      │   │ │
│   │   │  read: 0               read: 0                       │   │ │
│   │   │  inc: 1                inc: 1                        │   │ │
│   │   │  write: 1              write: 1                       │   │ │
│   │   │                                                      │   │ │
│   │   │  결과: Counter = 1 (기대: 2)                         │   │ │
│   │   │                                                      │   │ │
│   │   │  해결: 동기화 (Mutex, Atomic)                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 경쟁 상태는 비원자적 연산으로 발생합니다.

**📢 섹션 요약 비유:** Race Condition = 동시 주문!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Read-Modify-Write** | 읽기-수정-쓰기 |
| **Atomic Operation** | 원자적 연산 |
| **Memory Barrier** | 메모리 장벽 |
| **Data Race** | 데이터 경쟁 |
| **TOCTOU** | Time-of-check to Time-of-use |

**핵심 알고리즸:** 공유 자원 → 동시 접근 → 불일치

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Race Condition 해결책**

| 해결책 | 설명 |
|:---|:---|
| **Mutex** | 상호 배제 |
| **Atomic** | 원자적 연산 |
| **Lock-free** | 락 프리 구조 |
| **Thread-local** | 스레드 로컬 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 카운터 → Atomic
**시나리오 2:** 복잡한 연산 → Mutex
**시나리오 3:** 성능 중요 → Lock-free
