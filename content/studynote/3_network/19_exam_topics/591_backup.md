# Backup - 백업

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 복사본 보관
> 2. **가치**: 데이터 손실 방지
> 3. **융합:** Full, Incremental, Differential, RTO/RPO

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Backup은 데이터를 복구 가능한 상태로 보관합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Backup                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Backup 유형                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Full Backup:                                        │   │ │
│   │   │  모든 데이터 복사                                    │   │ │
│   │   │  복구: 빠름 | 공간: 큼 | 시간: 김                    │   │ │
│   │   │                                                      │   │ │
│   │   │  Incremental Backup:                                 │   │ │
│   │   │  마지막 백업 이후 변경분만                            │   │ │
│   │   │  복구: 느림 | 공간: 작음 | 시간: 짧음                 │   │ │
│   │   │                                                      │   │ │
│   │   │  Differential Backup:                                │   │ │
│   │   │  마지막 Full 백업 이후 변경분                         │   │ │
│   │   │  복구: 중간 | 공간: 중간 | 시간: 중간                 │   │ │
│   │   │                                                      │   │ │
│   │   │  3-2-1 Rule:                                         │   │ │
│   │   │  3개 복사본, 2개 미디어, 1개 오프사이트               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Backup 전략은 RTO/RPO에 따라 결정됩니다.

**📢 섹션 요약 비유:** Backup = 보험!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RPO** | Recovery Point Objective |
| **RTO** | Recovery Time Objective |
| **Full** | 전체 |
| **Incremental** | 증분 |
| **Differential** | 차등 |

**핵심 알고리즸:** 백업 → 저장 → 복구

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Backup vs Archive**

| 항목 | Backup | Archive |
|:---|:---|:---|
| **목적** | 복구 | 보관 |
| **빈도** | 정기 | 일회 |
| **접근** | 빠름 | 느림 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일일 → Incremental
**시나리오 2:** 주간 → Full
**시나리오 3:** 장기 → Archive
