# Reassembly - 재조립

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단편 결합
> 2. **가치**: 원본 패킷 복원
> 3. **융합:** IP Reassembly, Timeout, Buffer

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Reassembly는 분할된 단편을 원래대로 결합합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Reassembly                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              재조립 과정                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  수신된 단편들:                                        │   │ │
│   │   │  ┌────────────────────────────────────────────────┐ │   │ │
│   │   │  │ Fragment 1: ID=X, Offset=0, MF=1              │ │   │ │
│   │   │  │ Fragment 2: ID=X, Offset=185, MF=1  ← 도착     │ │   │ │
│   │   │  │ Fragment 3: ID=X, Offset=370, MF=0  ← 대기중   │ │   │ │
│   │   │  └────────────────────────────────────────────────┘ │   │ │
│   │   │                      ↓                               │   │ │
│   │   │              재조립 버퍼                              │   │ │
│   │   │  ┌────────────────────────────────────────────────┐ │   │ │
│   │   │  │ [Frag 1] [Frag 2] [    대기    ]              │ │   │ │
│   │   │  │    ↓        ↓          ↓                      │ │   │ │
│   │   │  │  Offset    Offset    Offset                   │ │   │ │
│   │   │  │    0        185        370                    │ │   │ │
│   │   │  └────────────────────────────────────────────────┘ │   │ │
│   │   │                      ↓                               │   │ │
│   │   │              타임아웃 내 모두 도착                    │   │ │
│   │   │                      ↓                               │   │ │
│   │   │  재조립된 패킷:                                        │   │ │
│   │   │  ┌────────────────────────────────────────────────┐ │   │ │
│   │   │  │ IP Header │ Complete Data                      │ │   │ │
│   │   │  └────────────────────────────────────────────────┘ │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 재조립은 최종 목적지에서만 수행됩니다.

**📢 섹션 요약 비유:** Reassembly = 퍼즐 맞추기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Buffer** | 재조립 버퍼 |
| **Timeout** | 타임아웃 |
| **ID** | 식별자 |
| **Offset** | 오프셋 |
| **MF** | More Fragments |

**핵심 알고리즸:** 단편 수집 → 정렬 → 결합 → 완료

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**재조립 특성**

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
