# Dual Stack - IPv4/IPv6 이중 스택

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPv4와 IPv6 동시 지원

> 3. **융합**: 전환 기술의 기준

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Dual Stack은 네트워크 장비와 호스트가 IPv4와 IPv6를 동시에 지원하는 전환 기술입니다.

> **해설**: Dual Stack은 가장 자연스러운 IPv6 전환 방식입니다.

**💡 비유**: Dual Stack은 두 언어를 동시에 할 수 있는 것입니다.

**📢 섹션 요약 비유:** Dual Stack = 권장 방식!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | IPv4 | IPv6 |
|:---|:---|:---|
| **주소** | 있음 | 있음 |
| **라우팅** | 독립 | 독립 |
| **DNS** | A | AAAA |
| **애플리케이션** | 지원 | 지원 |

**비교표: 우선순위 (Happy Eyeballs)**

```
+------------+----------+----------+
| 상황       │ 우선     │ 이유     │
+------------+----------+----------+
| IPv6 지원  │ IPv6     │ 미래     │
| IPv6만     │ IPv6     │ 유일     │
| IPv4만     │ IPv4     │ 유일     │
| 둘 다      │ IPv6     │ Happy EB │
+------------+----------+----------+
```

**핵심 알고리즸:** Happy Eyeballs (RFC 8305)

**📢 섹션 요약 비유:** Happy Eyeballs = 빠른 연결!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: ISP** → Dual Stack
**시나리오 2: 기업** → Dual Stack
**시나리오 3: 데이터센터** → IPv6 Only

**도입 체크리스트:** 장비 지원, 보안, 주소 계획

**📢 섹션 요약 비유:** 모든 장비 = Dual Stack 지원 확인!
