# 라우팅 프로토콜 (Routing Protocols) - RIP/OSPF/EIGRP/BGP

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 경로 정보 교환
> 2. **가치**: 자동 경로 학습, 장애 복구
> 3. **융합**: IGP/EGP로 분류

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

라우팅 프로토콜은 라우터 간 경로 정보를 교환하여 라우팅 테이블을 자동으로 구성하는 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    라우팅 프로토콜                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              라우팅 프로토콜 분류                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IGP (Interior Gateway Protocol):                      │   │ │
│   │   │  • RIP: Distance Vector, Hop 수 기반                   │   │ │
│   │   │  • OSPF: Link State, Dijkstra SPF                      │   │ │
│   │   │  • EIGRP: Advanced Distance Vector, Cisco              │   │ │
│   │   │  • IS-IS: Link State, ISP                              │   │ │
│   │   │                                                      │   │ │
│   │   │  EGP (Exterior Gateway Protocol):                       │   │ │
│   │   │  • BGP: Path Vector, AS 간                             │   │ │
│   │   │                                                      │   │ │
│   │   │  분류:                                                 │   │ │
│   │   │  • Distance Vector: RIP, EIGRP                         │   │ │
│   │   │  • Link State: OSPF, IS-IS                             │   │ │
│   │   │  • Path Vector: BGP                                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              프로토콜 비교                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  프로토콜 │ 알고리즸 │ AD  │ 메트릭   │ 용도         │   │ │
│   │   │  ────────┼─────────┼─────┼──────────┼────────────│   │ │
│   │   │  RIP     │ DV      │ 120 │ Hop      │ 소규모       │   │ │
│   │   │  OSPF    │ LS      │ 110 │ Cost     │ 기업        │   │ │
│   │   │  EIGRP   │ Adv DV  │ 90  │ 복합     │ Cisco       │   │ │
│   │   │  IS-IS   │ LS      │ 115 │ Cost     │ ISP        │   │ │
│   │   │  BGP     │ PV      │ 20  │ AS_PATH  │ 인터넷      │   │ │
│   │   │                                                      │   │ │
│   │   │  DV = Distance Vector, LS = Link State               │   │ │
│   │   │  PV = Path Vector                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IGP는 내부용, BGP는 AS 간 연결에 사용합니다.

**💡 비유**: 라우팅 프로토콜은 라우터들이 서로 지도를 공유하는 것입니다.

**📢 섹션 요약 비유:** OSPF = 기업 표준, BGP = 인터넷!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 프로토콜 | 특징 | 컨버전스 |
|:---|:---|:---|
| **RIP** | 15홉 제한, 30초 업데이트 | 느림 |
| **OSPF** | 영역 분할, SPF | 빠름 |
| **EIGRP** | DUAL, 부분 업데이트 | 매우 빠름 |
| **BGP** | AS_PATH, 정책 | 중간 |

**비교표**

```
+------------+----------+----------+
| 항목       │ OSPF     │ BGP      │
+------------+----------+----------+
| 범위       │ IGP      │ EGP      │
│ 스케일     │ 수천     │ 수만     │
│ 정책       │ 제한     │ 강력     │
│ TCP/UDP    │ IP 89    │ TCP 179  │
+------------+----------+----------+
```

**핵심 알고리즸:** Bellman-Ford, Dijkstra, DUAL

**📢 섹션 요약 비유:** OSPF는 빠르고, BGP는 스마트!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: OSPF 영역**

| 영역 | 역할 | LSA |
|:---|:---|:---|
| **Backbone** | Area 0 | 모든 LSA |
| **Standard** | 일반 | Type 1-4 |
| **Stub** | 외부 없음 | Type 3 |
| **NSSA** | 일부 외부 | Type 7 |

**📢 섹션 요약 비유:** Area 0 = Backbone 필수!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 소규모** → RIP (레거시)
**시나리오 2: 기업** → OSPF
**시나리오 3: ISP** → IS-IS + BGP
**시나리오 4: 데이터센터** → BGP (RFC 7938)

**도입 체크리스트:** 규모, 정책 요구, 컨버전스

**📢 섹션 요약 비유:** 기업은 OSPF, 인터넷은 BGP!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 자동 경로 학습, 빠른 복구

**미래 전망:** Segment Routing (SR-MPLS, SRv6)

**참고 표준:** RFC 2328 (OSPF), RFC 4271 (BGP)

**📢 섹션 요약 비유:** 라우팅 프로토콜은 네트워크의 지도!
