# MSS - 최대 세그먼트 크기

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: TCP 페이로드 최대 크기
> 2. **가치**: 단편화 방지
> 3. **융합:** TCP, MTU, SYN

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MSS (Maximum Segment Size)는 TCP 데이터의 최대 크기입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MSS (Maximum Segment Size)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MSS 계산                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  공식:                                                │   │ │
│   │   │  MSS = MTU - IP Header - TCP Header                   │   │ │
│   │   │  MSS = 1500 - 20 - 20 = 1460 bytes                    │   │ │
│   │   │                                                      │   │ │
│   │   │  IP Packet 구조:                                       │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ IP Header (20B) │ TCP Header (20B) │ Data (MSS) ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │  ←────────────── MTU (1500B) ──────────────────→     │   │ │
│   │   │                                                      │   │ │
│   │   │  TCP 3-way Handshake에서 협상:                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Client → Server: SYN, MSS=1460                  ││   │ │
│   │   │  │ Server → Client: SYN-ACK, MSS=1460              ││   │ │
│   │   │  │ Client → Server: ACK                            ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  MSS 옵션:                                            │   │ │
│   │   │  - TCP Options (Kind=2, Length=4)                    │   │ │
│   │   │  - SYN 세그먼트에서만 전송                             │   │ │
│   │   │  - 양방향 다를 수 있음                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: MSS는 IP 단편화를 방지합니다.

**📢 섹션 요약 비유:** MSS = 택배 내용물 크기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **MSS** | 최대 세그먼트 크기 |
| **MTU** | 최대 전송 단위 |
| **TCP Options** | TCP 옵션 |
| **SYN** | 동기화 플래그 |
| **Negotiation** | 협상 |

**핵심 알고리즸:** MSS = MTU - 40 (IPv4 + TCP)

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**MSS vs MTU**

| 항목 | MTU | MSS |
|:---|:---|:---|
| **계층** | L2/L3 | L4 |
| **포함** | 헤더 포함 | 페이로드만 |
| **크기** | 1500 | 1460 |
| **기준** | 프레임 | 세그먼트 |
