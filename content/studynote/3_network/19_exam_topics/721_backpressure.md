# Backpressure - 백프레셔

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 역압력 흐름 제어
> 2. **가치**: 과부하 방지, 안정성
> 3. **융합:** Reactive Streams, Flow Control, Throttling

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Backpressure는 소비자가 생산자에게 압력을 전달합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Backpressure                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              백프레셔 흐름                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Producer                  Consumer                  │   │ │
│   │   │  ┌─────────┐              ┌─────────┐               │   │ │
│   │   │  │ 빠름    │ ── 데이터 ──→│ 느림    │               │   │ │
│   │   │  │ (100/s) │              │ (10/s)  │               │   │ │
│   │   │  └─────────┘              └─────────┘               │   │ │
│   │   │       ↑                        │                     │   │ │
│   │   │       │     ← 백프레셔 ←       │                     │   │ │
│   │   │       │    (속도 조절 요청)    │                     │   │ │
│   │   │       └────────────────────────┘                     │   │ │
│   │   │                                                      │   │ │
│   │   │  해결 방법:                                           │   │ │
│   │   │  - Buffer: 버퍼링                                     │   │ │
│   │   │  - Drop: 폐기                                        │   │ │
│   │   │  - Block: 차단                                       │   │ │
│   │   │  - Sample: 샘플링                                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 백프레셔는 소비자 속도에 맞춰 생산자를 조절합니다.

**📢 섹션 요약 비유:** Backpressure = 수도꼭지 조절!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Publisher** | 생산자 |
| **Subscriber** | 소비자 |
| **Buffer** | 버퍼 |
| **Request(n)** | n개 요청 |
| **Cancel** | 취소 |

**핵심 알고리즸:** 소비자 요청 → 생산자 생산 → 흐름 제어

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**전략 비교**

| 전략 | 설명 |
|:---|:---|
