# Redundancy - 이중화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 예비 자원 확보
> 2. **가치**: 단일 실패점 제거
> 3. **융합:** N+1, Active-Active, Geographic

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Redundancy는 장애 대비 예비 자원을 확보합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Redundancy                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              이중화 유형                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  이중화 계층:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  1. Network Redundancy:                         ││   │ │
│   │   │  │     이중 경로, 이중 NIC, 이중 스위치               ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  2. Server Redundancy:                          ││   │ │
│   │   │  │     다중 서버, 클러스터링                         ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  3. Storage Redundancy:                         ││   │ │
│   │   │  │     RAID, 이중 스토리지                           ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  4. Power Redundancy:                           ││   │ │
│   │   │  │     이중 전원, UPS, 발전기                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  5. Geographic Redundancy:                      ││   │ │
│   │   │  │     다중 데이터센터, DR 사이트                    ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 이중화는 SPOF (Single Point of Failure)를 제거합니다.

**📢 섹션 요약 비유:** Redundancy = 예비 타이어!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **N+1** | N개 + 예비 1개 |
| **2N** | 완전 이중화 |
| **SPOF** | 단일 실패점 |
| **Hot Spare** | 즉시 전환 |
| **Cold Spare** | 수동 전환 |

**핵심 알고리듬:** 장애 감지 → 예비 자원 전환

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**이중화 수준 비교**

| 수준 | 구성 | 가용성 |
|:---|:---|:---|
| **N+1** | 예비 1대 | 중간 |
| **2N** | 완전 이중 | 높음 |
| **2N+1** | 이중+예비 | 매우 높음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹서버 → N+1
**시나리오 2:** 데이터베이스 → 2N
**시나리오 3:** 핵심 서비스 → 2N+1
