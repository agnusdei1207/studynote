# Disaster Recovery - 재해 복구

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 재해 후 서비스 복구
> 2. **가치**: 비즈니스 연속성
> 3. **융합:** Backup, Replication, RTO/RPO

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Disaster Recovery는 재해 후 서비스를 복구합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Disaster Recovery                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DR 지표                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RTO (Recovery Time Objective):                      │   │ │
│   │   │  복구 목표 시간 - 얼마나 빨리 복구?                       │   │ │
│   │   │                                                      │   │ │
│   │   │  RPO (Recovery Point Objective):                      │   │ │
│   │   │  복구 시점 목표 - 얼마나 최신 데이터?                     │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ RTO/RPO  │ Hot Site │ Warm Site │ Cold Site    ││   │ │
│   │   │  ├─────────────────────────────────────────────────┤│   │ │
│   │   │  │ RTO      │ < 1시간   │ < 24시간  │ > 24시간     ││   │ │
│   │   │  │ RPO      │ 실시간    │ < 24시간  │ > 24시간     ││   │ │
│   │   │  │ 비용      │ 높음      │ 중간      │ 낮음         ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DR은 RTO/RPO를 목표로 복구 계획을 수립합니다.

**📢 섹션 요약 비유:** DR = 보험!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RTO** | 복구 시간 목표 |
| **RPO** | 복구 시점 목표 |
| **Hot Site** | 핫 사이트 |
| **Warm Site** | 웜 사이트 |
| **Cold Site** | 콜드 사이트 |

**핵심 알고리즸:** 재해 → 선언 → 복구 → 서비스 재개
