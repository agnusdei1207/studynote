# Loopback - 루프백

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자기 자신과 통신
> 2. **가치**: 로컬 테스트
> 3. **융합:** 127.0.0.1, localhost, Internal

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Loopback은 자신의 컴퓨터 내부 통신입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Loopback                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              루프백 주소                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  루프백 주소 범위:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  IPv4: 127.0.0.0/8                              ││   │ │
│   │   │  │    127.0.0.1 (localhost) - 가장 많이 사용          ││   │ │
│   │   │  │    127.0.0.2, 127.0.0.3 ... 사용 가능              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  IPv6: ::1                                      ││   │ │
│   │   │  │    0000:0000:0000:0000:0000:0000:0000:0001       ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  용도:                                                  │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │  1. 서버 테스트                                   ││   │ │
│   │   │  │     http://localhost:8080                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  2. 데이터베이스 연결                              ││   │ │
│   │   │  │     mysql://localhost:3306                       ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  3. API 테스트                                   ││   │ │
│   │   │  │     http://127.0.0.1:3000/api                    ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                  │   │ │
│   │   │  - 네트워크 인터페이스 거치지 않음                       │   │ │
│   │   │  - 외부로 전송되지 않음                                  │   │ │
│   │   │  - 항상 사용 가능                                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 루프백은 로컬 개발과 테스트의 필수입니다.

**📢 섹션 요약 비유:** Loopback = 자기 자신과 대화!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **127.0.0.1** | IPv4 루프백 |
| **::1** | IPv6 루프백 |
| **localhost** | 호스트명 |
| **lo0** | 인터페이스 |
| **Internal** | 내부 |

**핵심 알고리듬:** 패킷 → 커널 → 바로 반환

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**루프백 vs 실제 IP**

| 항목 | 루프백 | 실제 IP |
|:---|:---|:---|
| **대상** | 자신 | 다른 호스트 |
| **네트워크** | 불필요 | 필요 |
| **속도** | 매우 빠름 | 느림 |
| **용도** | 테스트 | 통신 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 개발 → localhost
**시나리오 2:** DB → 127.0.0.1
**시나리오 3:** 서비스 바인딩 → 127.0.0.1 (내부만)
