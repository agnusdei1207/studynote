# WRED (가중치 무작위 조기 감지) - Weighted Random Early Detection

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 혼잡 예방을 위한 선제적 패킷 폐기
> 2. **가치**: 테일 드롭 방지, TCP 동기화 방지
> 3. **융합**: RED, QoS, AQM

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

WRED는 혼잡 전에 패킷을 무작위로 폐기합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WRED 동작                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              WRED 동작 곡선                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  폐기 확률                                           │   │ │
│   │   │    100% │                           ●               │   │ │
│   │   │         │                        ╱                  │   │ │
│   │   │         │                     ╱                     │   │ │
│   │   │      0% ├──────────────────●────────────────────   │   │ │
│   │   │         │    min      avg      max                 │   │ │
│   │   │         └────────────────────────────────────────── │   │ │
│   │   │                   큐 깊이                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: WRED는 큐 깊이에 따라 폐기 확률을 조절합니다.

**📢 섹션 요약 비유:** WRED = 선제적 폐기!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Min Threshold** | 최소 임계값 |
| **Max Threshold** | 최대 임계값 |
| **MPD** | Mark Probability Denominator |
| **Weight** | IP Precedence 가중치 |

**핵심 알고리즸:** 확률적 폐기

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**RED vs WRED**

| 항목 | RED | WRED |
|:---|:---|:---|
| **가중치** | 없음 | IP Precedence |
| **클래스** | 단일 | 다중 |
| **복잡성** | 낮음 | 중간 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 2:** 우선순위 → WRED per-class
**시나리오 3:** 실시간 → 별도 큐
