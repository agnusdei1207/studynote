# DAMA (Demand Assignment Multiple Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 수요 기반 동적 자원 할당
> 2. **가치**: 효율적 대역폭 활용, 온디맨드
> 3. **융합**: 위성, VSAT와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DAMA(Demand Assignment Multiple Access)는 통신 수요가 있을 때만 채널을 동적으로 할당하는 다중 접속 방식입니다.

> **해설**: DAMA는 위성 통신, VSAT 네트워크에서 효율적인 대역폭 활용을 위해 사용됩니다.

**💡 비유**: DAMA는 호텔 방을 예약할 때만 체크인하는 것입니다.

**📢 섹션 요약 비유**: 필요할 때만 할당!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **요청 채널** | 할당 요청 | 제어 신호 | 예약 |
| **할당표** | 자원 관리 | 스케줄 | 프론트 |
| **데이터 채널** | 실제 전송 | 할당됨 | 객실 |

**비교표**

```
+------------+----------+----------+
| 항목       | PAMA     | DAMA     |
+------------+----------+----------+
| 할당       | 고정     | 동적     |
| 효율       | 낮음     | 높음     |
| 지연       | 없음     | 있음     |
| 비용       | 높음     | 낮음     |
+------------+----------+----------+
```

**핵심 알고리즸**: 요청 → 할당 → 사용 → 반납

**📢 섹션 요약 비유**: 위성 = DAMA!
