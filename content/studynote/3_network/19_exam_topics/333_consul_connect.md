# Consul Connect - 서비스 메시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: HashiCorp 서비스 메시 플랫폼
> 2. **가치**: 서비스 디스커버리, 서비스 메시
> 3. **융합**: Nomad, Kubernetes와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Consul Connect는 서비스 디스커버리와 서비스 메시를 제공하는 플랫폼입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Consul Connect                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Consul 아키텍처                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Services]                                         │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Service Mesh (Connect)                         │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           Consul Agents                     │    │   │ │
│   │   │  │  • Client Agent                            │    │   │ │
│   │   │  │  • Sidecar Proxy (Envoy)                   │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │

│   │   │  ┌─────────────────────────────────────────────┐    │   │ │

│   │   │  │  • Raft 합의                                 │    │   │ │
│   │   │  │  • 서비스 디스커버리                          │    │   │ │
│   │   │  │  • KV 스토어                                 │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │

│   │   │  │ Health     │ 헬스체크 │ 가용성   │               │   │ │
│   │   │  | Connect    │ 메시     │ 보안     │               │   │ │
│   │   │  │ KV Store   │ 설정     │ 구성     │               │   │ │
```

> **해설**: Consul Connect는 서비스 디스커버리와 메시를 통합합니다.

**💡 비유**: Consul은 서비스의 전화번호부이자 교통경찰입니다.

**📢 섹션 요약 비유:** Consul = 디스커버리 + 메시!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Server** | 서버 노드 |
| **Client** | 클라이언트 에이전트 |
| **Connect** | 서비스 메시 |
| **KV Store** | 키-값 저장소 |

**비교표: Consul vs 다른 메시**

```
+------------+----------+----------+
| 항목       │ Consul   │ Istio    │
+------------+----------+----------+
| 디스커버리 │ 내장     │ K8s      │
| 런타임     │ 범용     │ K8s      │
| 복잡도     │ 중간     │ 높음     │
| Nomad      │ 강함     │ 약함     │
+------------+----------+----------+
```

**핵심 알고리즸:** Gossip, Raft

**📢 섹션 요약 비유:** Gossip = 분산 동기화!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 디스커버리**

| 항목 | Consul | Eureka | Zookeeper |
|:---|:---|:---|:---|
| **CP/AP** | CP | AP | CP |
| **메시** | 지원 | 없음 | 없음 |
| **KV** | 지원 | 없음 | 지원 |

**📢 섹션 요약 비유:** 범용 = Consul!
