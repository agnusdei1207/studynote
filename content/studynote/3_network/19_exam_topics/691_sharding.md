# Sharding - 샤딩

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 수평 분할
> 2. **가치**: 확장성
> 3. **융합:** Partitioning, Scale-out, Database

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Sharding은 데이터를 여러 서버에 분할 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Sharding                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              샤딩 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Before Sharding:                                    │   │ │
│   │   │  ┌───────────────────────────────────────────────┐  │   │ │
│   │   │  │              Single Database                  │  │   │ │
│   │   │  │  User 1-1000000                               │  │   │ │
│   │   │  └───────────────────────────────────────────────┘  │   │ │
│   │   │                                                      │   │ │
│   │   │  After Sharding:                                     │   │ │
│   │   │  ┌───────────┐ ┌───────────┐ ┌───────────┐          │   │ │
│   │   │  │  Shard 1  │ │  Shard 2  │ │  Shard 3  │          │   │ │
│   │   │  │ Users     │ │ Users     │ │ Users     │          │   │ │
│   │   │  │ 1-333333  │ │333334-    │ │666667-    │          │   │ │
│   │   │  │           │ │666666     │ │1000000    │          │   │ │
│   │   │  └───────────┘ └───────────┘ └───────────┘          │   │ │
│   │   │                                                      │   │ │
│   │   │  샤딩 키: user_id (Range, Hash, Directory)           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 샤딩은 수평 확장을 가능하게 합니다.

**📢 섹션 요약 비유:** Sharding = 데이터 쪼개기!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Shard Key** | 샤딩 키 |
| **Range** | 범위 기반 |
| **Hash** | 해시 기반 |
| **Directory** | 디렉토리 기반 |
| **Rebalancing** | 재분배 |

**핵심 알고리즸:** 키 → 샤드 결정 → 해당 샤드 접근

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**샤딩 전략**

| 전략 | 장점 | 단점 |
|:---|:---|:---|
| **Range** | 범위 쿼리 | 핫스팟 |
| **Hash** | 균등 분배 | 범위 쿼리 어려움 |
| **Directory** | 유연함 | 오버헤드 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 대용량 DB → Sharding
**시나리오 2:** 지역 기반 → Range Sharding
**시나리오 3:** 균등 분배 → Hash Sharding
