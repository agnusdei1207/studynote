# MAC Address - MAC 주소

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: L2 하드웨어 주소
> 2. **가치**: NIC 고유 식별
> 3. **융합:** OUI, Burned-In, EUI-48/64

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

MAC Address는 네트워크 인터페이스의 물리적 주소입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MAC Address                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MAC 주소 구조                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  MAC 주소 형식:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  48비트 (6바이트)                                 ││   │ │
│   │   │  │  AA:BB:CC:DD:EE:FF                              ││   │ │
│   │   │  │  └────┬────┘└────┬────┘                          ││   │ │
│   │   │  │       │           │                              ││   │ │
│   │   │  │    OUI (24비트)  NIC ID (24비트)                  ││   │ │
│   │   │  │    제조사 식별    일련번호                         ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  주소 유형:                                              │   │ │
│   │   │  - Unicast: 첫 바이트 최하위 비트 0                     │   │ │
│   │   │  - Multicast: 첫 바이트 최하위 비트 1                   │   │ │
│   │   │  - Broadcast: FF:FF:FF:FF:FF:FF                       │   │ │
│   │   │                                                      │   │ │
│   │   │  표기법:                                                  │   │ │
│   │   │  - AA:BB:CC:DD:EE:FF (콜론)                            │   │ │
│   │   │  - AA-BB-CC-DD-EE-FF (하이픈)                          │   │ │
│   │   │  - AABB.CCDD.EEFF (점)                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: MAC 주소는 L2 통신의 핵심입니다.

**📢 섹션 요약 비유:** MAC Address = 주민등록번호!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **OUI** | 조직 고유 식별자 |
| **BIA** | 기본 MAC |
| **LAA** | 로컬 관리 주소 |
| **EUI-48** | 48비트 |
| **EUI-64** | 64비트 |

**핵심 알고리듬:** 제조사 할당 → NIC 저장 → 프레임에 사용

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**MAC vs IP 주소**

| 항목 | MAC | IP |
|:---|:---|:---|
| **계층** | L2 | L3 |
| **크기** | 48비트 | 32/128비트 |
| **변경** | 어려움 | 가능 |
| **범위** | 로컬 | 전역 |
