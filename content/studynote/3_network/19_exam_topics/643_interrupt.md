# Interrupt - 인터럽트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비동기 이벤트 알림
> 2. **가치**: 효율적 CPU 활용
> 3. **융합:** ISR, Exception, Signal

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Interrupt는 CPU에 비동기 이벤트를 알립니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Interrupt                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인터럽트 처리                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────┐                                        │   │ │
│   │   │  │  CPU    │ 실행 중                                 │   │ │
│   │   │  └────┬────┘                                        │   │ │
│   │   │       │                                             │   │ │
│   │   │       │ ← Interrupt                                 │   │ │
│   │   │       ↓                                             │   │ │
│   │   │  ┌─────────────────────────────────────┐            │   │ │
│   │   │  │ 1. Context 저장                     │            │   │ │
│   │   │  │ 2. ISR (Interrupt Service Routine)  │            │   │ │
│   │   │  │ 3. Context 복원                     │            │   │ │
│   │   │  │ 4. 실행 재개                        │            │   │ │
│   │   │  └─────────────────────────────────────┘            │   │ │
│   │   │       │                                             │   │ │
│   │   │       ↓                                             │   │ │
│   │   │  ┌─────────┐                                        │   │ │
│   │   │  │  CPU    │ 실행 재개                               │   │ │
│   │   │  └─────────┘                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 인터럽트는 폴링 없이 이벤트를 처리합니다.

**📢 섹션 요약 비유:** Interrupt = 알림!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **IRQ** | Interrupt Request |
| **ISR** | Interrupt Service Routine |
| **IVT/IDT** | 인터럽트 벡터 테이블 |
| **Priority** | 우선순위 |
| **Nesting** | 중첩 인터럽트 |

**핵심 알고리즸:** 인터럽트 → 컨텍스트 저장 → ISR → 컨텍스트 복원

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**인터럽트 종류**

| 종류 | 설명 |
|:---|:---|
| **Hardware** | 하드웨어 이벤트 |
| **Software** | 시스템 콜 |
| **Exception** | 오류 상황 |
| **Trap** | 의도적 전환 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** I/O 완료 → Hardware Interrupt
**시나리오 2:** 시스템 콜 → Software Interrupt
**시나리오 3:** 페이지 폴트 → Exception
