# SAN (Storage Area Network) - 스토리지 영역 네트워크

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전용 스토리지 네트워크
> 2. **가치**: 블록 수준 고성능 접근
> 3. **융합:** FC, iSCSI, FCoE

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SAN은 전용 네트워크로 스토리지에 접근합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SAN                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SAN 구성                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Server              SAN Switch           Storage    │   │ │
│   │   │  ┌─────┐              ┌─────┐             ┌─────┐   │   │ │
│   │   │  │ HBA │─────────────→│ FC  │────────────→│ LUN │   │   │ │
│   │   │  └─────┘              │Switch│             └─────┘   │   │ │
│   │   │                       └─────┘                        │   │ │
│   │   │  프로토콜:                                           │   │ │
│   │   │  - Fibre Channel (FC)                                │   │ │
│   │   │  - iSCSI (IP 기반)                                   │   │ │
│   │   │  - FCoE (FC over Ethernet)                           │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                               │   │ │
│   │   │  - 블록 수준 접근                                    │   │ │
│   │   │  - 고성능, 저지연                                    │   │ │
│   │   │  - 서버와 스토리지 분리                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SAN은 블록 수준으로 고성능 스토리지 접근을 제공합니다.

**📢 섹션 요약 비유:** SAN = 전용 도로!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **HBA** | Host Bus Adapter |
| **FC Switch** | 파이버 채널 스위치 |
| **LUN** | Logical Unit Number |
| **WWN** | World Wide Name |

**핵심 알고리즸:** I/O → HBA → Switch → Storage

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SAN 프로토콜**

| 프로토콜 | 속도 | 비용 |
|:---|:---|:---|
| **FC** | 높음 | 높음 |
| **iSCSI** | 중간 | 낮음 |
| **FCoE** | 높음 | 중간 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** DB → FC SAN
**시나리오 2:** VM → iSCSI
**시나리오 3:** 통합 → FCoE
