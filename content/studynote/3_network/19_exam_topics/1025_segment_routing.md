# Segment Routing - 세그먼트 라우팅

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소스 라우팅 기반
> 2. **가치**: MPLS 간소화
> 3. **융합:** SR-MPLS, SRv6, SID

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Segment Routing은 소스에서 경로를 지정하는 라우팅 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Segment Routing                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Segment Routing 구조                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  세그먼트 리스트:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Source ──► [SID1][SID2][SID3] ──► Destination  ││   │ │
│   │   │  │            │     │     │                        ││   │ │
│   │   │  │            ▼     ▼     ▼                        ││   │ │
│   │   │  │          ┌───┐┌───┐┌───┐                        ││   │ │
│   │   │  │          │N1 ││N2 ││N3 │                        ││   │ │
│   │   │  │          └───┘└───┘└───┘                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  SID (Segment Identifier):                       ││   │ │
│   │   │  │  - Node SID: 노드 식별                            ││   │ │
│   │   │  │  - Adjacency SID: 인접 링크                        ││   │ │
│   │   │  │  - Service SID: 서비스 체인                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  SR 유형:                                          ││   │ │
│   │   │  │  - SR-MPLS: MPLS 라벨 사용                         ││   │ │
│   │   │  │  - SRv6: IPv6 확장 헤더 사용                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Segment Routing은 LDP/RSVP 없이 MPLS를 구현합니다.

**📢 섹션 요약 비유:** Segment Routing = GPS 경로 안내!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **SID** | Segment Identifier |
| **Segment List** | 세그먼트 목록 |
| **SRGB** | Segment Routing Global Block |
| **SR-MPLS** | MPLS 기반 SR |
| **SRv6** | IPv6 기반 SR |

**핵심 알고리듬:** 경로 계산 → SID 리스트 생성 → 패킷 캡슐화 → 순차 처리

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SR vs 기존 MPLS**

| 항목 | Segment Routing | 기존 MPLS |
|:---|:---|:---|
| **프로토콜** | IGP 확장 | LDP/RSVP |
| **상태** | 소스만 | 전체 경로 |
| **복잡도** | 낮음 | 높음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 데이터센터 → SR-MPLS
**시나리오 2:** 5G → SRv6
**시나리오 3:** TE → SR-TE
