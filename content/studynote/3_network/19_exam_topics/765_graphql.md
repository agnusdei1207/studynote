# GraphQL - 그래프큐엘

> 3. **융합:** API Design, Schema, Type System

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

GraphQL은 데이터 쿼리 언어입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    GraphQL                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              GraphQL 구조                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Query (조회):                                       │   │ │
│   │   │  query {                                            │   │ │
│   │   │    user(id: 1) {                                    │   │ │
│   │   │      name                                           │   │ │
│   │   │      email                                          │   │ │
│   │   │      posts { title }                                │   │ │
│   │   │    }                                                │   │ │
│   │   │  }                                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Mutation (변경):                                    │   │ │
│   │   │  mutation {                                         │   │ │
│   │   │    createUser(input: {name: "Kim"}) {               │   │ │
│   │   │      id                                             │   │ │
│   │   │      name                                           │   │ │
│   │   │    }                                                │   │ │
│   │   │  }                                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Schema:                                             │   │ │
│   │   │  type User {                                        │   │ │
│   │   │    id: ID!                                          │   │ │
│   │   │    name: String!                                    │   │ │
│   │   │    posts: [Post!]!                                  │   │ │
│   │   │  }                                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: GraphQL은 클라이언트가 필요한 데이터만 요청합니다.

**📢 섹션 요약 비유:** GraphQL = 뷔페!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Schema** | 스키마 |
| **Query** | 조회 |
| **Mutation** | 변경 |
| **Resolver** | 리졸버 |
| **Subscription** | 구독 |

**핵심 알고리즸:** 쿼리 → 파싱 → 검증 → 실행 → 응답

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**GraphQL vs REST**

| 항목 | GraphQL | REST |
|:---|:---|:---|
| **엔드포인트** | 1개 | 여러 개 |
| **데이터** | 필요한 만큼 | 전체 |
| **버전 관리** | 없음 | 필요 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 복잡한 쿼리 → GraphQL
**시나리오 2:** 단순한 API → REST
**시나리오 3:** 하이브리드 → 혼합
