# JSON - 데이터 교환 형식

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 경량 데이터 교환 형식
> 2. **가치**: 가독성, 파싱 용이
> 3. **융합**: REST, NoSQL, 설정

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

JSON(JavaScript Object Notation)은 경량 데이터 교환 형식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    JSON                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              JSON 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  {                                                    │   │ │
│   │   │    "name": "홍길동",                                  │   │ │
│   │   │    "age": 30,                                         │   │ │
│   │   │    "address": {                                       │   │ │
│   │   │      "city": "서울",                                  │   │ │
│   │   │      "zip": "12345"                                   │   │ │
│   │   │    },                                                 │   │ │
│   │   │    "hobbies": ["독서", "운동"]                        │   │ │
│   │   │  }                                                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              JSON 데이터 타입                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 타입       │ 예시     │ 설명     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ string     │ "hello"  │ 문자열   │               │   │ │
│   │   │  │ number     │ 123      │ 숫자     │               │   │ │
│   │   │  | boolean    │ true     │ 참/거짓  │               │   │ │
│   │   │  │ object     │ {"k":"v"}│ 객체     │               │   │ │
│   │   │  │ array      │ [1,2,3]  │ 배열     │               │   │ │
│   │   │  │ null       │ null     │ 없음     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: JSON은 키-값 쌍으로 데이터를 표현합니다.

**💡 비유**: JSON은 데이터의 엑셀입니다. 구조화된 정보를 담습니다.

**📢 섹션 요약 비유:** JSON = 데이터 교환!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Object** | 중괄호 {} |
| **Array** | 대괄호 [] |
| **Key-Value** | 키:값 |
| **String** | 따옴표 |

**비교표: JSON vs XML**

```
+------------+----------+----------+
| 항목       │ JSON     │ XML      │
+------------+----------+----------+
| 크기       │ 작음     │ 큼       │
| 파싱       │ 빠름     │ 느림     │
| 가독성     │ 높음     │ 중간     │
| 스키마     │ 선택     │ DTD/XSD  │
+------------+----------+----------+
```

**핵심 알고리즸:** 직렬화, 파싱

**📢 섹션 요약 비유:** JSON = 경량!
