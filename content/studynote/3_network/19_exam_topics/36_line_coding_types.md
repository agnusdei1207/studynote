# Line Coding (단극성, 극성, 양극성)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디지털 신호의 전압 레벨 표현 방식
> 2. **가치**: DC 성분, 대역폭, 동기화 특성 결정
> 3. **융합**: NRZ, RZ, Manchester, AMI와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Line Coding은 디지털 데이터를 전송 신호로 변환할 때 전압 레벨을 어떻게 표현하는지에 따라 단극성, 극성, 양극성으로 분류됩니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Line Coding 분류                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              1. 단극성 (Unipolar)                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  특징:                                                 │   │ │
│   │   │  • 0: 무전압 (0V)                                      │   │ │
│   │   │  • 1: +V (양전압)                                      │   │ │
│   │   │  • 가장 간단한 방식                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  파형:                                                 │   │ │
│   │   │  데이터:  1   0   1   1   0                          │   │ │
│   │   │           ┌┐       ┌┐┌┐                              │   │ │
│   │   │         ──┘└───────┘└┘└────                          │   │ │
│   │   │                                                      │   │ │
│   │   │  문제: DC 성분 존재, 동기화 어려움                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              2. 극성 (Polar)                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  특징:                                                 │   │ │
│   │   │  • 0: -V (음전압)                                      │   │ │
│   │   │  • 1: +V (양전압)                                      │   │ │
│   │   │  • NRZ-L, NRZ-I가 대표적                               │   │ │
│   │   │                                                      │   │ │
│   │   │  파형:                                                 │   │ │
│   │   │  데이터:  1   0   1   1   0                          │   │ │
│   │   │           ┌┐       ┌┐┌┐                              │   │ │
│   │   │         ──┘└──┌───┘└┘└┌──                            │   │ │
│   │   │              └───────┘                                │   │ │
│   │   │                                                      │   │ │
│   │   │  장점: 단극성보다 DC 성분 적음                         │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              3. 양극성 (Bipolar)                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  특징:                                                 │   │ │
│   │   │  • 0: 무전압 (0V)                                      │   │ │
│   │   │  • 1: +V, -V 교대 (Alternate Mark Inversion)          │   │ │
│   │   │  • AMI, Pseudoternary가 대표적                         │   │ │
│   │   │                                                      │   │ │
│   │   │  파형:                                                 │   │ │
│   │   │  데이터:  1   0   1   1   0                          │   │ │
│   │   │           ┌┐       ┌┐┌┐                              │   │ │
│   │   │         ──┘└───────┘└┘└────                          │   │ │
│   │   │          +V       -V+V                               │   │ │
│   │   │                                                      │   │ │
│   │   │  장점: DC 성분 없음, 오류 검출 가능                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 양극성(AMI)은 전화망에서 널리 사용됩니다.

**💡 비유**: 단극성은 스위치 ON/OFF, 양극성은 좌/우 교대 전환입니다.

**📢 섹션 요약 비유**: 양극성이 DC 성분 제거!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 유형 | 0 표현 | 1 표현 | DC 성분 |
|:---|:---|:---|:---:|
| **단극성** | 0V | +V | 있음 |
| **극성** | -V | +V | 있음 |
| **양극성** | 0V | ±V 교대 | 없음 |

**비교표**

```
+------------+----------+----------+----------+----------+
| 유형       | 대역폭   | 동기화   | DC 성분  | 오류검출 |
+------------+----------+----------+----------+----------+
| 단극성     | 낮음     | 어려움   | 있음     | 없음     |
| 극성       | 낮음     | 어려움   | 적음     | 없음     |
| 양극성     | 낮음     | 보통     | 없음     | 있음     |
+------------+----------+----------+----------+----------+
```

**핵심 알고리즘**: 데이터 → 전압 매핑 → 전송

**📢 섹션 요약 비유**: AMI는 연속 1에서 극성 교대!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 응용 분야**

| 유형 | 응용 |
|:---|:---|
| **단극성** | TTL 논리 |
| **극성** | NRZ, 직렬 통신 |
| **양극성** | T1/E1 전화망 |

**📢 섹션 요약 비유**: T1 회선은 AMI 사용!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 전화망** → AMI (양극성)
**시나리오 2: 컴퓨터 내부** → NRZ (극성)
**시나리오 3: 이더넷** → Manchester/PAM

**도입 체크리스트**: DC 성분, 대역폭, 동기화

**📢 섹션 요약 비유**: 양극성은 AMI 위반으로 에러 검출!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: DC 성분 제거, 에러 검출

**미래 전망**: 고속 전송은 PAM 방식

**참고 표준**: ITU-T G.703 (AMI)

**📢 섹션 요약 비유**: Line Coding은 신호의 문법!
