### Ⅰ. 개요 (Context & Background)

**개념 정의**

ILP는 독립적인 명령어를 병렬로 실행합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ILP                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ILP 활용                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Sequential Code:                                    │   │ │
│   │   │  A = B + C                                           │   │ │
│   │   │  D = E + F     ← A와 독립                            │   │ │
│   │   │  G = A + D     ← A, D에 의존                         │   │ │
│   │   │                                                      │   │ │
│   │   │  ILP 활용:                                           │   │ │
│   │   │  Cycle 1: A = B + C, D = E + F  (병렬 실행)          │   │ │
│   │   │  Cycle 2: G = A + D                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  ILP = 2 (한 사이클에 2개 명령어)                     │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: ILP는 데이터 의존성이 없는 명령어를 동시 실행합니다.

**📢 섹션 요약 비유:** ILP = 동시 작업!
