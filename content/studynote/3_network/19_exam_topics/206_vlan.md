# VLAN (Virtual LAN) - 논리적 네트워크 분할

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리적 스위치를 논리적으로 분할
> 2. **가치**: 브로드캐스트 도메인 분리, 보안 강화
> 3. **융합**: 트렁킹, STP, L3 스위치와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

VLAN(Virtual LAN)은 하나의 물리적 스위치를 여러 개의 논리적 네트워크로 분할하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    VLAN (Virtual LAN)                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │

│   │              VLAN 개념                                         │ │

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [물리적 스위치]                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────┐  ┌─────────────┐                   │   │ │
│   │   │  │  VLAN 10    │  │  VLAN 20    │                   │   │ │
│   │   │  │  (영업팀)    │  │  (개발팀)   │                   │   │ │
│   │   │  │             │  │             │                   │   │ │

│   │   │                                                      │   │ │
```

> **해설**: VLAN은 물리적 위치와 무관하게 논리적으로 네트워크를 분리합니다.

**💡 비유**: VLAN은 아파트의 층별 출입카드입니다. 같은 건물이지만 층마다 접근 권한이 다릅니다.

**📢 섹션 요약 비유:** VLAN = 논리적 분할!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 | 범위 |
|:---|:---|:---|
| **VLAN ID** | 1-4094 | 0, 4095 예약 |
| **Access Port** | 단일 VLAN | PC 연결 |
| **Trunk Port** | 다중 VLAN | 스위치 간 |
| **Native VLAN** | 태그 없음 | 기본 1 |

**비교표: 802.1Q 태그 구조**

```
+------------+----------+----------+
| 필드       │ 비트     │ 설명     │
+------------+----------+----------+
| TPID       │ 16       │ 0x8100   │
| Priority   │ 3        │ CoS      │
| CFI        │ 1        │ 토큰링   │
| VLAN ID    │ 12       │ 1-4094   │
+------------+----------+----------+
```

**핵심 알고리즸:** 802.1Q 태깅, 프레임 전달

**📢 섹션 요약 비유:** 802.1Q = VLAN 태그 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 트렁킹 프로토콜**

| 프로토콜 | 표준 | 특징 |
|:---:|:---|:---|
| **802.1Q** | IEEE | 표준, Native VLAN |
| **ISL** | Cisco | 레거시, 캡슐화 |

**VTP (VLAN Trunking Protocol)**

| 모드 | 특징 |
|:---|:---|
| **Server** | VLAN 생성/수정/전파 |
| **Client** | 수신만 |
| **Transparent** | 전파 안함 |

**📢 섹션 요약 비유:** VTP = VLAN 동기화!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 부서별 분리** → VLAN 10, 20, 30
**시나리오 2: 보안 강화** → Voice VLAN, Data VLAN
**시나리오 3: 데이터센터** → VXLAN

**도입 체크리스트:** VLAN 수, 트렁크 대역폭, L3 라우팅

**📢 섹션 요약 비유:** 부서별 VLAN 분리!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 브로드캐스트 억제, 보안 강화

**미래 전망:** VXLAN, EVPN

**참고 표준:** IEEE 802.1Q, IEEE 802.1ad

**📢 섹션 요약 비유:** VLAN은 네트워크 분할의 기본!
