# Keepalive - 킵얼라이브

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연결 유지 확인
> 2. **가치**: 좀비 연결 탐지
> 3. **융합:** TCP Keepalive, HTTP Keep-Alive, Heartbeat

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Keepalive는 연결이 활성 상태인지 확인합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Keepalive                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Keepalive 유형                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  TCP Keepalive:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Client                    Server                ││   │ │
│   │   │  │    │                         │                   ││   │ │
│   │   │  │    │──── Keepalive ACK ─────→│                   ││   │ │
│   │   │  │    │←─── ACK ───────────────│                   ││   │ │
│   │   │  │    │                         │                   ││   │ │
│   │   │  │  (데이터 없이 연결 확인)                         ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  HTTP Keep-Alive:                                       │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Connection: keep-alive                          ││   │ │
│   │   │  │ Keep-Alive: timeout=5, max=100                  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  TCP 연결 재사용으로 성능 향상                      ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  TCP Keepalive 파라미터:                                │   │ │
│   │   │  - Idle Time: 7200초 (기본)                             │   │ │
│   │   │  - Interval: 75초                                       │   │ │
│   │   │  - Probes: 9회                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Keepalive는 연결 상태를 유지 관리합니다.

**📢 섹션 요약 비유:** Keepalive = "아직 살아있어?"

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Idle** | 유휴 시간 |
| **Interval** | 간격 |
| **Probe** | 프로브 |
| **Timeout** | 타임아웃 |
| **Heartbeat** | 하트비트 |

**핵심 알고리듬:** 유휴 → 프로브 전송 → 응답 확인 → 연결 유지/종료

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**TCP vs HTTP Keep-Alive**

| 항목 | TCP Keepalive | HTTP Keep-Alive |
|:---|:---|:---|
| **계층** | L4 | L7 |
| **목적** | 연결 확인 | 연결 재사용 |
| **데이터** | 없음 | HTTP 헤더 |
