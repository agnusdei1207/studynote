# PAMA (Pre-Assigned Multiple Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사전 고정 자원 할당
> 2. **가치**: 낮은 지연, 안정적 연결
> 3. **융합**: 위성, 전용선과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

PAMA(Pre-Assigned Multiple Access)는 통신 자원을 사전에 고정 할당하는 다중 접속 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PAMA (Pre-Assigned Multiple Access)              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PAMA 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  기본 구조:                                              │   │ │
│   │   │                                                      │   │ │
│   │   │  채널 할당 (사전 고정):                                 │   │ │
│   │   │  ┌─────┬─────┬─────┬─────┬─────┬─────┐              │   │ │
│   │   │  │ CH1 │ CH2 │ CH3 │ CH4 │ CH5 │ CH6 │              │   │ │
│   │   │  │노드A│노드B│노드C│노드D│노드E│노드F│              │   │ │
│   │   │  └─────┴─────┴─────┴─────┴─────┴─────┘              │   │ │
│   │   │    고정    고정    고정    고정    고정    고정        │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                 │   │ │
│   │   │  • 채널이 노드에 고정 할당                             │   │ │
│   │   │  • 할당이 변경되지 않음                                │   │ │
│   │   │  • 요청 없이 즉시 전송 가능                            │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                   │   │ │
│   │   │  • 낮은 지연 (할당 대기 없음)                           │   │ │

│   │   │  단점:                                                   │   │ │
│   │   │  • 유휴 자원 낭비                                      │   │ │

│   │   │  • 확장성 제한                                         │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PAMA 할당 방식                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. FDMA-PAMA:                                           │   │ │

│   │   │                                                      │   │ │
│   │   │  3. CDMA-PAMA:                                           │   │ │
│   │   │     • 코드 고정 할당                                     │   │ │
```

> **해설**: PAMA는 전용선, 고정 할당 위성 회선에서 사용됩니다.

**💡 비유**: PAMA는 매월 고정된 사무실 임대입니다.

**📢 섹션 요약 비유**: 고정 = 안정!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **고정 할당** | 자원 배정 | 사전 설정 | 임대 |
| **전용 채널** | 독점 사용 | 경쟁 없음 | 사무실 |
| **항시 대기** | 즉시 전송 | 지연 없음 | 열쇠 |

**비교표**

```
+------------+----------+----------+
| 항목       | PAMA     | DAMA     |
+------------+----------+----------+
| 할당       | 고정     | 동적     |
| 효율       | 낮음     | 높음     |
| 지연       | 없음     | 있음     |
| 안정성     | 높음     | 중간     |
+------------+----------+----------+
```

**핵심 알고리즸**: 사전 할당 → 즉시 전송 → 유지

**📢 섹션 요약 비유**: 전용선 = PAMA!
