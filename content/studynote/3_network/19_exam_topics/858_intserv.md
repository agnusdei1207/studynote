# IntServ - 통합 서비스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 종단간 자원 예약
> 2. **가치**: 하드 보장
> 3. **융합:** RSVP, Flow-based, Guarantees

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IntServ (Integrated Services)는 종단간 자원을 예약합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IntServ                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IntServ 아키텍처                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RSVP (Resource Reservation Protocol):                │   │ │
│   │   │                                                      │   │ │
│   │   │  Sender              Router             Receiver      │   │ │
│   │   │  ┌──────┐           ┌─────┐            ┌──────┐      │   │ │
│   │   │  │      │ ── PATH ─→│     │ ── PATH ──→│      │      │   │ │
│   │   │  │      │           │     │            │      │      │   │ │
│   │   │  │      │ ←─ RESV ──│     │ ←─ RESV ──│      │      │   │ │
│   │   │  └──────┘           └─────┘            └──────┘      │   │ │
│   │   │                                                      │   │ │
│   │   │  서비스 클래스:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Guaranteed: 하드 보장, 지연 한계                  ││   │ │
│   │   │  │ Controlled Load: 소프트 보장, 최선형              ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  문제점:                                                │   │ │
│   │   │  - 확장성: 플로우별 상태 유지                           │   │ │
│   │   │  - 복잡도: 모든 라우터 지원 필요                        │   │ │
│   │   │  - 배포: 제한적                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  해결: DiffServ + IntServ 혼합                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IntServ은 완벽한 QoS 보장을 제공하지만 확장성이 제한됩니다.

**📢 섹션 요약 비유:** IntServ = 예약석!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RSVP** | 자원 예약 프로토콜 |
| **PATH** | 경로 메시지 |
| **RESV** | 예약 메시지 |
| **Flow** | 트래픽 흐름 |
| **Admission Control** | 승인 제어 |

**핵심 알고리즸:** PATH → 자원 확인 → RESV → 예약

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**IntServ vs DiffServ**

| 항목 | IntServ | DiffServ |
|:---|:---|:---|
| **단위** | Flow | Class |
| **상태** | 유지 | 무상태 |
| **확장성** | 낮음 | 높음 |
| **보장** | 하드 | 소프트 |
