# RIP - 라우팅 정보 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 거리 벡터 라우팅
> 2. **가치**: 간단한 구성
> 3. **융합:** Hop Count, Bellman-Ford, Split Horizon

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RIP (Routing Information Protocol)는 거리 벡터 기반 라우팅 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RIP                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RIP 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  거리 벡터 원리:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │    Router A          Router B          Router C││   │ │
│   │   │  │    ┌──────┐          ┌──────┐          ┌──────┐││   │ │
│   │   │  │    │RIP   │──1hop───│RIP   │──1hop───│RIP   │││   │ │
│   │   │  │    │Table │          │Table │          │Table │││   │ │
│   │   │  │    └──────┘          └──────┘          └──────┘││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │    A → B: 1 hop                                 ││   │ │
│   │   │  │    A → C: 2 hops (via B)                        ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                  │   │ │
│   │   │  - 최대 홉 수: 15 (16 = 무한대)                         │   │ │
│   │   │  - 업데이트: 30초마다 브로드캐스트                        │   │ │
│   │   │  - 메트릭: 홉 수만 사용                                  │   │ │
│   │   │  - 포트: UDP 520                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RIP는 가장 오래된 라우팅 프로토콜입니다.

**📢 섹션 요약 비유:** RIP = 거리 표지만 보는 내비게이션!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Hop Count** | 홉 수 |
| **Update** | 업데이트 |
| **Invalid** | 무효 타이머 |
| **Flush** | 플러시 타이머 |
| **Holddown** | 홀드다운 |

**핵심 알고리듬:** Bellman-Ford 알고리즘
