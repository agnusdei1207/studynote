### Ⅰ. 개요 (Context & Background)

**개념 정의**

Replication은 데이터를 복제합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Replication                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              복제 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Synchronous (동기):                                  │   │ │
│   │   │  Primary ──→ Standby (확인 대기) ──→ 완료            │   │ │
│   │   │  장점: 무손실 / 단점: 지연                             │   │ │
│   │   │                                                      │   │ │
│   │   │  Asynchronous (비동기):                                │   │ │
│   │   │  Primary ──→ Standby (확인 없이) ──→ 완료            │   │ │
│   │   │  장점: 빠름 / 단점: 손실 가능                          │   │ │
│   │   │                                                      │   │ │
│   │   │  복제 토폴로지:                                        │   │ │
│   │   │  - Master-Slave                                      │   │ │
│   │   │  - Master-Master                                     │   │ │
│   │   │  - Multi-Master                                      │   │ │
│   │   │  - Circular                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 복제는 가용성과 성능을 위해 사용합니다.

**📢 섹션 요약 비유:** Replication = 복사본!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Primary/Master** | 주 노드 |
| **Replica/Slave** | 복제 노드 |
| **Sync** | 동기 |
| **Async** | 비동기 |
| **Lag** | 복제 지연 |

**핵심 알고리즸:** 쓰기 → 복제 → 확인(동기) or 생략(비동기)
