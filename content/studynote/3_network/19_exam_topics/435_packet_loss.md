# 패킷 손실 - Packet Loss

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 중 패킷 소실
> 2. **가치**: 품질 저하, 재전송 오버헤드
> 3. **융합**: TCP 재전송, FEC, QoS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

패킷 손실은 전송 중 패킷이 손실되는 현상입니다.

```

├─────────────────────────────────────────────────────────────────────────┤

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 네트워크 혼잡                                    │   │ │
```

> **해설**: 패킷 손실은 TCP에서는 재전송으로, UDP에서는 품질 저하로 나타납니다.

**💡 비유**: 패킷 손실은 우편물 분실입니다. 일부가 도착하지 않습니다.

**📢 섹션 요약 비유:** Packet Loss = 품질 저하!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Loss Rate** | 손실률 (%) |
| **Burst Loss** | 연속 손실 |
| **Random Loss** | 무작위 손실 |
| **FEC** | Forward Error Correction |

**비교표: 허용 손실률**

```
+------------+----------+----------+
| 애플리케이션│ 허용     │ 영향     │
+------------+----------+----------+
| VoIP       │ <1%      │ 음질     │
| 화상회의   │ <0.1%    │ 화면     │
| 게임       │ <0.5%    │ 랙       │
| TCP        │ <5%      │ 처리량   │
+------------+----------+----------+
```

**핵심 알고리즸:** TCP 재전송, FEC

**📢 섹션 요약 비유:** FEC = 손실 복구!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 대응 방안**

| 방안 | 설명 | 용도 |
|:---|:---|:---|
| **재전송** │ TCP | 신뢰성 |
| **FEC** │ 중복 데이터 | 실시간 |
| **인터리빙** │ 분산 | 무선 |

**📢 섹션 요약 비유:** 실시간 = FEC!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** TCP 앱 → 재전송 허용
**시나리오 2:** VoIP → FEC + 버퍼
**시나리오 3:** 높은 손실 → 원인 진단

**도입 체크리스트:** 모니터링, 원인 분석, FEC

**📢 섹션 요약 비유:** 모니터링 = 손실 감지!
