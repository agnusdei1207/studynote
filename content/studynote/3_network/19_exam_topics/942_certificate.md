# Certificate - 인증서

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공개키 신원 증명
> 2. **가치**: 신뢰할 수 있는 연결
> 3. **융합:** X.509, CSR, Chain

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Certificate은 공개키와 소유자 정보를 바인딩합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Certificate                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              X.509 인증서 구조                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  인증서 내용:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Version: 버전 (v3)                             ││   │ │
│   │   │  │  Serial Number: 일련번호                         ││   │ │
│   │   │  │  Issuer: 발급자 (CA)                             ││   │ │
│   │   │  │  Validity: 유효기간 (Not Before/After)           ││   │ │
│   │   │  │  Subject: 소유자 (도메인/조직)                     ││   │ │
│   │   │  │  Subject Public Key Info: 공개키                  ││   │ │
│   │   │  │  Extensions: 확장 필드                            ││   │ │
│   │   │  │    - Key Usage: 키 용도                           ││   │ │
│   │   │  │    - SAN: 주체 대체 이름                          ││   │ │
│   │   │  │    - CRL DP: 폐기 목록 위치                        ││   │ │
│   │   │  │  Signature: CA 서명                               ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  발급 과정:                                              │   │ │
│   │   │  1. 키 쌍 생성 → 2. CSR 생성 → 3. CA 제출              │   │ │
│   │   │  → 4. 검증 → 5. 인증서 발급                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 인증서는 HTTPS 보안의 기반입니다.

**📢 섹션 요약 비유:** Certificate = 디지털 신분증!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Subject** | 소유자 |
| **Issuer** | 발급자 |
| **Validity** | 유효기간 |
| **Public Key** | 공개키 |
| **Signature** | 서명 |

**핵심 알고리듬:** CSR → CA 검증 → 서명된 인증서

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**인증서 검증 레벨**

| 레벨 | 검증 내용 | 표시 |
|:---|:---|:---|
| **DV** | 도메인 소유 | 자물쇠 |
| **OV** | 조직 확인 | 회사명 |
| **EV** | 엄격한 확인 | 주소창 녹색 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹사이트 → DV/OV/EV 인증서
**시나리오 2:** API → 클라이언트 인증서
**시나리오 3:** 내부 → 자체 서명 CA
