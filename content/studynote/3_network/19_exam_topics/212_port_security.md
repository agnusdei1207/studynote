# 포트 보안 (Port Security) - 접근 제어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 포트별 MAC 주소 제한
> 2. **가치**: 무단 접근 방지, 보안 강화
> 3. **융합**: 802.1X, DHCP 스누핑과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

포트 보안(Port Security)은 스위치 포트에 연결할 수 있는 MAC 주소를 제한하여 무단 접근을 방지하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    포트 보안 (Port Security)                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              포트 보안 기능                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. MAC 주소 학습 방식:                               │   │ │
│   │   │     • Static: 수동 설정                              │   │ │
│   │   │     • Dynamic: 자동 학습                             │   │ │
│   │   │     • Sticky: 자동 학습 + 저장                       │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 최대 MAC 수:                                      │   │ │
│   │   │     • 기본: 1개                                      │   │ │
│   │   │     • 최대: 포트별 다름                              │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 위반 시 동작 (Violation):                         │   │ │
│   │   │     • Protect: 무시                                  │   │ │
│   │   │     • Restrict: 무시 + 로그                          │   │ │
│   │   │     • Shutdown: 포트 차단                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              포트 보안 설정 예시                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  interface FastEthernet0/1                          │   │ │
│   │   │   switchport mode access                            │   │ │
│   │   │   switchport port-security                          │   │ │
│   │   │   switchport port-security maximum 2                │   │ │
│   │   │   switchport port-security mac-address sticky       │   │ │
│   │   │   switchport port-security violation shutdown       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 포트 보안은 Sticky MAC으로 자동 학습과 저장을 동시에 수행합니다.

**💡 비유**: 포트 보안은 출입증입니다. 등록된 사람만 출입 가능합니다.

**📢 섹션 요약 비유:** Sticky = 자동 + 저장!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 위반 모드 | 동작 | 로그 | 포트 |
|:---|:---|:---:|:---|
| **Protect** | 폐기 | X | 유지 |
| **Restrict** | 폐기 | O | 유지 |
| **Shutdown** | 폐기 | O | 차단 |

**비교표: MAC 학습 방식**

```
+------------+----------+----------+
| 방식       │ 저장     │ 설정     │
+------------+----------+----------+
| Static     │ O        │ 수동     │
| Dynamic    │ X        │ 자동     │
| Sticky     │ O        │ 자동     │
+------------+----------+----------+
```

**핵심 알고리즸:** MAC 학습, 위반 감지

**📢 섹션 요약 비유:** Shutdown = 강제 차단!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 사무실** → Sticky + Restrict
**시나리오 2: 서버** → Static MAC
**시나리오 3: 공용** → Shutdown

**도입 체크리스트:** MAC 수, 위반 모드, 에이징

**📢 섹션 요약 비유:** Restrict = 로그 + 유지!
