# 혼잡 제어 (Congestion Control)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 혼잡을 방지하기 위한 송신 속도 조절
> 2. **가치**: 네트워크 붕괴 방지, 공정성, 효율성
> 3. **융합**: TCP, AIMD, BBR, ECN과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

혼잡 제어(Congestion Control)는 네트워크 내의 혼잡을 방지하고 완화하기 위해 송신 속도를 조절하는 메커니즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    혼잡 제어 (Congestion Control)                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              혼잡 제어 vs 흐름 제어                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Flow Control (흐름 제어):                             │   │ │
│   │   │  • 목적: 수신측 보호                                    │   │ │
│   │   │  • 대상: 송신↔수신                                     │   │ │
│   │   │  • 기법: Sliding Window, XON/XOFF                      │   │ │
│   │   │                                                      │   │ │
│   │   │  Congestion Control (혼잡 제어):                        │   │ │
│   │   │  • 목적: 네트워크 보호                                  │   │ │
│   │   │  • 대상: 전체 네트워크                                   │   │ │
│   │   │  • 기법: AIMD, Slow Start, BBR                         │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TCP 혼잡 제어 알고리즘                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. Slow Start (느린 시작):                            │   │ │
│   │   │     • cwnd = 1 MSS로 시작                              │   │ │
│   │   │     • ACK마다 cwnd 2배 (지수적 증가)                     │   │ │
│   │   │     • ssthresh 도달 시 AIMD로 전환                      │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Congestion Avoidance (혼잡 회피):                   │   │ │
│   │   │     • cwnd = cwnd + 1/cwnd (선형 증가)                  │   │ │
│   │   │     • RTT마다 1 MSS씩 증가                              │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Fast Retransmit (빠른 재전송):                      │   │ │
│   │   │     • 3개 중복 ACK 수신 시 즉시 재전송                   │   │ │
│   │   │     • Timeout 대기 없음                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  4. Fast Recovery (빠른 복구):                          │   │ │
│   │   │     • ssthresh = cwnd/2                                 │   │ │
│   │   │     • cwnd = ssthresh + 3                               │   │ │
│   │   │     • Slow Start 건너뜀                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 혼잡 제어는 네트워크 전체의 건강을 위한 것입니다.

**💡 비유**: 혼잡 제어는 도로 정체를 막기 위해 차량 속도를 조절하는 것입니다.

**📢 섹션 요약 비유**: AIMD = 가법 증가 승법 감소!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **cwnd** | 혼잡 윈도우 | 송신 제한 | 속도계 |
| **ssthresh** | 임계값 | 전환점 | 기준점 |
| **RTT** | 왕복 시간 | 측정 | 거리 |
| **RTO** | 타임아웃 | 재전송 | 알람 |

**상태 전이도**

```
         Timeout
    ┌──────────────────┐
    │                  ▼
┌───┴───┐         ┌─────────┐
│ Slow  │ 3 dupACK│ Fast    │
│ Start │────────→│ Recovery│
└───┬───┘         └────┬────┘
    │                  │
    │ ssthresh         │
    ▼                  │
┌───────────┐          │
│Congestion │←─────────┘
│Avoidance  │  new ACK
└───────────┘
```

**핵심 알고리즘**: Slow Start → AIMD → Loss → Backoff

**📢 섹션 요약 비유**: TCP Tahoe/Reno/NewReno!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 혼잡 제어 버전**

| 버전 | Loss 대응 | 특징 |
|:---|:---|:---|
| **Tahoe** | Slow Start | 보수적 |
| **Reno** | Fast Recovery | 중간 |
| **NewReno** | Partial ACK | 개선 |
| **BBR** | Bandwidth-based | 최신 |

**📢 섹션 요약 비유**: BBR이 현재 최신!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 일반 인터넷** → CUBIC (Linux 기본)
**시나리오 2: 고속 네트워크** → BBR
**시나리오 3: 데이터센터** → DCTCP

**도입 체크리스트**: RTT, 대역폭, 버퍼 크기

**📢 섹션 요약 비유**: CUBIC이 Linux 표준!
