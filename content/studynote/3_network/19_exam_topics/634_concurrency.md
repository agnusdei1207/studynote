# Concurrency - 동시성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동시 실행 관리
> 2. **가치**: 자원 효율성, 응답성
> 3. **융합:** Thread, Lock, Async

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Concurrency는 여러 작업을 동시에 진행하는 능력입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Concurrency                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Concurrency vs Parallelism                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Concurrency (동시성):       Parallelism (병렬성):   │   │ │
│   │   │  ┌─────────────────┐        ┌─────────────────┐    │   │ │
│   │   │  │ Core 1          │        │ Core 1 │ Core 2 │    │   │ │
│   │   │  │ A B A B A B     │        │ A A A │ B B B │    │   │ │
│   │   │  │ 인터리빙        │        │ 동시 실행        │    │   │ │
│   │   │  └─────────────────┘        └─────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  Concurrency: 논리적 동시 (싱글코어도 가능)          │   │ │
│   │   │  Parallelism: 물리적 동시 (멀티코어 필요)            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 동시성은 작업 전환으로 동시 진행 효과를 냅니다.

**📢 섹션 요약 비유:** Concurrency = 멀티태스킹!
