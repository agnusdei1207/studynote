# Authentication - 인증

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 신원 확인
> 2. **가치**: 접근 제어 기반
> 3. **융합:** Password, MFA, Certificate, Token

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Authentication은 사용자/시스템의 신원을 확인합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Authentication                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인증 요소 (3-Factor)                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Something You Know (지식):                            │   │ │
│   │   │  - 비밀번호 (Password)                                │   │ │
│   │   │  - PIN                                                │   │ │
│   │   │                                                      │   │ │
│   │   │  Something You Have (소유):                            │   │ │
│   │   │  - 스마트폰 (OTP, SMS)                                 │   │ │
│   │   │  - 하드웨어 토큰                                       │   │ │
│   │   │  - 스마트카드                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  Something You Are (생체):                             │   │ │
│   │   │  - 지문 (Fingerprint)                                 │   │ │
│   │   │  - 얼굴 인식 (Face ID)                                 │   │ │
│   │   │  - 홍채 (Iris)                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  MFA (Multi-Factor Authentication):                    │   │ │
│   │   │  2개 이상 요소 조합                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  인증 방식:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Basic Auth: ID/PW (취약)                         ││   │ │
│   │   │  │ Digest Auth: 해시 (기본)                          ││   │ │
│   │   │  │ OAuth 2.0: 토큰 기반                              ││   │ │
│   │   │  │ SAML: SSO                                        ││   │ │
│   │   │  │ JWT: JSON Web Token                              ││   │ │
│   │   │  │ Certificate: 클라이언트 인증서                     ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 인증은 보안의 첫 번째 관문입니다.

**📢 섹션 요약 비유:** Authentication = 신분 확인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Identity** | 신원 |
| **Credential** | 자격 증명 |
| **Token** | 토큰 |
| **Session** | 세션 |
| **MFA** | 다중 인증 |

**핵심 알고리즸:** 자격 증명 → 검증 → 토큰 발급

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**인증 방식 비교**

| 방식 | 편의성 | 보안 | 용도 |
|:---|:---|:---|:---|
| **Password** | 높음 | 낮음 | 기본 |
| **MFA** | 중간 | 높음 | 중요 |
| **Certificate** | 높음 | 높음 | 시스템 |
| **SSO** | 높음 | 중간 | 기업 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹 → OAuth 2.0
**시나리오 2:** API → JWT
**시나리오 3:** 관리 → MFA

---
