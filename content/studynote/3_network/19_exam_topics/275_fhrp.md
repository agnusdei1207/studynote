# FHRP - 퍼스트 홉 라우터 이중화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 게이트웨이 이중화 프로토콜 그룹
> 2. **가치**: 무중단 네트워크
> 3. **융합**: VRRP, HSRP, GLBP 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

FHRP(First Hop Redundancy Protocol)는 게이트웨이 이중화를 위한 프로토콜 모음입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FHRP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FHRP 개요                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  문제: 단일 게이트웨이                               │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ 라우터 장애 시                                │   │ │
│   │   │       │ 모든 호스트 통신 두절                         │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           FHRP 해결책                        │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • 여러 라우터를 하나의 가상 라우터로         │    │   │ │
│   │   │  │  • 가상 IP/MAC 공유                          │    │   │ │
│   │   │  │  • 자동 페일오버                             │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FHRP 프로토콜 비교                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  +------------+----------+----------+----------+     │   │ │
│   │   │  │ 프로토콜   │ 벤더     │ 로드밸런싱│ 표준     │     │   │ │
│   │   │  +------------+----------+----------+----------+     │   │ │
│   │   │  │ HSRP       │ Cisco    │ 없음     │ 전용     │     │   │ │
│   │   │  │ VRRP       │ 표준     │ 없음     │ RFC 5798 │     │   │ │
│   │   │  │ GLBP       │ Cisco    │ 있음     │ 전용     │     │   │ │
│   │   │  │ IRDP       │ 표준     │ 있음     │ RFC 1256 │     │   │ │
│   │   │  +------------+----------+----------+----------+     │   │ │
│   │   │                                                      │   │ │

│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: FHRP는 게이트웨이 단일 실패점을 제거합니다.

**💡 비유**: FHRP는 비상구 시스템입니다. 하나가 막히면 다른 출구로 나갑니다.

**📢 섹션 요약 비유:** FHRP = 게이트웨이 이중화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Virtual IP** | 공유 IP |
| **Virtual MAC** | 공유 MAC |
| **Active/Master** | 활성 |
| **Standby/Backup** | 대기 |

**비교표: FHRP 특징**

```
+------------+----------+----------+
| 특징       │ HSRP/VRRP│ GLBP    │
+------------+----------+----------+
| 동시 활성  │ 1개      │ 여러 개 │
| 대역폭     │ 일부 유휴 │ 전부 사용│
| 복잡도     │ 낮음     │ 중간    │
+------------+----------+----------+
```

**핵심 알고리즸:** 선출, 페일오버

**📢 섹션 요약 비유:** Active = 활성!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 멀티캐스트 주소**

| 프로토콜 | 주소 |
|:---|:---|
| **HSRP v1** | 224.0.0.2 |
| **HSRP v2** | 224.0.0.102 |
| **VRRP** | 224.0.0.18 |
| **GLBP** | 224.0.0.102 |

**📢 섹션 요약 비유:** VRRP = 표준!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: Cisco 환경** → HSRP 또는 GLBP
**시나리오 2: 멀티벤더** → VRRP
**시나리오 3: 대역폭 최적** → GLBP

**도입 체크리스트:** 벤더, 로드밸런싱, 표준

**📢 섹션 요약 비유:** GLBP = 대역폭 활용!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 고가용성, 무중단

**미래 전망:** VRRP가 표준으로 확대

**참고 표준:** RFC 5798

**📢 섹션 요약 비유:** FHRP는 네트워크 안정성 필수!
