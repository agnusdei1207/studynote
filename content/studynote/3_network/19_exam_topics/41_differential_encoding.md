# 차분 부호화 (Differential Encoding)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 절대값이 아닌 변화로 표현
> 2. **가치**: 극성 반전에 강함, 위상 모호성 해결
> 3. **융합**: NRZ-I, 차분 맨체스터, PSK와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

차분 부호화(Differential Encoding)는 데이터 비트를 절대적인 레벨이 아닌 신호의 변화(전이)로 표현하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    차분 부호화 (Differential Encoding)               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              기본 원리                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  절대 부호화 (Absolute Encoding):                      │   │ │
│   │   │  • 0 = 낮은 레벨                                      │   │ │
│   │   │  • 1 = 높은 레벨                                      │   │ │
│   │   │  • 극성 반전 시 모두 반전됨                            │   │ │
│   │   │                                                      │   │ │
│   │   │  차등 부호화 (Differential Encoding):                  │   │ │
│   │   │  • 0 = 변화 없음                                      │   │ │
│   │   │  • 1 = 레벨 변화 (전이)                                │   │ │
│   │   │  • 극성 반전에도 데이터 보존                           │   │ │
│   │   │                                                      │   │ │
│   │   │  예시 (NRZ-I):                                        │   │ │
│   │   │  데이터:    1   0   1   1   0   1                     │   │ │
│   │   │  신호:     ┌┐   ┌┐   ┌┐   ┌┐   ┌┐                   │   │ │
│   │   │          ──┘└───┘└───┘└───┘└───┘└──                 │   │ │
│   │   │            전이 유지 전이 전이 유지 전이               │   │ │
│   │   │                                                      │   │ │
│   │   │  극성 반전 후:                                        │   │ │
│   │   │  신호:     ┌┐   ┌┐   ┌┐   ┌┐   ┌┐                   │   │ │
│   │   │          ─┘ └───┘ └───┘ └───┘ └───┘ └──              │   │ │
│   │   │          (전이 위치 동일 → 데이터 동일)                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              장점과 단점                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  장점:                                                 │   │ │
│   │   │  • 극성 반전에 강함 (Polarity Independence)            │   │ │
│   │   │  • 위상 모호성 해결 (Phase Ambiguity)                  │   │ │
│   │   │  • 에러 전파 제한                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  단점:                                                 │   │ │
│   │   │  • 에러 전파 가능 (1비트 에러 → 2비트 에러)             │   │ │
│   │   │  • 초기 상태 필요                                      │   │ │
│   │   │  • 복호화 복잡도 증가                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 차등 부호화는 위상 동기가 어려운 시스템에서 필수적입니다.

**💡 비유**: 차등 부호화는 "이전과 비교해 변했는가?"로 정보를 전달합니다.

**📢 섹션 요약 비유**: NRZ-I, 차분 맨체스터, DPSK!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **전이** | 1 표현 | 레벨 변화 | 토글 |
| **유지** | 0 표현 | 레벨 유지 | 지속 |
| **이전 상태** | 기준 | 직전 비트 | 참조 |
| **극성 독립** | 장점 | 반전 영향 없음 | 강점 |

**인코딩/디코딩 알고리즘**

```
인코딩:
  이전 출력 ⊕ 현재 입력 = 현재 출력

디코딩:
  이전 수신 ⊕ 현재 수신 = 현재 데이터

XOR 연산 기반
```

**핵심 알고리즘**: 이전 ⊕ 입력 → 출력, 수신 ⊕ 이전수신 → 데이터

**📢 섹션 요약 비유**: XOR이 핵심!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 응용 분야**

| 응용 | 방식 | 목적 |
|:---|:---|:---|
| **USB** | NRZ-I | 극성 독립 |
| **토큰링** | Diff Manchester | 위상 모호성 |
| **위성 PSK** | DPSK | 위상 기준 불확실 |

**📢 섹션 요약 비유**: 위성 통신은 DPSK 필수!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: USB** → NRZ-I (차등)
**시나리오 2: 위성** → DPSK
**시나리오 3: 광통신** → 차등 검출

**도입 체크리스트**: 위상 동기, 극성, 에러 전파

**📢 섹션 요약 비유**: 코히어런트 검출 어려우면 차등!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 위상 모호성 해결

**미래 전망**: 코히어런트 수신으로 이동

**참고 표준**: USB, IEEE 802.5

**📢 섹션 요약 비유**: 차등 부호화는 위상의 해결사!
