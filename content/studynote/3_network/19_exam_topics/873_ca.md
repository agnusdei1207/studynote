# CA - 인증 기관

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인증서 발급자
> 2. **가치**: 신뢰 보증
> 3. **융합:** Root CA, Intermediate CA, Trust Chain

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CA (Certificate Authority)는 디지털 인증서를 발급합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CA (Certificate Authority)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CA 계층 구조                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Root CA:                                               │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ - 자체 서명 (Self-signed)                        ││   │ │
│   │   │  │ - 브라우저/OS에 내장                              ││   │ │
│   │   │  │ - 오프라인 보관                                   ││   │ │
│   │   │  │ 예: DigiCert, Let's Encrypt, GlobalSign         ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                      │                               │   │ │
│   │   │                      ↓                               │   │ │
│   │   │  Intermediate CA:                                       │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ - Root CA가 서명                                 ││   │ │
│   │   │  │ - 실제 발급 업무 수행                             ││   │ │
│   │   │  │ - 보안: Root CA 보호                              ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                      │                               │   │ │
│   │   │                      ↓                               │   │ │
│   │   │  End Entity:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ - 서버, 클라이언트, 코드                          ││   │ │
│   │   │  │ - Intermediate CA가 서명                          ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  주요 CA:                                               │   │ │
│   │   │  - DigiCert, Sectigo, GlobalSign, GoDaddy            │   │ │
│   │   │  - Let's Encrypt (무료)                               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: CA는 디지털 신뢰의 근간입니다.

**📢 섹션 요약 비유:** CA = 여권 발급소!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Root CA** | 루트 인증 기관 |
| **Intermediate** | 중간 인증 기관 |
| **Trust Chain** | 신뢰 체인 |
| **CPS** | 인증 실무 선언 |
| **Audit** | 감사 |

**핵심 알고리즸:** 요청 → 검증 → 서명 → 발급

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**CA 비교**

| CA | 유형 | 특징 |
|:---|:---|:---|
| **Let's Encrypt** | 무료 | 자동화 |
| **DigiCert** | 상용 | 높은 신뢰 |
| **Internal CA** | 사설 | 내부용 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 2:** 기업 → DigiCert
**시나리오 3:** 내부 → Internal CA
