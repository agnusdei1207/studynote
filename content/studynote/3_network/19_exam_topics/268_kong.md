# Kong - API 게이트웨이 플랫폼

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오픈소스 API Gateway
> 2. **가치**: 플러그인, 확장성
> 3. **융합**: K8s, Service Mesh와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Kong은 Nginx 기반의 고성능 API Gateway 플랫폼입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Kong                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Kong 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           Kong Gateway                       │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌─────────────────────────────────────┐    │    │   │ │
│   │   │  │  │  Core (Nginx + Lua)                 │    │    │   │ │
│   │   │  │  └─────────────────────────────────────┘    │    │   │ │
│   │   │  │       │                                      │    │   │ │
│   │   │  │       ↓                                      │    │   │ │
│   │   │  │  ┌─────────────────────────────────────┐    │    │   │ │
│   │   │  │  │  Plugin Chain                       │    │    │   │ │
│   │   │  │  │  • 인증 (JWT, OAuth2, Key Auth)    │    │    │   │ │
│   │   │  │  │  • 속도 제한 (Rate Limiting)        │    │    │   │ │
│   │   │  │  │  • 로깅 (File, Syslog)             │    │    │   │ │
│   │   │  │  │  • 변환 (Request/Response)         │    │    │   │ │

│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Kong 구성요소                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
```

> **해설**: Kong은 Nginx 기반으로 플러그인으로 기능을 확장합니다.

**💡 비유**: Kong은 레고 블록 같은 API Gateway입니다.

**📢 섹션 요약 비유:** Kong = 플러그인 기반!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Service** | 업스트림 API |
| **Route** | 라우팅 규칙 |
| **Plugin** | 기능 확장 |
| **Consumer** | API 사용자 |

**비교표: Kong 에디션**

```
+------------+----------+----------+
| 에디션     │ 기능     │ 용도     │
+------------+----------+----------+
| OSS        │ 기본     │ 오픈소스 │
| Enterprise │ 풀       │ 기업     │
| Konnect    │ 클라우드 │ 관리형   │
+------------+----------+----------+
```

**핵심 알고리즸:** 플러그인 체인, 라우팅

**📢 섹션 요약 비유:** Plugin = 확장!
