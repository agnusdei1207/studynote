# Infrastructure as Code - 코드로서의 인프라

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인프라를 코드로 관리
> 2. **가치**: 재현성, 버전 관리
> 3. **융합:** Terraform, GitOps, CI/CD

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Infrastructure as Code (IaC)는 인프라를 코드로 정의합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Infrastructure as Code                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IaC 워크플로우                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Code (Terraform):                                   │   │ │
│   │   │  resource "aws_instance" "web" {                    │   │ │
│   │   │    ami           = "ami-12345"                      │   │ │
│   │   │    instance_type = "t3.micro"                       │   │ │
│   │   │  }                                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Workflow:                                           │   │ │
│   │   │  ┌─────────┐    ┌─────────┐    ┌─────────┐         │   │ │
│   │   │  │  Write  │ ──→│  Plan   │ ──→│  Apply  │         │   │ │
│   │   │  │  Code   │    │  (Dry)  │    │ (Real)  │         │   │ │
│   │   │  └─────────┘    └─────────┘    └─────────┘         │   │ │
│   │   │       │                               │             │   │ │
│   │   │       ↓                               ↓             │   │ │
│   │   │  ┌─────────┐                    ┌─────────┐        │   │ │
│   │   │  │   Git   │                    │   AWS   │        │   │ │
│   │   │  │ Version │                    │  Cloud  │        │   │ │
│   │   │  └─────────┘                    └─────────┘        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IaC는 인프라를 코드처럼 관리합니다.

**📢 섹션 요약 비유:** IaC = 레고 조립说明书!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Declarative** | 선언적 |
| **Idempotent** | 멱등성 |
| **State** | 상태 |
| **Plan** | 계획 |
| **Apply** | 적용 |

**핵심 알고리즸:** 코드 → Plan → 검증 → Apply

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**IaC 도구**

| 도구 | 유형 | 설명 |
|:---|:---|:---|
