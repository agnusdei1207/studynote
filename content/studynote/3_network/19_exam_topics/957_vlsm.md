# VLSM - 가변 길이 서브넷 마스크

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 유연한 서브넷 크기
> 2. **가치**: 주소 낭비 최소화
> 3. **융합:** CIDR, Hierarchical, Efficient

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

VLSM (Variable Length Subnet Mask)은 다양한 크기의 서브넷을 허용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    VLSM                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VLSM 설계                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  예시 (192.168.1.0/24):                                 │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  본사 (100호스트): 192.168.1.0/25                ││   │ │
│   │   │  │    192.168.1.1 - 192.168.1.126 (126개)           ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  지점1 (50호스트): 192.168.1.128/26               ││   │ │
│   │   │  │    192.168.1.129 - 192.168.1.190 (62개)           ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  지점2 (20호스트): 192.168.1.192/27               ││   │ │
│   │   │  │    192.168.1.193 - 192.168.1.222 (30개)           ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  링크 (2호스트): 192.168.1.224/30                 ││   │ │
│   │   │  │    192.168.1.225 - 192.168.1.226 (2개)            ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                  │   │ │
│   │   │  - 필요한 만큼만 할당                                    │   │ │
│   │   │  - 주소 낭비 최소화                                      │   │ │
│   │   │  - 계층적 설계 가능                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: VLSM은 IP 주소 효율성을 극대화합니다.

**📢 섹션 요약 비유:** VLSM = 맞춤형 방 크기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Variable** | 가변 |
| **Requirement** | 요구사항 |
| **Allocation** | 할당 |
| **Hierarchy** | 계층 |
| **Efficiency** | 효율성 |

**핵심 알고리듬:** 큰 것부터 → 순차 할당

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**FLSM vs VLSM**

| 항목 | FLSM | VLSM |
|:---|:---|:---|
| **마스크** | 고정 | 가변 |
| **효율성** | 낮음 | 높음 |
| **복잡도** | 낮음 | 높음 |
| **지원** | RIP v1 | RIP v2, OSPF |
