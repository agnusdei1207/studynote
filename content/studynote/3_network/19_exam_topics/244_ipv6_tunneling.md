# IPv6 터널링 - IPv4 네트워크에서 IPv6 전송

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPv6 패킷을 IPv4로 캡슐화
> 2. **가치**: 전환 기간 연결성 보장
> 3. **융합**: 6to4, Teredo, ISATAP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IPv6 터널링은 IPv6 패킷을 IPv4 패킷으로 캡슐화하여 IPv4 네트워크를 통해 전송하는 기술입니다.

> **해설**: 터널링은 IPv4 네트워크를 통해 IPv6를 전송합니다.

**💡 비유**: 터널링은 IPv6 편지를 IPv4 봉투에 넣어 보내는 것입니다.

**📢 섹션 요약 비유:** 터널링 = IPv4 위에 IPv6!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 기술 | 주소 형식 | 용도 |
|:---|:---|:---|
| **6to4** | 2002:IPv4:: | 공인 |
| **ISATAP** | ::0:5efe:IPv4 | 사내 |
| **Teredo** | 2001:: | NAT 통과 |
| **6rd** | ISP 접두사 | ISP |

**비교표: 터널링 비교**

```
+------------+----------+----------+
| 기술       │ NAT 통과 │ 용도     │
+------------+----------+----------+
| 6to4       │ X        │ 공인 IP  │
| ISATAP     │ X        │ 사내망   │
| Teredo     │ O        │ NAT 환경 │
+------------+----------+----------+
```

**핵심 알고리즸:** 캡슐화/역캡슐화

**📢 섹션 요약 비유:** Teredo = NAT 통과!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 전환 기술**

| 기술 | 설명 |
|:---|:---|
| **Dual Stack** | IPv4/IPv6 동시 |
| **터널링** | IPv6 over IPv4 |
| **변환** | NAT64/DNS64 |

**📢 섹션 요약 비유:** Dual Stack = 권장!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: ISP** → 6rd

**도입 체크리스트:** NAT 여부, 보안, 성능

**📢 섹션 요약 비유:** 터널링 = 임시 방편!

### Ⅴ. 기대효과 및 결론 (Future & Standard)
