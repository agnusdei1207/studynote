# OpenFlow - SDN 표준 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SDN 제어부-데이터부 통신 프로토콜
> 2. **가치**: 표준화, 프로그래밍 가능
> 3. **융합**: Flow Table, Match-Action

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

OpenFlow는 SDN 스위치와 컨트롤러 간 통신 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    OpenFlow 아키텍처                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              OpenFlow 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Controller                                  │    │   │ │
│   │   │  │ - 전역 뷰                                   │    │   │ │
│   │   │  │ - 플로우 규칙 설치                          │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                     │ OpenFlow                      │   │ │
│   │   │                     ▼                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ OpenFlow Switch                            │    │   │ │
│   │   │  │                                             │    │   │ │
│   │   │  │  Flow Table:                               │    │   │ │
│   │   │  │  ┌─────────┬─────────┬─────────┐          │    │   │ │
│   │   │  │  │ Match   │ Action  │ Counter │          │    │   │ │
│   │   │  │  ├─────────┼─────────┼─────────┤          │    │   │ │
│   │   │  │  │ 헤더    │ 포워딩  │ 통계    │          │    │   │ │
│   │   │  │  │ 필드    │ 수정    │ 카운트  │          │    │   │ │

│   │   │  │ Ingress    │ 입력 포트│ port 1   │               │   │ │

└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: OpenFlow는 패킷 처리 규칙을 중앙에서 제어합니다.

**💡 비유**: OpenFlow는 교통 신호 제어 시스템입니다.

**📢 섹션 요약 비유:** OpenFlow = SDN 통신!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Flow Table** | 플로우 규칙 테이블 |
| **Match Fields** | 매칭 필드 |
| **Actions** | 수행 동작 |
| **Counters** | 통계 카운터 |

**비교표: OpenFlow 버전**

```
+------------+----------+----------+----------+
| 버전       │ 특징     │ 테이블   │ 기능     │
+------------+----------+----------+----------+
| 1.0        │ 기본     │ 단일     │ 기본     │
| 1.3        │ 확장     │ 다중     │ 그룹     │
| 1.5        │ 고급     │ 계층     │ 번들     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 패턴 매칭, 우선순위

**📢 섹션 요약 비유:** Flow Table = 규칙 저장소!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: Actions**

| Action | 설명 | 용도 |
|:---|:---|:---|
| **Output** | 포트 전송 | 포워딩 |
| **Drop** | 폐기 | 필터링 |
| **Modify** | 헤더 수정 | NAT |
| **Group** | 그룹 처리 | 로드밸런싱 |

**📢 섹션 요약 비유:** Match-Action = 패킷 처리!
