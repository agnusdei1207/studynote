# TCP Close - TCP 연결 종료

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 4-way Handshake 종료
> 2. **가치**: 양방향 독립 종료
> 3. **융합:** FIN, ACK, TIME_WAIT, RST

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TCP Close는 4-way Handshake로 연결을 종료합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TCP Close (4-way Handshake)                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              정상 종료                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  클라이언트                    서버                   │   │ │
│   │   │     │                            │                    │   │ │
│   │   │     │ ─── FIN ─────────────────→ │ (1)               │   │ │
│   │   │     │ ←── ACK ────────────────── │ (2)               │   │ │
│   │   │     │ ←── FIN ────────────────── │ (3)               │   │ │
│   │   │     │ ─── ACK ─────────────────→ │ (4)               │   │ │
│   │   │     │                            │                    │   │ │
│   │   │  TIME_WAIT → CLOSED         CLOSED                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 양쪽 모두 FIN을 보내야 완전히 종료됩니다.

**📢 섹션 요약 비유:** TCP Close = 이별 인사!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 단계 | 패킷 | 의미 |
|:---|:---|:---|
| **1** | FIN | 전송 완료 |
| **2** | ACK | 수신 확인 |
| **3** | FIN | 반대편 완료 |
| **4** | ACK | 최종 확인 |

**핵심 알고리즸:** FIN ↔ ACK 교환

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**정상 종료 vs 비정상 종료**

| 항목 | FIN | RST |
|:---|:---|:---|
| **순서** | 4-way | 즉시 |
| **상태** | 정리 | 강제 |
| **용도** | 정상 | 오류 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 정상 → FIN 종료
**시나리오 2:** 오류 → RST 종료
**시나리오 3:** Timeout → 강제 종료
