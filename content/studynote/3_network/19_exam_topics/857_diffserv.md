# DiffServ - 차별화 서비스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 코어 기반 QoS
> 2. **가치**: 확장성
> 3. **융합:** DSCP, PHB, Traffic Conditioning

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DiffServ (Differentiated Services)는 네트워크 코어에서 QoS를 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DiffServ                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DiffServ 아키텍처                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Ingress               Core                Egress   │   │ │
│   │   │  ┌──────┐         ┌──────────┐          ┌──────┐   │   │ │
│   │   │  │Class.│         │          │          │      │   │   │ │
│   │   │  │Mark  │ ──────→ │  PHB     │ ───────→ │ Que  │   │   │ │
│   │   │  │Shape │         │  Forward │          │      │   │   │ │
│   │   │  └──────┘         └──────────┘          └──────┘   │   │ │
│   │   │                                                      │   │ │
│   │   │  DSCP (6 bits):                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Class   DSCP    PHB           용도              ││   │ │
│   │   │  │ ──────────────────────────────────────────────  ││   │ │
│   │   │  │ EF      101110  Expedited      음성             ││   │ │
│   │   │  │ AF4x    100xxx  Assured        영상             ││   │ │
│   │   │  │ AF3x    011xxx  Assured        중요 데이터       ││   │ │
│   │   │  │ AF2x    010xxx  Assured        일반 데이터       ││   │ │
│   │   │  │ AF1x    001xxx  Assured        낮은 우선         ││   │ │
│   │   │  │ CS      xxx000  Class Selector  호환성          ││   │ │
│   │   │  │ BE      000000  Best Effort    일반             ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                 │   │ │
│   │   │  - 코어 라우터 부하 적음                                │   │ │
│   │   │  - 확장성 우수                                         │   │ │
│   │   │  - 복잡도 엣지에 집중                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DiffServ은 대규모 네트워크에서 확장 가능한 QoS를 제공합니다.

**📢 섹션 요약 비유:** DiffServ = 항공 클래스!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **DSCP** | DiffServ Code Point |
| **PHB** | Per-Hop Behavior |
| **EF** | Expedited Forwarding |
| **AF** | Assured Forwarding |
| **DS Domain** | DiffServ 도메인 |

**핵심 알고리즸:** 분류 → 마킹 → PHB 적용

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**PHB 유형 비교**

| PHB | 특징 | 보장 |
|:---|:---|:---|
| **EF** | 최저 지연 | 대역폭 |
| **AF** | 우선순위 | 최소 보장 |
| **BE** | 없음 | 없음 |
| **CS** | IP ToS 호환 | 선택적 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 3:** 데이터 → AF21
