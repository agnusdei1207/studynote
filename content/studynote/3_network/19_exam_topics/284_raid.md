# RAID - 디스크 배열

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 디스크 결합
> 2. **가치**: 성능, 안정성
> 3. **융합**: 스토리지, 서버와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RAID(Redundant Array of Independent Disks)는 여러 디스크를 하나로 결합하여 성능과 안정성을 높입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RAID                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RAID 레벨                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 0 (Striping):                                   │   │ │
│   │   │  ┌───┬───┬───┬───┬───┬───┬───┬───┐                │   │ │
│   │   │  │ D0│ D1│ D2│ D3│ D4│ D5│ D6│ D7│                │   │ │
│   │   │  └───┴───┴───┴───┴───┴───┴───┴───┘                │   │ │
│   │   │  [Disk1] [Disk2]                                     │   │ │
│   │   │  • 성능: 최고  • 안정성: 없음                        │   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 1 (Mirroring):                                  │   │ │
│   │   │  ┌───┬───┬───┬───┐    ┌───┬───┬───┬───┐            │   │ │
│   │   │  │ D0│ D1│ D2│ D3│    │ D0│ D1│ D2│ D3│            │   │ │
│   │   │  └───┴───┴───┴───┘    └───┴───┴───┴───┘            │   │ │
│   │   │  [Disk1]              [Disk2]                       │   │ │
│   │   │  • 성능: 읽기 좋음  • 안정성: 높음                  │   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 5 (Striping + Parity):                         │   │ │
│   │   │  ┌───┬───┬───┐  ┌───┬───┬───┐  ┌───┬───┬───┐        │   │ │
│   │   │  │ D0│ D1│ P0│  │ D2│ P1│ D3│  │ P2│ D4│ D5│        │   │ │
│   │   │  └───┴───┴───┘  └───┴───┴───┘  └───┴───┴───┘        │   │ │
│   │   │  [Disk1]        [Disk2]        [Disk3]               │   │ │
│   │   │  • 성능: 좋음  • 안정성: 1디스크 장애 허용          │   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 6 (Double Parity):                             │   │ │
│   │   │  • 성능: 중간  • 안정성: 2디스크 장애 허용           │   │ │
│   │   │                                                      │   │ │
│   │   │  RAID 10 (1+0):                                       │   │ │
│   │   │  • RAID 1 + RAID 0 조합                              │   │ │
│   │   │  • 성능: 높음  • 안정성: 높음                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RAID 레벨에 따라 성능, 안정성, 비용이 다릅니다.

**💡 비유**: RAID는 여러 사람이 짐을 나눠 드는 것입니다.

**📢 섹션 요약 비유:** RAID 5 = 밸런스!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Striping** | 분산 |
| **Mirroring** | 복제 |
| **Parity** | 오류 정정 |

**비교표: RAID 레벨 비교**

```
+------------+----------+----------+----------+
| RAID       │ 최소 디스크│ 장애 허용│ 용도     │
+------------+----------+----------+----------+
| RAID 0     │ 2        │ 0        │ 성능     │
| RAID 1     │ 2        │ 1        │ 안정성   │
| RAID 5     │ 3        │ 1        │ 일반     │
| RAID 6     │ 4        │ 2        │ 중요     │
| RAID 10    │ 4        │ 1+       │ DB       │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** XOR Parity, 스트라이핑

**📢 섹션 요약 비유:** RAID 10 = DB 최적!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 소프트웨어 vs 하드웨어**

| 방식 | 장점 | 단점 |
|:---|:---|:---|
| **소프트웨어** | 저렴 | CPU 사용 |
| **하드웨어** | 빠름 | 비용 |

**📢 섹션 요약 비유:** 하드웨어 RAID = 성능!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: DB 서버** → RAID 10
**시나리오 2: 파일 서버** → RAID 5
**시나리오 3: 아카이빙** → RAID 6

**도입 체크리스트:** 성능, 안정성, 비용

**📢 섹션 요약 비유:** Hot Spare = 자동 복구!
