# HTTP (HyperText Transfer Protocol) - 하이퍼텍스트 전송 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 웹 데이터 전송 프로토콜
> 2. **가치**: 요청-응답, 무상태
> 3. **융합:** TCP, URL, HTML

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HTTP는 클라이언트-서버 간 데이터 전송 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    HTTP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HTTP 요청-응답                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Client                              Server          │   │ │
│   │   │     │                                   │            │   │ │
│   │   │     │ ── HTTP Request ───────────────→ │            │   │ │
│   │   │     │    GET /index.html HTTP/1.1      │            │   │ │
│   │   │     │    Host: www.example.com         │            │   │ │
│   │   │     │                                   │            │   │ │
│   │   │     │ ←── HTTP Response ────────────── │            │   │ │
│   │   │     │    HTTP/1.1 200 OK               │            │   │ │
│   │   │     │    Content-Type: text/html       │            │   │ │
│   │   │     │    <html>...</html>              │            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HTTP는 무상태(Stateless) 요청-응답 모델입니다.

**📢 섹션 요약 비유:** HTTP = 식당 주문!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Request** | 요청 메시지 |
| **Response** | 응답 메시지 |
| **Method** | GET, POST, PUT, DELETE |
| **Header** | 메타데이터 |
| **Body** | 데이터 |
| **Status Code** | 결과 상태 |

**핵심 알고리즸:** 요청 → 처리 → 응답

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**HTTP 버전**

| 버전 | 특징 |
|:---|:---|
| **HTTP/1.0** | 기본 |
| **HTTP/1.1** | Keep-Alive, Pipelining |
| **HTTP/2** | Multiplexing, Header Compression |
| **HTTP/3** | QUIC, UDP |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 3:** 보안 → HTTPS
