# 백업 (Backup) - 데이터 보호

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 복사본 보관
> 2. **가치**: 데이터 손실 방지
> 3. **융합**: DR, 아카이빙과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

백업은 데이터의 복사본을 생성하여 손실 시 복구할 수 있도록 하는 것입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    백업 (Backup)                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              백업 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Full Backup (전체 백업):                          │   │ │
│   │   │     • 모든 데이터 복사                               │   │ │
│   │   │     • 오래 걸림, 많은 공간                           │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Incremental Backup (증분 백업):                   │   │ │
│   │   │     • 마지막 백업 이후 변경분만                       │   │ │
│   │   │     • 빠름, 적은 공간                                │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Differential Backup (차등 백업):                  │   │ │
│   │   │     • 마지막 전체 백업 이후 변경분                    │   │ │
│   │   │     • 중간 복구 시간                                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              3-2-1 백업 규칙                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌───────────┐  ┌───────────┐  ┌───────────┐        │   │ │
│   │   │  │  Copy 1   │  │  Copy 2   │  │  Copy 3   │        │   │ │
│   │   │  │ (Primary) │  │ (On-site) │  │ (Off-site)│        │   │ │
│   │   │  └───────────┘  └───────────┘  └───────────┘        │   │ │
│   │   │                                                      │   │ │
│   │   │  • 3개의 복사본                                      │   │ │
│   │   │  • 2개의 다른 미디어                                  │   │ │
│   │   │  • 1개는 오프사이트                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 백업은 유형에 따라 복구 속도와 공간이 다릅니다.

**💡 비유**: 백업은 보험입니다. 만약을 위해 미리 준비합니다.

**📢 섹션 요약 비유:** 백업 = 데이터 보험!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Full** | 전체 |
| **Incremental** | 증분 |
| **Differential** | 차등 |
| **Snapshot** | 스냅샷 |

**비교표: 백업 유형 비교**

```
+------------+----------+----------+----------+
| 유형       │ 백업 시간│ 복구 시간│ 공간     │
+------------+----------+----------+----------+
| Full       │ 길다     │ 짧다     │ 크다     │
| Incremental│ 짧다     │ 길다     │ 작다     │
| Differential│ 중간    │ 중간     │ 중간     │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 변경 감지, 압축

**📢 섹션 요약 비유:** 3-2-1 규칙 = 안전한 백업!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 백업 대상**

| 대상 | 기술 |
|:---|:---|
| **파일** | rsync, tar |
| **DB** | dump, replica |
| **VM** | image, snapshot |
| **클라우드** | S3, Glacier |

**📢 섹션 요약 비유:** Incremental = 효율적!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 일일 백업** → Full + Incremental
**시나리오 2: 빠른 복구** → Full + Differential
**시나리오 3: 장기 보관** → 아카이빙 + Glacier

**도입 체크리스트:** RPO, 보관 기간, 비용

**📢 섹션 요약 비유:** RPO에 따라 백업 주기 결정!
