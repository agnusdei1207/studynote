# GraphQL - 그래프QL

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 쿼리 언어 기반 API
> 2. **가치**: 유연한 데이터 요청
> 3. **융합:** REST, Schema, Type System

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

GraphQL은 클라이언트가 필요한 데이터를 지정합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    GraphQL                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              GraphQL 동작                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Query (요청):                                       │   │ │
│   │   │  {                                                    │   │ │
│   │   │    user(id: 1) {                                     │   │ │
│   │   │      name                                             │   │ │
│   │   │      email                                            │   │ │
│   │   │      posts {                                          │   │ │
│   │   │        title                                          │   │ │
│   │   │      }                                                │   │ │
│   │   │    }                                                  │   │ │
│   │   │  }                                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  Response (응답):                                     │   │ │
│   │   │  {                                                    │   │ │
│   │   │    "data": {                                          │   │ │
│   │   │      "user": {                                        │   │ │
│   │   │        "name": "John",                                │   │ │
│   │   │        "email": "john@example.com",                   │   │ │
│   │   │        "posts": [{"title": "Hello"}]                  │   │ │
│   │   │      }                                                │   │ │
│   │   │    }                                                  │   │ │
│   │   │  }                                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                               │   │ │
│   │   │  - Over-fetching 방지                                 │   │ │
│   │   │  - Under-fetching 방지                                │   │ │
│   │   │  - 단일 Endpoint                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: GraphQL은 필요한 데이터만 정확히 받습니다.

**📢 섹션 요약 비유:** GraphQL = 주문서!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Query** | 조회 |
| **Mutation** | 수정 |
| **Subscription** | 실시간 구독 |
| **Schema** | 타입 정의 |
| **Resolver** | 데이터 해결 |

**핵심 알고리즸:** Query → Parse → Resolve → Response

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**GraphQL vs REST**

| 항목 | GraphQL | REST |
|:---|:---|:---|
| **Endpoint** | 1개 | 여러 개 |
| **Over-fetch** | 없음 | 있음 |
| **캐시** | 어려움 | 쉬움 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 복잡한 데이터 → GraphQL
