# Prometheus - 시계열 데이터베이스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오픈소스 시계열 모니터링
> 2. **가치**: 풀 모델, 메트릭 수집
> 3. **융합**: Grafana, Alertmanager와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Prometheus는 시계열 데이터를 수집하고 저장하는 모니터링 시스템입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Prometheus                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Prometheus 아키텍처                               │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Targets]                                          │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ /metrics (Pull)                                │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           Prometheus Server                 │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • Retrieval (수집)                          │    │   │ │
│   │   │  │  • TSDB (저장)                               │    │   │ │
│   │   │  │  • PromQL (쿼리)                             │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       ├─────▶ [Grafana] 시각화                       │   │ │
│   │   │       │                                              │   │ │
│   │   │       └─────▶ [Alertmanager] 알림                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              메트릭 유형                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 유형       │ 설명     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Counter    │ 증가만   │ 요청 수   │               │   │ │
│   │   │  │ Gauge      │ 증감     │ 온도     │               │   │ │
│   │   │  | Histogram  │ 분포     │ 지연     │               │   │ │
│   │   │  │ Summary    │ 분포     │ 백분위   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Prometheus는 Pull 방식으로 메트릭을 수집합니다.

**💡 비유**: Prometheus는 데이터 수집기입니다. 주기적으로 가져옵니다.

**📢 섹션 요약 비유:** Prometheus = Pull + TSDB!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Server** | 서버 |
| **Exporter** | 익스포터 |
| **Alertmanager** | 알림 관리 |
| **Pushgateway** | 푸시 게이트웨이 |

**비교표: Prometheus vs Zabbix**

```
+------------+----------+----------+
| 항목       │ Prometheus│ Zabbix  │
+------------+----------+----------+
| 수집 방식  │ Pull     │ Push    │
| 저장소     │ TSDB     │ SQL     │
| 쿼리       │ PromQL   │ API     │
| 시계열     │ 최적     │ 일반    │
+------------+----------+----------+
```

**핵심 알고리즸:** Scrape, TSDB, PromQL

**📢 섹션 요약 비유:** PromQL = 강력한 쿼리!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: CNCF 모니터링**

| 항목 | Prometheus | Thanos | Cortex |
|:---|:---|:---|:---|
| **규모** | 단일 | 멀티 | 멀티 |
| **저장** | 로컬 | S3 | S3 |
| **복잡도** | 낮음 | 중간 | 높음 |

**📢 섹션 요약 비유:** Prometheus + Grafana = 표준!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 쿠버네티스** → Prometheus Operator
**시나리오 2: 대규모** → Thanos
**시나리오 3: 단일 서버** → Prometheus 단일

**도입 체크리스트:** Exporter, 스크래핑 간격, 보존 기간

**📢 섹션 요약 비유:** K8s = Prometheus 필수!
