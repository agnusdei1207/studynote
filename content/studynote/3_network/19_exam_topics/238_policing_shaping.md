# Policing vs Shaping - 트래픽 제어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트래픽 속도 제한
> 2. **가치**: 대역폭 관리, 혼잡 방지
> 3. **융합**: QoS와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Policing과 Shaping은 트래픽 속도를 제한하는 기술이지만, 동작 방식이 다릅니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Policing vs Shaping                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Policing                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  입력 ────> [Token Bucket] ────> 출력/폐기          │   │ │
│   │   │                  │                                  │   │ │
│   │   │              토큰 있음?                              │   │ │
│   │   │               │                                     │   │ │
│   │   │          Yes  │  No                                 │   │ │
│   │   │           ↓       ↓                                 │   │ │
│   │   │        전송     폐기                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  • 즉시 폐기                                          │   │ │
│   │   │  • 버스트 허용                                        │   │ │
│   │   │  • 재전송 없음                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Shaping                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  입력 ────> [버퍼] ────> [토큰 버킷] ────> 출력     │   │ │
│   │   │              │                                     │   │ │
│   │   │           대기/지연                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  • 버퍼링 후 전송                                     │   │ │
│   │   │  • 평활화 (Smoothing)                                │   │ │
│   │   │  • 지연 발생                                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Policing은 폐기, Shaping은 지연시킵니다.

**💡 비유**: Policing은 도로 차단, Shaping은 신호등입니다.

**📢 섹션 요약 비유:** Policing = 폐기, Shaping = 지연!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | Policing | Shaping |
|:---|:---|:---|
| **동작** | 폐기 | 버퍼링 |
| **지연** | 없음 | 있음 |
| **TCP 영향** | 재전송 | 없음 |
| **버스트** | 허용 | 평활화 |

**비교표: Token Bucket**

```
+------------+----------+----------+
| 항목       │ CIR      │ CBS      │
+------------+----------+----------+
| 의미       │ 평균 속도 │ 버스트   │
| 단위       │ bps      │ bytes    │
+------------+----------+----------+
```

**핵심 알고리즸:** Token Bucket, Leaky Bucket

**📢 섹션 요약 비유:** Token = 전송 권한!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 적용 위치**

| 위치 | 기술 |
|:---|:---|
| **Ingress** | Policing |
| **Egress** | Shaping |
| **ISP 경계** | Policing |
| **WAN 링크** | Shaping |

**📢 섹션 요약 비유:** 수신 = Policing, 송신 = Shaping!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: ISP 연결** → Policing
**시나리오 2: WAN 최적화** → Shaping
**시나리오 3: TCP 트래픽** → Shaping 권장

**도입 체크리스트:** CIR, CBS, 방향

**📢 섹션 요약 비유:** TCP = Shaping, UDP = Policing!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대역폭 관리

**미래 전망:** SDN 기반 동적 제어

**참고 표준:** RFC 2697, RFC 2698

**📢 섹션 요약 비유:** Policing/Shaping은 대역폭 관리의 핵심!
