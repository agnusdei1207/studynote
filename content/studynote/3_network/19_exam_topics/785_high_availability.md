# High Availability - 고가용성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서비스 중단 최소화
> 2. **가치**: 신뢰성, 비즈니스 연속성
> 3. **융합:** Redundancy, Failover, Load Balancer

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

High Availability는 서비스 중단을 최소화합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    High Availability                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HA 구성요소                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  가용성 계산:                                          │   │ │
│   │   │  Availability = (MTBF / (MTBF + MTTR)) × 100%        │   │ │
│   │   │                                                      │   │ │
│   │   │  99.9% = 8.76시간/년 중단                              │   │ │
│   │   │  99.99% = 52.56분/년 중단                              │   │ │
│   │   │  99.999% = 5.26분/년 중단                              │   │ │
│   │   │                                                      │   │ │
│   │   │  HA 원칙:                                              │   │ │
│   │   │  1. Redundancy (이중화)                                │   │ │
│   │   │  2. Failover (장애 조치)                               │   │ │
│   │   │  3. Monitoring (모니터링)                              │   │ │
│   │   │  4. Recovery (복구)                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HA는 이중화와 빠른 복구로 가용성을 높입니다.

**📢 섹션 요약 비유:** HA = 예비 타이어!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **MTBF** | 평균 고장 간격 |
| **MTTR** | 평균 복구 시간 |
| **Redundancy** | 이중화 |
| **SPOF** | 단일 실패 지점 |
| **SLA** | 서비스 수준 협약 |

**핵심 알고리즸:** 이중화 → 모니터링 → 장애 감지 → 복구

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**HA 아키텍처**

| 아키텍처 | 설명 |
|:---|:---|
| **Active-Active** | 동시 운영 |
| **Active-Passive** | 대기 운영 |
| **N+1** | N개 + 1 예비 |
| **Cluster** | 클러스터 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 99.9% → Active-Passive
**시나리오 2:** 99.99% → Active-Active
**시나리오 3:** 99.999% → 지역 이중화
