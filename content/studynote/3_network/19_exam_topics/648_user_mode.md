# User Mode - 사용자 모드

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제한된 실행 모드
> 2. **가치**: 시스템 보호, 격리
> 3. **융합:** Kernel Mode, API, System Call

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

User Mode는 제한된 권한으로 실행되는 모드입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    User Mode                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              User Mode 실행                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  User Mode (Ring 3)                                  │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Application                                  │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  - 제한된 명령어만 실행                      │    │   │ │
│   │   │  │  - 할당된 메모리만 접근                      │    │   │ │
│   │   │  │  - I/O 직접 접근 불가                        │    │   │ │
│   │   │  │  - 하드웨어는 System Call 통해              │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │            ↓ System Call                    │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                    ↓                                 │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Kernel Mode (Ring 0)                        │    │   │ │
│   │   │  │ - 하드웨어 접근                             │    │   │ │
│   │   │  │ - 커널 서비스 수행                          │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                    ↓                                 │   │ │
│   │   │                   Return                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: User Mode는 커널을 통해서만 하드웨어에 접근합니다.

**📢 섹션 요약 비유:** User Mode = 일반 사용자!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Restricted Instructions** | 제한 명령어 |
| **Virtual Memory** | 가상 메모리 |
| **API** | 응용 프로그래밍 인터페이스 |
| **Library Call** | 라이브러리 호출 |
| **Context Switch** | 문맥 교환 |

**핵심 알고리즸:** API 호출 → Library → System Call → Kernel
