### Ⅰ. 개요 (Context & Background)

**개념 정의**

Chaos Engineering은 고의로 장애를 유발합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Chaos Engineering                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 정상 상태 정의 (Steady State)                     │   │ │
│   │   │     └── 시스템가 정상적으로 동작하는 기준               │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 가설 수립                                         │   │ │
│   │   │     └── "X 장애가 발생해도 정상 유지"                  │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 실험 설계                                         │   │ │
│   │   │     └── 장애 주입 시나리오                             │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 장애 주입                                         │   │ │
│   │   │     ┌─────────────────────────────────────┐          │   │ │
│   │   │     │ - 서버 종료                          │          │   │ │
│   │   │     │ - 네트워크 지연                      │          │   │ │
│   │   │     │ - CPU/메모리 과부하                  │          │   │ │
│   │   │     │ - DB 연결 실패                      │          │   │ │
│   │   │     └─────────────────────────────────────┘          │   │ │
│   │   │                                                      │   │ │
│   │   │  5. 결과 분석 및 개선                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 카오스 엔지니어링은 장애 대비력을 높입니다.
