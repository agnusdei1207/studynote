# LDAPS - 보안 LDAP

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SSL/TLS로 보호된 LDAP
> 2. **가치**: 암호화된 디렉터리 접근
> 3. **융합**: 인증, Active Directory

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

LDAPS(LDAP over SSL/TLS)는 암호화된 LDAP 연결입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LDAPS                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LDAPS vs LDAP                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ LDAPS (636)          LDAP (389)               │   │ │
│   │   │     │     │                   │                      │   │ │
│   │   │     │     ▼                   ▼                      │   │ │
│   │   │     │  [TLS 핸드셰이크]    [평문]                    │   │ │
│   │   │     │     │                   │                      │   │ │
│   │   │     │     ▼                   ▼                      │   │ │
│   │   │     │  [암호화 채널]       [평문 채널]               │   │ │
│   │   │     │     │                   │                      │   │ │
│   │   │     │     ▼                   ▼                      │   │ │
│   │   │     │  [LDAP Server]       [LDAP Server]             │   │ │
│   │   │     │                                                │   │ │
│   │   │     └────────────────────────────────────────────────┘   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LDAPS 특징                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 특징       │ LDAP     │ LDAPS    │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 포트       │ 389      │ 636      │               │   │ │
│   │   │  │ 암호화     │ 없음     │ TLS/SSL  │               │   │ │
│   │   │  | StartTLS   │ 지원     │ 지원     │               │   │ │
│   │   │  │ 인증서     │ 불필요   │ 필요     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: LDAPS는 포트 636에서 TLS/SSL로 암호화합니다.

**💡 비유**: LDAPS는 보안 전화번호부입니다. 내용을 암호화합니다.

**📢 섹션 요약 비유:** LDAPS = 보안 LDAP!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **포트 636** | LDAPS 기본 포트 |
| **TLS/SSL** | 암호화 계층 |
| **인증서** | 서버 인증서 |
| **StartTLS** | 업그레이드 방식 |

**비교표: LDAPS vs StartTLS**

```
+------------+----------+----------+
| 항목       │ LDAPS    │ StartTLS │
+------------+----------+----------+
| 포트       │ 636      │ 389      │
| 연결       │ 암호화   │ 업그레이드│
| 호환성     │ 제한     │ 높음     │
| 설정       │ 간단     │ 복잡     │
+------------+----------+----------+
```

**핵심 알고리즸:** TLS 핸드셰이크, 인증서 검증

**📢 섹션 요약 비유:** 보안 = LDAPS!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 보안 프로토콜**

| 항목 | LDAPS | HTTPS | SFTP |
|:---|:---|:---|:---|
| **용도** │ 디렉터리 │ 웹 │ 파일 |
| **포트** │ 636 │ 443 │ 22 |
| **암호화** │ TLS | TLS | SSH |

**📢 섹션 요약 비유:** 디렉터리 보안 = LDAPS!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 보안 인증 → LDAPS
