# BGP (Border Gateway Protocol) - 인터넷 라우팅

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: AS 간 경로 벡터 라우팅
> 2. **가치**: 정책 기반 라우팅, 인터넷 연결
> 3. **융합**: MPLS, 데이터센터와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

BGP(Border Gateway Protocol)는 AS(Autonomous System) 간 경로 정보를 교환하는 EGP입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    BGP (Border Gateway Protocol)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              BGP 특징                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Path Vector Protocol:                            │   │ │
│   │   │     • AS_PATH 속성                                   │   │ │
│   │   │     • 루프 방지                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 정책 기반 라우팅:                                 │   │ │
│   │   │     • 속성 기반 경로 선택                            │   │ │
│   │   │     • 유연한 제어                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  3. TCP 기반:                                        │   │ │
│   │   │     • 포트 179                                      │   │ │
│   │   │     • 신뢰성 보장                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 종류:                                            │   │ │
│   │   │     • eBGP: AS 간                                    │   │ │
│   │   │     • iBGP: AS 내                                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              BGP 속성                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  속성           │ 필수 │ 전달 │ 설명                │   │ │
│   │   │  ──────────────┼──────┼──────┼────────────────│   │ │
│   │   │  ORIGIN        │ O    │ O    │ 경로 출처          │   │ │
│   │   │  AS_PATH       │ O    │ O    │ AS 경로            │   │ │
│   │   │  NEXT_HOP      │ O    │ O    │ 다음 홉            │   │ │
│   │   │  LOCAL_PREF    │ X    │ O    │ AS 내 우선순위      │   │ │
│   │   │  MED           │ X    │ X    │ AS 간 우선순위      │   │ │
│   │   │  COMMUNITY     │ X    │ O    │ 그룹                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: BGP는 AS_PATH로 루프를 방지하고 속성으로 경로를 선택합니다.

**💡 비유**: BGP는 국가 간 우편 배달입니다. 각 국가(AS)를 거치며 최적 경로를 찾습니다.

**📢 섹션 요약 비유:** BGP = 인터넷 라우팅!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 경로 선택 순서 | 속성 |
|:---:|:---|
| 1 | **Weight** (Cisco) |
| 2 | **LOCAL_PREF** |
| 3 | **LOCAL_ORIGIN** |
| 4 | **AS_PATH** |
| 5 | **ORIGIN** |
| 6 | **MED** |
| 7 | **eBGP > iBGP** |
| 8 | **IGP Cost** |

**비교표: eBGP vs iBGP**

```
+------------+----------+----------+
| 항목       │ eBGP     │ iBGP     │
+------------+----------+----------+
| AS         │ 다름     │ 같음     │
│ AD         │ 20       │ 200      │
│ TTL        │ 1        │ 255      │
│ 전파       │ O        │ X*       │
+------------+----------+----------+
```

**핵심 알고리즸:** Path Vector, 속성 기반 선택

**📢 섹션 요약 비유:** LOCAL_PREF = AS 내 최우선!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: AS 번호**

| 유형 | 범위 | 용도 |
|:---|:---|:---|
| **Public** | 1-64511 | 인터넷 |
| **Private** | 64512-65535 | 내부 |
| **4-byte** | 65536+ | 확장 |

**📢 섹션 요약 비유:** Private AS = 64512-65535!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 멀티홈** → BGP + LOCAL_PREF
**시나리오 2: 데이터센터** → BGP EVPN
**시나리오 3: ISP** → Full Routing

**도입 체크리스트:** AS 번호, 피어링, 필터링

**📢 섹션 요약 비유:** BGP 필터링 = 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 인터넷 연결, 정책 제어

**미래 전망:** BGP EVPN, Segment Routing

**참고 표준:** RFC 4271 (BGP-4), RFC 7432 (EVPN)

**📢 섹션 요약 비유:** BGP는 인터넷의 핵심!
