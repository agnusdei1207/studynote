# Restore - 복구

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 백업에서 데이터 복원
> 2. **가치**: 서비스 재개
> 3. **융합:** Backup, DR, RTO

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Restore는 백업에서 데이터를 복원합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Restore                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              복구 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Full Restore:                                        │   │ │
│   │   │  전체 백업 → 전체 복원                                  │   │ │
│   │   │  시간: 김 / 단순                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  Point-in-Time Restore (PITR):                        │   │ │
│   │   │  특정 시점으로 복원                                     │   │ │
│   │   │  Full + Incremental/Differential                      │   │ │
│   │   │                                                      │   │ │
│   │   │  Granular Restore:                                    │   │ │
│   │   │  특정 파일/테이블만 복원                                │   │ │
│   │   │  빠름 / 선택적                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  복구 과정:                                            │   │ │
│   │   │  1. 백업 선택                                          │   │ │
│   │   │  2. 대상 준비                                          │   │ │
│   │   │  3. 데이터 복원                                        │   │ │
│   │   │  4. 무결성 검증                                        │   │ │
│   │   │  5. 서비스 재개                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 복구는 백업 전략에 따라 달라집니다.

**📢 섹션 요약 비유:** Restore = 되돌리기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Full Restore** | 전체 복원 |
| **PITR** | 시점 복원 |
| **Granular** | 부분 복원 |
| **Verification** | 검증 |
