# Cilium - eBPF 기반 네트워킹

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: eBPF 기반 K8s 네트워킹
> 2. **가치**: 관찰 가능성, 보안
> 3. **융합**: Service Mesh, K8s와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Cilium은 eBPF를 활용하여 Kubernetes 환경에서 네트워킹, 보안, 관찰 가능성을 제공합니다.

> **해설**: Cilium은 eBPF를 통해 K8s 네트워킹을 최적화합니다.

**💡 비유**: Cilium은 K8s 네트워크의 스마트 교통 시스템입니다.

**📢 섹션 요약 비유:** Cilium = eBPF K8s!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Cilium Agent** | 데몬 |
| **Cilium Operator** | 관리 |
| **Hubble** | 관찰 |
| **Identity** | 보안 ID |

**비교표: Cilium vs Kube-Proxy**

```
+------------+----------+----------+
| 항목       │ Kube-Proxy│ Cilium  │
+------------+----------+----------+
| 구현       │ iptables │ eBPF    │
| 성능       │ O(n)     │ O(1)    │
| L7         │ 없음     │ 지원    │
| 관찰       │ 제한     │ 풍부    │
+------------+----------+----------+
```

**핵심 알고리즸:** Identity, eBPF Map

**📢 섹션 요약 비유:** Identity = 보안 기반!
