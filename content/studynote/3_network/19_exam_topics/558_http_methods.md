# HTTP Methods - HTTP 메서드

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 리소스 조작 방식
> 2. **가치**: RESTful API 설계
> 3. **융합:** CRUD, Idempotent, Safe

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HTTP Methods는 리소스에 대한 동작을 정의합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    HTTP Methods                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HTTP 메서드 분류                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Safe (읽기만):                                      │   │ │
│   │   │  - GET: 리소스 조회                                  │   │ │
│   │   │  - HEAD: 헤더만 조회                                 │   │ │
│   │   │  - OPTIONS: 지원 메서드 조회                         │   │ │
│   │   │                                                      │   │ │
│   │   │  Idempotent (멱등):                                  │   │ │
│   │   │  - PUT: 리소스 전체 교체                             │   │ │
│   │   │  - DELETE: 리소스 삭제                               │   │ │
│   │   │                                                      │   │ │
│   │   │  Non-Idempotent:                                     │   │ │
│   │   │  - POST: 리소스 생성/처리                            │   │ │
│   │   │  - PATCH: 리소스 부분 수정                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 메서드는 Safe, Idempotent 속성을 가집니다.

**📢 섹션 요약 비유:** HTTP Methods = 식당 주문 방식!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 메서드 | CRUD | Safe | Idempotent |
|:---|:---|:---|:---|
| **GET** | Read | O | O |
| **POST** | Create | X | X |
| **PUT** | Update | X | O |
| **PATCH** | Update | X | X |
| **DELETE** | Delete | X | O |
| **HEAD** | Read | O | O |
| **OPTIONS** | Info | O | O |

**핵심 알고리즸:** 메서드 → 동작 → 응답

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**POST vs PUT vs PATCH**

| 항목 | POST | PUT | PATCH |
|:---|:---|:---|:---|
| **용도** | 생성 | 전체 교체 | 부분 수정 |
| **Idempotent** | X | O | X |
| **URL** | 컬렉션 | 특정 리소스 | 특정 리소스 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 조회 → GET
**시나리오 2:** 생성 → POST
**시나리오 3:** 수정 → PUT/PATCH
