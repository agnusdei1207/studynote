# 네트워크 토폴로지 - Network Topology

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 구조 형태
> 2. **가치**: 설계, 확장성
> 3. **융합**: 스패닝 트리, 라우팅과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

네트워크 토폴로지는 장비 간 연결 구조를 정의합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    네트워크 토폴로지                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              물리적 토폴로지                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Star (스타):                                      │   │ │
│   │   │           [Switch]                                   │   │ │
│   │   │          /  |  |  \                                  │   │ │
│   │   │        PC  PC PC  PC                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Bus (버스):                                       │   │ │
│   │   │     PC──PC──PC──PC──PC                               │   │ │
│   │   │                                                      │   │ │
│   │   │  3. Ring (링):                                        │   │ │
│   │   │     PC──▶PC──▶PC                                     │   │ │
│   │   │     ▲         │                                      │   │ │
│   │   │     └─────────┘                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  4. Mesh (메시):                                      │   │ │
│   │   │     모든 노드가 서로 연결                             │   │ │
│   │   │                                                      │   │ │
│   │   │  5. Tree (트리):                                      │   │ │
│   │   │         Core                                         │   │ │
│   │   │        /    \                                        │   │ │
│   │   │      Agg    Agg                                      │   │ │
│   │   │      / \    / \                                      │   │ │
│   │   │    Acc Acc Acc Acc                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              논리적 토폴로지                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  • 데이터 흐름 기반                                   │   │ │
│   │   │  • 논리적 경로                                        │   │ │
│   │   │  • VLAN, 서브넷                                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 토폴로지는 물리적 연결과 논리적 흐름을 정의합니다.

**💡 비유**: 토폴로지는 도시의 도로망입니다. 어떻게 연결되어 있는지 보여줍니다.

**📢 섹션 요약 비유:** 토폴로지 = 네트워크 구조!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Star** | 중앙 집중 |
| **Mesh** | 완전 연결 |
| **Tree** | 계층 |
| **Hybrid** | 혼합 |

**비교표: 토폴로지 장단점**

```
+------------+----------+----------+
| 토폴로지   │ 장점     │ 단점     │
+------------+----------+----------+
| Star       │ 관리 용이│ 단일 실패점│
| Mesh       │ 이중화   │ 비용     │
| Tree       │ 확장성   │ 복잡     │
| Ring       │ 순환     │ 지연     │
+------------+----------+----------+
```

**핵심 알고리즸:** STP, 라우팅

**📢 섹션 요약 비유:** Star = 가장 일반적!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 데이터센터 토폴로지**

| 토폴로지 | 설명 |
|:---|:---|
| **Spine-Leaf** | Clos |
| **3-Tier** | Core-Agg-Access |
| **Collapsed** | 2-Tier |

**📢 섹션 요약 비유:** Spine-Leaf = 현대적!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: LAN** → Star
**시나리오 2: 데이터센터** → Spine-Leaf
**시나리오 3: WAN** → Mesh

**도입 체크리스트:** 규모, 이중화, 비용

**📢 섹션 요약 비유:** 설계 = 요구사항 기반!
