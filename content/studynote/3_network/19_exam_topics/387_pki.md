# PKI - 공개키 기반 구조

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공개키 인증서 관리 프레임워크
> 2. **가치**: 신뢰 체인, 보안 통신
> 3. **융합**: HTTPS, 이메일, 코드 서명

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

PKI(Public Key Infrastructure)는 공개키 인증서를 관리합니다.


> **해설**: PKI는 신뢰 체인을 통해 공개키를 검증합니다.

**💡 비유**: PKI는 공인인증서 시스템입니다. 누가 누구인지 증명합니다.

**📢 섹션 요약 비유:** PKI = 신뢰 체인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **CA** | Certificate Authority |
| **RA** | Registration Authority |
| **CRL** | Certificate Revocation List |
| **OCSP** | Online Certificate Status Protocol |

**비교표: CRL vs OCSP**

```
+------------+----------+----------+
| 항목       │ CRL      │ OCSP     │
+------------+----------+----------+
| 방식       │ 목록     │ 실시간   │
| 크기       │ 큼       │ 작음     │
| 속도       │ 느림     │ 빠름     │
| 최신성     │ 지연     │ 실시간   │
+------------+----------+----------+
```

**핵심 알고리즸:** 서명 검증, 인증서 체인

**📢 섹션 요약 비유:** HTTPS = PKI!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 키 관리**

| 항목 | PKI | PGP | S/MIME |
|:---|:---|:---|:---|
| **모델** │ 계층 │ 신뢰망 │ PKI |
| **용도** │ 범용 │ 이메일 │ 이메일 |
| **관리** │ 중앙 │ 분산 │ 중앙 |

**📢 섹션 요약 비유:** 인증서 = PKI!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹 HTTPS → PKI (공인 CA)
**시나리오 2:** 기업 내부 → 사설 CA
**시나리오 3:** 이메일 → S/MIME

**도입 체크리스트:** CA 선택, 인증서 정책, 폐기 관리

**📢 섹션 요약 비유:** 보안 통신 = PKI!
