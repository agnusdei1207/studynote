# CBWFQ (Class-Based WFQ) - 클래스 기반 가중치 공정 큐잉

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클래스별 대역폭 보장
> 2. **가치**: 유연한 QoS 정책
> 3. **융합**: LLQ, WRED와 통합

### Ⅰ. 개요 (Context & Background)

**개념 정의**

> **해설**: CBWFQ는 클래스별로 대역폭을 보장하고, 남은 대역폭을 공정하게 분배합니다.

**💡 비유**: CBWFQ는 도로 차선 배정입니다. 각 차선이 용도별로 할당됩니다.

**📢 섹션 요약 비유:** CBWFQ = 대역폭 보장!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Class Map** | 트래픽 분류 |
| **Policy Map** | 대역폭 할당 |
| **Service Policy** | 인터페이스 적용 |

**비교표: 큐잉 방식**

```
+------------+----------+----------+
| 방식       │ 기준     │ 특징     │
+------------+----------+----------+
| FIFO       │ 순서     │ 단순     │
| PQ         │ 우선순위 │ 기아     │
| WFQ        │ 플로우   │ 공정     │
| CBWFQ      │ 클래스   │ 보장     │
| LLQ        │ 클래스+PQ│ 실시간   │
+------------+----------+----------+
```

**핵심 알고리즸:** WFQ, 대역폭 보장

**📢 섹션 요약 비유:** CBWFQ + LLQ = 완벽한 QoS!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: VoIP** → LLQ (Priority Queue)
**시나리오 2: 비디오** → CBWFQ + 대역폭 보장
**시나리오 3: 데이터** → CBWFQ + WRED

**도입 체크리스트:** 클래스 정의, 대역폭 합 < 75%, WRED

**📢 섹션 요약 비유:** 대역폭 합 = 75% 이하!
