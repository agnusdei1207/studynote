# GraphQL - 쿼리 언어

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: API 쿼리 언어 및 런타임
> 2. **가치**: 유연한 데이터 조회, 오버페칭 방지
> 3. **융합**: REST, 마이크로서비스와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

GraphQL은 API를 위한 쿼리 언어이자 런타임입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    GraphQL                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              GraphQL 구조                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Query                                          │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           GraphQL Server                    │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  • Schema (타입 정의)                         │    │   │ │
│   │   │  │  • Resolver (데이터 해결)                     │    │   │ │
│   │   │  │  • Query (조회)                              │    │   │ │
│   │   │  │  • Mutation (변경)                           │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ Resolver                                     │   │ │
│   │   │       ▼                                              │   │ │
│   │   │  [Database] [API] [Service]                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              GraphQL 작업                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 작업       │ 설명     │ 예시     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ Query      │ 조회     │ GET      │               │   │ │
│   │   │  │ Mutation   │ 변경     │ POST/PUT │               │   │ │
│   │   │  | Subscription│ 실시간  │ WebSocket│               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │

│   └──────────────────────────────────────────────────────────────┘ │
```

> **해설**: GraphQL은 클라이언트가 필요한 데이터만 요청할 수 있습니다.

**💡 비유**: GraphQL은 뷔페입니다. 원하는 것만 골라서 가져갑니다.

**📢 섹션 요약 비유:** GraphQL = 필요한 것만!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Schema** | 타입 정의 |
| **Resolver** | 데이터 해결사 |
| **Query** | 조회 |
| **Mutation** | 변경 |

**비교표: GraphQL vs REST**

```
+------------+----------+----------+
| 항목       │ GraphQL  │ REST     │
+------------+----------+----------+
| 엔드포인트 │ 1개      │ 여러 개  │
| 오버페칭   │ 없음     │ 있음     │
| 버전관리   │ 없음     │ 있음     │
| 캐싱       │ 복잡     │ 쉬움     │
+------------+----------+----------+
```

**핵심 알고리즸:** 타입 시스템, 해결자

**📢 섹션 요약 비유:** Schema = 계약!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: API 스타일**

| 항목 | GraphQL | REST | gRPC |
|:---|:---|:---|:---|
| **유연성** | 높음 | 중간 | 낮음 |
| **성능** | 중간 | 중간 | 높음 |
| **학습** | 중간 | 낮음 | 높음 |

**📢 섹션 요약 비유:** 복잡한 API = GraphQL!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 복잡한 데이터** → GraphQL
**시나리오 2: 단순 CRUD** → REST
**시나리오 3: 마이크로서비스** → GraphQL Federation

**도입 체크리스트:** 스키마 설계, N+1 문제, 보안

**📢 섹션 요약 비유:** 모바일 = GraphQL!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 효율적 데이터 조회

**참고 표준:** GraphQL Foundation

**📢 섹션 요약 비유:** GraphQL은 API의 미래!
