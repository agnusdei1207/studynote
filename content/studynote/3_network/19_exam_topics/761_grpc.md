# gRPC - 지알피씨

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고성능 RPC 프레임워크
> 2. **가치**: 빠른 직렬화, 양방향 스트리밍
> 3. **융합:** Protocol Buffers, HTTP/2, Microservices

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

gRPC는 고성능 RPC 프레임워크입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    gRPC                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              gRPC 통신 모델                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Client                      Server                  │   │ │
│   │   │  ┌─────────┐                ┌─────────┐             │   │ │
│   │   │  │         │ ── Request ──→ │         │             │   │ │
│   │   │  │ Stub    │ ←─ Response ── │ Service │             │   │ │
│   │   │  │         │                │         │             │   │ │
│   │   │  └─────────┘                └─────────┘             │   │ │
│   │   │                                                      │   │ │
│   │   │  통신 패턴:                                           │   │ │
│   │   │  1. Unary: 1 요청 → 1 응답                           │   │ │
│   │   │  2. Server Streaming: 1 요청 → N 응답                │   │ │
│   │   │  3. Client Streaming: N 요청 → 1 응답                │   │ │
│   │   │  4. Bidirectional: N 요청 ↔ N 응답                   │   │ │
│   │   │                                                      │   │ │
│   │   │  프로토콜: HTTP/2 + Protocol Buffers                 │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: gRPC는 Protocol Buffers로 빠른 직렬화를 제공합니다.

**📢 섹션 요약 비유:** gRPC = 고속 열차!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Proto** | 프로토콜 정의 |
| **Stub** | 클라이언트 스텁 |
| **Channel** | 채널 |
| **Message** | 메시지 |
| **Service** | 서비스 |

**핵심 알고리즸:** Proto 정의 → 코드 생성 → 직렬화 → 전송

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**gRPC vs REST**

| 항목 | gRPC | REST |
|:---|:---|:---|
| **프로토콜** | HTTP/2 | HTTP/1.1 |
| **형식** | Protobuf | JSON |
| **성능** | 빠름 | 느림 |
| **브라우저** | 제한 | 지원 |
