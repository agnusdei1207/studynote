# DHCPv6 - IPv6 동적 주소 할당

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: IPv6용 동적 주소 할당
> 2. **가치**: 중앙 관리, DNS 제공
> 3. **융합**: SLAAC과 혼합 가능

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DHCPv6는 IPv6 네트워크에서 IP 주소와 네트워크 설정을 동적으로 할당하는 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DHCPv6                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DHCPv6 동작                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  클라이언트                DHCPv6 서버               │   │ │
│   │   │     │                         │                      │   │ │
│   │   │     │  1. Solicit (Multicast) │                      │   │ │
│   │   │     │ ───────────────────────>│                      │   │ │
│   │   │     │     (ff02::1:2)         │                      │   │ │
│   │   │     │                         │                      │   │ │
│   │   │     │  2. Advertise           │                      │   │ │
│   │   │     │ <───────────────────────│                      │   │ │
│   │   │     │                         │                      │   │ │
│   │   │     │  3. Request             │                      │   │ │
│   │   │     │ ───────────────────────>│                      │   │ │
│   │   │     │                         │                      │   │ │
│   │   │     │  4. Reply               │                      │   │ │
│   │   │     │ <───────────────────────│                      │   │ │
│   │   │     │                         │                      │   │ │
│   │   │                                                      │   │ │
│   │   │  포트: UDP 546 (클라), 547 (서버)                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DHCPv6 모드                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. Stateful (M=1):                                  │   │ │
│   │   │     • DHCPv6가 주소 할당                             │   │ │
│   │   │     • SLAAC 사용 안함                                │   │ │
│   │   │                                                      │   │ │
│   │   │  2. Stateless (O=1):                                 │   │ │
│   │   │     • SLAAC로 주소 생성                              │   │ │
│   │   │     • DHCPv6로 기타 정보 (DNS)                       │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 혼합 (A=1, M=1):                                 │   │ │
│   │   │     • SLAAC + DHCPv6 모두 가능                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DHCPv6는 Solicit/Advertise/Request/Reply로 동작합니다.

**💡 비유**: DHCPv6는 IPv6용 호텔 체크인입니다.

**📢 섹션 요약 비유:** DHCPv6 = 관리형 IPv6!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | DHCPv4 | DHCPv6 |
|:---|:---|:---|
| **포트** | 67/68 | 546/547 |
| **주소** | 유니캐스트 | 멀티캐스트 |
| **DUID** | 없음 | 있음 |
| **IA** | 없음 | IA_NA, IA_TA |

**비교표: RA 플래그와 DHCPv6**

```
+------------+----------+----------+
| 플래그     │ 설정     │ 동작     │
+------------+----------+----------+
| A=1, M=0   │ SLAAC    │ 자동     │
| A=0, M=1   │ DHCPv6   │ 관리     │
| A=1, O=1   │ 혼합     │ SLAAC+DNS│
+------------+----------+----------+
```

**핵심 알고리즸:** DUID, IA (Identity Association)

**📢 섹션 요약 비유:** O=1 = DNS 등 기타 정보!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: SLAAC vs DHCPv6**

| 항목 | SLAAC | DHCPv6 |
|:---|:---|:---|
| **주소** | 자동 | 관리 |
| **DNS** | RDNSS | O |
| **추적** | 어려움 | 가능 |
| **관리** | 없음 | 중앙 |

**📢 섹션 요약 비유:** 기업 = DHCPv6, 가정 = SLAAC!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 기업** → DHCPv6 (M=1)
**시나리오 2: DNS만** → SLAAC + DHCPv6 (O=1)
**시나리오 3: IoT** → SLAAC

**도입 체크리스트:** RA 플래그, 이중화, DNS

**📢 섹션 요약 비유:** SLAAC + DHCPv6 = 최적!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 중앙 관리, DNS 제공

**미래 전망:** SLAAC + RDNSS로 대체

**참고 표준:** RFC 8415

**📢 섹션 요약 비유:** DHCPv6는 관리형 IPv6의 핵심!
