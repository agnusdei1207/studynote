# Nagle Algorithm - 네이글 알고리즘

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 작은 패킷 지연 병합
> 2. **가치**: 네트워크 효율 향상
> 3. **융합:** TCP, Delayed ACK, SWS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Nagle Algorithm은 작은 패킷을 모아 전송합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Nagle Algorithm                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Nagle 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  데이터: "A" "B" "C" "D" (각 1바이트)               │   │ │
│   │   │                                                      │   │ │
│   │   │  Nagle OFF:                                          │   │ │
│   │   │    → 4개 패킷 (각 41바이트 헤더)                     │   │ │
│   │   │                                                      │   │ │
│   │   │  Nagle ON:                                           │   │ │
│   │   │    → 1개 패킷 ("ABCD", 44바이트)                     │   │ │
│   │   │    → ACK 수신 후 다음 전송                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 작은 데이터를 모아 MSS 단위로 전송합니다.

**📢 섹션 요약 비유:** Nagle = 패킷 합배!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **조건** | 미확인 데이터 없음 OR MSS 도달 |
| **지연** | ACK 대기 |
| **비활성** | TCP_NODELAY |
| **대상** | 소량 데이터 |

**핵심 알고리즘:** if (unacked == 0 || size >= MSS) send

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Nagle ON vs OFF**

| 항목 | ON | OFF |
|:---|:---|:---|
| **지연** | 있음 | 없음 |
| **효율** | 높음 | 낮음 |
| **용도** | 대량 | 실시간 |
