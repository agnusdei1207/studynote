# LDPC (저밀도 패리티 검사)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 희소 행렬로 구성된 선형 블록 코드
> 2. **가치**: Shannon 한계 근접, 5G/WiFi 표준
> 3. **융합**: Tanner 그래프, 메시지 전달, 반복 복호와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

LDPC(Low-Density Parity-Check)는 희소 패리티 검사 행렬을 사용하여 Shannon 한계에 근접하는 성능을 달성하는 오류 정정 코드입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LDPC (저밀도 패리티 검사)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LDPC 특징                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  장점:                                                  │   │ │
│   │   │  • Shannon 한계 0.0045dB 근접                          │   │ │
│   │   │  • 높은 코드 레이트                                     │   │ │
│   │   │  • 병렬 처리 가능                                       │   │ │
│   │   │  • 5G, WiFi 6, DVB-S2 표준                             │   │ │
│   │   │                                                      │   │ │
│   │   │  단점:                                                  │   │ │
│   │   │  • 높은 복호화 복잡도                                   │   │ │

│   │   │  • 구현 난이도                                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              패리티 검사 행렬                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  H = 패리티 검사 행렬 (m × n)                          │   │ │

│   │   │                                                      │   │ │
│   │   │  Tanner 그래프 표현:                                   │   │ │
│   │   │  • 변수 노드 (Variable Nodes): 비트                   │   │ │
│   │   │  • 체크 노드 (Check Nodes): 패리티 방정식              │   │ │
│   │   │  • 엣지: H 행렬의 1                                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
```

> **해설**: LDPC는 반복적 메시지 전달로 복호화합니다.

**💡 비유**: LDPC는 퍼즐을 맞추듯 반복적으로 오류를 수정합니다.

**📢 섹션 요약 비유**: LDPC = Shannon 한계 달성!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **H Matrix** | 검사 | 희소 행렬 | 규칙 |
| **Tanner Graph** | 그래프 | 노드 연결 | 지도 |
| **BP** | 복호화 | 믿음 전파 | 추론 |
| **Iteration** | 반복 | 수렴 | 시도 |

**복호화 과정**

1. 초기화: 채널 LLR
2. 변수→체크: 메시지 전달
3. 체크→변수: 메시지 업데이트
4. 판정: 수렴 확인
5. 반복 또는 종료

**핵심 알고리즘**: Sum-Product → Belief Propagation

**📢 섹션 요약 비유**: 반복할수록 정확!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FEC 코드**

| 코드 | Shannon 한계 | 복잡도 |
|:---|:---:|:---:|
| **Hamming** | 멀다 | 낮음 |
| **Turbo** | 0.5dB | 중간 |
| **LDPC** | 0.0045dB | 높음 |

**📢 섹션 요약 비유**: LDPC가 이론적 한계 최근접!
