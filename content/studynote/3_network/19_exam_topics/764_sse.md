# SSE (Server-Sent Events) - 서버 전송 이벤트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서버→클라이언트 단방향 스트림
> 2. **가치**: 단순함, HTTP 기반
> 3. **융합:** Real-time Updates, Notifications, Feeds

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SSE는 서버에서 클라이언트로 이벤트를 푸시합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SSE (Server-Sent Events)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SSE 통신                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Client Request:                                     │   │ │
│   │   │  GET /events HTTP/1.1                               │   │ │
│   │   │  Accept: text/event-stream                          │   │ │
│   │   │                                                      │   │ │
│   │   │  Server Response:                                    │   │ │
│   │   │  HTTP/1.1 200 OK                                    │   │ │
│   │   │  Content-Type: text/event-stream                    │   │ │
│   │   │  Cache-Control: no-cache                            │   │ │
│   │   │  Connection: keep-alive                             │   │ │
│   │   │                                                      │   │ │
│   │   │  data: {"message": "Hello"}                         │   │ │
│   │   │                                                      │   │ │
│   │   │  data: {"message": "World"}                         │   │ │
│   │   │                                                      │   │ │
│   │   │  Server ──→ Client (지속적 전송)                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SSE는 HTTP로 단방향 스트림을 제공합니다.

**📢 섹션 요약 비유:** SSE = 라디오 방송!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Event** | 이벤트 |
| **Data** | 데이터 |
| **ID** | 식별자 |
| **Retry** | 재시도 |
| **EventSource** | 클라이언트 API |

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SSE vs WebSocket**

| 항목 | SSE | WebSocket |
|:---|:---|:---|
| **방향** | 단방향 | 양방향 |
| **프로토콜** | HTTP | WS |
| **재연결** | 자동 | 수동 |
| **형식** | 텍스트 | 이진/텍스트 |
