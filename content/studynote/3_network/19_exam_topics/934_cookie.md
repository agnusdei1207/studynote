# Cookie - 쿠키

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클라이언트 상태 저장
> 2. **가치**: 사용자 추적과 인증
> 3. **융합:** Set-Cookie, HttpOnly, Secure

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Cookie는 브라우저에 저장되는 작은 데이터입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Cookie                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              쿠키 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  쿠키 전송 과정:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  1. 서버 응답:                                   ││   │ │
│   │   │  │     Set-Cookie: session_id=abc123;             ││   │ │
│   │   │  │                   HttpOnly; Secure; Path=/     ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  2. 브라우저 저장                                 ││   │ │
│   │   │  │     Cookie Store에 저장                          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  3. 이후 요청:                                   ││   │ │
│   │   │  │     Cookie: session_id=abc123                   ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  쿠키 속성:                                              │   │ │
│   │   │  - Expires/Max-Age: 만료 시간                          │   │ │
│   │   │  - Domain: 유효 도메인                                  │   │ │
│   │   │  - Path: 유효 경로                                      │   │ │
│   │   │  - Secure: HTTPS만                                     │   │ │
│   │   │  - HttpOnly: JS 접근 불가                               │   │ │
│   │   │  - SameSite: 크로스 사이트 제한                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 쿠키는 웹 상태 관리의 기본입니다.

**📢 섹션 요약 비유:** Cookie = 방문 증명서!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Name** | 쿠키 이름 |
| **Value** | 쿠키 값 |
| **Expires** | 만료일 |
| **Domain** | 도메인 |
| **Path** | 경로 |

**핵심 알고리듬:** Set-Cookie → 저장 → 자동 전송

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**쿠키 vs 로컬 스토리지**

| 항목 | Cookie | LocalStorage |
|:---|:---|:---|
| **용량** | 4KB | 5MB+ |
| **전송** | 자동 | 수동 |
| **만료** | 설정 가능 | 영구 |
| **접근** | 서버+클라이언트 | 클라이언트 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 인증 → HttpOnly, Secure
**시나리오 2:** 추적 → Third-party Cookie
**시나리오 3:** 설정 → 오래 보관
