# Event Sourcing - 이벤트 소싱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 상태를 이벤트로 저장
> 2. **가치**: 완전한 이력, 감사 추적
> 3. **융합:** CQRS, Event Store, Time Travel

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Event Sourcing은 상태 변경을 이벤트로 저장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Event Sourcing                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              이벤트 소싱 구조                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  传统 방식:                                           │   │ │
│   │   │  State: { id: 1, name: "Kim", balance: 100 }        │   │ │
│   │   │  (현재 상태만 저장)                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  Event Sourcing:                                     │   │ │
│   │   │  Events:                                             │   │ │
│   │   │  1. AccountCreated { id: 1, name: "Kim" }           │   │ │
│   │   │  2. MoneyDeposited { amount: 50 }                   │   │ │
│   │   │  3. MoneyWithdrawn { amount: 30 }                   │   │ │
│   │   │  4. MoneyDeposited { amount: 80 }                   │   │ │
│   │   │                                                      │   │ │
│   │   │  Current State = Replay(Events) = { balance: 100 }  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 이벤트 소싱은 모든 변경 이력을 보존합니다.

**📢 섹션 요약 비유:** Event Sourcing = 회계 장부!
