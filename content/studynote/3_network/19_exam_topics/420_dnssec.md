# DNSSEC - DNS 보안 확장

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: DNS 데이터 무결성/출처 인증
> 2. **가치**: DNS 스푸핑 방지
> 3. **융합**: PKI, 신뢰 체인, 루트 서명

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DNSSEC는 DNS 응답의 무결성과 출처를 보증합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DNSSEC                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DNSSEC 신뢰 체인                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Root Zone] (KSK 서명)                             │   │ │
│   │   │        │                                             │   │ │
│   │   │        │ DS 레코드                                   │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  [TLD Zone] (.com)                                  │   │ │
│   │   │        │                                             │   │ │
│   │   │        │ DS 레코드                                   │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  [Domain Zone] (example.com)                        │   │ │
│   │   │        │                                             │   │ │
│   │   │        │ RRSIG (서명된 레코드)                       │   │ │
│   │   │        ▼                                             │   │ │
│   │   │  [Resolver] ──── 검증 ────▶ 사용자                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DNSSEC 레코드                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 레코드     │ 설명     │ 용도     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ DNSKEY     │ 공개키   │ 검증     │               │   │ │
│   │   │  │ RRSIG      │ 서명     │ 무결성   │               │   │ │
│   │   │  | DS         │ Delegation│ 체인    │               │   │ │
│   │   │  │ NSEC/NSEC3 │ 부재증명 │ NXDOMAIN │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DNSSEC는 기밀성이 아닌 무결성을 제공합니다.

**💡 비유**: DNSSEC는 공문서의 인감입니다. 문서의 진위를 증명합니다.

**📢 섹션 요약 비유:** DNSSEC = DNS 무결성!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **KSK** | Key Signing Key |
| **ZSK** | Zone Signing Key |
| **DS Record** | Delegation Signer |
| **Trust Anchor** | 신뢰 앵커 |

**비교표: DNS vs DNSSEC**

```
+------------+----------+----------+
| 항목       │ DNS      │ DNSSEC   │
+------------+----------+----------+
| 무결성     │ 없음     │ 있음     │
| 출처인증   │ 없음     │ 있음     │
| 기밀성     │ 없음     │ 없음     │
| 스푸핑     │ 취약     │ 방어     │
+------------+----------+----------+
```

**핵심 알고리즸:** 서명 검증, 체인 검증

**📢 섹션 요약 비유:** 기밀성 = DoH/DoT!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: DNS 보안**

| 항목 | DNSSEC | DoH | DoT |
|:---|:---|:---|:---|
| **목적** │ 무결성 │ 기밀성 │ 기밀성 |
| **계층** │ 앱 | 앱 | 트랜스포트|
| **스니핑** │ 방어X | 방어O | 방어O |

**📢 섹션 요약 비유:** DNSSEC + DoH = 완전한 보안!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 기업 DNS → DNSSEC 서명
**시나리오 2:** Resolver → DNSSEC 검증
**시나리오 3:** 개인정보 → DoH + DNSSEC

**도입 체크리스트:** KSK/ZSK 롤오버, DS 등록, 모니터링

**📢 섹션 요약 비유:** 루트 신뢰 = DNSSEC 기반!
