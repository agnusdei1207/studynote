# LDP - 라벨 분배 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: MPLS 라벨 분배
> 2. **가치**: LSP 자동 구성
> 3. **융합:** TCP 646, LSR, FEC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

LDP (Label Distribution Protocol)는 MPLS 라벨을 분배합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LDP                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LDP 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  LDP 메시지 교환:                                       │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  ┌──────┐                ┌──────┐               ││   │ │
│   │   │  │  │ LSR1 │◄──Hello───────►│ LSR2 │               ││   │ │
│   │   │  │  │      │   Discovery    │      │               ││   │ │
│   │   │  │  │      │                │      │               ││   │ │
│   │   │  │  │      │◄──Session─────►│      │               ││   │ │
│   │   │  │  │      │   TCP 646      │      │               ││   │ │
│   │   │  │  │      │                │      │               ││   │ │
│   │   │  │  │      │◄──Label        │      │               ││   │ │
│   │   │  │  │      │   Mapping─────►│      │               ││   │ │
│   │   │  │  └──────┘                └──────┘               ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  LDP 메시지 유형:                                  ││   │ │
│   │   │  │  - Discovery: Hello (UDP)                         ││   │ │
│   │   │  │  - Session: Initialization (TCP)                  ││   │ │
│   │   │  │  - Advertisement: Label Mapping                   ││   │ │
│   │   │  │  - Notification: Error/Withdraw                   ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: LDP는 LSR 간 라벨 매핑을 자동으로 교환합니다.

**📢 섹션 요약 비유:** LDP = 라벨 배달부!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Hello** | 발견 메시지 |
| **Session** | 세션 설정 |
| **Label Mapping** | 라벨 매핑 |
| **FEC** | Forwarding Equivalence Class |
| **LSP** | Label Switched Path |

**핵심 알고리듬:** Hello → Session → Label Mapping → LSP 구성

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**라벨 분배 방식**

| 방식 | 설명 |
|:---|:---|
| **DU** | Downstream Unsolicited |
| **DoD** | Downstream on Demand |
| **Liberal** | 모든 라벨 유지 |
| **Conservative** | 사용 라벨만 유지 |
