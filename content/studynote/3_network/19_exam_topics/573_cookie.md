# Cookie - 쿠키

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 클라이언트 상태 저장
> 2. **가치**: 세션 관리, 추적
> 3. **융합:** HTTP, Session, SameSite

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Cookie는 브라우저에 저장되는 작은 데이터입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Cookie                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Cookie 동작                                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Server → Client:                                    │   │ │
│   │   │  Set-Cookie: session=abc123; Path=/; HttpOnly        │   │ │
│   │   │                                                      │   │ │
│   │   │  Client → Server:                                    │   │ │
│   │   │  Cookie: session=abc123                              │   │ │
│   │   │                                                      │   │ │
│   │   │  속성:                                               │   │ │
│   │   │  - Domain: 적용 도메인                                │   │ │
│   │   │  - Path: 적용 경로                                    │   │ │
│   │   │  - Expires/Max-Age: 만료                              │   │ │
│   │   │  - Secure: HTTPS만                                    │   │ │
│   │   │  - HttpOnly: JS 접근 금지                             │   │ │
│   │   │  - SameSite: CSRF 방지                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Cookie는 HTTP Stateless를 보완합니다.

**📢 섹션 요약 비유:** Cookie = 입장권!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 속성 | 설명 |
|:---|:---|
| **Domain** | 범위 |
| **Path** | 경로 |
| **Secure** | HTTPS |
| **HttpOnly** | XSS 방지 |
| **SameSite** | CSRF 방지 |
| **Max-Age** | 수명 |

**핵심 알고리즸:** Set-Cookie → 저장 → 자동 전송

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Cookie vs Session vs Token**

| 항목 | 저장 | 용도 |
|:---|:---|:---|
| **Cookie** | 클라이언트 | 상태 |
| **Session** | 서버 | 인증 |
| **Token** | 클라이언트 | 인증 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 세션 → Secure + HttpOnly
**시나리오 2:** 추적 → SameSite=None
**시나리오 3:** 보안 → HttpOnly 필수
