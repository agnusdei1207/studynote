# RARP (Reverse ARP) - 역순 주소 결정 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: MAC → IP 주소 변환
> 2. **가치**: 디스크리스 워크스테이션 IP 할당
> 3. **융합:** ARP, BOOTP, DHCP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RARP는 MAC 주소로 IP 주소를 찾습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RARP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RARP 동작                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Diskless Client (MAC: aa:bb:cc:11:22:33)           │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ "내 MAC aa:bb:cc:11:22:33의 IP가 뭐야?"      │   │ │
│   │   │       │ ── RARP Request (Broadcast) ──────────────→  │   │ │
│   │   │       │                                              │   │ │
│   │   │  RARP Server                                         │   │ │
│   │   │       │                                              │   │ │
│   │   │       │ ←── RARP Reply (Unicast) ─────────────────   │   │ │
│   │   │       │ "네 IP는 192.168.1.100이야"                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: RARP는 디스크리스 시스템이 IP를 얻는 데 사용됩니다.

**📢 섹션 요약 비유:** RARP = 주소 문의!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **RARP Request** | MAC → IP 요청 |
| **RARP Reply** | IP 할당 응답 |
| **RARP Server** | MAC-IP 매핑 서버 |
| **L2 Broadcast** | 로컬 네트워크 한정 |

**핵심 알고리즸:** MAC 알림 → 서버 lookup → IP 응답

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ARP vs RARP vs BOOTP vs DHCP**

| 프로토콜 | 요청 | 응답 | 용도 |
|:---|:---|:---|:---|
| **ARP** | IP | MAC | 통신 |
| **RARP** | MAC | IP | IP 획득 |
| **BOOTP** | MAC | IP + 설정 | 부팅 |
| **DHCP** | MAC | IP + 설정 | 동적 할당 |
