# Failover - 페일오버

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장애 시 예비 시스템 전환
> 2. **가치**: 서비스 연속성
> 3. **융합:** HA, Heartbeat, Cluster

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Failover는 장애 발생 시 대체 시스템으로 전환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Failover                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Failover 동작                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  정상:                                                │   │ │
│   │   │  Client ──→ Primary ──── Heartbeat ──→ Standby       │   │ │
│   │   │                                                      │   │ │
│   │   │  장애:                                                │   │ │
│   │   │  Primary ↓ (Fail)                                    │   │ │
│   │   │           Heartbeat 끊김                              │   │ │
│   │   │           ↓                                           │   │ │
│   │   │  Client ──→ Standby (Promote)                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Failback:                                            │   │ │
│   │   │  Primary 복구 → 다시 Primary로                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Failover는 자동 또는 수동으로 수행됩니다.

**📢 섹션 요약 비유:** Failover = 예비 타이어!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Active-Passive** | 대기 후 전환 |
| **Active-Active** | 동시 운영 |
| **Automatic** | 자동 전환 |
| **Manual** | 수동 전환 |

**핵심 알고리즸:** 장애 감지 → 전환 → 서비스 재개

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Failover 방식**

|:---|:---|:---|
