# Traffic Shaping - 트래픽 셰이핑

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트래픽 속도 제어
> 2. **가치**: 대역폭 관리와 혼합 방지
> 3. **융합:** Token Bucket, Leaky Bucket, Policing

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Traffic Shaping은 트래픽 전송 속도를 제어합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Traffic Shaping                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              셰이핑 알고리즘                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Token Bucket:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │         ┌─────────────┐                        ││   │ │
│   │   │  │         │   Bucket    │  Tokens (1/rate)       ││   │ │
│   │   │  │         │  ┌───────┐  │  ───────────→          ││   │ │
│   │   │  │         │  │○○○○○○ │  │                        ││   │ │
│   │   │  │         │  │ Tokens │  │                        ││   │ │
│   │   │  │         │  └───────┘  │                        ││   │ │
│   │   │  │         └──────┬──────┘                        ││   │ │
│   │   │  │                │                                ││   │ │
│   │   │  │     Packet →   │  → Remove token → Transmit    ││   │ │
│   │   │  │                │                                ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  Leaky Bucket:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │  패킷 → [버킷] → 일정 속도로 출력                  ││   │ │
│   │   │  │         (물이 새는 양동이)                         ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Traffic Shaping은 네트워크 혼잡을 방지합니다.

**📢 섹션 요약 비유:** Traffic Shaping = 교통 신호!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Burst** | 버스트 크기 |
| **Rate** | 전송 속도 |
| **Queue** | 대기열 |
| **Token** | 토큰 |
| **Bucket** | 버킷 |

**핵심 알고리듬:** 토큰 생성 → 토큰 소모 → 전송/대기

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Shaping vs Policing**

| 항목 | Shaping | Policing |
|:---|:---|:---|
| **동작** | 버퍼링 | 폐기 |
| **지연** | 발생 | 없음 |
| **용도** | 아웃바운드 | 인바운드 |
| **복구** | 가능 | 불가 |
