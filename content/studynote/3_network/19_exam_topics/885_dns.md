# DNS - 도메인 이름 시스템

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이름 → IP 변환
> 2. **가치**: 사용자 친화적 주소
> 3. **융합:** A, AAAA, CNAME, MX, NS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DNS (Domain Name System)는 도메인 이름을 IP로 변환합니다.

**📢 섹션 요약 비유:** DNS = 인터넷 전화번호부!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Resolver** | 리졸버 |
| **Recursive** | 재귀 쿼리 |
| **Authoritative** | 권한 있는 서버 |
| **Zone** | 영역 |
| **Cache** | 캐시 |

**핵심 알고리즸:** 도메인 → 리졸버 → 쿼리 → IP 반환

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**DNS 레코드 타입**

| 타입 | 용도 | 예시 |
|:---|:---|:---|
| **A** | IPv4 | 1.2.3.4 |
| **AAAA** | IPv6 | 2001:db8::1 |
| **CNAME** | 별칭 | www → example.com |
| **MX** | 메일 | 10 mail.example.com |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹사이트 → A/AAAA 레코드
**시나리오 2:** 이메일 → MX 레코드
**시나리오 3:** CDN → CNAME
