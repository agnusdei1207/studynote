# IPv6 - 차세대 인터넷 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 128비트 주소 공간
> 2. **가치**: 주소 부족 해결, 단순화
> 3. **융합**: NDP, SLAAC, DHCPv6와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IPv6는 128비트 주소를 사용하는 차세대 인터넷 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IPv6 (Internet Protocol version 6)                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IPv6 주소 형식                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  IPv4: 32비트 (4바이트)                              │   │ │
│   │   │  예: 192.168.1.1                                    │   │ │
│   │   │  주소 수: 43억                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  IPv6: 128비트 (16바이트)                            │   │ │
│   │   │  예: 2001:0db8:85a3:0000:0000:8a2e:0370:7334       │   │ │
│   │   │  축약: 2001:db8:85a3::8a2e:370:7334                 │   │ │
│   │   │  주소 수: 340간 (3.4 × 10^38)                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IPv6 주소 유형                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  유형        │ 접두어          │ 용도                │   │ │
│   │   │  ───────────┼────────────────┼───────────────────│   │ │
│   │   │  Unicast    │                 │ 1:1 통신            │   │ │
│   │   │  - GUA      │ 2000::/3        │ 글로벌              │   │ │
│   │   │  - ULA      │ fc00::/7        │ 고유 로컬           │   │ │
│   │   │  - LLA      │ fe80::/10       │ 링크 로컬           │   │ │
│   │   │  Multicast  │ ff00::/8        │ 1:N 통신            │   │ │
│   │   │  Anycast    │                 │ 최근 1:1            │   │ │
│   │   │                                                      │   │ │
│   │   │  참고: 브로드캐스트 없음                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IPv6는 브로드캐스트를 사용하지 않고 멀티캐스트를 사용합니다.

**💡 비유**: IPv6는 모든 모래알에 주소를 부여할 수 있습니다.

**📢 섹션 요약 비유:** IPv6 = 128비트!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | IPv4 | IPv6 |
|:---|:---|:---|
| **주소 길이** | 32비트 | 128비트 |
| **헤더** | 복잡 | 단순 |
| **체크섬** | O | X |
| **NAT** | 필수 | 선택 |
| **IPSec** | 선택 | 필수 |

**비교표: IPv6 기능**

```
+------------+----------+----------+
| 기능       │ IPv6     │ 설명     │
+------------+----------+----------+
| NDP        │ O        │ ARP 대체 │
| SLAAC      │ O        │ 자동 설정│
| DHCPv6     │ O        │ 동적 할당│
| Path MTU   │ O        │ 단편화 X │
+------------+----------+----------+
```

**핵심 알고리즸:** SLAAC, NDP

**📢 섹션 요약 비유:** SLAAC = IPv6 자동 설정!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 전환 기술**

| 기술 | 설명 |
|:---|:---|
| **Dual Stack** | IPv4/IPv6 동시 |
| **Tunneling** | IPv6 over IPv4 |
| **NAT64** | IPv6↔IPv4 변환 |

**📢 섹션 요약 비유:** Dual Stack = 권장!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 신규 네트워크** → IPv6 Only
**시나리오 2: 기존 네트워크** → Dual Stack
**시나리오 3: ISP** → IPv6 + NAT64

**도입 체크리스트:** 장비 지원, 보안, 전환 계획

**📢 섹션 요약 비유:** IPv6 보안 = IPv4와 동일!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)
