# Segment Routing (SR) - 세그먼트 기반 라우팅

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소스 라우팅 진화
> 2. **가치**: 단순화, 트래픽 엔지니어링
> 3. **융합**: MPLS, IPv6와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Segment Routing은 패킷 헤더에 경로 정보를 포함하여 소스에서 경로를 제어하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Segment Routing (SR)                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SR 구조                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Ingress]        [Transit]        [Egress]         │   │ │
│   │   │                                                      │   │ │
│   │   │  세그먼트 리스트:                                    │   │ │
│   │   │  [S1] → [S2] → [S3] → [S4]                          │   │ │
│   │   │                                                      │   │ │
│   │   │  S1: 첫 번째 노드                                    │   │ │
│   │   │  S2: 두 번째 노드                                    │   │ │
│   │   │  S3: 세 번째 노드                                    │   │ │
│   │   │  S4: 목적지                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                │   │ │
│   │   │  • 소스 기반 경로 제어                                │   │ │
│   │   │  • 상태 유지 불필요                                  │   │ │
│   │   │  • IGP 확장 (OSPF, IS-IS)                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SR 유형                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. SR-MPLS:                                          │   │ │
│   │   │     • MPLS 레이블 기반                               │   │ │
│   │   │     • 레이블 스택 = 세그먼트 리스트                  │   │ │
│   │   │     • 기존 MPLS 장비 활용                            │   │ │
│   │   │                                                      │   │ │
│   │   │  2. SRv6:                                             │   │ │
│   │   │     • IPv6 주소 기반                                  │   │ │
│   │   │     • SRH (Segment Routing Header)                   │   │ │
│   │   │     • 네이티브 IPv6                                  │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Segment Routing은 소스에서 경로를 프로그래밍합니다.

**💡 비유**: Segment Routing은 내비게이션의 경유지 설정입니다.

**📢 섹션 요약 비유:** SR = 소스 라우팅!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Prefix SID** | 노드 식별자 |
| **Adjacency SID** | 링크 식별자 |
| **SRGB** | Segment Routing Global Block |
| **SRH** | IPv6 Extension Header |

**비교표: SR-MPLS vs SRv6**

```
+------------+----------+----------+
| 항목       │ SR-MPLS  │ SRv6     │
+------------+----------+----------+
| 캡슐화     │ MPLS     │ IPv6     │
| 식별자     │ 레이블   │ IPv6 주소│
| 오버헤드   │ 낮음     │ 높음     │
| 호환성     │ 기존     │ 새로운   │
+------------+----------+----------+
```

**핵심 알고리즸:** 세그먼트 리스트 처리

**📢 섹션 요약 비유:** SRGB = 16000-23999!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: TE 기술**

| 기술 | 상태 | 복잡도 |
|:---|:---|:---|
| **RSVP-TE** | 유상태 | 높음 |
| **LDP** | 유상태 | 중간 |
| **SR** | 무상태 | 낮음 |

**📢 섹션 요약 비유:** SR = 무상태 TE!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: ISP** → SR-MPLS
**시나리오 2: 데이터센터** → SRv6
**시나리오 3: 5G** → SRv6

**도입 체크리스트:** IGP 확장, SRGB, 컨트롤러

**📢 섹션 요약 비유:** SDN + SR = 완벽!

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 단순화, TE

**미래 전망:** SRv6 확대
