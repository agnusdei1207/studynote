# High Availability - 고가용성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서비스 중단 최소화
> 2. **가치**: 비즈니스 연속성
> 3. **융합:** 99.99%, Redundancy, Failover

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HA (High Availability)는 서비스 중단을 최소화합니다.

```

│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Tier   Availability   Downtime/Year           ││   │ │

│   │   │  │  1      99.9%         8.76 hours               ││   │ │
```

**📢 섹션 요약 비유:** HA = 24시간 영업!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Redundancy** | 이중화 |
| **Uptime** | 가동 시간 |
| **Downtime** | 중단 시간 |
| **SLA** | 서비스 수준 |
| **MTBF** | 평균 고장 간격 |

**핵심 알고리듬:** 가용성 = MTBF / (MTBF + MTTR)

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**HA 구현 방식**

| 방식 | 설명 | 비용 |
|:---|:---|:---|
| **Active-Passive** | 대기 서버 | 중간 |
| **Active-Active** | 동시 운영 | 높음 |
| **Geo-Redundancy** | 지역 이중화 | 매우 높음 |
