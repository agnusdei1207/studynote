# BASE - BASE 속성

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 느슨한 일관성 모델
> 2. **가치**: 높은 가용성
> 3. **융합:** NoSQL, CAP, Eventual Consistency

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

BASE는 NoSQL 시스템의 일관성 모델입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    BASE                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              BASE 속성                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  BA - Basically Available                   │    │   │ │
│   │   │  │      기본적 가용성                           │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  S - Soft State                             │    │   │ │
│   │   │  │      상태 변화 가능                          │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  E - Eventually Consistent                  │    │   │ │
│   │   │  │      결과적 일관성                          │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  vs ACID:                                            │   │ │
│   │   │  ACID: 강한 일관성, 낮은 가용성                      │   │ │
│   │   │  BASE: 약한 일관성, 높은 가용성                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: BASE는 시간이 지나면 일관성을 보장합니다.

**📢 섹션 요약 비유:** BASE = 느슨한 규칙!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Basically Available** | 기본 가용성 |
| **Soft State** | 상태 변화 |
| **Eventually Consistent** | 결과적 일관성 |
| **Tunable Consistency** | 조정 가능 |

**핵심 알고리즸:** 쓰기 → 비동기 복제 → 결과적 일치

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**ACID vs BASE**

| 항목 | ACID | BASE |
|:---|:---|:---|
| **일관성** | 강함 | 약함 |
| **가용성** | 낮음 | 높음 |
| **복잡도** | 높음 | 낮음 |
| **용도** | RDBMS | NoSQL |
