# GitOps - 깃옵스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Git을 단일 진실 공급원으로
> 2. **가치**: 버전 관리, 감사 추적
> 3. **융합:** Kubernetes, ArgoCD, Flux

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

GitOps는 Git을 인프라 상태의 단일 진실 공급원으로 사용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    GitOps                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              GitOps 워크플로우                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Developer                Git Repository             │   │ │
│   │   │  ┌─────────┐            ┌─────────────┐             │   │ │
│   │   │  │  Code   │ ── Commit→ │  Manifests  │             │   │ │
│   │   │  │ Change  │            │  (YAML)     │             │   │ │
│   │   │  └─────────┘            └──────┬──────┘             │   │ │
│   │   │                                │                     │   │ │
│   │   │                                │ Diff                │   │ │
│   │   │                                ↓                     │   │ │
│   │   │                         ┌─────────────┐              │   │ │
│   │   │                         │   ArgoCD    │              │   │ │
│   │   │                         │   Flux      │              │   │ │
│   │   │                         └──────┬──────┘              │   │ │
│   │   │                                │ Sync                │   │ │
│   │   │                                ↓                     │   │ │
│   │   │                         ┌─────────────┐              │   │ │
│   │   │                         │ Kubernetes  │              │   │ │
│   │   │                         │  Cluster    │              │   │ │
│   │   │                         └─────────────┘              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: GitOps는 Git 상태를 클러스터에 자동 동기화합니다.

**📢 섹션 요약 비유:** GitOps = 자동 동기화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Repository** | 저장소 |
| **Manifests** | 매니페스트 |
| **Operator** | 오퍼레이터 |
| **Sync** | 동기화 |
| **Drift Detection** | 드리프트 감지 |

**핵심 알고리즸:** Git 변경 → 감지 → Diff → Sync
