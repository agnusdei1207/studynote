# 소프트스위치 - Softswitch

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소프트웨어 기반 전화 교환기
> 2. **가치**: 유연성, 비용 절감
> 3. **융합**: VoIP, SIP, MGCP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

소프트스위치는 소프트웨어로 전화 교환을 처리합니다.

```

│   │   │  │  Call Agent │ Media Gateway Controller      │    │   │ │
```

**💡 비유**: 소프트스위치는 가상 교환기입니다.

**📢 섹션 요약 비유:** Softswitch = 소프트웨어 교환!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Call Agent** | 호 제어 |
| **MGC** | Media Gateway Controller |
| **MG** | Media Gateway |
| **SG** | Signaling Gateway |

**비교표: 제어 프로토콜**

```
+------------+----------+----------+----------+
| 프로토콜   │ 용도     │ 특징     │ 세대     │
+------------+----------+----------+----------+
| MGCP       │ MG 제어  │ 중앙집중 │ 1세대    │
| H.248/MEGACO│ MG 제어  │ 유연     │ 2세대    │
| SIP        │ 호 제어  │ 분산     │ 3세대    │
+------------+----------+----------+----------+
```

**핵심 알고리즸:** 호 상태 머신, 라우팅

**📢 섹션 요약 비유:** SIP = 현대 표준!

---

### Ⅲ. 융합 비교 및 다각도 분산 (Comparison & Synergy)

**기술 비교: 게이트웨이**

| 게이트웨이 | 용도 | 설명 |
|:---|:---|:---|
| **TG** | Trunk | PSTN 연결 |
| **AG** | Access | analog 전화 |
| **MG** | Media | 코덱 변환 |

**📢 섹션 요약 비유:** TG = PSTN 연결!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 통신사 → 소프트스위치
**시나리오 2:** 기업 → IP-PBX
**시나리오 3:** 클라우드 → CPaaS

**도입 체크리스트:** 프로토콜, 용량, 이중화

**📢 섹션 요약 비유:** 이중화 = 가용성!

### Ⅴ. 기대효과 및 결론 (Future & Standard)
