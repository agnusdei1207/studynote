# CSRF (Cross-Site Request Forgery) - 크로스 사이트 요청 위조

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 의도와 무관한 요청 위조
> 2. **가치**: 권한 도용
> 3. **융합:** CSRF Token, SameSite, OWASP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

CSRF는 사용자가 의도하지 않은 요청을 전송합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CSRF                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              CSRF 공격                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 희생자가 공격자 사이트 방문                       │   │ │
│   │   │  2. 공격자 사이트에서 은행 사이트로 요청 전송         │   │ │
│   │   │     <img src="bank.com/transfer?to=attacker&amt=1000"> │   │
│   │   │  3. 희생자의 쿠키가 자동 포함                         │   │ │
│   │   │  4. 은행은 정상 요청으로 처리                         │   │ │
│   │   │                                                      │   │ │
│   │   │  방어:                                               │   │ │
│   │   │  - CSRF Token                                       │   │ │
│   │   │  - SameSite Cookie                                  │   │ │
│   │   │  - Referer 검증                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: CSRF는 사용자의 인증 상태를 악용합니다.

**📢 섹션 요약 비유:** CSRF = 도장 도용!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **CSRF Token** | 난수 토큰 |
| **SameSite** | 쿠키 정책 |
| **Referer 검증** | 요청 출처 확인 |
| **Double Submit** | 이중 전송 |

**핵심 알고리즸:** 토큰 검증 → 요청 수락

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SameSite 속성**

| 값 | 설명 |
|:---|:---|
| **Strict** | 완전 차단 |
| **Lax** | GET만 허용 |
| **None** | 허용 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 상태 변경 → CSRF Token
**시나리오 2:** 쿠키 → SameSite=Lax
**시나리오 3:** API → Stateless
