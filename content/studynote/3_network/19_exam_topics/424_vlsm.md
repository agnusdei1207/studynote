# VLSM - Variable Length Subnet Mask

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가변 길이 서브넷 마스크
> 2. **가치**: 주소 효율 극대화
> 3. **융합**: CIDR, 서브넷팅, 라우팅

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

VLSM은 서브넷마다 다른 크기의 마스크를 사용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    VLSM                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VLSM 예시                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  원본: 192.168.1.0/24                               │   │ │
│   │   │                                                      │   │ │
│   │   │  VLSM 적용:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ 본사:    192.168.1.0/26   (62 호스트)       │    │   │ │
│   │   │  │ 지사1:   192.168.1.64/27  (30 호스트)       │    │   │ │
│   │   │  │ 지사2:   192.168.1.96/27  (30 호스트)       │    │   │ │
│   │   │  │ 링크1:   192.168.1.128/30 (2 호스트)        │    │   │ │
│   │   │  │ 링크2:   192.168.1.132/30 (2 호스트)        │    │   │ │
│   │   │  │ 예비:    192.168.1.136/29 ...               │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VLSM vs FLSM                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ FLSM     │ VLSM     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 마스크     │ 고정     │ 가변     │               │   │ │
│   │   │  │ 효율       │ 낮음     │ 높음     │               │   │ │
│   │   │  | 낭비       │ 많음     │ 적음     │               │   │ │
│   │   │  │ 복잡도     │ 낮음     │ 높음     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: VLSM은 필요한 호스트 수에 맞춰 서브넷을 할당합니다.

**💡 비유**: VLSM은 옷을 맞춤 제작하는 것입니다. 딱 맞게 만듭니다.

**📢 섹션 요약 비유:** VLSM = 주소 절약!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Variable Mask** | 가변 마스크 |
| **Hierarchical** | 계층적 할당 |
| **Waste Reduction** | 낭비 감소 |
| **Summarization** | 경로 요약 |

**비교표: 서브넷 선택**

```
+------------+----------+----------+
| 호스트수   │ 마스크   │ 낭비     │
+------------+----------+----------+
| 60         │ /26      │ 2        │
| 30         │ /27      │ 0        │
| 2 (링크)   │ /30      │ 0        │
+------------+----------+----------+
```

**핵심 알고리즸:** 큰 것부터 할당, 2의 거듭제곱

**📢 섹션 요약 비유:** 큰 것부터 = VLSM!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 서브넷 크기별 용도**

| 크기 | /30 | /29 | /27 | /24 |
|:---|:---|:---|:---|:---|
| **용도** │ 링크 │ 소규모│ 지사 │ 본사 |
| **호스트** │ 2 | 6 | 30 | 254 |

**📢 섹션 요약 비유:** 링크 = /30!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 기업 네트워크 → VLSM
**시나리오 2:** 링크 주소 → /30
**시나리오 3:** 소규모 사이트 → /29-/27

**도입 체크리스트:** 호스트 수집, 큰 것부터, 문서화

**📢 섹션 요약 비유:** 문서화 = VLSM 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**참고 표준:** RFC 1878
