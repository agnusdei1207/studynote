# Cipher - 암호

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 암호화 알고리즘
> 2. **가치**: 기밀성 보장
> 3. **융합:** Block Cipher, Stream Cipher, AES

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Cipher는 데이터를 암호화하는 알고리즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Cipher                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              암호 유형                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Block Cipher (블록 암호):                              │   │ │
│   │   │  - 고정 크기 블록 단위 암호화                            │   │ │
│   │   │  - AES: 128비트 블록                                   │   │ │
│   │   │  - DES: 64비트 블록                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  Block Mode:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ ECB: 동일 블록 → 동일 암호문 (취약)              ││   │ │
│   │   │  │ CBC: 이전 암호문과 XOR (IV 필요)                 ││   │ │
│   │   │  │ CTR: 카운터 기반, 병렬 처리 가능                  ││   │ │
│   │   │  │ GCM: 인증 포함 암호화                             ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  Stream Cipher (스트림 암호):                           │   │ │
│   │   │  - 비트/바이트 단위 암호화                              │   │ │
│   │   │  - RC4, ChaCha20                                      │   │ │
│   │   │  - 실시간 데이터에 적합                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  비교:                                                 │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Block: 고정 크기, 블록 단위, 병렬 가능           ││   │ │
│   │   │  │ Stream: 가변 크기, 연속, 실시간                  ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 암호는 데이터 기밀성의 핵심 기술입니다.

**📢 섹션 요약 비유:** Cipher = 비밀 코드!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Block** | 블록 |
| **Key** | 키 |
| **IV** | 초기화 벡터 |
| **Mode** | 모드 |
| **Padding** | 패딩 |

**핵심 알고리즸:** 평문 → 키 적용 → 암호문

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**암호화 모드 비교**

| 모드 | 병렬 | 인증 | 용도 |
|:---|:---|:---|:---|
| **ECB** | 가능 | 없음 | X |
| **CBC** | 불가 | 없음 | 파일 |
| **CTR** | 가능 | 없음 | 스트림 |
| **GCM** | 가능 | 있음 | 네트워크 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 파일 → AES-CBC
**시나리오 2:** 네트워크 → AES-GCM
**시나리오 3:** 실시간 → ChaCha20
