# SLAAC (Stateless Address Autoconfiguration) - 무상태 주소 자동 설정

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: RA 기반 자동 주소 생성
> 2. **가치**: DHCP 없이 자동 설정
> 3. **융합**: NDP, EUI-64와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SLAAC(Stateless Address Autoconfiguration)는 라우터 광고(RA)를 기반으로 IPv6 주소를 자동 생성하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SLAAC (Stateless Address Autoconfiguration)       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SLAAC 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 호스트: 링크 로컬 주소 생성 (fe80::/10)          │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 호스트 → 라우터: RS (Router Solicitation)        │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 라우터 → 호스트: RA (Router Advertisement)       │   │ │
│   │   │     • 접두사: 2001:db8:1::/64                        │   │ │
│   │   │     • 플래그: A=1 (SLAAC 가능)                       │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 호스트: 주소 생성                                │   │ │
│   │   │     IPv6 = 접두사 + 인터페이스 ID                    │   │ │
│   │   │     예: 2001:db8:1::a1b2:c3d4:e5f6:1234             │   │ │
│   │   │                                                      │   │ │
│   │   │  5. DAD (중복 주소 감지)                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인터페이스 ID 생성                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  방식 1: EUI-64                                      │   │ │
│   │   │  MAC: 00:11:22:33:44:55                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ID: 0211:22ff:fe33:4455                            │   │ │
│   │   │  (ff:fe 삽입, 7비트 반전)                            │   │ │
│   │   │                                                      │   │ │
│   │   │  방식 2: 랜덤 (Privacy Extension)                     │   │ │
│   │   │  ID: 무작위 생성                                     │   │ │
│   │   │  (보안 강화)                                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SLAAC는 접두사 + 인터페이스 ID로 주소를 생성합니다.

**💡 비유**: SLAAC는 우편번호 + 상세주소로 자동 완성합니다.

**📢 섹션 요약 비유:** SLAAC = DHCP 없이 자동!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Prefix** | RA에서 수신 |
| **Interface ID** | EUI-64 또는 랜덤 |
| **DAD** | 중복 감지 |
| **Privacy** | 임시 주소 |

**비교표: SLAAC vs DHCPv6**

```
+------------+----------+----------+
| 항목       │ SLAAC    │ DHCPv6   │
+------------+----------+----------+
| 상태       │ 무상태   │ 유상태   │
| DNS        │ X*       │ O        │
| 관리       │ 없음     │ 있음     │
| 보안       │ Privacy  │ 필터링   │
+------------+----------+----------+
```

**핵심 알고리즸:** EUI-64, Privacy Extension

**📢 섹션 요약 비유:** Privacy Extension = 보안!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 주소 생성**

| 방식 | 장점 | 단점 |
|:---|:---|:---|
| **EUI-64** | 추적 가능 | 프라이버시 |
| **랜덤** | 프라이버시 | 추적 어려움 |

**📢 섹션 요약 비유:** 랜덤 ID = 프라이버시 보호!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 단순 네트워크** → SLAAC
**시나리오 2: DNS 필요** → SLAAC + RDNSS
**시나리오 3: 관리 필요** → DHCPv6

**도입 체크리스트:** Privacy Extension, RA 설정

**📢 섹션 요약 비유:** SLAAC + RDNSS = 최적!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 자동 설정, 관리 없음

**미래 전망:** Privacy Extension 기본화

**참고 표준:** RFC 4862, RFC 4941

**📢 섹션 요약 비유:** SLAAC는 IPv6 자동 설정의 핵심!
