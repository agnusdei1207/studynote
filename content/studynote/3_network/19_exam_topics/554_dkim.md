# DKIM (DomainKeys Identified Mail) - 도메인키 식별 메일

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이메일 디지털 서명
> 2. **가치**: 위조 방지, 무결성
> 3. **융합:** DNS TXT, 공개키, SPF, DMARC

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DKIM은 발신 도메인이 서명한 이메일을 검증합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DKIM                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DKIM 동작                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  송신측:                                             │   │ │
│   │   │  1. 개인키로 이메일 서명                              │   │ │
│   │   │  2. DKIM-Signature 헤더 추가                         │   │ │
│   │   │                                                      │   │ │
│   │   │  수신측:                                             │   │ │
│   │   │  1. DNS에서 공개키 조회                               │   │ │
│   │   │     selector._domainkey.example.com                  │   │ │
│   │   │  2. 서명 검증                                        │   │ │
│   │   │  3. 위조 여부 판단                                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DKIM은 공개키 암호로 이메일 진위를 검증합니다.

**📢 섹션 요약 비유:** DKIM = 공문 확인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Selector** | 키 식별자 |
| **Private Key** | 서명용 |
| **Public Key** | 검증용 (DNS) |
| **Hash** | SHA-256 |
| **Algorithm** | RSA/Ed25519 |

**핵심 알고리즸:** 해시 → 개인키 서명 → 공개키 검증

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SPF vs DKIM vs DMARC**

| 항목 | 검증 대상 | 방식 |
|:---|:---|:---|
| **SPF** | 발신 IP | 목록 |
| **DKIM** | 내용 | 서명 |
| **DMARC** | 정책 | 통합 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 기업 → DKIM 필수
**시나리오 2:** 키 교체 → Selector 변경
**시나리오 3:** DMARC → DKIM + SPF
