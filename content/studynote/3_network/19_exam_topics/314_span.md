# SPAN/RSPAN - 포트 미러링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스위치 포트 트래픽 복제
> 2. **가치**: 간편한 모니터링
> 3. **융합**: IDS, 분석과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SPAN(Switched Port Analyzer)는 스위치에서 포트 트래픽을 미러링합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SPAN/RSPAN                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SPAN 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Switch]                                           │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Source Port: Gi0/1                              │   │ │
│   │   │     │ (미러링할 포트)                                    │   │ │
│   │   │     │                                                │   │ │
│   │   │     │    ┌─────────────────────────────────────┐     │   │ │
│   │   │     │    │          SPAN Session                │     │   │ │
│   │   │     │    │                                   │     │   │ │
│   │   │     │    │  Source: Gi0/1                   │     │   │ │
│   │   │     │    │  Destination: Gi0/24            │     │   │ │
│   │   │     │    │  Direction: Both                 │     │   │ │
│   │   │     │    └─────────────────────────────────────┘     │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ Destination Port: Gi0/24                          │   │ │
│   │   │     │ (미러링 수신 포트)                                  │   │ │
│   │   │     │                                                │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [Monitor/IDS]                                       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SPAN vs RSPAN                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ SPAN     │ RSPAN    │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 범위       │ 로컬     │ 원격     │               │   │ │
│   │   │  │ VLAN       │ 불필요   │ 필요     │               │   │ │
│   │   │  | 설정       │ 간단     │ 복잡     │               │   │ │
│   │   │  │ 용도       │ 인접     │ 원격     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SPAN은 로컬, RSPAN은 원격 스위치로 트래픽을 미러링합니다.

**💡 비유**: SPAN은 스위치 내부의 CCTV입니다.

**📢 섹션 요약 비유:** SPAN = 간편!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Source** | 원본 포트 |
| **Destination** | 대상 포트 |
| **Direction** | rx/tx/both |
| **Filter** | VLAN 필터 |

**비교표: SPAN 방향**

```
+------------+----------+
| 방향       │ 설명     │
+------------+----------+
| rx         │ 수신만   │
| tx         │ 송신만   │
| both       │ 양방향   │
+------------+----------+
```

**핵심 알고리즸:** 포트 미러링

**📢 섹션 요약 비유:** Both = 양방향!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: SPAN vs TAP**

| 항목 | SPAN | TAP |
|:---|:---|:---|
| **비용** | 낮음 | 높음 |
| **손실** | 있음 | 없음 |
| **유연성** | 제한 | 높음 |

**📢 섹션 요약 비유:** SPAN = 비용 절감!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 로컬 IDS** → SPAN
**시나리오 2: 원격 IDS** → RSPAN
**시나리오 3: 정확한 분석** → TAP

**도입 체크리스트:** 스위치 지원, 비용, 손실

**📢 섹션 요약 비유:** RSPAN = VLAN 필요!
