# GSLB - 글로벌 서버 로드밸런싱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: DNS 기반 글로벌 분산
> 2. **가치**: 지역별 최적 서버 선택
> 3. **융합**: DNS, CDN과 통합

### Ⅰ. 개요 (Context & Background)

**개념 정의**

GSLB(Global Server Load Balancing)는 지리적 위치, 서버 상태를 기반으로 최적 서버를 선택합니다.


> **해설**: GSLB는 사용자 위치와 서버 상태를 기반으로 최적 경로를 선택합니다.

**💡 비유**: GSLB는 GPS 내비게이션입니다. 최적 경로를 안내합니다.

**📢 섹션 요약 비유:** GSLB = 글로벌 최적화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **DNS 기반** | DNS 응답 조작 |
| **Health Check** | 서버 상태 |
| **GeoIP** | 위치 파악 |
| **Metric** | 성능 지표 |

**비교표: GSLB vs DNS LB**

```
+------------+----------+----------+
| 항목       │ DNS LB   │ GSLB     │
+------------+----------+----------+
| 위치 기반  │ 없음     │ 있음     │
| 상태 확인  │ 없음/제한│ 있음     │
| 지능도     │ 낮음     │ 높음     │
| 비용       │ 낮음     │ 높음     │
+------------+----------+----------+
```

**핵심 알고리즸:** GeoIP, RTT 측정

**📢 섹션 요약 비유:** GeoIP = 위치 파악!
