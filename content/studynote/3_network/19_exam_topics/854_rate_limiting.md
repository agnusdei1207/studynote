# Rate Limiting - 속도 제한

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 요청 수 제어
> 2. **가치**: 과부하 방지
> 3. **융합:** Token Bucket, Sliding Window, API Gateway

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Rate Limiting은 요청 빈도를 제한합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Rate Limiting                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              속도 제한 알고리즘                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Fixed Window:                                         │   │ │
│   │   │  ┌─────────┬─────────┬─────────┐                     │   │ │
│   │   │  │  0-1분  │  1-2분  │  2-3분  │                     │   │ │
│   │   │  │ 100 req │ 100 req │ 100 req │                     │   │ │
│   │   │  └─────────┴─────────┴─────────┘                     │   │ │
│   │   │  경계에서 버스트 가능                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  Sliding Window:                                       │   │ │
│   │   │  ┌──────────────────────────────────────────────────┐│   │ │
│   │   │  │        ←── 윈도우 크기 ──→                       ││   │ │
│   │   │  │ [... req ... req ... req ... req ...]            ││   │ │
│   │   │  │         ↑_________________________↑              ││   │ │
│   │   │  │              현재 시점 기준                       ││   │ │
│   │   │  └──────────────────────────────────────────────────┘│   │ │
│   │   │  정확하지만 메모리 많이 사용                           │   │ │
│   │   │                                                      │   │ │
│   │   │  Token Bucket:                                         │   │ │
│   │   │  r = rate (tokens/sec)                                │   │ │
│   │   │  b = burst (bucket size)                              │   │ │
│   │   │                                                      │   │ │
│   │   │  Leaky Bucket:                                         │   │ │
│   │   │  일정 속도로 처리                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  응답:                                                 │   │ │
│   │   │  - 429 Too Many Requests                              │   │ │
│   │   │  - Retry-After 헤더                                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 속도 제한은 서비스 안정성과 공정성을 보장합니다.

**📢 섹션 요약 비유:** Rate Limiting = 입장 제한!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Rate** | 허용 속도 |
| **Burst** | 버스트 허용 |
| **Window** | 시간 윈도우 |
| **Quota** | 할당량 |
| **Throttle** | 제한 |

**핵심 알고리즸:** 요청 → 카운트 → 허용/거부
