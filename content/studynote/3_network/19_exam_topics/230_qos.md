# QoS (Quality of Service) - 서비스 품질

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트래픽 우선순위 및 대역폭 관리
> 2. **가치**: 지연敏感 트래픽 보장
> 3. **융합**: DiffServ, IntServ, CoS와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

QoS(Quality of Service)는 네트워크 트래픽에 우선순위를 부여하여 서비스 품질을 보장하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    QoS (Quality of Service)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              QoS 메커니즘                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 분류 (Classification):                          │   │ │
│   │   │     • ACL, DSCP, CoS, Port                          │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 마킹 (Marking):                                  │   │ │
│   │   │     • DSCP, CoS, IP Precedence                      │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 폴리싱/쉐이핑 (Policing/Shaping):                │   │ │
│   │   │     • 폴리싱: 초과 시 폐기                           │   │ │
│   │   │     • 쉐이핑: 버퍼링                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 혼잡 관리 (Congestion Management):               │   │ │
│   │   │     • WFQ, CBWFQ, LLQ                               │   │ │
│   │   │                                                      │   │ │
│   │   │  5. 혼잡 회피 (Congestion Avoidance):                │   │ │
│   │   │     • WRED                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DSCP 값                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  DSCP  │ PHB        │ 용도                          │   │ │
│   │   │  ─────┼────────────┼───────────────────────────│   │ │
│   │   │  46   │ EF         │ VoIP (지연 최소)              │   │ │
│   │   │  34   │ AF41       │ 화상회의                      │   │ │
│   │   │  26   │ AF31       │ 비디오                        │   │ │
│   │   │  0    │ BE         │ Best Effort                   │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: QoS는 분류, 마킹, 폴리싱, 큐잉 단계로 동작합니다.

**💡 비유**: QoS는 응급실 우선 진료입니다. 위급한 환자가 먼저 치료받습니다.

**📢 섹션 요약 비유:** EF = VoIP 우선!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **지연 (Latency)** | 전송 시간 |
| **지터 (Jitter)** | 지연 변화 |
| **손실 (Loss)** | 패킷 손실 |
| **대역폭** | 가용 속도 |

**비교표: QoS 모델**

```
+------------+----------+----------+
| 모델       │ 특징     │ 스케일   │
+------------+----------+----------+
| IntServ    │ 예약     │ 소규모   │
| DiffServ   │ 클래스    │ 대규모   │
| Best Effort│ 없음     │ 기본     │
+------------+----------+----------+
```

**핵심 알고리즸:** WFQ, CBWFQ, LLQ, WRED

**📢 섹션 요약 비유:** DiffServ = 현대 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: CoS vs DSCP**

| 계층 | 필드 | 비트 |
|:---|:---|:---|
| **L2** | CoS (802.1p) | 3비트 |
| **L3** | DSCP | 6비트 |

**📢 섹션 요약 비유:** L2 = CoS, L3 = DSCP!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: VoIP** → LLQ + EF 마킹
**시나리오 2: 데이터센터** → DiffServ
**시나리오 3: WAN** → 쉐이핑 + 폴리싱

**도입 체크리스트:** 트래픽 분석, 클래스 정의, 정책 수립

**📢 섹션 요약 비유:** 트래픽 분석 = QoS 시작!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 지연 감소, 서비스 보장

**미래 전망:** SDN 기반 QoS

**참고 표준:** RFC 2475 (DiffServ), IEEE 802.1p (CoS)

**📢 섹션 요약 비유:** QoS는 실시간 트래픽의 필수!
