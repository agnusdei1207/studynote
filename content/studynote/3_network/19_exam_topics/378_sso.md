# SSO - Single Sign-On

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 번 로그인, 모든 서비스 접근
> 2. **가치**: 편의성, 보안 강화
> 3. **융합**: SAML, OIDC, Kerberos

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SSO(Single Sign-On)는 한 번의 인증으로 여러 서비스에 접근합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SSO                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SSO 흐름                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [User]                                             │   │ │
│   │   │     │                                                │   │ │
│   │   │     │ 1. 로그인 요청                                  │   │ │
│   │   │     ▼                                                │   │ │
│   │   │  [IdP] ◀────────────────────────────────┐           │   │ │
│   │   │     │                                    │           │   │ │
│   │   │     │ 2. 인증 토큰                       │           │   │ │
│   │   │     ▼                                    │           │   │ │
│   │   │  [SP1] ─── [SP2] ─── [SP3]              │           │   │ │
│   │   │     │        │        │                  │           │   │ │
│   │   │     └────────┴────────┴──────────────────┘           │   │ │
│   │   │              3. 토큰으로 접근                         │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SSO 구성 요소                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 구성요소   │ 역할     │ 프로토콜 │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ IdP        │ 인증     │ SAML/OIDC│               │   │ │
│   │   │  │ SP         │ 서비스   │ SAML/OIDC│               │   │ │
│   │   │  | Token      │ 인증증명 │ JWT/SAML │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SSO는 중앙 집중식 인증으로 사용자 경험을 개선합니다.

**💡 비유**: SSO는 마스터키입니다. 하나로 모든 문을 엽니다.

**📢 섹션 요약 비유:** SSO = 한 번 로그인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **IdP** | Identity Provider |
| **SP** | Service Provider |
| **Assertion** | 인증 정보 |
| **Session** | 세션 관리 |

**비교표: SSO 프로토콜**

```
+------------+----------+----------+
| 항목       │ SAML     │ OIDC     │
+------------+----------+----------+
| 형식       │ XML      │ JSON/JWT │
| 용도       │ 기업     │ 웹/모바일│
| 복잡도     │ 높음     │ 낮음     │
| 최신성     │ 레거시   │ 현대적   │
+------------+----------+----------+
```

**핵심 알고리즸:** 토큰 검증, 세션 동기화

**📢 섹션 요약 비유:** 기업 = SAML!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 인증 방식**

| 항목 | SSO | 개별 인증 | OAuth |
|:---|:---|:---|:---|
| **편의성** │ 높음 │ 낮음 │ 중간 |
| **보안** │ 중앙 │ 분산 │ 위임 |
| **관리** │ 집중 │ 분산 │ 분산 |

**📢 섹션 요약 비유:** 기업 = SSO!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 기업 내부 시스템 → SSO (SAML)
**시나리오 2:** 웹/모바일 앱 → SSO (OIDC)
**시나리오 3:** 레거시 시스템 → Kerberos

**도입 체크리스트:** IdP 선택, 프로토콜, 페더레이션

**📢 섹션 요약 비유:** 중앙 관리 = SSO!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 사용자 편의성, 보안 강화

**미래 전망:** Passwordless, FIDO2

**참고 표준:** SAML 2.0, OpenID Connect

**📢 섹션 요약 비유:** SSO는 기업 인증 표준!
