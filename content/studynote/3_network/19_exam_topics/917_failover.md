# Failover - 장애 조치

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자동 서비스 이전
> 2. **가치**: 서비스 연속성
> 3. **융합:** Active-Passive, Health Check, VIP

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Failover는 장애 발생 시 자동으로 대체 서버로 전환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Failover                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              장애 조치 과정                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  정상 상태:                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │   VIP: 10.0.0.100                              ││   │ │
│   │   │  │        │                                        ││   │ │
│   │   │  │   ┌────┴────┐                                  ││   │ │
│   │   │  │   │ Primary │  ←── Active                      ││   │ │
│   │   │  │   │10.0.0.1 │                                  ││   │ │
│   │   │  │   └─────────┘                                  ││   │ │
│   │   │  │        │ (Heartbeat)                           ││   │ │
│   │   │  │   ┌────┴────┐                                  ││   │ │
│   │   │  │   │ Standby │  ←── Passive                     ││   │ │
│   │   │  │   │10.0.0.2 │                                  ││   │ │
│   │   │  │   └─────────┘                                  ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  장애 발생 시:                                          │   │ │
│   │   │  Primary 다운 → VIP 이동 → Standby가 Active 승격       │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Failover는 무중단 서비스의 핵심입니다.

**📢 섹션 요약 비유:** Failover = 예비 타이어 자동 교체!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Primary** | 주 서버 |
| **Secondary** | 보조 서버 |
| **VIP** | 가상 IP |
| **Heartbeat** | 상태 확인 |
| **Timeout** | 타임아웃 |

**핵심 알고리듬:** Heartbeat 실패 → VIP 이전 → 서비스 재시작

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Failover 유형 비교**

| 유형 | 전환 시간 | 복잡도 |
|:---|:---|:---|
| **Cold** | 분~시간 | 낮음 |
| **Warm** | 초~분 | 중간 |
| **Hot** | 밀리초~초 | 높음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 데이터베이스 → Hot Standby
**시나리오 2:** 웹서버 → Warm Standby
**시나리오 3:** 백업 → Cold Standby
