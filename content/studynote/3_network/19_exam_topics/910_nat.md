# NAT - 네트워크 주소 변환

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사설/공인 주소 변환
> 2. **가치**: IPv4 주소 절약
> 3. **융합:** PAT, Static NAT, Dynamic NAT

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NAT (Network Address Translation)은 사설 IP와 공인 IP를 변환합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NAT                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NAT 동작                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  주소 변환 과정:                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │   내부망               NAT              인터넷   ││   │ │
│   │   │  │   ┌─────┐          ┌─────┐            ┌─────┐  ││   │ │
│   │   │  │   │ PC  │          │     │            │     │  ││   │ │
│   │   │  │   │192. │  ──────→ │변환 │ ────────→  │서버 │  ││   │ │
│   │   │  │   │168. │          │     │            │203. │  ││   │ │

│   │   │  │   └─────┘          └─────┘            └─────┘  ││   │ │
│   │   │  │                     │                           ││   │ │
│   │   │  │              203.0.113.1                       ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
```

> **해설**: NAT는 IPv4 주소 고갈의 해결책입니다.

**📢 섹션 요약 비유:** NAT = 우편물 포워딩 서비스!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Inside Local** | 내부 사설 주소 |
| **Inside Global** | 내부 공인 주소 |
| **Outside Local** | 외부 사설 주소 |
| **Outside Global** | 외부 공인 주소 |
| **NAT Table** | 변환 테이블 |

**핵심 알고리듬:** 패킷 헤더의 IP/포트 수정

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**NAT 유형 비교**

| 유형 | 매핑 | 용도 |
|:---|:---|:---|
| **Static** | 1:1 | 서버 |
| **Dynamic** | N:M | 클라이언트 |
| **PAT** | N:1+Port | 가정 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹 서버 → Static NAT
**시나리오 2:** 가정 → PAT
**시나리오 3:** 기업 → Dynamic NAT
