# WebSocket - 웹소켓

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 양방향 실시간 통신
> 2. **가치**: 낮은 지연, 지속 연결
> 3. **융합:** Real-time Apps, Chat, Gaming

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

WebSocket은 양방향 실시간 통신을 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    WebSocket                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              WebSocket 통신                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Handshake (HTTP Upgrade):                           │   │ │
│   │   │  Client ──→ GET /chat HTTP/1.1                      │   │ │
│   │   │             Upgrade: websocket                       │   │ │
│   │   │             Connection: Upgrade                      │   │ │
│   │   │                                                      │   │ │
│   │   │  Server ──→ HTTP/1.1 101 Switching Protocols        │   │ │
│   │   │             Upgrade: websocket                       │   │ │
│   │   │                                                      │   │ │
│   │   │  Data Frame (양방향):                                 │   │ │
│   │   │  Client ←──→ Server                                  │   │ │
│   │   │        ↑                                               │   │ │
│   │   │   지속 연결                                            │   │ │
│   │   │                                                      │   │ │
│   │   │  장점:                                                │   │ │
│   │   │  - 실시간 양방향                                       │   │ │
│   │   │  - 낮은 오버헤드                                       │   │ │
│   │   │  - HTTP 포트 (80/443) 사용                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: WebSocket은 HTTP 업그레이드로 시작합니다.

**📢 섹션 요약 비유:** WebSocket = 전화 통화!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Handshake** | 핸드셰이크 |
| **Frame** | 프레임 |
| **Message** | 메시지 |
| **Ping/Pong** | 하트비트 |
| **Close** | 종료 |

**핵심 알고리즸:** HTTP Upgrade → 연결 → 프레임 교환 → 종료
