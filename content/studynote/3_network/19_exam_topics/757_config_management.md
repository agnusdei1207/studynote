# Config Management - 설정 관리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 구성 관리
> 2. **가치**: 일관성, 자동화
> 3. **융합:** Ansible, Chef, Puppet

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Config Management는 시스템 구성을 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Config Management                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              설정 관리 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Control Node              Target Nodes             │   │ │
│   │   │  ┌─────────────┐          ┌─────────┐              │   │ │
│   │   │  │  Config     │          │ Server1 │              │   │ │
│   │   │  │  Server     │ ───────→ │ Server2 │              │   │ │
│   │   │  │ ─────────── │          │ Server3 │              │   │ │
│   │   │  │ - Manifests │          │ ...     │              │   │ │
│   │   │  │ - Playbooks │          └─────────┘              │   │ │
│   │   │  │ - Roles     │                                    │   │ │
│   │   │  └─────────────┘                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  목표 상태 (Desired State):                           │   │ │
│   │   │  - 패키지 설치                                        │   │ │
│   │   │  - 파일 배포                                          │   │ │
│   │   │  - 서비스 실행                                        │   │ │
│   │   │  - 사용자 관리                                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 설정 관리는 목표 상태를 정의하고 적용합니다.

**📢 섹션 요약 비유:** Config Management = 공장 조립 라인!
