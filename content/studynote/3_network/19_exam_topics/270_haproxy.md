# HAProxy - 고가용성 프록시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고성능 TCP/HTTP 로드밸런서
> 2. **가치**: 안정성, 세션 유지
> 3. **융합**: Keepalived, K8s와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HAProxy는 고가용성을 위한 TCP/HTTP 로드밸런서 및 프록시입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    HAProxy                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HAProxy 구조                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]                                           │   │ │
│   │   │       │                                              │   │ │
│   │   │       ↓                                              │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │           HAProxy                            │    │   │ │
│   │   │  │                                              │    │   │ │
│   │   │  │  ┌─────────────────────────────────────┐    │    │   │ │
│   │   │  │  │  Frontend (수신)                    │    │    │   │ │
│   │   │  │  │  • bind *:80                        │    │    │   │ │
│   │   │  │  │  • mode http/tcp                   │    │    │   │ │
│   │   │  │  └─────────────────────────────────────┘    │    │   │ │
│   │   │  │       │                                      │    │   │ │
│   │   │  │       ↓                                      │    │   │ │
│   │   │  │  ┌─────────────────────────────────────┐    │    │   │ │
│   │   │  │  │  Backend (분산)                     │    │    │   │ │
│   │   │  │  │  • balance roundrobin              │    │    │   │ │
│   │   │  │  │  • server web1 10.0.0.1:8080       │    │    │   │ │
│   │   │  │  │  • server web2 10.0.0.2:8080       │    │    │   │ │
│   │   │  │  └─────────────────────────────────────┘    │    │   │ │
│   │   │  │                                              │    │   │ │

│   │   │       │          │                                   │   │ │
│   │   │       ↓          ↓                                   │   │ │
│   │   │  [Server 1]  [Server 2]                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HAProxy 기능                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. L4 로드밸런싱:                                    │   │ │
│   │   │     • TCP 모드                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  2. L7 로드밸런싱:                                    │   │ │
│   │   │     • HTTP 모드, 헤더 기반                           │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 헬스 체크:                                        │   │ │
│   │   │     • 활성/수동                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 세션 유지:                                        │   │ │
│   │   │     • sticky sessions                                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HAProxy는 Frontend에서 수신하고 Backend로 분산합니다.

**💡 비유**: HAProxy는 트래픽 교통경찰입니다. 차량을 안전하게 분산합니다.

**📢 섹션 요약 비유:** HAProxy = 안정성!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Frontend** | 수신 지점 |
| **Backend** | 서버 그룹 |
| **ACL** | 접근 제어 |
| **Stick-table** | 세션 저장 |

**비교표: HAProxy 모드**

```
+------------+----------+----------+
| 모드       │ 계층     │ 특징     │
+------------+----------+----------+
| TCP        │ L4       │ 단순     │
| HTTP       │ L7       │ 스마트   │
| Log        │ L4/L7    │ 분석     │
+------------+----------+----------+
```

**핵심 알고리즸:** 로드밸런싱, 헬스체크

**📢 섹션 요약 비유:** Sticky = 세션 유지!
