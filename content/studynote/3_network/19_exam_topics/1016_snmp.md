# SNMP - 단순 네트워크 관리 프로토콜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 관리 프로토콜
> 2. **가치**: 장비 상태 수집
> 3. **융합:** MIB, OID, Trap

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SNMP (Simple Network Management Protocol)는 네트워크 장비를 관리합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SNMP                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SNMP 구조                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  SNMP 아키텍처:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  ┌────────────┐         ┌────────────┐          ││   │ │
│   │   │  │  │    NMS     │◄──Trap──│   Agent    │          ││   │ │
│   │   │  │  │ (Manager)  │         │ (장비)     │          ││   │ │
│   │   │  │  │            │ Get/Set │            │          ││   │ │
│   │   │  │  │            ├────────►│   MIB      │          ││   │ │
│   │   │  │  └────────────┘ Response└────────────┘          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  메시지 유형:                                      ││   │ │
│   │   │  │  - GetRequest: 값 요청                            ││   │ │
│   │   │  │  - GetNextRequest: 다음 값 요청                     ││   │ │
│   │   │  │  - SetRequest: 값 설정                            ││   │ │
│   │   │  │  - GetResponse: 응답                              ││   │ │
│   │   │  │  - Trap: 이벤트 통지                               ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  포트: UDP 161 (Agent), UDP 162 (Trap)             ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SNMP는 장비 상태를 중앙에서 수집합니다.

**📢 섹션 요약 비유:** SNMP = 원격 리모컨!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Manager** | 관리 시스템 |
| **Agent** | 에이전트 |
| **MIB** | Management Information Base |
| **OID** | Object Identifier |
| **Community** | 커뮤니티 문자열 |

**핵심 알고리듬:** 요청 → OID 조회 → 값 반환

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**SNMP 버전 비교**

| 항목 | v1 | v2c | v3 |
|:---|:---|:---|:---|
| **보안** | 없음 | 없음 | 있음 |
| **인증** | Community | Community | USM |
| **암호화** | 없음 | 없음 | 있음 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 내부망 → SNMP v2c
**시나리오 2:** 보안 필요 → SNMP v3
**시나리오 3:** 트래픽 → SNMP + NetFlow
