# 시분할 다중화 (TDM)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시간을 슬롯으로 나누어 순차 전송
> 2. **가치**: 디지털 전송 효율, 대역폭 활용
> 3. **융합**: 전화망, 셀룰러, 데이터 네트워크와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TDM(Time Division Multiplexing)은 전송 시간을 여러 시간 슬롯으로 나누어 각 신호가 할당된 슬롯에만 전송되는 디지털 다중화 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TDM (시분할 다중화)                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TDM 원리                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  시간 슬롯 구조:                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  시간                                                 │   │ │
│   │   │    ├────┼────┼────┼────┼────┼────┼────┼────┤       │   │ │
│   │   │    │CH1 │CH2 │CH3 │CH4 │CH1 │CH2 │CH3 │CH4 │       │   │ │
│   │   │    │ S1 │ S2 │ S3 │ S4 │ S1 │ S2 │ S3 │ S4 │       │   │ │
│   │   │    └────┴────┴────┴────┴────┴────┴────┴────┘       │   │ │
│   │   │         ↑                ↑                          │   │ │
│   │   │      프레임 1         프레임 2                       │   │ │
│   │   │                                                      │   │ │
│   │   │  특징:                                                 │   │ │
│   │   │  • 각 채널에 고정 슬롯 할당                            │   │ │
│   │   │  • 순차적 전송                                        │   │ │
│   │   │  • 동기식/비동기식 존재                                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TDM 송수신 구조                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  송신측:                                                │   │ │
│   │   │  CH1 ─┐                                                 │   │ │
│   │   │  CH2 ─┼─→ [MUX] ─→ [직렬화] ─→ 전송로                 │   │ │
│   │   │  CH3 ─┤        ↑                                       │   │ │
│   │   │  CH4 ─┘      클럭 동기                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  수신측:                                                │   │ │
│   │   │  수신 ─→ [동기 검출] ─→ [DEMUX] ─┬─→ CH1              │   │ │
│   │   │                             ├─→ CH2                    │   │ │
│   │   │                             ├─→ CH3                    │   │ │
│   │   │                             └─→ CH4                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: TDM은 디지털 통신의 핵심 다중화 기술입니다.

**💡 비유**: TDM은 회의에서 발언권을 돌아가며 갖는 것입니다.

**📢 섹션 요약 비유**: TDM = 시간으로 나누기!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **타임슬롯** | 전송 단위 | 고정 길이 | 차례 |
| **프레임** | 슬롯 집합 | N개 슬롯 | 라운드 |
| **동기** | 정렬 | 프레임 동기 | 시작점 |
| **클럭** | 타이밍 | 비트 동기 | 박자 |

**비교표**

```
+------------+----------+----------+
| 항목       | 동기식   | 비동기식 |
+------------+----------+----------+
| 슬롯 할당  | 고정     | 동적     |
| 유휴 슬롯   | 있음     | 없음     |
| 효율       | 낮음     | 높음     |
| 복잡도     | 낮음     | 높음     |
+------------+----------+----------+
```

**핵심 알고리즸**: N입력 → 슬롯 할당 → 순차 전송 → 동기 → 분리

**📢 섹션 요약 비유**: 동기 TDM = 고정 슬롯!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: TDM 계층**

| 계층 | 전송률 | 채널 |
|:---|:---:|:---:|
| **DS-0** | 64kbps | 1 |
| **DS-1 (T1)** | 1.544Mbps | 24 |
| **DS-3 (T3)** | 44.736Mbps | 672 |
| **E1** | 2.048Mbps | 32 |

**📢 섹션 요약 비유**: T1 = 24채널 × 64kbps!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 전화망** → TDM (T1/E1)
**시나리오 2: GSM** → TDMA
**시나리오 3: 데이터망** → 통계적 TDM

**도입 체크리스트**: 동기, 버퍼, 지연

**📢 섹션 요약 비유**: TDM은 디지털 전화망 기반!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 디지털 효율, 동기 전송

**미래 전망**: OTN, FlexE 진화

**참고 표준**: ITU-T G.7xx, G.8xx

**📢 섹션 요약 비유**: TDM은 디지털 시대의 시작!
