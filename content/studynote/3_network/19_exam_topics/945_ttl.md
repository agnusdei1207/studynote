# TTL - 생존 시간

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 패킷 수명 제한
> 2. **가치**: 라우팅 루프 방지
> 3. **융합:** Hop Limit, Traceroute, Expiration

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

TTL (Time To Live)은 패킷이 거칠 수 있는 홉 수를 제한합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    TTL                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              TTL 동작                                           │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  TTL 동작 과정:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  Source                 TTL=64                   ││   │ │
│   │   │  │    │                                            ││   │ │
│   │   │  │    │ Router 1          TTL=63                    ││   │ │
│   │   │  │    │    │                                       ││   │ │
│   │   │  │    │    │ Router 2        TTL=62                ││   │ │
│   │   │  │    │    │    │                                  ││   │ │
│   │   │  │    │    │    │ Router 3  TTL=61                 ││   │ │
│   │   │  │    │    │    │    │                             ││   │ │
│   │   │  │    │    │    │    │ Destination                 ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  각 라우터에서 TTL -= 1                           ││   │ │
│   │   │  │  TTL = 0이면 패킷 폐기 + ICMP Time Exceeded       ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  기본 TTL 값:                                           │   │ │
│   │   │  - Linux: 64                                           │   │ │
│   │   │  - Windows: 128                                        │   │ │
│   │   │  - Cisco: 255                                          │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: TTL은 라우팅 루프를 방지합니다.

**📢 섹션 요약 비유:** TTL = 여권 도장 한도!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Hop** | 홉 |
| **Decrement** | 감소 |
| **Exceeded** | 초과 |
| **ICMP** | 오류 메시지 |
| **Loop** | 루프 |

**핵심 알고리듬:** 라우터 통과 시 TTL-- → 0이면 폐기

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**TTL vs Hop Limit**

| 항목 | IPv4 TTL | IPv6 Hop Limit |
|:---|:---|:---|
| **이름** | Time To Live | Hop Limit |
| **단위** | 홉 | 홉 |
| **기본값** | 64/128 | 64 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
