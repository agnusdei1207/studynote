# HTTP Cache - HTTP 캐시

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: HTTP 수준 캐싱 메커니즘
> 2. **가치**: 대역폭 절약, 응답 속도 향상
> 3. **융합:** Cache-Control, ETag, CDN

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

HTTP Cache는 HTTP 프로토콜 기반 캐싱을 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    HTTP Cache                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              HTTP Cache 헤더                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Request:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ Cache-Control: max-age=3600                 │    │   │ │
│   │   │  │ ETag: "abc123"                              │    │   │ │
│   │   │  │ Last-Modified: Wed, 21 Oct 2015 07:28:00   │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  Validation:                                         │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │ If-None-Match: "abc123" → 304 Not Modified  │    │   │ │
│   │   │  │ If-Modified-Since: ... → 304               │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: HTTP 헤더로 캐시 동작을 제어합니다.

**📢 섹션 요약 비유:** HTTP Cache = 유통기한!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Cache-Control** | 캐시 지시어 |
| **ETag** | 엔티티 태그 |
| **Last-Modified** | 마지막 수정일 |
| **Expires** | 만료일 |
| **Vary** | 변화 기준 |

**핵심 알고리즸:** 요청 → 캐시 확인 → (유효) 반환 / (만료) 재검증

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**Cache-Control 지시어**

| 지시어 | 설명 |
|:---|:---|
| **no-cache** | 재검증 필수 |
| **no-store** | 저장 금지 |
| **public** | 공유 캐시 가능 |
| **private** | 개인 캐시만 |
| **max-age** | 최대 수명 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 정적 리소스 → max-age 설정
**시나리오 2:** 동적 콘텐츠 → no-cache + ETag
**시나리오 3:** 민감 정보 → no-store
