# Authentication - 인증

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 신원 확인
> 2. **가치**: 접근 제어 기반
> 3. **융합:** Password, Token, Biometric

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Authentication은 사용자 신원을 확인합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Authentication                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인증 방식                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  인증 요소 (MFA):                                        │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  1. Something You Know (지식)                     ││   │ │
│   │   │  │     - 비밀번호, PIN                               ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  2. Something You Have (소유)                     ││   │ │
│   │   │  │     - 토큰, 스마트폰, OTP                          ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  3. Something You Are (생체)                      ││   │ │
│   │   │  │     - 지문, 홍채, 얼굴                             ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  4. Somewhere You Are (위치)                      ││   │ │
│   │   │  │     - GPS, IP 기반 위치                           ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  인증 프로토콜:                                          │   │ │
│   │   │  - Basic Auth: ID/PW Base64                          │   │ │
│   │   │  - Bearer Token: OAuth/JWT                            │   │ │
│   │   │  - Certificate: 인증서 기반                             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 인증은 보안의 첫 단계입니다.

**📢 섹션 요약 비유:** Authentication = 신분증 확인!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Credential** | 자격 증명 |
| **Token** | 인증 토큰 |
| **Session** | 세션 |
| **Factor** | 인증 요소 |
| **Principal** | 주체 |

**핵심 알고리듬:** 자격 제출 → 검증 → 토큰 발급

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**인증 vs 인가**

| 항목 | Authentication | Authorization |
|:---|:---|:---|
| **질문** | 누구인가? | 무엇을 할 수 있는가? |
| **시점** | 먼저 | 나중 |
| **결과** | 신원 확인 | 권한 확인 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 웹 → ID/PW + 2FA
**시나리오 2:** API → Bearer Token
**시나리오 3:** 기업 → SSO
