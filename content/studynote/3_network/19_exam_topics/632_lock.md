# Lock - 락

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동시성 제어 메커니즘
> 2. **가치**: 데이터 일관성 보장
> 3. **융합:** Database, Mutex, Semaphore

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Lock은 동시 접근을 제어하는 메커니즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Lock                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Lock 종류                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Shared Lock (S-Lock)        Exclusive Lock (X-Lock)│   │ │
│   │   │  ┌─────────────────┐        ┌─────────────────┐    │   │ │
│   │   │  │   읽기 허용     │        │   읽기/쓰기     │    │   │ │
│   │   │  │   쓰기 차단     │        │   독점          │    │   │ │
│   │   │  │   여러 트랜잭션 │        │   단일 트랜잭션 │    │   │ │
│   │   │  └─────────────────┘        └─────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  행 레벨 Lock           테이블 레벨 Lock            │   │ │
│   │   │  ┌─────────────────┐        ┌─────────────────┐    │   │ │
│   │   │  │   Row Lock      │        │   Table Lock    │    │   │ │
│   │   │  │   세분화        │        │   범위 넓음     │    │   │ │
│   │   │  │   동시성 높음   │        │   동시성 낮음   │    │   │ │
│   │   │  └─────────────────┘        └─────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Lock은 동시성과 일관성 사이의 균형을 관리합니다.

**📢 섹션 요약 비유:** Lock = 화장실 열쇠!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Shared Lock** | 읽기 잠금 |
| **Exclusive Lock** | 쓰기 잠금 |
| **Row Lock** | 행 수준 잠금 |
| **Table Lock** | 테이블 수준 잠금 |
| **Deadlock** | 교착 상태 |

**핵심 알고리즸:** Lock 획득 → 작업 → Lock 해제
