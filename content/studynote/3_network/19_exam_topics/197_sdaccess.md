# SD-Access (Software-Defined Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소프트웨어 정의 기업 LAN
> 2. **가치**: 자동화, 분리, 보안
> 3. **융합**: Cisco DNA, ISE와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SD-Access는 기업 LAN에 SDN 개념을 적용하여 자동화와 분리(Segmentation)를 제공하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SD-Access (Software-Defined Access)               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SD-Access 구조                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [DNA Center] ─────────────────────────────────────  │   │ │
│   │   │       │                    (오케스트레이션)           │   │ │
│   │   │       │                                              │   │ │
│   │   │  [ISE] ──────────────────────────────────────────    │   │ │
│   │   │       │                    (인증/정책)                │   │ │
│   │   │       │                                              │   │ │
│   │   │  [Control Plane] ────────────────────────────────    │   │ │
│   │   │       │                    (LISP)                      │   │ │
│   │   │       │                                              │   │ │
│   │   │  ┌──────────────────────────────────────────────┐   │   │ │
│   │   │  │          Fabric (오버레이)                     │   │   │ │
│   │   │  │   ┌────────┐   ┌────────┐   ┌────────┐       │   │   │ │
│   │   │  │   │VLAN 10 │   │VLAN 20 │   │VLAN 30 │       │   │   │ │
│   │   │  │   │HR 그룹 │   │IT 그룹 │   │Guest   │       │   │   │ │
│   │   │  │   └────────┘   └────────┘   └────────┘       │   │   │ │
│   │   │  │         (VXLAN 기반 마이크로 세그멘테이션)       │   │   │ │
│   │   │  └──────────────────────────────────────────────┘   │   │ │
│   │   │       │                                              │   │ │
│   │   │  [Underlay] ─────────────────────────────────────    │   │ │
│   │   │                   (물리 네트워크)                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SD-Access 특성                                   │ │
│   │                                                              │ │
│   │   장점:                                                   │   │
│   │   • 자동화된 네트워크 프로비저닝                            │   │
│   │   • 사용자/기기 기반 세그멘테이션                            │   │
│   │   • 간소화된 정책 관리                                      │   │
│   │   • 모바일 로밍                                             │   │
│   │   • 가시성 ( Assurance)                                      │   │
│   │                                                      │   │
│   │   기술:                                                   │   │
│   │   • LISP: 위치 식별자 (Control Plane)                        │   │
│   │   • VXLAN: 오버레이 (Data Plane)                            │   │
│   │   • TrustSec: SGT 기반 정책                                 │   │
│   │                                                      │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: SD-Access는 기업 LAN의 자동화와 보안을 동시에 해결합니다.

**💡 비유**: SD-Access는 사무실의 스마트 출입 시스템입니다. 누가 어디로 가는지 자동으로 관리합니다.

**📢 섹션 요약 비유:** SD-Access = 자동화된 LAN!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소 | 역할 | 기술 |
|:---|:---|:---|
| **DNA Center** | 오케스트레이션 | GUI 관리 |
| **ISE** | 인증 | 802.1X, SGT |
| **Control Plane** | 제어 | LISP |
| **Fabric Edge** | 액세스 | VXLAN |
| **Fabric Border** | 게이트웨이 | 외부 연결 |

**비교표**

```
+------------+----------+----------+
| 항목       │ 기존 LAN │ SD-Access│
+------------+----------+----------+
| 프로비저닝│ 수동     │ 자동     │
│ 세그멘테이션│ ACL/VLAN │ SGT     │
│ 로밍       │ 서브넷 제한│ 전역    │
│ 관리       │ 분산     │ 중앙     │
+------------+----------+----------+
```

**핵심 알고리즸:** LISP, VXLAN, SGT

**📢 섹션 요약 비유:** LISP = 위치 식별!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 세그멘테이션**

| 방식 | 기술 | 복잡도 |
|:---|:---|:---:|
| **VLAN** | L2 분리 | 중간 |
| **ACL** | L3 필터 | 높음 |
| **VRF** | L3 분리 | 높음 |
| **SGT** | 태그 기반 | 낮음 |

**📢 섹션 요약 비유:** SGT = 간단한 세그멘테이션!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 대규모 캠퍼스** → SD-Access, 전체 Fabric
**시나리오 2: 소규모** → 기존 LAN, 수동 관리
**시나리오 3: 보안 중요** → SGT 세그멘테이션

**도입 체크리스트:** 규모, 자동화 필요성, 보안 요구

**📢 섹션 요약 비유:** 500대 이상이면 SD-Access!
