# Half-duplex vs Full-duplex

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단방향/양방향 동시 전송
> 2. **가치**: 대역폭 효율, 충돌
> 3. **융합**: 이더넷, 무선, 직렬

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Half-duplex는 한 번에 한 방향, Full-duplex는 양방향 동시 전송입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Half-duplex vs Full-duplex                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              전송 방식                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Half-duplex (반이중):                               │   │ │
│   │   │                                                      │   │ │
│   │   │  [A] ────▶ [B]     송신                             │   │ │
│   │   │  [A] ◀──── [B]     수신                             │   │ │
│   │   │  (동시 불가, CSMA/CD 필요)                           │   │ │
│   │   │                                                      │   │ │
│   │   │  Full-duplex (전이중):                               │   │ │
│   │   │                                                      │   │ │
│   │   │  [A] ◀━━━━▶ [B]    동시 송수신                       │   │ │
│   │   │  (충돌 없음, 2배 대역폭)                              │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              비교표                                             │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 항목       │ Half     │ Full     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │  │ 동시전송   │ 불가     │ 가능     │               │   │ │
│   │   │  │ 충돌       │ 발생     │ 없음     │               │   │ │
│   │   │  | 대역폭     │ 1x       │ 2x       │               │   │ │
│   │   │  │ CSMA/CD    │ 필요     │ 불필요   │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: Full-duplex는 충돌이 없어 대역폭이 2배입니다.

**💡 비유**: Half-duplex는 무전기, Full-duplex는 전화입니다.

**📢 섹션 요약 비유:** Full-duplex = 2배!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **TX/RX** | 송신/수신 |
| **Collision** | 충돌 |
| **Auto-negotiation** | 자동 협상 |
| **Medium** | 전송 매체 |

**비교표: 매체별 지원**

```
+------------+----------+----------+
| 항목       │ Half     │ Full     │
+------------+----------+----------+
| 허브       │ O        │ X        │
| 스위치     │ O        │ O        │
| 무선       │ O        │ X*       │
+------------+----------+----------+
```

**핵심 알고리즸:** 자동 협상, 이중화 감지

**📢 섹션 요약 비유:** 스위치 = Full-duplex!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 이더넷 속도**

| 속도 | Half | Full |
|:---|:---|:---|
| **100Mbps** │ 100 | 200 |
| **1Gbps** │ 드묾 | 2000 |
| **10Gbps** │ X | 20000 |

**📢 섹션 요약 비유:** 1Gbps+ = Full-duplex 필수!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 스위치 포트 → Full-duplex
**시나리오 2:** 무선 → Half-duplex
**시나리오 3:** 레거시 허브 → Half-duplex

**도입 체크리스트:** 자동 협상, 매체 확인, 듀플렉스 불일치

**📢 섹션 요약 비유:** 듀플렉스 불일치 = 성능 저하!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 대역폭 2배

**미래 전망:** 모든 유선은 Full-duplex

**참고 표준:** IEEE 802.3

**📢 섹션 요약 비유:** Full-duplex는 현대 네트워크 표준!
