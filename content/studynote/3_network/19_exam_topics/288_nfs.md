# NFS - 네트워크 파일 시스템

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 파일 공유
> 2. **가치**: 투명한 파일 접근
> 3. **융합**: NAS, 리눅스와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NFS(Network File System)는 네트워크를 통해 원격 파일 시스템에 접근하는 프로토콜입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    NFS                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NFS 구조                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  [Client]            [Client]            [Client]   │   │ │
│   │   │      │                   │                   │       │   │ │
│   │   │      │ NFS Mount         │                   │       │   │ │
│   │   │      │ /mnt/nfs          │                   │       │   │ │
│   │   │      │                   │                   │       │   │ │
│   │   │      └───────────────────┼───────────────────┘       │   │ │
│   │   │                          │                            │   │ │
│   │   │                          │ NFS Protocol (TCP/UDP 2049)│   │ │
│   │   │                          │                            │   │ │

│   │   │                   │ Server   │                        │   │ │
│   │   │                   │ Export   │                        │   │ │
│   │   │                   │ /data    │                        │   │ │
│   │   │                   └──────────┘                        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NFS 버전                                          │ │

│   │   │  │ NFSv3      │ UDP/TCP  │ 상태 없음│               │   │ │

│   │   │  │ NFSv4.2    │ 확장     │ 최신     │               │   │ │
│   │   │  +------------+----------+----------+               │   │ │
│   │   │                                                      │   │ │
```

> **해설**: NFS는 클라이언트가 원격 파일을 로컬처럼 사용합니다.

**💡 비유**: NFS는 네트워크 드라이브입니다. 원격 폴더를 내 컴퓨터처럼 사용합니다.

**📢 섹션 요약 비유:** NFS = 리눅스 파일 공유!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Export** | 공유 디렉토리 |
| **Mount** | 마운트 포인트 |
| **rpcbind** | 포트 매퍼 |
| **idmap** | ID 매핑 |

**비교표: NFSv3 vs NFSv4**

```
+------------+----------+----------+
| 항목       │ NFSv3    │ NFSv4    │
+------------+----------+----------+
| 상태       │ 무상태   │ 유상태   │
| 보안       │ AUTH_SYS │ Kerberos │
| 잠금       │ NLM      │ 내장     │
| 방화벽     │ 복잡     │ 단순     │
+------------+----------+----------+
```

**핵심 알고리즸:** RPC, 캐싱

**📢 섹션 요약 비유:** NFSv4 = 방화벽 친화적!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 파일 공유 프로토콜**

| 프로토콜 | OS | 특징 |
|:---|:---|:---|
| **NFS** | 리눅스/유닉스 | 유닉스 권한 |
| **CIFS/SMB** | 윈도우 | 윈도우 권한 |
| **AFP** | macOS | 애플 |

**📢 섹션 요약 비유:** NFS = 리눅스!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 리눅스 환경** → NFSv4
**시나리오 2: 혼합 환경** → NFS + SMB
**시나리오 3: 고성능** → pNFS

**도입 체크리스트:** 버전, 보안, 성능

**📢 섹션 요약 비유:** Kerberos = 보안 강화!
