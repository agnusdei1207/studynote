# 트래픽 관리 (Traffic Management)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 트래픽 제어
> 2. **가치**: 성능, 안정성
> 3. **융합**: QoS, 로드밸런싱과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

트래픽 관리는 네트워크 트래픽을 제어하여 성능과 안정성을 보장합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    트래픽 관리                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              트래픽 관리 기술                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  1. 대역폭 관리 (Bandwidth Management):               │   │ │
│   │   │     • Rate Limiting                                  │   │ │
│   │   │     • Traffic Shaping                                │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 혼잡 제어 (Congestion Control):                   │   │ │
│   │   │     • TCP Congestion Control                         │   │ │
│   │   │     • AQM (Active Queue Management)                  │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 우선순위 (Prioritization):                        │   │ │
│   │   │     • QoS (Quality of Service)                       │   │ │
│   │   │     • DSCP                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  4. 로드밸런싱 (Load Balancing):                      │   │ │
│   │   │     • L4/L7 Load Balancer                            │   │ │
│   │   │     • GSLB                                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              트래픽 쉐이핑 vs 폴리싱                            │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  Traffic Shaping:                                     │   │ │
│   │   │     • 트래픽을 버퍼에 저장 후 천천히 전송              │   │ │
│   │   │     • 지연 발생, 패킷 손실 없음                        │   │ │
│   │   │     ┌───┬───┬───┐ → ┌───┐ → ┌───┐ → ┌───┐            │   │ │
│   │   │     │   │   │   │   │   │   │   │   │   │            │   │ │
│   │   │     └───┴───┴───┘   └───┘   └───┘   └───┘            │   │ │
│   │   │                                                      │   │ │
│   │   │  Traffic Policing:                                   │   │ │
│   │   │     • 초과 트래픽 폐기                                │   │ │
│   │   │     • 지연 없음, 패킷 손실 있음                        │   │ │
│   │   │     ┌───┬───┬───┐ → ┌───┐ → ┌───┐                    │   │ │
│   │   │     │   │   │ X │   │   │   │   │                    │   │ │
│   │   │     └───┴───┴───┘   └───┘   └───┘                    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 트래픽 관리는 대역폭, 혼잡, 우선순위를 제어합니다.

**💡 비유**: 트래픽 관리는 교통 신호등과 도로 용량 관리입니다.

**📢 섹션 요약 비유:** 트래픽 관리 = 성능 보장!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Rate Limiting** | 속도 제한 |
| **Shaping** | 평탄화 |
| **Policing** | 폐기 |
| **QoS** | 서비스 품질 |

**비교표: QoS 메커니즘**

```
+------------+----------+----------+
| 메커니즘   │ 계층     │ 용도     │
+------------+----------+----------+
| CoS        │ L2       │ 이더넷   │
| DSCP       │ L3       │ IP       │
| ToS        │ L3       │ 레거시   │
| ECN        │ L3       │ 혼잡     │
+------------+----------+----------+
```

**핵심 알고리즸:** Token Bucket, Leaky Bucket

**📢 섹션 요약 비유:** Token Bucket = 토큰 기반!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: ISP** → QoS + Shaping
**시나리오 2: 데이터센터** → ECN + DSCP
**시나리오 3: 클라우드** → Rate Limiting

**도입 체크리스트:** 대역폭, 우선순위, 정책

**📢 섹션 요약 비유:** DiffServ = L3 QoS!
