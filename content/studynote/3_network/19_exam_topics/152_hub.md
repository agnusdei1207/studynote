# 허브 (Hub) - 더미 / 스위칭 / 인텔리전트

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 포트 신호 분배 장치
> 2. **가치**: LAN 확장, 1계층 장비
> 3. **융합**: 이더넷, 스타 토폴로지와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

허브(Hub)는 여러 장비를 연결하고 수신한 신호를 모든 포트로 전송하는 1계층 장비입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    허브 (Hub)                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              허브 종류                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 더미 허브 (Dummy Hub / Passive Hub):                │   │ │
│   │   │     • 신호를 모든 포트로 브로드캐스트                    │   │ │
│   │   │     • 충돌 도메인 공유 (CSMA/CD)                         │   │ │
│   │   │     • 대역폭 공유                                       │   │ │
│   │   │     • 1계층 장비                                        │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 스위칭 허브 (Switching Hub):                         │   │ │
│   │   │     • MAC 주소 기반 포워딩                              │   │ │
│   │   │     • 충돌 도메인 분리                                   │   │ │
│   │   │     • 전이중 지원                                       │   │ │
│   │   │     • 2계층 장비 (스위치와 동일)                         │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 인텔리전트 허브 (Intelligent Hub):                   │   │ │
│   │   │     • SNMP 관리 기능                                    │   │ │
│   │   │     • 모니터링/진단                                     │   │ │
│   │   │     • 원격 관리                                         │   │ │
│   │   │     • 관리형 스위치와 유사                               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              더미 허브 동작                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │          ┌─────────────────────┐                    │   │ │
│   │   │          │       허브          │                    │   │ │
│   │   │          │  ┌───┬───┬───┬───┐ │                    │   │ │
│   │   │    PC A ─┤  │ 1 │ 2 │ 3 │ 4 │ ├─ PC B              │   │ │
│   │   │          │  └───┴───┴───┴───┘ │                    │   │ │
│   │   │          │        ▲           │                    │   │ │
│   │   │          │        │           │                    │   │ │
│   │   │          │  모든 포트로 전송   │                    │   │ │
│   │   │          └─────────────────────┘                    │   │ │
│   │   │                                                      │   │ │
│   │   │  문제:                                                   │   │ │
│   │   │  • A→B 전송 시 C, D도 수신 (불필요 트래픽)               │   │ │
│   │   │  • 모든 포트가 같은 충돌 도메인                          │   │ │
│   │   │  • 대역폭 공유 (100Mbps ÷ N)                            │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 더미 허브는 1계층 장비로 충돌 도메인을 공유, 스위칭 허브는 2계층 장비입니다.

**💡 비유**: 더미 허브는 누가 말하면 모든 사람이 듣는 방송입니다.

**📢 섹션 요약 비유:** 더미 허브 = 1계층, 스위치 = 2계층!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 더미 허브 | 스위칭 허브 |
|:---|:---|:---|
| **MAC 학습** | 없음 | 있음 |
| **포워딩** | 브로드캐스트 | 유니캐스트 |
| **충돌 도메인** | 공유 | 분리 |
| **전이중** | 불가 | 가능 |

**비교표**

```
+------------+----------+----------+
| 항목       | 더미     | 스위칭   |
+------------+----------+----------+
| 계층       | 1        | 2        |
| 대역폭     | 공유     | 독립     |
| 충돌       | 많음     | 없음     |
| 성능       | 낮음     | 높음     |
+------------+----------+----------+
```

**핵심 알고리즸:** CSMA/CD(더미), MAC 학습(스위칭)

**📢 섹션 요약 비유:** 5-4-3 규칙 = 5세그먼트 4리피터 3호스트!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 성능**

| 장비 | 대역폭 | 충돌 도메인 |
|:---|:---:|:---:|
| **더미 허브** | 공유 | 1개 |
| **스위치** | 포트별 | N개 |

**📢 섹션 요약 비유:** 스위치가 허브 대체!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 레거시** → 더미 허브 (폐기)
**시나리오 2: 현대 LAN** → 스위치
**시나리오 3: 관리 필요** → 인텔리전트/관리형 스위치

**도입 체크리스트:** 트래픽, 포트 수, 관리 필요성

**📢 섹션 요약 비유:** 더미 허브는 역사!

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: LAN 연결

**미래 전망:** 스위치로 완전 대체

**참고 표준:** IEEE 802.3

**📢 섹션 요약 비유:** 허브는 LAN의 시초!
