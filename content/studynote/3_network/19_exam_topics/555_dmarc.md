# DMARC (Domain-based Message Authentication) - 도메인 기반 메시지 인증

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SPF + DKIM 통합 정책
> 2. **가치**: 이메일 위조 방지, 리포팅
> 3. **융합:** SPF, DKIM, DNS TXT

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DMARC는 SPF와 DKIM 검증 결과에 따른 정책을 정의합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DMARC                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DMARC 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  DNS TXT: _dmarc.example.com                        │   │ │
│   │   │  "v=DMARC1; p=reject; rua=mailto:dmarc@example.com" │   │ │
│   │   │                                                      │   │ │
│   │   │  검증 흐름:                                          │   │ │
│   │   │  1. SPF 검증                                         │   │ │
│   │   │  2. DKIM 검증                                        │   │ │
│   │   │  3. 정책 적용 (none/quarantine/reject)              │   │ │
│   │   │  4. 리포트 발송                                      │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: DMARC는 SPF/DKIM 실패 시 처리 방법을 지정합니다.

**📢 섹션 요약 비유:** DMARC = 최종 판결!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **p=none** | 모니터링만 |
| **p=quarantine** | 스팸 처리 |
| **p=reject** | 거부 |
| **rua** | 집계 리포트 |
| **ruf** | 포렌식 리포트 |
| **pct** | 적용 비율 |

**핵심 알고리즸:** SPF/DKIM 검증 → 정책 적용 → 리포트

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**DMARC 정책 단계**

| 단계 | 정책 | 목적 |
|:---|:---|:---|
| **1** | none | 모니터링 |
| **2** | quarantine | 격리 |
| **3** | reject | 거부 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 도입 → none 시작
**시나리오 2:** 검증 → quarantine
**시나리오 3:** 완료 → reject
