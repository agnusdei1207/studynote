# IPC (Inter-Process Communication) - 프로세스 간 통신

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 간 데이터 교환
> 2. **가치**: 협업, 동기화
> 3. **융합:** Pipe, Socket, Shared Memory

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

IPC는 독립된 프로세스 간 통신을 제공합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    IPC                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              IPC 메커니즘                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │            Shared Memory                     │    │   │ │
│   │   │  │   Process A ←────────────────→ Process B     │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │            Message Queue                     │    │   │ │
│   │   │  │   Process A → [Queue] → Process B            │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │            Pipe                             │    │   │ │
│   │   │  │   Process A → [Pipe] → Process B            │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   │  ┌─────────────────────────────────────────────┐    │   │ │
│   │   │  │            Socket                           │    │   │ │
│   │   │  │   Process A ←────────────────→ Process B    │    │   │ │
│   │   │  └─────────────────────────────────────────────┘    │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: IPC는 다양한 방식으로 데이터 교환을 지원합니다.

**📢 섹션 요약 비유:** IPC = 부서 간 소통!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Pipe** | 단방향 통신 |
| **Named Pipe** | 이름 있는 파이프 |
| **Message Queue** | 메시지 큐 |
| **Shared Memory** | 공유 메모리 |
| **Semaphore** | 동기화 |
| **Socket** | 네트워크 통신 |

**핵심 알고리즸:** 송신 → IPC 메커니즘 → 수신

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**IPC 방식 비교**

| 방식 | 속도 | 복잡도 | 용도 |
|:---|:---|:---|:---|
| **Pipe** | 빠름 | 낮음 | 단순 통신 |
| **Socket** | 보통 | 높음 | 네트워크 |
| **Shared Memory** | 가장 빠름 | 높음 | 대용량 |
| **Message Queue** | 보통 | 보통 | 비동기 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 같은 머신 → Shared Memory / Pipe
**시나리오 2:** 다른 머신 → Socket
**시나리오 3:** 비동기 → Message Queue
