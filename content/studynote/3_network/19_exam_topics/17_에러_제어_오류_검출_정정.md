# 에러 제어 (오류 검출/정정)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 중 발생한 오류를 검출하거나 정정
> 2. **가치**: 데이터 무결성, 신뢰성, 서비스 품질
> 3. **융합**: ARQ, FEC, CRC와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

에러 제어(Error Control)는 전송 과정에서 발생한 오류를 검출하고 정정하는 메커니즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    에러 제어 (오류 검출/정정)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              오류 유형                                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  단일 비트 오류 (Single Bit Error):                    │   │ │
│   │   │  • 1개 비트만 변형                                     │   │ │
│   │   │  • 검출/정정 용이                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  다중 비트 오류 (Multiple Bit Error):                   │   │ │
│   │   │  • 여러 비트 변형                                      │   │ │
│   │   │  • 정정 어려움                                         │   │ │
│   │   │                                                      │   │ │
│   │   │  버스트 오류 (Burst Error):                            │   │ │
│   │   │  • 연속된 비트 변형                                     │   │ │
│   │   │  • 길이에 따라 분류                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              오류 제어 기법                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 오류 검출 (Error Detection):                       │   │ │
│   │   │     • 패리티 (Parity)                                 │   │ │
│   │   │     • 체크섬 (Checksum)                               │   │ │
│   │   │     • CRC (Cyclic Redundancy Check)                   │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 오류 정정 (Error Correction):                       │   │ │
│   │   │     • FEC (Forward Error Correction)                  │   │ │
│   │   │     • 해밍 코드 (Hamming Code)                         │   │ │
│   │   │     • 리드-솔로몬 (Reed-Solomon)                       │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 재전송 (Retransmission):                            │   │ │
│   │   │     • ARQ (Automatic Repeat reQuest)                  │   │ │
│   │   │     • Stop-and-Wait, Go-Back-N, Selective Repeat       │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 검출은 재전송 요청, 정정은 즉시 복구입니다.

**💡 비유**: 에러 제어는 택배의 파손 확인과 보상입니다.

**📢 섹션 요약 비유**: 검출은 재전송, 정정은 즉시!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Parity** | 1비트 검출 | 홀/짝 | 검사 |
| **CRC** | 다비트 검출 | 다항식 | 인증 |
| **Hamming** | 정정 | 거리 | 복구 |
| **ARQ** | 재전송 | ACK/NAK | 재요청 |

**오류 검출 능력**

| 방식 | 검출 | 정정 |
|:---|:---:|:---:|
| **Parity** | 1비트 | X |
| **CRC-16** | 다비트 | X |
| **Hamming** | 2비트 | 1비트 |
| **RS** | 다비트 | 다비트 |

**핵심 알고리즘**: 데이터 → 검출 코드 추가 → 수신 → 검사 → 오류 시 대응

**📢 섹션 요약 비유**: CRC-32가 Ethernet 표준!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: ARQ 방식**

| 방식 | 윈도우 | 효율 |
|:---|:---:|:---:|
| **Stop-Wait** | 1 | 낮음 |
| **Go-Back-N** | N | 중간 |
| **Selective** | N | 높음 |

**📢 섹션 요약 비유**: Selective Repeat가 최고!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 무선** → FEC + ARQ (HARQ)
**시나리오 2: 유선 LAN** → CRC + ARQ
**시나리오 3: 저장** → ECC (정정)

**도입 체크리스트**: 오류율, 지연, 대역폭

**📢 섹션 요약 비유**: 무선은 FEC 필수!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 신뢰성 99.99%+

**미래 전망**: Quantum Error Correction

**참고 표준**: IEEE 802.3 CRC-32, LTE HARQ

**📢 섹션 요약 비유**: 에러 제어는 신뢰성의 핵심!
