# Denormalization - 반정규화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 의도적 중복 허용
> 2. **가치**: 조회 성능 향상
> 3. **융합:** Normalization, OLAP, Data Warehouse

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Denormalization은 성능을 위해 중복을 허용합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Denormalization                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              정규화 vs 반정규화                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  정규화:                    반정규화:                 │   │ │
│   │   │  ┌─────────────┐           ┌─────────────────────┐  │   │ │
│   │   │  │ Orders      │           │ Orders              │  │   │ │
│   │   │  │ ID │ UserID │           │ ID │ UserID │UName  │  │   │ │
│   │   │  └──────┬──────┘           │    │        │(중복) │  │   │ │
│   │   │         │                  └─────────────────────┘  │   │ │
│   │   │  ┌──────┴──────┐           장점:                    │   │ │
│   │   │  │ Users       │           - 조인 감소              │   │ │
│   │   │  │ ID │ Name   │           - 조회 속도 향상         │   │ │
│   │   │  └─────────────┘                                    │   │ │
│   │   │                     단점:                           │   │ │
│   │   │  (조인 필요)        - 저장 공간 증가                │   │ │
│   │   │                     - 갱신 비용 증가                │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 반정규화는 읽기 성능과 쓰기 비용의 트레이드오프입니다.

**📢 섹션 요약 비유:** Denormalization = 의도적 복사!
