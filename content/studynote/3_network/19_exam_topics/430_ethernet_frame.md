# 이더넷 프레임 - Ethernet Frame

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: L2 데이터 단위
> 2. **가치**: MAC 주소 기반 전송
> 3. **융합**: 802.3, VLAN 태그, FCS

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

이더넷 프레임은 L2에서 데이터를 전송하는 단위입니다.

> **해설**: 이더넷 프레임은 MAC 주소로 목적지를 식별합니다.

**💡 비유**: 이더넷 프레임은 편지 봉투입니다. 보내는 사람, 받는 사람, 내용이 있습니다.

**📢 섹션 요약 비유:** 프레임 = L2 데이터!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Preamble** | 10101010 (7바이트) |
| **SFD** | 10101011 (1바이트) |
| **MAC Address** | 48비트 주소 |
| **FCS** | CRC-32 |

**비교표: 프레임 크기**

```
+------------+----------+----------+
| 항목       │ 최소     │ 최대     │
+------------+----------+----------+
| 표준       │ 64B      │ 1518B    │
| Jumbo      │ 64B      │ 9000B    │
| VLAN 태그  │ 64B      │ 1522B    │
+------------+----------+----------+
```

**핵심 알고리즸:** CRC 계산, MAC 학습

**📢 섹션 요약 비유:** 64바이트 = 최소 크기!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 이더넷 타입**

| 타입 | 값 | 용도 |
|:---|:---|:---|
| **IPv4** | 0x0800 | IP |
| **IPv6** | 0x86DD | IPv6 |
| **ARP** | 0x0806 | ARP |
| **VLAN** | 0x8100 | 802.1Q |

**📢 섹션 요약 비유:** 0x8100 = VLAN!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** 일반 이더넷 → 1518바이트
**시나리오 2:** VLAN → 1522바이트
**시나리오 3:** 스토리지 → Jumbo Frame (9000B)

**도입 체크리스트:** MTU, Jumbo Frame, VLAN 태그

**📢 섹션 요약 비유:** MTU = 프레임 크기!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: L2 데이터 전송
