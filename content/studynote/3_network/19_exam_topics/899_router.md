# Router - 라우터

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: L3 패킷 전달 장치
> 2. **가치**: 네트워크 간 연결
> 3. **융합:** Routing Table, ACL, NAT

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

Router는 IP 계층에서 패킷을 전달합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Router                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              라우터 동작                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │  패킷 처리 과정:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │                                                 ││   │ │
│   │   │  │  1. 패킷 수신                                    ││   │ │
│   │   │  │  2. 헤더 검사 (TTL, Checksum)                     ││   │ │
│   │   │  │  3. 라우팅 테이블 조회                             ││   │ │
│   │   │  │  4. Next Hop 결정                                 ││   │ │
│   │   │  │  5. TTL 감소                                      ││   │ │
│   │   │  │  6. 인터페이스로 전송                              ││   │ │
│   │   │  │                                                 ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   │  기능:                                                   │   │ │
│   │   │  - Routing: 경로 결정                                     │   │ │
│   │   │  - Forwarding: 패킷 전달                                  │   │ │
│   │   │  - NAT: 주소 변환                                         │   │ │
│   │   │  - ACL: 접근 제어                                         │   │ │
│   │   │  - QoS: 서비스 품질                                       │   │ │
│   │   │                                                      │   │ │
│   │   │  라우팅 테이블:                                            │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐│   │ │
│   │   │  │ Destination    Gateway      Iface   Metric      ││   │ │
│   │   │  │ 10.0.0.0/8     192.168.1.1  eth0    1           ││   │ │
│   │   │  │ 0.0.0.0/0      192.168.1.1  eth0    1           ││   │ │
│   │   │  └─────────────────────────────────────────────────┘│   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └────────────────────────────────────────ational──────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

> **해설**: 라우터는 네트워크 간 통신의 핵심입니다.

**📢 섹션 요약 비유:** Router = 교차로 신호등!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 항목 | 설명 |
|:---|:---|
| **Routing Table** | 라우팅 테이블 |
| **Interface** | 인터페이스 |
| **Forwarding** | 포워딩 |
| **Control Plane** | 제어 평면 |
| **Data Plane** | 데이터 평면 |

**핵심 알고리즸:** 수신 → 조회 → 전달

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**라우터 vs 스위치**

| 항목 | Router | Switch |
|:---|:---|:---|
| **계층** | L3 | L2 |
| **주소** | IP | MAC |
| **분리** | 브로드캐스트 | 충돌 |
| **기능** | 경로 결정 | 프레임 전달 |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1:** WAN → Router
**시나리오 2:** 인터넷 → Edge Router
**시나리오 3:** 데이터센터 → L3 Switch/Router
