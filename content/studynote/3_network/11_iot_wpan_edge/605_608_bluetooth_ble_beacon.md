# 605-608. 블루투스와 BLE 기술 분석

> **핵심 인사이트**: 블루투스는 '선 없애기'에서 시작해 '모든 사물 연결하기'로 진화했다. 특히 전력 소모를 획기적으로 줄인 BLE(Bluetooth Low Energy)의 등장은 코인 배터리 하나로 수개월을 버티는 센서 세상을 열어젖힌 IoT의 일등 공신이다.

---

## Ⅰ. 블루투스 (Bluetooth) 기본 원리
IEEE 802.15.1 표준을 사용하는 2.4GHz ISM 대역의 무선 기술입니다.

* **네트워크 구조**:
  - **Piconet (피코넷)**: 1대의 마스터와 최대 7대의 슬레이브로 구성된 최소 단위망.
  - **Scatternet (스캐터넷)**: 여러 개의 피코넷이 마스터/슬레이브 역할을 공유하며 연결된 거대 그물망.
* **주파수 호핑 (AFH)**: 간섭을 피하기 위해 초당 1,600번씩 채널을 빠르게 옮겨 다니며 통신합니다.

---

## Ⅱ. BLE (Bluetooth Low Energy / BT 4.0 이상)
전력 효율을 극대화하기 위해 설계된 완전히 새로운 방식의 블루투스입니다.

* **핵심 특징**:
  - **Duty Cycle 최적화**: 평소에는 잠자고 있다가 데이터를 보낼 때만 잠깐 깨어납니다.
  - **짧은 패킷**: 데이터 크기를 줄여 전송 시간을 단축했습니다.
* **Classic vs BLE 비교**:
  - **Classic**: 오디오 스트리밍, 대용량 파일 전송 (계속 연결).
  - **BLE**: 심박수 측정, 스마트 전구 제어, 위치 추적 (간헐적 통신).

---

## Ⅲ. 비컨 (Beacon) 기술
BLE의 광고(Advertising) 기능을 활용하여 특정 위치의 사용자에게 정보를 쏘는 기술입니다.

1. **iBeacon (애플)**: 위치 기반 서비스(LBS)를 위해 UUID 정보를 주기적으로 발송.
2. **Eddystone (구글)**: 오픈 소스 기반으로 URL 주소 등을 직접 전송 가능.
* **활용**: 박물관 전시 안내, 마트 쿠폰 발송, 실내 내비게이션.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[블루투스 네트워크 구조]

  [ Slave ]      [ Slave ]
      \          /
       [ Master ]  (Piconet)
      /          \
  [ Slave ]      [ Master/Slave ] ─── [ Slave ] (Scatternet 확장)
```

📢 **섹션 요약 비유**: **Classic 블루투스**는 물을 계속 뿜어내는 '소방 호스'입니다. 음질은 좋지만 물(에너지)을 많이 쓰죠. **BLE**는 필요할 때만 분무기로 물을 한 번씩 칙칙 뿌리는 '스프레이'입니다. 물을 아주 조금만 써서 오래 버틸 수 있습니다. **비컨**은 등대와 같아서, 배(스마트폰)가 가까이 오면 "여기는 항구입니다"라고 끊임없이 신호를 보내는 위치 알림판입니다.
