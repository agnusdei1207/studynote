# 266-271. 초기 광역 통신망(WAN): X.25와 프레임 릴레이

> **핵심 인사이트**: 1980년대에는 통신선의 에러율이 너무 높아 망(네트워크) 자체가 에러를 다 고쳐주는 깐깐한 X.25를 썼지만, 광케이블의 도입으로 선로 품질이 획기적으로 좋아지자 에러 검사를 과감히 생략하고 속도를 높인 '프레임 릴레이(Frame Relay)'가 WAN 시장을 장악했다.

---

## Ⅰ. 광역 통신망(WAN)의 기초
과거 기업이 서울 본사와 부산 지사를 연결하는 방식은 크게 두 가지였습니다.
* **전용선 (Leased Line)**: 통신사(KT 등)로부터 구리선이나 광케이블을 24시간 독점 임대하여 쓰는 방식입니다. (T1: 1.544Mbps, E1: 2.048Mbps, T3: 45Mbps) 혼자 쓰니 빠르고 안전하지만 비용이 천문학적으로 비쌉니다.
* **다이얼업 (Dial-up)**: 필요할 때만 공중전화망(PSTN)에 전화를 걸어 연결하는 방식입니다. 싸지만 너무 느리고 회선 연결 시간이 깁니다.

이 두 가지의 단점을 극복하고, **여러 기업이 하나의 거대한 통신사 망(Cloud)을 공유하여 비용을 나누되 전용선처럼 쓸 수 있게 만든 것**이 패킷 교환망입니다.

---

## Ⅱ. X.25 망 (초기 패킷 교환망)
1970~80년대에 ITU-T가 제정한 패킷 교환망의 원조 표준입니다.

* **특징**: 당시의 구리 전화선은 노이즈가 심해 데이터가 가다가 깨지는 일이 다반사였습니다.
* **오류 제어**: X.25는 노드(라우터)와 노드를 지날 때마다 프레임을 까서 에러가 없는지(CRC 검사), 순서는 맞는지 일일이 확인하고 고쳐주는(ARQ) **매우 무겁고 신뢰성 높은 동작**을 했습니다. (LAPB 프로토콜 사용)
* **단점**: 안전하긴 하지만, 교환기마다 에러 검사를 하느라 지연(Delay)이 너무 심해서 최대 64Kbps의 속도밖에 내지 못했습니다.

---

## Ⅲ. 프레임 릴레이 (Frame Relay)
1990년대, 광케이블이 널리 깔리면서 선로에서 에러가 날 확률이 극도로 낮아졌습니다. 이에 맞춰 X.25의 무거운 기능을 다 빼버리고 속도에 올인한 프로토콜입니다.

### 1. 주요 특징 (X.25와의 차이점)
* **에러/흐름 제어 생략**: 중간에 있는 스위치(교환기)들은 더 이상 에러 검사나 재전송을 하지 않습니다. 오직 프레임의 헤더만 보고 목적지로 빠르게 전달(Relay)만 합니다. 에러가 나면 그냥 프레임을 버리며, 재전송 책임은 양 끝단의 PC(TCP 계층)에게 떠넘깁니다.
* **가변 길이 프레임**: 데이터 길이를 유동적으로 조정할 수 있어 트래픽 처리가 유연합니다. (최대 45Mbps 속도 지원)

### 2. 가상 회선 (Virtual Circuit)
물리적인 선은 하나지만, 논리적으로 여러 개의 전용선처럼 쓰는 기술입니다.
* **PVC (Permanent Virtual Circuit)**: 통신사에 돈을 내고 **영구적으로 고정**해둔 가상 회선입니다. (기업 전용선 대체 용도로 가장 많이 쓰임)
* **SVC (Switched Virtual Circuit)**: 필요할 때만 연결을 맺고 다 쓰면 끊는 가상 회선입니다. (전화 방식)

### 3. 핵심 제어 파라미터
* **DLCI (Data Link Connection Identifier)**: IP 주소나 MAC 주소 대신, 프레임 릴레이 망 내부에서 출발지와 목적지를 식별하는 10비트 길이의 **가상 회선 번호(주소)**입니다.
* **CIR (Committed Information Rate)**: 통신사가 고객에게 "이 속도만큼은 무조건 100% 보장해 줄게!"라고 약속한 최소 대역폭입니다. CIR을 넘는 트래픽(Burst)은 여유가 있으면 보내주고 망이 바쁘면 가차 없이 버려버립니다.
* **FECN / BECN (혼잡 알림)**: 망 내부에 트래픽이 꽉 차서 터지기 직전(혼잡)일 때, 수신자 쪽으로 가는 프레임에는 FECN 비트를, 송신자 쪽으로 가는 프레임에는 BECN 비트를 1로 세팅하여 "지금 길 막히니까 데이터 좀 천천히 보내라"고 경고해 주는 기능입니다.

## Ⅳ. 개념 맵 및 요약

```ascii
[X.25 와 프레임 릴레이 비교]

(1980년대 구리선 환경)
[ PC ] ---> [ X.25 교환기 ] ──(CRC/ARQ 검사)──> [ X.25 교환기 ] ---> [ PC ]
  └─ "선로가 안 좋으니 내가 라우터마다 일일이 뜯어보고 에러 고쳐줄게!" (느림)

(1990년대 광케이블 환경)
[ PC ] ---> [ Frame Relay 스위치 ] ──(그냥 패스)──> [ Frame Relay 스위치 ] ---> [ PC ]
  └─ "선로가 좋으니까 에러 안 날 거야! 나는 검사 안 하고 그냥 릴레이(전달)만 할게!" (빠름)

[프레임 릴레이 핵심 용어]
- DLCI : 내가 타고 갈 가상의 차선 번호 (주소)
- PVC  : 영구 고정 차선
- CIR  : 통신사가 무조건 보장하는 최저 속도 한도
```

📢 **섹션 요약 비유**: **X.25**는 과거 비포장도로 시절, 화물이 깨질까 봐 검문소(라우터)마다 상자를 다 열어서 확인하고 다시 포장하는 아주 꼼꼼하고 느린 택배 시스템입니다. **프레임 릴레이**는 고속도로(광케이블)가 뚫린 후, 검문소에서 겉에 적힌 목적지(DLCI)만 쓱 보고 하이패스로 바로 통과시켜주는 엄청나게 빠른 택배 시스템입니다.
