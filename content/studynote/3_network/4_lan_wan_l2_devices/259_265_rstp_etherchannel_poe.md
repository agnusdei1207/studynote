# 259-265. 고급 STP 기술과 스위치 포트 기능(LACP, PoE)

> **핵심 인사이트**: 오리지널 STP의 50초나 되는 끔찍한 복구 시간을 1초 이내로 줄인 것이 RSTP이며, 여러 개의 물리적 선을 하나로 묶어 대역폭을 뻥튀기하는 기술이 이더채널(LACP)이다.

---

## Ⅰ. STP의 한계 극복 (PortFast와 RSTP)

### 1. PortFast와 BPDU Guard (Cisco 전용 기술)
* **PortFast**: PC나 서버가 연결된 스위치 포트는 루프(Loop)가 생길 일이 전혀 없습니다. 하지만 오리지널 STP는 PC를 꽂아도 무식하게 30초(Listening -> Learning)를 대기한 후 통신을 엽니다. PortFast를 설정하면, 선을 꽂자마자 **대기 시간 없이 즉시 포워딩(Forwarding) 상태로 진입**합니다.
* **BPDU Guard**: PortFast가 설정된 포트에 누군가 실수로 (또는 악의적으로) 스위치를 꽂아서 BPDU(스위치 인사말)가 들어오면, 루프 방지를 위해 즉시 해당 포트를 강제 셧다운(Error-Disable) 시켜버리는 보안 기능입니다.

### 2. RSTP (Rapid STP, IEEE 802.1w)
오리지널 STP의 50초 컨버전스 타임을 **1~2초 이내**로 획기적으로 단축한 국제 표준입니다.
* **새로운 포트 역할 추가**: 기존 STP의 차단 포트(Blocking Port)를 세분화하여, 대장으로 가는 길(RP)이 끊어졌을 때 즉시 대체할 **대체 포트(Alternate Port)**와, 지정 포트(DP)가 끊어졌을 때 즉시 대체할 **백업 포트(Backup Port)**를 미리 계산해 둡니다.
* **빠른 전환**: 메인 링크가 끊어지는 순간, 타이머를 기다리지 않고 미리 준비해둔 대체/백업 포트를 즉각 포워딩 상태로 올려버립니다.

---

## Ⅱ. MSTP (Multiple STP, IEEE 802.1s)
VLAN이 100개가 있다면, 스위치는 각 VLAN마다 따로따로 STP 계산을 해야 하므로 CPU가 터져나갑니다. (PVST+의 문제점)

* **원리**: 100개의 VLAN을 10개씩 묶어서 하나의 **인스턴스(Instance)**로 만듭니다. 그리고 이 인스턴스 단위로만 STP 트리를 계산합니다.
* **장점**: 스위치의 CPU 부하를 획기적으로 줄이면서도, VLAN 그룹별로 서로 다른 길(트리)을 타게 하여 로드밸런싱(트래픽 분산)을 완벽하게 구현할 수 있습니다.

---

## Ⅲ. 이더채널 / 링크 어그리게이션 (Link Aggregation)
스위치 간에 1Gbps 선 1개만 연결하면 대역폭이 부족합니다. 그렇다고 선을 4개 연결하면 STP가 루프라고 판단하여 3개를 강제로 차단(Blocking)해 버립니다.

* **개념**: 물리적으로 여러 개(예: 4개)의 케이블을 연결하되, 스위치 소프트웨어가 이를 **"하나의 거대한 논리적 파이프(4Gbps)"**로 인식하게 속이는 기술입니다. STP도 이 묶음을 하나의 포트로 보므로 차단하지 않습니다.
* **표준 프로토콜 (LACP, IEEE 802.3ad)**: 모든 벤더 장비 간에 호환되는 표준 이더채널 협상 프로토콜입니다.
* **시스코 전용 프로토콜 (PAgP)**: 시스코 장비끼리만 묶을 때 사용하는 과거의 프로토콜입니다. (현재는 대부분 LACP로 통일됨)

---

## Ⅳ. PoE (Power over Ethernet, IEEE 802.3af/at/bt)
천장에 달린 CCTV나 Wi-Fi 공유기(AP)에 인터넷 선(UTP)과 전기선(220V 파워)을 따로 끌어가려면 공사비가 두 배로 듭니다.

* **원리**: UTP 케이블(8가닥) 중 데이터 통신에 쓰지 않는 남는 선, 혹은 데이터가 흐르는 선 위에 직류(DC) 전기를 함께 실어서 보내는 기술입니다.
* **발전**:
  - **PoE (802.3af)**: 포트당 15.4W (구형 IP 전화기)
  - **PoE+ (802.3at)**: 포트당 30W (최신 Wi-Fi AP, 모터 달린 CCTV)
  - **PoE++ (802.3bt)**: 포트당 60W~90W (소형 디스플레이, POS 기기까지 랜선 하나로 전력 공급 가능)

---

## Ⅴ. 개념 맵 및 요약

```ascii
[스위치 포트 주요 기술 요약]

         (1Gbps)                 |
[ SW 1 ] ========= [ SW 2 ]      |-> [ LACP (이더채널) ] 
         (1Gbps)                 |   "선 2개를 묶어 2Gbps짜리 굵은 파이프로 만들자!"

[ SW 3 ] ---------------------- [ AP (무선 공유기) / CCTV ]
             (Data + 48V DC)     |-> [ PoE ] 
                                 |   "데이터도 주고 전기도 줄게! 파워선 꽂지 마!"

[ SW 4 ] ---------------------- [ User PC ]
                                 |-> [ PortFast ] 
                                 |   "PC니까 루프 안 돌아! 30초 대기 없이 즉시 오픈!"
```

📢 **섹션 요약 비유**: **RSTP**는 메인 다리가 끊어졌을 때를 대비해 옆에 예비 폰툰 다리를 미리 지어놓고, 사고 나면 즉시 차단기를 올리는 시스템입니다. **이더채널(LACP)**은 1차선 국도 4개를 시멘트로 덮어버려서 4차선 고속도로 하나로 확장하는 공사입니다. **PoE**는 수도관(데이터) 바깥쪽에 열선(전기)을 같이 감아서 보내, 상대방이 물도 마시고 보일러도 때게 해주는 일석이조 기술입니다.
