# 245-250. 가상 랜(VLAN)과 트렁킹 기술

> **핵심 인사이트**: 스위치는 충돌 도메인은 나눌 수 있어도 브로드캐스트 도메인은 나누지 못한다. 이를 극복하기 위해 물리적인 스위치를 소프트웨어적으로 쪼개어 여러 대의 가상 스위치처럼 작동하게 만드는 기술이 바로 VLAN이다.

---

## Ⅰ. 가상 랜 (VLAN, Virtual LAN)의 개념
물리적인 배치(1층, 2층)와 상관없이, 논리적으로 네트워크(브로드캐스트 도메인)를 분할하는 기술입니다.

* **도입 목적**:
  1. **브로드캐스트 제어**: 불필요한 브로드캐스트 트래픽이 전체 네트워크로 퍼지는 것을 막아 성능을 높입니다.
  2. **보안 강화**: 인사팀(VLAN 10)과 영업팀(VLAN 20)을 논리적으로 격리시켜, 서로 패킷을 엿보거나 통신할 수 없게 만듭니다. (서로 통신하려면 반드시 3계층 장비인 라우터나 L3 스위치를 거쳐야 함)
  3. **유연성**: 직원이 자리를 옮겨도 랜선을 다시 깔 필요 없이, 스위치 포트의 VLAN 번호만 소프트웨어적으로 바꿔주면 끝납니다.

---

## Ⅱ. 스위치 포트의 종류

### 1. 접근 포트 (Access Port)
* **특징**: 오직 **단 하나의 VLAN**에만 소속된 포트입니다.
* **용도**: 일반 PC나 프린터 같은 단말기가 연결되는 포트입니다. 단말기들은 자신이 어떤 VLAN에 속해있는지 전혀 모릅니다(태그 없음).

### 2. 트렁크 포트 (Trunk Port)
* **특징**: **여러 개의 VLAN 트래픽이 동시에 지나다닐 수 있는 통로**입니다.
* **용도**: 스위치와 스위치 사이, 또는 스위치와 라우터를 연결할 때 사용합니다. 스위치 간에 데이터를 보낼 때 "이 프레임은 VLAN 10꺼야!"라는 꼬리표(Tag)를 붙여서 보냅니다.

---

## Ⅲ. VLAN 태깅 (Tagging) 프로토콜
트렁크 포트를 지나는 프레임이 어느 VLAN 소속인지 구별하기 위한 표준입니다.

### 1. IEEE 802.1Q (가장 널리 쓰이는 국제 표준)
이더넷 프레임의 Source MAC과 Type 필드 사이에 **4바이트의 태그(Tag)**를 억지로 쑤셔 넣는(Insert) 방식입니다.
* **TPID (Tag Protocol Identifier)**: 2바이트. 항상 `0x8100` 값을 가지며, "나 802.1Q 태그 달고 있어!" 라고 알립니다.
* **TCI (Tag Control Information)**: 2바이트. 이 안에 우선순위(Priority)와 핵심인 **VID (VLAN ID, 12비트)** 가 들어있습니다. 12비트이므로 최대 $2^{12} = 4096$ 개의 VLAN을 생성할 수 있습니다. (0과 4095는 예약됨)

### 2. ISL (Inter-Switch Link)
시스코(Cisco)에서 독자적으로 만든 구형 태깅 프로토콜입니다. 기존 프레임을 건드리지 않고, 프레임 전체를 새로운 26바이트 헤더와 4바이트 꼬리로 캡슐화(Encapsulation)해 버립니다. 지금은 표준인 802.1Q에 밀려 거의 쓰이지 않습니다.

---

## Ⅳ. 주요 VLAN 관련 기술

### 1. Native VLAN (네이티브 VLAN)
802.1Q 트렁크 포트는 기본적으로 모든 프레임에 태그(Tag)를 달아서 보냅니다. 하지만, **VLAN 태그가 없는(Untagged) 쌩얼 프레임이 트렁크 포트로 들어왔을 때, 이를 어느 VLAN으로 소속시킬지 결정하는 기본값**이 바로 Native VLAN입니다. (보통 디폴트 값은 VLAN 1 이지만, 보안상 다른 번호로 변경하는 것이 강력히 권고됩니다.)

### 2. VTP (VLAN Trunking Protocol)
* **개념**: 스위치가 수십 대 있을 때, 한 스위치에서 VLAN 10을 만들면 나머지 스위치들에도 자동으로 VLAN 10이 동기화되게 해주는 시스코 전용 프로토콜입니다.

### 3. DTP (Dynamic Trunking Protocol)
* **개념**: 스위치끼리 선을 꽂았을 때, 서로 협상하여 포트를 Access로 쓸지 Trunk로 쓸지 자동으로 결정해 주는 시스코 전용 프로토콜입니다. (보안상의 이유로 실무에서는 수동으로 고정(Hardcoding)하는 것을 선호합니다.)

---

## Ⅴ. 개념 맵 및 요약

```ascii
[IEEE 802.1Q 프레임 태깅 구조]

(기존 이더넷) [ Dest MAC | Source MAC |                 | Type | Payload ]
                                        ▼ 4바이트 삽입 ▼
(802.1Q 삽입) [ Dest MAC | Source MAC | 802.1Q Tag (4B) | Type | Payload ]
                                       ├── TPID(0x8100) 
                                       └── TCI (VID 포함: 0~4095)

[스위치 연결 구성]
 [PC A] --(Access: VLAN 10)--> [Switch 1] 
                                    |
                               (Trunk: 802.1Q) -> "나 VLAN 10번 프레임이야!"
                                    |
 [PC B] --(Access: VLAN 10)--> [Switch 2]
```

📢 **섹션 요약 비유**: 스위치가 '아파트'라면, VLAN은 아파트 안의 '평수별 거주 구역'을 나누는 가벽입니다. 접근 포트(Access)는 자기 방 문입니다. 트렁크 포트(Trunk)는 아파트 동과 동을 잇는 공용 구름다리입니다. 이 구름다리를 건널 때는 길을 잃지 않도록 반드시 이마에 자기 구역 번호가 적힌 명찰(802.1Q Tag)을 붙여야 합니다. 단, 명찰을 안 붙이고 돌아다니는 길 잃은 영혼들을 모아두는 곳이 바로 네이티브 VLAN(Native VLAN)입니다.
