# 601-604. 사물인터넷(IoT)의 기초와 WPAN

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사물인터넷(IoT, Internet of Things)은 현물(Physical Object)에 센서와 통신 능력을 부여하여 디지털 세계와 물리 세계를 융합하는 '사이버-물리 시스템(CPS, Cyber-Physical System)'의 근간이다.
> 2. **가치**: 단순 연결을 넘어 데이터(Data)를 실시간 수집·분석함으로써 비즈니스 프로세스를 자동화하고, 노동력 절감 및 신규 서비스 창출이라는 경제적 가치를 창출한다.
> 3. **융합**: WPAN(Wireless Personal Area Network)과 같은 저전력 통신 기술, 그리고 WSN(Wireless Sensor Network) 아키텍처가 결합하여 초연결 시대의 인프라를 구성한다.

+++

### Ⅰ. 개요 (Context & Background)

사물인터넷(IoT, Internet of Things)은 센서(Sensor)와 액추에이터(Actuator)가 부착된 사물('Thing')이 유무선 네트워크를 통해 서로 정보를 교환하고, 클라우드(Cloud)에서 이를 분석하여 지능형 서비스를 제공하는 생태계를 의미합니다. 이는 단순히 기계를 원격 제어하는 M2M(Machine-to-Machine) 개념을 넘어, 사물을 인터넷의 주체(Internet of)로 끌어올려 데이터를 생산하고 소비하는 거대한 플랫폼으로 진화했습니다.

**💡 비유**
마치 과거에 전기가 없던 세상에 전기가 공급되어 가전제품이 작동하듯, 이제는 모든 사물에 '전기' 대신 '정보(Internet)'가 공급되어 스스로 생각하고 통신하는 세상입니다.

**등장 배경**
1.  **기존 한계**: 기존 인터넷은 사람(Human) 간의 소통(PC, 스마트폰)에 국한되었으며, 물리 세계의 데이터(온도, 진동 등)를 디지털화하여 자동으로 처리하는 데 한계가 있었습니다.
2.  **혁신적 패러다임**: MEMS(Micro-Electro-Mechanical Systems) 기술의 발전으로 센서의 초소형화·저렴화가 가능해지고, 사물 인식 기술(RFID, Radio Frequency Identification)이 상용화되면서 '사물'의 연결이 현실화되었습니다.
3.  **현재 비즈니스 요구**: 4차 산업혁명의 핵심인 데이터 기반 의사결정(Data-Driven Decision Making)을 위해 현실 세계의 모든 현상을 데이터로 수집해야 하는 필연적인 요구가 생겨났습니다.

**📢 섹션 요약 비유**: IoT의 도입은 마치 도시에 '신경망'을 깔아주는 것과 같습니다. 도시의 모든 구석(사물)에 미세한 신경 센서를 부착하여 중앙 뇌(클라우드)가 도시의 상태를 실시간으로 감지하고 제어하게 되는 것입니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

IoT 시스템은 크게 **感知(Perception)**, **傳送(Network)**, **應用(Application)**의 3계층으로 구성됩니다. 각 계층은 독립적으로 발전했으나, 현재는 상호 밀접하게 연결된 융합형 아키텍처로 진화하고 있습니다.

#### 1. 구성 요소 상세 분석
| 구성 요소 (Component) | 역할 (Role) | 내부 동작 및 기술 (Internal Mechanics) | 주요 프로토콜/기술 | 비유 |
|:---|:---|:---|:---|:---|
| **Device (Perception)** | 물리적 데이터 수집 및 제어 | 센서(아날로그→디지털 변환)와 액추에이터(신호→물리력) 제어 | ADC, GPIO, RFID, BLE | 눈과 귀, 손발 |
| **Network (Transmission)** | 데이터 전송 및 연결 | 캡슐화/복호화, 라우팅, 무선 주파수 변조 | MQTT, CoAP, IPv6, 6LoWPAN | 신경망/혈관 |
| **Platform (Application)** | 데이터 저장, 분석, 서비스 제공 | 빅데이터 처리, AI 분석, API 관리, 미들웨어 | AWS IoT, Azure IoT, Kafka | 뇌/지능 |

#### 2. IoT 데이터 수집 및 전달 구조도 (ASCII)

아래 다이어그램은 현실 세계의 데이터가 디지털 세계로 전달되어 가치로 전환되는 과정을 도식화한 것입니다. WPAN은 1단계에서 2단계로 넘어가는 '접근로(Access)' 역할을 담당합니다.

```ascii
+-----------------------------------------------------------------------+
|                         [ IoT Service Layer ]                         |
|   +---------------+    +---------------------+    +----------------+  |
|   | Data Analytics|<---|  Application Logic |<---|  User Device   |  |
|   |   (AI/BI)     |    |   (Smart Factory)  |    |  (Dashboard)   |  |
|   +---------------+    +---------------------+    +----------------+  |
+--------------------------------^--------------------------------------+
                                 | (Internet / Cloud / LTE)
+--------------------------------|--------------------------------------+
|                        [ Network Gateway Layer ]                      |
|   +-------------------------------------------------------------------+|
|   |                     [ Sink Node / Gateway ]                       ||
|   |    (Protocol Conversion: BLE/ZigBee -> TCP/IP)                    ||
|   +-------------------------------------------------------------------+|
+--------------------------------^--------------------------------------+
         :                        |                 :
         : WPAN (10m)             |                 :
         :                        |                 :
+-------+-------+    +-------+    |    +-------+---------+             */
|  Sensor Node  |    | Sensor |----+----| Wearable Device | (Watch)    */
|  (Temp/Hum)   |    | (Light)|         |  (Health)       |             */
+---------------+    +--------+             +-----------------+
   <--- 802.15.4 --->
   <--- BLE Mesh --->
```

**(다이어그램 해설)**
위 구조는 IoT의 표준적인 계층 모델을 보여줍니다.
1.  **Device Layer (좌측)**: 현실 세계의 온도나 조도 등을 감지하는 **Sensor Node**들이 배치됩니다. 이들은 주로 802.15.4 계열의 저전력 통신(예: **ZigBee**)이나 BLE를 사용하여 주변의 데이터를 모읍니다.
2.  **Gateway Layer (중앙)**: 수많은 센서가 쏟아내는 데이터를 한 곳으로 모아주는 **Sink Node** 역할을 합니다. 매우 중요한 점은 이 게이트웨이가 '네이티브 IoT 프로토콜(예: CoAP)'을 '인터넷 프로토콜(TCP/IP)'로 변환하여 사물이 인터넷이라는 거대한 바다에 나아갈 수 있도록 다리를 놓아준다는 점입니다.
3.  **Cloud/Service Layer (상단)**: 변환된 데이터는 인터넷을 거쳐 클라우드 서버로 전송되어 저장, 분석되며, 최종적으로 사용자가 모바일 앱이나 웹 대시보드를 통해 서비스를 소비하게 됩니다.

#### 3. 핵심 동작 원리 및 알고리즘
IoT 환경에서는 제한된 리소스(배터리, CPU) 때문에 경량화된 프로토콜이 필수적입니다.
*   **MQTT (Message Queuing Telemetry Transport)**: 발행/구독(Pub/Sub) 모델을 사용하여 네트워크 트래픽을 최소화하는 메시징 프로토콜입니다. 헤더가 2바이트로 매우 작아 대규모 센서 네트워크에 적합합니다.
*   **CoAP (Constrained Application Protocol)**: RESTful API 스타일을 사용하며, UDP 기반으로 동작하여 전력 소모가 적습니다. M2M(Machine-to-Machine) 통신에 최적화되어 있습니다.

**📢 섹션 요약 비유**: IoT 아키텍처는 마치 거대한 '우체물 시스템'과 같습니다. 센서는 편지를 쓰는 사람(발신자), 게이트웨이는 지역 우체국(분류 집하실), 그리고 클라우드는 중앙 우체물 처리 센터입니다. 여기서 WPAN은 집에서 우체국까지 편지를 배달하는 '소형 배송차'나 '자전거 배달'과 같습니다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

IoT의 실현을 위해서는 다양한 통신 거리(Distance)와 전력 소모(Power) 간의 트레이드-오프(Trade-off)를 고려한 통신 기술 선택이 필수적입니다.

#### 1. 무선 통신 기술 비교 분석표
| 구분 | WPAN (Wireless PAN) | WLAN (Wireless LAN) | WWAN (Wireless WAN) | WSN (Wireless Sensor Network) |
|:---|:---|:---|:---|:---|
| **대표 기술** | **Bluetooth**, ZigBee, **UWB** | Wi-Fi (802.11 a/b/g/n/ax) | **LTE**, 5G, LoRa | 저전력 802.15.4, ZigBee |
| **복coverage 영역** | 10m 이내 (Personal) | 100m 내외 (Local) | 수 km ~ 전국 (Wide) | 100m 내외 (Ad-hoc) |
| **전력 소모** | **초저전력** (CR coin cell) | 고전력 (Outlet 필요) | 중~고전력 | **초저전력** (Sleep mode 중심) |
| **데이터 전송률** | 중저속 (1 Mbps ~) | 초고속 (Gbps ~) | 중고속 (Mbps ~ Gbps) | 저속 (kbps ~) |
| **주요 용도** | 웨어러블, 헤드셋, 홈 오토메이션 | 스트리밍, 사무실 네트워크 | 자동차, 공장, 스마트시티 | 환경 모니터링, 산업 센서 |

#### 2. M2M vs IoT 비교 (정량적 관점)
| 비교 항목 | M2M (Machine to Machine) | IoT (Internet of Things) |
|:---|:---|:---|
| **연결성** | 폐쇄적, 1:1 또는 1:N (Silos) | 개방적, N:M (Web based) |
| **통신망** | 전용망(PSTN, PLC) 위주 | 인터넷 프로토콜(IP) 기반 |
| **지능(Intelligence)** | 하드웨어 중심 (Embedded) | 클라우드 중심 (Cloud AI) |
| **스케일** | 소규모 (수백~수천 대) | 초대규모 (수십억 대) |

#### 3. 기술 융합 및 시너지 (OS & Network)
*   **OS와의 융합**: IoT 디바이스의 리소스 제약을 극복하기 위해 **RTOS (Real-Time OS)**인 FreeRTOS 또는 경량 리눅스 분석(TI-RTOS 등)이 탑재됩니다.
*   **네트워크와의 융합**: IPv6의 확장으로 모든 사물에 고유 IP 할당이 가능해지면서, **6LoWPAN (IPv6 over Low-Power Wireless Personal Area Networks)** 기술을 통해 작은 패킷도 IPv6로 라우팅할 수 있게 되어 인터넷과의 완전한 연결성이 확보되었습니다.

**📢 섹션 요약 비유**: 통신 기술의 선택은 '이동 수단의 선택'과 같습니다. **WPAN**은 자전거나 도보(내 집 안), **WLAN**은 승용차(회사나 카페), **WWAN**은 고속열차나 비행기(전국/세계)와 같습니다. 짐을 많이 싣고 멀리 가려면 연료(전력)가 많이 드는 것이 당연하므로, 배달하려는 물건(데이터)의 크기와 거리에 따라 적절한 것을 골라야 합니다.

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

실무에서 IoT 시스템을 구축할 때는 단순히 기술을 도입하는 것이 아니라, 비즈니스 임팩트와 유지보수 효율성을 고려한 아키텍처 설계가 필요합니다.

#### 1. 실무 시나리오 및 의사결정
**시나리오 A: 스마트 팩토리 내 설비 진단 시스템 구축**
*   **상황**: 수천 대의 모터와 베어링의 진동/온도 데이터를 1초마다 수집해야 함.
*   **결정 과정**:
    1.  **Network**: 케이블링이 어려운 회전체 관점에서 **무선(Wireless)** 선정.
    2.  **Tech**: Wi-Fi는 전력 소모가 크고 간섭이 심하므로 배제. **BLE Mesh**나 **ZigBee**를 사용하여 저전력 메시 네트워크 구성.
    3.  **Edge Processing**: 모든 데이터를 클라우드로 보내면 비용이 폭발하므로, **Edge Computing**을 도입하여 이상 신호(Threshold 초과) 발생 시에만 데이터 전송하는 **Edge Analytics** 전략 수립.

#### 2. 도입 체크리스트
*   **기술적**: **QoS (Quality of Service)** 보장 가능 여부, **Latency (지연 시간)** 요구 조건 충족 여부, 보안 프로토콜(TLS/DTLS) 적용 가능성.
*   **운영/보안적**: 펌웨어 **OTA (Over-The-Air) 업데이트** 지원 여부, 디바이스 인증 및 키 관리 체계, 배터리 교체 주기 및 관리 비용 산정.

#### 3. 안티패턴 (Anti-Pattern)
*   **IoT Monolith**: 모든 센서 데이터를 무조건 중앙 서버로 전송하여 저장하려는 설계. 네트워크 대역폭 폭증 및 스토리지 비용 증가로 시스템이 붕괴될 수 있음.
*   **Single Point of Failure**: 중앙의 **Sink Node(게이트웨이)**가 고장 나면 전체 센서 네트워크가 마비되는 구조. 이중화(Redundancy)나 분산 처리를 고려해야 함.

**📢 섹션 요약 비유**: IoT 구축은 마치 '물류 센터'를 짓는 것과 같습니다. 모든 물건(데이터)을 본사(클라우드)로 보내려고 하면 운송비(네트워크 비용)가 천문학적입니다. 현장에서 필요 없는 것을 거르는 현물 분류 시스템(엣지 컴퓨팅)을 도입하고, 택배차가 다닐 수 있는 최적의 배송 경로(WPAN/Wi-Fi)
