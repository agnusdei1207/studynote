# 615-621. 저전력 광역 통신망(LPWAN) 기술 비교

> **핵심 인사이트**
> 1. **본질**: LPWAN (Low-Power Wide-Area Network)은 전력 효율과 커버리지를 극대화하여 아주 작은 데이터를 먼 거리(수km~数十km)까지 전송하는 것을 핵심으로 하는 IoT 전용 통신 패러다임입니다.
> 2. **가치**: 기존 셀룰러 망의 과도한 전력 소모와 비면허대역 통신의 짧은 거리 한계를 해소하여, 스마트 시티, 스마트 팩토리, 농업 등 사물 인터넷 시장의 폭발적인 성장을 견인합니다.
> 3. **융합**: 비면허 대역(LoRa, Sigfox)의 자립성과 면허 대역(NB-IoT, LTE-M)의 신뢰성/이동성이 상호 보완적으로 융합되어, 5G Massive IoT 구현의 핵심 하부 기망으로 작용합니다.

+++

## Ⅰ. 개요 (Context & Background)

### 개념 및 정의
**LPWAN (Low-Power Wide-Area Network)**은 저전력(Low-Power) 동작과 광역(Wide-Area) 커버리지를 동시에 만족시키도록 설계된 무선 통신 네트워크 기술군을 의미합니다. 일반적인 Wi-Fi나 Bluetooth와 달리 수 km에서 수십 km의 거리를 커버하면서, 배터리 하나로 5~10년 이상 동작할 수 있도록 전력 소모를 최소화한 것이 특징입니다. 데이터 전송률은 매우 낮아(보통 0.3kbps ~ 100kbps 수준) 텍스트나 센서 값과 같은 작은 패킷 전송에 특화되어 있습니다.

### 💡 비유
LPWAN은 **"사람 대신 소리 내어 말할 수 없는 사물들이, 에너지를 아끼기 위해 작은 속삭임을 아주 먼 곳까지 전달하는 특별한 우편 시스템"**과 같습니다. 

### 등장 배경 및 필요성
1.  **기존 기술의 한계**: 기존 2G/3G/4G(LTE) 망은 전력 소모가 커서 건전지나 소형 배터리로 구동되는 IoT 센서에 적합하지 않았고, Wi-Fi/Zigbee는 통신 거리가 짧아 기지국을 빽빽하게 설치해야 하는 비용 문제가 있었습니다.
2.  **혁신적 패러다임**: M2M(Machine-to-Machine) 통신의 증가로, 높은 데이터 전송률보다는 연결 가능한 디바이스 수(Long Range & Low Throughput)와 배터리 수명이 중요해지는 역설이 발생했습니다.
3.  **비즈니스 요구**: 스마트 시티(가로등, 주차관제), 스마트 그리드(가스/전기 검침), 환경 모니터링 등과 같이 **"느리지만, 멀리, 오래"** 연결되어야 하는 시장이 급격히 확대되었습니다.

> **📢 섹션 요약 비유**: 이는 비행기처럼 빠르지만 비싸고 연료를 많이 먹는 일반 택배(LTE) 대신, 배터리를 장착한 자전거로 느리게라도 며칠 밤을 새워가며 효율적으로 물건을 배달하는 특별 배송 시스템을 도입한 것과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

LPWAN은 주파수 대역 사용 여부에 따라 **비면허 대역(Unlicensed Band)** 기술과 **면허 대역(Licensed Band)** 기술로 나뉩니다. 각각의 변조 방식과 접속 제어 방식이 판이하게 다릅니다.

### 1. 구성 요소 상세 분석

| 구성 요소 | LoRa (Long Range) | Sigfox | NB-IoT (Narrowband IoT) | LTE-M (LTE Cat-M1) |
|:---|:---|:---|:---|:---|
| **표준/관리** | LoRa Alliance (비영리) | Sigfox Company (단일) | 3GPP Release 13 | 3GPP Release 13 |
| **변조 방식** | **CSS (Chirp Spread Spectrum)** | **UNB (Ultra Narrow Band)** | OFDMA / SC-FDMA | OFDMA / SC-FDMA |
| **대역폭** | 125kHz, 250kHz, 500kHz | 100Hz (극초소) | 180kHz (1 PRB) | 1.08MHz (6 PRBs) |
| **핵심 특징** | **주파수 호핑에 강한 내성** | **초단 메시지 전송 강제** | LTE In-band 설치 가능 | 음성 및 이동성 지원 |

### 2. LPWAN 기술별 아키텍처 및 동작 흐름

#### A. LoRaWAN (Long Range Wide Area Network) 아키텍처
LoRa는 물리계층(PHY)의 변조 방식이며, LoRaWAN은 MAC 계층 프로토콜입니다. **"스타형(Star-of-Stars)"** 토폴로지를 사용합니다.

```ascii
      [End Device]          [End Device]        (센서/노드)
           |                     |             
           +-------+         +---+---+         
                   |         |                 
              (LoRa PHY)  (LoRa PHY)           
                   |         |                 
        ~~~~~~~~~~~|~~~~~~~~_|~~~~~~~~~~~~~~~~~~ (Radio Link)
                   |         |                 
           +-------+         +-------+         
           |   LoRa Gateway  (Concentrator)    
           +-------+---------+-------+         
                   |         |                 
        ~~~~~~~~~~~|~~~~~~~~_|~~~~~~~~~~~~~~~~~~ (TCP/IP IPsec VPN)
                   |         |                 
        +----------+---------+----------+      
        |        Network Server          |      
        +---------------------------------+      
```

**[LoRa 동작 메커니즘]**
1.  **Chirp Spread Spectrum (CSS)**: 데이터를 선형 주파수 변화를 갖는 '처프(Chirp)' 신호로 실어 보냅니다. 이는 도플러 효과나 잡음에 매우 강한 내성을 가집니다.
2.  **Spreading Factor (SF)**: 확산 계수(SF7~SF12)를 조절하여 속도와 거리를 트레이드오프할 수 있습니다.
    *   **SF7 (High Data Rate)**: 짧은 거리, 빠름
    *   **SF12 (Low Data Rate)**: 긴 거리(최대 15km), 느림
3.  **Adaptive Data Rate (ADR)**: 네트워크 서버가 게이트웨이와의 신호 대 잡음비(SNR)를 분석하여 단말기의 SF를 자동으로 조정하여 배터리 수명을 늘립니다.

#### B. 면허 대역 LPWAN (Cellular IoT) 구조
NB-IoT와 LTE-M은 기존 LTE 기지국을 업그레이드하여 사용합니다.

```ascii
      [LTE Base Station (eNodeB)]         
           |    |    |    |    |           
           +----+----+----+----+----------- (RF Carrier)
           |        (OFDMA/SC-FDMA)        
           v                                
    [ MME / S-GW / EPC ]  <--->  [IoT Platform]
```

*   **NB-IoT**: **180kHz**의 좁은 대역폭 하나만 차치합니다. LTE 기지국의 인-band(캐리어 내)나 Guard-band(보호 대역), 혹은 Stand-alone(독립 주파수)으로 배치될 수 있습니다. **Single Carrier (SC-FDMA)**를 사용하여 피크 전력(PAPR)을 낮춰 RF 효율을 극대화합니다.
*   **LTE-M**: LTE의 대역폭(최대 1.4MHz)을 그대로 사용하지만, 시그널링을 간소화하고 **eDRX(Extended Discontinuous Reception)**와 **PSM(Power Saving Mode)**을 통해 심화 절전 모드를 지원합니다.

> **📢 섹션 요약 비유**: LoRa는 **"여러 사람이 동시에 떠들어도 알아들을 수 있도록 높고 낮은 성대(Tone)를 섞어 노래하듯 전파를 보내는 방식"**이며, NB-IoT는 **"아주 좁은 파이프(전화선)를 통해 규칙적으로 물방울을 떨어뜨리듯 데이터를 보내는 방식"**입니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 심층 기술 비교 분석표

| 비교 항목 | LoRaWAN (비면허) | Sigfox (비면허) | NB-IoT (면허) | LTE-M (면허) |
|:---|:---|:---|:---|:---|
| **통신 거리** | 2~15km (Urban) | 3~10km | 10~15km | 10km 이상 |
| **데이터 전송률** | 0.3~50 kbps | 100 bps (Up) | 20~250 kbps | 1 Mbps |
| **전력 소모** | 극저전력 (Sleep 중심) | 극저전력 (자동 wake-up) | 극저전력 (PSM) | 저전력 (상대적으로 높음) |
| **모듈 가격** | $5 ~ $10 (하락 중) | $3 ~ $5 (최저가) | $10 ~ $15 | $15 ~ $20 |
| **망 구축 비용** | 사용자 직접 구축 (Gate-way 구매) | 사용자 직접 구축 (Operator 의존적) | 통신사 기간망 재활용 | 통신사 기간망 재활용 |
| **심층 특징** | 망 구축의 자율성과 보안 설정의 유연성 보유 | 업링크(Uplink) 위주의 하이브리드 방식, 메시지 제한 있음 | 기지국 밀도가 높아 실내 커버리지 우수 | 음성(Voice) 및 핸드오버(Handover) 지원 가능 |

### 2. 주요 매트릭스 및 의사결정 기준
기술 선택 시 가장 중요한 고려 요소는 **"운영 주체"**와 **"이동성 유무"**입니다.

*   **망 구축권 (Sovereignty)**:
    *   **LoRa**: 공장, 농장, 캠퍼스 등 **폐쇄 룹(Closed Loop)** 구간에서 자체 망을 구축하고 싶을 때 선택.
    *   **NB-IoT/LTE-M**: 전국 단위 서비스(공공 인프라)로 통신사의 관리(MNNO)를 받고 싶을 때 선택.
*   **이동성 (Mobility)**:
    *   고정형 자산(주차, 계량기)은 **NB-IoT**가 유리함.
    *   이동형 자산(자전거, 화물 트럭, 유목형 가축)은 핸드오버가 가능한 **LTE-M**이 필수적임.

### 3. 타 과목 융합 관점
*   **OS/임베디드**: LPWAN 모듈은 MCU(Microcontroller Unit)와 UART/SPI 인터페이스로 연결되며, 전력을 아끼기 위해 OS 스케줄링이 **"수동형(Polling)"**이 아닌 **"이벤트 드리븐(Event-Driven)"** 또는 **주기적 슬립(Sleep Mode)"** 설계가 필수적입니다.
*   **네트워크(네트워크 계층)**: 비면허 대역은 사용자가 직접 보안 채널(AES-128) 구성을 해야 하지만, 면허 대역(LTE 기반)은 **USIM 카드**를 통해 망 자체에서 인증과 암호화가 제공됩니다.

> **📢 섹션 요약 비유**: LPWAN 기술 선택은 **"택시와 자가용의 선택"**과 같습니다. **LoRa**는 내가 차를 사서 스스로 운전하고 연료를 관리해야 하지만 내 마음대로 짐을 싣고 다닐 수 있는 **'자가용'**이며, **NB-IoT**는 택시 회사에 요금을 내고 호출만 하면 되지만 운행 구간이 제한된 **'일반 택시'**, **LTE-M**은 고속도로를 달리며 고객센터와 실시간으로 통화할 수 있는 **'고급 택시'**와 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 1. 실무 시나리오 및 의사결정 프로세스

#### 시나리오 A: 스마트 팩토리 설비 모니터링
*   **상황**: 5만 평 규모의 공장 지붕 위 탱크 및 지하 파이프 센서 데이터 수집. 외부 통신사 전파가 잘 들지 않음.
*   **의사결정**: **LoRaWAN** 도입.
*   **이유**:
    1.  건물 내부(심층) 통신을 위해 LoRa Gateway를 자체적으로 최적 위치에 설치 가능함.
    2.  데이터 유출 우려가 있어 사설 망(Private Network) 구성 필요.
    3.  센서 간 간섭 제어를 위해 **ADR** 기능을 활용하여 전력 효율 극대화.

#### 시나리오 B: 전국 단위 전기/가스 계량기 검침 (AMI)
*   **상황**: 전국 가정의 계량기 데이터를 통신사 망을 통해 수집해야 함. 설치 후 유지보수(배터리 교체 등)가 매우 어려움.
*   **의사결정**: **NB-IoT** 도입.
*   **이유**:
    1.  기존 LTE 기지국 커버리지 활용으로 별도 망 구축 비용(CAPEX) 제로.
    2.  통신사 보안 인증(MNO Level Security)으로 공공 데이터 무결성 확보.
    3.  20dB 이상의 커버리지 증설(MCL)으로 지하실 계량기도 수집 가능.

### 2. 도입 체크리스트

| 구분 | LoRa / Sigfox | NB-IoT / LTE-M |
|:---|:---|:---|
| **기술적** | ① 주파수 간섭 여부 체크<br>② 백홀(Backhaul) 인터넷 라인 확보<br>③ LoRa Server(ChirpStack 등) 구축 | ① 기지국 커버리지 맵 확인<br>② USIM 형태(MFF2/Plug-in) 결정<br>③ MTC(Machine Type Comm) 플랫폼 연동 |
| **운영/보안** | ① 내부 서버 보안(AES 키 관리)<br>② 전용 주파수 사용 허가 확인(국가별) | ① 통신사 로밍(Roaming) 정책 확인<br>② 개인정보 암호화 전송 준수 |

### 3. 치명적 결함 (Anti-Patterns)
*   **LTE-M
