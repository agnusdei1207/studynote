# 622-625. IoT 메시징 프로토콜: MQTT와 CoAP

> **핵심 인사이트**: IoT 장치들은 데이터 한 비트를 보내는 데도 신중해야 한다. 발행-구독(Pub/Sub) 방식으로 수천 개의 장치를 효율적으로 관리하는 MQTT와, 웹의 방식(HTTP)을 경량화하여 장치 간 직접 대화를 돕는 CoAP은 IoT 세상을 움직이는 두 가지 핵심 언어다.

---

## Ⅰ. MQTT (Message Queuing Telemetry Transport)
IBM에서 개발하고 OASIS 표준이 된 **발행-구독(Publish-Subscribe)** 기반의 경량 프로토콜입니다.

* **특징**:
  - **TCP** 기반으로 신뢰성 있는 전송을 보장합니다.
  - **Broker (중개자)** 가 중간에서 메시지를 배분하여 송수신자가 서로를 몰라도 통신이 가능합니다.
* **QoS 등급**:
  - **Level 0**: 최대 한 번 (확인 안 함).
  - **Level 1**: 적어도 한 번 (중복 가능).
  - **Level 2**: 정확히 한 번 (완벽한 신뢰성).

---

## Ⅱ. CoAP (Constrained Application Protocol)
제한된 자원을 가진 장치들을 위해 **HTTP를 가볍게 재설계**한 프로토콜입니다.

* **특징**:
  - **UDP** 기반으로 오버헤드가 극도로 적습니다.
  - **REST** 방식(GET, POST 등)을 그대로 사용하여 기존 웹 시스템과 연동이 매우 쉽습니다.
  - 브로커 없이 1:1로 직접 통신하는 구조입니다.

---

## Ⅲ. MQTT vs CoAP 비교

| 구분 | MQTT | CoAP |
| :--- | :--- | :--- |
| **전송 계층** | **TCP** | **UDP** |
| **통신 모델** | Publish / Subscribe (다대다) | Request / Response (1:1) |
| **헤더 크기** | 2바이트 이상 | 4바이트 이상 |
| **신뢰성** | TCP 및 QoS 등급으로 보장 | 메시지 재전송(Confirmable)으로 보강 |
| **주요 용도** | 중앙 집중형 제어 (스마트 팩토리) | 단순 센서 데이터 조회 (스마트 그리드) |

---

## Ⅳ. 통합 표준 및 관리
* **LwM2M (Lightweight M2M)**: CoAP을 기반으로 하여 IoT 장치의 설정, 펌웨어 업데이트 등을 원격에서 관리하기 위한 표준 인터페이스입니다.
* **oneM2M**: 파편화된 IoT 표준들을 하나로 통합하기 위한 글로벌 아키텍처 표준입니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[MQTT Pub/Sub 모델]

 [ Publisher ] ──("온도: 25도")──> [ Broker ] ──("온도: 25도")──> [ Subscriber ]
 (센서 장치)                       (중앙 서버)                    (모바일 앱)

[CoAP REST 모델]

 [ Client ] ───────(GET /temp)───────> [ Server ]
 (조회 장치) <──────(2.05 Content)───── (센서 장치)
```

📢 **섹션 요약 비유**: **MQTT**는 '신문사(Broker)'와 같습니다. 기자가 기사(데이터)를 써서 신문사에 보내면(Pub), 그 신문을 구독신청(Sub)한 독자들에게만 배달되는 방식입니다. **CoAP**은 '편의점 쇼핑'과 같습니다. 내가 직접 가서 물건을 달라고 요청(GET)하면 즉시 대답해주는 방식이며, 봉투(헤더)를 최소화하여 가볍게 들고 다니는 형태입니다.
