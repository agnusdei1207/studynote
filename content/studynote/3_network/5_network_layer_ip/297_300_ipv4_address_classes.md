# 297-300. IPv4 주소 체계와 클래스 (Classful Addressing)

> **핵심 인사이트**: IP 주소는 전 세계 컴퓨터를 구분하는 논리 주소로, 효율적인 관리를 위해 '동네 이름(Network ID)'과 '집 번호(Host ID)'로 나뉜다. 과거에는 이 경계를 고정적으로 나눈 '클래스' 체계를 사용했다.

---

## Ⅰ. IPv4 주소의 기본 구조
* **길이**: 32비트 (약 43억 개의 주소 표현 가능)
* **표기**: 8비트씩 4개 묶음(Octet)으로 나누고, 사람이 읽기 편하게 10진수로 변환하여 점(.)으로 구분합니다. (Dotted Decimal Notation)
  - 예: `11000000.10101000.00000000.00000001` ➔ `192.168.0.1`
* **구성**: **네트워크 주소(Network ID)** + **호스트 주소(Host ID)**
  - Network ID: 어떤 네트워크(동네)에 속해있는가?
  - Host ID: 해당 네트워크 내에서 어떤 장비(집)인가?

---

## Ⅱ. 클래스 기반 주소 할당 (Classful Addressing)
과거에 IP 주소를 효율적으로 배분하기 위해 고안된 방식입니다. 주소의 첫 번째 옥텟의 비트 패턴을 보고 클래스를 결정합니다.

### 1. 클래스 A (대규모 네트워크)
* **범위**: `0.0.0.0` ~ `127.255.255.255` (첫 비트가 **0**)
* **구조**: `[Network (8bit)]`.`[Host (24bit)]`
* **특징**: 네트워크 개수는 적지만(128개), 한 네트워크당 호스트를 약 1,600만 개($2^{24}$)나 가질 수 있어 대기업이나 국가 기관에 할당되었습니다.

### 2. 클래스 B (중규모 네트워크)
* **범위**: `128.0.0.0` ~ `191.255.255.255` (첫 두 비트가 **10**)
* **구조**: `[Network (16bit)]`.`[Host (16bit)]`
* **특징**: 한 네트워크당 약 65,000개($2^{16}$)의 호스트를 가질 수 있어 중대형 학교나 기업용으로 쓰였습니다.

### 3. 클래스 C (소규모 네트워크)
* **범위**: `192.0.0.0` ~ `223.255.255.255` (첫 세 비트가 **110**)
* **구조**: `[Network (24bit)]`.`[Host (8bit)]`
* **특징**: 가장 흔한 주소입니다. 한 네트워크당 최대 254개($2^8 - 2$)의 호스트만 가능하여 소규모 사무실이나 가정용으로 적합합니다.

### 4. 클래스 D와 E
* **클래스 D (1110...)**: `224.0.0.0 ~ 239.255.255.255`. **멀티캐스트(Multicast)** 전용 주소입니다.
* **클래스 E (1111...)**: `240.0.0.0 ~ 255.255.255.255`. 미래를 위한 연구용/실험용 예약 주소입니다.

---

## Ⅲ. 특수한 목적의 IP 주소

* **네트워크 주소**: 호스트 비트가 모두 **0**인 주소. 해당 네트워크 자체를 대표하는 이름입니다. (예: `192.168.1.0`)
* **브로드캐스트 주소**: 호스트 비트가 모두 **1**인 주소. 해당 네트워크의 모든 장비에게 동시에 보낼 때 씁니다. (예: `192.168.1.255`)
* **루프백 주소 (Loopback)**: `127.0.0.1`. 자기 자신에게 패킷을 보낼 때 쓰는 가상의 주소입니다. (내 랜카드가 잘 작동하나 테스트용)
* **사설 IP 주소 (Private IP)**: 인터넷에 직접 연결되지 않고 내부망에서만 쓰는 가짜 주소입니다. (A: `10.x`, B: `172.16.x`, C: `192.168.x`)

---

## Ⅳ. 개념 맵 및 요약

```ascii
[IPv4 클래스별 경계 요약]

Class A: (0)NNNNNNN . HHHHHHHH . HHHHHHHH . HHHHHHHH  (대형)
Class B: (10)NNNNNN . NNNNNNNN . HHHHHHHH . HHHHHHHH  (중형)
Class C: (110)NNNNN . NNNNNNNN . NNNNNNNN . HHHHHHHH  (소형)

* N = Network ID, H = Host ID
* ( ) 안의 비트는 클래스를 식별하는 고정 비트
```

📢 **섹션 요약 비유**: IP 주소는 전 세계를 대상으로 하는 '도로명 주소'입니다. **클래스 A**는 '서울특별시'처럼 아주 큰 행정구역 하나를 통째로 빌려주는 것이고, **클래스 C**는 '조그만 빌라 한 동'의 주소들만 나눠주는 것입니다. 하지만 이 방식은 주소가 너무 듬성듬성 낭비되는 단점이 있어, 현대에는 서브넷 마스크를 이용해 더 촘촘하게 나누는 방식으로 진화했습니다.
