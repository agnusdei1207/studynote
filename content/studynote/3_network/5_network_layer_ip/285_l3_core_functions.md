# 285. 네트워크 계층의 핵심 3기능

> **핵심 인사이트**: 전 세계에 흩어진 수많은 네트워크들 사이에서 최적의 길을 찾아(Routing) 패킷을 밀어 넣어주고(Forwarding), 교통 체증을 관리하는(Congestion Control) 것이 OSI 3계층(네트워크 계층)의 존재 이유이다.

---

## Ⅰ. 네트워크 계층(Layer 3)의 본질
데이터 링크 계층이 '직접 연결된 옆집'까지의 배달을 책임진다면, 네트워크 계층은 '서울에서 뉴욕까지'라는 **종단 간(End-to-End)** 논리적 연결을 책임집니다.

---

## Ⅱ. 핵심 기능 1: 라우팅 (Routing, 경로 설정)
패킷이 출발지에서 목적지까지 가기 위한 **전체적인 경로를 결정**하는 지능적인 과정입니다.

* **동작**: 라우팅 알고리즘(RIP, OSPF, BGP 등)을 사용하여 네트워크 지도를 그리고, 목적지까지 가는 최적의 경로를 계산합니다.
* **결과물**: 계산된 경로는 **라우팅 테이블(Routing Table)** 이라는 이정표 데이터베이스에 저장됩니다.
* **비유**: 여행을 떠나기 전, 지도 앱을 켜서 어떤 고속도로를 탈지 전체 경로를 미리 계획하는 것과 같습니다.

---

## Ⅲ. 핵심 기능 2: 포워딩 (Forwarding, 전달/디스패칭)
라우터의 입력 포트로 들어온 패킷을 라우팅 테이블을 참조하여 **적절한 출력 포트로 내보내는 실제 동작**입니다.

* **동작**: 패킷 헤더의 목적지 IP 주소를 읽어 테이블의 인덱스와 매칭시킨 후, 패킷을 스위칭 구조(Switching Fabric)를 통해 이동시킵니다.
* **특징**: 라우팅이 '지도 만들기'라면, 포워딩은 그 지도를 보고 실제로 차를 운전해 나가는 '개별 교차로에서의 판단'입니다. 매우 빠른 속도로 처리되어야 하므로 주로 하드웨어(ASIC) 레벨에서 수행됩니다.

---

## Ⅳ. 핵심 기능 3: 혼잡 제어 (Congestion Control)
네트워크 내에 돌아다니는 패킷의 양이 네트워크의 수용 능력을 초과하지 않도록 관리하는 기능입니다.

* **문제 현상**: 트래픽이 너무 몰리면 라우터의 버퍼가 가득 차서 패킷이 유실(Drop)되고 지연 시간이 급증합니다.
* **동작**: 
  - **예방**: 트래픽 셰이핑(Leaky Bucket 등)을 통해 유입량을 조절합니다.
  - **제거**: 혼잡이 발생하면 소스(송신자)에게 알리거나(ICMP Source Quench 등), 우선순위가 낮은 패킷을 먼저 버려 체증을 해소합니다.
* **차이점**: 흐름 제어(Flow Control)가 송수신자 1:1 사이의 속도 조절이라면, 혼잡 제어는 **네트워크 전체(망)**의 원활한 흐름을 위한 관리입니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[네트워크 계층 동작 메커니즘]

  (1) Routing (계획)         (2) Forwarding (실행)        (3) Congestion Control (운영)
  ┌──────────────┐           ┌──────────────┐           ┌──────────────┐
  │ 최적 경로 계산 │  ────>    │ 패킷 포트 전환 │  <───>    │ 교통 체증 관리 │
  └──────────────┘           └──────────────┘           └──────────────┘
   (Control Plane)            (Data Plane)               (Management Plane)
```

📢 **섹션 요약 비유**: **라우팅**은 내비게이션으로 목적지까지의 가장 빠른 길을 미리 검색하는 두뇌 활동입니다. **포워딩**은 고속도로 톨게이트나 교차로에서 내비가 시키는 대로 1번 게이트나 오른쪽 길을 선택해 실제로 차를 머리를 돌리는 행동입니다. **혼잡 제어**는 도로 공사가 명절 정체를 막기 위해 가변 차로를 열거나 진입로 통제를 하는 도로 관리국(망 관리)의 역할입니다.
