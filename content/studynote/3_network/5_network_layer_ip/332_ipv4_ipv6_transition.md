# 332. IPv4-IPv6 전환 및 공존 기술

> **핵심 인사이트**: 전 세계 모든 장비가 하루아침에 IPv6로 바뀔 수는 없다. 따라서 구형 IPv4 장비와 신형 IPv6 장비가 평화롭게 공존하며 데이터를 주고받을 수 있게 해주는 징검다리 기술들이 반드시 필요하다.

---

## Ⅰ. 전환 기술의 필요성
IPv4와 IPv6는 헤더 구조가 아예 다르기 때문에 서로 직접 대화가 불가능합니다. 이를 해결하기 위해 다음 3가지 전략을 사용합니다.

---

## Ⅱ. 주요 공존 및 전환 기술

### 1. 듀얼 스택 (Dual Stack)
* **개념**: 하나의 장비(호스트나 라우터)에 **IPv4와 IPv6 프로토콜 스택을 동시에 설치**하는 방식입니다.
* **동작**: 상대방이 IPv4면 IPv4로 대화하고, IPv6면 IPv6로 대화합니다.
* **장점**: 가장 단순하고 확실한 방법입니다.
* **단점**: 모든 장비가 두 가지 주소를 다 가져야 하므로, 여전히 IPv4 주소 부족 문제를 근본적으로 해결하지는 못합니다.

### 2. 터널링 (Tunneling)
* **개념**: IPv6 패킷이 IPv4만 지원하는 네트워크 구간을 지나야 할 때, **IPv6 패킷 전체를 IPv4 패킷의 데이터 영역에 쑤셔 넣는(Capsulation)** 방식입니다.
* **종류**:
  - **6to4**: IPv6 섬 사이를 IPv4 바다를 통해 연결하는 자동 터널링.
  - **ISATAP**: 인트라넷 내에서 IPv4 호스트 간에 IPv6 통신을 가능케 함.
  - **Teredo**: UDP에 IPv6를 실어 보내어 NAT 방화벽을 통과함.
* **비유**: 외국(IPv6)에서 보낸 편지를 국내용(IPv4) 큰 봉투에 한 번 더 담아서 배달하는 것과 같습니다.

### 3. 주소 변환 (Translation / NAT64)
* **개념**: IPv6 전용 단말과 IPv4 전용 단말이 대화할 수 있도록, 중간의 게이트웨이가 **헤더 자체를 변환(Rewrite)**해 주는 방식입니다.
* **NAT64 / DNS64**: IPv6 클라이언트가 IPv4 서버에 접속할 수 있도록 주소를 1:1로 매핑해 줍니다.
* **장점**: 구형 서버를 교체하지 않고도 신규 IPv6 사용자를 수용할 수 있습니다.

---

## Ⅲ. 개념 맵 및 요약

```ascii
[IPv4-IPv6 전환 기술 3총사]

1. Dual Stack (공존)
   [ IPv4/IPv6 Host ] --- (둘 다 알아들음) ---> [ Router ]

2. Tunneling (포장)
   [ IPv6 ] -> [ IPv4 봉투에 담기 ] -> (IPv4 Network 통과) -> [ IPv6 ]

3. Translation (통역)
   [ IPv6 Host ] --(Hello!)--> [ NAT64 Gateway ] --(안녕하세요!)--> [ IPv4 Server ]
```

📢 **섹션 요약 비유**: **듀얼 스택**은 한국어와 영어를 모두 할 줄 아는 '이중 언어 사용자'입니다. **터널링**은 영문 편지를 한국 우체국 박스에 담아 배달만 해주는 '특송 서비스'입니다. **주소 변환(NAT64)**은 영어를 한국어로 실시간 번역해 주는 '전문 통역사'와 같습니다.
