# 324-327. IPv6 프로토콜과 차세대 헤더 구조

> **핵심 인사이트**: IPv6는 단순히 주소 개수만 늘린 것이 아니라, 라우터의 부하를 줄이기 위해 헤더 구조를 혁신적으로 단순화하고(Fixed Length), 패킷의 흐름(Flow)을 미리 감지하여 처리 속도를 극대화한 현대적 프로토콜이다.

---

## Ⅰ. IPv6 (Internet Protocol Version 6)의 특징
IPv4의 주소 고갈 한계를 극복하기 위해 등장한 차세대 인터넷 프로토콜입니다.

* **광활한 주소 공간**: 128비트($2^{128}$) 주소 체계를 사용하여, 지구상의 모든 모래알에도 IP를 할당할 수 있을 만큼 무한에 가까운 주소를 제공합니다.
* **보안 및 자동 설정 내장**: 설계 단계부터 보안(IPsec)과 자동 주소 설정(SLAAC) 기능을 포함하고 있습니다.
* **단편화 생략**: 라우터는 더 이상 패킷을 쪼개지(Fragmentation) 않습니다. 오직 송신 호스트만 단편화할 수 있어 라우팅 속도가 매우 빠릅니다.

---

## Ⅱ. IPv6 단순화된 헤더 구조 (40 Bytes 고정)
IPv4 헤더가 복잡하고 가변적이었던 것과 달리, IPv6는 **40바이트 고정 길이 헤더**를 채택했습니다.

### 1. 주요 필드 분석
* **Version (4bit)**: IPv6 이므로 '6'이 들어갑니다.
* **Traffic Class (8bit)**: IPv4의 ToS와 비슷하게 패킷의 우선순위를 나타냅니다.
* **Flow Label (20bit)**: 특정 송수신자 사이의 패킷 흐름(예: 실시간 화상회의 영상)을 식별하여, 라우터가 매번 경로를 계산하지 않고 빠르게 처리하도록 돕는 혁신적인 필드입니다.
* **Payload Length (16bit)**: 헤더를 제외한 순수 데이터의 길이를 표시합니다.
* **Next Header (8bit)**: 다음에 오는 헤더(TCP, UDP 또는 확장 헤더)의 종류를 알려줍니다. IPv4의 Protocol 필드와 유사합니다.
* **Hop Limit (8bit)**: IPv4의 TTL과 동일한 역할을 합니다. (라우터 통과 시 1씩 감소)
* **Source / Destination Address (각 128bit)**: 128비트의 긴 주소가 들어갑니다.

### 2. IPv4 대비 삭제/변경된 필드
* **IHL 삭제**: 헤더가 40바이트로 고정이므로 길이를 표시할 필요가 없습니다.
* **Header Checksum 삭제**: 매번 계산하던 체크섬을 과감히 버렸습니다. (상위 계층이나 데이터 링크 계층의 에러 검출 능력을 믿기로 함)
* **Fragmentation 관련 필드 삭제**: 단편화 정보는 기본 헤더에서 빠지고, 필요할 때만 '확장 헤더'에 넣습니다.

---

## Ⅲ. IPv6의 장점
1. **성능 향상**: 고정 길이 헤더와 체크섬 생략으로 라우터의 패킷 처리 부하가 급감했습니다.
2. **효율적 주소 배분**: 클래스 구분 없이 계층적으로 주소를 할당하여 라우팅 테이블 요약이 쉽습니다.
3. **확장성**: 기본 헤더 뒤에 필요한 옵션을 '확장 헤더' 형태로 무한히 덧붙일 수 있습니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[IPv4 vs IPv6 헤더 비교]

   [ IPv4 Header (가변 20~60B) ]           [ IPv6 Header (고정 40B) ]
 ┌──────────────────────────────┐        ┌──────────────────────────────┐
 │ Version | IHL | ToS | Length │        │ Version | Class | Flow Label │
 ├──────────────────────────────┤        ├──────────────────────────────┤
 │ Ident | Flags | Offset       │        │ Payload Length | Next | Hop  │
 ├──────────────────────────────┤        ├──────────────────────────────┤
 │ TTL | Protocol | Checksum    │        │                              │
 ├──────────────────────────────┤        │      Source Address (128)    │
 │ Source IP Address (32)       │        │                              │
 ├──────────────────────────────┤        ├──────────────────────────────┤
 │ Destination IP Address (32)  │        │                              │
 └──────────────────────────────┘        │   Destination Address (128)  │
                                         │                              │
                                         └──────────────────────────────┘
```

📢 **섹션 요약 비유**: **IPv4 헤더**는 상자 겉면에 이것저것 잡다한 메모와 검사 도장이 어지럽게 찍힌 구형 택배 박스입니다. 내용물에 따라 상자 크기도 제각각이죠. 반면 **IPv6 헤더**는 딱 정해진 크기의 스마트 박스입니다. 꼭 필요한 정보만 아주 깔끔하게 바코드로 인쇄되어 있고, "이건 한꺼번에 가는 영상 데이터 조각들(Flow Label)"이라고 표시까지 되어 있어 자동 분류기가 엄청나게 빨리 처리할 수 있습니다.
