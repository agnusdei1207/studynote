# 701-708. 네트워크 기만 공격: 스니핑과 스푸핑

> **핵심 인사이트**: 네트워크 공격의 시작은 '속이는 것'에서 시작된다. 상대방의 주소를 내 것처럼 속여(Spoofing) 데이터를 가로채고, 그 안의 내용을 몰래 훔쳐보며(Sniffing), 급기야 주인인 척 세션을 가로채는(Hijacking) 일련의 과정은 인터넷의 구조적 신뢰 결함을 파고든다.

---

## Ⅰ. 도청과 가로채기 (Sniffing & MitM)
1. **스니핑 (Sniffing)**: 랜카드를 **프라미스큐어스(Promiscuous)** 모드로 설정하여, 나에게 오지 않은 패킷까지 전부 읽어들이는 도청 행위입니다.
2. **중간자 공격 (MitM, Man-in-the-Middle)**: 통신하는 두 주체 사이에 몰래 끼어들어 데이터를 읽거나 수정하는 공격의 총칭입니다.

---

## Ⅱ. 주소 기만 공격 (Spoofing)

### 1. ARP 스푸핑 (ARP Spoofing)
* **원리**: 가짜 ARP Reply를 지속적으로 보내어, 타겟 단말의 ARP 테이블에 **"게이트웨이의 MAC 주소는 내(해커) MAC 주소야"**라고 저장하게 만듭니다.
* **결과**: 타겟의 모든 트래픽이 해커를 거쳐 게이트웨이로 나갑니다. (도청 및 변조 가능)
* **방어**: ARP 테이블을 정적(**Static**)으로 고정하거나, 스위치 보안 기능(DAI)을 사용합니다.

### 2. IP 스푸핑 (IP Spoofing)
* **원리**: IP 패킷 헤더의 **출발지 IP 주소**를 신뢰할 수 있는 다른 IP로 변조하여 방화벽을 우회하거나 DDoS 공격의 근원지를 숨깁니다.

### 3. DNS 스푸핑 (DNS Cache Poisoning)
* **원리**: DNS 서버에 가짜 IP 주소 정보를 주입하여, 사용자가 `google.com`을 입력했을 때 해커가 만든 피싱 사이트로 연결되게 합니다.
* **방어**: **DNSSEC**을 도입하여 응답의 진위를 검증합니다.

---

## Ⅲ. 세션 및 재생 공격

### 1. 세션 하이재킹 (Session Hijacking)
* **개념**: 이미 인증이 완료되어 통신 중인 두 단말 사이의 **TCP 일련번호(Sequence Number)**를 가로채거나 추측하여, 세션의 주도권을 뺏는 공격입니다.
* **방어**: TLS 암호화 통신을 통해 데이터와 세션 정보를 보호해야 합니다.

### 2. 재생 공격 (Replay Attack)
* **개념**: 사용자가 보낸 정상적인 인증 패킷을 가로챘다가, 나중에 서버에 그대로 다시 보내어 인증을 통과하는 방식입니다.
* **방어**: 매번 바뀌는 일회용 값(**Nonce**)이나 **타임스탬프**를 포함하여 재사용을 막습니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[ARP 스푸핑 공격 흐름]

  [ Victim (PC) ] ──────── ( 원래는 게이트웨이행 ) ────────> [ Gateway ]
         │                                                     ▲
         └─────── ( 속아서 해커에게 전송 ) ───────┐             │
                                                ▼             │
                                          [ Hacker (PC) ] ────┘
                                          ( 데이터 훔쳐보고 전달 )
```

📢 **섹션 요약 비유**: **스니핑**은 남의 집 대화에 귀를 기울이는 '도청기' 설치입니다. **스푸핑**은 택배 기사에게 "저 집 주인이 저(해커)로 바뀌었으니 모든 택배를 저희 집으로 배달해 주세요"라고 거짓말하는 '주소 세탁'입니다. **하이재킹**은 달리는 버스에 몰래 올라타 운전사를 밀어내고 운전대를 잡는 것이고, **재생 공격**은 현관문 도어락 누르는 소리를 녹음했다가 주인 없을 때 틀어주는 속임수입니다.
