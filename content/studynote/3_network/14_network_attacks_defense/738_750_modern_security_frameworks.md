# 738-750. 현대 보안 프레임워크 (Zero Trust, SASE, SOAR)

> **핵심 인사이트**: 성벽을 높게 쌓는 것(Perimeter Security)만으로는 내부의 배신자와 클라우드로 흩어진 데이터를 지킬 수 없다. "아무도 믿지 말고 매번 검증하라(Zero Trust)"는 철학 아래, 네트워크와 보안을 클라우드에서 하나로 합치고(SASE), AI가 공격에 자동으로 대응하는(SOAR) 입체적 방어 체계로 패러다임이 바뀌고 있다.

---

## Ⅰ. 제로 트러스트 (Zero Trust)
"절대 믿지 말고, 언제나 검증하라 (Never Trust, Always Verify)"는 보안 원칙입니다.

* **핵심 요소**:
  - **마이크로 세그멘테이션**: 네트워크를 아주 잘게 쪼개어 한 곳이 뚫려도 옆으로 퍼지지 못하게 함. (Lateral Movement 차단)
  - **최소 권한 원칙**: 꼭 필요한 서비스에만 딱 필요한 만큼의 권한 부여.
  - **지속적 인증**: 한 번 로그인했다고 끝이 아니라, 위치나 기기 상태가 바뀌면 다시 검증.

---

## Ⅱ. 클라우드 네이티브 보안 (SASE & CASB)

### 1. SASE (Secure Access Service Edge)
* **개념**: **네트워크(SD-WAN)**와 **보안(FW, SWG, CASB)** 기능을 클라우드 서비스 하나로 통합하여 제공하는 모델입니다.
* **장점**: 사용자가 어디에 있든(집, 사무실, 카페) 일관된 보안 정책을 적용받습니다.

### 2. CASB (Cloud Access Security Broker)
* **역할**: 기업이 사용하는 다양한 클라우드 서비스(Office 365, Slack 등)와 사용자 사이에서 가시성을 확보하고 데이터를 보호하는 중개자입니다.

---

## Ⅲ. 보안 운영 자동화 (SIEM & SOAR)

### 1. SIEM (Security Information and Event Management)
* **역할**: 수많은 장비에서 쏟아지는 **로그를 한데 모아 분석**하여 위협을 탐지합니다. (빅데이터 분석 중심)

### 2. SOAR (Security Orchestration, Automation and Response)
* **역할**: SIEM이 찾아낸 위협에 대해, 미리 정해진 시나리오(**Playbook**)에 따라 방화벽 차단 등 **대응 조치를 자동으로 수행**합니다. 보안 전문가의 부족 문제를 해결하는 핵심 기술입니다.

---

## Ⅳ. 데이터 유출 방지: DLP (Data Loss Prevention)
* 사내의 기밀 데이터가 외부(이메일, USB, 클라우드)로 유출되는 것을 실시간으로 감시하고 차단하는 기술입니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[현대 보안 아키텍처의 결합]

  [ User ] ───( SASE / Zero Trust )───> [ Cloud Apps ]
      │               │                     │
      └───────( SIEM / SOAR 감시 )──────────┘
```

📢 **섹션 요약 비유**: **제로 트러스트**는 회사 건물의 모든 방문에 도어락을 달고, 복도를 지날 때마다 신분증을 검사하는 철저한 감시망입니다. **SASE**는 무거운 보안 장비를 들고 다닐 필요 없이, 하늘 위(클라우드)에 떠 있는 거대한 투명 보호막 속에서 일하는 것과 같습니다. **SOAR**는 불이 났을 때 사람이 직접 소화기를 들고 뛰기 전, 센서가 화재를 감지하고 즉시 스프링클러를 작동시킨 뒤 119에 신고까지 마치는 '자동 방재 시스템'입니다.
