# 723-732. 최신 보안 위협: APT와 웹 취약점

> **핵심 인사이트**: 현대의 공격은 한 번의 타격으로 끝나지 않는다. 목표를 정해놓고 끈질기게 파고드는 APT 공격과, 사용자가 신뢰하는 웹 사이트를 오염시키는 인젝션 공격은 기술적 결함뿐만 아니라 사람의 심리(사회공학)까지 파고드는 입체적인 위협이다.

---

## Ⅰ. APT (Advanced Persistent Threat)
특정 타겟을 정하고, 목표를 달성할 때까지 장기간에 걸쳐 다양한 수단을 동원하는 **지능형 지속 위협**입니다.

* **공격 단계 (Cyber Kill Chain)**:
  1. **침투**: 스피어 피싱(맞춤형 메일) 등을 통해 내부망 진입.
  2. **탐색**: 내부 구조 파악 및 권한 상승.
  3. **수집/유출**: 중요 데이터를 모아 외부로 전송.
  4. **파괴/은닉**: 흔적을 지우거나 시스템 마비.

---

## Ⅱ. 3대 웹 취약점 (OWASP Top 10)

### 1. SQL Injection (SQL 인젝션)
* **원리**: 입력창에 악의적인 SQL 구문을 넣어 DB 정보를 빼내거나 조작하는 공격.
* **방어**: 입력값 검증, **Prepared Statement** 사용.

### 2. XSS (Cross Site Scripting)
* **원리**: 게시판 등에 악성 스크립트를 삽입하여, 이를 읽는 **다른 사용자의 브라우저**에서 실행되게 함. (쿠키 탈취 등)
* **방어**: 특수문자 치환(HTML Encoding), **CSP** 설정.

### 3. CSRF (Cross Site Request Forgery)
* **원리**: 사용자가 자신의 의지와 무관하게 공격자가 의도한 행위(비번 변경, 송금 등)를 서버에 요청하게 만드는 공격.
* **방어**: **CSRF Token** 검증, SameSite 쿠키 설정.

---

## Ⅲ. 랜섬웨어 (Ransomware)
사용자의 파일을 암호화하여 인질로 잡고 금전(비트코인 등)을 요구하는 악성코드입니다.
* **전파**: 이메일 첨부파일이나 공유 폴더의 취약점(SMBv1 등)을 통해 자가 전파됩니다.
* **대응**: 주기적인 **오프라인 백업**이 유일한 해결책입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[웹 공격의 타겟 차이]

1. SQL 인젝션 : [ 사용자 ] ───(악성 입력)───> [ 서버 DB ] (서버 타격)
2. XSS        : [ 서버 ] ───(악성 스크립트)──> [ 다른 사용자 ] (사용자 타격)
3. CSRF       : [ 공격자 ] ──(사용자 권한 이용)─> [ 서버 ] (사용자 사칭)
```

📢 **섹션 요약 비유**: **APT**는 도둑이 집안 구조를 다 파악하고 냉장고 음식까지 꺼내 먹으며 몇 달간 숨어 지내다 금고를 털어가는 '기생충형 도둑'입니다. **SQL 인젝션**은 식당 주문서에 "음식값은 0원"이라고 적어 넣어 계산기를 속이는 것이고, **XSS**는 게시판에 독이 묻은 편지를 써서 읽는 사람마다 쓰러지게 만드는 것입니다. **CSRF**는 집주인의 인감도장을 몰래 빌려가서 엉뚱한 서류에 도장을 찍게 만드는 대리인 사기입니다.
