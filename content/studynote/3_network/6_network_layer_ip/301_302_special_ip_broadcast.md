# 301-302. 특수 IP 주소와 브로드캐스트

> **핵심 인사이트**: IP 주소는 단순히 단말을 식별하는 용도 외에도, 자동 할당 실패 시의 자구책(APIPA), 자기 자신과의 대화(Loopback), 그리고 네트워크 전체에 소리치는(Broadcast) 특수한 용도로 예약된 대역이 존재한다.

---

## Ⅰ. 링크 로컬 주소와 APIPA
DHCP 서버로부터 IP 주소를 받지 못했을 때, 단말이 스스로 부여하는 임시 주소 체계입니다.

### 1. APIPA (Automatic Private IP Addressing)
* **개념**: DHCP 서버가 고장 났거나 응답이 없을 때, 운영체제가 가용 IP 대역 중 하나를 무작위로 선택해 스스로 설정하는 기능입니다.
* **주소 대역**: **`169.254.0.1` ~ `169.254.255.254`** (B클래스 대역)
* **특징**: 이 주소를 가진 장비끼리는 같은 스위치(허브) 내에서 통신이 가능하지만, 라우터를 넘어 외부 인터넷으로 나갈 수는 없습니다. (링크 로컬 성격)

---

## Ⅱ. 루프백(Loopback) 주소
자기 자신을 가리키는 가상의 인터페이스 주소입니다.

* **주소 대역**: **`127.0.0.0 /8`** (주로 `127.0.0.1` 사용)
* **특징**: 패킷이 랜카드 밖으로 나가지 않고 OS 내부의 네트워크 스택에서 바로 턴(Turn)하여 돌아옵니다.
* **용도**: 
  - 네트워크 설정 테스트
  - 로컬 서버(Localhost) 개발 환경 구축
  - 자기 자신에게 서비스를 요청할 때

---

## Ⅲ. 브로드캐스트 (Broadcast) 주소의 종류
목적지 IP를 "모두"로 설정하여 패킷을 보낼 때, 그 범위에 따라 두 가지로 구분됩니다.

### 1. 리미티드 브로드캐스트 (Limited Broadcast)
* **주소**: **`255.255.255.255`**
* **범위**: 현재 내가 속한 **로컬 네트워크 세그먼트 전체**입니다.
* **특징**: 라우터를 절대 통과할 수 없습니다. (라우터가 이 주소를 보면 즉시 드랍합니다.) 
* **용도**: 내가 아직 내 IP가 없을 때(DHCP Discover 등) 주변에 도움을 요청하는 용도로 씁니다.

### 2. 디렉티드 브로드캐스트 (Directed Broadcast)
* **주소**: 특정 네트워크 ID 뒤에 호스트 비트를 모두 1로 채운 주소입니다. (예: `192.168.1.255`)
* **범위**: **특정 원격 네트워크 내의 모든 단말**입니다.
* **특징**: 이론상 다른 네트워크에서 라우터를 거쳐 해당 네트워크까지 전달될 수 있습니다. 
* **보안 문제**: 외부에서 내부망 전체를 공격하는 **Smurf Attack** 등에 악용될 수 있어, 대부분의 최신 라우터는 이 패킷의 통과를 기본적으로 막아둡니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[특수 IP 주소 상황별 사용]

상황 1: 인터넷 안됨, 공유기 고장남
  -> [ PC ] 스스로 "169.254.x.x" 할당 (APIPA)

상황 2: 내 컴퓨터에 웹 서버 띄우고 접속 테스트
  -> 브라우저에 "127.0.0.1" 입력 (Loopback)

상황 3: 나 아직 IP 없어! 주변에 DHCP 서버 있니?
  -> 목적지 "255.255.255.255"로 쏨 (Limited Broadcast)

상황 4: (서울에서) 부산 지사 네트워크의 모든 컴퓨터를 깨우고 싶어!
  -> 목적지 "부산_네트워크.255"로 쏨 (Directed Broadcast)
```

📢 **섹션 요약 비유**: **APIPA**는 통신사 기지국이 고장 났을 때 우리끼리만 무전기로 대화하는 임시 번호입니다. **루프백**은 거울을 보고 나 자신과 가위바위보를 하는 연습용 주소입니다. **리미티드 브로드캐스트**는 지금 내가 서 있는 교실 안에서 "누구 저 좀 도와주세요!"라고 소리 지르는 것이고, **디렉티드 브로드캐스트**는 옆 교실 창문에 대고 "거기 3반 학생들 다 들어!"라고 외치는 것과 같습니다.
