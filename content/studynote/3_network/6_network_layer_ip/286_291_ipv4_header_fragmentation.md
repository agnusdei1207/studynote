# 286-291. IPv4 프로토콜과 헤더 구조 및 단편화

> **핵심 인사이트**: IPv4는 신뢰성보다는 효율성을 추구하는 비연결형 프로토콜이다. 데이터가 너무 크면 길 중간에서 잘게 쪼개져야(단편화) 하는데, 이를 위해 헤더에는 '조각난 패킷'들을 다시 조립하기 위한 식별자와 오프셋 정보가 꼼꼼하게 기록된다.

---

## Ⅰ. IPv4 (Internet Protocol Version 4)의 특징
* **비연결성 (Connectionless)**: 데이터를 보내기 전 상대방과 연결을 맺지 않고 그냥 던집니다.
* **최선 노력 (Best Effort)**: 데이터가 잘 도착했는지 보장하지 않습니다. 에러 처리나 흐름 제어는 상위 계층(TCP)에 맡깁니다.
* **데이터그램 방식**: 각 패킷이 독립적인 경로를 통해 목적지로 이동합니다.

---

## Ⅱ. IPv4 헤더 구조 (20 ~ 60 Bytes)
표준 헤더는 **20바이트** 고정이며, 옵션 추가 시 최대 60바이트까지 늘어납니다.

### 1. 기본 필드 (상위 12바이트)
* **Version (4bit)**: IP 버전을 나타냅니다. (IPv4 이므로 '4')
* **IHL (Header Length, 4bit)**: 헤더의 총 길이를 4바이트 단위로 표시합니다. (기본값 5 = 20바이트)
* **ToS (Type of Service, 8bit)**: 패킷의 우선순위(QoS)를 지정합니다. (현재는 DSCP/ECN으로 세분화됨)
* **Total Length (16bit)**: 헤더와 데이터를 합친 패킷 전체 길이를 바이트 단위로 표시합니다. (최대 65,535바이트)

### 2. 단편화 관련 필드 (중간 8바이트)
* **Identification (식별자, 16bit)**: 하나의 거대한 원본 패킷에서 쪼개져 나온 조각들이 모두 **같은 출신**임을 증명하는 번호입니다.
* **IP Flags (3bit)**:
  - **DF (Don't Fragment)**: '1'이면 쪼개지 마라! (만약 MTU보다 크면 패킷은 폐기됨)
  - **MF (More Fragments)**: '1'이면 뒤에 조각이 더 있다! '0'이면 내가 마지막 조각이다!
* **Fragment Offset (13bit)**: 쪼개진 조각이 원본 패킷의 **어느 지점(위치)**에 해당하는지를 8바이트 단위로 기록합니다.

### 3. 제어 및 주소 필드 (하위 20바이트 이상)
* **TTL (Time to Live)**: 패킷이 살아있을 수 있는 시간(라우터 홉 수)입니다. 라우터를 지날 때마다 1씩 감소하며 0이 되면 패킷은 폐기됩니다. (무한 루프 방지)
* **Protocol (8bit)**: 상위 계층이 무엇인지 알립니다. (ICMP=1, TCP=6, UDP=17)
* **Header Checksum**: 헤더가 전송 중 깨졌는지 확인합니다.
* **Source / Destination IP Address**: 각각 32비트(4바이트)의 IP 주소가 들어갑니다.

---

## Ⅲ. IP 단편화 (Fragmentation)와 재조립 (Reassembly)

### 1. 단편화가 발생하는 이유
각 물리 네트워크 매체는 한 번에 보낼 수 있는 최대 데이터 크기인 **MTU (Maximum Transmission Unit)**를 가집니다.
* 이더넷(Ethernet) MTU: 1,500바이트
* 송신 측에서 4,000바이트짜리 패킷을 보냈는데, 중간에 MTU가 1,500바이트인 구간을 만나면 라우터는 이를 세 조각으로 쪼개서(단편화) 보내야 합니다.

### 2. 재조립 메커니즘
* **위치**: 중간 라우터들은 쪼개기만 할 뿐, **최종 목적지 호스트(PC)**에 도달해서야 다시 하나로 합쳐집니다.
* **방법**: 수신 측은 `Identification` 번호가 같은 조각들을 모으고, `Fragment Offset` 순서대로 정렬한 뒤, `MF` 비트가 0인 패킷이 올 때까지 기다려 조립을 완성합니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[IPv4 단편화 과정 예시 (MTU 1500인 경우)]

 원본 패킷 (4000 B) [ ID: 123 | Offset: 0 | MF: 0 ]
        │
        ▼ (라우터에서 쪼개짐)
 조각 1 (1500 B) [ ID: 123 | Offset: 0   | MF: 1 ]
 조각 2 (1500 B) [ ID: 123 | Offset: 185 | MF: 1 ] (1480/8 = 185)
 조각 3 (1040 B) [ ID: 123 | Offset: 370 | MF: 0 ]
```

📢 **섹션 요약 비유**: IPv4 패킷은 1,500kg까지만 실을 수 있는 좁은 골목길(이더넷)을 통과해야 하는 거대한 이삿짐 트럭입니다. 짐이 너무 많으면(MTU 초과), 짐을 작은 박스로 여러 개 나누어 담고(단편화), 박스마다 "전체 짐 중 몇 번째 박스(Offset)"인지 적어둡니다. 최종 도착지에 이 박스들이 모두 도착하면 주인이 순서대로 뜯어 다시 이삿짐을 완성(재조립)합니다.
