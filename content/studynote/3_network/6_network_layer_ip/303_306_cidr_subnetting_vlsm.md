# 303-306. 클래스리스(Classless) 라우팅과 CIDR/VLSM

> **핵심 인사이트**: 과거의 클래스 기반 방식은 네트워크를 무조건 '통'으로 나눠줘서 주소 낭비가 심했다. 이를 해결하기 위해 비트 단위로 주소를 쪼개고 합치는 CIDR와 VLSM 기술이 등장하여 인터넷 수명을 획기적으로 연장했다.

---

## Ⅰ. CIDR (Classless Inter-Domain Routing)
주소 할당 시 A, B, C 클래스라는 고정된 경계를 무시하고, **비트 단위로 네트워크 영역을 지정**하는 방식입니다.

* **슬래시(/) 표기법**: IP 주소 뒤에 `/24` 처럼 네트워크 비트의 개수를 직접 적습니다.
  - `192.168.1.0/24`: 앞에서부터 24비트가 네트워크 영역이라는 뜻입니다.
* **장점**: 클래스에 얽매이지 않고 필요한 만큼만(예: 500개 주소면 /23) 유연하게 할당할 수 있어 IP 주소 고갈 문제를 늦추는 데 일등 공신이 되었습니다.

---

## Ⅱ. 서브네팅 (Subnetting)
하나의 커다란 네트워크를 여러 개의 작은 네트워크(Subnet)로 쪼개는 기술입니다.

* **원리**: 호스트(Host) 영역의 비트를 빌려와서 네트워크(Network) 영역으로 확장합니다.
* **서브넷 마스크 (Subnet Mask)**: 어디까지가 네트워크인지 알려주는 필터입니다. IP 주소와 **AND 연산**을 수행하여 네트워크 주소를 추출합니다.
  - 예: `210.10.10.0/24` (C클래스)를 2개로 쪼개면? ➔ `/25` 가 됨.
  - 서브넷 1: `210.10.10.0 ~ 127`
  - 서브넷 2: `210.10.10.128 ~ 255`

---

## Ⅲ. 슈퍼네팅 (Supernetting) / 경로 요약
서브네팅과 반대로, 여러 개의 작은 네트워크를 하나의 큰 네트워크로 합치는 기술입니다.

* **목적 (경로 요약)**: 라우터가 관리해야 할 라우팅 테이블의 크기를 줄이기 위해 사용합니다. (Route Summarization)
  - `192.168.0.0/24`, `192.168.1.0/24`, `192.168.2.0/24`, `192.168.3.0/24` 4개를 합치면?
  - 공통 비트를 묶어 **`192.168.0.0/22`** 로 요약 가능합니다.

---

## Ⅳ. VLSM (Variable Length Subnet Mask)
하나의 네트워크 안에서 **서로 다른 크기의 서브넷 마스크**를 동시에 사용하는 기술입니다.

* **필요성**: 부서마다 인원수가 다른데(인사팀 10명, 영업팀 100명), 모두 똑같은 크기(/24)로 나눠주면 낭비가 생깁니다.
* **동작**: 쪼개진 서브넷을 필요에 따라 한 번 더 쪼개는(Sub-subnetting) 방식입니다.
  - 영업팀: `/25` (128개 주소)
  - 인사팀: `/28` (16개 주소)
  - 라우터 사이 연결: `/30` (2개 주소)

---

## Ⅴ. 개념 맵 및 요약

```ascii
[Subnetting vs Supernetting]

<--- (합침) Supernetting / Summary      Subnetting (쪼갬) --->
[ 10.1.0.0/16 ] <==================== [ 10.1.1.0/24 ]
                                      [ 10.1.2.0/24 ]
                                      [ 10.1.3.0/24 ]

[VLSM의 효율적인 주소 배분]
전체 파이: [ 192.168.1.0/24 ]
  ├── 부서 A: [ 192.168.1.0/25 ] (128개)
  ├── 부서 B: [ 192.168.1.128/26 ] (64개)
  └── 나머지: 한 번 더 쪼개서 소규모 팀 배분
```

📢 **섹션 요약 비유**: **클래스 방식**은 피자를 무조건 1판, 10판, 100판 단위로만 파는 가게입니다. 혼자 먹으려 해도 1판을 다 사야 하니 버리는 게 많았죠. **CIDR/서브네팅**은 피자를 조각(비트) 단위로 주문할 수 있게 바꾼 혁명입니다. **슈퍼네팅**은 여러 조각을 큰 박스 하나에 담아 배달(라우팅 테이블 감소)하는 것이고, **VLSM**은 대식가에게는 큰 조각을, 소식가에게는 작은 조각을 나눠주는 맞춤형 배급 시스템입니다.
