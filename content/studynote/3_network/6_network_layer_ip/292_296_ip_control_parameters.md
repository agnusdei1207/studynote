# 292-296. IP 패킷의 제어 지표 (MTU, TTL, Checksum)

> **핵심 인사이트**: 패킷이 인터넷이라는 거대한 바다를 표류하지 않게 막아주는 수명(TTL)과, 최적의 상자 크기를 찾아 효율을 높이는 기술(PMTU), 그리고 머리표가 훼손되지 않았는지 검사하는 체크섬은 IP 통신의 안정성을 지탱하는 3대 요소이다.

---

## Ⅰ. MTU (Maximum Transmission Unit)
물리적인 네트워크 인터페이스에서 **한 번에 보낼 수 있는 가장 큰 데이터 단위**입니다.

* **이더넷 표준**: 기본적으로 **1,500바이트**를 사용합니다.
* **영향**: MTU보다 큰 패킷이 들어오면 라우터는 이를 여러 개로 쪼개야(단편화) 하므로 부하가 발생합니다.
* **PMTU (Path MTU Discovery)**:
  - 송신지와 수신지 사이의 수많은 라우터 중 **가장 작은 MTU**를 찾아내어, 처음부터 그 크기에 맞춰 패킷을 보내는 기술입니다.
  - **원리**: IP 헤더의 DF(Don't Fragment) 비트를 1로 세팅하고 보냅니다. 중간에 나보다 작은 MTU를 가진 라우터를 만나면 "너무 커서 못 보내!"라는 ICMP 에러를 뱉게 되는데, 이를 통해 경로상의 최소 MTU를 알아냅니다.

---

## Ⅱ. TTL (Time to Live)
패킷이 네트워크상에서 **무한 루프(Loop)를 도는 것을 방지**하기 위한 수명 장치입니다.

* **동작**: 라우터를 하나 통과(1 Hop)할 때마다 TTL 값은 **1씩 감소**합니다.
* **폐기**: TTL이 0이 되면 라우터는 해당 패킷을 즉시 버리고, 송신자에게 `ICMP Time Exceeded` 메시지를 보냅니다.
* **활용**: `tracert` (또는 `traceroute`) 명령어는 TTL을 1부터 하나씩 늘려가며 보내는 원리를 이용하여 목적지까지의 경로를 추적합니다.

---

## Ⅲ. 프로토콜(Protocol) 필드와 체크섬(Checksum)

### 1. 프로토콜 필드 (8bit)
IP 패킷의 등 뒤에 업혀 있는 '진짜 데이터'가 누구인지 알려주는 코드입니다.
* **1**: ICMP (네트워크 제어)
* **6**: TCP (신뢰성 전송)
* **17**: UDP (비연결 전송)

### 2. 헤더 체크섬 (Header Checksum)
전송 중에 IP **헤더(Header)** 부분이 노이즈에 의해 훼손되었는지 확인하는 수학적 검사 도구입니다.
* **범위**: 데이터 부분은 검사하지 않고 오직 **IP 헤더**만 검사합니다. (데이터 무결성은 상위 계층인 TCP/UDP가 책임집니다.)
* **특징**: 라우터를 지날 때마다 TTL 값이 바뀌므로, 체크섬 값도 **매 홉마다 라우터가 다시 계산**해서 갱신해야 합니다. (라우터의 부하를 유발하는 원인 중 하나가 되어 IPv6에서는 이 필드가 아예 삭제되었습니다.)

---

## Ⅳ. 개념 맵 및 요약

```ascii
[IP 패킷의 주요 제어 요소]

  [ 송신자 ] ───────────────── (Path MTU: 1400) ─────────────────> [ 수신자 ]
      │                                                               ▲
      │ (1) PMTU 발견: "아, 이 경로의 병목은 1400이구나. 1400으로 쏘자!"      │
      │                                                               │
      │ (2) TTL 설정: "수명은 64로 줄게. 라우터 64개 넘으면 소멸해!"           │
      │                                                               │
      └─────────────────── [ 라우터 (Hop) ] ──────────────────────────┘
                           - TTL 1 감소
                           - Checksum 재계산
```

📢 **섹션 요약 비유**: **MTU**는 택배 트럭이 지나갈 수 있는 '터널의 높이'입니다. **PMTU**는 미리 터널 높이를 조사해 가장 낮은 터널에 맞춰 트럭을 고르는 지혜입니다. **TTL**은 택배 상자에 적힌 '유통기한'입니다. 배달원이 집을 못 찾고 헤매다가 유통기한(TTL=0)이 지나면 상자를 폐기합니다. **체크섬**은 운송장이 찢어져서 주소를 못 읽게 되진 않았는지 확인하는 검사 필증입니다.
