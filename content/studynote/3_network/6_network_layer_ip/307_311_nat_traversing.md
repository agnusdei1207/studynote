# 307-311. 네트워크 주소 변환(NAT)과 횡단 기술

> **핵심 인사이트**: 부족한 공인 IP 문제를 해결하기 위해 내부망에서는 사설 IP를 쓰고, 밖으로 나갈 때만 공인 IP로 변신시키는 기술이 NAT이다. 단순히 주소만 바꾸는 것을 넘어, 포트 번호(PAT)를 이용해 하나의 공인 IP를 수백 명이 나눠 쓰기도 한다.

---

## Ⅰ. NAT (Network Address Translation) 개요
IP 패킷의 헤더에 있는 IP 주소를 다른 주소로 변환하여 전송하는 기술입니다. 주로 기업 내부망이나 가정용 공유기에서 사용됩니다.

* **도입 배경**: IPv4 주소 부족 문제 해결, 내부 네트워크 주소 은닉을 통한 보안 강화.
* **동작 위치**: 라우터나 방화벽과 같은 네트워크 경계 장비에서 수행됩니다.

---

## Ⅱ. NAT의 종류

### 1. Static NAT (정적 NAT, 1:1)
* **특징**: 내부 사설 IP 하나와 외부 공인 IP 하나를 **고정적으로 1:1 매핑**합니다.
* **용도**: 내부망에 있는 웹 서버나 메일 서버를 외부 인터넷에 공개해야 할 때 사용합니다.

### 2. Dynamic NAT (동적 NAT, M:N)
* **특징**: 여러 개의 공인 IP 주소 풀(Pool)을 만들어두고, 내부 사용자가 밖으로 나갈 때 **남는 공인 IP를 임시로 할당**받아 나가는 방식입니다.

### 3. PAT (Port Address Translation / NAPT, 1:N)
* **특징**: **가장 많이 쓰이는 방식**입니다. **단 하나의 공인 IP**를 수많은 내부 사용자가 공유합니다. 
* **원리**: 각 사용자를 구분하기 위해 IP 주소뿐만 아니라 **포트(Port) 번호**를 다르게 할당하여 NAT 테이블을 관리합니다. (공유기의 핵심 원리)

---

## Ⅲ. NAT 횡단 (Traversing) 기술
NAT 내부에 있는 두 단말(예: 두 집의 PC)이 서버를 거치지 않고 서로 직접 통신(P2P)하려 할 때, NAT 방화벽에 막히는 문제를 해결하는 기술입니다.

### 1. ALG (Application Layer Gateway)
* **역할**: FTP처럼 패킷 본문(Payload) 안에 자신의 IP/포트 정보를 담아 보내는 특수 프로토콜들을 위해, 라우터가 본문 내용까지 열어서 주소를 바꿔주는 지능형 기능입니다.

### 2. STUN / TURN / ICE (VoIP, WebRTC 필수 기술)
* **STUN**: 외부 서버를 통해 "내 공인 IP와 포트가 뭐지?"라고 물어보고 알아내는 방식입니다.
* **TURN**: NAT 보안이 너무 강력해 직접 통신이 불가능할 때, 외부 중계 서버를 통해 데이터를 전달(Relay)하는 방식입니다.
* **ICE**: STUN과 TURN 방식을 모두 시도하여 최적의 경로를 자동으로 찾아내는 프레임워크입니다.

### 3. Hairpin NAT (NAT Loopback)
* 내부 사용자가 자신의 공인 IP 주소를 통해 내부 서버에 접속하려고 할 때, 라우터가 이를 감지하여 외부로 나가지 않고 다시 내부로 돌려보내 주는 기능입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[PAT (Port Address Translation) 동작 예시]

[ 사설망 (Inside) ]             [ NAT 라우터 (공유기) ]             [ 인터넷 (Outside) ]
  PC 1 : 192.168.1.10:5000  ──┐                                ┌──> Target Server
  PC 2 : 192.168.1.20:5000  ──┼──  Public IP: 1.1.1.1  ───>  Target Server
                              │
[ NAT Table ]                 │
Inside IP : Inside Port  <==> Outside IP : Outside Port
192.168.1.10 : 5000      <==> 1.1.1.1    : 10001
192.168.1.20 : 5000      <==> 1.1.1.1    : 10002
```

📢 **섹션 요약 비유**: **Static NAT**는 회사 직통 전화번호를 가진 사장님 자리입니다. **PAT(공유기)**는 회사 대표 번호(공인 IP) 하나를 두고, 각 자리마다 내선 번호(포트 번호)를 부여해 수백 명이 전화를 쓰는 시스템입니다. 외부에서 전화를 걸면 안내원(NAT 테이블)이 내선 번호를 보고 담당자에게 연결해 줍니다. **STUN/TURN**은 철통 보안 아파트(NAT)에 사는 두 친구가 서로의 정확한 주소를 몰라 경비실이나 제3의 장소에서 만남을 주선받는 것과 같습니다.
