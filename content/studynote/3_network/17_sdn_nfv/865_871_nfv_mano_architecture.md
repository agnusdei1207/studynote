# 865-871. 네트워크 기능 가상화(NFV)와 MANO

> **핵심 인사이트**: 전용 하드웨어(방화벽, 라우터)의 시대가 가고, 범용 서버에 설치하는 '앱'의 시대가 왔다. NFV는 네트워크 장비를 소프트웨어(VNF)로 바꿔버렸으며, 이 수많은 가상 장비들의 탄생과 소멸을 지휘하는 마에스트로가 바로 MANO다.

---

## Ⅰ. NFV (Network Functions Virtualization)
고가의 전용 네트워크 장비(라우터, IPS, 가속기 등)를 표준화된 범용 서버나 스토리지, 스위치 위에서 **소프트웨어 형태**로 동작시키는 기술입니다.

* **가치**: 장비 도입 비용(CAPEX)과 운영 비용(OPEX) 절감, 서비스 출시 속도 향상.
* **VNF (Virtual Network Function)**: 소프트웨어로 구현된 개별 네트워크 기능. (예: 가상 방화벽, 가상 EPC)

---

## Ⅱ. NFV 표준 아키텍처 (ETSI)
1. **NFVI (NFV Infrastructure)**: VNF가 돌아가는 물리적/가상적 기초 환경. (하드웨어 + 하이퍼바이저)
2. **VNF**: 가상화된 네트워크 기능 소프트웨어 그 자체.
3. **NFV MANO (Management and Orchestration)**: 관리를 담당하는 핵심 레이어.

---

## Ⅲ. MANO의 3대 구성 요소
가상 네트워크 자원을 유기적으로 관리하기 위한 컨트롤 센터입니다.

1. **NFVO (NFV Orchestrator)**: 전체적인 네트워크 서비스의 설계와 자원 할당을 총괄하는 **최종 지휘자**.
2. **VNFM (VNF Manager)**: 개별 VNF의 라이프사이클(생성, 수정, 삭제, 에러 복구)을 담당하는 **중간 관리자**.
3. **VIM (Virtualized Infrastructure Manager)**: 물리 하드웨어와 가상화 자원을 직접 제어하는 **현장 소장**. (예: OpenStack, Kubernetes)

---

## Ⅳ. 서비스 체이닝 (Service Chaining / SFC)
사용자의 트래픽이 여러 개의 가상 장비(VNF)를 **순서대로 통과**하도록 엮어주는 기술입니다.
* 예: [패킷 유입] $\rightarrow$ [가상 방화벽] $\rightarrow$ [가상 IPS] $\rightarrow$ [가상 로드밸런서] $\rightarrow$ [목적지].

---

## Ⅴ. 개념 맵 및 요약

```ascii
[NFV MANO 관리 체계]

  [ NFVO ] (자원 지휘) ───┐
      │                  │
  [ VNFM ] (VNF 개별 관리) ├─────> [ VNF (가상장비) ]
      │                  │             │
  [  VIM ] (인프라 제어)  ───┘     [ NFVI (기반환경) ]
```

📢 **섹션 요약 비유**: **NFV**는 거실에 TV, 오디오, 게임기를 따로 두던 방식에서, 컴퓨터 한 대에 '프로그램'으로 깔아 쓰는 것과 같습니다. **VNF**는 그 각각의 프로그램(게임, 영화 플레이어)이고, **MANO**는 이 프로그램들을 실행하고 끄며 메모리를 나눠주는 '운영체제(OS)' 역할을 합니다. **서비스 체이닝**은 공장에서 물건이 컨베이어 벨트를 따라 조립, 검사, 포장 단계를 차례로 거치는 공정과 같습니다.
