# 872-879. 가상화 심화: P4와 텔레메트리 (P4, NETCONF/YANG)

> **핵심 인사이트**: 네트워크는 이제 '하드웨어의 한계'를 완전히 넘어섰다. P4는 칩셋의 패킷 처리 방식까지 직접 코딩하게 해 주며, 텔레메트리는 아주 미세한 성능 변화까지 실시간 스트리밍으로 쏟아낸다. 이제 네트워크는 거대한 하나의 '살아있는 소프트웨어'다.

---

## Ⅰ. P4 (Programming Protocol-independent Packet Processors)
하드웨어(ASIC)가 패킷을 어떻게 처리할지 정의하는 **네트워크 전용 프로그래밍 언어**입니다.

* **차별점**: 기존 SDN(OpenFlow)은 정해진 규칙(Match-Action)만 쓸 수 있었으나, P4는 개발자가 **새로운 프로토콜이나 헤더 구조를 직접 정의**할 수 있습니다.
* **유연성**: 칩셋을 새로 설계하지 않고도 코드만 컴파일해서 올리면 커스텀 스위치를 만들 수 있습니다.

---

## Ⅱ. 네트워크 설정 자동화: NETCONF와 YANG
장비 설정을 텍스트 기반으로 자동화하기 위한 표준 규격입니다.

1. **NETCONF**: 장비와 대화하기 위한 '통신 프로토콜'입니다. (SSH 위에서 XML/JSON으로 통신)
2. **YANG**: 장비가 가진 데이터(설정값, 통계 등)를 어떻게 표기할지 정의한 '데이터 모델링 언어'입니다.
* **가치**: 제조사마다 제각각이었던 명령어(CLI)를 하나로 통일하여 자동화(IaC)를 실현합니다.

---

## Ⅲ. 스트리밍 텔레메트리 (Streaming Telemetry)
네트워크 상태를 실시간으로 중앙 서버에 '스트리밍'하는 차세대 모니터링 기술입니다.

* **전통 방식 (SNMP)**: 매니저가 주기적으로 묻고 답을 기다림. (Pull 방식, 느림)
* **텔레메트리 방식 (Push)**: 장비가 상태 변화가 생길 때마다 즉시 데이터를 쏴줌. (gRPC 사용)
* **장점**: 마이크로초 단위의 정밀한 트래픽 분석이 가능하여 장애 예측에 탁월합니다.

---

## Ⅳ. SONiC (Software for Open Networking in the Cloud)
마이크로소프트가 주도하는 오픈 소스 네트워크 운영체제(NOS)입니다.

* **특징**: 컨테이너 기반으로 설계되어 특정 기능을 업데이트할 때 시스템 전체를 끄지 않아도 됩니다. (데이터센터 스위치의 표준으로 부상 중)

---

## Ⅴ. 개념 맵 및 요약

```ascii
[차세대 네트워크 프로그래밍 모델]

  [ 개발자 ] ───( P4 코딩 )───> [ P4 Compiler ] ───> [ ASIC / SmartNIC ]
                                                     (패킷 처리 로직 구현)

  [  NMS  ] <──( Telemetry )── [ Network Device ] <──( NETCONF/YANG )── [ IaC 코드 ]
   (감시)       (실시간 스트림)      (장비)             (자동 설정)
```

📢 **섹션 요약 비유**: **P4**는 오븐의 온도 조절만 하는 게 아니라, 오븐의 열선 구조 자체를 내가 원하는 요리에 맞춰 개조하는 것과 같습니다. **NETCONF/YANG**은 장비마다 다른 사투리(CLI)를 표준어(XML)로 통일하여 대화하는 비즈니스 통번역 서비스입니다. **텔레메트리**는 건강검진 때만 혈압을 재는 게 아니라, 24시간 심박수와 혈압을 병원 서버로 실시간 전송하는 '스마트 워치'와 같습니다.
