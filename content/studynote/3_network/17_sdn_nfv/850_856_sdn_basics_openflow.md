# 850-856. 소프트웨어 정의 네트워킹(SDN)의 이해

> **핵심 인사이트**: 전통적인 네트워크는 장비 하나하나가 각자의 두뇌(제어)를 가지고 움직이는 파편화된 구조였다. SDN은 이 두뇌들을 뽑아내어 '중앙 컨트롤러'로 합치고, 실제 장비는 시키는 대로만 움직이는 '단순 통로'로 만듦으로써 네트워크 전체를 마치 하나의 소프트웨어처럼 프로그래밍할 수 있게 했다.

---

## Ⅰ. SDN (Software Defined Networking)의 핵심 원리
네트워크 장비의 수직적 구조를 수평적으로 완전히 분리하는 혁신적인 아키텍처입니다.

1. **Control Plane (제어 평면)**: 네트워크의 전체 경로를 결정하고 정책을 수립하는 **두뇌**. (중앙 집중식 소프트웨어 컨트롤러)
2. **Data Plane (데이터 평면)**: 컨트롤러가 시키는 대로 패킷을 전달만 하는 **팔다리**. (단순한 스위칭 하드웨어)

---

## Ⅱ. SDN의 인터페이스 (API)
중앙 컨트롤러를 중심으로 위아래로 대화하는 통로입니다.

1. **Southbound API (사우스바운드)**: 컨트롤러와 하단 스위치 간의 대화 통로. 대표적으로 **오픈플로우(OpenFlow)**가 있습니다.
2. **Northbound API (노스바운드)**: 컨트롤러와 상위 비즈니스 앱(보안, 모니터링 등) 간의 대화 통로. 주로 RESTful API를 사용하여 네트워크를 프로그래밍합니다.

---

## Ⅲ. 오픈플로우 (OpenFlow)
SDN의 첫 번째이자 가장 대표적인 사우스바운드 표준 프로토콜입니다.

* **Flow Table**: 스위치 내부에 저장되는 규칙 장부입니다.
  - **Match Fields**: 어떤 패킷인가? (목적지 IP, 포트 등 대조)
  - **Actions**: 무엇을 할 것인가? (전달, 드랍, 헤더 수정 등)
  - **Statistics**: 얼마나 지나갔나? (패킷 통계 수집)

---

## Ⅳ. SDN의 장점
* **중앙 집중 관리**: 수만 대의 장비를 한곳에서 통제 가능.
* **유연성 및 민첩성**: 하드웨어 교체 없이 소프트웨어 업데이트만으로 새로운 기능을 즉시 적용.
* **벤더 종속성 탈피**: 표준 프로토콜(OpenFlow)만 지원하면 어떤 회사의 저렴한 스위치도 사용 가능.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[SDN 아키텍처 구조]

  [ Application Layer ] (보안, 모니터링, 관리 앱)
           │
  ( Northbound API )
           │
  [ SDN Controller ] (중앙 집중식 두뇌 / Control Plane)
           │
  ( Southbound API - OpenFlow )
           │
  ┌────────┼────────┐
  ▼        ▼        ▼
[Switch] [Switch] [Switch] (단순 전달 장치 / Data Plane)
```

📢 **섹션 요약 비유**: **전통적 네트워크**는 모든 사원이 자기 마음대로 판단하고 일하는 '각자도생' 사무실입니다. **SDN**은 명석한 '총괄 본부장(컨트롤러)' 한 명이 모든 지시를 내리고, 사원들은 매뉴얼(Flow Table)에 적힌 대로만 움직이는 '일사불란한 체계'입니다. 본부장의 지시 사항이 바뀌면(소프트웨어 변경) 사무실 전체의 업무 방식이 1초 만에 바뀌는 혁명적인 시스템입니다.
